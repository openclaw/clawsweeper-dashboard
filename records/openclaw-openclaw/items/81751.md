## Summary

Current main and the latest release still use exact top-level argv parsing for the gateway service audit, so a `/bin/zsh -lc "... gateway --port ..."` LaunchAgent payload can still produce the reported false `gateway-command-missing` warning. This issue should stay open because the linked open fix PR owns the narrow implementation path and uses closing syntax, so the issue should close only after that PR or an equivalent patch lands.

## What This Changes

`gateway status` reports a misleading missing-gateway-subcommand service-audit warning for macOS LaunchAgent commands where `gateway --port` is inside a shell wrapper.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81778, or an equivalent narrow service-audit parser/test fix, then close this issue after merge.

## Reproduction Assessment

Yes. A high-confidence source reproduction is clear from current main: pass ProgramArguments shaped as `/bin/zsh`, `-lc`, and an inline `... gateway --port 18890` command into `auditGatewayServiceConfig`; the current parser only accepts exact top-level `gateway` and `--port` tokens.

## Solution Assessment

Yes. The linked PR is the narrowest maintainable fix path because it changes the service-audit parser boundary and adds regression coverage without changing LaunchAgent generation or gateway runtime behavior.
