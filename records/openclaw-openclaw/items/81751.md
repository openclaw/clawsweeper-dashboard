## What This Changes

The issue reports that `gateway status` flags shell-wrapped macOS LaunchAgent commands as missing the `gateway` subcommand even when the wrapper executes `openclaw gateway --port`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81778 or an equivalent narrow service-audit tokenization fix, then close this issue after the merge reaches main.

## Reproduction Assessment

Yes. Current main reads LaunchAgent `ProgramArguments` into the status audit and only checks exact argv elements for `gateway`, so the reported `/bin/zsh -lc ... gateway --port ...` wrapper is source-reproducible as a false `gateway-command-missing` warning.

## Solution Assessment

Yes. A narrow service-audit command-token normalization fix with regression tests for the shell wrapper and near matches is the maintainable solution; the active linked PR is already taking that path.
