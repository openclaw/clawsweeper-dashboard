## What This Changes

The issue asks `gateway status` to recognize shell-wrapped macOS LaunchAgent commands that execute `openclaw gateway --port <port>` instead of reporting the gateway subcommand as missing.

## Best Possible Solution

Land the narrower proof-backed service-audit fix in https://github.com/openclaw/openclaw/pull/81778 or an equivalent tokenized implementation, then close this issue and supersede the weaker overlapping PR if needed.

## Reproduction Assessment

Yes from source inspection. A `/bin/zsh -lc` LaunchAgent wrapper keeps `gateway --port` inside one shell payload string, while current main only accepts a top-level argv element exactly equal to `gateway`.

## Solution Assessment

Yes. The best fix is a narrow service-audit command normalization/tokenization change with regression tests; the open proof-backed PR is the current best implementation path.
