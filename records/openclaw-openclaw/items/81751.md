## Summary

Keep open: current main still has the source-reproducible false positive, and https://github.com/openclaw/openclaw/pull/81778 is an open focused fix linked with GitHub closing syntax, so closing this issue before that PR lands would be premature.

## What This Changes

The issue reports that `openclaw --profile enterprise gateway status` falsely reports a missing gateway subcommand for a shell-wrapped macOS LaunchAgent command that runs `openclaw ... gateway --port`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81778 or an equivalent narrow service-audit tokenization fix, then close this issue after the fix is on main.

## Reproduction Assessment

Yes. Current main reads LaunchAgent `ProgramArguments` into the service audit and only checks exact argv elements for `gateway`, so a `/bin/zsh -lc` payload containing `gateway --port` inside one string is source-reproducible as a false `gateway-command-missing` warning.

## Solution Assessment

Yes. A narrow service-audit command-token expansion before the existing `gateway` and `--port` checks is the maintainable fix, and the active linked PR is already scoped to that path with targeted coverage.
