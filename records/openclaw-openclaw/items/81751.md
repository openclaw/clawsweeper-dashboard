## What This Changes

The issue asks gateway status service audit to recognize shell-wrapped macOS LaunchAgent commands that execute `openclaw gateway --port` instead of warning that the gateway subcommand is missing.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/81778 or an equivalent narrow service-audit tokenization fix with wrapper and near-match regression coverage, then close this issue after merge.

## Reproduction Assessment

Yes. Current main passes LaunchAgent `ProgramArguments` into the audit unchanged and only accepts an exact `gateway` argv element, so a `/bin/zsh -lc` payload containing `gateway --port` is source-reproducible as `gateway-command-missing`.

## Solution Assessment

Yes. The narrow maintainable fix is to expand/tokenize shell wrapper command fragments for the existing subcommand and port checks, with regression coverage for the reported wrapper and near-match negatives.
