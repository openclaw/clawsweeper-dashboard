## What This Changes

Fix gateway status service audit so shell-wrapped macOS LaunchAgent commands that execute `openclaw ... gateway --port` are recognized as gateway commands.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81778 or an equivalent narrow service-audit fix, then close this issue after merge.

## Reproduction Assessment

Yes. Current main reads LaunchAgent `ProgramArguments` into the audit unchanged, and `hasGatewaySubcommand` only checks exact argv elements, so the `/bin/zsh -lc ... gateway --port ...` wrapper is source-reproducible as a false `gateway-command-missing` warning.

## Solution Assessment

Yes. A narrow command-token normalization fix in `src/daemon/service-audit.ts`, with regression tests for the reported wrapper and near matches, is the best maintainable path.
