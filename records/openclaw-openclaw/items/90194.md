## Summary

Keep open: the revised PR fixes the earlier live-job metadata-loss risk by preserving owned base rows, but the ownership helper now protects `cron:<jobId>` base rows for every live job, including main-target jobs that do not own that base key.

## What This Changes

The branch extends cron session retention sweeping to prune stale base cron keys only when no live job owns them, wires agent-aware ownership keys through the cron timer, and adds reaper regression tests.

## Best Possible Solution

Narrow the protected-key set to cron base rows that current jobs actually own, then keep the guarded orphan-only reaper path with regression coverage for target changes and deleted-job cleanup.

## Reproduction Assessment

Yes. Source inspection shows current main only prunes `:run:` cron keys while isolated cron jobs write stable base keys; the remaining PR defect is source-reproducible from target-change and main-target ownership paths.

## Solution Assessment

No, not yet. The orphan-only design is the right shape, but the current helper over-approximates ownership and can preserve base rows for jobs that no longer write them.
