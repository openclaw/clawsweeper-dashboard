## Summary

Keep open; the patch fixes the reported source path but is not merge-ready because the widened retention predicate can sweep documented persistent custom cron sessions and archive their transcripts.

## What This Changes

The PR changes the cron session reaper to evaluate all cron-prefixed session keys against retention and adds focused tests for stale base cron sessions and transcript archival.

## Best Possible Solution

Keep the isolated base-session retention fix, but make the sweep operate only on generated cron-owned keys or explicitly approved cleanup targets while preserving documented persistent `session:<id>` history.

## Reproduction Assessment

Yes. Source inspection shows current main gates `sweepCronRunSessions` on `isCronRunSessionKey`, while isolated cron base keys lack `:run:` and therefore skip retention.

## Solution Assessment

No. The reaper is the right owner path, but using `isCronSessionKey` is too broad because documented persistent custom sessions can normalize to the same cron-prefixed key shape.
