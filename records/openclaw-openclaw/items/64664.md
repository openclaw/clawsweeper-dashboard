## Summary

Keep open. Current main and v2026.6.6 still keep pending approvals in process-local managers and the `/approve` path can still surface the raw stale-id text; the Telegram retry-wedge symptom is fixed, and an open gateway-taxonomy PR now covers part of the UX but is not merged and still has consumer-migration blockers.

## What This Changes

The issue asks OpenClaw to handle pending approval IDs lost across gateway restarts by preserving them or giving actionable stale-approval guidance in Telegram and `/approve` flows.

## Best Possible Solution

Finish the stale-approval terminal UX path through a reviewed gateway taxonomy and channel consumer migration, then make an explicit security/product decision before any durable approval persistence is added.

## Reproduction Assessment

Yes from source inspection, not from a live restart run in this read-only review. A pending approval lives in an in-memory manager, a gateway restart creates fresh managers, and current `/approve` tests still expect the raw stale-id text.

## Solution Assessment

Unclear as originally proposed. A narrow stale-terminal UX fix is appropriate, but durable approval persistence is not automatically the best fix without a maintainer security decision and replay-safe binding design.
