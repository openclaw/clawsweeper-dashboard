## Summary

Keep open. Current main still recreates approval managers in memory on gateway startup, and both the generic `/approve` path and Telegram inline callback path still expose or rethrow the raw not-found failure instead of actionable expiry/restart guidance; related source PRs #67982 and #64406 are closed unmerged.

## What This Changes

The issue requests that stale approval buttons after a gateway restart either survive or show clear expiry/restart guidance instead of a generic failed approval error.

## Best Possible Solution

Map approval-not-found failures to clear re-request guidance in `/approve` and Telegram callback handling, with focused regression tests, while leaving durable approval persistence for a separate maintainer design decision.

## Reproduction Assessment

Yes. The static reproduction is high-confidence: current code stores pending approvals in fresh process-local maps, and existing tests already mock `unknown or expired approval id` for `/approve` and Telegram callback paths that still expect raw failure behavior.

## Solution Assessment

Yes for the immediate bug. Translating the existing approval-not-found classifier at the user-facing surfaces is the narrowest maintainable fix; persisting approvals across restarts is broader than needed for the confusing stale-button error.

## Repair Work Prompt

Fix openclaw/openclaw#64664 with a focused UX repair only. Current main loses pending approval IDs on gateway restart because approval managers are process-local; do not add persistence or alter approval lifecycle/auth/protocol semantics. Instead, translate approval-not-found failures into actionable guidance in both user-facing surfaces: `src/auto-reply/reply/commands-approve.ts` for `/approve` replies and `extensions/telegram/src/bot-handlers.runtime.ts` for Telegram inline approval callbacks. Use the existing `isApprovalNotFoundError` classifier where appropriate. The message should say the approval request is no longer available, mention expiry or gateway restart as likely causes, and tell the user to re-request the action; keep manual-ID wording accurate. Add/tighten regression tests in `src/auto-reply/reply/commands-approve.test.ts` and `extensions/telegram/src/bot.test.ts`. Preserve legacy exec-to-plugin fallback behavior. Add a user-facing `CHANGELOG.md` fix entry unless maintainers classify this as internal-only. Source context: closed unmerged PRs #67982 and #64406 contain useful prior work but should not be treated as merged.
