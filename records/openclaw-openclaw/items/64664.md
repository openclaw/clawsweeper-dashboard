## Summary

Keep open. Current main still drops pending approval IDs on gateway restart and still surfaces the stale not-found path through raw `/approve` failure text and retryable Telegram callback errors; related PRs are closed unmerged or only cover part of the issue.

## What This Changes

The issue asks for stale approval callbacks after gateway restart to give clear recovery guidance, and optionally for pending approvals to survive restart.

## Best Possible Solution

Implement a narrow UX fix that maps approval-not-found failures to actionable re-request guidance in both `/approve` replies and Telegram inline callbacks, leaving durable approval persistence for a separate design.

## Reproduction Assessment

Yes, at source level. A restart creates fresh in-memory approval managers, and submitting a stale ID reaches current `/approve` and Telegram callback paths that still surface or rethrow the approval-not-found sentinel.

## Solution Assessment

Yes for the immediate UX bug. Translating the existing approval-not-found classifier at the user-facing handlers is narrower and safer than adding disk-backed approval persistence in the same change.
