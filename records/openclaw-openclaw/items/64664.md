## Summary

Keep open: current main and v2026.6.10 still create fresh process-local exec/plugin approval managers, so gateway restart loses pending approval IDs; Telegram retry-loop cleanup has landed, but the remaining restart-persistence versus explicit-invalidation behavior needs maintainer security/product direction.

## What This Changes

The issue asks for restart-stale approval IDs to survive or fail with clearer user guidance instead of confusing Telegram and approval errors.

## Best Possible Solution

Choose an explicit security/product direction: either fail restart-stale approvals closed with clear operator-facing copy across all approval surfaces, or persist pending approvals in SQLite with expiry, requester/approver binding, replay limits, and focused restart coverage.

Unclear. Persistence is plausible, but the best fix needs maintainer/security choice between explicit restart invalidation UX and durable SQLite-backed pending approval replay.
