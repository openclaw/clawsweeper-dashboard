Keep open. Current main fixed the Telegram retry wedge and now attaches structured remediation to stale approval lookup failures, but approval managers are still process-local and the generic /approve path still exposes the raw stale-ID error.

## What This Changes

The issue asks for pending approvals to survive gateway restarts or, at minimum, for stale Telegram approval buttons and /approve retries to give clear recovery guidance.

## Best Possible Solution

Keep the terminal Telegram callback behavior already on main, add actionable stale-ID recovery copy for /approve, and treat durable approval persistence as a separate secure storage/lifecycle decision.

## Reproduction Assessment

Yes, at source level. Pending approval IDs are stored only in fresh in-memory managers, and current /approve tests still expect the raw stale-ID error after the manager cannot find an approval.

## Solution Assessment

Unclear for the full issue. A narrow recovery-copy fix is maintainable, but preserving approvals across restart changes approval storage and authorization lifecycle semantics and needs maintainer design approval.
