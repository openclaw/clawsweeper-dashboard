## Summary

Keep open: current main and the latest release still keep pending exec/plugin approvals in fresh process-local managers, so a gateway restart loses approval IDs. The merged Telegram fix prevents the retry-loop/wedge symptom, but the remaining approval persistence versus explicit invalidation behavior still needs maintainer product/security direction.

## What This Changes

The issue asks OpenClaw to handle in-flight approvals after gateway restart so stale Telegram approval buttons either still resolve safely or give clear recovery guidance.

## Best Possible Solution

Keep the issue open until maintainers choose the approval lifecycle contract, then implement either SQLite-backed pending approval restoration or a consistent explicit invalidation UX that preserves the exec approval security boundary.

## Reproduction Assessment

Yes. Source inspection shows fresh process-local approval managers backed by private `Map` state, so restarting the gateway necessarily drops pending approval IDs; I did not run a live restart in this read-only review.

## Solution Assessment

Unclear. The narrow stale-message UX fix is a safe interim improvement, but the best complete solution depends on whether maintainers want durable pending approvals or explicit restart invalidation as the supported security model.
