## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Use the current group-policy default:** `src/telegram/settings-panel.ts:49`
  - body: Current Telegram config defaults `groupPolicy` to `allowlist`, but this panel falls back to `open` when the key is unset. On a fresh config the menu would display groups as open, and tapping `open` would no-op, even though current runtime policy is still allowlist.
  - confidence: 0.9
- **[P2] Write streaming.mode instead of replacing streaming:** `src/telegram/settings-panel.ts:33`
  - body: The current Telegram contract uses the nested `streaming.mode` field, with scalar `streaming` treated as legacy. Mapping the button to `channels.telegram.streaming` replaces the whole streaming object, losing nested preview/block/progress settings instead of only changing the mode.
  - confidence: 0.86
- **[P2] Handle the allowlist required by dmPolicy open:** `src/telegram/settings-panel.ts:154`
  - body: The panel offers `DM Policy: open`, but the mutation only writes `dmPolicy`. Current Telegram validation requires `dmPolicy="open"` to be paired with `allowFrom: ["*"]`, so the advertised pairing-to-open workflow fails validation for normal configs instead of applying the change.
  - confidence: 0.82
