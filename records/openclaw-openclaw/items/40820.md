## Summary

Keep open: the `/settings` UX is plausible, but this PR is not mergeable against current main because it targets stale `src/telegram` paths, conflicts with the nested Telegram streaming contract, and lacks required real Telegram proof for a privileged visible callback UI.

## What This Changes

Adds a Telegram DM `/settings` inline-keyboard panel under `src/telegram` to view or toggle DM policy, group policy, streaming, and allowlist state, with command registry changes and tests.

## Best Possible Solution

Port the panel into `extensions/telegram/src`, reuse plugin-local native command/callback auth and config-mutation seams, preserve nested `streaming.mode` including `progress`, add focused extension tests, and attach redacted real Telegram proof.

## Reproduction Assessment

Yes for the review findings: source inspection shows current main runs Telegram from `extensions/telegram/src` while the PR changes `src/telegram`, writes a scalar `streaming` value, and maps `progress` to `partial`. No live after-fix Telegram proof is attached.

## Solution Assessment

No. A Telegram `/settings` panel could be the right direction, but this branch needs to be ported to the current plugin runtime and aligned with the nested streaming contract before it is a maintainable solution.
