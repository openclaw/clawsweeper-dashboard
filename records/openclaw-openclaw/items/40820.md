## Summary

Keep open: the Telegram settings panel is a plausible feature, but this branch is not merge-ready because it targets the retired `src/telegram` path, writes streaming selections to the stale scalar path, omits the current `progress` streaming mode, and lacks real Telegram proof for a privileged config-write surface.

## What This Changes

The PR adds a Telegram DM `/settings` inline keyboard under `src/telegram`, registers a Telegram-only native command, dispatches `cfg_*` callbacks, mutates Telegram account config values, and adds unit tests.

## Best Possible Solution

Port the panel into `extensions/telegram/src` using the current plugin command, callback, runtime-config, and config-mutation seams; write `streaming.mode` including `progress`; then add plugin-local tests and real Telegram callback/auth proof.

## Reproduction Assessment

Yes. Static current-main inspection is enough: `src/telegram` is gone, active Telegram command/callback code lives under `extensions/telegram/src`, and current Telegram streaming is a nested `streaming.mode` contract with `progress` support.

## Solution Assessment

No. The feature direction is reasonable, but this diff is not the best current solution because it edits a retired path and uses a stale streaming contract instead of the current Telegram plugin/runtime seams.
