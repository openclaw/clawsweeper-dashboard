## Summary

Keep open: the feature is still unique and not implemented on current main, but the branch targets the retired Telegram path and conflicts with current plugin-owned config and access-control contracts, so it cannot merge as-is.

## What This Changes

This PR adds a Telegram DM `/settings` inline-keyboard panel for viewing and changing account-level `dmPolicy`, `groupPolicy`, streaming mode, and allowlist state.

## Best Possible Solution

Port the panel into `extensions/telegram/src`, reuse plugin-local command/callback/config-write contracts, preserve nested `streaming.mode` including `progress`, keep group access fail-closed, add focused extension tests, and attach redacted real Telegram proof.

## Reproduction Assessment

Yes for the review findings: source inspection establishes the retired runtime path and the streaming/group-policy contract mismatches against current main. I did not run a live Telegram session because the branch lacks after-fix proof and still targets the wrong runtime path.

## Solution Assessment

No. A Telegram `/settings` panel may be useful, but this implementation needs to move to the current plugin runtime and align with current streaming and access-control contracts before it is the maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Port the settings handler into the Telegram plugin:** `src/telegram/bot-native-commands.ts:788-792`
  - body: Current main runs Telegram native commands from `extensions/telegram/src`; there is no active `src/telegram` runtime after the plugin migration. Registering `/settings` here leaves the command outside the code path that will run after rebasing.
  - confidence: 0.95
- **[P1] Write the nested streaming mode value:** `src/telegram/settings-panel.ts:30-34`
  - body: This maps the stream setting to `streaming` and writes a string there. Current config stores the canonical mode at `streaming.mode` with sibling `preview`, `progress`, and `block` settings, so a panel save would replace the object and discard nested settings.
  - confidence: 0.92
- **[P1] Keep the group-policy default fail-closed:** `src/telegram/settings-buttons.ts:113-115`
  - body: The panel defaults a missing group policy to `open`, but current Telegram config defaults `groupPolicy` to `allowlist`. For fresh or partially resolved configs this displays broad group access instead of the fail-closed default.
  - confidence: 0.88
- **[P2] Preserve the progress streaming mode:** `src/telegram/settings-buttons.ts:27`
  - body: Current Telegram supports `progress`, but the panel omits it from the selectable stream values and normalizes existing `progress` to `partial`. Users with progress mode would see the wrong state and could lose that mode on save.
  - confidence: 0.88
