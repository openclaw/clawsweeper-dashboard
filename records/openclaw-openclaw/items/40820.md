## Summary

Keep open; this is a useful Telegram feature, but the patch is not correct for current main because it targets the obsolete `src/telegram` runtime, writes the legacy scalar streaming key, and still lacks real Telegram proof for a privileged callback/config-write surface.

## What This Changes

The branch adds a Telegram DM `/settings` native command with inline keyboard callbacks and tests for viewing or changing DM policy, group policy, streaming, and allowlist state.

## Best Possible Solution

Implement a smaller Telegram-plugin `/settings` panel that reuses current command authorization and callback seams, writes canonical nested streaming config, and ships with focused tests plus real Telegram proof.

## Reproduction Assessment

Yes for the review findings: source inspection shows current main runs Telegram from `extensions/telegram/src`, while the PR edits `src/telegram`, and the PR writes the legacy scalar streaming key. I did not run live Telegram proof because this external PR lacks an inspectable after-fix artifact.

## Solution Assessment

No. A settings panel is plausible, but this branch should be ported into `extensions/telegram/src`, use the current config-mutation and authorization seams, write `streaming.mode`, and then be proven in a real Telegram session.
