Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Port the settings handler into the Telegram plugin:** `src/telegram/bot-native-commands.ts:788-792`
  - body: Current main runs Telegram native commands from `extensions/telegram/src`; there is no active `src/telegram` runtime. Registering `/settings` here leaves the feature outside the code path that will actually run after rebasing.
  - confidence: 0.95
- **[P1] Write the nested streaming mode value:** `src/telegram/settings-panel.ts:30-34`
  - body: This maps the stream setting to `streaming` and writes a string there. Current config treats scalar `streaming` as legacy and stores the canonical mode at `streaming.mode`, so saving from the panel would replace `preview`, `progress`, and `block` siblings.
  - confidence: 0.92
- **[P1] Keep the group-policy default fail-closed:** `src/telegram/settings-buttons.ts:113-115`
  - body: The panel defaults a missing group policy to `open`, but current Telegram config defaults `groupPolicy` to `allowlist`. For partially resolved or fresh account configs this can show the wrong selected access-control state and steer users toward broad group access.
  - confidence: 0.86
- **[P2] Preserve the progress streaming mode:** `src/telegram/settings-buttons.ts:216-219`
  - body: Current Telegram supports `progress`, but the panel omits it from the selectable stream values and normalizes existing `progress` to `partial`. Users with progress mode would see the wrong state and lose that mode on save.
  - confidence: 0.88
