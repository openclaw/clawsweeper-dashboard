## Summary

Keep open: the feature is useful, but this branch is not correct against current main because it targets stale `src/telegram` runtime files, writes the legacy scalar streaming key, hides the current `progress` streaming mode, and still lacks inspectable real Telegram proof.

## What This Changes

The PR adds a Telegram DM `/settings` inline keyboard for viewing or changing DM policy, group policy, streaming, and allowlist state, plus command registry changes and tests.

## Best Possible Solution

Implement the panel inside `extensions/telegram/src`, reuse the current native-command and callback authorization seams, preserve nested streaming settings including `progress`, and ship focused tests plus real Telegram proof.

## Reproduction Assessment

Yes for the review findings: source inspection shows current main runs Telegram from `extensions/telegram/src`, while the PR edits `src/telegram`, and the PR writes `streaming` instead of nested `streaming.mode`. Live Telegram reproduction is not available because the PR lacks an inspectable after-fix artifact.

## Solution Assessment

No. A `/settings` panel is plausible, but this branch should be ported into the Telegram plugin, use current config-mutation and authorization contracts, preserve nested streaming/progress state, and then be proven in a real Telegram session.
