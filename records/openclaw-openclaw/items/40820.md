## Summary

Keep open: the feature is still useful, but the branch is not mergeable as-is because it targets the removed core Telegram path, conflicts with the current nested streaming contract, and lacks required real Telegram proof for a privileged callback/config-write UI.

## What This Changes

Adds a Telegram DM `/settings` inline keyboard under `src/telegram` with config mutation helpers, callback handling, command registry metadata, and tests.

## Best Possible Solution

Port the panel into `extensions/telegram/src`, reuse plugin-local native command/callback and config-write contracts, preserve nested `streaming.mode` including `progress`, add focused extension tests, and attach redacted real Telegram proof.

## Reproduction Assessment

Yes for the review findings: source inspection shows current main has no active `src/telegram` runtime, while the PR registers handlers there and writes scalar `streaming` while mapping `progress` to `partial`. I did not run a live Telegram session because the branch lacks the required after-fix proof and still targets the wrong runtime path.

## Solution Assessment

No. A Telegram `/settings` panel may be a good feature, but this branch needs to be ported to the current plugin runtime and aligned with the nested streaming contract before it is the maintainable solution.
