## Summary

Keep open: the feature direction is plausible, but this branch is not merge-ready because current main has moved Telegram runtime code into the plugin, the streaming config contract is now nested, and the external PR still lacks real Telegram behavior proof.

## What This Changes

The branch adds a Telegram `/settings` inline keyboard for viewing and changing `dmPolicy`, `groupPolicy`, and streaming settings, plus allowlist display, command-registry filtering, and tests.

## Best Possible Solution

Port the panel into `extensions/telegram/src` using the current plugin command, callback, runtime-config, and config-write seams; write streaming choices to `streaming.mode`; then add plugin-local tests and real Telegram callback/auth proof.

## Reproduction Assessment

Yes. Static current-main inspection is enough: there is no `src/telegram` directory, Telegram command and callback runtime lives under `extensions/telegram/src`, and the current config contract treats streaming mode as `streaming.mode` under a nested object.

## Solution Assessment

No. A Telegram settings panel may be useful, but this branch is not the best implementation for current main because it targets retired paths, writes the wrong streaming shape, and lacks real Telegram proof for a privileged callback surface.
