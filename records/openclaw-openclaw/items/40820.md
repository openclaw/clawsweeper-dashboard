## Summary

Keep open: the feature is still not implemented on current main, but this branch is not mergeable as-is because it targets the retired Telegram path, mismatches current streaming/default-policy contracts, and lacks required real Telegram behavior proof.

## What This Changes

The PR adds a Telegram `/settings` inline-keyboard panel intended to view and change DM policy, group policy, streaming mode, and allowlist state from Telegram DMs.

## Best Possible Solution

Port the panel into `extensions/telegram/src`, use the plugin-local command/callback/config mutation contracts, preserve `streaming.mode` including `progress`, keep `groupPolicy` fail-closed, add focused extension tests, and attach redacted real Telegram proof.

## Reproduction Assessment

I did not run a live Telegram session because the branch lacks after-fix proof and the runtime blockers are visible in source.

## Solution Assessment

No. A Telegram settings panel may be useful, but this implementation needs to move to the current plugin runtime and align with current streaming and access-control contracts before it is maintainable.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Port the settings handler into the Telegram plugin:** `src/auto-reply/commands-registry.data.ts:540-545`
  - body: This registry entry exposes `/settings`, but the dedicated command and callback handlers are added under `src/telegram`, which current main no longer runs. After rebasing, Telegram native commands are served from `extensions/telegram/src`, so this would route through the generic slash-command flow instead of opening the panel.
  - confidence: 0.95
- **[P1] Write the nested streaming mode value:** `src/telegram/settings-panel.ts:30-34`
  - body: The `stream` setting maps to `streaming` and `applySettingChange` writes the selected string at that path. Current config stores streaming as an object with `streaming.mode` plus sibling `preview`, `progress`, and `block` settings, so this would replace the object and discard nested settings.
  - confidence: 0.92
- **[P2] Keep the group-policy default fail-closed:** `src/telegram/settings-buttons.ts:113-115`
  - body: The menu falls back missing `groupPolicy` to `open`, but current Telegram config defaults it to `allowlist`. The settings panel should display and select the fail-closed default, especially because this surface controls who can interact with the bot in groups.
  - confidence: 0.88
- **[P2] Preserve the progress streaming mode:** `src/telegram/settings-buttons.ts:24-28`
  - body: Current Telegram supports `progress`, but the panel omits it from selectable stream values and normalizes existing `progress` to `partial`. Users with progress mode would see the wrong state and could lose that mode on save.
  - confidence: 0.88
