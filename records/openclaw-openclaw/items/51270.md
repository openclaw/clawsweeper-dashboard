## Summary

The TUI problem is still real on current main, so this should stay open, but the current branch is not merge-ready because it bypasses shared model-command context, reduces `/model status` diagnostics, and lacks after-fix TUI proof.

## What This Changes

The branch adds TUI handling and tests for `/model list` and `/model status` so they no longer fall through to session model patching.

## Best Possible Solution

Route TUI `/model list` and `/model status` through shared model-command semantics or equivalent backend methods that preserve active agent, workspace, session, auth context, and full diagnostics.

## Reproduction Assessment

Yes, at source level. Current main parses `/model list` and `/model status` as non-empty `/model` args and sends them to `sessions.patch`, while docs and shared command tests define list/status as command actions.

## Solution Assessment

No. Reserving `list` and `status` is directionally right, but this patch duplicates shared model-command logic and loses active-session catalog and status behavior.
