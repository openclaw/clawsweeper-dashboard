## Summary

Keep open, but do not merge this PR as-is: the underlying TUI bug is still source-reproducible on current main, while the proposed shortcut loses shared model-list and model-status semantics and still lacks contributor real behavior proof.

## What This Changes

The PR adds TUI branches and tests for `/model list` and `/model status`.

## Best Possible Solution

Route TUI `/model list` and `/model status` through shared model-command semantics or a backend method with the same active agent/workspace/session/auth context and diagnostics.

## Reproduction Assessment

Yes, at source level. Current main parses `/model list` and `/model status` as non-empty `/model` args and sends them to `patchSession` as model refs, while docs define them as list/status commands.

## Solution Assessment

No. Reserving `list` and `status` is directionally right, but this patch duplicates model command logic and loses shared active-session catalog and status behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve session-aware `/model list` resolution:** `src/tui/tui-command-handlers.ts:295-297`
  - body: This aliases `/model list` to `openModelSelector()`, which calls the generic `models.list` path. The shared `/model list` command resolves models with active agent, workspace, session, and auth context, so this can show or select the wrong catalog in multi-agent/profile setups.
  - confidence: 0.87

  - body: This short-circuits `/model status` to a cached provider/model string or `unknown`. The shared status command reports selected vs active model, default model, agent, auth file, and provider auth details, so this regresses documented troubleshooting output.
  - confidence: 0.88
