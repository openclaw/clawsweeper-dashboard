## Reproduction Assessment

Yes. Source review on current main gives a high-confidence path: `/model list` and `/model status` parse as `model` plus non-empty args, then the TUI handler sends those args to `patchSession` as model refs.

## Solution Assessment

No. The PR fixes the literal fallthrough, but `/model list` uses the bare model picker path and `/model status` replaces the shared detailed diagnostics with one cached line.
