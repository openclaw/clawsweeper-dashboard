## What This Changes

The PR adds TUI `/model list` and `/model status` branches plus tests so those inputs no longer call `patchSession` as literal model refs.

## Best Possible Solution

TUI `/model list` and `/model status` should share the same model-command backend semantics as chat, preserving active agent/workspace/session auth context and detailed status output.

## Reproduction Assessment

Yes, at source level. Current main parses `/model list` and `/model status` into non-empty `args`, and the TUI `/model` case sends any non-empty arg to `patchSession` as a model ref.

## Solution Assessment

No. Reserving `list` and `status` is the right direction, but this patch should preserve the shared agent/workspace-aware model list behavior and full `/model status` diagnostics instead of adding a TUI-only shortcut.
