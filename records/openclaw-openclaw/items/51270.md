## Summary

Keep open: this PR addresses the literal TUI fallthrough only partially, but its `/model list` and `/model status` branches bypass the shared session-aware model command behavior and no after-fix real TUI proof is present.

## What This Changes

The PR changes the TUI `/model` command handler so `list` opens the model selector and `status` prints a cached sessionInfo line, with colocated tests.

## Best Possible Solution

Route TUI `/model list` and `/model status` through the shared session-aware model command behavior, or match that behavior exactly, with focused regression coverage and real TUI proof.

## Reproduction Assessment

Yes. Source review against current main gives a high-confidence path: `/model list` and `/model status` parse as `model` plus non-empty args, then the TUI handler sends those args to `patchSession` as model refs.

## Solution Assessment

No. The PR prevents the literal fallthrough, but `/model list` uses the bare picker path and `/model status` replaces the documented shared diagnostics with one cached line.
