## Summary

Keep this PR open but not merge-ready: it targets a real current-main TUI bug and the later commits addressed Greptile's fallback-test comments, but the branch still bypasses shared model command semantics and lacks after-fix real TUI proof.

## What This Changes

This PR adds TUI command-switch branches for `/model list` and `/model status` plus unit tests for selector and status fallback behavior.

## Best Possible Solution

Route TUI `/model list` and `/model status` through shared model-command semantics or an equivalent backend method that preserves active agent, workspace, session, auth context, and full diagnostics.

## Reproduction Assessment

Yes, at source level. Current main parses `/model list` and `/model status` as non-empty `/model` args and sends them to `patchSession` as model refs, while docs define them as list/status commands.

## Solution Assessment

No. Reserving `list` and `status` is directionally right, but this patch duplicates model command logic and loses shared active-session catalog and status behavior.

## Review Findings

- **[P2] Preserve session-aware `/model list` resolution:** `src/tui/tui-command-handlers.ts:295-297`

  - confidence: 0.88
