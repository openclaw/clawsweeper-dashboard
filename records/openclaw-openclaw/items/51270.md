## Summary

Keep open: the underlying TUI bug is real, but this PR bypasses the shared session-aware model command behavior and drops detailed `/model status` diagnostics, so it should be revised or replaced before merge.

## What This Changes

This PR changes the TUI `/model` switch to reserve `list` and `status`, adds tests for those subcommands, and lets the test harness inject extra client methods.

## Best Possible Solution

TUI should reserve `list` and `status` before model setting while delegating to or matching the shared model command implementation, with regression coverage and a user-facing changelog entry.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists on current main: `/model list` and `/model status` parse into the TUI `model` case with non-empty args and are sent to `sessions.patch` as model refs.

## Solution Assessment

No. The PR fixes the literal model-ref failure, but it routes `list` through a bare model picker and replaces `status` with cached session info instead of preserving the shared session-aware diagnostics.
