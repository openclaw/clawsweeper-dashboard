## What This Changes

The PR adds TUI `/model list` and `/model status` branches plus focused command-handler tests.

## Best Possible Solution

Route TUI `/model list` and `/model status` through shared model-command behavior, or an equivalent backend contract that carries active agent, workspace, and session context while preserving full status diagnostics.

## Reproduction Assessment

Yes, at source level. `parseCommand()` leaves `list` and `status` as args, and current main's TUI `/model` case sends every non-empty arg to `patchSession` as a model ref.

## Solution Assessment

No. Reserving `list` and `status` is the right direction, but this implementation should preserve the shared agent/workspace-aware model list and full `/model status` diagnostics instead of adding TUI-only shortcuts.
