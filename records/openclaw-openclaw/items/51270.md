## Summary

Keep open: current main still has the TUI `/model list` and `/model status` routing bug, but this PR is not merge-ready because it bypasses the shared session-aware model command behavior and lacks real after-fix behavior proof.

## What This Changes

The PR adds TUI `/model list` and `/model status` branches plus unit tests so those inputs do not fall through to `patchSession` as model IDs.

## Best Possible Solution

Add a TUI model-command route that treats `list` and `status` as reserved modes and reuses or faithfully matches the shared session-aware model command implementation, with focused regression coverage.

## Reproduction Assessment

Yes. Source review on current main shows `/model list` and `/model status` parse into the TUI `model` case and are sent to `patchSession` as model refs, despite the docs reserving them as command modes.

## Solution Assessment

No. The PR fixes the literal model-ref failure, but `/model list` uses a bare picker path and `/model status` replaces the shared detailed diagnostics with one cached line.
