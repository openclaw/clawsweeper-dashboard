## Summary

Keep open. The PR targets a real TUI bug on current main, but it should not merge as-is because the proposed branches bypass the shared session-aware model command behavior, reduce `/model status` diagnostics, and omit the required changelog entry for a user-facing fix.

## What This Changes

The PR changes the TUI `/model` handler to reserve `list` and `status`, with tests and harness support for injected `listModels` behavior and status output.

## Best Possible Solution

TUI should reserve `list` and `status` before model setting while preserving the shared command contract: session-aware catalog output, detailed status diagnostics, and regression tests proving reserved words do not call `sessions.patch`.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: `/model list` and `/model status` parse into the TUI `model` case with non-empty args, then get sent to `sessions.patch` as model refs.

## Solution Assessment

No. The PR fixes the immediate literal model-ref failure, but it is not the best solution because `list` uses the bare model picker path and `status` replaces the shared detailed diagnostics with cached session info.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Route `/model list` through the session-aware command path:** `src/tui/tui-command-handlers.ts:298-299`
  - body: The new `args === "list"` branch calls `openModelSelector()`, which uses the bare `models.list` path. That path resolves catalog visibility from the default agent workspace, while the shared `/model list` command receives the active agent, workspace, and session entry. In multi-agent or workspace-scoped auth setups this can show the wrong catalog, so preserve the shared command path or pass equivalent session context.
  - confidence: 0.84
- **[P2] Preserve `/model status` diagnostics:** `src/tui/tui-command-handlers.ts:300-306`
  - body: The new `status` branch answers with one cached `state.sessionInfo` line or `unknown`. The shared `/model status` output includes selected vs active runtime model, default model, agent, auth file, endpoint/API, and provider auth labels, and can answer even before the TUI cache is populated. Keep that diagnostic path instead of replacing it with a local cache read.
  - confidence: 0.88
- **[P3] Add the required changelog entry:** `src/tui/tui-command-handlers.ts:298-306`
  - body: This branch changes user-facing TUI command behavior but only updates source and tests. Repo policy requires user-facing fixes to add a `CHANGELOG.md` entry under the active release section, without forcing bot or maintainer credit.
  - confidence: 0.82
