## Summary

Keep open. Current main still sends `/model list` and `/model status` through the TUI model-setting path, but this PR should not merge as-is because it bypasses the shared session-aware model command behavior and the branch is currently dirty against main.

## What This Changes

The PR adds early TUI `/model` branches for `list` and `status`, injects `listModels` into the TUI test harness, and adds focused command tests.

## Best Possible Solution

Reserve `/model list` and `/model status` in the TUI while routing them through the shared session-aware model command behavior, with regression coverage and a changelog entry.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: `/model list` and `/model status` parse into the TUI `model` case with non-empty args, then get sent to `sessions.patch` as model refs.

## Solution Assessment

No. The PR stops the immediate reserved-word patch request, but it replaces the shared session-aware list/status contract with a bare selector path and cached status line.
