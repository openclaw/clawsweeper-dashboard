## Summary

Keep open: the reported TUI bug is still source-reproducible on current main, but this PR is not mergeable as-is because it bypasses the shared session-aware model command behavior, drops the documented status diagnostics, is currently conflicting, and has no after-fix real behavior proof.

## What This Changes

The PR adds TUI handling and tests for `/model list` and `/model status` by routing `list` to the model selector and `status` to cached session info instead of patching the session model.

## Best Possible Solution

Implement TUI `/model list` and `/model status` through the shared model directive/model command behavior, or an equivalent backend contract that carries current agent, workspace, session entry, and full status diagnostics.

## Reproduction Assessment

Yes, at source level. `parseCommand()` leaves `list` and `status` in `args`, and current main's TUI `/model` case sends any non-empty arg to `patchSession` as a model ref.

## Solution Assessment

No. Reserving `list` and `status` is the right direction, but this patch should preserve the shared session-aware model list and full status diagnostics instead of adding a TUI-only shortcut.

## Review Findings
