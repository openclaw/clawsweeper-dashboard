## Summary

Keep open: current main still has the TUI `/model list` and `/model status` fallthrough, but this PR is not mergeable as written because it bypasses the shared session-aware model command behavior and still lacks real TUI proof.

## What This Changes

The PR changes TUI `/model` command handling and tests so `list` opens the model selector and `status` prints cached session model information instead of treating those words as model IDs.

## Best Possible Solution

Route TUI `/model list` and `/model status` through the shared model directive behavior, or match it exactly while preserving active agent, workspace, session context, and full status diagnostics.

## Reproduction Assessment

Yes, at source level. `parseCommand()` leaves `list` and `status` in `args`, and current main's TUI `/model` case sends non-empty args to `patchSession` as model refs.

## Solution Assessment

No. The PR avoids the literal fallthrough, but `/model list` uses a context-poor picker path and `/model status` replaces the documented shared diagnostics with one cached line.
