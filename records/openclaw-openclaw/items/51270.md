## Summary

Keep open, but this PR is not merge-ready: it targets a real TUI `/model` command bug, but the patch bypasses the shared agent/session-aware model command path, reduces `/model status` diagnostics, and still lacks real TUI behavior proof.

## What This Changes

The PR changes the TUI `/model` case so `list` opens the model selector and `status` prints `state.sessionInfo`, with unit tests for those branches.

## Best Possible Solution

Route `/model list` and `/model status` through shared model-command semantics, or a TUI adapter that carries the same session, agent, workspace, and auth context, then prove the behavior in a real TUI run.

## Reproduction Assessment

Yes, at source level. Current main sends non-empty TUI `/model` arguments to `sessions.patch`, while the shared command path and docs reserve `list` and `status` as command actions.

## Solution Assessment

No. Reserving `list` and `status` is the right direction, but this patch solves it with duplicate local behavior instead of the shared session-aware model command path.
