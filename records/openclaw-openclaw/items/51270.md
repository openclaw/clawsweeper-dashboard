## What This Changes

The PR adds TUI branches and tests so `/model list` opens the model selector and `/model status` prints cached session model information instead of patching the model ref.

## Best Possible Solution

Route TUI `/model list` and `/model status` through the shared model directive/model command behavior, or an equivalent backend contract that carries agent, workspace, and session context while preserving full status diagnostics.

## Reproduction Assessment

Yes, at source level. `parseCommand()` leaves `list` and `status` in `args`, and current main's TUI `/model` case sends any non-empty arg to `patchSession` as a model ref.

## Solution Assessment

No. Reserving `list` and `status` is the right direction, but this patch should preserve the shared session-aware model list and full status diagnostics instead of adding a TUI-only shortcut.
