## Summary

Keep open: the TUI bug is still source-reproducible on current main, but this PR is not mergeable as-is because it lacks real TUI proof and its `list`/`status` branches bypass the shared model-command behavior.

## What This Changes

The PR changes the TUI `/model` command handler so `/model list` opens the model selector and `/model status` prints cached session model information, with related tests.

## Best Possible Solution

Route TUI `/model list` and `/model status` through the same session-aware model command behavior used by chat directives, or match that behavior exactly, with focused regression coverage and redacted real TUI proof.

## Reproduction Assessment

Yes. Source inspection shows `/model list` and `/model status` parse as `model` with non-empty args, and the current TUI handler sends those args to `client.patchSession({ model: args })`.

## Solution Assessment

No. The PR prevents the literal `patchSession` fallthrough, but `/model list` uses a context-poor picker path and `/model status` replaces the documented shared diagnostics with one cached line.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Route `/model list` through the session-aware path:** `src/tui/tui-command-handlers.ts:295-297`
  - body: The new `args === "list"` branch calls `openModelSelector()`, whose gateway backend sends unparameterized `models.list` and resolves visibility from the default agent workspace. The shared `/model list` behavior carries active agent/workspace/session context, so multi-agent or workspace-scoped auth setups can show the wrong catalog.
  - confidence: 0.88
- **[P2] Preserve full `/model status` diagnostics:** `src/tui/tui-command-handlers.ts:298-305`
  - body: The new `status` branch reports only cached `state.sessionInfo` or `unknown`. The shared status response includes selected versus runtime model, default model, agent, auth file, provider auth, endpoint, and API details, so this regresses troubleshooting output and can say `unknown` when the shared path can still answer.
  - confidence: 0.9
