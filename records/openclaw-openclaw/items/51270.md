## Summary

Keep open. The PR targets a real current-main TUI `/model list` and `/model status` parsing bug, but the proposed branch bypasses the richer shared model-command contract and is currently dirty, so it should not merge as-is.

## What This Changes

The PR changes the TUI `/model` command handler to reserve `list` and `status`, adds injected client support to the TUI command test harness, and adds tests for those subcommands.

## Best Possible Solution

Keep the PR open for a semantic revision that reserves `list` and `status` in TUI while preserving the shared session-aware catalog/status behavior, with regression coverage and a changelog entry.

## Reproduction Assessment

Yes. Source inspection on current main shows `/model list` and `/model status` enter the TUI `model` case with non-empty args and are forwarded to `sessions.patch` as model refs.

## Solution Assessment

No. The branch fixes the literal model-ref failure, but `list` goes through a bare picker path without the shared session context and `status` replaces the documented detailed diagnostics with a cached one-line response.
