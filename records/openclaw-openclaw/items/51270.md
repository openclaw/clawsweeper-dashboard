## Summary

Keep open: current main still has the TUI fallthrough this PR targets, but the branch is not mergeable as-is because it bypasses shared session-aware model command behavior and lacks after-fix TUI proof.

## What This Changes

The branch teaches the TUI `/model` switch to treat `list` as the model picker and `status` as a cached sessionInfo status line, with colocated tests.

## Best Possible Solution

Route TUI `/model list` and `/model status` through the shared session-aware model command behavior, or match that behavior exactly, with focused regression coverage and real TUI proof.

## Reproduction Assessment

Yes. Source review gives a high-confidence current-main path: `/model list` and `/model status` parse as `model` plus non-empty args, then the TUI handler sends those args to `patchSession` as model refs.

## Solution Assessment

No. The PR prevents the literal fallthrough, but `/model list` uses the bare model picker path and `/model status` replaces the documented shared diagnostics with one cached line.
