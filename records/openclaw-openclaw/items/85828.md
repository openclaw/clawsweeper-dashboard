## Summary

Keep open. The Perplexity override direction is plausible, but the patch exposes an ungated model-facing override under operator credentials and also carries an unrelated Telegram streaming change that would retain stale preview messages.

## What This Changes

This PR adds a `model` parameter to generic and Perplexity `web_search` schemas, routes Perplexity calls with that override through chat completions, documents/tests it, and also changes Telegram draft-stream superseded-preview retention.

## Best Possible Solution

Add a provider-owned, allowlisted Perplexity model override path with docs/tests/proof, and keep Telegram draft retention unchanged unless a separate Telegram PR proves that behavior.

## Reproduction Assessment

Yes, from source inspection rather than an executed test: the PR exposes `model` in the tool schema and passes it straight to Perplexity transport, and the Telegram change marks generation-mismatch previews retained so the dispatcher skips deletion. I did not run tests because this review is read-only.

## Solution Assessment

No. The useful Perplexity direction needs an explicit allowlist/config policy, and the unrelated Telegram retention change should be removed or handled in a separately proved Telegram PR.
