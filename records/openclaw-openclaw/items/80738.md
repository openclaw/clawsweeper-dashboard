## Summary

Keep open: the patch has a blocking correctness issue in the forced-refresh path and still lacks real behavior proof. The in-memory expired store does not survive the real locked OAuth refresh path, which reloads the persisted store before deciding whether to refresh.

## What This Changes

The PR changes Codex app-server forced token refresh to pass an in-memory expired auth profile store instead of persisting `expires: 0`, and adds a regression test.

## Best Possible Solution

Keep the no-stale-write goal, but make forced refresh explicit in the locked OAuth refresh path so the persisted-store reload still refreshes without writing a temporary expired credential.

## Reproduction Assessment

Yes, source-reproducible: use a persisted `openai-codex` OAuth credential whose `expires` is more than five minutes in the future, then call `refreshCodexAppServerAuthTokens`; the patched code would enter the locked refresh path, reload the still-usable persisted credential, and skip provider refresh.

## Solution Assessment

No. Passing an expired in-memory copy to `resolveApiKeyForProfile` is not sufficient because the canonical refresh lock reloads from disk; the fix should be honored inside that locked refresh contract or by an equivalent API that avoids the stale disk write.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Force refresh inside the locked OAuth path:** `extensions/codex/src/app-server/auth-bridge.ts:491`
  - body: `refreshStore` only makes the credential look expired before calling `resolveApiKeyForProfile`. The real OAuth manager then enters `refreshOAuthTokenWithLock`, reloads `auth-profiles.json`, and returns the still-usable persisted credential without calling the provider. A forced app-server refresh for a token more than five minutes from expiry will return the stale access token; the new test misses this because it mocks the resolver and uses a +60s expiry, which the real margin already treats as unusable.
  - confidence: 0.91
