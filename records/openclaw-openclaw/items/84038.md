## What This Changes

The issue reports that `openclaw doctor --fix` rewrites a deliberate `openai-codex/*` plus `agentRuntime.id: "pi"` setup into canonical `openai/*` with the Codex runtime, losing the user's PI/OAuth route.

## Best Possible Solution

Land the linked doctor migration fix that preserves explicit non-default model-scoped runtime pins during canonicalization; handle broader native Codex runtime opt-in or cost behavior separately.

## Reproduction Assessment

Yes. Source inspection on current main shows the combined config path: `model.primary` is canonicalized and a Codex runtime policy is synthesized before the legacy model-map entry carrying `agentRuntime.id: "pi"` is merged.

## Solution Assessment

Yes for the narrow bug. The maintainable solution is to preserve explicit concrete runtime policy during the existing `openai-codex/*` to `openai/*` migration; https://github.com/openclaw/openclaw/pull/84362 is the active fix path.
