## Summary

Keep open: current `main` still has a source-level doctor migration bug where canonical Codex runtime policy can be created before the legacy model-map entry carrying explicit PI intent is merged, so `agentRuntime.id: "pi"` can be overwritten. The related prompt-cache issue may explain the token inflation symptom, but it does not supersede this narrow config-preservation bug.

## What This Changes

The issue requests that `openclaw doctor --fix` preserve intentional `openai-codex/*` plus PI runtime/OAuth routing metadata instead of silently migrating it to native Codex routing.

## Best Possible Solution

Keep the canonical `openai/*` migration, but merge legacy model-map metadata so concrete explicit runtime policy such as `agentRuntime.id: "pi"` wins over generated Codex policy for the same canonical model ref.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: current `main` can create `openai/gpt-5.4` with Codex runtime policy before merging the legacy `openai-codex/gpt-5.4` model-map entry with `agentRuntime.id: "pi"`; this review did not run doctor or tests because it is read-only.

## Solution Assessment

Yes for the narrow bug. The maintainable fix is to preserve explicit legacy model-entry metadata during the documented canonical route migration; changing the default Codex runtime policy is a separate product decision.
