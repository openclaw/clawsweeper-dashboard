## Summary

Keep open: the feature is not implemented on current main, and the PR head still has a correctness/security blocker where agent defaults can suppress non-agent lifecycle hooks before real agent scope is confirmed.

## What This Changes

The branch adds global/default-agent/per-agent internal hook policy overlays, threads config and agent metadata through hook dispatch call sites, updates schema/types/docs, and adds policy tests.

## Best Possible Solution

Rework the branch so agent hook overlays apply only to confirmed agent-scoped events, preserve non-agent lifecycle hooks under global policy, document exact event-key semantics, refresh generated config artifacts, and attach redacted real-run proof.

## Reproduction Assessment

Yes for the blocking PR finding. Source inspection of PR head shows gateway:startup passes config without agentId, agent resolution falls back from the non-agent session key to the default agent, and resolveEffectiveInternalHookPolicy applies agents.defaults.hooks.

## Solution Assessment

No. The feature direction may be useful, but this patch should not merge until agent overlays are gated on confirmed agent scope, generated config artifacts are refreshed, exact event semantics are documented, and real behavior proof is provided.
