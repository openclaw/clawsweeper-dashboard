## Summary

Keep open: current main still lacks this agent-scoped hook policy feature, but this PR is not mergeable or correct as-is because agent defaults can suppress non-agent gateway hooks.

## What This Changes

This PR adds optional `agents.defaults.hooks` and `agents.list[].hooks` internal-hook policy overlays, schema/docs/tests, and call-site config threading for runtime hook dispatch.

## Best Possible Solution

Rebase onto current main, preserve current hook runtime gates, and apply agent-scoped overlays only when a real runtime agent scope exists, with regression coverage for gateway and other non-agent hooks.

## Reproduction Assessment

Yes, for the blocking PR behavior from source inspection. `gateway:startup` passes config without an agent id, while the resolver applies `agents.defaults.hooks` unconditionally and then exact-filters registrations by event key.

## Solution Assessment

No. The feature direction is plausible, but applying agent-default policy globally is not the narrowest maintainable behavior; agent policy should only participate after an actual agent scope is resolved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep agent policies scoped to agent events:** `src/hooks/internal-hooks.ts:307-308`
  - body: As written, `resolveEffectiveInternalHookPolicy` reads `agents.defaults.hooks` before proving the event is agent-scoped. `gateway:startup` now calls `triggerInternalHook` with `config` but no `agentId`, so a default allowlist like `events: ["command:new"]` filters out the `gateway:startup` registration and suppresses startup hooks such as `boot-md`. Apply agent defaults/per-agent overlays only after a real agent scope is resolved.
  - confidence: 0.91

## Security Review

Status: needs_attention

Summary: The diff does not add dependencies or secret handling, but its hook-policy bug can unintentionally suppress lifecycle hooks that operators may rely on for audit or startup controls.

  - body: Because `agents.defaults.hooks` is applied to gateway events, an agent default event allowlist can disable non-agent lifecycle hooks such as `gateway:startup`; that is a policy-control regression for hook-based audit/startup workflows.
