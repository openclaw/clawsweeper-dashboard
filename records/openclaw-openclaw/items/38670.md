## Summary

Keep open. Current main does not implement agent-scoped internal hook policy overlays, and this PR is still the only exact implementation attempt, but it should not merge as-is because the branch has a source-proven lifecycle-hook filtering bug, lacks contributor real behavior proof, and needs rebase/security review for the new hook-policy contract.

## What This Changes

This PR adds internal hook policy config under `hooks.internal`, `agents.defaults.hooks`, and `agents.list[].hooks`, threads config and agent metadata through internal hook dispatch, and updates hook docs and tests.

## Best Possible Solution

Rebase onto current main, apply agent overlays only for explicit or parseable agent scope, settle and document the event-key matching contract, update generated config baselines, and add focused tests plus redacted real-run proof.

## Reproduction Assessment

Yes for the PR defect by source inspection: `gateway:startup` is dispatched with config and no `agentId`, while the resolver applies agent defaults and can infer the configured default agent from a non-agent session key. I did not run tests because this review is read-only.

## Solution Assessment

No. The feature direction may be useful, but this patch needs agent-scope gating, generated baseline alignment, event-contract clarification, rebase work, and real behavior proof before it is maintainable.
