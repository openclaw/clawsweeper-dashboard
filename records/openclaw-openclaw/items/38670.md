## Summary

## What This Changes

The PR adds `agents.defaults.hooks` and `agents.list[].hooks` internal-hook policy config, resolves effective hook policy at dispatch time, threads config through hook call sites, and updates docs/tests for the new policy.

## Best Possible Solution

Define the hook-policy trust model first, then rework this as a current-main patch that applies agent overlays only to real agent-scoped events, preserves lifecycle/security hooks by default, documents and tests event matching semantics, and includes real gateway proof.

## Reproduction Assessment

Yes for the blocking PR defects by source inspection: PR head applies agent defaults before real agent scope is proven, and exact registration-key filtering can skip catch-all handlers. I did not run tests because this is a read-only review.

## Solution Assessment

No. The feature may be useful, but this patch needs current-main rebase work, agent-scope gating, event-contract confirmation, generated config metadata alignment, and real behavior proof before it is the maintainable solution.
