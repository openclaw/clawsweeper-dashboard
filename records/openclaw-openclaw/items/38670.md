## Summary

This PR should stay open because the requested agent-scoped hook policy feature is still not implemented on current main, but the current PR head remains unsafe to merge: it can let agent defaults filter non-agent lifecycle hooks, the generated config baseline is not updated, and contributor real behavior proof is missing.

## What This Changes

The PR adds global/default/per-agent internal hook policy fields, resolves effective policy during internal hook dispatch, threads config through hook call sites, and updates hooks/config docs and tests.

## Best Possible Solution

Land a rebased hook-policy implementation that applies agent overlays only to confirmed agent-scoped dispatches, preserves gateway/system lifecycle hooks under global policy, updates generated config artifacts and docs, and includes focused regression coverage plus redacted real-run proof.

## Reproduction Assessment

Yes for the blocking PR finding: PR-head source inspection shows config-only gateway lifecycle dispatch can merge agent defaults and filter gateway:startup through an allowlist intended for agent events. I did not live-run it because this is a read-only review, but the call path is direct.

## Solution Assessment

No. The feature direction may be useful, but this branch should not merge until agent policy overlays are gated on confirmed agent scope, generated config baselines are refreshed, and real behavior proof is provided.
