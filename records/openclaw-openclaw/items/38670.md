## Summary

Keep open. Current main does not implement this new hook-policy surface, and the PR branch still has a source-reproducible lifecycle-hook suppression bug plus missing real behavior proof before merge.

## What This Changes

The PR adds global/default/per-agent internal hook policy config, resolves that policy during internal hook dispatch, threads config through hook call sites, and updates schema docs and tests.

## Best Possible Solution

Rebase onto current gateway startup code and land a scoped resolver that applies agent overlays only for explicit or parseable agent dispatch, with regenerated config baselines, docs, tests, and redacted real setup proof.

## Reproduction Assessment

Yes for the blocking review finding: source inspection of the PR head shows config-only lifecycle events can resolve to a default agent and merge agents.defaults.hooks, so an events allowlist can filter gateway:startup. I did not live-run it, but the call path is clear from the PR source.

## Solution Assessment

No. The feature direction may be useful, but this implementation should first keep gateway/system lifecycle hooks governed by global policy unless the event has a confirmed agent scope.
