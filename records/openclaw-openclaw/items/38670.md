## Summary

Keep open: current main does not implement agent-scoped hook policy overrides, and this PR still needs changes because agent defaults can apply to non-agent lifecycle hooks, the generated config baseline is missing, real behavior proof is absent, and the branch conflicts with main.

## What This Changes

This PR adds global/default/per-agent internal hook policy config, applies it during internal hook dispatch, threads config through hook call sites, and updates hooks/config docs plus tests.

## Best Possible Solution

Land a scoped hook-policy resolver in the current dispatch/startup architecture that applies agent overlays only for confirmed agent dispatch while preserving global lifecycle hooks, docs, generated config baseline, and focused regression coverage.

## Reproduction Assessment

Yes for the blocking PR finding: source inspection of the PR head shows config-only lifecycle events can resolve a default agent and merge `agents.defaults.hooks`, so an allowlist like `["command:new"]` can filter `gateway:startup`. This was not live-run, but the call path is direct in the PR source.

## Solution Assessment

No. The feature direction may be useful, but this implementation should first gate agent overlays on explicit or parseable agent scope, keep gateway/system lifecycle hooks governed by global policy, update the generated config baseline, and provide redacted real behavior proof.
