## Summary

Keep this PR open: current main still lacks agent-scoped internal hook policy overrides, but the PR needs revision because its policy merge can apply agent defaults to non-agent lifecycle hooks and suppress them unexpectedly.

## What This Changes

The PR adds internal hook policy overlays under `hooks.internal`, `agents.defaults.hooks`, and `agents.list[].hooks`, threads config/agent metadata into dispatch, and updates hook/config docs and tests.

## Best Possible Solution

Revise and rebase the PR so agent overlays apply only to confirmed agent-scoped events, gateway/system lifecycle hooks remain governed by global policy, and schema metadata, docs, changelog, and focused regressions land together.

## Reproduction Assessment

Yes. The PR diff shows a source-level reproduction: `gateway:startup` passes config with no explicit agent id, the fallback derives a default agent from a non-agent session key, and `agents.defaults.hooks.events` can then filter out the lifecycle registration.

## Solution Assessment

No. The requested direction may be useful, but this implementation is not the narrowest maintainable fix because agent-scoped defaults apply outside real agent scope; the safer path is explicit/parseable agent-scope resolution with non-agent lifecycle hooks left on global policy.
