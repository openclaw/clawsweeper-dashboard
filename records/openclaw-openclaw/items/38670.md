## Summary

Keep open. Current main still lacks the requested agent-scoped internal hook policy surface, but this PR is not merge-ready because the patch can apply agent defaults to non-agent lifecycle hooks, lacks real behavior proof, and predates the current gateway startup split.

## What This Changes

The PR adds `hooks.internal`, `agents.defaults.hooks`, and `agents.list[].hooks` policy overlays, threads config into internal hook dispatch, and updates hook config docs and tests.

## Best Possible Solution

Rebase onto current main, apply agent overlays only for explicit or parseable agent scope, settle and document the event-key matching contract, regenerate config baselines, and add redacted real-run proof.

## Reproduction Assessment

Yes for the PR defect by source inspection: `gateway:startup` is dispatched with config and no `agentId`, while the resolver applies agent defaults and can infer the configured default agent from a non-agent session key. I did not run tests because this review is read-only.

## Solution Assessment

No. The feature direction may be useful, but the patch needs agent-scope gating, generated baseline alignment, event-contract clarification, rebase work, and real behavior proof before it is the maintainable solution.
