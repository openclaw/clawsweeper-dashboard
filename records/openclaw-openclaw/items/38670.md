## Summary

Keep this PR open: current main does not implement the requested agent-scoped hook policy, and the PR branch still has a source-reproducible lifecycle-hook suppression bug plus missing real behavior proof.

## What This Changes

The PR adds global/default/per-agent internal hook policy config, threads config and agent metadata through internal hook dispatch call sites, and updates hooks/config docs and tests.

## Best Possible Solution

Land a rebased hook policy feature that applies agent overlays only to confirmed agent dispatch and keeps gateway/system lifecycle hooks under global policy unless maintainers intentionally change that contract.

## Reproduction Assessment

Yes for the blocking review finding: source inspection of the PR branch shows config-only lifecycle events can fall back to the default agent and merge `agents.defaults.hooks`, so an allowlist like `["command:new"]` can filter `gateway:startup`. I did not live-run this path, but the call path is clear from the PR source.

## Solution Assessment

No. The feature direction may be useful, but this implementation should first gate agent overlays on explicit or parseable agent scope and document or revise exact event-key matching semantics.
