## Summary

Keep open: current main does not implement the feature, and this branch still needs human review because the policy resolver can apply agent defaults to non-agent lifecycle hooks while real behavior proof is missing.

## What This Changes

The PR adds optional global/default/per-agent internal hook policy config, resolves effective policy during internal hook dispatch, threads config through hook call sites, and updates docs/tests.

## Best Possible Solution

Rebase onto current main and land a scoped resolver that applies agent overlays only when dispatch has confirmed agent scope, with schema baseline/docs/tests and redacted real behavior proof.

## Reproduction Assessment

Yes for the blocking review finding: source inspection of the PR head shows config-only lifecycle dispatch can resolve a default agent and merge `agents.defaults.hooks`, so an allowlist like `events: ["command:new"]` can filter `gateway:startup`. This was not live-run, but the call path is clear.

## Solution Assessment

No. The feature direction may be useful, but the implementation should first gate agent overlays on explicit or parseable agent scope and keep gateway/system lifecycle hooks governed by global policy unless maintainers intentionally change that contract.
