## Summary

Keep open. Current main still does not implement agent-scoped internal hook policy overrides, but this PR is not mergeable as-is because the branch is conflicting, has a source-reproducible lifecycle-hook suppression bug, is missing generated config baseline follow-through, and lacks after-fix real behavior proof.

## What This Changes

The PR adds global, default-agent, and per-agent internal hook policy configuration, threads config and agent context through internal hook dispatch sites, and updates hooks/config docs and tests.

## Best Possible Solution

Rebase onto current hook and gateway startup code, apply agent overlays only for confirmed agent-scoped dispatch, regenerate config baseline/docs/tests, and require redacted real behavior proof before merge.

## Reproduction Assessment

Yes for the blocking review finding: source inspection of the PR head shows config-only lifecycle events can resolve to the default agent and merge `agents.defaults.hooks`, so `events: ["command:new"]` can filter `gateway:startup`. I did not live-run the branch because the call path is clear and the PR is currently conflicting.

## Solution Assessment

No. The feature direction may be useful, but the implementation should first keep gateway/system lifecycle hooks governed by global policy unless the dispatch has explicit or parseable agent scope.

## Review Findings
