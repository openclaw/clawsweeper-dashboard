## Summary

Keep open. Current main still lacks the requested agent-scoped internal hook policy surface, but this PR should not merge as-is because source inspection shows agent defaults can suppress non-agent lifecycle hooks, the generated config baseline is not updated, and the external contributor has not provided real behavior proof.

## What This Changes

The PR adds `hooks.internal`, `agents.defaults.hooks`, and `agents.list[].hooks` policy fields, resolves them during internal hook dispatch, threads config through hook call sites, and updates docs/tests.

## Best Possible Solution

Land a rebased implementation that applies agent hook overlays only after confirmed agent scope, preserves non-agent lifecycle hooks under global policy, refreshes generated config docs artifacts, and includes focused tests plus redacted real-run proof.

## Reproduction Assessment

Yes for the blocking PR finding: source inspection shows `gateway:startup` dispatch passes config without `agentId`, then PR-head policy resolution can apply `agents.defaults.hooks` and filter the `gateway:startup` registration. I did not live-run it because this is a read-only review.

## Solution Assessment

No. The feature direction may be useful, but this branch should not merge until agent overlays are gated on confirmed agent scope, the generated config baseline is refreshed, and real behavior proof is provided.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Gate agent defaults on real agent scope:** `src/hooks/internal-hooks.ts:307-308`
  - body: `resolveEffectiveInternalHookPolicy` always reads `cfg.agents?.defaults?.hooks`, and `triggerInternalHook` can derive a default agent from non-agent session keys like `gateway:startup`. With `agents.defaults.hooks.events: ["command:new"]`, the gateway startup hook registered as `gateway:startup` is filtered out even though it is not agent-scoped. Only merge defaults/per-agent hook overlays after an explicit or parseable agent scope is confirmed.
  - confidence: 0.91

  - confidence: 0.78
