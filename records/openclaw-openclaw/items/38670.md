## Summary

Keep open: current main does not implement this feature, and the PR is not merge-ready because its resolver can apply agent defaults to non-agent lifecycle hooks and it lacks real behavior proof.

## What This Changes

The PR adds global/default/per-agent internal hook policy config, threads config and agent metadata through internal hook dispatch call sites, and updates hook/config docs and tests.

## Best Possible Solution

Land a rebased, maintainer-approved hook policy config that applies agent overlays only in confirmed agent scopes, updates generated config baselines and docs, and includes redacted real behavior proof.

## Reproduction Assessment

Yes for the blocking PR finding: source inspection of the PR head shows config-only gateway startup dispatch can resolve a default agent and merge `agents.defaults.hooks`, so an allowlist like `["command:new"]` can suppress `gateway:startup`. I did not live-run the branch, but the code path is direct.

## Solution Assessment

No. The feature direction may be useful, but the patch should apply agent overlays only for explicit or parseable agent scopes, update generated config baselines/docs, and include real setup proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Gate agent defaults on confirmed agent scope:** `src/hooks/internal-hooks.ts:307-308`
  - body: When `triggerInternalHook` is called with config but no explicit agent, the PR can derive a default agent from non-agent session keys and then merge `agents.defaults.hooks`. A config like `agents.defaults.hooks.events: ["command:new"]` can therefore suppress lifecycle handlers such as `gateway:startup`; only apply agent overlays after an explicit or parseable agent scope is confirmed.
  - confidence: 0.91
- **[P3] Regenerate the config baseline hash:** `src/config/zod-schema.hooks.ts:123`
  - body: This adds accepted config fields, but the PR file list does not update the tracked `docs/.generated/config-baseline.sha256`. Regenerate the config docs/schema baseline so config checks and consumers agree with the new schema surface.
  - confidence: 0.78
- **[P3] Document exact-match event allowlists:** `docs/automation/hooks.md:525`
  - body: The docs expose `events` as an allowlist, but the implementation checks exact registration keys, so `command:new` does not include broad `command` handlers and `command` does not include specific command handlers. Document that exact-match contract or implement the intended wildcard/prefix behavior before exposing this operator policy.
  - confidence: 0.74

## Security Review

Status: needs_attention

Summary: The diff changes hook execution policy and can unintentionally suppress gateway lifecycle hooks when agent defaults are merged outside a confirmed agent scope.

Concerns:

- **[medium] Agent defaults can suppress lifecycle hooks:** `src/hooks/internal-hooks.ts:307`

Deployments relying on those hooks for audit, DLP, compliance, or startup checks could unintentionally bypass controls.
