## Summary

This PR should stay open: current main still lacks the requested agent-scoped internal hook policy surface, and the branch contains useful unique feature work, but it has a blocking policy-scope bug and still lacks real behavior proof.

## What This Changes

The PR adds optional global, default-agent, and per-agent internal hook policy config, threads config into internal hook dispatch, and updates hook/config docs and tests.

## Best Possible Solution

Land a documented policy resolver that applies agent overlays only for explicit or parseable agent-scoped dispatch and keeps gateway/system lifecycle hooks governed by global policy unless maintainers intentionally define otherwise.

## Reproduction Assessment

Yes for the blocking review finding: source inspection of the PR diff shows config-only lifecycle dispatch can merge `agents.defaults.hooks`, allowing an agent event allowlist to suppress `gateway:startup`. I did not live-run the branch, but the call path is clear from the diff.

## Solution Assessment

No. The feature direction may be useful, but this implementation should first gate agent overlays on confirmed agent scope and document the exact-match `events` contract before becoming the maintained config surface.

## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P2] Gate agent overlays on real agent scope:** `src/hooks/internal-hooks.ts:307-308`
