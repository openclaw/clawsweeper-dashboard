## Summary

Keep open. Current main still does not implement agent-scoped internal hook policy overrides, and the PR remains a meaningful feature proposal, but the patch still has a source-proven lifecycle-hook suppression bug plus a missing real behavior proof gate.

## What This Changes

The PR adds global, agent-default, and per-agent internal hook policy overlays with config schema/docs changes, runtime dispatch filtering, and focused hook/config tests.

## Best Possible Solution

Rebase onto current main, apply agent overlays only for explicit or parseable agent-scoped events, settle and document the event-key matching contract, regenerate config baselines, add focused tests, and provide redacted real runtime proof.

## Reproduction Assessment

Yes for the PR defect by source inspection: gateway:startup uses a non-agent session key while the PR resolver can apply agents.defaults.hooks and infer a default agent from that key. I did not run tests because this review is read-only.

## Solution Assessment

No. The feature direction may be useful, but this patch needs agent-scope gating, generated baseline alignment, event-contract clarification, rebase work, and real behavior proof before it is maintainable.
