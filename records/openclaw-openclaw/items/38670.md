## Summary

Keep open: current main still lacks the requested agent-scoped hook policy surface, but this PR is not ready because the implementation can apply agent defaults to non-agent lifecycle hooks and has no real behavior proof.

## What This Changes

The PR adds global/default/per-agent internal hook policy config, dispatch-time policy resolution, config/agent metadata threading, schemas, docs, and focused tests.

## Best Possible Solution

Land a scoped hook-policy design that applies agent overlays only for confirmed agent dispatch while preserving global lifecycle hooks, schema/docs baselines, regression tests, and real setup proof.

## Reproduction Assessment

Yes for the blocking PR finding: source inspection of the PR head shows a config-only lifecycle event can resolve a default agent and merge `agents.defaults.hooks`, so a narrow allowlist can filter `gateway:startup`. I did not run the branch, but the call path is direct in the PR source.

## Solution Assessment

No. The feature direction may be useful, but this implementation should first gate agent overlays on explicit or parseable agent scope, keep gateway/system lifecycle hooks under global policy, regenerate config baselines after rebase, and provide redacted real behavior proof.
