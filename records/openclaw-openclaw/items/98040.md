## What This Changes

The PR wraps Android `camera.snap` processing in `try/finally` and unbinds the invocation-owned `ImageCapture` use case after completion.

## Best Possible Solution

Merge the scoped CameraX cleanup after redacted Android device proof shows `camera.snap` releases the camera and does not interrupt an immediate follow-up capture; keep the separate `camera.list` privacy path tracked in https://github.com/openclaw/openclaw/issues/92737.

## Reproduction Assessment

Yes for the source-level root cause: current main binds a CameraX still-capture use case without a matching snap cleanup path, and CameraX keeps bound use cases until unbound or lifecycle destruction. No live Android device reproduction was established in this review.

## Solution Assessment

Yes, this is the best narrow code shape: `provider.unbind(capture)` in `finally` releases only the invocation-owned still use case, while a final `unbindAll()` would risk tearing down newer camera work. The implementation still needs real Android proof before merge.
