## Summary

Keep open: this is a viable fix PR for a live linked bug, not a cleanup-close candidate. Current main still accepts known-priced nonzero-token transcript entries with recorded zero cost as real zero spend, while this branch covers the summary, cache, and session-log consumers; the remaining concern is maintainer acceptance of the cache-version invalidation.

## What This Changes

The branch recomputes known-priced transcript entries with zero recorded cost and nonzero tokens, clears stale cost breakdowns for summaries/session logs, bumps the usage-cost cache version to 5, and adds regression coverage.

## Best Possible Solution

Land this focused aggregation/cache/log fix once maintainers accept the cache rebuild behavior, then let it resolve https://github.com/openclaw/openclaw/issues/97047 and supersede https://github.com/openclaw/openclaw/pull/97054.

## Reproduction Assessment

## Solution Assessment

Yes: scanTranscriptFile is the central normalization point for summaries and cache rebuilds, and the PR adds matching loadSessionLogs handling plus coverage for summary, cache, and logs. A DeepSeek-specific provider special case would be narrower but more brittle than correcting the generic known-priced recorded-zero invariant.
