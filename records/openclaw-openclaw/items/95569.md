## What This Changes

The PR changes the `sessions_yield` tool description to warn that cron jobs do not resume yielded sessions and to suggest cron polling or `sessions_spawn` for cron workflows.

## Best Possible Solution

Revise the tool text to point cron waits to `openclaw cron run --wait` or `cron runs --run-id`, update the generated prompt snapshots, and keep the linked issue open until a corrected PR lands.

## Reproduction Assessment

Yes. Source inspection shows `sessions_yield` only records yield intent and relies on a later event, while current cron completion inspection flows through `runId`, `cron.runs`, or `cron run --wait`.
