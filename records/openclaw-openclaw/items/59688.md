## Summary

Keep open. Current main still rejects and warns on managed-skill symlinks whose realpath points into another known skill root, while this PR is a relevant but stale implementation candidate that needs maintainer policy selection alongside #41651 and #45912 before merge.

## What This Changes

The branch threads an allowed symlink target root list through skill containment checks and applies it to managed-skill loading so managed links into other known skill roots do not warn.

## Best Possible Solution

Land one source-preserving configured-root symlink policy, adapting either this managed-only carveout or the broader #45912 approach, with explicit bundled-root semantics, regression coverage, and a changelog entry.

## Reproduction Assessment

Yes. A managed skill symlink whose realpath points into another known skill root is evaluated against the managed root, fails resolveContainedSkillPath, and warns before the target root is scanned and merged separately.

## Solution Assessment

Unclear. The managed-only allowlist is narrower than #45912 and avoids bundled roots, but current main needs rebase work, tests, and an explicit maintainer choice on the cross-root symlink contract.
