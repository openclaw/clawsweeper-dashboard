## What This Changes

The PR threads an allowed symlink target root list through skill containment checks and applies it only while loading managed skills so managed symlinks into other known skill roots do not warn.

## Best Possible Solution

Adapt the narrow managed-symlink allowlist onto current main, preserve source/allowlist semantics, add regression tests for managed links into personal/project/workspace/extra roots and external escapes, and coordinate the result with #41651 and #45912.

## Reproduction Assessment

Yes. On current main, a managed skill symlink whose realpath points into another skill root fails resolveContainedSkillPath for the managed root and reaches warnEscapedSkillPath before roots are merged; the related issue provides the same observed warning pattern.

## Solution Assessment

Unclear. The PR's managed-only direction is safer than allowing bundled roots through non-bundled sources, but it is stale against current loader code and needs tests/changelog before it can be treated as the best merge path.
