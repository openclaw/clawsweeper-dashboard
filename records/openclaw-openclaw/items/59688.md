## Summary

Keep open for maintainer review. Current main has an explicit configured-target escape hatch, but it has not adopted this PR’s automatic managed-skill known-root policy, and the canonical symlink-containment discussion is still active.

## What This Changes

The PR threads an allowed symlink target root list through the skills loader and applies it to managed skills so managed symlinks into other scan roots do not warn.

## Best Possible Solution

Settle one canonical skill symlink policy through https://github.com/openclaw/openclaw/issues/41651, https://github.com/openclaw/openclaw/pull/45912, or an equivalent rebased PR, preserving containment and source identity while covering managed links into trusted roots with regression tests.

## Reproduction Assessment

Yes. By source inspection, a managed skill symlink whose realpath points into another scanned root is checked against the managed root and warns unless the current config-based allowSymlinkTargets path explicitly trusts the target.

## Solution Assessment

Unclear. The managed-only automatic allowlist is plausible, but current main now documents explicit trusted targets and the broader open PR has focused tests, so maintainers should choose the canonical containment policy before merge.
