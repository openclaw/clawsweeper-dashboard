## Summary

Keep open for maintainer approval rather than cleanup: the patch looks code-correct and has sufficient real-path proof, but it changes a public subagent tool security boundary and should be accepted deliberately as a narrow one-shot embedded-native API.

## What This Changes

This PR adds `sessions_spawn.toolsAllow`, forwards it through Gateway agent calls into embedded native runs, rejects ACP/persistent/CLI-backed cases, and updates docs, protocol models, tests, and prompt snapshots.

## Best Possible Solution

Ship this only as a narrowing-only one-shot embedded-native subagent control, and leave durable session policy, denylists, ACP semantics, and CLI semantics in the broader policy track.

## Solution Assessment

Yes, with maintainer approval: the patch reuses the existing embedded-runner allowlist path and fails closed where enforcement is unavailable. If maintainers want durable policy or deny semantics first, the safer path is to keep this paused behind the broader design issue.
