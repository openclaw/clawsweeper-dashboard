## Summary

The least-privilege feature is still useful and the supplied Gateway proof demonstrates its embedded-runner behavior, but the current branch is not merge-safe: it reverts unrelated current-main session rollover fixes, contains broad conflict-resolution churn, persists a spawn-time security policy without a public clear/replace lifecycle, and does not present one consistent contract for unsupported ACP and CLI-backed harnesses.

## What This Changes

Adds `toolsAllow` to native `sessions_spawn` and Gateway agent runs, persists the allowlist on child sessions, and forwards it into later embedded or CLI-backed child execution.

## Best Possible Solution

Reconstruct the feature as a narrow patch on current main, preserve all current session behavior, and expose `toolsAllow` only after agent and Gateway owners define whether it is turn-scoped or durable, how callers clear or replace it, and how every unsupported harness fails closed.
