## Summary

Keep this PR open, but it is not ready to merge: the useful `sessions_spawn.toolsAllow` plumbing is still needed, yet the current patch leaves a security-boundary bug where a persisted child-session allowlist can be widened on a later run, and the branch is reported dirty against its base.

## What This Changes

The PR adds `toolsAllow` to `sessions_spawn` and Gateway `agent.run`, persists it as `runtimeToolsAllow` on child sessions, forwards it into embedded/CLI attempt paths, and updates protocol docs, tests, and prompt snapshots.

## Best Possible Solution

Refresh the branch against current main, keep the additive API only if maintainers accept the public contract, and make persisted `runtimeToolsAllow` an authoritative upper bound with regression coverage for attempted widening.
