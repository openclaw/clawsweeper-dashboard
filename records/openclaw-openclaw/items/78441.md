The PR adds `toolsAllow` to `sessions_spawn` and Gateway `agent.run`, persists it on native child sessions, forwards it into embedded execution, updates generated protocol/docs/snapshots, and includes unrelated formatting and test changes.

## Best Possible Solution

Expose one owner-approved spawned-child tool restriction that is normalized once, applied as a final intersection with existing inherited allow/deny policies, persisted only when persistent native child sessions intentionally need it, and unavailable on general Gateway ingress unless an explicit trusted internal contract requires that surface.
