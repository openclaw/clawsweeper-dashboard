## Summary

## What This Changes

## Best Possible Solution

Keep the additive `sessions_spawn.toolsAllow` surface, but make persisted `runtimeToolsAllow` an upper bound for every later child-session embedded run before merge.

## Reproduction Assessment

The PR head keeps the explicit opts instead of the stored policy and forwards them into embedded tool construction.

## Solution Assessment

No, not yet. The additive control-plane surface is the right general layer, but the best fix must make stored child-session `runtimeToolsAllow` authoritative or an upper bound before this becomes a safe public policy field.
