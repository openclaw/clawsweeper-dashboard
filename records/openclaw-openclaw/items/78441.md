The PR adds normalized `toolsAllow` plumbing for `sessions_spawn` and Gateway `agent.run`, persists native child allowlists on sessions, rejects unsupported ACP/CLI paths, and updates docs, tests, generated protocol models, and prompt snapshots.

## Best Possible Solution
