## Summary

This remains a useful, unique feature PR: current `main` has embedded-runner allowlist machinery but does not expose `toolsAllow` through `sessions_spawn` or direct Gateway agent requests. The implementation uses the existing tool-schema filtering path, persists native child-session policy, rejects unsupported ACP/CLI harnesses, and includes convincing Gateway-backed proof; no discrete correctness defect was found, but the new public, persisted authorization contract still needs explicit agent/security owner approval before merge.

## What This Changes

Adds normalized `toolsAllow` inputs to `sessions_spawn` and Gateway agent runs, persists the allowlist for native child sessions, applies it to embedded child turns, and rejects runtimes that cannot enforce it.

## Best Possible Solution

Approve and document one canonical least-privilege contract: normalize once, persist only on native spawned-child sessions, inherit it on later embedded turns, permit overrides only from the intentionally trusted control plane, and reject every harness that cannot enforce the resulting tool schemas.
