## Summary

Keep this PR open, but it still needs changes before merge: the happy-path `sessions_spawn.toolsAllow` plumbing is useful and has credible current-head proof, yet the PR still lets explicit follow-up Gateway input widen a persisted child-session tool allowlist.

## What This Changes

The PR adds `toolsAllow` to `sessions_spawn` and Gateway `agent.run`, persists it on spawned native child sessions, forwards it to embedded/CLI attempt paths, updates generated protocol models, docs, prompt snapshots, and regression tests.

## Best Possible Solution

Land the feature only after stored child-session allowlists are authoritative on every later turn, with widening rejected before the run and focused regression coverage for omitted, narrower, and wider follow-up inputs.
