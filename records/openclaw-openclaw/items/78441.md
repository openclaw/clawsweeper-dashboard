## Summary

Keep open; the feature is useful and the supplied Gateway-backed proof is strong, but current head is not merge-ready because a stored child-session `runtimeToolsAllow` is only used when no explicit per-run `toolsAllow` is provided, allowing a later Gateway run to widen the persisted tool policy.

## What This Changes

The branch adds `toolsAllow` to `sessions_spawn` and Gateway `agent.run`, persists native subagent allowlists on child sessions, forwards them into runtime boundaries, and updates docs, protocol models, tests, and prompt snapshots.

## Best Possible Solution

Land the feature only after stored child-session allowlists are authoritative, or conflicting per-run values are rejected or intersected, with a regression for stored `["read"]` plus explicit `["read","exec"]` exposing no `exec`.

## Reproduction Assessment

## Solution Assessment

No. The feature direction is reasonable, but the implementation needs stored-vs-explicit precedence fixed so durable child-session tool policy cannot be widened by a later run.
