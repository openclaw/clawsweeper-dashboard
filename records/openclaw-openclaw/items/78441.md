## Summary

Keep open for maintainer review. Current main still lacks the public `sessions_spawn.toolsAllow` entry point, gateway forwarding, durable child-session policy field, and later-turn inheritance added by this branch; I found no line-level correctness defect, but the new public API and persisted session-state semantics need explicit maintainer acceptance before merge.

The branch adds `toolsAllow` to `sessions_spawn` and the gateway agent protocol, persists it as `runtimeToolsAllow` on child sessions, forwards and inherits it for embedded native subagent runs, rejects unsupported ACP and CLI-backed native paths, and updates docs, protocol models, tests, and Codex prompt snapshots.

## Best Possible Solution

Merge only after maintainers explicitly accept the `toolsAllow` public API name, the durable child-session inheritance contract, and the fail-closed behavior for runtimes that cannot enforce the allowlist.
