## Summary

Keep this PR open: current main and v2026.6.1 already have embedded-runner `toolsAllow` enforcement, but they still do not expose or persist that policy from `sessions_spawn` or Gateway agent params, and the PR fills that missing control-plane path without a concrete code-review blocker.

## What This Changes

The PR adds a `toolsAllow` parameter to `sessions_spawn` and Gateway agent requests, persists it as child-session runtime state, forwards it into embedded native runs, rejects unsupported ACP/CLI-backed paths, and updates docs, tests, Swift protocol models, and prompt snapshots.

## Best Possible Solution

Land the branch after maintainer confirmation that `toolsAllow` is the intended public API name and that durable child-session inheritance plus fail-closed unsupported-runtime behavior are the desired contract.
