## Summary

Keep open: the reported third-party npm package is malformed, and current main already rejects new source-only package installs and documents built runtime output, but the existing startup warning path still lacks a clear user action for already-installed broken third-party packages.

## What This Changes

Request clearer diagnostics, install-time validation, and plugin-author docs when a third-party TypeScript plugin package lacks compiled JavaScript output.

## Best Possible Solution

Keep the packaged-runtime policy intact, but make the startup warning identify the package/plugin and point users to update, disable, or report the missing compiled output to the plugin publisher, with WeChat troubleshooting docs if useful.
