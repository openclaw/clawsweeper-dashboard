## Summary

Keep this PR open: current main still lacks transcript mouse-wheel/Page Up/Page Down scrolling, and the PR is a useful implementation candidate, but it still has concrete TUI lifecycle, scroll anchoring, docs, and changelog issues before merge.

## What This Changes

The PR adds ChatLog viewport scroll state, Page Up/Page Down editor callbacks, SGR mouse-wheel parsing/listening, terminal mouse-reporting enable/disable writes, and focused transcript paging tests.

## Best Possible Solution

Land a rebased TUI implementation that keeps transcript viewport state local to ChatLog, routes Page Up/Page Down and SGR wheel input without changing compose focus, restores mouse modes through shared cleanup for normal and error exits, preserves scroll anchors for append and in-place transcript growth, and updates docs/changelog.

## Reproduction Assessment

Yes. Source inspection shows current main lacks the requested transcript scroll input path, and source inspection of the PR diff reproduces the remaining lifecycle and in-place anchoring defects without needing a live terminal.

## Solution Assessment

No, not as currently proposed. The feature direction is narrow and maintainable, but the mergeable solution should centralize mouse-mode cleanup and wrap all rendered-line-count-changing transcript mutations, plus add the required docs and changelog metadata.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Restore mouse mode on every exit path:** `src/tui/tui.ts:1111`
  - body: The PR enables SGR mouse reporting after `tui.start()` but disables it only inside `requestExit()`. If `client.start()` or another startup path throws after this line, `requestExit()` is skipped and pi-tui will not reset the custom mouse modes, leaving the terminal emitting mouse escape sequences; put enable/disable behind shared lifecycle cleanup.
  - confidence: 0.87

  - body: The new scrollback compensation only runs in `append()`. Streaming assistant updates, BTW/tool updates, and tool expansion can grow existing rendered components, so a user scrolled up can still be pushed away from the text they were reading when those components gain lines.
  - confidence: 0.85
