## Summary

Keep open. Current main still does not implement TUI transcript mouse-wheel or Page Up/Page Down scrolling, and this PR remains useful, but the patch is not mergeable as-is because terminal mouse-mode cleanup and scrollback anchoring still have concrete defects. Related #44130 is still open and supports that TUI scroll stability remains a relevant user problem.

## What This Changes

The PR adds ChatLog viewport scroll state, Page Up/Page Down editor callbacks, SGR mouse-wheel parsing/listening, terminal mouse-reporting enable/disable writes, and focused tests for transcript paging and mouse event parsing.

## Best Possible Solution

Land a rebased TUI implementation that keeps transcript viewport state local to ChatLog, routes Page Up/Page Down and SGR wheel input without changing compose focus, restores terminal mouse modes through shared lifecycle cleanup for normal and error exits, preserves scroll anchors for append and in-place transcript growth, and updates the user-facing docs/changelog if the controls are accepted.

## Reproduction Assessment

Yes. Source inspection of current main reproduces the missing feature path, and source inspection of the PR patch reproduces the remaining lifecycle and in-place anchoring defects without needing a live terminal.

## Solution Assessment

No, not as currently proposed. The feature direction is narrow and maintainable, but the mergeable solution should centralize mouse-mode cleanup and wrap all rendered-line-count-changing transcript mutations, not only append().

## Review Findings

  - body: The PR enables SGR mouse reporting after `tui.start()`, but the matching reset only runs inside `requestExit()`.
