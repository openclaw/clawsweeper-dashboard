## Summary

Keep this PR open: the TUI transcript scrolling feature is still useful and not implemented on current main, but the current patch still has lifecycle and scroll-anchor blockers plus missing real terminal proof from the external contributor.

## What This Changes

The PR adds ChatLog viewport scroll state, Page Up/Page Down transcript paging, and SGR mouse-wheel handling for scrolling the TUI transcript.

## Best Possible Solution

Land a rebased or replacement TUI patch that keeps transcript viewport state in ChatLog, routes Page Up/Page Down and SGR wheel input without stealing compose focus, restores mouse reporting through shared start/stop/suspend/error cleanup, preserves anchors for append and in-place growth, and documents the new controls.

## Reproduction Assessment

Yes. Source inspection shows current main lacks the requested transcript scroll input path, and PR diff inspection shows the remaining lifecycle and in-place anchoring defects without needing a live terminal.

## Solution Assessment

No, not as currently proposed. The feature direction is narrow and maintainable, but the mergeable solution should centralize mouse-mode lifecycle cleanup, preserve scroll anchors around all rendered-line-count-changing mutations, and add the user-facing docs entry.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Restore mouse modes through shared cleanup:** `src/tui/tui.ts:1111`
  - body: The PR enables SGR mouse reporting after `tui.start()`, but the reset only runs in `requestExit()`. Startup failures after the enable or TUI suspension paths such as local auth can leave the terminal or child flow in mouse-reporting mode because pi-tui does not know about these custom modes; put enable/disable behind shared lifecycle cleanup.
  - confidence: 0.87
- **[P2] Preserve anchors during in-place transcript growth:** `src/tui/components/chat-log.ts:49-64`
  - body: Scroll compensation only wraps `append()`. Streaming assistant updates, final text replacement, BTW/tool result updates, and tool expansion can grow existing components without changing `scrollOffset`, so a user paged up can still be pushed away from the text they were reading.
  - confidence: 0.85
- **[P3] Document transcript scrolling controls:** `src/tui/components/custom-editor.ts:39-46`
  - body: The PR adds Page Up/Page Down and mouse-wheel transcript controls, but the TUI docs still list only the existing shortcuts. Add the user-facing docs entry so the controls are discoverable.
  - confidence: 0.82
