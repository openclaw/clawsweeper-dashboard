## Summary

Keep this PR open. Current main still does not implement TUI transcript mouse-wheel scrolling or Page Up/Page Down transcript paging, but the proposed branch is not merge-ready because terminal mouse-mode cleanup and scroll anchoring still need to be made robust.

## What This Changes

The PR adds TUI transcript viewport state, Page Up/Page Down controls, SGR mouse-wheel parsing, terminal mouse reporting, wheel routing over the chat log, and focused TUI tests.

## Best Possible Solution

Land a rebased TUI implementation that keeps transcript scrolling local to ChatLog, restores terminal mouse modes through a shared lifecycle/finally path, preserves scroll anchors for both appended and in-place updated transcript content, and documents the shipped mouse and paging shortcuts if accepted.

## Reproduction Assessment

Yes. Source inspection of current main reproduces the missing behavior path: there is no ChatLog scroll state, no SGR mouse listener, and no Page Up/Page Down editor callback; source inspection of the PR patch reproduces the remaining cleanup and anchoring defects.

## Solution Assessment

No, not as currently proposed. The product direction is narrow and useful, but the mergeable solution should centralize terminal mouse-mode cleanup and anchor scrollback across append and in-place update paths before landing.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Restore mouse mode on every exit path:** `src/tui/tui.ts:1100`
  - body: The PR enables terminal mouse reporting after `tui.start()`, but the reset sequence is only sent from `requestExit()`. If startup or `client.start()` throws after enabling, or another error path leaves `runTui()` without `requestExit()`, the user's terminal can stay in mouse-reporting mode. Put mouse-mode enable/disable behind shared lifecycle cleanup that runs for normal and error exits.
  - confidence: 0.86
- **[P2] Preserve anchors during in-place transcript updates:** `src/tui/components/chat-log.ts:51-58`
  - body: The new scrollback anchor compensation only runs in `append()`. Streaming assistant updates and tool result updates grow existing components in place, so when a user is scrolled up the rendered line count can increase without adjusting `scrollOffset`, pushing the viewport away from what they were reading.
  - confidence: 0.84
