## What This Changes

The PR adds ChatLog viewport scroll state, Page Up/Page Down shortcuts, SGR mouse-wheel parsing, and focused tests for scrolling the TUI transcript.

## Best Possible Solution

Land a rebased or replacement TUI patch that keeps transcript viewport state local to ChatLog, routes Page Up/Page Down and SGR wheel input without changing compose focus, restores mouse reporting across start/stop/suspend/error paths, preserves anchors for append and in-place growth, and updates docs/changelog.

## Solution Assessment

No, not as currently proposed. The feature direction is narrow and maintainable, but the mergeable solution should centralize mouse-mode lifecycle cleanup, preserve scroll anchors around all rendered-line-count-changing mutations, and add the required docs/changelog updates.
