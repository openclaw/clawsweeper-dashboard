Keep open. Current main still has Slack dispatch paths where object-form `streaming.preview.toolProgress: false` lacks a tested delivery path that preserves in-flight preview, and the latest discussion says this is a sibling failure rather than the flat-DM gate tracked by #56544.

The issue asks Slack Socket Mode to hide tool-progress preview lines without making verbose in-flight preview delivery silent.

## Best Possible Solution

Slack should keep preview eligibility, native/draft delivery, and tool-progress visibility independent, with regression tests proving object-form `streaming.preview.toolProgress: false` still streams answer previews.
