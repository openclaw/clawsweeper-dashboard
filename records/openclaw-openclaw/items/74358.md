## Summary

Current main now preserves the Slack draft-preview path when `streaming.preview.toolProgress` is false: object-form Slack streaming still resolves to preview-enabled partial mode, flat DMs fall back to draft post-and-edit preview, partial text updates are wired independently of tool-progress visibility, and standalone tool-progress chatter is suppressed while a draft stream is active. The central fix shipped in `v2026.5.12`.

## What This Changes

The issue asks for Slack verbose streaming previews to keep showing in-flight answer text while hiding tool-progress lines when `streaming.preview.toolProgress` is false and no explicit `mode` or `nativeTransport` is configured.

## Best Possible Solution

Keep the shipped Slack draft-preview/tool-progress contract from `v2026.5.12`; a small docs-only cleanup can separately remove the stale Slack-channel sentence.

## Reproduction Assessment

Yes, source-level: the reported object-form config maps to Slack partial preview mode, and the old failure shape is the interaction between draft preview, native-thread fallback, and tool-progress suppression. I did not run a live Slack socket-mode turn in this read-only review.

## Solution Assessment

Yes. The current solution keeps answer preview delivery and tool-progress visibility as separate concerns, using the Slack draft path for flat DMs and the shared suppression helper instead of adding another Slack-specific policy branch.
