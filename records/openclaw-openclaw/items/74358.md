## Summary

Keep open. Current main still leaves Slack answer-preview delivery coupled to the native/draft streaming branch while `streaming.preview.toolProgress: false` only disables progress lines; the shared docs contract says that setting should keep preview streaming and hide only tool-progress text. The latest discussion also treats #56544 as a sibling flat-DM thread-gate fix, not the complete fix for this silent multi-tool preview path.

## What This Changes

Slack verbose turns should continue showing in-flight answer preview text when `streaming.preview.toolProgress` is `false`.

## Best Possible Solution

Slack should keep answer-preview delivery independent from tool-progress visibility across native and draft paths, with regression coverage for object-form config and `toolProgress: false`.

## Reproduction Assessment

Yes. The reporter supplied a stable Slack socket-mode config, and current main has a source-level reproduction path where object-only config defaults to partial/native streaming while the disabled-progress path lacks Slack coverage.

## Solution Assessment

Yes for the requested direction, but no patch is proposed here. The narrow maintainable fix belongs in Slack dispatch/streaming behavior, touching shared channel-streaming helpers only if needed to preserve the documented contract.
