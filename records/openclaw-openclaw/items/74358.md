## Summary

Keep open. Current main still has Slack preview paths where the reporter's object-only `streaming.preview.toolProgress: false` config can suppress visible in-flight feedback, and the latest discussion distinguishes this from the flat-DM thread-gate work in #56544.

## What This Changes

Slack should keep answer preview streaming active while hiding tool/progress lines when `streaming.preview.toolProgress` is set to `false`.

## Best Possible Solution

Slack should treat answer-preview delivery and tool-progress visibility as independent concerns: `toolProgress: false` should hide progress labels while preserving in-flight answer text for object-form config, with regression coverage for the relevant draft/native and DM/thread paths plus aligned Slack docs/help.

## Reproduction Assessment

Yes. The reporter supplied a concrete config and live confirmation, and current main has a source-level path where object-form Slack streaming defaults into partial/native behavior while partial answer callbacks or draft setup can be skipped.

## Solution Assessment

No for the narrow suggested line-only fix. The maintainable fix should route answer partial updates independently from tool-progress visibility and explicitly cover Slack native streaming versus draft-preview fallback behavior.
