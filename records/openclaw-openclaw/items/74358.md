## Summary

Keep open. Current main still has a credible Slack source path where object-form `streaming.preview.toolProgress: false` does not disable Slack streaming config, but the draft answer-preview callback is skipped when Slack native streaming is active, and the remaining open flat-DM PR is explicitly discussed as a sibling rather than a complete fix.

## What This Changes

The issue requests that Slack keep in-flight answer preview streaming visible while hiding tool-progress lines when `streaming.preview.toolProgress` is set to `false`.

## Best Possible Solution

Slack should make `streaming.preview.toolProgress: false` mean answer-preview-on and tool-lines-off across the object-form config path, including native streaming and draft fallback behavior, then lock it with Slack regression tests and align Slack docs/config help with the shared streaming contract.

## Reproduction Assessment

Yes, for a high-confidence source-level reproduction path: the reporter supplied the exact object-form config, current main resolves it to partial/native streaming, and the Slack dispatch path still omits `onPartialReply` when native streaming is active. A live Slack run was not performed.

## Solution Assessment

No, the current implementation is not the best solution for the reported behavior, and the issue body's one-line coupling hypothesis is not sufficient by itself. The safer fix is to separate answer-preview delivery from tool-progress visibility and deliberately handle Slack native streaming versus draft preview fallback.
