## Summary

Keep open. Current main still has Slack paths where disabling `streaming.preview.toolProgress` can remove visible in-flight feedback, and the shared streaming contract says that flag should hide progress lines without turning answer preview off.

## What This Changes

Restore Slack in-flight answer preview when `streaming.preview.toolProgress: false` is configured to hide tool-progress lines.

## Best Possible Solution

Slack should treat answer-preview delivery and tool-progress visibility as independent controls, with object-form config tests proving `toolProgress: false` means answer preview remains on and tool/progress labels stay hidden.

## Reproduction Assessment

Yes. The reporter supplied a stable Slack config and current main has a source-level reproduction path: object-only config defaults into partial/native behavior while disabled tool progress is not covered by Slack preview tests.

## Solution Assessment

No for the line-only hypothesis in the issue body. The maintainable fix should decouple partial answer delivery from progress-label visibility and cover native streaming, draft fallback, and the #56544 flat-DM sibling path explicitly.
