## Summary

Keep open. Current main still has Slack paths where `streaming.preview.toolProgress: false` can remove visible in-flight feedback even though the shared streaming contract says preview text should remain enabled.

## What This Changes

Slack should continue showing answer-preview updates while hiding tool-progress lines when `streaming.preview.toolProgress` is false.

## Best Possible Solution

Patch Slack dispatch so `toolProgress: false` only hides tool-progress text and cannot block answer-preview delivery, with regression tests for object-only Slack streaming config and native versus draft fallback behavior.

## Reproduction Assessment

Yes. The reporter supplied a concrete Slack socket-mode config, and current main maps that object-only config into partial/native streaming paths where draft partial updates are not consistently wired when tool progress is disabled.

## Solution Assessment

Yes for a narrow Slack fix, but no patch is attached here. PR #56544 should remain a sibling for the flat-DM thread gate; this issue needs its own regression coverage for disabled tool-progress preview delivery.
