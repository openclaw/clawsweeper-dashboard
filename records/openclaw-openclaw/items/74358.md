## Summary

Keep open. Current main still has Slack streaming paths where `streaming.preview.toolProgress: false` disables tool-progress preview lines but does not guarantee in-flight answer preview delivery, especially when the object-form config defaults into Slack native streaming and `onPartialReply` is not wired.

## What This Changes

The issue asks Slack to keep answer draft preview streaming active while hiding only tool-progress lines when `streaming.preview.toolProgress` is `false`.

## Best Possible Solution

Slack should separate answer-preview delivery from tool-progress visibility for object-form streaming config, covering both native streaming and draft-preview fallback so `toolProgress: false` means answer preview stays on and progress lines stay hidden. Add regression tests for the reporter's config and align Slack docs/help with the shared streaming contract.

## Reproduction Assessment

Yes, at source level. The reporter supplied a stable object-form config, and current main still defaults that config to partial/native streaming while omitting the answer partial callback on the native path; a live Slack run was not performed.

## Solution Assessment

No, changing only `suppressDefaultToolProgressMessages` would not fix the partial-text path. The maintainable fix should route answer partial updates independently from tool-progress visibility and account for Slack native streaming versus draft fallback.
