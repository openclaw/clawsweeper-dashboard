## Summary

Keep open. Current main still has Slack streaming paths where `streaming.preview.toolProgress: false` can disable tool-progress preview updates without independently guaranteeing answer-preview delivery, and the discussion confirms this is a separate failure mode from the flat-DM thread-gate work in #56544.

## What This Changes

The issue requests restoring Slack in-flight answer preview when object-form streaming config disables tool-progress lines.

## Best Possible Solution

Slack should treat answer-preview delivery and tool-progress visibility as independent concerns: `toolProgress: false` should hide progress labels while preserving in-flight answer text, with regression coverage for object-form config, native streaming, draft fallback, and the #56544 flat-DM sibling path.

## Reproduction Assessment

Yes. The reporter supplied exact object-form Slack config and a stable repro, and current-main source still shows a matching branch where disabled tool progress is not independently covered by answer-preview delivery tests.

## Solution Assessment

Unclear for the line-level hypothesis in the issue body. The maintainable fix is broader than flipping only `suppressDefaultToolProgressMessages`: decouple text preview from tool-progress visibility and lock the Slack native/draft branches with focused tests.
