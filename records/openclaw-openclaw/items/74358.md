## Summary

Keep open. Current main still leaves the reported Slack behavior plausible and actionable: object-form Slack streaming config defaults into partial/native streaming, `toolProgress: false` only disables draft progress-line updates, and the answer partial callback is not wired on the native-streaming path. The latest discussion also separates this from the flat-DM gate in #56544, so there is no merged or open canonical fix that should close this issue yet.

## What This Changes

The issue requests that Slack verbose streaming keep showing in-flight answer preview text when `streaming.preview.toolProgress` is `false`, while hiding only tool-progress lines.

## Best Possible Solution

Slack should treat `streaming.preview.toolProgress: false` as a visibility toggle for progress lines only. The implementation should keep answer-preview delivery active for the object-form config across native streaming and draft fallback, then update Slack docs/config help so they match the shared streaming contract.

## Reproduction Assessment

Yes, at source level. The reporter supplied a concrete object-only Slack config, and current main still defaults that config to partial/native streaming while omitting the answer partial callback on the native path; a live Slack run was not performed.

## Solution Assessment

Yes, a narrow Slack fix is the right direction. The fix should decouple answer preview delivery from tool-progress visibility rather than closing this under #56544, which targets the separate flat-DM thread gate.
