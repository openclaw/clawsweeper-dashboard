## Summary

Keep open. Current main still has Slack paths where object-form `streaming.preview.toolProgress: false` can leave users without visible in-flight answer preview, and the discussion establishes this as a separate failure mode from the flat-DM thread-gate work in #56544.

## What This Changes

The issue asks Slack streaming to keep answer preview text visible while hiding tool-progress lines when `streaming.preview.toolProgress` is `false`.

## Best Possible Solution

Slack should treat answer-preview delivery and tool-progress visibility as independent concerns. `toolProgress: false` should hide progress labels while preserving in-flight answer text for object-form Slack config, with regression coverage for native versus draft preview paths and aligned Slack docs/config help.

## Reproduction Assessment

Yes. The reporter supplied exact Slack config and confirmed a stable repro, and current main still has a source-level path where object-form config defaults into partial/native behavior while draft partial callbacks can be skipped or gated away.

## Solution Assessment

No for the narrow line-only hypothesis in the issue body. The safer fix is to decouple answer partial updates from tool-progress visibility and explicitly test Slack native streaming, draft fallback, and the #56544 flat-DM sibling path.
