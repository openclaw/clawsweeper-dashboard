## Summary

Keep open. Current main still has Slack dispatch paths where `streaming.preview.toolProgress: false` can suppress visible in-flight feedback even though the shared streaming contract says that flag should hide progress lines without disabling answer preview.

## What This Changes

The issue reports that Slack object-form streaming config with `streaming.preview.toolProgress: false` makes verbose multi-tool runs silent until the final reply.

## Best Possible Solution

Slack should make answer-preview delivery independent from tool-progress visibility, then add regression tests proving `toolProgress: false` means preview-on and progress-lines-off for object-form Slack streaming config.

## Reproduction Assessment

Yes. The reporter provided a stable Slack config, and current main has a source-level reproduction path through object-form config defaults, Slack dispatch callback gating, and missing disabled-progress coverage.

## Solution Assessment

Yes, a narrow Slack plugin fix is the best route. Do not close this with #56544 or only flip the `suppressDefaultToolProgressMessages` line; the durable fix needs Slack dispatch tests for native/default and draft-preview fallback behavior.
