## Summary

Keep open. Current main still has Slack streaming paths where `streaming.preview.toolProgress: false` is coupled to preview/tool-progress delivery, while the shared streaming contract says that flag should hide progress lines without disabling answer preview.

## What This Changes

The issue asks Slack to keep in-flight answer preview working while hiding tool-progress preview lines when `streaming.preview.toolProgress` is false.

## Best Possible Solution

Slack should make answer-preview delivery independent from tool-progress visibility, then align Slack docs/help text with the shared streaming contract.

## Reproduction Assessment

Yes. The reporter supplied a stable config, and current main has a source-level reproduction path through object-form Slack streaming defaults, preview/native gating, and `toolProgress`-dependent suppression; live Slack was not run here.

## Solution Assessment

Yes for the requested decoupling, but not for a line-only fix. The maintainable fix should cover Slack native streaming, draft fallback, object-form config, and the sibling flat-DM gate explicitly in tests.
