## Summary

Keep open. Current main still has a Slack-specific path where `streaming.preview.toolProgress: false` is not covered by tests and the dispatch wiring couples progress suppression to draft-preview behavior, while the shared streaming contract says this flag should hide tool-progress lines without disabling answer previews.

## What This Changes

Restore Slack in-flight answer preview for object-form `streaming.preview.toolProgress: false` configs while keeping tool/progress lines hidden.

## Best Possible Solution

Slack should treat answer-preview delivery and tool-progress visibility as independent controls, matching the shared contract and Telegram-style suppression behavior while keeping the fix inside the Slack plugin boundary.

## Reproduction Assessment

Yes. The reporter supplied a stable object-only Slack config, and current main has a source-level reproduction path: that config defaults to partial/native behavior while Slack has no disabled-`toolProgress` regression coverage.

## Solution Assessment

Yes. A focused Slack plugin fix with tests for `toolProgress: false` is the narrowest maintainable path; #56544 should remain sibling flat-DM context, not the sole fix vehicle for this issue.
