## Summary

Keep open. Current main still treats Control UI `[embed]` rendering as height-only, and the existing global chat-message width setting does not provide the requested per-embed width contract.

## What This Changes

Add a per-embed width parameter to the Control UI `[embed]` directive so iframe previews can request desktop-sized widths.

## Best Possible Solution

Add a typed, validated `preferredWidth` hint through shortcode parsing, normalized preview contracts, iframe rendering/CSS, prompt/docs, and focused tests while preserving current sandboxing and URL allow rules.

## Reproduction Assessment

Yes, at source level. A Control UI assistant message such as `[embed ref="cv_demo" title="Demo" height="600" width="900" /]` follows the shortcode parser, but current main only carries `height`/`preferredHeight` through parser, types, normalizer, renderer, CSS, docs, and tests.

## Solution Assessment

Yes for the feature direction, with a safer boundary than the issue sketch. The maintainable path is a bounded `preferredWidth` presentation hint, not arbitrary inline style passthrough or sandbox policy changes.
