## Summary

Keep open. Current main still treats Control UI `[embed ...]` sizing as height-only: the shortcode/parser contract, normalized UI shape, renderer, CSS, docs, and prompt guidance have no width/preferredWidth path, so the requested capability remains missing.

## What This Changes

Request to add a `width` parameter to Control UI `[embed ...]` directives so iframe previews can render wider than the chat column.

## Best Possible Solution

Add a bounded `preferredWidth` presentation hint end to end for hosted Control UI embeds, carrying it through parser/types/normalization/sidebar rendering/docs/tests while preserving existing sandboxing and URL restrictions.

## Reproduction Assessment

Yes. A static reproduction is an assistant message such as `[embed ref="cv_demo" title="Demo" height="600" width="900" /]`: current main parses and applies the height path, but the width attribute is ignored and the iframe remains container-width.

## Solution Assessment

Yes. The narrow maintainable direction is to extend the existing canvas preview contract with a bounded width hint rather than allowing arbitrary inline styles or changing iframe security policy.
