## Summary

Keep open. Current main still has only a height hint for Control UI `[embed ...]`; the parser, normalized UI type, renderer, CSS, docs, and tests do not carry or apply a per-embed width value.

## What This Changes

The issue requests a width parameter for Control UI `[embed]` directives so embedded iframes can render wider than the chat message column.

## Best Possible Solution

Add a bounded `preferredWidth` presentation hint end to end for hosted Control UI embeds, carrying it through parser/types/normalization/rendering/sidebar/docs/tests while preserving existing sandboxing and URL restrictions.

## Reproduction Assessment

Yes. A static current-main reproduction is an assistant message like `[embed ref="cv_demo" title="Demo" height="600" width="900" /]`: source inspection shows height is normalized and rendered, while width is ignored and the iframe remains `width: 100%` of its container.

## Solution Assessment

Yes for the feature direction, with one adjustment: the maintainable fix is a typed, bounded `preferredWidth` hint, not arbitrary style passthrough or a change to iframe security policy.
