## What This Changes

The PR changes CLI run preparation and result construction so `claude-stdio`/JSONL/stdin warm-stdin sessions skip prepare-time native transcript invalidation and keep CLI session bindings when post-turn flush probes miss.

## Best Possible Solution

Use one warm-stdin backend contract before both prepare-time and post-turn native transcript probes, add focused second-turn no-native-transcript coverage, and merge one canonical fix with redacted live or packaged Claude CLI proof.

## Reproduction Assessment

Yes at source level: current main clears the CLI binding when `bindingFlushOk` is false, and the linked issue supplies the warm-stdin no-native-transcript scenario. I did not run a fresh live Claude CLI session in this read-only review.

## Solution Assessment

No. The branch now covers prepare-time invalidation and suppresses destructive clearing, but the best fix should avoid running the impossible post-turn native transcript probe for warm-stdin at all or validate the correct OpenClaw-owned transcript artifact.
