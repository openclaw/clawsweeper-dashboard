## Summary

Keep this PR open: the reported claude-cli continuity regression is real and actionable, but this branch removes the write-side flush gate for every claude-cli miss instead of distinguishing the warm-stdin backend from genuine missing or unflushed native transcripts.

## What This Changes

The PR makes successful CLI turns always keep the effective session id by disabling the `bindingFlushOk === false` path that currently clears claude-cli session bindings.

## Best Possible Solution

Make the flush-miss handling backend-aware: validate warm-stdin claude-cli sessions against OpenClaw's captured transcript or treat only that backend's pure native-transcript miss as non-destructive, while preserving stale native-transcript invalidation for real flush failures.

## Reproduction Assessment

Yes, source-reproducible with high confidence: the linked issue traces the warm-stdin claude-cli backend to a native transcript probe that can never pass, and current code shows that a failed probe clears persisted session state.

## Solution Assessment

No; the patch is a plausible mitigation, but it is not the best fix because it removes the protection for all claude-cli flush misses instead of only the backend mode that lacks native transcripts.
