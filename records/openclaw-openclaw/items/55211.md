## Summary

Keep open: the patch is focused and the copied Node runtime proof is sufficient, but merging it intentionally changes the hook dispatch contract by suppressing same-key redispatch from a handler's async lineage, so hook-owner sign-off is still needed.

## What This Changes

The branch adds an AsyncLocalStorage-backed same-key re-entrancy guard around `triggerInternalHook` plus tests for recursive, sequential, concurrent, cross-session, and cross-action dispatch.

## Best Possible Solution

Land the ALS guard if hook owners accept same async-lineage suppression; otherwise adjust the guard to permit delayed child redispatch and add a regression test for that contract.

## Reproduction Assessment

Yes. Current main awaits hook handlers without any active same-key guard, so a handler can recursively call `triggerInternalHook` with the same key; the PR discussion also includes copied Node output showing unguarded amplification and guarded behavior.

## Solution Assessment

Mostly yes. ALS is a clean fit for blocking same async-chain recursion while preserving independent concurrent dispatches, but maintainers still need to accept the same-lineage delayed redispatch contract.
