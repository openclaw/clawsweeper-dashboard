## Summary

Keep open: this is a focused bugfix PR for a real source-reproducible hook recursion path that is not implemented on current main. The patch looks likely correct, but it changes the same-key internal hook redispatch contract and needs hook-owner acceptance before merge.

## What This Changes

The PR wraps `triggerInternalHook` in an AsyncLocalStorage same-key guard and adds regression tests for recursive, sequential, concurrent, cross-session, and cross-event dispatch.

## Best Possible Solution

Land the ALS guard if hook owners accept same-key async-lineage suppression; otherwise define delayed same-key redispatch semantics and add a focused regression test before merge.

## Reproduction Assessment

Yes. Current main awaits hook handlers without a same-key active-dispatch guard, so a handler that calls `triggerInternalHook` again with the same key recurses through the same source path.

## Solution Assessment

Mostly yes. AsyncLocalStorage is a maintainable way to block same-chain recursion while preserving independent concurrent dispatch, but delayed child-async redispatch semantics need maintainer acceptance.
