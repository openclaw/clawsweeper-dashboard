## Summary

Keep open. Current main still dispatches internal hooks without a re-entrancy guard, and this PR is a focused fix with relevant tests and terminal proof; the remaining concern is maintainer acceptance of the intentional compatibility change for nested same-key hook delivery.

## What This Changes

The PR adds an AsyncLocalStorage same-key dispatch guard to `triggerInternalHook` and expands internal hook tests for recursive, sequential, concurrent, cross-key, and delayed redispatch behavior.

## Best Possible Solution

Land the AsyncLocalStorage guard if maintainers accept same-chain same-key suppression as the internal hook contract, with the focused regression tests kept alongside the runtime change.

## Reproduction Assessment

Yes from source inspection: current main recursively invokes matching handlers with no active-dispatch guard, so a handler that triggers the same key again can re-enter indefinitely. I did not run a mutating local repro because this review is read-only.

## Solution Assessment

Yes, with maintainer sign-off on the contract change: AsyncLocalStorage is a narrow fit because it blocks only same-chain re-entry while preserving independent concurrent dispatches. A broader config knob would be less clean unless maintainers decide nested same-key delivery must remain compatible.
