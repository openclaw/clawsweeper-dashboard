## Summary

Keep open. Current main still lacks a re-entrant hook guard, and the latest PR fixes the prior global-Set concurrency regression with an AsyncLocalStorage-shaped approach, but the new slow-handler regression test can hang because it resolves only one of two pending handler calls; the user-facing hook behavior also needs a changelog entry.

## What This Changes

The PR adds an AsyncLocalStorage-backed re-entrancy guard to `triggerInternalHook` and expands internal hook tests for recursive, sequential, concurrent, cross-session, and cross-action dispatch.

## Best Possible Solution

Land the AsyncLocalStorage guard after fixing the hanging regression test and adding a changelog entry, preserving same-call-chain recursion suppression without dropping independent concurrent hook events.

## Reproduction Assessment

Yes. The introduced test blocker reproduces by starting two dispatches whose slow handler overwrites one shared resolver, resolving once, and observing that the first dispatch remains pending; the original loop is also modeled by a recursive same-key handler on current main, which has no guard.

## Solution Assessment

No as submitted. The AsyncLocalStorage direction is the narrow maintainable fix after the prior global-Set feedback, but the PR branch needs the hanging test repaired and the user-facing hook fix documented before merge.
