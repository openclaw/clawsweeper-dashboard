## Summary

Keep this PR open: current main still has unguarded internal hook dispatch, and the PR contains a focused, actively maintained fix with sufficient terminal proof, no blocking code finding, and one compatibility contract decision for hook owners.

## What This Changes

The PR adds an AsyncLocalStorage-based guard to `triggerInternalHook` plus regression tests for re-entrant, sequential, concurrent, cross-session, and cross-event dispatch behavior.

## Best Possible Solution

Land the ALS guard if hook owners accept same-key async-lineage suppression; otherwise add an explicit delayed-redispatch contract test and adjust the guard before merge.

## Reproduction Assessment

Yes, source-reproducible. Current main awaits hook handlers without a same-key active-dispatch guard, so a handler that calls `triggerInternalHook` again with the same `type:action:sessionKey` recurses through the same path.

## Solution Assessment

Mostly yes. AsyncLocalStorage is a maintainable way to block same-chain recursion while preserving independent concurrent dispatch, but the delayed child-async suppression semantics need maintainer acceptance.
