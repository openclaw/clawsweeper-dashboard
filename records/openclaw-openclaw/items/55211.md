## Best Possible Solution

Land the ALS guard if hook owners accept suppressing same-key redispatches inherited from a handler async lineage; otherwise define that delayed follow-up contract and adjust coverage/implementation before merge.

## Solution Assessment

Mostly yes. AsyncLocalStorage is a maintainable way to block same-chain recursion while preserving independent concurrent calls, but maintainers need to accept or refine delayed-child async suppression semantics.

## Risks / Open Questions

- AsyncLocalStorage propagates into child async resources created inside a handler, so a delayed same-key redispatch spawned from that handler lineage can be silently skipped even after the outer handler returns.
- The PR changes internal hook delivery semantics for same-key redispatches; maintainers should explicitly accept that compatibility tradeoff or define a narrower delayed-follow-up contract before merge.
