## Summary

Keep open for maintainer review: current main still lacks a same-key re-entrancy guard, and this focused PR appears correct, but the AsyncLocalStorage contract can suppress delayed same-key redispatches spawned from a handler lineage.

## What This Changes

This PR adds an AsyncLocalStorage guard to `triggerInternalHook` plus internal-hook tests for re-entrant, sequential, concurrent, cross-session, and cross-event dispatch.

## Best Possible Solution

Land the ALS guard if hook owners accept same-key async-lineage suppression; otherwise narrow the guard contract and add delayed-child redispatch coverage before merge.

## Reproduction Assessment

Yes. Current main has no same-key guard in `triggerInternalHook`, so a handler that calls the same hook can re-enter indefinitely; the supplied terminal proof demonstrates the unguarded amplification and guarded behavior.

## Solution Assessment

Mostly yes. AsyncLocalStorage is a clean way to block same-chain recursion while preserving independent concurrent calls, but maintainers need to accept or refine delayed-child async suppression semantics.
