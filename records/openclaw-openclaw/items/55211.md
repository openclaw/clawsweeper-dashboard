## Summary

Keep open: the PR is a focused, plausible fix for a source-reproducible internal-hook recursion path, but it still needs contributor-supplied real behavior proof and maintainer confirmation of the compatibility tradeoff before merge.

## What This Changes

The PR adds an AsyncLocalStorage re-entrancy guard to `triggerInternalHook` plus regression tests for recursive, sequential, concurrent, cross-session, and cross-action dispatch.

## Best Possible Solution

Land the guard after real proof is added and hooks owners either accept child-lineage suppression or narrow the guard to active dispatch only if delayed same-key child hooks must remain deliverable.

## Reproduction Assessment

Yes by source inspection, though I did not run a live repro in this read-only review. Current main awaits handlers without a same-key active guard, so a handler can recursively call `triggerInternalHook` with the same type/action/sessionKey.

## Solution Assessment

Yes for the general code direction, with one compatibility decision still needed. AsyncLocalStorage is a narrow way to block same-chain recursion while preserving outside concurrent dispatch, but maintainers should explicitly accept the child-lineage suppression semantics.
