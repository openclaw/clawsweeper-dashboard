## Summary

Keep open: this PR still addresses a source-reproducible internal-hook re-entrancy bug on current main, and the current AsyncLocalStorage patch has sufficient runtime proof with no blocking code-review findings. The remaining decision is compatibility: whether same-key hook work spawned inside a handler's async lineage should be suppressed.

## What This Changes

Adds an AsyncLocalStorage-backed same-key re-entrancy guard to `triggerInternalHook` and regression tests for recursive, sequential, concurrent, cross-session, and cross-action dispatch.

## Best Possible Solution

Land the ALS guard if hook owners accept same async-lineage suppression; otherwise adjust the guard to permit delayed child redispatch and add a regression test for that contract.

## Reproduction Assessment

Yes. Current main awaits hook handlers with no active same-key guard, so a handler can recursively call `triggerInternalHook` with the same event key; the contributor's Node terminal proof also shows unguarded amplification and guarded behavior.

## Solution Assessment

Mostly yes. ALS is a clean fit for blocking same async-chain recursion while preserving independent concurrent dispatches, but the hook-owner contract needs to accept suppression of delayed child async same-key redispatch.
