## Summary

Keep open: the patch appears focused and correct, and the latest copied terminal proof is enough to clear the contributor proof concern, but the same-key child async suppression is a compatibility contract change that still needs hook-owner confirmation before merge.

## What This Changes

The PR adds an AsyncLocalStorage same-key re-entrancy guard to internal hook dispatch and regression tests for recursive, sequential, concurrent, cross-session, and cross-action dispatch.

## Best Possible Solution

Land the ALS guard with the regression tests once hook owners accept same-key child async suppression; if delayed same-key child hooks must remain deliverable, narrow the guard before merge.

## Reproduction Assessment

Yes by source inspection. Current main awaits handlers without a same-key active guard, so a handler that awaits triggerInternalHook with the same type/action/sessionKey can recursively redispatch itself.

## Solution Assessment

Mostly yes. AsyncLocalStorage is a better fit than the earlier global Set because it preserves independent concurrent calls, but maintainers still need to accept the inherited child-async suppression semantics.
