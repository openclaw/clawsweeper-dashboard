Keep open for maintainer review. The PR is a focused, active hook-runtime bug fix that is not on current main and has no blocking code/security findings, but it deliberately changes same-key redispatch semantics and needs hook-owner acceptance before merge.

## What This Changes

The branch adds an AsyncLocalStorage-backed same-key re-entrancy guard around `triggerInternalHook` and regression tests for recursive, sequential, concurrent, cross-session, and cross-action dispatch.

## Best Possible Solution

Land the ALS guard if hook owners accept same async-lineage same-key suppression; otherwise adjust the guard to preserve the desired delayed-child redispatch contract and add a regression test for that case.

## Reproduction Assessment

Yes from source inspection: current main awaits hook handlers without a same-key active-dispatch guard, so a handler can recursively call `triggerInternalHook` with the same key. I did not run a local repro because this review was read-only, but the PR body includes copied Node output demonstrating the before/after behavior.

## Solution Assessment

Mostly yes. ALS is a clean fit for blocking same async-chain recursion while preserving independent concurrent dispatches, but the silent same-lineage drop semantics still need maintainer acceptance.
