## Summary

Keep open for maintainer review: the focused fix looks sound and the latest proof is sufficient, but merging it changes the hook dispatch contract for same-key async work spawned inside hook handlers, so hook-owner sign-off is still needed.

## What This Changes

Adds an AsyncLocalStorage-backed same-key re-entrancy guard to `triggerInternalHook` plus regression tests for recursive, sequential, concurrent, cross-session, and cross-action dispatch.

## Best Possible Solution

Land the ALS guard if hook owners accept same async-lineage suppression; otherwise adjust the guard to permit delayed child redispatch and add a regression test for that contract.

## Reproduction Assessment

Yes, by source inspection rather than an executed current-main repro: current main awaits handlers with no active same-key guard, so a handler can recursively call `triggerInternalHook` with the same event key. The contributor's terminal proof also demonstrates unguarded recursive amplification and guarded behavior in a Node runtime.

## Solution Assessment

Mostly yes: ALS is a clean fit for blocking same async-chain recursion while preserving independent concurrent dispatches, and the PR tests those boundaries. The remaining question is whether suppressing delayed child async same-key redispatch is the intended hook contract.
