## What This Changes

The PR adds an AsyncLocalStorage-based guard around internal hook dispatch to suppress same-key re-entrant hook calls and expands hook tests for recursive, sequential, concurrent, cross-session, and delayed redispatch behavior.

## Best Possible Solution

Establish the concrete integrated recursion path, then either approve and document same-lineage same-key suppression as the shared hook contract or move cycle prevention to the narrow recursive producer.

## Reproduction Assessment

Yes at the source level: a current-main handler can recursively await `triggerInternalHook` with the same `type:action:sessionKey` and redispatch repeatedly. No integrated production path has been proven for the claimed embedded-agent loop.

## Solution Assessment

Unclear. The AsyncLocalStorage guard is a plausible dispatcher-level mitigation, but the safer best fix may be a narrower guard at the actual recursive producer unless maintainers approve the shared hook contract change.
