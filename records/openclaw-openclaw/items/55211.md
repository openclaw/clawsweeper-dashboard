## Summary

Keep open. Current main still lacks any re-entrant internal hook guard, and the latest AsyncLocalStorage approach addresses the earlier global-Set concurrency concern, but the new slow-handler regression test can hang because it resolves only one of two pending handler promises.

## What This Changes

The PR adds an AsyncLocalStorage-scoped re-entrancy guard to internal hook dispatch and expands internal hook tests for recursive, sequential, concurrent, cross-session, and cross-action cases.

## Best Possible Solution

Land the ALS-scoped guard after fixing the slow-handler concurrency test and, if maintainers treat hook-loop prevention as user-facing, add a single-line changelog entry.

## Reproduction Assessment

Yes. For the proposed patch, start the new slow-handler test's two dispatches: the second `slowHandler` invocation overwrites the single resolver, so resolving it leaves `firstDispatch` pending and the final `Promise.all` can time out.

## Solution Assessment

No as submitted. The AsyncLocalStorage guard is a maintainable direction for call-chain re-entry, but the added regression test needs separate resolvers or a resolver list so both pending slow handlers are released.

## Review Findings

Overall correctness: patch is incorrect
