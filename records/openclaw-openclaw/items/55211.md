## Summary

Keep open. Current main still lacks a same-key re-entrancy guard, and the PR’s AsyncLocalStorage direction addresses the earlier global Set concurrency regression, but the latest added slow-handler test deadlocks by overwriting the first pending resolver.

## What This Changes

The PR branch adds an AsyncLocalStorage-based same-key re-entrancy guard around `triggerInternalHook` and expands `src/hooks/internal-hooks.test.ts` with recursive and concurrent dispatch coverage.

## Best Possible Solution

Land a recursion guard that preserves independent same-key hook delivery, after making the concurrent slow-handler regression test deterministic and keeping the internal hook API unchanged.

## Reproduction Assessment

Yes. On current main, a same-key handler can recursively call `triggerInternalHook` because no guard exists; on the PR head, the added slow-handler test can be source-reproduced by starting two dispatches and observing that only the second pending promise is released.

## Solution Assessment

No as submitted. The ALS implementation is a narrower approach than the earlier global Set, but the proposed test suite has a blocking async harness bug that must be fixed before merge.
