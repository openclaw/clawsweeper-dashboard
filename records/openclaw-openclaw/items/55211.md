## Summary

Keep open for maintainer merge review: the patch looks focused and correct, but it intentionally changes same-key hook redispatch from unbounded recursion to silent debug-logged suppression within one async lineage.

## What This Changes

The branch adds an AsyncLocalStorage-backed same-key guard around `triggerInternalHook` and tests recursive, sequential, concurrent, cross-session, and cross-action dispatch behavior.

## Best Possible Solution

Land the ALS guard if hook owners accept same async-lineage suppression; otherwise adjust the guard contract and add a regression test for delayed child redispatch.

## Reproduction Assessment

Yes from source inspection. Current main awaits hook handlers without a same-key guard, and the PR discussion includes copied Node output showing unguarded recursive amplification and guarded behavior.

## Solution Assessment

Mostly yes. ALS is a narrow fit for blocking same-chain recursion while preserving independent concurrent dispatches, but maintainers still need to accept the same-lineage delayed redispatch contract.
