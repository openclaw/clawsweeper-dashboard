- `merge-risk: 🚨 compatibility`: Merging changes hook dispatch semantics by dropping same-key triggers inherited through child async context, which could break handlers that rely on delayed same-key redispatch.

## Summary

Keep this PR open for human hook-contract review. The source path shows the reported recursion is possible on current main and the AsyncLocalStorage patch looks focused with no blocking code finding, but merging changes same-key hook redispatch semantics for async work spawned inside handlers.

## What This Changes

The PR adds an AsyncLocalStorage same-key re-entrancy guard to internal hook dispatch and regression coverage for recursive, sequential, concurrent, cross-session, and cross-action behavior.

## Best Possible Solution

Land the guard after hook owners accept or document the child-async-lineage suppression semantics; if delayed same-key redispatch must remain deliverable, narrow the guard to active recursion and add that regression case.

## Reproduction Assessment

Yes by source inspection. Current main and v2026.5.18 await hook handlers without an active same-key guard, so a handler can recursively call triggerInternalHook with the same type/action/sessionKey.

## Solution Assessment

Yes for the general implementation direction, with a compatibility decision still needed. AsyncLocalStorage is a narrow way to block same-chain recursion while preserving independent concurrent calls, but maintainers must accept its child-async-lineage behavior.
