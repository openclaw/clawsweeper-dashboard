## Summary

Keep open for maintainer review: the patch is focused and appears correct, but it changes the internal hook redispatch contract by silently skipping same-key calls inherited from a handler async lineage.

## What This Changes

This PR wraps `triggerInternalHook` in an `AsyncLocalStorage` same-key re-entrancy guard and adds internal-hook tests for recursive, sequential, concurrent, cross-session, cross-event, and slow-handler cases.

## Best Possible Solution

Land the ALS guard once hook owners accept same-key async-lineage suppression, or refine the semantics and tests if delayed child redispatch must still deliver.

## Reproduction Assessment

Yes. Current main's `triggerInternalHook` awaits handlers without an active same-key guard, so a handler that awaits the same hook again can recurse; the PR body also supplies copied Node output showing the before/after behavior.

## Solution Assessment

Mostly yes. `AsyncLocalStorage` is a maintainable fit for blocking same-chain recursion while preserving independent concurrent calls, but maintainers need to accept or refine delayed-child async suppression semantics.
