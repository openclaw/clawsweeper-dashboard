## Summary

No blocking code or security findings. This should stay open for maintainer hook-contract review because the patch deliberately suppresses same-key redispatch within one AsyncLocalStorage lineage, including delayed child work spawned by a handler.

## What This Changes

The PR adds an AsyncLocalStorage-backed re-entrancy guard to `triggerInternalHook` and regression tests for recursive, sequential, concurrent, cross-session, cross-action, and slow-handler dispatch behavior.

## Best Possible Solution

Land the ALS guard if hook owners accept same-lineage suppression; otherwise define queue, depth-limit, delayed redispatch, or hard-error semantics and cover that contract with a regression test.

## Reproduction Assessment

Yes from source inspection and supplied terminal output. Registering a same-key handler that awaits `triggerInternalHook` again recurses on current main, while the PR's proof shows the ALS guard blocks that path.

## Solution Assessment

Mostly yes. ALS is a narrow fit for blocking same-chain recursion while preserving independent concurrent dispatch, but the silent same-lineage suppression contract needs hook-owner acceptance before merge.
