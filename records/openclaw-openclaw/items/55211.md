## Summary

Keep this PR open: current main still dispatches internal hooks without a same-key active guard, and the patch is a focused fix, but external merge readiness still needs target-code real behavior proof plus a maintainer decision on the AsyncLocalStorage compatibility tradeoff.

## What This Changes

Adds an AsyncLocalStorage re-entrancy guard to `triggerInternalHook` and tests recursive, sequential, concurrent, cross-session, and cross-action dispatch behavior.

## Best Possible Solution

Land the guard after target-code real behavior proof is added and hook owners confirm the same-key child-lineage suppression contract, or narrow the guard if delayed child hooks must remain deliverable.

## Reproduction Assessment

Yes by source inspection: current main awaits each registered handler without a same-key guard, so a handler that calls `triggerInternalHook` again with the same type/action/sessionKey can recurse. I did not execute a repro because this review is read-only.

## Solution Assessment

Mostly yes: AsyncLocalStorage is a maintainable direction for same-chain recursion prevention while preserving independent top-level concurrent dispatch. The remaining question is whether suppressing same-key child async lineage is the intended compatibility contract.
