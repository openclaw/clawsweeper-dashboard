## Summary

Keep open for maintainer review: current main still has no active-dispatch guard, the PR now addresses prior concurrency and guard-lifetime feedback, and no blocking code or security findings remain. The remaining decision is whether nested same-chain same-key hook delivery should be intentionally suppressed.

## What This Changes

Adds an AsyncLocalStorage-based same-key re-entrancy guard around `triggerInternalHook` and expands internal hook tests for recursive, sequential, concurrent, cross-key, and delayed redispatch behavior.

## Best Possible Solution

Land the guard with regression tests if maintainers accept same-chain same-key suppression as the internal hook contract; otherwise request a different loop-prevention design that preserves nested delivery.

## Reproduction Assessment

Yes from source inspection: current main recursively invokes matching handlers with no active-dispatch guard, so a handler that triggers the same key again can re-enter indefinitely. I did not run a local repro because this review is read-only.

## Solution Assessment

Yes, with maintainer sign-off on the contract change: AsyncLocalStorage is a narrow fit because it blocks only same-chain re-entry while preserving independent concurrent dispatches. A broader config knob would only be preferable if nested same-key delivery must remain compatible.
