## Summary

Keep open for maintainer review: current main still has unguarded recursive internal hook dispatch, and this PR is a focused, likely correct fix with strong runtime proof, but it changes the same-key redispatch delivery contract for async work spawned from a handler lineage.

## What This Changes

This PR wraps `triggerInternalHook` in an AsyncLocalStorage same-key re-entrancy guard and adds regression coverage for recursive, sequential, concurrent, cross-session, and cross-event dispatch.

## Best Possible Solution

Land the ALS guard if hook owners accept same-key async-lineage suppression; otherwise define delayed follow-up delivery semantics and add focused regression coverage before merge.

## Reproduction Assessment

Yes, source-reproducible. Current main awaits hook handlers without a same-key active-dispatch guard, so a handler that calls `triggerInternalHook` again with the same `type:action:sessionKey` recurses through the same path.

## Solution Assessment

Mostly yes. AsyncLocalStorage is a maintainable way to block same-chain recursion while preserving independent concurrent dispatch, but the delayed child-async suppression semantics need maintainer acceptance.

## Risks / Open Questions
