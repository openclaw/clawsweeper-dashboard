## Summary

Keep open for maintainer review: the PR addresses a source-reproducible recursive hook loop with a focused AsyncLocalStorage guard and tests, but merging changes same-key redispatch semantics for async work spawned from a handler.

## What This Changes

Adds an AsyncLocalStorage same-key re-entrancy guard to `triggerInternalHook` and expands internal-hook tests for recursive, sequential, concurrent, cross-session, cross-event, and slow-handler cases.

## Best Possible Solution

Land the ALS guard after hook owners accept same-key async-lineage suppression, or refine the guard and tests if delayed child redispatch must still deliver.

## Reproduction Assessment

Yes. Current main's `triggerInternalHook` awaits handlers without an active same-key guard, so a handler that awaits the same hook again can recurse; the PR body also supplies live Node output showing before/after behavior.

## Solution Assessment

Mostly yes. AsyncLocalStorage is a maintainable fit for blocking same-chain recursion while preserving independent concurrent calls, but maintainers need to accept or refine the delayed-child async suppression semantics.
