## Summary

Keep open for maintainer review: current main still has unguarded recursive internal hook dispatch and the PR is focused with sufficient proof, but merging it changes the delivery contract for same-key redispatches inherited through an async handler lineage.

## What This Changes

The PR adds an AsyncLocalStorage same-key re-entrancy guard to `triggerInternalHook` and regression tests for recursive, sequential, concurrent, cross-session, and cross-event hook dispatch.

## Best Possible Solution

Land the ALS guard if hook owners accept suppressing same-key redispatches inherited from a handler async lineage; otherwise define the delayed-follow-up contract and add coverage before merge.

## Reproduction Assessment

Yes, source-reproducible. Current main awaits hook handlers without a same-key active-dispatch guard, and a handler that triggers the same `type:action:sessionKey` can recurse through the same dispatch path.

## Solution Assessment

Mostly yes. AsyncLocalStorage is a maintainable way to block same-chain recursion while preserving independent concurrent calls, but the delayed-child-async suppression semantics need maintainer acceptance.
