## Summary

Keep open: the fix is focused and source-reproducible, but the current AsyncLocalStorage store can silently suppress delayed same-key hook redispatches after the original dispatch has completed.

## What This Changes

The PR adds an AsyncLocalStorage-based same-key re-entrancy guard to `triggerInternalHook` and tests recursive, sequential, concurrent, cross-session, and cross-event dispatch.

## Best Possible Solution

Keep the same-key guard, but make its lifetime match the active dispatch and add coverage for delayed same-key redispatch after completion.

## Reproduction Assessment

Yes, source-reproducible. Current main awaits hook handlers without a same-key guard, and a handler that awaits `triggerInternalHook` again with the same event recurses through the same path.

## Solution Assessment

No, not as written. AsyncLocalStorage is a reasonable mechanism, but the guard key must be cleared when the active dispatch finishes so delayed follow-up events are not treated as still re-entrant.
