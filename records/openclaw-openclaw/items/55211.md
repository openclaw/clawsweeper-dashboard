## What This Changes

The PR adds an AsyncLocalStorage guard to `triggerInternalHook` and tests recursive, sequential, concurrent, cross-session, cross-event, and slow-handler dispatch behavior.

## Best Possible Solution

Land the ALS guard if hook owners accept same-key async-lineage suppression; otherwise add explicit delayed-child redispatch coverage and adjust the guard semantics before merge.

## Reproduction Assessment

Yes, source-reproducible. Current main awaits handlers without a same-key active-dispatch guard, so a handler that awaits the same hook can recurse; the PR body also supplies live Node before/after output.

## Solution Assessment

Mostly yes. AsyncLocalStorage is a maintainable way to block same-chain recursion while preserving independent concurrent calls, but maintainers need to accept or refine delayed-child async suppression semantics.
