## Summary

Keep open for maintainer review: the patch is focused and appears correct, with supplied terminal proof and no blocking code findings, but it intentionally changes same-key internal hook redispatch semantics and needs hook-owner acceptance before merge.

## What This Changes

The PR adds an AsyncLocalStorage-based same-key re-entrancy guard to `triggerInternalHook` and regression tests for recursive, sequential, concurrent, cross-session, and cross-event dispatch behavior.

## Best Possible Solution

Land the ALS guard if hook owners accept suppressing same-key redispatches inherited from a handler async lineage; otherwise define the delayed-follow-up contract and adjust the implementation and coverage before merge.

## Reproduction Assessment

Yes, source-reproducible. Current main awaits hook handlers without a same-key active-dispatch guard, and the PR body supplies copied Node output showing recursive amplification before the guard and suppression after it.

## Solution Assessment

Mostly yes. AsyncLocalStorage is a maintainable way to block same-chain recursion while preserving independent concurrent calls, but maintainers need to accept or refine delayed-child async suppression semantics.
