## Summary

Keep open for maintainer review: the patch is a focused and likely correct fix for a source-reproducible internal hook recursion bug, but it intentionally changes same-key redispatch semantics for hook handlers.

## What This Changes

This PR adds an AsyncLocalStorage-based same-key re-entrancy guard to `triggerInternalHook` and regression tests for recursive, sequential, concurrent, cross-session, and cross-event dispatch.

## Best Possible Solution

Land the ALS guard if hook owners accept same-key async-lineage suppression; otherwise define delayed follow-up delivery semantics and add a focused regression test before merge.

## Reproduction Assessment

Yes, source-reproducible. Current main awaits hook handlers without a same-key active-dispatch guard, and a handler that calls `triggerInternalHook` again with the same `type:action:sessionKey` recurses through the same path.

## Solution Assessment

Mostly yes. AsyncLocalStorage is a maintainable way to block same-chain recursion while preserving independent concurrent dispatch, but the delayed child-async suppression semantics need maintainer acceptance.
