## Summary

Keep open: current main still has unguarded recursive internal hook dispatch, and the PR’s AsyncLocalStorage direction is scoped to that bug, but the latest added slow-handler test can still hang because it releases only one of two pending dispatches.

## What This Changes

The PR adds an AsyncLocalStorage-based same-key re-entrancy guard to `triggerInternalHook` and expands `src/hooks/internal-hooks.test.ts` with recursive, sequential, and concurrent dispatch coverage.

## Best Possible Solution

Land the ALS recursion guard after making the concurrent slow-handler regression test deterministic and confirming the core hook lane is green.

## Reproduction Assessment

Yes, by source inspection: current main allows a same-key handler to call `triggerInternalHook` again with no active-dispatch guard. I did not execute the Vitest test in this read-only checkout, so this is source-reproducible rather than locally reproduced.

## Solution Assessment

No as submitted. The ALS runtime approach is narrower than the earlier global guard and preserves independent dispatches, but the added slow-handler test has a blocking async harness bug that should be fixed before merge.
