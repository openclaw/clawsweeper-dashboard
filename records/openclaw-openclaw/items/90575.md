## Summary

Keep open. Current main has same-state multi-agent OAuth inheritance, refresh serialization, and stale-shadow repair, but it does not provide the requested first-class shared auth-store/config surface across isolated `--profile` state directories; that design needs maintainer product and security review.

## What This Changes

Request to add config and CLI support for sharing one OAuth auth-profile store across multiple OpenClaw profiles or instances while keeping per-agent runtime auth state local.

## Best Possible Solution

Design a core auth-store sharing model that works with current SQLite/agentDir ownership, uses one cross-profile refresh lock namespace, keeps per-agent auth state local, and ships with doctor migration plus security docs before adding a config or CLI surface.

## Reproduction Assessment

Not applicable as a strict bug reproduction: this is a feature request for a new shared auth-store surface. Source and docs do give high-confidence evidence that current main only supports same-state main-agent read-through, not a configured cross-profile auth store.

## Solution Assessment

Unclear. The problem is valid, but the proposed `auth.profilesFile` JSON shape is probably not the best final form without maintainer design work because current main uses per-agent SQLite auth persistence and state-dir-scoped locks.
