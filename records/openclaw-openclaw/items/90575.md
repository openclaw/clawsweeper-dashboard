## Summary

Keep open. Current main and the latest release solve adjacent same-profile agent refresh races and legacy JSON symlink hazards, but they still do not provide a first-class cross-profile shared auth-store configuration surface; this remains a product/security design item.

## What This Changes

Request first-class config and CLI support for sharing one canonical model OAuth auth store across multiple named OpenClaw profiles or instances.

## Best Possible Solution

Design a maintainer-approved shared OAuth store for the current SQLite model, with explicit credential ownership, cross-profile locking, doctor migration or repair behavior, and public docs.

## Reproduction Assessment

Yes, source inspection gives a high-confidence way to reproduce the missing capability: create two separate --profile state roots and there is no config/schema/store resolver path that points both at one shared auth store.

## Solution Assessment

No for the literal JSON-file proposal: current main stores auth profiles in per-agent SQLite, so the maintainable solution needs an explicit shared SQLite/store owner, cross-profile lock design, doctor handling, and docs.
