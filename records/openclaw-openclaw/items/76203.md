## Summary

Keep open. Current main has provider-hook caching and active-registry reuse for part of the reported stream-setup cost, but minimal-profile tool construction still performs capability metadata availability work before final profile filtering, so the core pre-stream regression is not fully resolved.

## What This Changes

The issue asks OpenClaw to avoid several seconds of pre-stream latency by making built-in media/web provider discovery and provider hook resolution lazy, cached, or profile-aware.

## Best Possible Solution

Land a narrow performance fix that applies the resolved effective tool allowlist before media/image/PDF/web availability checks, preserves current provider-hook caches, and adds regression coverage proving minimal-profile runs do not load capability metadata for excluded tools.

## Reproduction Assessment

Yes for a source-level reproduction path: run a minimal-profile Gateway or embedded turn with the existing `recordToolPrepStage` timing callback and inspect `core-plugin-tools` while checking that excluded media/image/web availability paths are still entered. The exact reported seconds still need live macOS profiling.

## Solution Assessment

Unclear for the full broad fix because current main already addresses provider-hook and active-registry reuse. The best remaining fix is narrower: make built-in tool construction profile-aware before capability metadata or runtime provider discovery is performed.
