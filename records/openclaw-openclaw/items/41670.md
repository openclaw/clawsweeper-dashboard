This PR should close as superseded by #43910: the one-line hardcoded `encoding_format: "float"` change conflicts with current provider-compatibility policy, while the related open PR tracks the safer configurable/omit-by-default direction.

## What This Changes

The PR adds `encoding_format: "float"` to the remote embedding request body in `src/memory-host-sdk/host/embeddings-remote-provider.ts`.

## Best Possible Solution

Close this branch in favor of #43910 or an equivalent provider-scoped config/API path that preserves omit-by-default behavior and covers inline, batch, docs, and changelog surfaces.

## Reproduction Assessment

Yes for the request shape: current main sends OpenAI-compatible embedding bodies without `encoding_format` in the active package helper and OpenAI adapter. The reporter's internal proxy error was not live-tested, so the external failure is source-reproducible rather than live-reproduced.

## Solution Assessment

No. Hardcoding `float` in a shared helper is not the narrowest maintainable fix because current docs and implementation preserve omit-by-default behavior for providers that reject the field.
