## Summary

Keep open, but this PR should not merge as-is: the reported compatibility problem is real OpenAI API surface, while the patch now changes a generic helper instead of the current OpenAI memory embedding provider path.

## What This Changes

The PR adds `encoding_format: "float"` to the remote embedding request body in the memory host SDK helper.

## Best Possible Solution

Put the compatibility fix on the current OpenAI memory embedding path, or add a narrowly documented `encodingFormat` config option, with regression coverage for OpenAI remote `baseUrl` requests and no broad generic-helper behavior change without maintainer approval.

## Reproduction Assessment

No live reproduction against the internal proxy is available. Source inspection does show the active OpenAI embedding provider omits `encoding_format`, while this PR changes a helper that current OpenAI memory embeddings do not use.

## Solution Assessment

No. The one-line change is aimed at a valid API parameter, but on current main it lands in the wrong path for the reported OpenAI-compatible provider behavior and may need a configurable or OpenAI-specific fix instead.
