## Summary

Keep open. The PR targets a real current-main exec/message-delivery bug, but the proposed shell-script guard still has a concrete bypass for direct executable shell scripts without a shell-like suffix.

## What This Changes

The PR modifies exec preflight code, tests, exec docs, and the changelog to block OpenClaw message delivery commands from agent `exec`.

## Best Possible Solution

Keep the exec preflight boundary, extend direct script inspection to cover readable executable or shebang shell scripts without relying on file suffixes, and add focused regression coverage for that bypass.

## Reproduction Assessment

Yes. Source inspection of PR head shows a direct executable `./send` shell script without a `.sh`-style suffix is not returned by the suffix-only direct script extractor, so a body containing `openclaw message send` bypasses the new shell-file preflight.

## Solution Assessment

No. Blocking message delivery at the exec preflight boundary is the narrow maintainable direction, but this patch is incomplete until direct no-extension shell scripts are inspected or safely failed closed.

## Review Findings
