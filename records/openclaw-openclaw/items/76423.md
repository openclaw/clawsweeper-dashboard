## Summary

Keep open. The PR fixes the right boundary on current main, but the proposed shell-script coverage still misses direct extensionless executable shell scripts, leaving a practical bypass for the new message-delivery guard.

## What This Changes

The PR adds exec preflight parsing to reject OpenClaw message delivery commands, plus focused tests, exec docs, and a changelog entry.

## Best Possible Solution

Keep the exec preflight boundary, but extend it to inspect direct executable shell scripts without shell-like filename extensions or explicitly narrow the documented guarantee.

## Reproduction Assessment

Yes. Current main source shows `exec` does not block `openclaw message send`, and the PR bypass is source-reproducible because `./send` without a shell-like extension is not selected for shell-script preflight.

## Solution Assessment

No. The proposed boundary is the right place to solve this, but the implementation should also cover extensionless direct shell scripts before it can reliably claim shell-script delivery commands are blocked.
