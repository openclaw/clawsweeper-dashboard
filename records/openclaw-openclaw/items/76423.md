## Summary

Keep open. Current main still lacks an exec guard for OpenClaw message delivery commands, while this PR adds a focused preflight guard with docs, changelog credit, and broad regression coverage; I found no blocking correctness or security issue on the latest head.

## What This Changes

The PR adds exec preflight parsing to reject OpenClaw message delivery commands across direct CLI, package-runner, shell-wrapper, stdin, and shell-script forms, with regression tests plus exec docs and changelog credit.

## Best Possible Solution

Land the PR after the remaining exact-head checks finish, keeping the block at the exec preflight boundary while preserving normal chat replies, the message tool, and direct CLI message behavior outside exec.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: on current main, an agent `exec` command containing `openclaw message send` reaches the message CLI delivery path because the unsafe exec guard only blocks approvals and channel login.

## Solution Assessment

Yes. The exec preflight boundary is the narrowest maintainable fix because the message CLI should remain usable outside `exec`, and the PR preserves non-delivery commands such as help, dry-run, read, search, and thread list.
