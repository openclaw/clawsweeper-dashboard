## Summary

Keep open. The PR targets a valid exec/message-delivery bug, but the current patch still misses direct executable shell scripts without a `.sh`/similar suffix, so a normal `./send` shebang script can bypass the new file preflight.

## What This Changes

The PR adds exec-side parsing, shell-script preflight checks, tests, docs, and a changelog entry to block OpenClaw message delivery commands from agent `exec`.

## Best Possible Solution

Keep the exec preflight boundary, extend shell-script inspection to executable or shebang shell scripts without relying only on filename suffixes, then rebase and land after exact-head checks.

## Reproduction Assessment

Yes. Source inspection shows current main lets `openclaw message send` reach the message CLI delivery path from exec, and the PR-head bypass follows directly from the suffix-only direct shell-script target extractor.

## Solution Assessment

No. The exec preflight boundary is the right place to solve this, but the current implementation is incomplete because direct no-extension executable shell scripts skip the new shell-file inspection.
