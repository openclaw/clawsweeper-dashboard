## Summary

Keep open. The PR targets a real exec boundary bug and the direction is right, but PR head still has a concrete bypass: direct path-like shell scripts with arbitrary dotted names are not inspected before execution, so a shebang script such as `./send.v1` can still run `openclaw message send` through agent `exec`.

## What This Changes

The PR adds exec preflight logic, tests, docs, and a changelog entry to block OpenClaw message delivery commands from agent `exec` while preserving non-delivery message CLI forms.

## Best Possible Solution

Keep the exec preflight boundary, inspect path-like direct shell script targets regardless of arbitrary dotted basenames or by shebang, add a focused regression test, then rerun the targeted exec checks.

## Reproduction Assessment

Yes. Source inspection shows current main lets message CLI delivery reach `messageCommand` from exec, and PR head still lets a direct executable `./send.v1` shell script skip file inspection before execution.

## Solution Assessment

No. Blocking OpenClaw message delivery at the exec preflight boundary is the narrow maintainable fix, but this implementation is incomplete until dotted path-like direct shell scripts are inspected.

## Review Findings
