## Summary

Keep open. The PR targets a valid current-main exec/message-delivery gap, but the patch still misses direct executable shell scripts without a shell-looking suffix, so it is not merge-ready.

## What This Changes

The PR adds exec preflight parsing, tests, docs, and a changelog entry to reject OpenClaw message delivery commands from agent `exec`.

## Best Possible Solution

Keep the exec preflight boundary, extend direct shell-script inspection beyond suffix-only detection, and add regression coverage for a shebang shell script named without `.sh` or similar suffix.

## Reproduction Assessment

Yes. Source inspection shows current main can reach `runExecProcess` for `openclaw message send`, and PR-head source shows a direct no-extension shell script skips the new file preflight.

## Solution Assessment

No. Blocking message delivery at exec preflight is the right boundary, but this implementation is incomplete because direct executable shell scripts are recognized only by filename suffix.

## Review Findings
