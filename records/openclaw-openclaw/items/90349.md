## Summary

Keep open. Current main and v2026.6.8 have model context-window resolution, pre-prompt overflow compaction, compaction tuning, skill prompt limits, Tool Search, and opt-in localModelLean mitigation, but they still do not provide the requested automatic relevance-based prompt, skill, memory, and tool compactor for small-context models.

## What This Changes

Request automatic context-budget compaction for small-context models, including model-window detection, prompt budget allocation, relevance-based skill/prompt/memory trimming, summarization, and compactness levels.

## Best Possible Solution

Design a maintainer-approved context-budget policy that composes the existing budget, compaction, Tool Search, skills, memory, and context-engine surfaces before adding any new compactness modes or defaults.

## Reproduction Assessment

Not applicable; this is a feature request rather than a current-main bug report. Source inspection and related issues show partial capabilities and remaining gaps, but this issue does not provide a failing run to reproduce.

## Solution Assessment

No; a monolithic context-compactor with compactness levels is not yet proven to be the best permanent shape. The safer path is a maintainer-owned design that reuses existing context budget, compaction, tool, skill, and memory surfaces.
