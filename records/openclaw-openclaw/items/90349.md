## Summary

Keep open. Current main already has several context-budget mitigations, but the requested automatic compactness policy with relevance-based prompt, skill, memory, and tool-surface slimming is not fully implemented as one product behavior.

## What This Changes

The issue requests model-aware context-budget management that can automatically slim prompts, skills, memory, and conversation history for smaller context-window models.

## Best Possible Solution

Design a maintainer-approved core context-budget policy that composes existing contextTokens, compaction, Tool Search/localModelLean, skills limits, and memory/contextLimits surfaces before adding any new compactness config.

## Reproduction Assessment

Not applicable; this is a feature request, not a bug report with a high-confidence failing current-main reproduction path. Source review shows adjacent mitigations exist, but no specific 8K-32K model/config failure was provided.

## Solution Assessment

No; a new monolithic context-compactor with full/compact/minimal levels is not yet proven to be the best shape. The safer path is a product design that reuses the existing context-budget surfaces and only adds new config where current knobs cannot express the behavior.
