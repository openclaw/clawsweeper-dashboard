## Summary

Keep open. Current main has important partial mechanisms for context windows, overflow compaction, `/compact`, context engines, split subsystem budgets, skill prompt caps, and opt-in local-model lean mode, but it still does not provide the requested automatic relevance-based compactor across prompt sections, skills, memory, tools, and compactness levels.

## What This Changes

The issue requests a model-aware context compactor that slims prompt sections, skills, memory, tools, and conversation context for smaller models.

## Best Possible Solution

Design a maintainer-approved model-aware context-budget policy that composes existing context-window, compaction, context-engine, Tool Search, skill, memory, and localModelLean surfaces before adding compactness modes or defaults.
