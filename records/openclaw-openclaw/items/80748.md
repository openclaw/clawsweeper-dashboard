## Summary

Keep open. The report is credible and not fully resolved by current main: main now hides one subagent-announce history case, but the broader async/internal-system provenance path described here is still visible from source and the latest shipped tag is older than the current projection change.

## What This Changes

The issue asks OpenClaw to stop persisting or rendering internal provenance wrappers, runtime context blocks, and async completion/heartbeat instructions in Control UI, TUI, and external chat surfaces.

## Best Possible Solution

Finish a provenance-based sanitizer that tags internal heartbeat/cron/exec-originated turns at source, hides those internal-system turns from all user-visible history/projection paths, and preserves intentionally visible `sessions_send` content unless it is an internal announce/completion path.

## Reproduction Assessment

Yes, from source rather than a live run: the async completion path still creates the reported internal text, the heartbeat runner does not attach `internal_system` provenance, and current projection only drops the narrower subagent-announce inter-session case.

## Solution Assessment

No, current main is only a partial fix. The maintainable direction is source tagging plus shared display/delivery projection filtering, not broad string stripping or hiding every inter-session message.
