## Summary

Current main already contains the central history projection and prompt-persistence suppression needed for the reported internal provenance leak; the important caveat is that this is current-main-only and not included in v2026.5.7.

## What This Changes

The issue reports that internal provenance, inter-session, runtime context, and async completion text still appears in user-visible chat on release v2026.5.7.

## Best Possible Solution

Keep the current main projection and persistence fix, then let the next release carry it; any remaining channel-delivery leak should be filed with fresh current-main logs.

## Reproduction Assessment

No current-main failing reproduction path remains from source inspection: the reported wrapper and runtime-context row are now filtered by the shared chat projection and covered by a regression test. The reporter's v2026.5.7 release reproduction remains credible, but that release predates the current-main proof.

## Solution Assessment

Yes: centralizing visible-history filtering in `chat-display-projection` and suppressing persistence for subagent announce task-completion prompts is the narrow maintainable fix. Broadly stripping every inter-session prefix from all model output would be riskier because `sessions_send` has documented transcript provenance semantics.
