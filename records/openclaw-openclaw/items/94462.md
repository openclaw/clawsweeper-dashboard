## Summary

## What This Changes

The PR changes `src/agents/bash-tools.shared.ts` so `resolveWorkdir()` expands a leading `~` with `resolveUserPath()` before checking whether the workdir exists.

## Best Possible Solution

Close this weaker duplicate and continue review on the viable linked candidates, with https://github.com/openclaw/openclaw/pull/94449 carrying the clean OS-home expansion path and https://github.com/openclaw/openclaw/pull/94441 carrying the fail-fast alternative.

## Solution Assessment

No. Expanding before `statSync()` is the right layer, but this branch uses the OpenClaw-home resolver; the stronger linked expansion PR uses OS-home semantics and adds the missing regression coverage.
