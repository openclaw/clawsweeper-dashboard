## Summary

Keep open: current main and the latest release still pass bare `claude` from the Anthropic CLI backend into a supervisor allowlist that does not append `.cmd`, and an open linked PR already targets this issue. Closing before a merged fix would leave the Windows Claude CLI provider blocked.

## What This Changes

The issue reports that Windows 11 Claude CLI provider launches fail with `spawn claude ENOENT` even though Claude Code is installed and works from the shell.

## Best Possible Solution

Land one canonical Windows supervisor shim fix that resolves affected CLI shims and launches `.cmd`/`.bat` through a safe invocation path, then close this issue and the canonical duplicate from the merged implementation.

## Reproduction Assessment
