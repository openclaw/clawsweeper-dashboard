## What This Changes

The issue reports Windows 11 Claude CLI provider requests failing before model execution with `spawn claude ENOENT` even though Claude Code is installed and works on PATH.

## Best Possible Solution

Land or replace one canonical Windows supervisor shim fix so the spawn boundary resolves npm-installed CLI backend commands such as `claude` before spawning them on Windows.

## Reproduction Assessment

Yes. The current-main path is source-reproducible: the Claude backend supplies bare `claude`, and the supervisor Windows shim resolver does not include `claude`, so it remains a bare spawn target on Windows.
