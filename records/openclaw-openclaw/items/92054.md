## Summary

## What This Changes

Make the Claude CLI provider launch correctly on Windows when npm installs it as a `.cmd` command shim.

## Best Possible Solution

Land one canonical fix that extends the existing trusted Windows shim-resolution path for bundled CLI backends, includes focused regression coverage, and proves a real Claude supervisor launch on Windows without broad shell parsing.

## Reproduction Assessment

Yes. The reporter gives a concrete Windows 11 command-level reproduction, and current main shows the bare `claude` command bypassing the supervisor’s `.cmd` resolution list.

## Solution Assessment

Yes for the narrow direction: reuse the existing Windows shim helper and add regression coverage. The broader shell-mode alternative is not the best default because it introduces unnecessary command-parsing and security-sensitive behavior.
