## Label Justifications

- `P1`: The regression blocks every Claude CLI-backed agent request for the affected Windows configuration.

## Summary

Current main still omits the bundled Claude CLI from the Windows command-shim resolution path, and the reporter provides a concrete Windows 11 reproduction. This issue should remain open because https://github.com/openclaw/openclaw/pull/92119 explicitly closes it but has not merged, while https://github.com/openclaw/openclaw/pull/91490 proposes a competing repair with different batch-file execution semantics.

## What This Changes

Fix Windows execution of the npm-installed Claude CLI command so the provider no longer fails with `spawn claude ENOENT` before model execution.

## Best Possible Solution

Land one canonical repair that resolves the bundled CLI shim and proves a complete Claude CLI-backed agent turn on supported Windows and Node versions, with focused supervisor-level regression coverage and no unnecessary shell expansion.

## Reproduction Assessment

Yes. The reporter supplies a repeatable Windows 11 command sequence and an independent Node spawn comparison, and current-main source confirms the bare Claude command bypasses the existing `.cmd` allowlist.

## Solution Assessment

Not yet. Adding Claude to the allowlist addresses discovery, but maintainers should select the narrowest implementation that also passes a real Windows launch and full provider-turn proof without introducing unnecessary shell behavior.
