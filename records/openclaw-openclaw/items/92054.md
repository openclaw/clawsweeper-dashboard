## What This Changes

The issue reports that Windows 11 users with Claude Code installed cannot run the Claude CLI provider because OpenClaw spawns bare `claude` and fails with ENOENT.

## Best Possible Solution

Review and land one canonical generic Windows supervisor spawn fix that resolves npm-style CLI shims and invokes `.cmd`/`.bat` through a trusted wrapper or shared helper, with regression tests and redacted Windows proof.

## Reproduction Assessment

Yes, at source level. Current main passes bare `claude` from the Anthropic CLI backend into a supervisor adapter that only suffixes package-manager shims, and the reporter supplied Windows 11 ENOENT evidence.

## Solution Assessment

No, cleanup closure is not the best solution while current main and the latest release still contain the failing path. The best path is to choose and land a safe linked Windows batch-shim fix, then close this issue from the merged implementation.
