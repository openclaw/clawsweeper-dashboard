## Summary

The prior canonical fix PR was closed unmerged by its author on June 8, 2026, so no viable open implementation currently owns the regression.

## What This Changes

Fix Claude CLI provider execution on Windows where direct process spawning cannot resolve the npm-installed `claude` command shim.

## Best Possible Solution

Adopt the no-shell shared Windows resolver approach from the closed fix PR, resolving npm-installed CLI shims to a safe executable invocation while preserving argv boundaries and caller `PATH`/`PATHEXT` semantics.

## Reproduction Assessment

Yes at source level with high confidence: current main leaves the bundled `claude` command bare on Windows, exactly matching the reporter's direct-spawn ENOENT and two earlier independent reports. Native Windows execution remains the final post-fix validation step.

## Solution Assessment

Yes in direction: a shared no-shell executable resolver is safer and more maintainable than a Claude-only allowlist or blanket `shell: true`, and the closed prior PR demonstrates a bounded implementation and validation plan.
