## Summary

Keep open: this is a real cron data-loss fix path, but current head still lets a successful manual due run clear the preserved one-shot job's `nextRunAtMs` during post-run maintenance, and the current proof does not show that path surviving end to end.

## What This Changes

The branch adds manual-run discrimination to cron result handling, persists `lastRunWasManual` in cron state and gateway schemas, and expands cron regression/failure-alert/restart coverage.

## Best Possible Solution

Preserve the scheduled slot for manual terminal runs through the post-run recompute path, keep scheduled-run delete/backoff behavior unchanged, and prove current head with focused tests plus redacted CLI output.

## Reproduction Assessment

Yes. Source inspection shows `finishPreparedManualRun` applies the manual result and then runs expired maintenance; for a successful due at-job, that maintenance path treats the manual run as the scheduled execution and can clear `nextRunAtMs`.

## Solution Assessment

No. The shared `isManual` flag is the right layer, but the implementation is incomplete because recompute still treats a manual successful due at-job as an executed scheduled slot.
