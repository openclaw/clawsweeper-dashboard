## Summary

Keep this PR open: current main and v2026.6.8 still route manual cron completion through the scheduled `deleteAfterRun` and retry/counter state path, and this branch remains the live candidate fix for the linked P1 cron data-loss/session-state bug. ## What This Changes

The PR adds manual-run awareness to cron result application, preserves scheduled cron state for manual runs, records `lastRunWasManual`, updates gateway cron state schemas, and expands cron regression coverage.

## Best Possible Solution

Land this PR after explicit maintainer acceptance that manual cron runs are verification/debug operations that must not mutate scheduled retry, deletion, or alert state.
