## Label Justifications

- `P1`: The PR targets a destructive cron bug where manual runs can consume one-shot `deleteAfterRun` jobs and lose scheduled future fires.
- `merge-risk: 🚨 compatibility`: The diff changes the accepted gateway cron state shape and manual-run failure-alert semantics that existing operators may observe.
- `merge-risk: 🚨 session-state`: The patch changes persisted cron run state, restart catch-up eligibility, and backoff gating based on whether the last run was manual.

## Summary

This PR is still necessary: current main and the latest release still route manual cron completions through `applyJobResult` without a manual-run discriminator, while the linked destructive `deleteAfterRun` issue remains open. The branch has strong proof and no concrete line-level blocker, but it should stay open for maintainer acceptance of the new persisted/protocol state field and manual failure-alert semantics.

## What This Changes

The PR marks manual cron completions, prevents manual runs from consuming `deleteAfterRun` one-shot jobs or scheduled retry/backoff counters, adds `lastRunWasManual` to cron state, and expands cron regression coverage.

## Best Possible Solution

Land the PR after cron/protocol owners explicitly accept `lastRunWasManual` as the durable state marker and confirm manual-run alert suppression is the intended operator behavior.

## Reproduction Assessment

Yes from source: current main and v2026.6.5 both call `applyJobResult` for manual runs without an `isManual` discriminator, so a manual successful at-job can satisfy the same `deleteAfterRun` condition as a scheduled run. I did not run tests because this review is read-only.

## Solution Assessment

Yes for the central bug: marking manual completions at the `applyJobResult` boundary is the narrow owner-layer fix because both manual completion paths already converge there while timer-driven paths remain unchanged. The only unresolved part is maintainer acceptance of the public state marker and manual alert policy, not a concrete code defect.
