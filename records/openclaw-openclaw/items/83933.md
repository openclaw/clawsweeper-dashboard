## Summary

Keep open: the patch is a plausible fix for the destructive manual cron-run behavior, but it changes compatibility-sensitive cron semantics and adds persisted/gateway state. The remaining blockers are maintainer acceptance of the non-consuming manual-run contract and stronger current-head proof of a successful manual `deleteAfterRun` Gateway run preserving the job and schedule.

## What This Changes

The PR adds manual-run awareness to cron result application, preserves jobs/counters/backoff state for manual runs, records `lastRunWasManual`, updates gateway cron state schemas, and adds cron regression tests.

## Best Possible Solution

Land a maintainer-accepted version that makes manual cron runs non-destructive, preserves scheduled state by default, and proves client/schema tolerance for the additive manual-run state field.
