## Summary

Keep open. The PR is a focused fix for a real cron bug, but current head still does not prove or fully protect the successful manual due-run path, and it adds a protocol-visible cron state field that needs explicit upgrade/compatibility attention before merge.

## What This Changes

The PR marks manual cron runs with `isManual`, preserves deleteAfterRun/counter/backoff state for manual runs, records `lastRunWasManual`, extends the cron protocol state schema, and adds cron regression coverage.

## Best Possible Solution

Preserve manual-run isolation through both `applyJobResult` and post-run maintenance recompute, add a full successful manual due-run regression, and either get maintainer acceptance for the additive protocol field or keep the scheduler marker out of public cron state.
