## What This Changes

The PR adds an `isManual` result-application path for manual cron runs so `deleteAfterRun` and one-shot state mutation are skipped, with cron regression tests.

## Best Possible Solution

## Reproduction Assessment

Yes by source inspection, though not by a live run in this read-only review. Current main routes manual cron completion through `applyJobResult` without a manual discriminator, and that function deletes successful `deleteAfterRun` one-shot jobs or disables one-shot ok/skipped jobs.

## Solution Assessment

Not yet. The manual-run discriminator is the right shape for the core bug, but manual error alert side effects must be suppressed before rollback and the PR needs real queued-run proof.
