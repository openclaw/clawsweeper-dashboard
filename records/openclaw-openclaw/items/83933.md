## Summary

Preserve the scheduled at-slot through the full manual finish and recompute path, keep the new state marker only if maintainers accept the compatibility shape, and require redacted current-head proof for a successful manual `deleteAfterRun` run before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve manual due slots through post-run recompute:** `src/cron/service/ops.ts:949`
  - body: Passing `isManual: true` into `applyJobResult` preserves the job only until `finishPreparedManualRun` immediately calls `recomputeNextRunsForMaintenance(..., { recomputeExpired: true })`. For a successful manual due `at` job, `lastRunAtMs` is now at or after the scheduled `nextRunAtMs`, so maintenance treats that scheduled slot as already executed and clears it; the job remains enabled with no future wake, which can still silently lose the scheduled fire this PR is meant to protect. Please keep the manual-run slot out of expired recompute or teach recompute about `lastRunWasManual`, and add a regression that exercises the full manual finish path.
  - confidence: 0.91
