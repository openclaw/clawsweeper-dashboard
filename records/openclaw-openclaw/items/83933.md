## Summary

Keep open: current main still has the manual-run deletion bug, and the PR head still has a blocking manual due-run state defect before it can safely land.

## What This Changes

The PR changes cron manual run completion so manual runs set `lastRunWasManual`, skip `deleteAfterRun`, avoid scheduled retry/backoff/failure-alert mutations, add that state field to the gateway cron schema, and expand cron regression coverage.

## Best Possible Solution

Preserve the manually-run job's scheduled slot through manual post-run maintenance, keep scheduled runs' existing delete/retry semantics unchanged, and provide current-head redacted CLI/Gateway proof for successful and error manual `deleteAfterRun` cases.

## Reproduction Assessment

Yes. Source inspection at PR head shows a successful manual due run can set `lastRunWasManual` but still flow through expired-slot maintenance recompute, which can clear an `at` job's due `nextRunAtMs`.

## Solution Assessment

No. The shared `applyJobResult` discriminator is the right layer, but the manual finish path also needs to preserve the just-run scheduled slot through maintenance recompute and prove the real CLI/Gateway path at current head.
