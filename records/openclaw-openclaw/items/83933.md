## Summary

The PR fixes a real destructive cron issue, but it is not merge-ready because the current merge-ref cron-service shard fails existing disabled-heartbeat one-shot tests after `cron run --due` is also marked manual and bypasses scheduled retry/disable state.

The branch marks manual cron completions with `isManual`, preserves one-shot jobs and retry/alert counters for manual runs, adds `lastRunWasManual` to cron state/protocol, and expands cron regression coverage.

## Best Possible Solution

Land a version that fixes non-destructive force/debug manual runs while preserving `cron run --due` scheduled retry and disable semantics, with protocol/state upgrade proof.

## Solution Assessment

No; preserving deleteAfterRun jobs for manual/debug runs is the right direction, but applying the same state-isolation rule to `cron run --due` is too broad against current tests and CLI semantics.
