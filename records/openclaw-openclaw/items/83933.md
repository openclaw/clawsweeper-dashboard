## Summary

Keep this PR open for maintainer review: the patch appears to fix the reported cron data-loss path with strong proof and no blocking code findings, but it intentionally changes persisted manual-run semantics and needs explicit maintainer acceptance before merge.

## What This Changes

The PR adds an `isManual` option to cron result application, passes it from manual completion paths, and expands cron regression/failure-alert tests so manual runs no longer consume one-shot jobs or mutate scheduled retry, backoff, skip, disable, and alert state.

## Best Possible Solution

Land this only after a maintainer accepts non-consuming manual cron runs as the product contract, with release-note context for the changed operator behavior.
