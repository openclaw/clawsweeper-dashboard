## What This Changes

This PR adds an `isManual` path to cron result handling so manual runs preserve one-shot jobs and scheduled retry/backoff state, then updates cron regression and failure-alert coverage.

## Best Possible Solution

Land manual-run isolation after the cron docs state the exception and maintainers explicitly accept the compatibility shift.

## Solution Assessment

Yes with a merge gate: adding `isManual` at result application is the right narrow seam, and latest head covers at-job, recurring-job, counter, and alert-cooldown state. The remaining gap is documentation plus maintainer acceptance of the compatibility change.

- Docs still say one-shot jobs delete after success without explaining the manual-run exception, which can confuse users after this behavior changes.
