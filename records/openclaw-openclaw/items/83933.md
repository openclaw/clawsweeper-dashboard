## What This Changes

The branch adds an isManual option to cron result application, passes it from manual cron run paths, and adds timer/failure-alert regression tests around deleteAfterRun, counters, alerts, and at-job state.

## Best Possible Solution

Land a manual-run boundary that preserves deleteAfterRun jobs and scheduled retry/backoff/cooldown/next-run state for every schedule kind, with regression tests and redacted live CLI proof showing manual run preservation and scheduled deletion control.

## Solution Assessment

No. The isManual direction is the right boundary, but the current patch does not apply it to non-at error backoff and still needs real CLI/Gateway proof before merge.
