## What This Changes

The branch adds an `isManual` option to cron result application, passes it from manual run completion paths, and expands timer and failure-alert regression coverage for non-destructive manual runs.

## Best Possible Solution

Land only after maintainers accept manual cron runs as non-destructive verification that records the last outcome but does not advance scheduled execution state.

Yes. Current main is source-reproducible because `applyJobResult` has no manual discriminator and both manual completion paths call it without one; the related report supplies the observable `deleteAfterRun` loss scenario.

## Solution Assessment

Yes, subject to maintainer product acceptance. Putting the discriminator in shared result application and setting it only from manual completion paths is the narrow maintainable fix; the unresolved part is the compatibility-sensitive contract, not a code defect I can point to.
