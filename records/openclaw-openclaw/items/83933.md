## Summary

Keep open: the patch is a plausible cron bug fix, but it changes compatibility-sensitive persisted cron state and manual-run semantics, and it still needs current-head real behavior proof for the successful manual `deleteAfterRun` path before merge.

## What This Changes

The PR marks manual cron result application with `isManual`, prevents manual runs from deleting one-shot jobs or changing scheduled retry/alert state, adds `lastRunWasManual` to cron state/protocol schema, and expands cron tests.

## Best Possible Solution

Land this only after maintainers accept the manual-versus-scheduled cron state contract and the contributor adds redacted current-head success-path CLI/log proof for a manual `deleteAfterRun` run.
