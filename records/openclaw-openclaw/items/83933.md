## Summary

## What This Changes

The branch adds an `isManual` option to cron result application, passes it from manual run paths, and adds regression tests for preserving one-shot `deleteAfterRun` jobs during manual runs.

## Best Possible Solution

Keep the manual-run discriminator, move manual error handling before failure-alert/backoff mutation, and merge only after redacted queued-run proof shows the job remains scheduled.

## Reproduction Assessment

Yes, by source inspection rather than live execution. Current main sends manual cron completion into `applyJobResult` without a manual flag, and successful one-shot `deleteAfterRun` jobs return the deletion path there.

## Solution Assessment

No, not yet. The manual discriminator is the right direction, but manual error handling still needs to avoid alert/cooldown side effects before common failure-alert logic runs, and the branch needs real queued-run proof.
