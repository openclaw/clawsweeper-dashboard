## What This Changes

## Best Possible Solution

Keep the manual-run discriminator, prevent manual error and skipped outcomes from mutating failure-alert/backoff state before emission, and merge after redacted real CLI proof shows the scheduled one-shot remains.

## Reproduction Assessment

Yes, by source inspection rather than live execution. Current main sends manual cron completion into `applyJobResult` without a manual flag, and successful one-shot `deleteAfterRun` jobs return the deletion path there.

No, not yet. The `isManual` seam is the right place, but the current patch handles manual error preservation after failure-alert side effects have already run.
