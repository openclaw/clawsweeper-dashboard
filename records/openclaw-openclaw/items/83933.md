## Summary

## What This Changes

The PR adds an `isManual` option to cron result application, passes it from manual cron run paths, and adds regression tests so manual one-shot runs preserve scheduled jobs while scheduled runs still delete them.

## Best Possible Solution

Keep the manual-run discriminator, move preserved manual one-shot handling before failure-alert/backoff/skip alert mutation, and merge only after redacted real CLI proof shows the job remains scheduled.
