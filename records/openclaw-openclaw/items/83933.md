## Best Possible Solution

Land a cron-owned fix that preserves scheduled state for manual runs, with maintainer acceptance of whether `lastRunWasManual` should remain visible and patchable in the Gateway cron schema.

## Reproduction Assessment

Yes at the source level: current main and v2026.5.28 have no manual discriminator in `applyJobResult`, so manual completions flow through the same delete/counter/backoff logic as scheduled completions. I did not run a live repro because this review is read-only, but the PR body includes Docker Gateway CLI after-fix proof.

## Solution Assessment

Likely yes, subject to maintainer acceptance of the additive cron state field: the fix sits at the shared result-application boundary used by manual and scheduled runs rather than duplicating CLI behavior. A safer alternative, if public patchability is not desired, is to keep the marker internal while still excluding manual runs from scheduled-state effects.
