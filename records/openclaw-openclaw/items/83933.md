## Summary

## What This Changes

The PR adds an `isManual` option to cron result handling, passes it from manual cron run paths, and adds regression tests for preserving `deleteAfterRun` one-shot jobs during manual runs.

## Best Possible Solution

Keep the `isManual` seam, suppress manual one-shot deletion/disable/retry/alert state mutations before failure-alert bookkeeping, and merge after redacted `openclaw cron run`/`cron get`/`cron runs` proof.

## Solution Assessment

No, not yet. The `isManual` discriminator is the right seam, but it needs to guard manual one-shot alert/counter/cooldown mutations before stateful failure handling and then prove the CLI flow.
