## Summary

Keep open: the PR fixes the main manual-run delete/counter bug path, but it still needs one scheduler-state repair before merge because preserved manual at-jobs can be skipped by restart catch-up after the scheduled slot is missed.

## What This Changes

This PR marks manual cron result application as manual so manual runs preserve delete-after-run jobs and avoid mutating scheduled retry, skipped, backoff, disable, and alert state.

## Best Possible Solution

Keep the manual-run isolation design, but separate manual outcome observability from scheduler-owned terminal state or teach startup catch-up that preserved manual at-job state remains eligible when nextRunAtMs is after the manual run.
