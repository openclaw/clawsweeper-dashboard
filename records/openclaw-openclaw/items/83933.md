- **[P1] Preserve successful manual due slots after recompute:** `src/cron/service/ops.ts:948`
  - body: Passing `isManual: true` into `applyJobResult` preserves the job only until the later maintenance recompute. For a successful manual due `at` job, `lastRunAtMs` is now at or after the scheduled `nextRunAtMs`, so `recomputeNextRunsForMaintenance(..., { recomputeExpired: true })` can recompute that slot to `undefined`; the job remains enabled but loses the scheduled fire the PR is meant to protect. Please carry the manual exemption through this recompute path and add a service-level `cron.run(job.id, "due")` success regression for a `deleteAfterRun` at-job.
