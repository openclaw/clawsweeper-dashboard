## Best Possible Solution

Land the manual-result boundary after all scheduled retry and failure-alert state is preserved for manual runs and redacted patched Gateway/CLI proof shows manual runs preserve the job while scheduled runs still delete it.

## Reproduction Assessment

Yes, source-reproducible. Current main manual-run completion reaches `applyJobResult` without a manual flag, and `applyJobResult` deletes successful `deleteAfterRun` at-jobs regardless of manual versus scheduled origin.

## Solution Assessment

No, not yet. The `isManual` boundary is the right shape, but the implementation should also preserve failure-alert cooldown state and be backed by live patched Gateway/CLI proof.
