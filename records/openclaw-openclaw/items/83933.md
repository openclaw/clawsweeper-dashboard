## What This Changes

The PR marks cron manual-run completions as manual, prevents manual runs from consuming `deleteAfterRun` or scheduled retry/alert counters, adds `lastRunWasManual` to cron state/protocol shape, and expands cron regression coverage.

## Best Possible Solution

## Solution Assessment

Yes, this is a good owner-boundary fix: both manual completion paths converge on `applyJobResult`, while scheduled timer paths can omit `isManual` and keep existing behavior. The tradeoff is the additive persisted/protocol marker, which needs maintainer acceptance rather than an automated repair.
