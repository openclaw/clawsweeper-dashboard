## Summary

Keep open: current main and the latest release still share manual and scheduled cron outcome handling, so the linked bug is not implemented on main. This PR remains a meaningful, proof-backed fix candidate; I found no line-level blocking defect, but merge still needs maintainer acceptance of the compatibility-sensitive manual-run cron state contract.

## What This Changes

The PR adds manual-run discrimination to cron outcome handling, preserves scheduled state for manual runs, exposes `lastRunWasManual`, and updates cron regression coverage.

## Best Possible Solution

Land this PR after required CI finishes if maintainers accept manual runs as non-consuming verification; otherwise narrow the change to the accepted cron manual-run contract before merge.
