## Summary

Keep open: manual npm global install on native Windows is a documented path, and the report gives concrete evidence of a broken package tree; current main confirms the entrypoint is expected but does not explain or fix this partial install state.

## What This Changes

The issue asks OpenClaw to handle a Windows `npm install -g openclaw@2026.5.4` that leaves the global package directory without `openclaw.mjs` or `package.json`, causing the CLI shim to fail with `MODULE_NOT_FOUND`.

## Best Possible Solution

A reliable Windows global npm install should either populate the full `openclaw` package tree including `openclaw.mjs` or fail/recover through a documented path without leaving a broken shim.

## Reproduction Assessment

Unclear/no high-confidence reproduction yet. The issue includes a concrete Windows/npm failure path and logs, but this read-only sweep did not run a Windows global install; current source only confirms that `openclaw.mjs` is expected to exist.

## Solution Assessment

Unclear. Current main already declares and verifies the packaged entrypoint in release tooling, so the narrowest maintainable fix depends on first proving whether the broken state comes from OpenClaw packaging, npm registry contents, or a package-manager/local partial install.
