## Summary

Keep open. Current main still lacks MCP Resources, but the branch is not mergeable as-is because the config resource uses local redaction that can miss arbitrary secret-like MCP server fields and the plugin-tools server predates the current handler seam that enforces owner-only filtering and before_tool_call approval hooks.

## What This Changes

The branch adds MCP resource support for channel bridge status/conversations and plugin tool/config inspection, wires resource handlers into the channel and plugin-tools MCP servers, and expands plugin-tools resource tests.

## Best Possible Solution

Ship MCP resources only after rebasing onto current main's MCP server/helper structure, serializing config resources through shared redaction, preserving owner-only and before_tool_call controls, and aligning docs/tests with the final channel/plugin/gateway resource contract.

## Reproduction Assessment

Yes. The issue is reproducible by static review of the PR head: readResource on openclaw://config/mcp would return unknown MCP server properties unchanged, and the PR branch's plugin-tools server directly calls tool.execute rather than using current main's policy-enforcing handler.

## Solution Assessment

No. The feature direction is plausible, but this patch is not the narrowest maintainable solution until it reuses current main's handler boundary and shared config redaction instead of adding parallel MCP server and redaction logic.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Use shared redaction for MCP config resources:** `src/mcp/plugin-resources.ts:105-143`

  - confidence: 0.86
