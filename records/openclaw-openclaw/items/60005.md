## Summary

Keep open. Current `main` does not implement this MCP resource feature, and the PR still has blocking source-visible security/correctness issues plus missing real MCP client proof before merge.

## What This Changes

The PR adds MCP resources to the channel and plugin-tools MCP servers, including version/status/conversation/tool/config resources and plugin-tools resource tests.

## Best Possible Solution

Rebase the feature onto current main's shared MCP tool handlers, use shared redaction or a strict allowlist for config-derived resources, and settle the resource API boundary with the gateway MCP Apps work before merge.

## Reproduction Assessment

Yes for the review defects: PR-head source shows `openclaw://config/mcp` returns arbitrary server fields after local masking, and plugin-tool calls use a direct `tool.execute` path. The feature itself still lacks live MCP client proof.

## Solution Assessment

No. The feature direction is plausible, but this patch duplicates sensitive config scrubbing and plugin-tool handling instead of extending current shared helpers and resolving the overlapping MCP resource boundary.
