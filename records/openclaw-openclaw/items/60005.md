The PR adds MCP resource list/read support for channel bridge metadata and plugin tool/config inspection, plus plugin-tools resource tests.

## Best Possible Solution

Rebase the feature onto current main's shared MCP tool server/helpers, use shared redaction or a strict allowlist for config-derived resource payloads, and align the resource API with the gateway MCP Apps boundary before merge.
