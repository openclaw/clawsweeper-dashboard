## Summary

Keep open. Current main still does not implement MCP resource reads for the channel or plugin-tools MCP servers, but this PR is not merge-ready because the branch adds config resources with custom redaction and predates current main's plugin-tool policy handler seam.

## What This Changes

The PR adds MCP resource registration for channel bridge status/conversations and plugin tool/config inspection, with list/read resource handlers and plugin-tools tests.

## Best Possible Solution

Ship MCP resources only after rebasing onto current main's shared MCP server/helper structure, using shared config redaction for all exposed config, preserving owner-only and before_tool_call controls, and coordinating with the gateway MCP Apps/resource contract.

## Reproduction Assessment

Yes. Static review is enough: readResource on openclaw://config/mcp would serialize arbitrary MCP server fields outside the PR's small redaction list, and the branch's plugin-tools server shape predates current main's policy-enforcing helper.

## Solution Assessment

No. The feature direction is plausible, but the maintainable solution should extend current shared handlers and redaction utilities instead of adding parallel MCP server and config-scrubbing logic.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Rebase the resource support around that helper instead of restoring the older direct execution path.
