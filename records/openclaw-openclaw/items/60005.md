## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Route config resources through shared redaction:** `src/mcp/plugin-resources.ts:105-143`
  - body: `openclaw://config/mcp` clones each server entry, masks a few known fields, and returns every other key. `McpServerConfig` permits arbitrary extra keys, so `apiKey`, `authToken`, or `clientSecret`-style fields can leak to any MCP client that reads the resource; build this payload from a strict allowlist or the shared config redaction path.
  - confidence: 0.92
- **[P1] Preserve shared plugin-tool handling:** `src/mcp/plugin-tools-serve.ts:77-87`
  - body: This branch replaces current main's shared plugin-tool MCP server with a local `CallTool` handler that calls `tool.execute` directly. That drops owner-only filtering, `before_tool_call` wrapping, result coercion, and abort-signal forwarding from `createPluginToolsMcpHandlers`; add resources on top of the shared server/handlers instead.
  - confidence: 0.91
- **[P1] Keep standalone plugin-tool policy resolution:** `src/mcp/plugin-tools-serve.ts:35-39`
  - body: The branch resolves tools with a direct `resolvePluginTools` call, while current main loads the standalone registry and threads configured allow/deny policy into both registry loading and tool resolution. That can expose denied plugin tools or miss registered tools after upgrade; keep the current `resolveTools` path and layer resources around it.
  - confidence: 0.87
