## Summary

Keep open: current main still does not implement this channel/plugin MCP resource feature, and the PR should not merge as-is because it exposes config over MCP with incomplete shared redaction and revives an older direct plugin-tool execution path.

## What This Changes

The PR adds MCP resource listing/reading for channel bridge metadata and plugin-tool/config inspection, plus plugin-tools resource tests.

## Best Possible Solution

Rebase the feature onto current main's shared MCP server/helper structure, expose resources only through the agreed MCP Apps/resource boundary, and use shared config redaction for every config-derived payload.

## Reproduction Assessment

Yes for the review defects: static source review shows the PR serializes arbitrary MCP server fields in openclaw://config/mcp and replaces the current shared plugin-tool MCP handler with direct execution. Current main also has tests proving before_tool_call approval must block MCP tool execution.

## Solution Assessment

No. The feature direction is plausible, but this patch duplicates config scrubbing and plugin-tool handling instead of extending current shared helpers or the related gateway MCP Apps resource contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Route MCP config resources through shared redaction:** `src/mcp/plugin-resources.ts:105-143`
  - body: `openclaw://config/mcp` clones each server entry and masks only headers, env, and selected URL pieces before returning every other property. Since `McpServerConfig` permits arbitrary keys, auth-like fields such as `apiKey`, `authToken`, or `clientSecret` would be exposed to any MCP client reading the resource; build this payload from the shared config redaction path instead of a local allowlist.
  - confidence: 0.92
- **[P2] Preserve the shared plugin-tool MCP handler:** `src/mcp/plugin-tools-serve.ts:53-86`
  - body: The branch reconstructs a local tool map and calls `tool.execute` directly, while current main routes plugin MCP calls through `createToolsMcpServer`/`createPluginToolsMcpHandlers` to filter `ownerOnly` tools and run `before_tool_call` approval hooks. Rebase the resources onto that helper instead of reintroducing direct execution.
  - confidence: 0.86
