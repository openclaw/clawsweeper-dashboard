## Summary

Keep this PR open: the feature direction is plausible, but the current branch still has blocking security/correctness issues against current main and lacks required real behavior proof.

## What This Changes

The PR adds MCP resource list/read support to the channel and plugin-tools MCP servers, including version, status, conversation, tool, and config resources plus tests.

## Best Possible Solution

Rebase resource support onto current shared MCP handlers, expose config only through strict allowlists or central redaction, coordinate the resource boundary with https://github.com/openclaw/openclaw/pull/66168, and require redacted real MCP client proof before merge.

## Reproduction Assessment

Yes for the review defects: PR-head source inspection shows the config resource returns arbitrary server fields after local masking and the plugin-tools path calls tool.execute directly. The requested MCP resource surface is a new feature, so there is no broken current-main behavior to reproduce.

## Solution Assessment

No. The feature direction may be useful, but this patch should extend current shared handlers and central redaction instead of adding duplicate sensitive config and tool-execution paths.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Route config resources through shared redaction:** `src/mcp/plugin-resources.ts:105-143`
  - body: `openclaw://config/mcp` clones each MCP server entry, masks only selected fields, and returns every other key. `McpServerConfig` permits arbitrary additional keys, so credential-like fields such as `apiKey`, `authToken`, or `clientSecret` can leak to any MCP client that reads this resource; build this payload through shared redaction or a strict allowlist.
  - confidence: 0.92
- **[P1] Preserve shared plugin-tool handling:** `src/mcp/plugin-tools-serve.ts:77-87`
  - body: This branch rebuilds the plugin tool map and calls `tool.execute` directly, while current main routes plugin MCP tools through shared handlers that filter `ownerOnly` tools, wrap `before_tool_call` enforcement, and forward abort signals. Add resources on top of the shared server/handlers instead of restoring this direct path.
  - confidence: 0.9
