- **[P1] Preserve the current sqlite-vec fallback path:** `packages/memory-host-sdk/src/host/sqlite-vec.ts:69-70`
  - body: Current main falls back to `resolveSqliteVecPlatformVariant()` when the bundled `sqlite-vec` package import or load fails, which covers installs where only the platform package is present. This branch's replacement catches the bundled load error and rethrows on non-Windows without trying that variant path, so a rebase that keeps this shape would regress current supported installs. Add the Windows suffixless retry inside the current fallback flow instead of replacing it.
  - confidence: 0.88
- **[P1] Keep verifying vec_version after manual loads:** `packages/memory-host-sdk/src/host/sqlite-vec.ts:61-62`
  - body: Current main verifies `SELECT vec_version()` after every direct `loadExtension()` so ABI mismatches or silent loads do not mark vector search available. The Windows retry path here calls `params.db.loadExtension(suffixlessPath)` and returns success without that health check, which would reintroduce false-ready vector state on the same memory path. Route the retry through the existing load-and-verify helper when rebasing.
  - confidence: 0.86
