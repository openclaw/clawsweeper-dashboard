## Summary

Keep open: the suffixless retry targets a real Windows memory vector-search failure, but the branch is stale against current main's platform-variant fallback and would not cover installs that load `sqlite-vec-windows-x64/vec0.dll` through that path.

## What This Changes

The PR adds a Windows-only sqlite-vec loader retry that strips `.dll` after a bundled load failure, adds focused loader tests, and adds a changelog entry.

## Best Possible Solution

Rebase or replace the branch with one small Windows `.dll` suffixless load helper used by both automatic load paths, while keeping explicit `extensionPath` pass-through unchanged.

## Reproduction Assessment

Yes for source-level review: current main still has a Windows platform-variant path ending in `vec0.dll`, sqlite-vec delegates loading to `db.loadExtension`, and the PR body includes Windows terminal before/after output. I did not run an independent Windows smoke in this read-only pass.

## Solution Assessment

No, not as the branch currently stands against main. The suffixless retry is the right narrow fix, but it must be integrated with the newer platform-variant fallback before merge.
