## Summary

Keep open: this is a plausible focused Windows memory bug fix, but the PR branch is stale against current main and scopes the suffixless retry only to the bundled `sqlite-vec` load path. Current main now has a platform-variant fallback for installs with only `sqlite-vec-windows-x64`, and that fallback still loads `vec0.dll` directly.

## What This Changes

The PR adds a Windows-specific sqlite-vec retry that strips `.dll` after bundled loading fails, plus focused loader tests and a changelog entry.

## Best Possible Solution

Rebase the PR onto current main, factor the Windows suffixless load attempt so both bundled and platform-variant paths use it, keep explicit extensionPath untouched, and validate the focused loader tests plus a Windows memory smoke before merge.

## Reproduction Assessment

Yes for source-level review: current main still loads suffixed Windows sqlite-vec paths without a suffixless retry, and the PR body includes Windows before/after terminal output. I did not independently run the Windows scenario in this read-only review.

## Solution Assessment

No, not as the branch stands. The suffixless retry is the right narrow direction, but it must be integrated with current main's platform-variant fallback before merge.
