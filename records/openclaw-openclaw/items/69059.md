## Summary

Keep open: the PR addresses the bundled sqlite-vec load path, but it is incomplete against current main because the newer platform-variant fallback still directly loads the suffixed Windows DLL without the suffixless retry.

## What This Changes

The PR adds a Windows `.dll` suffixless retry for bundled sqlite-vec loading, loader regression tests, and a changelog entry for the Windows memory-vector failure.

## Best Possible Solution

Rebase or replace the branch with a small shared loader helper that leaves explicit `extensionPath` untouched and applies the Windows suffixless retry to both bundled and platform-variant sqlite-vec loads.

## Reproduction Assessment

Yes for source-level review: current main has suffixed Windows sqlite-vec paths in both bundled and platform-variant load paths, and the PR body includes Windows before/after terminal output. I did not independently run the Windows scenario in this read-only review.

## Solution Assessment

No, not as the branch stands. The suffixless retry is the right narrow fix, but it must be shared with current main's platform-variant fallback before merge.
