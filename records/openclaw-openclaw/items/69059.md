## Summary

Keep open. The Windows suffixless retry is a valid direction and current main still lacks that fallback, but the PR head is stale against the current package-owned loader and can break configured explicit sqlite-vec paths before the retry logic runs.

## What This Changes

This PR adds a Windows-only sqlite-vec fallback that retries bundled `.dll` loads without the suffix and returns the effective loaded path in both memory-host loader copies.

## Best Possible Solution

Rebase the retry onto the current package-owned loader, preserve explicit `extensionPath` as an early no-import path, return the suffixless path after a successful Windows retry, and cover the behavior with focused tests and a changelog entry.

## Reproduction Assessment

No for the original Windows runtime failure in this Linux checkout. Yes for the PR regression: current main has a focused test proving explicit `extensionPath` works without importing `sqlite-vec`, while PR head imports it first.

## Solution Assessment

No, not as written. The suffixless retry is a narrow maintainable fix, but the branch needs to be reshaped around the current package loader contract before merge.
