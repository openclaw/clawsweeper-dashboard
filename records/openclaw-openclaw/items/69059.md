## Summary

Keep open: the Windows suffixless retry is the right bug-fix direction and the PR includes real terminal proof, but the branch still needs repair because it only covers the bundled `sqlite-vec` load path and does not preserve/apply the same retry to current main's platform-variant fallback.

## What This Changes

Adds a Windows-only retry without the `.dll` suffix for bundled sqlite-vec loading, focused loader tests, and a changelog entry for Windows memory vector loading.

## Best Possible Solution

Keep the explicit `extensionPath` short-circuit unchanged, then add one shared Windows suffixless retry helper used by both bundled sqlite-vec loading and platform-variant loading.

## Reproduction Assessment

Yes for source-level review: current main has suffixed Windows `vec0.dll` paths in both bundled and platform-variant load paths, and the PR body includes Windows before/after terminal output for `openclaw memory index` and `openclaw doctor`. ## Solution Assessment

No, not as the branch stands. The suffixless retry is the narrow maintainable fix, but it needs to be shared with current main's platform-variant fallback before merge.
