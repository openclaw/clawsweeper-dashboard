## Summary

Keep open. The Windows suffixless sqlite-vec retry is still useful on current main, but this PR needs to be rebased into the current package-owned loader contract and still needs changelog/test coverage before it is mergeable.

## What This Changes

This PR adds a Windows-only sqlite-vec load fallback in the memory-host loaders, retrying bundled `.dll` loads without the suffix and returning the effective loaded path.

## Best Possible Solution

Reapply the suffixless Windows retry on top of the current package-owned loader, keep explicit extensionPath as an early path that does not import sqlite-vec, add focused tests, and leave broader sqlite ABI/function-registration failures to separate issues.

## Reproduction Assessment

No for the original Windows runtime failure in this Linux checkout. Yes for the PR regression: current main has a focused test proving explicit extensionPath must work without importing sqlite-vec, while the PR head imports it first.

## Solution Assessment

No, this branch is not the best merge shape as-is. The retry idea is appropriate, but the maintainable fix is a rebased package-loader change with tests and a changelog entry.
