## Summary

Keep open: the Windows sqlite-vec fix is plausible and the contributor proof is sufficient, but this PR is currently conflicting with main and its head drops the platform-variant fallback that current main already ships for installs with only `sqlite-vec-windows-x64`.

## What This Changes

The PR adds a Windows-only `.dll` suffixless retry after bundled `sqlite-vec.load()` fails, updates focused sqlite-vec loader tests, and adds a changelog entry.

## Best Possible Solution

Preserve current main's explicit-path and platform-variant behavior, then factor the Windows suffixless retry into a small helper used by both bundled and variant auto-discovery paths with focused tests for each path.

## Reproduction Assessment

Yes for source-level review: current main still has Windows paths that can pass a suffixed `vec0.dll` to `loadExtension`, and SQLite documents suffix omission as the portable convention. The PR body also includes copied Windows before/after CLI output, but no independent Windows smoke was run in this read-only pass.

## Solution Assessment

No, not as the branch currently stands. The suffixless retry is the right narrow fix, but it needs to be integrated with current main's platform-variant fallback before merge.
