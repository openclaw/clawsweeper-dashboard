## Summary

Keep open: the branch is still based before current main's platform-variant sqlite-vec loader, so the Windows suffixless retry needs to be shared with both bundled and variant load paths before merge.

## What This Changes

The branch adds a Windows `.dll` suffixless retry for sqlite-vec loading, focused regression tests, and a changelog entry.

## Best Possible Solution

Rebase or replace the branch with a small shared loader helper that preserves explicit paths and applies the Windows suffixless retry to both bundled and platform-variant sqlite-vec loads.

## Reproduction Assessment

Yes for source-level review: current main can load `sqlite-vec-windows-x64/vec0.dll` directly on the variant path, sqlite-vec passes suffixed paths to `loadExtension()`, and the PR body shows Windows before/after terminal output. I did not run an independent Windows smoke in this read-only review.

## Solution Assessment

No, not as the branch stands. The suffixless retry is the right narrow fix, but current main added a platform-variant fallback after the branch base, and the same retry must be applied there before merge.
