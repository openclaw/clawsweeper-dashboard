## Summary

Keep open: this is a plausible Windows sqlite-vec bug fix with sufficient terminal proof, but the latest PR head is stale against current main and only applies the suffixless retry to the bundled `sqlite-vec.load()` path. Current main also has a platform-variant fallback that resolves `sqlite-vec-windows-x64/vec0.dll` directly, and that path needs the same retry before this PR is correct.

## What This Changes

The PR adds a Windows-only retry that strips `.dll` from bundled sqlite-vec load paths, adds focused loader tests, and adds a changelog entry.

Rebase onto current main and use one Windows suffixless load helper for both bundled sqlite-vec loading and the platform-variant fallback, while preserving explicit `extensionPath` pass-through and persisting the effective suffixless path.
