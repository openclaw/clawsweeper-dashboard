## Summary

Keep open: this is a focused Windows sqlite-vec fix with useful proof, but the branch predates current main's platform-variant fallback and only retries the bundled `sqliteVec.load()` path, so it can still miss Windows installs that load `sqlite-vec-windows-x64/vec0.dll` directly.

## What This Changes

The branch adds a Windows-specific sqlite-vec loader retry without the `.dll` suffix, focused loader tests, and an Unreleased changelog entry.

## Best Possible Solution

Rebase onto current main and factor a small loader helper so both bundled and platform-variant sqlite-vec paths use the Windows suffixless retry, while explicit `extensionPath` remains untouched.

## Reproduction Assessment

Yes for source-level review: current main passes suffixed Windows sqlite-vec paths through both bundled and variant load paths, and the PR body includes Windows before/after terminal output. I did not run the Windows scenario independently.

## Solution Assessment

No, not as the branch stands. The suffixless retry is the right narrow direction, but it must be applied to current main's platform-variant fallback before merge.
