## Summary

Keep open: the PR is a useful Windows sqlite-vec bug-fix direction and has adequate terminal proof, but it still misses current main's platform-variant fallback path, so variant-only Windows installs can keep passing a suffixed `vec0.dll` to `loadExtension()`.

## What This Changes

The PR adds a Windows-specific sqlite-vec `.dll` suffix retry, focused loader tests, and a changelog entry for the linked Windows memory issue.

## Best Possible Solution

Rebase onto current main and introduce a small shared loader helper so bundled sqlite-vec and platform-variant loads both get the Windows suffixless retry while explicit `extensionPath` remains untouched.

## Reproduction Assessment

Yes for source-level review: current main passes suffixed Windows sqlite-vec paths through bundled and variant load paths, and the PR body includes Windows before/after terminal output. I did not independently run the Windows scenario.

## Solution Assessment

No, not as the branch stands. The suffixless retry is the right narrow fix, but it needs to be shared with current main's platform-variant fallback before merge.
