## Summary

Keep this open: current main still feeds `event.prompt` into Active Memory's recall conversation context, so the narrow prompt-envelope bloat remains even though the memory search query is separately sanitized and clamped.

## What This Changes

The issue asks Active Memory to derive recall context from the actual current user request, or a bounded sanitized fallback, instead of the full OpenClaw prompt envelope.

## Best Possible Solution

Fix the Active Memory plugin to build recall conversation context from a sanitized current-user request while preserving `queryMode` behavior for real recent/full conversation turns and the existing bounded search query path.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: call the Active Memory `before_prompt_build` hook with an envelope-shaped `event.prompt`, and current main places that envelope under `Conversation context` while only the search query is sanitized. I did not run a live Telegram gateway reproduction in this read-only review.

## Solution Assessment

Yes, a narrow plugin-local fix is the best path: sanitize or derive the latest user request before `buildQuery`, add regression coverage, and leave broader Active Memory blocking/reliability work to the related open issues.
