## Summary

Keep this open: current main and the latest release still pass the raw prompt-build prompt into Active Memory's recall context, and the linked closing pull request is still open rather than merged.

## What This Changes

The issue asks Active Memory to build recall context from the actual current user request or a bounded fallback instead of the full OpenClaw prompt envelope.

## Best Possible Solution

Review and repair https://github.com/openclaw/openclaw/pull/88078, or replace it with a narrow Active Memory fix that derives a bounded current-user-message recall context while preserving existing search-query clamping.

## Reproduction Assessment

Yes, from source inspection: current main still passes `event.prompt` into `buildQuery`, while only `buildSearchQuery` sanitizes and clamps the search string. I did not run a live Telegram gateway reproduction in this read-only review.

## Solution Assessment

Yes for the issue direction: the maintainable fix belongs in the Active Memory plugin and should derive a bounded latest-user-message value before building recall context. The linked PR is a candidate but still needs conflict repair and maintainer review of the envelope-boundary heuristic.
