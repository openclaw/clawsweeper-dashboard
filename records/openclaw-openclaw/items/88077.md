Keep this open: current main and the latest release still pass the raw `before_prompt_build` prompt into Active Memory recall context, and the linked closing pull request remains open and conflicting rather than merged.

## What This Changes

Active Memory should build recall subagent context from a bounded current user request instead of the full OpenClaw prompt envelope.

## Best Possible Solution

Land or replace the linked Active Memory plugin fix with a bounded latest-user-message derivation that is covered by realistic Telegram/OpenClaw envelope tests.

Yes, by source inspection: current main still passes `event.prompt` into `buildQuery` while only `buildSearchQuery` is sanitized and clamped. I did not run a live Telegram gateway in this read-only review.
