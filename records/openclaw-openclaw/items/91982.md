## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/91982. Current main lets the Anthropic Vertex `onPayload` wrapper reapply `applyAnthropicPayloadPolicyToParams` to a payload that native Anthropic shaping may already split and mark, so active-memory recall plus tools can exceed Anthropic's four `cache_control` markers. Make the shared Anthropic payload policy or Vertex wrapper idempotent for pre-shaped Anthropic payloads: do not add `cache_control` to a dynamic system suffix when the system array already carries a cache marker, and ensure the final system/tools/messages payload stays at or below four markers while preserving the intended stable system, last tool, and latest user/tool-result cache points. Add focused regression coverage for a pre-split system payload with stable-prefix CC, dynamic suffix without CC, a cached tool, and active-memory-style user content; verify Vertex policy application does not produce a fifth marker. Do not add config/env flags, do not change active-memory trust semantics, and do not try to solve the broader cache-hit-rate product issue in https://github.com/openclaw/openclaw/issues/91223.
