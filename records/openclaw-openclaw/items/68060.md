Full review comments:

- **[P1] Sync smartExtraction with the manifest schema:** `extensions/memory-lancedb/config.ts:139`
  - body: This makes the runtime parser accept `smartExtraction`, but `openclaw.plugin.json` still has `additionalProperties: false` and no matching property. Plugin config is validated against the manifest before runtime registration, so a user who enables the new option can get an invalid-config error instead of the feature loading.
  - confidence: 0.94
- **[P1] Gate smart extraction with capture safety filters:** `extensions/memory-lancedb/index.ts:636`
  - body: The smart branch passes raw collected user `texts` to the LLM extractor before applying the existing `shouldCapture(..., { maxChars })` gate. That bypasses the configured length limit and prompt-injection/capture eligibility checks for exactly the data being sent to a chat model and later persisted.
  - confidence: 0.95
- **[P1] Treat existing memories as untrusted prompt input:** `extensions/memory-lancedb/smart-extractor.ts:110`
  - body: `existingMemories` are durable user-controlled strings, but the extractor appends them verbatim as prompt text. A stored memory containing instruction-like text can steer extraction or deduplication; the prompt should escape/label these as untrusted data or pass them through a structured boundary.
  - confidence: 0.91
- **[P2] Surface extraction failures before returning no memories:** `extensions/memory-lancedb/smart-extractor.ts:202-203`
  - body: Catching every chat-completion error and returning `{ memories: [] }` makes invalid models, auth errors, bad base URLs, and transient provider failures indistinguishable from a valid no-op extraction. At minimum this needs logging or a safe fallback so smart auto-capture does not silently stop working.
  - confidence: 0.89
