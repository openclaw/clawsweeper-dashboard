- **[P1] Mirror smartExtraction in the plugin manifest:** `extensions/memory-lancedb/config.ts:139`
  - body: This adds `smartExtraction` to the runtime parser, but the PR does not add the same property to `extensions/memory-lancedb/openclaw.plugin.json`.   - confidence: 0.94
- **[P1] Run capture safety gates before extraction:** `extensions/memory-lancedb/index.ts:636`
  - body: The smart-extraction branch sends all collected user text to the LLM extractor, while current capture first sanitizes and gates text through `shouldCapture`, custom triggers, and `captureMaxChars`. That lets long, envelope-wrapped, or prompt-injection-like content reach the extraction model and potentially get persisted.
  - confidence: 0.96
- **[P1] Escape existing memories before prompt use:** `extensions/memory-lancedb/smart-extractor.ts:110`
  - body: `existingMemories` comes from previously stored user-controlled memory text, but the extractor appends it verbatim to the prompt. The current recall path escapes and labels memories as untrusted; this new prompt path should do the same so stored instructions cannot steer extraction or deduplication.
  - confidence: 0.91
- **[P2] Surface extraction failures instead of no-oping:** `extensions/memory-lancedb/smart-extractor.ts:202-203`
  - body: The extractor catches every chat-completion failure and returns an empty result, and the caller treats that as a normal no-op. Invalid model, auth, or endpoint settings would silently disable smart capture, so this should log, surface, or safely fall back in a way users can diagnose.
  - confidence: 0.88
