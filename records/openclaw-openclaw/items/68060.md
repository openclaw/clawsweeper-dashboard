## Summary

Keep open: this is a plausible memory-lancedb feature PR, but it is not safe to merge as-is because source review still shows manifest/config drift, capture-safety bypasses, untrusted prompt input, silent extractor failures, and no real behavior proof.

## What This Changes

The PR adds opt-in smartExtraction config, an OpenAI-compatible smart extractor module, parser tests, and an auto-capture branch for LLM-based atomic memory extraction in memory-lancedb.

## Best Possible Solution

Keep this PR open for maintainer-directed rework that aligns manifest, runtime config, UI hints, docs, tests, capture gating, prompt-trust boundaries, diagnostics, and real runtime proof before merge.

## Reproduction Assessment

Yes. The blocking issues are source-reproducible from the PR head: smartExtraction is accepted by the runtime parser but not the strict manifest, and the smart branch sends raw texts and existing memories to the extractor before current safety/trust handling.

## Solution Assessment

No. Single-shot LLM extraction may be a valid opt-in feature, but this implementation is not the narrowest maintainable merge path until it preserves existing memory-lancedb contracts, safety gates, untrusted-data handling, and operator diagnostics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Sync smartExtraction with the manifest schema:** `extensions/memory-lancedb/config.ts:139`
  - body: The parser now accepts `smartExtraction`, but `openclaw.plugin.json` still has `additionalProperties: false` with no matching property. The loader validates plugin config against the manifest before registration, so enabling the new option will fail before this parser runs.
  - confidence: 0.95
- **[P1] Filter texts before smart extraction:** `extensions/memory-lancedb/index.ts:633-637`
  - body: This passes every collected user text to `extractMemories` before applying `shouldCapture` or `captureMaxChars`. Messages the current auto-capture path rejects for length or prompt-injection patterns can now be sent to the extraction provider and stored.
  - confidence: 0.96
- **[P1] Treat existing memories as untrusted input:** `extensions/memory-lancedb/smart-extractor.ts:109-110`
  - body: `existingMemories` are persisted user-controlled strings, but the extractor prompt interpolates them verbatim as bullet text. A stored instruction-like memory can steer extraction or dedup behavior unless these entries are escaped and clearly delimited as untrusted data.
  - confidence: 0.91
- **[P2] Surface extractor failures:** `extensions/memory-lancedb/smart-extractor.ts:202-203`
  - body: If the chat completion fails because of auth, model, base URL, or provider outage, this catch returns an empty result and the caller treats it as a normal no-op. That silently disables smart auto-capture and gives operators no warning or fallback path.
  - confidence: 0.88
