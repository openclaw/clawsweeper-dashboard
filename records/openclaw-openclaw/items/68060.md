Full review comments:

- **[P1] Sync smartExtraction with the manifest schema:** `extensions/memory-lancedb/config.ts:139`
  - body: The runtime parser now accepts `smartExtraction`, but `openclaw.plugin.json` still has `additionalProperties: false` with no matching property. Plugin config is validated from the manifest before this parser runs, so enabling the new option will reject the plugin config instead of enabling extraction.
  - confidence: 0.95
- **[P1] Filter texts before smart extraction:** `extensions/memory-lancedb/index.ts:633-637`
  - body: This calls `extractMemories` with every collected user text before applying `shouldCapture`, `captureMaxChars`, or custom trigger policy. Messages the current auto-capture path rejects for length or prompt-injection patterns can now be sent to the extraction provider and stored.
  - confidence: 0.96
- **[P1] Treat existing memories as untrusted input:** `extensions/memory-lancedb/smart-extractor.ts:109-110`
  - body: `existingMemories` are persisted user-controlled strings, but the extractor prompt interpolates them verbatim as bullet text. A stored instruction-like memory can steer extraction or dedup behavior unless these entries are escaped and clearly delimited as untrusted data.
  - confidence: 0.91
- **[P2] Surface extractor failures:** `extensions/memory-lancedb/smart-extractor.ts:202-203`
  - body: If the chat completion fails because of auth, model, base URL, or provider outage, this catch returns an empty result and the caller treats it as a normal no-op. That silently disables smart auto-capture and gives operators no warning or fallback path.
  - confidence: 0.88
