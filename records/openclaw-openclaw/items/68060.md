- **[P1] Sync smartExtraction with the manifest schema:** `extensions/memory-lancedb/config.ts:139`
  - body: The runtime parser now accepts `smartExtraction`, but the PR does not update `openclaw.plugin.json`, whose config schema is strict.   - confidence: 0.94
- **[P1] Reuse the capture gate before smart extraction:** `extensions/memory-lancedb/index.ts:636`
  - body: The smart branch sends raw collected user text to `extractMemories` without first running the existing sanitize/`shouldCapture` path. That bypasses `captureMaxChars`, envelope-sludge rejection, and prompt-injection checks, so long or unsafe content can be sent to the chat model and later persisted as memory.
  - confidence: 0.96
- **[P1] Mark existing memories as untrusted prompt data:** `extensions/memory-lancedb/smart-extractor.ts:110`
  - body: Stored memory text is user-controlled, but this prompt appends `existingMemories` verbatim as instructions-adjacent text. A prior memory containing instruction-like content can steer extraction or dedup behavior; reuse the current recall path's untrusted labeling/escaping pattern before including these strings.
  - confidence: 0.92
- **[P2] Do not silently disable smart capture on extraction errors:** `extensions/memory-lancedb/smart-extractor.ts:202-203`
  - body: `extractMemories` catches every chat-completion failure and returns an empty result, so auth, model, base URL, rate-limit, or provider errors look the same as 'nothing to store'. The caller then returns without warning, making smart auto-capture fail closed and hard to diagnose.
  - confidence: 0.9
