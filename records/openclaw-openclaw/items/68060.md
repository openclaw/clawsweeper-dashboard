- **[P1] Mirror smartExtraction in the plugin manifest:** `extensions/memory-lancedb/config.ts:139`
  - body: The runtime parser now accepts `smartExtraction`, but `extensions/memory-lancedb/openclaw.plugin.json` still has a strict `additionalProperties: false` config schema with no matching property. Because plugin config is validated against the manifest before loading, a user who enables this feature can get an invalid-config failure before this new runtime parser runs.
  - confidence: 0.96
- **[P1] Run capture safety gates before extraction:** `extensions/memory-lancedb/index.ts:633-638`
  - body: This path passes every collected user message to `extractMemories` before applying the existing `sanitizeForMemoryCapture` and `shouldCapture(..., { maxChars })` gates. With `smartExtraction.enabled`, over-limit text, envelope sludge, and prompt-injection-like content that the current auto-capture path rejects can now be sent to the extractor model and stored.
  - confidence: 0.97
- **[P1] Escape stored memories before prompt injection:** `extensions/memory-lancedb/smart-extractor.ts:109-112`
  - body: Stored memories are user-controlled data, but this prompt appends `existingMemories` verbatim as live model instructions. The current recall path filters and escapes stored memories before prompt use; this new extractor should treat them as untrusted too or a previously stored instruction-like memory can steer extraction and dedup behavior.
  - confidence: 0.95
- **[P2] Surface extraction failures instead of no-oping:** `extensions/memory-lancedb/smart-extractor.ts:201-203`
  - body: Catching every extraction error and returning an empty memory list makes auth, model, endpoint, rate-limit, and provider outages indistinguishable from nothing to store. The caller treats that as a normal no-op, so enabling smart extraction can silently disable memory capture with no operator-visible signal.
  - confidence: 0.9
