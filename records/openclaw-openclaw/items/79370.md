## Summary

Keep open. The current head addresses the earlier TTL preservation and `none` sanitizer issues, but the explicit OpenRouter retention gate is still provider-id based, so it misses custom provider ids targeting `openrouter.ai` and can pass cache-retention hints into non-Anthropic OpenRouter models.

## What This Changes

The PR threads resolved `cacheRetention` into OpenRouter Anthropic cache marker insertion, preserves `ttl: "1h"` for long retention, skips new markers for `none`, and adds focused regression tests.

## Best Possible Solution

Make cache-retention resolution use the same verified OpenRouter Anthropic endpoint/model contract as marker insertion, preserving unsupported-model no-ops and custom-provider native OpenRouter support.

## Reproduction Assessment

Yes. Current main is source-reproducible: `resolveCacheRetention` returns `undefined` before reading explicit config for OpenRouter Anthropic refs, and the OpenRouter marker helper hardcodes no TTL; the PR also supplies before/after outbound request logs.

## Solution Assessment

No. The direction is right, but the current implementation should follow the existing OpenRouter endpoint/model policy instead of using `provider === "openrouter"` as the explicit-retention gate.
