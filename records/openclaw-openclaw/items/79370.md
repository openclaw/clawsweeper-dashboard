## Summary

Keep open. The reported OpenRouter Anthropic cache-retention bug is real and the PR is pointed at the right area, but the latest head is not mergeable: the updated proxy wrapper test file still uses a stale wrapper call shape and undefined variable, and the resolver change still returns explicit retention before provider/model eligibility.

## What This Changes

The PR changes cache-retention resolution and OpenRouter Anthropic cache-control marker insertion so explicit `long`, `short`, and `none` retention can add `ttl: "1h"` or skip markers, with regression tests.

## Best Possible Solution

Keep the fix in the agent cache-retention and OpenRouter wrapper boundary, but pass retention/facts consistently, preserve provider/model eligibility checks, repair the tests, and refresh real proxy/log proof for the exact final head.

## Reproduction Assessment

Yes. Current main is source-reproducible from `resolveCacheRetention` returning before explicit OpenRouter `cacheRetention` is read and the OpenRouter marker helper hardcoding no-TTL markers; the PR discussion also includes before/after outbound request logs for the intended route.

## Solution Assessment

No. The intended direction is maintainable, but the current patch is not the best form yet because the tests do not typecheck and the resolver ignores the eligibility boundary it should preserve.
