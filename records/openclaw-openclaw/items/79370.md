## Summary

Keep open: the underlying cache-retention bug is real and worth fixing, but this PR is not correct as written because the production plumbing passes the wrong shape into the new wrapper and the new OpenRouter eligibility check can bypass the current endpoint-class boundary.

## What This Changes

The branch changes prompt-cache retention resolution and OpenRouter Anthropic payload wrapping/tests so explicit `cacheRetention` can add `ttl: "1h"` or skip cache markers.

## Best Possible Solution

Keep the current endpoint-class OpenRouter boundary, pass the actual params or a typed retention value consistently through the wrapper, and cover the full `applyExtraParamsToAgent` path for builtin OpenRouter, custom OpenRouter base URLs, arbitrary proxies, and `none`/`short`/`long`.

## Reproduction Assessment

Yes, source-reproducible. Current main resolves no OpenRouter retention before reading explicit config and then hardcodes short `cache_control` markers; the PR body also includes before/after request payload logs from a real OpenRouter route.

## Solution Assessment

No. The direction is right, but this patch is not the best implementation until it preserves endpoint-class gating and passes the correct retention/params shape through the production wrapper path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Pass cache params into the OpenRouter wrapper:** `src/agents/pi-embedded-runner/extra-params.ts:697`
  - body: `createOpenRouterSystemCacheWrapper` now expects a params object and reads `extraParams?.cacheRetention`, but this call passes the resolved retention string. That is type-invalid, and at runtime a value like `"long"` has no `.cacheRetention`, so the production path still falls back to short markers instead of emitting `ttl: "1h"`.
  - confidence: 0.94
- **[P2] Keep OpenRouter retention endpoint-gated:** `src/agents/pi-embedded-runner/prompt-cache-retention.ts:42`
  - body: This marks every `provider === "openrouter"` Anthropic model as retention-eligible before the endpoint policy has checked `baseUrl`. Because this resolver also feeds the pre-plugin stream options, an OpenRouter model repointed at an arbitrary proxy can still pass `cacheRetention` downstream and receive Anthropic cache markers, violating the current endpoint-class boundary from the merged OpenRouter gating change.
  - confidence: 0.86
