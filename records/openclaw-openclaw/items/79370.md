Keep this PR open: the reported cache-retention bug is still present on current main, and the branch contains a focused, endpoint-class-aware fix with sufficient real behavior proof and no blocking review findings.

The PR threads explicit `cacheRetention` and legacy `cacheControlTtl` through OpenRouter Anthropic cache-control handling so long retention preserves `ttl: "1h"` and `none` skips marker insertion.

## Best Possible Solution

Land this endpoint-class-aware fix after focused tests and build, preserving the current OpenRouter route boundary and avoiding cache-control leakage to arbitrary custom proxies.
