## Summary

Keep open: the PR targets a real OpenRouter Anthropic cache-retention bug and the supplied logs are useful, but the latest patch still introduces two narrow regressions around unsupported-provider gating and `cacheRetention: "none"` cleanup.

## What This Changes

This PR reorders cache-retention resolution, threads resolved retention into the OpenRouter system-cache wrapper, preserves `ttl: "1h"` in Anthropic cache markers, skips marker insertion for `none`, and adds focused tests.

## Best Possible Solution

Land a narrow cache fix that honors explicit retention for supported Anthropic/OpenRouter cache routes, keeps unsupported providers as no-ops, preserves thinking-block cleanup, and includes regression tests plus a changelog entry.

## Reproduction Assessment

Yes. Current main is source-reproducible through the early `resolveCacheRetention` return for OpenRouter Anthropic models and the hardcoded no-TTL OpenRouter marker helper; the PR also includes before/after proxy logs.

## Solution Assessment

No, not as currently submitted. The cache-code direction is right, but the implementation should narrow explicit retention pass-through to supported routes and keep the `none` path's cleanup sanitizer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep explicit retention behind supported cache families:** `src/agents/pi-embedded-runner/prompt-cache-retention.ts:28-30`
  - body: Moving the explicit `cacheRetention` return ahead of the family/eligibility check makes every provider honor this cache mode before we know it is supported. For example, the existing non-Anthropic Bedrock guard expects `resolveCacheRetention({ cacheRetention: "long" }, "amazon-bedrock", "openai-completions", "amazon.nova-micro-v1:0")` to stay `undefined`, and the docs say unsupported providers ignore this knob. Please special-case the OpenRouter Anthropic route instead of bypassing the existing family gate for all providers.
  - confidence: 0.94
- **[P2] Still strip thinking cache markers when retention is none:** `src/agents/pi-embedded-runner/proxy-stream-wrappers.ts:165-167`
  - body: `cacheRetention === "none"` returns the underlying stream before `applyAnthropicEphemeralCacheControlMarkers` can remove existing `cache_control` from assistant `thinking`/`redacted_thinking` blocks. That breaks the current sanitizer covered by `extra-params.openrouter-cache-control.test.ts` and can still send cache markers despite the explicit opt-out. Keep the payload patch path for cleanup while skipping system/developer insertion.
  - confidence: 0.86
