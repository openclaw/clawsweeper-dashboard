## Summary

Keep open: the current-main bug is real and the PR direction fixes the long-retention path, but the latest patch still re-adds a short cache marker for explicit `cacheRetention: "none"`.

## What This Changes

The PR reorders prompt-cache retention resolution, exports/reuses Anthropic cache-control marker shaping, threads resolved retention into the OpenRouter system cache wrapper, and adds regression tests.

## Best Possible Solution

Land a version that preserves 1-hour TTL for explicit long retention and bypasses OpenRouter Anthropic marker insertion entirely for explicit `none`, with focused regression coverage and a changelog entry.

## Solution Assessment

No, not yet. The approach is the narrow maintainable fix for explicit long retention, but the wrapper must skip the payload patch for explicit `none` instead of falling back to `{ type: "ephemeral" }`.
