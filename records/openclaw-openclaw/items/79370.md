## Summary

Keep open: the reported OpenRouter Anthropic cacheRetention bug is real, but the latest PR head still has blocking correctness issues in the production call path and resolver eligibility.

## What This Changes

The PR changes OpenRouter Anthropic prompt-cache retention plumbing so explicit long/short/none values can shape cache_control markers, with focused regression tests.

## Best Possible Solution

Thread the original effective extra params plus provider/model facts into the OpenRouter wrapper, keep explicit retention gated to supported OpenRouter Anthropic and Anthropic-family routes, then refresh tests and latest-head real behavior proof.

## Reproduction Assessment

Yes. Current main is source-reproducible: explicit OpenRouter cacheRetention is dropped before it is read, and the marker helper always writes short ephemeral markers; the PR body also includes before/after outbound request dumps for the intended route.

## Solution Assessment

No. The direction is right, but this patch recomputes retention from missing arguments and moves explicit retention ahead of provider/model eligibility, so it is not the narrow maintainable fix yet.
