## Summary

Keep open: the PR fixes the primary built-in OpenRouter path and the prior legacy-key review gap, but latest head still resolves stream cache retention by provider id while the system wrapper is endpoint-class gated, so custom proxy/OpenRouter routes can diverge.

## What This Changes

The branch updates OpenRouter Anthropic prompt-cache retention plumbing so explicit long/short/none and legacy cacheControlTtl can affect cache_control markers, with focused regression tests.

## Best Possible Solution

Use one endpoint-class-aware retention decision for both pi-ai stream options and the OpenRouter system/developer marker wrapper, preserving the custom-proxy guard and covering long, none, and legacy TTL cases in regression tests.

## Reproduction Assessment

Yes, source-reproducible. Current main drops explicit built-in OpenRouter Anthropic retention before stream option insertion, and the remaining PR finding is visible from the patched resolver lacking the endpoint-class/baseUrl data used by the wrapper gate.

## Solution Assessment

No, not quite as submitted. The direction is narrow and appropriate, but stream cacheRetention eligibility should reuse the same verified OpenRouter endpoint boundary as the system wrapper instead of checking only provider id.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Keep retention behind the OpenRouter endpoint gate:** `src/agents/pi-embedded-runner/prompt-cache-retention.ts:41-42`
  - body: This new eligibility check treats any `provider === "openrouter"` Anthropic model as safe for explicit `cacheRetention`, but it cannot see `baseUrl`. If a user repoints the OpenRouter provider at a custom OpenAI-compatible proxy, the system wrapper still skips markers via the endpoint-class gate, while `resolveCacheRetention()` now passes `long` through to pi-ai; pi-ai then emits Anthropic `cache_control` with `ttl: "1h"` for that proxy path. Please make the stream retention decision use the same endpoint-class/baseUrl gate as `createOpenRouterSystemCacheWrapper` and add a regression for an OpenRouter provider on a non-OpenRouter base URL.
  - confidence: 0.87
