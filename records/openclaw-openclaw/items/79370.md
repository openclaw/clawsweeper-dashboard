## Summary

Keep open: the reported cache-retention bug is real, but the current PR head has blocking correctness issues in the production call path and resolver eligibility, and the supplied runtime proof does not establish this latest forced-push head.

## What This Changes

The PR updates OpenRouter Anthropic prompt-cache payload shaping so explicit `cacheRetention` can add `ttl: "1h"` or suppress cache markers, with focused unit tests.

## Best Possible Solution

Thread the original effective extra params plus provider/model API/model id into the endpoint-gated OpenRouter Anthropic wrapper, or pass a resolved retention value directly, while keeping unsupported providers and non-Anthropic OpenRouter refs outside the cache-retention path.

## Reproduction Assessment

Yes. Current main is source-reproducible: `resolveCacheRetention` exits before explicit OpenRouter config is read, and the system marker helper hardcodes `{ type: "ephemeral" }`; the PR discussion also includes outbound before/after request dumps for the intended route.

## Solution Assessment

No. The direction is right, but this patch passes the wrong argument shape into the wrapper and broadens explicit retention outside provider eligibility, so it is not the narrow maintainable fix yet.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.96

Full review comments:

- **[P1] Pass the real cache params into the wrapper:** `src/agents/pi-embedded-runner/extra-params.ts:692-695`
  - body: The new wrapper signature requires the original extra params plus provider/model facts, but this call passes only the resolved retention string. That should fail typecheck; if emitted, the wrapper recomputes retention from missing facts and `cacheRetention: "long"` still falls back to short markers in production. Pass `ctx.effectiveExtraParams`, `ctx.provider`, model API, and `ctx.modelId`, or change the wrapper to accept the resolved retention directly.
  - confidence: 0.97
- **[P2] Keep explicit retention behind provider eligibility:** `src/agents/pi-embedded-runner/prompt-cache-retention.ts:30-33`
  - body: This early return accepts explicit `cacheRetention` for every provider before the family or route eligibility checks. That contradicts the new tests for unsupported providers and OpenRouter non-Anthropic refs, and it can pass cache params into providers where the docs say the knob has no effect. Gate explicit values to supported families or the verified OpenRouter Anthropic boundary before returning them.
  - confidence: 0.9
