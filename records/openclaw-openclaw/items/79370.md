## Summary

Keep open: the reported OpenRouter Anthropic cache-retention bug is valid, but the latest PR head is not merge-ready because it drops explicit retention for native OpenRouter routes behind custom provider ids and has source-visible test/type defects.

## What This Changes

The PR threads explicit `cacheRetention` into the OpenRouter Anthropic cache marker wrapper so `long` can emit `ttl: "1h"` and `none` can skip system/developer cache markers, with focused regression tests.

## Best Possible Solution

Keep the fix in the agent cache-retention/OpenRouter wrapper boundary, but resolve retention with the same endpoint-class facts used for marker insertion, repair the tests, and refresh proof for the final head.

## Reproduction Assessment

Yes, source-reproducible. Current main returns before explicit OpenRouter `cacheRetention` is read and then hardcodes no-TTL system/developer markers; the PR discussion also includes before/after outbound proxy logs for the intended route.

## Solution Assessment

No. The intended approach is the right area, but the latest patch is not the best form because retention eligibility is not aligned with endpoint-class OpenRouter routing and the new tests have blocking defects.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.96

Full review comments:

- **[P2] Resolve retention for endpoint-class OpenRouter routes:** `src/agents/pi-embedded-runner/prompt-cache-retention.ts:41-42`
  - body: This eligibility check only recognizes `provider === "openrouter"`, but the cache-marker wrapper also supports custom provider ids whose `baseUrl` is native OpenRouter. Since `applyPostPluginStreamWrappers` resolves retention before the wrapper sees endpoint-class facts, explicit `long` on those native OpenRouter aliases is dropped and the wrapper still emits 5-minute markers.
  - confidence: 0.93
- **[P1] Pass the retention enum to the helper:** `src/agents/pi-embedded-runner/proxy-stream-wrappers.test.ts:268`
  - body: `runSystemCacheWrapper` now accepts `"short" | "long" | "none"`, but this call passes `{ cacheRetention: "long" }`. That cannot typecheck, and at runtime the wrapper would treat the object as an unknown retention value and omit the `ttl`, so the test cannot pass or prove the custom-provider case.
  - confidence: 0.98
- **[P2] Use an eligible model in the none-retention wrapper test:** `src/agents/pi-embedded-runner/proxy-stream-wrappers.test.ts:256`
  - body: The wrapper returns early unless the model is an Anthropic ref on a verified OpenRouter route, but this test calls it with `{}`. The payload patch never runs, so the thinking block keeps its stale `cache_control` and the test does not exercise the sanitizer path it claims to cover.
  - confidence: 0.96
