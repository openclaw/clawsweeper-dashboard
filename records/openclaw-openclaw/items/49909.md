## Summary

Keep open. Current main and the latest release still build ClawDock Docker Compose invocations from an explicit file list that includes only `docker-compose.yml` plus optional `docker-compose.extra.yml`, so a user-created `docker-compose.override.yml` is still omitted on helper-driven recreate paths.

## What This Changes

Request ClawDock helpers to include user-created `docker-compose.override.yml` when building Docker Compose file arguments for start/recreate commands.

## Best Possible Solution

Extend `_clawdock_compose()` to append the standard override file deliberately, document the supported overlay order, and add focused coverage for the generated Compose argument list.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: create `docker-compose.override.yml`, then run a helper recreate path such as `clawdock-stop` followed by `clawdock-start`; the wrapper passes an explicit `-f` list that never includes that file.

## Solution Assessment

Yes. Since ClawDock already constructs an explicit Compose file list, the narrow maintainable fix is to append every supported overlay in a documented order instead of relying on Compose auto-discovery.
