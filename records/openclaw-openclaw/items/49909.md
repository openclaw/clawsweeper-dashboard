## Summary

Keep this open. Current main and the latest shipped release still build ClawDock helper Compose commands from `docker-compose.yml` plus optional `docker-compose.extra.yml`, so a standard `docker-compose.override.yml` remains silently omitted from recreate flows.

## What This Changes

The issue asks ClawDock's helper wrapper to include a standard `docker-compose.override.yml` so `clawdock-start` and stop/start flows preserve user-supplied Compose overrides.

## Best Possible Solution

Add conditional `docker-compose.override.yml` loading through `_clawdock_compose()` in a documented order, preferably base plus generated extra plus user override, with focused syntax and ordering proof.

## Reproduction Assessment

Yes for source-level reproduction: current main's `_clawdock_compose()` never appends `docker-compose.override.yml`, so helper commands using explicit `-f` flags cannot load that file. I did not run Docker in this read-only review because source plus the Docker Compose contract is enough to prove the path.

## Solution Assessment

Yes. The narrow maintainable fix is to append the existing standard override file through the same helper seam, document the intended merge order, and add focused coverage or command proof.
