## Summary

Keep this open: current main and v2026.5.20 still build ClawDock compose commands from the base file plus optional generated extra overlay only, so standard override files remain omitted from stop/start recreate flows.

## What This Changes

The issue requests that ClawDock helper compose commands include `docker-compose.override.yml` when explicit `-f` arguments are used.

## Best Possible Solution

Add `docker-compose.override.yml` to `_clawdock_compose()` when present, after the generated extra overlay, and document the supported overlay order without adding a new config knob.

## Reproduction Assessment

Yes for source-level reproduction: current main's `_clawdock_compose()` never appends `docker-compose.override.yml`, while it uses explicit `-f` flags that define the complete Compose file list. I did not run Docker in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is to append the standard override file through the existing helper seam, keep the order explicit, and add focused syntax or command proof.
