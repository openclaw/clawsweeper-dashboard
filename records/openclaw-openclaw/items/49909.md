## Summary

Keep this open: current main and the latest shipped release still omit `docker-compose.override.yml` from ClawDock's explicit Compose file list, but an open linked PR now owns the narrow fix path.

## What This Changes

The issue asks for ClawDock helper commands to include `docker-compose.override.yml` when explicit Compose `-f` arguments are used.

## Best Possible Solution

Yes for source-level reproduction: current main's `_clawdock_compose()` passes explicit `-f` arguments for only the base and extra Compose files. I did not run Docker in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is to append the standard override file through the existing helper seam, keep overlay order explicit, and cover the argument order with a focused regression test.
