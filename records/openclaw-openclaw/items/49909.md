## Label Justifications

- `P2`: This is a normal-priority ClawDock helper bug with a clear narrow fix and limited Docker setup blast radius.
- `impact:data-loss`: The helper can silently drop user Compose overlay configuration such as env vars, mounts, and ports during recreate paths.

## Summary

Keep open. Current main and the latest release still build ClawDock Compose commands from an explicit file list that includes only the base file plus the optional extra overlay, and the related open PR only covers the sandbox overlay, not the standard override file.

## What This Changes

The issue asks ClawDock to preserve the standard Docker Compose override file when its helper builds explicit Compose `-f` arguments.

## Best Possible Solution

Extend `_clawdock_compose()` to append the standard override file in a deterministic documented order, then add focused regression coverage and docs for the supported overlay list.

## Reproduction Assessment

Yes, at source level. Create `docker-compose.override.yml`, then run a recreate path such as `clawdock-stop` followed by `clawdock-start`; current `_clawdock_compose()` never includes that file in its explicit `-f` list.

## Solution Assessment

Yes. Since ClawDock already constructs an explicit Compose file list, adding supported overlays in a documented order is the narrow maintainable fix rather than relying on Docker auto-discovery.
