## Summary

Keep open. Current main and the latest release still build ClawDock Compose commands from an explicit file list that includes only the base file plus optional extra overlay, so the reported standard override file is still skipped.

## What This Changes

The issue requests that ClawDock include `docker-compose.override.yml` when `_clawdock_compose()` builds its explicit Docker Compose file list.

## Best Possible Solution

Extend `_clawdock_compose()` to append the standard override file in a deterministic documented order, then add focused regression coverage and docs for the supported overlay list.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: create `docker-compose.override.yml`, then run a helper recreate path such as `clawdock-stop` followed by `clawdock-start`; the wrapper passes an explicit `-f` list that never includes that file.

## Solution Assessment

Yes. Since ClawDock already constructs an explicit Compose file list, appending supported overlays in a documented order is the narrow maintainable fix.
