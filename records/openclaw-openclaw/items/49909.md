## Summary

Keep this open. Current main and the latest release still route ClawDock through an explicit Compose file list that includes only the base file plus optional `docker-compose.extra.yml`, so a user-created `docker-compose.override.yml` is still omitted on helper-driven recreate paths.

## What This Changes

Load `docker-compose.override.yml` from ClawDock helper commands when present so user overrides survive `clawdock-start` and stop/start recreates.

## Best Possible Solution

Add explicit `docker-compose.override.yml` inclusion in `scripts/clawdock/clawdock-helpers.sh`, document the overlay order in ClawDock and Docker install docs, and add focused coverage for the generated Compose argument list. Treat status visibility for effective running config as a separate follow-up.

## Reproduction Assessment

Yes. The source path is enough to reproduce the bug: any helper call through `_clawdock_compose()` will pass an explicit `-f` list that never includes `docker-compose.override.yml`, so a stop/start recreate cannot apply that file.

## Solution Assessment

Yes, the requested direction is the narrowest maintainable fix. Because ClawDock already uses explicit `-f` arguments, relying on Compose auto-discovery is not available; the helper must append every intended overlay in a deliberate order.
