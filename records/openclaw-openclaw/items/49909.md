## Summary

Keep this open. Current main and the latest release still build ClawDock Compose invocations from an explicit file list that includes only `docker-compose.yml` plus optional `docker-compose.extra.yml`, so user-created `docker-compose.override.yml` remains omitted on helper-driven recreate paths.

## What This Changes

The issue asks ClawDock to include a standard `docker-compose.override.yml` overlay when its helper wrapper starts or recreates the Docker gateway.

## Best Possible Solution

Extend the ClawDock helper to append `docker-compose.override.yml` deliberately, document the supported overlay order, and add focused coverage for the generated Compose argument list.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: any ClawDock command that recreates the gateway goes through `_clawdock_compose()`, which passes an explicit `-f` list that never includes `docker-compose.override.yml`.

## Solution Assessment

Yes. Since ClawDock already constructs an explicit Compose file list, the narrow maintainable fix is to append every supported overlay in a documented order rather than relying on Compose auto-discovery.
