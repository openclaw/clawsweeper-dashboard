## Summary

Keep this open. Current main still builds ClawDock compose commands from only `docker-compose.yml` plus optional `docker-compose.extra.yml`, so a standard `docker-compose.override.yml` is still silently omitted from `clawdock-start`/`clawdock-stop` recreate flows.

## What This Changes

The issue asks for ClawDock helper commands to include `docker-compose.override.yml` when they pass explicit Docker Compose `-f` files.

## Best Possible Solution

Add `docker-compose.override.yml` to the ClawDock compose-file list with documented ordering, then cover the helper behavior with a focused test or command proof without changing unrelated Docker setup behavior.

## Reproduction Assessment

Yes for source-level reproduction: current main's `_clawdock_compose()` never appends `docker-compose.override.yml`, so `clawdock-start` cannot pass that file to Docker Compose. I did not run Docker because the review was read-only and source plus dependency contract evidence is sufficient.

## Solution Assessment

Yes. The proposed fix shape is the narrow maintainable path: append an existing `docker-compose.override.yml` through the same helper seam, document the supported overlay order, and add focused coverage or command proof.
