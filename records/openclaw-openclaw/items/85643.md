## Summary

Keep open for maintainer review: current main is source-reproducible, and the PR appears coherent with focused coverage and current-head terminal proof. ## What This Changes

The PR updates session model override persistence, status reporting, Gateway `sessions.patch`, docs, and regression tests so explicit configured-default model selections can be preserved as user session pins while clear paths still unpin.

## Best Possible Solution

Land after maintainers explicitly accept the strict configured-default chat pin semantics, or adjust the chat command policy while preserving exact user pins through a distinct clear-versus-pin contract.

## Reproduction Assessment

Yes at source level: current main clears stored override fields when `selection.isDefault` is true, so an explicit selection equal to the configured default cannot survive as a user pin. I did not run a failing current-main command because this review is read-only.

## Solution Assessment

Mostly yes: centralizing the distinction in `applyModelOverrideToSessionEntry` and threading an explicit preserve flag is a maintainable fix. The open question is whether maintainers want the chat `/model <configured-default>` compatibility break or a separate strict-pin path.
