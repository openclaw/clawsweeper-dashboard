The issue reports that sandbox-enabled agents cannot deliver generated files through WebChat or QQBot: WebChat marks them unavailable outside allowed folders, and QQBot leaves media undispatched.

## Best Possible Solution

Review, land, request changes on, or replace https://github.com/openclaw/openclaw/pull/92872 for the QQBot scoped-media bridge, while keeping the WebChat empty-roots overlap coordinated with https://github.com/openclaw/openclaw/issues/67915 and https://github.com/openclaw/openclaw/pull/78034.

## Reproduction Assessment

Yes. Source inspection shows current main drops scoped media access before QQBot media resolution and can hard-deny WebChat local attachments before server metadata authorization; I did not run the live Docker/WebChat/QQBot scenario in this read-only review.

## Solution Assessment

Yes, the best direction is to preserve the existing scoped mediaAccess/localRoots/readFile contract through QQBot-owned send paths and handle the WebChat UI overlap through its focused Control UI issue, not add broad mounts or a new config bypass.
