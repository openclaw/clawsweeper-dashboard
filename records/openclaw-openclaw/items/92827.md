Fix OpenClaw issue https://github.com/openclaw/openclaw/issues/92827. Sandboxed agents with workspaceAccess=rw can generate files under /workspace, but outbound file delivery to WebChat and QQBot fails because sandbox media paths and QQBot host media paths are not consistently bridged through scoped local-media access. Keep core plugin-agnostic and do not add config/env options, widen allowed roots to arbitrary host paths, or mount all ~/.openclaw/media into the sandbox. Prefer existing helpers/contracts such as resolveSandboxedMediaSource, normalizeSandboxMediaParams, mediaLocalRoots, and mediaReadFile. Cover singular media fields, top-level mediaUrls arrays, structured attachments, and QQBot <qqmedia>/auto media paths. Make QQBot media sends consume scoped media access/read-file context instead of resolving and uploading raw host paths directly. Add focused tests for sandbox /workspace path normalization and QQBot outbound media delivery, plus WebChat attachment availability where practical.
