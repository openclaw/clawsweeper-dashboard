## Summary

Keep open: the PR targets a plausible sandbox path bug, but it is not merge-ready because it duplicates the existing normalized home-dir contract, lacks focused sandbox regression coverage, and still has no redacted real behavior proof for the reported systemd/sub-agent setup.

## What This Changes

The PR adds a local sandbox home resolver that prefers HOME/USERPROFILE over `os.homedir()` for `~` expansion and short-path display in `src/agents/sandbox-paths.ts`.

## Best Possible Solution

Reuse the shared OS-home resolver for sandbox tilde expansion/display, add focused sandbox coverage, require redacted real systemd/sub-agent proof before merge, and leave broader OPENCLAW_HOME semantics to https://github.com/openclaw/openclaw/issues/54014.

## Reproduction Assessment

No high-confidence live reproduction is present. Current main clearly routes sandbox `~` paths through `os.homedir()` via the filesystem bridge, but a Node 24.15 probe returned HOME from `os.homedir()`, so the reported systemd/sub-agent divergence still needs redacted proof or a focused failing test.

## Solution Assessment

No, not as-is. The maintainable fix is to reuse `resolveOsHomeDir()`/`expandHomePrefix()`, add sandbox regression coverage for edge HOME values, and keep broader OPENCLAW_HOME behavior in the existing tracker.
