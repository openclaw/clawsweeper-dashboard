## Summary

Keep open. Current main still has the sandbox `~` expansion path the PR targets, but the branch is not ready to merge because it adds a raw local HOME resolver instead of reusing the existing normalized OS-home contract, and the external PR still has no after-fix real behavior proof.

## What This Changes

The PR changes sandbox path expansion and display shortening so `~` prefers `process.env.HOME` or `USERPROFILE` before falling back to `os.homedir()`.

## Best Possible Solution

Reuse the shared `resolveOsHomeDir()`/`expandHomePrefix()` contract for sandbox OS-home path resolution and display shortening, add focused sandbox regression coverage, attach redacted real behavior proof, and leave broader `OPENCLAW_HOME` semantics to https://github.com/openclaw/openclaw/issues/54014.

## Reproduction Assessment

No high-confidence live reproduction is present. Source inspection shows sandbox filesystem inputs route `~` through `os.homedir()` on current main, but the reported systemd/sub-agent HOME divergence still needs real output, logs, or a focused failing test.

## Solution Assessment

No, not as-is. The fix direction is narrow, but the PR should reuse the shared normalized OS-home helper and include regression coverage plus real behavior proof before merge.
