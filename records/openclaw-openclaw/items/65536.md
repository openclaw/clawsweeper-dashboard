## Summary

Keep open. Current main still resolves sandbox `~` paths through `os.homedir()`, so this PR addresses a real remaining sandbox write/read path problem, but it still needs maintainer review before merge because the repo now has a shared OS-home helper contract and the PR lacks focused sandbox regression coverage.

## What This Changes

The PR adds a sandbox-local home resolver that prefers `process.env.HOME`, then `process.env.USERPROFILE`, then `os.homedir()`, and uses it for sandbox tilde expansion plus `shortPath()` display.

## Best Possible Solution

Land a sandbox-specific fix that uses the shared OS-home tilde helpers where possible, keeps sandbox root and mount checks unchanged, and adds focused sandbox resolver or bridge tests for the reported HOME/os.homedir mismatch while leaving broader OPENCLAW_HOME policy to #54014.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: with `$HOME` set to the intended user home while `os.homedir()` returns another directory, current `expandPath('~/...')` resolves through `os.homedir()`, and sandbox filesystem resolution calls that path resolver.

## Solution Assessment

No, not as the best final form yet. The PR is directionally correct, but reusing the existing shared OS-home helper and adding focused sandbox tests would be narrower and easier to maintain than introducing an untested local resolver.
