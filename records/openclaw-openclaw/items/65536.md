## Summary

Keep open: this is a plausible sandbox tilde bug fix, but the branch still needs real behavior proof and should reuse the existing normalized OS-home contract before it is safe to merge.

## What This Changes

The PR changes `src/agents/sandbox-paths.ts` so sandbox `~` expansion and short-path display prefer `HOME` or `USERPROFILE` before falling back to `os.homedir()`.

## Best Possible Solution

Reuse `resolveOsHomeDir()` with `expandHomePrefix()` or an equivalent shared helper in sandbox path expansion/display, add focused sandbox tests for HOME/USERPROFILE edge cases, and require redacted real setup proof before merge.

## Reproduction Assessment

No high-confidence live reproduction is present. Current main clearly routes sandbox `~` paths through `os.homedir()`, but a local Node probe showed `os.homedir()` following `HOME`, so the reported service/sub-agent divergence still needs redacted proof or a focused failing test.

## Solution Assessment

No, not as-is. The maintainable fix is to reuse the shared home-dir contract, add sandbox regression coverage, and keep broader `OPENCLAW_HOME` semantics in https://github.com/openclaw/openclaw/issues/54014.
