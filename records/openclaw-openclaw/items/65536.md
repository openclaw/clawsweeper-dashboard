## Summary

Keep open, but not merge-ready: current main still routes sandbox `~` paths through `os.homedir()`, while this PR bypasses the shared normalized OS-home helper and still lacks after-fix proof from the reported systemd/sub-agent setup.

## What This Changes

The PR changes `src/agents/sandbox-paths.ts` so sandbox tilde expansion and short-path display prefer `HOME`/`USERPROFILE` before `os.homedir()`.

## Best Possible Solution

Reuse `resolveOsHomeDir()` with `expandHomePrefix()` or an equivalent shared helper in sandbox tilde expansion/display, add focused sandbox tests for edge HOME values, require redacted real setup proof, and leave broader OPENCLAW_HOME semantics to https://github.com/openclaw/openclaw/issues/54014.

## Reproduction Assessment

No high-confidence live reproduction is present. Source inspection shows current main still routes sandbox `~` paths through `os.homedir()`, but a Node v24.15.0 probe returned HOME from `os.homedir()`, so the reported systemd/sub-agent divergence still needs redacted runtime proof or a focused failing test.

## Solution Assessment

No, not as-is. The maintainable fix is to reuse the shared OS-home resolver contract, add sandbox regression coverage for edge HOME values, and avoid duplicating raw env semantics in a sandbox boundary path.
