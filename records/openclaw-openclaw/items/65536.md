## Summary

Keep open. Current main and v2026.4.27 still contain the sandbox `os.homedir()` behavior this PR targets, and sandbox write/read/edit operations still route tool paths through that resolver. The host-tool fix for #50227 does not cover this sandbox path, while #54014 remains the broader OPENCLAW_HOME policy issue.

## What This Changes

The PR adds a local sandbox home resolver and uses it for `~` expansion in `expandPath()` and path abbreviation in `shortPath()`.

## Best Possible Solution

Land a sandbox-specific fix that reuses `resolveOsHomeDir()` and `expandHomePrefix()` for OS-home tilde expansion, updates `shortPath()` consistently, and adds focused sandbox resolver or bridge tests for HOME/USERPROFILE/fallback behavior while keeping broader OPENCLAW_HOME policy work in #54014.

## Reproduction Assessment

Yes. Source inspection on current main shows `resolveSandboxPath()` reaches `expandPath()`, which still expands `~` and `~/...` with `os.homedir()`; sandbox write/read/edit operations pass paths through that resolver.

## Solution Assessment

No, not as the best final form. The behavior change is directionally correct, but current main already has a shared OS-home helper contract; reusing it with focused sandbox tests is narrower and more maintainable than adding a local helper.
