## Summary

Keep open. The PR is a narrow sandbox path change and current main still uses `os.homedir()` in sandbox tilde resolution, but the contributor has not provided real after-fix proof and the reported HOME/os.homedir divergence still needs live confirmation.

## What This Changes

The PR adds a sandbox-local home resolver in `src/agents/sandbox-paths.ts` and uses it for tilde expansion plus sandbox error path abbreviation.

## Best Possible Solution

Use the shared OS-home tilde helpers in sandbox path resolution, preserve the existing sandbox root and mount checks, add focused sandbox resolver or bridge tests, and leave the broader OPENCLAW_HOME policy to https://github.com/openclaw/openclaw/issues/54014.

## Reproduction Assessment

Unclear. Current main routes sandbox `~` through `os.homedir()` and the sandbox FS bridge uses that resolver, but a Node 24.14.1 probe showed `os.homedir()` follows `process.env.HOME`, so the reported systemd/sub-agent mismatch still needs live output or a focused failing test.

## Solution Assessment

No, not as the final merge shape yet. The direction is narrow, but reusing `resolveOsHomeDir()`/`expandHomePrefix()` and adding sandbox regression coverage would better match the current home-dir contract than introducing an untested local helper.
