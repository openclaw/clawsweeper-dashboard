## Summary

Keep open. Current main still has the sandbox tilde-expansion bug this PR targets, but the proposed local resolver bypasses the existing normalized OS-home contract and needs a small revision plus changelog coverage before merge.

## What This Changes

The PR changes sandbox path expansion and short path display to prefer `process.env.HOME`, then `process.env.USERPROFILE`, then `os.homedir()` instead of calling `os.homedir()` directly.

## Best Possible Solution

Land a narrow sandbox resolver fix that reuses the shared OS-home helper contract, keeps sandbox root and mount checks unchanged, adds focused regression coverage for HOME/USERPROFILE/os.homedir mismatch and poisoned env values, and leaves broader `OPENCLAW_HOME` policy to #54014.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: when `$HOME` points at the intended user home but `os.homedir()` returns another directory, current `resolveSandboxInputPath('~/...')` expands through `os.homedir()`, and sandbox filesystem operations call that resolver.

## Solution Assessment

No, not in the current PR form. The direction is right, but the maintainable fix should use `resolveOsHomeDir()` or `resolveRequiredOsHomeDir()` plus `expandHomePrefix()` instead of adding a local unnormalized resolver.
