## Summary

Keep open. The target is plausible, but this PR still needs real behavior proof and the patch should reuse the existing normalized OS-home helper before changing sandbox path expansion.

## What This Changes

The PR adds a local `resolveHome()` helper in `src/agents/sandbox-paths.ts` and uses it for sandbox `~` expansion and short-path display, preferring `HOME` or `USERPROFILE` before `os.homedir()`.

## Best Possible Solution

Reuse `resolveOsHomeDir()` with `expandHomePrefix()` or an equivalent shared helper in sandbox expansion/display, add focused sandbox tests for HOME/USERPROFILE edge values, and require redacted real setup proof before merge.

## Reproduction Assessment

No high-confidence live reproduction is present. Current main routes sandbox `~` through `os.homedir()`, but Node docs and a Node 24.15.0 probe show `os.homedir()` following `HOME`, so the reported service/sub-agent divergence still needs redacted proof or a focused failing test.

## Solution Assessment

No, not as-is. The maintainable fix is to reuse the shared home-dir contract and add sandbox regression coverage rather than introducing another raw HOME/USERPROFILE resolver.
