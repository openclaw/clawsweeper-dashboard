## Summary

Keep open. The PR targets a real sandbox path-resolution surface that still uses `os.homedir()` on current `main`, but the patch should not merge as-is because it duplicates raw home parsing instead of reusing the shared normalized helper, adds no sandbox regression coverage, and still lacks real behavior proof from the reported setup.

## What This Changes

The PR adds a sandbox-local `resolveHome()` helper and uses it for tilde expansion and sandbox error path shortening in `src/agents/sandbox-paths.ts`.

## Best Possible Solution

Reuse the shared `resolveOsHomeDir()`/`expandHomePrefix()` contract for sandbox path resolution and display shortening, add focused sandbox regression coverage, attach redacted real behavior proof, and leave broader `OPENCLAW_HOME` semantics to https://github.com/openclaw/openclaw/issues/54014.

## Reproduction Assessment

No high-confidence live reproduction is present. Source inspection shows sandbox paths route `~` through `os.homedir()`, but the reported systemd/sub-agent HOME divergence still needs real output, logs, or a focused failing test.

## Solution Assessment

No, not as-is. The fix direction is narrow, but the PR should reuse the shared normalized OS-home helper and include regression coverage plus real behavior proof before merge.
