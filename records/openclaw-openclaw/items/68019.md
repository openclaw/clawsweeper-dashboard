## Summary

Keep this PR open as an implementation candidate, but it is not merge-ready. Current main still only supports fixed-size memory chunking, while the PR branch is stale/dirty and has a blocking memory-host-sdk/plugin-SDK facade issue plus security/product questions around opt-in LLM-assisted indexing.

## What This Changes

The PR adds selectable memory-search chunking strategies, strategy-specific config/schema/help fields, per-strategy reindex metadata, memory-host-sdk exports, and chunking/reindex tests for memory-core.

## Best Possible Solution

Keep this as a candidate only after maintainers choose the memory chunking API shape, rebase it onto current main, move implementation through the package-owned memory-host-sdk and plugin SDK facades, and require focused proof for default compatibility, schema/help alignment, per-strategy reindex metadata, and model-call failure/privacy semantics.

## Reproduction Assessment

Not applicable as a feature PR, but there is a high-confidence current-main verification path. Source inspection shows current main only resolves `tokens`/`overlap` and still indexes text through `chunkMarkdown`, so selectable strategies are not implemented yet.

## Solution Assessment

No, not in the current branch shape. The direction may be useful, but the patch needs rebase, package/facade alignment, and explicit security/product review for model-assisted strategies before it is the narrowest maintainable solution.
