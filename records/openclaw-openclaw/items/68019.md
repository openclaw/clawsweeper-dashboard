## Summary

Keep open. Current main still has fixed-size memory chunking only, so this PR is not obsolete; however the latest branch is incorrect because the new chunking symbols are added under `src/memory-host-sdk` while the plugin SDK facade exports the package source, and it still needs real behavior proof plus model-call security/product review.

## What This Changes

The PR adds configurable memory-search chunking strategies, config/schema/help fields, indexing and reindex metadata changes, SDK exports, and chunking/reindex tests.

## Best Possible Solution

Rework the feature through the package-owned memory-host-sdk source and plugin SDK facade, settle the approved chunking API and model-call behavior, then rebase and validate with real indexing/search proof plus focused tests.

## Reproduction Assessment

Yes for the blocking PR correctness issue: source inspection shows the branch imports new chunking symbols from a package-backed SDK facade that does not export them. The feature itself is not a bug reproduction; current main only supports fixed-size chunking.

## Solution Assessment

No, not in the current branch shape. The feature direction may be useful, but the implementation should live in the package memory-host-sdk boundary and receive security/product review for model-assisted chunking before merge.
