## Summary

Keep this PR open. Current main still only supports fixed-size memory chunking, so the feature is not obsolete, but this branch is not merge-ready because it bypasses the package-owned memory-host-sdk/Plugin SDK facade and introduces model-assisted memory indexing behavior that needs maintainer API and security review.

## What This Changes

The PR adds selectable memory-search chunking strategies, strategy-specific config/schema/help, per-strategy reindex metadata, model-assisted chunking hooks, and chunking/reindex tests.

## Best Possible Solution

Land a maintainer-approved chunking API through the package-owned memory-host-sdk and Plugin SDK surfaces, with explicit docs/config for selectable strategies and model-assisted data handling.

## Reproduction Assessment

Not applicable as a feature PR, but there is a high-confidence current-main verification path: inspect the schema/config and indexing code and confirm only fixed-size `chunkMarkdown` with `tokens`/`overlap` is supported today. The PR’s facade issue is reproducible by checking the import path against the package-backed SDK exports.

## Solution Assessment

No, not in the current branch shape. The direction may be useful, but the implementation should move through the package source of truth and SDK facade, then receive maintainer API/security review for model-assisted strategies.
