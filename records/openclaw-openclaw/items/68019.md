- **[P1] Warn before sending memory text to completion models:** `src/config/schema.help.ts:1053-1054`
  - body: The new LLM-assisted strategies are exposed as ordinary chunking config, but users are not warned that memory contents can be sent to the configured completion model during indexing. Add explicit help/docs/security guidance or a maintainer-approved gate before exposing this path.
  - confidence: 0.86
- **[P2] Bound LLM chunking completions:** `extensions/memory-core/src/memory/manager-embedding-ops.ts:651-658`
  - body: The indexing completion call passes only model, auth, and prompt context, so lumber/hichunk can stall indexing or create unbounded model-call cost if the provider is slow or verbose. Wire this path to an indexing timeout/output limit before merge.
  - confidence: 0.82
- **[P2] Ensure HiChunk windows always advance:** `src/memory-host-sdk/host/chunking/hichunk.ts:708-710`
  - body: A configured windowSize smaller than the prompt plus the first sentence leaves sentenceCount at 0, and the loop advances with startIdx += sentenceCount, so an opt-in hichunk config can hang indexing indefinitely. Reject too-small windows or force at least one sentence per iteration.
  - confidence: 0.78
