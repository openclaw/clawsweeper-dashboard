## Summary

Keep open for maintainer review: current main has not implemented selectable memory chunking, and the branch still has a blocking SDK facade/package-boundary mismatch, missing real behavior proof, and security/product questions around model-assisted indexing.

## What This Changes

The PR adds selectable memory-search chunking strategies, config/schema/help fields, indexing and reindex metadata changes, SDK exports, and chunking/reindex tests.

## Best Possible Solution

Keep this as an implementation candidate, but rebase it onto current main, put the chunking implementation and exports through the package-owned memory-host-sdk and SDK facade, document/approve model-call behavior, and add redacted real indexing/search proof before merge.

## Reproduction Assessment

Yes for the blocking PR correctness issue: source inspection shows the branch imports new chunking symbols from the plugin SDK storage facade while current main backs that facade with the package engine-storage barrel that does not export them. The feature request itself is not a bug reproduction; current main has fixed-size chunking only.

## Solution Assessment

No, not in the current branch shape. The feature direction may be useful, but the implementation should first move through the package-owned memory-host-sdk boundary and receive security/product review for model-assisted chunking.
