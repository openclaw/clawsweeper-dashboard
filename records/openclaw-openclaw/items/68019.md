- **[P2] Avoid splitting every flat semantic distance:** `src/memory-host-sdk/host/chunking/semantic.ts:73-74`
  - body: With identical or otherwise flat embeddings, the percentile threshold becomes `0`, so `d >= threshold` returns every adjacent sentence as a breakpoint. That makes the semantic strategy fragment a uniform document into one chunk per sentence instead of preserving it as no-change content; require a strictly positive/different signal before adding a breakpoint.
  - confidence: 0.9
- **[P2] Bound LLM chunking completions:** `extensions/memory-core/src/memory/manager-embedding-ops.ts:651-658`
  - body: `lumber` and `hichunk` call the configured completion model during indexing, but this call passes no timeout, cancellation signal, or output-token cap even though the helper supports those options. A slow or verbose model can stall memory indexing for each prompt; pass explicit bounds or thread the indexing cancellation/timeout through this call.
  - confidence: 0.85
