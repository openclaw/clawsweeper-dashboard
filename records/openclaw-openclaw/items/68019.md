  - body: The new lumber/hichunk config help says these are LLM-assisted strategies, but it does not warn that memory file text is sent to the configured completion model during indexing.   - confidence: 0.87
- **[P2] Bound LLM chunking completions:** `extensions/memory-core/src/memory/manager-embedding-ops.ts:651-658`
  - body: This completion call passes only model/auth/context, so lumber and hichunk indexing have no visible timeout signal or output-token cap at the call site. A slow or verbose completion provider can stall indexing or create unbounded model-call cost; wire bounded options into this path or tie it to an existing indexing timeout policy.
  - confidence: 0.82
- **[P2] Do not split every tied semantic distance:** `src/memory-host-sdk/host/chunking/semantic.ts:72-74`
  - body: When all adjacent semantic distances tie, the percentile threshold equals that same value and `d >= threshold` marks every boundary as a breakpoint. Identical or flat embeddings would produce one chunk per sentence instead of one stable semantic segment; add a tie/min-spread guard or use a strict above-threshold comparison.
  - confidence: 0.9
