## Summary

## What This Changes

The PR passes `model.id` into the DeepSeek DSML filter/recoverer gate and adds a model-id substring fallback for proxy-hosted DeepSeek models.

## Best Possible Solution

## Reproduction Assessment

Yes at source level: current main only enables the DSML filter when `compat.thinkingFormat` is `deepseek`, so a custom/proxy DeepSeek model without that compat value will not create the filter. I did not reproduce the Feishu feedback loop in a live setup.

## Solution Assessment

No, not yet: the response-side DSML filter is the right owner path, but this patch uses a case-sensitive substring check and lacks regression coverage and real Feishu/proxy proof. A normalized, bounded predicate is safer.
