## Summary

Current main and v2026.6.5 already solve the original unavoidable token-cost problem by returning static acknowledgements for bare resets, but no typed policy exists to select a greeting model or opt between acknowledgement and startup turn; active canonical work may restore model greetings, so this configuration decision remains necessary.

## What This Changes

Add a configurable bare-reset policy that can suppress the model-generated greeting or run it with an explicitly cheaper model.

## Best Possible Solution

Define one typed per-agent-capable bare-reset policy with explicit `ack` and `startupTurn` modes plus an optional model override valid only for `startupTurn`, then apply it consistently to auto-reply and Gateway entry points with the current static acknowledgement as the compatibility default unless maintainers intentionally choose otherwise.

## Reproduction Assessment

Yes for the missing capability: current source and shipped v2026.6.5 have no reset policy or greeting-model override, while both bare reset entry points are directly inspectable. The original unavoidable model-cost behavior itself no longer reproduces because current bare resets use static acknowledgements.

## Solution Assessment

No, the two independent proposed options are not the best final API because they can conflict and omit cross-surface semantics. A single closed reset policy coordinated with https://github.com/openclaw/openclaw/issues/77733 and https://github.com/openclaw/openclaw/issues/45501 is safer and more maintainable.
