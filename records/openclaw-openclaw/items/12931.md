## Summary

Keep open. Current main reduces token spend for ordinary bare chat resets by returning fixed acknowledgements, but the requested typed suppression/model policy is still missing and the reply-producing plus Gateway agent reset paths still build and dispatch the built-in greeting prompt.

## What This Changes

The issue requests configuration to choose a cheaper model for bare `/new`/`/reset` greetings or suppress the greeting/model turn entirely.

## Best Possible Solution

Define one shared, typed reset-greeting policy across ordinary chat acknowledgements, reply-producing reset runs, and Gateway agent resets, coordinated with https://github.com/openclaw/openclaw/issues/45501, https://github.com/openclaw/openclaw/issues/77733, and https://github.com/openclaw/openclaw/issues/75314.

## Reproduction Assessment

Not applicable as a feature request. Source inspection is high-confidence that current main lacks the requested reset-greeting model/suppression config while only ordinary chat bare resets skip the model today.

## Solution Assessment

Unclear. The requested direction is reasonable, but the maintainable solution should be one cross-surface reset-greeting policy rather than ad hoc keys that affect only one reset path.
