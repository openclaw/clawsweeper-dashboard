## Summary

Keep open. Current main and v2026.5.12 reduce one cost path by acknowledging ordinary bare chat resets without a model call, but the requested typed policy for reset-greeting suppression or cheaper greeting-model selection is still missing, and Gateway plus reply-producing reset paths still use the built-in prompt with normal session model selection. Related work on https://github.com/openclaw/openclaw/issues/45501 and https://github.com/openclaw/openclaw/issues/77733 does not fully supersede this feature request.

## What This Changes

Add configuration to use a cheaper model for the bare `/new` or `/reset` greeting turn, or suppress that greeting entirely.

## Best Possible Solution

Define one typed reset-greeting policy across ordinary chat acknowledgements, reply-producing reset runs, and Gateway agent resets, then wire schema, types, help/docs, and focused tests around the chosen contract while reconciling adjacent reset-prompt and no-greeting work.

## Reproduction Assessment

Not applicable as a feature/config request. Source inspection gives high confidence that the requested config keys are absent while Gateway and reply-producing reset paths still use the built-in prompt with normal model selection.

## Solution Assessment

Unclear until maintainers choose the public contract. The maintainable direction is one cross-surface reset-greeting policy rather than separate ad hoc keys that affect only one reset path.
