## Summary

Keep this open. Current main has removed the model call for ordinary bare chat `/new` and `/reset`, but the requested configurable greeting policy is still missing and the gateway agent reset path still builds the hardcoded reset prompt and dispatches an agent run.

## What This Changes

Add configuration to choose a cheaper model for bare `/new`/`/reset` greeting turns or suppress the greeting/model turn entirely.

## Best Possible Solution

Define one typed reset-greeting policy across chat and gateway reset surfaces, then wire it through schema/types/help/docs/tests so users can suppress reply-producing greeting runs or route them to an explicit cheaper model without changing normal reset semantics.

## Reproduction Assessment

Not applicable as a feature request. Source inspection does show the remaining current-main behavior: ordinary chat bare resets are acknowledged without a model run, while the gateway reset path still sends the hardcoded greeting prompt into an agent run.

## Solution Assessment

Unclear. The requested direction is reasonable, but the exact solution needs a maintainer choice for a single cross-surface reset-greeting policy instead of adding ad hoc keys to only one path.
