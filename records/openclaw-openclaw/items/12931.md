## Summary

Current main solves the original token-cost problem by returning fixed acknowledgements for ordinary bare resets, but it does not provide the remaining opt-in agent greeting or reset-specific model selection. The prompt-only request at https://github.com/openclaw/openclaw/issues/45501 overlaps without fully owning these mode and routing requirements, so this issue remains useful pending a maintainer product decision.

## What This Changes

Add a configurable bare-reset execution policy that can use a static acknowledgement or run an agent greeting with an optional reset-specific model.

## Best Possible Solution

Adopt one typed reset-turn policy with static-acknowledgement and agent-run modes, permit a model override only in agent-run mode, and either include prompt customization in that contract or explicitly defer it.

## Reproduction Assessment

Yes. Current source, tests, and docs establish that bare resets produce fixed acknowledgements without a model invocation, while repository search finds no supported configuration for restoring an agent greeting or choosing its model.

## Solution Assessment

No, the two loosely coupled proposed options are not the best final shape. A discriminated reset-turn policy would prevent contradictory settings and give chat and gateway paths one canonical contract.
