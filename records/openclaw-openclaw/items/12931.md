## Summary

Keep open: current main avoids the expensive model call for ordinary bare `/new` and `/reset` by returning static acknowledgements, but the requested configurable reset-turn policy and reset-specific model choice are still missing and need maintainer product direction.

## What This Changes

The issue requests a configurable reset greeting policy so bare `/new` or `/reset` can either use a cheaper greeting model or suppress the model-run greeting.

## Best Possible Solution

Define one typed reset-turn policy that keeps acknowledgement-only behavior as the current default, then explicitly supports any approved startup/greeting turn and reset-specific model override across schema, help, docs, chat, gateway, bootstrap, startup-context, and tests.

## Reproduction Assessment

Yes, source-level. Current main exposes static reset acknowledgements and `agents.defaults.startupContext`, but no `resetGreeting`, `resetGreetingModel`, `resetPrompt`, or shared reset-turn policy.

## Solution Assessment

No, not as two independent keys from the issue body. The maintainable fix is a single closed reset-turn policy so prompt text, startup context, acknowledgement behavior, and optional model routing do not drift.
