## Summary

Keep open: current main has removed the ordinary bare chat model call, but it still lacks a typed reset-greeting suppression/model policy and Gateway/reply-producing reset paths still route the built prompt through normal model selection.

## What This Changes

The issue requests a configurable reset-greeting policy so bare `/new` or `/reset` can use a cheaper greeting model or suppress the greeting/model turn entirely.

## Best Possible Solution

Define one typed reset-greeting policy covering chat acknowledgements, reply-producing reset runs, and Gateway agent resets, then wire schema, types, help/docs, and focused tests for suppression and optional model selection.

## Reproduction Assessment

Not applicable as a feature/config request. Source inspection gives high confidence that the requested keys are absent while reply-producing and Gateway reset paths still use the built-in prompt with normal model selection.

## Solution Assessment

Unclear until maintainers choose the public contract. The maintainable direction is one cross-surface reset-greeting policy rather than separate ad hoc keys that affect only one reset path.
