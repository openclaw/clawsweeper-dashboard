## Summary

Keep open: current main and v2026.6.8 suppress ordinary bare reset model turns, but they do not provide the requested typed reset-greeting policy, reset-specific model choice, or maintainer-approved relationship to the open prompt-override and greeting-restoration work.

## What This Changes

The issue requests a configurable bare `/new` or `/reset` greeting policy, either by choosing a cheaper greeting model or suppressing the greeting/model turn.

## Best Possible Solution

Define one maintainer-approved reset-start policy that covers acknowledgement-only resets, startup/greeting turns, optional prompt override, and optional reset-specific model selection across auto-reply, Gateway, schema, docs, and tests.
