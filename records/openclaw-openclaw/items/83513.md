Keep open: the reported false reading-indicator path is still present on current main, and an open linked PR closes this issue, so the issue should remain open until that PR or an equivalent fix merges.

## What This Changes

Control UI webchat should stop rendering a reading-indicator bubble after a completed assistant response when the chat stream has already been cleared to null.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/83515 or an equivalent focused fix that keeps `stream: null` as null while preserving explicit empty-stream pending feedback, with a DOM regression test for the completed-response state.

## Reproduction Assessment

Yes from source inspection: render `renderChat` with `canAbort: true`, an `onAbort` handler, and `stream: null`, and current main passes an empty stream into `buildChatItems`, which emits the reading indicator. I did not run a live browser/gateway reproduction in this read-only sweep.

## Solution Assessment

Yes: removing or narrowing the render-time `canAbort` fallback is the narrow maintainable fix, provided tests keep the explicit empty-stream loading indicator behavior intact.
