- `impact:session-state`: The failure came from stale abortable session/run state being converted into visible chat stream state after the stream was cleared.

## Summary

Current main implements the requested render-path fix and includes regression coverage; the merged replacement PR supersedes the closed contributor PR and the earlier stale attempt. The fix is current-main-only in this checkout because the latest stable tag still contains the old fallback.

## What This Changes

The issue asks to stop Control UI chat from converting a cleared `null` stream into an empty stream when `canAbort` remains true, which leaves the reading indicator visible after a response.

## Best Possible Solution

Keep the merged render-path fix; if left-panel run status still sticks on a build containing a4f80f905d94f58f78f201cfbec650c6780be675, track that distinct behavior under https://github.com/openclaw/openclaw/issues/84041.

## Reproduction Assessment

Yes. The pre-fix and `v2026.5.18` source path turns `stream: null` plus `canAbort` into `""`, and `buildChatItems` renders that empty stream as the reading indicator; current main no longer follows that path.

## Solution Assessment

Yes. The merged fix keeps `null` as the cleared/no-stream state and preserves `""` as the explicit pending-stream signal, avoiding broader lifecycle or session refresh changes.
