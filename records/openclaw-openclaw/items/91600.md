## Summary

This PR has a real fix for app-agent transcript lookup and the new replay proof satisfies the proof gate, but it should not merge as-is because the new transcript-only stalled classification can permanently suppress stuck-session recovery after one current-turn assistant line.

## What This Changes

The PR resolves app-agent diagnostic transcript files, records a transcript byte cursor on processing turns, uses post-cursor assistant transcript text in attention classification, and adds focused regression coverage.

## Best Possible Solution

Keep the app-agent transcript resolution and cursor boundary, but make transcript-only assistant evidence diagnostic context or a freshness-bounded signal that cannot permanently suppress stuck-session recovery without active work.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: current main only resolves cron-form agent session keys, so non-cron app-agent keys cannot find their transcript. I did not run tests because this review is read-only, but the PR body supplies replayed diagnostic logs for the changed path.

## Solution Assessment

No, not as currently shaped: resolving app-agent transcripts and cursoring old output is the right boundary, but using any post-cursor assistant text to disable recovery indefinitely is broader than the bug requires. A safer fix would preserve transcript context while keeping recovery governed by active work or freshness.
