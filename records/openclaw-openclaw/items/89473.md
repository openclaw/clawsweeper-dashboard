## Label Justifications

- `P1`: Hidden model reasoning can reach user-facing chat channels, making this an urgent privacy and channel-safety bug.
- `impact:security`: The issue crosses a sensitive-information boundary by delivering internal reasoning text to chat recipients.

## Summary

Keep open: current main has merged MiniMax `mm:` tag improvements, but the core final sanitizer, embedded tag promotion/extraction, and block-delivery scanners still do not consistently recognize the reported `<reasoning>` / `<antml:reasoning>` tag family, and a Matrix-specific slice is still open.

## What This Changes

The issue reports hidden reasoning blocks leaking into user-facing chat output when streamed and final assistant sanitizers do not consistently strip reasoning-style tags.

## Best Possible Solution

Unify the reasoning-tag taxonomy across the shared sanitizer, embedded extraction/promotion, stream partitioner, block-delivery scanners, and channel monitor fallbacks, then add focused security regression coverage for closed, split, attributed, namespaced, and literal/code tag cases.

## Reproduction Assessment

Yes, source-reproducible: current main still has sanitizer and embedded delivery regexes that do not recognize the reported `<reasoning>` tag family while those paths prepare user-visible chat text. I did not establish a fresh live MiniMax/LiteLLM channel run in this read-only review.

## Solution Assessment

No, the landed MiniMax namespace fixes are useful but not the complete solution for this issue. The best fix is one canonical reasoning-tag grammar shared by final sanitization, embedded extraction, streaming, and channel delivery boundaries.
