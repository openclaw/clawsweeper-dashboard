## Summary

Keep open: current main still has inconsistent reasoning-tag handling. The merged stream partitioner covers `<reasoning>` in one OpenAI-compatible streaming path, but the final assistant sanitizer, embedded tag promotion/extraction, and embedded delivery scanners still omit the reported `<reasoning>` / `<antml:reasoning>` tags, and the direct fix PR for chat replies was closed unmerged.

## What This Changes

The issue reports hidden model reasoning inside `<reasoning>`-style tags being delivered as visible chat text when providers stream interleaved reasoning and text blocks.

## Best Possible Solution

Unify the reasoning-tag grammar across the shared sanitizer, embedded tag promotion/extraction, stream partitioning, and channel delivery stripping, with focused regression tests for closed, split, attribute-bearing, and literal/code `<reasoning>` cases.

## Reproduction Assessment

Yes, source-reproducible: current main's final sanitizer and embedded delivery regexes do not match the reported `<reasoning>` tag family even though those paths prepare user-visible chat text. I did not run a live MiniMax/LiteLLM channel reproduction in this read-only review.

## Solution Assessment

No, the issue's initial one-line wrapper is not the best complete fix because the shared sanitizer already runs but lacks the tag names, and the embedded scanners also need alignment. The safer fix is a single canonical tag taxonomy applied consistently across final and streaming delivery paths.
