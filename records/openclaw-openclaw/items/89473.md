## Summary

Keep open: current main has partial stream-level protection, but the final sanitizer and embedded delivery tag scanners still omit `<reasoning>` while the report is about reasoning-tagged text reaching chat channels.

## What This Changes

The issue asks OpenClaw to prevent model reasoning/thinking-tagged content from reaching user-facing chat channels when OpenAI-compatible streams interleave visible text with reasoning tags.

## Best Possible Solution

Align the canonical reasoning-tag matcher across final sanitization, embedded delivery stripping, tag promotion, and stream extraction so real `<reasoning>` and relevant `tool_call_thinking` variants are hidden outside literal/code contexts, with focused regression coverage.

## Reproduction Assessment

Yes, source-reproducible: current main's final sanitizer and embedded delivery regexes do not match `<reasoning>` even though chat delivery relies on them. I did not run a live MiniMax/LiteLLM reproduction in this read-only pass.

## Solution Assessment

No, the proposed one-line `sanitizeAssistantText` wrapper is not the best fix because `sanitizeAssistantVisibleText` already calls the reasoning-tag stripper. The maintainable fix is to make the shared tag taxonomy and embedded delivery scanners consistent, then cover the reported stream/final delivery cases.
