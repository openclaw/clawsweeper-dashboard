## Label Justifications

- `P1`: Hidden model reasoning can still reach user-facing chat channels through inconsistent sanitizer coverage, making this an urgent privacy and channel-safety bug.
- `impact:security`: The issue crosses a sensitive-information boundary by exposing internal reasoning text to chat recipients.

## Summary

Keep open: current main has useful MiniMax `mm:` fixes and a stream partitioner that recognizes `<reasoning>`, but the shared final sanitizer, embedded tag promotion/extraction, and some channel monitor fallbacks still use narrower tag grammars, so the reported reasoning-leak class is not fully fixed.

## What This Changes

The issue asks OpenClaw to prevent streamed reasoning/thinking tag content from reaching user-facing chat replies when providers emit interleaved text and reasoning blocks.

## Best Possible Solution

Unify the reasoning-tag taxonomy across the shared sanitizer, embedded extraction/promotion, stream partitioner, block-delivery scanners, and channel monitor fallbacks, then add focused security regression coverage for closed, split, attributed, namespaced, literal, and code-tag cases.

## Reproduction Assessment

Yes, source-reproducible: current main still has final sanitizer and embedded/channel delivery regexes that do not consistently recognize the reported `<reasoning>` tag family. I did not run a fresh live MiniMax/LiteLLM channel reproduction in this read-only pass.

## Solution Assessment

No, the current partial fixes are useful but not the best complete solution. The maintainable fix is a single shared reasoning-tag grammar applied at every delivery boundary, with channel-specific fallbacks aligned or removed where possible.
