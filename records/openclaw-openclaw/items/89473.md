## Label Justifications

- `P1`: Hidden model reasoning can still reach user-facing chat channels through inconsistent sanitizer coverage, making this an urgent privacy and channel-safety bug.
- `impact:security`: The issue crosses a sensitive-information boundary by exposing internal reasoning text to chat recipients.

## Summary

Keep open: current main has useful MiniMax `mm:` fixes, but the shared final sanitizer and embedded/channel fallbacks still do not consistently recognize the reported `<reasoning>` tag family, so the user-visible reasoning leak class is not fully fixed.

## What This Changes

The issue asks OpenClaw to prevent streamed model reasoning/thinking tag content from reaching user-facing chat replies, even when providers interleave visible text and hidden reasoning blocks.

## Best Possible Solution

Unify the reasoning-tag taxonomy across the shared sanitizer, embedded extraction/promotion, stream partitioner, block-delivery scanners, and channel monitor fallbacks, then add focused security regression coverage for closed, split, attributed, namespaced, literal, and code-tag cases.

## Reproduction Assessment

Yes, at source level: current main's delivery sanitizer calls the shared reasoning-tag stripper, and that stripper omits the reported `reasoning` tag family while a sibling stream partitioner recognizes it. I did not run a live MiniMax/LiteLLM channel reproduction in this read-only review.

## Solution Assessment

No, the merged MiniMax namespace fixes are useful partial repairs but not the complete fix. The maintainable solution is one shared reasoning-tag grammar applied consistently across final sanitization, embedded extraction, streaming, and channel fallback boundaries.
