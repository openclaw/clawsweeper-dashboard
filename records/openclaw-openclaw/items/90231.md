## Summary

Keep open for maintainer merge review: current main still treats complete `agent:` values in `to` as sender/delivery context, while this PR repairs that selector boundary with focused coverage and no blocking findings.

## What This Changes

The branch promotes complete agent-scoped `to` values to session selectors across CLI gateway dispatch, gateway agent RPC, reset acknowledgements, and direct embedded agent command delivery.

## Best Possible Solution

Land the boundary fix after maintainer approval, preserving explicit selector precedence and the consumed-selector regression coverage.

## Reproduction Assessment

Yes. Source inspection shows current main derives the session from `to` as sender context and later reuses `request.to`/`opts.to` for delivery planning, so a complete `agent:` key is not consistently treated as the selected session key.

## Solution Assessment

Yes. This is the best fix shape because it repairs the selector boundary and strips the consumed value before delivery, instead of adding a WeChat-specific workaround or a downstream fallback that would leave sibling paths inconsistent.
