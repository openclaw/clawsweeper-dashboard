## Summary

This is a valid, focused root-cause fix for the linked session-routing bug. Current main treats a complete agent-scoped key in `to` as destination context, while the PR consistently promotes it only when stronger selectors are absent and removes the consumed value before delivery; no blocking correctness finding was found, so the PR should remain open for the assigned maintainer’s merge review.

## What This Changes

The PR promotes complete agent-scoped session keys supplied through `to` into session selectors across embedded and gateway agent paths, while preventing those consumed selectors from becoming outbound delivery targets.

## Best Possible Solution

Land the current early selector promotion and delivery sanitization after the assigned maintainer confirms the shared precedence invariant, keeping complete `agent:` keys out of outbound peer fields while leaving ordinary external `to` values unchanged.

## Reproduction Assessment

Yes, from source with high confidence: current main does not promote a complete `agent:` value in `to` to the explicit session selector before destination-derived resolution. The PR body also records the observed wrong-main-session behavior and an after-fix gateway result.

## Solution Assessment

Yes. Classifying the complete key at each ingress before session loading, preserving stronger selector precedence, and stripping the consumed value before delivery addresses the root invariant without adding WeChat-specific routing policy.
