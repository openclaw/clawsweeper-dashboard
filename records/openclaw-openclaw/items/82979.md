## Summary

Keep open for author or maintainer revision, but the PR should not merge as written: the docs changes reverse current main's Codex auto-routing behavior and introduce a manual credential-copy flow that conflicts with the current implementation and docs.

## What This Changes

The PR rewrites OpenAI Codex onboarding docs to require explicit Codex runtime setup and manual auth mirroring, clarifies Codex harness auto mode, and appends an allow-list hint to the agent model override error.

## Best Possible Solution

Keep the current auto Codex routing documentation from main, and split any useful model-override error hint into a narrow change with focused assertion coverage and real CLI output.

The disallowed model override path is source-visible, but the PR body does not show after-patch runtime output for the new error hint.

## Solution Assessment

No. The additive error hint is plausible, but the docs rewrite is not the best fix because it contradicts current main's auto Codex selection and managed auth forwarding; the safer alternative is a small error-message PR that leaves the routing docs aligned with #82864.
