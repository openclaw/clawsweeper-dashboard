## Summary

Keep open. The PR addresses a real current-main gateway channel restart wedge, but it has the protected `maintainer` label and still leaves an explicit lifecycle/security policy tradeoff around replacing a channel while plugin-owned transport code may still be running.

## What This Changes

The PR adds a health-monitor-only force-retire stop option, stale-lifecycle status/restart guards, task-scoped runtime/native approval cleanup hardening, and gateway/infra regression tests for hung channel restart recovery.

## Best Possible Solution

Maintainers should explicitly choose the lifecycle policy: accept this force-retire boundary with documented plugin-owned abort responsibility, or require a generation fence, hard-stop contract, or opt-in recovery mode before automatic replacement lifecycles start.

## Reproduction Assessment

Yes. Current main has a deterministic unit-level reproduction in `src/gateway/server-channels.test.ts` where a hung `startAccount()` ignores abort, `stopChannel` times out, and a later `startChannel` no-ops; related #71412 adds observed Telegram sleep/wake evidence.

## Solution Assessment

Unclear. The PR is a focused recovery patch and hardens OpenClaw-owned state, but it is not clearly the best final fix until maintainers accept or mitigate the concurrent old/new plugin-owned transport risk.
