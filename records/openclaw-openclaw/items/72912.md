## Summary

Keep open. The PR addresses a real current-main channel restart wedge, but it has the protected `maintainer` label, the author explicitly asked for maintainer discussion, and the remaining force-retire overlap tradeoff needs lifecycle/security policy review before merge.

## What This Changes

The PR adds a health-monitor-only `forceRetireOnTimeout` option to `stopChannel`, gates stale lifecycle state updates and cleanup, hardens task-scoped runtime/native approval shutdown, and adds gateway/infra regression tests.

## Best Possible Solution

Maintainers should explicitly choose the lifecycle policy: accept this recovery boundary with documented plugin-owned abort responsibility, or add a generation fence, hard-stop contract, or opt-in mode before automatic replacement lifecycles start.

## Reproduction Assessment

Yes. Current main has a deterministic unit reproduction in `src/gateway/server-channels.test.ts:281`, and related #71412 plus closed duplicate evidence #75480/#75519 describe observed Telegram/Slack restart wedges.

## Solution Assessment

Unclear. The PR is a focused recovery patch and hardens OpenClaw-owned state, but it is not clearly the best final fix until maintainers accept or mitigate the concurrent old/new plugin-owned transport risk.
