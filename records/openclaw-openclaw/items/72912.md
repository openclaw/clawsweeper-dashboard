## What This Changes

The PR adds a health-monitor-only force-retire stop option, stale-lifecycle guards, task-scoped runtime/native approval cleanup hardening, and gateway/infra regression tests for wedged channel restarts.

## Best Possible Solution

Keep the PR open until maintainers choose the lifecycle policy explicitly: either accept automatic force-retire with its plugin-owned transport boundary, add the changelog entry, and require exact-head checks to pass, or require generation fencing, a hard-stop plugin contract, or an opt-in recovery policy before automatic replacement lifecycles are started.

## Reproduction Assessment

Yes. Current main has a deterministic source/test reproduction: a channel task that ignores abort makes `stopChannel` time out, leaves `store.tasks` registered, and causes the following `startChannel` to no-op; #71412 adds real Telegram sleep/wake context.

## Solution Assessment

Unclear. The PR is a narrow bookkeeping recovery for the wedge and hardens OpenClaw-owned cleanup paths, but it is not clearly the best final fix unless maintainers accept concurrent old/new plugin-owned transport risk; generation fencing or a hard-stop/opt-in contract would be safer.
