## Summary

Keep open for maintainer review: the PR targets a real restart wedge, but the protected `maintainer` label, explicit author request for discussion, and unresolved old/new lifecycle overlap make this unsafe to close or automate.

## What This Changes

The PR adds health-monitor-only force retirement for channel stop timeouts, stale lifecycle gating and cleanup, native approval shutdown hardening, regression tests, and a changelog entry.

## Best Possible Solution

Maintainers should either explicitly accept the health-monitor-only recovery tradeoff or add a generation fence, hard-stop contract, or documented plugin abort policy before replacement lifecycles can process events.

## Reproduction Assessment

Yes. Current main has a focused unit reproduction in `src/gateway/server-channels.test.ts:304`, and related reports describe the same stop-timeout wedge for Telegram and Slack after sleep or stale transport recovery.

## Solution Assessment

Unclear. The patch is a narrow repair for the observed wedge and now cleans up OpenClaw-owned runtime state, but it is not clearly the best final fix until the plugin-owned transport overlap is accepted or fenced.
