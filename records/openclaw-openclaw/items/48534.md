## Summary

Keep open. Current `main` still leaves unconfigured legacy `before_agent_start` handlers without a default timeout, and both agent setup and prompt construction can await that hook before the run reaches the model; the linked fixing PR is closed unmerged.

## What This Changes

The issue requests timeout and diagnostic protection for plugin `before_agent_start` hooks that can permanently hang agent runs.

## Best Possible Solution

Add a default fail-open timeout for the legacy `before_agent_start` compatibility hook in the generic hook runner, keep existing per-plugin timeout overrides, log plugin/hook/run context on timeout, and cover the hang with focused regression tests.

## Reproduction Assessment

Yes, from source inspection. A `before_agent_start` handler that returns a never-settling promise is awaited by the embedded setup and prompt helper paths, and current `main` supplies no default timeout for that unconfigured hook.

## Solution Assessment

Yes. The narrow maintainable fix is to give the existing legacy hook-runner path a default timeout while preserving per-plugin overrides; plugin-local OpenViking fixes do not protect other plugins from the same hang class.
