## Summary

Keep open. Current `main` still supports legacy `before_agent_start` but does not give that modifying hook a default timeout, so an unconfigured never-settling handler can still block the model-resolution or prompt-build path; the linked fixing PR is closed unmerged.

## What This Changes

The issue requests timeout protection and diagnostics so a hung plugin `before_agent_start` hook cannot permanently block agent runs.

## Best Possible Solution

Add a default fail-open timeout for the legacy `before_agent_start` compatibility hook in the generic hook runner, preserve explicit per-plugin timeout overrides, log enough hook/plugin/run context on timeout, and add focused regression coverage.

## Reproduction Assessment

Yes, source-reproducible. A `before_agent_start` handler that returns a never-settling promise is awaited by setup and prompt compatibility paths, and current `main` supplies no default timeout for that unconfigured hook.

## Solution Assessment

Yes. The narrow maintainable fix is in the generic hook runner/default hook budget so all legacy hook providers are protected; plugin-local OpenViking fixes do not cover the same failure class for other plugins.
