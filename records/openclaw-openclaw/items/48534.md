## Summary

Keep open. Current main still has no default timeout for unconfigured legacy `before_agent_start` modifying hooks, and agent setup/prompt construction still await that hook before the model call; the related attempted fix is closed unmerged.

## What This Changes

The issue requests a timeout and diagnostic guard for plugin `before_agent_start` hooks so a hung plugin cannot block agent runs indefinitely.

## Best Possible Solution

Add a default fail-open timeout for legacy `before_agent_start` in the generic hook runner, preserve existing per-plugin overrides, log plugin/hook/run context on timeout, and add focused regression coverage.

## Reproduction Assessment

Yes, from source inspection rather than a live run. A `before_agent_start` handler returning a never-settling promise is awaited in agent setup and prompt construction, and current `main` gives that hook no default timeout.

## Solution Assessment

Yes. The narrow maintainable fix is a default hook-runner timeout for the legacy compatibility hook; plugin-local OpenViking fixes and opt-in config do not protect other unconfigured plugins.
