## Summary

Keep open. Current main has per-plugin hook timeout configuration, but an unconfigured legacy `before_agent_start` handler can still hang forever because that hook has no default modifying-hook timeout and agent preparation still awaits it before model/prompt processing.

## What This Changes

The issue asks OpenClaw to bound `before_agent_start` plugin hooks so a hung plugin cannot silently block an agent run.

## Best Possible Solution

Add a default fail-open timeout for legacy `before_agent_start` hooks in the hook runner, keep the existing per-plugin overrides for tuning, log the plugin and hook context on timeout, and add focused regression coverage.

## Reproduction Assessment

Yes, from source inspection rather than a live run. A `before_agent_start` handler that returns a never-settling promise is awaited by the agent setup/prompt paths, and current main does not assign that hook a default timeout.

## Solution Assessment

Yes. The narrowest maintainable fix is in the generic hook runner default policy; plugin-local fixes help individual plugins, but core should fail open around hung compatibility hooks without adding another top-level config knob.
