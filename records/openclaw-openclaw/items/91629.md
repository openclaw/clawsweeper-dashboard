## Best Possible Solution

Keep the AST guard, but replace active-runtime whole-file exemptions with exact baselines for the currently accepted callsites or patterns, then add a regression test proving that a second legacy write in each baselined file still fails.

## Solution Assessment

No, not in its current boundary. An AST guard is appropriate, but exact existing-callsite baselines are safer and more faithful to the requested “prevent new regressions” contract than skipping whole production files.
