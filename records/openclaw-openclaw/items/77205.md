## Summary

Keep open: current main has not superseded this lifecycle work, and this draft/protected PR still has source-visible correctness blockers plus a missing runtime behavior proof gate.

## What This Changes

This PR adds a `plugin-sdk/channel-message` lifecycle surface, durable send-context delivery, message receipts and rendered batch plans, reply-pipeline facade migrations across bundled channels, plugin-loader fallback changes, docs, and changelog updates.

## Best Possible Solution

Align adapter-declared durable capabilities with the public lifecycle contract, pass durability through to outbound delivery, then merge only after maintainer SDK/channel review and representative runtime proof or an explicit proof override.

## Reproduction Assessment

Not applicable as a user bug, but yes for the PR blockers: source inspection at f99a463f shows the capability map/type mismatch and the hard-coded required queue policy.

## Solution Assessment

No. The lifecycle direction looks plausible, but this patch should first make the public capability contract match adapter declarations, honor best-effort durability, and provide representative runtime proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Expose side-effect durable capabilities on adapters:** `src/channels/plugins/outbound.types.ts:54-64`
  - body: The new lifecycle surface advertises `reconcileUnknownSend`, `afterSendSuccess`, and `afterCommit`, and requirement checks validate requested capabilities against `outbound.deliveryCapabilities.durableFinal`. This adapter type omits those keys, so typed channel adapters cannot declare support for the advertised side-effect capabilities.
  - confidence: 0.91
- **[P2] Honor best-effort send contexts:** `src/channels/message/send.ts:214`
  - body: `withDurableMessageSendContext` accepts `durability: "best_effort"`, but `ctx.send` always calls `deliverOutboundPayloads` with `queuePolicy: "required"`. A best-effort caller will still fail closed if queue persistence fails instead of falling back to direct send.
  - confidence: 0.9
