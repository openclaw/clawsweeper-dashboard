## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Require reconciliation for durable final sends:** `src/channels/turn/durable-delivery.ts:128-135`
  - body: `deliverInboundReplyWithMessageSendContext` derives requirements without `reconcileUnknownSend`, then can enqueue a required durable send. Recovery later refuses blind replay for `unknown_after_send`, so a crash after platform send start can move a required final to failed instead of proving or replaying it.
  - confidence: 0.94
- **[P1] Bootstrap adapters before unknown-send reconciliation:** `src/infra/outbound/delivery-queue-recovery.ts:143-145`
  - body: Recovery calls `loadChannelMessageAdapter` directly, but that loader only reads the active registry. On startup before plugin hydration, reconciliation can be skipped and an `unknown_after_send` entry can be treated as unrecoverable even if the bundled channel has a reconciler.
  - confidence: 0.92
- **[P2] Respect requested best-effort durability:** `src/channels/message/send.ts:128-133`
  - body: `withDurableMessageSendContext` accepts `durability: "best_effort"`, but `ctx.send` always passes `queuePolicy: "required"`. Best-effort callers will throw on queue persistence failures instead of falling back to direct send.
  - confidence: 0.95
- **[P2] Run commit hooks for non-queued sends:** `src/infra/outbound/deliver.ts:1139-1140`
  - body: When delivery runs without a queue id, this branch returns the direct-send results before `runOutboundDeliveryCommitHooks(...)` can run. Adapters using `afterCommit` for cleanup miss the hook on best-effort fallback and skip-queue sends.
  - confidence: 0.9
- **[P2] Preserve the original send error when cleanup fails:** `src/infra/outbound/deliver.ts:265-271`
  - body: The send lifecycle catch path awaits `afterSendFailure` directly, so a cleanup-hook failure can replace the original platform send error. Retry and diagnostics should continue to report the real send failure.
  - confidence: 0.88
- **[P2] Preserve structured failed send results:** `src/channels/message/send.ts:205-209`
  - body: When `ctx.send` returns `{ status: "failed" }`, `sendDurableMessageBatch` awaits `ctx.fail(result.error)` before returning. If the failure hook throws, callers lose the structured failed result needed by retry and recovery paths.
  - confidence: 0.87
- **[P2] Re-export reply-prefix helpers from channel-message:** `src/plugin-sdk/channel-message.ts:21-25`
  - body: The new facade re-exports `createReplyPrefixContext` but omits `createReplyPrefixOptions` and `createTypingCallbacks`, even though the PR body directs migration users to get those helpers through `plugin-sdk/channel-message`. That creates an SDK compatibility regression for plugin authors leaving the deprecated subpath.
  - confidence: 0.9
