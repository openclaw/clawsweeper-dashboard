## Summary

Keep open: current main does not include the new channel-message SDK surface, and the latest PR head still has two lifecycle correctness issues plus insufficient current-head real behavior proof for the broad durable delivery change.

## What This Changes

The PR adds `openclaw/plugin-sdk/channel-message` and rewires durable outbound/final reply delivery, recovery, live previews, receipts, message-tool sends, docs, changelog, and bundled channel adapters onto that lifecycle.

## Best Possible Solution

Fix the lifecycle hook isolation and recovered-send commit semantics, update the PR body with current-head runtime proof for a durable final-send or recovery path, then land only after SDK/channel owner review and green CI.

## Reproduction Assessment

Yes for the PR findings by source inspection: a successful platform send followed by a throwing `afterSendSuccess` hook is treated as failure, and reconciled-sent recovery returns before commit lifecycle handling. This is not a current-main bug reproduction for the broader feature.

## Solution Assessment

No: the architecture direction fits the plugin SDK/channel boundary, but this head is not ready because post-send hook failures can trigger duplicate delivery handling and reconciled recovery skips `afterCommit`. The safer path is to isolate hook failures, run recovered commit semantics after ack, and require current-head runtime proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Isolate afterSendSuccess hook failures:** `src/infra/outbound/deliver.ts:254`
  - body: `afterSendSuccess` still runs inside the same `try` as the platform send. If the send succeeds and the hook throws, the catch path treats the whole delivery as failed, runs failure cleanup, and rethrows; queued or direct callers can then retry an already-delivered message. Post-send hook failures should be logged/isolated without changing the send result.
  - confidence: 0.89
- **[P2] Run commit hooks after reconciled sent recovery:** `src/infra/outbound/delivery-queue-recovery.ts:267-272`
  - body: When `reconcileUnknownSend` returns `sent`, this branch acks the queue entry and returns immediately. Unlike the replay branch below, it never runs the commit lifecycle, so adapters that rely on `afterCommit` cleanup after a crash can remain inconsistent even though recovery marks the delivery recovered.
  - confidence: 0.78
