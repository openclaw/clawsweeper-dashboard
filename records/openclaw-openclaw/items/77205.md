## Summary

Keep open: current main does not contain this new SDK/message-lifecycle surface, and the exact PR head still has blocking lifecycle correctness issues plus missing real behavior proof for the migrated channel delivery paths.

## What This Changes

The PR adds a new `openclaw/plugin-sdk/channel-message` lifecycle API, durable final-delivery queue and receipt primitives, rendered recovery plans, live-preview finalizers, bundled channel migrations, docs, changelog entries, and tests.

## Best Possible Solution

Honor the requested durability policy, make lifecycle hooks run consistently across queued and direct fallback sends, preserve original send failures when cleanup fails, then merge only after representative runtime durable-delivery proof or an explicit maintainer proof override.

## Reproduction Assessment

Not applicable as a user bug. For the PR blockers, yes: source inspection on exact head shows the hard-coded required queue policy, the queue-only commit hook path, and the cleanup path that can mask the original send error.

## Solution Assessment

No. The lifecycle direction is plausible, but this exact patch should first fix the three send-lifecycle regressions and provide representative runtime proof for the migrated durable/live channel behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Respect the requested best-effort durability:** `src/channels/message/send.ts:132`
  - body: `withDurableMessageSendContext` accepts `durability: "best_effort"` and exposes it on the context, but `ctx.send` still always calls outbound delivery with `queuePolicy: "required"`. Best-effort callers will fail closed on queue persistence failure instead of falling back to direct send, contradicting the new contract.
  - confidence: 0.95
- **[P2] Run commit hooks without a queued intent:** `src/infra/outbound/deliver.ts:1174-1183`
  - body: When delivery falls back to direct send because `queueId` is null, this function returns the successful results without running `runOutboundDeliveryCommitHooks`. Adapters that rely on `afterCommit` for durable lifecycle cleanup or state publication will miss that hook on the best-effort fallback path.
  - confidence: 0.9
- **[P2] Preserve the original send error when cleanup fails:** `src/infra/outbound/deliver.ts:265-269`
  - body: The lifecycle catch block awaits `afterSendFailure` before rethrowing, so if plugin cleanup fails it replaces the original platform send error. Durable delivery and callers should still see the send failure, with cleanup failures logged or attached separately.
  - confidence: 0.88
