## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Require reconciliation before required durable enqueue:** `src/channels/turn/durable-delivery.ts:129`
  - body: `deliverInboundReplyWithMessageSendContext` derives required capabilities without `reconcileUnknownSend`, but recovery now refuses blind replay for `unknown_after_send` entries. A crash after platform send start can therefore move a required delivery to `failed/` for channels that passed preflight but cannot reconcile unknown sends.
  - confidence: 0.94
- **[P1] Bootstrap adapters before unknown-send reconciliation:** `src/infra/outbound/delivery-queue-recovery.ts:143-145`
  - body: Recovery calls `loadChannelMessageAdapter` directly, and that loader only reads the active registry. On startup before plugin hydration, reconciliation can be skipped and an `unknown_after_send` delivery can be moved to `failed/` even if the bundled channel has a reconciler.
  - confidence: 0.92
- **[P2] Respect requested best-effort durability:** `src/channels/message/send.ts:132`
  - body: `withDurableMessageSendContext` accepts `durability: "best_effort"`, but `ctx.send` always passes `queuePolicy: "required"`. Best-effort callers will throw on queue persistence failures instead of falling back to direct send.
  - confidence: 0.95
- **[P2] Run afterCommit hooks for non-queued sends:** `src/infra/outbound/deliver.ts:1178-1183`
  - body: When delivery succeeds without a queue id, this branch returns results without running `runOutboundDeliveryCommitHooks`. Adapters using `afterCommit` for cleanup miss the hook on best-effort fallback and skip-queue sends.
  - confidence: 0.9
- **[P2] Preserve original send error when cleanup fails:** `src/infra/outbound/deliver.ts:265-269`
  - body: The lifecycle catch path awaits `afterSendFailure` directly, so a cleanup-hook error can replace the original platform send error. Durable retry and diagnostics should report the real send failure.
  - confidence: 0.88
- **[P2] Preserve structured failed send results:** `src/channels/message/send.ts:208`
  - body: When `ctx.send` returns `{ status: "failed" }`, `sendDurableMessageBatch` awaits `ctx.fail(result.error)` before returning. If the failure hook throws, callers lose the structured failed result needed by retry and recovery paths.
  - confidence: 0.87
- **[P2] Re-export reply-prefix helpers from channel-message:** `src/plugin-sdk/channel-message.ts:21-25`
  - body: The new facade re-exports `createReplyPrefixContext` but omits `createReplyPrefixOptions` and `createTypingCallbacks`, even though those helpers remain in the reply core. Migrating plugin code can hit missing-export errors.
  - confidence: 0.9
