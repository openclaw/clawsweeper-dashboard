## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Honor best-effort durability:** `src/channels/message/send.ts:132`
  - body: `withDurableMessageSendContext` accepts `durability: "best_effort"`, but `ctx.send` always passes `queuePolicy: "required"`. Best-effort callers will fail closed on queue persistence errors instead of falling back to direct send, contradicting the new contract.
  - confidence: 0.95
- **[P2] Run commit hooks after direct fallback sends:** `src/infra/outbound/deliver.ts:1178-1182`
  - body: When delivery proceeds without a queued intent, this branch returns successful results without running `runOutboundDeliveryCommitHooks`. Adapters that use `afterCommit` for state cleanup or publication miss that hook on best-effort fallback and explicit skip-queue paths.
  - confidence: 0.9
- **[P2] Preserve original platform send failures:** `src/infra/outbound/deliver.ts:265-269`
  - body: The lifecycle catch path awaits `afterSendFailure` directly, so a cleanup-hook error replaces the original platform send error. Durable retry and diagnostics should still see the send failure, with cleanup failure logged or attached separately.
  - confidence: 0.88
- **[P2] Keep failed send results structured:** `src/channels/message/send.ts:208`
  - body: When `ctx.send` returns `{ status: "failed" }`, this path awaits `ctx.fail(result.error)` and can reject with the failure-hook error instead of returning the original structured failed result. Callers relying on the failed status for retry or recovery lose the real send error.
  - confidence: 0.87
