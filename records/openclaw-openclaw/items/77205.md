- **[P2] Expose every durable-final capability on adapters:** `src/channels/plugins/outbound.types.ts:54-64`
  - body: The new durable capability list includes `reconcileUnknownSend`, `afterSendSuccess`, and `afterCommit`, and support validation checks requirements against `outbound.deliveryCapabilities.durableFinal`. This adapter type omits those keys, so TypeScript plugin adapters cannot declare support for the advertised side-effect capabilities.
  - confidence: 0.9
- **[P2] Honor best-effort send contexts:** `src/channels/message/send.ts:214`
  - body: `withDurableMessageSendContext` accepts `durability: "best_effort"` and exposes it on the context, but `send` always calls outbound delivery with `queuePolicy: "required"`. A best-effort caller will still fail closed if queue persistence fails, contrary to the new public durability policy.
  - confidence: 0.84
