## Summary

Keep open: this draft/protected PR is still useful, but the current head introduces a blocking iMessage delivery parity regression by routing durable sends ahead of the existing monitor sanitizer.

## What This Changes

The PR adds message-lifecycle refactor docs, opt-in durable final-reply delivery for selected channel paths, outbound queue identity persistence, and plugin module-loader fallback hardening.

## Best Possible Solution

Preserve the durable-delivery migration, but keep iMessage behavior parity by reusing the monitor sanitizer in the durable path or leaving iMessage channel-owned until parity is covered.

## Reproduction Assessment

Yes for the review finding: source inspection shows the new iMessage durable path can return before deliverReplies, while the old path applies sanitizeOutboundText. No live channel reproduction was run, and the PR feature itself is not a bug report.

## Solution Assessment

No: the durable migration is directionally maintainable, but opting iMessage into durable delivery before preserving its monitor sanitizer is not the safest implementation.
