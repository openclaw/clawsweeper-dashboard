## Summary

Keep this PR open: the cron warning implementation is useful and has supplied real run-log proof, but the current branch still has a Swift source-compatibility blocker in the generated protocol model.

## What This Changes

This PR adds a configurable cron ghost-run warning for fast successful main-session `next-heartbeat` `systemEvent` handoffs, persists warning codes in cron run logs, exposes them through the Gateway/Swift/UI types, renders them in Control UI run history, and documents the new config.

## Best Possible Solution

Land the cron warning after preserving Swift source compatibility for the new optional run-log field and confirming maintainers accept the default-enabled warning threshold.

## Reproduction Assessment

Yes, from source inspection: current main’s `next-heartbeat` main-session cron path enqueues the event, requests a heartbeat, and returns `ok` without a completion signal, and current main has no ghost-run warning implementation. I did not run a live failing scheduled cron in this read-only review.

## Solution Assessment

No, not yet: the warning approach is a reasonable bounded fix, but the Swift generated model change is not source-compatible as written. The safer path is to keep the new run-log field optional at the wire level and default it to `nil` in the Swift initializer or update all call sites with compile proof.
