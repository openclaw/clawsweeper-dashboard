## Summary

Keep open for maintainer landing review: current main still lacks the requested ghost-run warning, this PR implements the linked observability fix with real behavior proof, and I found no blocking correctness or security finding in the diff.

## What This Changes

The PR adds a config-gated gateway cron warning for fast successful main-session `systemEvent` `next-heartbeat` handoffs, persists warning codes in cron run logs, and updates config schema, protocol models, docs, tests, and changelog.

## Best Possible Solution

Merge the additive warning/run-log marker as the narrow observability fix, and keep stronger delivery-confirmation semantics as a separate cron scheduler design change.

## Reproduction Assessment

Yes. Source inspection on current main shows a main-session `systemEvent` job with `wakeMode: "next-heartbeat"` enqueues the event, requests a heartbeat, and returns `ok` without awaiting agent processing or adding a warning.

## Solution Assessment

Yes for this PR's scope. A scoped warning plus persisted run-log marker is the narrowest maintainable observability change; changing cron to wait for confirmed processing would be a larger scheduler behavior change.
