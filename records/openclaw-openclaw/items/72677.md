## What This Changes

The PR adds a configurable cron ghost-run warning threshold, persists `possible-main-next-heartbeat-ghost-run` on fast successful main-session `systemEvent` `next-heartbeat` handoffs, and updates run-log parsing, protocol models, docs, changelog, and focused tests.

## Best Possible Solution

Land this scoped warning if maintainers accept the additive config/protocol shape, and track any deeper confirmed-processing semantics separately from this observability fix.

## Reproduction Assessment

Yes. Source inspection on current main shows a `main` `systemEvent` job with `wakeMode: "next-heartbeat"` requests a heartbeat and returns `ok`, while the finished-event run-log append path has no warning annotation.

## Solution Assessment

Yes for the narrow observability problem. The PR scopes the warning to the fast successful main-session `next-heartbeat` handoff, keeps delivery behavior unchanged, and uses focused run-log/config/protocol coverage; a confirmed-processing guarantee would be a larger separate change.
