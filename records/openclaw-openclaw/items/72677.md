## What This Changes

The branch adds a cron ghost-run warning for fast successful main-session `systemEvent` `next-heartbeat` handoffs, persists warning codes in run logs, exposes them through Gateway/Swift/UI types, renders them in Control UI history, and documents `cron.ghostRunWarningThresholdMs`.

## Best Possible Solution

Land this as an additive diagnostic if maintainers accept the default threshold/noise tradeoff, keeping the Swift initializer default and protocol/UI/docs surfaces aligned.
