The PR adds a config-gated gateway cron warning and persisted run-log annotation for fast successful main-session `systemEvent` jobs using `wakeMode: "next-heartbeat"`, with schema, docs, protocol model, tests, and changelog updates.

## Best Possible Solution

Merge the additive warning as the narrow observability fix, and keep stronger delivery-confirmation semantics as a separate cron scheduler design change.

## Reproduction Assessment

## Solution Assessment

Yes for this PR's scope. A scoped warning plus persisted run-log marker is the narrowest maintainable observability change; making cron wait for confirmed agent processing would be a larger behavior change.
