## Summary

Keep open: this PR is the active linked implementation for the cron ghost-run warning, and current main still lacks the warning, run-log annotation, config key, and protocol field. The patch looks scoped and green, with sufficient real behavior proof; the remaining step is maintainer acceptance of the new cron config and observability semantics.

## What This Changes

The PR adds `cron.ghostRunWarningThresholdMs`, emits and persists `possible-main-next-heartbeat-ghost-run` for fast successful main-session `systemEvent` `next-heartbeat` handoffs, and updates cron docs, schema, protocol models, tests, and changelog.

## Best Possible Solution

Land this scoped warning after maintainer acceptance, and track any deeper confirmed-processing behavior separately from the run-log observability fix.

## Reproduction Assessment

Yes, from source inspection. Current main’s `main`/`systemEvent`/`next-heartbeat` path requests a heartbeat and returns `ok`, and the current run-log append path has no warning annotation.

## Solution Assessment

Yes for the narrow observability problem. The PR scopes the annotation to the fast `ok` handoff path, keeps scheduler behavior unchanged, makes the threshold configurable, and adds focused coverage; confirmed-processing semantics would be a separate larger change.
