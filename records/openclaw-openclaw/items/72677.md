## Summary

Keep open: current main still has the silent fast main-session next-heartbeat handoff, while this PR provides a focused warning/run-log annotation with sufficient real behavior proof and no blocking code-review findings.

## What This Changes

The PR adds a configurable cron warning for fast successful main-session systemEvent jobs using wakeMode="next-heartbeat", persists the warning in run logs, updates protocol/config/docs, and adds focused tests.

## Best Possible Solution

Merge the additive warning/run-log/config/docs change if maintainers accept the heuristic threshold, and leave stronger delivery-confirmation semantics for a separate scheduler design change.

## Reproduction Assessment

Yes. Source inspection on current main shows a `main` `systemEvent` job with `wakeMode: "next-heartbeat"` enqueues the event, requests a heartbeat, and returns ok without waiting for agent processing or writing a warning.

## Solution Assessment

Yes for this PR's scope. The scoped warning plus persisted run-log marker is the narrowest maintainable observability fix; making cron wait for confirmed agent processing would be a larger behavior change.
