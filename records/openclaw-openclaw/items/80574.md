## Summary

Keep open: this is a plausible direct-Anthropic cache fix, but the patch currently changes the shared helper used by Anthropic Vertex, where upstream automatic caching is not yet supported, and the PR still lacks after-fix real behavior proof.

## What This Changes

The PR removes trailing user-message Anthropic cache markers, sets top-level `cache_control` in the shared payload policy, and updates Anthropic payload-policy tests.

## Best Possible Solution

Gate automatic caching to provider routes where upstream supports it, keep explicit block-level markers for Vertex, and require redacted live Anthropic usage proof before merge.

## Reproduction Assessment

No high-confidence live reproduction is present. Source confirms current main writes trailing user-message cache markers and the PR body reports before-session telemetry, but there is no current-main failing test or live usage transcript that reproduces the billed-token thrash end to end.

## Solution Assessment

No, not as written. Top-level automatic caching may be the right direction for direct Claude API requests, but the shared helper must preserve explicit cache markers for Anthropic Vertex until upstream supports automatic caching there.
