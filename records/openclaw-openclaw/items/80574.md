## Summary

Keep this PR open: the direct-Anthropic direction is plausible, but the patch applies top-level automatic caching through a shared helper that also shapes Anthropic Vertex requests, where upstream docs say automatic caching is not supported yet, and the PR also lacks after-fix real behavior proof.

## What This Changes

The PR replaces trailing-message Anthropic cache breakpoints with top-level automatic `cache_control` and updates the shared payload-policy tests.

## Best Possible Solution

Gate automatic caching to provider routes where upstream supports it, keep explicit block-level markers for Vertex, and add redacted live Anthropic usage proof that demonstrates the cache-write/read change.

## Reproduction Assessment

No high-confidence live reproduction is present. Source confirms current main writes trailing user-message cache markers, and the PR body reports before-session telemetry, but there is no current-main failing test or live usage transcript that reproduces the billed-token thrash end to end.

## Solution Assessment
