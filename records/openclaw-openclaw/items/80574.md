## Summary

Keep this PR open: the direct Anthropic optimization direction matches the new automatic-caching documentation, but the patch currently applies the top-level automatic mode to Vertex/custom Anthropic-message routes where the checked platform docs still require block-level cache markers, and it lacks after-fix live behavior proof.

## What This Changes

The PR removes the trailing-message Anthropic cache breakpoint, adds a top-level `cache_control` field to Anthropic payloads, keeps system-block cache control, and updates the payload-policy tests.

## Best Possible Solution

Gate automatic caching to platforms that document top-level support, keep explicit block-level cache markers for Vertex/custom providers until their contracts support automatic mode, and add provider-specific regression coverage plus live redacted usage proof.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not for the reported live cache-thrash itself: the PR body gives plausible before-fix usage observations, but I did not see or run a real multi-turn after-fix Anthropic session; the Vertex regression is source-reproducible from the shared policy call path and platform docs.

## Solution Assessment

Is this the best way to solve the issue? No as written: the direct Anthropic automatic-caching path is plausible, but the patch should be provider-gated instead of replacing explicit cache markers for Vertex/custom Anthropic-message transports.
