## Summary

Keep open: current `main` and the affected `v2026.5.18` release still contain the source path where non-ack heartbeat `pendingFinalDeliveryText` is replayed before a fresh model call, then swallowed by `target: "none"` delivery. The linked closing PR is still open and unmerged, so this issue should remain open until that fix path lands or is replaced.

## What This Changes

The issue reports that `target: "none"` heartbeats can stop making fresh model calls after stale pending final delivery state is persisted.

## Best Possible Solution

Land a narrow heartbeat/final-delivery fix that clears or bypasses heartbeat-owned stale pending delivery only for internal-only `target: "none"` runs, while preserving durable final delivery recovery for normal user and externally delivered heartbeat turns.

## Reproduction Assessment

Yes, source-reproducible. Current code can replay non-ack `pendingFinalDeliveryText` for a heartbeat, then the `target: "none"` path records skipped delivery without clearing the stale marker or making a fresh model call.

## Solution Assessment

Yes, the best fix belongs in the existing heartbeat/final-delivery state path. The open linked PR has the right narrow shape, but this issue should stay open until that PR lands or a replacement fix is merged.
