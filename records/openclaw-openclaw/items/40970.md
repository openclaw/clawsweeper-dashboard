## Summary

Keep open. Current main still returns a configured `logging.file` path literally, so the requested placeholder behavior is not implemented there. The PR is useful but not merge-ready because the added tests are timezone-dependent, the resolver stays private to logger writes instead of the shared status/log path helper, and the user-facing fix needs a changelog entry.

## What This Changes

The PR adds `YYYY-MM-DD` substitution for configured logging file paths in `src/logging/logger.ts` and adds logger settings plus log-tail regression tests.

## Best Possible Solution

Land a focused shared logging-path resolver used by logger writes, log tailing, channel logs, and daemon status; pin date tests to host-local-safe instants or derive expectations from the same local-date helper; and add the required active changelog entry for the user-facing logging fix.

## Solution Assessment
