## Summary

Keep open. The PR is useful and targets a real logging-path gap, but the current diff still has timezone-fragile tests, leaves daemon status on the raw `YYYY-MM-DD` config path, and needs the required changelog entry before merge.

## What This Changes

The PR adds `YYYY-MM-DD` substitution for configured logger file paths in `src/logging/logger.ts` and adds logger settings plus log-tail coverage.

## Best Possible Solution

Land a shared logging file-path resolver for `YYYY-MM-DD` that is used by logger writes, log tailing, and daemon status, with host-local-safe tests and an active changelog entry.

## Reproduction Assessment

Yes. Current main is source-reproducible because configured `logging.file` values are returned unchanged for settings, tailing, and status, and the PR's new tests are reproducibly timezone-fragile with the frozen instants in the diff.

## Solution Assessment

No. The direction is narrow and maintainable, but the implementation should share placeholder resolution across logging surfaces and make the tests honor host-local date semantics before merge.
