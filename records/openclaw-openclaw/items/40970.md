## What This Changes

The PR replaces `YYYY-MM-DD` in configured `logging.file` paths and adds logger settings plus log-tail regression tests.

## Best Possible Solution

Use one shared logger file resolver that expands leading `~`, substitutes the host-local date for the documented template, preserves rolling pruning and log-tail behavior, and is backed by focused tests plus redacted real-run proof.

## Reproduction Assessment

Yes, source-reproducible. Current docs advertise the dated `logging.file` pattern, while current settings/log-tail code can still follow the literal placeholder instead of the active dated file.
