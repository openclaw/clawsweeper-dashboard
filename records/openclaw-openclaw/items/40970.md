## Summary

Keep open: current main still has a documented `logging.file` placeholder/tail gap, but this PR is not merge-ready because its resolver drops existing home-prefix expansion and the external real-proof gate is still missing.

## What This Changes

The PR expands `YYYY-MM-DD` placeholders in configured `logging.file` paths and adds logger settings plus log-tail regression tests.

## Best Possible Solution

Land a focused logging resolver that expands `~` and `YYYY-MM-DD` consistently for settings, writes, and tailing, with regression tests and redacted real runtime proof.

## Reproduction Assessment

Yes, at source level: docs show `logging.file` with `YYYY-MM-DD`, current settings return the configured file unchanged, and log tailing uses that unresolved path while its fallback only recognizes digit-dated filenames. I did not run a live OpenClaw repro in this read-only review.

## Solution Assessment

No, not as submitted. Placeholder expansion is the right narrow direction, but the resolver must preserve `expandHomePrefix()` and needs real after-fix runtime proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9
