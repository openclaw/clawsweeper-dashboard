## Summary

Keep open: the linked diagnostic gap is valid, but this PR records errors only from the outer bootstrap rejection path while current pricing catalog fetch failures are caught inside the refresh routine, so the described network-failure path would still not surface in health output; the external PR also lacks required real behavior proof.

## What This Changes

The PR adds a modelPricingError field to HealthSummary, tracks a last pricing bootstrap error in cache state, renders it in openclaw health, and adds a changelog entry.

## Best Possible Solution

Record structured per-source pricing catalog failure state where OpenRouter/LiteLLM fetch errors are caught, clear it after a successful refresh, expose the summarized degraded state through health/status surfaces, and add focused regression coverage.

## Reproduction Assessment

Yes, source-reproducible: current main catches rejected OpenRouter/LiteLLM fetches at the per-source catalog calls and HealthSummary currently has no pricing degradation field. A live gateway repro was not run during this read-only review.

## Solution Assessment

No. Tracking a health-visible degraded state is the right direction, but this implementation attaches it to the wrong catch boundary; it should record the failures at the inner per-source catch sites and prove the health/status output with tests and real behavior evidence.
