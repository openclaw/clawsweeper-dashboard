## Summary

Keep open: the patch is directionally useful, but it still misses the common source-level failure path where pricing catalog fetch failures are caught and converted to resolved empty catalogs, and the supplied proof is not real after-fix runtime evidence.

## What This Changes

The PR stores a model-pricing bootstrap error in cache state, exposes it on `HealthSummary`, prints `model-pricing: degraded (...)` from `openclaw health`, and adds a changelog entry.

## Best Possible Solution

Record structured pricing-catalog failure state at the per-source catch paths, clear it only after an actually successful refresh, surface the degraded state through health and gateway status, and add focused regression coverage for both CLI outputs.

## Reproduction Assessment

Yes, source-reproducible: make the OpenRouter/LiteLLM pricing fetch reject or time out on current main, and `refreshGatewayModelPricingCache` catches those failures, logs warnings, resolves, and leaves health/status without pricing degradation state. I did not run a live gateway repro in this read-only review.

## Solution Assessment

No. Tracking health-visible degraded pricing state is the right direction, but this PR attaches the state to the wrong catch boundary and does not yet cover the linked status-command surface.
