## Summary

Keep open: current main has related plugin lifecycle coverage for stale daily rollovers, but the bundled internal `session-memory` hook still only listens to manual command events, so the requested automatic daily reset capture is not fully implemented.

## What This Changes

Request adding a pre-reset lifecycle hook for automatic daily session rotation and updating `session-memory` so it can capture the previous session during that rollover.

## Best Possible Solution

Define one supported reset lifecycle surface for implicit daily and idle rollovers, then update `session-memory`, docs, and focused tests so manual and automatic resets are covered consistently.

## Reproduction Assessment

Yes, by source inspection. Daily reset is documented, `session-memory` is registered and guarded for command events only, and the automatic rollover path emits plugin lifecycle hooks without triggering internal bundled hooks.

## Solution Assessment

Unclear: a new internal event would solve the request, but current main already has typed plugin lifecycle hooks for `session_end` with reason `daily` and `before_reset` for reset flows. Maintainers should choose the supported lifecycle surface before implementation.
