## Summary

Keep open: current main has related plugin lifecycle support for stale daily rollovers, but the bundled `session-memory` hook still only listens to manual command events, so the reported user-facing gap is not fully solved.

## What This Changes

Request an automation hook or lifecycle event for automatic daily session rotation so `session-memory` and other hooks can save or react before rollover.

## Best Possible Solution

Define one supported lifecycle surface for implicit daily and idle rollovers, then update `session-memory`, docs, and focused tests so manual and automatic resets are covered consistently.

## Reproduction Assessment

Yes, by source inspection rather than a live run: daily rollover is documented and implemented, while `session-memory` is registered and guarded for command events only. The existing plugin lifecycle hook covers part of the rollover story but does not make the bundled internal hook fire.

## Solution Assessment

Unclear: a new internal event would solve the request, but reusing or bridging the existing typed plugin lifecycle hooks may be the narrower long-term API. Maintainers should choose the event surface before implementation.
