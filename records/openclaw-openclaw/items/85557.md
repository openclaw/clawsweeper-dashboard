## Summary

Keep open. Current `main` still lets `stripInvalidThinkingSignatures` mutate the latest assistant turn, which conflicts with Anthropic's latest-turn thinking replay contract; the adjacent work at https://github.com/openclaw/openclaw/pull/84448 and https://github.com/openclaw/openclaw/issues/80625 does not make this exact guard obsolete.

## What This Changes

The issue reports that `stripInvalidThinkingSignatures` rewrites the latest assistant message and can make Anthropic reject continued sessions.

## Best Possible Solution

Preserve the latest assistant message verbatim in `stripInvalidThinkingSignatures` while continuing to strip invalid thinking signatures from older replay turns, with focused regression coverage.

## Reproduction Assessment

Yes at source level: a replay history whose latest assistant message contains an unsigned thinking block is modified by `stripInvalidThinkingSignatures`, and the current unit tests assert stripping for a single assistant message. I did not live-reproduce the Anthropic API rejection.

## Solution Assessment

Yes. Adding the same latest-assistant guard used by `dropThinkingBlocks` is the narrowest maintainable fix for this exact bug; broader cache-miss recovery remains complementary.
