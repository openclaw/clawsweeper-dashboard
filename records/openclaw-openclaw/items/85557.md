## Summary

Keep open. Current main still lets `stripInvalidThinkingSignatures` rewrite every assistant message, including the latest assistant turn, while Anthropic's signed-thinking contract and OpenClaw's sibling `dropThinkingBlocks` logic require latest assistant thinking blocks to remain unchanged. Related work at https://github.com/openclaw/openclaw/issues/80625 and https://github.com/openclaw/openclaw/pull/84448 is adjacent, but neither makes this exact latest-turn sanitizer bug safe to close.

## What This Changes

The issue asks OpenClaw to stop `stripInvalidThinkingSignatures` from removing or replacing thinking blocks on the latest assistant message before Anthropic replay.

## Best Possible Solution

Preserve the latest assistant message verbatim in `stripInvalidThinkingSignatures` while continuing to strip invalid thinking signatures from older replay turns, with focused regression coverage and coordination with the open replay-sanitizer PR.

## Reproduction Assessment

Yes at source level: a replay history whose latest assistant message contains an unsigned thinking block is modified by `stripInvalidThinkingSignatures`, and the current unit tests assert that behavior for a single/latest assistant message. I did not live-reproduce the Anthropic API rejection.

## Solution Assessment

Yes for the exact bug: adding the same latest-assistant guard used by `dropThinkingBlocks` is the narrowest maintainable fix. Broader one-shot recovery work in https://github.com/openclaw/openclaw/issues/80625 remains complementary, not a substitute for preserving latest turns before the request.
