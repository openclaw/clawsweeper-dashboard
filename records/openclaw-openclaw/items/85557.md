Keep open. Current main and the latest shipped release still let `stripInvalidThinkingSignatures` rewrite the latest assistant turn, which matches the reported Anthropic replay failure and is not fully covered by the adjacent open replay-sanitizer PR.

## What This Changes

The issue requests preserving the latest assistant message in `stripInvalidThinkingSignatures` while still stripping invalid thinking signatures from older replay turns.

## Best Possible Solution

Preserve the latest assistant message verbatim in `stripInvalidThinkingSignatures`, continue stripping invalid thinking signatures from older replay turns, and add focused regression coverage for both latest and older assistant messages.

Yes for the exact bug: adding the same latest-assistant guard used by `dropThinkingBlocks` is the narrowest maintainable fix. Broader one-shot recovery work in https://github.com/openclaw/openclaw/issues/80625 remains complementary, not a substitute for preserving latest turns before the request.
