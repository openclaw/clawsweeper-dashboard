## Summary

Keep open. Current main has partial final-answer suppression for some cases, but it still does not provide the requested opt-in coalescing/final-only delivery mode; block-streamed text is still flushed before tool execution, and the coalescer has no hold/resume mechanism.

## What This Changes

Request to merge tool-interleaved assistant text blocks from one agent turn into fewer outbound channel messages, likely behind an agent or channel config option.

## Best Possible Solution

Choose an opt-in shared reply-pipeline delivery mode, keep current defaults until maintainers approve otherwise, and cover both final payload assembly and block-streamed tool boundaries with config/schema/docs and regression tests.

## Reproduction Assessment

Yes at the source level: current main still flushes block replies at tool start and lacks hold/resume or requested config keys. I did not run a live Telegram or BlueBubbles reproduction in this read-only review.

## Solution Assessment

No current implementation is a complete solution. The prior hold/resume PR is useful prior art, but the best fix needs a maintainer-approved shared delivery mode with coverage for final payloads and block-streamed tool turns.
