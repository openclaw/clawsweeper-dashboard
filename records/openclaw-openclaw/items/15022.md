## Summary

Keep open. Current main and v2026.6.10 still lack the requested opt-in coalescing/final-only delivery policy for tool-interleaved assistant text; the open related PR only targets a narrower streaming-off ordering bug.

## What This Changes

The issue requests an opt-in way to merge tool-interleaved assistant text blocks from a single agent turn into one outbound channel message.

## Best Possible Solution

Add a maintainer-approved opt-in core delivery policy that covers both final payload assembly and block-streamed text around sequential and parallel tool execution, with docs and regression coverage for text-tool-text-tool-text turns.

## Reproduction Assessment

Yes at source level, but not by a live channel run in this read-only review. Current payload assembly, block-delivery behavior, coalescer API, and tests show that multiple tool-interleaved assistant text blocks can remain separate outbound sends or final reply items.

## Solution Assessment

Unclear until maintainers choose the delivery policy. A narrow coalescer-only patch is not obviously the best fix because current block streaming, direct-send ordering, final-only delivery, and coalescing modes have different compatibility tradeoffs.
