## Summary

Keep open. Current main has partial cleanup for final payloads, but the block-streamed tool-interleaved path still flushes before tool execution and still preserves separate assistant blocks, so the requested one-message coalescing/final-only delivery mode remains unresolved.

## What This Changes

The issue requests a configurable way to merge or defer assistant text blocks from a single tool-interleaved agent turn into one outbound channel message.

## Best Possible Solution

Keep this as the focused core reply-delivery design: add an opt-in coalescing or final-only mode that covers both final payload assembly and block-streamed text around sequential/parallel tool execution, with schema/docs and regression coverage.

## Reproduction Assessment

Yes, at source level: current tests and handlers show text -> tool -> text turns still produce multiple block replies, and tool start still flushes block replies. I did not run a live Telegram or iMessage repro in this read-only review.

## Solution Assessment

No current implementation is sufficient. The narrow maintainable solution is a deliberate opt-in core reply-delivery mode, not prompt-only guidance or the existing blockStreamingCoalesce setting.
