## Summary

Keep open. Current main still lacks an opt-in coalescing/final-only mode for tool-interleaved assistant text, and the latest BlueBubbles repro adds relevant channel impact rather than making the request obsolete.

## What This Changes

Add an opt-in agent or channel setting that merges or defers tool-interleaved assistant text blocks into one outbound message.

## Best Possible Solution

Define one explicit core reply-delivery mode covering final payload assembly and block-streamed text around tool execution, then add schema, docs, and regression tests for text-to-tool turns.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this requests new opt-in delivery behavior. Source inspection and the Telegram/BlueBubbles reports give a high-confidence current behavior path: multiple assistantTexts become separate reply payloads, and block streaming flushes before tool execution.

## Solution Assessment

Unclear. The reply pipeline is the right surface, but maintainers still need to choose the delivery semantics and config scope before a narrow implementation would be safe.
