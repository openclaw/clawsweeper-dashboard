## Summary

Keep open. Current main still lacks an opt-in mode that coalesces or defers tool-interleaved assistant text into one outbound channel message; the current final-payload and block-streaming paths still preserve multiple delivery boundaries.

## What This Changes

Request an opt-in agent or channel config that merges text blocks from one tool-using assistant turn into a single outbound message.

## Best Possible Solution

Define one explicit core reply-delivery mode covering both final payload assembly and block-streamed text around tool execution, then add schema, docs, and regression coverage for text-to-tool-to-text turns.

## Reproduction Assessment

Not applicable as a bug reproduction because this is an enhancement request. Source inspection gives a high-confidence current behavior path: multiple assistantTexts become multiple reply payloads, and block streaming flushes before tool execution.

## Solution Assessment

Unclear. The reply pipeline is the right area, but maintainers still need to choose the delivery semantics and config surface before a narrow implementation would be safe.
