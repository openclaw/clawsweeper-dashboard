## Summary

Keep open. Current main still lacks an opt-in coalescing/final-only delivery policy for interleaved assistant text, and the remaining behavior needs maintainer product judgment rather than conservative cleanup closure.

## What This Changes

Request to add an opt-in reply delivery mode that merges interleaved assistant text blocks from one agent turn into a single outbound channel message.

## Best Possible Solution

Define a maintainer-approved opt-in reply-delivery policy that covers both final payload assembly and block-streamed tool-boundary text, coordinate it with https://github.com/openclaw/openclaw/issues/25592 and https://github.com/openclaw/openclaw/pull/58445, then land schema/docs, regression tests, and live channel proof.

## Reproduction Assessment

Yes, at source level. Current main can still preserve multiple unphased assistant text entries as separate final payloads and still flush buffered block-streamed text at tool-start boundaries; this read-only review did not run live Telegram or BlueBubbles proof.

## Solution Assessment

Unclear. Coalescing is a plausible option, but the narrowest maintainable solution depends on a maintainer choice between merge-all, final-only, suppression, explicit-send, or channel-specific delivery semantics.
