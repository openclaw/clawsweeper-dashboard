## Summary

Keep open. Current main still preserves multiple assistant text blocks as separate final reply payloads and still flushes streamed block replies before tool execution, so the requested opt-in coalescing/final-only delivery mode remains unimplemented.

## What This Changes

Request an opt-in config mode that coalesces text blocks from one tool-interleaved agent turn into a single outbound channel message.

## Best Possible Solution

Implement an opt-in core reply-pipeline delivery mode that covers both final assistantTexts and block-streamed text around sequential and parallel tool execution, with schema/docs and regression coverage.

## Reproduction Assessment

Yes. A source-level reproduction is clear: multiple assistantTexts become multiple final reply items, and streamed text is flushed at tool_execution_start by existing code and tests.

## Solution Assessment

Unclear. Coalescing or final-only delivery is a plausible direction, but the safest maintainable solution needs maintainer agreement on config shape and metadata-boundary rules before implementation.
