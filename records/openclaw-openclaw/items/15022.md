## Summary

Keep open. Current main has partial final-answer hardening, but it still lacks an opt-in policy that coalesces or suppresses tool-interleaved assistant text across both final payload assembly and block-streamed tool-boundary delivery.

## What This Changes

The issue requests an opt-in policy to merge tool-interleaved assistant text blocks from one agent turn into a single outbound channel message.

## Best Possible Solution

Define a maintainer-approved opt-in reply-delivery policy that covers both final payload assembly and block-streamed tool-boundary text, coordinate it with https://github.com/openclaw/openclaw/issues/25592 and https://github.com/openclaw/openclaw/pull/58445, then land schema/docs, regression tests, and live channel proof.

## Reproduction Assessment

Yes, at source level. Current main still flushes buffered block-streamed text before tool execution and can still preserve multiple unphased assistant text entries as separate final payloads; this read-only review did not run live channel proof.

## Solution Assessment

Unclear. Coalescing is plausible, but the narrowest maintainable fix depends on a maintainer choice between merge-all, final-only, suppression, explicit-send, or plugin-hook delivery semantics.
