## Summary

Keep open. Current main still lacks an opt-in tool-interleaved text coalescing mode: final payload assembly preserves multiple assistant text blocks, generic dispatch sends each reply item separately, and block streaming still flushes before tool execution; the closed related PR #16275 remains useful prior art but was not merged.

## What This Changes

The issue requests an opt-in config mode that merges or withholds assistant text blocks interleaved with tool calls so one agent turn produces one outbound channel message.

## Best Possible Solution

Keep this as a focused core reply-pipeline feature: add a documented opt-in delivery mode covering both final payload assembly and block-streamed tool-interleaved turns, with config schema/docs and regression tests for text -> tool -> text -> tool -> text flows.

## Reproduction Assessment

Not applicable as a feature request. Source inspection still confirms the requested option is absent and current main preserves multiple outbound sends for multiple reply items and tool-boundary block flushes.

## Solution Assessment

Unclear. The requested direction is plausible, and #16275 is useful prior art, but maintainers still need to choose the exact opt-in delivery semantics and ensure both final and block-streamed paths are covered.
