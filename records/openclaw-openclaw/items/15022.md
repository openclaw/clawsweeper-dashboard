## Summary

Keep open: current main has partially improved final-payload selection, but the requested turn-level opt-in coalescing/final-only behavior is still missing for block-streamed tool-interleaved text, and the remaining work needs maintainer product judgment.

## What This Changes

The issue requests coalescing assistant text blocks interleaved with tool calls into one outbound channel message instead of rapid-fire channel messages.

## Best Possible Solution

Keep this as the canonical issue for a maintainer-sponsored opt-in reply-pipeline mode that covers both final payload assembly and block-streamed tool-interleaved text, with config/schema/docs and regression tests for text-to-tool-to-text turns.
