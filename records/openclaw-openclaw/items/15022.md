## Summary

Keep open. Current main has a useful partial final-payload canonicalization, but it still lacks the requested configurable coalescing/final-only mode for text emitted around tool calls; the block-reply path still flushes before tool execution, and the recent BlueBubbles repro shows the delivery-policy gap is still active.

## What This Changes

Add an opt-in mode/config to merge or final-only deliver assistant text blocks interleaved with tool calls so messaging channels receive one coherent outbound message per agent turn.

## Best Possible Solution

Define one explicit core reply-delivery mode covering final payload assembly and block replies around tool execution, then add schema/docs and regression coverage for text-tool-text turns.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this requests new opt-in delivery behavior. Source and tests give a high-confidence current behavior path: block replies flush before tool execution and no requested coalescing config exists.

## Solution Assessment

Unclear. The reply pipeline is the right surface, but maintainers still need to choose coalesce versus final-only versus suppression semantics before a narrow implementation is safe.
