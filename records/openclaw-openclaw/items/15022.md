## Summary

Keep open. Current main and v2026.6.10 still lack an opt-in turn-level mode that coalesces or final-only-delivers tool-interleaved assistant text, and the open related Discord ordering PR addresses a different delivery-order bug rather than this single-message policy request.

## What This Changes

Request an opt-in config mode that merges assistant text blocks interleaved with tool calls into one outbound chat message, or otherwise final-only-delivers the turn.

## Best Possible Solution

Keep this as the canonical product decision, then implement an opt-in core reply-pipeline mode covering both final payload assembly and block-streamed tool boundaries with schema, docs, and regression coverage.

## Reproduction Assessment

Yes, at source level. Current main still preserves multiple assistant text blocks as separate final reply items and still flushes block replies at tool start; I did not run a live Telegram, Discord, or BlueBubbles reproduction in this read-only review.

## Solution Assessment

Unclear until maintainers choose the product shape. A hold/defer or final-only mode like the prior candidate is plausible, but it must be opt-in, cover both final and block-streamed paths, and avoid breaking existing message-boundary semantics.
