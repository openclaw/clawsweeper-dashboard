## Summary

Keep open. Current main and the latest release still preserve multiple tool-interleaved assistant text blocks as separate outbound reply payloads, while the available message-tool-only mode is a related workaround rather than the requested opt-in coalescing/final-only delivery policy.

## What This Changes

The issue requests an opt-in way to merge assistant text blocks interleaved with tool calls into one outbound channel message for a single agent turn.

## Best Possible Solution

Add a maintainer-approved, opt-in core reply delivery policy that covers both final payload assembly and block-streamed text around sequential and parallel tool execution, with docs and regression coverage for text-tool-text-tool-text turns.

## Reproduction Assessment

Yes at source level, but not by a live channel run in this read-only review. Current payload assembly, dispatch, coalescer API, and tests show that multiple tool-interleaved assistant text blocks can remain separate outbound sends.

## Solution Assessment

Unclear until maintainers choose the delivery policy. A narrow coalescer-only patch is not obviously the best fix because current message-tool-only mode, broader explicit-send semantics, and final-only/coalesce modes overlap but have different compatibility tradeoffs.
