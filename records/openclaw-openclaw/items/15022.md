## Summary

Keep open. Current main still lacks an opt-in mode that coalesces or final-only-delivers tool-interleaved assistant text; the existing pipeline still preserves multiple final text payloads and intentionally flushes block-streamed text at tool boundaries.

## What This Changes

Request an opt-in configuration mode that merges text blocks from one tool-interleaved agent turn into a single outbound channel message.

## Best Possible Solution

Keep this issue as the canonical product decision, then implement an opt-in core reply-pipeline mode covering both final payload assembly and block-streamed tool boundaries with schema, docs, and regression coverage.

## Reproduction Assessment

Yes at source level. Current main still preserves multiple assistant text blocks as separate final reply items and still flushes block replies at tool start; I did not run a live Telegram or BlueBubbles reproduction in this read-only review.

## Solution Assessment

Unclear until maintainers choose the product shape. A hold/defer or final-only mode like the prior candidate is plausible, but it must be opt-in, cover both final and block-streamed paths, and avoid breaking existing message-boundary semantics.
