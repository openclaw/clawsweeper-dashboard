## What This Changes

The issue requests a configurable mode to merge text blocks from one tool-interleaved agent turn into a single outbound channel message.

## Best Possible Solution

Keep this as a focused core reply-delivery design: add an opt-in coalescing or final-only mode across final payload assembly and block-streamed tool boundaries, with schema/docs and regression coverage.

## Reproduction Assessment

Yes, at source level: current handlers and tests still show text-to-tool boundaries flushing block replies and preserving separate assistant blocks. I did not run a live Telegram or iMessage repro in this read-only review.

## Solution Assessment

No current implementation is sufficient. The maintainable path is an explicit opt-in core reply-delivery mode, not prompt-only guidance or only tuning blockStreamingCoalesce.
