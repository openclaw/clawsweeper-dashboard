## Summary

Keep open. Current main still lacks the requested configurable coalescing or final-only delivery mode: source shows only streamed-block coalescing, a hard pre-tool flush boundary, and tests that preserve the split, while related reports point to this issue as the canonical tracker.

## What This Changes

Request to add a delivery option that merges or defers text blocks from one tool-interleaved agent turn into a single outbound message.

## Best Possible Solution

Define one explicit core reply-delivery policy for tool-interleaved turns, then wire config schema, docs, and regression coverage across final payload assembly and block replies.

## Reproduction Assessment

Yes for the current behavior path, though this is a feature request rather than a broken defined contract. Current main still flushes block replies at tool_execution_start, has no requested coalescing config, and the May 5 BlueBubbles comment reports the same behavior on OpenClaw 2026.5.3.

## Solution Assessment

Unclear. The reply pipeline is the right surface and https://github.com/openclaw/openclaw/pull/16275 is useful prior art, but maintainers need to decide the delivery semantics and metadata boundaries before a narrow implementation is safe.
