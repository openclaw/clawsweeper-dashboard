## Summary

Keep open. Current main still lacks an opt-in coalescing or final-only mode for tool-interleaved assistant text, and recent Telegram, BlueBubbles, and broader channel-leak reports show this is still an active delivery-policy problem rather than completed cleanup.

## What This Changes

Add configurable outbound delivery behavior that merges or defers text blocks from one tool-using agent turn into a single channel message.

## Best Possible Solution

Define one explicit core reply-delivery mode covering final payload assembly and block-streamed text around tool execution, then add schema, docs, and regression coverage for text-tool-text turns.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this asks for new opt-in delivery behavior. Source inspection gives a high-confidence current behavior path: multiple assistantTexts remain separate reply payloads, and block streaming flushes before tool execution.

## Solution Assessment

Unclear. The reply pipeline is the right surface, but maintainers still need to choose coalesce versus final-only versus suppression/message-tool policy before a narrow implementation is safe.
