## Summary

Keep open. Current main has partial final-answer cleanup and an adjacent merged Telegram draft-preservation fix, but it still does not provide the requested opt-in turn-level coalescing/final-only mode for tool-interleaved assistant text across final payload and block-streaming delivery.

## What This Changes

Request an opt-in delivery mode that coalesces or final-only buffers assistant text blocks interleaved with tool calls into one outbound channel message.

## Best Possible Solution

Keep this as the canonical core reply-pipeline design item and implement a maintainer-approved opt-in turn-level coalescing or final-only delivery mode with schema/docs and regression coverage for final payloads, block streaming, sequential and parallel tools, media, and reply metadata.

## Reproduction Assessment

Yes at source level, not live: current main still flushes block replies at tool_execution_start and preserves assistant-message boundaries in the block pipeline, while docs only cover streamed-block coalescing. I did not run a live Telegram or BlueBubbles reproduction in this read-only review.

## Solution Assessment

No current implementation is not the full solution: recent main improves some final-answer and Telegram draft cases, but it does not add the requested generic opt-in coalescing/final-only mode. The best path is a product-approved core reply-pipeline mode rather than a narrow channel-only workaround.
