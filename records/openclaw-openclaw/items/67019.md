## Summary

Close as implemented on current main. Commit cc8a8f1df1cda06518e4888102d686cd18f1a1c9 disables Pi's internal auto-compaction for z.ai/GLM-style routes, including bare `glm-4.7` gateway models, preventing the session-state collapse that the discussion identified as the likely cause of corrupted GLM output.

## What This Changes

The issue asks OpenClaw to stop returning garbled GLM-4.7 output through the Cloudflare AI Gateway/Z.AI route after session/config state becomes corrupted.

## Best Possible Solution

Keep the targeted z.ai/GLM auto-compaction guard on main and let the next release carry it; users with already-corrupted sessions should start fresh or clear the affected session history.

## Reproduction Assessment

Yes for the fixed root cause: #75799 captured z.ai-style OpenAI-compatible requests dropping prior session messages while OpenClaw still recorded full context, and this report's discussion also pointed to session-state corruption. I did not live-test the reporter's exact Cloudflare credentials/route.

## Solution Assessment

Yes. Disabling Pi auto-compaction only for silent-overflow-prone z.ai/GLM routes is narrower and more maintainable than filtering visible text or changing the Cloudflare stream path blindly.
