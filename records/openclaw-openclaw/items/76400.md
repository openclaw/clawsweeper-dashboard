## Summary

Keep open. The reported gateway starvation path is still source-backed on current main, but this PR drops oversized `chat.history` records before the existing placeholder behavior can run, and the title-extraction part now overlaps a newer main fix.

## What This Changes

The PR adds 256 KiB guards before transcript JSON parsing, changes async title extraction to use the bounded sync reader, and adds session-utils tests.

## Best Possible Solution

Land a rebased or replacement fix that avoids multi-MB transcript `JSON.parse` work while preserving the `chat.history` oversized-row placeholder, retaining the current bounded async title reader, and adding focused regression coverage plus a changelog entry.

## Reproduction Assessment

Yes. Current main still has direct `JSON.parse` calls in the recent transcript message and usage paths, and the PR regression is source-reproducible because the new guard returns `null` before `chat.history` can replace an oversized visible row with its placeholder.

## Solution Assessment

No. Skipping expensive parses is the right direction, but this patch should preserve placeholder semantics and avoid replacing current main's newer bounded async title-reader implementation with the older sync-delegation approach.
