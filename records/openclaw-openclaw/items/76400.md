## Summary

Keep open: current main still has the oversized JSON.parse hot paths, and this PR is useful, but the latest diff needs revision before merge because the synthetic oversized record drops active-tree linkage and the branch should preserve current main's async bounded title reader.

## What This Changes

The PR adds a 256 KiB transcript JSONL parse guard, synthesizes oversized chat-history placeholders, skips oversized usage lines, changes async title extraction to reuse bounded title reads, and adds tests plus a changelog entry.

## Best Possible Solution

Rebase the PR onto current main, keep main's async bounded title-reader path, and make the oversized-line guard preserve enough tree metadata to keep active-branch chat.history placeholders without parsing the full payload.

## Reproduction Assessment

Yes, source-reproducible. Current main parses recent transcript and usage lines with `JSON.parse` before any oversized-line guard, and the remaining PR defect is reproducible from source with v3 tree records whose active branch depends on `id`/`parentId`.

## Solution Assessment

No. Skipping oversized JSON parsing and adding the changelog entry are the right direction, but the patch should preserve active-tree linkage for placeholders and keep the current async bounded title-reader implementation.
