## Summary

Keep open: current main and the latest v2026.6.10 release still record a delivered message_tool_only source reply without returning a termination hint, and the open candidate fix has not merged yet.

## What This Changes

The issue reports a regression where delivered message_tool_only source replies keep the embedded agent loop running and can produce repeated visible self-replies in one run.

## Best Possible Solution

Land a targeted guard that prevents repeated implicit current-source message(send) deliveries in one message_tool_only run while preserving legitimate follow-up tool work, with passing regression tests and redacted live Telegram proof.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.10 still return no terminate hint after a delivered message_tool_only source reply, and the core loop continues unless finalized tool results terminate. I did not rerun live Telegram proof in this read-only review.

## Solution Assessment

Unclear. Restoring unconditional terminate would match the reporter workaround but conflicts with https://github.com/openclaw/openclaw/pull/92343; the best fix is a narrower lifecycle invariant accepted by an agent-runtime owner.
