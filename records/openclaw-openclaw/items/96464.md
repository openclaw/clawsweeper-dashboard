## Summary

This PR is still necessary as a focused companion fix for the open Android dependency update: the linked dependency branch raises AndroidX artifacts whose AAR metadata requires compile SDK 37, current main remains on SDK 36, and this PR’s hosted Android CI now passes with the SDK 37 alignment.

## What This Changes

The PR raises the Android app and benchmark `compileSdk` to 37, updates CI to install/cache the Android 37.0 platform, and adjusts the workflow guard to compare the installed platform major against Gradle `compileSdk`.

## Best Possible Solution

Land this SDK alignment with the linked Android dependency update, preserving the app/benchmark/CI guard alignment now that hosted Android CI proves the SDK 37.0 package and Gradle tasks pass.

## Reproduction Assessment

Yes, for the linked dependency branch rather than current main. The dependency PR’s CI logs show AAR metadata failures against android-36, and official Google Maven AAR metadata for the bumped AndroidX artifacts requires compile SDK 37.

## Solution Assessment

Yes. Since the dependency branch intentionally takes AndroidX artifacts that require SDK 37 and hosted CI now installs `platforms;android-37.0`, aligning both Gradle modules and the CI guard is narrower than reverting or special-casing the dependency update.
