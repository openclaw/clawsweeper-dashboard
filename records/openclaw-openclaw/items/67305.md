## Summary

Keep this open. Current main still lacks an operator-selectable readiness-aware public queue/collect mode; existing collect/followup delivery remains debounce-based, while recent steering and subagent announce work is narrower internal prior art.

## What This Changes

Request a public queue collection mode that buffers status/completion notices until the session is ready, then flushes or coalesces them into the next safe reply slot.

## Best Possible Solution

Define a documented core queue contract that waits on generic session/run readiness, coalesces only intended status/completion notices, preserves existing collect/debounce behavior, and includes a bounded fallback.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a feature request; source and docs inspection give high-confidence verification that the requested public mode is absent from current main.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers define the readiness signal, notice classes, coalescing rules, and starvation cap; the likely home is the core queue contract rather than a channel-specific workaround.
