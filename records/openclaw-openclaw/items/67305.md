## Summary

Keep this open. Current main still lacks an operator-selectable readiness-aware queue mode, and the remaining work is a public queue/config policy decision rather than a narrow bug fix.

## What This Changes

Add a readiness-aware queue collection mode that buffers status/completion notices until a session is ready, then flushes or coalesces them safely.

## Best Possible Solution

Define a documented core queue contract that waits on a generic session/run readiness signal, coalesces only eligible notices, preserves existing debounce modes, and includes a bounded fallback.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior. Source and docs inspection give high confidence that the requested public mode is absent from current main.

## Solution Assessment

Unclear until maintainers define the readiness signal, notice eligibility, coalescing rules, and starvation cap. The maintainable direction appears to be a core queue contract rather than a channel-specific workaround.
