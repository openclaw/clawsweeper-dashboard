## Summary

Keep this open. Current main still exposes only the existing timer/debounce-based queue modes, while the one readiness-style deferral path is internal subagent announce prior art rather than a public `messages.queue` mode for ordinary queued chat/status/completion notices.

## What This Changes

Request a public readiness-aware queue mode that holds or coalesces queued status/completion messages until a session is safe to reply, with bounded fallback behavior.

## Best Possible Solution

Add a documented public queue option in core, covering config types/schema/help, `/queue` parsing, queue docs, focused tests, a generic session/run readiness signal, coalescing rules, and a bounded fallback while preserving existing collect/debounce behavior.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main still lacks the requested public readiness-aware queue mode and only offers timer/debounce-based public queue options.

## Solution Assessment

Unclear until maintainers define the readiness contract. The requested surface belongs in core queue handling, but the exact mode name, readiness signal, participating async notices, and starvation cap need owner judgment before implementation is safe.
