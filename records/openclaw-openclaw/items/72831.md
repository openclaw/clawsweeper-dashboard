- **[P2] Use id= for plugin commands:** `skills/obsidian/SKILL.md:135-136`
  - body: Official Obsidian CLI docs list `plugin:enable` and `plugin:disable` with required `id=<id>`, but these examples pass `name="dataview"`. Users copying the cheat sheet will pass the wrong parameter, so update both examples to `id=dataview`.
  - confidence: 0.92
- **[P3] Put vault= before the command:** `skills/obsidian/SKILL.md:45-46`
  - body: Official docs require `vault=<name>` or `vault=<id>` to be the first parameter before the command. This text says to use it on any command but omits that placement rule, so users may append it after the subcommand and not target the intended vault.
  - confidence: 0.85
- **[P3] Point setup at Settings > General:** `skills/obsidian/SKILL.md:30`
  - body: Official Obsidian CLI setup is under Settings > General, where users enable Command line interface and follow the registration prompt. This text sends first-time users to a separate Command-line interface settings page, which can be hard to follow or wrong for the current app.
  - confidence: 0.87
- **[P3] Keep link updates conditional:** `skills/obsidian/SKILL.md:107`
  - body: Official docs say `move` and `rename` update internal links only when the vault's automatic link-update setting is enabled. This sentence guarantees link repair unconditionally, which can make users trust a refactor that Obsidian may not apply.
  - confidence: 0.84
