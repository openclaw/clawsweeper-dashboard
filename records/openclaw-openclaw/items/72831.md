## Summary

Keep open: current main still ships the yakitrak-based bundled Obsidian skill, so the PR is not obsolete. The PR is narrow and mostly docs-only, but it still has one concrete correctness blocker: the official CLI plugin examples use the wrong required parameter.

## What This Changes

The PR rewrites `skills/obsidian/SKILL.md` to require the official `obsidian` binary, remove yakitrak Homebrew installer metadata, and document official Obsidian CLI setup and command examples.

## Best Possible Solution

If maintainers want the bundled Obsidian baseline to move to the official CLI, fix the incorrect plugin examples and keep the setup/command notes aligned with official Obsidian Help. If they do not want this as the bundled default, publish the same official-CLI content as a ClawHub or workspace skill override.

Not applicable as a feature/docs migration. The high-confidence review path is comparing the PR head `SKILL.md` against current main, official Obsidian CLI documentation, the prior PR discussion, and the existing memory-wiki official-CLI adapter.

## Solution Assessment
