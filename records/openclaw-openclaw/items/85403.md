## Summary

Keep open: this PR addresses a real Telegram disappearing-preview/message-delivery bug, but it still needs maintainer direction before merge because it promotes three reply-policy helpers into the public Plugin SDK and duplicates a core dispatch decision earlier in Telegram.

## What This Changes

## Best Possible Solution

Land the Telegram suppression only after maintainers choose and document the reply-runtime API shape, or replace it with a narrower bundled/internal seam while coordinating with https://github.com/openclaw/openclaw/pull/83161.
