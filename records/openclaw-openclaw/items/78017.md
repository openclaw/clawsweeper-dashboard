## Summary

Keep open: current main and the latest release still pin the affected pi-tui renderer, and the pinned dependency still clears terminal scrollback on the content-driven `firstChanged < prevViewportTop` full-redraw path. The open overlapping PR is dirty and implements a different opt-in preservation mode, so it is not a safe canonical replacement for this bug.

## What This Changes

The issue asks OpenClaw to stop long TUI assistant messages from disappearing when pi-tui content-driven full redraws clear terminal scrollback with `ESC[3J`.

## Best Possible Solution

Land a maintainer-approved fix that prevents content-driven TUI redraws from erasing terminal scrollback, preferably in the pi-tui renderer or a narrowly owned dependency patch, with PTY/real-terminal proof and no default behavior regression.

## Reproduction Assessment

Yes, at source level: current main still streams assistant deltas through the pi-tui Markdown path, and the pinned renderer still clears scrollback on `firstChanged < prevViewportTop`. A live PTY reproduction was not run in this read-only review.
