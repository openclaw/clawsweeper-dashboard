## Summary

Keep open: current main still pins `@mariozechner/pi-tui` 0.73.0, whose render path clears terminal scrollback when `firstChanged < viewportTop`, and OpenClaw streaming assistant updates can feed that path. The related TUI scroll issue and open viewport PR do not yet ship a fix for this specific scrollback wipe.

## What This Changes

The issue asks OpenClaw to stop long TUI assistant messages from disappearing from terminal scrollback after pi-tui content full redraws.

## Best Possible Solution

Change the TUI render path so non-resize content redraws never emit `\x1b[3J`, preferably in pi-tui or via a vetted dependency upgrade, with focused regression coverage and live terminal proof; keep broader viewport controls tracked by #44130/#51128.
