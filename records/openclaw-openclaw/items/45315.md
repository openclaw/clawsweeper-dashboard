## Summary

Keep this PR open: it has useful proof and fixes the simple direct abort path, but the current branch is conflicting and the patch still seeds from config defaults rather than active session/channel model state, so routed abort replies and override cases can remain wrong.

## What This Changes

The PR pre-seeds reply-prefix template context from configured default model/provider/thinking values and adds unit tests for seeded defaults plus runtime override behavior.

## Best Possible Solution

Populate abort-prefix context from the same active session/channel model selection used by normal replies and carry that context through routed delivery, with focused regression coverage.

Yes. Source inspection shows current main can format fast-abort replies before model context is populated, and the inspected WhatsApp screenshots show raw placeholders before the patch plus a resolved direct abort reply after it.

## Solution Assessment

No. Default seeding is a useful direct-path mitigation, but the best fix should derive abort prefixes from active session/channel model selection and pass that context through routed delivery.
