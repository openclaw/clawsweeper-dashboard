Keep this PR open: the linked abort-prefix bug is still present on current main and in v2026.6.9, but this branch is not merge-ready because it seeds only config defaults and does not carry prefix context through routed abort delivery.

## What This Changes

The branch pre-seeds reply-prefix template context from configured default model, provider, and thinking level and adds unit coverage for seeded defaults plus runtime override behavior.

## Best Possible Solution

Populate abort-prefix context from the same active session/channel model selection used by normal replies and carry that context through routed delivery, with focused regression coverage.

## Solution Assessment

No. Default seeding is a useful mitigation for simple direct replies, but the best fix should derive abort prefixes from active session/channel model selection and pass that context through routed delivery.
