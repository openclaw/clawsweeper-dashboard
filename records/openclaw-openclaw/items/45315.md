## Summary

Keep open. The PR fixes the dispatcher-based placeholder leak and includes real WhatsApp screenshot proof, but it misses routed abort replies because `routeReply` still normalizes a templated `responsePrefix` without any `responsePrefixContext`.

## What This Changes

The branch pre-seeds `createReplyPrefixContext` with config-default provider/model/thinking values and adds unit coverage for default seeding and `onModelSelected` overrides.

## Best Possible Solution

Extend the fix so every early abort delivery path, including routed replies, normalizes with a populated response-prefix context, preferably from session/runtime metadata with config defaults as fallback, plus focused regression tests.

## Reproduction Assessment

Yes, at source level. Current main can hit fast abort before `onModelSelected` fills model/thinking context, and routed abort normalization still applies a templated prefix without context.

## Solution Assessment

No. Pre-seeding `createReplyPrefixContext` is useful for dispatcher paths, but the routed abort path bypasses that context; the safer fix is to carry or construct prefix context where routed replies normalize.
