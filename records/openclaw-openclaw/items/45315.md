## Summary

Keep this PR open: it targets a real source-reproducible abort-prefix bug and has usable screenshot proof, but the current patch is not merge-ready because it seeds from config defaults instead of the active model state and the branch is currently unmergeable.

## What This Changes

The PR seeds `createReplyPrefixContext` with default model/provider/thinking values and adds tests for seeded defaults, runtime overrides, and empty config.

## Best Possible Solution

Rebase on current `main` and seed abort reply prefixes from canonical active model metadata, falling back to config defaults only when no session, turn, channel, runtime, or catalog facts exist.

## Reproduction Assessment

Yes. Current main leaves `{model}` and `{thinkingLevel}` unset until `onModelSelected`, while fast abort formats and sends the abort reply before that callback can run.

## Solution Assessment

No, not as-is. Pre-seeding from config defaults fixes the simplest placeholder leak but bypasses active session, channel, and catalog-backed model selection, so the better fix should carry those selected model facts into the abort prefix path.
