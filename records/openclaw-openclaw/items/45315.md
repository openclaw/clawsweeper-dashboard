## Summary

Keep this PR open: it targets a real documented behavior bug and has useful real-world proof, but the branch is incomplete because routed abort delivery still normalizes templated prefixes without a response-prefix context.

## What This Changes

The PR pre-seeds `ResponsePrefixContext` from config defaults and adds unit coverage for default seeding, runtime override, and fallback behavior.

## Best Possible Solution

Carry a populated, session/runtime-aware response-prefix context into both direct dispatcher sends and routed `routeReply()` delivery, with regressions for direct and routed fast aborts.

## Reproduction Assessment

Yes, at source level. Configure `messages.responsePrefix` with `{model}` or `{thinkingLevel}`, trigger stop before `onModelSelected()` runs, and the fast-abort route can normalize a templated prefix without interpolation context.

## Solution Assessment

No. Config-default pre-seeding is useful, but the maintainable fix also needs to pass populated prefix context into `routeReply()` and prefer runtime/session metadata where available.
