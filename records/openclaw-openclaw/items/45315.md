## Summary

Keep this PR open, but it still needs changes before merge: the patch improves default prefix context seeding, yet routed abort replies still normalize with a templated responsePrefix and no responsePrefixContext, so the claimed all-channel fix is incomplete.

## What This Changes

The PR seeds createReplyPrefixContext with config-default model/provider/thinking values and adds unit coverage for default seeding, runtime overwrite, and empty-config fallback.

## Best Possible Solution

Build a session/runtime-aware ResponsePrefixContext once at the dispatch or routing boundary and use it for both direct dispatcher and routed abort normalization, with direct and routed stop-reply regression tests.

## Reproduction Assessment

Yes, source-level: current main can format a fast-abort payload before onModelSelected and normalize a templated responsePrefix without model/thinking context, matching the issue steps and screenshots. I did not run a live channel test in this read-only review.

## Solution Assessment

No. Config-default pre-seeding improves one fallback path, but the patch misses routeReply normalization and does not carry override-aware runtime metadata through the delivery boundary.
