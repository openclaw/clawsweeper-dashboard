## Summary

Keep open. The reported bug is valid and the PR has credible WhatsApp before/after proof, but the patch only seeds the dispatcher prefix context and still leaves routed abort replies able to normalize a templated responsePrefix without model/thinking context.

## What This Changes

The PR pre-seeds `createReplyPrefixContext()` with configured default model/provider/thinking values and adds unit coverage for default seeding, runtime override, and fallback defaults.

## Best Possible Solution

Carry a populated, session/runtime-aware `ResponsePrefixContext` through both direct dispatcher delivery and routed abort delivery, with regression tests for templated fast-abort prefixes in both paths.

## Reproduction Assessment

Yes, at source level. Current main can enter fast abort before `onModelSelected()`, and missing responsePrefix context values remain literal during normalization.

## Solution Assessment

No. Default seeding is a useful partial fallback, but the narrow maintainable fix should also pass or build prefix context for `routeReply()` so routed abort replies cannot leak placeholders.

## Review Findings
