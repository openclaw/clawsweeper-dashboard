## Summary

Keep open: the reported bug is real and the screenshots show the intended improvement, but the branch should not merge as-is because it seeds abort prefixes from config defaults instead of the active model-selection state and is dirty against current main.

## What This Changes

The PR initializes reply-prefix model/provider/thinkingLevel from configured defaults in `createReplyPrefixContext` and adds unit coverage for default seeding and runtime override.

## Best Possible Solution

Resolve abort response-prefix templates from the same selected-model state normal replies use, including stored session/channel overrides and catalog-backed thinking defaults, while keeping the prefix formatter itself unchanged.

## Reproduction Assessment

Yes. Source inspection shows the fast-abort path sends before `onModelSelected`, and the inspected WhatsApp screenshots show the placeholder symptom and the central after-fix behavior; I did not run a live repro in this read-only review.

## Solution Assessment

No. The PR is a plausible mitigation, but the best fix is to seed abort prefixes from active model-selection state rather than config defaults that can diverge from session and channel overrides.

Full review comments:

- **[P1] Seed abort prefixes from the active model:** `src/channels/reply-prefix.ts:44-56`
  - body: This seeds `{model}` from `resolveDefaultModelForAgent`, but fast abort returns before `onModelSelected` while normal model selection honors stored `/model` overrides and channel model overrides. Users with an active override would get a confident but wrong model in the abort reply; compute or pass the active selection state before filling the prefix context.
  - confidence: 0.9
- **[P2] Use catalog-backed thinking defaults:** `src/channels/reply-prefix.ts:45-49`
  - body: `resolveThinkingDefault` is called without the catalog path used by normal selection, so reasoning-capable models without an explicit config default can show `off` in abort prefixes while normal replies resolve a catalog-backed thinking level. Pass the same catalog-backed default used by model selection before setting `thinkingLevel`.
  - confidence: 0.85
