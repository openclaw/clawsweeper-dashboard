## Summary

Keep open. The bug is still present on current main, and this PR is a useful but incomplete fix because routed abort delivery still normalizes templated prefixes without any response-prefix context.

## What This Changes

The PR pre-seeds response-prefix context from config defaults and adds unit coverage so abort replies can resolve `{model}` and `{thinkingLevel}` placeholders.

## Best Possible Solution

Build a session/runtime-aware `ResponsePrefixContext` and pass it through both direct dispatcher delivery and `routeReply()`, with direct and routed fast-abort regressions.

## Reproduction Assessment

Yes, at source level. Configure `messages.responsePrefix` with `{model}` or `{thinkingLevel}`, trigger `stop` before `onModelSelected()` runs, and current routed fast-abort delivery can normalize a templated prefix without interpolation context.

## Solution Assessment

No. Config-default pre-seeding improves one path, but the maintainable fix must also carry populated prefix context into routed delivery and prefer runtime/session metadata where available.
