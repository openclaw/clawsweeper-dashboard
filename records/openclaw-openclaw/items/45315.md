## Summary

Keep open: current main still has the fast-abort interpolation bug, but this PR should not merge as-is because it can replace raw placeholders with incorrect model/thinking values for override and catalog-backed cases.

## What This Changes

The PR pre-seeds reply prefix context from config defaults and adds tests for default seeding, runtime override, and fallback behavior.

## Best Possible Solution

Land one canonical fix that formats early abort prefixes from effective persisted/runtime model metadata and catalog-backed thinking defaults, with focused regression coverage and a changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure messages.responsePrefix with {model} or {thinkingLevel}, start a run, then send stop or /stop; the fast-abort branch replies before onModelSelected fills the context.

## Solution Assessment

No. Pre-seeding from config defaults is narrower than the real effective-model contract, so the maintainable fix should reuse persisted/runtime model and catalog-backed thinking metadata instead of guessing early.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Seed abort prefixes from the effective runtime model:** `src/channels/reply-prefix.ts:42`
  - body: This pre-seeds fast-abort prefix values from config defaults, but normal runs can use stored session overrides and channel model overrides. Because fast abort returns before onModelSelected runs, users who changed models can receive a resolved prefix with the wrong model; seed or format from effective runtime/persisted model metadata instead.
  - confidence: 0.88
- **[P2] Use catalog-backed thinking defaults before seeding:** `src/channels/reply-prefix.ts:43-47`
  - body: resolveThinkingDefault is called without a catalog here, so reasoning-capable models with no explicit thinkingDefault can seed off while the normal selection path hydrates catalog metadata and may resolve a non-off default. Use the same catalog-backed or persisted runtime thinking value before formatting abort prefixes.
  - confidence: 0.84
