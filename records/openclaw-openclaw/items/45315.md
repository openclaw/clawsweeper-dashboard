- **[P1] Carry prefix context through routed abort replies:** `src/channels/reply-prefix.ts:48-51`
  - body: This seeds the dispatcher context, but fast-abort replies first try `routeReplyToOriginating()`. `routeReply()` resolves `responsePrefix` and calls `normalizeReplyPayload()` without a context, so routed aborts still send `{model}` / `{thinkingLevel}` literally.   - confidence: 0.87

  - body: The seed comes from `resolveDefaultModelForAgent()`, while normal replies can use session `/model`, `channels.modelByChannel`, auto fallback, and runtime catalog thinking defaults before `onModelSelected()` fires. Fast abort skips that callback, so this can replace raw placeholders with the wrong model or thinking level; seed from the same active context used by model selection.
  - confidence: 0.83
