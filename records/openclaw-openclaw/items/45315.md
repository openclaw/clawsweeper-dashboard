## Summary

Keep open. The PR targets a valid source-reproducible abort-reply bug, but the current diff only seeds dispatcher-created prefix contexts; routed abort replies can still normalize templated prefixes without any responsePrefixContext.

## What This Changes

Pre-seeds reply-prefix context from config default model/provider/thinking values and adds helper tests for default seeding and onModelSelected overrides.

## Best Possible Solution

Carry a populated runtime/session-aware ResponsePrefixContext into both routed fast-abort delivery and the direct dispatcher fallback, with regression coverage for templated prefixes on both paths.

## Reproduction Assessment

Yes. Configure messages.responsePrefix with {model} or {thinkingLevel}, trigger stop before onModelSelected runs, and current main can normalize a routed abort reply without model/thinking context.

## Solution Assessment

No. Config-default seeding improves the dispatcher fallback, but the maintainable fix should pass runtime/session-aware prefix context into routed delivery as well.
