## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Pass prefix context through routed abort replies:** `src/channels/reply-prefix.ts:44-49`
  - body: This seeds the dispatcher prefix context, but fast abort first calls `routeReplyToOriginating()`. That routed path calls `routeReply()`, which normalizes with `responsePrefix` but no `responsePrefixContext`, so cross-surface abort replies can still send `{model}` or `{thinkingLevel}` literally. Please pass a resolved context into routed abort delivery and add a routed-abort regression test.
  - confidence: 0.87
- **[P2] Seed abort prefixes from active model state:** `src/channels/reply-prefix.ts:44-56`
  - body: `resolveDefaultModelForAgent()` only gives the config default. Normal replies use session overrides, channel `modelByChannel`, fallback-selected state, and catalog-aware thinking defaults before calling `onModelSelected`, so abort replies after `/model` or channel overrides can show the wrong model/thinking label. The fix should derive from active session/channel context when available instead of baking in only defaults.
  - confidence: 0.82
