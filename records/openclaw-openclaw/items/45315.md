## Summary

Keep open. Current main still has the source-reproducible abort-prefix gap, and this PR only seeds the dispatcher prefix context; routed abort delivery still normalizes the configured template without any `responsePrefixContext`, so raw `{model}` or `{thinkingLevel}` can still leak.

## What This Changes

The branch pre-seeds `createReplyPrefixContext()` with config-default provider/model/thinking values and adds tests for default seeding plus `onModelSelected()` overrides.

## Best Possible Solution

Carry a populated runtime/session-aware `ResponsePrefixContext` into both routed fast-abort delivery and the direct dispatcher fallback, with regression coverage for templated prefixes on both paths.

## Reproduction Assessment

Yes, at source level. Configure `messages.responsePrefix` with `{model}` or `{thinkingLevel}`, trigger `stop` or `/stop` before `onModelSelected()` runs, and current main can normalize an abort reply without model/thinking context.

## Solution Assessment

No. Config-default pre-seeding improves only the dispatcher fallback; the maintainable fix should pass runtime/session-aware prefix context into routed delivery as well.
