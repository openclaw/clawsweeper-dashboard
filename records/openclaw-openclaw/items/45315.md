## Summary

Keep open: this is a useful, proof-backed mitigation for raw abort-prefix placeholders, but it is not merge-ready because the patch seeds config-default model context, misses active session/channel model state, misses routed abort delivery, and the live PR is currently conflicting.

## What This Changes

The PR pre-seeds `createReplyPrefixContext()` with default provider/model/thinking values and adds unit coverage for the seeded context and later `onModelSelected()` overrides.

## Best Possible Solution

Populate abort-prefix context from the same active session/channel model selection used by normal replies and carry that context through routed delivery, with focused regression coverage.

## Reproduction Assessment

Yes, source-level: current main can format a fast-abort reply before model context is populated, and the PR screenshots show the raw-placeholder failure and a resolved direct WhatsApp abort reply after the patch. I did not run a live WhatsApp or Telegram flow in this read-only review.

## Solution Assessment

No. Default seeding is a useful mitigation for simple direct replies, but the best fix should derive abort prefixes from active session/channel model selection and pass that context through routed delivery.
