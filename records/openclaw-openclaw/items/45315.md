## Summary

This PR fixes a real user-visible abort-prefix bug, but the patch is incomplete because it only seeds the dispatcher-owned prefix context. Routed abort replies still normalize without `responsePrefixContext`, so templated placeholders can still leak on routed delivery.

## What This Changes

The PR pre-seeds `createReplyPrefixContext()` with config-default provider/model/thinking metadata and adds unit coverage for default seeding, runtime override, and empty-config fallback.

## Best Possible Solution

Carry a populated prefix context through both direct dispatcher delivery and routed abort normalization, using available session/runtime model metadata and falling back to config defaults only when needed.

## Reproduction Assessment

Yes, at source level. Configure `messages.responsePrefix` with `{model}` or `{thinkingLevel}`, trigger `stop` before `onModelSelected()`, and the direct or routed abort normalization paths can run without model/thinking context.

## Solution Assessment

No. Config-default pre-seeding is a useful fallback, but this patch misses routed abort normalization and does not carry override-aware metadata through the delivery path.

## Review Findings
