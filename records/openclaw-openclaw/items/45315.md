Overall correctness: patch is incorrect

Full review comments:

- **[P2] Resolve abort prefixes from active model context:** `src/channels/reply-prefix.ts:44-56`
  - body: This seed uses resolveDefaultModelForAgent, so fast-abort replies can show the config-default model/thinking when the current session has a /model override or the channel has channels.modelByChannel. Normal reply selection honors those active overrides before onModelSelected, but abort returns before that callback; derive this prefix from the same active session/channel selection instead.
  - confidence: 0.86
- **[P2] Pass prefix context into routed abort delivery:** `src/channels/reply-prefix.ts:51-56`
  - body: This context only reaches dispatcher normalization. When dispatch-from-config sends an abort through routeReplyToOriginating, routeReply calls normalizeReplyPayload with responsePrefix but no responsePrefixContext, so routed abort replies can still leak raw {model} or {thinkingLevel} placeholders. Pass the populated abort context through routed delivery too.
  - confidence: 0.82
