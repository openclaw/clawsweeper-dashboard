## Summary

Keep open: the PR addresses a real user-facing abort-prefix bug and includes useful screenshot proof, but the branch is currently merge-conflicting and the patch can still report stale model/thinking values for existing session, channel, and catalog-derived model-selection paths.

## What This Changes

The branch pre-seeds reply-prefix context with config default model/provider/thinkingLevel values and adds tests for seeded defaults, runtime override, and empty-config fallback.

## Best Possible Solution

Rebase the PR and seed early-abort prefix context from the same active session/channel/runtime model facts, including catalog-backed thinking defaults, that normal replies use.

## Reproduction Assessment

Yes, at source level and with attached visible proof. Configure responsePrefix with {model} or {thinkingLevel}, trigger stop before onModelSelected runs, and current main can route an abort reply without populated model/thinking prefix context.

## Solution Assessment

Not yet. Pre-seeding config defaults fixes the basic placeholder leak, but the maintainable fix should carry the active session/channel/runtime model and catalog-backed thinking facts into the abort path.

Full review comments:

- **[P2] Seed abort prefixes from the active model context:** `src/channels/reply-prefix.ts:44`
  - body: This seeds the abort prefix from resolveDefaultModelForAgent, but fast abort returns before normal selection while normal replies honor session model overrides and channels.modelByChannel. Users who switched models or use channel overrides can now see a plausible but wrong model/thinking prefix; thread the active runtime/session/channel model context into the abort path before resolving the prefix.
  - confidence: 0.87

  - body: resolveThinkingDefault is called without catalog data here, while the normal selection path loads catalog data before choosing a default thinking level. Reasoning-capable models without explicit thinkingDefault can therefore show off in abort prefixes even when normal replies would use the catalog-derived default.
  - confidence: 0.82
