## Summary

Keep open. The PR targets a real documented behavior bug and has useful proof, but the patch is incomplete because routed abort replies still normalize a templated responsePrefix without any responsePrefixContext.

## What This Changes

The PR pre-seeds reply prefix context with config-default model, provider, and thinking values and adds unit coverage for default seeding and runtime override.

## Best Possible Solution

Build a session/runtime-aware ResponsePrefixContext and pass it through both dispatcher fallback delivery and routeReply normalization, with direct and routed fast-abort regression coverage.

## Reproduction Assessment

Yes, at source level. Configure messages.responsePrefix with {model} or {thinkingLevel}, trigger stop before onModelSelected, and current main can normalize an abort reply with a templated prefix and missing context.

## Solution Assessment

No. Config-default pre-seeding improves the direct dispatcher fallback path, but it misses routed abort delivery and should be replaced or extended with context passed into routeReply.
