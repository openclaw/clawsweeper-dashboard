## Summary

Current `main` still has the reported fast-abort placeholder path, but this PR's default-only seed can report the wrong model or thinking level when normal reply selection would use session overrides, channel overrides, or catalog-derived thinking defaults.

## What This Changes

The PR pre-seeds reply-prefix context from configured default model/provider/thinking values and adds unit coverage for default seeding, runtime override, and fallback defaults.

## Best Possible Solution

Carry or persist the effective runtime model metadata into the fast-abort prefix path, then keep `onModelSelected()` as the normal reply override and cover default, session override, channel override, and catalog-derived thinking cases.

## Reproduction Assessment

Yes. Source inspection shows fast abort can deliver before `onModelSelected()` runs, while current `main` only seeds `identityName` in the prefix context.

## Solution Assessment

No. Default seeding is better than leaking raw template tokens, but it is not the best fix because normal replies already compute a richer effective model context than this branch uses.
