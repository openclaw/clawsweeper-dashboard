Fix PR https://github.com/openclaw/openclaw/pull/45315 for https://github.com/openclaw/openclaw/issues/45314. The bug is that early stop/abort replies with `messages.responsePrefix` templates can expose raw `{model}` or `{thinkingLevel}` before `onModelSelected()` runs. Keep the fix bounded to reply-prefix context propagation: carry a populated runtime/session `ResponsePrefixContext` into both dispatcher fallback and routed `routeReply()` fast-abort delivery, or normalize the abort payload with that context before routing. Preserve normal `onModelSelected()` override behavior, avoid broad model-selection rewrites, do not add config options, and do not change abort semantics. Add regression coverage for routed stop-command abort replies with `{model}`/`{thinkingLevel}` plus existing direct/default override cases. Resolve the current main conflict. Changelog is maintainer-owned at landing time.
