Overall correctness: patch is incorrect

Full review comments:

- **[P1] Carry prefix context into routed abort replies:** `src/channels/reply-prefix.ts:53-56`
  - body: This seeds `prefixContext`, but the fast-abort path first calls `routeReplyToOriginating()` and `routeReply()` normalizes the payload with `responsePrefix` but no `responsePrefixContext`. Routed-origin abort replies can still send literal `{model}` or `{thinkingLevel}` after this patch.
  - confidence: 0.87
- **[P2] Seed abort prefixes from active model context:** `src/channels/reply-prefix.ts:44-49`
  - body: These values come from config defaults before any turn context is considered. Normal reply selection can use session `/model`, `channels.modelByChannel`, auto fallback, and catalog-derived thinking defaults before `onModelSelected()` fires, but fast abort skips that callback, so abort prefixes can show the wrong model or thinking state.
  - confidence: 0.84
