## Summary

Keep open: the reported bug is real and the screenshots show the useful direction, but this patch is not merge-ready because it seeds abort prefixes from default-only state instead of the effective session/channel/catalog-backed model state, and GitHub currently reports the branch as conflicting with main.

## What This Changes

The PR pre-seeds reply-prefix template context with configured default model/provider/thinking values and adds tests for default seeding plus later `onModelSelected` overrides.

## Best Possible Solution

Carry the effective model, provider, and thinking facts used by normal model selection into fast-abort prefix rendering, with a deliberate fallback only when active facts are unavailable.

## Reproduction Assessment

Yes. Current main sends fast-abort replies before `onModelSelected` populates dynamic prefix context, and the supplied before screenshot shows literal placeholders in a real WhatsApp abort reply.

## Solution Assessment

No. The patch is a useful mitigation, but the best fix is to pass effective active model-selection facts into the fast-abort and routed reply paths rather than seeding from config defaults only.

- **[P1] Seed abort prefixes from active model state:** `src/channels/reply-prefix.ts:44-56`
  - body: The new pre-seed uses `resolveDefaultModelForAgent`, but fast abort can happen after a session `/model`, parent-session, or channel model override is active.
