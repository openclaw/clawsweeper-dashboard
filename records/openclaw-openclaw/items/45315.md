- **[P2] Resolve abort prefixes from active model context:** `src/channels/reply-prefix.ts:44-49`
  - body: The new seed uses `resolveDefaultModelForAgent`, so fast-abort replies can show the config-default model/thinking when the current session has a `/model` override or the channel has `channels.modelByChannel`. Normal reply selection honors those active overrides before `onModelSelected`, but abort returns before that callback; derive this prefix from the same active session/channel selection instead.

Pass the populated abort context through routed delivery too.
