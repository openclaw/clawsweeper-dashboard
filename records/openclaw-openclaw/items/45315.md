## Best Possible Solution

Rebase and carry canonical active model metadata into early abort and routed reply prefix normalization, using config defaults only as fallback and adding regression coverage for session overrides, channel overrides, and catalog-backed thinking defaults.

## Reproduction Assessment

## Solution Assessment

No, not as-is. Pre-seeding from config defaults improves the simplest placeholder leak, but the maintainable fix should use active session/channel/runtime model state and cover routed reply normalization.
