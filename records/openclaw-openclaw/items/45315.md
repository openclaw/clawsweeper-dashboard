## Summary

Keep open: the bug is valid and this PR improves the direct prefix-context path, but the patch is incomplete because routed abort replies still normalize templated response prefixes without a responsePrefixContext.

## What This Changes

The PR pre-seeds createReplyPrefixContext with default model/provider/thinking values and adds unit coverage for default seeding and onModelSelected overrides.

## Best Possible Solution

Build a session/runtime-aware ResponsePrefixContext and pass it through both direct dispatcher delivery and routeReply, with direct and routed fast-abort regressions.

## Reproduction Assessment

Yes, at source level. Configure messages.responsePrefix with {model} or {thinkingLevel}, trigger stop before onModelSelected runs, and current main can normalize the abort reply with missing model/thinking context.

## Solution Assessment

No. Config-default pre-seeding helps one path, but the maintainable fix also needs to carry populated prefix context into routed delivery and prefer runtime/session metadata where available.
