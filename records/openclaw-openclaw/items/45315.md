## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Pass prefix context into routed abort replies:** `src/channels/reply-prefix.ts:44`
  - body: This seed only reaches dispatcher-owned normalization. Current main's fast-abort path can call `routeReplyToOriginating()`, and `routeReply()` resolves `responsePrefix` without any `responsePrefixContext`, so routed abort replies can still send raw `{model}` / `{thinkingLevel}` placeholders. Please carry the abort prefix context into routed delivery or normalize the routed payload with context before sending.
  - confidence: 0.9
- **[P2] Seed abort prefixes from the active model context:** `src/channels/reply-prefix.ts:44-49`
  - body: `resolveDefaultModelForAgent({ cfg, agentId })` ignores session `providerOverride`/`modelOverride`, channel model overrides, and auto-fallback runtime state, while fast abort skips the later `onModelSelected()` correction. This can replace raw placeholders with the wrong model/thinking values for users who changed models or route by channel; the seed needs to come from the active session/channel selection path instead of only config defaults.
  - confidence: 0.87
- **[P2] Resolve thinking defaults with catalog data:** `src/channels/reply-prefix.ts:45-49`
  - body: The new call to `resolveThinkingDefault()` omits runtime catalog data, while the normal model-selection path can load catalog entries before resolving the default thinking level. For reasoning-capable defaults without an explicit configured `thinkingDefault`, abort prefixes can show `off` even though normal replies would resolve a different default; pass catalog-aware thinking context or avoid rendering this token until it is accurate.
  - confidence: 0.82
