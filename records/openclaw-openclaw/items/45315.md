## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Pass prefix context into routed abort replies:** `src/channels/reply-prefix.ts:44`
  - body: This seed only helps dispatcher-owned normalization. The fast-abort branch can call `routeReplyToOriginating()`, and `routeReply()` resolves `responsePrefix` without any `responsePrefixContext`, so routed abort replies can still send raw `{model}` or `{thinkingLevel}` placeholders.
  - confidence: 0.9
- **[P2] Seed abort prefixes from the active model context:** `src/channels/reply-prefix.ts:44-49`
  - body: `resolveDefaultModelForAgent({ cfg, agentId })` ignores session `/model` overrides, channel model overrides, and runtime fallback selection, while fast abort skips the later `onModelSelected()` correction. That can replace raw placeholders with the wrong model/thinking values for users whose active model differs from the config default.
  - confidence: 0.87
- **[P2] Resolve thinking defaults with catalog data:** `src/channels/reply-prefix.ts:45-49`
  - body: The new `resolveThinkingDefault()` call omits catalog data, while the normal model-selection path loads catalog entries before resolving default thinking. Reasoning-capable defaults without an explicit configured `thinkingDefault` can therefore show `off` in abort prefixes even when normal replies would resolve a different default.
  - confidence: 0.82
