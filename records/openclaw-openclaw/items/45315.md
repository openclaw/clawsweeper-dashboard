## Summary

Keep this PR open. Current main still has the abort-prefix interpolation bug, and this PR is a relevant implementation candidate for the linked open bug, but the proposed default-only seeding bypasses effective model metadata that normal replies already use.

## What This Changes

This PR pre-seeds `createReplyPrefixContext()` with config-default provider, model, and thinking level values, and adds tests for default seeding, runtime override, and fallback defaults.

## Best Possible Solution

Land one canonical fix that populates response-prefix context from the effective session/runtime model metadata before abort reply normalization, with focused regression coverage for templated fast-abort replies and explicit handling or deferral of channel overrides, session overrides, routed replies, and catalog-backed thinking defaults.

## Reproduction Assessment

Yes. Configure `messages.responsePrefix` with `{model}` or `{thinkingLevel}`, start a run, then send `/stop`; current main’s fast-abort path returns before `onModelSelected()` fills the prefix context, so the template resolver can leave those placeholders literal.

## Solution Assessment

No. The PR is a useful narrow improvement for the config-default case, but it is not the best fix because it bypasses effective model selection inputs that the normal reply path already honors; #45454 appears closer to the runtime-metadata direction, though it is still draft.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Seed abort prefixes from the effective session model:** `src/channels/reply-prefix.ts:42`
  - body: This seeds the fast-abort prefix from `resolveDefaultModelForAgent()`, but normal runs can select a different model from stored session overrides or `channels.modelByChannel`. Because fast abort returns before `onModelSelected()` runs, users who changed models can now receive a resolved prefix with the wrong model/thinking values. Thread the effective session/runtime model metadata into this path instead of using only config defaults.
  - confidence: 0.87
- **[P2] Use catalog-backed thinking defaults before seeding:** `src/channels/reply-prefix.ts:43-47`
  - body: `resolveThinkingDefault()` is called without a catalog here, so reasoning-capable models that rely on catalog metadata fall back to `off` even though the normal selection path hydrates a catalog and resolves a reasoning default. Pass the same catalog-backed default, or use persisted runtime thinking metadata, before pre-seeding abort prefixes.
  - confidence: 0.82
