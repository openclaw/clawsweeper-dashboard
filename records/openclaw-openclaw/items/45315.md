## Summary

Keep open. The PR improves one direct abort path, but routed abort replies still normalize templated response prefixes without any `ResponsePrefixContext`, so `{model}` and `{thinkingLevel}` can still leak literally.

## What This Changes

The PR seeds `createReplyPrefixContext()` with default model/provider/thinking values and adds unit coverage for seeded defaults, runtime override, and empty-config fallback.

## Best Possible Solution

Carry a populated reply-prefix context through both direct dispatcher delivery and routed abort normalization, using session/runtime metadata where practical, with direct and routed regression tests.

## Reproduction Assessment

Yes, at source level. Configure `messages.responsePrefix` with `{model}` or `{thinkingLevel}`, trigger `stop` before `onModelSelected()`, and current main can normalize a routed abort reply with a templated prefix but no prefix context.

## Solution Assessment

No. Pre-seeding config defaults is a useful partial fallback, but the patch is incomplete until routed abort delivery passes or builds the same prefix context before normalization.
