## Summary

Keep open: the bug is real and the screenshots show the intended user-visible improvement, but this branch is merge-conflicting and the implementation seeds abort prefixes from config defaults rather than the active session/channel model state.

## What This Changes

The branch pre-seeds `createReplyPrefixContext` with default model/provider/thinking values and adds unit coverage for seeded prefix context plus `onModelSelected` overrides.

## Best Possible Solution

Rebase on current `main` and seed abort reply prefixes from canonical active model metadata, falling back to config defaults only when no session, turn, channel, or runtime selection exists, with regression tests for the base abort case and override cases.

## Reproduction Assessment

Yes, source-reproducible. Current main leaves model variables unset until `onModelSelected`, while the fast-abort branch sends the abort reply before that callback can run.

## Solution Assessment

No, not as-is. Pre-seeding from config defaults fixes the simple placeholder leak but bypasses active model/session/channel selection, so the maintainable fix should carry the selected model facts into the abort prefix path.
