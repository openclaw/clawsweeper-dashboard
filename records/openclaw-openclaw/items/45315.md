## Summary

Keep open. The bug is still present on current main, and this PR improves only the dispatcher fallback while leaving routed fast-abort delivery without prefix template context.

## What This Changes

The branch pre-seeds `createReplyPrefixContext()` with config-default provider/model/thinking values and adds unit coverage for seeded defaults plus `onModelSelected()` overrides.

## Best Possible Solution

Carry a populated runtime/session-aware `ResponsePrefixContext` into both routed fast-abort delivery and dispatcher fallback, then land this PR or a narrow replacement with regression coverage.

## Reproduction Assessment

Yes, at source level. Configure `messages.responsePrefix` with `{model}` or `{thinkingLevel}`, trigger `stop` or `/stop` before `onModelSelected()` runs, and routed abort delivery can normalize the reply without template context.

## Solution Assessment

No. Config-default pre-seeding is an improvement for the dispatcher fallback, but the maintainable fix must also pass populated prefix context through routed delivery.
