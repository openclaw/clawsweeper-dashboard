## Summary

Keep this PR open, but it still needs changes before merge: it fixes the raw placeholder symptom, yet the proposed default-only seeding can show the wrong active model or thinking level for supported override paths.

## What This Changes

The PR pre-seeds reply-prefix template context from configured default model, provider, and thinking values for early abort replies.

## Best Possible Solution

Carry the effective model, provider, and thinking facts already selected for the active message/session into the abort reply path, using a deliberate fallback only when active facts do not exist.

## Reproduction Assessment

## Solution Assessment

No. Config defaults remove raw tokens, but the best fix is to pass effective active model-selection facts into the abort reply path so supported overrides and catalog-backed thinking defaults stay consistent.

Full review comments:

- **[P1] Seed abort prefixes from active model state:** `src/channels/reply-prefix.ts:44-56`
  - body: The new pre-seed uses `resolveDefaultModelForAgent`, but fast abort can happen after a session `/model` override or channel model override. Normal replies honor those overrides before the run; with this patch, abort replies replace raw placeholders with stale config-default model/thinking values. Pass the effective selected facts into the prefix context and cover those override cases.
  - confidence: 0.97
- **[P2] Use catalog-backed thinking defaults:** `src/channels/reply-prefix.ts:45-49`
  - body: `resolveThinkingDefault` is called without a catalog here, while normal model selection resolves thinking defaults through configured, manifest, or runtime catalog data. Reasoning-capable models without explicit config can therefore show `off` in abort prefixes instead of the normal catalog-derived level.
  - confidence: 0.95
