## Summary

The bug remains relevant on current main, but this implementation knowingly replaces unresolved placeholders with metadata that can contradict the effective active session state. The branch should stay open for a bounded repair rather than merge with that correctness gap.

## What This Changes

The PR pre-seeds response-prefix model, provider, and thinking values from configured defaults so early abort replies interpolate template variables.

## Best Possible Solution

Carry the effective model, provider, and thinking facts already selected for the active message or session into the abort reply path, using a narrow explicit fallback only when active facts do not exist.

## Reproduction Assessment

## Solution Assessment

No. Config defaults eliminate raw tokens but can replace them with incorrect values for supported session overrides, channel overrides, and catalog-derived thinking defaults.
