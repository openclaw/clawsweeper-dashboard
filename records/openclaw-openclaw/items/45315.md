## Best Possible Solution

Rebase or replace this branch with a fix that derives abort-prefix context from active session/channel model state, carries that context through routed and queued abort delivery, and adds regressions for direct, routed, and override cases.

## Solution Assessment

No. The PR is a useful mitigation for direct default-config cases, but the best fix must use active model context and pass prefix context through routed abort delivery instead of only pre-seeding config defaults.
