## Summary

Close as implemented: current main and v2026.5.7 already ship restart-safe Discord component/modal registry persistence through the SDK-backed plugin state store, while this PR's home-cache JSON approach is superseded.

## What This Changes

The PR adds best-effort disk persistence for Discord component and modal registry entries under a home-cache JSON file with validation and registry tests.

## Best Possible Solution

Keep the shipped openKeyedStore-backed implementation from https://github.com/openclaw/openclaw/pull/75584 and avoid merging this superseded JSON-file branch.

## Reproduction Assessment

Yes, at source level: register a Discord component, restart so the process-global Maps are empty, then click the still-visible Discord component; the pre-fix path would miss the registry and reply expired. I did not run a live Discord gateway smoke in this read-only review.

## Solution Assessment

No for this PR branch: it stores JSON under a home-cache path and ignores the active OpenClaw state system. The shipped solution is narrower and more maintainable because it uses the runtime-bound plugin keyed store under the state directory.
