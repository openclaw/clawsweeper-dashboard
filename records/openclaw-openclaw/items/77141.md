## Summary

Keep this PR open. The branch fixes the successful fresh-retry path, but the persisted stale CLI binding can still survive when the fresh retry fails, so the earlier blocking review finding remains.

## What This Changes

The branch widens `runPreparedCliAgent` reused-session `FailoverError` recovery to all reasons, logs a redacted session fingerprint, adds timeout/unknown regression tests, and adds a changelog entry for the linked CLI-session issue.

## Best Possible Solution

Land a corrected narrow fix that clears the stored CLI session binding for any `FailoverError` from a reused session, retries once fresh, persists a replacement binding on success, and preserves existing fallback behavior when the retry fails.

## Reproduction Assessment

Yes. Source inspection of current main shows watchdog failures become `FailoverError` reason `timeout`, while both the runner retry and the caller-side store clear are still limited to `session_expired`.

## Solution Assessment

No. Widening the retry is the right direction, but the clear must happen in the store-aware caller path or through an explicit callback before retry/fallback can leave the stale binding behind.

## Review Findings
