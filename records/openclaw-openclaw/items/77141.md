## Summary

Keep open: the PR targets a real current-main CLI session recovery bug, but the patch still has an incomplete stale-binding clear path when the fresh retry fails, and the branch currently reports as dirty against main.

## What This Changes

The PR widens CLI-session failover recovery for resumed sessions, adds timeout/unknown regression tests, and adds a changelog entry for #77089.

## Best Possible Solution

Land a corrected narrow fix that clears the stored CLI session binding for any FailoverError from a reused session, retries once fresh, persists a replacement binding on success, and preserves fallback behavior on retry failure.

## Reproduction Assessment

Yes. From source inspection, current main converts CLI watchdog failures into FailoverError reason "timeout" and only retries or clears the reused session path for "session_expired".

## Solution Assessment

No, not as-is. Widening the fresh retry is the right direction, but the stored binding should be cleared before a fresh retry can fail and fall through, or the widened recovery should live in attempt-execution where the store is available.
