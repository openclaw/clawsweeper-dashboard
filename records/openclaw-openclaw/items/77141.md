## Summary

Keep this PR open: current main still has the stale CLI-session recovery bug, and the PR targets the right area, but the proposed patch needs small corrections before merge.

## What This Changes

The PR widens CLI-session recovery so reused-session `FailoverError` failures retry once with a fresh session, adds timeout/unknown regression tests, and adds a changelog entry.

## Best Possible Solution

Land a corrected narrow fix that retries a fresh CLI session for stale reused-session FailoverErrors, preserves the replacement binding on success, masks session identifiers in logs, and keeps the changelog reference accurate.

## Reproduction Assessment

Yes, from source inspection: current main converts watchdog failures to `FailoverError(reason: "timeout")` and then only retries fresh for `session_expired`. I did not execute the PR tests in this read-only sweep.
