## Summary

Keep open: current main still has the Signal daemon hot-reload race, and the open closing PR is still unmerged.

## What This Changes

The issue asks OpenClaw to make Signal daemon shutdown await process exit during hot reload/restart so old signal-cli processes cannot race new daemon startup.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/71863 or an equivalent Signal-owned lifecycle fix that awaits daemon exit, bounds SIGKILL escalation, and covers daemon/monitor shutdown with regression tests.

## Reproduction Assessment

Yes, source-level. Enable Signal auto-start and trigger a `channels.signal` hot reload; current main stops then starts the channel while Signal daemon cleanup only sends SIGTERM and returns.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Signal daemon lifecycle and monitor teardown; broader child-process cleanup in iMessage or Gmail can be handled separately.
