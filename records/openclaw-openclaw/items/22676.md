## Summary

Keep open: current main and the latest release still use fire-and-forget Signal daemon shutdown, and https://github.com/openclaw/openclaw/pull/71863 is still the open implementation candidate that explicitly closes this issue.

## What This Changes

The issue requests that Signal daemon shutdown during hot restart wait for signal-cli exit, escalate if needed, and avoid orphaned processes or send failures.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/71863 or an equivalent focused Signal plugin fix, then close this issue after merge with the fixing commit and release provenance.

## Reproduction Assessment

Yes, source-level. Enable Signal auto-start and trigger a channels.signal hot reload; current main stops then starts the channel while Signal daemon cleanup only sends SIGTERM and returns before the child process exits.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Signal daemon lifecycle and monitor teardown; broader iMessage or Gmail child-process cleanup should stay separate unless independently reproduced.
