## Summary

Keep open: current main and the latest release still have the reported synchronous Signal daemon shutdown path, and the direct closing PR is open but unmerged, so this issue is still the canonical tracker.

## What This Changes

The issue reports that Signal daemon hot-reload restarts can orphan signal-cli because stop sends SIGTERM without awaiting process exit before the channel starts again.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/71863, or an equivalent focused Signal fix that makes daemon stop async/idempotent, waits for exit after SIGTERM, escalates to SIGKILL after a bounded timeout, and awaits shutdown through monitor teardown before restart.
