## Summary

Keep open: current main and the latest release still use the synchronous Signal daemon shutdown path, and the focused closing fix at https://github.com/openclaw/openclaw/pull/71863 is open but unmerged.

## What This Changes

The issue reports a Signal daemon shutdown race during hot reload/restart where OpenClaw sends SIGTERM to signal-cli but restarts before the old process exits.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/71863 or an equivalent focused Signal lifecycle fix with regression coverage, then close this issue with fixing commit and release provenance.

## Reproduction Assessment

Yes, at source level. With Signal autoStart enabled, a channels.signal hot reload stops the channel and immediately restarts it while Signal cleanup only sends SIGTERM and returns before signal-cli exits.

## Solution Assessment

Yes. Making Signal daemon stop async and idempotent, waiting for process exit with bounded SIGKILL escalation, and awaiting monitor teardown is the narrow maintainable fix.
