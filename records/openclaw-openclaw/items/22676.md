## Summary

Keep open. Current main and the latest release still have the Signal daemon stop race, and https://github.com/openclaw/openclaw/pull/71863 is an open closing PR rather than a merged fix.

## What This Changes

The issue requests fixing Signal daemon restart shutdown so gateway hot reload waits for signal-cli to exit before starting a replacement daemon.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/71863, or land an equivalent Signal-owned async/idempotent daemon stop that waits for exit and is awaited through monitor teardown.

## Reproduction Assessment

Yes by source inspection: current main still returns from Signal daemon stop immediately after SIGTERM, and monitor teardown does not await daemon process exit. I did not run a live linked Signal restart.

## Solution Assessment

Yes in direction. A Signal-owned async, idempotent daemon stop awaited through monitor teardown is the narrow maintainable fix; the open linked PR is the active implementation candidate.
