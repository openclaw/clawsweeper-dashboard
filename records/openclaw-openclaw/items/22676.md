Keep open: current main still has the Signal daemon hot-reload race, and the active closing PR is open and unmerged.

## What This Changes

The issue asks for Signal auto-start restarts to await `signal-cli` shutdown so SIGUSR1/config hot reloads do not leave orphaned daemons or cause send failures.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/71863 or an equivalent Signal-owned lifecycle fix that awaits daemon exit, bounds SIGKILL escalation, and covers daemon/monitor shutdown with regression tests.

## Reproduction Assessment

Yes, source-level. Enable Signal auto-start and trigger a `channels.signal` hot reload; current main stops then starts the channel while Signal daemon cleanup only sends SIGTERM and returns.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Signal daemon lifecycle and monitor teardown; broader child-process cleanup in other integrations can be handled separately.
