## Summary

Keep open. Current main still has the reported Signal daemon restart race: Signal shutdown sends SIGTERM without awaiting process exit, monitor teardown does not keep the channel task pending for daemon cleanup, and PR #71863 remains the active open fix candidate for this issue.

## What This Changes

Make Signal daemon shutdown asynchronous and awaited during config/SIGUSR1 restarts so the old signal-cli process exits, or is killed after a bounded timeout, before a replacement daemon starts.

## Best Possible Solution

Keep this issue open and review or land PR #71863, or land an equivalent focused Signal-owned fix that makes daemon stop idempotent and async, waits for signal-cli exit after SIGTERM, escalates to SIGKILL after a bounded timeout, and awaits that shutdown through monitor teardown before channel restart.
