## Summary

Keep open: current main still has the Signal daemon hot-reload race, and the active closing PR is open and unmerged.

## What This Changes

The issue requests making Signal daemon shutdown await signal-cli process exit during hot reload so restarts do not race the old daemon.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/71863 or an equivalent Signal-owned fix that makes daemon stop async/idempotent, waits for exit after SIGTERM, bounds SIGKILL escalation, and covers daemon/monitor shutdown with regression tests.

## Reproduction Assessment

Yes, source-level. Enable Signal auto-start and trigger a `channels.signal` hot reload; current main stops then starts the channel while Signal daemon cleanup only sends SIGTERM and returns before child exit.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Signal daemon lifecycle and monitor teardown, and the active PR targets that surface with focused regression coverage.
