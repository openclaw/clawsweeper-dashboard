## Summary

Keep open: current main still has the Signal daemon hot-reload race, and https://github.com/openclaw/openclaw/pull/71863 is an open, unmerged closing PR for the same Signal lifecycle fix.

## What This Changes

The issue asks OpenClaw to make Signal daemon shutdown during hot reload asynchronous and bounded so restarts do not leave old signal-cli processes behind.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/71863 or an equivalent Signal-owned lifecycle fix that awaits daemon exit, adds bounded SIGKILL escalation, and covers daemon/monitor shutdown with regression tests.

## Reproduction Assessment

Yes, source-level. Enable Signal auto-start and trigger a `channels.signal` hot reload; current main stops then starts the channel while Signal daemon cleanup only sends SIGTERM and returns.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Signal daemon lifecycle and monitor teardown; broader iMessage or Gmail child-process cleanup can be handled separately.
