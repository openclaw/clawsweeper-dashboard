## Summary

Keep open: current main still has the Signal daemon hot-reload shutdown race, and the open closing PR is still unmerged.

## What This Changes

Make Signal daemon hot-reload shutdown wait for `signal-cli` exit, with bounded escalation, so restarts do not leave orphaned daemons or stale send paths.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/71863 or an equivalent focused Signal-owned fix that awaits daemon exit, bounds SIGKILL escalation, and adds daemon/monitor regression coverage.

## Reproduction Assessment

Yes, source-level. Enable Signal auto-start and trigger a `channels.signal` hot reload; current main stops then starts the channel while Signal daemon cleanup only sends SIGTERM and returns.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Signal daemon lifecycle and monitor teardown; broader child-process cleanup for iMessage or Gmail should be handled separately if still needed.
