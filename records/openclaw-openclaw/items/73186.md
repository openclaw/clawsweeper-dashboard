## Summary

Keep open: current main still drops or ignores the reasoning marker in the shared isolated cron announce path, and open PR #73363 already uses closing syntax while #40534 overlaps the broader fix.

## What This Changes

Prevent Matrix and Feishu isolated cron announce delivery from exposing reasoning/thinking payloads or duplicating delivered content.

## Best Possible Solution

Preserve isReasoning through embedded-runner payloads, exclude reasoning from cron summary/output/deliverable and error-recovery selection, and add focused Matrix/Feishu announce regression coverage.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: reasoning text is marked internally, the returned payload drops that marker, and cron announce selectors choose deliverable text using only content and isError checks.

## Solution Assessment

Yes. The maintainable fix belongs in the shared embedded payload and cron outcome path; Matrix- or Feishu-only filtering would leave other direct cron announce consumers exposed.
