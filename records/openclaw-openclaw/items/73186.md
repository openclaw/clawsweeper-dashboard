## Summary

Keep open: current main still drops or ignores `isReasoning` before the shared cron announce selectors can reliably suppress reasoning text, and a direct open fix PR already closes this issue.

## What This Changes

The issue reports reasoning content and duplicate-looking delivery leaking into Matrix and Feishu cron announce messages.

## Best Possible Solution

Land one canonical current-main fix that preserves `isReasoning` through embedded payload construction and treats reasoning as non-user-visible across cron summary, output, delivery payloads, synthesized fallback, and post-error recovery, with Matrix/Feishu regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: reasoning text is marked internally, the returned payload drops the marker, and cron announce selectors choose payload text without checking `isReasoning`.

## Solution Assessment

Yes. The maintainable fix belongs in the shared embedded-payload and cron outcome path; Matrix- or Feishu-only filtering would leave other cron announce selectors exposed.
