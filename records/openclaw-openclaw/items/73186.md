## Summary

Keep open: current main still leaves the shared cron announce path able to select reasoning text, and open PR #73363 already uses closing syntax for this issue while #40534 overlaps the broader end-to-end fix.

## What This Changes

Cron announce delivery for Matrix and Feishu should stop exposing reasoning/thinking payloads and avoid duplicate delivery when thinking is enabled.

## Best Possible Solution

Preserve `isReasoning` through the embedded runner result contract and make the shared cron outcome/delivery selectors exclude reasoning from summary, output, deliverable payloads, and error-recovery decisions, with Matrix/Feishu announce regression coverage.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists on current main: reasoning text is marked internally, the returned payload drops that marker, and cron announce selectors choose text using content and `isError` checks only.

## Solution Assessment

Yes. The maintainable repair belongs in the shared embedded-payload and cron outcome path; Matrix- or Feishu-only filtering would leave other cron announce selectors exposed.
