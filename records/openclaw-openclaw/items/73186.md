## Summary

Keep open: current main still has a source-reproducible cron announce reasoning leak, and open PR #73363 directly uses closing syntax for this issue while #40534 overlaps the broader end-to-end fix.

## What This Changes

Prevent Matrix and Feishu cron announce delivery from leaking thinking/reasoning content and duplicating delivered text.

## Best Possible Solution

Preserve `isReasoning` through embedded payloads and exclude reasoning from shared cron summary, output, deliverable, and error-recovery selection, with Matrix/Feishu announce regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows reasoning is created with `isReasoning`, dropped by the returned payload map, and then eligible for cron announce selection because the selectors skip only errors.

## Solution Assessment

Yes. The maintainable repair belongs in the shared embedded-payload and cron outcome path; Matrix- or Feishu-only filtering would leave other cron announce selectors exposed.
