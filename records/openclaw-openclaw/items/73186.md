## Summary

Keep open. Current main still has the shared isolated-cron announce path that can select reasoning text, and open PR #73363 is already paired to this issue with closing syntax, so the issue should stay open until a canonical fix merges.

## What This Changes

The issue requests preventing thinking/reasoning payloads from leaking into Matrix and Feishu isolated cron announce delivery, including duplicate user-facing delivery cases.

## Best Possible Solution

Land one canonical fix that preserves isReasoning through embedded-runner payloads, includes that flag in the cron payload contract, excludes reasoning from summary/output/deliverable and error-recovery selectors, and adds focused Matrix/Feishu announce regression coverage before closing through the merged PR.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: reasoning text is marked internally, the returned payload drops that marker, and cron announce selectors choose text using only isError/content checks.

## Solution Assessment

Yes. The narrow maintainable solution belongs in the shared embedded-payload and cron outcome path; Matrix- or Feishu-only filtering would leave other cron selectors and direct announce paths exposed.
