## Summary

Keep open: current main still allows reasoning text to enter the shared cron announce outcome path, and open PR #73363 uses closing syntax for this issue while #40534 overlaps the broader fix.

## What This Changes

The issue requests that Matrix and Feishu cron announce delivery suppress reasoning payloads and avoid duplicate delivery when thinking is enabled.

## Best Possible Solution

Preserve isReasoning through the embedded runner payload contract and make the shared cron outcome path exclude reasoning from summary, output, deliverable payloads, and recovery decisions, with Matrix/Feishu announce regression coverage.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: reasoning text is marked internally, the returned payload drops that marker, and cron selectors can choose the resulting text for delivery.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the shared embedded-payload and cron outcome selectors; Matrix- or Feishu-only filtering would leave other cron announce consumers exposed.
