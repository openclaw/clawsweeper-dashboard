## Summary

Keep open: current main still has the shared cron announce leak path, and open PR #73363 is already linked with closing syntax, so this issue should wait for that canonical fix or an explicit replacement to merge.

## What This Changes

The issue reports Matrix and Feishu isolated cron announce delivery leaking model reasoning text, with duplicate delivery in some cases, because embedded-runner payloads drop or ignore the `isReasoning` marker before cron summary and delivery selection.

## Best Possible Solution

Land one canonical shared fix that preserves `isReasoning` through embedded-runner payloads, includes it in the cron payload contract, excludes reasoning from summary/output/deliverable and error-recovery selectors, and adds focused Matrix/Feishu cron announce regression coverage before this issue closes.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: reasoning text is marked internally, the returned payload drops that marker, and cron announce selectors choose text using only content and `isError` checks.

## Solution Assessment

Yes. The right fix boundary is shared embedded-payload and cron outcome handling; Matrix- or Feishu-only filtering would leave other cron selectors and direct announce paths exposed.
