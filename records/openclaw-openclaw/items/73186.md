## Summary

Keep open: current main still has the shared cron announce reasoning leak path, and open PR #73363 already uses closing syntax for this issue while overlapping PR #40534 remains unmerged.

## What This Changes

The issue asks OpenClaw to stop leaking thinking/reasoning content and duplicate cron announce messages for Matrix and Feishu deliveries.

## Best Possible Solution

Land one canonical shared fix that preserves `isReasoning` through the embedded-runner result contract, excludes reasoning payloads from cron summary/output/deliverable and error-recovery selectors, and adds Matrix/Feishu announce regression coverage.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current `main`: reasoning text is marked internally, the marker is dropped from returned payloads, and cron announce selectors choose text and deliverable payloads without checking reasoning state.

## Solution Assessment

Yes for the shared-fix direction. Matrix- or Feishu-only filtering would leave the common cron announce path exposed, and a reasoning-only fallback should be avoided unless maintainers explicitly accept that product behavior.
