Keep open: current main still has the primary shared cron announce path that can leak reasoning text, and open PR #73363 already references this issue with closing syntax while PR #40534 overlaps the broader fix. Closing before one canonical fix lands would be premature.

## What This Changes

The issue requests preventing thinking/reasoning payloads from leaking into Matrix and Feishu isolated cron announce delivery, including related duplicate delivery behavior.

## Best Possible Solution

Land one canonical shared fix that preserves isReasoning through the embedded-runner result contract, excludes reasoning from cron summary/output/deliverable and error-recovery selectors, and adds focused Matrix/Feishu announce regression coverage before this issue closes.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: reasoning text is marked internally, the returned payload drops that marker, and cron announce selectors choose text using only content and isError checks.

## Solution Assessment

Yes. The maintainable solution belongs in the shared embedded-payload and cron outcome path; Matrix- or Feishu-only filtering would leave other cron selectors and direct announce paths exposed.
