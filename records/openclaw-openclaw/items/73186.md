## Summary

Keep open: current main still drops or ignores the reasoning marker in the shared cron announce path, and open PR #73363 is already paired to this issue while #40534 overlaps the broader fix.

## What This Changes

Cron isolated announce delivery should avoid exposing model thinking/reasoning content or causing duplicate Matrix/Feishu deliveries.

## Best Possible Solution

Land one canonical shared fix that preserves isReasoning through the embedded-runner result contract, excludes reasoning from cron summary/output/deliverable and error-recovery selectors, and adds focused Matrix/Feishu announce regression coverage.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: reasoning text is marked internally, the returned payload drops that marker, and cron announce selectors choose text using only content and isError checks.

## Solution Assessment

Yes. The maintainable fix belongs in the shared embedded-payload and cron outcome path; Matrix- or Feishu-only filtering would leave other cron announce selectors exposed.
