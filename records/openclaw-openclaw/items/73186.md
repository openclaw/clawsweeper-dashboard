## Summary

Keep open: current main still has the shared cron announce leak path, and an open PR already uses closing syntax for this issue while another open PR overlaps the broader fix.

## What This Changes

Cron announce delivery for Matrix and Feishu should avoid exposing model reasoning text and duplicate content when thinking is enabled.

## Best Possible Solution

Land one canonical shared fix that preserves isReasoning through the embedded-runner result contract, excludes reasoning from cron summary/output/deliverable and error-recovery selectors, and adds focused Matrix/Feishu announce regression coverage.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: reasoning text is marked internally, the returned payload drops that marker, and cron announce selectors choose payload text using only content and isError checks.

## Solution Assessment

Yes. The best fix is in the shared embedded-payload and cron outcome path; Matrix- or Feishu-only filtering would leave other cron announce selectors exposed.
