## Summary

Keep open: current main still has a source-reproducible reasoning leak path, and an open PR explicitly closes this issue. The issue should stay open until one canonical fix lands and proves Matrix/Feishu cron announce behavior.

## What This Changes

The issue reports Matrix/Feishu cron announce delivery exposing model reasoning content and sometimes duplicate delivery when thinking output is present.

## Best Possible Solution

Land one canonical current-main fix that carries `isReasoning` through embedded payloads and treats reasoning as non-user-visible across cron summary, output, delivery, fallback, and post-error recovery, with Matrix/Feishu regression coverage and redacted real cron proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: reasoning blocks become internal payload items, the returned payload drops `isReasoning`, and cron announce selectors choose text using content and `isError` checks only.

## Solution Assessment

Yes. The shared embedded-payload and cron outcome helpers are the right fix boundary; channel-only Matrix or Feishu filtering would leave other cron announce selectors, fallback delivery, and recovery checks exposed.
