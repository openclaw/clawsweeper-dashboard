## Summary

Keep open: current main still has a static cron announce path where reasoning payload text can be selected for delivery, and open PR #73363 uses closing syntax for this issue while #40534 overlaps the broader shared fix.

## What This Changes

Cron announce delivery should prevent Matrix and Feishu recipients from receiving model reasoning/thinking payloads or duplicated content.

## Best Possible Solution

Land one canonical shared fix that preserves isReasoning through EmbeddedPiRunResult payloads, excludes reasoning from cron summary/output/deliverable and error-recovery selectors, and adds focused Matrix/Feishu cron announce regression coverage.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: reasoning text is marked in the embedded runner, dropped from the returned payload, and cron announce selectors choose text and deliverable payloads without checking isReasoning.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the shared embedded-payload and cron outcome path; Matrix- or Feishu-only filtering would leave other announce selectors exposed.
