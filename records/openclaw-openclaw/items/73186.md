## Summary

Keep open: current main still has a source-reproducible cron announce reasoning leak path because reasoning payloads are created, then the cron delivery selectors choose text/delivery payloads using `isError` and content checks without suppressing `isReasoning`. The linked open PRs target the same bug, including an open closing PR at https://github.com/openclaw/openclaw/pull/73363, so this issue should remain open until one canonical fix lands.

## What This Changes

Report that Matrix and Feishu isolated cron announce delivery can expose thinking/reasoning content and sometimes duplicate output when reasoning models are used.

## Best Possible Solution

Land one canonical fix that preserves `isReasoning` through embedded payload construction and treats reasoning as non-user-visible across cron summary, output, delivery, synthesized fallback, and post-error recovery paths, with Matrix/Feishu regression coverage and redacted real cron proof.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: reasoning blocks become internal payload items, the returned payload drops `isReasoning`, and cron announce selectors choose text/delivery payloads using content and `isError` checks only. I did not run a live Matrix or Feishu cron job in this read-only review.

## Solution Assessment

Yes, the shared embedded-payload and cron outcome helpers are the right fix boundary. Channel-only Matrix or Feishu filtering would leave other cron announce selectors, fallback paths, and recovery checks exposed.
