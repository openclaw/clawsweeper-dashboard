## Summary

Keep open. Current main still drops or ignores reasoning markers in the shared cron announce path, and open PR #73363 uses closing syntax while #40534 overlaps the broader end-to-end fix. The issue should close only after a canonical PR lands.

## What This Changes

The issue asks OpenClaw to prevent Matrix and Feishu isolated cron announce delivery from exposing model thinking/reasoning text and duplicate visible content.

## Best Possible Solution

Land one canonical fix that preserves `isReasoning` through embedded-runner payloads and excludes reasoning payloads from shared cron outcome and delivery selectors, with Matrix and Feishu announce coverage plus focused regression tests.

## Reproduction Assessment

Yes for the main leak path. A static current-main reproduction exists: reasoning text is created as a reasoning reply item, the returned payload drops that marker, and cron selectors can pick that text for summary/output/direct delivery.

## Solution Assessment

Yes, the shared payload-propagation and cron-selector boundary is the right fix. A Matrix- or Feishu-only filter would leave other isolated cron announce selectors exposed.
