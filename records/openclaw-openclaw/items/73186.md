## Summary

Keep open: current main still leaves the shared cron announce path able to select reasoning text, and open PR #73363 uses closing syntax for this issue while #40534 overlaps the broader end-to-end filtering work.

## What This Changes

The issue asks OpenClaw to prevent thinking/reasoning payloads from being selected and delivered by Matrix/Feishu cron announce runs, including duplicate delivery cases.

## Best Possible Solution

Preserve isReasoning through the embedded runner result contract and make cron outcome selectors exclude reasoning from summary, output, delivery, and error-recovery decisions, with Matrix/Feishu announce regression coverage.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: reasoning text is marked internally, the returned payload drops that marker, and cron announce selectors choose text using only content and isError checks.

## Solution Assessment

Yes. The maintainable fix belongs in the shared embedded-payload and cron outcome path; Matrix- or Feishu-only filtering would leave other cron announce selectors exposed.
