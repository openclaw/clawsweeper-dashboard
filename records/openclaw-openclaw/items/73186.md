## Summary

Keep open: current main still lets reasoning payloads enter the shared isolated-cron announce delivery path, and open PR #73363 already uses closing syntax for this issue while #40534 overlaps the broader fix.

## What This Changes

Report that Matrix and Feishu isolated cron announce delivery can expose model thinking/reasoning content and sometimes duplicate delivered content.

## Best Possible Solution

Land one canonical shared fix that preserves isReasoning through the embedded-runner result contract, excludes reasoning from cron summary/output/deliverable/error-recovery selectors, and adds focused Matrix/Feishu announce regression coverage.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: reasoning is marked internally, the returned payload drops that marker, and the cron announce selectors then treat the reasoning text as normal deliverable content.

## Solution Assessment

Yes. The maintainable fix belongs in the shared embedded-payload and cron outcome path; Matrix- or Feishu-only filtering would leave other cron announce consumers exposed.
