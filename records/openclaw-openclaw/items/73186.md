## Summary

Keep open: current main still exposes the shared cron announce path to reasoning payload text, and open PR #73363 already uses closing syntax for this issue while #40534 overlaps the broader end-to-end fix.

## What This Changes

Issue requests preventing Matrix/Feishu cron announce delivery from exposing model thinking/reasoning content and avoiding related duplicate delivery.

## Best Possible Solution

Preserve isReasoning through the embedded runner result contract and make cron outcome selectors exclude reasoning from summary, output, delivery, and error-recovery decisions, then land #73363 or the broader #40534 with Matrix/Feishu regression coverage.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: reasoning text is marked internally, the returned payload drops that marker, and cron announce selectors choose text using only content and isError checks.

## Solution Assessment

Yes. The maintainable fix belongs in the shared embedded-payload and cron outcome path; Matrix- or Feishu-only filtering would leave other cron announce selectors exposed.
