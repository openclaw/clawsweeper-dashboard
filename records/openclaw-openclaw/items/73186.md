## Summary

Keep open: current main still leaves the Matrix/Feishu cron announce path able to select reasoning text, and PR #73363 already uses closing syntax for this issue while PR #40534 overlaps the broader end-to-end fix.

## What This Changes

Cron announce delivery should keep reasoning/thinking blocks out of Matrix and Feishu messages and avoid related duplicate delivery.

## Best Possible Solution

Preserve isReasoning through EmbeddedPiRunResult and make the shared cron summary, output, deliverable, and error-recovery selectors ignore reasoning, then land one focused linked PR with Matrix/Feishu regression coverage.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists on current main: buildEmbeddedRunPayloads() creates reasoning payloads, drops isReasoning before returning, and cron selectors choose non-error text without any reasoning filter.

## Solution Assessment

Yes. The maintainable repair belongs in the shared embedded-payload and cron outcome path; Matrix- or Feishu-only filtering would leave other cron announce selectors exposed.
