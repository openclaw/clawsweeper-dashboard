## Summary

Keep open: current main still leaves the shared cron announce path able to select reasoning text, and open PR #73363 already targets this issue while #40534 overlaps the fuller end-to-end repair.

## What This Changes

Cron announce delivery should exclude thinking/reasoning content from Matrix and Feishu messages and avoid duplicate delivery when reasoning is enabled.

## Best Possible Solution

Keep the issue open until the shared payload/helper fix lands: preserve isReasoning through EmbeddedPiRunResult and prevent reasoning payloads from being selected for summaries, deliverable payloads, and recovery decisions, with Matrix/Feishu cron announce regression coverage.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists on current main: reasoning text is marked internally, the returned payload drops that marker, and cron announce selectors choose text while checking only content and isError.

## Solution Assessment

Yes. The best repair is in the shared embedded payload contract and cron outcome helpers; Matrix- or Feishu-only filtering would leave other cron announce selectors exposed.
