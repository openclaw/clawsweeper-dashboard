## Summary

Keep open: current main still has the shared cron announce path that can expose reasoning payload text, and open PR #73363 already references this issue with closing syntax while PR #40534 overlaps the broader end-to-end fix. Closing or spawning another repair lane before one canonical fix lands would be premature.

## What This Changes

Cron announce delivery for Matrix and Feishu can include model thinking/reasoning text, with occasional duplicate delivery, when isolated cron jobs run with reasoning enabled.

## Best Possible Solution

Land one canonical shared fix that preserves `isReasoning` through the embedded-runner result contract, excludes reasoning from cron summary/output/deliverable and error-recovery selectors, and adds focused Matrix/Feishu cron announce regression coverage before this issue closes.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: reasoning text is marked internally, the returned payload drops that marker, and cron announce selectors choose text using only content and `isError` checks.

## Solution Assessment

Yes. The maintainable fix belongs in the shared embedded payload and cron outcome path; Matrix- or Feishu-only filtering would leave other cron selectors and direct announce paths exposed.
