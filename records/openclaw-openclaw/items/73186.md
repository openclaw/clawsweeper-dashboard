## Summary

Keep open: current main still creates reasoning payloads but drops or ignores `isReasoning` before cron announce selection, and the latest release has the same cron helper gap. An open closing PR targets this issue, so the issue should stay open until one canonical fix lands.

## What This Changes

The issue reports reasoning/thinking text and occasional duplicate delivery leaking into Matrix/Feishu cron announce messages when thinking-enabled models run in isolated announce mode.

## Best Possible Solution

Land one current-main fix that preserves `isReasoning` through embedded payload construction and treats reasoning as non-user-visible across cron summary, output, delivery payloads, synthesized fallback, and post-error recovery, with Matrix/Feishu regression and redacted real cron proof.

## Reproduction Assessment

Yes, at source level. `buildEmbeddedRunPayloads()` marks reasoning internally, returns payloads without the marker, and cron selectors pick or send text using only `isError` and content checks.

## Solution Assessment

Yes. The shared embedded-payload and cron outcome helpers are the narrowest maintainable boundary; channel-only Matrix or Feishu filtering would leave cron selectors, fallback text, and recovery checks exposed.
