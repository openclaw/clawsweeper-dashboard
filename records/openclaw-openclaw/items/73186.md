## Summary

Keep open: current main still drops the reasoning marker before cron announce selection, and the latest release has the same selector gap. The linked open PRs target this exact bug, so this issue should stay open until one canonical fix lands.

## What This Changes

The issue reports Matrix and Feishu cron announce deliveries exposing model thinking/reasoning content and sometimes duplicating delivered text.

## Best Possible Solution

Land one current-main fix that preserves `isReasoning` through embedded payload construction and excludes reasoning from cron summary, output, delivery, synthesized fallback, and post-error recovery paths, with Matrix/Feishu regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: reasoning blocks become internal payload items, the returned payload drops `isReasoning`, and cron announce selectors choose text using content and `isError` checks only.

## Solution Assessment

Yes. The shared embedded-payload and cron outcome helpers are the right fix boundary; channel-only Matrix or Feishu filtering would leave other cron announce selectors exposed.
