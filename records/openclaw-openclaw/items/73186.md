## Summary

Keep open: current main still has the reasoning-marker propagation and cron selector gaps, and overlapping open PRs already target this bug. This issue should stay open until one canonical fix lands and is verified.

## What This Changes

The issue asks OpenClaw to prevent Matrix and Feishu isolated cron announce delivery from exposing thinking/reasoning payloads or sending duplicate reasoning-derived content.

## Best Possible Solution

Land one canonical fix that carries `isReasoning` through embedded payloads and excludes reasoning from cron summary, output, delivery, synthesized fallback, and post-error recovery, with Matrix/Feishu regression coverage and real cron proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: reasoning payloads are created, the marker is dropped before returning payloads, and cron announce selectors still choose text using content and `isError` checks only.

## Solution Assessment

Yes. The shared embedded-payload and cron outcome helpers are the right fix boundary; channel-only Matrix or Feishu filtering would leave other cron announce selectors exposed.
