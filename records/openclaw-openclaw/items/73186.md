## Summary

Keep open: current main still has the reported shared cron announce leak path, and open PR #73363 is already paired to this issue with closing syntax, so closing before a canonical fix merges would be premature.

## What This Changes

The issue reports that isolated cron announce delivery can expose model thinking/reasoning text, especially for Matrix and Feishu targets, and may duplicate content.

## Best Possible Solution

Land one canonical shared fix that preserves isReasoning through embedded-runner payloads, includes it in the cron payload contract, excludes reasoning from summary/output/deliverable and error-recovery selectors, and adds focused Matrix/Feishu announce regression coverage before this issue closes.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: reasoning text is marked internally, the returned payload drops that marker, and cron announce selectors choose text using only content and isError checks.

## Solution Assessment

Yes. The best fix belongs in the shared embedded-payload and cron outcome path; Matrix- or Feishu-only filtering would leave other cron selectors and direct announce paths exposed.
