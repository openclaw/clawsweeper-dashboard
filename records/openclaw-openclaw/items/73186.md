## Summary

Keep open: current main still creates reasoning payloads but drops or ignores the `isReasoning` marker before the shared cron announce selectors and Matrix/Feishu delivery paths can reliably suppress it. The linked open PRs already target the same bug, so this issue should stay open until one canonical fix lands.

## What This Changes

The issue asks OpenClaw to stop Matrix/Feishu cron announce delivery from exposing reasoning/thinking payloads and duplicate content when thinking-enabled DeepSeek output is present.

## Best Possible Solution

Preserve `isReasoning` through embedded payload construction and make the shared cron outcome selectors exclude reasoning from summaries, output, delivery payloads, synthesized fallback, and error-recovery decisions, with Matrix/Feishu regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: reasoning text is marked internally, then the returned payload drops the marker and cron announce selectors choose payload text without checking `isReasoning`.

## Solution Assessment

Yes. The maintainable fix belongs in the shared embedded-payload and cron outcome path; Matrix- or Feishu-only filtering would leave other cron announce selectors exposed.
