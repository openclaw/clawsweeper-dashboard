## Summary

Keep open: current main still has partial channel-specific and CLI pieces, but it does not provide the requested cross-channel agent-visible final-reply delivery-failure contract. Teams injects a local system event and `agent --json --deliver` reports `deliveryStatus`, while Feishu, Discord, and Telegram still do not consistently feed final-reply delivery failure back into the agent context.

## What This Changes

The issue requests an agent-visible notification or delivery-status contract when outbound channel message delivery fails.

## Best Possible Solution

Define a documented cross-channel delivery-failure contract in the reply-dispatch or message-lifecycle surface, then implement it consistently in owning channel paths while preserving existing CLI `deliveryStatus` and hook observability.

## Reproduction Assessment

Not applicable as a strict bug reproduction because current docs and tests do not promise automatic agent-visible final-reply delivery-failure injection. Source inspection does show the current feature gap across shared dispatch, Feishu, Discord, and Telegram, with Teams and CLI covering narrower paths only.

## Solution Assessment

Unclear. System-event injection is plausible and already proven for Teams, but maintainers need to choose the durable shared API and failure semantics before this becomes a narrow implementation task.
