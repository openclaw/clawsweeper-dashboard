## Summary

Keep open. Current main has useful delivery-status foundations, Telegram/automation observation hooks, and a Teams-only system-event fix, but it still does not provide the requested shared cross-channel agent-visible delivery-failure contract for ordinary final replies and message-tool sends.

## What This Changes

The issue requests an agent-visible notification or status contract when channel message delivery fails so agents can recover from missed sends.

## Best Possible Solution

Define an additive shared delivery-failure contract for final replies and message-tool sends, build it on the existing durable delivery status primitives, and add cross-channel tests before mapping individual channel failures into it.

## Reproduction Assessment

Yes, for the source-level feature gap: current main still lacks a shared agent-visible delivery-failure signal across Feishu, Telegram, and generic message-tool/default final-reply sends. I did not run a live failed channel send.

## Solution Assessment

Unclear as a concrete implementation. A shared additive delivery-failure contract is the maintainable direction, but maintainers still need to choose event shape, tool-result semantics, retry behavior, and compatibility rules.
