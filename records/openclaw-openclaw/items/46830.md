## Summary

Keep open: current main has useful pieces, but not the requested cross-channel agent-visible final-reply delivery failure contract. Teams queues a delivery-failure system event and CLI `--deliver` reports `deliveryStatus`, while Feishu and Discord final-reply failures still only log through their channel error paths and Telegram exposes failure through observation hooks rather than automatic agent context injection.

## What This Changes

The issue requests an agent-visible notification or delivery status contract when final message delivery to channels such as Feishu, Telegram, or Discord fails.

## Best Possible Solution

Define a documented cross-channel delivery-failure contract in the message lifecycle/reply-dispatch surface, then implement the chosen default consistently in the owning channel paths while preserving plugin hook observation semantics.

## Reproduction Assessment

Not applicable as a strict bug reproduction: current docs do not promise automatic agent-visible delivery-failure injection for final replies. Source inspection does show the feature gap across current main: Teams injects a system event, but Feishu and Discord log errors and Telegram exposes hook events only.

## Solution Assessment

Unclear. System-event injection is plausible and already proven in Teams, but maintainers still need to choose whether the durable contract belongs in the shared dispatcher, per-channel implementations, or an optional plugin policy layer.
