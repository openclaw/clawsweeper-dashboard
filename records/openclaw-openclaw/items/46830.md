## Summary

Keep open: current main has a Teams-specific delivery-failure system event and Telegram delivery-failure hooks, but there is no consistent cross-channel agent-visible failure notification for the Feishu/Telegram/Discord-style final-reply paths described here.

## What This Changes

The issue requests agent-visible notification when outbound channel message delivery fails so the agent can retry, recover, or explain the missing reply.

## Best Possible Solution

Define and document the cross-channel delivery-failure contract, then implement the chosen default in the owning channel paths or as a documented plugin using the existing hook and system-event surfaces.

## Reproduction Assessment

Not applicable as a strict bug reproduction: no current documented contract requires automatic agent-visible delivery-failure injection. Source inspection does show the gap the feature is about: the shared dispatcher calls `onError`, while only some channel paths turn delivery failure into system-event or hook signals.

## Solution Assessment

Unclear. System-message injection is plausible and Teams already uses that shape, but maintainers still need to choose whether this belongs in the shared dispatcher, each channel plugin, or an optional ClawHub/plugin policy layer.
