## Summary

Keep open. Current main and the latest release still lack the requested shared, cross-channel, agent-visible delivery-failure contract; Microsoft Teams has a narrow channel-specific system-event implementation, and CLI delivery status exists for a different surface, but Feishu, shared final replies, and generic message-tool sends do not expose a normalized failure notification to the agent.

## What This Changes

The issue requests a shared agent-visible notification or status contract when channel message delivery fails.

## Best Possible Solution

Define an additive shared delivery-failure contract for final replies and message-tool sends, then map channel-specific failures into it with compatibility tests and at least one live failed-delivery proof path.

## Reproduction Assessment

Yes for the source-level feature gap: current main and v2026.5.22 still show Feishu and shared reply paths without a shared agent-visible delivery-failure signal. I did not reproduce a live failed channel send.

## Solution Assessment

Unclear as a concrete implementation. A shared additive delivery-failure contract is the maintainable direction, but maintainers need to choose event shape, tool-result semantics, retry behavior, and compatibility rules first.
