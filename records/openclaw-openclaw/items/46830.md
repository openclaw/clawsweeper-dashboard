## Summary

Keep open: current main logs and observes delivery failures, but it does not provide the requested model-visible failure notification across channel final replies. The remaining work needs a maintained core/plugin contract choice rather than cleanup closure.

## What This Changes

Request model-visible delivery failure awareness or structured delivery status when channel API sends fail.

## Best Possible Solution

Define an additive delivery-failure contract that preserves existing logging/hooks while giving the originating session a bounded, model-visible failure signal for recovery.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source inspection does confirm the current behavior: delivery exceptions flow to onError logging/observation rather than a model-visible failure event.

## Solution Assessment

Unclear until maintainers choose the contract. The narrowest maintainable direction is likely an additive shared delivery-failure event or next-turn injection path, not channel-specific ad hoc prompts.
