## Summary

Keep open. Current main and v2026.6.10 have durable delivery outcome primitives, Gateway/CLI delivery status for narrower agent-command delivery, and channel-specific mitigations, but the shared agent-visible cross-channel delivery-failure notification/status contract requested here is still missing.

## What This Changes

The issue asks OpenClaw to notify agents when channel message delivery fails, either by injecting a system event into context or by returning a normalized failed delivery status.

## Best Possible Solution

Define an additive shared delivery-failure/status contract for final replies and message-tool sends, reuse the existing durable outcome primitives, and add SDK/channel conformance coverage instead of another one-channel patch.

## Reproduction Assessment

Yes, at source level: durable failed/partial_failed outcomes exist below the send layer, while Feishu final replies only log failure and MessageSendResult still lacks a normalized failed/partial_failed agent-facing response. I did not perform a live failed-send replay.

## Solution Assessment

Unclear as a concrete implementation because maintainers still need to choose system event versus tool-result status semantics. The best maintainable direction is a shared additive contract on the channel/agent boundary, not one-off plugin-local behavior.
