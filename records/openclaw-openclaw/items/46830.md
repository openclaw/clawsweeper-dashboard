## Summary

Keep open: current main still has useful partial pieces, but not the requested cross-channel agent-visible final-reply delivery-failure contract. Teams queues a channel-specific system event and CLI `--deliver` exposes `deliveryStatus`, while Feishu and Discord final-reply failures still only log through channel error paths and Telegram exposes failure through hooks/logging rather than automatic agent context injection.

## What This Changes

Request a shared agent-visible notification or delivery status when channel message delivery fails.

## Best Possible Solution

Define a documented cross-channel delivery-failure contract in the message lifecycle or reply-dispatch surface, then implement the chosen default consistently in the owning channel paths while preserving existing CLI `deliveryStatus` and plugin hook semantics.

## Reproduction Assessment

Not applicable as a strict bug reproduction because current docs and tests do not promise automatic agent-visible final-reply delivery-failure injection. Source inspection does show the feature gap on current main across Feishu, Discord, and Telegram, with Teams and CLI covering only narrower paths.

## Solution Assessment

Unclear. System-event injection is plausible and already proven in Teams, but maintainers still need to choose whether the durable contract belongs in the shared dispatcher, per-channel implementations, or an optional plugin policy layer.
