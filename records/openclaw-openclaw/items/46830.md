## Summary

Keep open: current main only partially addresses the request. Microsoft Teams now queues a delivery-failure system event, but other final-reply paths such as Feishu, Telegram, and Discord still primarily log delivery errors, so the broad cross-channel agent-notification capability remains unresolved.

## What This Changes

The issue requests notifying agents when outbound channel message delivery fails so they can retry, recover, or inform users.

## Best Possible Solution

Define an additive shared delivery-failure notification contract for final replies, then wire channels through it while preserving plugin-owned retry and replay policies.

## Reproduction Assessment

Partly. Source inspection gives a clear current-main path showing several channel final-reply failures log errors without agent-visible system events, but this is a feature request rather than a documented bug reproduction.

## Solution Assessment

Unclear. The requested system-event approach matches the existing Teams partial implementation, but making it cross-channel should be designed as a shared reply/outbound contract rather than copied into each plugin ad hoc.
