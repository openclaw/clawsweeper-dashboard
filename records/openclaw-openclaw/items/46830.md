## Summary

Keep open: current main has useful partial delivery-failure observability, but it still lacks the requested shared, agent-visible final-reply delivery-failure contract across channels. Teams queues a local system event and CLI `--json --deliver` exposes `deliveryStatus`, while Feishu, Discord, and Telegram still use channel-specific logging/fallback behavior instead of one documented cross-channel status or context injection surface.

## What This Changes

The issue requests a cross-channel agent-visible notification or delivery-status contract when message delivery fails.

## Best Possible Solution

Define a documented cross-channel delivery-failure contract in shared reply dispatch or channel-turn delivery, then wire channel-owned adapters through it while preserving existing Teams system events and CLI `deliveryStatus`.

## Reproduction Assessment

Not applicable as a strict bug reproduction because current docs and tests do not promise automatic agent-visible final-reply delivery-failure injection. Source inspection does show the current feature gap across shared dispatch and Feishu/Discord/Telegram paths, with Teams and CLI covering narrower paths only.

## Solution Assessment

Unclear. System-event injection and delivery-status responses are both plausible, but maintainers need to choose the durable shared API, failure semantics, and channel fallback interaction before this becomes a narrow repair.
