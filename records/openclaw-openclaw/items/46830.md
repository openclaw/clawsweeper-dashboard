## Summary

Keep open: current main still lacks the requested cross-channel, agent-visible final-reply delivery-failure contract. Teams and CLI delivery paths cover narrower cases, while Feishu, Discord, and Telegram still rely on logging, internal hooks, or channel-specific fallback behavior rather than a shared notification/status surface; the open cron PR at https://github.com/openclaw/openclaw/pull/80139 is related reliability work but not a replacement for this issue.

## What This Changes

Request a cross-channel notification or status mechanism so agents can detect failed final message delivery to channels such as Feishu, Telegram, and Discord.

## Best Possible Solution

Define a documented shared delivery-failure contract in the reply-dispatch or channel-turn delivery surface, then implement it consistently in owner channel paths while preserving existing CLI deliveryStatus and hook observability.

## Reproduction Assessment

Not applicable as a strict bug reproduction because current docs and tests do not promise automatic agent-visible final-reply delivery-failure injection. Source inspection does show the current feature gap across Feishu, Discord, and Telegram compared with the narrower Teams and CLI coverage.

## Solution Assessment

Unclear. System-event injection is plausible and already proven for Teams, but maintainers should choose the durable shared API and failure semantics before this becomes a narrow implementation task.
