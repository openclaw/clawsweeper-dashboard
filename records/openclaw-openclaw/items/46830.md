## Summary

Keep open. Current main and v2026.6.6 have useful delivery-status foundations and a Teams-only system-event fix, but they still do not provide the requested shared, cross-channel, agent-visible delivery-failure contract for Feishu, Telegram, and ordinary message-tool/default final-reply sends.

## What This Changes

The issue requests a shared mechanism to notify the agent when channel message delivery fails, either through a system event or normalized message-tool delivery status.

## Best Possible Solution

Define an additive shared delivery-failure contract for final replies and message-tool sends, then map channel-specific failures into it with cross-channel tests and at least one live failed-delivery proof path.

## Reproduction Assessment

Yes for the source-level feature gap: current main and v2026.6.6 still lack a shared agent-visible delivery-failure signal across Feishu, Telegram, and generic message-tool/default final-reply sends. I did not run a live failed channel send.

## Solution Assessment

Unclear as a concrete implementation. A shared additive delivery-failure contract is the maintainable direction, but maintainers still need to choose event shape, tool-result semantics, retry behavior, and compatibility rules.
