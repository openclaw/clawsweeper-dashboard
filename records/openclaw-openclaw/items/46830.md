## Summary

Keep open: current main has partial adjacent pieces, including Teams-specific delivery-failure context and delivery observation hooks, but it does not provide a channel-agnostic delivery-failure notification back into agent context or a consistent message-tool response contract.

## What This Changes

Request a cross-channel way for agents to learn when outbound channel message delivery fails, either through injected context or delivery status returned to the sending tool.

## Best Possible Solution

Define a shared delivery-result contract that turns confirmed final-send failures into bounded agent-visible context or explicit message-tool errors, with channel-specific retry and idempotency rules preserved.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection does show the implementation gap: some channel handlers only log delivery errors while Teams has a channel-specific system-event path, so the requested cross-channel behavior is not consistently present.

## Solution Assessment

Unclear until maintainers choose the runtime contract. The issue’s two proposed directions are plausible, but a maintainable fix should avoid duplicating retries or turning observation-only hooks into hidden recovery policy.
