## Summary

Keep open. Current main and the latest release have partial delivery-status handling and a Teams-specific system-event implementation, but they still do not provide the requested shared, cross-channel agent-visible delivery-failure contract for Feishu, Telegram, and generic message-tool sends.

## What This Changes

Request a cross-channel notification or status contract so agents can detect failed channel message delivery and recover.

## Best Possible Solution

Define an additive shared delivery-failure contract for final replies and message-tool sends, then map channel-specific failures into it with compatibility tests for success, partial failure, suppression, and failed delivery.

## Reproduction Assessment

Yes for the source-level feature gap: current main and v2026.5.20 still show Feishu and shared reply paths without a shared agent-visible delivery-failure signal. I did not reproduce a live failed channel send.

## Solution Assessment

Unclear as a concrete implementation. A shared additive delivery-failure contract is preferable to one-off channel injections, but maintainers need to choose event shape, tool-result semantics, retry behavior, and compatibility first.
