## Summary

Keep open: current main has partial delivery-failure visibility, but not the consistent cross-channel agent-visible delivery-failure notification requested for Feishu, Telegram, Discord, and similar final-reply paths.

## What This Changes

Request cross-channel notification when outbound channel delivery fails, preferably by injecting an agent-visible system event or returning delivery status to the sending path.

## Best Possible Solution

Define one documented delivery-failure contract, then make bundled final-reply paths consistently emit it or provide a reliable plugin hook plus system-event policy for agent-visible recovery.

## Reproduction Assessment

Not applicable as a strict bug reproduction: no current documented contract requires automatic agent-visible delivery-failure injection. Source inspection does confirm the feature gap across current main, especially the Feishu final-reply path and the shared dispatcher behavior.

## Solution Assessment

Unclear. System-event injection matches the Teams-specific shape, but maintainers still need to choose the cross-channel default and whether plugin hooks are sufficient for some channels.
