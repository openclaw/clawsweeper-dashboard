## Summary

Keep open: current main still has partial delivery-failure handling, but not the requested cross-channel agent-visible final-reply failure contract. Teams injects a local system event and CLI `--json --deliver` exposes `deliveryStatus`; Feishu, Discord, and Telegram still rely on logging or channel-specific fallback behavior rather than a shared agent-context notification/status surface.

## What This Changes

The issue requests an agent-visible notification or delivery status when channel message delivery fails.

## Best Possible Solution

Define a documented cross-channel delivery-failure contract in the reply-dispatch or channel-turn delivery surface, then implement it consistently in owning channel paths while preserving existing CLI `deliveryStatus` and hook observability.

## Reproduction Assessment

Not applicable as a strict bug reproduction because current docs and tests do not promise automatic agent-visible final-reply delivery-failure injection. Source inspection does show the current feature gap across shared dispatch and Feishu/Discord/Telegram paths, with Teams and CLI covering narrower paths only.

## Solution Assessment

Unclear. System-event injection is plausible and already proven for Teams, but maintainers need to choose the durable shared API and failure semantics before this becomes a narrow implementation task.
