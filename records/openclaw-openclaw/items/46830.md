## Summary

Keep open. Current main and v2026.6.8 have durable delivery-status primitives and some channel-specific signals, but the shared cross-channel agent-visible delivery-failure notification/status contract requested here is still missing.

## What This Changes

The issue asks for channel delivery failures to be reported back to the agent via system context or a delivery-status response.

## Best Possible Solution

Define an additive shared delivery-failure/status contract for final replies and message-tool sends, map channel failures through SDK-owned seams, and reuse existing durable send outcome primitives.

## Reproduction Assessment

Yes, source-level evidence shows current main has failure outcomes below the send layer, but ordinary cross-channel final replies and message-tool sends still lack one shared agent-visible delivery-failure contract. I did not run a live failed Feishu or Telegram send.

## Solution Assessment

Unclear as a concrete implementation because maintainers still need to choose the public status/event shape. The maintainable direction is additive and shared, using durable send outcomes rather than another channel-local patch.
