## What This Changes

The PR makes Feishu bot identity probing return an explicit aborted result, stops aborted startup before applying empty identity state, and adds a regression test for replacement identity preservation.

## Best Possible Solution

Land the Feishu-local fix after redacted real reload proof or an explicit maintainer proof override confirms the aborted old lifecycle no longer clears replacement identity.

## Reproduction Assessment

Yes at source level: current main returns empty identity on abort, applies that state, and skips recovery when the lifecycle signal is aborted. I did not run a live Feishu tenant reload.

## Solution Assessment

Yes, the implementation is the right owner-boundary fix: distinguish aborted probes from resolved unknown identity and avoid letting a stopped lifecycle write empty state. A fresh independent retry signal would risk stale lifecycle writes unless paired with stronger lifecycle ownership.
