## Summary

Keep this PR open, but it is not merge-ready: the latest patch fixes the previously undefined formatter, yet the startup session lookup still runs on reconnect and is not scoped to the current agent, which conflicts with the TUI session model and can surface another agent's recent session preview.

## What This Changes

The PR adds a TUI startup-summary formatter and a non-local Gateway `sessions.list` call after history load to render the previous session's title or last-message preview in the TUI chat log.

## Best Possible Solution

Revise this PR before merge so the startup summary is inserted only on the initial non-local connection, the session lookup is scoped to the current agent, the best-effort failure behavior remains nonblocking, focused tests cover reconnect and cross-agent ordering, and the user-facing change is recorded in `CHANGELOG.md`.

## Reproduction Assessment

Yes for the PR regressions: trigger `client.onConnected` after a disconnect to observe duplicate summary insertion, and mock `sessions.list` with a newer session from another agent to confirm the unscoped lookup can select it.

## Solution Assessment

No in its current form. Using Gateway session preview data is a reasonable direction, but the maintainable solution needs current-agent scoping, one-time startup behavior, tests, and changelog coverage before merge.
