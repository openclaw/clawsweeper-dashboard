## Summary

The latest branch fixed the earlier undefined-helper and reconnect-duplication issues, but it still has merge-blocking defects: the startup lookup can summarize the wrong same-agent session when the restored session is outside the newest-10 window, and the PR head committed an unresolved changelog conflict marker.

## What This Changes

The branch adds a Gateway-backed TUI startup summary, scopes the session lookup to the current agent, skips local/reconnect fetches, adds focused TUI tests, and adds a changelog entry.

## Best Possible Solution

Keep the current-agent scoped, first-connect-only direction, but explicitly resolve the current/restored session before any recent-session fallback and remove the changelog marker before normal maintainer UX review.

## Reproduction Assessment

Yes for the PR findings by source inspection: the branch only searches within the newest 10 same-agent rows before falling back, while current main sorts limited session rows newest-first, and the PR head visibly contains a changelog conflict marker. This is not a current-main bug reproduction because the startup summary is new branch behavior.

## Solution Assessment

No. The scoped params and reconnect guard are good, but the implementation still needs an explicit current-session lookup and the committed conflict marker must be removed before merge.
