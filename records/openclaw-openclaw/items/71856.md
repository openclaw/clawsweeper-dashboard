## Summary

Keep open: the TUI scoping and reconnect guard are useful, but PR head can still display the wrong startup summary when the restored current session is outside the bounded sessions.list page, and latest-head real behavior proof is still insufficient.

## What This Changes

The PR adds a Gateway-backed non-local TUI startup summary scoped to current-agent sessions, skips reconnect/local fetches, adds TUI unit coverage, updates the changelog, and carries one import-format-only cron test change.

## Best Possible Solution

Resolve the current session by key before using a same-agent fallback, keep the diff TUI-scoped, and refresh latest-head real behavior proof.

## Reproduction Assessment

Yes. Source inspection is enough: PR head requests only ten newest current-agent sessions, while current main can restore an older currentSessionKey before loadHistory, so the fallback can pick a different session.

## Solution Assessment

No. Scoping and reconnect gating are the right direction, but the implementation should fetch, search, or describe the active current session before falling back to another recent row.
