## Summary

Keep open: the Gateway-backed TUI startup summary is a reasonable direction, but the current PR can duplicate summary lines on reconnect and can render a newer session preview from another agent because the new session lookup is unscoped.

## What This Changes

This PR adds a non-local TUI startup path that calls Gateway `sessions.list` after history load and renders the most recent non-current session title or last-message preview as system lines.

## Best Possible Solution

Revise the PR so the summary is inserted only once on the initial non-local connection, queries only the current agent's sessions, preserves best-effort failure handling, adds focused reconnect and cross-agent tests, and records the user-facing feature in `CHANGELOG.md`.

## Reproduction Assessment

Yes for the PR regressions: trigger `client.onConnected` after a disconnect to observe duplicate summary insertion, and mock `sessions.list` with a newer `agent:other:*` row to confirm unscoped preview selection.

## Solution Assessment

No in its current form. A Gateway-backed summary is maintainable only if it follows the TUI current-agent session model, avoids reconnect duplication, and includes the expected tests and changelog entry.
