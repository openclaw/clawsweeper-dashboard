## Summary

Keep open: the PR is useful and focused, but it is not merge-ready because the live branch is conflicting and source review shows the startup summary can describe a different same-agent session when the opened session is outside the top-10 recent list.

## What This Changes

The PR adds a Gateway-backed TUI startup summary after history load, helper-tested formatting and session selection, current-agent `sessions.list` params, first-connect gating, a changelog entry, and one incidental import-format change.

## Best Possible Solution

Use the existing session APIs to make the startup block describe the session the TUI actually opened, then let maintainers decide whether the automatic preview UX should land.

## Reproduction Assessment

Yes for the review finding by source inspection: `sessions.list` returns a limited recent list and the selector falls back to a non-current row when the current key is absent. The original linked request is a feature request, so a failing current-main reproduction is not applicable.

## Solution Assessment

No, not yet. The maintainable path is to use an exact current-session lookup, such as `sessions.describe` or a searched list, before any explicit recent-session fallback is allowed.
