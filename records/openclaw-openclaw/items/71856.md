## What This Changes

The PR adds a TUI startup conversation summary fetched once on initial non-local connection, adds `describeSession` backend methods, and includes unrelated support/test/script stabilizations.

## Best Possible Solution

Keep the TUI summary owner boundary, but either make the Gateway describe contract safely support agent-scoped global sessions or avoid this startup summary for global sessions, then rebase and drop unrelated support-test/script drift.

## Reproduction Assessment

Yes, source-reproducible: a non-local TUI opened on `global` with a non-default `currentAgentId` passes that agent to history loading but not to the new Gateway summary describe path.

## Solution Assessment

No; using `sessions.describe` is plausible for direct agent-session keys, but it is incomplete unless selected-agent scope is preserved for `global` sessions or the feature is gated away from that case.
