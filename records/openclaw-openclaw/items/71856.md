The branch adds a non-local TUI startup summary fetched through Gateway sessions.list, scopes it to the current agent, skips reconnect fetches, adds TUI unit coverage, updates the changelog, and includes one format-only cron test import change.

## Best Possible Solution

Keep the PR TUI-scoped, fetch or search the currentSessionKey before any fallback, only use a recent same-agent fallback after the current session is known absent, and refresh latest-head real behavior proof.

## Reproduction Assessment

Yes. Source inspection is enough: PR head asks for a limit-10 newest-first sessions.list page, while current main may restore an older currentSessionKey that is not in that page.

## Solution Assessment

No. Scoping to the current agent and skipping reconnects are the right direction, but the implementation should resolve the active/current session before falling back to another recent session.
