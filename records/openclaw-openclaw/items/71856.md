## Summary

Keep open: the PR still has blocking review findings because the new startup summary can run on every reconnect and can render an unscoped session preview from another agent; it also needs the required changelog entry before merge.

## What This Changes

This PR adds a TUI startup-summary formatter and a non-local Gateway `sessions.list` call after history load to display a previous session title or last-message preview.

## Best Possible Solution

Revise the PR so the summary is inserted only once on the initial non-local connection, queries only the current agent's sessions, preserves nonblocking failure behavior, adds focused reconnect and cross-agent tests, and records the feature in `CHANGELOG.md`.

## Reproduction Assessment

Yes for the PR regressions: trigger `client.onConnected` after a disconnect to observe duplicate summary insertion, and mock `sessions.list` with a newer session from another agent to confirm the unscoped preview can be selected.

## Solution Assessment

No in its current form. A Gateway-backed summary is a reasonable direction, but the maintainable solution needs current-agent scoping, one-time startup behavior, test coverage, and changelog coverage before merge.
