## Summary

Keep open: the latest head fixed the missing formatter, reconnect guard, and changelog marker, but the startup summary can still show the newest same-agent session instead of the active restored or explicit session when that active session is outside the bounded newest list.

## What This Changes

The PR adds a Gateway-backed TUI startup summary that formats recent session context, scopes session lookup to the current agent, skips local mode and reconnects, and adds focused TUI tests plus a changelog entry.

## Best Possible Solution

Keep the scoped, first-connect-only TUI lookup, but resolve or fetch the active `currentSessionKey` row first and only use the newest same-agent fallback when there is no active session to summarize.

## Reproduction Assessment

Yes. The remaining review finding is reproducible by source inspection: the PR head still requests a bounded newest session list and then falls back to the first row when the active session is absent from that list.

## Solution Assessment

No. The scoped params and reconnect guard are the right direction, but the current patch is not merge-ready until the startup summary prefers the active restored or explicit session before any newest-session fallback.
