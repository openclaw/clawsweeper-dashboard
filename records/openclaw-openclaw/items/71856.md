## Summary

Keep this PR open: the missing formatter and reconnect duplication from earlier review are fixed, but the latest head can still show a startup summary from the wrong same-agent session when the restored or explicit current session is older than the bounded newest-session list.

## What This Changes

The PR adds a Gateway-backed TUI startup conversation summary scoped to the current agent, skips local mode and reconnects, and adds focused tests plus a changelog entry.

## Best Possible Solution

Keep the scoped, first-connect-only startup summary, but resolve or fetch the active `currentSessionKey` row first and only use the newest same-agent fallback when there is no active session to summarize.

## Reproduction Assessment

Yes. The remaining issue is reproducible by source inspection: the PR head asks for a bounded newest-session list and falls back to `sessions[0]` when the active `currentSessionKey` is absent.

## Solution Assessment

No. The scoped params and reconnect guard are the right direction, but the current selection logic should search or fetch the active restored or explicit session before falling back to any newest same-agent row.
