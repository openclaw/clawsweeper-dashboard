## Summary

Keep this PR open: the TUI direction is useful, but the latest head can still display a summary for the wrong session when the active session is outside the bounded list response, and the branch also carries duplicate changelog text plus unrelated Microsoft Teams Graph changes.

## What This Changes

The PR adds TUI startup-summary helpers/tests and a first-connect Gateway `listSessions` lookup, updates `CHANGELOG.md`, and currently also changes Microsoft Teams Graph fetch handling.

## Best Possible Solution

Keep the current-agent/reconnect guard direction, but resolve or fetch the active `currentSessionKey` before any fallback, remove unrelated Microsoft Teams drift, keep only one active changelog entry, and add latest-head runtime proof.

## Reproduction Assessment

Yes. Source inspection is enough: PR head asks for only the newest 10 current-agent sessions, while current main applies that limit newest-first, so an older active session can be absent and the fallback can choose a different session.

## Solution Assessment

No. The scoped params and reconnect guard are the right direction, but the patch should not fall back from a bounded response until the active session is known missing, and it should not carry unrelated Microsoft Teams changes.
