## Summary

Keep this PR open. Current main still does not implement the dynamic TUI startup summary, and the PR is a plausible implementation path, but the latest diff still has blocking reconnect and session-scope defects before it should merge.

## What This Changes

The PR adds a TUI startup-summary formatter and a non-local Gateway `client.listSessions` call that renders a previous session title or last-message preview after startup history loads.

## Best Possible Solution

Revise this PR before merge so the startup summary is a one-time initial-connect behavior, uses the current-agent session scope, has focused regression coverage, and includes the required user-facing changelog entry if the feature ships.

## Reproduction Assessment

Yes for the PR regressions: trigger `client.onConnected` again after a disconnect to see duplicate summary lines, and mock `sessions.list` with a newer session from another agent to see it selected because the call omits `agentId`.

## Solution Assessment

No in its current form. The Gateway sessions preview API is a reasonable source, but the patch needs reconnect guarding and current-agent scoping to be the narrow maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Guard startup summaries on reconnect:** `src/tui/tui.ts:1105`
  - body: The new block runs inside `client.onConnected`, which current main uses for both the first connection and reconnects via `reconnected = wasDisconnected`. Because the condition is only `!isLocalMode`, every Gateway reconnect re-runs `listSessions` and appends another copy of the summary lines to the chat log; add a reconnect guard or a one-time insertion flag.
  - confidence: 0.93
- **[P2] Scope the session lookup to the current agent:** `src/tui/tui.ts:1106-1111`
  - body: This `listSessions` call omits `agentId`, while `sessions.list` returns all non-global/non-unknown sessions when no agent filter is supplied. In a multi-agent setup, opening one agent's TUI can display a newer title or preview from another agent even though the docs and session picker scope sessions to the current agent.
  - confidence: 0.88

  - body: This is a user-facing `feat` change, but the PR only changes `src/tui/tui.ts`. Repo policy requires a single-line `CHANGELOG.md` entry under Unreleased with contributor credit before shipping this TUI behavior.
  - confidence: 0.82
