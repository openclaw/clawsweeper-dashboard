## Summary

Keep this PR open: the TUI scoping/reconnect direction is useful, but the current head still has a source-reproducible fallback bug, duplicate changelog placement, and unrelated Microsoft Teams Graph changes that are not covered by the supplied proof.

## What This Changes

The PR adds TUI startup-summary helpers/tests, fetches recent session summary data from Gateway `listSessions`, updates `CHANGELOG.md`, and currently also changes Microsoft Teams Graph fetch handling.

## Best Possible Solution

Keep the current-agent and reconnect guard approach, but fetch or search for the active `currentSessionKey` before fallback, remove duplicate changelog placement, drop or separately review the Microsoft Teams drift, and add latest-head runtime proof.

## Reproduction Assessment

Yes. Source inspection is enough: PR head asks for only the newest 10 current-agent sessions, while current main applies that limit newest-first, so an older restored/current session can be omitted and the fallback can choose a different session.

## Solution Assessment

No. The scoped list params and reconnect guard are the right direction, but the patch should not fall back from a bounded response until the active row is known absent, and the unrelated Microsoft Teams changes should not ride along with this TUI feature.
