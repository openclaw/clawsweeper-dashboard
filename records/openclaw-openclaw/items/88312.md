## Summary

Keep open. Current main contains several Codex app-server timeout and replay mitigations, but the core failure shape is still an explicit runtime path: when Codex emits tool/raw-item progress but no terminal `turn/completed`, OpenClaw times out, marks the turn incomplete, and blocks unsafe replay after tool activity; related open reports remain active for the same session/message-loss family.

## What This Changes

The issue reports a regression where long, tool-heavy Codex app-server turns fail before terminal completion after OpenClaw 2026.5.27, causing missing final replies or delivery across cron, Telegram, Discord, and hooks.

## Best Possible Solution

Keep this issue open as the canonical Codex app-server turn-completion/recovery problem until maintainers decide the safe terminal-state policy and prove it against a live long, tool-heavy Codex turn.
