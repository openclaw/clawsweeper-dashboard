## Summary

Keep open: current main still has a source-level visibility gap where plugin-owned `afterTurn()` compactions cannot report a count back to the session store that `/status` and `session_status` display.

## What This Changes

The issue requests a way for plugin context engines that own compaction to report post-turn/proactive compactions into OpenClaw session status.

## Best Possible Solution

Add an additive, documented plugin-facing compaction accounting seam and update the status/session-store path so owning context engines can report successful background compactions without reaching into core internals.

## Reproduction Assessment

Yes, at source level. Configure an `ownsCompaction: true` engine that performs successful compaction in `afterTurn()`; current main provides no way for that hook to report a count, while status renders only `sessionEntry.compactionCount`.

## Solution Assessment

Unclear. Option A is plausible, but the API shape is a public plugin contract decision and should be chosen with SDK/docs/API-baseline compatibility in mind.
