## Summary

Keep open. Current main and the latest release still have the core accounting gap: plugin `afterTurn()` can perform engine-owned compaction without any count notifier/result path, while `/status` and `session_status` render only persisted session `compactionCount`.

## What This Changes

The issue asks OpenClaw to let plugin-owned context-engine compactions increment or report the session compaction count shown by `/status`.

## Best Possible Solution

Choose a backward-compatible plugin-facing compaction reporting contract, persist successful engine-owned compaction deltas into the existing session store, and cover both `/status` and `session_status` without breaking existing `/compact` accounting.

## Reproduction Assessment

Yes, by source inspection. A plugin-owned engine can perform proactive compaction in `afterTurn()`, but current main provides no count notifier/result path back to the persisted session store that `/status` reads.

## Solution Assessment

Unclear as an implementation choice. The issue correctly identifies the gap, but maintainers still need to choose the stable plugin API shape before code changes.
