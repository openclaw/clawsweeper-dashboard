## Summary

Keep open. Current main and the latest release still have no plugin-owned afterTurn/maintenance compaction accounting path into the persisted session counter that /status and session_status render; the related Codex logging PR fixed an adjacent observability gap, not this plugin-engine count gap.

## What This Changes

The issue asks OpenClaw to let plugin-owned context-engine compactions update the session compaction count shown by /status and session_status.

## Best Possible Solution

Add one additive plugin-facing compaction accounting contract that lets successful engine-owned compaction deltas persist into SessionEntry.compactionCount for both /status and session_status, without engine-specific core reads.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: an ownsCompaction engine can compact inside afterTurn/maintenance, but current main has no callback, return value, or stats method that can increment the persisted session counter. I did not run a live third-party engine in this read-only review.

## Solution Assessment

Unclear as an implementation choice: callback, return value, and stats-query shapes are all plausible, but the best fix is an additive, idempotent plugin API contract selected by maintainers rather than a core special-case.
