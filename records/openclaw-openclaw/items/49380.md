## Summary

Keep open. Current main and the latest release still have the reported accounting gap: plugin-owned `afterTurn()` compactions have no notifier/result channel, while `/status` renders only the persisted session `compactionCount`; the duplicate request at https://github.com/openclaw/openclaw/issues/60332 is closed in favor of this issue, and the open Codex notification issue is adjacent rather than canonical.

## What This Changes

The issue asks OpenClaw to let plugin-owned context engine compactions update the `/status` and session compaction counter.

## Best Possible Solution

Keep this canonical issue open until maintainers choose and implement a plugin-safe compaction reporting contract for status/session metadata.

## Reproduction Assessment

Yes. Source inspection shows `afterTurn()` still returns `Promise<void>` with no notifier/result channel, while `/status` renders only `SessionEntry.compactionCount`.

## Solution Assessment

No. The current embedded compaction event reconcile path helps runtime-emitted compaction events, but it does not solve plugin-owned background compactions without a plugin reporting contract.
