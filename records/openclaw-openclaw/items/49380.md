## Summary

Keep open. Current main and the latest release still have the reported accounting gap: plugin-owned `afterTurn()` compactions have no notifier/result channel, while `/status` and `session_status` render only the persisted session `compactionCount`; the duplicate request at https://github.com/openclaw/openclaw/issues/60332 was already closed in favor of this canonical issue.

## What This Changes

Plugin-owned context engine compactions should be able to update the session compaction count shown by `/status` and `session_status`.

## Best Possible Solution

Add one additive, generic compaction-reporting mechanism for context engines and persist successful plugin-owned compaction deltas into the existing session counter used by both `/status` and `session_status`.

## Reproduction Assessment

Yes for source-level reproduction: current `afterTurn()` has no return value or notifier, and the status surfaces render only the persisted session counter. I did not run a live third-party engine setup.

## Solution Assessment

Unclear until maintainers choose the plugin API shape. The narrow fix should be additive and count only successful plugin-owned compactions, not every `afterTurn()` or maintenance run.
