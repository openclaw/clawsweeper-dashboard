## Summary

Keep open. Current main and the latest release still lack a host-visible way for plugin-owned `afterTurn()` compactions to update the persisted session counter that `/status` and `session_status` render.

## What This Changes

The issue asks for plugin-owned context-engine compactions to update the persisted `/status` and `session_status` compaction counter.

## Best Possible Solution

Choose an additive plugin-owned compaction reporting contract, persist validated deltas into the existing session counter, and update SDK docs plus focused status/session-store tests.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: an `ownsCompaction` engine can compact inside `afterTurn()`, but that hook returns `Promise<void>` and current status surfaces read only persisted `SessionEntry.compactionCount`.

## Solution Assessment

Unclear as an implementation choice. The issue identifies the right gap, but maintainers need to choose the stable plugin API shape before code changes.
