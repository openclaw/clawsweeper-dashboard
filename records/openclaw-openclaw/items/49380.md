## Summary

Keep open. Current main and the latest release still lack a reporting path from plugin-owned `afterTurn()` compactions into the persisted session counter that `/status` and `session_status` render; the newly merged Codex PR only added app-server logging and explicitly preserved existing count behavior.

## What This Changes

The issue requests a plugin-facing way for context-engine compactions performed in `afterTurn()` to update the persisted `/status` and `session_status` compaction count.

## Best Possible Solution

Choose an additive context-engine compaction reporting contract and thread confirmed plugin-owned compaction deltas into the existing persisted session-status counter with focused tests and docs.

## Reproduction Assessment

Yes, by source inspection. A plugin-owned engine can perform proactive compaction in `afterTurn()`, but current main provides no count notifier or result path back to the persisted session store that `/status` reads.

## Solution Assessment

Unclear as an implementation choice. The issue correctly identifies the gap, but the best fix requires maintainers to choose the stable plugin API shape before code changes.
