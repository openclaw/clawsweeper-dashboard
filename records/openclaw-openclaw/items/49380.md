## Summary

Keep open. Current main still has a source-level gap: plugin-owned context engines can do proactive compaction in `afterTurn`, but that hook has no compaction-reporting channel and `/status` still renders only the persisted session-store counter.

## What This Changes

The issue asks for plugin-owned context-engine compactions that happen in `afterTurn` to increment the `/status` session compaction counter.

## Best Possible Solution

Define an additive context-engine compaction reporting contract, persist confirmed plugin-owned deltas into `SessionEntry.compactionCount`, and cover `afterTurn`, manual compact, overflow, and built-in runtime paths without double-counting.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: an `ownsCompaction` engine can compact in `afterTurn`, but that hook returns `Promise<void>` and status reads only `SessionEntry.compactionCount`.

## Solution Assessment

Unclear as an implementation choice. The reported gap is valid, but maintainers still need to choose the stable plugin API shape before a code fix should proceed.
