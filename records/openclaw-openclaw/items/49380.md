## Summary

Keep open. Current main still has no accounting channel for successful plugin-owned proactive compactions that happen from `afterTurn`, and the related duplicate was closed specifically to keep this issue as the canonical tracker.

## What This Changes

The issue requests a way for plugin-owned context-engine compactions to be reflected in `/status` and `session_status` compaction counters.

## Best Possible Solution

Add a documented, backward-compatible context-engine accounting signal that lets owning engines report successful compaction deltas into `SessionEntry.compactionCount`, with shared `/status` and `session_status` coverage.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: an `ownsCompaction` engine can compact from `afterTurn`, but current main gives that hook no return value or callback that can update the persisted counter read by `/status` and `session_status`.

## Solution Assessment

Unclear. The suggested callback is plausible, but the best fix requires a maintainer choice of public SDK shape and idempotent counting semantics rather than an automated one-off patch.
