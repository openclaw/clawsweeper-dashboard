## Summary

Keep open. Current `main` and latest release still lack a host-visible reporting path from plugin-owned `afterTurn()` compactions into the persisted session counter that `/status` and `session_status` render; the recent Codex logging PR is adjacent and explicitly out of scope for this count behavior.

## What This Changes

The issue asks OpenClaw to let plugin-owned context-engine compactions update the `/status` and `session_status` compaction counter.

## Best Possible Solution

Choose an additive context-engine compaction reporting contract and persist confirmed plugin-owned compaction deltas into the existing session-status counter with focused tests and docs.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: an `ownsCompaction` engine can compact inside `afterTurn()`, but that hook returns `Promise<void>` and current status surfaces read only persisted `SessionEntry.compactionCount`.

## Solution Assessment

Unclear as an implementation choice. The issue identifies the right gap, but the best fix requires maintainers to choose the stable plugin API shape before code changes.
