## Summary

Keep open: current main still has the visibility gap. Plugin context engines can run proactive compaction from `afterTurn`, but the hook has no way to report successful compactions and status still displays only the session-store counter.

## What This Changes

The issue asks OpenClaw to count plugin-owned context-engine compactions in `/status` and `session_status`.

## Best Possible Solution

Add a backwards-compatible context-engine reporting seam, reconcile reported compactions into the session store, and cover `/status` plus `session_status` with regression tests and docs.

## Reproduction Assessment

Yes, from source inspection. A plugin can compact in `afterTurn`, but OpenClaw neither passes a notification callback nor reads a count, while status renders only `SessionEntry.compactionCount`.

## Solution Assessment

Unclear. Option A from the issue is plausible, but the best fix requires a maintainer API decision because it changes the public context-engine contract.
