## Best Possible Solution

Review and either land or close https://github.com/openclaw/openclaw/pull/96293 as the canonical fix, preserving omitted-field merge semantics while carrying explicit null clears through UI, protocol schema, normalization, merge logic, CLI parity, docs, and tests.

## Solution Assessment

Yes. The narrow maintainable fix is to send explicit null clears only for stored overrides and preserve `thinking: null` through the same schema, normalization, and merge layers already used for `model: null`.
