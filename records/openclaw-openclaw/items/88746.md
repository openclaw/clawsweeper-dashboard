## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Store Sentinel keys in the existing secret boundary:** `scripts/docker/full-local.mjs:230-233`
  - body: `seedNvidiaVaultFromRuntime` writes NVIDIA API keys into a new `vault.json` under the OpenClaw state/workspace tree. OpenClaw already has SecretRef/auth-profile surfaces for user-supplied provider keys, so this adds a second persistent credential source with weaker review and migration coverage; wire Sentinel to the existing credential boundary instead of materializing provider keys here.
  - confidence: 0.94
- **[P1] Keep Docker publishing upgrade-safe:** `docker-compose.yml:64-66`
  - body: These defaults change the existing compose Gateway/bridge/Teams host publishing from Docker's normal all-interface binding to `127.0.0.1`. That is a security-hardening direction, but merged as the default it can break existing LAN clients, callbacks, and remote Docker setups without a migration or explicit maintainer-approved compatibility story.
  - confidence: 0.91
- **[P1] Allow documented ticket states in SQLite:** `scripts/docker/sidecars/blackboard-cli.cjs:150-151`
  - body: The new Agent OS contract documents `WAITING_APPROVAL` and `BLOCKED`, and the normalizer accepts them, but the Blackboard table check constraint still rejects both states. Any adapter that follows the documented lifecycle will fail at runtime when updating a ticket to one of those states, and the signal-hub schema has the same shortened list.
  - confidence: 0.96
- **[P1] Preserve existing per-agent skill choices:** `scripts/agents/capability-agent-profile.mjs:190-191`
  - body: When `apply` refreshes an existing capability agent, it unconditionally replaces `skills` and `contextInjection`. Existing config docs define `agents.list[].skills` as a user-owned full replacement, so refreshing these profiles can silently overwrite a user's per-agent skill policy instead of only adding missing Agent OS metadata.
  - confidence: 0.88
