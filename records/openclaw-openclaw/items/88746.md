Full review comments:

- **[P1] Keep remote Docker publishing upgrade-safe:** `docker-compose.yml:64-66`
  - body: Changing the base compose ports to default to 127.0.0.1 makes existing Docker deployments stop accepting remote Gateway, bridge, or Teams traffic unless operators discover the new *_PUBLISH_HOST variables. Keep the shipped publish behavior by default, move loopback binding to the opt-in full-local path, or add an explicit maintainer-approved migration path with docs/tests.
  - confidence: 0.9
- **[P1] Reuse the shared workspace dotenv guard:** `extensions/nvidia/sentinel-gateway.cjs:169`
  - body: This writes workspace dotenv values into process.env with a Sentinel-local partial blocklist, while the shared loader blocks provider credentials, ClawHub, browser-control, OPENCLAW_SKIP_*, OPENCLAW_UPDATE_*, and host-env override keys. A checkout-controlled .env can change Sentinel/provider behavior before the trusted state env is read, so reuse the shared policy or generated shared guard instead of maintaining a second one.
  - confidence: 0.87
- **[P2] Allow every documented ticket status in storage:** `scripts/docker/sidecars/blackboard-cli.cjs:131`
  - body: The Agent OS contract advertises WAITING_APPROVAL and BLOCKED, and the normalizer accepts them, but the Blackboard table CHECK constraint only allows OPEN, CLAIMED, IN_PROGRESS, DONE, FAILED, and ARCHIVED. Updating a ticket to either documented status will fail with a SQLite constraint error, so align the schema and sibling sidecar schemas with the contract and add a regression case.
  - confidence: 0.92
