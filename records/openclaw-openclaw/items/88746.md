Overall confidence: 0.89

Full review comments:

- **[P1] Store Sentinel keys in the existing secret boundary:** `scripts/docker/sidecars/nvidia-key-validator.cjs:12-13`
  - body: `VAULT_FILE` defaults to a JSON file under `.openclaw/workspace_nvidia_key_sentinel`, and `saveVault` writes seeded `NVIDIA_API_KEYS` there with plain `writeFileSync`. That bypasses the existing SecretRef/credentials storage boundary for provider secrets and can leave NVIDIA keys as ordinary workspace/state files; keep these keys in the credential/SecretRef path or add a restrictive, documented migration before merge.
  - confidence: 0.91
- **[P1] Keep Docker publishing upgrade-safe:** `docker-compose.yml:64-66`
  - body: Changing the existing Gateway, bridge, and Teams compose mappings to default to `127.0.0.1` will make Docker installs that currently rely on host/LAN reachability or external callback services stop reaching those ports unless operators discover the new `*_PUBLISH_HOST` variables. Please preserve the existing upgrade default and make loopback an explicit hardening mode, or add a tested migration/docs path that maintainers intentionally approve.
  - confidence: 0.88
- **[P1] Preserve existing per-agent skill policy:** `scripts/agents/capability-agent-profile.mjs:190-191`
  - body: When `apply` refreshes a profile whose id already exists, it unconditionally replaces `skills` and `contextInjection`. `agents.list[].skills` is a documented final allowlist, so this can silently grant or remove an existing agent's configured skill policy; only add the Agent OS metadata by default or require an explicit overwrite mode.
  - confidence: 0.86
- **[P2] Allow documented ticket states in SQLite:** `scripts/docker/sidecars/blackboard-cli.cjs:151`
  - body: The contract and docs list `WAITING_APPROVAL` and `BLOCKED`, but the new Blackboard schema only allows `OPEN`, `CLAIMED`, `IN_PROGRESS`, `DONE`, `FAILED`, and `ARCHIVED`. Any agent that sets a documented blocked/approval state will normalize successfully and then hit a SQLite CHECK constraint; update both Blackboard schemas and add coverage for every documented status.
  - confidence: 0.9
