## Summary

Keep open: current main still has the ACPX test/resolver mismatch, and an open linked fix PR already owns the narrow test-helper change, so this issue should stay open until that PR lands or is closed.

## What This Changes

The issue reports an ACPX config test that expects source-mode MCP server args even though the resolver returns built dist MCP server paths when they exist.

## Best Possible Solution

Land or revise the linked PR so the ACPX test helper mirrors the resolver's dist-first/source-fallback contract while preserving the default-off bridge assertions.

## Reproduction Assessment

Yes. In a built checkout where dist/mcp/plugin-tools-serve.js exists, current source inspection and the reported pnpm test failure show the test expects source args while the resolver returns a dist path.

## Solution Assessment

Yes. The narrow maintainable solution is a test-only helper update that follows the existing resolver contract; changing production resolver behavior would be broader and is not supported by the docs or current code.
