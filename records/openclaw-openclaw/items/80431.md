## What This Changes

The issue asks for the ACPX config test helper to match the built-in MCP resolver when `dist/mcp/*.js` exists.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/80475 or an equivalent test-only helper fix that mirrors the resolver's dist-first contract while preserving the explicit bridge enablement checks.

## Reproduction Assessment

Yes. Source inspection plus the issue's failing `pnpm test` output show that a built checkout with `dist/mcp/*.js` makes the resolver return dist args while the current test expects source TSX args unconditionally.

## Solution Assessment

Yes. The narrow maintainable solution is to update the ACPX config test helper to mirror the resolver's dist-first path construction, not to change the production resolver behavior.
