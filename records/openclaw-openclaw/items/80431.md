## Summary

Keep open: current main still has an environment-sensitive ACPX test mismatch where the resolver returns the built `dist` MCP entry when it exists but the config test always expects the source `tsx` entry.

## What This Changes

The issue requests aligning the ACPX plugin-tools MCP bridge config test with the resolver's dist-first/source-fallback behavior without weakening the explicit opt-in assertion.

## Best Possible Solution

Keep the resolver contract and make the ACPX config test hermetic for both dist-present and source-fallback cases while preserving the explicit `pluginToolsMcpBridge` opt-in behavior.
