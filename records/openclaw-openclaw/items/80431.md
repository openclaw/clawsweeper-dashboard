## Repair Work Prompt

Fix the ACPX config test mismatch reported in https://github.com/openclaw/openclaw/issues/80431. The resolver in `extensions/acpx/src/config.ts` intentionally returns `node dist/mcp/plugin-tools-serve.js` when the built entry exists and falls back to `node --import tsx src/mcp/plugin-tools-serve.ts` otherwise, while `extensions/acpx/src/config.test.ts` currently expects the source path unconditionally. Make the test/helper hermetic or environment-aware so it covers the dist-present and source-fallback behavior without weakening the assertion that the plugin-tools and OpenClaw-tools MCP bridges are injected only when explicitly enabled and reserved names still conflict. Do not change runtime preference unless the code evidence shows the resolver is wrong. No changelog is expected for a test-only fix; add one only if runtime behavior changes.
