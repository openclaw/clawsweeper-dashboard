## Summary

This should stay open: current main still has an environment-dependent ACPX config test contract where the resolver returns the built plugin-tools MCP server when `dist/` exists, but the test always asserts the source `tsx` path.

## What This Changes

The issue asks to align the ACPX plugin-tools MCP config test with the resolver path returned when built `dist` output exists.

## Best Possible Solution

Keep the resolver behavior unchanged unless maintainers want different runtime semantics, and make the ACPX config test deterministic by covering or controlling the source and dist path cases.
