## Summary

Keep open: current main and the latest release still expose `sessions tail` and `sessions export-trajectory`, but not the requested `sessions diagnose` failure-summary/fix-suggestion surface. The linked fork PR is useful implementation context, but this changes public CLI/JSON output and redaction behavior, so it needs a maintainer product/API decision before a repair lane should implement it.

## What This Changes

The issue requests a new `openclaw sessions diagnose` command with failure summaries, suggested fixes, per-round failure tags, brief-mode context, and JSON fields.

## Best Possible Solution

Define a core sessions-diagnosis contract that reuses trajectory events, ACP metadata, and diagnostic activity snapshots, defaults to safe redaction, and documents stable text/JSON output before implementation.

## Reproduction Assessment

## Solution Assessment
