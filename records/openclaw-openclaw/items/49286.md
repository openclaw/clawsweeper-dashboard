The Discord bug is source-reproducible on current main and the PR is aimed at the right mapper, but the branch is not current-main-compatible: its tests still use the old Carbon import path, its preflight regression mocks the wrong runtime module, its changelog entry lacks required contributor credit, and #49308 overlaps the same fix area.

## What This Changes

The PR expands Discord native MessageType system-event mapping, adds mapper and inbound preflight regression coverage, updates a Discord preflight test helper, and adds a changelog fix entry.

## Best Possible Solution

Land one current-main-compatible Discord plugin fix that maps the missing native system MessageType cases through resolveDiscordSystemEvent, preserves user-authored passthrough types, covers mapper and preflight enqueue/drop behavior, and credits the changelog.

## Reproduction Assessment

Yes. Current main's mapper returns null for the reported native Discord system MessageTypes, while preflight only queues and drops messages after resolveDiscordSystemEvent returns text; the pinned discord-api-types enum exposes those values.

## Solution Assessment

No. Expanding the mapper is the right narrow direction, but this branch needs current-main Discord shim imports, the current system-event runtime mock, changelog credit, and coordination with #49308 before it is landable.
