## Summary

Keep open: current main still contains the Feishu streaming-card delivery paths described in the report, including queued final updates that are not awaited, block updates appended as deltas, broad late-final suppression, and pending-text merge state that can retain stale content.

## What This Changes

The issue requests fixes for Feishu/Lark streaming cards losing final text, preserving stale content, or duplicating text across block, final, idle-timeout, and throttled update delivery paths.

## Best Possible Solution

Fix the Feishu plugin dispatcher and streaming-session state so final card updates are flushed deterministically, block snapshots do not accumulate unrelated text, stale pending content cannot overwrite finals, and duplicate suppression only skips content already delivered.

## Reproduction Assessment

Yes at source level: current main has the exact dispatcher/session branches described in the report, and existing tests already assert duplicated block output and broad distinct-late-final suppression. I did not run a live Feishu card reproduction in this read-only review.

## Solution Assessment

Yes, a narrow Feishu plugin fix is the right direction because the broken behavior is contained in `reply-dispatcher.ts`, `streaming-card.ts`, and their tests. The safest implementation should update those paths without changing core channel streaming contracts or adding config.

## Repair Work Prompt

Fix the Feishu/Lark streaming-card delivery bugs reported in https://github.com/openclaw/openclaw/issues/77685. Work only in the Feishu plugin and necessary changelog/tests. In `extensions/feishu/src/reply-dispatcher.ts`, make final streaming-card updates flush deterministically before delivery returns/idle close can clear state; treat REACT/block payloads as snapshot-style content unless preserving block-only fallback requires a more precise suffix strategy; allow distinct final text after an idle-closed streaming card while still deduping text already delivered; and cover the final-suffix case where final text contains prior block context. In `extensions/feishu/src/streaming-card.ts`, prevent stale `pendingText` and pending flush timers from being used as merge input for authoritative final/card updates, and adjust close ordering so queued updates are not canceled before they can settle. Add focused regression coverage in `extensions/feishu/src/reply-dispatcher.test.ts` and `extensions/feishu/src/streaming-card.test.ts` for block duplication, distinct late final fallback, final flush ordering, and stale pending text. Preserve existing media delivery, duplicate-final suppression for already-delivered text, reasoning preview behavior, and `blockStreaming` defaults. Add a single-line user-facing `CHANGELOG.md` fix entry; do not change core channel streaming contracts unless a generic seam is strictly necessary.
