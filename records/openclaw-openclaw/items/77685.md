## Repair Work Prompt

Fix the Feishu streaming-card content delivery bugs tracked by https://github.com/openclaw/openclaw/issues/77685. Work in TypeScript source, not built JavaScript or node_modules. Keep the repair local to the Feishu plugin: likely `extensions/feishu/src/reply-dispatcher.ts`, `extensions/feishu/src/streaming-card.ts`, and their colocated tests. Coordinate with https://github.com/openclaw/openclaw/pull/74146 or include an equivalent block snapshot/dedupe fix so overlapping block payloads do not duplicate text, but also handle the broader cases: final delivery must wait for queued streaming updates before close, close must not discard pending flush state before the queue is drained, distinct late final text after streaming close must fall back to normal delivery unless that exact final was already delivered, and stale `pendingText`/flush timers must not overwrite or suppress final card content. Preserve independent block-only fallback behavior and avoid dispatcher access to private session internals by adding an appropriate session method if needed. Add focused regression coverage in `reply-dispatcher.test.ts` and `streaming-card.test.ts`; add a changelog entry because this is user-visible channel behavior. Do not change core channel contracts or introduce a new config option.
