## Summary

## What This Changes

The PR changes transcript header creation to accept an explicit cwd, resolves that cwd from the target agent workspace before mirrored assistant transcript appends, and adds a regression test for cwd isolation.

## Best Possible Solution

Rebase or replace this with a current-main patch that derives transcript header cwd from the target session/agent using the already-supplied runtime config, preserves best-effort mirror failure handling, and includes focused regression plus redacted real transcript proof.

## Reproduction Assessment

Yes. By source inspection on current main, a stored mirrored session entry without `spawnedCwd` reaches header creation without an explicit cwd, where `createSessionTranscriptHeader()` falls back to `process.cwd()`.

No. The fix direction is right, but this branch is not the best current solution because it reloads config in the mirror append path and predates the current transcript writer seam.
