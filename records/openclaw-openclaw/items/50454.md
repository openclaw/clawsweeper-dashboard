## Summary

Keep this PR open, but do not merge it as submitted. The underlying bug is still present on current main and the branch moves in the right direction, but it introduces config reload and agent-id consistency problems, lacks real after-fix behavior proof, and now conflicts with current main plus an overlapping open fix path.

## What This Changes

The PR changes mirrored transcript header creation to write an explicit agent workspace cwd instead of `process.cwd()`, and adds a mocked regression test for cwd isolation.

## Best Possible Solution

Land one current-main-compatible fix that derives one effective agent id and workspace from existing runtime config for every create-if-missing transcript header path, with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows a high-confidence path: create a brand-new mirrored transcript while process cwd differs from the target agent workspace, and `ensureSessionHeader()` writes the process-global cwd into the header.

## Solution Assessment

No. The direction is right, but the submitted patch reloads config after send and uses different agent identity inputs for cwd and path resolution; the safer fix is to thread existing config and one effective agent id into header creation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Pass existing config into mirrored transcript appends:** `src/config/sessions/transcript.ts:162`
  - body: The added `loadConfig()` runs inside post-send transcript mirroring. If the on-disk config is temporarily invalid or unreadable, a successful delivery can be reported as failed and retried because bookkeeping now throws; use the caller's existing runtime config instead.
  - confidence: 0.86
- **[P2] Use one effective agent id consistently:** `src/config/sessions/transcript.ts:163-167`
  - body: The patch derives a trimmed/default `agentId` for `sessionCwd`, but session store and file resolution still use `params.agentId`. Blank or omitted ids can create a transcript whose header cwd points at one agent while the transcript path is resolved under another/raw agent context.
  - confidence: 0.78
