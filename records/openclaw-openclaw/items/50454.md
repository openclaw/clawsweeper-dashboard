## Summary

Keep open for maintainer consolidation. The current-main bug remains valid, but this branch is merge-conflicting, lacks contributor real-behavior proof, and introduces split config/agent-id handling that can affect delivery and session state.

## What This Changes

The PR changes mirrored transcript header creation to pass an explicit cwd resolved from the target agent id and adds a regression test for cwd isolation.

## Best Possible Solution

Land one consolidated current-main-compatible fix that derives transcript file, store, and header cwd from the caller runtime config and one normalized effective agent id, with proof for the real transcript path.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: create a brand-new mirrored transcript while `process.cwd()` differs from the target agent workspace, and current main writes the process-global cwd into the header; I did not run a write-producing test because this review is read-only.

## Solution Assessment

No. Explicit workspace resolution is the right direction, but this PR should use the caller runtime config and one effective agent id for both file/store and header cwd resolution instead of reloading config and splitting identity handling.
