## Summary

Keep open. The underlying transcript-header bug remains on current main, and this PR points in the right direction, but the submitted patch should not merge because it adds a hot-path config reload, uses inconsistent agent-id resolution, lacks real behavior proof, and overlaps an open broader fix candidate.

## What This Changes

The branch changes mirrored transcript header creation to resolve a target agent workspace before writing `cwd` and adds a mocked regression test.

## Best Possible Solution

Land one current-main-compatible fix that removes `process.cwd()` from transcript header create-if-missing paths, threads existing runtime config and one effective agent id consistently, and adds focused regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows a high-confidence path: create a brand-new mirrored transcript while process cwd differs from the target agent workspace, and `ensureSessionHeader()` writes the process-global cwd into the header.

## Solution Assessment

No. The direction is right, but this patch reloads config in the post-send mirror append path and applies a resolved/default agent id only to header cwd while store/session-file resolution still uses the raw parameter.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Thread runtime config into mirror appends:** `src/config/sessions/transcript.ts:162`
  - body: `deliverOutboundPayloads()` and plugin send actions await transcript mirroring after sends can already have succeeded. Adding `loadConfig()` here means an invalid or unreadable config can turn bookkeeping into a failed or retryable delivery; pass the already-available runtime config/workspace context instead.
  - confidence: 0.86
- **[P2] Use one effective agent id consistently:** `src/config/sessions/transcript.ts:163-167`
  - body: The patch computes a trimmed/default agent id for the header cwd, but store path and session-file resolution still use `params.agentId`. Calls with an omitted or whitespace id can mix one agent's header cwd with another store/path; compute the effective id before store/path resolution and pass it through.
  - confidence: 0.78
