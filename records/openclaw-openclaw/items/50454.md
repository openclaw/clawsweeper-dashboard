## Summary

Keep open. Current main still has the mirrored transcript workspace-isolation bug, but this external branch is not merge-ready because it reloads config during post-send mirroring, splits effective agent identity between cwd and session path handling, lacks real behavior proof, and overlaps an open bug plus a broader open candidate fix.

## What This Changes

The PR changes mirrored transcript header creation to pass an explicit cwd resolved from agent config and adds a regression test for per-agent cwd isolation.

## Best Possible Solution

Land one current-main-compatible explicit-workspace fix that derives a single effective agent id and workspace from existing runtime config before create-if-missing transcript header writes, with focused regression coverage and redacted runtime proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: create a brand-new mirrored transcript while `process.cwd()` differs from the target agent workspace, and current main writes that process-global cwd into the header.

## Solution Assessment

No. The direction is right, but this patch should use the caller's runtime config and one effective agent id/workspace consistently instead of reloading config and splitting cwd/path identity.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Use runtime config for header cwd resolution:** `src/config/sessions/transcript.ts:162`
  - body: The new `loadConfig()` runs during mirrored transcript append even though the outbound caller already passes runtime config. A temporarily invalid on-disk config can make post-send bookkeeping fail and mark or retry a delivery that already succeeded, so resolve the header cwd from the existing runtime config instead.
  - confidence: 0.86
- **[P2] Use one effective agent id for cwd and session file:** `src/config/sessions/transcript.ts:163-167`
  - body: The patch defaults and trims `agentId` only for `sessionCwd`, while session store and transcript file resolution still use raw `params.agentId`. Blank or omitted ids can leave a transcript whose header workspace does not match the path/store context.
  - confidence: 0.78
