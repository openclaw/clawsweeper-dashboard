## Summary

Keep open: current main still has the transcript-header workspace bug, and this PR is aimed at the right surface, but the diff introduces a hot-path config reload and inconsistent effective-agent handling that should block merge until repaired.

## What This Changes

The PR changes mirrored session transcript creation to pass an explicit agent-workspace `cwd` into new transcript headers and adds focused transcript regression coverage.

## Best Possible Solution

Land one current-main-compatible fix that writes mirrored and gateway-created transcript headers from the resolved agent workspace, threads existing runtime config and one effective agent id consistently, adds regression coverage, and records the user-facing fix in the changelog.

## Reproduction Assessment

Yes. Source inspection on current main shows new mirrored transcript headers and gateway-created transcript headers still use `process.cwd()`, with the mirror append awaited after successful sends.

## Solution Assessment

No. The direction is right, but this patch reloads config in the mirror append path and applies the resolved/default agent id only to header cwd; the safer fix should thread already-available config and one effective agent id through all affected header writers.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Thread runtime config into mirror appends:** `src/config/sessions/transcript.ts:162`
  - body: `deliverOutboundPayloads()` and plugin send actions await transcript mirroring after sends can already have succeeded. Adding `loadConfig()` here means an invalid or unreadable config can turn bookkeeping into a failed/retryable delivery; pass the already-available runtime config/workspace context instead.
  - confidence: 0.86
- **[P2] Use one effective agent id consistently:** `src/config/sessions/transcript.ts:163-167`
  - body: The patch computes a trimmed/default agent id for the header cwd, but store path and session-file resolution still use `params.agentId`. Calls with an omitted or whitespace id can therefore mix one agent's header cwd with another store/path; compute the effective id before store/path resolution and pass it through.
  - confidence: 0.78
