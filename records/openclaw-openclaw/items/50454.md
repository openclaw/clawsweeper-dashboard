## Summary

Keep open: current main still has the transcript-header workspace bug, but this branch is not merge-ready because it adds config loading inside the mirror append path, uses inconsistent agent IDs, and lacks after-fix real behavior proof.

## What This Changes

The PR changes mirrored transcript header creation to pass an explicitly resolved agent workspace cwd and adds a regression test for per-agent cwd isolation.

## Best Possible Solution

Land one current-main-compatible fix that removes process.cwd() from transcript header create-if-missing paths, passes explicit workspace/runtime context, and covers mirrored delivery plus gateway-created transcript paths with regression tests.

## Reproduction Assessment

Yes. Source inspection against current main shows a clear path: create a brand-new mirrored transcript while process cwd differs from the target agent workspace, and ensureSessionHeader() writes the process-global cwd into the header.

## Solution Assessment

No. The direction is right, but this patch reloads config in the post-send mirror append path and applies the resolved/default agent id only to header cwd while store/session-file resolution still uses the raw parameter.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Thread runtime config into mirror appends:** `src/config/sessions/transcript.ts:162`
  - body: `appendAssistantMessageToSessionTranscript()` is awaited after sends and its callers already have runtime config. Loading config inside the transcript helper adds an avoidable config dependency to bookkeeping; pass explicit cfg/workspace context instead.
  - confidence: 0.86
- **[P2] Use one effective agent id consistently:** `src/config/sessions/transcript.ts:163-167`
  - body: The patch computes a default/trimmed id only for the header cwd while store path and session-file resolution still use `params.agentId`. Omitted or blank ids can mix one agent's header cwd with another store/path; compute the effective id before store/path resolution and pass it through.
  - confidence: 0.78
