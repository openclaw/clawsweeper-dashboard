## Summary

Keep this PR open: current main and the latest release still write mirrored transcript headers from ambient `process.cwd()`, but this branch is not merge-ready because it is conflicting, lacks real behavior proof, and introduces split config/agent identity risks.

## What This Changes

This PR changes mirrored transcript header creation to pass an explicit agent workspace cwd instead of reading `process.cwd()`, and adds a regression test for per-agent cwd isolation.

## Best Possible Solution

Land one current-main-compatible fix that uses caller runtime config plus one normalized effective agent id/workspace for transcript file/store/header resolution, with focused regression coverage and redacted runtime proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Use caller config for header cwd resolution:** `src/config/sessions/transcript.ts:165`
  - body: The new `loadConfig()` runs during transcript append even though delivery paths already pass runtime config. If on-disk config is invalid or differs after delivery succeeds, mirroring can fail post-send or write a header for the wrong workspace; resolve cwd from the caller config or a threaded workspace instead.
  - confidence: 0.89
- **[P2] Use one effective agent id for file and header:** `src/config/sessions/transcript.ts:166-170`
  - body: The patch trims/defaults `agentId` only for header cwd, while session file persistence still receives raw `params.agentId`. For absent or whitespace ids, the header can describe the default workspace while the transcript file/store are resolved in a different context; normalize once and pass the same id through both paths.
  - confidence: 0.78
