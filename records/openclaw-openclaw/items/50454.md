## Summary

Keep open: current main still has the transcript header cwd bug and this PR targets a real issue, but the submitted patch is not merge-ready because it reloads config in the post-send mirroring path, uses inconsistent agent identity for cwd versus path resolution, lacks real behavior proof, and overlaps with https://github.com/openclaw/openclaw/pull/56133.

## What This Changes

This PR changes mirrored session transcript creation to write a resolved agent workspace cwd instead of `process.cwd()` and adds a focused regression test.

## Best Possible Solution

Land one current-main-compatible explicit-workspace fix that derives one effective agent id and workspace from existing runtime config for every create-if-missing transcript header, with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows a high-confidence path: create a brand-new mirrored transcript while the process cwd differs from the target agent workspace, and `ensureSessionHeader()` writes the process-global cwd into the header.

## Solution Assessment

No. The direction is right, but this patch reloads config after delivery and uses different agent identity inputs for cwd and path resolution; the safer fix is to use existing config and one effective agent id consistently.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Pass existing config into mirrored transcript appends:** `src/config/sessions/transcript.ts:162`
  - body: The added `loadConfig()` runs inside post-send transcript mirroring. If on-disk config is temporarily invalid or unreadable, a successful delivery can be reported as failed and retried because bookkeeping now throws; use the caller's existing runtime config instead.
  - confidence: 0.86
- **[P2] Use one effective agent id consistently:** `src/config/sessions/transcript.ts:163-167`
  - body: The patch derives a trimmed/default `agentId` for `sessionCwd`, but session store and file resolution still use `params.agentId`. Blank or omitted ids can create a transcript whose header cwd points at one agent while the transcript path is resolved under another/raw agent context.
  - confidence: 0.78
