## Summary

Keep open. The workspace-isolation bug is still present on current main and the latest release, but this PR is not merge-ready: the patch adds a hot-path config load inside transcript mirroring, which can turn bookkeeping into a delivery failure after a send has already succeeded. The work also overlaps the open #49523/#56133 transcript-cwd cluster, so it needs maintainer consolidation rather than auto-close.

## What This Changes

This PR changes mirrored transcript header creation to pass an explicitly resolved agent workspace as `cwd` and adds a regression test for per-agent cwd isolation.

## Best Possible Solution

Land one current-main-compatible fix that removes `process.cwd()` from mirrored transcript header creation, uses already-available runtime config or explicitly threaded workspace context instead of rereading config in the append helper, preserves current lazy import boundaries, and coordinates with the gateway transcript-cwd fix tracked by #56133.

## Reproduction Assessment

Yes. On current main, creating a missing mirrored transcript through `appendAssistantMessageToSessionTranscript()` while the process cwd differs from the target agent workspace will stamp the process cwd in the first JSONL header line.

## Solution Assessment

No. The requested direction is right, but this patch is not the best implementation because it reloads config inside a post-delivery mirror append path and resolves an effective agent id only for `cwd`, not consistently for session/store resolution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Avoid reloading config while mirroring transcripts:** `src/config/sessions/transcript.ts:162`
  - body: `deliverOutboundPayloads()` awaits this mirror append after channel sends have succeeded. The new `loadConfig()` call can still parse and throw when no runtime snapshot is pinned or after snapshot reset; that exception bypasses the existing result-style error handling and can make a successful send look failed or retryable. Thread the already-resolved config/workspace into this helper or use a non-throwing best-effort path.
  - confidence: 0.86
- **[P2] Use the resolved agent id for session paths:** `src/config/sessions/transcript.ts:163-169`
  - body: The patch derives `agentId` for `sessionCwd`, but store/session file resolution still uses `params.agentId`. For supported calls that omit or pass whitespace `agentId`, the header can be derived from the configured default agent while the transcript path stays under the legacy/default session store. Compute the effective agent id before resolving the store and pass it consistently.
  - confidence: 0.78
