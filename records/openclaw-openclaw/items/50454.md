## Summary

Keep open. The reported workspace-isolation bug is still source-reproducible on current main, but this PR is conflicting and has blocking review issues: it reloads config in post-send transcript bookkeeping, uses a different effective agent id for the header cwd than for session path resolution, and lacks external after-fix real behavior proof.

## What This Changes

This PR changes transcript header creation to accept an explicit cwd, resolves that cwd from agent config inside mirrored transcript appends, and adds a mocked regression test.

## Best Possible Solution

Land one current-main-compatible fix that derives one effective agent id and workspace from existing runtime config for every create-if-missing transcript header path, with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows new mirrored transcript headers still read `process.cwd()` while the relevant outbound paths already carry agent/config context.

## Solution Assessment

No. The direction is right, but the submitted patch reloads config after send and uses different agent identity inputs for cwd and path resolution; the safer fix is to thread existing config and one effective agent id into header creation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Pass existing config into mirrored transcript appends:** `src/config/sessions/transcript.ts:162`
  - body: The new `loadConfig()` runs inside post-send transcript mirroring. If the on-disk config is temporarily invalid or unreadable, a successful delivery can be reported as failed and retried because bookkeeping now throws; use the caller's existing runtime config instead.
  - confidence: 0.86
- **[P2] Use one effective agent id consistently:** `src/config/sessions/transcript.ts:163-167`
  - body: The patch derives a trimmed/default `agentId` for `sessionCwd`, but session store and file resolution still use `params.agentId`. Blank or omitted ids can create a transcript whose header cwd points at one agent while the transcript path is resolved under another/raw agent context.
  - confidence: 0.78
