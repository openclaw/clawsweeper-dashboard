## Summary

Keep open. The PR targets a real workspace-isolation bug, but it still has blocking review findings, lacks contributor after-fix real behavior proof, and overlaps the open current-main fix candidate at https://github.com/openclaw/openclaw/pull/56133 plus the still-open linked bug report.

## What This Changes

This PR changes mirrored transcript header creation to resolve an agent workspace cwd instead of reading `process.cwd()` and adds a focused transcript regression test.

## Best Possible Solution

Land one current-main-compatible fix that removes `process.cwd()` from transcript create-if-missing header paths, derives cwd from explicit runtime config plus one effective agent id, and covers mirrored delivery, sessions.create, and chat-created transcript paths with regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: create a brand-new mirrored transcript while process cwd differs from the target agent workspace, and current main writes the process-global cwd into the header.

## Solution Assessment

No. The direction is right, but this patch reloads config inside post-send bookkeeping and uses the resolved/default agent id only for header cwd while store/session-file resolution still uses the raw parameter.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Pass existing config into mirrored transcript appends:** `src/config/sessions/transcript.ts:162`
  - body: `appendAssistantMessageToSessionTranscript()` is awaited after delivery has already sent the message. Loading config here means a temporarily invalid or unreadable config can turn mirror bookkeeping into a delivery failure and possible retry, even though callers already carry runtime config.
  - confidence: 0.86
- **[P2] Use one effective agent id consistently:** `src/config/sessions/transcript.ts:163-167`
  - body: The patch resolves a trimmed/default agent id only for `sessionCwd`, but store path and session-file resolution still receive `params.agentId`. Omitted or blank ids can mix one agent's header cwd with a different store/path; compute one effective id before resolving both.
  - confidence: 0.78
