## Summary

Keep open. The workspace-isolation bug is still source-reproducible on current main, but this external PR should not merge as-is because it adds config loading in the post-send mirror path, uses different effective agent ids for related decisions, overlaps a broader open fix candidate, and lacks after-fix real behavior proof.

## What This Changes

The PR changes mirrored session transcript creation to pass a resolved agent workspace `cwd` into new transcript headers and adds a focused regression test.

## Best Possible Solution

Land one current-main-compatible explicit-workspace fix for all transcript create-if-missing paths, using already-available runtime config and one effective agent id consistently with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: create a brand-new mirrored transcript while the process cwd differs from the target agent workspace, and `ensureSessionHeader()` writes the process-global cwd into the header.

## Solution Assessment

No. The direction is right, but this patch reloads config inside the post-send mirror append path and derives a default/trimmed agent id only for the header cwd; the safer fix should thread existing runtime config or resolved workspace context and use one effective agent id for store, path, and header decisions.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Thread runtime config into mirror appends:** `src/config/sessions/transcript.ts:162`
  - body: `deliverOutboundPayloads()` and plugin send actions await transcript mirroring after sends can already have succeeded. Adding `loadConfig()` here lets config loading turn bookkeeping into a failed or retryable delivery; pass the already-available runtime config or resolved workspace context instead.
  - confidence: 0.86
- **[P2] Use one effective agent id consistently:** `src/config/sessions/transcript.ts:163-167`
  - body: The patch computes a trimmed/default agent id for the header cwd, but store path and session-file resolution still use `params.agentId`. Calls with an omitted or whitespace id can therefore mix one agent's header cwd with another store/path; derive the effective id before store/path resolution and pass it through.
  - confidence: 0.78
