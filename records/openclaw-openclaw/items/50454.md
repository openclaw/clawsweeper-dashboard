## What This Changes

The PR changes transcript mirroring so new session headers use a resolved agent workspace instead of `process.cwd()`, and adds a regression test for that behavior.

## Best Possible Solution

Rebase or replace this PR with a current-main patch that passes an explicit target cwd from the existing runtime config/session owner into transcript header creation without a fresh post-send config reload, then add real behavior proof.

## Reproduction Assessment

Yes. By source inspection, a stored session entry without `spawnedCwd` can still reach header creation without an explicit cwd, and this PR's new fallback can choose the default agent instead of the session owner.

## Solution Assessment

No. The direction is right, but this branch is not the best fix because it reloads config inside the mirror append path and does not derive the workspace from the owning session when `agentId` is absent.
