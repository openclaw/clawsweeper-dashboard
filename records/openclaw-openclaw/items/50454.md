## Summary

Keep open. The target transcript-header cwd bug is still present on current main, but this PR is not ready to merge because its fix reloads config inside the post-delivery mirror append path and computes an effective agent id only for the header cwd, not for the session store/path resolution. The related #49523 bug and overlapping #56133 gateway fix should stay linked while one current-main-compatible patch is prepared.

## What This Changes

The PR changes mirrored session transcript header creation to use a resolved agent workspace cwd instead of `process.cwd()` and adds a regression test for cwd isolation.

## Best Possible Solution

Land one current-main-compatible fix that removes `process.cwd()` from new transcript headers for both mirrored append and the gateway-created transcript path, threads already-available config/workspace context instead of reloading config inside delivery bookkeeping, and uses one effective agent id consistently for store, session file, and header cwd resolution.

## Reproduction Assessment

Yes. On current main, create a missing mirrored transcript via `appendAssistantMessageToSessionTranscript()` while the process cwd differs from the target agent workspace; the first JSONL header line is written from `process.cwd()` at `src/config/sessions/transcript.ts:41`.

## Solution Assessment

No. The requested direction is right, but this patch is not the best implementation because it reloads config inside a post-delivery append helper and applies the resolved agent id only to the header cwd, leaving session path resolution on the raw parameter.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Avoid reloading config while mirroring transcripts:** `src/config/sessions/transcript.ts:165`
  - body: `deliverOutboundPayloads()` awaits this mirror append after channel sends have succeeded. The new `loadConfig()` call can still parse and throw when no runtime snapshot is pinned or after a config reset, which turns a bookkeeping-only mirror failure into a failed or retryable delivery even though the user already received the message. Thread already-resolved config/workspace context into this helper, or keep this path best-effort.
  - confidence: 0.86
- **[P2] Use the resolved agent id for session paths:** `src/config/sessions/transcript.ts:166-180`
  - body: The patch derives a trimmed/default `agentId` for `sessionCwd`, but store and session file resolution still use `params.agentId`. For calls that omit or pass whitespace `agentId`, the header can be derived from the configured default agent while the transcript path/store resolution uses a different default path. Compute the effective agent id before resolving the store and pass it consistently.
  - confidence: 0.78
