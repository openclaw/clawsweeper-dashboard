## Summary

Keep open: current main still has the mirrored transcript cwd bug, but this PR is not mergeable as-is because it reloads config during post-send mirroring, uses different effective agent ids for cwd versus session-file resolution, lacks contributor real behavior proof, and overlaps with https://github.com/openclaw/openclaw/pull/56133 plus https://github.com/openclaw/openclaw/issues/49523.

## What This Changes

The PR changes mirrored transcript header creation to use a resolved agent workspace cwd and adds a regression test for per-agent cwd isolation.

## Best Possible Solution

Land one current-main-compatible fix that derives a single effective agent id and resolved workspace from existing runtime config for every create-if-missing transcript header, with regression coverage for mirrored delivery and adjacent gateway-created paths.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: create a brand-new mirrored transcript while process cwd differs from the target agent workspace, and `ensureSessionHeader()` writes the process-global cwd into the header.

## Solution Assessment

No. The direction is right, but reloading config inside post-send transcript mirroring and resolving cwd with a different agent id from session-file resolution is not the narrowest maintainable fix; thread existing config and one effective agent id instead.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Pass existing config into mirrored transcript appends:** `src/config/sessions/transcript.ts:162`
  - body: The new `loadConfig()` runs after delivery has already succeeded. If the config file is temporarily invalid or unreadable, mirror bookkeeping can make the send look failed and trigger retry even though the recipient got the message; thread the caller's existing runtime config instead.
  - confidence: 0.86
- **[P2] Use one effective agent id consistently:** `src/config/sessions/transcript.ts:163-167`
  - body: The patch derives a trimmed/default `agentId` for `sessionCwd`, but session store and file resolution still use `params.agentId`. Blank or omitted ids can create a transcript whose header cwd points at one agent while the session file path is resolved under another/raw agent context.
  - confidence: 0.78
