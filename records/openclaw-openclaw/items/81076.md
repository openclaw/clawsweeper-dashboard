## Summary

Keep open for normal PR review and CI. The patch targets a real source-reproducible browser plugin bug, follows the schema compatibility boundary, and I found no blocking correctness issue in the diff.

## What This Changes

The PR backfills allowlisted top-level browser act fields into nested `request` objects when those nested fields are missing, and adds regression coverage for backfill and nested-field precedence.

## Best Possible Solution

Land the narrow parser-level compatibility fix in the browser plugin, preserving nested request precedence and covering the mixed-payload regression in the browser tool tests.

## Reproduction Assessment

Yes. Source inspection on current main shows that `readActRequestParam` returns nested `request` before copying allowlisted top-level fields, so the mixed payload reaches `/act` without `ref` or `selector`.

## Solution Assessment

Yes. Normalizing at the browser tool parser boundary is the narrowest maintainable fix because it honors the advertised tool schema while leaving the `/act` service contract and browser runtime behavior unchanged.
