## Summary

Keep open. Current main and v2026.5.22 have fixed built-in allowlists for same-origin microphone, OpenAI realtime, and tweakcn, but they still do not expose the requested operator-configurable CSP or Permissions-Policy surface for custom Control UI roots and local services; the related open PR at https://github.com/openclaw/openclaw/pull/66783 only covers part of the Permissions-Policy side.

## What This Changes

The issue requests configurable Control UI CSP and Permissions-Policy headers for custom Control UI roots and local browser integrations.

## Best Possible Solution

Add a maintainer-approved, typed Control UI header-policy config that preserves hardened defaults and validates allowed CSP directives, origins, and Permissions-Policy directives with schema, docs, and focused gateway tests.

## Reproduction Assessment

Yes. Source inspection on current main and v2026.5.22 shows the CSP and Permissions-Policy values are fixed, while the strict config contract has no accepted keys for CSP origins or policy overrides.

## Solution Assessment

Unclear as proposed. The use case is valid, but raw CSP or Permissions-Policy overrides would be too broad without a maintainer-approved typed, allowlisted API.
