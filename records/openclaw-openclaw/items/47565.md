## Summary

Keep open. Current main has fixed allowlists for the built-in OpenAI/tweakcn browser fetches and same-origin microphone access, but it still has no supported config surface for custom Control UI CSP or Permissions-Policy changes; the remaining request is a security-sensitive feature/API decision.

## What This Changes

The issue requests configurable Control UI Permissions-Policy and CSP settings so custom Control UI roots can allow microphone access and local fetch origins without patching generated gateway files.

## Best Possible Solution

Keep hardened defaults; if maintainers accept the use case, add a constrained typed Control UI header-policy config with validation, schema/help/docs updates, and focused gateway/header tests.

## Reproduction Assessment

Yes, for the feature gap. Source inspection on current main shows fixed CSP and Permissions-Policy headers plus strict config schemas with no requested keys, so no live browser run is needed to prove the missing config surface.

## Solution Assessment

Unclear as proposed. The use case is reasonable, but raw cspOverrides or free-form Permissions-Policy values are not the safest maintainable API; constrained typed config or same-origin gateway/plugin routing would be safer.
