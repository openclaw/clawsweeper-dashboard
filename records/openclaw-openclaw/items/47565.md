## Summary

Keep open. Current main and v2026.6.6 partially address the microphone example and known bundled origins, but Control UI CSP, frame policy, and Permissions-Policy remain fixed and no supported config surface exists for custom Control UI roots to add local connect-src targets or header policy overrides.

## What This Changes

Requests new gateway config for Control UI Permissions-Policy and CSP overrides so custom Control UI roots can use browser permissions and local fetch targets without patching dist files.

## Best Possible Solution

Add a maintainer-approved, narrow, typed Control UI header policy config with safe defaults, strict validation, schema/help/docs updates, and focused tests for serialization, rejected unsafe values, and HTTP header behavior.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a new configuration capability. Source inspection gives high confidence that current main still emits fixed Control UI CSP/Permissions-Policy headers and rejects the proposed config fields.

## Solution Assessment

Unclear until maintainers decide the header-policy contract. A narrow typed policy surface is safer than a raw arbitrary header override because it can preserve hardened defaults and validate risky directives.
