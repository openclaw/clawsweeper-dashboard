## Summary

Keep open. Current main has fixed-origin allowances for same-origin microphone, OpenAI realtime, and tweakcn, but the requested configurable Control UI CSP and Permissions-Policy config surface is still absent and remains a security-sensitive API decision.

## What This Changes

The issue requests configurable Control UI Permissions-Policy and CSP header settings for custom Control UI deployments.

## Best Possible Solution

Add a maintainer-approved, typed Control UI header-policy config with hardened defaults, strict directive/origin validation, config metadata and docs, and focused CSP/Permissions-Policy serialization plus HTTP header tests.

## Reproduction Assessment

Yes, for the feature gap. Current source shows the proposed config keys are absent from the public type and strict schema while the gateway still emits fixed CSP and Permissions-Policy values.

## Solution Assessment

Unclear as proposed. The capability is reasonable for custom Control UI roots, but raw override strings are not the safest maintainable shape; a constrained typed schema with allowlisted directives and origins is safer.
