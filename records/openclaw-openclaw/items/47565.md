## Summary

Keep open. Current main and v2026.5.18 still have fixed Control UI CSP and Permissions-Policy header surfaces; the same-origin microphone and specific origin allowlist fixes help, but they do not provide a supported config path for custom local `connect-src` origins or policy overrides.

## What This Changes

The issue requests configurable Control UI Permissions-Policy and CSP header overrides for custom Control UI roots.

## Best Possible Solution

Keep hardened defaults, then add a maintainer-approved typed Control UI header-policy config with strict directive/origin validation, schema/help/docs coverage, and focused gateway header tests.

## Reproduction Assessment

Yes, for the missing configuration surface. Source inspection shows fixed CSP and Permissions-Policy headers plus strict config schemas with no `permissionsPolicy`, `cspOverrides`, or configurable `connect-src` key.

## Solution Assessment

Unclear as proposed. The use case is concrete, but free-form CSP overrides or raw Permissions-Policy strings are not obviously the safest public API; a constrained typed policy or approved gateway route would be safer.
