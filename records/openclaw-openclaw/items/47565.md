## Summary

Keep open. Current main and v2026.6.8 still hardcode the Control UI CSP, frame policy, and Permissions-Policy; the same-origin microphone case is fixed, but custom Control UI roots still cannot add local connect-src targets or choose a supported header policy.

## What This Changes

The issue requests configurable Control UI Permissions-Policy and CSP directives so custom Control UI roots can use microphone access and local HTTP services without patching installed dist files.

## Best Possible Solution

Add a maintainer-approved, typed Control UI header policy config with safe defaults, strict validation, schema/help/docs updates, and focused tests for config rejection and served headers.

## Reproduction Assessment

Yes for the current limitation at source level: current main and v2026.6.8 hardcode the relevant headers and the strict config schema has no requested policy fields. This is not a broken-existing-contract bug reproduction; it is a missing configuration capability.

## Solution Assessment

Unclear until maintainers decide the security contract. A narrow typed policy surface is safer than a broad raw header override because it can preserve hardened defaults and reject unsafe directives.
