## Summary

Keep open. Current main and the latest release still have fixed Control UI CSP and fixed default Permissions-Policy behavior, while the open related Permissions-Policy PR is conflicting and only covers part of the requested CSP/Permissions-Policy configurability; the remaining work needs maintainer product and security-header policy review.

## What This Changes

The issue requests configurable Control UI Permissions-Policy and CSP headers so custom Control UI roots can use browser permissions and local service origins without patching installed bundles.

## Best Possible Solution

Add a maintainer-approved, narrow, typed gateway security-header policy surface with strict validation, schema/help/docs updates, and tests for CSP serialization, Permissions-Policy serialization, config rejection, and HTTP header behavior.

## Reproduction Assessment

Yes. Source inspection is enough to reproduce the missing capability: current main hardcodes the Control UI CSP and default Permissions-Policy and exposes no config field for the requested local origin or directive override.

## Solution Assessment

No, not exactly as proposed. A broad raw override surface is not yet proven to be the safest fix; the better path is a narrow typed config/API contract approved by maintainers.
