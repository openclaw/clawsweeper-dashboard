## Summary

Keep open. Current main and the latest release still hard-code the Control UI CSP and gateway Permissions-Policy surfaces, with no supported config key for custom local `connect-src` origins or policy overrides; the related open PR only covers Permissions-Policy and still leaves the CSP half of this request unresolved.

## What This Changes

The issue requests configurable Control UI Permissions-Policy and CSP headers so custom `gateway.controlUi.root` deployments can allow microphone access and local inference-server fetches without patching built files.

## Best Possible Solution

Keep hardened defaults, then add a maintainer-approved typed Control UI header-policy config with strict directive/origin validation, public docs, schema metadata, and focused gateway header tests.

## Reproduction Assessment

Yes, for the missing configuration surface. Source inspection shows fixed CSP and Permissions-Policy headers plus strict config schemas with no `permissionsPolicy`, `cspOverrides`, or configurable `connect-src` key.

## Solution Assessment

Unclear as proposed. The custom-Control-UI use case is concrete, but free-form CSP overrides or raw Permissions-Policy strings are not obviously the safest public API; a constrained typed policy or approved gateway route would be safer.
