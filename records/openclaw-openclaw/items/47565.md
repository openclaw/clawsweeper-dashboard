## Summary

Keep open. Current main partially covers the microphone example and OpenAI realtime calls, but it still has no configurable Control UI CSP or Permissions-Policy surface; adding one is a security-sensitive config/API decision.

## What This Changes

Request configurable `gateway.controlUi` CSP and Permissions-Policy settings so custom Control UI roots can use microphone access and local `connect-src` origins without patching installed files.

## Best Possible Solution

Add a maintainer-approved, typed Control UI header-policy config with hardened defaults, constrained directive/origin validation, generated config metadata/help/docs, and focused HTTP/header serialization tests.

## Reproduction Assessment

Yes, for the reported feature gap. Source inspection on current main shows fixed CSP and Permissions-Policy generation plus a strict `gateway.controlUi` schema that lacks the proposed policy keys.

## Solution Assessment

Unclear as proposed. The capability is reasonable for custom Control UI roots, but raw override strings are not the safest maintainable shape; a constrained typed schema is the better direction.
