## Summary

Keep open. Current main partially covers the examples by allowing same-origin microphone access and OpenAI realtime connections, but the requested configurable Control UI CSP and Permissions-Policy config surface is still absent and needs maintainer security/API review.

## What This Changes

The issue requests configurable `gateway.controlUi` options for Control UI CSP and Permissions-Policy headers so custom Control UI roots can allow specific browser permissions and local fetch origins.

## Best Possible Solution

Add a maintainer-approved, typed Control UI header-policy config with hardened defaults, directive/origin validation, generated schema metadata/help/docs, and focused CSP, Permissions-Policy, config, and HTTP header tests.

## Reproduction Assessment

Yes, for the reported feature gap. Source inspection on current main shows the proposed config keys are absent from the public type and strict schema while the gateway still emits fixed CSP and Permissions-Policy values.

## Solution Assessment

Unclear as proposed. The capability is reasonable for custom Control UI roots, but raw override strings would be riskier than a constrained typed schema with allowlisted directives and validated origins.
