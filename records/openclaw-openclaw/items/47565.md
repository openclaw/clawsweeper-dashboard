## Summary

Keep open. Current main partially covers the microphone example by allowing same-origin microphone access, but the requested configurable Control UI CSP and Permissions-Policy surface is still absent and needs maintainer security/API-shape review before implementation.

## What This Changes

Request configurable `gateway.controlUi` options for Control UI Permissions-Policy directives and CSP overrides so custom Control UI roots can use browser permissions and local service connections without patching installed bundles.

## Best Possible Solution

Add a maintainer-approved, typed Control UI header-policy config in the gateway owner area, keep hardened defaults, allow only narrowly validated directives/origins, update generated config metadata/help/docs, and add focused tests for serialization, rejection, and HTTP header behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: the proposed `gateway.controlUi` keys are absent from the type and strict schema, while current header helpers still emit fixed CSP and Permissions-Policy values.

## Solution Assessment

Unclear as proposed. The capability is reasonable for custom Control UI roots, but raw `cspOverrides` or arbitrary policy strings would be riskier than a constrained typed schema with validation and allowlisted directives.
