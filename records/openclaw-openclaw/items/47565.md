## Summary

Keep open. Current main has added specific allowances for same-origin microphone and OpenAI realtime calls, but the requested configurable Control UI CSP and Permissions-Policy surface is still absent and remains a security-sensitive config/API decision.

## What This Changes

The issue requests configurable Control UI `Permissions-Policy` and CSP header policy under `gateway.controlUi` so custom Control UI roots can allow browser permissions and local fetch origins without patching installed bundles.

## Best Possible Solution

Add a maintainer-approved, narrow, typed Control UI header-policy config with hardened defaults, validated directives/origins, generated config metadata/help/docs, and focused CSP/Permissions-Policy serialization plus HTTP header tests.

## Reproduction Assessment

Yes, for the reported feature gap. Source inspection on current main shows the proposed config keys are absent from the public type and strict schema while the gateway still emits fixed CSP and Permissions-Policy values.

## Solution Assessment

Unclear as proposed. The capability is reasonable for custom Control UI roots, but raw override strings are not the best maintainable shape; a constrained typed schema with allowlisted directives/origins would be safer.
