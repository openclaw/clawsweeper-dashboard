## Best Possible Solution

Add a maintainer-approved, typed Control UI header-policy config that preserves hardened defaults and validates allowed CSP directives, origins, and Permissions-Policy directives with schema, docs, and focused gateway tests.

## Reproduction Assessment

## Solution Assessment

Unclear as proposed. The use case is valid, but raw CSP or Permissions-Policy overrides would be too broad without a maintainer-approved typed, allowlisted API.
