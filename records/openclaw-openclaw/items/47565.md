## Summary

Keep open. Current main only partially covers the microphone example by allowing same-origin microphone access; it still has fixed Control UI CSP and Permissions-Policy headers and no supported config surface for adding custom `connect-src` origins or policy directives. Because this would broaden browser security policy, it needs maintainer API-shape and validation review before implementation.

## What This Changes

The issue requests configurable `gateway.controlUi` options for Control UI Permissions-Policy directives and CSP overrides, especially custom microphone access and extra local `connect-src` origins for custom Control UI roots.

## Best Possible Solution

Add a narrow, typed Control UI header-policy config in the gateway owner area, with allowlisted CSP directives, strict origin validation, generated config metadata/help/docs, and focused tests for serialization, config rejection, and HTTP header behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: the proposed `gateway.controlUi` keys are absent from the exported type and strict schema, while current header helpers still emit fixed CSP and Permissions-Policy values.

## Solution Assessment

Unclear as proposed. The capability is reasonable for custom Control UI roots, but raw `cspOverrides` or arbitrary policy strings would not be the safest path; a constrained typed schema with validation is the maintainable solution.
