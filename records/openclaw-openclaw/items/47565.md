## Summary

Keep open. Current main still emits fixed Control UI CSP and Permissions-Policy headers and has no public config surface for the requested overrides, so this remains valid but needs maintainer security/API review.

## What This Changes

Request configurable `gateway.controlUi` Permissions-Policy and CSP settings so custom Control UI roots can use browser permissions and local service origins without patching installed bundles.

## Best Possible Solution

Add a maintainer-approved, narrow, typed Control UI header-policy config with hardened defaults, validated directives/origins, generated config metadata/help/docs, and focused header serialization tests.

## Reproduction Assessment

Yes, for the reported feature gap. Source inspection shows the proposed config keys are absent from the public type and strict schema while the gateway still emits fixed CSP and Permissions-Policy values.

## Solution Assessment

Unclear as proposed. The capability is reasonable for custom Control UI roots, but raw override strings are not the safest shape; a constrained typed schema with allowlisted directives and origins would be more maintainable.
