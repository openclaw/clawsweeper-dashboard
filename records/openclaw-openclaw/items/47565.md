## Summary

Keep open. Current main still has fixed Control UI CSP and Permissions-Policy headers plus a strict config schema with no supported override surface; the remaining work is valid but security-sensitive and needs maintainer API-shape approval.

## What This Changes

Add `gateway.controlUi` configuration for Control UI CSP and Permissions-Policy so custom Control UI roots can allow selected browser permissions and outbound origins.

## Best Possible Solution

Add a maintainer-approved, typed Control UI header-policy config with hardened defaults, strict directive/origin validation, generated schema/help/docs updates, and focused CSP/Permissions-Policy HTTP tests.

## Reproduction Assessment

Yes, for the reported feature gap. Source inspection on current main shows the requested config keys are absent from the public type and strict schema while the gateway still emits fixed CSP and Permissions-Policy values.

## Solution Assessment

Unclear as proposed. The capability is reasonable for custom Control UI roots, but raw override strings are not the best maintainable shape; a constrained typed schema with allowlisted directives and origins would be safer.
