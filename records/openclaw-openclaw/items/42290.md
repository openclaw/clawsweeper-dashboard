## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Replace the OOM hint with an available command:** `src/commands/backup.ts:369`
  - body: The new OOM path tells users to retry with `--exclude-logs` or `--exclude-media`, but this branch does not register either flag for `backup create`. In the failure path this PR adds, the suggested recovery command would be rejected by the CLI, so the hint should use an existing option such as `--no-include-workspace` or avoid naming unavailable flags.
  - confidence: 0.94
- **[P2] Destroy the tar stream on destination errors:** `src/commands/backup.ts:355-364`
  - body: When the destination write stream errors, `fail()` only destroys `destStream`; the tar pack can keep walking and compressing input until it later notices the broken pipe. For the large backups this PR targets, use `pipeline()` or explicitly destroy the tar stream as well so the source side stops immediately on write failure.
  - confidence: 0.82
