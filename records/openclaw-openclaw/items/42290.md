## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Do not suggest flags the command does not register:** `src/commands/backup.ts:366-368`
  - body: The new ENOMEM hint tells users to retry with `--exclude-logs` or `--exclude-media`, but `backup create` does not register those options in the CLI. In the exact recovery path this patch adds, users would be directed to commands that fail option parsing.
  - confidence: 0.91
- **[P2] Stop the tar stream when the destination fails:** `src/commands/backup.ts:358`
  - body: On a destination stream error, `fail()` destroys only `destStream`; the tar pack can keep walking and compressing the source tree until it later sees a pipe error. Use `stream/promises.pipeline()` or explicitly destroy the tar stream so a failed write target stops the large backup work immediately.
  - confidence: 0.82
