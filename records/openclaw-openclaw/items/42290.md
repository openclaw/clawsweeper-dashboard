  - confidence: 0.91
- **[P2] Stop the tar stream when the destination fails:** `src/commands/backup.ts:358`
  - body: `fail()` destroys only the destination stream. If the destination errors, the tar pack can keep walking and compressing a large backup until it later sees a broken pipe, preserving the resource-burn failure mode this PR is meant to avoid. Use `pipeline()` or explicitly destroy the tar stream if this branch is revived.
  - confidence: 0.82
