## Summary

Keep open. Current main has a config-version guard for service-managed downgrades, but it does not implement the requested data-layout marker or a generic Docker/plain gateway startup halt, and the exact corruption claim lacks runtime proof.

## What This Changes

The issue requests a boot-time data-layout/version gate to prevent older OpenClaw Docker/runtime versions from writing a legacy root data layout after a newer workspace subdirectory layout has been used.

## Best Possible Solution

Define a forward-compatible state/layout version contract in the state root, document downgrade behavior, and apply any fail-closed startup check consistently across managed service, Docker Compose, and plain gateway paths if maintainers want to support it.

## Reproduction Assessment

No for the exact reported corruption: there are no logs or file listings, and the cited `main.sqlite` path does not match the current source I found. Source inspection does show the requested Docker/plain gateway layout gate is absent while a service-mode config guard exists.

## Solution Assessment

Unclear. A `.openclaw_version` marker or symlink compatibility layer might be viable, but the best solution depends on maintainer policy for downgrade support and cannot retroactively protect already-shipped older binaries.
