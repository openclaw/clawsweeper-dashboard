## Summary

Keep open for maintainer review. Current main already has a newer-config guard for service operations, but the Docker/foreground gateway path described here is still not clearly covered, and changing downgrade startup behavior needs an explicit compatibility policy.

## What This Changes

The issue asks OpenClaw to detect a newer data or workspace layout during downgrade and halt startup, or safely route paths, instead of allowing split state.

## Best Possible Solution

Define the supported downgrade contract, then either extend the existing meta.lastTouchedVersion guard to Docker/foreground gateway startup or document the recovery-only exception with Docker-focused proof.

## Reproduction Assessment

No, not for the full 2026.5.7 Docker downgrade and SQLite corruption claim. Source inspection does show a current-main gap where the hard future-config startup exit requires OPENCLAW_SERVICE_MARKER, while Docker starts the foreground gateway path.

## Solution Assessment

Unclear. A new .openclaw_version marker or symlink routing may not be the narrowest fix because current OpenClaw already has meta.lastTouchedVersion guarding; maintainers should first decide whether that guard should also block foreground Docker startup.
