## Summary

Keep this open for maintainer policy review: current main has a newer-config guard for service-mode gateway operations, but the Docker/foreground startup path is not clearly covered and the exact reported split-state downgrade was not reproduced.

## What This Changes

The issue requests startup downgrade gating or layout-version routing so older OpenClaw runtimes do not run against data written by a newer workspace/state layout.

## Best Possible Solution

Define the supported downgrade and recovery contract, then either extend the existing meta.lastTouchedVersion guard to the Docker/foreground gateway path or add a layout marker with targeted Docker tests and troubleshooting docs.

## Reproduction Assessment

No, not for the full report: I did not reproduce a real 2026.5.7 Docker rollback with split SQLite state. Source inspection does show a credible current-main gap where the hard startup halt only applies when OPENCLAW_SERVICE_MARKER is set.

## Solution Assessment

Unclear; the requested .openclaw_version marker is not obviously the narrowest fix because OpenClaw already stamps meta.lastTouchedVersion and guards managed service mutations. The safer direction is a maintainer decision on whether the existing guard should also block foreground/Docker startup, then add focused proof.
