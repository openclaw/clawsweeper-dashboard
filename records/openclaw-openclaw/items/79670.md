## Summary

Keep open: current main and the latest release still make Google API-key onboarding pick `google/gemini-3.1-pro-preview`, and the docs/tests still present preview Gemini defaults without quota-tier warning. The linked local fix commits that tried to switch to `gemini-2.5-flash` are not contained in any local main/release branch or tag, so they do not close the issue.

## What This Changes

The issue requests that Google onboarding stop recommending Gemini preview models without quota-tier validation or warnings, preferably by defaulting paid AI Studio users to stable Gemini models.

## Best Possible Solution

Make Google API-key onboarding and docs avoid an unqualified preview-model default for fresh setup: either default to a stable model such as `google/gemini-2.5-flash` or add quota-aware warning/copy around preview choices while preserving explicit existing user selections.

## Reproduction Assessment

Yes for the OpenClaw recommendation path: current source, docs, and tests all still point Google API-key onboarding at `google/gemini-3.1-pro-preview`. I did not live-reproduce a Tier 1 quota exhaustion scenario because that requires a real paid Google AI Studio project and workload.

## Solution Assessment

Unclear without maintainer product input: the narrow stable-default patch is plausible, but default-model selection and quota-warning UX are compatibility-sensitive. The safest fix should include docs/tests and avoid overwriting explicit existing model choices unexpectedly.
