## Summary

Keep open. Current main still applies and documents `google/gemini-3.1-pro-preview` for Google API-key onboarding, and recent history shows a stable-default fix was deliberately reverted, so this is an unresolved provider default/quota policy decision rather than cleanup closure material.

## What This Changes

The issue asks OpenClaw to avoid recommending Gemini preview models during Google API-key onboarding without quota awareness, either by warning users or defaulting to stable models.

## Best Possible Solution

Make an explicit provider-owned Google onboarding policy: choose a stable production-safe API-key default or keep the preview default with a clear quota warning, then align docs, tests, and changelog while leaving broader fallback/cost-control fixes on their canonical issues.

## Reproduction Assessment

Yes for the source-level onboarding path: current main deterministically recommends and applies `google/gemini-3.1-pro-preview` for Google API-key setup. No for the live Tier 1 quota exhaustion path, because this read-only review had no Google API key or quota-limited account.

## Solution Assessment

No for an automated one-line default change right now. The stable-default direction is plausible, but current main deliberately reverted it, so maintainers need to choose the quota/default policy and update the surrounding docs/tests together.
