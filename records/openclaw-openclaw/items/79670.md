## Summary

Keep open. Current main and the latest release still apply the Gemini 3.1 Pro Preview default during Google API-key onboarding, the public Google docs and tests still encode that preview default, and the related stable-default PR was closed unmerged, so this needs provider/default-policy review rather than cleanup closure.

## What This Changes

The issue asks OpenClaw to stop recommending Google Gemini preview models during onboarding without warning about Tier 1 quota limits, preferably by defaulting to stable models or showing quota information.

## Best Possible Solution

Decide the Google API-key onboarding policy explicitly: use a stable production-safe default, or retain the preview default with clear quota/rate-limit warning text and aligned docs/tests while leaving failover cascade work to its own track.

## Reproduction Assessment

Yes for the source-level onboarding path: current main and `v2026.5.12` deterministically apply `google/gemini-3.1-pro-preview` during Google API-key onboarding. No for the live Tier 1 quota-exhaustion path because this read-only review did not use a quota-limited Google account.

## Solution Assessment

No for an automated one-line default change right now. The stable-default direction is plausible, but the shipped preview-default restoration and docs/tests alignment mean the maintainable fix needs provider policy review first.
