Keep open. Current main and the latest shipped release still set Google API-key onboarding to `google/gemini-3.1-pro-preview`, and the short-lived stable-default fix was later reverted, so this is an active provider default-policy issue rather than cleanup closure.

## What This Changes

The issue requests that Google onboarding avoid or warn about Gemini preview defaults whose API-tier quota can be much lower than stable alternatives.

## Best Possible Solution

Decide the Google API-key onboarding policy explicitly, then either use a stable production-safe default or keep the preview default with clear quota warnings, with docs, tests, and related quota-failover scope aligned.

## Solution Assessment

No for an automated one-line default change right now. The stable-default direction is plausible, but the later revert means the maintainable fix needs provider-owner policy review and synchronized docs/tests or warning UX.
