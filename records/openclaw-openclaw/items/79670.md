## Summary

Keep open. Current main still applies and documents the Google API-key onboarding default as `google/gemini-3.1-pro-preview`, while Google’s current rate-limit docs say limits vary by model/tier and are more restricted for preview models, so the reported quota-risk surface has not been resolved. ([ai.google.dev](https://ai.google.dev/gemini-api/docs/rate-limits))

## What This Changes

The issue asks Google onboarding to avoid recommending Gemini preview models without quota validation, either by warning users, showing per-model limits, or preferring stable defaults for Tier 1 paid accounts.

## Best Possible Solution

Decide the Google API-key onboarding policy explicitly: use a stable production-safe default, or keep the preview default with clear quota/rate-limit warning text and aligned docs/tests while tracking failover cascade behavior separately.

## Reproduction Assessment

Yes for the source-level onboarding path: current main deterministically applies `google/gemini-3.1-pro-preview` during Google API-key onboarding. No for the live Tier 1 quota-exhaustion path, because this review did not exercise a quota-limited Google account.

## Solution Assessment

No for an automated one-line default change right now. The stable-default direction is plausible, but current main, docs, tests, and recent history intentionally encode the preview default, so the maintainable fix needs provider-owner policy review and synchronized docs/tests.
