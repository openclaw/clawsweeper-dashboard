## Summary

Keep open. Current main still applies the Google API-key onboarding default `google/gemini-3.1-pro-preview`, the public Google docs in this repo still recommend that preview default without a quota-tier warning, and the related stable-default PR was closed unmerged, so this is not cleanup-closeable.

## What This Changes

The issue asks OpenClaw to stop recommending Gemini preview models during Google onboarding without warning about paid-tier quota limits, either by defaulting to a stable model or by showing quota guidance.

## Best Possible Solution

Decide the Google onboarding policy explicitly: either switch API-key onboarding and docs/tests to a stable production-safe default, or keep the preview default with a clear quota/rate-limit warning and leave fallback behavior to the already merged failover fixes.

## Reproduction Assessment

Yes for the source-level onboarding recommendation path: current main deterministically applies `google/gemini-3.1-pro-preview` during Google API-key onboarding. No for the live Tier 1 quota exhaustion path, because this read-only review did not use a quota-limited Google account.

## Solution Assessment

No automated cleanup close or one-line default patch is justified. The maintainable fix needs provider-owner policy alignment plus synchronized docs/tests, because current main intentionally pins and documents the preview default.
