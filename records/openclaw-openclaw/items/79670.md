## Summary

Keep open. Current main still sets Google API-key onboarding to `google/gemini-3.1-pro-preview`, and the public docs/tests still present that preview default without quota-tier warning; because a stable-default attempt was later reverted on main, this needs provider/default-policy review rather than cleanup closure.

## What This Changes

The issue asks OpenClaw to stop recommending Gemini preview models during Google onboarding without warning about low Tier 1 quota limits, either by using stable defaults or showing per-model quota limits.

## Best Possible Solution

Decide the Google API-key onboarding policy explicitly: use a stable production-safe default or keep the preview default with clear quota/rate-limit warning, then align docs, tests, and related failover/account-scope work.

## Reproduction Assessment

Yes for the source-level onboarding path: current main deterministically applies `google/gemini-3.1-pro-preview` during Google API-key setup. No for the live Tier 1 quota exhaustion path, because this read-only review did not exercise a quota-limited Google account.

## Solution Assessment

No for a narrow automated default flip right now. The stable-default direction is plausible, but the later main restoration of the preview default means the maintainable fix needs provider policy review plus synchronized docs and tests.
