## Summary

Keep open. Current main still applies the Google API-key onboarding default `google/gemini-3.1-pro-preview`, docs and tests still describe that preview default, and the prior stable-default fix was closed unmerged after a later main commit intentionally reverted the default back to the preview model.

## What This Changes

The issue requests that Google API-key onboarding stop recommending Gemini preview models without quota guidance, either by using a stable default or by warning/displaying per-model quota limits.

## Best Possible Solution

Choose and document the Google API-key onboarding policy explicitly: either use a stable production-safe default or keep the preview default with clear quota/rate-limit guidance, then update Google plugin docs, tests, and any changelog entry together.

## Reproduction Assessment

Yes for the source-level onboarding path: current main deterministically applies the preview model during Google API-key onboarding. No for the full live Tier 1 quota exhaustion path, because that requires a specific Google AI Studio account tier and real usage.

## Solution Assessment

No automated cleanup closure is appropriate. The safest fix is a provider-owned policy change that keeps onboarding behavior, docs, tests, and external quota guidance consistent instead of applying an isolated default swap.
