## Summary

Keep open. Current main still sets Google API-key onboarding to `google/gemini-3.1-pro-preview`, and a short-lived stable-default fix was later reverted back to the preview model, so this needs provider default-policy review rather than cleanup closure.

## What This Changes

The issue asks Google onboarding to avoid or warn about Gemini preview models with low Tier 1 quota and prefer stable Google models by default.

## Best Possible Solution

Decide the Google API-key onboarding policy explicitly: either use a stable production-safe default with docs/tests/changelog aligned, or keep the preview default but add clear quota/rate-limit warnings and route quota-cascade fixes through the existing failover work.

## Reproduction Assessment

Yes for the source-level onboarding path: current main deterministically applies the preview default during Google API-key onboarding. No for the live Tier 1 quota exhaustion path, because this review did not have Google credentials or a quota-limited account to exercise.

## Solution Assessment

No for an automated one-line default change right now. The closed PR shows the stable-default direction is plausible, but the later main revert means the best solution needs provider-owner policy review and synchronized docs/tests before implementation.
