## Summary

Keep open. Current main still sets Google API-key onboarding to `google/gemini-3.1-pro-preview`, the docs and tests still encode that preview default, and a recent stable-default change was explicitly reverted on main, so this is an active provider default-policy issue rather than cleanup closure.

## What This Changes

The issue asks Google API-key onboarding to avoid recommending Gemini preview models without a quota warning or quota-aware validation for paid Tier 1 users.

## Best Possible Solution

Decide the Google API-key onboarding policy explicitly: either use a stable production-safe default with docs/tests aligned, or keep the preview default but add a clear quota/rate-limit warning and route remaining cascade behavior through the failover work.

## Reproduction Assessment

Yes for the source-level onboarding path: current main deterministically applies `google/gemini-3.1-pro-preview` during Google API-key onboarding. No for the live Tier 1 quota exhaustion path, because this read-only review did not exercise a quota-limited Google account.

## Solution Assessment

No for an automated one-line default flip right now. The stable-default direction is plausible, but the later main restoration of the preview default means the maintainable fix needs provider-owner policy review plus synchronized docs and tests.
