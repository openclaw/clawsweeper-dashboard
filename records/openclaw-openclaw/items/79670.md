## Summary

Keep open: current main still intentionally applies and documents Gemini preview defaults for Google onboarding, and the requested change is a provider/default policy decision rather than stale cleanup.

## What This Changes

The issue requests that Google/Gemini onboarding avoid, validate, or warn about preview model defaults when AI Studio tier quotas make those models unsuitable.

## Best Possible Solution

Provider owners should decide whether Google first-run defaults move to a stable model such as `google/gemini-2.5-flash`, add an explicit preview quota warning, or expose provider-owned quota guidance without adding Google-specific policy to core.

## Reproduction Assessment

Yes for the OpenClaw behavior: current source shows Google API-key onboarding applies `google/gemini-3.1-pro-preview`, while Gemini CLI defaults to `gemini-3-flash-preview`. The external Tier 1 quota exhaustion and cost cascade were not live-reproduced in this read-only review.

## Solution Assessment

Unclear. A provider-owned default change or warning is plausible, but live tier-quota validation would be brittle unless Google exposes a reliable API for it; a static warning/default policy is the safer first maintainer decision.
