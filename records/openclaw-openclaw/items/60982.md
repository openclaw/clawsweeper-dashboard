## Summary

Keep open: current main already covers normal Discord final delivery through the merged reply-safety path, but the draft-preview finalization path still has a source-reproducible leak route. This branch is not safe as submitted because it adds a stale separate sanitizer profile that can leave malformed reasoning visible and can mark media replies delivered when sanitized text becomes empty.

## What This Changes

The PR adds a Discord outbound sanitizer, applies it to final reply delivery and draft-preview finalization, and adds regression tests for stripping leaked internal scaffolding.

## Best Possible Solution

Rebase around current `reply-safety.ts`, apply the strict Discord front-channel sanitizer to draft-preview final edits, preserve non-text payloads when text scrubs empty, and add focused tests plus real behavior proof.

## Reproduction Assessment

Yes, by source inspection. Current main's draft-preview finalization can edit a one-chunk final payload such as `<thinking>internal</thinking>\nVisible answer` without passing through the Discord front-channel sanitizer.

## Solution Assessment

No, not as submitted. The maintainable fix should reuse the current strict Discord reply-safety path on draft-preview finalization and avoid treating sanitized-empty media replies as delivered.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve media when sanitized preview text is empty:** `extensions/discord/src/monitor/message-handler.process.ts:786-793`
  - body: This early return runs whenever sanitized text is empty and a preview was streamed, before allowing the normal delivery path to handle media. A final image or voice reply with an all-internal caption would be marked delivered and the media payload would never be sent.
  - confidence: 0.89
- **[P2] Use the strict Discord delivery sanitizer:** `extensions/discord/src/monitor/sanitize-outbound.ts:48`
  - body: `sanitizeOutboundText` calls `stripAssistantInternalScaffolding`, whose profile preserves unfinished reasoning text. A malformed final such as `<thinking>secret` can still reach Discord, so final delivery should reuse `sanitizeDiscordFrontChannelText` or the strict assistant-visible delivery profile.
  - confidence: 0.86
