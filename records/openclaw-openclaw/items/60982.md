The PR adds a Discord outbound sanitizer, applies it to final delivery and draft-preview finalization, and adds regression tests for leaked thinking and memory scaffolding.

## Best Possible Solution

Yes, by source inspection: current main's draft-preview finalization sanitizes only directive tags before processDiscordMessage can edit the preview message and bypass deliverDiscordReply. I did not run a live Discord session in this read-only review.

## Solution Assessment

No, not as submitted. The narrow maintainable fix is to reuse the current strict Discord front-channel sanitizer and keep media/non-text payloads on the normal delivery path rather than adding a separate stale sanitizer profile.
