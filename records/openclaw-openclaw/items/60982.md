The PR adds a Discord outbound sanitizer, wires it into final delivery and draft-preview finalization, and adds regression tests for leaked assistant-internal scaffolding.

Rebase around current reply-safety.ts, sanitize preview-final edits with the strict Discord front-channel sanitizer, preserve non-text payload delivery, and add focused regression coverage plus real Discord/runtime proof.

No, not as submitted. The narrow maintainable fix is to reuse current main's strict Discord reply-safety helper on the active preview-final path and avoid the branch's separate sanitizer profile and media-dropping early return.
