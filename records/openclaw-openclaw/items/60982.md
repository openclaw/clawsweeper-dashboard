## Summary

Keep open. The underlying Discord leak is valid and still has a draft-preview source path on current main, but this PR is stale against the current delivery architecture and its latest patch still has merge-blocking sanitizer defects plus no real behavior proof.

## What This Changes

The PR adds a Discord outbound sanitizer, wires it into final reply delivery and draft-preview finalization, and adds regression tests for stripping leaked internal scaffolding.

## Best Possible Solution

Rebase around current `reply-safety.ts`, apply the strict Discord front-channel sanitizer to draft-preview final edits as well as normal delivery, preserve non-text payloads when text scrubs empty, and add focused tests plus real proof.

## Reproduction Assessment

Yes, by source inspection. A one-chunk final payload such as `<thinking>internal</thinking>\nVisible answer` can still be edited through the draft-preview finalization path on current main without the front-channel sanitizer.

## Solution Assessment

No, not as submitted. The maintainable fix is to reuse the current strict Discord front-channel sanitizer on the active draft-preview path and preserve media/components when sanitized text becomes empty, rather than adding a separate stale sanitizer profile.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve media when sanitized preview text is empty:** `extensions/discord/src/monitor/message-handler.process.ts:786-793`
  - body: This returns after a sanitized-empty final whenever a preview was streamed, before checking `hasMedia`. An image or voice reply whose caption strips to empty would mark the preview delivered and skip normal delivery, dropping the media payload.
  - confidence: 0.89
- **[P2] Use the strict delivery sanitizer for Discord finals:** `extensions/discord/src/monitor/sanitize-outbound.ts:48`
  - body: `stripAssistantInternalScaffolding` uses the internal-scaffolding profile, which preserves unfinished reasoning text. A malformed final like `<thinking>secret` can still reach Discord; final delivery should reuse `sanitizeAssistantVisibleText` or `sanitizeDiscordFrontChannelText`.
  - confidence: 0.86
