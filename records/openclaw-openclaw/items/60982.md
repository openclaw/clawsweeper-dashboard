## Summary

Keep open. Current main still does not strip `<thinking>` or `<relevant-memories>` from Discord final delivery or preview-final edits, and the PR is useful but stale against the current shared outbound/draft-preview architecture. The patch also still has concrete correctness and privacy-sensitive sanitizer issues before merge.

## What This Changes

The PR adds a Discord outbound sanitizer for assistant-internal scaffolding, wires it into final Discord delivery and draft-preview finalization, and adds Discord delivery/draft-stream regression tests.

## Best Possible Solution

Rebase the fix onto current main by putting strict user-visible scaffolding sanitization on the Discord outbound adapter/shared delivery path and the draft-preview final-edit path. Preserve media delivery when captions sanitize away, keep Discord markup/code fences intact, add focused regression tests for final delivery and preview finalization, and add the required changelog entry.

## Reproduction Assessment

Yes. A deterministic source-level reproduction is a Discord final payload such as `<thinking>internal</thinking>\nVisible answer`: current main sends final replies through `deliverOutboundPayloads`, and Discord's adapter sanitizer only strips `system-reminder`/`previous_response`; preview finalization strips only inline directives.

## Solution Assessment

No, not as-is. The proposed direction is right, but the patch should be adapted to current main's shared outbound adapter and draft-preview controller, and it should use the strict delivery sanitizer while preserving media finals when sanitized text is empty.
