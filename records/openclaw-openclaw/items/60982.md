## Summary

Keep open: current main already sanitizes durable Discord delivery, but the draft-preview finalization path can still edit the preview message with only directive-tag stripping, and this PR remains a relevant but currently unsafe fix path because it has blocking correctness findings and no after-fix real Discord/runtime proof.

## What This Changes

The branch adds a Discord outbound sanitizer, applies it to final delivery and draft-preview finalization, and adds regression tests for leaked thinking and memory scaffolding.

## Best Possible Solution

Rebase on current main, reuse sanitizeDiscordFrontChannelText for preview-final edits, preserve media/non-text fallback delivery, and add focused tests plus redacted runtime proof.

## Reproduction Assessment

Yes, by source inspection: current main's draft-preview finalization strips only directive tags before processDiscordMessage can edit the preview message and bypass deliverDiscordReply. I did not run a live Discord session in this read-only review.

## Solution Assessment

No, not as submitted. The narrow maintainable fix is to reuse the current strict Discord front-channel sanitizer and keep media/non-text payloads on the normal delivery path rather than adding a separate stale sanitizer profile.
