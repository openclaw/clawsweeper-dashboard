## Summary

Keep open. Current main still does not sanitize Discord final replies for leaked `<thinking>` or `<relevant-memories>` scaffolding, and this PR is useful source work but is not ready as-is: it is based on an older Discord delivery shape and still has concrete correctness issues around sanitized-empty media finals and unclosed reasoning tags.

## What This Changes

The PR adds a Discord-specific outbound text sanitizer, wires it into Discord final delivery and draft-preview finalization, and adds regression tests for leaked thinking and relevant-memory scaffolding.

## Best Possible Solution

Rebase this work onto current main and implement the sanitizer through the Discord outbound adapter/shared delivery path plus the draft-preview controller. The final version should use a strict, code-region-aware user-visible sanitizer for reasoning and memory scaffolding, preserve legitimate Discord markup/code fences, and keep media delivery on the normal path when only the caption sanitizes away.

## Reproduction Assessment

Yes. A deterministic source-level reproduction is a non-reasoning Discord final payload containing `<thinking>internal</thinking>\nVisible answer`: current main routes it through `deliverDiscordReply` and the Discord adapter sanitizer only removes `system-reminder`/`previous_response`, while preview finalization strips only inline directives.

## Solution Assessment

No, not as-is. The proposed direction is right, but it should be adapted to the current shared outbound/draft-preview architecture and should fix the sanitized-empty media and unclosed-reasoning cases before merge.
