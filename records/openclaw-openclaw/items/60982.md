## Summary

Keep open. Current main now sanitizes ordinary Discord durable final payloads, but draft-stream preview finalization still builds its final edit from directive-tag stripping only, and this PR has unique preview-path work plus blocking sanitizer/media-loss issues and missing real behavior proof.

## What This Changes

The PR adds a Discord outbound sanitizer, applies it to legacy final delivery and draft-preview finalization, and adds regression tests for stripping thinking and relevant-memory scaffolding.

## Best Possible Solution

Rebase onto current main by applying the canonical strict delivery sanitizer to Discord front-channel payload scrubbing and draft-preview finalization, while preserving non-text and media payloads when captions scrub empty.

## Reproduction Assessment

Yes, by source inspection. On current main, durable final replies are scrubbed, but `resolvePreviewFinalText` can still send a fitting final preview edit after only directive-tag stripping, so `<thinking>...</thinking>` can remain visible in that path.

## Solution Assessment

No, not as submitted. The direction is right, but the patch targets an older delivery shape and uses the weaker internal-scaffolding profile instead of the current strict delivery sanitizer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Let media finals fall through when captions scrub empty:** `extensions/discord/src/monitor/message-handler.process.ts:786-793`
  - body: The sanitized-empty shortcut marks final delivery handled whenever a streamed preview exists, even if the final payload has media. If an image or voice reply has a caption made only of stripped scaffolding, this returns before normal delivery and drops the media; gate this shortcut to text-only finals or fall through for media.
  - confidence: 0.89
- **[P2] Use the strict delivery sanitizer for outbound reasoning:** `extensions/discord/src/monitor/sanitize-outbound.ts:48`
  - body: `sanitizeOutboundText` delegates to `stripAssistantInternalScaffolding`, whose profile preserves unfinished reasoning text. A malformed final payload like `<thinking>private tail` can still expose internal text to Discord users, so outbound delivery should use the strict assistant-visible delivery sanitizer.
  - confidence: 0.86
