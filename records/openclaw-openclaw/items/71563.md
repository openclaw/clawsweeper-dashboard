## Summary

Keep open: the direction is useful, but the current predicate also moves `/reset soft` onto the control lane, which can race with active same-session mutation, and the external PR still lacks after-fix real behavior proof.

## What This Changes

The PR extracts a shared Telegram command resolver, routes `/new` and `/reset` text commands to the Telegram control sequentializer lane, and adds sequential-key tests for `/new`, `/reset`, and bot-suffixed `/new`.

## Best Possible Solution

Land a repaired version that bypasses only `/new` and hard `/reset` forms, leaves `/reset soft` on the chat/topic lane, adds focused sequentializer and middleware regressions, and lets maintainers handle the changelog during landing.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence. On the PR head, `/reset soft` resolves through the argument-accepting `/reset` alias and reaches the new control-lane predicate even though current soft-reset code mutates the existing session in place.

## Solution Assessment

No. The maintainable fix is narrower: control-lane bypass for `/new` and hard `/reset`, while `/reset soft` remains serialized on the normal Telegram chat/topic key.
