## Summary

Keep open: current main still lacks the requested Telegram session-control bypass, but this PR should not merge as-is because its reset predicate also moves `/reset soft` onto the control lane, allowing an in-place session mutation to interleave with an active turn. The prior duplication review was addressed by the shared resolver, and the remaining work is a narrow predicate, regression, and changelog repair.

## What This Changes

The PR extracts shared Telegram command resolution and routes registered `/new` and `/reset` commands to the Telegram `:control` sequentializer lane with focused unit coverage.

## Best Possible Solution

Keep the control-lane bypass for `/new` and hard `/reset` forms, but explicitly keep `/reset soft` variants on the normal Telegram chat/topic lane; then add focused sequential-key and middleware regressions plus a single-line Unreleased Fixes changelog entry crediting the contributor.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: on the PR head, `/reset soft` is accepted as a `/reset` alias and the new predicate returns the Telegram control key instead of the normal chat/topic key.

## Solution Assessment

No. The PR direction is close, but the broad command-key predicate is not the narrowest maintainable fix because it conflates hard reset with in-place soft reset; excluding soft-reset forms before returning the control lane is safer.
