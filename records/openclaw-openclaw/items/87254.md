## Summary

## What This Changes

This PR raises plugin-state and Telegram persistent message-cache row budgets to 3000, evicts oldest rows from the namespace being written before failing plugin-wide cap writes, and updates plugin-state and Telegram regression tests.

## Best Possible Solution

Land this focused fix after maintainers explicitly accept the global plugin-state eviction and Telegram cache-retention tradeoff, with normal CI gating the exact head.
