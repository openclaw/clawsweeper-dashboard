## Summary

Keep this open: current main and v2026.5.12 still lack a SKILL.md/frontmatter model-routing contract, and the directly related implementation at https://github.com/openclaw/openclaw/pull/58142 is open with product, provider-routing, proof, and security questions unresolved. The overlapping config request at https://github.com/openclaw/openclaw/issues/8441 remains relevant but does not supersede this issue's SKILL.md contract.

## What This Changes

The issue requests a public SKILL.md frontmatter contract that lets a skill declare model routing preferences for its own execution.

## Best Possible Solution

Define one public per-skill routing contract across SKILL.md and skills.entries, including precedence, consent, allowlists, fallbacks or ceilings, and provider/auth-safe early resolution, then land or replace https://github.com/openclaw/openclaw/pull/58142 with docs, focused tests, and runtime proof.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection of current main and v2026.5.12 gives high confidence that the requested SKILL.md model-routing field and runtime router are absent.

## Solution Assessment

Unclear until maintainers choose the public contract. The frontmatter direction is plausible, but the best implementation needs explicit precedence, consent, fallback, model allowlist, and provider/auth routing semantics, especially alongside https://github.com/openclaw/openclaw/issues/8441.
