Keep this open. ## What This Changes

The issue requests an optional SKILL.md frontmatter model setting so individual skills can route execution to a preferred model instead of always using the agent default.

## Best Possible Solution

Define the core skill model-routing contract first, then land or replace the related implementation with preserved agent skill allowlists, documented precedence/security behavior, and focused parser/runtime/fallback tests.
