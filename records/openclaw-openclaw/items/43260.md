## Summary

Keep open: current main and the latest release still do not define or consume a SKILL.md per-skill model-routing contract. The related implementation candidate at https://github.com/openclaw/openclaw/pull/58142 is still open, conflicting, and carries unresolved provider/auth and security-contract concerns, so this should remain in maintainer review.

## What This Changes

The issue requests an optional SKILL.md frontmatter field that lets a skill declare a preferred model for its own execution instead of always using the agent default.

## Best Possible Solution

Define one maintainer-approved, operator-controlled per-skill routing contract, then implement it before provider/model/auth planning with precedence, allowlist or consent policy, fallback behavior, docs, focused tests, and real provider-routing proof.

Not applicable as a bug reproduction; this is a feature/config-contract request. Source, docs, tests, latest-release inspection, and live related GitHub state show the requested SKILL.md model-routing contract is still absent.

## Solution Assessment

Unclear until maintainers choose the routing and security contract. A SKILL.md field is plausible, but the maintainable solution needs explicit precedence, fallback, allowed-model, consent, provider/auth, turn-boundary, and observability semantics.
