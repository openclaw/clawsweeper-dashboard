The issue requests an optional SKILL.md frontmatter `model` field so individual skills can route execution to a preferred model instead of always inheriting the agent default.

## Best Possible Solution

Define one maintainer-approved per-skill routing contract across SKILL.md and any config override, then land or replace https://github.com/openclaw/openclaw/pull/58142 with provider-aware resolution, consent/allowlist policy, fallback and precedence semantics, docs, tests, and redacted runtime proof.
