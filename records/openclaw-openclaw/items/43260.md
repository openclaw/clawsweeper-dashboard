## Summary

Keep open: current main still has no per-skill model-routing contract, and the linked open PR #58142 is the active implementation candidate rather than a merged fix.

## What This Changes

Add an optional SKILL.md/OpenClaw skill metadata model field so a skill can request a specific model instead of inheriting the agent or session default.

## Best Possible Solution

Define a versioned skill metadata contract for preferred model routing, wire it through skill loading and the chosen runtime path, document precedence/fallback behavior, and land #58142 or a successor with focused regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high confidence current main lacks the requested feature because skill metadata types, parser, docs, and model-routing docs omit per-skill model fields.

## Solution Assessment

Unclear. The direction belongs in core skill/runtime model selection, but maintainers should settle field shape, fallback policy, and current-session versus subagent behavior before accepting an implementation.
