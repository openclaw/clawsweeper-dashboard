## Summary

Keep this open: current main still lacks a SKILL.md model-routing contract, and the active implementation candidate is an open PR with unresolved contract/security questions rather than a merged or shipped fix.

## What This Changes

Add optional SKILL.md model metadata so individual skills can request model routing instead of always inheriting the agent default.

## Best Possible Solution

Settle the public skill-routing contract, then land or replace https://github.com/openclaw/openclaw/pull/58142 with an implementation that preserves skill allowlists, documents consent/precedence/fallback behavior, and covers parser, cached-snapshot, routing, and security cases.

## Reproduction Assessment

Not applicable. This is a feature request, but the current-main gap is high-confidence from source inspection: skill metadata and parsing do not include model-routing fields, and no skill model router exists.

## Solution Assessment

No, not yet. The direction is plausible, but the field names, fallback/ceiling semantics, precedence, and consent/observability policy need maintainer agreement before the open PR or a successor is the best maintainable solution.
