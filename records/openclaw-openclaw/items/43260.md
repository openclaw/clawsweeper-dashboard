## What This Changes

The issue requests a SKILL.md frontmatter model-routing contract so individual skills can declare a preferred model instead of inheriting only the agent/default model.

## Best Possible Solution

Define one maintainer-approved per-skill routing contract across SKILL.md and any config override, then land or replace the open implementation with provider-aware resolution, consent or allowlist policy, fallback and precedence semantics, docs, tests, and redacted runtime proof.
