## Summary

Keep open: current main and the latest release still lack a SKILL.md model-routing contract, and the active implementation candidate remains open with unresolved provider, compatibility, and security-boundary questions.

## What This Changes

The request asks OpenClaw to support a per-skill model field in SKILL.md frontmatter so skill executions can route to a declared model instead of always using the agent default.

## Best Possible Solution

Define one maintainer-approved per-skill routing contract across SKILL.md and any config override, then land or replace the active PR with provider-aware resolution, allowlist/consent policy, fallback and precedence semantics, docs, tests, and redacted runtime proof.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config-contract request. Source, docs, release state, and related GitHub state show the requested SKILL.md model-routing contract is still absent.

## Solution Assessment

Unclear until maintainers choose the routing and security contract. A SKILL.md field is plausible, but the maintainable solution needs explicit precedence, fallback, allowed-model, consent, provider/auth, prompt-cache, observability, and session-boundary semantics.
