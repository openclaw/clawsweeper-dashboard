## Summary

Keep this feature request open for maintainer design review: current main supports generic pre-turn model overrides, but it does not expose a typed active-skill identity or a per-skill routing contract, so the central requested behavior remains missing. The production examples support the need, while the closed implementation PR does not provide a canonical solution because it inferred skill activation from SKILL.md file reads.

## What This Changes

Add per-skill model routing so skill execution can use an operator-approved model different from the parent agent’s default.

## Best Possible Solution

Introduce a typed skill-intent or deterministic skill-dispatch fact that reaches the existing pre-model-resolution seam, then let explicit operator configuration authorize and map that intent to allowed primary and fallback models; treat package-authored model metadata as advisory unless the operator opts into it.

## Reproduction Assessment

Not applicable: current source confirms that per-skill model routing is absent, but this is a request for a new contract rather than broken documented behavior.

## Solution Assessment

No, not in the proposed enforceable-frontmatter form alone. The maintainable solution is a typed skill-intent boundary combined with operator-authorized routing through the existing before_model_resolve seam, avoiding file-read inference and preventing downloaded skills from unilaterally choosing credentials or cost tiers.
