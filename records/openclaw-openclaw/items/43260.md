## Summary

Keep open. Current main still has the skill metadata and model-selection gap described here, and the matching implementation candidate remains open with unresolved contract/security review questions rather than being merged or shipped.

## What This Changes

Request an optional SKILL.md model-routing field so a skill can declare which configured model should handle that skill.

## Best Possible Solution

Define a public skill model-routing contract with allowlisted resolution, precedence, fallback/ceiling behavior, and auditability, then land a rebased implementation or replacement for https://github.com/openclaw/openclaw/pull/58142.

## Reproduction Assessment

Not applicable as a bug reproduction. The feature gap is source-visible: current skill metadata, parser, docs, and runtime paths do not expose or consume a skill-level model preference.

## Solution Assessment

No, not as a raw frontmatter field alone. The maintainable solution needs a settled contract for precedence, fallbacks, allowlists, and provider/cost visibility before code is merged.
