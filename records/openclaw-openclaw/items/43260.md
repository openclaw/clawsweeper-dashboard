## Summary

Keep open. Current main still has model selection for agent defaults, session overrides, cron payloads, and subagents, but no SKILL.md or plugin-skill metadata contract that lets an individual skill declare a model preference; the InvestorClaw and KinthAI comments add concrete production validation, and #58142 remains an open implementation candidate rather than a merged fix.

## What This Changes

Request an optional SKILL.md/frontmatter model-routing contract so individual skills can prefer a configured model independent of the agent default.

## Best Possible Solution

Define and land a documented skill metadata model-routing contract that reuses existing model resolution and policy enforcement, covers fallback and ceiling semantics, and includes parser, snapshot, prompt/runtime, slash-invocation, and security-policy tests.

## Reproduction Assessment

Not applicable as a bug reproduction. The product gap is high-confidence from source inspection: the current parser, metadata type, snapshot, prompt formatter, docs, and embedded runner expose no per-skill model-routing contract.

## Solution Assessment

Unclear as proposed. A bare frontmatter model field is plausible, but the best maintainable solution should specify alias/allowlist enforcement, fallbacks, ceilings, and whether routing uses live model switching or isolated execution before merging.
