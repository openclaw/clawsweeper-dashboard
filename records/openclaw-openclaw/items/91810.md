## Summary

The PR is still useful: current main can drop the deferred persisted thinking block when the terminal WebChat final payload has renderable text, and this patch fixes that by reloading only the selected/equivalent transcript path that already observed assistant thinking.

## What This Changes

The PR updates WebChat final-event reload logic to remember deferred selected assistant `session.message` thinking and repaint from `chat.history` when the live final payload omits that thinking, with focused unit coverage.

## Best Possible Solution

Land the focused UI reload fix after normal maintainer review and CI; keep broader live reasoning-stream delivery work in its separate gateway/agent PRs.

## Reproduction Assessment

Yes at source level: current main defers selected `session.message` reloads during active runs but suppresses final reload when the final assistant payload has renderable text. The PR body also supplies after-fix browser WebChat/gateway proof for the reported event order.

## Solution Assessment

Yes. Extending the existing final-event reload decision with a scoped deferred-thinking fact is narrower than reloading all renderable finals or changing gateway payload contracts.
