The PR tracks deferred selected-session assistant thinking in Control UI WebChat and reloads `chat.history` after final chat events whose live payload lacks that persisted thinking, with focused UI tests.

## Best Possible Solution

Land the focused UI reload behavior if maintainers accept the linked issue as fixed by post-run transcript repaint, while keeping broader live thinking-stream delivery in separate gateway work.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: selected `session.message` defers reload during an active run, then a renderable final `chat` payload makes current main skip `loadChatHistory()`. I did not execute the browser flow in this read-only review.

## Solution Assessment

Yes, this is the narrowest maintainable fix I found: it preserves the no-flicker path for ordinary renderable finals and reloads only when selected deferred assistant transcript thinking would otherwise be missing. Always reloading renderable finals would reintroduce the flicker the current predicate was designed to avoid.
