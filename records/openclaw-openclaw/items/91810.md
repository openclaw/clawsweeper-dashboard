## Summary

Keep open: current main still has the renderable-final no-reload path, and this PR adds a focused deferred-thinking reload path with selected-session guards, coverage, and sufficient live WebChat/gateway proof. I found no blocking patch findings.

## What This Changes

The PR tracks selected-session deferred assistant thinking from `session.message` and reloads WebChat history on terminal final events when the live final payload lacks that thinking.

## Best Possible Solution

Land the focused WebChat reload fix after the normal changed-file CI/check gates stay green; broader live thinking-stream delivery remains separate from this post-run history repaint path.

## Reproduction Assessment

Yes, source-reproducible: current main defers `session.message` reload during an active run but skips history reload when the final event carries renderable text. I did not run the browser repro in this read-only pass.

## Solution Assessment

Yes. The patch preserves the existing no-flicker path for ordinary renderable finals and reloads only when selected-session deferred assistant thinking exists and the final payload lacks thinking; always reloading would be a broader performance and flicker regression.
