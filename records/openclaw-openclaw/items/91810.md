## Summary

Keep open; the patch targets the right deferred WebChat history-reload path, but it is not merge-ready because the supplied live proof and source path show the final answer can be rendered twice after the reload.

## What This Changes

The PR tracks deferred assistant `session.message` thinking content in the Control UI and forces a WebChat history reload when the terminal final chat event lacks that thinking block.

## Best Possible Solution

Keep the deferred-thinking reload, but reconcile the terminal final with the persisted transcript so WebChat shows one assistant answer with reasoning and cover that through the real history reconciliation path.

## Reproduction Assessment

Yes for the blocking PR defect: the PR's own live proof reports two final-answer renderings, and source inspection shows the reload decision happens after the final chat event has already been applied.

## Solution Assessment

No; reloading history for deferred thinking is a plausible fix, but the best fix must also reconcile the terminal final with the persisted transcript and test the real history merge behavior.
