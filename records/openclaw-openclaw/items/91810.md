## What This Changes

This PR updates Control UI WebChat gateway event handling, final-event reload decisions, and chat-history reconciliation so persisted thinking can repaint after a text-only terminal final without leaving duplicate assistant finals.

## Best Possible Solution

Land this PR or an equivalent narrow fix after maintainer review of the WebChat session/message ordering risk, then close the linked regression issue once the fix reaches main.

## Reproduction Assessment

Yes, at source level. Current main skips history reload for renderable text-only finals while WebChat renders reasoning from the displayed message, so persisted thinking can remain hidden until a manual refresh.

## Solution Assessment

Yes. This is the narrowest maintainable fix I found: track only thinking-bearing deferred session messages and reload only when the terminal final lacks that thinking, rather than restoring unconditional final reloads that current code avoids for flicker reasons.
