Keep open for maintainer review: the PR is a focused bug fix and current main still has the reported WebChat deferred-thinking gap; I found no blocking defects after reading the reload state machine, message extraction, session scoping, adjacent tests, and supplied browser proof.

## What This Changes

The PR teaches Control UI WebChat to remember when a selected deferred assistant `session.message` contained thinking and to reload `chat.history` after a terminal final payload that lacks that thinking.

## Best Possible Solution

Land the focused UI reload guard once normal checks are green, preserving the source-reply no-flicker path and the session/agent scoping guards.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: on current main, a selected `session.message` can be deferred during an active run, and a later renderable assistant final makes `shouldReloadHistoryForFinalEvent` false, so history is not reloaded until manual refresh.

## Solution Assessment

Yes, this is the best local fix I found: it reuses the existing history-reload path only when persisted thinking is known to be missing from the live final payload, instead of duplicating transcript projection in the live event handler or changing gateway payloads.
