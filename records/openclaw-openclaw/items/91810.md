## Summary

Keep open: the PR targets the right WebChat regression path, but its own after-fix proof still shows the final answer twice after the deferred-thinking history reload, so it needs one more reconciliation fix before merge.

## What This Changes

The PR tracks deferred assistant thinking from selected `session.message` events and reloads WebChat history on terminal final events when the live final payload lacks that thinking block.

## Best Possible Solution

Preserve the deferred-thinking history reload, but make the reload/reconciliation path collapse the live final answer into the persisted thinking+answer transcript instead of rendering both.

## Reproduction Assessment

Yes for the PR defect: the PR body's after-fix WebChat proof shows two visible copies of the final answer, and the source path appends the terminal final before triggering the new history reload.

## Solution Assessment

No. Reloading history for deferred thinking is the right direction, but it is not the best fix until the same path de-duplicates the terminal final answer that history already represents.
