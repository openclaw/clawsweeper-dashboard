## Summary

Keep this PR open: current main and the latest release still have the session-status denominator using a raw `.jsonl` suffix while the indexer uses the shared usage-counted session predicate. The code change is the right narrow fix, but the external contributor proof still does not show the changed `openclaw memory status` behavior in a real sessions directory.

## What This Changes

The PR changes `extensions/memory-core/src/cli.runtime.ts` so `scanSessionFiles()` counts session files with `isUsageCountedSessionTranscriptFileName()` instead of a raw `.jsonl` suffix check.

## Best Possible Solution

Land this shared-predicate fix after the contributor adds redacted live `openclaw memory status` proof, then close the canonical linked issue when the fix reaches main.

## Reproduction Assessment

Yes. Source inspection shows current main renders the status denominator from `scanSessionFiles()` while session indexing uses `isUsageCountedSessionTranscriptFileName()`, though I did not run a live CLI reproduction in this read-only review.

## Solution Assessment

Yes for the patch shape. Reusing the existing indexer filename predicate in the CLI denominator is the narrow maintainable fix, but the PR still needs real `openclaw memory status` proof before merge.
