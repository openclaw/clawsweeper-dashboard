## Summary

Keep open: the code change is narrow and appears to be the right fix, but this external PR still lacks real after-fix `openclaw memory status` proof and its body misstates the reset/deleted archive contract.

## What This Changes

The PR changes memory-core CLI session scanning to count session transcript files with the existing usage-counted filename predicate instead of a raw `.jsonl` suffix check.

## Best Possible Solution

Land the narrow predicate-sharing change after the contributor supplies real CLI proof and corrects the PR-body wording about reset/deleted archive inclusion.

## Reproduction Assessment

Yes at source level: current main renders `status.files` against `scan.totalFiles`, and `scanSessionFiles()` still uses a different filename predicate than the indexer file list. I did not run live `openclaw memory status` in this read-only review.

## Solution Assessment

Yes for code shape: reusing `isUsageCountedSessionTranscriptFileName()` in `scanSessionFiles()` aligns the status denominator with the indexer contract without adding a second predicate. The PR still needs better real behavior proof and corrected wording before merge.
