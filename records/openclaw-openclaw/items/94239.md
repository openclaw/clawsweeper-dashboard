## Summary

Keep open: the code change is a focused fix for the open memory-status counter bug, but it still needs contributor-supplied real CLI proof and the PR body currently misstates reset/deleted archive semantics.

## What This Changes

The PR changes `scanSessionFiles` in the memory-core CLI runtime to use `isUsageCountedSessionTranscriptFileName` instead of a strict `.jsonl` suffix filter for the memory status session counter.

## Best Possible Solution

Merge the focused predicate alignment after the PR body and real CLI proof show the status denominator matches the usage-counted session helper.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: create usage-counted reset/deleted archive files and excluded backup/checkpoint/trajectory artifacts under the agent sessions directory, then current `memory status` uses a strict `.jsonl` denominator while the indexer uses the helper-selected set.

## Solution Assessment

Yes for the code path. Reusing the existing usage-counted session transcript predicate is the narrow maintainable fix; excluding reset/deleted archives from indexing would conflict with current protected memory-search behavior.
