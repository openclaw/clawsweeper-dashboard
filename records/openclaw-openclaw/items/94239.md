## Summary

Keep open: the patch appears to be the right narrow code fix for the memory status denominator mismatch, but this external PR still lacks after-fix real `openclaw memory status` proof and its body misdescribes the reset/deleted archive contract.

## What This Changes

The PR changes memory-core CLI status scanning to count session files with `isUsageCountedSessionTranscriptFileName` instead of a strict `.jsonl` suffix filter.

## Best Possible Solution

Merge the predicate alignment after the PR body describes reset/deleted archives as included by the helper and the contributor adds redacted terminal output or a terminal screenshot from `openclaw memory status` on a setup containing normal, reset, deleted, backup, and trajectory artifacts.

## Reproduction Assessment

Yes. Source inspection shows `memory status` currently counts strict `.jsonl` filenames while the session indexer uses the usage-counted helper, so a sessions directory with reset/deleted archives can produce a mismatched denominator.

## Solution Assessment

Yes for the narrow counter mismatch. Reusing the indexer's public helper is the smallest maintainable fix, while excluding reset/deleted archives would conflict with current protected archive-recall behavior.
