## What This Changes

The PR adds gateway URL derivation metadata, load-time resolution, page-derived scoped localStorage writes, and storage tests for path-prefixed Control UI gateway persistence.

## Best Possible Solution

Resolve the legacy fallback during loadSettings for path-prefixed sibling instances while preserving explicit overrides, add focused regression coverage, and land with redacted nginx/browser or terminal proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: seed the shared legacy settings key with gatewayUrl wss://gateway.example/a, set the page under /b, and call loadSettings; current main and PR head both keep /a on the first upgraded load.

## Solution Assessment

No. The flag and page-scoped cache direction is useful, but the undefined-flag branch preserves the stale legacy URL before any page-scoped write can correct it.
