## Summary

Keep open: the reverse-proxy storage bug is valid, but this PR still preserves unflagged legacy gateway URLs on the first upgraded load and lacks required real behavior proof from the contributor’s setup.

## What This Changes

The PR adds gateway URL auto-derived metadata, page-derived scoped localStorage writes, and storage test updates for path-prefixed Control UI gateway persistence.

## Best Possible Solution

Resolve the legacy fallback during loadSettings for path-prefixed sibling instances while preserving explicit overrides, add focused regression coverage, and land with redacted nginx/browser or equivalent terminal/log proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: seed the shared legacy settings key with gatewayUrl wss://gateway.example/a, set the page under /b, and call loadSettings; current main and PR head both keep /a on the first upgraded load.

## Solution Assessment

No. The metadata and page-scoped cache direction is useful, but the undefined-flag branch preserves the stale legacy URL before any page-scoped write can correct the initial upgraded load.

## Review Findings
