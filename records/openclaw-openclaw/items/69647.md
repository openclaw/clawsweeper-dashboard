## Summary

Keep open: this PR targets a real Control UI reverse-proxy storage bug, but the current head still preserves unflagged legacy gateway URLs on the first upgraded load and it lacks required real behavior proof.

## What This Changes

The PR adds Control UI persisted gateway URL metadata, a resolver, page-derived scoped localStorage writes, and focused storage test updates.

## Best Possible Solution

Resolve legacy fallback during loadSettings so the first upgraded visit selects the page-derived WebSocket URL while preserving explicit same-host custom gateway paths, then add focused regression coverage and real reverse-proxy proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: seed openclaw.control.settings.v1 with gatewayUrl "wss://gateway.example/a", set the page location under /b, and call loadSettings; current main and PR head both preserve the stored /a URL on first load.

## Solution Assessment

No. The flag and page-scoped-key direction is useful, but returning stored for an undefined flag defers the fix until after a later save and leaves the first upgraded load broken.
