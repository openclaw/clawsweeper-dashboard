## What This Changes

The branch adds Control UI gatewayUrlAutoDerived metadata, resolves stored gateway URLs through that flag, writes a page-derived scoped localStorage entry, and updates storage tests.

## Best Possible Solution

Fix loadSettings so unflagged legacy sibling-path records are corrected on the initial upgraded load while preserving explicit same-host overrides, then land with focused storage coverage and redacted real reverse-proxy proof.

## Reproduction Assessment

Yes, by source inspection. Seed the shared legacy settings key with gatewayUrl wss://gateway.example/a, set the page under /b, and call loadSettings; current main and PR head both keep /a on the first upgraded load.

## Solution Assessment

No. The metadata and page-scoped cache direction is useful, but the undefined-flag branch preserves the stale legacy URL before any page-scoped write can correct the initial upgraded load.
