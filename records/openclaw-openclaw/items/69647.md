## Summary

Keep open: the patch still preserves unflagged legacy fallback gateway URLs on first upgraded load, so the reported reverse-proxy path switch can still select the old backend; after-fix real behavior proof is also missing.

## What This Changes

The branch adds a persisted `gatewayUrlAutoDerived` flag, a gateway URL resolver, page-derived scoped localStorage writes, and focused Control UI storage test updates.

## Best Possible Solution

Resolve legacy fallback data during `loadSettings()` so `/b` selects its page-derived WebSocket immediately without breaking explicit custom gateway paths, then add focused storage regression coverage and real reverse-proxy proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: seed `openclaw.control.settings.v1` with `gatewayUrl: "wss://gateway.example/a"`, set the page location under `/b`, and call `loadSettings()`; current main and PR head both preserve the stored `/a` URL on first load.

## Solution Assessment

No. The flag and page-scoped-key direction is useful, but returning `stored` for an undefined flag defers the fix until after a later save, leaving the first upgraded load broken.
