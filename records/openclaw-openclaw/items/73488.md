## Summary

Keep open. Current main still exposes provider `createStreamFn` context without per-conversation identity, and open PR #73492 is an implementation candidate with closing syntax, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue requests adding optional `sessionId` and `sessionKey` to provider plugin stream context so provider plugins can scope per-conversation state.

## Best Possible Solution

Land a reviewed version of PR #73492, or an equivalent narrow core change, that adds optional session identity to provider stream context and forwards it only from session-bearing paths.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: current main has session identity in the runner, compaction, and `/btw` flows, but the provider context type and registration helper cannot deliver it to provider plugins.

## Solution Assessment

Yes. Optional fields on the existing provider stream context are the narrowest backward-compatible fix because existing plugins can ignore them while conversation-aware providers can scope state by `sessionKey` or `sessionId`.
