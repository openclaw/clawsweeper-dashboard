## Summary

Keep open. Current main still omits `sessionId` and `sessionKey` from provider plugin `createStreamFn` context, and open PR #73492 uses closing syntax for the same issue, so this should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue asks OpenClaw to expose per-conversation `sessionId` and `sessionKey` to provider plugin `createStreamFn` contexts so providers can scope upstream session state by conversation.

## Best Possible Solution

Land a reviewed version of #73492, or an equivalent narrow core change, that adds optional identity fields only to session-bearing provider stream paths while leaving one-off completions unset.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence, and the issue gives a concrete runtime repro using a Glueclaw-backed provider in two concurrent DMs where current main cannot pass conversation identity to the plugin.

## Solution Assessment

Yes. Optional fields on the existing provider stream context are the narrowest backward-compatible fix because existing plugins can ignore them while conversation-aware providers can scope state by `sessionKey` or `sessionId`.
