## Summary

Keep open: current main and latest release still omit per-conversation identity from provider `createStreamFn` context, and the open implementation candidate has not merged yet.

## What This Changes

The issue requests optional `sessionId` and `sessionKey` in provider `createStreamFn` context so custom provider transports can scope state per conversation.

## Best Possible Solution

Review and either land or deliberately decline https://github.com/openclaw/openclaw/pull/93025; if accepted, keep the fields optional, document the privacy boundary, and leave one-off completion paths identity-free.

## Reproduction Assessment

Yes. Source inspection shows `ProviderCreateStreamFnContext` and `registerProviderStreamForModel` omit `sessionId`/`sessionKey` even though the agent turn and compaction paths already carry session identity.

## Solution Assessment

Yes, pending maintainer API/privacy sign-off. Optional fields on the existing provider `createStreamFn` context are the narrowest core seam; plugin-side hashing, prompt parsing, or filesystem watching would be weaker.
