## Summary

Keep open. Current main and latest release still omit `sessionId`/`sessionKey` from provider `createStreamFn` context, while session-bearing runner, compaction, and `/btw` paths already have conversation identity available; open PR #73492 is the active closing implementation candidate.

## What This Changes

Request to add optional `sessionId` and `sessionKey` to provider plugin `createStreamFn` context and plumb them from session-bearing agent paths.

## Best Possible Solution

Land a reviewed version of PR #73492, or an equivalent narrow API change, that adds optional session identity fields to the provider stream context, forwards them only from session-bearing paths, and updates public SDK docs/changelog if maintainers consider the API user-facing.

## Reproduction Assessment

Yes. Source inspection on current main shows session identity exists in the runner and `/btw` inputs, but `ProviderCreateStreamFnContext` and `registerProviderStreamForModel` cannot deliver it to provider plugins.

## Solution Assessment

Yes. Optional fields on the existing provider stream context are the narrowest backward-compatible API extension because existing plugins can ignore them while conversation-aware providers key state by `sessionKey` or `sessionId`.
