## Summary

Close as ClawHub-scoped: this is a useful optional model-provider idea, but current main already has the provider-owned custom stream transport seam the proposal needs, and OpenClaw vision/policy routes optional provider plugins to external ownership and ClawHub when no core API gap is shown.

## What This Changes

Request to add a bundled Unix-domain-socket model provider plugin under `extensions/unixsocket/` with a custom `createStreamFn` transport and new provider config.

## Best Possible Solution

Keep core unchanged and publish the Unix-socket model provider as a ClawHub/provider plugin using `createStreamFn`; open a narrow SDK-gap issue only if implementation proves that seam cannot support the protocol.

## Reproduction Assessment

Not applicable; this is a new provider/plugin request, not a current-main bug report. Source inspection confirms the requested custom transport shape can be expressed through the existing provider plugin seam.

## Solution Assessment

No for bundling in core; the maintainable path is an external ClawHub provider plugin because current SDK seams already cover custom StreamFn transports. A core issue would be warranted only for a specific missing SDK/API gap found while building that plugin.
