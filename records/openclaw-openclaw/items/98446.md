## Summary

Keep open. Current main still has the reported mismatch: OpenShell’s plugin config rejects `/workspace`, while outbound media normalization still maps only `/workspace/...` container paths back to the sandbox workspace; no merged fix or canonical duplicate was found.

## What This Changes

The issue reports that OpenShell’s `/sandbox` workspace contract conflicts with media export paths that still assume `/workspace`, breaking startup or outbound media delivery depending on configuration.

## Best Possible Solution

Thread the active sandbox `containerWorkdir` through outbound media normalization and message-action media policy so OpenShell `/sandbox/...` media paths map correctly without relaxing the plugin’s `/sandbox` and `/agent` managed-root validation.

## Reproduction Assessment

Yes from source for both halves: OpenShell rejects `/workspace`, and outbound media normalization only maps `/workspace/...` as a container workspace path. I did not run the live Kubernetes/OpenShell scenario.

## Solution Assessment

Yes, the best fix is a narrow contract repair: use the active sandbox container workdir in media path normalization instead of adding another config knob or broadening OpenShell roots by default.
