## Summary

Keep open: the reset archive fix is plausible, but the PR also makes deleted transcript archives indexable without a matching visibility model after deletion removes the session store key, so it needs changes before merge.

## What This Changes

The PR indexes reset/deleted session transcript archives, maps archive hit paths back to live stems during session visibility filtering, adds focused tests, and updates the changelog.

## Best Possible Solution

Land the reset archive indexing and stem-resolution fix with regression coverage, while keeping deleted archives opaque unless maintainers explicitly approve a deleted-session recovery and visibility model.

## Reproduction Assessment

Yes, from source inspection: current main lists reset/deleted archives as usage-counted but empties archive entries and drops archive hit paths during visibility filtering. I did not run the reset/reindex/search flow because this was a read-only review.

## Solution Assessment

No. The reset half is narrow and maintainable, but indexing `.jsonl.deleted.<iso>` archives needs a reviewed visibility policy or should stay out of this fix.
