## Summary

Keep open. The reset-archive visibility gap is real, but the updated patch still broadens shared archive handling into deleted-session and dreaming-ingestion paths that do not have the same visibility or dedupe model.

## What This Changes

The PR makes reset/deleted session transcript archives produce indexed content, maps archive hit paths back to live transcript stems, adds focused tests, and updates the changelog.

## Best Possible Solution

Land a narrower reset-archive memory-search fix that preserves the existing dreaming archive skip behavior and keeps `.jsonl.deleted.<iso>` archives opaque unless maintainers explicitly define a deleted-session recovery policy.

## Reproduction Assessment

Yes. Source inspection shows current main drops archive hit paths and empties archive entries, while the PR’s changed skip predicate makes reset/deleted archives contentful for every `buildSessionEntry` consumer.

## Solution Assessment

No. The reset stem/indexing direction is narrow enough, but the current patch couples it to deleted archives and dreaming ingestion; splitting those paths or limiting the change to reset archives is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep archive reads out of dreaming ingestion:** `packages/memory-host-sdk/src/host/session-files.ts:74-75`
  - body: `buildSessionEntry` is also used by dreaming ingestion, whose current tests expect archive artifacts not to be re-ingested or double-counted. Letting reset/deleted archives fall through here means archived turns can enter the dreaming corpus under a different path from the live transcript. Please split the memory-indexing path or keep the dreaming skip intact.
  - confidence: 0.88
- **[P2] Keep deleted archives out of searchable session indexing:** `packages/memory-host-sdk/src/host/session-files.ts:74-75`
  - body: This makes `.jsonl.deleted.*` transcripts produce searchable content, but `sessions.delete` removes the store row that the agent visibility guard needs, and CLI memory search does not run that guard. Please leave deleted archives opaque here or add an explicit deleted-session visibility model.
  - confidence: 0.84
