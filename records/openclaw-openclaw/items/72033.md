## Summary

Keep open. Current main still lacks `diagnostics.pricing`, but the submitted diff is not mergeable as-is because it imports a pricing-cache meta helper that is not exported from the target module and leaves the new discoverable Gateway RPC undocumented.

## What This Changes

The PR adds a read-scoped `diagnostics.pricing` Gateway RPC, advertises it in method discovery, scopes it under `operator.read`, and adds diagnostics handler tests for empty and populated pricing-cache metadata.

## Best Possible Solution

Land the additive read-scoped diagnostics RPC after exporting a wrapper that returns real pricing-cache metadata, documenting the Gateway protocol and changelog entry, and keeping broader pricing timeout resilience in the companion resilience work.

## Reproduction Assessment

Not applicable as a feature/observability PR. The verification path is static and high-confidence: current main lacks the handler, advertised method, and scope entry, while the proposed import target is not exported on current main.

## Solution Assessment

Yes, once repaired. A read-scoped diagnostics method is a narrow maintainable way to expose cache state without changing pricing refresh behavior, but the PR needs the current-main export/TTL and documentation contract fixes before merge.
