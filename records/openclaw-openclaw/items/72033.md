## Summary

Keep open: the feature is useful, but the current PR is not mergeable against main because it imports a pricing-cache meta helper that is not exported from that module, and the new public Gateway method still lacks protocol docs and a changelog entry.

## What This Changes

This PR adds a read-scoped `diagnostics.pricing` Gateway RPC, advertises it in method discovery, classifies it under `operator.read`, and tests empty and populated pricing-cache metadata responses.

## Best Possible Solution

Land the read-scoped pricing diagnostics RPC with a `model-pricing-cache.ts` wrapper that returns the real refresh TTL plus protocol docs and an Unreleased changelog entry; leave pricing timeout, disk cache, and circuit-breaker behavior to #75067.

## Reproduction Assessment

Not applicable as a feature/observability PR. The high-confidence verification path is static: current main lacks the RPC, and this PR currently references an unexported cache-meta helper on the current source tree.

## Solution Assessment

Yes in direction, but not as patched. Exposing pricing-cache metadata through the diagnostics handler pattern is the narrowest maintainable visibility fix once the cache-meta export/TTL and public contract documentation are repaired.
