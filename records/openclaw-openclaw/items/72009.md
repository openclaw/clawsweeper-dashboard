## Summary

Keep open: current main still lacks the requested configurable typing TTL, and the PR covers the runtime/schema surface, but the latest diff still describes TTL as a hard per-run maximum even though the controller refreshes the timeout while keepalive ticks continue.

## What This Changes

The PR adds `typingTtlSeconds` to session and agent-default config, wires it into the typing controller, raises the default TTL to 300 seconds, and updates docs, schema, changelog, and focused tests.

## Best Possible Solution

Land the runtime/config/schema change after the docs and generated schema help describe TTL as the stale-refresh timeout since the last successful typing refresh.

## Reproduction Assessment

Yes, by source inspection: current main hardcodes a two-minute TTL, does not pass TTL from config into `createTypingController`, and lacks `typingTtlSeconds` in the relevant schemas. I did not run local tests in this read-only sweep; the latest head has green GitHub check-run proof by API.

## Solution Assessment

No, not quite as written: the runtime/config threading is narrow, but the docs and schema help should describe TTL as the timeout since the last successful typing refresh, not a hard maximum duration for a single run.
