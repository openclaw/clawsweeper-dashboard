## Summary

Keep open. Current main still has the hardcoded two-minute typing TTL and no public `typingTtlSeconds` key, while this PR provides the active implementation path with sufficient real CLI proof and no blocking correctness or security finding found in the reviewed diff.

## What This Changes

The branch adds `typingTtlSeconds` to session and agent-default config, passes it into auto-reply typing, raises the default TTL to 300 seconds, updates docs/tests/changelog, and carries merge-cleanup formatting/test-fixture changes outside the TTL path.

## Best Possible Solution

Land a TTL-focused version if maintainers accept the public `typingTtlSeconds` option and 300-second default, preserving session-first precedence and config/docs/test coverage.

## Reproduction Assessment

Not applicable as a feature/config PR. Source inspection of current main gives a high-confidence baseline: the controller still defaults to two minutes and the public schemas do not expose `typingTtlSeconds`.

## Solution Assessment

Yes for the requested direction: adding the key to the existing session and agent-default schemas and passing a session-first TTL into `createTypingController` is the narrow maintainable implementation. The remaining question is whether maintainers want this public option and default change.
