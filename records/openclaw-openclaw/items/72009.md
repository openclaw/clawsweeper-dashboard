## Summary

Keep this PR open: current main still lacks configurable typing TTL, and the branch mostly wires the right surface, but the new public docs/schema-help wording misstates the TTL as a hard per-run maximum while the controller refreshes it on each keepalive tick. Latest-head CI also is not fully green yet.

## What This Changes

The PR adds `typingTtlSeconds` to session and agent-default config, raises the typing controller default TTL to 300 seconds, and updates schema, docs, changelog, and focused tests.

## Best Possible Solution

Correct the docs/schema help to describe `typingTtlSeconds` as the stale keepalive watchdog, regenerate the config schema, then land after maintainer approval and green latest-head CI.

## Reproduction Assessment

Yes, by source inspection: current main hardcodes the two-minute TTL, does not pass a config-derived TTL into `createTypingController`, and lacks `typingTtlSeconds` in session and agent-default schemas/docs. I did not run tests because this review was read-only.

## Solution Assessment

No, not yet: the runtime wiring is focused, but the public docs/help describe TTL as a hard per-run maximum even though the implementation refreshes it on each keepalive tick. The narrow fix is wording/schema regeneration, not a runtime change.
