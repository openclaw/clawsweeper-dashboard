## Summary

Keep open: current main still lacks the configurable typing TTL, and this PR is the focused implementation, but the new public docs/schema-help wording describes the TTL as a hard per-run cap even though the controller refreshes the timeout on each typing keepalive.

## What This Changes

The PR adds `typingTtlSeconds` to session and agent-default config, raises the default typing TTL to 300 seconds, and updates schema, docs, changelog, and focused tests.

## Best Possible Solution

Correct the docs/schema-help wording to match the refreshed-timeout semantics, then land the focused config/schema/docs/test change after maintainer approval and green exact-head checks.

## Reproduction Assessment

Yes, by source inspection: current main hardcodes a 2-minute `typingTtlMs`, does not pass TTL from config into `createTypingController`, and has no `typingTtlSeconds` config schema/docs key. I did not run tests because this review was read-only.

## Solution Assessment

Not quite as written: the runtime/config threading is narrow, but the public docs and schema help should describe TTL as the timeout since the last successful typing refresh, not the maximum duration for a single run.
