## Summary

Keep open: current main still lacks configurable typing TTL, while this active PR provides a coherent implementation with sufficient CLI proof and green live checks; the remaining gate is maintainer acceptance of the new public config/default.

## What This Changes

The branch adds `typingTtlSeconds` for session and agent defaults, passes it into the typing controller with a 300-second default, updates docs/config metadata/tests/changelog, and carries some merge-cleanup churn outside the TTL path.

## Best Possible Solution

Keep the TTL implementation in the generic auto-reply/config layer and land it after maintainer acceptance of the public option and 300-second default, ideally with unrelated merge-cleanup churn trimmed if maintainers want a narrower diff.

## Reproduction Assessment

Not applicable as a feature/config PR. Current-main source gives a high-confidence baseline: the typing controller still defaults to two minutes and public schemas do not expose `typingTtlSeconds`.

## Solution Assessment

Yes for the requested direction: adding the key to the existing session and agent-default schemas and passing a session-first TTL into `createTypingController` is the narrow maintainable path. The remaining question is whether maintainers want the new public option/default and whether to trim non-TTL churn before landing.
