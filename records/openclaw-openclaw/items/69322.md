## Summary

Keep this PR open: current main still lacks the requested `doctor.memory.status.perAgent` field, and this PR is the canonical implementation path, but the branch is conflicting and has blocking correctness issues plus missing real behavior proof.

## What This Changes

This PR adds a `perAgent` array to `doctor.memory.status`, expands gateway doctor tests, and documents the new protocol field.

## Best Possible Solution

Land an additive current-main implementation that returns the default agent first, collects secondary statuses best-effort with per-entry errors, preserves cached-by-default embedding checks, and updates tests and protocol docs.

## Reproduction Assessment

Yes for the PR defects: source review shows secondary manager acquisition happens before the helper's local catch while callers use `Promise.all`, so a secondary throw can fail the whole RPC. The requested feature itself is not a bug reproduction; current main source shows the RPC is still default-agent only.

## Solution Assessment

No, not as currently written. Adding `perAgent` to the existing status RPC is the narrow direction, but this branch needs acquisition isolation, a current-main rebase that preserves cached status semantics, and real behavior proof before merge.

## Review Findings
