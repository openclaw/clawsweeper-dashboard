## Summary

Keep open. Current main and the latest release still lack the requested per-agent memory status, but this PR is not merge-ready because it is conflicting, lacks real behavior proof, and has source-reviewed correctness blockers.

## What This Changes

This PR adds a `perAgent` array to `doctor.memory.status`, updates gateway doctor tests, and documents the new gateway protocol field.

## Best Possible Solution

Land a rebased additive implementation that returns default-agent status first, reports secondary agents best-effort with per-entry errors, preserves cached-by-default embedding checks, and includes focused tests, protocol docs, and redacted real RPC proof.

## Reproduction Assessment

Yes for the PR defects by source inspection: the branch calls secondary manager acquisition before local error handling and still live-probes embeddings unconditionally, while current main lacks `perAgent`. I did not run tests because this was a read-only review.

## Solution Assessment

No, not as currently written. Adding `perAgent` to the existing status RPC is the narrow maintainable direction, but the branch needs acquisition isolation, a rebase onto current cached status semantics, and real behavior proof before merge.
