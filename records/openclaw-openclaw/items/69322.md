## Summary

Keep this PR open because current main still lacks the requested per-agent memory status field, but the branch is not merge-ready: it is conflicting, lacks contributor real-behavior proof, and has two source-level correctness issues against current main semantics.

## What This Changes

This PR adds a `perAgent` array with `{ agentId, dirty, files, chunks, error? }` to `doctor.memory.status`, adds gateway handler tests, and updates the protocol docs.

## Best Possible Solution

Land a rebased additive implementation that returns default-agent status first, reports secondary agents best-effort with per-entry errors, preserves cached-by-default embedding checks, and includes focused tests, protocol docs, and redacted real RPC proof.

## Reproduction Assessment

Yes for the PR defects by source inspection: the PR head calls secondary manager acquisition before local error handling and live-probes embeddings unconditionally, while current main still lacks `perAgent`. I did not run tests because this is a read-only review.

## Solution Assessment

No, not as currently written. Adding `perAgent` to the existing status RPC is the narrow maintainable direction, but the branch needs acquisition isolation, a rebase onto current cached status semantics, and real behavior proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

  - confidence: 0.9
