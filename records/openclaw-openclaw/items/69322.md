## Summary

Keep this PR open. Current main still lacks the requested per-agent `doctor.memory.status` payload, so the PR is not obsolete, but the proposed helper does not isolate manager acquisition failures per agent and the branch needs to be refreshed against current main before merge.

## What This Changes

The PR adds a `perAgent` array to `doctor.memory.status`, updates gateway doctor tests, and documents the new protocol field.

## Best Possible Solution

Revise the PR so `doctor.memory.status` remains additive, returns the default agent first, converts each non-default acquisition/status failure into that agent's `perAgent` entry, preserves current main's cached-by-default embedding behavior, and includes tests, docs, and changelog coverage for the new RPC field.

## Reproduction Assessment

Yes. Mock `getActiveMemorySearchManager` to throw for a secondary agent while the default manager succeeds; the PR's `collectMemoryAgentStatus` awaits acquisition before its `try/catch`, so the surrounding `Promise.all` rejects instead of returning an errored `perAgent` entry.

## Solution Assessment

No, not as currently written. Adding `perAgent` to the existing status method is the narrow right direction, but the implementation needs per-entry acquisition isolation and a current-main rebase before it is the maintainable fix.
