Land an additive current-main implementation that returns the default agent first, collects secondary statuses best-effort with per-entry errors, preserves cached-by-default embedding checks, and includes focused tests, protocol docs, and redacted real RPC proof.

Yes for the PR defects: source review shows secondary manager acquisition happens before the helper's local catch while callers use `Promise.all`, and the branch's handler still live-probes embeddings by default. ## Solution Assessment

No, not as currently written. Adding `perAgent` to the existing status RPC is the narrow direction, but the branch needs acquisition isolation, a current-main rebase that preserves cached status semantics, and real behavior proof before merge.
