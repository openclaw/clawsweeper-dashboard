## Summary

Keep open. Current main still does not expose per-agent memory status, so this PR is not obsolete, but the branch needs changes before merge because it can fail the whole RPC on one secondary-agent acquisition error and is stale against the current cached status path.

## What This Changes

The PR adds a `perAgent` array to the `doctor.memory.status` gateway RPC payload, expands gateway doctor tests for default and multi-agent status, and documents the protocol field.

## Best Possible Solution

Keep the PR open and revise it as an additive current-main change: return the default agent first, collect other configured agents best-effort with per-entry errors, preserve runtime config and explicit deep-probe behavior, and ship protocol docs plus an active-version changelog entry.

## Reproduction Assessment

Yes. Mock `getActiveMemorySearchManager` to throw for a secondary agent while the default manager succeeds; the PR head awaits acquisition before local error handling, so `Promise.all` rejects instead of returning an errored `perAgent` entry.

## Solution Assessment

No, not as currently written. Adding `perAgent` to the existing status RPC is the narrow maintainable direction, but the branch needs acquisition isolation, a current-main rebase that preserves cached status semantics, and release-note coverage.
