## Summary

Keep open. The requested Telegram topic `/status` visibility is useful and not already present on current main, but this PR still needs a plugin-boundary redesign and real Telegram proof before it is mergeable.

## What This Changes

The PR adds Telegram forum-topic routing, configured/live ACP binding, drift, and ACP runtime lines to `/status`, with context plumbing and focused status tests.

## Best Possible Solution

Add a generic channel/plugin-owned `/status` line hook and implement the configured/live/drift ACP topic state inside the Telegram plugin, then verify it with redacted live Telegram topic proof.

## Reproduction Assessment

Yes by source inspection for the feature gap: current main’s `/status` path has no message context or channel-owned status-line hook for Telegram topic routing metadata. I did not run a live Telegram forum-topic reproduction.

## Solution Assessment

No. The visibility feature is useful, but this patch makes shared status code parse Telegram topic grammar directly; a plugin-owned status hook is the narrower maintainable shape.
