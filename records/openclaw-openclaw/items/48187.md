## Summary

Keep open: current main still lacks the Telegram topic `/status` visibility, but this PR is not ready because it puts Telegram-specific routing in shared status code and omits the required changelog entry.

## What This Changes

The PR passes message context into `/status`, adds a status-message slot for channel-owned lines, and renders Telegram forum-topic configured/live ACP binding state with tests.

## Best Possible Solution

Add a generic channel/plugin-owned `/status` line hook and have Telegram render configured/live/drift ACP topic state through the plugin, plus a single-line Unreleased changelog entry.

## Reproduction Assessment

Yes for the feature gap: current main lacks message-context inputs and a channel-owned status-line slot, so `/status` cannot show Telegram topic routing metadata. I verified this by source inspection rather than a live Telegram run.

## Solution Assessment

No. The visibility feature is useful, but the PR hard-codes Telegram parsing and ACP binding lookup under shared `src/status` code instead of using a generic channel/plugin seam.
