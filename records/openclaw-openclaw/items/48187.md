## Summary

Keep open: the requested Telegram topic `/status` visibility is useful and is not implemented on current main, but the patch still puts Telegram routing logic inside shared status rendering and lacks live Telegram proof for the visible topic behavior.

## What This Changes

The PR adds Telegram topic routing metadata to chat `/status`, threads Telegram message context into status rendering, adds a `channelStatusLines` output slot, and updates focused tests/type fixes.

## Best Possible Solution

Expose a generic channel/plugin-owned status-line hook, have Telegram compute the topic routing snapshot at its plugin routing boundary, and merge only after redacted live Telegram `/status` proof.

## Reproduction Assessment

Not applicable: this is a feature PR, not a bug report. Source inspection shows current main lacks a chat `/status` channel-status slot, but there is no failing current-main bug path to reproduce.

## Solution Assessment

No. The display slot is the right direction, but Telegram-specific parsing and binding lookup should be owned by the Telegram plugin and passed through a generic seam.
