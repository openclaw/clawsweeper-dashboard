## What This Changes

The PR adds Telegram forum-topic routing, delivery, configured/live binding, drift, and ACP runtime lines to chat `/status`, plus a generic display slot and focused tests/type fixes.

## Best Possible Solution

Add a generic channel/plugin-owned status-line hook, implement Telegram topic snapshotting inside the Telegram plugin using its existing routing/binding helpers, and merge only after redacted live Telegram `/status` proof.

## Solution Assessment

No. The output slot is useful, but the current approach makes core parse Telegram routing; the safer maintainable path is a generic plugin-owned status hook with Telegram supplying its own topic snapshot.
