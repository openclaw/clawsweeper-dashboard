## Summary

Keep open. The Telegram topic visibility slice is useful, and the author responded to the configured/live/drift state discussion, but the current patch still puts Telegram-specific topic parsing and binding rendering in shared status code and lacks required live Telegram proof.

## What This Changes

The PR adds Telegram forum-topic routing, configured/live ACP binding, drift, and ACP runtime lines to `/status` output.

## Best Possible Solution

Add a generic channel/plugin-owned `/status` line hook and implement the configured/live/drift ACP topic state inside the Telegram plugin, then verify it with redacted live Telegram topic proof.

## Reproduction Assessment

Yes for the feature gap by source inspection: current main has no `/status` input or hook that can render Telegram topic routing metadata. I did not run a live Telegram forum-topic reproduction.

## Solution Assessment

No. The visibility feature is useful, but this patch makes shared status code parse Telegram topic grammar directly; a plugin-owned status hook is the narrower maintainable shape.
