## Summary

Keep open: the visibility feature is useful, but the latest patch still puts Telegram-specific topic parsing and ACP binding lookup in shared status code, and it is missing the required changelog entry before merge.

## What This Changes

The PR passes Telegram command context into `/status`, adds a status-message slot for channel-owned lines, and renders Telegram forum-topic configured/live ACP binding state with tests.

## Best Possible Solution

Keep the visibility feature, but expose a generic channel/plugin-owned status-line hook and implement Telegram configured/live/drift ACP topic rendering inside the Telegram plugin, with a single-line Unreleased changelog entry.

## Reproduction Assessment

Yes for the feature gap: current main lacks message context/command target inputs and has no status-message slot for channel-owned topic routing lines. I verified this by source inspection rather than a live Telegram run.

## Solution Assessment

No. The display slot is a useful start, but the current diff hard-codes Telegram topic parsing and binding lookup in shared status code instead of routing it through a generic channel/plugin seam.
