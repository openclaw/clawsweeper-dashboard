## Summary

Keep open: the visibility goal is useful, but the patch still misses the native slash `/status` path and keeps Telegram routing grammar in shared status code despite the channel ownership boundary. The supplied proof is terminal output from a direct helper call, not a live Telegram `/status` run.

## What This Changes

The branch adds Telegram topic routing, configured/live binding, drift, and ACP runtime lines to `/status` output and threads message context into several status reply callers.

## Best Possible Solution

Add a channel/plugin-owned status-line hook for Telegram topic routing state, thread topic context through every `/status` path including native slash, and verify with redacted live Telegram output.

## Reproduction Assessment

Yes by source inspection: the native slash fast path has the Telegram message context but the PR does not update that caller, while the new topic lines depend on a supplied message context. I did not run a live Telegram reproduction in this read-only review.

## Solution Assessment

No. The status visibility slice is worthwhile, but the maintainable fix should cover native slash `/status` and keep Telegram-specific session grammar behind a channel/plugin-owned seam.
