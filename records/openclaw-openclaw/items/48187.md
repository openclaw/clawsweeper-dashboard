## Summary

Keep open: the requested `/status` visibility is useful and is not already present on current main, but this patch still puts Telegram routing logic in shared status code and needs real Telegram proof before merge.

## What This Changes

The PR adds Telegram topic routing metadata to chat `/status`, threads message context into status rendering, adds a `channelStatusLines` output slot, and updates focused tests/type fixes.

## Best Possible Solution

Expose a generic channel/plugin-owned status-line hook, have Telegram compute the topic routing snapshot with its existing helpers, and merge only after redacted live Telegram `/status` proof.

## Reproduction Assessment

Not applicable: this is a feature PR, not a bug report. Source inspection confirms current main lacks a chat `/status` channel-status slot, but there is no failing current-main behavior to reproduce.

## Solution Assessment

No. The display slot is useful, but Telegram-specific parsing and binding lookup should stay in the Telegram plugin and flow through a generic seam.
