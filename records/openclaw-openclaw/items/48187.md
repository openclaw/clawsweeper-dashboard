## Summary

Keep open. The visibility goal is useful, but current main does not already have this narrow `/status` topic-routing output, and the branch still hardcodes Telegram routing/binding logic in shared status rendering while lacking live Telegram `/status` proof.

## What This Changes

The PR adds Telegram topic routing lines to chat `/status`, threads Telegram message context into status rendering, adds a `channelStatusLines` status slot, and updates focused tests/type checks.

## Best Possible Solution

Expose a generic channel/plugin-owned status-line hook, have Telegram build the topic routing snapshot at its plugin routing boundary, and merge it only with redacted live Telegram `/status` proof.

## Reproduction Assessment

Not applicable: this is a feature PR, not a bug report. Source inspection confirms current main lacks this chat `/status` topic-routing visibility, but there is no failing current-main bug path to reproduce.

## Solution Assessment

No. The visible output is useful, but shared status rendering should receive channel/plugin-owned status lines through a generic seam instead of importing Telegram-specific parsing and binding lookup logic.
