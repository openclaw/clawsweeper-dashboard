## Summary

Close as implemented on current main. The newer macOS update-restart hardening sets a 10s LaunchAgent throttle, adds a 20s shutdown timeout, blocks unsafe in-gateway package updates, refreshes/restarts from the updated install, verifies the running gateway version, and re-bootstraps unloaded LaunchAgents; the fix is current-main-only and not yet in a tagged release.

## What This Changes

The issue reports macOS LaunchAgent downtime after auto-update caused by a one-second launchd throttle, missing shutdown timeout, and bootout-based update/restart paths that can leave the gateway unloaded.

## Best Possible Solution

Keep the current main lifecycle hardening, ship it in the next release, and leave broader launchd crash-loop/backoff policy to narrower follow-up items if maintainers want more than the 10s throttle and shutdown window.

## Reproduction Assessment

Yes. The report gives concrete macOS LaunchAgent/update steps, and source inspection ties the original failure to the one-second throttle, missing shutdown timeout, and update/restart lifecycle paths; I did not run a live macOS launchd repro in this read-only sweep.

## Solution Assessment

Yes. Current main addresses the reported failure mechanisms narrowly in the LaunchAgent plist and package-update restart flow without adding a new config surface; broader launchd policy changes can be evaluated separately.
