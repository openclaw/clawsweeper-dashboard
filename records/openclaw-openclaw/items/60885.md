## Summary

Current main implements the central LaunchAgent auto-update downtime fix: generated plists now use a 10s throttle and 20s shutdown timeout, and package-update restart handling now refreshes, verifies, and recovers the managed service instead of leaving the gateway silently down. The related missing-config retry/log-storm policy remains separate at https://github.com/openclaw/openclaw/issues/66436.

## What This Changes

The issue asks OpenClaw to prevent macOS LaunchAgent auto-updates from leaving the gateway unloaded by changing the plist throttle/timeout policy and hardening fragile restart behavior.

## Best Possible Solution

Keep the current main lifecycle hardening for this auto-update downtime path, and handle remaining crash-loop/backoff policy under the separate missing-config retry report.

## Reproduction Assessment

No against current main. The historical failure path is well described, but current source now renders the requested throttle/timeout values and verifies or recovers package-update LaunchAgent restarts.

## Solution Assessment

Yes. Current main does not implement the exact “no bootout except uninstall” suggestion, but it fixes the observable auto-update downtime path in the owning launchd/update code with focused tests and documented stop semantics.
