## Summary

Current main now implements the central LaunchAgent auto-update hardening: generated plists use a 10s throttle and 20s shutdown timeout, package-update restarts refresh and verify the updated LaunchAgent, and the merged fix shipped in beta after the last stable release.

## What This Changes

The issue asks OpenClaw to prevent macOS LaunchAgent auto-updates from leaving the gateway unloaded by relaxing launchd throttle/shutdown timing and avoiding unrecoverable bootout restart paths.

## Best Possible Solution

Keep the merged macOS update restart hardening and leave broader missing-config respawn/backoff policy to https://github.com/openclaw/openclaw/issues/66436.

## Reproduction Assessment

No live macOS reproduction was run in this read-only Linux checkout. Current main no longer has the original source conditions because the plist throttle/timeout and package-update restart paths have changed.

## Solution Assessment

Yes. The merged fix addresses the reported auto-update downtime mechanism without adding a new config knob, while keeping the broader launchd crash-loop/backoff concern separate.
