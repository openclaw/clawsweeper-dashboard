## Summary

Keep open: current main and v2026.6.6 still lack durable TTS archive support, but the requested `saveAudioTo` behavior is a new opt-in feature/config/API decision with path-safety and privacy implications.

## What This Changes

Request to add an opt-in `saveAudioTo` setting for messages/Talk TTS so synthesized audio can be archived to a local directory during synthesis.

## Best Possible Solution

Design an opt-in generic TTS archive policy or plugin/runtime hook that covers message TTS, Talk, streaming, and telephony with safe path handling, naming, retention, and privacy docs.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection confirms current main and v2026.6.6 do not expose durable TTS audio archiving, only temporary delivery files and Talk base64 responses.

## Solution Assessment

Unclear; the capability is reasonable, but the proposed provider-local `saveAudioTo` field is not clearly the best owner boundary. Maintainers should first choose whether this belongs in generic TTS runtime policy, a plugin/SDK hook, or provider-owned config.
