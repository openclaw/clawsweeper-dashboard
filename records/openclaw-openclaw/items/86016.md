## What This Changes

Request to add configurable `saveAudioTo` paths for Talk and message TTS providers so synthesized audio can be saved to disk while it is generated.

## Best Possible Solution

Define a maintainer-approved archival contract first, then implement it across the owner TTS paths with explicit opt-in config, safe path handling, retention/privacy docs, and regression coverage for both Talk and message TTS flows.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior report. Source and release inspection confirm current OpenClaw has temporary delivery files and Talk base64 responses, but no durable TTS archive option.

## Solution Assessment

Unclear: saving the in-memory audio is plausible, but the proposed provider-local `saveAudioTo` field may not be the best owner boundary. Maintainers should first choose the generic TTS archival policy and storage/export shape.
