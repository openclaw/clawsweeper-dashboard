## Summary

Keep open: the doctor repair boundary is useful, but this patch is not merge-ready because it treats `models.providers` as the full provider registry, can synthesize invalid fallback refs, and lacks real doctor before/after proof.

## What This Changes

Adds a model-catalog helper and core doctor check to detect and repair configured model refs whose provider is absent from `models.providers`, plus unit coverage.

## Best Possible Solution

Revise the repair to validate against the runtime provider/model catalog, rewrite only to concrete known refs or delete optional stale refs, then add redacted real doctor before/after proof.

## Solution Assessment

  - body: `models.providers` is only the custom provider map, so this treats valid built-in refs as orphaned.   - confidence: 0.94
- **[P1] Stop inventing provider/default model refs:** `packages/model-catalog-core/src/configured-model-refs.ts:179`

That can turn a stale ref into `myproxy/default` or `ollama/default`, which current runtime can still report as an unknown model; choose an existing concrete configured/default model ref or leave/delete the stale optional field instead.
