## Summary

Keep this PR open, but it is not merge-ready: the doctor repair is the right general boundary, while this patch can classify valid built-in provider refs as orphaned, can rewrite stale refs to synthetic invalid model ids, and still lacks real before/after doctor proof.

## What This Changes

Adds a model-catalog prune helper, unit tests, and a core doctor check/repair that removes or rewrites configured model refs whose provider is absent from `models.providers`.

## Best Possible Solution

Revise the repair to validate against the runtime provider/model catalog, rewrite only to concrete known refs or delete optional stale refs, and add redacted real doctor before/after proof.

## Reproduction Assessment

Yes, source-level. On the PR branch, a config with `agents.defaults.model.primary: "openai/gpt-5.5"` and no `models.providers.openai` makes the new provider set empty, so the doctor check treats a valid built-in ref as orphaned.

## Solution Assessment

No. Doctor cleanup is the right boundary, but this implementation uses the wrong provider source and can synthesize invalid model refs; a catalog-backed repair is safer.
