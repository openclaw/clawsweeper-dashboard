## What This Changes

Adds a model-catalog prune helper and a core doctor check/repair to detect and rewrite or remove configured model refs whose provider is absent from `models.providers`, with unit coverage.

## Best Possible Solution

Revise doctor repair to validate against runtime/catalog-known provider-model refs, delete optional stale refs, rewrite only to a concrete known fallback, and then add redacted real doctor before/after proof.

## Reproduction Assessment

Yes, source-level. The patch builds its known provider set from only `ctx.cfg.models?.providers`, so a valid config that relies on built-in `openai/gpt-5.5` without a custom provider block would be classified as orphaned.

## Solution Assessment

No. Doctor cleanup is the right boundary, but this implementation uses the wrong provider source and can synthesize invalid model refs; a catalog-backed repair is safer.
