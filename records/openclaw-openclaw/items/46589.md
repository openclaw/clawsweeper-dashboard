## Summary

Keep open. Current main still lacks the PR’s cron chaining fields and still has the specific Control UI cron i18n gaps, so the work is not implemented on main; however, the proposed patch is not ready as-is because the chaining behavior is split across run paths and the locale updates do not follow the generated-locale workflow.

## What This Changes

The PR adds Control UI cron i18n/test-registration fixes plus success/failure cron job chaining across cron runtime, gateway protocol/generated clients, and the Control UI form.

## Best Possible Solution

Land or replace the narrow Control UI i18n fixes with generated locale metadata, then handle pipeline orchestration through either Task Flow or a centralized cron post-run dispatcher that covers scheduled, queued manual, and direct service runs with protocol, docs, and tests updated together.

## Reproduction Assessment

Yes. At PR head, configure onSuccessJobId or onFailureJobId and invoke the public CronService.run()/ops.run() path directly; the diff shows that path returns after finishPreparedManualRun() while chain dispatch is only added around enqueueRun and timer completion paths.

## Solution Assessment

No. The i18n fixes are narrow, but the chaining implementation is not the best solution as-is because post-run behavior is split across entry points and generated locale outputs are hand-maintained instead of regenerated.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Dispatch chains from direct cron runs:** `src/cron/service/ops.ts:550-551`
  - body: `CronService.run()`/`ops.run()` returns a status but never calls `triggerChainedJob`; only queued manual runs and timer completions dispatch the next job. Direct service callers can finish a configured job without firing `onSuccessJobId` or `onFailureJobId`, so the new chaining contract differs by entry point.
  - confidence: 0.87
- **[P2] Regenerate all Control UI locales:** `ui/src/i18n/locales/en.ts:338-343`
  - body: Adding new English cron strings requires regenerating the generated locale bundles and `.i18n` metadata. This patch updates only Spanish and Chinese alongside English, leaving the other generated locale files without the new keys and causing locale drift after rebase.
  - confidence: 0.78
