## Summary

Keep this open. Current main and the latest shipped tag still make the bundled Cloudflare onboarding path Anthropic/Claude-only, while Workers AI model selection remains a provider-surface decision rather than a completed fix.

## What This Changes

Request support for selecting Cloudflare Workers AI models during non-interactive Cloudflare AI Gateway onboarding.

## Best Possible Solution

Add a provider-owned Workers AI endpoint mode/model selector in `extensions/cloudflare-ai-gateway`, or explicitly document the exact custom-provider command as the supported Workers AI onboarding path.

## Reproduction Assessment

Yes. Source inspection on current main and `v2026.5.12` shows the bundled Cloudflare non-interactive path accepts only account/gateway/API-key inputs and generates Anthropic/Claude config, with no Workers AI selector.

## Solution Assessment

Unclear. The requested capability is valid, but maintainers should choose whether the supported path is a provider-owned Workers AI mode or a documented generic custom-provider workflow.
