## Summary

Keep open. Current main still configures the bundled Cloudflare AI Gateway onboarding path as Anthropic/Claude-only and does not expose a Workers AI endpoint or @cf model selector for non-interactive setup.

## What This Changes

Add Workers AI model selection to `openclaw onboard --non-interactive --auth-choice cloudflare-ai-gateway-api-key`.

## Best Possible Solution

Add a provider-owned Workers AI setup mode or model selector in `extensions/cloudflare-ai-gateway` that writes an OpenAI-compatible `/workers-ai/v1` Gateway config with @cf model IDs, plus docs, tests, and a changelog entry.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to trace `cloudflare-ai-gateway-api-key` through `runNonInteractive`, which only consumes account ID, gateway ID, and API key before writing the Anthropic config; no live command was run because this review is read-only.

## Solution Assessment

No. The current custom-provider workaround can express Workers AI manually, but it bypasses the bundled Cloudflare auth-choice flow requested here; the narrow maintainable fix belongs in the Cloudflare provider plugin.
