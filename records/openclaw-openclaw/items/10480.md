## Summary

Current main and the latest release still provide only Anthropic/Claude onboarding for the bundled Cloudflare AI Gateway provider; the requested Workers AI flow remains useful, but its permanent endpoint, auth, and CLI shape requires maintainer product judgment.

## What This Changes

Add non-interactive onboarding support for selecting Cloudflare Workers AI endpoints and `@cf/...` models through the bundled Cloudflare AI Gateway auth choice.

## Best Possible Solution

Define a single provider-plugin-owned Workers AI mode that keeps endpoint family, authentication semantics, model catalog or explicit model ID, generated provider config, and documentation together; alternatively, formally document the exact generic custom-provider command if maintainers decide first-class Cloudflare branding is unnecessary.

## Reproduction Assessment

Yes at source level. Current main and v2026.6.5 expose only the Anthropic endpoint and Claude default through the bundled Cloudflare auth choice, with no Workers AI model or endpoint selector.

## Solution Assessment

Unclear. A plugin-owned Workers AI mode is maintainable, but maintainers must first choose whether it should be a first-class Cloudflare flow or documented use of the existing generic custom-provider onboarding path.
