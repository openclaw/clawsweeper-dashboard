## Summary

Keep open: current main and the latest release still configure the bundled Cloudflare AI Gateway onboarding path as Anthropic/Claude only, while Workers AI setup remains either manual or a generic custom-provider workaround.

## What This Changes

The issue asks for non-interactive Cloudflare AI Gateway onboarding to support Workers AI endpoint/model selection instead of only writing the default Anthropic/Claude provider config.

## Best Possible Solution

A supported Workers AI setup path should live in the Cloudflare provider docs/onboarding, preserve the existing Anthropic Gateway default, and configure Workers AI model IDs through the maintainer-chosen first-class mode or documented custom-provider command.

## Reproduction Assessment

Yes, at source level for this feature gap. Current main accepts only Cloudflare account ID, gateway ID, and API key for the bundled auth choice, then writes a fixed Anthropic/Claude provider config.

## Solution Assessment

Unclear. The proposed model flag is plausible, but maintainers need to choose whether Workers AI should be a first-class Cloudflare mode or an explicitly documented generic custom-provider setup based on Cloudflare's current Gateway contract.
