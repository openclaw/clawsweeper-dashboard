## Summary

Keep open: current main still configures the bundled Cloudflare AI Gateway onboarding path as Anthropic/Claude only, while the requested Workers AI `@cf/...` model selection path remains a real provider setup gap.

## What This Changes

The issue requests Workers AI model selection or endpoint-mode support for non-interactive Cloudflare AI Gateway onboarding.

## Best Possible Solution

Add a provider-owned Workers AI endpoint mode or model selector in `extensions/cloudflare-ai-gateway`, with docs and focused config-generation tests, while preserving the current Anthropic default.

## Reproduction Assessment

Yes, at source level. Current main shows the Cloudflare non-interactive onboarding path accepts only account ID, gateway ID, and API key, then writes a fixed Anthropic/Claude provider config.

## Solution Assessment

Unclear. The proposed `--model` shape is plausible, but maintainers need to choose whether Workers AI should be a first-class Cloudflare mode or an explicitly documented generic custom-provider setup.
