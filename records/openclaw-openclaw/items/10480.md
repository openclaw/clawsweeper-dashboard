## Summary

Keep open: current main and the latest release still configure Cloudflare AI Gateway onboarding for Anthropic Messages only, while the requested Workers AI model-selection path remains missing and needs a provider/API-shape product decision.

## What This Changes

The issue requests non-interactive onboarding support for selecting Cloudflare Workers AI models through Cloudflare AI Gateway.

## Best Possible Solution

Keep the issue open until maintainers choose the supported Cloudflare Workers AI shape, then implement it inside `extensions/cloudflare-ai-gateway` with docs and focused onboarding/config tests, or document the exact custom-provider command as the supported path.

## Reproduction Assessment

Yes, source-level. Current main exposes only Cloudflare account/gateway/API key inputs and writes an Anthropic Messages default, with no Workers AI model or endpoint selector in the bundled non-interactive flow.

## Solution Assessment

Unclear. The request is valid, but a plain `--model` passthrough may not be the best fix because Workers AI also changes endpoint, API family, gateway header, and credential semantics; maintainers should choose explicit Workers AI mode versus a documented custom-provider path.
