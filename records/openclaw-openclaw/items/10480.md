## Summary

Keep open: current main still configures the bundled Cloudflare AI Gateway onboarding path as an Anthropic/Claude provider, while the requested Workers AI model-selection path remains missing.

## What This Changes

Add support for selecting Cloudflare Workers AI models during non-interactive Cloudflare AI Gateway onboarding.

## Best Possible Solution

Add a provider-owned Workers AI endpoint mode or model selector in `extensions/cloudflare-ai-gateway`, with docs and focused tests for the generated OpenAI-compatible Workers AI provider config; if maintainers prefer the generic custom-provider route, document the exact supported command before closing.

## Reproduction Assessment

Yes. Source inspection on current main shows the Cloudflare non-interactive onboarding path accepts account ID, gateway ID, and API key only, then writes a fixed Anthropic/Claude provider config.

## Solution Assessment

Unclear. The requested `--model` shape is plausible, but the best maintainable solution needs maintainer choice between a Cloudflare Workers AI mode and an explicitly documented generic custom-provider route.
