## Summary

Keep this open: current main and the latest shipped tag still configure the bundled Cloudflare AI Gateway onboarding path as Anthropic/Claude only, while the requested Workers AI `/workers-ai/v1` or compatible model-selection path remains absent and needs a maintainer product/API decision.

## What This Changes

Add a Cloudflare Workers AI endpoint/model selection path to non-interactive onboarding for `cloudflare-ai-gateway-api-key`.

## Best Possible Solution

Add a plugin-owned Workers AI onboarding mode/model selector, or explicitly document the exact custom-provider command, so users can generate `/workers-ai/v1` OpenAI-compatible config with `@cf/...` model ids without manual config edits.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: the Cloudflare non-interactive auth choice accepts only account id, gateway id, and API key, then writes Anthropic/Claude config rather than Workers AI config.

## Solution Assessment

Unclear until maintainers choose the product contract. A Cloudflare plugin-owned Workers AI mode is likely the narrowest maintainable direction; a generic core shim or silently changing the existing Anthropic default would be riskier.
