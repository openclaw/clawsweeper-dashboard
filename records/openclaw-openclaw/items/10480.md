## Summary

Keep open: current main still shows Cloudflare AI Gateway onboarding as an Anthropic/Claude setup path, while Workers AI model selection remains an unsolved provider onboarding feature that needs a maintainer product choice.

## What This Changes

The issue asks non-interactive Cloudflare AI Gateway onboarding to accept or select Cloudflare Workers AI models instead of only writing the default Anthropic/Claude provider config.

## Best Possible Solution

Add a provider-owned Workers AI endpoint mode or model selector in `extensions/cloudflare-ai-gateway` with docs and focused config-generation tests, while preserving the current Anthropic default; if maintainers prefer the generic custom-provider route, publish the exact Workers AI command as the supported path before closing.

## Reproduction Assessment

Yes. Source inspection on current main shows the Cloudflare non-interactive onboarding path accepts account ID, gateway ID, and API key only, then writes a fixed Anthropic/Claude provider config.

## Solution Assessment

Unclear. The requested `--model` shape is plausible, but the maintainable fix needs a product choice between a first-class Cloudflare Workers AI mode/model selector and an explicitly documented generic custom-provider route.
