## Summary

Keep open. Current main still configures the bundled Cloudflare AI Gateway onboarding path as Anthropic/Claude-only, so the requested Workers AI model selection flow remains missing.

## What This Changes

The issue requests Workers AI model selection support for `openclaw onboard --non-interactive --auth-choice cloudflare-ai-gateway-api-key`.

## Best Possible Solution

Add provider-owned Workers AI setup support in `extensions/cloudflare-ai-gateway` that preserves the Anthropic default, writes `/workers-ai/v1` OpenAI-compatible config for selected `@cf/...` models, and documents the supported command.

## Reproduction Assessment

Yes for the reported gap. Source inspection of the current `cloudflare-ai-gateway-api-key` non-interactive path shows it accepts only account ID, gateway ID, and API key, then writes Anthropic config without Workers AI model or endpoint input.

## Solution Assessment

Unclear on the exact interface. The maintainable direction is provider-local, but maintainers still need to choose whether to expose a model selector, endpoint mode, separate auth choice, or a documented custom-provider route.
