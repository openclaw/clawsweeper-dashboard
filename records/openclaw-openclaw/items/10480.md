## Summary

Keep open. Current main still configures the bundled Cloudflare AI Gateway onboarding path as Anthropic/Claude-only, and the generic custom-provider workaround does not satisfy the requested Workers AI model selection flow.

## What This Changes

Add non-interactive Cloudflare AI Gateway onboarding support for selecting/configuring Workers AI `/workers-ai/v1` models instead of only the built-in Anthropic/Claude default.

## Best Possible Solution

Add a provider-owned Workers AI setup mode/model selector that preserves the current Anthropic default, writes the correct `/workers-ai/v1` OpenAI-compatible provider config, and documents the supported non-interactive command.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to trace `cloudflare-ai-gateway-api-key` through `runNonInteractive`: current main consumes account ID, gateway ID, and API key, then writes Anthropic config without any Workers AI model or endpoint input.

## Solution Assessment

Yes on direction. The maintainable fix is provider-local Cloudflare onboarding support; the existing custom-provider workaround can express Workers AI manually but bypasses the bundled auth-choice flow requested here.
