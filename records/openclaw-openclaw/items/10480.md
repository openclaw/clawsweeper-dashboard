## Summary

Keep open: current main and the latest release still configure the bundled Cloudflare AI Gateway onboarding path as Anthropic/Claude only; the generic custom-provider path is a workaround, but it is not the requested or documented Workers AI setup surface.

## What This Changes

The issue requests a way to select Cloudflare Workers AI models during `openclaw onboard --non-interactive --auth-choice cloudflare-ai-gateway-api-key`.

## Best Possible Solution

Add a provider-owned Workers AI endpoint mode or model selector in `extensions/cloudflare-ai-gateway` with docs and focused config-generation tests, while preserving the current Anthropic default; if maintainers choose the generic custom-provider route, document the exact Workers AI command before closing.

## Reproduction Assessment

Yes, at source level. Current main accepts only account ID, gateway ID, and API key for bundled Cloudflare onboarding, then writes a fixed Anthropic/Claude provider config.

## Solution Assessment

Unclear. The proposed model flag is plausible, but maintainers need to choose whether Workers AI should be a first-class Cloudflare mode or an explicitly documented generic custom-provider setup.
