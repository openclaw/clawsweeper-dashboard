## Summary

Keep this open: current main and the latest release still only onboard Cloudflare AI Gateway as an Anthropic/Claude route, while the requested Workers AI `/workers-ai/v1` model-selection path remains missing and needs maintainer product/API direction.

## What This Changes

The issue requests non-interactive Cloudflare AI Gateway onboarding support for selecting or generating Workers AI `@cf/...` models.

## Best Possible Solution

Add a plugin-owned Workers AI onboarding mode/model selector, or explicitly document the exact custom-provider command, so users can generate `/workers-ai/v1` OpenAI-compatible config with `@cf/...` model ids without manual config surgery.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: the Cloudflare non-interactive auth choice accepts only account id, gateway id, and API key, then writes Anthropic/Claude config.

## Solution Assessment

Unclear until maintainers choose the product contract. A Cloudflare plugin-owned Workers AI mode is likely the narrow direction; changing the existing Anthropic default or adding a generic core shim would be less maintainable.
