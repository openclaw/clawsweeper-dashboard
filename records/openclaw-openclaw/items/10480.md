## Summary

Keep open: current main and v2026.6.10 still generate only the Anthropic Cloudflare AI Gateway provider during non-interactive onboarding, so the Workers AI model-selection request remains unresolved. ## What This Changes

Request support for selecting Cloudflare Workers AI models during non-interactive Cloudflare AI Gateway onboarding.

## Best Possible Solution

Define a provider-owned Cloudflare endpoint-family and model-selection contract that can generate Workers AI `@cf/...` config while preserving the current Anthropic Gateway setup, then align docs and focused onboarding tests.

## Reproduction Assessment

Yes, at source level: the Cloudflare non-interactive auth choice has no Workers AI model or endpoint option and always applies the Anthropic provider config. I did not run onboard because this was a read-only cleanup review.

## Solution Assessment

Unclear. The user need is valid, but maintainers need to choose whether the durable solution is a first-class Workers AI mode, the broader Cloudflare REST bridge, or a documented custom-provider workflow.
