## Summary

Keep open. Current main and v2026.5.12 still configure `cloudflare-ai-gateway-api-key` onboarding for the Anthropic/Claude Gateway route only, while the requested Workers AI `@cf/...` model-selection flow remains unsupported and needs a maintainer UX/API decision.

## What This Changes

Request support for selecting Cloudflare Workers AI `@cf/...` models during non-interactive Cloudflare AI Gateway onboarding.

## Best Possible Solution

Add provider-owned Workers AI setup in `extensions/cloudflare-ai-gateway` that preserves the current Anthropic default, writes an OpenAI-compatible Workers AI provider config for selected `@cf/...` models, and documents the supported non-interactive command with focused tests.

## Reproduction Assessment

Yes for the product gap. Source inspection on current main shows the bundled non-interactive Cloudflare auth path accepts account ID, gateway ID, and API key, then writes Anthropic/Claude provider config without Workers AI model or endpoint input.

## Solution Assessment

Unclear on the exact interface. The maintainable boundary is provider-local, but maintainers need to choose whether the supported path is a Cloudflare model selector, endpoint mode, separate auth choice, or documented custom-provider command.
