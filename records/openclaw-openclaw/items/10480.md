## Summary

Keep open. Current main still configures the bundled Cloudflare AI Gateway onboarding path as Anthropic/Claude-only, with no Workers AI endpoint mode or @cf model selector for non-interactive setup.

## What This Changes

Add Workers AI model selection to non-interactive Cloudflare AI Gateway onboarding so it can write a /workers-ai/v1 OpenAI-compatible config with @cf model IDs.

## Best Possible Solution

Add a Cloudflare provider-owned Workers AI setup mode/model selector that preserves the current Anthropic default, writes the correct /workers-ai/v1 provider config, and documents the supported non-interactive command.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to trace cloudflare-ai-gateway-api-key through runNonInteractive: current main consumes account ID, gateway ID, and API key, then writes the Anthropic config without any Workers AI model or endpoint input.

## Solution Assessment

No. The current custom-provider workaround can express Workers AI manually, but it bypasses the bundled Cloudflare auth-choice flow requested here; the narrow maintainable fix belongs in the Cloudflare provider plugin.

## Repair Work Prompt

Implement support for Workers AI model selection during `openclaw onboard --non-interactive --auth-choice cloudflare-ai-gateway-api-key`. Keep the fix provider-local to `extensions/cloudflare-ai-gateway` plus the onboard CLI/type surface. Prefer a conservative explicit UX such as `--cloudflare-ai-gateway-provider workers-ai` (defaulting to the existing Anthropic path when omitted) and `--cloudflare-ai-gateway-model <@cf/...>` for the selected Workers AI model. When Workers AI is selected, generate `models.providers.cloudflare-ai-gateway` with base URL `https://gateway.ai.cloudflare.com/v1/<account>/<gateway>/workers-ai/v1`, `api: "openai-completions"`, a model definition for the selected @cf ID, and matching default/allowlist config. Preserve current Anthropic/Claude behavior when no Workers AI mode/model is requested. Update `docs/providers/cloudflare-ai-gateway.md`, add focused non-interactive/provider tests, and add a single-line CHANGELOG.md entry. Do not change unrelated providers, generic core model selection, dependency pins, or the existing Anthropic stream-wrapper behavior. Stop for maintainer input if the implementation would require a new generic plugin SDK/onboard API rather than provider-owned flags.
