## Summary

Keep this open. Current main and the latest shipped tag still route the bundled Cloudflare AI Gateway onboarding path to Anthropic/Claude only, while the requested Workers AI model selection remains an unresolved provider/onboarding product decision.

## What This Changes

Add non-interactive Cloudflare AI Gateway onboarding support for selecting Workers AI `@cf/...` model IDs instead of always generating Anthropic/Claude config.

## Best Possible Solution

Add a provider-owned Workers AI endpoint mode/model selector in `extensions/cloudflare-ai-gateway`, or explicitly document the exact custom-provider command as the supported Workers AI onboarding path.

## Reproduction Assessment

Yes. Source inspection on current main and `v2026.5.12` shows the bundled Cloudflare non-interactive path accepts only account/gateway/API-key inputs and generates Anthropic/Claude config, with no Workers AI selector.

## Solution Assessment

Unclear. The requested capability is valid, but maintainers should decide whether the supported solution is a Cloudflare provider mode or a documented custom-provider flow.
