## Summary

Keep this open: current main still configures the bundled Cloudflare AI Gateway onboarding path for Anthropic/Claude only, while the requested Workers AI model selection path remains absent. The external moltworker thread was closed in favor of tracking this upstream, and no current PR or canonical issue supersedes the remaining work.

## What This Changes

Add a way for non-interactive Cloudflare AI Gateway onboarding to select or generate Workers AI models such as `@cf/...` instead of only using the bundled Anthropic default.

## Best Possible Solution

Add a plugin-owned Workers AI onboarding mode or non-interactive selector in `extensions/cloudflare-ai-gateway`, generate an OpenAI-compatible Workers AI provider config with `@cf/...` models, and update Cloudflare provider docs plus focused onboarding tests.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: the current Cloudflare non-interactive auth choice always resolves account, gateway, and key, then writes the Anthropic/Claude config. I did not run a live Workers AI onboarding flow because the missing selector is visible in the current CLI/options/plugin source.

## Solution Assessment

No final fix is present yet. The narrowest maintainable direction is a Cloudflare plugin-owned Workers AI mode or documented custom-provider decision, not a core-level shim or silent change to the existing Anthropic default.
