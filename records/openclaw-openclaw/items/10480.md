## Summary

Keep this open: current main and the latest release still configure the bundled Cloudflare AI Gateway onboarding path for Anthropic/Claude only, while Workers AI model selection remains an unresolved provider onboarding contract.

## What This Changes

Support selecting Cloudflare Workers AI `@cf/...` models during `openclaw onboard --non-interactive --auth-choice cloudflare-ai-gateway-api-key`.

## Best Possible Solution

Define one provider-owned Cloudflare onboarding contract that selects endpoint family and model, generates Workers AI/OpenAI-compatible config for `@cf/...` ids, preserves existing Anthropic setups, and documents scripted behavior.

## Reproduction Assessment

Yes, at source level: non-interactive Cloudflare onboarding has no Workers AI model or endpoint option and always applies the Anthropic provider config.

## Solution Assessment

Unclear. A provider-owned Workers AI mode is likely the right boundary, but maintainers need to choose whether it belongs in the bundled Cloudflare auth choice or in documented custom-provider setup coordinated with the REST API direction.
