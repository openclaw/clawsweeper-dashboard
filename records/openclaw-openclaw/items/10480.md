## Summary

Keep open: current main still has no Workers AI model selection for the bundled Cloudflare AI Gateway non-interactive onboarding path, and the linked moltworker thread explicitly upstreamed this gap here.

## What This Changes

Request support for selecting Cloudflare Workers AI `@cf/...` models during non-interactive Cloudflare AI Gateway onboarding.

## Best Possible Solution

Keep this issue open until maintainers choose and document the supported path: preferably a provider-local Workers AI mode/model selector that writes an OpenAI-compatible `/workers-ai/v1` provider config while preserving the current Anthropic default.

## Reproduction Assessment

Yes, this is source-reproducible on current main: the Cloudflare non-interactive auth choice accepts account id, gateway id, and API key only, then writes an Anthropic Claude default. I did not run onboarding because this review is read-only, but the code path is direct.

## Solution Assessment

Unclear until the product decision is made. A provider-local Workers AI mode/model selector is the narrow maintainable fix if bundled Cloudflare onboarding should own this flow; otherwise the exact custom-provider command should be documented before closing.
