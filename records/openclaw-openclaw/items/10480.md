## Summary

Keep open: current main and v2026.5.22 still configure the bundled Cloudflare AI Gateway onboarding path as Anthropic/Claude only, while Workers AI remains a manual or generic custom-provider setup rather than the requested supported Cloudflare auth-choice flow.

## What This Changes

Requested support for choosing Cloudflare Workers AI models during non-interactive onboarding with the bundled Cloudflare AI Gateway auth choice.

## Best Possible Solution

Preserve the existing Anthropic Gateway default and add or document a supported Workers AI setup path in the Cloudflare provider onboarding/docs, using the maintainer-chosen endpoint mode and model-id contract.

## Reproduction Assessment

Yes, at source level for this feature gap. Current main accepts only Cloudflare account ID, gateway ID, and API key for the bundled auth choice, then writes a fixed Anthropic/Claude provider config.

## Solution Assessment

Unclear. The proposed model flag is plausible, but maintainers need to choose whether Workers AI should be a first-class Cloudflare mode or an explicitly documented generic custom-provider setup based on Cloudflare's current Gateway contract.
