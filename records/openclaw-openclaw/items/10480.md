## Summary

Keep open: current main still does not implement Workers AI model selection for the bundled Cloudflare AI Gateway non-interactive onboarding path, and the remaining choice is the supported product/API shape rather than stale cleanup.

## What This Changes

The issue requests a non-interactive Cloudflare AI Gateway onboarding path that can choose Workers AI `@cf/...` models or a Workers AI endpoint mode.

## Best Possible Solution

Add a provider-owned Workers AI endpoint mode or model selector in `extensions/cloudflare-ai-gateway`, with docs and focused config-generation tests, while preserving the current Anthropic default.

## Reproduction Assessment

Yes, at source level. Current main accepts only account ID, gateway ID, and API key for bundled Cloudflare onboarding, then writes a fixed Anthropic/Claude provider config.

## Solution Assessment

Unclear. The proposed `--model` shape is plausible, but maintainers need to choose whether Workers AI should be a first-class Cloudflare mode or an explicitly documented generic custom-provider setup.
