## Summary

Keep open: current main and v2026.6.8 still expose only the Anthropic-shaped Cloudflare AI Gateway onboarding path, so Workers AI endpoint/model selection remains missing and needs maintainer product direction.

## What This Changes

Request support for selecting Cloudflare Workers AI models during non-interactive Cloudflare AI Gateway onboarding.

## Best Possible Solution

Define a provider-owned Cloudflare Workers AI/REST endpoint-family and model-selection contract that can generate `@cf/...` OpenAI-compatible config, preserve the current Anthropic setup, and update docs plus focused onboarding tests.

## Reproduction Assessment

Yes, at source level: non-interactive Cloudflare onboarding has no Workers AI model or endpoint option and always applies the Anthropic provider config. I did not run onboard because this was a read-only cleanup review.

## Solution Assessment

Unclear. The requested behavior is valid, but maintainers need to choose whether it belongs as a bundled Workers AI mode, a broader Cloudflare REST/API direction, or a documented custom-provider path.
