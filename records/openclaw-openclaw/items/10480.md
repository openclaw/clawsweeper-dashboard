## Summary

Keep open. Current main and shipped v2026.5.12 still configure the bundled Cloudflare AI Gateway onboarding path for Anthropic/Claude only; the requested Workers AI model selection remains a valid enhancement and prior maintainer/Codex comments already identified the same remaining gap.

## What This Changes

Request to let non-interactive Cloudflare AI Gateway onboarding select Workers AI models or generate a Workers AI provider config.

## Best Possible Solution

Add provider-owned Workers AI setup in `extensions/cloudflare-ai-gateway`, preserving the Anthropic default while documenting and testing the supported non-interactive Workers AI path.

## Reproduction Assessment

Yes for the product gap. Source inspection on current main shows the bundled Cloudflare non-interactive auth path accepts account ID, gateway ID, and API key, then writes Anthropic/Claude provider config without Workers AI model or endpoint input.

## Solution Assessment

Unclear on the exact public interface. The maintainable boundary is provider-local, but maintainers need to choose whether the supported path is a model selector, endpoint mode, separate auth choice, or documented custom-provider command.
