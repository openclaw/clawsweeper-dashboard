## Summary

Keep open: current main and v2026.6.8 still route Cloudflare AI Gateway onboarding to the Anthropic/Claude setup only, so Workers AI model selection remains missing and overlaps with an unresolved Cloudflare REST/API product decision.

## What This Changes

The issue requests a non-interactive onboarding path for selecting Cloudflare Workers AI `@cf/...` models under the `cloudflare-ai-gateway-api-key` auth choice.

## Best Possible Solution

Define a provider-owned Cloudflare endpoint-family and model-selection contract that can generate Workers AI `@cf/...` config while preserving the current Anthropic Gateway setup, then align docs and focused onboarding tests.

## Reproduction Assessment

Yes, at source level: the Cloudflare non-interactive auth choice has no Workers AI model or endpoint option and always applies the Anthropic provider config. I did not run onboard because this was a read-only cleanup review.

## Solution Assessment

Unclear. The user need is valid, but maintainers need to choose whether the durable solution is a first-class Workers AI mode, the broader Cloudflare REST `/ai/v1` direction, or a documented custom-provider workflow.
