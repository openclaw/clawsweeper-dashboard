## Summary

Keep open. Current main still implements the bundled Cloudflare AI Gateway onboarding path as Anthropic/Claude-only, while this issue asks for a supported Workers AI model path during non-interactive onboarding.

## What This Changes

Request to let non-interactive Cloudflare AI Gateway onboarding select or generate Workers AI models instead of requiring manual Workers AI config edits.

## Best Possible Solution

Add provider-owned Workers AI setup in extensions/cloudflare-ai-gateway that preserves the current Anthropic default, writes an OpenAI-compatible Workers AI provider config for selected @cf/... models, and documents the supported non-interactive command with focused tests.

## Reproduction Assessment

Yes for the product gap. Source inspection of current main shows the bundled non-interactive Cloudflare auth path accepts account ID, gateway ID, and API key, then writes Anthropic config without Workers AI model or endpoint input.

## Solution Assessment

Unclear on the exact interface. The maintainable implementation boundary is provider-local, but maintainers need to choose whether the supported path is a Cloudflare model selector, endpoint mode, separate auth choice, or documented custom-provider command.
