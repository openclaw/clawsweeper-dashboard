## Summary

Keep this issue open: current main still configures the bundled Cloudflare AI Gateway auth choice exclusively for Anthropic, while Workers AI requires manual or generic custom-provider configuration. A newer REST API proposal overlaps the provider design but does not yet replace this concrete onboarding requirement.

## What This Changes

Add Workers AI endpoint and model selection support to non-interactive Cloudflare AI Gateway onboarding.

## Best Possible Solution

Define one provider-owned Cloudflare onboarding contract that selects the endpoint family and model, generates the appropriate OpenAI-compatible Workers AI configuration for `@cf/...` IDs, preserves existing Anthropic setups, and documents both scripted and interactive behavior.

## Reproduction Assessment

Yes, at source level: run non-interactive onboarding with the Cloudflare auth choice and inspect the generated configuration; the code always selects the Anthropic endpoint, API, and Claude default and has no Workers AI input.

## Solution Assessment

Unclear. A provider-owned Workers AI mode is the narrowest first-class solution, but maintainers must coordinate it with the REST API proposal and decide whether Workers AI belongs under the bundled auth choice or the documented generic custom-provider path.
