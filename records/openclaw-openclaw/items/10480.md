## Summary

Keep this open: current main and the latest release still configure the bundled Cloudflare AI Gateway onboarding path for Anthropic/Claude only, while the requested Workers AI model selection remains a provider contract decision.

## What This Changes

Add a way for non-interactive Cloudflare AI Gateway onboarding to select or generate Cloudflare Workers AI model configuration.

## Best Possible Solution

Define one provider-owned Cloudflare onboarding contract that selects endpoint family and model, generates the correct Workers AI/OpenAI-compatible config for `@cf/...` ids, preserves existing Anthropic setups, and documents both scripted and interactive behavior.

## Reproduction Assessment

Yes, at source level: non-interactive Cloudflare onboarding has no Workers AI model or endpoint option and always applies the Anthropic provider config.

## Solution Assessment

Unclear. A provider-owned Workers AI mode is likely the right boundary, but maintainers need to choose whether it belongs in the bundled Cloudflare auth choice or in documented custom-provider setup coordinated with the REST API proposal.
