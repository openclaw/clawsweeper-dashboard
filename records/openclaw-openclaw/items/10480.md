## Summary

Keep open. Current main still configures `cloudflare-ai-gateway-api-key` onboarding as an Anthropic/Claude path and does not expose a bundled Workers AI endpoint or `@cf/...` model selector for non-interactive onboarding.

## What This Changes

Add a way for `openclaw onboard --non-interactive --auth-choice cloudflare-ai-gateway-api-key` to configure Cloudflare Workers AI models through AI Gateway.

## Best Possible Solution

Add a provider-owned Workers AI setup path that preserves the existing Anthropic default, accepts a Workers AI `@cf/...` model for non-interactive onboarding, writes an OpenAI-compatible Cloudflare AI Gateway provider config, and documents the scripted command.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to trace `cloudflare-ai-gateway-api-key` through `runNonInteractive`, which only consumes account ID, gateway ID, and API key before writing the Anthropic config. No live command was run because this review is read-only.

## Solution Assessment

No. The current custom-provider workaround can express a Workers AI endpoint manually, but it does not solve the bundled Cloudflare auth-choice onboarding behavior requested here; the narrow maintainable fix belongs in the Cloudflare provider plugin with focused docs and tests.
