## Summary

Keep open: current main still onboards Cloudflare AI Gateway as the Anthropic/Claude path, while Workers AI `@cf/...` model selection remains missing and needs a provider/API product decision.

## What This Changes

Request that `openclaw onboard --non-interactive --auth-choice cloudflare-ai-gateway-api-key` can select Cloudflare Workers AI `@cf/...` models or generate their provider config.

## Best Possible Solution

Add a provider-owned Workers AI endpoint mode or model selector in `extensions/cloudflare-ai-gateway`, with docs and focused config-generation tests, while preserving the current Anthropic default.

## Reproduction Assessment

Yes. Source inspection on current main shows the Cloudflare non-interactive onboarding path accepts only account ID, gateway ID, and API key, then writes a fixed Anthropic/Claude provider config.

## Solution Assessment

Unclear. The proposed `--model` shape is plausible, but maintainers need to choose whether Workers AI should be a first-class Cloudflare mode or an explicitly documented generic custom-provider setup.
