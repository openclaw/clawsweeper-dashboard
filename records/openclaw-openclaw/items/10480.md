## Summary

Keep open. Current main and the latest tagged release still route the bundled Cloudflare AI Gateway onboarding path to Anthropic/Claude only, while the requested Workers AI non-interactive model path remains unsupported.

## What This Changes

Allow non-interactive Cloudflare AI Gateway onboarding to select or populate Cloudflare Workers AI models.

## Best Possible Solution

Add provider-owned Workers AI setup in extensions/cloudflare-ai-gateway that preserves the current Anthropic default, writes an OpenAI-compatible Workers AI provider config for selected @cf/... models, and documents the supported non-interactive command with focused tests.

## Reproduction Assessment

Yes for the product gap. Source inspection shows the bundled non-interactive Cloudflare auth path accepts account ID, gateway ID, and API key, then writes Anthropic config without Workers AI model or endpoint input.

## Solution Assessment

Unclear on the exact interface. The maintainable boundary is provider-local, but maintainers need to choose whether the supported path is a Cloudflare model selector, endpoint mode, separate auth choice, or documented custom-provider command.
