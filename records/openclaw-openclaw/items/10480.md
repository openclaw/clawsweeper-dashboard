## Summary

Keep open. Current main still treats the bundled Cloudflare AI Gateway onboarding path as Anthropic/Claude-only, while the issue asks for a supported Workers AI model path during non-interactive onboarding.

## What This Changes

Add a way for `openclaw onboard --non-interactive --auth-choice cloudflare-ai-gateway-api-key` to select or generate Cloudflare Workers AI `@cf/...` model config.

## Best Possible Solution

Add provider-owned Workers AI setup in `extensions/cloudflare-ai-gateway` that preserves the Anthropic default, writes `/workers-ai/v1` OpenAI-compatible config for selected `@cf/...` models, and documents the supported command and tests.

## Reproduction Assessment

Yes for the reported product gap. Source inspection of the current non-interactive Cloudflare auth path shows it accepts account ID, gateway ID, and API key, then writes Anthropic config without Workers AI model or endpoint input.

## Solution Assessment

Unclear on the exact interface. The maintainable boundary is provider-local, but maintainers still need to choose whether the supported path is a model selector, endpoint mode, separate auth choice, or documented custom-provider command.
