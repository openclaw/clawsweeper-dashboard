## Summary

Keep this open: current main still lacks a Cloudflare Workers AI-specific model or endpoint selection path in non-interactive onboarding, and the bundled Cloudflare AI Gateway auth choice continues to generate an Anthropic/Claude provider config. The issue is concrete and source-reproducible, but the exact onboarding UX and config contract need maintainer product direction before an automated fix PR is safe.

## What This Changes

Request support for selecting Cloudflare Workers AI models during `openclaw onboard --non-interactive --auth-choice cloudflare-ai-gateway-api-key`.

## Best Possible Solution

Add a Cloudflare plugin-owned Workers AI onboarding mode, or explicitly document the custom-provider path, so setup generates the correct `/workers-ai/v1` OpenAI-compatible config with `@cf/...` model ids.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: the Cloudflare non-interactive auth choice has no model or endpoint selector and always writes the Anthropic/Claude Cloudflare config.

## Solution Assessment

Unclear until maintainers choose the product contract. A plugin-owned Workers AI mode is the narrow likely direction, while a core shim or silent change to the current Anthropic default would be harder to maintain.
