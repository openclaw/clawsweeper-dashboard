## Summary

Keep open: current main and v2026.6.6 still expose only account/gateway/API-key Cloudflare onboarding and generate Anthropic/Claude config, so Workers AI model selection remains missing and needs maintainer product direction.

## What This Changes

The issue requests first-class Workers AI model and endpoint selection for non-interactive Cloudflare AI Gateway onboarding instead of post-onboard custom config edits.

## Best Possible Solution

Define a provider-owned Cloudflare endpoint-family and model-selection contract that can generate Workers AI @cf config, preserve the current Anthropic setup, and align docs/tests with the Cloudflare REST/API direction.

## Reproduction Assessment

Yes, at source level: non-interactive Cloudflare onboarding has no Workers AI model or endpoint option and always applies the Anthropic provider config. I did not run onboard because this was a read-only cleanup review.

## Solution Assessment

Unclear. The requested behavior is valid, but maintainers need to choose whether it belongs as a bundled Cloudflare Workers AI mode, as part of the broader REST API direction, or as a documented custom-provider path.
