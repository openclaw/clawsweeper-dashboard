## Summary

Keep open: current main and the latest release still configure the bundled Cloudflare AI Gateway onboarding path as Anthropic/Claude only, while Workers AI remains a custom-provider/manual setup and the current Cloudflare docs make the exact endpoint mode a maintainer/provider decision.

## What This Changes

The issue requests a Cloudflare Workers AI model-selection path for `openclaw onboard --non-interactive --auth-choice cloudflare-ai-gateway-api-key`.

## Best Possible Solution

Preserve the existing Anthropic Gateway default and add or document a supported Workers AI setup path in the Cloudflare provider onboarding/docs using the maintainer-approved endpoint mode and model-id contract.

## Reproduction Assessment

Yes, at source level for this feature gap. Current main accepts only Cloudflare account ID, gateway ID, and API key for the bundled auth choice, then writes a fixed Anthropic/Claude provider config.

## Solution Assessment

Unclear. The requested model flag is plausible, but maintainers need to choose whether Workers AI is a first-class Cloudflare mode, a REST-header-backed provider config, or an explicitly documented generic custom-provider setup.
