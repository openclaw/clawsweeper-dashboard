## Summary

Keep this open: current main still has no Workers AI endpoint/model selection path for the bundled Cloudflare AI Gateway non-interactive onboarding flow, and prior review comments already kept the same concrete gap open for maintainer product direction.

## What This Changes

The issue requests Cloudflare Workers AI model and endpoint selection during `openclaw onboard --non-interactive --auth-choice cloudflare-ai-gateway-api-key`.

## Best Possible Solution

Add a Cloudflare plugin-owned Workers AI onboarding mode, or explicitly document the exact custom-provider setup, so users can generate `/workers-ai/v1` OpenAI-compatible config with `@cf/...` model ids without manual config surgery.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: the Cloudflare non-interactive auth choice has no model or endpoint selector and always writes the Anthropic/Claude Cloudflare config.

## Solution Assessment

Unclear until maintainers choose the product contract. A plugin-owned Workers AI mode is the narrow likely direction; silently changing the existing Anthropic default or adding a core shim would be harder to maintain.
