## Summary

Keep open: current main still onboards Cloudflare AI Gateway as an Anthropic/Claude route only, while the requested Workers AI model-selection path remains missing and the linked moltworker discussion points back to this issue as the upstream tracker.

## What This Changes

The issue requests non-interactive onboarding support for selecting Cloudflare Workers AI models when using the Cloudflare AI Gateway auth choice.

## Best Possible Solution

Add a provider-owned Workers AI endpoint mode or model selector in `extensions/cloudflare-ai-gateway`, update Cloudflare/onboarding docs, and cover the generated OpenAI-compatible provider config with focused tests; if maintainers prefer the existing custom-provider path, document the exact Workers AI command before closing.

## Reproduction Assessment

Yes. Source inspection on current main shows the Cloudflare non-interactive onboarding path accepts account ID, gateway ID, and API key only, then writes a fixed Anthropic/Claude provider config.

## Solution Assessment

Unclear. The requested `--model` shape is plausible, but the best maintainable solution likely needs maintainer choice between a Cloudflare Workers AI mode in the provider plugin and an explicitly documented generic custom-provider route.
