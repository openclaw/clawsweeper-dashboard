## Summary

Keep this open. Current main and the latest shipped tag still route the bundled Cloudflare AI Gateway onboarding path to Anthropic/Claude only, while the requested Workers AI model-selection path remains absent despite a generic custom-provider workaround.

## What This Changes

Request support for selecting Cloudflare Workers AI models during non-interactive onboarding for the bundled `cloudflare-ai-gateway-api-key` auth choice.

## Best Possible Solution

Add a provider-owned Workers AI mode/model selector in `extensions/cloudflare-ai-gateway`, or formally document the exact `custom-api-key` Workers AI command as the supported path before closing.

## Reproduction Assessment

Yes. Source inspection on current main and `v2026.5.12` shows the bundled Cloudflare non-interactive path accepts only account/gateway/API-key inputs and generates an Anthropic/Claude config, with no Workers AI model or endpoint selector.

## Solution Assessment

Unclear. The requested direction is valid, but maintainers should first decide between adding a Cloudflare provider-owned Workers AI mode and blessing the generic custom-provider route with exact documentation.
