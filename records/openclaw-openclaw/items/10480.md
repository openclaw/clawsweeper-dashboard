## Summary

Keep open: current main still has no Workers AI model or endpoint selection for the Cloudflare AI Gateway non-interactive auth choice, and the remaining fix is product-sensitive rather than already implemented or cleanly superseded.

## What This Changes

The issue requests Workers AI model selection during `openclaw onboard --non-interactive --auth-choice cloudflare-ai-gateway-api-key`.

## Best Possible Solution

Define a provider-owned Cloudflare endpoint-family and model-selection contract that can generate Workers AI `@cf/...` config while preserving current Anthropic Gateway behavior, then update docs and focused onboarding tests.

## Reproduction Assessment

Yes, at source level: the Cloudflare non-interactive auth choice has no Workers AI model or endpoint option and always applies the Anthropic provider config. I did not run a credentialed onboard flow because this was a read-only cleanup review.

## Solution Assessment

Unclear. The user need is valid, but maintainers should choose whether the best path is a first-class Workers AI mode, the broader Cloudflare REST bridge, or a documented custom-provider workflow.
