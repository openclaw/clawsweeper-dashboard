## Summary

Keep open. Current main still configures Cloudflare AI Gateway onboarding as Anthropic/Claude-only, so the requested Workers AI model selection path is still missing.

## What This Changes

Add a way for non-interactive Cloudflare AI Gateway onboarding to select or configure Workers AI `@cf/...` models.

## Best Possible Solution

Add provider-owned Workers AI setup support in `extensions/cloudflare-ai-gateway` that preserves the Anthropic default, writes `/workers-ai/v1` OpenAI-compatible config for selected `@cf/...` models, and documents the supported command.

## Reproduction Assessment

Yes for the reported gap. Source inspection of `cloudflare-ai-gateway-api-key` through `runNonInteractive` shows current main accepts only account ID, gateway ID, and API key, then writes Anthropic config without Workers AI model or endpoint input.

## Solution Assessment

Directionally yes, but the exact interface is not settled. The maintainable fix belongs in the Cloudflare provider plugin, with maintainers choosing whether to expose a model selector, endpoint mode, or a documented custom-provider route.
