## Summary

The issue remains necessary as a product/docs decision: current main can encode Workers AI through generic custom-provider onboarding, but the bundled and documented Cloudflare AI Gateway flow still selects only Anthropic/Claude and gives users no Workers AI mode.

## What This Changes

Add a supported way to select Cloudflare Workers AI endpoints and `@cf/...` models during non-interactive onboarding.

## Best Possible Solution

Choose one canonical plugin-owned workflow: preferably document and regression-test an exact Workers AI command through existing custom-provider onboarding, or add an explicit Workers AI endpoint mode to the Cloudflare plugin if maintainers require shared Cloudflare auth/catalog semantics.

## Reproduction Assessment

Yes. Current source provides a high-confidence reproduction: selecting the bundled Cloudflare auth choice always generates the Anthropic endpoint and Claude default, with no Workers AI endpoint or model input.

## Solution Assessment

Unclear. The proposed Cloudflare-specific `--model` flag is plausible but may duplicate current custom-provider onboarding; maintainers should first decide whether a documented generic command or an explicit plugin-owned endpoint mode is the canonical product path.
