## Summary

Current main and the latest release still leave the bundled Cloudflare AI Gateway onboarding flow Anthropic-only, while the generic custom-provider flow can express Workers AI manually; keep this open for a provider-contract decision because first-class Workers AI onboarding remains missing and Cloudflare now offers multiple viable endpoint contracts.

## What This Changes

Request support for selecting Cloudflare Workers AI models during non-interactive onboarding with the bundled Cloudflare AI Gateway auth choice.

## Best Possible Solution

Define one provider-owned Workers AI onboarding contract using the current Cloudflare endpoint strategy, generate the canonical OpenAI-compatible provider/model config with focused fresh-install tests, and document the exact generic custom-provider command until that mode ships.

## Reproduction Assessment

Yes at source level: selecting `cloudflare-ai-gateway-api-key` in non-interactive onboarding can only produce the Anthropic gateway configuration and fixed Claude default, with no Workers AI model or endpoint input.

## Solution Assessment

No, the exact endpoint proposal is no longer clearly the best fix; maintainers should first choose between Cloudflare’s current unified API, legacy Workers AI gateway path, or a documented generic custom-provider flow, while keeping implementation inside the provider plugin.
