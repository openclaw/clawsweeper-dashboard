## Summary

Keep open: current main still supports Cloudflare AI Gateway onboarding only for the Anthropic/Claude route, while the requested Workers AI `@cf/...` model setup remains absent from the bundled Cloudflare auth-choice flow.

## What This Changes

Add support to choose or configure Cloudflare Workers AI `@cf/...` models during non-interactive onboarding for `cloudflare-ai-gateway-api-key`.

## Best Possible Solution

Add a provider-owned Workers AI endpoint mode or model selector in `extensions/cloudflare-ai-gateway`, with docs and focused config-generation tests, while preserving the current Anthropic default; if maintainers choose the generic custom-provider route instead, document the exact Workers AI command before closing.

## Reproduction Assessment

Yes, at source level. Current main shows the Cloudflare non-interactive onboarding path accepts only account ID, gateway ID, and API key, then writes a fixed Anthropic/Claude provider config.

## Solution Assessment

Unclear. The proposed `--model` shape is plausible, but maintainers need to choose whether Workers AI should be a first-class Cloudflare mode or an explicitly documented generic custom-provider setup.
