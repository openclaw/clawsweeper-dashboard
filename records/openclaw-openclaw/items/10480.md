## Summary

Keep open. Current main still configures the bundled Cloudflare AI Gateway onboarding path as Anthropic/Claude-only, and the requested Workers AI model selection flow remains missing.

## What This Changes

The issue requests a way to select Cloudflare Workers AI `@cf/...` models during non-interactive Cloudflare AI Gateway onboarding.

## Best Possible Solution

Add provider-owned Workers AI setup support in `extensions/cloudflare-ai-gateway` that preserves the Anthropic default, writes the `/workers-ai/v1` OpenAI-compatible config for selected `@cf/...` models, and documents the supported non-interactive command.

## Reproduction Assessment

Yes. Source inspection of `cloudflare-ai-gateway-api-key` through `runNonInteractive` shows current main accepts only account ID, gateway ID, and API key, then writes Anthropic config without any Workers AI model or endpoint input.

## Solution Assessment

Yes on direction, but not yet on exact interface. The maintainable fix belongs in the Cloudflare provider plugin rather than generic core policy, with maintainers choosing the CLI/API shape before implementation.
