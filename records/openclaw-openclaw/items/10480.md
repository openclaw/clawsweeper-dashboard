## Summary

Keep open: current main still configures the bundled Cloudflare AI Gateway onboarding path as an Anthropic/Claude route only, while the requested Workers AI model-selection path remains missing and the linked Cloudflare moltworker discussion continues to point here as the upstream tracker.

## What This Changes

Request non-interactive Cloudflare AI Gateway onboarding to select Workers AI models or generate a Workers AI provider config.

## Best Possible Solution

Add a provider-owned Workers AI endpoint mode or model selector in `extensions/cloudflare-ai-gateway`, update Cloudflare and onboarding docs, and cover the generated OpenAI-compatible Workers AI provider config with focused tests; alternatively, document the exact custom-provider command as the supported path before closing.

## Reproduction Assessment

Yes. Source inspection on current main shows the Cloudflare non-interactive onboarding path accepts account ID, gateway ID, and API key only, then writes a fixed Anthropic/Claude provider config.

## Solution Assessment

Unclear. The proposed `--model` shape is plausible, but maintainers need to choose between a Cloudflare Workers AI mode in the provider plugin and an explicitly documented generic custom-provider route.
