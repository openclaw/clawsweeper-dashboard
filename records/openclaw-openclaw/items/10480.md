## Summary

Keep open: current main and latest release still configure the bundled Cloudflare AI Gateway onboarding path as Anthropic Messages with the Claude default, while the linked moltworker thread is asking for Workers AI `@cf/...` model selection. The broader REST API issue overlaps but does not replace this exact onboarding UX decision.

## What This Changes

Request to let non-interactive Cloudflare AI Gateway onboarding select or configure Workers AI `@cf/...` models instead of only the bundled default.

## Best Possible Solution

Choose the supported product path, then either add a Workers AI mode/model selector inside `extensions/cloudflare-ai-gateway` with focused config/catalog/docs tests, or document the exact `custom-api-key` Workers AI command as the supported setup flow before closing.

## Reproduction Assessment

Not applicable as a bug reproduction, but yes for source-level verification: current main's Cloudflare onboarding path only accepts account ID, gateway ID, and API key, then writes an Anthropic/Claude provider config.

## Solution Assessment

Unclear. The requested `--model` shape is plausible, but maintainers need to choose between a bundled Workers AI mode/model selector and documenting the existing custom-provider path as the supported solution.
