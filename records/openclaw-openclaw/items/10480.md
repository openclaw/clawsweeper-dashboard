## Summary

Current main still lacks a supported Workers AI path under the bundled Cloudflare auth choice. The generic custom-provider onboarding surface appears capable of expressing the official Cloudflare contract, but that exact flow is neither documented nor validated, and maintainers must decide whether it is canonical before this issue can close.

## What This Changes

Provide a supported non-interactive onboarding path for selecting Cloudflare Workers AI models routed through AI Gateway.

## Best Possible Solution

Make the existing generic `custom-api-key` onboarding command the documented Workers AI path, add focused generated-config coverage and live validation, and introduce a Cloudflare-specific selector only if that proof exposes behavior the generic seam cannot represent.

## Reproduction Assessment

Yes. Current source deterministically shows that the bundled Cloudflare choice always generates Anthropic/Claude configuration, while the official Workers AI contract requires a different endpoint, compatibility mode, and `@cf/...` model ID.

## Solution Assessment

No, the proposed new bundled selector is not yet proven to be the best solution. The narrower maintainable path is to document, test, and live-validate the existing generic provider onboarding surface before adding parallel Cloudflare-specific CLI options.
