## Summary

Keep open. Current main still treats the bundled Cloudflare AI Gateway onboarding flow as Anthropic/Claude-only, while this issue asks for a supported Workers AI model path during non-interactive onboarding.

## What This Changes

Add a way for `openclaw onboard --non-interactive --auth-choice cloudflare-ai-gateway-api-key` to configure Cloudflare Workers AI endpoint/model selection instead of only the default Anthropic gateway setup.

## Best Possible Solution

Add provider-owned Workers AI setup in `extensions/cloudflare-ai-gateway` that preserves the Anthropic default, writes OpenAI-compatible Workers AI config for selected `@cf/...` models, and documents the supported command with focused tests.

## Reproduction Assessment

Yes for the reported product gap. Source inspection of current main shows the non-interactive Cloudflare auth path accepts account ID, gateway ID, and API key, then writes Anthropic config without Workers AI model or endpoint input.

## Solution Assessment

Unclear on the exact interface. The maintainable boundary is provider-local, but maintainers still need to choose whether the supported path is a model selector, endpoint mode, separate auth choice, or documented custom-provider command.
