## Summary

Keep open. Current main still disables Telegram answer draft streaming whenever non-off `replyToMode` populates the native-quote candidate map, and the latest release tag contains the same guard and test expectation.

## What This Changes

Restore Telegram partial preview streaming for plain replies when `channels.telegram.replyToMode` is non-off, while still disabling streaming for real selected native quotes.

## Best Possible Solution

Restore the quote-only streaming guard in the Telegram dispatcher, keep native quote candidates available for final delivery, add a regression test for `replyToMode: "first"` plus `streaming.mode: "partial"` on a plain inbound message, and add a user-facing changelog entry crediting @choury.

## Reproduction Assessment

Yes. Configure Telegram with non-off `replyToMode` and partial streaming, send a plain text message, and the dispatcher adds the inbound message to `replyQuoteByMessageId` before disabling `canStreamAnswerDraft`; the current test suite already asserts that no draft stream is created.

## Solution Assessment

Yes. The narrowest maintainable fix is to compute `hasNativeQuoteReply` from real selected quote state, such as `replyQuoteText != null && replyQuoteMessageId != null`, while preserving the candidate map passed to final delivery.
