## Summary

Keep open. Current `main` still matches the reported Telegram regression: normal nonblank Telegram messages populate the native-quote candidate map when `replyToMode` is enabled, and the preview-stream guard still disables answer draft streaming for any populated candidate map. The closed, unmerged #73534 remains useful source context; open #39905 is broader scoped-configuration work, not a landed fix for this bug.

## What This Changes

Fix Telegram partial preview streaming so plain `replyToMode` replies can stream, while only true selected Telegram native quotes disable the preview edit path.

## Best Possible Solution

Land a narrow Telegram dispatcher fix that separates final-delivery quote candidate collection from preview-stream compatibility, so plain reply threading can stream while true selected quotes still use the non-preview path.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: `replyToMode: "first"`, `streamMode: "partial"`, and a plain nonblank Telegram message populate `replyQuoteByMessageId`, make `hasNativeQuoteReply` true, and prevent answer draft stream creation.

## Solution Assessment

Yes. The narrowest maintainable solution is to key the preview-stream skip on actual selected quote metadata, such as `replyQuoteText != null && replyQuoteMessageId != null`, while preserving the candidate map for final delivery fallback.
