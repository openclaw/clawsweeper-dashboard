## Summary

Keep open. Current main still has the Telegram regression: non-off `replyToMode` fills the native-quote candidate map for ordinary inbound text, and the draft-stream guard still treats any populated candidate map as incompatible with preview streaming. The closed unmerged #73534 remains useful source work; the open #39905 is broader per-conversation configuration work, not a landed fix for this specific bug.

## What This Changes

Fix Telegram preview streaming so `streaming.mode: "partial"` still creates and edits a draft preview for plain replies when `replyToMode` is non-off, while selected native quote replies continue to use the non-preview path.

## Best Possible Solution

Land a narrow Telegram dispatcher fix that separates final-delivery native quote candidate collection from preview-stream compatibility, then pin it with regression tests for plain `replyToMode: "first"` partial streaming and selected native quote non-streaming behavior.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: with `replyToMode: "first"`, `streamMode: "partial"`, and a plain nonblank Telegram message, current main populates `replyQuoteByMessageId`, makes `hasNativeQuoteReply` true, and prevents the answer draft stream from being created.

## Solution Assessment

Yes. The narrowest maintainable fix is to key the preview-stream skip on actual selected quote metadata, such as `replyQuoteText != null && replyQuoteMessageId != null`, while preserving `replyQuoteByMessageId` for final delivery fallback.
