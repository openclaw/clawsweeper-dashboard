## Summary

Keep open. Current main still disables Telegram answer draft streaming whenever non-off reply threading populates the native quote candidate map, so the reported `replyToMode` plus `streaming.mode: "partial"` regression remains valid; the useful source PR #73534 is closed unmerged and adjacent PR #39905 does not replace this narrow fix.

## What This Changes

Telegram partial preview streaming should continue to work for plain reply threading when `channels.telegram.replyToMode` is `first` or another non-`off` value, while selected Telegram quote replies may still skip preview streaming.

## Best Possible Solution

Land a narrow Telegram dispatcher fix that separates final-delivery native quote candidate collection from draft-stream compatibility, adds focused regression coverage, and credits the reporter plus the useful closed source PR in the changelog.

## Reproduction Assessment

Yes. With `replyToMode: "first"`, `streaming.mode: "partial"`, and a normal nonblank Telegram message, current source populates `replyQuoteByMessageId`, makes `hasNativeQuoteReply` true, and prevents `createTelegramDraftStream` from being created.

## Solution Assessment

Yes. The narrowest maintainable fix is to make the draft-stream skip depend on actual selected quote metadata, such as `replyQuoteText != null && replyQuoteMessageId != null`, while keeping `replyQuoteByMessageId` for final delivery fallback.
