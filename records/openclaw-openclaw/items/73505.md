## Summary

Keep this open. Current `main` still disables Telegram answer draft streaming whenever non-off `replyToMode` creates any native quote candidate, including a plain current-message reply, and the latest release tag still contains the same guard and test expectation.

## What This Changes

The issue asks for Telegram `streaming.mode: "partial"` to continue working with `replyToMode: "first"` or `"all"` for plain reply threading, while still skipping draft streaming for real selected Telegram quote replies.

## Best Possible Solution

Restore the quote-only streaming guard in the Telegram dispatcher, preserve native quote candidates for final delivery, add a regression test for `replyToMode: "first"` plus `streaming.mode: "partial"` on a plain inbound message, and add a user-facing changelog entry crediting @choury.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: use Telegram `replyToMode: "first"` with partial preview streaming on a plain inbound message, and the dispatcher populates `replyQuoteByMessageId` before disabling the answer draft stream.

## Solution Assessment

Yes. The narrow maintainable fix is to gate draft-stream suppression on an actual selected quote with quote text and a resolvable quote message id, while leaving the native quote candidate map available for final delivery fallback.
