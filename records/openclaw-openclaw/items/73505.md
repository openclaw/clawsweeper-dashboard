## Summary

Keep this open. Current main still has the reported Telegram regression: plain replies with non-off `replyToMode` populate the native-quote candidate map, and that map is still used to disable answer draft streaming. The prior focused fix in #73534 was closed unmerged for queue-cap reasons, so it does not resolve the issue.

## What This Changes

The issue asks for Telegram `streaming.mode: "partial"` live previews to keep working for plain `replyToMode` replies, while only disabling preview streaming for real Telegram quote replies.

## Best Possible Solution

Restore the narrow quote-only streaming guard in the Telegram dispatcher, keep the native quote candidate map for final delivery, add a regression test for `replyToMode: "first"` plus `streaming.mode: "partial"` on a plain inbound message, and include a user-facing changelog entry crediting @choury.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: configure Telegram with non-off `replyToMode` and partial streaming, send a plain text message, and the dispatcher adds the inbound message to `replyQuoteByMessageId` before disabling `canStreamAnswerDraft`; the current test suite already asserts the resulting no-draft-stream behavior.

## Solution Assessment

Yes. The suggested #73534-style guard is the narrowest maintainable fix because it restores the earlier quote-text plus resolvable-message-id condition while preserving the newer native quote candidate map for final delivery.
