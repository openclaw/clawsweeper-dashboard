## Summary

Keep open: current main still lets bare `openclaw acp` reach CLI startup bootstrap with `suppressDoctorStdout` false, and the linked open PR is the active fix candidate using closing syntax.

## What This Changes

The issue asks for bare `openclaw acp` startup to keep stdout reserved for ACP JSON-RPC while routing or suppressing human diagnostics elsewhere.

## Best Possible Solution

Land a narrow CLI startup-policy fix that makes bare ACP server startup protocol-pure while preserving intended human-facing `openclaw acp client` behavior.

## Reproduction Assessment

Yes from source inspection: current main computes startup stdout suppression only from JSON mode before invoking bare `openclaw acp`, while doctor preflight notes are suppressed only when that flag is true. I did not run a live Workbench or CLI repro in this read-only review.

## Solution Assessment

Yes, the best fix belongs in CLI startup policy before preaction/bootstrap output can reach stdout, not only inside `serveAcpGateway`. The open PR is the right candidate to review, with the `acp client` compatibility boundary preserved.
