## Summary

Keep open. Current main still lacks a full `openclaw message send` command timeout and a message-scoped SIGTERM/SIGINT cancellation path, and #76077 is an open, unmerged closing PR for the same issue.

## What This Changes

Add bounded timeout and prompt supervisor signal handling for `openclaw message send` so cron-invoked deliveries cannot hang indefinitely.

## Best Possible Solution

Land a repaired #76077 or a narrow replacement that bounds the entire `message send` path, aborts promptly on SIGTERM/SIGINT, cleans up listeners, and adds regression coverage plus docs/changelog updates.

## Reproduction Assessment

Yes at source level: hold `messageCommand` or pre-dispatch plugin loading pending and current `runMessageAction` has no command-level timer or signal cancellation. I did not establish a live process-level CPU-spin reproduction in this read-only pass.

## Solution Assessment

Yes, a command-level timeout plus scoped signal abort is the narrow maintainable direction. #76077 is aligned with that direction, but it should be reviewed or replaced so the timeout covers pre-dispatch work and signal handlers are cleaned up.
