## Summary

Keep open. Current main still lacks a command-level timeout and scoped SIGTERM/SIGINT cancellation for `openclaw message send`, and #76077 is an open, unmerged closing PR that targets the same report.

## What This Changes

Request an internal timeout and prompt SIGTERM handling for `openclaw message send` so stuck deliveries fail with a clear terminal error instead of hanging cron jobs.

## Best Possible Solution

Land a repaired #76077 or a narrow replacement that starts the timeout before plugin loading, scopes and disposes SIGTERM/SIGINT cancellation, and adds regression coverage plus user-facing docs/changelog if a new flag ships.
