## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

Yes, the proposed code path is the narrowest maintainable fix: reuse the existing `agent:<id>` query token and supported `sessions.usage.agentId` contract, with a compatibility retry for older gateways. The remaining blocker is proof, not a different implementation direction.
