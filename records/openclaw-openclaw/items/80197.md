## Summary

Keep open: the report is concrete and current source does not prove the stale managed-plugin/core mismatch is already repaired. The update path intends to sync plugins, but managed npm-root dependencies are still written as resolved versions, and changing that contract needs maintainer review because it intersects plugin integrity and package-resolution policy.

## What This Changes

Ensure managed default-line npm plugins such as `@openclaw/codex` are updated or clearly warned about after `openclaw update` so stale plugin versions do not break native hook relays.

## Best Possible Solution

Keep the issue open and decide a minimal plugin/update fix that reliably repairs or warns about stale official managed npm plugins while preserving install integrity guarantees.

## Reproduction Assessment

No live current-main reproduction was run in this read-only pass. The issue logs plus current source make it source-reproducible by constructing a managed npm root with a floating install record but an old exact dependency in package.json.

## Solution Assessment

Unclear. The safest direction is probably not a blind unpin of managed npm-root dependencies; maintainers should choose between stronger post-update sync, stale-version doctor/update warnings, or a narrower dependency-spec change with integrity coverage.
