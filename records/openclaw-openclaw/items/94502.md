## Summary

Keep open: this is a useful candidate for the linked Control UI manifest 403 report, but the patch currently expands the route boundary by serving known root public assets for non-read methods when a basePath is configured.

## What This Changes

The PR changes the gateway Control UI HTTP handler to serve known root public assets from the host root when `gateway.controlUi.basePath` is configured and adds GET regression coverage.

## Best Possible Solution

Land one canonical fix that keeps public assets reachable for basePath deployments while preserving the existing GET/HEAD-only static asset boundary.

## Reproduction Assessment

Yes, source-reproducible. Current main classifies root public assets outside a configured basePath as `not-control-ui`, and the PR-introduced route can be inspected directly against the existing read-method invariant; I did not run tests in this read-only review.

## Solution Assessment

No. The fallback is plausible, but this implementation skips the existing GET/HEAD guard; a narrow method check plus regression coverage would make this branch safer, while maintainers may still prefer a sibling URL-rewrite strategy.
