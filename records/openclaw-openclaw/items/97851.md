## What This Changes

The PR removes the Mattermost error parser's `!res.body` native `json()`/`text()` fallback so error reads use the existing 8 KiB bounded text helper, with regression tests.

## Best Possible Solution

Land the bounded-error-reader cleanup after redacted runtime proof or explicit maintainer acceptance confirms the null-body error-path behavior is desired.

## Reproduction Assessment

Yes for the source condition: current main calls native `json()`/`text()` when `readMattermostError()` sees a null body, and a Node `Response(null)` probe exercises that state. The claimed unbounded network-buffering impact is not proven beyond mocked tests.

## Solution Assessment

Yes for the code shape: using the existing bounded helper removes a second error-body read strategy without adding new API or config. The missing piece is real behavior proof or maintainer acceptance, not a line-level code defect.
