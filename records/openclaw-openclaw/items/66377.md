## Summary

Keep open. Current main still leaves the reported multi-location cleanup gap: agent deletion purges config, core session-store entries, and files, but does not run cron cleanup or generic plugin-owned session cleanup.

## What This Changes

The issue requests that deleting or resetting an agent clear all related persisted session, cron, and plugin-owned state so corrupted sessions cannot survive recreation.

## Best Possible Solution

Define one agent deletion/reset lifecycle that enumerates affected sessions, runs generic session and plugin cleanup per session, and applies an explicit cron-job policy without hard-coding plugin storage internals.

## Reproduction Assessment

Yes for the source-level cleanup gap: current main deletes config, core session-store entries, and files, but not cron jobs or generic plugin-owned session state. No for the exact TypeError trigger because the issue lacks a corrupt fixture or complete stack/log path.

## Solution Assessment

No; the current partial purge is not the complete solution. The maintainable path is a generic deletion/reset lifecycle with explicit cron policy and plugin cleanup hooks rather than ad hoc cleanup of one plugin database or one cron file shape.
