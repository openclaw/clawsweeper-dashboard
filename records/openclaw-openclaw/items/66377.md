## Summary

Keep open. Current main still has a real agent-scoped cleanup gap: agent deletion removes config entries, core session-store entries, and files, but it does not apply a cron-job policy or run the generic plugin/session cleanup lifecycle for all sessions owned by the deleted agent.

## What This Changes

The issue asks for agent deletion or reset to clear all agent-scoped persisted state, including sessions, cron references, and plugin/context-engine records.

## Best Possible Solution

Define one coherent agent deletion/reset lifecycle that enumerates affected sessions, runs existing session/plugin cleanup per session, and applies an explicit cron-job policy without hard-coding plugin storage internals.

## Reproduction Assessment

Yes for the source-level cleanup gap: current main deletion paths prune config, core session-store entries, and files, but do not prune cron jobs or invoke plugin/session cleanup for deleted-agent sessions. No for the exact reported `TypeError`, because the issue does not include a deterministic corrupt fixture or full logs.

## Solution Assessment

No, the current partial purge is not the best complete fix. The maintainable path is a generic agent deletion/reset lifecycle plus a decided cron policy, rather than ad hoc cleanup of one plugin database or one cron file shape.
