## Summary

Keep open. Current main still only removes agent config, core session-store entries, and filesystem directories; it does not run cron cleanup or generic plugin-owned session cleanup, so the multi-location persistence gap remains source-proven even though the exact TypeError trigger is not fixture-reproduced.

## What This Changes

The issue requests a complete agent deletion/reset lifecycle that clears all persisted agent-related state, including session files, plugin-owned state, and cron references.

## Best Possible Solution

Define one deletion/reset lifecycle that enumerates affected sessions, runs existing session and plugin cleanup per session, and applies an explicit cron-job policy without hard-coding plugin storage internals.

## Solution Assessment

No; the current partial purge is not the best complete solution. The maintainable path is a generic agent deletion/reset lifecycle with explicit cron policy and plugin cleanup hooks rather than ad hoc cleanup of one plugin database or one cron file shape.
