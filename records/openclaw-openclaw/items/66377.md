## Summary

Keep open. Current main still only performs partial agent deletion cleanup: config entries, core session-store entries, and filesystem state are handled, but persistent cron jobs and plugin/context-engine-owned state are not routed through an agent-scoped cleanup lifecycle.

## What This Changes

The issue asks for agent deletion/recreation, or a new agent reset flow, to clear all agent-scoped session, cron, and plugin/context-engine state.

## Best Possible Solution

Add a coherent agent deletion/reset lifecycle that enumerates affected sessions through the existing mutation cleanup path, applies an explicit cron-job policy, and exposes generic plugin/context-engine cleanup hooks for agent-scoped state.

## Reproduction Assessment

Yes for the remaining cleanup gap: current source shows agent deletion does not prune cron jobs or invoke session/plugin cleanup. The exact corruption trigger behind the reported `TypeError` remains unclear because no deterministic corrupt fixture or full logs were provided.

## Solution Assessment

No, the current partial session-store purge is not the complete fix. The maintainable path is a generic agent deletion/reset lifecycle rather than ad hoc cleanup of one plugin database or one cron file shape.
