## Summary

Keep open. Current main still only removes config entries, core session-store entries, and files during agent deletion; cron jobs and plugin-owned session cleanup are not part of that lifecycle, so the central cleanup gap remains source-proven even though the exact reported TypeError still lacks a deterministic fixture.

## What This Changes

The issue requests agent deletion, recreation, or a reset path to clear all agent-scoped session, plugin, and cron state so corrupted sessions do not survive deletion.

## Best Possible Solution

Define one coherent agent deletion/reset lifecycle that enumerates affected sessions, runs existing session and plugin cleanup per session, and applies an explicit cron-job policy without hard-coding plugin storage internals.

## Reproduction Assessment

Yes for the source-level cleanup gap: current main deletion paths prune config, core session-store entries, and files, but not cron jobs or generic plugin cleanup for deleted-agent sessions. The exact reported TypeError is not reproducible from the available report because there is no corrupt fixture or complete log path.

## Solution Assessment

No, the current partial purge is not the best complete solution. The maintainable path is a generic agent deletion/reset lifecycle with an explicit cron policy, rather than ad hoc cleanup of one plugin database or one cron file shape.
