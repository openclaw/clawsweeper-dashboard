## Summary

Keep open. Current main still performs partial agent deletion cleanup only: config entries, core session-store entries, and filesystem state are handled, but cron jobs and plugin/context-engine-owned state are not routed through an agent-scoped cleanup lifecycle.

## What This Changes

Agent deletion/recreation should clear or repair all agent-scoped state, including core sessions, cron references, and plugin/context-engine persistence.

## Best Possible Solution

Define an agent-scoped deletion/reset lifecycle that enumerates affected sessions through existing session cleanup, applies an explicit cron-job policy, and lets plugins/context engines clean their own state without hard-coded database surgery.

## Reproduction Assessment

Yes for the remaining cleanup gap: current source shows agent deletion does not prune cron jobs or invoke session/plugin cleanup. The exact corrupt transcript or context-engine `TypeError` trigger remains unclear without a deterministic fixture or full logs.

## Solution Assessment

No, the current partial deletion path is not the complete solution. The maintainable fix is a generic agent deletion/reset lifecycle with explicit cron policy and plugin cleanup hooks, not ad hoc cleanup of one cron file or one plugin database.
