## Summary

Keep open. Current main has partial deleted-agent hardening, but it still does not provide an agent-scoped reset, prune cron jobs that target the deleted agent, or route agent deletion through the existing session/plugin cleanup lifecycle needed for plugin-owned state such as LCM.

## What This Changes

The issue requests that deleting, recreating, or resetting an agent clear all related persistent state across session files, session stores, cron references, and plugin/context-engine state, with better corruption recovery messaging.

## Best Possible Solution

Keep this open until OpenClaw has a coherent agent deletion/reset lifecycle: enumerate affected sessions through the existing mutation cleanup path, handle cron jobs that target the agent with an explicit policy, and expose or reuse generic plugin/context-engine cleanup hooks so LCM and other plugins can remove agent/session state without core knowing their database layout.

## Reproduction Assessment

Yes for the remaining cleanup gap: current source shows agent deletion does not prune cron jobs or invoke session/plugin cleanup. The original corruption trigger is still unclear because no corrupt fixture, full logs, or deterministic trigger was provided.

## Solution Assessment

No, current main is not the best complete solution because it only purges/rejects some core session-store state. The safer maintainable fix is a generic agent deletion/reset lifecycle across sessions, cron, and plugin-owned state rather than ad hoc database cleanup.
