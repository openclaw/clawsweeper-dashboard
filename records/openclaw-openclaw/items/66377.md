## Summary

Keep open. Current main still does not tie agent deletion to cron-job cleanup or generic plugin-owned session cleanup, so the reported multi-location state persistence remains source-proven even though the exact TypeError trigger still lacks a deterministic fixture.

## What This Changes

Clear all persistent state tied to a deleted or recreated agent, including sessions, cron references, plugin-owned state, and recovery/reset behavior.

## Best Possible Solution

Define one coherent agent deletion/reset lifecycle that enumerates affected sessions, runs existing session and plugin cleanup per session, and applies an explicit cron-job policy without hard-coding plugin storage internals.

## Reproduction Assessment

Yes for the source-level cleanup gap: current main deletes config, core session-store entries, and files, but not cron jobs or generic plugin-owned session state. The exact reported TypeError is not reproducible from the available report because there is no corrupt fixture or complete log path.

## Solution Assessment

No; the current partial purge is not the best complete solution. The maintainable path is a generic agent deletion/reset lifecycle with explicit cron policy and plugin cleanup hooks, rather than ad hoc cleanup of one plugin database or one cron file shape.
