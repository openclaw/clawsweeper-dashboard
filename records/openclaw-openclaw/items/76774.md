## Summary

Keep open. The ClawHub-first direction matches the install metadata contract for installable artifacts, but the Prometheus ClawHub package currently has no installable latest and the existing fallback path does not treat that error as recoverable, so this PR would make that optional onboarding selection fail instead of using the existing npm default.

## What This Changes

The PR changes the official diagnostics optional plugin catalog defaults from npm to ClawHub, adds wizard coverage for that selection, and adds a changelog entry.

## Best Possible Solution

Only make a diagnostics entry ClawHub-first when its ClawHub spec resolves to an installable artifact, or add an explicit covered fallback for `no_installable_version` before changing Prometheus.

## Reproduction Assessment

Yes. Source inspection plus the live ClawHub readiness endpoint show that selecting Diagnostics Prometheus would choose a ClawHub spec with no installable latest, and the current fallback code does not recover from `no_installable_version`.

## Solution Assessment

No. The OTel ClawHub-first change is directionally consistent, but applying the same default to Prometheus before its ClawHub package is installable is not the narrowest maintainable fix.
