## Summary

Keep open. Current main and v2026.5.12 still keep the active doctor startup optimization helper Linux-only for this memory-target path, and https://github.com/openclaw/openclaw/pull/47291 is an open closing PR for the same helper and tests.

## What This Changes

Include macOS/Darwin hosts in startup optimization memory-target detection so doctor can emit the same hints it emits on Linux ARM or low-memory hosts.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/47291, or an equivalent focused patch, with Darwin regression tests; track broader agent memory telemetry separately if maintainers want that behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence unit path: call `noteStartupOptimizationHints` with `platform: "darwin"`, ARM64 or low memory, and a `/tmp` compile cache; current main returns before `noteFn` because the target predicate is Linux-only.

## Solution Assessment

Yes for the proven doctor helper bug. Widening the existing predicate and adding colocated Darwin cases is the narrow maintainable fix; broader `os.freemem()` or agent memory-context telemetry should remain separate scope.
