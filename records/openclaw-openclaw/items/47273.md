## Summary

Keep open. Current main and the latest release still skip Darwin in the active doctor startup optimization helper, and the open linked PR at https://github.com/openclaw/openclaw/pull/47291 is the right implementation path to review before closing this issue.

## What This Changes

The issue asks for macOS Darwin hosts to receive memory/startup optimization detection instead of being skipped by the Linux-only helper.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/47291, or replace it with an equivalent focused patch that includes Darwin ARM and low-memory hosts in the doctor startup optimization helper with Darwin regression tests.

## Reproduction Assessment

Yes. A unit-level call to `noteStartupOptimizationHints` with `platform: "darwin"`, `arch: "arm64"`, low `totalMemBytes`, and a `noteFn` follows the current source path that returns before `noteFn` can be called.

## Solution Assessment

Yes for the proven bug surface. Updating the doctor helper predicate and colocated tests is the narrow maintainable fix; adding general `os.freemem()` prompt telemetry would be separate work.
