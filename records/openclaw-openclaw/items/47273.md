## Summary

Keep open. Current main and the latest release still exclude Darwin from the active doctor startup optimization helper, and open PR #47291 already covers the same helper/test path, so closing before a merge would be premature.

## What This Changes

The issue asks OpenClaw to include macOS/Darwin hosts in the memory-aware startup detection path instead of silently skipping them behind a Linux-only platform guard.

## Best Possible Solution

Review and land PR #47291, or replace it with an equivalent focused patch that includes Darwin ARM and low-memory hosts in `noteStartupOptimizationHints` with Darwin regression tests.

## Reproduction Assessment

Yes. A unit-level call to `noteStartupOptimizationHints` with `platform: "darwin"`, `arch: "arm64"`, low `totalMemBytes`, and a `noteFn` reproduces the early return before the note function is called.

## Solution Assessment

Yes for the proven bug surface. Updating the doctor helper predicate and colocated tests is the narrow maintainable fix; adding general `os.freemem()` prompt telemetry would exceed the verified code path.
