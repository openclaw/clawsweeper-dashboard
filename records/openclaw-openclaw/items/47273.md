## Summary

Keep open. Current main still excludes Darwin from the active startup optimization helper, and an open PR already references this issue with closing syntax, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue requests that macOS hosts be included in startup/memory detection so Darwin can use available memory values in the agent/doctor path.

## Best Possible Solution

Review and land PR #47291, or replace it with an equivalent focused patch that includes Darwin ARM and low-memory hosts in `noteStartupOptimizationHints` with Darwin regression tests.
