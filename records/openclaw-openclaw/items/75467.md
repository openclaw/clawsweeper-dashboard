## Summary

Keep open. Current main still drops model compatibility before OpenAI tool-schema normalization, and the issue is already tied to open closing PR #75476, so it should remain open while that PR is reviewed.

## What This Changes

Request to honor `compat.unsupportedToolSchemaKeywords` for OpenAI-completions tool schemas and add Fireworks Kimi K2.5 Turbo metadata to strip JSON Schema `not`.

## Best Possible Solution

Review and land PR #75476, or an equivalent narrow fix, so extracted `ModelCompatConfig` reaches both OpenAI Responses and Completions tool-schema normalization and the Fireworks plugin owns the affected model metadata with focused regression tests.

## Reproduction Assessment

Yes. A focused source-level reproduction can build an OpenAI-completions tool schema containing `{ "not": {} }` with `compat.unsupportedToolSchemaKeywords: ["not"]`; current main has no call path that passes that compat into the OpenAI normalizer, so the keyword remains.

## Solution Assessment

Yes. Threading extracted model compat into the existing schema normalizer and adding provider-owned Fireworks catalog metadata is the narrow maintainable fix; Fireworks-specific branching in core would be a worse fit for the existing compat contract.
