## Summary

Keep open. Current main still does not pass model compat into the OpenAI Responses or Completions tool-schema normalizer, the Fireworks Kimi router catalog row still lacks the requested unsupported-keyword metadata, and open PR #75476 already targets this issue with closing syntax, so the issue should remain open until that PR is reviewed and either landed or rejected.

## What This Changes

The issue requests that OpenAI Responses/Completions tool-schema normalization honor `model.compat.unsupportedToolSchemaKeywords` and that Fireworks Kimi router catalog metadata declare `not` as unsupported.

## Best Possible Solution

Review and land a focused fix that threads existing `ModelCompatConfig` into the OpenAI tool-schema normalization path, adds Fireworks-owned unsupported-keyword catalog metadata, covers strict and non-strict regression cases, and includes the required changelog entry.

## Reproduction Assessment

Yes. A source-level reproduction can use an OpenAI-compatible model with `compat.unsupportedToolSchemaKeywords: ["not"]` and a tool schema containing `{ "not": {} }`; current main never passes that compat into the OpenAI tool normalizer, so the keyword remains.

## Solution Assessment

Yes. The narrow maintainable direction is to pass the existing model compat into the existing schema-normalization contract and keep Fireworks-specific metadata in the Fireworks provider catalog, instead of adding Fireworks-specific branching in core.
