## Summary

Keep open. Current main still omits model compat from OpenAI tool-schema normalization and the Fireworks Kimi catalog metadata, while open PR #75476 is the active closing implementation path.

## What This Changes

Honor `compat.unsupportedToolSchemaKeywords` for OpenAI Responses/Completions tool schemas, including the Fireworks Kimi model metadata.

## Best Possible Solution

Land a focused fix that threads `ModelCompatConfig` into OpenAI tool-schema normalization, adds the Fireworks unsupported-keyword metadata, and keeps regression coverage for strict and non-strict paths.

## Reproduction Assessment

Yes. A source-level reproduction is an OpenAI-compatible model with `compat.unsupportedToolSchemaKeywords: ["not"]` and a tool schema containing `{ "not": {} }`; current main never passes that compat into the OpenAI tool normalizer, so the keyword remains.

## Solution Assessment

Yes. The narrow maintainable fix is to reuse the existing `normalizeToolParameterSchema` compat option and keep Fireworks-specific unsupported-keyword data in the Fireworks provider catalog.
