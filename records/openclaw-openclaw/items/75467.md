## Summary

Keep open. Current main still drops model compatibility before OpenAI tool-schema normalization, and the linked open PR is the active closing implementation path, so this issue should remain the tracker until that PR lands or is replaced.

## What This Changes

Honor `compat.unsupportedToolSchemaKeywords` for OpenAI tool-schema normalization so Fireworks Kimi tool schemas strip unsupported keywords such as `not`.

## Best Possible Solution

Review and land or replace #75476 so OpenAI Responses/completions tool-schema normalization receives `ModelCompatConfig`, Fireworks declares unsupported keywords, and regression coverage protects strict and non-strict paths.

## Reproduction Assessment

Yes. By source inspection, an OpenAI-compatible model with `compat.unsupportedToolSchemaKeywords: ["not"]` and a tool schema containing `{ "not": {} }` still reaches `normalizeToolParameterSchema` without `modelCompat`, so the keyword is not stripped.

## Solution Assessment

Yes. The narrow maintainable fix is to thread the existing model compatibility object into OpenAI tool-schema normalization and keep Fireworks-specific unsupported-keyword data in the provider catalog, without adding new config or provider-specific branching.
