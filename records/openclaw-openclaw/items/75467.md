## Summary

Keep open. Current main still does not thread model compat into OpenAI Responses/Completions tool-schema normalization, the Fireworks Kimi catalog row still lacks unsupported-keyword metadata, and open PR #75476 is the active closing implementation path.

## What This Changes

Thread `compat.unsupportedToolSchemaKeywords` through OpenAI-compatible tool-schema normalization so Fireworks Kimi tool schemas strip unsupported JSON Schema keywords such as `not`.

## Best Possible Solution

The end state should be generic OpenAI tool-schema normalization honoring `ModelCompatConfig`, with Fireworks unsupported-keyword metadata in its plugin catalog and regression coverage for strict and non-strict paths.

## Reproduction Assessment

Yes. A source-level reproduction can use an OpenAI-compatible model with `compat.unsupportedToolSchemaKeywords: ["not"]` and a tool schema containing `{ "not": {} }`; current main never passes that compat into the OpenAI tool normalizer, so the keyword remains.

## Solution Assessment

Yes. The narrow maintainable direction is to pass the existing model compat into the existing schema-normalization contract and keep Fireworks-specific unsupported-keyword data in the Fireworks provider catalog.
