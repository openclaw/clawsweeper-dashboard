## Summary

Keep open. Current main still drops `unsupportedToolSchemaKeywords` before OpenAI tool-schema normalization, and #75476 is the open closing implementation path, so this issue should remain the tracker until that PR lands or is replaced.

## What This Changes

The issue asks for OpenAI-compatible tool-schema normalization to honor `compat.unsupportedToolSchemaKeywords` for Fireworks/Kimi tool schemas containing `not`.

## Best Possible Solution

Land or replace #75476 so OpenAI tool-schema normalization receives `ModelCompatConfig`, Fireworks has verified unsupported-keyword metadata, and regression coverage locks both strict and non-strict paths.

## Reproduction Assessment

Yes, by source inspection. On current main, an OpenAI-compatible model with `compat.unsupportedToolSchemaKeywords: ["not"]` and a tool schema containing `{ "not": {} }` reaches OpenAI tool normalization without `modelCompat`, so the keyword remains.

## Solution Assessment

Yes. The narrow maintainable fix is to thread existing model compatibility into OpenAI tool-schema normalization and keep provider-specific unsupported-keyword data in the Fireworks plugin catalog.
