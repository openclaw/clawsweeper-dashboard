## Summary

Keep open. Current main still omits model compat from OpenAI tool-schema normalization, and open PR #75476 is the active unmerged closing implementation path.

## What This Changes

OpenAI-compatible provider tool schemas should honor `compat.unsupportedToolSchemaKeywords` so Fireworks can strip unsupported JSON Schema keywords such as `not`.

## Best Possible Solution

Review and land or reject #75476; the desired end state is generic OpenAI tool-schema normalization honoring `ModelCompatConfig`, Fireworks catalog metadata for unsupported keywords, and regression coverage for strict and non-strict paths.

## Reproduction Assessment

Yes. A source-level reproduction uses an OpenAI-compatible model with `compat.unsupportedToolSchemaKeywords: ["not"]` and a tool schema containing `{ "not": {} }`; current main never passes that compat into the OpenAI normalizer, so the keyword remains.

## Solution Assessment

Yes. The narrow maintainable fix is to pass existing model compat into the existing schema-normalization contract and keep Fireworks-specific unsupported-keyword data in the Fireworks plugin catalog.
