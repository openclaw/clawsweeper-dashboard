## Summary

Keep open. Current main still omits model compat when normalizing OpenAI Responses and Completions tool schemas, the Fireworks Kimi router catalog row still lacks the requested unsupported-keyword metadata, and open PR #75476 already targets this issue with closing syntax, so the issue should stay open until that PR is reviewed and either landed or rejected.

## What This Changes

The issue asks for OpenAI-compatible tool schema normalization to honor `compat.unsupportedToolSchemaKeywords`, especially stripping `not` for Fireworks Kimi tool schemas.

## Best Possible Solution

Review and land a focused fix that passes existing model compat into OpenAI Responses and Completions tool-schema normalization, keeps Fireworks-specific unsupported-keyword metadata in the Fireworks provider catalog, and adds targeted regression coverage plus the required changelog entry.

## Reproduction Assessment

Yes. A source-level reproduction can construct an OpenAI-compatible model with `compat.unsupportedToolSchemaKeywords: ["not"]` and a tool schema containing `{ "not": {} }`; current main never passes that compat into the OpenAI tool normalizer, so the keyword remains.

## Solution Assessment

Yes. The narrow maintainable fix is to thread the existing `ModelCompatConfig` into the existing schema-normalization contract and add provider-owned Fireworks metadata, rather than adding Fireworks-specific branching in core.
