## Summary

Keep open. Current main still omits model compat from OpenAI tool-schema normalization, and open PR #75476 is the active unmerged closing path for this issue.

## What This Changes

Honor `compat.unsupportedToolSchemaKeywords` for OpenAI-completions tool schema normalization, especially Fireworks schemas containing the `not` keyword.

## Best Possible Solution

Review and land or reject #75476; the desired end state is generic OpenAI tool-schema normalization honoring model compat plus Fireworks catalog metadata and regression coverage.

## Reproduction Assessment

Yes. A source-level reproduction uses an OpenAI-compatible model with `compat.unsupportedToolSchemaKeywords: ["not"]` and a tool schema containing `{ "not": {} }`; current main never passes that compat into the OpenAI normalizer, so the keyword remains.

## Solution Assessment

Yes. Threading the existing `ModelCompatConfig` into `normalizeToolParameterSchema` is the narrow maintainable fix, with Fireworks-specific unsupported keyword data kept in the Fireworks plugin catalog.
