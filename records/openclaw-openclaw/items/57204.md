## Summary

Keep this open: current `main` still rebuilds immediately for a dirty watched source tree before checking source mtimes, and an open related fork PR with closing syntax remains the focused implementation candidate.

## What This Changes

The issue requests that `gateway:watch` skip startup rebuilds when the working tree is dirty but no watched source file is newer than the build stamp.

## Best Possible Solution

Review and land the open related PR or an equivalent narrow patch in `scripts/run-node.mjs`, with focused coverage for dirty-but-not-newer and dirty-and-newer source trees.

## Reproduction Assessment

Yes. Source inspection on current `main` shows the dirty watched source path returns before the mtime check, and the current unit test still asserts that rebuild reason; I did not run tests because this review is read-only.

## Solution Assessment

Yes. Letting dirty watched source trees fall through to the existing mtime check is the narrow maintainable fix, provided the existing force-build, HEAD/config, missing-output, and newer-source paths stay intact.
