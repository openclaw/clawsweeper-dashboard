## Summary

Keep this open: current main still rebuilds immediately for a dirty watched source tree before checking source mtimes, and an open related fork PR with closing syntax is still the implementation candidate.

## What This Changes

The issue requests that `gateway:watch` skip rebuilds when the working tree is dirty but no build-relevant source file is newer than the build stamp.

## Best Possible Solution

Review and land the open related PR or an equivalent narrow patch in `scripts/run-node.mjs`, with focused coverage for dirty-but-not-newer and dirty-and-newer source trees plus any needed regression-helper expectation updates.

## Reproduction Assessment

Yes. Source inspection on current main shows the dirty watched source path returns before the mtime freshness check, and the current unit test still asserts that rebuild reason; I did not run tests because this review is read-only and test commands create artifacts.

## Solution Assessment

Yes. Letting dirty watched source trees fall through to the existing mtime check is the narrow maintainable fix, provided the existing clean-tree, newer-source, HEAD/config, missing-output, and force-build paths stay intact.
