## What This Changes

The issue asks for `gateway:watch` startup rebuild detection to skip rebuilding when the watched source tree is dirty but no source file is newer than the build stamp.

## Best Possible Solution

Confirm the intended dirty-tree freshness contract, then land a narrow upstream patch that preserves clean-checkout, HEAD-change, config, and missing-output guards while letting dirty-but-already-built source trees use the mtime fallback.

## Reproduction Assessment

Yes. ## Solution Assessment

Unclear. The proposed mtime fallthrough is the narrow plausible fix, but current tests and the regression helper now encode dirty-tree rebuilds as intentional enough that maintainers should confirm the contract before adoption.
