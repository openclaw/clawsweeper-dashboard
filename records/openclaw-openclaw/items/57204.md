## Summary

Keep this open: current main and latest release `v2026.5.22` still return `dirty_watched_tree` before the source mtime freshness check, and the open fork PR at https://github.com/BingqingLyu/openclaw/pull/1646 remains the focused implementation candidate.

## What This Changes

Make `pnpm gateway:watch` skip rebuilds for dirty watched source trees when no source file is newer than the build stamp.

## Best Possible Solution

Review and land https://github.com/BingqingLyu/openclaw/pull/1646 or an equivalent narrow patch that lets dirty watched source trees fall through to source mtime freshness with regression coverage for both dirty-but-not-newer and dirty-and-newer cases.

## Reproduction Assessment

Yes at source level. Current `resolveBuildRequirement()` returns `dirty_watched_tree` before the mtime check, and current tests still assert that dirty watched source forces a build; I did not execute tests because this review is read-only.

## Solution Assessment

Yes. Falling through to the existing mtime freshness check is the narrow maintainable fix direction, as long as the linked PR is updated or replaced against current main and preserves the other rebuild triggers.
