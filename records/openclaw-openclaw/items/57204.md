## Summary

Keep open: current main still forces a build for dirty watched source trees before the source-mtime freshness check can run, and an open related PR with closing syntax is already the implementation candidate.

## What This Changes

The issue asks for `gateway:watch`/`run-node` to skip TypeScript rebuilds when the working tree is dirty but no watched source file is newer than the build stamp.

## Best Possible Solution

Land BingqingLyu/openclaw#1646 or an equivalent narrow patch that lets dirty watched source trees fall through to the existing mtime check, with regression coverage for dirty-but-not-newer and dirty-and-newer source trees.

## Reproduction Assessment

Yes. Source inspection on current main shows `resolveBuildRequirement()` returns `dirty_watched_tree` immediately for dirty watched source paths, and the existing test asserts that rebuild behavior before the mtime check can skip it.

## Solution Assessment

Yes. Falling through to the existing `hasSourceMtimeChanged()` path for dirty trees is the narrow maintainable fix, provided the existing clean, HEAD, config, missing-dist, dirty-and-newer, and force-build paths remain covered.
