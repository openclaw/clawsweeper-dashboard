## Summary

Keep this open: current main still forces a rebuild for a dirty watched source tree before the source mtime check can run, and the open fork PR with closing syntax remains the focused implementation candidate.

## What This Changes

Requests that `pnpm gateway:watch` skip rebuilds for dirty watched trees when no source file is newer than the build stamp.

## Best Possible Solution

Review and land the linked fix or an equivalent narrow patch so dirty watched source trees fall through to the existing mtime check, with regression coverage for dirty-but-not-newer and dirty-and-newer source files.

## Reproduction Assessment

Yes. Source inspection on current main shows the dirty watched source path returns before the mtime check, and the current unit test still asserts that rebuild reason; I did not run tests because this review is read-only.

## Solution Assessment

Yes. Letting dirty watched trees fall through to the existing mtime freshness check is the narrow maintainable fix, provided the existing force-build, HEAD/config, missing-output, and newer-source paths stay intact.
