The issue requests that `run-node` and `gateway:watch` skip rebuilds for dirty watched trees when source mtimes are not newer than the build stamp.

## Best Possible Solution

Refresh and land the focused run-node/test fix against current main, preserving current HEAD, config, and bundled-plugin dist-output rebuild guards.

## Solution Assessment

Yes. The narrow maintainable fix is to let dirty or unavailable git status fall through to mtime freshness while keeping the existing clean-tree fast path and missing-output guards.
