## Summary

Keep this open: current main still forces a build for any dirty watched source tree before checking source mtimes, and an open related PR with closing syntax should be reviewed or closed before resolving the issue.

## What This Changes

The issue requests that `pnpm gateway:watch` skip rebuilds for dirty but already-built source trees by falling through to source mtime freshness checks.

## Best Possible Solution

Review and land #1646 or an equivalent narrow patch in `scripts/run-node.mjs`, with focused coverage in `src/infra/run-node.test.ts` for dirty-but-not-newer and dirty-and-newer source trees.

## Reproduction Assessment

Yes. A live watch run was not performed in this read-only review, but current source and tests show that a dirty watched source path still returns `dirty_watched_tree` before the mtime freshness check.

## Solution Assessment

Yes. Letting dirty watched source trees fall through to the existing mtime check is the narrow maintainable fix, provided the existing clean, HEAD-changed, config-newer, missing-dist, and force-build paths remain intact.
