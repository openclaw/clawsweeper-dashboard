## Summary

Keep this open: current main still forces a build for any dirty watched source tree before checking source mtimes, and the provided context shows an open related PR with closing syntax that should be reviewed or closed before resolving this issue.

## What This Changes

Avoid rebuilding TypeScript on `gateway:watch` startup when the watched source tree is dirty but no build-relevant source file is newer than the build stamp.

## Best Possible Solution

Review and land the open related PR or an equivalent narrow patch in `scripts/run-node.mjs`, with focused coverage for dirty-but-not-newer and dirty-and-newer source trees.

## Reproduction Assessment

Yes. Source inspection on current main shows the dirty watched source path returns before the mtime freshness check, and the unit test still asserts that behavior.

## Solution Assessment

Yes. Letting dirty watched source trees fall through to the existing mtime check is the narrow maintainable fix, provided the existing clean-tree, HEAD/config, missing artifact, and force-build paths stay intact.
