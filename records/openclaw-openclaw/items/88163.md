## What This Changes

The branch changes device pairing approval to reuse an existing valid same-key untagged device token during metadata refreshes while preserving rotation for scope, public-key, issuer, revoked, and malformed-token boundaries, with regression tests.

## Best Possible Solution

Finalize the draft after targeted pairing tests and changed-check proof run on the final merge head, then land the guarded reuse implementation if maintainers accept the credential-stability tradeoff.

## Reproduction Assessment

## Solution Assessment

Yes, the proposed shape is narrow: it centralizes reuse eligibility in one approval guard and adds regression coverage for both stable reuse and unsafe reuse boundaries. The safer pre-merge step is final merge-head validation rather than a broader rewrite.
