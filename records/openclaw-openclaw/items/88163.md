## Risks / Open Questions

- The public native approval snapshot schema currently accepts empty or metadata-only objects, which can let protocol consumers accept a response the Control UI cannot classify as either file-backed or host-native.
- Existing operator scripts or diagnostics that mutate node exec approvals through raw node.invoke system.execApprovals.* will fail after merge and must use exec.approvals.node.* instead.
- Stable Windows nodes with stale, unbound, revoked, or mismatched device metadata can be forced through re-pairing or quarantine during upgrade; that is safer but user-visible.
- Loopback shared token/password CLI calls now use backend gateway-client identity with least-privilege scopes, which can change local auth/session behavior for tooling that assumed CLI device identity.
- The PR body has strong earlier Windows pond proof and focused current-head tests, but it explicitly does not claim an after-rebase native Windows or WSL2 rerun.
