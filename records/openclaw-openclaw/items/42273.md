## Summary

Keep open. Current main and `v2026.6.8` have a shipped volatile-file retry/skip mitigation for live backup races, but they still lack the broader exclude/smart-exclude behavior requested for multi-GB state directories, and the open linked PR is the current implementation candidate rather than a reason to close early.

## What This Changes

The issue asks for `openclaw backup create` to complete or fail clearly on multi-GB `.openclaw` directories, with possible log/media excludes, size warnings, progress, or streaming behavior.

## Best Possible Solution

Keep the shipped volatile-file retry/filter mitigation, then review, repair, or replace the linked exclude/smart-exclude PR so large backups can omit rebuildable state without weakening restore-critical coverage.
