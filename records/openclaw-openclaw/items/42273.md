## Summary

Keep open. Current main and v2026.6.8 include a real mitigation for live-write tar races and volatile state files, but they still do not implement the broader large-directory exclusion, smart-exclude, size/progress, or configurable backup policy requested here; the linked candidate PR remains open and still needs maintainer review/proof.

## What This Changes

The issue reports that full backup creation stalls or silently exits on large OpenClaw state directories and asks for completion, clear failure, exclusions, size/progress handling, or streaming behavior.

## Best Possible Solution

Land or replace the linked backup exclusion design with a maintainer-approved core backup policy that handles large rebuildable state safely, preserves recovery-critical paths by default, documents upgrade/output behavior, and includes real behavior proof on a large installation.

## Reproduction Assessment

Yes at source level, but not as a live 4GB repro in this review. Current code still archives the whole state tree by default except for selected volatile files and extension dependency trees, while `--no-include-workspace` only removes workspaces and would not exclude large state directories like browser caches, media, or session indexes.

## Solution Assessment

Unclear as a final product design. A focused exclusion or smart-exclude path could solve the reported large-state behavior, but it needs maintainer agreement on protected paths, defaults, manifest compatibility, and real behavior proof before it is the best fix.
