## Summary

Keep open. Current main and v2026.5.22 now mitigate live volatile-state races, but the reported large-installation backup problem is not fully closed: `backup create` still has only coarse `--no-include-workspace` and `--only-config` controls, no restore-safe large-state exclude surface, and the linked first-party implementation remains open, conflicting, and proof-blocked.

## What This Changes

The issue asks for `openclaw backup create` to complete or fail clearly on multi-GB `.openclaw` state, with suggested large-directory excludes, size/progress feedback, or streaming/error handling.

## Best Possible Solution

Review and either land a corrected https://github.com/openclaw/openclaw/pull/44288 or replace it with a narrower native backup fix that preserves current retry/volatile-filter behavior while adding restore-safe large-state exclusions and clear progress or failure behavior.
