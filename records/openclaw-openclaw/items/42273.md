## Summary

Keep open. Current main has backup volatile-file skip/retry hardening, but it still lacks smart/custom large-backup exclusion controls or progress/size behavior, and the open implementation PR with closing syntax for this issue remains unmerged and conflicting.

## What This Changes

`openclaw backup create` should handle multi-GB state directories by completing, failing clearly, or allowing nonessential data such as logs/media/cache to be excluded.

## Best Possible Solution

Review and either land a corrected https://github.com/openclaw/openclaw/pull/44288 or replace it with a narrower first-party backup exclude/progress/error fix while keeping timeout cleanup tracked separately.

## Reproduction Assessment

Unclear for a live 4GB current-main stall. The original report has concrete 2026.3.8 steps, and source inspection confirms broad state archiving without smart excludes, but I did not run a multi-GB backup after current volatile-file hardening.

## Solution Assessment

Unclear until the linked implementation PR is refreshed and reviewed. A first-party fix belongs in the backup CLI/archive path, but the exact default exclusions and protected restore paths need maintainer judgment.
