## Summary

Keep open. Current main and v2026.6.6 partially harden backup creation, but they do not fully solve the reported large-installation workflow: there is still no smart/custom exclude surface, size warning, progress UI, or live proof that a 4GB+ state backup now completes or fails clearly. The active native fix path is still https://github.com/openclaw/openclaw/pull/44288, which is open and conflicting, while KeepMyClaw is useful workaround context rather than a core fix.

## What This Changes

The issue asks `openclaw backup create` to complete or fail clearly on 4GB+ `.openclaw` directories, with suggested exclusions, size/progress controls, and streaming/error handling improvements.

## Best Possible Solution

Keep this issue open until maintainers either land an upgrade-safe native backup exclusion/progress/failure-handling path or close the linked implementation and choose a narrower canonical follow-up for the remaining backup reliability work.

## Reproduction Assessment

No high-confidence live 4GB+ current-main reproduction was run. Source and docs inspection do show that current main and v2026.6.6 still lack the requested smart/custom exclusions, size controls, and progress surface, while only partially mitigating live-file tar races.
