The issue requests that `openclaw backup create` handle large `.openclaw` directories without silently stalling or leaving only a tiny temporary archive.

Review and either land a corrected https://github.com/openclaw/openclaw/pull/44288 or replace it with a narrower native backup fix that provides restore-safe large-state exclusions plus clear progress, failure, and cleanup behavior in the backup CLI/archive path.
