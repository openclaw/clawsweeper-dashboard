## Summary

Keep open. Current main now has a merged volatile-file filter and retry path that likely addresses one live-backup EOF/stall cause, but this issue is still tied to an open closing PR for broader restore-safe exclusion controls, so maintainers should review that PR or explicitly decide the merged fix is enough.

## What This Changes

Full `openclaw backup create` reportedly stalls on multi-GB state directories while config-only backups complete, leaving only a small temporary archive.

## Best Possible Solution

Maintainers should either land or replace https://github.com/openclaw/openclaw/pull/44288 if broader exclusions are still desired, or close this issue as implemented by the merged volatile-file fix after validating that current behavior resolves the reported stall.

## Reproduction Assessment

No, not for the exact 4GB stall on current main. Source and PR history show a plausible live-backup EOF cause was fixed, but the current checkout was not exercised against a multi-GB `.openclaw` directory matching the report.

## Solution Assessment
