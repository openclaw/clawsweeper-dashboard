## Summary

Keep open. Current main still has the reported Codex native compaction accounting gap: unknown post-compaction usage is not preserved as unknown, and transcript fallback can still repopulate a fresh-looking total after the boundary.

## What This Changes

After Codex native compaction, OpenClaw should persist reliable `tokensAfter` values and otherwise report post-compaction context usage as unknown or stale instead of reviving pre-compaction usage.

## Best Possible Solution

Honor the native compaction boundary: store a finite reliable `tokensAfter` as fresh, otherwise preserve an explicit unknown/stale post-compaction state and keep transcript fallback from crossing that boundary.

## Reproduction Assessment

Yes, at source level. Complete Codex native compaction without `tokensAfter`, let `sessions.compact` clear stored token freshness, then session listing can read prior transcript usage and expose it as a fresh current context total again.

## Solution Assessment

Yes. The narrow maintainable fix is to preserve an explicit unknown/stale post-compaction state and teach transcript fallback to respect the compaction boundary, rather than fabricating a precise context percentage.
