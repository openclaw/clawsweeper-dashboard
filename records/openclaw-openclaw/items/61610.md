## Summary

Keep open. Current main has useful partial fixes, but source still shows unresolved recovery gaps: cron task cancellation can still hit the unsupported path, offline task maintenance is deliberately non-authoritative for cron liveness, TaskFlow cancellation can still stop on active children, and the remaining stale-running/restart work is split across open follow-up items rather than fully implemented here.

## What This Changes

The issue asks for task cancellation, task maintenance, cron deletion cleanup, and gateway restart behavior to recover stale persisted `running` task records.

## Best Possible Solution

Define one operator recovery contract: gateway-owned maintenance reconciles proven-stale cron/task rows, CLI cancellation either coordinates with the live gateway or clearly limits offline authority, and TaskFlow cancellation gains a non-deadlocking recovery path for unrecoverable children.

## Reproduction Assessment

Yes, source-backed. Current main still has concrete paths where cron task cancel is unsupported, offline cron maintenance is non-authoritative, and TaskFlow cancel can return with active children still blocking completion.

## Solution Assessment

No. Current main is only a partial solution; the best fix needs a maintainer-approved recovery contract rather than another isolated age-only prune, cancel escape hatch, or restart-drain policy change.
