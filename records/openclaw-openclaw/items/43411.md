## Summary

Keep open: this is a valid redaction bug fix area, but the PR head still leaves mode-only redaction callers on a defaults-only path, lacks after-fix real behavior proof, and is currently conflicting against main.

## What This Changes

The PR adds a default-plus-custom redaction pattern merge path, updates tool-detail/tool-payload redaction, adds focused redaction tests, and changes one doctor install fixture `git add` command.

## Best Possible Solution

Centralize config-derived redaction resolution so no-options, tool-detail, tool-payload, transcript, and mode-only safety callers get custom patterns plus defaults, while explicit `patterns` arrays keep exact replacement semantics.

## Reproduction Assessment

Yes, by source inspection. Configure `logging.redactPatterns` with an email regex and send text containing that email through a mode-only caller such as exec approval display; the PR head still resolves defaults only and skips the configured custom pattern.

## Solution Assessment

No. The merge helper is the right direction, but branching on the presence of any options object leaves mode-only callers incomplete; the safer fix is to distinguish explicit `patterns` from explicit `mode` or use one config-derived merge path for safety callers.
