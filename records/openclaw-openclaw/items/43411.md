## Summary

Keep open: current main only partially addresses the linked transcript redaction bug, and this PR still leaves mode-only safety callers on a defaults-only path while lacking required real behavior proof.

## What This Changes

This PR adds a default/custom redaction merge helper and tests, changes tool-detail and tool-payload redaction resolution, and tweaks one doctor install fixture from `git add -A` to `git add -- .`.

## Best Possible Solution

Centralize config-derived redaction resolution so no-options and mode-only safety callers apply configured custom patterns plus defaults, while explicit `patterns` arrays remain exact caller-supplied replacements.

## Reproduction Assessment

Yes. Source inspection shows the PR head still branches on any options object, and current main has exec approval and diagnostic callers using the mode-only `{ mode: "tools" }` form.

## Solution Assessment

No. The merge helper is directionally right, but the PR should branch on explicit `patterns` or use a shared config-derived resolver for mode-only calls before it can be the maintainable fix.
