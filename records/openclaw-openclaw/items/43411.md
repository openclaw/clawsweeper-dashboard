## Summary

Keep open; the patch is directionally useful, but it leaves a config-derived transcript redaction path where custom patterns can still replace the built-in secret defaults.

## What This Changes

The PR adds a default-plus-custom redaction helper, routes no-options and tool redaction through it, adds focused redaction tests, and narrows one doctor install fixture `git add` command.

## Best Possible Solution

Centralize additive config-pattern resolution for every `logging.redactPatterns` sink, including persisted transcripts, while preserving deliberate explicit replacement semantics only behind a clearly tested low-level call path.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure `logging.redactSensitive: "tools"` with only a custom email regex, then persist transcript text containing that email and a default-matched API key; the PR still routes transcript patterns through explicit replacement semantics.

## Solution Assessment

No. The PR fixes no-options and tool-payload redaction, but the narrow maintainable fix must also route persisted transcript config redaction through the additive default-plus-custom resolver and add regression coverage for that sink.
