The PR adds a helper for additive logging redaction patterns, rewires redaction helpers and tests, and changes one doctor E2E git fixture command.

## Best Possible Solution

Centralize redaction option resolution so omitted `patterns` use configured custom patterns plus defaults across generic and safety callers, while explicitly supplied patterns remain exact.

## Reproduction Assessment

## Solution Assessment

No. The merge helper is the right direction, but the PR should branch on whether `patterns` was explicitly supplied or use a shared config-derived resolver instead of branching on the presence of any options object.
