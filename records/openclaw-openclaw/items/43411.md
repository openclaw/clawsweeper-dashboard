## Best Possible Solution

Centralize config-derived redaction option resolution so no-options, tool-detail, tool-payload, transcript, and mode-only safety callers receive custom patterns plus defaults, while explicit `patterns` arrays keep replacement semantics and docs/tests match that contract.

No. The shared merge helper is the right direction, but the patch keys off the presence of any options object instead of explicit `patterns`, so mode-only safety callers remain incomplete.
