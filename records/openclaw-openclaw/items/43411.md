## Summary

Keep open: this is a plausible focused redaction bug fix, and the earlier Greptile test-quality concern has been addressed, but the current patch still misses a persisted-transcript path where configured custom patterns can replace the built-in secret patterns.

## What This Changes

The PR adds a helper that merges default and user redaction patterns for config-derived no-options/tool paths, adds focused `redactSensitiveText` tests, and changes one doctor install fixture from `git add -A` to `git add -- .`.

## Best Possible Solution

Land a rebased fix that centralizes default-plus-custom pattern resolution for all config-derived logging, transcript, and tool-payload sinks, preserves explicit low-level replacement semantics with tests, and aligns the docs.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure `logging.redactSensitive: "tools"` with only a custom email regex, then persist transcript text containing both that email and a default-matched API key; the PR still routes transcript config patterns through explicit replacement semantics.

## Solution Assessment

No. The PR fixes no-options and tool-payload redaction paths, but the narrow maintainable fix must also route persisted transcript config redaction through the additive resolver and cover custom-plus-default transcript persistence.
