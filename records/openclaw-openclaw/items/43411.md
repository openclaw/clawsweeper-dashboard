## Summary

Keep open: this is a focused security-redaction fix, but the latest PR head still skips configured `logging.redactPatterns` for mode-only callers and lacks after-fix real behavior proof, so it should not merge as-is.

## What This Changes

The PR adds a default-pattern merge helper for logging redaction, changes redaction option resolution and tool payload helpers, adds redaction tests, and changes one doctor E2E fixture `git add` invocation.

## Best Possible Solution

Centralize redaction option resolution so omitted `patterns` load configured custom patterns plus defaults, explicitly supplied patterns remain exact, and transcript/tool-payload safety helpers keep focused regression coverage.

## Reproduction Assessment

Yes, by source inspection: the PR head treats `{ mode: "tools" }` as explicit options and never reads configured `logging.redactPatterns`, while current main has safety callers using that mode-only form. I did not run a live runtime repro in this read-only review.

## Solution Assessment

No. The merge helper is useful, but the PR should branch on whether `patterns` was explicitly supplied or use a shared config-derived resolver instead of branching on the presence of any options object.
