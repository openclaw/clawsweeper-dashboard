## Summary

Keep open: current main still only gives google-vertex a provider-owned ADC transport for authorized_user credentials, while the reported service-account ADC plus proxychains path remains unsupported or undocumented; related PR #65023 is still open rather than merged.

## What This Changes

The issue asks for google-vertex authentication to work, or fail with clear proxy guidance, when OpenClaw runs through proxychains with gcloud ADC service-account credentials.

## Best Possible Solution

Land a maintainer-approved google plugin and docs path that supports service-account ADC through OpenClaw-owned Vertex transport/proxy semantics, or clearly documents proxychains as unsupported with actionable diagnostics.

## Reproduction Assessment

Unclear for the full proxychains 401 because live proxychains and Vertex credentials were not available. Source inspection does show a current-main gap: google-vertex can be marked configured by any ADC file, but the provider-owned Vertex transport only accepts authorized_user ADC.

## Solution Assessment

Unclear; current main is only a partial authorized_user ADC fix, and #65023 is still an open candidate. The narrow maintainable direction should be chosen by maintainers because this touches provider auth, service-account credentials, and proxy policy.
