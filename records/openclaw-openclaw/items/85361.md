## Summary

Keep open: this is a focused, plausible Telegram bug fix with sufficient real behavior proof and no blocking code finding, but it needs maintainer acceptance of the direct-DM deliver-all policy because it changes overlapping same-session reply semantics.

## What This Changes

The branch makes normal Telegram direct-chat turns non-superseding in the reply-fence policy and adds a regression test that two overlapping same-session DM replies are both delivered.

## Best Possible Solution

Land a Telegram-owned fix that keeps normal direct-chat replies deliverable while preserving authorized /stop cancellation, group latest-wins behavior, and regression coverage for the direct-DM overlap case.

Yes, with maintainer policy sign-off: using non-interrupting fences for normal direct chats is a narrow Telegram-layer fix that preserves group latest-wins and /stop behavior. The remaining question is whether deliver-every-reply is the desired direct-DM invariant.
