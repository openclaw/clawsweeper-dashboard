## What This Changes

This PR changes `openclaw logs` to use local-time formatting by default and replaces `--local-time` with a `--utc` option in `src/cli/logs-cli.ts`.

## Best Possible Solution

Land a focused CLI change that defaults human-readable logs to local time, keeps `--local-time` accepted for compatibility, adds a clear UTC opt-out, and updates docs/tests with real behavior proof.

## Reproduction Assessment

Yes, at source level. Current main parses the local `time` field but formats it through UTC unless `--local-time` or a valid `TZ` sets `localTime`, and tests assert UTC default output.

## Solution Assessment

No. Default-local may be the right direction, but this PR is not the narrowest compatible solution because it removes a released flag and omits the docs, tests, and real behavior proof needed for the changed CLI contract.
