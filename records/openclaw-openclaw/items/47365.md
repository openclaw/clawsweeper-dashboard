## What This Changes

The PR changes `openclaw logs` to render local-time timestamps by default, keeps `--local-time` accepted, adds `--utc`, and includes unrelated image parser type cleanup.

## Best Possible Solution

Make the logs timestamp contract explicit: either keep UTC as the compatible default, or land default-local only with maintainer signoff, docs/tests for both modes, and redacted real non-UTC log-stream proof.

## Reproduction Assessment

## Solution Assessment

No, not as currently proposed. The `!opts.utc` code path is small, but the best solution needs maintainer acceptance of changing a shipped CLI default plus docs, tests, and real output proof before merge.
