## Summary

Keep this PR open for normal maintainer review: current main and the latest release still have the old UTC-default behavior, but the branch is not ready to land because the supplied proof does not demonstrate the changed timestamp rendering and the docs/tests still describe the old contract.

## What This Changes

The PR changes `openclaw logs` to default human-readable timestamps to local time, keeps `--local-time` accepted, adds `--utc`, and removes redundant `unknown` unions in image response parsing.

## Best Possible Solution

Land or replace this with a narrow logs-only change that defaults human-readable `openclaw logs` output to local time, preserves compatibility, documents and tests `--utc`, and leaves console-log scope separate.

## Reproduction Assessment

Yes, at source level. Current main and `v2026.5.12` parse the local-offset `time` field but render human-readable output as UTC unless `--local-time` or a valid `TZ` enables local mode.

## Solution Assessment

No, not as submitted. The core `localTime = !opts.utc` direction is narrow, but the PR still needs docs, focused default-local and `--utc` coverage, stronger real behavior proof, and maintainer acceptance of the CLI default change.
