## Summary

Keep open: the default-local logs request is still not implemented on current main, but this PR is not merge-ready because it removes the shipped `--local-time` CLI flag, leaves docs/tests behind the new contract, and lacks after-fix real behavior proof.

## What This Changes

This PR changes `openclaw logs` to render local timezone timestamps by default and replaces the existing `--local-time` opt-in with a new `--utc` flag.

## Best Possible Solution

Land a focused logs CLI change that defaults human-readable output to local time, keeps `--local-time` accepted for compatibility, adds an explicit UTC opt-out, and updates docs plus focused CLI tests together.

## Reproduction Assessment

Yes, at source level. Current main leaves `localTime` false unless `--local-time` or a valid `TZ` is present, and `formatLogTimestamp()` then emits UTC output; I did not execute the CLI because this review is read-only.

## Solution Assessment

No. Defaulting logs to local time is a reasonable direction, but removing the released `--local-time` flag is not the narrowest compatible solution; keep it as an accepted alias/no-op while adding `--utc`.
