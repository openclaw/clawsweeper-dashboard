## Summary

Keep open. The useful default-local direction still targets the open logs-timezone issue, but this branch drops the shipped `--local-time` CLI flag, leaves docs/tests on the old contract, and lacks after-fix real behavior proof.

## What This Changes

The PR changes `openclaw logs` timestamp rendering to local time by default, removes `--local-time`, and adds `--utc` as the UTC opt-out in `src/cli/logs-cli.ts`.

## Best Possible Solution

Land one focused logs CLI change that defaults human-readable output to local time, keeps `--local-time` accepted for compatibility, adds an explicit UTC opt-out, and updates tests, docs, and changelog together.

## Reproduction Assessment

Yes, at source level. Current main `openclaw logs --plain` without `--local-time` and without a valid `TZ` leaves `localTime` false and formats with `toISOString()`; I did not execute the CLI because this review is read-only.

## Solution Assessment

No. Defaulting human-readable logs to local time is a reasonable direction, but deleting the shipped `--local-time` flag is not the narrowest compatible implementation; keep it as an alias/no-op or use an explicit deprecation path while adding `--utc`.
