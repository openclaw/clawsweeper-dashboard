## Summary

Keep this PR open, but do not merge it as-is. Current main still has the reported default-UTC behavior, so the PR is not obsolete; the blocker is that the branch removes the already shipped and documented `--local-time` CLI flag while adding `--utc`, and it overlaps the open alternative PR #46782 for the same linked issue.

## What This Changes

This PR changes `openclaw logs` so human-readable timestamps default to local time by replacing the `--local-time` option/type with `--utc` and deriving display mode from `!opts.utc` in `src/cli/logs-cli.ts`.

## Best Possible Solution

Land one coordinated logs CLI change that defaults human-readable output to local time, keeps `--local-time` accepted as a compatibility alias or follows an explicit deprecation path, exposes a clear UTC opt-out, updates tests/docs/changelog, and then resolves #46748 plus the overlapping PR #46782.

## Reproduction Assessment

Yes. Current main provides a source-level reproduction: `formatLogTimestamp` returns UTC ISO output by default, and `registerLogsCli` only enables local time through `--local-time` or a valid `TZ` environment value.

## Solution Assessment

No. Defaulting logs to local time is a reasonable direction, but replacing the released `--local-time` flag outright is not the safest maintainable fix; preserving compatibility while adding an explicit UTC opt-out is narrower.
