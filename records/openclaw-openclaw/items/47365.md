## Summary

Keep open: current main still defaults human-readable `openclaw logs` timestamps to UTC unless `--local-time` or a valid `TZ` is present, so the linked user problem remains. This branch is not merge-ready because it removes the released `--local-time` flag, leaves the docs/tests on the old contract, lacks contributor real behavior proof, and is currently conflicting with main.

## What This Changes

This PR changes `openclaw logs` to default timestamp rendering to local time by replacing `--local-time` with a new `--utc` option in `src/cli/logs-cli.ts`.

## Best Possible Solution

Refresh this as a logs-only change that preserves `--local-time` as an accepted compatibility flag, adds a clear UTC opt-out, updates docs/tests, and includes redacted real CLI output for both modes.

## Reproduction Assessment

Yes, at source level. Current main registers `--local-time` as the local-rendering path, computes local mode only from that flag or valid `TZ`, and focused tests/docs still encode UTC-by-default plus local-time opt-in behavior.

## Solution Assessment

No. The default-local direction may be right, but this branch is not the narrowest maintainable solution because it removes a released flag and does not update the docs/tests/proof required for the new CLI contract.
