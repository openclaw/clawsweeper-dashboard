## What This Changes

The branch changes `openclaw logs` to render local timestamps by default, adds `--utc`, keeps `--local-time` accepted, and carries image-response typing cleanup already present on main.

## Best Possible Solution

Make the logs timestamp contract explicit: either preserve UTC as the compatible default, or land default-local only with maintainer signoff, docs/tests for both modes, and redacted real non-UTC log-stream proof.

## Reproduction Assessment

Yes, at source level. Current main parses the local-offset `time` field, but `registerLogsCli` enables local rendering only from `--local-time` or valid `TZ`, and `formatLogTimestamp` returns UTC when local mode is false.

## Solution Assessment

Unclear as-is. The code path is small, but changing a shipped CLI output default and adding `--utc` is only the best solution if maintainers accept the compatibility tradeoff and the PR adds docs, tests, and real output proof.
