## What This Changes

The PR changes `openclaw logs` to render local-time timestamps by default, adds `--utc` while keeping `--local-time`, and carries already-on-main image response type cleanup.

## Best Possible Solution

Make the logs timestamp contract explicit: either preserve UTC as the compatible default, or land default-local only with maintainer signoff, docs/tests for both modes, and redacted real non-UTC log-stream proof.

## Reproduction Assessment

Yes, at source level. Current main parses the local-offset `time` field, but `registerLogsCli` enables local rendering only from `--local-time` or valid `TZ`, and `formatLogTimestamp` returns UTC when local mode is false.

## Solution Assessment

Unclear. The code path is small, but changing a shipped CLI default and adding `--utc` is only the best solution if maintainers accept the compatibility tradeoff and the PR adds docs, tests, and real output proof.
