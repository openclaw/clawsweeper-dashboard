## Summary

Keep open: the PR targets a valid logs timestamp pain point, but the patch is not merge-ready because it removes the shipped `--local-time` option and does not update the published docs, tests, or changelog for the new default/`--utc` contract.

## What This Changes

This PR changes `openclaw logs` to render local timestamps by default, adds `--utc` as the UTC opt-out, and removes the existing `--local-time` option from `src/cli/logs-cli.ts`.

## Best Possible Solution

Land one coordinated logs CLI update that defaults human-readable `openclaw logs` output to local time, keeps `--local-time` accepted for compatibility, adds a clear UTC opt-out, and updates focused tests, docs, and changelog together.

## Reproduction Assessment

Yes, at source level. Current main still formats plain log timestamps as UTC unless `--local-time` or a valid `TZ` enables local formatting, and the PR diff visibly removes the released compatibility flag.

## Solution Assessment

No. Defaulting logs to local time is a reasonable direction, but replacing the shipped `--local-time` flag outright is not the narrowest maintainable fix; add `--utc` while preserving or deliberately deprecating the old flag.
