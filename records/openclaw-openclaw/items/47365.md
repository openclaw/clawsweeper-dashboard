## Summary

Keep open: the logs timezone request is valid and current main still defaults human-readable log output to UTC unless `--local-time` or a valid `TZ` is present, but this branch is not merge-ready because it removes the shipped `--local-time` CLI flag while adding `--utc`.

## What This Changes

The branch changes `src/cli/logs-cli.ts` so `openclaw logs` defaults human-readable output to local time, removes `--local-time`, and adds `--utc` as the UTC opt-out.

## Best Possible Solution

Land one coordinated logs CLI change that defaults display output to local time, keeps `--local-time` accepted for compatibility, adds a clear UTC opt-out, updates docs/tests/changelog, and resolves #46748 plus the overlapping #46782 path.

## Reproduction Assessment

Yes. Current main provides a source-level reproduction: `formatLogTimestamp` returns UTC ISO output by default, and `registerLogsCli` only enables local display through `--local-time` or a valid `TZ`.

## Solution Assessment

No. Defaulting logs to local time is a reasonable fix, but removing the released `--local-time` flag is not the safest implementation; preserving it as an alias or compatibility no-op while adding an explicit UTC opt-out is narrower.
