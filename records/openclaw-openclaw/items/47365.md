## Summary

Keep open: current main still has the UTC-by-default behavior this PR targets, but the patch is not merge-ready because it removes the shipped and documented `--local-time` flag while adding a new UTC opt-out without the matching docs/tests/changelog updates.

## What This Changes

The branch modifies `src/cli/logs-cli.ts` to default `openclaw logs` timestamp formatting to local time and replace `--local-time` with a new `--utc` flag.

## Best Possible Solution

Land one coordinated logs CLI update that defaults display output to local time, keeps `--local-time` accepted for compatibility, adds a clear UTC opt-out, and updates tests, docs, and changelog together.

## Reproduction Assessment

Yes, at source level. Current main returns UTC ISO output by default and only enables local display through `--local-time` or a valid `TZ`; I did not execute the CLI because this was a read-only review.

## Solution Assessment

No. Defaulting logs to local time is a reasonable fix direction, but removing the released `--local-time` flag is not the narrowest maintainable implementation.
