## What This Changes

The branch changes `openclaw logs` to use local-time timestamp rendering by default, adds a `--utc` opt-out while keeping `--local-time`, and narrows two image response parser type signatures.

## Best Possible Solution

Make the logs timestamp contract explicit: either keep UTC as the compatible default, or land default-local only with maintainer signoff, docs/tests for both modes, and redacted real non-UTC log-stream proof.

## Reproduction Assessment

Yes, at source level. Current main parses the local-offset `time` field, but `formatLogTimestamp` returns UTC unless local-time mode is enabled by `--local-time` or a valid `TZ`.

## Solution Assessment

Unclear. The `!opts.utc` code path is small, but the best solution depends on maintainer acceptance of changing a shipped CLI default and needs docs, tests, and real output proof before merge.
