## Summary

Keep open: current main still uses UTC by default unless `--local-time` or a valid `TZ` is present, and this PR is the active implementation for the linked logs-timezone issue. It needs real behavior proof plus docs/tests for the new CLI contract before merge.

## What This Changes

The PR changes `openclaw logs` to default human-readable timestamps to local time, keeps `--local-time` accepted, and adds `--utc` for UTC output.

## Best Possible Solution

Land a refreshed logs-only change that defaults human-readable logs to local time, preserves `--local-time` compatibility, documents `--utc`, adds focused regression coverage, and includes redacted real CLI output for both modes.

## Reproduction Assessment

Yes, at source level. Current main formats default plain output with `parsed.toISOString()` and only enables local rendering from `--local-time` or a valid `TZ`, so the old UTC-default behavior is clear without running a live gateway.

## Solution Assessment
