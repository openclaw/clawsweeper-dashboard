## Summary

Keep open. The PR is aimed at a real remaining behavior gap: current main still formats an ended run with pending cleanup as done and does not count a direct cleanup-pending child as active. The branch is not merge-ready as-is because current main has moved the central list/status paths, so the fix needs a rebase/update and maintainer review, especially against the later quota concern in #66187.

## What This Changes

The PR adds cleanupCompletedAt and suppressAnnounceReason checks to subagent active-run predicates/status formatting, plus regression tests, so ended runs stay active/completing until cleanup finalizes.

## Best Possible Solution

Rebase the fix onto current main and centralize the cleanup-pending active predicate so the subagents tool, slash-command list, status summary, target ordering, and countActiveRunsForSession all agree. Add focused tests for direct cleanup-pending runs, suppressed steer/killed runs, and quota/list behavior, then reconcile the intended spawn-limit semantics with #66187.

## Reproduction Assessment

Yes. A source-level reproduction is a SubagentRunRecord with endedAt set, cleanupCompletedAt absent, suppressAnnounceReason absent, and no pending descendants: current main formats it as done and excludes it from direct active-run counts.

## Solution Assessment

Unclear as submitted. The direction is plausible and addresses the duplicate-orchestrator race, but the branch needs a current-main rebase and a shared predicate update across the moved paths before it is the narrowest maintainable fix.
