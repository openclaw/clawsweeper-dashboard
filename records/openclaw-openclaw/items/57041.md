## Summary

Keep open. The reported cleanup-pending status race is still source-reproducible on current main, but this PR is merge-conflicting and updates older paths while current list/status behavior now runs through moved helpers; it also needs maintainer policy judgment against the open quota/zombie report.

## What This Changes

The PR adds cleanup-pending checks to subagent active-run predicates and status formatting so ended runs without `cleanupCompletedAt` stay active and show `completing`.

## Best Possible Solution

Centralize one cleanup-aware subagent lifecycle/status predicate on current main, then apply it consistently to list/info/status/tool output and registry quota only after maintainers decide the quota semantics against the related zombie-run report.

## Reproduction Assessment

Yes, at source level. A `SubagentRunRecord` with `endedAt` set, no `cleanupCompletedAt`, no `suppressAnnounceReason`, and no pending descendants currently formats as `done` and is excluded from the direct active list/count paths.

## Solution Assessment

Unclear as submitted. The direction addresses a real cleanup-pending race, but the branch must be rebased onto the moved helpers and maintainers need to choose whether cleanup-pending ended runs should count against spawn quota.
