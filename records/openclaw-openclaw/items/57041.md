## Summary

Keep open. The cleanup-pending race is still source-reproducible on current main, but this branch targets older status/list helpers and needs a rebase, contributor real-behavior proof, and maintainer judgment on the cleanup-pending quota boundary before merge.

## What This Changes

The PR changes subagent run liveness and status formatting so ended runs without completed cleanup remain active and display `completing`, with focused regression tests.

## Best Possible Solution

Centralize cleanup-aware liveness/status in the current subagent list, registry counter, status line, and shared formatter paths, after maintainers decide how cleanup-pending ended runs should affect spawn quota.

Yes, at source level. A `SubagentRunRecord` with `endedAt`, no `cleanupCompletedAt`, no `suppressAnnounceReason`, and no pending descendants currently formats as `done` and is excluded from the direct active list/status count paths.

## Solution Assessment

No as submitted. The direction addresses a real race, but the patch must be rebased onto the moved current-main helpers and the cleanup-pending active/quota boundary needs maintainer judgment.
