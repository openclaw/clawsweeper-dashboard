## Summary

Keep open: current main still resolves model params and status from the active `openai-codex` runtime provider, so canonical `openai/gpt-5.5` `textVerbosity` can be missed; the linked closing PR is open and not merged.

## What This Changes

The issue requests that canonical `openai/gpt-5.5` model params, especially `textVerbosity`, be honored when a Codex OAuth session runs through `openai-codex`.

## Best Possible Solution

Land a compatibility-tested fix that resolves params/status from the selected canonical model ref while preserving legacy `openai-codex/*` behavior, then close this issue after the fix merges.

## Reproduction Assessment

Yes, from source. The documented `openai/gpt-5.5` Codex route can rewrite the active provider to `openai-codex`, and the resolver/status paths then miss the canonical model params before defaulting GPT-5 verbosity to `low`.

## Solution Assessment

Yes, the maintainable solution is to fix the parameter lookup/status boundary rather than auth routing itself. The open linked PR should be reviewed and adjusted as needed before this issue closes.
