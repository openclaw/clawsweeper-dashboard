## Summary

Keep open: current main still has the source-proven Windows node service lifecycle bug, and https://github.com/openclaw/openclaw/pull/85597 is an open closing fix PR that should resolve this issue after merge.

## What This Changes

The issue reports Windows `openclaw node start/stop/restart` disrupting WSL2 loopback Gateway reachability when the Windows node service uses a Gateway running inside WSL2.

## Best Possible Solution

Land the scoped daemon fix in https://github.com/openclaw/openclaw/pull/85597, then close this issue once the fix reaches main and, ideally, a release.

## Reproduction Assessment

Yes, source-reproducible with high confidence: current main routes node service stop/restart through shared Gateway Scheduled Task port cleanup, and the node command’s `--port` is the remote Gateway connection port. I did not run a live Windows/WSL2 VM repro in this review.

## Solution Assessment

Yes. The narrow maintainable fix is to use existing service-kind metadata and node command matching so node lifecycle cleanup does not reclaim Gateway listener ports, while preserving Gateway service cleanup.
