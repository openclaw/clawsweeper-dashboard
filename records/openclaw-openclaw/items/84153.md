## Label Justifications

- `P1`: A stuck compaction lifecycle hook can block serialized Codex notification processing and leave an agent turn unresponsive.
- `merge-risk: 🚨 compatibility`: Existing plugins that relied on unbounded compaction hook completion may now time out after 30 seconds unless they configure an override.
- `merge-risk: 🚨 session-state`: Compaction hooks may flush memory or session-related state, and fail-open timeout can let OpenClaw proceed before slow hook work finishes.

## Summary

No blocking findings; keep this PR open for the existing automerge/exact-head gate. The source path is real and the patch is narrow, with the main merge question being the intentional 30-second fail-open compatibility tradeoff for slow compaction hooks.

## What This Changes

The PR adds default 30-second fail-open timeouts for `before_compaction` and `after_compaction` plugin hooks, plus focused regression tests and a changelog entry.

## Best Possible Solution

Land the bounded default if maintainers accept 30 seconds as the lifecycle-hook default, and rely on the existing per-hook timeout override path for legitimately slow compaction plugins.

## Reproduction Assessment

Yes. Source inspection shows current main routes both compaction lifecycle hooks through `runVoidHook`, and with no default or registration timeout a never-settling handler keeps the awaited caller open; I did not run a live hanging turn because this review is read-only.

## Solution Assessment

Yes, if maintainers accept the timeout tradeoff. Adding these two hooks to the existing void-hook default table is the narrowest maintainable fix and preserves existing per-hook override mechanisms.
