## Summary

Keep open for merge review: the patch is source-aligned and has sufficient terminal proof now, but it intentionally changes compaction lifecycle hooks from unbounded to a 30s fail-open default, so maintainers should own that compatibility tradeoff before merge.

## What This Changes

The PR adds 30-second default timeouts for `before_compaction` and `after_compaction` void plugin hooks and adds Vitest coverage for hung and fast handlers.

## Best Possible Solution

Land the timeout-table fix if maintainers accept the 30s fail-open default, while preserving the documented per-plugin and per-hook timeout override path for slow compaction plugins.

## Reproduction Assessment

Yes. Source inspection shows current main awaits `before_compaction` and `after_compaction` through `runVoidHook` with no default timeout, including the Codex app-server `contextCompaction` notification path; I did not run a live hung plugin in this read-only review.

## Solution Assessment

Yes, subject to maintainer approval. Reusing the existing default void-hook timeout table is the narrow maintainable fix, and the documented hook timeout override remains available for plugins that need a longer compaction budget.
