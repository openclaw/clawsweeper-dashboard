## Summary

Keep open for merge processing: the patch addresses a source-proven unbounded compaction hook path with a narrow timeout-table change, supplied real hook-runner proof, and has no blocking review findings; the remaining concern is the explicit compatibility/session-state tradeoff of changing these hooks from unbounded to fail-open after 30 seconds.

## What This Changes

The PR adds 30-second default void-hook timeouts for `before_compaction` and `after_compaction` and adds focused Vitest coverage for hung and fast compaction hook handlers.

## Best Possible Solution

Merge the bounded default if maintainers accept 30 seconds as the lifecycle-hook safety budget, while preserving the documented per-hook timeout override for legitimately slow compaction plugins.

## Reproduction Assessment

Yes. Source inspection shows current main awaits `before_compaction` and `after_compaction` through `runVoidHook`, and without a default or registration timeout a never-settling handler keeps the caller open; the PR body adds live hook-runner output showing the fixed timeout path.

## Solution Assessment

Yes. Adding these hooks to the existing void-hook default timeout table is the narrowest maintainable fix and matches the `agent_end` precedent, with the remaining compatibility tradeoff made explicit for maintainers.
