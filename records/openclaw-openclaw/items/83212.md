## Summary

Keep open: current main and the latest shipped release still have a silent mismatch between the Discord setup docs and plugin trust/load policy for the external Discord plugin.

## What This Changes

The issue asks OpenClaw to warn, fail fast, or enable the Discord plugin when `channels.discord.enabled` is true but the explicit plugin entry is missing.

## Best Possible Solution

Keep the explicit trust boundary for external plugins, but make Discord setup and diagnostics require or write the explicit Discord plugin entry instead of letting the channel appear silently disabled.

## Reproduction Assessment

Yes from source, though not from a live gateway run: current and shipped code require explicit trust for the external Discord plugin while the Discord docs configure only `channels.discord`. The existing doctor warning path does not cover a missing plugin entry.

## Solution Assessment

Yes for an operator-facing warning, validation error, or setup/doc repair; no for a blanket implicit enable that relaxes external plugin trust. The narrowest maintainable fix is to diagnose or write the missing explicit Discord plugin enablement for official external channel setup.
