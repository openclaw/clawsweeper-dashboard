## Summary

Keep open: current main still lets a Discord channel config miss the explicit external-plugin trust path without a doctor warning, and an open draft PR already references this issue as the fix path.

## What This Changes

The issue requests an operator-facing warning, fail-fast error, or setup/docs repair when `channels.discord.enabled` is set but the external Discord plugin is not explicitly enabled or trusted.

## Best Possible Solution

Keep the external-plugin trust boundary, but make doctor/setup/docs surface or write the required explicit Discord plugin enablement so configured Discord does not appear silently disabled.

## Reproduction Assessment

Yes from source: the docs show a channel-only Discord config, current policy requires explicit trust for the external Discord plugin, and doctor preview warnings skip the missing-entry case. I did not run a live gateway reproduction in this read-only pass.

## Solution Assessment

Yes for a focused diagnostic/setup repair; no for a blanket implicit enable that would weaken the external-plugin trust model. The narrow maintainable fix is to warn, validate, or write the explicit Discord plugin trust entry in the setup/doctor path.
