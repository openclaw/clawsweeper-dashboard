## What This Changes

The issue asks OpenClaw to warn, fail clearly, or otherwise avoid silently leaving Discord disabled when `channels.discord.enabled` is set without explicit external plugin trust under `plugins.entries.discord.enabled` or `plugins.allow`.

## Best Possible Solution

Keep the external plugin trust boundary, but land https://github.com/openclaw/openclaw/pull/83252 or a replacement that wires the missing-trust diagnostic through the real doctor/setup path and adds focused regression coverage.

## Reproduction Assessment

Yes, from source rather than a live gateway run. Current main requires explicit trust for the external Discord plugin while the Discord setup docs configure only `channels.discord`, and the doctor blocker path exits before checking a missing plugin entry.

## Solution Assessment

Yes for an operator-facing warning, validation/setup diagnostic, or setup repair; no for blanket implicit enablement that weakens external plugin trust. The narrow maintainable fix is to diagnose or write the missing explicit Discord plugin enablement for official external channel setup.
