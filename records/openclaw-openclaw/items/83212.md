## What This Changes

The issue reports that Discord can be configured at `channels.discord` while the external Discord plugin remains disabled unless `plugins.entries.discord.enabled` or `plugins.allow` explicitly trusts it.

## Best Possible Solution

Keep the external plugin trust boundary, then land a diagnostic or setup repair that tells operators to set `plugins.entries.discord.enabled=true` or `plugins.allow` when Discord is configured.

## Reproduction Assessment

Yes, from source inspection. Current main and `v2026.5.12` configure Discord in docs without plugin trust, require explicit trust for non-bundled channel owners, and skip the doctor warning path when the plugin entry is missing.

## Solution Assessment

Yes for an operator-facing warning, validation/setup repair, or doctor fix; no for implicit enablement that weakens plugin trust. The narrowest maintainable solution is to diagnose or write the missing explicit Discord plugin enablement.
