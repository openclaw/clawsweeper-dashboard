## Summary

Keep open: the PR fixes the default wildcard runtime-policy gap, but the branch still misorders scoped agent wildcard precedence and needs a small docs update before merge.

## What This Changes

The PR adds `provider/*` matching to agent model runtime policy resolution, adds resolver and harness-selection tests, and records the fix in the changelog.

## Best Possible Solution

Keep the focused runtime-policy fix, reorder scoped wildcard precedence before default model entries, document the new wildcard runtime-policy form, and land one canonical fix for https://github.com/openclaw/openclaw/issues/82243.

## Reproduction Assessment

Yes, at source level. Current main only exact-matches model-map runtime keys, so `agents.defaults.models["vllm/*"].agentRuntime` does not apply when resolving `vllm/qwen-local`, and harness selection calls that resolver.

## Solution Assessment

No, not quite. The direction is right, but this branch checks default exact entries before scoped agent wildcards; the safer implementation preserves per-agent override precedence and adds docs for the new supported wildcard form.
