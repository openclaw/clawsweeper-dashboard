## Summary

Keep open. Current main still renders the floating message-level chat actions without applying the existing spacing class to normal assistant bubbles, and this PR is a focused layout fix for that gap. The newer open #73633 addresses code-copy payload resolution, but its diff does not reserve bubble space, so it does not supersede this overlap fix.

## What This Changes

The PR adds the existing `has-copy` class to Control UI chat bubbles whenever assistant copy or expand actions are rendered.

## Best Possible Solution

Land this fix or an equivalent patch on current main by adding `has-copy` under the same condition used to render `.chat-bubble-actions`, preserving the current `chat-bubble--tool-shell` class, and adding a small DOM regression test for assistant bubbles with and without actions. Treat #73633 as a separate handler-fidelity candidate rather than a replacement for the spacing fix.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction path exists: render an assistant message with fenced code so the code-block header reaches the bubble edge, then show the message-level copy/expand actions; current main lacks `has-copy`, so the existing absolute action bar can overlap the code-block copy control described in #69605.

## Solution Assessment

Yes, for the overlap root cause. Reusing the existing `.chat-bubble.has-copy` CSS contract under the exact action-rendering condition is the narrowest maintainable fix; the safer landing detail is to apply it to the current multiline class array and preserve the newer tool-shell class.
