## Summary

Keep open. Current main and the latest release still render message-level chat actions without applying the existing `has-copy` spacing class, and this PR remains the focused fix for that overlap; #73633 addresses copy-text resolution but does not reserve bubble space.

## What This Changes

The PR adds the existing `has-copy` class to Control UI chat bubbles when assistant copy or expand actions are rendered.

## Best Possible Solution

Land this fix or an equivalent current-main patch that adds `has-copy` under the same condition used to render `.chat-bubble-actions`, preserves `chat-bubble--tool-shell`, and includes the required changelog entry plus focused class-contract coverage.

## Reproduction Assessment

Yes. Render an assistant message with fenced code and message-level copy/expand actions; current main lacks `has-copy`, so the absolutely positioned `.chat-bubble-actions` can overlap the code-block copy control described in #69605.

## Solution Assessment

Yes for the overlap root cause. Reusing the existing `.chat-bubble.has-copy` CSS contract is the narrowest maintainable layout fix, with the landing detail that the branch needs a current-main refresh and changelog entry.
