## Summary

Keep this PR open: current main still lacks the `has-copy` spacing class while rendering the floating message actions, and the related open handler PR does not cover this layout contract. The code change looks correct, but this external PR still needs real after-fix Control UI proof before merge.

## What This Changes

The PR adds the existing `has-copy` class to Control UI assistant chat bubbles when message-level copy or expand actions are rendered.

## Best Possible Solution

Land this layout fix or an equivalent current-main patch that adds `has-copy` under the same condition used for `.chat-bubble-actions`, preserving current bubble classes and adding focused class-contract coverage if maintainers want a guardrail.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: render an assistant Control UI message with fenced code and message-level copy or expand actions, and current main can place the absolute `.chat-bubble-actions` over code-block chrome because the bubble never gets `has-copy`. I did not run a live browser repro in this read-only review.

## Solution Assessment

Yes. Reusing the existing `.chat-bubble.has-copy` CSS contract is the narrowest maintainable fix for the overlap root cause, and it avoids changing markdown parsing or clipboard behavior.
