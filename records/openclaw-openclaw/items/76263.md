## Summary

Keep open. Current main still lets WhatsApp direct chats inherit the shared-main DM default, and #76314 is an open closing PR that should be reviewed and landed or rejected before cleanup.

## What This Changes

The issue requests per-contact WhatsApp direct-chat session keys so distinct senders no longer share `agent:<agentId>:main` transcripts and model context.

## Best Possible Solution

Land a maintainer-approved WhatsApp isolation change with regression coverage and migration guidance for deployments that intentionally relied on shared DM continuity.

## Reproduction Assessment

Yes. Source inspection is enough: with `session.dmScope` unset, WhatsApp direct messages pass a direct peer to `resolveAgentRoute`, which defaults to `main` and returns the agent main session key.

## Solution Assessment

Unclear. Per-channel peer scoping is technically narrow and uses existing session-key support, but changing WhatsApp's default conflicts with documented shared-main continuity and needs maintainer security/product approval.
