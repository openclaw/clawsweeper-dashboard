## Label Justifications

- `impact:session-state`: The affected behavior is shared session keys, JSONL transcripts, and model context across distinct WhatsApp contacts.

## Summary

Keep open: current main and the latest release still allow WhatsApp direct inbound chats to inherit the shared `main` DM session lane, while the account-aware fix PR remains open and conflicting.

## What This Changes

The issue reports that WhatsApp direct chats for different contacts can share one agent session key, transcript, and model context instead of using per-contact session isolation.

## Best Possible Solution

Land a narrow account-aware WhatsApp direct-session isolation fix, with last-route metadata and upgrade behavior covered, then close this issue from the merged fix.

## Reproduction Assessment

Yes at source level. Current main routes WhatsApp direct inbound messages through `resolveAgentRoute`, the resolver defaults unset `dmScope` to `main`, and the current test contract still expects `agent:main:main` for an unconfigured WhatsApp direct peer; I did not run a live WhatsApp account in this read-only review.

## Solution Assessment

Yes, account-aware per-contact WhatsApp DM isolation is the right fix direction because it uses the existing per-account session-key contract instead of changing the global DM default. The remaining decision is upgrade/session compatibility, not whether the bug is still present.
