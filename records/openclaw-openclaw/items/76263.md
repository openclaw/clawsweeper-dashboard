## Summary

Keep open. Current `main` still routes WhatsApp direct messages to the shared agent main session when `session.dmScope` is unset, so the reported multi-contact context-sharing risk remains real and needs maintainer policy on security defaults and compatibility.

## What This Changes

The issue requests per-chat WhatsApp session keys so distinct direct contacts and groups do not share one `agent:${agentId}:main` transcript/context by default.

## Best Possible Solution

Multi-user WhatsApp deployments should end up with isolated direct-message sessions by default or an explicit secure-mode policy, plus migration guidance for users who intentionally keep shared main-session continuity.

## Reproduction Assessment

Yes. Source inspection is enough: with `session.dmScope` unset, WhatsApp direct inbound routing passes a direct peer into `resolveAgentRoute`, which defaults to `main` and returns the agent main session key; setting `session.dmScope` to a per-peer mode changes the key.

## Solution Assessment

Unclear. A WhatsApp-only default flip would address the reported leak, but the current security guide documents shared DM continuity as the product default, so maintainers should decide the default, migration path, and release-note policy before code changes.
