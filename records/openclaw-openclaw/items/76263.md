## Label Justifications

- `P1`: The report describes a live WhatsApp channel workflow that can mix distinct contacts' transcripts and model context.
- `impact:session-state`: The affected behavior is shared session keys, JSONL transcripts, and model context across distinct WhatsApp contacts.
- `impact:security`: Shared context can expose one contact's conversational information through another contact's model reply.

## Summary

Keep open: current main and the latest release still route WhatsApp direct inbound messages through the global `session.dmScope` default, so the reported shared-session behavior remains; the active closing PR is open, conflicting, and still needs maintainer review.

## What This Changes

Make WhatsApp inbound direct messages use chat/account-scoped session keys instead of the agent main session so distinct contacts do not share transcripts or model context.

## Best Possible Solution

Land a refreshed account-aware WhatsApp direct-session isolation fix with last-route metadata, upgrade behavior explicitly accepted, and real behavior proof, then close this issue after merge.

## Reproduction Assessment

Yes. This is source-reproducible: WhatsApp direct inbound routing calls `resolveAgentRoute` without a channel-owned direct-message scope override, and the core default collapses unset DMs to `agent:<agentId>:main`.

## Solution Assessment

Yes, account-aware per-contact WhatsApp DM isolation is the right fix shape; it is narrower than changing the global DM default, but maintainers still need to accept the upgrade/session-continuity tradeoff.
