## What This Changes

The issue asks WhatsApp inbound direct chats to resolve to per-contact/account scoped session keys instead of the shared agent main lane.

## Best Possible Solution

Review and land the account-aware WhatsApp direct-session isolation fix with broadcast coverage, docs/changelog, and migration guidance, then close this issue from the merged PR.

## Reproduction Assessment

Yes. Source inspection is enough: WhatsApp direct inbound messages pass a direct peer to resolveAgentRoute, the resolver defaults unset session.dmScope to main, and the direct-session builder returns the agent main session key under that scope.
