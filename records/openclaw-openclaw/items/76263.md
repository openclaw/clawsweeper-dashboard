- `impact:session-state`: The affected behavior merges separate WhatsApp contacts into one session key and transcript context.

## Summary

Keep open: current main and v2026.6.8 still route WhatsApp direct messages through the global `session.dmScope` default, and the open closing PR has not merged. This remains a high-confidence session isolation and security issue, not a cleanup close candidate.

## What This Changes

The issue requests WhatsApp inbound direct messages to use account/contact-scoped session lanes instead of the shared agent main lane.

## Best Possible Solution

Repair and land, or replace, the account-aware WhatsApp direct-message isolation PR so ordinary and broadcast WhatsApp DMs use per-account/per-contact session keys with focused tests and an explicit upgrade/session-continuity decision.

## Reproduction Assessment

Yes. Source inspection on current main shows WhatsApp direct messages pass a peer to `resolveAgentRoute` without an override, the resolver defaults `dmScope` to `main`, and the session-key helper returns the agent main key under that scope.

## Solution Assessment

No current-main solution exists yet. The linked account-aware PR is the right direction, but maintainers still need to resolve the conflicting branch and the upgrade/session-continuity tradeoff before this issue can close.
