## Summary

Keep open: current main and the latest release still route WhatsApp direct messages through the global DM-scope default, and the open closing PR has not merged. This is a high-confidence session isolation and security issue, not a stale cleanup candidate.

## What This Changes

The issue requests account/contact-scoped WhatsApp inbound direct-message session keys instead of routing distinct contacts into the shared agent main lane.

## Best Possible Solution

Repair and land, or replace, https://github.com/openclaw/openclaw/pull/76730 so ordinary and broadcast WhatsApp DMs use account-aware per-contact session keys with focused tests and an explicit upgrade/session-continuity decision.

## Reproduction Assessment

Yes. Source inspection on current main shows WhatsApp direct messages pass a direct peer to resolveAgentRoute without an override, the resolver defaults dmScope to main, and the session-key helper returns agent main for direct peers in that mode.

## Solution Assessment

No current-main solution exists yet. The linked account-aware PR is the right direction, but maintainers still need to resolve broadcast routing and upgrade/session-continuity before this issue can close.
