## Summary

Keep open. Current main still routes WhatsApp direct inbound messages through the shared-main DM default, and two open PRs already use closing references for this issue, with the account-aware maintainer-labeled PR being the stronger fix path.

## What This Changes

Request that WhatsApp inbound direct chats use per-account/contact session keys instead of the shared agent main session.

## Best Possible Solution

Review and land the account-aware WhatsApp isolation fix with docs/changelog and migration guidance, then close this issue from the merged PR.

## Reproduction Assessment

Yes. Source inspection of current main is enough: WhatsApp direct inbound messages pass a direct peer to resolveAgentRoute, the resolver defaults unset session.dmScope to main, and the direct-session builder returns the agent main session key under that scope.

## Solution Assessment

Unclear until maintainer review completes. Account-aware per-contact isolation is the best visible direction, but it changes the documented WhatsApp direct-chat default and needs compatibility/security approval before landing.
