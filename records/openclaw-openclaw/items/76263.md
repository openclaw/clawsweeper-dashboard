## Summary

Keep open. Current main and the shipped v2026.4.29 release already expose `session.dmScope` isolation and warn about multi-user DM context leakage, but WhatsApp direct chats still default to the shared agent main session; changing that default is a security/product compatibility decision, not a safe cleanup close.

## What This Changes

The issue asks WhatsApp inbound direct chats to use per-contact session keys instead of sharing the agent main session across contacts.

## Best Possible Solution

Multi-user WhatsApp deployments should end up with isolated DM session keys by default or through an explicit safe-mode configuration, with migration guidance for users who intentionally keep shared main-session continuity.

## Reproduction Assessment

Yes. Source inspection is enough to reproduce the reported default path: with `session.dmScope` unset, WhatsApp direct messages route through `resolveAgentRoute` and the session-key builder returns the agent main key; setting `session.dmScope` to an isolated mode changes the expected key.

## Solution Assessment

Unclear. A WhatsApp-only default flip may solve the SaaS leak, but current docs treat shared DM main sessions as intentional cross-channel continuity, so the safest fix needs maintainer approval for default, migration, and release-note policy.
