## Summary

Keep open: current main and v2026.6.10 still route WhatsApp direct inbound messages through the default `main` DM scope, and the account-aware closing PR is open, unmerged, and conflicting.

## What This Changes

The issue requests WhatsApp direct-message session isolation so distinct contacts do not share the same session transcript and model context.

## Best Possible Solution

Land a narrow account-aware WhatsApp direct-session isolation fix with last-route metadata, upgrade behavior, and proof covered, then close this issue after the PR merges.

## Reproduction Assessment

Yes. Source inspection shows WhatsApp direct inbound routing passes a direct peer into `resolveAgentRoute`, core defaults unset `session.dmScope` to `main`, and the current route test expects `agent:main:main`; I did not run a live WhatsApp account in this read-only review.

## Solution Assessment

Yes. Account-aware per-contact WhatsApp DM isolation is the best current fix shape; the earlier per-channel-peer attempt was too narrow because it omitted account id, while changing the global DM default would be broader than needed.
