## Summary

Keep open. Current main still collapses WhatsApp direct-message sessions to the agent main lane when `session.dmScope` is unset or `main`, and the report raises a real multi-customer privacy boundary question rather than a stale cleanup item.

## What This Changes

The issue requests WhatsApp inbound chats to resolve to per-contact or per-group session keys instead of sharing the default agent main session.

## Best Possible Solution

Define a safe WhatsApp/SaaS multi-user default: either isolate WhatsApp direct DMs by account and sender automatically for that deployment class, or enforce/configure `session.dmScope: "per-account-channel-peer"` with clear migration diagnostics.

## Reproduction Assessment

Yes. Source and tests provide a high-confidence reproduction: a WhatsApp direct peer with empty config resolves to `agent:main:main`, and the WhatsApp inbound path feeds direct contacts into that generic route.

## Solution Assessment

Unclear. The existing generic `session.dmScope` mechanism can isolate these chats, but the report challenges whether the documented default is safe for SaaS WhatsApp use; a default change needs maintainer review because it affects compatibility.
