## Summary

Keep open. Current main still routes WhatsApp direct chats through the documented shared-main DM default, and an open PR already uses closing syntax for this issue, so cleanup should wait for that PR to land or be rejected.

## What This Changes

The issue asks for WhatsApp inbound direct chats to receive chat-scoped session keys instead of sharing `agent:<agentId>:main` across contacts.

## Best Possible Solution

Review the open WhatsApp scoping PR and land a policy-approved isolation change, with migration guidance for operators who intentionally used shared DM continuity.

## Reproduction Assessment

Yes. Source inspection is enough: with `session.dmScope` unset, WhatsApp direct messages pass a direct peer to `resolveAgentRoute`, which defaults to `main` and returns the agent main session key.

## Solution Assessment

Unclear. Forcing `per-channel-peer` matches the recommended secure DM mode, but the current default is documented as shared continuity, so the default and migration policy need maintainer security/product approval.

## Security Review

Status: needs_attention

Summary: The issue describes a real multi-user privacy risk in the existing WhatsApp direct-message default path.
