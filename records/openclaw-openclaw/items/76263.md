## Summary

## What This Changes

The issue asks for WhatsApp inbound chats to get per-contact and per-account session isolation instead of sharing the agent main lane.

Repair, land, or replace https://github.com/openclaw/openclaw/pull/76730 with account-aware isolation for every WhatsApp direct-message path, including broadcast fan-out, then close this issue with merged-fix and release provenance.

## Solution Assessment

Yes, keeping this issue open for the linked fix path is the right maintainer action. The fix itself should be account-aware, cover broadcast/direct paths, and include upgrade-sensitive proof before the issue closes.
