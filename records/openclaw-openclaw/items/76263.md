## Summary

Keep open: current main and the latest release still route WhatsApp direct messages through global `session.dmScope` with the default `main` scope, and the open closing PR has not merged.

## What This Changes

The issue asks for WhatsApp inbound direct chats to use per-chat, account-aware session keys instead of sharing `agent:<agentId>:main`.

## Best Possible Solution

Repair and land, or replace, the account-aware WhatsApp DM isolation PR so ordinary and broadcast direct routes use isolated per-account/per-contact session keys with focused tests, then close this issue.

## Reproduction Assessment

Yes. Source inspection shows a WhatsApp direct peer reaches `resolveAgentRoute`, the resolver defaults `dmScope` to `main`, and the session-key helper returns `agent:<agentId>:main` for direct peers in that mode.

## Solution Assessment

Not yet. The account-aware linked PR is the right direction, but current main has not solved the issue and the PR still needs maintainer resolution for broadcast routing and upgrade continuity.
