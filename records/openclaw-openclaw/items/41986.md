## What This Changes

The issue asks for the Usage dashboard header cost and token totals to match the same selected date-range scope as the daily usage charts.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/70205 or an equivalent focused patch that makes `usage.cost` resolve, cache, and merge over the same configured-agent scope as `sessions.usage`, then separately verify any provider-specific unit normalization concern.

Yes. By source inspection, configure multiple gateway agents with usage-bearing transcripts outside the default agent, then request `sessions.usage` and `usage.cost` for the same range; current main fans out `sessions.usage` but keeps `usage.cost` on the default transcript scope.

## Solution Assessment

Yes. Aligning backend aggregation and cache scope is the narrow maintainable fix; frontend formatting alone would leave the dashboard's two data sources inconsistent.
