## What This Changes

The issue asks for the Usage dashboard header cost/token totals to match the daily cost/activity totals for the selected date range.

## Best Possible Solution

Review and land #70205 or an equivalent focused fix that makes `usage.cost` resolve, cache, and merge over the same configured-agent scope as `sessions.usage`, then separately check whether a provider cost-unit issue remains.

## Reproduction Assessment

Yes, by source inspection: current main still lets `sessions.usage` scan configured agents while `usage.cost` loads the cached cost summary without an agent scope. I did not live-run the reporter's MiniMax/dashscope dataset, so the exact displayed dollar magnitude is not verified.

## Solution Assessment

Yes. The narrow maintainable fix is to align `usage.cost` aggregation and cache scope with `sessions.usage`; frontend formatting alone would leave the two dashboard data sources inconsistent.
