## Summary

Keep open. Current main still has the dashboard split described in the report: the Usage UI renders header totals from `sessions.usage` while daily bars come from `usage.cost`, and the unscoped `usage.cost` path still loads one default transcript scope instead of the configured-agent scope that `sessions.usage` scans. The active open fix candidate is #70205; #42063 was a prior unmerged attempt in the same direction.

## What This Changes

The issue asks for Usage dashboard header cost/token totals to match the daily cost and activity totals for the selected date range.

## Best Possible Solution

Land #70205 or an equivalent gateway/UI fix that makes unscoped `usage.cost` aggregate across the same resolved configured-agent scope as `sessions.usage`, preserves explicit agent scoping, includes the resolved scope in cache keys, keeps costs in USD, and adds regression coverage for multi-agent date ranges. After the cost totals match, validate whether Activity by Time token totals need a separate follow-up.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence for multi-agent installs: current main requests `sessions.usage` and `usage.cost` separately, `sessions.usage` scans configured agents, and `usage.cost` still loads the default transcript directory when no `agentId` is provided.

## Solution Assessment

Yes, the linked PR direction is the narrowest maintainable fix for the confirmed bug. A frontend-only unit conversion would not address the proven endpoint-scope mismatch and the micro-dollar theory is not yet source-backed.
