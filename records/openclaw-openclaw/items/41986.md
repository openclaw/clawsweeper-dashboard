## Summary

Keep open: current main still has a source-visible Usage dashboard scope mismatch. The header totals come from `sessions.usage`, which scans configured agents, while `usage.cost` still reads the unscoped default cost cache; the focused fix is tracked by https://github.com/openclaw/openclaw/pull/70205 and is not shipped yet.

## What This Changes

The issue reports that the Usage dashboard header cost and token totals can be orders of magnitude larger than the daily chart and activity totals for the same date range.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/70205 or an equivalent focused patch that makes `usage.cost` resolve, cache, and merge over the same configured-agent scope as `sessions.usage`, then re-check whether any provider-specific unit normalization remains.

## Reproduction Assessment

Yes. By source inspection, configure multiple gateway agents with usage-bearing transcripts outside the default agent, then call `usage.cost` and `sessions.usage` for the same range; current main lets `sessions.usage` scan configured agents while `usage.cost` reads the unscoped default cost cache.

## Solution Assessment

Yes. Aligning backend aggregation and cache scope is the narrow maintainable fix; frontend formatting alone would leave the two dashboard data sources inconsistent.
