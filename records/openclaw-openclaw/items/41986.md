The issue asks for the Usage dashboard header totals and daily/activity charts to use consistent cost, token, and date-range semantics.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/70205 or an equivalent patch that makes `usage.cost` use the same configured-agent scope as `sessions.usage`, with scoped cache keys and UI date-label coverage.

## Solution Assessment
