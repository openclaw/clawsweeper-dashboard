## Summary

Keep open: current main still has the Usage page comparing `sessions.usage` totals from the configured-agent scope with `usage.cost` daily data loaded through the unscoped/default cost cache, and the focused implementation PR remains open rather than merged.

## What This Changes

The issue asks for the Usage dashboard header cost and token totals to be consistent with the daily cost chart and Activity by Time data for the selected date range.

## Best Possible Solution

Review and land #70205 or an equivalent focused fix that makes `usage.cost` resolve, cache, and merge over the same configured-agent scope as `sessions.usage`, then verify whether any provider-specific cost-unit issue remains.

## Reproduction Assessment

Yes, by source inspection for the current-main aggregation mismatch: `sessions.usage` scans configured agents, while `usage.cost` still reads the unscoped/default cost cache. I did not live-run the reporter's MiniMax dataset, so the exact displayed dollar magnitude remains a verification detail.

## Solution Assessment

Yes, the narrow maintainable fix is to align `usage.cost` aggregation and cache scope with `sessions.usage`; changing only frontend formatting would leave the endpoint mismatch intact.
