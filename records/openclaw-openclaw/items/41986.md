## Summary

Keep this open: current main still has a source-reproducible mismatch where the Usage header can use `sessions.usage` totals across configured agents while the daily cost chart uses the unscoped `usage.cost` summary, and open PR #70205 is the active fix path.

## What This Changes

The issue asks for the Usage dashboard header cost/tokens to align with the daily usage and activity breakdowns for the selected date range.

## Best Possible Solution

Review and land an equivalent of #70205 so unscoped `usage.cost` resolves the same configured-agent scope as `sessions.usage`, merges per-agent daily/totals, scopes the cache key, and keeps regression coverage.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: configure multiple agents with cost-bearing transcripts outside the default agent, then compare the Usage header from `sessions.usage` with the daily chart from unscoped `usage.cost`. I did not run a live UI or test reproduction in this read-only review.

## Solution Assessment

Yes, the narrowest maintainable fix is to make `usage.cost` use the same configured-agent scope as `sessions.usage` and merge the summaries; open PR #70205 follows that direction and should be reviewed rather than closing this issue now.
