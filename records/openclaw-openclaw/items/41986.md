## Summary

Keep open: current main still has a source-visible Usage endpoint scope mismatch, and the focused fix PR #70205 remains open rather than shipped.

## What This Changes

The issue asks for the Usage dashboard header cost and token totals to match the daily cost and activity totals for the same selected date range.

## Best Possible Solution

Review and land #70205 or an equivalent focused fix that makes `usage.cost` resolve, cache, and merge over the same configured-agent scope as `sessions.usage`, then separately verify whether a provider cost-unit issue remains.

## Reproduction Assessment

Yes, by source inspection: current main still lets `sessions.usage` scan configured agents while `usage.cost` reads the unscoped/default cost cache. I did not live-run the reporter's MiniMax dataset, so the exact displayed dollar magnitude remains unverified.

## Solution Assessment

Yes, the narrow maintainable solution is to align gateway `usage.cost` aggregation and cache scope with `sessions.usage`. Frontend formatting alone would leave the two dashboard data sources inconsistent.
