## Summary

Keep open: current main still has a source-visible Usage dashboard data-source mismatch, and the focused aggregation fix is tracked by an open PR rather than shipped behavior.

## What This Changes

The issue reports that Usage dashboard header cost and token totals are far larger than the daily cost chart and activity totals for the same selected range.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/70205 or an equivalent focused patch that makes `usage.cost` resolve, cache, and merge over the same configured-agent scope as `sessions.usage`, then verify whether any provider-specific unit normalization remains.

## Reproduction Assessment

Yes, by source inspection: configure multiple gateway agents with usage-bearing transcripts outside the default agent, then call `usage.cost` and `sessions.usage` for the same date range. Current main lets `sessions.usage` scan configured agents while `usage.cost` reads the unscoped default cost cache.

## Solution Assessment

Yes. The narrow maintainable solution is to align gateway `usage.cost` aggregation and cache scope with `sessions.usage`; frontend formatting alone would leave the two dashboard data sources inconsistent.
