## Summary

Keep open: current main still has a source-level mismatch between the Usage header scope and the daily cost endpoint, and the linked implementation PR is open rather than merged.

## What This Changes

The issue asks for the Usage dashboard header cost/token totals to match the selected date range's daily and activity aggregates.

## Best Possible Solution

Land or revise the focused fix so unscoped `usage.cost` resolves, caches, and merges the same configured-agent scope as `sessions.usage`, then separately track any independently reproduced cost-unit bug.

## Reproduction Assessment

Yes, by source inspection for the aggregation mismatch: the UI compares `sessions.usage` totals across configured agents with `usage.cost` daily data from an unscoped/default cache. I did not live-run the reporter's MiniMax dataset, so the exact million-dollar display remains a verification detail for the linked fix.

## Solution Assessment

Yes, the narrow maintainable direction is to align `usage.cost` agent scoping with the existing `sessions.usage` scope; the open #70205 direction matches that boundary without adding a new product mode.
