## Summary

Keep open: current main still lets the Usage page compare `sessions.usage` totals from the configured-agent scope with `usage.cost` daily data from the default/unscoped cache, and the focused implementation PR is still open rather than merged.

## What This Changes

The issue asks for Usage dashboard header cost/token totals to be consistent with the daily cost chart and activity totals for the selected date range.

## Best Possible Solution

Review and land #70205 or an equivalent focused fix that resolves, caches, and merges `usage.cost` over the same configured-agent scope as `sessions.usage`, then verify any remaining cost-unit concern separately.

## Reproduction Assessment

Yes, by source inspection for the current-main aggregation mismatch: `sessions.usage` scans configured agents, while `usage.cost` still reads the unscoped/default cost cache. I did not live-run the reporter's MiniMax dataset, so the exact displayed dollar magnitude remains a verification detail for the fix PR review.

## Solution Assessment

Yes, the best current direction is the narrow `usage.cost` agent-scope alignment in #70205; changing only frontend formatting would leave the endpoint mismatch intact.
