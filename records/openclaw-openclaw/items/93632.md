## Summary

## What This Changes

The branch changes macOS `VoiceWakeRuntime` so the Voice Wake HUD asynchronously reads gateway `talk.silenceTimeoutMs` for its post-trigger capture silence window and cancels the pending fetch on stop.

## Best Possible Solution

Wire or explicitly narrow the refresh behavior, document the accepted shared-key contract if maintainers approve it, preserve the 2000 ms HUD fallback or floor for upgrades, and attach redacted native macOS HUD proof before merge.

## Reproduction Assessment

## Solution Assessment

No for merge readiness. Moving consumption into `VoiceWakeRuntime` is the right owner boundary, but the branch still needs a real refresh trigger or narrower claim, docs for the expanded setting scope, maintainer acceptance of the shared-key upgrade behavior, and native HUD proof.
