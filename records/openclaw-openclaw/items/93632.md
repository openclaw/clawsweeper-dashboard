## What This Changes

The branch changes `apps/macos/Sources/OpenClaw/VoiceWakeRuntime.swift` so Voice Wake reads gateway `talk.silenceTimeoutMs` in a cancellable background task, updates the mutable HUD silence window on same-config refresh, and cancels pending fetches on stop.

## Best Possible Solution

Wire or explicitly scope real Talk-config refresh behavior, update the Talk and macOS Voice Wake docs for the final shared-key contract, preserve the 2000 ms HUD fallback, and attach redacted native macOS HUD proof before merge.

## Reproduction Assessment

Yes from source inspection, but not from a live macOS run. Current main and `v2026.6.9` hardcode the HUD post-trigger silence window at 2.0 seconds while `talk.silenceTimeoutMs` is parsed only through Talk config paths.

## Solution Assessment

No for merge readiness. Moving consumption into `VoiceWakeRuntime` is a plausible owner-boundary fix, but this branch still needs a real refresh trigger or narrower claim, docs for the expanded config scope, maintainer acceptance of shared-key compatibility, and native HUD proof.
