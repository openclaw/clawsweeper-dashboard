The PR changes macOS `VoiceWakeRuntime` to asynchronously read gateway `talk.silenceTimeoutMs` for the HUD post-trigger capture silence window, refresh it on same-config `refresh(state:)`, and cancel pending fetches on stop.

## Best Possible Solution

A mergeable version should wire real talk-config refresh behavior or narrow the claim, document the accepted shared-key semantics, preserve the 2000 ms HUD fallback, and include redacted native macOS HUD proof.

## Reproduction Assessment

## Solution Assessment

No for merge readiness. Moving consumption into `VoiceWakeRuntime` is the right owner-boundary direction, but this branch still needs a real refresh trigger or narrower claim, docs for the expanded config scope, maintainer acceptance of shared-key compatibility, and native HUD proof.
