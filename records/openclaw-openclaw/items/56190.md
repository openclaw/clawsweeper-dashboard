## What This Changes

Adds `${VAR:-default}` fallback parsing and preservation for config env substitutions, a shared brace scanner, and unit tests for substitution and write-back.

## Best Possible Solution

Land this only after defaulted tokens are handled consistently across substitution, unresolved-ref detection, config.env injection, write-back preservation, public docs, and upgrade compatibility proof.

## Reproduction Assessment

Yes, source-reproducible: the PR head parses `${VAR:-fallback}` as `kind: "substitution"`, `containsEnvVarReference` returns true for that kind, and config-env collection skips those values before substitution can resolve the fallback.

## Solution Assessment

No; parser and write-back support are plausible, but the best solution must also cover unresolved-ref detection, `env.vars` injection, docs, and explicit compatibility semantics.
