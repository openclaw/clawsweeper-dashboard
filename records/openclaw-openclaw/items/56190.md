## Summary

Keep open: current main still only supports plain `${VAR}` substitution, while this PR adds useful fallback syntax but still has a blocking config-env injection gap and lacks contributor real behavior proof.

## What This Changes

The PR adds `${VAR:-default}` fallback parsing, brace-aware scanning, env-reference preservation, and unit tests for config env substitution.

## Best Possible Solution

Land the feature only after fallback-resolvable env refs are handled correctly during config env injection, with chaining regression coverage, public docs, and redacted real config-load proof.

## Reproduction Assessment

Yes, for the PR defect by source inspection: `env.vars.API_KEY = "${UPSTREAM:-demo}"` is still treated as an env reference and skipped before fallback substitution, so a later `${API_KEY}` consumer cannot receive `demo`. I did not run tests because this review is read-only.

## Solution Assessment

No. The parser and write-back direction is reasonable, but the patch needs to distinguish fallback-resolvable refs from unresolved required refs during config env injection and document the new syntax before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88
