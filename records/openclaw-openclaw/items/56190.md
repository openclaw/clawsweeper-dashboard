## Summary

Keep open: this feature is not implemented on current main, but the PR still has a blocking config-env chaining defect and lacks real behavior proof from an OpenClaw config-load run.

## What This Changes

Adds `${VAR:-default}` fallback parsing, env-reference write-back preservation, and unit tests for config environment substitution.

## Best Possible Solution

Land the feature only after fallback-resolvable refs work through config env injection, public docs describe the syntax, and the contributor provides redacted real config-load proof.

## Reproduction Assessment

Yes for the PR defect by source inspection: `env.vars.API_KEY = "${UPSTREAM:-demo}"` is still treated as an unresolved env reference and skipped before later `${API_KEY}` substitution. The requested fallback syntax itself is a new feature, not a current-main bug reproduction.

## Solution Assessment

No. The parser and preservation direction is reasonable, but the patch needs to distinguish fallback-resolvable refs from unresolved required refs during config env injection and document the new public syntax before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Resolve defaulted env refs before injection:** `src/config/env-substitution.ts:82`
  - body: `${VAR:-default}` is still returned as `kind: "substitution"`, so `containsEnvVarReference()` treats it as unresolved.
