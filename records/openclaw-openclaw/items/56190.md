## Summary

Keep open: current main still only supports plain `${VAR}` config substitution, and this PR is the live implementation candidate, but the branch still needs a correctness fix for config-env chaining plus contributor real behavior proof before merge.

## What This Changes

The PR adds `${VAR:-default}` fallback parsing for config env substitution, write-back preservation, a shared brace scanner, and unit coverage.

## Best Possible Solution

Land the feature only after fallback-resolvable refs are distinguished from unresolved required refs in config env injection, with chaining regression coverage, public docs, and redacted real runtime proof.

## Reproduction Assessment

Yes for the PR defect by source inspection: `env.vars.API_KEY = "${UPSTREAM:-demo}"` is still treated as an env reference and skipped before fallback substitution, so a later `${API_KEY}` consumer cannot receive `demo`. I did not run tests because this review is read-only.

## Solution Assessment

No. The parser/write-back direction is reasonable, but the patch needs a small contract change or helper so fallback-resolvable tokens are not treated like unresolved required refs during config env injection.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88
