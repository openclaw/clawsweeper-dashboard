## Summary

Keep open. The feature is not implemented on current main, but this PR is not merge-ready because the PR head still treats fallback placeholders as unresolved refs during config env injection and the external PR lacks required real behavior proof.

## What This Changes

Adds `${VAR:-default}` fallback parsing to config env substitution and write-back preservation, with escape, brace, and round-trip unit coverage.

## Best Possible Solution

Land the feature only after fallback-resolvable tokens are distinguished from unresolved required refs during config env injection, with chaining regression coverage, public docs, and real config-load proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `env.vars.API_KEY = "${UPSTREAM:-demo}"` is classified as containing an env reference and skipped before substitution, so a later `${API_KEY}` consumer cannot see the fallback-resolved value.

## Solution Assessment

No. The parser and write-back direction is reasonable, but the branch needs a small contract change or helper so fallback-resolvable tokens are not treated like unresolved required refs during config env injection.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Resolve defaulted env refs before env injection:** `src/config/env-substitution.ts:82`
