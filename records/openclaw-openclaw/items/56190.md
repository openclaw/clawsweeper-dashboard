## Summary

Keep open: current main still only supports plain `${VAR}` substitution, and this PR is useful, but the current patch leaves fallback-bearing config env values on the unresolved-reference skip path and lacks the required public docs/changelog updates.

## What This Changes

This PR extends config env substitution and write-back preservation to parse `${VAR:-default}` fallbacks with unit coverage.

## Best Possible Solution

Land the feature after fallback-resolvable refs are distinguished from truly unresolved refs in config env injection, with chaining regression coverage plus public docs and an active changelog entry.

## Reproduction Assessment

Yes. From the PR diff and current read path, `env.vars.API_KEY = "${UPSTREAM:-demo}"` is classified as an env reference, skipped by `applyConfigEnvVars`, and therefore not available to a later `${API_KEY}` consumer even though the fallback is resolvable.

## Solution Assessment

No. The parser approach is reasonable, but the PR should resolve fallback-bearing config env values before the unresolved-reference skip, or expose a helper that distinguishes required refs from fallback-resolvable refs, then document the public syntax.
