## Summary

Keep this PR open, but it is not ready to merge: the proposed public config fallback feature still has a concrete fallback-chaining defect and lacks contributor-supplied real behavior proof.

## What This Changes

This PR adds `${VAR:-default}` config env fallback parsing, write-back preservation, a shared brace scanner, and focused unit tests under `src/config`.

## Best Possible Solution

Land a revised public config-contract change that handles fallback templates consistently across substitution, `env.vars` injection, write-back preservation, docs, and focused tests.

## Reproduction Assessment

Yes for the PR-introduced defect by source inspection: defaulted tokens are still ordinary `substitution` tokens, while current config read applies `env.vars` before substitution and skips values flagged by `containsEnvVarReference()`. A current-main bug repro is not applicable because fallback syntax is not on main.

## Solution Assessment

No. The parser and preservation direction is plausible, but the PR should distinguish fallback-resolvable refs during unresolved-ref checks, update public docs, and include redacted real config-load proof before merge.
