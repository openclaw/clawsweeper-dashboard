## Summary

## What This Changes

The branch adds `${VAR:-default}` parsing to config env substitution and write-back preservation with a shared brace scanner and focused unit tests.

## Best Possible Solution

Land a revised config-contract change that distinguishes required env refs from fallback-resolvable refs, covers `env.vars` chaining and write-back preservation, documents semantics and escaping, and includes redacted real config-load proof.

## Reproduction Assessment

Yes for the PR-introduced defect by source inspection: fallback tokens still return ordinary `substitution` tokens, while `applyConfigEnvVars()` skips any value reported by `containsEnvVarReference()` before substitution runs. A current-main bug repro is not applicable because fallback syntax is not implemented on main.

## Solution Assessment

No. The parser and preservation direction is plausible, but the branch should resolve fallback templates consistently across config fields, `env.vars` injection, write-back, docs, and real runtime proof before merge.
