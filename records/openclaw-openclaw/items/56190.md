## Summary

Keep this PR open, but not merge-ready: the feature is useful, current main does not implement it, and the branch still has a source-level `env.vars` chaining defect plus missing real behavior proof for a real config load.

## What This Changes

The PR adds `${VAR:-default}` config env fallback parsing and preservation, a shared brace scanner, and focused unit coverage in config env substitution and preserve tests.

## Best Possible Solution

Land a revised config-contract change that distinguishes required env refs from default-resolvable refs across substitution, `env.vars` injection, and write-back, with public docs and real config-load proof.

## Reproduction Assessment

Yes for the PR-introduced defect by source inspection: defaulted refs remain ordinary `substitution` tokens, and current `applyConfigEnvVars()` skips substitution-looking `env.vars` values before placeholder resolution runs. This is not a current-main bug because the fallback syntax is not implemented on `main`.

## Solution Assessment

No as-is. The parser and preservation approach are plausible, but the runtime injection path needs fallback-aware handling and the new public config syntax needs maintainer acceptance, docs, and real behavior proof.
