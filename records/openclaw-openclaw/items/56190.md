## Summary

Keep open: this is a focused feature PR, but the current branch still mishandles fallback templates in `env.vars`, lacks public docs for the new config syntax, and is missing contributor-supplied real behavior proof.

## What This Changes

The PR adds `${VAR:-default}` config env substitution, write-back preservation support, a shared brace scanner, and focused unit tests under `src/config`.

## Best Possible Solution

Land a revised config-contract change that resolves fallback templates consistently across substitution, `env.vars` injection, write-back preservation, docs, and focused tests.

## Reproduction Assessment

Yes for the PR-introduced defect by source inspection: defaulted tokens are still ordinary `substitution` tokens, while config read applies `env.vars` before substitution and skips values reported by `containsEnvVarReference()`. A current-main bug repro is not applicable because fallback syntax is not on main.

## Solution Assessment

No. The parser and preservation direction is plausible, but the branch should handle fallback-capable `env.vars` values, update public docs, and include redacted real config-load proof before merge.

- **[P2] Resolve defaulted env refs before injecting env.vars:** `src/config/env-substitution.ts:82`
