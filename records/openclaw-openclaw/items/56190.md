## Best Possible Solution

Yes for the PR-introduced defect by source inspection: defaulted refs remain ordinary `substitution` tokens, and current `applyConfigEnvVars()` skips substitution-looking `env.vars` values before placeholder resolution runs. ## Solution Assessment

No as-is. The parser and preservation approach are plausible, but the runtime injection path needs fallback-aware handling and the new public config syntax needs maintainer acceptance, docs, and real behavior proof.
