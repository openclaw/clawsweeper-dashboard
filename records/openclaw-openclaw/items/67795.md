## Summary

Keep open: current main still rejects a documented Linux install path when Bash reports `OSTYPE=linux`, and the one-line predicate change is a narrow fix, but the external PR still needs after-fix real behavior proof or maintainer proof handling before merge.

## What This Changes

This PR changes `scripts/install.sh` to classify any Bash `$OSTYPE` beginning with `linux` as Linux instead of only `linux-gnu` or WSL.

## Best Possible Solution

Merge the predicate fix after the proof gate is satisfied or explicitly waived, ideally with a small `install.sh` regression test for `OSTYPE=linux`; keep automatic zypper dependency installation as separate follow-up work.

## Reproduction Assessment

Yes. A guarded current-main source probe with `OSTYPE=linux` exits through the unsupported-OS path, while `OSTYPE=linux-gnu` and the PR head script accept Linux.

## Solution Assessment

Yes. Broadening the existing Bash prefix match from `linux-gnu*` to `linux*` is the narrow maintainable fix for the documented Linux detection gap; package-manager support for zypper is separate.
