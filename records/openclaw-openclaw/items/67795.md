## Summary

Keep open. This is a valid narrow installer compatibility fix against current main, with no code or security findings, but it still needs after-fix real behavior proof from the patched installer and normal handling of the failed Install Smoke check before merge.

## What This Changes

The PR changes `scripts/install.sh` so `detect_os_or_die` treats any Bash `OSTYPE` beginning with `linux` as Linux instead of only `linux-gnu`.

## Best Possible Solution

Merge a narrow Linux detection fix, ideally with a small regression test for bare `OSTYPE=linux`, after contributor-provided after-fix proof and green relevant checks.

## Reproduction Assessment

Yes. Current main can be reproduced by sourcing `scripts/install.sh` with no-run mode, setting `OSTYPE=linux`, and calling `detect_os_or_die`, which exits through the unsupported operating system path despite the documented Linux installer flow.

## Solution Assessment

Yes. Matching `linux*` is the narrowest maintainable fix for Linux-family Bash `OSTYPE` variants while preserving the existing `linux-gnu*` and WSL behavior; zypper auto-install support would be a separate feature, not required for this bug.
