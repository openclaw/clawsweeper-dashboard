## Summary

Keep open: the one-line OS detection change addresses the first openSUSE symptom, but it would route openSUSE/SLES into Linux installer paths that still lack zypper coverage for build tools, Node.js, and Git.

## What This Changes

This PR changes `scripts/install.sh` to treat any Bash `OSTYPE` beginning with `linux` as Linux instead of only `linux-gnu` or WSL.

## Best Possible Solution

Complete openSUSE/SLES support by pairing `OSTYPE=linux` detection with zypper coverage for build tools, Node.js, and Git plus focused tests/docs, or keep detection scoped to package managers the installer can bootstrap.

## Reproduction Assessment

Yes. Source inspection on current main shows `OSTYPE=linux` falls through OS detection, and the PR would then enter Linux installer branches that still do not handle zypper for build tools, Node.js, or Git.

## Solution Assessment

No. The pattern change is useful but incomplete; the maintainable fix is to add zypper support through the affected installer paths or avoid accepting Linux variants the installer cannot bootstrap.
