## Summary

Keep open: this PR addresses a plausible openSUSE compatibility gap, but it is incomplete for the installer contract because accepted zypper-only systems still cannot bootstrap build tools, Node.js, or Git when those dependencies are missing, and the latest proof link shows the workaround rather than an after-fix run from the branch.

## What This Changes

The PR changes `scripts/install.sh` so any Bash `OSTYPE` beginning with `linux` is treated as Linux during installer OS detection.

## Best Possible Solution

Pair broader Linux detection with explicit zypper/openSUSE support or a documented manual-dependency path so every accepted Linux variant either bootstraps the installer contract or fails with clear supported-package-manager guidance.

## Reproduction Assessment

Yes. A focused current-main shell call reproduces `OSTYPE=linux` being rejected, and source/stub checks show the PR would then hit zypper gaps when Node, Git, or build tools are missing.

## Solution Assessment

No. Broadening the pattern is useful for users who preinstall dependencies, but the maintainable fix is to either add tested zypper coverage or intentionally document a manual-dependency path instead of silently expanding supported Linux variants.
