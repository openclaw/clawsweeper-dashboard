## Summary

Keep open. Current main still routes WebChat and Gateway-backed TUI through `chat.send`, where final delivery and Gateway-owned transcript appends bypass the existing message lifecycle hook seams.

## What This Changes

The issue requests that outbound message lifecycle hooks fire for WebChat and openclaw-tui assistant replies, including `message_sent` and `before_message_write`.

## Best Possible Solution

Add a narrow Gateway chat lifecycle hook seam for `chat.send` final delivery and Gateway-owned transcript appends, while leaving the broader cross-channel observer design to the open umbrella issue.

## Reproduction Assessment

Yes, as a source-level reproduction. WebChat docs and the TUI client both enter `chat.send`, and current main still shows that path bypassing hook-aware dispatch, sent-hook emission, and `before_message_write` for Gateway assistant transcript appends; no live run was performed in this read-only review.

## Solution Assessment

Yes. A focused Gateway chat fix is the narrowest maintainable solution for this issue, reusing existing hook runner and reply-dispatch seams without taking on the full outbound observer architecture.
