## Summary

Keep open. Current main still routes WebChat and Gateway-backed TUI through `chat.send`, where final UI delivery and Gateway-owned assistant transcript appends bypass the existing outbound/write hook seams.

## What This Changes

The issue requests that WebChat and openclaw-tui outbound assistant replies fire message lifecycle hooks such as `message_sent`, `message_sending`, `agent_end`, and `before_message_write`.

## Best Possible Solution

Add a focused Gateway chat lifecycle seam so WebChat and Gateway-backed TUI final delivery and Gateway-owned assistant transcript appends invoke the existing hooks exactly once with session/run context, while leaving the broader cross-channel observer design to the umbrella issue.

## Reproduction Assessment

Yes, at source level. WebChat docs and the TUI client both enter `chat.send`, and current main still shows that path bypassing hook-aware dispatch, sent-hook emission, and `before_message_write` for Gateway-owned assistant transcript appends; no live run was performed in this read-only review.

## Solution Assessment

Yes. A focused Gateway chat fix is the narrowest maintainable path for this issue; a larger exactly-once outbound observer belongs to https://github.com/openclaw/openclaw/issues/50126 if maintainers choose that architecture.
