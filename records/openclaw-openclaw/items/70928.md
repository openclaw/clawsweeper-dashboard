## Summary

Keep this open. Current main still leaves the WebChat and Gateway-backed TUI `chat.send` outbound path outside the existing hook-aware dispatch and transcript-write seams, so the reported hook coverage gap remains valid and actionable.

## What This Changes

The issue requests that outbound lifecycle hooks such as `message_sending`, `message_sent`, `agent_end`, and `before_message_write` fire for WebChat and openclaw-tui assistant replies.

## Best Possible Solution

Add focused Gateway chat hook coverage: apply the existing `message_sending` before-delivery seam to `chat.send` replies, emit plugin `message_sent` plus internal `message:sent` after final WebChat/TUI delivery outcomes, run `before_message_write` before Gateway-injected assistant transcript appends, and cover WebChat plus Gateway TUI with regression tests while leaving the broader outbound observer architecture to #50126.

Yes. ## Solution Assessment

Yes. A narrow Gateway chat fix is the best next step; the broader outbound observer design can remain tracked by #50126, but this default-surface bug can be repaired with existing hook seams and focused tests.
