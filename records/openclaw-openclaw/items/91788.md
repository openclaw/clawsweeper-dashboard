## Summary

Keep this open: current main and the latest release still disable the message tool for native spawned subagents, and no canonical merged or open item fully owns the requested opt-in. The report is source-reproducible, but the fix requires maintainer product/security direction because it would intentionally restore direct channel-message capability to child sessions.

## What This Changes

The issue requests an explicit config or allowlist opt-in that lets spawned native subagent sessions construct and use the `message` tool again while keeping the default disabled.

## Best Possible Solution

Keep the default parent-mediated handoff, then add a maintainer-approved explicit opt-in with schema/docs/tests that prove default-off behavior and operator-requested message-tool availability for spawned native subagents.

## Reproduction Assessment

Yes. Source inspection shows the child gateway call passes `disableMessageTool: true`, and `createOpenClawTools` skips constructing `message` whenever that flag is set.

## Solution Assessment

Unclear. An explicit opt-in is plausible and aligned with operator-controlled capability, but maintainers need to choose the permanent config/API shape before implementation.
