## Summary

Keep open: current main and the latest stable release still omit passive context usage from the system-prompt Runtime line, while the two related closed issues were closed as duplicates in favor of this issue and the prior exact implementation PR is closed unmerged.

## What This Changes

The issue requests adding per-turn context-window usage to the system prompt Runtime metadata so agents do not need to call `session_status` every turn.

## Best Possible Solution

Keep this as the canonical tracker until maintainers either approve a bounded Runtime metadata field from a fresh session usage snapshot with focused tests/docs, or decide that `/status`, `/context`, and `session_status` remain the intended product boundary.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives a high-confidence reproduction of the missing capability because current main has no context usage carrier in Runtime prompt params, prompt rendering, or embedded-runner prompt assembly.

## Solution Assessment

Unclear as an implementation decision. Adding a bounded Runtime field is plausible and technically small, but maintainers should first decide the freshness/format/cache-stability contract versus keeping context usage on explicit status surfaces.
