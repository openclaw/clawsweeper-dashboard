## Repair Work Prompt

Implement passive context-window usage in the system-prompt Runtime line for embedded runs. Compute a bounded `context=NN%` value only from fresh session usage and a resolved positive context window, thread it through `RunEmbeddedPiAgentParams`, `EmbeddedRunAttemptParams`, `RuntimeInfoInput`, and `buildRuntimeLine`, and omit the field when usage/window data is missing, invalid, or explicitly stale. Keep the dynamic field in the volatile Runtime section below `SYSTEM_PROMPT_CACHE_BOUNDARY`; do not change compaction thresholds, model routing, `/status` behavior, provider APIs, plugin APIs, or CLI-provider behavior beyond omission when unavailable. Add focused tests for runtime-line rendering, stale/missing/invalid omission, reply-runner/embedded propagation, cache-boundary placement, and update the system-prompt docs if they enumerate Runtime fields.
