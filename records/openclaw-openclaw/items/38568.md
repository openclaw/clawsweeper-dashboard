## Summary

Keep open: current main still does not inject passive context-window usage into the system-prompt Runtime line, the duplicate reports are consolidated here, and the prior exact implementation PR was closed unmerged.

## What This Changes

The issue requests passive context-window usage metadata in the system prompt Runtime section so agents do not need to call `session_status` each turn.

## Best Possible Solution

Add bounded Runtime-line context usage plumbing from fresh session usage into embedded-runner prompt params, omit unknown or stale values, and cover prompt formatting, cache-boundary behavior, runner propagation, and docs.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection on current `main` verifies the passive Runtime-line context usage field is absent while `/status` and `/context` remain explicit inspection surfaces.

## Solution Assessment

Yes; focused Runtime-line plumbing from fresh session usage metadata is the narrowest maintainable path. Response-footer/template work and workspace-sync plugins are useful adjacent features, but they do not give agents passive per-turn system-prompt context pressure.
