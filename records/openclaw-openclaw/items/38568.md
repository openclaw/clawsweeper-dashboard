## Summary

Keep open: current main and the latest release still omit passive context-window usage from the system-prompt Runtime line, while duplicate reports point here as the canonical tracker. Existing `/status`, `/usage`, `/context`, `session_status`, reply usage footer, and diagnostics surfaces expose context data on demand or after the run, but they do not provide the per-turn prompt metadata requested here.

## What This Changes

Request to add current context-window usage, such as `context=49%` or token/window counts, to the agent system-prompt Runtime section.

## Best Possible Solution

Have maintainers define the freshness, omission, formatting, and prompt-cache contract; if accepted, implement a bounded Runtime context field from existing usage snapshots with focused prompt, runner, model-switch/compaction, and docs coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that passive Runtime-line context usage is still absent while explicit status/context surfaces exist.

## Solution Assessment

Yes for keeping it open: the requested core prompt path is plausible, but a fix should wait for maintainer approval of freshness and prompt-cache semantics. Existing explicit usage surfaces are useful workarounds but do not solve passive per-turn Runtime prompt metadata.
