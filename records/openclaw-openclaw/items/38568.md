## Summary

Keep this issue open. Current main still exposes context usage through status/session_status-style surfaces, but the Runtime prompt metadata has no context usage carrier or rendered context percentage, and the closed related #76683 appears to be the same request rather than a canonical replacement.

## What This Changes

Request to inject current context-window usage percentage or token count into the system prompt Runtime metadata so agents can read it without calling session_status every turn.

## Best Possible Solution

Add a typed optional context-usage value to core prompt assembly, render a compact Runtime field only when the value is reliable, and keep existing status/session_status behavior unchanged.
