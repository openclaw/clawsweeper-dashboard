## Repair Work Prompt

Implement the Runtime context-usage feature for embedded agent runs. Compute an optional context percentage from the active session's fresh `totalTokens` and positive `contextTokens`; omit the field when usage is missing, stale, non-finite, or invalid. Thread the optional value through reply-runner params, embedded runner params/attempt params, `RuntimeInfoInput`, and `buildRuntimeLine()` so the Runtime line can render `context=X%` below the prompt cache boundary. Keep the change additive, do not add a new CLI/status surface or rewrite token accounting, and add focused coverage for rendering, stale/missing usage omission, and propagation through the runner path. Add a changelog entry only if maintainer landing policy requires it for this user-visible feature.
