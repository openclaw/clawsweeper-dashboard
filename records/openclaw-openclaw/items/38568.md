## Summary

Keep this issue open. Current main still has no context-usage field in the system prompt Runtime path, while the later related #76683 appears to have been closed as the duplicate rather than implementing the feature here.

## What This Changes

The issue requests adding current context-window usage to the system prompt Runtime metadata so agents can read it without calling `session_status` every turn.

## Best Possible Solution

Add a typed optional context-usage snapshot to prompt assembly, render a compact Runtime field below the cache boundary only when the value is reliable, and leave existing status/session_status behavior unchanged.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection on current main gives high confidence that Runtime prompt metadata still lacks context usage while status surfaces expose it separately.

## Solution Assessment

Yes. A narrow core prompt-runtime addition is the best fit because the system prompt Runtime line is assembled in core and the request is specifically for zero-tool-call prompt metadata, not a plugin or status-only surface.
