## Summary

Current main still exposes context usage through `session_status` but does not pass it into system-prompt runtime metadata; the request remains the canonical open product decision because the related issues were closed in favor of it, and a volatile per-turn prompt field needs an explicit contract for snapshot timing and cache behavior.

## What This Changes

Expose current session context-window usage in the system-prompt Runtime metadata so agents can observe pressure without calling `session_status`.

## Best Possible Solution

Approve a canonical previous-turn context-usage snapshot, then add it to the existing dynamic Runtime suffix with omission when unavailable and focused coverage for compaction, model-window changes, and cache-boundary stability.

## Reproduction Assessment

Yes at the source level: current main clearly omits context usage from Runtime metadata while exposing accounting through `session_status`; this is a missing capability rather than a failing existing contract.

## Solution Assessment

Unclear until maintainers approve the metadata contract; the Runtime suffix is the narrowest likely surface, provided it reuses canonical accounting and explicitly represents a possibly one-turn-stale snapshot.
