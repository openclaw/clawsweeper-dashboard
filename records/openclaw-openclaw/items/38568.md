## Repair Work Prompt

Implement https://github.com/openclaw/openclaw/issues/38568 as a focused OpenClaw core prompt-runtime change. Add passive context usage to the system prompt Runtime line for embedded agent runs by threading a typed optional context usage value through the reply runner / embedded runner params, `buildSystemPromptParams`, `RuntimeInfoInput`, and `buildRuntimeLine`. Derive the value from reliable prompt-side context snapshots such as `promptTokens` or `lastCallUsage` via the existing context-token helpers plus `contextTokensUsed`; do not use cumulative provider `usage.total` when it can include output or multi-call totals. Omit `context=` when the value is unavailable, non-finite, stale, or lacks a positive context window. Keep the field in the volatile Runtime section below the prompt cache boundary, do not add a config option or migration, and leave `/status`, `/context`, and `session_status` behavior unchanged. Add focused tests for Runtime rendering, param propagation, stale/missing guards, and cache-boundary behavior; update `docs/concepts/system-prompt.md`, `docs/reference/token-use.md`, and `CHANGELOG.md` if the user-visible prompt behavior changes.
