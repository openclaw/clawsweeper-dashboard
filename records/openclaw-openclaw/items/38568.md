Implement passive context-window usage in the system-prompt Runtime line for embedded agent runs. Derive a simple `context=NN%` value only from fresh session token/window data; respect `totalTokensFresh === false`, omit unavailable or stale values, clamp to a sane range such as 0-999, and avoid raw token counts unless maintainers ask for them. Thread the value through the existing run path into `RunEmbeddedPiAgentParams`, `EmbeddedRunAttemptParams`, `buildSystemPromptParams`, `RuntimeInfoInput`, and `buildRuntimeLine()`. Keep the dynamic field below the prompt cache boundary, do not add a new config option or plugin API, do not change `/status` semantics, and add focused renderer/propagation/cache-boundary tests. If this is promoted as a maintainer repair PR, include a concise `CHANGELOG.md` Changes entry.
