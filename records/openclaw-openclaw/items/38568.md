## Repair Work Prompt

Implement the Runtime context-usage feature tracked by https://github.com/openclaw/openclaw/issues/38568. Add an optional fresh context usage percentage to the system prompt Runtime line, preferably `context=N%`, by deriving it from the active session's fresh total token snapshot and effective context token window. Omit the field when token usage or context window data is missing, invalid, non-finite, non-positive, or stale (`totalTokensFresh === false`). Keep the dynamic value in the existing volatile Runtime section below the prompt cache boundary. Likely implementation touches RuntimeInfoInput, buildRuntimeLine(), embedded runner params/attempt plumbing, and the reply runner path that has access to the active session entry. Preserve existing /status and /context behavior, do not add a config option, and do not change compaction policy. Add focused regression tests for formatting, stale/unknown omission, cache-boundary placement, and runner propagation. Add or update user-facing docs and a changelog entry if the maintainer lane expects one for the user-visible prompt feature.
