## Repair Work Prompt

Implement https://github.com/openclaw/openclaw/issues/38568. Add passive context-window pressure to the embedded-agent system prompt Runtime line as `context=NN%` only when session usage is fresh: derive it from the latest fresh session total tokens and the effective context window, round/clamp consistently with status formatting, and omit it when total/window data is unavailable or `totalTokensFresh === false`. Thread a small optional field through `agent-runner-execution` into `RunEmbeddedPiAgentParams`, `EmbeddedRunAttemptParams`, `buildSystemPromptParams`, `RuntimeInfoInput`, and `buildRuntimeLine`. Keep the value below the prompt cache boundary; do not add a config option, change `/status` or `/context` output, add raw token counts unless they use the same freshness guard, or broaden CLI-only behavior without a clear data source. Add focused tests for rendering, stale/unknown omission, and embedded-runner propagation; update `docs/concepts/system-prompt.md` and add a concise `CHANGELOG.md` entry if the behavior is user-visible.
