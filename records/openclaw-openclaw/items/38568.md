## Repair Work Prompt

Implement context-window usage in the system prompt Runtime line for embedded agent runs. Compute an optional percentage from fresh active session `totalTokens` and `contextTokens` snapshots; do not emit when values are unavailable, invalid, or `totalTokensFresh === false`. Thread the value through the appropriate reply/embedded-runner params into `buildSystemPromptParams`, `RuntimeInfoInput`, and `buildRuntimeLine`; keep the dynamic output in the volatile Runtime section below the cache boundary and do not change `/status`, `/context`, `/usage`, or token-accounting semantics. Prefer concise `context=49%` formatting unless the implementation already has a clean count/window formatter. Add focused tests for formatting, stale/invalid omission, embedded-runner propagation, and cache-boundary stability; update the system-prompt docs and add a user-facing changelog entry if maintainers land it as a feature.
