## Summary

Keep open: current main and the latest release still omit passive context-window usage from the system-prompt Runtime line, and the remaining question is a maintainer product decision about adding volatile usage metadata to every prompt turn.

## What This Changes

The issue requests adding context window usage, such as `context=49%`, to the system prompt Runtime metadata so agents do not need to call `session_status` every turn.

## Best Possible Solution

Define the snapshot and omission contract, then add a bounded `context=...` Runtime segment using existing session usage accounting with focused prompt, runner, compaction/model-change, and docs coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main lacks the requested Runtime-line context usage field while explicit `/context`, `session_status`, and response usage surfaces exist.

## Solution Assessment
