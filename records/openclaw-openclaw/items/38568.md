## Summary

Keep open: current main and the latest shipped release still lack a passive Runtime `context=` field, while related duplicates point to this as the canonical tracker and the prior exact implementation PR was closed unmerged.

## What This Changes

Add current context-window usage percentage to the system prompt Runtime section so agents can see context pressure without calling `session_status`.

## Best Possible Solution

Add a typed optional context-usage snapshot to prompt assembly and render a compact Runtime `context=...` field only when the value is reliable, with focused tests, docs, and a changelog entry.

## Solution Assessment

Yes. A narrow core prompt-runtime addition is the best fit because the requested zero-tool-call signal belongs in OpenClaw-owned system prompt assembly, not in a plugin or status-only surface.
