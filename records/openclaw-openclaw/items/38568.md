## Summary

Current `main` still does not expose context-window utilization in the system prompt, and https://github.com/openclaw/openclaw/issues/76683 was closed in favor of this issue as the canonical tracker. The feature fits the context-management roadmap, but implementation should wait for a maintainer decision on snapshot timing and runtime parity because embedded OpenClaw and native Codex own token state at different lifecycle boundaries.

## What This Changes

Add current context-window usage to the agent system prompt runtime metadata so agents can observe context pressure without calling `session_status` every turn.

## Best Possible Solution

Define one canonical, explicitly labeled previous-turn or preflight context snapshot, carry it with the resolved context budget through the runtime plan, and render it below the prompt cache boundary only for runtimes where the measurement is trustworthy.

## Reproduction Assessment

Yes for the feature gap: current `main` has no context-utilization field in the prompt runtime contract or renderer. This confirms absence of the requested capability, not a failure of an existing documented contract.

## Solution Assessment

No, not exactly as proposed. The Runtime line is a reasonable presentation surface, but the value must come from canonical runner or harness accounting with defined staleness and cache behavior rather than an independent prompt-renderer calculation.
