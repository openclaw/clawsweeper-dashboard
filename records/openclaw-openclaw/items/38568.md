## Summary

Keep open: current main still lacks passive context-window usage in the system-prompt Runtime line, and the related closed reports now point to this issue as the canonical tracker rather than a shipped fix.

## What This Changes

The issue requests adding current context-window usage to the system-prompt Runtime metadata so agents do not need to call `session_status` each turn.

## Best Possible Solution

Add a bounded optional context-usage field to the Runtime prompt path using only fresh session token snapshots, omit it when unknown or stale, and cover formatting plus runner propagation with focused tests.

Not applicable as a bug reproduction; this is a feature request. Source inspection on origin/main shows the requested passive Runtime context field is absent while explicit `/status` and `/context` surfaces exist.

## Repair Work Prompt

Implement https://github.com/openclaw/openclaw/issues/38568 as a focused prompt/runtime feature. Add an optional context usage percentage to the system-prompt Runtime line for embedded runs, using existing fresh session token metadata only: compute from `SessionEntry.totalTokens` via `resolveFreshSessionTotalTokens()` and a positive finite `contextTokens`, round to a small integer percentage, clamp display to a sane range such as 0-999, and omit the field when data is missing, invalid, or stale. Thread the optional value through `src/auto-reply/reply/agent-runner-execution.ts`, `src/agents/pi-embedded-runner/run/params.ts`, `src/agents/pi-embedded-runner/run.ts`, `src/agents/pi-embedded-runner/run/types.ts`, `src/agents/pi-embedded-runner/run/attempt.ts`, `src/agents/system-prompt-params.ts`, and `src/agents/system-prompt.ts`. Keep this below the existing prompt cache boundary in the Runtime section, do not add a config option, do not estimate from stale transcript fallback, and do not add raw token counts unless the implementation can do so without extra product surface. Add or update focused tests for Runtime-line formatting, stale/invalid omission, and reply-runner propagation; add a single user-facing changelog entry if repo policy requires it for this prompt-visible feature.
