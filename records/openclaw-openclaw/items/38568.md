## Summary

Keep open: current main still lacks a context-usage carrier or rendered `context=` segment in the system-prompt Runtime path, while the existing status surfaces expose usage separately and related reports were closed in favor of this tracker.

## What This Changes

Request to expose current context-window usage in the system prompt Runtime metadata so agents do not need to call `session_status` every turn.

## Best Possible Solution

Add a typed optional prompt/context snapshot to prompt assembly and render a compact Runtime `context=...` field only when the usage value is reliable, with focused tests and docs while leaving `/status` and `session_status` unchanged.

Add bounded optional context usage plumbing from the latest reliable session token snapshot into embedded-runner prompt params, `RuntimeInfoInput`, and `buildRuntimeLine()`, rendering a compact `context=N%` field only when total/context token data is finite, fresh, and positive.
