## Summary

Keep open: current main still lacks a context-usage carrier or rendered Runtime `context=` segment, the latest release lacks it too, related reports were closed in favor of this tracker, and the prior exact implementation PR was closed unmerged.

## What This Changes

Add current context-window usage to the system prompt Runtime metadata so agents can see it without calling `session_status`.

## Best Possible Solution

Add a typed optional context-usage snapshot to prompt assembly and render a compact Runtime `context=...` field only when the value is reliable, with focused tests, docs, and a changelog entry.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection on current main shows the Runtime prompt type, renderer, and embedded prompt path still lack context usage while status surfaces expose it separately.

## Solution Assessment

Yes. A narrow core prompt-runtime addition is the best fit because the requested zero-tool-call signal belongs in OpenClaw-owned system prompt assembly, not in a plugin or status-only surface.
