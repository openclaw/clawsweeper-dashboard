## Repair Work Prompt

Implement the canonical context-usage Runtime-line feature for https://github.com/openclaw/openclaw/issues/38568. Add a bounded `context=X%` field to the system prompt Runtime line for embedded agent runs when a fresh session token snapshot and effective context window are available; omit the field when usage is unavailable, invalid, or stale (`totalTokensFresh === false`). Keep the value in the volatile Runtime section below the prompt cache boundary, do not add a config knob, and do not change existing `/status`, `/context`, `/usage`, or response-footer behavior except as needed to reuse formatting helpers safely. Likely path: compute/resolve the percentage from session `totalTokens` plus effective `contextTokens`, thread it through `RunEmbeddedPiAgentParams`/attempt params into `RuntimeInfoInput`, render it in `buildRuntimeLine()`, and add focused tests for formatting, stale/invalid omission, runner propagation, and cache-boundary placement. Add a changelog entry if the repair PR lands as a user-facing feature. Update public docs only if the Runtime section description changes. Do not revive unrelated response footer/template or workspace-sync work.
