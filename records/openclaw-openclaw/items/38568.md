## Repair Work Prompt

Implement https://github.com/openclaw/openclaw/issues/38568 as a narrow prompt/runtime feature. Add a passive context-usage field to the system-prompt Runtime line for embedded agent runs when a fresh session token snapshot is available; omit it when total tokens are unavailable, stale, non-finite, or the context window is unavailable. Prefer a compact percentage such as `context=49%`, derived from fresh total tokens and a positive context-token window, with sane integer formatting/clamping. Thread the value through the reply runner and embedded runner params into `RuntimeInfoInput` and `buildRuntimeLine()` without changing token accounting, `/status`, `/context`, provider selection, or adding any config flag. Keep the dynamic field in the volatile Runtime section below the prompt cache boundary, update focused tests for formatting and propagation, and add/update docs for the Runtime field. Add a single changelog entry if landing policy treats this as user-visible.
