## Summary

Keep open: current main still does not inject context-window usage into the system-prompt Runtime line, and related closed items point to this issue as the canonical tracker rather than proving it is implemented. Existing `/status`, `/context`, `/usage`, and `session_status` surfaces expose context data on demand, but they do not provide the passive per-turn prompt metadata requested here.

## What This Changes

The issue requests adding current context-window usage, such as `context=49%`, to the injected system-prompt Runtime metadata.

## Best Possible Solution

Have maintainers define the freshness, omission, formatting, and prompt-cache contract; if accepted, implement a bounded Runtime context field from existing session usage snapshots with focused prompt, runner, compaction/model-change, and docs coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Current-main source inspection gives high confidence that passive Runtime-line context usage is still absent while explicit status/context surfaces exist.

## Solution Assessment

Yes for keeping it open: the requested core prompt path is plausible, but a fix should wait for maintainer approval of freshness and prompt-cache semantics. Existing `/status`, `/context`, `/usage`, and `session_status` are useful workarounds but do not solve passive per-turn Runtime prompt metadata.
