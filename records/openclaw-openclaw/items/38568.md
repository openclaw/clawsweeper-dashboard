## Repair Work Prompt

Implement the feature tracked by https://github.com/openclaw/openclaw/issues/38568. Add a typed optional fresh context-usage snapshot/percentage to the embedded prompt runtime path, thread it through `RunEmbeddedPiAgentParams` / `EmbeddedRunAttemptParams` / `RuntimeInfoInput`, and render a compact Runtime `context=...` field only when the value is reliable and finite. Prefer the latest fresh prompt/context snapshot over cumulative provider totals; omit the field when usage is unavailable or stale. Keep the dynamic field in the volatile Runtime section below the prompt cache boundary. Do not add a config option, change `/status` or `session_status` semantics, or implement threshold hooks. Add focused regression coverage for runtime-line rendering and reply-runner/embedded-runner propagation, update system-prompt/token-use docs if behavior text changes, and add a user-facing changelog entry.
