Add bounded Runtime-line context usage plumbing from fresh session token/window snapshots into embedded prompt params, omit unavailable or stale values, and cover formatting, propagation, cache-boundary, and docs behavior.

## Repair Work Prompt

Implement the feature requested in https://github.com/openclaw/openclaw/issues/38568: expose passive context-window usage in the system prompt Runtime line for embedded runs, preferably as `context=N%`, without adding a config option. Use only fresh session usage data: derive the percentage from a finite nonnegative fresh `totalTokens` snapshot and a positive finite `contextTokens` window, omit the field when usage/window data is unavailable or stale (`totalTokensFresh === false`), and keep the dynamic value in the existing volatile Runtime section below the prompt cache boundary. Do not broaden this into a new settings surface, a status command redesign, or channel-specific behavior. Add/update user-facing docs if the Runtime fields documented in `docs/concepts/system-prompt.md` change, and include a maintainer-owned changelog entry if the PR is prepared for landing.
