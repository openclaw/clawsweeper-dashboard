## Summary

Keep open: current main and the latest release still omit passive context usage from the system-prompt Runtime line. This issue is now the canonical tracker for that prompt-runtime feature, with older duplicate reports closed into it and a prior exact implementation PR closed unmerged.

## What This Changes

Request to inject current context-window usage into the system-prompt Runtime metadata so agents can see it without calling `session_status`.

## Best Possible Solution

Define the snapshot/freshness contract, then add a bounded optional `context=...` Runtime segment below the cache boundary using existing session usage accounting with focused prompt, runner, compaction, and model-change coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main lacks the requested Runtime-line context usage field while `/context` and `session_status` remain explicit inspection paths.

## Solution Assessment

Yes for keeping it open: the narrow maintainable path is a core prompt-runtime change after maintainers confirm the usage snapshot semantics. Status commands, response footers, and runtime self-context proposals are adjacent but do not replace this passive per-turn Runtime signal.
