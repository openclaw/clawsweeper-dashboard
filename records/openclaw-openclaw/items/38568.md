Implement https://github.com/openclaw/openclaw/issues/38568 as a focused prompt-runtime feature. Add typed optional context-usage plumbing from the latest reliable session usage snapshot into embedded-runner params, `RuntimeInfoInput`, and `buildRuntimeLine()`, then render a compact Runtime field such as `context=49%` only when usage and window values are finite and trustworthy. Omit the field when unknown or stale, clamp/round defensively, keep the dynamic field below the prompt cache boundary, and do not add a config option or change `/status` / `session_status` semantics.
