## Summary

Keep this PR open for a focused repair: the current head still permits an in-place package update after local override inspection fails, contradicting the preservation guarantee despite otherwise strong proof and validation.

## What This Changes

The PR preserves local packaged `dist` overrides during self-update, adds content-hash inventory verification, an opt-in replay flag, recovery and conflict reporting, and related release-package checks.

## Best Possible Solution

Abort before invoking the package manager whenever an installed content inventory exists but cannot be safely inspected, while retaining the no-inventory compatibility window only for legitimately pre-cutover released packages.

## Reproduction Assessment

Yes, from source and the PR's own regression test: a malformed content inventory yields `unsupported`, proceeds with the in-place update, and replaces the installed file. I did not execute the test because this review was required to keep the checkout read-only.

## Solution Assessment

No. The overall preservation architecture is plausible, but warning-only continuation after an unsafe capture result leaves the central loss-prevention guarantee incomplete; fail-before-mutation is the narrower safe behavior.
