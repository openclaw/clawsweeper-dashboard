## Summary

Keep this PR open for a focused repair: the overall inventory-and-recovery design is appropriate and the supplied behavior proof is credible, but the branch intentionally continues an update after local-override inspection fails, undermining its central preservation guarantee.

## What This Changes

The PR adds hashed package-content inventories, preserves locally changed packaged `dist` files in a recovery bundle during self-update, and provides an opt-in replay mode with conflict reporting.

## Best Possible Solution

Abort before invoking npm or pnpm whenever an installation has a content inventory but local-override inspection or capture cannot complete; return a structured failed `local overrides` step and leave the installed package untouched.

## Reproduction Assessment

Yes, from source and explicit tests. Supply a malformed content inventory or another capture error and the PR path still executes the global update without creating a recovery plan.

## Solution Assessment

No, not in its current form. A packaged hash baseline and default recovery bundle are the right ownership boundary, but preservation must fail before mutation whenever the updater cannot safely establish that bundle.
