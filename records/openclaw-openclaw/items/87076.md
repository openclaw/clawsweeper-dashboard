## Summary

The PR still needs maintainer review: the pre-manager QMD skip is useful in shape, but this head still drops the manager-side denial diagnostic below the default file log threshold, contains a test reset typo that leaves warn-log assertions contaminated, and has no real denied-scope runtime proof beyond unit tests.

## What This Changes

The PR adds a QMD scope check before `memory_search` resolves the memory manager, returns an empty `debug.skipped = "skipped_by_scope"` result when denied, lowers the QMD manager fallback denial log, and extends memory-core tests.

## Best Possible Solution

Keep the pre-manager skip, but preserve an info-or-higher manager fallback diagnostic, fix the warn/debug mock reset so the full QMD manager test file is stable, update the diagnostic docs if the observable wording changes, and add redacted live or terminal proof for a denied-scope QMD search.

## Solution Assessment
