Keep open: the narrow fix is still useful and current main does not already short-circuit denied QMD `memory_search` before manager resolution, but this PR needs real behavior proof, a current-main conflict refresh, and a small docs alignment before merge.

## What This Changes

The PR short-circuits QMD-backed `memory_search` when `memory.qmd.scope` denies the current session, returns an empty success with `debug.skipped = "skipped_by_scope"`, downgrades the manager guard log, and adds focused tests.

## Best Possible Solution

Land a refreshed narrow PR that preserves the current QMD scope policy, reuses the shared scope predicate before manager resolution, updates QMD diagnostics docs, and includes real after-fix QMD proof.

## Reproduction Assessment

Yes, at source level: current main opens the memory manager before the manager's QMD scope guard returns `[]` for a denied session. I did not run a live QMD repro in this read-only review.

## Solution Assessment

Yes, the core approach is the best narrow fix: move the existing QMD scope denial earlier while preserving the manager guard. It should be completed with docs alignment, real proof, and a current-main refresh rather than replaced by broader default-scope changes.
