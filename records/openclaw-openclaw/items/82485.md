## Summary

Keep open: current main still has a source-visible race in the approval handler cleanup path, and an open unmerged contributor PR already references this issue with closing syntax and proposes the focused fix.

## What This Changes

The issue reports that stopping a native approval handler while `deliverTarget` is in flight can leave a bound pending approval entry without a later `unbindPending` call.

## Best Possible Solution

Review, validate, and land https://github.com/openclaw/openclaw/pull/82482 or an equivalent current-main fix that guards in-flight delivery and adds regression coverage, then close this issue as implemented.

## Reproduction Assessment

Yes, source-level: current main still has the awaited `deliverPending`/`bindPending` window before `activeEntries.set`, and `onStopped` clears only entries already in the map. The issue also provides a deterministic gated stub reproduction, but I did not rerun it during this read-only review.

## Solution Assessment

Yes, the best path is the linked focused fix or an equivalent guard in `src/infra/approval-handler-runtime.ts` with a regression test for stop-during-delivery. Opening a separate repair PR would duplicate the already-open implementation PR.
