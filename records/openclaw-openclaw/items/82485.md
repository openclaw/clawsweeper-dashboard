## What This Changes

The issue reports an approval-handler runtime race where an in-flight delivery can be inserted into `activeEntries` after `onStopped` already cleared it, leaving `unbindPending` uncalled.

## Best Possible Solution

Land a reviewed, lifecycle-safe fix for the in-flight native approval delivery cleanup with deterministic regression coverage, then close this issue with merge commit proof.

## Reproduction Assessment

Yes. Keeping the issue open while the linked focused PR is reviewed is the right path; the implementation should use a lifecycle-safe guard and add the parked-delivery regression test.
