## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Prefer existing display names before derived titles:** `ui/src/ui/session-display.ts:100-102`
  - body: The UI now returns `derivedTitle` before `displayName`. Because the gateway already fills `displayName` for group/channel rows from `buildGroupDisplayName` while `derivedTitle` is derived only from entry fields or transcript content, enabling `includeDerivedTitles` can make those rows show a first-message/generated title instead of the stable channel/group label. Keep `displayName` ahead of `derivedTitle`, or only set `derivedTitle` when no stronger display name exists.
  - confidence: 0.87
- **[P2] Skip title generation when a stronger title exists:** `src/auto-reply/reply/session-title-generator.ts:47-48`
  - body: `maybeGenerateSessionTitle` only bails out for an existing `autoTitle`. Sessions that already have a manual `displayName` or `subject` still send transcript content to a background model and persist an unused `autoTitle`, which expands the opt-in privacy/cost boundary for titles that will not be displayed. Return before the model call when a stronger title already exists.
  - confidence: 0.82
