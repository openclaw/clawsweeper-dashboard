## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep credential steering out of plugin runtime auth:** `src/plugins/runtime/index.ts:144-148`
  - body: This wrapper is the plugin-facing `api.runtime.modelAuth` surface, and current tests document that plugin callers must not forward `agentDir`/credential-steering inputs. Forwarding `agentDir`, `profileId`, `preferredProfile`, and `lockedProfile` lets a plugin choose another agent/profile when resolving model auth. The title generator can call the internal runtime helper directly; keep these params off the plugin runtime wrapper and add coverage for stripping them.
  - confidence: 0.9
- **[P2] Preserve the session provider/model pair for title calls:** `src/auto-reply/reply/session-title-generator.ts:57-58`
  - body: When a session entry has stored runtime `modelProvider`/`model` but no `modelOverride`, this code passes the provider while leaving the model undefined. `generateConversationLabel` then falls back to the agent default model, which can pair the wrong model with the session provider. Pass the selected reply provider/model or the canonical session model pair instead.
  - confidence: 0.82
