## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Use the full session model for title generation:** `src/auto-reply/reply/session-title-generator.ts:58-59`
  - body: This only passes `modelProvider` plus `modelOverride` into the label generator. For sessions whose selected model is stored as runtime `modelProvider` + `model` with no override, `generateConversationLabel` receives no `modelId` and falls back to the agent default model, so transcript summarization can run on a different model than the session used. Reuse the canonical session model resolution path or pass `sessionEntry.model` when no override is present.
  - confidence: 0.91
- **[P1] Skip titled sessions before calling the model:** `src/auto-reply/reply/session-title-generator.ts:46-48`
  - body: `maybeGenerateSessionTitle` only returns early for an existing `autoTitle`. Sessions with a manual `displayName` or channel `subject` still send transcript content to a background model even though those stronger titles take precedence over `autoTitle`, so the model call is unused and widens the privacy surface. Add the stronger-title guard before reading transcripts or generating a label.
  - confidence: 0.88
