Keep open: current `main` and the latest release still contain the duplicate `Off` selector path, and the open linked fix PR has not merged yet.

## What This Changes

The issue reports that the Control chat thinking selector shows two indistinguishable `Off` choices for some Ollama models and remains selectable when no real thinking choice exists.

## Best Possible Solution

Fix the Control UI selector so duplicate `Off` is suppressed only when the inherited default is actually off, while preserving an explicit `Off` override for non-off defaults and adding focused regression coverage.

## Reproduction Assessment

Yes. Current main source and test fixtures show `value=""` inherited `Off` rendered alongside an explicit `off` option, and the reporter supplied screenshots plus a video from OpenClaw 2026.5.18.

## Solution Assessment

Yes, the narrowest fix belongs in the Control UI selector option-state logic rather than gateway thinking policy; the linked PR should be corrected before merge to preserve explicit `Off` for non-off inherited defaults.
