## What This Changes

The issue requests that the Control chat thinking selector show only one `Off` option and become disabled or unselectable when the selected model has no usable thinking choice.

## Best Possible Solution

Collapse default-off and explicit off into one clear selector state, disabling or making the control reset-safe for off-only models while keeping explicit `off` available when it meaningfully overrides a non-off default.

## Reproduction Assessment

Yes. The reporter supplied screenshots and a video from v2026.5.18, and current source plus tests still show the `value=""` Off and explicit `value="off"` Off options rendering together.

## Solution Assessment

Yes. The narrow maintainable fix is in the Control chat selector option-state logic with regression coverage for off-only and multi-level thinking metadata, not in gateway or provider thinking policy.
