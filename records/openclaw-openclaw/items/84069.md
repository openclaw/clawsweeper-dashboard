## Summary

Keep open: current main still has the duplicate `Off` selector path, and the two open linked fix PRs have not merged.

## What This Changes

The issue asks for the Control chat thinking selector to show a single `Off` choice and avoid an interactive chooser when no real thinking levels exist.

## Best Possible Solution

Land a corrected Control UI fix that suppresses duplicate `Off` only when the inherited default is actually off, disables the picker for off-only/no-real-option states, and preserves explicit `Off` for non-off inherited defaults.

## Reproduction Assessment

Yes. Current main renders the inherited empty-value option and then the explicit `off` option from thinking levels, and the reporter supplied visual proof from v2026.5.18.

## Solution Assessment

Yes. The maintainable fix belongs in the Control UI selector state/options layer, not in gateway thinking policy, with coverage for default-off, off-only, and inherited non-off defaults.
