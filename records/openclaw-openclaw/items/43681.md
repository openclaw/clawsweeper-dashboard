## Summary

Keep open. Current main still has no supported way for OAuth login to create a second OpenAI Codex profile for the same email; rotation works only after distinct profile ids already exist, and the related alias PR is closed unmerged.

## What This Changes

Allow OpenAI Codex OAuth login to register multiple auth profiles for the same provider/email/team and rotate across those profiles on rate limits.

## Best Possible Solution

Add a backward-compatible OAuth profile override to `models auth login` and the provider auth context, let OpenAI Codex use it when building the profile id, and document ordering/rotation for the resulting profiles.

## Reproduction Assessment

Yes, at source level. `models auth login` has no alias/profile-id input, OpenAI Codex maps email-bearing OAuth logins to `profileName: email`, and upsert overwrites the same profile id.

## Solution Assessment

Yes. A narrow supported profile-id/alias override is the best known fix because storage and failover already work once distinct profiles exist; changing rotation semantics would not solve profile creation.
