## Summary

Keep open. Current main still supports rotation once distinct auth profiles exist, but the supported OpenAI Codex OAuth login path cannot create distinct profiles for the same email account, and the related alias PR was closed unmerged.

## What This Changes

The issue requests supported same-provider OAuth profile separation so one OpenAI Codex account can use multiple team subscriptions through auth-profile rotation.

## Best Possible Solution

Add a backward-compatible OAuth login alias/profile override and wire it through the generic auth context into OpenAI Codex OAuth result construction, while leaving existing rotation and cooldown semantics intact.

## Reproduction Assessment

Yes at source level: `openclaw models auth login` has no alias/profile-id input, and OpenAI Codex maps email-bearing OAuth logins to the same `openai-codex:<email>` profile. That is enough to explain same-email multi-team profile overwrites without a live account.

## Solution Assessment

Yes, a narrow alias/profile override is the best known repair. The lower-level profile storage and runtime rotation already consume distinct profiles, so changing failover behavior is not the right first fix.
