## Summary

Keep open. Current main has per-agent auth stores and same-provider auth-profile rotation once multiple profiles exist, but `models auth login` still has no alias/profile-id input and OpenAI Codex OAuth still derives the profile name from the email first, so repeated logins for the same email can target the same profile id. The related alias PR was closed unmerged, so this is not implemented on main.

## What This Changes

Request support for registering multiple OAuth auth profiles for the same model provider/account, then rotating among those profiles for multi-team subscription failover.

## Best Possible Solution

Add a backward-compatible OAuth login alias/profile override path and wire it through the OpenAI Codex auth methods so users can create distinct same-provider profiles intentionally; leave the existing auth-order, cooldown, and model-fallback rotation semantics as the lower layer that consumes those profiles.

## Reproduction Assessment

Yes, at source level: inspect `openclaw models auth login` options and the OpenAI Codex identity resolver. The current path has no alias input and maps any OAuth result with an email to `openai-codex:<email>`, which is enough to explain same-email overwrites without needing a live multi-team account.

## Solution Assessment

No, current main is not the best solution yet. The maintainable direction is a narrow supported alias/profile-id input for OAuth login, optionally enhanced later with provider-confirmed team/org identity, rather than asking users to edit `auth-profiles.json` manually or changing rotation behavior that already exists.
