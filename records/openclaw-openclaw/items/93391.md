## Summary

Keep this PR open: the source fix is focused and current main still has the stale reply-context fallback, but the supplied proof only shows a patched installed bundle parses and does not demonstrate the Telegram reply-target behavior after the fix.

## What This Changes

This PR changes durable inbound reply fallback from `ReplyToId*` context fields to current-message `MessageSid*` fields and adds regression coverage for explicit-null and stale-reply ancestry cases.

## Best Possible Solution

Land the shared resolver change only after redacted live Telegram proof shows durable final replies anchor to the current inbound message and maintainers accept that as the shared fallback invariant.

## Reproduction Assessment

Yes as a source-level path: current main and the latest release resolve implicit durable replies from `ReplyToIdFull`/`ReplyToId`, while Telegram context stores current message ids separately as `MessageSid*`. I did not confirm a live Telegram misreply in this read-only review.

## Solution Assessment

Yes for the code shape, but not merge-ready: the shared resolver is the right layer because Telegram durable final delivery calls it before durable send. A Telegram-only workaround would split durable fallback semantics from the shared delivery path.
