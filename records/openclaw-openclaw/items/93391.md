Keep open: the code change looks like the right shared resolver fix for the source-level Telegram reply-target bug, but the supplied proof only shows a patched installed bundle parses and does not demonstrate the after-fix Telegram reply anchoring behavior.

## What This Changes

This PR changes durable inbound reply fallback from stale reply-context ids to current inbound message ids and updates regression coverage for explicit-null and stale-reply ancestry cases.

## Best Possible Solution

Land the shared resolver change after redacted live Telegram proof shows durable final replies anchor to the current inbound message and maintainers accept that as the shared fallback invariant.

## Reproduction Assessment

Yes as a source-level path: current main uses `ReplyToId*` for implicit durable reply fallback while Telegram stores the current inbound id in `MessageSid*`. I did not reproduce the visible Telegram misreply live in this read-only review.

## Solution Assessment

Yes for the code layer, but not merge-ready: the shared durable resolver is where Telegram final delivery resolves implicit reply targets before send. A Telegram-only workaround would split durable fallback semantics from the shared delivery path.
