## What This Changes

The PR changes durable inbound reply fallback from `ReplyToIdFull`/`ReplyToId` to current-message `MessageSidFull`/`MessageSid` and adds focused resolver regression tests.

## Best Possible Solution

Land the shared resolver change after redacted live Telegram proof confirms durable final replies anchor to the current inbound message and maintainers accept that as the shared fallback invariant.

## Reproduction Assessment

Yes as a source-level path: current main and the latest release resolve implicit durable replies from `ReplyToIdFull`/`ReplyToId`, while Telegram context stores current message ids separately as `MessageSid*`. I did not confirm a live Telegram misreply in this read-only review.

## Solution Assessment

Yes for the code shape, but not merge-ready: the shared resolver is the right layer because Telegram durable final delivery calls it before durable send. A Telegram-only workaround would split durable fallback semantics from the shared delivery path.
