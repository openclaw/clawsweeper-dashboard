## Summary

Keep this PR open: the source change appears narrow and relevant, but it still needs real Telegram behavior proof before merge because it changes user-visible reply anchoring for durable final delivery.

## What This Changes

The PR changes durable inbound reply fallback from `ReplyToId*` context fields to `MessageSid*` context fields and updates the durable-delivery regression tests.

## Best Possible Solution

Land the narrow resolver change after live Telegram proof shows durable final replies target the current inbound message and do not reuse stale reply ancestry.

## Reproduction Assessment

Yes, source-reproducible. Current main returns `ReplyToIdFull`/`ReplyToId` when no explicit reply target exists, while the inbound context also carries the current `MessageSid*`, so a context with stale reply ancestry and current message ids demonstrates the wrong fallback path without a live Telegram run.

## Solution Assessment

Yes, the source fix is the narrowest maintainable direction: choose the current inbound message ids as the ambient fallback while preserving explicit reply targets. It still needs real Telegram proof because the affected behavior is visible reply delivery.
