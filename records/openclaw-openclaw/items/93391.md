## What This Changes

The PR changes durable inbound reply fallback from `ctxPayload.ReplyToId*` to `ctxPayload.MessageSid*` and updates focused durable-delivery regression tests.

## Best Possible Solution

Land the shared resolver change after redacted live Telegram proof confirms durable final replies anchor to the current inbound message and maintainers accept that as the shared durable-delivery fallback invariant.

## Reproduction Assessment

Yes from source inspection: current main and v2026.6.6 fall back to `ReplyToId*`, while Telegram context separately carries the current inbound `MessageSid*`. I did not establish a live Telegram reproduction in this read-only review.

## Solution Assessment

Yes, the shared durable resolver is the right source layer because Telegram durable final delivery routes through it; a Telegram-only workaround would split the fallback invariant. The PR still needs real Telegram behavior proof before merge.
