## Best Possible Solution

Land the shared resolver change after redacted live Telegram proof confirms the durable final reply anchors to the current inbound message and maintainers accept that as the shared fallback invariant.

## Reproduction Assessment

## Solution Assessment

Yes, the shared durable resolver is the narrowest maintainable layer because Telegram final delivery and generic channel-turn durable paths route through it. A Telegram-only workaround would split the invariant, but merge should wait for real transport proof.
