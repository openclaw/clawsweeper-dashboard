## Summary

Keep open: current main and the latest release still have the Dashboard fast path using `config.token` while the async path resolves native/operator Control UI auth. The linked draft fix at https://github.com/openclaw/openclaw/pull/98473 is the active implementation candidate, so this issue should track until that PR lands or is replaced.

## What This Changes

The issue requests that the macOS Dashboard configured-window fast path use the same resolved Control UI auth token as the async Dashboard startup path.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/98473 so both Dashboard entry paths derive one URL/auth payload from `controlUiAutoAuthToken(config:)` with focused Swift regression coverage.

## Reproduction Assessment

Yes for source reproduction: current main calls the fast path from AppNavigationActions/MenuBar and uses `config.token` directly, while the async path resolves native/operator auth. No live macOS app run was attempted in this read-only review.

## Solution Assessment

Yes directionally: unifying both Dashboard startup paths around one resolved-token presentation path is the narrow maintainable fix. The open draft PR appears to pursue that direction, but it still needs normal PR review and validation before resolving this issue.
