## Summary

Keep open. This PR adds useful generic compat plumbing, but it is incomplete for the linked Fireworks failure because the affected bundled Kimi model still has no `compat.unsupportedToolSchemaKeywords` metadata for the new plumbing to read.

## What This Changes

The PR threads optional `modelCompat` through OpenAI strict tool-schema normalization and OpenAI Responses/Completions tool conversion, adds normalizer regression tests, and adds a changelog entry.

## Best Possible Solution

Keep the new generic OpenAI compat plumbing, add provider-owned Fireworks Kimi `compat.unsupportedToolSchemaKeywords` metadata, and cover the catalog-to-transport path so the bundled model strips `not` before OpenAI-compatible requests are built.

Yes.
