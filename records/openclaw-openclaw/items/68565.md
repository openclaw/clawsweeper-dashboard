## Summary

Keep open: the bug family is real, but this PR's global preservation rule would replay opaque Anthropic signatures into OpenAI Responses, where current main parses `thinkingSignature` as JSON; the PR also lacks after-fix real behavior proof.

## What This Changes

The branch changes `transformTransportMessages` to preserve signed or redacted thinking blocks across all model transitions, adds unit tests, and adds a changelog entry.

## Best Possible Solution

Implement target-compatible replay: preserve Anthropic signed/redacted thinking only for Anthropic-compatible targets, preserve OpenAI Responses JSON reasoning only for OpenAI-family targets, and cover the provider matrix with focused tests.

## Reproduction Assessment

Yes, by source inspection of the PR diff and current main. With the PR guard, an Anthropic `thinkingSignature` such as `sig_opaque` is retained for an OpenAI Responses target and then reaches `JSON.parse(block.thinkingSignature)`.

## Solution Assessment

No. The bug class is real, but a global signed/redacted preservation rule is not the narrowest maintainable fix because replay compatibility depends on the target transport.
