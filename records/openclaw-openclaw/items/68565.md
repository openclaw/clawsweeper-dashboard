## Summary

Keep open: the signed-thinking bug family is real, but this PR's global preservation rule is not safe because OpenAI Responses treats retained `thinkingSignature` as JSON replay metadata. The branch still has the same target-transport blocker noted in prior review, and the PR body/comments do not contain after-fix real behavior proof.

## What This Changes

The PR changes `transformTransportMessages` to preserve signed or redacted thinking blocks across all model transitions, with unit tests and a changelog entry.

## Best Possible Solution

Implement target-compatible replay: preserve Anthropic signed/redacted blocks only for Anthropic-compatible targets, preserve OpenAI Responses JSON reasoning only for OpenAI-family targets, and cover the provider matrix with focused tests.

## Reproduction Assessment

Yes, by source inspection of the PR branch. With the PR guard, an Anthropic `thinkingSignature: "sig_opaque"` replayed to an `openai-responses` target is retained and reaches `JSON.parse(block.thinkingSignature)`.

## Solution Assessment

No. The underlying bug class is real, but a global signed/redacted preservation rule is not the narrowest maintainable fix because replay compatibility depends on the target transport.
