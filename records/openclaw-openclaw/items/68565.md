## Summary

Keep this PR open but do not merge it as-is: the Anthropic replay bug class is plausible, yet the patch preserves signed/redacted thinking blocks globally and can break OpenAI Responses replay before request assembly. The existing ClawSweeper review already identified the same blocker, and there has been no later branch update or real after-fix behavior proof.

## What This Changes

The PR changes `transformTransportMessages` to preserve signed or redacted thinking blocks across model transitions, adds focused unit tests, and adds a changelog entry.

## Best Possible Solution

Implement target-compatible replay: preserve Anthropic signed/redacted blocks only for Anthropic-compatible targets, preserve OpenAI Responses JSON reasoning only for OpenAI-family targets, and cover the provider matrix with focused tests.

## Solution Assessment

No. The underlying bug class is real, but a global signed/redacted preservation rule is not the narrowest maintainable fix because replay compatibility depends on the target transport.
