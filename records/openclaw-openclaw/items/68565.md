## Summary

Keep open: the underlying Anthropic thinking replay bug class is real, and the linked reports are useful, but this PR’s global preservation rule introduces an OpenAI Responses replay failure and still lacks external-contributor real behavior proof.

## What This Changes

The PR globally preserves signed or redacted thinking blocks in `transformTransportMessages`, adds thinking-block preservation unit tests, and adds a changelog entry.

## Best Possible Solution

Implement target-compatible replay: preserve Anthropic signed/redacted thinking only for Anthropic-compatible targets, preserve OpenAI Responses JSON reasoning only for OpenAI-family targets, and cover the provider matrix with focused tests.

## Reproduction Assessment

Yes, by source inspection of the PR branch and current main. With the PR guard, an Anthropic `thinkingSignature` such as `sig_opaque` replayed to an OpenAI Responses target is retained and then reaches `JSON.parse(block.thinkingSignature)`.

## Solution Assessment

No. The bug class is real, but a global signed/redacted preservation rule is not the narrowest maintainable fix because replay compatibility depends on the target transport.
