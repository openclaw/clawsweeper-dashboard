## Summary

Keep open. The underlying Anthropic replay bug is credible, but this PR is not mergeable as written because it preserves opaque Anthropic thinking signatures for every target transport, which breaks OpenAI Responses request assembly.

## What This Changes

The PR changes `transformTransportMessages` to preserve signed or redacted thinking blocks across all model transitions, adds regression tests for those cases, and adds an Unreleased changelog entry.

## Best Possible Solution

Use a target-aware replay boundary that preserves Anthropic signed/redacted thinking only for compatible Anthropic-family targets, preserves OpenAI Responses JSON reasoning only on OpenAI-family replay, and downgrades or drops incompatible thinking blocks with matrix coverage.

## Reproduction Assessment

Yes. A static reproduction is enough: replay an Anthropic assistant thinking block with `thinkingSignature: "sig_opaque"` to an `openai-responses` model after the PR guard, and the OpenAI builder reaches `JSON.parse("sig_opaque")`.

## Solution Assessment

No. The intended bug fix is valid, but a global `signed or redacted` preservation rule is broader than the provider contracts allow; the safer fix is target-compatible preservation plus explicit downgrade/drop behavior for incompatible transports.
