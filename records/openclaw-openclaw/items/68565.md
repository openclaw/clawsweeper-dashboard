## Summary

Keep open: this PR targets a real Anthropic thinking replay failure, but the patch is not safe as-is because it preserves provider-specific signed/redacted thinking blocks globally and can feed opaque Anthropic signatures into OpenAI Responses JSON replay parsing. It also still lacks after-fix real behavior proof from a real setup.

## What This Changes

The PR changes `transformTransportMessages` to preserve signed or redacted thinking blocks across model transitions, adds unit tests for that behavior, and adds a changelog entry.

## Best Possible Solution

Implement provider/transport-aware replay that preserves Anthropic signed/redacted blocks only for compatible Anthropic-family targets, preserves OpenAI Responses JSON reasoning only for OpenAI-family targets, and covers the matrix with focused tests.

## Reproduction Assessment

Yes for the PR regression by source inspection: with the PR guard, an Anthropic `thinkingSignature: "sig_opaque"` replayed to an `openai-responses` target is retained and reaches `JSON.parse(block.thinkingSignature)`. I did not run a live provider repro.

## Solution Assessment

No. The underlying bug is real, but a global signed/redacted preservation rule is not the narrowest maintainable fix; the replay decision needs to be target-compatible.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9
