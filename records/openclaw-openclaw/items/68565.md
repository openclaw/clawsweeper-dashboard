## Summary

Keep open: the bug-fix direction is plausible, but this PR is not safe to merge because it globally preserves opaque Anthropic thinking signatures into provider paths that expect different replay metadata.

## What This Changes

The PR changes `transformTransportMessages` to preserve signed or redacted thinking blocks across all model transitions, adds unit coverage for that behavior, and adds a changelog entry.

## Best Possible Solution

Use a target-aware replay boundary that preserves Anthropic signed/redacted thinking only for compatible targets, preserves OpenAI Responses JSON reasoning only on OpenAI-family replay, and explicitly downgrades or drops incompatible thinking blocks with matrix coverage.

## Reproduction Assessment

Yes. A static source path is enough: after this PR's guard, an Anthropic `thinkingSignature: "sig_opaque"` replayed to `openai-responses` reaches `JSON.parse(block.thinkingSignature)`.

## Solution Assessment

No. The requested fix addresses a real class of Anthropic thinking replay failures, but the current branch applies a global preservation rule where the maintainable fix needs target-compatible preservation and downgrade/drop behavior.
