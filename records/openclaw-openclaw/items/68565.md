## Summary

Keep open. This is a real Anthropic replay bug, but the PR’s broad guard is not safe to merge because it preserves Anthropic opaque thinking signatures for OpenAI Responses replay, where current code expects JSON-encoded OpenAI reasoning items.

## What This Changes

The PR changes `transformTransportMessages` to preserve signed or redacted thinking blocks across all model transitions, adds replay regression tests, and adds an Unreleased changelog entry.

## Best Possible Solution

Add a target-aware replay policy: preserve Anthropic signed/redacted thinking verbatim only for Anthropic-compatible replay, preserve OpenAI Responses JSON reasoning items only for OpenAI Responses-compatible replay, and downgrade or drop incompatible thinking blocks for other targets with matrix coverage across Anthropic, OpenAI Responses, and Google routes.

## Reproduction Assessment

Yes. A high-confidence static reproduction is to apply the PR guard, replay an Anthropic assistant block with `thinkingSignature: "sig_opaque"` to an `openai-responses` target, and observe that `convertResponsesMessages` preserves the block then calls `JSON.parse("sig_opaque")`.

## Solution Assessment

No. Preserving signed/redacted thinking for every target is broader than the provider contracts allow; the maintainable fix is to decide preservation after considering both source and target transport compatibility.
