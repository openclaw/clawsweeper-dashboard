## Summary

Keep open: the underlying Anthropic thinking replay bug class is real, but this PR should not merge as-is because it preserves provider-specific thinking blocks globally and can break OpenAI Responses replay; it also lacks real behavior proof.

## What This Changes

The PR changes `transformTransportMessages` to preserve signed or redacted thinking blocks across all model transitions, adds unit tests, and adds a changelog entry.

## Best Possible Solution

Implement target-compatible replay: preserve Anthropic signed/redacted thinking only for Anthropic-compatible targets, preserve OpenAI Responses JSON reasoning only for OpenAI-family targets, and cover provider transitions with focused tests plus real proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence failing path: under the PR guard, an Anthropic opaque `thinkingSignature` is retained for an OpenAI Responses target, where `convertResponsesMessages` calls `JSON.parse` on it.

## Solution Assessment

No. The bug class is real, but global signed/redacted preservation is not the narrowest maintainable fix because replay compatibility depends on the target transport.
