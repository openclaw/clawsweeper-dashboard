Full review comments:

- **[P1] Add privacy to the runtime config schema:** `src/config/types.openclaw.ts:134-136`
  - body: This adds `privacy` to the TypeScript config type, but the strict `OpenClawSchema` does not include a matching key. A user who follows the documented config block will have normal config validation reject it as an unknown top-level field, so the new controls are unusable from real configuration.
  - confidence: 0.95
- **[P1] Thread privacy config through embedded prompts:** `src/agents/system-prompt.ts:472-474`
  - body: The new masking/redaction hook only works when `buildAgentSystemPrompt` receives `privacyConfig`, but the embedded messaging wrapper still does not pass that value through. That leaves Discord, Slack, Telegram, WhatsApp, and other embedded channel prompts unmasked even when the user enables the new system-prompt privacy settings.
  - confidence: 0.9
- **[P1] Wire transcript encryption before advertising at-rest protection:** `src/privacy/types.ts:94-96`
  - body: The PR documents `privacy.atRest.enabled` as encrypting session transcript files, but the production transcript writer still appends plaintext JSONL entries and no encrypted append/read path is integrated. That creates a security footgun where the config appears to protect session history while active sessions remain plaintext.
  - confidence: 0.95
- **[P1] Use async key derivation on runtime encryption paths:** `src/privacy/session-encrypt.ts:95-97`
  - body: `deriveKey` uses `pbkdf2Sync` with 210,000 iterations inside async public APIs. If this helper is wired into gateway/session paths, each encrypt/decrypt operation blocks the event loop and directory encryption multiplies that stall across files.
  - confidence: 0.86
- **[P2] Close the descriptor when encrypted-session probing throws:** `src/privacy/session-encrypt.ts:135-143`
  - body: `isEncryptedSessionFile` closes the file only after `fd.read()` succeeds. If the read throws, the outer catch returns `false` but the descriptor remains open, which can exhaust file descriptors in a long-running gateway that probes many session files.
  - confidence: 0.84
- **[P2] Mask repo paths that contain spaces:** `src/privacy/payload-redact.ts:224-229`
  - body: The repo-path masking regex stops at whitespace, so paths like `C:/Users/Alice Smith/workspace` only mask the prefix and still leak the rest of the home path into the runtime line. Use the pipe delimiter rather than whitespace as the field boundary.
  - confidence: 0.82
