Messaging channels therefore keep sending host/repo/context data even when the config says to mask it.
  - confidence: 0.93
- **[P1] Wire at-rest encryption into transcript writes:** `src/privacy/types.ts:98-99`
  - body: The config type says `privacy.atRest.enabled` will encrypt session transcript files, but no production caller reads `privacy.atRest` or invokes `encryptSessionFile`; the current transcript append path still writes JSONL directly. Enabling the documented option would leave sessions plaintext.
  - confidence: 0.91
- **[P2] Derive encryption keys asynchronously:** `src/privacy/session-encrypt.ts:95-96`
  - body: `deriveKey` uses `pbkdf2Sync` inside async encryption/decryption APIs. At the documented 210k iteration default, any runtime use would block the gateway event loop during session encryption/decryption and can compound across `encryptSessionDirectory`.
  - confidence: 0.88
- **[P2] Close the encryption probe fd in a finally block:** `src/privacy/session-encrypt.ts:140-143`
  - body: If `fd.read()` throws after `fs.open()` succeeds, the catch returns `false` without closing the file descriptor. Repeated probes on malformed or flaky files can leak descriptors in a long-running gateway process.
  - confidence: 0.82
- **[P2] Mask repo paths that contain spaces:** `src/privacy/payload-redact.ts:202`
  - body: The repo masking regex captures only up to whitespace, so paths like `C:/Users/Alice Smith/project` are partially rewritten and still leak the rest of the identifying path when `maskRepoPath` is enabled.
  - confidence: 0.78

## Risks / Open Questions

- Users who copy the documented `privacy:` block into `openclaw.yml` can hit strict config-validation failures because the runtime schema is not updated.
- Messaging-channel users would still leak hostname, repo path, shell/OS, and unredacted context-file content in system prompts because the embedded prompt path never receives `privacyConfig`.

- If the encryption helper is wired into runtime paths without changing `pbkdf2Sync`, gateway responsiveness can stall during key derivation, especially for directory encryption.
