Overall confidence: 0.93

Full review comments:

- **[P1] Add privacy to the runtime config schema:** `src/config/types.openclaw.ts:138`
  - body: The PR only adds `privacy` to the TypeScript config type, but `OpenClawSchema` remains strict and does not include the key. A user who copies the documented `privacy:` block into `openclaw.yml` can get an unknown-field validation failure before any of the new controls work.
  - confidence: 0.96
- **[P1] Thread privacy config through embedded prompts:** `src/agents/system-prompt.ts:472-474`
  - body: The new prompt masking is wired through the CLI helper, but messaging channels build prompts through `buildEmbeddedSystemPrompt`, which still has no `privacyConfig` field and does not forward one. That makes hostname/path/context-file masking inactive for Discord, WhatsApp, Slack, Telegram, and other embedded runs.
  - confidence: 0.94
- **[P1] Wire at-rest encryption into transcript writes before advertising it:** `src/privacy/types.ts:94-96`
  - body: `privacy.atRest.enabled` is documented as encrypting session transcript files, but this PR only adds bulk helper functions; production transcript writes still append plaintext JSONL entries. Users enabling the option would believe transcripts are protected when active session data is still stored in cleartext.
  - confidence: 0.93
- **[P2] Use async key derivation in gateway-facing encryption code:** `src/privacy/session-encrypt.ts:95-97`
  - body: `deriveKey` calls `pbkdf2Sync` with 210k iterations inside async file operations. If this helper is used from session or directory encryption paths, each derivation blocks the event loop and can stall gateway message handling while encryption runs.
  - confidence: 0.88
- **[P2] Close the encrypted-file descriptor on read failures:** `src/privacy/session-encrypt.ts:135-150`
  - body: `isEncryptedSessionFile` closes the fd only after `fd.read()` succeeds. A read error after open skips the close and leaks a descriptor in the long-running gateway process.
  - confidence: 0.84
- **[P2] Mask repo paths that contain spaces:** `src/privacy/payload-redact.ts:202-203`
  - body: The repo-path masking regex stops at whitespace, so paths like `C:/Users/Alice Smith/workspace` are only partially replaced and still leak the user-identifying remainder when `maskRepoPath` is enabled.
  - confidence: 0.82
