## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Add privacy to the runtime config schema:** `src/config/types.openclaw.ts:138`
  - body: The new `privacy` section is only added to the TypeScript config type. `OpenClawSchema` remains strict and has no top-level `privacy` key, so `validateConfigObjectRaw` rejects the documented `openclaw.yml` settings as unrecognized and the feature cannot be enabled through normal config.
  - confidence: 0.95
- **[P1] Wire at-rest encryption into session persistence:** `src/privacy/types.ts:98-99`
  - body: `privacy.atRest.enabled` is documented as encrypting session JSONL files, but the only references to the encryption helpers in the PR are the helper module and tests. Since the session writers never call `encryptSessionFile` or an append-time encrypted writer, enabling this setting does not actually protect active session transcripts at rest.
  - confidence: 0.9
- **[P1] Thread privacy config through embedded prompts:** `src/agents/system-prompt.ts:475-476`
  - body: `buildAgentSystemPrompt` only redacts context files and masks runtime fields when `privacyConfig` is supplied, but the embedded messaging prompt wrapper has no such parameter and passes no privacy value through. Messaging channels therefore keep sending unmasked system prompts even when the new settings are enabled.
  - confidence: 0.9
- **[P2] Use async key derivation for session encryption:** `src/privacy/session-encrypt.ts:95-96`
  - body: `deriveKey` calls `pbkdf2Sync` with the 210k-iteration default from async encrypt/decrypt APIs. Once wired into gateway/session flows, each encrypt/decrypt call can freeze the Node event loop, and directory encryption repeats that block once per file.
  - confidence: 0.86
- **[P2] Close the encrypted-file probe descriptor in a finally:** `src/privacy/session-encrypt.ts:140-143`
  - body: If `fd.read()` throws after `fs.open()` succeeds, the catch returns `false` without closing the file descriptor. A long-running gateway that probes many session files can leak descriptors under transient read errors.
  - confidence: 0.82
