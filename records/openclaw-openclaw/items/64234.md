## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Add privacy to the runtime config schema:** `src/config/types.openclaw.ts:138`
  - body: The branch adds `privacy` to the TypeScript config type but not to the strict runtime schema. Any normal config that sets the documented `privacy.*` keys will be rejected before redaction, media blocking, or encryption can be enabled.
  - confidence: 0.95
- **[P1] Honor documented PII defaults:** `src/privacy/payload-redact.ts:117-118`
  - body: `PrivacyConfig` says `pii.enabled` defaults to true when `privacy.enabled` is true, but `redactPii` returns without scanning unless `config.pii.enabled` is explicitly set. Users setting only the master switch would silently get no text redaction.
  - confidence: 0.9
- **[P1] Wire at-rest encryption into session writes:** `src/privacy/types.ts:98-99`
  - body: `privacy.atRest.enabled` is documented as encrypting session JSONL files, but the new encryption code is only a helper with tests. Active transcript writers still append plain JSONL, so enabling this setting would not protect newly written transcripts.
  - confidence: 0.9
- **[P1] Pass privacy config into embedded prompts:** `src/agents/system-prompt.ts:475-476`
  - body: `buildAgentSystemPrompt` only redacts runtime fields and context files when `privacyConfig` is supplied, but the embedded messaging prompt wrapper does not accept or pass it. Messaging channels therefore keep sending unmasked system prompts even when these settings are enabled.
  - confidence: 0.9
- **[P2] Preserve the resolved CLI provider:** `src/auto-reply/reply/agent-runner-execution.ts:996`
  - body: The CLI path computes `cliExecutionProvider` for configured CLI runtime overrides, but the branch passes the fallback `provider` into `runCliAgent`. That can route a CLI-backed run through the wrong backend or auth profile even though current main passes the resolved CLI provider.
  - confidence: 0.86
- **[P2] Derive encryption keys without blocking the event loop:** `src/privacy/session-encrypt.ts:95-96`
  - body: `deriveKey` calls `pbkdf2Sync` with a high iteration count from async encryption APIs. Once this is wired into gateway/session flows, each encrypt/decrypt operation can block the Node event loop, and directory encryption repeats that CPU-bound pause per file.
  - confidence: 0.86
- **[P2] Close the encrypted-file probe descriptor in a finally:** `src/privacy/session-encrypt.ts:140-143`
  - body: If `fd.read()` throws after `fs.open()` succeeds, the catch returns false without closing the file descriptor. A long-running gateway probing many session files can leak descriptors under transient read errors.
  - confidence: 0.82
- **[P2] Mask repo paths that contain spaces:** `src/privacy/payload-redact.ts:200-202`
  - body: The repo-path masking regex captures only up to whitespace, so paths such as `C:/Users/Alice Smith/workspace` are partially rewritten and still leak identifying path fragments when `maskRepoPath` is enabled.
  - confidence: 0.84

## Security Review

Status: needs_attention

Summary: Security review needs attention because several advertised privacy controls can be rejected, inactive, or only helper-level on production paths.

Concerns:

- **[high] Advertised privacy controls can be rejected or no-op:** `src/config/types.openclaw.ts:138`
  - body: The runtime schema does not accept the documented top-level `privacy` key, missing `pii.enabled` disables redaction despite the documented default, and embedded messaging prompts never receive the new privacy config.
  - confidence: 0.92
- **[high] At-rest encryption is not wired into active persistence:** `src/privacy/types.ts:98`
  - body: The AES-GCM code can encrypt an existing file in bulk, but active session writers are not changed to write encrypted records when `privacy.atRest.enabled` is set, so users could believe transcripts are protected when new JSONL remains plaintext.
  - confidence: 0.9
- **[medium] Synchronous PBKDF2 can stall the gateway when wired in:** `src/privacy/session-encrypt.ts:95`
  - body: The async-looking encryption APIs call `pbkdf2Sync` with 210k iterations, which can block message handling and UI/gateway work once used on production session paths.
  - confidence: 0.86
