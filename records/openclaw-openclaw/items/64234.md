## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Add privacy to the runtime config schema:** `src/config/types.openclaw.ts:138`
  - body: The branch adds `privacy` to the TypeScript config type but not to the strict runtime schema. A normal config containing the documented `privacy.*` keys will be rejected before these controls can be enabled.
  - confidence: 0.95
- **[P1] Honor documented PII defaults:** `src/privacy/payload-redact.ts:117-118`
  - body: `PrivacyConfig` says `pii.enabled` defaults to true when `privacy.enabled` is true, but `redactPii` returns without scanning unless `config.pii.enabled` is explicitly set. Users setting only the master switch would silently get no redaction.
  - confidence: 0.9
- **[P1] Wire at-rest encryption into session writes:** `src/privacy/types.ts:98-99`
  - body: `privacy.atRest.enabled` is documented as encrypting session JSONL files, but the new encryption code is only a bulk helper. Active transcript writers still append plaintext JSONL, so new session records remain unprotected.
  - confidence: 0.9
- **[P1] Pass privacy config into embedded prompts:** `src/agents/system-prompt.ts:475-476`
  - body: `buildAgentSystemPrompt` only masks runtime fields and context files when `privacyConfig` is supplied, but the embedded messaging wrapper never forwards it. Messaging channels therefore keep sending unmasked system prompts.
  - confidence: 0.9
- **[P2] Preserve the resolved CLI provider:** `src/auto-reply/reply/agent-runner-execution.ts:996`
  - body: The CLI path computes `cliExecutionProvider` for configured CLI runtime overrides, but this branch passes the fallback `provider` into `runCliAgent`. That can route a CLI-backed run through the wrong backend or auth profile.
  - confidence: 0.86
- **[P2] Derive encryption keys without blocking the event loop:** `src/privacy/session-encrypt.ts:95-96`
  - body: `deriveKey` calls `pbkdf2Sync` with a high iteration count from async encryption APIs. Once wired into gateway/session flows, each encrypt/decrypt operation can block message handling and other async work.
  - confidence: 0.86
- **[P2] Close the encrypted-file probe descriptor in a finally:** `src/privacy/session-encrypt.ts:140-143`
  - body: If `fd.read()` throws after `fs.open()` succeeds, the catch returns false without closing the descriptor. Repeated probes in a long-running process can leak file descriptors.
  - confidence: 0.82
- **[P2] Mask repo paths that contain spaces:** `src/privacy/payload-redact.ts:200-202`
  - body: The repo-path masking regex stops at whitespace, so paths such as `C:/Users/Alice Smith/workspace` are only partially rewritten and still leak identifying fragments when path masking is enabled.
  - confidence: 0.84
