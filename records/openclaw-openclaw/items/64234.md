## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Add privacy to the runtime config schema:** `src/config/types.openclaw.ts:138`
  - body: The branch adds `privacy` only to the TypeScript config type. `OpenClawSchema` is strict and still has no top-level `privacy` key, so normal config validation rejects the documented settings before any redaction, media blocking, or encryption controls can be enabled.
  - confidence: 0.95
- **[P1] Honor the documented PII defaults:** `src/privacy/payload-redact.ts:117-118`
  - body: `PrivacyConfig` says `pii.enabled` defaults to true when `privacy.enabled` is true, but `redactPii` returns without scanning unless `config.pii.enabled` is explicitly set. A user setting only the master privacy switch would silently get no text redaction.
  - confidence: 0.9
- **[P1] Wire at-rest encryption into session persistence:** `src/privacy/types.ts:98-99`
  - body: `privacy.atRest.enabled` is documented as encrypting session JSONL files, but the new encryption code is only a helper with tests. Active transcript writers still append plain JSONL, so enabling this setting would not protect newly written transcripts.
  - confidence: 0.9
- **[P1] Thread privacy config through embedded prompts:** `src/agents/system-prompt.ts:475-476`
  - body: `buildAgentSystemPrompt` only redacts runtime fields and context files when `privacyConfig` is supplied, but the embedded messaging prompt wrapper does not accept or pass it. Messaging channels therefore keep sending unmasked system prompts even when these settings are enabled.
  - confidence: 0.9
- **[P2] Preserve the resolved CLI execution provider:** `src/auto-reply/reply/agent-runner-execution.ts:996`
  - body: The CLI path computes `cliExecutionProvider` for configured CLI runtime overrides, but the branch passes the fallback `provider` into `runCliAgent`. That can route a CLI-backed run through the wrong backend or auth profile even though current main passes the resolved CLI provider.
  - confidence: 0.86
- **[P2] Use async key derivation for session encryption:** `src/privacy/session-encrypt.ts:95-96`
  - body: `deriveKey` calls `pbkdf2Sync` with a high iteration count from async encryption APIs. Once this is wired into gateway/session flows, each encrypt/decrypt operation can block the Node event loop, and directory encryption repeats that CPU-bound pause per file.
  - confidence: 0.86
- **[P2] Close the encrypted-file probe descriptor in a finally:** `src/privacy/session-encrypt.ts:140-143`
  - body: If `fd.read()` throws after `fs.open()` succeeds, the catch returns false without closing the file descriptor. A long-running gateway probing many session files can leak descriptors under transient read errors.
  - confidence: 0.82
- **[P2] Mask repo paths that contain spaces:** `src/privacy/payload-redact.ts:200-202`
  - body: The repo-path masking regex captures only up to whitespace, so paths such as `C:/Users/Alice Smith/workspace` are partially rewritten and still leak identifying path fragments when `maskRepoPath` is enabled.
  - confidence: 0.84
