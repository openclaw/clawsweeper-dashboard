## What This Changes

The PR adds a core auto-reply classifier that routes obvious small-talk turns through lightweight context with tools disabled, plus tests for that lane and Codex turn notification watchdog behavior.

## Best Possible Solution

Keep the optimization only after whole-message small-talk admission and runtime tool-policy support are proven, preserve the full path for mixed/current-info turns and always-on native CLI backends, then add latest-head Telegram proof with diagnostics.

## Reproduction Assessment

Yes, by source inspection: a message like `good morning, what's the weather today?` satisfies the broad greeting matcher, misses the narrow current-info denylist, and then receives `disableTools: true`. Current main disables Codex web search under `disableTools` and throws for always-on native CLI backends.

## Solution Assessment

No; the branch is a plausible optimization, but it is not the best fix while admission happens before runtime capability is known and while a single greeting token can admit a mixed-intent request. The safer shape is a whole-message small-talk classifier plus a runtime support gate before changing default options.
