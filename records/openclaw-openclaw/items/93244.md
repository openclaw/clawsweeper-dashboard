## Summary

Keep open: the latest head fixes the prior skill-filter admission bug, but the PR still introduces a global denylist-based reply shortcut and a Codex five-minute pre-item watchdog that can break existing tool/current-info replies and valid slow Codex startup paths.

## What This Changes

The branch adds a heuristic lightweight reply lane for eligible plain text replies, a five-minute Codex initial-progress watchdog, and focused tests plus Codex shard configuration.

## Best Possible Solution

Narrow lightweight routing to positively identified small-talk or an explicit owner-approved mode, preserve full tools for ambiguous/current-info turns, and keep Codex timeout behavior on the configured attempt budget until startup/prewarm can be separated from a stuck accepted turn with latest-head proof.

## Solution Assessment

No as submitted: the lightweight optimization and watchdog are plausible, but the best fix needs narrower admission, preserved tool/current-info behavior, and a timeout boundary that does not count valid prewarm silence as stalled progress.
