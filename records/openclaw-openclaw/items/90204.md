  - body: When `fs.symlink` throws `EPERM`, this test continues without creating `linkPath`, but the later 404 assertion can then come from a missing file rather than the symlink escape guard. Skip explicitly or use a meaningful Windows-compatible fallback so the security invariant is still exercised.
  - confidence: 0.9
