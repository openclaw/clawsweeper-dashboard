- **[P2] Keep POSIX conversion out of generic support strings:** `src/logging/diagnostic-support-redaction.ts:369-373`
  - body: `redactSupportString` handles ordinary diagnostic/error text as well as paths.   - confidence: 0.9
- **[P2] Do not pass symlink escape checks without a symlink:** `extensions/canvas/src/host/server.test.ts:460-466`
  - body: When `fs.symlink` throws `EPERM`, this test continues without creating `linkPath`, so the later 404 can come from a missing file rather than the symlink escape guard. Skip explicitly or use a meaningful Windows-compatible fallback so the security invariant is still exercised.

- **[P2] Keep trajectory session-file expectations platform neutral:** `src/trajectory/runtime.test.ts:48`
  - body: On Windows, `path.resolve('/tmp/session.trajectory.jsonl')` becomes a drive-qualified backslash path, but `resolveTrajectoryFilePath` returns the POSIX-looking string derived from the supplied `sessionFile`. This change makes the Windows-stability test fail instead of stabilizing it.
  - confidence: 0.88
- **[P3] Skip unsupported memory symlink cases explicitly:** `packages/memory-host-sdk/src/host/backend-config.test.ts:377-383`
  - body: Returning from these Vitest cases on `EPERM` reports symlink workspace coverage as passed rather than skipped. Use an explicit skip/helper or a meaningful fallback such as a junction where appropriate so CI output shows the invariant was not exercised.
  - confidence: 0.84
