## Repair Work Prompt

Fix the Matrix E2EE outbound failure reported in https://github.com/openclaw/openclaw/issues/82515. Current main lets matrix-js-sdk request MSC4222 `org.matrix.msc4222.use_state_after=true`; on homeservers that return incomplete `org.matrix.msc4222.state_after`, Rust crypto may never see `m.room.encryption` for a newly joined encrypted room, leaving outbound sends unconfigured. Implement a narrow Matrix-plugin compatibility mitigation, likely in the Matrix guarded fetch/client sync path, that avoids requesting MSC4222 state-after for `/sync` unless a safer explicit path is added. Preserve SSRF-pinned dispatcher and redirect behavior in `extensions/matrix/src/matrix/sdk/transport.ts`; do not move this policy into core. Add focused regression tests showing the `/sync` request no longer carries the problematic query parameter while unrelated Matrix requests are unchanged, and add/update Matrix E2EE/sync tests where practical. Add a user-facing changelog entry for the Matrix plugin/root changelog if repo convention requires both.
