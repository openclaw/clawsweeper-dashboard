## Summary

Keep open. The underlying memory supplement fan-out bug is still present on current main and the latest release, but this branch is not merge-ready because it converts all rejected supplement searches into a normal empty result and still lacks committed regression coverage plus non-test real behavior proof.

## What This Changes

The PR changes memory-core supplement search from fail-fast `Promise.all` aggregation to `Promise.allSettled` and drops rejected supplement results.

## Best Possible Solution

Yes. Source inspection gives a high-confidence path: register two memory corpus supplements, make one `search()` reject, and current main's `Promise.all` rejects the full supplement fan-out.

## Solution Assessment

No, not as-is. `searchMemoryCorpusSupplements` is the right owner boundary for isolating partial supplement failures, but this implementation also turns total requested-supplement failure into a normal empty search result.
