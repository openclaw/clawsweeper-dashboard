## Summary

Keep this PR open, but it is not merge-ready: current main and the latest release still have the fail-fast supplement bug, while this branch changes total supplement failure into a successful empty result and still lacks committed regression coverage plus non-test real behavior proof.

## What This Changes

The PR changes `searchMemoryCorpusSupplements` from fail-fast `Promise.all` aggregation to `Promise.allSettled` and drops rejected supplement results.

## Best Possible Solution

Preserve fulfilled sibling supplement results inside `searchMemoryCorpusSupplements` while still surfacing unavailable/error output when no requested supplement succeeds, with focused committed tests and redacted non-test proof before merge.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: register two memory corpus supplements, make one `search()` reject, and current main's `Promise.all` rejects the full supplement fan-out for `memory_search` with `corpus="wiki"` or `corpus="all"`.

## Solution Assessment

No, not as-is. Isolating partial supplement failures in `searchMemoryCorpusSupplements` is the right owner boundary, but this implementation also turns total requested-supplement failure into a normal empty result.
