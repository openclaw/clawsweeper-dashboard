## Summary

This PR should close because a later open PR tracks the same memory-core bug with the more complete fix direction, while this branch still has a correctness gap around silently swallowed supplement failures.

## What This Changes

The PR changes memory-core corpus supplement search aggregation from fail-fast `Promise.all` to `Promise.allSettled` so fulfilled supplement results are retained when another supplement rejects.

## Best Possible Solution

Use the successor PR's owner-module approach: isolate supplement failures, preserve operator diagnostics with plugin IDs, cover helper and tool-path regressions, and keep the linked issue open until that fix lands.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: register one resolving and one rejecting memory corpus supplement, then call `memory_search` with `corpus="all"`; current main's `Promise.all` rejects the supplement fanout before sibling results can be returned.

## Solution Assessment

No. `Promise.allSettled` is the right primitive, but this branch drops rejected outcomes without diagnostics and is superseded by https://github.com/openclaw/openclaw/pull/78035, which tracks the same bug with tests and a fuller runtime policy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve rejected supplement diagnostics:** `extensions/memory-core/src/tools.shared.ts:169`
  - body: This `flatMap` drops every rejected supplement result, so a broken only registered supplement, or a `corpus=wiki` call where memory-wiki fails, becomes a successful empty result with no unavailable response or warning. Keep partial successes, but preserve a diagnostic for rejected supplements so operators can identify the failing corpus provider.
