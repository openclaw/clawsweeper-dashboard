## Summary

Keep this PR open, but it is not merge-ready: the useful partial-success direction is in the right helper, while the current patch also turns wiki-only or all-failed supplement searches into normal empty successes and still lacks committed regression coverage plus real behavior proof.

## What This Changes

The PR changes memory-core supplement search aggregation from fail-fast `Promise.all` to `Promise.allSettled` so fulfilled supplement results survive rejected siblings.

## Best Possible Solution

Preserve fulfilled sibling supplement results while still surfacing unavailable/error output when no requested supplement succeeds, with focused committed tests and redacted non-test proof before merge.

Yes. Source inspection gives a high-confidence path: register two memory corpus supplements, make one `search()` reject, and current main's `Promise.all` rejects the full supplement fan-out for `memory_search` with `corpus="wiki"` or `corpus="all"`.

## Solution Assessment

No, not as-is. `searchMemoryCorpusSupplements` is the right owner boundary for isolating partial supplement failures, but this implementation also turns total requested-supplement failure into a normal empty result.
