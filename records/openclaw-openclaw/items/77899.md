## Summary

Keep open: the PR targets a valid current-main fail-fast bug, but the patch would also hide a complete supplement outage by returning an apparently successful empty result.

## What This Changes

The PR changes memory corpus supplement aggregation from `Promise.all` to `Promise.allSettled` so successful supplement results survive when another supplement rejects.

## Best Possible Solution

Use `Promise.allSettled` for partial success, but preserve the existing unavailable/error path when every registered supplement rejects and add focused regression coverage for both cases.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: register at least two memory corpus supplements and make one `search()` reject; current main’s `Promise.all` rejects the whole supplement aggregate.

## Solution Assessment

No. `allSettled` is the right primitive, but this patch should not convert an all-failed supplement set into a silent empty success.

## Review Findings

Overall correctness: patch is incorrect
