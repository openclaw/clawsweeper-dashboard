## Summary

Keep open for maintainer handling: current main still has the reported fail-fast bug, but this PR is not mergeable as-is because it silently drops all supplement failures and lacks external real behavior proof. A newer open PR tracks a fuller fix at https://github.com/openclaw/openclaw/pull/78035, but it is still open/conflicting, so this branch should not be auto-closed by cleanup.

## What This Changes

The PR changes memory corpus supplement aggregation from fail-fast `Promise.all` to `Promise.allSettled` so fulfilled supplement results can be returned when another supplement rejects.

## Best Possible Solution

Use an `allSettled`-style isolation in memory-core that preserves sibling results, retains or logs diagnostics for rejected supplements, and lands with focused regression coverage; https://github.com/openclaw/openclaw/pull/78035 appears to be the stronger current candidate if its remaining cleanup and conflict are resolved.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: register at least two memory corpus supplements, make one `search()` reject, then call `memory_search` with `corpus="all"`; current main rejects through `Promise.all` before returning healthy supplement results.

## Solution Assessment

No. `Promise.allSettled` is the right primitive for partial success, but this patch also turns an all-failed supplement set into a silent empty success and does not add committed regression coverage or real behavior proof.

## Review Findings

Overall correctness: patch is incorrect
