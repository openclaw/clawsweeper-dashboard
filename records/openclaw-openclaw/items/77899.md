## Summary

Keep open: the linked memory retrieval bug is real, but this branch is not merge-ready because it turns every rejected requested supplement search into an empty successful result and only supplies test-style proof.

## What This Changes

The PR changes `searchMemoryCorpusSupplements` to use `Promise.allSettled` and flatten only fulfilled supplement search results.

## Best Possible Solution

Keep the all-settled direction for partial successes, but preserve explicit unavailable/error behavior when every requested supplement fails and add focused regression coverage plus real behavior proof.

## Reproduction Assessment

Yes, source-reproducible: current main and v2026.6.6 still use `Promise.all`, so a rejecting supplement rejects the supplement fan-out. I did not run a live failing memory setup in this read-only review.

## Solution Assessment

No, not as-is: `Promise.allSettled` is the right direction for partial successes, but this implementation also turns total requested-supplement failure into a normal empty result. The safer fix preserves partial successes while still surfacing unavailable/error state when nothing requested succeeded.
