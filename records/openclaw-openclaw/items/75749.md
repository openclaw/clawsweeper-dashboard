## Summary

Keep open: current main and v2026.6.10 still short-circuit Telegram fallback suppression when `turnSourceChannel` is missing, and the direct fix candidate is open but unmerged.

## What This Changes

The issue reports duplicate Telegram plugin approval prompts when a null `turnSourceChannel` lets both generic forwarding fallback and Telegram native delivery send the same approval.

## Best Possible Solution

Land or replace the linked focused fix so Telegram native/fallback suppression handles missing `turnSourceChannel` without breaking sibling-channel fallbacks, then close this issue after merge.

## Reproduction Assessment

Yes, at source level: current main skips fallback suppression on null `turnSourceChannel`, while Telegram native matching can still accept a session/account-bound request. I did not run a live Telegram DM reproduction in this read-only review.

## Solution Assessment

Yes in direction: the maintainable fix is to suppress the generic fallback only when Telegram native delivery can handle the same null-source request. The open PR is the current direct candidate, but it still needs rebase/proof before merge.
