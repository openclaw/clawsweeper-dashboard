## What This Changes

The issue reports duplicate Telegram plugin approval prompts when a request without `turnSourceChannel` is delivered by both the forwarding fallback and Telegram native approval runtime.

## Best Possible Solution

Unify forwarding fallback suppression with Telegram native session/account matching for missing `turnSourceChannel`, then add regression coverage for plugin approvals with null turn-source plus preserved foreign-channel and disabled-native cases.

## Reproduction Assessment

Yes from source, not live execution: current fallback suppression returns false for missing `turnSourceChannel`, while Telegram native matching can still accept the same request through persisted session/account binding.

## Solution Assessment

Yes: the best fix is a bounded alignment of the shared suppression helper or generic session-derived effective channel/account path, not a Telegram-only send workaround.

## Repair Work Prompt

Fix the duplicate Telegram plugin approval delivery reported in https://github.com/openclaw/openclaw/issues/75749. Current `main` lets `src/plugin-sdk/approval-delivery-helpers.ts` return false from `shouldSuppressForwardingFallback` when Telegram requires a matching `turnSourceChannel` and the request has no `turnSourceChannel`, while `extensions/telegram/src/exec-approvals.ts` can still match the same request via session/account binding and native runtime delivery. Keep core plugin-agnostic; prefer a generic effective channel/account/session-binding check over Telegram-specific core policy. Add focused regression coverage proving a plugin approval with null or missing `turnSourceChannel`, a Telegram persisted session target, and native delivery enabled does not call the forwarding fallback delivery path. Preserve behavior for explicit non-Telegram `turnSourceChannel`, explicit targets, and disabled native delivery. Coordinate with open PR https://github.com/openclaw/openclaw/pull/84205 if it is still open, but do not duplicate its agent-side turn-source propagation work. No changelog edit.
