## Summary

Keep open: current main still has the null `turnSourceChannel` suppression gap, and the linked fix candidate is open but unmerged.

## What This Changes

The issue reports duplicate Telegram plugin approval prompts when generic forwarding fallback and Telegram native approval delivery both handle a request with missing `turnSourceChannel`.

## Best Possible Solution

Land or replace the linked focused fix so Telegram native/fallback suppression handles missing `turnSourceChannel` without dropping sibling-channel approval fallbacks, then close this issue after merge.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current main skips fallback suppression on null `turnSourceChannel` while Telegram native matching can still accept the same request by account/session. I did not run a live Telegram DM repro in this read-only review.

## Solution Assessment

Yes in direction, but the safest implementation should scope null-source suppression to native handlers that can actually deliver the request. The open shared-helper PR needs sibling-channel repair or replacement before it is the best mergeable fix.
