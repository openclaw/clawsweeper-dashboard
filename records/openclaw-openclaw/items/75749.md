## Summary

Current main still has the reported mismatch: Telegram native plugin approvals can resolve a session-bound request without `turnSourceChannel`, while forwarding fallback suppression returns before the native-delivery/account checks when that field is absent. The bug remains narrow and actionable.

## What This Changes

The issue asks OpenClaw to prevent duplicate Telegram plugin approval prompts when a request has a Telegram session binding but no `turnSourceChannel`.

## Best Possible Solution

Add regression-covered suppression that derives the effective approval channel/account from explicit turn-source fields or persisted session binding, then suppresses Telegram fallback only when native delivery is enabled for that route.

## Reproduction Assessment

Yes. A focused test can create a Telegram session-store entry, submit a plugin approval request with only `sessionKey`, and assert that native Telegram handles it while the forwarding fallback is currently not suppressed.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the shared approval delivery suppression/session-binding path with Telegram supplying existing channel/account facts; #74003 should remain separate metadata propagation work.

## Repair Work Prompt

Fix the Telegram plugin approval duplicate-delivery bug reported in openclaw/openclaw#75749. When `plugin.approval.request` has a Telegram session binding but no `turnSourceChannel`, the Telegram native runtime handles the request, but the forwarding fallback is not suppressed. Update the shared approval delivery suppression/Telegram capability path so the effective channel/account can be derived from explicit turn-source fields or persisted session binding, then suppress Telegram fallback only when native delivery is enabled for that effective route. Preserve fallback behavior for explicit non-Telegram turn sources, disabled native Telegram approvals, targets with no session binding, and multi-account account scoping. Do not fold in the broader #74003 metadata propagation fix except as needed for test fixtures. Add focused regression coverage for plugin approval with `sessionKey` and no `turnSourceChannel`, and add a single-line `CHANGELOG.md` fix entry because this is user-visible.
