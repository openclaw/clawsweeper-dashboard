## Summary

Current main still has the reported mismatch: Telegram native plugin approvals can resolve a session-bound request without `turnSourceChannel`, while forwarding fallback suppression returns before checking the session/native-delivery route. This is narrow, current, and actionable.

## What This Changes

The issue requests suppressing Telegram approval forwarding fallback when a plugin approval without `turnSourceChannel` is already handled by Telegram native session-bound delivery.

## Best Possible Solution

Add regression-covered fallback suppression that derives the effective channel/account from explicit turn-source fields or persisted session binding, then suppresses Telegram fallback only when native delivery is enabled for that route.

## Reproduction Assessment

Yes. A focused test can create a Telegram session-store entry, submit a plugin approval request with only `sessionKey`, and assert that native Telegram handles it while the forwarding fallback is currently not suppressed.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the shared approval-delivery suppression/session-binding path with Telegram supplying existing channel/account facts; #74003 should remain separate metadata propagation work.

## Repair Work Prompt

Fix openclaw/openclaw#75749. Current main lets Telegram native plugin approvals handle a session-bound request without `turnSourceChannel`, but the forwarding fallback suppression short-circuits before session/account/native-delivery checks, so Telegram can receive duplicate approval prompts. Keep the fix narrow: update the shared approval delivery suppression path and/or Telegram suppression account/channel resolver so an absent turn-source channel can use persisted session binding, but do not suppress fallback for foreign channels, missing sessions, disabled native delivery, or mismatched Telegram accounts. Keep #74003 as separate metadata propagation work. Add focused regression coverage for a Telegram plugin approval with only `sessionKey` proving native handling suppresses the forwarder fallback, plus negative coverage for foreign/missing session routes. Add a user-facing `CHANGELOG.md` fix entry if production behavior changes. Do not alter unrelated approval routing, Telegram callback handling, or broad plugin SDK contracts beyond the minimal compatible seam needed.
