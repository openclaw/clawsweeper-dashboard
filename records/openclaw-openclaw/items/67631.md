## Summary

Keep open. Current main and the latest release still expose the reported doctor/LaunchAgent ownership path, and the issue now has an open closing PR that should be reviewed or replaced before the issue is closed.

## What This Changes

The issue reports duplicate macOS gateway processes when OpenClaw.app owns `ai.openclaw.mac` but terminal `openclaw doctor --fix` installs or repairs `ai.openclaw.gateway`.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/81725 or a corrected successor that guards every CLI Gateway lifecycle mutation on a precise app-owned local-mode signal while preserving named profile services.

## Reproduction Assessment

Yes, source-backed rather than live macOS. Current main reaches Gateway LaunchAgent bootstrap, install, restart, and service-config repair paths without an app-owner guard, and the issue comments describe the matching conflict and manual recovery path.

## Solution Assessment

Yes as a direction, but not by closing this issue now. The narrow maintainable solution is to review and land the open closing PR or an equivalent corrected successor, then close this issue after merge.
