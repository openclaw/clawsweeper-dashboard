## Summary

Keep open. Current main still lets the terminal doctor path install, start, or restart the standalone macOS Gateway LaunchAgent without detecting that the macOS app LaunchAgent owns the local runtime, and open PR #73974 is already the closing implementation candidate.

## What This Changes

The issue asks the macOS app and CLI launchd paths to prevent duplicate `ai.openclaw.gateway` gateway processes when `ai.openclaw.mac` owns the local runtime.

## Best Possible Solution

Make macOS gateway ownership explicit so CLI doctor/gateway repair skips or defers standalone Gateway LaunchAgent install/start/restart when `ai.openclaw.mac` is installed or loaded, with focused regression tests and docs.

## Reproduction Assessment

Yes. The issue gives concrete macOS launchd steps, and current main source shows `doctor --fix` can still install or restart `ai.openclaw.gateway` without an `ai.openclaw.mac` ownership guard; I did not run a live macOS launchd reproduction.

## Solution Assessment

Yes as a direction. The narrow maintainable path is to review and land PR #73974 or an equivalent replacement that adds app-owner detection before CLI launchd repair.
