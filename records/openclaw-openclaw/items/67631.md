## Summary

Keep open. Current main still lacks an OpenClaw.app LaunchAgent ownership guard in the terminal doctor/gateway repair paths, and the open closing PR for the same bug remains unmerged with a blocking review finding and conflicts.

## What This Changes

The issue asks macOS doctor/gateway repair to avoid installing or restarting the standalone `ai.openclaw.gateway` LaunchAgent when OpenClaw.app owns the local runtime through `ai.openclaw.mac`.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/73974 or an equivalent corrected fix that uses app-specific launchd evidence before skipping CLI install/start/restart/config repair, with focused regression tests and docs.

## Reproduction Assessment

Yes, source-backed. The issue gives concrete macOS LaunchAgent steps, and current main still has doctor paths that can install, start, restart, or repair `ai.openclaw.gateway` without checking for an app-owned `ai.openclaw.mac` runtime.

## Solution Assessment

Yes as a direction, but not by closing the issue now. The narrow maintainable solution is to review and land the linked PR or a replacement that validates app-owned launchd state rather than relying on label-only checks.
