## What This Changes

The issue asks OpenClaw to prevent the macOS app and terminal CLI doctor flow from creating competing Gateway LaunchAgent ownership that can desynchronize Mac App and Telegram sessions.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/81725 or a corrected successor that guards every default CLI Gateway lifecycle mutation on a precise app-owned local-mode signal while preserving named profiles and attach-only mode.

## Reproduction Assessment

Yes, source-backed rather than live macOS. Current main reaches default LaunchAgent bootstrap, install, start/restart, and service-config repair paths without an app-owned launchd guard, and the issue comments describe the matching conflict and recovery path.

## Solution Assessment

Yes as a direction, but not by closing this issue now. The narrow maintainable solution is to guard CLI doctor Gateway lifecycle repair when the app owns local launchd, while preserving attach-only and named-profile behavior.
