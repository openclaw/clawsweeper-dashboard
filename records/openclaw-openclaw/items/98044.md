## Summary

Keep open: current main and v2026.6.10 still start the Android QR scanner without a CAMERA permission preflight and only show a generic scanner failure, while camera permission is requested later in onboarding/settings, so the reported QR setup blocker remains actionable.

## What This Changes

Request Android onboarding to ask for camera permission before QR setup scanning and explain or route to settings when permission blocks scanning.

## Best Possible Solution

Handle CAMERA permission in the Android onboarding QR scan path itself: request it on scan tap, show clear inline permission-required copy when denied, and expose an Android app settings action when the runtime prompt will not appear.

## Reproduction Assessment

Yes, as a high-confidence source reproduction: deny CAMERA on Android, tap Scan setup code, and current main calls startScan without a permission request and reports only a generic scanner error. I did not perform a live device run in this read-only review.

## Solution Assessment

Yes, the best fix is to keep this in Android onboarding's scan handler and reuse the app's existing permission/settings patterns; changing gateway pairing or auth diagnostics would treat the symptom at the wrong boundary.
