## Summary

The report points to an incompatible/outdated macOS Command Line Tools/Homebrew environment, while current OpenClaw intentionally delegates gateway-backed skill dependency installs to the selected host installer and returns that installer failure.

## What This Changes

Triage a Mac app skill-install failure where `skills.install` returns Homebrew's CLT error on macOS 26.

## Best Possible Solution

Leave OpenClaw's host-installer contract intact and have affected users update or reinstall Xcode/CLT, or opt out of Homebrew-backed skill installs through existing config when a node/download installer is available.

## Reproduction Assessment

No high-confidence end-to-end reproduction was run on a macOS 26 host. Source inspection does show that a selected brew installer failure will be propagated through `skills.install` as `UNAVAILABLE`, which matches the reported message.

## Solution Assessment

Not applicable as an OpenClaw core fix. The narrow remedy is to fix the host Xcode/CLT/Homebrew setup or use existing `skills.install.preferBrew` and node-manager configuration when a non-brew installer is available.
