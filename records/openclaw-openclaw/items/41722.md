## Summary

Keep open. Current main still routes macOS Skills installs through gateway `skills.install`, prefers Homebrew when `brew` exists, and returns nonzero installer failures as `UNAVAILABLE` without a macOS 26 / Apple CLT remediation classifier or safe alternate-installer retry.

## What This Changes

Installing a skill from the macOS app on macOS 26 can fail when the gateway selects a Homebrew-backed installer and Homebrew reports unsupported Command Line Tools.

## Best Possible Solution

Add narrow gateway installer/status handling for Homebrew Apple CLT unsupported-macOS failures, with targeted remediation text and fallback only to another declared installer when existing policy permits it.

## Reproduction Assessment

Yes, source-backed. On a macOS 26 host where `brew` exists but Homebrew fails because Apple CLT is unsupported, the macOS app calls `skills.install`, status selection prefers brew, and the gateway surfaces the raw installer failure as `UNAVAILABLE`.

## Solution Assessment

No. Current main is not the best solution because it preserves the raw Homebrew failure; the maintainable fix belongs in the gateway installer/status selection layer, not only in macOS UI copy.
