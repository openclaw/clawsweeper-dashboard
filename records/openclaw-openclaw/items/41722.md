## Summary

Keep open. Current main still prefers a brew installer when Homebrew is present, then surfaces the resulting installer failure through `skills.install` as `UNAVAILABLE` without a macOS 26 / Apple CLT remediation classifier or safe alternate-installer retry.

## What This Changes

Installing skills from the macOS app on macOS 26 should not fail with a raw Homebrew/CLT unsupported-macOS error.

## Best Possible Solution

Add a narrow gateway installer/status fix that detects Homebrew/Apple CLT unsupported-macOS failures, returns targeted remediation, and only falls back to another installer when the skill metadata and existing installer policy allow it.

## Reproduction Assessment

Yes, source-backed. On a macOS 26 host where `brew` is present but the Apple CLT is unsupported, the macOS Skills UI calls `skills.install`, current status selection prefers brew, and current installer code returns the brew failure as `UNAVAILABLE`.

## Solution Assessment

No. Current main is not the best solution because it preserves the raw Homebrew failure; the maintainable fix belongs in the gateway skill installer/status selection layer rather than only in macOS UI copy.
