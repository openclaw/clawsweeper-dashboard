## Summary

Keep open. Current main still routes macOS Skills installs through gateway `skills.install`, prefers Homebrew when `brew` exists, and returns nonzero Homebrew failures as `UNAVAILABLE` without a macOS 26 Apple CLT classifier, remediation message, or safe alternate-installer retry.

## What This Changes

Make macOS app Skills installs avoid or clearly remediate the Homebrew Apple CLT unsupported-macOS failure instead of surfacing the raw installer error.

## Best Possible Solution

Add a narrow Homebrew Apple CLT unsupported classifier/remediation path in the gateway skill installer, and only retry another declared installer when the existing selection policy safely allows it.

## Reproduction Assessment

Yes, source-backed. On a macOS 26 host where `brew` exists but Homebrew fails with the Apple CLT unsupported message, the macOS app calls gateway `skills.install`, the gateway executes the selected brew installer, and the raw failure is returned as `UNAVAILABLE`.

## Solution Assessment

No. Current main preserves the raw Homebrew failure; the maintainable fix belongs in shared gateway installer/status handling with targeted CLT remediation and safe alternate selection, not UI copy alone.
