## Summary

Keep open. Current main still routes macOS Skills installs through the gateway, selects Homebrew when `brew` is present, and formats nonzero installer exits generically, so the reported macOS 26 CLT/Homebrew failure would still surface as `UNAVAILABLE` without targeted remediation.

## What This Changes

The issue asks for macOS app skill installation to stop failing on macOS 26 when Homebrew exits with Apple CLT unsupported output.

## Best Possible Solution

Add a gateway-side Homebrew Apple CLT unsupported classifier with actionable remediation, and only retry another declared installer when the existing selection policy safely permits it.

## Solution Assessment

No. Current main preserves the raw Homebrew failure; the maintainable fix belongs in shared gateway installer/status handling with targeted CLT remediation, not only macOS UI copy.
