## Summary

Keep open: current main no longer has the old direct-resume relay, but ACP completion still reaches the parent through system-event plus heartbeat wake, and that path can still skip when heartbeat delivery is unavailable or disabled. The same-author PR #60402 remains the narrow implementation candidate, while the later native-child comments broaden the desired endpoint beyond a simple cleanup close.

## What This Changes

The issue asks OpenClaw to resume yielded parent sessions automatically after ACP or child completion without depending on optional heartbeat wake behavior.

## Best Possible Solution

Land or replace #60402 for the narrow ACP heartbeat-disabled gap, then converge ACP and native child completion on a maintained yielded-parent delivery primitive with focused heartbeat-disabled and yielded-parent regression coverage.

## Reproduction Assessment

Yes, source-level: ACP completion enqueues a parent-session event and schedules a heartbeat wake, while the wake path can return with no handler or no enabled heartbeat agent. I did not run the live browser or Matrix repro in this read-only review.

## Solution Assessment

No for current main: routing completion through heartbeat wake is not self-sufficient when heartbeat delivery is disabled or unavailable. The safer direction is a shared yielded-parent completion delivery path rather than another relay-specific direct re-entry workaround.
