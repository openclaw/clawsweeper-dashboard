## Label Justifications

- `P1`: Completed child work can leave ACP/native parent sessions wedged until a user refresh, poke, or later inbound event in real agent/channel workflows.
- `impact:session-state`: The requester session can remain yielded/running instead of promptly resuming after child completion.
- `impact:message-loss`: Child completion events can be delayed or fail to produce the expected visible parent reply, which users experience as lost delivery.

## Summary

Keep open: the original direct-resume root-cause text is stale, but current main still has a source-backed yielded-parent wake gap. ACP `streamTo:"parent"` still queues system events and depends on heartbeat wake delivery, related native completion issues remain open, and https://github.com/openclaw/openclaw/pull/60402 is still open, conflicting, and proof-blocked rather than a safe canonical fix.

## What This Changes

The issue requests reliable automatic resumption for yielded parent sessions after ACP or native child completion, without relying on stale transcript inspection or optional heartbeat delivery.

## Best Possible Solution

Define one yielded-parent completion delivery contract for ACP and native subagents that wakes active requester sessions independently of optional heartbeat scheduling, with focused regression coverage for heartbeat-disabled ACP and native `sessions_yield` completion cases.

## Reproduction Assessment

Yes, source-level: explicit ACP parent streaming still queues a parent-session system event and requests a heartbeat wake, while the heartbeat runner can skip before targeted delivery when heartbeat is disabled or unconfigured. I did not run a live ACP/control-ui or native channel repro in this read-only review.

## Solution Assessment

No, the original direct-resume explanation is no longer the best fix target because that old path is absent on current main. The better fix is at the shared yield/announce/wake contract so ACP and native completions resume the requester without coupling correctness to periodic heartbeat configuration.
