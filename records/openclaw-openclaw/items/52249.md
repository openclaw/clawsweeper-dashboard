## Summary

Keep open: current main still has a source-level ACP wake gap when heartbeat delivery is disabled or unscheduled, and the newest native-subagent Telegram evidence shows the broader yielded-parent resumption problem is still current. The paired targeted PR remains open, and the broader delivery/resumption contract is already tied to related open work rather than fixed on main.

## What This Changes

ACP/native child completion should wake a yielded parent session promptly without requiring UI refresh or a later user message.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/60402 for the narrow heartbeat-disabled ACP wake gap, then settle the broader ACP/native yielded-parent completion contract through https://github.com/openclaw/openclaw/issues/44925 and https://github.com/openclaw/openclaw/pull/56442 or an equivalent maintained design.

## Reproduction Assessment

Yes, source-level. ACP `streamTo:"parent"` enqueues session-scoped system events and requests a heartbeat wake, while the runner can return `disabled` before targeted delivery when heartbeat is disabled or no heartbeat agents are scheduled; the latest Telegram trace corroborates a related native-subagent yielded-parent delay.

## Solution Assessment

No for current main: the heartbeat-coupled relay is not self-sufficient for this behavior. The narrow paired PR is the right first fix boundary, while the broader ACP/native completion delivery semantics need maintainer design review.
