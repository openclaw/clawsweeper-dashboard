## Summary

Keep open: current main and v2026.6.8 still lack one durable yielded-parent resume contract, with ACP completions depending on system-event plus heartbeat delivery and native completions still using separate active-run/direct-handoff paths.

## What This Changes

The issue asks OpenClaw to wake or resume a yielded parent session after ACP or native child completion without requiring manual refresh, poke, or delayed follow-up activity.

## Best Possible Solution

Adopt one maintainer-owned yielded-session completion contract that resumes the intended parent session across ACP, native subagent, and Codex-adapter paths without depending on optional heartbeat polling or direct child-text fallbacks.

## Reproduction Assessment

Yes at source level: current main still queues ACP completion through heartbeat wake and native completion through active-run/direct-handoff paths while sessions_yield leaves the parent inactive; no live current-main channel run was performed.

## Solution Assessment

No current implementation is complete; the best fix is a shared yielded-parent resume contract rather than another one-sided heartbeat, direct fallback, or adapter-specific repair.
