## Summary

Keep open: current main contains partial wake/delivery fixes, but the broader yielded-parent completion contract is still unresolved across ACP and native subagent paths, and two open PRs cover only narrower slices.

## What This Changes

Issue reports that parent sessions yielded for ACP or native subagent completion can remain wedged or delayed until external activity wakes them.

## Best Possible Solution

Adopt one maintainer-owned yielded-parent completion contract that resumes the intended requester across ACP, native subagent, and adapter paths without relying on optional heartbeat polling, final give-up, or later inbound user activity.

## Reproduction Assessment

Yes at source level: current main still queues ACP parent updates through in-memory system events plus heartbeat wake, while native successful subagent terminal task auto-delivery is explicitly suppressed and relies on announce/direct handoff paths. I did not run a live current-main ACP/native replay.

## Solution Assessment

No current path is complete yet. The best fix is a shared yielded-parent resume contract across ACP and native completion delivery, not another one-sided heartbeat, direct-fallback, or terminal-task slice.
