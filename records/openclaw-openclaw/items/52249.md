## Summary

Keep open: current main removed the stale direct-resume ACP relay, but yielded parent completion still relies on heartbeat/system-event or active-run steering paths that do not establish a durable resume contract for the reported ACP/native cases.

## What This Changes

The issue requests automatic wake/resume of a yielded parent session when ACP or native child completion arrives, without manual UI refresh or heartbeat-only delivery.

## Best Possible Solution

Adopt one maintainer-owned yielded-session completion contract that resumes the intended parent session across ACP, native subagent, and Codex-adapter paths without depending on optional heartbeat polling.

## Reproduction Assessment

Yes, at source level: current main still queues completion events through heartbeat wake or active-run steering while sessions_yield leaves the parent inactive; I did not run a live ACP/native channel repro.

## Solution Assessment

No current fix is complete: the Phase B yield-wait direction is plausible, but the best repair needs a shared paused-parent resume contract rather than a one-sided ACP heartbeat patch.
