## Summary

Keep open. Current main has moved past the original direct-resume relay theory, but the ACP `streamTo:"parent"` completion path still relies on session-scoped system events plus heartbeat wake delivery, and the code still has no independent delivery path when the heartbeat handler or heartbeat-enabled agent schedule is absent. The issue is also paired with open PR #60402, so this should remain open for that implementation review rather than being closed by cleanup.

## What This Changes

Reported ACP child completion leaves a yielded parent session stuck until manual refresh; later discussion narrows the active gap to heartbeat-disabled ACP completion and possibly native yielded child completion.

## Best Possible Solution

Add or land a shared yielded-parent completion delivery path that does not depend on optional heartbeat scheduling, then cover ACP `streamTo:"parent"` completion and any confirmed native child completion case with heartbeat-disabled and split-heartbeat regression tests.

## Reproduction Assessment

Yes for the narrowed ACP path: enqueueing ACP `streamTo:"parent"` completion with no heartbeat handler or with the parent agent outside the heartbeat schedule leaves only an in-memory system event plus a skipped wake. The native-child report is plausible from the discussion, but still needs its own focused reproduction.

## Solution Assessment

Unclear for the open paired PR as the complete solution: moving targeted wake handling earlier should help the ACP no-handler case, but the maintainable endpoint appears to be a shared yielded-parent completion primitive that also covers native child completion if that extension reproduces.
