## Summary

Keep open: current main has several partial delivery and wake fixes, but the broad yielded-parent wake contract remains unresolved across ACP task completion and native subagent completion paths, with open PRs still covering important slices.

## What This Changes

The issue reports that a parent session using `sessions_yield` can remain stuck instead of resuming when ACP or native child work completes.

## Best Possible Solution

Keep this issue as the canonical maintainer thread until maintainers choose and land a cross-runtime parent continuation contract that covers ACP task completion, native subagent completion, blocked/give-up outcomes, and heartbeat-disabled deployments.

## Reproduction Assessment

Yes, at source level: docs promise push-based `sessions_yield` completion, while current ACP stream routing remains heartbeat-gated and the requested explicit yield-wait path is absent. I did not run a live Matrix/Telegram reproduction in this read-only pass.

No, current main is not yet the complete solution; the heartbeat relay, task terminal events, and native active-requester steering are useful partial paths but do not form one owner-approved ACP/native yielded-parent resume contract.
