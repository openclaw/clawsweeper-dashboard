## Summary

Keep open: current main still routes ACP parent completion through system events plus heartbeat wake, and the wake path can still skip as disabled before targeted delivery. The same reporter has an open targeted PR for the ACP heartbeat-disabled gap, and the latest discussion adds related subagent evidence rather than showing the issue is fixed.

## What This Changes

The issue requests that yielded parent sessions resume automatically when ACP or native child sessions complete, without requiring manual refresh or a later user message.

## Best Possible Solution

Review, rebase, and land or replace https://github.com/openclaw/openclaw/pull/60402 for the narrow ACP heartbeat-disabled gap, then keep the broader yielded-parent completion contract aligned with https://github.com/openclaw/openclaw/issues/44925 or a maintained follow-up that covers native subagents too.

## Reproduction Assessment

Yes, source-level: ACP streamTo:"parent" completion still queues session events and requests heartbeat wake, while the wake path can skip as disabled before targeted handling when heartbeat is disabled or unscheduled. I did not run the live UI, Matrix, or Telegram flow.

## Solution Assessment

No for current main: heartbeat wake is still not a self-sufficient delivery path for yielded parent completion. The narrow paired PR is the right first review target, while the broader native subagent/yield lifecycle behavior needs maintainer design alignment.
