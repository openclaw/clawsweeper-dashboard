## Summary

Keep open: current main still source-reproduces the yielded-parent completion gap. The old direct-resume theory is stale, but ACP parent completion still depends on system-event plus heartbeat wake delivery, disabled heartbeat paths can still skip the wake, and related open PRs show the fix is active rather than shipped.

## What This Changes

The issue reports that yielded parent sessions can remain stuck after ACP or native child completion instead of resuming automatically.

## Best Possible Solution

Converge ACP and native child completion onto one maintained yielded-parent resumption contract that does not depend on optional heartbeat scheduling and only marks delivery complete after the parent turn or visible channel result has actually succeeded.

## Reproduction Assessment

Yes, source-level. Current ACP `streamTo:"parent"` enqueues system events and requests a heartbeat wake, while disabled or unregistered heartbeat paths can return before targeted delivery; recent comments add matching ACP and native production traces.

## Solution Assessment

No for current main. The heartbeat-coupled relay is not a complete yielded-parent completion contract; the safer direction is a dedicated, maintained resumption path that covers ACP and native completion without duplicating delivery semantics.
