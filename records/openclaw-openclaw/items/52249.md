## Summary

Keep open: current main has partial fixes and active candidate PRs, but the broad yielded-parent wake contract is still unresolved across ACP and native subagent completion paths.

## What This Changes

The issue asks OpenClaw to resume yielded parent sessions automatically when ACP or native child session completion arrives, without requiring refresh or a manual user poke.

## Best Possible Solution

Adopt one maintainer-owned yielded-parent completion contract that resumes the intended requester across ACP, native subagent, and adapter paths without relying on optional heartbeat polling, final give-up, or later inbound user activity.

## Reproduction Assessment

Yes at source level: current main still queues ACP parent updates through in-memory system events plus heartbeat wake, while native successful subagent terminal task auto-delivery is explicitly suppressed and relies on announce/direct handoff paths. Multiple issue comments report production reproductions across 2026.4.x through 2026.5.18.

## Solution Assessment

No current or proposed path is complete yet. The best fix is a shared yielded-parent resume contract across ACP and native completion delivery, not another one-sided heartbeat, direct-fallback, or terminal-task slice.
