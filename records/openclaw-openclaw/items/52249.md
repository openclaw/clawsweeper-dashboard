## Summary

Keep open: current main still routes ACP parent completion through system events plus heartbeat wake, and that wake path can skip when heartbeat delivery is unavailable or disabled.

## What This Changes

Request automatic resumption of yielded ACP parent sessions when child completion arrives, without requiring a manual UI refresh or heartbeat coupling.

## Best Possible Solution

Land or replace #60402 for the narrow ACP heartbeat-disabled wake gap, then converge ACP and native child completion on a maintained yielded-parent delivery primitive with focused regression coverage.

## Reproduction Assessment

Yes, at source level: ACP completion enqueues a parent-session event and schedules a heartbeat wake, while current heartbeat wake/runner code can skip with no handler, disabled heartbeats, or no enabled heartbeat agent. I did not rerun the live browser or Matrix repro in this read-only review.

## Solution Assessment

No for current main: heartbeat-coupled completion delivery is not self-sufficient when heartbeat delivery is disabled or unavailable. The narrower maintainable fix is a targeted parent wake path that works without periodic heartbeat, with a broader shared yielded-parent delivery primitive as the long-term endpoint.
