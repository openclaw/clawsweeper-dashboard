## Summary

Keep open: current main still routes ACP parent-stream completion through session-scoped system events plus heartbeat wake delivery, and the runner can return `disabled` before targeted session wake handling when heartbeat is off or unscheduled. The same-author targeted PR at https://github.com/openclaw/openclaw/pull/60402 remains open and directly addresses this narrow gap.

## What This Changes

ACP child completion should automatically wake a yielded parent session, including when heartbeat delivery is disabled or unavailable.

## Best Possible Solution

Review, rebase, and land or replace https://github.com/openclaw/openclaw/pull/60402 for the narrow heartbeat-disabled ACP wake gap, then handle the broader ACP/native yielded-parent notify contract through https://github.com/openclaw/openclaw/pull/56442 or an equivalent maintained design.

## Reproduction Assessment

Yes, source-level. Explicit ACP `streamTo:"parent"` relays enqueue parent-session events and request a heartbeat wake, while the runner can return `disabled` before targeted handling when heartbeat is disabled or no heartbeat agents are scheduled; I did not run the live UI or Matrix flow.

## Solution Assessment

No for current main: heartbeat wake is still not a self-sufficient delivery path for this case. The narrow paired PR is the right first fix boundary, while the broader notify/yield lifecycle design should stay a separate maintainer review path.
