## Summary

Keep open: current main and v2026.6.8 still lack the Ollama adapter `batchEmbed` hook, while the remaining question is whether Ollama's synchronous `/api/embed` array request should count as OpenClaw memory `remote.batch` behavior.

## What This Changes

The issue requests exposing a `batchEmbed` runtime method from the Ollama memory adapter so configured memory remote batch indexing can activate for Ollama.

## Best Possible Solution

Maintainers should decide and document whether Ollama inline array embedding belongs behind memory `remote.batch`, then either add a provider-owned bridge with focused coverage or clarify that Ollama remains inline-only.

## Reproduction Assessment

Yes. Source inspection reproduces the gap: memory batch mode requires `providerRuntime.batchEmbed`, and the current Ollama adapter runtime omits it; I did not run a live Ollama indexing scenario.

## Solution Assessment

Unclear. Adding `batchEmbed` is a plausible narrow implementation only after maintainers decide whether Ollama's synchronous `/api/embed` array batching belongs behind the memory `remote.batch` runtime contract.
