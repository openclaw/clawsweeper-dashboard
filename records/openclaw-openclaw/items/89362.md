## Summary

Keep open. Current main and v2026.6.8 still lack the Ollama adapter `batchEmbed` hook, but the documented `remote.batch` path currently describes provider batch-job APIs while Ollama already batches inline `/api/embed` array requests, so this needs maintainer product/API direction before implementation.

## What This Changes

Request to expose an Ollama memory adapter `batchEmbed` runtime bridge so configured memory batch indexing can use Ollama batched embedding support.

## Best Possible Solution

Maintainers should decide and document Ollama `remote.batch` semantics, then either clarify unsupported status or add a provider-owned `batchEmbed` bridge with focused coverage that preserves current inline safeguards.

## Reproduction Assessment

Yes. Source inspection shows the exact gap: memory batch mode requires `providerRuntime.batchEmbed`, and the current Ollama memory adapter does not provide it; I did not run a live Ollama indexing scenario in this read-only review.

## Solution Assessment

Unclear. Adding `batchEmbed` is a plausible narrow implementation only after maintainers decide whether Ollama's synchronous `/api/embed` array batching belongs behind the memory `remote.batch` runtime contract.
