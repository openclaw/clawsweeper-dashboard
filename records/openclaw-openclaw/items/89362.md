## Summary

Keep open: current main and v2026.6.9 still lack the Ollama adapter `batchEmbed` runtime hook, but this is a product/API semantics decision because docs currently reserve `remote.batch` for provider batch APIs while Ollama already uses synchronous `/api/embed` array requests through the inline embedding path.

## What This Changes

Requests that the Ollama memory embedding adapter expose a `batchEmbed` runtime hook so existing remote batch indexing config can activate for Ollama-backed memory.

## Best Possible Solution

Maintainers should decide and document whether Ollama inline array embedding is supported `remote.batch` behavior, then either add a provider-owned adapter hook with focused tests/docs or keep the docs explicit that Ollama remains inline-only.

## Reproduction Assessment

Yes. Source inspection reproduces the gap: memory batch mode requires `providerRuntime.batchEmbed`, current Ollama runtime omits it, and the Ollama provider has only inline `embedBatch` today.

## Solution Assessment

Unclear. Adding `batchEmbed` is a plausible narrow implementation only after maintainers decide whether Ollama's synchronous `/api/embed` array batching belongs behind the memory `remote.batch` runtime contract.
