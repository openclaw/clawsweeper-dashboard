## Summary

Keep open. Current main still parses Feishu/Lark DM root/thread metadata but routes direct messages through the sender Open ID and chat-scoped lanes; open PR #63148 remains the focused implementation candidate, while #73972/#38806 cover only the narrower P2P thread-reply bug.

## What This Changes

Add Feishu/Lark DM topic/thread-aware session isolation so different topics from the same direct-message sender can use separate sessions.

## Best Possible Solution

Land a Feishu-plugin change that gives topic-bearing DMs stable topic-specific conversation ids and queue keys while preserving existing flat sender sessions for non-topic DMs, with focused tests, docs, and changelog coverage.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: two Feishu p2p events from the same sender/chat with different root_id/thread_id values parse those ids but still route through peer kind direct with id ctx.senderOpenId and the same chat-scoped queue key.

## Solution Assessment

Unclear until maintainer review of #63148. The fix belongs in the Feishu plugin, but maintainers still need to choose the requested opt-in config shape versus the PR’s default-on behavior for topic-bearing DMs.
