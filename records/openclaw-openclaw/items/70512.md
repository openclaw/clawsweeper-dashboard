## Summary

Keep open. Current main still has the Feishu DM topic-session gap: inbound root/thread metadata is parsed, but only group messages use topic-aware session resolution, while DMs still route by sender and share a chat-wide queue key. Open PR #63148 is the focused implementation candidate for the same remaining work; #73972 is narrower reply-in-thread handling and does not cover full session isolation.

## What This Changes

Add Feishu direct-message session isolation by topic/thread ID so separate DM topics from the same user keep independent OpenClaw sessions.

## Best Possible Solution

Review and land #63148 or an equivalent Feishu-plugin patch that derives stable topic-specific DM conversation IDs and queue keys from root_id/thread_id, keeps non-topic DMs on the existing sender-level behavior, preserves in-thread replies, and updates tests, docs, and changelog for the chosen default-vs-config policy.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to construct two Feishu p2p/private events from the same sender and chat with different root_id/thread_id values; current main parses those ids but routes both through peer kind direct with id ctx.senderOpenId and the same chatId-based sequential key.

## Solution Assessment

Unclear until maintainer review. The narrow maintainable fix belongs in the Feishu plugin and #63148 covers the likely runtime/test boundary, but maintainers still need to decide whether the behavior should be default-on or configurable.
