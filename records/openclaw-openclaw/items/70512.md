## Summary

Keep open. Current main still parses Feishu/Lark DM root and thread metadata, but direct-message session routing, thread-reply flags, debounce, and sequential queue keys remain flat; the open Feishu PR is the right implementation candidate rather than proof that the issue is fixed.

## What This Changes

The issue requests topic/thread-based session isolation for Feishu direct messages, optionally through a new DM session-scope config.

## Best Possible Solution

Land or replace a Feishu plugin change that gives topic-bearing DMs stable topic-specific conversation IDs, parent routing, thread replies, per-topic queue/debounce keys, focused tests, docs, and changelog coverage while preserving flat sessions for non-topic DMs.

## Reproduction Assessment

Yes, source-level. Two Feishu p2p events from the same sender/chat with different root_id or thread_id values parse those IDs, but current main still routes both through ctx.senderOpenId and the same chat-scoped dispatch key.

## Solution Assessment

Unclear until maintainer review of https://github.com/openclaw/openclaw/pull/63148. The fix belongs in the Feishu plugin, but maintainers still need to settle opt-in config versus default-on isolation and require real Feishu proof before merge.
