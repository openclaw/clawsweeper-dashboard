## Summary

Keep open. Current main still parses Feishu/Lark DM root/thread metadata but direct-message routing, reply-thread flags, debounce, and sequential queue keys remain flat for the sender/chat; https://github.com/openclaw/openclaw/pull/63148 is the focused implementation candidate rather than a reason to close before merge.

## What This Changes

Support Feishu/Lark direct-message topic/thread session isolation, potentially via a DM session-scope config, so separate DM topics do not share one agent session.

## Best Possible Solution

Land or replace a Feishu plugin change that gives topic-bearing DMs stable topic-specific conversation IDs, parent routing, thread replies, per-topic queue/debounce keys, focused tests, docs, and changelog coverage while preserving flat sessions for non-topic DMs.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: two Feishu p2p events from the same sender/chat with different root_id or thread_id values parse those IDs, but current main still routes both through direct peer id ctx.senderOpenId and the same chat-scoped sequential key.

## Solution Assessment

Unclear until maintainer review of https://github.com/openclaw/openclaw/pull/63148. The fix belongs in the Feishu plugin, but maintainers still need to choose opt-in config versus default-on isolation for topic-bearing DMs and require real Feishu proof before merge.
