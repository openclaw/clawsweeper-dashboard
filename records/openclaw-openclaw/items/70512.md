## Summary

Keep open. Current main and the latest shipped tag still keep Feishu/Lark DM routing flat by sender even when inbound events carry root/thread metadata, and the active implementation PR is a review/proof candidate rather than a landed fix.

## What This Changes

Request Feishu/Lark DMs to support topic/thread-scoped session isolation, likely with a DM session-scope option, so separate DM topics do not share context.

## Best Possible Solution

Land or replace a focused Feishu plugin change that gives topic-bearing DMs stable topic-specific conversation IDs, reply/queue/debounce behavior, tests, and docs while preserving flat main-surface DMs.

## Reproduction Assessment

Yes, source-level. Current main parses p2p root_id/thread_id into FeishuMessageContext, but the direct path still resolves peerId to ctx.senderOpenId and thread reply flags to false.

## Solution Assessment

Unclear until maintainer review. The Feishu plugin is the right surface, but maintainers still need to choose opt-in config versus default-on topic-bearing DM isolation and review https://github.com/openclaw/openclaw/pull/63148 with real Feishu proof.
