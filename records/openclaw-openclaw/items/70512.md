## Summary

Keep open. Current main still has the reported Feishu DM topic gap: thread/root metadata is parsed, but direct messages route by sender Open ID and flat chat queue keys, while topic-aware session routing remains group-scoped. The active related PR #63148 appears to cover the same remaining work, so this should stay open for maintainer review of that patch or an equivalent Feishu-plugin implementation.

## What This Changes

Add Feishu DM session isolation by topic/thread ID so different direct-message topics from the same user route to separate sessions.

## Best Possible Solution

Review and land #63148 or an equivalent Feishu-plugin patch that scopes only topic-bearing p2p events to stable topic-specific conversation IDs, keeps main-surface DM behavior unchanged, makes replies and sequential/debounce keys thread-aware, and updates docs/config if maintainers choose an opt-in setting.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to construct two Feishu p2p events from the same sender with different root_id/thread_id values; current main parses the metadata but resolves both through peer kind direct with id ctx.senderOpenId and the same chatId-based sequential key.

## Solution Assessment

Unclear until the active PR is reviewed. The narrow maintainable solution belongs in the bundled Feishu plugin, but maintainers should choose between the issue's opt-in config proposal and #63148's default-on behavior for topic-bearing DMs.
