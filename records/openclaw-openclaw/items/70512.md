## Summary

Keep open. Current main still parses Feishu/Lark DM root/thread metadata but routes direct messages through the sender Open ID and chat-scoped queue key; #63148 remains the focused open implementation candidate, while #73972/#38806 only cover the narrower reply-inside-thread path.

## What This Changes

Add Feishu/Lark DM topic/thread-aware session isolation so different topics from the same direct-message sender can use separate sessions.

## Best Possible Solution

Land a Feishu-plugin change that gives topic-bearing DMs stable topic-specific conversation ids and queue keys while preserving existing flat sender sessions for non-topic DMs, with tests, docs, and changelog coverage.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: two Feishu p2p events from the same sender/chat with different root_id/thread_id values currently parse those ids but still route through peer kind direct with id ctx.senderOpenId and the same chat-scoped sequential key.

## Solution Assessment

Unclear until maintainer review of #63148. The fix belongs in the Feishu plugin, but maintainers still need to choose the requested opt-in config shape versus the PR’s default-on behavior for topic-bearing DMs.
