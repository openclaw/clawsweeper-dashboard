## Summary

Keep open. Current main still parses Feishu/Lark DM root/thread metadata but direct messages route through the sender Open ID and chat-scoped queue/debounce keys, so different DM topics from the same sender still share one session; open PR #63148 remains the focused implementation candidate, while #73972/#38806 cover only the narrower reply-inside-thread path.

## What This Changes

The issue requests Feishu direct-message topic/thread session isolation so different DM topics from the same user can use independent OpenClaw sessions.

## Best Possible Solution

Land a Feishu plugin change that gives topic-bearing DMs stable topic-specific conversation IDs, parent routing, reply-in-thread behavior, and per-topic queue/debounce keys while preserving flat sessions for non-topic DMs, with focused tests, docs, and changelog coverage.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: two Feishu p2p events from the same sender/chat with different root_id or thread_id values parse those IDs but still route through peer kind direct with id ctx.senderOpenId and the same chat-scoped sequential key.

## Solution Assessment

Unclear until maintainer review of #63148. The fix belongs in the Feishu plugin, but maintainers still need to choose the requested opt-in config shape versus default-on isolation for topic-bearing DMs.
