## Summary

Keep open. Current main still preserves Feishu/Lark direct-message root/thread metadata but routes direct messages through the sender Open ID, with no DM topic session key, no DM topic queue key, and no DM session-scope config or docs. The remaining work is real and already has a focused open implementation candidate in #63148, while #73972 is only the narrower P2P thread-reply path.

## What This Changes

The issue requests Feishu/Lark direct-message topic/thread session isolation so separate DM topics from the same user create separate OpenClaw sessions instead of sharing the sender-level session.

## Best Possible Solution

Review and land #63148 or an equivalent Feishu-plugin patch that gives topic-bearing DMs stable topic-specific conversation IDs and queue keys, keeps non-topic DMs on the existing sender-level session, makes replies stay in the DM thread, and updates tests/docs/changelog for the chosen default-vs-config policy.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: construct two Feishu p2p events from the same sender/chat with different root_id/thread_id values; current main parses those ids but still routes both through peer kind direct with id ctx.senderOpenId and an ordinary chat-scoped sequential key.

## Solution Assessment

Unclear until maintainer review of #63148. The narrow maintainable fix belongs in the Feishu plugin, but the compatibility decision is whether to adopt default-on topic isolation for topic-bearing DMs or add the explicit opt-in config requested in this issue.
