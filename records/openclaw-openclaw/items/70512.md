## Summary

Keep open. Current main still carries Feishu/Lark DM root/thread metadata but routes direct messages through the flat sender session and chat-scoped queue; open PR #63148 remains the focused implementation candidate for the remaining work.

## What This Changes

Request Feishu/Lark DMs with topic/thread metadata to route to isolated per-topic sessions, optionally via a DM sessionScope config, instead of sharing one sender session.

## Best Possible Solution

Land a Feishu plugin change that gives topic-bearing DMs stable topic-specific conversation ids and dispatch keys while preserving existing flat sender sessions for non-topic DMs, with focused tests, docs, and changelog coverage.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: two Feishu p2p events from the same sender/chat with different root_id or thread_id values are parsed, but current main still routes both through ctx.senderOpenId and the same chat-scoped sequential key.

## Solution Assessment

Unclear until maintainer review of #63148. The fix belongs in the Feishu plugin, but the requested opt-in config shape versus default topic-aware behavior remains a product decision.
