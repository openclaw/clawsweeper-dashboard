## Summary

Keep open: current main still lacks word timing metadata and desktop/web highlight rendering, and changing generated TTS into text-then-audio follow-up delivery would alter shared channel media semantics. The related PR at https://github.com/openclaw/openclaw/pull/80646 is open, partial, and review-blocked, so there is no merged or shipped implementation to close against.

## What This Changes

Request to add word-level synchronized TTS highlighting in desktop/web UI and send generated TTS audio as a separate follow-up message after text.

## Best Possible Solution

Design this as two explicit changes: an optional timestamp-bearing TTS provider/result contract propagated into desktop/web chat highlighting, and a separate channel/product decision for text-then-audio delivery ordering.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection on current main confirms the requested timestamp propagation, highlight rendering, and separate TTS follow-up delivery are not implemented today.

## Solution Assessment

Unclear as an immediate implementation plan: optional timing metadata is a plausible SDK/provider extension, but globally changing TTS delivery ordering needs maintainer product/API design. The current related PR is not the best complete solution because it is partial, mixed with unrelated memory work, and review-blocked.
