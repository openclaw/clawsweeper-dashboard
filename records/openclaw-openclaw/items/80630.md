## What This Changes

Request to add TTS word-level playback highlighting in desktop/web UI and send generated TTS audio as a separate follow-up message instead of bundling it with text.

## Best Possible Solution

Design an optional timestamp-bearing TTS result contract, propagate it through transcript/webchat state with graceful fallback highlighting, and coordinate separate-delivery policy with the canonical TTS delivery-mode work.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives high confidence that current main lacks timestamp propagation and highlight rendering, and that separate-delivery behavior is only partial and policy-dependent.

## Solution Assessment
