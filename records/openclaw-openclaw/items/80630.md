## Summary

Keep open: current `main` still lacks timestamp-bearing TTS metadata and desktop/web word highlighting, and the requested text-then-audio delivery order would change channel delivery semantics rather than close as conservative cleanup.

## What This Changes

The issue requests karaoke-style word highlighting for desktop/web TTS playback and TTS audio delivery as a separate follow-up message after the text.

## Best Possible Solution

Split the work into an optional timestamp-bearing speech result contract with desktop/web fallback highlighting, and a separate maintainer-approved channel delivery design for text-first TTS audio ordering.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source and test inspection on current `main` confirms the requested timestamp sync and text-first TTS delivery behavior are not implemented end to end.

## Solution Assessment

Unclear as a direct cleanup fix: carrying optional timing metadata is plausible, but changing TTS delivery order across surfaces needs explicit product/API design because current Telegram and shared media paths intentionally use caption/media sequencing.
