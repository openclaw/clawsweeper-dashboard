## Summary

Keep open. Current main still contains the direct no-real-conversation skip and positive-only compaction token accounting that can leave zero-token or fallback compaction outcomes from refreshing session state; a partial branch-message fallback exists, but the related implementation candidate is still open rather than merged or shipped.

## What This Changes

The issue asks OpenClaw compaction to summarize real persisted conversation history instead of silently skipping when prepared or session message arrays arrive empty.

## Best Possible Solution

Keep the guard for truly empty compaction input, but make preparation, direct compaction, and token bookkeeping agree so real persisted turns are summarized and zero-token fallback outcomes clear stale trigger state.

## Reproduction Assessment

Yes, there is a high-confidence source-backed reproduction path: the comments provide multiple concrete runtime repros, and current main still has the direct empty-content skip plus positive-only accounting paths. I did not run a live current-main multi-channel session in this read-only review.

## Solution Assessment

No to the report's workaround of deleting the guards, because that can send empty compaction input to providers. The best fix is the narrower invariant/accounting repair represented by https://github.com/openclaw/openclaw/pull/69270 or an equivalent maintainer-owned patch.
