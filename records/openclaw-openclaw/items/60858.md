## Summary

Keep open: current main still contains the empty-content compaction skip and fallback-boundary paths that match the runtime reports, while the related fix PRs remain open and conflicting rather than shipped.

## What This Changes

The issue asks OpenClaw to stop compaction from silently skipping or writing empty summaries when real session history exists but the compaction inputs arrive empty or filtered out.

## Best Possible Solution

Keep the true-empty-session guard, but make manual/direct compaction, safeguard preparation, branch fallback, and token bookkeeping agree on the persisted transcript so real history summarizes and zero-token outcomes clear stale trigger state.

## Reproduction Assessment

Yes, source-backed but not live in this review: the comments provide multiple concrete runtime reproductions, and current main still contains the direct empty-content skip, safeguard fallback boundary, and positive-only accounting paths.

## Solution Assessment

No to the report's workaround of deleting all guards, because genuinely empty sessions still need to avoid provider dispatch. The maintainable fix is an invariant/accounting repair, a threshold-aware outer-skip bypass, or a coordinated replacement that preserves the empty-session protection.
