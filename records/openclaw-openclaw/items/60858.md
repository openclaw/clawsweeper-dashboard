## Summary

Keep open. Multiple runtime reports still match a source-backed compaction invariant problem, and current main only has partial mitigations while the broader zero-token/session-invariant repair remains in an open PR rather than merged or shipped.

## What This Changes

The issue reports that compaction skips or writes placeholder summaries for real sessions because empty/unpopulated message arrays trip `hasRealConversationContent` guards.

## Best Possible Solution

Keep the guard for truly empty compaction input, but make preparation, direct compaction, and token bookkeeping agree on the persisted transcript so real turns are summarized and zero-token fallback outcomes clear stale trigger state.

## Reproduction Assessment

Yes, source-backed but not live: the comments provide multiple concrete runtime repros, and current main still has the direct empty-content skip plus positive-only accounting paths. I did not run a live current-main multi-channel session in this read-only review.

## Solution Assessment

No to the report's workaround of deleting the guards, because that can send empty compaction input to providers. The narrow maintainable solution is the invariant/accounting repair represented by https://github.com/openclaw/openclaw/pull/69270 or an equivalent maintainer-owned patch after proof.
