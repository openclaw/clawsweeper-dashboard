## Summary

Keep open: the metadata-preservation fix is still useful and not implemented on current main, but this PR head is not merge-ready because it disables default routed-final transcript mirroring when no source-reply mirror metadata exists and still has only test/autoreview proof.

## What This Changes

The PR widens routed reply and outbound transcript mirror metadata so `expectedSessionId`, `storePath`, and `deliveryMirror` can flow through dispatch, `routeReply`, and outbound transcript append, with focused regression tests.

## Best Possible Solution

Keep the richer metadata forwarding, leave `mirror` undefined for no-source-metadata routed finals, reserve `false` for transcript-owned replies, add default-path regression coverage, and require redacted real setup proof before merge.

## Solution Assessment
