## Summary

Keep this PR open: it has a useful routed transcript-metadata fix, but the current branch still regresses the ordinary routed-final path by passing `mirror: false` when no source metadata exists, which can stop successful routed replies from appending assistant turns to the owning transcript.

## What This Changes

This PR changes routed reply and outbound delivery mirror plumbing so `sourceReplyTranscriptMirror` metadata can reach transcript append, with focused regression tests.

## Best Possible Solution

Keep the richer metadata forwarding, leave `mirror` undefined for no-source-metadata routed finals, use `false` only for transcript-owned replies, add default-path regression coverage, and require redacted real setup proof before merge.

## Reproduction Assessment

Yes, at source level: PR head passes `mirror: false` when source metadata is absent, while current main mirrors routed replies by default whenever a session key exists. I did not run a live channel reproduction in this read-only review.

## Solution Assessment

No as written. Extending richer mirror metadata through `routeReply` and outbound delivery is the right layer, but dispatch must preserve the ordinary default mirroring path.
