## Summary

Keep open: the metadata-preservation direction is useful, but the current patch still converts routed finals with no sourceReplyTranscriptMirror into `mirror: false`, which overrides routeReply's existing default transcript mirror and can drop delivered assistant turns from session history. The external PR also still has only targeted tests/autoreview output, not real behavior proof from a channel/transcript setup.

## What This Changes

The PR widens routed reply transcript mirror metadata through dispatch, routeReply, outbound delivery, and focused regression tests.

## Best Possible Solution

Keep the rich metadata forwarding, but leave routeReply's mirror option undefined when no explicit source-reply mirror exists, keep `false` only for transcript-owned finals, add the missing default-path regression test, and then provide redacted real setup proof.

## Reproduction Assessment

Yes at source level: ownerless routed finals with a sessionKey and no sourceReplyTranscriptMirror would pass `mirror: false`, while current routeReply defaults to transcript mirroring when the mirror option is undefined. I did not run a live channel reproduction in this read-only review.

## Solution Assessment

No as written. Extending rich mirror metadata through routeReply and outbound delivery is the right layer, but dispatch must not turn the ordinary no-source-metadata path into an explicit mirror opt-out.
