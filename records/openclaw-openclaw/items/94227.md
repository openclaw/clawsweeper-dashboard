## Label Justifications

## Summary

Keep open: the branch fixes a real metadata-preservation gap, but it also disables the existing default transcript mirror for ordinary routed final replies without source-reply metadata, which would make those cross-surface replies stop appending assistant turns.

## What This Changes

The PR widens routed/outbound transcript mirror metadata so expectedSessionId, storePath, and deliveryMirror can flow through dispatch, routeReply, durable delivery, and regression tests.

## Best Possible Solution

Keep the metadata-preservation direction, but pass mirror:false only for runtime-owned finals, pass the explicit source-reply mirror object when present, and otherwise leave mirror undefined so routeReply keeps its default session transcript mirror.

## Reproduction Assessment

Yes, at source level: an ownerless routed final reply with a sessionKey and no sourceReplyTranscriptMirror now passes mirror:false. Current routeReply defaults to mirroring in exactly that undefined-mirror case.

## Solution Assessment

No, not as written. Extending mirror metadata is the right layer, but the dispatch change must preserve routeReply's existing default mirror for ordinary routed finals before this is safe.
