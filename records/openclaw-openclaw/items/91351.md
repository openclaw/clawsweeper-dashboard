## Summary

Keep open: the provider-local approach is appropriate and the supplied proof is strong, but the current ranges charge exactly 256,000 input tokens at the high rate even though the official OpenCode pricing table places that value in the low `≤256K` tier.

## What This Changes

The PR adds two-tier pricing metadata for OpenCode Go’s `qwen3.7-plus` model and adds catalog-resolution and high-context cost-estimator coverage.

## Best Possible Solution

Represent the official inclusive low tier using ranges compatible with the estimator’s half-open integer convention, then cover exactly 256,000 and 256,001 input tokens in focused tests.

## Reproduction Assessment

Yes. With this PR’s ranges and the current half-open selector, exactly 256,000 input tokens deterministically select the high tier despite the official `≤256K` low-tier contract.

## Solution Assessment

No as currently written. Provider-local tier metadata feeding the shared estimator is the narrowest maintainable solution, but the transition must honor the estimator’s half-open range semantics.
