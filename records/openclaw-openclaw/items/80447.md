## Summary

This PR should stay open for normal maintainer review: current main still invokes the Codex app-server during onboarding migration detection, and the branch makes a focused code/test change to avoid that pre-import side effect. I found no blocking correctness issue in the proposed patch.

## What This Changes

The PR makes Codex migration provider detection skip app-server-backed plugin inventory and adds regression tests that detection does not call the app-server boundary.

## Best Possible Solution

Land the focused PR after maintainer review, keeping onboarding detection filesystem-only while preserving app-server-backed inventory for explicit Codex migration flows.

## Solution Assessment

Yes. The option on discoverCodexSource is the narrowest maintainable fix because only lightweight detection disables app-server plugin inventory, while explicit migration planning keeps the existing full inventory behavior.
