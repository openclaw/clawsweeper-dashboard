## Summary

Keep open: current main and v2026.6.6 have generic Talk gateway-relay plumbing plus a legacy Azure deployment bridge, but they do not prove first-class Azure Foundry GA GPT Realtime Talk with server-side Azure auth, diagnostics, docs, and live proof. A maintainer also explicitly left the feature open because Azure model/quota availability blocked live verification.

## What This Changes

Request first-class Azure Foundry / Azure OpenAI GPT Realtime Talk through Gateway relay with server-side Azure credentials, validation, diagnostics, tests, and docs.

## Best Possible Solution

Implement Azure Foundry GA realtime in the bundled OpenAI realtime Gateway relay with server-side Azure API-key and Entra handling, Azure-specific validation/doctor diagnostics, public docs, regression tests, and live Azure proof when quota is available.
