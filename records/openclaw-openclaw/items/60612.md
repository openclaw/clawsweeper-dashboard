## Summary

Keep open. Current main and the latest release still intentionally warn on version-manager Gateway runtimes, current main has no durable `--runtime-path` or acknowledgement surface, and the related runtime-path PR is still open, conflicting, and maintainer-labeled rather than merged.

## What This Changes

The issue requests a permanent way to pin the Gateway service Node executable or suppress/acknowledge doctor warnings when NVM, ASDF, or fnm is intentionally used.

## Best Possible Solution

Adopt a clear daemon runtime policy: either land a validated explicit runtime-path pin with documented audit suppression for acknowledged pins, or document that version-manager Gateway runtimes always warn and require supported system Node for clean doctor output.

## Reproduction Assessment

Yes, source-level. A Gateway service using an NVM, ASDF, fnm, Volta, mise, or similar Node path still triggers the warning path when no supported system Node is available, and current main has no durable pin or acknowledgement option.

## Solution Assessment

Unclear. The current system-Node warning is documented and coherent, but the least risky support path would be an explicit validated runtime pin rather than blanket suppression.
