## Summary

Keep open. Current main and the latest release still warn on version-managed Gateway Node paths, only migrate when a supported system Node exists, and do not expose a durable Node-path pin or warning acknowledgement; the ASDF and fnm follow-up comments show this is a broader runtime-policy request rather than an NVM-only support case.

## What This Changes

Request a durable way to pin the Gateway service Node path or suppress/acknowledge doctor warnings when a version-managed Node runtime is intentionally used.

## Best Possible Solution

Keep the system-Node migration default, then add a documented durable Node-path pin or explicit acknowledgement only if maintainers decide intentional version-manager Gateway runtimes are supported.

## Reproduction Assessment

Yes, source-level. A Gateway service whose Node executable is under NVM, ASDF, fnm, Volta, or a similar version manager while no supported system Node exists will still warn and fall back to that runtime, with no durable pin or acknowledgement option exposed.

## Solution Assessment

Unclear. The existing system-Node preference is coherent and documented, but adding a durable pin or warning acknowledgement is a maintainer product/config-policy decision rather than a narrow bug fix.
