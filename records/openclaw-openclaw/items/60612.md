## Summary

Keep open. Current main partially improved reinstall behavior when a supported system Node exists, but it still warns for intentional NVM/ASDF-style Gateway runtimes and does not expose a durable public node-path selector or scoped acknowledgement.

## What This Changes

The issue asks for a durable way to pin or acknowledge a version-manager Node path for Gateway services without recurring doctor warnings.

## Best Possible Solution

Define a documented Gateway service runtime policy that keeps the warning by default, migrates to supported system Node when available, and gives intentional users a durable explicit pin or scoped acknowledgement across common version managers.

## Reproduction Assessment

Yes. A Gateway service command whose first program argument is under `.nvm` or `.asdf` reproduces the warning in current source, and the runtime-path test covers the no-supported-system-Node fallback path.

## Solution Assessment

No current solution is complete. The recent system-Node preference fix addresses reinstall loops when a system Node exists, but this request still needs an explicit product decision for intentional version-manager use.
