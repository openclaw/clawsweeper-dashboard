## Summary

Keep open: current main and v2026.6.10 still intentionally warn on version-manager Node daemon paths and do not expose a supported exact runtime pin or warning acknowledgement path. The closest implementation path is still the open, conflicting, proof-needed https://github.com/openclaw/openclaw/pull/82290.

## What This Changes

Request a supported way to pin or acknowledge the Gateway daemon Node runtime when doctor warns about version-manager Node paths such as nvm, asdf, or fnm.

## Best Possible Solution

Maintainers should decide the daemon runtime policy, then land or replace a validated explicit runtime-pin or acknowledgement path with live supervisor proof, or document that supported system Node is the only way to clear this warning.

## Reproduction Assessment

Yes. Source inspection shows that a Gateway service command using an nvm/fnm/asdf/mise-style Node path still triggers the version-manager warning, while current CLI/docs expose no supported exact runtime pin or acknowledgement path.

## Solution Assessment

Unclear. Explicit validated runtime pinning is plausible, but warning suppression alone is likely the wrong layer because current policy treats version-manager daemon runtimes as fragile.
