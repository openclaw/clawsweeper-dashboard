## Summary

Keep open. Current main still warns on Gateway services whose direct runtime is an NVM/ASDF-style Node path and only offers automatic repair when a supported system Node is available; recent PATH cleanup work did not add a durable Node pin or warning acknowledgement. https://github.com/openclaw/openclaw/issues/60612

## What This Changes

The issue asks for a durable way to select or acknowledge an intentional version-manager Gateway Node runtime so doctor stops regenerating an unfixable warning.

## Best Possible Solution

Keep the diagnostic by default, continue migrating to supported system Node when available, and add a documented durable escape hatch for explicit Node pinning or scoped acknowledgement across NVM, ASDF, FNM, Volta, and similar managers.

## Reproduction Assessment

Yes. A source-level reproduction is a Gateway service command whose first program argument is under `~/.nvm` or `~/.asdf`; current code and tests show that path emits the version-manager warning, and the doctor repair path has no migration target without supported system Node.

## Solution Assessment

No current solution is complete. Manual plist edits are overwritten by design, and wrapper support can persist an external launcher, but the maintainable fix should be an explicit runtime pin or acknowledgement rather than an undocumented bypass.
