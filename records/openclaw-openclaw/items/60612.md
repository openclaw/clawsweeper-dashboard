## Summary

Keep open. Current main and v2026.5.12 still intentionally warn on version-manager Gateway Node paths, migrate only when a supported system Node exists, and expose no durable Node pin or warning acknowledgement; the ASDF and fnm follow-up comments show this is a broader runtime-policy gap rather than an NVM-only support question.

## What This Changes

Request to make the Gateway doctor warning for intentional version-manager Node runtimes permanently fixable, suppressible, or clearly documented as safe to ignore.

## Best Possible Solution

Keep the system-Node migration default, then add a documented durable Node pin or explicit acknowledgement only if maintainers decide intentional version-manager Gateway runtimes are a supported policy.

## Reproduction Assessment

Yes, source-level. A Gateway service whose Node executable is under NVM, ASDF, fnm, Volta, or similar while no supported system Node exists will still warn and fall back to that runtime, and no durable pin or suppression option is exposed.

## Solution Assessment

Unclear. The existing system-Node preference is documented and coherent, but adding a durable pin or warning acknowledgement is a product/config-policy decision rather than a narrow bug fix.
