## Summary

Keep open: current main still lacks the requested passive `context=X%` Runtime-line field, and the recent issue discussion distinguishes existing `/status`, `/context`, post-compaction refresh, and workspace-sync mitigation from this specific prompt-surface feature.

## What This Changes

Request adding a passive context-window usage percentage such as `context=45%` to the embedded-agent system-prompt Runtime line.

## Best Possible Solution

If accepted, plumb a bounded context usage percentage from fresh session token/window metadata into embedded-runner params, `RuntimeInfoInput`, and `buildRuntimeLine()` with focused coverage; otherwise document that explicit context commands and recovery plugins are the intended boundary.

## Reproduction Assessment

Yes for the missing requested surface: source inspection on current main shows no carrier or formatter for a Runtime-line context percentage. The broader real-world compaction/reset state-loss scenario was not live replayed.

## Solution Assessment

Unclear until maintainers decide the product direction. If they accept an always-visible Runtime field, using fresh session usage/window metadata and omitting stale or unavailable values is the narrowest maintainable path.
