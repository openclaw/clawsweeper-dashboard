## Summary

Keep open: current main still lacks the requested passive `context=X%` field in the system-prompt Runtime line. Existing `/status`, `/context`, token counters, plugin prompt hooks, and the linked `openclaw-workspace-sync` community plugin are useful adjacent surfaces, but they do not implement this per-turn agent-visible prompt primitive.

## What This Changes

Request to add passive `context=X%` metadata to the Runtime line so agents can checkpoint before compaction or session reset.

## Best Possible Solution

Implement a bounded, omitted-when-unavailable Runtime context percentage derived from reliable embedded-runner session token/window data, with focused prompt/runtime tests and docs noting it complements existing `/status`, `/context`, and sync/recovery plugins.

## Reproduction Assessment

Yes for the missing requested surface: current source inspection shows no `context=` carrier in `RuntimeInfoInput`, `buildRuntimeLine()`, or embedded runtime assembly. The real-world compaction loss was not live-replayed.

## Solution Assessment

Yes at the direction level: a bounded, omitted-when-unavailable `context=X%` Runtime field is the narrowest maintainable way to satisfy this request. The safer implementation detail is to derive it from fresh session prompt/context-window metadata and keep CLI runtimes excluded unless they gain reliable tracking.
