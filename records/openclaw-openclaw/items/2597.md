## Summary

Keep open: current main still lacks the requested passive `context=X%` field in the system-prompt Runtime line, and the latest issue discussion treats the linked workspace-sync plugin as a recovery mitigation rather than this prompt/runtime feature.

## What This Changes

Request to expose context-window usage as `context=X%` in the per-turn Runtime line so embedded agents can checkpoint state before compaction or reset.

## Best Possible Solution

Implement an omitted-when-unavailable `context=X%` Runtime field for embedded runs, derived from reliable session token/window metadata and covered by focused prompt and runner tests plus a short docs note.

## Reproduction Assessment

Yes for the missing requested surface: current source inspection shows no `context=` carrier in `RuntimeInfoInput`, `buildRuntimeLine()`, or embedded runtime assembly. The reported real-world compaction/reset state loss was not live-replayed.

## Solution Assessment

Unclear until maintainers decide the product direction. If accepted, a bounded, omitted-when-unavailable Runtime field is the narrowest maintainable implementation; otherwise the intended boundary should be documented around explicit status commands and recovery plugins.
