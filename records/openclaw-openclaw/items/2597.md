## Summary

Keep open: current main still lacks the requested passive `context=X%` Runtime-line field, and the latest issue discussion treats existing status/recovery paths as mitigations rather than a fix for this prompt primitive.

## What This Changes

The issue requests an agent-visible context usage percentage in the injected Runtime line so embedded agents can checkpoint before compaction or reset.

## Best Possible Solution

If maintainers accept the product behavior, plumb an omitted-when-unavailable bounded context percentage from reliable session token/window metadata into embedded Runtime prompt params and `buildRuntimeLine()` with focused prompt, runner, and docs coverage; otherwise document the status/recovery/plugin boundary explicitly.

## Reproduction Assessment

Yes for the missing requested surface: source inspection on current main shows no `contextPercent` carrier or `context=` Runtime-line segment in the prompt or embedded-runner path. The broader real-world compaction/reset state-loss scenario was not live replayed.

## Solution Assessment

Unclear until maintainers decide the product direction. If accepted, bounded omitted-when-unavailable runtime metadata plumbing is the narrowest maintainable path; workspace sync remains a recovery plugin rather than this prompt primitive.
