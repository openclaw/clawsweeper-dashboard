## Summary

Keep open: current main still lacks the requested passive `context=X%` Runtime-line field. Existing `/status`, `/context`, `/usage tokens`, `session_status`, and the linked `openclaw-workspace-sync` community plugin are useful inspection or recovery mitigations, but they do not provide this per-turn prompt primitive.

## What This Changes

The issue requests adding a passive context usage percentage to the system-prompt Runtime line for embedded agent runs.

## Best Possible Solution

If accepted, plumb an omitted-when-unavailable bounded context usage percentage from reliable fresh session token/window metadata into embedded Runtime prompt params with focused prompt and runner coverage; otherwise document the explicit status/recovery/plugin boundary.

## Reproduction Assessment

Yes for the missing requested surface: source inspection on current main shows no `contextPercent` or `context=` carrier in runtime prompt params, embedded runner params, or `buildRuntimeLine()`. The broader real-world compaction/reset state-loss scenario was not live replayed.

## Solution Assessment

Unclear until maintainers decide the product direction. If they accept an always-injected Runtime field, omitted-when-unavailable bounded plumbing from fresh session usage metadata is the narrowest maintainable path.
