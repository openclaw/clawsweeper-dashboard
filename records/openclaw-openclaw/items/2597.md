## Summary

Keep open: current main still lacks the requested passive `context=X%` field in the system-prompt Runtime line, and recent issue discussion explicitly treats status commands and workspace-sync as mitigations rather than this feature.

## What This Changes

Add an agent-visible context usage percentage to the Runtime line injected into embedded agent system prompts.

## Best Possible Solution

If accepted, plumb an omitted-when-unavailable bounded context usage percentage from reliable session token/window metadata into embedded Runtime prompt params and cover it with focused prompt and runner tests; otherwise document the status/recovery/plugin boundary explicitly.

## Reproduction Assessment

Yes for the missing requested surface: source inspection on current main shows no context percentage carrier in `RuntimeInfoInput`, `buildRuntimeLine()`, or embedded runtime assembly. The broader real-world compaction/reset state loss was not live-replayed.

## Solution Assessment

Unclear until maintainers decide the product direction. If they accept an always-injected Runtime field, bounded omitted-when-unavailable plumbing is the narrowest maintainable path; otherwise the intended boundary should be documented around `/status`, `/context`, `/usage tokens`, and recovery plugins.
