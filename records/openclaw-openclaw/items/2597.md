## Summary

Keep open: current main still does not implement the requested passive `context=X%` field in the Runtime line, and the latest issue discussion already distinguishes existing status/context commands and workspace-sync mitigation from this prompt-surface request.

## What This Changes

The issue requests adding a passive `context=X%` field to the system-prompt Runtime line so agents can see context pressure before compaction or resets.

## Best Possible Solution

If accepted, plumb a bounded context usage percentage from fresh session token/window metadata into embedded-runner params, `RuntimeInfoInput`, and `buildRuntimeLine()` with focused prompt and runner coverage; otherwise document that status commands, context reports, and recovery plugins are the intended boundary.

## Reproduction Assessment

Yes for the missing requested surface: source inspection on current main shows no carrier or formatter for a Runtime-line context percentage. The broader real-world compaction/reset state-loss scenario was not live replayed.

## Solution Assessment

Unclear until maintainers decide the product direction. If they accept an always-visible Runtime field, omitted-when-unavailable plumbing from fresh session usage metadata is the narrowest maintainable implementation path.
