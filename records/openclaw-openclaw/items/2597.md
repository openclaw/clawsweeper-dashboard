## Summary

Keep open: current main still lacks the requested passive `context=X%` Runtime-line field. Existing `/status`, `/context`, `/usage tokens`, and the linked `openclaw-workspace-sync` community plugin are useful inspection or recovery mitigations, but they do not provide this per-turn system-prompt primitive.

## What This Changes

Request adding an omitted-when-unavailable `context=X%` usage percentage to the system-prompt Runtime line for embedded agent runs.

## Best Possible Solution

If accepted, plumb a bounded context usage percentage from reliable fresh session token/window metadata into embedded runner Runtime prompt params and `buildRuntimeLine()` with focused prompt and runner coverage; otherwise document that status commands, context reports, and recovery plugins are the intended boundary.

## Reproduction Assessment

Yes for the missing requested surface: source inspection on current main shows no carrier or formatter for a Runtime-line context percentage. The broader real-world compaction/reset state-loss scenario was not live replayed.

## Solution Assessment

Unclear until maintainers decide the product direction. If they accept an always-visible Runtime field, omitted-when-unavailable plumbing from fresh session usage metadata is the narrowest maintainable implementation path; the existing plugin/status mitigations do not solve the same prompt-surface request.
