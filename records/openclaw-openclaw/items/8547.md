Keep open. Current main still lacks user-adjustable Control UI/WebChat font-size or font-scale controls, and the related closed reports preserve this issue as the canonical accessibility tracker rather than superseding it.

## What This Changes

## Best Possible Solution

Add a bounded browser-local text-size/font-scale setting applied through CSS variables across chat, composer, and dense Control UI surfaces; extend gateway `ui` schema/types/docs only if maintainers choose server-side sizing as a public contract.
