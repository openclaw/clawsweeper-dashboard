## Summary

Keep open. Current main still lacks a user-adjustable Control UI/WebChat font-size or font-scale control, and the related closed reports preserve this issue as the canonical accessibility tracker rather than superseding it.

## What This Changes

The issue requests configurable font sizing or font scaling for the OpenClaw web chat/UI, potentially through browser-local settings or gateway `ui` config.

## Best Possible Solution

Add a bounded browser-local text-size/font-scale setting in Appearance and Quick Settings, apply it through CSS variables across chat and dense UI surfaces, and extend gateway `ui` config only if maintainers explicitly choose server-side sizing as a public contract.

## Reproduction Assessment

Not applicable. This is a feature request rather than a broken existing contract, and current-main source/docs inspection shows the requested font-size/font-scale controls are absent.

## Solution Assessment

Unclear. The accessibility goal is sound, but the proposed gateway `ui` config shape is a product/API choice; a browser-local Appearance setting appears to be the narrower first implementation.
