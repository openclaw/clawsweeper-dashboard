## Summary

Keep open. Current main still lacks user-adjustable Control UI/WebChat font-size or font-scale controls in browser-local settings, Appearance/Quick Settings, gateway `ui` schema, CSS type variables, or public docs, and the related closed reports/PR do not supersede this remaining accessibility feature request.

## What This Changes

Request adding user-configurable font size or font scale controls for the Control UI/WebChat, potentially through gateway `ui` configuration.

## Best Possible Solution

Add a bounded browser-local text-size/font-scale setting applied through CSS variables across chat, composer, and core Control UI, extending gateway `ui` schema/types/docs only if maintainers choose server-side sizing as a public contract.
