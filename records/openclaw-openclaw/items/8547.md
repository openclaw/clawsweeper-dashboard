## Summary

Keep open. Current main still lacks a user-facing Control UI/WebChat font-size or font-scale setting, and the closed related reports do not supersede the remaining accessibility enhancement.

## What This Changes

Add a configurable font-size or font-scale control for the web chat interface to improve accessibility and user preference support.

## Best Possible Solution

Add a bounded, persisted browser-local Control UI text-size/font-scale setting, expose it in Appearance and Quick Settings, apply it through CSS custom properties across chat and surrounding UI, and document the behavior.

## Solution Assessment

Yes for a browser-local first pass: it matches the existing theme, roundness, and custom-theme settings pattern without prematurely expanding the gateway config schema. Gateway ui.fontSize/fontScale should be added only with explicit maintainer direction.
