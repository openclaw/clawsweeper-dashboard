The issue requests a configurable font-size or font-scale control for the OpenClaw web chat and Control UI, potentially via UI settings or gateway `ui` configuration.

## Best Possible Solution

Add a bounded, persisted Control UI text-size/font-scale setting, expose it in Appearance and Quick Settings, apply it through CSS custom properties across chat and surrounding UI, and document the browser-local behavior. Extend gateway `ui` schema/types/docs only if maintainers explicitly want server-side font sizing as a public config contract.
