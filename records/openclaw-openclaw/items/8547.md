## Summary

Keep open. Current `origin/main` still lacks a user-adjustable Control UI/WebChat font-size or font-scale setting in browser-local settings, Appearance/Quick Settings, gateway config schema, docs, or setting-driven CSS; related font-size reports and the old font-size PR are closed rather than canonical active fixes.

## What This Changes

Request adjustable font-size/font-scale controls for the Control UI/WebChat, potentially through browser-local UI settings and/or gateway `ui` config.

## Best Possible Solution

Add a bounded browser-local Control UI text-size/font-scale setting in `UiSettings`, expose it in Appearance and Quick Settings, apply it through CSS custom properties across chat and dense UI, and add gateway config/docs only if maintainers approve that public contract.

## Reproduction Assessment

Not applicable. This is a feature request, and source/docs inspection of `origin/main` shows the requested font-size/font-scale controls are absent.

## Solution Assessment

Unclear. The accessibility goal is sound, but the proposed gateway `ui.fontSize`/`ui.fontScale` schema is a product/API decision; a browser-local Appearance setting appears to be the narrower first implementation.
