## Summary

Keep open. Current main still does not expose user-adjustable font size or font scale in browser-local Control UI settings, Appearance/Quick Settings, gateway `ui` config, exported config types, or public docs; the related closed issues do not supersede this accessibility feature request.

## What This Changes

Request adding configurable Web UI/WebChat font size or scale controls, potentially through gateway `ui` config and CSS variables.

## Best Possible Solution

Add a bounded browser-local text-size or font-scale setting applied through CSS variables across chat, compose input, and core Control UI, extending gateway `ui` schema/types/docs only if maintainers choose server-side sizing as a public contract.

## Reproduction Assessment

Not applicable. This is a feature request rather than a broken existing contract, and source/docs inspection on current main shows the requested font-size/font-scale setting is absent.

## Solution Assessment

Unclear. The accessibility goal is sound, but the proposed gateway `ui` config shape is a product/API choice; a browser-local Appearance setting may be the narrower first step.
