## Summary

Keep open. Current main still lacks a user-adjustable Control UI/WebChat font-size or text-scale setting, and the latest MEMBER comment explicitly confirms this remains a valid accessibility enhancement.

## What This Changes

Request to add configurable font size or text scale controls for the OpenClaw web chat and Control UI.

## Best Possible Solution

Implement a bounded browser-local Control UI text-scale setting in UiSettings, expose it in Appearance and Quick Settings, apply it via shared CSS custom properties across chat and dense UI surfaces, and document it; add gateway ui schema only after explicit maintainer approval.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable: this is a feature request, and source/docs inspection of current main shows the requested font-size/font-scale controls are absent.

## Solution Assessment

Is this the best way to solve the issue? Unclear as proposed: the accessibility goal is sound, but a browser-local Appearance text-scale setting is the narrower first step than a public gateway ui.fontSize/ui.fontScale contract.
