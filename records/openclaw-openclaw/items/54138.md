## Summary

Keep open. Current main still passes heartbeat reply payloads through `buildReplyPayloads` unchanged before final channel delivery, while the shared plain-text bracket stripper is narrower than the reported `[TOOL_CALL]...[/TOOL_CALL]` and `[TOOL_RESULT]...[/TOOL_RESULT]` shapes and is not applied on that heartbeat payload branch. The issue is also paired with open closing-syntax PR #54165, so it should remain open until that PR or an equivalent fix lands.

## What This Changes

Heartbeat final replies should strip bracket-format `[TOOL_CALL]` and `[TOOL_RESULT]` blocks before Telegram/Discord or other channel delivery.

## Best Possible Solution

Keep this issue open and review/land PR #54165, or land an equivalent focused fix in `src/auto-reply/reply/agent-runner-payloads.ts` that strips uppercase `[TOOL_CALL]...[/TOOL_CALL]` and `[TOOL_RESULT]...[/TOOL_RESULT]` blocks from heartbeat reply text before channel delivery, with regression coverage in `src/auto-reply/reply/agent-runner-payloads.test.ts`. Track the broader interactive/XML leak separately if it still reproduces.
