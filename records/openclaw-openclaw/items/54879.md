## Summary

Keep open. Current main still has no browser `act` kind, request type, CLI command, or docs surface for CDP-backed IME/native text insertion; managed text input still goes through Playwright `fill`/`type`, and existing-session text input still delegates to Chrome MCP `fill`. The report is recent, concrete, and still maps to a narrow browser plugin capability gap.

## What This Changes

Requests an additive browser action that focuses a target and inserts text through CDP `Input.insertText` for IME/non-Latin contenteditable rich editors.

## Best Possible Solution

Add a documented, additive browser action for native text insertion that focuses the requested element/tab and sends CDP `Input.insertText` for managed/raw CDP profiles, with schema, route normalization, CLI/docs, and regression coverage. Existing-session Chrome MCP profiles should either implement equivalent native insertion when supported or return a precise unsupported error that preserves the action contract.
