## Summary

Keep open: current main and the latest release still expose only the existing click/type/fill/press-style browser action surface, while the issue and FSB comment describe a real rich-editor IME gap that needs maintainer API and live-proof decisions before implementation.

## What This Changes

Request adds a browser act action that inserts text through CDP/native IME-style input for contenteditable rich editors, especially X composer with Korean text.

## Best Possible Solution

Add a focused native text insertion browser act kind in the browser plugin, using CDP where supported, with explicit unsupported behavior for driver gaps and regression plus live rich-editor proof.

## Reproduction Assessment
