## Summary

Keep open: current main and the latest release still lack a CDP/IME/native text insertion browser action, and the existing public action schema, route normalization, managed Playwright path, Chrome MCP path, CLI, and docs only expose type/fill/press-style text entry. The remaining work is a public browser-tool capability that needs maintainer API and transport-support decisions, plus live repro/proof for the X/Korean IME case.

## What This Changes

The issue requests a browser action that inserts IME/native text through CDP for contenteditable rich editors where paste/type/fill are unreliable.

## Best Possible Solution

Add a focused browser act action for native/CDP text insertion where supported, keep type/fill behavior unchanged, document unsupported transports, and cover managed CDP plus existing-session behavior with tests and live rich-editor proof.

## Reproduction Assessment

No high-confidence live reproduction is available from this review. Source inspection does show current main has no native/CDP text insertion browser action and routes current text entry through type/fill paths.

## Solution Assessment

Unclear for final API shape: the browser action layer is the plausible owner, but maintainers still need to choose the public name, focus contract, driver support matrix, and guard behavior before implementation.
