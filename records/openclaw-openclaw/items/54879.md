## Summary

Keep open: current main and v2026.6.6 still do not expose a CDP/IME text insertion browser action; the existing text paths route through Playwright type/fill or Chrome MCP fill, while the FSB comment supports keeping this as a separate action that needs maintainer API and guard decisions.

## What This Changes

The issue requests a CDP-backed browser action for IME/native-like text insertion in contenteditable rich editors such as the X composer.

## Best Possible Solution

Add a focused browser act action for native text insertion through CDP where supported, keep current type/fill semantics unchanged, and document/test explicit unsupported behavior for drivers that cannot send native text.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection shows the current action set has no CDP/IME insertion action and existing text paths route through fill/type actions.

## Solution Assessment

Unclear for final API shape: the requested layer is plausible and no existing action/config solves it, but maintainers still need to choose the public action name, supported drivers, focus semantics, and guard behavior.
