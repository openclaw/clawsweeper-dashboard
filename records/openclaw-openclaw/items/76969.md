## Summary

Keep this PR open: it targets a real current-main transcript persistence bug, but the proposed rewrite path can delete a non-tail active user entry from persisted history and needs a narrow guard before merge.

## What This Changes

The PR adds a helper to persist orphaned trailing user leaf removal, wires the orphan-repair path through it, adds focused unit tests, and wraps Discord draft boundary callbacks.

## Best Possible Solution

Persist the orphan removal only for a proven trailing file-backed leaf, keep the branch/reset fallback for all other managers or non-tail states, add a regression test for the non-tail branch case, and then land with green changed checks.

## Reproduction Assessment

Yes. Source inspection shows current main reaches the orphaned-user repair block for a file-backed trailing user leaf and only moves the in-memory leaf pointer, leaving the persisted JSONL unchanged.

## Solution Assessment

No. Persisting the removal is the right direction, but the PR should guard the rewrite path to true trailing leaves instead of splicing the first matching id from the full transcript history.
