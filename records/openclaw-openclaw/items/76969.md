## Summary

Keep open: the PR addresses a valid source-reproducible transcript persistence bug that is not fixed on current main, but the proposed helper needs a small correctness fix around root/no-assistant session files before it is safe to merge.

## What This Changes

The PR adds a helper that removes an orphaned user leaf from file-backed `SessionManager` internals and rewrites the transcript, with unit coverage for persisted removal and fallback branching.

## Best Possible Solution

Land a narrow repair that persists orphaned leaf deletion while preserving first-assistant flush semantics, adds parent and root/no-assistant regression coverage, and includes the required changelog entry.

## Reproduction Assessment

Yes by source inspection: current main removes the orphaned user leaf only from the in-memory branch, while the file-backed manager persists from `fileEntries`. The linked issue provides matching field evidence and logs, but this read-only pass did not run an end-to-end replay.

## Solution Assessment

No as written: rewriting the underlying file-backed entries is the right direction, but the helper must preserve the no-assistant/root-turn flush contract before calling `_rewriteFile()`. A changelog entry is also required for this user-visible fix.
