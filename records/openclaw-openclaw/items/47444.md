The issue requests that workflow and Telegram-visible replies strip internal XML tool-call scaffolding, including `<function_calls>` and `<function_response>`, from user-visible chat.

## Best Possible Solution

## Solution Assessment

Yes. The narrow maintainable solution is to update the shared sanitizer and adjacent tests; a Telegram-only workaround would leave the same leak possible in other delivery and history paths.
