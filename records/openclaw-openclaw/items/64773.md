## Summary

Keep open: current main does not implement this sidecar, and the PR still has source-provable correctness blockers plus a missing real-behavior-proof gate after the author follow-up.

## What This Changes

The PR adds a `session-state.json` sidecar snapshot writer, tests, and a call from session-store saves.

## Best Possible Solution

If maintainers want this observable-state contract, make the sidecar best-effort, collision-free for custom store paths, per-entry agent-aware, documented or explicitly accepted, and backed by focused tests plus redacted runtime proof.

## Reproduction Assessment

Yes for the PR defects: source inspection of the PR head and current-main session-store contracts shows the unguarded auxiliary write, custom-store sidecar collision, and path-derived row attribution. I did not run tests because this review is read-only.

## Solution Assessment

No. The sidecar direction may be useful, but this patch is not the best merge path until it preserves existing session-store guarantees, handles custom/shared stores correctly, and has maintainer agreement on the new state-file surface.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Isolate snapshot failures from session store saves:** `src/config/sessions/store.ts:540-543`
  - body: `saveObservableSessionState` runs after `sessions.json` has already been written and caches updated, but it writes through `saveJsonFile`, which can throw. That makes callers observe a failed save even though the primary store was committed, so the sidecar write needs to be best-effort or otherwise isolated from the primary write path.
  - confidence: 0.93
- **[P2] Make sidecar paths unique for custom store files:** `src/config/sessions/observable-state.ts:45`
  - body: `resolveStorePath` supports templates like `/tmp/sessions-{agentId}.json`, where multiple agent stores live in the same directory. Writing every sidecar as that directory's `session-state.json` lets the last agent overwrite the others, so the sidecar path should be store-derived or explicitly aggregated.
  - confidence: 0.9
- **[P2] Derive entry agent IDs from session keys:** `src/config/sessions/observable-state.ts:81`
  - body: Shared stores are supported when `session.store` omits `{agentId}`, and they can contain rows such as `agent:main:room` and `agent:ops:room`. Mapping every snapshot row to the single agent ID inferred from the store path mislabels entries and breaks per-agent observability filters.
  - confidence: 0.9
