## What This Changes

Reports Telegram direct-chat messages being silently lost when polling offset handling can advance past an in-flight update before OpenClaw processes it.

## Best Possible Solution

Review, land, or replace https://github.com/openclaw/openclaw/pull/73993 with a current-main Telegram polling durability fix that separates fetched/accepted and handler-completed state while preserving token-fingerprint rotation and isolated spool behavior.
