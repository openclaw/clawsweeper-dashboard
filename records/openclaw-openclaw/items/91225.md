This PR makes light dreaming consider recent Dream Diary history before applying the snippet limit, adds sweep-date and recent-diary context to narrative prompts, and covers the selection and unreadable optional-context paths with tests.

## Best Possible Solution

Land this bounded memory-core fix after an area owner confirms that recent diary coverage should demote otherwise high-ranked recall entries, while keeping config, schema, and DREAMS.md format unchanged.

## Reproduction Assessment

Yes. Current main's light dreaming path sorts by last recall time and recall count, slices to the phase limit, and prompts only with those capped snippets, which gives a source-level reproduction for high-recall early memories monopolizing the diary input.

## Solution Assessment

Yes. Moving diary-history awareness into light selection before the phase limit is the narrowest maintainable fix; prompt-only context or append-time dedupe would not stop stale snippets from winning the staging slot.
