## Summary

Keep open. ## What This Changes

The report asks for `session_status` to resolve explicit `sessionKey="current"` to the requesting channel-plugin session after preserving literal `current` key/session-id precedence.

## Best Possible Solution

Keep the issue open until one canonical fix PR lands. The implementation should add a narrow explicit-`current` requester fallback after literal `current` key/sessionId resolution, preserve self-visibility handling, and include a focused channel-style requester regression test.
