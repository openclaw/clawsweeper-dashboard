## Summary

Keep open: the reported Windows update race is real, but this PR is not merge-ready because it changes generic Scheduled Task stop/restart semantics and still lacks after-fix real Windows proof.

## What This Changes

The PR adds `schtasks /Change /DISABLE` to Windows Scheduled Task stop and `schtasks /Change /ENABLE` to Scheduled Task restart.

## Best Possible Solution

Land an update-scoped suspend/resume guard around mutable package replacement, restoring only tasks this invocation disabled while preserving ordinary Windows gateway stop semantics.

## Reproduction Assessment

Yes. Current main stops the managed service before package replacement, and the Windows Scheduled Task stop implementation uses `/End` without disabling already-registered repeated triggers; I did not run a native Windows update in this read-only review.

## Solution Assessment

No. This is a plausible mitigation, but it is broader than the bug because it changes generic stop/restart behavior instead of owning temporary suspension inside the update lifecycle.
