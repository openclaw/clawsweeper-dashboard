## What This Changes

The PR adds `/DISABLE` after Windows Scheduled Task stop and `/ENABLE` before Scheduled Task restart in `src/daemon/schtasks.ts`.

## Best Possible Solution

Land or replace an update-scoped suspend/resume guard that disables only enabled Windows Scheduled Task autostart during mutable package replacement and restores only tasks this invocation disabled.

Yes, source-reproducible. Current main stops the managed service before package replacement, and the Windows Scheduled Task stop implementation uses `/End` without disabling repeated triggers; I did not run a native Windows update in this read-only review.

## Solution Assessment

No. This is a plausible mitigation, but it is broader than the bug because it changes generic stop/restart behavior instead of owning temporary suspension inside the update lifecycle.
