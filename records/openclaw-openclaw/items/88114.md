## Label Justifications

- `P2`: This is a bounded Windows package-update reliability fix with limited platform blast radius but real availability impact.
- `merge-risk: 🚨 compatibility`: The patch changes existing Windows `gateway stop` semantics by persistently disabling Scheduled Task autostart.
- `merge-risk: 🚨 availability`: A persistently disabled Scheduled Task can leave the gateway unavailable after logon or reboot if restart or recovery does not run.

## Summary

Keep open: the Windows update race is still source-reproducible on current `main`, but this PR is not merge-ready because it moves a durable Scheduled Task disable into generic Windows stop/restart instead of scoping it to the package-update window.

## What This Changes

The PR changes Windows Scheduled Task stop/restart handling to run `schtasks /Change /DISABLE` after stop and `/ENABLE` before restart.

## Best Possible Solution

Land or adapt an update-scoped suspend/restore guard that disables only enabled Windows Scheduled Tasks during mutable package replacement, restores only what that update invocation disabled, and preserves normal `gateway stop` semantics.

## Reproduction Assessment

Yes. Current main stops the Windows Scheduled Task with `/End` only before package replacement, and Microsoft documents `/End` as stopping only running task instances; I did not run a native Windows update in this read-only review.

## Solution Assessment

No. The PR is a plausible mitigation but not the best fix because the durable disable belongs around the package-update replacement window and should restore only tasks suspended by the same update invocation.
