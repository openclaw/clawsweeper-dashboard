## What This Changes

The branch adds `/DISABLE` after Windows Scheduled Task stop and `/ENABLE` before Scheduled Task restart in `src/daemon/schtasks.ts`.

## Best Possible Solution

Land an update-scoped suspend/resume guard around mutable package replacement, restoring only tasks this update invocation disabled while preserving normal Windows gateway stop/restart semantics.

## Reproduction Assessment

Yes, source-reproducible: current main stops the managed service before mutable package update work, and the Windows Scheduled Task stop path uses `/End` without disabling the registered task. I did not run a native Windows package update in this read-only pass.

## Solution Assessment

No. The proposed fix is broader than the bug: the safer shape is update-scoped suspension/restoration, not persistent enable-state changes in generic stop and restart.
