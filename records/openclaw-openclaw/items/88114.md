## Summary

Keep open: the Windows update race is source-reproducible, but this PR changes generic Windows Scheduled Task stop/restart semantics instead of limiting suspension to the package-update window. ## What This Changes

The PR adds `schtasks /Change /DISABLE` in `stopScheduledTask()` and `schtasks /Change /ENABLE` in `restartScheduledTask()` for Windows Scheduled Task lifecycle control.

## Best Possible Solution

Land an update-scoped guard around package replacement that disables only enabled Scheduled Tasks, restores only tasks this invocation disabled, and validates native Windows update behavior.

## Reproduction Assessment

Yes, source-reproducible. Current main stops the managed service before package replacement, and Windows Scheduled Task stop uses `/End` without disabling repeated triggers; I did not run a native Windows update in this read-only review.

## Solution Assessment

No. This PR is a plausible mitigation, but it is broader than the bug because it changes generic Windows stop/restart behavior instead of owning temporary suspension inside the update lifecycle.
