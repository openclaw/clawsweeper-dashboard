## Summary

Keep this PR open for revision: the mitigation targets a source-reproducible Windows update race, but the new disable operation can fail silently while stop reports success, and no real Windows behavior proof demonstrates the legacy-task update flow.

## What This Changes

The PR disables the Windows gateway Scheduled Task after stopping it and re-enables it before restart to prevent legacy repeat-trigger relaunches during updates.

## Best Possible Solution

Disable the registered task before ending and cleaning up its runtime, reject nonzero disable/enable results with clear errors, add focused ordering and failure tests, and prove the full legacy-task update flow on Windows.

## Reproduction Assessment

Yes at source level: the update path stops the managed service before package replacement, and a legacy repeating task remains eligible to relaunch until it is successfully disabled. No real Windows execution proof is attached.

## Solution Assessment

No as written. Task disablement is the appropriate direction, but it must occur before the stop window and fail loudly when Task Scheduler rejects the state change.
