## Label Justifications

- `P2`: This is a bounded Windows package-update reliability bug with real availability impact but limited platform blast radius.
- `merge-risk: 🚨 compatibility`: The patch changes existing Windows `gateway stop` behavior by persistently disabling Scheduled Task autostart.

## Summary

Keep open: the Windows update race is still real on current main and the latest release, but this PR puts persistent enable/disable mutations into generic Windows stop/restart instead of the update-owned window. The update-scoped maintainer PR at https://github.com/openclaw/openclaw/pull/88292 is related and better-shaped, but it is still open, draft, and dirty, so it does not safely supersede this branch yet.

## What This Changes

The PR adds `schtasks /Change /DISABLE` after Windows Scheduled Task stop and `schtasks /Change /ENABLE` before Scheduled Task restart.

## Best Possible Solution

Land an update-scoped suspend/resume guard around mutable package replacement, restoring only tasks suspended by that update invocation while preserving ordinary Windows gateway stop semantics.

## Reproduction Assessment

Yes, source-reproducible. Current main stops the managed service before package replacement, and the Windows Scheduled Task stop implementation uses `schtasks /End` without disabling already-registered repeated triggers; I did not run a native Windows update in this read-only review.

## Solution Assessment

No. Disabling inside generic `stopScheduledTask()` is broader than the bug requires; the narrower maintainable fix is update-scoped suspension and restoration around package replacement.

- **[P1] Keep persistent disable out of generic stop:** `src/daemon/schtasks.ts:1324`
  - body: This new `/DISABLE` call runs for every Windows gateway stop because `src/daemon/service.ts` maps win32 service stop to `stopScheduledTask`, not only package updates.   - confidence: 0.91

  - body: `restartScheduledTask()` can be invoked without a preceding stop from this PR, so this unconditional `/ENABLE` can re-enable an operator-disabled Scheduled Task. Track whether the update flow suspended an enabled task and only restore that recorded state.
  - confidence: 0.86
