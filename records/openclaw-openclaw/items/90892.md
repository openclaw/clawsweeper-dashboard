## Summary

Keep open: current main and the latest stable release still route ordinary Feishu/Lark paragraph-break text through the same post text path, and there is no merged fix or canonical open replacement for this narrower outbound rendering bug.

## What This Changes

The issue asks OpenClaw's Feishu/Lark outbound text rendering to preserve multi-line and multi-paragraph messages instead of showing only the last line in clients.

## Best Possible Solution

Make Feishu auto text rendering preserve paragraph breaks by using a valid paragraph-aware post payload or consistent interactive-card routing for paragraph-break text, with focused tests across send, outbound, and reply dispatch paths.

## Reproduction Assessment

No high-confidence live reproduction was run because this review did not exercise a real Feishu/Lark account. Source inspection does show paragraph-break text can reach the single-element post path under `renderMode: "auto"`, which is enough to keep the issue open.

## Solution Assessment

Unclear until live Feishu behavior is checked: routing paragraph-break text to cards is the simplest mitigation, while making the post payload paragraph-aware may be the cleaner transport-level fix if mentions and markdown semantics are preserved.
