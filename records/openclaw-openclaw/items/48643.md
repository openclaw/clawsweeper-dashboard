## Summary

Keep open: current main does not include this dedicated parallel-thread dialog, and the PR still has source-confirmed correctness blockers, missing real behavior proof, and overlapping Control UI direction with the pinned-chat work.

## What This Changes

The PR adds a Control UI new-thread modal, state wiring, handler, and CSS for creating a parallel chat thread with an agent, optional label, and first message.

## Best Possible Solution

Rework the feature into one Control UI parallel-session flow that uses sessions.create with parent linkage, label, and first-message support, preserves prior chat state on rollback, and aligns the visible affordance with the pinned-chat direction.

## Reproduction Assessment

Yes for PR review: source inspection of the PR diff plus current main contracts shows the direct chat.send path and rollback defects. This is not a current-main bug reproduction because the dedicated create-thread modal is not on main.

## Solution Assessment

No. The branch should use the existing sessions.create contract, preserve the previous transcript and actionable error on failure, and settle the UI alongside the pinned-chat work before merge.
