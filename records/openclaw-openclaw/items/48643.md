## Summary

Keep this PR open, but it needs changes before merge. Current main still lacks the dedicated New thread dialog, while this branch bypasses the current `sessions.create` contract and has source-reproducible rollback, error-display, keyboard, and changelog blockers.

## What This Changes

The PR adds a Control UI New thread dialog, app callback/state wiring, modal styles, and a client-generated `chat.send` flow for creating a parallel chat thread.

## Best Possible Solution

Rework the branch on current main so the dialog uses `sessions.create` with `parentSessionKey`, label, and initial message support, preserves rollback state, handles modal keyboard focus robustly, and ships with focused tests plus a changelog entry.

## Reproduction Assessment

Yes, for this PR review. Source inspection of the PR head reproduces the blockers, and current main source shows the `sessions.create` contract and `loadChatHistory` behavior that make the branch stale and lossy.

## Solution Assessment

No. The UI direction is useful, but the branch should be rebuilt on the current session creation contract instead of creating a client-side key and treating `chat.send` as the session creation API.
