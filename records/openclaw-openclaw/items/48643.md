Full review comments:

- **[P1] Route thread creation through `sessions.create`:** `ui/src/ui/app-chat.ts:181-186`
  - body: This creates a generated key, switches local state, and calls `chat.send` directly.   - confidence: 0.92

  - body: The catch path switches back to the previous session but clears messages and stream/tool state before attempting `loadChatHistory`. If the gateway is disconnected or the reload fails, the user returns to the previous session with an empty transcript even though no reset was requested.
  - confidence: 0.89
- **[P2] Preserve the create failure message after rollback:** `ui/src/ui/app-chat.ts:205-206`
  - body: After setting the specific create-thread error, the following `loadChatHistory` call can clear `lastError` and leave it null when history reload succeeds, so the dialog reports the generic failure instead of the real reason.
  - confidence: 0.9
- **[P3] Keep Escape handling active for the dialog:** `ui/src/ui/views/chat.ts:904`
  - body: The Escape handler is attached to the overlay, so it only receives key events while focus remains inside a descendant. If focus leaves the dialog, Escape no longer closes or guards it; use a document-level listener or a focus trap while the modal is open.
  - confidence: 0.78
