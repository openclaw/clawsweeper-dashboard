Feishu group-message sessions should use the `oc_*` group chat id for `groupId` instead of the sender `ou_*` open_id so group replies are not dropped by tool policy.

## Best Possible Solution

Keep the merged resolver-based fix on main and let the next release carry it; handle overlapping Feishu PRs as separate superseded PR cleanup if they have no unique remaining work.

## Reproduction Assessment

No current-main failing reproduction remains. The former source path is now covered by a Feishu-shaped regression test where `From` stays on the sender but `resolveGroupSessionKey` returns the `oc_*` chat id from `OriginatingTo`.

## Solution Assessment
