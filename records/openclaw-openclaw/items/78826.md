## Summary

Close as implemented: current main now resolves Feishu group context from the trusted chat target while preserving sender identity, so the reported `ou_*` sender-as-groupId mismatch is fixed by the merged resolver change.

## What This Changes

Feishu group session metadata should use the `oc_*` group chat ID instead of the sender `ou_*` open_id so group replies pass tool-policy validation.

## Best Possible Solution

Keep the merged resolver behavior from https://github.com/openclaw/openclaw/pull/79483; any remaining slash-command work in https://github.com/openclaw/openclaw/pull/79556 can be handled separately.

## Reproduction Assessment

No current-main failing reproduction remains. The pre-fix path was source-reproducible from `From=feishu:ou_*` plus `To/OriginatingTo=chat:oc_*`, and current main now has regression coverage proving the resolver returns the `oc_*` group id.

## Solution Assessment

Yes. The merged fix uses the trusted `OriginatingTo`/`To` chat target for group context while preserving speaker identity in `From`/`SenderId`, which is narrower and safer than weakening the tool-policy guard or adding an unsupported `GroupId` field.
