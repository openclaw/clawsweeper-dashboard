## Summary

Close: current main fixes the Feishu groupId mismatch via the merged resolver-level change in https://github.com/openclaw/openclaw/pull/79483. Feishu group turns still preserve sender identity in `From`, but `resolveGroupSessionKey` now uses the trusted `chat:oc_*` target for group context and focused regression coverage asserts the expected `oc_*` group id.

## What This Changes

The issue reports Feishu group message metadata storing the sender `open_id` as `groupId` instead of the `oc_*` group chat ID.

## Best Possible Solution

Keep the merged resolver-level fix from https://github.com/openclaw/openclaw/pull/79483 and treat the overlapping open alternative PR as separate stale follow-up cleanup.

## Reproduction Assessment

No, not against current main. The old Feishu-shaped payload with `From: feishu:ou_*` and `OriginatingTo: chat:oc_*` now resolves the group id from the `oc_*` chat target, and the Feishu regression test covers that path.

## Solution Assessment

Yes. The merged resolver-level fix uses the channel-owned chat target for group context without weakening the tool-policy fail-closed check or losing sender attribution in `From`/`SenderId`.
