## What This Changes

The issue asks Feishu group-message session metadata to use the group chat `oc_*` ID instead of the sender `ou_*` open_id so group replies are not dropped.

## Best Possible Solution

Keep the merged resolver/test fix on main and close this report as implemented, with release notes carrying the Feishu fix forward from `Unreleased`.

## Reproduction Assessment

No current-main failing reproduction remains: the former Feishu-shaped source path now resolves the group id from `OriginatingTo`/`To` to the `oc_*` chat id, and the regression test covers that payload. The original bug was source-reproduced before the merged fix.

## Solution Assessment

Yes. The merged resolver fix is the narrow maintainable solution because it preserves speaker-shaped `From`/`SenderId` while deriving trusted group context from the channel-owned chat target, without weakening the tool-policy guard or adding an unsupported `GroupId` field.
