## Summary

Close: current main fixes the Feishu groupId mismatch by resolving group context from the trusted chat target while preserving sender identity in the inbound speaker fields. The fix landed after v2026.5.7, so this is current-main-only until the next release.

## What This Changes

Feishu group messages should derive session and embedded-run groupId from the Feishu group chat ID (`oc_*`) instead of the sender open_id (`ou_*`).

## Best Possible Solution

Keep the merged resolver-based fix from https://github.com/openclaw/openclaw/pull/79483 and handle the overlapping open PR https://github.com/openclaw/openclaw/pull/79556 separately as obsolete or partial work.

## Reproduction Assessment

No, not as a failing current-main path. The pre-fix source path is clear, but current main now resolves the Feishu-shaped group payload to the `oc_*` chat id and has a regression test for that exact shape.

## Solution Assessment

Yes. The current implementation fixes the trusted group-resolution path instead of relaxing the tool-policy mismatch guard or adding an unused `GroupId` field.
