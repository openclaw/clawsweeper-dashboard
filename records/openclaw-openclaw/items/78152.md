## Summary

Keep open. The PR is a useful narrow fix for silent memory-flush error handoff, but it is not merge-ready because the new returned error payload can still be suppressed in message-tool-only delivery, and the supplied proof is only tests/check output rather than real runtime/chat behavior.

## What This Changes

The PR preserves `isError` payloads during silent auto-reply maintenance, forwards memory-flush error payloads into `runReplyAgent` before the main run, adds regression tests, and updates the changelog.

## Best Possible Solution

Mark memory-flush runtime error payloads for source-suppression delivery, keep the `isError` silent-turn exception, add coverage for message-tool-only delivery, and require real runtime/chat proof before merge.

No live high-confidence reproduction was established in this read-only review. Source inspection does trace current main through silent memory-flush execution, dropped text-only `isError` payloads, and ignored memory-flush result payloads.
