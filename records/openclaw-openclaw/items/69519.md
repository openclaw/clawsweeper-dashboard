## Summary

Keep open: the Sessions Limit validation problem is source-reproducible on current main, but this PR still has correctness blockers around stored limit coercion and blank-limit handling, and it lacks after-fix real behavior proof.

## What This Changes

This PR changes the Control UI Sessions Limit filter to an HTML number input, moves selected UI/controller state toward numeric limits, updates focused tests, and includes CSS formatting churn.

## Best Possible Solution

Finish a narrow Sessions Limit fix with one consistent state model: preserve editable blank state and coerce at the request boundary, or fully migrate the app/view/controller limit state to `number | undefined` and only send positive integers.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: the Sessions Limit field is string-backed text input, while the controller and Gateway schema show only positive integer limits should reach `sessions.list`.

## Solution Assessment

No. The numeric-input direction is reasonable, but this implementation removes request-boundary coercion before all state/default paths are numeric and uses `0` as a blank/invalid sentinel.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Coerce stored limit before sessions.list:** `ui/src/ui/controllers/sessions.ts:159`
  - body: `loadSessions` now forwards `state.sessionsFilterLimit` directly when no override is passed. The PR head still initializes that field as the string `"120"`, and opening the Sessions tab calls `loadSessions(app)` without overrides, so the request can send `limit: "120"` instead of the schema-required integer.
  - confidence: 0.93
- **[P2] Preserve blank limits without a zero sentinel:** `ui/src/ui/views/sessions.ts:293`
  - body: The input handler maps blank or invalid values to `0`. That stores an invalid limit value and relies on downstream guards, making cleared input behave as an omitted/unbounded limit rather than the PR's stated `undefined` blank state.
  - confidence: 0.82
