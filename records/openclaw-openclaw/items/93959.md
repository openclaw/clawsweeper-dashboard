## Summary

## What This Changes

The PR adds an in-memory `pendingCatchupDeferralJobIds` Set to cron service state and threads it into selected maintenance recomputes so overflow catch-up deferrals survive read/status and timer recompute paths.

## Best Possible Solution

Make the pending deferral exemption lifecycle-owned and central: every default maintenance recompute consults it, IDs clear when due or consumed, unrelated stale-future repair still works, and redacted real runtime proof is added before merge.

## Reproduction Assessment

## Solution Assessment

No. Persisting deferral IDs in service state is the right direction, but this patch is not the best fix until it expires those IDs, covers the remaining cleanup recomputes, and supplies real runtime proof.
