## Summary

Keep open. The PR fixes the right provider-local path for #76435, but it still leaves the `version` attribution header in browser `offerHeaders`, and OpenAI's current CORS preflight for the SDP endpoint allows only `authorization,content-type`, so the browser flow would still fail after `originator` is removed.

## What This Changes

The PR updates the OpenAI realtime browser offer-header filter and provider test so `originator` is no longer returned in browser `offerHeaders` while Gateway-side session minting keeps attribution headers.

## Best Possible Solution

Finish the provider-local fix by returning no OpenAI attribution-only headers in browser `offerHeaders` for direct SDP offers, keep Gateway/native OpenAI attribution intact, cover that in the focused provider test, and add the required changelog fix entry.

## Solution Assessment

No. The filter belongs in the OpenAI provider, but this patch only removes `originator` and leaves `version`, so the same CORS failure remains under the current endpoint policy.
