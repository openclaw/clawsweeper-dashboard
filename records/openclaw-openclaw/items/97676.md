## Summary

Keep open: this PR is a useful candidate for the open security-audit false-positive bug, but the current diff still uses a post-substitution snapshot for the source-value check. That means the real CLI path can keep reporting env-backed secrets, and merging as-is could make stored-secret suppression depend on mismatched source/resolved inputs.

## What This Changes

The branch passes a source config to the security-audit stored-secret collector, changes `openclaw security audit` to read `getRuntimeConfigSourceSnapshot()`, and adds direct audit regression tests for env-ref suppression and plaintext warnings.

## Best Possible Solution

Carry authored env-reference provenance or raw parsed field values for `gateway.auth.password` and `hooks.token` into security audit, cover the CLI/config-snapshot route, and keep plaintext literal warnings intact.

## Reproduction Assessment

Yes by source inspection. `resolveConfigForRead` substitutes `${ENV}` before `ConfigFileSnapshot.sourceConfig` is stored, and this PR reads that snapshot at `openclaw security audit`, so a real env-backed config can still reach the collector as the resolved secret.

## Solution Assessment

No. This is a plausible direct-collector change, but current `sourceConfig` is already post-substitution; the best fix is to carry authored field values or explicit env-ref provenance into the audit path and test it through the CLI/config snapshot route.
