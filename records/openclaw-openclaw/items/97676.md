## Summary

Keep open: the patch is not merge-ready because it threads `sourceConfig` through the collector, but the CLI source snapshot available on current `main` is already after `${ENV}` substitution, so the real `openclaw security audit` path still cannot see the authored env-ref text.

## What This Changes

The PR adds a `sourceConfig` argument to security-audit secret-in-config checks, passes a runtime source snapshot from `openclaw security audit`, and adds regression tests for env-reference suppression and plaintext flagging.

## Best Possible Solution

Use actual authored pre-env-substitution config or field-level provenance for `gateway.auth.password` and `hooks.token`, then cover the real CLI/config-snapshot path plus plaintext-literal cases.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence repro path: the config snapshot source is populated after env substitution, and the CLI audit path reads that snapshot before calling the secret-in-config collector; I did not run a live CLI repro in this read-only review.

## Solution Assessment

No. Passing `getRuntimeConfigSourceSnapshot()` is not the best fix because that snapshot is already env-substituted; the maintainable fix needs authored-value provenance or a snapshot field that retains pre-env templates.
