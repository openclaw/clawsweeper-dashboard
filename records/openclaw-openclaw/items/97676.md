## Summary

Keep open; the branch addresses the original false-positive shape, but it can also suppress stored-secret findings when the referenced env value is still authored in `openclaw.json` through `config.env` or `config.env.vars`.

## What This Changes

The PR changes security-audit stored-secret checks to consult `ConfigFileSnapshot.parsed` values before suppressing gateway password and hooks token findings, with regression tests for env refs and plaintext literals.

## Best Possible Solution

Keep the parsed-authored-value fix, but only suppress these findings for externally supplied env refs; config-owned env values should still warn or have equivalent audit coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows the audit passes only resolved `cfg` into `collectSecretsInConfigFindings`, so authored `${VAR}` strings are gone before `looksLikeEnvRef` when the env var resolves.

## Solution Assessment

No. The PR is close, but it is not the best fix until it distinguishes external env references from `config.env` or `config.env.vars` values stored in the same config file.
