## Summary

Keep open: the documentation clarification is still useful, but the PR is not correct as submitted because current main now appends non-task HEARTBEAT.md directives to cron-triggered heartbeat prompts and the branch still documents the older blanket behavior.

## What This Changes

The PR adds cron/HEARTBEAT.md guidance to the scheduled-tasks docs and AGENTS.md template, plus a non-runtime comment above the cron event prompt builder.

## Best Possible Solution

Revise the docs and template to state the current contract: cron system events use the scheduled-reminder prompt, non-task HEARTBEAT.md directives may be appended, and structured task-block execution remains owned by heartbeat due-task handling.

## Reproduction Assessment

Yes. Source inspection shows cron-tagged events route through buildCronEventPrompt, and tests cover both omission of the literal Read HEARTBEAT.md prompt and separate appending of non-task HEARTBEAT.md directives.

## Solution Assessment

No, not as submitted. Documentation is the right surface, but the wording needs to match current HEARTBEAT.md directive semantics and the AGENTS.md template should give direct agent-facing instructions instead of nested setup guidance.
