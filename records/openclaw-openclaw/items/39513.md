## Summary

This PR should stay open, but not merge as submitted: it targets a real cron/heartbeat documentation gap, yet the branch wording is stale against current main's HEARTBEAT.md directive handling and it still lacks required real behavior proof.

## What This Changes

This PR adds cron documentation, AGENTS.md template guidance, and a non-runtime source comment about main-session cron prompts not using the default HEARTBEAT.md prompt.

## Best Possible Solution

Revise the docs and template to document the current contract: cron wakes use the scheduled-reminder prompt, non-task HEARTBEAT.md directives may be appended, and structured task-block execution belongs to due-task heartbeat handling or isolated cron jobs.

## Reproduction Assessment

Yes, at source level. Current main routes cron-tagged events through buildCronEventPrompt, tests assert the literal 'Read HEARTBEAT.md' prompt is omitted, and separate tests cover appended non-task HEARTBEAT.md directives.

## Solution Assessment

No, not as submitted. Documentation is the right surface, but the wording needs to match current HEARTBEAT.md directive semantics and the AGENTS.md template should give direct agent-facing instructions instead of nested setup guidance.
