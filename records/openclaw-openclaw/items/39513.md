The PR adds scheduled-jobs documentation, AGENTS.md template guidance, and a non-runtime source comment about main-session cron prompts not including the default HEARTBEAT.md read instruction.

## Best Possible Solution

Revise the docs and template to state the current contract: cron system events use the scheduled-reminder prompt, non-task HEARTBEAT.md directives may be appended, and structured task-block execution remains owned by heartbeat due-task handling.

## Reproduction Assessment

## Solution Assessment

No, not as submitted. Documentation is the right surface, but the wording needs to match current HEARTBEAT.md directive semantics and the AGENTS.md template should provide direct agent-facing instructions instead of nested setup guidance.
