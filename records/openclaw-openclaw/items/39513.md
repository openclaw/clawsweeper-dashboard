## Summary

## What This Changes

This PR adds cron/HEARTBEAT documentation, an AGENTS.md template section, and a non-runtime comment about `buildCronEventPrompt`.

## Best Possible Solution

Revise the docs/template to state that cron wakes use a scheduled-reminder base prompt, non-task HEARTBEAT.md directives can be appended, and structured task execution belongs to due-task heartbeat handling or isolated cron jobs.

## Reproduction Assessment

Yes, at source level. Current main routes cron-tagged events through `buildCronEventPrompt`, existing tests assert the literal `Read HEARTBEAT.md` prompt is omitted, and separate tests cover appended non-task HEARTBEAT.md directives.

## Solution Assessment

No, not as submitted. Documentation is the right surface, but the wording needs to match current HEARTBEAT.md directive semantics and the AGENTS.md template should give direct agent-facing instructions instead of nested setup guidance.
