## Summary

## What This Changes

The PR adds cron documentation, AGENTS.md template guidance, and a TypeScript comment clarifying that main-session cron system-event prompts do not automatically include the default HEARTBEAT.md instruction.

## Best Possible Solution

Land a docs-only clarification after making the template text direct agent-facing instructions and cleaning whitespace, while leaving any runtime prompt unification or new config option to separate tested work.

## Reproduction Assessment

Not applicable as a docs clarification PR. The underlying behavior has a high-confidence source and test path: current main selects buildCronEventPrompt for cron events, and tests assert that prompt omits 'Read HEARTBEAT.md'.

## Solution Assessment

Mostly yes: documenting the existing self-contained cron-event prompt is a narrow maintainable path for this PR. The safer version inlines the AGENTS.md cron handling steps as direct instructions and removes the trailing whitespace before merge.
