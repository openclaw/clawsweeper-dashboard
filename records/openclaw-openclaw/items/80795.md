## Summary

Close as not actionable in the source repo: the reported Telegram group symptom matches the documented shipped visible-reply policy, where normal group/channel final answers stay private unless `messages.groupChat.visibleReplies` is set to `automatic` or the agent explicitly uses `message(action=send)`.

## What This Changes

The issue asks for text-only assistant replies in Telegram group chats to be posted visibly without requiring a message tool call.

## Best Possible Solution

Keep the current policy and have affected deployments set `messages.groupChat.visibleReplies: "automatic"` when they want legacy room-visible text-only final replies.

## Reproduction Assessment

No high-confidence bug reproduction is present; source inspection shows the reported behavior is expected for Telegram group turns under the default group visible-reply policy. The report does not say `messages.groupChat.visibleReplies` was set to `automatic`, which is the documented configuration for visible text-only room replies.

## Solution Assessment

Yes, closing as configuration/support is the narrowest maintainable outcome. Changing Telegram send code would bypass a documented cross-channel group policy; the existing supported fix is `messages.groupChat.visibleReplies: "automatic"` or reliable message-tool use.
