## Summary

The reported no-send symptom matches current group/channel visible-reply policy rather than a Telegram delivery failure: normal final assistant text is private in group rooms unless the agent uses the message tool or the deployment opts back into automatic room replies.

## What This Changes

Issue asks OpenClaw to automatically deliver text-only assistant final responses back to Telegram group chats.

## Best Possible Solution

Keep the current policy and direct affected deployments to use `messages.groupChat.visibleReplies: "automatic"` when they want legacy automatic Telegram group final replies.

## Reproduction Assessment

Yes. Source and tests show that a group/channel turn without automatic visible replies or an explicit message-tool send will produce private final assistant text and no Telegram send.

## Solution Assessment

Yes. The narrow maintainable answer is configuration or message-tool use; changing Telegram dispatch to auto-send text-only group finals would bypass the current group visible-reply policy.
