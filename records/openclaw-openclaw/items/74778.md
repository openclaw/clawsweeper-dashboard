## What This Changes

This PR updates the Bonjour advertiser watchdog to treat ciao `probing` and fresh name/hostname conflict-renames as in-progress states, adds regression tests, and documents the behavior.

## Best Possible Solution

Land a focused Bonjour plugin fix that treats ciao negotiation and fresh conflict renames as in-progress while preserving bounded disable behavior for genuinely stuck advertisers; keep the broader bundled-plugin default policy tracked separately in #74209 and the WSL2 environment-specific behavior in #74820.

## Reproduction Assessment

Yes. A unit-level reproduction on current main can hold the mocked ciao service in `probing`; the watchdog does not skip that state and reaches the non-announced re-advertise path before the stuck-state restart path.

## Solution Assessment

Yes. The PR is the narrowest maintainable fix for this scoped bug: it stays inside the Bonjour plugin, uses ciao's own state and conflict-event contract, adds focused regression tests, and updates the public Bonjour troubleshooting doc.
