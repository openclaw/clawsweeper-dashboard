## Summary

Keep open. Current main has source-backed mitigations for the refresh, heartbeat display, and typing slices, but outbound delivery mirroring still collapses media URLs to filename-only text before writing session history, so Control UI cannot render the reported generated-media preview path.

## What This Changes

The issue reports active-run responsiveness regressions across Control UI reconnect/refresh, Telegram typing, heartbeat transcript noise, and outbound media mirroring.

## Best Possible Solution

Keep the landed refresh, heartbeat, typing, and gateway responsiveness mitigations, then preserve outbound mirror media as previewable managed image or attachment content in session history instead of filename-only text.

## Reproduction Assessment

Partially yes. The remaining media mirror defect is reproducible from source through deliverOutboundPayloads and the current test expectation; the broader Docker/WSL2 reconnect and Telegram typing symptoms were not live-reproduced in this read-only review.

## Solution Assessment

Yes for a focused repair, no for closure. The narrow maintainable path is to preserve media URLs through the mirror append path and add focused tests, while leaving already-mitigated gateway latency, heartbeat, refresh, and typing slices alone unless new current-main evidence shows they still fail.
