## Summary

Keep open. Current main has mitigations for active-run refresh and heartbeat visibility, but the outbound delivery mirror still converts media URLs into basename-only text and writes a text-only `delivery-mirror` turn, so the reported Control UI media-preview defect remains source-confirmed.

## What This Changes

The issue reports a v2026.4.26 regression where active runs can stall Control UI reconnects, interrupt Telegram typing, expose heartbeat noise, and mirror delivered media as filenames instead of previews.

## Best Possible Solution

Keep the landed responsiveness and heartbeat mitigations, then repair outbound delivery mirroring so successful channel media is persisted as previewable managed media or structured attachment content in session history.

## Reproduction Assessment

Yes, partially. The remaining media mirror defect is source-reproducible through `deliverOutboundPayloads` into the text-only delivery-mirror transcript append; the broader Docker/WSL2 reconnect and Telegram typing symptoms were not live-reproduced in this read-only pass.

## Solution Assessment

Yes for a focused repair, no for closure. The narrow maintainable path is to preserve media references through the mirror append/history path and add regression coverage without reopening the already-mitigated refresh and heartbeat slices unless fresh evidence shows they still fail.
