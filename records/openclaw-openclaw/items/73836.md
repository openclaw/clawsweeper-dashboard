## Summary

Keep open because current main still has a source-confirmed delivery mirror defect: outbound media is reduced to filename text before session history, so Control UI cannot render the reported media preview even though several other active-run symptoms now have mitigations.

## What This Changes

The issue asks OpenClaw to keep Control UI/Gateway responsive during active runs, preserve Telegram typing, hide heartbeat transcript noise, and mirror outbound media as previews instead of filenames.

## Best Possible Solution

Keep the landed refresh, heartbeat, and typing mitigations, then make delivery-mirror transcript entries carry previewable media content through session history.

## Reproduction Assessment

Partially yes. The remaining media mirror defect reproduces from source inspection and current tests: mirror media URLs are converted to filename-only text before session history; the broader Docker/WSL2 active-run symptoms were not live-reproduced in this read-only pass.

## Solution Assessment

Yes for a focused repair, not for closure. The narrow maintainable fix is to preserve previewable media in the mirror append path using existing managed-media or MEDIA attachment conventions, without changing unrelated gateway refresh, typing, or heartbeat behavior.
