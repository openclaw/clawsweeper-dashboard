## Summary

Keep open. Current main has fixes or mitigations for the active-run refresh footgun, heartbeat transcript visibility, Telegram typing critical path, and related Docker/VPS latency paths, but the outbound delivery mirror still reduces media to filename text instead of persisting previewable media blocks for Control UI history.

## What This Changes

The issue asks for active-run Control UI/Gateway responsiveness to remain stable, Telegram typing and heartbeat UI noise to be cleaned up, and outbound Telegram media to mirror into Control UI as image previews.

## Best Possible Solution

Keep the landed responsiveness, heartbeat, and typing fixes, then add a narrow delivery-mirror fix that stores outbound media as structured managed image/media content in the session transcript so Control UI history can render authenticated previews after reload.
