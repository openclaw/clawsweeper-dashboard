The issue reports active-run regressions where Control UI reconnect/refresh can stall, Telegram typing has gaps, heartbeat content appears in visible chat history, and outbound media mirrored from Telegram appears as a filename instead of a previewable attachment.

## Best Possible Solution

Keep the landed refresh, heartbeat filtering, and typing mitigations, then change the outbound delivery-mirror transcript path so successfully delivered media is persisted as previewable managed media or attachment content rather than basename-only text.
