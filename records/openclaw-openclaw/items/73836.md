## Summary

Keep open. Current main has source-backed mitigations for refresh, heartbeat visibility, typing keepalive, and model-list responsiveness, but the outbound delivery mirror still stores successful media sends as basename-only text, so the reported Control UI media preview gap remains actionable.

## What This Changes

The issue reports active-run Control UI reconnect stalls, Telegram typing gaps, heartbeat transcript noise, and missing media previews in Control UI delivery mirrors after upgrading to v2026.4.26.

## Best Possible Solution

Keep the landed responsiveness mitigations, then update delivery mirroring so successful outbound image media is appended as previewable managed session content while preserving idempotency, dedupe, and safe non-image fallback behavior.
