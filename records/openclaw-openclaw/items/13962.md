## Summary

Keep open: current main and v2026.6.10 still lack per-mention model routing and per-group mention-context sizing. Existing skipped-message history buffering and `channels.modelByChannel` are adjacent, but they do not process unmentioned traffic through a background model or choose models by mention state, and that new path needs explicit product and security design.

## What This Changes

The issue requests WhatsApp/group mention-gating controls for separate mentioned/background models and a configurable prior-message context window when the bot is mentioned.

## Best Possible Solution

Design this as an explicit WhatsApp/group capability or trusted pre-gating plugin/API hook with safe defaults, clear model-routing precedence, bounded background turns, per-group context limits, schema/docs, and focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source and release inspection give high confidence that current main and v2026.6.10 lack the requested config keys and equivalent mention-state routing behavior.

## Solution Assessment

Unclear until maintainers choose the product and security contract. The suggested direct config is plausible, but the safer solution needs explicit routing-precedence, provider/auth, and group-context security design first.
