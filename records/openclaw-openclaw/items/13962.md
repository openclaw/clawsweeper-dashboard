## Summary

Keep open. Current main still lacks the requested per-mention model routing and per-group mention context sizing; the prior steipete review comment is still consistent with the current WhatsApp code, docs, and strict config schema.

## What This Changes

The issue requests per-group controls to route mentioned WhatsApp group messages to a premium model, unmentioned background traffic to a cheaper model, and cap the context window used when a mention triggers a reply.

## Best Possible Solution

Keep this open until WhatsApp/group handling either implements mention-state-aware model routing plus per-group context sizing, or exposes a deliberate trusted pre-gating plugin/API hook that can process unmentioned group messages safely.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source and docs inspection give high confidence that current main buffers skipped mention-gated group messages as pending context and lacks the requested config/model-routing surfaces.

## Solution Assessment

Unclear. The requested shape is plausible, but the best maintainable solution needs maintainer design for safety, precedence, and whether this belongs in the WhatsApp plugin pipeline or a more generic channel/plugin seam.
