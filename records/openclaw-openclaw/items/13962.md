## Summary

Keep open: current main still buffers skipped WhatsApp group messages for later context, but it does not route mentioned versus unmentioned messages to different models or expose the requested per-group context-size controls.

## What This Changes

Add per-group controls to route mentioned and unmentioned group messages to different models and include a configurable number of prior group messages when responding to mentions.

## Best Possible Solution

Add a deliberate WhatsApp/group pipeline design for mention-state-aware routing and per-group context sizing, or expose a trusted pre-gating room-event hook that preserves access-control, context-visibility, and model-precedence contracts.
