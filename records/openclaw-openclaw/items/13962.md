## Summary

Keep open. Current main still buffers unmentioned WhatsApp group messages as pending history only, does not route them through a background model, and has no per-group `mentionModel`, `backgroundModel`, or `mentionContextMessages` config.

## What This Changes

Request per-group controls to route mentioned and unmentioned group messages to different models and tune how much pending group context is included on a mention.

## Best Possible Solution

Design an explicit opt-in group/channel contract for mention-state-aware routing and per-group pending-context limits, then update WhatsApp config/schema/docs/tests with clear privacy and model-precedence rules.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source and docs inspection give high confidence that current main lacks the requested config fields and only buffers skipped mention-gated WhatsApp messages as pending context.

## Solution Assessment

Unclear. The suggested config is plausible, but the best maintainable solution needs maintainer design for privacy, routing precedence, and whether this belongs in WhatsApp-specific config or a shared channel/plugin contract.
