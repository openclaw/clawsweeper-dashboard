## Summary

Current main still lacks mention-state model selection or background model execution for skipped WhatsApp group messages; bounded pending context already exists, but per-group sizing and the routing capability remain product- and security-sensitive work with no canonical replacement item.

## What This Changes

Requests mention-state-specific model routing and configurable recent-message context for mention-gated group chats.

## Best Possible Solution

Keep the existing pending-history mechanism as the canonical context path, add per-group history sizing only if a distinct contract is justified, and design mention-state model selection through a typed generic routing/policy seam with explicit provider-exposure consent and precedence tests.
