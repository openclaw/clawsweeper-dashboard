## Summary

Keep open: current main still does not provide mention-state-aware model routing or per-group mention-context sizing for WhatsApp groups, and the remaining work needs product/security/config design rather than conservative cleanup.

## What This Changes

Requested per-group WhatsApp controls to route mentioned messages and background group chatter to different models and set a mention-specific context window.

## Best Possible Solution

Design this as an explicit WhatsApp/group capability or trusted plugin/API hook that preserves current safe defaults, defines model-routing precedence, and adds schema/docs/tests for background turns and per-group context limits.

## Solution Assessment

Unclear until maintainers choose the product/security contract. The direct suggested config is plausible, but the safer solution needs an explicit routing-precedence and group-context security design first.
