## Summary

Keep open. Current main still buffers unmentioned WhatsApp group messages as pending history only, does not route them through a background model, and does not expose per-group `mentionModel`, `backgroundModel`, or `mentionContextMessages` config.

## What This Changes

The issue requests per-mention model routing plus a configurable mention-context window for group chats.

## Best Possible Solution

Add a deliberate WhatsApp/group or generic channel design for mention-state-aware routing and per-group mention-context sizing, with schema, docs, tests, privacy guidance, and model-override precedence rules.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source and docs inspection give high confidence that current main lacks the requested config fields and only buffers skipped mention-gated WhatsApp messages as pending context.

## Solution Assessment

Unclear. The suggested config shape is plausible, but the best maintainable solution needs maintainer design for privacy, routing precedence, and whether the seam belongs in WhatsApp-specific config or a shared channel/plugin contract.
