- `impact:session-state`: The request changes how skipped group messages are retained and how much prior group context enters a mentioned turn.
- `impact:security`: Processing unmentioned group traffic through a background model changes the current trigger and content-visibility boundary for shared rooms.
- `impact:auth-provider`: The requested mention/background model controls directly affect provider/model selection and auth-profile behavior.

## Summary

Keep open: current main still does not provide per-mention/background model routing or per-group mention context sizing. WhatsApp does buffer skipped mention-gated messages as later context, but skipped unmentioned messages still return before model dispatch and the only shipped routing override is keyed by channel/session rather than mention state.

## What This Changes

The issue requests per-group controls for routing mentioned and unmentioned group messages to different models, plus a configurable mention-context window.

## Best Possible Solution

Design this as an explicit WhatsApp/group capability or trusted pre-gating plugin/API hook with safe defaults, clear model-routing precedence, bounded background turns, per-group context limits, schema/docs, and focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source and release inspection give high confidence that current main and v2026.6.10 lack the requested config keys and equivalent mention-state routing behavior.

## Solution Assessment

Unclear until maintainers choose the product and security contract. The suggested direct config is plausible, but the safer solution needs explicit routing-precedence, provider/auth, and group-context security design first.
