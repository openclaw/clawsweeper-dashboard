## Summary

Keep open: current main still lacks mention-state-aware model routing and background-model processing for skipped WhatsApp group messages. The existing implementation buffers skipped group messages as pending context, but the requested per-group config and routing semantics remain feature, config, product, and security design work.

## What This Changes

The issue requests per-mention model routing plus a configurable context window for mention-gated group chats.

## Best Possible Solution

Add an intentional WhatsApp or shared channel contract for mention-state routing and per-group context sizing, with schema, docs, privacy guidance, and focused gating/routing tests.
