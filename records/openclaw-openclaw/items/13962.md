## Summary

Current `main` still lacks the requested mention-state-aware model routing, background processing path for unmentioned WhatsApp group messages, and per-group mention context size control. This should stay open because it is a real feature/API request with security and routing precedence implications, not conservative cleanup to close.

## What This Changes

Request per-group controls that route mentioned and unmentioned group messages to different models and include a configurable number of prior group messages when a mention is processed.

## Best Possible Solution

Add a deliberate WhatsApp or shared channel contract for mention-state-aware routing and configurable mention-context sizing, with schema, docs, tests, privacy guidance, and model-override precedence rules.
