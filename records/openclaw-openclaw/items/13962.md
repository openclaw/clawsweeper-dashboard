## Summary

Keep open. Current main still treats mention-gated WhatsApp group misses as pending context only, lacks per-group mention/background model fields, and needs maintainer design for privacy and model-precedence semantics before implementation.

## What This Changes

The issue requests per-group routing of mentioned versus unmentioned group messages to different models plus a configurable N-message context window for mention-triggered group replies.

## Best Possible Solution

Add an explicit opt-in WhatsApp/group or generic channel design for mention-state-aware routing and per-group context limits, with schema, docs, tests, privacy guidance, and model-override precedence rules.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source, docs, and tests give high confidence that current main lacks the requested config fields and only buffers skipped mention-gated WhatsApp messages as pending context.

## Solution Assessment

Unclear. The suggested config shape is plausible, but the maintainable solution needs maintainer design for privacy, routing precedence, and whether the seam belongs in WhatsApp-specific config or a shared channel/plugin contract.
