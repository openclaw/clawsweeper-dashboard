## Summary

Keep open: current main and the latest release still lack mention-state-aware model routing, background processing for skipped WhatsApp group messages, and per-group mention context sizing; the issue also has product/security review labels rather than a close signal.

## What This Changes

The issue requests per-group configuration to route mentioned messages to one model, process unmentioned group traffic with a cheaper background model, and inject a configurable number of prior group messages when a mention triggers the bot.

## Best Possible Solution

Add an intentional WhatsApp or shared channel contract for mention-state routing and configurable mention-context sizing, with schema, docs, privacy guidance, and focused routing/gating tests.

## Reproduction Assessment

Not applicable; this is a feature/config request rather than broken existing behavior. Source and docs inspection on current main show the requested fields are absent and skipped mention-gated WhatsApp messages only enter pending history.

## Solution Assessment

Unclear. The suggested config shape is plausible, but the maintainable solution needs maintainer design for opt-in background processing, privacy boundaries, and model-routing precedence.
