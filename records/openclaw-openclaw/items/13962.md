## Summary

Keep open. Current main still implements pending-only WhatsApp group context and channel/session model overrides, but not mention-state-aware model routing or per-group mention context sizing; the prior maintainer/Codex comments are still consistent with the current source.

## What This Changes

The issue requests WhatsApp group controls for separate mentioned-message and background models plus a per-group context-window size when mention gating is enabled.

## Best Possible Solution

Add a WhatsApp-owned, documented mention-state routing contract with per-group context controls, backed by schema/types/docs and focused tests; only add a shared core model-selection seam if multiple channels need the same contract.

## Reproduction Assessment

Not applicable as a bug reproduction, but the feature gap is high-confidence by source inspection. With requireMention enabled, unmentioned WhatsApp group messages are stored and skipped before reply processing, and the proposed config fields are absent from the strict schema.

## Solution Assessment

Unclear. The requested direction is useful, but the exact implementation needs maintainer design for background-processing safety and model-routing precedence before it should become an automated fix PR.
