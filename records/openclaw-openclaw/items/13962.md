## Summary

Current main still buffers mention-gated WhatsApp group messages without invoking a background model, uses a shared account/channel-level history limit, and exposes no mention-state model or per-group mention-context controls. No canonical issue or PR now owns the same work, so this remains a valid product- and security-sensitive feature request rather than a cleanup close.

## What This Changes

Add separate model routing for mentioned versus unmentioned group messages and a configurable prior-message context window for mention-triggered turns.

## Best Possible Solution

Define a secure, plugin-owned mention-state routing contract with explicit model precedence and privacy semantics, then implement background processing and bounded pending context as one coherent path with the smallest necessary config surface.
