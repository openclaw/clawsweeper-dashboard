## Summary

Keep open: current main still does not implement mention-state-aware model routing or per-group mention context sizing for WhatsApp groups. The existing pipeline buffers skipped unmentioned group messages for later context, but it returns before message processing, and the strict WhatsApp config still lacks the requested `mentionModel`, `backgroundModel`, and `mentionContextMessages` controls; related early-hook and contextual-activation work is adjacent, not a complete substitute.

## What This Changes

The issue requests per-group model routing based on whether a group message mentioned the bot, plus configurable context-window sizing for mention-gated group replies.

## Best Possible Solution

Define a maintainer-approved WhatsApp/group contract for mention-state routing and per-group context sizing, including security, cost, precedence, config/schema/docs, and focused tests for skipped-message buffering, background processing, mentioned-message routing, and context limits.
