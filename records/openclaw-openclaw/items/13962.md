## Summary

Current main provides bounded pending context for mention-gated WhatsApp groups, but it still neither processes unmentioned messages through a background model nor chooses a model from mention state. No canonical replacement issue or PR owns the remaining product work, and the requested routing requires explicit privacy, precedence, and cross-channel ownership decisions.

## What This Changes

Add separate model routing for mentioned and unmentioned group messages plus configurable per-group context sizing for mention-triggered turns.

## Best Possible Solution

Define a generic typed channel-turn policy that carries prepared admission and mention facts into routing, supports explicitly opted-in background processing, and bounds pending context through the existing history abstraction; expose only the smallest cross-channel configuration surface justified by that contract.

## Reproduction Assessment

Not applicable; this requests new behavior rather than reporting broken established behavior. Source inspection confirms current main intentionally buffers skipped group messages without invoking a background model.

## Solution Assessment

No, the suggested configuration fields alone are not the best complete solution. Maintainers should first define the shared turn-policy owner, explicit privacy consent, routing precedence, and cross-channel applicability, then add only the necessary configuration.
