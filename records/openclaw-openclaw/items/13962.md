## Summary

Current main still buffers mention-gated WhatsApp group messages without running a background model, uses one account/channel-level pending-history limit, and has no mention-state model fields; no canonical issue or PR now owns the same work. This remains a valid but product- and security-sensitive enhancement rather than a cleanup close.

## What This Changes

Requests mention-state-aware model routing and configurable prior-message context for mention-gated group chats.

## Best Possible Solution

Introduce a typed mention-state turn policy at the existing WhatsApp/channel-to-model routing boundary, preserve all inbound authorization and tool restrictions, and let a per-group bounded pending-context override feed the existing history window while still honoring contextVisibility.

## Reproduction Assessment

Not applicable: this is a request for new behavior, not a failure of an existing documented contract. Source and docs provide a high-confidence current-behavior check showing that unmentioned messages are context-only and the requested controls are absent.

## Solution Assessment

No, the suggested standalone fields are not yet the best proven design. The maintainable solution should extend the canonical turn-routing and history contracts with explicit precedence and security semantics rather than create a parallel WhatsApp-only model pipeline.
