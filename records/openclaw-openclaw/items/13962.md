## Summary

Keep open: current main still lacks mention-state-aware model routing and per-group mention-context sizing for WhatsApp groups, and the remaining work is a product/security/config decision rather than conservative cleanup.

## What This Changes

The issue requests WhatsApp group mention behavior that can route mentioned messages to one model, optionally process non-mentioned group messages with a background model, and include a configurable recent-message context window when the bot is mentioned.

## Best Possible Solution

Define a secure, plugin-owned mention-state routing contract with explicit model precedence and privacy semantics, then implement background processing and bounded pending context through the smallest compatible config surface.
