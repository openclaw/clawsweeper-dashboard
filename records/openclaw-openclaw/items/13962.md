## Summary

Keep open. Current main still does not implement per-mention model routing or a per-group mention-context window; the WhatsApp pipeline only stores skipped unmentioned messages as pending group history, and model selection remains keyed by channel/group/session rather than mention state.

## What This Changes

Request per-WhatsApp-group controls that route mentioned messages and unmentioned background context to different models, plus a configurable context-message count for mention-gated groups.

## Best Possible Solution

Keep this open until WhatsApp/group handling deliberately supports mention-state-aware model routing plus per-group context sizing, or exposes a trusted pre-gating plugin/API hook that can process unmentioned group messages safely.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source and docs inspection give high confidence that current main buffers skipped mention-gated group messages as pending context and lacks the requested config/model-routing surfaces.

## Solution Assessment

Unclear. The requested config shape is plausible, but the best maintainable solution needs maintainer design for safety, precedence, and whether the behavior belongs in the WhatsApp plugin pipeline or a more generic channel/plugin seam.
