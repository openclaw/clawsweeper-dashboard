## Summary

Keep open: current main partially buffers skipped WhatsApp group messages for later context, but it still lacks mention-state-aware model routing, a background-model path for unmentioned messages, and per-group mention context sizing.

## What This Changes

Request per-group controls that route mentioned and unmentioned WhatsApp group messages to different models and include a configurable number of prior group messages when a mention is processed.

## Best Possible Solution

Add a deliberate WhatsApp/group pipeline design for mention-state-aware routing and per-group context sizing, or expose a trusted pre-gating room-event hook that preserves allowlist, context-visibility, and model-precedence contracts.
