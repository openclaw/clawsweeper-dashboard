## Summary

Keep open. Current main still does not implement mention-state-aware model routing, background-model processing for unmentioned WhatsApp group messages, or a per-group mention context window; the issue remains a real feature/API request rather than cleanup that can be closed.

## What This Changes

The issue requests per-group WhatsApp mention routing so mentioned messages can use one model, unmentioned background context another model, and mention-triggered replies can include a configurable number of prior group messages.

## Best Possible Solution

Add a deliberate WhatsApp or shared channel contract for mention-state-aware routing and configurable mention-context sizing, with schema, docs, tests, privacy guidance, and model-override precedence rules.

## Reproduction Assessment

Not applicable; this is a feature request, not a broken existing behavior report. Source and docs inspection on current main show the requested config fields are absent and skipped mention-gated WhatsApp messages only enter pending history.

## Solution Assessment

Unclear. The suggested config shape is plausible, but the maintainable solution needs maintainer design for privacy, opt-in background processing, routing precedence, and whether this belongs in WhatsApp-specific config or a shared channel/plugin seam.
