## Summary

Keep open: current main and v2026.5.12 still lack mention-state-aware model routing and background-model processing for skipped WhatsApp group messages. Current code already buffers pending group context, but the requested per-group mention context sizing and routing semantics remain a feature/config/security design decision.

## What This Changes

Request per-group WhatsApp controls for mention-state model routing and configurable context windows when mention-gated group messages trigger the bot.

## Best Possible Solution

Add an intentional WhatsApp or shared channel contract for mention-state routing and per-group context sizing, with schema, docs, privacy guidance, and focused gating/routing tests.
