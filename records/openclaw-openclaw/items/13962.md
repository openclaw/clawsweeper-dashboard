## Summary

Keep open. Current main still lacks mention-state-aware model routing, does not dispatch unmentioned WhatsApp group messages to a background model, and exposes no per-group mention-context sizing config; related open issues cover adjacent hooks/routing but not this exact product/API shape.

## What This Changes

Request per-group config for different models on mentioned versus unmentioned group messages and a configurable context window for mention-gated group replies.

## Best Possible Solution

Add a deliberate WhatsApp/group or shared channel contract for mention-state-aware routing and configurable mention-context sizing, with schema, docs, tests, privacy guidance, and model-override precedence rules.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source and docs inspection on current main show the requested config fields are absent and skipped mention-gated WhatsApp messages only enter pending history.

## Solution Assessment

Unclear. The suggested config shape is plausible, but the maintainable solution needs maintainer design for privacy, opt-in behavior, routing precedence, and whether this belongs in WhatsApp-specific config or a shared channel/plugin seam.
