## Summary

Current main and the latest release still lack mention-state-aware model routing and per-group mention context sizing. The request fits OpenClaw’s channel roadmap, but background processing of unmentioned messages and routing precedence require explicit maintainer product and security decisions before implementation.

## What This Changes

Add mention-state-aware model selection and configurable recent-message context for mention-gated group chats.

## Best Possible Solution

Keep mention gating and its configuration in the WhatsApp plugin, reuse canonical visible-history and model-routing contracts, and implement the capability only after maintainers define authorization, context filtering, and model-override precedence.

## Reproduction Assessment

Not applicable: this requests new routing and context capabilities rather than reporting broken established behavior. Source and release inspection confirm that the capability is currently absent.

## Solution Assessment

No, the suggested configuration alone is not yet the best complete solution because it leaves authorization, context filtering, and model-override precedence undefined; those contracts should be settled before implementing plugin-owned settings.
