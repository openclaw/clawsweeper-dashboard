## Summary

Keep open. Current main still lacks per-mention model routing and per-group mention-context sizing; skipped WhatsApp mention-gated group messages are buffered as pending context only, while model selection remains keyed by channel/conversation/session rather than mention state.

## What This Changes

The issue requests per-group controls for routing mentioned messages to one model, unmentioned background traffic to another model, and limiting how many skipped group messages are injected as context when the bot is mentioned.

## Best Possible Solution

Add a deliberate WhatsApp/group or generic channel design for mention-state-aware routing plus per-group context sizing, with schema, docs, tests, privacy guidance, and clear precedence rules.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source and docs inspection give high confidence that current main only buffers skipped mention-gated group messages and lacks the requested config/model-routing surfaces.

## Solution Assessment

Unclear. The suggested config shape is plausible, but the maintainable solution needs maintainer design for safety, routing precedence, and whether this belongs in the WhatsApp plugin path or a generic channel/plugin seam.
