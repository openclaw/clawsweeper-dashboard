## Summary

Keep open. Current main still lacks per-mention model routing and a per-group mention-context window; skipped WhatsApp group messages are buffered as pending context only, while model selection remains based on channel/group/session identity rather than mention state.

## What This Changes

Request per-group controls for routing mentioned and unmentioned WhatsApp group messages to different models and for limiting the context included when a mention triggers a reply.

## Best Possible Solution

Add a deliberate WhatsApp/group or generic channel design for mention-state-aware routing plus per-group context sizing, with schema/docs/tests and clear precedence and authorization rules.
