## Summary

Keep open. Current main still lacks mention-state-aware model routing and per-group mention-context sizing for WhatsApp groups: skipped unmentioned messages are buffered for pending context and then ignored, while model selection remains keyed to channel/group/session rather than whether the bot was mentioned. The prior issue-thread review also kept this open for the same remaining product gap, and the current source still matches that assessment.

## What This Changes

Add group mention-aware routing so mentioned messages can use a premium model, unmentioned group traffic can optionally use a cheaper background model, and mention-triggered replies can include a configurable amount of recent group context.

## Best Possible Solution

Add a WhatsApp-owned, documented mention-state routing contract with per-group context controls, plus focused tests and docs. If background processing of unmentioned group messages is supported, it should be gated deliberately and respect existing group authorization, visibility, routing, and broadcast semantics.

## Reproduction Assessment

Yes. The current gap is reproducible by inspection: with requireMention true, unmentioned WhatsApp group messages are stored and the handler returns before processMessage, and the proposed config fields are absent from the strict schema.

## Solution Assessment

Unclear. The user-facing goal is valid, but the exact fix needs maintainer design for safe background-model processing and routing precedence before this should be turned into an automated implementation PR.
