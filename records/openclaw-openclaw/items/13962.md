## Summary

Keep open. Current main still lacks mention-state-aware model routing, a background-model path for unmentioned WhatsApp group messages, and per-group mention-context sizing, so this remains a product/API feature request rather than cleanup that can be closed.

## What This Changes

Request per-group config that routes mentioned and unmentioned group messages to different models and controls how many skipped group messages are added as mention context.

## Best Possible Solution

Add a deliberate WhatsApp or shared channel contract for mention-state-aware routing and configurable mention-context sizing, with schema, docs, tests, privacy guidance, and model-override precedence rules.

## Solution Assessment
