## Summary

Keep open. Current main still lacks mention-state-aware model routing, background-model processing for unmentioned WhatsApp group messages, and per-group mention-context sizing; the open discussion already points to the same unresolved product/API gap.

## What This Changes

Request per-group controls to route mentioned and unmentioned group messages to different models and include a configurable number of skipped messages as context when `requireMention` is enabled.

## Best Possible Solution

Add a deliberate WhatsApp or shared channel contract for mention-state-aware routing and configurable mention-context sizing, with schema, docs, tests, privacy guidance, and model-override precedence rules.

## Reproduction Assessment

Not applicable because this is a feature request, not a broken existing behavior report. Source and docs inspection on current main show the requested config fields are absent and skipped mention-gated WhatsApp messages only enter pending history.

## Solution Assessment

Unclear. The suggested config shape is plausible, but the maintainable solution needs maintainer design for privacy, opt-in behavior, routing precedence, and whether this belongs in WhatsApp-specific config or a shared channel/plugin seam.
