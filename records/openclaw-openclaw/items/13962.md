## Summary

Keep open. Current main still lacks mention-state-aware model routing, does not dispatch skipped WhatsApp group messages to a background model, and has no per-group `mentionModel`, `backgroundModel`, or `mentionContextMessages` config; adjacent model-router and raw-inbound-hook work does not fully cover this request.

## What This Changes

The issue requests per-group routing that uses different models for mentioned versus unmentioned group messages plus a configurable mention-context window.

## Best Possible Solution

Add a deliberate WhatsApp/group or shared channel contract for mention-state-aware routing and configurable mention-context sizing, with schema, docs, tests, privacy guidance, and model-override precedence rules.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source inspection on current main verifies the requested config fields are absent and skipped mention-gated WhatsApp group messages only enter pending history.

## Solution Assessment

Unclear. The proposed config shape is plausible, but the maintainable solution needs maintainer design for privacy, opt-in behavior, routing precedence, and whether the surface belongs in WhatsApp config, shared channel config, or a trusted plugin/API seam.
