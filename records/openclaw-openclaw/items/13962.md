Keep open. Current main still lacks mention-state-aware model routing, background-model processing for unmentioned WhatsApp group messages, and a per-group mention context window; this remains a feature/API design request, not cleanup that can be closed.

## What This Changes

Request per-group mention-state model routing, cheaper background processing for non-mentioned group messages, and a configurable mention context window.

## Best Possible Solution

Add a deliberate WhatsApp or shared channel contract for mention-state-aware routing and configurable mention-context sizing, with schema, docs, tests, privacy guidance, and model-override precedence rules.
