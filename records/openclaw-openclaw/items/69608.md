## Summary

Keep open. Current main does not implement the shared daily-memory/session-summary model, and PR head still has a concrete path-boundary defect in remembered summary fallback that should be patched before merge.

## What This Changes

This PR centralizes daily-memory/session-summary classification and rewires startup context, session-memory, doctor, dreaming/REM, short-term promotion, docs, and runtime exports around shared helpers.

## Best Possible Solution

Keep the shared-helper direction, but constrain remembered-summary fallback to normalized in-workspace probe candidates or a deliberately documented migrated-path mechanism, with regression coverage for foreign absolute basename collisions before merge.

## Reproduction Assessment

Yes. A focused test can remember `2026-04-19-vendor-pitch.md` as a session summary in the current workspace, then call `isSessionSummaryDailyMemoryPath` with `/old/workspace/memory/2026-04-19-vendor-pitch.md`; PR-head code reaches the raw-input remembered fallback after producing no in-workspace probe candidates.

## Solution Assessment

No. The overall centralization is maintainable, but the current implementation is not the safest fix until remembered provenance stops trusting arbitrary raw path basenames outside the workspace boundary.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Constrain remembered fallback to in-workspace probes:** `src/memory-host-sdk/host/daily-session-summary-probes.ts:576`
  - body: When probe normalization blocks a foreign absolute path, this fallback still extracts the daily filename from the raw input and looks it up in the current workspace's recent-daily index. A persisted entry from another workspace can therefore inherit local session-summary provenance when basenames collide, causing durable memory to be skipped or purged. Only trust remembered provenance derived from normalized in-workspace probe candidates, or add an explicit migrated-path path that cannot match unrelated workspaces.
  - confidence: 0.89

## Security Review
