## Summary

Keep open. Current main does not implement the shared daily-memory/session-summary model, so the PR is not obsolete, but the latest PR head still appears to have a path-boundary data-integrity bug in the remembered-summary fallback for Windows-style absolute aliases.

## What This Changes

The PR adds shared daily-memory/session-summary helpers and rewires startup context, session-memory, doctor, dreaming, REM backfill, short-term promotion, bootstrap handling, docs, and scripts to use those rules.

## Best Possible Solution

Continue with the shared-helper approach, but first constrain remembered summary provenance so raw foreign Windows-style absolute paths cannot inherit current-workspace session-summary state by basename; then add a required changelog entry and rerun the targeted memory/startup checks plus the changed gate.

## Reproduction Assessment

Yes. A focused reproduction is to call isSessionSummaryDailyMemoryPath with workspaceDir on POSIX, filePath `C:/old/workspace/memory/2026-04-19-vendor-pitch.md`, no live probe candidates, and a current workspace recent-daily index that remembers `2026-04-19-vendor-pitch.md` as a session summary; the current PR-head code reaches the raw Windows-path remembered fallback.

## Solution Assessment

No. Centralizing the daily-memory/session-summary model is the right direction, but this implementation is not the safest version until remembered provenance is limited to normalized in-workspace candidates or a deliberately proven migrated-path case.
