## What This Changes

This PR adds Discord inbound queue caps and stale-job drops, coalesces Discord preview/media work, caches transcript idempotency checks, serializes Codex app-server writes with RPC timeouts, and adds targeted async session-store reads.

## Best Possible Solution

Land a tightened branch that cancels timed-out queued writes, documents and approves the Discord queue controls, preserves current main behavior during rebase, and includes redacted runtime proof.

## Reproduction Assessment

## Solution Assessment

No. The direction is plausible, but this is not the best mergeable form until timed-out queued RPCs cannot be sent later and the new Discord drop controls are documented and approved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Avoid sending timed-out queued RPCs:** `extensions/codex/src/app-server/client.ts:229-230`
  - body: The timeout is armed before the serialized stdin write has actually run. If an earlier write keeps the chain backpressured past the timeout, rejectPending removes the id, but the queued writeMessage still sends the request later, so the app-server can execute work after the caller saw it fail.
  - confidence: 0.88
- **[P2] Document the Discord queue drop controls:** `src/config/types.discord.ts:411-416`
  - body: This adds inboundWorker.maxPendingPerSession and maxQueuedAgeMs with drop/skip behavior, but the public Discord docs still describe only the listener queue contract and say queued agent turns are governed by lifecycle completion. Operators need the config paths, defaults, and drop behavior before enabling this.
  - confidence: 0.9
