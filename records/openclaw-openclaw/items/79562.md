## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Avoid sending timed-out queued RPCs:** `extensions/codex/src/app-server/client.ts:203-205`
  - body: The new timeout starts before the serialized `writeMessage` has actually written to stdin. If stdin stays backpressured past the timeout, `rejectPending` removes the id but the queued write still runs later, so the app server can execute a request after the caller already saw a timeout; start the timer after the write or cancel the send when the pending id is gone.
  - confidence: 0.86
- **[P2] Document the new Discord queue controls:** `src/config/types.discord.ts:411-416`
  - body: The diff adds user-facing `inboundWorker.maxPendingPerSession` and `maxQueuedAgeMs` drop semantics, but the Discord docs still describe the old queue contract and the PR has no changelog entry. Please update the public Discord/config docs and required `CHANGELOG.md` entry so users know how these knobs behave.
  - confidence: 0.9
