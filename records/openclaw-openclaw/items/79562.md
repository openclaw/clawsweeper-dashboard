## Best Possible Solution

Tighten the queued RPC cancellation path, document the public Discord queue controls and drop behavior, and require redacted real runtime proof before maintainer merge review.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Avoid sending timed-out queued RPCs:** `extensions/codex/src/app-server/client.ts:229-230`
  - body: The timeout starts before the serialized stdin write has actually run. If an earlier write keeps the chain backpressured past the timeout, `rejectPending` removes the id, but the queued `writeMessage` still sends the request later, so the app server can execute work the caller already saw fail.
  - confidence: 0.88

Users need the config paths, defaults, and drop behavior before this ships.
  - confidence: 0.9
