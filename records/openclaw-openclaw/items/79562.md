## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Do not add a blanket 60s Codex RPC timeout:** `extensions/codex/src/app-server/client.ts:203-206`
  - body: Several current Codex callers intentionally send long-running requests with only a lifecycle abort signal, for example `turn/start`. Applying `DEFAULT_CODEX_APP_SERVER_RPC_TIMEOUT_MS` whenever no timeout is passed makes normal turns, compaction, or thread work fail after 60 seconds even though the caller did not opt into that deadline. Keep timeouts at the specific operations that have a bounded protocol expectation.
  - confidence: 0.91
- **[P1] Preserve skipped-job cleanup when deactivating the Discord queue:** `extensions/discord/src/monitor/message-run-queue.ts:138-140`
  - body: This `deactivate()` path only clears `pendingBySession` before deactivating the run queue. Current main also runs cleanup for accepted but not-started jobs so replay keys are released and typing feedback is cleaned up; without that, shutdown or abort can leave queued Discord deliveries suppressed or stale.
  - confidence: 0.9
- **[P2] Do not persist transcript idempotency as a sidecar cache:** `src/config/sessions/transcript.ts:449-453`
  - body: Writing `${sessionFile}.idempotency.json` creates a second runtime state artifact next to transcripts that cleanup, migration, and storage ownership paths do not currently manage. Store this index through the owned session state model or keep it rebuildable in an existing managed store instead of adding a per-transcript sidecar.
  - confidence: 0.78
