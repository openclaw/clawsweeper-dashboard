## Summary

Keep this PR open: it is a focused fix for a current P1 OpenAI/ChatGPT subscription regression with strong live proof, but maintainer acceptance is still needed for the shared missing-content-type compatibility tradeoff.

## What This Changes

The PR relaxes the OpenAI SDK streamed-response guard so missing or empty content-type headers are allowed, while preserving rejection for present non-stream content types, and adds a ChatGPT Codex regression test.

## Best Possible Solution

Land this focused fix after maintainers accept the shared missing-header tolerance, or ask for a Codex-only narrowing if preserving custom-endpoint diagnostics is more important.

## Reproduction Assessment

Yes. Current main and v2026.6.1 reject an OK streamed response with a missing content-type once stream:true is detected, and the linked reports plus PR proof provide the live ChatGPT OAuth reproduction path.

## Solution Assessment

Yes, this is a narrow and maintainable regression fix if maintainers accept the shared guard semantics. The safer alternative is a Codex-only allowance, but that requires adding more context to the guard for a smaller diagnostic-preservation benefit.
