## Summary

Keep open. Current main still routes image-tool inputs through shared media optimization before the provider call and has no valid-under-cap original-image fallback; open PR #74243 explicitly remains the closing implementation candidate.

## What This Changes

Request that the image tool/shared media loader stop failing valid JPEG/PNG inputs with `Failed to optimize image` before the VLM provider call.

## Best Possible Solution

Land #74243 or an equivalent guard-aware `src/media/web-media.ts` fallback that returns the original only when it is valid, under cap, and not a HEIC/pixel-limit/unsafe guard failure.

## Reproduction Assessment

Yes. The report and comments provide concrete JPEG/PNG failures across LINE, WhatsApp, Telegram, macOS, and Linux, and current source still shows optimizer exceptions propagating before the provider call.

## Solution Assessment

Yes. A loader-level fallback is the narrowest maintainable fix because it can preserve the original MIME/content type while keeping oversized, unsafe, HEIC, and pixel-limit failures closed.
