## Summary

Keep open. Current main still lets image preprocessing fail before the configured VLM call, and the linked open PR #73451 remains the implementation candidate to review, replace, or close.

## What This Changes

The issue reports valid JPEG/PNG image inputs failing with `Failed to optimize image` in the shared media preprocessing path before the image model is called.

## Best Possible Solution

Review, land, or supersede #73451 with a guard-aware `loadWebMedia()` fallback that preserves MIME/content type and keeps HEIC conversion, pixel-limit, unsafe-input, and over-cap failures closed.

## Reproduction Assessment

Yes. The issue and comments provide concrete JPEG/PNG failures across LINE, WhatsApp, Telegram, macOS, and Ubuntu, and current source still shows optimizer exceptions propagating before the provider call.

## Solution Assessment

No. Current main is not the final fix; a loader-level fallback is safer than returning original bytes from `optimizeImageToJpeg()` because it can preserve MIME type and enforce existing safety guards.
