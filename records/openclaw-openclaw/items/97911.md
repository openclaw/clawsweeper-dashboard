## Summary

Keep open. The issue requests that `tools.deny: ["skill_workshop"]` remove Skill Workshop from direct tools, Codex deferred dynamic tools, `tool_search`, and injected developer guidance.

## Best Possible Solution

Add a focused Codex app-server regression, then make the dynamic-tool handoff use only the effective post-policy tool inventory for both developer instructions and native `tool_search`.
