## Summary

Keep open: current main and the latest release still only accept Claude CLI Keychain or `.claude/.credentials.json` credentials for the synthetic `claude-cli` auth path, while this issue is already paired with open fix PRs that should be reviewed and landed or rejected before the issue closes.

## What This Changes

The issue requests that Claude CLI `apiKeyHelper` auth from `.claude/settings.json` satisfy OpenClaw's pre-spawn auth gate for `claude-cli` runtime use.

## Best Possible Solution

Land one linked PR after review that adds a Claude CLI helper credential/marker through the existing Anthropic-owned auth seam, with focused regression coverage for runtime auth, status/doctor labels, profile sync, and non-secret marker handling.

## Reproduction Assessment

Yes. Source inspection shows that on Linux with no `.claude/.credentials.json`, the current resolver returns null because it does not read Claude settings, then synthetic auth returns undefined and provider auth can throw `missing-provider-auth`.

## Solution Assessment

Yes. The best fix is a narrow Anthropic/Claude CLI credential-presence repair in the existing synthetic-auth seam, not a new OpenClaw config option or a generic fallback.
