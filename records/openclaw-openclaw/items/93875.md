## What This Changes

This issue reports that successful Discord `message(action=send)` source replies in `message_tool_only` mode can be followed by a stale tool-failed warning from an earlier non-terminal tool error.

## Best Possible Solution

Review and land or reject the open closing PR; if it lands, close this issue as implemented and retire the duplicate fix attempts.

## Reproduction Assessment

Yes. Source inspection of current main shows the reported path: `message_tool_only` can set `deliveredSourceReplyViaMessageTool` while leaving `hasUserFacingAssistantReply` false, so a non-recoverable non-mutating `lastToolError` can append the stale warning.

## Solution Assessment
