## Best Possible Solution

Land one canonical narrow fix PR, preferably the earlier clean candidate https://github.com/openclaw/openclaw/pull/93906 if it remains viable, then close this issue and duplicate fix PRs from the merged fix.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence current-main path: successful `message_tool_only` delivery sets `deliveredSourceReplyViaMessageTool`, but that value is not counted as `hasUserFacingAssistantReply` before warning resolution.

## Solution Assessment

Yes. The best fix is the narrow gate alignment already proposed by the open PRs: count successful message-tool source delivery as user-facing reply evidence without changing config or delivery ownership.
