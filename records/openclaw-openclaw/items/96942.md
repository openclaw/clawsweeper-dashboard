## Summary

Keep this PR open but do not merge it as-is: the patch adds thread context to successful Telegram send logs, while the claimed problem is missing context in failure diagnostics, and the failure path still logs before any of the new success-log lines can run. The PR also lacks after-fix real behavior proof for a Telegram topic failure.

## What This Changes

This PR adds Telegram threadId/threadScope fields to text send success logs and adds a direct auto-reply test that asserts a forum thread is sent and logged.

## Best Possible Solution

Put the thread fields into the shared Telegram failure-log context for `sendTelegramWithThreadFallback`, keep success logging only if useful, and prove the rejected forum-topic path with a focused test plus redacted real/runtime log output.

## Reproduction Assessment

Yes, from source inspection: a rejected Telegram send flows through `withTelegramApiErrorLogging` before the added success log can run, and the existing test harness already models `message thread not found` failures. I did not run tests because this review is read-only.
