## Summary

Keep open: this maintainer-labeled PR is a focused candidate fix for a valid agent compaction retry bug, current main still has the 60-second idle predicate gap, and I found no blocking patch defect in the reviewed code path.

## What This Changes

The PR changes the embedded agent compaction retry aggregate wait to treat an active post-compaction retry model run as live work and adds focused regression coverage for active and idle retry paths.

## Best Possible Solution

Land this focused active-work predicate and regression coverage after maintainer review and CI, then let the linked bug close from the merged fix rather than replacing the idle guard with a broader fixed timeout.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: current main starts the aggregate timeout after compaction is no longer in flight even when the retry model run can still be streaming. The linked issue provides concrete 143-187 second provider trajectories that exceed the 60-second guard, but this review did not run a live 200K-token compaction session.

## Solution Assessment

Yes, this looks like the best narrow fix: it preserves the existing 60-second idle recovery guard and only extends while the session runtime reports active retry work. The competing constant/config timeout PRs are plausible but broader and can still discard active work if a valid retry exceeds the selected cap.
