## What This Changes

This PR routes Z.AI disabled-thinking `reasoning_content` chunks to visible text, preserves enabled-thinking chunks as hidden thinking, adds regression coverage, and records the fix in the changelog.

## Best Possible Solution

Land the scoped stream parser fix after exact-head checks finish green, keeping disabled-thinking Z.AI output visible while preserving hidden routing for enabled/preserved thinking.

## Reproduction Assessment

Yes. On current main, a Z.AI `openai-completions` stream chunk with empty `delta.content` and non-empty `delta.reasoning_content` is emitted as a thinking block, which leaves no visible text; the PR adds focused fixtures for the disabled and enabled-thinking paths.

## Solution Assessment

Yes. The latest patch uses the already-patched request payload plus Z.AI compat to distinguish disabled thinking from preserved thinking, which is narrower and safer than routing every Z.AI `reasoning_content` field as visible text.
