## Summary

Keep open. This PR fixes a real current-main Z.AI stream-routing bug, the latest diff is narrowly scoped to disabled-thinking GLM output, and I found no blocking review findings.

## What This Changes

This PR changes the OpenAI-completions stream parser to treat Z.AI disabled-thinking `reasoning_content` chunks as visible text, adds disabled/enabled-thinking regression tests, and adds a changelog fix entry.

## Best Possible Solution

Land the scoped stream parser fix after normal maintainer review, keeping disabled-thinking Z.AI output visible while preserving hidden routing for enabled/preserved thinking.

## Reproduction Assessment

Yes. On current main, a Z.AI `openai-completions` stream chunk with empty `delta.content` and non-empty `delta.reasoning_content` is classified as a thinking block, while visible delivery only includes text blocks.

## Solution Assessment

Yes. The PR uses the final patched request payload plus Z.AI compat to distinguish disabled thinking from preserved thinking, which is narrower than routing every Z.AI `reasoning_content` field as visible text.
