## Summary

Keep open. The PR targets a valid Z.AI empty-response regression, but the current diff makes every Z.AI `reasoning_content` stream delta visible, which is too broad for `/think on` or preserved-thinking sessions where that field is intentionally hidden reasoning.

## What This Changes

The PR adds a changelog entry, passes `compat.thinkingFormat` into the OpenAI completions stream reasoning parser, and tests that a Z.AI `reasoning_content` delta becomes a visible text block.

## Best Possible Solution

Land a scoped fix that converts Z.AI `reasoning_content` to visible text only for disabled-thinking answer fallback, while preserving hidden reasoning blocks for enabled and preserved-thinking sessions.

## Reproduction Assessment

Yes. The linked issue and added test provide a source-level reproduction for the empty visible reply on current main; a complementary Z.AI thinking-enabled fixture would reproduce the PR-introduced leak.

## Solution Assessment

No. Passing compat into the stream parser is a reasonable direction, but using `thinkingFormat === "zai"` alone is not the narrowest maintainable fix because it ignores the provider-owned thinking state.
