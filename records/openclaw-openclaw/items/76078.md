## Summary

Keep open for maintainer review. Current main still hides the reported Z.AI GLM-5 visible response as a thinking block, while the PR scopes visible routing to the disabled-thinking Z.AI path and preserves hidden thinking when enabled.

## What This Changes

This PR changes OpenAI completions stream parsing so Z.AI disabled-thinking `reasoning_content` deltas become visible text, adds disabled/enabled-thinking regression coverage, and records the fix in the changelog.

## Best Possible Solution

Land the scoped parser change after mergeability and exact-head checks are current, so disabled-thinking Z.AI GLM-5 replies are visible while preserved thinking remains hidden and replayable.

## Reproduction Assessment

Yes. On current main, a Z.AI completions stream chunk with `content: ""` and nonempty `reasoning_content` falls through to the fallback reasoning parser, becomes a `thinking` block, and is then excluded by user-facing text extraction.

## Solution Assessment

Yes. The PR uses the already-injected Z.AI `thinking: { type: "disabled" }` payload plus the existing compat signal, which is narrower than provider-wide routing and avoids exposing preserved thinking.
