## Summary

Keep open. This PR fixes a real current-main Z.AI GLM-5 stream routing bug with a narrow disabled-thinking gate, adds focused regression coverage and a changelog entry, and latest exact-head checks are green; I found no blocking review findings.

## What This Changes

The PR routes Z.AI disabled-thinking `reasoning_content` chunks through the OpenAI completions stream as visible text, adds disabled/enabled-thinking regression tests, and records the fix in the changelog.

## Best Possible Solution

Land the scoped stream parser fix so disabled-thinking Z.AI `reasoning_content` is visible to users while preserved thinking remains hidden and replayable.

## Reproduction Assessment

Yes. Current main with a Z.AI `openai-completions` stream chunk whose delta has `content: ""` and nonempty `reasoning_content` classifies that text as a thinking block, and visible extraction only includes text blocks.

## Solution Assessment

Yes. The PR uses the already-patched request payload plus the existing Z.AI compat flag, which is narrower than routing every Z.AI `reasoning_content` field as visible text.
