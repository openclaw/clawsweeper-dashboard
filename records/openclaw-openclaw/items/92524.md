## Summary

The proposed manager-boundary fix addresses the reported mixed-result loss, but it introduces two regressions: it removes current main’s fallback for overlapping vector-and-keyword hits when every weighted score is below `minScore`, and its final score sort undoes MMR ordering. The branch needs a narrow correction before merge; the other open PR is not a safe superseding target because it has the same fallback defect and weaker proof.

## What This Changes

The PR changes hybrid memory search selection to retain deduplicated keyword-only FTS matches below the configured minimum score alongside strict vector results, with regression coverage.

## Best Possible Solution

Extend rather than replace the existing selection policy: retain every keyword-backed merged result when the strict set is empty, and when strict results exist additionally retain deduplicated keyword-only entries while preserving the ordering already produced by temporal decay and MMR.

## Reproduction Assessment

Yes. An overlapping merged entry with both component scores, a combined score below `minScore`, and no strict peer is returned by current main but excluded by this PR; enabling MMR also shows that the new final score sort overrides the helper’s diversity order.

## Solution Assessment

No. The manager filtering boundary is appropriate, but the implementation must extend the existing fallback and preserve upstream ranking rather than introducing competing selection semantics.
