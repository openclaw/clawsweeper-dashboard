## Summary

Keep this PR open: the auth recovery direction is useful and source-reproducible, but the latest head still has a blocking auth-store persistence bug where a no-op locked update can fall back to saving the pre-lock caller snapshot outside the lock.

## What This Changes

The PR adds provider-scoped cooldown cleanup after direct/plugin auth refreshes, changes billing fallback probing to account for later auth availability, and adds focused auth/fallback regression tests.

## Best Possible Solution

Keep the same-auth billing probe and reauth cleanup direction, but remove the post-lock stale-save path and land it only as part of a coordinated auth-provider recovery decision with https://github.com/openclaw/openclaw/pull/87694.

## Solution Assessment
