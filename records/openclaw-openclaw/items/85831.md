## What This Changes

The issue requests that Control UI config saves succeed when `models.providers.openai.baseUrl` is omitted and the provider overlay relies on defaults.

## Best Possible Solution

Land a focused Control UI serialization fix that omits schema-derived empty optional strings for absent provider fields while preserving strict core validation for explicitly configured empty `baseUrl` values.

## Reproduction Assessment

Yes, at source level: a built-in provider overlay can omit `baseUrl`, the Control UI can save the materialized empty string, and `config.set` rejects that explicit empty value through the current schema. I did not run a live browser repro in this read-only pass.

## Solution Assessment
