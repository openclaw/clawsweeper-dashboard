## Summary

Keep open: current main still has the source-level Control UI round-trip path that can submit an explicit empty provider baseUrl, and the linked implementation PR is open and unmerged.

## What This Changes

The issue reports that Control UI config saves fail when an omitted OpenAI provider baseUrl is round-tripped as an explicit empty string instead of staying unset.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/85850, keeping the core provider baseUrl schema strict while omitting schema-derived empty optional strings during Control UI submit.

## Reproduction Assessment

Yes at source level: a built-in provider overlay may omit baseUrl, the snapshot/form path can carry the materialized empty string, and config.set rejects that explicit empty value. I did not run a live browser session in this read-only review.

## Solution Assessment

Yes: the maintainable fix is to normalize Control UI form submission so absent optional strings stay absent, not to loosen core validation for explicitly configured empty provider URLs.
