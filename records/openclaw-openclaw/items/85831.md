## What This Changes

Control UI should save provider config changes without serializing omitted built-in provider defaults, such as `models.providers.openai.baseUrl`, as invalid empty strings.

## Best Possible Solution

Land a focused Control UI serialization fix that omits empty optional constrained strings derived from schema defaults while keeping core provider validation strict.
