## Summary

Keep open: current main has Discord voice DAVE mitigations, but the issue points at a still-plausible bundled dependency failure. The latest stable `@discordjs/voice` in current main is still `0.19.2`, and its published source still treats `lastTransitionId === 0` as falsy in the decrypt-failure recovery branch.

## What This Changes

The issue requests fixing Discord voice receive after DAVE became mandatory by handling `UnencryptedWhenPassthroughDisabled` gracefully and treating transition id `0` as valid.

## Best Possible Solution

Keep this open until OpenClaw either consumes an upstream `@discordjs/voice` release that fixes the `lastTransitionId === 0` recovery path, or lands an approved plugin-local mitigation with live Discord voice proof.

## Reproduction Assessment

Yes from source, but not from a live Discord run. Current main uses `@discordjs/voice@0.19.2`, and that published source still has the `if (this.lastTransitionId)` branch that drops transition id `0` from recovery.

## Solution Assessment

Unclear. The reported dependency changes are plausible and narrow, but OpenClaw policy makes dependency patches/overrides maintainer-owned, so the safer path is maintainer review of upstream status or an explicitly approved local mitigation plus live voice validation.
