## Summary

Keep open: current main and v2026.5.12 include Discord voice DAVE mitigations, but the bundled stable @discordjs/voice source still has the reported transition-id-0 recovery bug and no canonical OpenClaw follow-up owns it.

## What This Changes

The issue asks OpenClaw to restore Discord voice reception under mandatory DAVE by handling UnencryptedWhenPassthroughDisabled packets and treating lastTransitionId === 0 as a valid first transition in @discordjs/voice.

## Best Possible Solution

Land or consume an approved @discordjs/voice fix that treats transition id 0 as valid and tolerates transition-time receive packets, then prove Discord voice reaches STT/realtime in a live DAVE-required room.

## Reproduction Assessment

Yes from source, but not from a live Discord run. Current main and v2026.5.12 use @discordjs/voice@0.19.2, whose published decrypt recovery branch still gates recovery on `if (this.lastTransitionId)` and therefore treats 0 as missing.

## Solution Assessment

Unclear. The reported dependency changes are plausible and narrow, but OpenClaw policy makes dependency patches or overrides maintainer-owned and the observable voice path still needs live Discord validation.
