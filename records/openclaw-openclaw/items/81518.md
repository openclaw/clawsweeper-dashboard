The issue reports Discord voice reception failing under mandatory DAVE encryption because passthrough-disabled decrypt errors and first transition id zero can leave the bot connected but unable to receive speech.

## Best Possible Solution

Keep this as the canonical Discord DAVE receive tracker until maintainers either patch or upgrade @discordjs/voice, or explicitly accept the current OpenClaw mitigation after live DAVE voice proof shows speech reaches STT and agent handling without decrypt-error floods.
