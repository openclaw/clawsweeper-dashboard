## Summary

Keep open: current main has Discord voice DAVE passthrough and rejoin mitigations, but the bundled stable `@discordjs/voice@0.19.2` still contains the reported falsy `lastTransitionId` recovery guard, so the dependency-level receive failure is not fully resolved.

## What This Changes

The issue asks OpenClaw to restore Discord voice receive under mandatory DAVE by tolerating transition-time unencrypted packets and treating `lastTransitionId === 0` as a valid first transition.

## Best Possible Solution

Consume an upstream `@discordjs/voice` release that fixes the DAVE recovery guard, or land an approved Discord-plugin-local mitigation with focused tests and live Discord voice proof.

## Reproduction Assessment

Partly: the dependency source path is high-confidence, because current main pins `@discordjs/voice@0.19.2` and that package still skips recovery when `lastTransitionId` is `0`. The full Discord voice failure was not live-reproduced in this review.

## Solution Assessment

Unclear. The reported upstream changes are plausible and narrow, but OpenClaw should either take a fixed upstream release or make an explicitly approved plugin-local mitigation with live voice validation.
