Keep open: current main has OpenClaw-side Discord DAVE mitigations, but the reported dependency failure still appears valid. OpenClaw still pins `@discordjs/voice@0.19.2`, and that published runtime still treats `lastTransitionId === 0` as falsy in decrypt-failure recovery.

## What This Changes

The issue asks OpenClaw to restore Discord voice reception under mandatory DAVE by addressing `@discordjs/voice` passthrough errors and first-transition-id-zero handling.
