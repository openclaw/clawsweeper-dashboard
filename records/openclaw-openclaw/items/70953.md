## Summary

Keep this open. Current main still returns model-visible `approval-pending` results for foreground gateway and node exec approvals while detached continuations wait for approve/deny/timeout, and the docs/tests still encode that async pending/follow-up contract.

## What This Changes

Request that foreground exec approval requests block or hard-stop the agent turn until approval is resolved instead of returning a pending result to the model.

## Best Possible Solution

Define one core foreground exec-approval turn-control contract, then update the gateway/node implementations, docs, and regression coverage for approval, denial, timeout, heartbeat, and follow-up delivery paths.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: configure gateway or node exec to require approval, call a foreground command, and current main returns `details.status = "approval-pending"` while an async continuation waits for the final decision.

## Solution Assessment

Unclear. Awaiting approval may address the reported flow, but current docs and tests intentionally encode the async pending/follow-up model, so maintainers should choose whether the new contract blocks the tool call or hard-stops/resumes the turn after resolution.
