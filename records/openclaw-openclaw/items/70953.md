## Summary

Keep this open. Current main still treats foreground gateway/node exec approvals as model-visible `approval-pending` results while detached continuations wait for approve/deny/timeout and send follow-up events later, so the requested turn pause or hard stop is not implemented.

## What This Changes

The issue asks foreground exec approval requests to pause the agent turn until approve, deny, or timeout is known.

## Best Possible Solution

Define one core foreground exec-approval turn-control contract, then update gateway/node implementation, docs, and regression coverage for approval, denial, timeout, heartbeat, and follow-up delivery paths.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: configure gateway or node exec to require approval, call a foreground command, and current main returns `details.status = "approval-pending"` while an async continuation waits for the final decision.

## Solution Assessment

Unclear. Awaiting the approval decision matches the issue request, but current docs and tests intentionally encode an async pending/follow-up model, so maintainers should choose whether the safer contract is blocking the tool call or hard-stopping/resuming the turn after approval resolution.
