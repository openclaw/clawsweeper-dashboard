## Summary

Keep open. Current main still implements foreground gateway/node exec approvals as model-visible `approval-pending` tool results with detached follow-up continuations, and the public docs/tests still encode that async contract rather than a paused agent turn.

## What This Changes

The issue requests that foreground exec approval requests suspend the agent turn until the approval is allowed, denied, or times out.

## Best Possible Solution

Define one foreground exec-approval turn-control contract, then update gateway/node exec, follow-up delivery, prompts, docs, and regression coverage for approval, denial, timeout, heartbeat, and audit paths.

## Reproduction Assessment

Yes, source-level. Configure gateway or node exec to require foreground approval and current main returns a tool result with `details.status = "approval-pending"` while a detached continuation waits for the decision.

## Solution Assessment

Unclear. Awaiting approval may solve the reported flow, but current docs and tests intentionally encode async pending/follow-up behavior, so maintainers need to choose the new pause, resume, timeout, and audit semantics before implementation.
