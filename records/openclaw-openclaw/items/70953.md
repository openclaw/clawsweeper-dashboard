## Summary

Keep open. Current main has partial inline-wait fixes for WebChat and native gateway approval channels, but the source, tests, and docs still preserve immediate `approval-pending` behavior for node-host approvals, explicit follow-up modes, and non-native/headless gateway paths, so the broad approval-scheduling issue is not implemented yet.

## What This Changes

The issue requests that foreground exec approvals block the active agent turn until approval, denial, or timeout is resolved before further tool calls run.

## Best Possible Solution

Keep this as the canonical approval-scheduling issue and design a security-reviewed fix that consistently blocks foreground gateway and node approvals while preserving explicit async, background, headless, unavailable-route, and timeout semantics.
