## Summary

Keep this open. Current main still exposes foreground exec approvals as model-visible `approval-pending` tool results while detached continuations wait for approve, deny, or timeout, and the docs/tests still describe that async follow-up contract.

## What This Changes

The issue requests that foreground exec approval requests pause the agent turn until approve, deny, or timeout is resolved.

## Best Possible Solution

Define one core foreground exec-approval turn-control contract, then update gateway/node implementations, prompts, docs, and regression coverage for approval, denial, timeout, heartbeat, and follow-up delivery paths.

## Reproduction Assessment

Yes. Configure gateway or node exec to require approval and call a foreground command; source inspection shows current main returns `details.status = "approval-pending"` while a detached continuation waits for the final decision.

## Solution Assessment

Unclear. Awaiting approval may address the reported flow, but current docs and tests intentionally encode the async pending/follow-up model, so maintainers should choose whether the new contract blocks the tool call or hard-stops and resumes the turn after resolution.
