## Summary

Keep this open. Current main has a partial inline-await path only for internal gateway approval calls, while channel-origin gateway and node exec approvals still return `approval-pending` and continue through detached follow-up handling; the remaining behavior needs maintainer product/security policy rather than cleanup closure.

## What This Changes

The issue requests that foreground exec approval requests pause the agent turn until approve or deny is resolved instead of returning `approval-pending` immediately.

## Best Possible Solution

Make an explicit maintainer decision on whether channel-origin exec approvals should block the turn, stay asynchronous with stronger queue/follow-up controls, or split behavior by surface and background mode, then update code, docs, and regression coverage together.

## Reproduction Assessment

Yes, from source. Channel-origin gateway and node approval paths still return `approval-pending` while a detached continuation waits on `exec.approval.waitDecision`; only internal gateway calls take the inline await branch.

## Solution Assessment

Unclear. Pausing approvals may be the right deterministic behavior, but current docs and tests intentionally encode asynchronous approval handling, so maintainers need to choose the policy boundary before a repair PR.
