## Label Justifications

## Summary

Keep open. Current main only awaits gateway approvals inline for the internal channel; channel-origin gateway approvals and node approvals still return `approval-pending` while a detached follow-up waits for the decision, so the central requested behavior remains unresolved.

## What This Changes

The issue requests that foreground exec approval requests pause the active agent turn until the approval is approved, denied, or times out.

## Best Possible Solution

Define one explicit approval scheduling contract: foreground interactive exec approvals should block the active turn until resolution or timeout, while background and non-resumable flows keep a documented follow-up path with regression coverage.

## Reproduction Assessment

Yes. Source inspection shows non-internal gateway and node approval paths still return `approval-pending` while a detached follow-up waits for the decision.

## Solution Assessment

No. The current main behavior is only a partial mitigation because inline waiting is limited to internal-channel gateway approvals and does not cover channel-origin or node foreground approvals.
