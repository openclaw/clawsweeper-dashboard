## Summary

Keep open. Current main and the latest release still return `approval-pending` for channel-origin gateway approvals and node-host approvals while a detached follow-up waits for the decision, so the central approval-pause request remains unresolved.

## What This Changes

The issue asks for foreground exec approval requests to pause the agent turn until the user approves, denies, or times out the command.

## Best Possible Solution

Define one explicit approval scheduling contract: foreground interactive exec approvals should block the active turn until resolution or timeout, while background and non-resumable flows keep a documented follow-up or deterministic-failure path with regression coverage.

## Reproduction Assessment

Yes. Source inspection shows non-internal gateway and node approval paths still return `approval-pending` while a detached follow-up waits for the decision.

## Solution Assessment

No for closure. Current main is only a partial mitigation because inline waiting is limited to internal-channel gateway approvals and does not cover channel-origin or node foreground approvals.
