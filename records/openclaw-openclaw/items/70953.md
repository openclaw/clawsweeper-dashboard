## Summary

Keep this open. Current main still returns foreground gateway/node exec approvals as model-visible `approval-pending` tool results while a detached continuation waits for approval, denial, or timeout, and the docs/tests still encode that async follow-up contract.

## What This Changes

The issue requests foreground exec approval requests to pause the agent turn until the user approves, denies, or the request times out.

## Best Possible Solution

Define one core foreground exec-approval turn-control contract, then update gateway/node implementations, prompts, docs, and regression coverage for approval, denial, timeout, heartbeat, and follow-up delivery paths.

## Reproduction Assessment

Yes, source-level. Configure gateway or node exec to require approval and call a foreground command; current main returns `details.status = "approval-pending"` while a detached continuation waits for the final decision.

## Solution Assessment

Unclear. Awaiting approval may address the reported flow, but current docs and tests intentionally encode the async pending/follow-up model, so maintainers should choose whether the new contract blocks the tool call, hard-stops and resumes the turn, or keeps async delivery with scheduler safeguards.
