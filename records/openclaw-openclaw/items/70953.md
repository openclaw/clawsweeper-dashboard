## Summary

Keep open. Current main and the latest release still preserve immediate `approval-pending` behavior for non-internal gateway approvals and node-host approvals, while the WebChat inline wait work is only a partial mitigation.

## What This Changes

The issue requests that foreground exec approval requests suspend the active agent turn until the approval is approved, denied, or times out.

## Best Possible Solution

Define one explicit approval scheduling contract: foreground interactive exec approvals should block the active turn until resolution or timeout, while background and non-resumable flows keep a documented follow-up or deterministic-failure path with regression coverage.

## Reproduction Assessment

Yes at source level: current tests and code show non-WebChat gateway/node approvals still return `approval-pending` while a detached continuation waits for the decision. I did not run a live channel repro in this read-only review.

## Solution Assessment

No for closure. Current main is only a partial mitigation because inline waiting is limited to internal-channel gateway approvals and does not cover channel-origin or node foreground approvals.
