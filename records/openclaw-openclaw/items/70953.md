## Summary

Keep this open. Current main still exposes required foreground exec approvals as immediate `approval-pending` tool results while a detached continuation waits for approval, denial, or timeout; changing that is a security-sensitive product contract decision, not a stale cleanup close.

## What This Changes

Request that foreground exec approval requests pause the agent turn until the user approves, denies, or the request times out.

## Best Possible Solution

Define one foreground exec-approval turn-control contract, then update gateway/node implementations, prompts, docs, and regression coverage for approval, denial, timeout, heartbeat, and follow-up delivery paths.

## Reproduction Assessment

Yes, source-level. Configure gateway or node exec to require approval and call a foreground command; current main returns `details.status = "approval-pending"` while a detached continuation waits for the final decision.

## Solution Assessment

Unclear. Awaiting approval may be the right user-visible flow, but current docs and tests intentionally encode the async pending/follow-up model, so maintainers should choose between blocking the tool call, hard-stopping and resuming the turn, or keeping async delivery with stronger scheduler safeguards.
