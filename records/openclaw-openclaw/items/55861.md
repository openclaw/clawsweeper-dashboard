## Summary

Keep open: the PR addresses a real Control UI Safari/sidebar clipping report, but the patch widens a shared rail variable in a way that also changes tablet drawer behavior and needs a small test/changelog cleanup before merge.

## What This Changes

The PR widens the Control UI collapsed nav rail variable from 78px to 96px and adds a browser test asserting collapsed content starts outside the nav rail.

## Best Possible Solution

Land a desktop-scoped Control UI layout fix that keeps collapsed desktop content clear of the nav rail, preserves tablet/mobile drawer widths, waits for settled layout in regression coverage, and records the user-facing fix in the changelog.

## Reproduction Assessment

Yes for the PR-introduced regression by source inspection: the widened base variable feeds tablet drawer mode at `max-width: 1100px`, while only `max-width: 768px` overrides it back to 78px. I did not live-run Safari for the original linked rendering bug in this read-only pass.

## Solution Assessment

No, not as written. A desktop-scoped rail-width change or equivalent Safari-safe CSS fix is narrower and avoids changing tablet drawer behavior while still covering the linked bug.
