## Summary

Keep open. The PR is a useful candidate for open #55850, but the current patch is too broad because it changes a shared rail-width variable that also controls the tablet drawer rail, not only the desktop collapsed grid.

## What This Changes

The PR changes `ui/src/styles/layout.css` to widen the shared collapsed navigation rail from 78px to 96px and adds a browser regression test comparing collapsed nav and content bounds.

## Best Possible Solution

Land a desktop-scoped Control UI layout fix that keeps collapsed desktop content clear of the sidebar rail, preserves the existing tablet/mobile collapsed drawer width, adds focused browser coverage for the settled desktop layout and responsive drawer boundary, and includes the required user-facing changelog entry.

## Reproduction Assessment

Yes for the PR-introduced regression: inspecting current CSS shows the 769-1100px tablet drawer uses the same `--shell-nav-rail-width` that the PR widens. The original Safari bug has concrete steps in #55850, but I did not live-run Safari in this read-only review.

## Solution Assessment

No, not as written. Widening the collapsed desktop rail may be the right direction, but changing the base variable is not the narrowest maintainable fix because it changes tablet drawer behavior too; scope the 96px value to desktop or override the drawer rail back to 78px.
