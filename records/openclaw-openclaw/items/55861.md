## Summary

Keep open. Current main still has the 78px collapsed rail and no regression coverage for the Safari overlap, but this PR’s base-variable change is too broad because it also widens the tablet drawer rail outside the desktop layout it intends to fix.

## What This Changes

The branch widens the Control UI collapsed nav rail from 78px to 96px and adds a browser test comparing collapsed nav and content bounds.

## Best Possible Solution

Land a desktop-scoped Control UI layout fix that keeps collapsed desktop content clear of the nav rail, preserves the existing tablet/mobile drawer width, adds focused settled-layout coverage, and records the user-facing fix in the changelog.

## Reproduction Assessment

Yes for the PR-introduced regression: current CSS inspection shows the tablet drawer uses the same rail variable that the PR widens. The original Safari bug has clear steps in #55850, but I did not run Safari in this read-only pass.

## Solution Assessment

No, not as written. Widening the collapsed desktop rail may be reasonable, but the 96px value should be scoped to desktop layout or overridden for drawer breakpoints instead of changing the base variable globally.
