## What This Changes

Telnyx and other generic voice-call webhook speech events should trigger the same inbound auto-response loop as Twilio media-stream transcripts.

## Best Possible Solution

Refresh and review the linked webhook-path auto-response PR, then close this issue only after that fix or an equivalent replacement lands.

Yes. Source inspection shows Telnyx `call.transcription` becomes `call.speech`, `processParsedEvents` sends it only to the manager, and the manager records it without invoking the agent response loop.

Yes. The narrow maintainable fix is to add the missing generic webhook auto-response handoff while preserving transcript-waiter and dedupe behavior; the open linked PR is the active implementation path.
