## Summary

Keep this issue open: current main and v2026.6.10 still lack the generic webhook speech auto-response handoff, and the open linked PR remains the active fix candidate.

## What This Changes

The issue requests that webhook-delivered final speech transcripts for Telnyx and other non-streaming voice-call providers trigger the same auto-response loop as media-stream transcripts.

## Best Possible Solution

Land the generic webhook speech auto-response handoff through https://github.com/openclaw/openclaw/pull/79336 or an equivalent replacement, with transcript-waiter and replay behavior preserved.

## Reproduction Assessment

Yes. Source inspection shows Telnyx call.transcription normalizes to call.speech, processParsedEvents only calls manager.processEvent, and manager speech handling records/listens without response generation; I did not run a live carrier call.

## Solution Assessment

Yes. The narrow maintainable fix is to add the missing webhook-server auto-response handoff after final call.speech while preserving transcript-waiter, dedupe, and replay semantics; the linked PR is the active implementation path.
