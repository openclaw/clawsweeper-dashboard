## Summary

Keep open: current main still lacks the provider-webhook auto-response handoff, and https://github.com/openclaw/openclaw/pull/79336 is an open same-author fix PR with supplied Telnyx proof and closing syntax for this issue.

## What This Changes

Route final Telnyx and other generic provider webhook speech transcripts into the voice-call auto-response loop after an inbound greeting.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/79336, or an equivalent voice-call plugin patch, while preserving waiter and replay safeguards.

## Reproduction Assessment

Yes. Source inspection shows Telnyx `call.transcription` becomes `call.speech`, then current main records the transcript and transitions to `listening` without invoking the response loop; the reporter also supplied two real Telnyx call observations.

## Solution Assessment

Yes. A provider-agnostic webhook-path handoff owned by the voice-call runtime is the narrow maintainable fix, gated so active transcript waiters and replays do not produce duplicate replies.
