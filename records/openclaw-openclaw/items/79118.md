## What This Changes

The issue asks for Telnyx/non-streaming provider `call.speech` webhook transcripts to trigger the same voice-call auto-response loop used by streaming transcripts.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/79336 or an equivalent voice-call plugin patch that routes final provider-webhook speech into the guarded auto-response handoff.

## Reproduction Assessment

Yes. A Telnyx `call.transcription` webhook becomes `call.speech`, then current main only records it through the manager and transitions to `listening`; no live carrier call was run in this review.

## Solution Assessment

Yes. The narrow maintainable fix is to add the response handoff at the voice-call webhook/runtime layer with waiter, dedupe, and transcript-persistence guards, matching the open PR direction.
