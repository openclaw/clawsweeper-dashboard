## Summary

Keep open: current main still lacks a native `openclaw tui --agent <id>` startup selector. Agent-scoped session keys, cwd workspace inference, remembered sessions, and post-launch `/agent <id>` switching are useful adjacent paths, but they do not satisfy the requested arbitrary-directory startup flag.

## What This Changes

The issue requests a native `openclaw tui --agent <id>` flag that starts the TUI against a configured non-default agent.

## Best Possible Solution

Add a typed `openclaw tui --agent <id>` startup selector with validation, documented precedence, focused TUI coverage, and CLI/TUI docs updates if maintainers accept the UX.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence the requested CLI surface is absent on current main: there is no TUI flag, no option field, and startup agent resolution only uses session key, cwd, or the default agent.

## Solution Assessment

Unclear until maintainers decide the CLI UX and precedence contract. If accepted, a first-class `--agent <id>` selector is the narrow direct solution; existing cwd inference, canonical session keys, remembered sessions, and `/agent` switching do not satisfy arbitrary-directory startup selection.
