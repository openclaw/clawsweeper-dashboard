- `P2`: This is a bounded TUI multi-agent usability improvement with meaningful workflow impact but no urgent outage or data-loss signal.
- `impact:session-state`: Startup agent selection controls which agent-scoped session and workspace context the TUI attaches to.

## Summary

Keep open: current main and the latest release still lack a native `openclaw tui --agent <id>` startup selector, and the existing session-key, cwd-inference, and post-launch `/agent` paths do not fully cover generic-directory startup.

## What This Changes

Add `openclaw tui --agent <id>` so a TUI session can start directly on a specific configured agent and use that agent's model, workspace, and context.

## Best Possible Solution

Add one validated TUI startup agent selector that feeds the existing agent/session initialization path, with documented precedence for explicit sessions, cwd inference, remembered sessions, and unknown IDs.

## Reproduction Assessment

Yes. Source inspection shows the TUI CLI has no `--agent` option, `TuiOptions` has no startup agent field, and `agent:ollama` is not parsed as an agent-scoped session key.

## Solution Assessment

Yes in direction, but not yet as an automation task. A native startup flag is the narrow user-facing solution, while maintainers still need to decide precedence, validation, and remembered-session semantics.
