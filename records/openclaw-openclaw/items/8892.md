## Summary

Keep open: current main still lacks a native `openclaw tui --agent <id>` startup selector. Existing canonical session keys, cwd workspace inference, remembered sessions, and post-launch `/agent <id>` cover adjacent flows, but not the requested arbitrary-directory startup flag.

## What This Changes

The issue requests adding `openclaw tui --agent <id>` so the TUI starts with a selected configured agent and uses that agent's model, workspace, and context.

## Best Possible Solution

If maintainers accept the UX, add a typed `--agent <id>` selector with validation, documented precedence, focused TUI coverage, and updates to the CLI/TUI docs.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection on current main gives high confidence the requested CLI surface is absent: no TUI flag, no option field, and startup resolution only uses session key, cwd, or the default agent.

## Solution Assessment

Unclear until maintainers decide the CLI UX and precedence contract. If accepted, a first-class `--agent <id>` startup selector is the direct solution; existing cwd inference, canonical session keys, remembered sessions, and `/agent` switching do not satisfy arbitrary-directory startup selection.
