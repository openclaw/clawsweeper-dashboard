## Summary

Keep open: current main still does not expose a native `openclaw tui --agent <id>` startup selector. The existing canonical session-key, workspace-inference, remembered-session, and post-launch `/agent` paths are useful adjacent behavior, but they do not implement the requested first-class flag.

## What This Changes

Request adds a native `openclaw tui --agent <id>` startup selector so TUI sessions can launch against a selected configured agent.

## Best Possible Solution

Add a typed `openclaw tui --agent <id>` selector with validation, documented precedence, focused TUI tests, and CLI/TUI docs updates.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main gives high confidence that the requested surface is absent: no CLI flag, no TUI option field, and startup agent resolution only from session key, cwd, or the default agent.

## Solution Assessment

Yes, a first-class startup selector is the narrow direct solution if maintainers accept the UX. Existing cwd inference, canonical session keys, remembered sessions, and `/agent` are useful but do not satisfy arbitrary-directory startup selection with `--agent <id>`.
