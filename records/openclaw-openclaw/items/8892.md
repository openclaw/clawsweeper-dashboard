## Label Justifications

- `impact:session-state`: Startup agent selection determines which agent-scoped session and workspace context the TUI attaches to.

## Summary

Current main and v2026.6.5 still lack a native TUI startup selector for choosing a configured agent from an arbitrary directory. Existing agent-scoped session keys, cwd workspace inference, remembered sessions, and post-launch switching are adjacent paths but do not satisfy the requested workflow.

## What This Changes

Add an `openclaw tui --agent <id>` startup option that selects the configured agent, model, workspace, and session context.

## Best Possible Solution

Add one validated startup agent selector that feeds the canonical agent/session initialization path, with documented precedence and focused coverage for explicit sessions, cwd inference, remembered sessions, unknown IDs, and both local and Gateway modes.

## Reproduction Assessment

Yes, from source inspection: current main exposes no explicit startup agent option, and the reported two-part session shorthand is not recognized as an agent-scoped key. This establishes the missing capability without needing a failing runtime test.

## Solution Assessment

Yes in direction, but the contract is incomplete. A native flag is the narrowest maintainable user-facing solution, while maintainers still need to decide its precedence over explicit sessions, cwd inference, and remembered sessions.
