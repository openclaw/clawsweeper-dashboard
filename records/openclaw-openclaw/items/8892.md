## Summary

Keep open. Current `main` and latest release still lack a native `openclaw tui --agent <id>` startup selector; the shipped alternatives are workspace-directory inference, canonical agent-scoped session keys, and post-launch `/agent`, which do not satisfy the arbitrary-directory startup flow requested here.

## What This Changes

The issue requests a native `openclaw tui --agent <id>` startup flag so a TUI session can route directly to a configured non-default agent.

## Best Possible Solution

Keep this issue open as the canonical request until maintainers choose and implement an additive startup selector with explicit precedence against existing session and workspace selection paths.

## Reproduction Assessment

Yes. Source inspection of `origin/main` shows no TUI `--agent` option, no startup-agent field in `TuiOptions`, and startup resolution limited to canonical session keys, cwd workspace inference, then the default agent.

## Solution Assessment

No, closing is not the best way to solve this issue. A native typed startup selector is still the narrowest maintainable solution for arbitrary-directory launches, but maintainers need to settle precedence first.
