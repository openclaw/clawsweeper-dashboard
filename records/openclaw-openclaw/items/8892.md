- `impact:session-state`: The startup agent determines the TUI session owner, remembered-session behavior, workspace context, and bootstrap context.
- `impact:auth-provider`: The selected agent controls model/provider routing and can select different auth-backed provider configuration.

## Summary

Keep open. Current main and the latest shipped tag still lack a native `openclaw tui --agent <id>` startup selector; canonical session keys, cwd workspace inference, and post-launch `/agent` switching are useful workarounds but do not satisfy the requested arbitrary-directory launch flow.

## What This Changes

Request adds a `--agent <id>` flag to `openclaw tui` so startup routes through a specific configured agent's model, workspace, and session context.

## Best Possible Solution

Add a typed `--agent <id>` startup path through the TUI CLI, options contract, startup resolver, validation, focused tests, and TUI docs, with documented precedence against existing session and workspace selection paths.

## Reproduction Assessment

Yes for the reported current behavior. Source and shipped-tag inspection show no `--agent` TUI option, no startup-agent field in `TuiOptions`, and startup resolution limited to canonical session keys, cwd workspace inference, then default agent.

## Solution Assessment

Yes, with maintainer-defined precedence. A native typed startup selector is the narrowest maintainable way to choose a non-default configured agent from arbitrary directories; the existing workspace workaround and post-launch `/agent` command do not cover that launch case.
