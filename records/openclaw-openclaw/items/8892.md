## Summary

Keep open: current main still lacks a native `openclaw tui --agent <id>` startup selector, and the existing canonical session-key, cwd workspace inference, remembered-session, and post-launch `/agent` paths do not cover selecting a non-default agent from an arbitrary directory at launch.

## What This Changes

Add an `openclaw tui --agent <id>` startup flag to choose which configured agent handles the TUI session.

## Best Possible Solution

Add a typed `openclaw tui --agent <id>` selector with validation, documented precedence, focused TUI coverage, and CLI/TUI docs updates.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence the requested surface is absent: no CLI flag, no TUI option field, and startup agent resolution only from session key, cwd, or the default agent.

## Solution Assessment

Yes, a first-class startup selector is the narrow direct solution if maintainers accept the UX. Existing cwd inference, canonical session keys, remembered sessions, and `/agent` are useful but do not satisfy arbitrary-directory startup selection with `--agent <id>`.
