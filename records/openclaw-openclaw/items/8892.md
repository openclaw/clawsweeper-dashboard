## Summary

Keep open: current main and v2026.6.6 still do not expose a native TUI `--agent` startup selector; the existing session-key, workspace-inference, and post-launch switching paths do not satisfy arbitrary-directory startup. Because adding the flag defines a CLI contract plus precedence and validation behavior, this needs maintainer product review rather than an automated fix PR.

## What This Changes

Request to add `openclaw tui --agent <id>` so terminal sessions can start directly under a non-default configured agent from arbitrary directories.

## Best Possible Solution

Add one validated TUI startup agent selector that feeds the existing agent/session initialization path, with documented precedence for explicit sessions, cwd inference, remembered sessions, and unknown IDs.

## Reproduction Assessment

Yes. Source inspection shows the TUI CLI has no `--agent` option, `TuiOptions` has no startup agent field, and `agent:ollama` is not parsed as an agent-scoped session key.

## Solution Assessment

Yes in direction, but not yet as an automation task. A native flag is the narrow user-facing solution, while maintainers still need to decide precedence, validation, and remembered-session semantics.
