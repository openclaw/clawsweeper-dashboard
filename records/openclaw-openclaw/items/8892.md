## Summary

Current main and v2026.6.5 still lack a native startup selector for choosing an agent from an arbitrary directory. Agent-scoped session keys, workspace inference, the in-TUI agent picker, and the workspace-routing work in https://github.com/openclaw/openclaw/pull/40176 are useful adjacent paths, but none provides `openclaw tui --agent <id>`; this issue remains the canonical request.

## What This Changes

Add an `openclaw tui --agent <id>` startup option that selects the configured agent, model, workspace, and context for the TUI session.

## Best Possible Solution

Add a validated startup agent id to the TUI option contract and use it as one canonical selection input with explicitly documented precedence, while preserving all current behavior when the option is omitted.

## Reproduction Assessment

Not applicable as a bug reproduction. Source and latest-release inspection provide a high-confidence capability check: the requested option is absent, while only agent-scoped session keys, workspace inference, defaults, and post-launch switching exist.

## Solution Assessment

Yes in direction: a native CLI selector is the narrowest user-facing solution for arbitrary directories. The exact precedence and remembered-session semantics need maintainer agreement before implementation.
