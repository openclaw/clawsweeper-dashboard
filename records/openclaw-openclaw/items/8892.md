## Summary

Keep open. Current main still has no native `openclaw tui --agent <id>` startup selector: the CLI does not register or forward an agent option, TUI startup resolution only uses canonical agent-scoped session keys, CWD workspace inference, then the default agent, and the public docs still document only those alternatives plus post-launch agent selection.

## What This Changes

Request to add an `openclaw tui --agent <id>` startup flag that selects a configured agent's model, workspace, and context instead of always starting from the default agent outside agent workspaces.

## Best Possible Solution

Add a native typed startup agent selector for the TUI: parse and normalize `--agent <id>` in the CLI, carry it through TUI options, make it override CWD inference while detecting conflicts with canonical agent-scoped `--session` values, validate it against the Gateway/local agent list when available, and document the new startup behavior.

## Reproduction Assessment

Yes. The source-level reproduction is high-confidence: current main has no `--agent` option in `registerTuiCli`, no TUI option field for a startup agent id, and no explicit agent branch in `resolveInitialTuiAgentId`.

## Solution Assessment

Yes. A dedicated `--agent <id>` startup option is the narrowest maintainable fix because it preserves existing session-key and workspace inference behavior while solving the arbitrary-directory startup use case that workarounds do not cover.

## Repair Work Prompt

Implement native `openclaw tui --agent <id>` support for https://github.com/openclaw/openclaw/issues/8892. Keep the change narrowly scoped to TUI startup routing. Register `--agent <id>` in `src/cli/tui-cli.ts`, normalize and reject blank values, extend `TuiOptions`/`RunTuiOptions` with a typed startup agent id, and update `resolveInitialTuiAgentId`/startup application so `--agent` selects the initial agent from arbitrary directories. Define deterministic precedence: canonical `--session agent:<id>:<key>` and `--agent <id>` may agree; if they conflict, fail with a clear user-facing error rather than silently choosing one. A bare `--session <key>` with `--agent <id>` should use that agent's session key. `--agent` should override CWD workspace inference. Validate against the available agent list when it is available, and surface an actionable error for unknown agents. Preserve existing `/agent <id>`, agent picker, workspace inference, and agent-scoped session behavior. Update focused TUI tests, add CLI option coverage if no existing test covers the parser, update `docs/cli/tui.md` and `docs/web/tui.md`, and add an active `CHANGELOG.md` user-facing enhancement entry crediting the reporter (`Thanks @nomadonwheels196`) unless maintainers decide changelog is not needed. Do not introduce bundled-agent ids into core and do not change unrelated session routing behavior.
