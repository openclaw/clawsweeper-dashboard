## Summary

Keep open: current main still has no native `openclaw tui --agent <id>` startup selector, and the existing session-key, cwd inference, remembered-session, and in-TUI switching paths do not cover selecting an agent from an arbitrary directory at launch.

## What This Changes

Request a native `openclaw tui --agent <id>` flag that starts the TUI already routed to a configured non-default agent.

## Best Possible Solution

Add a typed `openclaw tui --agent <id>` startup selector with validation, documented precedence, focused TUI tests, and docs updates for the CLI and TUI guide.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main gives high confidence the requested surface is missing: no CLI flag, no TUI option field, and startup agent resolution only from session key, cwd, or default agent.

## Solution Assessment

Yes, a first-class TUI startup selector is the narrow direct solution for this workflow. Existing cwd inference and canonical session keys are useful workarounds, but they do not let users start from arbitrary directories with a selected agent.
