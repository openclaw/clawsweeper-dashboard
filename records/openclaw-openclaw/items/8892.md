## Summary

Keep open. Current main still lacks a native `openclaw tui --agent <id>` startup selector; existing `--session agent:<id>:...`, CWD workspace inference, and post-launch `/agent` switching are useful but do not satisfy the requested generic-directory startup flow.

## What This Changes

The issue requests a native `openclaw tui --agent <id>` flag to start the TUI with a selected configured agent rather than the default agent.

## Best Possible Solution

Add a typed TUI startup agent selector, validate it against configured agents, preserve existing session/CWD behavior, and document the precedence with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence reproduction path: there is no TUI `--agent` option, no TUI option field to carry it, and no resolver branch that can select a startup agent directly.

## Solution Assessment

Yes. A dedicated `--agent <id>` startup option is the narrowest maintainable solution because it leaves session keys, CWD workspace inference, and post-launch `/agent` switching intact while covering the generic-directory startup use case.
