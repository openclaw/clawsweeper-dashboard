## Summary

Keep open: current main still has no native `openclaw tui --agent <id>` startup selector. Existing agent-scoped session keys, cwd workspace inference, remembered sessions, and post-launch `/agent <id>` are adjacent mechanisms, but they do not satisfy the requested arbitrary-directory startup flag.

## What This Changes

Request adds `openclaw tui --agent <id>` so a TUI session can start with a chosen configured agent, model, workspace, and context.

## Best Possible Solution

If maintainers accept the UX, add a typed `--agent <id>` selector with validation, documented precedence, focused TUI coverage, and CLI/TUI docs updates.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence the requested CLI surface is absent on current main: no TUI flag, no option field, and startup resolution only uses session key, cwd, or the default agent.

## Solution Assessment

Unclear until maintainers decide the CLI UX and precedence contract. If accepted, a first-class `--agent <id>` selector is the narrow direct solution; existing cwd inference, canonical session keys, remembered sessions, and `/agent` switching do not satisfy arbitrary-directory startup selection.
