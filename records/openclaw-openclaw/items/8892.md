## Summary

Keep open: current main still lacks a native `openclaw tui --agent <id>` startup selector, and the existing session-key, cwd inference, remembered-session, and post-launch `/agent` paths do not cover selecting a non-default agent from an arbitrary directory at launch.

## What This Changes

Add `openclaw tui --agent <id>` so a TUI session can start directly under a configured non-default agent.

## Best Possible Solution

Add a typed `openclaw tui --agent <id>` selector with validation, documented precedence, focused TUI tests, and CLI/TUI docs updates.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main gives high confidence the requested surface is absent: no CLI flag, no TUI option field, and startup agent resolution only from session key, cwd, or default agent.

## Solution Assessment

Yes, a first-class startup selector is the narrow direct solution if maintainers accept the UX. Existing cwd inference, canonical session keys, remembered sessions, and `/agent` are useful but do not satisfy arbitrary-directory startup selection.
