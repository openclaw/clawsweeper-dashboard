## Summary

Keep open: current main still lacks a native `openclaw tui --agent <id>` startup selector. The issue discussion and related items show useful adjacent paths, but cwd workspace inference, canonical `agent:<id>:...` sessions, remembered sessions, and post-launch `/agent <id>` do not satisfy starting from an arbitrary directory with an explicit agent flag.

## What This Changes

Add a native `openclaw tui --agent <id>` flag so the TUI starts against a configured non-default agent.

## Best Possible Solution

If maintainers accept the UX, add a typed `openclaw tui --agent <id>` selector with validation, documented precedence, focused TUI coverage, and CLI/TUI docs updates.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence the requested CLI surface is absent on current main: there is no TUI flag, no option field, and startup resolution only uses session key, cwd, or the default agent.

## Solution Assessment

Unclear until maintainers decide the CLI UX and precedence contract. If accepted, a first-class `--agent <id>` selector is the narrow direct solution; existing cwd inference, canonical session keys, remembered sessions, and `/agent` switching do not satisfy arbitrary-directory startup selection.
