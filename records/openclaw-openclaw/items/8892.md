## Summary

Keep open. Current main still lacks a native TUI startup agent selector; the existing CWD workspace inference, canonical `agent:<id>:...` session keys, and post-launch `/agent` command are useful adjacent behavior but do not satisfy launching from an arbitrary directory with `openclaw tui --agent <id>`.

## What This Changes

The issue requests a native `openclaw tui --agent <id>` startup selector so TUI sessions can start with a configured non-default agent.

## Best Possible Solution

Add a typed `--agent <id>` startup selector for `openclaw tui`, validate it against configured agents, preserve existing session/CWD behavior, document precedence, and cover it with focused TUI CLI/resolver tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: current main has no `--agent` option in `registerTuiCli`, no TUI option field for a startup agent id, and no explicit startup-agent branch in `resolveInitialTuiAgentId`.

## Solution Assessment

Yes. A dedicated `--agent <id>` startup option is the narrowest maintainable solution because it leaves session keys, CWD workspace inference, remembered sessions, and post-launch `/agent` switching intact while covering arbitrary-directory startup.
