## Summary

Keep this open: current main still lacks a native `openclaw tui --agent <id>` startup selector. The existing canonical session-key path, cwd workspace inference from https://github.com/openclaw/openclaw/pull/40176-adjacent work, remembered-session restore, and post-launch `/agent <id>` command are useful but do not satisfy selecting a configured agent from an arbitrary directory at TUI startup; https://github.com/openclaw/openclaw/issues/19247 was closed as a duplicate, so this remains the canonical request.

## What This Changes

Request adding `openclaw tui --agent <id>` so TUI startup routes to a configured non-default agent, model, workspace, and bootstrap context.

## Best Possible Solution

Add a typed TUI startup agent selector with validation, explicit precedence, focused TUI tests, and matching public TUI CLI/guide docs.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: current main registers no `--agent` flag, has no TUI option field for a startup agent id, and routes startup through session parsing or cwd inference only.

## Solution Assessment

Yes, if maintainers accept the new CLI mode. A native typed `--agent <id>` startup selector is the narrowest maintainable direction; it should not be modeled as the invalid `--session agent:<id>` shorthand.
