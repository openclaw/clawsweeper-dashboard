## Summary

Keep open. Current main still lacks a native `openclaw tui --agent <id>` startup selector; existing support for agent-scoped session keys, CWD workspace inference, and post-launch `/agent` switching does not cover the requested arbitrary-directory startup flow.

## What This Changes

The issue requests a native `openclaw tui --agent <id>` startup selector so the TUI can start directly against a configured non-default agent.

## Best Possible Solution

Add a typed TUI startup agent selector that validates the requested agent, preserves existing session/workspace behavior, and documents the new precedence.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: current main has no `--agent` option in `registerTuiCli`, no TUI option field for a startup agent id, and no explicit agent branch in `resolveInitialTuiAgentId`.

## Solution Assessment

Yes. A dedicated `--agent <id>` startup option is the narrowest maintainable solution because it leaves session keys, CWD workspace inference, and post-launch `/agent` switching intact while covering the generic-directory startup use case.

## Repair Work Prompt

Implement a native `openclaw tui --agent <id>` startup selector for https://github.com/openclaw/openclaw/issues/8892. Treat this as a user-facing enhancement, not a bug fix. Add the CLI option in `src/cli/tui-cli.ts`, carry a typed startup agent id through `TuiOptions`, preserve it in `src/tui/tui-launch.ts`, and update `resolveInitialTuiAgentId`/`runTui` so an explicit `--agent` selects the starting agent from arbitrary directories. Define and test precedence against canonical `--session agent:<id>:<session>` and CWD workspace inference; prefer an explicit conflict/error if `--agent` and an agent-scoped `--session` name different agents. Validate/normalize the agent id against the available configured/Gateway agent list where the current TUI startup architecture can do so without adding a broad core seam. Add focused tests for CLI forwarding, startup resolution, relaunch argument preservation, and current workaround preservation. Update `docs/cli/tui.md`, `docs/web/tui.md`, and `CHANGELOG.md`. Do not change the existing canonical `agent:<id>:<session>` session-key contract or remove CWD workspace auto-discovery.
