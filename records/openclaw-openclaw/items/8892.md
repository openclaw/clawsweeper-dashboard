## Summary

Keep open: current `main` still does not provide a native TUI startup agent selector, and the existing session-prefix, cwd workspace inference, and post-launch `/agent` paths do not cover the requested generic-directory workflow.

## What This Changes

The issue requests a native `openclaw tui --agent <id>` flag that starts the TUI against a configured non-default agent.

## Best Possible Solution

Implement a typed TUI startup agent id path that validates a configured agent and documents precedence across explicit session keys, cwd inference, and remembered sessions.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current `main` shows the missing surface directly: no `--agent` CLI option, no `TuiOptions` field, and startup resolution only from canonical session key, cwd, or default agent.

## Solution Assessment

Yes, a first-class `openclaw tui --agent <id>` startup selector is the direct solution for this workflow. The existing cwd inference and post-launch `/agent` command are useful but do not let users start from arbitrary directories with a selected agent.
