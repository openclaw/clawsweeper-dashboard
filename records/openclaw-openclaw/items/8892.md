## Summary

Keep this open: current main still lacks a native `openclaw tui --agent <id>` startup selector. The existing paths cover canonical agent-scoped session keys, workspace-directory inference, and post-launch `/agent` switching, but they do not satisfy the requested generic-directory startup flag.

## What This Changes

The issue requests a native `openclaw tui --agent <id>` flag so a TUI session can start directly on a configured non-default agent.

## Best Possible Solution

Add a typed TUI startup agent selector with validation and documented precedence, then cover it in focused TUI tests and the public TUI CLI/guide docs.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path for the missing capability: current main registers no `--agent` flag and has no TUI option field to carry a startup agent id.

## Solution Assessment

Yes, if maintainers accept the behavior. A native typed `--agent <id>` startup selector is the cleanest fit; it should not be modeled as the invalid shorthand `--session agent:<id>` and needs explicit precedence rules.
