## Summary

Keep this open: current main still lacks a native `openclaw tui --agent <id>` startup selector. Existing canonical session keys, cwd workspace inference, remembered-session restore, and post-launch `/agent <id>` are useful adjacent paths, but they do not satisfy selecting a configured agent from an arbitrary directory at TUI startup.

## What This Changes

The issue requests a native `openclaw tui --agent <id>` flag that starts the TUI on a selected configured agent instead of the default agent.

## Best Possible Solution

Add a typed TUI startup agent selector with validation and documented precedence, then cover it in focused TUI tests and the public TUI CLI/guide docs.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path for the missing capability: current main registers no `--agent` flag, has no TUI option field to carry a startup agent id, and routes startup through session parsing or cwd inference only.

## Solution Assessment

Yes, if maintainers accept the new CLI mode. A native typed `--agent <id>` startup selector is the narrowest maintainable direction; it should not be modeled as the invalid `--session agent:<id>` shorthand and needs explicit precedence rules.
