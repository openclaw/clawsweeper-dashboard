## Summary

Keep open: current main still lacks a native TUI startup agent selector, and the existing session-prefix, cwd workspace inference, and post-launch agent switching paths do not cover the requested arbitrary-directory `openclaw tui --agent <id>` workflow.

## What This Changes

Request a TUI startup `--agent <id>` flag so `openclaw tui` can start with a configured non-default agent from arbitrary directories.

## Best Possible Solution

Add a typed `openclaw tui --agent <id>` startup selector that validates the configured agent and documents/tests precedence against explicit agent session keys, cwd inference, and remembered sessions.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main gives high confidence that the requested surface is missing: no CLI flag, no TUI option field, and startup agent resolution only from session key, cwd, or default agent.

## Solution Assessment

Yes, a first-class TUI startup selector is the narrow direct solution for this workflow. Existing cwd inference and canonical session prefixes are useful context, but they do not let users start from arbitrary directories with a selected agent.
