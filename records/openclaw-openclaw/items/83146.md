## Summary

Keep open. The fail-closed registry write is the right direction, but the latest orphan-listing path ignores configured session stores and the supplied real proof only covers strict registration, not the new orphan listing behavior.

## What This Changes

This PR adds a strict initial subagent registry persistence path, rolls back registration on durable-write failure, and adds orphaned child-session reconstruction to subagent listing tests and code.

## Best Possible Solution

Thread the runtime config or resolved session.store path through orphan reconstruction, keep the strict initial persistence fix, and require after-patch proof that both the fail-closed path and orphan list output work before merge.

## Reproduction Assessment

Yes for the narrow invariant: current main’s source shows durable registry write failures are swallowed after in-memory registration. I did not run the full QQ or embedded-channel multi-subagent scenario in this read-only review.

## Solution Assessment

No as submitted. The strict initial write is a narrow maintainable fix, but the orphan-listing fallback should honor the existing session.store config contract before this is safe to merge.
