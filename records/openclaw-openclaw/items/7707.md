## Summary

Keep open: current main and v2026.5.22 still have coarse memory source labels, session visibility filtering, Active Memory untrusted framing, and Memory Wiki provenance, but not durable memory trust metadata, trust-policy config, review/expiration gates, or trust-weighted retrieval.

## What This Changes

Request durable trust tags for memory entries based on origin, with policy-driven expiration, review/isolation, and retrieval-time weighting or filtering.

## Best Possible Solution

Define a framework-owned durable memory trust/provenance contract and enforce it across ingestion, storage, compaction/dreaming, SDK/config/docs, retrieval filtering or weighting, review, and expiration policy.

## Reproduction Assessment

Not applicable. This is a feature/security-hardening request, and current-main source inspection shows the requested trust tags, policy config, and retrieval gates are absent rather than an existing command path failing.

## Solution Assessment

Yes as a product direction, but not as an isolated metadata-field patch. The maintainable path is a framework-owned cross-backend trust/provenance contract with retrieval enforcement, safe defaults, and compatibility planning.
