## Summary

Keep this issue open: current main still has adjacent provenance and untrusted-context mitigations, but it does not implement durable memory trust metadata, trust-policy config, review/expiration gates, or trust-weighted retrieval for the default memory path.

## What This Changes

Request memory entries tagged by source trust level and retrieval policies that can expire, review, filter, or downweight lower-trust memories.

## Best Possible Solution

Design and implement a first-class cross-backend memory provenance/trust contract across ingestion, persistence, retrieval, compaction/dreaming, plugin SDK surfaces, UI, and docs, evaluating AKF as prior art rather than treating it as shipped behavior.

## Reproduction Assessment

Not applicable. This is a feature/security design request, and source inspection confirms the requested trust metadata and enforcement surfaces are not present on current main.

## Solution Assessment

Unclear. The direction is plausible, but the maintainable fix needs maintainer product/security design across memory contracts and backends rather than a one-file config addition.
