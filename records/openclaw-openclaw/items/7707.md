## Summary

Keep open: current main still has adjacent source labels, session visibility filtering, Active Memory untrusted framing, Memory Wiki provenance, and memory docs that recommend review discipline, but it does not implement durable memory trust metadata, trust-policy config, review/expiration gates, or trust-weighted retrieval.

## What This Changes

The issue requests source-based trust metadata and retrieval policies for durable memory so untrusted content cannot be promoted or reused as trusted context.

## Best Possible Solution

Define one framework-owned durable memory trust/provenance contract and enforce it across ingestion, storage, compaction/dreaming, SDK/config/docs, retrieval filtering or weighting, review, and expiration policy.

## Solution Assessment

Yes as a product direction, but not as an isolated metadata-field patch. The maintainable path is a framework-owned cross-backend trust/provenance contract with retrieval enforcement and compatibility planning.
