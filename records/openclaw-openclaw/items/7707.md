## Summary

Keep open: current main has coarse memory/session source labels, session-visibility filtering, Active Memory untrusted framing, and Memory Wiki provenance, but not durable source trust metadata, trust-policy config, review/expiration gates, or trust-weighted retrieval for persistent memory.

## What This Changes

The issue requests source-based trust metadata and trust-policy enforcement for persistent memory entries.

## Best Possible Solution

Design a first-class framework-assigned memory trust/provenance contract, then enforce durable metadata and retrieval policy consistently across memory ingestion, persistence, retrieval, compaction, SDK/config/docs/UI, and optional memory paths.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a runtime bug; this is a feature/security design request, and current-main source inspection verifies the requested trust metadata, trust config, review gates, expiration policy, and trust-weighted retrieval are absent.

## Solution Assessment

Is this the best way to solve the issue? Yes: keeping it open for maintainer/product/security design is the best current path, because a narrow metadata field or one-backend filter would not cover ingestion provenance, persistence, retrieval, compaction, SDK contracts, config, and UI/docs.
