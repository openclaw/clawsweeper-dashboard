## Summary

Current main and v2026.6.5 still provide only corpus-level source labels, relevance ranking, and advisory untrusted-memory guidance; they do not persist or enforce source-derived trust, and the narrower provenance MVP does not cover this issue’s remaining policy, review, expiry, or weighting scope.

## What This Changes

Add durable source-derived trust metadata and configurable retrieval, review, isolation, weighting, and expiration policies for memory entries.

## Best Possible Solution

Define one backend-neutral provenance record with framework-assigned causal origin and separate endorsement/review fields, preserve it through promotion and compaction, then add explicit operator-selected audit-only, filter, approval, decay, and ranking policies across every supported memory backend.

## Reproduction Assessment

Yes, at high confidence from source and release inspection: current main and v2026.6.5 lack persistent source-trust metadata and retrieval enforcement. This establishes the feature gap, not a failure of an existing documented contract.

## Solution Assessment

No, not fully as proposed. Source tagging is the right foundation, but fixed source-to-trust mappings and numeric influence weights should follow a canonical framework-owned provenance, endorsement, and policy design rather than becoming an isolated memory-core configuration.
