## Summary

Keep open: current main still lacks durable source trust metadata, trust-policy config, review or expiration gates, and trust-weighted retrieval for memory, while the discussion and related items show this remains an active product/security design gap.

## What This Changes

The issue requests source-based trust tagging for durable memory entries, with policy controls for expiration, review/isolation, and retrieval weighting to mitigate persistent memory poisoning.

## Best Possible Solution

Define one framework-owned durable memory trust/provenance contract, then enforce it across ingestion, storage, compaction/dreaming, bridge paths, SDK/config/docs, retrieval filtering or weighting, review, and expiration policy.

## Reproduction Assessment

Not applicable. This is a feature/security-hardening request, and current-main plus v2026.5.19 source inspection shows the requested trust tags, trust config, and retrieval gates are absent rather than an existing behavior failing.

## Solution Assessment

Yes as a product direction, but not as a narrow automated patch. The maintainable path is a framework-owned cross-backend trust/provenance contract rather than an isolated metadata column, prompt-only wrapper, or agent-reported trust flag.
