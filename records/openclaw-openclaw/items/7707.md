## Summary

Keep open: current main still has adjacent source labels, untrusted prompt framing, Memory Wiki provenance, and optional memory-lancedb heuristics, but it does not implement durable trust metadata, trust-policy config, review/expiration gates, or trust-weighted retrieval for memory.

## What This Changes

The issue requests source-based trust metadata and policies for durable agent memory so untrusted content can be filtered, reviewed, expired, or downweighted before influencing future context.

## Best Possible Solution

Define one framework-owned durable memory trust/provenance contract, then enforce it across ingestion, storage, compaction/dreaming, SDK/config/docs, retrieval filtering or weighting, review, and expiration policy.

## Reproduction Assessment

Not applicable. This is a feature/security-hardening request, and current-main source inspection shows the requested trust tags, policy config, and retrieval gates are absent rather than an existing behavior failing.

## Solution Assessment

Yes as a product direction, but not as a narrow automated patch. The maintainable path is a framework-owned cross-backend trust/provenance contract rather than an isolated metadata field, prompt-only wrapper, or agent-reported trust flag.
