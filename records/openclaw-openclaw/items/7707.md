## Summary

Keep open: current main has adjacent source labels, citations, session visibility filtering, Active Memory untrusted framing, and Memory Wiki provenance, but it still lacks persistent source trust metadata, trust-policy config, review/expiration gates, and trust-weighted retrieval for durable memory.

## What This Changes

Add persistent source-based trust metadata and policies for memory ingestion, expiration, review, and trust-weighted retrieval.

## Best Possible Solution

Define one durable memory trust/provenance contract, then enforce it through ingestion, storage, compaction/dreaming, Memory Wiki bridge paths, SDK/config docs, and retrieval-time filtering or weighting.

## Reproduction Assessment

Not applicable. This is a feature/security-hardening request, and current-main source inspection shows the requested trust tags, config, and retrieval gates are absent rather than a broken existing contract.

## Solution Assessment

Yes as a product direction, but not as a narrow automated patch. The maintainable path is a cross-backend memory trust contract rather than adding an isolated field or prompt-only wrapper.
