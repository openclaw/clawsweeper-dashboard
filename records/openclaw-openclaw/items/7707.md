## Best Possible Solution

Keep this open for maintainer product/security design, then implement one host-authored memory trust/provenance contract across ingestion, storage, retrieval, review, expiration, SDK, docs, and UI surfaces.

## Solution Assessment

Unclear as the exact proposed config shape, because maintainers still need to decide who assigns trust and how policy composes with memory backends. The best direction is a host-authored trust/provenance contract rather than agent-self-reported memory metadata.
