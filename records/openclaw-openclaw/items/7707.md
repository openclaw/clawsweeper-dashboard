## Summary

Keep open: current main and the latest release still lack durable memory trust metadata, trust-policy config, review/expiration gates, or trust-weighted retrieval; the discussion adds useful AKF and framework-owned trust-assignment prior art, but no shipped implementation closes the gap.

## What This Changes

The issue requests source-based trust tagging for durable agent memory with policy controls for expiration, review, isolation, and retrieval weighting.

## Best Possible Solution

Define one framework-owned durable memory trust/provenance contract, then enforce it across ingestion, storage, compaction/dreaming, bridge paths, SDK/config/docs, retrieval filtering or weighting, review, and expiration policy.

## Reproduction Assessment

Not applicable. This is a feature/security-hardening request, and current-main plus v2026.5.19 source inspection shows the requested trust tags, policy config, and retrieval gates are absent rather than an existing behavior failing.

## Solution Assessment

Yes as a product direction, but not as a narrow automated patch. The maintainable path is a framework-owned cross-backend trust/provenance contract rather than an isolated metadata field, prompt-only wrapper, or agent-reported trust flag.
