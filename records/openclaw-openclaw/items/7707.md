## Summary

Keep open. Current main still has adjacent source labels, session visibility filtering, Active Memory untrusted framing, and Memory Wiki provenance, but it does not implement durable trust metadata, trust-policy config, review/expiration gates, or trust-weighted retrieval for the default memory path.

## What This Changes

The issue requests source-based trust tagging for memory entries with expiration, review/isolation, and retrieval weighting policies.

## Best Possible Solution

Define a typed durable memory trust contract first, then carry it through ingestion, storage, SDK results, retrieval policy, compaction/dreaming, optional backends, and user-facing docs/UI.

## Reproduction Assessment

Not applicable as a feature/security-hardening request. The current gap is high-confidence from static inspection of the memory schema, search result contract, config schema, tool schema, and retrieval path.

## Solution Assessment

Unclear for the exact proposed config, but yes for the overall direction. A config-only addition would not be enough; the maintainable path needs durable typed provenance/trust metadata plus enforcement across every recall backend and promotion path.
