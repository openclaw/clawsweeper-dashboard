## Summary

Keep open: current main still has source labels, session visibility filtering, Active Memory untrusted framing, and Memory Wiki provenance, but not durable memory trust metadata, trust-policy config, review/expiration gates, or trust-weighted retrieval.

## What This Changes

Add durable source-based trust metadata and trust-policy controls for memory ingestion, storage, retrieval weighting/filtering, review, and expiration.

## Best Possible Solution

Define a framework-owned durable memory trust/provenance contract and enforce it through ingestion, storage, compaction/dreaming, Memory Wiki bridging, SDK/config/docs, retrieval filtering or weighting, and review/expiration policy.

## Reproduction Assessment

Not applicable. This is a feature/security-hardening request, and current-main source inspection shows the requested trust tags, policy config, and retrieval gates are absent rather than an existing behavior failing.

## Solution Assessment

Yes as a product direction, but not as a narrow automated patch. The maintainable solution is one cross-backend, framework-assigned trust/provenance contract rather than an isolated metadata field, prompt-only wrapper, or agent-reported trust flag.
