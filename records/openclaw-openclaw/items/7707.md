## Summary

Keep open: current main and v2026.6.8 still have source labels, untrusted prompt framing, citations, and Memory Wiki provenance-adjacent features, but they do not implement persistent memory trust metadata, trust-policy config, review/expiration gates, or trust-weighted retrieval across the default memory path.

## What This Changes

The issue requests source-based trust tagging for durable agent memory, with configurable trust levels, expiration/review policy, isolation, and trust-weighted retrieval to reduce memory-poisoning risk.

## Best Possible Solution

Define a host-authored memory trust/provenance contract first, then persist it through memory storage and promotion, expose it through SDK/search results, and enforce retrieval-time filtering, downweighting, review, and expiration policy across default and optional memory paths.

## Reproduction Assessment

Not applicable as a broken existing-behavior repro, but yes for source-level gap proof: current main lacks trust fields in memory contracts, config, schemas, and retrieval enforcement. I did not run a live poisoning scenario because the requested work is a feature/security design surface.

## Solution Assessment

Unclear as a direct implementation because the proposal sketches policy but not the trusted authority model. The best solution is a maintainer-approved, host-authored trust/provenance contract rather than ad hoc model- or plugin-supplied tags.
