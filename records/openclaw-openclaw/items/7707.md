## Summary

Keep open: current main has coarse memory source labels, session visibility filtering, and adjacent prompt/provenance mitigations, but it still lacks durable memory trust metadata, trust-policy config, review/expiration gates, and trust-weighted retrieval.

## What This Changes

The issue requests source-based trust tagging for agent memory, with configurable trust levels, expiration/review policy, and trust-aware retrieval to mitigate memory poisoning.

## Best Possible Solution

Design a first-class cross-backend memory trust/provenance contract, then implement durable metadata and retrieval enforcement consistently across memory-core, QMD, session memory, Active Memory, Memory Wiki bridge, SDK/config, docs, UI, and regression tests.

## Reproduction Assessment

Not applicable. This is a feature/security design request rather than a broken existing contract, and current-main source inspection verifies the requested trust metadata, trust config, review gates, expiration policy, and trust-weighted retrieval are absent.

## Solution Assessment

Unclear. The direction is plausible, but the maintainable solution needs product/security design across memory contracts and backends rather than a one-file metadata or config addition.
