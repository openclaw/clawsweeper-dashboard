## Summary

Keep this issue open: current main still has coarse memory sources and adjacent untrusted-context mitigations, but not durable trust metadata, trust-policy config, review/expiration gates, or trust-weighted retrieval for default memory recall.

## What This Changes

Request to tag memory entries by source trust, configure trust policies, and enforce expiration, review, filtering, or downweighting during retrieval.

## Best Possible Solution

Design and implement a first-class cross-backend memory provenance/trust contract that preserves source trust through ingestion, storage, retrieval, compaction/dreaming, plugin SDK surfaces, UI, and docs.
