## Summary

Keep this open: current main and v2026.6.6 still have source labels, session visibility filtering, Active Memory untrusted framing, and Memory Wiki provenance, but they do not implement durable memory trust metadata, trust-policy config, review/expiration gates, or trust-weighted retrieval across the default memory path.

## What This Changes

The issue requests source-based trust tags for durable agent memory, with configurable expiration, review isolation, and retrieval-time weighting to reduce memory-poisoning risk.

## Best Possible Solution

Define a framework-owned durable memory trust/provenance model, persist it through canonical memory records, expose a documented policy contract, and enforce retrieval-time filtering, weighting, review, and expiration consistently across default and optional memory backends.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes, at source level: current memory contracts, schemas, config, tool execution, builtin retrieval, and QMD retrieval paths have no durable trust metadata or trust-policy enforcement.

## Solution Assessment

Is this the best way to solve the issue? Not as a concrete config-only patch yet: source trust tagging is the right security direction, but maintainers still need to decide source authority, source/review axes, policy modes, decay semantics, and backend/API compatibility.
