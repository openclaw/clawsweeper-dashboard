## Summary

Keep open: current main and the latest release still have adjacent source labels, session visibility checks, Active Memory untrusted framing, and Memory Wiki provenance, but not durable memory trust metadata, trust-policy config, review/expiration gates, or trust-weighted retrieval.

## What This Changes

The issue requests tagging durable memory entries by source trust level and enforcing trust-based review, expiration, isolation, and retrieval weighting.

## Best Possible Solution

Define a maintainer-approved durable source/trust model, persist it across default and QMD memory paths, and enforce review, expiration, filtering, and weighting at retrieval with trust assigned by framework-owned context rather than agent-authored fields.

## Reproduction Assessment

Yes, at source level: current main lacks trust fields and policy hooks in memory result contracts, SQLite schema, config schema, default retrieval, and QMD retrieval. This is a feature/security design gap rather than a runnable broken-behavior reproduction.

## Solution Assessment

Unclear as a concrete implementation shape. The direction is valid, but the best solution needs maintainer product/security design across memory-core, QMD, SDK/config contracts, retrieval policy, and migration behavior rather than an isolated config patch.
