## Summary

Keep open: current main and v2026.5.20 still lack durable source trust metadata, trust-policy config, review/expiration gates, or trust-weighted retrieval for memory. The discussion and related open issues add useful provenance and framework-owned trust-assignment prior art, but none is a shipped implementation that closes this gap.

## What This Changes

Request to tag durable memory entries with source-based trust metadata and enforce expiration, review/isolation, and trust-weighted retrieval policies.

## Best Possible Solution

Define one framework-owned durable memory trust/provenance contract, then enforce it across ingestion, storage, compaction/dreaming, bridge paths, SDK/config/docs, retrieval filtering or weighting, review, and expiration policy.

## Reproduction Assessment

Not applicable. This is a feature/security-hardening request, and current-main plus v2026.5.20 source inspection shows the requested trust tags, policy config, and retrieval gates are absent rather than an existing behavior failing.

## Solution Assessment

Yes as a product direction, but not as a narrow automated patch. The maintainable path is a framework-owned cross-backend trust/provenance contract rather than an isolated metadata field, prompt-only wrapper, or agent-reported trust flag.
