## Summary

Current main still lacks durable memory trust metadata and enforcement, and the active discussion shows this remains a distinct product/security design beyond the narrower provenance MVP. Keep it open for coordinated maintainer design rather than closing it as implemented or duplicate.

## What This Changes

Add framework-assigned source trust metadata and configurable review, expiration, isolation, and retrieval-weighting policies to durable memory.

## Best Possible Solution

First define a canonical SQLite-backed provenance and endorsement record populated by framework-owned ingestion paths, then add explicit per-agent audit-only and enforcement policies through the shared memory plugin contract, with filtering, review, and decay semantics consistent across builtin, QMD, session, compaction, and Active Memory paths.

## Reproduction Assessment

Yes at the source level: current main clearly lacks persisted trust/review metadata and trust-policy inputs in the shared memory schema, result contract, and retrieval path. This demonstrates a capability gap rather than a broken existing contract.

## Solution Assessment

No, the original fixed source-to-trust mapping is not yet the best complete solution. The discussion's separation of causal origin, endorsement, operator-selected policy mode, and framework-owned assignment is safer and more maintainable.

## Risks / Open Questions

- Trust assignment authority is unresolved: allowing the model or memory writer to self-label origin would let injected content promote itself.
- Provenance, endorsement, and policy are currently conflated in the proposal; implementing one trust field could erase causal origin or create contradictory state after review and merges.
- The design spans builtin memory, QMD, session memory, Active Memory, compaction/dreaming, plugin SDK contracts, config, and persisted SQLite state, so an isolated patch could create divergent backend behavior.
- Expiration and trust-weighted influence are product-policy choices with false-positive costs for operator-curated and long-running research deployments.
