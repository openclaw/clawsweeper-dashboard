## Summary

Keep open: current main still has adjacent source labels, citations, Active Memory untrusted framing, recall tracking, and provenance annotations, but it does not implement durable memory trust metadata, trust-policy config, review/expiration gates, or trust-weighted retrieval.

## What This Changes

Request source-based trust tagging for agent memory, including persisted trust metadata, policy config, expiration/review gates, and trust-aware retrieval weighting.

## Best Possible Solution

Design one canonical memory trust/provenance contract that records framework-assigned source and review metadata at ingestion, persists it through memory backends and promotions, and enforces trust-aware retrieval with documented operator policy modes.

## Reproduction Assessment

Yes at source level: current main and v2026.6.6 expose source/citation/provenance-adjacent fields but no durable trust metadata, trust-policy config, review gate, expiration-by-trust, or trust-weighted retrieval. This is a feature/security design gap rather than a failing bug reproduction.

## Solution Assessment

Unclear as a concrete implementation: source trust tagging is directionally aligned, but maintainers still need to decide source authority, source versus review axes, policy modes, decay semantics, and backend/API compatibility before code work is safe.
