## Summary

Keep open: current main still has adjacent source labels, citations, session visibility filtering, Active Memory untrusted framing, and Memory Wiki provenance, but it does not implement persistent source trust metadata, trust-policy config, review/expiration gates, or trust-weighted retrieval for durable memory.

## What This Changes

Add persistent trust metadata and policies to memory ingestion, storage, and retrieval so lower-trust sources can expire, require review, or be downweighted.

## Best Possible Solution

Define a durable, framework-assigned memory trust/provenance contract and enforce it through ingestion, storage, compaction/dreaming, Memory Wiki bridge, SDK/config docs, and retrieval-time filtering or weighting.

## Reproduction Assessment

Not applicable. This is a feature/security-hardening request, and current-main source inspection shows the requested trust tags, policy config, and retrieval gates are absent rather than an existing contract failing.

## Solution Assessment

Yes as a product direction, but not as a narrow automated patch. The maintainable solution is one cross-backend, framework-assigned trust/provenance contract rather than adding an isolated field or prompt-only wrapper.
