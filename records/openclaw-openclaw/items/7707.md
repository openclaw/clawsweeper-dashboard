## Summary

Keep this issue open: current main has adjacent provenance and untrusted-context mitigations, but it still lacks persistent memory trust metadata, trust-policy config, review/expiration gates, and trust-weighted retrieval across the default memory path.

## What This Changes

Request source-derived trust metadata and policy controls for persistent memory ingestion, storage, retrieval, expiration, and review.

## Best Possible Solution

Design and implement a first-class memory provenance/trust contract across ingestion, persistence, retrieval, compaction/dreaming, plugin SDK surfaces, UI, and docs, using AKF as prior art rather than treating it as shipped behavior.

Not applicable. This is a feature/security design request, and source inspection confirms the requested trust metadata and enforcement surfaces are not present on current main.

## Solution Assessment

Unclear. The direction is plausible, but the maintainable solution needs maintainer product/security design across memory contracts and backends rather than a one-file config addition.
