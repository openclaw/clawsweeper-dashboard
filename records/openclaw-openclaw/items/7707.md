## Summary

Keep open: current main and v2026.6.8 still have adjacent source labels, external-content wrappers, session visibility filtering, Active Memory untrusted framing, and Memory Wiki provenance, but not the requested durable trust metadata, trust-policy config, review/expiration gates, or trust-weighted retrieval for the default memory path.

## What This Changes

The issue requests persistent trust/source metadata for agent memory with trust-policy config, expiration/review gates, and trust-weighted retrieval to reduce memory poisoning risk.

## Best Possible Solution

Keep this as a maintainer-led memory security design item: define one durable source/trust model, persist it across default and QMD memory paths, expose policy config only after the model is settled, and enforce review/expiration/filtering/weighting at retrieval without agent-forgeable trust assignment.

## Reproduction Assessment

Yes, at source level: current main lacks trust fields and policy hooks in memory result contracts, SQLite schema, config schema, default retrieval, and QMD retrieval. This is a feature/security design gap rather than a runnable broken-behavior reproduction.

## Solution Assessment

Unclear as a concrete implementation shape. The direction is valid, but the best solution needs maintainer product/security design across memory-core, QMD, SDK/config contracts, retrieval policy, and migration behavior rather than an isolated config patch.
