## Summary

Keep open: current main and v2026.6.8 still have adjacent source labels, session visibility filtering, Memory Wiki provenance, and Active Memory untrusted framing, but not persistent trust metadata, trust-policy config, review/expiration gates, or trust-weighted retrieval for default memory.

## What This Changes

Request to tag persistent agent memory by source trust level and enforce trust-aware expiration, review, isolation, and retrieval weighting to reduce memory-poisoning risk.

## Best Possible Solution

Keep this as a maintainer-led memory security design item: define one canonical provenance/trust data model, persist it through default and QMD memory paths, expose documented policy config only after the model is settled, and enforce review/expiration/filtering/weighting at retrieval without agent-forgeable trust assignment.

## Reproduction Assessment

Not applicable as a runnable bug reproduction, but the source-level gap is high-confidence: current memory contracts, schema, config, retrieval, docs, release code, and tests expose source/corpus labels but no trust metadata or trust-policy enforcement.

## Solution Assessment

Unclear as an implementation shape because the requested direction is valid but broad. The best path is maintainer security/product design across memory-core, QMD, SDK/config contracts, durable state, and retrieval rather than an isolated config patch.
