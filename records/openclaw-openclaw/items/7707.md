## Summary

Keep https://github.com/openclaw/openclaw/issues/7707 open: current main still has coarse memory source labels and adjacent untrusted-context/provenance mitigations, but not durable memory trust metadata, trust-policy config, review/expiration gates, or trust-weighted retrieval.

## What This Changes

The issue requests source-based trust tags for persistent memory, with configurable expiration, review/isolation policy, and retrieval-time trust weighting to mitigate memory poisoning.

## Best Possible Solution

Design and implement a first-class cross-backend memory trust/provenance contract that persists through ingestion, storage, recall, compaction/dreaming, SDK surfaces, UI, and docs, with enforceable retrieval policy rather than prompt-only labeling.
