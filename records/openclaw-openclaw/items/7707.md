## Summary

Keep this open: current main and the latest release still have source labels and adjacent untrusted framing, but no durable memory trust metadata, trust-policy config, review/expiration gates, or trust-weighted retrieval across the default memory path.

## What This Changes

Request persistent trust tagging for agent memory by source, with policy-driven expiration, review isolation, and retrieval weighting to mitigate memory poisoning.

## Best Possible Solution

Define a framework-owned durable memory trust/provenance model, persist it through canonical memory records, expose a documented policy contract, and enforce retrieval-time filtering, weighting, review, and expiration consistently across default and optional memory backends.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes, at source level: current memory contracts, schemas, config, tool execution, builtin retrieval, and QMD retrieval paths have no durable trust metadata or trust-policy enforcement.

## Solution Assessment

Is this the best way to solve the issue? Not as a concrete implementation yet: source trust tagging is the right security direction, but the proposed config alone is insufficient until maintainers decide source authority, source/review axes, policy modes, decay semantics, and backend/API compatibility.
