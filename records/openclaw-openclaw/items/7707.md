## Summary

Keep open: current main and v2026.5.20 still have source labels, citations, session visibility filtering, Active Memory untrusted framing, and Memory Wiki provenance, but they do not implement durable memory trust metadata, trust-policy config, review/expiration gates, or trust-weighted retrieval; recent comments add useful prior art and operator feedback rather than superseding the request.

## What This Changes

Requests durable source-based trust tags for memory entries, with trust-aware expiration, review/isolation policy, and retrieval weighting.

## Best Possible Solution

Define one framework-owned durable memory trust/provenance contract and enforce it across ingestion, storage, compaction/dreaming, SDK/config/docs, retrieval filtering or weighting, review, and expiration policy.

## Reproduction Assessment

Not applicable. This is a feature/security-hardening request, and current-main source inspection shows the requested trust tags, policy config, and retrieval gates are absent rather than an existing command path failing.

## Solution Assessment

Yes as a product direction, but not as an isolated metadata-field patch. The maintainable path is a framework-owned cross-backend trust/provenance contract with retrieval enforcement and compatibility planning.
