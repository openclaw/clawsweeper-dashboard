## Summary

Keep open: current main and the latest release still lack durable memory trust metadata, trust-policy config, review/expiration gates, and trust-weighted retrieval; the related open issues cover provenance, inbound wrapping, and autoCapture filtering as narrower pieces rather than superseding this broader security design.

## What This Changes

Request structured trust tags and policy controls for memory entries based on source so untrusted content cannot poison future agent context.

## Best Possible Solution

Define a framework-owned durable memory trust/provenance contract and enforce it across ingestion, storage, compaction/dreaming, SDK/config/docs, retrieval filtering or weighting, review, expiration, and operator opt-in semantics.

## Reproduction Assessment

Not applicable. This is a feature/security-hardening request, and current-main source inspection verifies that the requested trust tags, trust-policy config, and retrieval gates are absent rather than an existing command path failing.

## Solution Assessment

Yes as a product/security direction, but not as an isolated metadata-field patch. The maintainable path is a cross-backend framework-owned trust/provenance contract with retrieval enforcement, safe defaults, and compatibility planning.
