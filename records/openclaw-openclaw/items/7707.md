## Summary

Keep open: current main still has coarse source labels, citations, session visibility filtering, Active Memory untrusted framing, and Memory Wiki provenance, but not persistent memory trust metadata, trust-policy config, review/expiration gates, or trust-weighted retrieval. The discussion and related items reinforce this as a real product/security design gap rather than something already shipped or cleanly superseded.

## What This Changes

The issue requests durable memory entries be tagged by source trust level, with configurable expiration, review/isolation, and retrieval weighting policies.

## Best Possible Solution

Define one framework-assigned durable memory trust/provenance contract and enforce it through ingestion, storage, compaction/dreaming, Memory Wiki bridging, SDK/config/docs, and retrieval-time filtering or weighting.

## Reproduction Assessment

Not applicable. This is a feature/security-hardening request, and current-main source inspection shows the requested trust tags, policy config, and retrieval gates are absent rather than an existing behavior failing.

## Solution Assessment

Yes as a product direction, but not as a narrow automated patch. The maintainable solution is one cross-backend, framework-assigned trust/provenance contract rather than an isolated metadata field, prompt-only wrapper, or agent-reported trust flag.
