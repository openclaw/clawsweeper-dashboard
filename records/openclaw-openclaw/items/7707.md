## Summary

Keep open: current main and v2026.5.22 still have source-only memory contracts and adjacent untrusted framing, but no durable trust metadata, trust-policy config, review/expiration gates, or trust-weighted retrieval for default memory.

## What This Changes

The issue requests source-based trust tagging for agent memory, with configurable expiration, review/isolation, and retrieval weighting policies to reduce memory-poisoning risk.

## Best Possible Solution

Define a framework-owned durable memory trust/provenance contract and enforce it across ingestion, storage, SDK/config/docs, compaction/dreaming, retrieval filtering or weighting, review, and expiration policy.

## Reproduction Assessment

Not applicable as a broken-behavior reproduction. This is a feature/security-hardening request, and source inspection shows the requested trust tags, policy config, and retrieval gates are absent rather than an existing command path failing.

## Solution Assessment

Yes as a product direction, but not as an isolated metadata-field patch. The maintainable path is a cross-backend, framework-owned trust/provenance contract with retrieval enforcement, safe defaults, and compatibility planning.
