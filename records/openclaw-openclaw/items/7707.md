## Summary

Keep open: current main still has adjacent source labels, citations, session visibility filtering, Active Memory untrusted framing, and Memory Wiki provenance, but not persistent memory trust metadata, trust-policy config, review/expiration gates, or trust-weighted retrieval for durable memory.

## What This Changes

Request to add durable memory trust metadata, policy config, review/expiration gates, and trust-weighted retrieval based on memory source.

## Best Possible Solution

Define a durable, framework-assigned memory trust/provenance contract and enforce it through ingestion, storage, compaction/dreaming, Memory Wiki bridge, SDK/config docs, and retrieval-time filtering or weighting.

## Reproduction Assessment

Not applicable. This is a feature/security-hardening request, and current-main source inspection shows the requested trust tags, policy config, and retrieval gates are absent rather than an existing behavior failing.

## Solution Assessment

Yes as a product direction, but not as a narrow automated patch. The maintainable solution is one cross-backend, framework-assigned trust/provenance contract rather than an isolated metadata field or prompt-only wrapper.
