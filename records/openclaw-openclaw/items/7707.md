## Summary

Current main and v2026.6.8 still have adjacent source/provenance signals, untrusted-content framing, session visibility filtering, and memory ranking controls, but not durable trust tiers, review/expiration gates, or trust-weighted retrieval.

## What This Changes

The issue requests source-based trust metadata and retrieval policy for durable agent memories to reduce memory poisoning risk.

## Best Possible Solution

Keep this open for maintainer product/security design that defines one durable provenance/trust contract and enforces it consistently at ingestion, storage, retrieval, review, expiration, and UI/docs surfaces.
