## Summary

Keep this issue open: current main has adjacent provenance and untrusted-context mitigations, but it still lacks persistent memory trust metadata, trust-policy config, review/expiration gates, and trust-weighted retrieval for the default memory path.

## What This Changes

Request source-based trust tagging for persistent memory, with config-driven expiration, isolation, review gates, retrieval weighting, and UI visibility.

## Best Possible Solution

Design and implement a first-class memory provenance/trust contract across ingestion, persistence, retrieval, compaction/dreaming, plugin SDK surfaces, UI, and docs, using AKF as prior art rather than treating it as shipped behavior.

Not applicable. This is a feature/security design request, and source inspection on current main shows the requested trust metadata and enforcement surfaces are not present to exercise as a broken existing behavior.

## Solution Assessment
