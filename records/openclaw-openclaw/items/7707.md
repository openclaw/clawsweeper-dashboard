## Summary

Keep open: current main still has adjacent source labels, prompt wrapping, session visibility checks, Active Memory untrusted framing, Memory Wiki provenance, and recall/promotion counters, but it still lacks durable memory trust metadata, trust-policy config, review/expiration gates, and trust-weighted retrieval. The recent issue discussion adds concrete product/security requirements, so this is live design work rather than cleanup closure.

## What This Changes

The issue requests source-based trust tagging for agent memories, with policy-based expiration, review isolation, and retrieval weighting to reduce memory poisoning risk.

## Best Possible Solution

Define a framework-owned durable memory trust/provenance model, persist it through the canonical memory record path, expose a documented policy contract, and enforce retrieval-time filtering, weighting, review, and expiration consistently across default and optional memory backends.

## Reproduction Assessment

Yes, at source level: current memory contracts, schemas, config, memory_search execution, builtin/QMD retrieval, and short-term promotion paths have no durable trust metadata or trust-policy enforcement. This is a feature/security-hardening gap rather than a failing bug reproduction.

## Solution Assessment

Unclear as a direct implementation: source trust tagging is the right security direction, but the proposed config alone is insufficient until maintainers decide source authority, source/review axes, audit-only versus enforcement modes, decay semantics, and backend/API compatibility.
