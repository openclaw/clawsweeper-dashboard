## Summary

Keep open. Current main has adjacent source labels, session visibility filtering, Active Memory untrusted framing, Memory Wiki provenance, and memory-lancedb prompt-injection mitigations, but the default memory path still lacks durable trust metadata, trust-policy config, review/expiration gates, or trust-weighted retrieval; the issue discussion also already kept this gap live and called out AKF as useful prior art.

## What This Changes

Request durable source-based trust tagging, expiration/review policy, and trust-weighted retrieval for persisted memory entries.

## Best Possible Solution

Define a typed durable trust/provenance contract, then persist and enforce it across memory ingestion, storage, SDK results, recall backends, compaction/dreaming, bridge/docs, and review/UI surfaces.

## Reproduction Assessment

Not applicable as a feature/security-hardening request. The gap is high-confidence from static inspection of the current memory contract, schema, config, tool schema, retrieval managers, docs, and adjacent plugins.

## Solution Assessment

Unclear whether the exact proposed config is the best shape. The maintainable direction is a durable typed metadata and enforcement contract, not only a retrieval-time knob or prompt wording change.
