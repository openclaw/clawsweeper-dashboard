## Summary

Keep this issue open: current main still has coarse memory source labels and adjacent prompt-framing/provenance mitigations, but not durable source trust metadata, trust-policy config, review/expiration gates, or trust-weighted retrieval for the default memory path.

## What This Changes

Request durable trust tags for memory entries by source, with configurable expiration, review isolation, and retrieval-time trust weighting.

## Best Possible Solution

Design a first-class cross-backend memory trust/provenance contract with enforceable retrieval policy, then implement it consistently through storage, SDK/config, compaction/dreaming, UI, docs, and regression tests.

Not applicable. This is a feature/security design request, and current-main source inspection confirms the requested trust metadata, trust config, review gates, expiration policy, and trust-weighted retrieval are absent.
