## Summary

Keep open: current main and v2026.6.5 still lack durable memory trust metadata, trust-policy config, review/expiration gates, and trust-weighted retrieval; the related open issues cover narrower provenance, inbound wrapping, and autoCapture filtering pieces rather than superseding this broader security design.

## What This Changes

Request source-based trust tagging for durable memory entries, with trust-aware retention, review isolation, and retrieval weighting to reduce persistent memory poisoning risk.

## Best Possible Solution

Define a framework-owned durable memory trust/provenance contract and enforce it across ingestion, storage, compaction/dreaming, SDK/config/docs, retrieval filtering or weighting, review, expiration, and any explicit operator trusted-source opt-in semantics.

## Solution Assessment

Yes as a product/security direction, but not as an isolated metadata-field patch. The maintainable path is a cross-backend, framework-owned trust/provenance contract with retrieval enforcement, safe defaults, and compatibility planning.
