## Summary

Current main has adjacent safeguards—source corpus labels, citations, session visibility filtering, temporal decay, and Active Memory untrusted framing—but still lacks durable origin-trust metadata and trust-aware retrieval policy. The open provenance work is a prerequisite, not a duplicate, because this issue uniquely covers security policy, review gates, trust weighting, and expiration.

## What This Changes

Add framework-derived source trust metadata and configurable policy controls for durable memory ingestion, persistence, review, retrieval weighting, isolation, and expiration.

## Best Possible Solution

First establish framework-assigned provenance as the canonical durable memory metadata, then define one backend-neutral trust policy that preserves provenance through writes, merges, compaction, and dreaming; enforces filtering, review, weighting, and expiration at retrieval; and exposes explicit operator overrides without allowing model-authored trust escalation.

## Reproduction Assessment

Yes at the source level. Current main clearly exposes corpus source and relevance controls without durable origin-trust metadata or trust-aware retrieval enforcement, although no end-to-end poisoning scenario was executed during this read-only review.

## Solution Assessment

No, not fully as proposed. Source-derived trust is the right direction, but the maintainable solution must derive authority from framework provenance, settle the canonical persisted schema first, and define consistent semantics across all memory backends rather than adding isolated labels and weighting knobs.
