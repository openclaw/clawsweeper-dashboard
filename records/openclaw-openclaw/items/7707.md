## Summary

Keep this open: current main and v2026.6.6 still have source labels, visibility filtering, untrusted prompt framing, and provenance-adjacent docs, but they do not implement durable memory trust metadata, trust-policy config, review/expiration gates, or trust-weighted retrieval across the default memory path.

## What This Changes

The issue requests source-based trust tagging for durable memory plus policy enforcement for review, expiration, isolation, and retrieval weighting.

## Best Possible Solution

Define a framework-owned durable memory trust/provenance model, persist it through canonical memory records, expose a documented policy contract, and enforce retrieval-time filtering, weighting, review, and expiration consistently across default and optional memory backends.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes, at source level: current main and v2026.6.6 memory contracts, schemas, config, tool execution, builtin retrieval, and QMD retrieval paths have no durable trust metadata or trust-policy enforcement.

## Solution Assessment

Is this the best way to solve the issue? The security direction is sound, but the issue is not ready as a config-only patch; maintainers need to decide source authority, source/review axes, policy modes, decay semantics, and backend/API compatibility first.
