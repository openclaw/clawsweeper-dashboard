## Summary

Keep open. Current main still has adjacent provenance and untrusted-context mitigations, but it does not provide durable source trust metadata, trust-policy config, review/expiration gates, or trust-weighted retrieval for default memory recall.

## What This Changes

Request durable trust tagging for memory entries by source, with policy-based expiration, review/isolation, and retrieval weighting.

## Best Possible Solution

Define a typed durable trust/provenance contract first, then carry it through ingestion, storage, SDK results, builtin/QMD/session memory, Active Memory, compaction/dreaming, Memory Wiki bridge, docs, and any review/UI surfaces.

## Reproduction Assessment

Not applicable as a feature/security-hardening request. The current gap is high-confidence from static inspection of the memory schema, result contract, config schema, tool schema, retrieval managers, docs, and adjacent plugins.

## Solution Assessment

Unclear whether the exact proposed config is the best shape, but the overall direction is sound. A config-only addition would not be enough; the maintainable solution needs durable typed metadata plus enforcement across every recall backend and promotion path.
