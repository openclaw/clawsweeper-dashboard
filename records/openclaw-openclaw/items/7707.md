## Summary

Keep this open: current main and the latest release still have source labels, citations, session visibility filtering, Active Memory untrusted framing, and Memory Wiki provenance, but not durable memory trust metadata, trust-policy config, review/expiration gates, or trust-weighted retrieval. The recent discussion adds useful operational detail and reinforces that this is a live product/security design item, not cleanup-ready stale work.

## What This Changes

The issue requests source-based trust metadata and trust-policy enforcement for durable agent memories to reduce memory-poisoning risk.

## Best Possible Solution

Define a framework-owned durable memory trust/provenance model, persist it through the canonical memory record path, expose a documented policy contract, and enforce retrieval-time filtering, weighting, review, and expiration consistently across default and optional memory backends.

## Reproduction Assessment

Yes, at source level. Current memory search contracts, schema, config, and tool execution have no durable trust metadata or trust-policy controls, but I did not run a live poisoning scenario because this is a feature/design request rather than a broken documented behavior.

## Solution Assessment

Unclear as a concrete implementation. The security direction is aligned, but the proposed config alone is not enough; the maintainable solution needs a maintainer decision on framework-owned source authority, policy modes, persistence, migration, UI/docs, and backend parity.
