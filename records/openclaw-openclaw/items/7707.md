## Summary

Keep open: current main and v2026.6.6 still have only coarse memory/session source labels, citations, Active Memory untrusted framing, and adjacent provenance work; they do not implement durable memory trust metadata, trust-policy config, review/expiration gates, or trust-weighted retrieval.

## What This Changes

Request durable memory trust tags and policies based on source origin so untrusted or low-trust content cannot silently influence later agent behavior.

## Best Possible Solution

Define one framework-owned memory provenance and trust contract, then enforce it consistently at ingestion, durable storage, promotion/compaction, retrieval, review gates, expiration, docs, and UI surfaces.

## Reproduction Assessment

Yes at source level: current main and v2026.6.6 expose memory/session source labels and scoring, but source inspection shows no durable trust metadata, review gates, expiration policy, or trust-weighted retrieval. This is a feature/security design gap rather than a failing command reproduction.

## Solution Assessment

Unclear as a direct implementation: the request points in the right direction, but the proposed config is only one policy shape. The best fix needs maintainer decisions on provenance authority, plugin/API contracts, storage migrations, defaults, and enforcement modes.
