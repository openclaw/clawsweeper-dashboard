## Summary

Keep open: current main still has source labels, citations, session visibility filtering, Active Memory untrusted framing, and Memory Wiki provenance, but it does not implement durable trust metadata, trust-policy config, review/expiration gates, or trust-weighted retrieval for memory.

## What This Changes

Request durable source-based trust metadata and trust-policy enforcement for agent memory ingestion, storage, retrieval, review, weighting, and expiration.

## Best Possible Solution

Define one framework-owned durable memory trust/provenance contract, then enforce it across ingestion, storage, compaction/dreaming, SDK/config/docs, retrieval filtering or weighting, review, and expiration policy.

## Reproduction Assessment

Not applicable. This is a feature/security-hardening request, and current-main source inspection shows the requested trust tags, policy config, and retrieval gates are absent rather than an existing command path failing.

## Solution Assessment

Yes as a product direction, but not as a narrow automated patch. The maintainable path is a framework-owned cross-backend trust/provenance contract rather than an isolated metadata field, prompt-only wrapper, or agent-reported trust flag.
