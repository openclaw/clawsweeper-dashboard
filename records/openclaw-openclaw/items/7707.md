## Summary

Keep this issue open: current main and v2026.6.8 still have adjacent source buckets, citations, session visibility filtering, Active Memory untrusted framing, and Memory Wiki provenance, but not durable memory trust tiers, review/expiration gates, or trust-weighted retrieval enforcement.

## What This Changes

The issue requests source-derived trust tags for durable agent memory, trust-policy config, expiration/review isolation, and retrieval-time weighting or filtering to mitigate memory poisoning.

## Best Possible Solution

Define one maintainer-approved, host-authored durable memory provenance/trust contract and enforce it consistently at ingestion, storage, retrieval, review, expiration, UI/docs, and plugin SDK boundaries.

## Reproduction Assessment

Yes, at source level: current main can be checked by reading the memory result type, schema, config schema, and retrieval paths, all of which lack durable trust metadata and trust-policy enforcement. This is not a failing bug reproduction.
