## Summary

Keep this issue open: current main still has coarse source labels and adjacent untrusted-context mitigations, but not durable memory trust metadata, trust-policy config, review/expiration gates, or trust-weighted retrieval.

## What This Changes

Add durable trust metadata and policy controls to memory ingestion, storage, and retrieval so untrusted sources can be reviewed, expired, filtered, or downweighted.

## Best Possible Solution

Design and implement a first-class cross-backend memory provenance/trust contract that preserves source trust through ingestion, storage, retrieval, compaction/dreaming, plugin SDK surfaces, UI, and docs.
