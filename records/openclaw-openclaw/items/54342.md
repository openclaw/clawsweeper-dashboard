## Summary

Keep open: current main still rejects persistent ACP session spawns without thread binding, and hook-triggered agent turns still enter isolated cron execution rather than an authorized persistent ACP turn or affinity path. Task/run tracking and hook delivery suppression cover parts of the original report, but ACP lifecycle, resume identity, and follow-up routing remain product and security contract work.

## What This Changes

The issue requests persistent ACP/subagent sessions from hook-triggered, non-threaded channel workflows with safe resume/follow-up routing and controlled delivery behavior.

## Best Possible Solution

Define a first-class headless persistent ACP lifecycle or affinity contract with explicit status/list/close/retention, safe opaque resume identity, and an authorized follow-up path through `sessions_send` or guarded hook ACP dispatch while preserving existing thread-bound behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows `sessions_spawn`/ACP `mode: 'session'` without `thread: true` still hits `thread_required`, and hook-triggered runs still use isolated cron execution rather than ACP `runTurn` dispatch.

## Solution Assessment

No, not as a simple guard removal. The maintainable fix needs explicit headless ACP lifecycle, delivery, ownership, and resume-identity rules before changing the public contract.
