## Summary

Keep this issue open: current main still contains the standalone CLI status context-window resolver, and the linked fix path is an open pull request rather than merged code.

## What This Changes

The issue reports that `openclaw status` uses a copied context-window resolver in `src/commands/status.summary.runtime.ts` and can fall back to the 200k default for models not explicitly configured.

## Best Possible Solution

Land a narrowed fix that removes the standalone CLI status resolver, preserves the intended read-only lookup behavior, and then close this issue from the merged pull request.

## Reproduction Assessment

Yes, source-reproducible with high confidence: current main still routes status summary context resolution through a copied resolver that falls back to 200k when config has no matching model entry. I did not run the CLI because this review was read-only.

## Solution Assessment

Yes for the requested direction, but the current open pull request is broader than the narrowest repair. The best fix is to delegate the CLI status runtime to the shared resolver while preserving existing read-only lookup contracts.
