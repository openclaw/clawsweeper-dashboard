## Summary

Keep open: current main and latest release still have a standalone CLI status-summary context resolver that can return the 200k default instead of the shared resolver/cache behavior, and the related open PRs do not touch this status-summary runtime path.

## What This Changes

The issue reports that `openclaw status` can show `Context: X/200k` for non-configured models because `src/commands/status.summary.runtime.ts` carries a standalone context-window resolver instead of the shared agent resolver.

## Best Possible Solution

Refactor status context-window resolution onto a shared lightweight resolver/cache path that matches `src/agents/context.ts` semantics while preserving `openclaw status` as a fast read-only command.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: route a status summary row through `statusSummaryRuntime.resolveContextTokensForModel` with no configured provider context window and a discovered/cache-backed model window, and the local resolver falls through to 200k instead of the shared cache lookup.

## Solution Assessment

No for literal import delegation as stated, because history shows that direct shared-context imports were removed to keep status startup slim. The best fix is a small shared lightweight resolver boundary plus regression coverage for status rows and read-only behavior.

## Repair Work Prompt

Fix the CLI status context-window resolution bug reported in https://github.com/openclaw/openclaw/issues/92760. Current main's `src/commands/status.summary.runtime.ts` has a standalone `resolveContextTokensForModel` that ignores the shared cache lookup and falls back to `DEFAULT_CONTEXT_TOKENS`, while `src/commands/status.summary.ts` calls it for default and session rows. Implement a small shared, lightweight resolver path so status uses the same provider/model/context-window semantics as `src/agents/context.ts` without reintroducing heavy status startup/plugin discovery imports or mutating status' read-only behavior. Add regression coverage for a non-configured provider/model whose context window is available from the model context cache/discovery path and for `allowAsyncLoad: false` read-only behavior. Check whether related open PRs https://github.com/openclaw/openclaw/pull/92709 and https://github.com/openclaw/openclaw/pull/90889 still apply; do not edit `CHANGELOG.md`.
