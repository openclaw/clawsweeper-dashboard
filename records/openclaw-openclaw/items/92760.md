## Repair Work Prompt

Fix the source-reproducible CLI `openclaw status` context-window bug from https://github.com/openclaw/openclaw/issues/92760. Current main has a standalone `statusSummaryRuntime.resolveContextTokensForModel` in `src/commands/status.summary.runtime.ts` that ignores `allowAsyncLoad`, scans only configured provider model entries, and falls back to `DEFAULT_CONTEXT_TOKENS`, while chat `/status` uses `src/agents/context.ts`. Replace the duplicate CLI resolver behavior with the shared agents resolver or a narrow shared helper so cached/discovered context windows are honored, while preserving the read-only `openclaw status` contract: do not start provider discovery or write models/config state from status summary lookups. Add focused regression coverage for CLI status summary resolving a cached provider/model window when no configured model `contextTokens` exists, and keep config overrides authoritative. Do not edit CHANGELOG.md.
