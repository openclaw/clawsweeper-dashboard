The issue asks gateway status/install to report and manage a working Linux user systemd service instead of treating `systemctl --user` bus errors as unavailable when the service is enabled and active.

## Best Possible Solution

Land a narrow daemon/systemd fix, likely through https://github.com/openclaw/openclaw/pull/73888, that synthesizes only the needed DBus/XDG env for user `systemctl` children while preserving root/sudo target-user scope and service-secret boundaries.

## Solution Assessment
