## Summary

Keep open because current main still has the reported permission-denied user-bus false-negative path, and the open closing PR #73888 has not merged.

## What This Changes

The issue asks `openclaw gateway status` and `openclaw gateway install` to recognize an enabled, active Linux user systemd service when `systemctl --user` works in the same shell.

## Best Possible Solution

End state should be a narrow `src/daemon/systemd.ts` fix that recovers missing Linux user-bus spawn environment for the intended user and includes status/install regression coverage.

## Reproduction Assessment

Yes. The issue has concrete Ubuntu commands and logs, and current main has a code-level reproduction path by mocking `systemctl --user` to return `Failed to connect to bus: Permission denied`.

## Solution Assessment

Yes. Recovering the systemctl spawn environment in the daemon/systemd layer is the narrow maintainable direction; status-only wording or hint changes would still leave install management broken.
