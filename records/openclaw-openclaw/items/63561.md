## Summary

Close: current main and v2026.5.18 now recover the missing DBus/XDG user-bus environment for `systemctl --user` children and carry regression coverage for the reported false-unavailable gateway status/install path.

## What This Changes

The issue asks `openclaw gateway status` and `openclaw gateway install` to recognize a working user-systemd service instead of reporting `systemctl --user` unavailable when the caller environment is missing DBus/XDG user-bus variables.

## Best Possible Solution

Keep the shipped systemctl child-env recovery in v2026.5.18 and treat the still-open replacement PR as obsolete unless it contains a separate follow-up.

## Reproduction Assessment

No current-main failure path remains: the source now synthesizes the missing DBus/XDG env before `systemctl --user` when the runtime bus exists. The original report itself was high-confidence from concrete Ubuntu logs and commands.

## Solution Assessment

Yes. Fixing the `systemctl` child-process environment in `src/daemon/systemd.ts` is the narrow maintainable solution; warning-only changes would not make status/install manage the working service.
