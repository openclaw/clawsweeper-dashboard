## Summary

Keep open. Current main still routes `systemctl --user` through the inherited process environment and still classifies the reported permission-denied user-bus failure as systemd-unavailable; the linked closing PR #73888 remains the correct implementation candidate to review before this issue can be closed.

## What This Changes

The issue requests that Linux gateway status/install report an already active user systemd gateway service correctly instead of falsely treating `systemctl --user` as unavailable.

## Best Possible Solution

Review and land #73888, or an equivalent narrow fix in `src/daemon/systemd.ts`, that recovers missing Linux user-bus environment for `systemctl --user` spawns, preserves existing env values and target-user scope, and adds regression coverage for gateway status and install.

Yes. The issue includes detailed Ubuntu steps and logs, and current main gives a code-level reproduction path: a direct `systemctl --user` permission-denied bus failure is still classified as unavailable and drives the same status/install messaging.
