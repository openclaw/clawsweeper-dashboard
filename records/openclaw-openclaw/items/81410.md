## Summary

Keep open: current main still has the reported scope mismatch, and the related systemd fixes/open PRs address other sudo/user-bus cases rather than a root-owned user service with stale sudo variables.

## What This Changes

Report that Linux gateway lifecycle commands in a root shell with stale sudo environment variables manage the sudo user's systemd scope instead of the root-owned user service.

## Best Possible Solution

Update Linux systemd scope resolution so root-owned user services are managed through root's direct user manager when the effective user and unit path indicate root, while preserving existing sudo-to-non-root fallback behavior.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: effective UID 0 plus non-root `SUDO_USER` makes OpenClaw choose `--machine <sudo-user>@ --user` even when `HOME=/root` points the unit path at root's user service. I did not run a destructive live systemd stop/start in this review.

## Solution Assessment

No, current main is not the best solution because it treats all root-plus-non-root-`SUDO_USER` shells as intentional non-root wrappers. The narrow fix belongs in `src/daemon/systemd.ts` with focused tests that keep sudo-u and user-bus fallbacks intact.
