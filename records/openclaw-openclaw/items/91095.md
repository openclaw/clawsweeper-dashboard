## Summary

Keep open: current main and latest release still exclude `sessions_spawn` from embedded TUI tool materialization unless `allowGatewaySubagentBinding` is set, and the two linked closing PRs are still open with unresolved proof/completeness concerns.

## What This Changes

Expose `sessions_spawn` in interactive TUI sessions when configured subagent allowlists permit delegation.

## Best Possible Solution

Land one canonical fix that mirrors the effective per-agent/default subagent allowlist and either proves local embedded spawning works or keeps/document local mode as unsupported while gateway-backed TUI works.

## Reproduction Assessment

Yes, by source inspection for the local embedded path: TUI sets embedded mode, the tool gate excludes `sessions_spawn` without `allowGatewaySubagentBinding`, and interactive ingress does not pass that option. I did not run a live Windows/npm or non-local URL-backed TUI reproduction.

## Solution Assessment

Unclear. `createOpenClawTools` is the right surface, but the best fix must share the executor's effective allowlist precedence and settle whether local embedded mode should execute subagent spawns or document them as Gateway-only.
