## Summary

Keep open: current main still gates `sessions_spawn` out of embedded TUI tool materialization unless `allowGatewaySubagentBinding` is set, and two open PRs already claim this issue with closing syntax, so the issue should remain open until a reviewed fix lands or maintainers choose the local-mode contract.

## What This Changes

Interactive TUI sessions should expose and execute `sessions_spawn` when subagent allowlists permit it instead of returning `Tool sessions_spawn not found`.

## Best Possible Solution

Define the TUI contract, then land one fix that mirrors the effective per-agent/default allowlist, keeps local embedded hidden unless it can execute, or implements and proves a local spawn bridge.

## Reproduction Assessment

Yes for the local embedded path by source inspection: local TUI sets embedded mode, and the current tool gate omits `sessions_spawn` unless `allowGatewaySubagentBinding` is true. The full gateway-connected Windows/npm claim still needs live confirmation.

## Solution Assessment

Unclear: the report is valid enough to keep open, but the best fix must separate gateway-bound TUI behavior from local embedded execution and mirror the effective subagent allowlist. The current open PRs should be reviewed against that boundary before merge.
