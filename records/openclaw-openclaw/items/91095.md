Interactive TUI sessions should expose `sessions_spawn` when the effective tool policy and configured subagent allowlist permit delegation.

## Best Possible Solution

Land one canonical fix that derives `sessions_spawn` exposure from the effective per-agent/default allowlist and either supports and proves local embedded spawning or keeps that path explicitly gated/documented.

## Reproduction Assessment

The reported non-local gateway TUI variant still needs live confirmation.

## Solution Assessment

Unclear: the linked PR direction is plausible, but the best fix must mirror the effective per-agent/default allowlist and prove whether local embedded spawning is executable or should stay gated.
