## Summary

This PR is still necessary: current main and the latest release still let identity-less outbound receipts flow through text/formatted/media delivery accounting, while the PR applies the shared identity check and adds focused cron/outbound regression coverage. Keep it open for maintainer acceptance of the stricter cross-channel receipt contract.

## What This Changes

The PR filters identity-less shared outbound delivery receipts from text, formatted-text, fallback text, and media sends, and adds cron/outbound regression tests for not reporting empty platform receipts as delivered.
