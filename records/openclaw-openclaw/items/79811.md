## Summary

Keep open. Current main and the latest release still have the false-success accounting path for identity-less text, formatted-text, and media outbound results, while this PR is a focused fix candidate that needs maintainer acceptance because it tightens the shared outbound receipt contract.

## What This Changes

The PR changes shared outbound delivery accounting so text, formatted-text, and media adapter results without a platform delivery identity are suppressed instead of counted as sent.

## Best Possible Solution

Land this narrow accounting fix after maintainers accept the empty-identity receipt contract, while keeping https://github.com/openclaw/openclaw/issues/87561 open for the broader cross-channel durable delivery contract.

## Reproduction Assessment

Yes, at source level: current main can still append identity-less text/formatted/media delivery results and sendDurableMessageBatch maps any non-empty result set to sent. I did not run live WeChat or Feishu credentials in this read-only review.

## Solution Assessment

Yes, with maintainer acceptance. The shared outbound delivery layer is the narrow maintainable boundary because cron consumes durable-send results; channel-specific workarounds would leave other direct delivery paths inconsistent.
