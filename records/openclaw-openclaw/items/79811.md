## Summary

Keep open. This PR remains the active, clean fix candidate for the linked cron false-delivered bug: current main and the latest release still count identity-less text, formatted-text, fallback-text, and media results by non-empty result count, while the patch filters those through the existing delivery-identity invariant and I found no blocking patch defect.

## What This Changes

The PR filters shared outbound text, formatted-text, and media delivery results so empty or identity-less adapter receipts no longer mark cron/direct sends or message_sent hooks as successful, with focused regression tests.

## Best Possible Solution

Land this narrow accounting fix after maintainers accept the empty-identity receipt contract, while keeping https://github.com/openclaw/openclaw/issues/87561 open for the broader cross-channel durable delivery contract.

## Solution Assessment

Yes, with maintainer acceptance. The shared outbound delivery layer is the narrow maintainable boundary because cron consumes durable-send results; channel-specific workarounds would leave other direct delivery paths inconsistent.
