## Best Possible Solution

Keep the narrow lifecycle fix, but clear per-run dedupe state only after retry delivery evidence has been captured, add focused regression coverage for normal compaction and retry compaction, and rebase the branch down to the relevant files with real post-compaction proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: seed `ctx.state.messagingToolSentTextsNormalized`, process a normal `compaction_end` with `willRetry: false`, then process a later `message_end` whose final text includes that stale normalized entry.

## Solution Assessment

No. Clearing the arrays is the right direction, but this placement runs before `resetForCompactionRetry()` records messaging delivery evidence; the safer fix is to clear on the non-retry path or capture retry evidence before clearing.
