## Summary

Keep open: the reported cross-run dedupe leak is valid and the PR is pointed at the right lifecycle area, but the current patch clears messaging-send evidence before the retry reset can snapshot it, which breaks compaction-retry side-effect accounting. The external PR also still lacks after-fix proof from a real setup.

## What This Changes

The branch adds an unconditional messaging-tool sent-state reset during `compaction_end`, updates the compaction test fixture, and carries older release/workflow/package commits outside the stated fix.

## Best Possible Solution

Clear messaging sent-state for normal completed compactions while preserving the existing retry snapshot-before-clear contract, then add focused regression coverage and real setup proof that a post-compaction follow-up reply is delivered.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: seed `messagingToolSentTextsNormalized`, process a normal `compaction_end` with `willRetry: false`, then process a later `message_end` whose final text includes that stale normalized entry; the PR-introduced retry regression is similarly visible by following `willRetry: true` into `resetForCompactionRetry`.

## Solution Assessment

No, not as currently written. The reset should happen only after retry side-effect evidence is captured, or only in the non-retry path, rather than before `resetForCompactionRetry()` reads the arrays.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Preserve retry side-effect evidence before clearing:** `src/agents/pi-embedded-subscribe.handlers.compaction.ts:90-93`
  - body: For `willRetry=true`, these clears run before `ctx.resetForCompactionRetry()`. That helper currently records `hasCommittedMessagingToolDeliveryEvidence(...)` into `state.hadDeterministicSideEffect` before it clears the arrays; clearing here makes the retry path see empty messaging evidence, so a compaction retry after a successful `message(action=send)` or `sessions_send` can lose side-effect accounting and end as abandoned. Move the clear to the non-retry path, or snapshot the evidence before clearing on retry.
  - confidence: 0.91
