## Summary

Keep open: the reported bug is source-reproducible, but this PR currently clears retry evidence too early, lacks real behavior proof, and carries unrelated release/dependency churn.

## What This Changes

The PR clears messaging-tool sent text, target, and media arrays during compaction end, updates the compaction test fixture, and currently includes unrelated release, version, workflow, and dependency changes.

## Best Possible Solution

Clear per-run messaging dedupe state only after normal compaction completes, or preserve retry side-effect evidence before clearing; add focused regression coverage for both normal compaction and retry evidence, then submit a clean branch with real behavior proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: seed `messagingToolSentTextsNormalized`, process a normal `compaction_end` with `willRetry: false`, then process a later `message_end` whose final text includes that stale normalized entry; I did not run a live harness in this read-only review.

## Solution Assessment

No. Clearing the arrays is directionally right for normal compaction, but doing it before the retry branch breaks deterministic side-effect accounting; the safer fix is to clear only the non-retry path or snapshot side-effect evidence before clearing.
