## Summary

Keep this PR open, but it should not merge as written: the intended normal-compaction cleanup is valid, yet the new clearing runs before the retry path can preserve messaging-tool delivery evidence for deterministic side-effect accounting.

## What This Changes

This PR clears messaging-tool sent text, target, and media tracking arrays on every `compaction_end` and updates the compaction handler test fixture.

## Best Possible Solution

Keep the lifecycle cleanup, but perform it after retry-side-effect evidence is captured or snapshot the evidence before clearing so normal compaction starts clean without changing retry liveness semantics.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path for both the original issue and the PR regression: stale normalized sent text can suppress a later `message_end`, and the proposed early clearing makes retry-side-effect evidence disappear before `resetForCompactionRetry()` reads it.

## Solution Assessment

No. Clearing the arrays unconditionally is the right direction for normal compaction, but the current placement is not the best solution because it changes retry behavior; move the cleanup after the retry snapshot or snapshot evidence first.

## Review Findings

- **[P1] Preserve delivery evidence before clearing retry state:** `src/agents/pi-embedded-subscribe.handlers.compaction.ts:90-93`
  - body: The new cleanup runs before the `willRetry` branch calls `resetForCompactionRetry()`. Move this cleanup after the retry reset path, or snapshot the evidence before clearing.
