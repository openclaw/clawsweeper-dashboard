## Best Possible Solution

## Solution Assessment

No, not as currently implemented: clearing per-run messaging state is the right direction, but doing it before the `willRetry` branch breaks retry-side delivery evidence accounting. The safer fix is to clear only after retry evidence is snapshotted or only on the normal non-retry transition.
