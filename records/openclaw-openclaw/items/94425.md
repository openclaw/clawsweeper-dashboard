## What This Changes

## Best Possible Solution

Keep the issue open until a PR creates the host skill directories before writable sandbox mount resolution, adds regression coverage through the sandbox context path, and provides real sandbox proof.

## Reproduction Assessment

Yes from source inspection: the writable sandbox setup path creates only the workspace root, and mount resolution drops missing `skills` and `.agents/skills` sources. I did not run a live Docker reproduction in this read-only review.
