## Summary

Keep open: current main and the latest release still omit read-only skill overlays when the host skill directories are absent, and https://github.com/openclaw/openclaw/pull/94820 is an unmerged candidate rather than an implemented fix.

## What This Changes

The issue reports that first-launch Docker sandboxes skip read-only overlays for `/workspace/skills` and `/workspace/.agents/skills` when the host skill directories do not yet exist.

## Best Possible Solution

Keep this issue as the canonical tracker and land a small sandbox fix that creates or materializes missing skill mount sources while preserving canonical-root validation, with regression coverage and first-launch sandbox proof.

## Reproduction Assessment

Yes, at source level: current main and `v2026.6.8` return false for missing skill mount sources, so the read-only overlay is filtered out. I did not run a live Docker first-launch repro in this read-only review.

## Solution Assessment

Yes, the requested boundary is the right one, but the best implementation must preserve canonical-root validation after creating missing directories. The open candidate PR should be reviewed or revised rather than closing this issue early.
