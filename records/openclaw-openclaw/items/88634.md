## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Recompute active chunks after final stop:** `extensions/telegram/src/lane-delivery-text-deliverer.ts:276-279`
  - body: `activeChunkIndex` is read before `stopDraftLane()` can trigger `onSupersededPreview({ retain: true })`. With the real draft stream, final overflow retention happens during that stop call, so this finalization still builds `remainingChunks` from the old index and can send chunks that the stream just retained as visible preview messages. Recompute from the post-stop retained prefix or have the draft stream report the retained chunk count before sending follow-ups.
  - confidence: 0.88
- **[P1] Only skip blocks after an answer draft streamed:** `extensions/telegram/src/bot-message-dispatch.ts:1806-1810`
  - body: This skips every text-only block whenever a partial-mode answer stream object exists. In the normal partial-mode path the stream is created at turn start, before any answer text is visible, so a first block payload can hit `continue` and the dispatcher returns without `deliverLaneText` or `sendPayload`. Gate this on `answerLane.hasStreamedMessage` or equivalent delivered-prefix proof.
  - confidence: 0.92
