## Summary

Keep open: this targets a real Telegram streaming duplication bug and includes useful live Telegram proof, but the patch introduces message-delivery blockers. Generic lane resets can make later long replies skip early chunks, and the partial-mode block skip can report unsent block text as delivered when no answer draft exists.

## What This Changes

The PR adds `activeChunkIndex` tracking to Telegram draft lanes, starts final delivery from that chunk, and skips redundant text-only block delivery in partial streaming mode.

## Best Possible Solution

Scope chunk advancement to retained overflow pages for the same final payload, reset it for new logical messages, and suppress text-only blocks only when an active partial draft has already carried the same text.

## Reproduction Assessment

Yes for the review findings: source inspection shows generic lane resets increment the chunk index, and text-only block skips can return as delivered without calling the delivery path. I did not run a live Telegram repro in this read-only review.

## Solution Assessment

No. The intended direction is sound, but this implementation puts same-payload chunk accounting on a generic reset path and suppresses block sends without proving a live draft owns that text.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep active chunk state scoped to one payload:** `extensions/telegram/src/bot-message-dispatch.ts:1127`
  - body: `resetDraftLaneState` is used for ordinary lane rotations, including tool-progress handoff and new-message setup. Incrementing `activeChunkIndex` here means the next long final answer starts at `chunks[activeChunkIndex]` and slices away earlier chunks, so a later long reply can silently lose its prefix. Track retained overflow pages for the same final payload, or reset this index for new logical messages.
  - confidence: 0.9
- **[P1] Do not mark skipped text blocks as delivered:** `extensions/telegram/src/bot-message-dispatch.ts:1803-1811`
  - body: In partial mode with block streaming explicitly enabled or forced for reasoning, `canStreamAnswerDraft` is false, so there is no answer draft stream carrying this text. This branch still skips `deliverLaneText`, sets `blockDelivered = true`, and lets the block pipeline treat the text as sent, which can suppress the matching final payload and drop the message. Gate the skip on an active streamed answer draft, or report it as not delivered.
  - confidence: 0.86
