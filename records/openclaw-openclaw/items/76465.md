## Summary

Keep open: the PR addresses a real doctor/install diagnostic gap, but it has a blocking correctness issue because inaccessible or otherwise unreadable live targets would be reported as stale symlinks, and the changelog entry still needs contributor credit.

## What This Changes

The PR adds a read-only doctor check for dangling global Node-prefix plugin-runtime symlinks, wires it into doctor startup, adds unit tests, and adds a changelog entry.

## Best Possible Solution

Land the diagnostic after narrowing stale detection to missing-target errors and adding changelog credit; keep actual symlink removal as a separate doctor --fix or postinstall follow-up for #76466.

## Reproduction Assessment

Yes for a source-level path: create a global node_modules/@scope/pkg symlink to a pruned plugin-runtime-deps target after the postinstall prune, and current main's doctor flow has no matching warning. I did not live-run the scenario because this sweep is read-only.

## Solution Assessment

No, not yet: a read-only doctor warning is the right narrow diagnostic shape, but the implementation must only classify missing-target stat failures as stale and must add the required changelog credit.
