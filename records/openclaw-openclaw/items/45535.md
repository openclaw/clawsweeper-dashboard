## What This Changes

This PR changes usage normalization to prefer positive token aliases over zero fields and adds prompt/completion token aliases to the old CLI runner usage helper.

## Best Possible Solution

Rework the fix on current main in `src/agents/usage.ts` and `src/agents/cli-output.ts`, with regression coverage for mixed-zero aliases and CLI prompt/completion aliases plus redacted real provider proof.

## Reproduction Assessment

Yes at source level: current main still lets `input_tokens: 0` win before later positive prompt-token aliases, and the current CLI parser still omits prompt/completion aliases. I did not run live Bailian, llama.cpp, or provider proof in this read-only review.

## Solution Assessment

No for the current branch. The direction is useful, but the implementation needs current-main rework and tests because part of the diff targets an old CLI helper and the zero fallback can overwrite prior nonzero stream usage.
