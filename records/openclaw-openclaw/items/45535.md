## Summary

Keep open. Current main still has the mixed zero/positive token-alias bug, and this PR is useful source work, but the submitted patch is stale and would miss the active CLI parser while introducing a zero-only JSONL usage overwrite risk.

## What This Changes

The PR changes usage normalization and an older CLI runner usage helper to prefer positive token aliases over zero-valued aliases for OpenAI-compatible provider usage payloads.

## Best Possible Solution

Port the positive-first alias resolver to the current `normalizeUsage` and active `cli-output` parser, preserve zero-only events as non-updates, and add focused regression tests plus the required changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: feed `normalizeUsage` a payload with `input_tokens: 0` and positive `prompt_tokens`, or feed the active CLI parser OpenAI-style prompt/completion-only usage; current main selects zero or misses the aliases.

## Solution Assessment

No. The positive-first normalization direction is right, but this patch updates an obsolete CLI helper and would let zero-only CLI usage events replace prior nonzero totals.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve prior nonzero JSONL usage:** `src/agents/cli-runner/helpers.ts:141-148`
  - body: The new finite fallback returns a usage object when every parsed token field is `0`. JSONL parsing keeps the latest parsed usage with `usage = toUsage(parsed.usage) ?? usage`, so a trailing zero-only bookkeeping event can replace earlier real totals and make the run look like it used no tokens. Keep zero-only usage returning `undefined` before updating stream state.
  - confidence: 0.91
- **[P2] Update the active CLI usage parser:** `src/agents/cli-runner/helpers.ts:130-138`
  - body: Current main no longer parses CLI usage from this helper; CLI execution imports `parseCliOutput` and streaming parsing from `src/agents/cli-output.ts`. Adding prompt/completion aliases here will not fix CLI usage reporting unless the active `toCliUsage` implementation is updated too.
  - confidence: 0.89
