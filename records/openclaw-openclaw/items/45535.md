## Summary

Keep open: the alias-normalization intent is still plausible, but this branch is unsafe to merge because it can turn all-zero CLI usage events into real usage objects, lacks required real behavior proof, and needs to be rebased/reworked against the current `src/agents/cli-output.ts` parser.

## What This Changes

The PR changes agent token usage normalization to prefer positive alias values over zero-valued fields and adds OpenAI prompt/completion aliases to the older CLI usage helper.

## Best Possible Solution

Reapply the useful alias-precedence behavior against current `src/agents/usage.ts` and `src/agents/cli-output.ts`, preserve all-zero usage records as absent, add focused regression coverage, and require redacted real provider or CLI-output proof before merge.

## Reproduction Assessment

Yes for the PR-introduced regression: the diff's fallback pass returns zero and JSONL parsing keeps the latest defined usage, so an all-zero bookkeeping event can overwrite earlier nonzero usage. The original Bailian symptom is source-explainable, but the linked reports were closed as fixed elsewhere on current main.

## Solution Assessment

No. The useful alias-precedence idea should be reworked against current main with all-zero overwrite guards, tests, and real behavior proof rather than merging this stale branch as-is.
