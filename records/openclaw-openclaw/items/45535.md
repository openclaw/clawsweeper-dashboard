## Summary

Keep this PR open. Current main still has the mixed zero/positive token-alias gap the PR targets, but the submitted patch is stale against the active CLI parser and has a JSONL usage regression, so it should be refreshed or replaced rather than merged as-is.

## What This Changes

The PR changes `src/agents/usage.ts` and the older CLI runner usage helper to prefer positive token aliases before falling back to finite zero aliases, while adding OpenAI-style prompt/completion and cached-token aliases for CLI usage parsing.

## Best Possible Solution

Refresh this PR or replace it with a current-main patch that adds a positive-first alias resolver to `normalizeUsage` without weakening cache normalization, updates `toCliUsage` in `src/agents/cli-output.ts`, preserves zero-only JSONL usage as `undefined`, and adds regression tests for mixed zero/positive aliases on both provider and CLI paths.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction: feed `normalizeUsage` a usage object with `input_tokens: 0` and positive `prompt_tokens`, or feed the CLI parser prompt/completion-only usage; current main still chooses the zero or misses the CLI aliases.

## Solution Assessment

No, not as currently submitted. The positive-first direction is right, but the patch targets an obsolete CLI helper and needs a guarded current-main implementation with tests rather than a direct merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve prior nonzero JSONL usage:** `src/agents/cli-runner/helpers.ts:141-148`
  - body: The new fallback returns a usage object when every parsed token field is `0`. JSONL parsing keeps the latest parsed usage with `usage = toUsage(parsed.usage) ?? usage`, so a trailing zero-only bookkeeping event can replace earlier real totals and make the run look like it used no tokens. Keep zero-only usage returning `undefined` before updating the stream state.
  - confidence: 0.91
- **[P2] Update the active CLI usage parser:** `src/agents/cli-runner/helpers.ts:130-138`
  - body: Current main no longer parses CLI usage from this helper; CLI execution imports `parseCliOutput` and streaming parsing from `src/agents/cli-output.ts`. Adding prompt/completion aliases here will not fix CLI usage reporting unless the active `toCliUsage` implementation is updated too.
  - confidence: 0.89
