## Summary

Keep open: the token-alias bug is still source-reproducible on current main, and the PR’s direction is useful but the branch is not correct enough to merge because it updates an obsolete CLI helper and can let zero-only CLI usage overwrite real totals.

## What This Changes

The PR changes `normalizeUsage` and the older CLI runner usage helper to prefer positive token aliases over zero-valued aliases for OpenAI-compatible usage payloads.

## Best Possible Solution

Implement positive-first alias selection in the current `normalizeUsage` and active `cli-output` parser, keep zero-only usage events as non-updates, and add regressions plus the required changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: pass `normalizeUsage` a payload with `input_tokens: 0` and positive `prompt_tokens`, or feed the active CLI parser OpenAI-style prompt/completion-only usage; current main selects zero or misses the aliases.

## Solution Assessment

No. The positive-first normalization direction is right, but this PR updates an obsolete CLI helper and would let zero-only usage objects overwrite prior nonzero JSONL totals; the narrower maintainable fix belongs in current `usage.ts` and `cli-output.ts`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve prior nonzero JSONL usage:** `src/agents/cli-runner/helpers.ts:141-148`
  - body: The new finite fallback returns a usage object when every parsed token field is `0`. JSONL parsing keeps the latest parsed usage with `usage = toUsage(parsed.usage) ?? usage`, so a trailing zero-only bookkeeping event can replace earlier real totals and make the run look like it used no tokens. Keep zero-only usage returning `undefined` before updating stream state.
  - confidence: 0.91
- **[P2] Update the active CLI usage parser:** `src/agents/cli-runner/helpers.ts:130-138`
  - body: Current main no longer parses CLI usage from this helper; CLI execution imports `parseCliOutput` and streaming parsing from `src/agents/cli-output.ts`. Adding prompt/completion aliases here will not fix CLI usage reporting unless the active `toCliUsage` implementation is updated too.
  - confidence: 0.89
