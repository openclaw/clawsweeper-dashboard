## What This Changes

The PR changes usage normalization to prefer positive token aliases over zero values and adds OpenAI prompt/completion aliases to the older CLI runner usage helper.

## Best Possible Solution

Rework the normalization on current main with all-zero overwrite guards, cache-subtraction-safe alias precedence, focused tests, and redacted provider or CLI proof, while coordinating with https://github.com/openclaw/openclaw/pull/78085 for the narrower CLI alias fallback.
