## Summary

Keep this PR open: it addresses a valid linked session-lock cleanup bug, but the current patch still preserves some unrelated live-PID owners because the new argv classifier accepts generic entrypoint filenames without an OpenClaw-specific marker.

## What This Changes

The PR adds owner-argv inspection to session lock cleanup and doctor, with regression tests, doctor docs, and changelog coverage so `doctor --fix` can remove fresh live-PID locks owned by non-OpenClaw processes.

## Best Possible Solution

Land a corrected version that removes fresh live locks only when the owner argv is readable and proven non-OpenClaw, while preserving active OpenClaw and unknown owners.

## Reproduction Assessment

Yes, source-level. Current main leaves a fresh live-PID lock at stale=no, and the PR's remaining bug is source-reproducible with an unrelated argv such as `node /srv/app/dist/index.js`.

## Solution Assessment

No. The repair direction is narrow and appropriate, but the current classifier should require a stronger OpenClaw-specific owner marker before preserving a fresh live-PID lock.
