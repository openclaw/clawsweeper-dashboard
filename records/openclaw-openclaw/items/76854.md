## Summary

Keep open: this PR addresses a valid false-live session-lock bug, but the proposed argv classifier is still too broad and can preserve the same class of stale locks for unrelated Node/Bun services with common entrypoint names.

## What This Changes

The PR adds owner-argv inspection to session lock cleanup and doctor reporting, with regression tests, docs, and a changelog entry for removing fresh live-PID locks owned by non-OpenClaw processes.

## Best Possible Solution

Land a corrected version that preserves readable live owners only with a strong OpenClaw-specific argv signal, adds a generic `dist/index.js` non-owner regression, and then closes the linked issue after merge.

## Reproduction Assessment

Yes. Source inspection of current main shows a fresh lock with a live PID is not stale unless the PID is dead, recycled by starttime, malformed, or too old; the linked issue's `stale=no` output matches that path.

## Solution Assessment

No. Owner-argv cleanup is the right narrow direction, but this implementation needs a stricter OpenClaw-specific marker before preserving a readable live owner.

## Review Findings

Overall correctness: patch is incorrect

- **[P2] Require a real OpenClaw argv marker:** `src/agents/session-write-lock.ts:397-409`
