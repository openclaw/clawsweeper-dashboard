## Summary

Keep open: the PR targets a valid false-live session lock bug and includes real terminal proof, but the branch still preserves generic Node entrypoints as OpenClaw owners, so it can leave the reported false-live locks behind.

## What This Changes

The PR adds session lock owner-argv inspection to `doctor --fix` cleanup, with tests, docs, and changelog coverage for removing fresh live-PID locks owned by non-OpenClaw processes.

## Best Possible Solution

Tighten owner classification to preserve only verified OpenClaw argv, ideally by sharing the established gateway argv logic or requiring an explicit OpenClaw marker, and add a regression for generic entrypoint false positives.

## Reproduction Assessment

Yes, at source level. Current main does not classify fresh live non-OpenClaw owner PIDs as stale, and PR head still preserves generic entrypoint argv such as `node /srv/app/dist/index.js`.

## Solution Assessment

No. Owner argv inspection is the right narrow direction, but this implementation needs a stronger OpenClaw-specific marker before preserving fresh live locks.
