## Summary

Keep open: the repair direction matches the linked bug, but the PR's argv classifier is too broad and can still preserve unrelated Node processes as OpenClaw owners, so the earlier ClawSweeper blocker still applies.

## What This Changes

The PR adds owner argv inspection to session lock cleanup and doctor so fresh live-PID locks can be removed when the owner is readable and non-OpenClaw, with tests, docs, and a changelog entry.

## Best Possible Solution

Land a corrected version that uses a strong OpenClaw-specific owner classifier, keeps unknown or active owners preserved, and then closes #76823 after merge.

## Reproduction Assessment

Yes. Source inspection of current main plus the linked issue output gives a high-confidence path: create a fresh lock with a live non-OpenClaw PID, run doctor repair, and it remains stale=no because no existing stale reason is set.

## Solution Assessment

No. The overall repair is narrow, but the current classifier should require an OpenClaw-specific marker rather than accepting generic dist/index.js-style argv suffixes.
