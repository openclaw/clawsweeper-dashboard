## Summary

Keep open: current main still lacks false-live owner-argv cleanup, and this PR is a focused fix with docs, changelog, and regression coverage. I found no blocking functional or security issue in source/diff review, but local tests were not runnable because `pnpm` is unavailable here and PR CI was still in progress.

## What This Changes

This PR adds conservative owner-argv detection to session lock cleanup so fresh live-PID locks owned by non-OpenClaw processes can be removed, with tests, docs, and a changelog entry.

## Best Possible Solution

Land this narrow owner-argv cleanup after normal CI passes, preserving the fail-closed rule for active, unreadable, or unknown OpenClaw owners.

## Reproduction Assessment

Yes. Source inspection shows that on current main a fresh `.jsonl.lock` with an alive reused PID gets no stale reason, so doctor `--fix` reports it but does not remove it.

## Solution Assessment

Yes. The PR is narrower than deleting all locks when the gateway is down: it removes a fresh live-PID lock only when owner argv is readable and not recognizable as OpenClaw, while preserving unknown owners.
