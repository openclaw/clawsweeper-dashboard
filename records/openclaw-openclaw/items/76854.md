## Summary

Keep open: the PR addresses a real current-main lock cleanup gap and includes tests/docs/changelog, but the proposed argv classifier is too broad and can still preserve false-live locks owned by unrelated Node processes with common entrypoint names.

## What This Changes

The PR adds owner argv inspection to session lock cleanup and doctor so readable live-PID locks from non-OpenClaw processes can be marked stale and removed, with tests, docs, and changelog coverage.

## Best Possible Solution

Land a corrected version that only preserves fresh live-PID locks when argv contains a strong OpenClaw-specific owner marker and keeps unknown or unreadable owners preserved.

## Reproduction Assessment

Yes, source-level. Current main leaves a fresh live reused-PID lock at stale=no, and the PR's remaining blocker is source-reproducible because a generic argv like node /srv/app/dist/index.js is classified as OpenClaw.

## Solution Assessment

No. The repair direction is narrow and appropriate, but the current PR should require an OpenClaw-specific argv marker before preserving a live-PID owner.
