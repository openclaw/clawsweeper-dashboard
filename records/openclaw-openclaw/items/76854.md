## Summary

Keep open: this PR targets a real source-reproducible doctor/session-lock cleanup gap and includes sufficient terminal proof, but the current classifier still treats generic Node entrypoints as OpenClaw owners, so it can preserve the false-live locks it is meant to remove.

## What This Changes

The branch adds owner argv inspection to session lock cleanup, threads it through doctor repair, adds regression tests, and updates doctor docs plus the changelog.

## Best Possible Solution

Tighten the owner classifier to require a strong OpenClaw-specific marker, preserve active or unknown owners, and cover generic Node entrypoints plus the doctor repair path with regression tests.

## Reproduction Assessment

Yes. Current main is source-reproducible because cleanStaleLockFiles leaves fresh live-PID locks stale=false unless they are dead, old, recycled, invalid, or orphan-self, and doctor repair cannot inspect owner argv. The PR blocker is source-reproducible with argv such as node /srv/app/dist/index.js.

## Solution Assessment

No. Owner argv inspection is the right repair path, but this implementation should require an OpenClaw-specific argv marker, or reuse the existing isGatewayArgv-style guard, before preserving a fresh live lock.
