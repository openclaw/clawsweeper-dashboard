## Summary

Keep open: the repair direction matches the linked bug, but the current classifier still preserves generic Node entrypoints as OpenClaw owners, so a false-live lock can survive.

## What This Changes

The PR adds owner argv inspection to session lock cleanup and doctor so readable live-PID locks from non-OpenClaw processes can be marked stale and removed.

## Best Possible Solution

Land a corrected version that preserves live locks only for strong OpenClaw owner markers and treats readable generic app entrypoints as non-OpenClaw.

## Reproduction Assessment

Yes, source-level. Current main leaves a fresh live-PID lock at stale=no, and the PR's remaining blocker is reproducible from the new classifier accepting a generic argv like node /srv/app/dist/index.js.

## Solution Assessment

No, not as written. Owner argv inspection is the narrow maintainable direction, but the classifier needs an OpenClaw-specific marker before it preserves a live owner.
