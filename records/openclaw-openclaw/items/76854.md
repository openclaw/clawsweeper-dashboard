## Summary

Keep open: the current-main bug is source-reproducible and this PR targets the right cleanup path, but the proposed argv classifier is too broad and can still preserve false-live locks owned by unrelated Node services.

## What This Changes

The PR adds owner argv inspection to session lock cleanup and doctor repair so fresh live-PID locks proven to belong to non-OpenClaw processes can be removed, with tests, docs, and changelog coverage.

## Best Possible Solution

Land a corrected version that uses a strong OpenClaw/gateway argv predicate, preserves active or unknown owners, and removes only live owners proven to be non-OpenClaw.

## Reproduction Assessment

Yes, source-level. On current main, a fresh lock with an alive PID and no recycled-starttime or age reason returns stale=false, and doctor only removes locks already marked stale.

## Solution Assessment

No. The PR is in the right module, but the classifier should require an OpenClaw-specific argv marker instead of treating generic entrypoint suffixes as proof of ownership.
