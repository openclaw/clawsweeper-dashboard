## Summary

Keep open: the PR targets a real stale-lock bug and supplies real terminal proof, but the current classifier still preserves false-live locks for unrelated Node services whose argv ends in generic entrypoint names.

## What This Changes

The PR adds owner argv inspection to session lock cleanup and doctor repair, with tests, doctor docs, and a changelog entry for removing fresh live-PID locks proven to be owned by non-OpenClaw processes.

## Best Possible Solution

Tighten session owner classification to require a strong OpenClaw-specific argv marker while preserving active OpenClaw owners and unknown or unreadable owners.

## Reproduction Assessment

Yes. Source inspection shows current main leaves a fresh .jsonl.lock with a live PID non-stale, and the PR branch's remaining blocker is source-reproducible with argv like ["node", "/srv/app/dist/index.js"].

## Solution Assessment

No. Owner argv inspection is the right narrow repair path, but this implementation should require an OpenClaw-specific marker instead of accepting generic entrypoint filenames.
