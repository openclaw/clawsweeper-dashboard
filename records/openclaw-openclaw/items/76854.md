## Summary

Keep open: the PR targets a valid false-live session lock bug, but the current classifier is too broad and can still preserve unrelated Node processes as OpenClaw owners.

## What This Changes

The PR adds owner-argv inspection to session lock cleanup, doctor tests, docs, and changelog so fresh live-PID locks can be removed when the owner is readable and non-OpenClaw.

## Best Possible Solution

Land a corrected owner-check implementation that requires a strong OpenClaw marker, preserves unreadable or unknown owners, and closes the linked issue after merge.

## Reproduction Assessment

Yes. Source inspection shows current main leaves a fresh live-PID lock non-stale, and the existing test baseline expects that fresh live lock to remain during cleanup.

## Solution Assessment

No. The owner-argv direction is narrow and maintainable, but the PR needs a stricter classifier so generic Node entrypoints do not bypass false-live cleanup.
