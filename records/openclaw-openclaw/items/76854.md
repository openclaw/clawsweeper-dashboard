## Summary

Keep this PR open because the proposed owner-argv repair is aimed at a real false-live lock bug, but the current patch still preserves unrelated Node processes whose argv ends in generic entrypoint names, so it can miss the exact cleanup case it is trying to fix.

## What This Changes

The PR adds owner-argv inspection to session lock cleanup and doctor repair, with tests, docs, and changelog coverage for removing fresh live-PID locks owned by non-OpenClaw processes.

## Best Possible Solution

Tighten owner classification to preserve only verified OpenClaw argv and add a regression for generic entrypoint false positives.

## Reproduction Assessment

Yes, at source level. Current main does not classify fresh live non-OpenClaw owner PIDs as stale, and PR head still preserves generic entrypoint argv such as node /srv/app/dist/index.js.

## Solution Assessment

No. Owner argv inspection is the right narrow direction, but this implementation needs a stronger OpenClaw-specific marker before preserving fresh live locks.
