## Summary

Keep open: the PR targets a valid false-live session lock bug, but the owner argv classifier still preserves unrelated Node processes with generic entrypoint names, so the linked issue can remain unresolved for common reused-PID cases.

## What This Changes

The PR adds owner argv inspection to session lock cleanup and doctor reporting so readable live-PID locks owned by non-OpenClaw processes can be marked stale and removed.

## Best Possible Solution

Land a corrected version that uses a strict OpenClaw-specific owner classifier, preserves unknown or unreadable owners, and closes the linked bug after merge.

## Reproduction Assessment

Yes. Source inspection of current main gives a high-confidence path: create a fresh lock with a live PID and age below staleMs, and cleanStaleLockFiles will keep staleReasons empty and remove nothing; the linked issue's stale=no output matches that path.

## Solution Assessment

No. The repair direction is narrow and appropriate, but this implementation needs a stricter OpenClaw-specific argv marker so generic Node entrypoints do not bypass cleanup.
