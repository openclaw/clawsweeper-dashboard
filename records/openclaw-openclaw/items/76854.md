## Summary

Keep open: the underlying false-live session-lock bug is source-reproducible and the PR is directionally right, but the new argv classifier is too broad and can still preserve locks owned by unrelated Node processes with common entry names.

## What This Changes

The PR updates session lock inspection and doctor repair to remove fresh live-PID `.jsonl.lock` files when the PID argv is readable and non-OpenClaw, with docs, changelog, and regression tests.

## Best Possible Solution

Land a corrected version that only preserves readable live owners with a strong OpenClaw-specific argv signal, while continuing to preserve unreadable or unknown owners and close the linked issue after merge.

## Reproduction Assessment

Yes. Source inspection of current main shows a fresh lock with a live reused PID is not stale unless the PID is dead, recycled by starttime, malformed, or too old; the linked issue’s `stale=no` output matches that path.

## Solution Assessment

No. The repair direction is narrow and appropriate, but the PR’s owner classifier needs a stricter OpenClaw-specific marker before merge so common non-OpenClaw Node entry paths do not bypass cleanup.
