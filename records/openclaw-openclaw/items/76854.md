## Summary

Keep open: the bug and repair direction are valid, but the latest PR head still has a blocking classifier bug because generic Node entrypoints can be treated as OpenClaw owners, so false-live locks can survive.

## What This Changes

The PR adds session-lock owner argv inspection so doctor/acquisition can remove fresh live-PID locks proven to belong to non-OpenClaw processes, with regression tests, docs, and changelog coverage.

## Best Possible Solution

Land a corrected version that requires a strong OpenClaw-specific owner marker, preserves unknown or active OpenClaw owners, and cleans only proven non-OpenClaw live owners.

## Reproduction Assessment

Yes, source-level. Current main leaves a fresh live-PID lock at stale=no when the PID is alive and not recycled/old, and the PR blocker is source-reproducible with argv like node /srv/app/dist/index.js.

## Solution Assessment

No. The approach is the right narrow area, but the classifier needs an OpenClaw-specific marker instead of treating generic entrypoint suffixes as sufficient proof.
