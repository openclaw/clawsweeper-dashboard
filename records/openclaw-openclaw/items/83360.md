## Summary

Keep open: current main still has the conflicting behavior described here. The gateway auto-update scheduler launches `openclaw update` as a child process, while the update CLI intentionally refuses package updates when the running gateway PID is in the updater ancestry; the related open marker-stripping PR is useful context but does not address this systemd ancestry failure.

## What This Changes

The issue reports that systemd-managed auto-update cannot complete because the gateway-spawned updater is blocked by the updater's own gateway-ancestry safety guard.

## Best Possible Solution

Route gateway auto-updates through a supervisor-owned handoff that can safely stop/update/restart the managed service, preserve the manual ancestry guard, and surface failed handoffs through update status.

## Reproduction Assessment

Yes for a source-level reproduction: the scheduler launches the updater as a gateway child, and the CLI explicitly blocks when the gateway PID is an ancestor. I did not run a live systemd install in this read-only pass.

## Solution Assessment

Yes, keeping this open for a handoff fix is the best path. The safer fix is not to weaken the guard, but to make auto-update run from outside the gateway-owned process tree or through the existing managed-service handoff pattern.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/83360. Current main runs gateway auto-update from `src/infra/update-startup.ts` via `runCommandWithTimeout`, making `openclaw update` a child of the managed gateway; `src/cli/update-cli/update-command.ts` then correctly refuses package updates when the active gateway PID is an ancestor. Implement a service-safe auto-update handoff for package installs, preferably by reusing or extracting the managed-service handoff pattern from `src/gateway/server-methods/update.ts` / `update-managed-service-handoff.ts`. Do not weaken the manual gateway-ancestry guard, do not change the `update.auto.*` config contract, and do not relax systemd `KillMode=control-group` without explicit maintainer approval. Add regression coverage for the systemd/gateway-ancestor auto-update path and update user-facing docs/changelog if behavior changes. Coordinate with https://github.com/openclaw/openclaw/issues/78492 and https://github.com/openclaw/openclaw/pull/78494 so the marker-leak fix does not mask the remaining ancestry failure. Before landing, get real systemd package-update proof on a managed service install.
