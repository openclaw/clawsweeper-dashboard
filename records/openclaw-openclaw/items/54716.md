## Summary

Keep open: this PR targets a valid sessions.list bug and has useful before/after proof, but it is not merge-ready because current main moved the active combined-store loader to `src/config/sessions/combined-store-gateway.ts`, the PR is conflicting, and the proposed multi-store path can still canonicalize legacy rows under the wrong agent when configured agent order differs.

## What This Changes

The branch changes gateway session-store combining so literal `session.store` paths under `agents/<id>/sessions/sessions.json` discover sibling agent stores, with regression tests for the linked sessions.list undercount.

## Best Possible Solution

Refresh or replace this PR against current main by moving the fix into `src/config/sessions/combined-store-gateway.ts`, scoping literal agent-layout discovery to that agents root, deriving target agent IDs from the on-disk path, and keeping true shared literal stores unchanged.
