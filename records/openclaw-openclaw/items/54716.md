## Summary

Keep open. The underlying bug is still present on current main, and the PR has useful proof, but the branch is stale and patches the old gateway-local loader instead of the active config/sessions loader.

## What This Changes

The PR changes gateway session utilities to treat literal per-agent session store paths as multi-agent discovery roots and adds sibling-store regression coverage.

## Best Possible Solution

Move literal agent-layout handling into src/config/sessions so same-root sibling stores are discovered, true shared literal stores remain single-store, resolved paths and on-disk agent IDs are preserved, and regression coverage lives with the active loader/targets.

## Reproduction Assessment

Yes. Configure session.store to a literal .../agents/main/sessions/sessions.json path with a sibling agent store and call sessions.list; current-main source still returns from the literal-store branch before sibling discovery, and the linked proof shows count=1 before versus count=2 after the branch.

## Solution Assessment

No, not as currently patched. The intended behavior is right, but the fix needs to move into the active src/config/sessions loader and resolver while preserving resolved-path and on-disk-agent-id semantics.
