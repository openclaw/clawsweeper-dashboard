## Repair Work Prompt

Port the proof-backed fix from https://github.com/openclaw/openclaw/pull/54716 to current main. The bug is that a literal `session.store` path shaped like `.../agents/<id>/sessions/sessions.json` is treated as a single shared store by `loadCombinedSessionStoreForGateway`, so `sessions.list` misses sibling agent stores. Implement in `src/config/sessions/combined-store-gateway.ts` and/or `src/config/sessions/targets.ts`, not the stale in-file gateway implementation. Preserve true shared literal store paths as single-store. For agent-layout literal paths, discover sibling stores only under the same resolved agents root, avoid pulling default state-dir stores, return resolved store paths for downstream usage/transcript lookup, and canonicalize legacy keys using the on-disk agent ID. Add focused tests for sibling discovery, no default-root leakage, resolved path return behavior, and wrong-config-order/on-disk agent ID canonicalization. Do not change unrelated `sessions.list` filtering, configuredAgentsOnly behavior, provider routing, or plugin/session policy. No changelog should be required until maintainer landing unless policy says otherwise.
