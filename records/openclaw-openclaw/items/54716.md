## Summary

Keep open: this is a valid, well-proved bug fix direction, but the branch is not currently mergeable or correct against current main because it patches the old gateway-local loader while the active loader lives under `src/config/sessions` and still returns early for literal non-template stores.

## What This Changes

The branch changes gateway session aggregation so literal `agents/<id>/sessions/sessions.json` store paths discover sibling per-agent stores, scopes that discovery to the literal agents root, and adds regression coverage.

## Best Possible Solution

Land an equivalent current-main repair in `src/config/sessions/combined-store-gateway.ts` and `src/config/sessions/targets.ts` so literal agent-layout stores discover only same-root sibling stores, preserve resolved paths, and derive agent identity from disk layout.

## Reproduction Assessment

Yes. A literal `.../agents/main/sessions/sessions.json` config with a sibling agent store follows the active loader's literal-store early return, and `sessions.list` consumes that combined store, so sibling sessions are omitted; the contributor's API-backed artifact shows the same count changing from 1 to 2 after the branch fix.

## Solution Assessment

No, not as written. The direction is right, but the patch must be ported to the active `src/config/sessions` loader and target resolver before it is the narrow maintainable solution.
