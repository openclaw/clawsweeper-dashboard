The branch updates gateway combined session-store loading and target discovery so literal `agents/<id>/sessions/sessions.json` paths use scoped sibling-agent discovery, with regressions for sibling stores, root scoping, and on-disk agent ownership.

## Best Possible Solution

Land this original scoped fix, or an equivalent narrow patch, after maintainers accept the literal agent-tree `session.store` behavior change and keep the added regression coverage.

## Reproduction Assessment

## Solution Assessment

Yes. The PR is the narrowest maintainable fix I found: only literal paths that structurally match the per-agent store layout join scoped discovery, while true shared literal stores keep the old single-store behavior.
