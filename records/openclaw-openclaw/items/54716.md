## Summary

Keep open: the bug and intended fix are valid, but this PR still patches the old gateway-local loader while current main routes `sessions.list` through `src/config/sessions`, and the branch needs a focused repair or replacement before merge.

## What This Changes

The PR changes gateway session aggregation so literal `agents/<id>/sessions/sessions.json` store paths discover same-root sibling agent stores, with regression tests.

## Best Possible Solution

Repair or replace this PR on current main by moving literal per-agent store handling into `src/config/sessions`, preserving same-root discovery, resolved store paths, and on-disk agent IDs with focused regressions.

## Reproduction Assessment

Yes. Configure `session.store` to a literal `.../agents/main/sessions/sessions.json` path with a sibling agent store and call `sessions.list`; current-main source returns from the literal-store branch before sibling discovery, and the linked proof shows one row before versus two after.

## Solution Assessment

No, not as currently patched. The intended behavior is the right narrow fix, but it needs to move into the active `src/config/sessions` loader/resolver and preserve resolved-path plus on-disk-agent-id semantics.
