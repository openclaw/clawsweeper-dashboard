## Summary

Keep open: the bug and fix direction are valid, and the linked artifacts show real before/after behavior, but the branch now patches a stale gateway-local loader while current main uses the active `src/config/sessions` loader.

## What This Changes

The PR modifies gateway session-store aggregation so literal agent-layout `session.store` paths discover sibling agent stores, with regression tests for literal main-agent paths and scoped discovery.

## Best Possible Solution

Land a current-main fix in `src/config/sessions` that treats literal agent-layout stores as scoped multi-agent roots while preserving true shared literal stores as single-store paths.

## Reproduction Assessment

Yes. Configure `session.store` to a literal `.../agents/main/sessions/sessions.json` with a sibling agent store, then call `sessions.list`; current-main source still returns from the literal single-store branch before sibling discovery, and the linked artifacts show count improving from one to two after the branch change.

## Solution Assessment

No, not as currently patched. The direction is right, but the implementation must move into the active `src/config/sessions` loader and resolver, preserve resolved literal paths, and canonicalize with the on-disk agent id.
