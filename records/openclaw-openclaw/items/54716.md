## Summary

Keep open. The intended bug fix is valid and has real before/after proof, but this PR is not mergeable as-is: current main now uses `src/config/sessions/combined-store-gateway.ts`, while the branch edits the stale gateway-local implementation and still needs resolved-path and on-disk-agent-id handling preserved in the active loader/resolver.

## What This Changes

The PR changes gateway session aggregation so literal per-agent `session.store` paths discover sibling agent stores, and adds regression tests for that scenario.

## Best Possible Solution

Port the proof-backed behavior into `src/config/sessions` on current main, keeping true shared literal stores single-store while literal `agents/<id>/sessions/sessions.json` paths discover same-root sibling stores with resolved paths and correct agent IDs.

## Reproduction Assessment

Yes, source-reproducible. Configure `session.store` to a literal `.../agents/main/sessions/sessions.json` path with a sibling agent store and call `sessions.list`; current main returns from the literal-store branch before sibling discovery, and the linked proof shows count 1 before versus count 2 after.

## Solution Assessment

No, not as currently patched. The intended behavior is the right narrow fix, but the implementation must move into the active `src/config/sessions` loader/resolver and preserve same-root scoping, resolved store paths, and on-disk agent IDs.
