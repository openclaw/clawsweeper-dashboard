## Summary

Keep open: the proof-backed behavior is useful, but this PR is now conflicting and patches the stale gateway-local loader instead of current main's active `src/config/sessions` implementation.

## What This Changes

The PR changes gateway session aggregation so literal per-agent `session.store` paths can discover sibling agent session stores.

## Best Possible Solution

Port the proven behavior into `src/config/sessions`, keeping shared literal stores single-store while literal `agents/<id>/sessions/sessions.json` paths discover same-root siblings with resolved paths and on-disk agent ids.

## Reproduction Assessment

Yes. Configure `session.store` to a literal `.../agents/main/sessions/sessions.json` path with a sibling agent store and call `sessions.list`; current main returns from the literal-store branch before sibling discovery, and the linked proof shows count 1 before versus 2 after.

## Solution Assessment

No as-is. The intended behavior is the right narrow fix, but it must be ported to the active `src/config/sessions` loader and preserve same-root scoping, resolved store paths, and on-disk agent ids.
