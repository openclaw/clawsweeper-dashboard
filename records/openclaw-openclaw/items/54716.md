## Summary

Keep open. The reported bug is source-reproducible and the PR has useful before/after proof, but the current PR head is stale against current main: it edits the old gateway-local loader while current main uses `src/config/sessions/combined-store-gateway.ts`, and the branch is currently conflicting.

## What This Changes

The PR changes gateway session aggregation so literal per-agent `session.store` paths discover same-root sibling agent stores, with regression tests.

## Best Possible Solution

Repair or replace this PR on current main by implementing literal per-agent store handling in `src/config/sessions`, preserving same-root scoping, resolved store paths, and on-disk agent IDs with focused regressions.

## Reproduction Assessment

Yes. Configure `session.store` to a literal `.../agents/main/sessions/sessions.json` path with a sibling agent store and call `sessions.list`; current-main source returns before sibling discovery, and the linked proof shows count 1 before versus count 2 after.

## Solution Assessment

No, not as currently patched. The intended behavior is the right narrow fix, but the implementation must move into the active `src/config/sessions` loader/resolver and preserve resolved-path plus on-disk-agent-id semantics.
