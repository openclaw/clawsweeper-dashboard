## Summary

Keep open. The PR targets a real `sessions.list` bug and has useful proof, but the branch still patches the older gateway-local implementation while current `main` loads sessions through `src/config/sessions`; as written it will not repair the active path.

## What This Changes

The PR changes gateway session aggregation so literal `agents/<id>/sessions/sessions.json` store paths discover sibling per-agent stores, scopes that discovery to the literal agents root, and adds regression coverage.

## Best Possible Solution

Port the useful branch direction into `src/config/sessions/combined-store-gateway.ts` and `src/config/sessions/targets.ts`, keeping true shared literal stores single-file while literal per-agent stores discover same-root siblings with regression coverage.

## Reproduction Assessment

Yes. Configure `session.store` to a literal `.../agents/main/sessions/sessions.json` path with a sibling agent store, then call `sessions.list`; current source returns from the literal-store branch before sibling discovery, and the linked artifact shows the count changing from 1 to 2 after the branch fix.

## Solution Assessment

No, not as written. The direction is right, but the patch must move to the active `src/config/sessions` loader and target resolver before it is the narrow maintainable fix.
