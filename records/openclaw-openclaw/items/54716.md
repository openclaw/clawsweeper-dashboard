## What This Changes

The PR changes gateway session aggregation so literal per-agent `session.store` paths discover sibling agent stores and adds regression coverage for that scenario.

## Best Possible Solution

Port the proof-backed behavior into the active `src/config/sessions` loader and resolver so literal `agents/<id>/sessions/sessions.json` paths discover same-root sibling stores while true shared literal stores remain single-store.

## Reproduction Assessment

Yes. Configure `session.store` to a literal `.../agents/main/sessions/sessions.json` path with a sibling agent store and call `sessions.list`; current-main source returns from the literal-store branch before sibling discovery, and the linked proof shows count 1 before versus 2 after.

## Solution Assessment

No, not as currently patched. The intended behavior is the right narrow fix, but the implementation needs to move into the active `src/config/sessions` loader/resolver and preserve same-root scoping, resolved store paths, and on-disk agent ids.
