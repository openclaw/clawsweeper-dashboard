## Summary

Keep open. The PR targets a real current-main bug and has useful proof, but the branch is now conflicting and still patches the old gateway-local loader instead of the active `src/config/sessions` implementation, so it needs repair before merge.

## What This Changes

The PR changes gateway session-store loading and regression tests so literal per-agent `sessions.json` paths discover sibling agent stores.

## Best Possible Solution

Port the proven literal agent-layout behavior into the active `src/config/sessions` loader and target resolver, with focused regressions for the literal `agents/main/sessions/sessions.json` configuration.

## Reproduction Assessment

Yes. Configure `session.store` to a literal `.../agents/main/sessions/sessions.json` path with a sibling agent store and call `sessions.list`; current-main source returns from the literal-store branch before sibling discovery, and the linked proof shows the before/after API counts.

## Solution Assessment

No, not as written. The fix direction is right, but it needs to move from the stale gateway-local implementation into `src/config/sessions/combined-store-gateway.ts` and `src/config/sessions/targets.ts` with the path and agent-id details preserved.
