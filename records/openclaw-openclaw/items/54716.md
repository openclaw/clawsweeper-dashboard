## Summary

Keep open: the reported bug and fix direction are valid, but this branch still patches the old gateway-local loader while current main uses the extracted `src/config/sessions` loader, so it needs a focused port before merge.

## What This Changes

The PR changes gateway session aggregation so literal `agents/<id>/sessions/sessions.json` store paths discover sibling agent stores, scopes discovery to the literal agents root, and adds regression coverage.

## Best Possible Solution

Land an equivalent current-main repair in `src/config/sessions/combined-store-gateway.ts` and `src/config/sessions/targets.ts` so literal agent-layout stores discover only same-root sibling stores while true shared literal stores stay single-store.

## Reproduction Assessment

Yes. A literal `.../agents/main/sessions/sessions.json` config with a sibling agent store follows the active loader's literal-store early return, and the inspected proof artifact shows the real `sessions.list` count changing from 1 to 2 after the branch fix.

## Solution Assessment

No, not as written. The direction is right, but the patch must be ported to the active `src/config/sessions` loader and target resolver, preserving resolved paths and disk-derived agent IDs.
