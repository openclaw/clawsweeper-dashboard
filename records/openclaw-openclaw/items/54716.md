## Summary

Keep open. This PR targets a real `sessions.list` bug and has sufficient before/after proof, but the branch now edits a stale gateway-local loader while current `main` uses `src/config/sessions/combined-store-gateway.ts`; the branch is also conflicting and needs a focused repair before merge.

## What This Changes

The PR changes session-store loading so literal agent-layout `session.store` paths discover sibling agent stores and adds regressions for that scenario.

## Best Possible Solution

Port the proven literal agent-layout behavior into the active `src/config/sessions` loader and target resolver, with focused regressions that preserve single-store behavior for non-agent-layout literal paths.

## Reproduction Assessment

Yes. Source inspection on current `main` shows literal non-template `session.store` returns after reading one default-agent store, and the linked API-backed proof shows the same scenario improving from one session to two after the branch fix.

## Solution Assessment

No, not as currently patched. The direction is right, but the implementation must move to the active `src/config/sessions` loader/resolver and preserve resolved-path and on-disk-agent canonicalization details.
