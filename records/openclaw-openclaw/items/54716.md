## Summary

Keep open: the PR targets a real `sessions.list` bug and has sufficient before/after proof, but it is not mergeable as written because current main moved the active combined-store loader into `src/config/sessions` while this branch still patches the stale gateway-local implementation.

## What This Changes

The PR changes gateway session-store loading so literal `session.store` paths under `agents/<id>/sessions/sessions.json` discover sibling agent stores, with regression coverage.

## Best Possible Solution

Port the proven literal agent-layout behavior into the current `src/config/sessions` loader and target resolver, add focused regressions, then land this PR or a narrow replacement.

## Reproduction Assessment

Yes. Configure `session.store` to a literal `.../agents/main/sessions/sessions.json` path with a sibling agent store and call `sessions.list`; current-main source returns from the literal-store branch before sibling discovery, and the linked proof shows `apiCount=1` before the branch fix.

## Solution Assessment

No, not as written. The fix direction is right, but the implementation must move to `src/config/sessions/combined-store-gateway.ts` and `src/config/sessions/targets.ts` with the resolver/path details preserved.
