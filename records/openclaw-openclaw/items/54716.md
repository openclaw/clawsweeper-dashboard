## Summary

Keep open: the underlying bug remains source-reproducible on current main, and the contributor's direction is useful, but the branch is stale against the active config/sessions loader and has correctness gaps that should block merge.

## What This Changes

The PR changes gateway session aggregation so literal per-agent `session.store` paths discover sibling agent session stores and adds regression coverage.

## Best Possible Solution

Land an equivalent current-main fix in the config/sessions loader and target resolver so literal agent-layout stores discover only sibling stores under the same resolved agents root while preserving resolved paths and disk-derived agent identity.

## Reproduction Assessment

Yes. Source inspection shows a literal `.../agents/main/sessions/sessions.json` config enters the non-template early return in the active loader, and `sessions.list` consumes that single combined store, so sibling agent stores are skipped.

## Solution Assessment

No, not as written. The direction is right, but the patch needs to move to the current `src/config/sessions` implementation and cover resolved path propagation, disk-derived agent identity, root scoping, and changelog before merge.
