## Summary

Keep this PR open: it is the only linked implementation for the session-list undercount, and the attached text/JSON proof shows the intended API improvement, but the branch is now conflicting with current main and still needs a session-state repair before it can land.

## What This Changes

The PR changes gateway combined session-store discovery so a literal `session.store` path under `agents/<id>/sessions/sessions.json` can include sibling agent stores, with regression coverage for the linked `sessions.list` bug.

## Best Possible Solution

Port the fix to the current combined-store loader, keep true shared literal stores on the single-store path, scope literal agent-layout discovery to that agents root, and derive the target agent from the on-disk path before merging entries.

## Solution Assessment
