## Summary

Keep open: the reported `sessions.list` bug is valid and the PR direction is useful, but the branch now edits a stale gateway-local loader and has unresolved correctness issues that should block merge.

## What This Changes

The branch modifies gateway session utilities so literal per-agent session store paths discover sibling agent stores, and adds subagent session regression tests.

## Best Possible Solution

Port the proven literal agent-layout handling into `src/config/sessions/combined-store-gateway.ts` and `src/config/sessions/targets.ts`, preserving true shared literal stores as single-store and adding focused regression coverage.

## Reproduction Assessment

Yes. Configure `session.store` to a literal `.../agents/main/sessions/sessions.json` path with a sibling agent store and call `sessions.list`; current-main source still returns after reading only the literal default-agent store, and the linked proof shows `apiCount=1` before the branch fix.

## Solution Assessment

No, not as written. The fix direction is right, but it must be ported to the active `src/config/sessions` loader and resolver while preserving the resolved literal path and on-disk agent IDs.
