## Summary

## What This Changes

The branch changes gateway combined session-store loading so literal paths shaped like `agents/<id>/sessions/sessions.json` discover sibling agent stores while preserving single-store behavior for other literal paths.

## Best Possible Solution

Port the fix into `src/config/sessions/combined-store-gateway.ts`, scope literal agent-layout discovery to that agents root, derive ownership from the store path, and keep non-agent literal stores on the existing single-store path.

## Reproduction Assessment

Yes, by source and attached proof: configure `session.store` as a literal `.../agents/main/sessions/sessions.json` path with a sibling agent store, and current main returns before all-agent discovery. I did not run a live gateway, but the PR's raw proof shows the real API count changing from 1 to 2 after the branch fix.

## Solution Assessment

No, not as submitted for current main: the intended fix shape is narrow and appropriate, but it must be ported to the current combined-store owner and repaired so literal agent-layout targets derive the agent id from the store path rather than config-order dedupe.
