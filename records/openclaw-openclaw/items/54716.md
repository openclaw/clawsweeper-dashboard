## Summary

Keep open. The bug/fix direction is valid and the linked proof is sufficient, but the branch now patches a stale gateway-local loader while current main routes `sessions.list` through `src/config/sessions/combined-store-gateway.ts`; it needs a current-main port or replacement before merge.

## What This Changes

The PR changes gateway session-store aggregation and regression tests so literal `agents/<id>/sessions/sessions.json` store paths discover sibling agent stores under the same agents root.

## Best Possible Solution

Land a current-main fix in `src/config/sessions` that treats literal agent-layout stores as scoped multi-agent roots while preserving true shared literal stores as single-store paths.

## Reproduction Assessment

Yes, by source inspection and linked proof. Configure `session.store` to a literal `.../agents/main/sessions/sessions.json` path with a sibling agent store and call `sessions.list`; current main returns from the literal single-store branch before sibling discovery.

## Solution Assessment

No, not as currently patched. The intended behavior is right, but the fix needs to move into the active `src/config/sessions` loader/resolver and preserve resolved-path and on-disk-agent-id semantics.
