## Summary

Keep open: the bug is still valid and this PR has useful proof, but current main moved the active combined-store loader under `src/config/sessions`, so the branch's gateway-local edit will not fix `sessions.list` after rebase.

## What This Changes

The PR makes literal `session.store` paths under `agents/<id>/sessions/sessions.json` discover same-root sibling session stores and adds regressions for that path.

## Best Possible Solution

Port literal agent-layout handling into the active `src/config/sessions` combined loader and target resolver, preserving single-store behavior for non-agent-layout literal paths and adding current-main regressions.

## Reproduction Assessment

Yes. Source inspection on current main shows literal non-template `session.store` returns after reading one default-agent store, and the linked API-backed proof shows the same scenario improving from one session to two after the branch fix.

## Solution Assessment

No, not as currently patched. The direction is right, but the implementation must move to the active `src/config/sessions` loader/resolver and preserve resolved-path and on-disk-agent canonicalization details.
