## Summary

Keep open: the PR targets a real, source-backed bug and has useful before/after proof, but the branch is not mergeable as written because it changes the old gateway-local implementation instead of the active current-main session loader and target resolver.

## What This Changes

The PR updates gateway session aggregation and regression tests so literal `agents/<id>/sessions/sessions.json` store paths discover same-root sibling agent stores.

## Best Possible Solution

Land an equivalent current-main repair in `src/config/sessions/combined-store-gateway.ts` and `src/config/sessions/targets.ts` so literal agent-layout stores discover only same-root sibling stores while true shared literal stores remain single-store.

## Reproduction Assessment

Yes. Configure `session.store` to a literal `.../agents/main/sessions/sessions.json` path with a sibling agent store, then call `sessions.list`; current-main source inspection shows the active loader reads only the literal default-agent store and returns before sibling discovery.

## Solution Assessment

No, not as written. The direction is right, but the implementation must move to the active `src/config/sessions` loader and resolver, while preserving resolved paths and disk-derived agent IDs.
