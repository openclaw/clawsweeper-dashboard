## Summary

Keep open. The bug and proof are solid, but this branch is now conflicting and patches the old gateway-local loader while current main uses the extracted `src/config/sessions` loader.

## What This Changes

The PR changes gateway session-store aggregation and regression tests so literal `agents/<id>/sessions/sessions.json` paths discover same-root sibling agent stores.

## Best Possible Solution

Repair or replace this PR on current main by implementing literal per-agent store handling in `src/config/sessions`, preserving same-root discovery, resolved paths, and on-disk agent IDs with focused regressions.

## Reproduction Assessment

Yes. Configure `session.store` to a literal `.../agents/main/sessions/sessions.json` path with a sibling agent store and call `sessions.list`; current-main source returns from the literal-store branch before sibling discovery, and the linked proof shows one row before versus two after.

## Solution Assessment

No, not as currently patched. The intended behavior is the right narrow fix, but the implementation needs to move into the active `src/config/sessions` loader/resolver and preserve resolved-path plus on-disk-agent-id semantics.
