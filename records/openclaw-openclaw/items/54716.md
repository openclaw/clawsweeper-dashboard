## Summary

Keep open: the bug and fix direction are valid, but the branch currently patches the stale gateway-local loader while current main routes `sessions.list` through `src/config/sessions/combined-store-gateway.ts`; the remaining review findings are narrow and repairable.

## What This Changes

The branch changes session-store loading so literal `agents/<id>/sessions/sessions.json` paths discover sibling agent stores and adds gateway session regressions for that scenario.

## Best Possible Solution

Land an equivalent current-main fix in `src/config/sessions` that treats literal agent-layout stores as same-root multi-agent roots while preserving true shared literal stores as single-store paths.

## Reproduction Assessment

Yes. Configure `session.store` to a literal `.../agents/main/sessions/sessions.json` with a sibling agent store, then call `sessions.list`; current-main source returns from the single-store branch before sibling discovery, and the linked API-backed proof shows the same scenario improving from one row to two after the branch fix.

## Solution Assessment

No, not as currently patched. The direction is right, but the implementation needs to move into the active `src/config/sessions` loader and resolver while preserving resolved store paths and on-disk agent-id canonicalization.
