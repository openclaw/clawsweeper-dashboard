## Summary

Keep open: current main has the internal watchdog option and tests, but no public TUI config or CLI path passes it into the live TUI handlers, so the useful work remains.

## What This Changes

Request to expose the TUI streaming watchdog timeout through config or a CLI flag so operators can tune or disable the 30-second stale-stream notice.

## Best Possible Solution

Add one supported millisecond tuning surface, thread it through `TuiOptions`/`runTui` into `createEventHandlers`, preserve `0` as disable, and update schema, docs, and focused TUI/CLI tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection confirms current main cannot set the handler's `streamingWatchdogMs` from `openclaw tui` or config.

## Solution Assessment

Unclear until maintainers choose the public contract. A schema-backed config key is the likely operator-friendly end state, possibly with a CLI override, but either path should reuse the existing handler option rather than changing watchdog internals.
