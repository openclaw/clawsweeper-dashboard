## What This Changes

Adds `onModelDiagnosticEvent` to `openclaw/plugin-sdk/diagnostic-runtime` so plugins can opt into allowlisted trusted `model.*` diagnostic events, with docs and regression tests.

## Best Possible Solution

Merge only after a Plugin SDK owner explicitly accepts the narrow `diagnostic-runtime` observability contract and its relationship to the separate model-failover hook work.

## Reproduction Assessment

Yes. Current source shows `onDiagnosticEvent` drops trusted events while OpenClaw emits the requested model lifecycle events through trusted diagnostic emitters, and the PR body includes after-fix gateway logs from a real plugin receiving `model.usage`.

## Solution Assessment

Unclear until maintainers approve the public API, but the implementation is a narrow and maintainable alternative to broadening `onDiagnosticEvent` or encouraging plugins to consume all internal diagnostic events.
