## Summary

Current main still withholds trusted model lifecycle events from third-party diagnostic subscribers, and this focused patch has credible live proof with no concrete correctness defect. The PR should remain open because shipping the new callback creates a permanent Plugin SDK contract and maintainers still need to accept diagnostics callbacks—rather than ordinary plugin hooks—as the owner of this external telemetry surface.

## What This Changes

The PR adds `onModelDiagnosticEvent` to the focused diagnostic Plugin SDK subpath, exposing five allowlisted trusted model lifecycle events with tests and public documentation.

## Best Possible Solution

Adopt one explicitly approved external model-observability architecture: keep this narrow diagnostic callback if raw lifecycle and usage telemetry belongs in `plugin-sdk/diagnostic-runtime`, document its versioning boundary, and define the related high-level failure hooks as complementary rather than overlapping.

## Solution Assessment

Unclear pending product ownership approval. The implementation is a focused, maintainable diagnostic seam and does not duplicate current bundled OTel internals, but maintainers must decide whether raw model telemetry belongs in diagnostic callbacks or the ordinary plugin-hook contract.
