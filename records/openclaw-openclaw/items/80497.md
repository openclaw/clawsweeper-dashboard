## Summary

Keep this PR open. The PR addresses a real plugin observability gap and has sufficient real behavior proof, but it still needs maintainer/security API judgment because the new listener treats plugin-callable trusted diagnostic emissions as runtime-origin model telemetry.

## What This Changes

Adds a plugin-sdk diagnostic-runtime `onModelDiagnosticEvent` listener for five trusted model lifecycle events, with tests, compat export, and a changelog entry.

## Best Possible Solution

Expose model diagnostics through a public SDK contract backed by non-forgeable core provenance, or explicitly scope and document the stream as non-integrity-safe telemetry before merge.

## Reproduction Assessment

Yes. Current main source shows broad plugin listeners drop trusted events while model lifecycle events are emitted through trusted diagnostic paths; the PR body adds real after-fix gateway logs showing delivery.

## Solution Assessment

No, not yet. A narrow opt-in listener is the right shape, but the implementation should not rely only on `metadata.trusted` while plugins can publicly emit trusted diagnostics themselves.
