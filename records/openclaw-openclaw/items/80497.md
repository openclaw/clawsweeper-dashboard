## Summary

Keep open for maintainer review: current main does not expose trusted `model.*` diagnostics through the plugin SDK diagnostic listener, and the PR is a focused implementation with tests, docs, and real log proof. The remaining blocker is not a code defect I can point to; it is maintainer approval of a new public plugin SDK observability contract.

## What This Changes

The PR adds `onModelDiagnosticEvent` to `openclaw/plugin-sdk/diagnostic-runtime`, exposes five allowlisted trusted `model.*` diagnostic events, documents the non-integrity telemetry contract, and adds focused diagnostic event tests.

## Best Possible Solution

Land this narrow subpath API if maintainers want diagnostics-stream telemetry as a public plugin SDK contract; otherwise keep telemetry plugins on the existing `model_call_*` and `llm_output` hooks instead of adding a second contract.

## Reproduction Assessment

Yes. Source inspection shows the current plugin-facing `onDiagnosticEvent` drops trusted events while model lifecycle diagnostics are emitted through trusted paths, and the PR body adds live logs showing the after-fix stream arriving in a real gateway.

## Solution Assessment

Unclear pending maintainer API direction. The patch is a narrow, tested solution for diagnostic-stream subscribers, but OpenClaw already has model-call and LLM output hooks, so maintainers should decide whether this parallel public SDK contract should exist.
