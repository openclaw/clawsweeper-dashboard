## Summary

Keep open. The PR addresses a real plugin observability gap and includes real deployed-gateway proof, but the current patch still treats `metadata.trusted` as an integrity boundary even though plugins can import the public trusted emitter from the same SDK subpath.

## What This Changes

Adds `onModelDiagnosticEvent` as a diagnostic-runtime/compat SDK export for five trusted `model.*` lifecycle events, with regression tests and a changelog entry.

## Best Possible Solution

Expose model diagnostics through a public SDK contract backed by non-forgeable core provenance, or explicitly scope the stream as non-integrity-safe telemetry, then document the diagnostic-runtime API and trust limits.

## Reproduction Assessment

Yes. Current main source shows broad plugin listeners drop trusted events while the embedded runner emits the relevant model lifecycle through trusted diagnostic paths; the PR body adds real after-fix gateway logs showing delivery.

## Solution Assessment

No, not yet. A narrow opt-in listener is the right shape, but the implementation should not rely on `metadata.trusted` while plugins can publicly emit trusted diagnostics themselves.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Use non-forgeable provenance for model telemetry:** `src/infra/diagnostic-events.ts:934-936`
  - body: `metadata.trusted` is still the only provenance check here, but `openclaw/plugin-sdk/diagnostic-runtime` publicly exports `emitTrustedDiagnosticEvent`. A plugin can emit allowlisted `model.usage` or `model.call.completed` events through that public emitter and have them pass this new stream, corrupting downstream attribution that expects runtime-origin model telemetry.
  - confidence: 0.9
- **[P3] Document the new diagnostic runtime listener:** `src/plugin-sdk/diagnostic-runtime.ts:14`
  - body: This adds a public diagnostic-runtime SDK export, but the plugin SDK docs still describe the subpath without this listener or its trust limits. Please update the SDK subpath or migration docs so plugin authors can discover the opt-in API and understand the provenance caveat.
  - confidence: 0.78
