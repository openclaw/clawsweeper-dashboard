## Summary

Keep open. The revised PR fixes the prior trust-gating and root-export blockers and has source, CI, and real-behavior proof, but it adds a new public SDK telemetry surface that needs maintainer API-boundary approval rather than cleanup closure.

## What This Changes

The PR adds a plugin SDK diagnostic-runtime helper for trusted allowlisted model lifecycle diagnostic events, exports it through diagnostic-runtime and compat, adds regression tests, and updates the changelog.

## Best Possible Solution

Land a narrow trusted-only diagnostic-runtime subscription after maintainer approval of the public telemetry boundary, while keeping the broad diagnostic stream’s trusted-event filter intact.

## Reproduction Assessment

Yes. Source inspection of current main shows onDiagnosticEvent filters trusted events while the model call, usage, and failover paths emit through trusted diagnostic helpers, so the observability gap is source-reproducible.

## Solution Assessment

Yes, the revised implementation is the narrowest maintainable direction I found: a focused diagnostic-runtime subpath with trusted metadata gating, an allowlist, and regression coverage; final API-boundary approval remains a maintainer decision.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Require trusted metadata before delivering model events:** `src/infra/diagnostic-events.ts:924`
  - body: `onModelDiagnosticEvent` is documented as exposing trusted runtime `model.*` events, but this callback ignores `metadata.trusted` and filters only by event type. Since `emitDiagnosticEvent` is still public, an untrusted `model.usage` or `model.call.completed` event would be delivered on the trusted-model subscription, causing forged or duplicate attribution for plugins that subscribe to both streams. Gate on `metadata.trusted` and add a regression for untrusted `model.*` events.
  - confidence: 0.93
- **[P2] Update the root SDK contract for the new export:** `src/plugin-sdk/index.ts:141`
  - body: This adds `onModelDiagnosticEvent` to the root `openclaw/plugin-sdk` export, but the existing root surface contract still expects the small runtime export list without it. `pnpm test src/plugins/contracts/plugin-sdk-index.test.ts` will reject the extra export unless the contract/API baseline is updated, or the helper stays scoped to `plugin-sdk/diagnostic-runtime`.
  - confidence: 0.88
