## What This Changes

The PR adds `onModelDiagnosticEvent` to `openclaw/plugin-sdk/diagnostic-runtime`, allowlists five trusted model diagnostic event types, and documents/tests the observability-only contract.

## Best Possible Solution

Land the narrow diagnostic-runtime API if maintainers accept observability-only telemetry semantics; otherwise keep plugins on the existing `model_call_*` and `llm_output` hooks rather than adding a parallel listener contract.
