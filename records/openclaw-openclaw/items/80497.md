## What This Changes

The PR adds an opt-in `onModelDiagnosticEvent` plugin SDK subscription for five trusted model lifecycle diagnostics, re-exports it from `diagnostic-runtime` and `compat`, documents the trust caveat, and adds focused diagnostics tests plus a changelog entry.

## Best Possible Solution

Land a narrow, documented `plugin-sdk/diagnostic-runtime` subscription if maintainers accept this observability-only contract for model telemetry.

## Reproduction Assessment

Yes. Source inspection on current main shows public `onDiagnosticEvent` drops trusted diagnostics while model lifecycle and usage events are emitted through trusted diagnostic paths; the PR body adds after-fix deployed logs showing delivery through the new listener.

## Solution Assessment

Yes, with maintainer API judgment. The proposed allowlisted listener preserves the broad diagnostic filter and documents the remaining trust limitation instead of presenting the stream as provenance-attested.
