## Summary

Keep open: the current-main limitation is source-backed and the API direction is plausible, but the patch needs fixes for trusted-event filtering and SDK contract coverage before merge.

## What This Changes

The PR adds `onModelDiagnosticEvent` to the diagnostic event infrastructure and plugin SDK exports, with tests and a changelog entry for exposing five trusted model lifecycle events to plugins.

## Best Possible Solution

Ship a narrow trusted-only model diagnostic subscription with metadata gating, regression coverage for untrusted model events, and synchronized SDK contract/API baseline updates.

## Reproduction Assessment

Yes. Source inspection shows `onDiagnosticEvent` drops trusted events while the relevant model lifecycle events are emitted through trusted diagnostic paths, so the current invisibility claim is source-reproducible without running the gateway.

## Solution Assessment

No, not as written. The focused opt-in API is a maintainable direction, but it must check `metadata.trusted` and align the root SDK contract tests or avoid adding the root export.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Require trusted metadata before delivering model events:** `src/infra/diagnostic-events.ts:924`
  - body: `onModelDiagnosticEvent` is documented as exposing trusted runtime `model.*` events, but this callback ignores `metadata.trusted` and filters only by event type. Since `emitDiagnosticEvent` is still public, an untrusted `model.usage` or `model.call.completed` event would be delivered on the trusted-model subscription, causing forged or duplicate attribution for plugins that subscribe to both streams. Gate on `metadata.trusted` and add a regression for untrusted `model.*` events.
  - confidence: 0.93
- **[P2] Update the root SDK contract for the new export:** `src/plugin-sdk/index.ts:141`
  - body: This adds `onModelDiagnosticEvent` to the root `openclaw/plugin-sdk` export, but the existing root surface contract still expects the small runtime export list without it. `pnpm test src/plugins/contracts/plugin-sdk-index.test.ts` will reject the extra export unless the contract/API baseline is updated, or the helper stays scoped to `plugin-sdk/diagnostic-runtime`.
  - confidence: 0.88
