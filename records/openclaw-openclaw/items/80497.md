## Summary

Keep open: the PR is directionally useful, but the new trusted-model subscription still relies on `metadata.trusted` as an integrity boundary even though plugins can import the public trusted diagnostic emitter from the same SDK area.

## What This Changes

The PR adds `onModelDiagnosticEvent` for trusted model lifecycle diagnostics, exports it through `diagnostic-runtime` and `compat`, adds regression tests, and updates the changelog.

## Best Possible Solution

Expose the model diagnostic subscription only with a non-forgeable internal provenance contract, or explicitly document that the stream is plugin-observable telemetry without integrity guarantees, then update the plugin SDK docs.

## Reproduction Assessment

Yes. Source inspection shows current `onDiagnosticEvent` filters trusted metadata while the relevant model lifecycle events are emitted through trusted diagnostic paths.

## Solution Assessment

No, not yet. A narrow opt-in API is the right shape, but the current trust gate is not enough while the trusted emitter remains publicly importable by plugins.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Use non-forgeable provenance for model telemetry:** `src/infra/diagnostic-events.ts:934-936`
  - body: At this gate, `metadata.trusted` is the only provenance check, but `openclaw/plugin-sdk/diagnostic-runtime` already exports `emitTrustedDiagnosticEvent`. A plugin can import that public emitter, forge `model.usage` or `model.call.completed`, and have it pass this new trusted-model stream, which breaks the attribution/integrity guarantee this API is meant to provide.
  - confidence: 0.88
- **[P3] Document the new diagnostic runtime export:** `src/plugin-sdk/diagnostic-runtime.ts:14`
  - body: This adds a public plugin SDK export, but the plugin SDK docs still describe `diagnostic-runtime` without this listener or its trust limits. Please update the SDK subpath/migration docs so plugin authors can discover the opt-in API and understand the provenance caveat.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: The diff introduces a concrete telemetry-integrity concern because the new trusted-model listener trusts a flag that public plugin SDK code can currently emit.

Concerns:

- **[medium] Plugin-forgeable trusted model telemetry:** `src/infra/diagnostic-events.ts:934`

  - confidence: 0.88
