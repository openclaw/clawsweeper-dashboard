## Summary

Keep open: this is a plausible additive plugin SDK feature with no blocking review findings, sufficient after-fix real behavior proof, and green PR checks. It still needs maintainer/API review because it expands the plugin-visible diagnostic surface for trusted model telemetry.

## What This Changes

The PR adds an opt-in `onModelDiagnosticEvent` SDK subscription for allowlisted trusted model lifecycle diagnostics, with docs, compat export, changelog, and regression tests.

## Best Possible Solution

Land a narrow diagnostic-runtime subpath API, keeping the root SDK surface unchanged, if maintainers accept the documented observability-only trust model.

## Reproduction Assessment

Yes. Source inspection shows current main emits the relevant model lifecycle through trusted diagnostics while the existing public listener filters trusted events, and the PR body adds deployed gateway logs showing after-fix delivery.

## Solution Assessment

Yes, with maintainer API judgment. The allowlisted listener is narrower than exposing all internal diagnostics and preserves the broad `onDiagnosticEvent` filter; the official diagnostics-exporter-only `internalDiagnostics` path does not solve the general plugin observability use case.
