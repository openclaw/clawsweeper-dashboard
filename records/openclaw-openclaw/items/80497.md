## Summary

Keep open. The SDK API shape is useful and the contributor supplied real deployed-gateway proof, but the PR still treats `metadata.trusted` as non-forgeable even though the public diagnostic-runtime subpath exports `emitTrustedDiagnosticEvent`, so plugins can inject allowlisted model telemetry into the new stream.

## What This Changes

This PR adds `onModelDiagnosticEvent` as a diagnostic-runtime/compat SDK export for five trusted `model.*` lifecycle events, with regression tests and a changelog entry.

## Best Possible Solution

Expose model diagnostics through a public SDK contract backed by core-only provenance, or explicitly document that the stream is telemetry-only and not integrity-safe, then update the plugin SDK docs.

## Reproduction Assessment

Yes. Current main source shows `onDiagnosticEvent` drops trusted events while model lifecycle diagnostics are emitted through trusted paths, and the PR body adds real after-fix gateway logs showing delivery.

## Solution Assessment

No, not yet. A narrow opt-in listener is the right maintainable shape, but the implementation should not rely on `metadata.trusted` while plugins can publicly emit trusted diagnostics themselves.
