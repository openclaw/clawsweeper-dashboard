## Summary

Keep open. Current main still generates `models.json` from implicit bundled provider catalogs plus explicit provider config, so the requested config/onboard-only behavior is a model-catalog contract change rather than conservative cleanup.

## What This Changes

The issue asks for generated `models.json` to exclude bundled provider catalog entries unless a provider was onboarded or explicitly configured.

## Best Possible Solution

Define the intended generated `models.json` contract for bundled provider catalogs, then align planner behavior, provider discovery docs, and focused regression tests with that contract.

## Reproduction Assessment

Yes, at source level. Current main resolves implicit plugin provider catalogs before writing `models.json`, and Codex and Arcee still have catalog paths that can contribute provider rows outside a direct `models.providers` entry.

## Solution Assessment

Unclear. A blanket config/onboard-only change is not the narrowest maintainable fix while docs and tests describe the default as built-in catalog plus config overlay; maintainers should decide whether to preserve that contract or introduce a stricter mode/behavior.
