## Summary

Keep open. Current main still builds generated `models.json` from implicit provider catalogs plus explicit config, and the requested config/onboard-only behavior conflicts with the documented provider-catalog contract.

## What This Changes

The issue asks for generated `models.json` to include only providers introduced through onboard or explicit `openclaw.json` config, excluding implicit bundled catalog rows such as Arcee and Codex.

## Best Possible Solution

Define the intended `models.json` contract for bundled provider catalogs, then align planner behavior, provider discovery docs, and focused regression tests with that contract.

## Reproduction Assessment

Yes, at source level. Current main resolves implicit plugin provider catalogs before writing `models.json`, and the Codex and Arcee catalog paths still produce rows outside a direct `models.providers` entry.

## Solution Assessment

Unclear. A blanket config/onboard-only change is not obviously the best fix because current code and docs define provider catalogs as a source for `models.providers`; maintainers should decide the contract first.
