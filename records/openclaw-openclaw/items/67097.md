## Summary

Keep open. Current main and the latest shipped release still let implicit bundled provider catalogs contribute to generated model state, but the requested onboard/config-only default conflicts with the documented merge-mode contract; the narrower replace-mode bug is already tracked by an open canonical issue and fix PR.

## What This Changes

The issue requests that generated models.json only include providers introduced by onboarding or explicit openclaw.json configuration, excluding bundled Arcee and Codex catalog contributions.

## Best Possible Solution

Define the generated models.json contract for merge mode versus replace mode, land the canonical replace-mode fix if accepted, and update docs/tests for any intentional change to bundled provider catalog visibility.

## Reproduction Assessment

Yes, at source level. Current main and v2026.5.18 still resolve implicit plugin provider catalogs before models.json planning, and Codex/Arcee can contribute catalog rows outside a direct models.providers entry.

## Solution Assessment

Unclear. A blanket onboard/config-only filter is not yet the best solution because documented merge mode intentionally keeps built-ins, while the existing replace mode is the safer opt-out path and has its own open fix track.
