## Summary

Keep open. Current main still has the bundled-provider catalog paths that can surface Codex and Arcee rows, while changing generated `models.json` from catalog-plus-config to config-only needs a maintainer contract decision rather than cleanup closure.

## What This Changes

The issue requests that generated `models.json` include only providers introduced through onboarding or explicit `openclaw.json` configuration.

## Best Possible Solution

Define whether generated `models.json` is meant to be config-only or catalog-plus-config, land the replace-mode skip through the existing related PR if accepted, then align docs and provider catalog planning with that contract.

## Reproduction Assessment

Yes, at source level. Current main resolves implicit provider catalogs before writing `models.json`, Codex provides fallback catalog rows via synthetic auth, and Arcee can add rows when Arcee or OpenRouter auth is available.

## Solution Assessment

Unclear. A blanket config-only change is not obviously the best fix because current merge-mode docs say built-ins are kept; the safer path is a maintainer contract decision plus the narrower replace-mode fix already tracked by the related PR.
