## Summary

Keep open. Current main still plans generated `models.json` by resolving implicit plugin provider catalogs before mode handling, and the Codex and Arcee catalog paths named in the report are still present; the overlapping #66957 replace-mode bug does not cover the broader default/merge manifest contract.

## What This Changes

The issue asks for generated `models.json` to include only providers that were onboarded or explicitly configured, instead of bundled plugin catalog rows such as Codex and Arcee.

## Best Possible Solution

Define the generated `models.json` contract explicitly, then either document the provider-catalog behavior or gate generated rows on explicit provider activation/auth/config intent while preserving read-only discovery surfaces.
