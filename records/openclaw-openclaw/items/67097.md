## Summary

Keep open. Current main still writes generated `models.json` from implicit plugin provider catalogs plus explicit config, and the requested config/onboard-only behavior conflicts with the documented provider-catalog contract, so this needs a maintainer product/API decision rather than conservative cleanup.

## What This Changes

The issue requests making generated `models.json` reflect only providers introduced through onboarding or explicit user config, excluding bundled plugin catalog rows such as Codex and Arcee.

## Best Possible Solution

Define the intended `models.json` contract for bundled provider catalogs, then align planner behavior, docs, and focused regression tests with that decision.

## Reproduction Assessment

Yes, at source level. Current main still resolves implicit plugin provider catalogs before writing `models.json`, and the Codex and Arcee catalog paths can produce rows outside a direct `models.providers` entry.

## Solution Assessment

Unclear. A blanket config/onboard-only filter is not yet the best fix because current code, docs, and tests define provider catalogs as inputs to generated `models.providers`; maintainers should decide the contract first.
