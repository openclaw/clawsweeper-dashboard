## Summary

Keep open. Current main still builds generated `models.json` from implicit plugin/provider catalogs plus explicit config, and Codex/Arcee can still contribute provider rows outside a direct `models.providers` entry; changing that default to onboard/config-only needs an explicit model-catalog contract decision.

## What This Changes

The issue asks for generated `models.json` to include only providers intentionally introduced through onboarding or explicit `openclaw.json` configuration.

## Best Possible Solution

Define the intended `models.json` contract for bundled provider catalogs, then align planner behavior, provider discovery docs, and focused regression tests with that contract.

## Reproduction Assessment

Yes, at source level. Current main resolves implicit plugin provider catalogs before writing `models.json`, Codex has fallback/synthetic catalog rows, and Arcee can add rows when Arcee or OpenRouter auth is available.

## Solution Assessment

Unclear. A blanket config-only change is not clearly the best fix because current docs and code treat the generated catalog as built-in/catalog-plus-config by default; the maintainable path is to decide the contract first.
