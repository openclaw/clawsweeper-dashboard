## Summary

Keep open. Current main still has the reported behavior path: generated models.json planning resolves implicit plugin provider catalogs before mode-specific handling, while Codex and Arcee still expose the catalog hooks named in the report. The remaining question is a maintainer contract decision, not a stale cleanup close.

## What This Changes

The issue asks for generated models.json to reflect only providers that were explicitly onboarded or configured, instead of including bundled Codex and Arcee catalog entries through plugin-owned discovery paths.

## Best Possible Solution

Define the generated models.json contract explicitly. If default/merge generation is meant to include bundled provider-owned catalogs, document that and keep #66957 focused on replace-mode ordering; otherwise gate generated catalog rows on explicit activation, auth, or config intent while preserving read-only discovery views such as models list --all.

## Reproduction Assessment

Yes. The source-level reproduction path is high-confidence: generated models.json planning resolves implicit providers before mode handling, and implicit resolution still reaches the Codex and Arcee catalog hooks named in the report. I did not run a live generation because this review was read-only.

## Solution Assessment

Unclear. A replace-mode short-circuit is likely the right narrow fix for #66957, but this issue's broader requested behavior needs a maintainer decision on whether default/merge generated models.json should be configured-only or catalog-like.
