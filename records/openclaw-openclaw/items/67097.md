## Summary

Keep open. Current main still generates models.json from implicit plugin provider catalogs plus explicit config, so the Arcee/Codex manifestation remains source-reproducible; the requested onboard/config-only default conflicts with the documented merge/provider-catalog contract, while the narrower replace-mode bug is already tracked by https://github.com/openclaw/openclaw/issues/66957 and has an open fix PR.

## What This Changes

The issue requests that generated models.json include provider models only after explicit onboarding or models.providers configuration, rather than accepting bundled plugin catalog injections.

## Best Possible Solution

Define the generated models.json contract for default merge mode versus replace mode, land the canonical replace-mode fix if accepted, and then update docs/tests for any intentional change to bundled provider catalog visibility.

## Reproduction Assessment

Yes, at source level. Current main still resolves implicit plugin provider catalogs before writing models.json, and the Codex and Arcee catalog paths can contribute rows outside a direct models.providers entry.

## Solution Assessment

Unclear. A blanket onboard/config-only filter is not yet the best fix because current docs and plugin internals define provider catalogs as inputs to generated models.providers; maintainers should decide whether default merge behavior should change or whether replace mode is the intended opt-out.
