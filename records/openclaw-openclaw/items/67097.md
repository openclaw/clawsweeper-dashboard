## Summary

Keep open. Current main still builds generated `models.json` from implicit plugin/provider catalogs plus explicit config; Codex and Arcee can still contribute provider rows outside a direct `models.providers` entry, so making the manifest config/onboard-only is a model-catalog contract decision rather than conservative cleanup.

## What This Changes

The issue requests that generated `models.json` include only providers introduced through onboarding or explicitly configured in `openclaw.json`.

## Best Possible Solution

Define the intended `models.json` contract for bundled provider catalogs, then align planner behavior, provider discovery docs, and focused regression tests with that contract.

## Reproduction Assessment

Yes, at source level. Current main resolves implicit plugin provider catalogs before writing `models.json`, and the Codex and Arcee provider catalog paths still produce rows outside a direct `models.providers` entry.

## Solution Assessment

Unclear. A blanket config/onboard-only change is not obviously the best fix because current docs, tests, and code treat the generated catalog as built-in/catalog-plus-config; the maintainable path is to decide the contract first.
