## Summary

Keep open: current main still writes whole `exec-approvals.json` snapshots from caller-held approval objects in the allow-always paths, so the reported last-write-wins loss remains source-reproducible. The KeepMyClaw comment is a backup mitigation rather than a core fix, the focused fix PR closed unmerged, and the related security hardening PR is still open with blockers.

## What This Changes

The issue asks for concurrent `allow-always` exec approvals to preserve all allowlist entries instead of overwriting earlier writes in `exec-approvals.json`.

## Best Possible Solution

Land a maintainer-reviewed serialized exec approvals mutation path that reloads and merges under a lock or CAS guard, covering gateway and node-host allow-always plus metadata writes; https://github.com/openclaw/openclaw/pull/78226 may be a starting point but needs fixes/proof or replacement.

## Reproduction Assessment

Yes, at source level. Current main loads approval state into request-local objects and later writes the whole object through `addAllowlistEntry`/`persistAllowAlwaysPatterns`, so concurrent handlers can overwrite each other's additions.

## Solution Assessment

Yes in direction. A locked reload-and-merge or CAS-guarded mutation helper is the narrow maintainable fix; external backup tooling or caller-local writes do not close the core race.
