## What This Changes

The issue reports that concurrent `allow-always` exec approvals can overwrite each other in `exec-approvals.json` and silently drop earlier allowlist entries.

## Best Possible Solution

Keep this issue open and land a maintainer-reviewed shared exec-approvals store mutation that merges or rejects writes against the latest file state for both gateway and node-host allow-always persistence.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: concurrent handlers can hold old `ExecApprovalsFile` snapshots and `addAllowlistEntry` saves the whole stale object after appending. I did not execute the live race, so this is source-reproducible rather than directly reproduced.

## Solution Assessment

No current fix is present on main. The best fix is a shared store-level mutation or freshness-guarded merge used by both gateway and node-host callers, not a caller-local workaround.
