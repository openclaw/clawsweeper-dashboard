## Label Justifications

- `P1`: Concurrent approval loss affects a real command-approval workflow and can silently corrupt durable exec approval state.
- `impact:data-loss`: The issue is about silently lost persisted allowlist entries in `exec-approvals.json`.
- `impact:security`: The affected file stores durable exec approval policy and command trust decisions.

## Summary

Keep open: current main still saves exec approval allowlist mutations from caller-held snapshots, so concurrent allow-always approvals can still overwrite each other. The external backup suggestion is only a workaround, the targeted fix PR closed unmerged, and the broader open PR is not merged and still has security/proof blockers.

## What This Changes

The issue asks OpenClaw to preserve all concurrent allow-always exec approval allowlist entries instead of silently losing earlier writes to `exec-approvals.json`.

## Best Possible Solution

Route exec approval mutations through one locked, freshness-guarded reload-and-merge or CAS helper that covers gateway and node-host allow-always writes plus allowlist metadata writeback, with focused regression coverage and security review.

## Reproduction Assessment

Yes, at source level. Two approval handlers can hold separate `ExecApprovalsFile` snapshots and later save through `addAllowlistEntry`, `persistAllowAlwaysPatterns`, or `recordAllowlistUse`, so the later whole-file save can replace the earlier one.

## Solution Assessment

Yes in direction. A locked reload-and-merge or CAS-guarded mutation helper is the maintainable core fix; backup tooling or a plain re-read without serialization would not fully close the race.
