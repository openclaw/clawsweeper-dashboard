## Summary

Keep open: current main still saves whole exec-approvals snapshots from caller-held objects in the allow-always and allowlist metadata paths, so the reported last-write-wins loss remains source-reproducible. The backup-skill suggestion in the discussion is only a mitigation, the focused fix PR closed unmerged, and the related security hardening PR is still open with review blockers.

## What This Changes

Concurrent allow-always approvals should persist all new exec allowlist entries instead of overwriting entries from earlier approvals.

## Best Possible Solution

Add a serialized, freshness-guarded exec approvals mutation API that reloads and merges current JSON under a file lock or CAS guard, then route gateway and node-host allow-always plus allowlist-use metadata writes through it with regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows two approval handlers can retain separate ExecApprovalsFile snapshots and later call addAllowlistEntry or persistAllowAlwaysPatterns, so the later whole-file save can replace the earlier save.

## Solution Assessment

Yes in direction. A locked reload-and-merge or CAS-guarded mutation helper is the narrow maintainable fix; backup tooling or caller-local writes do not close the core race.
