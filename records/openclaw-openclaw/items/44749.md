## Label Justifications

- `impact:data-loss`: The issue is about silently lost persisted allowlist entries in `exec-approvals.json`.
- `impact:security`: The affected file stores durable exec approval policy and command trust decisions.

## Summary

Keep open: current main and the latest release still save whole `exec-approvals.json` objects from caller-held snapshots in the allow-always and allowlist-use paths, so the reported last-write-wins allowlist loss remains source-reproducible.

## What This Changes

The issue reports concurrent `allow-always` exec approvals silently overwriting earlier allowlist entries in `exec-approvals.json`.

## Best Possible Solution

Route all exec approval mutations through a serialized, freshness-guarded API that reloads and merges current JSON under a lock or CAS guard, with regression coverage for gateway and node-host allow-always plus metadata writeback.

## Reproduction Assessment

Yes, at source level. Two approval handlers can hold separate `ExecApprovalsFile` snapshots and later save through `addAllowlistEntry`, `persistAllowAlwaysPatterns`, or `recordAllowlistUse`, so the later whole-file save can replace the earlier one.

## Solution Assessment

Yes in direction. The maintainable repair is a locked reload-and-merge or CAS-guarded mutation helper; backup tooling, caller-local writes, or a plain re-read without serialization would not fully close the core race.
