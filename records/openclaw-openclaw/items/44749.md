## Summary

Keep open: current main and the latest release still save whole exec approval files from caller-held snapshots in the allow-always and allowlist metadata paths, while the related fix work remains unmerged and still needs maintainer/security review.

## What This Changes

The issue reports that concurrent allow-always exec approvals can overwrite `exec-approvals.json`, silently losing earlier allowlist entries.

## Best Possible Solution

Route exec approval mutations through a serialized, freshness-guarded reload-and-merge or CAS helper, with regression coverage for gateway and node-host allow-always plus allowlist metadata writeback.

## Reproduction Assessment

Yes, at source level. Two approval handlers can hold separate `ExecApprovalsFile` snapshots and later save through `addAllowlistEntry`, `persistAllowAlwaysPatterns`, or `recordAllowlistUse`, so the later whole-file save can replace the earlier one.

## Solution Assessment

Yes in direction. The maintainable repair is a locked reload-and-merge or CAS-guarded mutation helper; backup tooling or a plain re-read without serialization would not fully close the core race.
