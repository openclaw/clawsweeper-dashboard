## Summary

Keep open: current main still mutates and saves whole `exec-approvals.json` snapshots from caller-held objects in the allow-always and allowlist-use paths, so the reported last-write-wins loss remains source-reproducible. A prior focused fix PR closed unmerged, and the newer related stale-writeback PR is still open with review blockers rather than shipped.

## What This Changes

The issue requests race-safe persistence for concurrent `allow-always` exec approvals so multiple approvals do not silently drop entries from `exec-approvals.json`.

## Best Possible Solution

Route all exec approval mutations through one serialized, freshness-guarded API that reloads and merges current JSON under a lock or CAS guard, then cover gateway and node-host allow-always plus metadata writeback with focused regression tests.

## Reproduction Assessment

Yes. Source inspection on current main shows two approval handlers can hold separate `ExecApprovalsFile` snapshots and later save through `addAllowlistEntry`, `persistAllowAlwaysPatterns`, or `recordAllowlistUse`, so the later whole-file save can replace the earlier one.

## Solution Assessment

Yes in direction. The maintainable repair is a locked reload-and-merge or CAS-guarded mutation helper; backup tooling, caller-local writes, or a plain re-read without serialization would not fully close the core race.
