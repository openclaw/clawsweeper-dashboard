## Summary

Keep open: current main still saves caller-held exec-approvals snapshots from allow-always and allowlist metadata paths, so concurrent approvals can clobber earlier allowlist entries. The focused fix PR closed unmerged, and the related hardening PR is still open with review blockers.

## What This Changes

The issue asks to prevent concurrent allow-always exec approvals from overwriting each other in `exec-approvals.json`.

## Best Possible Solution

Add a serialized, freshness-guarded exec approvals mutation API that reloads and merges current JSON under a file lock or CAS guard, then route gateway and node-host allow-always plus allowlist-use metadata writes through it with regression coverage.

## Reproduction Assessment

Yes, at source level. Current main lets two approval handlers carry separate `ExecApprovalsFile` snapshots and later save those whole snapshots through `addAllowlistEntry` or `recordAllowlistUse`, so a later save can replace an earlier approval entry.

## Solution Assessment

Yes in direction. A locked reload-and-merge or CAS-guarded mutation helper is the narrow maintainable fix; backup tooling or caller-local writes do not address the core race.
