## Summary

Keep open: current `main` and `v2026.5.19` still save whole exec approval snapshots from the allow-always and allowlist metadata paths, so the reported last-write-wins race remains source-reproducible. The earlier fix PR at https://github.com/openclaw/openclaw/pull/52403 closed unmerged, and the broader open PR at https://github.com/openclaw/openclaw/pull/78226 still has review/security/proof blockers rather than a merged fix.

## What This Changes

The issue reports that concurrent `allow-always` exec approvals can overwrite each other in `exec-approvals.json`, losing earlier allowlist entries.

## Best Possible Solution

Route exec approval mutations through one serialized freshness-guarded reload-and-merge or CAS helper, covering gateway and node-host allow-always writes plus allowlist metadata writeback with regression coverage.

## Reproduction Assessment

Yes, at source level. Two approval handlers can hold separate `ExecApprovalsFile` snapshots and later save through `addAllowlistEntry`, `persistAllowAlwaysPatterns`, or `recordAllowlistUse`, so the later whole-file save can replace the earlier one.

## Solution Assessment

Yes in direction. A locked reload-and-merge or CAS-guarded mutation helper is the maintainable core fix; backup tooling or a plain re-read without serialization would not fully close the race.
