## Summary

Keep open: current `main` and latest release still persist allow-always and allowlist-use updates from caller-held approval snapshots, so the reported last-write-wins loss remains source-reproducible. The external backup suggestion is only a workaround, the earlier targeted fix PR closed unmerged, and the broader related PR is still open with review/proof blockers rather than a merged fix.

## What This Changes

The issue reports that concurrent `allow-always` exec approvals can silently overwrite earlier allowlist entries in `exec-approvals.json`.

## Best Possible Solution

Route exec approval mutations through one locked, freshness-guarded reload-and-merge or CAS helper covering gateway and node-host allow-always writes plus allowlist metadata writeback, with focused regression coverage and security review.

## Reproduction Assessment

Yes, at source level. Two approval handlers can hold separate `ExecApprovalsFile` snapshots and later save through `addAllowlistEntry`, `persistAllowAlwaysPatterns`, or `recordAllowlistUse`, so the later whole-file save can replace the earlier one.

## Solution Assessment

Yes in direction. A locked reload-and-merge or CAS-guarded mutation helper is the maintainable core fix; backup tooling or a plain re-read without serialization would not fully close the race.
