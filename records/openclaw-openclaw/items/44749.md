## Summary

Keep open: current `main` and the latest release still mutate a stale in-memory `ExecApprovalsFile` and save it back for allowlist usage/additions, so the reported last-write-wins race remains valid by source inspection. The earlier direct fix PR closed unmerged, and the closest open stale-write PR covers adjacent admin/metadata freshness with unresolved security review blockers rather than safely superseding this issue.

## What This Changes

The issue reports concurrent `allow-always` exec approvals overwriting each other in `exec-approvals.json`, silently dropping earlier allowlist entries.

## Best Possible Solution

Centralize exec approval mutations behind one locked, freshness-aware store helper that reloads and merges the latest file for gateway and node allow-always writes plus usage metadata.

## Reproduction Assessment

Yes by source path, not by a live run: concurrent handlers can each hold a stale `ExecApprovalsFile`, and the store helper writes that snapshot back without merging newer entries.

## Solution Assessment

Yes, the best fix is a locked reload-and-merge store path; making the final write atomic prevents torn files but does not preserve concurrent allowlist entries.
