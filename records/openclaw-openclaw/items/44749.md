## Summary

Keep open: current main still mutates caller-held exec approval snapshots and saves the whole `exec-approvals.json` file, so concurrent `allow-always` approvals can overwrite earlier allowlist additions. The keepmyclaw backup suggestion is useful mitigation context and https://github.com/openclaw/openclaw/pull/52403 was a relevant fix attempt, but that PR is closed unmerged and the core persistence race remains.

## What This Changes

The issue reports that concurrent `allow-always` exec approvals can overwrite each other in `exec-approvals.json`, losing earlier allowlist entries.

## Best Possible Solution

Add a serialized exec-approvals mutation helper that locks the approvals file, reloads the latest JSON inside the lock, applies additive allowlist/durable/usage changes, and routes gateway plus node-host persistence through it without new config or dependencies.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: load two `ExecApprovalsFile` snapshots, add different allowlist entries through each snapshot, and the later whole-file save drops the earlier entry. I did not execute a runtime repro because this review is read-only.

## Solution Assessment

Yes. A locked reload-before-mutate store helper is the narrowest maintainable fix; external backups can reduce data loss impact but cannot make OpenClaw approval persistence correct.
