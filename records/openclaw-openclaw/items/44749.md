## Summary

Keep open: current main and the latest release still write `exec-approvals.json` from caller-held approval snapshots, and the same-scope fix PR was closed unmerged. This remains a core exec approval persistence bug with data-loss and security-boundary impact.

## What This Changes

The issue reports that concurrent `allow-always` exec approvals can silently overwrite earlier allowlist entries in `exec-approvals.json`.

## Best Possible Solution

Implement one central serialized approvals-file mutation path that reloads current state before updating allow-always entries and usage metadata, then cover concurrent gateway and node-host approvals with focused regression tests.

## Reproduction Assessment

Yes at source level. Current main still sends caller-held approval snapshots into full-file saves for both gateway and node-host `allow-always` persistence, matching the reporter's concurrent last-write-wins steps.

## Solution Assessment

Yes on direction, but the durable fix should be centralized in the exec approvals store. A fresh read plus lock or equivalent serialized mutation is better than an external backup workaround or isolated caller-side patch.
