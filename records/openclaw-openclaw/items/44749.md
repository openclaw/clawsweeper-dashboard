## Summary

Keep open: current main still mutates and saves caller-held exec approval snapshots for `allow-always`, so concurrent approvals can overwrite each other. The keepmyclaw comment is useful backup mitigation context and https://github.com/openclaw/openclaw/pull/52403 was a relevant attempted fix, but that PR is closed unmerged and the internal persistence race remains.

## What This Changes

Fix concurrent `allow-always` exec approvals so all approved allowlist entries are merged into `exec-approvals.json` instead of losing earlier writes.

## Best Possible Solution

Add a serialized async exec-approvals mutation helper that locks the file, reloads current JSON inside the lock, applies additive allowlist/durable/usage changes, and routes gateway plus node-host persistence through it without new config or dependencies.

## Reproduction Assessment

Yes, at source level. Two handlers can each hold an old `ExecApprovalsFile`, call `addAllowlistEntry` through gateway or node-host `allow-always`, and the later `saveExecApprovals` writes its stale whole-file object over the earlier entry.

## Solution Assessment

Yes. A lock plus reload-before-mutate store helper is the narrowest maintainable fix; backup skills can reduce data-loss impact but cannot make approval persistence correct.
