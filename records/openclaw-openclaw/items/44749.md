## Summary

Keep open: current main still has source-reproducible stale-snapshot writes for exec approval allowlist state, and the focused fix attempt closed unmerged. The related broader hardening PR is open but still has review blockers, so this issue should remain available for maintainer-led resolution.

## What This Changes

The issue reports a race where concurrent `allow-always` exec approvals overwrite `exec-approvals.json` and silently drop earlier allowlist entries.

## Best Possible Solution

Add a shared serialized exec-approvals mutation API that reloads and merges current JSON under a file lock or CAS guard, then route allow-always, allowlist metadata, initialization, gateway, node-host, and admin writes through it with regression coverage.

## Reproduction Assessment

Yes, at source level. Two concurrent approval handlers can resolve separate `ExecApprovalsFile` snapshots and later save those stale objects through `addAllowlistEntry` or `recordAllowlistUse`, so the later whole-file save can drop the earlier mutation.

## Solution Assessment

Yes in direction. A locked reload-and-merge or CAS-guarded mutation helper is the narrow maintainable fix; a simple backup workflow or caller-local writeback does not close the core race.
