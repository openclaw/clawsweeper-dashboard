## Summary

Keep open: current `main` still mutates and saves durable exec approval state from caller-held `ExecApprovalsFile` snapshots, so the reported last-write-wins loss remains source-reproducible. The only focused fix PR was closed unmerged, and the existing labels correctly route this as maintainer/security review rather than a new automated fix job.

## What This Changes

The issue reports that concurrent `allow-always` exec approvals can overwrite each other in `exec-approvals.json`, losing earlier allowlist entries.

## Best Possible Solution

Add a serialized, freshness-guarded exec approvals mutation API that reloads and merges current JSON under a file lock or CAS guard, then route gateway and node-host `allow-always` plus allowlist-use metadata writes through it with regression coverage.

## Reproduction Assessment

Yes. At source level, two approval handlers can each retain an old `approvals.file`; the later `addAllowlistEntry`, `persistAllowAlwaysPatterns`, or `recordAllowlistMatchesUse` save can replace newer durable entries with the stale whole-file snapshot.

## Solution Assessment

Yes. The requested fix direction is the right one, but current `main` has not implemented it; the maintainable solution is a locked reload-and-merge or CAS-guarded mutation helper rather than backup tooling or caller-local writes.
