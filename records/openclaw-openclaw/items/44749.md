## Summary

Keep open: current main and v2026.5.12 still persist durable exec approval mutations from caller-held snapshots, and the focused fix PR is closed unmerged, so the data-loss/security bug remains.

## What This Changes

Fix concurrent allow-always approval persistence so multiple gateway/node approvals merge without overwriting exec-approvals.json entries.

## Best Possible Solution

Add a serialized, freshness-guarded exec approvals mutation API that reloads and merges current JSON under a file lock or CAS guard, then route gateway and node-host allow-always and allowlist-use writes through it with regression coverage.

## Reproduction Assessment

Yes. Source inspection shows two approval handlers can each retain an old `approvals.file`, and the later `addAllowlistEntry`, `persistAllowAlwaysPatterns`, or `recordAllowlistMatchesUse` save can replace newer durable entries with a stale whole-file snapshot.

## Solution Assessment

Yes. A locked reload-and-merge or CAS-guarded mutation helper is the narrow maintainable fix; backup tooling or caller-local writes would not address the internal race.
