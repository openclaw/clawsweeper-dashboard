## Summary

Keep open: current main and the latest release still preserve the reported stale-snapshot write path for exec approval allowlist mutations, and the only exact fix candidate was closed unmerged.

## What This Changes

Concurrent allow-always exec approvals can overwrite each other's persisted allowlist entries in exec-approvals.json because write helpers save stale per-request approval snapshots.

## Best Possible Solution

Move allowlist mutations to a single reload-and-merge persistence path with locking or an equivalent durable store, shared by gateway and node-host, with regression coverage for concurrent allow-always and allowlist-use updates.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes at source level: current main still writes the whole approvals file from request-local snapshots in both allow-always caller paths, so concurrent stale snapshots can clobber earlier entries.

## Solution Assessment

Is this the best way to solve the issue? Yes for the reported direction: the maintainable fix belongs in the shared exec approvals persistence path, not in per-channel workarounds or backup/merge tooling.
