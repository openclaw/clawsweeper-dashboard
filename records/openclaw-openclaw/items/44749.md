## Summary

Keep open: current `main` and the latest release still save exec approval allowlist mutations from caller-held snapshots, so concurrent allow-always approvals can still overwrite each other. A prior targeted fix PR closed unmerged, and the related open repair PR is still unmerged with proof and security-review blockers.

## What This Changes

The issue reports that concurrent `allow-always` exec approvals can overwrite each other in `exec-approvals.json`, silently dropping persisted allowlist entries.

## Best Possible Solution

Route exec approval mutations through one locked, freshness-guarded reload-and-merge or CAS helper that covers gateway and node-host allow-always writes plus allowlist metadata writeback, with focused regression coverage and security review.

## Reproduction Assessment

Yes, at source level. Two approval handlers can hold separate `ExecApprovalsFile` snapshots and later save through `addAllowlistEntry`, `persistAllowAlwaysPatterns`, or `recordAllowlistUse`, so the later whole-file save can replace the earlier one.

## Solution Assessment

Yes in direction. A locked reload-and-merge or CAS-guarded mutation helper is the maintainable core fix; backup tooling or a plain re-read without serialization would not fully close the race.
