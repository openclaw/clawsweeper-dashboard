## Summary

Keep open: current main and the latest release still mutate and save stale exec approval snapshots for allow-always persistence, and the direct fix PR is closed unmerged while the overlapping stale-write PR is still open and not merge-ready.

## What This Changes

The issue reports silently lost exec allowlist entries when concurrent allow-always approvals write stale snapshots to exec-approvals.json.

## Best Possible Solution

Keep this issue open until exec approval allowlist mutations use one shared, serialized fresh read-modify-write or CAS contract across gateway and node-host allow-always paths, with regression coverage for concurrent approvals.

## Reproduction Assessment

Yes at source level: both gateway and node-host allow-always paths pass a loaded approvals snapshot into helpers that write the whole file back. I did not run a live concurrent Telegram approval repro in this read-only review.

## Solution Assessment

Yes: the best fix belongs in the shared exec approval persistence boundary, not in only one caller, because gateway and node-host both use the same stale snapshot mutation helpers.
