## Summary

Keep open: current main and the latest release still persist exec approval allowlist changes from request-local snapshots, and the related fix attempts are either closed unmerged or still open with conflicts and unresolved security-review feedback.

## What This Changes

The issue reports silent loss of exec allowlist entries when concurrent allow-always approvals save stale exec-approvals.json snapshots.

## Best Possible Solution

Merge a current-main fix that performs exec approval read-modify-write under one shared freshness/locking contract for gateway and node-host writes, with regression proof for concurrent allow-always approvals and stale metadata writeback.

## Reproduction Assessment

Yes from source inspection: concurrent approval handlers can each hold a stale ExecApprovalsFile and later save that whole snapshot through addAllowlistEntry or recordAllowlistUse. I did not run a live Telegram race in this read-only review.

## Solution Assessment

Yes for the issue boundary: the fix should live in shared exec approval persistence so both gateway and node-host callers get the same atomic/freshness behavior. The open related PR uses the right general direction but is not currently a complete safe landing path.
