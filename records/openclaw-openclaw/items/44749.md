## Summary

Keep open: current main still persists exec approval allowlist and metadata mutations by saving caller-held `ExecApprovalsFile` snapshots, so concurrent approval handlers can overwrite each other. The keepmyclaw backup comment is useful mitigation context, but it does not fix the internal persistence race, and the related fix PR #52403 is closed unmerged.

## What This Changes

The issue asks OpenClaw to preserve all `allow-always` exec allowlist entries when multiple approval handlers persist concurrently.

## Best Possible Solution

Add a shared exec-approvals mutation helper that serializes writes, reloads the latest file inside the critical section, applies additive allowlist/durable-approval/metadata changes without dropping unrelated fields, and routes gateway plus node-host persistence through it with regression coverage.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to load two `ExecApprovalsFile` snapshots, add different allowlist entries through each snapshot, and observe the later `saveExecApprovals` write replace the earlier entry because both helpers save stale whole-file objects.

## Solution Assessment

Yes. A serialized reload-before-mutate store helper is the narrowest maintainable fix; backup or external merge tooling can reduce data-loss impact but cannot make OpenClaw’s internal approval persistence correct.

## Repair Work Prompt

Fix the stale-snapshot race from https://github.com/openclaw/openclaw/issues/44749. Current main saves caller-held `ExecApprovalsFile` objects in `addAllowlistEntry`, `addDurableCommandApproval`, `persistAllowAlwaysPatterns`, `recordAllowlistUse`, and `recordAllowlistMatchesUse`; gateway and node-host paths pass already-loaded snapshots into those helpers. Add a shared write-side mutation helper in `src/infra/exec-approvals.ts` that serializes approval-file mutations, reloads the latest `exec-approvals.json` inside the critical section, applies additive allowlist, durable approval, and usage-metadata updates without dropping unrelated defaults/socket/agent fields, and saves once per mutation batch where possible. Route `src/agents/bash-tools.exec-host-gateway.ts` and `src/node-host/invoke-system-run.ts` through the helper while keeping approved executions best-effort if persistence fails. Add regression tests that simulate two stale snapshots adding different allow-always entries for the same non-default agent and confirm both persist. Also cover allowlist-use metadata not dropping a concurrently added entry. Include a user-facing `CHANGELOG.md` Fixes entry, crediting Thanks @aukei. Do not change exec policy defaults, approval UX text, or unrelated macOS/Swift behavior unless the TypeScript fix requires a documented generic seam.
