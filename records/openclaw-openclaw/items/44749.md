## Summary

Keep open: current main still writes exec approval mutations from caller-held `ExecApprovalsFile` objects, so concurrent allow-always handlers can overwrite entries saved by earlier handlers. The focused fix PR was closed unmerged, and the broad open security PR only changes approvals read-error handling, not stale-snapshot merging.

## What This Changes

Fix concurrent allow-always exec approvals so all entries persist in `~/.openclaw/exec-approvals.json` instead of being lost to last-write-wins stale snapshot saves.

## Best Possible Solution

Add one serialized, freshness-guarded exec approvals mutation API that reloads and merges current JSON under a lock or CAS guard, then route gateway and node-host allow-always plus usage-metadata writes through it with focused regression coverage.

## Reproduction Assessment

Yes. At source level, two approval handlers can each hold an old `approvals.file`; the later `addAllowlistEntry` or `recordAllowlistUse` call then saves its stale whole-file snapshot over a prior durable entry.

## Solution Assessment

Yes, the requested direction is the right fix shape, but it is not implemented on current main. The narrow maintainable solution is a locked reload-and-merge or CAS-guarded helper rather than backup tooling or another caller-local write.
