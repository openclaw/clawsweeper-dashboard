## Summary

Keep open: current main and the latest release still save exec approval mutations from caller-held `ExecApprovalsFile` snapshots, so concurrent `allow-always` handlers can overwrite entries persisted by earlier handlers. The focused fix PR was closed unmerged, leaving the reported durable approval-state race unresolved.

## What This Changes

The issue asks OpenClaw to prevent concurrent `allow-always` exec approvals from silently overwriting each other's allowlist entries in `~/.openclaw/exec-approvals.json`.

## Best Possible Solution

Add a serialized, freshness-guarded exec approvals mutation API that reloads and merges current JSON under an existing file-lock or CAS guard, then route gateway and node-host `allow-always` plus usage-metadata writes through it with regression coverage.

## Reproduction Assessment

Yes. At source level, two approval handlers can each hold an old `approvals.file`; the later `addAllowlistEntry`, `persistAllowAlwaysPatterns`, or `recordAllowlistMatchesUse` save can replace a newer durable entry with the stale whole-file snapshot.

## Solution Assessment

Yes. The requested direction is the right fix shape, but current main has not implemented it; the maintainable solution is a locked reload-and-merge or CAS-guarded mutation helper rather than backup tooling or another caller-local write.
