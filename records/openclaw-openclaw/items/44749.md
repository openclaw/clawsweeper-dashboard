## Summary

Keep open: current main and the latest release still write allow-always allowlist updates from the passed in-memory approvals object, so concurrent stale snapshots can still clobber each other. The only focused fix PR found is closed unmerged, and no merged duplicate or superseding item owns the exact stale-write race.

## What This Changes

The issue reports concurrent allow-always exec approvals overwriting each other in exec-approvals.json, causing earlier allowlist entries to be lost.

## Best Possible Solution

Keep this issue open for an owner-reviewed fix that serializes or transactionally merges exec approval allowlist writes across gateway and node-host paths.

## Reproduction Assessment

Yes. Current source still passes a previously loaded approvals.file into persistAllowAlwaysPatterns/addAllowlistEntry, and addAllowlistEntry writes that whole object back, so two concurrent stale objects can overwrite each other's additions.

## Solution Assessment

No. The current implementation is not the best solution because it keeps a read-modify-write shape without a visible lock or fresh merge at the allowlist mutation boundary.
