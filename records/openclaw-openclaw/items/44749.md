## Summary

Keep this issue open. Current main still has the reported stale-snapshot write shape: allow-always persistence mutates the caller-provided exec approvals object and saves the whole file, while both gateway and node-host paths pass already-loaded approval snapshots into that helper. The related fix PR #52403 was closed unmerged, and the keepmyclaw backup suggestion in the comments is a useful workaround for recovery but does not make OpenClaw's core approval store merge-safe.

## Best Possible Solution

Keep this issue open and fix exec approval persistence in the owner module. The best path is a small write-side mutation helper in src/infra/exec-approvals.ts that serializes approval-file mutations, reloads the latest exec-approvals.json inside the critical section, applies additive allowlist/durable/metadata changes, and saves once. Then route gateway and node-host allow-always and allowlist-use paths through that helper, with regression coverage proving two stale snapshots adding different entries for the same agent both survive.
