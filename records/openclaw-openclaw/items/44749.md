Fix the exec approval persistence race so concurrent allow-always approvals do not silently lose persisted allowlist entries.

## Best Possible Solution

Land a current-main security-reviewed approvals-store fix that performs the read-modify-write under one shared freshness or locking contract for gateway and node-host writes, with regression proof for concurrent allow-always approvals.
