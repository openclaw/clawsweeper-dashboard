## Summary

Keep open: current main still saves caller-held exec-approvals snapshots in the allow-always and allowlist metadata paths, so concurrent approvals can still clobber durable allowlist entries. The backup-skill suggestion is only mitigation, https://github.com/openclaw/openclaw/pull/52403 closed unmerged, and the overlapping hardening work in https://github.com/openclaw/openclaw/pull/78226 is still open with review blockers.

## What This Changes

The issue asks OpenClaw to preserve all allow-always exec approval allowlist entries when multiple gateway or node approvals resolve concurrently.

## Best Possible Solution

Add a serialized, freshness-guarded exec approvals mutation API that reloads and merges current JSON under a file lock or CAS guard, then route gateway and node-host allow-always plus allowlist-use metadata writes through it with regression coverage and real race proof.

## Reproduction Assessment

Yes. Source inspection on current main shows approval handlers can hold separate ExecApprovalsFile snapshots and later call addAllowlistEntry or persistAllowAlwaysPatterns, so the later whole-file save can replace entries written by an earlier approval.

## Solution Assessment

Yes in direction. A locked reload-and-merge or CAS-guarded mutation helper is the narrow maintainable fix; backup tooling or caller-local whole-file writes do not close the core race.
