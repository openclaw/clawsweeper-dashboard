## Summary

Keep open: current main and the latest release still persist allow-always entries by mutating a request-local approvals snapshot and saving the whole file, so concurrent approvals can still clobber earlier entries; the direct fix PRs are closed unmerged and the broader open stale-write fix is still conflicting with unresolved security-review feedback.

## What This Changes

The issue requests a core fix for concurrent allow-always exec approvals losing allowlist entries in exec-approvals.json due to stale full-file writeback.

## Best Possible Solution

Land a reviewed core exec-approvals persistence fix that reloads and merges under one lock/freshness contract for gateway and node allow-always and usage writes, with regression coverage for concurrent approvals and admin-revocation races.

## Reproduction Assessment

Yes. The issue has concrete reporter steps, and source inspection on current main shows both gateway and node allow-always paths save request-local ExecApprovalsFile snapshots without a re-read, merge, or hash guard; I did not run a live concurrent approval flow in this read-only sweep.

## Solution Assessment

Yes, the best fix belongs in the core exec approvals persistence boundary, not in a backup workaround. The keepmyclaw backup suggestion can help recover a clobbered file, but it does not prevent the internal stale-write race that loses entries.
