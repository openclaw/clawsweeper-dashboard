## Summary

Keep open: current main still saves stale caller-held exec approval snapshots from gateway and node-host `allow-always` paths, so the last writer can drop earlier allowlist entries. The keepmyclaw comment is useful mitigation context and #52403 is relevant prior work, but that PR is closed unmerged and the internal persistence race remains.

## What This Changes

The issue reports concurrent `allow-always` exec approvals overwriting each other in `exec-approvals.json`, silently dropping earlier allowlist entries.

## Best Possible Solution

Introduce an async exec-approvals mutation helper that takes the existing file lock, reloads the latest JSON inside the lock, applies additive allowlist/durable/usage metadata changes, and routes gateway plus node-host persistence through it without new config or dependencies.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: load two `ExecApprovalsFile` snapshots, add different allowlist entries through each snapshot, and the later `saveExecApprovals` call writes its stale whole-file object over the earlier entry.

## Solution Assessment

Yes. A serialized reload-before-mutate store helper is the narrowest maintainable fix; external backup/merge tooling can mitigate data loss but cannot make OpenClaw approval persistence correct.
