## Summary

Current main still misses the reported top-level `detail` API error shape in the shared assistant error classifier and formatter, so the raw JSON chat leak remains a valid open bug. The issue is also paired with open PR #45216 using closing syntax, so maintainers should review or replace that implementation path rather than close this issue early.

## What This Changes

The issue requests that unconfigured OpenAI/Codex provider errors, especially JSON payloads like `{"detail":"..."}`, not leak verbatim into user chat surfaces.

## Best Possible Solution

Land or replace PR #45216’s focused shared parser/formatter fix with regression coverage for top-level string `detail` payloads. After that, verify whether the unconfigured Codex/OpenAI spawn path still fires and, if it does, add credential gating in the provider or ACP discovery boundary rather than Telegram-specific suppression.
