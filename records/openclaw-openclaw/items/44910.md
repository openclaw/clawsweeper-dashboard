## What This Changes

The issue reports Codex/FastAPI-style `detail` API errors being forwarded as raw JSON into Telegram/user chat for users without configured Codex/OpenAI credentials.

## Best Possible Solution

Land, rebase, or replace the linked shared parser fix with regression coverage, then address unconfigured-provider probe suppression only if it still reproduces after formatting is fixed.

## Reproduction Assessment

Yes for the source-level parser leak: current main does not classify or message-extract a top-level string `detail` payload, and short unclassified assistant errors can still return raw. The exact live Telegram spawn/probe trigger was not reproduced.

## Solution Assessment

Yes for the first repair: recognizing top-level string `detail` in the shared formatter is the narrow channel-agnostic fix. Broader suppression of unconfigured provider probes should remain separate unless it still reproduces after the formatter fix lands.
