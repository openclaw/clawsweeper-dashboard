## Summary

Keep open. Current main and the latest shipped release still route the affected ACPX path through dependency schemas that reject nullable usage_update fields before OpenClaw can project or ignore them.

## What This Changes

The issue reports ACP session/update validation failures when an ACP usage_update notification includes used: null.

## Best Possible Solution

Normalize or drop nullable usage_update fields before ACP SDK notification validation, preferably upstream in acpx or the ACP SDK contract, then update OpenClaw's pin and add focused regression coverage for used: null.

## Reproduction Assessment

Yes, at source level: current main resolves ACPX through @agentclientprotocol/sdk 0.21.1, whose session/update schema requires numeric used and validates before callback dispatch. I did not run a live Discord ACP session in this read-only review.

## Solution Assessment

Unclear at implementation-boundary level. The desired behavior is clear, but maintainers should choose whether the fix lands upstream in acpx / ACP SDK, as a dependency bump, or as an approved local compatibility normalization path.
