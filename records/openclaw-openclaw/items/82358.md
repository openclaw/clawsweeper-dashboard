## Summary

Current main already addresses the reported macOS LaunchAgent false-warning path: package updates probe for the expected Gateway version after service metadata refresh, skip the redundant restart when launchd already brought the Gateway back, and the macOS restart helper uses launchd-native `kickstart -k`. The fix is current-main-only in this checkout; no release tag contains the proof commit yet.

## What This Changes

The issue requests that `openclaw update` avoid falsely warning when a macOS LaunchAgent with `KeepAlive=true` has already restarted a healthy Gateway on the updated version.

## Best Possible Solution

Keep the current main behavior and ship it in the next release; if the warning appears on a build containing 472523360d88522006da27634ba3f85022bb7f29 or later, reopen with the restart log and `openclaw gateway status --deep` output.

## Reproduction Assessment
