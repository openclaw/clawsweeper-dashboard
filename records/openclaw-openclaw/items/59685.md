## Summary

Keep open. The underlying status false-negative is not fully implemented on current main, but this PR head is not safe to merge as-is because it conflicts with newer scoped runtime-loading and cached doctor-probe behavior.

## What This Changes

The PR adds a live-gateway memory status fallback for `openclaw status --deep`, splits doctor memory runtime versus embedding health, restores active plugin registry state, and adds regression tests.

## Best Possible Solution

Land a rebased, narrow fix that lets status trust live gateway memory runtime evidence when the local status path cannot materialize the configured memory plugin, while preserving scoped plugin loading, cached doctor probes, and changelog policy.

## Reproduction Assessment

Yes. The linked report gives concrete steps where `openclaw status --deep` reports an enabled memory plugin as unavailable while live gateway memory operations succeed, and current main still has no status-row fallback to live gateway memory status.

## Solution Assessment

No, not as-is. The live-gateway fallback is the right direction, but this PR must be reconciled with current main's scoped runtime loader and cached/non-deep doctor probe safeguards before it is the narrowest maintainable fix.
