## Summary

Keep open: the PR is directionally correct, but it still misses JSON status and legacy provider-only mixed-version cases, so it should not merge until those focused gaps are fixed.

## What This Changes

The PR adds live Gateway memory-status fallback plumbing, splits doctor memory runtime and embedding health, restores cached plugin memory state, and updates tests plus the changelog.

## Best Possible Solution

Land a narrowed fix that consistently trusts live Gateway memory runtime/provider evidence across human and JSON status while keeping embedding readiness separate from runtime health.

## Reproduction Assessment

Yes. Source inspection shows current main can render an enabled memory plugin as unavailable when the local memory snapshot is null, and the linked report plus PR discussion provide concrete live-Gateway success evidence.

## Solution Assessment

No, not as-is. The runtime/embedding split is the right direction, but the PR should share the fallback with JSON status and treat non-empty legacy provider evidence as live memory health when embedding was only skipped.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Feed the Gateway fallback into JSON status:** `src/commands/status.command.ts:200-207`
  - body: The JSON branch returns before this new fallback is resolved, so `status --deep --json` still emits the original scan (`memory: null`) even when the live Gateway reports a healthy memory runtime. Move this data path into the shared JSON runtime or resolve it before the JSON return.
  - confidence: 0.9
- **[P2] Trust provider-only legacy memory payloads:** `src/commands/status.command-sections.ts:168-169`
  - body: This call does not ask older Gateways to probe embeddings, so a legacy response can contain a non-empty provider with skipped `embedding.ok=false` and no `runtime` field. Since this expression only accepts `runtime.ok` or `embedding.ok`, mixed-version status can still show a working live memory plugin as unavailable.
  - confidence: 0.88
