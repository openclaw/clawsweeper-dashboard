Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Clear stale runtime markers when context is absent:** `src/agents/bash-tools.exec-runtime.ts:265-268`
  - body: This helper spreads the inherited environment and only writes `OPENCLAW_SESSION_KEY` when `opts.sessionKey` exists, so a parent or caller marker can survive into a child process that has no bound session. Clear or overwrite the reserved marker keys in every runtime env resolver before applying the current context so scripts cannot route on stale session metadata.
  - confidence: 0.88
- **[P2] Forward agentId through approved gateway runs:** `src/agents/bash-tools.exec-host-gateway.ts:247-263`
  - body: `ProcessGatewayAllowlistParams` carries `agentId`, but this approved gateway execution path does not pass it to `runExecProcess`. In legacy or global session contexts the resolver falls back to `main`, so approved commands from another agent can receive the wrong `OPENCLAW_AGENT_ID`.
  - confidence: 0.86
