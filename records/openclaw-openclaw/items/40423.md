## Summary

Keep open: current main still does not expose these agent/session markers on exec, ACP client, or TUI local-shell child processes, and the open exec-focused replacement does not cover all of this PR's requested runtime scope. The branch also still has source-reproducible correctness issues, missing real behavior proof, and unresolved session-key exposure semantics.

## What This Changes

The PR adds `OPENCLAW_AGENT_ID` and `OPENCLAW_SESSION_KEY` injection for exec, ACP client, and TUI local-shell child processes, plus docs and focused tests.

## Best Possible Solution

Ship one trusted runtime marker contract with explicit override/clearing semantics and redaction docs; land the exec path through https://github.com/openclaw/openclaw/pull/73349 if approved, then explicitly decide ACP client and TUI local-shell scope.

Yes for the PR review findings by source inspection: the PR head preserves inherited reserved markers and the approved gateway follow-up path still omits explicit `agentId` forwarding. I did not run tests because this review is read-only.

## Solution Assessment

No as-is. The direction is useful, but the branch should not merge until trusted marker clearing, explicit agent forwarding, session-key exposure policy, canonical scope, and contributor real behavior proof are resolved.
