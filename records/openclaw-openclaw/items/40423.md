## Summary

Keep open. Current main still only documents and injects the existing shell-context markers, while this PR is not ready to land because it can mislabel approved exec runs and it deliberately broadens session-key exposure without a settled runtime contract; the open #72980 overlaps only the exec/session-key subset and does not supersede the full PR.

## What This Changes

The PR adds `OPENCLAW_AGENT_ID` and `OPENCLAW_SESSION_KEY` runtime environment markers to exec, ACP client, and TUI local-shell child processes, with matching docs and tests.

## Best Possible Solution

Rework this into a current-main runtime-marker contract that preserves explicit agent-id precedence in every exec path, applies ACP changes in `client-helpers.ts`, keeps TUI local-shell context lazy and current, and documents a deliberate session-key exposure policy before shipping.

## Reproduction Assessment

Yes. A focused test can exercise a gateway approval follow-up run with an explicit non-main `agentId` and a legacy/global session key, then assert the spawned env keeps the explicit id instead of falling back to `main`; the source path is clear even without running tests in this read-only review.

## Solution Assessment

No. The direction is reasonable, but this patch is not the best landing shape until it fixes the approval-followup identity propagation, rebases onto the current ACP helper layout, and settles whether session-key env exposure should be default or opt-in.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Preserve explicit agent IDs in approved exec runs:** `src/agents/bash-tools.exec-runtime.ts:263`
  - body: `resolveExecRuntimeEnv` falls back to a session-key-derived id, which becomes `main` for missing or legacy keys, but the gateway approval follow-up path still calls `runExecProcess` without forwarding `agentId`. Approved commands for a non-main/global context can therefore receive `OPENCLAW_AGENT_ID=main`, breaking the deterministic identity contract this PR adds.
  - confidence: 0.87

## Security Review

Status: needs_attention

Summary: The diff has no dependency or CI supply-chain change, but it intentionally exposes session keys to child-process environments and needs explicit maintainer policy before merge.

Concerns:

- **[medium] Define session-key exposure policy:** `src/agents/bash-tools.exec-runtime.ts:267`
  - body: The patch injects `OPENCLAW_SESSION_KEY` into exec, ACP client, and TUI local-shell children. Environment variables are inherited by subprocesses and may appear in logs, traces, shell debugging, crash reports, or process inspection, so default exposure and redaction expectations should be decided before this becomes a public contract.
  - confidence: 0.74
