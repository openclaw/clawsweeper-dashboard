## Summary

Keep open. Current main still only provides the existing OPENCLAW_SHELL markers for exec, acp-client, and tui-local; the requested OPENCLAW_AGENT_ID and OPENCLAW_SESSION_KEY contract is not implemented. The PR remains useful, but it is merge-conflicted, stale against the current ACP helper layout, still has an explicit-agent propagation bug in approval follow-up paths, and needs maintainer security semantics for exposing session keys in child environments.

## What This Changes

The PR adds runtime env resolution, docs, and tests to expose OPENCLAW_AGENT_ID and OPENCLAW_SESSION_KEY to child processes spawned by exec, acp-client, and TUI local shell commands.

## Best Possible Solution

Rework this against current main as one coherent runtime-marker contract: share or align env resolution across exec, gateway approval follow-up, ACP client helper, and TUI local shell; define default exposure and override/redaction behavior for session keys; preserve explicit agent-id precedence; document the public contract; and coordinate with #72980 so only one implementation of the exec session-key behavior lands.

## Reproduction Assessment

Not applicable as a traditional bug report, but there is a high-confidence source-level verification path. Current-main searches and the relevant env construction sites show only OPENCLAW_SHELL is injected today; a focused runtime proof would run exec, ACP client, and TUI local-shell children that print the requested variables after the rework.

## Solution Assessment

No, this PR is not the best landing shape as-is. The feature direction is reasonable, but the patch needs a rebase onto current helpers, an explicit-agent propagation fix, session-key exposure policy, and overlap reconciliation with #72980 before it is maintainable.

## Review Findings

Overall confidence: 0.84

Full review comments:

- **[P2] Avoid defaulting unthreaded agent identity to main:** `src/agents/bash-tools.exec-runtime.ts:263`
  - body: `resolveExecRuntimeEnv` falls back to `resolveAgentIdFromSessionKey(sessionKey)`, which returns `main` for missing or legacy keys. ## Security Review

Status: needs_attention

Summary: The PR intentionally exposes session keys through child-process environments, which needs explicit maintainer policy before landing.

Concerns:

- **[medium] Define session-key exposure policy:** `src/agents/bash-tools.exec-runtime.ts:263`
  - body: The diff injects `OPENCLAW_SESSION_KEY` into exec, acp-client, and tui-local child environments. Session keys are trusted runtime routing/binding values, and env vars are inherited by subprocesses and commonly captured in logs, traces, shell debugging, and crash reports, so the default/opt-in behavior, caller override rules, and redaction expectations should be decided before merge.
  - confidence: 0.73
