## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Forward agent IDs through approved exec runs:** `src/agents/bash-tools.exec-runtime.ts:263`
  - body: `resolveAgentIdFromSessionKey` falls back to `main` when no explicit agent id is provided, but the approved gateway follow-up path still reaches `runExecProcess` without forwarding `params.agentId`. Approved commands from non-main/global agents can therefore expose the wrong `OPENCLAW_AGENT_ID`; thread the explicit id through that path and cover an explicit-id mismatch.
  - confidence: 0.88
- **[P2] Clear reserved markers before injecting runtime values:** `src/agents/bash-tools.exec-runtime.ts:265-268`
  - body: The resolver spreads `baseEnv` before conditional session injection, so stale or caller-provided `OPENCLAW_SESSION_KEY` can survive when no current key exists; the ACP and TUI helpers use the same pattern. Delete or overwrite reserved marker keys before returning child envs so the values are always the trusted current context.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: No dependency or workflow change was found, but the patch intentionally exposes session-routing metadata through inherited child-process environments and needs trusted-marker semantics.

Concerns:

- **[medium] Approve session marker exposure:** `src/agents/bash-tools.exec-runtime.ts:268`
  - body: `OPENCLAW_SESSION_KEY` would become visible to exec, ACP client, and TUI local-shell children. Environment variables are inherited by descendants and can appear in logs or process inspection, so maintainers should confirm these keys are routing context rather than standalone authorization secrets and document redaction expectations.
  - confidence: 0.84
- **[low] Prevent stale marker spoofing:** `src/agents/bash-tools.exec-runtime.ts:265`
  - body: Because reserved markers are copied from `baseEnv` before conditional injection, an old or caller-provided marker can remain visible when the current runtime has no bound session key or agent context.
  - confidence: 0.86
