## Summary

Keep open: current main still exposes only the existing `OPENCLAW_SHELL` markers, and this PR remains a useful feature proposal, but the patch is not merge-ready because the approved gateway exec path can still lose explicit agent identity and the session-key env contract overlaps with #72980.

## What This Changes

The PR adds `OPENCLAW_AGENT_ID` and `OPENCLAW_SESSION_KEY` runtime markers to exec, ACP client, and TUI local-shell child-process environments with matching docs and tests.

## Best Possible Solution

Land a current-main runtime-marker contract after maintainer policy approval, preserving explicit agent-id precedence across every exec path and reconciling the exec session-key scope with #72980.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: run an approved gateway exec follow-up with explicit non-main `agentId` and a legacy/global session key, then assert the spawned env is not allowed to fall back to `OPENCLAW_AGENT_ID=main`.

## Solution Assessment

No. The feature direction is reasonable, but the patch should first thread explicit agent identity through all exec paths, rebase onto the current ACP helper layout, and settle the session-key exposure policy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Preserve explicit agent IDs in approved exec runs:** `src/agents/bash-tools.exec-runtime.ts:263-266`
  - body: `resolveExecRuntimeEnv` falls back to a session-key-derived id, which becomes `main` for missing or legacy keys, but the gateway approval follow-up path still calls `runExecProcess` without forwarding `agentId`. Approved commands from a non-main/global context can therefore expose `OPENCLAW_AGENT_ID=main`, breaking the deterministic identity contract this PR adds.
  - confidence: 0.87

## Security Review

Status: needs_attention

Summary: The diff has no dependency or CI supply-chain change, but default session-key exposure through inherited child-process environments needs maintainer policy before merge.

Concerns:

- **[medium] Define session-key exposure policy:** `src/agents/bash-tools.exec-runtime.ts:267`
  - body: The patch injects `OPENCLAW_SESSION_KEY` into exec, ACP client, and TUI local-shell children. Environment variables are inherited by subprocesses and may appear in logs, traces, shell debugging, crash reports, or process inspection, so default exposure and redaction expectations should be decided before this becomes a public contract.
  - confidence: 0.74
