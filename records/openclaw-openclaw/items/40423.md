## Summary

Keep open: the runtime-marker feature is not implemented on current main and the PR still has unique ACP client/TUI scope beyond the open exec-only follow-up, but this branch is not merge-ready because proof is missing, GitHub reports conflicts, and the marker trust/session exposure semantics need maintainer approval.

## What This Changes

The PR adds `OPENCLAW_AGENT_ID` and `OPENCLAW_SESSION_KEY` injection to exec, ACP client, and TUI local-shell child process environments, with docs and focused tests.

## Best Possible Solution

Ship one maintainer-approved trusted runtime marker contract with explicit clearing/override semantics and redaction docs, land the canonical exec path if approved, then decide whether ACP client and TUI local-shell should share the same contract.

## Solution Assessment

Overall confidence: 0.88

Full review comments:

- **[P2] Clear reserved markers before injecting runtime values:** `src/agents/bash-tools.exec-runtime.ts:263-268`
  - body: The helper copies `baseEnv` first and only conditionally adds `OPENCLAW_SESSION_KEY`, so a stale or caller-provided reserved value survives whenever the current runtime lacks a bound session key; ACP and TUI helpers use the same pattern. Strip or overwrite `OPENCLAW_AGENT_ID` and `OPENCLAW_SESSION_KEY` consistently before returning child envs.
  - confidence: 0.87
- **[P2] Forward explicit agent IDs through approved exec runs:** `src/agents/bash-tools.exec-runtime.ts:260-266`
  - body: `resolveAgentIdFromSessionKey` can fall back to `main` when no explicit id is passed, but the approved gateway follow-up path still calls `runExecProcess` without `params.agentId`. Approved commands from non-main/global agents can expose the wrong `OPENCLAW_AGENT_ID`; pass the explicit id through that path and cover an explicit-id mismatch.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: The PR intentionally exposes session-routing metadata to child-process environments and currently allows stale marker spoofing, so the trust/redaction contract needs maintainer approval.

Concerns:

- **[medium] Approve session marker exposure:** `src/agents/bash-tools.exec-runtime.ts:268`
  - body: `OPENCLAW_SESSION_KEY` would be visible to exec, ACP client, and TUI local-shell children. Environment variables are inherited by descendants and can appear in logs or process inspection, so maintainers should confirm these keys are routing context rather than standalone authorization secrets and document redaction expectations.
  - confidence: 0.84
- **[low] Prevent stale marker spoofing:** `src/agents/bash-tools.exec-runtime.ts:263`
  - body: Because reserved markers are copied from `baseEnv` before conditional injection, an old or caller-provided marker can remain visible when the current runtime has no bound session key or agent context.
  - confidence: 0.86
