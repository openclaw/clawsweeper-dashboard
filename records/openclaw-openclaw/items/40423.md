## Summary

Keep open: current main does not implement the requested markers, and this PR still needs rework before merge because explicit agent identity can be lost on the approved gateway exec path and the session-key exposure policy is unresolved.

## What This Changes

The PR adds `OPENCLAW_AGENT_ID` and `OPENCLAW_SESSION_KEY` to exec, ACP client, and TUI local-shell child-process environments, with matching docs and tests.

## Best Possible Solution

Land a current-main runtime-marker contract after maintainer policy approval, preserving explicit agent-id precedence across every exec path and reconciling the exec session-key scope with #72980.

## Reproduction Assessment

Yes. A focused source-level reproduction is clear: run an approved gateway exec follow-up with an explicit non-main `agentId` and a legacy/global session key, then assert the spawned env does not fall back to `OPENCLAW_AGENT_ID=main`.

## Solution Assessment

No. The feature direction is reasonable, but this patch is not the best landing shape until it forwards agent identity through the approval follow-up path, rebases onto the current ACP helper layout, and settles the session-key exposure policy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Preserve explicit agent IDs in approved exec runs:** `src/agents/bash-tools.exec-runtime.ts:263`
  - body: `resolveExecRuntimeEnv` falls back to a session-key-derived id, which becomes `main` for missing or legacy keys, but the gateway approval follow-up path still calls `runExecProcess` without forwarding `agentId`. Approved commands for a non-main/global context can therefore receive `OPENCLAW_AGENT_ID=main`, breaking the deterministic identity contract this PR adds.
  - confidence: 0.87

## Security Review

Status: needs_attention

Summary: The diff has no dependency or CI supply-chain change, but it intentionally exposes session keys to child-process environments and needs maintainer policy before merge.

Concerns:

- **[medium] Define session-key exposure policy:** `src/agents/bash-tools.exec-runtime.ts:267`
  - body: The patch injects `OPENCLAW_SESSION_KEY` into exec, ACP client, and TUI local-shell children. Environment variables are inherited by subprocesses and may appear in logs, traces, shell debugging, crash reports, or process inspection, so default exposure and redaction expectations should be decided before this becomes a public contract.
  - confidence: 0.74
