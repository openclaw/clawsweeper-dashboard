## Summary

Keep open: current main does not implement the requested runtime markers, and this PR still has blocking source-level issues, missing real behavior proof, merge conflicts, and overlap with other open env-marker PRs that need maintainer consolidation.

## What This Changes

This PR adds `OPENCLAW_AGENT_ID` and `OPENCLAW_SESSION_KEY` injection for exec, ACP client, and TUI local-shell child processes, plus docs and tests.

## Best Possible Solution

Define one maintained runtime-marker contract, reconcile it with the overlapping exec-marker PRs, then apply it consistently to the selected exec, ACP client, and TUI surfaces with stale-marker scrubbing, explicit agent-id precedence, docs, tests, and real behavior proof.

## Reproduction Assessment

Yes for source-level PR findings. The PR head gateway approval follow-up path has `params.agentId` but does not pass it to `runExecProcess`, and the resolver can preserve inherited stale markers because it spreads `baseEnv` before conditional injection; I did not run tests because this review is read-only.

## Solution Assessment

No. The feature direction is reasonable, but this branch needs rebase, consolidation with overlapping env-marker PRs, explicit agent-id forwarding, stale marker scrubbing, and maintainer acceptance of the public env contract before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Forward agent IDs through approved exec follow-ups:** `src/agents/bash-tools.exec-runtime.ts:263`
  - body: The new resolver falls back to a session-key-derived id, which becomes `main` for legacy/global keys. The gateway approval follow-up path still calls `runExecProcess` without `agentId`, so approved commands from a non-main legacy/global context can expose `OPENCLAW_AGENT_ID=main`.
  - confidence: 0.88
- **[P2] Scrub inherited session markers before injecting runtime env:** `src/agents/bash-tools.exec-runtime.ts:265-268`
  - body: The resolver starts from inherited `baseEnv` and only overwrites `OPENCLAW_SESSION_KEY` when a current key exists. A stale parent or caller-provided marker can therefore survive in a context with no bound session, contradicting the runtime-marker contract and misrouting downstream scripts.
  - confidence: 0.82
