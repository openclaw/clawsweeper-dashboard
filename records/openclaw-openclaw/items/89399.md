## Summary

Keep open: current main still has scoped command/gateway SecretRef resolution plus explicit exec opt-ins, not a consistent exec-backed resolver across CLI, gateway, doctor, and headless runtime contexts. The remaining work is security- and product-sensitive rather than a conservative cleanup close.

## What This Changes

Standardize exec-backed SecretRef resolution so CLI, gateway, diagnostics, scheduled, and headless runtime contexts behave consistently without changing SecretRef semantics.

## Best Possible Solution

Define a secops-approved resolver/delegation contract that preserves explicit exec execution, documents gateway delegation versus local execution, and proves upgrade-safe behavior across CLI, gateway, doctor, services, and headless agents.

## Reproduction Assessment

No high-confidence live reproduction is available. Source inspection does show the relevant divergence: command paths use targeted gateway/local resolution, while doctor/config/audit paths intentionally skip exec providers unless --allow-exec is set.

## Solution Assessment

Unclear as a direct universal resolver because that would change exec execution and credential delegation policy. The best path is an explicit security/product design that decides when to delegate to the gateway, when to execute locally, and when to degrade with diagnostics.
