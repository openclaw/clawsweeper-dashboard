## Summary

Keep open: current main and v2026.6.9 still have explicit exec opt-ins, runtime snapshots, command-path fallback, and pluginIntegration seams, but they do not define the requested cross-context OS/session/headless exec SecretRef contract.

## What This Changes

The issue requests a standardized exec SecretRef resolution contract so CLI, gateway, doctor, and agent runtime contexts resolve provider commands consistently across interactive, service, and headless environments.

## Best Possible Solution

Define a maintainer-approved generic exec SecretRef context contract or plugin API extension that preserves explicit exec opt-in, documents headless limits, and proves at least one real service-context keychain flow before implementation.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection shows current main still lacks a shared cross-context exec resolver contract and resolves exec providers from the caller/provider environment model.

## Solution Assessment

Unclear: a shared resolver contract may be the right direction, but the best boundary depends on maintainer choices about core versus plugin ownership, explicit exec consent, and credential-session security.
