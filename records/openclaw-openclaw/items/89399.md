## Summary

Keep open: current main and the latest release still lack the requested cross-context exec SecretRef contract. The existing implementation has runtime snapshots, explicit exec opt-ins, and pluginIntegration materialization, but no shared CLI/gateway/runtime OS-session or headless normalization layer.

## What This Changes

Request a consistent exec-based SecretRef resolution layer across CLI, gateway, agent runtime, diagnostics, and headless service contexts.

## Best Possible Solution

Define a narrow, security-reviewed SecretRef execution contract or plugin API extension that keeps provider-specific OS keychain behavior in plugins or resolver packages while giving core consistent diagnostics and runtime context handling.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not fully: source inspection confirms context-dependent exec behavior and diagnostic skips, but the report still needs live OS/session proof for specific keychain providers and headless service contexts.

## Solution Assessment

Is this the best way to solve the issue? Unclear: a shared core contract may be right for execution semantics, but provider-specific keychain/session behavior should remain plugin-owned unless maintainers confirm a missing core seam.
