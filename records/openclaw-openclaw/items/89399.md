## Summary

Keep open: current main has SecretRef snapshots, command-path gateway resolution, explicit exec opt-ins, and pluginIntegration provider seams, but it still runs exec resolvers with caller/provider env only and does not define the requested cross-context OS keychain or headless-service contract.

## What This Changes

Request standardizing exec-based SecretRef resolution so CLI, gateway, agent runtime, diagnostics, and headless service contexts resolve providers consistently.

## Best Possible Solution

Define a maintainer-approved generic SecretRef exec context contract or plugin API extension that preserves explicit exec opt-in, documents headless limitations, and proves at least one real service-context keychain flow before implementation.

## Reproduction Assessment

No: the source shows why CLI, gateway, and service contexts can differ, but this review did not establish a high-confidence live reproduction for a specific OS keychain, scheduler, service context, and failing command.

## Solution Assessment

Unclear: a shared resolver contract may be the right direction, but the best boundary depends on maintainer choices about core versus plugin ownership, explicit exec consent, and credential-session security.
