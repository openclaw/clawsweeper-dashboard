## Summary

Keep open: current main has fixed the shared outbound and Telegram reply slices, but WebChat, Gateway chat finalization, embedded TUI, and plugin-handled message actions still do not show uniform outbound lifecycle hook coverage, so the central inconsistency remains valid.

## What This Changes

The issue requests consistent internal `message:sent` and plugin `message_sent` hook emission across outbound dispatch paths, including Telegram replies, plugin-handled sends, and Gateway/RPC-style final delivery.

## Best Possible Solution

Define a single outbound lifecycle observer contract, or split the remaining Gateway/TUI/plugin-action gaps into maintainer-approved narrow fixes, with tests proving pre-send and post-send hooks fire exactly once with stable session context.

## Reproduction Assessment

Yes at source level. Current main documents outbound lifecycle hooks and shows shared outbound/Telegram paths wired, while WebChat, Gateway chat finalization, embedded TUI, and plugin-handled send paths still have direct delivery or final broadcast paths without those hook calls.

## Solution Assessment

Unclear as a single minimal patch. The Telegram slice is already fixed, but the remaining work is best solved by a shared observer seam or maintainer-approved narrower fixes that settle streaming, plugin-action, and exactly-once semantics.
