## Summary

Keep open. Current main has partial fixes for shared outbound delivery and Telegram replies, but WebChat/Gateway final broadcasts, Gateway chat.send dispatch, embedded TUI final events, plugin-handled send actions, and plugin message_sent session context still show source-level hook coverage gaps with narrower related issues still open.

## What This Changes

The issue reports inconsistent firing of internal message:sent and plugin message_sent/message_sending hooks across outbound message dispatch paths, causing downstream hooks such as conversation-stream to miss outbound replies.

## Best Possible Solution

Define one documented outbound lifecycle observer seam, or explicitly split the remaining surfaces into canonical fixes, so every outbound delivery path fires the applicable existing hooks exactly once with stable session/run context.

## Reproduction Assessment

Yes, as a high-confidence source-level path rather than a live run: register a sent-message hook and produce a WebChat/Gateway final reply, embedded TUI final reply, or plugin-handled message action. Current main directly broadcasts, emits, or returns from those paths without the relevant hook emission.

## Solution Assessment

No for closing or treating the partial Telegram/shared outbound fixes as complete. The maintainable solution is either a shared exactly-once lifecycle observer or maintainer-approved split fixes for WebChat/TUI, plugin actions, and canonical context propagation.
