## Summary

Keep open: source review found a narrow compatibility blocker in the follow-up because explicit `typingIntervalSeconds` config can still be ignored when Discord `message_tool_only` disables core keepalive. The stale-typing fix direction is plausible and the real Discord proof is sufficient, but this branch needs that settings repair before merge.

## What This Changes

The PR adds a per-run `typingKeepalive` option, disables the Discord callback keepalive, and passes `typingKeepalive: false` for unconfigured Discord `message_tool_only` source replies with focused regression tests.

## Best Possible Solution

Keep the nested Discord callback keepalive disabled, but only pass `typingKeepalive: false` when Discord `message_tool_only` has no explicit typing mode or refresh cadence; then land after checks and coordinate with the broader Discord lifecycle PR.

## Reproduction Assessment

Yes. The source path on current main clearly wires Discord callback keepalive and the core typing loop for the same reply run; I did not rerun live Discord in this read-only review, but the PR body includes live after-fix log proof.

## Solution Assessment

No, not yet. Disabling the nested Discord callback keepalive is well-scoped, but the PR should also preserve explicit `typingIntervalSeconds` before disabling the core keepalive for unconfigured message-tool-only replies.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Respect configured typing cadence before disabling keepalive:** `extensions/discord/src/monitor/message-handler.process.ts:223-225`
  - body: This predicate only treats `typingMode` as explicit typing configuration, so a Discord `message_tool_only` setup with `agents.defaults.typingIntervalSeconds` or `session.typingIntervalSeconds` but no `typingMode` still gets `typingKeepalive: false`. That silently ignores the documented refresh-cadence setting that `getReplyFromConfig` reads for the core typing controller, so existing configs lose periodic typing after upgrade. Include configured cadence in the explicit-config check or keep the core loop when either typing mode or interval is set.
  - confidence: 0.9
