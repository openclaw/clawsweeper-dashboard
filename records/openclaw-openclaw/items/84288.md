The PR adds a keepalive toggle to the shared typing controller, makes unconfigured `message_tool_only` replies one-shot, disables the Discord callback keepalive interval, and adds focused typing lifecycle tests.

## Best Possible Solution

Land a fix that removes duplicate Discord post-delivery typing refresh while preserving explicit `typingMode` behavior and either intentionally accepting the global default change or scoping the one-shot behavior to Discord.

## Solution Assessment

Unclear: disabling the nested Discord keepalive is well-scoped, but making the shared `message_tool_only` default one-shot changes cross-channel behavior. The merge path needs a maintainer decision on whether that default change is intended.
