## What This Changes

## Best Possible Solution

Review and land a single canonical fix, preferably the reply-gated lifecycle-owner shape in https://github.com/openclaw/openclaw/pull/94928, then close this issue after merge.

## Reproduction Assessment

Yes, source-level. Persist a CLI transcript, make post-turn runCliTurnCompactionLifecycle throw after result payloads exist, and current main reaches the post-turn error catch before pending final delivery or deliverAgentCommandResult.

## Solution Assessment

No, current main is not the best behavior because post-success compaction remains fatal to a reply that is already available. The narrow maintainable solution is to keep pre/no-reply failures fail-closed while treating post-success compaction as recovery metadata plus warning before the next turn.
