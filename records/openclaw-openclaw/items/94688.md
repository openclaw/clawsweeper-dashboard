## Summary

Keep open. Current main and the latest release still run post-turn CLI compaction before pending final delivery, and open closing PRs now own the exact repair path.

## What This Changes

The issue asks OpenClaw to avoid surfacing a generated Codex/OpenAI fixed-session reply as unavailable when a later post-turn compaction step fails.

## Best Possible Solution

Land one narrow agent-command lifecycle fix that treats post-success compaction as non-fatal only when deliverable assistant text exists, while keeping no-reply compaction failures fatal.

## Reproduction Assessment

Yes. Source-level reproduction is clear: make CLI transcript persistence succeed with reply payloads, then make runCliTurnCompactionLifecycle throw and observe current main reaches the post-turn error catch before pending final delivery or deliverAgentCommandResult.

## Solution Assessment

Yes. The best fix belongs in the agent-command turn lifecycle: catch post-turn compaction failure only when a deliverable reply exists, leaving the compaction helper's fail-closed contract intact.
