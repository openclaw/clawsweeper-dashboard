## Summary

Keep open. Current main and latest release v2026.6.10 still await post-turn CLI compaction before pending final delivery and channel delivery, and open candidate PR https://github.com/openclaw/openclaw/pull/94928 is the cleanest linked fix path.

## What This Changes

The issue reports that a Codex/OpenAI fixed session can generate and persist a reply, then fail post-turn compaction and surface the turn as unavailable instead of delivering the reply.

## Best Possible Solution

Land one narrow agent-command lifecycle fix that treats post-success compaction as non-fatal only when deliverable assistant content exists, while keeping pre-reply and no-reply compaction failures fatal.

## Reproduction Assessment

Yes. Source-level reproduction is clear: make CLI transcript persistence succeed with reply payloads, then make runCliTurnCompactionLifecycle throw and current main reaches the post-turn error catch before pending final delivery or deliverAgentCommandResult.

## Solution Assessment

Yes for the fix direction, but it is not on main yet. The best path is a reply-gated lifecycle fix in agent-command, like https://github.com/openclaw/openclaw/pull/94928, rather than making the compaction helper unconditionally swallow failures.
