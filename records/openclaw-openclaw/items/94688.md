## Summary

Keep open. Current main and the latest release still run post-turn CLI compaction before durable final delivery, and terminal compaction failures still throw, so the reported reply-generated-then-unavailable path remains; an open linked PR now owns the best canonical fix path.

## What This Changes

The issue reports that a Codex/OpenAI fixed session can generate and persist a reply, then surface the turn as unavailable when post-turn CLI compaction fails before delivery.

## Best Possible Solution

Land one canonical lifecycle-owner fix that preserves generated reply delivery while keeping empty/no-reply compaction failures fatal, then close or supersede sibling PRs after merge.

## Reproduction Assessment

Yes from source, but not from a live run in this review. Current main still awaits post-turn compaction before delivery, and the compaction helper still throws the exact reported error shape.

## Solution Assessment

Yes. The narrowest maintainable fix is to handle post-turn compaction failure in `agentCommandInternal` only when deliverable content exists, leaving the compaction helper and no-reply failure semantics fail-closed.
