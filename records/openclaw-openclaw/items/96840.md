Keep open: current main and the latest release still have the documented WebChat targetless `message.send` mismatch, but two open fix PRs now explicitly close this issue, so the issue should stay open until maintainers choose and land one candidate.

## What This Changes

Targetless WebChat `message.send` should route through the current-run internal source-reply sink instead of failing target validation.

## Best Possible Solution

Land one narrow fix PR that defaults the message tool for the current WebChat source to `message_tool_only` only when the internal sink is intended, while preserving ordinary automatic final replies and explicit overrides.

## Reproduction Assessment

Yes from source inspection: current main leaves implicit WebChat room events without `message_tool_only`, the internal sink requires that mode, and the next validation step throws the reported target error. I did not run a live WebChat session in this read-only review.

## Solution Assessment

No for the broad suggested shape that would make every WebChat room event message-tool-only, because current tests preserve automatic final delivery. The best fix is the narrower message-tool construction or mode-resolution repair already represented by the open fix candidates.
