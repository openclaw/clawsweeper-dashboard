## Summary

Keep open: current main still has the documented WebChat targetless `message.send` gap, and two open PRs now reference this issue as their closing fix path, so the issue should remain open until one lands.

## What This Changes

Targetless WebChat `message(action="send")` should route to the active WebChat source-reply sink instead of failing target validation.

## Best Possible Solution

Land one canonical focused PR that defaults the active internal WebChat message tool to `message_tool_only` while preserving ordinary WebChat final-answer delivery, then let this issue close from the merged fix.

## Reproduction Assessment

Yes, from source inspection: the internal source sink requires `message_tool_only`, current WebChat message-tool construction can leave that undefined, and normalization then throws the reported target error. I did not run a live WebChat session in this read-only review.

## Solution Assessment

Yes for the current repair direction: the narrow message-tool construction fix proposed by the linked PRs is the maintainable path. The broader issue-body sketch that changes all WebChat room events would be too broad because current tests intentionally keep ordinary WebChat final delivery automatic.
