Current main and v2026.6.10 keep WebChat out of the `message_tool_only` path in `src/gateway/tool-resolution.ts`, while `src/infra/outbound/internal-source-reply.ts` only handles targetless sends when that mode is active and `docs/web/webchat.md` documents active-run targetless WebChat sends as internal source replies. Implement the narrow owner-boundary fix without making WebChat a reusable outbound channel or breaking automatic final delivery for normal WebChat turns. Add or adjust regression tests around gateway tool resolution and message-action send validation; do not add config options, migrations, or changelog edits.
