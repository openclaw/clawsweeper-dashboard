## Summary

Keep open: current main and v2026.5.12 still drop the Feishu card-action sibling callback fields, while this PR is a plausible narrowed fix that still needs external real Feishu proof and maintainer reconciliation with broader callback-field work.

## What This Changes

The branch preserves Feishu card-action `option`, `options`, and `form_value` fields through typing, parser normalization, fallback serialization, focused tests, and changelog text.

## Best Possible Solution

Land or supersede one canonical Feishu plugin patch that preserves callback sibling fields from raw parser through fallback/agent dispatch, with regression coverage, a clear decision on `input_value`/`name`, and redacted live Feishu proof.

## Reproduction Assessment

Yes, source-level. A raw `card.action.trigger` payload with `action.option`, `action.options`, or `action.form_value` enters `parseFeishuCardActionEventPayload`, which returns an event containing only `value` and `tag`, and fallback text then serializes only `action.value`.

## Solution Assessment

Yes for the narrowed field set. Preserving these fields at the parser boundary and merging them into legacy fallback JSON after `text`/`command` precedence fits the current design, but the broader `input_value` and `name` scope needs an explicit maintainer decision before landing.
