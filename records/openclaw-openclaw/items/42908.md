## Summary

Keep open: the PR addresses a real Feishu callback field loss on current main, and I found no blocking code-review finding, but it still needs real Feishu after-fix behavior proof before merge.

## What This Changes

This PR adds Feishu card-action passthrough for `action.option`, `action.options`, and `action.form_value` through parsing and legacy fallback serialization, with regression tests and a changelog entry.

## Best Possible Solution

Land a rebased version of this Feishu plugin fix after real Feishu callback proof, preserving the three sibling fields from raw parser through fallback dispatch while leaving broader callback fields to https://github.com/openclaw/openclaw/issues/43202 or a follow-up.

## Reproduction Assessment

Yes, source-level. A raw Feishu `card.action.trigger` payload with `action.value` plus sibling `option`, `options`, or `form_value` is normalized into an event containing only `value` and `tag`, then fallback text serializes only `action.value`.

## Solution Assessment

Yes for this PR's scoped fields. Preserving the fields at the Feishu parser boundary and fallback serializer is the narrow maintainable fix; broader `input_value`/`name` handling should stay in the separate broader item.
