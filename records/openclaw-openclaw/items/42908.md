## Summary

Keep open: current main still drops Feishu card-action `option`, `options`, and `form_value` before dispatch, and this PR remains the canonical narrow implementation path for that data-loss bug. The diff looks directionally correct, but it still needs contributor-supplied redacted live Feishu proof before merge.

## What This Changes

The PR preserves Feishu card-action sibling fields from raw callback parsing through fallback dispatch and adds focused Feishu regression tests plus a changelog entry.

## Best Possible Solution

Land this PR or an equivalent narrow Feishu plugin patch after redacted live proof and branch refresh, then handle broader callback fields through https://github.com/openclaw/openclaw/issues/43202 or https://github.com/openclaw/openclaw/pull/43953.

## Reproduction Assessment

Yes, source-level: a raw `card.action.trigger` payload with sibling `action.option`, `action.options`, or `action.form_value` is normalized into only `value` and `tag`, then fallback text serializes only `action.value`. No live Feishu replay was run in this read-only review.

## Solution Assessment

Yes for the narrow scope: preserving the fields at the parser boundary and carrying them through fallback JSON after legacy `text` and `command` handling is the maintainable fix. Broader `input_value` and `name` support should remain with the related follow-up unless maintainers choose to fold it in.
