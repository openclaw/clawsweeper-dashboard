## Summary

Keep open: current main and v2026.5.7 still discard these Feishu callback fields, and this same-author PR remains the active focused implementation path, but it still lacks contributor real Feishu behavior proof before merge.

## What This Changes

The branch extends Feishu card-action parsing and fallback serialization to carry `option`, `options`, and `form_value`, adds Feishu regression tests, and adds a changelog entry.

## Best Possible Solution

Land this PR or an equivalent Feishu plugin patch after redacted live Feishu proof, a fresh mergeability and CI refresh, and targeted Feishu tests; keep broader `input_value` and `name` support in https://github.com/openclaw/openclaw/issues/43202.

## Reproduction Assessment

Yes, at source level. A raw `card.action.trigger` payload containing sibling `action.option`, `action.options`, or `action.form_value` is normalized into an event with only `value` and `tag`, then fallback text serializes only `action.value`.

## Solution Assessment

Yes for the narrow scope. Preserving these fields at the Feishu parser/event boundary is the maintainable fix because a handler-only patch would not see production callbacks after normalization.
