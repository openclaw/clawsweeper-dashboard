## Summary

Current main still discards Feishu card-action selection and form fields, while this repaired branch preserves them across the raw parser and legacy fallback with focused coverage. The implementation is reasonable, but the external PR still needs after-fix proof from a real Feishu interaction before merge.

## What This Changes

The PR preserves Feishu card-action `option`, `options`, and `form_value` fields from raw event parsing through legacy fallback dispatch and adds unit and lifecycle coverage.

## Best Possible Solution

Keep text and command precedence unchanged, validate and preserve the three sibling fields at the raw Feishu parser boundary, include them only in the legacy JSON fallback, and demonstrate the resulting agent input with a real redacted Feishu card interaction.

## Reproduction Assessment

Yes, from source with high confidence: current main strips the sibling fields in `parseFeishuCardActionEventPayload`, and the legacy fallback serializes only `action.value`. A real Feishu callback was not executed during this read-only review.

## Solution Assessment

Yes. Preserving validated fields at the parser boundary and merging them only into the existing legacy JSON fallback is narrower and safer than changing the structured interaction protocol, and the branch retains command/text precedence.
