## Summary

Keep open: current main still drops Feishu `option`, `options`, and `form_value`, and this PR is the canonical open branch for that narrow fix. The patch looks correct after earlier feedback was addressed, but it still needs real Feishu after-fix proof and maintainer landing checks.

## What This Changes

This PR adds Feishu `option`, `options`, and `form_value` pass-through from card-action parsing into legacy fallback serialization, with Feishu card-action tests and a changelog entry.

## Best Possible Solution

Land this PR or an equivalent Feishu plugin patch after redacted live Feishu proof, a fresh rebase/CI pass, and targeted tests; keep broader `input_value` and `name` support in https://github.com/openclaw/openclaw/issues/43202.

## Reproduction Assessment

Yes, source-level. A raw `card.action.trigger` payload containing sibling `action.option`, `action.options`, or `action.form_value` is normalized into an event with only `value` and `tag`, then fallback text serializes only `action.value`.

## Solution Assessment

Yes for the narrow scope. The branch preserves the fields at the raw parser boundary, carries them through fallback serialization, and keeps legacy `value.text` and `value.command` precedence; broader callback-field support should remain separate.
