## Label Justifications

- `P3`: This is a useful but speculative enhancement for multi-agent ergonomics rather than a regression or broken documented behavior.
- `impact:session-state`: The request is about automatically moving user work between main sessions and subagent sessions, which affects session routing and context ownership.

## Summary

Keep open: the reported `subsession-router` workaround is a real plugin/API gap, not something current main or ClawHub alone resolves. Current hooks can add prompt guidance, block a run, or observe/coordinate an already-started subagent, but they cannot force the initial user task into `sessions_spawn` before the main model run.

## What This Changes

Requests an opt-in gateway/plugin API or config that can force selected tasks into `sessions_spawn` subagent runs instead of relying on `before_prompt_build` prompt guidance.

## Best Possible Solution

Design an explicit opt-in routing contract that runs before the main model call, preserves existing `sessions_spawn` allowlists/sandbox/depth limits, records auditable routing decisions, and documents how existing prompt-only delegation remains compatible.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The capability gap is source-verifiable from current hook result types, delegation docs, and `sessions_spawn` policy tests.

## Solution Assessment

Unclear until maintainers choose the product and security model. The current prompt-injection workaround is not the best long-term solution because it relies on model compliance, but unconditional forced routing also needs an explicit trust boundary.
