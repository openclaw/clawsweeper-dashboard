## Summary

Keep open: current main and the latest release still provide prompt-only delegation guidance, pass/block gate hooks, and explicit `sessions_spawn`; the requested hard pre-model delegation contract remains a security-sensitive plugin/API product decision.

## What This Changes

Requests a first-class gateway/plugin routing contract or config path that can force selected prompts into subagent sessions instead of relying on prompt-only `appendSystemContext` guidance.

## Best Possible Solution

Design an explicit opt-in pre-model routing contract that reuses the existing `sessions_spawn` policy path, keeps prompt-only delegation as the default, and documents the plugin/security guarantees before implementation.

## Reproduction Assessment

Yes for the source-level feature gap, not as a broken-behavior repro: current main shows prompt-only delegation and hook gates that cannot return a spawn/routing decision. The latest release has the same public limitation.

## Solution Assessment

Unclear until maintainers choose the API and security model. The safer direction is a first-class opt-in routing contract that reuses the existing spawn policy path rather than a soft prompt workaround, broad REST spawn endpoint, or unconditional config override.
