## Summary

Keep open: current main still only supports prompt guidance, pass/block gates, and post-spawn lifecycle hooks; the requested hard pre-model delegation contract remains a product and security-sensitive feature gap.

## What This Changes

The issue requests a gateway/plugin API or opt-in config path that can force selected tasks into subagent sessions instead of relying on prompt guidance to call `sessions_spawn`.

## Best Possible Solution

Design an explicit opt-in pre-model routing contract that keeps `sessions_spawn` policy checks authoritative, records routing decisions, and leaves prompt-only delegation as the default behavior.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an additive feature request. Source inspection confirms current main has prompt guidance and hook gates but no hard pre-model delegation result.

## Solution Assessment

Unclear until maintainers choose the API and security model. The maintainable direction is an explicit opt-in routing contract rather than a prompt-injection workaround or unconditional hard-routing default.
