## Summary

Keep open: current main still only offers prompt/context hooks, pass/block gates, and post-spawn lifecycle hooks; forced pre-model subagent routing remains an unimplemented product and security-sensitive feature rather than stale or duplicate.

## What This Changes

The issue requests a gateway or plugin API/config contract that can force selected tasks into subagent sessions instead of relying on prompt-only delegation guidance.

## Best Possible Solution

Design an explicit opt-in pre-model routing contract that preserves the existing sessions_spawn policy boundary, records auditable routing decisions, and leaves prompt-only delegation as the default behavior.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an additive feature request. Source inspection gives a high-confidence absence check: current hooks provide prompt mutation, pass/block gates, and post-spawn lifecycle coordination, but no forced initial routing result.

## Solution Assessment

Unclear until maintainers choose the API and security model. The maintainable direction is an explicit opt-in routing contract, not a prompt-injection workaround or unconditional hard-routing default.
