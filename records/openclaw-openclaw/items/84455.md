## Summary

Keep open: this is a real plugin/API feature gap rather than an already-implemented behavior. Current main supports prompt guidance, blocking, and post-spawn subagent lifecycle hooks, but not a pre-model hard-routing contract that forces the original user task into `sessions_spawn`; the reported `subsession-router` workaround is useful evidence that ClawHub alone cannot solve the missing core extension point.

## What This Changes

The issue requests a gateway-level API or opt-in config path for plugins to force selected user tasks into subagent sessions instead of relying on prompt-only delegation guidance.

## Best Possible Solution

Design an explicit opt-in routing contract that runs before the main model call, preserves existing `sessions_spawn` allowlists, sandbox/depth/child limits, and auditability, and leaves prompt-only delegation unchanged by default.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The capability gap is source-verifiable from the current hook result types, prompt-only delegation docs, and `sessions_spawn` HTTP/policy boundary tests.

## Solution Assessment

Unclear until maintainers choose the product and security model. The prompt-injection workaround is not a durable solution, but unconditional forced routing should not be added without an explicit opt-in trust boundary.
