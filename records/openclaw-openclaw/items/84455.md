## Summary

Keep open: current main and v2026.6.8 still only provide prompt guidance, pass/block hook gates, and explicit `sessions_spawn`; the requested hard pre-model delegation contract remains a security-sensitive product/API design gap.

## What This Changes

The issue requests a plugin or gateway routing API/configuration that can force selected tasks into subagent sessions instead of relying on prompt-injected guidance.

## Best Possible Solution

Design an explicit opt-in pre-model routing contract that preserves `sessions_spawn` policy checks, sandbox guards, depth/child limits, and audit visibility, while keeping prompt-only delegation as the default.

## Reproduction Assessment

Yes for source behavior, not for a bug repro: current main shows prompt-only delegation and hook gates that cannot return a spawn/routing decision. The latest release has the same public limitation.

## Solution Assessment

Unclear until maintainers choose the API and security model. The safer direction is a first-class opt-in routing contract that reuses the existing spawn policy path rather than a soft prompt workaround or direct HTTP spawn endpoint.
