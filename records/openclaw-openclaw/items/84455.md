## Summary

Keep open: current main still offers prompt-only delegation guidance, pass/block gate hooks, and lifecycle hooks around already-created subagents; a forced pre-model routing contract remains a product and security-sensitive feature gap.

## What This Changes

Request a gateway or plugin routing contract that can force selected prompts into subagent sessions instead of relying on prompt-only `sessions_spawn` guidance.

## Best Possible Solution

Design an explicit opt-in pre-model routing contract that preserves existing `sessions_spawn` policy boundaries and records auditable routing decisions while leaving prompt-only delegation unchanged by default.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an additive feature request. Source inspection confirms the current contracts provide prompt guidance, pass/block gates, and post-spawn lifecycle hooks, but no forced initial routing result.

## Solution Assessment

Unclear until maintainers choose the API and security model. The maintainable direction is an explicit opt-in routing contract, not a prompt-injection workaround or unconditional hard-routing default.
