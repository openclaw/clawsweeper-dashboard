- `impact:session-state`: Forced delegation would decide which session owns the task, context, transcript, and child completion state.

## Summary

Keep open: current main and v2026.6.8 still provide prompt guidance, pass/block gates, and existing session-spawn tooling, but not the requested plugin/gateway hard-routing contract. The reporter's `subsession-router` workaround is useful signal, but forcing automatic delegation would add product and security-sensitive session authority that maintainers still need to design.

## What This Changes

This issue requests a plugin hook, gateway API, or config surface that can force selected tasks into subagent sessions instead of relying on `appendSystemContext` prompt guidance.

## Best Possible Solution

Design an explicit opt-in pre-model routing contract only after maintainers settle the product and security model, keeping `sessions_spawn` policy checks authoritative and prompt-only delegation as the default behavior.

## Reproduction Assessment

Not applicable as a bug reproduction: this is an additive feature request. Source and release inspection give high confidence that current main and v2026.6.8 only provide prompt guidance or pass/block hooks, not hard delegation.

## Solution Assessment

Unclear as an implementation path until maintainers choose the API and security model. The maintainable direction is an explicit opt-in routing contract rather than treating prompt injection or a generic REST spawn endpoint as session authority.
