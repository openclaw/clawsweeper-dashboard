## Summary

Keep open: current main and the latest shipped release still provide prompt-only delegation guidance plus pass/block or post-spawn lifecycle hooks, not a gateway-enforced pre-run route into `sessions_spawn`. The reported `subsession-router` workaround points at a real plugin/API gap that needs product and trust-boundary review rather than cleanup closure.

## What This Changes

The issue requests a gateway/plugin API or opt-in config surface that can force selected tasks into subagent sessions instead of relying on prompt-injected delegation guidance.

## Best Possible Solution

Design an explicit opt-in pre-run routing contract that preserves existing `sessions_spawn` policy checks, records auditable routing decisions, and keeps prompt-only delegation as the default compatibility path.

## Reproduction Assessment

No bug reproduction is applicable because this is a feature request. The capability gap is source-reproducible from current hook result types, config schema, docs, and `sessions_spawn` policy tests.

## Solution Assessment

Unclear until maintainers choose the product and security model. Prompt-injected routing suggestions are not a robust long-term answer, but hard routing should be opt-in and policy-aware rather than a bypass around the existing `sessions_spawn` trust boundary.
