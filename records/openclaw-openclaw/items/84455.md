## Summary

Keep open: current main still only supports prompt guidance, prompt-mutating hooks, pass/block input gates, and explicit `sessions_spawn` guardrails; the requested hard-routing contract is a new product and security-sensitive capability rather than implemented core behavior.

## What This Changes

The issue asks for a gateway/plugin/config API that can force selected prompts into subagent sessions instead of relying on model compliance with prompt instructions.

## Best Possible Solution

Design an explicit opt-in pre-model routing contract that preserves current `sessions_spawn` authority checks and auditability while leaving prompt-only delegation as the default behavior.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an additive feature request. Source inspection confirms current main has prompt guidance and pass/block gates but no hard initial-turn routing result.

## Solution Assessment

Unclear until maintainers choose the API and security model. The maintainable direction is an explicit opt-in routing contract, not a prompt-injection workaround or unconditional default delegation mode.
