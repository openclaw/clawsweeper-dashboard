## Summary

Keep open: the reporter's `subsession-router` workaround is credible evidence that plugins can classify work today, but current main still only offers prompt guidance, pass/block gates, plugin-owned background subagent runs, and post-spawn lifecycle hooks. A forced pre-model routing contract remains a product and security-sensitive feature gap rather than something already implemented or suitable for ClawHub alone.

## What This Changes

The issue requests a gateway-level or plugin-hook API that can force selected tasks into subagent sessions instead of relying on prompt-only delegation guidance.

## Best Possible Solution

Design an explicit opt-in pre-model routing contract that leaves prompt-only delegation unchanged by default and routes forced delegation through the same policy and audit checks as `sessions_spawn`.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an additive feature request. Source inspection gives high confidence that current main has prompt guidance, pass/block gates, plugin-owned background subagent runs, and post-spawn hooks, but no forced current-turn routing result.

## Solution Assessment

Unclear until maintainers choose the API and security model. The maintainable direction is a narrow opt-in routing contract, not a prompt-injection workaround or an unconditional default that bypasses `sessions_spawn` policy.
