## Summary

Keep open. Current main still has the reported `/status` display gap, and an open closing PR already targets the same bug, so the issue should remain open until that PR lands or maintainers choose a different canonical fix.

## What This Changes

The issue reports that `/status` can show an Anthropic env API-key auth label even when the active equivalent `claude-cli` route is using OAuth.

## Best Possible Solution

Review and land the canonical open fix PR, or replace it with an equivalent narrow renderer fix that prefers active CLI OAuth over selected env API-key for equivalent runtime aliases and covers the regression.

## Reproduction Assessment

Yes for the source-level renderer path: selected `anthropic/*` plus active `claude-cli/*`, `modelAuth: "api-key"`, and `activeModelAuth: "oauth"` still lets the selected API-key label win on current main. I did not establish a live gateway reproduction in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is to adjust the shared status renderer/auth-label choice and cost gating for equivalent runtime aliases, with a regression for selected env API-key plus active Claude CLI OAuth.
