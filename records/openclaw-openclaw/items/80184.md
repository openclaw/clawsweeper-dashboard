## What This Changes

The issue asks for `/status` to show the active Claude CLI OAuth auth label instead of an unrelated Anthropic env API-key label when a runtime alias is in use.

## Best Possible Solution

Land or revise the linked fix PR so status rendering prefers the active OAuth label for equivalent runtime aliases when the selected route resolved from an env API key, with focused regression coverage.

## Reproduction Assessment

Yes for the core renderer path: current source still lets a concrete selected `api-key` label suppress the active OAuth label when the selected and active model refs are runtime aliases. I did not establish a live gateway reproduction.

## Solution Assessment

Yes. The narrowest maintainable fix is in status rendering plus regression coverage for the env-key shadow case; changing model auth resolution would be broader than the display-only bug requires.
