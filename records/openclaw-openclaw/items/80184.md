## Summary

Keep open. Current main and latest v2026.5.12 still have the reported `/status` display gap, and the linked open fix PR should be reviewed or landed before this issue closes.

## What This Changes

The issue asks `/status` to show active `claude-cli` OAuth instead of an env API-key label when both a runtime alias and `ANTHROPIC_API_KEY` are present.

## Best Possible Solution

Land the existing narrow renderer fix in https://github.com/openclaw/openclaw/pull/80260 after normal CI and maintainer review, then close this issue as fixed.

## Reproduction Assessment

Yes for the core renderer path: selected `api-key` auth remains concrete and wins over active OAuth for equivalent runtime aliases on current main. I did not run a live gateway reproduction with real Claude CLI credentials.

## Solution Assessment

Yes. The narrowest maintainable fix is to make the shared status renderer prefer active non-unknown OAuth for equivalent runtime aliases and suppress API-key cost display in that case; broad auth-resolution policy changes are not needed for this display bug.
