## What This Changes

The issue asks /status to show the active Claude CLI OAuth auth path instead of an Anthropic env API-key label when the selected Anthropic model is routed through an equivalent claude-cli runtime alias.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/80260 or an equivalent focused fix with regression coverage for selected env API-key auth plus active OAuth on an equivalent CLI runtime alias.

## Reproduction Assessment

Yes for the core renderer path: current main resolves Anthropic env auth to a concrete `api-key` label and then preserves that selected label instead of the active OAuth label for equivalent runtime aliases. I did not run the full live gateway/OAuth setup.

## Solution Assessment

Yes. The narrow maintainable fix belongs in status auth-label selection, with a regression for `api-key` selected auth and active `oauth`; changing general auth resolution would be broader than the display bug.
