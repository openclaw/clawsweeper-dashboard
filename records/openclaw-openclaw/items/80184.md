## What This Changes

The issue reports that `/status` can show `api-key (env: ANTHROPIC_API_KEY)` even when the active equivalent `claude-cli` runtime path is using OAuth.

## Best Possible Solution

Review and land a focused fix that prefers the active non-unknown OAuth label for equivalent CLI runtime aliases when the selected label is an env API-key, with regression coverage for the `ANTHROPIC_API_KEY` shadow case.

## Reproduction Assessment

Yes for the renderer path: source inspection shows a selected `api-key (...)` label is treated as concrete and prevents borrowing the active OAuth label for equivalent runtime aliases. I did not perform a full live gateway run.

## Solution Assessment

Yes. The narrow maintainable solution is a status-renderer/test change; changing model auth resolution or runtime config policy would be broader than this display-only bug.
