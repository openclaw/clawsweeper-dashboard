## What This Changes

Request `/status` to show active Claude CLI OAuth instead of an env API-key label when an equivalent `claude-cli` runtime alias is actually handling the turn and `ANTHROPIC_API_KEY` is also present.

## Best Possible Solution

Land one canonical focused fix that makes status label and cost display use the active non-unknown auth for equivalent CLI runtime aliases, with regression coverage for selected env API-key plus active Claude CLI OAuth.

## Reproduction Assessment

Yes. The core renderer path is source-reproducible by using selected `anthropic/<model>`, active `claude-cli/<model>`, selected `api-key`, and active `oauth`, which current main resolves to the selected API-key label.

## Solution Assessment

Yes. The narrow maintainable solution is to adjust the status renderer's alias auth selection and cost decision, with focused regression coverage; broad auth-provider policy changes are not required for this display bug.
