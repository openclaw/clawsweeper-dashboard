## Summary

Keep open. Current main and the latest shipped tag still have the source-backed `/status` rendering gap, and the linked fix at https://github.com/openclaw/openclaw/pull/80260 has not merged yet.

## What This Changes

The issue asks `/status` to show the active Claude CLI OAuth auth path instead of an ambient Anthropic env API-key label when selected and active model refs are runtime aliases.

## Best Possible Solution

Land or otherwise resolve the focused status-renderer fix with regression coverage for selected API-key plus active OAuth runtime aliases, including cost-line behavior.

## Reproduction Assessment

Yes. Source inspection of current main gives a high-confidence renderer path: selected `anthropic/<model>` with `modelAuth: "api-key"`, active `claude-cli/<model>` with `activeModelAuth: "oauth"`, and runtime-alias equivalence still leaves the selected API-key label in place.

## Solution Assessment

Yes. The narrowest maintainable solution is to adjust status label and cost selection for equivalent runtime aliases; changing global auth resolution would be broader than the reported display bug.
