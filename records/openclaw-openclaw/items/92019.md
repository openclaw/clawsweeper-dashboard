## Summary

The report identifies a credible unreleased regression on current main, but its root-cause theory is only partly supported: OpenClaw already detects model changes, and Anthropic documents signatures as cross-platform compatible. The remaining high-confidence bug is unsafe signed-thinking replay for unowned or proxied Anthropic transports, which needs a focused fix and live proxy validation rather than a new config flag.

## What This Changes

Prevent proxied or otherwise unowned `anthropic-messages` sessions from replaying signed thinking blocks that the destination rejects and permanently poisoning subsequent turns.

## Best Possible Solution

Use the existing transcript-policy and model-snapshot sanitation pipeline to preserve complete signed blocks only for explicitly owned Anthropic-family transports and compatible active tool turns, while dropping unverifiable older blocks for unowned proxy transports; validate the exact Bifrost or LiteLLM failure live before landing.

## Reproduction Assessment

Yes at source level with high confidence: current main deliberately preserves newer-Claude thinking blocks for an unowned custom proxy, and the issue provides the matching 400 response plus a verified forced-drop workaround. A live Bifrost or LiteLLM run is still needed to prove the endpoint-specific condition before landing.

## Solution Assessment

No, the proposed environment flag is not the best primary solution. The narrow maintainable fix is automatic sanitation in the existing replay boundary, with the current model-snapshot path handling model switches and no additional config surface.
