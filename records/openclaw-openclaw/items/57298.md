## Summary

Keep open: the linked Telegram bug is real, but this PR head still guards the older direct reply-media helper and does not protect current main's `resolveReplyMediaForChain` fetch path. It also lacks required after-fix real Telegram proof for the visible reply behavior.

## What This Changes

The PR adds a numeric self-authored sender guard for Telegram reply media, regression tests, and a changelog entry for replies to bot-authored photos.

## Best Possible Solution

Refresh the branch to add the numeric bot-id guard in `resolveReplyMediaForChain` before `bot.api.getFile`, preserve user and other-bot reply media, keep focused regression coverage, and attach redacted live Telegram proof.

## Reproduction Assessment

Yes. Source inspection shows a Telegram text update with bot-authored `reply_to_message.photo` can enter `buildTelegramReplyChain` and reach `resolveReplyMediaForChain`, where `bot.api.getFile` is called without a self-authored reply-target check.

## Solution Assessment

No as-is. The numeric sender-id guard is the right narrow fix, but this PR places it in the older `resolveReplyMediaForMessage` path instead of current main's reply-chain fetch path.
