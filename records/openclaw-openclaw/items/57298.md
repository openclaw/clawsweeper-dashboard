## Summary

Keep open: this is a focused fix for a real Telegram reply-media bug, but the current branch is not mergeable as-is because it guards the older direct reply-media helper while current main resolves reply media through `resolveReplyMediaForChain`; external real Telegram proof is also still missing.

## What This Changes

The PR adds a numeric bot-id guard and Telegram bot tests intended to prevent replies to bot-authored photos from reinjecting the bot's own media, plus a changelog entry.

## Best Possible Solution

Refresh or replace the PR with a numeric self-authored check in `resolveReplyMediaForChain` before `bot.api.getFile`, preserve user and other-bot reply media, keep focused regression coverage, and attach redacted live Telegram proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a Telegram text update whose `reply_to_message.photo` is authored by `ctx.me` can enter the reply chain and reach `bot.api.getFile` before any self-authored reply-target check.

## Solution Assessment

No as-is. The numeric bot-id guard is the right narrow fix, but it needs to protect the live reply-chain media fetch path and be backed by real Telegram proof before merge.
