## Summary

Keep open: the PR still targets the older direct Telegram reply-media resolver while current main resolves reply media through the reply-chain fetch path, and the external real Telegram proof gate is still missing.

## What This Changes

The PR adds a Telegram bot-ID guard to skip media from bot-authored reply targets, related regression tests, and a changelog entry.

## Best Possible Solution

Refresh or replace the branch so the numeric self-authored check runs in `resolveReplyMediaForChain` before `bot.api.getFile`, preserves user and other-bot reply media, keeps focused tests, and adds redacted live Telegram proof.

## Reproduction Assessment

Yes. Source inspection shows a Telegram text reply with `reply_to_message.photo` can fall back into `buildTelegramReplyChain`, and `resolveReplyMediaForChain` then calls `bot.api.getFile` for media nodes without comparing the reply-node sender to `ctx.me`.

## Solution Assessment

No as-is. The numeric bot-ID guard is the right narrow direction, but it needs to be applied to the current reply-chain fetch path and backed by live Telegram proof before merge.
