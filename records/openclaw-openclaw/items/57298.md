## Summary

Keep open: this is a valid Telegram bug-fix PR, but the branch is now conflicting with current main and its guard is attached to the older reply-media resolver instead of the current reply-chain media fetch path; it also needs real Telegram proof before merge.

## What This Changes

The PR adds a numeric bot-ID guard before Telegram reply-target media fetches, focused regression tests, and a changelog entry for the linked bug.

## Best Possible Solution

Refresh or replace this PR with a narrow patch that skips self-authored Telegram reply-chain media before `bot.api.getFile`, preserves user and other-bot reply media, keeps focused regression coverage, and includes redacted live Telegram proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a user text reply with `reply_to_message.photo` can enter the reply chain and current `resolveReplyMediaForChain` fetches that media without checking whether the chain node was authored by `ctx.me`.

## Solution Assessment

No as-is. The intended numeric self-authorship guard is the right narrow fix, but it must be adapted to the current reply-chain fetch path and backed by real Telegram proof before merge.
