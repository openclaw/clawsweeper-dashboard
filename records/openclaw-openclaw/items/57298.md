## Best Possible Solution

Refresh the branch to guard self-authored reply-chain nodes before file fetch, preserve user and other-bot reply media, keep focused regression tests, and attach redacted live Telegram proof.

Yes, there is a high-confidence source-level reproduction path: a Telegram text update with bot-authored `reply_to_message.photo` can fall into `buildTelegramReplyChain` and then reach `resolveReplyMediaForChain`, where current main calls `bot.api.getFile` without a self-authored sender check.

## Solution Assessment

No as-is. The numeric bot-ID guard is the right narrow fix, but it needs to be placed in current main's reply-chain fetch path before `bot.api.getFile`, not only in the older reply-media helper shape.
