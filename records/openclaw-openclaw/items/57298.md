## Summary

Keep open. Current main still fetches Telegram reply-target media without checking whether the reply target was authored by the bot, while this PR’s repaired head adds the narrow numeric-ID guard, regression coverage, and changelog entry needed for #57278. There are no blocking code-review findings from the provided diff; the remaining action is maintainer refresh, validation, and landing judgment.

## What This Changes

The PR adds a numeric Telegram bot-ID guard before reply-target media resolution, adds regression tests for self-authored, different-bot, and debounced reply media, and adds a changelog entry for #57278.

## Best Possible Solution

Refresh and land this PR or an equivalent narrow patch that skips only self-authored Telegram reply-target media when both bot and reply sender ids are numeric and equal, while preserving reply media from users and other bots and keeping the regression tests plus changelog credit.

## Reproduction Assessment

Yes. A synthetic Telegram text message from a user with `reply_to_message.photo` authored by `ctx.me.id` reaches `resolveReplyMediaForMessage` on current main and proceeds to `bot.api.getFile`; the existing reply-image test already asserts the fetch path for reply media.

## Solution Assessment

Yes. The current PR approach is the narrow maintainable fix because it gates only the reply-target media fetch, uses numeric id checks to avoid optional-field false positives, and leaves ordinary user and different-bot reply media behavior intact.
