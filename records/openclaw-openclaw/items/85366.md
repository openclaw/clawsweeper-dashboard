The issue asks OpenClaw to prevent ACP startup identity reconciliation and session-lock cleanup from running large serial fanout on the gateway event loop.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/85399, with the desired end state being bounded, yielding startup-sidecar fanout that preserves ACP/session metadata contracts.
