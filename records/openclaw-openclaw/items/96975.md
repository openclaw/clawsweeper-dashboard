## Summary

Keep open: current main and v2026.6.10 still carry child result text into parent prompt and completion delivery paths, while this issue asks for a new metadata-only subagent completion contract. The linked lock and delivery reports are real adjacent work, but none owns the status/session-link-only default, so this needs maintainer product and compatibility design rather than cleanup close.

## What This Changes

Request subagent completion isolation so parent sessions and user channels receive only a bounded status/session handle by default and fetch child details explicitly.

## Best Possible Solution

Define a core subagent completion envelope that keeps child results durable in session/artifact storage, injects only bounded status and session handles by default, and makes full-result transfer explicit with a documented compatibility plan.
