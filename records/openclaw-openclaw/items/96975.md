## Summary

Keep open: current main and v2026.6.10 still carry child completion text into the parent completion event, and the existing docs/tests describe that behavior rather than the requested metadata-only envelope. The remaining work is a compatibility-sensitive completion contract decision, not conservative cleanup.

## What This Changes

The issue asks OpenClaw to isolate subagent completion so parent sessions receive only a bounded status/session-handle envelope by default instead of raw child output.

## Best Possible Solution

Define a stable subagent completion contract that keeps parent/user delivery bounded by default while preserving an explicit, bounded way to fetch child results when the parent needs them.

## Reproduction Assessment

Yes, at source level: current main reads child assistant output and renders it into the parent completion event, with tests and docs explicitly expecting that result-bearing handoff. I did not run a live heavy subagent overflow workflow in this read-only review.

## Solution Assessment

Unclear: metadata-only completion is a plausible safer contract, but it is not a small bug fix because current docs/tests expect Result to carry child assistant text and compatibility policy must be chosen first.
