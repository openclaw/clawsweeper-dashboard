## Summary

Keep open: current main and the latest release still put child result text into the subagent task-completion event and parent prompt path, while the related open delivery PR only covers durability/lock handling rather than the requested metadata-only completion contract.

## What This Changes

The issue requests a default subagent completion contract that keeps child transcript/result content out of the parent context and returns only bounded status/session metadata unless explicitly fetched.

## Best Possible Solution

Define a bounded subagent completion envelope with status/session identifiers and an explicit bounded child-result reader, then update the protocol schema, docs, and regression tests around that contract.

## Reproduction Assessment

Yes, at source level: current main still captures child result text, requires it in the internal event schema, and tests assert it appears in the parent prompt. I did not rerun the live Feishu workflow that triggered the report.

## Solution Assessment

Unclear: metadata-only completion is a strong direction for isolation, but the exact envelope fields, bounded-summary rule, explicit reader tool, and compatibility mode need maintainer product/API direction.
