## Summary

Current main still treats subagent completion as a payload-bearing internal event and direct fallback can send the child result text, so the metadata-only completion contract requested by #96975 is not implemented or superseded; keep the issue open for maintainer product direction on the default contract and compatibility path.

## What This Changes

Issue #96975 requests changing subagent completion delivery to send metadata-only completion envelopes by default, with concise summaries or an explicit compatibility/config path for including full child output.

## Best Possible Solution

Keep the issue open until maintainers decide the subagent completion contract, then implement one canonical metadata-first path with explicit compatibility behavior and focused tests for parent context, direct channel fallback, protocol schema, docs, and durable completion state.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: current main builds completion events from child result text, documents that Result in the handoff, and tests assert that payload-bearing child output reaches parent prompts or direct channel fallback.

## Solution Assessment

Unclear as an implementation until maintainers decide the contract: metadata-only completion is a plausible best direction for isolation, but the exact envelope fields, explicit result-reader path, and compatibility/config behavior are product-sensitive because payload-bearing results are current documented and tested behavior.
