## Summary

Keep open: current main and v2026.6.10 still carry child result text through the subagent completion event and requester prompt path, while the requested metadata-only completion envelope remains a product/API contract decision.

## What This Changes

Request metadata-only subagent completion by default, with status/session identifiers and an explicit child-session/history reader instead of automatic child-result injection into the parent context.

## Best Possible Solution

Define a bounded subagent completion envelope, update protocol/docs/tests around that contract, and keep full child details available only through explicit bounded history or artifact reads.
