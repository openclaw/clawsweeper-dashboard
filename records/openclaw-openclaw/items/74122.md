## Summary

Keep open: current main partially addresses config hot-reload coalescing, but it still has a 300 ms default debounce, watcher scheduling does not preserve the triggering filesystem path, and the scoped plugin/channel reload pieces are still active design work rather than a fully implemented behavior.

## What This Changes

Request coalescing rapid watched-file edits into fewer Gateway reloads, scoped reloads for non-config/plugin/channel changes, and clearer reload-attribution logging.

## Best Possible Solution

Split the config reloader coalescing and trigger-attribution work from the broader plugin hot-reload and channel reload designs tracked by https://github.com/openclaw/openclaw/issues/14438 and https://github.com/openclaw/openclaw/issues/72997.

## Reproduction Assessment

Partially. The config watcher slice is source-reproducible: the default debounce is 300 ms and chokidar callbacks do not retain the triggering path; the non-config/plugin-file slice is unclear on current main because the managed watcher is config-path scoped.

## Solution Assessment

Unclear. Longer/default coalescing and path logging are plausible narrow changes, but scoped non-config/plugin/channel reload needs maintainer scoping before it is the best maintainable solution.
