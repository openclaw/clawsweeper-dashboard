## Summary

Keep open: current main and v2026.6.6 still provide raw reset-memory capture, bounded startup-memory loading, and compaction-scoped memory flush, but not the requested structured `/new` synthesis, durable continuation package, or anchored transcript-link workflow. This remains a product and security boundary decision, not a narrow bug cleanup item.

## What This Changes

The issue requests automatic preservation and synthesis of session memory around `/new` resets, including summaries and links to full conversation transcripts.

## Best Possible Solution

Define a memory-plugin or session-lifecycle continuation package with bounded sanitized summaries, transcript receipts/links, explicit opt-in retention and redaction controls, and tests for reset hydration plus unsafe raw-memory input.

## Reproduction Assessment

Not applicable: this is a feature/design request, not a broken existing behavior report. Source inspection confirms current main has partial raw capture and startup loading, not the requested synthesis workflow.

## Solution Assessment

No: the proposed raw `.session_memory_preserve.json` and broad transcript dump is not the safest default. The better solution is a bounded, sanitized, opt-in continuation package with explicit retention and trust boundaries.
