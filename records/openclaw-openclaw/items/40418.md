## Summary

Keep open: current main partially preserves reset context through the bundled session-memory hook and startup memory loading, but it still lacks the requested structured synthesis, full transcript linking/anchors, and settled privacy/security policy. The related implementation PR was closed as superseded in favor of this issue, and narrower open issues only cover parts of the remaining lifecycle problem.

## What This Changes

Request automatic preservation and synthesis of session memory across `/new` resets, including durable summaries and links to full transcripts.

## Best Possible Solution

Define one opt-in memory lifecycle policy for reset/session-end continuity that separates raw audit artifacts from curated durable memory, preserves privacy boundaries, and coordinates with the existing startup-context and memory-plugin surfaces.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/product request. Source inspection gives high confidence that current main only provides bounded raw reset-memory capture plus startup loading, not the requested synthesis and transcript-linking behavior.

## Solution Assessment

Unclear until maintainers choose the memory lifecycle contract. The requested direction is useful, but a direct full-transcript/synthesis hook is not automatically the best solution without opt-in defaults, privacy review, and coordination with existing memory plugins and reset-flush work.
