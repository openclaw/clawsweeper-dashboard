## Summary

Keep open: current main has raw session-memory capture on `/new`/`/reset` plus bounded startup hydration, but the requested synthesized continuation package, transcript receipts/anchors, and default retention/safety policy are still unresolved and overlap with open #50584 and #69943.

## What This Changes

Add automatic `/new` session memory preservation with transcript export, synthesized daily summaries, and links back to full conversations.

## Best Possible Solution

Define a bounded session-memory continuation flow on the existing hook/startup-context path, resolving synthesis, receipt links, retention defaults, and sanitization together before changing shipped behavior.

Unclear as requested. The maintainable direction is a compact, sanitized continuation package with optional transcript receipts, not automatic promotion of full raw transcripts into future sessions by default.
