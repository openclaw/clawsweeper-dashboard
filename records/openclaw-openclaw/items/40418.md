## Summary

Keep open: current main and the latest release still provide partial mechanical reset-memory capture and bounded startup loading, but not the requested structured synthesis, dedupe, durable continuation package, transcript anchoring, or start-of-session summary generation. The remaining work is a product and security boundary decision, not conservative cleanup.

## What This Changes

The issue requests automated preservation and synthesis of session memory across `/new` resets, including saved session state, transcript exports, summaries, and links back to full conversations.

## Best Possible Solution

Define a maintainer-approved memory lifecycle contract first: a safe opt-in continuation package with curated summaries, transcript receipts, retention and redaction controls, untrusted historical-context framing, and tests across `/new`, `/reset`, compaction, and rollover paths.

## Reproduction Assessment

Not applicable: this is a feature/design request, not a broken existing behavior report. Source inspection confirms current main and v2026.6.8 have partial raw capture and bounded startup loading, but not the requested synthesis workflow.

## Solution Assessment

No for the raw full-dump shape in the issue body: automatic broad transcript export is not the safest maintainable default. A bounded, sanitized, opt-in continuation package with explicit retention, provenance, and trust boundaries is the better direction.
