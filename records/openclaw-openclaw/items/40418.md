## Summary

Keep open: current main has raw session-memory capture and bounded startup hydration, but it still does not provide the requested synthesized continuation package, transcript receipt links/anchors, retention policy, or safety model. The active related PR and issues cover parts of the space, but none has merged or fully superseded this broader product request.

## What This Changes

Request automatic preservation and synthesized memory continuity across `/new` resets, including durable summaries, transcript links, and startup hydration.

## Best Possible Solution

Define a bounded, sanitized continuation flow on the existing hook/startup-context path, coordinating synthesis, receipt links, retention defaults, and sanitization with the related open threads.

## Solution Assessment

Unclear as proposed. The maintainable direction is a compact, sanitized continuation package with optional transcript receipts, not automatic full raw transcript promotion into future sessions by default.
