## Summary

Keep open: current main preserves some reset context mechanically and hydrates bounded memory on startup, but it still does not provide the requested synthesized continuation package, transcript receipts/anchors, retention policy, or safety model. The overlapping open PR and related issues cover parts of the space but do not fully supersede this broader request.

## What This Changes

The issue requests automatic session memory preservation across `/new`, including synthesized summaries and links to full prior-session transcripts.

## Best Possible Solution

Land a maintainer-approved continuation design that writes a compact sanitized summary with optional transcript receipts, while keeping raw transcript retention gated, bounded, and documented.

## Reproduction Assessment

Not applicable. This is a feature request; source inspection verifies current main has raw capture and bounded startup hydration rather than a documented behavior failing to reproduce.

## Solution Assessment

Unclear as requested. A compact, sanitized continuation package with optional transcript receipts is safer than automatically exporting and promoting full raw transcripts into future-session context by default.
