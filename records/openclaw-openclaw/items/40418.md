## Summary

Keep open: current main has reset-time mechanical capture and bounded startup hydration, but it still lacks the requested synthesized continuation package, transcript receipts/anchors, retention policy, and sanitization model. The open related threads cover parts of the space but do not fully replace this feature request.

## What This Changes

Request automatic preservation and synthesized continuation memory around `/new` resets, including transcript exports/links and future-session hydration.

## Best Possible Solution

Define a bounded, sanitized continuation flow on the existing hook/startup-context path, coordinating synthesis, optional transcript receipts, dedupe, retention defaults, and sanitizer policy with the related open threads.

Not applicable. This is a feature request; source inspection shows current main has raw reset-time capture plus bounded startup loading, not a broken documented behavior to reproduce.

## Solution Assessment

Unclear as requested. The maintainable direction is a compact, sanitized continuation package with optional transcript receipts, not automatic full raw transcript promotion into future prompts by default.
