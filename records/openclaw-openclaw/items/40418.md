## Summary

Keep open: current main has an opt-in session-memory hook and runtime startup-context preload, but the requested synthesis, durable continuation package, transcript receipt/anchor behavior, and default policy are not implemented and overlap with open follow-up work.

## What This Changes

Request automatic preservation across `/new` resets with compact synthesized memory, full-transcript receipts, and links between summaries and prior session transcripts.

## Best Possible Solution

Define a bounded session-memory continuation flow in the existing hook/startup-context path, reviewing #50584 together with the sanitization and trust concerns raised in #69943 before changing defaults.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection confirms existing raw session-memory capture and startup hydration pieces rather than a current-main bug reproduction.

## Solution Assessment

No. Automatically dumping everything into memory is not the safest default; the maintainable direction is a compact, bounded continuation package with optional receipt links and sanitization on top of the existing session-memory seams.
