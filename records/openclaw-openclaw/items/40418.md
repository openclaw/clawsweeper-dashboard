## Summary

Keep open: current main and v2026.5.22 partially preserve reset/new-session context through raw background session-memory files and bounded startup hydration, but they still lack synthesized continuation packages, transcript receipts or anchors, and retention/redaction/provider-exposure policy.

## What This Changes

Request automatic cross-session memory preservation on `/new` or reset, including transcript export, synthesized summaries, and links from summaries back to prior transcripts.

## Best Possible Solution

Define an opt-in, bounded session-continuation design on the existing session-memory, startup-context, and memory-flush paths that creates compact sanitized summaries, durable facts/TODOs, and optional transcript receipts with explicit retention and provider policy.

Not applicable. This is a feature request, and source inspection shows partial raw capture plus bounded hydration rather than broken documented behavior to reproduce.

## Solution Assessment

Unclear as proposed. The maintainable direction is a compact, sanitized, opt-in continuation package with optional transcript receipts, not automatic full-transcript promotion into future sessions by default.
