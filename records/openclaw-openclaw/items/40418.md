## Summary

Keep open: current main and the latest release provide raw reset-time capture plus bounded startup hydration, but they do not implement synthesized continuation packages, transcript receipt links/anchors, or an explicit retention/redaction/trust contract. Related work overlaps this request, but the synthesis and sanitizer paths remain open and do not supersede the broader product/security decision.

## What This Changes

Requests automatic cross-session memory preservation on `/new`, including session-state saving, transcript export, synthesized summaries, and links back to full transcripts.

## Best Possible Solution

Define an opt-in, bounded session-continuation design on the existing hook/startup-context path, coordinated with raw-memory sanitization, transcript receipts, retention/redaction rules, docs, and focused tests.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection shows partial raw capture plus bounded hydration rather than a broken documented behavior to reproduce.

## Solution Assessment

Unclear as proposed. The maintainable direction is a compact, sanitized, opt-in continuation package with optional transcript receipts, not automatic raw transcript promotion into future sessions by default.
