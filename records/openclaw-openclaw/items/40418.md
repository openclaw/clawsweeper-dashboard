## Summary

Keep open: current main and v2026.6.6 have partial reset-memory capture, bounded startup daily-memory loading, and compaction-scoped memory flush, but they still do not provide the requested structured `/new` synthesis, durable continuation package, dedupe, anchored transcript links, or start-of-session summary generation. Related comments and open issues show this remains a product and security boundary decision rather than conservative cleanup.

## What This Changes

The issue requests automatic preservation, synthesis, transcript linking, and rehydration of session memory around `/new` resets.

## Best Possible Solution

Define a maintainer-approved memory lifecycle contract first: a safe opt-in continuation package with curated summaries, transcript receipts, retention/redaction controls, untrusted historical-context framing, and tests across `/new`, `/reset`, compaction, and stale rollover.

## Reproduction Assessment

Not applicable: this is a feature/design request, not a broken existing behavior report. Source inspection confirms current main has partial raw capture, bounded startup loading, and compaction flushes, but not the requested synthesis workflow.

## Solution Assessment

No for the raw-dump shape in the issue body: automatic `.session_memory_preserve.json` plus broad transcript export is not the safest maintainable default. The better path is a bounded, sanitized, opt-in continuation package with explicit retention, provenance, and trust boundaries.
