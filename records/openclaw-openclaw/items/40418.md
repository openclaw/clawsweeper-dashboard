## Summary

Keep open. Current main preserves bounded raw user/assistant turns on `/new` and `/reset` and can preload recent memory files, but it still does not implement structured synthesis, anchored transcript receipts, or a compact continuation package; open PR #50584 and bug #69943 are the live design blockers.

## What This Changes

Request automatic session-memory preservation and synthesized startup continuity across `/new`, with links back to full transcripts.

## Best Possible Solution

Land or replace #50584 with an opt-in, bounded session-memory design that writes compact durable decisions, TODOs, preferences, and receipt links while sanitizing raw artifacts and documenting retention behavior.

## Reproduction Assessment

Yes. A source-level reproduction path exists: enable `session-memory`, trigger `/new` or `/reset` after user/assistant turns, and current main writes recent role-prefixed transcript text into a slugged memory file that startup context can later preload.

## Solution Assessment

Unclear. The layered summary-with-receipts direction is safer than unconditional transcript promotion, but the implementation should be settled through #50584 or a maintainer-approved replacement that covers #69943's sanitization and retention concerns.
