## Summary

Keep open: current main and v2026.5.22 provide raw bounded session-memory capture plus bounded startup hydration, but not the requested synthesized continuation package with transcript receipts, anchors, retention/redaction policy, and provider-exposure decisions.

## What This Changes

Request automatic preservation and synthesis of session memory across `/new` resets, including durable summaries and links back to full transcripts.

## Best Possible Solution

Define an opt-in, bounded session-continuation design on top of the existing session-memory and startup-context paths, with sanitized compact summaries, durable facts/TODOs, optional transcript receipts, and explicit retention/provider policy.
