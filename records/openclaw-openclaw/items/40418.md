## Summary

Keep open: current main and v2026.6.6 preserve some reset context through raw session-memory artifacts and startup loading, but they do not implement the requested structured synthesis, anchored transcript links, or settled privacy/security lifecycle.

## What This Changes

Request automatic preservation of session context on `/new`, with transcript export, synthesized summaries, and links from summaries back to full transcripts.

## Best Possible Solution

Define an opt-in memory lifecycle policy that separates raw audit logs from curated durable memory, uses existing memory/plugin surfaces where possible, and resolves privacy, poisoning, and transcript-link retention before implementation.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main only provides bounded raw reset-memory capture plus startup daily-memory loading, not structured synthesis or anchored transcript links.

## Solution Assessment

Unclear until maintainers choose the memory lifecycle contract. The requested direction is useful, but the best solution needs opt-in defaults, privacy review, and coordination with existing memory plugins and reset-flush work.
