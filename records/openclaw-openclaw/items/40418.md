## Summary

Keep open. Current main only partially covers the requested continuity flow: the bundled session-memory hook preserves bounded raw user/assistant turns and startup context can preload recent memory files, but there is still no merged structured synthesis mode, anchored transcript receipt workflow, or compact continuation package. Open PR #50584 remains the live implementation candidate, and open #69943 makes sanitization and retention policy a required maintainer/security design point before expanding this feature.

## What This Changes

Request automatic session memory preservation across `/new` resets, including synthesized summaries, links to full transcripts, and compact startup continuation across sessions.

## Best Possible Solution

Land or replace #50584 with an opt-in, bounded design that distills durable decisions, TODOs, preferences, and receipts into compact startup memory; keeps raw transcripts available only as bounded, untrusted, non-default references; sanitizes unsafe role/tool/control-token artifacts before persistence or synthesis; and updates tests/docs for dedupe-aware startup loading.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: enable `session-memory`, trigger `/new` or `/reset`, and current main writes recent raw user/assistant transcript text into a slugged memory file that startup context can later preload. No live run was attempted because this review was explicitly read-only.

## Solution Assessment

Unclear as originally proposed. The discussion’s layered summary-with-receipts direction is safer than unconditional transcript promotion, but the final implementation should be settled through the open PR or a maintainer-approved replacement that covers privacy, retention, synthesis, and sanitization policy.
