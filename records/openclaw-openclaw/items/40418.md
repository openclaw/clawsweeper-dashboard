## Summary

Keep open: current main partially preserves reset/new session context through raw background capture and bounded startup hydration, but it still lacks synthesized continuation packages, transcript receipts or anchors, and the retention/redaction/provider-exposure policy needed for this request.

## What This Changes

Request automatic cross-session memory preservation on `/new`, including saved session state, transcript export, synthesized summaries, daily memory updates, and links back to full transcripts.

## Best Possible Solution

Define an opt-in, bounded session-continuation design on the existing session-memory, startup-context, and memory-flush paths, coordinated with https://github.com/openclaw/openclaw/issues/45608, https://github.com/openclaw/openclaw/pull/50584, and https://github.com/openclaw/openclaw/issues/69943.
