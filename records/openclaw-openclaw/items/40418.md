## Summary

Keep open. Current main has partial session-memory capture and bounded startup memory loading, but it does not implement the requested synthesis, durable-fact dedupe, full-transcript references, or compact successor-session hydration; the related synthesis PR was closed unmerged and the remaining design has real privacy and prompt-safety implications.

## What This Changes

Requested automated preservation and synthesis of useful session memory around `/new` so context can continue across resets.

## Best Possible Solution

Define one memory-owner lifecycle that preserves canonical transcript references, generates a small sanitized and deduplicated continuation artifact for durable facts, decisions, preferences, and open work, and hydrates startup through bounded untrusted context with clear retention controls.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives a high-confidence current-main baseline: OpenClaw saves recent raw excerpts and can load bounded recent memory at startup, but does not synthesize or dedupe continuation memory.

## Solution Assessment

No. The proposed JSON plus full Markdown sidecar workflow is not the best repository-aligned solution until maintainers choose the memory owner, storage shape, retention policy, sanitization boundary, and successor-session hydration contract.
