## Summary

Keep open: current main and v2026.5.12 provide raw reset-time session-memory capture plus bounded startup hydration, but synthesized continuation packages, transcript receipt links/anchors, and explicit retention/redaction/trust policy are still unresolved.

## What This Changes

Request automatic preservation, synthesis, and linked transcript receipts when `/new` or reset starts a new session.

## Best Possible Solution

Define an opt-in, bounded session-continuation design on the existing session-memory/startup-context path with sanitization, retention/redaction rules, transcript receipts, docs, and focused tests.

Not applicable. This is a feature request, and source inspection shows partial raw capture plus bounded hydration rather than a broken documented behavior to reproduce.

## Solution Assessment

Unclear as proposed. The maintainable direction is a compact, sanitized, opt-in continuation package with optional transcript receipts, not automatic raw transcript promotion into future sessions by default.
