## Summary

Keep open: current main and v2026.6.8 still provide mechanical reset-memory capture and bounded startup loading, but not the requested synthesized continuation package, dedupe, anchored transcript links, or start-of-session summary generation. The remaining work is product- and security-sensitive because it changes durable transcript retention and later model hydration.

## What This Changes

The issue requests automatic session memory preservation and synthesis around `/new`, including durable transcript export, summary generation, and links from summaries back to full conversations.

## Best Possible Solution

Define a maintainer-approved memory lifecycle contract first: opt-in continuation packages with curated summaries, transcript receipts, retention/redaction controls, untrusted historical-context framing, and coverage across `/new`, `/reset`, compaction, and stale rollover.

Not applicable: this is a feature/design request, not a broken existing behavior report. Source inspection confirms current main has partial raw capture, bounded startup loading, and compaction flushes, but not the requested synthesis workflow.

## Solution Assessment

No for the raw default-dump shape in the issue body: automatic `.session_memory_preserve.json` plus broad transcript export is not the safest maintainable default. A bounded, sanitized, opt-in continuation package with explicit retention, provenance, and trust boundaries is the safer path.
