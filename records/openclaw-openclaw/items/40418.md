## Summary

Keep open: current main and the latest release provide background raw session capture plus bounded startup hydration, but they still do not implement the requested synthesized continuation package, transcript receipt links/anchors, or explicit retention and sanitization contract. The related synthesis PR and reset-memory issues overlap this request, but they remain open and do not fully supersede the broader product/security decision.

## What This Changes

Request automatic memory preservation on `/new`, including transcript export, synthesized cross-session summaries, and links back to full transcripts.

## Best Possible Solution

Define an opt-in, bounded session-continuation design on the existing hook/startup-context path, coordinated with reset-time memory flush and raw-memory sanitization, including synthesis, transcript receipts, retention, redaction, docs, and focused tests.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection shows raw capture plus bounded hydration rather than a broken documented behavior to reproduce.

## Solution Assessment

Unclear as proposed. The maintainable direction is a compact, sanitized continuation package with optional transcript receipts, not automatic raw transcript promotion into future sessions by default.
