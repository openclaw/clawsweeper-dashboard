## Summary

Keep open: current main has mechanical session-memory capture on `/new` and `/reset` plus bounded startup hydration, but the requested synthesized continuation package, transcript receipts/anchors, retention policy, and safety model are still unresolved. ## What This Changes

Request automatic session memory preservation around `/new`, including summary synthesis, transcript links or receipts, and new-session hydration.

## Best Possible Solution

Define a bounded, sanitized continuation flow on the existing hook/startup-context path, coordinating synthesis, transcript receipt links, retention defaults, and sanitization with https://github.com/openclaw/openclaw/pull/50584, https://github.com/openclaw/openclaw/issues/69943, and https://github.com/openclaw/openclaw/issues/45608.
