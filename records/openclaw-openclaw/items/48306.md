## What This Changes

The issue asks OpenClaw to prevent macOS updater/appcast releases from advertising missing app artifacts that can leave the app and CLI/gateway on incompatible versions.

## Best Possible Solution

Land or replace a release-owner-approved post-publish guard that verifies required stable macOS assets and appcast state before users can hit a dead updater path.

## Reproduction Assessment

Yes for the release-artifact failure class, but not as a live current-latest app run. Old public releases still have zero macOS assets and 404 zip URLs, while current main still lacks the public verifier that would catch that state.

## Solution Assessment

Yes directionally: the focused verifier/CODEOWNERS guard is the narrow maintainable fix for the release-pipeline gap. This issue should stay open until https://github.com/openclaw/openclaw/pull/66213 is repaired and merged or replaced by an equivalent guard.
