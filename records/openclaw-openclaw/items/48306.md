## Summary

Keep open: the latest appcast and macOS release assets are currently healthy, and current main has app/gateway version mismatch visibility, but the public repository still lacks a merged post-publish verifier/CODEOWNERS guard for the missing-asset failure class and the focused repair PR remains open with review blockers.

## What This Changes

The issue asks OpenClaw to stop advertising missing macOS appcast assets and to prevent failed app updates from leaving the app and gateway in a noisy version-skew state.

## Best Possible Solution

Land or replace a release-owner-approved post-publish guard that verifies stable macOS assets, appcast reachability, and appcast ownership before users can be pointed at a missing zip.

## Reproduction Assessment

Yes for the release-artifact failure class, but not as a live current-latest app update. The old v2026.4.12 and v2026.3.13-1 release paths still show zero assets or 404 zip URLs, while current main still lacks a merged public verifier for that state.

## Solution Assessment

No close yet. The narrow maintainable direction is the open verifier/CODEOWNERS repair path, but https://github.com/openclaw/openclaw/pull/66213 remains unmerged and needs its review blockers addressed or a replacement landed.
