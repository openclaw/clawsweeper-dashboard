## Summary

Keep open: the latest appcast is healthy, but current main still lacks the public post-publish asset/appcast guard and the focused verifier PR remains open with review blockers.

## What This Changes

The issue asks for macOS updates to advertise only reachable signed app assets and to avoid noisy app/gateway version drift when the app cannot update.

## Best Possible Solution

Land or replace the post-publish macOS release guard so stable releases verify required assets and appcast state before users can hit a dead updater path.

## Reproduction Assessment

Yes for the release artifact failure class, but not as a live current-app run. Old affected releases still have zero macOS assets and 404 zip URLs, while current source still lacks the public guard that would catch that state.

## Solution Assessment

No close yet. The focused verifier direction is maintainable, but the open PR still needs its workflow fixes or an equivalent release-owner-approved replacement before this issue is resolved.
