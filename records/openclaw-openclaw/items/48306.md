## Summary

Keep open: the latest appcast is no longer advertising a dead macOS asset, but current main still lacks a public post-publish asset/appcast guard, and the focused verifier PR is still open with review blockers.

## What This Changes

The issue asks OpenClaw to prevent macOS appcast and release-asset mismatches that break Sparkle updates and leave the macOS app behind the CLI/gateway.

## Best Possible Solution

Land or replace the release guard so stable macOS releases verify assets and appcast before clients see them, while keeping the app/gateway version check as operator-facing defense-in-depth.

## Reproduction Assessment

Yes for the release artifact failure class, but not as a live current-app run: the old affected releases still have zero macOS assets and 404 zip URLs, while the current main appcast now points at a valid v2026.5.7 asset. Source inspection also shows the public post-publish guard has not landed.

## Solution Assessment

No close yet. The narrowest maintainable path is to repair and land the related asset-verification PR or an equivalent release-owner-approved guard; broader client fallback work should be a follow-up only if current builds still show bad UX.
