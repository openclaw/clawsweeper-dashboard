## Summary

Keep open: the latest appcast and v2026.5.7 macOS assets are healthy, and current main now exposes app/gateway version skew, but main still lacks a merged public post-publish verifier or root appcast CODEOWNERS guard for the missing-asset failure class; the focused repair PR remains open with review blockers.

## What This Changes

The issue reports macOS Sparkle appcast/release asset drift that can break in-app updates and leave the app and gateway on incompatible versions.

## Best Possible Solution

Land or replace a release-owner-approved post-publish guard that verifies stable macOS assets, appcast reachability, and appcast ownership before users can be pointed at a missing zip.

## Reproduction Assessment

Yes for the release-artifact failure class, but not as a live current-latest app update. The old v2026.4.12 and v2026.3.13-1 zip URLs still return 404, and current main still lacks the verifier that would prevent recurrence.

## Solution Assessment

No close yet. The narrow maintainable direction is to review, repair, or replace https://github.com/openclaw/openclaw/pull/66213 so the release pipeline enforces the documented macOS asset/appcast invariant.
