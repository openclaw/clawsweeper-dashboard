## Summary

## What This Changes

This PR flips `tools.exec.notifyOnExitEmptySuccess` to default true in exec setup/runtime, shortens the exit wake coalesce delay, and updates config help/type comments plus tests.

## Best Possible Solution

Land a rebased update or narrow replacement that makes empty-success exec completion notifications opt-out consistently while preserving `exec-event` wake metadata, updating generated/public docs, and adding changelog coverage.

Yes, source-level. On current main, omitted config stays false in `createExecTool` and `runExecProcess`, `maybeNotifyOnExit` suppresses completed empty-output runs, and the no-op test matrix asserts the default case queues no event.

## Solution Assessment
