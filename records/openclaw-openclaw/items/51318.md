## Summary

Keep open. Current main still does not provide a per-invocation skip-bootstrap option for `openclaw agents add`, but the PR is not merge-ready because the proposed CLI watchlist change would make `--skip-bootstrap` force non-interactive mode and block the interactive wizard path without `--workspace`.

## What This Changes

The PR adds a `--skip-bootstrap` option to `openclaw agents add`, plumbs it into the agents-add command and tests, and also changes image-generation runtime tests to use the active plugin registry seam.

## Best Possible Solution

Revise the PR so `--skip-bootstrap` is a behavior modifier that works in both interactive and non-interactive `agents add` flows: keep the command plumbing and OR-with-config behavior, exclude it from explicit non-interactive flag detection, add CLI-level coverage for the wizard path, update `docs/cli/agents.md`, and split or drop the unrelated image-generation test change.

## Reproduction Assessment

Yes. The source-level reproduction is `openclaw agents add myagent --skip-bootstrap` on the PR branch: the added watchlist entry makes `hasFlags=true`, and current `agentsAddCommand` treats that as non-interactive mode that exits when `--workspace` is absent.

## Solution Assessment

No. The intended feature is narrow and maintainable, but the current patch chooses the wrong non-interactive detection boundary; the safer fix is to forward `skipBootstrap` without making it a flag that forces non-interactive mode.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

- **[P2] Keep --skip-bootstrap out of non-interactive detection:** `src/cli/program/register.agent.ts:192`

Remove this option from the watchlist and add CLI-level coverage for that path.
