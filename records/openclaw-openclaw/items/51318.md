## Summary

Keep open: current main still lacks per-invocation skip-bootstrap support, but this PR as written breaks the interactive wizard path and lacks real behavior proof, so it needs updates rather than cleanup close.

## What This Changes

The PR adds a `--skip-bootstrap` option to `openclaw agents add`, threads it into bootstrap suppression, adds focused tests, and updates image-generation runtime tests.

## Best Possible Solution

Land a rebased version that treats `--skip-bootstrap` as a per-invocation modifier usable in both interactive and non-interactive `agents add`, while preserving existing config defaults and current optional-bootstrap behavior.

## Reproduction Assessment

Yes for the PR review finding. The diff adds `skipBootstrap` to the explicit-option watchlist, and current `agentsAddCommand` turns any `hasFlags` value into non-interactive mode that exits when `--workspace` is absent.

## Solution Assessment

No, not as currently written. The command plumbing is the right narrow direction, but `--skip-bootstrap` should be forwarded without making it a non-interactive trigger, and the user-facing docs need the new exception.
