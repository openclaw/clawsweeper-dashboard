## Summary

Keep open: current main still lacks a per-invocation `agents add --skip-bootstrap` option, so the PR is not obsolete, but the current patch needs changes before merge because it makes the new flag force non-interactive mode and omits required user-facing docs/changelog coverage.

## What This Changes

The PR adds an `--skip-bootstrap` option to `openclaw agents add`, threads it into agent-add bootstrap setup, adds focused CLI/command tests, and updates image-generation runtime tests to use active plugin registry state.

## Best Possible Solution

Land a rebased version that treats `--skip-bootstrap` as a per-invocation modifier usable in both interactive and non-interactive `agents add`, while preserving existing config defaults and optional-bootstrap behavior.

## Reproduction Assessment

Yes. The PR source adds `skipBootstrap` to `hasExplicitOptions`, and `agentsAddCommand` turns any `hasFlags` value into non-interactive mode that exits when `--workspace` is absent.

## Solution Assessment

No, not as currently written. The command plumbing is the right narrow direction, but `--skip-bootstrap` must stay out of non-interactive detection and the user-facing flag needs docs/changelog coverage.
