## Summary

Keep open: current main still does not have a per-invocation `agents add --skip-bootstrap` flag, so the useful feature is not obsolete. The PR is not merge-ready because the flag is incorrectly treated as a non-interactive trigger, public CLI docs would become inaccurate, and the external contributor has only test-based proof rather than real behavior proof.

## What This Changes

The PR adds a `--skip-bootstrap` option to `openclaw agents add`, threads it into workspace bootstrap suppression, adds focused agents tests, and updates image-generation runtime tests.

## Best Possible Solution

Land a rebased version that treats `--skip-bootstrap` as a per-invocation modifier valid in both wizard and non-interactive `agents add`, with CLI docs and regression coverage that preserve current bootstrap config behavior.

## Reproduction Assessment

Yes for the PR review finding from source inspection: the PR adds `skipBootstrap` to the explicit-option watchlist, and current `agentsAddCommand` turns any `hasFlags` value into non-interactive mode that requires `--workspace`. I did not run the branch because this is a read-only review.

## Solution Assessment

No, not as currently written. The narrow command plumbing is appropriate, but `--skip-bootstrap` should be forwarded without becoming a non-interactive trigger, and the CLI docs should describe that exception.
