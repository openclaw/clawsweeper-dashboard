## Summary

Keep this PR open: current main still lacks the requested per-invocation `agents add --skip-bootstrap` flag, but the branch has a source-proven CLI regression and missing contributor real behavior proof before it is mergeable.

## What This Changes

The PR adds a per-invocation `--skip-bootstrap` option to `openclaw agents add`, threads it into workspace bootstrap suppression, adds focused agents tests, and updates image-generation runtime tests.

## Best Possible Solution

Land a rebased version that treats `--skip-bootstrap` as a per-invocation modifier valid in both wizard and non-interactive `agents add`, preserves current bootstrap option plumbing, documents the CLI semantics, and includes real behavior proof.

## Reproduction Assessment

Yes for the PR-introduced blocker from source inspection: the branch adds `skipBootstrap` to the explicit-option watchlist, and `agentsAddCommand` treats any `hasFlags` value as non-interactive mode requiring `--workspace`. I did not run the PR branch because this review is read-only.

## Solution Assessment

No, not as currently written. The narrow command plumbing is appropriate, but `--skip-bootstrap` should be forwarded without becoming a non-interactive trigger, and the CLI docs should describe that exception.
