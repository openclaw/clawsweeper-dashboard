## Summary

Keep open: the requested per-invocation `agents add` flag is not implemented on current main and is not superseded by the merged onboard flag, but this PR cannot merge as-is because it makes `--skip-bootstrap` force non-interactive mode, leaves the CLI docs inconsistent, is conflicting, and has only test/mock proof.

## What This Changes

The PR adds `openclaw agents add --skip-bootstrap`, forwards it through the agents add command, and adds CLI/command tests plus an unrelated image-generation test adjustment.

## Best Possible Solution

Land a rebased version that adds the agents-add flag as a wizard-safe modifier, preserves current bootstrap options, documents the exception, and shows redacted real CLI behavior.

## Reproduction Assessment

Yes for the PR-introduced blocker: source inspection shows the branch adds `skipBootstrap` to the explicit-option watchlist, and `agentsAddCommand` treats `hasFlags` as non-interactive mode requiring `--workspace`. I did not run the branch because this is a read-only review.

## Solution Assessment

No, not as currently written. The forwarding logic is the right direction, but `--skip-bootstrap` should not trigger non-interactive mode, and the branch needs docs, rebase work, and real behavior proof.
