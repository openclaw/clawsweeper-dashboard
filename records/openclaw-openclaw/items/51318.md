## Summary

Keep open: current main still does not provide a per-invocation `agents add --skip-bootstrap` flag, but this PR is not merge-ready because the new flag is incorrectly treated as a non-interactive trigger and contributor real behavior proof is still missing.

## What This Changes

The PR adds `openclaw agents add --skip-bootstrap`, forwards it into workspace bootstrap suppression, adds agents command tests, and changes image-generation runtime tests.

## Best Possible Solution

Land a rebased version that treats `--skip-bootstrap` as a per-invocation modifier valid in both wizard and non-interactive `agents add`, preserves current bootstrap option plumbing, updates the agents CLI docs, and includes real behavior proof.

## Reproduction Assessment

Yes for the PR-introduced blocker from source inspection: the branch adds `skipBootstrap` to the explicit-option watchlist, and `agentsAddCommand` treats any `hasFlags` value as non-interactive mode requiring `--workspace`. I did not run the PR branch because this review is read-only.

## Solution Assessment

No, not as currently written. The command plumbing is the right direction, but `--skip-bootstrap` should be forwarded without becoming a non-interactive trigger, the docs need the new semantics, and the branch needs real behavior proof plus a rebase.
