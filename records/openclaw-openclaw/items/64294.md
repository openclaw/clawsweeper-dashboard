## Summary

Keep this PR open. Current main still lacks the requested Plugin SDK LLM completion surface, and the latest PR head fixes several earlier review gaps, but the patch still needs maintainer API/security review because it exposes model calls to plugins with spoofable audit metadata and default model/agent override authority.

## What This Changes

The PR adds `api.runtime.llm.complete()` and PluginLlmComplete SDK types, injects an LLM completion capability into context-engine runtime contexts, documents/changelogs the SDK surface, updates the API baseline, and includes broad generated/formatting churn.

## Best Possible Solution

Ship this as a session/request-scoped LLM helper with non-spoofable runtime attribution, explicit plugin authority and cost controls, clean generated assets, updated docs/changelog/API baseline, and then close the open feature tracker after merge.

## Reproduction Assessment

Yes for the PR review findings: the override authority and caller attribution issues reproduce directly from static inspection of the PR head, and the unrelated generated asset churn is visible in the PR file list. This is a PR review rather than an end-user runtime reproduction.

## Solution Assessment

No. The feature direction is plausible, but this patch is not the safest final form until model/agent overrides are explicitly governed, audit caller identity is bound by the runtime rather than caller input, and unrelated generated JavaScript churn is removed or separately justified.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Bind audit caller to the runtime scope:** `src/plugins/runtime/runtime-llm.runtime.ts:61-69`
  - body: `resolveScopedCaller` trusts `params.caller` before the runtime authority or plugin request scope, so a plugin can attribute completions to another plugin or even `host` in the new audit metadata. Use the scoped plugin/context-engine caller as the authority and treat request caller fields as optional descriptive metadata at most.
  - confidence: 0.89
- **[P2] Gate plugin model and agent overrides:** `src/plugins/runtime/index.ts:106-112`
  - body: The default plugin facade enables both agent and model overrides for every plugin completion. That lets a loaded plugin choose another agent's credentials or an arbitrary configured model without the opt-in policy that `api.runtime.subagent` already requires for model overrides, so this should be tied to explicit plugin authority or removed from the default surface.
  - confidence: 0.85
- **[P3] Drop the unrelated viewer bundle churn:** `extensions/diffs/assets/viewer-runtime.js:1`
  - body: This PR rewrites the diffs viewer runtime asset by over twenty thousand changed lines even though the functional change is the Plugin SDK LLM API. Please remove this generated asset change or split it into a separately justified PR so the LLM capability review does not include an unrelated browser code bundle.
  - confidence: 0.81
