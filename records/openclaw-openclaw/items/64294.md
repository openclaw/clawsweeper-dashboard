## Summary

This PR should stay open: current main does not yet provide the requested plugin SDK LLM runtime surface, and the proposed public inference capability still has correctness and security/API-boundary blockers before merge.

## What This Changes

The PR adds `api.runtime.llm.complete()` and SDK types, then injects the LLM capability into context-engine compact, afterTurn, and maintain runtime contexts.

## Best Possible Solution

Ship this SDK inference surface only after maintainer API/security review settles plugin authority, session binding, cost controls, docs/changelog, tests, and the generated SDK API baseline.

## Reproduction Assessment

Yes for the PR review findings: source inspection of the PR patch shows the ambient `loadConfig()` call, default-agent fallback, and missing generated SDK baseline. This is a PR review of a new feature rather than an end-user bug reproduction.

## Solution Assessment

No. The direction is plausible, but this implementation is not the safest merge shape because the LLM facade should be request/session-bound and explicitly reviewed for plugin authority, credential use, and cost/audit behavior before becoming a public SDK contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Use request-scoped config for completions:** `src/plugins/runtime/runtime-llm.runtime.ts:20`
  - body: `complete()` reloads config with `loadConfig()` inside the plugin runtime module, so the new facade can bypass the active runtime's config snapshot and conflicts with the repo's runtime/config boundary. Pass the current runtime config or an explicit process-boundary accessor into the facade instead of ambiently loading config at call time.
  - confidence: 0.9
- **[P2] Bind completions to the active session agent:** `src/plugins/runtime/runtime-llm.runtime.ts:21`
  - body: When `agentId` is omitted, context-engine hooks in a non-default session will use `resolveDefaultAgentId(cfg)`, which can route completions through another agent's model and auth profile. The injected capability should be bound to the active session agent or require an explicit agent selection.
  - confidence: 0.86
- **[P2] Commit the SDK API baseline update:** `src/plugin-sdk/index.ts:63-67`
  - body: Exporting `PluginLlmComplete*` changes the public plugin SDK API, but the PR does not update the generated plugin SDK API baseline. `pnpm plugin-sdk:api:check` is expected to report drift until the baseline hash is regenerated and committed.
  - confidence: 0.82
- **[P3] Document and changelog the new SDK surface:** `src/plugin-sdk/index.ts:63-67`
  - body: This adds a user-facing plugin SDK capability, but the PR does not add SDK docs or a `CHANGELOG.md` entry. Plugin authors need the new contract documented and release notes need the feature recorded before merge.
  - confidence: 0.78
