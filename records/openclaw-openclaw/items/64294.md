## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Preserve the bound LLM capability during maintenance:** `src/agents/pi-embedded-runner/context-engine-maintenance.ts:290-296`
  - body: `buildContextEngineMaintenanceRuntimeContext` spreads a freshly resolved `llm` over `params.runtimeContext`, but `compact.queued.ts` already built a session-bound runtime context with `agentId: sessionAgentId` and then calls `runContextEngineMaintenance` without passing that agent id. For legacy or otherwise non-canonical session keys, this overwrites the working capability with one that rejects as not bound to an active session agent, so engines can lose `runtimeContext.llm` in `maintain()` immediately after compaction. Reuse an existing bound `runtimeContext.llm` or thread the active agent id through that maintenance call.
  - confidence: 0.87
