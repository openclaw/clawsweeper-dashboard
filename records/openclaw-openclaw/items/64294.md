## Summary

Keep open. Current main still lacks the proposed plugin/runtime LLM capability, and the patch has blocking API/runtime issues around config snapshot use, active-agent binding, and public SDK baseline drift. The linked #45865 design discussion is still open, so this belongs in maintainer review rather than cleanup closure.

## What This Changes

The branch adds an `api.runtime.llm.complete()` facade, public `PluginLlmComplete*` SDK types, a `runtime-llm.runtime.ts` implementation, context-engine `llm` injection for lifecycle hooks, and related test/mocking updates.

## Best Possible Solution

Ship an additive LLM runtime seam only after it is bound to the active agent/session and runtime config snapshot, has explicit credential/cost controls acceptable to maintainers, includes docs and SDK baseline updates, and has tests for default and non-default agent model/auth resolution.

## Reproduction Assessment

Yes for the review findings: static inspection of the supplied PR diff plus current-main guard/docs is enough to reproduce the config-boundary and SDK-baseline failures, and the multi-agent bug follows directly from the default-agent fallback in the new facade. This is a PR review rather than a user bug reproduction.

## Solution Assessment

No. The current patch is not the safest way to expose this capability because it reloads config from inside a runtime module and does not bind session-injected completions to the active agent; a safer path is a request/session-bound runtime facade using the existing runtime config snapshot and explicit maintainer-approved controls.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Use the runtime config snapshot for completions:** `src/plugins/runtime/runtime-llm.runtime.ts:20`
  - body: `runtime-llm.runtime.ts` calls `loadConfig()` directly under `src/plugins/runtime`, but this tree is covered by the deprecated-internal-config-api guard and long-lived runtime modules must use a passed/runtime config snapshot. This will fail the config-boundary check and can bypass the active runtime config view.
  - confidence: 0.9
- **[P2] Bind completions to the active agent:** `src/plugins/runtime/runtime-llm.runtime.ts:21`
  - body: When callers omit `agentId`, this facade falls back to the config default agent. Because the same facade is injected into context-engine hooks for active sessions, non-default agent sessions can silently run completions with another agent's model/auth profile unless the capability is bound to the session agent or requires explicit selection.
  - confidence: 0.86
- **[P2] Commit the SDK API baseline update:** `src/plugin-sdk/index.ts:63-67`
  - body: Exporting `PluginLlmComplete*` from the public SDK changes the generated API baseline, but the PR file list does not include the updated `docs/.generated/plugin-sdk-api-baseline.sha256`. `pnpm plugin-sdk:api:check` will report drift until the baseline hash is regenerated and committed.
  - confidence: 0.82
