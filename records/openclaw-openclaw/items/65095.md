## Summary

Keep this open. Current main has partial v3-v4 protocol compatibility and a restart-probe exception, but ordinary older clients whose range omits the current protocol are still rejected and the macOS app still blocks older gateway packages, so the remote/offline version-skew problem remains a product and protocol policy item.

## What This Changes

The issue requests a grace period, compatibility mode, stable/LTS channel, or offline deployment path so older OpenClaw clients and gateways keep working after new releases.

## Best Possible Solution

Define a supported client/app/gateway skew policy, implement negotiated compatibility for safe protocol ranges, and document remote/offline expectations with any bypass treated as a narrow break-glass mode.

## Reproduction Assessment

Yes. A high-confidence source-level path is to connect to a current v4 Gateway as a non-probe client advertising only protocol v3; the handler rejects it, and the macOS app also marks older installed gateways incompatible.

## Solution Assessment

No. A broad skipVersionCheck flag is not the best standalone solution; explicit protocol negotiation plus a maintainer-approved skew policy and docs is safer.
