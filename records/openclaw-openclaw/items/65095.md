## Summary

Keep this open. Current main partially widens compatibility for newer clients and restart probes, but ordinary older clients are still rejected by a newer Gateway and the macOS app still treats older gateway packages as incompatible, so the remote/offline version-skew problem remains a product and protocol policy item.

## What This Changes

Request a compatibility grace period, compatibility/offline mode, or stable-channel policy so older OpenClaw clients and gateways remain usable across releases.

## Best Possible Solution

Define a maintainer-approved Gateway compatibility policy for app, client, and gateway skew; implement explicit negotiated support for safe version ranges; and document remote/offline deployment expectations with any bypass treated as a narrow break-glass mode.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to connect to a current v4 Gateway as a non-probe client advertising only protocol v3; the handler rejects it as protocol mismatch, and macOS also marks an installed gateway older than the app-required version as incompatible.

## Solution Assessment

No, a broad skipVersionCheck flag is not the best standalone solution. The maintainable direction is explicit protocol negotiation plus a documented skew policy; the stable-channel portion is already covered by https://github.com/openclaw/openclaw/issues/65081.
