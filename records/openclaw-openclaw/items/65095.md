## Summary

Keep this open. Current main has partial protocol-range compatibility for v3/v4 clients and restart probes, but ordinary older clients can still be rejected by a newer Gateway and the macOS app still blocks older gateway packages, so the remote/offline version-skew problem remains a product and protocol policy item.

## What This Changes

Request a grace period, compatibility mode, stable channel, or offline guidance so OpenClaw remains usable across app/client/gateway version skew.

## Best Possible Solution

Define the supported app/client/gateway skew policy, implement explicit negotiated compatibility for safe ranges, and document remote/offline expectations with any bypass treated as a narrow break-glass mode.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to connect to a current v4 Gateway as a non-probe client advertising only protocol v3; the handler rejects it, and the macOS app also marks older installed gateways incompatible.

## Solution Assessment

No. A broad skipVersionCheck flag is not the best standalone solution; the safer path is explicit protocol negotiation plus a maintainer-approved skew policy and docs.
