## Summary

Keep open: the reported overwrite is still source-reproducible on current main. Codex app-server emits reasoning deltas, while Discord formats each callback payload with a reasoning/thinking display prefix before the draft-preview merge logic treats that prefixed payload as a full snapshot and replaces the previous progress line.

## What This Changes

Fix Discord progress-mode reasoning rendering so delta-style reasoning chunks accumulate instead of each formatted chunk replacing the prior progress line.

## Best Possible Solution

Merge raw reasoning semantics before display formatting, or carry explicit delta/snapshot metadata, then format once for Discord progress display while preserving final-answer delivery and tool-progress rows.

## Solution Assessment

Yes, the best fix is the narrow semantic split described in the report: merge raw delta/snapshot reasoning first, or pass explicit metadata, and only then apply Discord display formatting. A fix should add regression coverage for both true deltas and full snapshots.
