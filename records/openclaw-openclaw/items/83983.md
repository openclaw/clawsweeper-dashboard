## Summary

Keep open: current main still has the reported source-level Discord progress bug where Codex reasoning deltas are display-formatted before the draft merge, causing prefix-based snapshot detection to replace prior visible reasoning text.

## What This Changes

The issue requests that Discord progress-mode reasoning deltas from the Codex runtime accumulate into a readable progress line instead of overwriting earlier chunks.

## Best Possible Solution

Keep raw reasoning delta versus snapshot semantics separate from display formatting, merge the stable reasoning value first, then format the Discord progress line once with focused delta and snapshot regression coverage.

## Reproduction Assessment

Yes, at source level: current main sends Codex reasoning callbacks as deltas, formats each delta with a Thinking/Reasoning prefix, and then treats that formatted text as a full snapshot. I did not run a live Discord edit-history reproduction.

## Solution Assessment

Yes: the best narrow fix is to merge raw reasoning delta/snapshot data before display formatting, or carry explicit metadata, rather than inferring semantics from the display prefix.
