## Summary

Keep open: current main still has the reported source-level Discord progress bug. Codex app-server forwards reasoning as deltas, Discord formats each delta as a Thinking block before merging, and the merge logic treats that prefix as a full snapshot replacement.

## What This Changes

The issue reports that Discord progress-mode reasoning deltas are formatted as full reasoning snapshots and overwrite earlier visible reasoning chunks.

## Best Possible Solution

Update Discord reasoning progress to merge raw delta/snapshot semantics before display formatting, then format the stable progress line once while preserving true snapshot replacement, final replies, and tool progress rows.

## Solution Assessment

Yes: the narrow maintainable fix is to separate raw reasoning payload semantics from display formatting, with regression coverage for fragment deltas and true full snapshots.
