## Summary

Keep open: current main still has the reported Discord progress-mode reasoning overwrite path, and https://github.com/openclaw/openclaw/pull/84202 is an open linked implementation candidate rather than a reason to close the issue before merge.

## What This Changes

The issue asks Discord progress-mode reasoning deltas to accumulate or coalesce instead of replacing prior reasoning chunks when Codex app-server streams reasoning text.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/84202 or an equivalent narrow fix that merges raw reasoning delta/snapshot semantics before applying Discord display formatting, with regression coverage for both delta accumulation and true snapshot replacement.

## Reproduction Assessment

Yes, at source level: current main still sends Codex reasoning as deltas, formats each delta for Discord display, and then uses display-prefix snapshot detection that can replace prior reasoning progress text. I did not establish a live Discord edit-history reproduction in this read-only review.

## Solution Assessment

Yes: the narrow maintainable solution is to keep raw reasoning delta/snapshot semantics separate from Discord display formatting, then format the merged progress line once and preserve true snapshot replacement behavior.
