## Summary

Keep open: current main still has the source-level Discord progress rendering bug. Codex app-server emits reasoning deltas, Discord formats each delta before merging, and the draft-preview merge still treats formatted reasoning/thinking text as a full snapshot replacement.

## What This Changes

Discord progress-mode reasoning deltas should accumulate into one readable progress line instead of each formatted delta overwriting the prior chunk.

## Best Possible Solution

Keep raw reasoning delta/snapshot semantics separate from display formatting, then format the merged Discord progress line once while preserving true snapshot replacement and tool-progress behavior.
