## Label Justifications

- `P2`: This is a focused Discord progress-mode rendering bug with visible reasoning-progress loss but no evidence of final-answer loss or runtime unavailability.
- `impact:message-loss`: Earlier reasoning chunks are overwritten in the visible Discord progress draft, effectively dropping prior progress message content.

## Summary

Keep open: current main still has the source-level Discord progress rendering bug. Codex reasoning is forwarded as deltas, Discord formats each delta before merging, and the merge path treats the display prefix as a full reasoning snapshot, so earlier visible reasoning chunks can be replaced.

## What This Changes

The issue asks Discord progress-mode reasoning deltas from the Codex app-server runtime to accumulate or coalesce instead of overwriting prior progress text.

## Best Possible Solution

Keep raw reasoning delta/snapshot semantics separate from display formatting, then format the stable Discord progress line once while preserving final-answer and tool-progress behavior.

## Reproduction Assessment

Yes, at source level: the current main code path sends Codex reasoning as deltas, formats each delta with a Thinking/Reasoning prefix, and then treats that prefix as a snapshot marker during Discord progress merging. I did not run a live Discord guild edit-history reproduction.

## Solution Assessment

Yes: the narrow maintainable fix is to merge raw reasoning payload semantics before display formatting, with coverage for both fragment deltas and true full snapshots.
