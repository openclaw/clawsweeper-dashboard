## Summary

This PR still needs review work before merge: the latest head fixes the earlier exact o-series eligibility gap, but the default Copilot web-search routing change remains undocumented and changes auto-mode behavior from managed provider discovery to GitHub Copilot hosted search.

## What This Changes

Adds a GitHub Copilot provider-local native web-search wrapper and tests that inject a Responses `web_search` tool for eligible Copilot OpenAI-family models.

## Best Possible Solution

Keep the provider-owned Copilot native search path only after the default hosted-search behavior and opt-out are documented, and consider sharing the duplicated native payload patch helper with the existing OpenAI implementation before merge.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature PR. Source inspection shows current main lacks Copilot native-search injection, and the PR body supplies terminal/live proof for `github-copilot/gpt-5.4`.

## Solution Assessment

Partly: the provider-local wrapper is the right ownership boundary, and the upstream Responses contract supports the tool shape, but the default routing change needs docs and maintainer acceptance before it is the best merge path.
