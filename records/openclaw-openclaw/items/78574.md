## Summary

Keep open: the feature is directionally useful, but the patch is not merge-ready because the eligibility check is broader than the stated Copilot GPT scope, real OpenClaw after-fix proof is still insufficient, and the user-facing feature lacks a changelog entry.

## What This Changes

The PR adds a GitHub Copilot provider-local native web search payload wrapper and unit tests, then wraps the Copilot Responses stream to replace managed `web_search` with the Responses native tool when enabled.

## Best Possible Solution

Keep the provider-local approach, but gate native search to verified Copilot GPT/OpenAI-family models, add the required release note, and require after-fix OpenClaw output before merge.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature PR. Source inspection does provide high-confidence proof of the PR regression path because Copilot Gemini catalog entries inherit `openai-responses` and would pass the new provider/API-only gate.

## Solution Assessment

No: a provider-local wrapper is a maintainable direction, but this implementation should be narrowed to GPT/OpenAI-family Copilot models and backed by real OpenClaw output before merge.
