Keep open: the PR targets a valid Bun bootstrap regression, but the branch weakens the direct-vs-transitive module-miss guard and still lacks after-fix real Bun behavior proof.

## What This Changes

The PR adds Bun `err.specifier` and relative error-message matching to `openclaw.mjs` so optional `warning-filter` module misses can be ignored under Bun.

## Best Possible Solution

Land a narrow launcher fix that recognizes Bun direct optional-import misses by matching specifier plus launcher referrer, with focused regression coverage and redacted live Bun CLI proof.

## Reproduction Assessment

Yes, source-backed: the linked bug gives `bun openclaw.mjs --version`, current main still rethrows Bun-shaped optional `warning-filter` misses, and Bun source exposes `specifier` plus `referrer`. I did not run Bun in this read-only review.

## Solution Assessment

No, not as written. The fix direction is right, but the Bun-specific branches should require the launcher `referrer` instead of accepting any matching relative specifier or message text.
