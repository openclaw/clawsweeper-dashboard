## Summary

Keep open: current main still lacks Bun-shaped direct module-miss handling, but this PR is not merge-ready because the new matching is not scoped to Bun's `referrer` and the external PR still lacks after-fix real Bun runtime proof.

## What This Changes

The PR changes `openclaw.mjs` so launcher bootstrap treats Bun-shaped `ERR_MODULE_NOT_FOUND` errors for the optional `warning-filter` import as direct misses.

## Best Possible Solution

Land a narrow launcher fix that recognizes Bun direct optional-import misses by matching both `specifier` and the launcher `referrer`, with focused regression coverage and redacted live Bun CLI proof.

## Reproduction Assessment

Yes, source-backed: the linked bug report gives `bun openclaw.mjs --version`, current main still rethrows Bun-shaped optional warning-filter misses, and Bun source exposes `specifier` and `referrer`. I did not run Bun in this read-only review, so this is source-reproducible rather than locally reproduced.

## Solution Assessment

No, not as written. The direction is right, but the implementation should require Bun's `referrer` to identify a direct launcher miss instead of accepting any matching relative specifier or message text.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Check Bun referrer before swallowing direct misses:** `openclaw.mjs:63-75`
