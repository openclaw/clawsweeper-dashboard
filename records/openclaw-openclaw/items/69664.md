## Summary

Keep this PR open: it targets a real Bun launcher regression, but the current patch still widens direct-miss detection without checking Bun's `referrer`, and the PR still lacks after-fix Bun runtime proof.

## What This Changes

The PR extends `openclaw.mjs` direct module-not-found detection to treat Bun `err.specifier` and relative-message matches as missing launcher bootstrap artifacts.

## Best Possible Solution

Land a narrow launcher fix that recognizes Bun direct optional-import misses by matching both specifier and launcher referrer, with focused regression coverage and redacted live Bun CLI proof.

## Reproduction Assessment

Yes, source-backed: the linked report gives `bun openclaw.mjs --version`, and current `main` still rethrows Bun-shaped optional `warning-filter` misses because the helper lacks Bun `specifier` and `referrer` handling. I did not run Bun in this read-only review.

## Solution Assessment

No, not as written. The fix direction is right, but the Bun-specific branches should require the launcher `referrer` instead of accepting any matching relative specifier or message text.
