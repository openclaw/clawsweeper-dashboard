## Summary

Keep open: current main and the latest release still lack Bun-shaped direct-miss handling, but this PR is not ready because it accepts Bun `specifier` and relative-message matches without checking `referrer`, and it has no attached real Bun runtime proof.

## What This Changes

The PR updates `openclaw.mjs` to recognize Bun-shaped module-not-found errors for the optional warning-filter bootstrap import.

## Best Possible Solution

Land a narrow launcher fix that recognizes Bun direct optional-import misses by matching both `specifier` and the launcher `referrer`, with focused regression coverage and redacted live Bun CLI proof.

## Reproduction Assessment

Yes, source-backed. The linked report gives concrete `bun openclaw.mjs --version` steps, current main still rethrows Bun-shaped optional warning-filter misses, and Bun source shows the relevant `specifier` and `referrer` error fields.

## Solution Assessment

No, not as written. The fix should use Bun's `referrer` as well as `specifier` so the helper continues to swallow only direct launcher import misses instead of same-specifier transitive failures.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Check Bun referrer before swallowing direct misses:** `openclaw.mjs:63-75`
  - body: `isDirectModuleNotFoundError` is the guard that keeps transitive import failures from being treated as missing launcher output.
