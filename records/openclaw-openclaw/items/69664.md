## What This Changes

The PR updates `openclaw.mjs` module-not-found detection to recognize Bun `err.specifier` and relative-message matches for missing optional `warning-filter` imports.

## Best Possible Solution

Land a narrow launcher fix that recognizes Bun direct optional-import misses by matching both specifier and launcher referrer, with focused regression coverage and redacted live Bun CLI proof.

## Reproduction Assessment

Yes, source-backed: the linked bug gives `bun openclaw.mjs --version`, and current main still rethrows Bun-shaped optional `warning-filter` misses because the helper lacks Bun `specifier` and `referrer` handling. I did not run Bun in this read-only review.

## Solution Assessment

No, not as written. The fix direction is right, but the Bun-specific branches should require the launcher `referrer` instead of accepting any matching relative specifier or message text.
