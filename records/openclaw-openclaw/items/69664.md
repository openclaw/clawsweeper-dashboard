## Summary

Keep this PR open: it targets a real Bun bootstrap regression that current main still has, but the patch should be tightened so Bun-shaped errors are still proven to be direct launcher import misses, and it needs real after-fix Bun proof before merge.

## What This Changes

The PR updates `openclaw.mjs` so the launcher treats Bun-shaped `ERR_MODULE_NOT_FOUND` errors with `err.specifier` or relative-specifier messages as direct optional import misses.

## Best Possible Solution

Land a narrow launcher fix that recognizes Bun direct optional-import misses by matching both Bun `specifier` and the launcher `referrer`, with focused regression coverage or live Bun proof.

## Reproduction Assessment

Yes. The linked report gives concrete `bun openclaw.mjs --version` steps, and current main still has the Node-only direct-miss helper on the optional warning-filter bootstrap path; I did not run Bun locally because it is not installed in this checkout.

## Solution Assessment

No, not as written. The fix should use Bun's `referrer` as well as `specifier` so the helper continues to swallow only direct launcher import misses instead of same-specifier transitive failures.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Check Bun referrer before swallowing direct misses:** `openclaw.mjs:63`
  - body: `isDirectModuleNotFoundError` is specifically meant to avoid masking transitive import failures. Bun's `ResolveMessage` also carries `referrer`, but this branch accepts any `ERR_MODULE_NOT_FOUND` whose `specifier` string matches, so a nested module that happens to import the same relative string would be treated as a direct launcher miss. Please constrain the Bun path, and the relative-message fallback, to the launcher referrer before returning `true`.
  - confidence: 0.78
