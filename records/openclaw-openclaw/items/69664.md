The PR updates `openclaw.mjs` so `isDirectModuleNotFoundError` recognizes Bun-shaped `ERR_MODULE_NOT_FOUND` errors through `err.specifier` and relative-specifier message matching.

## Best Possible Solution

Land this narrow launcher fix after adding the required changelog entry and refreshing validation; ideally add a focused regression test for Bun-shaped direct optional import misses while preserving transitive-miss behavior.
