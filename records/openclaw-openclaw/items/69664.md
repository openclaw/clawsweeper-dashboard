## Summary

Keep open: the underlying Bun bootstrap bug is valid and still not fixed on current main, but this external PR lacks real Bun runtime proof and broadens direct-miss detection without checking Bun's `referrer`, so it needs changes before merge.

## What This Changes

The PR updates `openclaw.mjs` so `isDirectModuleNotFoundError` treats Bun `ERR_MODULE_NOT_FOUND` errors with `err.specifier` or relative-specifier messages as direct misses.

## Best Possible Solution

Land a narrow launcher fix that matches Bun `specifier` and launcher `referrer`, keeps the transitive-miss guard intact, adds focused regression coverage, and includes redacted live Bun CLI proof.

## Reproduction Assessment

Yes, source-backed. The linked report at https://github.com/openclaw/openclaw/issues/69589 gives concrete `bun openclaw.mjs --version` steps, current main still lacks Bun-shaped direct-miss handling, and upstream Bun source shows the relevant `specifier` and `referrer` fields.

## Solution Assessment

No, not as written. The direction is right, but the Bun branch should require `referrer` to point at the launcher so direct optional-import misses are swallowed without hiding transitive failures.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Check Bun referrer before swallowing direct misses:** `openclaw.mjs:63-75`
  - body: `isDirectModuleNotFoundError` is the guard that keeps transitive import failures from being treated as missing launcher output.
