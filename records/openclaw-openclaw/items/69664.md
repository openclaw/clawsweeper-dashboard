## Summary

This PR addresses a real Bun source-checkout launcher regression, but it is not merge-ready because the Bun-specific match is not scoped to Bun's `referrer` and the PR still lacks after-fix Bun CLI proof.

## What This Changes

The PR updates `openclaw.mjs` to treat Bun `ERR_MODULE_NOT_FOUND` errors with a matching `specifier` or relative-specifier message as direct launcher missing-module misses.

## Best Possible Solution

Land a narrow launcher fix that recognizes Bun direct optional-import misses by matching both specifier and launcher referrer, with regression coverage and redacted live Bun CLI proof.

## Reproduction Assessment

Yes, source-backed: the linked report gives `bun openclaw.mjs --version`, upstream Bun exposes `specifier` and `referrer`, and current `main` only checks Node-style `url` or absolute-path messages. I did not run Bun in this read-only review.

## Solution Assessment

No, not as written. The fix direction is right, but the Bun-specific branches should require the launcher `referrer` instead of accepting any matching relative specifier or message text.
