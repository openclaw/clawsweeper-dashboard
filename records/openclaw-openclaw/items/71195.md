## Summary

Keep open: current main and v2026.6.10 still leave native macOS Talk on the local speech-recognition -> chat.send -> talk.speak/TTS loop, while realtime Talk exists for browser and gateway-relay clients. The remaining work is a maintainer-sponsored macOS client integration, not a new gateway voice stack.

## What This Changes

The issue requests native macOS Talk Mode support for OpenAI Realtime speech-to-speech using the existing realtime voice/provider architecture where possible.

## Best Possible Solution

Implement native macOS realtime Talk on the current talk.client.create or talk.session.* contracts, wire AVAudioEngine/WebRTC or PCM relay playback into TalkModeRuntime and the overlay, and keep the existing STT/chat/talk.speak fallback.

## Solution Assessment
