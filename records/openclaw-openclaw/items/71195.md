## Summary

## What This Changes

The issue requests native macOS Talk Mode support for OpenAI Realtime speech-to-speech using the existing realtime voice stack while keeping the current STT/chat/TTS path as fallback.

## Best Possible Solution

Implement native macOS realtime Talk on the existing `talk.client.*` or `talk.session.*` contracts, wire AVAudioEngine/WebRTC or PCM relay playback into the Talk runtime and overlay, keep the current STT/chat/`talk.speak` path as fallback, and track the ElevenLabs PCM 403 behavior separately.
