## Summary

Keep open. Current main still has the same MCP loopback gap: cron preserves Matrix room ID casing only when a current delivery context reaches the tool, but the loopback HTTP path still forwards only the lowercased session-key-derived context.

## What This Changes

Preserve canonical Matrix room ID casing for implicit cron announce delivery when `mcp__openclaw__cron action=add` uses `sessionTarget: "current"` through the Claude CLI MCP loopback path.

## Best Possible Solution

Thread the preserved current channel target through the CLI MCP env/header, request context, loopback tool cache, and `resolveGatewayScopedTools` into `createOpenClawTools`, while keeping session-key lowercase normalization as the fallback contract.

## Reproduction Assessment

Yes, from source inspection rather than a live Matrix run. The loopback request path lacks a preserved current channel target, so implicit cron delivery can fall back to `parseAgentSessionKey`, which lowercases the Matrix room ID.

## Solution Assessment

Yes. Propagating the existing current delivery context through MCP loopback is the narrow maintainable fix; changing `parseAgentSessionKey` or adding Matrix-specific recanonicalization would broaden a stable generic routing contract.
