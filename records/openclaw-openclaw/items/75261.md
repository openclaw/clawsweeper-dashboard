## Summary

Current main implements the reported Matrix cron delivery fix: implicit cron delivery now uses current or stored delivery context instead of reconstructing case-sensitive provider targets from lowercased session keys, and the merged fix is present on main but not yet in a release tag.

## What This Changes

Ensure cron jobs created via MCP loopback with `sessionTarget: "current"` preserve mixed-case Matrix room IDs when inferring implicit announce delivery.

## Best Possible Solution

Keep the current stored delivery-context fallback and only add explicit loopback current-channel threading if future live evidence shows sessions without routable stored delivery metadata.

## Reproduction Assessment

Yes for the original 2026.4.27 failure: the reporter supplied concrete Matrix logs and current source/history shows the old lowercased session-key fallback was replaced. I did not run a live Matrix cron instance in this read-only review, but current main no longer has the failing source path for stored routed sessions.

## Solution Assessment

Yes. Resolving implicit cron delivery from current or stored delivery context is the narrow maintainable fix because it preserves session-key canonicalization while sourcing case-sensitive provider IDs from route metadata.
