## Summary

Keep open. Current main still has the source-level gap: cron preserves a mixed-case Matrix target only when `currentDeliveryContext.to` reaches `createCronTool`, but the Claude CLI MCP loopback path still omits the current channel target and can fall back to lowercased session-key parsing. The issue is also paired with open PR #75493, so it should stay open until that PR or a successor lands.

## What This Changes

The issue requests preserving canonical Matrix room ID casing for `cron` jobs created through Claude CLI MCP loopback with `sessionTarget: "current"` and implicit delivery.

## Best Possible Solution

Review and land PR #75493 or an equivalent successor that generically threads the canonical current delivery target and thread context through the Claude CLI MCP loopback request/env/header/tool-resolution path into `createOpenClawTools`, with regression coverage for mixed-case Matrix room IDs. Avoid a Matrix-only cron special case unless the generic delivery-context seam proves insufficient.

## Reproduction Assessment

Yes. The issue provides concrete v2026.4.27 steps, stored job JSON, and Matrix delivery errors, and current main source inspection shows the same loopback path still lacks current-channel target propagation.

## Solution Assessment

Yes. Extending the existing `currentDeliveryContext` seam into loopback tool resolution is the narrow maintainable fix because cron already preserves casing when that context is present; changing `parseAgentSessionKey` broadly would risk destabilizing canonical session routing.
