## Summary

Keep open. Current main still has the loopback propagation gap: cron can preserve a mixed-case Matrix target when `currentDeliveryContext` is populated, but the Claude CLI MCP loopback path does not carry the current channel target into tool resolution. Open PR #75493 explicitly targets this issue with closing syntax, so the issue should stay open until that PR or a narrow successor lands.

## What This Changes

Cron jobs created through the Claude CLI MCP loopback with `sessionTarget: "current"` should preserve mixed-case Matrix room IDs when inferring announce delivery.

## Best Possible Solution

Land PR #75493 or a narrow successor that threads the preserved current channel target through the Claude CLI MCP loopback config/env/header, request context, cache key, and `resolveGatewayScopedTools` into `createOpenClawTools`/`createCronTool`, with regression coverage for mixed-case Matrix room IDs.

## Reproduction Assessment

Yes. The issue provides a concrete v2026.4.27 Matrix plus Claude CLI MCP loopback reproduction and stored job/error evidence, and current main still shows the source-level path that falls back to lowercased session-key inference when loopback context lacks the target.

## Solution Assessment

Yes. Extending the existing generic `currentDeliveryContext`/current-channel seam into the loopback path is the narrow maintainable fix; changing the session-key parser contract or adding Matrix-specific recovery would be broader and riskier.
