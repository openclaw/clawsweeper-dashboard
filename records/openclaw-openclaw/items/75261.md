## Summary

Keep open. Current main still omits the current channel target from the MCP loopback request, CLI environment, and tool-resolution path, so implicit cron delivery can still fall back to lowercased session-key parsing; open PR #75493 references and targets this exact bug but is not merged.

## What This Changes

Fix cron jobs created via Claude CLI MCP loopback with `sessionTarget="current"` so implicit Matrix delivery preserves the canonical mixed-case room ID instead of falling back to lowercased session-key parsing.

## Best Possible Solution

Review and land PR #75493, or an equivalent generic fix, so the canonical current channel target is threaded through the Claude CLI MCP loopback path into `createOpenClawTools`/`createCronTool` before cron delivery falls back to session-key parsing.

## Reproduction Assessment

Yes. The reporter provides concrete v2026.4.27 reproduction steps, stored job JSON, and Matrix delivery errors, and current main still shows the source-level path where loopback cron creation lacks canonical target propagation.

## Solution Assessment

Yes. Threading the current channel ID through the existing loopback request/tool-resolution seam is the narrow maintainable fix; a Matrix-only cron special case would leave the generic case-sensitive target problem unresolved.
