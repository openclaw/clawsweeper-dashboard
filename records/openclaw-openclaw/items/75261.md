## Summary

Keep open. Current main still has the MCP loopback gap: cron preserves Matrix room ID casing only when a current delivery context is supplied, while loopback requests still provide session key, account, and channel metadata without the case-preserved current channel target.

## What This Changes

The issue asks OpenClaw to preserve mixed-case Matrix room IDs for implicit cron announce delivery when `mcp__openclaw__cron action=add` uses `sessionTarget: "current"` through the Claude CLI MCP loopback path.

## Best Possible Solution

Thread the preserved current channel target through the CLI MCP env/header, request context, loopback cache key, and `resolveGatewayScopedTools` into `createOpenClawTools`, while preserving the lowercase session-key fallback contract.

## Reproduction Assessment

Yes, source-reproducible on current main. The loopback path does not carry `currentChannelId`, so cron can fall back to `parseAgentSessionKey`, which lowercases the Matrix room ID.

## Solution Assessment

Yes. Propagating the existing case-preserving current channel context through the loopback path is the narrow maintainable fix; changing `parseAgentSessionKey` or adding Matrix-specific recanonicalization would broaden a stable generic contract.
