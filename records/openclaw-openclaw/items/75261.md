## Summary

Keep open. Current main and the latest release still omit a case-preserved current channel target from the MCP loopback path, so cron can still fall back to session-key delivery inference that lowercases mixed-case Matrix room IDs; the two linked fix attempts are closed unmerged.

## What This Changes

The issue asks for cron jobs created via Claude CLI MCP loopback with `sessionTarget: "current"` to preserve the current Matrix room ID casing in implicit announce delivery.

## Best Possible Solution

Thread the preserved current channel target through the CLI MCP env/header, request context, loopback tool cache key, and `resolveGatewayScopedTools` into `createOpenClawTools`, while leaving session-key lowercase normalization as the fallback contract.

## Reproduction Assessment

Yes, source-reproducible rather than live-reproduced. Reporter logs show the failed mixed-case Matrix cron job, and current main shows loopback supplies session key, message channel, and account ID but no preserved current channel target before cron falls back to lowercased `parseAgentSessionKey`.

## Solution Assessment

Yes. Propagating the existing case-preserving current channel context through the loopback path is the narrow maintainable fix; changing `parseAgentSessionKey` or adding Matrix-specific session-store recanonicalization would broaden a stable generic contract.
