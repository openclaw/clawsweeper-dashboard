## Summary

Keep open. Current `main` still has the MCP loopback propagation gap: cron preserves mixed-case Matrix targets only when `currentDeliveryContext` is populated, but the Claude CLI loopback path still resolves tools without a case-preserving current channel target, and the linked fix PRs are closed unmerged.

## What This Changes

Cron jobs created through the Claude CLI MCP loopback path with `sessionTarget: "current"` should preserve mixed-case Matrix room IDs when inferring implicit announce delivery.

## Best Possible Solution

Carry the case-preserving current channel target from inbound run context into loopback tool resolution so cron uses `currentDeliveryContext` and only falls back to session-key inference when no preserved target exists.

## Reproduction Assessment

Yes. The issue provides a concrete Matrix + Claude CLI MCP loopback reproduction, and current `main` still shows the deterministic fallback path that lowercases the Matrix target when loopback context lacks a preserved current channel ID.

## Solution Assessment

Yes. Threading `currentChannelId` through CLI params, MCP env/header/config, request context, cache key, and `resolveGatewayScopedTools` is the narrow maintainable fix; changing session-key normalization or adding Matrix-specific recovery would be broader and riskier.
