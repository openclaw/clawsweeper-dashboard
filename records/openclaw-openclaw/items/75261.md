## Summary

Keep open. Current main still preserves Matrix room ID casing only when `currentDeliveryContext` is populated, but the MCP loopback path does not carry a case-preserved current channel target into tool resolution, so cron can still fall back to lowercasing session-key inference; the two linked fix PRs are closed unmerged and no open PR currently tracks this exact issue.

## What This Changes

The issue requests preserving the case-sensitive Matrix room ID when `mcp__openclaw__cron action=add` uses `sessionTarget: "current"` through the Claude CLI MCP loopback path without explicit delivery.

## Best Possible Solution

Thread the preserved current channel target through the CLI MCP env/header, request context, loopback tool cache key, and `resolveGatewayScopedTools` into `createOpenClawTools`, while leaving session-key lowercase normalization as the fallback contract.

## Reproduction Assessment

Yes, from source inspection rather than a live Matrix run. A loopback request carries session key, message channel, and account ID but no preserved current channel target, so cron falls back to `parseAgentSessionKey`, which lowercases the Matrix room ID.

## Solution Assessment

Yes. Propagating the existing case-preserving current channel context through the loopback path is the narrow maintainable fix; changing `parseAgentSessionKey` or adding Matrix-specific recanonicalization would broaden a stable generic contract.
