## Summary

Keep open. Current main still lacks the MCP loopback target propagation needed for implicit cron delivery, and open PR #75493 explicitly targets this issue with closing syntax, so the issue should remain open until that PR or a successor merges.

## What This Changes

The issue reports that cron jobs created through the Claude CLI MCP loopback path with `sessionTarget: "current"` still derive Matrix delivery from a lowercased session key instead of preserving the canonical room ID case.

## Best Possible Solution

Review and land PR #75493, or a narrow successor, that threads the preserved current channel target through the Claude CLI MCP loopback request, runtime/cache, and tool-resolution path into `createOpenClawTools`/`createCronTool`, with regression coverage for mixed-case Matrix room IDs.

## Reproduction Assessment

Yes. The issue gives concrete v2026.4.27 reproduction steps and stored job/error evidence, and current main still shows the causal source path: loopback provides no current target, so implicit cron delivery falls back to lowercased session-key parsing.

## Solution Assessment

Yes. Extending the existing generic delivery-context seam into the loopback path is the narrow maintainable fix; changing the session-key parser contract or adding a Matrix-only special case would be broader and riskier.
