## Summary

Keep open. Current main still lacks the MCP loopback current-target propagation needed for implicit cron delivery, and open PR #75493 explicitly references this issue with closing syntax, so this issue should remain open until that PR or a successor merges.

## What This Changes

The issue requests that cron jobs created through MCP loopback with `sessionTarget: "current"` preserve case-sensitive Matrix room IDs when inferring implicit delivery.

## Best Possible Solution

Review and land PR #75493, or a narrow successor, that threads the preserved current channel target through the Claude CLI MCP loopback request, cache key, and tool-resolution path into `createOpenClawTools`/`createCronTool`, with regression coverage for mixed-case Matrix room IDs.

## Reproduction Assessment

Yes. The issue provides detailed v2026.4.27 reproduction steps and stored job/error evidence, and current main still shows the loopback path lacks the target context needed to avoid lowercased session-key fallback.

## Solution Assessment

Yes. Extending the existing generic `currentDeliveryContext` seam into the loopback path is the narrow maintainable fix; changing the session-key parser contract or adding a Matrix-only special case would be riskier.
