## Summary

Keep open. Current `main` still has the MCP loopback propagation gap: cron can preserve a mixed-case Matrix target when `currentDeliveryContext` is populated, but the Claude CLI MCP loopback path still does not carry a preserved current channel target into tool resolution, and the linked fix PRs are closed unmerged.

## What This Changes

Fix implicit cron announce delivery for `sessionTarget: "current"` when invoked through the Claude CLI MCP loopback path so mixed-case Matrix room IDs are preserved.

## Best Possible Solution

Thread a case-preserving current channel target through the CLI run params, loopback MCP env/header, request context, cache key, tool resolution, and `createOpenClawTools` so cron uses `currentDeliveryContext` instead of reconstructing delivery from the lowercased session key.

## Reproduction Assessment

Yes. The issue gives a concrete Matrix plus Claude CLI MCP loopback reproduction with stored job/error evidence, and current `main` still has the source path that lowercases the fallback when loopback context lacks the target.

## Solution Assessment

Yes. Extending the existing generic `currentDeliveryContext`/current-channel seam into the loopback path is the narrow maintainable fix; changing `parseAgentSessionKey` or adding Matrix-specific recovery would be broader and riskier.

## Repair Work Prompt

Fix issue #75261. Current `main` still loses mixed-case Matrix room IDs for `mcp__openclaw__cron action=add` with `sessionTarget: "current"` when called from a Claude CLI subprocess through the MCP loopback path. Implement the narrow generic fix: add a case-preserving `currentChannelId` or equivalent current delivery target to `RunCliAgentParams`, pass it from the agent run context into `prepareCliRunContext`, export it as an MCP loopback env var/header, read it in `resolveMcpRequestContext` without lowercasing, include it in the loopback tool cache key, pass it through `McpLoopbackToolCache.resolve` and `resolveGatewayScopedTools`, and populate `createOpenClawTools` so `createCronTool` receives `currentDeliveryContext.to` and avoids `inferDeliveryFromSessionKey`. Add regression coverage proving a mixed-case Matrix room ID in the loopback header reaches tool resolution unchanged, and update CLI prepare/config tests for the new env/header. Do not change the `parseAgentSessionKey` lowercase contract and do not add Matrix-specific special cases. Add a `CHANGELOG.md` fix entry if this is user-facing under current repo policy.
