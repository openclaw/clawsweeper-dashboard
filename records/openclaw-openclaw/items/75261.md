## Summary

Keep open. Current main still does not pass a case-preserved current channel target through the MCP loopback path, so Matrix cron jobs created with implicit delivery can still fall back to lowercased session-key inference.

## What This Changes

The issue requests preserving mixed-case Matrix room IDs for cron jobs created through the MCP loopback path with `sessionTarget: current` and no explicit delivery target.

## Best Possible Solution

Thread the preserved current channel target through CLI MCP env, loopback headers, request context, cache key, and `resolveGatewayScopedTools` into `createOpenClawTools`, then cover the Matrix casing path with focused MCP and cron regression tests.

## Reproduction Assessment

Yes. The reporter supplied a concrete Matrix/Claude CLI loopback reproduction and current main still shows the exact source path where missing delivery context falls back to lowercased session-key inference.

## Solution Assessment

Yes. Propagating the existing case-preserving current-channel context through the loopback path is the narrow maintainable fix; changing `parseAgentSessionKey` or adding Matrix-specific recanonicalization would broaden a stable generic contract.
