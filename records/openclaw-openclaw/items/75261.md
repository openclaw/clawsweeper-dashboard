## Summary

Keep open. Current main still has the MCP loopback path carrying session key, message channel, and account ID, but not a case-preserved current channel target, so cron can still fall back to lowercased session-key delivery inference for mixed-case Matrix room IDs.

## What This Changes

Cron jobs created through MCP loopback with `sessionTarget: "current"` should preserve mixed-case Matrix room IDs when implicit announce delivery is inferred.

## Best Possible Solution

Thread the preserved current channel target through the CLI MCP env/header, request context, loopback cache key, and gateway tool resolution into `createOpenClawTools`, while keeping session-key lowercase normalization as the fallback contract.

## Reproduction Assessment

Yes, source-reproducible: the loopback request path lacks a current-channel target, and cron falls back to `parseAgentSessionKey`, which lowercases the Matrix room ID. I did not run a live Matrix server, but the current-main code path is direct and matches the reporter's stored job/log evidence.

## Solution Assessment

Yes. Propagating the existing case-preserving current delivery context through the loopback path is the narrow maintainable fix; changing `parseAgentSessionKey` or adding Matrix-specific recanonicalization would broaden a stable generic contract.
