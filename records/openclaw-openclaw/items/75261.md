## Summary

Current main fixes the reported Matrix cron lowercasing path: cron delivery inference now falls back to stored delivery context that preserves provider target casing instead of rebuilding `delivery.to` from the lowercased session key.

## What This Changes

Fix cron jobs created through MCP loopback with `sessionTarget: "current"` so implicit Matrix announce delivery preserves the case-sensitive room ID.

## Best Possible Solution

Keep the current stored delivery-context fallback as the implemented fix, and only add explicit loopback current-channel threading if future live evidence shows a session without routable stored delivery metadata.

## Reproduction Assessment

Yes for the original 2026.4.27 failure: the reporter provided concrete Matrix logs and prior source showed the lowercased session-key fallback. Current main no longer has that failing fallback path for stored routed sessions because cron reads the stored delivery context before creating implicit announce delivery.

## Solution Assessment

Yes. The current implementation is a narrow maintainable fix because it preserves the existing session-key normalization contract while sourcing case-sensitive provider targets from stored delivery context rather than inventing Matrix-specific parsing.
