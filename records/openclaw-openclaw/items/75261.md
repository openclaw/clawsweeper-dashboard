## Summary

Keep open. Current main still lacks the MCP loopback propagation needed for implicit cron delivery to preserve a mixed-case Matrix room ID, and the linked open PR #75493 is the right pending implementation path rather than a reason to close the issue before merge.

## What This Changes

The issue requests preserving the canonical Matrix room ID when `mcp__openclaw__cron action=add` uses `sessionTarget: "current"` through the Claude CLI MCP loopback path without explicit delivery.

## Best Possible Solution

Land PR #75493 or a narrow successor that threads the preserved current channel target through the Claude CLI MCP loopback request, cache, and tool-resolution path into `createOpenClawTools`/`createCronTool`, with regression coverage for mixed-case Matrix room IDs.

## Reproduction Assessment

Yes. The issue provides a concrete v2026.4.27 reproduction and stored-job evidence, and current main still shows the source path that lets loopback cron fall back to lowercased session-key inference.

## Solution Assessment

Yes. Extending the existing generic `currentDeliveryContext` seam into the loopback path is the narrow maintainable fix; changing the session-key parser contract or adding a Matrix-specific fallback would be riskier.
