## Summary

Keep open. Current main still preserves Matrix room casing only when `currentDeliveryContext` is already present, but the Claude CLI MCP loopback path does not carry the current channel id into tool resolution, so cron can still fall back to delivery inferred from a lowercased session key.

## What This Changes

The issue requests preserving mixed-case Matrix room ids for cron jobs created via Claude CLI MCP loopback with `sessionTarget: "current"` and no explicit delivery.

## Best Possible Solution

Propagate a case-preserving current channel target through the existing generic CLI loopback and tool-resolution seams so cron uses `currentDeliveryContext` instead of reconstructing delivery from the normalized session key.

## Reproduction Assessment

Yes. The report gives concrete Matrix plus Claude CLI MCP loopback steps and stored job/error evidence, and current main still has the source path where missing loopback context falls back to the lowercased session-key parser.

## Solution Assessment

Yes. Extending the existing `currentChannelId`/`currentDeliveryContext` seam through the loopback path is narrower and safer than changing session-key normalization or adding Matrix-specific recovery.
