## Summary

Keep open. Current main still has the source-level gap described by the report: cron can preserve a case-sensitive delivery target only when `currentDeliveryContext` is populated, but the MCP loopback request/cache path still carries session, channel provider, account, and owner state without the current channel target. PR #75493 is an open linked implementation candidate, so this issue should remain open until that PR or a successor lands.

## What This Changes

Cron jobs created through MCP loopback with `sessionTarget: "current"` should preserve case-sensitive Matrix room IDs in implicit announce delivery instead of deriving a lowercased target from the session key.

## Best Possible Solution

Review and land PR #75493, or a narrow successor, that threads the preserved current channel target through the CLI runner env/header, MCP request context, loopback tool cache, and gateway tool-resolution path into `createOpenClawTools`/`createCronTool`, with regression coverage for mixed-case Matrix room IDs.

## Reproduction Assessment

Yes. The issue provides a detailed reproduction path and stored job/error evidence, and current main still shows the missing loopback target propagation that allows cron to fall back to lowercased session-key parsing.

## Solution Assessment

Yes. Extending the existing generic delivery-context seam into the loopback path is the narrow maintainable fix; changing the session-key parser contract or adding a Matrix-specific recovery path would be broader and riskier.
