## Summary

Keep open. Current main still preserves Matrix casing only when `currentDeliveryContext.to` is populated, while the Claude CLI MCP loopback path still does not pass a current channel target into tool resolution; the linked fix PRs are closed unmerged.

## What This Changes

Cron creation through the Claude CLI MCP loopback should preserve mixed-case Matrix room IDs when `sessionTarget` is `current` and delivery is inferred.

## Best Possible Solution

Thread the canonical current channel target from inbound run context into CLI params, MCP env/header/config, request context, cache key, and `resolveGatewayScopedTools`, while keeping the lowercased session-key parser contract as fallback only.

## Reproduction Assessment

Yes. The issue provides concrete Matrix and Claude CLI MCP loopback steps, and current main still shows a deterministic path where loopback-created cron tools lack `currentDeliveryContext.to` and fall back to lowercased session-key inference.

## Solution Assessment

Yes. Generic context propagation through the loopback path is the narrow maintainable fix; changing session-key normalization or adding Matrix-specific recovery would be broader and could destabilize routing contracts.
