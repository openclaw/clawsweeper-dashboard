## Summary

Keep open. Current main still has the MCP loopback propagation gap: cron preserves a mixed-case Matrix delivery target only when current delivery context is populated, but the loopback path does not carry the current channel target into tool resolution. Open PR #75493 explicitly targets this issue with closing syntax, so the issue should stay open until that PR or a narrow successor lands.

## What This Changes

The issue reports that `mcp__openclaw__cron action=add` with `sessionTarget: "current"` via Claude CLI MCP loopback stores a lowercased Matrix room ID in implicit cron delivery.

## Best Possible Solution

Review and land PR #75493 or a narrow successor that threads the preserved current channel target through the Claude CLI MCP loopback config, env/header, request context, cache key, and `resolveGatewayScopedTools` into `createOpenClawTools`/`createCronTool`, with regression coverage for mixed-case Matrix room IDs.

## Reproduction Assessment

Yes. The issue provides concrete v2026.4.27 Matrix plus Claude CLI MCP loopback steps and stored-job/error evidence, and current main source still shows the same path falling back to lowercased session-key inference when loopback context lacks a target.

## Solution Assessment

Yes. Extending the existing generic `currentDeliveryContext`/current-channel seam into the loopback path is the narrow maintainable fix; changing the session-key parser contract or adding Matrix-specific recovery would be broader and riskier.
