## Summary

Keep open. Current main still has the source-level propagation gap described by the report, and open PR #75493 explicitly targets this issue with closing syntax, so the issue should remain open until that PR or a narrow successor lands.

## What This Changes

The issue reports that cron jobs created through the MCP loopback path with `sessionTarget: "current"` still infer Matrix announce delivery from a lowercased session key instead of preserving the mixed-case room ID.

## Best Possible Solution

Review and land PR #75493, or a narrow successor, that threads the preserved current channel target through the Claude CLI MCP loopback request, cache, and tool-resolution path into `createOpenClawTools`/`createCronTool`, with regression coverage for mixed-case Matrix room IDs.

## Reproduction Assessment

Yes. The issue provides detailed v2026.4.27 reproduction steps and stored job/error evidence, and current main still shows the missing loopback target propagation that lets cron fall back to lowercased session-key parsing.

## Solution Assessment

Yes. Extending the existing generic delivery-context seam into the loopback path is the narrow maintainable fix; changing the session-key parser contract or adding a Matrix-specific special case would be riskier and less general.
