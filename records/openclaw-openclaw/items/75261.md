## Summary

Keep open. Current main still lacks a case-preserving current-channel value on the MCP loopback path, so cron can still fall back to lowercased session-key inference; the two linked fix PRs are closed unmerged.

## What This Changes

Report a regression where cron jobs created via MCP loopback with `sessionTarget=current` store Matrix room IDs lowercased when implicit delivery is inferred.

## Best Possible Solution

Thread the inbound current channel target through CLI MCP env/header, request context, loopback tool cache, gateway tool resolution, and `createOpenClawTools`, leaving session-key inference as a fallback only.

## Reproduction Assessment

Yes. The issue provides a concrete Matrix plus Claude CLI MCP loopback scenario, and current source deterministically shows that the loopback path cannot pass a case-preserved channel ID into cron before the lowercasing fallback runs.

## Solution Assessment

Yes. Propagating the case-preserved current channel through the MCP loopback path is the narrow maintainable fix; changing session-key normalization would be broader and would risk stable routing semantics.

## Repair Work Prompt

Fix the Matrix cron MCP loopback regression where `mcp__openclaw__cron action=add` with `sessionTarget=current` and no explicit `delivery` lowercases mixed-case Matrix room IDs. Thread a case-preserved current channel target from CLI run params into the bundled MCP loopback env/header/config, parse it in `McpRequestContext` without lowercasing, include it in `McpLoopbackToolCache` params and cache key, pass it through `resolveGatewayScopedTools`, and into `createOpenClawTools` so cron receives `currentDeliveryContext.to`. Keep `parseAgentSessionKey` and existing session-key normalization semantics unchanged; session-key inference should remain only the fallback. Add regression coverage for a mixed-case Matrix room ID through MCP loopback tool resolution and update CLI prepare/config tests. Add a single-line `CHANGELOG.md` fix entry because this is user-facing. Do not broaden Matrix-specific recovery or change unrelated cron retry behavior.
