## Summary

Keep open. Current main still has the reported source-level gap: cron can preserve a mixed-case Matrix room only when current delivery context reaches the cron tool, but the MCP loopback path still does not parse or forward a current channel target. Open PR #75493 now specifically references this issue and proposes the narrow fix, so the issue should remain open until that PR or an equivalent change lands.

## What This Changes

The issue asks OpenClaw to preserve mixed-case Matrix room IDs for implicit cron delivery when `mcp__openclaw__cron action=add` uses `sessionTarget: "current"` through the Claude CLI MCP loopback path.

## Best Possible Solution

Land PR #75493 or an equivalent generic fix that carries the case-sensitive current channel target, and any thread context if applicable, through the MCP loopback request/runtime/tool-resolution path into `createOpenClawTools`/`createCronTool`, with regression coverage for mixed-case Matrix room IDs.

## Reproduction Assessment

Yes. The issue includes concrete 2026.4.27 job JSON and Matrix errors, and current-main source inspection shows the same reachable path: loopback cron lacks current delivery context and can fall back to lowercased session-key parsing.

## Solution Assessment

Yes. Propagating the existing current-delivery-context seam through loopback is the narrow maintainable fix; changing the global session-key parser or adding a Matrix-only cron special case would be broader and riskier.
