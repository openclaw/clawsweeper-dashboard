## Summary

Keep open. Current main still preserves mixed-case Matrix targets only when `currentDeliveryContext` is populated, while the Claude CLI MCP loopback path still omits a preserved current channel target and can fall back to lowercased session-key inference. Open PR #75493 is the active closing-syntax fix, so this issue should remain open until that PR or a narrow successor lands.

## What This Changes

The issue reports that cron jobs created through the Claude CLI MCP loopback path with `sessionTarget: "current"` still store lowercased Matrix delivery targets.

## Best Possible Solution

Land PR #75493 or a narrow successor that threads the preserved current channel target through CLI run params, loopback env/header/request context, cache key, and tool resolution into `createOpenClawTools`, with a mixed-case Matrix regression test.

## Reproduction Assessment

Yes. The issue provides concrete v2026.4.27 Matrix plus Claude CLI loopback steps and stored job/error evidence, and source review shows current main can still fall back to lowercased session-key inference when loopback context lacks a preserved target.

## Solution Assessment

Yes. Extending the existing generic `currentDeliveryContext`/`currentChannelId` seam through the loopback path is the narrow maintainable fix; changing the session-key parser contract or adding Matrix-specific recovery would be broader and riskier.
