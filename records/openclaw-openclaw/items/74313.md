## Summary

Keep open. Current main still contains the reported doctor/model-status mismatch, and linked PR #74438 remains open and unmerged with closing syntax, so this issue should stay open until that PR lands or is replaced.

## What This Changes

The issue asks doctor not to present `agents/main` as removable orphan state when a non-`main` default agent coexists with commands that still use the compatibility main auth/session store.

## Best Possible Solution

Land PR #74438, or an equivalent narrow patch, so models status/probe honor the configured default agent and doctor only exempts the live compatibility main store from removable-orphan guidance.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: configure a non-`main` default agent, leave `agents/main/agent` on disk, and current main still classifies `main` as orphaned while omitted `models status` resolves auth state through the compatibility main agent dir.

## Solution Assessment

Yes. The narrow maintainable fix is to align omitted models status/probe resolution with the configured default agent while limiting the doctor change to the live compatibility-store case; PR #74438 is already the linked vehicle for that direction.
