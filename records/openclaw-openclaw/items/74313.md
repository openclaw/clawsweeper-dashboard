## Summary

Keep open. Current main still has the reported mismatch: doctor can classify an unconfigured `agents/main/agent` directory as orphaned, while omitted `models status` still resolves auth state through the compatibility `main` agent dir. The open, unmerged #74438 PR is already the closing fix vehicle for the same behavior.

## What This Changes

The issue asks doctor to stop presenting `agents/main` as removable orphaned state when the install still uses that compatibility/default agent store.

## Best Possible Solution

Review and land #74438, or an equivalent narrow fix, so omitted `models status` and `models status --probe` resolve the configured default agent unless an env override is active, while doctor exempts only the current live compatibility `main` store and continues reporting real orphan directories.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: configure a non-`main` default agent, leave `agents/main/agent` on disk, and current main classifies `main` as orphaned while omitted `models status` still reads the compatibility path.

## Solution Assessment

Yes. The best direction is the narrow #74438-style fix because it aligns command resolution with the documented configured-default contract and limits the doctor exemption instead of suppressing all orphan warnings.
