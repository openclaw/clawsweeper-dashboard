## Summary

Keep open. Current main still has the reported doctor/status mismatch, and the linked closing PR is open and unmerged, so this issue should stay open until that fix or an equivalent replacement lands.

## What This Changes

The issue asks for `openclaw doctor` to stop treating live `agents/main` compatibility state as safely removable when the configured default agent is not `main`.

## Best Possible Solution

Review and land #74438, or an equivalent narrow patch, so omitted `models status` and `--probe` use the configured default agent while doctor preserves true orphan warnings without flagging live compatibility `main` state.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: with a non-`main` configured default and `agents/main/agent` on disk, doctor classifies `main` as orphaned while omitted `models status` still resolves the auth store through the compatibility `main` path.

## Solution Assessment

Yes. The linked fix direction is the narrow maintainable path because it aligns command path resolution with the documented configured-default contract and limits the doctor behavior change to the live compatibility-store case.
