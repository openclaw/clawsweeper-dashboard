## Summary

Keep open. Current main still has the reported doctor/model-status mismatch, and open PR #74438 is the linked unmerged implementation vehicle, so this issue should remain open until that PR lands or is replaced.

## What This Changes

The issue requests that `openclaw doctor` stop presenting the live `agents/main` compatibility store as removable orphaned state when the configured default agent is not `main`.

## Best Possible Solution

Review and land PR #74438, or an equivalent narrow patch, so omitted `models status` and `--probe` honor the configured default agent while doctor exempts only the live compatibility `main` store from removable orphan guidance.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: configure a non-`main` default agent, leave `agents/main/agent` on disk, and current main still classifies `main` as orphaned while omitted `models status` resolves auth state through the compatibility `main` agent dir.

## Solution Assessment

Yes. The linked fix direction is the narrow maintainable path because it aligns status/probe path resolution with the documented configured-default contract and limits the doctor change to the live compatibility-store case instead of suppressing all orphan warnings.
