## Summary

Keep open. Current main still contains the reported doctor/status mismatch, and open PR #74438 is the unmerged closing implementation vehicle for the same bug.

## What This Changes

`openclaw doctor` should stop presenting the live `agents/main` compatibility store as removable orphaned state when the configured default agent is not `main`.

## Best Possible Solution

Review and land #74438, or an equivalent narrow patch, so omitted `models status` and probe use the configured default agent while doctor preserves the real orphan-dir warning without flagging the live compatibility store.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: configure a non-`main` default agent, leave `agents/main/agent` on disk, and current main will still classify `main` as orphaned while omitted models status/probe can resolve through the compatibility `main` agent dir.

## Solution Assessment

Yes. The linked fix direction is the narrow maintainable path because it aligns command path resolution with the configured-default contract and limits the doctor exception to the compatibility-store case rather than suppressing orphan detection broadly.
