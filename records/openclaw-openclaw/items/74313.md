## Summary

Keep open. Current main still has the reported mismatch: doctor can classify an unlisted `agents/main` directory as orphaned while omitted `models status` still resolves its auth store through the compatibility `main` path. The open, unmerged #74438 PR uses closing syntax for the same bug, so this issue should stay open until that PR is reviewed, landed, or replaced.

## What This Changes

`openclaw doctor` should stop presenting `agents/main` as removable orphaned state when a non-`main` configured default agent still leaves the compatibility `main` auth/session store live.

## Best Possible Solution

Review and land #74438, or an equivalent narrow patch, so omitted `models status` and `--probe` resolve the configured default agent unless an explicit env override is active, while doctor exempts only the live compatibility `main` store and continues reporting true orphan directories.

## Reproduction Assessment

Yes. Configure `agents.list` with a non-`main` default, leave `agents/main/agent` on disk, and current source still has doctor classify unconfigured `main` as orphaned while omitted `models status` reads through `resolveOpenClawAgentDir()` to `agents/main/agent`.

## Solution Assessment

Yes. The linked fix direction is the narrow maintainable path because it aligns command resolution with the documented configured-default contract and limits the doctor change to the compatibility-store false positive instead of weakening all orphan detection.
