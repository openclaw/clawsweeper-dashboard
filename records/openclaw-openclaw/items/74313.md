## Summary

Keep open. Current main still has the reported doctor/status mismatch, and open PR #74438 is the unmerged closing implementation vehicle for the same behavior, so this issue should remain open until that PR lands or is replaced.

## What This Changes

The issue requests that `openclaw doctor` avoid presenting the compatibility `agents/main` store as safely removable when a non-main configured default agent is present but live commands can still read state from `agents/main`.

## Best Possible Solution

Review and land #74438, or an equivalent narrow patch, so omitted `models status` and `--probe` resolve the configured default agent unless an env override is active, while doctor exempts only the live compatibility `main` store and continues reporting true orphan directories.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: configure a non-`main` default agent, leave `agents/main/agent` on disk, and current main still classifies `main` as orphaned while omitted `models status` reads the compatibility `main` auth store.

## Solution Assessment

Yes. The linked fix direction is the narrow maintainable solution because it aligns command path resolution with the documented configured-default contract and limits the doctor change to the live compatibility-store case.
