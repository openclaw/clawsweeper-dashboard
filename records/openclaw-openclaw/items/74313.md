## Summary

Keep open. Current main still has the reported doctor/status mismatch, and #74438 is an open, unmerged PR with closing syntax for the same behavior, so this issue should remain open until that fix lands or is replaced.

## What This Changes

The issue requests that `openclaw doctor` stop presenting the compatibility `agents/main` state directory as safely removable when a non-`main` configured default agent is active but `main` may still be live state.

## Best Possible Solution

Review and land #74438, or an equivalent narrow patch, so omitted `models status` and `--probe` resolve the configured default agent unless an env override is active, while doctor exempts only the live compatibility `main` store and continues reporting true orphan directories.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: configure a non-`main` default agent, leave `agents/main/agent` on disk, and current main still classifies `main` as orphaned while omitted `models status` reads the compatibility `main` auth store.

## Solution Assessment

Yes. The narrow maintainable solution is to align models status/probe agent-scope resolution with the documented configured-default contract and limit the doctor change to the live compatibility-store case rather than suppressing all orphan warnings.
