## Summary

Keep open. Current main still has the reported doctor/model-status mismatch, and open PR #74438 is the unmerged closing implementation vehicle, so this issue should remain open until that PR lands or is replaced.

## What This Changes

The issue reports that `openclaw doctor` can classify the live compatibility `agents/main` store as orphaned when a different configured agent is default.

## Best Possible Solution

Review and land PR #74438, or an equivalent narrow patch, so omitted `models status` and `--probe` honor the configured default agent while doctor avoids removable-orphan guidance for the live compatibility `main` store.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: configure a non-`main` default agent, leave `agents/main/agent` on disk, and current main still classifies `main` as orphaned while omitted `models status` resolves auth state through the compatibility `main` agent dir.

## Solution Assessment

Yes. The linked fix direction is the narrow maintainable path because it aligns status/probe path resolution with the configured-default contract and limits the doctor change to the live compatibility-store case instead of suppressing all orphan warnings.
