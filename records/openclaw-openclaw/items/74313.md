## Summary

Keep open. Current main still contains the reported doctor/status mismatch, and open PR #74438 is the unmerged closing implementation vehicle, so this should remain open until that PR lands or is replaced.

## What This Changes

The issue reports that `openclaw doctor` can mark the legacy `agents/main` store as orphaned when the configured default agent is not `main`, even though model/status paths can still read that store.

## Best Possible Solution

Review and land PR #74438, or an equivalent narrow patch, so omitted `models status` and `--probe` honor the configured default agent while doctor stops treating the live compatibility `main` store as removable stale state.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: configure a non-`main` default agent, leave `agents/main/agent` on disk, and current main still classifies `main` as orphaned while omitted model status/probe can read the compatibility `main` agent dir.

## Solution Assessment

Yes. The linked fix direction is the narrow maintainable solution because it aligns command path resolution with the documented configured-default contract and limits the doctor change to the live compatibility-store case instead of suppressing all orphan warnings.
