## Summary

Keep open. Current main still has the reported doctor/status mismatch, and open PR #74438 is the unmerged closing implementation vehicle, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue requests that doctor stop presenting `agents/main` as removable orphan state when a named default agent coexists with a live compatibility store.

## Best Possible Solution

Review and land PR #74438, or an equivalent narrow patch, so omitted `models status`/`--probe` resolve the configured default agent while doctor exempts the live compatibility `main` store without hiding real orphan dirs.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: configure a non-`main` default agent, leave `agents/main/agent` on disk, and current main will warn on `main` while omitted status/probe paths can still read the compatibility dir.

## Solution Assessment

Yes. The linked fix direction is the narrow maintainable path because it aligns command path resolution with the configured-default contract and limits the doctor change to the live compatibility-store case.
