## What This Changes

Fix `openclaw doctor` so live `agents/main` compatibility state is not presented as removable orphaned data when the configured default agent is not `main`.

## Best Possible Solution

Review and land #74438, or an equivalent narrow patch, so omitted `models status` and `--probe` honor the configured default agent while doctor exempts only the live compatibility `main` store and still reports true orphan directories.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: configure a non-`main` default agent, leave `agents/main/agent` on disk, and current main still classifies `main` as orphaned while omitted `models status` reads the compatibility `main` auth store.

## Solution Assessment

Yes. The linked fix direction is the narrow maintainable solution because it aligns command path resolution with the documented configured-default contract and limits the doctor change to the live compatibility-store case.
