## Summary

Keep open. Current main still has the reported doctor/status mismatch, and open PR #74438 is the unmerged closing implementation vehicle for this exact behavior.

## What This Changes

The issue reports that `openclaw doctor` can suggest removing `agents/main` as orphaned even while other commands still use it as live compatibility state.

## Best Possible Solution

Review and land #74438, or an equivalent narrow replacement, so omitted `models status` and `--probe` honor the configured default agent while doctor exempts only live compatibility `main` state and still reports true orphan dirs.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: configure a non-`main` default agent, keep `agents/main/agent` on disk, and doctor classifies `main` as orphaned while omitted `models status` still reads the compatibility main auth store.

## Solution Assessment

Yes. The linked fix direction is the narrow maintainable path because it aligns command path resolution with the documented configured-default contract and limits the doctor change to the compatibility-store case.
