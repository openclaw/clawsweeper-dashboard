## Summary

Keep open. Current main still has the reported doctor/status mismatch, and #74438 is an open, unmerged PR with closing syntax for the same behavior, so this issue should remain open until that PR lands or is replaced.

## What This Changes

The issue asks doctor to stop presenting `agents/main` as removable orphan state when current commands can still use it as live auth/session state under a non-`main` configured default agent.

## Best Possible Solution

Land #74438 or an equivalent narrow patch that aligns omitted `models status` and `--probe` with the configured-default contract, while doctor exempts only the live compatibility `main` store and keeps reporting true orphan directories.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: configure a non-`main` default agent, leave `agents/main/agent` on disk, and current main still classifies `main` as orphaned while omitted `models status` reads the compatibility `main` auth store.

## Solution Assessment

Yes. The linked fix direction is the narrow maintainable solution because it aligns command path resolution with the documented configured-default contract and limits the doctor change to the compatibility-store case.
