## Summary

Keep open. Current main still has the reported mismatch: doctor can classify an unconfigured `agents/main/agent` directory as orphaned while omitted `models status` still resolves through the compatibility `main` agent dir, and #74438 is already the open, unmerged closing PR for this behavior.

## What This Changes

Fix `openclaw doctor` so it does not present `agents/main` as an orphaned/removable agent directory when a non-`main` configured default coexists with live compatibility auth/session state.

## Best Possible Solution

Review and land #74438, or an equivalent narrow fix, so omitted `models status` and probe use the configured default agent while doctor exempts only the current live compatibility `main` store and continues reporting real orphan directories.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: configure a non-`main` default agent, leave `agents/main/agent` on disk, and current main will classify `main` as orphaned while omitted `models status` still reads through the compatibility path.

## Solution Assessment

Yes. The linked fix direction is the narrow maintainable path because it aligns command resolution with the documented configured-default contract and limits the doctor exemption instead of suppressing all orphan warnings.
