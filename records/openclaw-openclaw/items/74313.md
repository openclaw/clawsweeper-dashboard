## Summary

Keep open. Current main still has the reported doctor/status mismatch, and the open, unmerged #74438 PR is already the closing implementation vehicle for the same behavior.

## What This Changes

The issue asks OpenClaw to stop `openclaw doctor` from presenting `agents/main` as safely removable orphan state when a named default agent is configured but the compatibility `main` store is still live.

## Best Possible Solution

Review and land #74438, or an equivalent narrow patch, so omitted `models status` and `--probe` resolve the configured default agent unless an env override is active, while doctor exempts only the live compatibility `main` store and continues reporting true orphan directories.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: configure a non-`main` default agent, leave `agents/main/agent` on disk, and current main still classifies `main` as orphaned while omitted `models status` reads the compatibility `main` auth store.

## Solution Assessment

Yes. The linked fix direction is the narrow maintainable solution because it aligns the command path with the documented configured-default contract and limits the doctor change to the live compatibility-store case.
