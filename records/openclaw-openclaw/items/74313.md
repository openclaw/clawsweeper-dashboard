## Summary

Keep open. Current main still contains the reported doctor/models-status mismatch, and the provided context shows #74438 is an open, unmerged PR with closing syntax for the same behavior, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue requests that `openclaw doctor` stop presenting `agents/main` as removable orphaned state when current commands still use that compatibility store.

## Best Possible Solution

Land #74438 or an equivalent narrow fix that makes omitted `models status` and `models status --probe` resolve the configured default agent while doctor exempts only the live compatibility `main` store and keeps warning on real orphan directories.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: configure a non-`main` default agent, leave `agents/main/agent` on disk, and current main classifies `main` as orphaned while omitted `models status` still resolves auth/model state through the compatibility `main` path.

## Solution Assessment

Yes. The linked fix direction is the narrow maintainable path: align status/probe agent-scope resolution with the documented configured-default contract and keep doctor’s orphan warning focused on truly unreachable stores.
