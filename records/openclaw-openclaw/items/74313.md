## Summary

Keep open. Current main still has the reported doctor/model-status mismatch, and #74438 is an open closing PR for the same bug, so this issue should remain open until that PR lands or is replaced.

## What This Changes

The issue requests that `openclaw doctor` stop treating a live `agents/main` compatibility store as removable orphan state when the configured default agent is not `main`.

## Best Possible Solution

Review and land #74438, or an equivalent narrow patch, so status/probe use the configured default agent and doctor exempts only the live compatibility `main` store from removable-orphan guidance.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: configure a non-`main` default agent, leave `agents/main/agent` on disk, and current main still classifies `main` as orphaned while omitted `models status` resolves through the compatibility agent dir.

## Solution Assessment

Yes. The linked fix direction is the narrow maintainable path because it aligns status/probe resolution with configured-default semantics while keeping real orphan-agent warnings intact.
