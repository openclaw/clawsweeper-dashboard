## Summary

Keep open. Current main still has the reported mismatch: the doctor orphan-directory scan can classify `agents/main` as unconfigured while `models status` still falls back to the compatibility agent dir when `--agent` is omitted. The open, unmerged #74438 PR has closing syntax for this exact issue, so this issue should remain open until that PR lands or is replaced.

## What This Changes

The issue asks `openclaw doctor` to stop presenting `agents/main` as removable orphaned state when a non-`main` default agent is configured but current commands still use `agents/main` as live compatibility/default state.

## Best Possible Solution

Review and land #74438, or an equivalent narrow fix, so omitted `models status` and probe paths resolve the configured default agent while doctor exempts only the live compatibility `main` store and continues reporting real orphan directories.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: configure a non-`main` default agent, leave `agents/main/agent` on disk, and current main will classify `main` as orphaned while omitted `models status` still resolves auth state through the compatibility path.

## Solution Assessment

Yes for the linked fix direction. The maintainable solution is to align the status/probe agent-scope resolution with the documented configured-default contract and keep the doctor exemption narrowly tied to the live compatibility store rather than suppressing all orphan warnings.
