## Summary

Keep open. Current main still has the reported mismatch: doctor can classify an unconfigured `agents/main/agent` directory as orphaned, while omitted `models status` still reads the compatibility agent dir; the open, unmerged #74438 PR is already the closing fix vehicle.

## What This Changes

The issue requests that `openclaw doctor` stop presenting the live compatibility `agents/main` store as removable orphaned state when a different configured agent is default.

## Best Possible Solution

Review and land #74438, or an equivalent narrow fix, so omitted `models status` and probe resolve the configured default agent while doctor exempts only the current live compatibility agent dir and continues reporting real orphan directories.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: configure a non-`main` default agent, leave `agents/main/agent` on disk, and current main will classify `main` as orphaned while omitted `models status` still resolves auth state through the compatibility path.

## Solution Assessment

Yes. The linked fix direction is the narrow maintainable path because it aligns command resolution with the documented configured-default contract and limits the doctor exemption to the live compatibility store rather than suppressing all orphan warnings.
