## Summary

Keep open. The PR addresses a real source-reproducible gateway wake-routing bug, but the live head is not mergeable because it still uses the obsolete heartbeat wake API on current main and still allows a literal `global` direct wake key outside global session scope.

## What This Changes

The PR threads optional wake `sessionKey` and `agentId` metadata through direct and mapped HTTP hook wake dispatch, adds session-policy guards, and adds gateway hook regression tests.

## Best Possible Solution

Rebase onto current main, use requestHeartbeat with explicit hook wake metadata, reject literal `global` outside global session scope, preserve the policy regressions, and attach redacted end-to-end proof before merge.

## Reproduction Assessment

Yes. Current main is source-reproducible: wake mappings drop `agentId` and `sessionKey` before dispatch, and the dispatcher enqueues every wake into the main session.

## Solution Assessment

No, not yet. Threading wake targets through the existing hook session-policy path is the right direction, but the branch needs the current heartbeat API and the literal `global` edge fixed before it is the narrow mergeable solution.
