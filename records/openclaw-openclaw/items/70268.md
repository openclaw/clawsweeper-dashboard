## Summary

Keep open. Current main still ignores wake-hook session targeting, so this PR remains a useful implementation candidate for the linked bug, but the patch is not ready to merge because it can still accept an explicit `global` wake target outside global session scope and enqueue an event that the heartbeat runner will not drain; it also needs the required user-facing changelog entry.

## What This Changes

The PR branch adds sessionKey and agentId routing to direct and mapped HTTP wake hooks, canonicalizes wake dispatch targets, enforces request/prefix/global-scope policy, and expands gateway hook tests.

## Best Possible Solution

Keep the PR open and land the focused wake-session routing fix after the explicit `global` edge is made drain-safe, the required changelog entry is added, and the targeted gateway tests plus changed gate pass on the final head.

## Reproduction Assessment

Yes. Current main reproduces by inspection: direct wake normalizes only `text`/`mode`, mapped wake actions do not carry routing fields, and dispatch always uses the main session. The PR’s remaining edge can be reproduced with non-global session scope plus explicit wake `sessionKey: "global"`, which queues `global` while the heartbeat runner targets an agent-scoped key.

## Solution Assessment

No, not yet. The PR is the right narrow direction for #64556, but it needs to reject or canonicalize explicit `global` wake keys outside `session.scope: "global"` so the queued system event and heartbeat target stay aligned, plus a changelog entry for the user-facing fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Reject global wake keys outside global scope:** `src/gateway/server/hooks-request-handler.ts:284-291`
  - body: When `session.scope` is not `"global"`, an explicit wake `sessionKey: "global"` can pass validation and reach `dispatchWakeHook`. That queues the event under `global`, but the heartbeat runner treats the forced key as `agent:<default>:global`, so `mode: "now"` returns 200 while the queued wake event is not drained. Please reject `global` outside global scope or canonicalize it to the same key the heartbeat runner will process.
  - confidence: 0.86
- **[P3] Add the required changelog entry:** `src/gateway/hooks.ts:218`
  - body: This is a user-facing gateway fix: hook operators can now target wake events at explicit sessions instead of silently falling back to main. Repository policy requires a `CHANGELOG.md` entry for this kind of fix, credited to the contributor, before landing.
  - confidence: 0.82
