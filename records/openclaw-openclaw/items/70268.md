## What This Changes

The PR threads `sessionKey` and `agentId` through direct and mapped HTTP wake hooks, validates hook session policy, canonicalizes wake dispatch targets, and adds gateway hook regressions.

## Best Possible Solution

Rebase onto current main, use the current heartbeat wake API, reject literal `global` outside global session scope, keep the policy regressions, and attach redacted end-to-end proof before merge.

## Reproduction Assessment

Yes. Current main is source-reproducible: wake mappings drop `agentId` and `sessionKey` before dispatch, and `dispatchWakeHook` enqueues to the main session; the PR defects are also source-reproducible from the branch and current heartbeat code.

## Solution Assessment

No, not yet. Threading wake targets through the existing hook session-policy path is the right direction, but the branch needs the current heartbeat API and the literal `global` edge fixed before it is mergeable.
