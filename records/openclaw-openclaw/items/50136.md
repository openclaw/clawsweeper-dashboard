Full review comments:

- **[P1] Preserve the Windows unknown-listener recovery contract:** `src/cli/daemon-cli/lifecycle.ts:209-214`
  - body: This drops `includeUnknownListenersAsStale` from the managed Windows restart waits, undoing the shipped recovery path from https://github.com/openclaw/openclaw/pull/24734. With runtime stopped or unknown and only an unknown stale listener on the port, restart can now leave the stale gateway listener in place instead of recovering; preserve the fallback or replace it with a narrower tested contract plus native Windows proof.
  - confidence: 0.86
- **[P2] Preserve listener PIDs on probe-success snapshots:** `src/cli/daemon-cli/restart-health.ts:140-146`
  - body: When runtime is stopped or unknown and the loopback probe succeeds, this early return marks the restart healthy but sets all `listenerKinds` arrays empty. The unchanged-PID warning depends on `listenerKinds.gateway`, so this path can skip the new identity diagnostic even when the old PID still owns the listener.
  - confidence: 0.87
- **[P2] Add identity warnings to the shared warnings payload:** `src/cli/daemon-cli/lifecycle.ts:249-253`
  - body: In non-JSON mode this logs the unchanged-PID warning but never pushes it into the shared `warnings` array that the restart result uses. Push the warning before the `if (!json)` branch so human output and structured restart diagnostics stay consistent.
  - confidence: 0.82
- **[P2] Restore process.platform after Windows tests:** `src/cli/daemon-cli/restart-health.test.ts:214`
  - body: The new restart-health tests set `process.platform` to `win32` but never restore it. Wrap both overrides in `try/finally` or a spy restore so later tests do not silently inherit the wrong platform.
  - confidence: 0.82
