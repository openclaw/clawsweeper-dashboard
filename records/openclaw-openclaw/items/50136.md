## Summary

Keep open: current main still has the Windows unknown-listener stale fallback and loose OpenClaw substring listener classification, while this stale PR would regress newer restart-health and restart-probe hardening if merged as-is.

## What This Changes

The PR changes Windows gateway restart handling to avoid killing unknown listeners, tighten listener classification, add PID/diagnostic restart logging, and update daemon/status/port tests.

## Best Possible Solution

Land a rebased or replacement patch that separates diagnostic unknown listeners from verified kill candidates, uses the existing argv-based gateway classifier, preserves current health gates and exact auth-close allowlist, and adds focused regression coverage plus changelog.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: on win32, a stopped runtime plus a busy unknown listener with includeUnknownListenersAsStale enabled returns that listener PID in staleGatewayPids, and the restart path can terminate it.

## Solution Assessment

No, not as-is. The PR direction is useful, but the branch must be rebased or replaced so the listener-safety changes are layered onto current restart-health contracts and security hardening instead of replacing them.
