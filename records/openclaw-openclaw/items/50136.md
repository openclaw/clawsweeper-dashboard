## Summary

Keep open. Current main still contains the Windows unknown-listener stale cleanup path and the loose `openclaw` substring listener classifier, so the PR is not obsolete; however, the branch is stale against current restart-health behavior and would need a careful rebase before it is mergeable.

## What This Changes

The PR changes Windows gateway restart handling to avoid killing unknown listeners, tighten gateway listener classification, add restart diagnostics and a PID identity warning, and update related daemon/status/port tests.

## Best Possible Solution

Land one current-main patch that preserves the existing restart version/plugin/channel health contract and exact restart-probe close allowlist, separates diagnostic unknown listeners from verified kill candidates, tightens listener classification through the existing argv parser, and updates daemon/update/status/port tests together.

## Reproduction Assessment

Yes. On current main, a focused source-level reproduction is to run the restart-health path as `win32` with runtime stopped, a busy port listener classified as `unknown`, and a failed probe; with `includeUnknownListenersAsStale` enabled, the unknown PID is returned in `staleGatewayPids` and `runDaemonRestart()` can terminate it.

## Solution Assessment

No, not as-is. The PR's direction is useful, but the branch predates current restart-health version/plugin/channel checks and exact auth-close handling; the safer solution is a rebase or replacement patch layered onto the current contract.
