- **[P2] Preserve listener PIDs on probe-success snapshots:** `src/cli/daemon-cli/restart-health.ts:146`
  - body: When runtime is stopped or unknown and the loopback probe succeeds, this early return marks restart healthy with `listenerKinds.gateway` empty. `runDaemonRestart` relies on that list to warn when the pre-restart PID still owns the listener, so this path can skip the new unchanged-PID diagnostic.

  - body: In non-JSON mode this logs the unchanged-PID warning but never pushes it into the shared `warnings` array that `runServiceRestart` emits in the result payload. Push the warning before the `if (!json)` block so human output and structured restart diagnostics stay consistent.
  - confidence: 0.82
- **[P2] Restore process.platform in the added Windows tests:** `src/cli/daemon-cli/restart-health.test.ts:214`
  - body: The new restart-health tests set `process.platform` to `win32` without restoring it, so later tests can run under the leaked platform value and silently exercise different branches. Guard each override with `try`/`finally` or shared cleanup.
  - confidence: 0.78
