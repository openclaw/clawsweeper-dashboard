The issue asks for caller-supplied custom HTTP headers on OpenAI-compatible requests to be available inside a Python skill script.

## Best Possible Solution

Keep the remaining design on https://github.com/openclaw/openclaw/issues/80329, with an explicit allowlisted, redacted, request-scoped credential/env contract if maintainers accept the feature.

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence that current main has no supported arbitrary inbound header map from OpenAI-compatible requests into skill execution.
