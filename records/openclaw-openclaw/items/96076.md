## Summary

Keep open: this PR targets a real Gateway stale-index regression, but the patch is not merge-correct because it appends missing source-root plugins only to the manifest registry while runtime startup still plans from the unchanged installed index. ## What This Changes

The PR modifies `src/plugins/plugin-metadata-snapshot.ts` to append unindexed source-root plugin manifests when a persisted installed-plugin index exists.

## Best Possible Solution

Fix stale persisted registry handling so the metadata snapshot index and manifest registry stay aligned, add focused stale-index coverage, and require redacted real Gateway or runtime-registry proof before merge.

## Reproduction Assessment

Yes. Source inspection shows current main validates `plugins.slots.memory` against manifest registry IDs while Gateway runtime startup loads only ids derived from `metadataSnapshot.index`.

## Solution Assessment

No. Appending manifest records is only a partial fix; the maintainable fix should re-derive stale registry state or update the index and manifest view atomically so validation and startup agree.
