## Summary

Keep open. Current main still lets `sessions_history` forward an oversized limit to `chat.history`, and open PR #66650 already targets this exact bug with closing syntax, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that `sessions_history` can send `limit: 2000` to `chat.history`, exceeding the Gateway protocol maximum of 1000 and losing history context after validation fails.

## Best Possible Solution

Land PR #66650 or an equivalent narrow fix that caps the `sessions_history` tool schema and outgoing `chat.history` request limit at 1000 with regression coverage.

## Reproduction Assessment

Yes. A static reproduction is high confidence: call `sessions_history` with `limit: 2000`; current main accepts and forwards it, then the Gateway `chat.history` validator rejects it before the handler clamp runs.

## Solution Assessment

Yes. The linked PR direction is the narrowest maintainable fix because it aligns the tool boundary with the existing Gateway protocol limit instead of raising the protocol maximum or parsing validation errors for retry.
