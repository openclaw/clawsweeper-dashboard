## Summary

Keep open. Current `main` still forwards oversized `sessions_history` limits into `chat.history`, and the open implementation PR at https://github.com/openclaw/openclaw/pull/66650 already uses closing syntax for this issue, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that `sessions_history` can request `chat.history` with `limit: 2000`, exceeding the Gateway protocol maximum of 1000 and losing history context after validation failure.

## Best Possible Solution

Review, update, and land https://github.com/openclaw/openclaw/pull/66650 or an equivalent narrow fix that caps the tool schema and outgoing request at 1000 while keeping the Gateway protocol cap unchanged.

## Reproduction Assessment

Yes. A source-level repro is high confidence: invoke `sessions_history` with `limit: 2000`; current `main` normalizes it to 2000 and forwards it, while `chat.history` rejects values above 1000 before its handler clamp.

## Solution Assessment

Yes. The linked caller-side clamp is the narrowest maintainable direction because it aligns the tool boundary with the existing Gateway protocol limit instead of raising the protocol maximum or parsing validation errors for retry.
