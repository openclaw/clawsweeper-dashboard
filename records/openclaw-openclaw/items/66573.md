## What This Changes

The issue reports that `sessions_history` can request `chat.history` with `limit: 2000` even though the Gateway protocol rejects values above 1000, causing lost conversation context after validation failure.

## Best Possible Solution

Review, update, and land https://github.com/openclaw/openclaw/pull/66650 or an equivalent narrow caller-side clamp while keeping the Gateway `chat.history` cap at 1000.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to invoke `sessions_history` with `limit: 2000`; current main forwards 2000, while `chat.history` rejects values above 1000 before its handler clamp.

## Solution Assessment

Yes. Clamping at the `sessions_history` tool boundary is the narrowest maintainable direction because it aligns the caller with the existing Gateway protocol instead of raising the protocol maximum or parsing validation errors for retry.
