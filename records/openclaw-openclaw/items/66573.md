## Summary

Keep open. Current main still has the reported caller/protocol mismatch: `sessions_history` can forward `limit: 2000`, while `chat.history` rejects values above 1000 before the handler clamp runs; the live open implementation PR at https://github.com/openclaw/openclaw/pull/66650 already references this issue with closing syntax, so this should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue asks OpenClaw to avoid sending an oversized `chat.history` limit that exceeds the Gateway's 1000-item maximum.

## Best Possible Solution

Review, update, and land https://github.com/openclaw/openclaw/pull/66650 or an equivalent caller-side clamp while keeping the Gateway `chat.history` cap at 1000.

## Reproduction Assessment

Yes. A high-confidence source-level path is to invoke `sessions_history` with `limit: 2000`: current main forwards 2000 to `chat.history`, whose validator rejects values above 1000 before the handler clamp runs.

## Solution Assessment

Yes. Clamping at the `sessions_history` tool boundary is the narrowest maintainable direction because it aligns the caller with the existing Gateway protocol instead of raising the protocol maximum or parsing validation errors for retry.
