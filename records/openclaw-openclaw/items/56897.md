## Summary

Current main still has the xurl safety gap: the bundled skill lists public/account-mutating X commands and raw POST usage, but the safety text only covers credential handling and one-line 429 backoff. This remains a valid security-policy issue for maintainer review rather than cleanup closure.

## What This Changes

The issue requests xurl skill instructions for confirm-before-write, redacted write-operation audit logging, and safer rate-limit/backoff handling.

## Best Possible Solution

Update skills/xurl/SKILL.md with explicit read-only/write classifications, confirmation wording that shows account/action/target/content, redacted bounded audit guidance, and conservative Retry-After/backoff rules; track hard runtime enforcement separately if maintainers require it.

## Reproduction Assessment

Yes, for a source-level gap: current main and v2026.5.22 show the xurl skill still has write-capable examples without xurl-specific confirmation, audit, or detailed backoff instructions. This is not a runtime bug reproduction.

## Solution Assessment

Partly yes: SKILL.md hardening is the narrowest first fix for agent-followed guidance. If maintainers want non-bypassable enforcement, that should be a separate runtime or plugin-hook design rather than implied by skill text.
