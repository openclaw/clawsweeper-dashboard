## Summary

Keep open. Current main contains the bundled xurl skill with an active X write surface, but the skill instructions still lack the requested explicit confirm-before-write workflow and redacted write audit trail; rate-limit guidance is present only as a brief generic note. This is actionable in the bundled skill, not a closeable external/ClawHub request.

## What This Changes

The issue requests xurl skill instructions that require user confirmation before X write actions, document redacted audit logging for writes, and add practical rate-limit/backoff guidance.

## Best Possible Solution

Update the bundled xurl skill instructions so agents classify read-only versus write actions, show the exact account/action/target/content before any X write, obtain explicit user confirmation, record redacted write outcomes to a bounded audit trail, and handle 429/Retry-After with conservative backoff. If maintainers want hard guarantees, track a separate plugin/trusted-policy classifier rather than hiding that design inside a skill text change.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/56897 with a focused bundled-skill safety update. Edit `skills/xurl/SKILL.md` to add explicit xurl write-safety instructions: classify read-only commands versus write/mutating commands; require explicit user confirmation before every write action, showing the exact account/app/user if known, command/action, target, and content/payload summary; include shortcut writes such as post, reply, quote, delete, like/unlike, repost/unrepost, bookmark/unbookmark, follow/unfollow, block/unblock, mute/unmute, DM send, and raw API non-GET/POST/PUT/PATCH/DELETE writes; keep read-only operations such as read/search/timeline/mentions/whoami/user/list/status free of confirmation. Add redacted audit-log instructions for write outcomes using a bounded path such as `/tmp/ttl=30d/xurl-audit/YYYY-MM-DD.log`, with timestamp, action type, target, redacted content summary, and success/failure/result, and explicitly forbid logging secrets, tokens, authorization headers, ~/.xurl, or full sensitive DM bodies. Improve rate-limit guidance to respect 429 and Retry-After, serialize write bursts, use conservative exponential backoff, and stop after repeated 429/403 or scope failures. Do not add runtime before_tool_call enforcement, new dependencies, installer changes, credential handling changes, or xurl command behavior in this PR. If a changelog entry is added, put it under Unreleased and credit `Thanks @xinbenlv` on a single line.
