## Best Possible Solution

Full review comments:

- **[P1] Preserve exec approval semantics when sourcing snapshots:** `src/agents/shell-snapshot.ts:380`
  - body: The approval/safe-bin path approves the original command before `runExecProcess`, but this wrapper then sources user startup aliases/functions and re-parses the same text with `eval`. With snapshots enabled, a startup function or alias named like an allowlisted or safe-bin executable can replace the approved binary after the decision, so the operator approves one command while a different shell body runs. Disable alias/function snapshots for approval-gated exec, or make approval analysis account for the snapshot-resolved command before execution.
