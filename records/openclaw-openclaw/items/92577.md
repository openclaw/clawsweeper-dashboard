## Summary

Keep open: the linked bug is valid, but this patch is too broad because it suppresses any adjacent same-text assistant entry instead of proving the entry is the cleaned thinking-stripped child of the prior assistant message; the PR also still has only unit-test proof.

## What This Changes

This PR adds a `lastAssistantText` guard to session-memory transcript extraction and adds focused transcript unit tests for duplicate, distinct, and non-consecutive assistant messages.

## Best Possible Solution

Constrain session-memory deduplication to the raw thinking assistant entry plus its cleaned child, add a regression for unrelated adjacent same-text assistant entries, and require redacted live or terminal proof of the memory file path.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: current main extracts identical text from adjacent raw and cleaned assistant entries and appends both to session-memory output. I did not run tests because this review is read-only and tests would create artifacts.

## Solution Assessment

No. Hook-layer dedupe is an acceptable mitigation, but this implementation is not the best fix because it should identify the raw/cleaned parent-child transcript pair instead of collapsing every adjacent identical assistant text.
