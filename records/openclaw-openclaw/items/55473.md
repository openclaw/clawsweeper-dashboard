## Summary

Keep open: the gateway problem is still real on current main, but this PR is not merge-ready because the recovery path can report success without bringing recovered channels back online and can replay stale channel state after operator edits.

## What This Changes

The PR adds startup-only channel secret degradation, recovery snapshot plumbing, merge-patch support, regression coverage, and node/build debugging tweaks.

## Best Possible Solution

Implement channel-only startup secret degradation in the current startup activator, preserve operator edits through recovery, restart recovered channels via the existing reload plan, document the contract, and keep non-channel or permanent secret failures fatal.

## Reproduction Assessment

Yes. Source inspection shows current main still wraps startup SecretRef preparation failures as fatal, and the existing startup secret test locks in that failure path; the user-level repro is an active channel token SecretRef with the backing env var unset.

## Solution Assessment

No. The direction is maintainable, but this branch is not the best current implementation because recovery needs to use the current channel restart/rollback seam and preserve operator edits made during the degraded window.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Restart recovered channels during secrets reload:** `src/gateway/server.impl.ts:1348-1352`
  - body: In the degraded-startup path the channel was never started because the active snapshot has `enabled: false`. This reload path can swap to the recovery config and report success after secret activation while the channel runtime remains offline, especially with the channel health monitor disabled or delayed. Route recovery through the existing reload-plan stop/start path.
  - confidence: 0.88
- **[P2] Preserve explicit channel disables in recovery config:** `src/secrets/runtime.ts:307-308`
  - body: When an operator sets `channels.<id>.enabled=false` during a degraded session, both the degraded source config and new source config can already contain `enabled: false`, so this delta is empty and `recoveryConfig` can keep the old enabled state. A later `secrets.reload` may re-enable a channel the operator explicitly disabled; track disable writes or merge against the pre-degrade baseline.
  - confidence: 0.78
- **[P3] Document the startup degradation contract:** `src/gateway/server.impl.ts:608-621`
  - body: This changes the operator-facing activation contract from startup fail-fast to channel-local degradation for one class of SecretRef failure, but current docs still say startup failure aborts and there is no changelog entry. Add docs and changelog coverage so recovery behavior is discoverable.
  - confidence: 0.72
