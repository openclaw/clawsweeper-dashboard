## Summary

Keep open: current main still fails startup on missing active-surface SecretRefs, and the PR addresses a real gateway problem, but this branch is stale against the current gateway split and still has recovery correctness gaps before merge.

## What This Changes

The PR adds startup-only channel SecretRef degradation, a recovery snapshot field, merge-patch support, and regression coverage for recovering a disabled channel via `secrets.reload`.

## Best Possible Solution

Implement channel-only startup SecretRef degradation in the current split startup activator, keep non-channel and policy failures fatal, preserve operator edits through recovery, restart recovered channels through the existing reload plan, and document the new contract.

## Reproduction Assessment

Yes. Source inspection shows current main still turns startup SecretRef preparation failures into the normalized fatal startup error, and the existing startup secret test asserts that behavior.

## Solution Assessment

No. The direction is appropriate, but this PR is not the best current implementation because recovery bypasses the current channel restart/rollback seam and can replay a stale recovery baseline over operator disables.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Restart recovered channels during secrets reload:** `src/gateway/server.impl.ts:1348-1352`
  - body: In the degraded-startup path the channel was never started because the active snapshot has `enabled: false`. This reload path can swap to the recovery config and report success after secret activation while the channel runtime remains offline, especially with the channel health monitor disabled or delayed; route recovery through the existing reload-plan stop/start path.
  - confidence: 0.88
- **[P2] Preserve explicit channel disables in recovery config:** `src/secrets/runtime.ts:307-308`
  - body: When an operator sets `channels.<id>.enabled=false` during a degraded session, both the degraded source config and new source config can already contain `enabled: false`, so this delta is empty and `recoveryConfig` can keep the old enabled state. A later `secrets.reload` may re-enable a channel the operator explicitly disabled; track disable writes or merge against the pre-degrade baseline.
  - confidence: 0.78
- **[P3] Document the startup degradation contract:** `src/gateway/server.impl.ts:608-621`
  - body: This changes the operator-facing activation contract from startup fail-fast to channel-local degradation for one class of SecretRef failure, but the gateway secrets docs still say startup failure aborts and there is no changelog entry. Add docs and changelog coverage so recovery behavior is discoverable.
  - confidence: 0.72
