## Summary

Keep open. Current main still fails gateway startup on unresolved active-channel SecretRefs, so the underlying bug is real, but this PR is dirty against main and still has blocking recovery bugs in the degraded-startup reload path.

## What This Changes

The branch adds channel-only startup secret degradation, a recoveryConfig snapshot for secrets.reload recovery, merge-patch support, and gateway reload regression coverage.

## Best Possible Solution

Rework the degradation on current main so only missing active-channel env secrets degrade startup, recovery preserves operator edits, and secrets.reload uses the existing channel restart and rollback plan.

## Reproduction Assessment

Yes. Configure an active channel such as Discord with an env-backed token SecretRef, leave the env var unset, and start the gateway; current main's startup secret activator still follows the fatal startup path, and existing tests assert that behavior.

## Solution Assessment

No. The direction is maintainable, but this branch is not the best current implementation because it predates the current reload handler split and can leave recovered channels offline or re-enable channels that an operator explicitly disabled.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Restart recovered channels during secrets reload:** `src/gateway/server.impl.ts:1348-1352`
  - body: In the degraded-startup path the channel was never started because the active snapshot has `enabled: false`. This reload path swaps to `recoveryConfig` and returns after secret activation, so once the env secret appears `secrets.reload` can report success while the channel runtime remains offline. Route recovery through the current reload-plan stop/start path.
  - confidence: 0.88
- **[P2] Preserve explicit channel disables in recoveryConfig:** `src/secrets/runtime.ts:307-308`
  - body: When an operator sets `channels.<id>.enabled=false` during a degraded session, both the old degraded sourceConfig and the new sourceConfig already contain `enabled: false`, so this delta can be empty and recoveryConfig keeps the old enabled state. A later `secrets.reload` can re-enable a channel the operator disabled; merge against the pre-degrade baseline or explicitly track disable writes.
  - confidence: 0.78
- **[P3] Document the new startup degradation path:** `src/gateway/server.impl.ts:608-621`
  - body: This changes the documented activation contract from startup fail-fast to channel-local degradation for one class of SecretRef failure, but the PR does not update docs or CHANGELOG. Add the operator-facing docs and changelog entry so recovery behavior is clear.
  - confidence: 0.72
