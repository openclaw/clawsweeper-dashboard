## Summary

Keep open. Current main still has the reported Unix/macOS stale-gateway PID discovery bug, and the issue is already paired with open, unmerged PR #70681 using closing linkage, so the issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue reports that macOS stale gateway cleanup drops Node-backed `openclaw-gateway` listener PIDs because Unix `lsof -Fpc` output is filtered by the `c` command token before full argv verification.

## Best Possible Solution

Land PR #70681 or an equivalent narrow fix that treats lsof as raw listener-PID discovery, verifies Unix/macOS candidates via full argv inspection, preserves self and ancestor exclusions, and updates regression coverage for `cnode` gateway listeners and unrelated non-gateway listeners.

## Reproduction Assessment

Yes. The issue's `lsof -Fpc` sample with a gateway PID reported as `cnode` deterministically reproduces against current source because the parser drops that PID before any argv verifier can run.

## Solution Assessment

Yes. The proposed direction is the narrowest maintainable fix because it mirrors the Windows command-line verification path and avoids using lsof's kernel command field as the identity signal for Node-based gateway processes.
