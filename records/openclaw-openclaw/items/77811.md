## What This Changes

The PR adds ISO timestamp text to repeated cron failure/skipped alerts and schedule-error auto-disable notifications, plus updates failure-alert test assertions.

## Best Possible Solution

Revise this PR to thread an explicit cron event/run timestamp through repeated failure/skipped alerts, single failure-destination announce/webhook messages, and auto-disable notices, with focused tests and contributor-supplied patched runtime proof.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is an additive feature request. Source inspection gives high confidence that current main's repeated-alert, single-failure, and auto-disable templates omit event-time text while cron events carry run metadata.

## Solution Assessment

No. This PR should use explicit event times such as `startedAt`/`endedAt` or `evt.runAtMs` and update all affected alert paths, rather than formatting `state.deps.nowMs()` in only part of the surface.
