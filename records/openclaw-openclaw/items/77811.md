## Summary

Keep open: the direction is useful, but the PR is still incomplete because it stamps repeated alerts with application time, leaves the single-failure destination alert unchanged, and lacks after-fix real behavior proof plus the required changelog entry.

## What This Changes

The PR adds ISO 8601 timestamps to repeated cron failure/skipped alerts and schedule-error auto-disable notifications, and updates one failure-alert test expectation.

## Best Possible Solution

Land a narrow cron change that uses explicit event/run timestamps for single-failure, repeated failure/skipped, and auto-disable notifications, with regression tests, changelog coverage, and real patched-output proof.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main verifies the relevant cron notification templates omit event timestamps today.

## Solution Assessment

No. A maintainable fix should thread explicit event times such as result.endedAt or evt.runAtMs into every affected alert body rather than formatting nowMs() in only part of the surface.
