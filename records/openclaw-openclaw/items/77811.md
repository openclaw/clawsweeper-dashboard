## Summary

Keep open: the PR targets a useful cron-alert enhancement, but the current patch is not merge-ready because it uses emission time for repeated alerts, leaves the single-failure notification path untimestamped, lacks changelog coverage, and still needs patched runtime proof.

## What This Changes

The PR adds ISO 8601 timestamps to repeated cron failure/skipped alert text and schedule-error auto-disable notifications, and updates one failure-alert test expectation.

## Best Possible Solution

Thread explicit run/event timestamps through every cron failure notification path, cover repeated alerts, single-failure delivery, and auto-disable notifications with regression tests, add changelog coverage, and require patched-output proof before merge.

## Reproduction Assessment

Not applicable as a feature request. Source inspection confirms current main's repeated alert, single-failure notification, and auto-disable templates omit event-time text.

## Solution Assessment

No. The patch should use explicit event times such as result.endedAt or evt.runAtMs and update all affected alert paths instead of formatting state.deps.nowMs in only part of the surface.
