## Summary

Keep open. Current main has solved important discovery, watcher, install-policy, verification, and diagnostic pieces, but the umbrella still contains unresolved product/security decisions for skill permission manifests, signing/trust semantics, post-install audit hooks, trigger/composition metadata, agent packs, and external ClawHub publish ownership.

## What This Changes

The issue asks OpenClaw maintainers to close the gap between community skills and ClawHub’s promised workflow across discovery, publishing, refresh, security, composition, and sharing.

## Best Possible Solution

Keep this as a consolidation tracker or split it into maintained narrow trackers: mark discovery, watcher, install policy, and verification pieces as solved; route standalone ClawHub publish/service bugs to the ClawHub owner; and define OpenClaw-side contracts for permission/trust manifests, post-install audit hooks, trigger/composition metadata, and agent-pack packaging.

## Reproduction Assessment

Yes, for the remaining source-level gaps: current metadata and frontmatter code do not define a permission/trust manifest, and current hook/docs evidence shows pre-install policy without a complete post-install audit contract. No single live reproduction covers the whole broad ecosystem umbrella.

## Solution Assessment

Unclear as one implementation. The maintainable path is to keep the solved plumbing, route external ClawHub publishing to the right owner, and have maintainers choose one public security/schema design before new core or plugin API work starts.
