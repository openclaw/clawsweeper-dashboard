## Summary

Keep open. Current main has solved important discovery, watcher, and diagnostic pieces, but this remains a broad ecosystem tracker with unresolved OpenClaw-side skill trust, direct ClawHub install scanning, permission/provenance, trigger-quality, and external ClawHub publish/service decisions.

## What This Changes

The issue asks OpenClaw to make community skills and ClawHub reliable across discovery, publishing, watching, trust/security, composition, and ecosystem workflows.

## Best Possible Solution

Keep this as a consolidation tracker or split it into maintained narrow trackers: mark discovery, watcher, and skills-check diagnostics as solved; route pure ClawHub service/publish bugs to ClawHub; and scope OpenClaw API/core work for direct install scanning, permission/provenance manifests, lifecycle hooks, and trigger/lint schema.

## Reproduction Assessment

No high-confidence reproduction exists for the full umbrella because it combines feature requests, security policy, and external service behavior. Individual remaining gaps are source-verifiable: direct ClawHub skill install still bypasses the install scan hook, and the proposed manifest/trigger fields are not implemented in the current skill contract.

## Solution Assessment

No single code change is the best way to solve this issue. The maintainable path is to split solved diagnostics from unresolved security/API and ClawHub service work, then land narrow designs or PRs per surface.
