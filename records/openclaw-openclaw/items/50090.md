## Summary

Keep open. Current main has addressed major discovery, watcher, ClawHub install, and agent-visibility diagnostics, but the umbrella still has unresolved OpenClaw-side skill trust, permission/capability manifest, native ClawHub install policy, post-install audit, trigger-quality, and product-boundary work that is not fully superseded by one canonical tracker.

## What This Changes

The issue asks OpenClaw to mature community skills and ClawHub around discovery, publishing, watcher refresh, trust/security, composition, trigger quality, and agent-pack workflows.

## Best Possible Solution

Keep this as a consolidation tracker or split it into maintained narrow trackers: mark discovery, watcher, and diagnostics as solved; route pure ClawHub publish/service bugs to ClawHub-owned code; and define OpenClaw API work for native install scanning, permission/provenance manifests, lifecycle/audit hooks, trigger/lint schema, and any accepted agent-pack boundaries.

Not applicable. This is a broad feature/security ecosystem tracker rather than a single runtime bug, and static inspection gives high-confidence evidence of which requested surfaces exist and which remain absent.

Yes for the current review outcome: keeping this open for maintainer scoping is the narrowest maintainable path. A one-shot automated fix would have to choose public trust, manifest, ClawHub, hook, trigger, CLI, and packaging contracts prematurely.
