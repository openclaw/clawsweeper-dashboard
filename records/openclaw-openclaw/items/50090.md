## Summary

Keep open. Current main has solved important discovery, watcher, and agent-visibility diagnostics, but the umbrella still has unresolved OpenClaw-side skill trust, native ClawHub install policy, permission/provenance, trigger-quality, external ClawHub publish, and agent-pack/product-scope work that is not fully superseded by narrower trackers.

## What This Changes

The issue asks OpenClaw to close gaps in community skill discovery, ClawHub publishing, watcher refresh, trust/security policy, trigger quality, composition, and agent-pack workflows.

## Best Possible Solution

Preserve this as a consolidation tracker or split it into maintained narrow trackers: mark discovery, watcher, and diagnostics as solved, route pure ClawHub publish/service bugs to ClawHub-owned code, and define OpenClaw API work for native install scanning, permission/provenance manifests, lifecycle/audit hooks, trigger/lint schema, and agent-pack boundaries.
