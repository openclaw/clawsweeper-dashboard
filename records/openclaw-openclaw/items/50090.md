## Summary

Keep open. Current main has addressed several discovery, watcher, and diagnostics pieces, but the issue remains a broad ecosystem/security tracker with unresolved OpenClaw-side direct install scanning, permission/provenance manifest, trigger-quality, and external ClawHub publish/service work.

## What This Changes

Request maturation of community skills and ClawHub across skill discovery, publish reliability, trust/scanning, invocation quality, and composition.

## Best Possible Solution

Keep this as a consolidation tracker or split it into maintained narrow trackers: mark discovery, watcher, and skills-check diagnostics as solved; route pure ClawHub publish/service bugs to ClawHub; and scope OpenClaw API/core work for direct install scanning, permission/provenance manifests, lifecycle hooks, and trigger/lint schema.
