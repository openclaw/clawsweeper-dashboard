Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep DeepSeek V4 non-native compatibility guards:** `src/agents/embedded-agent-runner/extra-params.ts:831`
  - body: This replaces the current compat-aware DeepSeek V4 wrapper with an unconditional model-id match and removes the sanitizer, while deleting the regression tests that protected `microsoft-foundry-*` aliases and `compat.thinkingFormat: "openai"`. Those routes can again receive DeepSeek-native `thinking`/`reasoning_content` fields that current main intentionally suppresses for Foundry/OpenAI-compatible deployments.
  - confidence: 0.9
- **[P1] Preserve the browser profile SDK type surface:** `src/plugin-sdk/browser-types.ts:62-73`
  - body: The new `ResolvedBrowserProfile` copy omits fields that `resolveProfile()` still returns, including `mcpCommand`, `mcpArgs`, `executablePath`, and `headlessSource`. External TypeScript callers importing this public SDK type lose supported browser profile fields, so keep this facade type in parity with the runtime result or derive it from the source type.
  - confidence: 0.86
- **[P2] Validate raw chunk sizing before slicing media:** `src/plugin-sdk/outbound-media.ts:182-186`
  - body: `rawChunkBytes` is a public helper option but only max entry counts are validated. A non-positive or non-integer chunk size can compute an invalid chunk count and slice buffers with corrupted boundaries while metadata still records the original byte length, so reject invalid chunk sizes before `prepareUrl()` stores rows.
  - confidence: 0.84
- **[P2] Keep gateway readiness scanning incremental:** `scripts/lib/openclaw-e2e-instance.sh:380-383`
  - body: This reverts the current-main incremental log scan back to grepping the whole gateway log on every wait loop and deletes the tests that covered large logs, split ready markers, and ready-port parsing. On large gateway logs this can reintroduce the readiness stalls the current helper was changed to avoid.
  - confidence: 0.82
- **[P2] Restore bounded Docker E2E log printing:** `scripts/lib/docker-e2e-logs.sh:79-81`
  - body: `docker_e2e_print_log()` now cats the entire log file and removes the configured byte cap plus its regression test. Long or noisy Docker runs can flood CI logs and artifact output; keep the bounded tail behavior unless this PR proves the full-log behavior is required.
  - confidence: 0.78
