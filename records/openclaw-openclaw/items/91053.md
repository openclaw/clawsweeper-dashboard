## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Restore the DeepSeek non-native compatibility guards:** `src/agents/embedded-agent-runner/extra-params.ts:831`
  - body: This line now applies the DeepSeek V4 native thinking wrapper to every OpenAI-compatible DeepSeek V4 model except the exact `microsoft-foundry` provider. The base branch also excluded `microsoft-foundry-*` aliases and honored `compat.thinkingFormat !== "deepseek"` while stripping replay-only `reasoning_content`; removing those guards can make Azure/Foundry-compatible turns send rejected `thinking` or replay fields again.
  - confidence: 0.93
- **[P2] Serialize capacity pruning with chunk writes:** `src/plugin-sdk/outbound-media.ts:258`
  - body: `pruneForCapacity()` runs before the chunk rows are registered, but plugin-state enforces `maxEntries` independently after each `register()`. Two concurrent hosted-media preparations near the chunk-row limit can both pass pruning, then one call's chunk inserts can evict chunks from the other just-returned URL before metadata is complete, causing a valid media reply URL to read as missing/corrupt.
  - confidence: 0.86
- **[P2] Validate rawChunkBytes before slicing:** `src/plugin-sdk/outbound-media.ts:252`
  - body: `rawChunkBytes` is a public option but is never checked before `Math.ceil(byteLength / rawChunkBytes)` and `Buffer.subarray(...)`. A zero, negative, non-integer, or non-finite value can throw late or store truncated/corrupt chunks while metadata still records the original byte length; reject anything except a positive safe integer when creating the store.
  - confidence: 0.9
- **[P2] Keep the incremental gateway readiness scan:** `scripts/lib/openclaw-e2e-instance.sh:380`
  - body: This reverts the base branch back to grepping the entire gateway log on every poll. The removed incremental scanner and tests covered large logs and ready markers split across writes; with this PR also changing gateway-watch CI and current branch comments reporting gateway-watch timeouts, dropping that guardrail is a regression that should not ride with the Zalo refactor.
  - confidence: 0.88
- **[P2] Keep Docker E2E log output bounded:** `scripts/lib/docker-e2e-logs.sh:81`
  - body: The PR removes the existing `OPENCLAW_DOCKER_E2E_LOG_PRINT_BYTES` cap and prints the full captured Docker E2E log. Long successful or failing container runs can now flood CI logs again; keep the bounded tail behavior and its regression test unless this automation policy change is intentional and separately reviewed.
  - confidence: 0.82
