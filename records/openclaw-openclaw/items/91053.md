## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Validate rawChunkBytes before slicing hosted media:** `src/plugin-sdk/outbound-media.ts:182`
  - body: `rawChunkBytes` is part of the new public helper options but is never validated. Passing `0`, a negative number, or a non-integer can make `chunkCount` and `Buffer.subarray()` disagree with `byteLength`, producing corrupt rows or unexpected failures instead of a clear SDK error.
  - confidence: 0.91
- **[P2] Serialize capacity pruning before writing chunks:** `src/plugin-sdk/outbound-media.ts:258-263`
  - body: `prepareUrl()` prunes capacity before writing chunks, but each `chunkStore.register()` independently enforces `maxEntries` and protects only that one row. Concurrent preparations near the chunk-row limit can evict chunks from a just-returned URL while metadata remains, causing `read()` to return null and the Zalo route to serve 404.
  - confidence: 0.84
- **[P2] Preserve resolved browser profile fields:** `src/plugin-sdk/browser-types.ts:62-72`
  - body: The new SDK copy of `ResolvedBrowserProfile` omits fields still returned by the browser runtime, including `mcpCommand`, `mcpArgs`, `executablePath`, and `headlessSource`, and it makes `headless` optional. External TypeScript callers importing this facade lose access to supported runtime fields, so keep the facade type in parity with the browser plugin result.
  - confidence: 0.9
