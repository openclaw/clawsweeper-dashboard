## Summary

Keep open. Current main still has the reported wrong-root caller: the auto-compaction completion hook reads post-compaction context from `process.cwd()`, while the adjacent preflight compaction path already uses the queued run workspace, so the bug remains valid and narrowly repairable.

## What This Changes

The issue requests fixing post-compaction context injection so the auto-compaction hook reads AGENTS.md from the follow-up run workspace instead of the runner process cwd.

## Best Possible Solution

Use `followupRun.run.workspaceDir` for the auto-compaction completion hook, preserve the existing trusted enqueue and best-effort error handling, and add a regression test proving the hook reads from the queued workspace when cwd differs.

## Reproduction Assessment

Yes. The source-level reproduction is clear: with `autoCompactionCount > 0`, a `sessionKey`, and cwd different from `followupRun.run.workspaceDir`, current main passes cwd into `readPostCompactionContext`.

## Solution Assessment

Yes. Passing the already-required queued run workspace is the narrowest maintainable fix and aligns this hook with the neighboring preflight compaction path without adding config or API surface.
