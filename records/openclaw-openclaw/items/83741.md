## Summary

Keep open for maintainer review: current main and v2026.5.20 still bypass ClickClack `allowFrom` on inbound dispatch, and this PR appears to fix that without blocking findings. The remaining blocker is maintainer acceptance of the intentional fail-closed compatibility and message-delivery impact for restrictive configs.

## What This Changes

The PR adds a ClickClack access resolver that gates gateway/inbound dispatch on `allowFrom` and updates inbound/gateway tests for allowed, denied, and DM-target allowlist cases.

## Best Possible Solution

Land the plugin-local shared ingress enforcement once maintainers accept ClickClack `allowFrom` as a hard inbound boundary, with release-note or docs follow-through if they want the restrictive-config impact called out.

## Solution Assessment
