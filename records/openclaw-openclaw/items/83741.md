## Summary

The PR should stay open for maintainer review: the patch is a focused fix for a real ClickClack authorization gap, with sufficient local runtime proof, and no blocking code finding from this review.

## What This Changes

The PR adds a ClickClack inbound access resolver, gates gateway and inbound handling on `allowFrom`, derives `CommandAuthorized` from the shared ingress decision, and adds focused gateway/inbound regression tests.

## Best Possible Solution

Land the plugin-local shared-ingress enforcement once maintainers accept the intentional fail-closed `allowFrom` upgrade behavior, with release-note or docs follow-through during landing if they want the behavior called out.

## Solution Assessment

Yes for the code direction: using the shared channel ingress resolver inside the ClickClack plugin keeps the authorization decision at the owning plugin boundary. The remaining decision is whether maintainers accept the deliberate fail-closed upgrade behavior for restrictive `allowFrom` configs.
