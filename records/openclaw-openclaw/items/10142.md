## Summary

Keep open. Current main still lacks the requested internal `session:end` hook event; the existing `session_end` lifecycle support is a plugin hook on a separate documented surface, and open PRs #56672 and #61675 still reference this issue as implementation candidates.

## What This Changes

The issue requests an internal `session:end` hook event that fires when an agent session completes so external workflow systems can react without agent-authored cleanup commands.

## Best Possible Solution

Define the intended contract explicitly: either add a typed internal `session:end` event with tests and docs across the agreed completion paths, or document that plugin `session_end` is the supported replacement.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is a feature/API gap. The absence is high-confidence from current-main source and docs inspection of internal hook types, internal hook docs, and completion paths.

## Solution Assessment

No; current main does not solve the requested internal hook contract. The maintainable path is either a narrow internal-hook implementation with focused tests/docs or an explicit maintainer-documented decision to use plugin `session_end` instead.
