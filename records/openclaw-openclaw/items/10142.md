## Summary

Keep open. Current main still lacks the requested internal `session:end` hook event; the existing `session_end` lifecycle hook is a separate plugin API, and open implementation PRs still reference this issue as candidate fixes.

## What This Changes

Request an internal `session:end` hook event that fires when an agent session completes so external workflow orchestrators can react without agent-authored cleanup commands.

## Best Possible Solution

Define the supported lifecycle contract explicitly: either add a typed internal `session:end` event with focused tests and docs across the agreed completion paths, or document that plugin `session_end` is the supported integration surface instead.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is a feature/API gap. The absence is high-confidence from current-main source and docs inspection of internal hook types, internal hook docs, and adjacent plugin lifecycle hooks.

## Solution Assessment

No; current main does not solve the requested internal hook contract. The maintainable path is either a narrow internal-hook implementation with tests/docs or an explicit maintainer-documented decision to use plugin `session_end` instead.
