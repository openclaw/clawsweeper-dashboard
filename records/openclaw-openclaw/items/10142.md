## Summary

Keep open. Current main and v2026.6.10 still lack the requested internal `session:end` hook event; the existing `session_end` support is the typed plugin hook surface, and the open linked PR is a narrower reset/session-memory candidate rather than a merged implementation of this contract.

## What This Changes

Add an internal `session:end` hook event so operator hooks can react when sessions complete.

## Best Possible Solution

Maintainers should choose the permanent contract: add and document a typed internal `session:end` event across the relevant lifecycle paths, or explicitly document plugin `session_end` as the supported replacement and retarget the linked reset/memory work accordingly.

## Reproduction Assessment

Yes, at source level: current main has plugin `session_end` lifecycle hooks, but internal hook docs/types and inspected completion paths do not define or emit the requested `session:end` event. This is a feature/API gap rather than a broken existing behavior repro.

## Solution Assessment

Unclear. The narrowest maintainable solution depends on maintainer direction: either add the internal hook contract, or make the typed plugin `session_end` hook the documented supported surface for this use case.
