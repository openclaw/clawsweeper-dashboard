## Summary

Keep open. Current main still lacks the requested internal `session:end` hook event; the existing lifecycle surface is the separate typed plugin `session_end` hook, and open implementation candidates at https://github.com/openclaw/openclaw/pull/56672 and https://github.com/openclaw/openclaw/pull/61675 still reference this issue with closing syntax.

## What This Changes

Request to add an internal `session:end` hook event that fires when a session completes so workflow orchestration can react without an explicit agent command.

## Best Possible Solution

Maintain one supported lifecycle contract: either land a narrowed internal `session:end` implementation with docs/tests across the relevant completion paths, or explicitly document plugin `session_end` as the supported replacement and resolve the linked PRs.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection on current main and v2026.5.12 confirms the requested internal event is absent while the adjacent plugin `session_end` lifecycle hook exists.

## Solution Assessment

Unclear. A direct internal hook would satisfy the request, but the maintainable solution depends on maintainer selection between the internal hooks contract and the existing plugin lifecycle hook surface.
