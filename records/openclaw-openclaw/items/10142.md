## Summary

Keep open: current main still lacks the requested internal `session:end` HOOK.md event, and the existing `session_end` implementation is a separate typed plugin hook. The open linked PR covers daily/idle reset hook dispatch, not the broader internal session-completion contract requested here.

## What This Changes

The issue requests a new internal `session:end` hook event that fires when a session completes so workflow orchestration can react without agent-authored cleanup commands.

## Best Possible Solution

Maintainers should choose and document the supported contract: either add a narrow internal `session:end` event with tests across completion paths, or declare typed plugin `session_end` the supported replacement and close this request with that guidance.

## Reproduction Assessment

Yes, at source level: current main exposes `session:patch` internally and `session_end` as a typed plugin hook, but no documented or typed internal `session:end` event exists.

## Solution Assessment

Unclear. Adding an internal event is plausible, but the existing typed plugin `session_end` surface may be the better owner, so maintainers should decide the supported hook contract before implementation.
