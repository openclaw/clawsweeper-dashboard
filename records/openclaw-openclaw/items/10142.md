## Summary

Keep open. Current main and v2026.5.22 still lack the requested internal `session:end` hook; the available `session_end` lifecycle signal is a typed plugin hook, and two open conflicting PRs still claim this issue with different lifecycle/API shapes.

## What This Changes

Requested addition of an internal `session:end` hook event that fires when an agent session completes so external workflow orchestration can be signaled without an agent-authored final command.

## Best Possible Solution

Establish one supported session-completion lifecycle contract, then either land an internal `session:end` event with typed guard, docs, and tests across completion paths or document and extend typed plugin `session_end` as the supported replacement.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source and docs inspection give high confidence that current main lacks the requested internal event while exposing adjacent typed plugin lifecycle hooks.

## Solution Assessment

Unclear. A session completion signal is reasonable, but maintainers need to choose whether the maintainable API is an internal `HOOK.md` event or the existing typed plugin lifecycle hook before accepting one implementation path.
