## Summary

Keep open. Current main still lacks the requested internal `session:end` hook contract; the existing `session_end` surface is a plugin lifecycle hook, and open implementation PRs still reference this issue with closing syntax.

## What This Changes

The issue requests an internal `session:end` hook event that fires when an agent session completes so external workflow orchestrators can react without agent-authored cleanup commands.

## Best Possible Solution

Define the supported session-completion lifecycle contract, then land one canonical implementation with docs and focused tests or explicitly document plugin `session_end` as the supported replacement.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is a feature/API contract gap. Source and docs inspection on current main gives high-confidence evidence that internal hooks still lack `session:end`.

## Solution Assessment

Unclear. The requested internal hook may be the right solution, but maintainers should first choose between adding internal `session:end` and treating plugin `session_end` as the supported integration surface.
