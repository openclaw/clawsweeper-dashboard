## Summary

Keep open: current main and the latest release still lack a documented internal `session:end` hook event, while the existing `session_end` implementation is a typed plugin hook and the linked implementation path is still open/unmerged.

## What This Changes

Request adds a `session:end` internal hook event that fires when agent sessions complete so external workflow orchestrators can react without agent-issued CLI commands.

## Best Possible Solution

Keep this issue open until maintainers either land or close the linked implementation path and explicitly choose the internal-hook versus plugin-hook lifecycle contract.

## Reproduction Assessment

Yes, at source level. Current main has typed plugin `session_end` lifecycle hooks, but the internal hook registry and public internal hooks docs still lack a `session:end` event.

## Solution Assessment

Unclear. Adding an internal `session:end` event is plausible, but the existing typed plugin `session_end` surface may be the better long-term contract unless maintainers explicitly want HOOK.md operator scripts to observe session completion.
