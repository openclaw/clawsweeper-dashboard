## Summary

Keep open. Current main still lacks the requested internal `session:end` HOOK.md event; the existing `session_end` support is a typed plugin hook on a separate surface, and the linked implementation PRs remain open rather than merged fixes.

## What This Changes

The issue requests a new internal `session:end` hook event that fires when an agent session completes so external orchestration can react without agent-authored cleanup commands.

## Best Possible Solution

Choose the supported session-completion contract, then either land a focused internal `session:end` implementation with docs/tests or document typed plugin `session_end` as the supported replacement.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/API request. Source and docs checks show current main and v2026.6.5 still omit an internal `session:end` event while exposing typed plugin `session_end`.

## Solution Assessment

Unclear. Adding the internal HOOK.md event is plausible, but maintainers should first decide whether typed plugin `session_end` is the cleaner core-lean contract or whether a parallel internal hook is required.
