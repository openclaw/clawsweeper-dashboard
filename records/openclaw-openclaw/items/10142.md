## Summary

Keep open. Current main still lacks the requested operator internal `session:end` hook; the existing `session_end` lifecycle surface is a separate plugin hook, and open implementation PRs with closing references are still pending.

## What This Changes

Request adding an internal `session:end` hook event when sessions complete so external workflow orchestrators can signal completion without agent-authored cleanup commands.

## Best Possible Solution

Define the supported session-completion lifecycle contract, then land one canonical implementation with docs and focused tests, or explicitly document plugin `session_end` as the supported replacement and close the competing PRs.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is a feature/API gap. Source and docs inspection on current main gives high-confidence evidence that the internal hook contract still lacks `session:end`.

## Solution Assessment

Unclear. The requested internal hook may be the right operator contract, but maintainers should first choose between adding `session:end` to internal hooks and treating plugin `session_end` as the supported integration surface.
