## Summary

Keep open. Current main and v2026.6.5 still omit the requested internal `session:end` HOOK.md event; the available `session_end` signal is a typed plugin hook, and https://github.com/openclaw/openclaw/pull/61675 is still an open linked implementation candidate that needs maintainer API-direction review.

## What This Changes

Add an internal `session:end` hook event so external workflow orchestrators can react when an OpenClaw session completes.

## Best Possible Solution

Define one supported session-completion contract, then either land a focused internal `session:end` implementation with docs/tests or document typed plugin `session_end` as the replacement and retarget the linked candidates accordingly.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/API request. Source and docs checks show current main and v2026.6.5 still omit an internal `session:end` event while exposing typed plugin `session_end`.

## Solution Assessment

Unclear. A direct internal hook would satisfy the request, but OpenClaw already has a typed plugin lifecycle hook and maintainers need to decide which surface should be the supported session-completion contract.
