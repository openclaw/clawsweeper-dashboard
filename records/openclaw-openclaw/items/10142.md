## Summary

Keep open. Current main still lacks the requested internal `session:end` hook contract; the existing `session_end` support is a plugin lifecycle hook, and open implementation candidates still need maintainer selection before this issue can close: https://github.com/openclaw/openclaw/pull/56672 and https://github.com/openclaw/openclaw/pull/61675.

## What This Changes

Add an internal `session:end` hook event that fires when an agent session completes so external workflow orchestrators can react without agent-authored cleanup commands.

## Best Possible Solution

Keep this issue open until maintainers land one canonical implementation with docs and focused tests, or explicitly document plugin `session_end` as the supported replacement and close the overlapping PRs.
