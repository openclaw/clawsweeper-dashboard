## Summary

Keep open. Current main still lacks the requested internal `session:end` hook contract; the adjacent `session_end` implementation is a typed plugin hook, and the linked open pull request is a human-review candidate rather than a merged fix.

## What This Changes

The issue requests an internal `session:end` hook event that fires when an agent session completes so external workflow systems can react without agent-authored cleanup commands.

## Best Possible Solution

Keep this issue open until maintainers either land a refreshed implementation, explicitly reject the internal hook in favor of the typed plugin `session_end` contract, or document the supported session-completion automation path in the internal hooks docs.

## Reproduction Assessment

Yes, at source level. Current main defines and documents `session:patch` for internal hooks while `session:end` exists only as a typed plugin hook, and cron/subagent completion paths do not emit an internal `session:end` event.

## Solution Assessment

Unclear as a final product path. Adding the internal event is plausible, but the existing typed plugin `session_end` surface may be the better supported boundary if maintainers explicitly document it for this workflow use case.
