## Summary

Current main still lacks the requested internal `session:end` contract, and the only linked OpenClaw PR remains open and conflicting while targeting lazy reset hooks rather than the broader run-completion semantics; this needs a maintainer product/API decision, not cleanup closure.

## What This Changes

Add an internal `session:end` hook emitted when an agent session completes so workflow orchestration can react without explicit agent cleanup commands.

## Best Possible Solution

Define separate, documented lifecycle boundaries for persisted-session termination and finite run completion, reuse typed plugin hooks where possible, and add an operator internal hook only if maintainers explicitly want `HOOK.md` scripts to observe one of those boundaries.

## Reproduction Assessment

## Solution Assessment
