## Summary

Keep open. Current main still lacks the requested internal `session:end` hook contract, the existing `session_end` support is a separate plugin hook surface, and open implementation PRs with closing references still target this issue.

## What This Changes

The issue requests an internal `session:end` hook event that fires when an agent session completes so external workflow systems can react without agent-authored cleanup commands.

## Best Possible Solution

Define the supported session-completion lifecycle contract, then either land one canonical implementation with docs and focused tests or document plugin `session_end` as the supported replacement and close the competing PRs.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is a feature/API gap. Source and docs inspection on current main gives high-confidence evidence that the internal hook contract still lacks `session:end`.

## Solution Assessment

Unclear. The requested internal hook may be the right contract, but maintainers should first choose between adding internal `session:end` and treating the existing plugin `session_end` lifecycle hook as the supported extension point.
