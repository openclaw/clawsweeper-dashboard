## Summary

Keep open. Current main and the latest release still lack a true internal `session:end` hook event; the existing `session_end` lifecycle surface is a typed plugin hook, and the open linked PR still needs maintainer hook-contract judgment rather than making this issue obsolete.

## What This Changes

Request a new internal `session:end` hook event for session completion so operator hooks can signal workflow orchestrators without agent-instruction workarounds.

## Best Possible Solution

Keep this issue as the canonical decision point until maintainers either land a refreshed implementation or document the typed plugin `session_end` hook as the supported replacement for internal session-completion automation.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes, at source level: current internal hook types and docs have `session:patch` but no internal `session:end`, while the only session-completion hook is the typed plugin `session_end` surface.

## Solution Assessment

Is this the best way to solve the issue? Unclear: adding an internal event is plausible, but current plugin hooks already cover part of the lifecycle, so maintainers should decide the supported contract before a new fix lane starts.
