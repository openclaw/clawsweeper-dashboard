## Summary

Keep open. Current main still lacks the requested internal `session:end` hook event; the existing `session_end` lifecycle hook is a plugin API, and open implementation PRs #56672 and #61675 still reference this issue with closing syntax rather than being merged fixes.

## What This Changes

Request an internal `session:end` hook event that fires when sessions complete so external workflow orchestrators can be signaled without agent instructions.

## Best Possible Solution

Define the supported lifecycle contract, then either land a narrow internal `session:end` implementation with focused tests and docs for the agreed completion paths or document plugin `session_end` as the supported replacement.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is a feature/API gap. Source and docs inspection on current main gives high-confidence evidence that the internal hook contract still lacks `session:end`.

## Solution Assessment

No, current main is not yet the best solution because it has only the adjacent plugin `session_end` surface. The maintainable path is to resolve the open implementation PRs or make and document an explicit product decision to use plugin hooks instead.
