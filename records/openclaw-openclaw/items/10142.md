## Summary

Keep open. Current main still lacks the requested internal `session:end` hook contract; the existing `session_end` behavior is a separate plugin lifecycle hook, and open implementation PRs still reference this issue with closing syntax.

## What This Changes

Request an internal `session:end` hook event that fires when an agent session completes for workflow orchestration integrations such as Temporal.

## Best Possible Solution

Define the supported session lifecycle contract, then either land a focused internal `session:end` implementation with docs/tests for agreed completion paths or document plugin `session_end` as the supported replacement and close the competing implementation PRs.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is a feature/API gap. Source and docs inspection on current main gives high-confidence evidence that the internal hook contract still lacks `session:end`.

## Solution Assessment

No, current main is not the best end state because it only provides the adjacent plugin `session_end` hook. The narrow maintainable path is to resolve the open implementation PRs or make and document an explicit product decision to use plugin hooks instead.
