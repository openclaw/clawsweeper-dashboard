## Summary

Keep open. Current main still lacks the requested internal `session:end` hook event and documentation; the existing `session_end` support is a typed plugin hook on a separate surface, and open implementation candidates #56672 and #61675 still reference this issue with closing syntax.

## What This Changes

Request to add an internal `session:end` hook event that fires when an agent session completes so external workflow orchestrators can react without relying on agent instructions.

## Best Possible Solution

Keep this issue open while maintainers review #56672 and #61675 or make an explicit product decision. A direct fix should add a typed internal `session:end` event and guard, wire the agreed completion paths, and update focused tests plus the public internal hooks docs; if plugin hooks are the intended replacement, document that contract clearly.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is a feature/API gap. The absence is high-confidence from source and docs inspection of the internal hook types, event table, and completion paths on current main.

## Solution Assessment

No; current main does not solve the requested internal hook contract. The maintainable path is either a narrow internal-hook implementation with tests/docs or an explicit maintainer-documented decision that the typed plugin `session_end` hook is the replacement.
