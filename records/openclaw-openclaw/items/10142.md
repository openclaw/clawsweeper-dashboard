## Summary

Keep open. Current main and the latest release still lack the requested internal `session:end` hook event; the adjacent `session_end` implementation is a typed plugin lifecycle hook, and the open linked PRs remain implementation candidates with unresolved API direction.

## What This Changes

Requested addition of an internal `session:end` hook event that fires when an agent session completes so external workflow systems can be signaled without agent instructions.

## Best Possible Solution

Establish one supported session-completion lifecycle contract, then either land an internal `session:end` event with docs and coverage across completion paths or document typed plugin `session_end` as the supported replacement.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source, docs, tests, release, and related-PR inspection give high confidence that current main lacks the requested internal event while exposing adjacent typed plugin lifecycle hooks.

## Solution Assessment

Unclear. A session completion signal is reasonable, but maintainers need to choose whether the maintainable API is an internal `HOOK.md` event or the existing typed plugin lifecycle hook before accepting an implementation path.
