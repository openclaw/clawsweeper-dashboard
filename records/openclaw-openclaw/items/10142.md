## Summary

Keep open. Current main and the latest release still lack the requested internal `session:end` hook; the existing `session_end` support is the typed plugin hook surface, and the live linked implementation PR is still open, conflicting, and narrower than the requested contract.

## What This Changes

Add an internal `session:end` hook event that fires when an agent session completes so workflow orchestration can react without agent-authored commands.

## Best Possible Solution

Keep this issue open until maintainers either land or close the linked implementation path, or explicitly document that typed plugin `session_end` is the supported replacement for internal-hook automation.

## Reproduction Assessment

Yes, at source level: current main has plugin `session_end` lifecycle hooks, but the internal hook registry and internal hooks docs do not define or emit the requested `session:end` event. This is a feature gap rather than a broken existing behavior repro.

## Solution Assessment

Unclear. The narrowest maintainable outcome depends on maintainer direction: either add a typed internal `session:end` contract across the relevant lifecycle paths, or make the plugin `session_end` hook the documented supported surface for this use case.
