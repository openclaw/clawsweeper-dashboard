## Summary

Keep open: current main still lacks the requested internal `session:end` event, and the existing plugin lifecycle hooks do not fully provide the operator-installed internal-hook contract. The linked open PR addresses lazy reset and memory archival behavior rather than this complete request.

## What This Changes

Add an internal `session:end` hook event that fires when an agent session completes for workflow orchestration and related automation.

## Best Possible Solution

Define the exact terminal lifecycle semantics, then either add a typed internal `session:end` contract with complete coverage for every promised caller or explicitly document plugin `session_end` and `subagent_ended` as the supported replacement APIs.

## Reproduction Assessment

Yes at source level: current main has no internal `session:end` dispatch, while the adjacent plugin session and subagent lifecycle contracts are directly traceable.

## Solution Assessment

Unclear: a parallel internal event is plausible for hook packs, but maintainers should first decide whether existing plugin lifecycle APIs are the canonical boundary and precisely define what constitutes a session ending.
