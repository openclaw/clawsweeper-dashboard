## Summary

Keep open. Current main and v2026.6.6 still lack the requested internal `session:end` hook; current code has a separate typed plugin `session_end` lifecycle surface, and the open linked PR only partially addresses daily/idle reset behavior without settling the internal-hook contract.

## What This Changes

Request to add an internal `session:end` hook event that fires when a session completes so external workflow orchestrators can be signaled without agent-instruction workarounds.

## Best Possible Solution

Maintainers should decide the permanent contract, then either implement and document a focused internal `session:end` event across the relevant completion paths or explicitly document typed plugin `session_end` as the supported replacement for this workflow use case.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.6 expose plugin `session_end` but no internal `session:end` event in internal hook types or docs. I did not run a live hook harness because the missing contract is clear from source and docs.

## Solution Assessment

Unclear. A direct internal `session:end` event is plausible, but maintainers need to decide whether the existing typed plugin `session_end` hook should be the supported lifecycle surface instead of adding another internal hook contract.
