## Summary

Keep open. Current main and the latest release still do not provide the requested internal `session:end` hook; the available `session_end` signal is typed plugin API surface, and the linked open PRs remain unmerged implementation candidates with an unresolved API-boundary decision.

## What This Changes

Add an internal `session:end` hook event that fires when an agent session completes so external workflow orchestration can react without agent-authored shutdown commands.

## Best Possible Solution

Establish one supported session-completion lifecycle contract, then either land an internal `session:end` event with docs and focused tests across completion paths or document and extend typed plugin `session_end` as the supported replacement.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source and docs inspection give high confidence that current main lacks the requested internal event while exposing adjacent typed plugin lifecycle hooks.

## Solution Assessment

Unclear. A session completion signal is reasonable, but maintainers need to choose whether the maintainable API is an internal `HOOK.md` event or the existing typed plugin lifecycle hook before accepting one implementation path.
