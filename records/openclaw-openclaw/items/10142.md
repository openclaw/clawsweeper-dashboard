## Summary

Keep open. Current main and v2026.6.6 still do not provide a documented internal `session:end` hook event; the adjacent typed plugin `session_end` hook is a separate surface, and the linked implementation PR is still open/unmerged and needs maintainer product/API review.

## What This Changes

The issue requests an internal `session:end` hook event emitted when an agent session completes so workflow orchestrators can react without agent-authored commands.

## Best Possible Solution

Maintainers should choose the permanent lifecycle contract: land or reject the linked PR if reset/memory behavior is the target, document typed plugin `session_end` as the supported replacement if that is the intended surface, or implement a focused internal `session:end` event with tests and docs.

## Reproduction Assessment

Yes, at source level: current main lacks a typed internal `session:end` event or documented internal event entry, while adjacent plugin `session_end` support exists. This is a feature/API gap rather than a broken documented behavior repro.

## Solution Assessment

Unclear as a final product direction. Adding an internal event would satisfy the request, but the current docs and VISION.md make typed plugin hooks the stronger lifecycle-control surface unless maintainers confirm `HOOK.md` automation should grow this event.
