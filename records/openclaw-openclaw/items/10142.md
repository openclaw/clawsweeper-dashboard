## Summary

Current main still lacks the requested operator-installed internal event. The existing plugin lifecycle is a separate contract, and the only open linked PR is a conflicting reset-specific change rather than a complete implementation.

## What This Changes

Requests a typed internal `session:end` hook for external workflow orchestration when agent sessions complete.

## Best Possible Solution

Define a typed generic lifecycle contract with explicit reasons and a complete terminal-path matrix, then either expose it consistently to internal hooks or document the existing plugin `session_end` plus task-specific hooks as the supported orchestration API.

## Reproduction Assessment

Yes at source level: current internal-hook types, tests, and operator-hook documentation omit `session:end`, while a separate plugin-only `session_end` lifecycle is implemented and documented.

## Solution Assessment

No, the proposed undifferentiated event is not yet the best solution; maintainers should first distinguish persisted-session termination from cron, webhook, subagent, and turn completion, then expose one coherent generic contract.
