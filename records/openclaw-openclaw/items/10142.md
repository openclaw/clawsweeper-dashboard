## Summary

Keep open. Current main and the latest release still do not expose a typed internal `session:end` HOOK.md event; the existing `session_end` lifecycle support is a separate plugin hook surface, and the open linked repair work only partially overlaps the requested contract.

## What This Changes

The issue requests a new internal HOOK.md `session:end` event that fires when agent sessions complete so external workflows can be signaled without agent-instruction workarounds.

## Best Possible Solution

Maintain this as the canonical lifecycle-contract issue until maintainers either document plugin `session_end` as the supported replacement or approve and land a focused internal `session:end` event with typed guards, completion-path wiring, docs, and regression coverage.

## Reproduction Assessment

Yes, at source level. Current internal hook types and public internal-hook docs lack `session:end`, while current completion/reset code emits typed plugin `session_end` hooks instead.

## Solution Assessment

Unclear as a final implementation. A direct internal `session:end` hook is plausible, but the safer maintainer decision is to first settle whether the existing plugin `session_end` lifecycle surface should be the supported replacement for operator-installed internal hooks.
