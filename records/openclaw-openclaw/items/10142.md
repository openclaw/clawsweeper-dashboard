## What This Changes

The issue requests an internal `session:end` hook event that fires when sessions complete so external workflow orchestration can proceed without agent-instruction workarounds.

## Best Possible Solution

Keep this issue open until maintainers either land or close https://github.com/openclaw/openclaw/pull/61675 and explicitly decide whether typed plugin `session_end` is the supported replacement or a focused internal `session:end` event should be added with docs and tests.

## Reproduction Assessment

## Solution Assessment

Unclear as a final product direction. A direct internal `session:end` event would satisfy the request, but the existing typed plugin `session_end` hook already covers part of the lifecycle use case, so maintainers need to decide whether to document that replacement or add a new internal hook contract.
