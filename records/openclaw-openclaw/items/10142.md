## Summary

Keep open. Current main still lacks the requested internal `session:end` hook contract; the similarly named `session_end` surface is a plugin lifecycle hook, and open implementation PRs still need maintainer selection before this issue can close.

## What This Changes

Add an internal `session:end` hook event that fires when a session completes so workflow orchestration systems can react without relying on agent instructions.

## Best Possible Solution

Keep this issue open until maintainers land one canonical implementation with docs and focused tests, or explicitly document plugin `session_end` as the supported replacement and close the overlapping PRs.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is a feature/API contract gap. Source and docs inspection on current main gives high-confidence evidence that internal hooks still lack `session:end`.

## Solution Assessment

Unclear.
