## Summary

Current main and v2026.6.5 still lack the requested internal `session:end` event. The existing typed plugin `session_end` hook covers persistent-session lifecycle boundaries rather than ordinary cron or subagent task completion, and an open linked PR still references this issue, so maintainers must first settle the canonical lifecycle semantics.

## What This Changes

Add a session-completion hook that lets external orchestrators react without relying on agent-authored end-of-task commands.

## Best Possible Solution

Define one canonical typed contract with explicit semantics for persistent-session termination versus individual run completion, cover each intended path exactly once, and document whether internal-hook users should bridge through that plugin API or receive a dedicated internal event.

## Reproduction Assessment

## Solution Assessment
