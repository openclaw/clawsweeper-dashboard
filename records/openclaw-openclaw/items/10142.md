## Summary

Keep open. Current main still does not define or document an internal `session:end` hook event, and two open implementation PRs still target this issue with closing syntax, so this should stay open until maintainers land or reject that work.

## What This Changes

Add an internal `session:end` hook event that fires when an agent session completes so external orchestrators can react without agent-authored CLI workarounds.

## Best Possible Solution

Define the supported session lifecycle contract, then either land a focused internal `session:end` implementation with docs/tests for agreed completion paths or explicitly document plugin `session_end` as the supported replacement and close the competing PRs.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is a feature/API gap. Source and docs inspection on current main gives high-confidence evidence that the internal hook contract still lacks `session:end`.

## Solution Assessment

No, current main is not the best end state because it only provides the adjacent plugin `session_end` hook. The maintainable path is to resolve the open implementation PRs or make and document an explicit product decision to use plugin hooks instead.
