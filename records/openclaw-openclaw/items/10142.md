## Summary

Keep open: current main and the latest release still lack the requested operator-installed internal `session:end` hook event; the existing `session_end` surface is a typed plugin hook, and the linked implementation candidate remains open and not merge-ready.

## What This Changes

Request to add a `session:end` internal hook event that fires when an agent session completes so external workflow systems can react without agent-authored cleanup commands.

## Best Possible Solution

Settle the lifecycle surface first: either land or replace a reviewed implementation of the internal event, or explicitly document typed plugin `session_end` as the supported replacement for this use case.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes, at source level: current main exposes internal `session:patch` and typed plugin `session_end`, but no internal `session:end` event or guard is present.

## Solution Assessment

Is this the best way to solve the issue? Unclear: a direct internal `session:end` event is plausible, but maintainers still need to decide whether the typed plugin `session_end` hook is the intended contract instead.
