## Summary

Keep open. Current main still lacks the requested internal `session:end` hook event; the available `session_end` lifecycle signal is typed plugin API surface, and the linked open PRs at https://github.com/openclaw/openclaw/pull/56672 and https://github.com/openclaw/openclaw/pull/61675 remain unmerged implementation candidates with unresolved API direction.

## What This Changes

Request an internal `session:end` hook event emitted when an agent session completes so operator hooks can notify external workflow orchestrators.

## Best Possible Solution

Establish one supported session-completion lifecycle contract, then either land an internal `session:end` event with docs and tests across completion paths or document and extend typed plugin `session_end` as the supported replacement.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source and docs inspection give high confidence that current main lacks the requested internal event while exposing adjacent typed plugin lifecycle hooks.

## Solution Assessment

Unclear. A session completion signal is reasonable, but maintainers need to choose whether the maintainable API is an internal `HOOK.md` event or the existing typed plugin lifecycle hook before accepting one implementation path.
