## Summary

Keep open. Current main still lacks the requested internal `session:end` hook event; the existing `session_end` coverage is a typed plugin hook surface, and the open linked implementation path is unmerged and still needs maintainer contract review.

## What This Changes

Add an internal `session:end` hook event that fires when an agent session completes for workflow orchestration integrations.

## Best Possible Solution

Maintainers should choose and document the supported session-completion contract, then either add a focused internal `session:end` event or explicitly direct workflow orchestration users to typed plugin `session_end`.

## Reproduction Assessment

Yes, at source level: current main has plugin `session_end` lifecycle hooks, but the internal hook registry and internal hooks docs do not define or emit `session:end`. I did not run a live agent session because the missing event is directly visible in source/docs.

## Solution Assessment

Unclear: adding an internal `session:end` event is plausible, but the maintainable product answer may be to make typed plugin `session_end` the supported workflow-orchestration surface instead.
