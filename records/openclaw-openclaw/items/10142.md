## Summary

Keep open. Current main still lacks an internal `session:end` hook contract, and live GitHub metadata shows open implementation PRs with closing references still targeting this issue.

## What This Changes

Add an internal `session:end` hook event when an agent session completes so workflow orchestrators can react without asking agents to run explicit completion commands.

## Best Possible Solution

Define the supported session lifecycle contract, then land one focused implementation with docs/tests for the agreed completion paths or explicitly document plugin `session_end` as the supported replacement and close the competing PRs.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is a feature/API gap. Source and docs inspection on current main gives high-confidence evidence that the internal hook contract still lacks `session:end`.

## Solution Assessment

No, current main is not the best end state because it only provides the adjacent plugin `session_end` hook. The maintainable path is to resolve the open implementation PRs or make and document an explicit product decision to use plugin hooks instead.
