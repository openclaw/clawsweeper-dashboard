## Summary

Keep open. Current main and the latest release still lack the requested internal `session:end` hook event; the existing `session_end` lifecycle surface is a plugin hook, and the open linked implementation path still needs maintainer/product review before this issue can be resolved.

## What This Changes

Request to add a `session:end` internal hook event that fires when an agent session completes so workflow orchestration can react without agent instructions.

## Best Possible Solution

Decide the lifecycle contract first: either land a maintainer-approved internal `session:end` design with docs/tests across completion paths, or document the typed plugin `session_end` hook as the supported replacement and close this issue after the linked PR path is resolved.

## Reproduction Assessment

Yes, at source level. Current main has plugin `session_end` lifecycle hooks, but internal hook source and public internal hook docs still do not define or dispatch a `session:end` event.

## Solution Assessment

Unclear. Adding a new internal hook is plausible, but maintainers should first decide whether the existing typed plugin `session_end` lifecycle hook should be the canonical extension point instead of adding a parallel internal event.
