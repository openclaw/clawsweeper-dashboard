The issue requests an internal `session:end` hook event emitted when a session completes so workflow orchestrators can react without agent-instruction workarounds.

## Best Possible Solution

Choose one supported lifecycle contract, then either land an internal `session:end` event with typed guard, docs, and coverage across completion paths, or document typed plugin `session_end` as the supported replacement and close/redirect the overlapping PRs.
