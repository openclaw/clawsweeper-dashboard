Keep open. Current main still lacks the requested internal `session:end` hook contract; the existing `session_end` implementation is a typed plugin hook, and the only live closing PR is still open and conflicting rather than a merged fix.

## What This Changes

The issue requests an internal `session:end` hook event fired when an agent session completes so external orchestration can react without agent-authored workaround commands.

## Best Possible Solution

Keep this issue open until maintainers either land or close the linked implementation PR, or explicitly document that typed plugin `session_end` is the supported replacement for internal hook automation; if implemented directly, add a typed internal event, wire the relevant completion paths, and update focused tests plus public hooks docs.
