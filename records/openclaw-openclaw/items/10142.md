## Summary

Keep open. Current main and v2026.6.6 still do not provide a documented internal `session:end` hook event; the existing typed plugin `session_end` hook is a separate lifecycle surface, and the open linked PR at https://github.com/openclaw/openclaw/pull/61675 is unmerged and explicitly avoids adding new hook event names.

## What This Changes

Request adds an internal `session:end` hook event emitted when an agent session completes so external workflow orchestration can react without agent-side instructions.

## Best Possible Solution

Maintainers should choose one canonical lifecycle contract: land a focused internal `session:end` event with docs and tests across completion paths, or document typed plugin `session_end` as the supported replacement and close the linked candidates accordingly.

## Reproduction Assessment

Yes, at source level: current main lacks a typed/documented internal `session:end` event while adjacent typed plugin `session_end` support exists. This is a feature/API gap rather than a broken documented behavior repro.

## Solution Assessment

Unclear as a final direction. Adding the internal event would satisfy the request, but current docs and VISION.md make typed plugin hooks the stronger lifecycle-control surface unless maintainers confirm internal hooks should grow this event.
