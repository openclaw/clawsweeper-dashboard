## Summary

Keep open. Current main and v2026.6.10 still lack the requested internal `session:end` hook event; the implemented `session_end` surface is a typed plugin lifecycle hook, and the open linked implementation candidate is unresolved and narrower than the requested contract.

## What This Changes

Add a `session:end` internal hook event that fires when an agent session completes so external workflow orchestration can react without agent-authored cleanup commands.

## Best Possible Solution

Define one supported session-completion extension contract: either add and document typed internal `session:end` across the relevant completion paths, or document plugin `session_end` as the supported replacement and retarget the linked work accordingly.

## Reproduction Assessment

Yes, at source level: current main lacks a typed internal `session:end` event and docs entry, while adjacent plugin `session_end` support exists separately. This is a feature/API gap rather than a failing runtime bug reproduction.

## Solution Assessment
