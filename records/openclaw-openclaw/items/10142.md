## Summary

Keep open. Current main and v2026.6.8 still lack the requested internal `session:end` hook event; the existing `session_end` surface is a typed plugin lifecycle hook, and the linked implementation PR remains open rather than merged.

## What This Changes

The issue requests a new internal `session:end` hook event for session completion so workflow orchestration can react without requiring agents to run explicit CLI commands.

## Best Possible Solution

Maintainers should choose the permanent contract: add and document a typed internal `session:end` event across the relevant lifecycle paths, or explicitly document plugin `session_end` as the supported replacement and close or retarget the linked PRs.

## Reproduction Assessment

Yes, at source level: current main has plugin `session_end` lifecycle hooks, but internal hook types, docs, and inspected completion paths do not define or emit the requested `session:end` event. This is a feature gap rather than a broken existing behavior repro.

## Solution Assessment

Unclear. The narrowest maintainable solution depends on maintainer direction: either add the internal hook contract, or make the typed plugin `session_end` hook the documented supported surface for this use case.
