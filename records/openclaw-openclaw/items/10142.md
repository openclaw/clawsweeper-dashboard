## Summary

Keep open. Current main still lacks the requested internal `session:end` hook contract; the similarly named `session_end` support is a plugin lifecycle hook, and open implementation PRs still need maintainer selection before this issue can close.

## What This Changes

The issue requests an internal `session:end` hook event that fires when an agent session completes so orchestration systems can react without agent-authored cleanup commands.

## Best Possible Solution

Keep this issue open until maintainers land one canonical implementation with docs and focused tests, or explicitly document plugin `session_end` as the supported replacement and close the overlapping PRs.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is a feature/API contract gap. Source and docs inspection on current main gives high-confidence evidence that internal hooks still lack `session:end`.

## Solution Assessment

Unclear. Adding the internal hook matches the request, but maintainers should first choose between that contract, the existing plugin `session_end`, and the overlapping implementation PRs at https://github.com/openclaw/openclaw/pull/56672 and https://github.com/openclaw/openclaw/pull/61675.
