## Summary

Keep open. Current main and v2026.5.22 still lack the requested internal `session:end` hook event; the existing `session_end` surface is a typed plugin hook, and the open linked PRs remain implementation candidates rather than merged fixes.

## What This Changes

Request to add an operator-facing internal `session:end` hook event that fires when an agent session completes so workflow orchestrators can react without agent-authored CLI commands.

## Best Possible Solution

Establish one supported session-completion lifecycle contract, then either land an internal `session:end` event with docs and coverage across completion paths or document typed plugin `session_end` as the supported replacement.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source, docs, release-tag inspection, and linked-PR state give high confidence that current main lacks the requested internal event while exposing adjacent typed plugin lifecycle hooks.

## Solution Assessment

Unclear. A session completion signal is reasonable, but maintainers need to choose whether the durable API should be an internal HOOK.md event or the existing typed plugin `session_end` lifecycle hook.
