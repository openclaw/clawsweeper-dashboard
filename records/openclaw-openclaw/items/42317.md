## Summary

## What This Changes

The PR preserves configured `session.mainKey`, threads it into several session-route builders, and adds defaults/routing unit tests for custom main session keys.

## Best Possible Solution

Yes for source-level behavior: current main still rewrites `session.mainKey` to `main` and route resolution still uses `DEFAULT_MAIN_KEY`. I did not establish a live Telegram or WhatsApp reproduction in this read-only review.

## Solution Assessment

No: honoring the existing field may be the right end state, but this branch needs a rebase, an explicit compatibility/security decision, and complete current-path delivery coverage before it is the best mergeable solution.
