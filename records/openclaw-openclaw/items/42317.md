## Summary

Keep open: current main still does not honor custom `session.mainKey`, and this PR remains the canonical linked implementation path, but the branch is conflicting, misses current route-builder and delivery-guard coverage, and lacks external real behavior proof.

## What This Changes

The PR changes defaults and route/session-key builders so configured `session.mainKey` can become the canonical main session key, with unit tests for custom routing.

## Best Possible Solution

Land a rebased implementation that consistently honors normalized `session.mainKey` across defaults, central routing, current plugin/outbound route builders, delivery guards, docs, and focused tests, with live Telegram proof.

## Reproduction Assessment

Yes for current-main source behavior: `applySessionDefaults()` rewrites configured `session.mainKey` to `main`, and central route builders still use `DEFAULT_MAIN_KEY`. I did not run a live Telegram or WhatsApp setup in this read-only review.

## Solution Assessment

No. The direction is reasonable, but this branch is not the best merge unit until it is rebased onto current main, covers delivery-guard alias handling and moved plugin route builders, aligns docs, and includes real behavior proof.
