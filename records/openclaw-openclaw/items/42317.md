## Summary

Keep this PR open: current main still does not implement custom `session.mainKey` routing, and the closed source issue explicitly points here as the canonical remaining work. The branch is not merge-ready because it lacks real behavior proof and needs a current-main port across routing, delivery guards, bundled plugin paths, and docs.

## What This Changes

The PR preserves configured `session.mainKey`, threads it into selected session-route builders, and adds unit coverage for custom main-session keys.

## Best Possible Solution

Land a current-main implementation that either fully supports `session.mainKey` as canonical routing or formally keeps it legacy, with routing guards, bundled plugin paths, docs, tests, and live proof aligned.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the current behavior: source inspection shows config materialization rewrites custom `session.mainKey` to `main` and route builders still hard-code the default main key.

## Solution Assessment

Is this the best way to solve the issue? No. The direction is plausible, but the branch is not the best merge unit until it is ported to current main, covers main-alias delivery guards and bundled plugin route builders, aligns docs, and includes real behavior proof.
