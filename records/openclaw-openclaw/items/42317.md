## Summary

Keep open. Current main still does not honor custom `session.mainKey`, and this PR remains the canonical implementation path, but the branch is not mergeable as-is because it lacks real behavior proof, is conflicting/outdated against current route-builder locations, and leaves a privacy-sensitive delivery-guard gap.

## What This Changes

This PR preserves configured `session.mainKey` in defaults and threads it through selected session route-key builders with unit coverage.

## Best Possible Solution

Land a rebased implementation that preserves normalized `session.mainKey` across defaults, central routing, outbound/plugin route builders, delivery guards, docs, and focused tests, with live Telegram proof and relevant outbound proof.

## Reproduction Assessment

Yes. Source inspection on current main shows `applySessionDefaults()` rewrites configured `session.mainKey` to `main`, while central route builders still pass `DEFAULT_MAIN_KEY`; I did not run a live Telegram or WhatsApp setup in this read-only review.

## Solution Assessment

No. The direction is reasonable, but this branch is not the best merge unit until it is rebased onto current main, covers delivery-guard alias handling and moved plugin route builders, aligns docs, and includes real behavior proof.
