## Summary

Keep open: current main still ignores configured `session.mainKey`, but this PR is conflicting, lacks after-fix real behavior proof, and does not yet cover delivery guards or the current plugin/outbound route-builder surface.

## What This Changes

The PR preserves configured `session.mainKey`, threads it through selected route builders for Telegram/WhatsApp/outbound paths, and adds config/routing unit tests.

## Best Possible Solution

Land a rebased implementation that treats normalized `session.mainKey` as the canonical main key across defaults, routing, outbound/plugin builders, delivery guards, docs, and focused tests, then prove the Telegram or WhatsApp path in a redacted real setup.

## Reproduction Assessment

Yes for source-level reproduction: current main’s `applySessionDefaults()` rewrites any configured `session.mainKey` to `main`, and `resolveAgentRoute()` still builds main keys with `DEFAULT_MAIN_KEY`. I did not run a live Telegram or WhatsApp setup in this read-only review.

## Solution Assessment

No. The direction is plausible, but the branch needs a rebase plus custom-main delivery guard coverage, current outbound/plugin route-builder updates, docs alignment, and real behavior proof before it is the right merge unit.
