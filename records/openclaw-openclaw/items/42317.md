## Summary

Keep open: current main still does not honor custom `session.mainKey` in the defaults and central route builders, but this PR is not merge-ready because it lacks real behavior proof and needs a current-main port plus routing-guard coverage.

## What This Changes

The PR preserves normalized `session.mainKey`, threads it into selected session/route-key builders, and adds unit coverage for custom main-session routing.

## Best Possible Solution

Land a rebased implementation that preserves normalized `session.mainKey` across defaults, route builders, outbound/plugin routes, delivery guards, docs, and focused tests, with live Telegram and relevant outbound proof.

## Reproduction Assessment

Yes. Source inspection on current main shows `applySessionDefaults()` rewrites custom `session.mainKey` to `main`, while central route builders and delivery guards still use `DEFAULT_MAIN_KEY` or literal `main`; I did not run a live Telegram/WhatsApp setup in this read-only review.

## Solution Assessment

No. The direction is reasonable, but this branch is not the best merge unit until it is rebased onto current main, covers delivery-guard alias handling and moved plugin route builders, aligns docs, and includes real behavior proof.
