## Summary

Keep open. Current main still ignores the documented `session.mainKey` at config materialization and hard-codes `main` in central route builders, so this PR is not obsolete; however, the branch is not safe to merge because it misses current moved outbound/bundled plugin paths and does not update delivery guards that still treat only literal `main` as a main session.

## What This Changes

The PR preserves configured `session.mainKey`, threads it through selected route/session-key builders for routing, outbound, Telegram, and WhatsApp broadcast paths, and adds config/routing tests.

## Best Possible Solution

Port the feature on top of current main by preserving normalized `session.mainKey`, threading it through central SDK/outbound/bundled plugin route builders, teaching delivery-continuity guards custom main aliases, and adding changelog plus regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows `materializeRuntimeConfig()` calls `applySessionDefaults()`, which rewrites custom `session.mainKey` to `main`, while route resolution still passes `DEFAULT_MAIN_KEY`.

## Solution Assessment

No. The direction is right, but the proposed branch is not the best fix as-is because it must be rebased or replaced with a current-main port that covers moved route builders and delivery guards.
