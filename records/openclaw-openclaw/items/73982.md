## Summary

Keep this PR open. It addresses a real current-main gap in chat bash exec-default inheritance, but the patch also makes persisted session exec overrides take precedence for direct host bash, which is security-sensitive and needs an explicit maintainer trust-model decision before merge.

## What This Changes

The PR updates chat `/bash` and `!` execution to inherit global, agent, and session `tools.exec` defaults, forwards `sessionEntry` into the bash handler, adds regression/routing tests, and adds a changelog entry.

## Best Possible Solution

Land a focused fix that makes chat bash inherit the same effective global and agent `tools.exec` defaults as the normal exec tool, preserves elevated gates and bang routing, and explicitly defines how session-level exec overrides are allowed for direct bash.

## Reproduction Assessment

Yes. The functional gap is reproducible by source inspection or a mocked unit test asserting `createExecTool` arguments on current main. The security concern is also reproducible from the diff: an authorized non-internal session can persist `/exec` defaults, and the PR gives those stored fields precedence for later `!` or `/bash` execution.

## Solution Assessment

Unclear. The global/agent inheritance direction is narrow and maintainable, but the session override precedence is not clearly the best or safest way to solve direct chat bash policy inheritance without a maintainer decision on the trust boundary.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.8

Full review comments:

- **[P1] Gate chat bash session exec overrides:** `src/auto-reply/reply/bash-command.ts:65-72`
  - body: This resolver gives stored `sessionEntry.execHost`, `execSecurity`, `execAsk`, and `execNode` precedence for `/bash`. Those fields can be persisted by authorized non-internal chat directives, and `/bash` is a direct host-command route, so an elevated-allowlisted sender could weaken approval policy for later bang commands. Please gate these session overrides for direct bash, or limit them to stricter settings, before consuming them here.
  - confidence: 0.78
