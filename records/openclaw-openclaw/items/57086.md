## Summary

Keep open: current main already has the stronger gateway-agent freshness guard, but this PR still contains adjacent node-event changes that are incomplete and it lacks after-fix real behavior proof.

## What This Changes

The PR removes stored sessionId forwarding from gateway agent, voice.transcript, and agent.request dispatches when a sessionKey is present, with focused test assertions.

## Best Possible Solution

Rebase on current main, keep the shipped gateway-agent freshness implementation, and either fix node-event handlers before store touches with rollover tests and real proof or split that adjacent work into a separate follow-up.

## Reproduction Assessment

Yes, at source level. Seed a stale keyed legacy/no-header session row and trigger voice.transcript or agent.request; current node-event code selects and touches the stored sessionId before downstream freshness can own rollover.

## Solution Assessment

No. Removing only the ingress sessionId is directionally useful, but the node-event paths need freshness-aware id selection before touching the store, or the touch should be deferred until the session resolver owns the decision.
