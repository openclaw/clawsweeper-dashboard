## Summary

Keep open: current main already has the stronger gateway-agent freshness fix, but this PR also carries node-event changes that are still incomplete and lacks external real behavior proof required before merge.

## What This Changes

The PR suppresses forwarded `sessionId` values for gateway agent, `voice.transcript`, and `agent.request` dispatches when a `sessionKey` is present, with focused test assertions.

## Best Possible Solution

Keep the shipped gateway-agent freshness implementation and replace the node-event portion with freshness-aware touch/dispatch behavior, regression tests, and real behavior proof.

## Reproduction Assessment

Yes, at source level for the remaining PR findings. A stale keyed `voice.transcript` or `agent.request` row without reliable lifecycle/header timestamps can still be touched before downstream freshness evaluation, and current source shows the old `sessionId` selected before dispatch.

## Solution Assessment

No. Suppressing the forwarded field is directionally useful, but the node-event paths need freshness-aware id selection before the store touch or should defer the write to the session-resolution path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Resolve freshness before voice dispatch:** `src/gateway/server-node-events.ts:304`
  - body: Deleting the forwarded `sessionId` is not enough while this handler still selects `entry.sessionId` and queues a store touch before dispatch. For stale legacy rows without lifecycle/header timestamps, that touch can refresh `updatedAt` before `resolveSession` evaluates freshness, so the old session may still be reused; resolve/select the fresh id first or defer the touch.
  - confidence: 0.8
- **[P2] Resolve freshness before agent-request dispatch:** `src/gateway/server-node-events.ts:436`
  - body: The `agent.request` path still chooses `entry.sessionId` and awaits `touchSessionStore` before handoff. Removing only the ingress field leaves stale legacy rows refreshed before freshness evaluation, so move freshness-aware selection before the touch or let the session resolver own the write.
  - confidence: 0.79
