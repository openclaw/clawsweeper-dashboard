## Summary

Keep open. This is useful source work for the Linux/Windows desktop companion tracked by the related platform request, but the branch is not merge-ready because the new Tauri app still reimplements Gateway config/dashboard/restart behavior and introduces a concrete PATH-based restart execution concern.

## What This Changes

This PR adds a new `apps/desktop` Tauri v2 React/Rust companion app with tray controls, embedded Gateway Control UI, Gateway health polling/restart wiring, workspace registration, root `desktop:*` scripts, and lockfile updates.

## Best Possible Solution

Land a desktop MVP only after maintainers approve the first-party Tauri and restart model, then make the app reuse canonical config, dashboard URL, auth, and restart contracts with matching docs, changelog, labeler coverage, and Linux/Windows validation proof.

## Reproduction Assessment

Yes for the static review findings: the PR-head files can be compared directly with current main's config loader, Control UI link helper, routing, docs, labeler, and changelog policy. Full runtime reproduction is incomplete because the discussion does not show cargo check, frontend type/build proof, or a desktop smoke run.

## Solution Assessment

No. A Tauri companion may be the right direction for the related platform request, but this branch should not ship while it reimplements canonical Gateway contracts and restarts through an ambient shell/PATH command.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Load Gateway config through canonical contracts:** `apps/desktop/src-tauri/src/main.rs:296-383`
  - body: This reads `openclaw.json` with strict `serde_json` and hand-extracts selected fields, so valid OpenClaw configs using JSON5 comments, `$include`, env substitution, or SecretRef-managed gateway auth can be treated as missing or different from the running Gateway. The desktop app should consume an existing config snapshot, CLI contract, or dashboard contract instead of parsing a partial Rust copy.
  - confidence: 0.9
- **[P2] Include the configured Control UI base path:** `apps/desktop/src-tauri/src/main.rs:492-504`
  - body: `build_dashboard_url()` always embeds `/`, but current Gateway config supports `gateway.controlUi.basePath`. When the Control UI is mounted under a path such as `/openclaw`, the iframe opens the wrong route instead of the configured dashboard URL.
  - confidence: 0.9
- **[P2] Permit every generated dashboard origin in CSP:** `apps/desktop/src-tauri/tauri.conf.json:14`
  - body: The Rust URL builder can produce HTTPS dashboard URLs and custom-host URLs, but this CSP only allows HTTP/WS localhost and 127.0.0.1 frame/connect origins. Valid generated dashboard URLs can be blocked by the WebView policy.
  - confidence: 0.86
- **[P2] Avoid restarting through the ambient shell PATH:** `apps/desktop/src-tauri/src/main.rs:538-558`
  - body: The tray/UI restart action invokes `openclaw gateway restart` through `cmd` or `sh -lc`, so a first-party desktop control executes whichever `openclaw` the ambient PATH resolves. Resolve a trusted CLI path or call a controlled Gateway restart surface before exposing this as a GUI action.
  - confidence: 0.88
- **[P2] Add the required release-note entry:** `package.json:1322-1323`
  - body: This adds a user-facing desktop companion and root `desktop:*` commands but does not update `CHANGELOG.md`. Repo policy requires user-facing feature work to include a changelog entry before landing.
  - confidence: 0.84
- **[P3] Update public platform docs for the new app surface:** `apps/desktop/README.md:1-4`
  - body: The app-local README says this is a first Linux/Windows PR, but current public platform docs still say native Linux and Windows companion apps are planned. Landing this first-party app needs matching platform docs so users see the new supported surface.
  - confidence: 0.8
- **[P3] Route desktop app changes through labeler:** `pnpm-workspace.yaml:1-6`
  - body: Registering `apps/desktop` creates a new app surface, but `.github/labeler.yml` does not route desktop app changes. Future desktop PRs will miss the expected app label unless this path is added.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: The diff introduces a concrete command-execution concern in the new desktop restart action.

Concerns:

- **[medium] Ambient shell restart execution:** `apps/desktop/src-tauri/src/main.rs:538`
  - body: `run_gateway_restart()` invokes `cmd /C openclaw gateway restart` on Windows and `sh -lc "openclaw gateway restart"` elsewhere. A first-party desktop app should not execute a PATH/cwd-hijackable command for a GUI restart action; it should resolve a trusted CLI path or use a controlled Gateway restart surface.
  - confidence: 0.88
