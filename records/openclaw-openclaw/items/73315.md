## Summary

Keep open. Current main still has no Linux/Windows desktop companion app, and this PR is a useful active attempt for #75 that supersedes earlier desktop PR work, but the patch is not merge-ready because it reimplements config/dashboard/restart behavior and misses required docs, changelog, and labeler follow-through.

## What This Changes

The PR adds a new Tauri v2 Linux/Windows desktop companion under apps/desktop with a React shell, Rust tray/backend, gateway health and restart wiring, root desktop scripts, workspace registration, and lockfile entries.

## Best Possible Solution

Land a desktop MVP only after it reuses canonical config, dashboard URL, auth, and restart contracts, updates public platform docs plus changelog and labeler coverage, and has Linux/Windows build/runtime validation.

## Reproduction Assessment

Yes for the static defects: the PR head can be inspected directly against current main's config IO, dashboard link helper, platform docs, labeler, and changelog policy. Full runtime reproduction is incomplete because the PR discussion says cargo check and frontend validation were not run.

## Solution Assessment

No. A Tauri companion may be a reasonable product direction, but this branch should first delegate to existing OpenClaw config/dashboard contracts and use a trusted restart surface or resolved executable path instead of duplicating those behaviors in Rust.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Use the canonical Gateway config reader:** `apps/desktop/src-tauri/src/main.rs:296-383`
  - body: This reader parses the config as strict JSON and manually extracts a few fields, so valid OpenClaw configs using JSON5 comments, $include, env-template SecretRefs, or runtime migrations can fall back to defaults or lose auth. The desktop app should consume the same config snapshot or CLI contract as the Gateway before deriving URLs.
  - confidence: 0.92
- **[P2] Preserve the Control UI base path:** `apps/desktop/src-tauri/src/main.rs:492-504`
  - body: build_dashboard_url() always points the iframe at /, but the Gateway can serve Control UI under gateway.controlUi.basePath such as /openclaw. Those valid setups will open the wrong page instead of the configured dashboard route.
  - confidence: 0.91
- **[P2] Keep generated dashboard URLs inside the CSP:** `apps/desktop/src-tauri/tauri.conf.json:14`
  - body: The Rust code can generate https loopback URLs when TLS is enabled and custom-host URLs when gateway.bind is custom, but this CSP only permits http/ws localhost and 127.0.0.1. Valid generated dashboard origins can therefore be blocked by the WebView policy.
  - confidence: 0.9
- **[P2] Avoid restarting through the ambient shell PATH:** `apps/desktop/src-tauri/src/main.rs:556-558`
  - body: The tray and UI restart action runs openclaw gateway restart through sh -lc or cmd /C, so a first-party desktop action executes whichever openclaw the ambient shell resolves. Resolve a trusted CLI path or call a controlled restart surface instead of shell/PATH lookup.
  - confidence: 0.84
- **[P2] Update platform docs for the new desktop surface:** `apps/desktop/README.md:3-4`
  - body: This adds a first-party Linux/Windows desktop companion, but the public platform docs still say native Linux and Windows companion apps are planned. Landing as-is would make the published platform status incorrect.
  - confidence: 0.86
- **[P2] Add the required changelog entry:** `package.json:1322-1323`
  - body: The branch exposes a new user-facing desktop companion and root desktop:* commands, but it does not update CHANGELOG.md. Repo policy requires user-facing feature work to add a release-note entry before landing.
  - confidence: 0.85
- **[P3] Route desktop changes through the labeler:** `pnpm-workspace.yaml:3`
  - body: Registering apps/desktop creates a new app surface, but .github/labeler.yml still routes only Android, iOS, macOS, and web UI app changes. Future desktop changes will miss the expected app label until this route is added.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff introduces a security-sensitive desktop restart action that executes openclaw through the ambient shell PATH.

Concerns:

- **[medium] Ambient shell restart execution:** `apps/desktop/src-tauri/src/main.rs:556`
  - body: run_gateway_restart() invokes cmd /C openclaw gateway restart on Windows and sh -lc "openclaw gateway restart" elsewhere. A first-party desktop app should not execute a PATH-resolved shell command for a privileged-feeling GUI action; it should resolve a trusted CLI path or use a controlled Gateway restart surface.
  - confidence: 0.84
