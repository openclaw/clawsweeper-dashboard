Full review comments:

- **[P2] Parse OpenClaw config through the canonical contract:** `apps/desktop/src-tauri/src/main.rs:297`
  - body: Current OpenClaw config is JSON5, can live at `OPENCLAW_CONFIG_PATH`, and can use `$include`/env resolution, but this reader uses strict JSON from the default file. Valid configs with comments, includes, or alternate config paths will be treated as missing/invalid and the desktop app will silently fall back to loopback defaults.
  - confidence: 0.91
- **[P2] Keep generated Gateway URLs inside the WebView CSP:** `apps/desktop/src-tauri/tauri.conf.json:14`
  - body: The backend can construct dashboard URLs from `gateway.bind: "custom"` and `gateway.customBindHost`, but the Tauri CSP only permits localhost/127.0.0.1 for frame/connect origins. A valid custom host can be generated by Rust and then blocked by the WebView unless the MVP restricts itself to loopback or derives the policy from the allowed configured origin.
  - confidence: 0.88
- **[P2] Update platform docs before landing the desktop app:** `apps/desktop/README.md:3-4`
  - body: This adds a Linux/Windows desktop companion while current public platform docs still say those companions are planned or unavailable. Landing the app without updating the platform pages would make the new user-facing surface contradict the docs immediately.
  - confidence: 0.84
- **[P3] Add labeler coverage for the desktop app:** `pnpm-workspace.yaml:3`
  - body: Registering `apps/desktop` as a new app workspace creates a new repository surface, but `.github/labeler.yml` still routes only Android, iOS, macOS, and web UI app changes. Future desktop changes will miss the expected app label unless this PR adds a desktop route.
  - confidence: 0.82
- **[P2] Add a changelog entry for the desktop feature:** `package.json:1324`
  - body: The branch exposes a new user-facing desktop companion and root `desktop:*` commands, but it does not update `CHANGELOG.md`. Repo policy requires user-facing feature work to add an Unreleased entry with contributor credit before landing.
  - confidence: 0.84

  - body: The new package adds a `typecheck` script that runs `tsc --noEmit`, but this repo standardizes TypeScript checks through the `tsgo` lanes and avoids adding standalone `typecheck`/`tsc --noEmit` entry points. Wire desktop validation into the existing check lane instead of creating a separate checker path.
  - confidence: 0.78
