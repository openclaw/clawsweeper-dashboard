- **[P2] Parse OpenClaw config through the canonical contract:** `apps/desktop/src-tauri/src/main.rs:297`
  - body: Current OpenClaw config is JSON5 and can use the configured config path and include layout, but this reader uses strict `serde_json::from_str` on the default file. Valid configs with comments, trailing commas, or included gateway settings can be treated as invalid/missing, causing the desktop app to fall back to loopback defaults and lose port/auth/bind settings.
  - confidence: 0.91
- **[P2] Keep generated gateway URLs inside the WebView CSP:** `apps/desktop/src-tauri/tauri.conf.json:14`
  - body: The backend can construct dashboard and health URLs from `gateway.bind: "custom"` and `gateway.customBindHost`, but the Tauri CSP only allows `127.0.0.1` and `localhost` for connections and frames. A valid custom host such as a LAN or tailnet address will be generated by Rust and then blocked by the WebView unless the app either restricts the MVP to loopback or safely permits the configured origin.
  - confidence: 0.88
