## Summary

Keep this PR open. Current main still has no Linux/Windows desktop companion, and the contributor has addressed useful prior feedback from #59075 and Greptile, but the branch is not merge-ready because it bypasses established config/dashboard contracts, has a security-sensitive restart path through the ambient shell/PATH, and lacks required docs/changelog/labeler follow-through for a new first-party app.

## What This Changes

The PR adds a new private Tauri v2 desktop companion app under `apps/desktop`, root `desktop:*` scripts, workspace and lockfile updates, plus small tool-display and channel module-loader test fixes.

## Best Possible Solution

Keep the Tauri branch as the active candidate only if maintainers want this first-party desktop direction, then make it reuse or faithfully mirror the canonical OpenClaw config, auth, dashboard-link, and restart contracts; update public platform docs, labeler, and changelog before landing; and require Linux/Windows build/runtime proof.

## Reproduction Assessment

Yes for the static review defects: compare the PR head files against current main's config loader, dashboard link resolver, CSP constraints, platform docs, labeler, and changelog policy. Full runtime reproduction is incomplete because the PR has not provided cargo check, frontend checks, or Linux/Windows smoke proof.

## Solution Assessment

No. A Tauri MVP may be an acceptable product direction, but this branch is not the best merge shape until it stops reimplementing config/dashboard behavior and replaces the ambient shell/PATH restart path with a trusted CLI resolution or controlled Gateway restart surface.

## Security Review

Status: needs_attention

Summary: Security review needs attention because the new desktop restart action executes an unqualified `openclaw` command through the ambient shell/PATH.

Concerns:

- **[medium] Ambient shell restart execution:** `apps/desktop/src-tauri/src/main.rs:556`
  - body: `run_gateway_restart()` invokes `cmd /C openclaw gateway restart` on Windows and `sh -lc "openclaw gateway restart"` elsewhere. A first-party desktop app should not execute a PATH/cwd-hijackable command for a privileged-feeling GUI action; it should resolve a trusted CLI path or call a controlled Gateway restart surface.
