## Summary

Keep open. Current main still has no desktop app, and the PR has useful follow-up work from #59075, but it remains not merge-ready because it reimplements config/dashboard contracts and introduces a shell/PATH restart security concern.

## What This Changes

The PR adds a new Tauri v2 React/Rust desktop companion under `apps/desktop`, root `desktop:*` scripts, workspace/lockfile entries, tray health/restart behavior, and a small tool-display export.

## Best Possible Solution

Land a desktop MVP only after maintainers accept the first-party Tauri direction and the app delegates config, dashboard URL, auth, and restart behavior to existing OpenClaw contracts with docs, changelog, labeler, and platform build proof updated.

## Reproduction Assessment

Yes for the static defects: the PR head files can be compared directly with current main's config reader, dashboard link helper, platform docs, labeler, and changelog policy. Runtime reproduction is incomplete because the discussion lacks cargo/frontend checks or a Linux/Windows desktop smoke run.

## Solution Assessment

No. A Tauri companion may be a valid direction, but this implementation should reuse the canonical config/dashboard/auth/restart surfaces instead of duplicating them in the new Rust app.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Load Gateway config through canonical contracts:** `apps/desktop/src-tauri/src/main.rs:296-383`
  - body: This reader reparses `openclaw.json` as strict JSON and manually extracts a few fields, so valid OpenClaw configs using JSON5, includes, env substitutions, migrations, or SecretRef-managed values can be treated as missing or different from the running Gateway. The desktop app should consume the same config snapshot or CLI/dashboard contract before deriving URLs.
  - confidence: 0.92
- **[P2] Include the configured Control UI base path:** `apps/desktop/src-tauri/src/main.rs:492-504`
  - body: `build_dashboard_url()` always embeds `/`, but the Gateway can serve the Control UI under `gateway.controlUi.basePath` such as `/openclaw`. Those valid setups will open the wrong page instead of the configured dashboard route.
  - confidence: 0.9
- **[P2] Permit every generated dashboard origin in CSP:** `apps/desktop/src-tauri/tauri.conf.json:14`
  - body: The Rust side can generate HTTPS dashboard URLs and custom-host URLs, but this CSP only permits HTTP/WS localhost origins. Valid generated dashboard origins can therefore be blocked by the WebView policy.
  - confidence: 0.9
- **[P2] Avoid restarting through the ambient shell PATH:** `apps/desktop/src-tauri/src/main.rs:538-558`
  - body: The tray/UI restart action runs `openclaw gateway restart` through `cmd` or `sh -lc`, so a first-party desktop action executes whichever `openclaw` the ambient shell resolves. Resolve a trusted CLI path or call a controlled Gateway restart surface instead of shell/PATH lookup.
  - confidence: 0.86
- **[P2] Add the required release-note entry:** `package.json:1322-1323`
  - body: The branch exposes a new user-facing desktop companion and root `desktop:*` commands, but it does not update `CHANGELOG.md`. Repo policy requires user-facing feature work to add a changelog entry before landing.
  - confidence: 0.84
- **[P2] Update platform docs for the new app surface:** `apps/desktop/README.md:3-4`
  - body: This adds a first-party Linux/Windows desktop companion, while the public Linux platform docs still say native Linux companion apps are planned. Landing as-is would make the shipped docs contradict the new app status.
  - confidence: 0.86
- **[P3] Route desktop app changes through labeler:** `pnpm-workspace.yaml:3`
  - body: Registering `apps/desktop` creates a new app surface, but `.github/labeler.yml` still routes only Android, iOS, macOS, and web UI app changes. Future desktop changes will miss the expected app label until this route is added.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff introduces a concrete command-execution concern in the new desktop restart action.

Concerns:

- **[medium] Ambient shell restart execution:** `apps/desktop/src-tauri/src/main.rs:538`
  - body: `run_gateway_restart()` invokes `cmd /C openclaw gateway restart` on Windows and `sh -lc "openclaw gateway restart"` elsewhere. A first-party desktop app should not execute a PATH/cwd-hijackable command for a privileged-feeling GUI action; it should resolve a trusted CLI path or use a controlled Gateway restart surface.
  - confidence: 0.86
