## Summary

Keep open. Current main still has no Linux/Windows desktop companion, and the PR is aligned with the first-party platform direction, but the branch is not merge-ready: it bypasses the canonical OpenClaw config contract, can generate gateway URLs that its Tauri CSP blocks, and misses required docs, labeler, changelog, and repo typecheck integration. The Greptile P2 items called out earlier appear addressed in the latest PR head.

## What This Changes

The PR adds a new `apps/desktop` Tauri v2 workspace with a React/Tailwind shell, Rust tray/health/restart backend, root `desktop:*` scripts, workspace registration, and npm/Cargo lockfile updates.

## Best Possible Solution

Keep this as the active desktop companion candidate if maintainers want the Tauri direction, but land it only after the app follows the OpenClaw config and dashboard-origin contracts, updates public platform docs and repository routing, adds a user-facing changelog entry, and has Linux/Windows build/runtime proof.

## Reproduction Assessment

Yes for the static review defects: they reproduce by comparing the PR head against current main config, dashboard, docs, labeler, and changelog contracts. Full desktop runtime reproduction was not attempted in this read-only review, and the PR itself notes missing GTK/WebKit validation.

## Solution Assessment

No. The Tauri MVP shape is plausible, but the current patch is not the narrowest maintainable solution until it reuses or faithfully mirrors the canonical config reader/path behavior and keeps generated dashboard origins aligned with the WebView policy; the safer path is to constrain the MVP to loopback or derive the CSP/URL policy from the same effective config contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Parse OpenClaw config through the canonical contract:** `apps/desktop/src-tauri/src/main.rs:297`
  - body: Current OpenClaw config is JSON5, supports `$include`, and can resolve through `OPENCLAW_CONFIG_PATH`, state-dir, and legacy candidates, but this reader uses strict `serde_json::from_str` on its selected file. Valid configs with comments, trailing commas, includes, or non-default path resolution can be treated as invalid/missing, causing the desktop app to fall back to loopback defaults and lose port/auth/bind settings.
  - confidence: 0.91
- **[P2] Keep generated gateway URLs inside the WebView CSP:** `apps/desktop/src-tauri/tauri.conf.json:14`
  - body: The backend can construct dashboard and health URLs from `gateway.bind: "custom"` and `gateway.customBindHost`, but the Tauri CSP only allows `127.0.0.1` and `localhost` for connections and frames. A valid custom host such as a LAN or tailnet address will be generated by Rust and then blocked by the WebView unless the app either restricts this MVP to loopback or safely permits the configured origin.
  - confidence: 0.88
- **[P2] Update platform docs before landing the desktop app:** `apps/desktop/README.md:3-4`
  - body: This adds a Linux/Windows desktop companion, while current public platform docs still say Linux and Windows companions are planned or unavailable. If this lands without docs updates, user-facing platform guidance will immediately contradict the new app surface.
  - confidence: 0.84
- **[P2] Add labeler coverage for the desktop app:** `pnpm-workspace.yaml:3`
  - body: Registering `apps/desktop` as a new workspace creates a new app surface, but `.github/labeler.yml` still has app labels only for Android, iOS, macOS, and web UI. Future desktop changes will miss the expected app label unless this PR adds a desktop route.
  - confidence: 0.82
- **[P2] Add a changelog entry for the desktop feature:** `package.json:1326-1327`
  - body: The PR exposes a user-facing desktop companion app and root `desktop:*` commands, but it does not update `CHANGELOG.md`. Repo policy requires user-facing feature work to add an Unreleased entry with contributor credit before landing.
  - confidence: 0.84
- **[P3] Use repo typecheck lanes instead of a tsc script:** `apps/desktop/package.json:12`
  - body: The new package adds a `typecheck` script that runs `tsc --noEmit`, but this repo standardizes TypeScript checking through the `tsgo` lanes and avoids new `typecheck`/`tsc --noEmit` entry points. Wire desktop validation into the existing check lane instead of adding a separate checker path.
  - confidence: 0.78
