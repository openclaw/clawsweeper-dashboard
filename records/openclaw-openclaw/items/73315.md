## Summary

Keep open. Current main still has no `apps/desktop` implementation, and the PR is active useful work for the Linux/Windows companion-app request, but the branch is not merge-ready because it reimplements existing config/dashboard/restart contracts and introduces a security-sensitive restart path without full validation.

## What This Changes

The branch adds a new Tauri v2 React/Rust desktop companion app under `apps/desktop`, root `desktop:*` scripts, workspace/lockfile entries, and a small tool-display export.

## Best Possible Solution

Land the desktop MVP only after maintainers accept the Tauri and restart model, the branch delegates to existing config/dashboard/auth contracts, public docs/changelog/labeler coverage is updated, and Linux/Windows build/runtime proof is available.

## Reproduction Assessment

Yes for the static defects: the PR head files can be compared directly with current main's config reader, dashboard link helper, platform docs, labeler, and changelog policy. Full runtime reproduction is incomplete because no cargo check, frontend build/typecheck, or desktop smoke proof is available in the discussion.

## Solution Assessment

No. A Tauri companion may be the right product direction, but this branch should reuse the canonical config/dashboard/token contracts and a trusted restart surface instead of reimplementing those behaviors inside a new Rust app.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Load Gateway config through canonical contracts:** `apps/desktop/src-tauri/src/main.rs:296-383`
  - body: This reader reparses `openclaw.json` as strict JSON and manually extracts a few fields, so valid OpenClaw configs using JSON5 comments, `$include`, env substitutions, migrations, or SecretRef-managed values can be treated as missing or different from the running Gateway. The desktop app should consume the same config snapshot or CLI/dashboard contract before deriving URLs.
  - confidence: 0.92
- **[P2] Include the configured Control UI base path:** `apps/desktop/src-tauri/src/main.rs:492-504`
  - body: `build_dashboard_url()` always embeds `/`, but the Gateway can serve the Control UI under `gateway.controlUi.basePath` such as `/openclaw`. Those valid setups will open the wrong page instead of the configured dashboard route.
  - confidence: 0.9
- **[P2] Permit every generated dashboard origin in CSP:** `apps/desktop/src-tauri/tauri.conf.json:14`
  - body: The Rust side can generate HTTPS dashboard URLs when Gateway TLS is enabled and custom-host URLs when `gateway.bind` is `custom`, but this CSP only permits HTTP/WS localhost origins. Valid generated dashboard origins can therefore be blocked by the WebView policy.
  - confidence: 0.9
- **[P2] Avoid restarting through the ambient shell PATH:** `apps/desktop/src-tauri/src/main.rs:538-558`
  - body: The tray/UI restart action runs `openclaw gateway restart` through `cmd` or `sh -lc`, so a first-party desktop action executes whichever `openclaw` the ambient shell resolves. Resolve a trusted CLI path or call a controlled Gateway restart surface instead of shell/PATH lookup.
  - confidence: 0.86
- **[P2] Add the required release-note entry:** `package.json:1322-1323`
  - body: The branch exposes a new user-facing desktop companion and root `desktop:*` commands, but it does not update `CHANGELOG.md`. Repo policy requires user-facing feature work to add a changelog entry before landing.
  - confidence: 0.84
- **[P2] Update platform docs for the new app surface:** `apps/desktop/README.md:3-4`
  - body: This adds a first-party Linux/Windows desktop companion, while the public platform docs still say native Linux and Windows companion apps are planned. Landing as-is would make the shipped docs contradict the new app status.
  - confidence: 0.86
- **[P3] Route desktop app changes through labeler:** `pnpm-workspace.yaml:3`
  - body: Registering `apps/desktop` creates a new app surface, but `.github/labeler.yml` still routes only Android, iOS, macOS, and web UI app changes. Future desktop changes will miss the expected app label until this route is added.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff introduces a concrete command-execution concern in the new desktop restart action.

Concerns:

- **[medium] Ambient shell restart execution:** `apps/desktop/src-tauri/src/main.rs:538`
  - body: `run_gateway_restart()` invokes `cmd /C openclaw gateway restart` on Windows and `sh -lc "openclaw gateway restart"` elsewhere. A first-party desktop app should not execute a PATH/cwd-hijackable command for a privileged-feeling GUI action; it should resolve a trusted CLI path or use a controlled Gateway restart surface.
  - confidence: 0.86
