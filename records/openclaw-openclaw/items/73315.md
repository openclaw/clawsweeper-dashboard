## Summary

Keep open. Current main still has no Linux/Windows desktop companion, so the PR is not obsolete, and it is the active successor to earlier desktop attempts; however, the current branch is not merge-ready because it bypasses canonical config/dashboard contracts, has a security-sensitive restart path, and misses required docs/changelog/labeler follow-through.

## What This Changes

The PR adds a private `apps/desktop` Tauri v2 workspace with a React/Tailwind shell, Rust tray/status/restart backend, root `desktop:*` scripts, workspace and lockfile updates, plus small tool-display and channel module-loader test changes.

## Best Possible Solution

Keep this PR open as the active Linux/Windows desktop companion candidate if maintainers want Tauri, but require it to use the canonical OpenClaw config and dashboard link contracts, constrain restart through a trusted path or Gateway API, align CSP with generated URLs, update platform docs/changelog/labeler coverage, and provide Linux/Windows build/runtime proof before merge.

## Reproduction Assessment

Yes for the static review defects: compare the PR head files against current main's config reader, dashboard URL builder, Control UI docs, labeler, and changelog policy. Full desktop runtime reproduction remains incomplete because the branch lacks `cargo check`, frontend build/typecheck, and platform smoke proof.

## Solution Assessment

No. A Tauri MVP may be a reasonable product direction, but this branch is not the narrowest maintainable merge path until it reuses established config/dashboard contracts and removes the ambient shell/PATH restart execution path.

## Security Review

Concerns:

- **[medium] Ambient shell restart execution:** `apps/desktop/src-tauri/src/main.rs:556`
  - body: `run_gateway_restart()` invokes `sh -lc "openclaw gateway restart"` on non-Windows and `cmd /C openclaw gateway restart` on Windows. A first-party desktop app should not execute a PATH/cwd-hijackable command for a privileged-feeling GUI action; it should resolve a trusted CLI path or call a controlled Gateway restart surface.
