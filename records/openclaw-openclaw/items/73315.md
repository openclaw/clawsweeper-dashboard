## Summary

Keep open. Current main has no desktop app; this PR is useful and supersedes earlier desktop-companion attempts, but it is still merge-blocked by config/dashboard/CSP/docs/changelog/labeler gaps plus a security-sensitive ambient shell restart path.

## What This Changes

Adds a Tauri v2 React/Rust desktop companion under `apps/desktop`, root `desktop:*` scripts, workspace and lockfile entries, and a small tool-display export.

## Best Possible Solution

Land a desktop MVP only after maintainers accept the Tauri/restart model and the branch reuses canonical config, dashboard URL, and restart contracts, updates docs/changelog/labeler coverage, and has Linux/Windows build/runtime proof.

## Reproduction Assessment

Yes for the static defects: compare the PR head against current main's config IO, dashboard helper, platform docs, labeler, and changelog. Full desktop runtime reproduction is incomplete because cargo check, frontend checks, and platform smoke proof are missing.

## Solution Assessment

No. A Tauri companion may be a reasonable direction, but this branch should first delegate to existing config/dashboard contracts and use a trusted restart surface or resolved executable path instead of reimplementing those behaviors in Rust.

Full review comments:

- **[P2] Load Gateway config through the canonical reader:** `apps/desktop/src-tauri/src/main.rs:296-383`
  - body: This reader parses `~/.openclaw/openclaw.json` with strict JSON and manually extracts a few fields, so valid OpenClaw configs using JSON5 comments, `$include`, env substitution, SecretRefs, or runtime migrations can be treated as missing or lose auth. The desktop app should consume the same config snapshot/CLI contract as the Gateway before deriving URLs.
  - confidence: 0.92
- **[P2] Preserve the configured Control UI base path:** `apps/desktop/src-tauri/src/main.rs:492-504`
  - body: `build_dashboard_url()` always points the iframe at `/`, but the Gateway can serve Control UI under `gateway.controlUi.basePath` such as `/openclaw`. Those valid setups will open the wrong page instead of the configured dashboard route.
  - confidence: 0.9
- **[P2] Allow every generated dashboard origin in CSP:** `apps/desktop/src-tauri/tauri.conf.json:14`
  - body: The Rust code can generate `https` loopback URLs when TLS is enabled and custom-host URLs when `gateway.bind` is `custom`, but this CSP only permits `http` loopback frame origins. Valid generated dashboard URLs can be blocked by the WebView policy.
  - confidence: 0.9
- **[P2] Avoid restarting through the ambient shell PATH:** `apps/desktop/src-tauri/src/main.rs:556-558`
  - body: The tray/UI restart action runs `openclaw gateway restart` through `sh -lc` or `cmd /C`, so a first-party desktop action executes whichever `openclaw` the ambient shell resolves. Resolve a trusted CLI path or call a controlled Gateway restart surface instead of shell/PATH lookup.
  - confidence: 0.84
- **[P2] Update platform docs before landing the app:** `apps/desktop/README.md:3-4`
  - body: This adds a first-party Linux/Windows desktop companion, but the public platform docs still say native Linux and Windows companion apps are planned. Landing as-is would make the new app contradict the published user-facing platform status.
  - confidence: 0.86

  - body: The branch exposes a new user-facing desktop companion and root `desktop:*` commands, but it does not update `CHANGELOG.md`. Repo policy requires user-facing feature work to add a release-note entry before landing.
  - confidence: 0.85
- **[P3] Route desktop changes through the labeler:** `pnpm-workspace.yaml:3`
  - body: Registering `apps/desktop` creates a new app surface, but `.github/labeler.yml` still routes only Android, iOS, macOS, and web UI app changes. Future desktop changes will miss the expected app label until this route is added.

## Security Review

Status: needs_attention

Summary: The diff introduces a GUI restart path that shells through ambient PATH resolution.

Concerns:

- **[medium] Ambient shell restart execution:** `apps/desktop/src-tauri/src/main.rs:556`
  - body: `run_gateway_restart()` invokes `sh -lc "openclaw gateway restart"` on non-Windows and `cmd /C openclaw gateway restart` on Windows. A first-party desktop app should not execute a PATH/cwd-hijackable command for a privileged-feeling GUI action; it should resolve a trusted CLI path or use a controlled Gateway restart surface.
  - confidence: 0.84
