## Summary

Keep this open: current main still has an in-process WebChat/image path that can reach sharp/libvips, and the related open optimizer-fallback PR explicitly excludes native SIGILL crashes from its scope.

## What This Changes

The issue reports a hard `openclaw-gateway` SIGILL/core dump in `libvips-cpp.so.8.17.3` during a WebChat image-message flow on Linux x64, Node 22, npm-global OpenClaw v2026.2.22-2.

## Best Possible Solution

Keep the issue open until maintainers reproduce on current main or the latest release, collect kernel/systemd logs plus a core/backtrace, identify the loaded libvips/sharp binary, and then choose a dependency pin, backend fallback, or process-isolation fix with a WebChat image-flow regression smoke.

## Reproduction Assessment

Unclear. The original report has a concrete scenario and crash logs, but there is not yet a high-confidence current-main reproduction because the exact image payload, loaded libvips variant, and core/backtrace are missing.

## Solution Assessment

No. Current main does not yet provide a crash-level solution, and #74243 is intentionally scoped to thrown optimizer failures rather than native process termination; the safer path is a maintainer-led native crash investigation before coding a fix.
