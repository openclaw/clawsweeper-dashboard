## Summary

Keep open: current main still routes WebChat image media through the in-process sharp/libvips optimizer, and the related optimizer-fallback work only handles thrown JavaScript errors, not a native SIGILL that kills the gateway process.

## What This Changes

The issue reports a Linux npm-global `openclaw-gateway` crash with SIGILL in libvips-cpp during a WebChat image-message flow.

## Best Possible Solution

Keep this as the native sharp/libvips SIGILL crash tracker, collect current-release crash artifacts, then choose the narrow packaging, backend-selection, or process-isolation fix with focused WebChat image-flow proof.

## Reproduction Assessment

No high-confidence current-main crash reproduction is available. Source inspection confirms the reported WebChat image path can still reach in-process sharp/libvips, but the native SIGILL itself needs a current-release payload, coredump/backtrace, CPU flags, and loaded binary paths.

## Solution Assessment

Unclear. The current optimizer fallback is the right shape for thrown JavaScript optimizer failures, but this issue needs native crash evidence before maintainers choose packaging repair, backend selection, or isolation.
