## Summary

Keep open. Current main partially improves this area by treating `preview-finalized` as delivered for internal hooks/transcript mirroring, but it still does not expose clear operator-visible delivery-mode parity: plugin `message_sent` hooks are bypassed for preview finalization, `preview-retained` is collapsed into generic delivered state, and the public hook context has no delivery outcome field.

## What This Changes

The issue requests clear Telegram partial-stream final reply observability for sent, preview-finalized, preview-retained, canceled, failed, and skipped outcomes.

## Best Possible Solution

Add an additive Telegram delivery outcome signal that preserves in-place preview finalization while making normal sends, preview-finalized, preview-retained, hook-canceled, transport-failed, and skipped outcomes visible in logs and message-sent events, with focused Telegram dispatch/lane tests and docs.

## Reproduction Assessment

No live high-confidence reproduction was run. Source inspection does show the current-main gap: preview-finalized only emits an internal success hook, preview-retained is marked delivered without equivalent observability, and hook contexts lack a delivery outcome field.

## Solution Assessment

No, closing this as implemented would not be the best solution. The maintainable path is an additive observability contract for Telegram final delivery outcomes, not a delivery-semantics change or a config-only strict mode.
