## Summary

Keep open: current `main` and latest release still have the Telegram partial-stream delivery-state ordering that can make preview-finalized finals look delivered before all final follow-up sends settle, and the open closing implementation at https://github.com/openclaw/openclaw/pull/84563 is not merged.

## What This Changes

The issue asks for clear Telegram partial-stream final reply observability across sent, preview-finalized, preview-retained, canceled, failed, and skipped delivery outcomes.

## Best Possible Solution

Rebase, review, and prove https://github.com/openclaw/openclaw/pull/84563 or a narrow replacement that delays delivered-state marking until all visible final delivery work settles and records clear preview-finalized versus sent versus failed outcomes.

## Reproduction Assessment

Yes at source level: current Telegram lane delivery marks final preview delivery before all follow-up chunks settle, and the linked PR discussion includes production logs showing the same symptom. I did not run a live Telegram reproduction in this read-only pass.

## Solution Assessment

Yes for the current fix direction: the narrowest maintainable path is to keep this in the Telegram delivery layer and delay delivered-state/hook success until the final visible delivery outcome is known. The open PR is not yet a final solution because it still needs rebase, review, and real Telegram proof.
