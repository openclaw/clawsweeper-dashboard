## Summary

This issue should stay open: current main and v2026.6.8 still have Telegram preview-finalized paths that mark delivery before all long-final follow-up chunks settle, and https://github.com/openclaw/openclaw/pull/84563 is the active but unmerged fix path.

## What This Changes

The issue asks for Telegram partial-stream finalization to expose clear final-delivery outcomes such as sent, preview-finalized, preview-retained, hook-canceled, transport-failed, and skipped.

## Best Possible Solution

Rebase, prove, and land https://github.com/openclaw/openclaw/pull/84563 or a current-main successor that keeps Telegram's one-preview lifecycle while delaying delivered-state success until all final delivery chunks and hooks settle.

## Reproduction Assessment

Yes at source level, but not from a live current-main run in this review. Current main and v2026.6.8 mark Telegram preview-finalized long finals delivered before continuation sends settle, and the linked PR plus production log comments provide the focused failure path.

## Solution Assessment

Yes as a direction: delaying delivered-state success until final follow-up sends and hook cancellation paths settle is the right Telegram-owned fix. The current merge path still needs maintainer review, rebase/proof handling, and a decision on whether any broader strict-mode or outcome-taxonomy surface belongs in this fix.
