## What This Changes

The PR rewrites `collectPromptCacheToolNames()` to iterate indexed tool rows with guarded row/name reads and adds a regression test for poisoned tool-name rows.

## Best Possible Solution

Rebase or replace this with a narrow current-main patch that shares a safe tool-name reader across prompt-cache observability and session allowlist collection, with focused tests for throwing rows, throwing getters, and non-string names.

## Reproduction Assessment

Yes. Source inspection shows `collectRegisteredToolNames()` reads `tool.name` before prompt-cache observation, and a focused Node probe confirmed poisoned indexed array rows throw before a loop body guard can run.

## Solution Assessment

No. Guarding only this diagnostics helper is too narrow, and the PR head also leaves `trim()` outside the guard; the safer fix is a shared safe name reader or equivalent upstream normalization.
