## Summary

## What This Changes

This PR changes collectPromptCacheToolNames() to skip poisoned tool rows or name getters and adds a focused regression test for that diagnostic helper.

## Best Possible Solution

Rebase the branch and move the guarded tool-name read into the shared allowlist/name-collection path so session allowlists and prompt-cache diagnostics both tolerate unreadable descriptors, with focused coverage for both helpers.

## Reproduction Assessment

Yes. Source inspection and a Node probe show current main's for-of collection can still throw on a poisoned array row, and run/attempt.ts reads the same allCustomTools list through collectRegisteredToolNames() before prompt-cache observability.

## Solution Assessment

No. Guarding only collectPromptCacheToolNames() is too narrow; the best fix is a shared guarded name reader or equivalent protection in the allowlist path that runs earlier.
