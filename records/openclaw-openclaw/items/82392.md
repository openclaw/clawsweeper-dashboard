## Summary

Keep open: the Android URL-tap bug is real and this PR targets the right renderer, but it should not merge yet because it turns every CommonMark destination into a tappable Android URI without the scheme filtering OpenClaw already applies to web markdown, and the submitted proof is only unit-level.

## What This Changes

The PR changes Android ChatMarkdown to wrap CommonMark link nodes in Compose LinkAnnotation.Url metadata and adds unit tests for bare URLs, explicit markdown links, and plain text.

## Best Possible Solution

Merge a scoped Android renderer fix that annotates only safe URL schemes, preserves existing link styling, and includes focused tests plus emulator or device tap proof.

## Reproduction Assessment

Yes from source, but not from a live Android run in this review. Current main parses bare URLs as CommonMark links, then renders them as styled text without URL metadata or a tap handler.

## Solution Assessment

No. The renderer location is right, but the current PR should filter unsafe schemes before adding LinkAnnotation.Url and should provide real Android tap proof before merge.
