## Summary

Keep open: current main still restarts non-manual stopped WhatsApp accounts without a terminal-disconnect guard, and an open closing PR remains the active fix candidate for the primary bug.

## What This Changes

The issue reports that terminal WhatsApp disconnects such as logged-out or replaced sessions are collapsed into generic stopped state and repeatedly restarted by gateway health recovery, with optional follow-up requests for richer status/events and restart-budget controls.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/78511 for the primary terminal-disconnect no-restart guard, then track status/event and restart-budget expansion separately if maintainers want those surfaces.

## Solution Assessment

Yes for the primary bug: carrying terminal disconnect state into the existing generic health/restart policy is the narrowest maintainable fix. The proposed new gateway event, top-level status reason, and restart-budget config need separate maintainer product/API decisions.
