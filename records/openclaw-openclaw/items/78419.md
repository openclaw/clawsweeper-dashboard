## Summary

Keep this open: current main still has a source-reproducible WhatsApp terminal-disconnect restart path, and an open linked fix pull request already references this issue with closing syntax, so the issue should track that review until the fix lands or is replaced.

## What This Changes

The issue requests that WhatsApp terminal disconnects such as logged-out or session-conflict stops stop automatic gateway restarts while still allowing retryable disconnects to recover.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/78511, or replace it with an equivalent current-main patch that propagates terminal WhatsApp state and guards both restart paths while deferring broader event/config policy.

## Reproduction Assessment

Yes. Source inspection on current main shows terminal WhatsApp closes can leave `healthState` as logged-out/conflict with `running=false`, while the generic policy returns `not-running` and both restart paths can start the channel again; I did not run a live 12-account WhatsApp setup.

## Solution Assessment

Yes for the primary bug. The narrow maintainable fix is to carry a terminal disconnect signal into the existing channel snapshot/status path and skip both automatic restart paths; new event fields and restart-budget configuration can be handled separately.
