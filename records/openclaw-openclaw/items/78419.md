## Summary

Keep open: current main still has a source-reproducible WhatsApp terminal-disconnect restart path, and an open closing pull request is already attached, so this issue should track that review until the fix lands or is replaced.

## What This Changes

The issue asks OpenClaw to classify terminal WhatsApp disconnects as requiring reauth, avoid automatic restarts for them, and expose enough status signal for operators.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/78511, or replace it with an equivalent current-main patch that propagates terminal WhatsApp disconnect state and skips both automatic restart paths while tracking optional event/config policy separately.

## Reproduction Assessment

Yes. Source inspection on current main shows logged-out/conflict WhatsApp stops can become generic stopped snapshots, and both the health monitor and ChannelManager restart non-manual stopped accounts; I did not run a live 12-account WhatsApp setup.

## Solution Assessment

Yes for the primary bug. Propagating a terminal disconnect signal through the existing channel snapshot/status path and guarding both restart paths is narrower than adding new events or restart-budget configuration in the same fix.
