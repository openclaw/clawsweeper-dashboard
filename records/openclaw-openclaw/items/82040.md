## Summary

Keep open: current main still builds the Telegram conversation-context block without same-session dedupe, and the open closing PR at https://github.com/openclaw/openclaw/pull/82194 is the canonical fix path. This issue should close only after that PR, or an equivalent fix, is proven and merged.

## What This Changes

The issue reports that Telegram direct-chat prompt context repeats messages already present in the persistent session transcript, wasting tokens and degrading response quality.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/82194 after live or equivalent Telegram proof, then close this issue as implemented.

## Reproduction Assessment

Yes, at source level: current main records Telegram messages into the cache, selects recent cached messages into `Conversation context`, and carries that current-turn context alongside the session transcript. I did not run a live Telegram bot session in this read-only review.

## Solution Assessment

Yes, the linked PR direction is the narrowest maintainable fix: add Telegram-local session-bound dedupe for direct-chat ambient context while preserving reply targets and group context. It still needs proof and merge before this issue should close.
