The issue asks OpenClaw to stop Telegram prompt conversation context from duplicating messages already present in persistent session history.

Yes for the current fix direction: a Telegram-local session-bound dedupe that preserves explicit reply targets is the narrowest maintainable solution. The issue should stay open until the linked PR or an equivalent fix lands and is proven.
