## Summary

Keep open: current main still attempts explicit overflow compaction before the fallback tool-result truncation check, and the open linked PR with closing syntax is the right path to resolve this after review and merge.

## What This Changes

The issue asks OpenClaw to try cheap tool-result truncation before waiting for full LLM auto-compaction when tool-heavy context overflow occurs.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81190 if normal checks remain clean, then close this issue as implemented by that merge.

## Reproduction Assessment

Yes. Source inspection on current main shows overflow compaction runs before the tool-result truncation check, and the supplied live log shows a 912840ms compaction timeout followed by immediate truncation of 281 tool results.

## Solution Assessment

Yes. The narrow maintainable solution is to reuse the existing tool-result truncation heuristic before explicit overflow compaction for tool-heavy snapshots, then fall back to compaction when truncation is unavailable or insufficient.
