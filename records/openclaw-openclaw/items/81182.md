## Summary

Keep open: current main still attempts explicit overflow compaction before the fallback tool-result truncation check, and an open linked PR with closing syntax is the right path to resolve this after review/merge.

## What This Changes

Request that tool-heavy context overflow recovery try deterministic tool-result truncation before waiting for the full LLM auto-compaction timeout.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81190 if normal CI and maintainer checks stay clean, then close this issue after the merge.

## Reproduction Assessment

Yes. Source inspection on current main shows overflow compaction runs before the tool-result truncation check, and the linked live logs show a 912840ms compaction timeout followed by immediate truncation of 281 tool results; I did not run a live Telegram overflow.

## Solution Assessment

Yes for the linked fix direction. The narrow maintainable solution is to reuse the existing tool-result truncation heuristic before explicit overflow compaction for tool-heavy snapshots, then fall back to compaction when truncation is unavailable or insufficient.
