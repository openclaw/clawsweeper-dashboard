## Summary

Keep open: current main and the latest release still run explicit overflow compaction before the general tool-result truncation fallback, while the linked implementation candidate is open and not merged.

## What This Changes

The issue asks for tool-heavy context-overflow recovery to try deterministic tool-result truncation before waiting for the full LLM auto-compaction timeout.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81190, preserving compaction as the fallback when early tool-result truncation does not reduce context.

## Reproduction Assessment

Yes, from source inspection: current main still starts explicit overflow compaction before the later tool-result truncation fallback, and adjacent tests assert that ordering. I did not run a live Telegram reproduction in this read-only review.

## Solution Assessment

Yes, the narrowest maintainable fix is to try the existing tool-result truncation heuristic before explicit overflow compaction, then keep compaction as fallback when truncation does not help. The open linked PR is the current implementation candidate.
