## What This Changes

The issue asks OpenClaw to truncate or prune accumulated tool results before waiting for full overflow auto-compaction in tool-heavy sessions.

## Best Possible Solution

Review and land the linked narrow run-loop PR if it preserves the existing compaction fallback; otherwise apply the same truncation-first ordering with focused regression coverage.
