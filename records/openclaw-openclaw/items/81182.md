## Summary

Keep open: current main still performs explicit overflow compaction before fallback tool-result truncation, and the linked open PR with closing syntax is the right resolution path.

## What This Changes

The issue asks OpenClaw to try deterministic tool-result truncation before waiting for full LLM auto-compaction timeout in tool-heavy overflow recovery.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81190 if it remains clean under normal maintainer/CI gates, then close this issue after merge.

## Reproduction Assessment

Yes, as a source-level reproduction. Current main calls contextEngine.compact before the truncation fallback, and the issue includes live logs showing a 912840ms compaction timeout followed immediately by truncating 281 tool results.

## Solution Assessment

Yes for the linked fix direction. The narrow maintainable solution is to reuse the existing tool-result truncation heuristic before explicit overflow compaction for tool-heavy snapshots, then fall back to compaction if truncation is unavailable or insufficient.
