## Summary

Keep this PR open. The security hardening direction is valid, but the branch is stale against current main: builtin memory_get now routes through packages/memory-host-sdk, while the PR still changes the older src/memory-host-sdk host implementation, so merging as-is would not reliably harden the current builtin runtime path.

## What This Changes

The PR adds realpath-based containment checks for memory_get builtin and QMD reads, exports the new helper, and adds regression coverage for directory-symlink escapes and transient missing-file races.

## Best Possible Solution

Move the containment helper, readMemoryFile changes, and package exports into packages/memory-host-sdk, keep src/memory-host-sdk bridge files thin, apply the same canonical containment primitive to QMD reads, and keep the added regression tests on the current package-backed runtime surface.

## Reproduction Assessment

Yes. The PR body's reproduction path matches current source: create an outside Markdown file, place an intermediate directory symlink under memory/, an allowed extra path, or a QMD collection, then call memory_get; current main checks lexical containment and final lstat, not canonical root containment.

## Solution Assessment

No, not as currently shaped for main. Realpath containment is the right fix primitive, but the PR needs to target the current package-owned memory-host-sdk runtime/export surface rather than the older src implementation.
