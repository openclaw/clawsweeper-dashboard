## Summary

Keep open: this is still a real memory-boundary hardening PR, and current main only shipped the extraPath portion through https://github.com/openclaw/openclaw/pull/80331. The branch is not merge-ready because it targets removed runtime files, uses overly broad workspace roots, and lacks external real behavior proof.

## What This Changes

The PR adds a realpath containment helper for memory_get workspace, extraPath, and QMD reads, plus symlink escape regression tests.

## Best Possible Solution

Rebase and port the fix to the package-owned memory runtime, reject symlink-parent escapes for workspace memory and QMD collection/default reads, and preserve the shipped extraPath behavior from https://github.com/openclaw/openclaw/pull/80331.

## Reproduction Assessment

Yes, source-reproducible. Current main has lexical workspace-memory and QMD read paths that stat/read the requested path without canonical containment against the intended memory or collection root; I did not run a live repro because this sweep is read-only.

## Solution Assessment

No, not as proposed. Realpath or symlink-parent containment is the right direction, but this branch must be ported to packages/memory-host-sdk, use memory/QMD roots rather than workspace fallback roots, and preserve https://github.com/openclaw/openclaw/pull/80331.
