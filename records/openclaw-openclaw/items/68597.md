## Summary

Keep open: the security bug is plausible and partly addressed elsewhere, but this PR is still useful only after a rebase/replacement because it conflicts with current main, targets an obsolete host path, and leaves builtin/QMD containment too broad.

## What This Changes

The PR adds realpath containment checks and regression tests for memory_get directory symlink escapes in builtin memory, extraPaths, and QMD-backed reads.

## Best Possible Solution

Rebase or replace this branch with package-owned builtin and QMD read hardening that preserves the merged extraPath guard, checks exact memory/collection roots, and includes redacted after-fix proof.

## Reproduction Assessment

Yes, source-reproducible. Current main lexically authorizes builtin/QMD read paths and fs-safe regular-file helpers reject the final symlink only, so an intermediate directory symlink can redirect the read path; I did not run a live repro because this review is read-only.

## Solution Assessment

No, not as proposed. Realpath or parent-symlink containment is the right direction, but the patch must target the active package runtime and use narrow memory/QMD roots instead of broad workspace roots.
