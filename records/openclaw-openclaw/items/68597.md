## Summary

Keep open. This is plausible security hardening and the related merged work only covers extraPaths, but this PR is not merge-ready because the branch is conflicting, targets the old memory-host runtime, leaves root selection too broad in the proposed fix, and lacks external real behavior proof.

## What This Changes

The PR adds realpath-based containment checks and regression tests for memory_get reads through workspace memory, extraPaths, and QMD symlinked directories.

## Best Possible Solution

Rebase or replace the branch with a package-runtime fix that preserves the merged extraPath hardening, adds canonical containment for workspace memory and QMD roots, and includes redacted real behavior proof.

## Reproduction Assessment

Yes, source-reproducible. Current main still has QMD read paths that lexically resolve collection/default memory paths and then stat/read absPath without canonical containment against the intended QMD root; I did not run a live repro because this review is read-only.

## Solution Assessment

No, not as proposed. Realpath or symlink-parent containment is the right direction, but this branch must be ported to packages/memory-host-sdk, use the memory/QMD roots rather than a broad workspace root, and preserve https://github.com/openclaw/openclaw/pull/80331.

  - confidence: 0.8
