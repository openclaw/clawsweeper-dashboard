## Summary

Keep open: the PR is a valid security-hardening direction, but it is not merge-ready because it targets stale runtime paths, leaves memory/QMD roots too broad, and lacks required real behavior proof.

## What This Changes

The branch adds a realpath containment helper under `src/memory-host-sdk`, applies it to builtin and QMD memory reads, and adds symlink-escape regression tests.

## Best Possible Solution

Rebase or replace the branch with a package-runtime fix that constrains builtin memory reads to canonical memory roots, constrains QMD reads to the matched collection root, preserves https://github.com/openclaw/openclaw/pull/80331, and includes redacted after-fix proof.

## Reproduction Assessment

Yes, source-reproducible: current main authorizes memory/QMD read paths lexically and the fs-safe file helpers only reject the leaf symlink unless parent/root containment is added. I did not run a live repro in this read-only review.

## Solution Assessment

No, not as proposed. Realpath or parent-symlink containment is the right direction, but the fix must land in `packages/memory-host-sdk`, use memory/QMD roots instead of the whole workspace, and preserve the merged extraPath hardening.
