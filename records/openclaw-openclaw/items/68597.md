## Summary

Keep open: the security hardening is valid and recent, but this branch is not mergeable as-is because it targets the old memory-host path, leaves some memory/QMD roots too broad, and lacks required real behavior proof. The merged extraPath fix in https://github.com/openclaw/openclaw/pull/80331 covers only part of the original scope.

## What This Changes

The branch adds a realpath containment helper under `src/memory-host-sdk`, applies it to builtin and QMD memory reads, and adds symlink-escape regression tests.

## Best Possible Solution

Replace or rebase the branch with a package-runtime fix that preserves the merged extraPath hardening, constrains builtin memory and QMD reads to their canonical memory or collection roots, and includes redacted after-fix proof.

## Reproduction Assessment

Yes. The source path is high-confidence: current QMD/default reads resolve paths lexically and fs-safe regular-file helpers only reject the final symlink unless parent checks are added; I did not run a live repro in this read-only review.

## Solution Assessment

No, not as proposed. Realpath or parent-symlink containment is the right direction, but it must land in `packages/memory-host-sdk`, use memory/QMD roots rather than the whole workspace, and preserve https://github.com/openclaw/openclaw/pull/80331.
