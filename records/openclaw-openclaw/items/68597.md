## Summary

Keep open. The PR addresses a real memory_get symlink-boundary bug, but current main only has the narrower extraPaths fix from https://github.com/openclaw/openclaw/pull/80331; workspace and QMD reads still need canonical containment, and this branch targets the old src memory-host path plus lacks external real behavior proof.

## What This Changes

The PR adds realpath containment checks and regression tests intended to make memory_get reject directory-symlink escapes for workspace, extraPath, and QMD reads.

## Best Possible Solution

Port the containment helper into packages/memory-host-sdk, export it through the current plugin SDK storage surface, apply it to workspace and QMD memory_get reads, preserve the extraPaths behavior already merged in https://github.com/openclaw/openclaw/pull/80331, and add focused regression tests.

## Reproduction Assessment

Yes, source-reproducible. Current main's workspace memory read and QMD read paths resolve lexically and then stat/read absPath without canonical root containment; I did not run a live test because this review is read-only.

## Solution Assessment

No, not as currently shaped. Realpath containment is the right direction, but the branch must be rebased onto packages/memory-host-sdk and the current plugin SDK export path; the already-merged extraPaths fix is only a partial overlap.
