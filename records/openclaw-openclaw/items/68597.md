## Summary

Keep open: the security problem is still only partly addressed on main, but this branch is not merge-ready because it is stale against the package-owned runtime and leaves root containment too broad.

## What This Changes

The PR adds realpath containment checks for memory_get reads, exports a new contained-file helper from the old memory-host SDK path, and adds builtin/QMD symlink-escape tests.

## Best Possible Solution

A rebased replacement should apply per-root canonical containment in packages/memory-host-sdk and QMD read paths while preserving the merged extraPaths hardening from https://github.com/openclaw/openclaw/pull/80331.

## Reproduction Assessment

Yes, source-reproducible: current main still has lexical workspace/QMD read authorization paths where a directory symlink can cross the intended memory or collection boundary. I did not run a live repro because this was a read-only review.

## Solution Assessment

No, not as proposed. Realpath or parent-symlink containment is the right direction, but the fix needs to land in packages/memory-host-sdk and QMD should authorize against exact memory or collection roots rather than broad workspace fallback.
