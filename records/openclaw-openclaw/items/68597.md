## Summary

Keep open: this PR targets a real memory read boundary issue, but the branch is not merge-ready because it patches the old `src/memory-host-sdk` path, leaves workspace and QMD roots too broad, conflicts with current `main`, and still lacks external real behavior proof. The merged extra-path fix at https://github.com/openclaw/openclaw/pull/80331 covers only one part of the original scope.

## What This Changes

The PR adds a realpath containment helper under the old memory-host SDK path, applies it to builtin and QMD memory reads, and adds symlink-escape regression tests.

## Best Possible Solution

Replace or rebase the branch with a package-runtime fix that preserves the merged extraPath hardening, constrains default memory and QMD reads to canonical memory or collection roots, and includes redacted after-fix proof.

## Reproduction Assessment

Yes. The issue is source-reproducible: current default-memory and QMD reads use lexical path authorization with file helpers that do not by themselves constrain parent-symlink targets to the intended memory or collection root; I did not run a live repro in this read-only review.

## Solution Assessment

No, not as proposed. Realpath or parent-symlink containment is the right direction, but it must land in `packages/memory-host-sdk`, preserve https://github.com/openclaw/openclaw/pull/80331, and use narrower memory/QMD roots than the whole workspace.
