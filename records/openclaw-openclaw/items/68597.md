## What This Changes

The branch adds a realpath containment helper under `src/memory-host-sdk`, applies it to memory_get/QMD reads, and adds directory-symlink escape regression tests.

## Best Possible Solution

Rebase or replace the branch with a package-runtime fix that preserves the merged extraPath hardening, adds canonical containment for default memory and QMD roots, and includes focused regression coverage plus redacted real behavior proof.

## Reproduction Assessment

Yes, source-reproducible. Current main still has QMD/default-memory read paths that lexically resolve the requested path and then stat/read `absPath` without canonical containment against the intended memory or QMD collection root; I did not run a live repro because this review is read-only.

## Solution Assessment

No, not as proposed. Realpath/canonical containment is the right direction, but this branch must be ported to `packages/memory-host-sdk`, use narrower memory/QMD roots instead of the broad workspace root, and preserve the merged extraPath fix from https://github.com/openclaw/openclaw/pull/80331.
