## Summary

Keep this PR open: the two touched conversions are useful, but the MSTeams Graph attachment flow still has an unbounded main message JSON read before hosted-content collection processing, so the OOM class is only partially covered.

## What This Changes

The branch replaces two MSTeams attachment-path `response.json()` calls with the shared `readProviderJsonResponse` bounded reader in Bot Framework attachment info and Graph hosted-content collection fetches.

## Best Possible Solution

Complete the MSTeams attachment JSON boundary by using the shared bounded reader for the main Graph message response too, preserving existing parse-failure degradation and adding focused regression proof.

## Reproduction Assessment

Yes. Source inspection shows current main uses unbounded `response.json()` in the touched attachment metadata and collection paths, and the PR body provides copied Node live output showing bounded reads canceling for the two changed paths.

## Solution Assessment

No. The proposed helper is the right mechanism, but the best fix should also cover the main Graph message JSON read in the same attachment flow before this lands.
