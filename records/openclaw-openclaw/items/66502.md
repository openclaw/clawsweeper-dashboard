## Summary

Keep open: this is a plausible gateway bug and current main still lacks the versioned `.pnpm/.../openclaw.mjs` to stable package-root fallback, but the PR still has a correctness blocker because the stable pnpm candidate is tried after the version-pinned dist candidate.

## What This Changes

The PR adds pnpm package-root candidate discovery and a regression test so gateway service ProgramArguments can resolve a versioned `.pnpm/.../openclaw.mjs` invocation to a stable `node_modules/openclaw/dist` entrypoint.

## Best Possible Solution

Update the resolver so versioned `.pnpm/<package>@<version>/node_modules/<package>/openclaw.mjs` inputs derive stable `node_modules/<package>/dist/*` candidates before version-pinned dist candidates, while preserving wrapper, `.bin`, and existing dist-symlink behavior.

## Reproduction Assessment

Yes, source inspection gives a high-confidence deterministic fixture: set `process.argv[1]` to a versioned pnpm `openclaw.mjs` path, make both versioned and stable dist entries accessible, and the current/PR ordering selects the versioned dist first. I did not run tests because this review was required to keep the checkout read-only.

## Solution Assessment

No. The PR is directionally right, but appending the stable pnpm package candidates after normal package-root dist candidates is not the narrow complete fix; the stable candidate needs priority and coverage where both paths exist.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Try stable pnpm package roots before versioned dist:** `src/daemon/program-args.ts:85`
  - body: This appends the stable `node_modules/openclaw` candidate after the normal package-root dist candidates. With the PR's versioned entrypoint path shape, the versioned package `dist/index.js` can exist and be returned before the stable symlink path is ever tried, leaving the service pinned to the old package version after an upgrade. Move the stable pnpm candidate ahead of the versioned package dist candidate and cover the case where both are accessible.
  - confidence: 0.9
