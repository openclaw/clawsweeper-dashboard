## Summary

Keep this PR open: current main still lacks the versioned pnpm `openclaw.mjs` fallback, but the proposed patch is not merge-ready because it appends the stable package-root candidate after the version-pinned package `dist` candidates and the PR still lacks required real behavior proof.

## What This Changes

The PR adds pnpm package-root candidate discovery in `src/daemon/program-args.ts` plus a regression test for versioned global `openclaw.mjs` gateway service entrypoint resolution.

## Best Possible Solution

Update the resolver so stable `node_modules/<package>/dist/*` candidates derived from versioned `.pnpm/.../openclaw.mjs` inputs are tried before version-pinned package `dist` candidates, then add real pnpm global service proof.

## Reproduction Assessment

Yes. Source inspection gives a deterministic fixture: set `process.argv[1]` to a versioned pnpm `openclaw.mjs` path, make both versioned and stable dist entries accessible, and the PR ordering still selects the versioned dist first.

## Solution Assessment

No. The PR is directionally right, but appending stable pnpm candidates after normal package-root dist candidates is not the narrow complete fix because the stable candidate needs priority when both paths exist.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Try stable pnpm package roots before versioned dist:** `src/daemon/program-args.ts:85`

Move the stable pnpm candidate ahead of the versioned package dist candidate and cover the case where both are accessible.
