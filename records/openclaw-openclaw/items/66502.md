## Summary

Keep open: the PR addresses a real gateway service path stability gap, but the proposed candidate ordering can still select a version-pinned `.pnpm` dist path before the stable symlinked package root, and the PR lacks required real behavior proof.

## What This Changes

The branch adds pnpm `.pnpm/.../node_modules/<package>` path rewriting to gateway service entrypoint candidate discovery and a regression test.

## Best Possible Solution

Prioritize stable `node_modules/<package>/dist/*` candidates before versioned `.pnpm` package dist candidates, cover the both-paths-accessible case, and attach redacted real pnpm global service proof.

## Reproduction Assessment

## Solution Assessment

No. The direction is right, but appending stable pnpm candidates after the normal package-root dist candidates is not the narrow complete fix because stable candidates need priority when both paths exist.
