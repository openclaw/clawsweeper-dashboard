## Summary

Keep open: current main still deletes the legacy timeout value without preserving it and still gives no valid-config runtime/CLI warning; #74940 remains the open closing PR, so this issue should stay open until that implementation lands or is declined.

## What This Changes

The issue asks `doctor --fix` to preserve or echo `agents.defaults.llm.idleTimeoutSeconds` and add a runtime warning for the retired key before users run doctor.

## Best Possible Solution

Keep the retired key out of runtime semantics, but land or decline #74940 or an equivalent narrow patch that echoes the removed numeric value and adds a one-shot legacy-key warning.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: current migration and tests delete the value, config loading silently applies that migration, and the runtime resolver cannot consume the legacy path.

## Solution Assessment

Yes. The requested diagnostic-focused fix is narrower and more maintainable than reviving `agents.defaults.llm` as a live runtime input; automatic provider writes should only happen if the target is unambiguous.
