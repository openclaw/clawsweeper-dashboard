## What This Changes

The issue reports that `openclaw doctor --fix` removes legacy `agents.defaults.llm.idleTimeoutSeconds` without preserving the configured timeout value, while normal runtime startup gives no warning before doctor runs.

## Best Possible Solution

Keep `agents.defaults.llm` retired, but update the doctor repair path so the removed numeric timeout is visible or carried to the correct provider timeout target, add a one-shot pre-doctor warning when the legacy block is still present, and resolve the issue through review of PR #74940 or an equivalent narrow patch.

## Solution Assessment

Yes. The best direction is a narrow migration/diagnostic fix that preserves user intent and warns on the retired key, not reintroducing `agents.defaults.llm` as a supported runtime timeout knob.
