## Summary

Keep open: current main and v2026.6.10 still provide only generic editable workspace files and narrow plugin UI descriptors, not a pinned-facts panel with clear always-injected context semantics or a settled security/product boundary.

## What This Changes

Request a Control UI panel or sidebar widget for adding, editing, and deleting user-defined pinned facts that are always injected into agent context.

## Best Possible Solution

Define a scoped memory/context UI and storage contract, coordinated with the memory-panel and contribution-slot decisions, before adding browser write access to always-injected facts.
