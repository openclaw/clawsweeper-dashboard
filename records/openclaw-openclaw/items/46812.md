## Summary

Keep this open for a maintainer product decision. Current main now provides safe raw editing of USER.md and MEMORY.md in the Control UI, but it does not provide the requested structured pinned-facts UX; the request also cannot treat those files as interchangeable because native Codex receives USER.md as instructions while MEMORY.md is usually retrieved on demand. The same author’s broader memory-panel issue remains open, so the clean path is to consolidate the UI design rather than close one half or start an automated fix.

## What This Changes

Add a Control UI panel where users can manage structured persistent facts that influence agent context on future turns.

## Best Possible Solution

Design pinned facts as a constrained, round-trippable USER.md section inside the broader memory UI, clearly disclose its persistent instruction effect, and handle any generic plugin panel contract as a separate proposal.

## Reproduction Assessment

Not applicable: this is a feature request. Source inspection gives a high-confidence gap check showing raw workspace-file editing exists, but no structured pinned-facts editor exists.

## Solution Assessment

No, not with USER.md and MEMORY.md treated interchangeably or with a new dedicated state file. A constrained USER.md-backed view coordinated with the open memory-panel design is the narrowest contract-aligned solution.
