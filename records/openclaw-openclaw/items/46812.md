## Summary

Keep open. Current main and v2026.6.6 only expose generic whole-file editing for allowlisted bootstrap files plus root MEMORY.md, and the current plugin UI descriptor surface is narrower than the persistent panel discussed in comments; the requested structured pinned facts and guaranteed-injection semantics remain a product and security design item.

## What This Changes

Request to add a Control UI pinned context panel for user-defined persistent facts that can be edited and guaranteed in future agent context.

## Best Possible Solution

Design a scoped pinned-facts or memory UI backed by an approved bootstrap/context contract, and route broader persistent plugin panels through the Control UI contribution-slots design rather than a one-off panel shortcut.
