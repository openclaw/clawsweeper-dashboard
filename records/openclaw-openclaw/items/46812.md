## Summary

Keep open. Current main has a generic Agents > Files editor for bootstrap files plus root MEMORY.md, a memory-wiki plugin, and a narrow Control UI descriptor foundation, but it does not provide the requested pinned key-value persistent-facts panel or a storage/injection contract for it.

## What This Changes

The issue requests a Control UI panel for adding, editing, deleting, and always injecting user-defined persistent facts without editing raw workspace files.

## Best Possible Solution

Keep this open until maintainers choose whether pinned facts belong in the broader Memory panel at https://github.com/openclaw/openclaw/issues/46809, the Control UI contribution-slot design at https://github.com/openclaw/openclaw/issues/71736, or a dedicated first-party pinned-context API with strict storage and injection rules.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high-confidence current behavior: there is no pinned-context route, structured facts editor, dedicated pinned facts API, or plugin panel surface that would satisfy the request.

## Solution Assessment

Unclear. A pinned key-value panel is plausible, but the maintainable direction should be decided with the memory-file API and Control UI contribution-slot work instead of adding an ad hoc always-injected panel.
