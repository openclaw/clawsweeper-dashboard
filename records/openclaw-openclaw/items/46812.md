## Summary

Keep open. Current main and v2026.6.6 still only expose a generic whole-file editor for allowlisted workspace files such as USER.md and MEMORY.md; the requested structured pinned facts panel, always-injected versus last-seen semantics, diff/save UX, and plugin panel boundary remain unresolved product and security work.

## What This Changes

Add a Control UI pinned context panel for user-defined persistent facts that can be edited from the UI and injected into future agent context.

## Best Possible Solution

Choose a scoped pinned-facts or memory UI contract that defines storage, always-injected versus last-seen semantics, authorization, diff/save UX, and its relationship to the broader Control UI contribution-slot RFC before implementation.
