## Summary

Keep open. Current main and v2026.5.19 still provide a generic allowlisted Markdown file editor plus a narrow plugin UI descriptor foundation, but not pinned facts storage, a guaranteed pinned-facts injection contract, or the requested Control UI panel; the related memory-panel and contribution-slot issues overlap without fully replacing this request.

## What This Changes

Add a Control UI panel/sidebar for user-defined key-value facts that users can edit and have injected into every agent turn.

## Best Possible Solution

Define a versioned pinned-facts storage and injection contract, then add scoped Gateway RPCs and a Control UI surface aligned with the memory-panel and contribution-slot decisions.

Unclear. A pinned panel is plausible, but maintainers should first choose the storage/API boundary and whether this belongs in dedicated memory UX, broader workspace-file management, or Control UI contribution slots.
