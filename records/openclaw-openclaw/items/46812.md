## Summary

Keep open: current main and v2026.6.10 provide a generic Agents > Files editor for bootstrap files plus root MEMORY.md, but they do not provide a pinned key-value facts panel, a dedicated pinned-context store, or settled always-injected/stale-fact semantics. The request overlaps https://github.com/openclaw/openclaw/issues/46809 and https://github.com/openclaw/openclaw/issues/71736, but neither currently owns the specific pinned persistent-facts UX end to end.

## What This Changes

The issue requests a Control UI panel for adding, editing, deleting, and always injecting user-defined persistent facts into agent context.

## Best Possible Solution

Keep this as a maintainer-owned feature/design item until OpenClaw chooses the pinned-facts storage model, stale-fact semantics, and safe Control UI write boundary, then implement it in coordination with the memory panel and plugin contribution-slot work.
