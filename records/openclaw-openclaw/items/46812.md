## Summary

Keep open. Current main still lacks a pinned facts editor, structured pinned-facts storage, and an every-turn context-injection guarantee; the existing implementation is whole-file workspace Markdown editing plus unrelated browser-local pinned chat messages. 1fanwang's sidebar/controlUiPanels comments and the open #71736 RFC are useful design input, but they are not a merged implementation of this request.

## What This Changes

Add a Control UI panel or sidebar widget for creating, editing, deleting, and persistently injecting user-defined key-value facts into agent context.

## Best Possible Solution

Keep this open until maintainers approve a design that covers both pieces: a safe Control UI contribution/panel contract and a pinned facts editor backed by a documented structured store or USER.md section, with tests proving how those facts enter normal agent context under budget, lightweight, continuation, and disabled-injection modes.

## Reproduction Assessment

Not applicable as a feature request. The source-level verification is high-confidence: current main has raw workspace-file editing and browser-local chat pins, but no pinned facts CRUD UI, storage format, or prompt guarantee.

## Solution Assessment

Unclear. The request points in a useful direction, but the narrowest maintainable solution should be chosen with the open Control UI contribution-slot work so pinned facts do not create a one-off panel/storage API that conflicts with the emerging plugin boundary.
