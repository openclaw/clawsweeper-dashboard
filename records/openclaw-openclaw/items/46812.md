## Summary

Keep open. Current main and v2026.6.6 expose a generic Core Files editor for allowlisted bootstrap files plus root MEMORY.md, and browser-local pinned chat display, but they do not provide structured pinned facts, guaranteed injection for a dedicated pinned-context file, or the persistent/plugin Control UI panel discussed in comments.

## What This Changes

Request to add a Control UI pinned-facts panel for user-defined persistent context that is editable from the UI and injected into every agent turn.

## Best Possible Solution

Design a scoped pinned-facts or memory UI backed by an approved bootstrap/context contract, and route broader persistent plugin panels through the plugin UI extension design thread rather than a one-off Control UI shortcut.
