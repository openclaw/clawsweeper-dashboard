## Summary

Keep open. Current main and v2026.6.8 provide a generic Core Files editor for bootstrap files plus MEMORY.md, but they do not implement a structured pinned-facts panel, key/value editing, explicit always-injected versus last-seen semantics, diff-preview save behavior, or a decided storage/security model.

## What This Changes

Request to add a Control UI panel for user-defined persistent facts that can be edited inline and injected into agent context on every turn.

## Best Possible Solution

Design a scoped pinned-facts Control UI and memory contract with explicit storage ownership, trust rules, always-injected versus last-seen semantics, visible save diffs, and targeted Gateway/UI tests before implementation.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing behavior report. Source and release inspection confirm the current product only has generic whole-file editing, not a structured pinned-facts panel.

## Solution Assessment

Unclear: the requested UX is useful, but raw USER.md sections versus a dedicated pinned-context store is not decidable without maintainer choices for storage ownership, freshness semantics, and browser write security.
