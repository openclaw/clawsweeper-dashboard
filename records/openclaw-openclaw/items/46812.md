## Summary

Keep open: current main has guaranteed workspace-file injection and a raw Control UI workspace-file editor, but not the requested pinned key-value facts panel or a mounted persistent panel surface for plugins.

## What This Changes

Add a Control UI panel for creating, editing, and deleting user-defined persistent facts that are injected into agent context every turn.

## Best Possible Solution

Define a first-class pinned-facts workflow, likely reusing the existing workspace-file injection guarantees while adding a structured Control UI editor or the missing manifest-first panel seam that can host it.
