## Summary

Keep open: current main and v2026.6.10 only provide raw allowlisted workspace-file editing, not a structured pinned key-value facts panel or guaranteed-injection contract. The remaining work needs maintainer product and security review before implementation.

Request a Control UI panel/sidebar for user-defined key-value facts that are persisted and injected into every agent turn.

## Best Possible Solution

Define a first-class pinned-facts contract before implementation: storage owner, always-injected versus last-seen semantics, safe save/audit flow, and tests proving bounded context injection.
