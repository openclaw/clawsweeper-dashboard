## Summary

Keep open: current main exposes raw agent workspace markdown file editing and session workspace viewing, but it does not provide the requested pinned key-value facts panel or a structured guaranteed-injection contract. The remaining work is a product and security decision, not a narrow bug fix.

## What This Changes

Add a Control UI panel for user-defined pinned key-value facts backed by persistent workspace context and injected on every agent turn.

## Best Possible Solution

Define a first-class pinned-facts contract before implementation: storage owner, always-injected versus last-seen semantics, safe save/audit flow, and tests proving bounded context injection.

## Reproduction Assessment

Not applicable as a feature request. Source inspection shows no current pinned-facts API or UI, only raw workspace-file editing for known markdown files.

## Solution Assessment

Unclear, because the requested direction is not yet specified enough to be the best implementation. The existing raw file editor is a useful partial surface, but maintainers still need to decide pinned-fact semantics and security boundaries before code should land.
