## Summary

Keep open. Current main still exposes raw workspace-file editing, local chat-message pinning, and a narrow plugin UI descriptor projection, but it does not provide a structured pinned facts editor, dedicated pinned-facts storage, or an every-turn context-injection guarantee. The related comments and open #71736 RFC point to a broader Control UI contribution design, so this remains valid but needs maintainer design review before implementation.

## What This Changes

Add a Control UI panel or sidebar widget for creating, editing, deleting, and persistently injecting user-defined key-value facts into agent context.

## Best Possible Solution

Design and land a safe manifest-first Control UI contribution surface plus a dedicated pinned-facts store or documented USER.md section, then wire a host-owned CRUD UI and tests proving injection behavior under bootstrap budgets, lightweight/continuation modes, and docs.
