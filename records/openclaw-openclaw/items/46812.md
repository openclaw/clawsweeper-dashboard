## Summary

Keep open. Current main still has a generic allowlisted agent file editor and a narrow plugin UI descriptor foundation, but it does not provide structured pinned facts, a guaranteed always-injected facts contract, or the requested Control UI panel; related memory-panel and contribution-slot issues are still open design work rather than a shipped replacement.

## What This Changes

Add a Control UI panel for user-defined pinned key-value facts that can be edited without opening raw workspace files and are injected into agent context on every turn.

## Best Possible Solution

Define a versioned pinned-facts storage and injection contract, then add scoped Gateway RPCs and a Control UI surface aligned with the memory-panel and contribution-slot decisions.

## Reproduction Assessment

Not applicable; this is a feature request, not a bug report. Source inspection on current main gives a high-confidence feature-gap check: there is generic agent file editing and narrow Control UI descriptors, but no pinned facts panel, storage contract, or persistent panel seam.

## Solution Assessment

Unclear. A pinned panel is plausible, but maintainers should first choose the storage/API boundary and whether this belongs in dedicated memory UX, broader workspace-file management, or Control UI contribution slots.
