## Summary

Keep open. Current main still has only a generic agent-file Markdown editor and a narrow plugin Control UI descriptor foundation; it does not provide a pinned key-value facts panel, a pinned-facts storage contract, or a persistent panel/native-page seam that would make this cleanup-ready.

## What This Changes

Add a Control UI panel for user-defined pinned facts that can be edited inline and injected into the agent context on every turn.

## Best Possible Solution

Define a versioned pinned-facts storage and injection model, then add a scoped Gateway/UI surface aligned with the memory panel and Control UI contribution-slot direction after product and security review.

Not applicable; this is a feature request, not a bug report. Source inspection gives a high-confidence absence check: current main has generic file editing and narrow plugin descriptors, but no pinned facts panel, storage contract, or persistent panel seam.

## Solution Assessment
