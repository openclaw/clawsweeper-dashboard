## Summary

Keep open. Current main still has a generic agent-file Markdown editor and a narrow plugin Control UI descriptor surface, but it does not provide a pinned key-value facts panel, a pinned-facts storage/injection contract, or a persistent panel seam that makes this cleanup-ready.

## What This Changes

Add a Control UI panel for user-defined persistent key-value facts that are always injected into the agent context.

## Best Possible Solution

Define a versioned pinned-facts storage and injection model, then add a scoped Gateway/UI surface aligned with memory UX and Control UI contribution-slot direction after product and security review.

## Reproduction Assessment

Not applicable; this is a feature request, not a bug report. Source inspection gives high-confidence absence evidence for a pinned facts panel, storage contract, and persistent panel seam on current main.

## Solution Assessment

Unclear. A pinned panel is plausible, but maintainers should first choose the storage/API boundary and whether this belongs in dedicated memory UX, broader workspace-file management, or Control UI contribution slots.
