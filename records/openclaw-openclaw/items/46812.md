## Summary

Keep open. Current main only exposes generic agent file editing for bootstrap files plus root MEMORY.md; it still lacks a pinned key-value facts UX, a pinned-facts storage contract, and a persistent Control UI panel/contribution seam. Related memory, workspace-file, and plugin-slot threads are useful context but are either open, unmerged, or broader than this request.

## What This Changes

Requests a Control UI pinned-facts panel or sidebar for adding and editing always-injected user facts without raw USER.md or MEMORY.md editing.

## Best Possible Solution

Define a versioned pinned-facts storage contract and Control UI surface, or deliberately fold the capability into the Memory/workspace file manager and plugin contribution-slot direction with memory-specific shortcuts.
