## Summary

Keep this open: current main has a Control UI raw Markdown editor for core agent files and a backend plugin UI descriptor seam, but it does not provide a pinned key-value facts panel, guaranteed quick-fact workflow, or SPA-mounted persistent plugin panel described in the discussion.

## What This Changes

Add a discoverable Control UI panel/sidebar widget for adding, editing, deleting, and always injecting user-defined persistent facts.

## Best Possible Solution

Make a maintainer product/API decision first, then implement either a first-class pinned facts editor backed by the existing bootstrap context contract or a generalized Control UI panel path that the SPA actually renders.
