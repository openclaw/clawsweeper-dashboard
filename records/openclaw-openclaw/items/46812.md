## Summary

Keep open. Current main only partially addresses raw file editing through the generic Agents > Files editor for USER.md and MEMORY.md; it still lacks a pinned key-value context panel, a pinned-facts storage contract, and a Control UI route/tab for this UX. The comments usefully identify a broader persistent Control UI panel seam, so this needs maintainer product/API direction rather than cleanup.

## What This Changes

Request a Control UI sidebar or panel for adding, editing, and deleting user-defined facts that are injected into every agent turn.

## Best Possible Solution

Add a scoped pinned-facts UX on top of the existing workspace/context contracts, or fold the capability into the memory/workspace panel direction after maintainers choose the storage and panel API shape.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high confidence that current main has a generic Agents > Files editor but no pinned-facts panel, storage contract, or Control UI route for this request.

## Solution Assessment

Unclear. A pinned panel is plausible, but the maintainable path should be decided alongside https://github.com/openclaw/openclaw/issues/46809, https://github.com/openclaw/openclaw/issues/61368, and https://github.com/openclaw/openclaw/pull/62417 rather than adding an ad hoc file or panel bypass.
