## Summary

Keep open. Current main and v2026.6.6 have a generic Agents > Files editor for allowlisted workspace files, but they do not provide a pinned key-value context panel, a dedicated pinned-context storage/injection contract, or a Control UI contribution slot that would make the request obsolete.

## What This Changes

Request to add a Control UI panel for user-defined persistent facts that can be edited inline and injected into agent context on every turn.

## Best Possible Solution

Define a maintainer-approved pinned-context model that distinguishes durable always-injected facts from stale-prone last-seen facts, exposes it through a scoped Gateway/UI contract, and coordinates with the broader Memory panel and Control UI contribution-slot work.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing behavior report. Source inspection gives a high-confidence feature-gap check: current main only has a generic allowlisted file editor, not a pinned facts panel or injection contract.

## Solution Assessment

Unclear; the UX goal is valid, but the issue’s proposed storage choices need maintainer product and security direction before implementation. A scoped pinned-context contract with stale-fact handling is safer than treating raw USER.md edits as the whole solution.

- A browser-facing editor for personal memory/context needs explicit authorization, workspace containment, and user-visible write confirmation before implementation.
