## Summary

Keep open. Current main still has a generic Agents Files Markdown editor and a narrow plugin Control UI descriptor foundation, but it does not provide a pinned key-value facts panel, a pinned-facts storage/injection contract, or a persistent panel/native-page seam that would make this cleanup-ready.

## What This Changes

The issue requests a Control UI pinned context panel for adding, editing, and deleting user-defined facts that are always available to the agent.

## Best Possible Solution

Define a versioned pinned-facts storage and injection model, then add a scoped Gateway and Control UI surface aligned with the memory panel and contribution-slot direction after product and security review.

## Reproduction Assessment

Not applicable; this is a feature request, not a bug report. Source inspection gives a high-confidence absence check: current main has generic file editing and narrow plugin descriptors, but no pinned facts panel, storage contract, or persistent panel seam.

## Solution Assessment

Unclear. A pinned panel is plausible, but maintainers should first choose the storage/API boundary and whether this belongs in dedicated memory UX, broader workspace-file management, or Control UI contribution slots.
