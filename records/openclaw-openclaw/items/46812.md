## Summary

Keep open. Current main still has only a generic allowlisted workspace-file editor plus a narrow plugin UI descriptor foundation; it does not provide pinned facts storage, a structured always-injected facts contract, or the requested Control UI panel, and the overlapping memory-panel and contribution-slot issues do not fully replace this feature.

## What This Changes

Add a Control UI pinned context panel for editable user-defined facts that are guaranteed to be injected into every agent turn.

## Best Possible Solution

Define a versioned pinned-facts storage and injection contract, then add scoped Gateway RPCs and a Control UI surface aligned with the memory-panel and contribution-slot decisions.

Unclear. A pinned panel is plausible, but maintainers should first choose the storage/API boundary and whether this belongs in dedicated memory UX, broader workspace-file management, or Control UI contribution slots.
