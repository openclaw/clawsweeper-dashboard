## Summary

Keep open: current main still lacks a primary cron delivery default, and the live discussion already preserves this as the focused tracker while the broader cron-defaults proposal remains open.

## What This Changes

Request adding `cron.defaults.delivery` so isolated cron jobs can inherit a default announce channel and target instead of repeating per-job delivery fields.

## Best Possible Solution

Ship a documented, schema-backed primary cron delivery default, or fold the same precedence into an approved unified `cron.defaults` design, with explicit job delivery winning before any config default and the existing fallback last.

## Reproduction Assessment

Not applicable as a bug reproduction. This is a new public config-option request, and source inspection of current main verifies the requested config path and precedence behavior are absent.

## Solution Assessment

Unclear until maintainers choose the public config contract. `cron.defaults.delivery` is focused and plausible, but it should be reconciled with https://github.com/openclaw/openclaw/issues/57638 before implementation.
