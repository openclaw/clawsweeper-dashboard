## Summary

Keep open: current main still lacks a `cron.defaults.delivery` or equivalent primary cron announce delivery default, while the narrower duplicate was closed back to this tracker and the broader cron-defaults proposal remains open rather than superseding it.

## What This Changes

The issue requests a new `cron.defaults.delivery` config option that supplies a default announce delivery channel/target while preserving per-job delivery overrides.

## Best Possible Solution

Ship a documented, schema-backed primary cron delivery default, or fold the same precedence into an approved unified `cron.defaults` design while keeping explicit per-job delivery first and the existing fallback last.
