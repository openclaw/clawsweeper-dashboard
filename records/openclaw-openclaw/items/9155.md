## Summary

Keep open: current main still lacks `cron.defaults.delivery` or an equivalent primary cron delivery default, and the issue discussion already preserves this as the focused tracker while the broader cron-defaults proposal remains open.

## What This Changes

The issue requests a `cron.defaults.delivery` config option that supplies a default cron announce delivery channel and target when a job omits per-job delivery fields.

## Best Possible Solution

Add a documented, schema-backed primary cron delivery default or fold the same precedence into an approved unified `cron.defaults` design, with per-job delivery winning before any config default and existing fallback behavior last.

## Reproduction Assessment

Not applicable as a bug reproduction. This is a new public config-option request, and source inspection on current main verifies the requested config path and precedence behavior are absent.

## Solution Assessment

Unclear until maintainers choose the public config contract. `cron.defaults.delivery` is focused and plausible, but it should be reconciled with https://github.com/openclaw/openclaw/issues/57638 before implementation.
