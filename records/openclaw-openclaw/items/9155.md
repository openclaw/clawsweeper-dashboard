## Summary

Keep open: current main still lacks `cron.defaults.delivery` or an equivalent primary cron announce delivery default, and the related broader defaults proposal remains open rather than shipped.

## What This Changes

Add a `cron.defaults.delivery` configuration default so cron jobs can inherit a channel/target instead of repeating per-job announce delivery fields.

## Best Possible Solution

Add a documented, schema-backed cron delivery default only after maintainers settle the defaults shape and precedence with the broader cron defaults design.
