## Summary

Keep open: current main and v2026.5.19 still lack `cron.defaults.delivery` or an equivalent primary cron announce-delivery default, and this remains the focused tracker while the broader cron-defaults issue is still open.

## What This Changes

Request to add a `cron.defaults.delivery` config path that supplies a default cron delivery mode/channel/target when a job omits explicit delivery.

## Best Possible Solution

Approve one cron delivery-default contract, then implement typed config, strict schema/help/docs, runtime precedence, CLI/UI behavior, and regression tests for per-job delivery over cron default over `last`.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection on current main and v2026.5.19 gives high-confidence proof that the requested config path and precedence behavior are absent.

## Solution Assessment

Unclear until maintainers choose the public cron-defaults contract. The requested path is plausible and focused, but recipient-routing safety and overlap with https://github.com/openclaw/openclaw/issues/57638 should be resolved before implementation.
