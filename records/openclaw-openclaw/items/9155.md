## Summary

Keep open: current main and the latest shipped release still lack `cron.defaults.delivery` or an equivalent primary cron announce-delivery default. The closed duplicate points back here as the focused tracker, while the broader open cron-defaults issue is not a replacement yet.

## What This Changes

Request to add a `cron.defaults.delivery` configuration path that supplies default cron delivery settings when per-job delivery is omitted.

## Best Possible Solution

Keep this as the focused tracker until maintainers approve `cron.defaults.delivery` or fold it into a broader cron defaults design, then implement typed config, strict schema/help/docs, runtime precedence, CLI/UI support, and regression tests for per-job delivery over cron default over `last`.
