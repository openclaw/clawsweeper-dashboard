## What This Changes

The issue requests a new `cron.defaults.delivery` config option so cron jobs can inherit a default announce delivery channel and target when per-job delivery is omitted.

## Best Possible Solution

Ship a documented, schema-backed primary cron delivery default, or fold the same precedence into an approved unified `cron.defaults` design while preserving per-job delivery first and the existing fallback last.
