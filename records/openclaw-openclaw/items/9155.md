## Summary

Keep open: current main and v2026.5.22 still lack `cron.defaults.delivery` or an equivalent primary cron announce target default. The narrower request remains useful despite the broader open cron-defaults proposal because the related duplicate was closed back to this focused tracker and the change needs product/security review before implementation.

## What This Changes

Add a `cron.defaults.delivery` config default that supplies a cron announce delivery channel/target before the existing `last` fallback.

## Best Possible Solution

Keep this as the focused delivery-default tracker until maintainers approve either `cron.defaults.delivery` or a broader cron-defaults contract, then implement typed config, strict schema/help/docs, runtime precedence, CLI and Control UI support, and regression coverage.

## Reproduction Assessment

Yes, at source level: current `CronConfig` and the strict zod schema have no `cron.defaults`, and job creation plus delivery planning still fall through explicit delivery, `{ mode: "announce" }`, and `last`. I did not run a runtime repro because this is a read-only feature review.

## Solution Assessment

Unclear until maintainers choose the public cron-defaults contract. The requested path is focused and plausible, but it should be reconciled with https://github.com/openclaw/openclaw/issues/57638 and reviewed for recipient-routing safety.
