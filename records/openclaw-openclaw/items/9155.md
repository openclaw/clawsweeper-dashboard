## Summary

Keep open: current main and the latest release still lack `cron.defaults.delivery` or an equivalent primary cron announce-delivery default. Prior triage also kept this as the focused tracker, while https://github.com/openclaw/openclaw/issues/57638 is broader and https://github.com/openclaw/openclaw/issues/41164 was closed back to this issue.

## What This Changes

Add a `cron.defaults.delivery` config option that supplies a default announce delivery channel and target when a cron job does not define one.

## Best Possible Solution

Keep this focused tracker open until maintainers either approve `cron.defaults.delivery` or explicitly fold it into the broader cron defaults design, then implement typed config, schema/help/docs, runtime precedence, CLI/UI support, and regression tests.

## Reproduction Assessment

## Solution Assessment
