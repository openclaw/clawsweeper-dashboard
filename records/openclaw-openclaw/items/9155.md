## Summary

Keep open: current main and the latest release still lack a primary `cron.defaults.delivery` or equivalent default announce target, and the broader cron-defaults issue remains open rather than a merged replacement. The request is useful but needs maintainer product and security review because it changes scheduled-output recipient routing.

## What This Changes

Add a `cron.defaults.delivery` config option so cron jobs can inherit a default announce delivery channel and target instead of repeating per-job delivery config.

## Best Possible Solution

Keep this as the focused delivery-default tracker until maintainers approve either `cron.defaults.delivery` or a broader cron-defaults contract, then implement typed config, strict schema/help/docs, runtime precedence, CLI and Control UI support, and regression coverage.

## Solution Assessment
