## What This Changes

Add a `cron.defaults.delivery` config option so cron jobs without per-job delivery targets can use a configured default channel and recipient before falling back to `last`.

## Best Possible Solution

Add a documented, schema-backed cron delivery default, or fold the same precedence into the approved unified `cron.defaults` design, with regression coverage for job-level delivery before config default before `last`.

## Reproduction Assessment

## Solution Assessment
