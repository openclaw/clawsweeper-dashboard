## Summary

Keep open: current main still lacks `cron.defaults.delivery` or an equivalent primary cron announce-delivery default, and the broader open cron-defaults request has not superseded this focused delivery-default tracker.

## What This Changes

Request to add `cron.defaults.delivery` so cron jobs can inherit a default announce channel and target when no per-job delivery is configured.

## Best Possible Solution

Keep this as the focused tracker until maintainers approve `cron.defaults.delivery` or fold it into one cron-defaults design, then implement typed config, strict schema/help/docs, runtime precedence, CLI/UI support, and regression tests for per-job delivery over cron default over `last`.

Not applicable as a bug reproduction. Source inspection on current main gives high-confidence proof that the requested config path and precedence behavior are absent.

## Solution Assessment

Unclear until maintainers choose the public cron defaults contract. The requested path is plausible and focused, but it should be reconciled with https://github.com/openclaw/openclaw/issues/57638 and reviewed for recipient-routing safety before implementation.
