## Summary

Keep open: current main and the latest release still lack `cron.defaults.delivery` or an equivalent primary cron announce-delivery default, and the broader open cron defaults issue has not replaced this focused tracker.

## What This Changes

Requests a new `cron.defaults.delivery` config option so cron jobs can inherit a default delivery channel and target before falling back to `last`.

## Best Possible Solution

Keep this as the focused tracker until maintainers approve `cron.defaults.delivery` or fold it into one cron-defaults design, then implement typed config, strict schema/help/docs, runtime precedence, CLI/UI support, and regression tests for per-job delivery over cron default over `last`.
