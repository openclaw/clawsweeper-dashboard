## Summary

Keep open: current main and latest release still lack `cron.defaults.delivery` or an equivalent primary cron announce delivery default, and the focused request is not fully superseded by the broader open cron-defaults proposal.

## What This Changes

The issue requests a `cron.defaults.delivery` configuration path so cron jobs can inherit a default announce delivery channel and target when per-job delivery is omitted.

## Best Possible Solution

Keep this as the focused delivery-default tracker until maintainers approve either `cron.defaults.delivery` or a broader cron-defaults contract, then implement typed config, schema/help/docs, runtime precedence, CLI/UI support, and regression coverage.

## Reproduction Assessment

Yes, at source level: current `CronConfig` and the strict zod schema have no `cron.defaults`, and job creation plus delivery planning still fall through explicit delivery, `{ mode: "announce" }`, and `last`. I did not run a runtime repro because this is a read-only feature review.

## Solution Assessment

Unclear until maintainers choose the public cron-defaults contract. The requested path is focused and plausible, but it should be reconciled with https://github.com/openclaw/openclaw/issues/57638 and reviewed for recipient-routing safety.
