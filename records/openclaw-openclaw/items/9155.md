## What This Changes

Request to add a schema-backed `cron.defaults.delivery` config option that supplies a default cron delivery mode/channel/target when a job omits explicit delivery.

## Best Possible Solution

Ship a documented, schema-backed primary cron delivery default, or fold the same precedence into an approved unified `cron.defaults` design, with explicit job delivery first, config default second, and the existing fallback last.

## Reproduction Assessment

## Solution Assessment
