Keep open: current main and the latest release still lack `cron.defaults.delivery` or an equivalent primary cron announce-delivery default, and the related thread history keeps this as the focused delivery-default tracker while the broader cron defaults proposal remains open.

The issue requests a new `cron.defaults.delivery` config option so cron jobs can inherit a default announce delivery channel and target when job-level delivery is omitted.
