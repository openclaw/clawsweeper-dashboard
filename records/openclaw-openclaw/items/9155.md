## Summary

Keep open: current main and the latest release still lack the requested primary cron delivery default, and the broader cron-defaults tracker is open rather than a merged replacement. The implementation path is source-clear but needs product and security review because it changes scheduled-output recipient routing.

## What This Changes

Add a `cron.defaults.delivery` configuration path so cron jobs without explicit delivery can inherit a configured default channel and target before falling back to current behavior.

## Best Possible Solution

Keep this as the focused delivery-default tracker until maintainers approve either `cron.defaults.delivery` or a broader cron-defaults contract, then implement typed config, schema/help/docs, runtime precedence, UI/CLI support, and regression coverage.
