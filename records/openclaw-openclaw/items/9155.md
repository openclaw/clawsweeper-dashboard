## Label Justifications

- `P2`: This is a normal-priority cron/config enhancement with clear user value and bounded implementation surface, but it needs product and security review.
- `impact:security`: A cron delivery default controls where scheduled output may be sent, which can affect sensitive-data routing.
- `impact:message-loss`: The request is motivated by cron output being misrouted or not delivered when jobs fall back to unstable `last` routing.

## Summary

Keep open: current main and the latest release still do not expose `cron.defaults.delivery` or an equivalent primary cron announce target default. The request remains useful, but it needs maintainer product and security review because it changes scheduled-output recipient routing.

## What This Changes

Add a `cron.defaults.delivery` config option that supplies a default cron delivery mode, channel, and target when an individual job omits delivery settings.

## Best Possible Solution

Keep this as the focused delivery-default tracker until maintainers approve either `cron.defaults.delivery` or a broader cron-defaults contract, then implement typed config, strict schema/help/docs, runtime precedence, CLI and Control UI support, and regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction, but source inspection is high-confidence: current main lacks `cron.defaults.delivery` and falls back only through explicit job delivery, `{ mode: "announce" }`, and `last` delivery planning.

## Solution Assessment

Unclear until maintainers choose the public cron-defaults contract. The requested path is focused and plausible, but it should be reconciled with https://github.com/openclaw/openclaw/issues/57638 and reviewed for recipient-routing safety.
