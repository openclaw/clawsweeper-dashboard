## Summary

Keep open: current main and v2026.5.12 still lack `cron.defaults.delivery` or an equivalent primary cron announce-delivery default, and the broader open cron-defaults tracker at https://github.com/openclaw/openclaw/issues/57638 has not superseded this focused request.

## What This Changes

The issue requests a new `cron.defaults.delivery` config option that supplies a default cron delivery mode, channel, target, and precedence below per-job delivery but above the existing `last` fallback.

## Best Possible Solution

Keep this as the focused tracker until maintainers approve `cron.defaults.delivery` or fold it into one cron-defaults design, then implement typed config, schema/help/docs, runtime precedence, UI/CLI support, and regression tests for per-job delivery over cron default over `last`.
