## Summary

Keep this open. Current main and the latest release still lack `cron.defaults.delivery` or an equivalent primary cron delivery default, and related closed items either point back here or cover adjacent default-policy work rather than the requested default target config.

## What This Changes

Add a `cron.defaults.delivery` config option that supplies a default cron announce channel and target when jobs omit explicit delivery fields.

## Best Possible Solution

Define one maintainer-approved primary cron delivery-default contract, then update config types/schema/help/docs, cron create/normalize paths, CLI and Control UI surfaces, and tests for per-job delivery first, default delivery second, and last-route fallback last.

## Reproduction Assessment

Yes at source level. Current main and `v2026.6.6` still have no `cron.defaults.delivery`; omitted detached cron delivery becomes announce-only and then plans as `channel: "last"`.

## Solution Assessment

Unclear. A configured default target is plausible and aligned with the reported need, but the best fix depends on maintainer policy for default target versus default none and create-time versus run-time persistence.
