## Summary

Keep open: the localization fix is directionally valid, but this patch is not merge-ready because it bypasses the generated Control UI i18n pipeline and lacks after-fix browser proof.

## What This Changes

The PR adds chat welcome/run-control translation keys, wraps selected Control UI chat labels/tooltips/placeholders with `t()`, and changes several zh-CN `skills` labels to Chinese.

## Best Possible Solution

Update the source strings through the Control UI i18n pipeline, commit the regenerated locale/metadata/raw-copy outputs, and provide redacted browser proof showing the zh-CN chat labels rendering.

## Reproduction Assessment

Yes, source-level: current main hardcodes the reported chat strings in `chat-welcome.ts`, `run-controls.ts`, and `views/chat.ts`, so selecting zh-CN cannot translate those labels because they bypass `t()`. I did not run a browser session, so this is source-reproducible rather than fully reproduced.

## Solution Assessment

No: wrapping these strings with `t()` is the right direction, but the current PR is incomplete until generated i18n artifacts are synced and browser behavior proof is provided.
