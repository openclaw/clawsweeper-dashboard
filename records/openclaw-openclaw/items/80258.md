## Summary

Keep open: the setup-fallback flag change is relevant for callers that reach `buildChannelSummary`, but the reported `openclaw status --json` path still disables channel summary construction before this helper can run.

## What This Changes

The PR flips `listChannelSummaryPlugins` to include setup-fallback plugins and adds a focused Vitest assertion for that resolver option.

## Best Possible Solution

Define the JSON status contract, then either populate `channelSummary` in the fast JSON path using read-only setup-fallback metadata without starting channel runtimes, or narrow/document this fix as applying only to non-CLI summary callers.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `openclaw status --json` uses `scanStatusJsonFast`, which passes `includeChannelSummary: false`, so the CLI JSON payload can still contain `channelSummary: []` before this PR's helper runs.

## Solution Assessment

No. The resolver flag change is a useful part of the lower-level fix, but the reported CLI command needs the fast JSON status path and its tests updated, or the PR needs to be scoped away from `openclaw status --json`.
