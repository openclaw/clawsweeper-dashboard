## Summary

Keep open. Current main does not implement the requested Android `http.request` node command, and the PR contains useful work, but the branch is not safe to merge as-is because it default-allows a general Android HTTP proxy without a settled security policy and still has an opt-in/reconnect behavior bug plus missing public release/docs coverage.

## What This Changes

The PR adds an Android `http.request` node invoke command, Android settings opt-in, gateway Android allowlist support, protocol constants, and Android/gateway tests.

## Best Possible Solution

Keep the Android-owned implementation path only if maintainers accept the feature, but require an explicit security policy for arbitrary node HTTP egress, fix opt-in command declaration refresh, and add changelog/docs before merge.

## Reproduction Assessment

Yes for the blocking PR behavior: at PR head, enabling HTTP Access only writes prefs, while command declarations are built into the node connect options and the gateway rejects non-declared commands. Current main does not contain `http.request`, so the feature itself is not reproducible on main yet.

## Solution Assessment

No. The Android node is the plausible owner for this capability, but default gateway allowlisting plus protocol-only URL validation is not the safest maintainable shape until the security/product policy is decided, and the switch needs to update the connected node session behavior.

  - body: Adding `http.request` to the Android platform defaults means any Android node that advertises the command can be used as an arbitrary HTTP(S) egress proxy without explicit gateway config or plugin policy.   - confidence: 0.9

  - body: This adds a public node command, Android setting, and JSON contract, but the branch does not update `CHANGELOG.md` or the nodes docs. Operators need the opt-in, parameters, timeout/body limits, and security expectations documented before shipping a user-facing `feat`.

  - body: `NODE_HTTP_REQUEST_COMMAND` is added to Android platform defaults, so once the Android app advertises it the gateway can forward arbitrary URL fetches without explicit operator config.
