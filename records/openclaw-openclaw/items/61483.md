## Summary

Keep this PR open as the canonical implementation candidate, but it is not mergeable as-is because the new Android HTTP egress path still needs maintainer policy/security decisions and has a source-visible toggle/session bug.

## What This Changes

The PR adds an Android `http.request` node command with a settings toggle, Android handler/registry/dispatcher wiring, gateway allowlist changes, and tests.

## Best Possible Solution

Land a narrower implementation after the Android HTTP egress and destination policy is chosen, then enforce that policy in the gateway/handler and refresh or reset the active node command snapshot when HTTP Access changes.

## Reproduction Assessment

Yes for the PR review blockers: source inspection shows default allowlisting, scheme-only URL validation, and no session refresh after enabling the toggle. I did not establish a live Android failure path in this read-only review.

## Solution Assessment

No. The Android handler is a plausible owner, but default gateway allowlisting plus unconstrained HTTP(S) destinations is not the narrowest maintainable boundary until maintainers choose the egress and destination policy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Require policy before defaulting Android HTTP egress:** `src/gateway/node-command-policy.ts:103`
  - body: Adding `NODE_HTTP_REQUEST_COMMAND` to Android `PLATFORM_DEFAULTS` lets any paired Android node that advertises it fetch arbitrary HTTP(S) destinations from the phone network without explicit operator config or a reviewed destination policy. Keep it out of defaults, or add and document the reviewed gateway policy before default-allowing this egress path.
  - confidence: 0.9
- **[P2] Refresh the node session when HTTP Access turns on:** `apps/android/app/src/main/java/ai/openclaw/app/NodeRuntime.kt:801-803`
  - body: `setHttpAccessEnabled` only writes the preference, but the node declares commands at connect time and the gateway checks the active session's declared command snapshot. Enabling HTTP Access after the node is already connected can leave `http.request` undeclared until reconnect or re-approval, making the normal off-to-on toggle path appear broken.
  - confidence: 0.86
