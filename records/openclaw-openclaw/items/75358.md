## Summary

Keep open. Current main still drops PDF page images when `@napi-rs/canvas` cannot load, and open PR #75370 already uses closing syntax for this issue, so it should wait for that PR to be reviewed, merged, or rejected.

## What This Changes

Request a Poppler `pdftoppm` fallback for PDF page image extraction when the bundled `document-extract` plugin cannot load optional `@napi-rs/canvas`.

## Best Possible Solution

Review and either land a bounded plugin-owned Poppler fallback in PR #75370 or explicitly surface and document canvas-missing image degradation if Poppler is rejected.

## Reproduction Assessment

Yes. Source inspection on current main shows a low-text PDF reaches `loadCanvasModule()`, catches a missing optional canvas import, and returns `{ text, images: [] }`; the agent PDF tool route does not pass the warning callback.

## Solution Assessment

Unclear. A `pdftoppm` fallback is plausible and belongs in the `document-extract` plugin, but external PDF rendering needs maintainer/security review before it is the accepted best solution.
