## Summary

Keep open. Current main still has the reported canvas-only PDF image extraction failure path, and open PR #75370 uses closing syntax for the same fix, so this issue should wait for that PR's review rather than be closed or queued separately.

## What This Changes

The issue requests a `pdftoppm`/Poppler PDF page-image rendering fallback when `@napi-rs/canvas` is unavailable.

## Best Possible Solution

Review and land the linked plugin-local Poppler fallback after changelog, tests, and execution-bound checks, or explicitly surface and document canvas-missing image degradation if Poppler is rejected.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: a low-text PDF reaches image rendering, the missing `@napi-rs/canvas` import is caught, and the extractor returns text with `images: []`; the issue also provides concrete Ubuntu 24.04 host evidence.

## Solution Assessment

Unclear. A bounded `pdftoppm` fallback in `document-extract` is the narrow owner-module direction, but it should be accepted through the linked PR after security and coverage review.
