## Summary

Keep open. Current main still drops PDF page images when `@napi-rs/canvas` cannot be imported, and open PR #75370 already uses closing syntax for this issue, so the issue should wait for that PR or a maintainer-approved alternative.

## What This Changes

The issue requests a Poppler/`pdftoppm` image-rendering fallback when the PDF extractor cannot load `@napi-rs/canvas`, instead of returning only text.

## Best Possible Solution

Land a bounded `document-extract` plugin fallback, or explicitly surface and document canvas-missing image degradation if maintainers reject Poppler.

## Reproduction Assessment

Yes. Source inspection on current main shows low-text PDFs reach `loadCanvasModule()`, catch a missing `@napi-rs/canvas` import, call only the optional callback, and return `{ text, images: [] }`; the agent PDF tool route does not pass that callback.

## Solution Assessment

Unclear. `pdftoppm` is a plausible owner-module fallback, but it should be reviewed for secure external process execution and resource bounds before it is accepted as the best solution.
