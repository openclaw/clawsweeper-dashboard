## Summary

Keep open. Current main still has the reported text-only degradation when canvas is unavailable, and open PR #75370 already uses closing syntax for this issue, so the issue should wait for that PR to be reviewed, merged, or rejected.

## What This Changes

Requests that the bundled document-extract plugin fall back to Poppler `pdftoppm` for PDF page images when `@napi-rs/canvas` is unavailable instead of returning text-only output.

## Best Possible Solution

Review and either land a bounded `document-extract` plugin fallback in PR #75370 or explicitly surface and document canvas-missing image degradation if Poppler is rejected.

## Reproduction Assessment

Yes. Source inspection on current main shows a low-text PDF path reaches `loadCanvasModule()`, catches a missing `@napi-rs/canvas` import, and returns `{ text, images: [] }`; the agent PDF tool route does not pass the warning callback.

## Solution Assessment

Unclear. A `pdftoppm` fallback is a plausible owner-module fix, but external PDF rendering of untrusted input needs bounded execution and maintainer/security review before it is the accepted best solution.
