## Summary

Keep open. Current main still has the reported text-only degradation when `@napi-rs/canvas` is unavailable, and https://github.com/openclaw/openclaw/pull/80837 is an open closing PR that should be fixed, proven, and landed or rejected before this issue closes.

## What This Changes

The issue requests a plugin-local Poppler `pdftoppm` fallback for low/no-text PDF page image extraction when `@napi-rs/canvas` cannot load.

## Best Possible Solution

Review and land a bounded plugin-local Poppler fallback in https://github.com/openclaw/openclaw/pull/80837, or explicitly document and surface canvas-missing image degradation if that fallback is rejected.

## Reproduction Assessment

Yes. Source inspection on current main shows low-text PDFs reach `loadCanvasModule()`, catch a missing `@napi-rs/canvas` import, and return `{ text, images: [] }`; the issue body gives matching Ubuntu/Poppler evidence.

## Solution Assessment

Unclear as currently proposed. A plugin-local `pdftoppm` fallback is the right owner boundary, but the open PR still needs resource-bound correction and real headless-Linux proof before it is the best accepted fix.
