## Summary

Keep open. Current main still degrades low-text PDF fallback to `{ text, images: [] }` when `@napi-rs/canvas` cannot load, there is no Poppler/`pdftoppm` fallback, and open PR #75370 already uses closing syntax for this issue.

## What This Changes

The issue requests a `pdftoppm`/Poppler fallback for PDF page image rendering when optional `@napi-rs/canvas` is missing.

## Best Possible Solution

Review and either land a bounded `document-extract` plugin fallback in PR #75370 or deliberately surface/document canvas-missing image degradation if Poppler is rejected.

## Reproduction Assessment

Yes. Source inspection on current main shows a low-text PDF path reaches `loadCanvasModule()`, catches a missing `@napi-rs/canvas` import, returns `{ text, images: [] }`, and the agent PDF tool route does not pass the warning callback.

## Solution Assessment

Unclear. A `pdftoppm` fallback in the owner plugin is plausible, but the best solution needs maintainer/security review around bounded external process execution before it is accepted.
