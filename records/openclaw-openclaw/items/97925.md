## What This Changes

Reports that stale xAI/Grok reasoning replay state is not stripped and retried, letting the provider error trip the per-model breaker for grok-4.3 traffic.

## Best Possible Solution

Review and land the narrow matcher/test fix in https://github.com/openclaw/openclaw/pull/97926 if it proves the xAI recovery path without broadening unrelated decrypt-error retries.

Yes at source level: construct an OpenAI.BadRequestError with status 400, no code, and the reported encrypted_content decrypt message, then pass it through the current retry helper with encrypted_content in the request. Current main's predicate returns false, so the retry path throws instead of stripping and retrying.

## Solution Assessment

Yes, the narrowest maintainable solution is to extend isInvalidEncryptedContentError and add focused positive and negative tests. Changing the breaker itself would be a broader workaround for a recoverable stale-replay predicate miss.
