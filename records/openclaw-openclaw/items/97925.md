## What This Changes

The issue reports that xAI/Grok stale reasoning replay returns a code-less encrypted_content decrypt 400 that is not recognized by the existing strip-and-retry recovery path, allowing the model breaker to block later Grok traffic.

## Best Possible Solution

Review and land the narrow matcher/test fix in https://github.com/openclaw/openclaw/pull/97926 if it proves the live xAI recovery path; keep this issue open until that PR merges or maintainers choose a different fix.

## Reproduction Assessment

Yes at source level: pass a 400 BadRequestError with the reported encrypted_content decrypt message and no code through the current retry helper with encrypted_content in the request. Current main's predicate returns false, so the helper rethrows instead of stripping and retrying.

## Solution Assessment

Yes, the best narrow fix is to extend isInvalidEncryptedContentError with a constrained xAI decrypt-message match and add positive and negative regression tests. The broader circuit-breaker classification idea in the discussion may be useful later, but it is larger than needed for this recoverable stale-replay miss.
