The PR moves channel delivery from parser-style target handling to target resolver/session-route APIs, deprecates parser-backed SDK surfaces, and updates bundled channel callers, docs, and tests.

## Best Possible Solution

Keep the session-route refactor direction, but preserve explicit topic-thread extraction across remaining sync/direct paths, make the SDK deprecation path explicit, and require a green current-head gate plus redacted real Telegram topic proof before merge.

## Solution Assessment

No as submitted. The resolver/session-route direction is maintainable, but the branch must keep topic identity intact for remaining sync/direct paths and document the plugin SDK compatibility decision before it is safe to merge.

- **[P1] Preserve topic thread IDs after parser removal:** `src/infra/outbound/targets-session.ts:152-155`

Keep a compat extraction path or move every remaining sync/direct caller onto `resolveOutboundSessionRoute` before dropping the parser.
