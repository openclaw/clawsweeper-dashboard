## Summary

Keep open: current main has the broad boolean `messages.suppressToolErrors` off-style suppression, but it still intentionally shows mutating-tool warnings after success-looking final replies and does not expose the requested `all | final | off` delivery policy. Recent operator comments confirm the broader Telegram/group-chat policy need remains after the narrow boolean fix.

## What This Changes

Request a normal-chat configuration option that controls user-visible delivery of transient tool-error warnings, such as all warnings, final-failure-only warnings, or no automatic warning bubbles.

## Best Possible Solution

Define one documented normal-chat tool-error delivery contract that preserves visible warnings by default, keeps explicit off/internal-only suppression, and only hides recovered transient failures when recovery can be identified reliably.

## Reproduction Assessment

Yes, at source level. Current tests still expect a success-looking assistant reply plus a failed mutating tool to emit a visible warning by default, and the requested delivery-mode config is absent; I did not run tests because this review was read-only.

## Solution Assessment

Unclear. The existing boolean suppression fix is a useful off/internal-only repair, but the requested `final` policy still needs maintainer agreement on safe recovery signals and user-visible failure semantics.
