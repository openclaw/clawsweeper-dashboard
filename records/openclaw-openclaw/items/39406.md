## Summary

Keep open: current main and v2026.5.18 still lack the requested normal-chat delivery policy, and current tests intentionally preserve warning bubbles after success-looking assistant text. The open PR at https://github.com/openclaw/openclaw/pull/81561 is useful for the narrower `messages.suppressToolErrors` gap, but it does not settle this issue's broader `all` / `final` / `off` contract.

## What This Changes

The issue requests a normal-chat config option for suppressing transient tool error warning bubbles after the agent recovers or succeeds.

## Best Possible Solution

Define one documented normal-chat tool-error delivery contract that can hide configured or proven-recovered transient failures while preserving explicit visibility for unresolved mutating side effects.

## Reproduction Assessment

Yes, at source level. Current source and regression tests show a success-looking assistant reply plus failed mutating tool still emits a warning, and the requested delivery-mode config is absent.

## Solution Assessment

Unclear. The requested direction is plausible for hosted and team-chat deployments, but the maintainable solution needs a maintainer product decision and coordination with the narrower open PR at https://github.com/openclaw/openclaw/pull/81561.
