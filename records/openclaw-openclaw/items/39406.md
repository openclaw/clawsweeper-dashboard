## Summary

Keep open. Current main still lacks the requested normal-chat delivery mode or global heartbeat-style suppression, and existing tests intentionally keep mutating-tool warnings visible when the assistant reply looks successful; the open overlapping PR only covers part of the existing boolean behavior.

## What This Changes

Request a normal-chat config option to suppress or defer transient tool-error warnings, especially when the final agent reply indicates success.

## Best Possible Solution

Define one documented normal-chat tool-error delivery contract that can hide configured or proof-recovered transient failures while preserving visible warnings for unresolved mutating side effects, then review or supersede https://github.com/openclaw/openclaw/pull/81561 within that scope.

## Reproduction Assessment

Yes, source-level. Current tests assert that success-looking assistant replies plus failed mutating tools still emit a warning payload, and schema/docs do not expose the requested delivery mode.

## Solution Assessment

Unclear. The requested suppression is plausible for hosted deployments, but the safest maintainable path needs a maintainer policy decision because current behavior intentionally protects against false-success mutating failures.
