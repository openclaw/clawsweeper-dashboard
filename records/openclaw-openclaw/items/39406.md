## Summary

Keep open: current main and the latest release have the global `messages.suppressToolErrors` off switch plus acknowledgement-only suppression, but the requested `all | final | off` normal-chat delivery policy is still missing and current tests intentionally keep warnings after success-looking replies.

## What This Changes

The issue requests a configurable normal-chat policy to suppress transient tool error warning bubbles when the agent recovers or finishes successfully.

## Best Possible Solution

Keep the issue open for a maintainer decision on whether to keep the existing boolean off switch or add a documented `messages` delivery-policy enum with safe defaults and focused payload/channel tests.

## Reproduction Assessment

Yes, from source: current payload tests build a success-looking assistant reply plus a tool failure and assert a visible warning is still emitted. I did not run tests because this review is read-only.

## Solution Assessment

Unclear: the existing `messages.suppressToolErrors` boolean solves the explicit off/internal-only case, but the requested `final` mode changes safety policy for mutating failures and needs a maintainer product decision before implementation.
