## Summary

Keep open: current main and v2026.6.1 have the global `messages.suppressToolErrors` off switch and acknowledgement-only suppression, but the requested `all | final | off` normal-chat delivery policy remains missing and current tests still preserve warnings after success-looking final replies.

## What This Changes

The issue requests a configurable normal-chat policy for transient tool error warnings, especially when an agent recovers and returns a useful final reply.

## Best Possible Solution

Keep the current safety default, and only add a documented `all | final | off` or equivalent policy after maintainers decide the exact safety contract for success-looking final replies.

## Reproduction Assessment

Yes, source-reproducible: current tests intentionally show a separate mutating-tool warning after success-looking assistant text, which is the remaining behavior this issue asks to make configurable.

## Solution Assessment

Unclear: the implemented boolean off switch is the right current workaround, but the requested final-success-aware mode needs a maintainer policy decision before it can be called the best fix.
