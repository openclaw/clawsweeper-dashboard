## Summary

Keep open: current main and v2026.6.6 only provide a coarse boolean suppression switch plus acknowledgement safeguards; the requested final-only or tri-state normal-chat warning policy remains missing and needs maintainer product/safety agreement.

## What This Changes

The issue requests a configurable normal-chat delivery policy for transient tool error warnings after recovered tool failures.

## Best Possible Solution

Define a maintainer-approved `messages` tool-warning delivery policy with safe defaults, documented semantics, and regression coverage across payload assembly plus Telegram/Discord delivery where applicable.

## Reproduction Assessment

Yes. Source and tests show current main still sends a warning after success-looking assistant output unless the coarse suppression switch is enabled; I did not run tests because this was a read-only review.

## Solution Assessment

Unclear as an implementation shape. The requested `all | final | off` direction is plausible, but maintainers need to choose the safety semantics before a code change should be queued.
