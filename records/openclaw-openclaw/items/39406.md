## Summary

Keep open. Current main has a useful partial fix for recovered same-target file mutations, but it still does not provide the requested normal-chat delivery policy or config knob, and existing tests intentionally keep success-looking replies plus unresolved mutating failures visible.

## What This Changes

The issue requests a config policy to suppress, defer, or disable transient tool-error warnings in normal chat when the agent later succeeds.

## Best Possible Solution

Define one documented normal-chat tool-error delivery policy that keeps proven recovered transient failures quiet while preserving a visible safety fallback for unresolved mutating failures.

## Reproduction Assessment

Yes, source-level. Current source and tests show the remaining policy split: same-target edit/write recovery can clear a prior error, but `messages.suppressToolErrors` still does not suppress unresolved mutating warnings and no all/final/off config exists.

## Solution Assessment

Unclear. The requested all/final/off direction is plausible for managed deployments, but maintainers need to choose the safety boundary before implementing a config that could hide failed writes, edits, execs, or sends.
