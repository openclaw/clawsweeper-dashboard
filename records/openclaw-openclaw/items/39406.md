## Summary

Keep open. Current main still lacks the requested normal-chat delivery policy for transient tool-error warnings: suppression is partial, heartbeat-scoped for the broad flag, and focused tests still assert that mutating failures can be delivered alongside success-looking replies.

## What This Changes

Request a configurable normal-chat policy to suppress transient tool error warnings when the agent recovers or succeeds, such as `all` / `final` / `off` delivery modes or a global extension of `suppressToolErrorWarnings`.

## Best Possible Solution

Define one documented normal-chat tool-error delivery policy that covers non-mutating failures, mutating failure safety, heartbeat-only suppression, and any `all` / `final` / `off` style option; then align config types, schema/help, docs, and focused payload/tool-handler tests around that contract.
