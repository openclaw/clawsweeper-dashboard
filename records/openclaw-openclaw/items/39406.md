## Summary

Keep open: current main and v2026.6.6 provide the coarse `off` path via `messages.suppressToolErrors`, plus acknowledged-failure suppression, but the requested `all | final | off` normal-chat delivery policy is still missing and needs maintainer product/safety direction.

## What This Changes

Requests a configurable normal-chat policy to suppress or defer transient tool-error warning bubbles after recovered agent turns.

## Best Possible Solution

Define one canonical `messages`-level tool-error delivery policy with conservative defaults, explicit `all | final | off` semantics, docs, and regression tests for recovered-success, ambiguous-success, acknowledged-failure, and unresolved-failure turns.

## Reproduction Assessment

Yes, source-reproducible. Current tests construct success-looking assistant output plus a failed mutating tool and assert that a separate warning remains visible; this review did not run tests because it is read-only.

## Solution Assessment

Unclear as proposed. The full `off` outcome is already available through `messages.suppressToolErrors`, so the remaining `final` mode likely belongs as an additive `messages` policy after maintainers define safety semantics.
