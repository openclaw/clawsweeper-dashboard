## Summary

Keep open: current main and v2026.6.6 provide the full opt-out path through `messages.suppressToolErrors`, but the requested conditional `all | final | off` normal-chat delivery policy remains missing and needs maintainer product/safety judgment.

## What This Changes

Request a normal-chat configuration policy that can suppress transient tool-error warning bubbles, especially after recovered or successful agent turns.

## Best Possible Solution

Define one canonical `messages`-level delivery policy with conservative defaults, explicit `all | final | off` semantics, docs, and regression tests for recovered-success, ambiguous-success, acknowledged-failure, and unresolved-failure turns.

## Reproduction Assessment

Yes. Source-level tests on current main construct success-looking assistant output plus a failed mutating tool and assert that a separate warning remains visible; I did not run tests because this review is read-only.

## Solution Assessment

Unclear as proposed. The full `off` outcome is already available through `messages.suppressToolErrors`, but the `final` mode needs maintainer-defined safety semantics and likely belongs at the existing `messages` config surface rather than a new agent-default enum.
