## Summary

Keep open: current main and v2026.6.10 include useful partial fixes, including `messages.suppressToolErrors`, mutating-tool suppression when explicitly enabled, and some channel-side post-final warning filtering, but they still do not provide the requested shared `all | final | off` normal-chat delivery policy. The remaining work is a product/config decision because the default path still intentionally preserves success-looking mutating and exec failure warnings unless an existing suppression path applies.

## What This Changes

Feature request for a normal-chat configuration policy that can suppress transient tool-error warning bubbles after a successful or useful final reply.

## Best Possible Solution

Maintainers should choose one shared user-facing contract, either documenting `messages.suppressToolErrors` as the off/internal-only mode or adding a default-compatible `all | final | off` delivery mode with runner and channel tests.

## Reproduction Assessment

Yes. Current-main source tests show success-looking assistant replies followed by visible mutating and exec warning payloads when suppression is not explicitly enabled.

## Solution Assessment

Unclear. A shared `all | final | off` policy may be cleaner than more channel-specific filters, but the default safety behavior and config shape need maintainer approval before implementation.
