## Summary

Keep open: current main and the latest release only cover the broad `messages.suppressToolErrors` opt-out, while the requested final-only normal-chat policy remains deliberately unresolved for mutating tool failures after successful-looking replies.

## What This Changes

The issue requests a normal-chat configuration policy for suppressing transient tool-error warning bubbles after successful or useful agent replies.

## Best Possible Solution

Keep this as the canonical maintainer decision point for a normal-chat tool-warning delivery policy, then implement any chosen policy with config/schema/docs alignment and payload plus channel regression coverage.

## Reproduction Assessment

Yes. Current source and tests reproduce the remaining gap: mutating tool warnings still appear after successful-looking replies unless the broad suppression flag is enabled or the assistant explicitly acknowledges the failed action.

## Solution Assessment

Unclear until maintainers choose the product policy. The existing broad `messages.suppressToolErrors` flag is a useful workaround, but the requested `final` mode changes the safety boundary for failed mutating actions after optimistic assistant text.
