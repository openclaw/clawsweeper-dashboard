## Summary

Keep open: current main still deliberately surfaces unresolved mutating tool warnings after success-looking replies, lacks the requested `all`/`final`/`off` delivery policy, and the active related PR is narrower than this normal-chat config request.

## What This Changes

Request a normal-chat config option to suppress transient user-visible tool error warnings when the agent recovers or otherwise succeeds.

## Best Possible Solution

Define one documented normal-chat tool-error delivery contract that can hide configured or proven-recovered transient failures while preserving explicit visibility for unresolved mutating side effects.

## Reproduction Assessment

Yes, source-level. Current main has tests asserting a success-looking assistant reply plus failed `write` still emits an error payload, and the schema/docs do not contain the requested delivery mode.

## Solution Assessment

Unclear. The requested success-based suppression is plausible for hosted deployments, but the safest maintainable path needs a product decision and coordination with https://github.com/openclaw/openclaw/pull/81561.
