## Summary

Keep open: current main now honors `messages.suppressToolErrors` as an off-style knob, but source, tests, and docs still preserve visible warnings for success-looking final replies and no `all | final | off` delivery policy exists. Recent operator comments confirm the broader Telegram/group-chat policy need remains after the narrow boolean fix.

## What This Changes

The issue requests a configurable normal-chat policy for transient tool-error warning delivery, such as all, final-success-only, or off/internal-only.

## Best Possible Solution

Define one documented normal-chat tool-error delivery contract that preserves visible warnings by default, keeps explicit off/internal-only suppression, and only hides recovered transient failures when recovery can be identified reliably.

## Reproduction Assessment

Yes, at source level. Current tests still expect a success-looking assistant reply plus a failed mutating tool to emit a visible warning by default, and the requested delivery-mode config is absent; I did not run tests because this review was read-only.

## Solution Assessment

Unclear. The merged boolean suppression fix is a good narrow repair for `messages.suppressToolErrors`, but the requested `final` policy still needs maintainer agreement on safe recovery signals and user-visible failure semantics.
