## Summary

Keep open: current main and the latest release now provide a broad `messages.suppressToolErrors` off switch, but they still deliberately show mutating tool-error warnings after success-looking assistant replies unless a broader suppression flag is enabled. The remaining `all | final | off` normal-chat policy is a real product/config decision, not a completed bug fix.

## What This Changes

Request a normal-chat configuration policy such as `all | final | off` to suppress or defer transient tool-error warning bubbles after recovered or successful agent turns.

## Best Possible Solution

Maintainers should choose one canonical warning-delivery policy that preserves safe visibility for real failed mutations, honors the existing broad suppression switch, and documents/tests any new final-only mode.

## Reproduction Assessment

Yes, at source level. Current policy and tests still show a mutating tool failure warning after a success-looking final reply unless broad suppression is enabled; I did not run a live Telegram/Discord replay in this read-only review.

## Solution Assessment

Unclear until maintainers choose the product policy. The existing broad `messages.suppressToolErrors` switch is a useful partial solution, but the proposed final-only behavior needs explicit safety semantics before implementation.
