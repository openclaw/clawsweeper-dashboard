## Summary

Keep open: current main now honors the existing boolean `messages.suppressToolErrors` knob for mutating-tool warnings, but it still lacks the requested normal-chat delivery policy that can distinguish `all`, final-success-only, and off/internal-only behavior.

## What This Changes

The issue requests a normal-chat configuration policy to suppress transient tool error warning bubbles after successful recovery, with possible `all`, `final`, and `off` modes.

## Best Possible Solution

Define one documented normal-chat tool-error delivery contract that preserves the current visible default, supports an explicit internal-only/off mode for operators, and only hides recovered transient failures when recovery can be identified reliably.

## Reproduction Assessment

Yes, at source level. Current tests still expect a success-looking assistant reply plus a failed mutating tool to emit a visible warning by default, and the requested delivery-mode config is absent.

## Solution Assessment

Unclear. The merged boolean suppression fix is the right narrow repair for `messages.suppressToolErrors`, but the requested final-success policy still needs a maintainer decision about safe recovery signals and user-visible failure semantics.
