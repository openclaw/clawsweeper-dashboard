## Summary

Keep open: current main and v2026.6.8 include useful partial fixes, including `messages.suppressToolErrors` and post-final/non-terminal warning suppression, but they do not implement the requested shared `all | final | off` normal-chat delivery policy. The remaining work is a maintainer product/config decision because success-looking mutating and exec failures are still intentionally visible unless explicitly suppressed or acknowledged.

## What This Changes

The issue requests an operator-facing config policy for suppressing transient tool-error warning bubbles in chat, especially after recovered or success-looking final replies.

## Best Possible Solution

Maintainers should decide whether the existing boolean should be the documented off/internal-only operator contract, or add one shared delivery mode with default-compatible behavior and focused runner plus channel tests.

## Reproduction Assessment

Yes. Current source tests show success-looking assistant replies followed by visible mutating/exec tool-error warnings, and Telegram tests preserve ordinary final error warnings after a final reply.

## Solution Assessment

Unclear. A shared `all | final | off` policy may be cleaner than more channel-specific filters, but the default safety behavior and config shape need maintainer approval before implementation.
