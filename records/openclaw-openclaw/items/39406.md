## Summary

Keep open. Current main still lacks the requested normal-chat configurable delivery policy: mutating tool warnings bypass `messages.suppressToolErrors`, heartbeat suppression remains a separate run-scoped option, and the changelog explicitly leaves this issue for a configurable delivery policy.

## What This Changes

Request a config option to control whether transient tool error warnings are delivered to normal chat after the agent recovers or reports success.

## Best Possible Solution

Define one documented normal-chat tool-error delivery policy, then align config types, schema/help, docs, payload handling, changelog text, and focused regression tests around that policy.

## Reproduction Assessment

Yes. Current main source and focused payload tests show the gap: `messages.suppressToolErrors` suppresses non-mutating warnings, but a failed mutating tool can still emit a warning after success-looking assistant text.

## Solution Assessment

Unclear. The requested `all`/`final`/`off` policy is plausible, but simply moving `suppressToolErrors` ahead of the mutating guard would change the existing safety contract; maintainers should choose the intended normal-chat policy first.
