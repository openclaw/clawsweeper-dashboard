## Summary

Keep open. Current main still lacks the requested normal-chat delivery policy/config, and the checked-in warning policy intentionally keeps unresolved mutating tool failures visible even when `messages.suppressToolErrors` is enabled.

## What This Changes

Add or broaden configuration so normal chat can suppress transient tool-error warnings after agent recovery.

## Best Possible Solution

Define one documented normal-chat tool-error delivery policy that suppresses proven recovered transient failures or explicit opt-in suppression while preserving visible warnings for unresolved mutating side effects.

## Reproduction Assessment

Yes, source-level. Current source and tests show `messages.suppressToolErrors` does not suppress unresolved mutating warnings and no requested all/final/off policy exists on current main.

## Solution Assessment

Unclear. The requested direction is plausible for managed deployments, but maintainers need to choose the safety boundary before adding a policy that can hide failed writes, edits, execs, or sends.
