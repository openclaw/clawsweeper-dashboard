## Summary

Keep open: current main and the latest release cover the broad `messages.suppressToolErrors` off switch and acknowledgement-only suppression, but they still intentionally show mutating-tool warnings after success-looking final replies and do not expose the requested `all | final | off` delivery policy.

## What This Changes

The issue requests a normal-chat configuration policy for suppressing transient tool-error warning messages after recovered or successful agent turns.

## Best Possible Solution

Define one documented normal-chat tool-error delivery contract that keeps visible warnings as the safe default, preserves explicit off/internal-only suppression, and only hides recovered transient failures when recovery can be identified reliably.

## Reproduction Assessment

Yes, at source level. Current main still has a test where `Done.` plus a failed mutating `write` tool emits a visible `Write` warning, and the requested delivery-mode config is absent.

## Solution Assessment

Unclear. The existing boolean suppression is a useful off/internal-only workaround, but the requested `final` policy still needs maintainer agreement on safe recovery signals and user-visible failure semantics.
