## Summary

Keep open: current main now honors the existing boolean `messages.suppressToolErrors` knob for mutating-tool warnings, but it still does not provide the requested normal-chat delivery policy for `all`, final-success-only, and off/internal-only behavior.

## What This Changes

The issue requests a configurable normal-chat policy to hide or defer transient tool-error warning bubbles after an agent recovers and returns a successful final reply.

## Best Possible Solution

Define one documented normal-chat tool-error delivery contract that preserves visible warnings by default, supports an explicit internal-only/off operator mode, and only hides recovered transient failures when recovery can be identified reliably.

## Reproduction Assessment

Yes, at source level. Current tests still expect a success-looking assistant reply plus a failed mutating tool to emit a visible warning by default, and the requested delivery-mode config is absent.

## Solution Assessment

Unclear. The merged boolean suppression fix is a good narrow repair for `messages.suppressToolErrors`, but the requested `final` policy still needs a maintainer decision about safe recovery signals and user-visible failure semantics.
