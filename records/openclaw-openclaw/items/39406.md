## Summary

Keep open: current main and v2026.6.10 have useful partial suppression paths, but they still do not implement the requested shared normal-chat `all | final | off` delivery policy. The remaining decision is a product/config safety contract because default-visible failed mutations are intentional today.

## What This Changes

The issue requests a normal-chat configuration policy for suppressing transient tool-error warning bubbles, especially after a recovered or successful agent turn.

## Best Possible Solution

Maintainers should choose one shared, default-compatible warning-delivery contract and align schema, docs, runner payloads, progress delivery, and channel tests around it.

## Reproduction Assessment

Yes, at source level: current main still has tests where `Done.` is followed by a visible failed-write warning, and no current config enum implements final-only delivery.

## Solution Assessment

Unclear; an `all | final | off` policy is plausible, but the best fix depends on a maintainer safety decision because failed mutations currently remain visible to avoid silent false success.
