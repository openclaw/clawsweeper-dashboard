## Summary

Keep open: current main and v2026.5.18 still lack the requested normal-chat delivery policy, and current tests intentionally preserve mutating-tool warning bubbles after success-looking assistant text. The open related PR at https://github.com/openclaw/openclaw/pull/81561 is a useful narrow fix for honoring `messages.suppressToolErrors`, but it does not settle the broader `all` / `final` / `off` product contract.

## What This Changes

Feature request to add a normal-chat configuration policy for suppressing transient tool error warnings after the agent recovers or succeeds.

## Best Possible Solution

Define one documented normal-chat tool-error delivery contract that can hide configured or proven-recovered transient failures while preserving explicit visibility for unresolved mutating side effects.

## Reproduction Assessment

Yes, at source level. Current source and regression tests show a success-looking assistant reply plus failed mutating tool still emits a warning, and the requested delivery-mode config is absent.

## Solution Assessment

Unclear. The requested direction is plausible for hosted/team-chat deployments, but maintainers need to choose the warning-visibility contract and coordinate it with the narrower open PR at https://github.com/openclaw/openclaw/pull/81561.
