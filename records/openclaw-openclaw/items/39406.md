## Summary

Keep open: current main and v2026.5.18 still lack the requested normal-chat delivery policy, and the checked-in tests intentionally preserve mutating-tool warning bubbles even after success-looking assistant text. The open related PR at https://github.com/openclaw/openclaw/pull/81561 is a useful narrow fix for `messages.suppressToolErrors`, but it does not settle the broader `all` / `final` / `off` product contract.

## What This Changes

The issue requests a configurable policy for suppressing transient user-visible tool error warnings in normal chat surfaces after the agent recovers.

## Best Possible Solution

Define one documented normal-chat tool-error delivery contract that can hide configured or proven-recovered transient failures while preserving explicit visibility for unresolved mutating side effects.

## Reproduction Assessment

Yes, at source level. Current source and regression tests show a success-looking assistant reply plus failed mutating tool still emits a warning, and the requested delivery-mode config is absent.

## Solution Assessment

Unclear. The requested direction is plausible for hosted/team-chat deployments, but the maintainable solution needs a maintainer product decision and coordination with the narrower open PR at https://github.com/openclaw/openclaw/pull/81561.
