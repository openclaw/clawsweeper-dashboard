## Summary

Keep open: current main and v2026.5.12 still lack the requested normal-chat `all`/`final`/`off` delivery policy, and current tests intentionally preserve visible mutating-tool warnings after success-looking assistant text. The open narrower PR at https://github.com/openclaw/openclaw/pull/81561 may make `messages.suppressToolErrors` stronger, but it does not settle the broader product/config request.

## What This Changes

Request a normal-chat configuration policy to hide transient tool-error warnings after successful recovery, with modes such as `all`/`final`/`off` or global `suppressToolErrorWarnings` behavior.

## Best Possible Solution

Define one documented normal-chat tool-error delivery contract that can hide configured or proven-recovered transient failures while preserving explicit visibility for unresolved mutating side effects.

## Reproduction Assessment

Yes. Source inspection and current regression tests show a success-looking assistant reply plus a failed mutating `write` still emits a user-visible warning, and the requested delivery-mode config is absent.

## Solution Assessment

Unclear. The requested direction is plausible for managed deployments, but the safest solution needs a maintainer decision about warning visibility and coordination with the narrower open PR at https://github.com/openclaw/openclaw/pull/81561.
