## Summary

Keep open: current main and v2026.6.6 cover the coarse `off` path with `messages.suppressToolErrors` and some post-final/acknowledged-warning suppression, but the requested normal-chat `all | final | off` delivery policy is still missing and needs maintainer product/safety direction.

## What This Changes

The issue requests a normal-chat configuration policy for suppressing transient tool-error warning bubbles after a successful agent turn.

## Best Possible Solution

Define one canonical `messages`-level tool-error delivery policy with conservative defaults, explicit `all | final | off` semantics, docs, schema/help, and regression coverage for recovered-success, ambiguous-success, acknowledged-failure, and unresolved-failure turns.

## Reproduction Assessment

Yes, from source inspection. Current tests construct success-looking assistant output plus a failed mutating tool and assert that a separate warning remains visible; this review did not run tests because it is read-only.

## Solution Assessment

Unclear as proposed. The full `off` outcome is already available through `messages.suppressToolErrors`, so the remaining `final` mode likely belongs as an additive `messages` policy after maintainers define safety semantics.
