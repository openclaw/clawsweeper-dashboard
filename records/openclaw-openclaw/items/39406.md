## Summary

Keep open: current main and the latest release have the broad `messages.suppressToolErrors` off-style suppression, but they still do not expose the requested `all | final | off` normal-chat delivery policy, and tests intentionally keep warnings visible after success-looking mutating-tool replies. Recent operator comments add current Telegram/group-chat cases after the narrow boolean fix, so this remains a product/config decision rather than a cleanup close.

## What This Changes

Request a configurable normal-chat policy for transient tool-error warning delivery, including hiding recovered warnings after successful final replies.

## Best Possible Solution

Define one documented normal-chat tool-error delivery contract that keeps warnings visible by default, supports explicit internal-only/off suppression, and hides recovered transient failures only from reliable recovery signals.

## Reproduction Assessment

Yes, at source level. Current tests still expect a success-looking assistant reply plus a failed mutating tool to emit a visible warning, and the requested delivery-mode config is absent; I did not run tests because this review is read-only.

## Solution Assessment

Unclear. The existing boolean suppression is a useful off/internal-only repair, but a `final` policy needs maintainer agreement on recovery signals and user-visible failure semantics.
