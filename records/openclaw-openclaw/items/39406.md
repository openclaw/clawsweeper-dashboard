## Summary

Keep open: current main now supports the boolean off/internal-only workaround via `messages.suppressToolErrors`, but it still does not provide the requested normal-chat delivery policy for `all`, final-success-only, and off/internal-only behavior.

## What This Changes

The issue requests a normal-chat configuration option to suppress, defer, or disable transient tool-error warning bubbles after an agent recovers and produces a useful final reply.

## Best Possible Solution

Define one documented normal-chat tool-error delivery contract that preserves visible warnings by default, treats `messages.suppressToolErrors` as the explicit off/internal-only mode or deliberately replaces it, and only adds a final-success mode if recovery can be detected without hiding real unresolved failures.

## Reproduction Assessment

Yes, at source level. Current tests still show a success-looking assistant reply plus a failed mutating tool emits a visible warning by default, and current schema/docs do not expose the requested `all | final | off` delivery policy.

## Solution Assessment

Unclear. The merged boolean suppression reorder is the right narrow fix for `messages.suppressToolErrors`, but the remaining final-success-aware policy requires maintainer agreement on recovery signals, default visibility, and whether a new enum should coexist with the boolean knob.
