## Summary

Keep open. Current main still lacks the requested normal-chat configurable delivery policy: mutating tool warnings bypass `messages.suppressToolErrors`, heartbeat suppression remains run-scoped, and the changelog explicitly leaves this issue for a configurable delivery policy.

## What This Changes

Request a config option or policy to hide transient tool failure warnings from user chat when an agent ultimately recovers, or to disable them globally.

## Best Possible Solution

Define and document one normal-chat tool-error delivery policy that preserves safety for unresolved mutating failures, then align config schema, docs, payload handling, changelog text, and focused regression tests.

## Reproduction Assessment

Yes. Source and focused tests show the requested behavior is still absent: `messages.suppressToolErrors` suppresses non-mutating warnings, but mutating `write`/`edit` warnings can still appear after success-looking assistant text.

## Solution Assessment

Unclear. The requested `all`/`final`/`off` policy is plausible, but the current mutating-warning behavior is an intentional safety contract, so maintainers should choose the policy before a code change.
