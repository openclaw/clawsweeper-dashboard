## Summary

Keep open: current main and v2026.6.6 implement the full opt-out path through `messages.suppressToolErrors`, but they still intentionally show mutating/exec warnings after success-looking final replies; the remaining `final`/normal-chat delivery policy is a product and safety decision, not an implemented feature.

## What This Changes

The issue requests a configurable normal-chat policy for suppressing transient tool-error warnings after the agent has recovered or produced a useful final response.

## Best Possible Solution

Define one canonical messages-level tool-error delivery policy that preserves the conservative default, keeps or migrates the existing full-suppression switch, and tests any final-only mode across core payload assembly plus affected channel paths.

## Reproduction Assessment

Yes, at source level. Current payload tests construct success-looking final replies plus failed mutating or exec tools and assert that a separate warning remains visible; I did not run tests because this review is read-only.

## Solution Assessment

No, not exactly as proposed. The `off`/internal-only outcome already exists through `messages.suppressToolErrors`; the remaining best fix is a maintainer-approved canonical messages-level policy with explicit safety semantics for final-only suppression.
