## Summary

Keep open: current main and the latest release still lack the requested normal-chat `all`/`final`/`off` delivery policy, and the existing tests still intentionally show mutating-tool warnings after success-looking assistant text. The narrower open PR at https://github.com/openclaw/openclaw/pull/81561 may improve `messages.suppressToolErrors`, but it does not settle the broader product/config request.

## What This Changes

Requested a normal-chat configuration policy for suppressing transient tool-error warnings after successful agent recovery.

## Best Possible Solution

Define one documented normal-chat tool-error delivery contract that can hide configured or proven-recovered transient failures while preserving explicit visibility for unresolved mutating side effects.

## Reproduction Assessment

Yes. Source inspection and existing regression tests show a success-looking assistant reply plus failed `write` still emits a warning, and the requested delivery-mode config is absent.

## Solution Assessment

Unclear. The requested direction is plausible for hosted deployments, but the safest maintainable solution needs a maintainer product decision and coordination with the narrower open PR at https://github.com/openclaw/openclaw/pull/81561.
