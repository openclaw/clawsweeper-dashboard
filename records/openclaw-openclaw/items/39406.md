## Summary

Keep open: current main and v2026.5.20 include the broad boolean off switch, but the requested `all | final | off` delivery policy is not implemented and tests intentionally preserve visible mutating-tool warnings after success-looking replies.

## What This Changes

Request a configurable normal-chat tool-error delivery policy that can suppress transient or recovered tool-warning bubbles on user channel surfaces.

## Best Possible Solution

Define one documented normal-chat tool-warning delivery contract that keeps visible warnings by default, preserves explicit off/internal-only suppression, and only hides recovered transient failures when recovery can be identified reliably.

## Reproduction Assessment

Yes, at source level. Current tests still expect a success-looking assistant reply plus a failed mutating tool to emit a visible warning by default, and the requested delivery-mode config is absent.

## Solution Assessment

Unclear. The existing boolean suppression path is a useful off/internal-only workaround, but the requested `final` mode needs maintainer agreement on safe recovery signals and user-visible failure semantics.
