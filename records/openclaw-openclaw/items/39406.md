## Summary

Keep open: current main still lacks the requested normal-chat `all`/`final`/`off` delivery policy or equivalent global config, and tests still assert that a success-looking assistant reply plus a failed mutating tool emits a visible warning. The open implementation PR at https://github.com/openclaw/openclaw/pull/81561 is narrower than this product/config request, so this still needs maintainer safety review.

## What This Changes

Request a normal-chat configuration policy to suppress or delay transient tool-error warnings after agent recovery or final success.

## Best Possible Solution

Define one documented normal-chat tool-error delivery contract that can hide configured or proven-recovered transient failures while preserving explicit visibility for unresolved mutating side effects.
