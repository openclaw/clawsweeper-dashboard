## Summary

Keep open. Current main still lacks the requested normal-chat tool-error delivery policy, and the related open pull request only addresses the narrower `messages.suppressToolErrors` behavior for mutating tools rather than the proposed all/final/off or global policy.

## What This Changes

Request to add a normal-chat configuration option that can suppress transient recovered tool-error warnings shown to users.

## Best Possible Solution

Define one documented normal-chat tool-error delivery contract that can suppress explicit opt-in or proven recovered transient failures while preserving visible fallback for unresolved mutating side effects.

## Reproduction Assessment

Yes, source-level. Current source and tests still show the requested normal-chat all/final/off policy is absent and that mutating warnings can surface even with success-looking assistant text.

## Solution Assessment

Unclear. The narrower fix in https://github.com/openclaw/openclaw/pull/81561 is maintainable for the existing boolean, but the requested success-based or all/final/off policy needs maintainer product and safety judgment.
