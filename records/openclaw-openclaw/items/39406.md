## Summary

Keep open: current main and `v2026.6.6` cover the explicit full opt-out with `messages.suppressToolErrors`, but the requested conditional `all | final | off` normal-chat policy is still missing and needs a maintainer product/safety decision.

## What This Changes

Request a normal-chat configuration policy to suppress or defer transient tool-error warning bubbles when an agent turn ultimately succeeds.

## Best Possible Solution

Define one canonical `messages`-level delivery policy with conservative default behavior, clear `all | final | off` semantics, docs, and tests for recovered-success, ambiguous-success, acknowledged-failure, and unresolved-failure turns.

## Reproduction Assessment

Yes, source-level: current payload tests construct a success-looking final reply plus a failed mutating tool and assert that a separate warning remains visible. I did not run tests because this was a read-only review.

## Solution Assessment

Unclear as proposed. The `off` outcome is already available through `messages.suppressToolErrors`, but the `final` mode needs maintainer-defined safety semantics before it can be the narrowest maintainable fix.
