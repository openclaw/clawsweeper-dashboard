## Summary

Keep open: current main provides a global full-suppression switch and acknowledgement-aware suppression, but intentionally retains mutating-tool warnings after success-looking replies; the remaining conditional-delivery behavior requires a maintainer product and safety decision.

## What This Changes

Requests a normal-chat tool-error delivery policy that can show all warnings, conditionally suppress recovered failures based on the final reply, or suppress warnings entirely while retaining internal error context.

## Best Possible Solution

Define one canonical messages-level warning policy that preserves the conservative default, reuses the existing full-suppression behavior, and conditionally hides failures only through explicit acknowledgement, verified recovery, or an intentional operator-selected policy.

## Reproduction Assessment

Yes. Current-main payload tests explicitly construct a successful-looking final reply plus a mutating tool failure and assert that the separate warning remains visible.

## Solution Assessment

No, not exactly as proposed. The full-off outcome already exists, while conditional suppression needs safer recovery semantics and a canonical messages-level contract rather than a parallel agent-default knob.
