## Summary

Keep open. Current main still lacks the requested normal-chat delivery policy/config, and source plus tests still show unresolved mutating tool warnings can bypass `messages.suppressToolErrors` and appear after success-looking assistant text; https://github.com/openclaw/openclaw/pull/81561 is an open narrower implementation for the existing boolean setting, not the broader all/final/off policy.

## What This Changes

The issue requests a normal-run config option or policy for suppressing transient tool-error warnings after the agent recovers and produces a successful final reply.

## Best Possible Solution

Define one documented normal-chat tool-error delivery contract that suppresses explicit opt-in or proven recovered transient failures while preserving visible fallback for unresolved mutating side effects.

## Reproduction Assessment

Yes, source-level. Current source and tests show the requested normal-chat all/final/off policy is absent and that mutating warnings still surface in cases covered by this issue.

## Solution Assessment

Unclear. The narrower `messages.suppressToolErrors` reorder in https://github.com/openclaw/openclaw/pull/81561 looks maintainable for the existing boolean, but the requested success-based or all/final/off policy needs product/safety judgment.
