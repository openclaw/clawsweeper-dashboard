## Summary

Keep open: current main and v2026.6.6 already provide the coarse `off` path through `messages.suppressToolErrors`, plus narrower Telegram/post-final fixes, but the requested conditional `all | final | off` normal-chat delivery policy is still missing and needs a maintainer product/safety decision.

## What This Changes

The issue requests a normal-chat configuration policy for suppressing transient tool error warnings after an agent recovers successfully.

## Best Possible Solution

Define one canonical `messages`-level delivery policy with conservative defaults, explicit `all | final | off` semantics, docs, and regression tests for recovered-success, ambiguous-success, acknowledged-failure, and unresolved-failure turns.

## Reproduction Assessment

Yes. Current source-level tests construct success-looking assistant output plus a failed mutating tool and assert that a separate warning remains visible; this review did not run tests because it is read-only.

## Solution Assessment

Unclear as proposed. The full `off` outcome is already available through `messages.suppressToolErrors`, but the `final` mode needs maintainer-defined safety semantics and likely belongs at the existing `messages` config surface rather than a new agent-default enum.
