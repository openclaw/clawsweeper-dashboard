## Summary

Keep open: current main and the latest release still lack a WhatsApp `notifyApproval` pairing hook, and the open implementation candidate should own the fix until it lands or is closed.

## What This Changes

Notify an approved WhatsApp sender through the channel pairing approval flow so they know access is ready.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/94386, or an equivalent focused WhatsApp plugin change, to implement the existing channel pairing `notifyApproval` hook without adding new config surface.

## Reproduction Assessment

Yes. Source inspection shows `--notify` dispatches through the generic pairing hook while the WhatsApp pairing adapter does not implement that hook.

## Solution Assessment

Yes for the narrow repair: add WhatsApp's `notifyApproval` implementation and focused tests for the existing `--notify` path. Making notification the default or replaying blocked messages would be broader product work.
