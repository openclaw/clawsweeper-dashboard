## Summary

Keep open. Current main still has a documented generic approval notification path, but WhatsApp does not implement the pairing `notifyApproval` adapter, so `pairing approve whatsapp <CODE> --notify` reaches a no-op instead of messaging the approved sender.

## What This Changes

Notify WhatsApp senders after DM pairing approval, using the existing approval-notification path where possible.

## Best Possible Solution

Implement WhatsApp support for the existing `pairing approve --notify` contract with account-aware routing, and leave default-on behavior and blocked-message replay as separate product decisions.

## Reproduction Assessment

Yes, at source level. WhatsApp creates pairing requests, `pairing approve --notify` reaches the generic notifier, and the current WhatsApp pairing adapter has no `notifyApproval`, so the notifier returns without sending.

## Solution Assessment

Yes for the narrow repair. Reusing the existing text pairing adapter contract and forwarding the approved request's account id is the smallest maintainable solution; adding defaults, config, or replay should be handled separately.
