## Summary

Keep open. Current main and the latest release still expose a generic `pairing approve --notify` contract, but WhatsApp registers pairing without any approval notification callback, so the sender is not messaged after approval.

## What This Changes

The issue requests notifying a WhatsApp sender after their DM pairing code is approved.

## Best Possible Solution

Implement WhatsApp support for the existing `pairing approve --notify` contract with account-aware routing, and leave default-on notification or blocked-message replay for separate product work.

## Reproduction Assessment

Yes, at source level. `pairing approve --notify` reaches `notifyPairingApproved`, and the current WhatsApp plugin has no `notifyApproval` callback, so the notifier returns without sending.

## Solution Assessment

Yes for the narrow repair. Reusing the existing pairing adapter notification contract and forwarding account scope is the smallest maintainable fix; making notifications default, configurable, or replaying the original message should be separate work.
