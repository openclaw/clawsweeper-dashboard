## Summary

Keep open. Current main and v2026.5.18 still have the generic pairing `--notify` path, but WhatsApp registers a pairing adapter without an approval notification callback, so the existing notification contract cannot message the approved sender.

## What This Changes

The issue asks for WhatsApp pairing approvals to notify newly approved senders, with possible default-on, configurable-message, or blocked-message replay behavior.

## Best Possible Solution

Implement WhatsApp support for the existing `pairing approve --notify` contract with account-aware routing, and keep default-on notification, configurable text, and blocked-message replay as separate product decisions.

## Reproduction Assessment

Yes, at source level. `pairing approve --notify` reaches `notifyPairingApproved`, which returns when `adapter.notifyApproval` is absent, and the current WhatsApp adapter does not define that callback.

## Solution Assessment

Yes for the narrow repair. Reusing the existing pairing adapter notification contract, forwarding account scope, and sending the standard approval message through WhatsApp outbound is the smallest maintainable path; default-on notification and replay should stay separate.
