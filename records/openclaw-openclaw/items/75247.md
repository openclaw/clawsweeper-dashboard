## Summary

Keep open. The PR addresses a valid source-reproducible CLI/update mismatch, but the branch prints user-controlled porcelain filename output without terminal sanitization and still lacks after-fix real behavior proof.

## What This Changes

The PR forwards update-step stdout to progress rendering, warns on dirty clean-check stdout, prints offending entries in dirty-skip output, updates tests, and adds a changelog entry.

## Best Possible Solution

Keep the stdoutTail plumbing and dirty messaging, sanitize each porcelain line before terminal styling/logging, use neutral clean-check failure copy, and require redacted real CLI output proof before merge.

## Reproduction Assessment

Yes. A high-confidence source reproduction is a git install where `git status --porcelain` emits an untracked or modified entry while exiting 0; current main then skips with `reason: dirty` while progress rendering only sees exit code 0.

## Solution Assessment

No, not as-is. The proposed shape is narrow and maintainable, but it needs terminal-safe rendering for dirty status lines and clearer clean-check failure wording before merge.
