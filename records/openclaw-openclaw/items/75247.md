## Summary

Keep open: the reported CLI/update mismatch is real and the PR takes the right narrow shape, but it should not merge while it prints user-controlled dirty filenames to the terminal without sanitization.

## What This Changes

The PR forwards update-step stdout tails into CLI update progress/result handling, uses them to show dirty clean-check output, updates focused tests, and adds a changelog fix entry.

## Best Possible Solution

Keep the stdoutTail plumbing and shared dirty clean-check display, but sanitize every dirty status line before applying theme coloring or logging, then land with the existing focused tests plus CI and the manual untracked-file update check.

## Reproduction Assessment

Yes. A high-confidence reproduction exists by making `git status --porcelain` emit dirty entries while exiting 0: current main can skip with `reason: dirty` while the progress renderer reports the clean-check success label.

## Solution Assessment

No, not as-is. The proposed design is the narrow maintainable fix, but the branch should route dirty filename output through `sanitizeTerminalText()` or an equivalent terminal-safe helper before merge.
