## Summary

Keep open. Current main still has the update-progress mismatch, but this PR is not merge-ready because the new dirty-entry display prints untrusted porcelain output and leaves one clean-check failure label misleading.

## What This Changes

The PR forwards update-step stdout tails, changes dirty clean-check rendering and dirty skipped-result guidance, adds focused tests, and adds a changelog fix entry.

## Best Possible Solution

Keep the stdoutTail plumbing and dirty-entry guidance, but sanitize every displayed porcelain line before coloring/logging and use a neutral clean-check failure label before landing.

## Reproduction Assessment

Yes. Make `git status --porcelain` emit a modified or untracked entry while exiting 0; current main can render `Working directory is clean` while the runner skips with `reason: dirty`.

## Solution Assessment

No, not as-is. The proposed shape is narrow and maintainable, but the PR should route dirty status text through `sanitizeTerminalText()` and special-case clean-check failures before merge.
