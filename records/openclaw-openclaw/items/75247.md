Keep open: the bug is real and the PR is directionally narrow, but it still renders user-controlled porcelain filenames without terminal sanitization and lacks contributor-provided real behavior proof.

## What This Changes

The PR forwards update-step stdout to CLI progress rendering, changes dirty clean-check display/output, updates related tests, and adds a changelog entry.

## Best Possible Solution

Keep the stdoutTail plumbing and dirty messaging, sanitize every printed porcelain line with `sanitizeTerminalText()` before styling, cover control-character filenames, and require redacted real CLI output proof.

## Reproduction Assessment

Yes. A high-confidence source reproduction is a git install where `git status --porcelain` emits an untracked or modified entry while exiting 0; current main then skips as dirty while progress rendering only sees a zero exit code.

## Solution Assessment

No, not as-is. The approach is narrow and maintainable, but it needs terminal-safe rendering for dirty status lines and contributor-provided real CLI proof before merge.
