## Summary

Keep open. The reported CLI/update mismatch is valid on current main, and the PR takes the right narrow shape by plumbing `stdoutTail` into update-step display, but the proposed patch introduces new raw terminal rendering of `git status --porcelain` entries in multiple paths. Existing review discussion already flagged the same terminal-safety issue, and current main has an existing sanitizer contract that should be used before merge.

## What This Changes

The PR forwards update-step stdout tails into update progress completion, uses that output to show dirty clean-check results, surfaces dirty entries with `git stash -u` guidance, and adds focused tests plus a changelog entry.

## Best Possible Solution

Keep the additive `stdoutTail` plumbing and shared display-helper direction, but sanitize every dirty porcelain line before terminal coloring/logging and add focused sanitizer assertions for the live progress, step summary, and skipped-result paths before landing.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction exists by making `git status --porcelain` emit dirty entries while exiting 0: current main returns `reason: dirty` from the runner while the progress renderer can still print the clean-working-tree success label.

## Solution Assessment

No, not as-is. The proposed approach is the narrow maintainable fix for the mismatch, but it must route untrusted dirty status lines through `sanitizeTerminalText()` before merge.

## Review Findings

Overall correctness: patch is incorrect

  - body: This renders `stdoutTail` from `git status --porcelain` directly with `theme.warn(line)` in the live progress path. Dirty filenames are local user-controlled terminal text, so crafted ANSI/control characters can spoof or alter output; sanitize each line before coloring/logging it.
  - confidence: 0.86

  - body: The non-progress `Steps:` summary also prints the new dirty `stdoutTail` lines directly. When the spinner is hidden, users still get unsanitized filename text, so sanitize here or in the shared rendering path before applying terminal styling.
  - confidence: 0.84
- **[P2] Sanitize dirty entries before skipped-result output:** `src/cli/update-cli/update-command.ts:1627`
  - body: The dirty skipped-result branch logs the same porcelain lines through `theme.muted(line)`. This leaves a second raw terminal path even if progress output is disabled, so sanitize each line before printing it.
  - confidence: 0.86
