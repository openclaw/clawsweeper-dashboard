## Summary

Keep open. The PR fixes a real current-main CLI/update mismatch, but the branch adds raw rendering of dirty `git status --porcelain` output in terminal paths and needs sanitization before merge.

## What This Changes

The PR forwards update-step stdout tails, uses them to display dirty clean-check status and skipped-result entries, updates focused CLI/update tests, and adds a changelog fix entry.

## Best Possible Solution

Keep the stdoutTail plumbing and dirty clean-check display, but sanitize every dirty status line before coloring/logging and use a neutral clean-check failure label before landing.

## Reproduction Assessment

Yes. A high-confidence reproduction is to make `git status --porcelain` emit a dirty entry while exiting 0; current main can then render `Working directory is clean` while the runner skips with `reason: dirty`.

## Solution Assessment

No, not as-is. The proposed shape is the narrow maintainable fix, but terminal-safe rendering should use `sanitizeTerminalText()` or an equivalent helper before merge.
