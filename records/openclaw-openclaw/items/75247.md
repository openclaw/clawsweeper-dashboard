## Summary

Keep open: this PR addresses a real CLI/update mismatch, but the patch is not mergeable as-is because it prints dirty worktree entries from filenames without terminal sanitization and the contributor has not provided after-fix real behavior proof.

## What This Changes

The branch forwards update-step stdout tails, changes update progress/result display for dirty clean-check output, updates dirty-skip guidance and tests, and adds a changelog entry.

## Best Possible Solution

Keep the stdoutTail plumbing and dirty messaging, sanitize each porcelain line before coloring/logging, use neutral clean-check failure copy, then add real after-fix CLI output proof.

## Reproduction Assessment

Yes. A high-confidence source reproduction is a git install where git status --porcelain emits an untracked or modified entry while exiting 0; current main then skips with reason dirty while progress rendering only sees exitCode 0.

## Solution Assessment

No, not as-is. The proposed shape is narrow and maintainable, but it must sanitize dirty status lines and avoid a misleading clean-check failure label before merge.
