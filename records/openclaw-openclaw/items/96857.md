## Label Justifications

- `P1`: Live Discord and Telegram sessions can lose ordinary tool output in agent-visible context, breaking active agent debugging and follow-up workflows.
- `impact:message-loss`: Tool-result text can be dropped or replaced before the model can read it, causing actionable command/status output to be lost.
- `impact:session-state`: The report ties the degradation to prompt-history truncation and per-session replay/materialization state that persisted for the affected session.

## Summary

Keep open: current main has partial fixes for visible display and empty non-image fallbacks, but model-visible provider/transport conversion still drops structured non-text tool-result content, and https://github.com/openclaw/openclaw/pull/97742 is the active open closing PR for the remaining fix.

## What This Changes

The issue reports that ordinary text/JSON tool results can reach the agent-visible model context as `(see attached image)` placeholders instead of readable text.

## Best Possible Solution

Land or supersede https://github.com/openclaw/openclaw/pull/97742 with one shared structured non-image tool-result text extractor across the provider and active transport replay paths, preserving image-only and empty-output fallbacks with focused tests and real behavior proof.

## Reproduction Assessment

Yes from source: current main still filters tool-result content down to `type: "text"` in OpenAI Responses shared, OpenAI transport, OpenAI-compatible completions, and Google/Gemini conversion paths. I did not run a live Discord or Telegram reproduction in this read-only review.

## Solution Assessment

Yes, the best current direction is a single bounded structured-text extraction path applied across every model-visible provider/transport converter, not another one-sided provider patch. The issue should stay open until the linked replacement PR lands or is replaced by an equivalent complete fix.
