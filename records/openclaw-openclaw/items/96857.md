## Summary

Keep open: current main includes the older MCP resource/materialization fix, but that does not prove the broader ordinary text tool-output placeholder failure is fixed. The remaining issue needs transcript/provider replay investigation around the affected session rather than cleanup closure.

## What This Changes

The issue reports that ordinary text tool results such as exec, read, session status, and GitHub CLI JSON can appear to the agent as literal `(see attached image)` placeholders instead of text.

## Best Possible Solution

Identify the boundary that converts or loses ordinary text toolResult content before model replay, keep the MCP-specific normalization as-is, and add regression coverage for exec/read/session_status output across the affected provider fallback path.
