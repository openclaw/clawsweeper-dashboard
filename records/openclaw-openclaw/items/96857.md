## Summary

Keep this issue open: current main still has provider/model replay paths that extract only explicit `type: "text"` tool-result blocks, while the live issue reports structured/text-like tool output becoming placeholder text. ## What This Changes

The issue asks OpenClaw to preserve normal tool text/JSON/stdout outputs in agent-visible transcript/model context instead of replacing them with `(see attached image)` placeholders.
