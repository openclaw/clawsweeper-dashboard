## Summary

Keep open: current `main` still has a source-level Anthropic Vertex externalization regression. The model path asks the generic facade loader for `anthropic-vertex/api.js`, but the installed-plugin fallback only checks package-root public surfaces while the npm plugin build emits top-level surfaces under package-local `dist/`.

## What This Changes

The issue reports that externalized Anthropic Vertex provider model calls fail in 2026.5.12 because provider runtime resolution cannot find `anthropic-vertex/api.js` from the installed npm plugin package.

## Best Possible Solution

Extend the generic installed-plugin public-surface resolver to locate built package surfaces such as `dist/api.js`, and update regression coverage to match the published Anthropic Vertex package layout.

## Reproduction Assessment

Yes, at source level: Anthropic Vertex model calls request `anthropic-vertex/api.js`, the installed-plugin resolver does not check `dist/api.js`, and the package build emits public surfaces under `dist/`. A packaged live run would be useful before landing but is not needed to see the broken path.

## Solution Assessment

Yes. The maintainable fix is a generic installed-plugin facade resolver update with a real package-layout regression test, not an Anthropic-specific shim or re-bundling the provider.
