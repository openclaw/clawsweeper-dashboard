## Summary

Keep open: the report describes a valid 2026.5.12 regression, and current main still appears to miss the real npm package layout for provider public surfaces. The existing Anthropic Vertex regression test uses `api.js` at the plugin package root, while OpenClaw's npm plugin build emits public surfaces under package-local `dist/`, so the reported installed-package path is not fully covered.

## What This Changes

The issue asks OpenClaw to resolve the externalized `@openclaw/anthropic-vertex-provider` model-provider `api.js` surface from an installed npm plugin instead of only the removed bundled plugin directory.

## Best Possible Solution

Extend the generic installed-plugin public-surface resolver to find package-local built public surfaces such as `dist/api.js`, and add a regression test that matches the real published Anthropic Vertex package layout.

## Reproduction Assessment

Yes, at source level: model calls load `anthropic-vertex/api.js`, the installed-plugin resolver does not check package-local `dist/api.js`, and the npm package build emits top-level public surfaces under `dist/`. A live packaged run would still be useful before landing.

## Solution Assessment

No, the current main test is not the best fix because it models a root-level `api.js` that the published npm package does not provide. The maintainable fix is a generic resolver update for installed plugin public surfaces, not an Anthropic-specific shim.
