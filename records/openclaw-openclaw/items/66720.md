## Summary

Keep open: the schema-filtering direction is useful, but the proposed option remains inert for normal provider requests because no production caller sets it. Current main already hides ACP-only fields when ACP is unavailable, so the remaining strict-gateway behavior needs deliberate activation plus real behavior proof.

## What This Changes

Adds a `simplifiedSchemaForGateway` option that filters selected `sessions_spawn` schema properties and passes it from `createOpenClawTools` into `createSessionsSpawnTool`.

## Best Possible Solution

Integrate property filtering into a provider/model-compat or documented config path that reaches runtime tool construction, while preserving current dynamic ACP/thread schema behavior and covering it with focused schema tests plus redacted strict-provider proof.

## Reproduction Assessment

Yes, by source inspection. The PR only forwards `options?.simplifiedSchemaForGateway`, and the current non-test runtime callers do not set that option, so normal provider requests still use the full strict-gateway-unfriendly schema path.

## Solution Assessment

No. The filter helper is a useful start, but an internal option with no production activation is not a complete or maintainable fix for the reported provider failure.
