## Summary

Keep open. The PR adds a plausible schema-filtering hook, but the hook is still opt-in only and no live provider/config path supplies it, so the reported strict-gateway failure would remain in normal runtime.

## What This Changes

The PR adds a `simplifiedSchemaForGateway` option that strips selected `sessions_spawn` schema properties and forwards that option through `createOpenClawTools`.

## Best Possible Solution

Add a deliberate provider/model-compat or explicit config activation path for the stripped schema, while preserving the full schema for native ACP/subagent users and covering both paths with focused schema tests.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: follow the live `pi-tools` to `createOpenClawTools` to `createSessionsSpawnTool` path and the strict-provider-unfriendly fields remain in the emitted schema because no caller supplies the new flag.

## Solution Assessment

No. The proposed schema builder is a useful starting point, but an unactivated internal option is not the narrowest complete fix for the reported provider failure.
