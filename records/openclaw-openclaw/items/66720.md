## Summary

Keep open. The PR targets a real provider-compatibility pain point, but the current patch is still inactive for normal runtime requests and the external contributor has not provided after-fix real behavior proof.

## What This Changes

The PR adds a `simplifiedSchemaForGateway` option to strip selected `sessions_spawn` schema properties and threads that option from `createOpenClawTools` into `createSessionsSpawnTool`.

## Best Possible Solution

Implement provider/model-compat or config-backed schema-property filtering in the live tool-preparation path, preserving native ACP/subagent affordances where supported and covering it with focused schema tests plus redacted strict-provider proof.

## Reproduction Assessment

Yes for the PR defect: source review shows the new option only affects `sessions_spawn` when an upstream caller supplies it, and the live provider and gateway callers do not. I did not reproduce the original strict-provider failure in a live proxy environment.

## Solution Assessment

No. The schema-filtering helper is useful, but forwarding an unset option is not a complete fix because normal strict-provider requests can still receive the unstripped schema.
