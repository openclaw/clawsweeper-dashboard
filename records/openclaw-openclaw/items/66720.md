## Summary

Keep open. The PR adds a useful schema-stripping helper and pass-through option, but the option is not activated by any current non-test runtime path, so the reported strict-provider compatibility failure remains for normal provider requests.

## What This Changes

The PR adds a `simplifiedSchemaForGateway` option that strips selected `sessions_spawn` schema fields and forwards that option through `createOpenClawTools`.

## Best Possible Solution

Add a deliberate provider/model-compat or explicit config activation path for the stripped schema, preserve the full schema for native ACP/subagent users, and cover both paths with focused schema tests.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: follow `pi-tools` to `createOpenClawTools` to `createSessionsSpawnTool`, and no caller supplies the new flag, so the strict-gateway-unfriendly fields remain in normal tool schemas.

## Solution Assessment

No. The schema builder is a useful start, but an unactivated internal option is not a complete fix for the provider rejection described by the linked report.
