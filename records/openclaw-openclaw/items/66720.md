## Summary

Keep open. The PR is aimed at a real compatibility surface, but its new schema-stripping flag is still inert in normal provider/gateway tool assembly and the external PR lacks after-fix strict-provider proof.

## What This Changes

Adds a `simplifiedSchemaForGateway` option that strips selected `sessions_spawn` schema properties and forwards it from `createOpenClawTools` into `createSessionsSpawnTool`.

## Best Possible Solution

Implement provider/model-compat or config-backed `sessions_spawn` schema property filtering in the live tool-preparation path, preserving native ACP/subagent affordances, with focused schema tests and redacted strict-provider proof.

## Reproduction Assessment

Yes for the PR defect: source inspection shows the added option is only usable if callers pass it, and the live provider, gateway, and inline-action callers do not. The original strict-provider/proxy failure still lacks live proof in the PR.

## Solution Assessment

No. A schema builder may be useful, but this PR does not activate it in the live provider/gateway path; the durable fix needs a clear provider/model-compat or config-backed activation contract.
