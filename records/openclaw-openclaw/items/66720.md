## Summary

Keep open: this PR is the focused schema-compatibility tracker for the closed linked provider issue, but the current branch still only adds an internal flag and pass-through; no live provider path sets it, and no real strict-provider proof is present.

## What This Changes

The PR adds a `simplifiedSchemaForGateway` flag to filter selected `sessions_spawn` schema properties and forwards that flag from `createOpenClawTools` into `createSessionsSpawnTool`.

## Best Possible Solution

Use a deliberate provider/model-compat or config-backed schema-property filter in the live tool-preparation path, preserving ACP/subagent affordances for native runtimes and covering it with focused schema tests plus redacted strict-provider proof.

## Reproduction Assessment

Yes. Source reproduction is high-confidence: the PR's option is only forwarded if an upstream caller supplies it, while the `pi-tools` and gateway paths on current main do not set a `sessions_spawn` property-stripping option.

## Solution Assessment

No. The schema filtering helper is a useful start, but the proposed wiring does not activate for a real provider path, so normal requests can still emit the strict-provider-unfriendly schema.
