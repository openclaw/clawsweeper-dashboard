## Summary

Keep open: the PR targets a real config persistence bug, but the latest head still misses numeric array-path writes, lacks required after-fix real behavior proof, and carries the protected `maintainer` label.

## What This Changes

The PR threads explicit `config set`/`config patch` paths through config write options, injects default-equal explicit values into persisted candidates, rejects include-owned explicit writes, adds regression tests, and updates the changelog.

## Best Possible Solution

Update the explicit-path injection helpers to traverse numeric array segments consistently with CLI path parsing, add regression coverage for `agents.list[0]` default-equal writes, then add redacted real behavior proof and land the canonical config persistence fix.

## Reproduction Assessment

Yes. Source inspection on current main shows the default-equal write can be dropped before persistence, and the linked user report gives concrete CLI steps against v2026.5.7; the PR's remaining array gap is also source-reproducible from the helper contracts.

## Solution Assessment

No. The explicit-path plumbing is the right narrow direction, but the current implementation should use array-aware get/has/set path helpers and add array-index regression coverage before it is the best fix.
