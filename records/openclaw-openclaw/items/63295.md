## Summary

Keep open. Current main still builds `inboundUserContext` but omits it from the `isBareSessionReset` prompt branch, so model-executed reset paths can still lose sender/history metadata. Bare no-tail `/new` now acknowledges locally, which changes the original greeting path, but the remaining reset-tail and soft-reset prompt assembly bug is still real and is already targeted by open PR #72866.

## What This Changes

The issue requests that session reset prompts preserve the untrusted sender envelope from channel metadata so agents can personalize reset turns.

## Best Possible Solution

Review and land PR #72866 or an equivalent focused change that includes `inboundUserContext` in the `isBareSessionReset` prompt array, with regression coverage that actually drives a soft-reset or true new-session bare-reset model execution path. Preserve the current local acknowledgement behavior for no-tail `/new` and `/reset` unless maintainers make a separate product decision.

## Reproduction Assessment

Yes. A high-confidence static reproduction is a soft reset or reset-tail turn with sender metadata or non-empty `InboundHistory`: current main lets the turn reach the agent, but the reset prompt branch drops `inboundUserContext` before `applySessionHints` and `runReplyAgent`. The original no-tail `/new` greeting path no longer reproduces because current main returns a local acknowledgement instead of invoking the model.

## Solution Assessment

Yes. Adding `inboundUserContext` to the reset prompt assembly is the narrow maintainable solution because the non-reset branch already uses the same untrusted metadata envelope and the formatter preserves the trust boundary. The paired PR should still verify the actual reset branch in tests before merge.
