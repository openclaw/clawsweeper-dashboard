## Summary

Keep open: current main still omits the untrusted inbound sender/history envelope from the model-executed reset prompt branch, while open PR #72866 is the focused implementation candidate for that remaining bug.

## What This Changes

Preserve the untrusted sender/history envelope on session reset prompt turns so agents can use available channel metadata after `/new` or `/reset`.

## Best Possible Solution

Review and land PR #72866 or an equivalent focused change that includes `inboundUserContext` in the `isBareSessionReset` prompt assembly with regression coverage for model-executed reset turns.

## Reproduction Assessment

Yes. Current main has a high-confidence static reproduction path: a model-executed reset turn can pass `hasUserBody` through inbound history, then feed a prompt assembled without `inboundUserContext`; the original bare no-tail `/new` greeting no longer reaches the model because it is locally acknowledged.

## Solution Assessment

Yes. Adding `inboundUserContext` to the reset prompt array is the narrowest maintainable fix for the remaining bug; restoring model-generated no-tail reset greetings would be a separate product decision.
