## Summary

Keep open: current main still omits `inboundUserContext` from the model-executed bare reset prompt branch, while open PR #72866 is the focused implementation candidate for the remaining bug. The original no-tail `/new` greeting path has changed to a local acknowledgement, so the remaining actionable surface is reset tails, soft resets, and inbound-history reset turns that still reach the model.

## What This Changes

The issue requests preserving the untrusted sender metadata envelope on `/new` or `/reset` session-reset prompts so first-turn replies can use known sender identity.

## Best Possible Solution

Land PR #72866 or an equivalent focused change that includes `inboundUserContext` in the `isBareSessionReset` prompt assembly with regression coverage, leaving no-tail reset greeting UX to #75314.

## Reproduction Assessment

Yes. Source inspection on current main shows reset tails or inbound history can satisfy the user-body guard, then the model prompt is assembled without `inboundUserContext`; no live channel run was performed in this read-only review.

## Solution Assessment

Yes for the remaining source-level bug. Adding `inboundUserContext` to the reset prompt branch is the narrowest maintainable fix; restoring model-generated no-tail `/new` greetings would be a separate product/UX decision.
