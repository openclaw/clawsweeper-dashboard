## Summary

Keep open: current main still omits the untrusted inbound sender/history envelope from the model-executed reset prompt branch, and open PR #72866 is the focused implementation candidate for that remaining bug.

## What This Changes

The issue asks for session reset prompts to preserve the sender envelope so agents can use channel-provided sender metadata on reset turns.

## Best Possible Solution

Review and land PR #72866 or an equivalent focused change that includes `inboundUserContext` in the `isBareSessionReset` prompt assembly with regression coverage for model-executed reset turns.

## Reproduction Assessment

Yes. Current main has a high-confidence source reproduction path: a model-executed reset can pass the inbound-history body guard, then feed a prompt assembled without `inboundUserContext`.

## Solution Assessment

Yes. Adding `inboundUserContext` to the reset prompt array is the narrowest maintainable fix for the remaining bug; restoring model-generated no-tail reset greetings should stay separate.
