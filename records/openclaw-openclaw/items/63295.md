## Summary

Keep open: current `main` still computes the inbound sender/history envelope but excludes it from the `isBareSessionReset` prompt branch, so model-executed reset paths can still run without the metadata that made the turn actionable. Bare no-tail `/new` and `/reset` now stop at a local acknowledgement, but open PR #72866 is the focused implementation candidate for the remaining prompt assembly bug.

## What This Changes

The issue requests preserving the `Sender (untrusted metadata)` envelope in session-reset prompts so agents can personalize or contextualize the first model turn after `/new` or `/reset`.

## Best Possible Solution

Review and land PR #72866 or an equivalent narrow change that includes `inboundUserContext` in the `isBareSessionReset` prompt assembly with regression coverage for reset-tail or soft-reset execution. Keep the current no-tail local acknowledgement behavior unless maintainers intentionally decide to restore model-generated reset greetings.

## Reproduction Assessment

Yes. Current `main` has a high-confidence static reproduction path: a model-executed reset turn with sender metadata or nonblank `InboundHistory` passes `hasUserBody`, but the reset branch omits `inboundUserContext` before `applySessionHints`. The original no-tail `/new` greeting path no longer reaches the model because it returns a local acknowledgement.

## Solution Assessment

Yes. Adding `inboundUserContext` to the reset prompt array, as PR #72866 proposes, is the narrowest maintainable fix for the remaining bug; changing no-tail resets to run a greeting would be a separate product decision.
