## Summary

Keep open: current main does not prove this stable-release failure is fixed. The visible direct cron announce branch already used durable outbound send in v2026.6.10, but source still permits cron announce runs to use the message tool, and WeChat's package has outbound send handlers without a message-action handler, matching the reported unsupported-action error path.

## What This Changes

The issue reports that isolated cron `delivery.mode="announce"` to `openclaw-weixin` fails with `Channel openclaw-weixin does not support action send`.

## Best Possible Solution

Trace the exact WeChat cron announce path, then make cron announce delivery avoid unsupported `message.action` for direct-send-only channels or bridge that action through the existing outbound send contract with focused regression coverage.

## Reproduction Assessment

No high-confidence current-main live reproduction is available. The stable-release report gives concrete steps and logs, and source inspection reproduces the unsupported-action path if cron announce uses the message tool against the WeChat plugin, but no live WeChat run proves the exact path on current main.

## Solution Assessment

Unclear: using outbound send is the right owner boundary for cron direct fallback, but that branch was already present in the reported stable release. The best fix likely needs a maintainer decision on cron message-tool policy versus a generic `message.action` outbound bridge.
