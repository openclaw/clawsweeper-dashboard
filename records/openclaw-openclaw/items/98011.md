## Summary

Keep open: current main does not provide enough source, history, or release proof that this stable WeChat cron announce failure is fixed, and the reported error matches a real Gateway message.action versus outbound-send contract mismatch.

## What This Changes

Isolated cron jobs with `delivery.mode=announce` should deliver to `openclaw-weixin` through the channel outbound send contract instead of failing with unsupported `message.action` send.

## Best Possible Solution

Fix the cron/message delivery boundary generically so isolated cron announce uses the channel outbound send contract and never requires `actions.handleAction('send')` for plugins that already implement `outbound.sendText`.

## Reproduction Assessment

No high-confidence live reproduction was run; source and dependency inspection show the reported error path is plausible because WeChat implements outbound send handlers while Gateway `message.action` requires an action handler.

## Solution Assessment

Yes: the best visible fix is a generic core delivery repair that routes cron announce through the existing outbound send contract, not a WeChat-specific plugin shim or new config option.
