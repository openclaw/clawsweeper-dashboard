## What This Changes

The branch adds a new `openclaw tools execute-plan` CLI, an `executeToolPlan` helper, docs/navigation entries, registration, and unit tests for replaying JSON tool plans through Gateway `tools.invoke`.

## Best Possible Solution

Keep the Gateway-backed `tools.invoke` design, reject invalid provided argument payloads before any invocation, then have maintainers decide whether to accept the new public CLI surface.

## Reproduction Assessment

Yes for the review finding: source inspection of the PR head shows scalar or array `input`/`args`/`arguments` is normalized to `{}` and sent to the invoker. The underlying PR is a feature addition, so current-main bug reproduction is otherwise not applicable.

## Solution Assessment

Mostly yes: replaying through `tools.invoke` is the right maintainable boundary for preserving policy and approval behavior. The implementation should fail closed on non-object provided args before calling the Gateway.
