## What This Changes

The PR adds a Gateway-backed `openclaw tools execute-plan` CLI, executor API, docs, registration, and unit tests for replaying JSON tool plans through `tools.invoke`.

## Best Possible Solution

Keep the Gateway-backed approach, but validate plan step arguments fail-closed before invocation and then have maintainers decide whether to accept the new public CLI surface.

## Reproduction Assessment

Yes for the review finding: source inspection shows a step with scalar or array `args` reaches `options.invoke` as `{}`. The original item is a feature PR, so current-main bug reproduction is otherwise not applicable.

## Solution Assessment

Mostly yes: `tools.invoke` is the narrowest maintainable boundary for preserving policy and approval behavior. The implementation should reject non-object provided argument payloads before invoking any tool.

## Review Findings

Overall correctness: patch is incorrect
