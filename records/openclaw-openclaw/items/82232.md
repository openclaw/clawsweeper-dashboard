## Summary

Keep open: this draft feature PR is not obsolete and current main lacks the new command, but the patch should not merge while malformed step arguments are silently coerced into empty tool invocations and the new public CLI surface still needs maintainer approval.

## What This Changes

Adds a Gateway-backed `openclaw tools execute-plan` CLI, executor API, docs, lazy registration, descriptors, and unit tests for replaying JSON tool plans through `tools.invoke`.

## Best Possible Solution

Keep the Gateway-backed `tools.invoke` design, require fail-closed validation for provided step arguments, then have maintainers decide whether to ship the new CLI surface.

## Reproduction Assessment

Yes for the review finding: source inspection of the PR head shows scalar or array step arguments normalized to `{}` before `options.invoke`. The original item is a feature PR, so current-main bug reproduction is otherwise not applicable.

## Solution Assessment

Mostly yes: `tools.invoke` is the narrowest maintainable boundary for preserving Gateway tool resolution, policy, and approval behavior. The implementation should reject malformed provided args before dispatch, and the new CLI surface still needs maintainer acceptance.
