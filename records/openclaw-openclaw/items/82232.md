## Summary

Keep open. Current main does not have this CLI or helper, the PR is active with sufficient live proof and no blocking correctness finding from this review, but adding deterministic tool-plan replay as a public core CLI surface still needs maintainer product/API approval before merge.

## What This Changes

The PR adds a Gateway-backed `openclaw tools execute-plan` CLI, an exported `executeToolPlan` helper, registration/descriptors, tests, and CLI docs for replaying JSON tool plans.

## Best Possible Solution

Land the Gateway-backed CLI if maintainers want this core surface; otherwise keep direct tool invocation planning in the broader feature discussion at https://github.com/openclaw/openclaw/issues/37131.

## Reproduction Assessment

Not applicable: this PR adds a new CLI feature rather than fixing a current-main bug. The behavior is source-reviewable, and the PR body supplies live Gateway proof for the new flow.

## Solution Assessment

Mostly yes: replaying through Gateway `tools.invoke` is the narrow maintainable boundary because it preserves existing tool resolution, policy, approval, and `before_tool_call` behavior. The remaining question is whether this public CLI belongs in core.
