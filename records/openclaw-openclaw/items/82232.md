## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P2] Preserve tool action payloads when `tool` is present:** `src/tools/execute-plan.ts:107-113`
  - body: Current Gateway `/tools/invoke` accepts `{ tool, action, args }`, with `action` mapped into args for action-style tools; the docs and tests use that shape. This normalizer prefers `raw.action` over `raw.tool`, so a plan copied from the documented Gateway shape would invoke `json` instead of `agents_list` and never forward the action. Please use `tool`/`name` as the tool name when present, and only treat `action` as the tool-name fallback or preserve it as `args.action`.
  - confidence: 0.88
