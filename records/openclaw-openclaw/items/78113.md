## What This Changes

The PR changes agent workspace resolution, tests, and changelog so non-default agents without explicit workspaces fall back to `stateDir/workspace-<id>` even when `agents.defaults.workspace` is set, while default agents still use that configured default.

## Best Possible Solution

Land a narrow resolver, test, and changelog update that restores sibling defaults for non-default agents while preserving default-agent and explicit per-agent workspace behavior.

## Reproduction Assessment

Yes from source inspection: current main joins `agents.defaults.workspace` with a non-default agent id, and `agents add` uses that resolver for the interactive workspace default. I did not run the CLI because this review is read-only and that path would mutate config/workspaces.

## Solution Assessment

Yes, the proposed code direction is the narrowest maintainable fix: apply `agents.defaults.workspace` only to the configured default agent and restore `stateDir/workspace-<id>` for implicit non-default workspaces. It still needs real behavior proof and the changelog credit fix before merge.
