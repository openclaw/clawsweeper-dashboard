## Summary

Keep open: current main supports manual `sessions_spawn.cwd` and workspace inheritance, but it does not provide SKILL.md workdir strategy metadata, automatic per-spawn WORKDIR creation/injection, or merge/output validation. This is a new skill/subagent runtime contract that needs maintainer product design rather than conservative cleanup closure.

## What This Changes

Request to add built-in workdir isolation for skills that spawn parallel subagents, including SKILL.md metadata, automatic temporary WORKDIR injection, and merge validation.

## Best Possible Solution

Design a narrow skill/subagent contract that lets skill authors opt into isolated per-spawn output directories and exposes those paths to the parent without turning core into a broad orchestration layer.

## Reproduction Assessment

Not applicable: this is a feature/API request, not a broken documented behavior with a current-main repro path. Source inspection confirms the requested automatic workdir strategy is absent while manual `cwd` support exists.

## Solution Assessment

Unclear: the proposed layers are plausible, but the best solution needs maintainer design around skill metadata, runtime injection, cleanup, and expected-output validation. A narrower manifest/runtime contract may be preferable to hard-coding merge behavior in core.
