## Summary

Keep open: current main and the latest release support caller-provided `sessions_spawn.cwd`, but they do not implement skill-declared workdir strategy metadata, automatic per-spawn WORKDIR creation/injection, parent-visible generated workdirs, or merge/output validation. This remains a new skill/subagent runtime contract that needs maintainer product design rather than conservative cleanup closure.

## What This Changes

The issue requests built-in workdir isolation for skills that spawn subagents, including SKILL.md metadata, automatic WORKDIR injection, and merge validation.

## Best Possible Solution

Design a narrow opt-in skill/subagent contract for isolated per-spawn output directories, including lifecycle, cleanup, and parent-visible outputs, without turning core into a broad skill-specific merge orchestrator.

## Reproduction Assessment

Not applicable: this is a feature/API request, not a broken documented behavior with a current-main reproduction path. Source inspection confirms manual `cwd` support exists while the requested automatic workdir strategy is absent.

## Solution Assessment

Unclear: the proposed layers are plausible, but merge validation needs a defined expected-output contract and the injection boundary needs maintainer design. A narrower manifest/runtime contract may be better than hard-coded merge behavior.
