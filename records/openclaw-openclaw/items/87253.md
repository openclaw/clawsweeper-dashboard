## Summary

Keep open: current main and the latest release still only support caller-provided `sessions_spawn.cwd`; they do not implement skill-declared `spawnsSubagents`/`workdirStrategy`, automatic per-spawn `WORKDIR` creation, parent-visible workdir reporting, or merge/output validation. This remains a new skill/subagent runtime contract that needs maintainer product design, not conservative cleanup closure.

## What This Changes

The issue asks OpenClaw to add skill-declared workdir isolation for parallel subagent spawns, including automatic `WORKDIR` creation/injection and output validation.

## Best Possible Solution

Design a narrow opt-in skill/subagent contract for isolated per-spawn output directories, with lifecycle, cleanup, and parent-visible output semantics, without making core a broad skill-specific merge orchestrator.

## Reproduction Assessment

Not applicable: this is a feature/API request, not broken existing documented behavior. Source inspection confirms current main and `v2026.6.9` support caller-provided `cwd` but not automatic skill-declared workdir isolation.

## Solution Assessment

Unclear: the proposed layers are plausible, but merge validation needs a defined expected-output contract and the injection boundary needs maintainer design. A narrower manifest/runtime contract may be better than hard-coded merge behavior.
