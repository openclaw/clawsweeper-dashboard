## Summary

Keep open: current main and v2026.6.8 still only support caller-provided `sessions_spawn.cwd`; they do not implement skill-declared workdir strategy metadata, automatic per-spawn `WORKDIR` creation/injection, parent-visible generated workdirs, or merge validation. This remains a new skill/subagent runtime contract that needs maintainer product design.

## What This Changes

The issue requests built-in skill/subagent workdir isolation through new skill metadata, automatic temporary workdir injection, and output/merge validation.

## Best Possible Solution

Design a narrow opt-in skill/subagent contract for isolated per-spawn output directories, including lifecycle, cleanup, and parent-visible outputs, without making core a skill-specific merge orchestrator.

## Reproduction Assessment

Not applicable: this is a feature/API request, not broken existing documented behavior. Source inspection confirms current main and v2026.6.8 support caller-provided `cwd` but not automatic skill-declared workdir isolation.

## Solution Assessment

Unclear: the proposed layers are plausible, but merge validation needs a defined expected-output contract and the injection boundary needs maintainer design. A narrower manifest/runtime contract may be better than hard-coded merge behavior.
