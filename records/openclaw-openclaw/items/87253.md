## Summary

Keep open: current main has manual `sessions_spawn.cwd` and persisted spawned workspace/cwd metadata, but it still lacks the requested SKILL.md workdir strategy metadata, automatic per-spawn temporary directory injection, parent workdir reporting, and merge/output validation contract. This remains a feature/API design item rather than conservative cleanup.

## What This Changes

The issue requests built-in workdir isolation for skills that spawn parallel subagents, driven by SKILL.md metadata and enforced by the runtime.

## Best Possible Solution

Design a narrow opt-in skill/runtime contract for isolated per-spawn output roots, including parent-visible workdir reporting and a declarative expected-output validation shape, without turning core into a broad workflow merge engine.

Not applicable: this is a feature/API request, not a broken documented behavior with a current-main repro path. Source inspection confirms the requested automatic workdir strategy is absent while manual `cwd` support exists.

## Solution Assessment
