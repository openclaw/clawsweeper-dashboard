## Summary

Keep open: current main and v2026.6.10 still only support caller-provided `sessions_spawn.cwd`, not skill-declared `spawnsSubagents`/`workdirStrategy`, automatic per-spawn `WORKDIR` creation, parent-visible workdir reporting, or merge/output validation. The linked plur1bus repository is useful adjacent context for isolated per-agent/workspace memory, but it does not replace the missing OpenClaw skill/subagent runtime contract.

## What This Changes

Request to add a public skill metadata and subagent runtime contract that automatically creates unique per-subagent workdirs and validates outputs for skills that spawn subagents.

## Best Possible Solution

Design a narrow opt-in skill/subagent contract for isolated per-spawn output directories, with explicit lifecycle, cleanup, parent-visible output, and validation semantics, without making core a broad skill-specific merge orchestrator.

## Reproduction Assessment

Not applicable: this is a feature/API request, not broken existing documented behavior. Source inspection confirms current main and v2026.6.10 support caller-provided `cwd` but not automatic skill-declared workdir isolation.

## Solution Assessment

Unclear: the proposed layers are plausible, but the skill invocation boundary and merge validation contract need maintainer design. The current supported path is caller-provided `sessions_spawn.cwd`, which is only a manual mitigation.
