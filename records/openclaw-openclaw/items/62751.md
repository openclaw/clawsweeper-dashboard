## Summary

Keep open. Current main still has the hook consistency gap: `sessions.patch` emits `session:patch`, but `/model` and `session_status(model=...)` mutate model state through direct session-store writes without dispatching the same hook.

## What This Changes

The issue asks for model changes made through `/model` or `session_status(model=...)` to emit the existing `session:patch` hook event.

## Best Possible Solution

Route successful user-driven model selection writes through a shared hook-emitting session patch helper, preserving existing live-switch, auth-profile, visibility, and queued-followup semantics.

## Reproduction Assessment

Yes, at source level. Register a `session:patch` hook, then follow the current `/model` or `session_status(model=...)` paths: both persist model changes without the hook dispatch used by `sessions.patch`.

## Solution Assessment

Yes. Option A is the narrow maintainable fix because the existing `session:patch` contract and model patch helper already exist; adding a new `session:model_changed` event would expand the API unnecessarily.
