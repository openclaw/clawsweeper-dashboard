## Summary

Keep open. Current `main` still has the documented hook consistency gap: `sessions.patch` emits `session:patch`, while `/model` and `session_status(model=...)` persist model changes through direct session-store writes without dispatching that hook.

## What This Changes

The issue requests that model changes from `/model` and `session_status(model=...)` emit the same `session:patch` hook event as `sessions.patch`.

## Best Possible Solution

Route successful user-driven model selection writes through a shared hook-emitting session patch path while preserving existing validation, auth-profile, live-switch, visibility, and queued-followup behavior.

## Reproduction Assessment

Yes, at source level. Register a `session:patch` hook, then follow current `/model` or `session_status(model=...)`: both persist model changes without the hook dispatch used by `sessions.patch`.

## Solution Assessment

Yes. Reusing the existing `session:patch` contract for model writes is the narrow maintainable fix; adding a separate `session:model_changed` event would expand the hook API unnecessarily.
