## Summary

Keep open. Current main still has the reported hook consistency bug: `sessions.patch` emits `session:patch`, while `/model` and `session_status(model=...)` persist model overrides through direct session-store writes without invoking the same hook path.

## What This Changes

The issue asks for model changes made through `/model` and `session_status(model=...)` to emit the existing `session:patch` hook event.

## Best Possible Solution

Route successful user-driven model mutations through one hook-emitting session patch path, preserving current model validation, auth-profile, live-switch, visibility, and status semantics.

## Reproduction Assessment

Yes, at source level. `sessions.patch` dispatches `session:patch`, while `/model` and `session_status(model=...)` mutate the stored session model directly without a hook dispatch.

## Solution Assessment

Yes. Option A is the narrow maintainable fix because the hook contract and model patch helper already exist; adding a new `session:model_changed` event would expand the API unnecessarily.
