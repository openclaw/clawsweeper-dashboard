## Summary

Keep open. Current main still has the reported hook consistency gap: `sessions.patch` emits `session:patch`, while `/model` and `session_status(model=...)` persist model changes through direct store updates that bypass the hook dispatch path.

## What This Changes

Request that model changes made through `/model` or agent-internal paths emit a `session:patch` hook event with the model patch.

## Best Possible Solution

Route all user-driven model-selection persistence through a shared hook-emitting session patch helper while preserving current live-switch, visibility, and exact-selection semantics.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: register a `session:patch` listener, then follow the current `/model` or `session_status(model=...)` code paths; both persist the model without the hook dispatch used by `sessions.patch`.

## Solution Assessment

Yes. Option A is the narrow maintainable fix because it preserves the existing `session:patch` contract instead of adding a separate event surface for the same patchable session field.
