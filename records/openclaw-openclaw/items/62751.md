## Summary

Keep open. Current main still has the reported hook consistency gap: `sessions.patch` emits `session:patch`, but `/model` and `session_status(model=...)` persist model selections through direct session-store updates without the same hook dispatch path.

## What This Changes

The issue requests that model changes made through `/model` or agent-internal paths emit the same `session:patch` hook event as `sessions.patch`.

## Best Possible Solution

Route all user-driven model-selection persistence through a shared hook-emitting session patch helper, while preserving existing live-switch, visibility, and exact-selection semantics.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: register a `session:patch` listener, then follow the current `/model` or `session_status(model=...)` paths, both of which persist model changes without the hook dispatch used by `sessions.patch`.

## Solution Assessment

Yes. Option A is the narrow maintainable fix because it restores the documented `session:patch` contract without introducing a separate `session:model_changed` event surface.
