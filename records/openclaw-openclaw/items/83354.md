## Summary

## What This Changes

The issue reports that helper commands may restart a stopped and disabled Linux user-level `openclaw-gateway` systemd unit.

## Best Possible Solution

Define the Linux gateway service lifecycle contract so diagnostic commands stay non-mutating, helper recovery does not override disabled or masked operator intent, and only explicit lifecycle commands start disabled units with docs and regression coverage.

## Reproduction Assessment

No high-confidence live reproduction for the exact `openclaw status` sequence. A closely related current-main path is source-reproducible because readiness and doctor flows can start or restart an installed stopped service.

## Solution Assessment

Unclear. The safer direction is to separate diagnostic/helper recovery from explicit lifecycle commands and teach Linux systemd handling to respect disabled or masked units, rather than changing recovery defaults before maintainers choose the contract.
