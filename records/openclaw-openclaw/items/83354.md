## Summary

Keep open. Current main still has helper, doctor, and configure lifecycle paths that can start or restart an installed stopped gateway service, so this fresh Linux systemd report is not a cleanup-close candidate even though the exact `openclaw status` path looks read-only.

## What This Changes

The issue asks OpenClaw helper commands to respect explicitly stopped or disabled user-level `openclaw-gateway` systemd units instead of silently starting them again.

## Best Possible Solution

Define the Linux service lifecycle contract so diagnostic commands remain non-mutating, helper recovery paths respect disabled or masked operator intent, and only explicit start/install/restart flows can bring a user-level unit back with docs and regression coverage.

## Reproduction Assessment

No high-confidence live reproduction was established for the exact `openclaw status` sequence. Source inspection does show current helper, doctor, and configure paths that can restart an installed stopped service, which is enough to keep this open but not enough to mark the full report reproduced.

## Solution Assessment

Unclear. A broad new flag may not be the narrowest first fix; the safer direction is to distinguish diagnostics from lifecycle repair and teach Linux systemd handling to respect disabled or masked units where operator intent should win.
