## Summary

Keep open. Current `openclaw status` looks read-only, but current main and the latest release still contain helper/doctor lifecycle paths that can restart an installed but stopped Linux user systemd gateway, so this needs a maintainer lifecycle-policy decision rather than cleanup closure.

## What This Changes

The issue asks OpenClaw helper and diagnostic flows to stop auto-starting a stopped or disabled user-level systemd gateway and to document disable versus mask behavior.

## Best Possible Solution

Define the Linux service lifecycle contract so diagnostic status stays non-mutating, helper recovery respects disabled or masked operator intent, and only explicit lifecycle commands can restart disabled user units with clear docs and tests.

## Reproduction Assessment

No high-confidence exact reproduction for the issue's `openclaw status` sequence: the current status source appears read-only and no live Ubuntu systemd run was performed. A closely related current-main path is source-reproducible because readiness and doctor flows can call `systemctl --user restart` for an installed stopped service.

## Solution Assessment

Unclear. Adding a broad `--ensure-running` flag may be more surface than needed; the better first step is a maintainer lifecycle decision that separates diagnostics and helper recovery from explicit start/install/restart commands.
