## Summary

Keep open. Current `main` still has source-reproducible helper and doctor recovery paths that can restart an installed stopped gateway service, even though the exact `openclaw status` path appears read-only; the policy for respecting disabled user-level systemd units needs maintainer follow-up.

## What This Changes

The issue asks OpenClaw helper commands to stop silently starting a previously stopped and disabled user-level `openclaw-gateway` systemd unit.

## Best Possible Solution

Define the Linux service lifecycle contract so diagnostic commands stay non-mutating, helper recovery does not override disabled or masked operator intent, and only explicit start/install/restart flows can bring a disabled unit back with docs and regression coverage.

## Reproduction Assessment

No high-confidence exact reproduction for the issue's `openclaw status` sequence was established; current source makes status look read-only. A closely related current-main helper/doctor path is source-reproducible because stopped installed services can flow into `systemctl --user restart`.

## Solution Assessment

Unclear. A broad `--ensure-running` flag is probably not the first fix; the safer direction is to distinguish diagnostics from explicit lifecycle commands and make Linux systemd handling respect disabled or masked units.
