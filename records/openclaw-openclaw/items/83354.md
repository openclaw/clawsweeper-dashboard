## Summary

Keep open. The exact `openclaw status` path appears read-only on current main, but current readiness and doctor paths can still start an installed, stopped gateway service, so this fresh Linux systemd report needs lifecycle review rather than cleanup closure.

## What This Changes

Respect operator-stopped or disabled user-level systemd gateway units so helper commands do not restart them unexpectedly.

## Best Possible Solution

Define the Linux service lifecycle contract so diagnostic status stays non-mutating, helper recovery does not override disabled or masked operator intent, and only explicit start/install/restart flows can bring a disabled unit back with clear docs and regression coverage.

## Reproduction Assessment

No high-confidence exact reproduction for the issue's `openclaw status` sequence: current source makes status look read-only and no live systemd run was performed. A closely related current-main helper/doctor path is source-reproducible because stopped installed services flow into `systemctl --user restart`.

## Solution Assessment

Unclear. A broad new flag is probably not the first fix; the safer direction is to distinguish diagnostic/helper recovery from explicit lifecycle commands and teach Linux systemd handling to respect disabled or masked units.
