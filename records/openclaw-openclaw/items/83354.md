## Summary

Keep open: current main still has source-visible helper and doctor paths that can start or restart an installed stopped Linux user-systemd gateway, while the exact `openclaw status` resurrection sequence is not confirmed on current source.

## What This Changes

The issue requests that helper commands stop silently starting a disabled user-level Linux `openclaw-gateway` service after an operator stops or disables it.

## Best Possible Solution

Define the Linux gateway lifecycle contract so diagnostic commands remain non-mutating, helper recovery does not override disabled or masked operator intent, and only explicit lifecycle commands start disabled units with docs and regression coverage.

## Reproduction Assessment

No high-confidence live reproduction for the exact `openclaw status` sequence; current source makes status look read-only. A closely related current-main helper/doctor path is source-reproducible because stopped installed services can flow into the daemon start/restart path.

## Solution Assessment

Unclear. The safer direction is to separate diagnostic/helper recovery from explicit lifecycle commands and teach Linux systemd handling to respect disabled or masked units before choosing any new flag or default behavior.
