## Summary

Keep open. Current main does not confirm the exact `openclaw status` resurrection path, but the source still has adjacent helper and doctor paths that can start or restart an installed stopped Linux user-systemd gateway, so this needs lifecycle-policy review rather than cleanup closure.

## What This Changes

The issue asks OpenClaw helper commands to stop silently restarting operator-stopped or disabled user-level Linux `openclaw-gateway` units.

## Best Possible Solution

Define the Linux gateway service lifecycle contract so diagnostic commands remain non-mutating, helper recovery does not override disabled or masked operator intent, and only explicit lifecycle commands start disabled units with clear docs and regression coverage.

## Reproduction Assessment

No high-confidence exact reproduction for the `openclaw status` sequence: current source makes status look read-only and no live systemd run was performed. A closely related current-main helper/doctor path is source-reproducible because stopped installed services can flow into `systemctl --user restart`.

## Solution Assessment

Unclear. The safer direction is to separate diagnostic/helper recovery from explicit lifecycle commands and teach Linux systemd handling to respect disabled or masked units, rather than adding a broad flag before maintainers choose the lifecycle contract.
