## Summary

Keep open. Current main still has source-confirmed helper and doctor paths that can start or restart a stopped installed Linux user-systemd gateway, while the exact `openclaw status` path appears read-only, so this needs lifecycle-policy review rather than cleanup closure.

## What This Changes

The issue asks OpenClaw helper commands to avoid silently restarting a stopped or disabled user-level `openclaw-gateway` systemd unit.

## Best Possible Solution

Define the Linux gateway service lifecycle contract so diagnostic commands stay non-mutating, helper recovery respects disabled or masked units, and only explicit lifecycle commands start disabled units with docs and regression coverage.

## Reproduction Assessment

Partially. The exact `openclaw status` sequence is not source-confirmed, but current main clearly shows doctor/readiness paths that can start or restart an installed stopped user-systemd gateway.

## Solution Assessment

Unclear. The maintainable direction is a lifecycle-policy decision that separates diagnostics from explicit start/install/restart operations and then tests disabled or masked Linux systemd units.
