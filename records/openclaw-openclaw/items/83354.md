Keep open. Current main makes `openclaw status` look read-only, but adjacent helper paths still can start or restart an installed stopped Linux user-systemd gateway, so this needs maintainer lifecycle-policy review and live systemd proof rather than cleanup closure.

## What This Changes

The issue asks OpenClaw helper and diagnostic commands to avoid restarting a stopped and disabled Linux user-systemd gateway unit without explicit operator intent.

## Best Possible Solution

Define the Linux gateway service lifecycle contract so read-only diagnostics remain non-mutating, helper recovery respects disabled or externally managed units, and only explicit lifecycle commands start disabled services with docs, tests, and live systemd proof.

## Solution Assessment

Unclear. The safer direction is to separate read-only diagnostics from explicit lifecycle recovery and teach Linux systemd handling to respect disabled, masked, or externally managed units before adding a new flag or changing defaults.
