## What This Changes

The PR changes inferParamBFromIdOrName to use a non-consuming trailing boundary and adds regression tests for adjacent B-size tokens separated by one delimiter.

## Best Possible Solution

Land the narrow shared-helper regex fix with the adjacent-token regression cases, keeping model-size inference centralized in src/shared/model-param-b.ts.

## Reproduction Assessment

Yes at source level: the current regex returns the first adjacent token for single-delimiter pairs, and the same focused probe shows the lookahead returns the largest token while preserving existing negative and multi-delimiter cases.

## Solution Assessment

Yes. Changing only the trailing boundary to a lookahead is the narrowest maintainable fix; broader parsing or caller-specific filtering would duplicate behavior outside the shared helper.
