## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Fail closed on unreadable command metadata:** `src/gateway/plugin-node-command-policy-registrations.ts:47-48`
  - body: When `entry.command` throws, this helper returns `null`, and both dangerous-command callers then treat that row as absent. If that unreadable row is the only registration marking the requested command dangerous, `applyPluginNodeInvokePolicy` can return `null` and allow the raw node.invoke path to continue; distinguish unreadable metadata from no match and return a controlled fail-closed result instead.
  - confidence: 0.86
- **[P1] Preserve the original policy receiver:** `src/gateway/node-invoke-plugin-policy.ts:167`
  - body: Current main invokes `entry.policy.handle(...)`, so method-style handlers receive the original policy object as `this`. This patch copies the unbound handle into a snapshot and calls `policy.handle(...)`, rebinding `this` to the snapshot and breaking policies that keep state on the policy object; store the original policy or call the handle with the original receiver.
  - confidence: 0.88
