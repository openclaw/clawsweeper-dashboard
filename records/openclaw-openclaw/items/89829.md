- **[P1] Preserve the original policy receiver:** `src/gateway/node-invoke-plugin-policy.ts:167`
  - body: The reader copies `policy.handle` onto a new snapshot object, so `policy.handle(...)` no longer invokes the method with the original `entry.policy` receiver. Current main calls `entry.policy.handle(...)`; method-style policies that read `this` will break, so keep the original policy object or call the copied handle with `.call(originalPolicy, ctx)`.
  - confidence: 0.9
- **[P1] Fail closed when dangerous metadata is unreadable:** `src/gateway/node-invoke-plugin-policy.ts:43-46`
  - body: The new lookup maps unreadable host-command rows to `null` and then treats them as no dangerous command. If an operator allowlists a command whose dangerous plugin registration cannot be inspected, `applyPluginNodeInvokePolicy` returns `null` and the raw node invoke can proceed without the required policy; carry an unreadable/blocked state instead of skipping it at this security boundary.
  - confidence: 0.86
