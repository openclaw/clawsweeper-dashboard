## Summary

## What This Changes

This PR adds eager creation of `skills` and `.agents/skills` inside `ensureAgentWorkspace` so sandbox read-only skill mounts have host sources before first container launch.

## Best Possible Solution

Create the host `skills` and `.agents/skills` directories in the writable sandbox setup path before mount resolution, preserve skip-bootstrap wipe detection, and prove the real sandbox path.

## Reproduction Assessment

Yes from source inspection. The `rw` sandbox setup path only creates the workspace root before mount resolution, and the added `.agents/skills` directory is visible to the existing skip-bootstrap content-evidence scanner.

## Solution Assessment

No. Adding the directories in `ensureAgentWorkspace` is useful for some setup paths but is not the narrowest safe fix for the container launch path and it regresses workspace wipe detection.
