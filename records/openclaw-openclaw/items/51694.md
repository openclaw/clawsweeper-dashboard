## Summary

Close as superseded. Current main shows `appPrincipal` is intentionally optional for valid Google Chat paths, while the remaining conditional Add-ons `appPrincipal` docs/setup gap is already tracked by the broader open Google Chat Add-ons issue.

## What This Changes

The issue asks to resolve a Google Chat config/docs mismatch where `channels.googlechat.accounts.*.appPrincipal` appears optional even though Add-ons app-url webhooks fail without a principal binding.

## Best Possible Solution

Follow the open canonical Add-ons issue for conditional appPrincipal documentation/setup and parser/auth polish, while keeping the config field optional for valid non-Add-ons Google Chat paths.

## Reproduction Assessment

Yes. Source inspection and existing tests show the conditional path: Add-ons app-url tokens without `expectedAddOnPrincipal` return `missing add-on principal binding`, while normal Chat app-url tokens can pass without it.

## Solution Assessment

Yes. Closing this as superseded is the cleanest maintainer outcome because making the field globally required was rejected, and the remaining conditional Add-ons docs/setup work is already centralized in https://github.com/openclaw/openclaw/issues/65326.
