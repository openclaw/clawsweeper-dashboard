## Summary

Keep open. Current main still has a Google Chat config/docs contract mismatch: `appPrincipal` and `channels.googlechat.accounts` remain optional in schema, exported types, generated metadata, and docs, while the app-url add-on token path rejects requests without a configured principal binding; the earlier blanket-required PR is closed unmerged.

## What This Changes

The issue asks OpenClaw to reconcile Google Chat `appPrincipal` config metadata/docs with the runtime requirement for app-url add-on authentication.

## Best Possible Solution

Make the Google Chat config contract, setup/status guidance, generated metadata, and docs clearly require `appPrincipal` only for app-url add-on token delivery while preserving project-number compatibility.

## Reproduction Assessment

Yes. Source and existing unit tests provide a high-confidence path: app-url add-on token verification without `expectedAddOnPrincipal` returns `missing add-on principal binding`, while the schema/types/docs still allow omitting it.

## Solution Assessment

Yes. The narrow maintainable solution is conditional validation or clear setup/status/docs guidance for the app-url add-on path, not making `appPrincipal` unconditionally required for every Google Chat account.
