## Summary

Keep open. Current main still has the Google Chat config contract mismatch: `appPrincipal` and `channels.googlechat.accounts` remain optional in schema/types/generated metadata/docs, while the app-url add-on auth path still passes the optional value into verification and rejects add-on tokens when no principal binding is configured. The prior related fix PR #51817 is closed unmerged, so no merged implementation supersedes this issue.

## What This Changes

Request to align Google Chat `appPrincipal` config docs/schema with runtime auth behavior because app-url add-on webhook verification fails when `appPrincipal` is omitted.

## Best Possible Solution

Make the Google Chat config contract and docs match the app-url add-on runtime deliberately: require or clearly validate `appPrincipal` only for the app-url add-on token path, preserve the project-number path, update generated config metadata/docs hashes, and add focused tests for both missing app-url principal and project-number compatibility.

## Reproduction Assessment

Yes. The source and existing unit tests provide a high-confidence reproduction path: omit `expectedAddOnPrincipal` for an app-url add-on token and `verifyGoogleChatRequest` returns `missing add-on principal binding`, while config schema/types still allow that omission.

## Solution Assessment

Yes, with a narrower direction than the closed blanket-required PR. The maintainable fix is conditional validation/docs/setup for the app-url add-on path, not making `appPrincipal` unconditionally required for every Google Chat account.
