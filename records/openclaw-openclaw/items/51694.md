## Summary

Keep open. Current main shows the title is too broad because `appPrincipal` is not universally required, but app-url add-on token delivery still fails without it while the Google Chat docs and config surface do not explain that conditional requirement.

## What This Changes

The issue asks OpenClaw to align Google Chat config and docs with the runtime requirement for `channels.googlechat.accounts.*.appPrincipal`.

## Best Possible Solution

Document and validate the add-on-token requirement narrowly: require or warn for `appPrincipal` only where app-url add-on delivery needs it, while preserving existing project-number and normal Chat issuer configs.

## Reproduction Assessment

Yes. Source and existing tests provide a high-confidence path: add-on app-url token verification without `expectedAddOnPrincipal` returns `missing add-on principal binding`, while schema/types/docs still allow omitting the field.

## Solution Assessment

Yes, with a narrower scope than the closed PR. The maintainable fix is conditional config/docs/setup guidance for add-on app-url delivery, not making `appPrincipal` unconditionally required for every Google Chat account.
