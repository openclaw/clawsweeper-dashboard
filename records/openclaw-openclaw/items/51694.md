## Summary

Keep open. Current main confirms the original wording is too broad because `appPrincipal` is not globally required, but the remaining Google Chat add-on `app-url` path can still fail when the principal binding is omitted and the docs/setup flow do not make that conditional requirement clear.

## What This Changes

Align Google Chat config docs, schema/setup guidance, and diagnostics with the runtime `appPrincipal` requirement for add-on `app-url` token delivery.

## Best Possible Solution

Document and surface the conditional add-on-token requirement, ideally through setup guidance and focused validation or warnings, while preserving existing valid Google Chat auth modes.

## Reproduction Assessment

Yes. Source inspection and existing tests provide a high-confidence path: add-on `app-url` token verification without `expectedAddOnPrincipal` returns `missing add-on principal binding`, while schema/docs/setup still allow omitting it.

## Solution Assessment

Yes, but narrower than the closed proposed PR. The maintainable fix is conditional docs/setup/validation for add-on `app-url` delivery, not making `appPrincipal` mandatory for every Google Chat account.
