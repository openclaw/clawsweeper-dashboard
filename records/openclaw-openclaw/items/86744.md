## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Close the opened database before quarantine:** `src/tasks/task-registry.store.sqlite.ts:525`
  - body: If `new DatabaseSync(pathname)` succeeds and a PRAGMA or `ensureSchema(db)` throws, this catch block keeps that handle open while `quarantineCorruptedDatabase(pathname)` tries to rename the file. On Windows or locked filesystems the rename can fail, so startup still crashes instead of recovering; close the partially opened `db` before moving the corrupt database aside.
  - confidence: 0.9
- **[P2] Avoid partial quarantine when sidecar moves fail:** `src/tasks/task-registry.store.sqlite.ts:490-494`
  - body: This helper renames the main database before moving `-shm` and `-wal`, but it returns `null` if a later sidecar rename fails, leaving the main file quarantined and stale sidecars at the live path. Make the move all-or-nothing or handle rollback/cleanup before rethrowing, otherwise the next startup can see a fresh main database paired with old WAL/SHM files.
  - confidence: 0.78
