## Summary

Keep open. Current main still has thinking effort controls, reasoning visibility controls, provider replay sanitizers, display projection, and plugin hook workarounds, but it does not provide a schema-backed mode that lets reasoning run while excluding thinking blocks from durable session transcripts and transcript API/ACP surfaces.

## What This Changes

The issue requests a silent reasoning mode that decouples reasoning production from persistence of `thinking` blocks in session transcripts.

## Best Possible Solution

Define a core transcript-persistence policy separate from model thinking effort and reasoning visibility, with documented behavior for durable storage, transcript APIs, ACP, compaction, memory consumers, plugin hooks, and provider-required replay state.

## Reproduction Assessment

Yes, at source level. Current main has no hidden transcript-persistence mode, the append path persists supplied assistant messages, and `sessions.get` plus ACP loadSession can expose stored `thinking` blocks.

## Solution Assessment

No, the exact proposed strip-before-write shape is not sufficient by itself. The maintainable solution needs an explicit policy that separates durable transcript consumers from provider replay state that may still need opaque reasoning blocks.
