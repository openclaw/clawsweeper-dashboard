## Summary

Keep open. Current main has provider replay sanitizers, chat/history display projection, ACP thought replay, and a plugin hook workaround, but it still lacks a schema-backed policy that lets reasoning run while excluding thinking blocks from durable session transcript and transcript API surfaces.

## What This Changes

Add a silent reasoning mode that keeps model reasoning enabled while preventing thinking blocks from being persisted in session transcripts.

## Best Possible Solution

Define a core transcript-persistence policy separate from model thinking effort and channel reasoning visibility, documenting which transcript/API/memory surfaces discard thinking and which provider replay paths preserve required opaque state.

## Reproduction Assessment

Yes, at source level. Current config only supports thinking effort and reasoning visibility, while durable transcript write/read and ACP replay paths still operate on stored messages that can include thinking blocks.

## Solution Assessment

No, the exact proposed strip-before-write shape is not sufficient on its own. The maintainable solution needs an explicit policy covering durable storage, provider-required replay state, sessions.get/ACP, compaction, memory indexing, config, docs, and UI semantics.
