## Summary

Keep open. Current main has reasoning visibility controls and provider replay sanitization, but it still lacks a schema-backed mode that runs reasoning while excluding thinking blocks from durable session transcripts and transcript consumers.

## What This Changes

The issue requests a silent reasoning mode that keeps model thinking enabled while excluding thinking blocks from durable session transcripts and downstream memory/history surfaces.

## Best Possible Solution

Define a core transcript-persistence policy that is separate from model thinking effort and user/channel reasoning visibility. Durable transcript and memory-facing surfaces should be able to omit thinking blocks, while provider-required replay state remains available through a tested internal path.

## Reproduction Assessment

Yes. A code-level reproduction path is clear: persist an assistant message containing `type: "thinking"`, then read it through `sessions.get` or ACP load-session replay; current main has no mode that keeps reasoning enabled while discarding those durable blocks.

## Solution Assessment

Unclear. The requested direction is sound, but the issue’s strip-before-storage sketch is not yet the safest implementation because provider replay and prompt-cache preservation need a maintained internal path.
