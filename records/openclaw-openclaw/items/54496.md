## Summary

Keep open. Current main and the latest release still separate thinking effort from reasoning visibility, but they do not provide a schema-backed mode that lets reasoning run while excluding thinking blocks from durable session transcripts and transcript API consumers.

## What This Changes

Request a silent reasoning mode where model reasoning can run while thinking blocks are discarded from persisted session transcript/history surfaces.

## Best Possible Solution

Define a core transcript-persistence policy separate from model thinking effort and reasoning visibility, with explicit behavior for storage, provider replay, sessions.get, ACP loadSession, compaction, memory-facing consumers, docs, and compatibility tests.

## Reproduction Assessment

Yes, at source level. Current schema/docs only expose thinking effort and reasoning visibility, while transcript append, sessions.get, and ACP replay still preserve stored assistant thinking content.

## Solution Assessment

Unclear pending maintainer API choice. The requested direction is plausible, but the best fix is a cross-surface transcript-persistence contract rather than stripping thinking blocks at only one write site.
