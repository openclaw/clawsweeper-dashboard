## Summary

Keep open. Current main has replay/display sanitizers and a plugin hook workaround, but it still lacks a core, schema-backed policy for running reasoning while excluding thinking blocks from durable session transcripts and transcript API consumers.

## What This Changes

The issue requests a silent reasoning mode that keeps model reasoning enabled while preventing thinking content from being persisted in session transcripts.

## Best Possible Solution

Define and document a core transcript-persistence policy that is separate from thinking effort and reasoning visibility, with explicit rules for durable storage, public transcript APIs, memory/compaction consumers, and provider replay preservation.

## Reproduction Assessment

Yes, at source level. Current config only supports thinking effort and reasoning visibility, while durable transcript write/read and ACP replay paths can still operate on stored messages containing thinking blocks.

## Solution Assessment

No, the exact strip-before-write proposal is not sufficient by itself. The maintainable path needs an explicit policy covering storage, provider-required replay state, sessions.get, ACP loadSession, compaction, memory indexing, config, docs, and UI semantics.
