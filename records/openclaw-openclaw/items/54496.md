## Summary

Keep open. Current main still has provider replay sanitization and display/history projection, but not the requested schema-backed mode that keeps reasoning enabled while excluding thinking blocks from durable transcripts and transcript-facing APIs.

## What This Changes

Request a silent reasoning mode that keeps model reasoning enabled but excludes thinking blocks from durable session transcripts and transcript consumers.

## Best Possible Solution

Define a core transcript-persistence policy separate from model thinking effort and reasoning visibility, with documented behavior for durable session storage, `sessions.get`, ACP loadSession, compaction, memory-facing consumers, and provider-required replay state.

## Reproduction Assessment

Yes, at source level. Enable reasoning today and persist an assistant message with a `type: "thinking"` block; current config, write, `sessions.get`, and ACP replay paths have no mode that keeps reasoning production on while dropping that durable block.

## Solution Assessment

Unclear as proposed. The direction is valid, but the maintainable fix should be a schema-backed transcript-persistence contract rather than an unconditional strip before storage.
