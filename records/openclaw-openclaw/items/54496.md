## Summary

Keep open. Current main and the latest release still expose thinking effort and reasoning visibility controls, plus provider replay sanitizers, but there is no supported mode that lets reasoning run while excluding thinking blocks from durable session transcripts and transcript API consumers.

## What This Changes

Request to add a silent reasoning mode that lets models produce reasoning while omitting thinking blocks from durable session transcripts.

## Best Possible Solution

Define a core transcript-persistence policy separate from model thinking effort and user/channel reasoning visibility, with explicit semantics for provider replay state, sessions.get, ACP loadSession, compaction, memory-facing consumers, docs, and compatibility tests.

## Reproduction Assessment

Yes, at source level. Current main has no hidden transcript-persistence setting, and source plus tests show assistant thinking can still be stored and replayed through session transcript consumers.

## Solution Assessment

Unclear pending maintainer API choice. The requested direction is plausible, but the best fix is a defined transcript-persistence contract rather than only stripping thinking blocks at one write site.
