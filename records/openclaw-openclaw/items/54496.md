## Summary

Keep open. Current main still has separate thinking-effort and reasoning-visibility controls plus provider replay sanitizers, but it does not provide the requested durable transcript-persistence mode that lets reasoning run while excluding thinking blocks from stored session transcripts and transcript APIs.

## What This Changes

Requests a silent reasoning mode that preserves model reasoning quality while discarding thinking blocks before durable session transcript persistence.

## Best Possible Solution

Define a core transcript-persistence policy separate from model thinking effort and reasoning visibility, with explicit behavior for storage, provider replay, sessions.get, ACP loadSession, compaction, memory indexing, docs, and compatibility tests.

## Reproduction Assessment

Yes, at source level. Current schema, docs, transcript append, sessions.get, and ACP replay show thinking effort and reasoning visibility exist, while durable transcript persistence still has no hidden/discard mode.

## Solution Assessment

Unclear pending maintainer API choice. Decoupling reasoning production from durable persistence is the right problem boundary, but the implementation should not be a one-site strip unless provider replay and transcript consumer semantics are defined.
