## Summary

Keep open. Current main still has reasoning/thinking visibility controls and provider replay sanitizers, but it does not define a mode that lets reasoning run while omitting thinking blocks from durable session transcripts and transcript consumers.

## What This Changes

Request a silent reasoning mode that preserves model reasoning quality while excluding thinking blocks from stored session transcripts, compaction, and memory-facing history.

## Best Possible Solution

Define a core transcript persistence contract separate from thinking effort and reasoning display, then implement the chosen mode across session storage, session APIs, ACP replay, compaction, session-history tools, memory-facing consumers, docs, and compatibility tests.

## Reproduction Assessment

Yes, at source level. Current main has no hidden transcript-persistence setting, and source plus tests show stored assistant thinking can still flow through `sessions.get`, ACP replay, and session-history consumers.

## Solution Assessment

Unclear pending maintainer API choice. The requested direction is plausible, but the durable solution needs an explicit compatibility contract rather than only stripping blocks at one write site.
