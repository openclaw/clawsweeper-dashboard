## Summary

Keep open. Current main still lacks a schema-backed silent-reasoning persistence mode; existing thinking controls, reasoning visibility settings, provider replay sanitizers, and plugin hooks do not define whether assistant thinking blocks are discarded from durable session transcripts, sessions.get, ACP replay, compaction, or memory-facing history.

## What This Changes

Request a mode that lets models produce reasoning while excluding thinking blocks from durable session transcripts and downstream history or memory surfaces.

## Best Possible Solution

Define and document a core transcript-persistence policy separate from model thinking effort and channel reasoning visibility, including provider-required replay state and how session APIs, ACP, compaction, memory, config, and UI observe it.

## Reproduction Assessment

Yes, at source level. Current main has no hidden transcript-persistence mode, and stored assistant thinking blocks can still be returned by sessions.get and replayed by ACP.

## Solution Assessment

No, the literal strip-before-write proposal is not sufficient by itself. The maintainable solution needs an explicit persistence policy that discards user/history/memory-visible thinking while preserving provider-required replay state where needed.
