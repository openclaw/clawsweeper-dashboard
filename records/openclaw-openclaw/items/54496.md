## Summary

Keep open. Current main still has thinking effort controls, reasoning visibility controls, and provider replay sanitizers, but it lacks a schema-backed mode that lets reasoning run while excluding thinking blocks from durable transcripts and transcript-facing APIs.

## What This Changes

Request a silent reasoning mode that runs model thinking while omitting thinking blocks from the session transcript.

## Best Possible Solution

Define a first-class transcript-persistence policy separate from thinking effort and reasoning visibility, covering durable storage, transcript APIs, ACP replay, compaction, memory-facing consumers, and provider-required replay state.

## Reproduction Assessment

Yes, at source level. Current schema accepts thinking effort and reasoning visibility only, while exact assistant append stores supplied content and sessions.get/ACP can return or replay stored thinking blocks.

## Solution Assessment

Unclear as proposed. The direction is valid, but the maintainable fix should be a documented transcript-persistence contract rather than a bare hidden alias or unconditional thinking strip before storage.
