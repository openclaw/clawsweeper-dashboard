## Summary

Keep open. Current main has thinking-effort controls, reasoning-visibility controls, provider replay sanitizers, and memory/display cleanup paths, but it still does not provide a schema-backed silent-reasoning mode that runs reasoning while excluding thinking blocks from durable session transcripts and transcript-facing APIs.

## What This Changes

Request a silent reasoning mode that lets models produce reasoning while omitting thinking blocks from durable session transcripts and transcript consumers.

## Best Possible Solution

Define a core transcript-persistence policy separate from model thinking effort and reasoning visibility, with explicit storage, replay, session API, compaction, memory, UI/config, and docs semantics.

## Reproduction Assessment

Yes, source-level. Current main exposes thinking effort and reasoning visibility, while the assistant transcript append path stores the received message and transcript-facing APIs still surface stored thinking blocks.

## Solution Assessment

Unclear pending maintainer API choice. Decoupling reasoning production from durable persistence is the right boundary, but a one-site strip would be too narrow without provider replay and transcript-consumer semantics.
