## Summary

Keep open. Current main still lacks a schema-backed silent-reasoning transcript persistence policy; existing replay sanitizers, visibility controls, and plugin hooks do not stop stored assistant thinking blocks from being persisted or exposed through session APIs and ACP replay.

## What This Changes

Request a silent reasoning mode that lets models produce thinking while excluding thinking blocks from durable session transcripts and transcript consumers.

## Best Possible Solution

Define and document a core transcript-persistence policy separate from model thinking effort and channel reasoning visibility, including storage, session APIs, ACP replay, compaction, memory consumers, provider-required replay state, config, and UI semantics.

## Reproduction Assessment

Yes, at source level. Current main has no hidden transcript-persistence mode, and stored assistant thinking blocks can still be returned by sessions.get and replayed by ACP.

## Solution Assessment

No, the literal strip-before-write proposal is not sufficient by itself. The safer solution needs an explicit persistence policy that can discard user/history/memory-visible thinking while preserving provider-required replay state where needed.
