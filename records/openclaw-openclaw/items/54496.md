## Summary

Keep open. Current main separates reasoning visibility from thinking effort and has provider replay sanitizers, but it still lacks a schema-backed transcript-persistence mode that runs reasoning while excluding assistant thinking blocks from durable session transcripts and transcript APIs.

## What This Changes

Request to add a silent reasoning mode that preserves model reasoning capability while dropping thinking blocks from stored session transcripts and transcript consumers.

## Best Possible Solution

Define a core transcript-persistence policy separate from model thinking effort and reasoning visibility, with explicit storage, provider replay, sessions.get, ACP, compaction, memory, docs, and compatibility-test behavior.

## Reproduction Assessment

Yes, at source level. Current schema and docs expose thinking effort and reasoning visibility, while transcript append and transcript API readers still preserve stored assistant thinking content unless a separate provider-replay sanitizer runs before model calls.

## Solution Assessment

Unclear pending maintainer API choice. The requested direction is plausible, but the maintainable fix should be a cross-surface transcript-persistence contract rather than stripping thinking at only one write site.
