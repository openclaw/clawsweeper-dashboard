## Summary

Keep open. Current main has provider replay sanitizers and reasoning visibility controls, but it still does not provide a schema-backed silent-reasoning mode that lets reasoning run while excluding thinking blocks from durable session transcripts and transcript consumers.

## What This Changes

Request to add a silent reasoning transcript-persistence mode that decouples model reasoning production from storing or exposing thinking blocks in session history.

## Best Possible Solution

Define and document a core transcript-persistence policy separate from model thinking effort and reasoning visibility, covering storage, sessions.get, ACP loadSession, compaction, memory consumers, UI/config semantics, and provider-required replay state.

## Reproduction Assessment

Yes, at source level. Current main has no hidden transcript-persistence mode, and stored assistant thinking blocks can still be persisted, returned by sessions.get, and replayed through ACP.

## Solution Assessment

No, the literal strip-before-write proposal is not sufficient by itself. The safer solution needs an explicit persistence policy that can discard user/history/memory-visible thinking while preserving provider-required replay state where needed.
