## Summary

Keep open. Current main has reasoning visibility controls, sanitized recall views, and provider replay sanitizers, but still lacks a schema-backed mode that keeps reasoning enabled while excluding thinking blocks from durable transcripts and transcript-facing APIs.

## What This Changes

Request a silent reasoning mode that keeps model reasoning enabled while preventing thinking blocks from being persisted in session transcripts.

## Best Possible Solution

Define a documented transcript-persistence policy separate from model thinking effort and reasoning visibility, covering durable storage, `sessions.get`/ACP, compaction, memory-facing consumers, and provider-required replay state.

## Reproduction Assessment

Yes, at source level. A stored assistant message with a `type: "thinking"` block is appended without a discard policy, returned by `sessions.get`, and replayed by ACP as `agent_thought_chunk`, while current config has no mode that keeps reasoning on and drops durable thinking.

## Solution Assessment

Unclear as proposed. The direction is valid, but the maintainable fix should be a schema-backed transcript-persistence contract rather than a bare `hidden` alias or unconditional strip before storage.
