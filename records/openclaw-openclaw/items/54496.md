## Summary

Keep open. Current main has reasoning visibility controls, provider replay sanitizers, session-history display sanitizers, and a plugin hook workaround, but it still lacks a core mode that keeps reasoning enabled while excluding thinking blocks from durable session transcripts and transcript APIs.

## What This Changes

The issue requests a silent reasoning mode that produces model reasoning while preventing thinking blocks from being persisted in session transcripts.

## Best Possible Solution

Define a schema-backed transcript persistence policy that strips thinking from user/history/memory-facing durable surfaces while preserving provider-required replay state through a tested internal path.

## Reproduction Assessment

Yes, at source level. Enable reasoning today and persist an assistant message with a `type: "thinking"` block; current config, write, `sessions.get`, and ACP replay paths have no mode that keeps reasoning production on while dropping that durable block.

## Solution Assessment

Unclear as proposed. The direction is valid, but the best maintainable fix is not an unconditional strip before storage; it needs an explicit transcript-persistence contract with provider replay, ACP, compaction, memory, UI/config, and docs behavior defined.
