## Summary

Keep open. Current main has provider replay sanitizers, display sanitizers, and a plugin hook workaround, but it still lacks a schema-backed policy that lets reasoning run while excluding thinking blocks from durable session transcripts and transcript API consumers.

## What This Changes

Add a silent reasoning mode that runs model reasoning while excluding thinking blocks from durable session transcripts and transcript-facing consumers.

## Best Possible Solution

Define and document a core transcript-persistence policy separate from model thinking effort and user/channel reasoning visibility, including explicit rules for storage, transcript APIs, memory/compaction consumers, and provider-required replay state.

## Reproduction Assessment

Yes, at source level. Current config only supports thinking effort and reasoning visibility, while transcript write/read and ACP replay paths can still operate on stored assistant thinking blocks.

## Solution Assessment

No, the exact strip-before-write proposal is not sufficient by itself. The maintainable path needs an explicit policy covering durable storage, provider-required replay state, sessions.get, ACP loadSession, compaction, memory indexing, config, docs, and UI semantics.
