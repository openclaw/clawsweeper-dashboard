## Summary

Keep open. Current main has thinking-effort controls, reasoning visibility controls, provider replay sanitizers, and some sanitized history/tool surfaces, but it still lacks a schema-backed mode that lets reasoning run while excluding thinking blocks from durable session storage and raw transcript consumers.

## What This Changes

Add a hidden/silent reasoning mode that allows model reasoning while preventing `type: "thinking"` blocks from being written to durable session transcripts and transcript consumers.

## Best Possible Solution

Define a core transcript-persistence policy separate from model thinking effort and user/channel reasoning visibility. Durable transcript, history, memory, and public session surfaces should omit thinking blocks under the new policy, while provider-required replay state is preserved through an explicit internal path with tests and docs.
