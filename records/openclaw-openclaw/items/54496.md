## Summary

Current main has improvements around reasoning visibility, provider replay sanitization, and channel delivery cleanup, but the requested durable transcript policy is still not implemented. The remaining work needs maintainer API/product design across thinking production, display visibility, provider replay requirements, stored session history, ACP replay, memory consumers, compaction, and docs.

## What This Changes

The issue requests a silent reasoning mode that allows model thinking/reasoning to run while excluding thinking blocks from durable session transcripts and downstream transcript consumers.

## Best Possible Solution

Define and document a core transcript-persistence policy separate from model thinking effort and user/channel reasoning visibility. Durable transcript/history/memory-facing surfaces should be able to drop thinking blocks, while provider-required replay state is preserved through a tested internal path.

## Reproduction Assessment

Yes. A code-level reproduction path is clear: enable thinking/reasoning today, persist an assistant message containing `type: "thinking"`, then read it back through session transcript APIs or ACP load; current main has no schema-backed mode that keeps reasoning enabled while discarding those durable blocks.

## Solution Assessment

Unclear as an implementation because the issue proposes the right product direction but not enough cross-surface contract detail. The best fix is a maintainer-designed transcript-persistence policy, not a narrow unconditional strip before storage.
