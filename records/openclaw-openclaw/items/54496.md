## Summary

Keep open. Current main has thinking-effort controls, reasoning-visibility controls, transcript display sanitizers, and provider replay sanitizers, but it still does not define a durable silent-reasoning mode that lets reasoning run while excluding thinking blocks from stored session transcripts and transcript-facing APIs.

## What This Changes

Add a silent reasoning mode that lets models produce thinking while preventing thinking blocks from being persisted to session transcripts and transcript consumers.

## Best Possible Solution

Define a core transcript-persistence policy separate from thinking effort and reasoning visibility, then document how storage, replay, session APIs, compaction, memory plugins, and UI/config observe it.

## Reproduction Assessment

Yes, source-level. Current main exposes thinking effort and reasoning visibility, while the message_end persistence path still stores assistant messages as received and no hidden persistence mode exists.

## Solution Assessment

Unclear pending maintainer API choice. Decoupling reasoning production from durable persistence is the right boundary, but a one-site strip would be too narrow without provider replay and transcript consumer semantics.
