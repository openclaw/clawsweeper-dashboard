## Summary

Keep open. Current main still has thinking-effort controls, reasoning-visibility controls, and provider replay sanitizers, but it does not provide a durable transcript-persistence mode that lets reasoning run while excluding thinking blocks from stored session transcripts and transcript APIs.

## What This Changes

Request a silent reasoning mode that allows model reasoning while preventing thinking blocks from being written to the session transcript.

## Best Possible Solution

Define a core transcript-persistence policy separate from model thinking effort and user/channel reasoning visibility, with explicit storage, replay, session API, compaction, memory, docs, and compatibility behavior.

## Reproduction Assessment

Yes, source-level. Current main exposes thinking effort and reasoning visibility, while storage and transcript APIs still have no hidden/discard persistence mode.

## Solution Assessment

Unclear pending maintainer API choice. Decoupling reasoning production from durable persistence is the right problem boundary, but a one-site strip would not define provider replay and transcript consumer semantics safely.
