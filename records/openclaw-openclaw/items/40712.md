## Summary

Keep open: current main has partial restart recovery and durable final-delivery replay, but arbitrary in-flight assistant generation is still kept in process-local buffers and assistant partial transcript tails are not resumed. The related drain-manifest PR remains open and only covers a producer slice, so it does not supersede this umbrella feature.

## What This Changes

Request durable persistence, recovery, or user notification for assistant message generation interrupted by a gateway restart or crash.

## Best Possible Solution

Keep this as the focused umbrella for durable in-flight assistant generation recovery, then split implementation across checkpointing, restart marking, boot-time recovery, final-delivery replay, and channel-safe user notification semantics.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence current-main path: assistant deltas are buffered in memory, shutdown clears those buffers, and restart recovery rejects assistant partial transcript tails.

## Solution Assessment

Unclear. Durable checkpointing is the right problem, but the proposed JSONL status sketch needs maintainer design review against existing restart-aborted markers, pending-final-delivery replay, drain manifests, and channel UX.
