## Summary

Keep open: current main and v2026.5.20 still keep live assistant generation in process-local buffers, clear that state during shutdown, and intentionally fail assistant-tailed restart recovery instead of resuming or delivering it. Related restart-recovery trackers cover broader adjacent work, but this focused interrupted-generation request remains a useful open product/design item.

## What This Changes

The issue requests durable recovery for assistant responses interrupted by gateway restarts.

## Best Possible Solution

Define a durable in-flight generation checkpoint and recovery contract that composes with abortedLastRun, pendingFinalDelivery, restart sentinels, drain behavior, and channel-safe interruption UX, then split it into focused gateway and agent changes.

## Reproduction Assessment

Yes, for a source-level reproduction. Live assistant deltas are buffered in ChatRunState maps, shutdown clears that state, and the recovery tests expect assistant-tailed transcripts to fail rather than resume.

## Solution Assessment

Unclear. Durable persistence or interruption recovery is the right direction, but the JSONL status sketch needs maintainer review against current abortedLastRun, pendingFinalDelivery, restart-sentinel, idempotency, privacy, and channel UX semantics.
