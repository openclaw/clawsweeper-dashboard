## Summary

Keep open: current main still buffers arbitrary in-flight assistant deltas in process-local state and restart recovery still refuses assistant-tailed transcripts. Existing restart markers and pending-final-delivery replay are useful partial mitigations, but they do not implement durable in-flight generation recovery.

## What This Changes

Request durable persistence and recovery so active assistant generation can resume or complete after a gateway restart instead of losing partial or final output.

## Best Possible Solution

Keep this as the focused product/architecture tracker for durable in-flight generation recovery across checkpointing, restart marking, boot-time recovery, final delivery replay, and channel-safe interruption UX.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: live assistant deltas are merged into in-memory chatRunState maps, shutdown clears that state, and restart recovery marks assistant-tailed transcripts failed.

## Solution Assessment

Unclear. Durable persistence is the right direction, but the JSONL status sketch and related manifest approach need maintainer review against abortedLastRun, pendingFinalDelivery, boot-time recovery, and channel UX semantics.
