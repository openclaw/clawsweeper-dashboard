- `impact:message-loss`: The user-visible failure is an incomplete or missing channel reply after restart.

## Summary

Keep open: current main marks some interrupted runs and persists completed final-delivery payloads, but arbitrary assistant deltas are still process-local and assistant-tailed transcripts remain non-resumable, so mid-generation recovery is not implemented.

## What This Changes

Add durable recovery for in-progress assistant generation so gateway restarts do not drop partial or completed replies.

## Best Possible Solution

Define a durable in-flight generation checkpoint and recovery contract that composes with abortedLastRun, pendingFinalDelivery, restart drain, shutdown session_end hooks, and channel-safe interruption UX, then split it into focused gateway/agent changes.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: live assistant deltas are held in chatRunState maps, shutdown clears that state, and restart recovery fails assistant-tailed transcripts; I did not run a live restart in this read-only review.

## Solution Assessment

Unclear. Durable persistence or interruption recovery is the right direction, but the issue's JSONL status sketch needs maintainer review against current abortedLastRun, pendingFinalDelivery, restart-drain, session_end, and channel UX semantics.
