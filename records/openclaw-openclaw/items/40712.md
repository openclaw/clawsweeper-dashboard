## Summary

Keep open: current main still keeps arbitrary in-flight assistant deltas in process memory, clears that state during shutdown, and restart recovery explicitly fails assistant-tailed transcripts; the durable pending-final-delivery path only covers completed payloads after generation has finished.

## What This Changes

The issue requests durable persistence and recovery for in-flight assistant generation so gateway restarts do not lose partial responses or final delivery state.

## Best Possible Solution

Define a durable in-flight generation checkpoint and recovery contract that composes with abortedLastRun, pendingFinalDelivery, restart drain, shutdown session_end hooks, and channel-safe interruption UX, then split it into focused gateway/agent changes.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: live assistant deltas are held in chatRunState maps, shutdown clears that state, and restart recovery fails assistant-tailed transcripts. I did not run a live restart in this read-only review.

## Solution Assessment

Unclear. Durable persistence is the right direction, but the issue's JSONL status sketch needs maintainer review against the current abortedLastRun, pendingFinalDelivery, restart-drain, and channel UX semantics.
