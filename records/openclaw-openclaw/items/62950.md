## Summary

Keep open. Current main still drops mixed-content think/verbose levels, so the PR addresses a real bug, but the patch is not mergeable as-is because it restores the same flags that session persistence uses and the external PR still lacks after-fix real behavior proof.

## What This Changes

The PR captures parsed think and verbose levels before mixed-content inline directives are cleared and re-adds them for authorized senders.

## Best Possible Solution

Carry authorized mixed-content think/verbose values into only the current run, avoid session-default persistence, and add focused regression coverage for both run-level application and non-persistence.

## Reproduction Assessment

Yes. Source inspection of current main shows a normal chat body with real text plus `/think <level>` or `/verbose <level>` reaches `clearInlineDirectives`, loses the parsed level, and then falls back during run resolution; I did not execute a live run in this read-only review.

## Solution Assessment

No. Preserving the levels is the right direction, but restoring `hasThinkDirective` and `hasVerboseDirective` is not the best fix because those flags also drive session persistence.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep preserved inline levels out of session defaults:** `src/auto-reply/reply/get-reply-directives.ts:298-305`
  - body: This restores `hasThinkDirective` and `hasVerboseDirective` after clearing a mixed-content message. `applyInlineDirectiveOverrides` later calls `persistInlineDirectives`, which writes session defaults from those flags, but normal-chat inline directives are documented as per-message only. Preserve the run-level values without reusing the persistence flags, or make persistence skip this preserved mixed-inline case.
  - confidence: 0.9
