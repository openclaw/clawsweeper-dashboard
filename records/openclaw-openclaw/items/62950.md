## Summary

Keep open: current main still drops mixed-content inline `/think` and `/verbose` levels, but this PR restores them by setting persistence-driving flags, so a one-message override can leak into stored session defaults. The external PR also still lacks after-fix real behavior proof and has a failed node test check at the current head.

## What This Changes

The PR captures parsed `thinkLevel` and `verboseLevel` before mixed-content inline directive clearing and re-injects them for authorized senders in `resolveReplyDirectives`.

## Best Possible Solution

Use a separate non-persistent carrier for mixed-content think and verbose overrides, feed it into run-level resolution, keep session persistence limited to directive-only messages, and add regression coverage plus real setup proof.

## Reproduction Assessment

Yes. Source inspection shows current `main` clears parsed mixed inline `/think` and `/verbose` values before `resolvedThinkLevel` and `resolvedVerboseLevel` are calculated.

## Solution Assessment

No. Preserving the values is the right direction, but setting `hasThinkDirective` and `hasVerboseDirective` reuses fields that the persistence layer treats as session-default directives; a separate run-only path is safer.
