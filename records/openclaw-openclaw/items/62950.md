## Summary

Keep open: current main still drops mixed-content think/verbose levels, but this PR reuses directive flags that downstream code treats as session-persistence markers, so it needs a narrow fix before merge.

## What This Changes

The PR modifies `resolveReplyDirectives` to preserve parsed think and verbose levels after clearing inline directives in mixed-content messages.

## Best Possible Solution

Carry authorized mixed-content think/verbose levels into the current run while keeping normal-chat inline hints non-persistent, then add focused regression tests and a changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists through `sessions.send`/`chat.send` with a `thinking` value and non-command message text, which synthesizes `/think <level> <message>` and currently loses the level when mixed directives are cleared.

## Solution Assessment

No. Preserving the levels is the right direction, but setting `hasThinkDirective`/`hasVerboseDirective` back to true is not the best fix because those flags are also used for session-default persistence.
