## Summary

Keep open: current main still drops mixed-content inline think/verbose levels, but this PR is not merge-ready because it reattaches persistence-driving flags, lacks real behavior proof, and has a failed CI check.

## What This Changes

The PR captures think/verbose levels before clearing mixed-content inline directives and re-injects them for authorized senders in `resolveReplyDirectives`.

## Best Possible Solution

Preserve mixed-content think/verbose values as run-only overrides, keep session defaults unchanged unless the message is directive-only, and add regression coverage for both run application and non-persistence.

## Reproduction Assessment

Yes. Source inspection of current main shows an authorized mixed-content message with `/think <level>` or `/verbose <level>` reaches `clearInlineDirectives`, which drops those parsed values before run-level resolution.

## Solution Assessment

No. Preserving the values is the right direction, but this branch couples run-only values to session persistence flags; the safer solution keeps the current-run value separate from persisted defaults.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep mixed-inline think and verbose out of session defaults:** `src/auto-reply/reply/get-reply-directives.ts:298-305`
  - body: The patch restores `hasThinkDirective` and `hasVerboseDirective` after `clearInlineDirectives`. Those flags are later treated as session-persistence directives, so a mixed-content `/think high` or `/verbose full` can overwrite stored session defaults even though docs say inline directives apply only to that message. Preserve the values for current-run resolution without marking them as persistent directives, or add an explicit no-persist path.
  - confidence: 0.9
