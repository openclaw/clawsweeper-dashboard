## Summary

Keep this PR open, but do not merge it as-is: it targets a real directive-clearing bug, yet the patch turns mixed-content `/think` and `/verbose` values into persistence-driving flags, lacks after-fix real behavior proof, and still has a failed node test lane.

## What This Changes

The branch preserves parsed think and verbose levels after mixed-content inline directive clearing in `src/auto-reply/reply/get-reply-directives.ts`, guarded to authorized senders.

## Best Possible Solution

Preserve mixed-content think and verbose values as run-only overrides, leave session defaults unchanged unless the message is directive-only, and add regression coverage for both run application and non-persistence.

## Reproduction Assessment

Yes, by source inspection. On current main, an authorized mixed-content message with `/think <level>` or `/verbose <level>` reaches `clearInlineDirectives`, which drops those parsed values before run-level resolution.

## Solution Assessment

No. Preserving the values is the right direction, but this branch couples run-only values to session persistence flags; the safer solution keeps current-run values separate from stored defaults.
