## What This Changes

The PR re-injects parsed think and verbose levels after mixed-content inline directive clearing in `src/auto-reply/reply/get-reply-directives.ts`, gated to authorized senders.

## Best Possible Solution

Preserve mixed-content think and verbose values as current-run overrides, leave session defaults unchanged unless the message is directive-only or explicitly patched, and add regression coverage for run application and non-persistence.

## Reproduction Assessment

Yes, by source inspection for generic mixed inline directives: current main clears `/think` and `/verbose` values before run-level resolution when real message text remains. The webchat `p.thinking` path is partly covered on current main by `thinkingLevelOverride`, so the remaining bug boundary should be scoped carefully.

## Solution Assessment

No. Preserving the values is the right direction, but restoring `hasThinkDirective` and `hasVerboseDirective` couples run-only values to session persistence; a safer fix should keep run overrides separate from persistent directive flags.
