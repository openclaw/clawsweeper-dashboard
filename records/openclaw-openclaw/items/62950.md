## Summary

Keep this PR open. Current main still drops mixed-content think/verbose directive levels, but the branch restores them by setting persistence-driving flags, so it can change stored session defaults and still lacks after-fix real behavior proof.

## What This Changes

The PR re-injects parsed `/think` and `/verbose` levels after `clearInlineDirectives` clears mixed-content inline directive state in reply directive resolution.

## Best Possible Solution

Preserve mixed-content think and verbose values as current-run overrides, leave session defaults unchanged unless the message is directive-only, and add regression coverage for both run application and non-persistence.

## Reproduction Assessment

Yes, by source inspection. On current main, an authorized mixed-content message with `/think <level>` or `/verbose <level>` reaches `clearInlineDirectives`, which drops those parsed values before run-level resolution.

## Solution Assessment

No. Preserving the values is the right direction, but this branch couples run-only values to session persistence flags; the safer solution keeps current-run values separate from stored defaults.

## Review Findings

Overall confidence: 0.9

- **[P1] Keep mixed-inline think and verbose run-only:** `src/auto-reply/reply/get-reply-directives.ts:298-305`
