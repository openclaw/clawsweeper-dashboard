## Summary

This PR should stay open: it targets a real current-main directive bug, but the patch still turns per-message mixed inline think/verbose hints back into session-persistence directives and the external contributor has not provided real behavior proof.

## What This Changes

The PR preserves parsed `thinkLevel` and `verboseLevel` after mixed-content inline directives are cleared in `resolveReplyDirectives`.

## Best Possible Solution

Carry authorized mixed-content think/verbose values into only the current run, then add focused regression coverage proving the values survive clearing without changing session defaults.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a normal chat body containing real text plus `/think <level>` or `/verbose <level>` reaches mixed-inline clearing, loses the parsed level on current `main`, and then resolves the run from fallback values.

## Solution Assessment

No. Preserving the levels is the right narrow fix, but setting the directive flags back to true is not the best path because those flags also drive session persistence for directives that docs define as per-message hints.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep preserved inline levels out of session defaults:** `src/auto-reply/reply/get-reply-directives.ts:298-304`
  - body: This restores `hasThinkDirective` and `hasVerboseDirective` after clearing a mixed-content message. `applyInlineDirectiveOverrides` then calls `persistInlineDirectives`, which writes session defaults from those flags, but normal-chat inline directives are documented as per-message only. Preserve the run-level values without reusing the persistence flags, or make persistence skip this preserved mixed-inline case.
