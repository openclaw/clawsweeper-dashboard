## Summary

Keep open: the branch targets a real directive bug, but the current patch marks preserved inline values as directives again, which can write run-only `/think` or `/verbose` values into session defaults. It also lacks contributor real behavior proof and has a failed node test check.

## What This Changes

The branch preserves parsed think and verbose levels after mixed-content inline directive clearing in `src/auto-reply/reply/get-reply-directives.ts` for authorized senders.

## Best Possible Solution

Preserve mixed-content think and verbose as current-run overrides without setting persistence flags, keep directive-only persistence unchanged, and add regression coverage for run application plus non-persistence.

## Reproduction Assessment

Yes, by source inspection: current main clears `/think` and `/verbose` values when a mixed directive message still has real text, before the run-level values are resolved. I did not run a live channel repro in this read-only review.

## Solution Assessment

No. Preserving the values is the right direction, but restoring the persistence flags is not the best fix because those flags are used to write session defaults.
