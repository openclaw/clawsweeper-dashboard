## Summary

Keep open; the PR targets a plausible WebChat attachment crash, but it is not merge-ready because the proposed guard misses pasted image attachments and the required after-fix real behavior proof is absent.

## What This Changes

The PR adds a 4 MB size check to the web UI chat attachment support helper.

## Best Possible Solution

Apply one attachment-size policy across picker, drop, and paste before FileReader or JSON-RPC serialization, align the limit and user feedback with the gateway attachment contract, add focused regression coverage, and provide real browser proof.

## Reproduction Assessment

No high-confidence live reproduction was established in this read-only review. Source inspection does show the linked failure path: accepted files become data URLs, then base64 JSON-RPC payloads, then a full-frame `JSON.stringify` before WebSocket send.

## Solution Assessment

No. A client-side guard is a reasonable mitigation, but this patch is incomplete because pasted images bypass it and the hard 4 MB silent-skip policy is not reconciled with the existing 20 MB gateway attachment limit.
