## Summary

Keep open: this is a plausible focused bug fix, but the proposed patch does not cover the paste attachment ingress path and the PR still lacks inspectable real behavior proof.

## What This Changes

The PR adds a 4 MB WebChat attachment size constant and rejects oversized files through the shared attachment support predicate.

## Best Possible Solution

Apply one attachment-size or safety preflight before FileReader across picker, drop, and paste, align user feedback with the gateway attachment contract, and coordinate with the gateway base64-validator fix path.

## Reproduction Assessment

No high-confidence live current-main reproduction was established in this review. Source inspection does show the path from FileReader data URLs to inline base64 chat.send attachments and full WebSocket frame serialization, plus a direct paste-path bypass of the PR's new helper.

## Solution Assessment

No. A client-side preflight can be part of the fix, but this PR only covers picker/drop files and needs paste coverage plus an agreed limit and feedback behavior that fits the gateway attachment contract.
