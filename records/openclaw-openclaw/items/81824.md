## Summary

Keep this PR open: the branch is a plausible mitigation for the linked WebChat attachment crash, but it is incomplete because pasted image files still bypass the new predicate and the external PR proof gate is still missing.

## What This Changes

The PR adds a 4 MB max-size predicate to WebChat attachment support so oversized selected or dropped files are rejected before base64 serialization.

## Best Possible Solution

Apply one attachment-size policy before FileReader across picker, drop, and paste, align the limit and user feedback with the Gateway attachment contract, add focused regression coverage, and provide real browser proof.

## Reproduction Assessment

No high-confidence live reproduction was established in this read-only review. Source inspection does show the reported path: accepted files become data URLs, base64 JSON-RPC attachments, and a full JSON.stringify WebSocket frame.

## Solution Assessment

No. A client-side guard is a reasonable boundary, but this patch is incomplete because pasted images bypass it and the hard 4 MB silent-skip policy is not reconciled with the existing 20 MB Gateway attachment contract.
