## Summary

Keep open: the PR targets a real WebChat attachment failure, but the patch only adds a picker/drop helper limit, leaves the paste attachment path uncovered, and still lacks inspectable after-fix real behavior proof.

## What This Changes

The PR adds a 4 MiB WebChat attachment size constant and rejects files over that limit in `isSupportedChatAttachmentFile()`.

## Best Possible Solution

Apply one attachment-size and validation policy across picker, drop, and paste before expensive read/send work, align it with the gateway attachment contract, add focused regression coverage, and provide redacted real browser proof.

## Reproduction Assessment

No high-confidence live reproduction was established here. Source inspection does show the current large-attachment path and a definite paste-path bypass of the PR's helper gate.

## Solution Assessment

No. A client-side guard may be part of the fix, but this implementation is incomplete because pasted images bypass it and the 4 MiB silent cap is not reconciled with the larger gateway attachment contract.
