## Summary

Keep open: current main still rejects signed non-message Nextcloud Talk webhook payloads with HTTP 400 before the existing ignore path can run. The related fix PR was focused and had useful real webhook proof, but it was closed unmerged, and no open replacement was found.

## What This Changes

The issue asks for signed Nextcloud Talk webhook events that are not chat messages, such as file-share and bot lifecycle events, to be acknowledged instead of returned as invalid payloads.

## Best Possible Solution

Implement the focused parser split from the closed PR: parse signed JSON first, preserve 400 for malformed JSON and malformed message-shaped payloads, and acknowledge signed non-message Talk events without invoking the message handler.

## Reproduction Assessment

Yes, with source-backed confidence rather than a live Nextcloud run: a signed JSON Activity Streams payload outside the strict Person/Note/Collection message schema reaches decodeWebhookCreateMessage and returns HTTP 400 Invalid payload format on current main.

## Solution Assessment

Yes. The narrow maintainable fix is to distinguish malformed JSON and malformed supported message payloads from signed non-message Talk events after signature verification; the closed related PR followed that shape but did not merge.
