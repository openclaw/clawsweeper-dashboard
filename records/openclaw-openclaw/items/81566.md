## Summary

Keep open: current main and the latest release still reject signed Nextcloud Talk webhook payloads that are valid Activity Streams but not strict chat-message Notes, and the open linked PR is the active fix path rather than a reason to close before merge.

## What This Changes

The issue asks for signed Nextcloud Talk non-message webhook events to be acknowledged without bot dispatch while preserving errors for malformed or unauthenticated requests.

## Best Possible Solution

Complete and land https://github.com/openclaw/openclaw/pull/90637 or an equivalent narrow fix, then close this issue after the merged commit and release status are known.

## Reproduction Assessment

Yes at source level: a signed JSON payload whose Activity Streams object is not a Note fails the current schema and returns HTTP 400. I did not run a live Nextcloud server in this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is to authenticate and parse JSON first, ignore signed unsupported Activity Streams events, and keep 400/401 behavior for malformed or unauthenticated requests. The open linked PR follows that shape, so a duplicate fix PR is not the best next step.
