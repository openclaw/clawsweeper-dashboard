## Summary

Keep open: current main and v2026.6.6 still reject signed non-message Nextcloud Talk webhook envelopes through the strict Note-schema path, but an open draft PR now formally closes this issue and targets the exact parser/test boundary.

## What This Changes

The issue reports that signed Nextcloud Talk webhook payloads for non-message events such as file shares return HTTP 400 instead of being acknowledged without bot dispatch.

## Best Possible Solution

Review the linked draft fix PR to completion: accept signed valid non-message Activity Streams events with 200/no dispatch while preserving 400/401 for malformed or unauthenticated webhook requests.

## Reproduction Assessment

Yes. At source level, a signed valid JSON Activity Streams payload outside the strict chat-message Note schema reaches decodeWebhookCreateMessage(), fails parseWebhookPayload(), and returns HTTP 400; this review did not run a live Nextcloud server.

## Solution Assessment

Yes. The narrow maintainable fix is to split signed JSON envelope parsing from supported chat-message decoding, acknowledge valid non-message events without dispatch, and keep strict errors for malformed or unauthenticated requests.
