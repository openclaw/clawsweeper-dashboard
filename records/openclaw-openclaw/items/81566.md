## Summary

Keep open: current main and the latest release still send signed non-message Nextcloud Talk webhook payloads through the strict chat-message schema and return HTTP 400 on schema miss, while the exact linked fix PR was closed unmerged.

## What This Changes

The issue asks the Nextcloud Talk webhook handler to acknowledge signed non-message Activity Streams events instead of returning 400 Invalid payload format.

## Best Possible Solution

Add a plugin-local webhook classifier that parses signed JSON before strict message validation, acknowledges non-message Activity Streams variants without invoking onMessage, preserves 400 for malformed JSON and malformed supported message payloads, and coordinates tests with adjacent structured-body parsing work.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a signed valid JSON Talk Activity Streams payload outside the strict Note-message schema reaches decodeWebhookCreateMessage and returns HTTP 400 Invalid payload format; I did not run live Nextcloud in this read-only pass.

## Solution Assessment

Yes. The maintainable fix is to distinguish malformed JSON and malformed supported message payloads from signed non-message Talk events after signature verification, matching the closed unmerged fix shape without broadening into attachment or mention parsing.
