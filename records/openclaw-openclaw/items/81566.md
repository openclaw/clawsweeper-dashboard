## Summary

Keep open: current main and the latest release still reject signed Nextcloud Talk webhook payloads that do not match the strict chat-message Note schema, and the active fixing path is an open draft PR that closes this issue rather than a merged implementation.

## What This Changes

The issue requests that signed Nextcloud Talk non-message webhook events, such as file-share or setup Activity Streams events, be acknowledged without dispatch instead of returning HTTP 400.

## Best Possible Solution

Review and either land or close the active narrow PR that separates signed JSON parsing from supported chat-message decoding, then close this issue once that path is merged.

## Reproduction Assessment

Yes. Source inspection shows a signed non-message Activity Streams payload reaches signature verification, then fails the strict Note schema and returns HTTP 400 before the existing ignore branch can run; I did not run a live Nextcloud server in this read-only review.

## Solution Assessment

Yes. The narrow maintainable solution is to authenticate and parse JSON first, ignore signed Activity Streams events outside the supported chat-message shape without dispatch, and keep 400/401 behavior for malformed JSON, malformed supported Notes, and auth failures.
