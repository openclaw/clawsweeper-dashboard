## Summary

Keep open: current main and v2026.6.6 still validate signed Nextcloud Talk webhooks against the strict chat-message Note schema before ignoring unsupported events, so signed non-message Activity Streams payloads can still return 400. The focused fix is already represented by the open draft PR at https://github.com/openclaw/openclaw/pull/90637, so this issue should stay open until that PR lands or is replaced.

## What This Changes

The issue asks the Nextcloud Talk webhook handler to acknowledge signed non-message Activity Streams events such as file shares instead of returning 400 Invalid payload format.

## Best Possible Solution

Review and land the focused acknowledgement fix in https://github.com/openclaw/openclaw/pull/90637, preserving 400/401 behavior for malformed JSON and authentication failures while returning 200 for signed non-message Talk events.

## Reproduction Assessment

Yes at source level: a signed valid JSON Activity Streams payload outside the strict Note schema passes signature verification, fails `parseWebhookPayload()`, and returns HTTP 400 on current main. I did not run a live Nextcloud server in this read-only review.

## Solution Assessment

Yes: splitting signed JSON parsing from supported message-schema validation is the narrow maintainable fix. The current best path is to finish the linked PR rather than queue a duplicate repair.
