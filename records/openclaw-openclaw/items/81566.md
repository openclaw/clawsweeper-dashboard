## Summary

Keep open: current main and v2026.5.18 still parse every signed Nextcloud Talk webhook through the strict Note-message schema before the ignore path, so signed non-message hooks can still return HTTP 400. The focused fix PR was closed unmerged, and the remaining open related PR handles rich-object message parsing rather than this acknowledgment contract.

## What This Changes

The issue asks the Nextcloud Talk webhook server to acknowledge signed non-message Activity Streams hooks instead of returning 400 Invalid payload format.

## Best Possible Solution

Update the Nextcloud Talk webhook classifier so signed JSON is parsed after signature verification, malformed JSON and malformed supported message payloads still return 400, and signed non-message Talk events are acknowledged without invoking the message handler.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a signed valid JSON Activity Streams payload outside the strict Note-message schema reaches decodeWebhookCreateMessage and returns 400 Invalid payload format before the ignore branch.

## Solution Assessment

Yes. The narrow maintainable fix is to split signed JSON parsing from strict message validation, preserve 400 for malformed JSON or message-shaped invalid payloads, and acknowledge other signed Talk event variants.
