## Summary

Keep this open: current main still returns HTTP 400 for signed Nextcloud Talk webhook payloads outside the strict chat-message schema, and an open PR already references this issue with closing syntax.

## What This Changes

The issue asks the Nextcloud Talk plugin to acknowledge signed non-message Talk webhook events instead of returning HTTP 400 Invalid payload format.

## Best Possible Solution

Review and land the focused plugin fix in https://github.com/openclaw/openclaw/pull/81569, preserving strict handling for missing auth, invalid signatures, malformed JSON, and malformed supported message payloads.

## Reproduction Assessment

Yes, from source inspection: a signed JSON Talk webhook whose Activity Streams shape is outside the strict Create/Note chat-message schema reaches decodeWebhookCreateMessage and returns HTTP 400 Invalid payload format. I did not establish a live Nextcloud reproduction in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is to classify signed JSON before strict chat-message validation so benign non-message Talk events are acknowledged while malformed/auth-invalid requests still fail.
