## Summary

Keep open. Current main still routes signed non-message Nextcloud Talk webhook payloads through the strict chat-message schema and returns HTTP 400 on schema misses; the linked open fix PR specifically closes this issue, so the issue should stay open until that PR lands or is replaced.

## What This Changes

The issue requests that the Nextcloud Talk webhook acknowledge signed non-message Talk events instead of returning 400 Invalid payload format.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81569, or replace it with an equivalent focused fix, then close this issue once current main acknowledges signed non-message Talk events.

## Reproduction Assessment

Yes from source inspection: a signed JSON payload whose Activity Streams shape is not the strict Create/Note chat-message schema reaches decodeWebhookCreateMessage and returns HTTP 400 Invalid payload format. I did not run a live Nextcloud instance in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is to classify signed JSON after signature verification, acknowledge benign non-message Talk events, and keep malformed JSON, missing headers, invalid signatures, and malformed supported message payloads on error paths.
