## Summary

Keep open: current main fixes the original Mattermost `streaming` config acceptance/off-mode gap, but active Mattermost preview modes still finalize through edited draft posts and the final-notification/silent-preview UX remains unresolved.

## What This Changes

Request to make Mattermost honor documented `channels.mattermost.streaming` modes and improve preview notification behavior so final answers are not silently delivered as edits.

## Best Possible Solution

Keep the shipped config support, then define and implement Mattermost preview-mode semantics so `progress` and/or `block` provide a clear final-delivery policy without surprising notifications.

## Reproduction Assessment

Partly. Source inspection on current main shows active Mattermost previews finalize through a PUT edit to an existing post, but I did not run a live Mattermost client to prove notification behavior.

## Solution Assessment

Unclear as an automated fix. The config-stripping bug is fixed, but final notification, silent draft creation, or boundary-split behavior needs a Mattermost UX decision coordinated with the open follow-up PR.
