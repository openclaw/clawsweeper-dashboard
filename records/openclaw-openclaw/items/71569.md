## Summary

Keep this open. Current main still documents Mattermost `streaming` mode support while the Mattermost schema/type/metadata omit the config key and the inbound runtime still wires draft-preview callbacks unconditionally; open PR #73252 remains the active implementation candidate rather than a merged fix.

## What This Changes

The issue requests Mattermost support for the documented `channels.mattermost.streaming` modes and a resolution for draft-preview notification/visual-noise behavior.

## Best Possible Solution

Land or replace PR #73252 with Mattermost-local canonical `streaming.mode` schema/type/metadata/runtime support while preserving legacy `blockStreaming`, then decide final-notification or silent-preview behavior as a separate product choice.

## Reproduction Assessment

Yes for the documented config mismatch: current main advertises Mattermost `streaming` modes while the strict Mattermost schema/type/metadata omit the key and the runtime always wires the draft-preview path. The notification symptom has credible reporter evidence but no live reproduction in this review.

## Solution Assessment

Yes, the best maintainable direction is to reuse the existing channel streaming mode helpers in Mattermost and add focused regression coverage for `streaming.mode: "off"` and final delivery. A new `finalNotification` option should wait until the documented mode contract is honored.
