Full review comments:

- **[P1] Handle every payload entry before sending:** `extensions/feishu/src/send.ts:110`
  - body: `firstOpenClawEnvelopePayload` returns only `payloads[0]`, and the resolver never inspects the rest of the array. Please iterate/send every entry or fail fast when a multi-payload envelope is unsupported.
  - confidence: 0.93
- **[P1] Preserve media payloads in the envelope path:** `extensions/feishu/src/channel.ts:711`
  - body: This branch catches every `payloads` envelope before the existing media path runs, but `sendOpenClawEnvelopeFeishu` only builds text or interactive create-message bodies. A payload containing `mediaUrl`, `mediaUrls`, or `audioAsVoice` will bypass `feishuOutbound.sendMedia` and lose the attachment while reporting success, so media-bearing envelopes need to delegate to the existing media/sendPayload flow or be rejected explicitly.
  - confidence: 0.91
