## Summary

This PR targets a real Feishu payload-envelope bug that is still present on current main, but the proposed patch is not merge-ready because it can truncate payload envelopes and lacks required real Feishu behavior proof.

## What This Changes

The PR adds a Feishu serializer for OpenClaw outbound envelopes, routes `send` and `thread-reply` payload envelopes through it, and adds mocked Feishu serialization/action tests.

## Best Possible Solution

Implement the Feishu-owned payload-envelope path so it preserves all payload entries and media semantics, then land it with focused regression coverage and redacted live Feishu send proof.

## Reproduction Assessment

Yes. Current main is source-reproducible: Feishu `handleAction` has no `payloads` branch and rejects payload-envelope-only sends before calling the create-message API; I did not run a live Feishu account reproduction in this read-only review.

## Solution Assessment

No as proposed. Fixing this inside the Feishu plugin action/send path is the right boundary, but this patch must process or reject multiple payload entries, preserve media payloads, and include real behavior proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Process every payload entry before sending:** `extensions/feishu/src/send.ts:105-106`
  - body: The serializer only reads `payloads[0]`, so additional payload entries are silently ignored while the action can still return success. OpenClaw outbound payloads can carry multiple visible entries, so this should either send each entry or fail fast when more than one payload is present.
  - confidence: 0.91
- **[P1] Preserve media payloads in envelope sends:** `extensions/feishu/src/channel.ts:708`
  - body: The new `payloads` branch catches the envelope before the existing media path, but the serializer only maps text and interactive cards. A payload containing `mediaUrl` or `mediaUrls` can now send only text or error through the wrong path instead of preserving the attachment behavior.
  - confidence: 0.9
