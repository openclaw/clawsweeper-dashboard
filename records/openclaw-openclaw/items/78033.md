## Summary

Keep open: this PR targets a real Feishu bug, but it still needs contributor-provided real behavior proof and has blocking review findings for payload truncation, media dropping, and the missing changelog entry.

## What This Changes

The PR adds Feishu envelope serialization for OpenClaw outbound payloads and routes Feishu send/thread-reply actions with `payloads` through the new serializer.

## Best Possible Solution

Land a Feishu-owned action fix that serializes payload envelopes without dropping multi-payload or media content, adds the required changelog entry, and includes redacted live Feishu send proof.

## Reproduction Assessment

Yes, source-reproducible: current main Feishu `handleAction` has no `payloads` branch, so a payload-only send errors before reaching the create-message API. I did not run a live Feishu account reproduction in this read-only review.

## Solution Assessment

No as proposed: fixing this inside the Feishu plugin send/action path is the right boundary, but the current patch reads only `payloads[0]`, bypasses media handling for payload envelopes, and lacks changelog and real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Process every payload entry before sending:** `extensions/feishu/src/send.ts:105-106`
  - body: The new serializer only reads `payloads[0]`, so additional payload entries are silently ignored while the action returns success. OpenClaw outbound payloads can carry multiple visible entries; either send each entry or fail fast when more than one payload is present.
  - confidence: 0.91
- **[P1] Preserve media payloads in envelope sends:** `extensions/feishu/src/channel.ts:708`
  - body: The new `payloads` branch runs before the existing media path, but the serializer only maps text and interactive card content. A payload with `mediaUrl` or `mediaUrls` will now bypass Feishu media sending and can drop attachments without a clear failure.
  - confidence: 0.9
- **[P2] Add the required changelog entry:** `extensions/feishu/src/send.ts:69`
  - body: This is a user-visible Feishu send fix, but the PR does not update `CHANGELOG.md`. OpenClaw policy requires a changelog entry for user-facing fixes before merge.
  - confidence: 0.92
