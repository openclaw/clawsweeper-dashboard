## Summary

Keep open. Current main still renders live WebChat/Control UI code blocks as escaped plain code with copy chrome and JSON collapse, while syntax highlighting exists only in the separate transcript export template.

## What This Changes

Add syntax highlighting for WebChat/Control UI code blocks so language-tagged snippets render with token colors instead of plain monospace text.

## Best Possible Solution

Live WebChat/Control UI code blocks should get theme-aware token highlighting in the shared markdown renderer while preserving copy buttons, language labels, JSON collapse, sanitization, caching, and oversized fallback behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows WebChat/Control UI chat flows through toSanitizedMarkdownHtml, and both fenced and indented code paths emit escaped pre/code output without a highlighter call or token markup.

## Solution Assessment

Yes in direction, but the issue body's marked.setOptions sketch is outdated for current main. The narrow maintainable solution is to extend the existing markdown-it renderer and sanitizer path rather than switching renderer stacks.
