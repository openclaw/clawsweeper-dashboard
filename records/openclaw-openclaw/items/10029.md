## Summary

Keep open. Current main still lacks syntax highlighting in the live WebChat/Control UI markdown path: code blocks get existing chrome, language labels, copy buttons, and JSON collapse, but the code text is escaped directly into `<pre><code>` with no highlighter/token markup. The existing syntax-highlighting code is limited to transcript export HTML, so it does not satisfy the live WebChat request.

## What This Changes

Request to add syntax highlighting for code blocks in the WebChat/Control UI markdown renderer.

## Best Possible Solution

Implement highlighting in the shared Control UI markdown layer used by live WebChat, with theme-aware token styles and regression coverage for fenced and indented code, language labels, copy buttons, JSON collapse, sanitization, cache behavior, and oversized-message fallback.

## Reproduction Assessment

Yes. Rendering a fenced block such as ```cpp through the live chat path currently flows through `toSanitizedMarkdownHtml` and emits escaped `<pre><code class="language-cpp">` output without highlighted token markup.

## Solution Assessment

Yes for a renderer-layer fix. Transcript-export-only highlighting or backend rendering would not solve the live WebChat surface; the exact library should be chosen with bundle size, lazy behavior, sanitization, and theme fit in mind.
