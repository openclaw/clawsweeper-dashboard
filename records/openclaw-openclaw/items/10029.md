## Summary

Keep open. Current main still lacks live WebChat syntax highlighting: code blocks have copy chrome, language labels, and JSON collapse, but the shared Control UI markdown renderer emits escaped plain code with no highlighter or token markup.

## What This Changes

Add syntax highlighting for fenced and indented code blocks in the WebChat/Control UI chat markdown renderer.

## Best Possible Solution

Add theme-aware highlighting in the shared Control UI markdown renderer while preserving copy buttons, language labels, JSON collapse, sanitization, caching, and oversized-message fallback behavior.

## Reproduction Assessment

Yes for the observable missing feature. Source inspection on current main shows WebChat/Control UI chat flows through toSanitizedMarkdownHtml, and both fenced and indented code render escaped pre/code output without a highlighter call or token markup.

## Solution Assessment

Yes in direction: the narrow maintainable fix belongs in the shared Control UI markdown renderer, not the transcript export template. The implementation should use the current markdown-it renderer and sanitizer path rather than the issue body's older marked.setOptions sketch.
