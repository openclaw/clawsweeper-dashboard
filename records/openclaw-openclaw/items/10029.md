## Summary

Keep open. Current main still does not apply syntax highlighting in the live Control UI/WebChat markdown renderer; code blocks keep copy/header chrome and escaping, but there is no highlighter dependency, highlighter hook, or highlighted token markup on the live chat path.

## What This Changes

The issue requests syntax highlighting for fenced and indented code blocks in the WebChat/Control UI markdown renderer.

## Best Possible Solution

Add theme-aware, sanitizer-safe syntax highlighting in the Control UI markdown renderer while preserving copy buttons, language labels, JSON collapse, caching, raw copy behavior, and oversized-message fallback.

## Reproduction Assessment

Not applicable as a bug reproduction because this is an enhancement request. Source inspection gives a high-confidence current-state check: live WebChat code blocks render as escaped pre/code text without highlighted token markup.

## Solution Assessment

Yes. A focused Control UI markdown-renderer change, with shared styling and regression coverage, is the narrowest maintainable path; transcript/export-only or backend highlighting would not solve the live WebChat request.
