## Summary

Keep open. Re-checking current main at 50da306 shows the prior review is still accurate: live WebChat/Control UI code blocks have copy chrome, language labels, and JSON collapse, but no syntax highlighter or token markup.

## What This Changes

Requested syntax highlighting for code blocks in WebChat/Control UI markdown rendering.

## Best Possible Solution

Add theme-aware highlighting in the shared Control UI markdown renderer, preserving copy buttons, language labels, JSON collapse, sanitization, caching, and oversized-message fallback behavior.

## Reproduction Assessment

Yes for the observable missing feature. Source inspection on current main shows WebChat markdown flows through toSanitizedMarkdownHtml, and both fenced and indented code render escaped pre/code output without a highlighter call or token markup.

## Solution Assessment

Yes in direction: the narrow maintainable solution is renderer-layer highlighting in the shared Control UI markdown path, not the transcript export path. The exact library and language bundle should be chosen to preserve sanitization, copy behavior, theme fit, and bundle size.
