## Summary

Keep open. Current main still renders live WebChat/Control UI code blocks as escaped plain code with existing code-block chrome, copy buttons, language labels, and JSON collapse, but no syntax highlighter or token markup.

## What This Changes

Add syntax highlighting to WebChat/Control UI markdown code blocks.

## Best Possible Solution

Add theme-aware syntax highlighting in the shared Control UI markdown renderer used by WebChat, preserving copy buttons, language labels, JSON collapse, DOMPurify sanitization, caching, and large-message fallbacks.

## Reproduction Assessment

Yes for the observable missing feature. Source inspection on current main shows WebChat markdown goes through toSanitizedMarkdownHtml, and both fenced and indented code render escaped pre/code output without a highlighter call or token markup.

## Solution Assessment

Yes in direction: the narrow maintainable path is renderer-layer highlighting in the shared Control UI markdown code. The exact library and language bundle should be chosen to preserve sanitization, copy behavior, theme fit, and bundle size.
