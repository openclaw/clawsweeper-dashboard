## Summary

Keep open. Current main still lacks live WebChat syntax highlighting: the Control UI chat path renders Markdown through the shared renderer, and fenced/indented code blocks are emitted as escaped plain code with language labels, copy chrome, and JSON collapse but no highlighter or token markup.

## What This Changes

Add syntax highlighting for code blocks in WebChat/Control UI Markdown rendering.

## Best Possible Solution

Add theme-aware highlighting in the shared Control UI Markdown renderer while preserving copy buttons, language labels, JSON collapse, sanitization, caching, and oversized-message fallback behavior.

## Reproduction Assessment

Not applicable; this is a feature request, not a broken existing contract. Source inspection still gives high confidence that the observable gap exists because the live WebChat path renders escaped plain code without token markup.

## Solution Assessment

Yes in direction, but the issue body's marked.setOptions sketch is stale because current main uses markdown-it. The narrow maintainable path is to integrate highlighting into the existing shared renderer/helper and sanitizer flow.
