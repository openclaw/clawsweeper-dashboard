## Summary

Keep open. Current main still lacks live WebChat/Control UI syntax highlighting for code blocks; the existing highlighting code is limited to transcript export HTML, while the live markdown renderer emits escaped plain code with copy chrome and JSON collapse.

## What This Changes

Request to add syntax highlighting for WebChat/Control UI markdown code blocks.

## Best Possible Solution

Add theme-aware token highlighting in the shared markdown-it renderer while preserving copy buttons, language labels, JSON collapse, sanitization, caching, and oversized fallback behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows WebChat/Control UI chat flows through toSanitizedMarkdownHtml, and both fenced and indented code paths emit escaped pre/code output without a highlighter call or token markup.

## Solution Assessment

Yes in direction, but the issue body's marked.setOptions sketch is outdated for current main. The narrow maintainable solution is to extend the existing markdown-it renderer and sanitizer path rather than switching renderer stacks.
