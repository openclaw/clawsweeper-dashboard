## Summary

Keep open. Current main still renders WebChat/Control UI code blocks through the shared markdown renderer as escaped plain code with copy/JSON chrome, with no live syntax highlighter or token markup.

## What This Changes

Add syntax highlighting to WebChat/Control UI Markdown code blocks so fenced languages like C++ render with theme-aware tokens instead of plain monospace text.

## Best Possible Solution

Add theme-aware highlighting in the shared Control UI markdown renderer while preserving copy buttons, language labels, JSON collapse, sanitization, cache behavior, and oversized-message fallback.

## Reproduction Assessment

Yes for the observable missing feature. Source inspection on current main shows WebChat/Control UI messages call toSanitizedMarkdownHtml, and both fenced and indented code render escaped pre/code without token markup or a highlighter call.

## Solution Assessment

Yes in direction: the maintainable fix belongs in the shared Control UI markdown renderer. The issue body's marked.setOptions sketch is outdated because current main uses markdown-it, so a PR should adapt the current renderer and sanitizer path.
