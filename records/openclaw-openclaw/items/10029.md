## Summary

Keep open. Current main still renders live WebChat/Control UI code blocks as escaped plain code with code-block chrome, copy buttons, language labels, and JSON collapse, but no syntax highlighter or token markup.

## What This Changes

The issue requests syntax highlighting for fenced and indented code blocks in the WebChat/Control UI markdown renderer.

## Best Possible Solution

Add theme-aware syntax highlighting in the shared Control UI markdown renderer used by live WebChat, with regression coverage that preserves existing code-block chrome and sanitization.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: WebChat markdown flows through toSanitizedMarkdownHtml, and fenced or indented code is escaped directly into pre/code output with no highlighter call.

## Solution Assessment

Yes for a renderer-layer implementation. Transcript-export-only highlighting or backend rendering would miss the live WebChat surface; the exact highlighter should be chosen with sanitization, lazy loading, theme fit, and bundle size in mind.
