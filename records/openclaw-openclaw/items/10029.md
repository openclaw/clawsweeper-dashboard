## Summary

Keep open. Current main and the latest shipped release still route live WebChat markdown through the Control UI renderer that escapes fenced and indented code as plain `<pre><code>` content, while existing syntax-highlighting code is limited to transcript export or the separate markdown preview surface.

## What This Changes

Add syntax highlighting to WebChat/Control UI markdown code blocks.

## Best Possible Solution

Implement highlighting in the shared Control UI markdown rendering layer with theme-aware styles and regression coverage for fenced code, indented code, copy buttons, JSON collapse, sanitization, caching, and oversized fallback behavior.

## Reproduction Assessment

Not applicable as a bug repro; this is a feature request. Source inspection gives a high-confidence confirmation path: WebChat calls `toSanitizedMarkdownHtml`, whose code-block renderers currently emit escaped plain code without highlighting.

## Solution Assessment

Yes, a focused renderer-level change is the right direction. Reusing the existing Control UI markdown path is narrower than adding a separate WebChat-only renderer, but the exact highlighter choice should preserve sanitization and performance constraints.
