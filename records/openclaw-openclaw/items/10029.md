Keep open. Current main and v2026.5.12 still render live WebChat code blocks as escaped plain code with code-block chrome, while syntax highlighting exists only in the transcript export path.

## What This Changes

The issue requests syntax highlighting for fenced and indented code blocks in WebChat.

## Best Possible Solution

Add renderer-owned, theme-aware highlighting in ui/src/ui/markdown.ts with regression coverage for highlighting, copy buttons, language labels, JSON collapse, sanitization, cache behavior, and oversized fallback.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives a high-confidence current-main path: WebChat uses toSanitizedMarkdownHtml, and that renderer emits escaped plain code blocks without syntax highlighting.

## Solution Assessment

Yes. The maintainable path is a focused renderer-level MarkdownIt implementation; the issue's marked.setOptions sketch is outdated for current main, and transcript-export highlighting does not affect live WebChat.
