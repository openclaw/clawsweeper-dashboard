## Summary

Keep open. Current main still lacks syntax highlighting in the live WebChat/Control UI markdown path: code blocks get chrome, language labels, copy buttons, and JSON collapse, but the code text is escaped directly into `<pre><code>` with no highlighter/token markup. The existing highlighter code is limited to transcript export HTML and does not satisfy the live WebChat request.

## What This Changes

The issue requests language-aware syntax highlighting for code blocks rendered in WebChat/Control UI chat.

## Best Possible Solution

Implement highlighting in the shared Control UI markdown layer used by live WebChat, with theme-aware token styles and focused regression coverage for fenced/indented code, copy buttons, JSON collapse, sanitization, cache behavior, and oversized fallback.

## Reproduction Assessment

Yes. Rendering a fenced block such as ```cpp through the WebChat grouped-render path currently flows through `toSanitizedMarkdownHtml` and emits escaped `<pre><code class="language-cpp">` output without highlighted token markup.

## Solution Assessment

Yes for a renderer-layer fix. Transcript-export-only highlighting or backend rendering would not solve the live WebChat surface; the exact highlighter should be chosen with sanitization, lazy loading, theme fit, and bundle size in mind.
