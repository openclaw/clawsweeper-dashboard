## Summary

Keep open. Current main still renders WebChat/Control UI code blocks as escaped plain code with code-block chrome, while syntax highlighting is only present in the separate session-export HTML path.

## What This Changes

Add syntax highlighting to WebChat/Control UI Markdown code blocks.

## Best Possible Solution

Add highlighting in the Control UI markdown rendering layer with theme-aware token styles and regression coverage for fenced/indented blocks, copy buttons, language labels, JSON collapse, sanitization, cache behavior, and oversized fallback.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives a high-confidence current-state check: WebChat uses `toSanitizedMarkdownHtml`, whose code-block renderers escape plain text and do not emit highlighted token markup.

## Solution Assessment

Yes. A renderer-level Control UI implementation is the narrowest maintainable direction; backend highlighting or unrelated WebChat rewrites are unnecessary, and the existing export-only highlighter should not be mistaken for live UI support.
