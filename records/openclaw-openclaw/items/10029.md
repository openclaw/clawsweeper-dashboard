## Summary

Keep open. Current main still renders WebChat code blocks as escaped plain text with code-block chrome, while the only syntax highlighter found is in the separate session-export HTML path.

## What This Changes

Request syntax highlighting for WebChat/Control UI markdown code blocks.

## Best Possible Solution

Add renderer-level highlighting in the Control UI markdown layer with theme-aware token styles and regression coverage for code-block chrome, copy text, JSON collapse, sanitization, caching, and oversized fallback.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives a high-confidence current-state check: WebChat uses toSanitizedMarkdownHtml, whose code-block renderers escape plain text and do not emit highlighted token markup.

## Solution Assessment

Yes. A small Control UI markdown-renderer implementation is the narrowest maintainable direction; backend highlighting or a broader WebChat rewrite is unnecessary.
