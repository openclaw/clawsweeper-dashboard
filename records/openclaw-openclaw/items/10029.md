## Summary

Keep open. Current `main` still renders live WebChat code blocks as escaped plain code with copy/JSON chrome, and the existing `highlight.js` usage is limited to exported transcript HTML rather than the Control UI markdown renderer.

## What This Changes

Request syntax highlighting for WebChat markdown code blocks in the Control UI renderer with styling that fits OpenClaw.

## Best Possible Solution

Implement highlighting in the Control UI markdown renderer or a small owned helper, with theme-aware styles and regression coverage for copy buttons, language labels, JSON collapse, sanitization, cache behavior, and oversized-message fallback.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence that live WebChat currently emits unhighlighted escaped code blocks through the Control UI markdown renderer.

## Solution Assessment

Yes. A focused renderer-level implementation is the narrow maintainable path; transcript export highlighting can be used as reference material, but it does not solve live WebChat rendering.
