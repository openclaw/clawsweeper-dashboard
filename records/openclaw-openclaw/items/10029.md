## Summary

Keep open. Current main and the latest release still render WebChat code fences as escaped plain code with copy/JSON chrome, while the existing highlight.js integration is limited to transcript export rather than the live Control UI renderer.

## What This Changes

Add syntax highlighting for fenced and indented code blocks in WebChat while preserving existing code-block chrome.

## Best Possible Solution

Implement highlighting in the Control UI markdown layer or a small helper used by it, with theme-aware styles and regression coverage for copy buttons, language labels, JSON collapse, sanitization, cache behavior, and oversized-message fallback.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The missing behavior is high-confidence from source: current main's WebChat renderer escapes code content and emits no highlighted token markup.

## Solution Assessment

Yes. A focused renderer-level implementation is the narrow maintainable path; reusing or adapting existing highlighting concepts is fine, but transcript export code should not be treated as the live WebChat solution.
