## Summary

Keep open. Current main still renders live WebChat code blocks as escaped plain code with copy-button chrome, while the only existing syntax highlighter is confined to session-export HTML and no open implementation PR appears to own this exact enhancement.

## What This Changes

Add syntax highlighting for fenced and indented code blocks rendered in WebChat/Control UI chat messages.

## Best Possible Solution

Add renderer-level highlighting in the Control UI markdown layer with theme-aware token styles and regression coverage for code-block chrome, copy text, JSON collapse, sanitization, caching, and oversized fallback.

## Reproduction Assessment

Not applicable as a bug reproduction because this is an enhancement request. Source inspection gives a high-confidence current-state check: WebChat uses `toSanitizedMarkdownHtml`, whose code-block renderers escape plain text and do not emit highlighted token markup.

## Solution Assessment

Yes. A small Control UI markdown-renderer implementation is the narrowest maintainable direction; backend highlighting or a broader WebChat rewrite is unnecessary.
