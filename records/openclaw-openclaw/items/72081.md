## Summary

Keep open. Current main still omits the safe manual-auth recovery hint when dashboard token delivery fails, and #72802 remains the open closing PR for the narrow fix.

## What This Changes

The issue reports that `openclaw dashboard` prints a dashboard URL without an accessible gateway token, leaving WSL/headless users unable to authenticate.

## Best Possible Solution

Land or revise #72802 so failed clipboard/browser delivery shows a redaction-preserving recovery hint that names the safe token sources without logging the token value or `#token=`.

## Reproduction Assessment

Yes. Configure a literal `gateway.auth.token`, make clipboard copy fail and browser opening unavailable, then inspect `dashboardCommand` logs; the current code emits the clean URL plus SSH/no-open fallback without a manual token-source hint.

## Solution Assessment

No, not by printing the tokenized URL. The narrow maintainable fix is the #72802-style recovery hint because existing security tests intentionally forbid logging token values or the `#token=` fragment marker.
