## Summary

Keep open. Current main still leaves the dashboard headless/failed-delivery path without a redaction-preserving token-source recovery hint, and open PR #72802 references this issue with closing syntax, so the issue should wait for that PR or an equivalent fix to land.

## What This Changes

The issue requests `openclaw dashboard` to provide a usable token-authenticated dashboard path when clipboard/browser token delivery fails in WSL or headless environments.

## Best Possible Solution

Review and land or revise PR #72802, or an equivalent patch, so failed clipboard/browser delivery shows token-source recovery guidance while runtime logs remain free of token values and `#token=` fragments.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure a literal `gateway.auth.token`, make clipboard and browser delivery fail, and current main logs the clean dashboard URL plus generic fallback text without manual token-source guidance.

## Solution Assessment

No, not as originally requested. Printing a tokenized URL in runtime logs would violate the existing security contract; the maintainable fix is a redaction-preserving hint that points to token sources without revealing the token.
