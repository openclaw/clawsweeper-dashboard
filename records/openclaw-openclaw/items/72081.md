## Summary

Keep open. Current main still logs the clean dashboard URL while sending the tokenized URL only to clipboard/browser, and the headless/WSL failure paths still fall back to an SSH hint or generic `--no-open` text without manual token-source guidance; the open closing PR should be reviewed or replaced before this issue is closed.

## What This Changes

The issue reports that `openclaw dashboard` prints a dashboard URL without a gateway token on WSL/headless setups and asks for a usable token-auth path.

## Best Possible Solution

Review and land or revise PR #72802, or an equivalent narrow fix, so failed clipboard/browser delivery shows redaction-preserving token-source guidance while token values and `#token=` stay out of runtime logs.

## Reproduction Assessment

Yes. Configure a literal `gateway.auth.token`, force clipboard copy and browser opening/support to fail, and current main logs the clean URL plus generic fallback text without manual token-source guidance.

## Solution Assessment

No, not as originally requested. Printing a tokenized URL in runtime logs would conflict with the existing security contract; the maintainable fix is a safe fallback hint that points to token sources without revealing the token.
