## Summary

This PR should stay open because it targets a real CLI model-ref normalization bug, but the current diff still puts non-read-only catalog discovery on the normal model-run dispatch path that current source and docs keep lean.

## What This Changes

## Best Possible Solution

Keep catalog-aware CLI-boundary canonicalization, but use a read-only or narrow lookup, or gate catalog work to refs that actually need case repair, so ordinary model smokes preserve the lean dispatch path.
