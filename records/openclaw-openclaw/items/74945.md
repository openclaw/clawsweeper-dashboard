## Summary

Keep open: current main still has the reported CLI model-case bug, but this PR is not merge-ready because the new canonicalization calls the non-read-only model catalog loader on ordinary explicit model runs, regressing the lean `skipPiDiscovery` probe path.

## What This Changes

The PR adds catalog-backed canonicalization of CLI `--model <provider/model>` input before local and gateway model-run dispatch, regression tests for mixed-case refs and auth-profile suffixes, and a changelog entry.

## Best Possible Solution

Keep the CLI-boundary canonicalization, but use a narrow or read-only lookup, or gate catalog work to refs that actually need case repair, so normal explicit model probes keep the lean path.

## Reproduction Assessment

Yes. The linked issue provides concrete CLI steps, and current-main source shows the raw `--model` value still reaches both local preparation and gateway override parsing.

## Solution Assessment

No, not as-is. Catalog-aware canonicalization is a reasonable fix direction, but the current patch performs full non-read-only catalog discovery before ordinary model-run dispatch instead of preserving the shipped lean probe path.
