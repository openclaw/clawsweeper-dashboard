## Summary

Keep open: current main still has the linked CLI model-ref bug, and this PR is useful, but the head patch still regresses the lean local model-run path by loading the full model catalog for ordinary explicit refs and it is missing the required changelog entry.

## What This Changes

The PR adds catalog-backed `--model` canonicalization for local and gateway model-run dispatch plus regression tests for case-mismatched refs, mixed-case catalog ids, auth-profile suffixes, and unknown custom refs.

## Best Possible Solution

Keep the pre-dispatch canonicalization, but gate catalog lookup to actual case-mismatched refs, preserve custom and auth-profile semantics, use existing provider/model normalization helpers where possible, and add an Unreleased Fixes changelog line.

## Reproduction Assessment

Yes. #73715 provides concrete CLI steps, and current main source shows the typed `--model` value still reaches both local preparation and gateway override parsing unchanged.

## Solution Assessment

No, not as-is. Catalog-aware canonicalization is the right direction, but calling full catalog discovery before ordinary lowercase local probes is broader than the existing lean model-run contract; a targeted case-mismatch lookup plus changelog is safer.
