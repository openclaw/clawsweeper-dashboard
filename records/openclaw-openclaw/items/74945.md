## Summary

Keep open: current main still has the raw CLI model-ref bug, but this PR is not ready because the latest fix still calls the full model catalog before every explicit model run, regressing the lean local probe path and widening provider-plugin catalog hook execution. The changelog gap has been addressed.

## What This Changes

The PR adds CLI `--model` catalog canonicalization before `capability model run` local/gateway dispatch, with regression tests and a changelog entry.

## Best Possible Solution

Keep catalog-aware CLI canonicalization for #73715, but make the lookup targeted or read-only so normal local probes preserve `skipPiDiscovery`, custom refs remain verbatim, and provider catalog hooks are not invoked unless needed.

## Reproduction Assessment

Yes from source, though I did not run a live provider call in this read-only review. Current main passes raw `params.model` into local preparation and gateway override parsing, and #73715 provides concrete CLI reproduction steps.

## Solution Assessment

No, not as-is. The branch's behavior and tests are the right direction, but full non-readOnly catalog loading before every explicit model run is broader than necessary and regresses the existing lean probe contract.

## Review Findings

Full review comments:

- **[P2] Avoid catalog discovery on normal model runs:** `src/cli/capability-cli.ts:697`
