## Summary

Keep open: the PR addresses a real CLI model-ref casing bug, but the latest diff still loads the non-read-only model catalog before ordinary explicit `infer model run --model` dispatch, regressing the lean probe path that current docs and recent fixes preserve.

## What This Changes

The PR adds `canonicalizeCliModelRef`, applies it before local and gateway model-run dispatch, and adds capability CLI regression tests plus a changelog entry.

## Best Possible Solution

Keep catalog-aware CLI-boundary canonicalization, but make it side-effect-light: return exact/lowercase refs without loading the full catalog, and only perform a narrow or read-only lookup for case-mismatch candidates that need canonicalization.

## Reproduction Assessment

Yes. The linked issue gives concrete CLI steps, and current main source inspection shows raw `--model` casing still reaches both local simple-completion preparation and gateway override parsing.

## Solution Assessment

No, not as-is. Catalog-aware canonicalization is the right direction, but the PR should preserve the lean `skipPiDiscovery` model-run contract by avoiding non-read-only catalog discovery for ordinary exact refs.
