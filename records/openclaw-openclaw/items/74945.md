## Summary

Keep open. The PR addresses the reported case-mismatched model-ref symptom and adds useful regression coverage, but the latest patch still puts full model-catalog discovery on every explicit provider/model run before the existing lean local dispatch path.

## What This Changes

The PR adds catalog-aware canonicalization of CLI `--model` refs for local and gateway `infer model run`, regression tests for mixed-case and auth-profile suffix behavior, and a changelog entry.

## Best Possible Solution

Keep the CLI-boundary canonicalization, but restrict catalog lookup to refs that actually need case canonicalization or use a narrow read-only lookup so canonical lowercase model runs stay on the lean `skipPiDiscovery` path.

## Reproduction Assessment

Yes. The linked issue gives concrete CLI steps, and current main source inspection shows the raw `--model` value still reaches both local preparation and gateway override parsing.

## Solution Assessment

No, not as-is. Catalog-aware canonicalization is a good fix direction, but the current PR loads the full model catalog before normal explicit runs instead of preserving the existing lean probe path.
