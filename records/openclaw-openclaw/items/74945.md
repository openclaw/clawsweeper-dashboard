## Summary

Keep open. The PR fixes the reported model-case path in the right general area, but it is not ready as-is because it loads the full model catalog before ordinary explicit local model probes and still needs the required changelog entry.

## What This Changes

The PR adds catalog-aware CLI model-ref canonicalization for `infer model run --model` across local and gateway dispatch plus regression tests.

## Best Possible Solution

Keep the CLI-boundary canonicalization, but make the catalog lookup narrow enough to preserve the lean local model-run path, keep auth-profile/custom-ref semantics, and add the missing changelog entry before landing.

## Reproduction Assessment

Yes. #73715 provides concrete CLI steps, and current main source inspection shows the raw `--model` value still reaches both local preparation and gateway override parsing.

## Solution Assessment

No, not as-is. Catalog-aware canonicalization is the right direction, but calling full catalog discovery before ordinary lowercase local probes regresses the existing lean model-run contract; a targeted case-mismatch lookup plus changelog is safer.
