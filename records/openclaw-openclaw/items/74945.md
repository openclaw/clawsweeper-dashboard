## Summary

Keep open: the linked bug is valid and this PR is pointed at the right CLI boundary, but the current patch still adds full non-read-only model catalog discovery to ordinary explicit model probes, which conflicts with the documented lean `model run` path.

## What This Changes

The PR adds catalog-backed CLI `--model` canonicalization for local and gateway model-run dispatch, regression tests, and a changelog entry.

## Best Possible Solution

Canonicalize only case-mismatched catalog refs with a read-only or narrowly gated lookup, while leaving canonical and custom explicit refs on the existing lean dispatch path.

## Reproduction Assessment

Yes. The linked issue gives concrete CLI steps, and current-main source shows the raw `--model` value reaches both local preparation and gateway override parsing.

## Solution Assessment

No, not as-is. Catalog-aware canonicalization is a good direction, but the PR should preserve the shipped lean probe path by avoiding full non-read-only catalog discovery for ordinary explicit refs.
