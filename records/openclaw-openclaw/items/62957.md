## Summary

Current main and release v2026.4.29 implement the 16k-specific request: the fixed 16k/32k context-window preflight was replaced by dynamic thresholds with 4k/8k safety floors, so sub-16k local models such as a 6k-window model warn instead of being blocked. A smaller 4k safety block remains intentionally documented and tested.

## What This Changes

Remove the fixed 16k context-window hard minimum so sub-16k models can be attempted with warning behavior instead of being rejected outright.

## Best Possible Solution

Keep the shipped dynamic context-window guard: it admits small local models below the former 16k floor, preserves warning-only behavior for 4k-8k windows, and retains a documented safety block for invalid or extremely tiny context windows.

## Reproduction Assessment

Yes. The observable blocker is reproducible through the guard tests: current main allows a 6,000-token model with a warning and only blocks below the 4,000-token hard floor.

## Solution Assessment

Yes. Replacing the fixed 16k/32k thresholds with dynamic 4k/8k floors is the narrowest maintainable fix because it solves the sub-16k block while preserving a deliberate preflight guard for unusably small or invalid windows.
