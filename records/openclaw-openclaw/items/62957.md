## Summary

Current main removes the fixed 16k model context-window hard block. The guard now uses dynamic thresholds with 4k/8k safety floors, runtime setup no longer passes the old fixed 16k/32k overrides, and tests/docs cover warning-only behavior for sub-16k models above the floor.

## What This Changes

The issue requests removing the fixed 16k model context-window hard limit so smaller models can be attempted, preferably with warning behavior instead of rejection.

## Best Possible Solution

Keep the dynamic context-window guard now on main, with warning behavior for small-but-usable models and a documented 4k safety floor. Open a narrower product-policy issue only if maintainers want to remove every hard floor.

## Reproduction Assessment

Yes. The current source and focused guard tests give a high-confidence reproduction path for the original policy: a 6,000-token model now warns without blocking, while only below-floor values block.

## Solution Assessment

Yes. Deriving thresholds from the effective model window and removing fixed runtime overrides is the narrow maintainable fix because it unblocks sub-16k experiments while preserving diagnostics and a minimum safety floor.
