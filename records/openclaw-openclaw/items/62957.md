## Summary

Current main no longer enforces a 16k hard context-window minimum: the guard now uses 4k/8k safety floors with percentage-based thresholds, runtime permits 6k models with warnings, custom-provider onboarding clamps to 4k, and docs/changelog describe the fixed-cutoff removal. This is implemented on main but not yet proven shipped in a release.

## What This Changes

Remove the 16k model context-window hard minimum and allow smaller models with warning behavior instead of blocking them.

## Best Possible Solution

Close as implemented on main and keep the current dynamic guard policy: warning behavior for small 4k+ contexts, with a separate issue if maintainers want to remove the remaining 4k safety floor.

## Reproduction Assessment

Not applicable as a feature/policy request. Source and tests give high-confidence current behavior: 6,000-token models now warn without blocking, while only below-floor or invalid windows block.

## Solution Assessment

Yes. The current implementation is a maintainable solution for the reported 16k block because it removes the fixed cutoff, scales thresholds from model metadata, keeps warning behavior for small usable contexts, and documents the remaining 4k safety floor.
