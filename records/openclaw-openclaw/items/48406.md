  - body: Current main registers safeguard compaction from `src/agents/pi-hooks`, but this branch threads `guardEnabled` and `escalationMode` through the older `pi-extensions` runtime path. Enabling the documented guard would not affect the active safeguard runtime after rebase.

  - confidence: 0.84

  - body: The added help says `compaction.guard` is reserved and has no runtime effect, while the same branch wires `guard.enabled` and `guard.escalation` into runtime behavior and the new guide describes the guard as implemented. Users would get contradictory guidance.
  - confidence: 0.91
- **[P2] Pass unresolved items into post-compaction validation:** `src/agents/pi-extensions/compaction-safeguard.ts:1117-1124`
  - body: The validator can only check missing pending items when `unresolvedItems` is present, and `compact.ts` passes `details.unresolvedItems`, but the safeguard details object records only the latest user goal. Live diagnostics would skip the pending-item check the tests claim to cover.
  - confidence: 0.82
- **[P2] Preserve non-Latin goals during validation:** `src/agents/post-compaction-validator.ts:180-184`
  - body: The validator strips every non-ASCII letter and treats an empty normalized source as already reflected. A latest goal written in Chinese, Japanese, or another non-Latin script can be omitted from the summary while validation still passes.
  - confidence: 0.91
