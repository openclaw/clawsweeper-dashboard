## Summary

Keep open: the patch is useful but too narrow because it can emit `totalTokens: 0` while the row still marks the total as not fresh, and the TUI still lacks the freshness/estimated fields needed to distinguish exact zero from stale or estimated usage.

## What This Changes

The PR changes `buildGatewaySessionRow` to accept zero-valued total token counts by swapping `resolvePositiveNumber` for `resolveNonNegativeNumber`.

## Best Possible Solution

Implement freshness-aware TUI token rendering that preserves valid fresh zero totals, carries `totalTokensFresh` and `contextBudgetStatus` into TUI session metadata, adds focused regression coverage, and includes real TUI proof.

## Reproduction Assessment

Yes, at source level: current main can hold context/window data without a fresh exact total, and the TUI footer deterministically formats that state through `formatTokens` as `tokens ?/<context>`. I did not run a live TUI session in this read-only review.

## Solution Assessment

No. The PR is a plausible narrow change, but the best fix should preserve the zero-token freshness invariant and carry the existing gateway/status freshness semantics into TUI rendering.
