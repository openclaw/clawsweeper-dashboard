## Summary

Keep open: current main still has gateway file-log tailing, session list/export, and structured trajectory capture, but no first-class human-readable live progress tail for active work. The related logs issue is adjacent file-log provenance work, not a duplicate of this operator progress surface.

## What This Changes

The issue requests a first-class human-readable live progress log or tail view over active session and trajectory events for heavy operators.

## Best Possible Solution

Add a narrow core CLI progress reader over existing session/trajectory data with safe redacted previews, documented behavior, and focused tests for multi-session follow, rotation, and disabled trajectory capture.

## Reproduction Assessment

Not applicable as a feature request. Source, docs, and focused test inspection confirm the current surfaces are gateway file logs plus session list/export, while the requested live human-readable trajectory tail is absent.

## Solution Assessment

Unclear until maintainers choose the public CLI/status surface and safe preview contract. Once that boundary is set, building on the existing trajectory recorder and sessions command area looks like the maintainable path.
