## Summary

Keep open: current main has structured trajectory capture plus session listing/export, but no first-class live, human-readable progress tail for active work. This is a useful operator-facing feature request, not an implemented-on-main or stale cleanup candidate.

## What This Changes

Request a CLI/status/TUI surface that renders active session trajectory and progress events in concise human-readable live lines instead of requiring manual tailing and JSONL parsing.

## Best Possible Solution

Add a narrow core CLI progress reader over existing session/trajectory data with safe redacted previews, documented behavior, and focused tests for multi-session follow, rotation, and disabled trajectory capture.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection confirm the current surfaces are gateway file logs and session list/export, while the requested live human-readable trajectory tail is absent.

## Solution Assessment

Unclear until maintainers choose the CLI/status surface and safe preview contract. Once that product boundary is set, building on the existing trajectory recorder and sessions command area looks like the maintainable path.
