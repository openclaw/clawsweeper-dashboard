## Summary

Keep open: current main has gateway file-log tailing, session listing/export, trajectory JSONL capture, and chat-visible progress drafts, but no first-class human-readable live progress tail for active session work. The request is a valid operator-facing feature that needs maintainer choice on command shape and safe redaction before implementation.

## What This Changes

Request a first-class human-readable live progress tail or view for active session trajectory events instead of manual JSONL tailing.

## Best Possible Solution

Add a narrow core CLI progress reader over existing session/trajectory run events with safe redacted summaries, multi-session follow behavior, docs/tests, and no regression to the existing gateway file-log tail.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection give high confidence the requested surface is absent: logs tail gateway file logs, sessions lists metadata/export, and progress drafts are chat-visible rather than an operator CLI aggregate.

## Solution Assessment

Unclear until maintainers choose the public command/status surface and redaction policy. Once that boundary is set, building on the existing trajectory recorder and compact progress formatter looks like the maintainable path.
