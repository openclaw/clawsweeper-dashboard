## Summary

Keep open: current main has structured trajectory capture plus session listing/export, but it does not provide a first-class live, human-readable progress tail for active work. The latest contributor comment usefully narrows a possible MVP, but maintainers still need to choose the public command shape and redaction contract.

## What This Changes

Request a human-readable live progress surface over session trajectory activity so operators can follow active sessions, tool calls, results, model completion, and run status without hand-parsing JSONL files.

## Best Possible Solution

Add a narrow CLI progress reader over existing trajectory/session data, with safe default redaction, multi-session follow behavior, disabled-capture handling, docs, and focused tests, while leaving broader Activity panel and watchdog work to their existing trackers.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and docs inspection give a high-confidence gap check: trajectory data exists, but current logs and sessions commands do not expose a live human-readable progress tail.

## Solution Assessment

Unclear until maintainers choose the public command surface and redaction contract. Once that is settled, building on the existing trajectory recorder plus the sessions command area is the narrowest maintainable path.
