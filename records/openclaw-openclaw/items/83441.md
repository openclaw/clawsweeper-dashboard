## Summary

Keep open: current main and the latest shipped tag still expose gateway file logs plus session listing/export, but not a first-class live, human-readable progress tail over active trajectory/session events. Related activity/progress issues and the draft dashboard PR are useful coordination points, not a shipped or canonical replacement for this specific CLI/operator surface.

## What This Changes

Request a first-class operator-facing CLI/status surface that renders active session and trajectory progress as live human-readable lines instead of requiring raw JSONL tailing.

## Best Possible Solution

Scope a narrow core CLI progress reader over existing session/trajectory data, with safe redacted previews and tests for multi-session follow, rotation, disabled capture, and coordination with broader activity/progress work.

## Reproduction Assessment

Not applicable as a bug reproduction. Source, docs, and the v2026.5.12 tag give high confidence that OpenClaw has structured trajectory capture and export, but no live human-readable progress tail.

## Solution Assessment

Unclear until maintainers choose the public surface and redaction contract. Building on the existing trajectory recorder and sessions/logs CLI area looks maintainable once that product boundary is set.
