## Summary

Keep open: current main still has structured trajectory capture and session listing/export, but no first-class live, human-readable progress tail for active agent work. The recent contributor comment proposing `openclaw sessions tail --follow` reinforces that the remaining decision is command shape and redaction policy, not cleanup.

## What This Changes

Request a first-class human-readable live progress surface over session trajectory data so operators can tail active work without manually parsing JSONL files.

## Best Possible Solution

Add a narrow core CLI progress reader over existing session/trajectory data, likely under `openclaw sessions`, with safe redacted previews and focused tests for multi-session follow, rotation, and disabled trajectory capture.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection give high confidence that current main has gateway logs plus session list/export, but no live human-readable trajectory progress tail.

## Solution Assessment

Unclear until maintainers choose the public CLI/status surface and redaction contract. Building on the existing trajectory recorder and sessions command area looks like the maintainable direction once that product boundary is set.
