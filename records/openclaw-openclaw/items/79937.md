## Summary

Keep open: current main still has the reported hardcoded Control UI chat strings and missing chat welcome locale keys. The chat-localization part is actionable, but the requested zh-CN `Skills` wording conflicts with the current generated-locale glossary and should be split or approved by a locale/UI owner before repair.

## What This Changes

Request localization of Control UI chat welcome text, run-control labels/tooltips, composer placeholders, missing `chat.welcome` locale keys, and several zh-CN `Skills` labels.

## Best Possible Solution

Localize the chat controls through the existing Control UI i18n source and generator, and handle the `Skills` terminology through glossary policy rather than manual generated-file edits.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: set Control UI to zh-CN and open the chat panel, where the cited welcome, toolbar, placeholder, and Talk labels remain literal English. I did not run a browser session in this read-only review.

## Solution Assessment

Partly yes: wrapping the chat strings with existing `t()` keys and regenerating locales is the maintainable fix. The proposed direct zh-CN edits and `Skills` wording change are not the best path unless the glossary decision is made first.
