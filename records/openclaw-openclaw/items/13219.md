## Summary

Keep open. Current main still provides transcript-backed usage/cost summaries and separate provider quota snapshots, but it does not implement the requested durable per-call usage ledger; the issue discussion and adjacent open PRs explicitly treat transcript-summary work as non-closure for this request.

## What This Changes

The issue requests native per-model usage logging for cost tracking, with a dedicated per-call ledger containing model, session/source attribution, token/cache counts, and cost fields instead of requiring users to parse session JSONL transcripts.

## Best Possible Solution

Add a typed append-only UsageLedgerEntry written at the normalized model-call usage/cost seam, storing provider/model, agent/session/run/source/job attribution, token/cache/cost/status/latency metadata, and durable event IDs without prompt or completion content. Then make usage.cost, CLI/UI summaries, and an explicit backfill path prefer the ledger while preserving transcript scanning for older data and keeping provider quota snapshots separate.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. The gap is high-confidence by source/docs inspection: current main has transcript-backed summaries and provider quota surfaces, but no ledger type, usage.jsonl storage path, ledger writer, backfill command, or ledger-first reader.

## Solution Assessment

Yes for triage: keeping this open for maintainer review is the best current handling. A transcript-only CLI command or aggregation fix does not solve the durable per-call ledger request, and a narrow automated patch would be premature before storage, API, privacy, and idempotency decisions are made.
