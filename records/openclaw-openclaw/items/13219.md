## Summary

Keep open. Current main still exposes transcript/session-backed cost summaries and provider quota snapshots, not the requested native append-only per-call usage ledger with durable event IDs, source/run/job attribution, backfill, or ledger-first readers. Related discussion keeps this as the canonical ledger request: https://github.com/openclaw/openclaw/issues/14377 was closed as a duplicate of this issue, https://github.com/openclaw/openclaw/issues/11894 is a narrower auth-profile attribution follow-up, and the open CLI PR at https://github.com/openclaw/openclaw/pull/49064 is adjacent transcript-summary work rather than a ledger implementation.

## What This Changes

Request native per-model/per-call usage logging for cost tracking, with source/session attribution and a dedicated log instead of manual transcript parsing.

## Best Possible Solution

Add a typed core UsageLedgerEntry written at normalized model-call completion, then make cost summaries, CLI/UI readers, and a backfill command prefer that ledger while preserving transcript scanning for older data.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a feature request; the high-confidence verification path is source inspection showing current main lacks the requested ledger and still uses transcript-backed summaries.

## Solution Assessment

Is this the best way to solve the issue? Unclear for the exact API shape, but a native append-only ledger at the normalized usage/cost seam is the maintainable direction; transcript-only CLI summaries and external network interceptors are useful adjuncts, not replacements.
