## What This Changes

Requests native per-model, per-call usage logging for cost tracking and model/source attribution without manually scanning session transcripts.

## Best Possible Solution

Add a typed append-only UsageLedgerEntry written at the normalized model-call usage/cost seam, then make usage.cost, CLI/UI readers, and a backfill command prefer that ledger while preserving transcript scanning for older data.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The high-confidence verification path is source, docs, release-tag, and live issue inspection showing current OpenClaw still has transcript-backed summaries but no durable per-call usage ledger.

## Solution Assessment

Unclear for the exact storage/API shape, but a native append-only ledger at the normalized usage/cost seam is the maintainable direction. Transcript-only summaries, external parsers, and the open CLI PR are adjacent surfaces, not replacements for the requested accounting layer.
