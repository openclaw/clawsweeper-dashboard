## Summary

Keep open. Current main has transcript-backed cost summaries, cached transcript aggregates, provider quota/status surfaces, and adjacent open CLI/gateway PRs, but it still lacks the requested append-only per-call usage ledger with durable event IDs, source/run/job attribution, idempotent writes, backfill, and ledger-first readers.

## What This Changes

Request native per-model/per-call usage logging for cost tracking, model-mix optimization, source attribution, and future budget reporting.

## Best Possible Solution

Add a typed append-only UsageLedgerEntry at normalized model-call completion, then make cost summaries, CLI/UI readers, and a backfill command prefer that ledger while preserving transcript scanning for older data.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The high-confidence verification path is source inspection showing current main lacks the requested durable ledger and still serves usage summaries from transcripts and a transcript aggregate cache.

## Solution Assessment

Unclear for the exact public API shape, but a native append-only ledger at the normalized usage/cost seam is the maintainable direction. Transcript-only summaries, diagnostic events, external parsers, and the open transcript-summary CLI are useful adjacent surfaces, not replacements for the requested durable accounting layer.
