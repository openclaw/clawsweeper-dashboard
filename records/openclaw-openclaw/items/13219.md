## Summary

Keep open. Current main still provides transcript-backed usage summaries and a transcript-derived cache, while the open CLI PR is an adjacent reporting surface rather than the requested durable per-call usage ledger.

## What This Changes

Request a native append-only per-model/per-call usage log for cost tracking and model-mix attribution instead of requiring users to parse session transcripts.

## Best Possible Solution

Add a typed append-only `UsageLedgerEntry` at the normalized model-call usage/cost seam, then make cost summaries, CLI/UI readers, and a backfill command prefer that ledger while preserving transcript scanning for older data.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The high-confidence verification path is source inspection showing current main lacks a durable usage ledger and still serves usage summaries from transcripts and a transcript-derived cache.

## Solution Assessment

Unclear for the exact public API and storage shape, but a native append-only ledger at the normalized usage/cost seam is the maintainable direction. Transcript-only summaries, external parsers, and the open CLI PR are useful adjacent surfaces, not replacements for the requested accounting layer.
