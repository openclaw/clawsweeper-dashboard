## Summary

Keep this issue open. Current main and v2026.6.8 have transcript-backed cost summaries, a durable transcript aggregate cache, session usage logs, and diagnostic `model.usage` events, but not the requested durable per-call usage ledger with stable event IDs, source/run/job attribution, idempotent writes, backfill, and ledger-first readers.

## What This Changes

The issue requests a native per-model/per-call usage log or ledger for cost tracking and model-mix analysis.

## Best Possible Solution

Add a canonical core completed-call usage ledger at the normalized model-call usage/cost seam, likely SQLite-backed, with durable event IDs, agent/session/run/source/job attribution, provider/model/token/cache/cost/status/latency metadata, no prompt or completion content, a backfill path, and ledger-first usage readers with transcript fallback for older data.

## Reproduction Assessment

Yes. Source inspection of current main and v2026.6.8 reproduces the gap: OpenClaw exposes transcript-backed summaries, derived caches, per-session log readers, and diagnostic usage events, but no durable usage-ledger type, writer, backfill, or ledger-first reader.

## Solution Assessment

No current implementation fully solves it. The maintainable path is one canonical completed-call accounting contract, likely SQLite-backed under current storage policy, before deriving CLI, UI, gateway, diagnostics, or plugin views from it.
