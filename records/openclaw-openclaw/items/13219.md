## Summary

Keep open. Current main has transcript-backed cost/session summaries, provider quota/status snapshots, and partial cron run telemetry, but it still lacks the requested durable per-call usage ledger; the issue discussion also treats open CLI work such as #49064 as adjacent rather than closure.

## What This Changes

Request native append-only per-model usage logging for LLM calls so users can aggregate token and cost usage by model, session, source, cron/job, and provider.

## Best Possible Solution

Add a typed append-only UsageLedgerEntry at the normalized model-call usage/cost seam, then make usage.cost, CLI/UI summaries, and backfill prefer that ledger while preserving transcript scanning for historical data.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. The gap is high-confidence by source and docs inspection: current main exposes transcript-backed summaries and cron-specific run telemetry, but no usage.jsonl-style ledger, ledger writer, durable event IDs, backfill, or ledger-first reader.

## Solution Assessment

Unclear for exact implementation details, but transcript scanning or the open CLI PR is not sufficient for this request. A core ledger seam is the maintainable direction, with storage/API/privacy/idempotency scope decided by maintainers first.
