## Summary

Keep open. Current main still has transcript-backed usage summaries and a transcript-derived durable cache, but not the requested native append-only per-call usage ledger; the linked open CLI PR adds transcript reporting rather than the durable ledger described in the issue discussion.

## What This Changes

Request a native per-model usage log for per-call token, cache, cost, session/source, and model attribution to support cost tracking and model-mix optimization.

## Best Possible Solution

Add a typed append-only UsageLedgerEntry at the normalized model-call usage/cost seam, then make cost summaries, CLI/UI readers, and a backfill command prefer that ledger while preserving transcript scanning for older data.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The high-confidence verification path is source inspection showing current main still serves transcript-backed summaries and lacks a durable usage ledger.

## Solution Assessment

Unclear for the exact public API and storage shape, but a native append-only ledger at the normalized usage/cost seam is the maintainable direction. Transcript-only summaries, external parsers, and the open CLI PR are adjacent surfaces, not replacements for the requested accounting layer.
