## Summary

Keep open. Current main still has transcript-backed usage summaries and a transcript-derived cache, but it does not implement the requested native append-only per-call usage ledger with durable event ids, source/run/job attribution, backfill, or ledger-first readers; recent discussion also explicitly treats the open CLI and gateway PRs as adjacent rather than closure for this request.

## What This Changes

The issue requests native per-model usage logging via a dedicated append-only per-call ledger for token, cache, cost, session, and source attribution.

## Best Possible Solution

Add a typed append-only usage ledger at the normalized model-call usage/cost seam, store attribution and accounting metadata without prompt/completion content, and make CLI/UI/API summaries prefer the ledger with transcript fallback and backfill for older sessions.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. The gap is high-confidence from source, docs, and discussion: current main has transcript-backed summaries/caches and session usage logs, but no usage.jsonl-style ledger writer, durable event ids, backfill, or ledger-first reader.

## Solution Assessment

Unclear for exact implementation details, but the requested ledger direction is not satisfied by the current transcript cache or the open transcript-scanning CLI PR. A core ledger seam with transcript fallback is the maintainable direction, with storage/API/privacy/idempotency scope decided by maintainers first.
