## Summary

Keep open: current main and v2026.6.10 still provide transcript-backed summaries, provider quota/status views, UI/Gateway aggregation, and model diagnostic events, but not the native durable per-call usage ledger requested here. The burn0 and ClawMetry links in the discussion are useful interim/external approaches, but they still leave OpenClaw core without a canonical ledger, idempotent event IDs, source/run/job attribution, backfill, or ledger-first readers.

## What This Changes

Request native per-model usage logging for cost tracking, with a dedicated durable per-call usage ledger instead of manual transcript parsing.

## Best Possible Solution

Define one core usage-ledger contract at the normalized model-call usage/cost seam, store only accounting metadata with durable idempotency keys and no prompt/completion content, then make usage.cost, CLI/UI summaries, and backfill prefer that ledger while preserving transcript scanning for older data.

## Reproduction Assessment

Yes. At source level, current main exposes transcript-backed summaries, derived caches, UI/Gateway aggregation, and diagnostics, but no durable per-call usage-ledger type, writer, backfill, or ledger-first reader.

## Solution Assessment

No current implementation fully solves it. Transcript-backed summaries and model.usage diagnostics are useful adjacent surfaces, but the maintainable solution is a canonical durable accounting source with explicit privacy, storage, and migration semantics.
