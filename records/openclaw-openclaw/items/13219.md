## Summary

Keep open. Current main has transcript-backed cost/session summaries, provider quota/status snapshots, diagnostics metrics/spans, and cron-specific token telemetry, but it still lacks the requested durable append-only per-call usage ledger with event IDs, source/run/job attribution, backfill, and ledger-first readers; maintainer discussion also explicitly treats the open CLI summary work as adjacent rather than closure.

## What This Changes

The issue requests native per-model usage logging through a dedicated per-call usage ledger for cost tracking and model-mix analysis.

## Best Possible Solution

Add a core typed append-only `UsageLedgerEntry` written at the normalized model-call usage/cost seam, then make `usage.cost`, CLI/UI summaries, and a backfill command prefer that ledger while preserving transcript scanning for older sessions.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. The gap is high-confidence by source and docs inspection: current main has transcript-derived summaries, diagnostics telemetry, and cron run tokens, but no `usage.jsonl`-style ledger, ledger writer, durable event IDs, backfill, or ledger-first reader.

## Solution Assessment

Unclear for exact implementation details, but the current transcript scan/cache path and the open CLI summary PR are not sufficient for this request. A core ledger seam is the maintainable direction, with storage/API/privacy/idempotency scope decided by maintainers first.
