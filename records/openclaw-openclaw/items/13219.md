## Summary

Keep open. Current main still exposes transcript-backed cost/session summaries, a transcript-derived cache, provider quota/status surfaces, and adjacent open PRs, but it does not implement the requested native append-only per-call usage ledger with durable event IDs, source/run/job attribution, idempotent writes, backfill, or ledger-first readers.

## What This Changes

Request a native append-only per-call usage log for provider/model token and cost attribution across user, cron, heartbeat, and other agent sources.

## Best Possible Solution

Add a typed append-only UsageLedgerEntry at the normalized model-call usage/cost seam, then make cost summaries, CLI/UI readers, and a backfill command prefer that ledger while preserving transcript scanning for older data.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The high-confidence verification path is source inspection showing current main lacks the requested durable ledger and still serves usage summaries from transcripts and a transcript-derived cache.

## Solution Assessment

Unclear for the exact public API and storage shape, but a native append-only ledger at the normalized usage/cost seam is the maintainable direction. Transcript-only summaries, external parsers, and the open CLI PR are useful adjacent surfaces, not replacements for the requested accounting layer.
