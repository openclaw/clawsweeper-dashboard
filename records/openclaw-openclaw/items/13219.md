## Summary

Keep open. Current main still provides transcript-backed usage/cost summaries and provider quota surfaces, but not the requested durable per-call usage ledger; the linked CLI/aggregation work is adjacent and the maintainer discussion treats it as non-closure.

## What This Changes

Request a native append-only per-call usage log with per-model token, cost, session, and source attribution for cost tracking.

## Best Possible Solution

Add a typed append-only UsageLedgerEntry at the normalized model-call usage/cost seam, then make usage.cost, CLI/UI summaries, and backfill prefer that ledger while preserving transcript scanning for older data.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. Source and docs inspection on current main gives high confidence that OpenClaw has transcript-backed summaries but no ledger type, storage path, writer, backfill command, or ledger-first reader.

## Solution Assessment

Yes for triage: keeping this open for maintainer review is the best current handling. A transcript-only CLI or aggregation fix would not satisfy the durable ledger request, and an automated patch would be premature before storage, API, privacy, and idempotency scope is settled.
