## Summary

Keep open. Current main and the latest release have transcript-backed usage summaries, `model.usage` diagnostics, provider quota snapshots, and a transcript aggregate cache, but they still do not provide the requested durable per-call usage ledger with stable event ids, source/run/job attribution, backfill, or ledger-first readers.

## What This Changes

The issue requests a native per-call usage ledger for provider/model cost tracking, source attribution, budget analysis, and faster aggregation than parsing session transcripts.

## Best Possible Solution

Add a typed append-only `UsageLedgerEntry` at the normalized model-call usage/cost seam, then make `usage.cost`, CLI/UI readers, and a backfill command prefer that ledger while preserving transcript scanning for older data.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, live issue context, release inspection, and history give a high-confidence verification path showing current OpenClaw lacks the requested durable per-call usage ledger.

## Solution Assessment

Unclear for the exact storage/API shape, but a native append-only ledger at the normalized usage/cost seam is the maintainable direction. Transcript caches, `model.usage` diagnostics, and external parsers are useful adjacent surfaces but not replacements for the requested accounting layer.
