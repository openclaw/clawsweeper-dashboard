## Summary

Keep open. Current main still has transcript-backed usage summaries, a transcript aggregate cache, provider quota/status surfaces, and a gateway usage-cost command, but it does not provide the requested native append-only per-call usage ledger with durable event IDs, source/run/job attribution, idempotent writes, backfill, and ledger-first readers. The open CLI PR at https://github.com/openclaw/openclaw/pull/49064 references this issue with closing syntax, but its body says it parses existing session transcripts and adds no new collection schema.

## What This Changes

Request a native per-call usage log that records model/provider, tokens, cost, and source/session attribution for cost tracking.

## Best Possible Solution

Add a typed append-only UsageLedgerEntry at the normalized model-call usage/cost seam, then make cost summaries, CLI/UI readers, and a backfill command prefer that ledger while preserving transcript scanning for older data.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The high-confidence verification path is source inspection showing current main lacks the requested durable ledger and still serves usage summaries from transcripts and a transcript aggregate cache.

## Solution Assessment

Unclear for the exact public API shape, but a native append-only ledger at the normalized usage/cost seam is the maintainable direction. Transcript-only summaries, diagnostic events, and external network interceptors are useful adjacent surfaces, not replacements for the requested durable accounting layer.
