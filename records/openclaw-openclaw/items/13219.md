## Summary

Keep open. Current main has transcript-backed usage summaries, a transcript aggregate cache, Gateway `usage.cost`, and provider quota snapshots, but it still does not implement the requested durable per-call usage ledger with event ids, source/run/job attribution, backfill, or ledger-first readers.

## What This Changes

Request native append-only per-call usage logging with provider/model/source/session attribution so users can track cost and model mix without manually parsing transcripts.

## Best Possible Solution

Add a typed append-only `UsageLedgerEntry` at the normalized model-call usage/cost seam, then make `usage.cost`, CLI/UI readers, and a backfill command prefer that ledger while preserving transcript scanning for older data.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The high-confidence verification path is source, docs, live issue/PR context, and git history showing current main has transcript-backed summaries but no durable per-call usage ledger.

## Solution Assessment

Unclear for the exact storage/API shape, but a native append-only ledger at the normalized usage/cost seam is the maintainable direction. Transcript-only summaries, external parsers, and the adjacent CLI PRs are useful context but not replacements for the requested accounting layer.
