## Summary

Keep open. Current main still provides transcript-backed cost/session summaries and provider quota snapshots, but it does not implement the requested durable per-call usage ledger; the discussion and open CLI PR #49064 are adjacent transcript-summary work, not closure for this request.

## What This Changes

Request native per-model usage logging through a dedicated usage ledger for cost tracking, model-mix analysis, and cron/session attribution.

## Best Possible Solution

Add a typed append-only core UsageLedgerEntry at the normalized model-call usage/cost seam, then make usage.cost, CLI/UI summaries, and a backfill path prefer that ledger while preserving transcript scanning for old data.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. The gap is high-confidence by source/docs inspection: current main has transcript-backed summaries and provider quota surfaces, but no ledger type, usage.jsonl storage path, ledger writer, backfill command, or ledger-first reader.

## Solution Assessment

Unclear for implementation details, but the requested direction is broader than transcript scanning. A core ledger seam is the maintainable direction; the storage/API/privacy/idempotency boundary needs maintainer design before a fix PR.
