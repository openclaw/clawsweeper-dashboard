## Summary

Keep open: current main and v2026.6.10 have transcript-backed summaries, a derived usage-cost cache, and live `model.usage` diagnostics, but not the requested native durable per-call usage ledger with stable IDs, source/run/job attribution, backfill, and ledger-first readers. The closed CLI PR and open plugin diagnostics PR are useful adjacent work, not replacements for the core accounting design.

## What This Changes

Request native per-model usage logging so users can track model costs, model mix, and source attribution without manually parsing session transcripts.

## Best Possible Solution

Define one canonical, privacy-safe completed-model-call usage ledger with stable IDs and attribution, then derive usage.cost, sessions.usage, CLI/UI summaries, backfill, and optional plugin/export views from that source.

## Reproduction Assessment

Yes. Source inspection shows current main has transcript-derived summaries and live diagnostic events, but no durable per-call ledger with event IDs, source/run/job attribution, backfill, and ledger-first readers.

## Solution Assessment

Unclear. A native ledger is plausible and useful, but maintainers need to choose the durable storage/API/privacy boundary before implementation rather than letting automation add a new accounting store ad hoc.
