## Summary

Keep this issue open. Current main and v2026.6.6 still provide transcript-backed cost summaries, session usage logs, diagnostic events, and UI aggregation, but not the requested native per-call usage ledger with durable event IDs, source/run/job attribution, idempotency, backfill, and ledger-first readers.

## What This Changes

The issue requests native per-model, per-call usage logging for cost tracking rather than manual transcript parsing.

## Best Possible Solution

Add a canonical core completed-call usage ledger at the normalized model-call usage/cost seam, with durable event IDs, agent/session/run/source/job attribution, provider/model/token/cache/cost/status/latency metadata, no prompt or completion content, a backfill path, and ledger-first usage readers with transcript fallback for older data.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.6 expose transcript-backed summaries, derived caches, and telemetry, but no durable usage-ledger type, writer, backfill, or ledger-first reader. No live runtime repro was needed because this is a feature gap visible in source and docs.

## Solution Assessment

No current implementation fully solves it. The requested dedicated ledger direction is valid, but the maintainable OpenClaw shape should be a canonical core accounting contract, likely SQLite-backed under current storage policy, rather than another ad hoc JSONL runtime state file.
