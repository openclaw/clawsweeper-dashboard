## Summary

Keep this issue open. Current main has transcript-backed cost summaries, provider quota snapshots, `model.usage` diagnostics, and a durable aggregate cache, but it still lacks the requested canonical per-call usage ledger with durable event IDs, source/run/job attribution, idempotent writes, backfill, and ledger-first readers.

## What This Changes

The issue requests native per-model usage logging so users can track cost, source, sessions, and model mix without parsing all session transcripts manually.

## Best Possible Solution

Define one canonical privacy-preserving usage ledger contract, preferably at the normalized model-call usage/cost seam, with durable IDs, provider/model, agent/session/run/source/job attribution, token/cache/cost/status/latency metadata, no prompt or completion content, backfill for historical transcripts, and ledger-first readers for `usage.cost`, CLI/UI summaries, and plugin-safe observability.

## Reproduction Assessment

Yes, at source level: current main exposes transcript-backed summaries, diagnostic events, and cache-derived aggregates, but not a first-class durable usage ledger. This is a feature gap rather than a failing existing contract.

## Solution Assessment

Unclear as proposed: the requested native ledger is the right product problem, but the exact `logs/usage.jsonl` shape should be reconsidered against OpenClaw's SQLite-first state policy and plugin/API boundary.
