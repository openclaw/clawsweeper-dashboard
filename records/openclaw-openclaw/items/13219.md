## Summary

Keep open. Current main and the latest v2026.5.22 release still provide transcript-backed usage summaries, a transcript-derived aggregate cache, and provider quota/status surfaces, but not the requested durable per-call usage ledger with event IDs, source/run/job attribution, backfill, or ledger-first readers; the related open PRs and discussion are adjacent rather than closure for this request.

## What This Changes

Request a dedicated per-call usage log or ledger for provider/model/session/source token and cost attribution across user, cron, heartbeat, and background model calls.

## Best Possible Solution

Add a typed append-only UsageLedgerEntry at the normalized model-call usage/cost seam, then make usage.cost, CLI/UI readers, and a backfill command prefer that ledger while preserving transcript scanning for older data.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, release inspection, and live issue context give a high-confidence verification path showing current OpenClaw lacks the requested durable per-call usage ledger.

## Solution Assessment

Unclear for the exact storage/API shape, but a native append-only ledger at the normalized usage/cost seam is the maintainable direction. Transcript caches, provider quota snapshots, external parsers, and the open transcript-backed CLI PR are useful adjacent surfaces but not replacements for the requested accounting layer.
