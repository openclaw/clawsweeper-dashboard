## Summary

Keep open. Current main and v2026.5.20 provide transcript-backed usage summaries, provider quota snapshots, diagnostics, and a durable transcript-derived aggregate cache, but they still do not implement the requested append-only per-call usage ledger with stable event ids, source/run/job attribution, backfill, or ledger-first readers; the linked CLI/fork work is adjacent transcript-summary work, not a replacement.

## What This Changes

Request native per-model/per-call usage logging so users can track token and cost attribution across sessions, cron jobs, heartbeat runs, and model choices.

## Best Possible Solution

Add a typed append-only UsageLedgerEntry at the normalized model-call usage/cost seam, then make usage.cost, CLI/UI readers, and a backfill command prefer that ledger while preserving transcript scanning for older data.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, live issue context, release inspection, and history give a high-confidence verification path showing current OpenClaw lacks the requested durable per-call usage ledger.

## Solution Assessment

Unclear for the exact storage/API shape, but a native append-only ledger at the normalized usage/cost seam is the maintainable direction. Transcript caches, model.usage diagnostics, provider quota snapshots, and external parsers are useful adjacent surfaces but not replacements for the requested accounting layer.
