## Summary

Keep open. Current main and the latest release still provide transcript-backed summaries, a transcript aggregate cache, provider quota/status surfaces, diagnostic `model.usage` events, and an open transcript-summary CLI PR, but not the requested native append-only per-call usage ledger. Prior discussion also keeps this as the canonical ledger request: https://github.com/openclaw/openclaw/issues/14377 was closed as a duplicate of this issue, while https://github.com/openclaw/openclaw/pull/49064 only adds transcript-summary CLI output.

## What This Changes

Request a dedicated per-model/per-call usage log for cost tracking and model-mix optimization instead of requiring users to parse session JSONL transcripts.

## Best Possible Solution

Add a typed append-only `UsageLedgerEntry` at normalized model-call completion, then make cost summaries, CLI/UI readers, and a backfill command prefer that ledger while preserving transcript scanning for older data.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The high-confidence verification path is source inspection showing current main and the latest release still lack the requested durable ledger and serve usage summaries from transcripts and a transcript aggregate cache.

## Solution Assessment

Unclear for the exact public API and storage shape, but a native append-only ledger at the normalized model-call usage seam is the maintainable direction. Transcript-only summaries, diagnostic events, provider quota snapshots, and external interceptors are useful adjacent surfaces, not replacements for the requested accounting layer.
