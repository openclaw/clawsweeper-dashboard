## Summary

Keep open. Current main and v2026.5.22 still provide transcript-backed usage summaries, provider quota snapshots, UI aggregation, and a transcript-derived cache, but not the requested durable per-call usage ledger with event IDs, source/run/job attribution, backfill, or ledger-first readers.

## What This Changes

Request a native per-call usage log or ledger for model, token, cost, session, and source attribution so users can track spend and model mix without parsing transcripts.

## Best Possible Solution

Add a typed append-only UsageLedgerEntry at the normalized model-call usage/cost seam, then make usage.cost, CLI/UI readers, and a backfill command prefer that ledger while preserving transcript scanning for older data.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, release inspection, and live issue context give a high-confidence verification path showing current OpenClaw lacks the requested durable per-call usage ledger.

## Solution Assessment

Unclear for the exact storage/API shape, but a native append-only ledger at the normalized usage/cost seam is the maintainable direction. Transcript caches, model.usage diagnostics, provider quota snapshots, and external parsers are useful adjacent surfaces but not replacements for the requested accounting layer.
