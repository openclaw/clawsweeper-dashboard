## Summary

Current main still lacks the requested durable per-call accounting data: existing summaries are transcript-backed, and the newer sanitized model-call plugin hooks omit tokens, cost, auth-profile, and job/source fields. Keep this open for a maintainer architecture decision; the open gateway aggregation PR is adjacent rather than a complete implementation.

## What This Changes

Request a native per-call, per-model usage ledger with token, cost, session, and automation-source attribution.

## Best Possible Solution

Extend the generic sanitized model-call telemetry contract with normalized usage, cost status, and source/run attribution, persist canonical events in the appropriate SQLite store, and make existing usage summaries consume that store while consolidating auth-profile requirements with the related issue.

## Reproduction Assessment

Not applicable: this issue requests a new durable accounting capability, while current main's documented transcript-backed summaries are functioning as designed.

## Solution Assessment

No, the proposed `logs/usage.jsonl` sidecar is not the best repository-aligned implementation because OpenClaw-owned runtime state belongs in SQLite; one canonical database-backed ledger behind the generic model-call telemetry seam is safer and more maintainable.
