## Summary

Keep open. Current main still implements transcript-backed usage summaries/caches and provider quota snapshots, not the requested append-only per-call usage ledger; maintainer discussion also treats the open CLI summary work as adjacent rather than closure.

## What This Changes

Requests a native per-model, per-call usage log for cost tracking, cron/session attribution, and model-mix optimization.

## Best Possible Solution

Add a core typed append-only usage ledger written at the normalized model-call usage/cost seam, then make usage.cost, CLI/UI summaries, and backfill prefer that ledger while preserving transcript scanning for older data.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. The gap is high-confidence by source and docs inspection: current main has transcript-derived summaries and caches, but no usage.jsonl-style ledger, ledger writer, durable event IDs, backfill, or ledger-first reader.

## Solution Assessment

Unclear for exact implementation details, but the current transcript cache and open transcript-scanning CLI PR are not sufficient for this request. A core ledger seam with transcript fallback is the maintainable direction, with storage/API/privacy/idempotency scope decided by maintainers first.
