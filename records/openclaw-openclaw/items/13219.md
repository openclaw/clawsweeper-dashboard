## Summary

Keep open: current main still has transcript-backed usage summaries and provider quota snapshots, but no durable per-call usage ledger; open PR #49064 is adjacent CLI transcript-summary work, not closure for this request.

## What This Changes

Request native per-call usage logging with model, token, cost, session, and source attribution for per-model cost tracking without scanning session transcripts.

## Best Possible Solution

Add a typed append-only core UsageLedgerEntry written at the normalized model-call usage/cost seam, then make usage.cost, CLI/UI summaries, and a backfill path prefer that ledger while preserving transcript scanning for older data.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. Source inspection gives high confidence that current main has transcript-backed summaries and provider quota surfaces, but no native append-only usage ledger, ledger writer, backfill path, or ledger-first reader.

## Solution Assessment

Unclear for implementation details. A core ledger seam is the maintainable direction, but the storage/API/privacy/idempotency boundary needs maintainer design before a fix PR should attempt it.
