## Summary

Keep open. Current main still provides transcript-backed usage summaries and a durable transcript aggregate cache, but not the requested native append-only per-call usage ledger with event IDs, source/run/job attribution, backfill, or ledger-first readers.

## What This Changes

Request native per-model usage logging through a dedicated per-call usage ledger for cost tracking and model-mix analysis.

## Best Possible Solution

Add a typed append-only usage ledger at the normalized model-call usage/cost seam, store attribution and accounting metadata without prompt/completion content, and make CLI/UI/API summaries prefer the ledger with transcript fallback and backfill for older sessions.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. The gap is high-confidence by source and docs inspection: current main has transcript-backed summaries/caches and session usage logs, but no usage.jsonl-style ledger writer, durable event IDs, backfill, or ledger-first reader.

## Solution Assessment

Unclear for exact implementation details, but the current transcript cache and the open transcript-scanning CLI PR are not sufficient for this request. A core ledger seam with transcript fallback is the maintainable direction, with storage/API/privacy/idempotency scope decided by maintainers first.
