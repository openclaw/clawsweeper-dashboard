## Summary

Keep open. Current main and the latest release still provide transcript-backed usage summaries and a transcript-derived cache, not the requested append-only per-call usage ledger; the linked CLI PR is adjacent transcript-summary work and should not close this ledger request.

## What This Changes

The issue requests native per-model/per-call usage logging for cost tracking, source attribution, and model-mix optimization.

## Best Possible Solution

Add a typed append-only UsageLedgerEntry at the normalized model-call usage/cost seam, then make cost summaries, CLI/UI readers, and a backfill command prefer that ledger while preserving transcript scanning for older data.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The high-confidence verification path is source and docs inspection showing current main and `v2026.5.19` still serve transcript-backed summaries and lack a durable per-call usage ledger.

## Solution Assessment

Unclear for the exact public API and storage shape, but a native append-only ledger at the normalized usage/cost seam is the maintainable direction. Transcript-only summaries, external parsers, and the open CLI PR are adjacent surfaces, not replacements for the requested accounting layer.
