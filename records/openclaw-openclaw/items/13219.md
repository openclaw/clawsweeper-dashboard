## Summary

Keep open. Current main still has transcript-backed usage/cost summaries and provider quota snapshots, but it does not implement the requested durable per-call usage ledger; the discussion and adjacent open PRs treat transcript-summary work as non-closure for this request.

## What This Changes

The issue requests a native append-only per-call usage log with per-model token, cache, cost, session, and source attribution for cost tracking.

## Best Possible Solution

Add a typed append-only UsageLedgerEntry written at the normalized model-call usage/cost seam, then make usage.cost, CLI/UI summaries, and a backfill path prefer that ledger while preserving transcript scanning for older data.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. The gap is high-confidence by source/docs inspection: current main has transcript-backed summaries and provider quota surfaces, but no ledger type, usage.jsonl storage path, ledger writer, backfill command, or ledger-first reader.

## Solution Assessment

Yes for triage: keeping this open for maintainer review is the best current handling. A transcript-only CLI command or aggregation fix does not solve the durable per-call ledger request, and an automated patch would be premature before storage, API, privacy, and idempotency decisions are settled.
