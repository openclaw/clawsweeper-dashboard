## Summary

Keep open. Current main and the latest release still provide transcript-backed usage summaries, transcript-derived durable cache entries, provider quota/status surfaces, and UI aggregation, but not the requested native per-call usage ledger with durable event IDs, source/run/job attribution, idempotent writes, backfill, or ledger-first readers.

## What This Changes

The issue requests native per-model/per-call usage logging for cost tracking, model-mix optimization, budget monitoring, and cron/subagent attribution.

## Best Possible Solution

Add a core typed append-only usage ledger at the normalized completed-model-call usage/cost seam, preferably backed by OpenClaw-owned SQLite state, then make usage.cost, CLI/UI summaries, and a deduped backfill path prefer the ledger while preserving transcript summaries for older data.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, release inspection, and live issue context give a high-confidence verification path showing current OpenClaw lacks the requested durable per-call usage ledger.

## Solution Assessment

Unclear for the exact public API and storage shape, but a native ledger at the normalized usage/cost seam is the maintainable direction. The requested JSONL file should be reconsidered against the repository's SQLite-first state policy and privacy requirements.
