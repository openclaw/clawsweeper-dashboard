## Summary

Keep open: current main and v2026.6.10 still have transcript-backed summaries, derived usage caches, UI/Gateway aggregation, provider quota/status views, and model diagnostic events, but not the requested durable per-call usage ledger. The burn0 comment and related diagnostic/plugin work are useful adjacent paths, but they do not replace a native core accounting source with idempotent event IDs, source/run/job attribution, backfill, and ledger-first readers.

## What This Changes

Requested native per-model LLM usage logging with source/session attribution so users can track costs and optimize model mix.

## Best Possible Solution

Define one core usage-ledger contract at the normalized model-call usage/cost seam, store only accounting metadata with durable idempotency keys and no prompt/completion content, then make usage.cost, CLI/UI summaries, and backfill prefer that ledger while preserving transcript scanning for older data.

## Reproduction Assessment

Yes. Source inspection shows current main exposes transcript-backed summaries, derived caches, UI/Gateway aggregation, and diagnostics, but no durable per-call usage-ledger type, writer, backfill, or ledger-first reader.

## Solution Assessment

No current implementation fully solves it. Transcript-backed summaries and model.usage diagnostics are useful adjacent surfaces, but the maintainable solution is a canonical durable accounting source with explicit privacy, storage, and migration semantics.
