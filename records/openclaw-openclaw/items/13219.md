## Summary

Keep open. Current main still serves usage from transcript-derived summaries and a durable transcript aggregate cache; it does not implement the requested append-only per-call usage ledger, and the open CLI PR with closing metadata only adds transcript parsing.

## What This Changes

Request native per-model usage logging with a dedicated per-call usage log for cost tracking, model-mix optimization, and source/session attribution.

## Best Possible Solution

Add a typed append-only usage ledger at the normalized model-call usage/cost seam, store attribution and accounting metadata without prompt/completion content, and make CLI/UI/API summaries prefer the ledger with transcript fallback and backfill for older sessions.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. The gap is high-confidence by source and docs inspection: current main has transcript-backed summaries/caches and session usage logs, but no usage.jsonl-style ledger writer, durable event IDs, backfill, or ledger-first reader.

## Solution Assessment

Unclear for exact implementation details, but the current transcript cache and open transcript-scanning CLI PR are not sufficient for this request. A core ledger seam with transcript fallback is the maintainable direction, with storage/API/privacy/idempotency scope decided by maintainers first.
