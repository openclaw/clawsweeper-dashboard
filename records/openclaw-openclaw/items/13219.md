## Summary

Keep open. Current main still provides transcript-backed usage summaries and a transcript-derived cache, but it does not implement the requested native append-only per-call usage ledger; the linked CLI PR is open and also transcript-based rather than a durable ledger.

## What This Changes

Request native per-model and per-call usage logging so users can track costs, model mix, and automated job/session attribution without parsing session transcripts manually.

## Best Possible Solution

Add a typed append-only UsageLedgerEntry at the normalized model-call usage/cost seam, then make cost summaries, CLI/UI readers, and a backfill command prefer that ledger while preserving transcript scanning for older data.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The high-confidence verification path is source inspection showing current main still serves transcript-backed summaries and lacks a durable per-call usage ledger.

## Solution Assessment

Unclear for the exact public API and storage shape, but a native append-only ledger at the normalized usage/cost seam is the maintainable direction. Transcript-only summaries, external parsers, and the open CLI PR are adjacent surfaces, not replacements for the requested accounting layer.
