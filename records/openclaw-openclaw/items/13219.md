## Summary

Keep open. Current main and the latest release still provide transcript-backed usage summaries and a transcript-derived cache, not the requested append-only per-call usage ledger with durable event IDs, run/source/job attribution, idempotent writes, backfill, and ledger-first readers.

## What This Changes

Request a native append-only per-call usage log for model, token, cache, cost, session, and source attribution.

## Best Possible Solution

Add a typed append-only UsageLedgerEntry at the normalized model-call usage/cost seam, then make cost summaries, CLI/UI readers, and a backfill command prefer that ledger while preserving transcript scanning for older data.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. The high-confidence verification path is source, docs, tests, latest-release tag, and live issue inspection showing OpenClaw still has transcript-backed summaries but no durable per-call ledger.

## Solution Assessment

Unclear for the exact public API and storage shape, but a native append-only ledger at the normalized usage/cost seam is the maintainable direction. Transcript-only summaries, external parsers, and the open CLI PR are adjacent surfaces, not replacements for the requested accounting layer.
