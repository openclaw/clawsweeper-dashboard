## Summary

Keep open: current main still silently drops the writer cache on a stat mismatch and reloads from disk, while the requested structured warning is a new operator-facing observability contract that needs maintainer direction.

## What This Changes

Request to emit a structured `sessions/store` warning when the on-disk session store changes between writer cache loads.

## Best Possible Solution

Approve or adjust the logging contract, then add a narrow structured warn in the writer-cache invalidation path with regression coverage for external invalidation and no false-positive in-process queued writes.

## Reproduction Assessment

Yes, by source inspection: a writer-cache stat mismatch invalidates and returns `null`, and the writer reloads from disk without logging. I did not run a live write scenario because this review is read-only.

## Solution Assessment

Unclear: the proposed structured warn on `sessions/store` is the narrowest visible direction and fits the existing logger surface, but its payload and frequency should be approved before implementation.
