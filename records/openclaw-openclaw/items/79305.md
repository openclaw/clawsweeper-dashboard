## Summary

Keep open: current main and the latest release still silently invalidate and reload the mutable session-store writer cache on mtime/size mismatches, and the requested warn log is a small core-session observability contract that needs maintainer approval before implementation.

## What This Changes

Request a structured `sessions/store` warn log when writer cache stat checks detect that the on-disk sessions store changed before a mutable writer load.

## Best Possible Solution

If maintainers want this signal, add a narrow `sessions/store` warning on writer-cache invalidation with focused regression tests for stat mismatch and missing-file paths, without changing write behavior.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: populate the writer cache, externally change or remove the store file so `mtimeMs` or `sizeBytes` differs, and the next writer load invalidates the cache without logging.

## Solution Assessment

Unclear until maintainer direction: the proposed structured warn is a narrow implementation, but the warn level and payload shape are new operator-facing policy.
