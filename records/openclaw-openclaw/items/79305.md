## Summary

Keep open: current main confirms the silent cache-invalidation path, but the request adds a new operator-facing logging contract and needs maintainer direction rather than conservative cleanup.

## What This Changes

Request a structured warn log when the session writer cache detects that the on-disk session store changed between writer cache loads.

## Best Possible Solution

Approve or adjust the logging contract, then add a narrow structured warn in the writer-cache invalidation path with regression coverage for both external invalidation and no false-positive in-process queued writes.

## Reproduction Assessment

Yes, by source inspection: a writer-cache stat mismatch invalidates and returns `null`, and the writer reloads from disk without logging. I did not run a live write scenario because this review is read-only.

## Solution Assessment

Unclear: the proposed structured warn on `sessions/store` is the narrowest visible direction and fits the existing logger meta surface, but its payload and frequency should be approved as a new observability contract before implementation.
