## Summary

Keep open: current main still silently drops the writer cache on an mtime/size mismatch and reloads from disk, but the requested warn payload is a new operator-facing observability contract that should get maintainer direction before implementation.

## What This Changes

Add a structured `sessions/store` warn log when the session-store writer cache is invalidated by an on-disk stat mismatch or missing file.

## Best Possible Solution

Approve or revise the logging contract, then add a narrow structured warning in the writer-cache invalidation path with regression coverage for external invalidation and no false positives from queued in-process writes.

## Reproduction Assessment

Yes. Source inspection shows a writer-cache mtime/size mismatch invalidates the cache and returns `null`, after which the writer reloads from disk without a log signal.

## Solution Assessment

Unclear. A structured `sessions/store` warn is a narrow technical fit for the existing logger, but the event name, payload, and noise tolerance should be approved as a new observability contract first.
