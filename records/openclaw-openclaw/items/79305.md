## What This Changes

The issue requests a structured warning when the session store writer cache is invalidated by on-disk changes between writer cache loads.

## Best Possible Solution

A maintainer should first confirm the desired diagnostic contract, then implement a bounded fail-open signal in the session writer cache path with focused coverage for stat mismatch, missing file, and normal in-process queue serialization.

## Solution Assessment

Unclear until a maintainer confirms the logging contract. The proposed warn-only shape is plausible and narrow, but the repository should decide whether this hot-path diagnostic belongs as structured logging, an audit event, or no operator-facing signal.
