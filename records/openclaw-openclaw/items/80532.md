## Summary

Keep open, but the PR is not mergeable as submitted: the useful core idea matches the open feature request at https://github.com/openclaw/openclaw/issues/77096, while this branch relaxes security-sensitive exec approval hardening without the requested approval-plan UX, lacks real behavior proof, and includes unrelated dependency/pnpm policy churn.

## What This Changes

This PR adds a `tools.exec.allowSymlinkPath` config/schema path to relax symlink cwd denial for approval-backed `system.run`, plus a unit test and unrelated fs-safe/pnpm workspace and lockfile changes.

## Best Possible Solution

If maintainers accept the threat model, land a narrow default-off opt-in that displays requested and canonical cwd in approval UX, preserves canonical identity revalidation, updates docs/schema/tests, and drops dependency/pnpm churn.

Yes for the current behavior: current source and regression tests show approval-backed `system.run` rejects direct symlink cwd paths and mutable symlink path components. The PR is a feature/config opt-in rather than a regression fix.

## Solution Assessment
