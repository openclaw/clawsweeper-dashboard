## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/79082, or an equivalent centralized alias-aware status fix, while preserving real fallback diagnostics and the existing session usage snapshot contract.

Yes. Source-level reproduction is high-confidence: selected `anthropic/<model>` plus active `claude-cli/<model>` flows through literal selected/active comparisons and independently resolved auth labels, and a second reporter confirmed the same output on release 2026.5.7.

## Solution Assessment

Yes. The best fix is the linked PR direction: centralize runtime-alias equivalence for status/fallback decisions and mark CLI usage as an existing fresh context snapshot, without adding configuration or changing model execution.
