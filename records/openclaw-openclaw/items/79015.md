## Summary

Keep open: current main still has a source-visible `/status` display bug for Anthropic models executed through the `claude-cli` runtime alias, and an open closing PR is already tracking the narrow fix.

## What This Changes

Display `/status` correctly when Anthropic models execute through the `claude-cli` runtime alias, avoiding a spurious fallback line, unknown auth label, and unknown context numerator.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/79082 if its alias-equivalence approach preserves real fallback notices while carrying the active CLI auth label and fresh context tokens into `/status`.

Yes. Source-level reproduction is clear from the documented `anthropic/*` plus `agentRuntime.id: "claude-cli"` setup and current direct selected/active ref comparisons; a second reporter also reproduced the same output on release 2026.5.7.

## Solution Assessment

Yes. The narrow maintainable direction is to compare known runtime aliases by canonical physical route/auth for status display and persist a fresh CLI context snapshot when usage metadata is available; this should be completed through the open linked PR, not by closing the issue early.
