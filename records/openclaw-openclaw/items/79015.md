## Summary

Keep open: current main still has the source-visible alias mismatch for Anthropic models executed through the `claude-cli` runtime, and the issue is already paired with an unmerged closing PR that targets the same display/accounting bug.

## What This Changes

Make `/status` treat Claude CLI runtime aliases for Anthropic models as the selected route, show the resolved OAuth auth label, and render a real context numerator from trusted usage data.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/79082, or an equivalent maintainer patch, so alias equivalence and trusted CLI context accounting are fixed together.

## Reproduction Assessment

Yes. Current main source shows the selected `anthropic/<model>` route and active `claude-cli/<model>` route are compared literally, and a v2026.5.7 runtime comment confirms the same display symptoms in a real Docker setup.

## Solution Assessment

Yes. The narrow maintainable solution is to centralize known runtime-alias equivalence for fallback/status rendering and pass a trusted CLI context snapshot through the existing session-usage freshness contract, without changing model selection or adding configuration.
