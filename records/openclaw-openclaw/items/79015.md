## Summary

Keep open: current main still has source-visible literal selected-vs-active model comparisons for fallback/status display, and the runner still does not pass the CLI usage snapshot flag needed for the context numerator. ## What This Changes

The issue asks `/status` to treat `claude-cli/<model>` as an alias of the selected `anthropic/<model>` route for fallback display, auth labeling, and context usage display.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/79082 or an equivalent centralized alias-aware status/session fix while preserving real fallback notices when selected and active physical routes truly differ.
