## Summary

Keep open: current main still has source-visible fallback/context display gaps for Anthropic models executed through the Claude CLI runtime, and an open closing PR already targets this exact issue.

## What This Changes

The issue asks for `/status` to treat `anthropic/<model>` selected routes and `claude-cli/<model>` active runtime aliases as the same physical model/auth route and to show a real context numerator.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/79082 or an equivalent narrow patch that centralizes runtime-alias equivalence for status/fallback and persists fresh CLI usage context without hiding real fallback chains.

## Reproduction Assessment

Yes, source-level: an Anthropic selected model with `agentRuntime.id: "claude-cli"` can produce selected `anthropic/<model>` and active `claude-cli/<model>` refs, and current main still compares those refs literally in fallback/status code. I did not establish a live Anthropic Max OAuth reproduction in this read-only review.

## Solution Assessment

Yes. The maintainable fix is narrow alias-aware status/fallback equivalence plus CLI usage snapshot persistence; broader runtime-policy/docs work is separately tracked by https://github.com/openclaw/openclaw/issues/79906.
