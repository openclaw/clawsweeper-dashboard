## What This Changes

## Best Possible Solution

Land one focused Feishu plugin schema fix that emits non-empty provider-visible value schemas while preserving valid Bitable field payloads, then let the issue close from the merged PR.

## Reproduction Assessment

Yes. Current main and latest release still declare the three reported Feishu schemas with `Type.Record(Type.String(), Type.Any())`, and Bedrock forwards tool parameters directly into the provider input schema.

## Solution Assessment
