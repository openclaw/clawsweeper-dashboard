## Summary

Keep open: current main and latest release still expose the reported Feishu bitable empty sub-schema, and open closing PRs are implementation candidates rather than a reason to close this issue before merge.

## What This Changes

Reports that Feishu bitable write tools emit empty nested JSON Schema under `patternProperties`, causing strict Bedrock-backed Anthropic requests to reject the full tool list.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/94633 or an equivalent plugin-local Feishu schema repair with regression tests; keep broader provider schema sanitizer policy separate.

## Reproduction Assessment

Yes for the schema emission path: current main and v2026.6.8 still use `Type.Record(Type.String(), Type.Any())`, TypeBox 1.1.39 makes that record value an empty schema, and Bedrock forwards tool parameters directly. A live Bedrock call was not run in this read-only review.

## Solution Assessment

Yes, the best fix is a narrow Feishu bitable schema repair using an explicit non-empty flexible value schema plus regression coverage. A broader Anthropic/Bedrock sanitizer is a possible follow-up, but it is not required to fix this plugin-specific blocker.
