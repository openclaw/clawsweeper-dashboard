## Summary

Keep open: current main and the latest release still contain the reported Feishu bitable `Type.Any()` record schemas, and two open closing PRs are candidate fixes, so the issue should close only after one canonical fix lands.

## What This Changes

Fix the Feishu bitable write tools so their `fields` and `property` schemas emit non-empty JSON Schema accepted by strict Bedrock-backed Anthropic validation.

## Best Possible Solution

Land one focused Feishu-local non-empty value schema repair with registered-tool regression coverage, then let this issue close from the merged PR and close the duplicate PR attempts.

## Reproduction Assessment

Yes for source reproduction: current main and `v2026.6.9` still emit `Type.Any()` record value schemas for the three Feishu bitable write tools, TypeBox 1.1.39 makes those value schemas empty, and Bedrock receives tool parameters directly. A live Bedrock call was not run in this read-only review.

## Solution Assessment

Yes, the narrowest maintainable solution is a Feishu-local non-empty value schema plus regression coverage. A broader Anthropic/Bedrock sanitizer can be considered later, but it is not required to fix this bundled plugin bug.
