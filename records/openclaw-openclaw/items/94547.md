## Summary

Keep open: current main and v2026.6.9 still expose the Feishu bitable `Type.Any()` record value schemas, and two open closing PRs are candidate fixes that should be reviewed instead of closing this issue early.

## What This Changes

The issue asks OpenClaw to stop Feishu bitable write tools from emitting empty nested JSON Schema values that strict Bedrock-backed Anthropic validation rejects.

## Best Possible Solution

Land exactly one Feishu-local non-empty value schema repair with registered-tool regression coverage, then close this issue from the merged PR.

## Reproduction Assessment

Yes, source-level: current main and v2026.6.9 still use `Type.Record(Type.String(), Type.Any())` for the three affected schema maps, and Bedrock forwards those tool schemas directly into `inputSchema.json`. A live Bedrock call was not run.

## Solution Assessment

Yes: the maintainable fix is a Feishu-local explicit non-empty flexible value schema plus registered-tool regression tests. The immediate path is to review and land one of the open linked candidate PRs, not start another repair branch.
