## Summary

Keep open: current main and the latest v2026.6.9 release still define the three Feishu bitable write-tool free-form maps with Type.Any, and the issue has open closing PRs that should be reviewed and landed before this report closes.

## What This Changes

Fix Feishu bitable write tool schemas so Bedrock-backed Anthropic no longer rejects the provider tool list because of empty nested patternProperties sub-schemas.

## Best Possible Solution

Review and land one Feishu-local non-empty value schema repair with registered-tool regression coverage, then close this issue from the merged PR; avoid Type.Unknown-only fixes because Type.Unknown also serializes empty in TypeBox 1.1.39.

## Reproduction Assessment

Yes, source-level: current main and v2026.6.9 still emit the three Type.Any record value schemas, TypeBox 1.1.39 places the empty value under patternProperties, and Bedrock receives tool parameters directly. I did not run a live Bedrock request in this read-only review.

## Solution Assessment

Yes. A Feishu-local explicit non-empty JSON-value schema is the narrowest maintainable fix for this shipped plugin bug; a broad provider sanitizer is not needed unless maintainers want a separate hardening follow-up.
