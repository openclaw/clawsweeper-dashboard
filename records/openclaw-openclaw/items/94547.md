## Summary

Keep open: current main and shipped v2026.6.9 still expose the reported Feishu bitable schemas, and two open closing PRs already target the exact bug. The right cleanup is to review and land one of those PRs rather than close the issue or queue a duplicate fix.

## What This Changes

The issue reports that three Feishu bitable write tools emit empty `patternProperties` value schemas that strict Bedrock-backed Anthropic validation rejects.

## Best Possible Solution

Land one of the existing Feishu schema fix PRs after maintainer review, then close this issue as implemented by that merge.

## Reproduction Assessment

Yes for the schema emission path: current main and v2026.6.9 still use `Type.Record(Type.String(), Type.Any())`, and TypeBox 1.1.39 places that empty value schema under `patternProperties`. A live Bedrock request was not run in this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is to change the affected Feishu bitable value schemas to an explicit non-empty flexible JSON value schema with regression coverage. A broader Anthropic/Bedrock sanitizer could be follow-up hardening, but it is not required to fix this plugin bug.
