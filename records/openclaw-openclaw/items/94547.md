## Label Justifications

- `P1`: The bug can block all agent turns for affected users when Feishu bitable write tools are enabled with a strict Bedrock-backed Anthropic route.
- `impact:auth-provider`: The failure is in provider-facing tool-schema payload compatibility, causing the configured model provider to reject requests before agent work can proceed.

## Summary

Keep open: current main and the latest release still expose the reported Feishu bitable `Type.Any()` record value schemas, and two open linked PRs are candidate fixes rather than merged implementations.

## What This Changes

The issue reports that three Feishu bitable write-tool parameters emit empty nested JSON Schema under `patternProperties`, causing Bedrock-backed Anthropic requests to reject the whole tool list.

## Best Possible Solution

Land one focused Feishu bitable schema repair with non-empty provider-visible value schemas and regression coverage, then close this issue and the duplicate fix attempts.

## Reproduction Assessment

Yes, at source level: current main and `v2026.6.9` still use `Type.Record(Type.String(), Type.Any())` for the three affected Feishu bitable write parameters, and TypeBox 1.1.39 makes that record value an empty nested schema. I did not run a live Bedrock request in this read-only review.

## Solution Assessment

Yes. A Feishu-local non-empty JSON-value schema plus registered-tool regression tests is the narrowest maintainable fix; a broader Bedrock sanitizer can remain a follow-up provider policy decision.
