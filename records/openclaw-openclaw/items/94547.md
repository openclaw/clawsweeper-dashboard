## Summary

Keep open: current main and v2026.6.10 still expose the reported Feishu bitable `Type.Any()` record schemas, while two open closing PRs already own candidate fixes and should be reviewed before this issue closes.

## What This Changes

The issue reports that Feishu bitable write-tool schemas emit empty `patternProperties` value schemas that strict Bedrock-backed Anthropic validation rejects.

## Best Possible Solution

Land one Feishu-local non-empty value-schema repair with registered-tool regression coverage, then close this issue from the merged PR; avoid a `Type.Unknown()`-only fix because it does not address the empty-schema concern.

## Reproduction Assessment

Yes, source-level: current main and v2026.6.10 still define the three affected Feishu bitable schemas with `Type.Any()`, TypeBox 1.1.39 source makes that an empty record value schema, and Bedrock forwards tool parameters directly. A live Bedrock request was not run in this read-only review.

## Solution Assessment

Yes: a narrow Feishu-local non-empty JSON-value schema plus regression coverage is the maintainable fix. The remaining action is to review and land one open closing PR rather than queue a new repair branch or add a broad provider sanitizer.
