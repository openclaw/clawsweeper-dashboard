## Summary

Keep open: current main and latest release still define the three Feishu bitable write-tool maps with `Type.Record(Type.String(), Type.Any())`, and the open closing PRs are candidate fixes rather than a reason to close this issue before merge.

## What This Changes

The issue requests a Feishu bitable schema fix so write-tool free-form maps no longer emit empty nested JSON Schema sub-schemas that strict Bedrock-backed Anthropic validation rejects.

## Best Possible Solution

Land one Feishu-local non-empty value schema repair with registered-tool regression coverage, then close this issue from the merged PR; avoid a `Type.Unknown()`-only change because it does not address the empty-schema concern.

## Reproduction Assessment

Yes, source-level: current main and `v2026.6.9` still emit the three `Type.Any()` record schemas, and Bedrock receives `tool.parameters` directly as `inputSchema.json`. A live Bedrock request was not run in this read-only review.
