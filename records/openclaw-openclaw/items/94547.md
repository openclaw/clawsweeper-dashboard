## What This Changes

The issue reports that Feishu bitable write tools emit empty `patternProperties` sub-schemas that strict Bedrock Anthropic validation rejects, blocking tool-bearing agent turns for affected users.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/94633 or an equivalent tested Feishu schema repair, close duplicate fix attempts, then close this issue after the fix merges.

## Reproduction Assessment

Yes for the schema emission path: current main and v2026.6.8 still emit the three `Type.Any()` record schemas, TypeBox 1.1.39 makes those value schemas empty, and Bedrock forwards tool parameters directly. A live Bedrock call was not run.

## Solution Assessment

Yes. The narrow plugin-local Feishu schema repair is the best immediate fix; a broader provider sanitizer is separate policy work and should not be required to unblock this bug.
