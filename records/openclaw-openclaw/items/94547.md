## Summary

Keep open: current main and shipped v2026.6.8 still expose the reported Feishu bitable empty nested schema shape, and open closing PRs are candidate fixes rather than a reason to close this issue before merge.

## What This Changes

The issue asks OpenClaw to make three Feishu bitable write tool parameters emit non-empty JSON Schema values so strict Bedrock-backed Anthropic validation does not reject every tool request.

## Best Possible Solution

Land exactly one Feishu-local non-empty value schema repair with registered-tool regression coverage, then close this issue from the merged PR; do not use Type.Unknown alone because it has the same empty schema problem.

## Reproduction Assessment

Yes, source-level: current main and v2026.6.8 still emit the three Type.Any record schemas, TypeBox 1.1.39 places the empty value under patternProperties, and Bedrock receives tool parameters directly. A live Bedrock request was not run in this read-only review.

## Solution Assessment

Yes: the best fix is a Feishu-local schema repair using an explicit non-empty flexible JSON-value schema plus regression coverage. The open question is choosing one of the active candidate PRs rather than starting another repair path.
