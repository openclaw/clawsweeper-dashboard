## Summary

Keep open. The underlying packaged-layout bug is still source-reproducible on current main, but this PR still has resolver/test blockers and lacks inspectable real packaged-runtime proof, so it should not merge as-is.

## What This Changes

The PR changes workspace-template directory resolution to try module-relative source and bundled docs fallbacks before cwd and adds a bundled-layout regression test.

## Best Possible Solution

Land a narrow resolver fix that keeps package-root lookup first, adds the correct dist-relative fallback before cwd or overshooting paths, isolates the regression test from `argv1`, and includes redacted packaged-gateway proof.

## Reproduction Assessment

Yes, source-reproducible. Current main has no dist-relative `../docs/reference/templates` fallback, so a bundled module with null package-root discovery and a cwd without templates can resolve to a missing template path and then throw in `loadTemplate()`.

## Solution Assessment

No. The PR targets the right narrow bug, but the current branch should preserve the production-module fallback under injected module URLs, control `argv1` in the regression test, and prefer the bundled fallback before the source-layout fallback.

  - body: This test sets a fake cwd and module URL but leaves `process.argv[1]` pointing at the test runner. Since `resolveOpenClawPackageRoot()` also searches from `argv1`, it can discover the checkout package root and return checkout templates before exercising the synthetic `../docs` fallback.   - confidence: 0.88

  - body: For a bundled dist module with null package-root discovery, `../../docs` points outside the package and is checked before the correct `../docs` fallback. If a sibling docs tree exists, the wrong templates could win; checking the bundled candidate first also avoids the extra miss in the broken layout this PR is fixing.
  - confidence: 0.78
