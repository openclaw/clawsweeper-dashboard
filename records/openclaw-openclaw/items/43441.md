## Summary

Keep this PR open: it targets a real workspace-template resolution edge case that current main still does not directly cover, but the branch has blocking resolver/test issues and lacks inspectable packaged-runtime proof.

## What This Changes

The PR changes workspace-template directory resolution to compute module-relative source and bundled fallback directories before cwd, and adds a bundled-layout resolver test.

## Best Possible Solution

Land a narrow resolver fix that preserves package-root lookup, adds the correct dist-relative fallback before cwd or overshooting paths, isolates regression tests from process argv, and includes redacted packaged-gateway proof.

## Reproduction Assessment

Yes, source-reproducible. Current main has no dist-relative `../docs/reference/templates` fallback, so a bundled module with null package-root discovery and a cwd without templates can resolve to a missing path and then throw in `loadTemplate()`.

## Solution Assessment

No. The PR targets the right narrow bug, but the branch should preserve the production-module fallback under injected module URLs, control `argv1` in the regression test, prefer the bundled fallback first, and attach inspectable real behavior proof.

  - body: This test sets a fake cwd and module URL but leaves `process.argv[1]` pointing at the test runner.   - confidence: 0.88
- **[P3] Try the bundled fallback before the source fallback:** `src/agents/workspace-templates.ts:21-22`
  - body: For a bundled dist module with null package-root discovery, `../../docs` points outside the package and is checked before the correct `../docs` fallback. If a sibling docs tree exists, the wrong templates could win; checking the bundled candidate first also avoids the extra miss in the broken layout this PR is fixing.
  - confidence: 0.78
