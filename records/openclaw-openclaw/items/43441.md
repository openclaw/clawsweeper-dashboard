## Summary

Keep open: current main still lacks the bundled `dist/` template fallback, and this PR targets a valid resolver bug, but the latest head still has correctness issues in the fallback/test path and lacks inspectable real behavior proof.

## What This Changes

This PR changes workspace-template directory resolution to try module-relative source and bundled fallback directories before cwd, and adds a bundled-layout regression test.

## Best Possible Solution

Land a narrow resolver fix that preserves package-root discovery, uses the real module fallback correctly, tries the bundled fallback before overshooting paths, isolates regression tests from argv1, and includes redacted packaged-gateway proof.

## Reproduction Assessment

Yes, source-reproducible: current main lacks the bundled `../docs` fallback, and `loadTemplate()` throws when the resolved template directory lacks the requested template. I did not run a live packaged WSL2/systemd gateway reproduction.

## Solution Assessment

No. The PR is scoped to the right bug, but it should fix the fallback/test-isolation defects, prefer the bundled fallback first, and provide inspectable real behavior proof before merge.
