## Summary

Keep open: current main still has the packaged dist fallback gap, but this PR should not merge as-is because the test seam changes fallback behavior, the new bundled-layout test can be bypassed by argv1 package-root discovery, and the user-facing fix needs a changelog entry.

## What This Changes

The PR changes workspace template resolution to add source and bundled module-relative fallback candidates before cwd lookup, and adds a bundled-layout regression test.

## Best Possible Solution

Repair this PR before merge: preserve package-root-first resolution, add reliable bundled/source module-relative fallbacks ahead of cwd, keep existing fallback behavior covered, isolate the bundled-layout regression test from real package-root discovery, and document the fix in Unreleased Fixes.

## Reproduction Assessment

Yes. The original bug is reproducible by reasoning through current main with a dist-rooted module, packageRoot null, and a daemon cwd without templates; the PR defects are reproducible from the resolver candidate order and the root discovery candidates used by the tests.

## Solution Assessment

No. The production direction is narrow and appropriate, but the current patch is not the best form because the injected moduleUrl changes all fallback dirs and the new test does not force package-root discovery to stay null.
