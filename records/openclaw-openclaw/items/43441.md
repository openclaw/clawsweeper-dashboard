## Summary

Keep this PR open: current main still lacks the bundled `dist/` workspace-template fallback, so the change is useful, but the branch is not merge-ready because the resolver/test seam still has correctness issues and the external PR lacks inspectable real behavior proof.

## What This Changes

The PR changes workspace-template resolution to try module-relative source and bundled fallback directories before cwd, and adds a bundled-layout resolver test.

## Best Possible Solution

Land a narrow resolver fix that preserves package-root discovery, tries the correct dist-relative fallback before unreliable cwd or overshooting paths, isolates regression tests from argv1, and includes redacted packaged-gateway proof.

## Reproduction Assessment

Yes, source-reproducible. Current main lacks the dist-relative fallback, and `loadTemplate()` throws when the selected directory has no template; I did not run a packaged gateway repro in this read-only review.

## Solution Assessment

No. The PR targets the right narrow bug, but the patch should preserve the real production fallback under test injection, isolate argv1 in the bundled-layout test, prefer the bundled fallback first, and provide inspectable real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Keep the real fallback under injected module URLs:** `src/agents/workspace-templates.ts:47-52`
  - body: After this refactor, `opts.moduleUrl` drives both package-root discovery and fallback directory generation. The existing missing-template test injects a synthetic dist URL while expecting the resolver's real module fallback to find checkout templates, so keep the real `import.meta.url` fallback reachable or inject only the synthetic bundled candidate for the new test.
  - confidence: 0.9
- **[P2] Isolate the bundled-layout test from argv1:** `src/agents/workspace-templates.test.ts:74`
  - body: The new test controls `cwd` and `moduleUrl` but leaves `process.argv[1]` pointing at the test runner. Since root discovery also searches argv1, it can find the checkout package root before the synthetic `../docs` fallback is exercised.
  - confidence: 0.88
- **[P3] Try the bundled fallback before the source fallback:** `src/agents/workspace-templates.ts:21-22`
  - body: For a bundled dist module with null package-root discovery, `../../docs` points outside the package and is checked before the correct `../docs` fallback. If a sibling docs tree exists, the wrong templates could win, and the broken layout also pays an avoidable miss first.
  - confidence: 0.78
