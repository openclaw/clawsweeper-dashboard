## Summary

Keep open: current main still lacks the bundled `dist/` workspace-template fallback, and this PR is aimed at a real source-reproducible startup bug, but the branch still has resolver/test correctness issues and no inspectable after-fix real behavior proof.

## What This Changes

The PR changes workspace template directory resolution to try module-relative source and bundled `dist` fallbacks before cwd, and adds a bundled-layout regression test.

## Best Possible Solution

Keep the PR open and repair the resolver/test so package-root resolution remains first, the bundled `../docs/reference/templates` fallback is reliable before cwd or overshooting paths, tests isolate root discovery, and contributor proof is attached.

## Reproduction Assessment

Yes. Source inspection on current main shows that with package-root discovery null, cwd without templates, and a module URL under package `dist`, current main lacks the required `../docs/reference/templates` bundled fallback and can throw during template loading.

## Solution Assessment

No. The PR is the right narrow direction, but this branch should keep or explicitly inject the real-module fallback for synthetic-module calls, isolate argv1/root discovery in the new test, and prefer the bundled fallback before the overshooting source-layout path.
