## What This Changes

The PR changes workspace-template resolution to try module-relative source and bundled fallback directories before cwd, and adds a bundled-layout resolver test.

## Best Possible Solution

Land a narrow resolver fix that preserves package-root discovery, adds the correct dist-relative fallback ahead of cwd or overshooting paths, isolates regression tests from argv1, and includes redacted packaged-gateway proof.

## Reproduction Assessment

Yes, source-reproducible. Current main lacks the dist-relative fallback, and the template loader throws when the selected directory has no template; I did not run a packaged gateway repro in this read-only review.

## Solution Assessment

No. The PR targets the right narrow bug, but it should preserve the production-module fallback under injected module URLs, isolate argv1 in the test, prefer the bundled fallback first, and include inspectable real behavior proof.

  - confidence: 0.88

  - body: For a bundled dist module with null package-root discovery, `../../docs` points outside the package and is checked before the correct `../docs` fallback. If a sibling docs tree exists, the wrong templates could win, and the broken layout also pays an avoidable miss first.
  - confidence: 0.78
