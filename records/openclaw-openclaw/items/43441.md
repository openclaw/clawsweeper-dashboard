## Summary

Keep open: current main still lacks the bundled `dist` template fallback, but this PR has actionable resolver/test issues and is missing inspectable real behavior proof.

## What This Changes

The branch changes workspace template resolution to try module-relative source and bundled `dist` template directories before cwd, and adds a bundled-layout regression test.

## Best Possible Solution

Repair the resolver and test so package-root resolution stays first, the bundled `../docs/reference/templates` fallback is reliable before cwd or overshooting paths, existing fallback behavior is preserved, and redacted real behavior proof is attached.

## Reproduction Assessment

Yes. Source inspection on current main shows that when package-root discovery is null and cwd lacks templates, a module running from package `dist/` has no `../docs/reference/templates` fallback and template loading can throw `Missing workspace template`.

## Solution Assessment

No. The PR is pointed at the right narrow bug, but this branch should preserve the real-module fallback for injected module URLs, control argv1 in the bundled-layout test, and try the bundled fallback before the overshooting source-layout path.

  - body: In a bundled `dist` module with no package root, `../../docs` points outside the package and is checked before the correct `../docs` fallback. A sibling docs tree could be selected first, so put the bundled candidate before the source-layout candidate.
