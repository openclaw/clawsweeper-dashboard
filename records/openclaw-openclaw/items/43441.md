## Summary

Keep open: current main still lacks the bundled `dist/` workspace-template fallback, so the PR targets a real bug, but the PR head has blocking resolver/test issues and no inspectable real behavior proof.

## What This Changes

The PR changes workspace-template directory resolution to try module-relative source and bundled docs fallbacks before cwd and adds a bundled-layout regression test.

## Best Possible Solution

Land a narrow resolver fix that keeps package-root lookup first, adds the bundled `../docs/reference/templates` fallback before cwd or overshooting paths, preserves production fallback behavior under test injection, and includes redacted packaged-gateway proof.

## Reproduction Assessment

Yes, source-reproducible: current main has no dist-relative `../docs/reference/templates` fallback, so a bundled module with null package-root discovery and a cwd without templates can resolve to a missing template path and then throw in `loadTemplate()`.

## Solution Assessment

No. The PR targets the right narrow bug, but it should preserve the production fallback for injected module URLs, control `argv1` in the regression test, and prefer the bundled fallback before the source-layout fallback.
