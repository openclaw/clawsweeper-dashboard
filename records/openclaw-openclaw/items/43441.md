## Summary

Keep open: current main still lacks the bundled `dist/` `../docs/reference/templates` fallback, so the PR targets a real source-reproducible bug, but the patch has blocking review findings and no inspectable real behavior proof.

## What This Changes

The PR changes workspace-template directory resolution to try module-relative source and bundled template directories before cwd, and adds a bundled-layout regression test.

## Best Possible Solution

Land a narrow resolver update that keeps package-root lookup first, adds a reliable bundled `../docs/reference/templates` fallback before cwd or overshooting paths, preserves production-module fallback behavior under test injection, and includes redacted packaged-gateway proof.

## Reproduction Assessment

Yes, source-reproducible. Current main has no dist-relative `../docs/reference/templates` fallback, so when package-root discovery is null and cwd lacks templates, a bundled dist module can resolve to a missing template path and `loadTemplate()` throws.

## Solution Assessment

No. The PR targets the right narrow bug, but it should preserve the production fallback for injected module URLs, control argv1 in the bundled-layout test, and try the dist fallback before the overshooting source-layout path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Keep the production fallback reachable for injected URLs:** `src/agents/workspace-templates.ts:45-50`
  - body: Because fallback dirs now come only from the injected `moduleUrl`, callers that pass a synthetic module URL lose the production `import.meta.url` fallback. The existing missing-template test can return the temp package path after synthetic candidates miss instead of reaching checkout `docs/reference/templates`; keep the real-module fallback reachable or inject the synthetic bundled candidate separately.
  - confidence: 0.9
- **[P2] Control argv1 in the bundled-layout regression:** `src/agents/workspace-templates.test.ts:74`
  - body: This test sets fake cwd and moduleUrl but leaves `process.argv[1]` pointing at the Vitest runner. `resolveOpenClawPackageRoot` can discover the checkout package root from argv1 and return checkout templates before exercising the synthetic `../docs` fallback, so pass a fake argv1 or inject root discovery.
  - confidence: 0.88
- **[P3] Try the dist fallback before the source fallback:** `src/agents/workspace-templates.ts:21-23`
  - body: For a bundled dist module with null packageRoot, `../../docs` points outside the package and is checked before the correct `../docs` fallback. A sibling docs tree could be selected first, so the bundled candidate should come before the source-layout candidate.
  - confidence: 0.78
