## What This Changes

The PR adds module-relative source and bundled docs fallback candidates for workspace templates, moves cwd lookup later, and adds a bundled-layout resolver test.

## Best Possible Solution

Land a narrow resolver fix that preserves package-root lookup, tries the dist-relative docs path before cwd or overshooting paths, isolates regression tests from argv1, and includes redacted packaged-gateway proof.

## Reproduction Assessment

Yes, source-reproducible. Current main lacks the `dist/`-relative `../docs/reference/templates` fallback, and `loadTemplate()` throws if the resolved directory does not contain the requested template.

## Solution Assessment

No. The PR targets the right bug, but the branch should preserve a production-module fallback under injected test URLs, isolate argv1 in the regression test, prefer the bundled fallback first, and attach inspectable real behavior proof.
