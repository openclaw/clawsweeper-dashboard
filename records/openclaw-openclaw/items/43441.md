## Summary

Keep open: current main still lacks the dist-relative fallback this PR is trying to add, but the PR head has blocking test/resolver issues and lacks inspectable after-fix packaged-runtime proof.

## What This Changes

The PR changes workspace-template resolution to try module-relative source and bundled fallback directories before cwd, and adds a bundled-layout resolver test.

## Best Possible Solution

Land a narrow resolver fix that preserves package-root discovery, keeps a real source-layout fallback under test injection, tries the correct dist-relative fallback before cwd or overshooting paths, isolates regression tests from argv1, and includes redacted packaged-gateway proof.

## Reproduction Assessment

Yes, source-reproducible: current main lacks the bundled ../docs fallback, and loadTemplate() throws when the resolved template directory has no template. I did not establish a live packaged WSL2/systemd reproduction in this read-only review.

## Solution Assessment

No. The PR targets the right narrow bug, but the patch should preserve the real production fallback under test injection, isolate argv1 in the bundled-layout test, prefer the bundled fallback first, and provide inspectable real behavior proof.
