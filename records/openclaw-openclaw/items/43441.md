## Summary

Keep this PR open. Current main still has the bundled-dist workspace template fallback bug, and the PR addresses the right area, but the current branch introduces blocking resolver/test issues and still needs the required changelog entry before merge.

## What This Changes

## Best Possible Solution

Repair this PR before merge: keep package-root resolution first, add reliable module-relative fallbacks for bundled and source layouts ahead of cwd, preserve the real-module fallback contract or add a narrower test seam, isolate package-root discovery in the bundled-layout regression test, and add an Unreleased Fixes changelog entry with contributor credit.

No. The production direction is narrow and appropriate, but this patch is not the best form because opts.moduleUrl now changes all fallback dirs and the bundled-layout test does not isolate package-root discovery; a narrower fallback/module-dir or root-resolver test seam is safer.
