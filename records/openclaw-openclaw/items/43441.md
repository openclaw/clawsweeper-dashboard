## Summary

Keep open: current main still has the packaged dist fallback bug, and this PR is useful but needs resolver/test repairs plus the required changelog entry before merge.

## What This Changes

The branch changes workspace template directory resolution to try module-relative source and bundled dist fallbacks before cwd, and adds a bundled-layout regression test.

## Best Possible Solution

Repair this PR before merge: preserve package-root-first resolution, add reliable source and bundled module-relative fallbacks ahead of cwd without losing the real source fallback, isolate the bundled regression test from root discovery, and document the fix in Unreleased Fixes.

## Reproduction Assessment

Yes. The original bug is reproducible by code-path analysis with a dist-rooted module, `packageRoot` null, and a daemon cwd without templates; the PR defects are reproducible from the resolver candidate order and root-discovery candidates.

## Solution Assessment

No. The production direction is narrow and appropriate, but the current patch is not the best form because fallback injection is too broad and the new test does not force package-root discovery to stay null.
