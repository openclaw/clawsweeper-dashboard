## Summary

Keep open: the PR is focused, but the added global-hook chain can recurse into this repository's own pre-commit hook when the global hooksPath is relative or points at the same file.

## What This Changes

The PR modifies `git-hooks/pre-commit` to run an executable globally configured pre-commit hook before OpenClaw's existing staged-file formatting flow.

## Best Possible Solution

Keep the global-hook chaining idea only after resolving the candidate hook and skipping it when it is the same executable as the repo-local hook, with focused hook coverage for that case.

## Reproduction Assessment

Yes. Source inspection plus Git's hook-path contract shows that a global relative `core.hooksPath=git-hooks` resolves from the worktree root and can call `git-hooks/pre-commit` from inside itself.

## Solution Assessment

No. The hook is the right layer for this local hardening, but the implementation needs a same-file recursion guard before it is the narrow maintainable fix.
