## Summary

Keep open. Current main and the latest release still build Bedrock-discovered models with zero default pricing and expose no supported discovery-cost overlay; the only exact implementation PR was closed unmerged, while the related closed issue handled explicit aws-sdk pricing display rather than this discovery metadata gap.

## What This Changes

The issue requests a way to attach token cost metadata to Amazon Bedrock models discovered through `models.bedrockDiscovery` without duplicating explicit provider model definitions.

## Best Possible Solution

Keep this issue open as the canonical discovery-pricing overlay discussion until maintainers choose the API shape, then implement it in the Bedrock plugin discovery boundary or a generic pricing overlay with schema, docs, cache, migration, and focused tests aligned.

## Reproduction Assessment

Yes, at source level: current main builds discovered Bedrock models with default zero cost and exposes discovery config without a pricing overlay. No live AWS credentials are needed to verify this config/API gap.

## Solution Assessment

No for the exact old-path proposal: `models.bedrockDiscovery` is now a legacy migration path, so the best fix should use the plugin-owned `plugins.entries.amazon-bedrock.config.discovery` surface or a maintainer-approved generic model pricing overlay.
