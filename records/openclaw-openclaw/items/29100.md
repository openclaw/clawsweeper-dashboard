## Summary

Keep open. Current main and the latest release still lack a supported pricing overlay for auto-discovered Amazon Bedrock models, and the remaining work adds provider/plugin config surface that needs maintainer API-shape approval.

## What This Changes

The issue requests cost-per-token metadata for Bedrock models discovered through automatic IAM/AWS SDK discovery without duplicating an explicit provider model block.

## Best Possible Solution

Choose one canonical pricing-overlay API, preferably under the plugin-owned Bedrock discovery config unless maintainers want a broader model-pricing overlay, then update schema, docs, discovery cache keys, discovery tests, and usage proof around that surface.

## Reproduction Assessment

Yes. Source inspection is enough to reproduce the missing capability: discovered Bedrock models get zero cost, inference profiles inherit that value, and the supported discovery config exposes no pricing overlay.

## Solution Assessment

Unclear until maintainers choose the API shape. A plugin-owned discovery pricing overlay fits current ownership better than reviving the legacy `models.bedrockDiscovery` path, while a generic `models.costs` map would need broader config approval.
