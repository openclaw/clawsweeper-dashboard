## Summary

Keep open. Current main and v2026.5.12 still lack a supported Bedrock discovery cost overlay; the related implementation pull request was closed unmerged, and the separate aws-sdk cost display issue tracks a different remaining gate.

## What This Changes

The issue requests a supported cost metadata overlay for auto-discovered Amazon Bedrock models so usage/status can show estimated spend without duplicating explicit provider model entries.

## Best Possible Solution

Add a plugin-owned discovery costs overlay, apply it while building Bedrock model definitions, include it in the discovery cache key, and leave the aws-sdk display gate to the separate linked issue.

## Reproduction Assessment

Yes. Source inspection is enough: the strict Bedrock discovery schema and shared config type omit a pricing map, and current discovery assigns zero-cost metadata to discovered foundation models and profiles.

## Solution Assessment

Yes for the plugin-owned overlay direction. The legacy models.bedrockDiscovery shape is now a compatibility path, and a top-level models.costs map would be broader product surface than this issue needs.
