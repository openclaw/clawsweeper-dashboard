## Summary

Keep open. Current main and v2026.6.10 still have no local pricing overlay for auto-discovered Amazon Bedrock models; the closed implementation PR was not merged, and the closed related issue only covered explicit configured costs for `aws-sdk` providers.

## What This Changes

Request to add local cost metadata for Amazon Bedrock auto-discovered models so `/status` and `/usage` can estimate USD without duplicating explicit provider model entries.

## Best Possible Solution

Choose one canonical pricing-overlay API, preferably under the plugin-owned Bedrock discovery config unless maintainers want a generic model-pricing overlay, then wire schema, docs, cache-key behavior, discovery tests, and usage proof around that surface.

## Reproduction Assessment

Yes. Source inspection is enough: discovered Bedrock models are assigned zero cost, inference profiles inherit that value, and the current discovery schema/docs expose no local pricing overlay.

## Solution Assessment

Unclear until maintainers choose the API shape. A plugin-owned discovery pricing overlay fits the current owner boundary better than reviving the legacy `models.bedrockDiscovery` path, while generic `models.costs` needs broader config direction.
