## Summary

Keep open: current main and v2026.6.6 still have no supported operator-supplied cost overlay for auto-discovered Amazon Bedrock models; the old implementation PR closed unmerged and the separate aws-sdk display issue does not cover discovery metadata.

## What This Changes

The issue requests cost metadata for Amazon Bedrock models discovered through Bedrock discovery without duplicating explicit `models.providers.amazon-bedrock.models[]` entries.

## Best Possible Solution

Choose a canonical pricing-overlay contract, preferably in the current Amazon Bedrock plugin config unless maintainers want a generic model-pricing overlay, then update schema, discovery cache keys, tests, and public docs together.

## Reproduction Assessment

Yes. Source inspection shows discovered Bedrock models and inference profiles still receive zero cost metadata, while the current plugin discovery schema has no cost overlay field.

## Solution Assessment

No for the exact legacy shape. The requested behavior is valid, but `models.bedrockDiscovery.costs` should not be added as-is without a maintainer decision because Bedrock discovery now lives under the plugin config path.
