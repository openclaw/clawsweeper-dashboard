## Summary

Keep open. Current main and the latest release still have no local pricing overlay for auto-discovered Amazon Bedrock models; the remaining work is a useful provider cost-visibility feature, but it adds config/API surface and needs maintainer product direction before an automated fix PR should implement it.

## What This Changes

The issue requests cost metadata for Amazon Bedrock auto-discovered models without duplicating an explicit `models.providers.amazon-bedrock.models[]` block.

## Best Possible Solution

Choose one canonical pricing-overlay API, preferably under the plugin-owned Bedrock discovery config unless maintainers want a generic model-pricing overlay, then wire schema, docs, cache-key behavior, discovery tests, and usage proof around that surface.

## Reproduction Assessment

Yes. Source inspection is enough to reproduce the missing capability: discovered Bedrock models are assigned zero cost, inference profiles inherit that value, and the current discovery schema/type/docs expose no pricing overlay.

## Solution Assessment

Unclear until maintainers choose the API shape. A plugin-owned discovery pricing overlay fits current ownership better than reviving the legacy `models.bedrockDiscovery` path, while a generic `models.costs` map would need a broader config decision.
