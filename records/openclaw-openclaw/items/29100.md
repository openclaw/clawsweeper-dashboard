## Label Justifications

- `P2`: The gap affects recurring cost observability for Bedrock deployments but does not prevent authentication, discovery, or inference.
- `impact:auth-provider`: The missing pricing metadata specifically affects Amazon Bedrock models discovered through IAM and AWS SDK credential flows.

## Summary

Current main still cannot attach local pricing to auto-discovered Amazon Bedrock models: the canonical plugin discovery schema exposes region, filtering, refresh, and token-limit defaults, while discovery emits zero cost metadata. The earlier implementation PR closed unmerged, and the separate aws-sdk display issue only fixed use of already-configured pricing. Keep this issue open for a maintainer decision on the modern public contract because Bedrock discovery has since moved from `models.bedrockDiscovery` to plugin-owned configuration.

## What This Changes

Add a supported cost-per-token metadata overlay for automatically discovered Amazon Bedrock models so status and usage surfaces can estimate USD cost without duplicating an explicit provider model list.

## Best Possible Solution

Approve one canonical modern overlay—preferably plugin-owned unless multiple discovery providers need the same contract—then merge validated cost metadata into discovered `ModelDefinitionConfig` entries, include it in discovery cache identity, document USD-per-million-token units, and preserve explicit provider rows as the higher-precedence override if that precedence is intended.

## Reproduction Assessment

Yes, at source level with high confidence: current plugin schema and discovery construction show no accepted pricing overlay and zero cost metadata for discovered models. A live AWS account is not required to establish the missing configuration path.

## Solution Assessment

No, the original `models.bedrockDiscovery.costs` proposal is no longer the best exact shape because that key is now legacy and migrates to plugin-owned configuration. The capability remains useful, but maintainers should first select the plugin-local versus provider-neutral contract and define precedence with explicit model pricing.
