## Summary

Keep open. Current main still lacks a supported Bedrock discovery cost overlay: the plugin-owned discovery schema/type omit costs, discovered Bedrock models keep default cost metadata, and the only matching implementation PR was closed unmerged.

## What This Changes

Add cost metadata for Amazon Bedrock auto-discovered models so usage surfaces can estimate USD costs without duplicating the provider model list.

## Best Possible Solution

Approve a Bedrock plugin-owned discovery cost overlay with schema, types, docs, tests, and a cache-key pricing fingerprint; treat a top-level cross-provider models.costs map as separate product work.

## Reproduction Assessment

Yes. Source inspection on current main shows the strict Bedrock discovery schema and BedrockDiscoveryConfig have no cost overlay, and discovery emits default cost metadata for foundation and profile models.

## Solution Assessment

Yes, if maintainers approve the new public config surface. A Bedrock plugin-owned overlay matches the current plugin config boundary and is narrower than a new cross-provider model-cost overlay.
