## Summary

Keep open. Current main and the latest release still lack a local per-model cost overlay for auto-discovered Bedrock models; the only direct implementation attempt was closed unmerged, and the current plugin-owned discovery shape makes the remaining work a maintainer API/product decision.

## What This Changes

Request a configurable cost metadata overlay for automatically discovered Amazon Bedrock models without duplicating the full explicit provider model list.

## Best Possible Solution

Choose the permanent API shape first, then implement either plugin-owned Bedrock discovery costs or a generic model-pricing overlay with schema, docs, migration/doctor coverage as needed, discovery cache-key handling, and focused tests.

## Reproduction Assessment

Yes. Source inspection on current main is enough: the Bedrock discovery plugin emits zero-cost model definitions and its strict discovery schema/type expose no per-model local cost overlay, so no live AWS credentials are needed to reproduce the config gap.

## Solution Assessment

Unclear. The exact old `models.bedrockDiscovery.costs` proposal is not the best current path because discovery is now plugin-owned; maintainers should choose between a plugin-local `plugins.entries.amazon-bedrock.config.discovery` overlay and a broader `models` pricing overlay.
