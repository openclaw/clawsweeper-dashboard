## Summary

Keep open. Current main still lacks a supported Bedrock discovery cost overlay: the strict plugin schema/type omit costs, discovered models get zero/default cost metadata, and the exact implementation PR at https://github.com/openclaw/openclaw/pull/31160 was closed unmerged.

## What This Changes

The issue requests a supported way to attach cost metadata to Amazon Bedrock models discovered through Bedrock discovery config without duplicating explicit provider model entries.

## Best Possible Solution

Approve and implement a Bedrock plugin-owned discovery cost overlay with schema, types, docs, tests, and a cache-key pricing fingerprint; treat a top-level `models.costs` map as separate broader product work.

## Reproduction Assessment

Yes. Source inspection is enough: the strict Bedrock discovery schema and type omit any cost overlay, and discovered models receive `DEFAULT_COST` or inherited default cost metadata.

## Solution Assessment

Yes, if maintainers approve the new public config surface. A Bedrock plugin-owned overlay is the narrow maintainable path; a top-level `models.costs` map would be broader cross-provider product work.
