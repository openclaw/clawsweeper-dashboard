## Summary

Keep open. Current main and the latest release still lack a pricing overlay for auto-discovered Bedrock models, and the linked implementation PR was closed unmerged against the older `models.bedrockDiscovery` surface.

## What This Changes

Allow cost metadata to be attached to auto-discovered Amazon Bedrock models without duplicating an explicit provider model block.

## Best Possible Solution

Choose one canonical discovered-model pricing overlay contract, then implement it at the Bedrock plugin discovery boundary or as a broader model-pricing overlay with schema, docs, cache-key handling, and focused tests.

## Reproduction Assessment

Yes, from source inspection. Current Bedrock discovery assigns zero-cost metadata to discovered models, and the plugin discovery schema has no cost overlay field.

## Solution Assessment

Unclear. A Bedrock-specific `costs` map is plausible, but the issue also proposes a generic `models.costs` overlay, so maintainers should choose the product/API contract before a fix PR.
