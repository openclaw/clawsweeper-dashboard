## Summary

Keep open. Current main still has no supported cost overlay for Bedrock auto-discovery: the plugin discovery schema and typed config omit costs, discovered models still get zero-cost metadata, and the related implementation PR closed unmerged.

## What This Changes

The issue requests per-model cost metadata for Amazon Bedrock models discovered through Bedrock auto-discovery without duplicating explicit provider model entries.

## Best Possible Solution

Land a Bedrock plugin-owned discovery cost overlay on `plugins.entries.amazon-bedrock.config.discovery`, with legacy migration compatibility, cache-key handling, docs, tests, and coordinated aws-sdk display handling with #53286.

## Reproduction Assessment

Yes. Source inspection on current main is enough: the strict plugin discovery schema and typed config have no costs field, and the discovery implementation assigns zero-cost metadata to discovered Bedrock models.

## Solution Assessment

Yes. A Bedrock plugin-owned overlay is the narrow maintainable direction for this request; the proposed global `models.costs` overlay would be broader than needed and should wait for explicit maintainer direction.

## Repair Work Prompt

Implement cost metadata support for Amazon Bedrock auto-discovery from issue #29100. Add a plugin-owned `plugins.entries.amazon-bedrock.config.discovery.costs` map keyed by discovered model ID, with `input`, `output`, `cacheRead`, and `cacheWrite` values in USD per 1M tokens. Keep legacy `models.bedrockDiscovery` migration compatibility, validate the manifest schema strictly, apply partial overrides with zero fallback, include the normalized costs map in the discovery cache key, and ensure inference profiles can receive direct profile-ID costs while still inheriting base model metadata where appropriate. Update Bedrock and token/cost docs plus CHANGELOG.md for the user-facing config. Do not add a broad top-level `models.costs` overlay unless maintainers explicitly ask. Coordinate with #53286: do not claim Bedrock aws-sdk cost display is complete unless the configured cost path is shown by status/usage tests too.
