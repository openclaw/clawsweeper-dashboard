## Summary

Keep open. Current main and v2026.5.20 still lack a supported pricing overlay for auto-discovered Bedrock models, the prior implementation PR was closed unmerged, and the remaining choice is a new public Bedrock plugin config surface rather than a closeable duplicate.

## What This Changes

Request adding a supported cost metadata overlay for Bedrock auto-discovered models so cost estimates can work without duplicating an explicit provider model list.

## Best Possible Solution

Approve and implement a Bedrock plugin-owned discovery costs overlay with schema, types, docs, focused discovery tests, and deterministic cache-key pricing coverage; treat a repo-wide model-cost overlay as separate product work.

## Reproduction Assessment

Yes. Source inspection on current main shows the strict Bedrock discovery schema/type omit any cost overlay, and discovery emits DEFAULT_COST for foundation and inference-profile models with no user pricing path.

## Solution Assessment

Unclear until maintainers choose the public config shape. A Bedrock plugin-owned overlay under `plugins.entries.amazon-bedrock.config.discovery` is the narrowest maintainable direction; a repo-wide `models.costs` map is broader product work.
