## Summary

Keep open. Current main still lacks a supported cost metadata overlay for Bedrock-discovered models; the exact implementation PR was closed unmerged, and the other open Bedrock cost issue tracks a different explicit-provider display path.

## What This Changes

Allow users to attach cost metadata to auto-discovered Amazon Bedrock models without duplicating an explicit provider model list.

## Best Possible Solution

Approve and implement a plugin-owned Bedrock discovery cost overlay, with schema/types/docs/tests and a cache-key pricing fingerprint; consider a broader models.costs overlay only as a separate product decision.

## Reproduction Assessment

Yes. Source inspection is enough: the strict Bedrock discovery config schema and type omit costs, and discovered foundation/profile models receive DEFAULT_COST or inherited default cost metadata.

## Solution Assessment

Yes, if maintainers approve the new public config surface. A Bedrock plugin-owned overlay is the narrowest maintainable fix; a top-level models.costs map would be broader cross-provider product work.
