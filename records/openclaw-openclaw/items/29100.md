## Summary

Keep open. Current main and v2026.5.12 still lack a supported cost overlay for Bedrock-discovered models, and the related implementation pull request was closed unmerged rather than superseded by a merged fix.

## What This Changes

The issue requests cost metadata overlays for auto-discovered Amazon Bedrock models without duplicating explicit provider model entries.

## Best Possible Solution

Add a plugin-owned Bedrock discovery costs overlay, apply it when building discovered foundation models and inference profiles, include it in the discovery cache key, and document the USD-per-1M-token units.

## Reproduction Assessment

Yes. Source inspection is enough: the strict plugin schema and shared BedrockDiscoveryConfig type omit a costs field, and current discovery assigns zero-cost metadata to discovered models.

## Solution Assessment

Yes. A plugin-owned discovery overlay is the narrow maintainable direction; a top-level models.costs map would be broader product surface than this issue needs.
