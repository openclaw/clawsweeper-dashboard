## Summary

Keep open: current main and v2026.6.5 still lack a local cost overlay for Bedrock plugin discovery, while the direct implementation PR closed unmerged and the separate aws-sdk display fix only covers explicit provider model rows. The remaining blocker is maintainer approval of the public config shape and precedence rules.

## What This Changes

The issue requests cost metadata for auto-discovered Amazon Bedrock models without duplicating explicit provider model entries.

## Best Possible Solution

Approve one canonical overlay shape, preferably plugin-owned unless maintainers want a provider-neutral pricing patch contract, then apply it to discovered Bedrock ModelDefinitionConfig entries with cache-key, docs, migration, and precedence coverage.

## Reproduction Assessment

Yes. Source inspection is enough: current plugin discovery schema has no accepted cost map and discovery emits zero-cost metadata for discovered foundation models and profiles.

## Solution Assessment

No for the original exact proposal, because models.bedrockDiscovery is now a legacy migration source. The capability is still useful, but maintainers should choose the modern plugin-local or provider-neutral contract before implementation.
