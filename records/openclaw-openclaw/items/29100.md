## Summary

Keep open. Current main and v2026.6.10 still materialize auto-discovered Bedrock models with zero/default cost metadata and expose no supported discovery pricing overlay; the prior implementation PR is closed unmerged and the current config surface is plugin-owned, so this needs maintainer API-shape approval before a fix is queued.

## What This Changes

Add a supported cost metadata overlay for auto-discovered Amazon Bedrock models so usage/status cost estimates work without duplicating explicit provider model entries.

## Best Possible Solution

Choose one canonical pricing-overlay API, preferably under the plugin-owned Bedrock discovery config unless maintainers approve a broader model-pricing overlay, then update schema, docs, cache keys, discovery tests, and usage proof around that surface.

## Reproduction Assessment

Yes. Source inspection reproduces the missing capability: discovered Bedrock models get zero/default cost metadata, inference profiles inherit that value, and the supported discovery config exposes no pricing overlay.

## Solution Assessment

Unclear until maintainers choose the API shape. The old models.bedrockDiscovery.costs proposal is no longer the best exact path because current main migrated discovery into plugin-owned config, while a generic models.costs map needs broader config approval.
