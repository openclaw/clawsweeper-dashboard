## Summary

Keep open. Current main and the latest release still lack a supported cost overlay for Bedrock auto-discovered models, the earlier implementation PR was closed unmerged, and the remaining work needs a maintainer choice on the public config shape.

## What This Changes

Allow user-supplied cost metadata to be attached to Amazon Bedrock models discovered through Bedrock discovery without duplicating an explicit provider model list.

## Best Possible Solution

Approve and implement a Bedrock plugin-owned discovery cost overlay with schema, types, docs, focused discovery tests, and cache-key pricing coverage, while treating a repo-wide model-cost overlay as separate product work.

## Reproduction Assessment

Yes. Source inspection on current main shows the strict Bedrock discovery schema/type omit any cost overlay, and discovery emits DEFAULT_COST for foundation and inference-profile models with no user pricing path.

## Solution Assessment

Unclear until maintainers choose the public config shape. A Bedrock plugin-owned overlay under plugins.entries.amazon-bedrock.config.discovery is the narrowest maintainable direction; a repo-wide models.costs map is broader product work.
