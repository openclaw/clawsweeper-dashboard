## Summary

Keep open. Current main still lacks a supported cost overlay for auto-discovered Bedrock models: the plugin discovery schema and shared type omit pricing, discovery assigns zero-cost metadata, and the direct implementation PR was closed unmerged.

## What This Changes

Request adding per-model cost metadata for Amazon Bedrock models created by automatic discovery without duplicating an explicit provider model list.

## Best Possible Solution

Add a plugin-owned discovery cost overlay under `plugins.entries.amazon-bedrock.config.discovery.costs`, apply it during Bedrock model discovery, and keep `aws-sdk` cost-display policy coordinated with https://github.com/openclaw/openclaw/issues/53286.

## Reproduction Assessment

Yes. Source inspection is enough to verify the missing capability: the strict Bedrock discovery schema/type omit pricing, and discovered models are assigned zero-cost metadata.

## Solution Assessment

Yes. A Bedrock plugin-owned overlay is the narrowest maintainable direction; a top-level `models.costs` overlay would be broader product surface and should wait for explicit maintainer direction.
