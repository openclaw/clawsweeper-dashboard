## Summary

Keep open. Current main and v2026.5.12 still lack a supported Bedrock discovery cost overlay, and the only implementation PR for this request was closed unmerged rather than superseded by a landed fix.

## What This Changes

The issue requests cost metadata for Amazon Bedrock models discovered through `models.bedrockDiscovery` or the plugin-owned discovery config, without duplicating explicit provider model entries.

## Best Possible Solution

Add a plugin-owned Bedrock discovery costs overlay, apply it to discovered foundation models and inference profiles, include it in the discovery cache key, and document the USD-per-1M-token units.

## Reproduction Assessment

Yes. Source inspection on current main is enough: the strict Bedrock plugin discovery schema and BedrockDiscoveryConfig omit costs, while discovery assigns zero DEFAULT_COST metadata to discovered models.

## Solution Assessment

Yes, if the new surface is approved. A plugin-owned discovery overlay is narrower than a top-level models.costs map, while the aws-sdk display-policy issue should remain separately tracked.
