## Summary

Keep open. Current main and the latest release still lack a supported user pricing overlay for auto-discovered Bedrock models; the related explicit aws-sdk cost-display work is distinct, and adding a public config shape needs maintainer product approval.

## What This Changes

The issue requests cost metadata for auto-discovered Amazon Bedrock models without duplicating explicit provider model entries.

## Best Possible Solution

Approve one canonical public config shape for discovered-model pricing, preferably plugin-owned under `plugins.entries.amazon-bedrock.config.discovery` unless maintainers intentionally want broader cross-provider cost overlays, then implement schema, docs, cache-key, discovery, and regression coverage together.

## Reproduction Assessment

Yes, at source level. Current Bedrock discovery config/output lacks a user pricing overlay path, while the supported cost contract remains explicit `models.providers.*.models[].cost` entries or gateway/cache pricing.

## Solution Assessment

No clear best implementation is approved yet. A Bedrock plugin-local overlay is the narrowest likely shape, but a top-level cross-provider overlay is broader product work and should be a maintainer decision first.
