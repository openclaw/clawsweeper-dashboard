## Summary

Keep open. Current main and the latest release still lack a supported cost overlay for auto-discovered Amazon Bedrock models; the related explicit aws-sdk cost-display work covers configured provider model entries only, and the direct discovery-overlay PR closed unmerged.

## What This Changes

Request to attach per-model pricing metadata to auto-discovered Amazon Bedrock models without duplicating explicit provider model entries.

## Best Possible Solution

Approve the desired config shape, preferably a Bedrock plugin-owned discovery pricing overlay unless maintainers explicitly want a cross-provider model-cost map, then implement it with schema, docs, cache-key, and discovery tests.

## Reproduction Assessment

Yes, at source level. Current main's strict Bedrock discovery schema and type omit any cost overlay, and discovery emits DEFAULT_COST for foundation and inference-profile models.

## Solution Assessment

Unclear until maintainers choose the public config shape. A Bedrock plugin-owned overlay under plugins.entries.amazon-bedrock.config.discovery is the narrowest maintainable direction; a repo-wide models.costs map is broader product work.
