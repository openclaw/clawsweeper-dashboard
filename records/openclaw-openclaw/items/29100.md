## Label Justifications

- `P2`: The missing metadata prevents routine Bedrock spend estimates but does not block authentication, discovery, or inference.
- `impact:auth-provider`: The affected model catalog is produced through Amazon Bedrock IAM and instance-profile provider discovery.

## Summary

Keep open: current main and v2026.6.6 still create discovered Bedrock models with zero local cost, and the newer generic pricing cache has no Amazon Bedrock manifest mapping or local override contract. The earlier implementation PR closed unmerged, so maintainers still need to choose whether catalog-derived estimates are sufficient or operators should be able to supply plugin-owned pricing overrides.

## What This Changes

Allow operator-supplied token pricing metadata for IAM-discovered Amazon Bedrock models without duplicating explicit provider model definitions.

## Best Possible Solution

Approve one canonical plugin-owned pricing policy: use Bedrock manifest normalization for general catalog estimates and, if operator-specific prices are required, add a per-model override under `plugins.entries.amazon-bedrock.config.discovery` with explicit precedence, cache-key, schema, docs, and regression coverage.

## Reproduction Assessment

Yes at source level. Current discovery assigns zero pricing, the plugin schema exposes no local cost overlay, and no Bedrock external-pricing mapping closes that gap on current main or v2026.6.6.

## Solution Assessment

No for the original legacy-path proposal. The maintainable direction is a plugin-owned contract, preferably combining provider-manifest catalog mapping with an explicitly approved local override and documented precedence.
