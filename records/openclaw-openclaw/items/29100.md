## Summary

Keep open: current main and v2026.6.6 still lack an operator-supplied pricing overlay for auto-discovered Amazon Bedrock models, and the directly linked implementation PR closed unmerged while the adjacent aws-sdk cost-display issue covered explicit configured costs only.

## What This Changes

Allow cost metadata to be attached to auto-discovered Amazon Bedrock models without duplicating explicit provider model entries.

## Best Possible Solution

Choose a canonical pricing-overlay contract, then wire it through the Amazon Bedrock discovery schema, discovered model construction, cache key, tests, and public docs without changing model routing.

## Reproduction Assessment

Yes. Source inspection is high-confidence: the strict Bedrock discovery schema has no costs field, and discovered foundation/profile entries still receive zero cost metadata.

## Solution Assessment

No for the exact legacy shape. The requested behavior is valid, but maintainers should first choose whether the overlay belongs in the current Bedrock plugin config or in a generic model-pricing surface.
