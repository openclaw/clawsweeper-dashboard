## Summary

Keep open. Current main and v2026.5.12 still lack a supported cost overlay for auto-discovered Bedrock models, the matching implementation PR was closed unmerged, and the remaining change adds a public config surface that needs maintainer approval.

## What This Changes

Request adding user-configurable cost metadata for Bedrock auto-discovered models without duplicating an explicit amazon-bedrock provider block.

## Best Possible Solution

Approve and implement a Bedrock plugin-owned `plugins.entries.amazon-bedrock.config.discovery.costs` overlay with schema, types, docs, tests, and a discovery cache-key pricing fingerprint; treat a repo-wide `models.costs` overlay as separate product work.

## Reproduction Assessment

Yes. Source inspection on current main shows the strict Bedrock discovery schema/type omit any cost overlay, and discovery emits DEFAULT_COST for foundation/profile models with no user pricing path.

## Solution Assessment

Yes, if maintainers approve the new public config surface. A Bedrock plugin-owned overlay under the current plugin config path is narrower than a repo-wide `models.costs` map, which should be a separate product decision.
