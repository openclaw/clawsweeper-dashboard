## Summary

Keep open. Current main still has no supported cost overlay for auto-discovered Amazon Bedrock models; the request is a valid provider/config feature, but the exact config shape needs maintainer product direction before a fix PR should be queued.

## What This Changes

Add a way to attach per-model cost metadata to auto-discovered Amazon Bedrock models without duplicating the whole explicit provider model list.

## Best Possible Solution

Choose a canonical pricing-overlay contract, preferably plugin-owned Bedrock discovery config unless maintainers want a broader `models.costs` overlay, then update schema, docs, cache keying, discovery tests, and migration behavior together.

## Reproduction Assessment

Yes. Source inspection is enough to reproduce the missing capability: current discovery config, manifest schema, docs, and runtime all lack a cost overlay and discovered models still receive zero default cost.

## Solution Assessment

Unclear. The requested metadata overlay is useful, but maintainers should choose between plugin-owned Bedrock discovery config and a broader model-pricing overlay before implementation.
