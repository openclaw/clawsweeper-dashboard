## Summary

Keep open. Current main still lacks any supported cost overlay for auto-discovered Amazon Bedrock models: the plugin-owned discovery schema and shared discovery type omit pricing, discovery still assigns zero costs to discovered models, and the only direct implementation PR was closed unmerged; the related aws-sdk display issue tracks a separate remaining gate.

## What This Changes

Add cost-per-token metadata for Amazon Bedrock models discovered through Bedrock discovery without duplicating explicit provider model entries.

## Best Possible Solution

Add a Bedrock plugin-owned discovery cost overlay under plugins.entries.amazon-bedrock.config.discovery, including validation, legacy migration compatibility, cache-key handling, docs, and focused discovery tests.

## Reproduction Assessment

Yes. Source inspection on current main is enough to show the missing capability: the strict plugin discovery schema and BedrockDiscoveryConfig omit a costs field, and discovered models are assigned zero-cost metadata.

## Solution Assessment

Yes. The narrow maintainable path is a Bedrock plugin-owned overlay in the current discovery config; a global models.costs overlay would be broader and should wait for explicit maintainer direction.
