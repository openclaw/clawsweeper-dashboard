## Summary

Keep open. Current main still has no supported cost overlay for auto-discovered Amazon Bedrock models: the plugin discovery schema and typed config omit pricing, discovered foundation models still receive zero-cost metadata, and the only direct implementation PR was closed unmerged; https://github.com/openclaw/openclaw/issues/53286 is related but tracks a separate aws-sdk cost-display gate.

## What This Changes

Request adding cost-per-token metadata to auto-discovered Amazon Bedrock models without duplicating an explicit provider model list.

## Best Possible Solution

Add a Bedrock plugin-owned discovery cost overlay under plugins.entries.amazon-bedrock.config.discovery, with legacy migration compatibility, cache-key/fingerprint handling, docs, tests, and coordination with the aws-sdk display policy tracked separately.

## Reproduction Assessment

Yes. Source inspection on current main is enough: the strict Bedrock plugin discovery schema and typed config have no costs field, and discovery assigns zero-cost metadata to discovered models.

## Solution Assessment

Yes. A Bedrock plugin-owned overlay is the narrow maintainable direction under the current plugin-owned discovery config; a global models.costs overlay would be broader and should wait for explicit maintainer direction.

## Repair Work Prompt

Implement cost metadata overlay support for auto-discovered Amazon Bedrock models from https://github.com/openclaw/openclaw/issues/29100. Keep the fix owned by the Amazon Bedrock plugin: add a costs map under plugins.entries.amazon-bedrock.config.discovery keyed by discovered model ID, validate nonnegative input/output/cacheRead/cacheWrite values, apply full and partial overrides with zero defaults, include the overlay in the discovery cache key/fingerprint, and preserve existing discovery behavior. Add compatibility handling for legacy models.bedrockDiscovery so old configs migrate into the plugin-owned discovery config without prototype pollution risk. Update focused tests and Bedrock/token-use docs, and add a user-facing changelog entry. Do not add a global models.costs overlay unless a maintainer explicitly redirects the design, and keep core generic except for existing shared config types required by the plugin SDK. Coordinate but do not conflate with https://github.com/openclaw/openclaw/issues/53286, which tracks aws-sdk visible cost display eligibility.
