## Summary

Keep open. Current main and v2026.5.20 still lack a supported cost overlay for auto-discovered Bedrock models, the linked implementation PR closed unmerged, and the remaining choice is a new public Bedrock plugin config surface rather than a closeable duplicate.

## What This Changes

Request support for assigning cost metadata to auto-discovered Amazon Bedrock models without duplicating explicit provider model entries.

## Best Possible Solution

Approve and implement a Bedrock plugin-owned `plugins.entries.amazon-bedrock.config.discovery.costs` overlay with schema, types, docs, focused discovery/cache tests, and clear USD-per-1M-token units; treat a repo-wide `models.costs` overlay as separate product work.

## Reproduction Assessment

Yes. Source inspection on current main shows the strict Bedrock discovery schema/type omit any cost overlay, and discovery emits DEFAULT_COST for foundation and inference-profile models with no user pricing path.

## Solution Assessment

Unclear until maintainers choose the public config shape. A Bedrock plugin-owned overlay under the current plugin config path is the narrowest maintainable direction; a repo-wide `models.costs` map is broader product work.
