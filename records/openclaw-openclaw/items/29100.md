## Summary

Keep open. Current main still has no supported pricing overlay for auto-discovered Amazon Bedrock models: the plugin discovery schema, legacy config type/schema, cache key, discovery conversion, tests, and docs all omit a per-model cost map, and discovered foundation/profile models still get zero-cost metadata. The earlier implementation PR #31160 is closed unmerged, while #53286 remains open for the related aws-sdk display behavior.

## What This Changes

Requested support for attaching per-model cost metadata to auto-discovered Amazon Bedrock models without duplicating explicit provider model entries.

## Best Possible Solution

Land a backward-compatible, plugin-owned Bedrock discovery pricing overlay that applies configured rates to foundation model IDs and inference profile IDs, keeps cache entries deterministic, updates schema/help/docs/tests, and ensures configured Bedrock costs are visible on the affected usage surfaces.

## Repair Work Prompt

Implement cost metadata support for auto-discovered Amazon Bedrock models. Add a backward-compatible pricing overlay under `plugins.entries.amazon-bedrock.config.discovery.costs` keyed by model ID, with legacy `models.bedrockDiscovery.costs` compatibility only where the existing migration/runtime fallback needs it. Normalize/validate cost entries as USD per 1M tokens with `input`, `output`, `cacheRead`, and `cacheWrite`; partial entries should default missing rates to zero. Apply configured costs to discovered foundation model IDs and inference profile IDs, including profile IDs that map to a priced backing foundation model. Include normalized pricing in the Bedrock discovery cache key so changed pricing does not reuse stale cached model definitions. Update `extensions/amazon-bedrock/openclaw.plugin.json`, config types/schema/help/docs, and focused tests. Coordinate with #53286 so configured Bedrock costs are displayed for aws-sdk-auth status and usage surfaces, or clearly include that display fix in the same PR. Do not change AWS credential resolution, request routing, model selection, or hard-code AWS pricing catalogs. Add a user-facing changelog entry crediting @mg-mavios, and credit @Pan-Binghong too if the aws-sdk display fix is included.
