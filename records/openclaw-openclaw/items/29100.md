## Summary

Keep open. Current main and v2026.6.6 still materialize auto-discovered Bedrock foundation models and inference profiles with zero pricing, and the modern plugin-owned discovery config has no cost overlay; the old requested `models.bedrockDiscovery` path is now legacy input, so the remaining work needs a maintainer API-shape decision rather than cleanup closure.

## What This Changes

Requests a way to attach cost metadata to auto-discovered Amazon Bedrock models without duplicating explicit provider model entries.

## Best Possible Solution

Adopt one canonical pricing-overlay API, preferably a plugin-owned Bedrock discovery overlay unless maintainers intentionally choose a generic model-pricing map, then wire schema, docs, cache-key handling, discovery tests, and usage proof around that single surface.

## Reproduction Assessment

Yes. Source inspection on current main shows discovered Bedrock foundation models are assigned zero `DEFAULT_COST`, inference profiles inherit that value, and the discovery config schema/type/docs do not accept a cost overlay; no AWS credentials are needed to prove the missing metadata path.

## Solution Assessment

Unclear until maintainers choose the API shape. A narrow plugin-owned discovery pricing overlay matches current ownership better than reviving the legacy core `models.bedrockDiscovery` shape, while a generic `models.costs` map would need a broader config decision.
