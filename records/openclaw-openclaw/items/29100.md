## Summary

Keep open. Current main still has no supported pricing overlay for discovered Bedrock models: the plugin schema, typed config, docs, and discovery conversion path omit per-model cost metadata, discovered models still get zero-cost metadata, and the related implementation PR is closed unmerged; the display side also overlaps open issue #53286.

## What This Changes

Allow Amazon Bedrock auto-discovery config to attach per-model cost metadata without duplicating an explicit provider/model block.

## Best Possible Solution

Land a backward-compatible, plugin-owned Bedrock discovery pricing overlay that applies configured USD-per-1M-token rates to discovered foundation and inference-profile IDs, includes pricing in the cache key, updates schema/docs/tests, and coordinates with the aws-sdk display fix.

## Reproduction Assessment

Yes. Source inspection of current main is enough: the strict Bedrock discovery config accepts no costs field, and discoverBedrockModels assigns DEFAULT_COST to discovered foundation models and inherited profiles without requiring live AWS access.

## Solution Assessment

Yes. A plugin-owned discovery pricing overlay is the narrow maintainable path; duplicating explicit provider blocks preserves today’s workaround but conflicts with the auto-discovery flow, and hard-coding AWS pricing catalogs would be broader than requested.
