## Summary

Keep open. Current main and v2026.6.6 still have no supported pricing overlay for auto-discovered Bedrock models: discovery emits zero-cost model definitions, the plugin discovery schema has no `costs` field, and the only PR that attempted this exact feature was closed unmerged.

## What This Changes

Request to add cost metadata for auto-discovered Amazon Bedrock models so usage surfaces can estimate cost without duplicating the provider model list.

## Best Possible Solution

Approve one canonical pricing-overlay shape, then implement it at the Bedrock plugin discovery boundary or as a generic model-pricing overlay with schema, docs, discovery-cache, and cost-display tests.

## Reproduction Assessment

Yes. Source inspection shows Bedrock discovery still emits zero-cost model definitions and the plugin discovery schema/docs expose no pricing overlay field.

## Solution Assessment

Unclear. A plugin-owned discovery-cost overlay is plausible, but maintainers should first choose between that narrower shape and the broader `models.costs` overlay suggested in the issue.
