## Summary

Keep open. Current main and the latest release still lack a supported cost overlay for auto-discovered Amazon Bedrock models; the direct implementation PR was closed unmerged, and the remaining `aws-sdk` cost-display behavior is tracked separately.

## What This Changes

Allow users to attach per-model cost metadata to auto-discovered Amazon Bedrock models without duplicating an explicit provider model list.

## Best Possible Solution

Add a plugin-owned `plugins.entries.amazon-bedrock.config.discovery.costs` overlay, apply it during Bedrock discovery, include it in the discovery cache key, update docs/tests, and leave the `aws-sdk` display gate to the separate issue.

## Reproduction Assessment

Yes. Source inspection is enough: the strict plugin schema and shared type omit a pricing map, and discovered Bedrock models are assigned zero-cost metadata today.

## Solution Assessment

Yes for the plugin-owned overlay direction. A top-level `models.costs` map would be broader product surface and should wait for a separate maintainer decision.
