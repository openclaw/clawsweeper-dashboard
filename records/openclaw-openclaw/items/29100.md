## Summary

Keep open. Current main still lacks a supported cost metadata overlay for Bedrock auto-discovered models; https://github.com/openclaw/openclaw/pull/31160 attempted this exact change but was closed unmerged, and https://github.com/openclaw/openclaw/issues/53286 tracks a separate explicit-provider aws-sdk display path rather than superseding this request.

## What This Changes

The issue requests a Bedrock discovery cost overlay so IAM/auto-discovered models can show USD estimates without duplicating provider model entries.

## Best Possible Solution

Add a plugin-owned Bedrock discovery cost overlay under the current discovery config, apply it to discovered foundation models and inference profiles, include pricing in discovery cache identity, and document the USD-per-1M-token units.

## Reproduction Assessment

Yes. Source inspection on current main is enough: the Bedrock discovery schema and BedrockDiscoveryConfig omit costs, while discovered models are assigned DEFAULT_COST metadata.

## Solution Assessment

Yes, if maintainers approve the new config surface. A Bedrock plugin-owned overlay is the narrow maintainable direction; a top-level models.costs map would be a broader cross-provider product decision.
