## Summary

Keep open. Current main still lacks a supported cost overlay for auto-discovered Amazon Bedrock models: plugin discovery emits zero-cost metadata, the plugin discovery schema has no `costs` field, and the latest release has the same behavior.

## What This Changes

Request to attach per-model cost metadata to auto-discovered Amazon Bedrock models without duplicating explicit provider model entries.

## Best Possible Solution

Choose one canonical pricing-overlay API, preferably under the current plugin-owned Bedrock discovery config unless maintainers want a broader `models.costs` design, then add schema, docs, discovery cache-key handling, and focused cost-display tests.

## Reproduction Assessment

Yes. Source inspection on current main shows Bedrock discovery only accepts sizing/filter fields, discovered models receive `DEFAULT_COST`, and docs only document explicit model-entry pricing.

## Solution Assessment

Unclear as proposed. The user problem is valid, but the best fix needs maintainer direction on whether the overlay belongs in plugin-owned Bedrock discovery config or in a generic cross-provider pricing map.
