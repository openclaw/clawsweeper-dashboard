## Summary

Keep open. Current main and v2026.5.12 still do not provide a supported costs overlay for Bedrock-discovered models, and the prior implementation PR was closed unmerged rather than replaced by a landed fix.

## What This Changes

The issue requests a way to attach per-model cost metadata to Amazon Bedrock models discovered by the Bedrock plugin without duplicating an explicit provider model list.

## Best Possible Solution

Add a plugin-owned Bedrock discovery costs overlay, apply it to discovered foundation models and inference profiles, include it in the discovery cache key, and document the USD-per-1M-token units.

## Reproduction Assessment

Yes. Source inspection is enough: the strict plugin schema and shared BedrockDiscoveryConfig type omit a costs field, and current discovery assigns zero-cost metadata to discovered models.

## Solution Assessment

Yes, if the new surface is approved. A plugin-owned discovery overlay is narrower than a top-level models.costs map, while the related aws-sdk display-policy work should remain separately tracked.
