## Summary

Keep open. Current main and the latest release still do not provide a cost overlay for auto-discovered Bedrock models: discovery emits zero-cost model definitions, and local pricing is still only resolved from explicit provider model config or existing pricing caches.

## What This Changes

The issue requests a supported way to attach cost metadata to auto-discovered Amazon Bedrock models without duplicating an explicit provider model block.

## Best Possible Solution

Choose one canonical pricing-overlay surface, then implement it at the Bedrock plugin discovery boundary or as a generic model-pricing overlay with schema, docs, cache-key handling, and focused cost-display coverage.

## Reproduction Assessment

Yes at source level: the current Bedrock discovery path assigns zero-cost model definitions and the plugin discovery schema does not expose a cost overlay. I did not run live AWS discovery in this read-only review.

## Solution Assessment

Unclear until maintainers choose the API direction. The legacy `models.bedrockDiscovery.costs` path is no longer the best target by itself because current main migrated discovery config to the Amazon Bedrock plugin-owned config surface.
