## Summary

Keep open. Current main still lacks a supported pricing overlay for auto-discovered Amazon Bedrock models: the strict plugin discovery schema, typed discovery config, docs, cache key, and discovery conversion path all omit a per-model cost map, and discovered foundation/profile models still receive zero-cost metadata. The related implementation PR #31160 is closed unmerged, and #53286 remains open for the separate aws-sdk cost-display gate.

## What This Changes

Issue requests cost metadata overlays for auto-discovered Amazon Bedrock models so `/status` and `/usage` can estimate costs without duplicating explicit provider model entries.

## Best Possible Solution

Land a backward-compatible, plugin-owned Bedrock discovery pricing overlay that applies configured USD-per-1M-token rates to discovered foundation model IDs and inference profile IDs, includes normalized pricing in the discovery cache key, updates schema/help/docs/tests, and either coordinates with or includes the aws-sdk display fix so configured Bedrock costs are visible.

## Reproduction Assessment

Yes. A static current-main reproduction is enough: configure Bedrock discovery and inspect the strict discovery schema plus `discoverBedrockModels`; there is no accepted costs field and discovered models are assigned all-zero cost metadata without requiring live AWS access.

## Solution Assessment

Yes. Keeping this open for a focused implementation PR is the best path; the narrow maintainable direction is plugin-owned discovery pricing with legacy compatibility only where needed, not duplicating entire explicit provider blocks or hard-coding AWS pricing catalogs.

## Repair Work Prompt

Implement cost metadata support for auto-discovered Amazon Bedrock models. Add a backward-compatible pricing overlay under `plugins.entries.amazon-bedrock.config.discovery.costs` keyed by model ID, with legacy `models.bedrockDiscovery.costs` compatibility only where the existing migration/runtime fallback needs it. Normalize and validate cost entries as USD per 1M tokens with `input`, `output`, `cacheRead`, and `cacheWrite`; partial entries should default missing rates to zero. Apply configured costs to discovered foundation model IDs and inference profile IDs, including profile IDs that map to a priced backing foundation model. Include normalized pricing in the Bedrock discovery cache key so changed pricing does not reuse stale cached model definitions. Update `extensions/amazon-bedrock/openclaw.plugin.json`, config types/schema/help/docs, and focused tests. Coordinate with #53286 so configured Bedrock costs are displayed for aws-sdk-auth status and usage surfaces, or clearly include that display fix in the same PR. Do not change AWS credential resolution, request routing, model selection, dependency versions, or hard-code AWS pricing catalogs. Add a user-facing changelog entry crediting @mg-mavios, and credit @Pan-Binghong too if the aws-sdk display fix is included.
