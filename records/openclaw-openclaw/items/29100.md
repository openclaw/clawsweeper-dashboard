## Summary

Keep open. Current main and the latest release still lack a supported pricing overlay for auto-discovered Bedrock models, the direct implementation PR was closed unmerged, and the related explicit-cost display issue does not cover discovered-model pricing metadata.

## What This Changes

The issue requests a config path for attaching token cost metadata to auto-discovered Amazon Bedrock models without duplicating explicit provider model entries.

## Best Possible Solution

Approve and implement a Bedrock plugin-owned discovery costs overlay with schema, types, docs, focused discovery tests, and deterministic cache-key pricing coverage; treat a repo-wide model-cost overlay as separate product work.

## Reproduction Assessment

Yes, at source level. Current main's strict Bedrock discovery schema/type omit any cost overlay, and discovery emits DEFAULT_COST for foundation and inference-profile models with no user pricing path.

## Solution Assessment

Unclear until maintainers choose the public config shape. A Bedrock plugin-owned overlay under plugins.entries.amazon-bedrock.config.discovery is the narrowest maintainable direction; a repo-wide models.costs map is broader product work.
