## Summary

Keep open. Current main still has no supported pricing overlay for auto-discovered Amazon Bedrock models: the plugin discovery schema and typed discovery config omit a costs field, and discovered foundation models still receive zero-cost metadata.

## What This Changes

Add a Bedrock discovery pricing overlay so auto-discovered Amazon Bedrock model IDs can carry per-token cost metadata without duplicating explicit provider models.

## Best Possible Solution

Land a backward-compatible, Amazon Bedrock plugin-owned discovery costs overlay that enriches discovered foundation and inference-profile models, includes pricing in discovery cache identity, updates schema/docs/tests/changelog, and coordinates the aws-sdk cost-display path tracked by #53286.

## Reproduction Assessment

Yes. Source inspection on current main is enough: the strict plugin discovery schema rejects a costs field and discoverBedrockModels assigns DEFAULT_COST to discovered models without a pricing overlay path.

## Solution Assessment

Yes. A plugin-owned overlay is the narrow maintainable direction under the provider boundary; duplicating explicit provider blocks is the current workaround, and hard-coding an AWS pricing catalog would be broader than requested.

## Repair Work Prompt

Implement the linked feature as a backward-compatible Amazon Bedrock plugin discovery pricing overlay. Add a typed `costs` map under `plugins.entries.amazon-bedrock.config.discovery` and preserve legacy `models.bedrockDiscovery` compatibility if that legacy object is migrated. Apply configured USD-per-1M-token values to discovered foundation model IDs and inference profile IDs without changing model routing or AWS discovery calls; unspecified cost fields should default consistently with existing model cost normalization. Include the costs map in the discovery cache key so different pricing overlays do not reuse stale cached model metadata. Update plugin schema/UI hints, relevant config/types exports, docs, generated config artifacts if required, and CHANGELOG.md. Add focused tests for full, partial, missing, foundation-model, inference-profile, and cache-key behavior. Also inspect #53286: either include a narrow aws-sdk display fix so configured Bedrock costs can appear in status/usage, or explicitly leave that as a coordinated follow-up if the maintainer prefers to keep display behavior separate. Do not hard-code AWS pricing catalogs or move provider-specific behavior into core.
