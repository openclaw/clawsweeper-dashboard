## Summary

Keep open. The PR contains useful Control UI and Slack hardening that is not present on current main, but it is not mergeable as-is because it introduces a security-sensitive media-link regression and the live PR branch is dirty with maintainer edits disabled.

## What This Changes

The PR adds Control UI auth/media hardening, build provenance and checksum artifacts, Slack/channel health diagnostics, docs/i18n updates, workflow wiring, and regression tests.

## Best Possible Solution

Keep the hardening direction, but revise local attachment rendering so non-media documents stay on the authenticated Gateway route or become forced downloads, then refresh the branch on current main and run exact-head CI.

## Reproduction Assessment

Not applicable as a user bug report. The blocking review finding has a high-confidence static reproduction path: compare the PR head document-link rendering with current main and inspect the Gateway assistant-media security headers.

## Solution Assessment

No, not as currently implemented. Authorization-backed blob fetches are appropriate for validated media previews, but using blob URLs for document links is broader than the hardening goal and drops server-side response protections.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Keep document attachments on the hardened media route:** `ui/src/ui/chat/grouped-render.ts:1213`
  - body: Local document attachments now link to a client-created `blob:` URL. That bypasses the Gateway assistant-media response headers (`Content-Security-Policy`, `X-Content-Type-Options`, and `Referrer-Policy`) that protected the old authenticated route, so a local HTML/SVG file under an allowed preview root can be opened without those safeguards. Keep non-media links on the authenticated route or force a safe download, and reserve blob URLs for validated media previews.
  - confidence: 0.87
