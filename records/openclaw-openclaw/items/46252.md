## Summary

Keep open: current main counts reset/deleted archives for global usage totals, but per-session `sessions.usage` still selects one active transcript and does not aggregate same-stem `.jsonl.reset.*` archives, matching the reporter-confirmed undercount path.

## What This Changes

The issue asks for cost and usage aggregation to include `.jsonl.reset.*` archives created by `/new` or `/reset` so dashboard spend is not undercounted.

## Best Possible Solution

Aggregate all usage-counted transcript files that parse to the same session id in backend session usage summaries/cache, preserve a canonical transcript path for detail navigation, and keep `.jsonl.bak.*` excluded.

## Reproduction Assessment

Yes, source-reproducible. Create an active `sess.jsonl` and `sess.jsonl.reset.<timestamp>` with usage entries under one agent sessions directory, then request `sessions.usage`; current main resolves and summarizes only the active file for that session id.

## Solution Assessment

No, current main is still only a partial solution. The maintainable fix belongs in backend session usage aggregation/cache so session rows and totals stay consistent, not in a UI-only override.
