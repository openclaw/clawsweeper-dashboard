## Label Justifications

- `P2`: The bug can substantially understate spend for reset-heavy sessions, but it does not prevent agent execution or make the core runtime unavailable.
- `impact:session-state`: The accounting error is caused by archived transcript files not being associated with the current logical session lineage.

## Summary

Current main only partially addresses the report: global date-range totals include reset/deleted archives, but same-session detail still prefers the active transcript and omits its archived predecessors, matching the reporter’s confirmed `sessions.usage` undercount. Both linked fix PRs are closed unmerged, so no viable open implementation currently owns the remaining work.

## What This Changes

Aggregate active and archived reset transcripts for the same logical session in cost and usage reporting.

## Best Possible Solution

Add one canonical lineage-file resolver that returns ordered, deduplicated active plus same-stem reset/deleted transcripts, then reuse it for cached summaries, time-series, and logs while retaining the active transcript as the displayed canonical file.

## Reproduction Assessment

Yes—source inspection gives a high-confidence reproduction: create both `<id>.jsonl` and `<id>.jsonl.reset.<timestamp>`, then request per-session usage; the resolver selects the primary file and each detail loader scans only that file. The reporter also supplied concrete runtime values confirming this exact path.

## Solution Assessment

Yes, the best fix is a shared lineage-aware file-selection primitive rather than another caller-specific glob expansion, because summary, cache, time-series, and logs must apply identical ordering and deduplication rules.

Fix the same-lineage usage undercount described in https://github.com/openclaw/openclaw/issues/46252. Current main includes reset/deleted archives in broad totals, but `resolveExistingUsageSessionFile` prefers an existing active `.jsonl`, and `loadSessionCostSummary`, `loadSessionUsageTimeSeries`, `loadSessionLogs`, plus cache-backed summaries process only that file. Introduce one canonical helper that resolves the ordered, deduplicated usage transcript files for a logical session. Keep the active transcript as the returned/displayed canonical `sessionFile`, but aggregate same-stem `.jsonl.reset.*` and `.jsonl.deleted.*` predecessors. Exclude `.bak` and compaction snapshots to prevent double-counting. Add focused regression coverage for active-plus-reset aggregation, archive-only fallback, chronological ordering, cache reads, and backup exclusion. Keep the scope to established reset/deleted same-stem artifacts; do not add config, speculative checkpoint/trajectory aliases, or CHANGELOG.md edits.
