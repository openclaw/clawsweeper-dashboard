## Summary

Keep open: current main and the latest release still undercount per-session detail when an active transcript exists beside reset/deleted archives, and https://github.com/openclaw/openclaw/pull/92610 is the open closing PR that now owns the remaining fix.

## What This Changes

The issue asks the cost dashboard and session usage detail to count reset/deleted archived JSONL transcripts alongside active session JSONL files so `/new` rotations do not hide spend.

## Best Possible Solution

Keep this issue open until https://github.com/openclaw/openclaw/pull/92610 or an equivalent current-main fix lands, using the canonical usage-counted filename helpers to aggregate active plus reset/deleted lineage without double-counting backups/checkpoints.

## Reproduction Assessment

Yes. Source inspection shows current main resolves the active `<id>.jsonl` and then scans only that file for per-session summary, time series, and logs, so same-stem `.jsonl.reset.*` usage is omitted when the active file exists.

## Solution Assessment

Yes. The best path is to review and land the linked PR or an equivalent owner-boundary fix in `src/infra/session-cost-usage.ts`; queuing a second repair now would duplicate active open work.
