## Summary

Keep this open: current main includes the earlier global archive scan fix, but the Usage dashboard still has a source-reproducible undercount path when an active `.jsonl` and `.jsonl.reset.*` archive share the same session stem. The linked PR #46605 was closed unmerged, and the later merged work only partially covers the reporter's updated reproduction.

## What This Changes

The issue asks for cost and usage aggregation to include `.jsonl.reset.*` archived session files created by `/new` or `/reset`.

## Best Possible Solution

Make session usage totals aggregate all usage-counted transcript files for the same session stem while keeping `.bak` archives excluded and preserving a clear canonical file for logs/detail navigation.

## Reproduction Assessment

Yes, source-reproducible. Create `sess.jsonl` and `sess.jsonl.reset.<timestamp>` with usage entries under the same sessions directory, call `sessions.usage`, and current main will keep the active file for that session while excluding the reset archive from the returned session aggregate.

## Solution Assessment

No, the current partial fix is not the best complete solution. `usage.cost` now includes archives globally, but `sessions.usage` and the dashboard headline totals need same-stem active-plus-archive aggregation or an equivalent consistent total source.
