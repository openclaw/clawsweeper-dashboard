## Summary

Keep open: current main and the latest release still omit same-stem reset/deleted archives from active per-session usage detail, and https://github.com/openclaw/openclaw/pull/92610 is the open closing fix path.

## What This Changes

The issue asks OpenClaw usage/cost reporting to include archived session transcript files created by `/new` or `/reset` so daily and per-session spend is not undercounted.

## Best Possible Solution

Aggregate active plus same-stem reset/deleted transcripts for per-session summary, time-series, logs, and cache-backed detail while preserving the canonical usage-counted filename contract.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: create an active `<id>.jsonl` beside same-stem `.jsonl.reset.*` or `.jsonl.deleted.*` archives, then request per-session usage detail; current main resolves and scans only the active file.

## Solution Assessment

Yes for the linked PR direction. Reusing the canonical usage-counted filename helpers in the session-cost loader/cache paths is narrower and safer than broad prefix matching; the issue should remain open until the PR lands or maintainers choose a replacement.
