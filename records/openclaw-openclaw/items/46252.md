## Summary

Keep open: current main fixes broad archive scanning for global cost totals, but per-session `sessions.usage` still resolves a same-stem active transcript plus `.jsonl.reset.*` archives to one file, so the reporter-confirmed undercount can still happen.

## What This Changes

The issue asks OpenClaw usage and cost reporting to count `.jsonl.reset.*` archive transcripts together with active `.jsonl` files so `/new` and `/reset` rotations do not underreport spend.

## Best Possible Solution

Aggregate all usage-counted transcript files that parse to the same session id in backend session usage summaries/cache, preserve a canonical transcript for detail navigation, and keep `.jsonl.bak.*` excluded.

## Reproduction Assessment

Yes, source-reproducible. Create an active `sess.jsonl` and `sess.jsonl.reset.<timestamp>` with usage entries under one agent sessions directory, then request `sessions.usage`; current main resolves and summarizes only the active file for that session id.

## Solution Assessment

No, current main is still only a partial solution. The maintainable fix belongs in backend session usage aggregation/cache so session rows and headline totals stay consistent, not in a UI-only total override.
