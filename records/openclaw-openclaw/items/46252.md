## Summary

Keep open: current main and the latest release include reset/deleted archives in broad usage scans, but same-stem active-plus-reset `sessions.usage` still collapses to one transcript and can undercount the reporter-confirmed per-session path.

## What This Changes

The issue asks for usage and cost reporting to include `.jsonl.reset.*` archives created by `/new` or `/reset` so daily spend is not undercounted.

## Best Possible Solution

Aggregate every usage-counted transcript file that parses to the same session id in backend session usage summaries/cache, keep a canonical path for detail navigation, and continue excluding `.jsonl.bak.*` and checkpoint artifacts.

## Reproduction Assessment

Yes, source-reproducible. With an active `<id>.jsonl` and same-stem `<id>.jsonl.reset.<timestamp>`, current main returns and summarizes one active transcript path, matching the reporter's retest.

## Solution Assessment

Yes, the best fix is a backend aggregation/cache change rather than a UI-only glob change. The current implementation is only a partial fix because broad scans include archives while same-stem per-session summaries still do not merge them.
