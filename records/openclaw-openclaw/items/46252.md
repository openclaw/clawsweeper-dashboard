## Summary

Keep open: current main includes archive-aware global cost scanning and lineage rollups, but the same-stem active `.jsonl` plus `.jsonl.reset.*` path still resolves to one transcript file for `sessions.usage`, and the Usage UI headline uses those session totals.

## What This Changes

The issue asks the cost dashboard and usage summaries to count `.jsonl.reset.<timestamp>` archive transcripts alongside active `.jsonl` session files.

## Best Possible Solution

Aggregate all usage-counted transcript files that parse to the same session id in backend session usage summaries/cache, preserve a canonical transcript for detail navigation, and keep `.jsonl.bak.*` excluded.

## Reproduction Assessment

Yes, source-reproducible. Create an active `sess.jsonl` and `sess.jsonl.reset.<timestamp>` with usage entries under one agent sessions directory, then request `sessions.usage`; current main resolves and summarizes only the active file for that session id.

## Solution Assessment

No, the current main implementation is still only a partial solution for this issue. The narrow maintainable fix is in backend session usage aggregation/cache, not a UI-only total override, so session rows and headline totals stay consistent.
