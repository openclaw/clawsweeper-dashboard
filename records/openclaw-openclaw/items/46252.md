## Summary

Keep open: current main and v2026.6.6 still resolve per-session usage detail to the active transcript when one exists, so same-stem reset/deleted archives remain omitted from the detail path; https://github.com/openclaw/openclaw/pull/92610 is the open closing PR that owns the remaining fix.

## What This Changes

The issue requests per-session cost/usage aggregation include same-stem reset/deleted archives created by `/new` or `/reset`, not only the active `.jsonl` transcript.

## Best Possible Solution

Review and land, revise, or close https://github.com/openclaw/openclaw/pull/92610; the end state should aggregate active plus same-stem reset/deleted transcripts for per-session summary, time-series, logs, and cache-backed detail while preserving the canonical archive filename contract.

## Reproduction Assessment

Yes, source-reproducible: create an active `<id>.jsonl` beside same-stem `.jsonl.reset.*` or `.jsonl.deleted.*` archives and request per-session usage detail; current main resolves and scans only the active file. I did not run a write-based repro because this review is read-only, but the source path and issue comments give a high-confidence reproduction path.

## Solution Assessment

Yes, the linked PR direction is the best known fix shape: reuse the canonical usage-counted filename helpers and aggregate the resolved lineage in the session-cost loader/cache paths instead of broad prefix matching. The issue should remain open until that PR lands or maintainers choose a replacement.
