## Summary

Keep open: current main and the latest release include reset/deleted archives in broad usage totals, but same-stem active-plus-reset `sessions.usage` still resolves one transcript path and can undercount the reporter-confirmed per-session case.

## What This Changes

The issue reports that cost and usage views omit `.jsonl.reset.<timestamp>` session archives created by `/new` or `/reset`, causing understated spend.

## Best Possible Solution

Aggregate every usage-counted transcript that parses to the same session id in backend session usage summaries/cache, keep a canonical active path for detail navigation, and continue excluding `.jsonl.bak.*` and checkpoint artifacts.

## Reproduction Assessment

Yes, source-reproducible with high confidence. Current main returns the active `<id>.jsonl` before considering same-stem archives, then `sessions.usage` and the cache summarize that single path, matching the reporter's April 4 retest.

## Solution Assessment

Yes. The best fix is a backend aggregation/cache change rather than a UI-only glob change, because broad scans already include archives while per-session summaries still do not merge same-stem active and reset transcripts.
