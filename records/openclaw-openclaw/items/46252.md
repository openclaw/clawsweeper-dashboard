## Summary

Keep this open: current main includes the broad archive scan/cache work, but the same-stem active-plus-reset path still undercounts `sessions.usage`, and the Usage overview still uses those session totals for its headline cost.

## What This Changes

Include `.jsonl.reset.<timestamp>` archive files in Usage dashboard/session cost aggregation so `/new` or `/reset` spend is not undercounted.

## Best Possible Solution

Aggregate all usage-counted active/reset/deleted transcript files for the same session stem in backend session usage totals, preserve a canonical file for logs/detail navigation, and keep `.bak` archives excluded.

## Reproduction Assessment

Yes, source-reproducible. Create `sess.jsonl` and `sess.jsonl.reset.<timestamp>` with usage entries under the same agent sessions directory, call `sessions.usage`, and current main follows the active file while dropping the archive from the session aggregate.

## Solution Assessment

No, the current partial fix is not the complete solution. The narrow maintainable fix belongs in backend session usage aggregation, not as a UI-only total override, so session rows and headline totals stay consistent.
