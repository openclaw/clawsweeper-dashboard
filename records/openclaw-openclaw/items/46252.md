## Summary

Keep open: current main includes the global archive scan fix, but the per-session `sessions.usage` path still selects one transcript per session stem, so same-stem `.jsonl.reset.*` archives can be excluded from dashboard totals.

## What This Changes

The issue asks the Usage dashboard and cost summaries to include `.jsonl.reset.<timestamp>` session archives created by `/new` or `/reset`.

## Best Possible Solution

Make `sessions.usage` totals aggregate all usage-counted same-stem transcripts while keeping `.bak` excluded and preserving a canonical file for detail views.

## Reproduction Assessment

Yes, source-reproducible. A sessions directory with `sess.jsonl` and `sess.jsonl.reset.<timestamp>` follows code paths that discover one `sess` entry, prefer the active file, and load one per-session summary.

## Solution Assessment

No, the current partial fix is not the best complete solution. `usage.cost` includes archives globally, but `sessions.usage` and dashboard headline totals need same-stem active-plus-archive aggregation or a consistent aggregate source.
