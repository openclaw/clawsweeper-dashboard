## Summary

Keep open: current main and the latest release include reset/deleted archives in broad usage totals, but the same-stem active-plus-reset `sessions.usage` path still resolves and summarizes one transcript, matching the reporter's confirmed undercount.

## What This Changes

This issue asks for cost and usage aggregation to include `.jsonl.reset.*` archive transcripts so `/new` and `/reset` do not undercount spend.

## Best Possible Solution

Aggregate every usage-counted transcript that parses to the same session id in backend session usage summaries/cache, keep a canonical active path for navigation, and continue excluding `.jsonl.bak.*` and checkpoint artifacts.

## Reproduction Assessment

Yes. Source inspection shows current main returns the active `<id>.jsonl` before considering same-stem archives, then `sessions.usage` summarizes that single path, matching the reporter's April 4 retest.

## Solution Assessment

Yes. The narrow maintainable fix is a backend aggregation/cache change rather than a UI-only glob change, because broad scans already include archives while per-session summaries do not merge same-stem transcripts.
