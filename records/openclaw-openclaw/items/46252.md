## Summary

Keep open: current main has the broad archive-aware `usage.cost` scan, but per-session `sessions.usage` still resolves same-stem active-plus-reset transcripts to one file and can undercount the reporter-confirmed path.

## What This Changes

The issue asks for cost and usage aggregation to include `.jsonl.reset.*` archived transcripts alongside active `.jsonl` files so `/new` and `/reset` sessions are not undercounted.

## Best Possible Solution

Aggregate every usage-counted transcript file that parses to the same session id in backend session usage summaries/cache, keep a canonical path for detail navigation, and continue excluding `.jsonl.bak.*`.

## Reproduction Assessment

Yes, source-reproducible. Create an active `sess.jsonl` and `sess.jsonl.reset.<timestamp>` with usage entries under one agent sessions directory, then request `sessions.usage`; current main resolves and summarizes only one file for that session id.

## Solution Assessment

No, current main is only a partial solution. The maintainable fix should be in backend session usage aggregation/cache rather than a UI-only override or another file-glob patch.
