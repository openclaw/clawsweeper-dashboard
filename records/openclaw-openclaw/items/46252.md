## Summary

Keep open: current main and the latest release count reset/deleted archives in the global usage aggregate, but the per-session detail path still resolves the active transcript first and scans one file, so same-stem reset archives remain omitted until the open closing PR lands.

## What This Changes

The issue requests that cost and usage views include reset/deleted JSONL transcript archives, especially same-stem archives left by /new or /reset, instead of counting only active session files.

## Best Possible Solution

Review, refresh, and land or intentionally close https://github.com/openclaw/openclaw/pull/92610; after a merged fix, close this issue against that commit.

## Reproduction Assessment

Yes, there is a high-confidence source reproduction path: create an active <id>.jsonl plus same-stem .jsonl.reset.* usage file, then request per-session usage detail; current main resolves and scans only the active file.

## Solution Assessment

Yes, the best path is to aggregate active plus same-stem reset/deleted lineage files in the per-session loader/cache paths while excluding backup and checkpoint artifacts; the open closing PR is the current implementation candidate.
