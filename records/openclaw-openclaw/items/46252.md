## Summary

Keep open: current main and the latest shipped release have archive-aware global usage scans, but `sessions.usage` still resolves same-stem active-plus-reset transcripts to one file and can undercount the reporter-confirmed per-session path.

## What This Changes

The issue requests that cost and usage aggregation include `.jsonl.reset.<timestamp>` archive transcripts created by `/new` or `/reset` so spend is not undercounted.

## Best Possible Solution

Aggregate every usage-counted transcript file that parses to the same session id in backend session usage summaries/cache, keep a canonical path for detail navigation, and continue excluding `.jsonl.bak.*`.

## Reproduction Assessment

Yes, source-reproducible. Create an active `sess.jsonl` and `sess.jsonl.reset.<timestamp>` with usage entries under one agent sessions directory, then request `sessions.usage`; current main resolves and summarizes one file for that parsed session id.

## Solution Assessment

Yes, the best fix is a backend aggregation/cache change, not a UI-only patch. The current implementation is only a partial solution because it includes archives in broad scans but not same-stem per-session summaries.
