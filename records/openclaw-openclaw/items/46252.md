## Summary

Keep open. Current main has the archive-aware global cost scan, but the per-session usage path still resolves a session id to one transcript and returns the active `.jsonl` before considering reset/deleted archives, so the active-plus-reset undercount reported in the discussion remains valid.

## What This Changes

The issue requests that the cost dashboard and related usage summaries include `.jsonl.reset.<timestamp>` session archive files alongside active `.jsonl` transcripts after `/new` or `/reset`.

## Best Possible Solution

Add a shared per-session transcript-segment resolver that returns every usage-counted segment for a session id: active `.jsonl` plus `.jsonl.reset.*` and `.jsonl.deleted.*`. Use that segment list for summaries, timeseries, logs, exports, dashboard totals, and overview cards while continuing to exclude `.jsonl.bak.*`, checkpoint, and trajectory artifacts.

## Reproduction Assessment

Yes. The issue gives a concrete active-plus-reset file pair with expected and observed costs, and current source statically reproduces the gap because `loadSessionCostSummary({ sessionId })` resolves and scans only one transcript when the active file exists.

## Solution Assessment

Yes. The best fix is a narrow shared resolver/aggregation change in `src/infra/session-cost-usage.ts` with gateway tests; a UI-only adjustment would leave RPC totals, detail views, exports, and logs inconsistent.
