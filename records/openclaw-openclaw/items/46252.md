## Summary

Keep open: current main includes the global archive scan fix, but the per-session `sessions.usage` path still collapses same-stem active and reset/deleted transcripts to one active file, so the Usage dashboard can still undercount the reporter's updated reproduction.

## What This Changes

Cost dashboard and `sessions.usage` should include `.jsonl.reset.*` archive transcripts when calculating daily and per-session spend after `/new` or `/reset`.

## Best Possible Solution

Aggregate all usage-counted transcripts for a session stem in `sessions.usage` and dashboard totals, while preserving a canonical active/latest file for detail views and keeping `.jsonl.bak.*` excluded.

## Reproduction Assessment

Yes, source-reproducible. With `sess.jsonl` and `sess.jsonl.reset.<timestamp>` under the same agent, current main discovers one `sess` entry and scans only the selected active `sessionFile` for `sessions.usage` totals.

## Solution Assessment

No, the current partial fix is not the complete solution. The narrow maintainable fix is same-stem aggregation in the per-session usage path, not broadening backup archive counting or changing provider billing logic.
