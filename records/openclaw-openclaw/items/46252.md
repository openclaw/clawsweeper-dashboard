## Summary

Keep open: current main and v2026.6.6 still scan only one resolved active transcript for per-session usage detail when that file exists, so same-stem reset/deleted archives remain omitted. The open closing fix path is https://github.com/openclaw/openclaw/pull/92610.

## What This Changes

The issue reports that cost and usage reporting undercounts sessions rotated by `/new` or `/reset` because reset archive transcript files are not included with the active session lineage.

## Best Possible Solution

Review and land, revise, or close https://github.com/openclaw/openclaw/pull/92610; the end state should aggregate active plus same-stem reset/deleted transcripts for per-session summary, time-series, logs, and cache-backed detail while preserving the canonical archive filename contract.

## Reproduction Assessment

Yes, source-reproducible: create an active `<id>.jsonl` beside same-stem `.jsonl.reset.*` or `.jsonl.deleted.*` archives and request per-session usage detail; current main resolves and scans only the active file. I did not run a write-based repro because this review is read-only, but the issue comments and current code path give high confidence.

## Solution Assessment

Yes, the linked PR direction is the best known fix shape: reuse the canonical usage-counted filename helpers and aggregate the resolved lineage in the session-cost loader/cache paths instead of broad prefix matching. The issue should stay open until that PR lands or maintainers choose a replacement.
