## Summary

Keep open. Current main and v2026.6.8 still have CLI backup create/verify and narrow Control UI agent-file editing only, with no Web UI backup/restore flow; related workspace and native-backup items overlap but do not settle this UI product/security decision.

## What This Changes

The issue requests backup and restore buttons or a Web UI flow for users who cannot access the command line.

## Best Possible Solution

Design a security-reviewed Control UI export/import/restore flow that reuses the existing backup manifest logic, defines restore scope explicitly, and coordinates with the native backup umbrella and workspace-file-management track.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing contract. Source and release inspection give a high-confidence feature-gap check: backup is CLI create/verify only, and the Web UI exposes only narrow agent file editing/session file browsing.

## Solution Assessment

No, adding buttons alone is not the best way to solve this. The maintainable path needs a reviewed Gateway/API and restore policy that defines archive trust, scope, secret handling, destructive writes, rollback, and how it relates to existing backup/workspace-file efforts.
