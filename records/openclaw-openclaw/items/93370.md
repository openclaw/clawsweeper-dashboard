## Summary

Keep open: current main and v2026.6.9 still provide local CLI backup create/verify plus a narrow Control UI bootstrap-file editor, but no browser backup/restore flow; this remains product- and security-sensitive work rather than conservative cleanup.

## What This Changes

The issue requests browser-accessible backup and restore controls for users who cannot access the CLI.

## Best Possible Solution

Design a maintainer-approved Control UI export/import flow backed by a native restore/import contract, with explicit archive scope, authorization, validation, rollback/audit behavior, and coordination with the native backup and workspace-file-management threads.

## Solution Assessment

No, adding buttons alone is not the best way to solve the issue. The safer path is a maintainer-approved export/import contract with explicit restore scope, archive validation, authorization, rollback, and coordination with open native backup and workspace-file-management work.
