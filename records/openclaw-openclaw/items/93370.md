## Summary

Keep open for maintainer review. Current main and v2026.6.6 provide local CLI backup create/verify plus a narrow allowlisted Control UI file surface, but there is no Web UI backup/restore flow for users without CLI access; related open issues cover the native restore lifecycle and remote workspace file management, but neither has landed the requested end-user UI behavior.

## What This Changes

Add Control UI backup and restore buttons for OpenClaw users who cannot use the CLI.

## Best Possible Solution

Define a security-reviewed native backup/restore contract first, then expose authenticated Control UI actions that call that contract for users without CLI access and link them to the existing remote-file-management direction where appropriate.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing contract. Source and docs give high-confidence absence proof for Web UI backup/restore on current main and v2026.6.6.

## Solution Assessment

No as stated: adding buttons alone is not the best solution until the underlying restore/import lifecycle is safe. The maintainable path is a vetted backend contract plus a Control UI wrapper and clear user-facing restore semantics.
