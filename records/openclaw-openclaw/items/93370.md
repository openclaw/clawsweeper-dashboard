## Summary

Keep open: current main and v2026.6.9 still provide CLI backup create/verify plus a narrow Control UI agent-file editor, but no browser backup/restore flow; the remaining work is product- and security-sensitive rather than conservative cleanup.

## What This Changes

The issue requests adding browser-accessible backup and restore controls for users without CLI access.

## Best Possible Solution

Design a maintainer-approved Control UI export/import flow backed by a native restore contract, with explicit archive scope, authorization, validation, rollback/audit behavior, and coordination with the related backup and workspace-management threads.

## Solution Assessment

No, adding buttons alone is not the best way to solve the issue. The safer path is a maintainer-approved export/import contract with explicit restore scope, archive validation, authorization, rollback, and audit behavior before exposing browser controls.
