## Summary

Keep open. Current main and v2026.6.8 still provide local CLI backup create/verify plus a narrow Control UI bootstrap-file editor, but no browser backup/restore flow; the overlapping workspace-file-management and backup lifecycle threads do not fully own Web UI restore/import for OpenClaw state.

## What This Changes

The issue requests Web UI backup and restore buttons for users who cannot access the CLI.

## Best Possible Solution

Design a security-reviewed Control UI export/import and restore flow that reuses the existing backup planning and verification primitives, defines restore scope explicitly, and coordinates any workspace-file browsing pieces with the workspace-management track.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing contract. Source and release checks give a high-confidence feature-gap check: backup is CLI create/verify only, and the Web UI exposes only narrow agent bootstrap-file editing.

## Solution Assessment

No as stated: adding buttons alone is not the best way to solve this because restore needs a secure gateway contract and product decision about what can be imported or overwritten. The safer path is an explicit backup/export plus restore/import design that reuses existing backup logic and coordinates with workspace file management.
