## Summary

Keep open: current main and v2026.6.9 still provide local CLI backup create/verify plus narrow Control UI agent-file/session-file browsing, but no browser-accessible backup/restore flow. The remaining work is product- and security-sensitive because restore/export authority can expose or overwrite config, credentials, sessions, and workspace data.

## What This Changes

The issue requests backup and restore controls in the OpenClaw Web UI for users without CLI access.

## Best Possible Solution

Design a security-reviewed Control UI export/import flow that defines archive scope, restore authority, validation, rollback, and coordination with the native backup lifecycle and workspace-management tracks.

## Reproduction Assessment

Not applicable: this is a feature request, not a failing current behavior. Source and release inspection give high-confidence absence proof that current main and v2026.6.9 have CLI backup create/verify and narrow UI file browsing, but no Web UI backup/restore flow.

## Solution Assessment

No: adding buttons alone is not the best way to solve this. The safer path is a maintainer-approved export/import contract with explicit restore scope, authorization, archive validation, rollback, and coordination with overlapping backup and workspace-file work.
