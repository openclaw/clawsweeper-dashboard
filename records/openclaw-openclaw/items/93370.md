## Summary

Keep open. Current main and v2026.6.6 have a CLI-only backup create/verify path and a narrow Control UI editor for allowlisted agent bootstrap files, but no Web UI backup/restore flow; the related workspace-file-management issue and open PR overlap but do not fully cover full OpenClaw backup/restore.

## What This Changes

The issue requests backup and restore controls in the OpenClaw Web UI for users without command-line access.

## Best Possible Solution

Design a security-reviewed Control UI backup/export and restore/import flow that reuses the existing backup planning/verification logic where possible, defines the restore scope explicitly, and coordinates any workspace-file browser pieces with the existing workspace-management track.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing contract. Source inspection gives high-confidence absence proof on current main and v2026.6.6: backup is CLI create/verify only, while the Web UI exposes only narrow agent bootstrap-file editing.

## Solution Assessment

No, not as stated: adding buttons alone is not the best solution because restore needs a secure gateway contract and product decision about what can be imported or overwritten. The better path is an explicit backup/export plus restore/import design that reuses existing backup logic and coordinates with the workspace-file-management issue.
