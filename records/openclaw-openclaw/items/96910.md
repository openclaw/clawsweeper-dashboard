## What This Changes

The issue asks OpenClaw to add subjectAltName entries to auto-generated gateway TLS certificates so strict TLS clients can verify loopback or hostname connections.

## Best Possible Solution

Merge or reject the linked candidate fix after maintainers settle the generated SAN set and an explicit upgrade or doctor strategy for old OpenClaw-generated CN-only certs while preserving custom cert material.

## Reproduction Assessment

Yes, at source level. Current main generates a CN-only cert with no SAN, and Node's TLS identity check validates IP hostnames against SAN IP entries rather than the CN.

## Solution Assessment

Yes on the likely layer, but not fully settled. The fix belongs in the gateway TLS generator plus an upgrade-safe repair path; the exact SAN set and existing-cert replacement policy need maintainer/security acceptance.
