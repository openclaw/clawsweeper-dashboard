## Repair Work Prompt

Fix the Feishu streaming card regression reported in https://github.com/openclaw/openclaw/issues/90164. The official Feishu/Lark CardKit streaming text endpoint expects `content` to be the full cumulative text, but `extensions/feishu/src/streaming-card.ts` currently computes and sends only the unsent suffix via `resolveStreamingCardAppendContent`, and tests assert suffix bodies. Change `FeishuStreamingSession` so normal streaming updates and prefix-preserving final close updates send the cumulative text to `/elements/content/content`, while retaining the full element replace path for non-prefix rewrites such as removing transient status text. Update `extensions/feishu/src/streaming-card.test.ts` to expect full cumulative content and add/keep regression coverage for per-character or small-chunk streaming so the visible card content cannot collapse to the last chunk. Do not change the default `channels.feishu.streaming` behavior, add new config, or touch unrelated Feishu channel behavior. No changelog edit.
