## Summary

Keep open: this protected maintainer-labeled PR is a live narrow fix candidate for the linked yielded-parent wake bug. Source review found current main blocks the descendant-settle wake path for every `sessions_yield` pause, and the PR targets that gap without a line-level correctness finding, but real behavior proof is still test-only.

## What This Changes

The PR changes `resumeSubagentRun` so `sessions_yield`-paused parents marked `wakeOnDescendantSettle` can reach the existing descendant wake path, and adds a regression test for that case.

## Best Possible Solution

Have a maintainer review this narrow gate change against the linked bug and either accept the focused source proof or request redacted live/runtime proof before landing; keep the linked issue open until a fix merges.

## Reproduction Assessment

Yes, at source level: current main sets `wakeOnDescendantSettle` for deferred descendant cleanup but `resumeSubagentRun` returns early for every `sessions_yield` pause before the wake path can run. I did not run the live Telegram/github-copilot workflow in this read-only review.

## Solution Assessment

Yes, this appears to be the narrowest maintainable fix: ordinary yielded runs remain paused, while only yielded orchestrators marked for descendant-settle wake-up can reach the existing announce/wake path. The main alternative is the broader lifecycle PR, but that adds much larger public surface and is currently not a safe supersession.
