## Label Justifications

- `P1`: The original report described repeated gateway restarts, stalled health checks, and interrupted active sessions in normal use.
- `impact:crash-loop`: The reported failure included launchd KeepAlive restart loops and frequent SIGTERM gateway restarts.
- `impact:session-state`: The user-visible symptom was active webchat turns being interrupted by gateway restarts.
- `impact:auth-provider`: Provider auth pre-warm stalls and SecretRef resolution failures were central causes in the report.

## Summary

Current main and the shipped v2026.6.6 release cover the central gateway reliability problem: config reload planning is selective, no-op/self writes are skipped, provider auth pre-warm is delayed and worker-backed, and inactive SecretRefs no longer poison command-time `secrets.resolve`.

## What This Changes

The issue requests less aggressive Gateway config reloads, non-blocking provider auth pre-warm, and clearer/non-fatal handling for inactive SecretRefs during `secrets.resolve`.

## Best Possible Solution

Keep the shipped v2026.6.6 implementation; use the narrower Discord follow-up at https://github.com/openclaw/openclaw/issues/90865 for channel-specific post-start subscription behavior if that remains reproducible.

## Reproduction Assessment

No high-confidence current-main failure repro is available; the original v2026.5.22 failure is supported by concrete logs and the repaired current code paths are visible in source and tests.

## Solution Assessment

Yes. The current shipped solution addresses the right owner boundaries: reload planning stays in Gateway, provider auth warming runs in a bounded worker path, and inactive SecretRefs are handled by the secret-resolution layer.
