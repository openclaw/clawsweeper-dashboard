## Summary

Keep open: current main still lacks the requested per-server caller-context injection, and #75118 is an open, unmerged implementation candidate with closing syntax. The remaining decision is maintainer/security approval for forwarding caller and session identity headers to opted-in remote MCP URLs.

## What This Changes

Add a default-off per-server `mcp.servers.<name>.injectCallerContext` flag that forwards OpenClaw caller/session headers to opted-in remote bundle MCP servers without overwriting configured headers.

## Best Possible Solution

Review and land #75118 or an equivalent narrow implementation with config types/schema/docs, URL-only non-overwriting header injection, downstream flag stripping, placeholder env behavior, regression tests, changelog, and trusted-endpoint security wording.

## Reproduction Assessment

Not applicable as a feature request, but the absence is source-verifiable on current main. The typed config, schema, docs, bundle merge path, and CLI env preparation do not implement `injectCallerContext`.

## Solution Assessment

Yes, pending maintainer/security approval. A per-server, default-off, URL-only opt-in is narrower than a global switch and can preserve user header precedence while limiting identity disclosure.
