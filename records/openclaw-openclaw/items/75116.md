## Summary

Keep open: current main still lacks the requested per-server caller-context injection, and #75118 is an open, unmerged implementation candidate that explicitly closes this issue. The prior ClawSweeper comment correctly points to maintainer/security review rather than cleanup closure.

## What This Changes

Add a default-off `mcp.servers.<name>.injectCallerContext` option that forwards OpenClaw caller/session headers to opted-in remote MCP URL servers without overwriting user headers.

## Best Possible Solution

Review and land #75118 or an equivalent narrow change covering schema/types/docs, header injection, flag stripping, placeholder env population, tests, and changelog.

## Reproduction Assessment

Yes. Configure a remote `mcp.servers.<name>.url` entry with `injectCallerContext: true` and inspect the generated bundle MCP config; current source has no flag, injection branch, or non-loopback placeholder-env trigger.

## Solution Assessment

Yes, pending maintainer security approval. A per-server, default-off URL-only opt-in is the narrowest maintainable direction, provided it never overwrites user headers and strips the control flag before emitting downstream MCP config.
