## Summary

Keep open: current main still lacks the requested per-server caller-context injection, and #75118 is an open, unmerged implementation candidate with closing syntax for this issue.

## What This Changes

Add a per-server `mcp.servers.<name>.injectCallerContext` opt-in that forwards OpenClaw caller headers to remote bundle MCP servers without overwriting user headers.

## Best Possible Solution

Review and land #75118 or an equivalent narrow implementation with config type/schema/docs, default-off URL-only header injection, control-flag stripping, placeholder env coverage, tests, changelog, and trusted-endpoint security wording.

## Reproduction Assessment

Not applicable as a feature request, but the absence is source-verifiable on current main. The typed config, schema, docs, bundle merge path, and CLI env preparation do not implement the requested opt-in behavior.

## Solution Assessment

Yes, pending maintainer/security approval. A per-server, default-off, URL-only opt-in is narrower than a global switch and preserves user header precedence while avoiding broad identity disclosure.
