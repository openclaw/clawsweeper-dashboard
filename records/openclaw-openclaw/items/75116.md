## Summary

Keep open: current main still lacks the requested per-server caller-context injection behavior, and #75118 is an open, unmerged closing PR for this exact feature. The issue should stay open until maintainers review the caller-identity disclosure boundary and either land, replace, or close that PR.

## What This Changes

Add a default-off `mcp.servers.<name>.injectCallerContext` option for remote bundle MCP servers so trusted endpoints can receive OpenClaw caller/session headers without overwriting user-supplied headers.

## Best Possible Solution

Review #75118 for the trust boundary and implementation details, then land that PR or an equivalent narrow change if accepted: schema/types/docs plus default-off remote-only header injection, user-header precedence, downstream control-flag stripping, placeholder env population, focused tests, and changelog coverage.

## Reproduction Assessment

Yes. Configure a remote `mcp.servers.<name>.url` entry with `injectCallerContext: true` and inspect the generated bundle MCP config; source inspection shows current main has no typed/schema contract, injection branch, or non-loopback placeholder-env trigger.

## Solution Assessment

Yes, pending maintainer security approval. A per-server default-off opt-in is the narrow maintainable design for trusted internal endpoints; a global flag would carry broader accidental identity-disclosure risk.
