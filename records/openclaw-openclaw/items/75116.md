## Summary

Keep open: current main still lacks the requested per-server caller-context injection behavior, and #75118 is an open, unmerged closing PR for the same work, so this issue should stay open until that PR is reviewed, landed, or closed.

## What This Changes

Add an opt-in per-server `mcp.servers.<name>.injectCallerContext` flag for remote bundle MCP servers that forwards OpenClaw caller/session headers without overwriting user headers.

## Best Possible Solution

Review #75118 for the trust boundary and implementation details, then land that PR or an equivalent narrow change if accepted: schema/types/docs plus default-off remote-only header injection, user-header precedence, downstream control-flag stripping, placeholder env population, and focused tests.

## Reproduction Assessment

Yes. Configure a remote `mcp.servers.<name>.url` entry with `injectCallerContext: true` and inspect the generated bundle MCP config on current main; there is no injection branch, no typed/schema contract, and only the built-in loopback server gets caller header placeholders.

## Solution Assessment

Yes, pending maintainer security approval. A per-server default-off flag is the narrow maintainable approach; a global flag would create broader accidental identity-disclosure risk.
