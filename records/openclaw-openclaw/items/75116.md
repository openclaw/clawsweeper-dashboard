## Summary

Keep open: current main still lacks the requested per-server `injectCallerContext` option, and the linked closing PR remains open and unmerged, so this issue should stay open for maintainer/security review.

## What This Changes

Add an opt-in per-server MCP config flag that forwards OpenClaw caller/session headers to trusted remote bundle MCP servers without affecting stdio servers or overriding user headers.

## Best Possible Solution

Keep the issue open until #75118, or an equivalent maintainer-approved implementation, lands with schema/types/docs, default-off remote-only injection, user-header precedence, downstream flag stripping, placeholder env support, tests, and changelog coverage.

## Reproduction Assessment

Yes. Configure a remote `mcp.servers.<name>.url` entry with `injectCallerContext: true` and inspect the generated bundle MCP config; current main has no typed flag, schema entry, injection branch, or non-loopback placeholder-env trigger.

## Solution Assessment

Yes, pending maintainer security approval. A per-server default-off opt-in is the narrow maintainable direction for trusted internal remote MCP endpoints; a global flag would carry broader accidental identity-disclosure risk.
