## Summary

Keep open: current main still lacks the requested caller-context injection path, and the provided GitHub context shows #75118 is the open, unmerged closing PR for this work. The remaining decision is maintainer review of that implementation and the remote identity-disclosure boundary, not issue cleanup.

## What This Changes

The issue requests a default-off `mcp.servers.<name>.injectCallerContext` flag for remote bundle MCP servers to receive OpenClaw caller/session headers without overwriting user-configured headers.

## Best Possible Solution

Review and either land #75118 or an equivalent narrow implementation with schema/types/docs and focused tests: default off, remote-only, user headers win, downstream control flag stripped, and `OPENCLAW_MCP_*` placeholders populated when any merged bundle MCP layer opts in.

## Reproduction Assessment

Yes. Configure a remote `mcp.servers.<name>.url` entry with `injectCallerContext: true` and inspect the generated bundle MCP config; current main has no typed/schema contract, no merge branch, and no non-loopback env placeholder trigger for that behavior.

## Solution Assessment

Yes, pending maintainer security approval. A per-server default-off flag is the narrow maintainable direction for trusted internal endpoints; a global flag would create broader accidental identity-disclosure risk.
