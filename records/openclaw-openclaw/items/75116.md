## Summary

Keep open: current main still lacks the requested per-server caller-context injection, and #75118 is an open, unmerged implementation candidate with closing syntax for this issue. The remaining decision is maintainer/security approval for forwarding caller and session identity headers to opted-in remote MCP URLs.

## What This Changes

Add an opt-in `mcp.servers.<name>.injectCallerContext` option that forwards OpenClaw caller/session headers to configured remote bundle MCP servers without overwriting user headers.

## Best Possible Solution

Review and land #75118 or an equivalent narrow implementation covering schema/types, config docs, header injection, flag stripping, placeholder env population, regression tests, changelog, and trusted-endpoint security guidance.

## Reproduction Assessment

Not applicable as a feature request, but the absence is source-verifiable on current main: there is no typed flag, schema property, injection branch, or non-loopback placeholder-env trigger.

## Solution Assessment

Yes, pending maintainer/security review. A per-server, default-off, URL-only opt-in is narrower than a global switch and preserves user header precedence while avoiding broad identity disclosure.
