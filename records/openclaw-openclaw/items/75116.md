## Summary

Keep open: current main still lacks the requested per-server caller-context injection, and the provided GitHub context shows #75118 is an open, unmerged implementation candidate with closing syntax for this issue. The remaining decision is maintainer/security approval for forwarding caller and session identity headers to opted-in remote MCP URLs.

## What This Changes

Request to add an opt-in per-server `mcp.servers.<name>.injectCallerContext` config flag that injects OpenClaw caller/session headers into bundled remote MCP server configs without overwriting user headers.

## Best Possible Solution

Review and land #75118 or an equivalent narrow implementation with schema/types/docs, flag stripping, user-header precedence, placeholder env behavior for any opted-in merged layer, regression tests, changelog, and trusted-endpoint security wording.

## Reproduction Assessment

Not applicable as a feature request, but the absence is source-verifiable on current main. The typed config, schema, generated schema, docs, bundle merge path, and CLI env preparation do not implement the requested opt-in behavior.

## Solution Assessment

Yes, pending maintainer/security approval. A per-server, default-off, URL-only opt-in with user-header precedence is narrower than a global switch and avoids broad identity disclosure.
