## Summary

Keep open: current main still lacks the requested per-server caller-context injection, and #75118 is an open implementation candidate with closing syntax. The remaining decision is maintainer/security approval for forwarding caller and session identity headers to opted-in remote MCP URLs.

## What This Changes

Add an opt-in per-server `mcp.servers.<name>.injectCallerContext` config flag for URL-based bundled MCP servers to receive OpenClaw caller/session headers without overwriting user headers.

## Best Possible Solution

Review and land #75118 or an equivalent narrow implementation with schema/types/docs, flag stripping, placeholder env behavior, regression tests, changelog, and trusted-endpoint security guidance.

## Reproduction Assessment

Not applicable as a feature request; there is no broken existing contract to reproduce. The absence is source-verifiable on current main through config types, schema, bundle merge, CLI env setup, docs, and symbol searches.

## Solution Assessment

Yes, pending maintainer/security approval. A per-server, default-off, URL-only opt-in that preserves operator headers is the narrowest maintainable direction compared with a global switch.
