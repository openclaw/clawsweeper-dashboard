## Summary

Keep open: current main still lacks the requested per-server caller-context injection behavior, and the same author has an open, unmerged closing PR for this work. That PR is the implementation candidate, so this issue should stay open until maintainers review, land, or close it.

## What This Changes

The issue requests a default-off `mcp.servers.<name>.injectCallerContext` option for remote bundle MCP servers to inject OpenClaw caller/session headers without overwriting user-provided headers.

## Best Possible Solution

Review #75118 for the trust boundary and implementation details, then land that PR or an equivalent narrow change if accepted: schema/types/docs plus default-off remote-only header injection, user-header precedence, downstream control-flag stripping, placeholder env population, and focused tests.

## Reproduction Assessment

Yes. Configure a remote `mcp.servers.<name>.url` entry with `injectCallerContext: true` and inspect current main's generated bundle MCP config; source inspection shows no typed/schema contract, no injection branch, and no non-loopback env placeholder trigger.

## Solution Assessment

Yes, pending maintainer security approval. A per-server default-off flag is the narrow maintainable approach for trusted internal endpoints; a global flag would create broader accidental identity-disclosure risk.
