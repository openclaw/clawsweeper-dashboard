## Summary

Keep open: current `main` still lacks the requested per-server caller-context injection behavior, and the provided GitHub context shows an open, unmerged closing PR for this issue. The issue should stay open until maintainers review, land, replace, or close that PR.

## What This Changes

Add an opt-in `mcp.servers.<name>.injectCallerContext` option that forwards OpenClaw caller/session headers to trusted remote bundle MCP servers without changing default behavior.

## Best Possible Solution

Review #75118, or an equivalent narrow change, for the trust boundary and implementation details: schema/types/docs plus default-off remote-only header injection, user-header precedence, downstream control-flag stripping, placeholder env population, and focused tests.

## Reproduction Assessment

Yes. Configure a remote `mcp.servers.<name>.url` entry with `injectCallerContext: true` and inspect current main's generated bundle MCP config; source inspection shows no typed/schema contract, no injection branch, and no non-loopback env placeholder trigger.

## Solution Assessment

Yes, pending maintainer security approval. A per-server default-off flag is the narrow maintainable direction for trusted internal endpoints; a global flag would create broader accidental identity-disclosure risk.
