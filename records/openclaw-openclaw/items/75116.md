## Summary

Keep open: current main still lacks the requested per-server caller-context injection, and #75118 is an open, unmerged implementation candidate with closing syntax for this issue. The remaining decision is maintainer/security approval for forwarding caller and session identity headers to opted-in remote MCP URLs.

## What This Changes

Add a default-off `mcp.servers.<name>.injectCallerContext` option that injects OpenClaw caller identity placeholders into remote bundle MCP server headers without overwriting user headers.

## Best Possible Solution

Review and land #75118 or an equivalent narrow implementation with schema/types/docs, header merge and flag stripping, placeholder env behavior, tests, changelog, and trusted-endpoint security guidance.

## Reproduction Assessment

Not applicable as a feature request, but the absence is source-verifiable on current main. The typed config, schema, docs, bundle merge path, and CLI env preparation do not implement the requested opt-in behavior.

## Solution Assessment

Yes, pending maintainer/security approval. A per-server, default-off, URL-only opt-in is narrower than a global switch and preserves user header precedence while limiting identity disclosure.
