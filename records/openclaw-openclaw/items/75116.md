## Summary

Keep open: current main still lacks the requested caller-context injection behavior, and #75118 is an open, unmerged closing PR for the same feature. The remaining decision is maintainer/security review of the caller-identity disclosure boundary, not cleanup closure.

## What This Changes

Add a default-off per-server `injectCallerContext` option for remote `mcp.servers` entries so bundle MCP configs can forward OpenClaw caller identity headers without overwriting user headers.

## Best Possible Solution

Review #75118 or an equivalent narrow core change with schema/types/docs/changelog/tests, preserving default-off remote-only injection, user-header precedence, downstream flag stripping, and placeholder env population.

## Reproduction Assessment

Yes. Configure a remote `mcp.servers.<name>.url` entry with `injectCallerContext: true` and inspect the generated bundle MCP config; current main has no typed/schema flag, no injection branch, and no non-loopback placeholder-env trigger.

## Solution Assessment

Yes, pending maintainer security approval. A per-server default-off opt-in is the narrowest maintainable shape for trusted internal endpoints; a global flag would create broader accidental disclosure risk.
