## Summary

Keep open: current main still lacks the requested per-server caller-context injection, and #75118 is an open, unmerged implementation candidate with closing syntax for this issue. The remaining question is maintainer/security approval for forwarding caller and session identity headers to opted-in remote MCP URLs.

## What This Changes

Request to add a default-off `mcp.servers.<name>.injectCallerContext` option for URL MCP servers so bundle MCP can inject OpenClaw caller/session headers without overwriting user headers.

## Best Possible Solution

Review and land #75118 or an equivalent narrow implementation covering schema/types/docs, header injection, flag stripping, placeholder env population, regression tests, changelog, and trusted-endpoint security wording.

## Reproduction Assessment

Not applicable as a feature request, but the absence is source-verifiable: current main has no typed flag, schema property, injection branch, or non-loopback placeholder-env trigger.

## Solution Assessment

Yes, pending maintainer security approval. A per-server, default-off, URL-only opt-in is narrower than a global switch and preserves user header precedence while avoiding broad identity disclosure.
