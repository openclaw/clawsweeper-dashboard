## Summary

Keep open. Current main still lacks the requested per-server caller-context injection contract and behavior, and this issue is paired with open closing PR #75118, so it should stay open until that PR is reviewed, landed, or closed.

## What This Changes

Add an opt-in `mcp.servers.<name>.injectCallerContext` flag that forwards OpenClaw caller/session headers to trusted remote bundle MCP servers without overwriting user headers or leaking the control flag downstream.

## Best Possible Solution

Review #75118 for the trust boundary and implementation details, then land that PR or an equivalent narrow change if accepted: default-off, remote-only, non-overwriting caller-header merge, stripped downstream control flag, placeholder env population when any merged server opts in, plus matching schema, docs, and tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to configure a remote `mcp.servers.<name>.url` entry with `injectCallerContext: true` and inspect the generated bundle MCP config; current main has no injection branch and only builds `OPENCLAW_MCP_*` env values for the loopback runtime.

## Solution Assessment

Yes, pending maintainer approval of the trust boundary. A per-server default-off flag is the narrow maintainable direction; a global flag would increase accidental identity disclosure risk, and #75118 is the active implementation candidate.
