## Summary

Keep open: current main still lacks the requested per-server caller-context injection, and #75118 is an open implementation PR with closing syntax for this issue. The remaining decision is maintainer/security review of forwarding caller and session identity headers to opted-in remote MCP URLs.

## What This Changes

Add a default-off per-server `mcp.servers.<name>.injectCallerContext` option for remote bundle MCP servers that merges OpenClaw caller/session headers without overwriting user-supplied headers.

## Best Possible Solution

Review and land #75118 or an equivalent narrow implementation with schema/types/docs, header merge and flag stripping, placeholder env behavior, regression tests, changelog, and clear trusted-endpoint security wording.

## Reproduction Assessment

Not applicable as a feature request, but the absence is source-verifiable on current main. The typed config, schema, docs, bundle merge path, and CLI env preparation do not implement the requested opt-in behavior.

## Solution Assessment

Yes, pending maintainer/security approval. A per-server, default-off, URL-only opt-in is narrower than a global switch and preserves user header precedence while reducing accidental identity disclosure.
