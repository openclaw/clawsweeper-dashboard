## Summary

Keep open: current main still lacks the requested per-server caller-context injection behavior, and #75118 is an open closing PR for this exact feature. The issue should stay open until maintainers review the caller-identity disclosure boundary and land, replace, or close that PR.

## What This Changes

Add a default-off `mcp.servers.<name>.injectCallerContext` option that forwards OpenClaw caller/session headers to opted-in remote MCP servers without overwriting configured headers.

## Best Possible Solution

Review #75118 for the trust boundary and implementation details, then land that PR or an equivalent narrow change if accepted: schema/types/docs plus default-off remote-only header injection, user-header precedence, downstream flag stripping, placeholder env population, focused tests, and changelog coverage.

## Reproduction Assessment

Yes. Configure a remote `mcp.servers.<name>.url` entry with `injectCallerContext: true` and inspect the generated bundled MCP config; source inspection shows current main has no typed/schema contract, injection branch, or non-loopback placeholder-env trigger.

## Solution Assessment

Yes, pending maintainer security approval. A per-server default-off opt-in is the narrow maintainable design for trusted internal endpoints; the issue should not close before the linked implementation or an equivalent lands.
