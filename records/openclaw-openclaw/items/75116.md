## Summary

Keep open: current main still lacks the requested per-server caller-context injection, and #75118 is an open, unmerged implementation candidate with closing syntax for this issue. The remaining decision is maintainer/security review for forwarding caller and session identity headers to opted-in remote MCP URLs.

## What This Changes

Request per-server `mcp.servers.<name>.injectCallerContext` opt-in so bundle MCP remote `url` servers receive OpenClaw caller/session headers without overriding configured headers.

## Best Possible Solution

Review and land #75118 or an equivalent narrow implementation with schema/types/docs, flag stripping, non-overwriting header merge behavior, placeholder env coverage, regression tests, changelog, and trusted-endpoint security wording.

## Reproduction Assessment

Not applicable as a feature request. The absence is source-verifiable on current main: config types, schema, docs, bundle merge, and CLI env preparation do not implement the requested opt-in behavior.

## Solution Assessment

Yes, pending maintainer/security approval. A per-server, default-off, URL-only opt-in is narrower than global forwarding and preserves operator header precedence while reducing accidental identity disclosure.
