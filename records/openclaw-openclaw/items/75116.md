## Summary

Keep open: current main still lacks the requested per-server caller-context injection, and the linked #75118 remains an open implementation candidate with closing syntax for this issue. The remaining decision is maintainer/security review of forwarding caller and session identity headers to opted-in remote MCP URLs.

## What This Changes

Add a default-off `mcp.servers.<name>.injectCallerContext` config option for bundled remote MCP servers that merges OpenClaw caller/session headers without overwriting user headers and strips the control flag from emitted CLI config.

## Best Possible Solution

Review and land #75118 or an equivalent narrow implementation with schema/types/docs, header merge and flag stripping, placeholder env behavior, regression tests, changelog, and trusted-endpoint security guidance.

## Reproduction Assessment

Not applicable as a feature request, but the absence is source-verifiable on current main. The config type, schema, docs, bundle merge path, and CLI env preparation do not implement the requested opt-in behavior.

## Solution Assessment

Yes, pending maintainer/security approval. A per-server, default-off, URL-only, non-overwriting opt-in is narrower than a global switch and reduces accidental caller identity disclosure.
