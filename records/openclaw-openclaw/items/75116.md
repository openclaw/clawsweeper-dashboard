## Summary

Keep open. Current main still lacks the requested per-server caller-context injection behavior, and the issue is paired with open, unmerged PR #75118, so the tracking issue should remain open until that PR or an equivalent implementation lands.

## What This Changes

The issue requests a default-off `mcp.servers.<name>.injectCallerContext` option that injects OpenClaw caller/session headers into trusted remote bundle MCP server configs without overwriting user headers.

## Best Possible Solution

Review PR #75118 for the trust boundary and implementation details, then land that PR or an equivalent narrow change if accepted: default-off, remote-only, non-overwriting caller-header merge, stripped downstream control flag, placeholder env population when any merged server opts in, plus schema, docs, changelog, and focused tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to configure a remote `mcp.servers.<name>.url` entry with `injectCallerContext: true` and inspect the generated bundle MCP config; current main has no injection branch and only prepares `OPENCLAW_MCP_*` env values from the loopback runtime path.

## Solution Assessment

Yes, pending maintainer approval of the trust boundary. A per-server default-off flag is the narrow maintainable shape; a global flag would raise accidental disclosure risk, and the existing open PR is the active implementation candidate.
