## Summary

Keep open: current main still lacks the requested per-server caller-context opt-in, and https://github.com/openclaw/openclaw/pull/75118 is an open implementation PR with closing syntax for this issue. The remaining question is maintainer/security approval for forwarding caller and session identity headers to opted-in remote MCP URLs.

## What This Changes

Add a default-off per-server `mcp.servers.<name>.injectCallerContext` flag for remote bundle MCP servers to receive OpenClaw caller identity headers without overriding configured headers.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/75118 or an equivalent narrow implementation with types, schema/docs, flag stripping, non-overwriting header merge, regression tests, changelog, and trusted-endpoint wording.

## Reproduction Assessment

Not applicable as a feature request, but the absence is source-verifiable on current main. The typed config, schema, merge path, and CLI env preparation do not implement the requested opt-in behavior.

## Solution Assessment

Yes, pending maintainer/security approval. A per-server, default-off, URL-only opt-in is narrower than a global switch and preserves configured header precedence while avoiding broad identity disclosure.
