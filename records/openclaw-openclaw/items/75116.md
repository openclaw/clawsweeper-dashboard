## Summary

Keep open: current main still lacks the requested per-server caller-context injection contract, and #75118 is an open, unmerged closing PR for the same feature that needs maintainer/security review.

## What This Changes

Add a default-off `mcp.servers.<name>.injectCallerContext` option that forwards OpenClaw caller/session headers to opted-in remote bundled MCP servers without overwriting user headers.

## Best Possible Solution

Review #75118 for the security boundary and implementation details, then land that PR or an equivalent narrow change if maintainers accept the feature.

## Reproduction Assessment

Yes. Configure a remote `mcp.servers.<name>.url` entry with `injectCallerContext: true` and inspect the generated bundled MCP config; current-main source inspection shows no contract, injection path, or non-loopback placeholder-env trigger.

## Solution Assessment

Yes, pending maintainer security approval. A per-server default-off opt-in is the narrowest maintainable direction for trusted internal remote MCP servers; a global flag would create broader accidental disclosure risk.
