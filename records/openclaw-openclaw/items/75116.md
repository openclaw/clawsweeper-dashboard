## Summary

Keep open. Current main still lacks the requested per-server caller-context injection behavior, and the issue is paired with open closing PR #75118, so it should stay open until that implementation is reviewed, landed, or closed.

## What This Changes

Add an opt-in `mcp.servers.<name>.injectCallerContext` flag that forwards OpenClaw caller/session headers to trusted remote MCP servers without changing default behavior.

## Best Possible Solution

Review #75118 or an equivalent narrow change for the trust boundary, then land it if accepted with a default-off, remote-only, non-overwriting header merge, stripped downstream control flag, placeholder env population when any merged server opts in, and matching schema, docs, and unit tests.

## Reproduction Assessment

Yes. Configure a remote `mcp.servers.<name>.url` entry with `injectCallerContext: true` and inspect the generated bundle MCP config/env; current main has no injection branch and only builds `OPENCLAW_MCP_*` env values for the loopback runtime.

## Solution Assessment

Yes, pending maintainer approval of the disclosure boundary. A per-server, default-off flag is the narrow maintainable direction; a global flag would raise accidental identity disclosure risk, and #75118 is already the active implementation candidate.
