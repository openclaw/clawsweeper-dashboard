## Summary

Keep open: current main still lacks the requested per-server caller-context injection, and #75118 is an open, unmerged implementation candidate that explicitly closes this issue.

## What This Changes

Request an opt-in `mcp.servers.<name>.injectCallerContext` config flag that forwards OpenClaw caller/session headers to URL-based bundled MCP servers without overwriting user headers.

## Best Possible Solution

Review and land #75118 or an equivalent narrow implementation covering schema/types/docs, header injection, flag stripping, placeholder env population, regression tests, changelog, and trusted-endpoint security wording.

## Reproduction Assessment

Not applicable as a feature request; the absence is source-verifiable on current main. The config type, schema, bundle merge path, CLI env preparation, and docs do not implement the requested opt-in behavior.

## Solution Assessment

Yes, pending maintainer/security approval. A per-server, default-off, URL-only opt-in is the narrowest maintainable direction because it avoids a global identity disclosure switch and preserves user-supplied headers.
