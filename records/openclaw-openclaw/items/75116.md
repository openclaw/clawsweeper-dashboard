## Summary

Keep open: current main still lacks the requested per-server caller-context injection, and #75118 is an open, unmerged implementation candidate with closing syntax. The remaining decision is maintainer/security approval for forwarding OpenClaw caller and session identity headers to opted-in remote MCP URLs.

## What This Changes

Add an optional `mcp.servers.<name>.injectCallerContext` flag that injects OpenClaw caller/session headers into bundled remote MCP server configs without changing existing defaults.

## Best Possible Solution

Review and land #75118 or an equivalent narrow implementation with schema/types/docs, header merge and flag stripping, placeholder env behavior, regression tests, changelog, and trusted-endpoint security guidance.

## Reproduction Assessment

Not applicable as a feature request, but the absence is source-verifiable on current main: the typed config, schema, docs, bundle merge path, and CLI env preparation do not implement the requested opt-in behavior.

## Solution Assessment

Yes, pending maintainer/security approval. A per-server, default-off, URL-only opt-in is the narrowest maintainable shape compared with a global switch, and it preserves user header precedence while limiting identity disclosure.
