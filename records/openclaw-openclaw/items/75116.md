## Summary

Keep open: current main still lacks the requested per-server caller-context injection, and #75118 is an open, unmerged implementation candidate. Because the feature would forward caller and session identity to remote MCP URLs, it should stay open for maintainer/security review rather than be swept closed.

## What This Changes

Add a per-server `mcp.servers.<name>.injectCallerContext` opt-in so bundled remote MCP servers can receive OpenClaw caller identity headers without changing default behavior.

## Best Possible Solution

Review and land #75118 or an equivalent narrow implementation with schema/types/docs, flag stripping, non-overwriting header merge behavior, placeholder env coverage, regression tests, changelog, and trusted-endpoint security wording.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main lacks the requested typed/schema config, bundle merge behavior, and placeholder env trigger.

## Solution Assessment

Yes, pending maintainer/security approval. A per-server, default-off, URL-only opt-in is narrower than a global switch and matches the stated non-leakage and user-header-precedence requirements.
