## Summary

Keep open: current main still lacks the requested per-server caller-context injection, and https://github.com/openclaw/openclaw/pull/75118 is an open, unmerged implementation candidate that closes this issue. The remaining action is maintainer/security review of forwarding caller/session identity headers to opted-in remote MCP URLs.

## What This Changes

Add a default-off `mcp.servers.<name>.injectCallerContext` option that injects OpenClaw caller/session headers into trusted remote bundle MCP servers without overwriting user headers.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/75118 or an equivalent narrow change covering schema/types/docs, URL-only header injection, user-header precedence, flag stripping, placeholder env population, tests, and changelog.

## Reproduction Assessment

Yes. Configure a remote `mcp.servers.<name>.url` entry with `injectCallerContext: true` and inspect the generated bundle MCP config; current source has no injection path, while the unknown flag would be preserved rather than acted on or stripped.

## Solution Assessment

Yes, pending maintainer security approval. A per-server, default-off, URL-only opt-in is narrower than a global flag and is the maintainable direction if it preserves user header precedence and documents the trust boundary.
