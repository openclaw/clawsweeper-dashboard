## Summary

Keep open: current main still lacks the requested per-server caller-context injection, and #75118 is an open, unmerged implementation candidate with closing syntax. This should remain tracked until that security-sensitive config/API addition is reviewed and merged or replaced.

## What This Changes

Add a default-off `mcp.servers.<name>.injectCallerContext` flag that injects OpenClaw caller/session headers into opted-in remote bundle MCP server headers without overwriting user headers.

## Best Possible Solution

Review and land #75118 or an equivalent narrow core implementation with schema/types/docs, header merge and flag stripping, placeholder env behavior, regression tests, changelog, and trusted-endpoint security wording.

## Reproduction Assessment

Not applicable as a feature request; the absence is source-verifiable on current main. The typed config, schema, docs, bundle merge path, and CLI env preparation do not define the requested opt-in behavior.

## Solution Assessment

Yes, pending maintainer/security approval. A per-server, default-off, URL-only opt-in is narrower than a global switch and preserves user header precedence while avoiding broad identity disclosure.
