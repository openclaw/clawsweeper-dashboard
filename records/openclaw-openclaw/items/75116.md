## Summary

Keep open: current main still lacks the requested per-server caller-context injection, and #75118 is an open, unmerged implementation candidate for the same feature. The remaining decision is maintainer/security review of whether opted-in remote MCP URLs may receive caller and session identity headers.

## What This Changes

Request adds a default-off `mcp.servers.<name>.injectCallerContext` option that injects OpenClaw caller/session headers into trusted remote bundle MCP server configs without overwriting user headers.

## Best Possible Solution

Review and either land #75118 or an equivalent narrow change covering schema/types/docs, header injection, flag stripping, placeholder env population, tests, and changelog.

## Reproduction Assessment

Yes. Configure a remote `mcp.servers.<name>.url` entry with `injectCallerContext: true` and inspect the generated bundle MCP config; source inspection shows current main has no schema/type flag, injection branch, or non-loopback placeholder-env trigger.

## Solution Assessment

Yes, pending maintainer security approval. A per-server, default-off opt-in for URL servers is narrower than a global flag and preserves user header precedence while avoiding broad identity disclosure.
