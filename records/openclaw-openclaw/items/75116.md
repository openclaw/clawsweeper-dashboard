## Summary

Keep open. Current main still has only loopback MCP caller headers and lacks a typed/schema or bundle-merge path for per-server remote caller-context injection; the linked closing PR #75118 remains open and unmerged, so this issue should track maintainer and security review.

## What This Changes

Request an optional per-server `mcp.servers.<name>.injectCallerContext` flag that forwards OpenClaw caller/session headers to trusted remote bundle MCP servers without overwriting configured headers.

## Best Possible Solution

Review #75118 for the trust boundary and implementation details, then land that PR or an equivalent narrow change if accepted: schema/types/docs plus default-off remote-only header injection, user-header precedence, downstream control-flag stripping, placeholder env population, focused tests, and changelog coverage.

## Reproduction Assessment

Yes. Configure a remote `mcp.servers.<name>.url` entry with `injectCallerContext: true` and trace current bundle MCP generation; source inspection shows no typed/schema contract, injection branch, or non-loopback placeholder-env trigger.

## Solution Assessment

Yes, pending maintainer security approval. A per-server, default-off opt-in is the narrow maintainable direction for trusted internal endpoints; a global flag would carry broader accidental identity-disclosure risk.
