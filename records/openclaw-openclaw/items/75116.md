## Summary

Keep open: current main still lacks the requested per-server caller-context injection, and #75118 is an open, unmerged implementation candidate with closing syntax for this issue. The remaining decision is maintainer/security approval for forwarding caller and session identity headers to opted-in remote MCP URLs.

## What This Changes

Request a default-off per-server `mcp.servers.<name>.injectCallerContext` config option that injects OpenClaw caller/session headers into opted-in remote bundle MCP servers.

## Best Possible Solution

Review and land #75118 or an equivalent narrow implementation covering schema/types/docs, header injection, flag stripping, placeholder env population, regression tests, changelog, and trusted-endpoint security wording.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The absence is source-verifiable: current main has no typed flag, schema property, injection branch, or non-loopback placeholder-env trigger.

## Solution Assessment

Yes, pending maintainer/security approval. A per-server, default-off, URL-only opt-in is the narrowest maintainable direction compared with a global switch, and it preserves user header precedence while limiting identity disclosure.
