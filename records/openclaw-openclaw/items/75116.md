## Summary

Keep open: current main still lacks the requested per-server caller-context injection, and #75118 is an open, unmerged closing PR for the same feature. The remaining decision is maintainer/security review of whether trusted remote MCP URLs may receive caller/session identity headers.

## What This Changes

Add an opt-in `mcp.servers.<name>.injectCallerContext` setting that forwards OpenClaw caller identity headers to trusted remote bundle MCP servers without changing default behavior.

## Best Possible Solution

Review #75118 for the trust boundary and implementation details, then land that PR or an equivalent narrow change if accepted: schema/types/docs, default-off remote-only injection, user-header precedence, downstream flag stripping, placeholder env population, focused tests, and changelog coverage.

## Reproduction Assessment

Yes. Configure a remote `mcp.servers.<name>.url` entry with `injectCallerContext: true` and inspect the generated bundle MCP config; source inspection shows current main has no typed/schema contract, injection branch, or non-loopback placeholder-env trigger.

## Solution Assessment

Yes, pending maintainer security approval. A per-server default-off opt-in is the narrow maintainable design for trusted internal endpoints; a global flag would carry broader accidental identity-disclosure risk.
