## Summary

Keep open. Current main still lacks the requested per-server caller-context injection contract and behavior, and open PR #75118 is the active closing implementation candidate, so this issue should remain open until that PR is reviewed, landed, or closed.

## What This Changes

Add an opt-in per-remote-MCP-server config flag that injects OpenClaw caller/session headers into bundled MCP configs without overwriting user-supplied headers.

## Best Possible Solution

Review #75118 for the trust boundary and implementation details, then land that PR or an equivalent narrow change if accepted: typed/schema-backed config, default-off remote-only behavior, non-overwriting header merge, stripped downstream control flag, placeholder env population when any merged layer opts in, plus docs and targeted tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to configure a remote mcp.servers entry with url and injectCallerContext: true, then inspect the generated bundle MCP config path; current main has no branch that injects caller headers or populates placeholder env for that remote opt-in.

## Solution Assessment

Yes, pending maintainer security approval. A per-server default-off flag is the narrow maintainable shape for trusted internal remote MCP endpoints; a global flag would create a broader accidental identity-disclosure risk.
