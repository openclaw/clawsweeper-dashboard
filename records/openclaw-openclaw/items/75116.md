## Summary

Keep open: current main still lacks the requested per-server caller-context injection, and #75118 is an open, unmerged implementation candidate with closing syntax for this issue. The remaining decision is maintainer/security approval for forwarding caller and session identity headers to opted-in remote MCP URLs.

## What This Changes

The issue requests a new per-server `mcp.servers.<name>.injectCallerContext` option for remote bundled MCP servers so opted-in HTTP MCP endpoints receive OpenClaw caller identity headers without changing default behavior.

## Best Possible Solution

Review and land #75118 or an equivalent narrow implementation covering schema/types/docs, header injection, flag stripping, placeholder env population, regression tests, changelog, and trusted-endpoint security wording.

## Reproduction Assessment

Not applicable as a feature request, but the absence is source-verifiable on current main. The typed config, schema, bundle merge path, and CLI env preparation do not implement the requested opt-in behavior.

## Solution Assessment

Yes, pending maintainer/security approval. A per-server, default-off, URL-only opt-in is narrower than a global switch and preserves user header precedence while avoiding broad identity disclosure.
