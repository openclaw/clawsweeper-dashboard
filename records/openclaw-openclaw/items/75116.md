## Summary

Keep open: current main still lacks the requested per-server caller-context injection, and #75118 is an open, unmerged implementation candidate with closing syntax for this issue. The remaining decision is maintainer/security review for forwarding caller and session identity headers to opted-in remote MCP URLs.

## What This Changes

Add a default-off `mcp.servers.<name>.injectCallerContext` flag that injects OpenClaw caller headers into bundled remote MCP server configs without overwriting user headers.

## Best Possible Solution

Review and land #75118 or an equivalent narrow implementation with types, schema/help/docs, flag stripping, non-overwriting header merge, placeholder env behavior, regression tests, changelog, and trusted-endpoint security wording.

## Reproduction Assessment

Not applicable as a feature request, but the absence is source-verifiable on current main. The typed config, schema, docs, merge path, and CLI env preparation do not implement the requested opt-in behavior.

## Solution Assessment

Yes, pending maintainer/security review. A per-server, default-off, URL-only opt-in is the narrow maintainable shape compared with a global switch, provided user headers keep precedence and the emitted downstream config strips the control flag.
