## Summary

Keep open: current main still lacks the requested per-server caller-context injection, and #75118 is an open, unmerged implementation candidate that explicitly tracks this issue. The remaining decision is maintainer/security approval for forwarding caller and session identity headers to opted-in remote MCP URLs.

## What This Changes

Add a default-off `mcp.servers.<name>.injectCallerContext` option that forwards OpenClaw caller identity headers to opted-in remote bundle MCP servers without overwriting user headers.

## Best Possible Solution

Land #75118 or an equivalent narrow implementation after security review, covering schema/types/docs, bundle merge header injection, control-flag stripping, placeholder env population, tests, and changelog.

## Reproduction Assessment

Not applicable as a feature request, but the absence is source-verifiable: current main has no typed flag, schema property, injection branch, or non-loopback placeholder-env trigger.

## Solution Assessment

Yes, pending maintainer/security approval. A per-server, default-off, URL-only opt-in is narrower than a global switch and preserves user header precedence while limiting accidental identity disclosure.
