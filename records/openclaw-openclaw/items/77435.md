## Summary

Keep open: current main and the latest release still leave a Plugin SDK contract gap between documented hook support and Gateway startup selection, and the linked docs/runtime PRs are open rather than merged.

## What This Changes

The issue asks OpenClaw to document, fail fast, or change runtime loading for non-capability plugins whose chat-path hooks appear valid but do not fire in the Gateway.

## Best Possible Solution

Land or replace the hook runtime activation docs clarification, then handle any fail-fast validation or broader Gateway startup behavior as a separate explicit Plugin SDK decision.

## Reproduction Assessment

Yes, source-level: current main omits enabled non-capability hook plugins unless manifest hook activation or explicit hooks policy selects them, while the docs still present before_dispatch as an available plugin hook. I did not run the live Slack/Gateway reproduction.

## Solution Assessment

Unclear: a docs-only mitigation is the narrowest low-risk path, but fail-fast validation or broader runtime loading requires a maintainer Plugin SDK decision. Keeping this issue open while the linked PRs are reviewed is the best current handling.
