## Repair Work Prompt

Prepare a focused docs-only PR for https://github.com/openclaw/openclaw/issues/77435. Document the current chat-path typed-hook routing contract for plugin authors: typed lifecycle hooks such as `before_dispatch`, `reply_dispatch`, and `before_model_resolve` are registered with `api.on(...)`, not the internal `api.registerHook(...)`; an external hook-only or non-capability plugin must be imported by Gateway startup before those hooks can run; the intended opt-ins are `activation.onCapabilities: ["hook"]`, `activation.onStartup: true` when startup import is truly required, or explicit `plugins.entries.<id>.hooks.*` policy where appropriate. Clarify that `allowConversationAccess=true` gates `llm_input`, `llm_output`, `before_agent_finalize`, and `agent_end`, while `before_dispatch` is a dispatch hook that still needs startup activation but not conversation-access permission. Update `docs/plugins/hooks.md`, `docs/plugins/manifest.md`, and/or `docs/tools/plugin.md`; keep the change docs-focused, avoid runtime behavior changes, do not add a new config option, and mention that `plugins list` is inventory while `plugins inspect --runtime --json` verifies module-loaded registrations. No changelog is required for docs-only work unless maintainers request one.
