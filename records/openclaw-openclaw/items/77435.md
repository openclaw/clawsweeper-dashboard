## Summary

Keep open: current main and v2026.6.10 still have the hook startup-selection gap, current docs only partially explain the active-Gateway/runtime distinction, and the two candidate PR paths are open rather than merged.

## What This Changes

The issue asks OpenClaw to fail fast or document why hook registrations from non-capability plugins can appear loaded but not run in the Gateway messaging runtime.

## Best Possible Solution

Make the hook activation contract explicit for plugin authors, then have maintainers either accept docs-only closure or separately sponsor a fail-fast/runtime startup change.

## Reproduction Assessment

Yes, source-level: current main and v2026.6.10 still exclude enabled-only non-capability hook plugins unless manifest hook activation or hook-policy config selects them. I did not run the live Slack/Gateway reproduction.

## Solution Assessment

Unclear; docs-only is the narrowest low-risk mitigation, but fail-fast behavior or broader runtime loading needs maintainer confirmation because it affects plugin SDK compatibility and Gateway startup policy.
