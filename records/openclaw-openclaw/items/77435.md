## Summary

Keep this open: current main and v2026.6.6 still lack the requested hook-routing clarification or fail-fast behavior, and the open closing PR is the active fix path rather than a reason to close the issue early.

## What This Changes

The issue asks OpenClaw to clarify or fail fast when plugins register chat-path hooks such as `before_dispatch` without being loaded into the messaging runtime.

## Best Possible Solution

Land or replace the linked docs clarification that names `activation.onCapabilities: ["hook"]` or explicit hook-policy config as the supported chat-path hook loading contract, and leave fail-fast or broader runtime loading semantics to a separate Plugin SDK product decision if maintainers want them.

## Reproduction Assessment

Source-reproducible, but not live-reproduced in this read-only review. The code shows `before_dispatch` only runs from a loaded hook runner, while Gateway startup selection for hook plugins requires `activation.onCapabilities` including `hook` or explicit hook-policy config.

## Solution Assessment

No, closing this issue is not the best solution yet. The narrowest current path is to review and land or replace the linked docs PR; changing fail-fast or non-capability loading semantics would require a separate Plugin SDK decision.
