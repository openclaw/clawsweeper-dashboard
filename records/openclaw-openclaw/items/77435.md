## Summary

Keep this open because current main and the latest release still lack the requested hook-routing clarification, and an open linked PR with closing syntax is the active fix path rather than a reason to close the issue early.

## What This Changes

The issue asks OpenClaw to clarify or fail fast when non-capability Plugin SDK hooks such as `before_dispatch` are accepted but not loaded into the Gateway chat-path runtime.

## Best Possible Solution

Land a corrected docs clarification that names `activation.onCapabilities: ["hook"]` or explicit hook-policy config as the supported chat-path hook loading contract, and leave broader fail-fast/runtime-loading behavior to a separate SDK contract decision if maintainers want it.

## Reproduction Assessment

Source-reproducible, but not live-reproduced in this read-only review. The code shows `before_dispatch` only runs from loaded typed hooks, while Gateway startup selection for hook plugins depends on `activation.onCapabilities: ["hook"]` or explicit hook-policy config.

## Solution Assessment

No, the issue should not close yet. The narrowest current solution is the linked docs clarification after its example is corrected; changing fail-fast or non-capability loading semantics would require a separate Plugin SDK product decision.
