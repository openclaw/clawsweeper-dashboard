## Summary

Keep open. This PR is the open candidate for the Feishu localhost media-fetch bug, but the current patch weakens the shared remote-media SSRF boundary and still lacks after-fix real Feishu/gateway proof.

## What This Changes

The branch modifies `src/media/web-media.ts` to add `localhost` and `127.0.0.1` to the remote-media SSRF allowed hostnames when the requested media URL uses those loopback hosts.

## Best Possible Solution

Resolve only OpenClaw-owned inbound or gateway media through trusted local media paths or managed `media://` references, leave generic loopback HTTP(S) media blocked, then add regression coverage and real Feishu proof.

## Reproduction Assessment

Source-reproducible: literal `http://localhost` or `http://127.0.0.1` media URLs still enter `readRemoteMediaBuffer` and are blocked by the SSRF guard. I did not establish a live Feishu end-to-end reproduction or after-fix proof.

## Solution Assessment

No. The proposed patch is a plausible workaround, but it is not the best fix because it trusts generic loopback hosts in the shared remote-media loader instead of proving OpenClaw ownership through managed media resolution.
