## What This Changes

The issue asks OpenClaw to remove or more strongly warn on `dangerouslyAllowNameMatching` because Discord/Slack display-name matching can authorize an impersonating sender.

## Best Possible Solution

A maintainer should either reaffirm the documented break-glass/out-of-scope posture and close this as covered by https://github.com/openclaw/openclaw/issues/54288, or explicitly choose a compatibility-aware hardening path such as higher audit severity, setup warnings, deprecation, or removal with migration guidance.

## Reproduction Assessment

Yes for source-level reproduction, but no live Discord or Slack impersonation run was performed. Current code shows dangerous name aliases are ignored by default and authorize only when the explicit mutable-identifier policy is enabled.

## Solution Assessment

Unclear. Removing the flag is the strongest hardening option, but the shipped docs and SECURITY.md frame it as an intentional break-glass operator tradeoff, so maintainers should choose the policy before implementation.
