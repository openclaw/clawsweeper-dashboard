## Summary

This issue should stay open: current main still exposes `dangerouslyAllowNameMatching` for Discord/Slack-style channel authorization, and the active audit path treats the enabled break-glass mode as `info` rather than a higher-severity auth-boundary finding.

## What This Changes

The issue requests removing or strengthening warnings around display-name-based channel allowlist matching for Discord and Slack.

## Best Possible Solution

Keep sender authorization ID-first and either remove mutable-name authorization or make the retained break-glass path critical, setup-visible, and explicit in docs.

## Reproduction Assessment

No live high-confidence reproduction was run. Source inspection does show the risky aliases and audit severity path on current main, which is enough to keep this open for security review.

## Solution Assessment

Unclear. Removing the flag is the strongest security posture, but because this is documented break-glass compatibility across channels, maintainers should choose the compatibility/security tradeoff before implementation.
