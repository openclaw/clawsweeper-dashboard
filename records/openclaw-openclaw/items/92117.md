## Summary

Current main still contains a self-contradictory test setup: the injected asynchronous resolver mock calls the synchronous resolver mock, then the test asserts that synchronous mock was never called. The production branch correctly selects the supplied async resolver, no later commit or canonical item fixes this path, and the linked CI failures make this a clear, focused test regression worth repairing.

## What This Changes

Fix the asynchronous provider model-discovery test so its async resolver double does not itself record a synchronous model-resolution call.

## Best Possible Solution

Keep the production async-selection branch intact and give the affected test an independent async resolver result, preserving focused assertions that only the async resolver is selected and receives the expected arguments.

## Reproduction Assessment

Yes. Source inspection shows the async mock directly calls the sync mock before the test asserts that the sync mock was untouched, and the issue links two CI jobs with the matching assertion failure.

## Solution Assessment

Yes. Isolating the async test double is the narrowest maintainable fix; changing the production resolver branch or broadly clearing mocks would obscure the actual test-double defect.
