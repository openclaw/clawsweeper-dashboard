## Summary

Current main no longer matches the reported Mattermost preview-flattening behavior: regular inbound Mattermost posts are dispatched as normal channel turns with newline-preserving body fields, and a focused regression test asserts they are not enqueued as system events. The shipped v2026.4.29 tag contains the same proof, so the central user-visible problem is already resolved.

## What This Changes

Request an opt-in Mattermost or per-channel preview mode that preserves newlines in system-event previews instead of flattening multi-line Markdown into one line.

## Best Possible Solution

Keep the shipped Mattermost behavior: regular user posts should stay normal inbound turns with newline-preserving `BodyForAgent` and `RawBody`, while actual Mattermost system events remain limited to interactions and reactions that do not carry multi-line user Markdown previews.

## Reproduction Assessment

Yes. The focused source-level path is high confidence: send a Mattermost `posted` event through the monitor and verify no `enqueueSystemEvent` call occurs, with the user body dispatched through `BodyForAgent` instead.

## Solution Assessment

Yes. Removing the regular-post system-event preview path is a narrower and more maintainable resolution than adding a new config mode for a preview surface that Mattermost no longer uses for inbound user posts.
