Keep open: current main and v2026.6.8 still only make Android chat code blocks selectable, while normal message text has no message-level copy, select-copy, share, or reply action. The open linked fork PR references this issue but is conflicting, so maintainers should review, repair, replace, or close that implementation path rather than close this issue now.

## What This Changes

The issue requests Android chat message copy, selective copy, share, and reply actions for non-code message text.

## Best Possible Solution

Review, repair, or replace the linked Android implementation with a focused message-action UI that adds copy, selective copy, share, and optional reply while preserving LazyColumn scrolling, tappable links, code-block selection, and current chat rendering behavior.
