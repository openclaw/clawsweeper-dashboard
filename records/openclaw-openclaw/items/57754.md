## Summary

Keep open. Current main still only makes Android chat code blocks selectable, with no message-level long-press actions for whole-message copy, selective copy, share, or reply; an open implementation PR with closing syntax should be reviewed, landed, or replaced before this issue closes.

## What This Changes

Add Android chat message actions for copying whole messages, selecting partial text, sharing message text, and quoting messages into the composer.

## Best Possible Solution

Review and land #59603 or a narrow successor that adds a message-level Android chat action surface in `apps/android/app/src/main/java/ai/openclaw/app/ui/chat`, then close this issue after the implementation reaches main.

## Reproduction Assessment

Yes. The issue is reproducible by inspecting current main: only code blocks are inside `SelectionContainer`, and the message bubble/list path has no long-press action callbacks for normal message text.

## Solution Assessment

Yes, directionally. A message-level long-press action surface is the maintainable solution because wrapping all rendered text in `SelectionContainer` would keep the reported scroll/gesture conflict in the main chat list.
