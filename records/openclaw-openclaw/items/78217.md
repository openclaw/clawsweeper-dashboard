## Summary

Keep open. Current `main` still has the source-level pieces reported here: noninteractive doctor uses a 3s Gateway RPC timeout, falls into busy-port diagnostics when that RPC times out, and shared port formatting still labels any OpenClaw listener as a conflict; the exact SIGTERM path needs live Linux/systemd or gateway-child proof before closing or automating a fix.

## What This Changes

The issue asks for `openclaw doctor --fix --non-interactive` to avoid partial live-gateway repairs that end in SIGTERM, either by applying only safe-live fixes or by refusing cleanly when a running gateway is detected.

## Best Possible Solution

Define a live-gateway doctor policy, then make noninteractive repair either finish without restart-backed self-termination or fail before mutations with a clear explanation; share the expected-Gateway port-listener fix with #77939.

## Reproduction Assessment

No, not for the full SIGTERM behavior. Source inspection reproduces the 3s Gateway timeout and false busy-port diagnostic path, but the terminating signal itself still needs a live Linux/systemd or Gateway-child reproduction.

## Solution Assessment

Unclear until maintainers choose the live-gateway policy. The narrow technical direction is to avoid treating the expected Gateway listener as a conflict, use a less fragile liveness timeout, and prevent restart-required doctor writes from killing the invoking session.
