## Summary

Keep open. Current main still blocks LINE webhook ACK on event processing in the Node handler, Express middleware, and monitor dispatcher, while the open closing PR remains the right path to finish the fix instead of closing this issue early.

## What This Changes

The issue requests that LINE webhook ingress return HTTP 200 immediately after validation and process events asynchronously so slow agent turns do not cause LINE request timeouts or expired reply-token failures.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/78186, or replace it with an equivalent focused PR, so all LINE ingress paths ACK after validation and process events asynchronously with regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: send a signed LINE webhook whose handleWebhook or onEvents promise stays pending, and current main will not send HTTP 200 until that promise settles.

## Solution Assessment

Yes. The narrow maintainable fix is to ACK after signature/body/target validation, then run trusted LINE event processing fire-and-forget with runtime error logging; the open closing PR already follows that direction.
