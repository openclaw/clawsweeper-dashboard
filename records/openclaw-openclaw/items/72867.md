## Summary

Keep open. Current main still has a single Telegram polling fetch abort signal that `stopRunner` aborts, while existing liveness hardening only reduces some watchdog restarts and does not prove the original session-specific reply-loss symptom is resolved; the open related PR #73050 remains the active targeted fix path.

## What This Changes

The issue reports session-specific Telegram conversational replies failing while direct Telegram message-tool sends still deliver.

## Best Possible Solution

Keep this as the Telegram session-delivery tracker until #73050 or an equivalent fix scopes polling-restart aborts to `getUpdates`, preserves shutdown abort semantics, adds overlapping `sendMessage` regression coverage, and retests the reported session symptom.
