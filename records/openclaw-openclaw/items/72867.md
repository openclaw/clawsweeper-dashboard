## Summary

Keep open. Current main still lacks proof that the session-specific Telegram delivery failure is fixed: conversational replies and direct message-tool sends remain separate paths, the polling restart still aborts the shared Telegram fetch signal, and open PR #73050 is the active targeted fix path rather than shipped evidence.

## What This Changes

Reports Telegram conversational replies failing in one session while direct `message` tool sends still deliver and `/reset` temporarily restores delivery.

## Best Possible Solution

Keep this as the Telegram session-delivery tracker until PR #73050 or an equivalent fix lands with method-scoped polling-restart abort behavior, focused regression coverage, and retest evidence for the reported symptom.
