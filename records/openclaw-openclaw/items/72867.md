## Summary

Keep open. Current main still does not prove the Telegram session-delivery failure is fixed: conversational replies and direct message-tool sends remain separate paths, polling restarts still abort a shared fetch signal that is applied to all Telegram Bot API methods, and #73050 is still open as the targeted fix path.

## What This Changes

The issue requests consistent Telegram conversational reply delivery when direct Telegram message-tool sends still work in the same otherwise healthy connection.

## Best Possible Solution

Keep this as the Telegram session-delivery tracker until #73050 or an equivalent fix lands with scoped polling-restart abort behavior, focused regression coverage, and retest evidence for the reported stuck-reply symptom.
