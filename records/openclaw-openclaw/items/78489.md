## Summary

Keep open: the PR targets a source-visible ACPX Codex auth gap, but the patch introduces blocking credential-state and refresh-token rotation risks and still lacks inspectable real behavior proof.

## What This Changes

The PR adds an ACPX Codex auth sync script and wrapper launch step that writes OpenClaw `openai-codex` OAuth credentials into the isolated Codex ACP `CODEX_HOME`, with a regression test.

## Best Possible Solution

Implement the bridge through an OpenClaw-owned credential sync or refresh path that treats optional auth state as recoverable and keeps refresh-token rotation coordinated with the auth profile store.

## Reproduction Assessment

Yes, from source inspection: current main creates an isolated Codex ACP `CODEX_HOME` and the existing test asserts no OpenClaw OAuth auth is synthesized. I did not run a live Docker ACP session during this read-only review.

## Solution Assessment

No. The proposed direction is plausible, but copying the shared refresh token into an unsynchronized Codex `auth.json` is not the narrowest maintainable solution because it bypasses OpenClaw's existing refresh-token reuse protections.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Do not fork the Codex refresh token into auth.json:** `extensions/acpx/src/codex-auth-bridge.ts:393`
  - body: The sync script copies the same OpenClaw `openai-codex` refresh token into Codex CLI's isolated `auth.json`. Codex can then rotate that token independently of OpenClaw's per-profile refresh lock and source `auth-profiles.json`, leaving one side stale and causing future `refresh_token_reused` failures. Please keep refresh rotation in one OpenClaw-owned path or write back under the existing OAuth coordination mechanism.
  - confidence: 0.86
- **[P2] Treat corrupt auth state as optional:** `extensions/acpx/src/codex-auth-bridge.ts:357`
  - body: `auth-state.json` is only used to prefer `lastGood`, but this unguarded parse makes the wrapper exit before launching Codex if that optional state file is malformed. Wrap this read like `auth-profiles.json` and continue without `lastGood` when it cannot be parsed so valid profiles or existing isolated auth can still be used.
  - confidence: 0.83

## Security Review

Status: needs_attention

Summary: The patch handles OAuth refresh tokens and currently duplicates single-use OpenClaw refresh material into a second mutable Codex CLI auth store without coordinated rotation.

Concerns:

- **[medium] Unsynchronized OAuth refresh-token copy:** `extensions/acpx/src/codex-auth-bridge.ts:393`
  - body: Writing `profile.refresh` into Codex CLI `auth.json` lets the external Codex process refresh and rotate the same credential outside OpenClaw's auth-profile lock and persistence path, which can stale or invalidate the source profile.
