## Summary

This PR should close as superseded: current main and maintainer discussion have moved ACPX/Codex away from materializing Codex auth bridge files, while this branch reintroduces a mutable isolated auth.json bridge with OAuth refresh material.

## What This Changes

The PR changes ACPX to generate and run a Codex auth sync script that maps OpenClaw Codex OAuth credentials into the isolated Codex ACP home and adopts rotated Codex auth afterward.

## Best Possible Solution

Keep the current main contract that avoids materializing Codex auth bridge files; if operators need a different Codex home, handle that through an explicit, reviewed configuration path such as https://github.com/openclaw/openclaw/pull/77006 rather than copying OpenClaw OAuth refresh tokens into ACPX state.

## Reproduction Assessment

Yes. Source inspection shows current main intentionally creates an isolated Codex ACP home without synthesizing auth.json, and the current test covers that no-auth-copy behavior.

## Solution Assessment

No. This PR is not the best solution now because it reintroduces the auth-file materialization direction that current main and the related maintainer closeout superseded.
