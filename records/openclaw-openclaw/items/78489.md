## Summary

Keep open because credential rotation and optional-state error handling still block merge; the supplied Docker output is now enough for the real behavior proof gate.

## What This Changes

The PR adds a generated ACPX Codex auth sync script and wrapper prelaunch hook that converts OpenClaw's main `openai-codex` OAuth profile into isolated `CODEX_HOME/auth.json`, with regression coverage.

## Best Possible Solution

Bridge isolated ACP auth through an OpenClaw-owned sync or refresh path that keeps one refresh-token source of truth and treats optional auth state as recoverable.

## Reproduction Assessment

Yes, source-reproducible: current main creates an isolated Codex ACP `CODEX_HOME` and the existing test asserts no OpenClaw OAuth auth is synthesized. I did not run a live Docker ACP session during this read-only review.

## Solution Assessment

No. The bridge direction is plausible, but copying the shared refresh token into Codex `auth.json` bypasses OpenClaw's refresh lock and persistence path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Do not fork the Codex refresh token into auth.json:** `extensions/acpx/src/codex-auth-bridge.ts:393`
  - body: The sync script copies the same OpenClaw `openai-codex` refresh token into Codex CLI's isolated `auth.json`. Codex can then rotate that token outside OpenClaw's per-profile refresh lock and source store, leaving one side stale or triggering `refresh_token_reused`; keep refresh ownership in one coordinated path or write back under the existing lock.
  - confidence: 0.86
- **[P2] Treat corrupt auth-state.json as optional:** `extensions/acpx/src/codex-auth-bridge.ts:357`
  - body: `auth-state.json` only supplies the preferred `lastGood` profile, but this unguarded parse makes the wrapper exit before launching Codex if that optional state file is malformed. Catch parse failures and continue without `lastGood` so valid profiles or existing isolated auth can still be used.
  - confidence: 0.83

## Security Review

Status: needs_attention

Summary: The patch handles OAuth refresh material and currently duplicates it into a second mutable Codex CLI credential store without coordinated rotation.

Concerns:

- **[medium] Unsynchronized OAuth refresh-token copy:** `extensions/acpx/src/codex-auth-bridge.ts:393`
  - body: Writing `profile.refresh` into Codex CLI `auth.json` lets the external Codex process refresh and rotate the same credential outside OpenClaw's auth-profile lock and persistence path.
  - confidence: 0.84
