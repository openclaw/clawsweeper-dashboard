## What This Changes

This PR mirrors direct cron delivery output into the stored agent-scoped destination session transcript using the outbound payload mirror projection, and adds regression coverage for custom-session plus media/silent-token cron deliveries.

## Best Possible Solution

Keep the current narrow cron-owned implementation, add a single-line Unreleased Fixes changelog entry crediting the reporter/contributor, then proceed with normal maintainer review and exact-head CI gating.

## Reproduction Assessment

Yes. The linked bug provides concrete `openclaw cron add --session session:<id> ...` and `openclaw cron run <jobId>` steps, and current-main code shows direct delivery cannot update transcripts because it omits `params.mirror`.

## Solution Assessment

Yes for the code path. Mirroring the filtered outbound payload projection into the stored agent-scoped session key with the direct-delivery idempotency key is the narrow maintainable fix; the remaining issue is the missing changelog entry, not the implementation shape.
