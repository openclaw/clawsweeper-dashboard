## Summary

Keep this PR open. The branch targets the right cron-owned behavior, and maintainer discussion supports the narrow delivered-text mirror boundary, but the latest diff still mirrors before cron knows whether a best-effort delivery fully succeeded and it is missing the required changelog entry.

## What This Changes

The PR adds a destination-session transcript mirror to direct cron delivery and regression coverage for custom-session and media-only payload projection.

## Best Possible Solution

Land a small cron-owned fix that mirrors only after complete successful delivery, or mirrors only the actually successful payload subset with safe idempotency, plus the required changelog entry.

## Reproduction Assessment

Yes. The linked bug has concrete CLI steps using `openclaw cron add --session session:<id>` and current main shows no direct-delivery mirror; the PR-specific blocker is reproducible with a best-effort multi-payload send where one payload succeeds and another calls `onError`.

## Solution Assessment

No as-is. The proposed cron-owned mirror is the right maintainable surface, but passing the mirror into `deliverOutboundPayloads` before full success is known can record unsent payloads and block a later retry with the same idempotency key.

## Review Findings
