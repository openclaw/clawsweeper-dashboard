## Summary

Keep open. The PR targets a valid cron transcript gap and the discussion supports the narrow delivered-text mirror, but the current diff still mirrors before cron knows whether best-effort delivery fully succeeded and it is missing the required changelog entry.

## What This Changes

The PR adds direct cron delivery transcript mirroring into the stored destination session and regression tests for custom-session text/media projections.

## Best Possible Solution

Land a cron-owned fix that appends the exact delivered projection to the destination transcript only after full delivery success, with partial best-effort regression coverage and a changelog entry.

## Reproduction Assessment

Yes. The linked bug has concrete CLI steps, and the PR-specific regression is reproducible at the unit seam by making one best-effort payload fail while another succeeds.

## Solution Assessment

No as-is. Mirroring the delivered projection is the right direction, but the mirror must be deferred or gated until cron has confirmed `delivered=true`.
