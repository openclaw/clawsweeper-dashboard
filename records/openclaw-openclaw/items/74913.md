## Summary

Keep open: the PR is pointed at the right cron transcript bug and now mirrors the filtered direct-delivery projection, but it still mirrors before cron can reject a partial best-effort send and it is missing the required changelog entry.

## What This Changes

The PR adds direct cron delivery transcript mirroring for `session:<id>` targets and regression coverage for text plus media-only `NO_REPLY` projections.

## Best Possible Solution

Land the cron-owned mirror only after a fully successful direct delivery, preserve the filtered payload projection and idempotency semantics, add a partial best-effort regression test, and add the required changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: use best-effort direct cron delivery with multiple payloads where one send succeeds and another invokes `onError`; `deliverOutboundPayloads` appends the mirror while cron still returns `delivered=false` for retry.

## Solution Assessment

No as-is. Mirroring the filtered delivered projection into the destination session is the right direction, but the mirror must be gated on full delivery success or limited to the actually successful payload subset before merge.
