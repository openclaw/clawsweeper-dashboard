## Summary

Keep open. The cron-owned fix is still useful and matches the maintainer-supported boundary, but the current patch can mirror unsent payloads after partial best-effort delivery and is missing the required changelog entry.

## What This Changes

The PR adds direct cron delivery transcript mirroring for `session:<id>` targets and regression tests for text/media mirror projection.

## Best Possible Solution

Land the cron-owned mirror after it only appends on fully successful delivery, or only for the actually successful subset, and include the user-facing changelog entry.

## Reproduction Assessment

Yes. The linked bug has concrete CLI steps, docs define custom sessions as persistent history, and current main source shows direct cron delivery reaches outbound delivery without a transcript mirror; the PR-specific blocker is also source-reproducible from best-effort `onError` plus `results.length > 0` mirror append.

## Solution Assessment

No as-is. Mirroring the delivered direct payload into the destination session is the right maintainable boundary, but the mirror must wait until cron knows the delivery was fully successful or must be scoped to the successful payload subset.
