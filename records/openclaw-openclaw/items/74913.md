## Summary

Keep open: the PR targets the right cron-owned behavior, but the current diff still can write a destination transcript mirror during partial best-effort delivery before cron leaves the job retryable, and it is missing the required changelog entry.

## What This Changes

The PR adds direct cron delivery transcript mirroring for `session:<id>` jobs using the stored agent-scoped session key and filtered payload projection, plus regression coverage for text and media delivery.

## Best Possible Solution

Land a small cron-owned fix that mirrors exactly the delivered payload into the destination transcript only after full delivery success, with partial-failure coverage and a single-line changelog entry; keep #74286 and BlueBubbles-specific #75529 separate.

## Reproduction Assessment

Yes. Current main source shows direct cron delivery has no transcript mirror despite `session:<id>` being documented as persistent context; the PR-specific blocker is also source-reproducible because best-effort `onError` can occur while `deliverOutboundPayloads` still appends a mirror after any result.

## Solution Assessment

No as-is. The stored-session and filtered-projection approach is the right boundary, but mirroring must be gated until full delivery success or moved after cron determines `delivered=true`, and the required changelog entry must be added.
