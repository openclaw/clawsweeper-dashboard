## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Do not put fatal payloads in argv:** `src/infra/fatal-error-hooks.ts:61`
  - body: `payload` includes the raw error message and stack, then passes it as a command-line argument. Node treats `spawn` args as command-line args, and Linux exposes running process command lines through `/proc/<pid>/cmdline`, so a configured handler can leak API keys, prompts, or private paths before the handler even runs. Use a non-argv transport and the existing redaction/diagnostic contract instead. ([nodejs.org](https://nodejs.org/api/child_process.html))
  - confidence: 0.91
