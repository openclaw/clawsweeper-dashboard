Full review comments:

- **[P2] Avoid passing fatal payloads through argv:** `src/infra/fatal-error-hooks.ts:61`
  - body: The payload includes the raw error `message` and truncated `stack`, then passes it as a command-line argument. On normal hosts argv can be visible through process listings or audit logs before the handler can redact it, so crashes containing tokens, private endpoints, paths, or user data escape the diagnostics privacy model. Please redesign this to avoid argv exposure or send only a redacted/minimized payload before adding the env surface.
