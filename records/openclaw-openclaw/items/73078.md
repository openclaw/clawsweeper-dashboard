## Summary

Keep open. Current main still implements `non-main` as a per-agent main-session comparison, so a dynamically created peer agent's own `:main` session can inherit defaults and bypass sandboxing; the requested compact main-agent-only policy is still missing.

## What This Changes

The issue requests clearer or new sandbox policy semantics so only the primary `main` agent bypasses sandboxing while dynamic peer agents stay sandboxed by default.

## Best Possible Solution

Add a backward-compatible, schema-backed main-agent-only sandbox policy or equivalent selector, update `sandbox explain` and public docs, and preserve existing `non-main` behavior unless maintainers explicitly choose a migration.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: with defaults set to `sandbox.mode: "non-main"` and session `agent:feishu-ou_peer:main`, the resolved agent and resolved main session key both point to the peer agent, so `shouldSandboxSession` returns `false`.

## Solution Assessment

Unclear until maintainers choose the public security and compatibility contract. An additive mode appears safer than redefining `non-main`; documentation-only clarification reduces surprise but does not remove the reported misconfiguration hazard.
