## Summary

Keep open. Current main still implements `non-main` as a per-agent main-session comparison, so dynamic peer agents that inherit the default can leave their primary DM session unsandboxed; the docs clarify the session-based behavior and per-agent override workaround, but the requested main-agent-only policy does not exist.

## What This Changes

Request a sandbox policy or clarification so only the default `main` agent can bypass sandbox while dynamic peer agents remain sandboxed without verbose per-agent overrides.

## Best Possible Solution

Add a backward-compatible, schema-backed main-agent-only sandbox policy or equivalent selector, update `sandbox explain` and public docs, and preserve existing `non-main` semantics unless maintainers explicitly choose a migration.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: with defaults set to `sandbox.mode: "non-main"` and session `agent:feishu-ou_peer:main`, the resolved agent and resolved main session key both point to that peer agent, so `shouldSandboxSession` returns `false`.

## Solution Assessment

Unclear until maintainers choose the public security and compatibility contract. An additive mode appears safer than redefining `non-main`, while documentation-only clarification does not remove the reported misconfiguration hazard.
