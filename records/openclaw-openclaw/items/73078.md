## Summary

Keep open. Current main still treats `sandbox.mode: "non-main"` as a comparison against the resolved agent's own main session, and the schema/docs do not expose a main-agent-only sandbox policy for dynamic peer agents.

## What This Changes

Request a sandbox policy or clearer documentation for keeping only the `main` agent unsandboxed while dynamic or peer agents stay sandboxed.

## Best Possible Solution

Add a backward-compatible, schema-backed main-agent-only sandbox policy or equivalent selector, update `sandbox explain` and public docs, and preserve existing `non-main` semantics unless maintainers explicitly choose a migration.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: with defaults set to `sandbox.mode: "non-main"`, `agent:feishu-ou_peer:main` resolves to agent `feishu-ou_peer`, compares against that same agent's main session key, and returns `sandboxed: false`.

## Solution Assessment

Unclear until secops/product owners choose the public compatibility contract. An additive mode appears safer than redefining `non-main`, while docs alone do not remove the reported misconfiguration hazard.
