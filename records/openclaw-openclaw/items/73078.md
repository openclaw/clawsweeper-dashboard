## Summary

Keep open. Current main still implements `non-main` as a per-resolved-agent main-session comparison, so dynamic peer agents can inherit defaults and leave their primary DM session unsandboxed; docs clarify the session semantics, but the requested main-agent-only policy is still missing and needs maintainer/security policy judgment.

## What This Changes

Request a clearer or new sandbox policy so only the main agent bypasses sandbox while peer or dynamic agents remain sandboxed without verbose per-agent overrides.

## Best Possible Solution

Add a backward-compatible, schema-backed main-agent-only sandbox policy or equivalent selector, update `sandbox explain` and public docs, and preserve existing `non-main` semantics unless maintainers explicitly choose a migration.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with `agents.defaults.sandbox.mode: "non-main"` and session `agent:feishu-ou_peer:main`, the runtime resolves the session agent as `feishu-ou_peer`, computes that same agent's main key, and returns `sandboxed: false`.

## Solution Assessment

Unclear until maintainers choose the public security and compatibility contract. An additive main-agent-only mode appears safer than redefining `non-main`, while documentation-only clarification does not remove the reported misconfiguration hazard.
