## Summary

Keep open. Current main still treats `sandbox.mode: "non-main"` as a comparison against the resolved agent's own main session, and the public schema still accepts only `off`, `non-main`, and `all`, so the requested main-agent-only policy remains missing.

## What This Changes

Request a sandbox policy or docs/config change that lets only the default/main agent bypass sandboxing while dynamic peer agents stay sandboxed on their primary sessions.

## Best Possible Solution

Add a backward-compatible, schema-backed main-agent-only sandbox policy or equivalent selector, update `sandbox explain` and public docs, and preserve existing `non-main` semantics unless maintainers explicitly choose a migration.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: with default `sandbox.mode: "non-main"`, `agent:feishu-ou_peer:main` resolves to agent `feishu-ou_peer`, compares against that same agent's main session key, and returns `sandboxed: false`.

## Solution Assessment

Unclear until maintainers choose the public security and compatibility contract. An additive mode appears safer than redefining `non-main`, while documentation-only clarification does not remove the reported misconfiguration hazard.
