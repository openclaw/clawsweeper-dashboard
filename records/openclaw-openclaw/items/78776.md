## Summary

Keep open: current main has exec approval routing, config validation/backups, and subagent fan-out controls, but it still lacks native config-change approvals and the requested critic-agent review gate. The open policy PR and verifier-agent issue are useful adjacent work, but neither fully owns the config-write approval/restart workflow requested here.

## What This Changes

Add optional configurable approvals for exec commands and config writes/restarts, with session-bound routing, diff/backup prompts, and optional critic-agent review.

## Best Possible Solution

Design a narrow core config-write approval contract that reuses existing approval routing, schema validation, base-hash checks, restart follow-up, and backup machinery; decide separately whether critic-agent review is a core fail-closed hook or policy/plugin-backed verifier.

## Reproduction Assessment

Not applicable: this is a feature and security-design request, not a report of broken existing behavior. Source inspection gives a high-confidence current-main gap for native config-change approvals and critic review.

## Solution Assessment

Unclear until maintainer design review. The direction is plausible, but the best solution should avoid an ad hoc umbrella and instead define one approval contract for config writes/restarts plus a separately reviewed verifier hook.
