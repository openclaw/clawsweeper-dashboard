## Summary

Keep open: current main already has exec approvals, config validation/backups, and subagent fan-out controls, but it does not provide native config-change approvals or an optional critic-agent review gate. This is core gateway/security product work, not a ClawHub-only extension.

## What This Changes

Request an optional approval policy that gates exec commands and OpenClaw config changes, keeps approval prompts bound to the originating session/channel, and can run a critic agent with diff/backup context before applying changes.

## Best Possible Solution

Design a narrow core config-write approval extension that reuses the existing approval routing and config backup/validation machinery, then decide separately whether critic-agent review belongs in core or as a plugin-backed optional hook.
