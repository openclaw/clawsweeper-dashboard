## Summary

Keep open: current main has exec approval routing, partial config safety primitives, and subagent fan-out limits, but it still lacks native config-change approval and critic-agent review. Related open issues cover protected config approvals, and the related policy PR is only draft groundwork, so this broader approval-design request is not obsolete.

## What This Changes

Request an optional approval policy covering exec commands, OpenClaw config writes, same-channel routing, diffs/backups, and optional critic-agent review.

## Best Possible Solution

Converge this with the protected-config approval design so core can stage exact diffs bound to base hash, session, owner identity, expiry, and risk class, while deciding separately whether critic review is a core hook or plugin-provided check.

## Reproduction Assessment

Not applicable. This is a feature request; source review shows adjacent exec approval, config mutation, backup, and subagent guardrail behavior, but no existing contract that current main is failing.

## Solution Assessment

Unclear. The requested direction is plausible, but the best maintainable solution should reuse existing approval routing and config validation while coordinating with the open protected-config approval issues rather than bolting config writes onto exec approvals unchanged.
