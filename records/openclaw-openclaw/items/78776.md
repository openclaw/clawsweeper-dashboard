## Summary

Keep open: current main has exec/plugin approval routing, admin-scoped config writes with validation/backups, protected agent-side config guards, and subagent fan-out controls, but it still lacks native config-change approvals and critic-agent pre-approval. The overlap with existing open config-approval issues is useful, but the critic-review and umbrella approval-scope design still need maintainer product/security judgment.

## What This Changes

Request an optional approval framework for exec and config changes that routes by originating session and can run a critic-agent review before human approval.

## Best Possible Solution

Design a narrow core config-change approval contract that reuses existing approval routing, binds decisions to exact diff/base hash/session/owner identity, and treats critic review as a separately decided optional advisory hook.

## Reproduction Assessment

Not applicable. This is a feature request, and source review shows adjacent exec approval, config write, backup, and subagent guardrail behavior rather than a broken existing contract to reproduce.

## Solution Assessment

Unclear. The requested direction is plausible, but the maintainable path should split core config approval mechanics from optional critic-agent review and preserve the current fail-closed model/operator boundary.
