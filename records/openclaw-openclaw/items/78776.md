## Summary

Keep open: current main covers exec/plugin approval routing and already has config validation, base-hash checks, backups, and restart follow-up, but it still does not provide native config-change approvals or a critic-agent review gate. The related policy PRs are useful adjacent work, but they are open drafts and intentionally do not add the config-write approval/restart workflow requested here.

## What This Changes

The issue requests an optional approval system covering exec commands, gateway config changes, session-bound routing, critic-agent review, diffs, backups, and restart handling.

## Best Possible Solution

Design a narrow core config-write approval contract that reuses existing approval routing, schema validation, base-hash checks, restart follow-up, and backup machinery; treat critic-agent review as a separately reviewed policy/verifier hook.

## Reproduction Assessment

Not applicable: this is a feature and security-design request, not a broken existing behavior report. Source inspection gives high-confidence evidence that native config-change approvals and critic review are still absent on current main.

## Solution Assessment

Unclear until maintainer design review. The direction is plausible, but the best solution should avoid an umbrella ad hoc flow and define one approval contract for config writes/restarts plus a separately reviewed verifier hook.
