## Summary

Keep open: this PR targets a real provider-catalog image-input gap, but the branch is stale against current main and does not correctly update the manifest-backed Volcengine and BytePlus runtime catalogs. The proposed diff also changes the preview model despite the stated scope boundary.

## What This Changes

The PR updates Volc coding model capability metadata to advertise image input for selected shared coding-plan models, adds Volcengine/BytePlus assertions, and changes Volcengine preview-model metadata.

## Best Possible Solution

Update the current manifest-backed Volcengine and BytePlus coding-plan catalogs only for verified image-capable models, keep doubao-seed-code-preview-251028 text-only unless separately proven, align provider docs/changelog, and add extension-level regression tests proving both plugin catalogs expose the intended capability.

## Reproduction Assessment

Yes. A source-only reproduction is clear: inspect the current volcengine-plan and byteplus-plan manifest rows, then follow the transport gate that drops image parts when model.input lacks image.

## Solution Assessment

No. The proposed patch is not the best current-main solution because it changes stale/removed agent files, misses the current BytePlus manifest runtime path, and broadens the preview model outside the stated scope; the safer fix is provider-plugin manifest metadata plus focused extension tests.
