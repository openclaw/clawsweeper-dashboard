- `impact:message-loss`: The reported conflict loop blocks inbound Telegram updates and command replies, causing dropped or delayed channel messages.

## Summary

Keep open: current main and v2026.6.8 include useful Telegram 409 mitigations, but source still routes conflicts through restart/backoff and fire-and-forget dirty transport recovery rather than proving same-token stale long-poll overlap is eliminated. The recent comments add credible newer macOS, Windows, and isolated-ingress measurements, so this remains a live canonical Telegram polling lifecycle bug.

## What This Changes

The issue requests that a single local OpenClaw Telegram gateway maintain only one active getUpdates long-poll per bot token and stop self-conflicting with 409 errors.

## Best Possible Solution

Keep this issue as the canonical Telegram polling self-conflict tracker and land a Telegram-owned lifecycle fix that prevents same-token overlapping getUpdates, with redacted live Bot API proof on the affected path.

## Reproduction Assessment

No high-confidence live current-main repro was run in this read-only review. The report and later comments provide credible stop/start and timing evidence, and current source still contains the restart/dirty-rebuild path that matches the logs.

## Solution Assessment

No current fix is proven complete. The merged 409 handling is a useful mitigation, but the best fix should address Telegram polling cancellation, lifecycle timing, and restart behavior with live Bot API validation.
