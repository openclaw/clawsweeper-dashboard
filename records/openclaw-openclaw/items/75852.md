The issue reports Telegram `getUpdates` 409 self-conflicts in polling mode on a single gateway process, especially around stale/restarted long-poll lifecycle handling.

## Best Possible Solution

Keep this issue as the canonical Telegram polling lifecycle bug until maintainers either land a focused inter-poll/stale-lease timing fix with live Bot API proof or explicitly decide that the remaining behavior is covered elsewhere.
