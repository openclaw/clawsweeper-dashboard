## Summary

Keep open: current main and the latest release still have the non-awaited Telegram stale-transport close path that can match the reported self-conflict loop, and open PR #76153 already targets this issue with a narrow lifecycle fix.

## What This Changes

The issue reports Telegram polling 409 self-conflicts on macOS and asks OpenClaw to avoid overlapping long-poll transports when only one local gateway is running.

## Best Possible Solution

Land #76153 or an equivalent Telegram plugin fix that serializes stale transport teardown before the replacement polling cycle starts, with regression coverage for slow stale closes.

## Reproduction Assessment

Yes. The reporter gives a concrete macOS stop/start Bot API reproduction, and source inspection on current main shows the non-awaited dirty rebuild close path that can plausibly create overlapping Telegram long-poll lifecycles.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Telegram plugin lifecycle path by awaiting or otherwise serializing stale transport close before the next polling cycle starts; no core policy or config change is needed.
