## Summary

Keep open: current main has safeguards for explicit non-443 gateway ports and TLS failure feedback, but the Android pairing-required path still contains a source-visible retry loop and an open related PR is not yet a clean landed fix.

## What This Changes

The issue asks OpenClaw to stop Android gateway authorization from silently retrying forever when a custom TLS route cannot use port 443.

## Best Possible Solution

Land or replace the Android fix with a rebased, security-preserving patch that limits pairing-required retries, keeps explicit custom TLS ports intact, and includes focused Android tests plus redacted runtime proof.

## Reproduction Assessment

Partially yes: current main source clearly reproduces the pairing-required retry loop via PairingAutoRetryEffect, but I did not reproduce the full Android custom-port flow live and current source already preserves explicit non-443 ports.

## Solution Assessment

Unclear as a single issue-level fix: the maintainable path is to target the remaining retry churn and any setup-code route that omits a custom port, not to broadly rewrite Android auth or weaken gateway TLS policy.
