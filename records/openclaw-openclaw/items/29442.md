## Summary

Keep open. Current main has meaningful prompt-injection hardening for external cron, hook, and web content, but it still routes wrapped untrusted content into the normal cron agent prompt path. It does not implement the requested default read-only worker LLM, schema-validated worker-to-manager handoff, or privileged manager continuation boundary.

## What This Changes

Add privilege separation for cron jobs by routing untrusted external content through a read-only worker model and structured sanitizer/tagging before privileged manager turns continue.

## Best Possible Solution

Define an explicit security design for external-content ingestion that either adds a first-class read-only/tool-disabled reader-worker flow for cron and related sources, or introduces a structured taint/sanitizer continuation seam that can block or replace untrusted content before any privileged manager turn proceeds. The design should keep ClawMoat and the related memory-capture issues as inputs, then add regression coverage proving untrusted external content cannot silently reach privileged cron execution or memory capture as ordinary prompt text.
