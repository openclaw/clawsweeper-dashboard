## Summary

Keep open. Current main still starts manual `/compact` without an abort signal or active embedded-run registration, so the reported cancel paths still cannot discover or interrupt that compaction.

## What This Changes

Manual `/compact` should be cancellable through existing chat abort, `/stop`, and `abortEmbeddedPiRun(sessionId)` paths while compaction is in progress.

## Best Possible Solution

Wire manual compaction into the existing embedded-run cancellation lifecycle, forward or bridge the abort signal into compaction, clear registration in `finally`, and add focused cancellation regression coverage.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current main calls manual `/compact` without `abortSignal` or active-run registration, while `/stop` and `abortEmbeddedPiRun(sessionId)` depend on active-run or reply-run discovery.

## Solution Assessment

Yes. Reusing the existing `abortSignal` parameter and `ACTIVE_EMBEDDED_RUNS` lifecycle is the narrow maintainable fix; a separate cancel mechanism would duplicate current run lifecycle machinery.
