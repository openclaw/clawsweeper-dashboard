## Label Justifications

- `P2`: The bug can leave users unable to cancel a long-running manual compaction, but it is bounded to one command and is not data loss, a crash loop, or a broad outage.
- `impact:session-state`: The failure is caused by in-progress compaction not being represented in active session/run state, so cancellation state diverges from real work.

## Summary

Keep open: current main and the latest release still start manual `/compact` without a discoverable active-run handle, and the open linked PR is the candidate fix rather than merged implementation.

## What This Changes

The issue asks for manual `/compact` to be cancelable through `/stop`, `chat.abort`, and `abortEmbeddedAgentRun` by registering compaction as active work and wiring abort propagation.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/90821 or an equivalent narrow fix that registers only standalone manual queued compaction as abortable active work while preserving budget/preflight run ownership.

## Reproduction Assessment

Yes, source-level: current main starts manual `/compact` without registering the compaction as active work, while the abort primitive only cancels registered embedded or reply runs. I did not run a live slow compaction in this read-only review.

## Solution Assessment

Yes, keeping this issue open behind the linked fix PR is the best current path. The desired implementation should register manual queued compaction while it is executing, pass an abort signal through, and avoid replacing real active runs used by budget/preflight paths.
