## Label Justifications

- `P2`: The bug can leave users unable to cancel a long-running manual compaction, but it is bounded to one command and is not data loss, a crash loop, or a broad outage.
- `impact:session-state`: The failure is caused by in-progress compaction not being represented in active session/run state, so cancellation state diverges from real work.

## Summary

Keep open: current main and the latest release still start manual `/compact` without an active-run registration that cancel paths can discover, and https://github.com/openclaw/openclaw/pull/90821 is the open linked fix candidate rather than merged implementation proof.

## What This Changes

The issue asks for manual `/compact` to register as abortable active session work so `/stop`, `chat.abort`, and `abortEmbeddedAgentRun` can cancel an in-progress compaction.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/90821, or replace it with an equivalent narrow fix that registers only manual queued compaction as abortable active work and preserves budget/preflight compaction isolation.

## Reproduction Assessment

Yes, source-level: current main starts manual `/compact` without an active-run registration or command-created abort signal, while the abort primitive only cancels registered embedded or reply runs. I did not run a live slow compaction in this read-only review.

## Solution Assessment

Yes, keeping the issue open behind the linked fix PR is the best current path. The narrow implementation direction is to register manual queued compaction only while it is executing, pass an abort signal through, and avoid replacing real active runs used by budget/preflight paths.
