## Summary

Keep open. Current main still starts manual `/compact` without passing an abort signal or registering the compaction as an active embedded run, so the reported cancel paths remain unable to discover or interrupt it.

## What This Changes

The issue asks for manual `/compact` to be cancellable through the existing `chat.abort`, `/stop`, and `abortEmbeddedPiRun(sessionId)` paths.

## Best Possible Solution

Wire manual compaction into the existing embedded-run cancellation lifecycle, forward or bridge the caller abort signal, clear registration in `finally`, and add focused cancellation regression coverage.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: current main calls manual `/compact` without `abortSignal` or active-run registration, while the existing abort paths only reach registered active handles or active reply runs.

## Solution Assessment

Yes. Reusing the existing `abortSignal` parameter and `ACTIVE_EMBEDDED_RUNS` lifecycle is the narrow maintainable fix; adding a separate cancel mechanism would duplicate current run-lifecycle machinery.
