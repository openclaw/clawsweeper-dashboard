## Summary

Keep open. Current main still has the reported cancellation gap: manual `/compact` starts compaction without an abort signal or active-run registration, and the linked implementation PR is still open and unmerged.

## What This Changes

The issue requests making manual `/compact` cancellable through existing abort paths by passing an abort signal and registering the compaction in active embedded runs.

## Best Possible Solution

Land a focused fix that registers an in-flight compact operation as a compacting embedded run, passes its AbortController signal into compactEmbeddedPiSession, clears the handle in finally, and adds regression coverage for `/stop` or abortEmbeddedPiRun cancelling compaction. The gateway `sessions.compact` path should be fixed in the same patch or split into an explicit linked follow-up.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: `/compact` enters handleCompactCommand, the compaction call has no abortSignal, and abortEmbeddedPiRun only cancels registered active handles or reply runs.

## Solution Assessment

Yes. The narrow maintainable solution is to reuse the existing active-run registry and existing compaction abortSignal parameter rather than adding a parallel cancellation mechanism.
