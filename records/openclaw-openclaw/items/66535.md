## Summary

Keep open. Current main still starts manual `/compact` without passing an abort signal or registering the compaction as an active embedded run, and the latest linked fix PR is closed unmerged.

## What This Changes

Request cancellation support for in-progress manual `/compact` runs through existing chat abort, `/stop`, and embedded-run abort paths.

## Best Possible Solution

Add a focused compact-cancellation fix that registers manual compactions as compacting active runs, forwards/bridges abort signals into `compactEmbeddedPiSession`, clears the handle in `finally`, and adds regression coverage for `/stop` or `abortEmbeddedPiRun`.

## Reproduction Assessment

Yes, source-level reproduction is high confidence. Current main calls manual `/compact` without `abortSignal` or active-run registration, while `/stop` and `abortEmbeddedPiRun(sessionId)` depend on active-run or reply-run discovery.

## Solution Assessment

Yes. Reusing the existing `abortSignal` parameter and `ACTIVE_EMBEDDED_RUNS` lifecycle is the narrow maintainable fix; a separate cancellation mechanism would duplicate existing run lifecycle machinery.
