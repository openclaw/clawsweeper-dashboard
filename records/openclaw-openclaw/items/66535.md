- `impact:session-state`: The issue is about active-run/session lifecycle state failing to represent an in-progress manual compaction, so cancel paths cannot find it.

## Summary

Keep open: current main and `v2026.6.6` still start manual `/compact` without a discoverable abort handle for the new compaction, and the open closing implementation is https://github.com/openclaw/openclaw/pull/90821. The issue should remain open until that PR, or an equivalent narrow fix, lands.

## What This Changes

Manual `/compact` should be cancelable through existing embedded-run abort paths while compaction is in progress.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/90821, or an equivalent narrow fix, so manual context-engine `/compact` is registered as abortable only while it runs and is cleaned up afterward.

## Reproduction Assessment

Yes, source-level. Current main calls manual `compactEmbeddedAgentSession` without creating an abort signal or active-run handle, while `/stop` and direct abort paths only work against registered active runs.

## Solution Assessment

Yes. The best current direction is the linked PR's queued-task active-run lifecycle, because it scopes abortability to the actual compaction work instead of registering a command-entry handle that could race queue ordering.
