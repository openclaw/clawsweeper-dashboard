## Summary

Keep open. Current main still has the reported cancellation gap in the manual `/compact` path, and the linked implementation PR #66894 remains open and unmerged, so this issue should not be closed before that work lands or is replaced.

## What This Changes

The issue asks for `/compact` compaction to be cancellable through existing abort paths and visible in the active embedded-run registry while compaction is in progress.

## Best Possible Solution

Land or replace PR #66894 with a focused fix that registers an in-flight manual compaction as an active compacting embedded run, passes an AbortController signal into compactEmbeddedPiSession, clears the handle in finally, and adds regression coverage for `/stop` or abortEmbeddedPiRun cancelling compaction. The gateway `sessions.compact` path should be handled in the same patch or tracked as an explicit linked follow-up if maintainers intentionally split it.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: `/compact` reaches handleCompactCommand, the compaction call has no abortSignal or active-run registration, and `/stop`/fast abort ultimately rely on abortEmbeddedPiRun finding an active handle.

## Solution Assessment

Yes. Reusing the existing abortSignal parameter and ACTIVE_EMBEDDED_RUNS registration is the narrow maintainable fix; a parallel cancellation mechanism would duplicate existing run lifecycle machinery.
