## Summary

Keep open: current main and the latest release still do not give manual `/compact` a discoverable abort handle, and the open linked PR is the active implementation path. The issue should close only after that PR, or an equivalent narrow fix, lands.

## What This Changes

The issue reports that manual `/compact` starts long-running compaction without an active abort handle, so `/stop` and other abort paths cannot cancel it.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/90821, or an equivalent narrow fix, so manual queued compaction is registered as abortable only while it runs and cleaned up afterward.

## Reproduction Assessment

Yes, source-level. Current main starts manual `/compact` without a command-owned abort signal or active-run registration, while the abort path only finds active embedded/reply runs.

## Solution Assessment

Yes. The maintainable direction is to register an abortable handle at the queued manual compaction execution boundary, then clear it in cleanup; the open linked PR is the current candidate for that shape.
