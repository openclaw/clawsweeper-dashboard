## What This Changes

Reports that manual `/compact` cannot be canceled because the command-started compaction is not wired into the active embedded-run abort lifecycle.

## Best Possible Solution

Review and land the linked fix PR, or an equivalent narrow fix, so queued manual context-engine `/compact` is registered as abortable only while it runs and is cleaned up afterward.

## Reproduction Assessment

Yes, source-level. Current main shows `/compact` starts manual compaction without registering the new compaction in active-run state or passing a command-owned abort signal; I did not run a live gateway reproduction in this read-only review.

## Solution Assessment

Yes. The best current path is the linked PR's queued-task active-run registration, because it scopes abortability to the actual compaction work instead of registering at command entry; it still needs normal PR proof/review before this issue can close.
