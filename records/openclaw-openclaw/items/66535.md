## Summary

Keep open. Current main still starts manual `/compact` without passing an abort signal or registering the compaction as an active embedded run, so `/stop`, `chat.abort`, and direct `abortEmbeddedPiRun(sessionId)` still cannot discover or interrupt it; the linked fix PRs are all closed and unmerged.

## What This Changes

The issue reports that manual `/compact` cannot be canceled because the command does not wire compaction into the existing abort and active-run lifecycle.

## Best Possible Solution

Manual compaction should participate in the same active-run lifecycle as embedded turns so existing stop/abort APIs interrupt it and observers can see the in-flight state.

## Reproduction Assessment

Yes. Source inspection on current main shows `/compact` calls compaction without `abortSignal` and without active-run registration, while `/stop` and `abortEmbeddedPiRun` only reach registered handles or active reply runs.

## Solution Assessment

Yes. Reusing the existing `abortSignal` parameter and `setActiveEmbeddedRun`/`clearActiveEmbeddedRun` lifecycle is the narrow maintainable fix; a separate cancel mechanism would duplicate current run machinery.
