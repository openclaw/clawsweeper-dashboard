## Summary

Keep open. Current main still leaves manual `/compact` outside the embedded-run cancellation registry, and open PR #66894 references this issue with closing syntax but remains unmerged, so this should stay open until that PR lands or is replaced.

## What This Changes

The issue requests cancellable manual `/compact` runs by wiring an abort signal and active embedded-run registration into the compaction lifecycle.

## Best Possible Solution

Land or replace PR #66894 with a focused fix that registers manual compactions as active compacting runs, passes abortSignal into compactEmbeddedPiSession, clears the handle in finally, and adds regression coverage for `/stop` or abortEmbeddedPiRun cancellation.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: the manual `/compact` handler starts compaction without abortSignal or ACTIVE_EMBEDDED_RUNS registration, while `/stop` and abortEmbeddedPiRun depend on that registry to find cancellable work.

## Solution Assessment

Yes. Reusing the existing abortSignal parameter and active embedded-run lifecycle is the narrow maintainable direction; a separate cancellation mechanism would duplicate existing run lifecycle machinery.
