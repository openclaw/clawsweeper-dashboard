## Summary

Keep open: current main and the latest release still let manual `/compact` run without an active-run registration that `/stop` can discover, and the open linked pull request is the active fix candidate rather than merged implementation proof.

## What This Changes

The issue requests that manual `/compact` compaction be cancelable through the same active embedded-run abort paths used by normal agent runs.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/90821, or replace it with an equivalent narrow fix that registers only manual queued compaction as abortable active work and preserves budget/preflight compaction isolation.

## Reproduction Assessment

Yes, from source inspection. Current main still has `/compact` call the compaction runner without registering the new manual compaction in `ACTIVE_EMBEDDED_RUNS`, while the abort primitive only cancels registered active runs or reply runs.

## Solution Assessment

Yes, the linked PR's direction appears to be the right narrow fix shape: register manual queued compaction only while it is executing, pass an abort signal through, and avoid replacing real active runs used by budget/preflight paths.
