## Best Possible Solution

Land the stricter admin-only non-operator approval policy only after maintainers accept the compatibility break and the PR body includes redacted real trusted-proxy/shared-auth WebSocket output for the current head.

## Solution Assessment

Yes, with merge gates: moving the pending-request check to all non-admin callers is the narrow code fix and preserves the existing device-token ownership check. Because it tightens existing shared-auth behavior, maintainer acceptance and real behavior proof are still required before merge.
