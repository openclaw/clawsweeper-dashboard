## What This Changes

## Best Possible Solution

Land a focused status fix that uses the selected session provider/model for usage when stale runtime state lacks fallback/equivalent-runtime provenance, while preserving active-runtime usage for real fallback and runtime-alias cases.

## Reproduction Assessment

## Solution Assessment

No, not yet. Moving selected parsing into the shared resolver is the right shape, but usage selection needs the same stale-runtime versus fallback provenance guard that the context path already uses.
