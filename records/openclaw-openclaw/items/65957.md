## What This Changes

The branch adds a target-none heartbeat prompt fallback for direct internal WebChat sessions, moves main/direct session-key helpers into `src/sessions/session-key-utils.ts`, and adds regression tests.

## Best Possible Solution

Keep the WebChat target-none fallback, but restore strict direct-key scoping or explicit reserved-scope rejection, rebase on current main, and add redacted after-fix WebChat proof before merge.

## Reproduction Assessment

Yes by source inspection. Current main resolves `heartbeat.target: "none"` to no delivery and only enables relay wording when `delivery.channel !== "none"`, so the direct internal WebChat target-none path remains internal-only.

## Solution Assessment

No, not yet. The fallback is the right narrow direction, but the shared direct-session helper needs the current deliverable-channel/reserved-scope guard restored and the rebased branch needs real behavior proof.
