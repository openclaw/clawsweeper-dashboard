## What This Changes

This PR adds a heartbeat prompt fallback for direct internal WebChat routes when `heartbeat.target` is `none`, moves main/direct session-key helpers into `src/sessions/session-key-utils.ts`, and adds focused heartbeat/session-delivery tests.

## Best Possible Solution

Keep the target-none WebChat fallback, but preserve strict direct-session scoping or keep the stricter helper local to session delivery, add regression coverage for reserved/unknown direct-like keys, rebase, and provide redacted WebChat proof.

## Reproduction Assessment

Yes, by source inspection rather than a live run. Current main gates relay wording on `delivery.channel !== "none"`, and the PR's helper broadening is visible in the changed `isDirectSessionKey` implementation.

## Solution Assessment

No, not as written. The target-none WebChat fallback is a reasonable direction, but the shared helper must preserve the current strict session-key contract and the PR needs real WebChat proof before merge.
