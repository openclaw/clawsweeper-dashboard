## Summary

Keep open: the bug is source-reproducible and the PR targets the right heartbeat/WebChat seam, but current head still widens direct-session key classification and lacks contributor real behavior proof, so it should not merge yet.

## What This Changes

The PR adds a target-none direct WebChat heartbeat relay fallback, moves main/direct session-key helpers into `src/sessions/session-key-utils.ts`, and adds heartbeat/session delivery regression tests.

## Best Possible Solution

Keep the direct WebChat fallback, preserve current main's strict direct-session key contract for reserved/internal scopes, add a reserved-key heartbeat regression, and require redacted after-fix behavior proof before merge.

## Reproduction Assessment

Yes, by source inspection. Current main resolves `heartbeat.target: "none"` to no-delivery and gates relay wording on `delivery.channel !== "none"`, so a direct internal WebChat session with queued exec/cron work remains internal-only; I did not run a live heartbeat flow in this read-only review.

## Solution Assessment

No, not as currently written. The fallback direction is narrow, but the helper extraction drops the reserved-scope guard; the safer fix is to keep the fallback while matching current main's strict direct-session key contract.
