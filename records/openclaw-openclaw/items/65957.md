## Summary

Keep open. Current main still lacks the direct WebChat `heartbeat.target: "none"` relay-prompt exception, so the PR addresses a real remaining gap after #66073; however the refreshed patch appears to broaden direct-session key recognition too far and also needs the required changelog entry before it is ready to merge.

## What This Changes

The PR adds a heartbeat prompt relay fallback for direct internal WebChat sessions when heartbeat delivery resolves to `none`, moves main/direct session-key helpers into `src/sessions/session-key-utils.ts`, and adds targeted heartbeat regression tests.

## Best Possible Solution

Keep this PR open as the active implementation candidate, but tighten direct-session key detection so it preserves the old malformed-key exclusions while still supporting real plugin/direct WebChat routes, add the required changelog entry, and then run the focused heartbeat/session tests plus the normal changed gate before merge.

## Reproduction Assessment

Yes. The current-main behavior is reproducible from source: `heartbeat.target: "none"` returns no delivery target and `runHeartbeatOnce` therefore builds internal-only exec/cron prompts; the PR tests encode the missing direct WebChat path, and the helper regression can be checked by evaluating the new helper on a key like `agent:main:subagent:worker:direct:user-1`.

## Solution Assessment

No, not as currently written. The direct WebChat prompt fallback is the right narrow product direction, but the shared helper should not broaden malformed or channel-agnostic session keys, and the user-facing fix needs a changelog entry before merge.
