## Summary

Keep this PR open: current main still lacks the direct WebChat target-none prompt fallback, but the latest diff has a blocking direct-session helper regression and lacks after-fix real behavior proof.

## What This Changes

The PR adds a direct WebChat heartbeat target-none prompt relay fallback, moves main/direct session-key helpers into shared utilities, and adds heartbeat regression tests.

## Best Possible Solution

Keep the direct WebChat fallback, but preserve the current helper’s deliverable-channel or reserved-scope exclusion, add a regression for reserved internal direct-like keys, and require real behavior proof before merge.

## Reproduction Assessment

Yes. Current-main behavior is source-reproducible because `target: "none"` returns no delivery target and the relay gate remains false; the PR regression is also source-reproducible from the broadened direct-session helper.

## Solution Assessment

No, not as currently written. The direct WebChat fallback is the right narrow direction, but the shared helper should not classify reserved internal scopes as direct sessions.

## Review Findings

Full review comments:

- **[P2] Reject non-channel direct-like session keys:** `src/sessions/session-key-utils.ts:90-95`
