## Summary

Keep open: this PR targets a real source-reproducible heartbeat/WebChat routing gap, but the latest branch still broadens direct-session classification in a way that can misroute internal lanes, and contributor real behavior proof is still missing.

## What This Changes

The branch moves main/direct session-key helpers into `src/sessions/session-key-utils.ts`, adds a target-none heartbeat relay fallback for direct internal WebChat routes, and adds session-delivery/heartbeat regression tests.

## Best Possible Solution

Keep the WebChat target-none fallback, but preserve a strict direct-key contract that explicitly supports real WebChat direct keys while rejecting reserved internal scopes, then require redacted WebChat runtime proof on a current-main merge result.

## Reproduction Assessment

Yes, at source level. Current main keeps `heartbeat.target: "none"` internal-only through the delivery-channel gate, and the PR regression is visible in the broadened `isDirectSessionKey` helper without needing a live channel run.

## Solution Assessment

No, not as written. The target-none WebChat fallback is the right direction, but the shared helper must keep reserved scopes out of direct routing and the PR needs real WebChat behavior proof before merge.

- **[P1] Reject reserved scopes in direct-session detection:** `src/sessions/session-key-utils.ts:90-94`

Keep the WebChat direct case, but reject reserved/internal prefixes or require a known channel-like prefix before accepting this shape.
