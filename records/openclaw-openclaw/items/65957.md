## Summary

Keep open: current main still lacks the direct WebChat target-none prompt exception, but this PR still broadens direct-session key detection for internal/non-channel scopes and is missing the required changelog entry.

## What This Changes

The PR adds a heartbeat prompt relay fallback for direct internal WebChat sessions when heartbeat delivery resolves to none, extracts main/direct session-key helpers into shared utilities, and adds targeted heartbeat regression tests.

## Best Possible Solution

Keep this PR as the active implementation candidate, but preserve the old malformed-key exclusions in the shared direct-session helper, add the active-version changelog entry, and then run focused heartbeat/session tests plus the normal changed gate.

## Reproduction Assessment

Yes. Current-main behavior is reproducible from source because `target: "none"` returns no delivery target and the relay gate remains false; the PR regression can be checked by evaluating the new helper on a channel-agnostic key such as `agent:main:subagent:worker:direct:user-1`.

## Solution Assessment

No, not as currently written. The direct WebChat prompt fallback is the right narrow direction, but the shared helper should not classify reserved internal scopes as direct sessions, and the user-facing fix needs a changelog entry before merge.
