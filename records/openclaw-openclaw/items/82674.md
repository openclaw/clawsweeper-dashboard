## Summary

Keep open: this is a real P1 gateway diagnostics fix with sufficient Testbox proof, but the PR is draft/protected and the new config toggle does not actually hot-reload for direct config-file edits on the running heartbeat.

## What This Changes

The PR adds critical memory-pressure stability bundle persistence, redacted memory/session/cgroup evidence, CLI rendering, a `diagnostics.memoryPressureBundle.enabled` toggle, docs/changelog updates, and regression coverage.

## Best Possible Solution

Fix the new toggle so accepted direct config edits update the running diagnostic heartbeat, then land the privacy-preserving bundle and keep the linked OOM issue focused on memory-growth mitigation.

## Reproduction Assessment

No high-confidence reproduction of the production OOM is available here; current main source shows the missing pre-OOM bundle path and the PR body gives deterministic Testbox proof for the critical-pressure path. The hot-reload defect in this PR is source-reproducible from the pinned runtime-config path.

## Solution Assessment

Partly: using the existing stability-bundle writer and redaction path is the right diagnostic shape. The config-toggle portion is not complete until direct file edits are observed by the already-running heartbeat.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Update the heartbeat config on direct reloads:** `src/logging/diagnostic.ts:1020-1021`
  - body: The new toggle is documented and tested as hot-reloaded for direct config edits, but the running heartbeat reads `heartbeatConfig` through `getRuntimeConfig`, which is backed by a pinned runtime snapshot. `startGatewayConfigReloader` accepts direct file edits by updating its internal `currentConfig` and calling `onHotReload`; it does not update that snapshot, so editing `diagnostics.memoryPressureBundle.enabled: false` after startup leaves the heartbeat using the old value and still writing critical bundles until a restart or in-process config write. Please update the runtime snapshot/heartbeat config on accepted hot reload, and cover the heartbeat path rather than only the reload-plan callback.
  - confidence: 0.84
