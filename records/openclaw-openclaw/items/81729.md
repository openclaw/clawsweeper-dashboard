## Summary

Keep this PR open for explicit human security review. The latest branch appears to preserve non-owner execution for many queued system-event runs, but it still removes the model-visible external-event trust marker and deletes the GHSA-backed static enforcement without a replacement boundary.

## What This Changes

The PR removes queued system-event `trusted` metadata handling, standardizes drained event text to `System:`, adjusts owner-downgrade tests and logic, deletes the GHSA OpenGrep rule, and updates the changelog.

## Best Possible Solution

Keep an explicit queued-event authority boundary end to end before simplifying metadata: preserve the current marker and rule, or land a reviewed replacement that separates external event text from trusted runtime context and keeps static enforcement aligned.

## Reproduction Assessment

Yes, for source inspection. Current main stores `trusted`, renders false values as `System (untrusted)`, downgrades owner runs from that marker, and the PR diff removes the marker and OpenGrep rule while adding separate non-owner heuristics.

## Solution Assessment

No. The latest branch addresses sender-owner downgrading, but removing the model-visible and static trust boundaries without an approved replacement is not the narrowest maintainable security fix.
