## Summary

Keep open: this MEMBER-authored PR has protected security/maintainer labels and still has a security-relevant compatibility issue because it drops legacy `trusted: false` from published channel producers while their external install floors still admit hosts older than the new `forceSenderIsOwnerFalse` contract.

## What This Changes

The branch removes `trusted` options from queued system-event producers, changes inbound system-marker sanitization, updates the GHSA OpenGrep rule, adds memory-host coverage, and updates the changelog.

## Best Possible Solution

Preserve the structured authority model, but keep legacy `trusted: false` on externally installable plugin producer calls until the official package and catalog compatibility floors require hosts that understand `forceSenderIsOwnerFalse`.

## Reproduction Assessment

Yes. Source inspection is enough: the PR head removes `trusted: false` from published channel producer calls while package metadata and the non-bundled plugin loader still allow older hosts that may not understand `forceSenderIsOwnerFalse`.

## Solution Assessment

No. Removing model-visible trust labels is a good direction, but dropping the legacy compatibility flag before raising external plugin compatibility floors is not the narrowest safe solution.
