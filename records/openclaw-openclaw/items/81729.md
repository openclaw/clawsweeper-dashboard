## Summary

Keep open: this protected maintainer-labeled PR needs explicit maintainer handling, and source review found a blocking trust-boundary regression in the proposed patch.

## What This Changes

The PR removes `trusted` from queued system events, formats all drained event context as `System:`, deletes untrusted-event owner downgrades, and updates affected channel, gateway, agent, memory, and changelog files.

## Best Possible Solution

Preserve a non-privileged authority signal for externally sourced queued events, or replace it with an explicitly reviewed equivalent, before simplifying the visible system-event prefix.

## Reproduction Assessment

Yes for the review finding from source: current main tests expect untrusted hook:wake and interval queued events to set `ForceSenderIsOwnerFalse`, and the PR changes those expectations. I did not run branch tests in this read-only review.

## Solution Assessment

No. Removing display metadata and authorization metadata together is not the narrowest maintainable solution; a safer path is to decouple the visible `System:` prefix decision from the non-owner execution guard.
