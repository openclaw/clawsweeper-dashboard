## Summary

Keep this PR open: it is MEMBER-authored and carries protected `security` and `maintainer` labels, and the diff removes the model-visible and static system-event trust boundaries without a replacement design.

## What This Changes

The branch removes `SystemEvent` trust metadata and `trusted` enqueue options across core/plugin call sites, renders drained queued events with a single `System:` prefix, adjusts owner-downgrade tests/logic, deletes the GHSA OpenGrep rule, and updates the changelog.

## Best Possible Solution

Keep or replace the explicit system-event authority boundary end to end before simplifying prompt wording: enqueue contract, drain formatting, owner downgrade behavior, regression tests, and static enforcement should stay aligned.

## Reproduction Assessment

Yes for source inspection. Current main stores trust metadata, renders `trusted: false` as `System (untrusted):`, downgrades owner runs from that marker, and enforces the GHSA rule; the branch removes the visible marker and scanner.

## Solution Assessment

No. Preserving non-owner execution for queued event runs is useful, but removing the AI-visible and static trust boundaries without an approved replacement is not the narrowest maintainable security fix.
