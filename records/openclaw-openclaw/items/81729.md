The PR removes queued system-event trust storage/options, changes drained queued event prompt lines to `Event:`, replaces owner downgrades with event-source heuristics, deletes the GHSA OpenGrep rule, and updates affected tests/docs/changelog.

## Best Possible Solution

## Reproduction Assessment

Yes, for source inspection. Current main stores `trusted`, renders false values as `System (untrusted)`, downgrades owner runs from that marker, and the PR head changes those prompt lines to `Event:` while deleting the scanner.

## Solution Assessment

No. The owner-downgrade heuristics help runtime authority, but they do not replace the model-visible untrusted prompt boundary or the static enforcement that kept new external enqueue paths explicit.
