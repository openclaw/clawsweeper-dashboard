## Summary

Keep open: this is a MEMBER-authored PR with protected `security` and `maintainer` labels, and the diff still changes a GHSA-backed prompt-trust boundary in a way that needs human security review before merge.

## What This Changes

The PR removes queued system-event trust metadata, renders drained queued events with a single `System:` prefix, replaces trust-bit owner downgrades with queued-event heuristics, deletes the related OpenGrep rule, and updates tests/changelog text.

## Best Possible Solution

Keep an explicit queued-event authority boundary end to end: preserve the current marker and scanner, or land a reviewed replacement that separates external event text from trusted runtime context and keeps static enforcement aligned.

## Reproduction Assessment

Yes, for source inspection. Current main stores `trusted`, renders false values as `System (untrusted)`, downgrades owner runs from that marker, and the PR diff removes the marker and scanner while adding separate non-owner heuristics.

## Solution Assessment

No. The non-owner heuristics help runtime authority, but they do not replace the model-visible prompt boundary or the static enforcement that kept new external enqueue paths explicit.
