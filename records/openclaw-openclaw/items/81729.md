## Summary

Keep open. This is a MEMBER-authored PR with protected `security` and `maintainer` labels, and current head still removes a GHSA-backed queued-event prompt and authority boundary without a reviewed replacement.

## What This Changes

The PR removes queued system-event trust storage, trusted/untrusted prompt formatting, owner-downgrade plumbing, related channel/plugin call-site flags, the OpenGrep trust rule, tests, and changelog text.

## Best Possible Solution

Preserve or explicitly replace the queued-event trust boundary with human security/product approval: external queued text should stay visibly unprivileged, owner authority should stay downgraded for external or exec/system-event turns, and static enforcement should match the approved model.

## Reproduction Assessment

Yes. Source inspection shows current main stores `trusted`, formats untrusted queued events as `System (untrusted):`, and downgrades owner authority, while PR head removes those paths.

## Solution Assessment

No. The current PR is broader than a narrow cleanup because it removes prompt, runtime authority, and static scanner boundaries without a reviewed replacement.
