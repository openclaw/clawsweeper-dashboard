## Summary

Keep open. This member-authored PR has protected `security` and `maintainer` labels, and source review shows it removes the existing queued-event prompt trust marker, owner-authority downgrade, and GHSA OpenGrep enforcement without an equivalent reviewed replacement.

## What This Changes

The PR removes system-event `trusted` storage and formatting, strips trust flags from enqueue call sites across channels/gateway/agents, changes inbound spoof-marker sanitization, and deletes the GHSA OpenGrep rule.

## Best Possible Solution

Keep or explicitly replace the queued-event trust boundary with a reviewed security model that preserves unprivileged prompt labeling, owner downgrades for external queued/exec events, and static enforcement.

## Reproduction Assessment

Yes. Source comparison of current main and PR head shows current main stores trust metadata, labels untrusted queued prompt text, downgrades owner authority, and scans for missing trust flags, while the PR removes those paths.

## Solution Assessment

No. This patch is broader than cleanup because it removes a prompt/runtime authority boundary and its scanner without an approved replacement contract.
