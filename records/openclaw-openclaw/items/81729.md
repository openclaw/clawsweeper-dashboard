## Summary

Keep open: this protected security/maintainer PR removes the current system-event prompt trust metadata, sender-owner downgrade paths, and GHSA OpenGrep guard without a reviewed replacement boundary.

## What This Changes

This PR removes queued system-event trust metadata, standardizes queued event formatting on `System:`, drops queued-event owner downgrades, and deletes the GHSA OpenGrep trust rule.

## Best Possible Solution

Address the cron owner-tool flicker with a narrower authority model or an explicitly reviewed replacement boundary that still separates external queued text from trusted runtime context and preserves static enforcement.

## Reproduction Assessment

Yes, from source inspection: current main stores trust metadata, formats untrusted queued text separately, and downgrades owner authority; PR head removes those paths. I did not run runtime tests because this was a read-only review.

## Solution Assessment

No; the current PR is broader than the related cron owner-tool bug requires and removes multiple security boundaries at once. A safer solution is a targeted cron/heartbeat authority fix or a reviewed replacement for the trust metadata, prompt marker, and scanner.
