## Summary

Keep open: this protected maintainer/security PR intentionally removes a GHSA-backed queued-event trust boundary, and the current diff still drops untrusted prompt labeling, owner-authority downgrades, heartbeat overrides, and static enforcement without a reviewed replacement.

## What This Changes

The PR removes queued system-event trust storage, prompt formatting, duplicate-suppression metadata, static enforcement, and many plugin/channel/gateway/agent call-site `trusted` options.

## Best Possible Solution

Keep the existing trust boundary, or replace it with a reviewed ingress/tool-policy/execution-context model that preserves non-owner routing and equivalent static enforcement before deleting the metadata.

## Reproduction Assessment

Yes. A source comparison is enough: current main stores `trusted: false`, renders `System (untrusted):`, and forces non-owner routing, while the PR diff removes those paths.

## Solution Assessment

No. The patch is not the best solution unless a reviewed replacement preserves the prompt boundary, owner downgrade, heartbeat override, and scanner coverage.
