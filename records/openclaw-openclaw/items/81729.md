## Summary

Keep open: this MEMBER-authored PR has protected `security` and `maintainer` labels and still removes a GHSA-backed prompt-trust boundary without replacement enforcement.

## What This Changes

The PR removes queued system-event trust metadata, removes related OpenGrep enforcement, and formats drained queued event context with a single `System:` prefix.

## Best Possible Solution

Keep an explicit machine-readable or AI-visible external-event authority signal through enqueue, drain, heartbeat, and reply-run setup, then simplify wording only after replacement enforcement and regression coverage exist.

## Reproduction Assessment

Yes for source inspection. Current main stores trust, formats `trusted: false` as `System (untrusted)`, downgrades owner runs from that marker, and enforces the GHSA rule; the PR diff removes the marker and rule while rendering drained events as `System:`.

## Solution Assessment

No. The latest head fixes the owner-status part by downgrading runs with queued events, but removing the visible trust marker and static enforcement is not the safest maintainable solution without a replacement boundary.
