Keep open. This MEMBER-authored PR has protected `security` and `maintainer` labels and changes a prompt-trust boundary; current main still keeps explicit system-event trust metadata, an AI-visible untrusted marker, owner downgrades, and GHSA OpenGrep enforcement while the PR removes the marker and scanner without a replacement boundary.

## What This Changes

The PR removes queued system-event `trusted` metadata, renders drained queued events as `System:`, deletes the GHSA OpenGrep trust rule, and updates core/channel/plugin tests and call sites for that simplified contract.

## Best Possible Solution

Keep or replace the explicit system-event authority boundary end to end: enqueue contract, drain formatting, owner downgrade behavior, regression tests, and static enforcement should remain aligned before simplifying the prompt wording.

## Reproduction Assessment

Yes for source inspection. Current main stores trust metadata, renders `trusted: false` as `System (untrusted):`, downgrades owner runs from that marker, and enforces the GHSA rule; the PR diff removes the visible marker and scanner.

## Solution Assessment

No. The latest head appears to preserve the non-owner run behavior for queued system-event runs, but removing the AI-visible and static trust boundaries without a replacement is not the safest maintainable solution.
