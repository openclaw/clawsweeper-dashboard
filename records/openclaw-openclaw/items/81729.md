## Summary

Keep open: this MEMBER-authored PR has protected `security` and `maintainer` labels, and the current diff still removes the queued-event trust marker plus its GHSA OpenGrep enforcement without a reviewed replacement boundary.

## What This Changes

The PR removes queued system-event `trusted` metadata and enqueue options, renders drained events as `System:`, replaces owner downgrading with queued-event heuristics, deletes the GHSA OpenGrep rule, and updates tests/changelog.

## Best Possible Solution

Keep an explicit queued-event authority boundary end to end: preserve the current marker and scanner, or land a reviewed replacement that separates external event text from trusted runtime context and keeps static enforcement aligned.

## Reproduction Assessment

Yes, for source inspection. Current main stores `trusted`, renders false values as `System (untrusted)`, downgrades owner runs from that marker, and the PR diff removes the marker and OpenGrep rule while adding separate non-owner heuristics.

## Solution Assessment

No. The non-owner heuristics help runtime authority, but they do not replace the model-visible prompt boundary or the static enforcement that kept new external enqueue paths explicit.
