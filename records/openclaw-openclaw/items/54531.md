## Label Justifications

- `P1`: The issue describes user-visible channel replies silently missing from messaging clients in active Telegram/Discord-style workflows.
- `impact:message-loss`: The central symptom is generated assistant output appearing internally but not being delivered to the originating chat surface.
- `impact:session-state`: The failure mode depends on session route metadata, deliveryContext, lastChannel/lastTo, and current surface ownership drifting across UI and channel contexts.
- `impact:security`: Changing this behavior decides when internal UI or runner output may be sent to external chat recipients, which is a privacy boundary.

## Summary

Keep open: current main and v2026.5.22 preserve several external delivery routes and support explicit delivery, but they still intentionally do not force browser/WebChat dashboard replies onto inherited external chat routes. The remaining request is a product and security decision about when internal UI or runner output may be sent to Telegram, Discord, WhatsApp, or similar channels.

## What This Changes

Requests a config/default that forces agent final replies to deliver to the originating messaging channel, with fallback behavior when the origin is unclear.

## Best Possible Solution

Define a maintainer-approved explicit origin-delivery policy that keeps safe defaults, adds a documented config/schema surface if desired, and covers WebChat/UI/main/channel-scoped behavior with focused regression tests.

## Reproduction Assessment

Yes at source level: current main has explicit guards showing WebChat/internal turns do not route to an external origin unless an explicit delivery route is set. I did not run tests because this review is read-only, but the existing tests assert the guarded behavior.

## Solution Assessment

Unclear: the requested force-origin behavior may be useful, but a blanket default is not clearly the safest solution. The maintainable path is an explicit, documented operator policy with security review rather than changing the current guard opportunistically.
