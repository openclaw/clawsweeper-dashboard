## Summary

Keep open. The underlying outbound metadata leak is still present on current main, but this PR is not merge-ready because it applies the broad inbound sanitizer to all assistant delivery text and weakens the shared timestamp-stripping contract used by other surfaces.

## What This Changes

The PR calls `stripInboundMetadata()` during outbound payload normalization, adds an outbound regression test, and changes shared sanitizer timestamp-only test expectations.

## Best Possible Solution

Sanitize only echoed leading inbound metadata at the outbound plan boundary before directive parsing, preserve the existing shared `stripInboundMetadata()` timestamp-only behavior, and cover the privacy fix plus false-positive cases in focused tests.

## Reproduction Assessment

Yes. A `ReplyPayload` whose text begins with `Conversation info (untrusted metadata):` still reaches `parseReplyDirectives(payload.text ?? "")` on current main and then flows into delivery through `createOutboundPayloadPlan()`.

## Solution Assessment

No, not as-is. The fix belongs in outbound normalization before directive parsing, but it should use an outbound-safe leading-prefix helper or new narrow helper instead of changing the shared sanitizer contract and stripping metadata-looking examples anywhere in assistant text.
