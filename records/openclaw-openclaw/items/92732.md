## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve stale-run transcript recovery:** `ui/src/ui/app-gateway.ts:1170-1186`
  - body: This makes session.message update metadata with `reconcileCurrentChatRun: false` and then, while a run is active, only records `pendingTranscriptSync` until a matching terminal chat event arrives. Current main also starts a bounded session refresh/replay path so a stale active run can clear and reload history even when the terminal event is missed; dropping that fallback can leave WebChat stuck in an active run with the transcript never reconciled.
  - confidence: 0.9
- **[P1] Remove the unused debug preview block:** `ui/src/ui/chat/build-chat-items.ts:710-720`
  - body: The new `previews` array is built but never read, and the same block uses explicit `any` in production UI code. `tsconfig.core.json` has `noUnusedLocals: true` and oxlint enforces both `no-unused-vars` and `typescript/no-explicit-any`, so this patch will keep prod type/lint checks failing until the debug block is removed or rewritten with typed, used data.
  - confidence: 0.95
- **[P2] Remove production DEDUPE warnings:** `src/gateway/server-chat.ts:679-687`
  - body: The added chat delta/final warnings run on normal gateway chat hot paths, with similar DEDUPE logging also added in terminal delivery and UI rendering. That leaves temporary diagnostic noise in production traffic and contradicts the PR's stated removal of DEDUPE debug logs; please drop these logs before merge.
  - confidence: 0.89
