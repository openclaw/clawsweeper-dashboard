## Summary

Keep open: the reported duplicate-message race is a useful bugfix target, but this patch is not merge-ready because it introduces blocking type/lint/test failures, leaves production debug logging in hot paths, changes the renderable-final reload invariant, and lacks inspectable real behavior proof.

## What This Changes

The PR changes WebChat gateway event reconciliation so `session.message` during an active run records a pending transcript sync for the terminal chat event, updates related UI/session tests, and adds temporary `[DEDUPE]` logging in gateway, agent, and UI paths.

## Best Possible Solution

Land a narrower UI fix that prevents `session.message` from racing `chat final` while preserving the no-reload-after-renderable-final and stale-run reconciliation invariants, removes debug-only code, keeps checks green, and includes visible WebChat proof.

## Reproduction Assessment

Source reproduction is plausible: current code and tests show the `session.message` then terminal chat-event ordering path, and the PR targets a known WebChat duplicate/flicker cluster. I did not have high-confidence real-environment proof because the PR provides only a textual manual result, not an inspectable artifact.

## Solution Assessment

No: the single-terminal-owner direction is plausible, but this patch reloads after renderable finals, removes an existing stale-run safety path, leaves debug code, and fails type/lint/test checks. A safer fix should keep the current final-message no-reload invariant and only defer the racing history write.

Full review comments:

- **[P1] Preserve the renderable-final no-reload path:** `ui/src/ui/app-gateway.ts:1100-1101`
  - body: When `session.message` arrives before a normal `chat final` carrying a visible assistant message, `shouldConsumePendingTranscriptSync` is true and this block now calls `loadChatHistory` after `handleChatEvent` has already appended the final message. Current main has an explicit regression test that this path must not reload history, because the final payload already reconciles the UI and an immediate transcript reload is the flicker/duplicate class this area has been avoiding.
  - confidence: 0.88
- **[P2] Remove the terminal debug dereference:** `src/agents/embedded-agent-subscribe.handlers.lifecycle.ts:259-260`
  - body: This `[DEDUPE]` log reads `ctx.state.deferredAssistantEvents.length` before terminal delivery. The PR CI agent lifecycle shard fails 34 tests with `Cannot read properties of undefined (reading 'length')` at this call path, so the temporary log is not safe to merge.
  - confidence: 0.97
- **[P2] Remove the browser debug preview block:** `ui/src/ui/chat/build-chat-items.ts:710-720`
  - body: The added browser debug branch leaves `previews` unused and introduces explicit `any` casts, which fail `check-prod-types` and `check-lint`. `buildChatItems` is a render hot path, so this temporary diagnostic work should be removed or moved to a typed, gated diagnostic path before merge.
  - confidence: 0.98
- **[P2] Drop the stale sessions-refresh import:** `ui/src/ui/app-gateway.ts:18`
  - body: After removing the active-run `loadSessions` refresh path, `scopedAgentListParamsForSession` is still imported and now fails both production typechecking and lint as unused. Remove the import or restore the caller if the refresh safety path is kept.
  - confidence: 0.99
- **[P2] Fix the added test callback types:** `ui/src/ui/app-gateway.sessions.node.test.ts:868`
  - body: The added test callbacks do not match the hoisted mock signatures: the `state` parameter here is unused, and the later `handleChatEventMock.mockImplementation` callbacks take an argument where the mock is typed as zero-argument. This is why `check-test-types` fails on the new tests.
  - confidence: 0.95
- **[P3] Remove unconditional gateway DEDUPE logs:** `src/gateway/server-chat.ts:679-688`
  - body: This logs every visible chat delta in the gateway hot path, and the same PR also logs finals. The PR body says all `[DEDUPE]` logs were removed; leaving these warnings in production will spam operator logs during normal streaming.
  - confidence: 0.91
