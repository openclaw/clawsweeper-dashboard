## Summary

Keep open. Current main still drops Slack file objects that have an id but no private URL, and this PR is the right narrow direction, but it still needs a rebase-aware retry after stale event URL download failures plus the required changelog entry before merge.

## What This Changes

The branch threads an optional Slack WebClient through inbound Slack media resolution and adds a `files.info` fallback plus tests for file objects with an id but no private URL.

## Best Possible Solution

Land a rebased Slack plugin fix that refreshes metadata for missing URLs, retries once with fresh metadata after stale event URL download failures, preserves existing Slack URL/auth guards and timeout parameters, and records the user-facing fix in the changelog.
