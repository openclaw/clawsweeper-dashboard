Keep open. Current main still cannot refresh or download inbound Slack file objects that only carry a file id, and this PR is the narrow canonical fix path, but the branch still needs a complete fresh-metadata retry and the required changelog entry before merge.

## What This Changes

The branch threads an optional Slack WebClient through inbound Slack media preparation and adds a `files.info` fallback plus tests for Slack file objects that have an id but no private download URL.

## Best Possible Solution

Land a rebased Slack plugin fix that passes the WebClient through direct, forwarded attachment, and thread-starter media resolution, refreshes metadata when URLs are missing, retries once with fresh metadata after stale event URL download failure, preserves the Slack URL allowlist/auth fetch limits, and records the fix in the changelog.
