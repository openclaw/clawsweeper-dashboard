## Summary

Keep open for maintainer review, but the patch is not ready as proposed: it adds a new hook credential config surface despite related maintainer guidance toward a future mutable SecretRef path, and its file-backed resolver bypasses OpenClaw's shared secret-file safety checks.

## What This Changes

This PR adds `hooks.tokenFile` to hook config/schema/docs/runtime paths, shares token resolution with gateway/Gmail hook code, and adds regression coverage for tokenFile-only hook configs.

## Best Possible Solution

Route hook ingress secrets through the planned mutable SecretRef credential path, or explicitly accept `hooks.tokenFile` as an interim surface and implement it with the shared secret-file/runtime helpers plus aligned schema, audit, docs, and tests.

## Reproduction Assessment

Not applicable as a bug repro. Source inspection confirms current main only supports inline `hooks.token`, and the related discussion shows this is a requested new secret-backed config capability.

## Solution Assessment

No, not as-is. The direction needs maintainer confirmation against the planned mutable SecretRef work, and any accepted file-backed token path should use the existing secret-file safety helper instead of a raw file read.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Use the shared secret-file reader for tokenFile:** `src/hooks/token.ts:19`
  - body: This new credential resolver reads the configured path with raw `fs.readFileSync`, while existing tokenFile paths use `tryReadSecretFileSync` to enforce empty-path, max-size, regular-file, and symlink policy. A misconfigured hook token file can now follow symlinks or read an unbounded file, and the docs call it a regular secret file, so route this through the shared secret-file helper before accepting the new surface.
  - confidence: 0.91

## Security Review

Status: needs_attention

Summary: The PR introduces a new secret-file read path that bypasses OpenClaw's shared secret-file hardening.
