## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P2] Defer prose only after confirming tables:** `extensions/qqbot/src/engine/messaging/markdown-table-chunking.ts:207-210`
  - body: This now moves any preceding prose into `pendingParagraphBlock` as soon as a pipe-looking row appears, before a separator confirms that it is a table. For input like `Intro\n| maybe | header |\nplain continuation`, the later `flushText` emits the pipe row plus continuation first and the pending intro is flushed only at the end, so the delivered chunks are reordered. Keep the prose in normal text state until the table is actually confirmed, or restore it before flushing the header as plain text.
  - confidence: 0.94
- **[P2] Preserve empty fenced blocks:** `extensions/qqbot/src/engine/messaging/markdown-table-chunking.ts:674-675`
  - body: For an empty fence body, this branch enters the `bodyLines.length === 0` case and calls `flushCurrent()`, but `flushCurrent` immediately returns when `currentLines` is empty. Current main emits `render([])` for this case, so the PR drops a valid empty fenced block, and with a pending paragraph it can drop the fence while sending only the prose. Emit `render([])` or teach `flushCurrent` to handle the empty-fence case.
  - confidence: 0.96
