## Review Findings

Overall confidence: 0.93

Full review comments:

- **[P1] Keep fenced MEDIA examples deliverable:** `extensions/qqbot/src/engine/gateway/outbound-dispatch.ts:104`
  - body: This regex treats any line-start `MEDIA:` as a directive, even inside fenced code. Core media parsing deliberately preserves fenced MEDIA-looking lines as text, but the QQ Bot block callback now returns early, so a code example can be marked streamed and then omitted from the final reply path.
  - confidence: 0.93
- **[P2] Defer prose only after confirming tables:** `extensions/qqbot/src/engine/messaging/markdown-table-chunking.ts:207-210`
  - body: This drains accumulated prose as soon as a pipe-looking row appears, before a separator confirms a table. For `Intro\n| maybe | header |\nplain continuation`, the later pipe/plain text can be emitted before the pending intro, reordering the QQ Bot reply.
  - confidence: 0.94
- **[P2] Preserve empty fenced blocks:** `extensions/qqbot/src/engine/messaging/markdown-table-chunking.ts:674-675`
  - body: When `bodyLines` is empty, this branch calls `flushCurrent()`, but `flushCurrent` returns immediately because `currentLines` is empty. Current main emits an empty fenced block as a self-contained chunk, so this drops valid markdown content.
  - confidence: 0.96
