## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Keep MEDIA lines inside code fences deliverable:** `extensions/qqbot/src/engine/gateway/outbound-dispatch.ts:104`
  - body: This regex treats any line-start `MEDIA:` as a directive, but core parsing deliberately ignores `MEDIA:` inside fenced code spans. In partial mode, a block containing a fenced example like ```\nMEDIA:/tmp/example.png\n``` now returns without sending; the block pipeline treats the callback as successfully sent and can suppress the matching final text, so the QQ user loses the code block.
  - confidence: 0.93
- **[P2] Defer prose only after confirming tables:** `extensions/qqbot/src/engine/messaging/markdown-table-chunking.ts:207-210`
  - body: This drains accumulated prose into `pendingParagraphBlock` as soon as a pipe-looking row appears, before a separator confirms a table. For input like `Intro\n| maybe | header |\nplain continuation`, the later pipe/plain text is emitted before the pending intro, reordering QQ Bot output; keep prose in normal text state until table confirmation or restore it before flushing the header as text.
  - confidence: 0.94
- **[P2] Preserve empty fenced blocks:** `extensions/qqbot/src/engine/messaging/markdown-table-chunking.ts:674-675`
  - body: When `bodyLines` is empty, this branch calls `flushCurrent()`, but `flushCurrent` immediately returns because `currentLines` is empty. Current main emits `render([])`, so this drops valid empty fenced blocks; emit the empty rendered fence explicitly or make `flushCurrent` handle that case.
  - confidence: 0.96
