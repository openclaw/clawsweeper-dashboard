The issue reports that Anthropic OAuth bearer requests use a stale Claude CLI user-agent version, causing valid Claude CLI OAuth tokens to be rejected with `authentication_error`.

## Best Possible Solution

## Solution Assessment

Yes on the issue direction: the stale hardcoded user-agent should be replaced across both OAuth builders. The exact implementation should be resolved in one of the linked PR reviews rather than by closing this issue early.
