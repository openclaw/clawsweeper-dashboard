## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Verify tunnel ownership before signaling stored PIDs:** `src/cli/tunnel-cli.ts:291`
  - body: `tunnel down` sends SIGTERM/SIGKILL to the stored numeric PID after only checking that the PID exists. If the original SSH tunnel exits and the OS reuses that PID, this can terminate an unrelated user process, and `status` will also treat the reused PID as a live tunnel.
  - confidence: 0.94
- **[P1] Reject occupied ports before reporting tunnel readiness:** `src/cli/tunnel-cli.ts:222-224`
  - body: The startup loop treats any listener on `--port` as proof that the new SSH tunnel is ready. If another process already owns the port, SSH can still be briefly alive before `ExitOnForwardFailure` exits, so the command can write stale state and report success for the wrong listener.
  - confidence: 0.92
- **[P2] Create the state directory before writing tunnel metadata:** `src/cli/tunnel-cli.ts:38-39`
  - body: `writeTunnelState` writes under `STATE_DIR` without creating it first. On a fresh profile or custom `OPENCLAW_STATE_DIR`, `tunnel up` can start the detached SSH process and then throw `ENOENT`, leaving users without a PID file to stop or inspect the tunnel.
  - confidence: 0.91
- **[P2] Rebase registration onto the descriptor catalog:** `src/cli/program/register.subclis.ts:168-176`
  - body: Current main no longer registers subcommands with the old `entries` array this PR edits; subcommands are described in the descriptor catalog and registered from command-group specs. Rebase the command onto that surface or it will remain conflicting and miss the current help/registration path.
  - confidence: 0.88
- **[P2] Avoid adding another hard-coded SSH executable path:** `src/cli/tunnel-cli.ts:200-203`
  - body: The new command spawns `/usr/bin/ssh` directly, which fails on hosts where the trusted SSH client lives elsewhere and extends the open Unix-only SSH helper issue. Use or extract a shared SSH executable policy, or explicitly scope the command before exposing it as a general CLI feature.
  - confidence: 0.82
