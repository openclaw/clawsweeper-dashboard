## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Verify tunnel ownership before signaling stored PIDs:** `src/cli/tunnel-cli.ts:291`
  - body: `tunnel down` sends SIGTERM/SIGKILL to whatever process currently has the stored PID, but the state file does not store or verify process identity. After the original SSH process exits and the OS reuses that PID, this command can terminate an unrelated user process instead of cleaning stale tunnel state.
  - confidence: 0.93
- **[P1] Reject occupied ports before reporting tunnel readiness:** `src/cli/tunnel-cli.ts:222-224`
  - body: The readiness loop treats any listener on `--port` as proof that the spawned SSH tunnel is ready. If the port is already occupied, SSH can still be alive briefly before `ExitOnForwardFailure` exits, so the command can write stale PID state and report a tunnel that never started.
  - confidence: 0.9
- **[P2] Create the state directory before writing tunnel metadata:** `src/cli/tunnel-cli.ts:38-39`
  - body: `writeTunnelState` writes to `STATE_DIR/tunnel.pid.json` without ensuring the directory exists. On a fresh profile or new `OPENCLAW_STATE_DIR`, `tunnel up` can start detached SSH and then throw `ENOENT`, leaving users with an unmanaged tunnel.
  - confidence: 0.91
- **[P2] Avoid adding another hard-coded SSH executable path:** `src/cli/tunnel-cli.ts:199`
  - body: The new command spawns `/usr/bin/ssh` directly, so Windows OpenSSH and any platform-specific trusted SSH resolver work remain bypassed for this new surface. Reuse the shared SSH executable policy or extract one before exposing another user-facing SSH command.
  - confidence: 0.82
- **[P2] Register the command in the current descriptor catalog:** `src/cli/program/register.subclis.ts:168-176`
  - body: The branch adds `tunnel` to the old `register.subclis.ts` entry shape, but current main uses `subcli-descriptors.ts` plus `register.subclis-core.ts`. After rebasing, this command must be registered in both current descriptor and loader paths or root help/completion/lazy registration will drift.
  - confidence: 0.86
