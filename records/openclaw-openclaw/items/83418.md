## Summary

Keep open. The PR fixes the private-maintainer proof-gate path with a direct team-membership lookup, but it introduces a GitHub App token into a `pull_request_target` workflow without constraining the token permissions, so one narrow security fix is needed before merge.

## What This Changes

The PR adds GitHub App token minting to the real-behavior-proof workflow, checks the PR author’s `openclaw/maintainer` team membership before enforcing proof, and adds tests for the membership helper.

## Best Possible Solution

Add least-privilege `permission-members: read` inputs to both GitHub App token steps, keep the trusted base-checkout path, and merge after the targeted policy tests and CI confirm maintainer-skip and fallback behavior.

## Reproduction Assessment

Yes. A direct current-main Node probe of `evaluateRealBehaviorProof` with a `CONTRIBUTOR` PR author and no proof section returns `missing`, and the source path shows no private-team-membership check before the gate.

## Solution Assessment

Mostly yes. The direct membership lookup is the right narrow fix and avoids labeler ordering races, but the workflow should constrain the minted App token before merge.
