## Summary

Keep open. Current main and the latest release still lack the requested model-supplied exec `description` or `intent` metadata, and the remaining work is a security-adjacent exec/tool contract decision rather than cleanup.

## What This Changes

The issue requests an optional exec description or intent field so hooks and approval flows can see human-readable command-purpose metadata.

## Best Possible Solution

Add an additive optional exec intent metadata field only as advisory data, carry it through exec schema/runtime, hook-visible params, approval payloads, docs, and tests, and document that enforcement remains command-bound.

## Reproduction Assessment

Yes. Source inspection on current main shows the exec contract, runtime args, hook-visible params, approval schema, and public docs still do not contain a model-supplied `description` or `intent` field.

## Solution Assessment

Unclear until maintainers settle the field name and trust boundary. The requested direction is maintainable if the field is advisory metadata and cannot authorize execution by itself.
