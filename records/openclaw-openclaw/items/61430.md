## Summary

Keep open. Current main still lacks the requested optional exec `description`/`intent` call metadata, and the remaining work is a security-adjacent product/API decision rather than a cleanup close.

## What This Changes

Request an optional natural-language exec intent field so hooks and approval flows can inspect command purpose in addition to command text.

## Best Possible Solution

Add an additive optional exec intent metadata field only as advisory data, carry it through exec schema/runtime, hook-visible params, approval payloads, docs, and tests, and document that enforcement remains command-bound.

## Reproduction Assessment

Yes. Source inspection on current main shows the exec contract, runtime args, hook-visible params, approval schema, and docs still do not contain a `description` or `intent` field.

## Solution Assessment

Unclear until maintainers settle the field name and trust boundary. The requested direction is maintainable if the field is advisory metadata and cannot authorize execution by itself.
