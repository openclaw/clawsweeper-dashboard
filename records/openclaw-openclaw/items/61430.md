## Summary

Keep open. Current main still lacks the requested exec `description`/`intent` field across the tool schema, runtime arg shape, docs, hook-visible params, and approval payloads; the duplicate at https://github.com/openclaw/openclaw/issues/61562 is already closed in favor of this canonical issue.

## What This Changes

The issue requests an optional exec `description` or `intent` parameter so hooks and approval/audit surfaces can see model-supplied command intent.

## Best Possible Solution

Add a backward-compatible optional exec intent metadata field through the exec schema/runtime, hook-visible params, approval payload/docs, and tests, while keeping enforcement command-bound.

## Reproduction Assessment

Yes. Source and docs inspection on current main shows the feature gap: exec accepts no `description` or `intent` field, and hooks only receive params admitted by the tool contract.

## Solution Assessment

Unclear until maintainers choose the field name and trust model. The requested direction is maintainable only if the new field is informational and cannot authorize execution by itself.
