## Summary

Keep open. Current main still lacks an exec call-level `description` or `intent` parameter, and this is not ClawHub-closeable because plugins can only inspect params that the core exec tool exposes; the closed duplicate #61562 leaves this as the canonical tracker.

## What This Changes

Add an optional human-readable `description` or `intent` field to exec tool calls so hooks can inspect command intent without parsing the command string.

## Best Possible Solution

Expose a backward-compatible optional string field in the exec schema/runtime and docs, while explicitly documenting that approval decisions remain bound to the command and existing approval policy.

## Reproduction Assessment

Yes. Source and docs inspection on current main reproduces the feature gap: the exec schema, runtime arg type, and public parameter docs have no `description` or `intent` field, while hooks only see forwarded params.

## Solution Assessment

Unclear until maintainers choose the field name and trust model. The narrow implementation is feasible, but the maintainable solution must keep the field informational and avoid using natural-language text as an approval bypass.
