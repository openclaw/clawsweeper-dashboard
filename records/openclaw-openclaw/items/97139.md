## Risks / Open Questions

- The PR changes OpenAI Responses SDK requests from SDK-default fetch to OpenClaw's guarded transport, so custom or local OpenAI-compatible endpoints that previously relied on SDK fetch behavior may now hit SSRF, private-network, proxy, timeout, or retry policy differences.

- Non-OK error bodies remain outside this PR's shared sanitizer path, so this should land as scoped streaming/SSE hardening or wait for a broader cross-provider error-body policy.
