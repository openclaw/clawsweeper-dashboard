## Summary

Keep this PR open: the bug and narrow direction are well supported, and the updated PR body now supplies useful after-fix Linux ACL proof, but the current patch still has a filesystem boundary blocker and a red generated-shrinkwrap guard.

## What This Changes

The PR reroots memory-core dreaming private JSON reads and writes under `memory/.dreams`, adds a helper and regression tests, and refreshes several shrinkwrap files.

## Best Possible Solution

Land a narrow memory-core fix that keeps only `memory/.dreams` private, ties workspace parent containment to the same fs-safe/pinned write boundary, adds race-oriented regression coverage, regenerates shrinkwraps cleanly, and leaves the broader permission-policy decision with https://github.com/openclaw/openclaw/issues/56263.

## Reproduction Assessment

Yes. Current main source uses `privateFileStore(workspaceDir)` for `memory/.dreams` JSON state, fs-safe private writes chmod the supplied root and private parent directories, and the linked discussion plus PR body provide a Linux POSIX ACL reproduction.

## Solution Assessment

No, not as-is. Rerooting the private store is the right narrow shape for the chmod blast radius, but parent containment must be bound to the private write and the shrinkwrap guard must be green before merge.
