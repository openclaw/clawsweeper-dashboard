## Summary

Keep open. The PR targets a real memory-core ACL bug and the linked discussion gives useful causality evidence, but the patch appears to weaken fs-safe's workspace symlink boundary by making `memory/.dreams` the private store root without first proving `memory` is a real workspace child; after-fix real behavior proof is also test-only.

## What This Changes

The PR moves memory-core dreaming JSON state reads/writes from `privateFileStore(workspaceDir)` to a new helper rooted at `memory/.dreams`, with focused regression coverage for preserved parent directory modes.

## Best Possible Solution

Land a narrow memory-core fix that keeps only `memory/.dreams` private, preserves workspace symlink/realpath containment, adds focused regression coverage for that boundary, and leaves the broader permission-policy decision with https://github.com/openclaw/openclaw/issues/56263.

## Reproduction Assessment

Yes. Current main source uses `privateFileStore(workspaceDir)` for `memory/.dreams` JSON files, fs-safe private writes chmod the supplied root and parents, and the linked discussion includes a live Linux POSIX ACL reproduction.

## Solution Assessment

No, not as-is. Rerooting the private store is the right narrow shape for the chmod blast radius, but it must preserve the workspace symlink/realpath boundary before merge.
