## Summary

Keep open. The narrow memory-core fix is useful, but it is not merge-ready because the new store root drops the previous workspace-root symlink-parent guard, and the after-fix proof is only targeted tests rather than a real ACL/shared-workspace run.

## What This Changes

The PR adds a memory-core helper that reads and writes dreaming private JSON under `memory/.dreams`, rewires daily/session/short-term dreaming state to use it, and adds mode/path regression tests.

## Best Possible Solution

Keep the `memory/.dreams`-scoped private artifact root, but first reject symlink parents from `workspaceDir` through `memory/.dreams`, add a symlink regression, and include redacted after-fix ACL proof.

## Reproduction Assessment

Yes, at source level. Current main calls `privateFileStore(workspaceDir)` for `memory/.dreams` JSON state, fs-safe private writes chmod the supplied root and target parents, and the linked issue comment provides a live Linux ACL reproduction; I did not run a live ACL repro in this read-only review.

## Solution Assessment

No, not as written. Re-rooting private dreaming state under `memory/.dreams` is the right narrow direction, but it must preserve the workspace-root symlink-parent rejection before it is safe to merge.
