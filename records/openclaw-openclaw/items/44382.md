## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Remove the root openclaw override:** `package.json:447-448`
  - body: Adding a root `openclaw` override for this TUI feature rewrites workspace consumers in the lockfile from local links to a published package version. That can make packages test against stale registry code instead of the checkout and mixes supply-chain remediation into unrelated UI work.
  - confidence: 0.92
- **[P2] Preserve 256-color COLORFGBG detection:** `src/tui/theme/detect.ts:30-31`
  - body: Current main handles 0-255 `COLORFGBG` backgrounds, but this detector only treats 7 and 15 as light. Light terminals reporting 231 or 255 will fall back to the dark palette, regressing the shipped readability fix.
  - confidence: 0.9
- **[P2] Keep mounted theme objects live:** `src/tui/theme/theme.ts:223-235`
  - body: `applyPalette()` replaces exported theme objects, while mounted Markdown and selector components keep the old objects they were constructed with. A runtime `/theme` switch can therefore render mixed old/new colors after `requestRender()`.
  - confidence: 0.86
- **[P2] Rebuild static chrome after switching themes:** `src/tui/tui-command-handlers.ts:470-473`
  - body: The `/theme` handler sets the active theme and requests a render, but header, footer, and idle status text contain ANSI-colored strings built earlier. Re-rendering alone does not recolor those strings, so the visible chrome can remain on the old palette.
  - confidence: 0.82
