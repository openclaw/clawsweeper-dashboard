## What This Changes

The PR adds five named TUI palettes with CLI/env/runtime selection and docs/tests, plus Swift protocol model regeneration and pnpm package-resolution changes.

## Best Possible Solution

Land a narrow, rebased TUI-only theme change if maintainers want this feature: preserve existing light/dark and 256-color detection, keep live theme objects stable for mounted components, repaint runtime chrome consistently, and split protocol regeneration plus dependency/security work into owner-reviewed PRs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Remove the global openclaw override:** `package.json:447`
  - body: Adding a repo-wide `pnpm.overrides.openclaw` makes workspace consumers resolve a published npm package instead of local source, as reflected by the lockfile changes. That is unrelated to TUI theming and weakens local validation in a supply-chain-sensitive way.
  - confidence: 0.9
- **[P2] Preserve 256-color light background detection:** `src/tui/theme/detect.ts:28-31`
  - body: `detectThemeName()` only returns `light` for `bg === 7` or `bg === 15`, so 256-color light backgrounds like `COLORFGBG=0;231` or `0;255` now fall back to the dark palette even though current main handles them as light.
  - confidence: 0.91
- **[P2] Normalize explicit theme overrides:** `src/tui/theme/detect.ts:10-13`
  - body: The env value is returned verbatim, but palette lookup is exact. Mixed-case values such as `OPENCLAW_THEME=LiGhT` or `Dracula` therefore fall back to dark even though current main treats `OPENCLAW_THEME` case-insensitively for light/dark.
  - confidence: 0.86
- **[P2] Recompute chrome after runtime theme changes:** `src/tui/tui-command-handlers.ts:480-487`
  - body: The `/theme` handler changes the active theme and requests a render, but header, footer, and status text are already baked into `Text` nodes with old ANSI colors. They can remain in the previous palette until another state update rebuilds them.
  - confidence: 0.82
- **[P2] Keep mounted theme objects stable:** `src/tui/theme/theme.ts:223-235`
  - body: `applyPalette()` replaces exported theme objects such as markdown and select-list themes. Components constructed before `/theme` store those object references, so already-mounted messages and selectors can remain in the old palette after a runtime switch.
  - confidence: 0.82
