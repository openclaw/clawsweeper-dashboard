## Review Findings

Overall confidence: 0.9

Full review comments:

- **[P1] Remove the root openclaw override:** `package.json:447`
  - body: This TUI theme branch adds a root `openclaw` override and lockfile rewrites that convert local workspace links to registry `openclaw@2026.3.11` for workspace packages. That changes package resolution and supply-chain inputs outside the UI feature; split it into separately reviewed dependency work or drop it from this PR.
  - confidence: 0.94
- **[P2] Update the cached editor border on theme switches:** `src/tui/tui.ts:532-540`
  - body: The `liveEditorTheme` proxy does not update pi-tui's cached `this.borderColor`, which is copied from `theme.borderColor` in the Editor constructor and used during render. After `/theme`, the input border can stay on the old palette unless the editor cache is updated or the editor is rebuilt.
  - confidence: 0.91
- **[P2] Keep mounted theme objects stable across switches:** `src/tui/theme/theme.ts:223-235`
  - body: `applyPalette()` replaces exported markdown/list/editor theme objects, while existing Markdown, SearchableSelectList, and SelectList instances store the object passed at construction. Runtime switching can leave mounted messages and overlays with stale colors; use stable delegating objects or retheme/rebuild those components.
  - confidence: 0.9
- **[P2] Preserve 256-color COLORFGBG detection:** `src/tui/theme/detect.ts:28-31`
  - body: The new detector only returns `light` for background indices 7 and 15, but current main handles 0-255 `COLORFGBG` values with luminance tests. Light 256-color terminals such as background 231 or 255 would fall back to dark and regress the shipped light-background path.
  - confidence: 0.88
- **[P2] Refresh static chrome after applying a theme:** `src/tui/tui-command-handlers.ts:469-473`
  - body: The `/theme` handler calls `setTheme()` and requests a render, but header/footer/status `Text` nodes already contain ANSI-formatted strings. Recompute those strings on direct apply, picker select, and cancel so users do not see mixed old and new colors.
  - confidence: 0.84
- **[P3] Normalize explicit theme names before lookup:** `src/tui/theme/detect.ts:11-13`
  - body: `OPENCLAW_THEME` and `/theme` arguments are looked up exactly, so mixed-case values like `LiGhT` or `Dracula` fall back to dark even though current main treats `OPENCLAW_THEME=LiGhT` as light. Normalize user-provided names before resolving palettes.
  - confidence: 0.78
