  - body: This theme PR adds a root `openclaw` override, and the lockfile then resolves internal packages such as `packages/clawdbot` and `packages/moltbot` to the published `openclaw@2026.3.11` package instead of the workspace link.   - body: `setTheme()` replaces `markdownTheme`, list themes, and other exported theme objects, but existing `MarkdownMessageComponent`/`HyperlinkMarkdown` and select-list instances are constructed with the old object reference. After `/theme`, already-mounted chat content and overlays can remain on the old palette, producing a mixed and potentially low-contrast UI; use stable delegating objects or explicitly rebuild/retheme affected components.
  - confidence: 0.9
- **[P2] Preserve 256-color COLORFGBG light detection:** `src/tui/theme/detect.ts:28-31`
  - body: The new detector only returns `light` for background indices 7 and 15, so light 256-color terminals such as `COLORFGBG=0;231` or `0;255` fall back to the dark theme. Current main already handles 0-255 COLORFGBG values with luminance tests, so port that logic instead of replacing it with the narrower 0-15 heuristic.
  - confidence: 0.88
- **[P3] Normalize explicit theme names before lookup:** `src/tui/theme/detect.ts:11-13`
  - body: `OPENCLAW_THEME` is returned verbatim and `setTheme()` performs an exact palette lookup, so values like `LiGhT` or `Dracula` fall back to dark even though current main treats OPENCLAW_THEME case-insensitively. Normalize explicit theme names once before resolving them.
  - confidence: 0.78
