Full review comments:

- **[P1] Remove the root openclaw override:** `package.json:447`
  - body: Adding an `openclaw` override for a TUI theme PR rewrites workspace consumers like `packages/clawdbot` and `packages/moltbot` from local `workspace:*` links to a published package in the lockfile. That can make local packages test against a published OpenClaw build instead of the checked-out workspace and mixes a GHSA/supply-chain fix into unrelated UI work.
  - confidence: 0.94
- **[P2] Preserve 256-color COLORFGBG detection:** `src/tui/theme/detect.ts:29-30`
  - body: Current main handles 0-255 `COLORFGBG` backgrounds and tests values like 231 and 255 as light. This new detector only considers 7 and 15 light, so light terminals that report xterm 256-color backgrounds get the dark palette by default and lose the shipped readability fix.
  - confidence: 0.92
- **[P2] Retheme mounted components on runtime switch:** `src/tui/tui-command-handlers.ts:468-470`
  - body: `setTheme()` replaces exported theme objects and this handler only requests a render. Mounted chat content, selectors, header/status/footer text, and other already-formatted strings can keep old theme objects or ANSI colors, so `/theme` can leave a mixed old/new UI unless the switch path updates or remounts those surfaces.
  - confidence: 0.88
- **[P2] Normalize explicit theme names before lookup:** `src/tui/theme/detect.ts:11-13`
  - body: Current main treats `OPENCLAW_THEME=LiGhT` case-insensitively, but this detector returns the raw trimmed value and the palette lookup is exact. Existing mixed-case env values would fall back to dark and can reintroduce unreadable light-terminal startup behavior.
  - confidence: 0.83
