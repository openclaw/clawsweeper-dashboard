  - body: Adding `pnpm.overrides.openclaw` makes workspace consumers resolve a published `openclaw` package instead of the local workspace links shown on main. That is unrelated to TUI theming and changes package resolution in a supply-chain-sensitive way.

  - confidence: 0.91
- **[P2] Normalize explicit theme overrides:** `src/tui/theme/detect.ts:10-13`
  - body: The env value is returned verbatim but palette lookup is exact, so `OPENCLAW_THEME=LiGhT` or `Dracula` falls back to dark. Current main treats `OPENCLAW_THEME` case-insensitively for the light/dark contract.
  - confidence: 0.86
- **[P2] Recompute chrome after runtime theme changes:** `src/tui/tui-command-handlers.ts:480-487`
  - body: The `/theme` handler changes the active theme and requests a render, but header, footer, and idle status text are already stored as ANSI-colored strings. They can remain in the previous palette until another state update rebuilds them.
  - confidence: 0.82
- **[P2] Keep mounted theme objects stable:** `src/tui/theme/theme.ts:223-235`
  - body: `applyPalette()` replaces exported theme objects such as markdown and select-list themes. Components constructed before `/theme` store those object references, so already-mounted messages and selectors can stay in the old palette after a runtime switch.
  - confidence: 0.82
- **[P2] Keep legacy palette exports in sync:** `src/tui/theme/theme.ts:77-83`
  - body: `setTheme()` changes the active named theme, but legacy exports such as `lightMode` and `palette` remain derived from startup background detection. Consumers can observe contradictory theme state after startup overrides or runtime switching.
  - confidence: 0.78
