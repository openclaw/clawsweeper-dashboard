Overall confidence: 0.9

Full review comments:

- **[P1] Remove the root openclaw override from this PR:** `package.json:447`
  - body: This TUI theme branch adds a root `openclaw` override and the lockfile then resolves workspace package importers to a published package instead of local workspace links. That changes supply-chain inputs and validation scope in an unrelated feature PR; keep GHSA/dependency work separate.
  - confidence: 0.93
- **[P2] Do not rely on the editor theme proxy for border updates:** `src/tui/tui.ts:532-540`
  - body: `Editor` copies `theme.borderColor` into `this.borderColor` during construction, so the getter proxy still leaves the input border on the old color after `/theme` switches. Use an explicit editor retheme path or a stable function that delegates at call time.
  - confidence: 0.9
- **[P2] Keep exported theme objects stable after switching:** `src/tui/theme/theme.ts:223-235`
  - body: `applyPalette()` replaces `markdownTheme`, list themes, and `editorTheme`, but mounted Markdown/select-list components keep the object they received at construction. After `/theme`, existing chat content and overlays can render with stale colors; mutate stable delegating objects or rebuild/retheme those components.
  - confidence: 0.9
- **[P2] Preserve 256-color COLORFGBG light detection:** `src/tui/theme/detect.ts:28-31`
  - body: The new detector only returns `light` for background indices 7 and 15, so 256-color light terminals such as `COLORFGBG=0;231` or `0;255` fall back to dark. Current main handles the 0-255 range with luminance logic; port that instead of narrowing the heuristic.
  - confidence: 0.88
- **[P2] Refresh cached TUI chrome after theme changes:** `src/tui/tui-command-handlers.ts:470-473`
  - body: The `/theme` handler calls `setTheme()` and requests a render, but header/footer/status strings have already been formatted with ANSI colors. Refresh those cached strings when applying, selecting, or canceling themes so the shell does not stay partially themed.
  - confidence: 0.84
- **[P3] Normalize explicit theme names before lookup:** `src/tui/theme/detect.ts:11-13`
  - body: `OPENCLAW_THEME` is returned verbatim and `setTheme()` does an exact palette lookup, so mixed-case values like `LiGhT` or `Dracula` fall back to dark even though current main treats the env override case-insensitively.
  - confidence: 0.78
