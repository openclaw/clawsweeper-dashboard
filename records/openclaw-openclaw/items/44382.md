- **[P1] Remove the root openclaw override:** `package.json:447`
  - body: This TUI feature adds a root `openclaw` override and the lockfile then resolves some workspace packages to a published `openclaw` package instead of the repo checkout. That is unrelated to themes and can make local builds/tests exercise different code than the branch being reviewed.
  - confidence: 0.93
- **[P2] Preserve 256-color light-background detection:** `src/tui/theme/detect.ts:30`
  - body: Current main handles 0-255 `COLORFGBG` backgrounds and has tests for light values like 231 and 255. This detector only treats 7 and 15 as light, so users on common 256-color light terminals would regress to the dark palette by default.
  - confidence: 0.91
- **[P2] Keep mounted theme objects live on theme switches:** `src/tui/theme/theme.ts:223-235`
  - body: `applyPalette` replaces exported theme objects, but mounted Markdown, select-list, editor, and other components keep the object reference they were constructed with. After `/theme`, parts of the existing UI can remain in the old palette even though a render is requested.
  - confidence: 0.86
- **[P2] Normalize theme names before palette lookup:** `src/tui/theme/detect.ts:11-13`
  - body: Current main treats `OPENCLAW_THEME` case-insensitively for `light` and `dark`, but this path returns the env var verbatim for exact lookup. Values like `OPENCLAW_THEME=Light` would fall back to dark despite being a valid existing override spelling in practice.
  - confidence: 0.8
