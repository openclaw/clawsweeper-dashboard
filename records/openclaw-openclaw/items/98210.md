- `P1`: The PR targets a broken native iOS Talk workflow where configured SecretRef-backed ElevenLabs credentials are treated as missing and can push users toward plaintext workarounds.

## Summary

Keep open for maintainer review: the patch appears to fix the current iOS Talk SecretRef readback bug, but it intentionally changes a credential-bearing `talk.config` response and needs explicit acceptance of the `operator.talk.secrets` boundary.

## What This Changes

The PR changes Gateway `talk.config` so authorized `includeSecrets` callers get a resolved active Talk provider `apiKey` string while provider maps stay source-shaped and read-scope responses remain redacted.

## Best Possible Solution

Land this narrow Gateway response fix after maintainers accept the scoped credential boundary; otherwise replace it with a narrower native handoff that avoids returning provider credentials.

## Reproduction Assessment

Yes: source inspection on current main shows `includeSecrets` returns before runtime SecretRef resolution, while the iOS parser only accepts a string `apiKey`. I did not run a physical iOS session in this read-only review.

## Solution Assessment

Yes for the code layer: Gateway owns runtime SecretRef resolution and iOS cannot resolve SecretRef objects. The remaining question is maintainer acceptance of `operator.talk.secrets` as the credential boundary for native clients.
