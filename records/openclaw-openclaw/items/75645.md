## Summary

Keep open. The PR addresses the previously reported changelog gap and the main approach is well-scoped, but the new dispatcher suppression runs after TTS synthesis, so `messages.tts.mode: "all"` can still convert provider inventory text into an audio attachment and deliver that media on non-direct surfaces.

## What This Changes

The PR adds an internal provider-inventory classifier and ReplyPayload tag, tags image/video list tool results, suppresses tagged tool text on non-direct reply surfaces while preserving media/direct delivery, adds classifier and dispatcher tests, and records a changelog entry.

## Best Possible Solution

Keep the internal-shape tag approach, but make provider-inventory payloads non-deliverable on non-direct surfaces before TTS synthesis, or explicitly remove TTS-generated media and `spokenText` for that shape, while preserving direct/DM inventory visibility and real generated media delivery.

## Reproduction Assessment

Yes. There is a high-confidence static path: list actions return `details.providers` plus visible text, compact emission bypasses the normal tool-output gate, Discord guild channels are `ChatType: "channel"`, and the PR still lets TTS mode `all` turn the text into media before the new suppression branch preserves media.

## Solution Assessment

No. The current PR is close and the changelog is fixed, but suppressing after TTS is not the narrowest safe delivery fix; the visibility decision should happen before synthesis or strip synthesized audio for provider-inventory on non-direct surfaces.
