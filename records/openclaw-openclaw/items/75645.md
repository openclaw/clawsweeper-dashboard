## Summary

Keep open. The PR addresses a real provider-inventory leak, but the current diff suppresses tagged text after the tool-result TTS step, so `messages.tts.mode: "all"` can still turn provider/model/auth-hint inventory into audio and deliver it on shared surfaces.

## What This Changes

The PR tags image/video provider-inventory tool results with an internal shape marker, extends reply payload types, suppresses tagged text on non-direct dispatch surfaces, adds classifier/dispatcher tests, and updates the changelog.

## Best Possible Solution

Keep the internal-shape tagging approach, but make provider-inventory payloads non-deliverable on non-direct surfaces before TTS synthesis, or explicitly strip TTS-generated media and `spokenText` for that shape while preserving direct/DM inventory visibility and real generated media delivery.

## Reproduction Assessment

Yes. There is a high-confidence static reproduction path: list actions return `details.providers`, compact emission bypasses ordinary tool-output suppression, Discord guild messages use `ChatType: "channel"`, and this PR can still synthesize tagged text into audio before the new suppression branch runs.

## Solution Assessment

No. The proposed direction is close, but suppressing after TTS is not the narrowest safe fix; the visibility decision should happen before synthesis or remove synthesized audio for provider-inventory on non-direct surfaces.
