## Summary

This should stay open: current `main` still renders the `/sessions` table without column resize handles, and the inline Label editor is still width-capped enough to truncate useful label text. The narrow label-editor usability fix is clear, but the broader draggable-column behavior needs a maintainer UI scope decision.

## What This Changes

The issue asks for resizable Sessions table columns and a wider or focus-expanding inline Label editor in the Control UI `/sessions` view.

## Best Possible Solution

Fix the Label editor readability with a bounded Control UI change first, and only add reusable draggable table resizing if maintainers want that UX pattern across data tables.

## Reproduction Assessment

Yes from source inspection for the central usability problem: current `main` caps the inline Label input at 140px and provides no resize-handle UI in the Sessions table. I did not run a browser repro in this read-only pass, but the shipped source and reporter screenshots line up.

## Solution Assessment

Unclear for the full requested solution: widening or focus-expanding the Label editor is the narrow maintainable fix, while draggable column resizing needs an explicit Control UI product decision and probably a reusable table pattern.
