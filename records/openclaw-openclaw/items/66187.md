- `impact:session-state`: The affected behavior is subagent registry liveness, session-state drift, and quota accounting.

## Summary

Keep open: current main and the latest release still intentionally classify ended orchestrator runs as active when they have pending descendants, and the same predicate feeds the `sessions_spawn` max-children quota. The adjacent stale-unended orphan class was fixed, but this issue remains the canonical open item for the ended-parent quota/display semantics.

## What This Changes

The issue asks OpenClaw to stop ended subagent runs from appearing active or consuming subagent concurrency quota, with optional archival/pruning behavior.

## Best Possible Solution

Keep this issue as the canonical place to define separate predicates for subagent display/announce liveness and max-children quota accounting, preserving descendant cleanup visibility without letting ended controllers permanently occupy spawn slots.

## Reproduction Assessment

Yes at source level: current main’s quota path and regression tests preserve the reported ended-parent-with-pending-descendants active behavior. I did not run a live nested subagent scenario in this read-only review.

## Solution Assessment

No, the current implementation is not the best complete solution for this report. It is a valid cleanup/announce visibility model, but quota accounting should likely use a narrower predicate than list/display liveness.
