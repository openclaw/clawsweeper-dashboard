- `impact:session-state`: The issue is about subagent registry liveness and session quota accounting drifting from actual run completion.

## Summary

Keep open: current main and latest release still classify an ended parent run with pending descendants as active, and the spawn quota gate consumes that same active count. The stale-unended orphan variant has been fixed separately, but this issue remains the canonical open decision for splitting quota accounting from display/cleanup liveness.

## What This Changes

The issue reports that completed subagent entries with `endedAt` can still appear active and consume child-spawn quota when descendants remain pending.

## Best Possible Solution

Define separate predicates for display/announce liveness, cleanup-pending tracking, and spawn quota so ended controllers do not permanently consume child-spawn slots while descendant cleanup remains observable.

## Reproduction Assessment

Yes, at source level: current list and quota predicates keep ended parents active while descendants remain pending, and focused tests preserve that behavior. I did not run a live nested subagent scenario in this read-only review.

## Solution Assessment

No, the current implementation is not the best complete solution for this report because one active model drives both display/cleanup visibility and spawn quota. The safer fix is to split those semantics after maintainer confirmation, not add a broad new config knob first.
