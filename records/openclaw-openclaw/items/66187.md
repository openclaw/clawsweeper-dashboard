- `P1`: The issue describes a broken nested multi-agent workflow where stale subagent state can exhaust all spawn slots for real users.
- `impact:session-state`: The affected behavior is subagent registry liveness, session-store drift, and per-session quota accounting.

## Summary

Keep open: current main and v2026.6.6 still preserve the reported behavior where ended parent/orchestrator runs remain active while descendants are pending, and that same active-count path gates `sessions_spawn`. The stale-unended orphan class was fixed separately, but this quota/display semantic still needs a maintainer decision.

## What This Changes

The issue requests that subagent runs with `endedAt` stop appearing active and stop consuming the per-session `sessions_spawn` child quota.

## Best Possible Solution

Define separate predicates for cleanup/display liveness and spawn quota accounting, preserving descendant cleanup visibility while preventing ended controllers or session-store-completed registry rows from occupying child slots indefinitely.

## Reproduction Assessment

Yes at source level: current main’s quota and list paths still classify ended runs with pending descendants as active, and tests intentionally protect that behavior. I did not run a live nested subagent scenario in this read-only review.

## Solution Assessment

No, current main is only a partial adjacent fix: stale unended rows are bounded, but the ended-parent quota/display behavior remains. The cleaner fix is to split quota accounting from cleanup visibility and reconcile terminal session-store state into registry liveness.
