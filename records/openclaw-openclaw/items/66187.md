## Label Justifications

- `P1`: The report describes a broken multi-agent spawning workflow where stale subagent state exhausts child slots for real users.
- `impact:session-state`: The affected behavior is subagent registry liveness, session-state drift, and quota accounting.

## Summary

Keep open: current `main` and the latest release still classify an ended parent run with pending descendants as active, and that same predicate feeds the `sessions_spawn` child quota. The stale-unended orphan variant is fixed, but this issue remains the canonical item for deciding how display/cleanup liveness should differ from quota accounting.

## What This Changes

The issue reports that finished subagent runs with `endedAt` can remain in the active list and continue occupying the per-session child quota.

## Best Possible Solution

Define separate predicates for subagent display/announce liveness and max-children quota accounting, so descendant cleanup stays visible without letting ended controllers permanently consume spawn slots.

## Reproduction Assessment

Yes, at source level: current `main` tests and implementation preserve the reported ended-parent-with-pending-descendants active behavior, and both native and ACP spawn paths use that count for quota gating. I did not run a live nested subagent scenario in this read-only review.

## Solution Assessment

No, the current implementation is not the best complete solution for this report. It is a valid cleanup/announce visibility model, but quota accounting likely needs a narrower predicate than list/display liveness.
