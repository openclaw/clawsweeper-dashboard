## Summary

Keep open. Current main still treats ended subagent runs as active when they have pending descendants, and that same lifecycle predicate feeds the spawn quota path; existing tests intentionally preserve the behavior, so changing it needs maintainer lifecycle/quota policy rather than cleanup closure.

## What This Changes

The issue asks for ended subagent runs to leave the active list, stop counting against spawn concurrency, and be archived or pruned by the configured retention policy.

## Best Possible Solution

Define separate concepts for cleanup or announce settlement versus user-visible active work and spawn quota, then apply the chosen policy consistently across list output, registry counters, native spawn, ACP spawn, retention docs, and regression tests.

## Reproduction Assessment

Yes, at source level. An ended parent run with pending or cleanup-incomplete descendants still passes the current active predicate and active-child quota count, and tests explicitly expect that behavior.

## Solution Assessment

Unclear until maintainers choose the lifecycle policy. A list-only change would not fix spawn quota; the maintainable fix is to split cleanup-aware liveness from active-list and quota liveness if the reported behavior is accepted.
