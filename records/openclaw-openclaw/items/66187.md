## Summary

Keep open. Current main still makes ended orchestrator runs active when pending descendants or cleanup-incomplete descendants exist, and the same predicate feeds the spawn-quota rejection path; the related open PR is an opposing cleanup-policy change rather than a merged fix for this report.

## What This Changes

The issue asks for subagent runs with `endedAt` set to stop appearing active and stop consuming child-spawn quota.

## Best Possible Solution

Define separate lifecycle concepts for cleanup/announce settlement versus user-visible active rows and spawn quota, then apply that policy consistently across list output, registry counters, native spawn, ACP spawn, tests, and docs.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is an ended parent run with pending or cleanup-incomplete descendants; current source and tests show it remains active and quota-counted.

## Solution Assessment

Unclear until maintainers choose the lifecycle policy. The narrow maintainable fix is not to hide rows only, but to split cleanup-aware liveness from user-visible active/quota liveness if the reporter's behavior is accepted.
