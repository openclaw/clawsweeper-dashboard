## Summary

Keep open. Current main still classifies ended subagent orchestrators as active when descendants remain pending or cleanup-incomplete, and the same predicate feeds both `/subagents list` output and native/ACP spawn quota rejection; the related stale-unended orphan work only addresses a different no-`endedAt` liveness case.

## What This Changes

The issue asks OpenClaw to stop showing `endedAt`-set subagent runs as active, stop counting them against `maxChildrenPerAgent`/spawn quota, and eventually archive or prune them after a configurable duration.

## Best Possible Solution

Define separate liveness concepts for cleanup/announce settlement versus user-visible active rows and spawn quota, then update list formatting, registry counters, native/ACP quota checks, tests, and subagent docs consistently.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is an ended parent run with a pending or cleanup-incomplete descendant; current tests construct that shape and assert it remains active and quota-counted.

## Solution Assessment

Unclear until maintainers choose the policy. Hiding rows from `/subagents list` alone is not enough; the maintainable fix would split cleanup-aware liveness from list/quota liveness if the reporter's behavior is accepted.
