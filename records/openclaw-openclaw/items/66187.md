## Summary

Keep open. Current main still classifies ended subagent runs as active when descendant cleanup is pending and feeds that same predicate into the spawn quota path; existing tests preserve the behavior, while https://github.com/openclaw/openclaw/pull/57041 argues for an even more conservative cleanup-pending active state, so this needs maintainer lifecycle/quota policy rather than cleanup closure.

## What This Changes

The issue requests that ended subagent runs stop appearing active, stop consuming spawn quota, and be pruned or archived after the configured retention window.

## Best Possible Solution

Define separate lifecycle concepts for user-visible active work, cleanup or announce settlement, and spawn quota, then apply that contract consistently across list output, registry counters, native spawn, ACP spawn, archive retention, docs, and regression tests.

## Reproduction Assessment

Yes, at source level. An ended parent run with pending or cleanup-incomplete descendants still passes the current active-list predicate and active-child quota count, and tests explicitly lock in that behavior.

## Solution Assessment

Unclear until maintainers choose the lifecycle policy. A list-only change would not fix quota exhaustion; the maintainable fix is to split cleanup-aware liveness from active-list and quota liveness if the reported behavior is accepted.
