## Label Justifications

- `P1`: The reported active-slot leak can block nested subagent spawning once the per-session child quota is exhausted.
- `impact:session-state`: The issue concerns subagent lifecycle state being interpreted inconsistently across display, cleanup, and quota accounting.

## Summary

Keep open: current `main` still intentionally treats an ended orchestrator with pending descendants as active for both `/subagents list` display and the `sessions_spawn` child quota. The adjacent stale-unended orphan class is addressed, but this issue remains the canonical lifecycle/quota semantics decision for ended parents with descendants.

## What This Changes

The issue asks OpenClaw to stop showing `endedAt`-set subagent entries as active, stop counting them against child spawn quota, and consider configurable archive/prune behavior.

## Best Possible Solution

Keep this issue open as the canonical lifecycle/quota item, then define one explicit state model for live, ended-but-cleanup-pending, and descendant-pending subagent runs before changing list, status, and quota behavior together.

## Reproduction Assessment

Yes from source, without running a live nested orchestration: current code and tests show an ended orchestrator with pending descendants is counted active, and `sessions_spawn` gates new children on that active count.

## Solution Assessment

No final fix path yet: the desired direction is plausible, but the best solution needs a maintainer decision on parent-slot semantics and whether the requested configurable archive/prune knob should exist.
