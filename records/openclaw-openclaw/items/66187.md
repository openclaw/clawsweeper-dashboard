## Summary

Keep open. Current main still treats ended subagent orchestrators with pending descendants as active and uses that same count for native and ACP spawn quota rejection; the related stale-unended orphan fix addresses a different liveness case.

## What This Changes

The issue requests that subagent runs with `endedAt` set stop appearing as active and stop consuming active-child quota while still being archived or pruned after the configured duration.

## Best Possible Solution

Define a split liveness model: ended orchestrators with pending descendants should remain cleanup-aware for announce and descendant settlement, but should be excluded from user-active list entries and per-session spawn quota if maintainers accept that product behavior. Then update list/status formatting, registry count helpers, native and ACP spawn quota checks, tests, and subagent docs together.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is an ended parent run with a pending child; current tests build that shape and assert it remains active and quota-counted, which matches the reporter's failure path.

## Solution Assessment

Unclear until maintainers settle the policy. Hiding rows from `/subagents list` alone is not the best fix; the safer direction is to split cleanup/announce liveness from user-visible active and quota accounting.
