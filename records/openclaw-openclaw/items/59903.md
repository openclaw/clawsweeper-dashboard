## Summary

Keep open. Current main has not implemented the requested behavior: the shared `/status` task formatter can still select a recent successful terminal task, label it as recently finished, and render its `terminalSummary`; the PR remains useful but needs to be ported to the current `src/status/status-text.ts` path before it can be considered resolved.

## What This Changes

The PR hides completed-success background task output from `/status`, preserves active and recent-failure task summaries including agent-local failure fallback, and adds regression coverage for those cases.

## Best Possible Solution

Port the completed-success suppression into the current shared status rendering, preserve recent failure visibility including agent-local fallback, and add focused regression tests for completed-success-only task output on `/status` and the parallel session status surface.

## Reproduction Assessment

Yes. A recent succeeded task with `terminalSummary` and no active or failed task follows `buildTaskStatusSnapshot` into `formatSessionTaskLine`, where current main labels it `recently finished` and renders the terminal summary into the status card.

## Solution Assessment

Yes in direction, but not merge-ready against current main. The narrow maintainable fix is the PR’s guard of active or recent-failure state, applied to the current `src/status/status-text.ts` formatter and mirrored where the session status tool formats the same task line.
