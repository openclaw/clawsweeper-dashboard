## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Resolve client IP before enforcing request limits:** `src/gateway/server-http.ts:769`
  - body: This calls the limiter with `req.socket.remoteAddress`, which is the proxy hop in reverse-proxy deployments. With the default loopback exemption, a local proxy bypasses the limiter entirely; with a non-loopback proxy, unrelated clients share one bucket and can cross-throttle each other. Move this check after the config snapshot and key it with `resolveRequestClientIp(req, trustedProxies, allowRealIpFallback)`.
  - confidence: 0.96
- **[P2] Expose a supported startup and config path:** `src/gateway/server-runtime-state.ts:62`
  - body: The new runtime option is only an optional parameter and current startup never creates or disposes this limiter from documented Gateway config. As a result, the PR adds security-critical plumbing without a supported way for operators to enable, configure, or lifecycle-manage the feature it advertises.
  - confidence: 0.88
