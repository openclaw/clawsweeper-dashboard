- **[P1] Persist thread routes on the active session:** `extensions/mattermost/src/mattermost/monitor.ts:1393`
  - body: The new non-direct path writes every Mattermost route to `route.mainSessionKey` instead of the active thread/session entry. Cron delivery checks the requested thread session before falling back to the main entry, so thread-specific jobs still miss their own stored route and can inherit the latest unrelated thread from the shared main bucket.
  - confidence: 0.9
- **[P1] Avoid redirecting default heartbeats to public channels:** `extensions/mattermost/src/mattermost/monitor.ts:1393`
  - body: Because this update runs for every non-direct post and targets the shared main session, later heartbeats with no explicit session or target can read a public Mattermost channel as the main fallback destination. Store the route on the scoped session or make thread/channel heartbeat delivery explicit instead of overwriting the shared main route.
  - confidence: 0.86
