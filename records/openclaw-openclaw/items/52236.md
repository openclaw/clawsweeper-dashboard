Full review comments:

- **[P1] Persist routes on the thread session entry:** `extensions/mattermost/src/mattermost/monitor.ts:1380-1386`
  - body: Once this block runs for non-DM Mattermost turns, it still writes through the last-route session key derived from the base route rather than the active thread `sessionKey`. Cron delivery checks the targeted thread session before falling back to shared entries, so multiple Mattermost threads can inherit the wrong stored `threadId` and post to the wrong thread.
  - confidence: 0.88
- **[P1] Avoid redirecting main-session deliveries into public threads:** `extensions/mattermost/src/mattermost/monitor.ts:1380-1386`
  - body: Broadening this write updates shared route context for channel/thread traffic, not only the thread-scoped session that needs cron recovery. Later main-session heartbeat or wake deliveries can then reuse the public Mattermost thread as the last route, changing existing delivery behavior for users whose main session previously resolved elsewhere.
  - confidence: 0.82
- **[P2] Do not claim heartbeat routing is fixed here:** `extensions/mattermost/src/mattermost/monitor.ts:1381-1384`
  - body: The added comment and PR description still frame heartbeat wakeups as fixed by saving `threadId`, but heartbeat-mode target resolution intentionally drops session-derived thread IDs unless the turn source supplies one. This patch is at most a partial cron route-persistence prerequisite unless the heartbeat path is changed too.
  - confidence: 0.9
