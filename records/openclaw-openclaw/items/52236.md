## What This Changes

The PR removes the Mattermost direct-message guard around last-route persistence and adds `threadId` to the stored delivery context.

## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

No. This is only a partial and risky mitigation; the safer fix is to persist the route on the active thread/session boundary or a generic inbound fallback without overwriting shared main-session delivery state.
