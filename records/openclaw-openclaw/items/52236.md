- **[P1] Persist routes on the thread session entry:** `extensions/mattermost/src/mattermost/monitor.ts:1383-1386`
  - body: Once this block runs for non-DM Mattermost turns, it still writes through `route.mainSessionKey` instead of the active thread `sessionKey`. Multiple Mattermost threads can then share the same stored `threadId`, so a later cron or wake for one thread can inherit another thread's route and post to the wrong place.
  - confidence: 0.88
- **[P1] Avoid redirecting main-session deliveries into public threads:** `extensions/mattermost/src/mattermost/monitor.ts:1383-1386`
  - body: Broadening this write updates shared main-session route context for channel/thread traffic, not only the thread-scoped session that needs recovery. Later main-session heartbeat or wake deliveries can reuse that public Mattermost thread as the last route, changing existing delivery behavior.
  - confidence: 0.82
- **[P2] Do not claim heartbeat routing is fixed here:** `extensions/mattermost/src/mattermost/monitor.ts:1384-1386`
  - body: The new comment says cron and heartbeat wakeups keep thread context, but heartbeat-mode target resolution intentionally drops session-derived thread IDs unless the turn source supplies one. This patch is at most a partial cron route-persistence prerequisite unless the heartbeat path changes too.
  - confidence: 0.9
