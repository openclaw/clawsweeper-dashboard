## Summary

Keep this PR open but not merge-ready: the underlying Mattermost cron/thread delivery bug still exists on current main, yet this abandoned draft writes thread context onto the shared route session instead of the active thread session entry and lacks real behavior proof.

## What This Changes

The PR removes the Mattermost direct-message guard around route persistence and stores `threadId` in inbound delivery context for all session kinds.

## Best Possible Solution

Refresh or replace this with a Mattermost-owned fix that persists delivery context on the active thread session entry, keeps shared main-route behavior stable, handles cron and heartbeat semantics explicitly, adds focused regression coverage, and includes redacted real Mattermost proof.

## Reproduction Assessment

Yes, source-reproducible. Current Mattermost still persists `updateLastRoute` only for direct inbound messages, while cron delivery first checks the requested thread session before falling back to main-route state.

## Solution Assessment

No. The current PR is a plausible prerequisite, but the best fix must persist the route on the active thread session entry and separately prove cron/heartbeat delivery instead of unconditionally updating shared route state.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Write non-direct routes to the thread session entry:** `extensions/mattermost/src/mattermost/monitor.ts:1380-1396`
  - body: Removing the guard makes every non-direct Mattermost turn update `inboundLastRouteSessionKey`, but that key is computed from `route.sessionKey`, not the active `ctxPayload.SessionKey` that includes `:thread:<id>`. Cron delivery checks the requested thread session first and then falls back to main route state, so thread A can inherit thread B's latest stored `threadId` instead of its own route.
  - confidence: 0.9
- **[P2] Remove the heartbeat claim from this partial fix:** `extensions/mattermost/src/mattermost/monitor.ts:1381-1384`
  - body: The new comment says saving `threadId` here fixes cron and heartbeat wakeups, but heartbeat-mode target resolution intentionally ignores session-stored thread IDs unless the thread came from an explicit turn source or plugin route. Keep the claim scoped to the path this patch actually fixes, or add the missing heartbeat change and proof.
  - confidence: 0.82
