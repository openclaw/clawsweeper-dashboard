## Summary

Keep open. Current main still has the Mattermost thread-route persistence gap, but this draft is abandoned and the proposed patch does not fully cover the claimed heartbeat/cron behavior.

## What This Changes

The PR removes the Mattermost direct-message guard around inbound last-route recording and stores `effectiveReplyToId` as `deliveryContext.threadId`.

## Best Possible Solution

Keep the canonical Mattermost thread-context work centered on #45082/#52120 and land or replace a complete Mattermost-local fix covering persisted route context, heartbeat route-thread preservation, and cron/tool current-thread targeting.

## Reproduction Assessment

Yes for source-level reproduction: current main creates Mattermost thread-scoped session keys for channel/group posts, but last-route delivery context is only recorded for direct messages. Live Mattermost cron/heartbeat delivery was not exercised.

## Solution Assessment

No. Persisting `deliveryContext.threadId` for all Mattermost session types is a plausible prerequisite, but heartbeat delivery still ignores inherited session thread IDs unless the Mattermost plugin opts into route-thread preservation.
