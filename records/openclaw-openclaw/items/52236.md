## Summary

Keep open for maintainer handling: current main still shows the Mattermost non-direct route gap, but this draft's main-session write is unsafe, the author marked it abandoned, and related canonical work remains open.

## What This Changes

The PR removes the direct-only guard around Mattermost last-route persistence and stores `effectiveReplyToId` as `deliveryContext.threadId`.

## Best Possible Solution

Replace or consolidate this with a current-main fix that writes routable delivery context to the active non-direct session and keeps heartbeat stale-thread safeguards intact.

## Reproduction Assessment

Yes, at source level: current main builds non-direct Mattermost thread sessions with `OriginatingTo` and `MessageThreadId`, but only supplies `updateLastRoute` for direct messages. I did not run a live Mattermost workspace during this read-only review.

## Solution Assessment

No. Persisting route context is the right direction, but this patch writes non-direct thread context to the shared main session instead of the active thread session; the safer path is active-session persistence or the core fallback tracked by https://github.com/openclaw/openclaw/pull/77442.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Persist non-direct routes on the active session:** `extensions/mattermost/src/mattermost/monitor.ts:1380`
  - body: Removing the direct-only guard still writes the route through `route.mainSessionKey` while the active turn records a thread-scoped `SessionKey`. Cron delivery checks the requested thread session before falling back to main, so the intended thread entry can remain unroutable or inherit another thread's main-route write.
  - confidence: 0.88
- **[P1] Avoid storing public thread context on the main session:** `extensions/mattermost/src/mattermost/monitor.ts:1395`
  - body: The patch writes every non-DM Mattermost route, including public thread context, into the shared main session. Later main-session heartbeats or manual deliveries can then jump into the last public thread instead of preserving main-session routing safety.
  - confidence: 0.82
