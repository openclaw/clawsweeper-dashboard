## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Persist thread routes on the thread session entry:** `extensions/mattermost/src/mattermost/monitor.ts:1393`
  - body: This broadens `updateLastRoute` to non-DM messages but still writes to `route.mainSessionKey`. Thread-scoped cron delivery checks the requested session entry before falling back to main, so if thread A runs after thread B was last stored on main, A can inherit B's `threadId` and post to the wrong Mattermost thread. Store this route on the active thread/session entry instead.
  - confidence: 0.9
- **[P1] Do not overwrite main-session delivery with channel threads:** `extensions/mattermost/src/mattermost/monitor.ts:1393`
  - body: For every non-DM message, this writes a public Mattermost channel/thread route into the shared main session. Main-session heartbeats or manual wakes that use the last route can then leave the expected main/DM route and reply in that public thread. The fix needs to preserve the thread-scoped route without mutating the shared main route.
  - confidence: 0.86
