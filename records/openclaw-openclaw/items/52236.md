## Summary

Keep this PR open for maintainer handling: the reported Mattermost thread-route gap is still source-reproducible on current main, but this draft branch is abandoned, conflicting, missing real behavior proof, and the patch introduces blocking routing regressions.

## What This Changes

The PR removes Mattermost's direct-message-only guard around inbound last-route persistence and adds `effectiveReplyToId` as `deliveryContext.threadId`.

## Best Possible Solution

Replace or substantially revise this draft with a current-main fix that persists non-direct Mattermost route context on the active thread session, preserves main-session heartbeat safety, and includes focused regression tests plus redacted real Mattermost proof.

## Reproduction Assessment

Yes, by source inspection. Current main records Mattermost last-route context only for direct posts while non-direct thread turns carry `SessionKey`, `OriginatingTo`, and `MessageThreadId` without a matching route update for later cron or delivery lookup.

## Solution Assessment

No. Persisting non-direct route context is the right direction, but this patch writes it to the shared main session and does not resolve the heartbeat/current-thread cron policy the author called out.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Persist non-direct routes on the active session:** `extensions/mattermost/src/mattermost/monitor.ts:1380`
  - body: Removing the direct-only guard still writes the route through `route.mainSessionKey` while the active turn records a thread-scoped `SessionKey`. Cron delivery checks the requested thread session before falling back to main, so the intended thread entry can remain unroutable or inherit another thread's main-route write. Persist the route on the active session entry.
  - confidence: 0.88
- **[P1] Avoid storing public thread context on the main session:** `extensions/mattermost/src/mattermost/monitor.ts:1380`
  - body: The patch writes every non-DM Mattermost route, including public thread context, into the shared main session. Heartbeat `target: last` has explicit safeguards against stale session thread reuse, so later main-session heartbeats or manual deliveries can jump into the last public thread. Scope non-direct persistence before changing the shared route.
  - confidence: 0.82
