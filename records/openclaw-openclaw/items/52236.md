## Summary

## What This Changes

The PR removes the direct-message guard around Mattermost inbound last-route updates and adds `threadId` to the persisted delivery context in `extensions/mattermost/src/mattermost/monitor.ts`.

## Best Possible Solution

Replace or repair this branch with a narrow Mattermost route-persistence fix that writes complete delivery context to the correct active session entry, preserves heartbeat stale-thread safeguards, and includes redacted real Mattermost proof.

Yes, at source level: current main still omits `updateLastRoute` for Mattermost non-direct turns, and the PR can be traced through session persistence and cron/heartbeat delivery routing. I did not run a live Mattermost workspace in this read-only review.

## Solution Assessment

No. This is only a partial mitigation because it removes the guard without changing the target session key, and heartbeat routing still intentionally drops session-stored thread ids.
