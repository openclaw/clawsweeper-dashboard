## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Persist the route on the thread session entry:** `extensions/mattermost/src/mattermost/monitor.ts:1395`
  - body: Adding `threadId` to this write still records the route on the shared/base route entry in the PR head. Cron delivery loads the explicit thread-session entry first, so a job targeting thread A can still miss its own route context or inherit the last route from a different thread.
  - confidence: 0.92
- **[P1] Keep non-direct posts out of the shared main route:** `extensions/mattermost/src/mattermost/monitor.ts:1380`
  - body: Removing the direct-message guard makes group/channel posts update the same stored route used by later main-session deliveries. A normal public Mattermost mention can therefore redirect future main-session heartbeat or cron output into that public channel/thread.
  - confidence: 0.86
- **[P2] Do not claim heartbeat routing is fixed here:** `extensions/mattermost/src/mattermost/monitor.ts:1382`
  - body: The heartbeat resolver intentionally avoids inheriting session-stored thread IDs in heartbeat mode, so saving `threadId` here cannot fix thread-scoped heartbeats without a separate explicit turn-source or target change.
  - confidence: 0.9
