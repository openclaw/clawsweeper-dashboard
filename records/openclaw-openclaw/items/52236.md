## Summary

Keep open for maintainer handling: current main still has the Mattermost non-direct route persistence gap, but this draft branch is abandoned by its author, lacks real behavior proof, and the diff still introduces route-safety regressions.

## What This Changes

The PR removes Mattermost's direct-message guard around inbound last-route persistence and adds `effectiveReplyToId` to the saved delivery context.

## Best Possible Solution

Replace or substantially revise this draft with a current-main fix that records non-direct Mattermost delivery context on the active thread session, preserves main-session heartbeat safety, and includes focused regression tests plus redacted real Mattermost proof.

## Reproduction Assessment

Yes by source inspection. Current main records Mattermost last-route context only for direct posts, while non-direct thread turns carry `SessionKey`, `OriginatingTo`, and `MessageThreadId` without a route update for cron or later delivery lookup.

## Solution Assessment

No. Persisting non-direct route context is the right direction, but this patch writes it to the shared main session instead of the active thread session and does not settle the heartbeat/current-thread cron policy that the author called out.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Persist non-direct routes on the active session:** `extensions/mattermost/src/mattermost/monitor.ts:1380`
  - body: Removing the direct-only guard keeps `sessionKey: route.mainSessionKey` for channel/thread posts while the turn records `ctxPayload.SessionKey` on the thread-scoped key. Cron delivery checks the requested thread session before falling back to main, so the intended thread entry can stay unroutable or inherit another thread's main-route write. Persist the route on the active session entry.
  - confidence: 0.88
- **[P1] Avoid sending main-session heartbeats into public threads:** `extensions/mattermost/src/mattermost/monitor.ts:1380`
  - body: This now writes every non-DM Mattermost route, including public thread context, into the shared main session. Heartbeat `target: last` resolves from that main session and current main has explicit safeguards around stale thread reuse, so later main-session heartbeats or manual deliveries can jump into the last public thread. Scope non-direct persistence before changing the shared route.
  - confidence: 0.82
