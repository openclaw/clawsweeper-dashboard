## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Persist thread routes on the active session entry:** `extensions/mattermost/src/mattermost/monitor.ts:1393`
  - body: Once non-direct messages enter this block, every Mattermost channel/thread updates `route.mainSessionKey`. A cron for thread A after thread B was the latest inbound message can then inherit B's stored `threadId` from the shared main entry instead of A's thread, so the route update needs to target the active or explicitly selected session rather than the shared main session.
  - confidence: 0.91
- **[P2] Handle heartbeat routing before claiming it is fixed:** `extensions/mattermost/src/mattermost/monitor.ts:1385-1386`
  - body: The new comment says this fixes cron/heartbeat thread wakeups, but heartbeat resolution intentionally drops session-stored thread IDs in `mode: "heartbeat"` unless the turn source explicitly carries a thread. Saving `deliveryContext.threadId` here leaves that heartbeat case broken, so the patch should either handle that path or narrow the claim and tests to the cron path it actually addresses.
  - confidence: 0.88
