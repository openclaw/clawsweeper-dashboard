## Summary

Keep this PR open for maintainer handling, but it is not mergeable as-is: the underlying Mattermost thread-route bug still appears valid on current main, while this draft is abandoned, missing real behavior proof, and writes non-direct thread context onto the shared main route instead of the active thread session.

## What This Changes

The PR removes the Mattermost direct-message guard around `updateLastRoute` and adds `effectiveReplyToId` to persisted delivery context for inbound Mattermost sessions.

## Best Possible Solution

Replace or substantially revise this draft with a current-main fix that persists non-direct Mattermost delivery context on the active thread session, preserves main-session route safety, and separately resolves the heartbeat/current-thread cron semantics with regression tests and real Mattermost proof.

## Reproduction Assessment

Yes by source inspection, though not by live run here. Current main still omits `updateLastRoute` for non-direct Mattermost posts, while cron delivery depends on a populated thread-session entry before falling back to main.

## Solution Assessment

No. Persisting thread context for non-direct inbound Mattermost sessions is the right direction, but this patch writes it to the shared main session and does not address heartbeat mode’s deliberate thread-dropping behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Record non-direct routes on the active thread session:** `extensions/mattermost/src/mattermost/monitor.ts:1380`
  - body: Removing the direct-only guard makes channel/thread posts update the shared main session route while the inbound context is already using a thread-scoped `SessionKey`. Cron delivery checks the requested thread session first, so the intended thread entry remains without route context; if another thread was the latest main-route write, a job for one thread can inherit the other thread. Persist the route on the active session entry or otherwise maintain per-thread route state.
  - confidence: 0.88
- **[P1] Keep main-session heartbeats out of public threads:** `extensions/mattermost/src/mattermost/monitor.ts:1380`
  - body: This change updates the main session route for every non-DM Mattermost message. With heartbeat target `last`, later main-session heartbeat or manual delivery can resolve to that public channel/thread instead of the previous main route. Scope non-direct route persistence to the session that received the turn, or add an explicit heartbeat policy before changing the shared main route.
  - confidence: 0.82
