Full review comments:

- **[P1] Persist routes on the thread session entry:** `extensions/mattermost/src/mattermost/monitor.ts:1383-1399`
  - body: Once this block runs for non-DM Mattermost turns, it still writes through `route.mainSessionKey` rather than the active thread `sessionKey`. Session-keyed cron delivery can then inherit whichever public thread last updated the shared route instead of the thread that the job targeted.
  - confidence: 0.88
- **[P1] Avoid redirecting main-session deliveries into public threads:** `extensions/mattermost/src/mattermost/monitor.ts:1391-1399`
  - body: Broadening this write updates the shared main-session route for channel and thread traffic, not only the thread-scoped session that needs recovery. A later main-session cron or manual wake with an explicit main session can reuse that stored thread and post somewhere the user did not target.
  - confidence: 0.82
- **[P2] Do not claim heartbeat routing is fixed here:** `extensions/mattermost/src/mattermost/monitor.ts:1383-1386`
  - body: The added comment says saving `threadId` fixes heartbeat wakeups, but heartbeat-mode target resolution intentionally drops session-derived thread IDs unless the turn source supplies one. This patch is at most a partial cron route-persistence prerequisite.
  - confidence: 0.9
