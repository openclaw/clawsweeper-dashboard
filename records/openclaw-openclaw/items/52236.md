## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Persist the route on the thread session:** `extensions/mattermost/src/mattermost/monitor.ts:1380`
  - body: This widens `updateLastRoute` for non-direct Mattermost messages but still relies on the shared last-route target instead of the actual thread-scoped session. If thread A receives a cron after thread B was the latest inbound message, the job can inherit B's stored `threadId` and post into the wrong thread; persist the route on the specific session entry or use a shared fallback with tests for multiple active threads.
  - confidence: 0.86
- **[P2] Do not claim heartbeat routing is fixed here:** `extensions/mattermost/src/mattermost/monitor.ts:1381-1383`
  - body: The heartbeat resolver still runs in heartbeat mode and drops session-inherited thread ids unless the thread id is supplied by an explicit turn source. Saving `threadId` in this last-route update is not enough to keep Mattermost heartbeat replies in the thread, so the patch needs an explicit heartbeat change or narrower comments/body.
  - confidence: 0.9
