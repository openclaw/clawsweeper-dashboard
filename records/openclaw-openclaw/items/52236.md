## Summary

## What This Changes

The PR removes the Mattermost direct-message guard around last-route persistence and adds `threadId` to the stored delivery context for inbound Mattermost messages.

## Best Possible Solution

Keep the canonical issue open and replace this branch with a targeted repair that records routable context on the active thread session or explicit turn source, preserves stale-thread safeguards, and includes live Mattermost thread-delivery proof.

## Reproduction Assessment

Yes, at source level: Mattermost still only supplies `updateLastRoute` for direct inbound turns on current main, while cron delivery recovers route context from stored session delivery metadata. I did not run a live Mattermost workspace, so this is not a live reproduction.

## Solution Assessment

No. The patch is a plausible prerequisite, but not the best fix because it writes thread routes to the shared main session and does not complete the heartbeat path; the safer fix is per-session or turn-source route persistence with stale-thread tests.
