## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Persist thread routes on the active session entry:** `extensions/mattermost/src/mattermost/monitor.ts:1383-1386`
  - body: By broadening this block to all session types, the PR now writes channel/thread delivery context to `route.mainSessionKey`. Cron delivery resolves the requested thread session first and falls back to main only when that thread entry has no route, so multiple Mattermost threads can overwrite the same main entry and later wakeups can post to the wrong thread or channel. Write the route for the active `sessionKey`/thread entry instead of the shared main bucket.
  - confidence: 0.9
- **[P2] Do not claim this fixes heartbeat thread routing:** `extensions/mattermost/src/mattermost/monitor.ts:1384-1386`
  - body: The new comment includes heartbeat wakeups, but heartbeat target resolution still avoids inheriting session-stored thread IDs unless an explicit turn source supplies one. Saving `threadId` here does not make a Mattermost heartbeat target the active thread, so either narrow this to the cron path or add the explicit heartbeat route change and coverage.
  - confidence: 0.86
