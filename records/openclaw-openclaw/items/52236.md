## Summary

Keep open for maintainer consolidation: current main still has the Mattermost non-direct route-persistence gap, but this draft is abandoned, conflicting, lacks real behavior proof, and has blocking route-target findings while overlapping newer canonical Mattermost/session work.

## What This Changes

The PR removes Mattermost's direct-only last-route guard and records `effectiveReplyToId` as `deliveryContext.threadId` in the Mattermost monitor.

## Best Possible Solution

Consolidate or replace this draft with a current-main fix that persists non-direct Mattermost route context on the active session, preserves heartbeat thread-safety, and coordinates with the open canonical Mattermost/session work.

## Reproduction Assessment

Yes, at source level: current main records Mattermost last-route context only for direct posts while non-direct thread turns carry the active `SessionKey`, `OriginatingTo`, and `MessageThreadId`. I did not run a live Mattermost workspace during this read-only review.

## Solution Assessment

No. Persisting route context is the right direction, but this patch writes non-direct thread context to the shared main session and does not resolve the heartbeat/current-thread cron policy called out in discussion; the safer replacement path is the active-session/core fallback tracked by https://github.com/openclaw/openclaw/pull/77442 plus the canonical issue at https://github.com/openclaw/openclaw/issues/45082.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

  - body: Removing the direct-only guard still writes the route through `route.mainSessionKey` while the active turn records a thread-scoped `SessionKey`. Cron delivery checks the requested thread session before falling back to main, so the intended thread entry can remain unroutable or inherit another thread's main-route write.
  - confidence: 0.88

  - body: The patch writes every non-DM Mattermost route, including public thread context, into the shared main session. Later main-session heartbeats or manual deliveries can then jump into the last public thread instead of preserving main-session routing safety.
  - confidence: 0.82
