## Summary

## What This Changes

The issue reports Anthropic subscription setup-token requests failing through OpenClaw embedded/local agent runtime with an overage-style billing error while direct Claude Code succeeds with the same account and token.

## Best Possible Solution

Maintainers should confirm Anthropic's current subscription setup-token contract for OpenClaw's embedded Messages API shape, then either adjust the request shape/docs or make the billing-policy limitation explicit with a predictable API-key fallback path.
