## Review Findings

Full review comments:

- **[P2] Preserve no-overwrite recovery for missing configs:** `src/commands/doctor-config-preflight.ts:63`
  - body: When `targetExists` is false, this helper now renames the temp file into `openclaw.json`. On POSIX, `rename` replaces an existing destination, so a config created after the earlier access check can be clobbered; the previous path used `COPYFILE_EXCL`. Keep the temp/chmod flow, but fail or re-check without overwriting a newly-created canonical config.
  - confidence: 0.86
- **[P3] Keep gateway hint tests off shared temp paths:** `src/cli/gateway-cli/run.option-collisions.test.ts:224-225`
  - body: The test fixture writes and deletes a fixed sibling config file in the mocked temp root in every `beforeEach`. That can delete an unrelated developer file and leak across parallel tests; use a per-test state dir or mock `existsSync` for the sibling path instead.
  - confidence: 0.78
