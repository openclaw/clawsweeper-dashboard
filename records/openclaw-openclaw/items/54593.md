## What This Changes

The PR changes `getSubagentDepth()` to count leading legacy `subagent:` tokens after agent-key scoping and adds regression cases for unscoped legacy subagent keys.

## Best Possible Solution

Land the focused helper and regression-test fix once maintainers accept the corrected legacy depth semantics and current-head validation is green or explicitly waived.

## Reproduction Assessment

Yes. Source inspection is enough to reproduce the bug: current main and `v2026.6.6` split only on `:subagent:`, so a leading `subagent:worker` token returns depth 0 while the store fallback uses that helper when metadata is missing.

## Solution Assessment

Yes. Updating the shared session-key helper is the narrowest maintainable fix because direct helper callers and session-store fallback logic both rely on it; fixing only a caller would leave inconsistent depth semantics.
