## What This Changes

The PR changes `getSubagentDepth()` to count token-boundary `subagent:` segments after stripping any `agent:<id>:` prefix and adds two raw legacy session-key unit cases.

## Best Possible Solution

Land this PR or an equivalent minimal helper fix after current required checks pass, keeping the scope to session-key depth classification and regression coverage.

Yes. The helper is pure and current main deterministically returns `0` for `subagent:worker` because it only splits on `:subagent:`; the read-only trace confirms the PR’s intended behavior without needing runtime state.

## Solution Assessment

Yes. Counting token-boundary `subagent:` segments in the agent-scoped portion is the narrowest maintainable fix because it preserves canonical nested behavior while covering the accepted legacy prefix form.
