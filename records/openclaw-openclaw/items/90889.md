## Summary

This remains a useful, focused fix: current main still lets an explicit override bypass known model limits in shared resolver and session-store fallback paths, while another current path already defines the override as a ceiling. The patch aligns those paths and has no discrete correctness defect, but the upgrade-visible session semantics should remain explicit before merge.

## What This Changes

The PR caps positive context-token overrides by a selected model’s known smaller window in shared resolution and session persistence, with focused regression tests.

## Best Possible Solution

Keep one effective-window invariant across runtime and fallback paths: treat a positive override as a ceiling when the model limit is known, preserve it when the limit is unknown, and retain focused resolver plus persisted-session coverage.

## Reproduction Assessment

Yes. Current-main source has a high-confidence reproduction: a positive override returns before model-window lookup, and session persistence writes it directly when runtime context metadata is unavailable.

## Solution Assessment

Yes. Extending the existing override-as-ceiling invariant through the shared resolver is narrower and less drift-prone than adding separate caps to each status, runner, or persistence caller.
