## Label Justifications

- `P2`: This is a normal-priority correctness fix for over-reported agent/session context windows with limited blast radius.
- `merge-risk: 🚨 compatibility`: The PR changes how existing context-token override config is interpreted when a smaller known model window exists.

## Summary

Keep open: current main still short-circuits positive context-token overrides before consulting known model windows, and this PR is a focused repair with no blocking code-review findings. The main remaining gate is maintainer acceptance of the intentional config/session-state behavior change plus normal checks.

## What This Changes

This PR caps effective context-token override resolution and session-store persistence by known model context windows, with focused regression tests.

## Best Possible Solution

Land the shared resolver/session-store cap after maintainers accept override-as-ceiling semantics and checks pass, while leaving the broader model-switch and fallback work to https://github.com/openclaw/openclaw/issues/58957 and https://github.com/openclaw/openclaw/issues/9986.

## Reproduction Assessment

Yes. Source inspection on current main shows a positive override returns before model-window lookup, and the PR adds focused tests for the inflated resolver and persisted-session cases.

## Solution Assessment

Yes. Centralizing the cap in `resolveContextTokensForModel` and routing session persistence through that resolver is the narrowest maintainable fix; a local session-store-only cap would leave sibling resolver callers inconsistent.
