## Summary

Keep open: current main and the latest release still let a positive context-token override win before known model-window resolution and session-store persistence. ## What This Changes

The branch changes the shared agent context-window resolver to cap positive overrides by known model windows and updates post-run session persistence/tests to use the capped result.

## Best Possible Solution

Land the shared resolver/session-store correction after maintainer acceptance of the bounded override semantics, and keep separate CLI status and model-switch behavior in their canonical follow-ups.

## Reproduction Assessment

Yes. Source inspection on current main shows the resolver returns a positive override before model-window lookup, and the session store persists a positive override before calling the resolver.

## Solution Assessment

Yes. The shared resolver is the narrow owner for capping overrides against actual known windows, and removing the session-store bypass makes persisted metadata use the same decision path; the standalone CLI status copy remains separate work at https://github.com/openclaw/openclaw/issues/92760.
