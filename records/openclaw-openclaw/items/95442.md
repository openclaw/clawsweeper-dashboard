- **[P1] Preserve duplicate padded keys for validation:** `src/agents/tools/cron-tool-canonicalize.ts:122-123`
  - body: This always writes the padded value to the trimmed key and deletes the raw key. Keep the padded key visible when the canonical key already exists and cover nested plus flat duplicate conflicts.
  - confidence: 0.9

  - body: The flat recovery path now trims padded known keys into a synthetic job that is already canonicalized, but the add/update branches canonicalize that object again. A first pass that preserves a padded duplicate can still be re-trimmed on the second pass after the canonical shorthand has moved into `schedule` or `payload`, so flat conflicts like `cron`/`cron ` or `allowUnsafeExternalContent`/`allowUnsafeExternalContent ` can be silently resolved. Skip the second canonicalization for recovered flat objects or otherwise keep those conflicts rejectable.
  - confidence: 0.85
