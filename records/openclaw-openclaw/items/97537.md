## Summary

Keep open: current main only partially covers standard string hook delivery, while the requested Telegram `InputRichMessage`-level transformation seam is still missing and needs maintainer API/product direction.

## What This Changes

Request a Telegram rich-message outbound hook or SDK seam so plugins can intercept rich-message delivery and transform Markdown into Telegram `InputRichMessage` payloads when `richMessages` is enabled.

## Best Possible Solution

Define a narrow typed Telegram/channel outbound seam for rich-message shaping, coordinated with the generic hook-bypass work so standard hooks run exactly once and rich payload plugins do not reach into private internals.

## Reproduction Assessment

Yes, at source level for the missing rich-payload seam: current hook result types cannot return `InputRichMessage`, and direct rich send helpers build/send rich payloads without hook dispatch. The broader claim that every rich Telegram path skips `message_sending` is not fully reproduced because `deliverReplies` already runs it.

## Solution Assessment

Unclear as filed. Reusing existing string hooks may fix part of the bypass story, but the durable solution is a maintainer-approved typed SDK/channel seam rather than an ad hoc Telegram-only hook name.
