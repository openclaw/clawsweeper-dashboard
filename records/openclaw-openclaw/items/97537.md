## Summary

Keep open: current main has generic outbound and Telegram reply hook coverage, but it does not clearly satisfy the reported Telegram Rich Message plugin-seam need, especially for raw Rich Message conversion or lower-level rich send paths.

## What This Changes

The issue requests an outbound hook seam for Telegram rich messages so plugins can transform content before the Rich Messages send path.

## Best Possible Solution

Define the intended plugin contract for Telegram rich outbound conversion, then add focused coverage so rich final delivery uses that seam without double-running generic hooks.

## Reproduction Assessment

Partly: current source shows generic delivery paths do run hooks before Telegram send, while the lower-level rich send helper itself does not. The missing part is a concrete reproduction tying the reporter's plugin path to an unhooked helper rather than the already-hooked durable/reply pipeline.

## Solution Assessment

Unclear: firing existing hooks in the wrong lower-level helper could double-run them for durable sends, while adding a Telegram-specific Rich Message hook would be new plugin API surface requiring maintainer direction.
