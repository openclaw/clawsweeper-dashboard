## Summary

Keep open: current main only partially covers standard string hook delivery, while the requested Telegram rich-payload transformation seam still does not exist and needs a maintainer API decision.

## What This Changes

Request a plugin hook/API seam so Telegram `richMessages` delivery can be intercepted or transformed, especially for Markdown-to-`InputRichMessage` conversion.

## Best Possible Solution

Define a narrow typed Telegram/channel outbound seam for rich-message shaping, coordinated with the generic exactly-once hook repair and covered by SDK docs plus focused Telegram tests.

## Reproduction Assessment

Yes, at source level for the missing rich-payload seam: current hook result types cannot return `InputRichMessage`, and direct rich send helpers build/send rich payloads without such a hook. The broader claim that every rich Telegram path skips `message_sending` is only partially supported because `deliverReplies` already runs it.

## Solution Assessment

Unclear as filed. Reusing existing string hooks may address part of the bypass story, but the maintainable solution is a typed SDK/channel seam coordinated with https://github.com/openclaw/openclaw/issues/92374 rather than an ad hoc Telegram-only hook.
