## Summary

Keep open. Current main has durable transcripts, restart-sentinel continuations, and a narrow stale-lock main-session recovery path, but it does not implement the requested durable checkpoint/resume contract for task state or mid-turn tool context.

## What This Changes

The issue requests durable session checkpointing, restart detection, user notification, and optional resume support so in-flight agent work can survive gateway restarts.

## Best Possible Solution

Define a durable, versioned resume contract across the session/runtime boundary, then implement it with explicit user confirmation and idempotency while leaving the narrower delivery-queue and orphaned-session bugs tracked separately.

## Reproduction Assessment

Not applicable as a strict bug reproduction; this is a feature request for behavior not currently promised by docs or tests. Source inspection gives high confidence that current main has only partial transcript/restart recovery, not the requested checkpoint/resume path.

## Solution Assessment

Unclear. The proposed checkpoint file and tool replay direction need maintainer design first; the safer direction is to build on existing session store, transcript, and restart-recovery seams with explicit confirmation rather than adding ad hoc automatic tool replay.
