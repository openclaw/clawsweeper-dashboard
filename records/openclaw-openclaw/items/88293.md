## Summary

Keep open: current main and the latest release still lack a native Android node URL/deeplink command, and this is a real feature request rather than a stale or duplicate report. The right path needs maintainer product/security review because it adds a new Android intent-capability plus an explicit allowlist/config surface.

## What This Changes

Request for a disabled-by-default Android node command that launches validated local URLs/deeplinks through an explicit allowlist.

## Best Possible Solution

Design a narrow native Android node command with disabled-by-default allowlisting, Android-side URL validation, gateway policy/docs/tests, and an explicit maintainer decision on where the allowlist is configured.

## Reproduction Assessment

Not applicable as a bug reproduction. Source and docs inspection do confirm the missing capability: Android declares system.notify but no URL/deeplink command, and the gateway rejects commands that are not both declared and allowlisted.

## Solution Assessment

Unclear until maintainers choose the security and ownership shape. An allowlisted ACTION_VIEW command is plausible, but the best implementation depends on whether the allowlist belongs in Android settings, gateway config, plugin policy, or a combined model.
