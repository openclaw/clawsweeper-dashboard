## What This Changes

The PR adds Claude stream-json `tool_use` parsing, an optional `onToolEvent` callback, runner forwarding to `emitAgentEvent({ stream: "tool" })`, regression tests, and a changelog entry.

## Best Possible Solution

Merge this parser-level bridge after required checks pass, preserving sanitized `start`/`update`/`result` tool-event semantics for existing consumers.

## Reproduction Assessment

Yes. Feed current main's `createCliJsonlStreamingParser` a Claude stream-json `tool_use` block with `input_json_delta` and `content_block_stop`; there is no `onToolEvent` hook or runner forwarding path, so no `stream: "tool"` event can be emitted.

## Solution Assessment

Yes. An optional parser callback wired into both Claude CLI runner paths is the narrow maintainable fix because downstream web, Android, Swift, and macOS consumers already subscribe to the generic `stream: "tool"` contract.
