## Summary

Keep this PR open: the requested Claude CLI tool-event gap is real, but the latest head still has a security-sensitive raw argument fanout and does not match the established tool-event consumer contract.

## What This Changes

The PR adds Claude stream-json `tool_use` parsing to the CLI JSONL parser, threads `onToolEvent` through one-shot and live CLI runners, emits `stream: "tool"` agent events, adds parser tests, and updates the changelog.

## Best Possible Solution

Keep the parser callback and runner wiring, but shape events like the existing sanitized tool contract: sanitized args on `start`, no raw `partialJson` fanout, and update/result reserved for sanitized tool output.

## Reproduction Assessment

Yes. A Claude stream-json fixture with `content_block_start`, `input_json_delta`, and `content_block_stop` shows current main drops tool activity, and the PR fixture reproduces the remaining raw-payload and contract-shape issues.

## Solution Assessment

No. The callback boundary is appropriate, but the latest payload shape is not the narrowest safe solution because it exposes raw argument deltas and diverges from established consumers.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Stop broadcasting raw argument deltas:** `src/agents/cli-output.ts:447-450`
  - body: The new update event forwards Claude's raw `input_json_delta.partial_json` as `data.partialJson`. Gateway non-full tool fanout only strips `result` and `partialResult`, so sensitive argument fragments can be broadcast to WS/session subscribers before the final sanitized args path runs.
  - confidence: 0.92
- **[P2] Emit sanitized args on the start event:** `src/agents/cli-output.ts:434`
  - body: The start event contains only `name` and `toolCallId`, while parsed args are emitted later on `phase: "result"`. Current Control UI, Android, and ACP consumers read args from `phase: "start"` and treat result as output/completion, so Claude CLI tool cards can miss command/path details and complete at the wrong time.
  - confidence: 0.89
