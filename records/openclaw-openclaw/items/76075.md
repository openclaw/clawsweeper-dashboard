## Summary

Keep this PR open: the callback/wiring direction is right, but the post-fixup diff still forwards raw Claude argument deltas and puts sanitized parsed args on a phase that current consumers do not read for tool details.

## What This Changes

The PR adds a CLI parser `onToolEvent` callback for Claude stream-json `tool_use` blocks, wires it through one-shot and live Claude CLI runners, adds parser tests, and updates the changelog.

## Best Possible Solution

Keep the parser callback seam, accumulate Claude argument deltas internally, and emit only the established sanitized `stream: "tool"` contract that webchat, native clients, gateway subscribers, and macOS activity labels already consume.

## Reproduction Assessment

Yes. A Claude stream-json fixture with `content_block_start`, `input_json_delta`, and `content_block_stop` reproduces the current-main gap, and source inspection shows the proposed update/result payload would still bypass current consumer fields.

## Solution Assessment

No. The boundary is maintainable, but the payload shape should avoid raw `partialJson` fanout and place sanitized argument data where existing consumers read it.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Do not stream raw argument deltas:** `src/agents/cli-output.ts:448`
  - body: This forwards Claude's raw `input_json_delta.partial_json` as `partialJson` on every tool update. Gateway redaction only strips `result` and `partialResult`, so secrets embedded in tool arguments can still reach WebSocket/native subscribers before the final sanitized args are emitted. Accumulate the JSON internally and avoid broadcasting raw argument deltas.
  - confidence: 0.9
- **[P2] Put parsed args on the consumer-visible start payload:** `src/agents/cli-output.ts:466-471`
  - body: The parsed args are emitted on `phase: "result"`, but current webchat, TUI, Android, Swift, and macOS activity consumers read args only from `phase: "start"`; macOS also treats non-start phases as completion. This leaves Claude CLI tool cards/status without the exec/read details this PR is trying to restore.
  - confidence: 0.86
