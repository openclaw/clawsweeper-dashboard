## Summary

Keep this PR open: the current-main Claude CLI tool-event gap is real, but the updated branch still broadcasts raw streamed tool argument JSON and still puts parsed args on a phase that existing consumers do not use for tool details.

## What This Changes

The PR adds a `CliToolEvent` callback to Claude CLI JSONL parsing, threads it through live and one-shot runners, forwards those events via `emitAgentEvent`, adds parser tests, and updates the changelog.

## Best Possible Solution

Keep the parser callback seam, accumulate Claude argument deltas internally, and emit only the established sanitized tool-event contract that current web, gateway, and native consumers read.

## Reproduction Assessment

Yes. A Claude stream-json fixture with `content_block_start`, `input_json_delta`, and `content_block_stop` reproduces the current-main missing event path and shows the PR's current `partialJson` payload shape.

## Solution Assessment

No. The wiring direction is maintainable, but the event payload should avoid raw argument-delta fanout and provide sanitized args on the consumer-visible start contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Do not stream raw argument deltas:** `src/agents/cli-output.ts:450`
  - body: This forwards Claude's raw `input_json_delta.partial_json` as `partialJson` on each update. Gateway non-full tool fanout strips only `result` and `partialResult`, so sensitive tool arguments can reach WebSocket/session/native subscribers before the final sanitized args event.
  - confidence: 0.9
- **[P2] Emit sanitized args on the visible start event:** `src/agents/cli-output.ts:434`
  - body: The start event creates the consumer-visible tool card without args, while the parsed args are emitted later on `phase: "result"`. Current web, Android, Swift, and macOS consumers read args from `phase: "start"`, so Claude CLI tool cards/status still miss the exec/read details this PR is meant to restore.
  - confidence: 0.86

## Security Review
