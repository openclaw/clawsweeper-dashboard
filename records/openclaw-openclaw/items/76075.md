## Summary

Keep open: the PR targets a real Claude CLI streaming gap, but the current head still introduces a security-sensitive raw argument stream and does not satisfy the existing tool-event consumer contract.

## What This Changes

The PR adds a `CliToolEvent` callback to Claude stream-json parsing, threads it through live and one-shot CLI runners, emits `stream: "tool"` events, adds parser tests, and records a changelog fix for #75846.

## Best Possible Solution

Keep the parser callback seam, accumulate Claude tool argument fragments internally, and emit only sanitized data through the established `stream: "tool"` start/update/result contract used by gateway, webchat, Android, and macOS consumers.

## Reproduction Assessment

Yes. A Claude stream-json fixture with `content_block_start`, `input_json_delta`, and `content_block_stop` proves current main has no tool-event callback, and PR-head source/tests show the remaining raw-delta and result-only-args behavior.

## Solution Assessment

No. The proposed seam is the right area, but the payload shape should avoid raw `partialJson` fanout and put sanitized argument details where existing consumers read them.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Do not emit raw tool argument deltas:** `src/agents/cli-output.ts:447-451`
  - body: This forwards Claude's raw `input_json_delta.partial_json` as `partialJson` on every tool update. Gateway fanout only strips `result` and `partialResult`, so secrets embedded in tool arguments can reach WebSocket/session subscribers before the final sanitized args event. Accumulate the JSON internally instead of broadcasting these fragments.
  - confidence: 0.9
- **[P2] Emit args through the start contract:** `src/agents/cli-output.ts:469-474`
  - body: The parsed args are emitted only on `phase: "result"`, but current webchat and Android consumers read args only from `phase: "start"`, and macOS treats non-start phases as completion. This leaves the Claude CLI tool cards/status without the exec/read details the PR is trying to restore.
  - confidence: 0.86
