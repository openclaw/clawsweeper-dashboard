## Summary

Keep this PR open: it targets a real current-main Claude CLI tool-event gap, but the current head still forwards raw argument deltas and does not meet the established tool-event consumer contract.

## What This Changes

The PR adds Claude CLI stream-json tool-use parsing, threads an `onToolEvent` callback through one-shot/live CLI runners, forwards those events as `stream: "tool"`, adds parser tests, and updates the changelog.

## Best Possible Solution

Keep the parser callback seam, but emit sanitized consumer-compatible tool events: no raw `partialJson` fanout, args available on the established start contract, and coverage across gateway plus UI/native consumers.

## Reproduction Assessment

Yes. A Claude stream-json fixture with `content_block_start`, `input_json_delta`, and `content_block_stop` reproduces the current-main drop by source inspection, and the PR's own parser fixture demonstrates the missing branch.

## Solution Assessment

No. The boundary is maintainable, but the payload shape should avoid raw argument deltas and either provide sanitized args on the established start event or update every downstream consumer deliberately.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Stop broadcasting raw argument deltas:** `src/agents/cli-output.ts:448-450`
  - body: This still forwards Claude's raw `input_json_delta.partial_json` as `data.partialJson` on every tool update. Gateway non-full tool fanout strips only `result` and `partialResult`, so secrets in tool arguments can reach WebSocket/session subscribers before any sanitized final args are emitted.
  - confidence: 0.91
- **[P2] Put sanitized args on the start event:** `src/agents/cli-output.ts:434`
  - body: The start event still has only `name` and `toolCallId`, while parsed args are emitted on `phase: "result"`. Current Control UI, Android, Swift, TUI, and ACP consumers populate tool details from `phase: "start"` and treat `result` as completion/output, so Claude CLI tool cards can still miss command/path details.
  - confidence: 0.88
