## Summary

Keep open: the latest PR head still broadcasts raw Claude argument deltas and leaves parsed args on `phase: "result"`, so it has a security-sensitive payload leak and does not satisfy the established tool-event consumer contract.

## What This Changes

The PR adds a Claude CLI stream-json tool-event callback to the CLI JSONL parser, wires it through one-shot and live runners to emit `stream: "tool"`, adds parser coverage, and updates the changelog.

## Best Possible Solution

Keep the parser callback seam, accumulate Claude argument JSON internally, and emit only the established sanitized tool-event shape or update every consumer and gateway redaction rule in the same contract change.

## Reproduction Assessment

Yes. A Claude stream-json fixture with `content_block_start`, `input_json_delta`, and `content_block_stop` reproduces the current-main missing tool-event path, and the latest diff shows the remaining raw-delta and start-args contract issues directly.

## Solution Assessment

No. The parser boundary is a good direction, but this implementation should not fan out raw argument deltas and should make sanitized args visible through the contract current consumers already read.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Do not stream raw argument deltas:** `src/agents/cli-output.ts:450`
  - body: This forwards Claude's raw `input_json_delta.partial_json` as `partialJson` on every tool update. Gateway redaction only strips `result` and `partialResult`, so sensitive tool arguments can still reach WebSocket/native subscribers before sanitized final args are emitted.
  - confidence: 0.9
- **[P2] Put tool args on the start event contract:** `src/agents/cli-output.ts:434`
  - body: The start event emitted here has no `args`, while parsed args are attached only to the later result event. Control UI, Android, Swift chat, and macOS activity labels read args from `phase: "start"`, so Claude CLI tool details remain invisible or too late for current consumers.
  - confidence: 0.87

## Security Review

Status: needs_attention

Summary: The PR still introduces raw Claude tool argument delta fanout before the sanitized final args event.

Concerns:

- **[medium] Raw argument deltas can reach subscribers:** `src/agents/cli-output.ts:450`
  - body: `partialJson` is copied directly from Claude's `input_json_delta.partial_json`, and gateway fanout does not remove that new field when verbose output is not full, so sensitive tool argument fields can be exposed to agent-event subscribers.
  - confidence: 0.9
