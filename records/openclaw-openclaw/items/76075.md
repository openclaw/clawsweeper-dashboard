## Summary

This PR addresses a real current-main gap, but the latest head is still not mergeable because it broadcasts raw Claude tool argument deltas and does not match the established downstream tool-event contract.

## What This Changes

The PR adds Claude CLI stream-json `tool_use` parsing, threads an optional tool-event callback through live and one-shot CLI runners, forwards those events as `stream: "tool"`, and adds parser tests plus a changelog entry.

## Best Possible Solution

Keep the parser callback seam, but emit sanitized consumer-compatible tool events: no public raw argument deltas, sanitized args available on the established start contract, and result reserved for real tool output or completion.

## Reproduction Assessment

Yes. A Claude stream-json fixture with `content_block_start`, `input_json_delta`, and `content_block_stop` reproduces the current-main drop by source inspection, and the PR's own fixture demonstrates the proposed but incompatible event shape.

## Solution Assessment

No. The parser and runner threading are the right boundary, but the payload shape should avoid raw `partialJson` fanout and should align with existing gateway, UI, Android, TUI, and ACP tool-event semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Do not broadcast raw argument deltas:** `src/agents/cli-output.ts:447-450`
  - body: This copies Claude's raw `input_json_delta.partial_json` into `data.partialJson` for every tool update. Gateway non-full tool fanout strips only `result` and `partialResult`, so sensitive command paths or token-like argument fields can still reach WebSocket/session subscribers before the final sanitized args are emitted.
  - confidence: 0.91
- **[P2] Keep parsed arguments on the start contract:** `src/agents/cli-output.ts:470-473`
  - body: The patch emits parsed args on `phase: "result"` instead of on the established start event. Current Control UI ignores result-time `args`, Android removes pending tool calls on `result`, and other consumers use `result` as completion/output, so Claude CLI tool cards can lose details or disappear immediately after arguments finish streaming.
  - confidence: 0.88
