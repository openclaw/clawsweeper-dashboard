## Summary

Keep open: the PR targets a real current-main gap, but the latest head still has blocking payload-contract issues. It forwards raw Claude `partialJson` tool-argument deltas through agent events and emits sanitized args only on `phase: "result"`, while existing consumers build tool details from sanitized `phase: "start"` args.

## What This Changes

The PR adds Claude stream-json `tool_use` parsing in the CLI JSONL parser, threads an optional tool-event callback through one-shot and live CLI runners, emits `stream: "tool"` agent events, adds parser tests, and updates the changelog.

## Best Possible Solution

Keep the parser callback seam, but emit the existing sanitized tool-event contract: sanitized args on `phase: "start"`, no raw `partialJson` fanout, and coverage for gateway plus UI/native/ACP consumers.

## Reproduction Assessment

Yes. A Claude stream-json fixture with `content_block_start`, `input_json_delta`, and `content_block_stop` reproduces the current-main drop by source inspection, and the PR fixture demonstrates the missing branch.

## Solution Assessment

No. The callback and runner wiring are the right boundary, but the payload shape should avoid raw streamed arguments and match the existing start/update/result consumer contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Stop broadcasting raw argument deltas:** `src/agents/cli-output.ts:447-450`
  - body: The patch forwards Claude's raw `input_json_delta.partial_json` as `data.partialJson` on every tool update. Gateway non-full tool fanout strips only `result` and `partialResult`, so sensitive argument fields can reach WS/session subscribers before any sanitized final args are emitted.
  - confidence: 0.91
- **[P2] Put sanitized args on the start event:** `src/agents/cli-output.ts:434`
  - body: The start event still has only `name` and `toolCallId`, while parsed args are emitted on `phase: "result"`. Current Control UI, Android, Swift-style tests, and ACP consumers populate tool details from `phase: "start"` and treat result as completion/output, so Claude CLI tool cards can miss command/path details.
  - confidence: 0.88
