## Summary

Keep open. The PR targets a real current-main gap, but it is not mergeable yet because it maps a completed Claude `tool_use` input block to OpenClaw’s tool-completion event, which existing clients treat as the end of execution.

## What This Changes

The branch adds Claude stream-json `tool_use` parsing, threads an optional parser callback through one-shot and live CLI runners, emits `stream: "tool"` events, adds parser tests, and updates the changelog.

## Best Possible Solution

Keep the parser callback seam and runner wiring, but translate Claude `tool_use` input into OpenClaw’s existing sanitized tool-start contract and emit completion only when an actual tool result is available.

## Reproduction Assessment

Yes. A Claude stream-json fixture with `content_block_start`, `input_json_delta`, and `content_block_stop` shows current main drops tool activity, and the PR's own fixture shows it emits `result` at input-block close rather than at tool completion.

## Solution Assessment

No. The boundary is maintainable, but the payload mapping should put sanitized args on the start event and avoid treating parsed input JSON as result output.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Do not finish tools at content_block_stop:** `src/agents/cli-output.ts:468-472`
  - body: This emits `phase: "result"` as soon as the Claude `tool_use` input JSON closes, but that payload contains sanitized args, not execution output. Existing clients treat `result` as completion: Android/Swift remove the pending tool, ACP sends a completed update from `data.result`, and Control UI finalizes on `phase === "result"`. Emit the sanitized args on the start event once parsed, and reserve `result` for an actual tool result or omit it when unavailable.
  - confidence: 0.91
- **[P3] Preserve tool names on update events:** `src/agents/cli-output.ts:447-450`
  - body: The new update event contains only `toolCallId`. Control UI defaults missing names to `"tool"` and overwrites the existing entry name on every update, so active cards can lose the real tool name while the input stream is still being parsed. Include `name` on update events or avoid emitting empty updates.
  - confidence: 0.82
