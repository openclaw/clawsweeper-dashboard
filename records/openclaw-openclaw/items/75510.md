## Summary

Keep open. The cron retry itself matches a real current-main behavior gap, but this PR is not merge-ready because the added regression test has a typed helper argument mismatch, the branch also changes unrelated subagent/cron fallback delivery behavior, and the required changelog entry is missing.

## What This Changes

The branch reschedules interrupted isolated agent-turn cron jobs on Gateway startup, documents the retry, adds restart catch-up coverage, exports the startup deferral constant, and changes direct completion fallback text for subagent/cron events.

## Best Possible Solution

Land the cron retry implementation, docs, regression test, and changelog as one focused fix; split the fallback-delivery policy into a separate agent-delivery review if maintainers want that behavior.

## Reproduction Assessment

Yes. Persist an enabled one-shot isolated `agentTurn` cron job with stale `state.runningAtMs` and start `CronService`; current main marks it interrupted and disables the `at` job, while the PR test targets the deferred retry state.

## Solution Assessment

No, not in the current combined patch. The cron retry uses the right startup deferral seam, but the helper-argument typo must be fixed and the unrelated fallback-delivery change should be removed or split.
