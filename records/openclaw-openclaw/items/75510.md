## Summary

Keep open. The cron retry fix is useful, but this branch still mixes in a separate direct completion fallback behavior change that suppresses result text, and it lacks the required user-facing changelog entry before merge.

## What This Changes

The branch reschedules interrupted isolated agent-turn cron jobs on Gateway startup, updates cron docs/tests and the startup delay export, and also changes direct subagent/cron completion fallback text selection.

## Best Possible Solution

Land the cron retry implementation, docs, regression test, and a single-line Unreleased Fixes changelog entry as one focused fix; split the direct fallback delivery policy change into a separate agent-delivery PR if maintainers want that behavior.

## Reproduction Assessment

Yes. Persist an enabled one-shot isolated agent-turn cron job with stale state.runningAtMs and start CronService; current main marks it interrupted/failed and disables the at job, while the PR's new test covers the intended deferred retry state.

## Solution Assessment

No, not as currently packaged. The cron change is the narrow maintainable solution because it reuses the existing startup deferral seam, but the fallback-delivery behavior change is separate and the changelog entry is missing.
