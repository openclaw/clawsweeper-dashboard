## Summary

Keep open. Current main already wraps memory-core query and batch embeddings in Promise.race-style timeouts, so part of the report is overstated, but source review still confirms no AbortSignal propagation into remote embedding fetches and no circuit breaker for repeated memory_search embedding failures.

## What This Changes

The issue requests AbortController/deadline propagation and circuit-breaker protection for memorySearch and remote embedding provider paths.

## Best Possible Solution

Define a memory embedding resilience contract that propagates abort/deadline state through provider fetches, adds a scoped cooldown for repeated query-time embedding failures, and keeps config, docs, and tests aligned.

## Reproduction Assessment

No high-confidence runtime reproduction was established. Source inspection does confirm the missing abort propagation and circuit-breaker pieces, while also showing current main already has 60s remote query timeouts via Promise.race.

## Solution Assessment

Unclear as written. Mirroring active-memory exactly is probably not the best direct fix because memory embeddings use a public provider SDK contract; the safer direction is an explicit abort/deadline contract plus scoped failure cooldown at the memory-core/provider boundary.
