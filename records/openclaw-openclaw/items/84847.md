## What This Changes

Managed memory dreaming cron is reported to add too much gateway pressure by sweeping many workspaces in one cron tick under high memory and event-loop load.

## Best Possible Solution

Review the linked pressure-bounding PRs, choose the intended cron pressure policy, and land one tested fix that bounds or defers cron-triggered dreaming without changing direct non-cron dreaming semantics unnecessarily.

## Reproduction Assessment

## Solution Assessment
