## Summary

Keep open. Current main and v2026.5.7 still lack the requested durable OpenResponses Excel/generic-binary input_file contract: Excel MIME types are outside the default file allowlist, unsupported files fail before storage, and accepted files are decoded only into ephemeral prompt context. The merged WebChat attachment work and the open DOCX/Telegram items are adjacent, but they do not supersede this Gateway/OpenResponses request.

## What This Changes

Support Excel spreadsheets and generic binary streams as Gateway/OpenResponses input_file uploads with explicit MIME policy, size limits, durable workspace storage, metadata receipts, and documentation.

## Best Possible Solution

Define a documented OpenResponses upload contract that fail-closes on MIME, size, and URL policy, stores accepted binaries through managed workspace media with receipt metadata, exposes stable tool-readable paths, and then adds Excel coverage and docs.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a /v1/responses base64 input_file with application/vnd.openxmlformats-officedocument.spreadsheetml.sheet is outside the default allowlist and throws before any durable persistence path.

## Solution Assessment

Unclear. The direction is reasonable, but the narrow maintainable solution is not just adding Excel MIME strings; it needs a managed storage/receipt/redaction/sandbox contract for untrusted binary files.
