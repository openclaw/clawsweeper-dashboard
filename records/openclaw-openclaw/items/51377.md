## Summary

Keep open: current main and v2026.5.20 still do not provide durable Gateway/OpenResponses Excel or generic-binary input_file uploads; the existing WebChat attachment path is adjacent but not the requested receipt-backed OpenResponses contract.

## What This Changes

Request to support Excel spreadsheets and generic binary payloads as first-class Gateway/OpenResponses input_file uploads with MIME policy, size limits, durable workspace storage, metadata, and docs.

## Best Possible Solution

Define a documented OpenResponses binary upload contract that fail-closes on MIME, size, and URL policy, stores accepted binaries through managed workspace media with receipt metadata, exposes stable tool-readable paths, and then add Excel coverage and docs.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a /v1/responses base64 input_file with an Excel MIME is outside the default allowlist and rejects before durable storage or receipt creation.

## Solution Assessment

Unclear as a concrete implementation. The direction is valid, but the maintainable fix is a managed binary storage and security contract, not only adding Excel MIME strings.
