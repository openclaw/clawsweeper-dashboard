## Summary

Keep open. Current main and the latest release still lack the requested durable OpenResponses/Gateway Excel and generic-binary input_file contract; adjacent WebChat attachment support exists, but it does not persist /v1/responses input_file uploads with receipt metadata or stable tool-readable paths.

## What This Changes

Support Excel spreadsheets and generic binary uploads through Gateway/OpenResponses input_file handling, including allowlists, limits, durable workspace storage, receipts, and documentation.

## Best Possible Solution

Define a documented OpenResponses upload contract that fail-closes on MIME, size, and URL policy, stores accepted binaries through managed workspace media with receipt metadata, exposes stable tool-readable paths, and then add Excel coverage and docs.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a /v1/responses base64 input_file with an Excel MIME is outside the default allowlist and rejects before durable storage or receipt creation.

## Solution Assessment

Unclear as a concrete implementation. The direction is valid, but the maintainable fix is a managed binary storage and security contract, not only adding Excel MIME strings.
