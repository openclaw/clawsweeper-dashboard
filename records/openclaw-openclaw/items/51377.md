## Summary

Keep open. Current main still lacks the requested durable Gateway/OpenResponses Excel and generic-binary input_file contract; the merged WebChat attachment work is adjacent but does not make /v1/responses persist Excel/binary uploads with receipt metadata or stable tool-readable paths.

## What This Changes

The issue requests Excel spreadsheet and generic binary upload support for Gateway/OpenResponses input_file, including allowlists, size limits, durable workspace storage, metadata receipts, and docs.

## Best Possible Solution

Define a documented OpenResponses upload contract that fail-closes on MIME, size, and URL policy, stores accepted binaries through managed workspace media with receipt metadata, exposes stable tool-readable paths, and then adds Excel coverage and docs.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a /v1/responses base64 input_file with application/vnd.openxmlformats-officedocument.spreadsheetml.sheet is outside the default allowlist and throws before durable storage or receipt creation.

## Solution Assessment

Unclear. The direction is reasonable, but the narrow maintainable solution is not just adding Excel MIME strings; it needs a managed storage, receipt, redaction, and sandbox contract for untrusted binary files.
