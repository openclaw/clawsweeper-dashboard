## Summary

Keep open: current Discord behavior intentionally treats human `@everyone`/`@here` as mention-gate matches, and neither current main nor the latest release exposes an option to opt out of that mass-mention behavior.

## What This Changes

Request a Discord guild/channel config toggle to stop `@everyone` and `@here` from satisfying `requireMention` while preserving current behavior by default.

## Best Possible Solution

Add an opt-in Discord guild/channel activation policy, or deliberately extend `ignoreOtherMentions`, with current defaults preserved and focused preflight/docs/schema coverage.

## Reproduction Assessment

Yes, from source and tests: Discord preflight treats `mentionedEveryone` as `wasMentioned`, and the test suite expects `@everyone` to satisfy `requireMention`. I did not run a live Discord scenario in this read-only review.

## Solution Assessment

Unclear until maintainers choose the config shape. An opt-in guild/channel setting that preserves existing defaults is safer than changing `requireMention` or `ignoreOtherMentions` behavior globally.
