Keep open: current main and v2026.6.6 still treat human Discord `@everyone` and `@here` metadata as mention-gate matches, and no `ignoreMassMentions` config surface exists. ## What This Changes

Request to add a Discord guild/channel option so `@everyone` and `@here` do not satisfy `requireMention` unless the bot is explicitly mentioned.

## Best Possible Solution

Approve a narrow opt-in Discord config design that preserves current default behavior, then update preflight logic, schema/types, docs, and focused tests together.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: current main tests and preflight code show human `@everyone` satisfies `requireMention`, while `ignoreOtherMentions` intentionally does not cover mass mentions.

## Solution Assessment

Unclear. An opt-in Discord config key is likely the safest implementation shape because it preserves current default behavior, but maintainers still need to approve the name and config/docs contract.
