## Summary

Keep open: current main and v2026.6.6 still treat human `@everyone`/`@here` as Discord mention-gate matches, and no `ignoreMassMentions` option exists. The related implementation PR is closed unmerged, so the remaining work is valid but needs a maintainer product decision on the new config/default semantics.

## What This Changes

Request adds an `ignoreMassMentions` Discord guild/channel option so `@everyone` and `@here` no longer satisfy `requireMention` when enabled.

## Best Possible Solution

Approve a narrow opt-in Discord guild/channel mass-mention policy that preserves existing defaults, or explicitly fold the behavior into the broader Discord access-control design, then update schema, types, docs, and preflight tests together.

## Reproduction Assessment

Yes, source-level. With `requireMention: true`, current source and tests show a human `@everyone` message sets `wasMentioned=true`; no live Discord replay was run in this read-only review.

## Solution Assessment

Unclear until maintainers decide the config contract. The requested opt-in is narrow and plausible, but the safest implementation should preserve the current default and update config/schema/docs/tests in one focused change.
