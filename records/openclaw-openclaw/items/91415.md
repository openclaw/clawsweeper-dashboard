## Summary

Keep open: current main and the latest release still treat human `@everyone` and `@here` as valid Discord mention-gate matches, and no `ignoreMassMentions`-style option exists. Solving this adds Discord config semantics, so it needs maintainer product approval rather than conservative cleanup closure.

## What This Changes

The issue requests a Discord guild/channel option that prevents `@everyone` and `@here` from satisfying `requireMention: true` unless the bot is explicitly mentioned.

## Best Possible Solution

Approve a narrow Discord guild/channel config design for broadcast mention handling, keep the existing default behavior for upgrades, and update schema, types, docs, and focused preflight tests together.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: current tests assert that `@everyone` satisfies `requireMention: true` and that `ignoreOtherMentions=true` does not drop `@everyone`. I did not run a live Discord replay in this read-only review.

## Solution Assessment

No, not as an automatic fix: the requested behavior requires a new Discord config key or changed semantics for an existing key. The maintainable path is a maintainer-approved name/default decision before implementation.
