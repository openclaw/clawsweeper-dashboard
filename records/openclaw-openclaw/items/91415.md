## Summary

Keep open: current main and the latest release still treat human `@everyone` and `@here` as valid Discord mention-gate matches, and there is no `ignoreMassMentions` config surface. The related implementation PR is closed unmerged, so the remaining work is a maintainer product decision about adding a new Discord config key.

## What This Changes

The issue requests a Discord `ignoreMassMentions` option so `@everyone` and `@here` do not satisfy `requireMention` unless the bot is explicitly mentioned.

## Best Possible Solution

Approve a narrow Discord guild/channel config design that preserves the existing default for upgrades, then update runtime gating, schema, types, docs, and focused preflight tests together.

## Reproduction Assessment

Yes for the current behavior, though this is a feature request rather than a failing existing contract. Source and tests show a human `@everyone` message satisfies `requireMention: true`; I did not run a live Discord repro in this read-only review.

## Solution Assessment

No for automatic implementation yet. Adding `ignoreMassMentions` is a plausible shape, but the best fix needs maintainer approval of the config name, scope, and default before code changes.
