## Summary

Keep open: current main and v2026.6.8 still treat human Discord `@everyone` and `@here` as satisfying `requireMention`, while the only directly linked implementation PR is closed unmerged. The remaining question is whether maintainers want a new Discord guild/channel config surface for opting out of that behavior.

## What This Changes

Requests a Discord guild/channel option such as `ignoreMassMentions` so `@everyone` and `@here` no longer satisfy `requireMention` unless the bot is explicitly mentioned.

## Best Possible Solution

Approve a narrow Discord config contract that preserves the current default, then update Discord preflight, strict schema/types, metadata/docs, and focused regression tests together.

## Reproduction Assessment

Yes at source level: current Discord preflight and tests show a human `@everyone` message survives `requireMention` gating with `wasMentioned=true`. I did not run a live Discord replay in this read-only review.

## Solution Assessment

Unclear as a finalized product path: `ignoreMassMentions` is plausible, but the best fix needs maintainer approval of the config name, inheritance scope, and default before implementation.
