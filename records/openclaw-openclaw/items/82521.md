## Summary

Keep open: current main still has a source-level path where explicit OpenAI PI runs resolve the `openai` auth order before the Codex profile can be forwarded, and the OpenAI docs still advertise a `--device-code` flag that the CLI does not register.

## What This Changes

The issue asks OpenClaw to align documented OpenAI/Codex auth routing so `openai/*` with `auth.order.openai` can select `openai-codex:*` before API-key fallback, and to fix the advertised device-code login command.

## Best Possible Solution

Align the OpenAI provider runtime/auth plan with the documented routing and correct the device-code command surface or docs using the existing provider auth method.
