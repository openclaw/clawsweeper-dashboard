## Summary

Keep open. Current main has partially reduced the blast radius by bounding chat-history and sessions-list tail reads, but the central full-transcript reader still materializes large JSONL files and several gateway paths still call it before applying limits. The direct fix PR was closed unmerged, so there is no complete main-branch or shipped fix to close against.

## What This Changes

The issue asks OpenClaw to prevent oversized session JSONL transcripts from hanging the gateway by bounding transcript reads and adding diagnostic or maintenance guardrails.

## Best Possible Solution

Finish the gateway transcript-reader work by replacing remaining full-file transcript materialization with bounded, decoder-safe line or entry iteration for history, sequence counting, usage, artifact, reset, and image lookup paths. Treat doctor warnings, file rotation, and maintenance caps as follow-up guardrails after the raw reader can no longer block the event loop.

## Reproduction Assessment

Yes. The issue includes concrete `sample`, `lsof`, file-size, and recovery evidence, and current main still has a static reproduction path: create or retain a very large session JSONL and hit a gateway surface such as `sessions.history`, HTTP history, session event broadcasting, or `sessions.send` that calls `readSessionMessages`.

## Solution Assessment

Yes, the narrow reader fix is the best first solution. The current bounded-tail helpers show the right direction, but the maintainable fix is to remove the remaining shared full-read paths before adding broader rotation, doctor, or health-probe policy.
