## Summary

Current main has removed the gateway path that loaded large session transcripts into one string and synchronously split/replaced it; the async transcript index and bounded hot-history reads landed after the prior keep-open review and shipped in v2026.5.7.

## What This Changes

The issue asks OpenClaw to stop large session JSONL transcripts from hanging the gateway when session history or agent processing reads them.

## Best Possible Solution

Keep the shipped async/bounded transcript-reader implementation as the fix for the gateway hang; broader rotation, doctor warnings, or session-size guardrails can be handled as separate follow-up hardening if needed.

## Reproduction Assessment

Yes. The original report has strong `sample`, `lsof`, file-size, and recovery evidence, and the pre-fix source path matched a full JSONL string read/split; current main no longer has that gateway async reader path.

## Solution Assessment

Yes. Replacing full-string transcript reads with async chunked/indexed reading and bounding hot history/list paths is the narrow maintainable fix before optional rotation or monitoring policy.
