## Summary

Current main already fixes the central Feishu streaming-card truncation bug by sending full accumulated CardKit text instead of suffix deltas, with regression tests; the fix is main-only after v2026.6.1, so release users can still observe it until the next release.

## What This Changes

The issue reports that Feishu streaming interactive cards truncate longer responses when streaming is enabled by default.

## Best Possible Solution

Keep the current main full-content CardKit update behavior and let the next release carry it; any residual Feishu client caching problem should be reopened only with proof against a build containing `9fb8d87f91`.

## Reproduction Assessment

Yes at source/release level: the v2026.6.1-era path sent suffix content to the Feishu CardKit streaming endpoint, while current main now sends full merged text. I did not run a live Feishu client repro in this read-only review.

## Solution Assessment

Yes. The fix is in the Feishu plugin owner boundary, matches the Feishu CardKit full-text update contract, and includes focused regression tests for the truncation failure mode.
