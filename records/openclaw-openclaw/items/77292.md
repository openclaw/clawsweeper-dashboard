## Summary

Keep open: current main and latest release still preserve shared-main Telegram DM behavior unless `session.dmScope` is set to an isolation mode, and the merged single-user subagent announce fix does not prove the reported cross-user same-bot Telegram leak is resolved.

## What This Changes

The issue requests Telegram DM delivery and transcript isolation per `chat_id`/`sender_id` for subagent completions and parent replies when one bot serves multiple users.

## Best Possible Solution

Keep this open for maintainer security/product review: either require or migrate secure DM isolation for multi-user Telegram bots, or explicitly preserve shared-main behavior with stronger warnings and live proof of safe delivery semantics.

## Reproduction Assessment

No, not as a live current-main Telegram reproduction. The source path is clear enough to show that default-account Telegram DMs can still share `agent:main:main` under `dmScope="main"`, which is the isolation boundary the report challenges.

## Solution Assessment

Unclear; another narrow subagent routing patch is not obviously the best fix. Current main already has an isolation knob, onboarding/docs, and audit warnings, but preserving shared-main behavior for existing configs may still leave the reported cross-user risk.
