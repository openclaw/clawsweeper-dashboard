## Summary

Keep open: this is a plausible high-impact Telegram/session isolation report, but current evidence splits the issue between documented `dmScope` configuration, a current-main-only subagent ownership fix, and the still-intentional shipped default where DMs share the main session.

## What This Changes

The issue reports Telegram bot DM delivery and session context leaking between different users sharing one main agent, especially around subagent completions and cached `deliveryContext`.

## Best Possible Solution

Decide whether existing secure-DM configuration and warnings are sufficient, or add a compatibility-aware migration/default policy plus regression coverage for multi-user Telegram DMs and subagent completions.

## Reproduction Assessment

Partly. Source and docs show the shared-session premise when `session.dmScope` is `main`, but I did not establish a live current-main Telegram reproduction for the exact async subagent cross-user misdelivery after the current-main subagent ownership change.

## Solution Assessment

Unclear. The supported narrow mitigation is `session.dmScope: "per-channel-peer"`, while a code/default change for existing shared-main DMs would be compatibility-sensitive and needs maintainer policy judgment.
