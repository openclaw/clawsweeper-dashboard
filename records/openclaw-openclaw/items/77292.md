## Summary

Keep this open: current main documents and implements shared-main DM mode, but the report describes a possible cross-user Telegram disclosure that needs maintainer security/config triage before cleanup.

## What This Changes

The issue reports Telegram bot DM delivery context leaking between unrelated users when subagent or async results complete after another user messages the same bot.

## Best Possible Solution

Keep documented secure DM isolation as the supported multi-user mode, and if leakage reproduces with per-channel-peer or per-account-channel-peer, patch Telegram requester-origin/last-route preservation with focused subagent and async delivery regressions.

## Reproduction Assessment

Unclear as a current-main bug. Source inspection shows the shared-main DM route and last-route overwrite path, but not a high-confidence reproduction of leakage when secure DM isolation is enabled.

## Solution Assessment

Unclear. Scoping only deliveryContext by chat could conflict with documented shared-main DM semantics; the narrow maintainable path is to confirm the reporter's dmScope and add a secure-mode regression if that invariant fails.
