## Summary

Keeping this open because the report describes a plausible privacy boundary issue, but current main also documents shared-main DM behavior as the default and the issue does not state whether secure DM mode was enabled.

## What This Changes

The issue reports cross-user Telegram DM leakage from shared `deliveryContext` during subagent or delayed parent delivery when one bot serves multiple DM users.

## Best Possible Solution

First confirm the reporter's session configuration; if they are in shared-main mode, route them to secure DM mode, and if leakage reproduces under `per-channel-peer` or `per-account-channel-peer`, add a focused Telegram plus subagent announce regression that preserves the requester chat id through completion delivery.

## Reproduction Assessment

Unclear. Source inspection gives a high-confidence path for documented shared-main DM context sharing, but not for an unintended cross-DM leak when secure DM isolation is enabled.

## Solution Assessment

Unclear. Scoping `deliveryContext` by Telegram chat alone would be a partial and possibly incompatible change to the documented shared-main model; the maintainable fix depends on whether secure DM mode actually leaks.
