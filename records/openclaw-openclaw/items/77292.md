Keep open because the report is privacy-sensitive and current evidence separates into two cases: default shared-DM mode is documented behavior, but a secure-DM subagent delivery leak has not been ruled out.

## What This Changes

The issue requests Telegram subagent and parent delivery context isolation per chat/sender so async completions cannot cross DMs on a shared bot.

## Best Possible Solution

Confirm the reporter's effective session configuration, route bindings, and redacted traces; if secure DM mode leaks, fix requester-origin preservation in Telegram/subagent announce delivery, otherwise document or improve guidance for multi-user bots in shared-main mode.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not for the exact secure-DM regression; source inspection does reproduce the documented shared-main DM context sharing path when dmScope remains main.

## Solution Assessment

Is this the best way to solve the issue? Unclear; scoping only deliveryContext by chat would be incomplete if the transcript session remains agent:main:main, while changing the default DM scope is a product/security decision.
