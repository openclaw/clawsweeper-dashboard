## Summary

Keep open: current main documents and implements shared-main DM mode with warnings and secure `dmScope` alternatives, but this report describes a privacy-sensitive Telegram/subagent cross-user leak and is not safe to close as expected behavior or already fixed without the reporter's effective config and redacted routing proof.

## What This Changes

The issue asks OpenClaw to keep Telegram delivery context and subagent completion routing isolated per DM sender when one bot serves multiple users.

## Best Possible Solution

Confirm the reporter's effective Telegram session configuration and redacted routing traces; route shared-main setups to secure DM mode, and if peer-scoped DM mode still leaks, add a focused Telegram plus subagent announce regression that pins async delivery to the originating peer-scoped requester session.

## Reproduction Assessment

No high-confidence current-main bug reproduction was established. Source inspection shows a clear shared-main context-sharing path, but the report omits `dmScope` and I did not prove cross-DM subagent leakage with secure DM isolation enabled.

## Solution Assessment

Unclear. Scoping only `deliveryContext` by chat while leaving transcript context in `agent:main:main` would be partial; the maintainable path is secure DM configuration unless a focused regression proves requester-origin preservation fails in peer-scoped mode.
