## Summary

Keeping this open because the report describes a plausible cross-user privacy boundary, but current main also documents and tests shared-main DM behavior; the missing `session.dmScope` detail prevents a safe cleanup close.

## What This Changes

The issue reports Telegram DM delivery-context leakage between users when subagent or async completions resolve after another user contacts the same bot.

## Best Possible Solution

Confirm the reporter's DM-scope configuration first; if secure DM mode reproduces the leak, preserve requester-origin delivery through Telegram subagent announcements and add a focused multi-user regression test.

## Reproduction Assessment

Unclear. Source inspection proves default Telegram DMs can share `agent:main:main` by design and that `per-channel-peer` should isolate sender ids, but the report does not state the active dmScope or provide redacted trajectories for the secure-mode path.

## Solution Assessment

Unclear. Scoping `deliveryContext` by chat id in all modes is not clearly the best fix because it conflicts with documented shared-DM continuity; the safer path is to verify secure-DM behavior and patch requester-origin preservation only if that invariant fails.
