## Summary

Keep open: the report describes a possible cross-user disclosure, and current `main` shows both a documented shared-main DM mode and an isolated DM mode. The issue does not yet prove which configuration was active or whether leakage still reproduces under `session.dmScope` isolation, so this needs maintainer security/config triage rather than cleanup closure.

## What This Changes

Telegram DM delivery and subagent/parent follow-up context should be isolated per chat/user instead of using a shared main-agent delivery context.

## Best Possible Solution

Confirm the reporter's DM session configuration first; if they were in shared-main mode, route them to secure DM mode, and if leakage reproduces with peer-scoped DM sessions, add a focused Telegram plus subagent announce regression and patch requester-origin preservation.

## Reproduction Assessment

Unclear. Source inspection reproduces the documented shared-main DM context path, but there is not yet a high-confidence current-main reproduction of cross-DM leakage when `per-channel-peer` or `per-account-channel-peer` is enabled.

## Solution Assessment

Unclear. Scoping only `deliveryContext` by chat would be partial if the transcript is still shared; the maintainable path is to use existing DM-scoped sessions and patch announce delivery only if that invariant fails.
