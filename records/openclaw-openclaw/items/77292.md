## Summary

Keep open: current main and the latest release still expose default-account Telegram DMs sharing `agent:main:main` under `dmScope: "main"`, and the adjacent subagent-announce fixes do not prove the reported cross-user same-bot Telegram leak is resolved.

## What This Changes

The issue asks OpenClaw to isolate Telegram DM delivery context so subagent completions and parent replies from one user cannot surface in another user's DM on the same bot.

## Best Possible Solution

Keep open for maintainer security/product review: either require or migrate per-user DM isolation for multi-user Telegram bots, or explicitly preserve shared-main compatibility with live proof that delayed subagent delivery cannot cross users.

## Reproduction Assessment

No live Telegram run was established, but source inspection shows default-account Telegram DMs still share the main session under `dmScope: "main"`, and docs/audit identify that mode as a cross-user leakage risk.

## Solution Assessment

Unclear: a narrow subagent routing patch alone is not obviously the best fix because maintainers need to decide whether multi-user Telegram bots should be migrated to isolated DM scopes or keep shared-main compatibility with stronger proof and warnings.
