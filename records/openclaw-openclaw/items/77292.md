## What This Changes

The issue asks OpenClaw to prevent Telegram subagent and parent delivery context from crossing users when one bot serves multiple direct-message chats.

## Best Possible Solution

Confirm the reporter's effective Telegram session configuration and redacted routing traces, then either route shared-main deployments to documented DM isolation or add a focused regression proving peer-scoped subagent completion remains pinned to the originating requester session.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection shows a reproducible shared-main context-sharing path, but the issue does not prove cross-DM leakage when session.dmScope is per-channel-peer or per-account-channel-peer.

## Solution Assessment

Unclear. Scoping only deliveryContext by chat could be partial if the transcript/session state remains shared; the maintainable path is to verify the configured DM scope and pin subagent announce delivery to the recorded requester session when isolated DM mode is in use.
