## Summary

Current main has a working Telegram download-to-transcription-to-context path and partial operator diagnostics, but it does not provide the requested end-to-end handoff record. The remaining work is useful, while its shared schema, privacy rules, and model-visible trust behavior require maintainer product judgment.

## What This Changes

Add auditable telemetry connecting Telegram voice receipt and download, STT execution and transcript trust, and final insertion into the correct agent turn and session context.

## Best Possible Solution

Extend the shared media-understanding outcome with privacy-safe timing, transcript-state, and context-correlation fields; keep Telegram limited to transport facts and expose only an explicitly approved concise trust or degradation notice to the model.

## Reproduction Assessment

No, there is no high-confidence failed-message reproduction because the report describes an observability gap rather than one concrete malfunction. Source inspection does confirm that current transport, media-decision, and context types lack the complete requested correlation record.

## Solution Assessment

No, Telegram-specific telemetry is not the best implementation boundary. The narrow maintainable solution is to extend the existing shared media-understanding decision and context path, with sensitive transport details kept out of model context.
