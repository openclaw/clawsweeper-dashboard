## Summary

This remains a valid feature request: the latest release and current `main` expose partial transcription trust and failure diagnostics, but they do not correlate Telegram download, STT execution, and actual agent-context insertion. Maintainer design input is needed because the proposed raw telemetry conflicts with existing privacy-conscious logging boundaries and would make transcript failure states agent-visible.

## What This Changes

Add correlated Telegram voice-note receipt, STT execution, transcript trust, and agent-context insertion telemetry across the existing media pipeline.

## Best Possible Solution

Define a privacy-safe typed lifecycle with correlation tokens and closed outcome codes, enrich it at Telegram download and shared STT boundaries, and expose bounded support diagnostics plus explicit context-application state without logging transcript text or raw sensitive identifiers.

## Reproduction Assessment

No; the issue provides acceptance criteria but no concrete failing voice note, configuration, logs, or current-main reproduction, and it primarily requests new auditability.

## Solution Assessment

No as written; end-to-end correlation is useful, but emitting every proposed raw field conflicts with existing privacy-conscious logging and crosses multiple owner boundaries, so a typed redacted lifecycle is the safer design.
