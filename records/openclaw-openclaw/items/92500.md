## Summary

Keep this issue open: current main and the latest release have a Telegram voice-to-STT-to-context path, but they do not implement the requested structured handoff telemetry, and the linked implementation candidate at https://github.com/openclaw/openclaw/pull/92502 is still open with unresolved product/security review concerns.

## What This Changes

The issue requests structured Telegram voice-note, STT, and context-insertion telemetry so support tooling and the agent can distinguish received, downloaded, transcribed, trusted, truncated, missing, or untrusted audio transcripts.

## Best Possible Solution

Adopt a generic, privacy-safe media-processing outcome carried from media receipt through transcription and context construction, with detailed diagnostics kept operator-side and only compact transcript status/trust state exposed to agent context.
