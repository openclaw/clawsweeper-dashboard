## Summary

Keep this issue open. Current main partially handles Telegram and Signal through an opt-in internal ingest event, but normal plugins still lack the requested public observation contract, and the paired implementation at https://github.com/openclaw/openclaw/pull/83101 remains open and conflicting.

## What This Changes

Requests a passive, read-only plugin event for inbound group messages dropped before mention gating so archive plugins can retain full history without triggering sessions or replies.

## Best Possible Solution

Define an operator-consented, read-only observation contract that runs after transport authenticity and access-control enforcement but before activation or mention gating, then implement consistent channel-owned emission and documented skip metadata across supported transports.

## Reproduction Assessment

Yes at source level. Telegram's mention-skip branch emits only an opt-in internal event and returns before the later public plugin hook path; the reporter also supplied a concrete stable-deployment observation, although this read-only review did not send a live transport message.

## Solution Assessment

Unclear pending maintainer policy. A distinct observational hook is narrower and safer than moving `message_received`, but maintainers must decide consent, access-policy ordering, cross-channel scope, and whether the existing `ingest` setting should gate public delivery.
