## Summary

Keep open: current main still lacks a public early raw-inbound Plugin SDK hook, and representative requireMention skip paths still return before public message hooks can observe skipped group messages. The related Telegram/Signal ingest work is narrower than the archive-plugin use case, and the reporter’s later stable deployment update confirms ordinary non-mentioned Telegram group messages are still absent from the archive plugin without a local pre-return patch.

## What This Changes

The issue requests a passive, read-only early inbound/skipped-message hook so archive plugins can observe group messages dropped by requireMention without creating sessions or replies.

## Best Possible Solution

Define an explicit opt-in Plugin SDK early inbound observation contract with read-only payloads, privacy documentation, and per-channel wiring that preserves existing allowlist, sender-policy, requireMention, and reply-routing decisions.

## Reproduction Assessment

Yes, as a source-level reproduction: current main has no public early raw inbound hook, and representative requireMention skip branches return before public message_received dispatch. The reporter also provided a stable v2026.5.7 deployment result showing the linked archive plugin still misses ordinary non-mentioned Telegram group messages.

## Solution Assessment

Unclear for the exact hook name and payload, but the requested direction is valid. The best maintainable solution is a new opt-in, read-only observation contract rather than making skipped messages flow through existing reply-oriented hooks by default.
