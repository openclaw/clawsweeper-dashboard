## Summary

Keep open: current main still lacks a public early raw-inbound Plugin SDK hook, and representative `requireMention` skip paths still return before public `message_received` hooks can observe skipped group messages. The existing archive plugin and the reporter's validated branch make the need concrete, but the solution requires Plugin SDK, privacy, and multi-channel policy review before implementation.

## What This Changes

Request a public, opt-in, read-only early inbound hook so plugins can observe `requireMention`-skipped group messages without creating sessions, replies, or routing overrides.

## Best Possible Solution

Define an explicit opt-in Plugin SDK early inbound observation contract with read-only minimized payloads, privacy documentation, and per-channel wiring that preserves allowlists, sender policy, mention gating, and reply routing.

## Reproduction Assessment

Yes, as a source-level reproduction: current main has no public early raw-inbound hook, and representative `requireMention` skip branches return before public `message_received` dispatch. I did not run a live Telegram/Slack/Discord reproduction in this read-only review.

## Solution Assessment

Yes on the general direction: a separate read-only observation hook is more maintainable than moving `message_received` earlier and changing its dispatch-stage meaning. The exact hook contract still needs maintainer API, privacy, and channel-owner review.
