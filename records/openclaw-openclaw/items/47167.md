## Summary

Keep open: current main and the latest release still lack a public early inbound/skipped-message plugin hook, the source-level lifecycle gap is real, and https://github.com/openclaw/openclaw/pull/83101 is the active implementation and privacy/API decision path.

## What This Changes

Request a public read-only early inbound hook so archive plugins can observe requireMention-skipped group messages without creating sessions or replies.

## Best Possible Solution

Review and either land, gate, or explicitly narrow https://github.com/openclaw/openclaw/pull/83101 as the public read-only skipped-message hook, with the chosen privacy boundary and channel rollout documented.

## Reproduction Assessment

Yes at source level. Current main returns null from the Telegram requireMention skip path before dispatch-from-config can emit message_received; I did not run a live group send in this read-only review.

## Solution Assessment

Unclear until maintainers choose the plugin API and privacy contract. A dedicated read-only hook is narrower than moving message_received earlier, but https://github.com/openclaw/openclaw/pull/83101 still needs explicit security and rollout review.
