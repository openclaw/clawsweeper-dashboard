## Summary

Keep open. Current main has a partial Telegram/Signal internal ingest path from https://github.com/openclaw/openclaw/pull/60018, but it still lacks a public cross-channel Plugin SDK hook for early passive inbound observation, and representative Discord, Slack, LINE, and iMessage mention-gated paths still return before downstream `message_received` dispatch. The related raw-inbound request at https://github.com/openclaw/openclaw/issues/39232 was closed as superseded by this canonical thread, not as implemented.

## What This Changes

The issue asks for skipped group messages behind `requireMention` to remain visible to passive archive plugins through a read-only early inbound hook or equivalent documented hook ordering.

## Best Possible Solution

Design and document a channel-neutral, read-only, operator-opt-in early inbound Plugin SDK contract with payload privacy semantics, then wire affected channel plugins to emit it before mention gating without creating sessions or replies.

## Reproduction Assessment

Yes, source-reproducible. Configure an affected group channel with `requireMention: true` and send a non-mention group message; representative current-main paths return before downstream `message_received`, and Telegram’s partial `ingest` path is internal-only and opt-in.

## Solution Assessment

Unclear pending maintainer API/privacy design. The requested early read-only hook is plausible, but the current partial Telegram/Signal internal ingest path is not the full plugin-facing cross-channel solution, and channel-specific patches are not the best long-term shape.
