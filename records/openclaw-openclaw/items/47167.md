## Label Justifications

- `P2`: The issue affects plugin visibility for skipped channel messages but does not report a core outage.
- `impact:message-loss`: Skipped group messages are intentionally not delivered to plugin message hooks, preventing passive archive plugins from recording them.
- `impact:session-state`: The requested archive visibility affects what historical conversation state plugins can preserve for later retrieval.
- `impact:security`: An early passive inbound hook would expose pre-gating message content to plugins, so access-control and data exposure boundaries matter.

## Summary

Keep this open: current main and v2026.6.8 still lack the requested early passive plugin hook, and the shipped `ingest` path is only a partial internal-hook workaround. The same-author implementation PR remains open and conflicting, so the remaining action is maintainer review of the plugin API and security boundary.

## What This Changes

The issue requests a supported passive early inbound message hook so plugins can observe group messages skipped by `requireMention` without creating sessions or replies.

## Best Possible Solution

Keep the archive behavior plugin-owned, then land or replace https://github.com/openclaw/openclaw/pull/83101 with a narrow read-only hook after maintainer approval of the plugin API and privacy boundary.

## Reproduction Assessment

Yes at source level: the Telegram `requireMention` skip path can return before the shared public `message_received` hook is emitted downstream. I did not run a live channel repro in this read-only review.

## Solution Assessment

Yes as a direction, pending maintainer approval: a separate read-only observation hook is narrower than moving `message_received` earlier because it preserves existing hook semantics. The final public API and privacy gate remain maintainer-owned decisions.
