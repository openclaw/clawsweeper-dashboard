## Label Justifications

- `P2`: This is a normal-priority Plugin SDK and channel visibility gap with clear archive-plugin impact but no core outage.
- `impact:message-loss`: Mention-skipped group messages are intentionally not delivered to public plugin hooks, preventing passive archive plugins from recording them.
- `impact:session-state`: The missing hook affects what historical conversation context plugins can preserve for later retrieval without entering an agent session.
- `impact:security`: The requested hook would expose pre-gating conversation content to plugins, so privacy and access-control boundaries are central to the issue.

## Summary

Keep this open: current main and latest release still lack a public early read-only inbound hook, the shipped silent-ingest work is only a partial internal-hook workaround, and the linked implementation PR remains open and conflicting. The public `@dashhuang/conversation-archive` plugin makes the use case concrete, but the remaining decision is a maintainer-owned Plugin SDK and security-boundary choice.

## What This Changes

Request a read-only early inbound plugin hook so archive/audit plugins can observe group messages skipped by `requireMention` without creating sessions or replies.

## Best Possible Solution

Keep this as the canonical issue, then review or replace PR 83101 with a rebased implementation that uses an explicit read-only Plugin SDK contract and a maintainer-approved conversation-access/privacy gate.

## Reproduction Assessment

Yes, at source level: current main's Telegram `requireMention` skip path returns `null` before downstream public `message_received` hook dispatch. I did not run a live Telegram reproduction in this read-only review.

## Solution Assessment

Unclear until maintainer approval: a separate read-only observation hook is cleaner than moving `message_received` earlier, but the public API, security gate, and multi-channel rollout still need product and security decisions.
