## Summary

Keep open. Current main still lacks a supported cross-channel Plugin SDK event for passively observing group messages before `requireMention` drops them; Telegram has a partial internal `ingest` path, but Discord, Slack, LINE, iMessage, and BlueBubbles still return before the normal Plugin SDK `message_received` dispatch path. The reporter’s public archive plugin shows this is a concrete plugin-API gap rather than a core archive feature request.

## What This Changes

Request a read-only early inbound Plugin SDK hook/event so archive plugins can observe group messages skipped by `requireMention` without starting sessions or sending replies.

## Best Possible Solution

Design a documented Plugin SDK early inbound observation seam that is read-only, opt-in, fires before mention gating for supported channels, does not create sessions or replies by default, carries stable channel/account/conversation/thread/sender/message metadata, and adds regression coverage across the affected bundled plugins.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: configure a group-capable channel with `requireMention: true`, send a non-mention group message, and the affected ingress code returns before `dispatchReplyFromConfig` can emit Plugin SDK `message_received`. I did not run live channel tests because this review is read-only, but the current source path is direct.

## Solution Assessment

Unclear pending maintainer design. The requested read-only early hook is the right direction, but the best implementation needs a deliberate public SDK contract and privacy model rather than channel-local patches or reusing `message_received` after dispatch context creation.
