## Summary

Keep open. Current main still lacks a supported public Plugin SDK hook for passively observing inbound group messages before `requireMention` drops them, while representative Telegram, Discord, Slack, LINE, iMessage, and BlueBubbles paths still return before central `message_received` dispatch.

## What This Changes

This issue requests a read-only early inbound hook/event so archive plugins can see group messages skipped by `requireMention` without creating sessions or replies.

## Best Possible Solution

Design and document a channel-neutral, read-only, opt-in early inbound Plugin SDK seam that fires before mention gating and has regression coverage for the affected bundled plugins.

## Reproduction Assessment

Yes, source-reproducible. Configure an affected group channel with `requireMention: true`, send a non-mention group message, and the current channel ingress code returns before `dispatchReplyFromConfig` can emit Plugin SDK `message_received`.

## Solution Assessment

Unclear pending maintainer design. The requested early read-only hook is plausible, but the maintainable fix should be a deliberate Plugin SDK contract with privacy semantics rather than per-channel source patches.
