## Label Justifications

- `P2`: This is a focused plugin API/channel observability gap with real plugin impact, but it needs product/security review rather than emergency handling.
- `impact:message-loss`: Inbound group messages skipped by mention gating are not delivered to supported plugin archive hooks.
- `impact:session-state`: The requested archive/history plugin stores session-adjacent conversation chronology that currently cannot see mention-skipped messages.
- `impact:security`: The requested hook changes which group-message content installed plugins can observe before mention gating.

## Summary

Keep open: current main and latest v2026.6.6 still lack the requested public early inbound observation hook, while the linked implementation remains open and needs maintainer API/privacy review plus conflict refresh.

## What This Changes

The issue requests a passive read-only early inbound hook so archive plugins can observe group messages skipped by mention gating without creating sessions or replies.

## Best Possible Solution

Review and refresh https://github.com/openclaw/openclaw/pull/83101 or a replacement that keeps message_received semantics unchanged while adding a narrow read-only observed hook only after explicit maintainer approval of skipped-message plugin visibility.

## Reproduction Assessment

Yes. Source inspection shows Telegram mention-skipped group messages return from the channel path before the public message_received hook in dispatch runs; I did not run a live Telegram group test in this read-only review.

## Solution Assessment

Unclear until maintainers approve the contract. A separate read-only observed hook is cleaner than moving message_received earlier, but exposing skipped group content to public plugins is a product/security decision.
