## Summary

Keep open: current main still lacks a public early inbound Plugin SDK hook, source inspection still shows skipped requireMention messages returning before public message hooks, and an open contributor PR now references this issue with closing syntax, so the issue should stay open until that PR or a successor lands.

## What This Changes

The issue requests an opt-in, read-only early inbound hook so archive plugins can observe group messages skipped by requireMention without creating sessions or replies.

## Best Possible Solution

Define a privacy-reviewed, opt-in, read-only early inbound Plugin SDK contract with documented payload minimization and then close this issue after https://github.com/openclaw/openclaw/pull/83101 or a successor merges.

## Reproduction Assessment

Yes, as a source-level reproduction: current main lacks the public early inbound hook, and representative Telegram/Discord requireMention skip paths return before public message_received dispatch.

## Solution Assessment

Unclear for the exact API shape; the separate observational hook direction is safer than moving message_received earlier, but it needs maintainer Plugin SDK, privacy, and multi-channel scope review.
