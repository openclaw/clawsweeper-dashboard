## Summary

Keep open: current main and the latest shipped tag still lack a public early inbound plugin hook, representative requireMention skip paths still return before public message hooks, and the linked open PR is the active implementation path rather than a reason to close this issue.

## What This Changes

This issue requests a passive, read-only early inbound hook so archive plugins can observe group messages skipped by requireMention without creating sessions or replies.

## Best Possible Solution

Land a privacy-reviewed, opt-in, read-only early inbound Plugin SDK contract with documented payload minimization and close this after https://github.com/openclaw/openclaw/pull/83101 or a successor merges.

Yes, as a source-level reproduction: current main lacks the public early inbound hook, and representative Telegram and Discord requireMention skip paths return before public message_received dispatch. I did not run a live transport reproduction in this read-only review.

## Solution Assessment

Unclear for the exact API shape. A separate read-only observed hook is narrower than moving message_received earlier, but the Plugin SDK contract, privacy boundary, and multi-channel scope need maintainer approval.
