## Label Justifications

- `P2`: This is a source-reproducible Plugin SDK and channel-hook improvement with limited blast radius but real product and security review needs.
- `impact:message-loss`: The issue is about inbound group messages being suppressed before public plugin message hooks can observe or archive them.
- `impact:session-state`: The requested archive/history layer is intended to preserve raw conversation context for later retrieval outside live agent sessions.
- `impact:security`: An earlier hook would expose pre-filter group content and identifiers to installed plugins before existing routing and mention gates.

## Summary

Keep open: current main and the latest shipped tag still lack a public early inbound plugin hook for mention-skipped group traffic, and the same author’s open implementation PR at https://github.com/openclaw/openclaw/pull/83101 is the active path rather than a reason to close this issue.

## What This Changes

The issue requests a passive, read-only early inbound plugin hook so archive/history plugins can observe group messages skipped by `requireMention` without creating sessions or replies.

## Best Possible Solution

Land a privacy-reviewed, opt-in, read-only early inbound Plugin SDK contract through https://github.com/openclaw/openclaw/pull/83101 or a successor, then close this issue after the public hook and intended channel behavior are merged.

## Reproduction Assessment

Yes, as a source-level reproduction: current main lacks the public early inbound hook, and representative Telegram, Slack, LINE, and iMessage `requireMention` skip paths return before the downstream public `message_received` plugin dispatch. I did not run a live transport reproduction in this read-only review.

## Solution Assessment

Unclear for the exact API and rollout shape. A separate read-only observed hook is narrower than moving `message_received` earlier, but the Plugin SDK contract, privacy boundary, and channel coverage need maintainer approval.
