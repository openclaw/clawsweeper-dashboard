## Summary

Keep open: the external archive plugin use case is real, and current main still lacks a supported early plugin hook for messages dropped by mention gating. The merged silent-ingest work only partially covers internal Telegram/Signal hooks, while the requested cross-channel Plugin SDK contract remains a product and privacy design item.

## What This Changes

Request an early read-only inbound plugin hook or equivalent guarantee so `requireMention`-skipped group messages can be passively archived without entering agent sessions.

## Best Possible Solution

Design a channel-neutral, explicit-opt-in Plugin SDK observation hook for pre-filter inbound messages, with read-only minimized payloads and clear privacy/retention documentation.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current main has no public early raw-inbound hook, and representative channel paths return before the downstream plugin `message_received` hook on no-mention group messages. I did not run a live channel E2E in this read-only review.

## Solution Assessment

Unclear at implementation level. The requested direction is valid, but the maintainable fix should be a single privacy-gated Plugin SDK contract rather than more channel-specific patches.
