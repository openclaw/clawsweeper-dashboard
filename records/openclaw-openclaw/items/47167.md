## Summary

Keep open. Current main still lacks a supported cross-channel Plugin SDK hook that passively observes group messages before `requireMention` drops them; the source paths still return before `message_received`, and the remaining work needs maintainer API/privacy design.

## What This Changes

The issue requests a public, read-only early inbound message hook so plugins can observe group messages before `requireMention` mention gating drops them.

## Best Possible Solution

Design and document a public early inbound observation seam that is read-only, opt-in, privacy-aware, channel-neutral, fires before mention gating, and has regression coverage for the affected bundled plugins.

## Reproduction Assessment

Yes, source-reproducible. Configure an affected group channel with `requireMention: true`, send a non-mention group message, and the current ingress code returns or drops before `dispatchReplyFromConfig` can emit Plugin SDK `message_received`.

## Solution Assessment

Unclear pending maintainer design. The requested read-only early hook is a plausible direction, but the maintainable fix should be a deliberate Plugin SDK contract and privacy/permission model rather than per-channel source patches.
