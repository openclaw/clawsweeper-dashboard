## Summary

Keep open: current main still lacks a public early raw-inbound Plugin SDK hook, and several representative requireMention skip paths return before downstream plugin message hooks can observe the skipped message.

## What This Changes

Request a passive read-only early inbound hook so archive plugins can observe requireMention-skipped group messages without creating sessions or replies.

## Best Possible Solution

Define a channel-neutral, opt-in, read-only early inbound observation contract with minimized payloads, privacy documentation, and per-channel wiring that preserves existing routing decisions.

## Reproduction Assessment

Yes, as a source-level reproduction. The public hook catalog has no early raw inbound hook, and representative requireMention skip branches return before the shared message_received dispatch path.

## Solution Assessment

Unclear for the exact hook name, but the direction is valid. The safest solution is an explicit opt-in early observation contract, not making all skipped traffic visible to existing hooks by default.
