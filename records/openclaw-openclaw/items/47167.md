## Label Justifications

- `P2`: This is a focused plugin API/channel observability gap with real user plugin impact, but it needs product/security review rather than emergency handling.
- `impact:message-loss`: The issue is about inbound group messages being intentionally skipped before supported plugin message hooks can observe them.
- `impact:session-state`: The requested archive/history plugin stores session-adjacent conversation chronology that currently cannot see mention-skipped messages.
- `impact:security`: The requested hook changes which group-message content installed plugins can observe before mention gating.

## Summary

Keep open: current main and the latest release still lack a public early observed hook for mention-skipped inbound messages. The public @dashhuang/conversation-archive plugin and the open implementation PR show this is a real plugin API gap, but the remaining action is maintainer API/privacy review rather than conservative cleanup.

## What This Changes

The issue requests a passive read-only inbound hook so plugins can archive group messages skipped by mention gating without creating sessions or replies.

## Best Possible Solution

Review and refresh https://github.com/openclaw/openclaw/pull/83101 or a replacement that keeps message_received semantics unchanged while adding a narrow, read-only, no-session/no-reply observed hook with explicit maintainer approval for the plugin visibility boundary.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path for Telegram: with requireMention enabled and no mention, the current branch returns null before the public message_received hook path. I did not run a live Telegram group test in this read-only review.

## Solution Assessment

Unclear until maintainers approve the contract. A separate read-only observed hook is cleaner than moving message_received earlier, but exposing skipped group content to public plugins is a product/security decision and the open PR needs refresh/review before it can land.
