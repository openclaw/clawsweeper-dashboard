## Label Justifications

- `P2`: This is a focused Plugin SDK and channel-hook improvement with clear source reproduction but product/security review needs.
- `impact:message-loss`: The issue is about inbound group messages being suppressed before plugin message hooks can observe or archive them.
- `impact:session-state`: The requested archive/history layer is intended to preserve raw conversation context for later retrieval outside live agent sessions.
- `impact:security`: An earlier hook would expose skipped group content and identifiers before existing mention-gating boundaries.

## Summary

Keep open: current main and the latest shipped tag still lack a public early inbound plugin hook, representative mention-gated channel paths still return before public message hooks, and the open implementation PR is the active path rather than a reason to close this issue.

## What This Changes

The issue asks for a passive, read-only early inbound plugin hook so group messages skipped by mention gating can be archived without entering agent sessions or triggering replies.

## Best Possible Solution

Land a privacy-reviewed, opt-in, read-only early inbound Plugin SDK contract through https://github.com/openclaw/openclaw/pull/83101 or a successor, then close this issue after the implementation merges and ships.

## Reproduction Assessment

Yes, as a source-level reproduction: current main lacks the public early inbound hook, and representative Telegram and Discord requireMention skip paths return before the public `message_received` hook. I did not run a live transport reproduction in this read-only review.

## Solution Assessment

Unclear for the exact API and policy shape. A separate read-only observed hook is narrower than moving `message_received` earlier, but the Plugin SDK contract, privacy boundary, and multi-channel scope need maintainer approval.
