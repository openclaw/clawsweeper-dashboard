## Summary

Keep open. Current main has partial Telegram/Signal internal silent-ingest support, but it still lacks a public early inbound Plugin SDK hook, `message_received` remains downstream of normal dispatch, and representative Slack/Discord/LINE/iMessage mention-skip paths still return before plugin hooks can observe the message.

## What This Changes

The issue requests a passive early inbound hook so plugins can archive group messages skipped by `requireMention` without routing them into agent sessions.

## Best Possible Solution

Define one channel-neutral, opt-in, read-only early inbound observation contract with minimized payloads, explicit privacy documentation, and per-channel wiring that preserves mention/allowlist routing decisions.

## Reproduction Assessment

Yes, as a source-level reproduction rather than a live run: representative `requireMention` no-mention paths return before the shared dispatch hook, while the public hook catalog has no early raw inbound hook a plugin can register.

## Solution Assessment

Unclear for the exact requested hook name, but the direction is valid. The safest solution is an explicit opt-in early observation contract rather than making all skipped traffic visible to existing hooks by default.
