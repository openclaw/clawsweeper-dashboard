## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Defer table-tail chunks while streaming:** `src/agents/pi-embedded-block-chunker.ts:455`
  - body: When `force` is false and `maxChars` lands inside a table whose detected end is the current buffer tail, this returns `buffer.length` and emits a partial table. Later `append()` rows continue in another message, so the new protection can still split streaming tables.
  - confidence: 0.9
- **[P2] Include one-pipe rows in table spans:** `src/markdown/table-spans.ts:76-86`
  - body: Valid two-column Markdown tables without outer pipes have one `|` per header/body row, but this expansion requires at least two. The parser then protects only the delimiter row, leaving the header and body available for chunk splits.
  - confidence: 0.88
- **[P3] Add the required changelog entry:** `src/agents/pi-embedded-block-chunker.ts:3-4`
  - body: This changes user-visible streaming output, but the PR file list does not include `CHANGELOG.md`. OpenClaw policy requires user-facing `feat`, `fix`, and `perf` changes to add a changelog entry before merge.
  - confidence: 0.84
