## Summary

Keep this PR open, but do not merge it as-is. Current main still lacks table-aware block-streaming protection, while this branch has two concrete correctness gaps and now overlaps with the open replacement candidate #73981, so a maintainer should choose or reconcile the canonical fix.

## What This Changes

The PR adds a Markdown table span parser, wires table spans into `EmbeddedBlockChunker` break protection alongside code fences, and adds parser/chunker unit coverage.

## Best Possible Solution

Land one canonical table-aware block-streaming fix that protects complete and streaming Markdown tables, supports valid no-edge-pipe table syntax, includes the `subscribeEmbeddedPiSession` integration case, and then close the superseded PRs/issues against the landed SHA.

## Reproduction Assessment

Yes. The current-main bug path is `subscribeEmbeddedPiSession` with paragraph block chunking and a Markdown table, and the PR-specific defects reproduce from source by appending a still-growing table past `maxChars` and by parsing `A | B\n--- | ---\n1 | 2`.

## Solution Assessment

No. The direction is appropriate, but this PR is not the best mergeable fix until it defers tail-ended streaming tables and expands table spans over one-pipe rows; #73981 or a revised branch should carry the final canonical implementation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Defer table-tail chunks during streaming:** `src/agents/pi-embedded-block-chunker.ts:455`
  - body: When `force` is false and `maxChars` lands inside a table, `parseTableSpans` can report the current buffer tail as `table.end`. Returning that value emits a still-growing table as soon as the buffer crosses `maxChars`, so later `append()` rows continue in another message and the table is still split.
  - confidence: 0.9
- **[P2] Include one-pipe table rows in spans:** `src/markdown/table-spans.ts:76-86`
  - body: Valid two-column Markdown tables without outer pipes have only one `|` in header/body rows, but the level-1 expansion requires `countPipes(...) >= 2`. The parser then protects only the delimiter row, leaving the header and body available for chunk splits.
  - confidence: 0.88
