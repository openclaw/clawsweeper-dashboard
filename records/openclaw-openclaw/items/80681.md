## Summary

Keep open: current main still hard-caps trajectory events at 256 KiB, so the requested override is not already implemented, but this PR needs stronger after-fix proof, maintainer acceptance of the new env config surface, and fixes for the import/security/test regressions before merge.

## What This Changes

The PR adds `OPENCLAW_TRAJECTORY_RUNTIME_EVENT_MAX_BYTES` parsing to core and Codex trajectory writers, adds raw/suffix tests, and updates trajectory docs and changelog.

## Best Possible Solution

Accept the override only after maintainer agreement, keep the implementation limited to the existing writers with narrow SDK imports and the shared safe append helper, and add corrected focused tests plus redacted after-fix runtime proof.

## Reproduction Assessment

Yes for the current source behavior: current main hard-caps runtime trajectory events at 256 KiB in both core and Codex paths, and the truncation path is clear from source. I did not establish a live failing run in this read-only review.

## Solution Assessment

No, not as currently patched: the env override direction is plausible, but the implementation should preserve the narrow runtime SDK imports and shared safe append helper, and the fallback test needs a genuinely malformed value.
