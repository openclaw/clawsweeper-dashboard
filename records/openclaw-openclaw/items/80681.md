## Summary

Keep open: the requested trajectory cap override is plausible and current source makes the hard-coded limit clear, but this patch still has blocking implementation, security, test, and real-behavior-proof gaps.

## What This Changes

The PR adds an environment-variable override for the core and Codex trajectory per-event byte cap, plus parser tests, docs, and changelog coverage.

## Best Possible Solution

Land a narrower version that only makes the existing trajectory writers' per-event cap configurable, keeps the current SDK/security helper boundaries, fixes the fallback test, and includes redacted after-fix runtime proof.

## Reproduction Assessment

Yes for current source behavior: current main hard-codes 256 KiB caps in both core and Codex trajectory writers, and the truncation branches are visible from source. I did not run a live OpenClaw session in this read-only review.

## Solution Assessment

No, not as currently patched. The env override direction is plausible, but the patch should preserve the narrow runtime/security subpaths and shared append helper, fix the invalid-value test, and provide after-fix proof.
