## Summary

Keep open: the proposed override is a plausible trajectory feature, but the patch only updates the core recorder while the bundled Codex app-server trajectory recorder still uses its own hard-coded 256 KiB event cap, and the PR lacks real behavior proof.

## What This Changes

The PR adds `OPENCLAW_TRAJECTORY_RUNTIME_EVENT_MAX_BYTES` parsing in `src/trajectory/paths.ts`, new resolver tests, and a changelog entry for a configurable runtime trajectory per-event cap.

## Best Possible Solution

Update all runtime trajectory recorders that write this event shape, document the supported override and limits in the trajectory docs, then require contributor proof from a real run showing a larger event is preserved with private data redacted.

## Reproduction Assessment

Yes, for the PR defect by source inspection: the patch changes only the core trajectory cap, while `extensions/codex/src/app-server/trajectory.ts` still hard-codes and enforces 256 KiB. No live runtime reproduction was run in this read-only review.

## Solution Assessment

No. The core resolver is a reasonable direction, but the maintainable fix needs to cover the bundled Codex app-server recorder and public docs before the advertised override is complete.
