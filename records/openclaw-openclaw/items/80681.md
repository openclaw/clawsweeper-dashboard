## Summary

Keep open: current main still has the fixed 256 KiB trajectory event cap, but this PR needs stronger after-fix runtime proof and should avoid expanding the Codex trajectory file-write/security surface before merge.

## What This Changes

Adds an env-variable override for core and Codex trajectory runtime event byte caps, with parser tests, docs, and a changelog entry.

## Best Possible Solution

Keep the configurable cap only if maintainers accept the new env surface, and implement it by passing a resolved limit through the existing trajectory writers without replacing shared security helpers.

## Reproduction Assessment

Yes for source behavior: current main hard-codes 256 KiB caps in both core and Codex trajectory writers and the truncation path is visible in source. I did not run a live OpenClaw session in this read-only review.

## Solution Assessment

No, not as currently patched: the override direction is plausible, but the implementation should keep the existing runtime SDK/security subpaths and shared append helper. The fallback test should also use a genuinely malformed value.
