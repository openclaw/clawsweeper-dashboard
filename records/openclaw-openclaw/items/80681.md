## Summary

Keep open: this is an active external PR adding a new trajectory cap env setting, but the current patch still needs code changes, contributor-supplied after-fix proof, and maintainer acceptance of the new config contract before merge.

## What This Changes

The PR adds `OPENCLAW_TRAJECTORY_RUNTIME_EVENT_MAX_BYTES` parsing for core and Codex trajectory writers, tests suffix/raw-byte parsing, and documents the new trajectory limit setting.

## Best Possible Solution

Keep the env override narrowly scoped to the existing trajectory writers, preserve the existing runtime SDK/security helper boundaries, add the missing focused test correction, and require redacted after-fix runtime proof before merge.

## Reproduction Assessment

Yes for the current source behavior: current main hard-codes 256 KiB caps in both core and Codex trajectory writers, and the truncation path is clear from source. I did not run a live OpenClaw reproduction in this read-only review.

## Solution Assessment

No, not as currently patched: the env override direction is plausible, but the implementation should avoid switching Codex trajectory to the broad SDK barrel and should keep the shared safe append helper rather than reimplementing file-write security logic.
