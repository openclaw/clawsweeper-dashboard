## Summary

Keep open: current main and the latest release still lack the requested async external-plugin transform hook, while the only awaited tool-result rewrite path remains bundled-only. The related implementation at https://github.com/openclaw/openclaw/pull/64907 is open, synchronous, text-only, and explicitly does not cover this async contract; related duplicate cleanup has also kept this issue as the async design thread.

## What This Changes

The issue requests an async plugin hook that can mutate or suppress tool results after execution and before transcript writing or model visibility.

## Best Possible Solution

Choose one versioned Plugin SDK contract for async result mutation with explicit trust policy, ordering, timeout, failure, shape-bound, block/suppress, audit/provenance, docs, and Pi/Codex behavior before implementation.

## Reproduction Assessment

Yes for a source-level feature-gap check: current main lacks the requested hook names, async persistence hook returns are ignored by design and test, and external plugins cannot use the existing awaited middleware path.

## Solution Assessment

Unclear. The requested async hook addresses a real gap, but exposing it as an ordinary external-plugin hook is not clearly the safest solution until maintainers settle the Plugin SDK trust and security contract alongside the related synchronous pre-model work.
