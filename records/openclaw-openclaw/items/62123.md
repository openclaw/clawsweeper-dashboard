## Summary

Keep open: current main still only offers fire-and-forget `after_tool_call`, synchronous `tool_result_persist`/`before_message_write`, and bundled-only async middleware; related open work is narrower or adjacent and not merged.

## What This Changes

Add an awaited plugin hook that can asynchronously replace or block tool-result messages after tool execution and before transcript/model visibility.

## Best Possible Solution

Keep this open until maintainers choose a versioned contract: either document that async pre-model tool-result rewriting remains bundled-only, or expose a narrow trusted plugin API with explicit ordering, timeouts, failure behavior, block semantics, shape bounds, SDK docs, and Pi/Codex coverage.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: the requested hook names are absent, async `tool_result_persist` returns are ignored by design and test, and external plugins are blocked from the existing awaited tool-result middleware.

## Solution Assessment

Unclear. The requested async hook addresses a real gap, but exposing it as an ordinary external-plugin hook is not clearly the safest solution until maintainers settle the trust policy and reconcile #64907, #29132, #52849, and #58558.
