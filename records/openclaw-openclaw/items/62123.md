## Summary

Keep open: current main and the latest release still lack the requested async public Plugin SDK hook, and the existing awaited tool-result rewrite seam remains bundled-only. The related open implementation PR is a narrower synchronous text-only proposal, so it does not cover this issue's async mutation/suppression contract.

## What This Changes

The issue requests a new async plugin hook that can await validation and replace or suppress completed tool results before transcript write.

## Best Possible Solution

Choose a maintainer-approved Plugin SDK contract for awaited tool-result mutation, or explicitly document that async pre-transcript/pre-model result rewriting remains bundled/internal and route async scanning through a different trusted extension path.

## Reproduction Assessment

Yes for a source-level feature-gap check, not as a bug reproduction. Current main and v2026.5.18 lack the requested async public hook, intentionally ignore async persistence hook returns, and keep the awaited tool-result middleware path bundled-only.

## Solution Assessment

Unclear. The requested async hook targets a real extension gap, but the safest API shape requires maintainer Plugin SDK and security-policy decisions before implementation.
