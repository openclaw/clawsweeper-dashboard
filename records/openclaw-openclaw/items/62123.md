## Summary

Keep open: current main still lacks the requested async public Plugin SDK hook for mutating or suppressing tool results before transcript write, and the existing awaited pre-model rewrite seam remains bundled-only. The related open implementation PR is synchronous and text-only, so it does not cover this issue's async contract.

## What This Changes

Request an async plugin hook after tool execution and before transcript persistence that can replace or suppress the tool-result message.

## Best Possible Solution

Choose a maintainer-approved, versioned Plugin SDK contract for async tool-result mutation, or explicitly document that awaited pre-model/pre-transcript result rewriting remains bundled-only.

## Reproduction Assessment

Yes for a source-level feature-gap check, not as a bug reproduction. Current main lacks the requested async public hook names, ignores async persistence hook returns by design and test, and keeps the awaited middleware path unavailable to external plugins.

## Solution Assessment

Unclear. The requested async hook addresses a real extension gap, but the safest API shape needs maintainer Plugin SDK and security-policy decisions before implementation.
