## Summary

Keep open: current main still lacks the requested async public tool-result mutation hook, and the existing awaited rewrite path is intentionally bundled-only. The adjacent open PR and issues are related but do not implement this async replace-or-block contract on main.

## What This Changes

The issue requests an async plugin hook that can replace or suppress a tool result after execution but before transcript/model visibility.

## Best Possible Solution

Keep this open until maintainers choose a versioned contract: either document that awaited pre-model tool-result rewriting remains bundled-only, or expose a narrow trusted plugin API with explicit ordering, timeouts, failure behavior, block semantics, shape bounds, SDK docs, and Pi/Codex coverage.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: the requested hook names are absent, async `tool_result_persist` returns are ignored by code and test, and external plugins are rejected from the existing awaited middleware path.

## Solution Assessment

Unclear. The requested async hook addresses a real gap, but exposing it as an ordinary plugin hook is not clearly the safest solution until maintainers settle the trust policy and reconcile the adjacent open hook proposals.
