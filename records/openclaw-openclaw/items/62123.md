## Summary

Keep open: current main still lacks the requested async external-plugin tool-result mutation hook, and the only awaited pre-model result rewrite path remains reserved for bundled middleware. Related cleanup has kept this issue as the async design thread, while the open implementation candidate is synchronous, text-only, and does not cover this contract.

## What This Changes

Request an async Plugin SDK hook that can modify or suppress a tool result after execution and before transcript/model-visible persistence.

## Best Possible Solution

Choose one reviewed, versioned Plugin SDK contract for async result mutation, or explicitly document that pre-model/pre-transcript result rewriting remains bundled-only.

## Reproduction Assessment

Yes for a source-level feature-gap check, not as a bug reproduction. Current main lacks the requested hook names, ignores async persistence hook returns by design and test, and keeps the existing awaited middleware path unavailable to external plugins.

## Solution Assessment

Unclear. The requested async hook addresses a real gap, but exposing it as an ordinary external-plugin hook is not clearly the safest solution until maintainers settle trust policy, ordering, timeout, failure, block/suppress, bounds, and audit semantics.
