## Summary

Keep open: current main still has no public async tool-result transform hook, and the only awaited pre-model tool-result rewrite path remains bundled-only while related proposals are still open and not a merged solution.

## What This Changes

The issue requests an async plugin hook that can replace or suppress a tool result after execution and before transcript/model visibility.

## Best Possible Solution

Keep this open until maintainers choose a versioned contract: a trusted/bundled-only policy or a narrow public async hook with explicit ordering, timeouts, failure behavior, block semantics, size bounds, SDK docs, and Pi/Codex coverage.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: the requested hook names are absent, async tool_result_persist returns are ignored by implementation and test, and the existing awaited middleware path is blocked for external plugins.

## Solution Assessment

Unclear. The requested async transform hook fits the reported defensive workflow, but exposing it as a normal plugin hook may not be the safest maintainable answer without a settled trust and failure policy.

Status: needs_attention

Summary: The issue is security-sensitive because it asks to expose async pre-model/pre-transcript tool-result rewriting across plugin boundaries.
