## Summary

Keep open: current main still lacks the requested async public Plugin SDK hook, while the only awaited tool-result rewrite path remains bundled-only and related work deliberately does not cover the async contract.

## What This Changes

The issue requests an async Plugin SDK hook that can validate, replace, or suppress a tool result after execution but before transcript persistence.

## Best Possible Solution

Settle one versioned Plugin SDK contract for async result mutation with explicit trust policy, ordering, timeout, failure, bounds, block/suppress, audit/provenance, docs, and Pi/Codex behavior.

## Reproduction Assessment

Yes for a source-level feature-gap check: current main lacks the requested hook names, ignores async `tool_result_persist` returns by design and test, and blocks external plugins from the existing awaited middleware seam.

## Solution Assessment

Unclear. The requested async hook addresses a real gap, but exposing it as an ordinary external-plugin hook is not clearly the safest API until maintainers define the Plugin SDK trust and security contract.
