## Summary

Keep open. Current main still lacks an awaited public plugin hook that can mutate or suppress tool results before transcript/model visibility; `after_tool_call` is observation-only, `tool_result_persist` remains synchronous, and the existing async pre-model middleware is deliberately bundled-only. Related issue threads and PR #64907 are active but only cover adjacent or narrower contracts, so this async external-plugin request is not implemented or safely closeable.

## What This Changes

The issue requests an async plugin hook after tool execution and before transcript write that can replace or suppress tool results for external-content validation and prompt-injection defense.

## Best Possible Solution

Keep this open until maintainers consolidate the overlapping hook requests and choose a versioned contract. The end state should either document that async pre-model tool-result rewriting remains bundled-only, or expose a narrow trusted plugin API with explicit ordering, timeout/failure behavior, block/suppress semantics, shape bounds, prompt-injection policy, SDK docs, and Pi/Codex coverage.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes. A source-level reproduction is enough: an async `tool_result_persist` handler is ignored by the current hook runner, `after_tool_call` is void/fire-and-forget, and the requested hook names are absent from the public hook catalog.

## Solution Assessment

Is this the best way to solve the issue? Unclear. The reported gap is real, but exposing it as a normal async external-plugin hook is not clearly the safest design without a trust policy; a safer path may be to extend the existing bundled-only middleware contract or land the narrower text-only hook first.
