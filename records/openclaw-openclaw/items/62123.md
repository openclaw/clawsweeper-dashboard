## Summary

Current main and the shipped v2026.6.6 release now provide the requested async tool-result mutation boundary through `api.registerAgentToolResultMiddleware(...)`, so the central plugin API gap is implemented even though the final API name differs from the proposal.

## What This Changes

Request to add an async Plugin SDK hook that can transform or suppress tool results after execution and before transcript/model consumption.

## Best Possible Solution

Use the shipped `api.registerAgentToolResultMiddleware(...)` contract with `contracts.agentToolResultMiddleware` and explicit enablement for trusted async result transforms, while keeping `tool_result_persist` as the synchronous transcript-specific hook.

## Reproduction Assessment

Not applicable as a bug reproduction. Source and release inspection show the previous feature gap is now covered by the shipped async tool-result middleware API.

## Solution Assessment

Yes. The runtime-neutral, manifest-gated middleware API is the maintainable solution because it keeps trusted result mutation out of sync transcript hooks and works across OpenClaw and Codex tool paths.
