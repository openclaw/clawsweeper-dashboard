  - body: When `channels.telegram.presence` is absent, this factory still enables default reactions and the typing loop.   - body: The received hook runs before `handleInboundMessageLike`, so it can react to messages that current Telegram handling later drops for disabled groups/topics, unauthorized DMs, allowlists, or mention gating. Move visible presence behind the existing accepted-message/ack gate.
  - confidence: 0.94
- **[P2] Add presence config to the runtime schema:** `src/config/types.telegram.ts:223-227`
  - body: The PR advertises `channels.telegram.presence` only through TypeScript types. The Telegram Zod schema is strict and has no `presence` property, so users who try the new config will hit unknown-key validation and generated metadata will not expose it.
  - confidence: 0.91
- **[P2] Make reaction null disable working reactions:** `extensions/telegram/src/presence-indicator.ts:119-124`
  - body: The type docs say `reaction: null` disables reactions entirely, but `workingEmoji` still falls back to the default unless `workingReaction` is also null. That lets `onProcessingStart` set and later clear a working reaction despite the advertised disable switch.
  - confidence: 0.87
