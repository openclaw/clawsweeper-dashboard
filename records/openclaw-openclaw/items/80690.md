  - confidence: 0.91

  - body: This creates a second default-on Telegram reaction/typing lifecycle next to the existing `ackReaction`, `ackReactionScope`, `messages.statusReactions.enabled`, `reactionLevel`, `removeAckAfterReply`, and `typingMode` paths. Because Telegram has one reaction slot per message, the new lifecycle can overwrite or clear configured status reactions and ignores current opt-in/default semantics; fold this into the existing controller or prove why a separate gated path is needed.
