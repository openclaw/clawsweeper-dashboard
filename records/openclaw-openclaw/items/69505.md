## Summary

Keep open. Current main still lacks an assistant-prefix carrier across the cron payload contract, direct agent RPC, agent defaults, CLI flags, and provider execution flow; this is a new public API/config feature rather than a narrow bug fix.

## What This Changes

Request an optional assistantPrefix-style setting for cron agent turns, agent defaults, CLI agent/cron commands, and provider adapters so generated replies begin with fixed text.

## Best Possible Solution

Define one additive, documented assistant-continuation contract with clear precedence across cron payloads, direct agent RPC, agent defaults, CLI flags, and provider adapters, including explicit thinking and unsupported-provider behavior.

## Reproduction Assessment

Yes for the missing capability: current strict schemas and CLI builders omit an assistant-prefix field, so it cannot be accepted or forwarded. The production WhatsApp/Z.AI incident itself was not reproduced in this read-only pass.

## Solution Assessment

Unclear until maintainer API review. Provider prefill is plausible, but field naming, precedence, streaming behavior, thinking interactions, and unsupported-provider semantics should be chosen centrally.
