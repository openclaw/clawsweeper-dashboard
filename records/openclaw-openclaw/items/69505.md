## Summary

Keep open. Current main still lacks the requested assistant-prefix contract in cron payloads, direct agent runs, agent defaults, CLI flags, docs, and provider forwarding, and the remaining work is a public API/config feature that needs maintainer direction rather than a narrow bug-fix sweep.

## What This Changes

Request to add an `assistantPrefix` or equivalent public contract for cron `agentTurn` payloads, agent defaults, CLI agent/cron commands, and provider-side generation prefill so replies start with fixed text.

## Best Possible Solution

Define one additive documented assistant-continuation contract that flows through cron payloads, direct agent runs, agent defaults, CLI flags, and provider adapters with explicit precedence and no-op rules.

## Reproduction Assessment

Yes for the missing capability: strict current-main schemas and CLI builders do not accept or forward an assistant-prefix field. The live WhatsApp/model-fallback incident itself was not reproduced.

## Solution Assessment

Unclear until maintainer API review. Provider prefill is plausible and `messages.responsePrefix` is not equivalent, but the public name, precedence, thinking interaction, and provider fallback behavior should be chosen centrally.
