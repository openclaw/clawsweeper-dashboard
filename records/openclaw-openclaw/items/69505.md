## Summary

Keep open. Current main still does not provide an assistant-prefix/prefill contract across cron agentTurn payloads, direct agent RPC, agent defaults, CLI flags, or provider execution, and the requested behavior is a new public protocol/config feature that needs maintainer API direction.

## What This Changes

Add an optional assistantPrefix-style field for cron agentTurn payloads and agent defaults so generated replies can be provider-prefilled to start with fixed text.

## Best Possible Solution

Define one additive, documented assistant-continuation contract with clear precedence across cron payloads, direct agent RPC, agent defaults, CLI flags, and provider adapters.

## Reproduction Assessment

Yes for the missing capability: current strict schemas and CLI builders still omit any assistant-prefix carrier, so the requested field cannot be accepted or forwarded. The production WhatsApp/Z.AI incident itself was not live-reproduced in this read-only pass.

## Solution Assessment

Unclear until maintainer API review. Provider prefill is plausible, but field naming, precedence, streaming behavior, thinking interactions, and unsupported-provider semantics should be chosen centrally.
