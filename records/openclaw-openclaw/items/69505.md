## Summary

Keep open. Current main still has no assistant-prefix carrier in the cron payload, direct agent RPC, agent defaults, CLI surfaces, or generation path, and the remaining work is an additive public API/config feature rather than conservative cleanup.

## What This Changes

Request an assistant-prefix contract for cron agentTurn payloads, agent defaults, direct agent runs, CLI flags, docs, and provider adapters so replies can be forced to start with fixed text.

## Best Possible Solution

Define one additive, documented assistant-continuation contract that flows through cron payloads, direct agent RPC, agent defaults, CLI flags, and provider adapters with explicit precedence and unsupported-provider behavior.

## Reproduction Assessment

Yes for the missing capability: current strict schemas and CLI builders do not accept or forward an assistant-prefix field. The production WhatsApp/model-fallback incident itself was not reproduced in this read-only pass.

## Solution Assessment

Unclear until maintainer API review. Provider prefill is a plausible direction, but the public field name, precedence, thinking interaction, and fallback/no-op semantics should be chosen centrally.
