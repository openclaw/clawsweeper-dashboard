## Summary

Keep open. Current main still lacks the requested assistant-prefix carrier across cron payloads, direct agent RPC, agent defaults, CLI surfaces, and provider generation flow; this is an additive public API/config feature that needs maintainer design direction.

## What This Changes

Requests an additive assistantPrefix-style contract for cron agentTurn payloads, agent defaults, CLI entry points, and provider adapters so replies can be forced to start with fixed text.

## Best Possible Solution

Define one additive, documented assistant-continuation contract that flows through cron payloads, direct agent RPC, agent defaults, CLI flags, and provider adapters with explicit precedence and unsupported-provider behavior.

## Reproduction Assessment

Yes for the missing capability: strict schemas and CLI builders on current main do not accept or forward an assistant-prefix field. The production WhatsApp/Z.AI fallback incident itself was not reproduced in this read-only pass.

## Solution Assessment

Unclear until maintainer API review. Provider prefill is plausible, but the public field name, precedence rules, thinking interaction, and fallback/no-op semantics should be chosen centrally.
