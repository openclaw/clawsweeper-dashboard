## Summary

Keep open. Current main still lacks an assistant-prefix carrier across the cron payload contract, direct agent RPC, agent defaults, CLI builders, SDK run params, and provider execution path; this is valid feature/API work, not a narrow bug cleanup item.

## What This Changes

The issue requests an optional assistantPrefix-style field on cron agentTurn payloads, agent defaults, CLI/agent invocation, and provider adapters so generated replies can be forced to start with fixed text.

## Best Possible Solution

Define one additive, documented assistant-continuation contract with clear precedence across cron payloads, direct agent RPC, agent defaults, CLI/SDK callers, and provider adapters, including thinking-mode and unsupported-provider behavior.

## Reproduction Assessment

Yes for the missing capability: current strict schemas, CLI builders, SDK run params, and provider wrappers omit any assistant-prefix field, so it cannot be accepted or forwarded on current main. The reported production WhatsApp/Z.AI fallback incident itself was not live-reproduced in this read-only pass.

## Solution Assessment

Unclear until maintainer API review. Provider prefill is plausible, but field naming, precedence, streaming/block delivery behavior, thinking interactions, and unsupported-provider semantics should be chosen centrally before code is written.
