## Summary

Keep open. Current main still lacks the requested assistant-prefix contract in cron payloads, direct agent runs, agent defaults, CLI flags, docs, and provider runtime forwarding; adjacent `messages.responsePrefix` and assistant-prefill cleanup do not guarantee the model starts generation with fixed text.

## What This Changes

Request to add an assistant-prefix or prefill field across cron `agentTurn` payloads, agent defaults, CLI agent/cron surfaces, and provider adapters so generated replies begin with fixed text.

## Best Possible Solution

If maintainers accept the direction, add one documented additive contract that flows from cron payloads, direct agent runs, agent defaults, and CLI flags into a provider-facing continuation hook with explicit precedence and no-op rules.

## Reproduction Assessment

Yes for the missing capability: current strict schemas and typed config/payload surfaces have no accepted assistant-prefix field, and CLI/runtime surfaces have nothing to forward. The production WhatsApp/model-fallback incident itself was not live-reproduced.

## Solution Assessment

Unclear until maintainer API review. Provider prefill is a plausible implementation strategy and `messages.responsePrefix` is not equivalent, but the exact public name, precedence, thinking interaction, and provider fallback behavior should be chosen centrally.
