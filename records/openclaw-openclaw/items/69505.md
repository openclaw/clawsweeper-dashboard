## Summary

Keep open: current main still lacks a generation-time assistant-prefix contract for cron agentTurn payloads, direct agent RPC, agent defaults, CLI flags, and provider execution, and adding one is a new public protocol/config feature that needs maintainer API direction.

## What This Changes

The issue requests an assistantPrefix-style option for cron agentTurn payloads, direct agent runs, agent defaults, CLI, docs, and provider adapters so generated replies start with fixed text before channel delivery.

## Best Possible Solution

Define one additive, documented assistant-continuation contract with clear precedence across cron payloads, direct agent RPC, agent defaults, CLI flags, delivery responsePrefix, streaming behavior, and provider adapters.

## Reproduction Assessment

Yes for the missing capability: current strict schemas and CLI request builders omit any assistant-prefix carrier, so such a field cannot be accepted and forwarded today. The reported production WhatsApp/Z.AI incident itself was not live-reproduced in this read-only pass.

## Solution Assessment

Unclear until maintainer API review. Provider prefill is plausible, but field naming, precedence, interaction with delivery-side responsePrefix, streaming, thinking, and unsupported-provider semantics need a central product/protocol decision.
