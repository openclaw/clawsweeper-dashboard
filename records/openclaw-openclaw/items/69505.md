## Summary

Keep open. Current main still lacks the requested assistant-prefix contract, while the existing response-prefix feature only decorates already-produced outbound replies; the remaining work is an additive public protocol/config/provider feature that needs maintainer API direction.

## What This Changes

Add an assistant-prefix/prefill option to cron agentTurn payloads, agent defaults, CLI agent/cron flows, and provider adapters so generated replies begin with fixed text.

## Best Possible Solution

Define one additive, documented assistant-continuation contract that flows through cron payloads, direct agent RPC, agent defaults, CLI flags, and provider adapters with explicit precedence and unsupported-provider behavior.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high-confidence proof the requested field is absent from strict cron, direct-agent, defaults, and CLI surfaces; the production incident itself was not live-reproduced.

## Solution Assessment

Unclear until maintainer API review. Provider-native assistant prefill is plausible, but field naming, precedence with responsePrefix/thinking, fallback/no-op behavior, and streaming semantics should be chosen centrally.
