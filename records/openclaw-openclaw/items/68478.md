## Summary

Keep open. Current main still lacks a cross-surface suppression or grouping policy for routine success confirmations; existing controls such as visible-reply mode and cron `--no-deliver` are narrower, while skill commands, ClawHub installs, and isolated cron delivery still produce visible output.

## What This Changes

Request to suppress, group, or configure routine auto-generated chat confirmations such as cron, skill-install, and Done messages.

## Best Possible Solution

Add a centralized notification policy, likely under `messages` or `commands`, that can suppress or group routine success confirmations while preserving critical notices and explicit delivery controls.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: empty direct skill tool results still produce `✅ Done.`, ClawHub installs still return `Installed <slug>@<version>`, and isolated cron jobs default to announce unless explicitly disabled.

## Solution Assessment

No current implementation fully solves the issue. The maintainable fix is a shared event-class notification policy rather than a one-off flag for only cron, skill installs, or command fallback text.
