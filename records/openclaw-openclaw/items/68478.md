Keep open. Current main still has visible routine confirmations and only narrow per-surface controls, while the requested cross-surface suppression or grouping policy still needs a maintainer product/config decision.

## What This Changes

Request to silence, group, or make configurable routine auto-generated chat confirmations such as cron, skill-install, and generic done messages.

## Best Possible Solution

Define a centralized routine-notification policy that can suppress or group low-value success confirmations while preserving failures, approvals, security notices, and explicit delivery controls.

Unclear. A config flag is plausible, but the maintainable fix should classify routine success confirmations separately from failures, approvals, security notices, and explicit cron delivery rather than add one-off string filters.
