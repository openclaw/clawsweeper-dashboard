## Summary

Keep open. Current main still has only narrow controls for visible replies, task notifications, exec lifecycle notices, and cron fallback delivery; it does not provide the requested cross-surface way to suppress or group routine success confirmations such as skill-command `✅ Done.`, ClawHub skill install success text, or cron-created/delivered chat notices. The prior ClawSweeper comment is consistent with the current source re-check.

## What This Changes

The issue requests a default-silent or configurable policy to suppress or group routine auto-generated chat confirmations across cron, skill installation, and generic success replies.

## Best Possible Solution

Define one coherent notification policy, probably under `messages` or `commands`, that can suppress or group routine success confirmations by event class while preserving failures, approvals, security notices, reminders, failure alerts, and existing narrow controls such as `tools.exec.notifyOnExit`, cron delivery mode, and model silent-reply behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: a direct tool-dispatched skill command whose tool returns no text still produces `✅ Done.`, ClawHub `skills.install` returns an `Installed ...` success message, and isolated cron delivery defaults to `announce` unless explicitly disabled.

## Solution Assessment

No current implementation fully solves the issue. The maintainable path is a cross-surface policy with preserved critical notices, not another one-off flag for only cron, skill installs, or direct command fallback text.
