## Summary

Keep open: current main still has visible routine success confirmations and cron announce defaults, while the existing controls only cover narrower visible-reply, verbose, progress, or error-message cases. The remaining work needs maintainer UX and config-contract direction before a safe implementation.

## What This Changes

Request a way to silence, selectively disable, or group routine auto-generated system success messages such as cron, skill-install, and done confirmations.

## Best Possible Solution

Define a centralized routine-notification policy that can suppress or group low-value success confirmations while preserving failures, approvals, security notices, reminders, and explicit delivery controls.

## Reproduction Assessment

Yes. Source inspection on current main shows visible `✅ Done.`, skill-install success text, cron announce defaults, and no global routine-message suppression field in the strict messages schema.

## Solution Assessment

Unclear pending maintainer policy. A config flag is plausible, but the maintainable fix should classify routine confirmations separately from failures, approvals, security notices, reminders, and explicit cron delivery rather than add one-off string filters.
