## Summary

Keep open. Current main and the latest stable tag still expose visible routine confirmations with only narrow per-surface controls, while the requested cross-surface suppression or grouping policy needs maintainer product/config direction.

## What This Changes

Request a way to silence or group automatic success/system messages such as cron-created, skill-installed, and done confirmations across chat surfaces.

## Best Possible Solution

Define a centralized routine-notification policy that can suppress or group low-value success confirmations while preserving failures, approvals, security notices, reminders, and explicit delivery controls.

## Reproduction Assessment

Yes, by source inspection: current main still has visible `✅ Done.`, skill-install success replies, cron announce defaults, and no global routine-message suppression field in the strict messages schema. I did not run a live Telegram/Discord flow because the source paths and tests already show the behavior.

## Solution Assessment

Unclear. A config flag is plausible, but the maintainable fix should classify routine success confirmations separately from failures, approvals, security notices, reminders, and explicit cron delivery rather than add one-off string filters.
