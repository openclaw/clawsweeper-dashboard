## Summary

Keep open. Current main still only has narrow controls for visible replies, exec lifecycle events, cron fallback delivery, and tool-error warnings; it does not provide the requested cross-surface way to suppress or group routine success confirmations such as skill-command Done fallbacks, ClawHub skill install success text, or cron announce/system-event chat notices.

## What This Changes

Request a config or policy to silence or group routine auto-generated system confirmations such as cron-created, skill-installed, and Done messages across chat surfaces.

## Best Possible Solution

Add one coherent notification policy, likely under `messages` or `commands`, that can suppress or group routine success confirmations while preserving failures, approvals, security notices, reminders, failure alerts, and existing narrow controls such as exec notifyOnExit and cron delivery mode.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: direct skill tool commands with empty output still return `✅ Done.`, ClawHub installs return visible success text, and isolated cron jobs default to announce delivery unless explicitly disabled.

## Solution Assessment

No current implementation fully solves the issue. The maintainable solution is a cross-surface policy with protected critical notices, not another one-off flag for only cron, skill installs, or direct command fallback text.
