## Summary

Keep open. Current main still lacks a cross-surface suppression or grouping policy for routine success confirmations; the available controls are narrower, while direct skill commands, ClawHub skill installs, and cron delivery paths still produce visible messages.

## What This Changes

The issue requests a config or delivery policy to disable or group routine auto-generated success messages such as cron, skill install, and Done confirmations.

## Best Possible Solution

Add a centralized notification policy, likely under `messages` or `commands`, that can suppress or group routine success confirmations while preserving critical notices and existing explicit delivery controls.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: an empty direct skill command result produces `✅ Done.`, ClawHub `skills.install` returns `Installed <slug>@<version>`, and isolated cron jobs default to announce unless explicitly disabled.

## Solution Assessment

No current implementation fully solves the issue. The maintainable fix is a shared event-class policy rather than another one-off flag limited to only cron, skill installs, or command fallback text.
