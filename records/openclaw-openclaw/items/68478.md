## Summary

Keep open: current main and the latest release still expose visible routine success messages, and the requested suppression/grouping behavior is a cross-surface config/product decision rather than an already-defined bug fix.

## What This Changes

The issue requests a way to suppress or group routine auto-generated system messages such as cron creation, skill install, and generic done confirmations.

## Best Possible Solution

Define a centralized routine-notification policy and config contract that can suppress or group low-value success confirmations while preserving errors, approvals, security notices, reminders, and explicitly requested delivery.

## Reproduction Assessment

Yes, at source level. Current main and v2026.5.18 still expose visible `✅ Done.`, skill-install success text, isolated cron announce defaults, and no routine-success suppression key in the strict messages config schema.

## Solution Assessment

Unclear until maintainer policy is chosen. A single config flag is plausible, but the maintainable solution should classify routine confirmations across command replies, skill installs, cron delivery, and docs instead of filtering individual strings.
