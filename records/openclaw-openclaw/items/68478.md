## Summary

Keep open: current main still has visible routine-success paths and no suppression/grouping contract, but this is a cross-surface notification UX/config feature that needs maintainer policy before implementation.

## What This Changes

Request a global or per-action way to suppress or group routine auto-generated chat confirmations such as cron creation, skill installs, and generic done messages.

## Best Possible Solution

Define a centralized routine-notification policy and config contract that can suppress or group low-value successes while preserving failures, approvals, security notices, reminders, and explicit delivery controls.

## Reproduction Assessment

Yes, at source level. Current main still exposes visible `✅ Done.`, skill-install success text, cron announce defaults, and no routine-success suppression field in the strict messages config schema.

## Solution Assessment

Unclear pending maintainer policy. A config flag is plausible, but the maintainable fix should classify routine confirmations across command replies, skill installs, and cron delivery instead of filtering individual strings.
