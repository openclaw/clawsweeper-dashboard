## Summary

Keep open. Current main still has visible routine confirmations and only narrow per-feature controls, so the cross-surface suppression or grouping policy requested here remains unresolved.

## What This Changes

Add a way to suppress or group routine auto-generated system confirmations such as cron creation, skill install, and Done messages.

## Best Possible Solution

Define a centralized routine-notification policy or config surface for success confirmations while preserving critical notices and explicit delivery controls.

## Reproduction Assessment

Yes. Source inspection of current main shows empty direct skill tool results fall back to `✅ Done.`, ClawHub install returns `Installed <slug>@<version>`, and isolated cron agent jobs default to announce delivery unless explicitly set to none.

## Solution Assessment

No current implementation fully solves the issue. The maintainable path is a shared event-class notification policy rather than one-off switches for only cron, skill installs, or command fallback text.
