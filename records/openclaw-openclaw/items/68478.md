## Summary

Keep open. Current main still has visible routine success confirmations and only narrow per-surface controls, while the requested cross-chat suppression or grouping policy remains unresolved and has fresh user support.

## What This Changes

Request to make routine auto-generated success/system confirmations silent, suppressible, or grouped across chat surfaces such as Telegram and Discord.

## Best Possible Solution

Define a centralized routine-notification policy or config surface that can suppress or group low-value success confirmations while preserving critical notices and explicit delivery controls.

## Reproduction Assessment

Yes, by source inspection. Current main still has a visible `✅ Done.` fallback, visible skill-install success text, isolated cron announce defaults, and no global routine-message suppression field in the strict messages schema.

## Solution Assessment

Unclear. A config flag matches the request, but the maintainable fix should classify routine success confirmations separately from failures, approvals, security-sensitive notices, and explicit delivery alerts rather than adding one-off switches.
