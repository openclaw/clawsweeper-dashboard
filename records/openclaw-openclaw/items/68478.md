## Summary

Keep open: current main and v2026.6.10 still have visible command, session, skill-install, and cron/status paths, and the existing message visibility controls do not provide a canonical suppression or grouping policy for these confirmations.

## What This Changes

The issue requests default-silent behavior, configurable suppression, or grouping for automatic system confirmation messages across chat channels.

## Best Possible Solution

Define one cross-surface confirmation policy, then implement it centrally across command replies, skill install/status responses, session acknowledgements, and cron events while preserving safety-critical errors and approvals.

## Reproduction Assessment

## Solution Assessment

Unclear. A config flag is plausible, but the best fix needs maintainer policy on default silence, grouping, per-action classes, and which safety/status confirmations must remain visible.
