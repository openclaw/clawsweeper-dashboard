## Summary

Keep open. Current main still has only narrow noise controls for exec and heartbeat output; it does not provide the requested broad default-silent, suppress, or grouping policy for routine auto-generated chat confirmations across skill command dispatch, cron delivery, skill installation, and generic success replies.

## What This Changes

The issue requests a configurable way to silence or group routine auto-generated chat confirmations such as cron creation, skill installation, and generic done messages.

## Best Possible Solution

Define a coherent notification policy, likely under `messages` or `commands`, that can suppress or group routine success confirmations while preserving explicit failures, approvals, security notices, reminders, and existing narrow controls such as `tools.exec.notifyOnExit`, heartbeat visibility, and cron `delivery.mode`.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: direct tool-dispatched skill commands can still return `✅ Done.`, cron delivery defaults to announce, and skill install returns an `Installed ...` response on current main with tests covering parts of that behavior.

## Solution Assessment

No current solution fully solves the issue. The existing controls are narrow and useful, but the maintainable fix needs one cross-surface policy rather than another one-off suppression flag for a single command path.
