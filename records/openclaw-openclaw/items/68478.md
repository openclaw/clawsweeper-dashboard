## Summary

Keep open. Current main still has visible routine confirmations across command replies, skill install responses, and cron announce defaults, while there is no global suppression or grouping policy and the discussion has fresh user support for a config flag.

## What This Changes

Request to suppress, group, or configure routine auto-generated chat confirmations such as cron, skill-install, and done messages.

## Best Possible Solution

Define a centralized routine-notification policy or config surface that can suppress or batch low-value success confirmations while preserving failures, approvals, security notices, and explicit delivery controls.

## Reproduction Assessment

Yes, by source inspection. Current main still has a visible `✅ Done.` fallback, visible skill-install success text, isolated cron announce defaults, and no global routine-message suppression option in the strict messages config schema.

## Solution Assessment

Unclear. A config flag is plausible, but the maintainable fix should classify routine success confirmations separately from failures, approvals, reminders, and security-sensitive notices rather than add one-off switches for each example.
