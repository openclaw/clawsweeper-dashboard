## Summary

Keep open: current main and v2026.6.10 still expose visible automatic confirmation paths, and changing them needs a maintainer-owned confirmation policy rather than conservative cleanup closure.

## What This Changes

The issue asks for automatic system confirmations such as cron, skill, and done messages to be silent, configurable, or grouped so they stop spamming chat notifications.

## Best Possible Solution

Define a maintainer-approved confirmation/noise policy, then implement a typed suppression or grouping path across inline command replies, session acknowledgements, cron/system events, and skill install responses while preserving safety-critical notices.

## Reproduction Assessment

Yes. Source inspection on current main shows visible confirmation paths for inline skill/tool commands, session reset/new acknowledgements, cron system events, and skill install result messages; I did not run a live Telegram or Discord scenario in this read-only review.

## Solution Assessment

Unclear. The requested direction is valid, but broad silence or a global config flag needs a maintainer policy decision; a typed confirmation class is safer than raw text or channel-specific suppression filters.
