## Summary

Keep open. Current main still has visible routine success confirmations and only narrow per-feature controls, so the requested cross-surface suppression or grouping policy remains unresolved.

## What This Changes

Request a way to make routine autogenerated system messages silent, configurable, or grouped across cron, skill install, and generic done confirmations.

## Best Possible Solution

Define a centralized routine-notification policy or config surface for success confirmations while preserving critical notices and explicit delivery controls.

## Reproduction Assessment

Yes. Source inspection on current main shows empty direct skill tool results fall back to `✅ Done.`, ClawHub install returns visible success text, and cron only has narrow delivery controls rather than a shared routine-message suppression policy.

## Solution Assessment

No current implementation fully solves the issue. The maintainable path is a shared event-class notification policy rather than one-off switches for only cron, skill installs, or command fallback text.
