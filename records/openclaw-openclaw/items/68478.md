## Summary

Keep open. Current main still lacks a shared suppression or grouping policy for routine success confirmations; the available controls are narrower, while direct skill commands, ClawHub skill installs, and isolated cron delivery can still produce visible messages.

## What This Changes

The issue requests a way to silence, disable, or group routine auto-generated success/status messages such as cron creation, skill installation, and Done confirmations.

## Best Possible Solution

Add a centralized notification policy that classifies routine success confirmations separately from critical notices, then wire cron, skill install, and direct command fallback paths through that policy.

## Reproduction Assessment

Yes. Source and test inspection on current main show concrete paths: empty direct skill tool results produce `✅ Done.`, ClawHub installs return `Installed <slug>@<version>`, and isolated cron add defaults to announce delivery unless explicitly disabled.

## Solution Assessment

No current implementation fully solves it. The maintainable direction is a product-approved shared event-class policy rather than isolated one-off flags for cron, skills, or fallback command text.
