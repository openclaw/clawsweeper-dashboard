## Summary

Keep this open: current main still has visible success-confirmation paths for inline skill/tool commands, skill installs, and cron creation results, while the existing visible-reply controls only cover normal agent final replies. The remaining work is a product/config policy decision, not a stale or already-implemented bug fix.

## What This Changes

The issue requests silent-by-default or configurable suppression of automatic system/action confirmation messages across chat channels.

## Best Possible Solution

Define a class-based confirmation policy that suppresses low-value success acknowledgements while preserving errors, approvals, and explicit command responses, then implement it through shared command/reply surfaces with docs and regression tests.

## Reproduction Assessment

Yes, at source level: current main still has visible `✅ Done.`, skill-install success, and cron-add result paths. I did not run a live Telegram session in this read-only review.

## Solution Assessment

No automatic small fix is currently best. Maintainers first need to choose the visibility classes and default/config semantics so suppression does not hide important user-facing outcomes.
