## Summary

Keep open: current main still exposes routine success confirmations and cron announce defaults, while the requested cross-surface suppression/grouping policy needs maintainer UX and config-contract direction.

## What This Changes

Request a global or per-action way to silence or group routine auto-generated system confirmations such as cron, skill-install, and generic done messages.

## Best Possible Solution

Define a centralized routine-notification policy that can suppress or group low-value success confirmations while preserving failures, approvals, security notices, reminders, and explicit delivery controls.

## Reproduction Assessment

Yes. Source inspection on current main shows visible `✅ Done.`, skill-install success text, cron announce defaults, and no global routine-message suppression field in the strict messages schema.

## Solution Assessment

Unclear pending maintainer policy. A config flag is plausible, but the maintainable fix should classify routine success confirmations separately from failures, approvals, security notices, reminders, and explicit cron delivery rather than add one-off string filters.
