## Summary

Keep open: current main still has visible routine success and announce paths, and the messages config/schema still lacks a suppression or grouping contract. The request is valid at source level, but it is a cross-surface notification UX/config feature rather than a narrow bug cleanup.

## What This Changes

Add a way to silence, disable, or group routine auto-generated system messages such as cron, skill-install, and generic success confirmations.

## Best Possible Solution

Define a centralized routine-notification policy and config contract that can suppress or group low-value successes while preserving failures, approvals, security notices, reminders, and explicit delivery controls.

## Reproduction Assessment

Yes, at source level. Current main exposes visible `✅ Done.`, skill-install success text, cron announce defaults, and no routine-success suppression field in the strict messages config schema.

## Solution Assessment

Unclear pending maintainer policy. A config flag is plausible, but the maintainable fix should classify routine confirmations across command replies, skill installs, and cron delivery instead of filtering individual strings.
