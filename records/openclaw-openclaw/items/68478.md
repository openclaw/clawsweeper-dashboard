This is a cross-surface notification/config feature that needs maintainer policy before implementation.

## What This Changes

Request a config or policy to suppress or group routine system-generated success messages such as cron creation, skill install, and Done confirmations across chat channels.

## Best Possible Solution

Define a centralized routine-notification policy and config contract that can suppress or group low-value success confirmations while preserving errors, security-sensitive notices, approvals, reminders, and explicitly requested delivery.

## Reproduction Assessment

Yes, at source level. Current main and v2026.5.18 still expose visible `✅ Done.`, skill-install success text, isolated cron announce defaults, and no routine-success suppression key in the strict messages config schema.

## Solution Assessment

Unclear until maintainer policy is chosen. A single config flag is plausible, but the maintainable solution should classify routine confirmations across command replies, skill installs, cron delivery, and docs instead of filtering individual strings.
