## Summary

Keep open. Current main still has visible routine confirmations and only narrow per-surface controls, while the requested cross-surface suppression/grouping policy remains unresolved and now has fresh user support in the discussion.

## What This Changes

Request a way to silence, control, or group routine auto-generated success/system messages such as cron creation, skill installation, and generic done confirmations.

## Best Possible Solution

Define a centralized routine-notification policy or config surface that can suppress or group low-value success confirmations while preserving critical notices and explicit delivery controls.

## Reproduction Assessment

Yes, by source inspection. Current main still has a visible `✅ Done.` fallback, ClawHub install success text, and isolated cron announce defaults, while the config schema lacks a global routine-message suppression option.

## Solution Assessment

Unclear. A config flag is plausible, but the maintainable fix should classify routine success confirmations separately from failures, approvals, and security-sensitive notices rather than add one-off switches for each example.
