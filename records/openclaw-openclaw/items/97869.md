## Summary

Keep open: the patch looks correct and the new Gateway proof covers the denial/success behavior, but the PR still needs maintainer acceptance because it intentionally removes `/dreaming on|off` access from allowlisted non-owner channel callers.

## What This Changes

The PR gates Memory Core `/dreaming on|off` config mutations behind channel owner status or Gateway `operator.admin`, adds regression coverage, and updates docs for the privilege requirement.

## Best Possible Solution

Land the owner/admin hardening after maintainers explicitly accept the compatibility impact for non-owner channel callers and normal merge gates finish.

## Reproduction Assessment

Yes, from source inspection: current main only blocks Gateway scope arrays that lack `operator.admin`, while no-scope channel invocations fall through to `mutateConfigFile`. I did not run a live current-main reproduction in this read-only review.

## Solution Assessment

Yes, with maintainer acceptance: using `exposeSenderIsOwner` plus the existing Gateway `operator.admin` scope is the narrowest local fix and matches the Phone Control owner/admin mutation pattern.
