## Summary

Keep open: current main has addressed the Telegram isolated-ingress spool drain shape with per-lane handlers, but the Codex half still has a source-backed path where a current-turn `rawResponseItem/completed` for tool output can disarm the short completion watchdog and leave only the 30-minute terminal watchdog. The broader FIFO and Codex timeout work has related open trackers/PRs, but this report contains a specific remaining post-tool-output liveness gap that should not be swept closed yet.

## What This Changes

The issue reports Telegram isolated-ingress head-of-line blocking and Codex app-server post-tool-output silence that leads to a 30-minute idle timeout.

## Best Possible Solution

Keep the current Telegram per-lane drain behavior, route broader command FIFO work through https://github.com/openclaw/openclaw/issues/79589 and https://github.com/openclaw/openclaw/pull/75299, and fix Codex run-attempt liveness so raw custom-tool-output completion followed by silence fails fast instead of waiting for the terminal 30-minute guard.

## Reproduction Assessment

Yes, source-reproducible for the remaining Codex half: on current main a same-turn `rawResponseItem/completed` after tool output can still disarm the short completion watchdog and leave the 30-minute terminal watchdog as the next guard. The Telegram isolated-ingress half is not reproduced on current main because per-lane drain code and tests now cover interleaving.

## Solution Assessment

No, current main is only a partial solution. The narrowest maintainable path is a focused Codex app-server watchdog fix, possibly folded into the active progress-aware timeout work if maintainers want one liveness PR.
