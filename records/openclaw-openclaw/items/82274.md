## Summary

Keep open: the Telegram HOL part appears addressed on current main, but the Codex stall remains source-reproducible because current `run-attempt` logic can still fall back to the 30-minute terminal watchdog after a current-turn `rawResponseItem/completed` for tool-output bookkeeping.

## What This Changes

The issue asks OpenClaw to decouple Telegram isolated-ingress spool draining from long agent turns and to fail fast or diagnose Codex app-server JSON-RPC silence after `custom_tool_call_output`.

## Best Possible Solution

Keep the current Telegram per-lane isolated-ingress drain, and add a focused Codex app-server watchdog fix so `custom_tool_call_output` raw-response bookkeeping after an OpenClaw tool result does not defer recovery to the 30-minute terminal timeout.
