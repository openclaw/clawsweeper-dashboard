## Summary

Keep open. Current main still has a source-visible package/runtime dependency gap: the image tool can reach the sharp-backed optimizer, the root package does not declare `sharp`, and the packaged postinstall explicitly does not install plugin package dependencies.

## What This Changes

The issue requests fixing default image-tool attachment processing when the sharp-backed optimizer is missing from a packaged install.

## Best Possible Solution

Make packaged default installs resolve the media-understanding image optimizer, or provide a safe raw-image fallback for cases that do not require resizing, with package acceptance coverage for the installed package.

## Reproduction Assessment

Yes, with high confidence from source inspection and reporter live confirmation. A JPEG that requires optimization reaches `loadWebMedia`, then the media-understanding `sharp` import, while the root package does not install that dependency by default.

## Solution Assessment

Yes, the best direction is a package/runtime dependency fix with installed-package coverage, not a Telegram-specific workaround. Preserve the current in-limit fallback, but make default packaged installs work when optimization is required.
