## Summary

Keep open. Current main still has a credible `status --json` diagnostic gap, and the issue is paired with an open fixing PR that references it with closing syntax.

## What This Changes

The issue requests that `openclaw status --json` include a configured active Telegram channel in `channelSummary` instead of returning an empty array.

## Best Possible Solution

Update the JSON status path so configured setup-backed channels can appear in `channelSummary` without activating runtimes, then land or replace the linked fix PR with regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows `statusCommand({ json })` calls `scanStatusJsonFast`, which passes `includeChannelSummary: false`; the lower-level summary loader also disables setup-fallback channel plugin resolution.

## Solution Assessment

Unclear. Enabling setup fallback is likely necessary for summary callers, but the reported CLI command also needs the fast JSON contract changed or explicitly documented.
