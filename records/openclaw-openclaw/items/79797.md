## What This Changes

`openclaw status --json` should include a configured, working Telegram channel in `channelSummary` instead of returning an empty array.

## Best Possible Solution

Keep this issue paired with https://github.com/openclaw/openclaw/pull/79801; if maintainers want `channelSummary` populated in lean JSON output, update the fast JSON path and setup-fallback summary resolution without starting channel runtimes, then add focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `openclaw status --json` reaches `scanStatusJsonFast`, which passes `includeChannelSummary: false`, so `getStatusSummary` returns an empty `channelSummary`; I did not run a live WSL2 Telegram setup.

## Solution Assessment

Unclear. Enabling setup fallback is relevant for callers that reach `buildChannelSummary`, but the reported CLI path also disables channel summary entirely, so the maintainable fix should first settle the lean JSON contract and then cover both code paths.
