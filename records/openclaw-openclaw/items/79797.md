## Summary

Keep open. Current main still leaves the reported `status --json` path on the lean fast scanner with `channelSummary` disabled, while the lower-level summary resolver would still skip setup-fallback channel plugins such as Telegram if that path were enabled.

## What This Changes

The issue asks for `openclaw status --json` to populate `channelSummary` for a configured, active Telegram channel instead of returning an empty array.

## Best Possible Solution

Define the intended `status --json` contract, then either populate `channelSummary` from cold read-only setup-fallback metadata without runtime activation or document/remove the empty field from the lean default JSON payload.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: `openclaw status --json` reaches `scanStatusJsonFast`, which passes `includeChannelSummary: false`, so a configured Telegram channel can still yield an empty `channelSummary` without entering the summary builder.

## Solution Assessment

Unclear. The reporter's setup-fallback resolver direction is plausible for the lower-level summary builder, but maintainers first need to decide whether the lean default JSON contract should include channel summaries at all.
