## What This Changes

The issue reports that `openclaw status --json` returns an empty `channelSummary` even when Telegram is configured, enabled, and actively working.

## Best Possible Solution

Review and land a focused fix that makes configured setup-fallback channels appear in `openclaw status --json` without starting channel runtimes, or explicitly document/remove the field if lean JSON output is intended to stay empty.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence current-main path: `status --json` uses the fast scanner with `includeChannelSummary: false`, and the lower-level summary builder also disables setup-fallback plugin resolution; I did not run a live Telegram setup.

## Solution Assessment

Unclear. The setup-fallback resolver change in the linked PR is relevant, but the reported CLI path also needs the fast JSON contract addressed or explicitly scoped by maintainers.
