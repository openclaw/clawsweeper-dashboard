## Summary

Keep open. Current main and the latest release still leave the reported `status --json` surface unresolved: the fast JSON path disables `channelSummary`, and the lower-level summary resolver still opts out of setup-fallback channel plugins. The two related fix PRs were closed unmerged, and an open performance PR favors keeping default JSON lean, so the remaining work needs maintainer contract direction before a repair lane should change behavior.

## What This Changes

The issue asks for `openclaw status --json` to include a Telegram entry in `channelSummary` when Telegram is configured and active.

## Best Possible Solution

Define the lean `status --json` contract, then either populate `channelSummary` from read-only setup-fallback metadata without live probes or explicitly document that the default JSON path omits channel summaries.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: `openclaw status --json` routes through `scanStatusJsonFast`, which passes `includeChannelSummary: false`, so the field can remain empty even when Telegram is configured.

## Solution Assessment

Unclear. Enabling setup fallback in `buildChannelSummary` is necessary for setup-backed channels, but it does not settle whether the default fast JSON path should call that summary path at all.
