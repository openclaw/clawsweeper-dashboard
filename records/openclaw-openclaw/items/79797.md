## What This Changes

The issue reports that `openclaw status --json` returns an empty `channelSummary` even when a Telegram channel is configured, enabled, and working.

## Best Possible Solution

Yes. Source inspection on current main shows `status --json` reaches the fast JSON path with `includeChannelSummary: false`, and the lower-level summary resolver still excludes setup-fallback plugins when reached.

Unclear. Enabling setup fallback is likely necessary for summary callers, but the reported command also has a fast-path contract issue; the safest fix should handle both paths or document the omission intentionally.
