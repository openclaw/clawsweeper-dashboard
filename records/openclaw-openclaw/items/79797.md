## Summary

Keep open: current main and the latest release still keep default `status --json` off channel-summary construction, and the setup-fallback resolver path still stays disabled when summaries are built. The linked implementation attempts were closed unmerged, while the merged status JSON scan PR fixed a related performance path without deciding this public JSON contract.

## What This Changes

The issue reports that `openclaw status --json` returns an empty `channelSummary` even when Telegram is configured, enabled, and working.

## Best Possible Solution

Define the `status --json` channel-summary contract, then implement a lightweight configured-channel summary path or explicitly reserve populated channel summaries for `--all` with regression coverage.

## Reproduction Assessment

Yes, by source inspection. Default fast `status --json` sets `includeChannelSummary: false`, and when channel-summary construction is reached the resolver still disables setup fallback; no live Telegram setup was run in this read-only review.

## Solution Assessment

Unclear. Enabling setup fallback is relevant for summary callers, but the reported CLI command currently bypasses channel-summary construction by policy, so maintainers should first decide whether default JSON status must include configured channel rows.
