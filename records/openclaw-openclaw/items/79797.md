## Repair Work Prompt

Fix the `openclaw status --json` channelSummary bug for configured setup-fallback channels. On current main, `scanStatusJsonFast` passes `includeChannelSummary: false`, and `buildChannelSummary` resolves read-only channel plugins with `includeSetupFallbackPlugins: false`; configured Telegram/setup-fallback channels can therefore produce an empty `channelSummary`. Make the existing JSON status payload populate the current `string[]` channelSummary for configured channels without activating channel runtimes, and resolve summary plugins with setup fallback enabled. Update the fast JSON test that currently asserts the summary is disabled, add focused coverage for a configured Telegram/setup-fallback channel and for forwarding the resolver option, and do not change Telegram delivery/auth behavior, plugin registry contracts, or add a new config flag. Add a user-facing `CHANGELOG.md` Fixes entry unless maintainers decide to handle changelog only during landing.
