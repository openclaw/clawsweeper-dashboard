## Summary

Keep open. Current main still has the reported diagnostic gap: the default `status --json` path disables channel summaries before plugin resolution runs, and the lower-level summary resolver still disables setup-fallback channel plugins such as Telegram.

## What This Changes

Request that `openclaw status --json` include Telegram in `channelSummary` when Telegram is configured, enabled, and active.

## Best Possible Solution

Define the intended `status --json` contract, then either populate `channelSummary` through cheap read-only setup metadata without live probes or document/remove the empty field from the lean JSON payload.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: `status --json` reaches `scanStatusJsonFast`, which passes `includeChannelSummary: false`, and the fallback resolver would still skip Telegram setup plugins if invoked.

## Solution Assessment

Unclear. Enabling setup fallback is necessary for the lower-level summary builder, but it is not sufficient for the reported CLI JSON command because the fast JSON path disables channel summaries first.
