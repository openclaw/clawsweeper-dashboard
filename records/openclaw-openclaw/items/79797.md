## Summary

Keep open. Current main still has a credible `status --json` diagnostics gap, and the linked fix PR is open rather than merged and appears to miss the CLI fast path that the report names.

## What This Changes

The issue requests that `openclaw status --json` include a Telegram entry in `channelSummary` when Telegram is configured, enabled, and working.

## Best Possible Solution

Define the JSON status contract, then either populate `channelSummary` from cold read-only setup-fallback metadata for configured channels or remove/document the field for the lean CLI path.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `openclaw status --json` reaches `scanStatusJsonFast`, which passes `includeChannelSummary: false`, so configured Telegram can still produce an empty `channelSummary`.

## Solution Assessment

Unclear. Enabling setup fallback in `buildChannelSummary` is a plausible part of the fix, but it is not sufficient for the reported CLI path unless the fast JSON policy also requests channel summaries.
