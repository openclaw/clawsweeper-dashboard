## Summary

Keep open. Current main still has the reported diagnostic gap: the CLI JSON path disables channel summaries, and the lower-level summary resolver would skip setup-fallback plugins such as Telegram when it is reached. An open fixing PR references this issue, so the issue should wait for that PR to be corrected, landed, or closed.

## What This Changes

The issue asks for `openclaw status --json` to populate `channelSummary` for configured, active Telegram channels instead of returning an empty array.

## Best Possible Solution

Define the intended `status --json` contract, then land a fix that includes configured setup-backed channels in `channelSummary` without starting channel runtimes and adds regression coverage for the CLI fast path.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `statusCommand({ json })` reaches `scanStatusJsonFast`, which sets `includeChannelSummary: false`, and the lower-level summary resolver also disables setup fallback for Telegram-style setup plugins.

## Solution Assessment

Unclear. Enabling setup fallback in `buildChannelSummary` is necessary for summary callers that reach it, but the reported CLI command also needs the fast JSON path handled or the JSON contract documented differently.
