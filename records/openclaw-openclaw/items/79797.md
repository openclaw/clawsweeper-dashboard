## Summary

Keep open. Current main still has the reported source-level diagnostic gap for `openclaw status --json`: the routed JSON command uses the lean fast path that disables channel summaries, and the lower-level summary resolver still excludes setup-fallback channel plugins such as Telegram when it is reached. The open linked fix PR https://github.com/openclaw/openclaw/pull/82600 uses closing syntax, so this issue should remain open until that PR is correctly reworked/landed or maintainers explicitly define the lean JSON contract.

## What This Changes

The issue asks for `openclaw status --json` to include a Telegram `channelSummary` entry when Telegram is configured, enabled, and working.

## Best Possible Solution

Define the `status --json` channel-summary contract, then either document the lean empty-summary behavior or populate `channelSummary` through read-only setup-fallback metadata without starting channel runtimes.

## Reproduction Assessment

Yes, at source level. Routed `status --json` reaches `scanStatusJsonFast`, which disables channel summaries, and the lower-level summary resolver still excludes setup-fallback plugins when reached; I did not run a live Telegram setup.

## Solution Assessment

Unclear. The currently linked PR does not fix the actual fast JSON path and has its own review blockers, so the safer path is a maintainer contract decision followed by focused changes to the fast JSON and summary resolver paths.
