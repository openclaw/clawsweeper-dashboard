Keep open. Current main and the latest release still skip `channelSummary` on the fast JSON status path, and this issue is paired with an open PR using closing syntax, so it should stay open while that PR is reviewed or the JSON status contract is decided.

## What This Changes

The issue requests that `openclaw status --json` include a Telegram entry in `channelSummary` when Telegram is configured and working.

## Best Possible Solution

Define the JSON status contract, then either populate `channelSummary` through read-only setup-fallback metadata without starting channel runtimes or document that plain JSON intentionally omits it and expose the intended diagnostic mode.
