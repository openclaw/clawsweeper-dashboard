## Summary

Keep open. Current main still has the reported diagnostic gap: the CLI `status --json` path deliberately disables `channelSummary`, and the lower-level summary builder still disables setup-fallback plugin resolution for channels such as Telegram. The two related fix PRs were closed unmerged, and the earlier narrow fallback-only patch did not fully cover the reported CLI path.

## What This Changes

`openclaw status --json` should include a Telegram `channelSummary` entry when Telegram is configured, enabled, and working.

## Best Possible Solution

Define the intended `status --json` contract, then either populate `channelSummary` from read-only setup-fallback metadata without activating channel runtimes or document/remove the empty field from the lean JSON output.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `status --json` uses `scanStatusJsonFast`, which passes `includeChannelSummary: false`, and the fallback summary resolver also excludes setup-fallback channel plugins.

## Solution Assessment

Unclear. Enabling setup fallback is necessary for callers that reach `buildChannelSummary`, but the reported CLI command also needs a deliberate decision on the fast JSON summary policy.
