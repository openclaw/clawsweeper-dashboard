## Summary

Keep open. Current main still has the reported diagnostic gap: `status --json` disables channel-summary loading on the fast path, and the lower-level summary resolver still excludes setup-fallback channel plugins such as Telegram; both linked fix PRs were closed unmerged.

## What This Changes

The issue asks for `openclaw status --json` to include a Telegram `channelSummary` entry when Telegram is configured, enabled, and working.

## Best Possible Solution

Define the JSON status contract, then either populate `channelSummary` through read-only setup-fallback metadata without starting channel runtimes or document that plain `status --json` intentionally omits channel summaries.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: `status --json` reaches `scanStatusJsonFast`, which disables channel summaries, and the summary resolver also disables setup-fallback plugins. I did not run a live Telegram setup.

## Solution Assessment

Unclear. Flipping the setup-fallback resolver option is necessary for summary callers that reach it, but the reported CLI path also needs a maintainer decision on whether the lean JSON path should load channel summaries.
