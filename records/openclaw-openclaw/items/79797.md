## Summary

Keep open. Current main still source-reproduces the empty `channelSummary` path for `openclaw status --json`, and https://github.com/openclaw/openclaw/pull/79801 already references this issue with closing syntax, so this issue should wait for that PR to be updated, landed, or closed.

## What This Changes

The issue asks for `openclaw status --json` to include configured active Telegram channels in `channelSummary` instead of returning an empty array.

## Best Possible Solution

Decide the default JSON status contract, then either update the fast JSON path plus setup-fallback summary resolution with focused regression coverage and redacted real CLI proof, or document/remove the empty field from the lean JSON payload.

## Reproduction Assessment

Yes. Source inspection shows `openclaw status --json` reaches `scanStatusJsonFast`, which passes `includeChannelSummary: false`, so `channelSummary` can remain `[]` before Telegram setup fallback resolution is considered; I did not run a live WSL2 Telegram setup.

## Solution Assessment

Unclear. The suggested setup-fallback change is relevant for callers that reach `buildChannelSummary`, but the reported CLI command also needs a maintainer decision on whether the lean JSON fast path should include channel summary data.
