## Summary

Keep open. Current main still has a source-visible path where `openclaw status --json` can return an empty `channelSummary`, and the linked open fixing PR has not landed and appears incomplete for the reported CLI path.

## What This Changes

The issue asks for `openclaw status --json` to include a configured, active Telegram channel in `channelSummary` instead of returning an empty array.

## Best Possible Solution

Keep this issue open until the linked PR is amended and landed, or superseded by a fix that populates `channelSummary` for `openclaw status --json` from lightweight setup-fallback metadata without starting channel runtimes.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `status --json` calls `scanStatusJsonFast`, which passes `includeChannelSummary: false`, so the summary returns `channelSummary: []` before Telegram setup-fallback resolution can run.

## Solution Assessment

Unclear. Enabling setup fallback in `buildChannelSummary` is likely necessary for summary callers, but it is not sufficient for the reported CLI JSON path while the fast JSON policy disables channel summaries entirely.
