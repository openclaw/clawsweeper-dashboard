## Summary

Keep open. Current main and the latest release still suppress `channelSummary` on the `status --json` fast path, and the lower-level channel summary resolver still disables setup-fallback plugins, so the reported Telegram diagnostic gap remains credible.

## What This Changes

The issue reports that `openclaw status --json` leaves `channelSummary` empty even when Telegram is configured, enabled, and active.

## Best Possible Solution

Keep this issue open and define a lightweight `status --json` channel summary path that includes configured setup-backed channels without full plugin/runtime warmup, then update the fast-path and resolver tests together.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `status --json` reaches `scanStatusJsonFast`, which passes `includeChannelSummary: false`, and the lower-level summary resolver also disables setup fallback plugins.

## Solution Assessment

Unclear. Flipping only `includeSetupFallbackPlugins` in `buildChannelSummary` is not enough because the reported CLI path disables channel summaries before that function runs; the maintainable fix should preserve the lean JSON path while restoring configured-channel summary output.
