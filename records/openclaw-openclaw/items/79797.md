## Summary

Keep open. Current main still has a credible diagnostic gap: the `status --json` fast path suppresses `channelSummary`, and the lower-level summary loader would also skip setup-fallback channel plugins such as Telegram. Because tests currently codify the lean JSON behavior, this needs maintainer direction before a repair PR.

## What This Changes

The issue requests that `openclaw status --json` include Telegram in `channelSummary` when Telegram is configured and active.

## Best Possible Solution

Define the intended JSON contract, then either populate `channelSummary` from read-only setup-fallback metadata without live probes or document/remove the empty field from the lean JSON path.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: `status --json` uses the fast JSON path with `includeChannelSummary: false`, so the field can remain empty regardless of configured Telegram. I did not run a live Telegram setup in this read-only review.

## Solution Assessment

Unclear. The suggested setup-fallback resolver change is plausible for the lower-level summary builder, but the current JSON path intentionally disables channel summaries, so maintainers should first decide whether the lean JSON contract should change.
