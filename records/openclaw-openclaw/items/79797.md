## Summary

Keep open: current main and v2026.6.10 still keep default `status --json` on the lean path with channel-summary construction disabled, and the setup-fallback resolver remains disabled when summaries are built. The linked fix attempts were closed unmerged, while merged status JSON work addressed performance and metadata scanning without deciding this JSON contract.

## What This Changes

The issue asks for `openclaw status --json` to include a Telegram entry in `channelSummary` when Telegram is configured and active instead of returning an empty array.

## Best Possible Solution

Define the public `status --json` channel-summary contract, then either add a lightweight configured-channel summary path that does not start channel runtimes or explicitly reserve populated channel summaries for a richer status mode and document that behavior.

## Reproduction Assessment

Yes, by source inspection. Default fast `status --json` sets `includeChannelSummary: false`, and the channel-summary resolver still disables setup fallback when reached; no live Telegram setup was run in this read-only review.

## Solution Assessment

Unclear. Enabling setup fallback is relevant for summary callers, but the reported CLI command currently bypasses channel-summary construction by policy, so maintainers should first decide whether default JSON status must include configured channel rows.
