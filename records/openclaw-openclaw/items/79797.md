## Summary

Keep open. Current main still routes `openclaw status --json` through a fast JSON path that disables `channelSummary`, and the linked implementation PR at https://github.com/openclaw/openclaw/pull/79801 is open rather than merged.

## What This Changes

Report that `openclaw status --json` returns an empty `channelSummary` even when Telegram is configured, enabled, and delivering messages.

## Best Possible Solution

Decide and codify the JSON status contract; if `channelSummary` belongs in default `status --json`, update both the fast JSON path and read-only setup-fallback resolver with regression coverage, otherwise document the lean default and expose the populated summary through `--all` or channel-specific status commands.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence current-main path: `status --json` reaches `scanStatusJsonFast`, which passes `includeChannelSummary: false`, and the lower summary resolver would also skip setup-fallback channel plugins if reached.

## Solution Assessment

Unclear. The linked PR’s setup-fallback change is relevant for summary callers, but the reported CLI path also needs an explicit contract decision because default JSON currently avoids loading channel summaries by design.
