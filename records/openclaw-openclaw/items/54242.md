## Summary

Keep open. Current main and the latest release still rewrite only tar entry paths during backup creation and verify only tar entry paths, while the hardlink target lives in `linkpath`; the related exclude and backup-design items do not close this restore-safety gap.

## What This Changes

The issue asks OpenClaw backup creation and verification to avoid or reject tar hardlink targets containing traversal segments so broad archive extraction succeeds on macOS tar.

## Best Possible Solution

Fix the backup writer and verifier so hardlink targets are restore-safe or rejected, with regression coverage for a hardlinked payload whose generated tar `linkpath` would otherwise contain `..`.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: back up a tree containing hardlinked files where `tar@7.5.15` emits a `Link` entry with a relative `linkpath` containing `..`; current create and verify paths inspect only `entry.path`. ## Solution Assessment

Yes. The narrow maintainable fix belongs in backup archive emission plus verification; configurable excludes and a broader restore command are related but separate feature/design tracks.
