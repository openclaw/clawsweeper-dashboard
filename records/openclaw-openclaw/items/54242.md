## Summary

Keep open. Current main and v2026.5.7 still remap only tar entry paths and verify only entry paths, while `tar@7.5.15` stores hardlink targets separately in `linkpath`; no open or merged PR was found that fixes this specific gap.

## What This Changes

The issue requests restore-safe backup creation or verification for tar hardlink targets containing `..` so broad macOS `bsdtar` extraction does not fail after `backup verify` passes.

## Best Possible Solution

Keep this open until backup creation emits restore-safe hardlink entries or dereferences unsafe hardlinks, and `backup verify` rejects unsafe tar `linkpath` values with regression coverage.

## Reproduction Assessment

Yes. A high-confidence source reproduction is a backup source tree with hardlinked files where `tar@7.5.15` emits a `Link` entry whose target is stored in `linkpath`, while current create and verify paths inspect only `entry.path`.

## Solution Assessment

Yes. The narrow maintainable fix is in backup archive emission and verification; configurable exclusions and service-aware backup redesign belong to the separate related feature issues.
