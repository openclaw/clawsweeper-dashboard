## Summary

Keep open: current main still has the reported cold-registry gap for config-path plugin tools, and the linked fixing PR is open rather than merged.

## What This Changes

Path-based plugins loaded through `plugins.load.paths` should expose their registered agent tools after upgrade to `v2026.5.2`.

## Best Possible Solution

Review and land PR #76609 or an equivalent scoped fallback in `resolvePluginTools`, with regression coverage and a changelog entry.

## Reproduction Assessment

Yes. The source path on current main is enough: a manifest-selected tool plugin can be absent from loaded registries, and `resolvePluginTools` returns without invoking the existing scoped loader.

## Solution Assessment

Yes. The narrow maintainable fix is to keep the loaded-registry and descriptor-cache fast paths, then load only the remaining manifest-selected tool plugin ids when the registry is cold.
