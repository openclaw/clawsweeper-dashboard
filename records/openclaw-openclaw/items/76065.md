Keep this PR open; the proposed dependency override is not merge-ready because it forces Jimp runtime packages onto a Zod major that their published dependency metadata does not declare compatible.

## What This Changes

The PR adds a repo-wide `pnpm.overrides.zod` entry and regenerates `pnpm-lock.yaml` so Zod 3 transitives, including `@jimp/*`, resolve to Zod 4.4.1.

## Best Possible Solution

Prefer an upstream-compatible Jimp/Baileys dependency update or narrowly tested package-resolution fix that preserves declared dependency contracts unless a maintainer explicitly approves the override.
