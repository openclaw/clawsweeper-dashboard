## Summary

Keep this PR open, but it is not merge-ready as written because it forces a runtime dependency major version that the upstream Jimp packages do not declare support for.

## What This Changes

The PR adds a repo-wide `pnpm.overrides.zod` entry and regenerates the lockfile so `@jimp/*` transitives and other Zod consumers resolve to Zod 4.

## Best Possible Solution

Prefer an upstream-compatible Jimp/Baileys dependency update or narrowly tested package-resolution fix that preserves declared dependency contracts unless a maintainer explicitly approves the override.

## Reproduction Assessment

Unclear for the alleged type conflict: the dual-version dependency state is visible in the current lockfile, but the PR does not provide a concrete TypeScript error or runtime stack to reproduce. I did not run install or tests because this review was constrained to read-only inspection.

## Solution Assessment

No. Forcing all Zod consumers to v4 is broader than the reported Jimp transitive issue and overrides Jimp packages outside their declared runtime dependency range.
