## Summary

Keep this PR open for maintainer review. Current main still has the root `zod` dependency on `^4.4.1` while @jimp transitives resolve to `zod@3.25.76`, so the PR targets a real dependency-state mismatch, but it changes root dependency resolution for runtime packages outside their declared Zod range and should not be auto-cleaned up or auto-merged.

## What This Changes

The PR adds a root pnpm override for `zod` at `^4.4.1` and regenerates `pnpm-lock.yaml` so @jimp and other Zod entries resolve to `zod@4.4.1`.

## Best Possible Solution

Merge only after maintainer approval of the dependency-policy tradeoff, preferably with an exact `zod` override, regenerated lockfile, and install/build plus affected WhatsApp/Baileys or package validation; otherwise keep the existing dual-Zod lockfile until a concrete type/runtime failure is reproduced.

## Reproduction Assessment

Yes for the dependency-state mismatch: `origin/main` has root `zod@^4.4.1` and @jimp lockfile snapshots on `zod@3.25.76`. The claimed type conflict itself is not fully reproduced here because `pnpm` is unavailable for `tsgo` or test validation.

## Solution Assessment

Unclear. The override would remove the dual Zod versions and sampled @jimp source appears to use Zod APIs still present in v4, but a root override outside upstream @jimp's declared range is broad enough that maintainer approval and targeted validation are the safer path.
