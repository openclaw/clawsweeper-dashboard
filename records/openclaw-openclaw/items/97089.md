## Summary

Keep open: current `main` and latest release still have the `sessions_yield` early return that blocks the descendant-settle wake path, the issue has a protected `maintainer` label, and the direct closing PR is still open.

## What This Changes

The issue reports that a parent session paused by `sessions_yield` can remain idle after descendant subagents settle instead of receiving the completion wake-up.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/97090 or an equivalent narrow guard-and-regression fix, then close this issue after the fix reaches `main`.

## Reproduction Assessment

Yes. Current `main` has a source-level reproduction path: descendant cleanup sets `wakeOnDescendantSettle`, but `resumeSubagentRun` returns for all `sessions_yield` pauses before it can invoke the wake/announce cleanup path.

## Solution Assessment

Yes. The narrow best fix is to preserve ordinary yielded pauses while allowing only `wakeOnDescendantSettle` yielded orchestrators through the existing descendant wake path, with regression coverage for the parent wake.
