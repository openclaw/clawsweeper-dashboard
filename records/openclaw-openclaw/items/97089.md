## Label Justifications

- `P1`: The report describes a broken agent orchestration workflow where completed subagent work can leave the yielded parent idle until manual re-entry.
- `impact:session-state`: The parent session can remain paused while descendant state has already settled, causing session state drift until a later manual turn reconciles it.
- `impact:message-loss`: Completion events that should resume the yielded parent can be suppressed or delayed instead of arriving as the next message.

## Summary

Keep open: current main and v2026.6.10 still have the `sessions_yield` early return that blocks the descendant-settle wake path, the issue has a protected `maintainer` label, and the direct closing fix is still an open PR.

## What This Changes

The issue asks OpenClaw to wake or coalesce completion events for a yielded parent session after relevant subagent descendants settle.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/97090 or an equivalent narrow fix after maintainer review, then close this issue once the fix reaches main.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: current main sets `wakeOnDescendantSettle` during descendant deferral, but `resumeSubagentRun` returns early for all `sessions_yield` pauses before the wake path.

## Solution Assessment

Yes. The narrowest maintainable fix is to let only `wakeOnDescendantSettle` yielded orchestrators through the existing descendant wake path, while leaving ordinary yielded runs paused until explicit steer.
