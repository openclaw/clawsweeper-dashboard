## Label Justifications

- `P1`: The report describes a broken agent orchestration workflow where completed subagent work can leave a yielded parent idle until manual re-entry.
- `impact:session-state`: The yielded parent can remain paused while descendant state has settled, leaving session state stale until a later turn reconciles it.
- `impact:message-loss`: Completion events that should wake the yielded parent can be delayed or suppressed instead of arriving as the next message.

## Summary

Keep open: current `main` and `v2026.6.10` still have the `sessions_yield` early return that blocks the descendant-settle wake path, the issue has a protected `maintainer` label, and the direct closing fix is still open at https://github.com/openclaw/openclaw/pull/97090.

## What This Changes

The issue reports that a parent session paused by `sessions_yield` can remain idle after descendant subagents settle instead of being woken with completion context.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/97090 or an equivalent narrow fix after maintainer review, then close this issue once the fix reaches `main`.

## Reproduction Assessment

Yes. The source path is high-confidence: current `main` sets `wakeOnDescendantSettle` during descendant deferral, but `resumeSubagentRun` returns early for all `sessions_yield` pauses before the ended-run wake path.

## Solution Assessment

Yes. The narrow best fix is to let only yielded orchestrators marked for descendant-settle wake reach the existing cleanup/wake path while ordinary yielded runs remain paused.
