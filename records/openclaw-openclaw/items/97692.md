## Summary

Current main still lacks the requested memory-core dreaming crash-recovery contract: dream events are completed-only, there is no started/interrupted event pairing, and partial inline writes are not tagged. Cron startup recovery now marks interrupted cron jobs failed, but that only covers scheduler state, so this issue still has meaningful remaining product/API work.

## What This Changes

Request crash recovery for memory-core dreaming phases so interrupted phase runs are detected, recorded, and partial writes can be distinguished on restart.

## Best Possible Solution

Keep this open for a maintainer decision on a bounded memory-core/host event contract for interrupted dreaming runs, reusing existing cron-level recovery without duplicating scheduler state.

## Solution Assessment

Unclear: started/completed event pairing is plausible, but the best design needs maintainer agreement on the memory-host event contract and partial-write semantics.
