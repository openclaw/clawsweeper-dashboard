## Summary

Keep open: current main and v2026.6.10 still lack a `memory.dream.started` or interrupted-run recovery contract, and the related open PR only covers overlapping outcome normalization for completed events.

## What This Changes

Add dreaming-phase crash recovery that records interrupted runs and makes partial dreaming writes detectable on startup.

## Best Possible Solution

Define an additive memory-core dreaming lifecycle contract that preserves existing completed-event readers, coordinates with outcome normalization, and handles startup orphan recovery plus partial-write policy in the owning memory plugin.
