## Summary

Keep open: the linked Docker regression is real enough to keep this PR in play, but the proposed Compose-only fix is likely incomplete because current runtime workspace resolution does not read `OPENCLAW_WORKSPACE_DIR` after `agents.defaults.workspace` has a host path.

## What This Changes

The PR adds container-side `OPENCLAW_CONFIG_DIR` and `OPENCLAW_WORKSPACE_DIR` overrides to both Docker Compose services, a Docker setup regression assertion, and a changelog entry.

## Best Possible Solution

Repair Docker setup so host `.env` values remain only bind-mount sources while OpenClaw config/runtime inside the container stores and uses `/home/node/.openclaw/workspace`, then cover that behavior with a regression test beyond Compose string matching.

## Reproduction Assessment

Yes by source inspection for the failure once a host workspace is persisted: `resolveAgentWorkspaceDir` returns `agents.defaults.workspace`, and the reply path creates that path. The exact live Docker flow was not rerun, and current source does not support the PR's claim that runtime reads `OPENCLAW_WORKSPACE_DIR` directly.

## Solution Assessment

No. Service-level env overrides are useful defense-in-depth, but they are not the best complete fix because the failing reply path uses the persisted workspace config rather than those env vars.
