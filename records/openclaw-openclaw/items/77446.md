## Summary

Keep this PR open: current main still lets host-side Docker setup path values from `.env` enter both container environments, while the PR is a focused Compose boundary fix with no discrete correctness or security findings from this read-only review.

## What This Changes

The branch adds container-local `OPENCLAW_CONFIG_DIR` and `OPENCLAW_WORKSPACE_DIR` overrides to both Compose services, adds a Docker setup contract test, and records the macOS Docker regression fix in the changelog.

## Best Possible Solution

Land the focused Compose env override after CI is green or triaged, keeping host paths only for bind-mount sources and container paths for runtime configuration.

## Solution Assessment
