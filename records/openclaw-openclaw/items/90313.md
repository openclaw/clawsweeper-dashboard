## Summary

Keep open. Current main still does not fully solve the reported Dreaming corpus pollution: archive handling is only partially mitigated by message dedupe, and cron classification still does not follow subagent lineage; the open closing pull request at https://github.com/openclaw/openclaw/pull/93187 is the right implementation candidate to review before this issue closes.

## What This Changes

Prevent Dreaming session-corpus ingestion from admitting reset/deleted archive transcripts and from treating cron-spawned subagent sessions as ordinary user sessions.

## Best Possible Solution

Land a focused fix that filters Dreaming corpus archive inputs and classifies cron descendants through stored lineage such as `spawnedBy` or `parentSessionKey`, while preserving usage-counted archive indexing for non-Dreaming callers.

## Reproduction Assessment

Yes. Current main is source-reproducible: Dreaming still relies on usage-counted session listing plus session-key-only cron classification, so cron-spawned subagents with only lineage metadata are not excluded by the existing classifier.

## Solution Assessment

Yes, the best direction is a Dreaming-specific archive eligibility boundary plus lineage-aware cron classification. A shared-helper swap alone would risk breaking the documented memory_search archive-indexing contract.
