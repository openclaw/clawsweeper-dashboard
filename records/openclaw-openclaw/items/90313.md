## Summary

Keep open. Current main still leaves both reported Dreaming corpus defects source-reproducible, and the open closing pull request at https://github.com/openclaw/openclaw/pull/93187 is the right implementation candidate to review or replace.

## What This Changes

The issue reports Dreaming session-corpus pollution from archived session transcripts and cron-descended subagent transcripts that are not classified as generated by cron.

## Best Possible Solution

Land a focused fix that filters Dreaming archive inputs and classifies cron descendants through stored lineage such as `spawnedBy` or `parentSessionKey`, while preserving usage-counted archive indexing for `memory_search` and QMD.

## Reproduction Assessment

Yes. Current main is source-reproducible: Dreaming consumes the usage-counted session listing and cron classification still checks only direct cron session keys, so cron-descended subagents with lineage metadata are not excluded.

## Solution Assessment

Yes. The best direction is a Dreaming-specific archive eligibility boundary plus lineage-aware cron classification; changing the shared usage-counted helper alone would risk breaking `memory_search` and QMD archive indexing.
