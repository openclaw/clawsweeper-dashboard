## Summary

Keep open. Current main and the latest release still leave both reported Dreaming corpus defects source-reproducible, and the open closing pull request at https://github.com/openclaw/openclaw/pull/93187 is the active implementation candidate rather than a reason to close this issue early.

## What This Changes

The issue asks memory-core Dreaming to avoid re-ingesting reset/deleted archive transcripts and to classify cron-descended subagent transcripts through session parentage.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/93187 with a focused fix that filters Dreaming to primary transcripts and classifies cron descendants through stored lineage while preserving usage-counted archive indexing for QMD and memory search.

## Reproduction Assessment

Yes. Current main and v2026.6.9 are source-reproducible: Dreaming consumes the usage-counted transcript listing, and cron classification checks only direct cron session keys even though subagent lineage is stored as `spawnedBy`.

## Solution Assessment

Yes. The maintainable fix is a Dreaming-only primary-transcript boundary plus SDK-owned lineage-aware cron classification; narrowing the shared usage-counted lister would risk breaking QMD and memory-search archive indexing.
