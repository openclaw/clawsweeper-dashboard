## Summary

Keep open. Current main still leaves the reported Dreaming corpus pollution source-reproducible, and the open closing pull request at https://github.com/openclaw/openclaw/pull/93187 is the right implementation candidate for both archive exclusion and cron-descended subagent classification.

## What This Changes

The issue reports that memory-core Dreaming session-corpus ingestion admits reset/deleted archive transcripts and fails to treat cron-descended subagent transcripts as generated by cron.

## Best Possible Solution

Land a focused fix that filters Dreaming archive inputs and classifies cron descendants through stored lineage such as spawnedBy or parentSessionKey, while preserving usage-counted archive indexing for memory_search and QMD.

## Reproduction Assessment

Yes. Current main is source-reproducible: Dreaming still consumes the usage-counted session listing and cron classification still checks only direct cron session keys, so cron-descended subagents with lineage metadata are not excluded.

## Solution Assessment

Yes. The best direction is a Dreaming-specific archive eligibility boundary plus lineage-aware cron classification; changing the shared usage-counted helper alone would risk breaking memory_search and QMD archive indexing.
