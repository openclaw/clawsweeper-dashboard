## Summary

Keep this open. Current main has useful partial mitigations for pricing opt-out, Telegram request timeouts, HTTP/1-only Telegram transport, stale transport cleanup, and diagnostics, but it does not prove the broad Gateway runtime degradation is fixed; the issue discussion also reports a pricing-skip A/B test where pricing symptoms disappeared while chronic Telegram failures continued.

## What This Changes

Fix chronic Gateway runtime degradation where long-running Gateway processes see pricing fetch timeouts, Telegram polling/send failures, and slow RPC responses across Windows 11 and Node 24/25 installs.

## Best Possible Solution

Keep this as the canonical broad Gateway runtime-degradation tracker until maintainers validate current main or the next release on an affected Windows/Node Gateway, land or replace the pricing-startup subset, and then split any still-reproducing Telegram transport, stopChannel timeout, or Gateway RPC/event-loop root cause into narrow follow-ups.

Yes. The issue has detailed setup steps, repeated logs, same-host direct-fetch counterexamples, multiple corroborating reports, and an A/B pricing-skip isolation test; this review did not live-run the Windows reproduction.

## Solution Assessment

No complete solution is in place. The safest path is maintainer-led live validation plus targeted fixes for the pricing startup path and the still-source-backed channel stop/Telegram transport/RPC gaps, rather than closing from partial mitigations.
