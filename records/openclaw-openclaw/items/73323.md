## Summary

Keep this issue open. Current main has several relevant mitigations and diagnostics, but the broad cross-subsystem degradation is not proven fixed: pricing still begins from the initial runtime-services path when enabled, both remote pricing fetches still have 60s aborts, and the channel stop-timeout path still preserves the zombie-task behavior implicated by the Telegram recovery reports.

## What This Changes

The issue requests a fix for long-running Gateway degradation where pricing fetches time out, Telegram polling and outbound Bot API calls stall or fail, and Gateway RPCs become slow on affected Windows/Node installs.

## Best Possible Solution

Keep this as the canonical broad Gateway runtime-degradation tracker until maintainers validate a long-lived affected Gateway on current main or the next release, land or replace the pricing-startup deferral work, fix the source-backed stopChannel timeout recovery gap, and then split any still-reproducing Telegram transport or RPC/event-loop symptom into narrower follow-ups.
