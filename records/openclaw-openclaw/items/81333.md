Full review comments:

- **[P2] Use a defined translation key for Remove:** `ui/src/ui/views/nodes-exec-approvals.ts:613`
  - body: The allowlist remove button now calls `t("common.remove")`, but the English source map does not define `common.remove`. Since missing keys render as their raw key strings, this button will show `common.remove` in every locale; add the key and regenerate locales, or reuse an existing defined key.
  - confidence: 0.97

  - body: The localized default/use-default strings still receive raw config enum values such as `deny`, `on-miss`, and `allowlist`. In non-default agent rows, zh-CN will still show English/runtime values inside otherwise translated text; pass these values through the existing security/ask label helpers before interpolation.
  - confidence: 0.9
- **[P2] Localize token scope labels too:** `ui/src/ui/views/nodes.ts:200`
  - body: `renderTokenRow` still builds `scopes: ...` with a hardcoded English prefix after the PR localizes the surrounding token labels and actions. Paired devices with tokens will still show English in zh-CN; use the existing `nodes.scopesLabel` key or add a token-specific key here.
  - confidence: 0.86
- **[P3] Localize the allowlist never fallback:** `ui/src/ui/views/nodes-exec-approvals.ts:571-581`
  - body: The allowlist row localizes `Last used`, but still passes the literal `never` when an entry has no timestamp. That leaves English inside the zh-CN allowlist details; use `common.never` or an allowlist-specific translation for the fallback value.
  - confidence: 0.84
