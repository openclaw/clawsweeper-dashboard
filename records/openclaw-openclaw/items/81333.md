  - body: This button calls `t("common.remove")`, but the PR-head English `common` map does not define `remove`. Because missing translations render as the key, the allowlist remove button will show `common.remove` unless this uses an existing key or adds the source key and regenerates locales.

  - body: The localized default/use-default strings still receive raw config values like `deny`, `on-miss`, and `allowlist`. In zh-CN these agent-scope rows can still show runtime English values; pass the defaults through the localized security/ask label helpers before interpolation.

- **[P2] Localize token scope labels:** `ui/src/ui/views/nodes.ts:200`
  - body: Token rows still build `scopes: ...` with a hardcoded English prefix while surrounding token status and actions are localized. Paired devices with tokens will still show mixed English in zh-CN; reuse the scope translation key or add a token-specific one.

  - body: Allowlist rows translate the `Last used` label, but the empty timestamp remains the literal `never`, producing mixed output such as a Chinese label followed by `never`. Use `common.never` or a dedicated allowlist value.
  - confidence: 0.86

  - body: The new English source changes the existing re-approval message from the more specific reconnect-details text to `re-approval required`. Keep the old English source text and translate that value so this localization PR does not also change English behavior.
