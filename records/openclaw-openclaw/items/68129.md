## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] List every approval decision in pending text:** `src/infra/exec-approval-reply.ts:280-282`
  - body: `buildExecApprovalPendingText` builds descriptors for all allowed decisions but only prints `descriptors[0]`. In text-only clients, copied logs, or channels without interactive buttons, that hides `deny` and alternate approval commands, pushing operators toward approval when they may need to deny. Please render all allowed decision commands in the text payload, while keeping the command and full ID visible.
  - confidence: 0.92
- **[P1] Refresh session state before marking tasks lost:** `src/tasks/task-registry.maintenance.ts:381-392`
  - body: The mutating maintenance sweep creates one cached session-store context before the loop and then yields between batches. If a backing session is created or updated after a yield, later `shouldMarkLost` calls can still use the stale cached store and mark active work as `lost`. Refresh or invalidate session-store snapshots across yield boundaries before mutating task state.
  - confidence: 0.86
- **[P2] Make hidden exec stripping opt-in for rendered Markdown:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatMarkdownPreprocessor.swift:49-55`
  - body: `preprocess` defaults `stripHiddenSystemEvents` to true, and `ChatMarkdownRenderer` still calls it without an explicit role/source flag. A user message that quotes `System ... Exec ...` log lines can therefore be preserved during decode but stripped when rendered, hiding legitimate user content. Make the stripping opt-in for trusted assistant/system metadata paths or pass the role-aware flag through the renderer.
  - confidence: 0.84
- **[P2] Do not warn on healthy symlinked installs:** `src/infra/update-check.ts:397-399`
  - body: `detectInstallState` treats every symlinked package root as suspicious and `status-all` turns that into a warning plus reinstall/relink advice. Linked installs and version-manager symlinks can be healthy when the resolved root is consistent, so this creates upgrade/diagnostic noise instead of a real drift signal. Only warn for unsafe paths or actual mismatches.
  - confidence: 0.82
- **[P2] Recognize the openclaw binary without full argv:** `src/infra/ports-format.ts:28-34`
  - body: The stricter gateway listener fallback only accepts an `openclaw-gateway` executable basename. Listener probes can legitimately provide only `command: "openclaw"` without the full argv, so a real `openclaw gateway run` process can be classified as `unknown` and shown as a busy-port conflict. Accept the `openclaw` executable when that is the only available command metadata.
  - confidence: 0.78
