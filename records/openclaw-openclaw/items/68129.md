## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] List every approval action in pending text:** `src/infra/exec-approval-reply.ts:280-282`
  - body: `buildExecApprovalPendingText` renders only `descriptors[0]` under `Approve with`. Text-only channels and copied-log workflows lose `deny` and alternate allowed decisions, so the approval prompt must list every allowed command.
  - confidence: 0.96
- **[P1] Refresh session stores after maintenance yields:** `src/tasks/task-registry.maintenance.ts:381-389`
  - body: `runTaskRegistryMaintenance` creates one pass-wide session-store cache, then yields between batches. Session entries created after a yield can remain invisible to later `shouldMarkLost` checks, incorrectly marking active tasks as lost.
  - confidence: 0.88
- **[P2] Make hidden-event stripping opt-in for rendered text:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatMarkdownPreprocessor.swift:46-55`
  - body: The markdown preprocessor defaults hidden exec/system stripping to enabled, and the renderer calls it without message context. User-authored `System ... Exec ...` log quotes can disappear during normal rendering.
  - confidence: 0.9
- **[P2] Do not flag healthy symlink installs as suspicious:** `src/infra/update-check.ts:394-396`
  - body: `detectInstallState` marks any symlinked package root suspicious, so healthy version-manager or linked installs show recovery warnings even when the resolved root is valid. Treat symlinks as warning-worthy only with real drift or unsafe paths.
  - confidence: 0.82
- **[P2] Recognize the openclaw binary in listener fallback:** `src/infra/ports-format.ts:31-34`
  - body: When listener discovery only reports `command: "openclaw"`, the stricter fallback no longer classifies a real gateway listener as gateway-owned. That can turn a healthy gateway-owned port into a false busy-port warning.
  - confidence: 0.74
