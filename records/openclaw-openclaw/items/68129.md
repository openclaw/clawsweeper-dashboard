## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] List every allowed approval action in text prompts:** `src/infra/exec-approval-reply.ts:280-282`
  - body: `buildExecApprovalPendingText` now renders only `descriptors[0]`, so clients or copied logs without interactive buttons show the primary allow command but hide `deny` and `allow-always`. That regresses the current text-only safety path and can steer operators toward approval when they need to deny or choose a different allowed decision.
  - confidence: 0.92
- **[P1] Do not reuse session snapshots across sweep yields:** `src/tasks/task-registry.maintenance.ts:381`
  - body: `runTaskRegistryMaintenance()` creates one cached session store for the whole mutating sweep, then yields between batches. If a session is created or updated after a yield, later `shouldMarkLost` checks can still read the stale cached store and incorrectly mark active tasks as lost; keep this cache out of the mutating sweep or clear it across yields.
  - confidence: 0.84
- **[P2] Default hidden-event stripping to opt-in:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatMarkdownPreprocessor.swift:49`
  - body: The new parameter defaults to stripping hidden system events, so existing render callers such as `ChatMarkdownRenderer` still remove `System ... Exec ...` lines from user-authored log quotes even though the decode path now preserves them. Make stripping opt-in for trusted assistant/system paths or pass message context through the renderer.
  - confidence: 0.86
- **[P2] Do not warn on healthy symlink installs:** `src/infra/update-check.ts:397-398`
  - body: `detectInstallState` marks every symlinked package root suspicious and emits reinstall/relink recovery guidance. Version managers and linked installs can be healthy symlinks, so this should only warn when the resolved root is unsafe or mismatched rather than treating all symlinks as drift.
  - confidence: 0.78
