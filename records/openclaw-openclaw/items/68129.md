## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] List every approval action in pending text:** `src/infra/exec-approval-reply.ts:280-282`
  - body: `buildExecApprovalPendingText` renders only `descriptors[0]`, so text-only clients and copied logs show an allow command but not the deny or alternate allowed decisions. Approval prompts need to list every allowed command, because this text is part of the user's safety check when interactive buttons are absent.
  - confidence: 0.96
- **[P1] Refresh session stores after maintenance yields:** `src/tasks/task-registry.maintenance.ts:381-392`
  - body: `runTaskRegistryMaintenance` creates one cache context for the whole sweep, then deliberately yields between batches. Session entries created after a yield remain invisible to later `shouldMarkLost` checks, so active tasks can be marked lost from stale state.
  - confidence: 0.88
- **[P2] Make hidden-event stripping opt-in for rendered text:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatMarkdownPreprocessor.swift:49-55`
  - body: The Swift markdown preprocessor defaults hidden exec/system stripping to enabled, but generic render callers do not pass message-source context. A user-authored `System ... Exec ...` log quote can disappear during normal chat rendering even after decode preserved it.
  - confidence: 0.9
- **[P2] Warn on symlink installs only when they are actually unsafe:** `src/infra/update-check.ts:397-398`
  - body: `detectInstallState` marks every symlinked package root suspicious, so healthy version-manager or linked installs show recovery warnings even when the resolved root is valid. Treat symlink status as informational unless there is a real mismatch or unsafe path.
  - confidence: 0.82
- **[P2] Recognize bare openclaw gateway listeners:** `src/infra/ports-format.ts:32-34`
  - body: The listener fallback only accepts an `openclaw-gateway` basename. When listener discovery reports only `command: "openclaw"` for a real `openclaw gateway run` process, status can misclassify the gateway as an unknown busy-port conflict.
  - confidence: 0.74
