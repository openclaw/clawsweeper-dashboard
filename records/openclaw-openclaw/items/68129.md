Full review comments:

- **[P1] List every approval action in text prompts:** `src/infra/exec-approval-reply.ts:280-282`
  - body: `buildExecApprovalPendingText` renders only `primaryAction.command`, but text-only users still need to see every allowed decision, especially `deny`, when buttons are unavailable or copied into logs. Current main includes secondary actions in text, so this PR would make approval authorization less safe.
  - confidence: 0.94
- **[P2] Make hidden system stripping context-aware:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatMarkdownPreprocessor.swift:49-55`
  - body: The preprocessor defaults `stripHiddenSystemEvents` to true, and `ChatMarkdownRenderer` still calls it without user/assistant context. A user message that quotes `System ... Exec completed` lines can survive decode but disappear at render time, so normal chat display can silently drop user-authored logs.
  - confidence: 0.86
- **[P2] Do not warn on healthy symlinked installs:** `src/infra/update-check.ts:397-399`
  - body: `detectInstallState` treats any symlinked package root as suspicious and emits a reinstall/relink recovery hint. Linked installs and version-manager package roots can be healthy when the resolved root is consistent, so this will create false operator warnings unless the symlink is paired with a real mismatch or unsafe path.
  - confidence: 0.87
- **[P1] Refresh session cache across maintenance yields:** `src/tasks/task-registry.maintenance.ts:381-392`
  - body: `runTaskRegistryMaintenance` shares one cached session-store context for the whole sweep, but the sweep yields between batches. Session entries created after a yield stay invisible to later `shouldMarkLost` checks, so active tasks can be marked lost from stale session data.
  - confidence: 0.82
