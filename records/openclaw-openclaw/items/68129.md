## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] List every allowed approval action in pending text:** `src/infra/exec-approval-reply.ts:280-282`
  - body: `buildExecApprovalPendingText` renders only `descriptors[0]`, so clients without interactive buttons or copied-log workflows no longer show `deny` or alternate allow decisions. Exec approvals are a safety gate; the text prompt needs to list every allowed command.
  - confidence: 0.96
- **[P1] Refresh session stores after maintenance yields:** `src/tasks/task-registry.maintenance.ts:381`
  - body: The mutating maintenance sweep keeps a pass-wide session-store cache while yielding between batches. Session entries created after a yield can be invisible to later `shouldMarkLost` checks, causing active tasks to be marked `lost` from a stale snapshot.
  - confidence: 0.88
- **[P2] Make hidden-event stripping opt-in for rendered text:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatMarkdownPreprocessor.swift:49-55`
  - body: The markdown preprocessor defaults hidden exec/system stripping to enabled, and the renderer calls it without message context. User-authored quotes containing `System ... Exec ...` lines can disappear during normal rendering even if decode preserves them.
  - confidence: 0.9
- **[P2] Do not flag healthy symlink installs as suspicious:** `src/infra/update-check.ts:397-398`
  - body: `detectInstallState` treats any symlinked package root as suspicious, which makes status diagnostics warn and recommend relinking even when the resolved install root is healthy and consistent.
  - confidence: 0.82
