Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] List every allowed approval action in text prompts:** `src/infra/exec-approval-reply.ts:260-282`
  - body: `buildExecApprovalPendingText` renders only `descriptors[0]`, while the same payload can expose both Allow Once and Deny as interactive buttons. Text-only clients and copied logs therefore hide the denial/alternate commands and nudge operators toward approval, so the text prompt should list every allowed decision and full-ID recovery command.
  - confidence: 0.95
- **[P1] Refresh session snapshots before mutating lost tasks:** `src/tasks/task-registry.maintenance.ts:381-392`
  - body: The mutating sweep creates one cache context before the loop and then yields between batches. If a backing session is created or updated after a yield, later `shouldMarkLost` checks can still use the stale cached store and mark an active task lost; refresh or scope the cache so yielded mutation decisions see current session state.
  - confidence: 0.88
- **[P2] Do not strip hidden events by default for all renderers:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatMarkdownPreprocessor.swift:49-55`
  - body: Making hidden-system-event stripping the default means existing callers like `ChatMarkdownRenderer` still strip `System ... Exec ...` lines without knowing whether the message is user-authored. A user pasting those logs for debugging can lose visible content during render, so the default should preserve content or the renderer must pass role-aware intent.
  - confidence: 0.89
- **[P2] Keep symlinked installs out of drift warnings:** `src/infra/update-check.ts:397-398`
  - body: `detectInstallState` treats any symlinked active root as suspicious and attaches a reinstall/relink recovery hint. Healthy installs through version managers or linked package roots can be symlinked without drift, so this should warn only when resolved roots or unsafe paths prove an actual mismatch.
  - confidence: 0.82
