## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] List every allowed approval action in pending text:** `src/infra/exec-approval-reply.ts:280-282`
  - body: `buildExecApprovalPendingText` now renders only `descriptors[0]`, so text-only approval flows no longer show the other allowed decisions such as `deny` or `allow-always`. Interactive buttons are not guaranteed in every client or copied-log workflow, and hiding the deny path is a safety regression for exec approvals.
  - confidence: 0.96
- **[P1] Refresh session stores across maintenance yields:** `src/tasks/task-registry.maintenance.ts:381`
  - body: `runTaskRegistryMaintenance` creates one cached session-store context for the whole mutating sweep, but the loop yields between batches. A session created or updated after a yield can be invisible to later `shouldMarkLost` checks, letting the sweep mark an active task `lost` from a stale snapshot.
  - confidence: 0.88
- **[P2] Make hidden-event stripping opt-in for rendered user text:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatMarkdownPreprocessor.swift:49-55`
  - body: `preprocess(markdown:)` defaults `stripHiddenSystemEvents` to true, and `ChatMarkdownRenderer` still calls it without passing the message context. User-authored quotes containing `System ... Exec completed` lines can therefore disappear at render time even though the decode path tries to preserve user messages.
  - confidence: 0.9
- **[P2] Do not warn on symlinks without real install drift:** `src/infra/update-check.ts:397-398`
  - body: `detectInstallState` marks any symlinked package root as `suspicious`, and status-all turns that into an install-integrity warning plus reinstall/relink hint. Healthy symlinked installs should not be reported as drift unless the resolved root is actually unsafe or mismatched.
  - confidence: 0.82
