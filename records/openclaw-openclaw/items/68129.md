## Review Findings

Overall confidence: 0.89

Full review comments:

- **[P1] List every approval action in pending text:** `src/infra/exec-approval-reply.ts:280-282`
  - body: `buildExecApprovalPendingText` renders only the first descriptor under `Approve with`, so text-only clients, copied logs, and non-interactive channels lose the visible `deny` and alternate approval commands. List every allowed decision command in the text prompt.
  - confidence: 0.96
- **[P1] Refresh session stores after maintenance yields:** `src/tasks/task-registry.maintenance.ts:381-392`
  - body: `runTaskRegistryMaintenance` creates one pass-wide session-store context, then yields between batches. Session entries created after a yield can remain invisible to later lost checks, incorrectly marking active tasks as lost.
  - confidence: 0.88
- **[P2] Make hidden-event stripping opt-in for rendered text:** `apps/shared/OpenClawKit/Sources/OpenClawChatUI/ChatMarkdownPreprocessor.swift:49-55`
  - body: The markdown preprocessor defaults hidden exec/system stripping to enabled, and renderer callers do not pass message context. User-authored `System ... Exec ...` log quotes can disappear during normal chat rendering.
  - confidence: 0.9
- **[P2] Treat symlink installs as drift only with real mismatch:** `src/infra/update-check.ts:397-399`
  - body: `detectInstallState` marks every symlinked package root suspicious, so healthy version-manager or linked installs show recovery warnings even when resolved roots are valid. Warn only for actual drift or unsafe paths.
  - confidence: 0.82
- **[P2] Recognize the openclaw binary in listener fallback:** `src/infra/ports-format.ts:32-34`
  - body: When listener discovery only reports `command: "openclaw"`, the stricter fallback no longer classifies a real `openclaw gateway run` listener as gateway-owned, causing false busy-port warnings.
  - confidence: 0.74
