## Summary

Keep open: the report matches a documented `/stop` emergency-brake behavior, and current main still routes `/stop` through a fast-abort authorization path that can fall through to normal queued processing for non-native text command turns. I did not find a merged fix or duplicate owner for the remaining work in the provided context or local source search.

## What This Changes

Restore typed `/stop` handling so text slash commands interrupt active agent runs immediately instead of entering the follow-up queue.

## Best Possible Solution

Add a narrow regression around text `/stop` fast abort during an active run, then make fast-abort authorize owner/allowed text stop requests without requiring a native command turn.

## Reproduction Assessment

No, not from an executed live run in this read-only review. Source inspection gives a credible reproduction path: active run plus a text `/stop` turn that fails the fast-abort authorization gate and then enters the normal follow-up queue.

## Solution Assessment

Yes, keeping this open for a narrow fast-abort fix is the best path. The fix should stay inside abort/command authorization handling and preserve owner or allowlist checks rather than broadly trusting all text commands.
