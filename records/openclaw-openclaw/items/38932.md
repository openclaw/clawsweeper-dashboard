## Summary

Keep open. The proposed guide is not already on current main, but the security-facing docs are not merge-ready because the setup path omits the elevated PowerShell prerequisite and the verification command can pass without exercising the ACL deny boundary.

## What This Changes

The PR adds a native Windows no-Docker/WSL hardening docs page and cross-links it from docs navigation, sandboxing, and Windows platform docs.

## Best Possible Solution

Publish the fallback guide only after the setup prerequisites and verification target match the intended ACL boundary and maintainers approve the Windows host-hardening threat model.

## Reproduction Assessment

Yes for the review findings: source inspection of the proposed docs shows the missing elevated PowerShell prerequisite and the forbidden-path check that does not exercise the documented ACL deny step. I did not run the full Windows ACL setup live.

## Solution Assessment

No. A native Windows fallback guide may be useful, but this PR is not the best mergeable form until the command workflow is corrected and the security positioning is approved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] State that setup requires elevated PowerShell:** `docs/gateway/windows-no-docker-hardening.md:58`
  - body: The setup path says only "In PowerShell" before commands that create a local user and change ACLs. Those steps require an elevated/admin PowerShell session on Windows, so readers following from a normal shell will fail before the hardening flow is usable.
  - confidence: 0.86
- **[P2] Verify a path denied by the ACL step:** `docs/gateway/windows-no-docker-hardening.md:158`
  - body: The forbidden-path test checks `$env:USERPROFILE`, which is expanded by the caller and is not one of the sibling directories denied in step 4. This can pass even if the deny loop was skipped or aimed at the wrong parent, so the guide should test an explicitly blocked sibling path or placeholder blocked path.
  - confidence: 0.95

## Security Review
