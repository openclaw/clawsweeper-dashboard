## Summary

Keep open: the docs-only fallback guide is not implemented on current main, and the PR still has a concrete correctness/security review finding in its Windows ACL verification step. The direction may be useful, but it needs the verification example fixed plus secops/docs owner review before merge.

## What This Changes

The PR adds a Windows native no-Docker/WSL host-hardening fallback guide under docs, links it from sandboxing and Windows platform docs, and adds it to docs navigation.

## Best Possible Solution

Publish the guide only after the forbidden-path test validates an explicitly denied path, the docs preserve Docker/WSL as the recommended path, and security/docs owners approve the threat model.

## Reproduction Assessment

Yes for the PR defect: source inspection of the PR head shows the deny loop targets workspace-parent siblings while the forbidden-path test probes $env:USERPROFILE. I did not establish live Windows execution of the full hardening flow.

## Solution Assessment

No, not as-is. The docs direction is plausible, but the verification example should test a path the guide actually blocks and the security framing should receive owner review.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Verify an explicitly denied path:** `docs/gateway/windows-no-docker-hardening.md:158`
