Full review comments:

- **[P2] Document the elevated PowerShell requirement:** `docs/gateway/windows-no-docker-hardening.md:58`
  - body: The setup flow says only "In PowerShell" before `New-LocalUser`, group membership, and ACL grant/deny commands that require an elevated/admin shell on Windows. A normal session can fail partway through the hardening workflow, so the guide should state that setup snippets run from elevated PowerShell while the gateway runtime remains non-admin.
  - confidence: 0.9
- **[P2] Verify a path denied by the ACL step:** `docs/gateway/windows-no-docker-hardening.md:158`
  - body: The forbidden-path test probes `$env:USERPROFILE`, which can be inaccessible to `openclaw_bot` even when the sibling-deny loop was skipped or targeted incorrectly. Point the check at a specific sibling directory blocked in step 4, or a user-supplied denied path, so readers do not get a false pass for the claimed boundary.
  - confidence: 0.9
