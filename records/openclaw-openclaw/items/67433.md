## Summary

Keep open: this PR implements a real missing /hooks/agent result-return feature with sufficient live proof, but the latest diff still silently ignores idempotency for waitForResult and leaves the public contract inconsistent.

## What This Changes

The PR adds waitForResult and announceToMain handling to /hooks/agent, makes agent hook dispatch return status/sessionKey/result/error data, updates hook handler test fixtures, and declares Slack's typebox runtime dependency.

## Best Possible Solution

Ship a documented wait/result contract that either preserves or explicitly rejects idempotency for waitForResult, keeps hook trust/session safeguards, and includes focused regression coverage.

## Reproduction Assessment

Not applicable; this is a feature PR rather than a bug report. Source inspection of current main gives high confidence that waitForResult and announceToMain are absent and /hooks/agent is async runId-only today.

## Solution Assessment

No. Awaiting the isolated run is a plausible direction, but silently bypassing idempotent retries and exposing mismatched docs/comments is not the narrowest maintainable API shape yet.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Reject or preserve idempotency for waitForResult:** `src/gateway/server/hooks-request-handler.ts:293-295`
  - body: When waitForResult is true, replayKey is forced to undefined even if the caller supplies Idempotency-Key or idempotencyKey, so a retry after a client timeout starts another agent run. Existing /hooks/agent treats idempotency keys as dedupe; store/replay the completed result or explicitly reject idempotency for this mode instead of silently ignoring it.
  - confidence: 0.86
- **[P3] Align the waitForResult contract:** `src/gateway/hooks.ts:221-225`
  - body: The new field comment says the response includes a `text` field, but the handler emits `result`, and the public hook docs still list only the old fields. Update inline and public docs for request fields, response shape, errors, timeout, and retry behavior before exposing this API.
  - confidence: 0.9
