## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Keep async hook failures visible:** `src/gateway/server/hooks.ts:166-168`
  - body: `announceToMain:false` currently gates the entire result announcement policy. For async calls, the HTTP caller has already received `accepted`, so a later non-ok run is only logged and never surfaced to the target session. Keep the opt-out limited to successful summary noise, or add another failure path for async hooks.
  - confidence: 0.87
- **[P2] Preserve wait-mode idempotency semantics:** `src/gateway/server/hooks-request-handler.ts:314-320`
  - body: Setting `waitForResult:true` disables the existing replay cache entirely, so a retry after a dropped response can launch a second isolated agent run even when the caller supplied the same idempotency key. That breaks the endpoint's dedupe contract for the new mode and can duplicate downstream actions; cache the in-flight/final wait response or reject idempotency for wait mode explicitly.
  - confidence: 0.82
- **[P2] Document the new hook request and response contract:** `src/gateway/hooks.ts:476-477`
  - body: The PR adds `waitForResult`, `announceToMain`, `status`, `sessionKey`, `result`, and error responses, but the public hook docs still list only the old request fields and async run-id behavior. Please update the gateway hook docs so integrators know the final API shape and retry/failure semantics.
  - confidence: 0.78
