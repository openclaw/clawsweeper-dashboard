## Summary

Keep open: this is an active feature PR for a real gateway API gap, but it still needs after-fix real behavior proof and a fix or maintainer decision for waitForResult retry/idempotency semantics before merge.

## What This Changes

The branch adds waitForResult and announceToMain fields to POST /hooks/agent, optionally awaits isolated agent completion, returns status/sessionKey/result/error metadata, and updates hook handler test harness mocks.

## Best Possible Solution

Ship a documented /hooks/agent result-return contract that preserves or explicitly defines idempotency semantics, keeps hook trust/session protections, includes focused regression coverage, and is backed by redacted real behavior proof from a live run.

## Reproduction Assessment

Not applicable; this is a feature PR, not a bug report. Source inspection of current main confirms waitForResult and announceToMain are absent and /hooks/agent is currently an async runId-only endpoint.

## Solution Assessment

No. Awaiting the isolated run is a plausible implementation direction, but bypassing the existing hook idempotency contract and shipping mismatched/undocumented response fields is not ready as the maintainable public API path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep waitForResult retries idempotent:** `src/gateway/server/hooks-request-handler.ts:291-293`
  - body: The direct /hooks/agent path sets replayKey to undefined when waitForResult is true, so retrying a timed-out request with the same idempotency key starts another agent run. Current main dedupes /hooks/agent retries; preserve an in-flight/replay entry or define an equivalent retry response so blocking automation cannot repeat side effects.
  - confidence: 0.84
- **[P3] Align the public hook contract:** `src/gateway/hooks.ts:223-225`
  - body: The new field comment says waitForResult returns a text field, but the handler sends result, and the public hook docs still list only the old async contract. Update the inline and docs contract for request fields, response shape, errors, timeout, and retry behavior before exposing this API.
  - confidence: 0.9
