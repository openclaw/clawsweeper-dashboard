Full review comments:

- **[P2] Preserve idempotency for wait-mode retries:** `src/gateway/server/hooks-request-handler.ts:315-316`
  - body: By setting `replayKey` to `undefined` whenever `waitForResult` is true, same-key retries after a client timeout can start a second agent turn. That regresses the existing hook idempotency contract for the exact long-running path this mode introduces; store/replay an in-flight or completed wait result instead of disabling dedupe.
  - confidence: 0.87
- **[P2] Include announceToMain in the replay scope:** `src/gateway/server/hooks-request-handler.ts:327-334`
  - body: Async requests still use the replay cache, but the new `announceToMain` behavior is not part of `dispatchScope`. Two same-key requests that differ only by announcement intent can therefore collapse to the first run and either suppress or emit the main-session summary incorrectly.
  - confidence: 0.82
- **[P2] Document the new hook contract:** `src/gateway/hooks.ts:235-238`
  - body: This adds public `/hooks/agent` request fields and new response/error shapes, but the gateway docs still list only the old async contract. Callers need documented semantics for `waitForResult`, `announceToMain`, `result`, retries/idempotency, timeout behavior, and result disclosure before this API ships.
  - confidence: 0.9
