## Summary

Keep open. Current main still lacks a blocking/result-return mode for `POST /hooks/agent`, and the related product tracker remains open; however, this PR is stale against current Gateway hook code and would regress current hook routing/announcement protections if rebased mechanically.

## What This Changes

The PR adds an optional `blocking` field to agent hook payloads, makes agent hook dispatch awaitable with response text/error metadata, updates HTTP idempotency handling, and adds Gateway hook tests.

## Best Possible Solution

Rebase this PR onto current main’s `src/gateway/server/hooks-request-handler.ts` and `src/gateway/server/hooks.ts`, preserve current hook security and announcement policies, document the accepted blocking/result contract, and run focused Gateway hook tests. If maintainers prefer the broader #49019 API shape, revise this PR around that contract rather than landing the current names and response shape.

Yes. ## Solution Assessment

Unclear as currently proposed. Awaiting the isolated agent run and returning text is a plausible narrow implementation, but the branch is not the best merge path until it is rebased and preserves current idempotency, target-session routing, suppressed-announcement, documentation, and timeout semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve suppressed hook-success announcements:** `src/gateway/server/hooks.ts:95-99`
  - body: Current main only announces hook results when the run failed or delivery was requested but not completed; successful `deliver:false` hook runs are logged without waking the main session. This helper enqueues a system event whenever `result.delivered` is false, so machine-only blocking or non-blocking hooks can leak summaries into the main session. Reuse the current `shouldAnnounceHookRunResult` policy when rebasing.
  - confidence: 0.87
- **[P2] Route target-agent hook events to the target session:** `src/gateway/server/hooks.ts:96-98`
  - body: Current main resolves hook completion/error events through the target agent session when `agentId` is set. The PR's shared result/error handlers enqueue to `mainSessionKey`, which would route target-agent hook summaries and failures into the default main session after rebase. Carry over `resolveHookEventSessionKey` for both result and error paths.
  - confidence: 0.82
