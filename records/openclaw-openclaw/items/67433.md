Overall confidence: 0.89

Full review comments:

- **[P1] Preserve hook owner downgrades:** `src/gateway/server/hooks.ts:171-174`
  - body: Current main includes `forceSenderIsOwnerFalse: true` on hook result/error system events so webhook-originated artifacts cannot retain owner authority later. The new helper only sets `trusted: false` here, and the thrown-error helper has the same omission, so rebasing this would drop the structured downgrade metadata current hook trust coverage protects.
  - confidence: 0.92
- **[P2] Honor announceToMain on thrown failures:** `src/gateway/server/hooks.ts:232`
  - body: When `waitForResult` callers set `announceToMain: false`, non-throwing results respect it, but thrown import/runtime failures still call `handleRunError` and enqueue a hook error event plus heartbeat. Machine-callback flows that requested no main-session artifact can still inject one on the failure path.
  - confidence: 0.87
- **[P2] Keep waitForResult idempotent:** `src/gateway/server/hooks-request-handler.ts:293-295`
  - body: Setting `replayKey` to `undefined` for every `waitForResult` request means a retry with the same idempotency key starts another agent run. Existing hook calls dedupe retries, and synchronous callers are likely to retry after a lost response, so store/replay the completed result or reject idempotency explicitly instead of duplicating side effects.
  - confidence: 0.85
- **[P3] Return the new accepted shape on replay:** `src/gateway/server/hooks-request-handler.ts:316-318`
  - body: Fresh async responses now include `status` and `sessionKey`, but an idempotent replay still returns only `{ ok, runId }`. That makes retries observe a different response contract from the original accepted request, so cache enough data to replay the new shape or keep the async response contract unchanged.
  - confidence: 0.8
- **[P3] Document the result-return contract:** `src/gateway/hooks.ts:221-225`
  - body: The field comment still promises a `text` response while the handler returns `result`, and the public hook docs still list only the old async fields. Since this adds a public API mode, update the comment and docs with the shipped response, error, timeout, and retry semantics.
  - confidence: 0.88
