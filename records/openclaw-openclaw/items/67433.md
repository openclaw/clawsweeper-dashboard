## Summary

Keep open: current main still lacks the result-return hook API, and this is an active implementation with live proof supplied, but it should not merge until maintainers settle retry semantics and align the public contract.

## What This Changes

The branch adds waitForResult and announceToMain handling to POST /hooks/agent, makes agent hook dispatch awaitable with status/sessionKey/result/error responses, updates hook-handler test fixtures, and declares Slack's typebox runtime dependency.

## Best Possible Solution

Ship a documented /hooks/agent result-return contract with explicit idempotency semantics, preserved hook trust/session protections, focused tests, and no unrelated drive-by once main already covers it.

## Reproduction Assessment

Not applicable; this is a feature PR, not a bug report. Source inspection of current main gives high confidence that waitForResult and announceToMain are absent and /hooks/agent is async runId-only today.

## Solution Assessment

No. Awaiting the isolated run is a plausible direction, but silently bypassing idempotent retries and exposing mismatched docs/comments is not the narrowest maintainable API shape yet.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep waitForResult retries idempotent:** `src/gateway/server/hooks-request-handler.ts:293-295`
  - body: The direct /hooks/agent path sets replayKey to undefined when waitForResult is true, so retrying a timed-out request with the same idempotency key starts another agent run. Current /hooks/agent dedupes idempotent retries; preserve equivalent semantics, store a replayable result, or explicitly reject idempotency for this mode instead of silently bypassing it.
  - confidence: 0.86
- **[P3] Align the hook result contract:** `src/gateway/hooks.ts:221-225`
  - body: The new field comment says waitForResult returns a text field, but the handler sends result and the public docs still list only the old request fields. Align inline and public docs for request fields, response shape, errors, timeout, and retry behavior before exposing this API.
  - confidence: 0.9
