## Summary

Keep open: this is a valid active feature PR with live proof, but the latest diff still has correctness blockers around retry semantics and `announceToMain` behavior on thrown failures, plus contract alignment work before a public API merge.

## What This Changes

The branch adds `waitForResult` and `announceToMain` handling to `POST /hooks/agent`, makes agent hook dispatch awaitable with result/error responses, updates hook-handler test fixtures, and declares Slack's `typebox` runtime dependency.

## Best Possible Solution

Ship a documented `/hooks/agent` result-return contract that preserves or explicitly rejects idempotent retry semantics, honors `announceToMain` consistently, keeps hook events untrusted, and has focused gateway tests for success, failure, timeout, suppression, and retry behavior.

## Reproduction Assessment

Not applicable; this is a feature PR, not a bug report. Source inspection of current main and public docs gives high confidence that `/hooks/agent` remains async and runId-only today.

## Solution Assessment

No. Awaiting the isolated agent turn is a plausible direction, but the branch should settle idempotency and error-announcement semantics and align the public contract before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Honor announceToMain on thrown failures:** `src/gateway/server/hooks.ts:232-233`
  - body: In `waitForResult` mode, non-throwing error results respect `announceToMain: false`, but thrown/import/runtime failures always call `handleRunError`, which enqueues a `Hook ... (error)` system event. A machine callback can still inject an error artifact into a session even though the HTTP response already returns the error.
  - confidence: 0.87
- **[P2] Keep waitForResult retries idempotent:** `src/gateway/server/hooks-request-handler.ts:293-295`
  - body: The direct `/hooks/agent` path disables the replay key whenever `waitForResult` is true, so retrying a timed-out request with the same idempotency key starts another agent run. Preserve equivalent semantics, store a replayable result, or explicitly reject idempotency for this mode instead of silently bypassing it.
  - confidence: 0.86
- **[P3] Align the hook result contract:** `src/gateway/hooks.ts:223-225`
  - body: The new field comment promises a `text` response field, but the handler sends `result`, and the public hook docs still list only the old request/response contract. Align inline and public docs for request fields, response shape, errors, timeout, and retry behavior before exposing this API.
  - confidence: 0.9
