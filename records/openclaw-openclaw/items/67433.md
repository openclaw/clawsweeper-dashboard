## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve idempotency for wait-mode retries:** `src/gateway/server/hooks-request-handler.ts:318-320`
  - body: `waitForResult` requests currently skip the hook replay cache entirely, so a client retry with the same idempotency key after a timeout starts a second agent run instead of deduping the side effect. The existing `/hooks/agent` path has idempotency coverage; the new wait mode should preserve equivalent retry behavior or return an explicit conflict without dispatching duplicate work.
  - confidence: 0.88
- **[P2] Include announceToMain in the replay scope:** `src/gateway/server/hooks-request-handler.ts:324-338`
  - body: The direct async path's replay key includes fields such as `deliver`, channel, model, and timeout, but the new `announceToMain` flag is omitted. Two requests with the same idempotency key but different announcement intent can therefore collapse to the first run, which is wrong for a field that changes whether a session summary event is posted.
  - confidence: 0.82
- **[P2] Document the new hook contract:** `src/gateway/hooks.ts:229-238`
  - body: This adds public request fields and a richer response shape for `/hooks/agent`, but the gateway configuration reference still documents only the old async request shape. Callers need docs for `waitForResult`, `announceToMain`, returned `status`/`sessionKey`/`result`, timeout behavior, error responses, and idempotency semantics before this becomes a stable gateway API.
  - confidence: 0.9
