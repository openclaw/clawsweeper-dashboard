## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve idempotency for wait-mode retries:** `src/gateway/server/hooks-request-handler.ts:318-320`
  - body: This disables the existing hook replay cache whenever `waitForResult` is true. A caller that retries with the same idempotency key after a dropped response or client timeout will start another isolated agent run, which can duplicate messages or other side effects. Preserve/replay the waited result, track an in-flight result, or make the API reject unsupported idempotent wait requests instead of silently bypassing dedupe.
  - confidence: 0.88
- **[P2] Scope replay by announcement intent:** `src/gateway/server/hooks-request-handler.ts:324-337`
  - body: The async replay scope hashes the rest of the dispatch inputs but omits `announceToMain`. Two requests with the same idempotency key and identical payload except announcement intent can collapse to the first run, unexpectedly suppressing or producing a session event. Include the field in the replay scope or reject conflicting replays.
  - confidence: 0.84
- **[P2] Preserve async error surfacing when suppressing summaries:** `src/gateway/server/hooks.ts:166-168`
  - body: `announceToMain: false` wraps the existing `shouldAnnounceHookRunResult` policy, which currently always surfaces non-ok hook results. For non-wait async requests that later fail, this can turn an accepted hook request into a log-only failure with no session wake/event. Keep suppression limited to successful summaries, or document and test an explicit error-suppression contract.
  - confidence: 0.78
- **[P2] Document the new hook contract:** `src/gateway/hooks.ts:229-238`
  - body: The PR adds public request fields and new response shapes for `/hooks/agent`, but the public gateway/automation docs still list the old async-only request fields and examples. Update the docs for `waitForResult`, `announceToMain`, `status`, `sessionKey`, `result`, `error`, timeout behavior, and retry/idempotency semantics before exposing this API.
  - confidence: 0.9
