## Summary

Keep open: the feature remains useful and current main does not provide a wait/result hook mode, but this PR is not merge-ready because `announceToMain:false` can suppress async hook failures after the caller has only received an accepted response, and current-head real behavior proof does not exercise the real dispatcher and agent turn path.

## What This Changes

The PR adds a synchronous `waitForResult` response mode and `announceToMain` suppression option to `POST /hooks/agent`, changing hook dispatch to optionally await an agent run and return result/error/sessionKey data.

## Best Possible Solution

Preserve failure visibility for async hooks, document and test the new hook contract, then have maintainers explicitly accept the result-return, retry, timeout, and announcement-suppression semantics before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Keep async hook failures visible:** `src/gateway/server/hooks.ts:166-168`
  - body: `announceToMain: false` currently suppresses the result announcement before checking whether the run failed. In the default async path the HTTP caller has already received `accepted`, so a non-ok run with this flag would only be logged and would no longer create the failure system event that current hook tests preserve for otherwise-silent hooks. Keep failure events visible for async calls, or restrict this suppression to `waitForResult` calls that return the error to the HTTP caller.
  - confidence: 0.86
- **[P2] Document the new hook response contract:** `src/gateway/server/hooks-request-handler.ts:387-393`
  - body: The handler now returns `status`, `sessionKey`, `result`, and `error` shapes and accepts two new request fields, but the public gateway configuration reference still documents only the old async request payload. This is a new external HTTP API surface, so the docs should describe the opt-in mode, response fields, timeout behavior, and retry/idempotency semantics before merge.
  - confidence: 0.82
