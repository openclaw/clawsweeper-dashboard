## Risks / Open Questions

- Returning `result` turns a hook token from authenticated ingress into a readback capability; maintainers still need to approve result redaction, size limits, error disclosure, and whether hook-token auth is sufficient for that authority.
- `waitForResult` currently bypasses the idempotency cache, so a dropped or retried synchronous request can start another isolated agent run and duplicate side effects.
- `announceToMain:false` can suppress async non-ok hook results after the HTTP caller already received `accepted`, leaving failures visible only in logs.
- The current-head proof exercises real HTTP around the handler but stubs dispatch/provider execution, while the full live-container proof is from an older head.
- The branch adds a public request/response contract without committed behavior tests or public docs for response statuses, result/error fields, timeout behavior, idempotency semantics, or suppression behavior.
