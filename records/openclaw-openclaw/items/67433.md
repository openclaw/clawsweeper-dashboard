## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P1] Preserve idempotency for waitForResult retries:** `src/gateway/server/hooks-request-handler.ts:318-320`
  - body: `/hooks/agent` already dedupes retries by idempotency key, but this branch disables the replay key whenever `waitForResult` is true. A client retry after a dropped connection or timeout will start another agent turn instead of replaying/rejecting the original result, which can duplicate side effects in the same session. Please keep a defined idempotency behavior for the new mode and cover it with tests.
  - confidence: 0.86
- **[P2] Honor announceToMain on blocking errors:** `src/gateway/server/hooks.ts:249-250`
  - body: In the `waitForResult` branch the caller still gets the HTTP error, but the catch path always calls `handleRunError`, which enqueues `Hook ... (error)` into the session even when `announceToMain:false` was requested. That makes the new machine-only suppression option inconsistent exactly on thrown failures; the blocking path should either suppress this event or document and test that errors intentionally override the flag.
  - confidence: 0.78
- **[P2] Document the new hook contract:** `src/gateway/hooks.ts:476-489`
  - body: The code accepts `waitForResult` and `announceToMain` and returns `status`, `sessionKey`, `result`, and `error`, but the public hook docs still describe only the old request shape. Please update the gateway/automation docs with the new fields, response shapes, timeout and idempotency semantics, and the security/audit expectations for hook-token result readback.
  - confidence: 0.82
