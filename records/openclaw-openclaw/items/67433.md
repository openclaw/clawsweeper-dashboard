## Summary

Keep open. Current main still lacks the requested `/hooks/agent` result-return mode, and this active external feature PR remains useful, but the current branch still has correctness/API contract issues before merge.

## What This Changes

The branch adds `waitForResult` and `announceToMain` handling to `POST /hooks/agent`, makes agent hook dispatch return result/error data, updates gateway hook fixtures, and carries a Slack `typebox` dependency change.

## Best Possible Solution

Land a rebased gateway hook API update that returns agent results while preserving explicit retry semantics, honoring announcement suppression consistently, and documenting the public contract.

## Reproduction Assessment

Not applicable; this is a feature PR rather than a bug report. Source inspection of current main shows `/hooks/agent` is still async and runId-only, so the requested capability remains absent.

## Solution Assessment

No. Awaiting the isolated agent turn is a plausible implementation direction, but this branch still needs idempotency semantics, thrown-error announcement suppression, docs, and rebase cleanup settled before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Honor announcement suppression on thrown failures:** `src/gateway/server/hooks.ts:232`
  - body: In `waitForResult` mode, normal and non-ok results pass through the `announceToMain` gate, but thrown/import/runtime failures always call `handleRunError`, which enqueues a `Hook ... (error)` system event. A machine callback using `announceToMain: false` can still inject an error artifact into a session even though the HTTP response already reports the failure.
  - confidence: 0.88
- **[P2] Keep waitForResult retries idempotent:** `src/gateway/server/hooks-request-handler.ts:293-295`
  - body: This bypasses the replay cache whenever `waitForResult` is true, so retrying a timed-out request with the same idempotency key starts another isolated agent run. Either store/replay the completed result, return an in-flight/conflict response, or reject idempotency for this mode instead of silently ignoring the existing retry contract.
  - confidence: 0.86
- **[P3] Align the documented result field:** `src/gateway/hooks.ts:221-225`
  - body: The new payload comment says the response includes `text`, but the handler sends `result`, and the public hook docs still list only the old request shape. Align the inline and public docs for request fields, response shape, errors, timeout, and retry behavior before exposing this API.
  - confidence: 0.9
