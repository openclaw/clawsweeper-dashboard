## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Preserve idempotency for wait-mode retries:** `src/gateway/server/hooks-request-handler.ts:318`
  - body: `waitForResult` requests now skip the replay cache entirely. A caller retrying with the same idempotency key after a dropped response or client timeout will start a second isolated agent run, which can duplicate messages, tool actions, or other side effects. Wait mode should cache/replay the terminal result or define a safe pending/retry contract instead of bypassing dedupe.
  - confidence: 0.88
- **[P2] Keep non-ok hook runs visible when summaries are suppressed:** `src/gateway/server/hooks.ts:166-168`
  - body: `announceToMain:false` is applied before the non-ok announcement policy, so async hook failures and skipped runs can become log-only. Current main surfaces non-ok hook results into the resolved session even for silent hook paths; suppressing success summaries should not also hide failures unless maintainers explicitly choose and document that failure mode.
  - confidence: 0.84
- **[P2] Document the new hook contract:** `src/gateway/hooks.ts:229-238`
  - body: This adds public request fields and new response shapes for `/hooks/agent`, but the configuration reference still documents only the old async payload. Users will not have the retry, timeout, result, error, or `announceToMain` contract needed to use this safely.
  - confidence: 0.81
