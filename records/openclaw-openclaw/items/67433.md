- **[P1] Gate returned hook results behind approved authority:** `src/gateway/server/hooks-request-handler.ts:387-393`
  - body: This response now returns agent output to any caller that can authenticate to `/hooks/agent` and pass the existing hook policy. That changes hook-token auth from trigger-only ingress into a readback capability, so this needs a maintainer-approved gate or documented security contract before merge.
  - confidence: 0.86
- **[P2] Preserve idempotency semantics for wait-mode retries:** `src/gateway/server/hooks-request-handler.ts:318-319`
  - body: `waitForResult` disables the replay cache while still accepting idempotency keys, so a client retry after a dropped response or timeout can launch a second isolated agent run. Either replay/store the completed response, return a retry-safe in-progress/error shape, or reject idempotency for wait-mode requests.
  - confidence: 0.84
- **[P2] Keep async hook failures visible:** `src/gateway/server/hooks.ts:166-168`
  - body: Because `announceToMain` gates all announcements before checking status, an async request with `announceToMain:false` can hide a non-ok hook run after the HTTP caller already received `accepted`. Non-waiting callers still need an error surface for failed runs.
  - confidence: 0.82
- **[P2] Document and test the new hook contract:** `src/gateway/hooks.ts:476-477`
  - body: These fields create a new public endpoint contract, but the PR leaves the public hook docs async-only and the changed files do not add dedicated wait/result/error/retry tests. Add focused coverage and docs before exposing the new response shape.
