## Summary

Keep open. Current main still lacks the requested result-return mode, and this recent PR is the active implementation candidate for https://github.com/openclaw/openclaw/issues/49019. It is not merge-ready yet because the new waitForResult path bypasses idempotent retries, the public contract is not documented consistently, and the external contributor has not posted after-fix real behavior proof.

## What This Changes

The PR adds waitForResult and announceToMain fields to POST /hooks/agent, awaits isolated agent completion when requested, and returns status, sessionKey, result, or error metadata.

## Best Possible Solution

Ship a documented /hooks/agent result-return contract that preserves or explicitly defines idempotency semantics, keeps hook trust/session protections, adds focused regression coverage, and includes redacted real behavior proof from a live run.

## Reproduction Assessment

Not applicable as a feature PR. Source inspection of current main confirms waitForResult and announceToMain are absent and /hooks/agent returns a runId-only async response.

## Solution Assessment

Unclear. Awaiting the isolated run is a plausible implementation direction, but bypassing the established hook replay cache and shipping mismatched/undocumented response fields is not the narrowest maintainable public API path yet.
