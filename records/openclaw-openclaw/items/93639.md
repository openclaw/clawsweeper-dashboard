## Summary

## What This Changes

## Best Possible Solution

Apply the prefix in the shared send path after resolving the same cascade and template context used by visible replies, with focused coverage for static, idempotent, media-only, and templated prefixes.

## Reproduction Assessment

Yes. Current main builds and dispatches `message(action="send")` text without any responsePrefix handling, while the normal reply normalizer resolves and prepends configured prefixes.

## Solution Assessment

No. The PR is in the right shared send layer for static prefixes, but the best fix must resolve templated prefix values with the same context visible replies use or explicitly avoid applying unresolved templates.
