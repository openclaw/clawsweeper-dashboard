## Summary

Keep open: current main and the latest stable release still route Anthropic Vertex through the package-local @anthropic-ai/vertex-sdk 0.16.1 dependency chain that resolves google-auth-library 9.15.1 and gaxios 6.7.1, while the provider runtime constructs the SDK client without a custom Google Auth transporter. The merged Node 25 gaxios work and the open audit cleanup PR overlap, but neither currently gives a shipped, safe Anthropic Vertex fix for this report.

## What This Changes

Fix Anthropic Vertex OAuth token fetching on Node 25+ by moving the provider off, or safely injecting around, its google-auth-library 9.x and gaxios 6.x transport path.

## Best Possible Solution

Ship a current-main fix that keeps Anthropic Vertex ownership inside the provider package, removes or bypasses the google-auth-library 9/gaxios 6 token transport path, regenerates the provider shrinkwrap, and proves a Node 25+/26 ADC Vertex turn.

## Reproduction Assessment

Yes, source-reproducible: the reporter gives a Node 26.3.1 ADC path, and current main plus v2026.6.8 still route token fetching through the old SDK -> google-auth-library 9 -> gaxios 6 transport. I did not run live GCP credentials in this read-only review.

## Solution Assessment

Yes on direction, not yet on a specific PR: the maintainable fix should be provider-owned and either inject a safe GoogleAuth/gaxios fetch transport or remove the old SDK dependency chain. The existing open audit PR is relevant but needs refresh and auth-provider proof before it can be treated as the solution.
