## Summary

Keep open: current main still records the non-deliverable terminal marker but reopens the same transcript before any recovery gate, and the related open/closed PRs address retry or user-facing copy rather than this stale-transcript recovery path. The patch looks functionally reasonable, with the remaining merge question being maintainer acceptance of the session-state archive-and-reset behavior.

## What This Changes

The PR adds a pre-open embedded-runner recovery helper that archives the session transcript and trajectory when the latest trajectory `session.ended` event has `terminalError=non_deliverable_terminal_turn`, then starts the next turn as a fresh transcript.

## Best Possible Solution

Land a narrow pre-open recovery path only after maintainer sign-off that archive-and-reset is the intended recovery for poisoned non-deliverable terminal transcripts; keep broader retry and user-facing copy policy in the sibling diagnostic work.

## Reproduction Assessment

Yes at source level: current main records `non_deliverable_terminal_turn` in trajectory and reopens the same transcript before any recovery gate. I did not run the production Feishu scenario live in this read-only review.

## Solution Assessment

Mostly yes: the pre-open gate is the narrowest place to avoid replaying a known poisoned transcript, and it preserves backups. The remaining question is whether maintainers accept runtime archive-and-reset as the desired session-state behavior.
