## Summary

Keep open: this is a focused bug-fix PR for a still-missing current-main recovery path. I found no blocking line-level correctness finding, but the patch intentionally resets a poisoned persisted transcript by archiving it, so maintainers should treat the session-state tradeoff as an explicit pre-merge decision.

## What This Changes

The branch adds an embedded-runner pre-open recovery gate that archives a transcript and trajectory when the latest trajectory end is `non_deliverable_terminal_turn`, then starts the current run as a fresh transcript with focused helper coverage.

## Best Possible Solution

Land a narrow recovery path only if maintainers accept archived-transcript reset as the intended behavior for poisoned non-deliverable sessions, with the existing generic retry/copy work remaining separate.

## Reproduction Assessment

Yes at source level: current main records `non_deliverable_terminal_turn` in trajectory and then reopens the same transcript before any recovery gate. I did not run a live Feishu or production-session reproduction in this read-only review.

## Solution Assessment

Mostly yes: the pre-open gate is a narrow place to prevent replaying a known poisoned transcript and it preserves backups. The main remaining question is whether maintainers accept runtime archive-and-reset as the desired session-state behavior.
