## Summary

Keep this PR open: current main still records `non_deliverable_terminal_turn` and then reopens the same transcript without an archive/recovery gate, while no merged sibling PR supersedes this transcript-recovery behavior. The patch looks like a narrow, plausible fix, but it intentionally changes persisted session handling, so it needs maintainer sign-off before merge.

## What This Changes

The PR adds an embedded-agent pre-open recovery gate that archives a transcript and trajectory when the latest trajectory `session.ended` event is `non_deliverable_terminal_turn`, then opens the next attempt as a fresh session.

## Best Possible Solution

Land the narrow pre-open recovery path only if maintainers accept archive-and-reset as the intended recovery for poisoned non-deliverable sessions; keep broader retry and user-copy policy separate.

## Reproduction Assessment

Yes at source level: current main records `non_deliverable_terminal_turn` in trajectory and then prewarms/reopens the same transcript before any recovery gate. I did not run a live Feishu or production-session replay in this read-only review.

## Solution Assessment

Mostly yes: the pre-open gate is a narrow place to stop replaying a known poisoned transcript, and the PR preserves backups. The remaining question is whether maintainers accept runtime archive-and-reset as the right session-state policy.
