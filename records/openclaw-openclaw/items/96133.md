## Best Possible Solution

Land the narrow recovery only after maintainers accept archive-and-reset as the intended behavior for poisoned non-deliverable sessions, while keeping broader retry/copy/context-continuity work in its canonical trackers.

## Solution Assessment

Mostly yes: the pre-open gate is a narrow place to avoid replaying a known poisoned transcript and it preserves backups. The unresolved question is policy, namely whether maintainers accept runtime archive-and-reset instead of context carry-forward for this terminal state.
