## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

No, not yet: moving the native typing RPC earlier is a reasonable fix shape, but reusing that predicate for existing tool-progress behavior is too broad. The safer solution keeps those predicates separate and adds visible iMessage proof.
