## Summary

Keep open: current main has a source-reproducible DSML leak path and the PR targets the right stream seam, but the latest head still has blocking stream/parser bugs, lacks real behavior proof, and changes the tool-execution boundary.

## What This Changes

The PR adds a DeepSeek-scoped OpenAI-completions stream text processor and regression fixtures to suppress DSML `tool_use_error` text and parse DSML tool-call blocks before assistant stream events/content are emitted.

## Best Possible Solution

Keep a DeepSeek-scoped stream fix, but repair the ordering and close-token bugs, settle the DSML tool-call parsing boundary, add the required changelog entry, and require redacted real OpenClaw/DeepSeek stream proof before merge.

## Reproduction Assessment

Yes, at source level: current main appends `choice.delta.content` directly into assistant-visible text, so a DSML text delta leaks. I did not run a live DeepSeek scenario in this read-only review.

## Solution Assessment

No. Filtering provider-private text at the stream seam is likely the right boundary, but this patch still misorders buffered text, misses singular close tokens, and expands inline DSML into tool calls without clear maintainer approval.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Flush buffered text before non-content deltas:** `src/agents/openai-transport-stream.ts:1570-1575`
  - body: When a content chunk ends with a DSML open-token prefix such as `see <`, the processor keeps `<` buffered. If the next chunk has only reasoning or `tool_calls`, that buffered visible text is not emitted until final `flush()`, so it can move after the tool/reasoning block; flush non-suppressed buffered text before handling non-content branches.
  - confidence: 0.91
- **[P2] Accept singular DSML close tokens:** `src/agents/openai-transport-stream.ts:1717-1719`
  - body: The singular `<｜DSML｜tool_call>` definition only accepts plural `</｜DSML｜tool_calls>` close tokens. A complete singular block followed by normal text is treated as unterminated until `flush()`, causing the trailing visible text to be dropped; include singular close tokens here too.
  - confidence: 0.94
- **[P2] Keep DSML suppression separate from tool execution:** `src/agents/openai-transport-stream.ts:1511-1514`
  - body: This leak fix also converts DeepSeek DSML `tool_calls` text into OpenClaw `toolCall` blocks when no structured tool calls were seen. That broadens the command/tool execution surface beyond suppressing `tool_use_error` text, so either remove it from this PR or gate it behind an explicitly approved provider contract.
  - confidence: 0.82
- **[P3] Add the required changelog entry:** `src/agents/openai-transport-stream.ts:1326-1328`
  - body: This is a user-facing bug fix, so repo policy requires a single-line `CHANGELOG.md` entry. Add one under the active release fixes section without guessing forbidden bot or maintainer attribution.
  - confidence: 0.87
