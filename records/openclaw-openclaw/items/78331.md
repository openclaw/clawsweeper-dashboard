## Summary

Keep open: the narrowed revision removes the DSML-to-toolCall security boundary, but it still has blocking stream/parser bugs and only unit-level replay proof, so it is not ready to merge.

## What This Changes

The PR adds a DeepSeek-scoped OpenAI-completions text processor, regression tests, and a changelog entry to suppress DSML/shadow blocks and route inline thinking without inventing tool calls.

## Best Possible Solution

Fix the stream processor ordering, singular close-token handling, and inline-thinking gating, then add real DeepSeek/OpenClaw run proof while keeping DSML text suppress-only.

## Reproduction Assessment

Yes, at source level: current main appends captured DSML text from `choice.delta.content` directly into assistant-visible output. I did not establish a live DeepSeek API reproduction in this read-only review.

## Solution Assessment

No. Suppressing provider-private DSML at the OpenAI-compatible stream seam is directionally narrow and safer after dropping DSML-to-toolCall parsing, but this revision still needs the ordering, close-token, and inline-thinking fixes plus real runtime proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Flush buffered DeepSeek text before non-content deltas:** `src/agents/openai-transport-stream.ts:1534-1549`
  - body: When a content chunk ends with a prefix of a DSML open token, the processor can keep that visible suffix buffered. If the next chunk has only reasoning or native `tool_calls`, those blocks are emitted first and the visible suffix is flushed only at end-of-stream, changing output order.
  - confidence: 0.9
- **[P2] Accept singular DSML close tokens:** `src/agents/openai-transport-stream.ts:1665`
  - body: The singular `tool_call` definition opens on `<|DSML|tool_call>` but only recognizes plural `tool_calls` close tokens. A complete singular block followed by normal assistant text can be treated as unterminated and the trailing visible text dropped at final flush.
  - confidence: 0.94
- **[P2] Narrow inline thinking detection:** `src/agents/openai-transport-stream.ts:1820`
  - body: Returning true for any `reasoning_effort` enables the inline `</think>` scanner on normal DeepSeek thinking requests, even though DeepSeek's OpenAI-compatible stream already exposes thinking through `reasoning_content`. Short normal answers can stop producing text deltas until completion.
  - confidence: 0.87
