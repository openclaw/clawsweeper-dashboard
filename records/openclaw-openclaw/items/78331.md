## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve buffered text before same-chunk tool calls:** `src/agents/openai-transport-stream.ts:1522-1524`
  - body: When a DeepSeek chunk carries ordinary `content` plus native `tool_calls` or reasoning fields, `push()` can retain short safe text and this branch only flushes when content is absent. That buffered text is emitted later, after the tool-call/reasoning block, so a chunk like `content: "I'll check", tool_calls: [...]` records the tool call before the text.
  - confidence: 0.91
- **[P2] Emit safe text without waiting for flush:** `src/agents/deepseek-text-filter.ts:56-60`
  - body: When no DSML opener is present, the filter still keeps the last `MAX_OPEN_TOKEN_LEN - 1` characters unconditionally. Normal short DeepSeek chunks such as `hello` then produce no `text_delta` until final flush; retain only a suffix that is actually a prefix of a DSML open token.
  - confidence: 0.86
