## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Flush buffered DeepSeek text before non-content deltas:** `src/agents/openai-transport-stream.ts:1530`
  - body: When a DeepSeek content chunk ends with a prefix of a DSML open token, the processor can retain visible text in its buffer. The loop then handles reasoning or native `tool_calls` before that text is flushed, so output can move after a reasoning/tool block.
  - confidence: 0.9
- **[P2] Accept singular DSML close tokens:** `src/agents/openai-transport-stream.ts:1655-1662`
  - body: The filter opens on singular `tool_call` DSML tokens but only recognizes plural `tool_calls` close tokens. A complete singular block followed by normal assistant text is treated as unterminated, and the trailing visible text is dropped on final flush.
  - confidence: 0.94
- **[P2] Narrow inline thinking detection:** `src/agents/openai-transport-stream.ts:1800`
  - body: Returning true for any `reasoning_effort` enables the inline `</think>` scanner for ordinary DeepSeek V4 thinking requests, even though this path already receives native `reasoning_content`. Short normal answers can stop streaming text deltas until completion.
  - confidence: 0.87
