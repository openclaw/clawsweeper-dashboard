## Summary

Keep this issue open. Current main still does not make `status --deep` use live gateway memory runtime health; it skips memory inspection outside `--all` and renders an enabled memory plugin as `not checked`, while the broader target-repo fix PRs are closed unmerged.

## What This Changes

The issue asks `openclaw status --deep` to stop reporting a working live memory plugin as unavailable.

## Best Possible Solution

Land a focused target-repo fix that lets deep status surfaces trust live gateway memory provider/runtime evidence without adding a new config option or broad plugin-loader policy change.

## Reproduction Assessment

Yes, source-reproducible: current `status --deep` does not resolve a memory snapshot unless `--all` is set, so an enabled memory slot can only render as skipped/not checked rather than healthy from the live gateway. I did not run a live external memory backend in this read-only review.

## Solution Assessment

No, current main is only a partial mitigation. The best path is a narrow fix based on the closed implementation attempts that wires live gateway memory provider/runtime health into human and JSON deep status while keeping embedding readiness separate.
