## What This Changes

The branch adds a dedicated Active Memory recall lane, passes it to `runEmbeddedPiAgent`, adds a focused lane assertion, and records a changelog fix.

## Best Possible Solution

Land the plugin-owned dedicated-lane assignment with the focused regression test after real runtime proof and green required security/CI checks, keeping broader Active Memory fail-open and latency policy work separate.

Yes, at source level: current main awaits Active Memory recall inside `before_prompt_build`, the recall call omits `lane`, omitted embedded lanes resolve to `main`, and command lanes run one active task by default. The linked issue also includes matching Telegram gateway timeout logs.

## Solution Assessment

Yes, the proposed plugin-owned `active-memory` lane at the recall call site is the narrowest maintainable fix; changing embedded-runner defaults globally would affect unrelated callers. The remaining blockers are proof quality and CI state, not the code direction.
