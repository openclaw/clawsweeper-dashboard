## Best Possible Solution

Land the dedicated-lane Active Memory fix with the focused regression once contributor-supplied live runtime proof and required CI gates are acceptable, keeping broader fail-open or latency-policy work separate.

## Reproduction Assessment

Yes, at source level: current main awaits Active Memory recall inside `before_prompt_build`, the recall call omits `lane`, omitted lanes resolve to `main`, and command lanes are serial by default. ## Solution Assessment

Yes. Passing a plugin-owned `active-memory` lane at the recall call site is the narrowest maintainable fix because changing embedded-runner defaults globally would affect unrelated callers and plugin contracts.
