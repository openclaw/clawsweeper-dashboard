## Summary

Keep open: the PR addresses a real source-backed Claude CLI live-session reset bug, but the current patch removes all system-prompt invalidation rather than preserving stable prompt-change invalidation.

## What This Changes

The PR exports `buildClaudeLiveFingerprint`, removes `systemPromptHash` from the Claude CLI live-session fingerprint, and adds unit coverage for volatile prompt reuse plus existing fingerprint gates.

## Best Possible Solution

Replace the full volatile `systemPromptHash` with a stable Claude live-session prompt fingerprint that excludes per-turn inbound metadata but still rotates on real instruction changes.

## Reproduction Assessment

Yes, source-level: current main hashes the full `context.systemPrompt` and restarts the Claude live process on fingerprint mismatch, while chat prompt preparation injects volatile inbound metadata into that prompt. The PR body also supplies redacted live Telegram logs and a screenshot showing the before/after symptom.

## Solution Assessment

No. Removing the entire prompt hash fixes volatile metadata resets but lets a reused Claude live process ignore stable prompt changes; the safer fix is to hash stable prompt material or strip only volatile metadata.
