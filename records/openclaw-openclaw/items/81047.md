## Summary

Keep this PR open: it targets a real Claude CLI live-session context-loss bug with sufficient live proof, but the current patch overcorrects by removing all system-prompt invalidation instead of preserving stable prompt changes.

## What This Changes

The branch exports `buildClaudeLiveFingerprint`, removes `systemPromptHash` from the Claude CLI live-session fingerprint, and adds unit tests for volatile prompt reuse plus existing fingerprint gates.

## Best Possible Solution

Replace the full prompt hash with a stable-prefix prompt fingerprint, likely using the internal system-prompt cache boundary, and keep regression tests for both volatile metadata reuse and stable prompt invalidation.

## Reproduction Assessment

Yes. Source inspection shows current main hashes per-turn `context.systemPrompt` and restarts on fingerprint mismatch, and the PR body supplies live Telegram plus journal proof of the symptom and patched behavior.

## Solution Assessment

No. Removing the entire `systemPromptHash` fixes volatile metadata churn but is not the best fix because it also lets stable system-prompt changes be ignored by an already running Claude live process.
