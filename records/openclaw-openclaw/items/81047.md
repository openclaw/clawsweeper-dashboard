## Summary

Keep open: this PR targets a valid source-backed claude-cli chat context-loss bug and supplies sufficient live proof, but the current patch drops all prompt-change invalidation and can let stable system-prompt updates be ignored by a reused live process.

## What This Changes

The PR exports `buildClaudeLiveFingerprint`, removes `context.systemPrompt` from the Claude CLI live-session fingerprint, and adds unit tests for volatile prompt reuse plus existing fingerprint gates.

## Best Possible Solution

Preserve warm claude-cli process reuse across volatile chat metadata while still rotating the live process when stable prompt-affecting inputs change; after the PR is fixed and merged, close https://github.com/openclaw/openclaw/issues/81041.

## Reproduction Assessment

Yes, source-level: current main hashes the full assembled `context.systemPrompt`, chat reply preparation injects per-turn metadata into that prompt, and the live session is restarted when the fingerprint changes. The PR body also supplies redacted live Telegram before/after logs, though I did not rerun a live Telegram setup in this read-only review.

## Solution Assessment

No. Removing the whole `systemPromptHash` fixes the volatile metadata reset but also removes invalidation for stable system-prompt changes; the safer fix is to fingerprint stable prompt material and exclude only volatile chat metadata.
