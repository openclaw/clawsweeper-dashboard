## Best Possible Solution

## Reproduction Assessment

Yes, source-level and contributor-live proof are both strong: current main hashes the full per-turn `systemPrompt`, and the PR body includes live Telegram plus journal evidence showing restarts before a matching runtime patch and no restart afterward. ## Solution Assessment

No. The proposed deletion fixes the volatile metadata symptom, but it is too broad because reused live sessions do not receive updated system prompts; the safer fix preserves a stable static prompt invalidator while excluding only per-turn metadata.
