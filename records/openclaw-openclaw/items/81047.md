## Summary

Keep open: the PR targets a real source-backed Claude CLI live-session context-loss bug, but the patch removes all system-prompt invalidation instead of preserving stable prompt changes that still require a fresh live process.

## What This Changes

The PR exports the Claude live fingerprint helper, removes `context.systemPrompt` from that fingerprint, and adds focused unit tests for volatile prompt reuse plus model/workspace/static-prompt gates.

## Best Possible Solution

Add a stable Claude live-session prompt fingerprint that excludes volatile chat metadata while still rotating the process when stable instructions or runtime prompt inputs change.

## Reproduction Assessment

Yes. Source inspection shows current main hashes volatile chat prompt content and restarts on fingerprint mismatch, and the PR body supplies live Telegram/claude-cli logs plus an inspected screenshot showing the before/after symptom.

## Solution Assessment

No. Removing the full prompt hash fixes the per-turn metadata reset, but the safer solution preserves a stable prompt-change signal instead of allowing reused live processes to ignore real instruction changes.
