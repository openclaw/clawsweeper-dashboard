## Summary

Keep open: the reported bug is plausible and source-backed, but this patch drops the whole live-session system prompt fingerprint and lacks real behavior proof. That can preserve the Claude live process across real system prompt changes, not only across volatile chat metadata changes.

## What This Changes

The PR exports `buildClaudeLiveFingerprint`, removes `systemPromptHash` from the Claude CLI live-session fingerprint, and adds a unit test file for fingerprint stability.

## Best Possible Solution

Preserve stable system-prompt invalidation for Claude live sessions while excluding only per-turn volatile chat metadata, with a focused regression test and real chat proof.

## Reproduction Assessment

Yes, source-level: current `main` fingerprints the full `context.systemPrompt` and recreates the live session when it changes, while chat code builds volatile inbound metadata separately from static extra context. I did not run the live Telegram or Discord reproduction in this read-only review.

## Solution Assessment

No. Removing the whole `systemPromptHash` is broader than the bug boundary; the safer fix is to fingerprint stable system-prompt inputs or move volatile metadata into per-turn prompt context while keeping real prompt changes session-invalidating.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Keep stable system-prompt changes in the fingerprint:** `src/agents/cli-runner/claude-live-session.ts:276`
