## Summary

Keep open: the reported bug is source-backed, but this PR drops the live-session prompt guard too broadly and still lacks after-fix real behavior proof from a real chat/claude-cli setup.

## What This Changes

The PR exports `buildClaudeLiveFingerprint`, removes the full `context.systemPrompt` hash from the Claude live-session fingerprint, and adds focused unit coverage for volatile prompt text plus static/model/workspace invalidation.

## Best Possible Solution

Hash stable session-changing prompt material, such as the cache-boundary stable prefix or an equivalent digest, while excluding volatile inbound metadata; then cover both volatile and stable-prompt cases and attach redacted real chat proof.

## Reproduction Assessment

Yes, by source inspection: current main hashes the fully assembled system prompt and restarts the live process on fingerprint mismatch, while chat turns add volatile inbound metadata that the static prompt hash intentionally excludes. I did not run a live Telegram or Discord claude-cli repro in this read-only review.

## Solution Assessment

No: removing the full prompt hash addresses the volatile reset, but it also removes stable prompt invalidation for live processes. The narrower maintainable fix is to fingerprint stable prompt content while excluding the volatile suffix.
