## Summary

Keep open: current `main` and latest release still have the volatile full-prompt fingerprint bug, but this PR is not merge-ready because it removes the only broad prompt-content invalidator for a warm Claude CLI process and the added test fixture is stale against current types.

## What This Changes

The PR removes `systemPromptHash` from the Claude CLI live-session fingerprint, exports the fingerprint helper for tests, and adds a focused fingerprint unit test.

## Best Possible Solution

Preserve the volatile-metadata fix while replacing the full prompt hash with a stable prompt-content fingerprint that rotates warm Claude processes when non-volatile instructions actually change, then update the fixture and validation.

## Reproduction Assessment

Yes. Source inspection shows current `main` hashes the full per-turn `systemPrompt`, chat-channel metadata flows into that prompt, and the live-session path restarts when the fingerprint differs; the PR body also supplies live Telegram and journal evidence.

## Solution Assessment

No. Removing volatile full-prompt bytes is the right direction, but the proposed deletion is too broad because a reused Claude live process does not receive updated spawn-time prompt files unless the fingerprint still rotates on stable prompt-content changes.
