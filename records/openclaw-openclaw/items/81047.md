## Summary

Keep open: the underlying Claude CLI live-session reset bug is still present on current main and the latest release, but this PR is not merge-ready because it removes the whole full-system-prompt fingerprint axis and its new test fixture misses current required context fields.

## What This Changes

The PR exports `buildClaudeLiveFingerprint`, removes `systemPromptHash` from Claude live-session fingerprints, and adds a focused fingerprint unit test.

## Best Possible Solution

Exclude per-turn volatile metadata from the live fingerprint while preserving a stable static prompt invalidator, then repair the fixture and validate focused tests plus test-type checks.

## Reproduction Assessment

Yes. Source inspection shows current main hashes the full per-turn `systemPrompt`, chat-channel metadata flows into that prompt, and the live-session path restarts when the fingerprint differs; the PR body also supplies live Telegram and journal evidence.

## Solution Assessment

No. The proposed deletion fixes the volatile metadata symptom, but it is too broad because reused live sessions do not receive updated startup system prompts; the safer fix preserves a stable static prompt invalidator while excluding only turn-variable prompt material.
