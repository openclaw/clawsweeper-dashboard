## Summary

Keep open; the reported bug is real and the supplied proof is sufficient, but this patch is too broad because it removes stable system-prompt invalidation for a Claude live process that only receives the system prompt when the process is spawned.

## What This Changes

The PR exports `buildClaudeLiveFingerprint`, removes `context.systemPrompt` from the Claude CLI live-session fingerprint, and adds unit tests for volatile prompt reuse plus existing fingerprint gates.

## Best Possible Solution

Add or carry a stable Claude live-session prompt fingerprint that excludes volatile chat metadata but still rotates the process when stable system-prompt inputs change.

## Reproduction Assessment

Yes, source-level: current main hashes the full `context.systemPrompt` while chat prompt preparation injects volatile inbound metadata and separately computes a static prompt subset. The PR body also supplies redacted live Telegram logs and a screenshot showing the before/after symptom, though I did not rerun live Telegram in this read-only review.

## Solution Assessment

No. Removing the entire `systemPromptHash` fixes the volatile metadata reset but also lets stable prompt changes be ignored by a reused Claude live process; the safer fix is a stable prompt hash that preserves real prompt invalidation.
