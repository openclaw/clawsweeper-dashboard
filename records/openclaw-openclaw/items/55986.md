## Summary

Keep open. The OpenRouter audio transcription capability is still missing on current main, but this PR is stale against the existing OpenRouter media provider and has blocking transport-policy, metadata, and changelog issues before it can merge.

## What This Changes

The PR adds an OpenRouter media-understanding audio transcription provider using chat/completions input_audio, registers it, adds focused tests, and updates audio docs.

## Best Possible Solution

Land a refreshed implementation that extends the existing bundled OpenRouter media provider with audio support, preserves image support, uses provider-http policy helpers, updates manifest metadata/docs/changelog, and keeps auto-detection unchanged.

## Reproduction Assessment

Yes. Source inspection on current main is enough: OpenRouter advertises only image media-understanding support, and the audio runner requires a provider transcribeAudio hook for tools.media.audio provider entries.

## Solution Assessment

No. The feature direction fits the provider plugin surface, but this patch should be rebased and reshaped to extend the current provider, route through provider-http policy, and update metadata/changelog before merge.

  - confidence: 0.88
