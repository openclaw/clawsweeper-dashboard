## Summary

Keep open: current main still lacks OpenRouter audio transcription, and this draft PR is a useful narrow direction, but it is not merge-ready because it conflicts with the existing OpenRouter image provider, imports non-exported helpers, and bypasses provider HTTP policy handling.

## What This Changes

The PR adds an OpenRouter media-understanding audio transcription provider using chat/completions input_audio, registers it, adds request-shape tests, and documents tools.media.audio usage.

## Best Possible Solution

Land a refreshed implementation that extends the existing OpenRouter media provider with audio support, uses provider-http policy helpers, updates plugin metadata/docs/changelog, and keeps auto-detection unchanged.

## Reproduction Assessment

Yes. Source inspection on current main is enough: OpenRouter declares only image media-understanding hooks, and the audio runner requires transcribeAudio for provider audio entries.

## Solution Assessment

No. The feature direction is appropriate, but this patch should be refreshed against main, preserve image support, route through provider-http policy, and update metadata/changelog before merge.
