## Summary

Keep open. Current main and the latest release still have no supported Feishu `ignoreFileContent` or inbound-media opt-out, while Feishu media is downloaded into `MediaPaths` that the agent prompt path includes; this needs maintainer product/config review, not cleanup closure.

## What This Changes

The issue asks Feishu to honor an `ignoreFileContent` configuration value by skipping inbound file downloads and sending only a placeholder to the model.

## Best Possible Solution

Define a Feishu-owned inbound media opt-out in schema, docs, and runtime, apply it before downloads, preserve placeholder text, and cover the approved behavior with focused Feishu tests plus live proof.

## Reproduction Assessment

Yes, at source level: current main and `v2026.5.12` lack `ignoreFileContent` in the strict Feishu schema, and Feishu media messages still flow through `resolveFeishuMediaList` into `MediaPaths`. I did not run a live Feishu message.

## Solution Assessment

No for an automatic bug fix: current OpenClaw does not define `ignoreFileContent` as a Feishu contract. The maintainable solution is an explicit, maintainer-approved Feishu inbound-media opt-out with schema, docs, runtime behavior, and regression coverage.
