## Summary

Keep open. Current main still has no supported Feishu `ignoreFileContent` or inbound-media opt-out contract, while Feishu file/media/post resources are still resolved into agent media context; this needs a Feishu plugin config/product decision rather than conservative cleanup.

## What This Changes

The issue asks for Feishu `ignoreFileContent: true` to prevent inbound files or media from being downloaded and attached to model context.

## Best Possible Solution

Define a Feishu-owned inbound media opt-out in schema, types, docs, and runtime, then apply it before `resolveFeishuMediaList` downloads resources while preserving useful placeholders and focused tests.

## Reproduction Assessment

Yes, at source level: current Feishu schema/docs omit `ignoreFileContent`, while inbound file/media/post messages flow through `resolveFeishuMediaList` into `MediaPaths`/`MediaTypes`. I did not run a live Feishu message.

## Solution Assessment

No for immediate automation: treating the undocumented key as an existing bug would bake in a public contract without maintainer choice. The maintainable path is a Feishu-owned config contract with explicit scope and tests.
