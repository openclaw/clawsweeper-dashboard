## Summary

Keep open. Current main and the latest release still do not support a Feishu inbound file-content opt-out, while the Feishu runtime still downloads inbound media and passes media paths into finalized agent context. Because `ignoreFileContent` is not a documented or schema-supported Feishu setting today, this needs maintainer product/config review rather than cleanup closure.

## What This Changes

The issue asks for Feishu `ignoreFileContent: true` to stop inbound file content from being downloaded and included in model context.

## Best Possible Solution

Define a Feishu-owned inbound media opt-out in schema, docs, and runtime, apply it before downloading media while preserving placeholder text, and cover it with focused Feishu tests plus live or Testbox proof.

## Reproduction Assessment

Yes at source level: current main lacks a supported Feishu `ignoreFileContent` or equivalent opt-out, while Feishu media still flows through `resolveFeishuMediaList` into finalized context. I did not run a live Feishu event.

## Solution Assessment

No for an automatic bug fix: the named key is not an existing supported Feishu setting. The maintainable path is a maintainer-approved Feishu inbound-media opt-out with schema, runtime behavior, docs, regression tests, and live proof.
