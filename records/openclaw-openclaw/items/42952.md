## Summary

Keep open. Current main still has no supported Feishu `ignoreFileContent` or inbound-media opt-out, while Feishu media resources still resolve into agent `MediaPaths`/`MediaTypes`; the adjacent max-byte cap work addresses resource limits, not the requested opt-out contract.

## What This Changes

The issue requests a Feishu `ignoreFileContent` setting that prevents inbound file/media content from being downloaded and sent into model context.

## Best Possible Solution

Define a Feishu-owned inbound media opt-out in schema, types, docs, and runtime, then apply it before `resolveFeishuMediaList` downloads resources while preserving useful placeholders and focused tests.

## Reproduction Assessment

Yes, at source level: current Feishu schema/docs omit `ignoreFileContent`, while inbound media messages still flow through `resolveFeishuMediaList` into `MediaPaths`/`MediaTypes`. I did not run a live Feishu message.

## Solution Assessment

No for an automatic bugfix: treating the undocumented key as existing behavior would create a new public config contract. The maintainable path is a Feishu-owned opt-out with explicit scope and regression coverage.
