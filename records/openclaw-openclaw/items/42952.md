## Summary

Keep open. Current main still has no supported Feishu `ignoreFileContent` contract, while inbound Feishu media is downloaded and passed into agent media context; the remaining work is a Feishu config/product decision rather than conservative cleanup.

## What This Changes

Request Feishu to honor an `ignoreFileContent`-style option so inbound files use a placeholder instead of being downloaded and included in model context.

## Best Possible Solution

Define a Feishu-owned inbound media opt-out in schema/docs and apply it before `resolveFeishuMediaList` downloads resources, preserving useful placeholders and adding focused account/top-level tests.

## Reproduction Assessment

Yes, at source level: current Feishu schema/docs omit `ignoreFileContent`, while inbound file/media/post messages flow through `resolveFeishuMediaList` into `MediaPaths`/`MediaTypes` passed to the agent context. I did not run a live Feishu message.

## Solution Assessment

No for immediate automation. Adding this setting requires maintainer choice of option name, scope, inheritance, and media semantics before the narrow Feishu implementation can be written.
