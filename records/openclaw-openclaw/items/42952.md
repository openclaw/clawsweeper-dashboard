The issue requests a way for Feishu `ignoreFileContent: true` to prevent uploaded file contents from being downloaded and sent as model context.

## Best Possible Solution

Have the Feishu/plugin owner decide whether file-content suppression belongs in a Feishu setting, shared `tools.media` policy, or docs/doctor guidance, then implement that contract with focused Feishu media tests.

## Reproduction Assessment

Yes, partially from source: current main still downloads Feishu inbound media and passes saved media paths into the agent turn, while the reported opt-out key is not implemented. No live Feishu account repro was performed, so the exact file-text injection path remains unconfirmed.

## Solution Assessment

Unclear. A direct patch to honor `ignoreFileContent` is only one possible solution; maintainers should first decide whether this belongs as a Feishu-specific config key, a shared media policy, or documentation that the old key is unsupported.
