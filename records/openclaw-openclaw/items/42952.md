Keep open: current main and v2026.6.6 still lack a Feishu `ignoreFileContent` setting or equivalent file-content suppression path, while inbound Feishu files can still be downloaded and projected into agent media context. ## What This Changes

The issue requests that Feishu account config support `ignoreFileContent: true` so inbound Feishu files are not downloaded into model context.

## Best Possible Solution

Add a Feishu-owned, documented account-level inbound media content policy after maintainer approval of the config name, default, inheritance behavior, media coverage, placeholder context, and upgrade expectations.

Do we have a high-confidence way to reproduce the issue? No live Feishu reproduction was run, but source inspection gives a medium-confidence path: Feishu file/media messages still download into `MediaPaths`, and `ignoreFileContent` is not defined in the current Feishu schema or docs.

## Solution Assessment

Is this the best way to solve the issue? Unclear as framed: treating an undocumented config key as a broken shipped setting is not the best strict-bug path. The maintainable solution is a Feishu-owned, documented media-content policy chosen by maintainers.
