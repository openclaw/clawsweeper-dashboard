## Summary

Keep open. Current `main` and the latest release still download Feishu file/media attachments into inbound media context, while `ignoreFileContent` is not a documented or current Feishu config contract, so this is a product/config decision rather than an implemented bug that can be swept closed.

## What This Changes

Request that Feishu `ignoreFileContent: true` stop file content from being downloaded or included in model context, leaving only a file placeholder.

## Best Possible Solution

Define the plugin-owned Feishu inbound attachment policy: either document the current always-download behavior bounded by `mediaMaxMb`, or add a canonical per-account skip/download option with schema, docs, regression tests, and live Feishu proof.

## Reproduction Assessment

Not with high-confidence live proof; the clear reproduction is source-level. Current `main` has no supported `ignoreFileContent` schema entry, and the Feishu media path still downloads file/media resources into inbound media context.

## Solution Assessment

Unclear as an automatic fix. The requested key is not a current documented contract, so the maintainable solution needs a maintainer decision on the supported Feishu attachment policy before code changes.
