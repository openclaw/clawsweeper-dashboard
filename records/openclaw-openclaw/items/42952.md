## Summary

Keep open: current main and the latest release still do not implement an `ignoreFileContent` Feishu setting. Feishu media is still downloaded into saved media paths and added to inbound agent context, while the documented/configured control is only a size cap, so the remaining work is a Feishu plugin config and product-policy decision rather than an already-fixed bug.

## What This Changes

The issue asks for Feishu `ignoreFileContent` configuration to stop file downloads or extracted file content from entering model context and leave only a file notification.

## Best Possible Solution

Define a canonical Feishu plugin media-ingress policy, likely a documented metadata-only/ignore-content mode with schema, docs, and tests, while preserving the current default unless maintainers explicitly change it.

## Reproduction Assessment

Yes at source level, but not through a live Feishu tenant in this review. Current main does not define `ignoreFileContent`, and the Feishu inbound path still downloads supported media and forwards saved media paths into the finalized agent context.

## Solution Assessment

Unclear until maintainers choose the product contract. A narrow plugin-owned media policy is the maintainable path; treating the unrecognized `ignoreFileContent` key as an existing broken contract would be the wrong boundary.
