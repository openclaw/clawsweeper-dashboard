The issue asks for Feishu `ignoreFileContent: true` to prevent inbound file resources from being downloaded and sent to model context.

## Best Possible Solution

Add a Feishu-owned, documented inbound media suppression policy only after maintainer approval, preserving current default download behavior and proving both enabled and unset paths.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection shows `ignoreFileContent` is unsupported and Feishu still resolves inbound media into `MediaPaths`, but the reporter's exact Feishu instance was not exercised.

## Solution Assessment

No, treating this as a drop-in bug fix for an already supported setting is not the best path. The maintainable solution is a maintainer-approved Feishu media policy with schema, docs, focused tests, and live Feishu proof.
