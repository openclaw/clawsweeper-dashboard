## Summary

Keep open: current main and the latest release still have no Feishu `ignoreFileContent` or equivalent inbound-media suppression setting, and the Feishu path still downloads inbound media into prompt/context media fields before the agent turn. This remains useful work, but it needs a maintainer-approved public Feishu config contract and live proof scope before implementation.

## What This Changes

The issue requests a Feishu setting that prevents inbound file/media content from being downloaded and sent to the model context.

## Best Possible Solution

Add a Feishu-owned, documented, account-inheritable inbound media suppression policy after maintainer approval, preserving current download behavior by default and proving both enabled and disabled paths.

## Reproduction Assessment

No, not with high-confidence live proof. Source inspection gives a medium-confidence reproduction path: current Feishu file/media messages still save resources into `MediaPaths`, and `ignoreFileContent` is not a current schema or docs key.

## Solution Assessment

No, not as framed as a broken documented setting. The maintainable solution is a maintainer-approved Feishu media-content policy with explicit config scope, docs, regression tests, and live proof.
