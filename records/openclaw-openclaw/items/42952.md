## Summary

Keep open. Current main still has no Feishu `ignoreFileContent` setting or equivalent inbound attachment-content opt-out, while Feishu file/post attachments still become media paths that shared media-understanding can extract into the agent body.

## What This Changes

Request that Feishu `ignoreFileContent: true` prevent inbound file contents from being downloaded or extracted into model context while leaving a file placeholder.

## Best Possible Solution

Add a Feishu-owned file-content opt-out at top-level and per-account scope, enforce it before document/file attachments enter `MediaPaths`, keep a clear placeholder in the agent body, and document/test the behavior.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: a Feishu `file` or post-embedded media message reaches `resolveFeishuMediaList`, saves an inbound media path, and shared media-understanding then extracts text-like attachments into `ctx.Body`. I did not run a live Feishu webhook replay.

## Solution Assessment

Yes. The narrowest maintainable solution is a Feishu plugin config/schema/docs/test change that gates file/document media resolution before shared media-understanding; adding a Feishu special case in core would cross the owner boundary.
