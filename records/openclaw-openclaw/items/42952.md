## Label Justifications

- `P2`: Automatic Feishu resource attachment can cause context-limit failures, but the blast radius is limited to affected channel sessions and media messages.
- `impact:session-state`: Downloaded inbound file data enters agent context and can exhaust or destabilize long-running session state.

## Summary

Current main still retrieves Feishu message resources and forwards saved media into the agent’s inbound context, while neither current main nor the reported v2026.3.8 release defines `ignoreFileContent`. The context-exhaustion concern is valid, but the requested key would be a new configuration/product contract, so this should remain open for maintainer choice of the canonical policy.

## What This Changes

Add a supported Feishu policy that keeps inbound file messages metadata-only instead of downloading and attaching their contents to model context.

## Best Possible Solution

Define one canonical account-level inbound attachment policy in the Feishu plugin, evaluate it before `saveMessageResourceFeishu`, retain the existing placeholder and filename text when disabled, and document and test both metadata-only and attachment-enabled modes.

## Reproduction Assessment

Yes at source level: v2026.3.8 and current main both contain inbound Feishu resource-download paths, while neither schema recognizes `ignoreFileContent`; no credentialed live Feishu run was performed.

## Solution Assessment

No, the exact requested key is not established as the best contract. The narrow maintainable direction is a deliberately named inbound attachment policy with explicit media scope, defaults, documentation, and pre-download tests.
