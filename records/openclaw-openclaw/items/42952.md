## Summary

Keep open: current main and the latest release still do not implement a Feishu `ignoreFileContent` or equivalent inbound-media suppression setting, and Feishu still downloads inbound file/media resources into `MediaPaths` before the agent turn. This remains a real user problem, but the fix needs maintainer approval for the public Feishu config contract rather than a close or autonomous bug-fix lane.

## What This Changes

The issue requests that Feishu `ignoreFileContent: true` prevent inbound file contents from being downloaded and included in model context.

## Best Possible Solution

Add a Feishu-owned, documented, account-inheritable inbound media suppression policy after maintainer approval, preserving current download behavior by default and proving both enabled and disabled paths.

## Reproduction Assessment

No, we do not have a high-confidence live reproduction. Source inspection gives a medium-confidence path: current Feishu file/media messages still save resources into `MediaPaths`, and `ignoreFileContent` is not a current schema or docs key.

## Solution Assessment

No, not as framed as a broken documented setting. The maintainable solution is a maintainer-approved Feishu media-content policy with explicit config scope, docs, regression tests, and live proof.
