## Summary

Keep open. Current main and the reported v2026.3.8 release do not expose an `ignoreFileContent` Feishu config field, while inbound Feishu media is still downloaded and forwarded into the agent media context; this is a valid request, but it is a new config/product surface rather than a broken supported setting.

## What This Changes

The issue requests that Feishu `ignoreFileContent: true` prevent inbound files from being downloaded or included in model context, leaving only a file placeholder.

## Best Possible Solution

Define and document a Feishu-owned inbound media opt-out, then apply it before media download so matching messages keep placeholders without attaching saved files to the agent turn.

## Reproduction Assessment

Yes, at source level: current Feishu schema/docs have no `ignoreFileContent` field, and file/media messages flow through `resolveFeishuMediaList` into `MediaPaths` and prompt media notes. I did not run a live Feishu message.

## Solution Assessment

No for an automated bug fix. The named setting is not a supported contract on current main or v2026.3.8, so maintainers should first define the supported opt-out name, scope, account inheritance, and audio/post behavior.
