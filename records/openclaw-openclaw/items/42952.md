## Summary

Keep open. Current main still has no supported Feishu `ignoreFileContent` contract, and Feishu inbound media continues to be downloaded and forwarded as agent media context; the request is valid, but it is a new Feishu config/product surface rather than a broken existing setting.

## What This Changes

Request an account-level Feishu setting that suppresses inbound file/media content downloads and sends only a placeholder to the model.

## Best Possible Solution

Define and document a Feishu-owned inbound media opt-out, then apply it before media download so suppressed messages keep useful placeholders without adding saved files to the agent turn.

## Reproduction Assessment

Yes, at source level: current Feishu schema/docs do not support `ignoreFileContent`, while file/media messages still flow through `resolveFeishuMediaList` into saved media paths and prompt media notes. I did not run a live Feishu message.

## Solution Assessment

No for an automated bug fix. The requested setting is not an existing supported contract, so maintainers should first choose the opt-out name, scope, inheritance behavior, and media-type semantics.
