## Summary

Keep open. Current main still has no Feishu `ignoreFileContent` or equivalent opt-out, while Feishu `file` and rich `post` media are still downloaded into `MediaPaths` and can be appended to the agent body by shared media-understanding.

## What This Changes

Request a Feishu setting that prevents inbound file/document contents from being downloaded or added to model context, leaving only a placeholder.

## Best Possible Solution

Add a Feishu-owned, documented top-level and per-account opt-out after maintainer approval of the key name and covered media types, while leaving the generic core media-understanding pipeline unchanged.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: a Feishu `file` or post-embedded media message reaches `resolveFeishuMediaList`, becomes `MediaPaths`, and shared media-understanding can append extracted file blocks to `ctx.Body`. I did not run a live Feishu webhook replay.

## Solution Assessment

Yes directionally, but not yet as an automated fix: a Feishu plugin config/schema/docs/test change is the narrowest maintainable path. The exact config key and media-type scope need maintainer confirmation first.
