## Summary

Keep open. Current main still has no Feishu `ignoreFileContent` setting or equivalent inbound file-content opt-out, and Feishu file/post media still flows into the shared media-understanding path that can append extracted attachment text to the agent body.

## What This Changes

Request a Feishu inbound file-content opt-out so file messages leave placeholder context instead of downloaded and extracted file text.

## Best Possible Solution

Add a Feishu-owned, documented top-level and per-account opt-out that prevents file/document attachments from entering `MediaPaths` while preserving clear placeholder text and leaving the generic core media pipeline unchanged.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: a Feishu `file` or post-embedded media message reaches `resolveFeishuMediaList`, becomes `MediaPaths`, and shared media-understanding can append extracted file blocks to `ctx.Body`. I did not run a live Feishu webhook replay.

## Solution Assessment

Yes, directionally: a Feishu plugin config/schema/docs/test change is the narrowest maintainable path. The exact config name and covered media types need maintainer confirmation before implementation.
