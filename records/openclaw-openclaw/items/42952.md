## Summary

Keep open. Current main still has no Feishu `ignoreFileContent` config or equivalent inbound attachment-content opt-out, and the Feishu inbound path still turns file/post attachments into media paths that shared media-understanding can extract into the agent body.

## What This Changes

The issue requests that `ignoreFileContent: true` in Feishu config prevent incoming Feishu file contents from being downloaded or extracted into model context while leaving a placeholder that a file was sent.

## Best Possible Solution

Add a Feishu-owned config key at both top-level and per-account scope, document it, and enforce it before document/file attachments become `MediaPaths` for shared media-understanding; keep a clear placeholder in the agent body and leave the generic media-understanding pipeline owner-agnostic.

## Reproduction Assessment

Yes. The source-level reproduction is high-confidence: with a Feishu `file` or post-embedded file message, the current handler calls `resolveFeishuMediaList` regardless of any `ignoreFileContent` value and passes resulting media paths to shared file extraction.

## Solution Assessment

Yes. The narrowest maintainable fix is a Feishu plugin gate around inbound file/document media resolution or payload construction, plus focused Feishu regression tests and docs; a core media-understanding special case for Feishu would be a less appropriate owner-boundary fix.
