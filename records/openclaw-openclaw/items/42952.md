## Summary

Keep open: current main and the latest release still have no Feishu `ignoreFileContent` or equivalent inbound-media suppression setting, while Feishu still resolves inbound media into prompt/context media fields before the agent turn. The remaining work is useful, but it needs a maintainer-approved public Feishu config contract and live Feishu proof scope before implementation.

## What This Changes

The issue requests a Feishu account setting that prevents inbound file content from being downloaded and sent to the model context.

## Best Possible Solution

Add a Feishu-owned, documented, account-inheritable inbound media suppression policy after maintainer approval, preserving current download behavior by default and proving both enabled and disabled paths.

## Reproduction Assessment

No, not with high-confidence live proof. Source inspection gives a medium-confidence path: current Feishu file/media messages still save resources into `MediaPaths`, and `ignoreFileContent` is not a current schema or docs key.

## Solution Assessment

No, not as framed as a broken documented setting because current main does not define that key. The maintainable solution is a maintainer-approved Feishu media-content policy with explicit config scope, docs, regression tests, and live proof.
