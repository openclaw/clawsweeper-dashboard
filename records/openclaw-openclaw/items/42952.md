## Summary

Keep open: current main and the latest release still lack a Feishu `ignoreFileContent` or equivalent inbound media suppression setting, while Feishu file/media messages are still saved and projected into agent media context. This matches the prior ClawSweeper comment and remains a maintainer product/config decision rather than a closeable fixed issue.

Requests Feishu `ignoreFileContent: true` to prevent inbound files from being downloaded or included in model context.

## Best Possible Solution

Add a Feishu-owned, documented, account-inheritable inbound media suppression policy after maintainer approval, preserving current download behavior by default and proving both disabled and enabled paths.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? No live Feishu reproduction was run, but source inspection gives a medium-confidence path: current Feishu file/media messages still save resources into `MediaPaths`, and `ignoreFileContent` is not a current schema or docs key.

## Solution Assessment

Is this the best way to solve the issue? No, not as framed as a broken documented setting; the maintainable path is a maintainer-approved Feishu media-content policy with explicit scope, docs, tests, and live proof.
