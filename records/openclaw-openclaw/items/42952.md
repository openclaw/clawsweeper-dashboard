## Summary

Keep open. Current main still has no supported Feishu `ignoreFileContent` or inbound-media opt-out, while Feishu media messages still download into `MediaPaths` that downstream media handling can add to the agent turn; this needs a Feishu config/product decision rather than cleanup closure.

## What This Changes

Request that Feishu `ignoreFileContent: true` prevent file content from being downloaded and included in model context.

## Best Possible Solution

Define a Feishu-owned inbound media opt-out in schema, types, docs, and runtime, apply it before media download, preserve placeholder text, and cover it with focused Feishu tests plus live proof if approved.

## Reproduction Assessment

Yes, at source level: current main has no supported Feishu `ignoreFileContent` key while inbound Feishu media still flows through `resolveFeishuMediaList` into `MediaPaths`/`MediaTypes`. I did not run a live Feishu message.

## Solution Assessment

No for an automatic bugfix: treating this as a broken existing setting would create a new public Feishu config contract. The maintainable path is an explicit Feishu inbound-media opt-out with schema, docs, and regression coverage.
