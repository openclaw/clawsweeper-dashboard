## What This Changes

The issue reports that `openclaw@2026.4.29` still fails bundled plugin runtime-dependency installation on Node v22/npm v10 with `Invalid Version: ''`, breaking plugins that require staged runtime deps.

## Best Possible Solution

Add a targeted runtime-deps installer hardening path so npm v10 on Node v22 can converge the generated package-level dependency plan without leaving bundled plugins unavailable. The fix should keep the existing manifest-first package-level plan, add a regression test for the `Invalid Version:` npm failure, and validate the release-style gateway startup path before the next stable release.
