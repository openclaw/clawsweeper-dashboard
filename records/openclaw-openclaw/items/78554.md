## Summary

Keep open: current main still has the Google Chat auth transport regression path, and this active PR is a focused plugin-local fix with sufficient real behavior proof and no blocking correctness or security findings.

## What This Changes

This PR changes the Google Chat auth runtime, tests, package contract, lockfile, and changelog to use `google-auth-library`'s exported `Gaxios`, preserve native `Request` data through SSRF-guarded fetch, and remove the direct plugin `gaxios` dependency.

## Best Possible Solution

Land the plugin-local auth transport fix with the added regression coverage so https://github.com/openclaw/openclaw/issues/77307 closes on merge, while preserving the existing SSRF guard and avoiding any core API or config change.

## Reproduction Assessment

Yes at source and reporter-proof level: the linked issue provides live Google Chat steps and the `unsupported_grant_type` log, while current main shows how native `Request` data can be lost before Google auth reaches the guarded fetch. I did not run a live Google Chat service-account setup during this read-only review.

## Solution Assessment

Yes. Reusing `google-auth-library`'s exported `gaxios` and preserving `Request` data inside the existing Google Chat SSRF-guarded transport is the narrowest maintainable fix; no core seam, config option, or product decision is needed.
