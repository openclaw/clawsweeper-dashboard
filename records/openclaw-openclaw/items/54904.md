## Summary

Keep open: this is the right Feishu-owned fix path for a real webhookPath bug, but the branch still compares a raw safe incoming path against a normalized configured path, so same-route trailing-slash callbacks can be rejected after upgrade.

## What This Changes

This PR adds a Feishu webhook request-path gate and regression tests for wrong, malformed, network-path, and configured webhook paths.

## Best Possible Solution

Normalize safe incoming origin-form paths with normalizeWebhookPath after malformed/network-path rejection, add a trailing-slash regression, and then land the early Feishu route gate.

## Reproduction Assessment

Yes by source inspection. Current main and v2026.6.10 never compare the incoming request path to webhookPath before body/signature handling, and PR head would compare /feishu/events/ unequal to normalized /feishu/events.

## Solution Assessment

No. Enforcing Feishu routing before body and signature work is the right layer, but the branch should normalize safe incoming origin-form paths before comparison so same-route trailing-slash callbacks are not broken.
