## Best Possible Solution

Land a rebased Feishu plugin patch that enforces the normalized configured path before body parsing and dispatch, uses the modern webhook-ingress SDK subpath, and includes redacted runtime proof of wrong-path rejection plus configured-path success.

## Reproduction Assessment

## Solution Assessment

Yes, mostly. The early Feishu-owned path check is the right boundary; the patch should switch the new import from the deprecated webhook-path alias to openclaw/plugin-sdk/webhook-ingress before merge.
