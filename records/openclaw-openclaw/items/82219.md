## Summary

Keep open: the latest docs commit resolves the earlier docs mismatch, but the patch still introduces a migration polling regression. Target plugin inventory now returns as ready when any allowed marketplace appears, while migration still auto-plans only curated plugins, so curated plugin migration can be skipped if bundled or primary-runtime inventory appears first.

## What This Changes

The PR widens Codex native plugin marketplace handling from only `openai-curated` to include bundled and primary-runtime first-party marketplaces across config, runtime, migration, docs, changelog, and focused tests.

## Best Possible Solution

Land the expanded first-party marketplace contract only after preserving curated-only migration readiness until non-curated source migration is intentionally added.

## Reproduction Assessment

Yes. Current main has a source-level reproduction path for the reported rejection: non-curated `marketplaceName` values fail the Codex plugin schema and never reach policy resolution; the new PR regression is also source-reproducible from the changed readiness predicate.

## Solution Assessment

No, not yet. The centralized allowlist with matching docs/tests is the right shape if maintainers accept the boundary, but migration polling should remain keyed to the selected plugin marketplace instead of any supported marketplace.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep migration polling tied to the selected marketplace:** `extensions/codex/src/migration/apply.ts:322`
  - body: This now treats `plugin/list` as ready as soon as any allowed marketplace is present. Migration still auto-plans only source-installed `openai-curated` plugins, so a target inventory that exposes `openai-bundled` or `openai-primary-runtime` before `openai-curated` will stop polling early and then skip the selected curated plugin as `marketplace_missing`. Keep this readiness check tied to the planned plugin's marketplace, or keep it curated-only until non-curated source migration is added.
