## Summary

Keep open: the PR addresses a real current-main limitation in Codex native plugin config, but it widens a documented V1 support boundary without updating the public config/native-plugin docs.

## What This Changes

The PR widens Codex native plugin marketplace validation from only `openai-curated` to include `openai-bundled` and `openai-primary-runtime`, then updates activation, inventory, session binding, migration apply, and focused tests around that allowlist.

## Best Possible Solution

Adopt one typed first-party marketplace allowlist only if maintainers accept that support boundary, and keep the manifest, runtime validation, migration scope, and public docs in sync.

## Reproduction Assessment

Yes. Current main has a source-level reproduction path: a `codexPlugins` entry using `openai-bundled` or `openai-primary-runtime` fails schema validation and is dropped by `resolveCodexPluginsPolicy`, though I did not execute it in this read-only review.

## Solution Assessment

Partly. A shared typed allowlist is the right implementation shape if the first-party marketplace boundary is accepted, but the docs and support contract need to be aligned before merge.
