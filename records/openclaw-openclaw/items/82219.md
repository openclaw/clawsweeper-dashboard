## Summary

Keep open: the PR is a useful active fix path, but it currently makes migration polling treat any allowed first-party marketplace as readiness even though automatic migration still selects curated plugins only. That can make a partially loaded target inventory skip a curated plugin with `marketplace_missing` before `openai-curated` appears.

## What This Changes

The PR widens native Codex plugin config to accept `openai-curated`, `openai-bundled`, and `openai-primary-runtime`, threads marketplace identity through Codex plugin resolution paths, and updates docs, changelog, and focused tests.

## Best Possible Solution

Land a centralized allowlist only after migration polling remains keyed to the selected curated plugin or marketplace, with focused regression coverage and docs kept aligned with the accepted support boundary.

## Reproduction Assessment

Yes. Current main has a source-level reproduction path: non-curated `marketplaceName` values fail schema validation and are then absent from resolved Codex plugin policy; the review finding is also source-reproducible from the changed migration readiness predicate.

## Solution Assessment

No, not as written. The shared allowlist is a reasonable direction if maintainers accept the boundary, but migration polling should not stop on unrelated first-party marketplaces while automatic migration remains curated-only.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Wait for the selected migration marketplace:** `extensions/codex/src/migration/apply.ts:322`
  - body: Migration still creates automatic plugin install items from `openai-curated` only, but this helper now stops polling as soon as any allowed marketplace is visible. If `openai-bundled` or `openai-primary-runtime` loads before `openai-curated`, activation sees the selected curated plugin's marketplace as missing and skips config that would have migrated after the target inventory finished loading. Keep this readiness check tied to the selected policy marketplace/plugin, or leave it curated for migration, and add a partial-inventory regression test.
  - confidence: 0.88
