## Summary

Keep open: current main still has source-backed subagent risks. Thread-bound omitted context still falls back to `fork` until https://github.com/openclaw/openclaw/pull/79050 lands, and announce delivery still routes direct or queued completion turns without an age or session-lineage guard for stale delivery; the maintainer narrowing comment after https://github.com/openclaw/openclaw/pull/79726 explicitly says this issue should stay open for those remaining paths.

## What This Changes

Prevent subagent completion announces from replaying stale output into requester sessions and keep new subagent transcripts from inheriting unrelated history.

## Best Possible Solution

Land or supersede https://github.com/openclaw/openclaw/pull/79050 for implicit thread-bound isolation, then add a generic freshness/session-lineage guard in subagent announce delivery without changing the plugin contract.

## Reproduction Assessment

Yes by source inspection, though not by a live end-to-end replay in this read-only review. Omitted thread-bound `sessions_spawn` still resolves to `fork` on current main, and queued announces still deliver after the busy deferral cap instead of proving requester-session freshness.

## Solution Assessment

Yes: keeping this open is the right outcome because current main only removed the custom WebSocket branch, while the subagent announce/session-history branch remains live. The narrowest maintainable fix is the open isolated-default PR plus a follow-up lineage/freshness guard in the announce path.
