## Summary

Keep open: current main still has the thread-bound spawn path that can fork requester history, and the announce queue still has a source-visible stale-delivery path; the merged WebSocket refactor only covers the later custom-WS branch of the report.

## What This Changes

Prevent stale subagent completion announcements from becoming requester replies and ensure spawned subagent sessions do not inherit unrelated transcript history.

## Best Possible Solution

Land or replace the thread-bound isolation fix from https://github.com/openclaw/openclaw/pull/79050, then add a narrow freshness/session-lineage guard for subagent announce delivery or prove that path obsolete on current main.

## Reproduction Assessment

No high-confidence live reproduction was run in this read-only review. The current-main source path is reproducible by inspection: omitted thread-bound context still resolves to `fork`, and queued announces still deliver after the busy-parent deferral cap without an age or lineage drop.

## Solution Assessment

Yes, keeping this issue open is the best current outcome. Closing it from the merged WebSocket refactor would lose the remaining subagent announce/session-history behavior, while the active PR only covers the inherited thread-bound context branch.
