## Summary

Keep open: current main and the latest release still do not expose or read a Feishu `reactionLevel`, and existing Feishu reaction work covers manual reaction actions/API plumbing rather than this requested config contract.

## What This Changes

Request Feishu support for a `reactionLevel` config that controls automatic or agent-driven emoji reaction behavior.

## Best Possible Solution

Define the Feishu reaction-level contract first, then implement it inside the Feishu plugin using the shared reaction-level helper and align focused tests plus Feishu/reactions docs with the chosen behavior.

## Reproduction Assessment

Yes, at source level: current Feishu config and runtime only gate manual reaction actions through `actions.reactions`, and no Feishu `reactionLevel` reader or prompt-guidance hook exists. I did not run a live Feishu bot, so this is not a live reproduction.

## Solution Assessment

Unclear. Feishu can likely reuse the shared reaction-level pattern, but maintainers should first choose whether the config controls model reaction guidance, automatic ack/status reactions, or both.
