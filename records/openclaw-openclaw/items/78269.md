## Summary

Keep open: current main exposes Feishu manual `react`/`reactions` actions behind `actions.reactions`, but it does not define, document, or read a Feishu `reactionLevel` setting or add reaction-guidance prompt behavior for Feishu.

## What This Changes

The issue requests Feishu channel support for a `reactionLevel` configuration that controls automatic or agent-guided emoji reactions.

## Best Possible Solution

Implement Feishu-owned `reactionLevel` parity using the existing reaction-level helper and channel prompt seam, with Feishu docs and tests covering top-level and per-account behavior.

## Reproduction Assessment

Yes, at source level: current main has Feishu reaction actions but no Feishu `reactionLevel` schema, resolver, docs, or prompt guidance. I did not run a live Feishu bot because this is a missing feature rather than a current contract failure with a runnable local repro.

## Solution Assessment

Yes, a Feishu-local implementation is the narrowest maintainable direction. It should reuse the existing `resolveReactionLevel` and `agentPrompt.reactionGuidance` patterns instead of adding core channel-specific behavior.

Keep the change Feishu-owned: add `reactionLevel: off | ack | minimal | extensive` to `extensions/feishu/src/config-schema.ts` for top-level and account config, add a small Feishu resolver using `resolveReactionLevel` from `openclaw/plugin-sdk/text-runtime`, wire `agentPrompt.reactionGuidance` in `extensions/feishu/src/channel.ts`, and gate agent `react`/`reactions` discovery/handling so `off`/`ack` disable agent-controlled reactions while `minimal`/`extensive` enable guidance. Preserve existing `actions.reactions` behavior as the hard action gate, do not move Feishu-specific behavior into core, and avoid solving unrelated Feishu reaction API bugs already tracked by #66406/#66478 and #34528/#77653. Add focused tests for schema acceptance, account override behavior, discovery/gating, and prompt guidance. Update Feishu user docs and add a user-facing changelog entry if behavior ships.
