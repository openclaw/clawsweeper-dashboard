## Summary

Keep open: current main has Feishu reaction actions and reaction-event ingestion, but it does not expose or resolve `channels.feishu.reactionLevel` like the existing Telegram, WhatsApp, and Signal implementations.

## What This Changes

Request Feishu/Lark support for `reactionLevel` so `minimal`/`extensive` enables agent reaction guidance and `off`/`ack` suppresses agent-controlled emoji reactions.

## Best Possible Solution

Add Feishu-owned `reactionLevel` config, resolver, prompt guidance, action gating, tests, docs, and changelog without changing unrelated channel reaction semantics.

## Reproduction Assessment

Yes from source inspection: current main has no Feishu `reactionLevel` schema field or resolver, while the Feishu runtime only reads `actions.reactions`. I did not use a live Feishu account, so this is source-reproducible rather than live-reproduced.

## Solution Assessment

Yes: the narrow maintainable path is to implement this inside the bundled Feishu plugin using the existing shared reaction-level resolver and channel prompt/action hooks. A core change is not needed unless maintainers want new cross-channel ack behavior.
