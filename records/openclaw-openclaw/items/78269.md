## Summary

Keep open: current main still lacks Feishu `reactionLevel` config, resolver, prompt guidance, and action gating; related Feishu reaction issues and PRs address API reliability rather than this requested auto-reaction level.

## What This Changes

Add Feishu channel `reactionLevel` config so `minimal` and `extensive` can enable agent-driven emoji reactions automatically.

## Best Possible Solution

Implement this inside the bundled Feishu plugin using the shared reaction-level resolver, Feishu config/schema metadata, `agentPrompt.reactionGuidance`, reaction action gating, focused tests, docs, and a changelog entry.

## Reproduction Assessment

Yes, from source inspection on current main: the strict Feishu schema and plugin metadata have no `reactionLevel`, and `feishuPlugin.agentPrompt` does not provide `reactionGuidance`. I did not run a live Feishu account, so this is source-reproducible rather than live-reproduced.

## Solution Assessment

Yes: the narrow maintainable path is Feishu-owned parity with Telegram, WhatsApp, and Signal using the existing shared resolver and channel prompt/action hooks. A core architecture change is not needed for the requested behavior.
