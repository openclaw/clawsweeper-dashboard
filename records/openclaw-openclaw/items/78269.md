## Summary

Keep open: current main and v2026.6.6 still lack a Feishu `reactionLevel` runtime/config contract; existing related Feishu work covers manual reaction actions, native reaction tools, or reaction API bugs rather than this requested automatic level behavior.

## What This Changes

Request to implement Feishu `reactionLevel` so configured levels control automatic emoji reaction behavior instead of only exposing manual reaction actions.

## Best Possible Solution

Define the Feishu reaction-level product contract first, then implement it inside `@openclaw/feishu` with the shared resolver, focused Feishu tests, docs, and live Feishu proof.

## Reproduction Assessment

Yes, at source level: current Feishu schema, docs, prompt wiring, and runtime action paths have no `reactionLevel` reader or automatic level-based behavior. I did not run a live Feishu bot.

## Solution Assessment

Unclear. Reusing the shared reaction-level pattern is plausible, but maintainers need to decide whether Feishu levels should govern agent reaction guidance, ACK/status reactions, explicit reaction actions, or all of them.
