## Summary

Keep open: current main and the latest release still do not implement a Feishu `reactionLevel` contract, and related Feishu reaction items cover manual reaction actions or status-reaction lifecycle work rather than this requested config behavior.

## What This Changes

Requested behavior is for Feishu `reactionLevel` (`off`/`ack`/`minimal`/`extensive`) to control automatic or agent-guided emoji reactions instead of relying only on manual reaction actions.

## Best Possible Solution

Define the Feishu reaction-level product contract first, then implement it inside `@openclaw/feishu` with the shared resolver, focused Feishu tests, and public docs aligned to the chosen behavior.

## Reproduction Assessment

Yes, at source level for the missing capability: current Feishu schema, docs, and runtime have no `reactionLevel` reader or reaction-guidance hook. I did not run a live Feishu bot, and current main appears to reject or omit the key rather than silently honor it.

## Solution Assessment

Unclear. Feishu can likely reuse the shared reaction-level pattern, but maintainers first need to choose whether the setting controls prompt-guided reactions, ACK/status reactions, or both.
