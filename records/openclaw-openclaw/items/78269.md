## Summary

Keep open: current main and the latest tagged release still lack Feishu `reactionLevel` config, metadata, prompt guidance, and runtime gating; the related open Feishu reaction PRs address API reliability or typing-reaction behavior, not this policy/config feature.

## What This Changes

The issue requests Feishu `reactionLevel` support so automatic emoji reaction behavior can be controlled like other reaction-capable channels.

## Best Possible Solution

Implement Feishu-owned `reactionLevel` parity in the bundled plugin using the shared resolver, config schema/metadata, prompt guidance, action gating, docs, focused tests, and live or high-fidelity Feishu proof.

## Reproduction Assessment

Yes. Source inspection on current main shows Feishu has reaction actions gated only by `actions.reactions`, but no Feishu `reactionLevel` field, resolver, or prompt guidance; I did not run a live Feishu workspace.

## Solution Assessment

Yes. The narrow maintainable path is Feishu-plugin parity with the existing shared reaction-level resolver and channel prompt/action hooks, coordinated with the open Feishu reaction API fixes rather than a core architecture change.
