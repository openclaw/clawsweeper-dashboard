## Summary

Keep open: current main and v2026.6.8 still do not implement a Feishu `reactionLevel` runtime/config contract; related Feishu reaction work covers manual/native reaction plumbing or API fixes, not automatic level semantics.

## What This Changes

Request to add Feishu `reactionLevel` config semantics so configured levels automatically drive emoji reaction behavior on inbound messages.

## Best Possible Solution

Define the Feishu reaction-level product contract first, then implement it in `@openclaw/feishu` with schema, plugin metadata, docs, focused tests, and live Feishu proof.

## Reproduction Assessment

Yes, at source level: current Feishu schema, plugin metadata, runtime action handler, and inbound reaction monitor have no Feishu `reactionLevel` reader or automatic level-based branch. I did not run a live Feishu tenant.

## Solution Assessment

Unclear. Reusing the shared reaction-level resolver is plausible, but maintainers need to decide whether Feishu levels control prompt guidance, ack/status reactions, explicit actions, or all of them.
