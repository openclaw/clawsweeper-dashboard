## Summary

Keep open: current main still lacks Feishu `reactionLevel` schema, metadata, prompt guidance, and runtime gating, and the related open Feishu reaction PRs address API reliability rather than this requested policy/config feature.

## What This Changes

Add Feishu `reactionLevel` config so the Feishu channel can control automatic or agent-driven emoji reactions with `off`, `ack`, `minimal`, and `extensive` levels.

## Best Possible Solution

Implement Feishu-owned `reactionLevel` parity in the bundled plugin using the shared resolver, config schema/metadata, prompt guidance, action gating, docs, focused tests, and a changelog entry.

## Reproduction Assessment

Yes, from source inspection: current main has Feishu reaction actions but no Feishu `reactionLevel`, `reactionGuidance`, or resolver wiring. I did not run a live Feishu workspace, so this is source-reproducible rather than live-reproduced.

## Solution Assessment

Yes: the narrow maintainable path is Feishu-plugin parity with the existing shared reaction-level resolver and channel prompt/action hooks. A core architecture change is not needed, but the implementation should coordinate with the open Feishu reaction API reliability fixes.
