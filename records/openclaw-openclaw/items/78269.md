Keep open: current main still lacks Feishu `reactionLevel` schema, metadata, prompt guidance, and action gating, while the related Feishu reaction PRs address API reliability rather than this auto-reaction policy.

## What This Changes

Add a Feishu channel `reactionLevel` config that controls automatic emoji reactions for inbound messages.

## Best Possible Solution

Implement Feishu-owned `reactionLevel` parity in the bundled plugin using the shared resolver, config schema/metadata, prompt guidance, action gating, docs, focused tests, and a changelog entry.

## Solution Assessment
