## Summary

Keep open: the protected `maintainer` label applies, and the patch has a source-level regression in Telegram poll validation plus missing real behavior proof.

## What This Changes

The PR refactors Telegram outbound delivery into a shared outbound adapter, centralizes Telegram DM/command authorization helpers, and replaces local grammY fallbacks with native `API_CONSTANTS` and `InputFile` usage.

## Best Possible Solution

Keep the refactor open until it preserves the existing Telegram outbound contract, adds focused coverage for the poll option cap, and includes redacted real Telegram behavior proof.

## Reproduction Assessment

Yes. Source inspection shows that a Telegram poll with more than the channel cap no longer gets early CLI/gateway normalization once the PR adapter omits `pollMaxOptions`; I did not run the branch because this was a read-only review.

## Solution Assessment

No. The adapter consolidation is a reasonable direction, but it needs to preserve `pollMaxOptions` and add a regression check before it is the narrowest maintainable solution.
