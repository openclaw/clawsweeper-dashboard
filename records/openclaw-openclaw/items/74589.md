## Summary

Keep open because current `main` still builds Anthropic-compatible payloads from the converted message array without a final non-empty guard, and open PR #74731 is the linked implementation candidate rather than a reason to close before merge.

## What This Changes

Fix intermittent MiniMax M2.7 `chat content is empty` failures by preventing empty Anthropic-compatible message payloads after tool-call-heavy turns.

## Best Possible Solution

Land PR #74731 or an equivalent final non-empty guard at the Anthropic-compatible payload boundary, with focused regression coverage and a changelog entry.

## Reproduction Assessment

Yes for the core source-level failure path, but not for a deterministic live MiniMax session. Current `main` can still convert an empty or fully filtered replay into `messages: []` before sending the Anthropic-compatible request.

## Solution Assessment

Yes. A final guard in `buildAnthropicParams` or an immediately adjacent payload helper is the narrowest maintainable fix because it covers MiniMax and other strict Anthropic-compatible endpoints without provider-specific retry or transcript repair.
