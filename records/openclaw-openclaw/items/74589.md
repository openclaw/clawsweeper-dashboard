## Summary

Keep open. Current main still lacks a final non-empty guard at the Anthropic-compatible payload boundary, MiniMax is routed through that transport, and the provided GitHub context shows open PR #74731 is the focused closing implementation candidate rather than a merged fix.

## What This Changes

The issue asks OpenClaw to prevent MiniMax M2.7 Anthropic-compatible requests from sending an empty `messages` payload after filtered tool-call chains, avoiding 400/2013 errors and polluted session state.

## Best Possible Solution

Land PR #74731 or an equivalent final non-empty guard in `src/agents/anthropic-transport-stream.ts`, with regression coverage in `src/agents/anthropic-transport-stream.test.ts`; keep the broader `/new` empty-prompt race tracked under #73926.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? No for a deterministic live MiniMax session because the report is intermittent and lacks a minimal transcript; yes for the core code path because current main can convert empty or fully filtered replay input into an empty Anthropic `messages` array and forward it.

## Solution Assessment

Is this the best way to solve the issue? Yes: a final guard at the Anthropic-compatible transport payload boundary is narrower and more maintainable than MiniMax-specific retry logic or session-store repair, while preserving the separate `/new` prompt-guard work.
