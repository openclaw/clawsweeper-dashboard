## What This Changes

The issue reports intermittent MiniMax M2.7 400/2013 empty chat content errors after tool-call chains and requests a provider-boundary guard for empty Anthropic-compatible message payloads.

## Best Possible Solution

Land #74731 or an equivalent final non-empty guard at the Anthropic-compatible payload boundary, with focused regression coverage and a user-facing changelog entry.

## Reproduction Assessment

Yes for the core payload failure path, but not for a deterministic live MiniMax session. Current main can still convert empty or fully filtered replay history into `messages: []` for an Anthropic-compatible request.

## Solution Assessment

Yes. A final guard in `buildAnthropicParams` or an adjacent payload helper is the narrowest maintainable fix because it covers MiniMax and other strict Anthropic-compatible endpoints without provider-specific retry logic or transcript mutation.
