Keep open because current main still lets the Anthropic-compatible transport send an empty converted `messages` array, and the linked open PR #74731 is the implementation candidate that should be reviewed or landed before this issue closes.

## What This Changes

The issue requests preventing MiniMax Anthropic-compatible requests from sending an empty `messages` array after replay/tool-call filtering.

## Best Possible Solution

Review and land PR #74731, or an equivalent final non-empty guard at `buildAnthropicParams` with focused regression coverage and a changelog entry.

## Reproduction Assessment

Yes for the source-level failure path, unclear for a deterministic live MiniMax session. Current main can still convert empty or fully filtered replay history into `messages: []` and submit it through the Anthropic-compatible transport.

## Solution Assessment

Yes. A final guard at the Anthropic-compatible payload boundary is the narrowest maintainable fix because it covers MiniMax and other strict compatible endpoints without provider-specific retries or transcript repair.
