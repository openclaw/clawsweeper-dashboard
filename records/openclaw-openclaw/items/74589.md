## Summary

Keep open. Current main still builds Anthropic-compatible payloads from the converted message array without a final non-empty guard, MiniMax uses that transport, and open PR #74731 is already the focused closing implementation candidate for this report.

## What This Changes

The issue asks for MiniMax M2.7 Anthropic-compatible requests to stop reaching the provider with an empty `messages` array after tool-call replay/filtering.

## Best Possible Solution

Land PR #74731 or an equivalent final non-empty guard at the Anthropic-compatible transport payload boundary, with regression coverage in `src/agents/anthropic-transport-stream.test.ts`; keep the broader `/new` empty-prompt race tracked separately under #73926.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? No for a deterministic live MiniMax session because the report is intermittent and lacks a minimal transcript; yes for the core code path because current main can build an empty converted Anthropic `messages` array from empty or fully filtered replay content.

## Solution Assessment

Is this the best way to solve the issue? Yes: a final guard at `src/agents/anthropic-transport-stream.ts` is the narrowest maintainable fix because it protects MiniMax and other Anthropic-compatible providers after all replay filtering has happened.
