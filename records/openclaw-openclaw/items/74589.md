## Summary

Keep open because current main still forwards the converted Anthropic-compatible message array without a final non-empty guard, and #74731 is an open closing PR that should be reviewed or landed before this issue closes.

## What This Changes

The issue reports intermittent MiniMax M2.7 400/2013 errors after multi-tool-call turns, likely when the Anthropic-compatible transport sends an empty converted messages array.

## Best Possible Solution

Review and land #74731, or an equivalent final guard in the Anthropic-compatible payload builder, with focused regression coverage and a changelog entry.

## Reproduction Assessment

Yes for the source-level failure path: an empty or fully filtered replay can produce `messages: []`, and current main forwards that array. The live MiniMax sequence is still intermittent rather than deterministically reproduced.

## Solution Assessment

Yes. A final non-empty guard at `buildAnthropicParams` or an immediately adjacent payload helper is the narrowest maintainable fix because it protects MiniMax and other strict Anthropic-compatible endpoints without provider-specific retry logic.
