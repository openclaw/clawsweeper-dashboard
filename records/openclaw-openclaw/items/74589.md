## Summary

Keep open. Current main still allows the Anthropic-compatible transport to produce and forward an empty `messages` array, MiniMax is configured to use that transport, and open PR #74731 is the focused closing implementation candidate rather than a reason to close this issue before merge.

## What This Changes

The issue requests preventing MiniMax M2.7 Anthropic-compatible requests from sending an empty `messages` array after tool-call-heavy turns.

## Best Possible Solution

Review and land PR #74731, or an equivalent final non-empty guard at the Anthropic-compatible transport payload boundary, with regression coverage in `src/agents/anthropic-transport-stream.test.ts` and user-facing changelog coverage if the fix ships separately.

## Reproduction Assessment

Yes for the core payload condition, but not for a deterministic live MiniMax session. Current main can build an Anthropic-compatible request whose converted `messages` array is empty when replay content is empty or fully filtered, matching the reported MiniMax 2013 failure condition.

## Solution Assessment

Yes. A final guard at the Anthropic-compatible transport payload boundary is the narrowest maintainable fix because it covers MiniMax and other strict Anthropic-compatible endpoints without adding MiniMax-specific retry logic or session-store repair.
