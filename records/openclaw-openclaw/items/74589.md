## What This Changes

The report asks OpenClaw to prevent MiniMax/Anthropic-compatible provider requests from sending an empty `messages` payload after tool-call chains.

## Best Possible Solution

Review and land PR #74731, or an equivalent final non-empty guard in `buildAnthropicParams` with focused regression coverage and a changelog entry.

## Solution Assessment

Yes. A final guard at the Anthropic-compatible payload boundary is the narrowest maintainable fix because it covers MiniMax and other strict compatible endpoints without provider-specific retries or session repair.
