## Summary

Keep open. Current main still lacks a final non-empty invariant for Anthropic-compatible payloads, and the issue is already paired with open closing PR #74731 for the same transport-boundary guard.

## What This Changes

The issue requests a MiniMax/Anthropic-compatible transport guard so filtered tool-call histories cannot be sent as an empty `messages` array after chained tool calls.

## Best Possible Solution

Land #74731 or an equivalent narrow guard in `src/agents/anthropic-transport-stream.ts` that enforces a non-empty Anthropic Messages payload after conversion, with regression coverage for empty or fully filtered converted histories.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? No for a deterministic live MiniMax session because the report is intermittent; yes for the core path, since current main can produce and forward an empty converted Anthropic `messages` array by inspection.

## Solution Assessment

Is this the best way to solve the issue? Yes: the narrow maintainable fix is a final transport-boundary guard after conversion, rather than changing session storage, MiniMax-specific retries, or downstream plugin hooks.
