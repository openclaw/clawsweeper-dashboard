## Summary

Keep open because current `origin/main` still forwards the converted Anthropic-compatible message array without a final non-empty guard, and open PR #74731 is the linked implementation candidate rather than a reason to close before merge.

## What This Changes

Prevent MiniMax M2.7 Anthropic-compatible requests from sending `messages: []` after tool-call-heavy sessions, avoiding 400/2013 errors and transcript pollution.

## Best Possible Solution

Land PR #74731 or an equivalent final non-empty guard at the Anthropic-compatible payload boundary, with focused regression coverage and a changelog entry.

## Reproduction Assessment

Yes for the core failure path, but not for a deterministic live MiniMax session. Current main can still build an Anthropic-compatible request whose converted `messages` array is empty when replay history is empty or fully filtered.

## Solution Assessment

Yes. A final guard in `buildAnthropicParams` or an adjacent payload helper is the narrowest maintainable fix because it covers MiniMax and other strict Anthropic-compatible endpoints without provider-specific retry or transcript repair.
