## Summary

Keep open: current main and the latest release still only provide timer-based inbound debouncing plus active-run queue modes, not the requested user-controlled continuation marker that buffers marked fragments until an unmarked final message arrives.

## What This Changes

Request to add a configurable message continuation marker and timeout so marked message fragments are buffered and sent to the agent as one batched turn.

## Best Possible Solution

Keep the current timer-based batching as the default, and only add a minimal marker/timeout contract in shared inbound handling or a typed plugin hook after maintainers choose that product boundary.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing behavior report. Source inspection confirms current main follows documented timer debounce and active-run queue behavior, not marker-controlled batching.

## Solution Assessment

No, not yet; the requested marker, queue mode, and typing mode are implementation options rather than an approved canonical contract. Maintainers should first choose whether this belongs in core inbound handling, a narrow plugin/API seam, or remains covered by timer-based batching.
