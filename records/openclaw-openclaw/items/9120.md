## Summary

Current main still lacks user-controlled continuation-marker batching: it only provides quiet-window inbound coalescing and active-run queue modes. The request remains useful but needs a maintainer product decision because it adds cross-channel session semantics and new configuration rather than fixing broken documented behavior.

## What This Changes

Add a configurable continuation marker and timeout that buffer marked inbound fragments and submit them with the next unmarked message in one agent turn.

## Best Possible Solution

Select one canonical pre-agent batching owner—preferably the existing inbound coalescing path or a narrow typed plugin hook—then add only the minimum marker/timeout surface with explicit command, media, ordering, timeout, typing, and transcript tests.

## Reproduction Assessment

Not applicable; current main follows its documented timer-based debounce and queue behavior, while this issue requests a new user-controlled input mode.

## Solution Assessment

No, not yet; the proposed marker, queue mode, and typing mode are multiple possible implementations, and maintainers should first choose the narrowest canonical owner and contract.
