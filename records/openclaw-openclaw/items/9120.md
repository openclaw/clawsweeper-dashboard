## Summary

Keep open: current main and the latest release still expose timer-based inbound debounce and active-run collect batching, but not the requested user-controlled continuation marker, marker config, or continuation-specific typing mode.

## What This Changes

Requests a configurable marker such as `(cont)` that buffers multi-message input until an unmarked final message so OpenClaw can make one agent turn instead of several.

## Best Possible Solution

Define the shared continuation-batching contract first; if accepted, implement typed config/schema/docs plus marker parsing, marker stripping, timeout flushing, queue/typing integration, and focused inbound coverage.

## Reproduction Assessment

Not applicable. This is an enhancement request, and current-main source inspection shows the requested marker config and parser are absent rather than a broken existing contract.

## Solution Assessment

Unclear. The marker workflow is plausible, but maintainers need to decide whether shared core config, a plugin/channel seam, or existing debounce/collect modes should be the supported product direction.
