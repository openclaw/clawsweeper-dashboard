## Summary

Current main provides timer-based inbound debouncing and active-run collect queueing, but neither implements user-controlled continuation fragments; the request remains useful but needs maintainer judgment because it adds cross-channel policy and configuration rather than fixing broken existing behavior.

## What This Changes

Add an explicit continuation marker that buffers marked inbound message fragments until a later unmarked message triggers one agent turn.

## Best Possible Solution

Define one canonical cross-channel inbound aggregation contract that composes with debounce and collect behavior, including explicit flush and lifecycle semantics; if continuation syntax is optional policy, add the smallest generic plugin hook instead of hard-coding another core batching path.

## Reproduction Assessment

## Solution Assessment

No, not as currently specified. A configurable suffix is plausible UX, but the maintainable solution must first choose the canonical owner and define how it composes with existing debounce and collect lifecycles.
