## Summary

Keep open. Current main and v2026.6.10 still have timer-based inbound debounce and active-run collect queueing, but not the requested marker-controlled continuation buffer that waits for an unmarked final message before invoking the agent.

## What This Changes

Request adds a configurable message continuation marker that batches marked inbound fragments into one later agent turn for token-cost and UX savings.

## Best Possible Solution

Keep this as the focused marker-controlled continuation design item until maintainers approve a shared inbound/plugin API contract or explicitly decide that existing timer-based batching is the intended product answer.

## Reproduction Assessment

Not applicable: this is a feature request rather than a broken existing behavior report. Source inspection gives high-confidence evidence that current main supports timer-based inbound debounce and active-run collect batching, but not marker-controlled continuation buffering.

## Solution Assessment

No, not as an automatic fix: the proposed marker, timeout, queue mode, and typing mode are plausible but add core config and session turn semantics. The best path is maintainer-approved shared inbound or plugin/API continuation behavior, or an explicit product decision that existing batching is enough.
