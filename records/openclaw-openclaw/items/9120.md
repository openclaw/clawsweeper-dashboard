## Summary

Keep open. Current main still has timer-based inbound debouncing and active-run queue batching, but not the requested user-controlled continuation marker that buffers marked fragments until a later unmarked message; prior reviews in the issue discussion reached the same conclusion for https://github.com/openclaw/openclaw/issues/9120.

## What This Changes

The issue requests a configurable continuation marker that batches multiple inbound user messages before invoking the agent.

## Best Possible Solution

Define the cross-channel continuation contract first, then, if accepted, add typed config/schema/docs plus shared inbound buffering that strips markers and defines timeout, media, command, queue, and typing/status behavior.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence the requested marker-config and marker-parsing behavior is absent on current main.

## Solution Assessment

Unclear until maintainers choose the product/API shape. A shared inbound helper or documented plugin/SDK seam would be more maintainable than channel-specific ad hoc buffering.
