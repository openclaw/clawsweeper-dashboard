## Summary

Keep open: current main and v2026.5.22 still provide timer-based inbound debouncing and active-run queue batching, but not the requested user-controlled continuation marker that buffers marked fragments until a later unmarked message arrives.

## What This Changes

Request adding an opt-in configurable continuation marker that buffers marked inbound message fragments until a later unmarked message is sent as one agent turn.

## Best Possible Solution

Decide the cross-channel product/API contract first, then implement opt-in shared inbound continuation-marker handling with schema, types, docs, and focused behavior tests.
