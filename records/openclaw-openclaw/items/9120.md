## Summary

Keep open. Current main still has timer-based inbound debouncing and active-run collect queueing, but not the requested user-controlled continuation marker; existing comments and labels also route this to maintainer/product follow-up.

## What This Changes

Requests a configurable `(cont)`-style marker that buffers rapid multi-message input until an unmarked final message arrives, reducing agent turns.

## Best Possible Solution

Keep this open for a product decision; if accepted, define the semantic batching contract and implement it in shared inbound handling with typed config, schema, docs, and focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection against current main and latest release confirms existing debounce/collect paths, but no marker-based hold-and-flush behavior.

## Solution Assessment
