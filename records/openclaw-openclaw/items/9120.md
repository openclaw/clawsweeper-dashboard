## Summary

Keep open: current main still has timer-based inbound debouncing and active-run collect batching, but not the requested user-controlled continuation marker, marker config, or continuation-specific typing behavior. Prior Codex review comments on the issue reached the same conclusion and no closing implementation is linked.

## What This Changes

The issue requests a configurable message continuation marker that buffers marked inbound message fragments until a later unmarked message arrives, reducing redundant agent turns and context loads.

## Best Possible Solution

Define the cross-channel continuation-batching contract first; if accepted, implement typed config/schema/docs plus marker parsing, marker stripping, timeout flushing, queue/typing integration, and focused inbound coverage.
