## Summary

Keep open. Current main still has timer-based inbound debouncing and active-run collect queueing, but it does not implement user-controlled continuation markers that buffer marked fragments until a later unmarked message; the prior maintainer-side review in the issue also kept this open for the same missing behavior.

## What This Changes

Request to add configurable continuation-marker buffering so multi-part inbound messages can be batched into one agent invocation with optional typing suppression.

## Best Possible Solution

Define the shared continuation contract first, then implement typed config/schema/docs, marker stripping, timeout flushing, queue and typing interactions, and focused regression coverage in the inbound/queue path.

## Reproduction Assessment

Not applicable. This is an enhancement request, and source inspection on current main shows the requested marker config and marker parser are absent rather than a broken existing contract.

## Solution Assessment

Unclear. The requested direction is plausible, but adding a marker, timeout, stripping semantics, and typing behavior is a product/API decision before a narrow implementation should proceed.
