## Summary

Keep open. Current main still has timer-based inbound debouncing and active-run collect queueing, but not the requested user-controlled continuation marker that buffers marked fragments until a later unmarked message; prior automated maintainer-side reviews in the issue discussion reached the same conclusion.

## What This Changes

The request proposes configurable continuation marker and timeout settings, plus optional typing behavior, so multipart user messages can be buffered and sent through one agent turn.

## Best Possible Solution

Define the cross-channel continuation contract first, then implement shared inbound buffering with typed config, marker stripping, timeout flushing, and documented behavior for media, commands, queues, and typing/status indicators.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence the requested behavior is absent: the schema lacks the fields, inbound debounce flushes by timer, and inbound policy does not parse suffix markers.

## Solution Assessment

Unclear until maintainers choose the product/API shape. A shared inbound helper or documented plugin/SDK seam would be more maintainable than channel-specific ad hoc buffering.
