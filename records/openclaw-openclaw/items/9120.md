## Summary

Keep open: current main and v2026.6.10 still provide timer-based inbound debouncing and active-run queue modes, but not the requested user-controlled continuation marker that buffers marked fragments until an unmarked final message.

## What This Changes

Request a configurable message continuation marker that buffers multi-part inbound messages until an unmarked final message, then sends one batched agent turn.

## Best Possible Solution

Keep this as the focused marker-controlled continuation design item until maintainers approve a shared inbound or plugin API contract, or explicitly decide that timer-based batching is the intended product answer.
