## Summary

Keep open. Current main has removed `socat`, but the sandbox browser CDP path still uses a derived Chromium port plus an authenticated relay, so the remaining direct-CDP simplification is an architecture/security decision rather than completed cleanup.

## What This Changes

Evaluate whether the sandbox browser CDP path can remove the relay/port-offset layer and route directly to Chromium while preserving readiness and access control.

## Best Possible Solution

Have sandbox/browser owners either document the authenticated relay as the intentional replacement for socat or design a direct-CDP migration that preserves auth, loopback publishing, readiness, and ingress guarantees.

## Reproduction Assessment

Not applicable; this is an architecture/evaluation request, not a runtime bug. Static current-main inspection gives high confidence that `socat` is gone but the derived-port authenticated relay design remains.

## Solution Assessment

Unclear. Removing `socat` has already happened, but removing the relay entirely is not clearly the best solution without owner review because the relay now enforces CDP authentication and optional source filtering.
