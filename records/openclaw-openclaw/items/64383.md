## Summary

Keep open. Current main still uses the sandbox browser socat relay, derived Chromium CDP port, loopback host publish, and cdpSourceRange ingress model that this architecture cleanup asks maintainers to evaluate, so there is no conservative close reason.

## What This Changes

Evaluate removing the sandbox browser socat CDP relay and port-offset design in favor of direct Chromium CDP routing, with entrypoint, Docker, docs, and config follow-through if accepted.

## Best Possible Solution

Keep this as the sandbox browser CDP architecture tracker: either migrate to direct CDP with equivalent ingress, readiness, lifecycle, and bridge-auth guarantees, or document why the relay remains intentional and close after that decision.

## Reproduction Assessment

Not applicable; this is an architecture/evaluation request rather than a runtime bug. Static current-main inspection gives high confidence that the socat relay, CDP port offset, and cdpSourceRange design still exist.

## Solution Assessment

Unclear until maintainer architecture/security review decides whether direct CDP is the desired end state. The narrow maintainable path is either a deliberate direct-CDP migration with equivalent controls or explicit documentation that the relay remains intentional.
