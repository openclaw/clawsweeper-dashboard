## Summary

Keep open. Current main and the latest release have removed `socat`, but the sandbox browser CDP path still intentionally uses a derived Chromium port behind an authenticated relay, so the remaining direct-CDP simplification is a security-sensitive product decision rather than completed cleanup.

## What This Changes

Evaluate simplifying the sandbox browser CDP path by removing the remaining relay/port indirection while preserving access control and readiness guarantees.

## Best Possible Solution

Have sandbox/browser owners either document the authenticated relay as the intended post-`socat` design or approve a direct-CDP migration that preserves auth, loopback publishing, source-range filtering, and readiness guarantees.

## Reproduction Assessment

Not applicable; this is an architecture/evaluation request rather than a runtime bug. Static source, tests, docs, and release inspection show `socat` is gone but direct CDP is still not implemented because Chromium remains behind an authenticated relay.

## Solution Assessment

Unclear. Removing `socat` has already happened, but removing the relay entirely is not clearly the best solution without owner and security review because current code and tests rely on it for CDP auth and optional ingress filtering.
