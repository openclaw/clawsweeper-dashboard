## Summary

Keep open. Current main and the latest release have removed `socat`, but the sandbox browser CDP path still deliberately uses a derived Chromium port behind an authenticated Python relay, so the remaining direct-CDP simplification is a security-sensitive product decision rather than completed cleanup.

## What This Changes

Evaluate whether the sandbox browser CDP startup path should remove the proxy layer and route CDP directly to Chromium while preserving readiness and access controls.

## Best Possible Solution

Have sandbox/browser owners either document the authenticated relay as the intentional post-socat design or approve a direct-CDP migration that preserves auth, loopback publishing, source-range filtering, and readiness guarantees.

## Reproduction Assessment

Not applicable; this is an architecture/evaluation request, not a runtime bug. Static source, tests, docs, and release inspection provide high confidence that `socat` is gone but direct CDP is not implemented because Chromium still sits behind an authenticated relay.

## Solution Assessment

Unclear. Removing `socat` has already happened, but removing the relay entirely is not clearly the best solution without owner and security review because current code and tests rely on the relay for CDP auth and optional ingress filtering.
