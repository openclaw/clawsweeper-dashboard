## Summary

Keep open. Current main and v2026.6.10 have removed the literal socat dependency from the sandbox-browser image, but the CDP path still routes the configured port through a generated-token Python relay to a derived Chromium loopback port, so the remaining direct-CDP simplification is still a sandbox/browser security and product decision.

## What This Changes

Evaluate whether the sandbox browser CDP startup path should remove its relay/proxy layer and route CDP directly to Chromium.

## Best Possible Solution

Have sandbox/browser owners either document the authenticated Python relay as the intentional successor to socat, or approve a direct-CDP migration that preserves equivalent auth, source filtering, loopback publishing, readiness checks, tests, and docs.

## Reproduction Assessment

Not applicable; this is an architecture/evaluation request rather than a concrete runtime bug. Source, docs, tests, release inspection, and related PR history give high confidence that socat is gone but direct CDP is not implemented.

## Solution Assessment

Unclear. Removing socat has already happened, but removing the authenticated relay entirely is not clearly the best solution without owner and security review because current code and tests rely on that relay for CDP auth and optional ingress filtering.
