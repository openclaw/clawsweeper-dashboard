## Summary

Keep open. Current main still uses the sandbox browser socat relay, derived Chromium CDP port, loopback Docker publishing, and `cdpSourceRange` ingress model, so this remains an unresolved architecture/security evaluation rather than a cleanup-close candidate.

## What This Changes

Evaluate whether the sandbox browser CDP path can remove the socat relay and route CDP directly to Chromium while preserving readiness, lifecycle, access-control, and documentation behavior.

## Best Possible Solution

Keep this as the sandbox browser CDP architecture tracker until owners either migrate to direct CDP with equivalent ingress and lifecycle guarantees, or document why the relay remains intentional.

## Reproduction Assessment

Not applicable; this is an architecture/evaluation request rather than a runtime bug. Static inspection of current main gives high confidence that the socat relay, CDP port offset, and `cdpSourceRange` design still exist.

## Solution Assessment

Unclear until owner architecture/security review decides whether direct CDP is the desired end state. The maintainable path is either a deliberate direct-CDP migration with equivalent controls or explicit documentation that the relay remains intentional.
