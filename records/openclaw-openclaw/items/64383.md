## Summary

Keep open. Current main and the latest release have removed the old `socat` dependency from the sandbox-browser surface, but the requested direct-CDP simplification is not complete because the runtime still uses a derived Chromium loopback port behind an authenticated Python relay; replacing that relay needs sandbox/browser owner and security review.

## What This Changes

Evaluate whether the sandbox browser CDP path can remove its remaining proxy/relay indirection and route CDP directly to Chromium.

## Best Possible Solution

Have sandbox/browser owners either document the authenticated relay as the intentional post-socat design or approve a direct-CDP migration that preserves auth, loopback publishing, source filtering, and readiness guarantees.

## Reproduction Assessment

Not applicable; this is an architecture/evaluation request rather than a concrete runtime bug. Static source, tests, docs, release inspection, and related PR history give high confidence that `socat` is gone but direct CDP is not implemented.

## Solution Assessment

Unclear. Removing `socat` has already happened, but removing the authenticated relay entirely is not clearly the best solution without owner and security review because current code and tests rely on the relay for CDP auth and optional ingress filtering.
