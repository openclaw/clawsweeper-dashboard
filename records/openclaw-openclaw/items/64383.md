## Summary

Close as implemented: current main and v2026.5.12 removed socat from the sandbox browser CDP path via https://github.com/openclaw/openclaw/pull/81002 and replaced the old proxy with an authenticated relay, which is now the chosen CDP ingress boundary.

## What This Changes

Evaluate whether sandbox browser CDP can remove the socat relay and route directly to Chromium, moving any remaining responsibilities to Docker binding, bridge/auth, and readiness checks.

## Best Possible Solution

Keep the shipped no-socat authenticated relay design; open a narrower security architecture proposal only if maintainers want to replace the relay entirely.

## Reproduction Assessment

Not applicable; this was an architecture evaluation request rather than a runtime bug. Static inspection of current main gives high confidence that the old socat path is gone and the relay-auth CDP contract is present.

## Solution Assessment

Yes for this cleanup: current main removes socat while preserving CDP ingress controls through an authenticated relay and optional cdpSourceRange. Routing directly to Chromium without an equivalent boundary would be a separate security design, not the shipped solution.
