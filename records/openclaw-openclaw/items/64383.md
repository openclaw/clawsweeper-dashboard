## Summary

Keep open. Current main and the latest release have removed `socat`, but the sandbox browser CDP path still uses a derived Chromium loopback port behind an authenticated Python relay, so the remaining direct-CDP simplification is a security-sensitive product decision rather than completed cleanup.

## What This Changes

The issue requests evaluating whether the sandbox browser CDP path can drop its remaining proxy/port-indirection design and connect directly to Chromium.

## Best Possible Solution

Have sandbox/browser owners either document the authenticated relay as the intentional post-socat design or approve a direct-CDP migration that preserves auth, loopback publishing, source-range filtering, and readiness guarantees.

## Reproduction Assessment

Not applicable; this is an architecture/evaluation request rather than a runtime bug. Static source, tests, docs, release inspection, and related PR history give high confidence that `socat` is gone but direct CDP is not implemented.

## Solution Assessment

Unclear. Removing `socat` has already happened, but removing the authenticated relay entirely is not clearly the best solution without owner and security review because current code and tests rely on the relay for CDP auth and optional ingress filtering.
