## Summary

Keep open. Current main has removed `socat`, but the sandbox browser CDP path still uses a derived Chromium port behind an authenticated Python relay, and the remaining direct-CDP decision is a security-sensitive architecture choice rather than completed cleanup.

## What This Changes

Evaluate removing the sandbox browser CDP proxy and port-offset design now that readiness and keyring fixes have landed.

## Best Possible Solution

Have sandbox/browser owners either document the authenticated relay as the intentional replacement for `socat` or design a direct-CDP migration that preserves the existing auth, ingress, loopback, and readiness guarantees.

## Reproduction Assessment

Not applicable; this is an architecture/evaluation request, not a runtime bug. Static current-main inspection gives high confidence that `socat` is gone but direct CDP is not implemented because Chromium still sits behind an authenticated relay.

## Solution Assessment

Unclear. Removing `socat` has already happened, but removing the relay entirely is not clearly the best solution without owner/security review because current code and tests rely on the relay for CDP auth and optional ingress filtering.
