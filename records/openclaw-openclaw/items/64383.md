## Summary

Keep open. Current main removed the literal socat dependency, but the sandbox browser CDP path still intentionally routes through a generated-token Python relay on a derived Chromium loopback port, so direct CDP remains a sandbox/browser security and product decision.

## What This Changes

Evaluate whether to remove the sandbox browser CDP relay and port-offset path, or document the authenticated relay as the intended successor to socat.

## Best Possible Solution

Have sandbox/browser owners either document the authenticated Python relay as the intended design, or approve a direct-CDP migration that preserves equivalent auth, source filtering, loopback publishing, readiness checks, tests, and docs.

## Reproduction Assessment

Not applicable; this is an architecture/evaluation request rather than a concrete runtime bug. Source, docs, tests, and related PR history give high confidence that socat is gone but direct CDP is not implemented.

## Solution Assessment

Unclear. Literal socat removal is already done, but removing the authenticated relay entirely is not clearly the best solution without owner and security review because current code and tests rely on that relay for CDP auth and optional ingress filtering.
