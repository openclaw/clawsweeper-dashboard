## Summary

Keep open. Current main has removed the `socat` dependency, but the sandbox browser CDP path still uses a derived Chromium CDP port plus an authenticated relay, so the remaining direct-CDP simplification is an architecture/security decision rather than completed cleanup.

## What This Changes

Evaluate whether the sandbox browser CDP path can remove the intermediate relay/port-offset design and route CDP more directly while preserving readiness and ingress controls.

## Best Possible Solution

Have sandbox/browser owners either document the authenticated relay as the intentional replacement for socat, or design a direct-CDP migration that preserves auth, loopback publishing, readiness, and ingress guarantees.

## Reproduction Assessment

Not applicable; this is an architecture/evaluation request, not a runtime bug. Static current-main inspection gives high confidence that `socat` is gone but the derived-port authenticated relay design remains.

## Solution Assessment

Unclear. Replacing `socat` with an authenticated Python relay resolves one part of the request, but removing the relay entirely is not obviously the best solution without owner security review.
