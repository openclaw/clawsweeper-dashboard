## Summary

Keep open. Current main and v2026.6.10 have removed `socat` from the sandbox-browser runtime, but the central simplification is not complete because CDP still runs through a derived Chromium loopback port and an authenticated Python relay; replacing or blessing that relay needs sandbox/browser owner and security review.

## What This Changes

Evaluate whether the sandbox browser CDP path should remove the remaining proxy/relay indirection and route CDP directly to Chromium while preserving readiness, auth, and exposure controls.

## Best Possible Solution

Have sandbox/browser owners either document the authenticated Python relay as the intentional post-socat design or approve a direct-CDP migration that preserves auth, loopback publishing, optional source filtering, and readiness guarantees.

## Reproduction Assessment

Not applicable; this is an architecture/evaluation request rather than a concrete runtime bug. Static source, tests, docs, release inspection, and related PR history give high confidence that `socat` is gone but direct CDP is not implemented.

## Solution Assessment

Unclear. Removing `socat` has happened, but removing the authenticated relay entirely is not clearly the best solution without owner and security review because current code and tests rely on that relay for CDP auth and optional ingress filtering.
