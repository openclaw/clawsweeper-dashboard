## Summary

Current main still has the configured ACP collision path for inherited Discord thread routing: the parent-channel fallback identity is materialized into the configured ACP record, and the persistent key hashes that parent conversation id. This should stay open while the active paired fix PR #64322 is reviewed, landed, replaced, or closed.

## What This Changes

The issue asks for Discord configured ACP persistent bindings on parent-channel routes to resolve to distinct per-thread session keys instead of one shared parent-channel key.

## Best Possible Solution

Inherited Discord parent-channel ACP bindings should materialize persistent sessions using the live thread identity while preserving direct parent-channel and direct thread bindings, with regression coverage proving sibling thread keys differ.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: resolve two Discord configured binding routes with different messageChannelId values and the same earlyThreadParentId, and both can materialize through the parent conversation id into the same configured ACP key.

## Solution Assessment

Yes. The narrow maintainable fix is to make the inherited Discord parent-channel match preserve the live thread identity for materialization, rather than changing the global hash format or adding a new configuration knob.
