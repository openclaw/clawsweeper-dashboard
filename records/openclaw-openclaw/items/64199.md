## Summary

Current main still has the configured ACP collision path for inherited Discord thread routing, and the paired same-author fix PR #64322 remains open and unmerged. This issue should stay open until that PR or an equivalent narrow fix lands.

## What This Changes

The issue requests per-thread persistent ACP session keys for Discord threads that inherit a configured parent-channel ACP binding.

## Best Possible Solution

Review and land #64322 or an equivalent fix that materializes inherited Discord parent-channel ACP bindings with the live thread id, preserves direct bindings, and adds sibling-thread key regression coverage plus the required changelog entry.

## Reproduction Assessment

Yes. A high-confidence source path exists: resolve two configured Discord ACP bindings with different thread conversation ids and the same parentConversationId; current main’s parent fallback materializes both through the parent id and therefore the same configured ACP key.

## Solution Assessment

Yes. The narrow maintainable fix is to adjust the Discord inherited parent-channel match/materialization path so the live thread conversation id drives the persistent ACP key; changing the global key hash alone would risk unrelated channel remapping behavior.
