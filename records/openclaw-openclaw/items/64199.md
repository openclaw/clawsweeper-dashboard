## Summary

Current main still has the configured ACP collision path for Discord thread inheritance, and the same-author fix PR #64322 remains open and unmerged. This should stay open until that PR, or an equivalent narrow fix, lands.

## What This Changes

The issue asks for Discord parent-channel ACP configured bindings to create distinct persistent session keys for each child thread instead of sharing one parent-channel ACP process.

## Best Possible Solution

Keep this open until #64322 or an equivalent narrow fix lands: the Discord parent-match path should materialize inherited thread bindings with the live thread id, preserve channel-canonical remaps for other plugins, and add regression coverage plus the required changelog entry.

## Reproduction Assessment

Yes. A high-confidence source path exists: resolve two Discord configured binding routes with different messageChannelId values and the same earlyThreadParentId; both materialize through the parent conversation id and hash to the same configured ACP session key.

## Solution Assessment

Yes. The narrow maintainable fix is to adjust Discord's inherited parent-channel match identity to carry the runtime thread conversation id while keeping the shared registry contract intact; changing the global key hash would risk other channel remapping semantics.
