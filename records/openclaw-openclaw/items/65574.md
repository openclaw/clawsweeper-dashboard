Keep open: this targets a real MCP stdio proxy/NODE_OPTIONS inheritance bug, but the branch is stale against current main's stdio env safety filtering and needs maintainer security review plus real after-fix proof before merge.

## What This Changes

The PR adds MCP stdio launch-time propagation of ambient proxy-related environment variables and sanitized NODE_OPTIONS from the gateway process, plus tests for precedence, empty values, and inspect-flag filtering.

## Best Possible Solution

Land a refreshed current-main fix that propagates safe ambient proxy defaults with user-config precedence, preserves toMcpEnvRecord/onDroppedEnv filtering, and explicitly documents/tests any accepted host NODE_OPTIONS exception.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: start the gateway with HTTPS_PROXY or NODE_OPTIONS, configure a stdio MCP server without explicit env, and current main resolves no ambient env before spawning with SDK defaults that omit those keys.

## Solution Assessment

No, not as currently proposed. The resolver boundary is right, but the branch must be rebased around current toMcpEnvRecord/onDroppedEnv filtering and the NODE_OPTIONS inheritance policy.
