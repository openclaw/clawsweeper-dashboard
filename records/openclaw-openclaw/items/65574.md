## Summary

Keep open: the underlying MCP stdio proxy/NODE_OPTIONS inheritance bug is valid and current main still does not add ambient proxy defaults, but this PR is not safe to merge as-is because it bypasses current stdio env filtering and lacks after-fix real behavior proof.

## What This Changes

The branch adds MCP stdio launch-time propagation of gateway proxy/NODE_OPTIONS env defaults plus tests for precedence, empty proxy values, and inspect-flag filtering.

## Best Possible Solution

Land a refreshed current-main fix that propagates safe ambient proxy defaults with user-config precedence, preserves toMcpEnvRecord/onDroppedEnv filtering, and deliberately documents/tests any accepted host NODE_OPTIONS exception.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: start the gateway with HTTPS_PROXY or NODE_OPTIONS, configure a stdio MCP server without explicit env, and current main resolves no ambient proxy/default startup env before the transport adds only the MCP SDK allowlist.

## Solution Assessment

No, not as currently proposed. The resolver boundary is right, but the branch must be rebased around current toMcpEnvRecord/onDroppedEnv filtering and the NODE_OPTIONS inheritance policy.
