## What This Changes

The issue requests stable gateway self-restart/update respawn arguments so package-manager realpaths are not reused after updates.

Add a focused update respawn entrypoint planner that rewrites OpenClaw package-manager realpaths to stable package wrappers while preserving source/dev behavior and the managed-service handoff contract.

Yes. The narrow maintainable fix is a helper-level respawn argument planner that reuses or mirrors the existing stable entrypoint resolver without changing broader gateway restart policy.
