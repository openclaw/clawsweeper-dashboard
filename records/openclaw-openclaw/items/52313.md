## Summary

Keep open: current main still respawns detached gateway restart/update children with `process.execPath`, `process.execArgv`, and `process.argv.slice(1)`, so a pnpm versioned `dist/*` realpath can still be reused after package replacement. The closed related PR #52319 targeted the right problem but was not merged, and the adjacent open reports cover different restart/cutover failures rather than superseding this one.

## What This Changes

Gateway self-restart/update should respawn through a stable OpenClaw wrapper or package entrypoint instead of reusing package-manager versioned argv paths.

## Best Possible Solution

Add a focused respawn-argument planner in `src/infra/process-respawn.ts` that resolves OpenClaw package installs to a stable wrapper/package entrypoint for both normal SIGUSR1 restarts and `update.run`, while preserving direct dev/source entrypoints and covering pnpm realpath cases with regression tests.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to set `process.argv[1]` to a pnpm versioned OpenClaw `dist/*` realpath and observe that current main passes that same path into `spawn()` from `spawnDetachedGatewayProcess()`.

## Solution Assessment

Yes. The requested stable-entrypoint rewrite is the narrow maintainable fix, but it should be scoped to respawn planning and tests rather than replaying the broader closed PR wholesale.
