## What This Changes

## Best Possible Solution

Land the narrow runtime-first search gate after redacted provider-path proof and owner confirmation, then let the linked bug close from the merged fix.

Yes at source level: current main can set `preferRuntimeProviders` to false for a manifest-owned configured search provider, and the agent tool forwards that into `runWebSearch`. I did not live-reproduce the multi-hour gateway uptime failure in this read-only review.

## Solution Assessment

Likely yes, but not fully merge-proven. Removing the premature search manifest-owner lookup while preserving the fetch bundled-owner gate is the narrowest code path found; live provider-path proof and owner confirmation remain open.
