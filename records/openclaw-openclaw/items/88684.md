## Summary

## What This Changes

The PR changes agent `web_search` runtime-context resolution so search always prefers the live runtime provider registry while `web_fetch` keeps bundled-owner gating, and updates focused tests for that split.

## Best Possible Solution

Land this focused runtime-first search fix after redacted provider-path proof and provider/runtime owner confirmation, or explicitly supersede it with a merged broader routing contract.

## Solution Assessment

Likely yes, but not fully merge-proven. Keeping agent `web_search` runtime-first while preserving the `web_fetch` bundled-owner gate is the narrowest fix shape found; the remaining gaps are live provider-path proof and owner confirmation of the routing contract.
