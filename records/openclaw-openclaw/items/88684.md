## Summary

Keep open: current main still has the agent-side search owner gate that can bypass runtime provider discovery, and the PR’s code shape matches the downstream runtime scoping path, but merge should wait on live provider-path proof and maintainer provider-routing signoff.

## What This Changes

The PR keeps agent-side `web_search` on runtime provider discovery for configured manifest-owned search providers while leaving `web_fetch` bundled-owner gating and focused tests aligned.

## Best Possible Solution

Land the narrow runtime-first search gate after redacted live provider-path proof or explicit maintainer acceptance, while keeping the fetch bundled-owner guard unchanged.

## Solution Assessment

Yes, this appears to be the right layer: the agent tool should stop preempting runtime discovery, and `runWebSearch` already scopes configured provider loading by manifest ownership downstream. The remaining blocker is live proof and provider-routing signoff, not a concrete code defect in the patch.
