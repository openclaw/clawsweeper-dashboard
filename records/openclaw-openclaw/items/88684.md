The PR changes agent web_search runtime-context resolution so configured search providers keep runtime-provider discovery enabled, while preserving bundled-owner gating for web_fetch and updating focused tests.

Yes for code shape: removing the agent-tool manifest-owner gate for search while leaving runWebSearch's scoped runtime loading and web_fetch's bundled-owner gate intact is the narrowest maintainable fix I found. Merge readiness still depends on live proof and provider-owner sign-off.
