## Summary

Keep open: the patch is a plausible narrow fix for the linked production `web_search` provider-routing bug, but it still needs maintainer/provider-owner confirmation and stronger live provider-path proof before merge.

## What This Changes

This PR makes agent-side `web_search` always keep runtime provider discovery enabled while leaving bundled-owner `web_fetch` gating in place and updating focused tests.

## Best Possible Solution

Land the narrow runtime-first `web_search` gate only after redacted live provider-path proof and confirmation that search should intentionally differ from fetch.

## Reproduction Assessment

Yes at source level: current main can set `preferRuntimeProviders` to false for a manifest-owned configured search provider, and `createWebSearchTool` forwards that into `runWebSearch`. I did not live-reproduce the multi-hour gateway uptime failure in this read-only review.

## Solution Assessment

Likely yes, but not fully merge-proven. Removing the premature search manifest-owner lookup while preserving the fetch bundled-owner gate is the narrowest code path found; the remaining blockers are live provider-path proof and owner confirmation.
