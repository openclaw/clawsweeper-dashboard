## Summary

Keep open: current main and the latest release still create running cron task records, but cancellation falls through to the unsupported-runtime branch for cron. The existing CLI, Gateway RPC, docs, and Web UI task surfaces make this a valid operator-control bug rather than cleanup that can be closed.

## What This Changes

The issue reports that running cron tasks appear in the task ledger but `openclaw tasks cancel <taskId>` cannot cancel them and returns an unsupported-runtime error.

## Best Possible Solution

Make the live Gateway cron runtime own cancellation for active cron task rows and make CLI/Web UI cancellation report the same runtime result, with a clear escape hatch when the Gateway no longer owns the cron run.

## Reproduction Assessment

Yes. Source inspection shows current and shipped code create `runtime: "cron"` task records while task cancellation falls through to the unsupported-runtime branch for non-ACP/subagent/CLI records; this review did not run a live Gateway cron job.

## Solution Assessment

Yes for keeping this open and routing a bounded fix. The maintainable solution is a cron-runtime-owned abort/finalize path through the existing task cancellation surface, not a local registry-only status flip.
