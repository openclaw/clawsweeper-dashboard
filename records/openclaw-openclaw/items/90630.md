## What This Changes

The issue reports that running cron tasks appear in the task ledger but cannot be cancelled through `openclaw tasks cancel`.

## Best Possible Solution

Make the existing task cancel contract work for active cron task records by handing cancellation to the Gateway-owned cron runtime; keep an explicit unsupported/offline escape-hatch message only when no active Gateway cron handle can be reached.

## Reproduction Assessment

Yes from source. Current main and v2026.6.1 create `runtime: "cron"` task rows, and the cancellation path returns the exact reported unsupported-runtime error for cron tasks; I did not run a live Gateway cron job in this read-only review.

## Solution Assessment

Yes, the best fix is to wire active cron runs into the existing task cancellation contract rather than adding a separate operator command. If maintainers decide runtime cancellation is out of scope, the fallback should be a clearer error and docs, but that would not solve the runaway-run problem.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/90630. On current main, cron creates running `runtime: "cron"` task rows but `openclaw tasks cancel <taskId>` and Gateway `tasks.cancel` return `Task runtime does not support cancellation yet.` Implement cancellation for active cron task records through the Gateway-owned cron runtime using the existing task cancel contract and operator scope; do not add a config/env option. Reuse the cron timeout abort path where possible so cancellation aborts the underlying run, clears active-job ownership, finalizes the task as `cancelled` with the operator reason, and keeps cron run/job history coherent. Preserve existing ACP, subagent, childless Codex-native subagent, and CLI cancellation behavior. If the CLI cannot reach an active Gateway-owned cron runtime, report a clear unsupported/offline escape hatch rather than silently marking the task cancelled. Keep the Gmail MCP cleanup symptom out of scope unless the cron cancellation fix directly exposes a cleanup regression.
