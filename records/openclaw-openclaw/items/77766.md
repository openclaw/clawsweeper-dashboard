## Summary

Keep open: current main intentionally treats user-pinned session models as strict selections, so the requested fallback-only pin exception needs a maintainer policy decision rather than conservative cleanup.

## What This Changes

The issue asks OpenClaw to preserve or warn about configured fallback behavior when a session is pinned to a model that appears only in `agents.defaults.model.fallbacks`.

## Best Possible Solution

Decide and document whether user-pinned fallback models should remain strict, warn when the chain collapses, or use a defined next candidate, then align code and regression tests with that policy.

## Reproduction Assessment

Yes, source-reproducible: a user session model override is passed into fallback resolution, which returns an empty fallback override, leaving the runner with only the requested candidate. I did not run a live timeout transcript in this read-only review.

## Solution Assessment

No as an automatic bug fix: adding the configured primary behind a user-pinned model would conflict with the current strict session-selection contract. The safer path is a maintainer policy decision followed by code, docs, and tests that encode the chosen behavior.
