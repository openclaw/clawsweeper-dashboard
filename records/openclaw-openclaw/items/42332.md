## Summary

Keep open: current main still accepts direct and mapped same-session hook deliveries with 200/runId; idempotency replay and session-lane queueing do not provide the requested caller-visible 409/skip admission contract.

## What This Changes

Add a sessionKey-scoped webhook overlap guard so repeated hook deliveries can receive a conflict or skip signal instead of starting another accepted run.

## Best Possible Solution

Define an explicit, compatibility-safe webhook overlap policy at the gateway hook admission boundary, covering active and queued same-session work while preserving today's default unless maintainers approve a breaking change.

## Reproduction Assessment

Yes. Source inspection shows two direct or mapped hook-agent requests with the same resolved sessionKey and no shared idempotency key are both accepted with 200 and distinct run IDs instead of a conflict.

## Solution Assessment

No for an unconditional default 409. The best path is a maintainer-approved policy, likely opt-in, that decides reject, skip, queue, and compatibility semantics at admission time.
