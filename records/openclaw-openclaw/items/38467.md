## Summary

This is a useful WhatsApp contact-resolution idea, but the requested persistent phone-contact directory is an optional, privacy-sensitive WhatsApp capability that prior maintainer review already kept out of the bundled OpenClaw WhatsApp plugin.

## What This Changes

Expose WhatsApp phone contacts from Baileys contact events through directory lookup and name-based message target resolution.

## Best Possible Solution

Leave the bundled WhatsApp directory config-backed and pursue phone-contact sync as a ClawHub/wacli-style plugin or a plugin-API discussion if an external implementation proves a missing seam.

## Reproduction Assessment

Not applicable as a feature request. Source inspection confirms current main has no WhatsApp contact-store population path, but that is an unimplemented capability rather than a broken documented contract.

## Solution Assessment

Yes for cleanup: closing this as ClawHub/plugin-scope is the best current outcome because a prior contact-store PR was explicitly rejected for project-scope and privacy reasons, while plugin/skill paths remain available.
