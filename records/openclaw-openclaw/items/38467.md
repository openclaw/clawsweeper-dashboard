## Summary

Keep open: current main still has only config-backed WhatsApp peer directory entries, and there is no WhatsApp contact-event persistence path feeding directory/name resolution.

## What This Changes

Request WhatsApp to persist Baileys phone-contact updates and expose them through directory peers and outbound name resolution.

## Best Possible Solution

Add an account-scoped, opt-in WhatsApp contact cache in the WhatsApp plugin, populate it from validated Baileys contact data, expose it through directory peers, and document/test name-collision and privacy behavior.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives a high-confidence current-state check: WhatsApp peers are allowFrom-backed and there is no contacts.upsert handler.

## Solution Assessment

Unclear until maintainers decide the privacy and opt-in model. The narrow maintainable direction is WhatsApp-plugin-owned contact caching feeding the existing directory and target-resolver seams, not a core migration.
