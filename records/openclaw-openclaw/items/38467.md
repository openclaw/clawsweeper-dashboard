## Summary

Keep open: current main still has only config-backed WhatsApp peer directory entries, while the requested phone-contact ingestion is a new WhatsApp plugin capability with a privacy/defaults decision around persisting a local address book.

## What This Changes

Expose WhatsApp phone contacts received from Baileys as directory peers so name-based outbound target resolution can find contacts like “Emily”.

## Best Possible Solution

Implement this in the bundled WhatsApp plugin after the privacy policy is decided: persist minimal account-scoped contact records locally, expose them through `directory.listPeers`, and let the existing outbound directory resolver handle name matches.

## Reproduction Assessment

Not applicable as a strict bug reproduction, but the capability gap is source-reproducible: current WhatsApp peers come from `allowFrom`, and no contact event handler or contact store exists on current main.

## Solution Assessment

Yes on boundary, unclear on policy: the maintainable code path is WhatsApp-plugin-owned storage plus directory exposure, not a core resolver rewrite, but the default/opt-in choice needs maintainer review first.
