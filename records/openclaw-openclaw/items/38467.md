## Summary

Keep open. Current main still exposes WhatsApp peer directory entries from configured allowFrom values only, does not subscribe to or persist Baileys contacts.upsert data, and therefore cannot resolve saved WhatsApp contact names through the directory or message target resolver.

## What This Changes

Request to add an opt-in WhatsApp contact directory backed by Baileys contacts.upsert so saved phone contacts can be listed and resolved by name for outbound messages.

## Best Possible Solution

Add a WhatsApp-plugin-owned, account-scoped contact store that listens to Baileys contact events, persists local contact metadata with explicit privacy controls, exposes name-bearing entries through directory peers, and reuses the existing shared target resolver with strict ambiguity and allowlist enforcement.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection confirms current main cannot return saved WhatsApp contact names because peer listing is allowFrom-backed and the WhatsApp socket does not subscribe to contacts.upsert.

## Solution Assessment

Unclear without a maintainer product decision. The requested direction is plausible, but the maintainable solution should be an opt-in WhatsApp plugin feature with privacy, retention, ambiguity, and authorization rules rather than a core resolver workaround.
