## Summary

Keep open for maintainer direction. Current main still has the source-confirmed WhatsApp directory gap, but persisting phone contacts is a new privacy-sensitive plugin capability rather than a broken existing contract or safe automation fix.

## What This Changes

Expose WhatsApp phone contacts from Baileys `contacts.upsert` through directory peers so name-based WhatsApp sends can resolve contacts.

## Best Possible Solution

Decide the product/privacy policy first; if accepted, implement an opt-in WhatsApp-plugin-owned contact store that feeds the existing directory adapter and leaves core target resolution generic.

## Reproduction Assessment

Yes, as a source-confirmed feature gap rather than a broken documented contract. Current main lists WhatsApp peers from `allowFrom`, has no `contacts.upsert` listener/store, and Baileys exposes the contact event the issue names.

## Solution Assessment

Unclear. Consuming Baileys contact events is technically plausible, but the best maintainable solution depends on maintainer decisions about opt-in storage, retention/deletion behavior, privacy wording, and whether this belongs in the official WhatsApp plugin at all.
