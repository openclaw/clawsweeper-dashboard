## Summary

Keep open for maintainer direction. Current main still has the source-confirmed WhatsApp contact-directory gap, but persisting a user's phone contacts and resolving names is a new privacy-sensitive WhatsApp plugin capability rather than a narrow bug fix or stale cleanup item.

## What This Changes

Request to persist WhatsApp contacts received from Baileys `contacts.upsert`, expose them through `openclaw directory peers`, and let outbound messaging resolve contacts by saved names.

## Best Possible Solution

Decide the product/privacy boundary first; if accepted, implement an opt-in WhatsApp-plugin-owned contact directory with retention/deletion docs and strict authorization/ambiguity handling.

## Reproduction Assessment

Yes, as a source-confirmed feature gap rather than a broken documented contract. Current main lists WhatsApp peers from `allowFrom`, has no `contacts.upsert` listener/store, and Baileys exposes the contact event the issue names.

## Solution Assessment

Unclear. Consuming Baileys contact events is technically plausible, but the best maintainable solution depends on opt-in storage, retention/deletion behavior, authorization semantics, and whether this belongs in the official WhatsApp plugin or a separate plugin path.
