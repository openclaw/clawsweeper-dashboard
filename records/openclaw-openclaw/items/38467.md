Keep open for maintainer direction. Current main still has the source-confirmed WhatsApp contact-directory gap, but storing a user's phone contacts and resolving names is a new privacy-sensitive WhatsApp plugin capability, not a narrow stale cleanup or bug-fix item.

Expose WhatsApp phone contacts from Baileys `contacts.upsert` through directory peer lookup and message-tool name resolution.

## Best Possible Solution

Decide the product/privacy boundary first; if accepted, implement an opt-in WhatsApp-plugin-owned contact directory with retention/deletion docs and strict authorization plus ambiguity handling.

Unclear. Consuming Baileys contact events is technically plausible, but the maintainable solution depends on opt-in storage, retention/deletion behavior, authorization semantics, and official WhatsApp plugin scope.
