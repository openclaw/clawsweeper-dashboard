## Summary

This should close as plugin/ClawHub scope: current main intentionally keeps WhatsApp peers config-backed, and a closely related opt-in WhatsApp contact/message-store PR was already closed unmerged on VISION/privacy grounds.

## What This Changes

The issue requests persisting WhatsApp contacts received from Baileys `contacts.upsert` and exposing them through directory peer lookup for name-based outbound messaging.

## Best Possible Solution

Keep bundled WhatsApp privacy-preserving by default; pursue contact-name resolution as an external ClawHub/plugin/channel experiment, with a narrow plugin API proposal only if current directory or target-resolution seams are insufficient.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection confirms current WhatsApp peer listing is config-backed and there is no bundled `contacts.upsert` persistence path.

## Solution Assessment
