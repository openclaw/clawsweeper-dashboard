## Summary

Keep this issue open: current main and the latest release still route bundled LiteLLM through the OpenAI-compatible transport, while `cacheRetention` resolution and Anthropic `cache_control` injection do not cover the reported default LiteLLM path.

## What This Changes

The issue requests that explicit `cacheRetention` for LiteLLM-proxied Anthropic models inject Anthropic prompt-cache `cache_control` markers instead of being ignored.

## Best Possible Solution

Add a LiteLLM-owned stream wrapper, preferably in the bundled LiteLLM plugin using the existing provider `wrapStreamFn` seam and shared payload helpers, that injects Anthropic cache markers only for explicit `short` or `long` retention on Anthropic-like LiteLLM model refs.

## Reproduction Assessment

Yes, at source level: bundled LiteLLM remains `openai-completions`, `resolveCacheRetention` excludes that API family, and the only Anthropic `cache_control` payload injector is OpenRouter-scoped. I did not run a live LiteLLM backend cache-hit check in this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is a LiteLLM/provider-owned payload wrapper that activates only when resolved retention is `short` or `long` for Anthropic-backed LiteLLM refs, while preserving absent config, `none`, and non-Anthropic LiteLLM models.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/37966 on current main. The bug is that explicit `agents.defaults.models["litellm/<claude-ref>"].params.cacheRetention` is ignored on the bundled LiteLLM `openai-completions` path, so Anthropic `cache_control` markers are never injected. Implement a narrow LiteLLM-owned provider stream wrapper, preferably from `extensions/litellm/index.ts` via `wrapStreamFn`, using shared `openclaw/plugin-sdk/provider-stream-shared` helpers where possible. Inject Anthropic ephemeral cache markers only for Anthropic-like LiteLLM refs when the effective retention is `short` or `long`; do not inject for absent config, `cacheRetention: "none"`, invalid legacy values, or non-Anthropic LiteLLM models. Preserve OpenRouter endpoint gating and custom `anthropic-messages` behavior; do not move LiteLLM-specific policy into generic core unless a small SDK helper is needed. Add focused regression coverage for positive injection, no-config, `none`, and non-Anthropic cases, plus any existing cache-retention tests that prove no drift. Add a single-line changelog fix entry if the PR is user-facing.
