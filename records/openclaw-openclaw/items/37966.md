## Summary

Keep open. Current main still does not implement the LiteLLM `openai-completions` cache-control path: related PRs proposed the right shape, but they were closed unmerged and the shipped code remains OpenRouter/custom-Anthropic scoped.

## What This Changes

The issue asks OpenClaw to honor explicit `cacheRetention` for LiteLLM-proxied Anthropic models by adding Anthropic `cache_control` markers to the outbound payload.

## Best Possible Solution

Add a provider-owned LiteLLM stream wrapper that injects Anthropic cache markers only for recognized Claude/LiteLLM Anthropic models when caching is explicitly enabled, with regression coverage for short/long, legacy TTL, absent config, and `none`.

## Reproduction Assessment

Yes, source-level reproduction is high confidence. LiteLLM resolves to `openai-completions`, the retention resolver does not pass this path, and the only `cache_control` payload wrapper is OpenRouter-gated, so the reported request cannot receive Anthropic cache markers on current main.

## Solution Assessment

Yes. The narrow maintainable fix is a LiteLLM-owned payload wrapper, not a generic all-proxy cache injection, and it must explicitly skip `cacheRetention: "none"`.
