## Summary

Keep open. Current main and the latest release still omit LiteLLM Claude from both the cache-retention family gate and OpenAI-completions Anthropic `cache_control` payload shaping, while https://github.com/openclaw/openclaw/pull/92665 is an open linked fix candidate rather than a merged fix.

## What This Changes

The issue reports that explicit `cacheRetention` is ignored for LiteLLM-proxied Anthropic models, preventing Anthropic prompt-cache `cache_control` markers from being sent through the OpenAI-compatible adapter.

## Best Possible Solution

Review, repair if needed, and land https://github.com/openclaw/openclaw/pull/92665 or an equivalent explicit-gated LiteLLM Anthropic `cache_control` fix, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes, at source level: current main drops LiteLLM OpenAI-completions models before retention reaches payload shaping, and Anthropic `cache_control` compatibility only targets OpenRouter Anthropic refs. I did not run a live LiteLLM-to-Anthropic proxy probe.

## Solution Assessment

Yes in direction, but not complete on main: the maintainable fix belongs in the provider cache-retention family and OpenAI-completions payload compatibility seams. The linked PR is the current candidate, so this issue should stay open until that path lands or is replaced.
