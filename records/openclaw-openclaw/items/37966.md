## Summary

Keep open: current main still leaves bundled LiteLLM on the OpenAI-compatible route where explicit `cacheRetention` does not resolve for Anthropic/Claude models and no LiteLLM payload wrapper injects Anthropic `cache_control` markers. The related LiteLLM fix attempts are closed unmerged, and the open related OpenRouter PR does not cover this LiteLLM path.

## What This Changes

The issue asks OpenClaw to honor explicit `cacheRetention` for LiteLLM-proxied Anthropic models by injecting Anthropic `cache_control` markers on the LiteLLM OpenAI-compatible payload.

## Best Possible Solution

Add a narrow LiteLLM Anthropic prompt-cache payload wrapper for explicit `short`/`long` cache settings, honor `none`, add focused regression tests/docs/changelog, and leave unrelated OpenAI-compatible proxies unchanged.

## Reproduction Assessment

Yes, by source inspection. Configure `agents.defaults.models["litellm/claude-opus-4-6"].params.cacheRetention`, and current main keeps bundled LiteLLM on `openai-completions`, resolves no Anthropic cache family for that route, and only injects Anthropic `cache_control` markers for OpenRouter.

## Solution Assessment

Yes. The narrow maintainable fix is a LiteLLM/Claude-gated payload marker path that reuses the existing marker policy, respects `cacheRetention` values, and avoids widening behavior for arbitrary OpenAI-compatible providers.
