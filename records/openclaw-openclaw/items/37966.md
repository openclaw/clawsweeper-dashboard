## Summary

Keep open. Current main and v2026.6.10 still drop explicit LiteLLM Claude cache retention before the Anthropic `cache_control` payload path, and the strongest linked fix candidate was closed unmerged rather than landing.

## What This Changes

The issue asks OpenClaw to honor explicit `cacheRetention` for LiteLLM-proxied Anthropic models by injecting Anthropic `cache_control` markers into OpenAI-completions payloads.

## Best Possible Solution

Implement a narrow fix in the current LLM provider seams: detect LiteLLM Claude models only for explicit `cacheRetention: "short"` or `"long"`, inject Anthropic `cache_control`, preserve no-config and `"none"` as no markers, and update tests/docs.

## Reproduction Assessment

Yes, at source level: current main drops LiteLLM OpenAI-completions Claude models before retention reaches payload shaping, and Anthropic `cache_control` compatibility only targets OpenRouter Anthropic refs. I did not run a live LiteLLM-to-Anthropic proxy probe.

## Solution Assessment

No fix is present on main. The closed unmerged PR used the right general seams after follow-up, but the best path now is a narrow replacement or reopened fix with the explicit-opt-in guard, regression tests, docs, and live-provider proof when available.
