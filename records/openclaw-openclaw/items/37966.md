## Summary

Keep open: current main still leaves bundled LiteLLM on the OpenAI-compatible route, where explicit `cacheRetention` does not resolve for LiteLLM/Claude model refs and no LiteLLM payload wrapper injects Anthropic `cache_control` markers. The related LiteLLM fix PRs are closed unmerged, while the open OpenRouter PR covers a different provider path.

## What This Changes

The issue requests that explicit `cacheRetention` on bundled LiteLLM Anthropic model refs inject Anthropic `cache_control` markers instead of being ignored on the OpenAI-compatible route.

## Best Possible Solution

Add a narrow LiteLLM Anthropic prompt-cache payload wrapper for explicit `short`/`long`, honor `none`, and update focused tests, docs, and changelog without widening arbitrary OpenAI-compatible proxies.

## Solution Assessment
