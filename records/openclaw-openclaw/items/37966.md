## Summary

Keep this issue open: current main and the latest release still leave bundled LiteLLM on the `openai-completions` path without a LiteLLM-specific Anthropic `cache_control` injector, so explicit `cacheRetention` remains ineffective for the reported setup.

## What This Changes

`cacheRetention` configured for LiteLLM-proxied Anthropic models should cause Anthropic prompt-cache `cache_control` markers to be added to the LiteLLM OpenAI-compatible request payload.

## Best Possible Solution

Add a LiteLLM-owned stream wrapper for explicit `short` or `long` retention on Claude-like LiteLLM model refs, leaving absent config and `none` unchanged and keeping generic proxy providers out of scope.

## Reproduction Assessment

Yes, at source level: bundled LiteLLM is `openai-completions`, current cache-retention resolution excludes that route, and the only Anthropic `cache_control` injector is OpenRouter-scoped. I did not run a live LiteLLM backend cache-hit check in this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is LiteLLM provider request shaping, not a global resolver change for every OpenAI-compatible provider. The repair should reuse the public provider stream helper surface and preserve `cacheRetention: "none"` as an explicit opt-out.

Current main ignores explicit `agents.defaults.models["litellm/claude-opus-4-6"].params.cacheRetention` because bundled LiteLLM is `openai-completions`, `resolveCacheRetention` only admits direct/custom `anthropic-messages` and Google families, and the Anthropic `cache_control` payload wrapper is OpenRouter-scoped. Implement a LiteLLM-owned wrapper through the provider/plugin stream surface: for `provider === "litellm"` and Claude-like LiteLLM model ids (`claude-*`, `anthropic/claude-*`, `anthropic.claude*`), inject Anthropic `cache_control: { type: "ephemeral" }` into eligible system/developer content blocks only when effective `cacheRetention` is `"short"` or `"long"`; preserve absent config and `cacheRetention: "none"` as no marker injection. Do not globally enable Anthropic markers for arbitrary OpenAI-compatible providers, do not duplicate OpenRouter long-TTL work, and do not change non-Anthropic Bedrock semantics. Add focused regression tests for enabled short/long, explicit none, absent config, non-Anthropic LiteLLM models, and string/array system content.
