## Summary

Keep this issue open: current main and the latest shipped release still route LiteLLM through the OpenAI-compatible transport without resolving LiteLLM Anthropic cache retention or injecting Anthropic `cache_control` payload markers. The linked fix attempts are closed and unmerged, while current source makes the remaining bug narrow and repairable.

## What This Changes

The issue requests honoring explicit `cacheRetention` for LiteLLM-proxied Anthropic models by adding payload-level Anthropic cache-control handling on the LiteLLM OpenAI-compatible path.

## Best Possible Solution

Add LiteLLM-owned prompt-cache payload handling, or a shared provider-stream helper used by LiteLLM, that applies Anthropic cache-control markers only for explicit short/long retention on Anthropic-backed LiteLLM model refs and preserves none/no-config as no injection.

## Reproduction Assessment

Yes, at source level: LiteLLM is still `openai-completions`, cache-retention resolution excludes that API family, and the only Anthropic `cache_control` injector is OpenRouter-scoped. I did not run a live LiteLLM backend cache-hit reproduction in this read-only review.

## Solution Assessment

Yes, the best fix shape is a narrow LiteLLM/provider-owned wrapper or shared helper reuse, not broadening OpenRouter-specific behavior to every proxy. The repair should include payload-level regression tests for short, long, none, absent config, and non-Anthropic LiteLLM models.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/37966 on current main. LiteLLM models are registered as `openai-completions`, so explicit `agents.defaults.models["litellm/<anthropic model>"].params.cacheRetention` currently neither resolves to a stream option nor injects Anthropic `cache_control` blocks. Add a LiteLLM-owned `wrapStreamFn` path, or a shared provider-stream helper reused by the LiteLLM plugin, that injects Anthropic cache-control payload markers only when explicit LiteLLM config requests caching (`cacheRetention: "short"`/`"long"` or legacy TTL equivalents) and the LiteLLM model ref is Anthropic/Claude-shaped. Do not inject for absent config, `cacheRetention: "none"`, or non-Anthropic LiteLLM models. Preserve OpenRouter endpoint gating and do not broaden OpenRouter-specific behavior to arbitrary proxies. Add focused regression tests that inspect the outgoing payload, update LiteLLM/prompt-caching docs if the supported behavior changes, and add a user-facing changelog entry unless maintainers decide the docs/tests-only scope does not need one.
