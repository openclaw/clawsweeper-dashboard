## Summary

This remains a valid, bounded provider bug. Current main still excludes LiteLLM’s OpenAI-compatible Claude route from both cache-retention resolution and Anthropic payload-marker injection, while the only linked fix closed unmerged and covered only part of the required behavior.

## What This Changes

Honor explicit cache retention for Anthropic models routed through LiteLLM’s OpenAI-compatible endpoint.

## Best Possible Solution

Add a LiteLLM-owned or canonical proxy capability that reliably identifies Claude routes, translates explicit retention into Anthropic cache_control blocks on the OpenAI-completions payload, strips the consumed transport option, and proves eligible and non-eligible models separately.
