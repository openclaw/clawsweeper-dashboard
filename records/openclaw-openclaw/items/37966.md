## Summary

Keep open. Current main and latest release still omit the LiteLLM OpenAI-completions Claude route from both cache-retention family resolution and Anthropic `cache_control` payload shaping, while https://github.com/openclaw/openclaw/pull/92665 is an open linked implementation candidate rather than a merged fix.

## What This Changes

The issue reports that explicit `cacheRetention` is ignored for LiteLLM-proxied Anthropic models, preventing Anthropic prompt-cache markers from being sent through the OpenAI-completions adapter.

## Best Possible Solution

Review, repair if needed, and land https://github.com/openclaw/openclaw/pull/92665 or an equivalent explicit-gated LiteLLM Anthropic cache-control fix, then close this issue after the fix reaches main.
