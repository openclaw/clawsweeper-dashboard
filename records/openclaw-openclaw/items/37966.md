## Summary

Keep this issue open: current main still does not cover LiteLLM Anthropic models on the OpenAI-completions path, and the linked open fixing PR is the active implementation candidate.

## What This Changes

The issue reports that explicit `cacheRetention` is ignored for LiteLLM-proxied Anthropic models, so Anthropic prompt-cache `cache_control` markers are not added to the outgoing payload.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/92665 or an equivalent narrow fix, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes, from source: current main only enables custom Anthropic cache retention for `anthropic-messages`, while LiteLLM uses the OpenAI-completions path and only OpenRouter gets automatic Anthropic cache-control formatting. I did not run a live LiteLLM/Anthropic request in this read-only review.

## Solution Assessment

Yes, the best fix shape is a narrow provider-routing change in the shared cache-family resolver plus OpenAI-completions payload-marker detection. Starting a separate repair lane would duplicate the already-open fixing PR.
