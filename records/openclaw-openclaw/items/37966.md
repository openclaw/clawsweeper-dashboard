## Summary

Keep this issue open. Current main still does not honor explicit `cacheRetention`/`cacheControlTtl` for the bundled LiteLLM OpenAI-compatible Claude route: LiteLLM is configured as `openai-completions`, the cache-retention resolver only accepts Anthropic-family custom providers on `anthropic-messages`, the payload `cache_control` wrapper is OpenRouter-only, and the linked fix PR is closed unmerged and did not add the required payload-level injection.

## What This Changes

The issue requests that explicit `cacheRetention` or `cacheControlTtl` for LiteLLM-proxied Anthropic models be honored by injecting Anthropic `cache_control` markers on the LiteLLM OpenAI-compatible request path.

## Best Possible Solution

Add a narrow LiteLLM Anthropic prompt-cache pass-through for the bundled `openai-completions` LiteLLM route: only when the resolved model looks Claude/Anthropic-backed and explicit `cacheRetention` or `cacheControlTtl` resolves to `short` or `long`, inject Anthropic `cache_control` markers into system/developer content blocks before the proxy request; document the behavior and credit the reporter/contributors in the changelog.
