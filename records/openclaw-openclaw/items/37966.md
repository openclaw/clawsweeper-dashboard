## Summary

Keep this open: current main still leaves the bundled LiteLLM OpenAI-compatible Claude route without either `cacheRetention` resolution or Anthropic `cache_control` payload markers, while the related fix PR is closed unmerged and did not cover the payload marker path.

## What This Changes

The issue asks OpenClaw to honor explicit `cacheRetention` for LiteLLM-proxied Anthropic models by adding Anthropic prompt-cache markers through the bundled LiteLLM OpenAI-compatible route.

## Best Possible Solution

Add a narrow bundled-LiteLLM Anthropic prompt-cache pass-through: honor explicit `cacheRetention` only for Claude/Anthropic-looking LiteLLM models and inject provider-compatible `cache_control` markers without changing arbitrary OpenAI-compatible proxy behavior.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists on current main: configure `litellm/claude-opus-4-6` with explicit `cacheRetention`, then the LiteLLM route stays `openai-completions`, retention resolution returns `undefined`, and the only marker wrapper remains OpenRouter-scoped.

## Solution Assessment

Yes. The narrow maintainable fix is a LiteLLM-specific Anthropic payload wrapper or equivalent resolver path gated by explicit cache config and Claude-like model identity; broadly applying Anthropic markers to all OpenAI-compatible providers would be riskier.
