## Summary

Keep open. Current main still supports the documented Gmail webhook setup command but continues to invoke bare `gcloud`, `gog`, and `tailscale` in the affected paths, while the focused fix is still an open linked PR and should merge before this issue closes.

## What This Changes

Fix native Windows Gmail webhook setup so installed `gcloud`, `gog`, and Tailscale PATH/PATHEXT shims resolve before spawn instead of crashing with `spawn gcloud ENOENT`.

## Best Possible Solution

Land or replace the focused Windows helper-resolution fix from #74881 so Gmail setup, manual run, and gateway watcher paths resolve `gcloud`, `gog`, and `tailscale` to explicit Windows executables while keeping argv-based execution and `shell: false`; then let the issue close after merge.

## Reproduction Assessment

Yes. The reporter provided a high-confidence native PowerShell reproduction, and current main statically preserves the same failure path by spawning bare `gcloud` before the setup wizard can proceed.

## Solution Assessment

Yes. Resolving helper binaries through PATH/PATHEXT before spawn, without broad shell mode, is the narrow maintainable fix; expanding the global process shim or enabling `shell: true` would be riskier than threading explicit resolved helper paths through the Gmail surfaces.
