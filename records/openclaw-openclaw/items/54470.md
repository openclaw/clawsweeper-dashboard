## Summary

Keep open. Current main still documents the Gmail webhook setup flow and still reaches bare `gcloud`, `tailscale`, and `gog` spawn sites on that path; open PR #74881 is the focused implementation candidate and should land or be replaced before this issue closes.

## What This Changes

Native Windows Gmail webhook setup should use available PATH/PATHEXT helper shims instead of exiting with `spawn gcloud ENOENT`.

## Best Possible Solution

Land or replace the focused Windows executable-resolution fix so Gmail setup, manual run, and gateway watcher paths resolve `gcloud`, `gog`, and `tailscale` before shell-free spawn, with regression coverage for the Windows shim case.

## Reproduction Assessment

Yes, but by source inspection rather than a live Windows run in this review. The reporter supplied concrete native PowerShell steps, and current main still has dependency detection that can pass through PATHEXT while the affected Gmail paths spawn bare helper names.

## Solution Assessment

Yes. Resolving these specific helper binaries to explicit executable paths before argv-based spawn is the narrow maintainable fix; enabling broad shell execution would be a riskier global behavior change.
