## Summary

Keep open: current main still supports the documented Gmail webhook setup command, but the active setup path continues to invoke bare `gcloud`, `gog`, and `tailscale` commands. The Windows command shim in the process layer is limited to package-manager commands, so it does not cover the reporter's native Windows PATH/PATHEXT case.

## What This Changes

Fix native Windows Gmail webhook setup so installed helper tools such as `gcloud.cmd`, `gog`, and `tailscale` are resolved and invoked correctly instead of crashing with `spawn gcloud ENOENT`.

## Best Possible Solution

Add a Windows-aware helper-tool resolution path for Gmail commands that resolves PATH/PATHEXT shims to explicit invocations before spawning, keeps arguments structured, and covers setup, run, and gateway watcher behavior with win32 regression tests.
