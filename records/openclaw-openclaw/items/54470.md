## Summary

Keep open. Current main still exposes the documented Gmail webhook setup flow and still reaches bare `gcloud`, `tailscale`, and `gog` spawn sites on the affected paths; the focused closing fix is the still-open #74881 and should land or be replaced before this issue closes.

## What This Changes

The issue requests that native Windows Gmail webhook setup resolve installed `gcloud`, `gog`, and `tailscale` helpers instead of crashing with `spawn gcloud ENOENT`.

## Best Possible Solution

Merge or replace the focused Windows executable-resolution fix so Gmail setup, manual run, and gateway watcher paths resolve `gcloud`, `gog`, and `tailscale` through PATHEXT while preserving argv-based shell-free execution.

## Reproduction Assessment

Yes. The reporter supplied a high-confidence native PowerShell reproduction, and current main is source-reproducible because dependency detection can pass through PATHEXT while the Gmail setup and watcher paths still spawn bare helper names.

## Solution Assessment

Yes. Resolving these specific helper binaries to explicit executable paths before spawn is the narrow maintainable fix; enabling broad shell execution or changing global spawn policy would be riskier.
