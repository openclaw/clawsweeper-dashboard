## Summary

Keep open: current main still routes denied exec allowlist misses through the generic tool failure logger, which includes a redacted raw_params preview, and the current default redaction pattern can miss JSON-escaped quoted env assignments for non-prefix-matched credentials.

## What This Changes

The issue requests preventing denied exec raw_params log lines from persisting inline environment credentials in gateway error logs.

## Best Possible Solution

Treat this as a gateway logging security fix: stop raw denied exec command values from reaching persistent logs, add regression coverage for JSON-escaped quoted env assignments and provider token prefixes, and document any operator cleanup guidance maintainers choose.

## Reproduction Assessment

Yes, source-level: denied exec allowlist misses throw through the generic tool failure logger, which includes raw_params, and the current redaction regex can miss JSON-escaped quoted env assignments. I did not run a live gateway repro because that would write logs during a read-only review.

## Solution Assessment

Unclear which exact mitigation maintainers want, but the direction is sound: structurally redact command fields before logging, or replace denied raw_params with a bounded preview/hash, with focused regression coverage for the reported command shape.
