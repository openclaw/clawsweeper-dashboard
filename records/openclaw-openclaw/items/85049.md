## Label Justifications

- `P1`: Cleartext provider credential exposure in persistent gateway logs can affect real operators and needs urgent security handling without making the core runtime unusable.
- `impact:security`: The report is about sensitive credentials being persisted to local logs after a denied tool call.
- `impact:auth-provider`: The exposed values are provider API keys and access tokens used for model/tool authentication.

## Summary

Keep open: current main and v2026.5.19 still route denied exec allowlist failures through generic tool failure logging with `raw_params`, and the default redaction can miss JSON-escaped quoted env assignments for non-`sk-` provider tokens.

## What This Changes

The issue asks OpenClaw to stop persisting cleartext credentials embedded in denied `tools/exec` `raw_params.command` log entries.

## Best Possible Solution

Treat this as a gateway logging security fix: prevent denied exec command values from reaching persistent logs in cleartext, and add regression coverage for JSON-escaped quoted env assignments plus provider token prefixes.

## Reproduction Assessment

Yes, source-level: denied exec throws through the generic tool failure logger, which includes `raw_params`, and a focused probe shows the current redaction pattern misses escaped quoted env assignments for several token shapes. I did not run a live gateway repro because this review was read-only and live repro would write logs.

## Solution Assessment

Unclear which exact mitigation maintainers want, but the direction is sound: redact command fields before logging, log only a bounded hash/preview, or suppress raw_params on denial, with focused regression tests for the reported command shape.
