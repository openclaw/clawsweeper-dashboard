## Label Justifications

- `P1`: Cleartext provider credential exposure in persistent gateway logs can affect real operators and needs urgent security handling.
- `impact:security`: The issue is about sensitive credentials being written to local logs after a denied tool call.
- `impact:auth-provider`: The exposed values are provider API keys and access tokens used for model or service authentication.

## Summary

Keep open: current main still logs denied tool failures with a `raw_params` preview, and the current default redaction can miss JSON-escaped quoted env-var assignments for non-prefixed provider secrets.

## What This Changes

The issue asks OpenClaw to stop persisting cleartext credentials from denied `tools/exec` raw parameters into gateway error logs.

## Best Possible Solution

Treat this as a gateway logging security fix: prevent denied exec command values from reaching persistent logs in cleartext, and add regression coverage for JSON-escaped quoted env assignments plus provider token prefixes.

## Reproduction Assessment

Yes, source-level: denied exec allowlist misses flow into the generic tool failure logger, which includes `raw_params`, and the current redaction pattern misses escaped quoted env assignments for several token shapes. I did not run a live gateway repro because that would write logs during a read-only review.

## Solution Assessment

Unclear which exact mitigation maintainers want, but the requested direction is sound: redact command fields before logging, log only a bounded hash/preview, or suppress raw_params on denial with focused regression tests.
