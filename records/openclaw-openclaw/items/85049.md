## Summary

Keep open: current main and the latest release still route denied exec allowlist failures through generic raw_params failure logging, while the default env-assignment redaction can miss JSON-escaped quoted credentials for non-sk token shapes.

## What This Changes

Prevent denied exec tool failure logging from persisting raw_params.command values that contain inline env-var credentials.

## Best Possible Solution

Treat this as a gateway logging security fix: prevent denied exec raw command values from reaching persistent logs, add regression coverage for escaped quoted env assignments and provider token prefixes, and document operator cleanup for already-written logs.

## Reproduction Assessment

Yes, source-level: denied allowlist misses throw into the generic tool failure logger, which serializes raw_params, and the current default redaction pattern misses escaped quoted env assignments for several non-sk token shapes. I did not run a live gateway repro because it would write logs in a read-only review.

## Solution Assessment

Unclear which exact mitigation maintainers want, but the requested direction is sound: redact command fields structurally, log only bounded previews/hashes, or suppress raw_params on denial with focused regression coverage.
