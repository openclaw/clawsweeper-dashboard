## Summary

Current main now contains the missing xAI OAuth renewal path: the bundled provider registers a refresh callback that exchanges the stored refresh token at xAI’s discovered token endpoint and returns updated credentials through the shared auth-profile machinery. This implementation landed after the latest v2026.6.5 release, so the issue is resolved on main but not yet shipped; the prior Cloudflare claim should only be reopened with a redacted post-fix live failure.

## What This Changes

Ensure xAI OAuth credentials automatically renew through the stored refresh token instead of requiring browser reauthentication after access-token expiry.

## Best Possible Solution

Keep the provider-owned refresh callback on main and ask for a new issue with redacted post-fix request/response evidence only if xAI renewal still fails after a release containing b66f0b6c.

## Reproduction Assessment

Yes. Source history shows that the reported renewal gap is addressed by a newly registered xAI refresh callback, with focused tests covering the exact grant and credential update path; no post-fix live failure is currently reported.

## Solution Assessment

Yes. Keeping refresh vendor-owned in the xAI plugin while using the shared auth-profile refresh machinery is narrower and more maintainable than adding a new endpoint, config option, or parallel credential path.
