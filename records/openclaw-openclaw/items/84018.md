## Summary

Keep this issue open: current main correctly attempts xAI refresh-token grants, but a live request to xAI’s discovery-advertised token endpoint still receives a Cloudflare 403 before normal OAuth error handling, reproducing the reported provider-auth blocker.

## What This Changes

Resolve xAI OAuth renewal failures that force browser reauthentication whenever the roughly six-hour access token expires.

## Best Possible Solution

Keep refresh ownership in the xAI plugin, obtain a redacted live trace using a valid expired credential, and then either implement an xAI-approved token-request adjustment or clearly diagnose and document the provider-side limitation while retaining API-key authentication as the supported unattended path.

## Reproduction Assessment

Yes. Current main sends refresh grants to xAI’s discovery-advertised endpoint, and a live automated POST to that endpoint returned the reported Cloudflare 403 block page before an OAuth response; a valid-token trace is still needed to measure the exact account scope.

## Solution Assessment

No definitive repair is proven yet. The existing discovery-based provider hook is the correct implementation layer, but an alternate endpoint or browser-like workaround should not be adopted without xAI contract evidence and live valid-token verification.
