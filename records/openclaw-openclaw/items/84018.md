## Summary

## What This Changes

The issue reports that xAI OAuth access tokens expire after about six hours and automated refresh fails against the xAI token endpoint with a Cloudflare challenge.

## Best Possible Solution

Verify refresh with a live expired eligible xAI OAuth profile; if xAI supports a non-challenged refresh path, keep the fix in `extensions/xai`, otherwise document the limitation and surface clear xAI-specific reauth/API-key guidance.

## Reproduction Assessment

No high-confidence live reproduction yet. Source inspection confirms the expired-token refresh path and upstream discovery confirms a refresh endpoint, but reproducing the reported CAPTCHA requires a real expired SuperGrok/xAI OAuth profile.

## Solution Assessment

Unclear. An alternate endpoint should not be implemented without upstream or live contract proof; if refresh is externally blocked, the maintainable fix is provider-specific warning/docs plus reauth or API-key guidance.
