## Summary

Keep open: current main and v2026.6.5 already attempt xAI OAuth refresh through the provider hook, but the central user problem is a live xAI/Cloudflare block on that refresh endpoint, recently confirmed by multiple users and not resolved or documented on current main.

## What This Changes

The issue requests reliable automated renewal for xAI OAuth tokens when access tokens expire after roughly six hours and refresh attempts hit a Cloudflare challenge.

## Best Possible Solution

Verify a live expired xAI OAuth profile on current main or v2026.6.5; if xAI supports automated refresh, fix the provider-owned xAI refresh path, and otherwise document/warn about the upstream limitation with clear reauth/API-key guidance.

## Reproduction Assessment

No high-confidence live reproduction is available. Source and latest-release inspection show the expired OAuth path calls xAI's refresh endpoint, and multiple users report the six-hour failure, but the Cloudflare block itself needs a real expired xAI OAuth profile to verify.

## Solution Assessment

Unclear. An alternate refresh endpoint is not the best fix without xAI discovery or live contract proof; the safer path is live verification followed by either a provider-owned refresh fix or explicit warning/docs for an upstream limitation.
