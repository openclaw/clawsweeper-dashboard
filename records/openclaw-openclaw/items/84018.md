## Summary

Keep open: current main and the latest stable release already contain xAI OAuth refresh plumbing, but this report is about a live Cloudflare challenge on the provider token exchange path and there is not enough live xAI/SuperGrok evidence to treat it as fixed, duplicate, or external-only.

## What This Changes

The issue asks OpenClaw to handle or clearly document xAI OAuth refresh-token failures when Cloudflare blocks automated renewal after short-lived xAI access tokens expire.

## Best Possible Solution

Keep the fix provider-owned: verify an expired eligible xAI OAuth profile in a live run, then either update the xAI token endpoint/request contract with regression coverage or document and warn that xAI OAuth requires periodic reauthentication or API-key fallback when xAI blocks refresh.

## Reproduction Assessment

No high-confidence live reproduction yet. Source confirms expired xAI OAuth credentials flow into the provider refresh hook, but the Cloudflare CAPTCHA failure needs an expired live xAI/SuperGrok OAuth profile and redacted refresh logs to verify.

## Solution Assessment

Unclear. An alternate endpoint or request workaround should not be implemented without upstream contract or live proof; if xAI blocks automated refresh, the maintainable solution is xAI-specific docs/warnings plus reauthentication or API-key guidance.
