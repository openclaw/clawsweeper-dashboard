- `impact:auth-provider`: The issue concerns xAI OAuth credential expiry and provider token renewal.

## Summary

Current main and release `v2026.6.5` implement the missing automatic xAI OAuth renewal path: expired credentials are routed through the xAI plugin, which submits the stored refresh token to the trusted discovered token endpoint and returns updated credentials. The older duplicate reports describe pre-fix versions, while any continuing Cloudflare failure on `v2026.6.5` or newer would be a distinct post-fix live-provider report.

## What This Changes

Restore reliable automatic xAI OAuth renewal so expiring access tokens do not require browser reauthentication every six hours.

## Best Possible Solution

Keep the shipped discovery-backed provider refresh path; if `v2026.6.5` or newer still receives a Cloudflare challenge, track that separately with a redacted live refresh response rather than reopening the pre-implementation renewal gap.

## Reproduction Assessment

No for current main: the previously absent renewal route now has a registered xAI provider refresh hook plus focused source-level regression coverage. A remaining xAI or Cloudflare failure needs a fresh live trace from `v2026.6.5` or newer.

## Solution Assessment

Yes. Refreshing through the owning provider plugin with the trusted endpoint retained from OAuth discovery is narrower and safer than hard-coding an alternate endpoint, adding configuration, or introducing a core fallback.
