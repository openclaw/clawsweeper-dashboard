## Summary

Keep open: current main and the latest release already have xAI OAuth refresh plumbing, but the reported failure is a live Cloudflare/CAPTCHA block on that exact provider token path and is not documented, superseded, or safe to dismiss as external-only.

## What This Changes

The issue asks OpenClaw to handle xAI OAuth access-token expiry when the saved refresh token cannot be renewed automatically because the xAI token endpoint returns a Cloudflare challenge.

## Best Possible Solution

Keep the fix provider-owned: verify refresh with a live eligible xAI account, use a documented non-challenged token endpoint only if xAI supports one, and otherwise add xAI-specific diagnostics or docs that explain manual reauth and API-key fallback.

## Reproduction Assessment

No, not yet with high confidence. Source confirms expired xAI OAuth credentials should call the provider refresh hook against the saved or discovered token endpoint, but the Cloudflare CAPTCHA response needs a live expired SuperGrok/xAI OAuth profile to verify.

## Solution Assessment

Unclear. Implementing a different endpoint is not the best fix without xAI contract or live proof; if automated refresh is externally blocked, the maintainable answer is clear xAI-specific diagnostics and documentation plus supported reauth/API-key guidance.
