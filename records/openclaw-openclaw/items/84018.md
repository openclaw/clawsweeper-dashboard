## Summary

Keep open: current main and v2026.5.18 ship an xAI OAuth refresh hook, but the reported Cloudflare block is on that exact live token endpoint and still needs provider/account validation before OpenClaw can choose code, docs, or warning behavior.

## What This Changes

The issue reports that xAI OAuth access tokens expire after about six hours and automated refresh against the xAI token endpoint returns a Cloudflare challenge, forcing manual re-login.

## Best Possible Solution

Verify refresh with a live eligible xAI/SuperGrok account; if xAI supports a non-challenged refresh path, update the xAI plugin and focused tests, otherwise document and surface provider-specific reauth or API-key guidance.

## Reproduction Assessment

No high-confidence live reproduction yet. Source inspection confirms the expired-profile path calls the xAI provider refresh hook against the saved or discovered token endpoint, but the Cloudflare CAPTCHA failure needs a live expired SuperGrok OAuth profile to verify.

## Solution Assessment

Unclear. An alternate endpoint or request shape should not be implemented without upstream or live contract proof; if refresh is externally blocked, provider-owned warning/docs plus API-key or reauth guidance is safer.
