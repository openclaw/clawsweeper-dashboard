Keep open: current main sends xAI OAuth refreshes to the discovered xAI token endpoint, and a direct token-endpoint POST currently returns a Cloudflare block before OAuth error handling. This still needs maintainer/provider judgment rather than cleanup closure.

## What This Changes

The issue reports that xAI OAuth refresh-token renewal is blocked by Cloudflare, forcing manual browser reauthentication roughly every six hours.

## Best Possible Solution

Keep the behavior provider-owned: verify with a live eligible xAI account, then either implement a contract-backed refresh path with tests or document/warn that xAI OAuth may require manual reauth and API-key/device-code recovery.

## Reproduction Assessment

No full app-level reproduction with a real expired xAI OAuth profile is available. Source plus upstream proof is high confidence: current main posts refresh requests to xAI's discovered token endpoint, and that endpoint currently returns a Cloudflare block for the same form POST shape.

## Solution Assessment

Unclear. The proposed alternate endpoint should not be implemented without xAI contract or live proof; if xAI intentionally blocks automated refresh, the maintainable solution is provider-specific warning/docs and reauth or API-key guidance.
