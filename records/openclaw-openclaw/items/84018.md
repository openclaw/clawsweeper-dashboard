## Summary

The issue is still necessary: current main and v2026.6.5 already attempt standards-based xAI refresh, but recent cross-platform reports indicate that the official token endpoint can still block unattended renewal, and no merged targeted fix or verified supported workaround was found.

## What This Changes

Investigate xAI OAuth refresh failures that force browser reauthentication after access-token expiry.

## Best Possible Solution

Run a redacted, credentialed xAI refresh trace against current main, compare the request and response with xAI's advertised contract, then implement the narrow provider-owned compatibility or diagnostic change supported by that evidence and document a stable credential alternative if unattended subscription refresh is not supported.

## Reproduction Assessment

Yes at source level: an expired xAI profile invokes the registered refresh callback, which sends the advertised refresh-token request, and several recent users report failure after six to eight hours. A real credentialed current-main run is still required before calling the endpoint failure fully reproduced.

## Solution Assessment

No, trying an unspecified alternate API endpoint is not presently the best-supported fix because xAI discovery explicitly names the endpoint OpenClaw already uses. Live contract diagnostics should determine whether the maintainable outcome is a request-compatibility fix, clearer failure handling, or documentation of a supported credential alternative.
