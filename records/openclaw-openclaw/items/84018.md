## Summary

Keep open: xAI OAuth refresh is a shipped provider-auth path, and current main plus v2026.5.20 still refresh by posting the stored refresh token to xAI's token endpoint; the reported Cloudflare challenge on that path is not documented, superseded, or safely dismissible without live provider verification.

## What This Changes

The issue asks OpenClaw to make xAI OAuth token renewal work automatically, or clearly warn/document the Cloudflare-blocked refresh limitation for affected SuperGrok users.

## Best Possible Solution

Keep the fix provider-owned: live-test expired xAI OAuth refresh with an eligible account, then either update the xAI refresh endpoint/request with focused tests or document the upstream limitation with clear reauth/API-key guidance.

## Reproduction Assessment

No high-confidence live reproduction yet. Source confirms expired xAI OAuth credentials dispatch to the provider refresh hook and post to the saved/discovered xAI token endpoint, but the Cloudflare CAPTCHA failure requires a live expired xAI OAuth profile to verify.

## Solution Assessment

Unclear. An alternate token endpoint should not be implemented without upstream or live contract proof; if xAI blocks automated refresh, the maintainable solution is provider-specific warning/docs plus reauth or API-key guidance.
