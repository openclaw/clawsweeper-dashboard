## Summary

Keep open: current main still has a real xAI OAuth refresh path, but the reported Cloudflare failure is not high-confidence fixed or reproducible without an expired live xAI profile. Source and live endpoint checks suggest the correct discovery endpoint is `https://auth.x.ai/oauth2/token`, while the reported `/oauth/token` path returns Cloudflare HTML, so this needs maintainer/provider follow-up rather than cleanup closure.

## What This Changes

The issue requests reliable automated xAI OAuth refresh, or clear warning/docs, when SuperGrok OAuth access tokens expire and refresh attempts hit a Cloudflare challenge.

## Best Possible Solution

Verify an expired xAI OAuth profile with redacted live logs; if the profile is using stale `/oauth/token`, add an xAI-owned fallback to fresh discovery `/oauth2/token` with tests, otherwise document the upstream limitation and reauth/API-key guidance.

## Reproduction Assessment

No high-confidence live reproduction yet. Source confirms expired xAI OAuth credentials call the provider refresh hook, and endpoint probing distinguishes `/oauth2/token` from the Cloudflare-blocked `/oauth/token` path, but a real expired SuperGrok OAuth profile is still needed.

## Solution Assessment

Unclear. A stale-endpoint fallback may be a narrow code fix, but choosing that over docs or provider escalation depends on live proof of which endpoint the stored profile is actually using.
