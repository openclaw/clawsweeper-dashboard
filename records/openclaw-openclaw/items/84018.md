## Summary

Keep open: current main still has a provider-owned xAI refresh path, but the report points at a live upstream/token-endpoint failure that is not fully resolved by the recently merged device-code login work and needs maintainer/provider validation before a safe fix can be queued.

## What This Changes

The issue reports that xAI OAuth refresh tokens cannot automatically renew access tokens because the token endpoint returns a Cloudflare block, forcing browser reauthentication about every six hours.

## Best Possible Solution

Verify the refresh path with a real expired xAI OAuth profile; if the failure is stale tokenEndpoint metadata, canonicalize or rediscover the xAI token endpoint with regression coverage, and if xAI blocks valid refreshes, document/warn and guide users to API-key or manual reauth/device-code login.

## Reproduction Assessment

No high-confidence end-to-end reproduction yet because this needs a real expired xAI/SuperGrok OAuth profile. Source plus live endpoint probes show a plausible current-main failure path: saved /oauth/token endpoints are reused and that path returns Cloudflare HTML, while discovered /oauth2/token returns OAuth JSON.

## Solution Assessment

Unclear. Canonicalizing a stale stored endpoint would be a narrow code fix if live proof confirms it, but an alternate API endpoint or broader workaround should not be implemented without xAI contract evidence.
