## Summary

Keep this canonical issue open: current main and v2026.6.5 automatically attempt xAI token refresh, but no merged change addresses the reported Cloudflare challenge, and independent reports confirm that sustained xAI use still requires reauthentication. The closed reports https://github.com/openclaw/openclaw/issues/86520 and https://github.com/openclaw/openclaw/issues/86583 correctly consolidate the same symptom here.

## What This Changes

Resolve xAI OAuth renewal failures that force browser reauthentication after access tokens expire roughly every six hours.

## Best Possible Solution

Keep refresh provider-owned at the discovered official endpoint, capture a redacted current-main refresh failure with response status, content type, headers, and bounded body, then implement only an officially supported request-compatibility repair or a precise reauthentication diagnostic for the affected xAI account class.

## Reproduction Assessment

Yes at source level: expired xAI credentials deterministically route through the provider refresh hook, and several users report the same expiry-driven failure. A sanitized current-main live capture of the Cloudflare response is still required for high-confidence endpoint-level reproduction.

## Solution Assessment

No, the proposed undocumented API-endpoint substitution is not currently the best-supported fix. The narrow maintainable approach is to verify the official token endpoint response and repair or diagnose the existing xAI plugin refresh boundary.
