## Summary

Keep open: current main and v2026.5.18 ship xAI OAuth with `offline_access` and a provider refresh hook, but this issue reports a live Cloudflare block on that exact refresh path that is not documented, superseded, or safe to dismiss as external-only.

## What This Changes

Report that xAI OAuth refresh-token renewal is blocked by a Cloudflare challenge at the xAI token endpoint, forcing manual browser re-login about every six hours.

## Best Possible Solution

Keep the fix provider-owned: verify refresh with a live eligible xAI account; if xAI supports a non-challenged token endpoint, update `extensions/xai/xai-oauth.ts` with regression coverage, otherwise document the limitation and surface xAI-specific reauth/API-key guidance.

## Reproduction Assessment

No high-confidence live reproduction yet. Source confirms expired xAI OAuth credentials call the provider refresh hook against the saved/discovered xAI token endpoint, but the Cloudflare CAPTCHA failure needs a live expired SuperGrok OAuth profile to verify.

## Solution Assessment

Unclear. An alternate endpoint should not be implemented without upstream or live contract proof; if refresh is externally blocked, the maintainable fallback is provider-specific warning/docs plus API-key reauth guidance.
