Keep open: the docs fix is useful, but the new correlation sentence is too broad because `subagent_ended` also fires for ACP/session reset and delete cleanup paths that do not necessarily have a corresponding `subagent_spawned` event.

## Best Possible Solution

Keep the docs-only field clarification, but qualify the `targetSessionKey` correlation to spawned child sessions before merge; no runtime or plugin API change is needed.

## Reproduction Assessment

Yes, source-level. Current main and v2026.6.9 omit the `subagent_ended` payload docs, and source/tests show the PR wording needs one qualifier for non-spawned cleanup events.

## Solution Assessment

No, not quite as written. A docs-only clarification is the right layer, but the correlation sentence should say it matches `subagent_spawned.childSessionKey` for spawned child sessions or when a spawned event exists.
