## Best Possible Solution

Scope replace-mode filtering to the default authenticated catalog append, preserve explicit `--all` and provider-filtered browse paths, then land one canonical fix with regression coverage and real CLI proof.

## Reproduction Assessment

Yes, at source level: current main routes default `models list` through `appendConfiguredModelRowSources`, which then calls `appendAuthenticatedCatalogRows` without checking `models.mode`. The issue body also provides concrete replace-mode leaked-provider output, but I did not run a live CLI repro in this read-only review.

## Solution Assessment

No, not as-is. The default-list fix direction is right, but disabling the source-plan cascade also removes documented explicit `--all` and provider-filtered catalog browsing; a default-list-only gate is the safer repair.
