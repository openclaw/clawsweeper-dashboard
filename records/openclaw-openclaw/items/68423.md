## Summary

Keep this open. Current main still trusts the legacy `.bak` restore path after only checking that the backup fingerprint has `gatewayMode`, and the linked implementation candidate remains open and unmerged.

## What This Changes

The issue asks OpenClaw to reject redacted-secret placeholder pollution in config backups before restoring or propagating `.bak` files, with optional validation CLI and retention hardening follow-ups.

## Best Possible Solution

Review and land #68462, or replace it with a main-ready equivalent that validates parsed backup contents before both async and sync `.bak` restore copies. Treat backup creation validation and CLI backup scanning as explicit follow-ups unless maintainers want to widen the same patch.

## Reproduction Assessment

Yes. A source-level reproduction is clear: seed a valid-looking `.bak` with `gateway.mode` and a secret field such as `gateway.auth.token: "***"`, make the active config suspicious, then call either recovery function; current main copies the backup because it checks `gatewayMode` but not redacted placeholders.

## Solution Assessment

Yes for the immediate path. Refusing polluted `.bak` restore in `src/config/io.observe-recovery.ts` is the narrowest maintainable first fix; the backup-rotation and CLI validation ideas are useful hardening but do not need to block that guard.

## Security Review

Status: needs_attention

Summary: This non-PR issue is security-sensitive because config recovery can preserve redacted secret placeholders and cause credential loss or repeated auth failures.

Concerns:

- **[medium] Polluted backup can restore redacted secret placeholders:** `src/config/io.observe-recovery.ts:641`
  - body: The `.bak` restore paths can copy a backup containing placeholder values such as `***` over the live config, preserving credential pollution instead of requiring clean manual recovery.
  - confidence: 0.89
- **[medium] Backup rotation can propagate a polluted source config:** `src/config/backup-rotation.ts:120`
  - body: `maintainConfigBackups` copies the active config into `.bak` without checking whether secret-shaped fields already contain redacted placeholders, so a polluted live file can spread into the backup ring.
  - confidence: 0.82
