## Summary

The central user problem is already tracked in the older, more specific https://github.com/openclaw/openclaw/issues/90944: a yielded parent generates a completion reply that is recorded but not delivered while the raw child completion is delivered instead. This issue adds a proposed configuration shape, but the canonical report already captures the observable failure and required single-delivery product decision.

## What This Changes

Request configurable parent wake-turn delivery so a parent can publish a curated response after subagent completion.

## Best Possible Solution

Keep the delivery-semantics discussion in https://github.com/openclaw/openclaw/issues/90944 and implement one mutually exclusive completion-output owner, preferring a parent-curated public reply over a separate boolean that can double-post.

## Reproduction Assessment

Yes at source level: current main always disables delivery for the descendant wake turn, and the linked canonical report supplies the same sessions_yield transcript-level failure path.

## Solution Assessment

No. The proposed independent boolean can overlap the existing direct child announcement; the canonical issue should define a single parent-curated versus direct-child delivery policy.
