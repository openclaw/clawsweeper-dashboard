## Summary

Keep open. Current main still lacks a Slack account-level native command naming surface, the two implementation attempts are closed unmerged, and the remaining work is a public config/API choice rather than a cleanup-closeable duplicate.

## What This Changes

The issue requests per-account Slack native slash-command prefixes or names so multiple Slack apps can expose independently targetable OpenClaw commands in one workspace.

## Best Possible Solution

Expose a validated Slack account-level native command naming contract and thread it through registration, dispatch, interactive arg-menu payloads, docs, and tests while preserving the existing /agentstatus override.

## Reproduction Assessment

Yes for the currently unsupported path: current source lacks any account-level Slack native name or prefix field, and native registration/lookup remain provider-scoped. I did not perform a live multi-app Slack workspace test in this read-only review.

## Solution Assessment

Unclear until maintainers choose the public config shape. Account-level native naming is the likely boundary, but the closed PR history shows nativePrefix versus nativeNames and validation semantics remain a product/API decision.
