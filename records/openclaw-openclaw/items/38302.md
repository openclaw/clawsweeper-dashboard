## Summary

Keep open: current main and v2026.6.10 still lack account-scoped Slack native command names, the two linked implementation PRs are closed unmerged, and the remaining choice is a maintainer product/config contract decision rather than a safe cleanup close.

## What This Changes

Request account-level Slack native slash-command naming so multiple Slack apps or agents can register distinct native commands without collisions.

## Best Possible Solution

Choose the supported Slack account-scoped native command contract, then implement config schema, native spec generation, lookup, arg-menu roundtrip, docs, and regression tests together.
