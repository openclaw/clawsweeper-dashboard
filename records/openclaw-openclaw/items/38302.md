## Summary

Keep open: current main and the latest release still lack account-scoped Slack native command names, while the two direct implementation PRs were closed unmerged. The remaining work is valid but needs a maintainer product decision on prefix versus explicit mapping before code should be queued.

## What This Changes

Request account-level Slack native slash command naming so multi-agent Slack apps can register distinct commands and avoid Slack command-name collisions.

## Best Possible Solution

Choose the supported Slack account-scoped native command contract, then implement config schema, native spec generation, lookup, arg-menu roundtrip, docs, and regression tests together.
