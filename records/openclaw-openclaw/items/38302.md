## Summary

Keep open. Current main still lacks an account-level Slack native command naming surface: Slack slash-command config has no native prefix or custom native-name field, the strict schema rejects unknown slashCommand keys, and native command generation/lookup plus Slack dispatch are still provider-scoped. The exact prior implementation PRs #38326 and #38391 are closed unmerged, so there is no merged fix for this issue.

## What This Changes

Request account-level Slack native slash-command naming so each Slack account/app can register distinct commands for multi-agent setups.

## Best Possible Solution

Add a validated account-level Slack native naming option in the Slack plugin and thread it through config/schema/metadata, native spec generation, registration, dispatch lookup, argument-menu callbacks, docs, and focused tests while preserving current defaults and the /status to /agentstatus override.

## Reproduction Assessment

Yes. Source inspection is enough: the proposed account field is absent and rejected by the strict schema, and current lookup cannot resolve a prefixed native name such as oz-reset back to the reset command.

## Solution Assessment

Yes on the technical boundary, but the exact public config shape needs maintainer choice. A Slack account-level native-name seam is narrower and more maintainable than a manifest-only workaround because OpenClaw must map the registered Slack command back to an internal command key.
