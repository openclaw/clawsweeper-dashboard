## Summary

Keep open. Current main still lacks an account-level Slack native command naming surface, both implementation attempts are closed unmerged, and the related duplicate points back to this issue as the canonical remaining feature/API decision.

## What This Changes

Request account-level Slack native slash-command naming or prefixes so multiple Slack apps for different agents can avoid command-name collisions.

## Best Possible Solution

Expose a validated Slack account-level native command naming contract and thread it through registration, dispatch, interactive arg-menu payloads, docs, and tests while preserving the existing /agentstatus override.

## Reproduction Assessment

Yes for the currently unsupported path: current source lacks any account-level Slack native name or prefix field, and native registration/lookup remain provider-scoped. I did not run a live multi-app Slack workspace test in this read-only review.

## Solution Assessment

Unclear until maintainers choose the public config shape. Account-level native naming is the likely boundary, but prefix versus explicit nativeNames mapping, validation, and collision semantics remain product/API decisions.
