## Summary

Keep open. Current main still lacks an account-level Slack native command naming surface: config only exposes enabled/name/sessionPrefix/ephemeral, native command generation and lookup are provider-scoped, and Slack registration plus arg-menu dispatch still resolve unprefixed provider names. The prior implementation attempts #38326 and #38391 are closed unmerged, while #74343 adds related Slack multi-app evidence rather than a merged fix.

## What This Changes

The issue requests per-account Slack native slash-command prefixes so multiple Slack apps can register distinct commands for separate agents.

## Best Possible Solution

Add a validated account-level Slack native naming option in the bundled Slack plugin, preserve default unprefixed behavior and the existing /status to /agentstatus override, and thread the mapping through config/schema/metadata, native spec generation, Slack registration, dispatch lookup, arg-menu callbacks, docs, and focused tests.

## Reproduction Assessment

Yes. Source inspection is enough to reproduce the gap: the proposed account field is absent and rejected by the strict schema, and current lookup cannot resolve a prefixed native name such as oz-reset back to the reset command.

## Solution Assessment

Yes. The narrow maintainable path is an account-level Slack native-name seam shared by registration, dispatch, and argument-menu callbacks; a Slack-manifest-only change would leave OpenClaw unable to map prefixed commands back to internal command keys.
