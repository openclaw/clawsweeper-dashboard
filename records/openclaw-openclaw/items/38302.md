## Summary

Keep open. Current main and v2026.6.5 still lack account-scoped Slack native command names, both linked implementation PRs closed unmerged, and a later Slack multi-app cross-routing report was already closed as a duplicate of this canonical item.

## What This Changes

The issue requests account-scoped Slack native slash command naming so multiple Slack apps can register and target OpenClaw multi-agent commands without Slack command-name collisions.

## Best Possible Solution

Keep this as the canonical Slack multi-app native-command item and implement a maintainer-approved account-scoped naming contract with validation, collision handling, docs, and tests, or explicitly document duplicate Slack native names as unsupported.

## Reproduction Assessment

Yes at source level: current main still has provider-scoped native command registration and lookup, and Slack account config has no account-scoped native naming field. I did not run a live Slack workspace reproduction in this read-only review.

## Solution Assessment

Unclear. Account-specific native names would address the Slack platform collision, but maintainers still need to choose the canonical prefix versus nativeNames contract plus validation and collision semantics.
