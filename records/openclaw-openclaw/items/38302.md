## Summary

Keep open. Current main still lacks an account-level Slack native command naming surface: the Slack account config schema only supports the existing slash-command fields, and native command generation plus lookup remain provider-scoped rather than account-scoped.

## What This Changes

Add account-level Slack slash-command native naming so each Slack account can register collision-free native commands and map them back to built-in OpenClaw commands.

## Best Possible Solution

Add a validated account-level Slack native-name configuration in the Slack plugin, thread it through schema/metadata, native spec generation, registration, dispatch lookup, argument-menu callbacks, docs, and tests, while preserving the existing /status to /agentstatus override.

## Reproduction Assessment

Yes for the current limitation. Source inspection shows the proposed account field is absent and rejected by the strict schema, and the current Slack lookup path cannot map a prefixed native name such as oz-reset back to reset.

## Solution Assessment

Unclear until a maintainer chooses the public config shape. An account-level Slack native-name mapping appears more flexible than a blanket prefix, but the chosen surface must include validation, collision handling, and callback round-trip behavior.
