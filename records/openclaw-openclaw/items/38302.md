## Summary

Keep open. Current main still lacks a Slack account-level native command naming surface, and the issue describes a valid multi-account Slack limitation not covered by the closed duplicate or the separate Discord visibility request.

## What This Changes

The issue requests a Slack account-level `slashCommand.nativePrefix` option so each Slack app in a multi-agent deployment can register distinct native slash commands such as `/oz-reset`.

## Best Possible Solution

Add a validated Slack account-level native command naming contract, thread it through config metadata, native spec generation, registration, dispatch lookup, argument-menu callbacks, docs, and regression tests while preserving the existing `/status` to `/agentstatus` override.

## Reproduction Assessment

Yes for the current limitation. Source inspection shows the proposed account field is absent from strict Slack config validation, and current native command lookup cannot map a prefixed Slack command like `oz-reset` back to the internal `reset` command.

## Solution Assessment

Unclear until maintainers choose the public config shape. Account-level native naming is the right boundary, but the maintainable solution likely needs validation, collision handling, docs, and callback round-trip coverage rather than only minimal prefix threading.
