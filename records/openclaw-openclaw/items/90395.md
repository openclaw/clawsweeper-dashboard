## Summary

Keep open, but the patch is not merge-ready. Current main still has the missing-metadata early return, so the reported bug remains, but this branch stamps current metadata onto unverified chunks and conflicts with the existing stale-row safety invariant.

## What This Changes

The PR adds a `memory_index_meta_v1` write in `MemoryManagerSyncOps.runSync()` when metadata is absent but existing chunks prevent a full reindex.

## Best Possible Solution

Recover missing-metadata indexes by rebuilding or verifying and pruning the current rows before writing metadata, while leaving the safe-reindex race to the dedicated related PR.

## Reproduction Assessment

Yes from source, but not from a live run in this review. Current main still has the invalid-identity early return when chunks already exist, and the linked issue gives a concrete missing-meta-with-chunks path.

## Solution Assessment

No. The proposed fix is a plausible dead-loop mitigation, but it is not the best fix because it bypasses the metadata identity checks instead of rebuilding or proving the existing rows are current.
