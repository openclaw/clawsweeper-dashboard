## Summary

Keep this PR open: it addresses a real missing-metadata loop, but the patch still writes current identity metadata onto existing chunks without proving those chunks belong to the current provider, scope, sources, tokenizer, or live file set. That can turn stale or out-of-scope memory rows into trusted searchable memory.

## What This Changes

The PR adds a `runSync` recovery branch that writes memory index metadata when identity is missing but chunks exist, plus a manager regression test and a PR-specific proof script.

## Best Possible Solution

Recover missing metadata only after a verified rebuild or pruning sync, while keeping https://github.com/openclaw/openclaw/pull/90453 as the separate fix path for safe-reindex read races.
