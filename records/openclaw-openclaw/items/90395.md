## Summary

Keep this PR open, but do not merge it as-is. The missing-metadata dead loop is a real memory-core problem, yet this patch makes unverified existing chunks look valid before pruning or rebuilding them, which can surface stale or out-of-scope memory rows.

## What This Changes

The PR adds a memory-core `runSync` recovery branch that writes index metadata when metadata is missing but chunks exist, plus a manager regression test and a one-off proof script.

## Best Possible Solution

Recover missing metadata only after a verified rebuild or pruning sync, and keep https://github.com/openclaw/openclaw/pull/90453 as the separate fix path for safe-reindex read races.

## Reproduction Assessment

Yes from source inspection: current `runSync()` returns dirty when identity is missing and chunks exist, and the linked issue describes the resulting missing-metadata loop. I did not run the PR tests because this review is read-only.

## Solution Assessment

No. The patch is plausible for chunks that were just built by the same settings, but the best fix must verify, rebuild, or prune the chunk set before replacing identity metadata.
