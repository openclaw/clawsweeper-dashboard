## Summary

Keep open: this is a focused memory-core bug-fix PR, but the current patch is not merge-ready because it can mark unverified indexed chunks as valid metadata and the new proof script does not exercise the changed `runSync()` path.

## What This Changes

The PR adds a `writeMeta` call in `MemoryManagerSyncOps.runSync()` when identity is missing but chunks already exist, and adds a standalone memory-meta proof script.

## Best Possible Solution

Recover missing metadata by rebuilding or otherwise verifying the indexed rows against the current provider, source set, scope hash, chunking, tokenizer, and vector dimensions before search is made available.

## Reproduction Assessment

Yes, source inspection shows the current early-return path when identity is not valid and a full reindex is not selected. I did not establish a live gateway reproduction, and the PR's proof script does not exercise that manager path.

## Solution Assessment

No. The patch is a plausible symptom repair, but the safer fix is to rebuild or verify the index before writing metadata rather than making existing chunks authoritative by assertion.

That means the script can pass without exercising the changed branch or proving the reported runtime recovery.
