## Summary

Keep open: this targets a real memory-index dead loop, but the current branch can turn an unverified existing chunk set into valid searchable memory before a rebuild or prune verifies the rows.

## What This Changes

The branch adds a `runSync` recovery branch that writes memory index metadata when identity is missing but chunks exist, plus a manager regression test and a standalone proof script.

## Best Possible Solution

Recover missing metadata only after a verified rebuild or pruning sync, while keeping https://github.com/openclaw/openclaw/pull/90453 as the separate fix path for safe-reindex read races.

## Reproduction Assessment

Yes, from source inspection: current `runSync()` returns dirty when identity is missing and chunks exist, and the linked issue describes the resulting missing-metadata loop. I did not run tests because this review is read-only.

## Solution Assessment

No. The patch is plausible for chunks that were just built by the same settings, but the best fix must verify, rebuild, or prune the chunk set before replacing identity metadata.
