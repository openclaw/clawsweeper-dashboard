## Summary

Keep open: current main still lacks lazy ACP initialization for agent-configured TUI sessions, but this PR head remains unsafe/incomplete and lacks after-fix real behavior proof.

## What This Changes

The PR adds lazy ACP session initialization for agent-configured ACP sessions in auto-reply/TUI and CLI command paths, extends resolved agent config fields, adds tests, and updates the changelog.

## Best Possible Solution

Centralize lazy ACP initialization behind the existing ACP policy/init contract, guard raw and empty turns, recompute ACP prompt text after init, cover TUI/CLI regressions, and require real TUI ACP proof before merge.

## Reproduction Assessment

Yes. Current main is source-reproducible through tryDispatchAcpReply returning null for kind "none" before checking the agent runtime, and the PR regressions are source-reproducible from the latest head.

## Solution Assessment

No. Lazy initialization is the right direction, but this PR still needs the raw-run guard, ACP prompt recomputation after re-resolve, inline image-only first-turn handling, and real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Skip lazy ACP init for raw model runs:** `src/agents/agent-command.ts:495`
  - body: This condition is not gated by `!isRawModelRun`, while the ACP `runTurn` branch below still is. A raw `modelRun` or `promptMode: "none"` request on an ACP-configured session can start an external ACP runtime and write metadata before continuing down the embedded/raw path.
  - confidence: 0.88
- **[P2] Recompute the ACP prompt after lazy init:** `src/agents/agent-command.ts:521`
  - body: `body` is computed before lazy initialization, when `acpResolution` is still `none`. After this re-resolve makes the session ready, `runTurn` still receives the stale embedded-format body instead of the ACP-specific prompt form for internal-event turns.
  - confidence: 0.87
- **[P2] Include inline images in the lazy-init guard:** `src/auto-reply/reply/dispatch-acp.ts:391`
  - body: This guard returns before lazy init when there is no prompt text and no inbound channel media, but the normal ACP path also supports inline `params.images`. A first inline-image-only TUI/WebChat turn still falls back to embedded instead of initializing the ACP session.
  - confidence: 0.85
