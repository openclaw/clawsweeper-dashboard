## Best Possible Solution

Rebase onto current main, keep lazy ACP init after policy and non-empty-turn checks, exclude raw model runs, recompute ACP prompt text after re-resolve, include inline-image first turns, and prove the fixed TUI ACP path with redacted real output.

## Reproduction Assessment

Yes, source-reproducible. Current main returns null from `tryDispatchAcpReply` when ACP metadata is absent, so a first TUI turn for an agent configured with `runtime.type="acp"` falls back before any ACP session can be initialized; I did not run a live ACP harness.

## Solution Assessment

No. Lazy initialization is the right direction, but this PR is not yet the narrow maintainable fix because it can initialize ACP on raw CLI runs, keeps stale prompt formatting after re-resolve, and skips inline-image-only first turns.
