## Summary

Keep open: current main still has the reported ACP lazy-init gap, but the latest PR head introduces three source-level blockers around raw runs, prompt construction after re-resolution, and inline-image-only first turns.

## What This Changes

This PR adds lazy ACP session initialization for agent-configured ACP sessions in TUI/auto-reply and CLI command paths, expands resolved agent config fields, adds regression coverage, and updates the changelog.

## Best Possible Solution

Land a narrow lazy-init implementation after side-effect and empty-turn guards match ACP execution semantics, with regression tests for CLI raw runs, ACP prompt construction, and inline-image first turns.

## Reproduction Assessment

Yes. Current main can be source-reproduced with an ACP-configured TUI/auto-reply session whose ACP session metadata is absent, and the PR regressions are source-reproducible with raw model runs, internal-event first turns, and inline-image-only first turns.

## Solution Assessment

No. Lazy initialization is the right narrow direction, but this patch needs the raw-run guard, ACP prompt recomputation after re-resolve, and inline-image detection fixes before it is the maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Skip lazy ACP init for raw model runs:** `src/agents/agent-command.ts:483`
  - body: This condition is not gated by `!isRawModelRun`, while the ACP execution branch below still is. A `modelRun` or `promptMode: "none"` request on an ACP-configured session can start an external ACP runtime and write metadata, then continue down the embedded raw path.
  - confidence: 0.88
- **[P2] Recompute the ACP prompt after lazy init:** `src/agents/agent-command.ts:509`
  - body: `body` is computed before lazy initialization, when the ACP resolution is still not ready. After this re-resolve makes the session ready, `runTurn` still sends the stale embedded-formatted body instead of the ACP-specific prompt body.
  - confidence: 0.87
- **[P2] Include inline images in the lazy-init guard:** `src/auto-reply/reply/dispatch-acp.ts:393`
  - body: This guard returns before lazy init when there is no prompt text and no inbound channel media, but the normal ACP path also supports inline `params.images`. A first inline-image-only TUI/WebChat turn still falls back to embedded instead of initializing ACP.
  - confidence: 0.85

## Security Review
