## Summary

Keep open: current main still has the TUI/auto-reply ACP metadata gap, but the latest PR head is not ready to merge because three narrow control-flow blockers remain, including a security-sensitive raw-run side effect.

## What This Changes

The PR adds lazy ACP session initialization for agent-configured ACP sessions in auto-reply/TUI and CLI agent command paths, extends resolved agent config fields, adds tests, and updates the changelog.

## Best Possible Solution

Keep the lazy initialization direction, but make it side-effect-safe: skip CLI lazy init for raw runs, recompute the ACP prompt after re-resolve, include inline images in first-turn non-empty detection, and land with focused regression tests.

## Reproduction Assessment

Yes. Current main is source-reproducible by an auto-reply/TUI session for an agent with runtime.type="acp" and no stored ACP metadata, and the PR regressions are source-reproducible through raw model runs, internal-event first turns, and inline-image-only first turns.

## Solution Assessment

No. Lazy ACP initialization is the right narrow approach for the missing TUI path, but this PR still needs the three control-flow fixes before it is the maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Skip lazy ACP init for raw model runs:** `src/agents/agent-command.ts:483`
  - body: This block is not gated by `!isRawModelRun`, while the ACP `runTurn` branch below still is. A `modelRun` or `promptMode: "none"` request on an ACP-configured session can start an external ACP runtime and write metadata, then continue down the raw embedded path.
  - confidence: 0.88
- **[P2] Recompute the ACP prompt after lazy init:** `src/agents/agent-command.ts:509`
  - body: `body` is computed before lazy initialization, when `acpResolution` is still `none`. After this re-resolve makes the session ready, `runTurn` still receives the stale embedded-format body instead of the ACP-specific `resolveAcpPromptBody(...)` form for internal-event turns.
  - confidence: 0.87
- **[P2] Include inline images in the lazy-init nonempty guard:** `src/auto-reply/reply/dispatch-acp.ts:393`
  - body: This guard returns before lazy init when there is no prompt text and no inbound channel media, but the normal ACP path also supports inline `params.images`. A first inline-image-only TUI/WebChat turn still falls back to embedded instead of initializing the ACP session.
  - confidence: 0.85

## Security Review
