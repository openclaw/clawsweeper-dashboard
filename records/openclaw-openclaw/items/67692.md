## Summary

Keep open. Current main still has the ACP TUI lazy-init gap, but the latest PR head is not merge-ready because the new lazy-init path still has three source-level P2 regressions, including one medium security-sensitive side effect that can start an external ACP runtime from a raw model path.

## What This Changes

The PR adds lazy ACP session initialization for ACP-configured agents in TUI reply dispatch and CLI agent command execution, exposes runtime/default fields through resolved agent config, adds regression tests, and updates the changelog.

## Best Possible Solution

Keep the lazy ACP initialization direction, but make the implementation side-effect-safe and current-main-compatible: skip CLI lazy init for raw model runs, recompute the ACP prompt after lazy re-resolution, treat inline images as non-empty dispatch input, rebase the dirty branch, and add targeted tests for those paths.

## Reproduction Assessment

Yes. The original bug is reproducible by source on current main with an ACP-configured TUI session whose ACP resolution is `kind: "none"`; the PR-introduced issues are also source-reproducible from the raw-run, internal-event, and inline-image-only control flow.

## Solution Assessment

No. Lazy initialization is the narrow right direction for the missing TUI activation path, but this PR still starts ACP in a raw-run path and misses prompt/attachment edge cases that should be fixed before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Skip lazy ACP init for raw model runs:** `src/agents/agent-command.ts:472`
  - body: This new condition is not gated by `!isRawModelRun`, while the ACP execution branch below is. For `modelRun` or `promptMode: "none"`, an ACP-configured session with missing metadata can still initialize ACP and write metadata, then continue down the raw/embedded path. Add the raw-run exclusion before calling `initializeSession`.
  - confidence: 0.86
- **[P2] Recompute the ACP prompt after lazy init:** `src/agents/agent-command.ts:498`
  - body: `body` is computed before lazy init, while `acpResolution` is still `none`, so internal-event turns use the embedded/internal-context form. After re-resolving to `ready`, `runTurn` still sends that stale body. Recompute or defer the body once the final ACP resolution is known.
  - confidence: 0.88
- **[P2] Include inline images in the lazy-init guard:** `src/auto-reply/reply/dispatch-acp.ts:391`
  - body: This guard returns `null` when there is no prompt text and no inbound media, but normal ACP dispatch also supports `params.images` through `resolveAcpInlineImageAttachments`. A first inline-image-only turn still falls back to embedded instead of initializing ACP; include inline images in the non-empty check.
  - confidence: 0.84
