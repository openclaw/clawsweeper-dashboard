## Summary

Keep this PR open: the requested lazy ACP initialization is a real gap on current main, but the latest PR head still introduces raw-run side effects and misses prompt/inline-image edge cases before merge.

## What This Changes

The PR adds lazy ACP session initialization for agent-configured ACP sessions in TUI/auto-reply and CLI agent command paths, extends resolved agent config fields, adds regression tests, and updates the changelog.

## Best Possible Solution

Keep the lazy initialization direction, but make it side-effect-safe: skip CLI lazy init for raw runs, recompute ACP prompt text after lazy re-resolve, include inline images in first-turn non-empty detection, and land with focused regression tests.

## Reproduction Assessment

Yes. Current main can be reproduced by a TUI or auto-reply session for an agent with runtime.type="acp" and no stored ACP metadata; the PR regressions are source-reproducible through raw model runs, internal-event first turns, and inline-image-only first turns.

## Solution Assessment

No. Lazy ACP initialization is the right narrow fix for the missing TUI path, but this PR still needs the three control-flow fixes above before it is the maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Skip lazy ACP init for raw model runs:** `src/agents/agent-command.ts:483`
  - body: This lazy-init block is not gated by `!isRawModelRun`, while the ACP `runTurn` branch below still is. A `modelRun` or `promptMode: "none"` request on an ACP-configured session with missing metadata can start an external ACP runtime and write metadata, then continue down the raw embedded path.
  - confidence: 0.88
- **[P2] Recompute the ACP prompt after lazy init:** `src/agents/agent-command.ts:509`
  - body: `body` is computed before lazy initialization, when `acpResolution` is still `none`, so internal-event turns use the embedded prompt format. After this re-resolve makes the session ready, `runTurn` still sends that stale body instead of the ACP-specific `resolveAcpPromptBody(...)` form.
  - confidence: 0.87
- **[P2] Include inline images in the empty-turn guard:** `src/auto-reply/reply/dispatch-acp.ts:393`
  - body: This guard returns before lazy init when there is no prompt text and no inbound channel media, but the normal ACP path also supports inline `params.images`. A first inline-image-only TUI/WebChat turn still falls back to embedded instead of initializing the ACP session.
  - confidence: 0.85
