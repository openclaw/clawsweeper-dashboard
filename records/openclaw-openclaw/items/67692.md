## Summary

Keep this PR open: the underlying current-main ACP/TUI gap is real and the lazy-init direction is reasonable, but the latest head still has three source-proven blockers before merge.

## What This Changes

The PR adds lazy ACP session initialization for agent-configured ACP sessions in TUI/auto-reply and CLI agent command paths, extends resolved agent config fields, adds regression tests, and updates the changelog.

## Best Possible Solution

Keep the lazy initialization direction, but make it side-effect-safe: skip CLI lazy init for raw runs, recompute the ACP prompt after lazy re-resolve, include inline images in first-turn non-empty detection, and land with focused regression tests.

## Reproduction Assessment

Yes. Current main is source-reproducible by an ACP-configured TUI/auto-reply session with no stored ACP metadata; the PR regressions are source-reproducible through raw model runs, internal-event first turns, and inline-image-only first turns.

## Solution Assessment

No. Lazy ACP initialization is the right narrow fix for the missing TUI path, but this PR still needs the three control-flow repairs before it is the best maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Skip lazy ACP init for raw model runs:** `src/agents/agent-command.ts:495`
  - body: This condition is not gated by `!isRawModelRun`, while the ACP `runTurn` branch below still is. A `modelRun` or `promptMode: "none"` request on an ACP-configured session with missing metadata can start an external ACP runtime and write metadata, then continue through the raw embedded path.
  - confidence: 0.89
- **[P2] Recompute the ACP prompt after lazy init:** `src/agents/agent-command.ts:521`
  - body: `body` is computed before lazy initialization, when `acpResolution` is still `none`. After this re-resolve makes the session ready, `runTurn` still sends that stale embedded-format body instead of the ACP-specific `resolveAcpPromptBody(...)` form for internal-event turns.
  - confidence: 0.87
- **[P2] Include inline images in the empty-turn guard:** `src/auto-reply/reply/dispatch-acp.ts:391`
  - body: This guard returns before lazy init when there is no prompt text and no inbound channel media, but the normal ACP path also supports inline `params.images`. A first inline-image-only TUI/WebChat turn still falls back to embedded instead of initializing the ACP session.
  - confidence: 0.86
