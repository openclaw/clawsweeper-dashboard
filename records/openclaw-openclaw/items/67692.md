## Summary

Keep open: current main still lacks lazy ACP initialization for the reported TUI path, but this PR head remains blocked by missing real behavior proof plus correctness and security findings in the proposed implementation.

## What This Changes

The branch adds lazy ACP session initialization for agent-configured ACP runtimes in auto-reply/TUI and CLI agent command paths, extends resolved agent config fields/tests, and adds a changelog entry.

## Best Possible Solution

Centralize lazy ACP initialization after policy and non-empty-turn checks, exclude raw model runs, recompute ACP prompt text after re-resolve, handle inline-image first turns, and land with focused tests plus redacted real TUI ACP proof.

## Reproduction Assessment

Yes, source-reproducible. Current main returns null from tryDispatchAcpReply when ACP metadata is absent, so a first TUI turn for an agent configured with runtime.type="acp" falls back before any ACP session can be initialized; I did not run a live ACP harness.

## Solution Assessment

No. Lazy initialization is the right direction, but this implementation is not yet the narrow maintainable fix because it can initialize ACP on raw CLI runs, keeps stale prompt formatting after re-resolve, and skips inline-image-only first turns.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Gate lazy ACP init out of raw model runs:** `src/agents/agent-command.ts:495`
  - body: This condition is not gated by `!isRawModelRun`, while the ACP `runTurn` branch below still is. A raw `modelRun` or `promptMode: "none"` request on an ACP-configured session can start an external ACP runtime and write metadata before continuing down the embedded/raw path.
  - confidence: 0.88
- **[P2] Recompute the ACP prompt after lazy init:** `src/agents/agent-command.ts:521`
  - body: `body` is computed before lazy initialization, when `acpResolution` is still `none`. After this re-resolve makes the session ready, `runTurn` still receives the stale embedded-format body instead of the ACP-specific prompt form for internal-event turns.
  - confidence: 0.87
- **[P2] Include inline images in the lazy-init guard:** `src/auto-reply/reply/dispatch-acp.ts:391`
  - body: This guard returns before lazy init when there is no prompt text and no inbound channel media, but the normal ACP path also supports inline `params.images`. A first inline-image-only TUI/WebChat turn still falls back to embedded instead of initializing the ACP session.
  - confidence: 0.85
