## Summary

Keep open: current main still appears to have the TUI ACP fallback bug and this PR is a plausible fix path, but the latest diff still has blocking source-level issues plus missing after-fix real behavior proof.

## What This Changes

The branch adds lazy ACP session initialization for agent-configured ACP runtimes in auto-reply/TUI and CLI agent command paths, expands resolved agent config fields/tests, and adds a changelog entry.

## Best Possible Solution

Centralize lazy ACP initialization after policy and non-empty-turn checks, keep raw model runs embedded, preserve existing binding/runtime defaults, recompute ACP prompt text after init, handle inline images, and require real TUI ACP proof before merge.

## Reproduction Assessment

Yes, by source inspection. Current main returns null from tryDispatchAcpReply when ACP metadata is absent, so a first TUI turn for an agent configured with runtime.type="acp" falls back before any lazy ACP session initialization; I did not run a live ACP harness.

## Solution Assessment

No. Lazy initialization is the right direction, but this implementation still needs the raw-run guard, ACP prompt recomputation after re-resolve, inline-image handling, and real behavior proof before it is the maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Gate lazy ACP init out of raw model runs:** `src/agents/agent-command.ts:495`
  - body: This lazy-init condition is not gated by `!isRawModelRun`, while the ACP `runTurn` branch below still is. A raw `modelRun` or `promptMode: "none"` request on an ACP-configured session can start an external ACP runtime and write metadata before continuing down the embedded/raw path.
  - confidence: 0.88
- **[P2] Recompute the ACP prompt after lazy init:** `src/agents/agent-command.ts:521`
  - body: `body` is computed before lazy initialization, when `acpResolution` is still `none`. After this re-resolve makes the session ready, `runTurn` still receives the stale embedded-format body instead of the ACP-specific prompt form for internal-event turns.
  - confidence: 0.87
- **[P2] Include inline images in the lazy-init guard:** `src/auto-reply/reply/dispatch-acp.ts:391`
  - body: This guard returns before lazy init when there is no prompt text and no inbound channel media, but the normal ACP path also supports inline `params.images`. A first inline-image-only TUI/WebChat turn still falls back to embedded instead of initializing the ACP session.
  - confidence: 0.85
