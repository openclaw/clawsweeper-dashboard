Full review comments:

- **[P1] Resolve bridge links from source-origin targets:** `extensions/memory-wiki/src/lint.ts:75-77`
  - body: `extractWikiLinks` already rewrites Markdown links like `./target.md` against the generated wiki page path, so this helper receives `sources/target.md` rather than the original bridge-relative target. Joining that with `dirname(bridgeRelativePath)` produces paths like `memory/chatgpt/sources/target.md`, so the bridge-source false positive can remain.
  - confidence: 0.91
- **[P2] Guard malformed percent escapes before decoding:** `extensions/memory-wiki/src/lint.ts:69-71`
  - body: Imported bridge content can contain arbitrary Markdown link text. Calling `decodeURIComponent` on every candidate target means a malformed escape like `%ZZ` throws and makes `openclaw wiki lint` fail instead of reporting or ignoring a broken link.
  - confidence: 0.9
- **[P3] Keep bare reply_to targets lintable:** `extensions/memory-wiki/src/lint.ts:53-55`
  - body: The directive matcher makes the suffix optional, so `[[reply_to]]` is also ignored even though the PR only claims `reply_to_current` and `reply_to:<id>` are transcript directives. Keep the allowlist to known directive shapes so real pages or aliases named `reply_to` can still be linted.
  - confidence: 0.76
