The issue reports that an isolated cron job with an explicit opencode-go model resolves to the fallback provider even though OpenClaw lists and interactive sessions can use the plugin model.

Yes, at source level: current main forwards cron provider/model into the embedded runner, and that runner's second lookup lacks the static catalog fallback needed for bundled plugin catalog rows. I did not run a live Ubuntu/npm cron reproduction in this read-only review.

## Solution Assessment
