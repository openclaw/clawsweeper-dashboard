## Summary

Keep open: current main and latest release still let the Control UI save arbitrary Telegram cron announce `To` values, and the only closing PR is still open, conflicting, and not a landed fix.

## What This Changes

Request dashboard validation, helper text, and configured Telegram group/topic suggestions for cron announce `To` targets so invalid agent-like strings are not saved and deliverable targets are discoverable.

## Best Possible Solution

Land or replace #90841 with a fix that validates Telegram cron announce targets against the runtime contract and, if approved, exposes configured group/topic suggestions through an appropriate channel/plugin boundary.

## Reproduction Assessment

Yes by source inspection: current main saves arbitrary announce `To` values and Telegram runtime later throws the reported numeric chat id error for unresolvable values; no live dashboard run was performed in this read-only review.

## Solution Assessment

The best fix is not a numeric-only check; it should align with Telegram runtime target normalization and separately decide whether configured group/topic suggestions require a new safe exposure path.
