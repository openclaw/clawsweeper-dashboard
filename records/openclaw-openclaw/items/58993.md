## Summary

Keep open. Current main still only classifies Google Chat spaces with legacy `space.type`, and this PR is a focused classifier fix, but the new May 12 comment is current-release reproduction rather than after-fix proof from the PR head.

## What This Changes

The PR adds `spaceType` and `singleUserBotDm` to `GoogleChatSpace` and changes inbound direct/group classification to treat any legacy or modern DM signal as direct.

## Best Possible Solution

Land the narrow classifier fix after redacted PR-head Google Chat proof and a small regression test, while keeping broader Space/thread-routing work in the existing Google Chat routing lane.

## Reproduction Assessment

Yes for the classifier gap by source inspection: current main routes a modern `spaceType: "DIRECT_MESSAGE"` payload with no legacy `type` as group. I did not run live Google Chat traffic against the PR head.

## Solution Assessment

Yes for the narrow direct/group classifier: checking legacy `type`, modern `spaceType`, and `singleUserBotDm` in the Google Chat plugin is the right boundary. It is not yet proven as the full fix for the linked Space-message symptom.
