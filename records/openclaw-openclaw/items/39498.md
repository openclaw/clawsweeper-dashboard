## Summary

This PR should stay open, but the current patch should not merge: the ByteString crash is real, while the diff moves header-only sanitization into the generic SecretInput boundary and would silently mutate valid non-header credentials.

## What This Changes

The PR makes `normalizeSecretInputString` delegate to `normalizeOptionalSecretInput` and adds tests for trimming, blank input, non-Latin1 stripping, and Latin-1 preservation.

## Best Possible Solution

Keep generic SecretInput helpers trim/presence-only, and apply `normalizeSecretInput` or a clearly named header-safe helper only where resolved credentials are assembled into HTTP headers, with regressions for header safety and Unicode credential preservation.

## Reproduction Assessment

Yes. Node's `Headers` constructor reproduces the ByteString failure for U+2502, and current main has a source-visible Comfy path from trim-only API-key normalization into `X-API-Key`.

## Solution Assessment

No. The patch uses the right sanitizer at the wrong boundary; the narrower maintainable fix is scoped header sanitization while preserving generic credential semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve Unicode secrets outside HTTP headers:** `src/config/types.secrets.ts:116`
  - body: Delegating `normalizeSecretInputString` to `normalizeOptionalSecretInput` strips every code point above `0xff` for all SecretInput consumers. Matrix password auth resolves configured passwords through this generic path and sends them in a JSON login body, so a valid password like `пароль123` would become `123`; keep the generic helper trim-only and sanitize only header values.
  - confidence: 0.91
- **[P2] Keep presence checks independent from header sanitization:** `src/config/types.secrets.ts:116`
  - body: `hasConfiguredSecretInput` uses this helper as a presence check. After this change, an all-non-Latin1 configured value such as `пароль` normalizes to `undefined`, so gateway auth can treat a real credential as absent and take unrelated token-ref or missing-credential paths.
  - confidence: 0.87
