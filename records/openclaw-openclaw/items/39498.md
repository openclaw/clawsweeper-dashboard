## Summary

Keep this PR open, but the current patch should not merge because it fixes a real header ByteString crash by changing generic SecretInput semantics for every consumer, including non-header credentials and presence checks.

## What This Changes

The PR delegates `normalizeSecretInputString` to `normalizeOptionalSecretInput` and adds tests for trimming, blank input, non-Latin1 stripping, and Latin-1 preservation.

## Best Possible Solution

Keep generic SecretInput helpers trim-only, then apply ByteString-safe normalization only where secrets are assembled into HTTP headers, with regression coverage for both header safety and Unicode non-header credential preservation.

## Reproduction Assessment

Yes. Node's `Headers` constructor reproduces the ByteString failure for U+2502, and current main has a source-visible Comfy plugin path from trim-only API key normalization into the `X-API-Key` header.

## Solution Assessment

No. The PR uses the right sanitizer at the wrong boundary; the narrower maintainable fix is header-bound sanitization while preserving generic credential semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve Unicode secrets outside HTTP headers:** `src/config/types.secrets.ts:116`
  - body: Delegating `normalizeSecretInputString` to `normalizeOptionalSecretInput` strips every code point above `0xff` for all SecretInput consumers. Matrix password auth resolves configured passwords through this generic path and sends them in a JSON login body, so a valid password like `пароль123` would become `123`; keep the generic helper trim-only and sanitize only header values.
  - confidence: 0.91
- **[P2] Keep presence checks independent from header sanitization:** `src/config/types.secrets.ts:116`
  - body: `hasConfiguredSecretInput` uses this helper as a presence check. After this change, an all-non-Latin1 configured value such as `пароль` normalizes to `undefined`, so gateway auth can treat a real credential as absent and take unrelated missing-credential or token-ref paths.
  - confidence: 0.87
