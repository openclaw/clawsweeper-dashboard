## Summary

Keep open: the crash class is real, but this PR applies header-safety normalization at the generic SecretInput boundary, which would mutate valid non-header credentials and alter presence checks. It also lacks real after-fix runtime proof beyond unit-test output.

## What This Changes

The PR changes `normalizeSecretInputString` to delegate to `normalizeOptionalSecretInput` and adds direct tests for trimming, blank input, non-Latin1 stripping, and Latin-1 preservation.

## Best Possible Solution

Keep generic SecretInput string normalization trim-only, then apply ByteString-safe normalization only where secrets are assembled into HTTP headers, with regression coverage for both header safety and Unicode non-header credentials.

## Reproduction Assessment

Yes for the crash class: Node's `Headers` constructor reproduces the ByteString failure for U+2502, and current main has a source-visible path from trim-only configured API-key normalization into Comfy's `X-API-Key` header. I did not execute a full OpenClaw macOS command or plugin run in this read-only review.

## Solution Assessment

No. The PR uses the right sanitizer at the wrong boundary; the narrower maintainable fix is header-bound sanitization while preserving generic SecretInput semantics for JSON-body credentials and presence checks.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve Unicode secrets outside HTTP headers:** `src/config/types.secrets.ts:116`
  - body: Delegating `normalizeSecretInputString` to `normalizeOptionalSecretInput` strips every code point above `0xff` for all SecretInput consumers. Matrix password auth resolves configured passwords through this generic path and sends them in a JSON login body, so a valid password like `пароль123` would become `123`; keep the generic helper trim-only and sanitize only header values.
  - confidence: 0.91
- **[P2] Keep presence checks independent from header sanitization:** `src/config/types.secrets.ts:116`
  - body: `hasConfiguredSecretInput` uses this helper as a presence check. After this change, an all-non-Latin1 configured value such as `пароль` normalizes to `undefined`, so gateway auth can treat a real credential as absent and take unrelated token-ref or missing-credential paths.
  - confidence: 0.87
