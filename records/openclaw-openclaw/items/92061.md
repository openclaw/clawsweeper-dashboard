## What This Changes

Fix memory-core hybrid search so CJK queries using the FTS5 trigram tokenizer contribute a keyword score when relevant indexed text exists.

## Best Possible Solution

Add a regression fixture using the reporter's exact indexed chunk and query, then repair only a demonstrated planner, chunking, or ranking defect while preserving trigram MATCH/BM25 for contiguous terms of three or more characters.

## Solution Assessment

No, the suggested blanket character splitting and empty-result fallback are not yet the best solution because they replace valid ranked trigram matching with broader LIKE semantics. The exact failing text should first establish whether this is a planner defect or a request for non-contiguous/fuzzy matching.
