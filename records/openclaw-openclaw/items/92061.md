## Summary

Current main does not reproduce the claimed unconditional failure: a contiguous six-character Chinese query returns an FTS5 MATCH result with BM25 rank. The issue remains worth maintainer review because the reporter may instead expect fuzzy or noncontiguous CJK matching, which current behavior and tests intentionally do not provide.

## What This Changes

The issue asks memory-core trigram search to provide keyword-score credit for CJK queries when the full multi-character query does not produce a contiguous FTS5 MATCH hit.

## Best Possible Solution

Obtain a minimal query-plus-indexed-text fixture, then preserve ranked trigram MATCH for contiguous terms and add a narrowly specified zero-hit fallback only if maintainers explicitly choose fuzzy or noncontiguous CJK retrieval semantics.

## Reproduction Assessment

No. A direct current-runtime FTS5 trigram check returns a BM25-ranked result for the issue’s quoted Chinese phrase when that substring exists, while the report does not include its indexed text to establish a different failing case.

## Solution Assessment

No. Converting every CJK term into per-character LIKE clauses changes the product’s matching semantics, removes BM25 ranking on those results, and is broader than the demonstrated evidence; a fixture-backed and bounded fallback would be safer.
