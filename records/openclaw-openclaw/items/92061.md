## Summary

Current main and SQLite 3.51 do not reproduce the claimed unconditional CJK failure: a contiguous six-character Chinese query matches trigram FTS5 and receives BM25 rank. The report remains open because its indexed text is missing, so the remaining concern may be a request for fuzzy or noncontiguous CJK matching rather than a defect in the documented contiguous trigram behavior.

## What This Changes

Investigate whether trigram FTS5 hybrid memory search incorrectly assigns zero text score to CJK queries.

## Best Possible Solution

Preserve contiguous trigram MATCH behavior, obtain the exact indexed text and query, and add a focused regression only if that concrete pair fails; treat fuzzy or noncontiguous CJK retrieval as a separate product decision with precision tests.

## Reproduction Assessment

No. A direct SQLite 3.51 check and the current real-database regression test both show a contiguous six-character Chinese query matching with a nonzero BM25-derived text score; the report lacks the indexed text needed to reproduce its observed zero score.

## Solution Assessment

No. Splitting all CJK text into single-character LIKE predicates or falling back on every empty MATCH would change the feature into broad fuzzy matching; the narrow maintainable path is to reproduce the exact corpus/query pair first.
