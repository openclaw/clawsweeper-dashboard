## What This Changes

The issue asks for CJK trigram memory search to contribute a nonzero keyword text score in hybrid results instead of silently relying on vector score only.

## Best Possible Solution

Review and land one canonical memory-core fix with maintainer-approved CJK term decomposition, empty-MATCH fallback semantics, hybrid merge behavior, and focused regression coverage; then close this issue through the merged PR.

## Reproduction Assessment

Yes at source and dependency level: current main still routes multi-character CJK trigram tokens into quoted MATCH and only retries LIKE on exceptions, and a real in-memory FTS5 trigram query returns zero rows for the reported non-contiguous phrase. I did not run a full OpenClaw memory_search flow with Ollama and a populated CJK index.

## Solution Assessment

Unclear as a final fix: the issue is valid, but the best implementation still needs maintainer choice on CJK decomposition, ranking, and hybrid merge semantics. The right fix belongs in `memory-core` search planning and merge coverage, not in a new config option.
