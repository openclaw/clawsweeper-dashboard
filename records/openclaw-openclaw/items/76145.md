## Summary

Keep open. Current main still has a source-reproducible foreground/background lifecycle gap: the CLI returns the parent `agent` RPC result while `sessions_spawn` child completion and follow-up synthesis are handled asynchronously, and the remaining fix needs a maintainer decision on the CLI completion contract.

## What This Changes

The issue asks for `openclaw agent --json` to avoid returning partial parent-run output when a parent agent spawns parallel subagents and later synthesizes their results.

## Best Possible Solution

Define and implement a foreground CLI completion contract for spawned descendants: either drain pending child completions and return the latest parent synthesis, or explicitly mark JSON output as partial with documented timeout/liveness semantics.

## Reproduction Assessment

Yes, source-reproducible. Run `openclaw agent --json` against a parent agent that calls `sessions_spawn`; current source shows the CLI waits for the parent `agent` RPC final frame while child completion is awaited asynchronously in the subagent registry.

## Solution Assessment

Unclear. Waiting on descendants by default may be correct, but current docs intentionally make `sessions_spawn` non-blocking and #64281 proposes a separate synchronous primitive, so maintainers should choose the CLI contract before implementation.
