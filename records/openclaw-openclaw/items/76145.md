## Label Justifications

- `P2`: The issue affects a real multi-agent CLI workflow with limited blast radius, but it is not an emergency outage.
- `impact:session-state`: The report concerns stale or partial run liveness/final-state metadata while descendant session work continues.
- `impact:message-loss`: The foreground JSON can omit later synthesis turns produced after subagent replies, which is a lost-message surface for CLI consumers.

## Summary

The foreground CLI lifecycle gap is still present on current main: `openclaw agent --json` waits for the gateway agent response, while `sessions_spawn` descendant completion is handled asynchronously and can produce later parent follow-up runs the CLI output does not observe. This should stay open because the remaining choice is the user-facing contract for non-blocking spawn versus foreground JSON completion.

## What This Changes

The issue asks OpenClaw to stop returning incomplete `openclaw agent --json` output for parallel `sessions_spawn` workflows where descendant sub-agents continue running and later synthesis turns exist.

## Best Possible Solution

Maintainers should choose and document the `openclaw agent --json` contract for background descendants, then implement that chosen path with regression coverage for `sessions_spawn`, descendant drain or explicit non-drain metadata, and final JSON state.

## Reproduction Assessment

Yes for source reproduction: current main still has the foreground CLI resolve from the gateway agent response while `sessions_spawn` descendant completion is backgrounded with `void waitForSubagentCompletion(...)`. I did not run the reporter's Docker multi-agent scenario, so the status is source-reproducible rather than live-reproduced.

## Solution Assessment

Unclear: blocking until all descendants finish would address the reporter's observed workflow, but current docs define `sessions_spawn` as non-blocking. The best solution needs a maintainer-selected contract, such as explicit wait behavior, clearer incomplete-run metadata, or a separate blocking delegation primitive.
