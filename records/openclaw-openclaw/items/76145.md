## Summary

Keep open. Current main still has the foreground/background lifecycle gap: `openclaw agent --json` returns the parent Gateway run result while `sessions_spawn` child completion is tracked asynchronously, and the remaining repair needs a maintainer decision on the CLI completion contract.

## What This Changes

The issue requests that `openclaw agent --json` either wait for spawned descendant synthesis or clearly report partial output instead of returning only the parent announce run.

## Best Possible Solution

Define one foreground CLI contract for spawned descendants: either drain pending child completions and return the final parent synthesis, or return explicit partial-output metadata with documented wait/yield semantics.

## Reproduction Assessment

Yes, at source level: run `openclaw agent --json` against a parent agent that calls `sessions_spawn`; current main returns the parent Gateway run while child completion waits in the asynchronous subagent registry path. I did not run the reporter's Docker/SRE-MAS setup live in this read-only review.

## Solution Assessment

Unclear. The retracted “capture the last turn” fix is not sufficient; maintainers should choose between default blocking, an explicit wait mode, partial-output metadata, or the separate `sessions_delegate` direction.
