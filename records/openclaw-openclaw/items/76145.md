## Label Justifications

- `P2`: The issue affects advanced multi-agent CLI JSON workflows but does not make ordinary agent use unusable.
- `impact:session-state`: The foreground JSON result can describe a parent run while descendant session work is still active or later synthesizing.
- `impact:message-loss`: The reported CLI JSON output can omit later parent synthesis that operators expected from the multi-agent run.

## Summary

Keep open: current main and v2026.6.10 still return the foreground parent `agent` RPC result for `openclaw agent --json`, while `sessions_spawn` descendants complete through later registry-managed events that the CLI request does not drain. The remaining fix is real but needs maintainer direction because it defines public CLI JSON lifecycle semantics.

## What This Changes

The issue asks for `openclaw agent --json` to avoid reporting only the parent announce turn when parallel `sessions_spawn` child work later produces the actual synthesis.

## Best Possible Solution

Define the CLI JSON lifecycle contract, then either make the selected wait path drain descendant completions or surface explicit pending-descendant/partial-output state without overloading `replayInvalid`.

## Reproduction Assessment

Yes, at source level: current main returns the gateway parent-run response for `--json` while `sessions_spawn` returns accepted and descendant completion is tracked separately. I did not run the reporter's Docker MAS scenario.

## Solution Assessment

Unclear until maintainers choose the contract. The discussion already rules out changing only `finalAssistantVisibleText` extraction or overloading `replayInvalid`; the durable fix is a CLI lifecycle or explicit metadata decision.
