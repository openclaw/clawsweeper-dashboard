## Summary

Keep open. Current main still has a source-reproducible foreground/background lifecycle gap: the CLI returns the parent Gateway run result while `sessions_spawn` completions are handled later through background announce flows, and the remaining fix needs a maintainer decision on the CLI completion contract.

## What This Changes

The issue asks for `openclaw agent --json` to wait for spawned subagents or clearly report partial output instead of returning only the parent announce turn.

## Best Possible Solution

Define a foreground CLI completion contract for spawned descendants: either drain pending child completions and return the latest parent synthesis, or explicitly mark JSON output as partial with documented timeout/liveness semantics.

## Reproduction Assessment

Yes for a source-level reproduction path: run `openclaw agent --json` against a parent agent that calls `sessions_spawn`. Current source shows the CLI waits for the parent `agent` RPC final frame while child completion is awaited asynchronously.

## Solution Assessment

Unclear. Waiting on descendants by default may be correct, but current docs intentionally make `sessions_spawn` non-blocking and #64281 proposes a separate synchronous primitive, so maintainers should choose the contract before implementation.
