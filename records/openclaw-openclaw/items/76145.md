## Summary

Keep open. Current main still has a source-reproducible foreground/background lifecycle gap: `openclaw agent --json` returns the parent run's final Gateway frame while `sessions_spawn` child completion and parent synthesis are handled later through async announce flows, and the right CLI contract needs maintainer choice before a repair.

## What This Changes

The issue asks for `openclaw agent --json` to avoid returning an incomplete parent announce result while parallel `sessions_spawn` descendants are still running and later synthesis turns are pending.

## Best Possible Solution

Define a foreground CLI completion contract for spawned descendants, then implement it consistently in Gateway/CLI JSON output and docs so consumers either receive the final synthesized parent result or an explicit, documented partial-result state.

## Reproduction Assessment

Yes for a source-level reproduction path: run `openclaw agent --json` against a parent agent that calls `sessions_spawn`. Current source shows the CLI waits for the parent `agent` RPC final frame while child completion and announce handling are awaited asynchronously.

## Solution Assessment

Unclear. Waiting on descendants by default may solve the reported operator pain, but it conflicts with the documented non-blocking `sessions_spawn` contract and overlaps with the separate `sessions_delegate` direction, so maintainers should choose the contract first.
