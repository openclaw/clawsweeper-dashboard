## Summary

Keep open: current main and the latest stable release still return the foreground parent `agent` RPC result for `openclaw agent --json`, while `sessions_spawn` descendant completion remains a later background/session event. The source gap is clear, but the repair defines public CLI JSON lifecycle semantics and needs maintainer direction.

## What This Changes

The issue asks for `openclaw agent --json` to avoid reporting an incomplete parent announce result when parallel `sessions_spawn` subagents continue and later synthesize output.

## Best Possible Solution

Define the CLI JSON lifecycle contract, then either drain required descendant completions on the selected wait path or expose explicit pending-descendant/partial-output state without overloading `replayInvalid`.

## Reproduction Assessment

Yes, at source level: current main returns the gateway parent-run response for `--json` while `sessions_spawn` returns immediately and descendant completion is tracked separately; the exact Docker MAS scenario was not rerun.

## Solution Assessment

Unclear until maintainers choose the contract; the discussion and source rule out only changing text extraction or overloading `replayInvalid`, leaving a CLI lifecycle or explicit metadata decision.
