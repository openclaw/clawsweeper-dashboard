## Label Justifications

- `impact:session-state`: The foreground JSON result can describe a parent run while descendant session work is still active or later synthesizing.

## Summary

Keep open: current main and v2026.6.10 still return the foreground parent `agent` RPC result for `openclaw agent --json`, while `sessions_spawn` remains non-blocking and descendant completion or synthesis arrives through later parent-session events. The source gap is clear, but the fix defines public CLI JSON lifecycle semantics and still needs maintainer direction.

## What This Changes

The issue asks OpenClaw to make `openclaw agent --json` accurately represent runs whose parent agent spawns parallel subagents, instead of returning only the parent announce turn while descendant work is still active.

## Best Possible Solution

Define the CLI JSON lifecycle contract, then either make the selected wait path drain descendant completions or surface explicit pending-descendant/partial-output state without overloading `replayInvalid`.

## Reproduction Assessment

Yes, at source level: current main returns the gateway parent-run response for `--json` while `sessions_spawn` returns accepted and descendant completion is tracked separately. I did not run the reporter's Docker MAS scenario.

## Solution Assessment

Unclear until maintainers choose the contract. The discussion already rules out changing only `finalAssistantVisibleText` extraction or overloading `replayInvalid`; the durable fix is a CLI lifecycle or explicit metadata decision.
