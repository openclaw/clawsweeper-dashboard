## Summary

Keep open. Current main still returns the Gateway agent response when the parent turn ends, while `sessions_spawn` completions are handled later through background registry/announce flows, so the original CLI JSON is not guaranteed to contain the later synthesis turns.

## What This Changes

Ensure `openclaw agent --json` waits for or clearly reports push-based subagent completion instead of returning only the parent’s initial announcement text.

## Best Possible Solution

Define a foreground CLI completion contract for runs with spawned descendants: either drain pending child completions within the command timeout and update `finalAssistantVisibleText` from the latest parent assistant turn, or explicitly mark JSON output as partial with documented timeout semantics.

## Reproduction Assessment

Yes for a code-level reproduction path: a Gateway-backed `openclaw agent --json` run whose parent uses `sessions_spawn` can return the parent turn result while child completions are still handled asynchronously. The exact SRE-MAS container scenario was not live-reproduced in this review.

## Solution Assessment

Unclear. Waiting for every subagent by default may be the desired CLI behavior, but it needs maintainer confirmation because current subagent docs intentionally describe push-based non-blocking completion.
