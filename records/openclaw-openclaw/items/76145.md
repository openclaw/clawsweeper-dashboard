## Summary

Keep open: current main and v2026.6.8 still return the foreground parent `agent` RPC result for `openclaw agent --json`, while `sessions_spawn` remains non-blocking and descendant synthesis can arrive through later parent-session events. The source gap is clear, but the fix changes the public CLI JSON lifecycle contract and needs maintainer direction.

## What This Changes

Request for `openclaw agent --json` to avoid returning incomplete parent-run JSON while spawned sub-agents are still producing follow-up synthesis, with clearer completion metadata.

## Best Possible Solution

Define the CLI JSON lifecycle contract, then either make the chosen wait path drain descendant completions or surface structured pending-descendant/partial-output state without overloading `replayInvalid`.

## Reproduction Assessment

Yes, at source level: current main returns the gateway parent run response for `--json` while `sessions_spawn` returns accepted and completes through later parent-session events. I did not run the reporter's Docker MAS scenario.

## Solution Assessment

Unclear until maintainers choose the contract. The discussion already ruled out changing only `finalAssistantVisibleText` extraction or overloading `replayInvalid`; the durable fix is a CLI lifecycle or explicit metadata decision.
