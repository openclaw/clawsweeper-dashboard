## Summary

Keep open. Current main still lets `openclaw agent --json` return the parent Gateway agent result while `sessions_spawn` descendants complete through later background announce/queue flows, so the JSON result can be partial for this reported multi-agent pattern.

## What This Changes

The issue requests that `openclaw agent --json` either wait for parallel sub-agent completion or explicitly report partial output, with `finalAssistantVisibleText` reflecting the final synthesized parent answer.

## Best Possible Solution

Define a foreground CLI completion contract for spawned descendants, then either drain pending child completions within the command timeout and aggregate the latest parent visible text, or mark/document the JSON response as partial with explicit timeout semantics.

## Reproduction Assessment

Yes for a source-level reproduction path: a Gateway-backed `openclaw agent --json` parent run that calls `sessions_spawn` can return the parent run result while child completion handling continues asynchronously. The exact SRE-MAS container scenario was not live-reproduced here.

## Solution Assessment

Unclear. Waiting for every descendant may be the right CLI behavior, but current docs and tests intentionally describe non-blocking subagent completion, so the safest fix starts with a maintainer decision on the CLI/Gateway contract.
