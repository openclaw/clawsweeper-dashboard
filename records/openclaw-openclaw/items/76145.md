## Summary

Keep open. Current main still returns the Gateway `agent` RPC result when the parent turn resolves, while spawned subagent completions are registered as asynchronous background handoffs, so `openclaw agent --json` can still report the announce turn instead of later synthesis turns.

## What This Changes

The issue asks for `openclaw agent --json` to either wait for spawned subagent completion and report the latest parent synthesis, or clearly mark/document partial output.

## Best Possible Solution

Define an explicit foreground completion contract for `openclaw agent --json`: either drain spawned descendants within the command timeout and report the latest parent assistant text, or return a documented partial result with machine-readable pending/partial metadata.

## Reproduction Assessment

Yes for a source-level reproduction path: a Gateway-backed `openclaw agent --json` run whose parent uses `sessions_spawn` can return the parent run result while child completion handling continues asynchronously. The exact external Docker scenario was not live-reproduced in this read-only pass.

## Solution Assessment

Unclear. Waiting for every spawned descendant by default may solve the operator-visible bug, but it needs maintainer product judgment because current docs intentionally describe subagents as non-blocking, push-based background runs.
