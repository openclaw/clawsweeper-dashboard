## Summary

Keep open. Current main still returns the foreground `openclaw agent --json` response when the parent run resolves, while `sessions_spawn` descendants are tracked and completed through async follow-up paths that the CLI does not drain; the later comment correctly narrows this to a foreground-vs-background lifecycle contract gap.

## What This Changes

The issue asks for `openclaw agent --json` to avoid reporting only an interim parent announce when spawned sub-agents later produce the real synthesis, and to expose complete/partial state accurately.

## Best Possible Solution

Define a foreground CLI completion contract for descendant runs, then implement a targeted drain or explicit partial-result signal with tests and docs covering `openclaw agent --json` plus `sessions_spawn`.

## Reproduction Assessment

Yes for a code-level reproduction path: a Gateway-backed `openclaw agent --json` run whose parent calls `sessions_spawn` can resolve the parent announce result while child completion handling remains async. The reporter's exact container setup was not live-reproduced here.

## Solution Assessment

Unclear. Waiting for descendants may be right for foreground JSON output, but the reporter's last-text-only fix is insufficient because the field already uses the last assistant of the resolved run, and the default blocking behavior needs maintainer product/API judgment.
