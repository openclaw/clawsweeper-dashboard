## Summary

Keep open: current main and v2026.6.10 still return the foreground parent run result for CLI JSON, while `sessions_spawn` descendant completion remains separate push-based work; the source gap is real but needs a maintainer contract decision before code.

## What This Changes

The issue asks for `openclaw agent --json` to avoid reporting a parent announce turn as the final result when parallel `sessions_spawn` descendants are still producing later synthesis.

## Best Possible Solution

Define the CLI JSON lifecycle contract, then either make the selected wait path drain descendant completions or surface explicit pending-descendant/partial-output state without overloading `replayInvalid`.

## Reproduction Assessment

Yes, at source level: current main sends `openclaw agent --json` through a foreground gateway `agent` request and returns that result, while `sessions_spawn` child completion is monitored and delivered through separate later parent-session events. I did not run the reporter's Docker MAS scenario.

## Solution Assessment

Unclear until maintainers choose the public CLI contract. The discussion and source both rule out changing only `finalAssistantVisibleText` extraction or overloading `replayInvalid`; the durable fix is a CLI lifecycle or explicit metadata decision.
