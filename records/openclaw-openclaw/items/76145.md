## Summary

Keep open. Current main still returns the parent Gateway agent result while descendant `sessions_spawn` completions are handled asynchronously, so `finalAssistantVisibleText` can describe only the announce turn instead of the later synthesis; this needs a maintainer-defined CLI completion contract.

## What This Changes

The issue requests that Gateway-backed `openclaw agent --json` not report the first parent turn as final when `sessions_spawn` descendants are still running, and that JSON semantics make partial output explicit.

## Best Possible Solution

Define an explicit foreground completion contract for Gateway `openclaw agent --json` runs with spawned descendants, so JSON either contains the final synthesized parent turn or clearly reports partial output and pending work.

## Reproduction Assessment

Yes. A code-level reproduction path exists: a Gateway-backed `openclaw agent --json` run whose parent uses `sessions_spawn` can return the parent run result while child completion handling continues through asynchronous registry and announce flows.

## Solution Assessment

Unclear. Waiting for every descendant by default may be correct for this CLI surface, but the repo currently documents `sessions_spawn` as non-blocking, so maintainers should choose between default waiting, an explicit wait option, or a documented partial-output contract.
