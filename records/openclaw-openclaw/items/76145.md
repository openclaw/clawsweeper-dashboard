## Summary

Keep open. Current main still has a foreground-vs-background lifecycle gap: `openclaw agent --json` returns the parent Gateway run result while `sessions_spawn` completion is explicitly non-blocking and delivered later through the subagent registry/announce path.

## What This Changes

The issue asks `openclaw agent --json` to wait for or clearly report pending `sessions_spawn` descendants so JSON output contains the final synthesis instead of only the parent announce turn.

## Best Possible Solution

Define a foreground CLI completion contract for spawned descendants, then either add an explicit/default wait-and-drain path that reports the latest parent synthesis or return a clearly documented partial result with pending-descendant metadata.

## Reproduction Assessment

Yes for a code-level reproduction: a Gateway-backed `openclaw agent --json` run whose parent uses `sessions_spawn` can return when the parent announce run resolves while child completion is still handled asynchronously. The exact SRE-MAS container scenario was not live-reproduced.

## Solution Assessment

Unclear until maintainers choose the contract. The reporter's “last assistant text” framing is not sufficient because current code already takes the last assistant of the returned run, and overloading `replayInvalid` would conflate retry safety with partial output.
