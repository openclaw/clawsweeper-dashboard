## Summary

Keep open: current main and the latest release still expose the parent-turn result for `openclaw agent --json` while `sessions_spawn` descendants continue through the non-blocking completion path, so the central CLI contract gap remains unresolved.

## What This Changes

The issue asks for `openclaw agent --json` to avoid reporting the parent announce turn as a complete multi-agent result when `sessions_spawn` descendants are still running.

## Best Possible Solution

Define the `openclaw agent --json` lifecycle contract, then either wait for descendant completion in the chosen mode or surface a clear descendants-pending/incomplete state without overloading `replayInvalid`.

## Reproduction Assessment

Yes at source level: current main waits for the parent gateway agent RPC result and `sessions_spawn` is documented as non-blocking, so descendants can continue after the CLI JSON response. I did not run the reporter's Docker MAS scenario, so the live sub-agent-kill observation remains unverified.

## Solution Assessment

No, not as originally framed: the issue discussion correctly retracts the last-turn extraction and `replayInvalid` overload fixes. The maintainable path needs a maintainer decision on the CLI JSON lifecycle contract before code is queued.
