## Summary

Keep open. Current main still has the foreground/background lifecycle gap: `openclaw agent --json` returns the parent run's final Gateway response while `sessions_spawn` descendants complete later through background wait and announce paths, so the remaining repair needs a maintainer decision on the CLI completion contract.

## What This Changes

The issue reports that `openclaw agent --json` exits after the parent announce turn while parallel `sessions_spawn` sub-agents are still running, leaving JSON consumers with partial output.

## Best Possible Solution

Define a foreground CLI contract for spawned descendants: either drain pending child completions and return the latest parent synthesis, or explicitly mark JSON output as partial with documented timeout/liveness semantics while leaving `sessions_spawn` non-blocking.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: run `openclaw agent --json` against a parent agent that calls `sessions_spawn`, and the CLI waits for the parent `agent` RPC final frame while child completions are awaited asynchronously.

## Solution Assessment

Unclear. Waiting on descendants by default may solve the observable bug, but current docs intentionally describe `sessions_spawn` as non-blocking and #64281 proposes a separate blocking delegation primitive, so maintainers should choose the contract before implementation.
