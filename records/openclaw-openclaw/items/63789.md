Keep open: current main does not implement the proposed handoff_id, structured acknowledgement states, or append-only handoff ledger, and maintainer discussion says this lower-level primitive is still useful and should stay separate from the higher-level lifecycle work in #65403. ## What This Changes

Adds a docs-only proposal for a minimal Phase 1 agent-to-agent handoff protocol with handoff IDs, structured acknowledgements, control-only outcome visibility, and an append-only ledger.

## Best Possible Solution

Keep the draft open for maintainer architecture review. If accepted, either land it as a properly formatted RFC/proposal page or replace it with a narrow source implementation at the existing sessions_send A2A boundary, with tests covering NO_REPLY, REPLY_SKIP, and ANNOUNCE_SKIP visibility.

## Solution Assessment

Unclear for the final product fix. A proposal document is a reasonable discussion step, but the maintainable fix is a narrow implementation plus regression tests after maintainers settle the protocol boundary; the current docs patch also needs standard front matter before docs publication.
