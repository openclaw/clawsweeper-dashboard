## Summary

Keep open. The PR is a draft docs-only proposal for a lower-level handoff primitive, and current main has related `sessions_send` A2A behavior but not the requested `handoff_id`, structured ack states, or append-only handoff ledger. Maintainer discussion explicitly says this remains useful and should stay separate from the higher-level meeting lifecycle tracked in #65403.

## What This Changes

Adds `docs/concepts/a2a-handoff-phase1.md`, a proposal for a minimal agent-to-agent handoff protocol with handoff IDs, structured acknowledgements, control-only outcome visibility, and an append-only ledger.

## Best Possible Solution

Keep the draft open for maintainer architecture review. If accepted, either land it as a properly formatted RFC/proposal page or replace it with a narrow implementation PR that adds the handoff envelope and ledger at the existing `sessions_send` A2A boundary with focused regression tests for silent/control-only outcomes.

## Reproduction Assessment

Not applicable as a docs-only proposal. The observable gap can be checked from current main by inspecting `sessions_send` return metadata and the A2A control-token tests; there is no failing reproduction bundled in this PR.

## Solution Assessment

Unclear for the final product fix. A proposal document is a reasonable discussion step, but the maintainable solution is a narrow source implementation plus tests after maintainers settle the protocol boundary.
