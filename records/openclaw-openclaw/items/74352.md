## Summary

Keep open: current main still has the fixed 20s file-chooser client deadline, so this PR addresses a real source-reproducible browser upload timeout path, but the diff still leaves the explicit CLI timeout edge and lacks after-fix real behavior proof.

## What This Changes

The PR changes browser file-chooser arming to use the browser action timeout budget, raises CLI upload’s default transport timeout to 60 seconds, and adds client timeout assertions.

## Best Possible Solution

Land the narrow timeout alignment after padding the explicit CLI timeout path and adding redacted after-fix real behavior proof, while leaving broader upload-flow reliability to https://github.com/openclaw/openclaw/issues/38844.

## Reproduction Assessment

Yes, source-reproducible. Current main sends /hooks/file-chooser with a fixed 20,000ms client deadline while the route can perform tab resolution and Playwright/Chrome-MCP setup before responding; I did not run a live browser flow in this read-only review.

## Solution Assessment

No, not yet. The core browser client timeout direction is the narrow maintainable fix, but the CLI explicit-timeout wrapper should receive the same slack or a shared resolver, and the external PR still needs after-fix real behavior proof.
