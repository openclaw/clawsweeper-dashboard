## Summary

Keep open. The PR targets a real current-main timeout bug, but it still leaves the explicit CLI upload timeout with no transport slack, lacks after-fix real browser upload proof, and is currently reported as conflicting.

## What This Changes

The branch changes browser file-chooser arming to use action-style client timeouts, raises the CLI upload default transport timeout to 60s, and adds client timeout assertions.

## Best Possible Solution

Land the narrow timeout alignment after padding or sharing the explicit CLI timeout resolver, adding redacted after-fix live upload proof, and leaving broader upload-flow reliability to the canonical browser upload issue.

## Reproduction Assessment

Yes, source-reproducible. Current main sends `/hooks/file-chooser` with a fixed 20,000ms client deadline while the route can do tab resolution, upload-path validation, Playwright setup, and optional clicking before responding.

## Solution Assessment

No, not yet. The browser-client timeout direction is the narrow maintainable fix, but the CLI explicit-timeout wrapper should get the same slack or shared resolver and the external PR still needs after-fix real behavior proof.
