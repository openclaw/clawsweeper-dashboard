## Summary

Keep open. Current main still lacks Signal `ReplyToId` forwarding and `authorNumber` handling, while this PR is the narrow canonical implementation and its prior review concerns appear addressed on the branch.

## What This Changes

The PR updates the Signal monitor, quote typing, regression tests, and changelog so visible signal-cli quote IDs and authors are forwarded into inbound `ReplyTo*` context while blocked or debounced-merged quote metadata is suppressed.

## Best Possible Solution

Land this PR or an equivalent Signal plugin patch after normal maintainer review and CI, preserving the visibility-gated metadata handling, authorNumber/UUID routing, debounced-merge clearing, and focused regression tests.

## Reproduction Assessment

Yes. A Signal inbound `dataMessage.quote` containing `id` and `authorNumber` reproduces the gap on current main by static inspection: quote text/sender can reach context, but quote IDs cannot and `authorNumber` is neither typed nor mapped.

## Solution Assessment

Yes. The PR keeps the fix inside the Signal plugin monitor boundary, follows the upstream signal-cli quote shape, uses existing identity and context-visibility helpers, and does not require a new core API.
