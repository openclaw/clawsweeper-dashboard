## Summary

Keep this PR open for maintainer review: current main still has the OpenRouter Anthropic conversation-cache gap, but this branch is not ready to merge because it omits upstream-cacheable document blocks, leaves user-facing docs stale, lacks contributor real-behavior proof, and overlaps with https://github.com/openclaw/openclaw/issues/63034 plus https://github.com/openclaw/openclaw/pull/63062.

## What This Changes

This PR changes OpenRouter Anthropic payload shaping to mark the last system/developer message and trailing user turn with `cache_control`, adds focused tests, and adds a changelog entry.

## Best Possible Solution

Land one canonical OpenRouter Anthropic cache-marker policy that caps explicit breakpoints, supports upstream-cacheable trailing user blocks including documents, updates public docs, and includes redacted real OpenRouter proof.

## Reproduction Assessment

## Solution Assessment

No, not as-is. The direction is plausible, but the patch should handle document blocks or provide provider rejection proof, align docs, add real OpenRouter evidence, and be chosen against the overlapping implementation path.
