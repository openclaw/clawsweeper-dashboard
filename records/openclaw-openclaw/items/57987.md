## Summary

Keep open: current main still has the OpenRouter conversation-cache gap, and this PR is a plausible implementation candidate, but it still needs real OpenRouter proof plus fixes for the document-block and docs issues before merge.

## What This Changes

The PR changes Anthropic/OpenRouter payload shaping to mark only the last system/developer message plus the trailing user turn with ephemeral cache_control markers, and updates cache-control tests and changelog text.

## Best Possible Solution

Land one canonical OpenRouter Anthropic cache-marker policy that caps explicit breakpoints, marks upstream-supported cacheable trailing user blocks, updates public prompt-caching/OpenRouter docs, and includes redacted real OpenRouter proof before superseding overlapping work.

## Reproduction Assessment

Yes. Source inspection on current main shows the OpenRouter wrapper invokes the affected helper, the helper does not mark trailing user messages, and current tests assert that user messages remain unchanged.

## Solution Assessment

No, not as-is. The direction is plausible, but the PR excludes document blocks despite upstream cache support, leaves public docs stale, lacks real OpenRouter proof, and needs maintainer selection against the overlapping fix PR.
