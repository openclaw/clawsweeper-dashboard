## Summary

Keep open. Current main still lacks OpenRouter trailing-user conversation-history cache markers, but this external PR still needs real OpenRouter proof, a document-block correction, docs alignment, and maintainer selection against the overlapping open implementation.

## What This Changes

The PR changes Anthropic/OpenRouter payload shaping to mark only the last system/developer message plus the trailing user turn with ephemeral cache_control, updates focused tests, and adds a changelog note.

## Best Possible Solution

Land one canonical OpenRouter Anthropic cache-marker policy that caps explicit breakpoints, handles upstream-supported cacheable trailing user blocks, updates public docs, and includes redacted real OpenRouter proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence current-main path: an OpenRouter Anthropic payload with system plus user messages reaches the helper, which tags system/developer content and leaves trailing user content unchanged; current tests assert that behavior.

## Solution Assessment

No, not as-is. The direction is plausible, but the PR omits cacheable document blocks despite upstream support, leaves public docs stale, lacks real OpenRouter proof, and overlaps with another open implementation path.
