## Summary

Keep this PR open. Current main still lacks trailing user-turn cache markers on the OpenRouter Anthropic path, and the PR remains a plausible implementation candidate rather than obsolete cleanup. The remaining maintainer decision is whether this shared helper change should also affect GitHub Copilot and how to reconcile the overlapping #63034/#63062 work.

## What This Changes

The PR updates the shared Anthropic cache marker helper to mark only the last system/developer message plus the trailing user turn, backtrack to cacheable blocks, and adds OpenRouter/cache-control tests plus a changelog entry.

## Best Possible Solution

Land one focused implementation that gives OpenRouter Anthropic requests a bounded, cacheable trailing user-turn marker, updates the prompt-caching docs, and then close or supersede the overlapping tracker/PR. Prefer the implementation shape that keeps the four-breakpoint limit and cacheable-block rules explicit, while making the GitHub Copilot/shared-SDK impact intentional.

## Reproduction Assessment

Yes. A deterministic current-main reproduction is an OpenRouter Anthropic payload with a trailing user message passed through createOpenRouterSystemCacheWrapper; current tests and source show the user content remains unmarked.

## Solution Assessment

Unclear. The PR is a stronger implementation than the overlapping one-line #63062 because it caps generated system/developer markers and backtracks to cacheable blocks, but the shared helper/Copilot blast radius needs maintainer approval or an OpenRouter-specific split.
