## Summary

Keep open. Current main still lacks OpenRouter trailing-user cache markers and this PR has useful unique breakpoint-limit work, but it should not merge as-is because real OpenRouter proof is missing, document user blocks are excluded despite the upstream cache contract, docs remain system/developer-only, and the overlapping OpenRouter cache PR/issue need one canonical policy.

## What This Changes

The PR adds OpenRouter Anthropic trailing-user `cache_control` marking, limits system/developer marking to the last such message, updates focused tests, and adds a changelog entry.

## Best Possible Solution

Land one canonical OpenRouter Anthropic cache-marker policy that caps explicit breakpoints, supports upstream-cacheable trailing user blocks, updates public docs, and includes redacted real OpenRouter proof.

## Reproduction Assessment

Yes, source-level. On current main, an OpenRouter Anthropic payload with system plus trailing user messages goes through a helper that tags only system/developer content, and the focused tests assert the user message remains unchanged.

## Solution Assessment

No, not as-is. The direction is useful, but the branch should handle document blocks or provide OpenRouter-specific rejection proof, align docs, include real OpenRouter evidence, and be chosen against the overlapping implementation path.
