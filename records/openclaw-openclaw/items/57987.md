## Summary

Keep open. The PR addresses a real current-main OpenRouter cache gap, but it still lacks contributor real-behavior proof, has an actionable document-block cacheability issue, and overlaps the open canonical bug/fix discussion at https://github.com/openclaw/openclaw/issues/63034 and https://github.com/openclaw/openclaw/pull/63062.

## What This Changes

The branch changes shared Anthropic/OpenRouter payload shaping to mark only the last system/developer message plus the trailing user turn with ephemeral cache_control markers, updates focused tests, and adds a changelog entry.

## Best Possible Solution

Land one canonical OpenRouter Anthropic cache-marker policy that caps explicit breakpoints, handles upstream-supported cacheable trailing user blocks, updates public prompt-caching/OpenRouter docs, and includes redacted real OpenRouter proof before superseding overlapping work.

## Reproduction Assessment

Yes, source inspection gives a high-confidence current-main reproduction path: an OpenRouter Anthropic payload with system plus user messages reaches the helper, which tags the system/developer content and leaves the trailing user content unchanged. The current OpenRouter tests assert that old behavior.

## Solution Assessment

No, not as-is. The direction is plausible, but the PR omits cacheable document blocks despite upstream support, leaves public docs stale, lacks real OpenRouter proof, and needs maintainer selection against the overlapping fix PR.
