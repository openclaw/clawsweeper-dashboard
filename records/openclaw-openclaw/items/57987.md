## Summary

Keep open. Current main still has the OpenRouter conversation-history cache gap, but this PR needs real OpenRouter proof, document-block handling, docs alignment, and maintainer choice against the overlapping implementation path before merge.

## What This Changes

The PR adds trailing user-turn `cache_control` markers for OpenRouter Anthropic requests, limits system/developer marking to the last such message, updates focused tests, and adds a changelog note.

## Best Possible Solution

Land one canonical OpenRouter Anthropic cache-marker policy that caps explicit breakpoints, supports upstream-cacheable trailing user blocks including documents, updates public docs, and includes redacted real OpenRouter proof.

## Reproduction Assessment

Yes. A high-confidence source-level path is an OpenRouter Anthropic payload with system plus user messages: current main tags system/developer content and leaves trailing user content unchanged, with focused tests asserting that behavior.

## Solution Assessment

No, not as-is. The direction is plausible, but the patch should handle document blocks or provide OpenRouter-specific rejection proof, align docs, add real OpenRouter evidence, and be chosen against the overlapping implementation path.
