## Summary

Keep open. Current main still lacks OpenRouter conversation-history cache markers, but this PR needs changes before merge: the new cacheable-block guard skips document blocks that upstream Anthropic documents as cacheable, public docs would be stale, and the external PR still lacks real behavior proof.

## What This Changes

The PR changes Anthropic/OpenRouter payload shaping to mark only the last system/developer message plus the trailing user turn, adds regression tests, and adds a changelog entry.

## Best Possible Solution

Land one canonical OpenRouter Anthropic cache-marker policy that caps explicit breakpoints, marks all upstream-supported cacheable trailing user blocks, updates prompt-caching/OpenRouter docs, and includes real OpenRouter proof before superseding the overlapping items.

## Reproduction Assessment

Yes. Current-main source and tests give a high-confidence path: `createOpenRouterSystemCacheWrapper` applies the helper, the helper only marks system/developer content, and current tests assert the adjacent user message remains unmarked.

## Solution Assessment

No, not as-is. The overall direction is plausible, but the new type guard excludes document blocks despite upstream cache support, and the PR still needs docs alignment, contributor real behavior proof, and maintainer selection among overlapping strategies. ([docs.anthropic.com](https://docs.anthropic.com/en/docs/build-with-claude/prompt-caching?amp=&e45d281a_page=1&wtime=2418s))
