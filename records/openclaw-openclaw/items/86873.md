The branch adds a 30s per-chunk idle timeout around LINE inbound media downloads and regression tests for stalled headers, stalled body reads, partial stalls, iterator cleanup, and slow-progress transfers.

## Best Possible Solution

Land the PR after maintainers accept the fixed 30s LINE media idle policy, or ask for a specific configurable or overall-deadline timeout policy before merge.
