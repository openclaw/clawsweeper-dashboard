## Summary

## What This Changes

Adds a 30 second headers/per-chunk idle timeout around LINE inbound media downloads, plus tests for stalled headers, stalled chunks, iterator cleanup, and slow-progress streams.

## Best Possible Solution

Land the focused LINE timeout if maintainers accept the fixed idle cutoff; request a configurable cutoff or overall deadline only if they want a different delivery policy.

## Reproduction Assessment

Yes. Source inspection shows current main awaits LINE media before processMessage with no deadline, and the contributor's exact-head Node 22 harness exercises controlled stalled headers/body cases through the real handler path.

## Solution Assessment

Yes, with maintainer policy acceptance. A channel-local idle timeout is a narrow mitigation that matches the Telegram-style ceiling; direct LINE fetch with AbortSignal would cancel more completely but would be a larger SDK bypass.
