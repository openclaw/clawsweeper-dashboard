## Best Possible Solution

Land the focused LINE timeout if maintainers accept the fixed 30 second idle cutoff and non-abortable SDK tradeoff, leaving configurability or an overall deadline for follow-up only if real LINE traffic needs it.

## Reproduction Assessment

## Solution Assessment

Yes, with a maintainer policy decision: a channel-local idle timeout is a narrow fix for the existing hang and follows the Telegram-style ceiling. The safer alternative, if maintainers dislike fixed fail-open attachment behavior, is to request a configurable cutoff or overall deadline before merge.
