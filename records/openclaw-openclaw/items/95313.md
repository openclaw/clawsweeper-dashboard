## Summary

Keep open: current main still rejects name-keyed Slack read targets by channel ID, and this PR's latest head repairs the previous untrusted context shortcut. Merge still needs maintainer acceptance of the Slack authorization-boundary change and the unrelated failing tooling check rerun or cleared.

## What This Changes

This PR changes Slack read-like action authorization to resolve Slack channel names with `conversations.info` for channel-ID targets when `dangerouslyAllowNameMatching` is enabled, and adds focused regression coverage.

## Best Possible Solution

Land the Slack-owned API-lookup fix after maintainer security-boundary acceptance and green or cleared required checks, while keeping stable channel IDs as the preferred config shape.

## Reproduction Assessment

## Solution Assessment

Yes. The Slack plugin is the right owner boundary, and resolving the target name through Slack's API before rechecking the existing allowlist is narrower than trusting generic context fields or changing core channel contracts.
