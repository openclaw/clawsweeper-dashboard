## Summary

Keep open: this is a real Slack name-allowlist regression fix, but the patch still feeds the provider value into a read-authorization channel-name shortcut and can suppress the intended lazy Slack name lookup.

## What This Changes

The branch updates Slack read-like action authorization to resolve channel names for channel-ID targets, threads an optional current channel name through Slack context, and adds Slack regression tests.

## Best Possible Solution

Remove the untrusted context-name shortcut and rely on the Slack-owned conversations.info fallback for channel-ID reads, or source any shortcut from a trusted conversation label with focused core-built-context coverage.

## Reproduction Assessment

Yes, source-level: current main and v2026.6.8 check Slack read targets by ID only, while PR head can still fail the core-built same-context path because Channel is the provider id and lazy lookup is skipped when any channelName is present.

## Solution Assessment

No. The lazy conversations.info fallback is the right Slack-owned repair path, but the context shortcut is sourced from the wrong field and should be removed or backed by a trusted conversation label before merge.
