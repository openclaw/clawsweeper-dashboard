## What This Changes

The PR changes Slack read-like actions to resolve a Slack channel name from `conversations.info` when a read target is only a channel ID and the account opted into name matching.

## Best Possible Solution

## Reproduction Assessment

Yes. Current main and `v2026.6.8` resolve Slack read-target authorization by channel ID only, so a name-only allowlist entry is source-reproducibly rejected before the PR's API name lookup can occur.

## Solution Assessment

Yes, pending maintainer acceptance of the boundary. The fix stays inside the Slack plugin, uses the existing conversation-info helper, avoids trusting generic context fields, and fails closed when the Slack API name is absent or not allowed.
