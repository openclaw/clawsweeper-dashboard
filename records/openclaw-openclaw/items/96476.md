## What This Changes

The PR updates `extractLinksFromMessage` markdown-link stripping to tolerate `]` characters inside the label and adds a regression test for bracketed markdown labels.

## Best Possible Solution

Land the narrow detector regex and regression test after normal maintainer and CI handling so markdown citations remain suppressed before link-understanding fetches URLs.

## Reproduction Assessment

## Solution Assessment

Yes: fixing the markdown-stripping boundary inside `extractLinksFromMessage` is the narrowest maintainable fix for the existing contract that markdown links are ignored. Replacing the whole detector with a full markdown parser would be broader than this bug requires.
