## What This Changes

The branch changes heartbeat wake classification so background-task completion wakes run on the owning session despite heartbeat isolation, and changes subagent announce prompts to mark required progress-only completions as blocked.

## Best Possible Solution

## Reproduction Assessment

Yes at source level: current main queues background-task events on the owner session but does not classify those wakes as same-queue wake payloads, while isolated heartbeat runs use a separate :heartbeat session. I did not run a live Telegram/gateway reproduction in this read-only review.

## Solution Assessment

Yes, this looks like the narrow maintainable fix shape: it reuses existing task wake sources and the existing required-completion terminal classifier instead of adding a parallel route. The remaining blocker is real transport proof, not an obvious code defect.
