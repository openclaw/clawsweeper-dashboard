## Summary

Keep open. Current main still has provider preset and auth-patch paths that can replace an explicit `agents.defaults.model.primary`, and the linked closing PR #75748 is open but unmerged.

## What This Changes

Provider auto-onboarding and plugin preset helpers should register provider models without overwriting an explicit user default model selection.

## Best Possible Solution

Review and land #75748 or an equivalent narrow fix that makes provider setup additive by default while preserving explicit replacement through `models set` and `models auth login --set-default`.

## Reproduction Assessment

Yes. A high-confidence helper-level reproduction is to call `applyProviderConfigWithDefaultModelPreset`, `applyMoonshotConfig`, or `applyOpenAIConfig` with an existing `agents.defaults.model.primary` and observe the current helper replace it.

## Solution Assessment

Yes. Preserving existing defaults in provider preset/auth config application while leaving explicit set-default commands as the replacement path is the narrowest maintainable direction.
