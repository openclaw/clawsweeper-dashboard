## Summary

Keep open. Current main still has SDK/provider preset paths that replace an existing `agents.defaults.model.primary`; only the configure/provider-auth patch path appears partially protected, and the linked closing PR #75748 remains open and unmerged.

## What This Changes

Preserve user-set `agents.defaults.model.primary` and fallbacks when auto-onboarded provider plugin presets register their default models.

## Best Possible Solution

Land #75748 or an equivalent narrow fix that makes provider preset setup additive by default while keeping explicit `models set` and `models auth login --set-default` as the consent paths for replacement.

## Reproduction Assessment

Yes. A high-confidence helper-level reproduction is to call `applyProviderConfigWithDefaultModelPreset`, `applyMoonshotConfig`, or `applyOpenAIConfig` with an existing `agents.defaults.model.primary` and observe the current helper replace it.

## Solution Assessment

Yes. Preserving existing defaults in provider preset/auth setup while leaving explicit default-setting commands authoritative is the narrowest maintainable direction.
