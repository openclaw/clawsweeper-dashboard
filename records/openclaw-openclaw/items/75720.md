## Summary

Keep open. Current main still has provider preset paths that can replace an explicit `agents.defaults.model.primary`, and the linked closing PR #75748 is open and unmerged.

## What This Changes

Preserve user-set default model selection when auto-onboarded provider plugins register their own preset models.

## Best Possible Solution

Provider preset and auth config application should add provider models without replacing an existing primary/fallback selection unless the user runs an explicit default-setting path.

## Reproduction Assessment

Yes. A high-confidence helper-level reproduction is to call `applyProviderConfigWithDefaultModelPreset`, `applyMoonshotConfig`, or `applyOpenAIConfig` with an existing `agents.defaults.model.primary` and observe current main replace it.

## Solution Assessment

Yes. The narrowest maintainable direction is the one described by the linked PR: make provider setup additive by default while preserving explicit replacement through `models set` and `models auth login --set-default`.
