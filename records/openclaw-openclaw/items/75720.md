## Summary

Keep open. Current main still has provider preset/auth paths that can overwrite an explicit `agents.defaults.model.primary`; the allowlist merge and some configure flows are guarded, but the SDK preset helper and direct provider auth config patch path are not protected.

## What This Changes

The issue asks OpenClaw provider auto-onboarding and preset auth paths to preserve user-set model defaults unless the user explicitly chooses to change the default model.

## Best Possible Solution

Provider-owned setup should be additive by default: register provider models and aliases, preserve unrelated allowlist entries, and only fill `agents.defaults.model.primary` or fallback fields when they are absent. Explicit user actions such as `openclaw models set` and `models auth login --set-default` should remain the intentional replacement path.

## Reproduction Assessment

Yes. A high-confidence helper-level reproduction is to call `applyProviderConfigWithDefaultModelPreset`, `applyMoonshotConfig`, or `applyOpenAIConfig` with an existing `agents.defaults.model.primary`; current main routes through `applyAgentDefaultModelPrimary` and replaces that value.

## Solution Assessment

No current complete solution is present. The narrow maintainable fix is to add preserve-existing-primary/fallback semantics to provider preset/auth config application while keeping explicit set-default commands as the override path.
