## Summary

Keep open. Current main still contains provider preset and auth config paths that can overwrite an existing `agents.defaults.model.primary`, and this issue is already paired with open fix PR #75748, so it should stay open until that PR or an equivalent fix lands.

## What This Changes

Provider auto-onboarding and preset/auth setup should add provider models without silently replacing an explicitly configured agent default primary model or fallback chain.

## Best Possible Solution

Provider-owned setup should be additive by default: register provider models and aliases, preserve unrelated allowlist entries and existing primary/fallback selections, and reserve replacement for explicit commands such as `openclaw models set` or `openclaw models auth login --set-default`. Review and land #75748 or an equivalent shared-helper fix before closing this issue.

## Reproduction Assessment

Yes. A high-confidence helper-level reproduction is to call `applyProviderConfigWithDefaultModelPreset`, `applyMoonshotConfig`, or `applyOpenAIConfig` with an existing `agents.defaults.model.primary`; current main routes through `applyAgentDefaultModelPrimary` and replaces the value.

## Solution Assessment

Yes, the intended solution direction is the narrow maintainable one: preserve existing default model selection in shared provider preset/auth config helpers while keeping explicit set-default paths as replacements. This issue should not close until #75748 or an equivalent fix is merged.
