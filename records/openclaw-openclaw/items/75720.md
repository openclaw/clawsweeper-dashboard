## Summary

Keep open. Current main still has provider preset paths that can replace an existing default primary, and the linked closing PR is open and unmerged.

## What This Changes

The issue asks provider auto-onboard and preset setup to add provider models without replacing a user-set `agents.defaults.model.primary` or dropping custom model entries.

## Best Possible Solution

Review and land PR #75748 or an equivalent narrow fix that makes provider preset setup additive by default while leaving explicit default-setting commands authoritative.

## Reproduction Assessment

Yes. A helper-level reproduction is clear from source: pass a config with an existing `agents.defaults.model.primary` into `applyProviderConfigWithDefaultModelPreset`, `applyMoonshotConfig`, or `applyOpenAIConfig`, and current main routes to `applyAgentDefaultModelPrimary`.

## Solution Assessment

Yes. Preserving existing primary/fallbacks during provider preset/auth setup while keeping `openclaw models set` and `models auth login --set-default` as explicit replacement paths is the narrow maintainable fix.
