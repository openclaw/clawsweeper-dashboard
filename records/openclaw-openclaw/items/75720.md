## Summary

Keep open for https://github.com/openclaw/openclaw/issues/75720. Current main still has provider preset paths that replace an existing default primary, and the linked closing PR #75748 is open and unmerged.

## What This Changes

The issue asks OpenClaw provider auto-onboarding and plugin presets to register provider models additively without overwriting a user-set `agents.defaults.model.primary` or fallbacks.

## Best Possible Solution

Land #75748 or an equivalent narrow fix that makes provider preset setup additive by default while keeping `openclaw models set` and `models auth login --set-default` as explicit replacement paths.

## Reproduction Assessment

Yes. A high-confidence helper-level reproduction is to call `applyProviderConfigWithDefaultModelPreset`, `applyMoonshotConfig`, or `applyOpenAIConfig` with an existing `agents.defaults.model.primary` and observe current main replace it.

## Solution Assessment

Yes. Preserving existing defaults in provider preset/auth setup while leaving explicit default-setting commands authoritative is the narrowest maintainable direction.
