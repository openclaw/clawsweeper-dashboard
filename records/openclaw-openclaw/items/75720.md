## Summary

Keep open. Current `main` still contains provider preset/auth paths that can replace an explicit default model selection, and the issue is now paired with an open draft PR that specifically targets this bug, so cleanup should wait for that PR to merge, be replaced, or be closed.

## What This Changes

The issue reports that provider auto-onboarding and plugin preset application can overwrite a user-set `agents.defaults.model.primary` while registering new provider defaults.

## Best Possible Solution

Land #75748 or an equivalent narrow fix that makes provider setup additive by default: add/register provider models and aliases, preserve existing primary and non-empty fallback selections, and keep explicit `openclaw models set` or `models auth login --set-default` as the intentional replacement paths.

## Reproduction Assessment

Yes. A high-confidence helper-level reproduction is to call `applyProviderConfigWithDefaultModelPreset`, `applyMoonshotConfig`, or `applyOpenAIConfig` with an existing `agents.defaults.model.primary`; current `main` routes through `applyAgentDefaultModelPrimary` and replaces that value.

## Solution Assessment

Yes, the proposed direction is the narrow maintainable solution: preserve existing defaults in provider preset/auth config application while leaving explicit user commands as override paths. The linked PR should be reviewed as the implementation candidate before any separate repair lane is opened.
