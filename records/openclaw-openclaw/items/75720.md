## Summary

Keep open. Current main still contains provider preset paths that can overwrite an existing default primary, and the linked closing PR #75748 is open and unmerged.

## What This Changes

Provider auto-onboarding and preset application should add provider models without silently replacing user-configured `agents.defaults.model.primary` or fallbacks.

## Best Possible Solution

Land #75748 or an equivalent narrow fix that makes provider preset setup additive by default while keeping `openclaw models set` and `models auth login --set-default` as explicit replacement paths.

## Reproduction Assessment

Yes. A high-confidence helper-level reproduction is to call `applyProviderConfigWithDefaultModelPreset`, `applyMoonshotConfig`, or `applyOpenAIConfig` with an existing `agents.defaults.model.primary` and observe current main replace it.

## Solution Assessment

Yes. Preserving existing defaults in provider preset/auth setup while leaving explicit default-setting commands authoritative is the narrowest maintainable solution.
