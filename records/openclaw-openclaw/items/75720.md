## Summary

Keep open. Current main still has provider preset paths that replace an existing default primary, and the linked closing PR #75748 remains open and unmerged.

## What This Changes

The issue reports that auto-onboarded provider plugin presets can silently replace a user-configured `agents.defaults.model.primary` during provider registration.

## Best Possible Solution

Land #75748 or an equivalent narrow fix that makes provider preset/auth setup additive unless the user explicitly invokes `openclaw models set` or `models auth login --set-default`.

## Reproduction Assessment

Yes. A high-confidence helper-level reproduction is to call `applyProviderConfigWithDefaultModelPreset`, `applyMoonshotConfig`, or `applyOpenAIConfig` with an existing `agents.defaults.model.primary` and observe current main replace it.

## Solution Assessment

Yes. Preserving existing primary/fallbacks in provider preset/auth setup while leaving explicit replacement commands authoritative is the narrowest maintainable direction.
