## Summary

Keep open. Current main still has provider preset paths that replace an existing default primary, and PR #75748 is open and unmerged with closing syntax for this work.

## What This Changes

Preserve user-set `agents.defaults.model.primary` and fallbacks when provider auto-onboarding or plugin presets register new default models.

## Best Possible Solution

Review and land PR #75748 or an equivalent narrow fix that makes provider preset setup additive unless an explicit default-setting command is used.

## Reproduction Assessment

Yes. A source-level reproduction is clear: pass a config with an existing `agents.defaults.model.primary` into `applyProviderConfigWithDefaultModelPreset`, `applyMoonshotConfig`, or `applyOpenAIConfig`, and current main routes to `applyAgentDefaultModelPrimary`.

## Solution Assessment

Yes. Preserving existing primary/fallbacks during provider preset setup while keeping `openclaw models set` and `models auth login --set-default` as explicit replacement paths is the narrow maintainable fix.
