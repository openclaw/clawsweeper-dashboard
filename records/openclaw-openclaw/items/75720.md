## Summary

Keep open. Current main still has provider preset paths that can replace an existing `agents.defaults.model.primary`, and the linked closing PR #75748 is still an open implementation candidate rather than a merged fix.

## What This Changes

The issue requests additive provider preset onboarding so user-set `agents.defaults.model.primary` and related model selections are not replaced unless the user explicitly chooses a new default.

## Best Possible Solution

Review and land #75748 or an equivalent narrow fix that makes provider preset setup additive by default while keeping `models set` and `models auth login --set-default` as the explicit replacement paths.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to call `applyProviderConfigWithDefaultModelPreset`, `applyMoonshotConfig`, or `applyOpenAIConfig` with an existing `agents.defaults.model.primary` and observe that current main replaces it.

## Solution Assessment

Yes. Preserving existing defaults in provider preset/auth setup while leaving explicit default-setting commands authoritative is the narrowest maintainable direction for this bug.
