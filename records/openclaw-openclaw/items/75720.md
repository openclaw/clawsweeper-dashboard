## Summary

Keep open. Current main still has provider preset paths that can replace an explicit `agents.defaults.model.primary`; the auth/configure path has partial preservation coverage, but the Moonshot/OpenAI preset-onboard surface described here is not fully fixed and #75748 is still the active linked fix PR.

## What This Changes

The issue reports that provider auto-onboarding and preset appliers silently overwrite user-set default model selection while registering newly available provider plugins.

## Best Possible Solution

Review and land #75748, or an equivalent narrow fix, so provider setup adds models additively while explicit replacement remains limited to `openclaw models set` and `models auth login --set-default`.

## Reproduction Assessment

Yes. A high-confidence helper-level reproduction is to call `applyProviderConfigWithDefaultModelPreset`, `applyMoonshotConfig`, or `applyOpenAIConfig` with an existing `agents.defaults.model.primary` and observe current main replace it.

## Solution Assessment

Yes. Preserving existing primary/fallbacks in provider preset/auth setup while keeping explicit set-default commands as the replacement path is the narrowest maintainable solution.
