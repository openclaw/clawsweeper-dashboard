## Summary

Keep open. Current main still has provider preset paths that can replace an existing default primary, and the linked closing PR is open and unmerged in the provided GitHub context.

## What This Changes

Provider preset/onboard flows should register provider models additively without overwriting user-set `agents.defaults.model.primary` or fallbacks.

## Best Possible Solution

Review and land #75748, or an equivalent narrow fix that preserves existing default model selection during provider preset/auth setup while keeping explicit default-setting commands authoritative.

## Reproduction Assessment

Yes. A source-level reproduction is clear: pass a config with an existing `agents.defaults.model.primary` through `applyProviderConfigWithDefaultModelPreset`, `applyMoonshotConfig`, or `applyOpenAIConfig`, and current main routes to the primary overwrite helper.

## Solution Assessment

Yes. Additive provider setup with explicit `--set-default` or `models set` as the replacement path is the narrow maintainable fix, and #75748 is already scoped that way.
