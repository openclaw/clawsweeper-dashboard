## Summary

Keep open. Current main still has the provider preset and OpenAI/Moonshot setup paths that can replace an existing `agents.defaults.model.primary`, and the linked closing PR remains open rather than merged into `main`.

## What This Changes

The issue asks OpenClaw provider/plugin auto-onboarding to register new provider models additively without overwriting a user-set default primary or fallback chain.

## Best Possible Solution

Land #75748 or an equivalent narrow fix that makes provider preset setup additive by default while keeping explicit `models set` and `models auth login --set-default` as the consent paths for replacing defaults.

## Reproduction Assessment

Yes. A high-confidence helper-level reproduction is to call `applyProviderConfigWithDefaultModelPreset`, `applyMoonshotConfig`, or `applyOpenAIConfig` with an existing `agents.defaults.model.primary` and observe current main replace it.

## Solution Assessment

Yes. Preserving existing defaults in provider preset/auth setup while leaving explicit default-setting commands authoritative is the narrowest maintainable direction.
