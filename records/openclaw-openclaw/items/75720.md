## Summary

Keep open. Current main still has provider preset paths that can replace an explicit default primary, while only some interactive auth-choice paths restore the prior primary; the attached closing PR is still the right active fix path.

## What This Changes

Auto-onboarded provider presets should register provider models additively without replacing explicit model defaults or pruning custom provider allowlist entries.

## Best Possible Solution

Land PR #75748 or an equivalent narrow fix that makes provider setup additive by default while preserving explicit replacement through `models set` and `models auth login --set-default`.

## Reproduction Assessment

Yes. A high-confidence helper-level reproduction is to call `applyProviderConfigWithDefaultModelPreset`, `applyOpenAIConfig`, or `applyMoonshotConfig` with an existing `agents.defaults.model.primary` and observe the returned config replace it.

## Solution Assessment

Yes. Preserving existing defaults in provider preset and auth setup while leaving explicit default-setting commands as the replacement path is the narrowest maintainable direction.
