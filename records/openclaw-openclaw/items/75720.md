## Summary

Keep open. Current `origin/main` still contains the source-level overwrite path, and the issue is already paired with an open closing PR, so it should remain open until that PR or an equivalent fix lands.

## What This Changes

Provider auto-onboarding and provider auth presets should add provider models without silently replacing an explicitly configured `agents.defaults.model.primary` or fallback selection.

## Best Possible Solution

Review and land PR #75748, or an equivalent narrow fix, so provider setup remains additive by default while explicit commands such as `openclaw models set` and `openclaw models auth login --set-default` stay as the intentional replacement paths.

## Reproduction Assessment

Yes. A high-confidence reproduction is to call `applyProviderConfigWithDefaultModelPreset`, `applyMoonshotConfig`, or `applyOpenAIConfig` on a config that already has `agents.defaults.model.primary`; current `origin/main` routes through `applyAgentDefaultModelPrimary` and replaces it.

## Solution Assessment

Yes. Preserving existing primary/fallback selection during provider preset/auth application is the narrow maintainable direction, while explicit set-default command paths should continue to replace the default model.
