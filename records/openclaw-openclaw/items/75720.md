## Summary

Keep open. Current main still contains provider preset paths that replace an existing `agents.defaults.model.primary`, and the closing PR linked from the issue is still only an open follow-up in the provided GitHub context.

## What This Changes

The issue asks provider auto-onboarding and plugin presets to register new provider models additively without replacing user-set default primary/fallback model choices or dropping custom model entries.

## Best Possible Solution

Land PR #75748 or an equivalent narrow fix that makes provider preset setup additive by default while preserving explicit replacement through `openclaw models set` and `models auth login --set-default`.

## Reproduction Assessment

Yes. A helper-level reproduction is to call `applyProviderConfigWithDefaultModelPreset`, `applyMoonshotConfig`, or `applyOpenAIConfig` with an existing `agents.defaults.model.primary` and observe current main replace it.

## Solution Assessment

Yes. Preserving existing defaults during provider preset/auth setup, while keeping explicit default-setting commands authoritative, is the narrowest maintainable fix direction.
