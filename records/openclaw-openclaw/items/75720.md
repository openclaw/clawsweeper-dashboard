## Summary

Keep open because current main still has provider preset paths that can replace an existing default primary, and the linked closing PR remains open and conflicting.

## What This Changes

Auto-onboarded provider plugin presets should add their models without overwriting an existing user-set `agents.defaults.model.primary` or fallbacks.

## Best Possible Solution

Repair/rebase and land https://github.com/openclaw/openclaw/pull/75748, or an equivalent narrow fix, so provider presets add models without changing defaults unless the user takes an explicit default-setting path.

## Reproduction Assessment

Yes, at source level. Passing a config with an existing `agents.defaults.model.primary` through `applyProviderConfigWithDefaultModelPreset`, `applyOpenAIConfig`, or `applyMoonshotConfig` still reaches `applyAgentDefaultModelPrimary`; the reporter also confirmed the production update path.

## Solution Assessment

Yes, the best direction is a narrow additive-default fix that preserves existing primary/fallbacks during non-explicit provider setup while leaving `openclaw models set` and `models auth login --set-default` authoritative.
