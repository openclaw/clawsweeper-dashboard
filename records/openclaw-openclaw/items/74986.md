Keep open. ## What This Changes

The issue reports `openclaw infer` and `agents list` commands hanging while the spawned `openclaw-infer` process spins at 100% CPU before any gateway or provider network I/O.

## Best Possible Solution

Land or revise PR #75022 for a metadata-only infer catalog inspection path, then retest `infer model list`, `infer model inspect`, `infer model providers`, local and remote `infer model run`, and `agents list` on the affected setup. Any remaining model-run or all-agent spin should be split into a narrower follow-up with stack/profile evidence.
