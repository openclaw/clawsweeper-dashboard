## Summary

Keep open. Current main has mitigations for `agents list` and local `infer model run`, but `infer model list`, `infer model inspect`, and `infer model providers` still load the full model catalog and unconditionally enter provider-plugin augmentation before network I/O. Open PR #75022 is the active implementation candidate for that remaining catalog-inspection slice, so the issue should stay open until that PR lands or is replaced and the reported command matrix is retested.

## What This Changes

The issue reports `openclaw infer` and `agents list` commands hanging on v2026.4.27 with a pegged `openclaw-infer` child before any gateway or provider network traffic.

## Best Possible Solution

Review and land or revise PR #75022 for the metadata-only catalog inspection path, then retest `infer model list`, `infer model inspect`, `infer model providers`, local and gateway `infer model run`, and `agents list` on the affected WSL2/Node setup. Any remaining gateway-run CPU spin should be split into a focused follow-up with stack/profile evidence.
