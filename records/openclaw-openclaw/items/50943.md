## What This Changes

The PR changes Feishu session-thread parsing, agent run context, ACP/subagent spawn routing, Feishu reply targeting, and regression tests, with unrelated formatting churn in UI, Telegram, Discord, sandbox, and infra files.

## Best Possible Solution

Rebase into a narrower Feishu/agent patch that uses current main's plugin-owned session-conversation and agent-aware delivery-info contracts, binds Feishu ACP sessions to resolvable conversation ids, and includes live Feishu proof.

## Reproduction Assessment

No high-confidence live reproduction is present. Source inspection shows current main does not recover currentThreadTs from a Feishu sender-scoped session key, but the flow has not been exercised in a real Feishu workspace.

## Solution Assessment

No, not as-is. The direction is plausible, but the branch should be rebased through current main's session-store contracts and fixed so named-agent delivery lookup and Feishu ACP binding use resolvable routing keys before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Load delivery info from the session's agent store:** `src/agents/command/run-context.ts:14`
  - body: The new sessionKey fallback calls extractDeliveryInfo(trimmedSessionKey), but on this branch that helper only opens the default session store. Named-agent sessions can restore currentThreadTs without restoring the matching channel, so same-turn threaded sends still fall back to the room root; resolve delivery info through the session key's agent-specific store before relying on this fallback.
  - confidence: 0.84
- **[P2] Bind Feishu ACP sessions to a resolvable conversation:** `src/agents/acp-spawn.ts:120`
  - body: For sender-scoped Feishu keys this recovers only the topic root id, and the branch's test asserts a binding conversationId like om_x100abc123. Feishu inbound routing matches runtime bindings by the full live conversation id, so follow-up topic messages will not reach the accepted ACP session; use the Feishu conversation-id contract or avoid creating that binding without an exact conversation.
  - confidence: 0.78
