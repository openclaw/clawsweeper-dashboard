## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Preserve valid Telegram DM thread sessions:** `src/config/sessions/delivery-info.ts:32-35`
  - body: `resolveSessionThreadIdForRouting()` suppresses every `agent:main:telegram:dm:*:thread:*` key, but current coverage expects legacy Telegram DM thread keys such as `agent:main:telegram:dm:123456789:thread:123456789:99` to infer threadId `99`. The run-context, ACP, and subagent paths that use this helper would fall back to the DM root.
  - confidence: 0.87
- **[P2] Use the requester agent store for restored delivery context:** `src/agents/command/run-context.ts:13-14`
  - body: The new run-context fallback calls `extractDeliveryInfo(trimmedSessionKey)`, but the branch implementation opens only the default session store. Named-agent sessions can still miss `currentChannelId` and same-turn threaded delivery when resumed without `to`.
  - confidence: 0.8
- **[P2] Normalize restored channel targets before auto-threading:** `src/agents/command/run-context.ts:60-62`
  - body: This copies stored `deliveryContext.to` directly into `currentChannelId`. Slack stores prefixed targets such as `slack:C...` or `channel:C...`, while auto-threading compares the parsed bare channel id, so resumed Slack thread sessions can still miss auto-threading.
  - confidence: 0.78
