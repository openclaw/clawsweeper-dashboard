Full review comments:

- **[P2] Preserve legacy Telegram DM thread ids:** `src/config/sessions/delivery-info.ts:34-36`
  - body: `resolveSessionThreadIdForRouting()` drops every Telegram DM `:thread:` suffix. Current main still preserves keys like `agent:main:telegram:dm:123456789:thread:123456789:99`, so run-context, ACP, and native subagent routing from those sessions would fall back to the DM root.
  - confidence: 0.86
- **[P2] Use the requester agent store for inferred delivery context:** `src/agents/command/run-context.ts:12-13`
  - body: The new fallback calls `extractDeliveryInfo(trimmedSessionKey)` without selecting the agent store for that session key. Named-agent sessions resumed without `to` will read the default store, leaving `currentChannelId` empty and still missing same-turn threaded delivery context.
  - confidence: 0.8
- **[P2] Normalize restored channel ids before reusing them:** `src/agents/command/run-context.ts:57-59`
  - body: This copies stored `deliveryContext.to` directly into `currentChannelId`. Slack stores values such as `slack:C...` or `channel:C...`, while Slack auto-threading compares against the bare channel id, so resumed Slack thread sessions can still miss auto-threading.
  - confidence: 0.78
- **[P2] Keep failed Feishu explicit replies inside the topic:** `extensions/feishu/src/outbound.ts:52-56`
  - body: When an explicit `replyToId` is present, `resolveReplyContext()` forces `replyInThread: false` even if a topic `threadId` is also available. If Feishu rejects that quoted target as withdrawn or missing, the fallback posts at the chat root instead of failing closed inside the topic.
  - confidence: 0.76
