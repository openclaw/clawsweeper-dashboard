Keep open for human/contributor follow-up; the branch is a plausible Feishu routing bug-fix attempt, but it conflicts with current main's sender-scoped Feishu conversation contract, lacks real Feishu/Lark behavior proof, and is currently dirty/conflicting.

## What This Changes

The branch changes Feishu session-key/thread parsing, ACP/subagent/run-context routing, and Feishu outbound reply handling, with regression tests plus unrelated formatting churn in UI, Discord, Telegram, sandbox, and infra files.

## Best Possible Solution

Rebase narrowly onto current main, preserve Feishu conversation identity through plugin-owned conversation/session-binding APIs, keep sender-scoped ACP bindings exact, drop unrelated churn, and add real Feishu/Lark proof before merge.

## Reproduction Assessment

No live reproduction is present for this PR. Source inspection gives a high-confidence static reproduction of the PR regression: the branch collapses a sender-scoped Feishu topic session into a topic-only conversation id while current main requires exact sender-scoped binding matches.

## Solution Assessment

No, not as-is. The maintainable fix should build on current main's plugin-owned Feishu conversation and binding contracts instead of slicing Feishu `:topic:` session keys generically.
