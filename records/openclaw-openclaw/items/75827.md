## Summary

This issue remains valid on current main: the Docker publishing workflow still has no main-branch push trigger, the install docs still advertise the floating `main` image tag, and live GHCR metadata still shows `ghcr.io/openclaw/openclaw:main` pointing at the March 26 image revision. Because changing mutable image publishing is release/supply-chain policy, it should stay open for maintainer handling.

## What This Changes

The issue asks OpenClaw to either rebuild `ghcr.io/openclaw/openclaw:main` on every push to git `main` or stop presenting Docker `:main` as current main.

## Best Possible Solution

Define the Docker image channel contract and align workflow, docs, and maintainer guidance around it: either guarded `main`/`main-slim` publishing from main pushes, or explicit deprecation of Docker `:main` as a current-main test path.

## Reproduction Assessment

Yes. Current-main source inspection shows the Docker release workflow cannot run on ordinary main-branch pushes, and live GHCR inspection confirms the `:main` image is still a March 26 artifact rather than current main.

## Solution Assessment

Unclear because no patch is proposed and both plausible fixes carry release-policy implications. Re-enabling main-push publishing is the direct behavioral fix, while deprecating the Docker `:main` channel may be safer if releases are meant to be the only published image artifacts.
