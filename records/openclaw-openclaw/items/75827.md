## Summary

Keep open. Current main still lacks a branch-main Docker release trigger, the Docker docs still list `main` as a common pre-built tag, and a live GHCR manifest check still shows `:main` pointing at the March image; the prior ClawSweeper comment correctly kept this for release/Docker channel follow-up.

## What This Changes

Ensure the GHCR Docker `:main` tag either tracks git `main` pushes or is no longer presented as a current-main testing path.

## Best Possible Solution

Define one Docker image channel contract and align workflow, docs, and maintainer guidance around it: either guarded `main`/`main-slim` publishing from branch pushes with attestations, or explicit deprecation of Docker `:main` as a current-main test path.

## Reproduction Assessment

Yes. Inspecting `.github/workflows/docker-release.yml` reproduces the source-level issue, and querying GHCR for `ghcr.io/openclaw/openclaw:main` reproduces the stale-image state with a March 2026 OCI revision.

## Solution Assessment

Unclear until release maintainers choose the intended Docker channel contract. Auto-rebuilding `:main` is the direct technical fix, but documenting that Docker `:main` is not current-main may be safer if mutable branch images are not desired.
