## Summary

Keep open: the Docker `:main` image channel is still stale on current main. The workflow has no ordinary main-branch push trigger, the Docker docs still advertise `main` as a common pre-built tag, and live GHCR metadata still points `ghcr.io/openclaw/openclaw:main` at a March 26 image revision rather than the checked-out main SHA.

## What This Changes

The issue asks OpenClaw to make the GHCR `:main` Docker image track git `main`, or to stop directing users toward `:main` when only release images are current.

## Best Possible Solution

Define the Docker image channel contract and align workflow, docs, and maintainer guidance around it: guarded `main`/`main-slim` publishing from main pushes, or explicit deprecation of Docker `:main` as a current-main test path.

## Reproduction Assessment

Yes. Source inspection shows no normal main-branch trigger for Docker Release, and live GHCR inspection shows `ghcr.io/openclaw/openclaw:main` is still the March 26 image revision rather than current main.

## Solution Assessment

Unclear because no patch is proposed and both plausible fixes carry release-policy implications. Re-enabling main-push publishing is the direct behavioral fix, while deprecating Docker `:main` may be safer if release tags are intended to be the only supported published image artifacts.
