## Summary

Keep open. Current main still does not rebuild the GHCR `:main` image from main-branch pushes, the Docker docs still advertise `main` as a common pre-built tag, and live registry inspection still shows `:main` pointing to the March 26 image revision rather than current main.

## What This Changes

The issue asks OpenClaw to either keep `ghcr.io/openclaw/openclaw:main` current with git `main` or stop presenting that tag as a current-main testing path.

## Best Possible Solution

Define one Docker image channel contract and align workflow, docs, and maintainer guidance: either guarded `main`/`main-slim` publishing from main pushes with attestation checks, or explicit deprecation of Docker `:main` as a current-main test path.

## Reproduction Assessment

Yes. Source inspection shows no branch-main trigger in `.github/workflows/docker-release.yml`, and live `skopeo inspect` shows `:main` was created on 2026-03-26 at revision `4ed58956375be8b16fb3559699f1b0cb3661dbce`, far behind current main `ac607044f1ff1e911abbb4db0ff7f2151e564b70`.

## Solution Assessment

Unclear because no patch is proposed and the remaining choice is release policy. Re-enabling `:main` publishing is direct but supply-chain sensitive; deprecating the tag in docs and guidance is safer if Docker images should stay release-driven.
