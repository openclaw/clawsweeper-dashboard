## Summary

Keep open. Current main still does not rebuild the GHCR `main` Docker image from branch pushes, the Docker docs still present `main` as a common pre-built tag, and live registry inspection still shows the tag pointing to a March image rather than current main.

## What This Changes

Request that the GHCR Docker `main` tag either rebuild from each git `main` push or that docs and maintainer guidance stop treating it as current main.

## Best Possible Solution

Define the Docker image channel contract, then align workflow, docs, and maintainer guidance: either guarded `main`/`main-slim` publishing from branch pushes or explicit deprecation of Docker `main` as a current-main test path.

## Reproduction Assessment

Yes. Inspect `.github/workflows/docker-release.yml` to see the missing branch-main trigger, then run `skopeo inspect docker://ghcr.io/openclaw/openclaw:main` to confirm the live tag still points to the 2026-03-26 image revision.

## Solution Assessment

Unclear until release maintainers choose the desired channel contract. Auto-rebuilding `main` is the direct technical fix, but a docs/guidance change may be the safer product choice if OpenClaw does not want mutable Docker images from every main push.
