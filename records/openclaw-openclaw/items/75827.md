## Summary

Keep open. The report is valid against current main: the Docker release workflow no longer runs on branch pushes, the workflow still contains a `refs/heads/main` tag path that is therefore unreachable, docs still list `main` as a common pre-built image tag, and live GHCR metadata shows `:main` was last built from a March commit.

## What This Changes

The issue asks OpenClaw to either rebuild `ghcr.io/openclaw/openclaw:main` on every git `main` commit or stop directing users to that tag as a current-main test path.

## Best Possible Solution

Define one explicit Docker channel contract and make workflow plus docs match it: either restore guarded branch-build publication for `:main`/`:main-slim` and arch tags, or deprecate that Docker tag in favor of release tags, npm/git dev installs, or local source builds for current-main verification.

## Reproduction Assessment

Yes. Source inspection shows the missing branch trigger, docs still advertise the tag, and live GHCR config metadata confirms `:main` is built from a March 26 revision rather than current main.

## Solution Assessment

Unclear until maintainers choose the Docker channel policy. Auto-publishing `:main` is the direct technical fix, but a docs/guidance change may be safer if OpenClaw does not want mutable Docker images from every main push.
