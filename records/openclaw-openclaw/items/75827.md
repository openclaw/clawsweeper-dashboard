## Summary

Keep open. Current main still leaves Docker publishing tag-driven only, docs still advertise a `main` image tag, and live GHCR inspection shows `ghcr.io/openclaw/openclaw:main` is still the March 26 image rather than current git `main`.

## What This Changes

The issue asks OpenClaw to either keep the GHCR `:main` Docker image aligned with git `main` or stop using it as current-main retest guidance.

## Best Possible Solution

Define one Docker image channel contract and align workflow, docs, and maintainer guidance: either guarded `main`/`main-slim` publishing from branch pushes with attestations, or explicit deprecation of Docker `:main` as a current-main test path.

## Reproduction Assessment

Yes. Inspecting `.github/workflows/docker-release.yml` shows no branch-main trigger, and live `skopeo inspect` shows the GHCR `:main` image is revision `4ed58956375b` from 2026-03-26 while current main is `2f0c9358b138`.

## Solution Assessment

Unclear until release maintainers choose the desired channel contract. Auto-publishing `:main` is the direct technical fix, but updating docs/guidance may be safer if OpenClaw does not want mutable Docker images from every main push.
