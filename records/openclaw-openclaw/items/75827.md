## Summary

Keep open. Current main still does not auto-build the Docker `:main` image on branch pushes, docs still advertise `main` as a common pre-built image tag, and live GHCR inspection shows `:main` still points to the March 26 image revision.

## What This Changes

The issue asks OpenClaw to align the published GHCR Docker `:main` tag with git `main` or stop presenting it as current-main proof.

## Best Possible Solution

Define one Docker channel contract and align workflow, docs, and maintainer guidance: either guarded `main`/`main-slim` publishing from protected main with attestations, or explicit deprecation of Docker `:main` as a current-main testing path.

## Reproduction Assessment

Yes. Source inspection shows no branch-main trigger in `.github/workflows/docker-release.yml`, and live `skopeo inspect` shows `ghcr.io/openclaw/openclaw:main` was created on 2026-03-26 at revision `4ed58956375be8b16fb3559699f1b0cb3661dbce` while current main is `b22263720e79895e7ea742de5afbb9985f93d43d`.

## Solution Assessment

Unclear because no patch is proposed and the two viable fixes are release-policy choices. Re-enabling `:main` publishing is direct but supply-chain sensitive; deprecating the tag in docs/guidance may be safer if Docker images should remain release-driven.
