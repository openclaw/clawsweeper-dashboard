## Summary

This issue remains valid on current main: the Docker Release workflow still has no ordinary main-branch push trigger, the Docker docs still list `main` as a common pre-built image tag, and live GHCR metadata still points `ghcr.io/openclaw/openclaw:main` at the March 26 image revision rather than current `main`. The remaining fix is release-channel policy, so it should stay open for maintainer handling.

## What This Changes

The issue asks OpenClaw to make the GHCR `:main` image track git `main`, or to stop directing users toward `:main` when only release images are current.

## Best Possible Solution

Define the Docker image channel contract and align workflow, docs, and maintainer guidance around it: guarded `main`/`main-slim` publishing from main pushes, or explicit deprecation of Docker `:main` as a current-main test path.

## Reproduction Assessment

Yes. Current-main source inspection shows no normal main-branch trigger for Docker Release, and live `skopeo inspect` plus GitHub Actions history confirm `ghcr.io/openclaw/openclaw:main` is still the March 26 image revision.

## Solution Assessment

Unclear because no patch is proposed and both plausible fixes carry release-policy implications. Re-enabling main-push publishing is the direct behavioral fix, while deprecating Docker `:main` may be safer if release tags are intended to be the only supported published image artifacts.
