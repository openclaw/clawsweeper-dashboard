## Summary

Keep open. Current main still lacks a branch-push Docker Release trigger, docs still advertise the GHCR `main` tag, and live registry inspection still shows `ghcr.io/openclaw/openclaw:main` at the 2026-03-26 image revision instead of current main.

## What This Changes

Request that the GHCR `ghcr.io/openclaw/openclaw:main` Docker tag either rebuild from git `main` pushes or that docs and maintainer guidance stop treating it as current main.

## Best Possible Solution

Define one Docker image channel contract and align workflow, docs, and maintainer guidance: either guarded `main`/`main-slim` publication from branch pushes with attestation verification, or explicit deprecation of Docker `:main` as a current-main test path.

## Reproduction Assessment

Yes. The source-level reproduction is to inspect `.github/workflows/docker-release.yml` and see no `push.branches: [main]`; the live reproduction is GHCR manifest inspection, which still reports a 2026-03-26 `main` image revision.

## Solution Assessment

Unclear until release maintainers choose the channel contract. Auto-publishing `:main` is the direct technical fix, but a docs and guidance change may be safer if OpenClaw does not want mutable Docker images from every main push.
