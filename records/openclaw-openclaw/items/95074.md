## What This Changes

The PR changes the Control UI cron quick-create Silent delivery preset from main-session/no-delivery to isolated-session/no-delivery and adds a focused unit assertion.

## Best Possible Solution

Fix the quick-create mapper so Silent creates a backend-supported job while preserving the intended distinction between no notification and independent-session execution, unless maintainers explicitly redefine those presets.

## Reproduction Assessment

Yes. Source inspection shows current main maps Silent quick-create to a main-session agentTurn job, addCronJob submits that shape to cron.add, and the cron service rejects main-session jobs unless the payload is systemEvent.

## Solution Assessment

No. The patch makes the job backend-valid by changing Silent into isolated/no-delivery; the narrower maintainable fix is to keep Silent distinct as a supported main-session systemEvent job unless maintainers deliberately redefine the preset.
