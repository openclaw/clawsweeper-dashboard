## What This Changes

The PR changes the Control UI Silent cron quick-create preset from `sessionTarget: "main"` to `"isolated"` and adds a regression test for that preset mapping.

## Best Possible Solution

Preserve Silent as a backend-supported cron shape that stays distinct from Independent session, add focused mapper coverage, and include real Control UI/Gateway proof before merge.

## Reproduction Assessment

Yes. Source inspection shows current main maps Silent quick-create to a `main` `agentTurn` job, the UI sends that to `cron.add`, and the backend rejects main-session jobs unless the payload is `systemEvent`; I did not run a live browser/Gateway flow in this read-only review.

## Solution Assessment

No. The patch makes validation pass by turning Silent into isolated/no-delivery; the narrower maintainable fix is to keep Silent distinct as a supported main-session system event unless maintainers deliberately redefine the preset.
