## Summary

Keep open: this PR targets a real linked Silent cron quick-create bug, but the current diff changes Silent into an isolated agent-turn job instead of preserving the documented main-session/system-event distinction, and the external PR still has only test-output proof.

## What This Changes

The PR changes the Control UI Silent cron quick-create preset to target an isolated session and adds a node test asserting isolated/no-delivery mapping.

## Best Possible Solution

Preserve Silent as a backend-supported main-session `systemEvent` job, update the regression test for that contract, and add redacted live Control UI/Gateway proof before merge.

## Reproduction Assessment

Yes, source-level. Current main maps Silent quick-create to a main-session `agentTurn` job, `addCronJob` sends that to `cron.add`, and the backend rejects main-session cron jobs unless the payload is `systemEvent`; I did not run a live browser/Gateway flow in this read-only review.

## Solution Assessment

No. Changing `sessionTarget` to `isolated` is a plausible validation fix, but the narrower maintainable fix is to keep Silent distinct as a supported main-session `systemEvent` job unless maintainers deliberately redefine the preset.
