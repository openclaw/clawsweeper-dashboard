## Summary

Keep open: the PR fixes the backend rejection by changing Silent into an isolated agent-turn job, but current UI copy, docs, and runtime contracts distinguish Silent from Independent session, so this is not the best fix without maintainer approval to redefine the preset.

## What This Changes

The PR changes the Control UI cron quick-create Silent preset from `sessionTarget: "main"` to `sessionTarget: "isolated"` and adds a focused mapper test.

## Best Possible Solution

Fix Silent as a backend-supported distinct preset, likely by keeping `sessionTarget: "main"` and setting `payloadKind: "systemEvent"`, unless maintainers explicitly choose to redefine Silent as isolated/no-delivery.

## Reproduction Assessment

Yes, source-level: current main maps Silent quick-create to a main-session `agentTurn` job, `addCronJob` forwards that shape, and the cron service rejects main-session cron jobs unless the payload is `systemEvent`. I did not run a live browser/Gateway flow because this is a read-only review.

## Solution Assessment

No. This patch makes the job valid by changing Silent into isolated/no-delivery, but the narrower maintainable fix is to preserve a distinct Silent preset shape, likely `main` plus `systemEvent`, or get explicit maintainer direction to redefine the preset.
