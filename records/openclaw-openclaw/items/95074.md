## Summary

Keep this PR open, but the current head is not merge-ready: it avoids the backend rejection by changing Silent into an isolated agent turn, which collapses Silent and Independent session into the same preset instead of preserving the established main-session/system-event behavior.

## What This Changes

The PR changes the Control UI Silent cron quick-create preset from `sessionTarget: "main"` to `"isolated"` and adds a node test for the isolated/no-delivery mapping.

## Best Possible Solution

Fix the quick-create mapper while preserving the Silent versus Independent session distinction, likely by making Silent emit a supported main-session `systemEvent` job unless maintainers explicitly choose to redefine the preset labels and semantics.

## Reproduction Assessment

Yes, source-level: current main maps Silent quick-create to a main-session `agentTurn` job, and the backend rejects main-session cron jobs unless the payload is `systemEvent`. I did not run a live browser/Gateway flow in this read-only review.

## Solution Assessment

No. The patch makes the job backend-valid, but it does so by collapsing Silent into the Independent session preset; the narrower maintainable fix is to preserve the existing main-session Silent semantics with a supported payload shape, or get explicit maintainer product direction to rename/redefine the presets.
