## Summary

Keep open: the reported tilde-resolution bug is real, but this PR should not merge as written because it reimplements home expansion with `os.homedir()` and diverges from the existing OpenClaw user-path/QMD resolver when `OPENCLAW_HOME` is set. The PR also has only unit-test proof, not real behavior proof for the memory indexing path.

## What This Changes

The PR expands tilde-prefixed configured extra memory paths in memory-host-sdk before absolute/workspace resolution, adds unit coverage for `~` and `~/...`, and adds a changelog entry.

## Best Possible Solution

Use the existing package user-path normalization for tilde-prefixed extra memory paths so builtin and QMD resolve the same config value, with regression coverage for `~`, `~/...`, and `OPENCLAW_HOME`.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction: current main treats `~/...` as non-absolute and resolves it under the workspace. The PR regression is also source-reproducible by comparing the new `homedir()` helper with the existing QMD `resolveUserPath()` path under `OPENCLAW_HOME`.

## Solution Assessment

No, not as written. Expanding before workspace resolution is the right fix location, but the implementation should reuse the existing package resolver instead of creating a parallel tilde contract.
