## What This Changes

The issue reports Discord guild replies in OpenClaw 2026.5.7 producing malformed outbound send payloads without `message` and repeated reply loops.

## Best Possible Solution

Land or replace the focused schema-gating fix so Discord guild send tools only expose fields consumed by advertised actions, then close this issue with PR, release, and targeted Discord/runtime proof.
