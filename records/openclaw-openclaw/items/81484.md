## What This Changes

Fix a Discord guild/server regression where mixed message-tool schemas can allow malformed `send` payloads without `message`, causing failed, duplicate, or looping outbound replies.

## Best Possible Solution

Land or replace the focused schema-gating fix so Discord guild send tools only expose fields consumed by advertised actions, then close this issue with PR, release, and targeted Discord/runtime proof.
