## Summary

Keep open: this is a plausible focused bug fix, but the current patch still preserves cross-channel structured recipients in WebChat delivery contexts and the external PR has no after-fix real behavior proof.

## What This Changes

The PR changes `normalizeDeliveryContext` to drop unstructured WebChat `to` values while preserving colon-prefixed targets, and adds normalization tests for `webchat:user-123` and a phone-number-shaped recipient.

## Best Possible Solution

Normalize WebChat delivery so only WebChat-owned targets such as `webchat:...` survive, clear unstructured and non-WebChat structured recipients before fallback treats them as routable, and cover normalization plus extraction paths.

## Reproduction Assessment

Yes from source inspection. The PR would preserve `normalizeDeliveryContext({ channel: "webchat", to: "telegram:123" })` as routable, and current `extractDeliveryInfo` treats any `channel` plus `to` as a deliverable context; I did not run tests in this read-only review.

## Solution Assessment

No. Preserving any colon-prefixed target is too broad; the narrower maintainable fix is to preserve only WebChat-owned targets and clear cross-channel structured recipients.
