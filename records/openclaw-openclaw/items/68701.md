## Summary

Keep open: this is a plausible bug-fix PR, but the current patch does not fully clear stale WebChat recipients because it preserves any structured target, including non-WebChat targets, and it has no after-fix real behavior proof.

## What This Changes

The PR changes `normalizeDeliveryContext` and tests so WebChat contexts drop unstructured `to` values while preserving structured targets such as `webchat:user-123`.

## Best Possible Solution

Normalize WebChat delivery so only WebChat-owned direct targets survive, clear unstructured and non-WebChat structured recipients, and add regression coverage against the current route-helper and delivery-info paths.

## Reproduction Assessment

Yes, from source inspection: the PR predicate keeps `normalizeDeliveryContext({ channel: "webchat", to: "telegram:123" })` routable, and current `extractDeliveryInfo` treats any `channel` plus `to` as routable. I did not run tests because this review kept the checkout read-only.

## Solution Assessment

No. Structured versus unstructured is not the right boundary; the narrow maintainable fix is WebChat-owned versus cross-channel targets, implemented against the current route-normalization helper.
