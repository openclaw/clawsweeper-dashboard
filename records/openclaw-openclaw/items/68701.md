## What This Changes

The PR changes WebChat delivery-context normalization to preserve colon-prefixed targets, drop unstructured WebChat `to` values, and add two normalization tests.

## Best Possible Solution

Normalize WebChat delivery so only WebChat-owned direct targets survive, clear unstructured and cross-channel recipients, and cover the normalization and delivery-info paths with regression tests plus redacted runtime proof.

## Reproduction Assessment

Yes, from source inspection: the PR keeps `normalizeDeliveryContext({ channel: "webchat", to: "telegram:123" })` routable, and `extractDeliveryInfo` treats any `channel` plus `to` as routable. I did not run tests because this review kept the checkout read-only.

## Solution Assessment

No. Structured-prefix preservation is not the right boundary; the maintainable fix is to preserve WebChat-owned targets and clear non-WebChat targets before downstream fallback treats them as routable.
