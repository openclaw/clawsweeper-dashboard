- `impact:message-loss`: The issue reports Telegram media-bearing group messages being skipped before plugin `message:received` ingest hooks can receive them.

## Summary

Keep open: current main and the latest release still skip unaddressed Telegram group media before checking `ingest`, and https://github.com/openclaw/openclaw/pull/95180 is now the open fix candidate that references this issue.

## What This Changes

Honor Telegram group/topic `ingest: true` before the `requireMention` media pre-download skip so `message:received` hooks receive unaddressed media-bearing group messages.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/95180, or replace it with a narrow Telegram fix that mirrors text ingest precedence, preserves silent passive media failure behavior, and includes focused tests plus live Telegram proof.

## Reproduction Assessment

Yes. Source inspection shows current main can return from the Telegram media pre-download skip before any ingest check, while the sibling text path already honors topic, group, and default ingest.

## Solution Assessment

Mostly yes. The fix belongs in the existing Telegram media pre-download gate, but the best implementation should also preserve the old no-visible-error behavior for passive ingest media failures.
