## Summary

Keep open: current main and v2026.6.6 still skip unaddressed Telegram media before media download or hook dispatch, and the open linked fix PR has not landed.

## What This Changes

Telegram media-bearing group/topic messages should honor existing `ingest: true` config before the unaddressed mention skip so plugin `message:received` hooks can see them.

## Best Possible Solution

Align the media pre-download skip with the text path's topic, exact group, and wildcard/default ingest precedence, add focused media regression coverage, and land the linked PR or a narrow replacement with Telegram proof.

## Reproduction Assessment

Yes, at source level. Current main skips unaddressed group media before media download and hook dispatch, while the sibling text path already honors `ingest`; I did not run a live Telegram repro in this read-only pass.

## Solution Assessment

Yes, the fix belongs in the Telegram media pre-download skip helper, but it should mirror the existing text path's topic, exact group, and wildcard/default ingest precedence.
