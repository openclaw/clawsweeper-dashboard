## Summary

Keep open: current main and the latest release still skip unaddressed Telegram media before the ingest hook can run, and the open linked fix PR has not landed yet.

## What This Changes

Honor Telegram group/topic `ingest` before the media pre-download mention skip so unaddressed media reaches `message:received` hooks.

## Best Possible Solution

Align the media skip bypass with the existing topic, exact group, and wildcard/default ingest precedence, add focused media regression coverage, and land the linked PR or a narrow replacement.

## Reproduction Assessment

Yes at source level. Current main skips unaddressed group media before media download and hook dispatch, while the sibling text path already honors `ingest`; I did not run live Telegram proof in this read-only pass.

## Solution Assessment

Yes, the fix belongs in the Telegram media pre-download skip helper, but it should mirror the text path's topic, exact group, and wildcard/default ingest precedence rather than only checking exact config.
