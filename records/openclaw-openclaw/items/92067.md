- `impact:message-loss`: The reported behavior silently drops Telegram media-bearing group/topic messages before plugin ingest hooks can receive them.

## Summary

Keep open: current `main` and the latest release still skip unaddressed Telegram group/topic media before honoring explicit ingest, and open linked PRs are implementation candidates rather than a reason to close this issue before merge.

## What This Changes

Honor Telegram group/topic `ingest: true` before the unaddressed group media pre-download skip so media-bearing messages can reach `message:received` hooks.

## Best Possible Solution

Land a narrow Telegram handler fix that mirrors the text-path ingest precedence for topic, exact group, and wildcard/default group config, with focused regression coverage and real Telegram proof, then close this issue.

## Reproduction Assessment

Yes at source level. Current `main` can return from the unaddressed group media skip before any ingest check, while the sibling text path already fires `message:received` when ingest is enabled.

## Solution Assessment

Yes for the requested direction: the Telegram media skip helper is the right owner boundary. The final fix should use the same topic, exact group, and wildcard/default ingest precedence as the existing text mention-skip hook.
