## Summary

Keep open: current main and v2026.6.8 still return from the Telegram unaddressed group-media skip without checking `ingest`, while the text path already honors that config and all same-bug fix PRs are closed unmerged.

## What This Changes

The issue asks Telegram media ingestion to honor `ingest: true` before skipping unaddressed group or topic media.

## Best Possible Solution

Land a narrow Telegram fix that mirrors the text-path ingest hierarchy in the media pre-download gate, preserves silent handling for ingest-bypassed media failures, and adds focused regression plus live Telegram proof.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current main can return from the media pre-download skip before any ingest check, while the sibling text path already honors topic, exact group, and wildcard/default ingest. I did not run a live Telegram session in this read-only review.

## Solution Assessment

Yes, the best fix belongs in the existing Telegram media pre-download gate, not in a new config surface or hook system. It should mirror the established text ingest precedence and preserve the no-visible-error intent of the old unaddressed-media skip.
