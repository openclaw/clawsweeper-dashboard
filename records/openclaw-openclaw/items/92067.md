## Label Justifications

- `P2`: This is a bounded Telegram regression that silently suppresses media for valid existing ingest configurations.
- `impact:message-loss`: Media-bearing Telegram updates are discarded before the configured plugin receive hook can observe them.

## Summary

Current main still drops unaddressed Telegram media before the configured ingest hook can run. Keep this issue open while maintainers select and correct one of the two linked closing PRs; neither is merged, and both currently omit the established wildcard/default ingest fallback.

## What This Changes

Honor topic, group, or inherited Telegram `ingest: true` configuration before skipping unaddressed group media downloads.

## Best Possible Solution

Update one canonical linked PR so the early media check uses the same topic → specific group → wildcard/default ingest precedence as downstream hook delivery, with regression tests covering individual media and buffered media groups; then merge it and close the duplicate PR.

## Reproduction Assessment

Yes, from source with high confidence: current main returns before message-context creation, while existing tests define mention-skipped messages with inherited `ingest: true` as deliverable to `message:received`. A live Telegram run was not performed in this read-only review.

## Solution Assessment

The proposed decision point is the right boundary, but the exact linked implementations are not yet the best fix because they omit wildcard/default ingest inheritance and focused regression coverage.
