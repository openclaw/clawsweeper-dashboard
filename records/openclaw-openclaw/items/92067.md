## Label Justifications

- `P1`: A current Telegram workflow silently suppresses media-bearing events for real users who explicitly enabled group ingest, while text events still arrive and mask the failure.
- `impact:message-loss`: The pre-download return prevents configured Telegram media events from reaching the message-received hook.

## Summary

Current main still drops unaddressed Telegram media before the existing explicit-ingest hook path is evaluated, and no linked or searched fix PR currently owns the repair. The report provides a concrete production reproduction on v2026.6.1, while source inspection independently confirms the ordering defect, so this issue remains necessary and is a strong narrow repair candidate.

## What This Changes

Make explicitly ingest-enabled Telegram groups or topics receive unaddressed media updates instead of dropping them before media download and hook dispatch.

## Best Possible Solution

Resolve one canonical effective-ingest fact for topic, specific group, and wildcard/default policy before media preflight, and preserve the download skip only when that fact is not explicitly true; add media-specific regression coverage for both enabled and default behavior.

## Reproduction Assessment

Yes. Current source provides a high-confidence path: an uncaptioned photo in a mention-required group reaches the pre-download no-mention return before the later explicit-ingest hook logic; the reporter also supplies a matching v2026.6.1 production reproduction and observed patched result.

## Solution Assessment

Yes in intent, but the best implementation is not a free-standing one-line copy of the precedence expression. The fix should share or carry forward the canonical effective-ingest decision so media preflight and message-context behavior cannot diverge.
