## Summary

Keep open: current main and the latest release still skip unaddressed Telegram group media before checking `ingest`, and https://github.com/openclaw/openclaw/pull/95180 is the open closing PR that now owns the fix path.

## What This Changes

The issue asks Telegram to honor explicit group/topic `ingest: true` before skipping unaddressed `requireMention` group media.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/95180, or replace it with a narrow Telegram fix that mirrors text-path ingest precedence, preserves passive no-spam media failure behavior, and includes focused tests plus live Telegram proof.

## Reproduction Assessment

Yes, at source level. Current main can return from the Telegram media pre-download skip before any ingest check, while the sibling text path already honors topic, group, and default ingest.

## Solution Assessment

Mostly yes. The fix belongs in the existing Telegram media pre-download gate, but the best implementation should also carry an ingest-bypass signal into media failure handling so passive unaddressed media does not create visible group warnings.
