## Label Justifications

- `P1`: Affected Telegram setups can silently lose media-bearing group messages before plugin ingest hooks run.
- `impact:message-loss`: The reported failure drops media-bearing Telegram messages from the plugin message-ingest path.

## Summary

Keep open: current main and v2026.6.6 still skip unaddressed Telegram group media before the ingest hook path, and the open closing PR at https://github.com/openclaw/openclaw/pull/92198 has not merged yet.

## What This Changes

Honor Telegram group/topic ingest configuration before skipping unaddressed group media so plugins receive media-bearing `message:received` events.

## Best Possible Solution

Land one Telegram fix that honors topic, exact group, and wildcard/default ingest before the media skip, add focused regression coverage, and close this issue after the linked PR or an equivalent fix merges.

## Reproduction Assessment

Yes. Source inspection shows the Telegram media helper can return `true` before media resolution and before the hook path, while existing sibling tests show text ingest is already expected to honor `ingest`.

## Solution Assessment

Yes, fixing the Telegram media skip helper is the right layer, but the final patch should match the existing topic, exact group, and wildcard/default ingest precedence used by the text path.
