## Summary

Keep open: current main and the latest release still skip unmentioned Telegram group media before any ingest check, while the sibling text path already honors the ingest config. The linked pull request https://github.com/openclaw/openclaw/pull/92153 owns a plausible fix path, but it is still open and reviewed as needing precedence/proof work before merge.

## What This Changes

The issue asks Telegram media pre-download gating to honor group/topic `ingest: true` before skipping unaddressed group media so `message:received` hooks receive media messages.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/92153 or a narrow replacement that checks topic, exact group, and wildcard/default ingest before the media skip, with focused regression coverage and redacted live Telegram proof.

## Reproduction Assessment

Yes. Source inspection shows current main can return from the Telegram media pre-download skip before any ingest hook path, while the sibling text path already honors ingest.

## Solution Assessment

Yes, the right fix belongs in the Telegram media skip helper. The exact patch should mirror the text path's topic, exact group, and wildcard/default ingest precedence rather than checking only part of the config surface.
