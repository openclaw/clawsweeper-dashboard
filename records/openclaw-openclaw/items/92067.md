## Summary

Current main still skips unaddressed photo-only Telegram group media before the established ingest-enabled `message:received` path can run. The report is a focused, source-reproducible regression and is not superseded by the separate ingress-spool issue, so it should remain open for a narrow fix.

## What This Changes

Honor explicit Telegram group or topic ingest configuration before skipping unaddressed media downloads.

## Best Possible Solution

Use one canonical Telegram ingest-policy resolver in both the early media decision and the later silent-ingest hook, preserving the no-download optimization whenever ingest is not explicitly enabled.

## Reproduction Assessment

Yes. On current main, an unmentioned photo-only group message with `requireMention: true` reaches the pre-download no-mention return before the existing ingest-enabled hook path.

## Solution Assessment

Yes in principle. The narrowest maintainable fix is to reuse the existing topic, exact-group, and wildcard ingest precedence at the pre-download boundary rather than add a divergent config lookup.
