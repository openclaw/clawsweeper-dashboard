## What This Changes

The issue requests that Telegram ingest-enabled groups or topics bypass the unaddressed group-media pre-download skip so `message:received` hooks receive media-bearing messages.

## Best Possible Solution

Review and land a narrow Telegram plugin fix that checks topic, exact group, and wildcard/default ingest before the pre-download media skip, with focused media regression coverage and redacted live Telegram proof.

## Reproduction Assessment

Yes by source inspection: current main returns from the Telegram pre-download media skip before media resolution and hook dispatch, while the sibling text skip path already honors `ingest`. I did not run a live Telegram reproduction in this read-only review.

## Solution Assessment

Yes, the fix belongs in the Telegram pre-download media skip helper, but it should match the existing text ingest precedence rather than add a narrower one-off check. The issue should stay open until the linked closing PR merges or is replaced.
