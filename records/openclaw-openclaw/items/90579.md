## Summary

Keep open: the PR is a focused fix for a real Telegram message-delivery regression, but the current provenance design still has a security-boundary bug and a delivery-failure edge before it is merge-ready.

## What This Changes

The PR adds shared-state SQLite provenance for trusted generated HTML staged into outbound media, then consults that provenance when host-read reloads the staged HTML file.

## Best Possible Solution

Bind provenance to the exact staged artifact, fail or drop media when non-ENOENT provenance writes fail, and then add after-fix host-read-enabled Telegram proof for the second-load path.

## Reproduction Assessment

Yes. The reported failure is source-reproducible: reply media normalization stages the original file, then Telegram reloads the staged path through the host-read HTML gate that current main only trusts under the temp root.

## Solution Assessment

No. Provenance-carrying is the right layer, but the current path-only marker and broad marker-write catch are not the safest implementation for a host-read HTML trust boundary.

Full review comments:

- **[P1] Bind trusted markers to the staged artifact:** `src/media/web-media.ts:391-394`
  - body: The marker is keyed only by `realpath` and the reader only checks `kind`/`version`, so if the staged HTML file is replaced before cleanup, a different regular single-link HTML file at the same path inherits trusted-generated authorization. Store and verify artifact identity or content metadata captured at mark time so replacement files are rejected.
  - confidence: 0.88
- **[P2] Do not ignore failed provenance writes:** `src/media/outbound-attachment.ts:42-47`
  - body: This catch swallows every marker-write failure and still returns the staged path. For DB, permission, or schema failures, the later Telegram `loadWebMedia` reload has no provenance row and can reject the attachment at the same host-read gate this PR is trying to fix; only tolerate the missing-file race or fail normalization so the existing media-failure path can run.
  - confidence: 0.84
