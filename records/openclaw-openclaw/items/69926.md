## Summary

Keep open. Current main and the latest release still lack WhatsApp per-group sender allowlists, and the linked open implementation at https://github.com/openclaw/openclaw/pull/81053 should be reviewed, repaired, merged, or replaced before this issue closes.

## What This Changes

Add optional WhatsApp per-group `allowFrom` sender authorization at root and account-scoped group config entries, overriding the channel/account `groupAllowFrom` for that group.

## Best Possible Solution

Land or replace the linked implementation with a consistent WhatsApp-owned per-group sender allowlist across schema, runtime ingress, context visibility, docs, and focused tests.

## Reproduction Assessment

Yes, at source level: current main's strict WhatsApp group schema lacks `allowFrom`, and inbound authorization only receives channel/account `groupAllowFrom` or its `allowFrom` fallback. I did not run live WhatsApp traffic in this read-only review.

## Solution Assessment

Yes, the requested direction is the narrow maintainable shape: extend the existing per-group WhatsApp config map, matching sibling channel prior art, rather than overloading `requireMention`, `toolsBySender`, or global deny lists. The linked PR is the right implementation path but still needs review/repair before merge.
