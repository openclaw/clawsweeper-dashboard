## What This Changes

Add optional WhatsApp group-level `allowFrom` entries under root and account-scoped `groups` maps to override the channel/account `groupAllowFrom` for that group.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81053, or a narrow replacement, with schema, type, resolver, docs, generated metadata, and regression coverage aligned.

## Reproduction Assessment

Yes for the feature gap: current source shows the strict WhatsApp group schema omits `allowFrom`, and the runtime only passes a single resolved account/channel `groupAllowFrom`. I did not run tests in this read-only review.

## Solution Assessment

Yes, if maintainers accept the new config surface. A WhatsApp-owned additive per-group override is the narrowest maintainable path and matches the existing group-config boundary without changing unrelated denylist or admission behavior.
