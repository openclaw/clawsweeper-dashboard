## Summary

Keep open: current main still lacks WhatsApp per-group `allowFrom`, and https://github.com/openclaw/openclaw/pull/81053 is an open closing PR for the requested behavior, so this issue should remain open until that PR or a replacement lands.

## What This Changes

Add optional WhatsApp per-group sender allowlists under root and account-scoped group configuration, overriding channel-level `groupAllowFrom` per group JID.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81053, or a narrow replacement, with schema, type, resolver, docs, generated metadata, and regression coverage aligned while preserving existing channel-level fallback behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows the strict WhatsApp group schema omits `allowFrom`, and the runtime passes only one resolved account/channel `groupAllowFrom` into ingress authorization.

## Solution Assessment

Yes, if maintainers accept the new config surface. A WhatsApp-owned additive per-group override is the narrowest maintainable path, while `denyFrom`, `requireMention`, and `toolsBySender` do not provide per-group sender authorization.
