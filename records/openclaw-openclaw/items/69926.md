## Summary

Keep open: current main and the latest release still lack WhatsApp per-group `allowFrom`, and https://github.com/openclaw/openclaw/pull/81053 is the open linked implementation but remains unmerged with review follow-up.

## What This Changes

Add optional WhatsApp `groups.<jid>.allowFrom` at root and account scope so each group can override the channel/account-level `groupAllowFrom` sender allowlist.

## Best Possible Solution

Land a narrow WhatsApp plugin change that adds optional per-group `allowFrom` at root/account group scopes and applies the effective sender boundary consistently to ingress and supplemental context.

## Reproduction Assessment

Yes for the feature gap: current source shows WhatsApp group config rejects `allowFrom`, and runtime uses only account/channel `groupAllowFrom`. I did not run tests because this was a read-only review.

## Solution Assessment

Yes, if implemented consistently: an additive WhatsApp-owned group override matches sibling channel patterns and avoids changing unrelated denylist or group-admission behavior. The open PR shape is close, but its review finding needs to be resolved before merge.
