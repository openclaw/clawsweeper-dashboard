## Summary

Keep open. Current main still lacks WhatsApp per-group sender authorization: group config accepts only prompt/tools/mention fields, and both inbound and command authorization still enforce one account/channel-level group allowlist.

## What This Changes

Add optional `allowFrom` to WhatsApp root and account-scoped group entries so a specific group can override channel-level group sender authorization.

## Best Possible Solution

Implement a WhatsApp-owned additive config/runtime/docs change that honors a defined group `allowFrom` as the sender-authority override while preserving existing fallback behavior when it is absent.

## Reproduction Assessment

Yes. Static current-main reproduction is high-confidence: `channels.whatsapp.groups.<jid>.allowFrom` is absent from the public type and strict schema, and runtime access checks never read the matched group entry for sender authorization.

## Solution Assessment

Yes. The requested additive override is the narrowest maintainable fix because it mirrors existing channel prior art without changing WhatsApp group admission or the separate multi-account inheritance question.
