## Summary

Keep open: current main still lacks WhatsApp per-group sender allowlists, and the linked implementation PR is open and uses closing syntax for this issue, so the issue should remain open until that PR or a replacement lands.

## What This Changes

Add optional WhatsApp `groups.<jid>.allowFrom` and account-scoped equivalents so sender authorization can vary per group instead of using one channel-wide `groupAllowFrom`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81053, or a narrow replacement, with schema, type, resolver, docs, generated metadata, and regression coverage aligned.

## Reproduction Assessment

Yes for the feature gap: current source shows the strict WhatsApp group schema omits `allowFrom`, and runtime passes one resolved `groupAllowFrom` for all groups. I did not run tests in this read-only review.

## Solution Assessment

Yes, if maintainers accept the new config surface. A WhatsApp-owned additive per-group override is the narrowest maintainable path and avoids changing unrelated denylist, admission, or mention-gating behavior.
