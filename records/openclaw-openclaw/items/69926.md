## Summary

Keep open. Current main and v2026.6.10 still lack WhatsApp per-group `allowFrom`, and the open closing PR is the active implementation path but remains open/conflicting with an authorization-boundary review concern.

## What This Changes

Add optional WhatsApp per-group `allowFrom` at root and account group config scopes so group sender authorization can override channel-level `groupAllowFrom`.

## Best Possible Solution

Land or replace the linked implementation with one WhatsApp-owned effective per-conversation sender allowlist used consistently across schema, generated metadata, ingress, command authorization, supplemental context, docs, and tests.

## Reproduction Assessment

Yes. Source inspection shows current main's strict WhatsApp group schema lacks `allowFrom`, and inbound authorization only receives account/channel `groupAllowFrom` or its `allowFrom` fallback.

## Solution Assessment

Yes, directionally. Extending the existing WhatsApp per-group config map is the narrowest maintainable shape, but the implementation should use one effective per-conversation allowlist across ingress, command checks, and supplemental context.
