## Summary

Keep open. Current main and v2026.6.10 still lack WhatsApp per-group sender allowlists, and the open linked implementation is the active fix path but remains conflicting with a security-boundary review concern.

## What This Changes

Add optional WhatsApp per-group `allowFrom` sender authorization for `channels.whatsapp.groups.<jid>` and account-scoped group entries.

## Best Possible Solution

Land or replace the linked implementation with one WhatsApp-owned effective per-group sender allowlist used consistently across schema, generated metadata, ingress, command authorization, supplemental context visibility, docs, and focused tests.

## Reproduction Assessment

Yes. At source level, current main's strict WhatsApp group schema lacks `allowFrom`, and inbound authorization only receives account/channel `groupAllowFrom` or its `allowFrom` fallback.

## Solution Assessment

Yes, directionally. Extending the existing WhatsApp per-group config map is narrower than overloading mention gates, tool policy, or global deny lists, but the open PR needs repair so the boundary is applied consistently.
