## Summary

Keep open. Current main and v2026.6.8 still lack WhatsApp per-group sender allowlists, and the linked open implementation at https://github.com/openclaw/openclaw/pull/81053 should be repaired, reviewed, merged, or replaced before this issue closes.

## What This Changes

Add optional WhatsApp group-level `allowFrom` entries so sender authorization can vary per configured group instead of using one channel/account-wide `groupAllowFrom`.

## Best Possible Solution

Land or replace the linked implementation with one consistent WhatsApp-owned per-group sender allowlist across schema, runtime ingress, supplemental context visibility, docs, and focused tests.

## Reproduction Assessment

Yes, at source level: current main's strict WhatsApp group schema lacks `allowFrom`, and inbound authorization only receives account/channel `groupAllowFrom` or its `allowFrom` fallback. I did not run live WhatsApp traffic in this read-only review.

## Solution Assessment

Yes, the requested direction is the narrow maintainable shape: extend the existing per-group WhatsApp config map, matching sibling channel prior art, instead of overloading mention gates, tool policy, or global deny lists. The linked PR is the current implementation path but still needs repair/rebase before merge.
