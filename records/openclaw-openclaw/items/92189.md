This remains necessary and is a narrow regression repair, not a duplicate of the MCP injection report.

Fix `sessions_spawn` so requester-side tool restrictions do not erase tools explicitly allowed for the target agent.

## Best Possible Solution

Keep the fail-closed policy guard, but propagate only an explicit ancestor delegation ceiling to the child and evaluate the target agent’s own configured tool policy independently of the immediate requester’s self-deny settings.
