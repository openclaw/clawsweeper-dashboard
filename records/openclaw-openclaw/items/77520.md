## Summary

Keep open: current main honors `messages.groupChat.visibleReplies: "automatic"` at the policy layer, but the Discord turn/status path can still mark success from queued final dispatch rather than proven visible delivery, so this remains distinct from the closed workaround reports.

## What This Changes

The issue asks for Discord guild/channel turns with `visibleReplies: "automatic"` to deliver normal final assistant text visibly, or surface/log failure instead of showing terminal completion.

## Best Possible Solution

Keep the issue open and fix Discord/channel final-reply accounting so automatic turns only show terminal success after a visible final reply is delivered or the missing/failed dispatch is surfaced and retryable.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? No; the report has concrete live steps, but this read-only review did not run a Discord guild turn. ## Solution Assessment
