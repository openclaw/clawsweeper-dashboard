## Summary

Keep open: the related closed reports only cover the default tool-only behavior, while this report says the documented `automatic` workaround is already applied; current Discord code also has a source-visible path where a turn can set a terminal done reaction before verifying a final reply was dispatched.

## What This Changes

The issue asks for Discord guild/channel turns with `messages.groupChat.visibleReplies: "automatic"` to visibly deliver plain final replies, or not mark the turn done when no final reply was dispatched.

## Best Possible Solution

Add a focused Discord guild/channel regression for `visibleReplies: "automatic"` with no message-tool send, then make final status and dispatch accounting depend on actual final visible delivery or a surfaced delivery failure.
