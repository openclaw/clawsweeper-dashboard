## Summary

Keep open. Current main still has adjacent agent lifecycle, tool/item, SDK run/tool/task, and Task Flow surfaces, but not the requested durable session-level process/step event contract with stable step identity and related message/tool/artifact identifiers.

## What This Changes

The issue requests structured session process events such as process.started and step.completed so clients can render progress without parsing assistant markdown.

## Best Possible Solution

Add an additive Gateway/SDK process-event contract with typed, replayable per-session/per-run process and step events, then align schema, docs, client handling, persistence/reconnect behavior, and tests.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is a feature/API gap. Source and docs inspection gives high confidence that current main lacks the requested process/step event contract.

## Solution Assessment

Yes, keeping this open for core Gateway/SDK protocol design is the right path. Existing agent, tool, task, and Task Flow surfaces should inform the design, but they are not a substitute for the requested durable session process stream.
