## Summary

Keep open. Current `main` still has adjacent `agent`, `session.tool`, SDK run/tool/task, and Task Flow surfaces, but not the requested durable session-level `process.*` / `step.*` event contract with stable step identity and related message/tool/artifact identifiers.

## What This Changes

The issue requests durable, structured session-level `process.*` and `step.*` events with stable step identity and related message/tool/artifact identifiers.

## Best Possible Solution

Add an additive Gateway/SDK process-event contract with typed, replayable per-session/per-run process and step events, then align schema, docs, client handling, persistence/reconnect behavior, and tests.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is a feature/API gap. Source and docs inspection gives high confidence that current `main` lacks the requested process/step event contract.

## Solution Assessment

Yes, keeping this open for core Gateway protocol work is the right path. Existing agent, tool, task, and SDK event surfaces should inform the design, but they are not a substitute for the requested durable session process stream.
