## Summary

Keep open. Current main has adjacent agent, session tool/message, SDK run/tool/task, and Task Flow surfaces, but not the requested durable session-level process.* / step.* Gateway contract with stable step identity and related message/tool/artifact ids.

## What This Changes

Request session-level structured process and step events with stable step identity so clients can render progress without parsing assistant text.

## Best Possible Solution

Add an additive Gateway protocol surface for typed, replayable per-session/per-run process and step events, then align schema, docs, SDK/client handling, Control UI rendering, and reconnect/reload tests.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is a feature/API gap. Source and docs inspection gives high confidence that current main lacks the requested process/step event contract.

## Solution Assessment

Yes, keeping this open for core Gateway protocol work is the right path. Existing item/tool/task surfaces should inform the design, but they are not a substitute for the requested durable session process stream.
