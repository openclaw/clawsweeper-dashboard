## Summary

Keep open. Current main and v2026.6.10 still have adjacent agent, item, tool, session.tool, and agent.wait streams, but not the requested session-level process/step contract with stable step identity and replay/reload semantics.

## What This Changes

The issue requests structured session process events such as process.started and step.completed with session/run ordering, step identity, timestamps, and related tool/message/artifact links.

## Best Possible Solution

Define one additive, permission-checked process-event contract with stable step identity, bounded persistence/replay, redaction rules, and mappings from existing agent item, plan, tool, and task streams.

## Reproduction Assessment

Yes, at source level. Current main and v2026.6.10 have adjacent generic event streams, but source inspection does not show the requested process.* and step.* taxonomy, stable step identity, or replayable process-event API.

## Solution Assessment

Unclear. Structured process events are a plausible right layer, but maintainers still need to decide taxonomy, persistence, authorization, redaction, and how this relates to existing progress streams.
