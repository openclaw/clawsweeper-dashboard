## Summary

Keep open. Current main has generic agent, tool, item, plan, and lifecycle events, but it does not provide the requested session-level process/step event contract with stable step identity; the closest related work is still open and narrower than this request.

## What This Changes

The issue requests session-level structured process events with stable step identity so clients can render progress without parsing assistant text.

## Best Possible Solution

Define one additive core process-event contract that either extends the current agent event bus or maps Task Flow/agent item events into stable session-level process and step records, then decide how it relates to the open inter-session progress-streaming issue.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Source inspection shows the requested structured process-event surface is absent from current session/gateway APIs, but this is primarily a feature/API gap rather than a runnable bug reproduction.

## Solution Assessment

Is this the best way to solve the issue? Unclear; the need is valid, but the event taxonomy and its relationship to existing agent item/plan streams, Task Flow, and inter-session progress streaming need maintainer design first.
