## What This Changes

Request to add typed, replayable session process and step events so clients can render progress as source of truth instead of parsing assistant text.

## Best Possible Solution

Define an additive Gateway/SDK process-event contract with typed per-session/per-run process and step events, replay semantics, schema/docs/client support, authorization, redaction, retention, and regression coverage that reuses adjacent event surfaces instead of duplicating them.
