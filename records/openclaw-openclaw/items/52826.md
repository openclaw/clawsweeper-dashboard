## Summary

Keep open. Current main has generic agent item/plan/tool/session events and task summaries, but it still does not provide the requested session-level `process.*`/`step.*` event contract with stable step identity, replay/reload semantics, and explicit linkage to tool calls, messages, and artifacts.

## What This Changes

Requested stable session-level structured process and step events so clients can render progress as source of truth instead of parsing assistant text.

## Best Possible Solution

Define one additive, permission-checked core process-event contract with stable step identity, bounded persistence/replay, redaction rules, and mappings from existing agent item/plan/tool/task streams, then decide how the narrower inter-session progress issue consumes that contract.

## Reproduction Assessment

Yes, at source level: current main lacks the requested `process.*`/`step.*` event taxonomy, stable step identity, and replayable session process-event API. This is a feature/API gap rather than a failing runnable bug reproduction.

## Solution Assessment

Unclear: the requested direction is valid, but the exact contract is not yet the best-fix shape until maintainers decide taxonomy, persistence, authorization, redaction, and how it relates to the open inter-session progress-streaming work.
