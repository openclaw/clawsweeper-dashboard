## What This Changes

Request session-level process and step events with stable step identity and related message/tool/artifact identifiers so clients can render durable progress without parsing assistant text.

## Best Possible Solution

Add an additive Gateway/SDK process-event contract with typed, replayable per-session/per-run process and step events, then align persistence/reconnect behavior, redaction, docs, client handling, and regression tests.

## Reproduction Assessment

## Solution Assessment

Yes, a first-class Gateway/SDK process-event contract is the maintainable direction. The existing plan, item, tool, session, and Task Flow surfaces are useful design inputs but are not substitutes for the requested durable session process stream.
