## Summary

Keep open. Current main still has only adjacent generic agent streams, live session message/tool broadcasts, session lifecycle snapshots, background tasks, and Task Flow records; it does not expose the requested durable session-level process.* / step.* Gateway contract with stable step identity and related tool/message/artifact ids.

## What This Changes

Add structured, replayable session process and step events so clients can render progress without parsing assistant text.

## Best Possible Solution

Implement an additive Gateway protocol surface for durable session process events, likely as typed process/step event payloads plus a per-session/per-run replayable ledger. Reuse existing lifecycle, item, plan, tool, transcript, task, and artifact signals where possible, then update schema, docs, SDK/client handling, Control UI rendering, and reconnect/reload tests around the stable contract.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is a feature/API gap. The absence is high-confidence from current-main source and docs searches for the requested event names plus inspection of the relevant agent event, Gateway schema, event discovery, session broadcast, and task/Task Flow surfaces.

## Solution Assessment

Yes, keeping this open for core Gateway protocol work is the right direction. This does not fit a plugin-only/ClawHub close because ClawWork needs upstream session process events as a stable transport contract, not an optional external integration.
