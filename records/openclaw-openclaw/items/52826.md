## Summary

Keep open. Current main and the latest release have generic agent item/plan/tool/session event streams, but not the requested stable session-level process/step event contract; the related inter-session progress-streaming issue is still open and narrower than this request.

## What This Changes

The issue requests stable session-level process and step events so clients can render progress without parsing assistant text.

## Best Possible Solution

Define a single additive, permission-checked process-event contract with persistence/replay semantics, then map existing item/plan/tool streams into it or fold it into the related progress-streaming design.

## Reproduction Assessment

Yes at source level: current main has generic agent item/plan/tool/session events, but no stable process.started or step.* session-level process contract. I did not run a live ClawWork client reproduction in this read-only cleanup pass.

## Solution Assessment

Unclear as an implementation path: the need is valid, but maintainers need to choose the taxonomy, persistence/replay rules, redaction, and relationship to https://github.com/openclaw/openclaw/issues/54895 before a fix PR.
