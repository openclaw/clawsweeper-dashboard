## What This Changes

The issue requests structured session-level process and step events with stable identity and links to related tool calls, messages, and artifacts so clients can render progress without parsing assistant text.

## Best Possible Solution

Add an additive Gateway/SDK process-event contract with typed, replayable per-session/per-run process and step events, then align schema, docs, client handling, persistence/reconnect behavior, authorization, redaction, and regression tests.
