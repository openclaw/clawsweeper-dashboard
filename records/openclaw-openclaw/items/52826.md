Keep open. Current main and the latest release still expose adjacent agent, item, tool, session.tool, and agent.wait surfaces, but not the requested session-level process/step event contract with stable step identity and replay/reload semantics.

The issue requests session-level structured process and step events with stable identifiers, sequencing, timestamps, and links to related tool calls, messages, and artifacts.

## Best Possible Solution

Define one additive, permission-checked process-event contract with stable step identity, bounded persistence/replay, redaction rules, and mappings from existing agent item, plan, tool, and task streams.
