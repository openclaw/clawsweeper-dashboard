## Summary

Keep open. Current main has adjacent agent, item, plan, tool, session.tool, Task Flow, and SDK-normalized run events, but it still lacks the requested typed session-level process/step event contract with stable step identity and tool/message/artifact linkage.

## What This Changes

The issue requests an additive session-level process/step event contract with stable step IDs and links to tool calls, messages, and artifacts so clients can render durable progress.

## Best Possible Solution

Add an additive Gateway/SDK process-event contract with typed, replayable per-session/per-run process and step events, aligned with TaskFlow/run-trace semantics plus authorization, redaction, persistence, reconnect, docs, and tests.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is a feature/API gap. Source inspection gives high confidence that current main lacks the requested process/step event contract.

## Solution Assessment

Yes for keeping the item open, but the implementation needs maintainer-owned API design. Existing agent events, session.tool events, SDK normalization, TaskFlow, and the broader run-trace request should inform the contract rather than replace this issue with an ad hoc mapping.
