## Summary

Keep open. Current main and v2026.6.8 still expose generic agent/item/plan/tool/session event surfaces, but not the requested session-level process/step event contract with stable step identity, replay/reload semantics, and explicit tool/message/artifact linkage.

## What This Changes

The issue requests session-level structured process and step events so clients can render progress as source-of-truth state instead of parsing assistant text.

## Best Possible Solution

Define one additive, permission-checked core process-event contract with stable step identity, bounded persistence/replay, redaction rules, and mappings from existing agent item/plan/tool/task streams.

## Reproduction Assessment

Yes, at source level. Current main has adjacent agent/item/plan/tool/session events but lacks the requested process.* and step.* taxonomy, stable step identity, and replayable session process-event API; I did not run a live reload/reconnect scenario in this read-only review.

## Solution Assessment

Unclear. Structured process events are a plausible right layer, but the exact contract is not ready for an automated fix until maintainers decide taxonomy, persistence, authorization, redaction, and its relationship to inter-session progress and durable audit streams.
