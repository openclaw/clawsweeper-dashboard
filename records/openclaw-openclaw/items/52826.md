## Summary

Keep open. Current main and the latest release still have generic agent/item/plan/tool/session events, but not the requested session-level process/step event contract with stable step identity, replay/reload semantics, and explicit tool/message/artifact linkage.

## What This Changes

The issue requests structured session process and step events so clients can render progress from source-of-truth events instead of assistant text.

## Best Possible Solution

Define one additive, permission-checked core process-event contract with stable step identity, bounded persistence/replay, redaction rules, and mappings from existing agent item/plan/tool/task streams.

## Reproduction Assessment

Yes. By source inspection, current main lacks the requested process.* and step.* event taxonomy, stable step identity, and replayable session process-event API.

## Solution Assessment

Unclear. Structured process events are a plausible right layer, but the exact contract is not ready for an automated fix until maintainers decide taxonomy, persistence, authorization, redaction, and its relationship to inter-session progress streaming.
