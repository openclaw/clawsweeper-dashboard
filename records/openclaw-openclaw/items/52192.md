## Summary

Keep open. Current main and the latest release still have the Feishu text-without-media duplicate-send path, and this PR’s code change is the right narrow fix, but the user-facing fix is missing the required changelog entry before it is merge-ready.

## What This Changes

The PR changes the Feishu outbound sendMedia handler to send companion text before media only when mediaUrl is present, and updates the adjacent comment.

## Best Possible Solution

Land the minimal mediaUrl guard with a single-line Unreleased/Fixes changelog entry crediting the contributor, and preferably add a focused regression test for text plus no mediaUrl; leave broader media-upload fallback behavior to a separate follow-up.

## Reproduction Assessment

Yes. A direct feishuOutbound.sendMedia call with non-empty text and no mediaUrl reaches the current companion text send and then the no-media text fallback, which is visible from source inspection without a live Feishu account.

## Solution Assessment

Yes. The mediaUrl guard is the narrowest maintainable code fix because it preserves text-plus-media and media-only behavior while letting the existing text-only fallback handle the no-media case once.
