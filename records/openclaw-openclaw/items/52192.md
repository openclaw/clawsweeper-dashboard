## Summary

Keep open: the Feishu duplicate-send bug is still present on current main, and this contributor PR remains relevant but needs a refresh against later voice-media suppression work plus real behavior proof before merge.

## What This Changes

The PR changes Feishu outbound `sendMedia` so companion text is only sent before media when `mediaUrl` is present.

## Best Possible Solution

Refresh the Feishu companion-text guard to require `mediaUrl` while preserving `!suppressTextForVoiceMedia`, then add a focused no-media regression test and keep the voice-media tests green.

## Reproduction Assessment

Yes by source inspection: call non-comment Feishu `sendMedia` with non-empty text and no `mediaUrl`; current main sends text at the companion-text guard and then again in the no-media fallback. I did not run a live or unit repro because this pass was read-only.

## Solution Assessment

Yes with changes: requiring `mediaUrl` in the companion-text guard is the narrowest maintainable fix, but the refreshed patch must preserve current voice-media suppression behavior.
