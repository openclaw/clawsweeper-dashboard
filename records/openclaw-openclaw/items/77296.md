## Summary

Keep open: current main still consumes enabled plugin-declared skill directories in place and does not publish them into the managed or agent skill roots named in the report; an open closing PR (#77328) already targets this issue.

## What This Changes

Publish plugin manifest-declared skill directories into agent skill discovery so enabled plugin SKILL.md prompts are reachable.

## Best Possible Solution

Review and land #77328 or an equivalent generic fix that exposes enabled plugin skill roots to the agent read/discovery path without plugin-specific ids.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes, from source inspection: manifest-declared plugin skill dirs are loaded directly as extra skill dirs, while the managed and `.agents/skills` roots remain separate and unpopulated.

## Solution Assessment

Is this the best way to solve the issue? The requested direction is right; the safest fix should register enabled plugin skill roots with the agent discovery/read layer or create a guarded managed mirror with containment and collision checks.
