## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/78571. On current main, Telegram plus `claude-cli` on Windows can fail because the Anthropic backend default `command: "claude"` is passed through the generic process supervisor without PATH/PATHEXT or `.cmd/.bat` safe wrapper resolution, and an explicit `claude.cmd` path may hit Node's direct `.cmd` spawn rejection. Add a narrow Windows-safe resolution path for CLI backend process launch, preferably reusing `src/plugin-sdk/windows-spawn.ts` semantics to unwrap npm shims to Node/EXE entrypoints and avoid `shell: true` for arbitrary args. Also cover Claude stream-json/live-session output where assistant text exists but the final `result` is empty, so dispatch queues a visible final instead of ending in Telegram `silent-reply-dispatch`. Update `docs/gateway/cli-backends.md` with a complete `claude-cli` example and Windows guidance. Add a user-facing changelog entry. Do not add a new config flag or broaden shell execution policy.
