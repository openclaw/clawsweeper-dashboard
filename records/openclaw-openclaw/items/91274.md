- **[P1] Handle scoped duplicates when unscoped exists:** `extensions/memory-core/src/memory/qmd-manager.ts:797-800`
  - body: This early `continue` skips legacy scoped cleanup whenever the unscoped name already exists. In the upgrade case described by the PR, an index can contain stale `memory-dir` plus current `memory-dir-<agent>`; the stale unscoped entry is removed, then adding `memory-dir` conflicts with the still-present scoped collection, and the current code cannot verify/rebind it because `collection list` lacks paths for non-current names. The migration needs to inspect and remove the matching scoped duplicate before using the unscoped name.
  - confidence: 0.91
- **[P2] Update the rebind tests to the final unscoped names:** `extensions/memory-core/src/memory/qmd-manager.test.ts:5854`
  - body: These new tests configure an explicit QMD path named `workspace`, and the PR's final backend config resolves that to the unscoped collection `workspace`. Keeping `collectionName` as `workspace-${agentId}` makes the test exercise legacy migration instead of same-name rebind, and the expected add call for the scoped name will not be produced on this branch.
  - confidence: 0.96
- **[P2] Update the documented QMD filter names:** `packages/memory-host-sdk/src/host/backend-config.ts:348`
  - body: The code now creates default collections as `memory-root` and `memory-dir`, but the QMD docs still tell users to search `memory-root-main` and `memory-dir-main`. Fresh indexes after this change would not have those scoped names, so the public example needs to change with the runtime behavior.
