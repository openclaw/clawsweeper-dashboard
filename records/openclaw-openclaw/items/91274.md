## Summary

Keep this PR open: the cleanup is directionally reasonable and has useful live proof, but it is not safe to merge while the current QMD collection-list path cannot reliably distinguish a stale unscoped collection from the desired new target, and the public QMD docs still show the old scoped filter names.

## What This Changes

The branch removes agent-id suffixes from managed QMD memory collection names, reverses the legacy migration from scoped names to unscoped names, and updates memory/QMD tests for the new identifiers.

## Best Possible Solution

Land the unscoped collection-name cleanup only after path-aware collection rebind metadata is present, then include a regression for indexes containing both stale unscoped and current scoped names and update the QMD docs in the same merge.
