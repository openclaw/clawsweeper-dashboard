## Best Possible Solution

Approve one canonical default-off `web_fetch` SSRF policy surface, likely nested under `tools.web.fetch.ssrfPolicy` or per-agent overrides, with explicit metadata/link-local, redirect, DNS rebinding, allowlist, audit, cache, docs, and regression coverage.

## Solution Assessment

No for the exact top-level key as proposed. The safer maintainable path is a maintainer-approved nested or per-agent SSRF policy contract that composes with existing fake-IP flags and documents the security boundary.
