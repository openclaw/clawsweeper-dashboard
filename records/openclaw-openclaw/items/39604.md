## Summary

Keep open: current main and v2026.5.7 still do not expose a broad private-network opt-in for arbitrary `web_fetch` URLs, and the linked implementation remains a security-sensitive product/API decision rather than cleanup.

## What This Changes

Request adding a default-off `tools.web.fetch` private-network config opt-in so `web_fetch` can reach local or internal addresses.

## Best Possible Solution

Land one reviewed default-off `web_fetch` private-network opt-in under the accepted `ssrfPolicy` shape, with docs, generated config metadata, cache partitioning, metadata/redirect tests, and release notes.

## Reproduction Assessment

Yes. Source inspection of current main and v2026.5.7 shows the strict schema lacks the requested key, the runtime never forwards a broad private-network SSRF policy, and tests document the current blocking path.

## Solution Assessment

Unclear as proposed. A default-off opt-in is plausible, but maintainers still need to choose the public API shape and approve the metadata/LAN/redirect threat model before the open PR or a replacement is the best implementation.
