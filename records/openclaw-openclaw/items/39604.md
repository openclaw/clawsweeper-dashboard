## What This Changes

The issue requests a default-off tools.web.fetch private-network config option so web_fetch can reach localhost, LAN, and internal addresses when explicitly enabled.

## Best Possible Solution

Define one maintainer-approved, default-off web_fetch SSRF policy that composes with existing fake-IP flags, partitions cache by every policy bit, documents the operator trust boundary, and lands with focused tests plus security review.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.8 expose only narrow web_fetch fake-IP SSRF policy keys while arbitrary private/internal targets still go through the strict guard. A live localhost fetch is not needed to prove the missing public config surface.

## Solution Assessment

No for the exact top-level key as proposed; the safer likely solution is a nested, default-off ssrfPolicy capability that preserves current fake-IP behavior and receives explicit maintainer security approval.
