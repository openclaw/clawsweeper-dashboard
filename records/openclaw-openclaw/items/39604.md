## Best Possible Solution

Define one maintainer-approved, default-off nested `web_fetch` SSRF policy that composes with the existing fake-IP flags, documents the operator trust boundary, preserves cache isolation, and explicitly handles metadata/link-local and redirect behavior.

## Reproduction Assessment

Yes, at source level. Current main and the latest release have a strict `web_fetch` schema with only narrow fake-IP SSRF flags, and current tests/docs still show private/internal targets blocked.

## Solution Assessment

No, not as the original top-level `allowPrivateNetwork` key. The safer direction is a maintainer-approved nested SSRF policy contract with explicit metadata/link-local, redirect, cache, docs, and audit semantics.
