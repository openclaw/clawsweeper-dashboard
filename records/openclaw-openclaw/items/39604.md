## Summary

Keep open. Current main still lacks a broad private-network opt-in for arbitrary `web_fetch` URLs, this issue is paired with the still-open implementation PR at https://github.com/openclaw/openclaw/pull/39630, and maintainer discussion says https://github.com/openclaw/openclaw/pull/76887 did not close the security-sensitive config decision.

## What This Changes

Request a default-off config option that lets `web_fetch` reach private, internal, or localhost addresses when explicitly enabled.

## Best Possible Solution

Land or replace the paired work with one maintainer-approved, default-off nested `web_fetch` SSRF policy that preserves fake-IP flags, blocks or documents metadata behavior, partitions cache keys, covers redirects, and updates docs, config metadata, changelog, and targeted tests.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature/config request. Source inspection gives high confidence in the current-main gap: strict `web_fetch` config lacks the broad private-network key, and runtime forwarding only includes the two narrow fake-IP SSRF flags.

## Solution Assessment

Unclear for the exact proposed top-level key. The capability is valid, but the safer maintainable path is a maintainer-approved `ssrfPolicy` shape with explicit metadata, redirect, cache, docs, and operator-trust boundaries.
