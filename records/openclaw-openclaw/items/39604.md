## Summary

Keep open. Current main still lacks a broad private-network opt-in for arbitrary `web_fetch` URLs, and the linked implementation PR at https://github.com/openclaw/openclaw/pull/39630 is still open rather than merged. A recent maintainer comment also says https://github.com/openclaw/openclaw/pull/76887 does not close this because general `web_fetch` private-network access remains a separate security-sensitive config decision.

## What This Changes

The issue requests a default-off config option so `web_fetch` can reach private/internal network addresses when explicitly enabled.

## Best Possible Solution

Land or replace the open implementation with one canonical default-off `web_fetch` private-network opt-in under the approved SSRF policy shape, preserving the shipped RFC2544 and IPv6 ULA flags, cache isolation, generated config metadata, docs, changelog, and targeted SSRF tests.

## Reproduction Assessment

Yes, as source proof rather than a runtime bug repro: current main's strict `web_fetch` schema rejects the requested broad private-network key, and the runtime forwards only the two narrow fake-IP SSRF flags.

## Solution Assessment

No, not as the exact top-level key proposed. The capability is valid, but maintainer guidance points toward a nested `ssrfPolicy` shape and a security review of metadata, LAN, redirect, and cache behavior before shipping.
