## Summary

Keep open. Current main still lacks a broad private-network opt-in for arbitrary `web_fetch` URLs, the issue is paired with the still-open implementation PR at https://github.com/openclaw/openclaw/pull/39630, and recent maintainer discussion says the merged fake-IP policy work did not close this security-sensitive config decision.

## What This Changes

Add an opt-in `web_fetch` configuration key so explicitly trusted agents can fetch private or internal network URLs.

## Best Possible Solution

Land or replace the paired work with one maintainer-approved, default-off `web_fetch` SSRF policy contract that preserves fake-IP flags, clearly handles metadata/link-local policy, partitions cache keys by policy, covers redirects, and updates docs, config metadata, changelog, and targeted tests.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature/config request. Source inspection gives high confidence in the current-main gap: strict `web_fetch` config lacks a broad private-network key, and runtime forwarding only includes the two narrow fake-IP SSRF flags.

## Solution Assessment

Unclear for the exact proposed top-level key. The capability is valid, but the safer maintainable path is a maintainer-approved nested SSRF policy shape with explicit metadata, redirect, cache, docs, and operator-trust boundaries.
