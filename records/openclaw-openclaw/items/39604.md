## What This Changes

## Best Possible Solution

Define one default-off nested `tools.web.fetch.ssrfPolicy` private-network contract, document the operator trust boundary, preserve existing fake-IP flags and cache partitioning, and land it only after maintainer security approval and real configured-run proof.

## Reproduction Assessment

Yes, at source level: current main has a strict web_fetch schema without the requested key, and the runtime only forwards narrow fake-IP policy bits. I did not run a live localhost fetch because the missing config contract and blocking tests are directly visible in source.

## Solution Assessment

No for the proposed top-level key as the final shape. Current code and maintainer discussion point toward a nested `tools.web.fetch.ssrfPolicy` contract with explicit metadata/link-local, redirect, and cache semantics.
