## Summary

Keep open. Current main still lacks a broad private-network opt-in for arbitrary `web_fetch` URLs, the issue is paired with the open implementation PR at https://github.com/openclaw/openclaw/pull/39630, and maintainer discussion says this remains a security-sensitive API decision rather than shipped behavior.

## What This Changes

The issue requests a default-off `tools.web.fetch.allowPrivateNetwork` config option so `web_fetch` can reach private/internal network targets when explicitly enabled.

## Best Possible Solution

Choose one maintainer-approved, default-off `web_fetch` SSRF policy contract that preserves existing fake-IP flags, blocks credential metadata endpoints as intended, partitions cache keys by policy, covers redirects, and ships with docs, config metadata, changelog, targeted tests, and real private-target proof.

## Solution Assessment
