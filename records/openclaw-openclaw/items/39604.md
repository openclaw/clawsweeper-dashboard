## Summary

Keep open. Current main and the latest release still do not expose a broad private-network opt-in for arbitrary `web_fetch` URLs, and https://github.com/openclaw/openclaw/pull/39630 already references this issue while maintainer discussion says the metadata/LAN/redirect policy still needs review.

## What This Changes

The request is to add an explicit opt-in `tools.web.fetch.allowPrivateNetwork` config key so `web_fetch` can reach local or private network URLs.

## Best Possible Solution

Review or replace the linked implementation PR with one canonical default-off `web_fetch` private-network opt-in under the accepted SSRF policy shape, with maintainer-approved metadata handling, redirect behavior, docs, changelog, cache isolation, and targeted SSRF tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: strict web_fetch config rejects the requested key, the runtime only forwards narrow fake-IP SSRF flags, and the current tests/docs still describe private/internal hosts as blocked.

## Solution Assessment

Unclear. A default-off opt-in is plausible, but the exact top-level key proposed here is not clearly the best shape because maintainer discussion points toward a nested SSRF policy and requires security review for metadata, LAN, and redirect semantics.
