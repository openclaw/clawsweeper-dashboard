## Summary

Keep open: the ambient-proxy failure is valid and this branch is close, but the current gate would also bypass OpenClaw managed proxy routing, which conflicts with the documented network-proxy contract and needs a small repair before merge.

## What This Changes

The PR changes Feishu WebSocket proxy handling to ignore ambient proxy environment variables unless `OPENCLAW_FEISHU_WS_USE_PROXY=1` is set, and updates Feishu client tests for the new default and opt-in.

## Best Possible Solution

Preserve the safer direct default for unmanaged ambient proxy env while still honoring OpenClaw managed proxy routing, then add regression coverage plus Feishu docs and a changelog entry before merge.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure Feishu WebSocket mode and set HTTP(S) proxy env on current main, where `getWsProxyAgent()` always resolves a proxy agent; the linked bug report adds live logs and recovery evidence, though I did not run a live Feishu gateway in this read-only review.

## Solution Assessment

No. Gating only on `OPENCLAW_FEISHU_WS_USE_PROXY=1` is not the narrowest maintainable fix because it solves ambient proxy inheritance by also disabling the documented managed-proxy path; the safer repair should distinguish unmanaged ambient env from OpenClaw-managed proxy activation.
