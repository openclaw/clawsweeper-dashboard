## Summary

This should stay open: the Feishu ambient-proxy failure is valid and the PR has usable live proof, but the patch currently breaks OpenClaw's managed proxy contract for runtime WebSocket egress by requiring a second Feishu-specific opt-in.

## What This Changes

The PR changes Feishu WebSocket proxy handling to ignore ambient proxy environment variables unless `OPENCLAW_FEISHU_WS_USE_PROXY=1` is set, with updated Feishu client tests.

## Best Possible Solution

Keep Feishu WebSocket direct by default for unmanaged ambient proxy env, continue honoring OpenClaw managed proxy routing, and document/changelog the Feishu-specific escape hatch.

## Reproduction Assessment

Yes. On current main, Feishu WebSocket creation always resolves an ambient proxy agent when HTTP(S) proxy env is set, and #65799 plus the PR body provide live logs showing failure and recovery when that ambient proxy was removed.

## Solution Assessment

No. Gating only on `OPENCLAW_FEISHU_WS_USE_PROXY=1` fixes unmanaged ambient env inheritance but also disables the documented managed-proxy path; the narrower fix should distinguish `OPENCLAW_PROXY_ACTIVE=1` from unrelated ambient proxy env.
