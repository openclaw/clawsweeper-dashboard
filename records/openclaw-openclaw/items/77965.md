## Summary

Keep open: the reported Feishu WebSocket failure is valid and the PR has real runtime proof, but the current diff would also disable documented operator-managed proxy routing for Feishu WebSocket and needs docs/changelog coverage before merge.

## What This Changes

The PR changes Feishu WebSocket client construction to ignore ambient proxy environment variables unless `OPENCLAW_FEISHU_WS_USE_PROXY=1` is set, with updated Feishu client tests for the default and opt-in paths.

## Best Possible Solution

Keep Feishu WebSocket direct by default for unmanaged ambient proxy env, preserve OpenClaw managed proxy routing, and document/changelog the Feishu-specific opt-in escape hatch.

## Reproduction Assessment

Yes. Current main can be reproduced from source by setting HTTP(S) proxy env and calling `createFeishuWSClient()`, which currently creates and passes a proxy agent; #65799 and the PR body add live failure/recovery logs.

## Solution Assessment

No. Gating only on `OPENCLAW_FEISHU_WS_USE_PROXY=1` fixes unmanaged ambient env inheritance but breaks the documented managed-proxy path; the narrower fix should distinguish `OPENCLAW_PROXY_ACTIVE=1` from unrelated ambient proxy env.
