## Summary

Keep open: current main has bounded tail readers and opt-in compaction rotation, but the active embedded run and compaction paths still materialize the full session transcript before any default quarantine or bounded successor recovery, and full-bootstrap completion markers are still appended without a per-session cap.

## What This Changes

The issue requests recovery guards to stop oversized session transcripts and repeated bootstrap markers from causing crash-restart OOM loops.

## Best Possible Solution

Add a default load-before-open transcript safety boundary that stats and streams active JSONL before repair/open/compaction, rotates or quarantines pathological files into a bounded successor, and caps repeated full-bootstrap marker accumulation.

## Reproduction Assessment

Yes at source level: a synthetic oversized active JSONL will still reach whole-file repair, dependency-backed SessionManager.open, and whole-file transcript-state reads before default quarantine. I did not run the live OOM loop.

## Solution Assessment

Yes: the narrow maintainable fix is a default guard at the transcript load/open boundary plus bootstrap-marker accumulation protection, rather than relying on opt-in compaction settings or channel-specific workarounds.
