## Summary

Keep open: current main still lets oversized active session JSONL files reach whole-file repair and dependency-backed `SessionManager.open()` before a default quarantine or bounded successor recovery, while the bootstrap completion marker path still has no per-session accumulation cap.

## What This Changes

The issue requests crash-loop recovery for oversized session transcripts so gateway restarts do not repeatedly load, crash, and append bootstrap-context entries.

## Best Possible Solution

Add a default pre-open transcript safety boundary that stats and streams active JSONL before repair/open/compaction, rotates or quarantines pathological files into a bounded successor, and caps repeated full-bootstrap marker accumulation.

## Reproduction Assessment

Yes at source level: a synthetic oversized active JSONL still reaches whole-file repair, dependency-backed `SessionManager.open`, and whole-file transcript-state reads before default quarantine. I did not run the live OOM loop.

## Solution Assessment

Yes: the narrow maintainable fix is a default guard at the transcript load/open boundary plus bootstrap-marker accumulation protection, rather than relying on opt-in compaction settings or channel-specific workarounds.
