## Summary

Keep open: current main has useful bounded tail readers, bounded bootstrap-completion scanning, and opt-in compaction rotation, but the active transcript path still reaches full-file repair/open helpers before any default quarantine or bounded successor recovery, and repeated full-bootstrap completion markers are still appended without a per-session cap.

## What This Changes

The issue requests default recovery for oversized session transcripts that can crash or hang the gateway on restart and grow further through repeated bootstrap replay entries.

## Best Possible Solution

Add a default load-before-open transcript safety boundary that stats and streams active JSONL before repair/open/compaction, rotates or quarantines pathological files into a bounded successor, and prevents repeated full-bootstrap marker accumulation.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: a synthetic oversized active session JSONL still reaches whole-file repair, dependency-backed SessionManager.open, and whole-file transcript-state reads before any default quarantine. I did not run the live OOM loop.

## Solution Assessment

Yes, the narrow maintainable fix is a default guard at the transcript load/open boundary rather than relying on opt-in compaction settings or channel-specific workarounds.
