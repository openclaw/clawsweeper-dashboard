## Best Possible Solution

Keep this issue open for a maintainer-designed recovery boundary that bounds active transcript repair/open, avoids repeat crash loading, and gives operators a safe degraded reset or compaction path.

## Reproduction Assessment

No live current-main OOM reproduction was run. Source inspection gives a medium-confidence reproduction path because the active embedded attempt still repairs and opens the full JSONL before any degraded skip, and two users reported the restart/OOM pattern.

## Solution Assessment

No, current main is not the complete solution. The narrow bootstrap circuit breaker is only a partial mitigation; the better fix needs a maintainer-approved recovery boundary across transcript loading, repair, compaction/reset, and operator recovery semantics.
