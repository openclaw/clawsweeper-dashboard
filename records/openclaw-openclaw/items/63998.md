## Summary

Keep open: current main still exposes oversized active session JSONL files to whole-file repair, dependency-backed `SessionManager.open()`, and whole-file transcript-state reads before any default quarantine or bounded successor recovery; the repeated full-bootstrap marker path also remains uncapped outside the continuation-skip optimization.

## What This Changes

The issue reports a gateway OOM crash-restart loop caused by oversized session transcripts and repeated bootstrap-context entries growing persisted session state on each restart.

## Best Possible Solution

Add a default pre-open transcript safety boundary that stats and streams active JSONL before repair, prewarm, `SessionManager.open()`, or rotation helpers; preserve the old file as an archive/quarantine, create a bounded successor for recovery, and make full-bootstrap completion markers idempotent or capped per session.

## Solution Assessment

Yes: the maintainable fix is a default guard at the transcript load/open boundary plus bootstrap-marker accumulation protection. The current opt-in compaction byte guard is useful but not sufficient for default crash-restart recovery.
