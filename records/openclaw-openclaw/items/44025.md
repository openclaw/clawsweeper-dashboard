## Summary

Keep open. Current main supports explicit ordered fallback chains and provider wildcard visibility, but it still does not expand `provider/auto` or `provider/*` into provider-local fallback candidates; implementing that needs model ordering, eligibility, and cost/subscription policy decisions.

## What This Changes

Request provider-scoped model auto/wildcard fallback strings such as `openai-codex/auto` or `openai-codex/*` that try same-provider candidates before cross-provider routers.

## Best Possible Solution

Define a generic provider-scoped fallback contract that deterministically expands `provider/auto` or `provider/*` through configured or discovered provider catalog order, then align runtime fallback, validation, CLI/docs, and regression tests.
