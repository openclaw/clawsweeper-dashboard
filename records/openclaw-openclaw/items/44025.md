## Summary

Keep open: current main and v2026.6.6 still only support explicit flat model fallback lists, while wildcard/auto provider-family selection or fallbackFamilies would add provider-routing and config semantics that need maintainer product design.

## What This Changes

Requests provider-scoped wildcard/auto model fallback and optional ordered fallback families so OpenClaw stays within an intended provider family before cross-provider routers.

## Best Possible Solution

Choose one canonical provider-family fallback contract, then update schema/types, config normalization, docs, doctor guidance if needed, and fallback-chain tests around that contract.

## Reproduction Assessment

Not a bug reproduction; yes for source-level verification. Current schema, docs, and runtime fallback code show flat fallback lists only, with no wildcard/auto or fallbackFamilies path.

## Solution Assessment

Unclear as proposed. The goal fits provider-routing needs, but maintainers still need to choose one predictable public config shape rather than landing both implicit wildcard expansion and explicit grouped fallbacks by default.
