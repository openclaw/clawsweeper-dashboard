## Summary

Keep open. Current main supports explicit ordered model fallbacks and provider-wildcard allowlists, but it does not implement provider-scoped automatic model selection such as `openai-codex/auto` or expanding `provider/*` into a runtime fallback chain.

## What This Changes

The issue requests wildcard or auto model selection within a provider family before falling back to cross-provider routers.

## Best Possible Solution

Define a documented provider-scoped fallback contract that specifies syntax, ordering, catalog discovery, health probing, and interaction with existing explicit `fallbacks` before implementing it in one shared resolver.

## Reproduction Assessment

Not applicable. This is a feature request, and source inspection shows the existing behavior is explicit fallback lists plus provider-wildcard visibility rather than provider-scoped auto selection.

## Solution Assessment

Unclear. The direction is plausible, but the exact `auto`/wildcard semantics and priority policy need maintainer product/API review before a narrow implementation would be safe.
