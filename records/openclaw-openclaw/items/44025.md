## Summary

Keep open: current main has explicit ordered model fallbacks and provider wildcard allowlists for model visibility, but it does not implement provider-scoped automatic runtime fallback from `provider/auto` or `provider/*` model refs. This remains a product/config feature request rather than a stale or already-implemented bug.

## What This Changes

The issue requests provider-scoped automatic model fallback using refs such as `provider/auto` or `provider/*` so provider model rotations stay within the intended provider before falling back elsewhere.

## Best Possible Solution

Define a provider-scoped model pool contract that reuses the existing fallback engine while keeping `agents.defaults.models` visibility semantics, runtime ordering, docs, and status output aligned.

## Reproduction Assessment

Not applicable as a strict bug reproduction: this is a feature request for behavior not currently promised by docs or tests. Source inspection shows explicit fallbacks and visibility wildcards exist, but no provider-auto runtime fallback pool is implemented.

## Solution Assessment

Unclear. The requested direction is plausible, but overloading `provider/*` or adding `provider/auto` affects config semantics and model ordering, so the best fix needs maintainer product design before implementation.
