## Summary

Keep open. Current main has explicit ordered model fallbacks and provider wildcard catalog visibility, but it still does not implement provider-scoped `provider/auto` or `provider/*` fallback expansion; that remaining behavior needs a maintainer product/API decision around order, eligibility, and cost boundaries.

## What This Changes

The issue requests provider-scoped automatic model fallback syntax such as `openai-codex/auto` or `openai-codex/*`, plus optional provider-local priority ordering.

## Best Possible Solution

Define a generic provider-scoped fallback contract, then align runtime candidate expansion, config validation, CLI/docs, and regression coverage around that contract.

## Solution Assessment
