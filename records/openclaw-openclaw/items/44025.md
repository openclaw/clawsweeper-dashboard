## Summary

Keep open: current main and the latest release still support only explicit flat model fallback lists, while wildcard/auto provider-family fallback and `fallbackFamilies` remain absent. The existing plugin `before_model_resolve` hook can choose a model before a turn, but it does not settle the failure-driven family retry semantics requested here, so this still needs maintainer product/config direction.

## What This Changes

The issue requests provider-family model fallback using wildcard or auto model selectors so OpenClaw can stay within a chosen provider before falling back across providers.

## Best Possible Solution

Maintainers should choose one canonical provider-family fallback contract, then update schema/types, config normalization, runtime candidate construction, docs, and focused fallback tests around that contract.

## Reproduction Assessment

Yes for source-level verification: current main and `v2026.6.10` only support flat `fallbacks`, and targeted searches found no wildcard/auto family fallback or `fallbackFamilies` implementation. This is a feature request rather than a failing-bug reproduction.

## Solution Assessment

Unclear as proposed: the goal is valid, but wildcard/auto expansion and explicit fallback families are different public contracts. The best fix needs maintainer selection of one predictable config and provider-routing model before implementation.
