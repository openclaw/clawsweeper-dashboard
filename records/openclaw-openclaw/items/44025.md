## Summary

Keep open. Current main has explicit ordered model fallbacks and provider wildcard model visibility, but it still does not implement provider-scoped `provider/auto` or `provider/*` fallback expansion; that remaining work needs product/API policy for ordering, eligibility, and cost/subscription behavior.

## What This Changes

Request provider-scoped auto or wildcard model fallback strings, such as `openai-codex/auto` or `openai-codex/*`, that try provider-local models before cross-provider fallback.

## Best Possible Solution

Define a generic provider-scoped fallback contract that deterministically expands `provider/auto` or `provider/*` through configured or discovered catalog order, then align runtime fallback, validation, CLI/docs, and regression tests.

## Reproduction Assessment

Not applicable as a strict bug reproduction; this is a feature request for new model-selection/config semantics. Source inspection confirms current main walks explicit fallback refs and treats provider wildcards as visibility policy.

## Solution Assessment

Unclear as stated. The direction is plausible, but maintainers should first define provider-scoped ordering, eligibility, and cost/subscription policy before adding `provider/auto` or fallback expansion semantics.
