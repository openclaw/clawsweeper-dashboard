## Summary

Keep open: current main supports explicit fallback chains, OpenRouter's `openrouter:auto` alias, and `provider/*` catalog visibility, but it still does not implement provider-scoped `auto` or wildcard fallback expansion. The remaining work is a user-facing model config and routing-policy decision, not conservative cleanup.

## What This Changes

Request provider-scoped wildcard or `auto` model fallback so provider-pinned configurations can try same-provider model candidates before cross-provider routers.

## Best Possible Solution

Define a provider-scoped fallback contract first, then align parsing, schema/help, runtime candidate expansion, docs, and regression coverage while preserving explicit fallback chains and strict user selections.
