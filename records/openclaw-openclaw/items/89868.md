## Label Justifications

- `impact:data-loss`: The reported failure irreversibly removes usable transcript context from the active branch after the user requested cancellation.

## Summary

Keep open: current main and the latest release still have a source-proven abort propagation gap in the delegated compaction path, matching the reported stop-button compaction race and session context loss.

## What This Changes

The issue asks for stop-button aborts during auto-compaction to discard in-flight compaction results instead of applying them to the active session transcript.

## Best Possible Solution

Keep this issue open and propagate the compaction abortSignal through delegateCompactionToRuntime into compactEmbeddedAgentSessionDirect, with a regression test proving aborted delegated compaction cannot append a compacted transcript.

## Reproduction Assessment

Yes, at source level: overflow recovery forwards an abort signal to the context engine, but the delegate bridge drops it before calling the runtime compaction path that knows how to cancel and discard. A full live model repro was not run during this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is to pass params.abortSignal through the delegate bridge and test that delegated runtime compaction receives it and does not append after abort. This does not require a new user-facing mode or config option.
