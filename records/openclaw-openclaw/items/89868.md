## Summary

Keep open: current main and v2026.5.28 still drop the context-engine compaction abort signal at the delegate bridge, so the reported stop-during-compaction race remains a narrow runtime bug.

## What This Changes

The issue reports that stopping during auto-compaction aborts the wait but not the in-flight delegated compaction, allowing a compacted transcript to be applied after cancellation.

## Best Possible Solution

Thread the caller abort signal through the context-engine delegate bridge and keep delegated compaction cancellation on the existing embedded runtime abort contract.

## Reproduction Assessment

Yes for a source-level repro: call `delegateCompactionToRuntime` with an `abortSignal` and inspect the mocked `compactEmbeddedAgentSessionDirect` params; current main drops the signal. I did not run a live 200K-token compaction stop-button scenario.

## Solution Assessment

Yes: forwarding `params.abortSignal` at `src/context-engine/delegate.ts` is the narrowest maintainable fix because the public context-engine contract and embedded runtime params already support the signal.
