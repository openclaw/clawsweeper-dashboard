## Summary

Keep open: current main and the latest stable release still drop the context-engine compaction abort signal at the runtime delegate, while https://github.com/openclaw/openclaw/pull/89886 is an open linked fix candidate rather than merged implementation proof.

## What This Changes

The issue asks for stop-button abort during auto-compaction to cancel or discard the in-flight compaction result so the next model turn does not see an unintended compacted context.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/89886, or replace it with an equivalent narrow bridge fix that forwards the existing abort signal and keeps focused regression coverage.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: current main threads a run abort signal into context-engine compaction, but delegateCompactionToRuntime does not pass it to runtime compaction. I did not run the full live large-context stop-button scenario.

## Solution Assessment

Yes. Forwarding the already-defined abortSignal through the delegate bridge is the narrow maintainable fix because the public contract and downstream runtime already support cancellation; broader stop-button timeout and lock issues are separate.
