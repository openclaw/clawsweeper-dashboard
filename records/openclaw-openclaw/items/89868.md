## Summary

Keep open: current main and the latest release still drop the compaction abort signal at the context-engine delegate bridge, and the linked open PR is the active fix candidate rather than merged implementation proof.

## What This Changes

Stop-button abort during auto-compaction should cancel or discard in-flight compaction so the next turn does not see an unwanted compacted session.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/89886, or replace it with an equivalent narrow bridge fix that forwards the existing abort signal and keeps focused regression coverage.

## Reproduction Assessment

Yes for a high-confidence source-level reproduction: current main threads a run abort signal into context-engine compaction, but `delegateCompactionToRuntime` does not forward it to runtime compaction. I did not run the full live stop-button and large-context scenario in this read-only pass.

## Solution Assessment

Yes. Forwarding the already-defined `abortSignal` through the delegate bridge is the narrow maintainable fix because the public contract and downstream runtime already support it; broader stop-button lock or timeout cleanup remains separate.
