## Summary

Keep open: current main and the latest release still drop the compaction `abortSignal` at the context-engine delegate bridge, and the remaining fix is already owned by the open linked PR https://github.com/openclaw/openclaw/pull/89886.

## What This Changes

The issue reports that pressing Stop during auto-compaction can abort only the wait path while the in-flight compaction still applies a summary to the active session context.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/89886, or replace it with an equivalent narrow bridge fix that forwards the existing abort signal and keeps the focused regression coverage.

## Reproduction Assessment

Yes for a high-confidence source-level reproduction: current main threads a run abort signal into context-engine compaction but `delegateCompactionToRuntime` does not forward it to runtime compaction. I did not run the full live stop-button and large-context scenario in this read-only pass.

## Solution Assessment

Yes, the best fix shape is to forward the already-defined `abortSignal` through `delegateCompactionToRuntime`; the public contract and downstream runtime already support it, so no new API, config, or product decision is needed.
