## Summary

Keep open: current main still does not provide a general opt-in, domain-scoped agent attestation contract for arbitrary outbound HTTP requests, and the remaining work is a security/product design decision rather than stale cleanup.

## What This Changes

The issue requests optional OpenClaw agent attestation headers or signed tokens for outbound HTTP requests, with config and skill metadata to control where identity proof is sent.

## Best Possible Solution

Define the attestation trust model and outbound path scope first, then implement a privacy-preserving opt-in config/API with domain scoping, non-leakage guarantees, docs, and tests.

## Reproduction Assessment

Not applicable: this is a feature/security design request rather than broken existing behavior. Source, docs, live issue context, and related-item checks show the requested general attestation headers, config, and skill metadata surfaces do not exist today.

## Solution Assessment

Unclear until maintainers settle the product/security design. Static headers, signed JWTs, and per-skill metadata are plausible inputs, but a local header patch is not the right solution until trust, privacy, redirect behavior, and outbound scope are explicit.

## Security Review

Status: needs_attention

Summary: The request is security-sensitive because it would send identity or proof material to external HTTP services and needs a trust/privacy design before implementation.

Concerns:

- **[medium] Define attestation trust and replay controls:**
  - body: A signed attestation header needs a clear issuer, signing authority, public-key distribution model, audience binding, expiry/replay window, and rotation story before OpenClaw can safely emit verifiable identity claims.
  - confidence: 0.92
- **[medium] Prevent cross-service identity leakage:**
  - body: Static or signed headers could leak stable agent, user, channel, session, or skill identifiers to unintended hosts unless config and runtime enforce strict opt-in domain scoping, redirect handling, and non-leakage across fallback paths.
  - confidence: 0.9
