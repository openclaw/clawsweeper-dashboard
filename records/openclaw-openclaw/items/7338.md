## Summary

Keep open. Current main still lacks a general, opt-in, domain-scoped agent attestation surface for outbound HTTP requests, and the existing provider attribution code is narrower provider-transport attribution; the prior maintainer review also treats this as product/security design work.

## What This Changes

Request opt-in OpenClaw agent attestation on outbound external HTTP requests, including domain allowlists and optional skill-scoped signed headers.

## Best Possible Solution

Define the identity and outbound-scope model first, then implement a privacy-preserving opt-in attestation surface with strict domain allowlisting, redirect protections, docs, and non-leakage tests.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. There is a high-confidence current-main verification path: inspect web_fetch headers, runtime config/schema, skill metadata, docs, and provider attribution to confirm the requested general attestation surface is absent.

## Solution Assessment

Unclear until maintainers settle the design. Static headers alone are not the best solution for agent-only services; the maintainable path needs scoped consent, cryptographic or explicitly limited semantics, replay protection, and non-leakage guarantees.

## Security Review

Status: needs_attention

Summary: This issue is security-sensitive because it would assert OpenClaw agent identity to third-party services and could leak stable identifiers if scoped incorrectly.

Concerns:

- **[medium] Define the attestation trust model:**
  - body: Before implementation, maintainers need to decide who signs attestations, how services obtain verification keys, how replay is prevented, and what issuer/audience claims mean for OpenClaw agents.
  - confidence: 0.93
- **[medium] Prevent cross-domain identity leakage:** `src/agents/tools/web-fetch.ts:433`
  - body: Any attestation must be opt-in and allowlisted so agent ids, versions, skill context, or signed tokens are not attached to arbitrary external requests, redirects, custom proxies, browser traffic, or fallback providers.
  - confidence: 0.91
