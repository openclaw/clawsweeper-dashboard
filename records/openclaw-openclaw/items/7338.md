## Summary

Keep this open: current main still lacks a general, opt-in, domain-scoped agent attestation surface, and the existing provider attribution work is narrower than the requested external-request identity feature.

## What This Changes

Request an opt-in agent attestation mechanism for external HTTP requests, including optional signed headers plus skill/config allowlists.

## Best Possible Solution

Define the security model and outbound scope first, then add a generic opt-in config/API contract with domain allowlisting, privacy-preserving defaults, docs, and non-leakage tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a new identity/security capability request. Source and docs inspection gives high confidence that current main does not implement the requested general attestation surface.

## Solution Assessment

Unclear until maintainer design approval. A narrow header injection patch is not the best fix yet because trust, privacy, signing, and outbound-path scope need to be settled first.
