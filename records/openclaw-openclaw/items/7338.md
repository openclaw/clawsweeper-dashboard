## Summary

Keep open: current main still lacks general opt-in, domain-scoped agent attestation for arbitrary outbound HTTP requests, and the existing provider-attribution and plugin-hook surfaces do not provide a general agent-origin attestation contract.

## What This Changes

The issue requests optional OpenClaw agent attestation headers or signed attestations for outbound external HTTP requests, with domain scoping and skill/config opt-in.

## Best Possible Solution

Define the attestation trust model and outbound path scope first, then implement an opt-in, domain-scoped config/API with privacy-preserving defaults, documented semantics, and non-leakage tests.

## Reproduction Assessment

Not applicable: this is a feature/security design request rather than broken existing behavior. Static source, docs, related-item, and live issue checks show the requested headers, config, and skill metadata surfaces do not exist today.

## Solution Assessment

Unclear until maintainers settle the product/security design. Static headers, signed JWTs, and per-skill metadata are plausible directions, but a local header patch is not the best solution until trust, privacy, redirect behavior, and outbound scope are explicit.

## Security Review

Status: needs_attention

Summary: The requested capability is security-sensitive because it would emit identity or proof material across external HTTP trust boundaries.

Concerns:

- **[medium] Define attestation trust and replay controls first:**
  - body: A signed attestation header needs a clear issuer, signing authority, public-key distribution model, audience binding, expiry/replay window, and rotation story before OpenClaw can safely emit verifiable identity claims.
  - confidence: 0.92
- **[medium] Prevent cross-service identity leakage:**
  - body: Static or signed headers could leak stable agent, user, channel, or session identifiers to unintended hosts unless config and runtime enforce strict opt-in domain scoping, redirect handling, and non-leakage across fallback paths.
  - confidence: 0.9
