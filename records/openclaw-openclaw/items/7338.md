## Summary

Keep open: current main still lacks general opt-in, domain-scoped agent attestation for arbitrary external HTTP requests, and the existing provider-attribution and provider-plugin header seams are narrower than this request.

## What This Changes

The issue requests opt-in OpenClaw agent attestation headers for outbound external HTTP requests, including static headers or signed attestations, per-skill metadata, and domain allowlists.

## Best Possible Solution

Define the trust model and outbound path scope first, then implement an opt-in, domain-scoped attestation config/API with privacy-preserving defaults, docs, and tests for allowlisted injection plus non-leakage.

## Reproduction Assessment

Not applicable: this is a feature/security design request, not broken existing behavior. Static source and release inspection give high confidence that OpenClaw lacks the requested attestation headers, config, and skill metadata surfaces.

## Solution Assessment

Unclear until maintainers settle the product/security design. Static headers, signed JWTs, and per-skill metadata are plausible directions, but a local header patch is not the best solution until trust, privacy, redirect behavior, and outbound scope are explicit.
