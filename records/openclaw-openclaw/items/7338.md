## Summary

Keep open: current main still lacks opt-in, domain-scoped agent attestation for arbitrary external HTTP requests, and the related provider/MCP header work is narrower than this security/privacy design request.

## What This Changes

Request an opt-in OpenClaw agent attestation header or signed token for external HTTP requests, optionally controlled by config and skill metadata.

## Best Possible Solution

Define the trust model and outbound scope first, then add an opt-in, domain-allowlisted config/API contract with privacy-preserving claims, docs, schema validation, and non-leakage tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a new identity/security capability request. Source and docs inspection gives high confidence that current main does not implement the requested general attestation surface.

## Solution Assessment

Unclear until maintainer design approval. A narrow header injection patch is not the best fix yet because trust, privacy, signing, verifier semantics, and outbound-path scope need to be settled first.
