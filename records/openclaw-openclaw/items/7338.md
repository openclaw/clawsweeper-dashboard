## Summary

Keep this open: current main still lacks opt-in, domain-scoped agent attestation for arbitrary external HTTP requests, and the request needs security/product design before implementation.

## What This Changes

Requests optional static or cryptographic OpenClaw attestation headers for external HTTP requests, plus skill metadata and config allowlisting.

## Best Possible Solution

Define the trust model and outbound scope first, then add an opt-in, domain-allowlisted config/API contract with privacy-preserving claims, docs, and non-leakage tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a new identity/security capability request. Source and docs inspection gives high confidence that current main does not implement the requested general attestation surface.

## Solution Assessment

Unclear until maintainer design approval. A narrow header injection patch is not the best fix yet because trust, privacy, signing, verifier semantics, and outbound-path scope need to be settled first.
