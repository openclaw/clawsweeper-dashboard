## Summary

Keep this open. Current main still lacks a general opt-in, domain-scoped agent attestation surface for arbitrary external requests, and the adjacent provider-attribution and trust-plugin discussions do not implement the requested agent-origin attestation headers.

## What This Changes

Request an opt-in OpenClaw agent attestation mechanism for outgoing HTTP requests, with domain allowlisting, possible signed headers, and optional skill metadata/config control.

## Best Possible Solution

Define the trust model and outbound scope first, then implement a privacy-preserving opt-in attestation contract with domain allowlisting, schema/docs, and tests proving injection only on approved destinations and non-leakage elsewhere.

## Reproduction Assessment

Not applicable: this is a feature request, not a bug report. Source inspection confirms the requested attestation surface is absent on current main, but there is no failing existing behavior to reproduce.

## Solution Assessment

Unclear until maintainers choose the trust model. A signed, opt-in, domain-scoped design is safer than global static headers, but the exact implementation should wait for security/product approval and a chosen outbound scope.
