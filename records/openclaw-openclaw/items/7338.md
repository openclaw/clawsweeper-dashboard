## Summary

Keep open. Current main still lacks opt-in, domain-scoped OpenClaw agent attestation for arbitrary external HTTP requests; the existing provider attribution and trust-hook work covers adjacent, narrower surfaces, while this issue remains a product/security design tracker.

## What This Changes

Request to add opt-in OpenClaw agent attestation headers for outbound HTTP requests so external services can distinguish OpenClaw agents from ordinary clients.

## Best Possible Solution

Define the trust model and exact outbound scopes first, then add an opt-in, domain-scoped config/API surface with privacy-preserving defaults and non-leakage tests.
