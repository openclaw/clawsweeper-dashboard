## Summary

Keep open: current main still lacks opt-in, domain-scoped agent attestation for arbitrary external HTTP requests, and the existing OpenAI/OpenRouter attribution work is provider-transport attribution rather than the requested general agent-origin proof.

## What This Changes

Request to add opt-in OpenClaw agent attestation headers or signed tokens for allowlisted external HTTP requests and skill-declared audiences.

## Best Possible Solution

Define the trust model and outbound scope first, then add an opt-in, domain-allowlisted config/API contract with privacy-preserving claims, docs, schema validation, and non-leakage tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a new identity/security capability request. Source and docs inspection gives high confidence that current main does not implement the requested general attestation surface.

## Solution Assessment

Unclear until maintainers settle the design. A narrow header-injection patch is not the best fix yet because trust, privacy, signing, verifier semantics, and outbound-path scope need to be decided first.
