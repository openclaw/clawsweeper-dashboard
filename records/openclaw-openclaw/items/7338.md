## Summary

Keep open. Current main still lacks a general opt-in, domain-scoped agent attestation surface for outbound external HTTP requests, and the existing provider attribution work is narrower provider-transport metadata rather than a service-verifiable agent-origin mechanism.

## What This Changes

The issue requests optional OpenClaw agent attestation headers or signed JWTs for outbound HTTP requests, controlled by user config and possibly skill metadata.

## Best Possible Solution

Define the identity, trust, and outbound-scope model first, then implement privacy-preserving opt-in attestation with strict domain allowlisting, redirect protections, docs, and non-leakage tests.

## Reproduction Assessment

Not applicable as a runtime bug reproduction. There is a high-confidence current-main verification path: inspect web_fetch headers, runtime config/schema, skill metadata, docs, and provider-attribution behavior to confirm the requested general attestation surface is absent.

## Solution Assessment

Unclear until maintainers settle the design. Static headers alone are not enough for agent-only services; the maintainable path needs scoped consent, cryptographic or explicitly limited semantics, replay protection, and non-leakage guarantees.

## Security Review

Status: needs_attention

Summary: This feature is security-sensitive because it would assert OpenClaw agent identity to third-party services and could leak stable identifiers if scoped incorrectly.

Concerns:

- **[medium] Define the attestation trust model:**
  - body: Before implementation, maintainers need to decide who signs attestations, how services obtain verification keys, how replay is prevented, and what issuer/audience claims mean for OpenClaw agents.
  - confidence: 0.93
- **[medium] Prevent cross-domain identity leakage:** `src/agents/tools/web-fetch.ts:432`
  - body: Any attestation must be opt-in and allowlisted so agent ids, versions, skill context, or signed tokens are not attached to arbitrary external requests, redirects, custom proxies, browser traffic, or fallback providers.
  - confidence: 0.91
