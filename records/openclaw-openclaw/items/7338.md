## Summary

Keep open: current main still lacks a general opt-in, domain-scoped OpenClaw agent attestation contract for arbitrary outbound HTTP requests, and the remaining work is a product/security design decision rather than conservative cleanup.

## What This Changes

The issue requests optional static or cryptographic agent attestation headers, domain allowlisting, and skill-level metadata for outbound OpenClaw HTTP requests.

## Best Possible Solution

Use this issue as the canonical design tracker, then implement a privacy-preserving opt-in attestation contract only after maintainers define trust, scope, config/API shape, docs, and non-leakage tests.

## Reproduction Assessment

Not applicable: this is a feature/security design request rather than broken existing behavior. Source, docs, live issue context, and related-item checks show the requested general attestation headers, config, and skill metadata surfaces do not exist today.

## Solution Assessment

Unclear until maintainers settle the product/security design. Static headers, signed JWTs, and per-skill metadata are plausible inputs, but a local header patch is not the right solution until trust, privacy, redirect behavior, and outbound scope are explicit.

## Security Review

Status: needs_attention

Summary: The request is security-sensitive because it would send identity or proof material to external services and needs trust/privacy design before implementation.

Concerns:

- **[medium] Define attestation trust and replay controls:**
  - body: A signed attestation header needs an issuer, signing authority, public-key distribution model, audience binding, expiry/replay window, and rotation story before OpenClaw can safely emit verifiable identity claims.
  - confidence: 0.92
- **[medium] Prevent cross-service identity leakage:**
  - body: Static or signed headers could leak stable agent, user, channel, session, or skill identifiers to unintended hosts unless config and runtime enforce strict opt-in domain scoping, redirect handling, and non-leakage across fallback paths.
  - confidence: 0.9
