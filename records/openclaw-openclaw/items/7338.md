## Summary

Keep open: current main still lacks opt-in, domain-scoped agent attestation for arbitrary outbound HTTP requests, and the existing OpenAI/OpenRouter attribution code is provider-transport attribution rather than a general agent-origin proof. The prior issue discussion also keeps this as product/security design work, so there is no conservative cleanup close available.

## What This Changes

Request opt-in OpenClaw agent attestation metadata or signed headers for outbound HTTP requests to agent-targeted services.

## Best Possible Solution

Define the security and privacy model first, then add an opt-in, domain-allowlisted config/API contract with minimal claims, verifier semantics, docs, schema validation, and non-leakage tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a new identity/security capability request. Source and docs inspection gives high confidence that current main lacks the requested general attestation surface.

## Solution Assessment

Unclear until maintainers settle the design. A narrow header injection patch is not the best fix yet because signing, verifier trust, privacy, and outbound path scope need explicit decisions first.

## Security Review

Status: needs_attention

Summary: The request is security-sensitive because attestation headers would create a new outbound identity and external verification contract.

Concerns:

- **[high] Define verifier trust and signing model:**
  - body: A signed attestation header needs clear issuer, public-key distribution, replay protection, audience binding, expiration, and rotation semantics before external services can safely rely on it.
  - confidence: 0.9
- **[medium] Prevent stable identity leakage:** `src/agents/tools/web-fetch.ts:442`
  - body: Agent ids, skill ids, sender ids, or workspace-derived identifiers in outbound headers could enable cross-service tracking unless domain allowlisting, minimal claims, and user opt-in are enforced.
  - confidence: 0.88
- **[medium] Constrain outbound injection paths:** `src/config/types.tools.ts:617`
  - body: External HTTP traffic spans multiple surfaces, so implementation needs a shared scope contract to avoid accidental leakage or inconsistent behavior across tools, skills, plugins, and provider transports.
  - confidence: 0.84
