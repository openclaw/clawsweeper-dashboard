The PR adds an Active Memory formatter that renders internal `empty` recall results as user-visible `status=none` and updates status-line tests accordingly.

## Best Possible Solution

Keep the landed Active Memory status split on current main and close this branch rather than merging the older display-only mapping.

## Reproduction Assessment

Yes for the pre-fix behavior: source and tests showed normal `NONE`/no-summary recall surfacing as `status=empty`. Current main no longer has that failing path because the statuses are split in source and regression tests.

## Solution Assessment

No for this PR branch; mapping every `empty` display value to `none` is broader than the maintainable fix. The current-main implementation from https://github.com/openclaw/openclaw/pull/80015 is the better solution because it preserves distinct `no_relevant_memory`, `unavailable`, and `failed` outcomes.
