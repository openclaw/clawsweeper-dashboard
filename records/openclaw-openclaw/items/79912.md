## Summary

Keep open: current source review agrees with the existing ClawSweeper comment that the PR fixes the benign wording path but maps every internal `empty` result to `none`, which would hide unavailable/error Active Memory paths in the user-visible status line.

## What This Changes

The PR adds an Active Memory display formatter that renders internal `empty` recall results as user-visible `status=none` and updates status-line test expectations.

## Best Possible Solution

Distinguish benign no-useful-memory recall from unavailable/error/timeout states in Active Memory, render only the benign outcome as `none` or the chosen final label, and keep failure-oriented statuses visible with focused tests.

## Reproduction Assessment

Yes. Current main source shows `NONE`/empty-summary recall becomes `status: "empty"` and current tests assert the user-visible `status=empty` line, though this read-only review did not run the test suite.

## Solution Assessment

No. The PR’s display-only mapping is too broad because `empty` is also used for unavailable search records and generic failures; the safer fix is to classify those result reasons before formatting.
