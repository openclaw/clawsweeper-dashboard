Keep this PR open. Current main still has the reported policy gap, and the PR is an active security-hardening fix that routes inline skill tool dispatch through the shared effective tool pipeline; it should receive maintainer/security review and CI validation rather than being closed or replaced.

The PR changes inline skill `command-dispatch: tool` handling to use `createOpenClawCodingTools(...)` with resolved runtime policy, sandbox, session, sender, group, and model context, updates focused regression coverage, and removes the old raw-tool runtime shim.

## Best Possible Solution
