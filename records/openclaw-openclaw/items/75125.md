## What This Changes

The PR changes inline skill `command-dispatch: tool` handling to build tools through `createOpenClawCodingTools(...)` with runtime-policy session, sandbox, target-session, sender, and model context; it updates focused regression coverage and removes the old raw-tool runtime shim.

## Best Possible Solution

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: configure a restricted profile or leaf subagent, add a user-invocable skill dispatching to a denied tool such as `sessions_spawn`, and the current inline branch resolves it from raw `createOpenClawTools(...)` after only owner-only filtering.

## Solution Assessment

Yes. Reusing `createOpenClawCodingTools(...)` with runtime-policy session, sandbox, target-session, sender, and model context is the narrowest maintainable fix; duplicating those policy filters in inline dispatch would risk future drift.
