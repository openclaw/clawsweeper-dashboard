## Summary

Keep open. Current main still has the APNs managed-proxy bypass this PR targets, but the latest PR head still has concrete CONNECT tunnel hardening defects in a security-labeled egress path, so it should not merge as-is.

## What This Changes

The PR routes direct APNs HTTP/2 sends through the active managed proxy and adds APNs proxy validation CLI/docs, active-proxy state tracking, raw HTTP/2 guardrails, tests, and a changelog entry.

## Best Possible Solution

Keep the APNs-specific proxy seam, but harden CONNECT parsing and cleanup, add the test-secret allowlist marker, then rerun the targeted APNs/proxy tests and security guardrails.

## Reproduction Assessment

Yes. Current main's bypass is visible at `src/infra/push-apns.ts:662`, and the PR defects are reproducible by static inspection of malformed proxy credential handling and unterminated CONNECT header buffering in `src/infra/net/http-connect-tunnel.ts`.

## Solution Assessment

No, not as-is. The wrapper direction is narrow and maintainable, but the CONNECT implementation needs bounded response parsing and cleanup-safe request construction before it is the best fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Cap CONNECT response headers before buffering:** `src/infra/net/http-connect-tunnel.ts:121`
  - body: `onData()` keeps concatenating proxy response chunks into `responseBuffer` until it sees a header terminator, with no maximum size. A bad proxy can stream unterminated header bytes and grow memory until timeout or OOM, so the tunnel should enforce a fixed CONNECT header limit and fail closed.
  - confidence: 0.9
- **[P2] Route CONNECT request build failures through fail():** `src/infra/net/http-connect-tunnel.ts:116`
  - body: `writeConnectRequest()` decodes proxy credentials and can throw for malformed percent escapes. Because `onConnected()` calls it directly from the socket event handler, that exception bypasses `fail()`, skipping normal cleanup and rejection for proxy validation or APNs send failures.
  - confidence: 0.91
- **[P3] Allowlist the test APNs private key fixture:** `src/infra/push-apns.test.ts:18`
  - body: The new test embeds a private-key PEM without the repo's usual test secret allowlist marker. Add the marker or move the material into an existing fixture pattern so secret scanning does not create noisy alerts for this security PR.
  - confidence: 0.82
