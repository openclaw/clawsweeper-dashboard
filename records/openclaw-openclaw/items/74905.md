## Summary

Keep this PR open: the main-branch APNs proxy gap is real, but the latest PR head still has concrete CONNECT hardening defects and a test-secret-scanning issue in a security-labeled change.

## What This Changes

The PR routes direct APNs HTTP/2 delivery through a managed proxy wrapper, adds active proxy state and APNs proxy validation, adds raw HTTP/2 guardrails, updates docs/changelog, and adds targeted tests.

## Best Possible Solution

Keep the APNs-specific proxy seam, but bound CONNECT response parsing, route CONNECT request-build failures through cleanup, and allowlist or replace the PEM fixture before maintainer/security review and merge.

## Reproduction Assessment

Yes. Current main's bypass is visible at `src/infra/push-apns.ts:662`, and the PR defects are reproducible by static inspection of the latest head plus a direct Node check showing malformed proxy credentials throw during `decodeURIComponent()`.

## Solution Assessment

No, not as-is. The wrapper direction is the right narrow surface, but this security-sensitive CONNECT implementation needs bounded parsing and cleanup-safe request construction before it is the best fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Cap CONNECT response headers before buffering:** `src/infra/net/http-connect-tunnel.ts:121`
  - body: `onData()` keeps concatenating proxy response chunks into `responseBuffer` until it sees a header terminator, with no maximum size. A bad proxy can stream unterminated header bytes and grow memory until timeout or OOM, so the tunnel should enforce a fixed CONNECT header limit and fail closed.
  - confidence: 0.91
- **[P2] Route CONNECT request build failures through fail():** `src/infra/net/http-connect-tunnel.ts:116`
  - body: `writeConnectRequest()` decodes proxy credentials and can throw for malformed percent escapes, but `onConnected()` calls it directly from the socket event handler. That bypasses `fail()`, so wrap the build/write and reject through the normal cleanup path.
  - confidence: 0.9
- **[P3] Allowlist the test APNs private key fixture:** `src/infra/push-apns.test.ts:18`
  - body: The new test embeds a full PEM private key without the repo's usual `// pragma: allowlist secret` marker for test secret material. Add the marker or move the material into an established allowlisted fixture pattern to avoid noisy secret-scanning alerts.
  - confidence: 0.82
