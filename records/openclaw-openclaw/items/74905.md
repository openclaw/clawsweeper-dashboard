## Summary

Keep open. Current main still has the APNs raw HTTP/2 proxy bypass, but the latest head still has concrete CONNECT tunnel hardening issues on a protected security/maintainer PR.

## What This Changes

The PR routes APNs HTTP/2 sends and proxy validation through a managed-proxy-aware connection seam, adds APNs reachability CLI/docs, guardrails against raw HTTP/2 use, tests, and a changelog entry.

## Best Possible Solution

Keep the APNs-specific proxy seam, but harden the CONNECT path with bounded header parsing and cleanup-safe error handling, then rerun the targeted proxy/APNs tests and security review before merge.

## Reproduction Assessment

Yes. Current main’s bypass is visible at `src/infra/push-apns.ts:662`, and the PR defects are reproducible by inspecting malformed proxy credential handling at `http-connect-tunnel.ts:116` and unterminated CONNECT header buffering at `http-connect-tunnel.ts:121`.

## Solution Assessment

No, not as-is. The APNs wrapper direction is narrow, but the CONNECT implementation needs fixed bounds and failure routing, or an equivalent maintained-agent approach, before it is the safest solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Route CONNECT request failures through fail():** `src/infra/net/http-connect-tunnel.ts:116`
  - body: `writeConnectRequest()` decodes proxy credentials and can throw for malformed percent escapes. Because `onConnected()` calls it directly from the socket event handler, that exception bypasses `fail()`, skips normal cleanup/rejection, and can crash instead of returning a proxy validation/APNs failure.
  - confidence: 0.91
- **[P2] Cap CONNECT response headers before buffering:** `src/infra/net/http-connect-tunnel.ts:121`
  - body: `onData()` appends every chunk to `responseBuffer` until it sees `\r\n\r\n`, with no maximum size. A bad proxy can keep sending unterminated header bytes and grow memory until timeout or OOM, so the tunnel should enforce a fixed CONNECT header limit and fail closed.
  - confidence: 0.9
- **[P3] Allowlist the test-only APNs PEM fixture:** `src/infra/push-apns.test.ts:18`
  - body: The new test embeds a private-key PEM without the repo's test secret allowlist marker. Add the marker or move the material into an existing fixture pattern so secret scanning does not produce noisy alerts for this security PR.
  - confidence: 0.82
