## Summary

Current main already fixes the reported Windows `EPERM` media-save failure in the shared media store, and Feishu image downloads save through that same helper; the fix is on main but not in the latest `v2026.5.3` release.

## What This Changes

The issue reports Feishu inbound image media downloads failing on Windows because saving the downloaded buffer hits `EPERM: operation not permitted, fsync`.

## Best Possible Solution

Keep the shared media-store fallback on main; users on `v2026.5.3` need a build or release that includes `484195d14e6d22824b832dcaa92c05498e7ece42`.

## Reproduction Assessment

Yes for the root cause: Feishu saves inbound images through `saveMediaBuffer`, and the regression test reproduces `EPERM` from `handle.sync()` while confirming current main no longer fails. I did not run a native Windows Feishu session in this read-only review.

## Solution Assessment

Yes. The shared media-store fallback is the narrowest maintainable fix because Feishu, Telegram, WebChat, and other channels all rely on the same media persistence helper.
