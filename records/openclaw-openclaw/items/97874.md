## What This Changes

The PR tightens Talk Voice `/voice set` so persistent voice selection requires `operator.admin` when gateway scopes are present or explicit owner status when scopes are absent, while adding focused tests.

## Best Possible Solution

Land the narrow subcommand-level owner/admin gate after maintainers explicitly accept the compatibility impact and exact-head checks remain green.

## Reproduction Assessment

Yes. Current main’s source and tests show no-scope Telegram `/voice set` mutates config, and the PR’s posted proof exercises the production command path showing denial, owner/admin success, and read-only behavior.

## Solution Assessment

Yes. Subcommand-level gating is the best fit because it preserves `/voice status` and `/voice list`, while a whole-command `requiredScopes` gate would over-block read-only behavior; it also matches the trusted Phone Control owner/admin pattern.
