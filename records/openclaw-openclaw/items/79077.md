## Summary

Keep open: current main and the latest shipped tag still lack Telegram bot-sender policy, peer/delegation routing, Telegram loop protection, Guest Bot handling, and the requested doctor diagnostics. The fresh operator comments and upstream-library note support the need, but the remaining work is a Telegram product/API design rather than conservative cleanup.

## What This Changes

Request Telegram support for Guest Bots and Bot-to-Bot mode with explicit bot-sender policy, per-sender loop/rate protection, peer delegation routing, and doctor diagnostics.

## Best Possible Solution

Design an opt-in Telegram-owned bot-sender and peer policy first, wire it into the shared bot-loop guard, then add delegation/session semantics and doctor diagnostics once the Bot API surfaces are confirmed.

## Solution Assessment
