## Summary

Keep open: current main and v2026.6.6 still pin `acpx@0.10.0`, whose built-in OpenCode ACP command is `npx -y opencode-ai acp`, so the reported spawn path remains live and no canonical merged fix owns it yet.

## What This Changes

Requests replacing the default OpenCode ACP launch command from the npx wrapper with `opencode acp` to avoid intermittent ACP initialize/spawn failures.

## Best Possible Solution

Fix the OpenCode default in upstream acpx, publish that acpx release, bump OpenClaw's ACPX dependency, refresh the ACP router docs/skill defaults, and validate with the OpenCode live ACP bind lane.

## Reproduction Assessment

No live high-confidence reproduction was established for the intermittent failure rate. Source inspection does show the current `sessions_spawn(runtime="acp", agentId="opencode")` path still reaches an acpx registry entry that launches OpenCode through npx.

## Solution Assessment

The direct `opencode acp` direction is plausible, but the best maintainable fix is an upstream acpx registry change plus an OpenClaw dependency bump, not a private OpenClaw-only override.
