## What This Changes

The issue asks OpenClaw to launch the OpenCode ACP harness directly with `opencode acp` instead of `npx -y opencode-ai acp`.

## Best Possible Solution

Update the upstream acpx built-in OpenCode adapter command, publish the fixed acpx package, then bump OpenClaw’s ACPX plugin dependency and refresh the ACP router docs/skill text plus live ACP bind validation.

## Reproduction Assessment

No live high-confidence reproduction was established for the intermittent failure rate. Source inspection does prove that current main and the latest release still launch the OpenCode adapter through the reported npx command path.

## Solution Assessment

The direct `opencode acp` direction is plausible, but the best maintainable fix is at the acpx built-in registry boundary followed by an OpenClaw dependency bump, not a private OpenClaw-only runtime workaround.
