## Summary

Keep open: the PR targets a real false-positive class, but as written it can reduce deep audit coverage by skipping test-named files without preserving files reachable from plugin runtime entries, and the external contributor has not provided real behavior proof. The related open PR at https://github.com/openclaw/openclaw/pull/82488 covers the same issue but has the same security blocker, so there is not a clean merged or canonical superseding fix yet.

## What This Changes

The branch threads `excludeTestFiles` through the code-safety scanner and deep audit helper, enables it for plugin and skill deep audits, and adds scanner-focused tests.

## Best Possible Solution

Preserve manifest-entry runtime graph coverage before excluding non-runtime test fixtures, then prove the false-positive fix with focused tests and a real security audit run.

## Reproduction Assessment

Yes. Source inspection shows current main deep audits pass only explicit manifest entries while the scanner default includes test files, and the linked issue provides concrete Discord false-positive examples; I did not run the audit because the checkout must stay read-only.

## Solution Assessment

No. Excluding test files is the right direction only after deep audit preserves runtime-reachable files; this patch skips test-named runtime imports without adding that coverage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Preserve runtime coverage before excluding test files:** `src/security/audit-extra.async.ts:1389`
  - body: This enables test-file exclusion for deep plugin audits, but the deep audit path only force-includes manifest entries and does not include files imported by those entries.
