## Summary

Keep open: the PR targets a real false-positive class, but the patch turns on test-file exclusion for deep plugin audits before preserving runtime import-graph coverage, and it still lacks after-fix real behavior proof.

## What This Changes

The branch adds an `excludeTestFiles` scanner option, threads it into deep plugin and skill code-safety summaries, and adds scanner tests for test/spec/mock file exclusion.

## Best Possible Solution

Mirror or reuse the install scanner’s runtime graph coverage in deep audits, then exclude non-runtime test/mock/spec fixtures and prove the result with focused regression tests plus a real audit run.

## Reproduction Assessment

Yes, source-reproducible: current main’s deep plugin audit calls `scanDirectoryWithSummary` with only `includeFiles`, and the scanner default includes test files. The linked issue provides concrete Discord false-positive audit output; I did not run the audit because this review is read-only.

## Solution Assessment

No, not as written. Excluding non-runtime test files is the right direction, but deep audit should first preserve runtime-reachable file coverage so test-like imported modules are not skipped.
