## Summary

Keep open: current main still has a source-reproducible premature completion-output path, and the duplicate-announcement half is covered only by open, unmerged repair PRs rather than shipped or current-main code.

## What This Changes

Issue reports that subagent completion announcements can treat assistant text from tool-use turns as terminal output and can deliver the same terminal child result more than once.

## Best Possible Solution

Land a refreshed current-main fix, or reviewed replacement PRs, that treats mixed text/tool-call turns as progress until a later non-tool final assistant response and adds bounded terminal announce idempotency at the delivery boundary.

## Reproduction Assessment

Yes. Current main has a high-confidence source reproduction: mixed assistant text plus tool-call turns are still selected as completion output, and an adjacent test expects that behavior.

## Solution Assessment

Yes for the requested direction, but no complete fix is on main. The right boundaries are the existing subagent output selector and announce-delivery idempotency path, with current open PRs needing refresh or replacement.
