## Summary

Keep open: current main still has the reported source-level premature completion path and no local terminal announce idempotency guard, and the related search did not find a merged fix or narrower canonical replacement.

## What This Changes

The issue requests that subagent completion announcements ignore tool-use progress text as terminal output and deliver each child run completion only once.

## Best Possible Solution

Keep the fix inside the agents announce output and delivery flow: wait for a post-tool assistant final response, avoid promoting tool-result progress while waiting, and add bounded announceId-based terminal idempotency with focused regression coverage.

## Reproduction Assessment

Yes for a source-level/unit reproduction: a chat history containing assistant text plus a non-sessions_yield tool call still returns that text as subagent output. The duplicate half is source-reproducible from the missing local announceId guard, but it should be pinned with a focused test that invokes the same childSessionKey and childRunId through direct and queued delivery.

## Solution Assessment

Yes. The proposed direction is the narrowest maintainable fix because it changes only the agents announcement output and delivery identity paths, reusing the existing announce-id machinery instead of adding a new policy surface.

In the agents subagent announce path, treat assistant messages containing any tool call as progress unless a later assistant message without tool calls provides the terminal completion; after non-sessions_yield tool use, do not promote toolResult text while waiting for that final assistant response. Add bounded terminal announce idempotency keyed by the existing announce identity built from childSessionKey plus childRunId so repeat terminal delivery for the same child run is suppressed across the direct and queued paths. Add or update focused regression tests in the subagent announce output/flow/queue suites, including the existing non-sessions_yield fixture and a duplicate same childSessionKey plus childRunId case.
