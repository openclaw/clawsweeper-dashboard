## Summary

Keep open: current main still has a source-reproducible path for premature subagent completion output after non-`sessions_yield` tool-use turns, and the duplicate-announcement concern remains plausible enough to need a focused regression fix rather than cleanup close.

## What This Changes

The issue requests that subagent completion announcements ignore progress text on tool-use turns and dedupe terminal announcements by stable child-run identity.

## Best Possible Solution

Update the agents announcement selector and delivery flow so only a later assistant message without tool calls is terminal output, and terminal delivery is guarded by the stable child-run announce id with focused regression tests.

## Reproduction Assessment

Unclear for the whole issue: the premature-output half has a high-confidence source and unit-fixture path on current main, while the duplicate-delivery half still needs a focused regression or live proof. The clearest reproduction target is `readSubagentOutput` with a non-`sessions_yield` tool-use assistant turn plus direct/queued announce calls for the same stable child-run id.

## Solution Assessment

Yes, the proposed direction is the narrowest maintainable fix: keep output classification in `src/agents/subagent-announce-output.ts`, keep terminal idempotency in the announce flow, and update the focused tests instead of adding a new product option.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/80498 in the agents runtime. Keep the change scoped to subagent completion announcement output selection and terminal announcement idempotency. In `src/agents/subagent-announce-output.ts`, assistant messages that include any tool call should not be selected as terminal completion output; after non-`sessions_yield` tool use, wait for a later assistant message without tool calls, and do not promote intervening tool/toolResult text to terminal completion while waiting. In `src/agents/subagent-announce.ts` and/or the announce queue boundary, add bounded in-memory terminal dedupe keyed by the existing stable announce id from `childSessionKey + childRunId`; do not add config or change channel/provider behavior. Update `src/agents/subagent-announce-output.test.ts` and `src/agents/subagent-announce.test.ts` to cover non-yield tool-use continuation, no tool-result promotion while waiting, and duplicate direct/queued terminal delivery for the same child run. Check overlap with https://github.com/openclaw/openclaw/pull/80110 before editing the selector. No changelog is required unless maintainers decide the behavior change is user-visible enough for release notes.
