## Summary

Keep open: current main still has a source-reproducible path that treats assistant text on non-`sessions_yield` tool-use turns as completion output, while the duplicate-delivery claim is plausible but still needs a focused regression test.

## What This Changes

The issue requests tighter subagent completion announcement selection after tool-use turns and per-child-run terminal announce idempotency.

## Best Possible Solution

Change the agents announce output and delivery flow so only a later assistant message without tool calls is terminal completion output, and guard terminal announcements by the existing child-run announce identity.

## Reproduction Assessment

Unclear for the whole issue: the premature-output half has a high-confidence source and unit-fixture path on current main, but the duplicate-delivery half was not independently executed in this read-only review. A focused regression should cover both direct and queued announce paths for the same `childSessionKey + childRunId`.

## Solution Assessment

Yes, the proposed direction is the narrowest maintainable fix: keep it inside `src/agents` announcement output/flow code, reuse the existing announce-id machinery, and add regression tests for tool-use continuation and terminal idempotency.

## Repair Work Prompt

Fix the subagent completion announcement regression reported in https://github.com/openclaw/openclaw/issues/80498. In `src/agents/subagent-announce-output.ts`, treat assistant text attached to any assistant message containing tool calls as progress rather than terminal completion output; after non-`sessions_yield` tool use, wait for a later assistant message without tool calls before selecting final output, and do not promote tool-result text while waiting for that final assistant response. In the announce flow, add bounded terminal idempotency keyed by the existing stable announce identity from `childSessionKey + childRunId` so the same child terminal result cannot be delivered twice; preserve existing gateway idempotency and retry behavior. Update `src/agents/subagent-announce-output.test.ts` and `src/agents/subagent-announce.test.ts` with regression coverage for tool-use continuation and duplicate terminal delivery. Add a single `CHANGELOG.md` Fixes entry crediting @saphoroth if the maintainer treats this as user-visible. Do not add a config option, change channel/plugin routing policy, or broaden the agents architecture.
