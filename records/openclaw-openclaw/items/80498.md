## Summary

Keep open: current main still has a source-reproducible premature completion-output path and no local terminal announce dedupe at the delivery boundary. Open related PRs now target the same bug family, especially https://github.com/openclaw/openclaw/pull/81621 for tool-boundary completion handling and https://github.com/openclaw/openclaw/pull/81184 for same-key duplicate dispatch, but neither is merged.

## What This Changes

The issue asks OpenClaw to treat assistant text on tool-use turns as progress until a later final assistant message and to dedupe terminal subagent announcements by child session/run identity.

## Best Possible Solution

Review and land a focused fix through the open subagent completion PRs, then close this issue with the merged commit and release provenance once current main actually enforces final-output selection and terminal announce idempotency.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: current tests already show non-`sessions_yield` assistant text with a tool call being returned as completion output, and current delivery code can be invoked repeatedly with the same stable announce key without a local delivered guard.

## Solution Assessment

Yes. The narrow maintainable direction is to fix `src/agents` completion-output selection and delivery idempotency, with regression tests for tool-use continuation and same `childSessionKey + childRunId` terminal delivery.
