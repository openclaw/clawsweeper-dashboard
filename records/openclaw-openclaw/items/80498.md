## Summary

Keep open: current main still has the premature-output path for non-`sessions_yield` tool-use turns, and the related hardening PR is open but not merged.

## What This Changes

Reports that subagent completion handling can treat tool-use progress as terminal output and can emit duplicate terminal announcements for the same child session/run.

## Best Possible Solution

Land a focused agents-runtime fix that treats tool-use assistant text as non-terminal until a later no-tool assistant reply and guards terminal announcements by child-run identity.

## Reproduction Assessment

Partially yes: the premature-output half is source-reproducible on current main through `readSubagentOutput` and its current unit fixture. The duplicate-delivery half remains plausible from source inspection but still needs a focused regression that exercises repeated terminal delivery for the same child session/run.

## Solution Assessment

Yes for the proposed direction: the narrow fix belongs in the agents announcement output and delivery flow, with regression tests for non-yield tool boundaries and terminal idempotency. The open related PR should be reviewed or extended before this issue closes.
