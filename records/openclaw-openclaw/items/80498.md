## Label Justifications

- `impact:message-loss`: The reported failure can deliver stale or duplicate completion output instead of the child run's real final result.
- `impact:session-state`: The bug affects parent/child run completion state and can advance the parent session on non-terminal child progress.

## Summary

Keep open: current main still has a source-reproducible path where assistant text on a non-`sessions_yield` tool-use turn is selectable as child completion output. Duplicate delivery is partly guarded by stable idempotency and delivered-state checks, but the exact `childSessionKey + childRunId` duplicate claim still needs the focused repair path already represented by https://github.com/openclaw/openclaw/pull/81621.

## What This Changes

Fix subagent completion announcements so tool-use progress is not promoted to terminal child output and terminal results are announced once per child session/run.

## Best Possible Solution

Review and land a focused agents-runtime fix, likely through https://github.com/openclaw/openclaw/pull/81621, that waits for a no-tool final assistant turn after non-yield tool use and adds terminal announce idempotency coverage.

## Reproduction Assessment

Yes from source inspection, but not from an executed live repro in this read-only pass. Current main records generic tool-use assistant text as completion output, and the current test suite still expects that behavior.

## Solution Assessment

Yes, the requested fix direction is the narrow maintainable path: treat non-final tool-use text as progress, then add idempotency and regression coverage around terminal child results. The open related PR should be reviewed before starting a separate repair.
