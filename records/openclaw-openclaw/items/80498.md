## Summary

Keep open: current main still has the reported source path for premature tool-use completion output and the duplicate-delivery half is tracked by open, not-yet-merge-ready PR work.

## What This Changes

The issue requests subagent completion fixes so tool-use progress text is not selected as terminal output and the same child run cannot announce completion more than once.

## Best Possible Solution

Land a consolidated agents-runtime fix, likely by repairing the open tool-boundary and duplicate-delivery PRs, so terminal output only comes from a later assistant message without tool calls and same-run completion delivery is boundedly idempotent.

## Reproduction Assessment

Yes at source level: current main has a unit-fixture path where a non-`sessions_yield` assistant tool-use turn with text is returned as child output, and the duplicate-delivery path has a stable key without a local dispatch guard. I did not execute a live current-main repro.

## Solution Assessment

Yes, the proposed direction is the right narrow fix: update `src/agents/subagent-announce-output.ts` to wait for a later non-tool assistant completion and add bounded same-run announce idempotency at the delivery boundary. The current open PRs need their review blockers resolved before this issue can close.
