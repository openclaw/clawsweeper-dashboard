Update the agents announce output selector to wait after any assistant tool-call turn until a later assistant message without tool calls, and add bounded terminal announce-id dedupe with regression coverage for direct and queued delivery.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/80498 in the agents runtime. Keep the change scoped to subagent completion announcement output and terminal announce idempotency. In `src/agents/subagent-announce-output.ts`, assistant messages that include any tool call should be treated as progress, not terminal completion output, including non-`sessions_yield` tool calls; after such a turn, wait for a later assistant message without tool calls before selecting final output, and do not promote intervening tool-result text as terminal completion while waiting. In `src/agents/subagent-announce.ts` and related announce delivery/queue helpers, add bounded in-memory terminal announce idempotency keyed by the existing announce id built from `childSessionKey + childRunId`, covering direct and queued terminal paths without changing unrelated queue policy or public config. Update/add focused Vitest coverage in the named agents tests for non-yield tool-use continuation, tool-result suppression while waiting, final assistant selection after tools, and duplicate terminal announce suppression for the same child run. Do not add a config option or broaden plugin/channel behavior.
