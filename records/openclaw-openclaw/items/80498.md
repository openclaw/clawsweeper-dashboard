## Summary

Keep open: current main still has a source-reproducible path where assistant text on a non-`sessions_yield` tool-use turn is selected as completion output. The related open PR is relevant, but its diff does not fully cover this exact assistant-text-on-tool-call case or the requested terminal announce idempotency guard.

## What This Changes

The issue asks OpenClaw to avoid premature subagent completion output from tool-use turns and deduplicate terminal completion announcements by child run identity.

## Best Possible Solution

Fix `src/agents` announcement output selection so tool-call assistant turns remain progress until a later no-tool assistant completion, then add terminal announce idempotency coverage keyed by the existing child-run announce identity.

## Reproduction Assessment

Partially yes. The premature-output half has a high-confidence source and unit-fixture path on current main; the duplicate-delivery half still needs a focused regression that exercises repeated terminal delivery for the same child session and run id.

## Solution Assessment

Yes. The requested direction is the narrow maintainable fix: keep the behavior in `src/agents` output selection and announce delivery, reuse the existing announce-id machinery, and update the regression tests that currently assert the bad tool-use text behavior.
