## Summary

Keep open: current main still has a source-reproducible path where non-`sessions_yield` assistant tool-use text is selected as terminal subagent output, and the same announce-id path lacks a clear local terminal dedupe guard before direct or queued delivery.

## What This Changes

The issue requests that subagent completion output ignore progress text attached to tool-use turns and dedupe terminal announcements for the same child session/run.

## Best Possible Solution

Change the agents announce output and delivery flow so tool-use turns are treated as progress until a later assistant message without tool calls, and guard terminal direct/queued announcements by the existing child-run announce identity.

## Reproduction Assessment

Partially. The premature-output half has a high-confidence current-main source path and unit fixture; the duplicate-delivery half is plausible from the announce-id and queue paths but still needs a focused regression rather than only inspection.

## Solution Assessment

Yes. The proposed direction is the narrowest maintainable fix: keep it inside `src/agents` output selection and announce delivery, reuse the existing announce-id machinery, and add regression coverage for tool-use continuation plus terminal idempotency.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/80498 as a narrow agents-runtime bug. In `src/agents/subagent-announce-output.ts`, treat any assistant message that contains tool calls as progress rather than terminal completion output, including non-`sessions_yield` tools, and wait for a later assistant message without tool calls before selecting final output; do not promote intervening tool-result text as terminal while waiting for that continuation. In `src/agents/subagent-announce.ts` and/or the announce queue/delivery seam, add bounded terminal idempotency keyed by the existing `announceId` built from `childSessionKey + childRunId`, covering both direct and queued/fallback delivery without changing provider/channel behavior. Update `src/agents/subagent-announce-output.test.ts` and `src/agents/subagent-announce.test.ts` with focused regressions for non-yield tool-use progress, later final assistant output, no tool-result promotion while waiting, and duplicate terminal delivery for the same child run. Keep the fix in `src/agents`; do not broaden into PI subscription prelude filtering, `sessions_send`, stale session-history contamination, or channel/provider routing. Add a `CHANGELOG.md` fix entry if the final behavior is user-visible.
