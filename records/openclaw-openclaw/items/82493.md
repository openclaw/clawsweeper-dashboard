## Summary

Keep open: the report is plausible and current source still has a Feishu card-reference path that can collapse model-visible content to `[Interactive Card]`. The non-streaming card parser handles inline card bodies, but Feishu streaming cards are sent as `card_id` references and quoted-message hydration feeds that parsed content into the agent context.

## What This Changes

The issue asks for subagent completion events delivered through Feishu/interactive-card paths to include model-visible completion text for non-Claude models.

## Best Possible Solution

Preserve rich Feishu card rendering while guaranteeing a plain-text fallback for model context whenever an interactive card or subagent completion is quoted, fetched, or routed internally.

## Reproduction Assessment

No live high-confidence reproduction was run. Source inspection shows a credible current-main path: Feishu Card Kit `card_id` interactive messages parse to `[Interactive Card]`, and quoted message hydration can place that value into the agent context.

## Solution Assessment

The best fix is not model-aware Claude/non-Claude branching. A narrower, maintainable solution is to make Feishu interactive-card hydration and subagent completion handoff always provide model-visible fallback text while retaining native cards for users.

## Repair Work Prompt

Fix the Feishu interactive-card model-context fallback for subagent completion/quoted-card flows. Current source sends Feishu streaming cards as Card Kit references (`{ type: "card", data: { card_id } }`), while `getMessageFeishu` only extracts inline card elements and otherwise returns `[Interactive Card]`; quoted message hydration can pass that placeholder into the model context. Keep rich Feishu card rendering intact. Prefer a channel-owned fallback that preserves or retrieves final card text for model-visible contexts, and add regression coverage for Card Kit `card_id` interactive messages plus existing inline card extraction. Do not add model-family branching, do not change generic subagent dispatch semantics, and do not weaken Feishu allowlist/security behavior. Add a `CHANGELOG.md` fix entry if this repair branch is maintainer-owned.
