## Summary

Keep open: the ordinary subagent announce path already carries text, but current rich source-reply handling can preserve presentation/interactive/channelData-only payloads while the durable transcript/model-visible mirror only records text or media names. That leaves a plausible current-main gap for non-Claude coordinators when a completion is represented as a rich card rather than plain text.

## What This Changes

The issue asks OpenClaw to make subagent completion content readable to non-Claude coordinator models when completion output is represented as an interactive card or other rich payload.

## Best Possible Solution

Add a model-agnostic text fallback for rich subagent completion/source-reply payloads at the handoff and transcript mirror boundary, reusing existing presentation/interactive fallback helpers before considering any model-specific formatting branch.

## Reproduction Assessment

Yes at source level for the likely rich-payload variant: current main preserves presentation/interactive/channelData-only payloads but mirrors only text or media names into durable model-visible history. I did not establish a live Feishu plus DeepSeek failing run in this read-only review.

## Solution Assessment

Yes in direction: a plain-text fallback alongside the rich payload is the narrowest maintainable fix because it preserves the rich card UI while making the completion visible to all model runtimes. Model-specific card selection should be avoided unless a maintainer decides the generic fallback is insufficient.
