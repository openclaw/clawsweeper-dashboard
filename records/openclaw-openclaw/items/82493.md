## Summary

Keep open: the report describes a current, source-backed regression path where rich-only subagent completion payloads can be delivered without a model-visible text mirror, which can leave non-Claude requester models with only an opaque card placeholder.

## What This Changes

The issue asks for subagent completion handoffs to include readable completion text for non-Claude requester models instead of only an Interactive Card placeholder.

## Best Possible Solution

Add a generic plain-text fallback at the source-reply/transcript-mirror boundary for rich-only subagent completion payloads, preserving channel cards while ensuring the requester model always sees the completion text.

## Reproduction Assessment

No live high-confidence reproduction was run. Source inspection shows a plausible current-main path: rich-only `presentation` or `interactive` source replies are allowed with no text and the transcript mirror only receives `payload.text` when present.

## Solution Assessment

No, leaving the completion as rich-only card data is not the best fix for an internal model handoff. The safer maintainable fix is a channel-agnostic text fallback in the payload or mirror path rather than Claude-specific branching.
