## Summary

Keep open: current main still has a credible source-backed regression path where rich-only subagent completion payloads can be delivered without a model-visible text mirror, leaving non-card-aware requester models with only an opaque interactive-card placeholder.

## What This Changes

The issue asks OpenClaw to make subagent completion text readable to non-Claude requester models when completion delivery uses rich interactive-card payloads.

## Best Possible Solution

Add a channel-agnostic plain-text fallback at the rich source-reply or transcript-mirror boundary so subagent completion content remains visible to every requester model while retaining card presentation for capable clients.

No, not as a live high-confidence reproduction in this review. Source inspection shows a credible current-main path: rich-only completion payloads can carry presentation or interactive data without text, and Feishu can surface [Interactive Card] when no extractable text is present.

## Solution Assessment

Yes, a generic text fallback before model context injection is the narrowest maintainable direction. It avoids Claude/non-Claude branching while keeping channel cards intact.
