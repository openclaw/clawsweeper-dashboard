## Summary

Keep open for maintainer review: the patch is a focused gateway auth hardening change, but it changes the existing default lockout/upgrade contract and needs an explicit upgrade path/docs plus real behavior proof before merge.

## What This Changes

The PR makes the general gateway auth rate limiter unconditional with built-in defaults, disposes it unconditionally on shutdown, and adds a WebSocket regression test for remote no-Origin auth failures.

## Best Possible Solution

Land only after maintainers choose the default-on policy and the branch documents and validates upgrade behavior, ideally with a clear operator opt-out or an explicit accepted-risk rationale.

## Reproduction Assessment

## Solution Assessment

Unclear as-is: the code path is narrow and likely enforces the intended throttle, but unconditional default-on behavior changes a shipped opt-in posture and current docs still advertise configured-only semantics. A safer solution needs maintainer-approved default policy plus docs, upgrade proof, and possibly an opt-out.
