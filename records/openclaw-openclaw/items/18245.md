## Summary

Keep open. Current `main` does not implement the requested browser-side credential firewall/domain-pinned secret injection. The browser tool still exposes only generic `type`/`fill` actions that take plaintext values from tool input, the Playwright path forwards those raw strings into `locator.fill()`/`locator.type()`, there is no `fill_credential` surface anywhere in the repo, and the docs still recommend manual login specifically to avoid giving the model credentials.

## Evidence

- **No credential-firewall tool on main:** Repo search for `fill_credential` returned no matches in `src`, `extensions`, `docs`, or `test`. That strongly indicates the proposed atomic secret-injection tool was not added.
  - command: `rg -n "fill_credential" extensions/browser/src src docs test || true`
  - sha: [a59d1bd46d9f](https://github.com/openclaw/openclaw/commit/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3)
- **Browser tool schema only exposes generic typing/fill actions:** The browser tool supports `act` kinds such as `type` and `fill`, but no credential-specific action. The schema also accepts raw `text` and `fields` payloads.
  - file: [extensions/browser/src/browser-tool.schema.ts:4](https://github.com/openclaw/openclaw/blob/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3/extensions/browser/src/browser-tool.schema.ts#L4)
  - sha: [a59d1bd46d9f](https://github.com/openclaw/openclaw/commit/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3)
- **Browser action types carry plaintext values:** `BrowserActRequest` defines `kind: "type"` with a required `text: string`, and `kind: "fill"` with `fields` whose entries carry `value?: string | number | boolean`.
  - file: [extensions/browser/src/browser/client-actions.types.ts:19](https://github.com/openclaw/openclaw/blob/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3/extensions/browser/src/browser/client-actions.types.ts#L19)
  - sha: [a59d1bd46d9f](https://github.com/openclaw/openclaw/commit/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3)
- **Runtime forwards raw values into DOM fill/type:** `typeViaPlaywright()` takes `opts.text` and passes it directly to `locator.type()`/`locator.fill()`. `fillFormViaPlaywright()` converts each `field.value` to a string and passes it directly to `locator.fill()`. The dispatcher routes `action.text` and `action.fields` straight into those functions.
  - file: [extensions/browser/src/browser/pw-tools-core.interactions.ts:700](https://github.com/openclaw/openclaw/blob/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3/extensions/browser/src/browser/pw-tools-core.interactions.ts#L700)
  - sha: [a59d1bd46d9f](https://github.com/openclaw/openclaw/commit/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3)
- **Current docs still prescribe manual login instead of automated credential injection:** Browser login docs say to sign in manually in the host browser and explicitly say not to give the model credentials. That is the opposite of the requested shipped feature.
  - file: [docs/tools/browser-login.md:11](https://github.com/openclaw/openclaw/blob/a59d1bd46d9fc0bd2d9194b3f17db2561efc51e3/docs/tools/browser-login.md#L11)
