## Summary

Keep open: current main still has Google Chat paths that can propagate an inbound thread despite `replyToMode: "off"`, and an open overlapping PR needs maintainer reconciliation before this issue can close.

## What This Changes

Google Chat should honor `channels.googlechat.replyToMode: "off"` by suppressing implicit thread replies instead of forcing responses into Chat threads.

## Best Possible Solution

Land one Google Chat-owned fix that suppresses only implicit inbound thread propagation when the resolved account mode is `off`, preserves explicit valid thread targets, and adds monitor/outbound regression coverage.

## Reproduction Assessment

Yes, source-level. Configure Google Chat with `replyToMode: "off"` and process an inbound webhook with `message.thread.name`; current main still has monitor, typing-placeholder, and outbound adapter paths that can carry that thread through.

## Solution Assessment

Yes in direction, but it is not implemented on main yet. The maintainable path is a narrow Google Chat boundary fix that filters implicit thread sources for off mode while preserving explicit thread targets and reconciling the active overlapping PR.
