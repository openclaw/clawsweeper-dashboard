## Label Justifications

- `P1`: The issue breaks an existing Google Chat channel workflow for real users who configured off mode to avoid threaded replies.
- `impact:message-loss`: Replies can be delivered into unintended Google Chat threads instead of the configured flat conversation, which is message misrouting.

## Summary

Keep open: current main and v2026.6.8 still pass Google Chat thread metadata through send boundaries despite off-mode semantics, and the open closing PR has not landed yet.

## What This Changes

Google Chat should honor `channels.googlechat.replyToMode: "off"` by suppressing implicit thread metadata for typing, text, media, and durable replies.

## Best Possible Solution

Land or repair one Google Chat-owned fix path that clears implicit thread targets when off mode is configured, without adding a duplicate config surface.

## Reproduction Assessment

Yes. The source path shows Google Chat thread metadata still reaches `sendGoogleChatMessage` despite off-mode semantics, and the issue discussion includes a deployed-user confirmation.

## Solution Assessment

Yes. The best path is to finish the existing focused Google Chat delivery PR or an equivalent maintainer-chosen branch rather than opening another duplicate repair.
