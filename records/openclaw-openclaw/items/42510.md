## Summary

Keep open: current main still has Google Chat paths that can propagate an inbound thread despite `replyToMode: "off"`, and the remaining fix is already represented by an open overlapping Google Chat threading PR that needs maintainer reconciliation rather than cleanup closure.

## What This Changes

The issue requests that Google Chat honor `channels.googlechat.replyToMode: "off"` by suppressing implicit thread replies the same way Slack does.

## Best Possible Solution

Land one Google Chat-owned fix that suppresses only implicit inbound thread propagation when the resolved account mode is `off`, preserves explicit thread targets, and adds monitor/outbound regression coverage.

## Reproduction Assessment

Yes, source-level. Configure Google Chat with `replyToMode: "off"` and process an inbound webhook with `message.thread.name`; current main still has monitor and outbound paths that can carry that thread through.

## Solution Assessment

Yes in direction, but it is not shipped yet. The narrow maintainable solution is to filter implicit Google Chat thread sources at the plugin/reply boundary while preserving explicit reply directives and reconciling the active PR.
