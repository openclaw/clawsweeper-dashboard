- `P1`: The issue breaks an existing Google Chat channel workflow for real users who configured off mode to avoid threaded replies.

## Summary

This issue is still necessary: current main and v2026.6.8 still contain Google Chat delivery paths that can send thread metadata despite off-mode semantics, and an open closing PR already owns the focused fix path.

## What This Changes

The issue asks Google Chat to honor `channels.googlechat.replyToMode: "off"` by suppressing implicit thread replies for text, media, typing, and durable reply delivery.

## Best Possible Solution

Land or repair the existing Google Chat-owned fix path so off mode clears implicit thread targets consistently without adding another config surface.
