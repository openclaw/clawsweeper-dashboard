## Summary

Keep open: current main and the latest release still let Google Chat thread metadata reach send boundaries despite `replyToMode: "off"`, and the open closing PR at https://github.com/openclaw/openclaw/pull/89790 has not merged yet.

## What This Changes

The issue requests that Google Chat honor the existing `channels.googlechat.replyToMode: "off"` setting by suppressing thread replies for text, media, typing, and durable reply delivery.

## Best Possible Solution

Review and land the existing focused fix PR, or close that PR if review finds a better same-boundary repair, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes. The source path is clear: Google Chat accepts `replyToMode`, but current send and monitor delivery paths still pass thread metadata to `sendGoogleChatMessage` without an off-mode guard.

## Solution Assessment

Yes, directionally. The maintainable fix should honor the existing reply policy at Google Chat delivery boundaries without adding a new config option, and https://github.com/openclaw/openclaw/pull/89790 is the current implementation candidate.
