## Label Justifications

- `P1`: This breaks an existing Google Chat channel workflow for users who configured off mode to avoid threaded replies.
- `impact:message-loss`: Replies can land in unintended Google Chat threads instead of the configured flat conversation, which is message misrouting.

## Summary

Keep open: current main and v2026.6.10 still allow Google Chat thread metadata to reach send boundaries despite off-mode semantics, and the open closing PR has not merged yet.

## What This Changes

Google Chat should honor `channels.googlechat.replyToMode: "off"` by suppressing threaded reply targets instead of forcing replies into Chat threads.

## Best Possible Solution

Land or replace the existing open Google Chat fix PR so off-mode clears implicit thread targets without adding a duplicate config surface.

## Reproduction Assessment

Yes. Current source shows Google Chat thread metadata still reaches `sendGoogleChatMessage` despite off-mode semantics, and the issue discussion includes deployed-user confirmation on v2026.3.28.
