## Summary

This issue is still necessary: current main and v2026.6.8 still carry Google Chat thread metadata through outbound delivery despite `replyToMode: "off"`, and an open closing PR already owns the repair path.

## What This Changes

The issue requests that Google Chat suppress thread replies when `channels.googlechat.replyToMode` is set to `"off"` instead of always sending replies with Google Chat thread metadata.

## Best Possible Solution

Repair and land, or explicitly reject, https://github.com/openclaw/openclaw/pull/89790 so Google Chat off-mode suppresses implicit thread targets across typing, text/media, and durable final delivery without adding another config surface.
