## Label Justifications

- `P1`: This breaks an existing Google Chat channel workflow for users who configured off mode to avoid threaded replies.
- `impact:message-loss`: Replies can be routed into unintended Google Chat threads instead of the configured flat conversation, which is message misrouting.

## Summary

Keep open: current main and v2026.6.10 still let Google Chat thread targets reach outbound send paths despite `replyToMode: "off"`, and https://github.com/openclaw/openclaw/pull/89790 is the open closing fix candidate but has not merged.

## What This Changes

Google Chat should respect `channels.googlechat.replyToMode: "off"` by suppressing automatic thread targets for text, media, typing, and durable reply delivery.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/89790 or a maintainer-equivalent branch, then close this issue once the fix is on main.

## Reproduction Assessment

Yes. Source inspection shows a Google Chat group thread name can populate reply metadata and reach `sendGoogleChatMessage`, where any thread value becomes a threaded send.

## Solution Assessment

Yes. The narrow maintainable path is to finish https://github.com/openclaw/openclaw/pull/89790 or an equivalent branch rather than opening another duplicate repair.
