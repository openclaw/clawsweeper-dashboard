## Summary

Keep open: current main still has Google Chat paths that can propagate an inbound thread despite `replyToMode: "off"`, and open Google Chat threading PRs need maintainer reconciliation rather than cleanup closure.

## What This Changes

The issue reports that Google Chat ignores `channels.googlechat.replyToMode: "off"` and still sends replies with thread IDs.

## Best Possible Solution

Land one Google Chat-owned fix that suppresses only implicit thread propagation when the resolved account mode is `off`, preserves explicit reply targets, and includes monitor/outbound regression coverage.

## Reproduction Assessment

Yes, source-level. Configure a Google Chat account with `replyToMode: "off"` and process an inbound webhook with `message.thread.name`; current main can still pass that thread through monitor context, the typing placeholder, or outbound delivery paths.

## Solution Assessment

Yes in direction, but it is not shipped yet. The narrow maintainable solution is to filter implicit Google Chat thread sources at the plugin/reply boundary while preserving explicit reply directives and reconciling the active PRs.
