## Summary

Keep open: current main and the latest release still have a source-level Google Chat threading gap, and https://github.com/openclaw/openclaw/pull/89790 owns the fix path but is still open and conflicting.

## What This Changes

The issue reports that Google Chat ignores `channels.googlechat.replyToMode: "off"` and still sends replies into Google Chat threads.

## Best Possible Solution

Refresh and land the canonical Google Chat delivery fix so `replyToMode: "off"` clears thread metadata across typing placeholders, durable final delivery, text sends, and media sends.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a threaded Google Chat group message with `replyToMode: "off"` still creates a typing placeholder with the inbound thread before reply filtering runs.

## Solution Assessment

Yes. The right solution is a narrow Google Chat delivery-boundary fix that clears stale thread metadata consistently instead of adding a new config option or changing shared reply semantics.
