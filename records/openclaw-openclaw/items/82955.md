## Label Justifications

- `P2`: This is a bounded installer security hardening PR with real user impact but no current core runtime outage.
- `merge-risk: 🚨 compatibility`: The strict first-byte shebang rule can reject future upstream setup scripts that Bash might otherwise execute.
- `merge-risk: 🚨 availability`: The installer can intentionally stop before Homebrew or NodeSource setup when a downloaded response is malformed or no longer matches the strict script shape.

## Summary

This PR is still useful and should stay open: current `main` still executes the downloaded Homebrew and NodeSource setup scripts without structural validation, while the branch adds a focused fail-closed validation step with strong terminal proof. The remaining blocker is installer-owner acceptance of the strict first-line shebang compatibility contract, not a definite code defect.

## What This Changes

The branch adds `validate_downloaded_script()` to `scripts/install.sh`, calls it before Homebrew and NodeSource setup scripts are executed, and adds `scripts/test-install-validation.sh` as a shell validation harness.

## Best Possible Solution

Land centralized downloaded-script validation once installer owners accept the strict shebang contract, or adjust it to allow narrowly documented safe prefixes while still rejecting empty, HTML, JSON, binary, and diagnostic-injection payloads.

## Reproduction Assessment

Yes. Source inspection of current `main` shows the Homebrew and NodeSource download paths execute temp files with Bash without structural validation, and the PR body supplies after-fix terminal proof against malformed responses.

## Solution Assessment

Yes, with maintainer acceptance of the compatibility contract. A single shared validator at the download-before-execute boundary is the maintainable shape, but the exact strictness of the shebang rule is an installer policy choice.
