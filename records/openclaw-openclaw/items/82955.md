## Summary

Keep open for maintainer review rather than cleanup close: current main still executes the downloaded installer bodies without structural validation, and this PR is a focused hardening change with sufficient terminal proof, but it intentionally changes installer compatibility by failing closed on empty or non-shebang responses.

## What This Changes

Adds a reusable installer helper that rejects empty or non-shebang downloaded scripts before Homebrew and NodeSource execution, plus a standalone shell validation harness.

## Best Possible Solution

Land the scoped structural validator after maintainer acceptance of the fail-closed installer behavior and focused installer validation; treat signed or checksum-pinned script provenance as separate trust-model work.

## Reproduction Assessment

## Solution Assessment

Yes, with maintainer acceptance of the compatibility tradeoff. A shared non-empty/shebang validator with sanitized diagnostics is a narrow hardening path; signed or checksum-pinned inputs would be broader trust-model work.
