## Summary

Keep this PR open: the installer hardening is useful and the contributor supplied real terminal proof, but the proposed validator still checks the shebang after Bash has transformed the downloaded bytes, leaving a byte-level bypass in a security-sensitive install path.

## Best Possible Solution

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Check raw bytes before accepting shebang:** `scripts/install.sh:98-99`
  - body: This reads the downloaded header through Bash command substitution before deciding whether it starts with `#!`. Bash drops NUL bytes during command substitution, so a response that begins with `\0#!/bin/bash` is normalized to `#!/bin/bash` and passes the validator even though the raw file does not start with a shebang; check the first two raw bytes directly before any string assignment and add that regression.
  - confidence: 0.92
- **[P3] Use a portable control-byte assertion:** `scripts/test-install-validation.sh:140`
  - body: The new harness uses `grep -P`, which is GNU-specific and fails on the BSD grep shipped with macOS, one of the installer targets. Use a portable byte check for the C1 diagnostic assertion so maintainers can run this proof on both macOS and Linux.
  - confidence: 0.84
