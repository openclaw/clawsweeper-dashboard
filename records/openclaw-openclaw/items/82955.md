## Summary

Keep this PR open: current main still has the unvalidated remote-script execution path, and the branch is a useful hardening change, but it is not merge-ready because the shebang check can accept content after shell command substitution has altered the raw bytes, and the new harness has a macOS portability issue.

The PR adds `validate_downloaded_script()` before Homebrew and NodeSource setup-script execution, plus a shell harness for invalid downloads and sanitized diagnostics.

## Best Possible Solution

Keep the installer hardening direction, but make the shebang decision from raw file bytes, keep diagnostics sanitized, make the harness portable, and then have maintainers explicitly accept the stricter fail-closed installer policy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Check raw bytes before accepting shebang:** `scripts/install.sh:98-99`
  - body: `first_line` is populated through command substitution, so Bash can discard NUL bytes before the `#!` prefix check runs. A corrupt or hostile file with leading NUL/control bytes before `#!` can be accepted even though the downloaded file does not actually start with a shebang; read and compare the first two raw bytes before executing.
  - confidence: 0.9
- **[P3] Use a portable control-byte assertion:** `scripts/test-install-validation.sh:140`
  - body: `grep -P` is not available in the default BSD grep on macOS, so the new harness can fail on a supported developer platform even when the sanitizer is correct. Use a portable byte check such as `od`/`awk` or another repo-supported helper instead.
  - confidence: 0.82
