## Summary

Keep this PR open: current main and the latest release still execute downloaded Homebrew and NodeSource setup scripts without structural validation, and the proposed patch is a focused hardening change with sufficient terminal proof. There are no blocking correctness findings, but installer owners should explicitly accept the strict first-line shebang fail-closed behavior before merge.

## What This Changes

The branch adds a reusable shell-script validation helper before Homebrew and NodeSource setup-script execution and adds a standalone shell harness for malformed download cases.

## Best Possible Solution

Land centralized downloaded-script validation after installer owners accept the strict shebang contract, or narrowly allow documented safe prefixes and wire the malformed-response cases into normal installer validation.

## Solution Assessment

Yes, with maintainer acceptance. Central validation before remote-script execution is the narrow maintainable fix; the unresolved choice is how strict the shebang policy should be for third-party installer availability.
