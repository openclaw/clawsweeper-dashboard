## Summary

Keep open. This PR addresses a real installer hardening gap in remote script execution, but the strict first-line shebang rule changes how installs fail when Homebrew or NodeSource responses do not match the expected script shape, so it needs installer-owner acceptance rather than cleanup closure or automated repair.

## What This Changes

The branch adds centralized downloaded-script validation to `scripts/install.sh` for Homebrew and NodeSource setup scripts, plus a standalone malformed-download shell harness.

## Best Possible Solution

Land centralized downloaded-script validation after installer owners accept the strict shebang contract, or narrowly allow known-safe prefixes if they want a softer availability posture.

## Reproduction Assessment

Yes from source inspection: current main and `v2026.5.22` download Homebrew and NodeSource setup scripts to temp files and execute them with bash without validating content. The PR body also includes after-fix terminal proof against malformed responses.

## Solution Assessment

Yes, with maintainer acceptance. Centralized validation is a narrow maintainable fix for malformed downloads, but installer owners need to accept or soften the strict shebang policy before merge.

- The malformed-download coverage is a standalone shell harness rather than part of the existing `test/scripts/install-sh.test.ts` lane, so normal CI may not preserve the behavior unless maintainers wire it in.
