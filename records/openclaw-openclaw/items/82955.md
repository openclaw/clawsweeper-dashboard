## Summary

Keep open for maintainer review: the patch addresses a real installer hardening gap and has sufficient terminal proof, but merging intentionally changes installer failure behavior for third-party setup scripts and should be accepted by installer/security owners.

## What This Changes

The PR adds downloaded-script validation to `scripts/install.sh` before Homebrew and NodeSource setup scripts execute, plus a standalone malformed-download shell harness.

## Best Possible Solution

Land centralized downloaded-script validation once installer/security owners accept the strict shebang contract, or narrowly allow reviewed safe prefixes while still rejecting empty, HTML, JSON, binary, and diagnostic-injection payloads.

## Reproduction Assessment

Yes from source inspection. Current main and `v2026.5.22` download Homebrew and NodeSource setup scripts into temp files and invoke Bash without content validation.

## Solution Assessment

Yes, centralized validation before execution is the narrow maintainable fix. The remaining question is whether strict first-byte shebang validation is the installer contract maintainers want.
