## Summary

Keep open: this is a focused installer hardening PR with sufficient terminal proof and no blocking production-code finding, but the added standalone harness still has a portable-shell bug and the stricter fail-closed installer behavior needs maintainer acceptance.

## What This Changes

The PR adds `validate_downloaded_script()` to reject empty or non-shebang downloaded installer scripts before Homebrew and NodeSource setup execution, plus a standalone validation harness.

## Best Possible Solution

Keep the strict structural validation if maintainers accept the install-time fail-closed policy, but replace the harness C1-byte assertion with a POSIX-portable check before merge.

## Reproduction Assessment

Yes. Source inspection of current main shows the affected installer paths download temp files and execute them with `bash` without checking content first; the PR body also supplies terminal proof for malformed responses.

## Solution Assessment

Mostly yes. Non-empty plus shebang validation is a narrow maintainable hardening step, but the harness should avoid `grep -P` and maintainers should consciously accept the fail-closed install behavior.
