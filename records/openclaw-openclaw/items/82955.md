## What This Changes

The PR adds `validate_downloaded_script` to `scripts/install.sh` and a standalone shell harness so empty or non-shebang remote installer downloads are rejected before `bash` execution.

## Best Possible Solution

Land the focused installer hardening once maintainers accept the shebang-as-contract behavior, with the existing installer syntax, ShellCheck, dry-run, and smoke coverage green.

## Reproduction Assessment

Yes. Current main clearly downloads Homebrew and NodeSource setup scripts and executes the temp file with bash without validating the payload; the PR body also gives a concrete local HTTP/HTTPS server reproduction for empty, HTML, and valid-script responses.

## Solution Assessment

Yes, with a maintainer policy call. Structural non-empty plus shebang validation is a narrow maintainable hardening for these script URLs, but maintainers should own the compatibility choice that non-shebang downloads now fail closed.
