## Summary

Keep this PR open for maintainer review: current main still executes downloaded Homebrew and NodeSource setup payloads without structural validation, and the patch is a focused hardening change with no blocking code findings. The remaining decision is whether maintainers accept the stricter fail-closed installer behavior if an upstream script ever stops beginning with a raw shebang.

## What This Changes

The PR adds non-empty, raw-shebang, and sanitized diagnostic validation before executing downloaded Homebrew and NodeSource installer scripts, plus a standalone shell validation harness.

## Best Possible Solution

Land the validation if maintainers accept fail-closed installer hardening, ideally with the current upstream shebang check recorded and an installer smoke before merge if they want stronger release confidence.

## Reproduction Assessment

Yes. Source inspection on current main shows run_remote_bash and the NodeSource apt/dnf/yum branches execute downloaded temp files with bash before validating the payload shape.

## Solution Assessment

Yes, with maintainer acceptance of the stricter failure mode. A raw-byte shebang and empty-file check is a narrow hardening step for these script execution paths; checksum pinning would be brittle for frequently updated third-party setup scripts.
