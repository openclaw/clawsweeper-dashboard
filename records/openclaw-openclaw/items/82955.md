## Summary

Keep open: current main and v2026.6.10 still lack structural validation before executing downloaded Homebrew and NodeSource setup scripts, while this PR is the live candidate fix. The patch looks correct, but it intentionally changes installer behavior to fail closed for empty or non-shebang setup-script responses, so installer/security owner acceptance remains the merge decision.

## What This Changes

The PR adds a shared downloaded-script validator to `scripts/install.sh`, wires it into Homebrew and NodeSource setup-script execution, and adds shell/installer-test coverage.

## Best Possible Solution

Land this focused validation change after installer/security owner acceptance, then let https://github.com/openclaw/openclaw/issues/90013 close through the merged implementation.

## Reproduction Assessment

Yes, at source level. Current main and v2026.6.10 still pass Homebrew and NodeSource downloaded temp files to bash without structural validation; I did not run a mutating live installer repro in this read-only review.

## Solution Assessment

Yes, subject to installer/security owner acceptance. The shared validator is narrow for the four bash-executed download paths; checksum pinning appears brittle for frequently updated upstream setup scripts, and current `run_required_step` only handles failed downloads rather than malformed successful responses.
