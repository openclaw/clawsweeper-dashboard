## Summary

Keep open: the installer hardening is useful, but the added proof harness copies the validator instead of exercising the production `scripts/install.sh` function, and the new binary fixture remains probabilistically flaky. ## What This Changes

The PR adds `validate_downloaded_script` to `scripts/install.sh`, guards Homebrew and NodeSource downloaded-script execution, and adds a shell harness for invalid-download fixtures.

## Best Possible Solution

Have the harness call the production installer validator and guard paths with deterministic fixtures, then have a maintainer explicitly accept the stricter fail-closed shebang contract before merge.

## Reproduction Assessment

Yes from source inspection: current main downloads Homebrew and NodeSource setup scripts into temp files and invokes bash without structural validation. I did not run a live CDN failure reproduction in this read-only pass.

## Solution Assessment

No, not merge-ready as written: the production validator is a reasonable mitigation, but the proof harness should exercise production code and deterministic fixtures before maintainers accept the fail-closed compatibility tradeoff.
