## Summary

Keep open: current main still executes the downloaded installer setup scripts without structural validation, while this PR addresses that gap; the remaining blocker is maintainer acceptance of stricter fail-closed installer behavior.

## What This Changes

The PR adds structural validation for downloaded Homebrew and NodeSource setup scripts in `scripts/install.sh` and adds a shell harness for malformed-download cases.

## Best Possible Solution

Land the validation on top of current main if maintainers accept no-shebang or empty downloaded setup scripts as hard failures, with maintainer-owned release and changelog handling during landing.
