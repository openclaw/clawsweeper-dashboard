Full review comments:

- **[P2] Return on failed downloads before validating:** `scripts/install.sh:121`
  - body: `download_file` can return nonzero while leaving a temp file behind, but this path ignores that status and proceeds to `validate_downloaded_script`. If the failed or partial response starts with `#!`, it can still pass validation and be executed, so return immediately when the download fails.
  - confidence: 0.88
- **[P2] Guard NodeSource download failures before validation:** `scripts/install.sh:1699`
  - body: The NodeSource branches add validation after `run_quiet_step ... download_file`, but they never check whether that quiet download step failed. Guard each download step before validating or running `bash`/`sudo bash`, otherwise a failed partial setup script can continue into execution.
  - confidence: 0.85
