## Summary

Keep this PR open: the security hardening is useful, but the current patch is not merge-ready because the new rejected-response diagnostic still lets non-printable high-bit bytes from untrusted downloads reach terminal output.

## What This Changes

The PR adds a shared downloaded-script validator to `scripts/install.sh` for Homebrew and NodeSource setup-script paths, plus a standalone shell validation harness.

## Best Possible Solution

Keep the shared validator, encode or whitelist every non-printable diagnostic byte, make the validation harness deterministic, and merge only after maintainers accept strict Homebrew/NodeSource shebang enforcement.

## Reproduction Assessment

Yes, by source inspection: current main downloads Homebrew and NodeSource scripts and executes the temp files without validating that the body is a non-empty shell script. I did not run the live failing installer path, but the current code path and PR terminal proof make the bug source-reproducible with high confidence.

## Solution Assessment

No, not as submitted. The shared validator is the right narrow shape, but the diagnostic preview must be printable-safe before logging and the strict shebang compatibility tradeoff needs maintainer acceptance.
