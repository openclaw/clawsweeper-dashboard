## Summary

Keep open: the PR is useful installer hardening with good real-output proof, but it still does not make downloader failure authoritative before validation, so a failed partial download that begins with `#!` can still reach execution.

## What This Changes

The PR adds a structural validator for Homebrew and NodeSource downloaded setup scripts and adds a shell validation harness.

## Best Possible Solution

Keep the validator, but make every download-then-execute path stop on downloader failure before checking file structure, then add regression proof for failed downloads that leave partial shebang content.

## Reproduction Assessment

Yes, source-reproducible: current main downloads remote installer scripts and executes `bash "$tmp"` without structural validation. I did not run a mutating installer flow in this read-only review, but the PR body includes real terminal proof for the patched validator behavior.

## Solution Assessment

No, not yet: structural validation is the right direction, but downloader failure must be checked before validation so failed partial downloads cannot be accepted just because they start with `#!`.

- **[P2] Return on failed downloads before validating:** `scripts/install.sh:122`
  - body: `run_remote_bash` is executed through `run_quiet_step`'s `if "$@"` path, so `errexit` is not a reliable guard inside the function.   - confidence: 0.88

  - body: The NodeSource branches also validate only after the download step has returned through `run_quiet_step`. In any error-checked `install_node` call context, a failed download that leaves a partial shebang file can pass validation and then be executed by the following `bash "$tmp"` or `sudo bash "$tmp"`; guard each setup-script download before the new validation call.
  - confidence: 0.78
