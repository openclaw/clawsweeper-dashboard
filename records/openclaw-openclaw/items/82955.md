## Summary

Keep open: the installer hardening is useful and the production path looks directionally correct, but the standalone validation harness still has a rare nondeterministic binary-fixture flake. Maintainers should also explicitly accept the fail-closed shebang requirement for third-party setup scripts before merge.

## What This Changes

This PR adds downloaded-script validation and download-failure guards to the Homebrew and NodeSource installer paths, plus a standalone shell harness for bad-download cases.

## Best Possible Solution

Land the production hardening with a deterministic harness fixture and an explicit maintainer acceptance of the fail-closed shebang policy for third-party installer scripts.

## Reproduction Assessment

Yes. Current main source clearly downloads Homebrew and NodeSource scripts into temp files and executes them without structural validation; I did not run a live partial-download repro, but the unsafe path is source-reproducible.

## Solution Assessment

Yes, with one small harness repair. Download failure guards plus raw shebang/non-empty validation are a narrow maintainable hardening path, while full checksum pinning would be brittle for these frequently changing third-party setup scripts.
