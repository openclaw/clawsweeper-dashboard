## Summary

Keep open: current main still downloads and executes the Homebrew and NodeSource setup scripts without structural content validation, and this PR is a focused hardening change with sufficient terminal proof. I found no blocking correctness or security-regression finding, but installer owners should explicitly accept the strict first-line shebang fail-closed behavior before merge.

## What This Changes

The PR adds `validate_downloaded_script()` to `scripts/install.sh` before Homebrew and NodeSource setup-script execution, plus a standalone shell validation harness for malformed downloads.

## Best Possible Solution

Land centralized downloaded-script validation once installer owners accept the strict shebang policy, or narrowly broaden known-safe prefixes and add normal installer-lane coverage if they want a softer availability posture.

## Reproduction Assessment

Yes from source inspection. Current main downloads remote Homebrew and NodeSource setup scripts into temp files and invokes bash on them without structural validation; malformed non-empty responses would reach the shell path.

## Solution Assessment

Yes, with maintainer acceptance. Central validation before remote-script execution is the narrow maintainable hardening path; the only unresolved decision is how strict the acceptable-script prefix contract should be.
