## Summary

The PR is a useful installer hardening change, but it still needs one focused fix before merge: the Homebrew `run_remote_bash` path validates after `download_file` without first honoring a failed curl/wget status, so a failed partial transfer can still reach the validation/execution path.

## What This Changes

The PR adds downloaded shell-script validation to `scripts/install.sh` before executing Homebrew and NodeSource setup scripts, plus a new shell validation harness.

## Best Possible Solution

Keep the structural validation hardening, but add explicit failed-download guards before validation/execution and cover that transport-failure path in the harness.

## Reproduction Assessment

Yes, for the review finding: source inspection shows `run_remote_bash` validates after an unchecked `download_file`, and the bash/curl contract checks show why a failed partial transfer can continue to that point. I did not run the PR harness because this review is read-only.

## Solution Assessment

No, not yet. The validation direction is maintainable, but the best fix must preserve the download command as the first failure boundary before validating or executing any temp-file content.

Full review comments:

- **[P2] Honor failed downloads before validation:** `scripts/install.sh:122`
  - body: `run_remote_bash` is invoked under `run_quiet_step`'s `if "$@" ...` wrapper, so `set -e` does not stop the function when `download_file` returns non-zero.
