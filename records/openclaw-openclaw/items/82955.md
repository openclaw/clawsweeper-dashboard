## Summary

Keep open. The installer hardening is useful and current `main` still executes downloaded Homebrew and NodeSource setup scripts without structural validation, but this PR still has a small portable-test defect and a maintainer-owned fail-closed installer behavior decision before merge.

## What This Changes

The branch adds `validate_downloaded_script` to `scripts/install.sh` before Homebrew and NodeSource setup-script execution and adds a malformed-download shell validation harness.

## Best Possible Solution

Keep the shared installer validation, replace the GNU-only harness assertion with a portable control-byte check, and have installer owners explicitly accept the fail-closed availability tradeoff before merge.

## Reproduction Assessment

Yes. Source inspection of current `main` shows downloaded Homebrew and NodeSource setup scripts are executed with bash without a non-empty or shebang check; the PR body also includes terminal proof against malformed local HTTP/HTTPS responses.

## Solution Assessment

Mostly yes. The production fix is a bounded shared validator in the implicated installer path, but the added harness should avoid GNU-only `grep -P` and installer owners still need to accept the fail-closed policy.
