## What This Changes

The branch adds `validate_downloaded_script` to `scripts/install.sh` before executing Homebrew and NodeSource setup scripts, plus a standalone shell validation harness.

## Best Possible Solution

Land centralized downloaded-script validation after installer owners accept the strict shebang contract, ideally with CI-backed malformed-download coverage if they want a long-term guardrail.

## Reproduction Assessment

## Solution Assessment

Likely yes, subject to installer-owner policy. Centralized non-empty and shebang validation at the remote-script execution boundary is a narrow maintainable fix, but the byte-zero `#!` contract should be explicitly accepted or adjusted.
