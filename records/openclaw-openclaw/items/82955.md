## Summary

Keep open: current main still executes downloaded Homebrew and NodeSource setup scripts without content validation, while this PR adds a focused validation layer and provides sufficient terminal proof. The remaining blocker is maintainer acceptance of the fail-closed shebang policy for installer downloads.

## What This Changes

The PR adds `validate_downloaded_script` to `scripts/install.sh`, applies it before executing downloaded Homebrew and NodeSource scripts, and adds a standalone shell validation harness.

## Best Possible Solution

Land the structural validation if installer owners accept the fail-closed script-shape contract, and promote the malformed-response cases into the existing installer test lane only if maintainers want durable CI coverage before merge.

## Reproduction Assessment

Yes for the code path: current main downloads Homebrew and NodeSource setup scripts to temp files and executes them without validating the content first. I did not run current main during this read-only review, but the source path and the PR's after-fix terminal proof are high-confidence.

## Solution Assessment

Yes, with a maintainer policy caveat. Non-empty plus first-line shebang validation is a narrow hardening layer and current upstream scripts satisfy it, but unconditional fail-closed behavior needs installer-owner acceptance.
