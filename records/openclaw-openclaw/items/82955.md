## Summary

Keep open: the patch is focused and source inspection supports the installer hardening need, but the contributor proof still does not convincingly show the exact HTTPS-only downloader path and this security-boundary change needs maintainer proof review before merge.

## What This Changes

The PR adds non-empty and shebang validation with sanitized first-line diagnostics to `scripts/install.sh`'s `run_remote_bash`, plus a standalone shell validation harness.

## Best Possible Solution

Land this as narrowly scoped Homebrew `run_remote_bash` hardening after exact HTTPS downloader proof, or generalize the validation helper to the NodeSource setup-script executions before calling it broad downloaded-script validation.

## Reproduction Assessment

Yes from source inspection: current main downloads the Homebrew installer into a tempfile and immediately executes it with `/bin/bash` without checking for an empty or non-shell response. I did not run a mutating installer flow in this read-only review.

## Solution Assessment

Mostly yes for the Homebrew helper: non-empty and shebang validation is a narrow maintainable guard and upstream Homebrew currently starts with `#!/bin/bash`. If the intended scope is all downloaded setup scripts, the same guard should be shared with the NodeSource execution paths or the PR wording should be narrowed.
