## Summary

Keep open for maintainer review: current main still executes downloaded Homebrew and NodeSource setup scripts without structural validation, and this PR is a focused hardening patch with sufficient terminal proof and no blocking code findings, but the strict shebang-only fail-closed contract needs installer/security owner acceptance before merge.

## What This Changes

The PR adds downloaded shell-script validation to `scripts/install.sh` before Homebrew and NodeSource setup scripts are executed, plus a standalone shell harness for malformed and valid downloads.

## Best Possible Solution

Land centralized downloaded-script validation once installer/security owners accept the strict shebang contract, or narrowly allow reviewed harmless prefixes while still rejecting empty, HTML, JSON, binary, and diagnostic-injection payloads.

## Reproduction Assessment

Yes. Current main is source-reproducible because `scripts/install.sh` downloads temp files and invokes Bash on them directly; I did not run a current-main malformed-download server, but the execution path is direct and the PR proof exercises the patched path.

## Solution Assessment

Yes, with a maintainer tradeoff. Central validation before Bash execution is the narrow maintainable fix, while the only unresolved solution choice is whether strict first-line shebang validation should be accepted or relaxed for reviewed harmless prefixes.
