## Summary

Keep open: the PR is useful hardening with sufficient terminal proof, but it is not merge-ready because the new validation only protects the Homebrew helper path while the installer still downloads and executes NodeSource apt/dnf/yum setup scripts without the same guard.

## What This Changes

The branch adds non-empty/shebang validation and sanitized first-line diagnostics to `run_remote_bash`, plus a standalone shell validation harness.

## Best Possible Solution

Extract a reusable downloaded-shell-script validator and call it before every installer path that executes a downloaded setup script, preserving sudo/root behavior and avoiding checksum pinning.

## Reproduction Assessment

Yes, source-reproducible by inspection: current main downloads setup scripts into temp files and executes them with bash, while the PR body demonstrates malformed downloads being rejected for the guarded helper path.

## Solution Assessment

No. Guarding `run_remote_bash` is the right direction, but the maintainable fix should share the same validator with the NodeSource apt/dnf/yum setup-script paths too.
