## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Check raw header bytes before accepting shebang:** `scripts/install.sh:98-99`
  - body: `first_line` is populated through command substitution, and Bash drops NUL bytes there. A response beginning with `#!\0...` is normalized into a string that still matches `#!`, so the new validator can pass a malformed binary payload into `/bin/bash` instead of rejecting it. Compare the first two raw bytes directly and add a NUL-prefixed fixture so this stays covered.
  - confidence: 0.9
- **[P3] Use a portable C1 diagnostic assertion:** `scripts/test-install-validation.sh:140`
  - body: `grep -P` is not available on macOS/BSD grep, and because it is under `!`, an unsupported option can make the C1 assertion pass for the wrong reason. Use a portable byte check such as `od`, `perl`, or a `tr` round trip so the harness proves the sanitizer on supported installer environments.
  - confidence: 0.82
