## Summary

Keep this PR open for maintainer review: current main and the latest release still execute the Homebrew and NodeSource downloaded scripts without structural validation, and the branch is a bounded hardening fix with sufficient terminal proof. ## What This Changes

Adds `validate_downloaded_script` and download-failure guards to four `scripts/install.sh` Homebrew/NodeSource remote-script execution paths, plus a shell harness for malformed downloads and partial-download guard behavior.

## Best Possible Solution

Land this hardening once maintainers accept the fail-closed installer behavior and normal installer smoke/sync checks are green; treat checksum or signature pinning as separate future trust-model work.

## Reproduction Assessment

## Solution Assessment

Yes, this is the best bounded mitigation short of checksum or signature pinning: one shared raw-shebang validator plus download-failure guards covers all four affected paths. The remaining decision is whether maintainers accept fail-closed behavior for this published installer surface.
