## Summary

Keep this PR open: it addresses a real installer hardening gap and has strong terminal proof, but the diff still logs rejected untrusted bytes through an incomplete sanitizer before it is merge-ready.

## What This Changes

The PR adds `validate_downloaded_script()` to `scripts/install.sh`, applies it before Homebrew and NodeSource downloaded setup scripts are executed, and adds a standalone shell validation harness.

## Best Possible Solution

Keep the shared validator, encode or whitelist every non-printable diagnostic byte, make the validation harness deterministic, and merge only after maintainers accept strict Homebrew/NodeSource shebang enforcement.

## Reproduction Assessment

Yes, by source inspection: current main downloads Homebrew and NodeSource setup scripts into temp files and executes them without checking for non-empty shell-script content. I did not run the live installer, but the code path and PR proof make the bug source-reproducible with high confidence.

## Solution Assessment

Not yet; the shared validator is the right narrow direction, but it should encode every non-printable diagnostic byte and maintainer review should explicitly accept the strict third-party script-shape behavior.
