## Summary

Keep open: current main still executes downloaded Homebrew and NodeSource setup scripts without structural validation, while this PR adds a focused validation layer with sufficient terminal proof and no blocking code findings. The remaining blocker is maintainer acceptance of the stricter fail-closed installer behavior.

## What This Changes

Adds `validate_downloaded_script` to `scripts/install.sh` before Homebrew and NodeSource remote script execution, plus a standalone shell validation harness.

## Best Possible Solution

Land a current-main merge that preserves recent installer fixes and intentionally accepts fail-closed structural validation for remote setup scripts, or ask for an explicit compatibility-mode design if maintainers do not want that default.

## Reproduction Assessment

Yes. Current main has a clear source path that downloads Homebrew and NodeSource scripts and executes them without validation, and the PR body supplies terminal proof for malformed-response cases after the patch.

## Solution Assessment

Yes, with maintainer approval. Raw non-empty plus shebang validation is a narrow maintainable hardening step, while checksum pinning would be brittle for third-party scripts that update independently.
