## Summary

## What This Changes

## Best Possible Solution

Keep validation before every remote setup-script execution, then land after installer owners accept the fail-closed shebang contract or request a compatibility-preserving rule with wired regression coverage.

## Reproduction Assessment

## Solution Assessment

Yes, if maintainers accept the fail-closed installer contract. Otherwise the safer solution is to keep validation but relax the compatibility rule around known bash-compatible script formats.
