## Summary

Keep open: current main and the latest shipped tag still execute the downloaded Homebrew and NodeSource setup scripts without structural validation, while this PR adds focused fail-closed checks and I found no blocking patch defect. The remaining decision is whether maintainers accept the stricter installer behavior before merge.

## What This Changes

The branch adds `validate_downloaded_script` to `scripts/install.sh` for Homebrew and NodeSource setup downloads and adds a shell harness for malformed, control-byte, NUL-prefix, and valid-script cases.

## Best Possible Solution

Land the focused validation if maintainers accept the fail-closed installer behavior, with current-base smoke as the final proof gate if desired.

## Reproduction Assessment

Yes. Current main and v2026.5.22 show the Homebrew and NodeSource downloaded temp files are executed without structural validation; I did not run a current-main live repro because this review is read-only.

## Solution Assessment

Yes, with maintainer acceptance. The raw-byte shebang and non-empty checks are a narrow hardening for the affected download-then-execute paths, while checksum pinning would be brittle for frequently updated third-party setup scripts.
