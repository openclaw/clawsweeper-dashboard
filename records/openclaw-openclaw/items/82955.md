## Summary

Keep open: current main still executes downloaded Homebrew and NodeSource setup scripts without structural validation, while this PR adds a narrow non-empty/raw-shebang check with sanitized diagnostics and supplied real terminal proof. I found no blocking patch defect, but maintainers still need to accept the stricter fail-closed installer behavior before merge.

## What This Changes

The PR adds `validate_downloaded_script()` to `scripts/install.sh`, calls it before Homebrew and NodeSource setup-script execution, and adds a standalone shell validation harness.

## Best Possible Solution
