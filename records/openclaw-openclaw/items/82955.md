## Summary

Keep open: current main still executes Homebrew and NodeSource downloaded scripts without structural validation, and this PR is focused with sufficient terminal proof. One low-priority harness portability issue remains, and the fail-closed installer behavior needs maintainer acceptance before merge.

## What This Changes

Adds `validate_downloaded_script()` to `scripts/install.sh`, validates Homebrew and NodeSource setup-script downloads before Bash execution, and adds a Bash validation harness.

## Best Possible Solution

Land the validation after replacing the GNU-only harness check and with maintainer acceptance that malformed or non-shebang remote scripts should fail closed rather than be executed.

## Reproduction Assessment

Yes. Current main source shows the affected downloaded files are executed directly, and the PR body supplies terminal proof using the patched functions against malformed and valid local HTTP/HTTPS responses.

## Solution Assessment

Mostly yes. Non-empty plus shebang validation and sanitized diagnostics are a narrow hardening path, but the harness should be made portable and maintainers should explicitly accept the fail-closed installer behavior.
