## What This Changes

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Honor failed downloads before validating:** `scripts/install.sh:122`
  - body: `run_remote_bash` is executed through `run_quiet_step`'s `if "$@"` wrapper, so `set -e` is ignored inside the function. If curl or wget returns non-zero after leaving a partial file that still starts with `#!`, this validation passes and `/bin/bash "$tmp"` still runs the failed download. Check `download_file "$url" "$tmp" || return 1` before validating and add a harness case for a failed downloader that leaves a shebang-prefixed partial file.
  - confidence: 0.88
