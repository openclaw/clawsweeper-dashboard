## Risks / Open Questions

- The new shebang gate intentionally fails closed; if Homebrew or NodeSource publish a valid bash setup script without a leading `#!`, OpenClaw installation will stop before configuring Homebrew or Node.js.
- The added check is structural hardening, not integrity verification; syntactically valid downloaded scripts still rely on the existing HTTPS trust model.
- The standalone validation harness is useful proof but is not currently wired into installer CI, so maintainers must either accept it as manual proof or add equivalent automated coverage.
