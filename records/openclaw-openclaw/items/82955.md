Yes, with a maintainer policy call. Non-empty plus shebang validation is a narrow maintainable hardening that matches the live upstream scripts checked here, but maintainers need to own the fail-closed compatibility choice.

- This is structural validation, not integrity pinning or signature verification; syntactically valid downloaded scripts still rely on the existing HTTPS trust model.
