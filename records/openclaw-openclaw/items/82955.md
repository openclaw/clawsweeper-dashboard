## Summary

Keep this PR open for maintainer review: the patch is a focused installer hardening change with sufficient terminal proof and no blocking code findings, but it intentionally makes Homebrew/NodeSource setup-script downloads fail closed when content is empty, malformed, or not shebang-first.

## What This Changes

The PR adds downloaded shell-script validation to `scripts/install.sh` for Homebrew and NodeSource setup downloads and adds `scripts/test-install-validation.sh` coverage for malformed downloads and sanitized diagnostics.

## Best Possible Solution

Merge this as-is if maintainers accept fail-closed installer hardening; otherwise define the fallback or strict-mode policy first and test that upgrade path explicitly.

## Reproduction Assessment

## Solution Assessment

Yes, if maintainers want fail-closed installer hardening. The implementation is narrow and current upstream scripts start with `#!/bin/bash`, but the fail-closed policy still needs maintainer acceptance before merge.
