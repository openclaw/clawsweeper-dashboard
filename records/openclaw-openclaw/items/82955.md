## Summary

Keep open for maintainer review: the patch appears to harden current installer download-execute paths, and current main/latest release still lack that validation, but it intentionally changes installer compatibility by failing closed on non-shebang third-party setup-script downloads.

## What This Changes

Adds `validate_downloaded_script()` to `scripts/install.sh` before Homebrew and NodeSource setup-script execution and adds a standalone shell validation harness.

## Best Possible Solution

Land the validator if maintainers accept fail-closed handling for third-party installer scripts, with optional CI wiring for the new harness if recurring proof is required.

## Reproduction Assessment

Yes. Source inspection shows current main downloads remote installer scripts into temp files and executes them with bash without non-empty, raw-shebang, or diagnostic-safety validation; the PR body also supplies terminal proof for malformed HTTP/HTTPS responses.

## Solution Assessment

Yes, with maintainer risk acceptance. Raw shebang validation before the four download-then-execute paths is a narrow hardening step, while checksum pinning would be brittle for frequently changing Homebrew and NodeSource scripts.
