## Summary

Keep open: the PR is useful installer hardening with sufficient terminal proof, but the latest head still lets `run_remote_bash` continue after a failed downloader if the failed transfer leaves a shebang-prefixed partial file.

## What This Changes

The PR adds structural validation for downloaded Homebrew and NodeSource shell scripts in `scripts/install.sh` and adds a standalone shell validation harness.

## Best Possible Solution

Keep the validator, but require every download-then-execute path to stop on downloader failure before validation or execution and cover the failed-partial-file case in the harness.

## Reproduction Assessment

Yes. Source inspection plus the existing ClawSweeper shell-probe evidence show that a failed downloader can leave a shebang-prefixed partial temp file that still reaches `bash "$tmp"` under the PR head.

## Solution Assessment

No. The validator is the right direction, but the narrow maintainable fix is to check downloader status before validation or execution and add a failed-partial-download regression case.
