## Summary

Keep open: the patch hardens the Homebrew helper path, but the same downloaded-script execution boundary remains in the Linux NodeSource apt/dnf/yum paths, so this is not ready to merge as complete installer hardening.

## What This Changes

The branch adds non-empty/shebang validation and sanitized first-line diagnostics to `run_remote_bash`, plus a standalone shell validation harness.

## Best Possible Solution

Extract a reusable downloaded-shell-script validator and call it before every installer path that executes a downloaded setup script, preserving current sudo/root behavior and avoiding checksum pinning.

## Reproduction Assessment

Yes, by source inspection and the PR's terminal proof. Current main downloads setup scripts into temp files and executes them with bash, while the PR proof shows the patched helper rejecting bad downloaded content for the guarded path.

## Solution Assessment

No. Guarding `run_remote_bash` is the right direction, but the maintainable fix should share the validator with the NodeSource apt/dnf/yum setup-script paths too.
