## Best Possible Solution

## Risks / Open Questions

- The new validator intentionally rejects downloaded setup scripts unless byte 1 starts with `#!`; a future Homebrew or NodeSource script with a UTF-8 BOM, leading blank line, or leading comment would stop installation before setup runs.
- The open NodeSource installer PR at https://github.com/openclaw/openclaw/pull/82926 changes the same setup-script execution area, so landing order may require combining validation with its noninteractive and post-install verification changes.
- The added shell harness is useful proof but is not wired into the existing Vitest or changed-check lanes and duplicates the validation logic, so long-term regression coverage depends on maintainer choice.
