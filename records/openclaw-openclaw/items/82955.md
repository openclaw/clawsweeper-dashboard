## Summary

Keep open for maintainer review: the current installer source still executes downloaded Homebrew and NodeSource setup scripts without structural validation, and this PR is a focused hardening change with sufficient terminal proof. The remaining blocker is accepting the fail-closed first-line shebang policy for third-party installer scripts.

## What This Changes

This PR adds structural validation before executing downloaded Homebrew and NodeSource setup scripts in `scripts/install.sh` and adds a standalone shell validation harness.

## Best Possible Solution

Land the structural validation if installer owners accept the fail-closed shebang contract, with CI-backed malformed-response coverage added if maintainers want durable regression proof before merge.

Yes, at source level: current main downloads Homebrew and NodeSource setup scripts to temp files and executes them without checking the downloaded content first. I did not run the full installer end to end, but the code path and PR terminal proof make the behavior high-confidence.

## Solution Assessment

Yes, with a maintainer policy caveat. Non-empty plus first-line shebang validation and sanitized diagnostics is a narrow hardening layer, but unconditional fail-closed installer behavior needs owner acceptance.
