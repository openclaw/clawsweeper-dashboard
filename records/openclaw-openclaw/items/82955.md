## What This Changes

The PR adds non-empty/shebang validation and download-failure guards before Homebrew and NodeSource installer scripts are executed, plus a shell harness for malformed-download cases.

## Best Possible Solution

Keep the installer hardening, replace the nondeterministic binary fixture with fixed bytes, and have maintainers explicitly accept structural fail-closed validation as a mitigation rather than an integrity pin before merge.

## Reproduction Assessment

Yes. Current main source shows run_remote_bash and three NodeSource branches executing downloaded temp files without validation, and the PR body supplies terminal output from the malformed-download harness.

## Solution Assessment

Partly. Download-failure guards plus empty/shebang rejection are a bounded mitigation, but they are not integrity verification and the fail-closed boundary should be an explicit maintainer decision.
