## Summary

Keep this PR open: current main and the latest release still execute the Homebrew and NodeSource setup downloads without structural validation, and the branch is a bounded hardening path with one low-priority deterministic-test finding plus maintainer-owned compatibility/security-boundary risk.

## What This Changes

Adds downloaded-script validation and download-failure guards to `scripts/install.sh` before Homebrew and NodeSource setup scripts are executed, plus a shell harness for malformed-download cases.

## Best Possible Solution

Keep the centralized validator and download-failure guards, make the binary fixture deterministic, and merge only after maintainers accept structural fail-closed validation as the intended installer policy.

## Reproduction Assessment

Yes. Source inspection of current main and `v2026.5.28` shows Homebrew and NodeSource setup scripts are downloaded into temp files and invoked with `bash` without structural validation.

## Solution Assessment

Yes as a bounded mitigation, not as full integrity validation. The validator sits near `download_file`/`run_remote_bash` and covers all four affected execution paths; stronger integrity checks remain a maintainer policy choice.
