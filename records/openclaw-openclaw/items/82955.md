## Summary

Keep open: the validator improves the installer download path, but PR head still lets a failed downloader fall through when it leaves shebang-prefixed partial content that passes validation and reaches `bash`.

## What This Changes

The PR adds `validate_downloaded_script()` to `scripts/install.sh`, applies it to Homebrew and NodeSource setup-script execution paths, and adds `scripts/test-install-validation.sh`.

## Best Possible Solution

Keep the validation approach, but make all download-then-execute paths return on downloader failure and cover that failure mode before merge.

## Reproduction Assessment

Yes. Source inspection plus the existing ClawSweeper shell probe show that a failed downloader can leave a shebang-prefixed partial temp file that still reaches `bash "$tmp"` under the PR head.

## Solution Assessment

No. The validator is the right direction, but the narrow maintainable fix is to check downloader status before validation or execution in all four touched paths.
