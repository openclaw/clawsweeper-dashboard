## Summary

Keep open for maintainer review: the patch appears correct and has sufficient terminal proof, but merging intentionally makes installer downloads fail closed when upstream Homebrew or NodeSource setup scripts do not start with a first-line shebang.

## What This Changes

The PR adds shell-script structural validation before executing downloaded Homebrew and NodeSource installer scripts and adds a standalone shell validation harness.

## Best Possible Solution

## Solution Assessment

Yes, with a maintainer policy call. Non-empty plus first-line shebang validation is a narrow hardening path that matches the live upstream scripts checked during review, but unconditional fail-closed behavior needs installer-owner acceptance.
