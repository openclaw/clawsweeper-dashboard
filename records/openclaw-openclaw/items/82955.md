## Label Justifications

- `P2`: The PR hardens installer script execution but has a bounded install-path blast radius rather than an active emergency.
- `merge-risk: 🚨 compatibility`: The new raw-shebang requirement can break installs if a supported upstream setup script changes shape even though the download itself succeeds.
- `merge-risk: 🚨 availability`: The installer will now fail closed before Homebrew or NodeSource setup when validation rejects content, so a false rejection can make fresh installs unavailable on affected platforms.

## Summary

Keep open for maintainer review: current main and the latest release still execute the downloaded installer scripts without structural validation, and this PR is a narrow hardening patch with sufficient terminal proof, but it intentionally makes installer setup fail closed on third-party script shape so compatibility and availability risk needs maintainer acceptance.

## What This Changes

Adds `validate_downloaded_script` to `scripts/install.sh` before Homebrew and NodeSource setup-script execution, plus `scripts/test-install-validation.sh` malformed-download coverage.

## Best Possible Solution

Land the narrow validation after maintainers accept the fail-closed installer policy and verify the rebased/merged installer still passes the real malformed-download and normal-script paths.

## Reproduction Assessment

Yes. Source inspection of current main and `v2026.5.22` shows the Homebrew and NodeSource paths download to a temp file and run `bash` without validating that the file is non-empty or script-shaped first.

## Solution Assessment

Yes, with maintainer policy acceptance. The patch is a narrow structural guard around the existing downloader/execution path, but the fail-closed dependency on third-party shebangs is a compatibility decision rather than a purely mechanical fix.
