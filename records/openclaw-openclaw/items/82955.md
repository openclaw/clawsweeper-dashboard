## Label Justifications

- `P2`: This is a normal-priority installer security hardening change with limited blast radius but meaningful user-install impact.
- `merge-risk: 🚨 compatibility`: Strict first-line shebang validation can reject future upstream setup scripts that remain Bash-compatible but do not start exactly with `#!`.
- `merge-risk: 🚨 availability`: If an upstream setup script changes shape, affected installs would fail before Homebrew or NodeSource setup completes.

## Summary

Keep open for maintainer review: this is a focused installer hardening PR with sufficient terminal proof, but it intentionally changes remote setup-script execution from permissive Bash execution to fail-closed first-line shebang validation.

## What This Changes

The PR adds `validate_downloaded_script()` to `scripts/install.sh`, calls it before Homebrew and NodeSource setup-script execution, and adds a standalone shell validation harness.

## Best Possible Solution

Land centralized downloaded-script validation once installer/security owners accept the strict shebang contract, or explicitly allow reviewed safe prefixes while still rejecting empty, HTML, JSON, binary, and diagnostic-injection payloads.

## Reproduction Assessment

Yes from source inspection, though I did not run current main locally. Current main downloads Homebrew and NodeSource setup scripts into temp files and invokes Bash directly, and the PR body supplies after-fix terminal proof for malformed HTTP and HTTPS responses.

## Solution Assessment

Yes, centralizing structural validation is a maintainable fix for malformed remote-script downloads. The open decision is whether the installer should enforce a strict first-byte shebang or permit a small reviewed set of safe prefixes.
