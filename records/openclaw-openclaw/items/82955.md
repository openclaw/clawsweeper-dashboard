## Summary

Keep this PR open: it addresses a real installer hardening gap on current main with no blocking correctness finding, but the strict fail-closed shebang rule needs installer/security owner acceptance because it can make future upstream installer-script format changes abort OpenClaw installs.

## What This Changes

The branch adds `validate_downloaded_script` to `scripts/install.sh` before Homebrew and NodeSource downloaded scripts are executed, plus a standalone shell validation harness.

## Best Possible Solution

Land centralized downloaded-script validation once installer/security owners accept the strict shebang contract, or narrowly allow reviewed safe prefixes while still rejecting empty, HTML, JSON, binary, and diagnostic-injection payloads.

## Reproduction Assessment

Yes, from source inspection. Current main and `v2026.5.22` download Homebrew and NodeSource setup scripts into temp files and invoke Bash without validating the downloaded content first.

## Solution Assessment

Yes, with a maintainer policy caveat. A shared validator at the download-then-execute boundary is the narrow maintainable fix, but owners need to accept or refine the strict first-line shebang rule before merge.

- The validator intentionally aborts unless the downloaded file starts with `#!`; if Homebrew or NodeSource later publish a Bash-compatible setup script with a BOM, leading blank/comment prelude, or another safe prefix, OpenClaw installs could fail before package setup.

- The supplied proof covers local malformed HTTP/HTTPS responses rather than a live upstream CDN error page, which is reasonable for structural validation but leaves the strict upstream-format contract as owner judgment.
