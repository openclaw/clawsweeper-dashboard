## Summary

Keep open: current `main` and the latest release still execute downloaded Homebrew and NodeSource setup scripts without content validation, while this PR adds a focused hardening path with sufficient terminal proof. The remaining question is maintainer acceptance of the stricter fail-closed `#!` script-shape contract, not whether the PR is obsolete.

## What This Changes

The PR adds `validate_downloaded_script()` to `scripts/install.sh`, applies it before Homebrew and NodeSource setup-script execution, and adds a standalone malformed-download validation harness.

## Best Possible Solution

Land centralized downloaded-script validation after installer owners accept the strict shebang contract, or broaden only narrow known-safe prefixes before merge if they prefer a softer availability posture.

## Reproduction Assessment

Yes from source inspection: current `main` and `v2026.5.22` download Homebrew and NodeSource setup scripts to temp files and execute them with bash without validating content. The PR body also includes after-fix terminal proof against malformed responses.

## Solution Assessment

Yes, with maintainer acceptance: central structural validation is a narrow maintainable fix for malformed downloaded scripts. The unresolved part is the strict first-byte shebang policy, which is an installer availability tradeoff.

## Risks / Open Questions

- The strict first-line `#!` rule can break installs if Homebrew or NodeSource later publish a bash-compatible setup script with a BOM, leading comments, or no shebang.
- A future upstream format change would become an OpenClaw install-time failure instead of falling through to bash, so installer owners should explicitly accept that availability tradeoff.
- The added malformed-download coverage is a standalone shell harness rather than part of the existing installer Vitest lane, so normal CI may not protect this behavior unless maintainers wire it in later.
