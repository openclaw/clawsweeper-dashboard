## What This Changes

The PR adds a shared downloaded-script validator plus checked download-failure guards before Homebrew and NodeSource setup scripts are executed by `scripts/install.sh`, and adds a standalone shell validation harness.

## Best Possible Solution

Keep the centralized validator and checked download guards, make the binary fixture deterministic, and land only after maintainers accept structural fail-closed validation as the intended installer policy.

## Reproduction Assessment

Yes. Source inspection of current main and v2026.5.28 shows Homebrew and NodeSource setup scripts are downloaded to temp files and invoked with `bash` without structural validation.

## Solution Assessment

Yes as a bounded mitigation, not as full integrity validation. The validator sits near `download_file`/`run_remote_bash` and covers all four affected execution paths; the safer pre-merge shape is to fix the random fixture and record maintainer acceptance of the fail-closed policy.
