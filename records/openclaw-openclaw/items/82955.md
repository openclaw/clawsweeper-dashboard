## Summary

The PR is still necessary and should stay open for maintainer review: current main and the latest release still execute downloaded Homebrew and NodeSource setup scripts without structural validation, while the branch adds a focused validator with credible terminal proof and no blocking code finding. The remaining question is whether maintainers accept the stricter fail-closed installer behavior.

## What This Changes

The branch adds `validate_downloaded_script()` to `scripts/install.sh`, calls it before Homebrew and NodeSource downloaded setup scripts are executed, and adds `scripts/test-install-validation.sh` coverage for malformed downloads, diagnostic sanitization, and valid shebang scripts.

## Best Possible Solution

Land the shared validator after maintainer acceptance of the stricter installer contract, preserving the current proof and recording any final merged-head installer smoke required by landing policy.

## Reproduction Assessment

Yes. Current main and `v2026.5.22` both show download-then-bash paths for Homebrew and NodeSource without structural validation, and the PR body supplies after-fix terminal proof against empty, HTML, valid, and escape-attack responses.

## Solution Assessment

Yes, with maintainer acceptance of the compatibility tradeoff. A shared validator immediately before the four shell execution paths is the narrowest maintainable hardening; checksum pinning would be stronger but brittle for frequently changing third-party setup scripts.
