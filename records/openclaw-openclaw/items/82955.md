## Summary

Keep open: current main still executes downloaded Homebrew and NodeSource setup scripts without structural validation, and this PR is a focused hardening with sufficient terminal proof. The remaining blocker is installer-owner acceptance of the fail-closed shebang rule before merge.

## What This Changes

Adds `validate_downloaded_script()` to `scripts/install.sh` before Homebrew and NodeSource setup-script execution, plus a standalone shell validation harness.

## Best Possible Solution

Keep validation before every remote setup-script execution, then land after installer owners accept the fail-closed shebang contract or request a compatibility-preserving rule with wired regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows `run_remote_bash` and NodeSource branches download temp files and execute them with bash without structural validation; the path is deterministic even though this read-only review did not run a local repro.

## Solution Assessment

Yes, if maintainers accept the fail-closed installer contract. Otherwise the safer path is to preserve validation but relax the compatibility rule around known bash-compatible script formats.
