## Summary

Keep open: current main still executes downloaded Homebrew and NodeSource setup scripts without validating the response body first, and this PR is a focused security hardening. The implementation looks basically sound, but the added harness still contains a portable-shell issue and maintainers should explicitly accept the fail-closed installer behavior before merge.

## What This Changes

The branch adds `validate_downloaded_script` to `scripts/install.sh`, calls it before executing Homebrew and NodeSource setup scripts, and adds a shell validation harness.

## Best Possible Solution

Land the installer validation after replacing the GNU-only harness assertion and recording maintainer acceptance of the strict downloaded-script gate with focused install smoke proof.

## Reproduction Assessment

Yes. Current main directly passes downloaded Homebrew and NodeSource temp files to bash without validating the body, and the PR proof demonstrates the patched behavior against rejected and accepted response bodies.

## Solution Assessment

Mostly yes. Structural non-empty plus shebang validation is a bounded hardening path for the observed upstream scripts, but the proof harness should avoid GNU-only `grep -P` and maintainers should own the fail-closed policy choice.
