## Summary

## What This Changes

Adds non-empty/shebang validation and sanitized first-line diagnostics to `run_remote_bash`, plus a standalone installer validation harness.

## Best Possible Solution

Extract a reusable downloaded-shell-script validator and call it before every installer path that executes a downloaded setup script, while preserving existing root/sudo behavior and avoiding checksum pinning.

## Reproduction Assessment

Yes. Source inspection shows current main downloads and executes Homebrew through `run_remote_bash`, and separately downloads NodeSource setup scripts before executing them with `bash` or `sudo bash`.

## Solution Assessment

No. Validating `run_remote_bash` is the right shape for Homebrew, but the maintainable fix should reuse that validation for the NodeSource setup-script paths too.
