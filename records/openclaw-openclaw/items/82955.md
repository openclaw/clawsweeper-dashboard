Full review comments:

- **[P2] Return on failed remote downloads:** `scripts/install.sh:122`
  - body: This validation is reached even if `download_file` returns nonzero. Because `run_remote_bash` is invoked through `run_quiet_step`, the function can continue after a failed transfer; if the partial file starts with `#!`, validation passes and `/bin/bash "$tmp"` still runs it. Please return on the download failure before validating.
  - confidence: 0.88
- **[P2] Stop after failed NodeSource downloads:** `scripts/install.sh:1805`
  - body: The NodeSource branches ignore the `run_quiet_step "Downloading NodeSource setup script"` return value and then validate the temp file. A failed partial setup-script download that begins with `#!` would still pass this check and reach `bash "$tmp"` or `sudo bash "$tmp"`; guard the download step before validation in the apt, dnf, and yum paths.
  - confidence: 0.86
