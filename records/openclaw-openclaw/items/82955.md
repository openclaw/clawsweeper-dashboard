## What This Changes

The PR adds `validate_downloaded_script()` to `scripts/install.sh`, calls it before Homebrew and NodeSource setup-script execution, and adds a shell harness for malformed download validation.

## Best Possible Solution

Land the scoped structural validation after targeted installer checks, while keeping the trust claim limited to rejecting malformed downloaded scripts rather than script authenticity.

## Reproduction Assessment

## Solution Assessment

Yes. A shared non-empty/shebang validator with sanitized diagnostics is the narrow maintainable fix for malformed downloaded content; checksum or signature pinning would be a larger installer trust-model decision.
