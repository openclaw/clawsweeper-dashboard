## Summary

Keep open. Current main still downloads the Homebrew and NodeSource setup scripts into temp files and runs Bash directly, while this PR adds centralized non-empty/shebang validation with convincing terminal proof. I did not find a blocking code defect, but the strict fail-closed installer contract still needs maintainer acceptance.

## What This Changes

Adds `validate_downloaded_script()` to `scripts/install.sh`, calls it before Homebrew and NodeSource setup-script execution, and adds a standalone shell validation harness.

## Best Possible Solution

Land centralized downloaded-script validation once installer/security owners accept the strict shebang contract, or explicitly allow reviewed safe prefixes while still rejecting empty, HTML, JSON, binary, and diagnostic-injection payloads.
