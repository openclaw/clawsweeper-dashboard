## Summary

Keep open. The installer hardening is useful and the supplied terminal proof is credible, but the patch still validates before checking downloader failure; a failed download that leaves shebang-prefixed partial content can pass validation and execute, so this needs a narrow repair before merge.

## What This Changes

The PR adds `validate_downloaded_script()` to `scripts/install.sh`, applies it before Homebrew and NodeSource setup scripts are executed, and adds a standalone shell validation harness.

## Best Possible Solution

Land the validator only after every download-then-execute path returns immediately on downloader failure and has a regression covering a failed shebang-prefixed partial download; keep the fail-closed shebang policy only if maintainers accept that upstream script contract.

## Reproduction Assessment

Yes. Source inspection plus a focused shell probe against current main and PR head show that a downloader failure leaving a shebang-prefixed partial temp file can still execute through `run_remote_bash`.

## Solution Assessment

No. The validator is the right direction, but the patch must check downloader status before validation or execution so failed transfers cannot fall through to `bash "$tmp"`.
