## What This Changes

The PR adds `validate_downloaded_script` to `scripts/install.sh` for Homebrew and NodeSource download-then-execute paths, plus a standalone shell validation harness.

## Best Possible Solution

Land the scoped validator after maintainers accept the fail-closed installer behavior and validate the final combined installer result; treat signed or checksum-pinned third-party script provenance as separate trust-model work.

## Reproduction Assessment

## Solution Assessment

Yes, with maintainer acceptance of the compatibility tradeoff. A shared non-empty/shebang validator with sanitized diagnostics is a narrow hardening path; full signing or checksum pinning would be broader trust-model work.
