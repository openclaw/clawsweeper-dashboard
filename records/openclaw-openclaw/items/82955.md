## Summary

Keep open: current main still executes downloaded Homebrew and NodeSource setup scripts without checking for empty or non-shell content, and this PR is a focused hardening of that path. The remaining actionable issue is small: the supplemental Bash harness uses GNU-only `grep -P`, so it is not portable to macOS/BSD maintainer environments.

## What This Changes

The PR adds `validate_downloaded_script()` to `scripts/install.sh`, applies it before Homebrew and NodeSource downloaded scripts are executed, and adds a standalone Bash validation harness.

## Best Possible Solution

Keep the downloaded-script validation, make the harness portable, and have maintainers explicitly accept the fail-closed installer tradeoff before merge.

## Reproduction Assessment

Yes from source inspection: current main downloads Homebrew and NodeSource setup scripts and executes the temp file without checking for empty or non-shebang content. I did not run the full installer path in this read-only review.

## Solution Assessment

Mostly yes: non-empty and shebang validation is a narrow hardening of the implicated paths, and the PR also sanitizes rejected diagnostics. The best merge path is to make the harness portable and confirm the intended fail-closed installer policy.
