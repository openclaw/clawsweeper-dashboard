## Summary

Keep open for maintainer review: the PR addresses a real current-main installer hardening gap with sufficient terminal proof and no blocking code findings, but it intentionally changes first-install failure behavior by requiring downloaded third-party setup scripts to start with a shebang.

## What This Changes

Adds `validate_downloaded_script()` to `scripts/install.sh`, invokes it before Homebrew and NodeSource setup-script execution, and adds a standalone shell validation harness.

## Best Possible Solution

Land with centralized validation once installer owners accept the fail-closed shebang contract, or request a narrow compatibility prefix rule and CI-backed malformed-download coverage before merge.

## Reproduction Assessment

Yes. Current-main source inspection shows the Homebrew and NodeSource paths download temp files and execute them with bash without validation; I did not run the malformed-download current-main path in this read-only review.

## Solution Assessment

Likely yes. Centralizing structural validation before every remote setup-script execution is the narrow maintainable fix, with the remaining choice being whether installer owners want strict shebang-at-byte-zero compatibility or a slightly more tolerant prefix rule.
