## Risks / Open Questions

- The validator intentionally aborts unless the downloaded file starts with `#!`; if Homebrew or NodeSource later publish a Bash-compatible setup script with a BOM, leading blank line, comment prelude, or another safe prefix, OpenClaw installs could fail before package setup.
- The added shell harness is useful proof but is not wired into the maintained Vitest, Docker, or installer CI lanes, so regression coverage depends on maintainers choosing to run or port it.
- The proof exercises local malformed HTTP/HTTPS responses rather than an actual upstream CDN error page, which is reasonable for structure validation but still leaves live upstream behavior as owner judgment.
