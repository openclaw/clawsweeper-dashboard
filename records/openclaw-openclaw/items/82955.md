## Summary

Keep open for maintainer review: current main still executes downloaded Homebrew and NodeSource setup scripts without structural validation, and this PR is a focused hardening change with sufficient terminal proof. The remaining decision is whether installer owners accept the stricter fail-closed shebang contract for third-party setup scripts.

## What This Changes

Adds `validate_downloaded_script` to `scripts/install.sh` before Homebrew and NodeSource setup-script execution, plus a standalone shell validation harness for malformed downloads.

## Best Possible Solution

Land centralized downloaded-script validation once installer owners accept the strict shebang policy, or narrowly broaden known-safe prefixes and add normal installer-lane coverage if they want a softer availability posture.

## Reproduction Assessment

Yes. Current main has clear source paths that download Homebrew and NodeSource setup scripts into temp files and execute them with bash without structural validation; malformed non-empty 200 responses would reach bash.

## Solution Assessment

Yes, with maintainer acceptance. Central validation before remote-script execution is the narrow maintainable fix; the remaining policy choice is how strict the first-line shebang requirement should be.
