## Summary

The PR is a focused installer hardening change, and current main still executes downloaded setup scripts without structural validation. It should stay open for maintainer review because the strict first-line shebang rule can intentionally fail installs that current OpenClaw would attempt if an upstream installer remains bash-compatible but does not start with `#!`.

## What This Changes

The branch adds a reusable downloaded-script validator to `scripts/install.sh`, applies it before Homebrew and NodeSource setup-script execution, and adds a standalone malformed-download shell harness.

## Best Possible Solution

Land centralized downloaded-script validation once installer owners accept the strict shebang contract, or adjust it to allow narrowly documented safe prefixes while still rejecting empty, HTML, JSON, binary, and diagnostic-injection payloads.

## Reproduction Assessment

Yes, source inspection of current main is enough: `run_remote_bash` and three NodeSource branches download temp files and execute them with bash without structural validation. The PR body also supplies after-fix terminal proof against malformed local HTTP/HTTPS responses.

## Solution Assessment

Yes for the centralized validation shape; unclear only on the exact strict first-line shebang policy. A narrow safe-prefix allowance is the safer alternative if maintainers want to preserve more bash-compatible upstream script formats.
