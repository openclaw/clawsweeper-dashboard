## Summary

Keep open: current `main` and the latest shipped release still execute downloaded Homebrew and NodeSource setup scripts without structural validation, while this PR adds a focused fail-closed validation path with sufficient terminal proof. The remaining concrete code issue is the added harness's GNU-only control-byte assertion, and maintainers still need to accept the stricter installer failure mode before merge.

## What This Changes

The PR adds `validate_downloaded_script()` to `scripts/install.sh`, applies it before Homebrew and NodeSource downloaded setup scripts execute, and adds a shell validation harness.

## Best Possible Solution

Keep validation localized to the four remote-script execution paths, make the harness portable, and have installer owners explicitly accept the fail-closed behavior before merge.

## Reproduction Assessment

Yes by source inspection: current `main` and `v2026.5.22` download Homebrew and NodeSource setup scripts into temp files and run bash without validating the body first. I did not execute a malformed current-main download locally, but the code path and PR after-fix proof make the issue source-reproducible with high confidence.

## Solution Assessment

Yes, with a caveat. Non-empty plus shebang validation is a bounded hardening step without brittle checksum pins, but the harness should avoid GNU-only `grep -P` and maintainers must accept the fail-closed installer policy.
