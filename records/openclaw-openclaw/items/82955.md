## Summary

Keep open: this PR is useful and has sufficient terminal proof for the new `run_remote_bash` guard, but current main also downloads NodeSource apt/dnf/yum setup scripts and executes them with bash without using the same validation.

## What This Changes

The PR adds non-empty/shebang validation and sanitized first-line diagnostics to `scripts/install.sh`'s `run_remote_bash` helper, plus a standalone installer validation harness.

## Best Possible Solution

Extract a reusable downloaded-shell-script validator and call it before every installer path that executes a downloaded setup script, with focused coverage in the existing installer tests.

## Reproduction Assessment

Yes, as a source-reproducible installer boundary. Current main downloads setup scripts into temp files and executes them with bash, and the PR body demonstrates malformed downloads being rejected only for the guarded helper path.

## Solution Assessment

No. Guarding `run_remote_bash` is the right direction, but the maintainable fix should share the same validator with the NodeSource apt/dnf/yum setup-script paths too.
