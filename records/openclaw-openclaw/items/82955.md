## Summary

Keep open: current main and the latest release still execute downloaded Homebrew and NodeSource installer scripts without structural validation, and this PR addresses the right security-sensitive surface. The remaining concrete issue is a small nondeterministic harness fixture, plus maintainer acceptance of fail-closed validation for third-party installer scripts.

## What This Changes

The PR adds non-empty/raw-shebang validation and download-failure guards before Homebrew and NodeSource downloaded scripts are executed, plus a standalone shell validation harness.

## Best Possible Solution

Keep the download-failure guards and structural validation, replace the random fixture with deterministic non-shebang bytes, and land after maintainers accept fail-closed validation for these installer scripts.

## Reproduction Assessment

Yes, from source inspection: current main and v2026.5.22 download remote Homebrew and NodeSource setup scripts and execute the temp file without structural validation. The PR body supplies terminal proof for malformed files, escape payloads, valid scripts, and a failed partial-transfer pipeline.

## Solution Assessment

Mostly yes. The downloader-success gate plus non-empty/raw-shebang validation is a narrow maintainable fix for the observed shell-execution risk, but the random binary fixture should be replaced with deterministic bytes.
