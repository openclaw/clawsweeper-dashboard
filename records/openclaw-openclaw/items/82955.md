## Summary

Keep open for maintainer review: the PR fixes a source-reproducible installer hardening gap and supplies sufficient terminal proof, but strict shebang-only validation is an installer availability/compatibility choice that should be accepted by the relevant owners before merge.

## What This Changes

The branch adds `validate_downloaded_script` to `scripts/install.sh` before Homebrew and NodeSource downloaded scripts are executed, plus a standalone shell validation harness.

## Best Possible Solution

Land centralized downloaded-script validation once installer/security owners accept the strict shebang contract, or narrowly allow reviewed safe prefixes while still rejecting empty, HTML, JSON, binary, and diagnostic-injection payloads.

## Reproduction Assessment

Yes, source-reproducible: current main downloads Homebrew and NodeSource setup scripts into temp files and invokes Bash without validating the downloaded content first. I did not run a current-main live failure case, but the execution path is direct in `scripts/install.sh`.

## Solution Assessment

Yes, validating downloaded script structure before Bash execution is a narrow maintainable fix. The remaining solution choice is whether strict first-byte shebang validation is acceptable or should allow explicitly reviewed harmless prefixes.
