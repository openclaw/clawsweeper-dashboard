## Summary

Keep open: this is useful installer security hardening with sufficient terminal proof, but the rejected-body diagnostic still has an incomplete terminal-output sanitizer and the new fail-closed script-shape check needs maintainer-owned compatibility acceptance before merge.

## What This Changes

The PR adds `validate_downloaded_script()` to `scripts/install.sh`, gates Homebrew and NodeSource setup-script execution on non-empty/shebang checks, and adds a standalone shell validation harness.

## Best Possible Solution

Keep the shared validator, encode or strip every non-printable diagnostic byte before calling `ui_error`, add deterministic regression coverage, and merge only after maintainers accept strict fail-closed validation for third-party installer scripts.

## Reproduction Assessment

Yes, by source inspection. Current main downloads Homebrew and NodeSource setup scripts to temp files and invokes bash on those files before validating the downloaded body.

## Solution Assessment

No, not yet. The shared validator is the right repair shape, but the diagnostic preview needs a printable-safe sanitizer and maintainers still need to accept the strict third-party script-shape tradeoff.

## Review Findings

Full review comments:

- **[P2] Escape every non-printable byte before logging:** `scripts/install.sh:87-88`
  - body: `tr -d '\000-\037\177'` removes C0 controls and DEL, but it still preserves C1 bytes such as 0x9b. Because the next `ui_error` call reaches the non-gum `echo -e` path, a rejected response can still emit terminal-control bytes; whitelist or encode printable bytes before logging and cover C1/8-bit input.
  - confidence: 0.86
- **[P3] Make the binary fixture deterministic:** `scripts/test-install-validation.sh:85`
  - body: The binary negative fixture comes from `/dev/urandom`, so it can rarely start with `#!` and be accepted by the shebang check. Use fixed non-shell bytes so this harness cannot flip randomly.
  - confidence: 0.82
