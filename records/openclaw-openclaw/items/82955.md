## Summary

Keep this PR open: the core hardening is useful and the supplied terminal proof covers the main malformed-download behavior, but the patch still previews untrusted response bytes with an incomplete terminal-control sanitizer and carries an installer compatibility tradeoff maintainers should accept deliberately.

## What This Changes

The PR adds `validate_downloaded_script()` to `scripts/install.sh`, calls it before Homebrew and NodeSource downloaded scripts are executed, and adds a standalone malformed-download validation harness.

## Best Possible Solution

Keep the shared validator, encode or strip every non-printable diagnostic byte, add a focused regression fixture, then have maintainers explicitly accept or adjust the strict installer compatibility tradeoff.

## Reproduction Assessment

Yes, by source inspection. Current main downloads the Homebrew and NodeSource setup scripts to temp files and executes them with bash before validating the downloaded body.

## Solution Assessment

No, not yet. The shared validator is the right shape, but the diagnostic preview needs a stricter printable-safe sanitizer and maintainers should deliberately accept the fail-closed script-shape behavior.
