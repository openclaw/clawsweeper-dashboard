## Summary

## What This Changes

Adds `validate_downloaded_script()` to `scripts/install.sh` before Homebrew and NodeSource downloaded scripts execute, plus `scripts/test-install-validation.sh` coverage for bad and valid payloads.

## Best Possible Solution

Land the validation if maintainers accept fail-closed installer setup semantics; otherwise revise to a softer compatibility path or explicit strict mode before merge.

## Reproduction Assessment

Yes. Current main source shows the downloaded Homebrew and NodeSource temp files are executed without content validation, and the PR proof exercises the patched rejection paths against local HTTP/HTTPS fixtures.

## Solution Assessment

Yes, subject to maintainer policy. Non-empty plus raw-shebang validation is a narrow checksum-free hardening that current upstream scripts satisfy; the remaining question is whether fail-closed installer behavior is acceptable.
