## Summary

Keep this PR open: the downloaded-script validator is useful and well scoped, but `run_remote_bash` still ignores a failed downloader status before validating the file, so a failed partial response that starts with `#!` can still be executed.

## What This Changes

The PR adds `validate_downloaded_script()` to `scripts/install.sh`, applies it to the Homebrew and NodeSource download-then-execute paths, and adds `scripts/test-install-validation.sh` coverage for bad downloaded script bodies.

## Best Possible Solution

Land the validator after `run_remote_bash` returns immediately on downloader failure and the harness covers a failed shebang-prefixed partial download.

## Reproduction Assessment

Yes for the source-level issue: current main downloads and executes Homebrew and NodeSource setup scripts without structural validation. I did not run the installer end to end in this read-only review.

## Solution Assessment

No, not yet. The validator is the right repair shape, but `run_remote_bash` must preserve downloader failure status before validating or executing the temp file.
