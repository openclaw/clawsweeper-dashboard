## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Check raw bytes before accepting shebang:** `scripts/install.sh:98-99`
  - body: `first_line` is populated through command substitution before the `#!` check, and Bash drops NUL bytes there. A response starting with `\0#!/bin/bash` therefore passes this validator even though the raw file does not start with a shebang, leaving the structural validation bypassable for malformed binary/truncated content. Check the raw first two bytes before command substitution and add that regression case.
  - confidence: 0.93
- **[P3] Avoid GNU-only grep in the harness:** `scripts/test-install-validation.sh:140`
  - body: The new harness uses `grep -P` for the control-byte assertion, but BSD grep on macOS does not support `-P`. Since this installer surface is macOS and Linux, use a portable byte check so the validation script can run in both supported environments.
  - confidence: 0.82
