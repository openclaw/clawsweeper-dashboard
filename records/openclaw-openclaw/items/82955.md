## Summary

## What This Changes

The PR adds `validate_downloaded_script()` to `scripts/install.sh`, calls it before Homebrew and NodeSource setup scripts are executed, and adds a standalone shell validation harness.

## Best Possible Solution

Keep the shared validator, make the diagnostic preview printable-safe with deterministic coverage, and merge only after maintainers accept strict Homebrew/NodeSource shebang enforcement.

## Reproduction Assessment

Yes. Current `main` clearly downloads Homebrew and NodeSource setup scripts into temp files and executes them without checking for non-empty shell-script content; I did not run the full installer, but the source path is direct and the PR body includes after-fix terminal proof.

## Solution Assessment

Mostly yes. A shared structural validator at the execution boundary is the right shape, but the diagnostic must be printable-safe and maintainers need to accept the strict shebang fail-closed behavior before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Escape every non-printable byte before logging:** `scripts/install.sh:87-88`
  - body: `validate_downloaded_script` is meant to protect the `ui_error` path from untrusted downloaded content, but this sanitizer only removes C0/DEL bytes and escapes backslashes. C1/high-bit control bytes can still survive into the non-gum `echo -e` diagnostic, so please encode or whitelist the preview before logging and cover those byte classes.
  - confidence: 0.86
- **[P3] Make the binary fixture deterministic:** `scripts/test-install-validation.sh:85`
  - body: The harness uses `/dev/urandom` for the invalid binary fixture, which makes the test probabilistic and can emit byte-dependent shell warnings. Use a fixed binary payload so the fixture can never randomly begin with `#!` and the output stays stable.
  - confidence: 0.78
