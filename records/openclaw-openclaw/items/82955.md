## Summary

Keep open: the PR is useful installer hardening, but `run_remote_bash` still needs to stop on a failed downloader status before validating or executing the temp file. The intentional fail-closed behavior for installer setup scripts is also a compatibility/availability tradeoff for maintainers to accept before merge.

## What This Changes

The PR adds `validate_downloaded_script()` to `scripts/install.sh`, applies it before Homebrew and NodeSource setup-script execution, and adds a shell validation harness for rejected and accepted payloads.

## Best Possible Solution

Land the hardening after explicitly returning on `download_file` failure before validation/execution and keeping proof that live upstream scripts plus bad responses behave as expected.

## Reproduction Assessment

Yes. Source inspection shows `run_quiet_step` invokes shell functions inside an `if`, and a focused Bash check confirmed `errexit` is ignored there; the remaining path is a failed downloader that leaves a shebang-prefixed partial file.

## Solution Assessment

No. The validation direction is right, but the best fix also checks the downloader status before validation/execution so failed partial downloads cannot slip through.
