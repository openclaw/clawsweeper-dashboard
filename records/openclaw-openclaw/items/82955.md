## Summary

Keep open for maintainer review: current main and the latest release still have the unsafe execution path, the patch has sufficient real proof and no blocking correctness finding, but installer owners should accept the stricter fail-closed shebang policy before merge.

## What This Changes

The branch adds downloaded shell-script validation before Homebrew and NodeSource setup-script execution and adds a standalone shell harness for malformed download cases.

## Best Possible Solution

Land centralized downloaded-script validation once installer owners accept the strict shebang contract, or narrowly allow known-safe prefixes and wire the cases into normal installer validation if they prefer a softer availability posture.

## Reproduction Assessment

Yes from source inspection. Current main and v2026.5.22 both download Homebrew and NodeSource setup scripts into temp files and execute them with bash without structural validation.

## Solution Assessment

Yes, with maintainer acceptance. Central validation before remote-script execution is the narrow maintainable fix; the remaining judgment is whether the first-line shebang policy should be strict or allow a few known-safe prefixes.
