## Summary

## What This Changes

The PR adds `validate_downloaded_script` to `scripts/install.sh`, gates Homebrew and NodeSource setup-script execution on non-empty shebang-prefixed content, sanitizes rejection diagnostics, and adds a shell validation harness.

## Best Possible Solution

Land the focused validation if maintainers accept the fail-closed installer behavior, keeping the structural guard and proof while letting normal release verification handle the final merge.

## Reproduction Assessment

Yes. Current main source shows the Homebrew and NodeSource setup paths downloading into a temp file and executing it with bash, and the PR proof demonstrates the malformed-response cases against the patched functions.

## Solution Assessment

Yes, with maintainer acceptance. A non-empty plus raw shebang check is a narrow hardening step that avoids fragile checksum pinning for frequently updated third-party scripts, but the stricter fail-closed behavior is compatibility-sensitive.
