## What This Changes

The PR adds `validate_downloaded_script()` to `scripts/install.sh` for Homebrew and NodeSource setup scripts and adds a standalone shell validation harness.

## Best Possible Solution

Land the centralized validation if maintainers accept fail-closed installer behavior, ideally after a current-base smoke of the Homebrew and NodeSource setup-script paths.

## Reproduction Assessment

Yes. Current main directly executes the downloaded Homebrew and NodeSource setup-script temp files, so serving an empty or HTML response to those paths would reach bash without structural validation.

## Solution Assessment

Yes, with a maintainer acceptance caveat. Centralized structural validation before `bash` is a bounded hardening and avoids brittle hash pins, but maintainers should explicitly accept the stricter fail-closed installer policy.
