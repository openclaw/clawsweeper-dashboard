## Summary

Keep open for maintainer review. The patch looks technically correct and has strong terminal proof, but it intentionally changes installer behavior to fail closed when downloaded Homebrew or NodeSource setup scripts are empty or do not start with a shebang, so maintainers should explicitly accept that compatibility and availability tradeoff before merge.

## What This Changes

The PR adds structural validation before executing downloaded Homebrew and NodeSource setup scripts in `scripts/install.sh` and adds a shell harness covering valid and malformed downloads.

## Best Possible Solution

Land the hardening if maintainers accept the fail-closed installer policy, preferably after current-base CI or a focused live installer smoke; otherwise choose an explicit softer compatibility mode rather than leaving unvalidated remote script execution in place.

## Reproduction Assessment

I did not run a live installer repro in this read-only review, but the contributor supplied after-fix terminal proof against real extracted functions.

## Solution Assessment

Yes, if maintainers accept the fail-closed policy. Structural shebang and non-empty validation is a narrow hardening for dynamic third-party setup scripts, while full checksum pinning would be brittle for frequently changing upstream installers.
