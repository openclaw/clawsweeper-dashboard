## Summary

Current `scripts/install.sh` really does execute four downloaded setup scripts with bash after HTTPS download only. The PR’s structural validation is focused, and the remaining decision is the installer compatibility policy around failing closed on scripts without a first-byte shebang.

## What This Changes

The PR adds `validate_downloaded_script()` to `scripts/install.sh` before Homebrew and NodeSource setup-script execution and adds a standalone shell validation harness under `scripts/`.

## Best Possible Solution

Keep validation before every remote setup-script execution, but land it only after installer owners accept the fail-closed shebang policy or ask for a compatibility-preserving validation rule and a wired regression test.

## Reproduction Assessment

Yes. Current main is source-reproducible: the Homebrew and NodeSource paths download remote scripts into temp files and execute them with bash before structural validation.

## Solution Assessment

Yes, if maintainers accept the fail-closed contract. The helper covers the current download-then-execute paths and today's upstream scripts begin with shebangs; otherwise the better solution is to adjust the validator around the compatibility policy before merge.
