Keep open: the hardening is useful and the supplied terminal proof is convincing, but the patch still validates after a bare Homebrew `download_file` call, so a failed download that leaves shebang-prefixed partial content can still be executed.

## Best Possible Solution

Land the structural validation only after failed download statuses stop execution before validation, with one focused proof case for a failed downloader that leaves shebang-prefixed partial content.

## Solution Assessment

Not yet. Structural validation is the right direction, but the narrow maintainable fix must also stop on downloader failure before validating or executing the temp file.

- **[P2] Return before validating failed downloads:** `scripts/install.sh:122`
