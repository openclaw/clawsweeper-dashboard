## Summary

Keep open for maintainer review. The patch is focused and I found no blocking line-level defect, but it intentionally changes installer behavior from executing whatever the HTTPS download saved to failing closed unless the downloaded Homebrew or NodeSource setup script starts with raw `#!` bytes.

## What This Changes

Adds `validate_downloaded_script()` to `scripts/install.sh`, calls it before Homebrew and NodeSource downloaded setup scripts are executed, and adds a standalone shell validation harness.

## Best Possible Solution

Land the structural validation if maintainers accept the fail-closed installer behavior; otherwise keep compatibility as the default and add an explicit stricter validation path or softer diagnostic before aborting.

## Reproduction Assessment

Yes, source-reproducible: current main clearly downloads temp files and executes them without content validation in the Homebrew and NodeSource paths. I did not run the installer, but the patched PR proof exercises the same shell functions against bad local HTTP/HTTPS responses.

## Solution Assessment

Yes, subject to maintainer approval: non-empty plus raw-shebang validation is a narrow checksum-free guard for these current upstream scripts. The remaining question is whether failing closed is the desired installer policy for all users.
