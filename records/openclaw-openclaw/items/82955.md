Full review comments:

- **[P2] Return on failed downloads before validating:** `scripts/install.sh:123`
  - body: `run_remote_bash` is executed through `run_quiet_step`'s `if "$@"` path, so `errexit` is not a reliable guard inside the function. If `curl` or `wget` exits nonzero after writing a partial file that starts with `#!`, this new validation accepts it and `/bin/bash` still executes the partial download; check `download_file "$url" "$tmp" || return 1` before validating.
  - confidence: 0.88
- **[P2] Guard NodeSource download failures before validation:** `scripts/install.sh:1699`
  - body: The NodeSource branches validate only after the setup-script download step, so the safety of the following `bash "$tmp"` depends on shell `errexit` context instead of the local download result. Guard each setup-script download explicitly before the new validation call so a failed partial shebang file cannot be executed if this function is reused or called from an error-checked context.
  - confidence: 0.78
