## Summary

Keep open: this PR is still useful because current main and the latest release still execute the Homebrew and NodeSource downloaded scripts without structural validation. I found no blocking correctness finding, but the stricter fail-closed behavior needs maintainer acknowledgement because it can stop installs if an upstream installer script stops beginning with a raw shebang.

## What This Changes

The branch adds `validate_downloaded_script()` to `scripts/install.sh`, calls it before Homebrew and NodeSource setup scripts execute, and adds a standalone shell validation harness.

## Best Possible Solution

Land the validator if maintainers accept fail-closed handling for third-party installer scripts, then rely on the existing installer smoke/shellcheck lanes plus the supplied terminal proof for merge confidence.

## Reproduction Assessment

Yes, source-reproducible: current main downloads remote installer scripts and executes the temp files with bash without checking emptiness, shebang, or diagnostic safety. The PR body also includes terminal proof against local HTTP/HTTPS malformed responses at the final head.

## Solution Assessment

Yes, with maintainer risk acceptance: checking non-empty raw shebang bytes before the four download-then-execute paths is a narrow hardening step, and checksum pinning would be brittle for frequently changing Homebrew/NodeSource scripts.
