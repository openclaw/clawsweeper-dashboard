## Summary

Keep open. Current main and the latest release still execute Homebrew and NodeSource downloaded setup scripts with bash before content validation, while this PR adds a focused validation step with sufficient terminal proof; the remaining question is installer-owner acceptance of the stricter fail-closed shebang contract.

## What This Changes

This PR adds a reusable downloaded-shell-script validator to `scripts/install.sh`, applies it before Homebrew and NodeSource setup-script execution, and adds a supplemental shell validation harness.

## Best Possible Solution

Land centralized downloaded-script validation once installer owners accept the stricter shebang contract, or request a narrow compatibility-prefix rule plus CI-backed malformed-download coverage before merge.
