## Summary

This is a focused installer hardening PR with no blocking correctness finding from the read-only review. It should stay open for maintainer acceptance of the installer trust-model change: failing closed on empty or non-first-line-shebang Homebrew and NodeSource setup downloads.

## What This Changes

The PR adds `validate_downloaded_script()` to `scripts/install.sh`, calls it before executing Homebrew and NodeSource downloaded setup scripts, and adds a standalone shell validation harness.

## Best Possible Solution

Land the structural validation after installer owners accept the fail-closed shebang contract, with CI-backed installer coverage added first if maintainers want durable regression protection.

## Reproduction Assessment

Yes, source-reproducible: current main downloads Homebrew and NodeSource setup scripts into temp files and executes them with bash without structural validation. I did not execute the installer flows during this read-only review.

## Solution Assessment

Yes, with a maintainer policy caveat: the helper centralizes a narrow non-empty/shebang check and sanitizes diagnostics before every affected execution path. The remaining question is whether unconditional fail-closed validation is the desired installer trust contract.
