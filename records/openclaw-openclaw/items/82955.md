## Summary

Keep this PR open: the validation is useful security hardening, but the Homebrew download path still needs an explicit failed-download guard before validation/execution because `run_remote_bash` is invoked through `run_quiet_step` in a Bash conditional context.

## What This Changes

The PR adds structural validation for downloaded Homebrew and NodeSource setup scripts in `scripts/install.sh` and adds a standalone shell validation harness.

## Best Possible Solution

Keep the structural validation, but first preserve raw download failures with explicit guards before any validation or execution and add focused proof for the failed-download path.

## Reproduction Assessment

Yes, from source inspection and a focused Bash semantics probe. The PR merge still calls `validate_downloaded_script` after an unchecked `download_file` in `run_remote_bash`, and `run_quiet_step` invokes that function in a conditional context where Bash ignores `errexit` inside the callee.

## Solution Assessment

No, not quite yet. Structural validation is the right narrow direction, but the best fix returns immediately on a failed download before validating or executing any temp-file contents.
