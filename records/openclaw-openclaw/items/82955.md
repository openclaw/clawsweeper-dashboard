The PR adds a shell-script validation helper to `scripts/install.sh` for Homebrew and NodeSource setup downloads and adds a standalone installer-validation shell harness.

## Best Possible Solution

Land the hardening after installer owners accept fail-closed structural validation, preferably with malformed-download cases promoted into an existing installer test lane if they want durable coverage.

## Solution Assessment

Yes, with a maintainer policy call. Non-empty plus first-line shebang validation is a narrow hardening layer and current upstream scripts satisfy it, but unconditional fail-closed behavior needs installer-owner acceptance.
