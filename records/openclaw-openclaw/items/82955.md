## What This Changes

The PR adds a shared Bash validator for downloaded installer setup scripts, applies it before Homebrew and NodeSource bash execution, and adds a standalone validation harness.

## Best Possible Solution

Land the scoped validator if maintainers accept fail-closed installer behavior, coordinate it with the related NodeSource installer PR, and treat signed or checksum-pinned third-party provenance as separate trust-model work.

## Solution Assessment

Yes, with maintainer acceptance of the compatibility tradeoff. A shared non-empty/shebang validator with sanitized diagnostics is a narrow hardening path; full signing or checksum pinning would be broader trust-model work.
