The branch adds downloaded-script validation to `scripts/install.sh` before executing Homebrew and NodeSource setup scripts, plus a standalone shell validation harness.

## Best Possible Solution

Land centralized downloaded-script validation once installer owners accept the strict third-party script contract, or narrow the accepted prefix before merge if reducing future install outages matters more.

## Solution Assessment

Yes, the centralized helper is the narrow maintainable fix for this execution path. The only unsettled point is policy: whether strict first-line shebang validation is the right compatibility boundary for third-party scripts.
