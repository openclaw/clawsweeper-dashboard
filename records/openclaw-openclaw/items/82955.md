## What This Changes

The PR adds downloaded-shell-script validation to `scripts/install.sh` before Homebrew and NodeSource setup execution, plus a shell harness for malformed, empty, binary, and valid script cases.

## Best Possible Solution

Land the localized validation after replacing the harness's GNU-only assertion and after installer owners explicitly accept fail-closed malformed-download behavior, with live install smoke proof if availability is the remaining concern.

## Solution Assessment

Yes, with caveats. Non-empty plus shebang validation is bounded hardening without brittle checksum pins, but the harness should avoid GNU-only `grep -P` and maintainers must accept the fail-closed installer policy.
