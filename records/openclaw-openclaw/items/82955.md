## Summary

The PR should stay open: current main and v2026.5.22 still execute downloaded Homebrew and NodeSource setup scripts without content validation, while the branch adds focused structural validation with sufficient terminal proof. The remaining decision is installer-owner acceptance of the fail-closed shebang policy.

## What This Changes

The PR adds a `validate_downloaded_script()` helper to `scripts/install.sh`, uses it before Homebrew and NodeSource setup-script execution, and adds a standalone shell validation harness.

## Best Possible Solution

Land centralized downloaded-script validation once installer owners accept the strict shebang contract, or narrowly broaden safe prefixes and add CI-wired installer tests if they want a softer availability posture.

## Reproduction Assessment

Yes. Source inspection shows current main and v2026.5.22 download Homebrew and NodeSource setup scripts to temp files and execute them with bash without validating content; the PR body also supplies after-fix terminal proof against malformed responses.

## Solution Assessment

Yes, with maintainer acceptance. Centralized validation before downloaded-script execution is the narrow maintainable fix, but the exact first-line shebang acceptance rule is an installer availability policy choice.
