## What This Changes

This PR adds `validate_downloaded_script()` before Homebrew and NodeSource setup-script execution in `scripts/install.sh`, plus a shell harness for malformed-download validation.

## Best Possible Solution

Land the hardening once maintainers accept strict shebang validation as the default and record any final installer smoke they want for the current base.

## Solution Assessment

Yes, the proposed non-empty and raw shebang validation is a narrow dependency-free guard for the observed installer gap. The best remaining decision is maintainer acceptance of the fail-closed compatibility tradeoff.
