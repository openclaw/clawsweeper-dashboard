## Summary

Keep this PR open for maintainer/security review before merge. The patch is focused and appears correct with sufficient terminal proof, but it intentionally changes installer behavior by failing closed when Homebrew or NodeSource downloads are empty or lack a shebang.

## What This Changes

The PR adds downloaded-script validation to `scripts/install.sh`, applies it before Homebrew and NodeSource setup scripts are executed, adds a shell validation harness, and updates installer tests.

## Best Possible Solution

Land this focused installer validation after an installer/security owner accepts the fail-closed shebang contract, then let https://github.com/openclaw/openclaw/issues/90013 close through the merge.

## Reproduction Assessment

Yes. Source inspection of `origin/main` and `v2026.6.8` shows Homebrew and NodeSource temp files are still executed with bash or sudo bash before structural validation.

## Solution Assessment

Yes, with maintainer acceptance. A shared validator in `scripts/install.sh` is the narrowest maintainable fix for the four bash-executed download paths; checksum pinning third-party setup scripts would create frequent installer breakage.
