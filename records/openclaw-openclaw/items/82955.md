## What This Changes

The PR adds a shared downloaded-script validator to `scripts/install.sh` for Homebrew and NodeSource setup scripts, plus a standalone shell validation harness.

## Best Possible Solution

Land the scoped structural validator after maintainer acceptance of the fail-closed installer behavior and focused installer validation; treat signed or checksum-pinned script provenance as separate trust-model work.

## Reproduction Assessment

Yes. Source inspection on current main shows `run_remote_bash` and the NodeSource apt/dnf/yum branches download temp files and then invoke `/bin/bash`, `bash`, or `sudo bash` without structural validation.

## Solution Assessment

Yes, with maintainer acceptance of the compatibility tradeoff. A shared non-empty/shebang validator with sanitized diagnostics is the narrow maintainable hardening path; signed or checksum-pinned inputs would be broader trust-model work.
