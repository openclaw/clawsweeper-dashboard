## Summary

Keep open. This is useful installer hardening and current main still executes the downloaded Homebrew and NodeSource scripts directly, but the new validator is not merge-ready because the shebang check goes through Bash command substitution, which can drop NUL bytes and let a malformed download reach bash.

## What This Changes

The branch adds `validate_downloaded_script()` to `scripts/install.sh`, applies it before Homebrew and NodeSource setup scripts are executed, and adds `scripts/test-install-validation.sh`.

## Best Possible Solution

Keep the fail-closed validation direction, but make the shebang check compare raw first bytes without NUL elision, add that regression case, and keep the supplemental harness portable across Linux and macOS.

## Reproduction Assessment

Yes. Current main source shows the downloaded Homebrew and NodeSource setup files are executed directly after `download_file`; the PR-specific NUL bypass was also reproduced with the same shell expression in `/tmp`.

## Solution Assessment

No, not quite. Structural validation is the right narrow fix, but the implementation should inspect raw bytes for the shebang before it can reliably claim malformed or binary downloads are rejected before execution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Check raw header bytes before accepting shebang:** `scripts/install.sh:98`
  - body: `first_line="$(head -c 256 "$file" | head -1)"` runs the downloaded bytes through Bash command substitution before the `#!` check, and Bash drops NUL bytes there. A response that starts with NUL bytes followed by `#!/bin/bash` passes this validator and is still handed to `/bin/bash`, so the new guard does not reliably reject malformed binary downloads before execution. Compare the raw first two bytes with a NUL-safe path before producing the sanitized diagnostic.
  - confidence: 0.87
- **[P3] Use a portable C1 diagnostic assertion:** `scripts/test-install-validation.sh:117`
  - body: This harness is under the shared script surface, but `grep -P` is GNU-only and fails on macOS/BSD grep. Please rewrite the C1-byte assertion using portable shell tools, for example by comparing the diagnostic to the same `tr -d` sanitized form, so the proof script can run on both installer platforms.
  - confidence: 0.91
