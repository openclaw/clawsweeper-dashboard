## Summary

This PR remains useful and should stay open for maintainer review: current main still executes downloaded installer scripts without structural validation, while the PR adds focused fail-closed validation and has sufficient terminal proof. ## What This Changes

The PR adds `validate_downloaded_script()` to `scripts/install.sh`, applies it before Homebrew and NodeSource setup-script execution, and adds a shell harness for malformed-download validation cases.

## Best Possible Solution

Land the hardening if maintainers accept the fail-closed installer policy, ideally with required CI green and broader platform smoke before release if extra availability confidence is desired.

## Reproduction Assessment

## Solution Assessment

Yes, the patch is a narrow maintainable hardening path for frequently changing third-party setup scripts where checksum pinning would be brittle. The remaining question is maintainer policy: whether OpenClaw wants this path to fail closed by default.
