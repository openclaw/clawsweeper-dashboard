## Summary

Keep this PR open: current `main` and the latest release still execute the Homebrew and NodeSource downloaded scripts without structural validation, and the submitted fix is focused with sufficient terminal proof. ## What This Changes

The PR adds `validate_downloaded_script()` to `scripts/install.sh` before Homebrew and NodeSource setup-script execution and adds a standalone shell validation harness under `scripts/`.

## Best Possible Solution

Keep validation before every remote setup-script execution, but land it after installer owners accept the fail-closed shebang policy or request a compatibility-preserving rule and wired regression coverage.

## Reproduction Assessment

Yes. Current main is source-reproducible: the Homebrew and NodeSource paths download remote scripts into temp files and execute them with bash before structural validation.

## Solution Assessment

Yes, if maintainers accept the fail-closed contract. The helper covers the current download-then-execute paths and today's upstream scripts begin with shebangs; otherwise the safer solution is to adjust the validator around the compatibility policy before merge.
