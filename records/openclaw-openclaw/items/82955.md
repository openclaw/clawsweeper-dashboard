## Summary

Keep open: this is useful installer hardening with sufficient terminal proof, but the patch only guards the Homebrew `run_remote_bash` path while current main still downloads and executes NodeSource setup scripts with `bash`/`sudo bash` on Linux.

## What This Changes

Adds non-empty/shebang validation and sanitized first-line diagnostics before `run_remote_bash` executes a downloaded script, plus a standalone installer validation harness.

## Best Possible Solution

Extract a reusable downloaded-shell-script validator and call it before every installer path that executes a downloaded setup script, preserving existing sudo/root behavior and avoiding checksum pinning.

## Reproduction Assessment

Yes, by source inspection rather than live execution in this read-only review. Current main downloads setup scripts into temp files and executes them with bash, and the PR body shows terminal proof that the patched helper rejects empty, HTML, and escape-payload responses.

## Solution Assessment

No. Guarding `run_remote_bash` is the right direction, but the maintainable fix should share the validator with the NodeSource apt/dnf/yum setup-script paths too.
