## Summary

Keep open with changes requested. The PR is a focused installer hardening change with solid terminal proof, but the merge result still validates and can execute the temp file without first proving the downloader command succeeded.

## What This Changes

The PR adds structural validation for downloaded Homebrew and NodeSource setup scripts in `scripts/install.sh` and adds a shell harness for bad-download cases.

## Best Possible Solution

Keep the structural validation, but make every download-then-execute path return immediately on downloader failure before validating or executing the temp file, then preserve the real terminal proof for malformed and valid scripts.

## Reproduction Assessment

Yes. Current main source shows Homebrew and NodeSource setup downloads executed from temp files without structural validation, and the PR body includes terminal proof for malformed payload rejection in the patched functions.

## Solution Assessment

No as submitted. Structural validation is the right narrow direction, but the best fix also has to stop immediately on downloader failure before inspecting or executing any temp file.

Full review comments:

- **[P2] Return before validating failed downloads:** `scripts/install.sh:122`
  - body: `run_remote_bash` is invoked by `run_quiet_step` inside an `if "$@"` command, so `errexit` is not a reliable guard inside the helper.
