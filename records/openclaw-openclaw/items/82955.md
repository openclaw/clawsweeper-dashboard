## Summary

Keep open for maintainer review: the patch is focused, real behavior proof is sufficient, and I found no blocking correctness finding, but installer owners still need to accept the new fail-closed first-line-shebang policy for third-party setup scripts.

## What This Changes

This PR adds `validate_downloaded_script()` to `scripts/install.sh`, calls it before Homebrew and NodeSource setup-script execution, and adds a standalone shell validation harness.

## Best Possible Solution

Land the hardening after installer owners accept the fail-closed shebang policy, preferably with malformed-download cases moved into an existing installer test or CI lane if maintainers want ongoing regression coverage.

## Reproduction Assessment

Yes. Current main source shows the Homebrew and NodeSource setup scripts are downloaded to temp files and executed by bash or sudo bash without structural validation; I verified that by source inspection rather than running the full installer.

## Solution Assessment

Yes, with a maintainer policy call. Non-empty plus first-line shebang validation is a narrow hardening path and live upstream scripts currently satisfy it, but unconditional fail-closed behavior needs installer-owner acceptance.
