## What This Changes

The PR adds a `validate_downloaded_script` helper, calls it before executing Homebrew and NodeSource setup scripts, and adds a standalone shell validation harness.

## Best Possible Solution

Land the structural validation after installer owners accept the fail-closed shebang contract, with CI-backed installer coverage added first if maintainers want durable regression protection.

## Reproduction Assessment

Yes, source-reproducible: current main downloads Homebrew and NodeSource setup scripts into temp files and executes them with bash without structural validation. I did not execute installer flows during this read-only review.

## Solution Assessment

Yes, with a maintainer policy caveat: non-empty plus first-line shebang validation and sanitized diagnostics are a narrow hardening layer. The compatibility decision is whether unconditional fail-closed validation is acceptable for trusted upstream installer scripts.
