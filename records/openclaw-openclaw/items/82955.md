## Summary

## What This Changes

The branch adds `validate_downloaded_script()` to `scripts/install.sh`, applies it before executing Homebrew and NodeSource setup scripts, and adds a standalone shell validation harness.

## Best Possible Solution

Keep the shared validator, encode or strip every non-printable diagnostic byte before calling `ui_error`, add deterministic regression coverage, and merge only after maintainers accept strict fail-closed validation for third-party installer scripts.

## Reproduction Assessment

Yes, by source inspection. Current `main` downloads the Homebrew and NodeSource setup scripts to temp files and invokes bash on those files before validating the downloaded body.

## Solution Assessment

No, not yet. The shared validator is the right repair shape, but the diagnostic preview needs a printable-safe sanitizer and maintainers still need to accept the strict third-party script-shape tradeoff.

- **[P2] Escape every non-printable byte before logging:** `scripts/install.sh:87-88`
  - body: `tr -d '\000-\037\177'` removes C0 controls and DEL, but it still preserves C1 bytes such as 0x9b.
