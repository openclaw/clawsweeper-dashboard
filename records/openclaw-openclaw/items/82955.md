## Summary

Keep open. Current `main` and the latest shipped release still execute downloaded Homebrew and NodeSource setup scripts without structural validation, and this PR is a focused hardening patch with sufficient terminal proof; the remaining decision is installer-owner acceptance of the stricter fail-closed shebang policy.

## What This Changes

The PR adds `validate_downloaded_script()` to `scripts/install.sh`, calls it before Homebrew and NodeSource setup-script execution, and adds a standalone shell validation harness.

## Best Possible Solution

Land centralized downloaded-script validation once installer owners accept the fail-closed shebang contract, or request a narrow compatibility prefix rule plus CI-backed malformed-download coverage before merge.

## Reproduction Assessment

Yes from source inspection. Current `main` downloads temp files in the Homebrew and NodeSource paths and executes them with bash without validating content first; I did not run a malformed-download reproduction in this read-only review.

## Solution Assessment

Likely yes, subject to installer-owner policy. Centralizing structural validation before every remote setup-script execution is the narrow maintainable fix, with the remaining choice being how strict the accepted script prefix should be.
