## Summary

Keep open. The installer hardening is useful and the supplied proof covers the main validation behavior, but the Homebrew `run_remote_bash` path still needs to guard a failed `download_file` before validation or execution because `run_quiet_step` invokes it in a shell context where `errexit` does not stop after the failed download.

## What This Changes

The PR adds shell-script structural validation for the Homebrew and NodeSource download-then-execute installer paths and adds a standalone shell validation harness.

## Best Possible Solution

Land the structural validator only after `run_remote_bash` checks `download_file` failure before any validation or execution and the harness covers a shebang-prefixed partial failed download.

## Reproduction Assessment

Yes, source-level. Current main and the PR merge result show the unguarded download-then-bash path, and a focused Bash check confirms `errexit` does not protect commands inside a function invoked from `if "$@"` in `run_quiet_step`.

## Solution Assessment

No, not yet. The structural validator is the right narrow direction, but it must first make failed downloads terminal before checking script shape or executing the temporary file.
