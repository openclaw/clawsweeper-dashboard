## Summary

Keep open for maintainer review: the patch addresses a real installer hardening gap and has sufficient terminal proof, with no blocking code defect found. The remaining decision is whether installer owners accept making third-party setup scripts fail closed when downloaded content does not start with a shebang.

## What This Changes

The PR adds downloaded-script validation to `scripts/install.sh` before executing Homebrew and NodeSource setup scripts, plus a standalone malformed-download validation harness.

## Best Possible Solution

Land centralized downloaded-script validation after installer owners accept the strict shebang contract, or broaden only a narrow known-safe prefix before merge if they prefer a softer availability posture.

## Reproduction Assessment

Yes from source inspection: current `main` and `v2026.5.22` download Homebrew and NodeSource setup scripts to temp files and execute them with bash without validating content. The PR body also includes after-fix terminal proof against malformed responses.

## Solution Assessment

Yes, with maintainer acceptance: central structural validation is a narrow maintainable fix for malformed downloaded scripts. The unresolved part is the strict first-byte shebang policy, which is an installer availability tradeoff.

## Risks / Open Questions

- The strict first-line `#!` rule can break installs if Homebrew or NodeSource later publish a bash-compatible setup script with a BOM, leading comments, or no shebang.
