## Summary

The PR is useful installer hardening, but the latest head still needs a focused repair before merge because failed downloads can reach the new validation/execution path when Bash `errexit` is suppressed by `run_quiet_step`'s conditional command execution.

## What This Changes

The PR adds `validate_downloaded_script` to `scripts/install.sh` for Homebrew and NodeSource downloaded setup scripts and adds a standalone shell validation harness.

## Best Possible Solution

Keep the structural validation, but add explicit failed-download guards before every validation/execution decision and prove both failed download and bad-content paths.

## Reproduction Assessment

Yes. Source inspection plus the Bash `set -e` conditional-function check gives a high-confidence reproduction path for the failed-download continuation risk in `run_remote_bash`.

## Solution Assessment

Not yet. The shebang and diagnostic validation is the right narrow direction, but it is incomplete until failed downloads are explicitly stopped before validation or execution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Stop failed downloads before validating Homebrew scripts:** `scripts/install.sh:121-122`
  - body: `run_remote_bash` now validates the temp file after `download_file`, but it still ignores the download command's status. Because `run_quiet_step` invokes this function from a conditional, Bash `errexit` does not stop the function; a failed curl/wget that leaves a partial file starting with `#!` can still pass validation and reach `/bin/bash`. Return immediately when `download_file` fails before validating or executing the temp file.
  - confidence: 0.87
- **[P2] Guard NodeSource downloads explicitly too:** `scripts/install.sh:1698-1699`
  - body: The NodeSource paths add validation after the download step, but the download status is still only implicit through `run_quiet_step`/`errexit`. Make each setup-script download use an explicit `run_quiet_step ... || return 1` guard before `validate_downloaded_script` so these download-then-execute paths do not depend on caller `errexit` context.
  - confidence: 0.72
