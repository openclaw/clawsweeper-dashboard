## Summary

Keep open: the PR is useful installer hardening, but `run_remote_bash` can still validate and execute a shebang-prefixed partial file after a failed downloader when invoked through `run_quiet_step`.

## What This Changes

The PR adds structural shell-script validation for Homebrew and NodeSource downloaded setup scripts and adds a shell harness for malformed download fixtures.

## Best Possible Solution

Land the hardening after every download-then-execute path aborts on downloader failure before validation and the harness covers a shebang-prefixed partial-download failure.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `run_quiet_step` invokes `run_remote_bash` through an `if "$@"` wrapper, so failed `download_file` status can be ignored before the new validation line.

## Solution Assessment

No. The structural validation direction is good, but the patch needs to abort on downloader failure before validation/execution and cover that failed-transfer path.
