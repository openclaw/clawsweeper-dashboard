## Summary

This PR is still necessary: current main and the latest release still execute downloaded Homebrew and NodeSource setup scripts without content validation, and the proposed hardening is focused. There are no blocking code-review findings, but maintainers should explicitly accept the strict fail-closed shebang policy before merge.

## What This Changes

The branch adds `validate_downloaded_script()` to `scripts/install.sh` before Homebrew and NodeSource setup scripts are executed, plus a standalone shell validation harness.

## Best Possible Solution

Land centralized downloaded-script validation after installer owners accept the strict shebang contract, or narrow the accepted prefix and add malformed-download coverage to the existing installer tests if reducing future upstream-format outages matters more.

## Reproduction Assessment

Yes from source inspection: current main and `v2026.5.22` download Homebrew and NodeSource setup scripts into temp files and invoke bash on them without content validation. I did not run a malformed-download repro in the checkout because this review is read-only.

## Solution Assessment

Mostly yes: centralized validation before execution is the right maintainable shape for the installer. The strict first-byte shebang rule is the remaining maintainer policy choice because it trades compatibility for fail-closed safety.
