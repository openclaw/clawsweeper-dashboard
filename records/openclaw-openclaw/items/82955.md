## What This Changes

The PR adds `validate_downloaded_script` to reject empty or non-shebang downloaded installer scripts before executing Homebrew and NodeSource setup paths, plus a standalone shell validation harness.

## Best Possible Solution

Land the structural validation after installer owners accept the shebang fail-closed contract, with durable installer-lane coverage added first if maintainers want this guarded in CI.

## Reproduction Assessment

Yes, at source level: current main downloads Homebrew and NodeSource setup scripts into temp files and executes them with bash without validating downloaded content. I did not run a full installer repro because this review was read-only.

## Solution Assessment

Yes, with a maintainer policy caveat: non-empty plus first-line shebang validation and sanitized diagnostics is a narrow hardening layer, but unconditional fail-closed installer behavior needs owner acceptance.
