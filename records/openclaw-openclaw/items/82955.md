## Summary

No blocking correctness finding: current main still executes four downloaded installer scripts without structural validation, and this PR adds a focused fail-closed check plus diagnostic sanitization. It should stay open for maintainer merge judgment because the safer behavior can turn malformed or changed upstream installer responses into explicit install failures.

## What This Changes

This PR adds `validate_downloaded_script()` to `scripts/install.sh` before running Homebrew and NodeSource setup scripts, plus an executable shell harness for malformed-download validation.

## Best Possible Solution

Land the hardening after maintainer acceptance of the fail-closed installer behavior, preserving the current main installer fixes in the final merge result.

## Reproduction Assessment

Yes. Current main clearly downloads and executes the Homebrew and NodeSource setup scripts without validating content first; the contributor also supplied terminal proof for empty, HTML, escape, and valid-script cases on the patched path.

## Solution Assessment

Yes, with maintainer acceptance of the compatibility tradeoff. Non-empty plus raw shebang validation is a narrow maintainable hardening here, and checksum pinning would be brittle for frequently updated third-party setup scripts.
