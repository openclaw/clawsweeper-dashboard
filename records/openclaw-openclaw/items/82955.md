## What This Changes

The PR adds non-empty and shebang validation to `scripts/install.sh` before executing the downloaded Homebrew installer script, plus a standalone shell validation harness.

## Best Possible Solution

Keep the structural guard, but require redacted terminal proof through the real patched HTTPS-only download path or an explicitly documented equivalent before merging.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: current main downloads the Homebrew installer into a tempfile and executes it with `/bin/bash` without checking for an empty or non-shell response. I did not run a mutating installer flow in this read-only review.

## Solution Assessment

Mostly yes: non-empty and shebang validation is a narrow, maintainable guard and upstream Homebrew currently starts with `#!/bin/bash`. The remaining issue is proof quality, not the core patch shape.
