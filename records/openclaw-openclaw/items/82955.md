The PR adds `validate_downloaded_script()` to `scripts/install.sh`, calls it before Homebrew and NodeSource setup-script execution, and adds a standalone shell validation harness.

## Best Possible Solution

Land the hardening once installer owners accept shebang-as-contract behavior, optionally moving the malformed-download cases into an installer test lane.

## Solution Assessment

Yes, with a maintainer policy call. Non-empty plus shebang validation is a narrow maintainable hardening path that matches today’s upstream scripts, but installer owners need to accept the fail-closed compatibility behavior.
