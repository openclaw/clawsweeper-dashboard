## Summary

This PR is still useful: current main and the latest release still execute downloaded Homebrew and NodeSource setup files without structural validation, while the branch adds a focused fail-closed check with sufficient terminal proof. It should stay open for maintainer review because the strict first-line shebang contract can affect installer availability and overlaps the same NodeSource area as another open installer PR.

## What This Changes

The PR adds a shared `validate_downloaded_script()` helper to `scripts/install.sh`, calls it before Homebrew and NodeSource setup-script execution, and adds a standalone malformed-download shell harness.

## Best Possible Solution

Land centralized downloaded-script validation after installer/security owners accept the strict fail-closed script-format contract and reconcile the overlapping NodeSource installer PR.

## Reproduction Assessment

Yes. Current main is source-reproducible because `scripts/install.sh` downloads Homebrew and NodeSource setup files into temp files and executes them with Bash without checking content; the PR body supplies after-fix terminal proof, though I did not run a full installer E2E.

## Solution Assessment

Yes, with maintainer acceptance of the policy tradeoff. Centralizing non-empty/shebang validation before execution is a narrow hardening path; the remaining decision is strictness and whether to promote the harness into maintained CI.
