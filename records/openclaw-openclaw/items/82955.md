## Summary

Keep open: the patch is a focused security hardening change and current main plus the latest release still execute downloaded Homebrew and NodeSource setup scripts without structural validation. The remaining blocker is maintainer/security acceptance of the intentional fail-closed installer behavior, not a narrow code defect.

## What This Changes

The PR adds a shared downloaded-script validator to `scripts/install.sh`, applies it before Homebrew and NodeSource setup-script execution, adds a shell proof harness, and updates adjacent installer tests.

## Best Possible Solution

Land this focused validation change after installer/security owner acceptance, then let the linked installer-validation issue close through the merged implementation.

## Reproduction Assessment

Yes, at source level. Current main and `v2026.6.10` still pass Homebrew and NodeSource downloaded temp files to bash without structural validation; I did not run a mutating live installer repro in this read-only review.

## Solution Assessment

Yes, subject to installer/security owner acceptance. The shared validator is narrow for the four bash-executed download paths; checksum pinning would be brittle for frequently updated upstream setup scripts, and `run_required_step` only handles failed downloads rather than malformed successful responses.
