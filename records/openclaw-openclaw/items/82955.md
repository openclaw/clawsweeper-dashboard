## Summary

Keep open: current main and the latest release still execute downloaded installer scripts without structural validation, and this PR addresses the right installer paths with sufficient terminal proof. The remaining merge decision is owner acceptance of strict fail-closed shebang validation, plus one small portability issue in the supplemental harness.

## What This Changes

The PR adds shell-script validation before executing downloaded Homebrew and NodeSource setup scripts and adds a standalone validation harness.

## Best Possible Solution

Land this after installer/security owner acceptance of the fail-closed shebang contract, with the harness made portable and the NodeSource edits rebased or combined if https://github.com/openclaw/openclaw/pull/82926 lands first.

## Reproduction Assessment

Yes. Source inspection shows current main and v2026.5.22 execute downloaded Homebrew and NodeSource temp files with bash before validating content; I did not run the unsafe current-main installer path during this read-only review.

## Solution Assessment

Mostly yes. Structural non-empty plus shebang validation is a narrow maintainable hardening step for dynamic third-party setup scripts, but maintainers still need to accept the stricter fail-closed contract or choose a looser validator.
