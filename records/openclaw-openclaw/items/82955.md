## Summary

Keep open for maintainer review: the PR is focused and has sufficient terminal proof, but it intentionally changes install-time remote-script handling by failing closed on downloaded content that lacks a first-line shebang.

## What This Changes

Adds `validate_downloaded_script` to `scripts/install.sh` for Homebrew and NodeSource downloaded setup scripts, plus a standalone shell validation harness.

## Best Possible Solution

Land the hardening once installer owners accept the first-line shebang fail-closed contract and decide whether to promote the harness into an existing installer regression lane.

## Reproduction Assessment

Yes. Current main’s `run_remote_bash` accepted an empty downloaded temp file with exit 0 in a focused shell probe, and the source shows the NodeSource paths also execute downloaded temp files without validation.

## Solution Assessment

Yes, with a maintainer policy call. Non-empty plus leading-shebang validation is a narrow hardening path that matches current upstream scripts, but the unconditional fail-closed behavior needs installer-owner acceptance.
