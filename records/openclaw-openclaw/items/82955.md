## Label Justifications

- `P2`: This is a normal-priority installer hardening PR with limited but real security and install reliability impact.
- `merge-risk: 🚨 compatibility`: The new first-line shebang requirement can reject upstream scripts that are still bash-compatible but start with a BOM, blank line, or comment.
- `merge-risk: 🚨 availability`: A future upstream formatting change would make OpenClaw installation fail closed before Node or Homebrew setup runs.

## Summary

Keep this PR open: the hardening is still useful because current main and the latest release still execute downloaded installer scripts without structural validation, but the strict fail-closed shebang policy needs installer-owner acceptance before merge.

## What This Changes

The PR adds `validate_downloaded_script()` to `scripts/install.sh`, uses it before Homebrew and NodeSource setup scripts are run, and adds a standalone shell validation harness.

## Best Possible Solution

Land centralized downloaded-script validation once installer owners accept the strict shebang contract, or narrow it to explicitly allowed safe prefixes while still rejecting empty, HTML, JSON, binary, and diagnostic-injection payloads.

## Reproduction Assessment

Yes. Current main clearly downloads temp files and invokes Bash directly in the Homebrew and NodeSource paths, and the PR body supplies after-fix terminal proof against malformed local HTTP and HTTPS responses.

## Solution Assessment

Yes, with maintainer acceptance of the strict fail-closed contract. A centralized helper in the installer is the maintainable fix; the safer alternative is to allow only explicitly documented safe prefixes if installer owners are concerned about upstream formatting drift.
