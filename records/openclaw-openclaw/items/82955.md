## Summary

Keep open for maintainer review: the PR addresses a real installer hardening gap and I found no blocking correctness or security regression, but landing it should be an explicit installer-owner decision because it changes remote setup-script execution from permissive to fail-closed on the first-line shebang check.

## What This Changes

The PR adds downloaded-script validation to `scripts/install.sh`, applies it before Homebrew and NodeSource setup-script execution, sanitizes rejected first-line diagnostics, and adds a standalone shell validation harness.

## Best Possible Solution

Land centralized downloaded-script validation once installer owners accept the strict shebang contract, or adjust it to allow narrowly documented safe prefixes while still rejecting empty, HTML, JSON, binary, and diagnostic-injection payloads.

## Reproduction Assessment

Yes. Source inspection of current main shows `run_remote_bash` and the three NodeSource branches execute downloaded temp files with Bash without structural validation, and the PR body supplies after-fix terminal proof against malformed responses.

## Solution Assessment

Yes, with an owner-review caveat. Centralizing validation at the download-then-execute boundary is the narrow maintainable fix, but the strict first-line shebang contract should be explicitly accepted or adjusted before merge.
