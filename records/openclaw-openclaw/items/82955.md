## Summary

Keep open for maintainer review: this is a focused installer hardening PR with sufficient terminal proof and no blocking code findings, but the fail-closed shebang policy needs installer-owner acceptance because it can turn upstream script-format drift into a fresh-install failure.

## What This Changes

The PR adds `validate_downloaded_script()` to `scripts/install.sh`, calls it before Homebrew and NodeSource setup-script execution, and adds a standalone shell validation harness.

## Best Possible Solution

Keep validation before every remote setup-script execution, then land after installer owners accept the fail-closed shebang contract or request a compatibility-preserving rule with wired regression coverage.

## Reproduction Assessment

Yes. Current-main source inspection shows the Homebrew and NodeSource paths download temp files and execute them with bash without structural validation; the PR body also supplies after-fix terminal proof for malformed responses.

## Solution Assessment

Yes, if maintainers accept the fail-closed installer contract. Otherwise the safer path is to keep the malformed-response rejection while documenting and testing any compatibility allowance for bash-compatible scripts without a first-line shebang.
