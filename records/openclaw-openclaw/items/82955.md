## What This Changes

The PR adds `validate_downloaded_script` to guard Homebrew and NodeSource downloaded scripts before `bash` execution, plus `scripts/test-install-validation.sh` coverage for malformed downloads and diagnostic sanitization.

## Best Possible Solution

Land the narrow structural validation once maintainers accept the fail-closed installer behavior and CI is green or waived; keep checksum pinning out of these frequently changing third-party setup-script paths.

## Reproduction Assessment

Yes from source: current main downloads Homebrew and NodeSource setup scripts to temp files and executes them with bash without content validation. I did not run the current-main installer path locally because this review is read-only, but the vulnerable control flow is direct.

## Solution Assessment

Yes, with maintainer acceptance of the compatibility tradeoff: non-empty plus raw shebang validation and sanitized diagnostics are a narrow hardening that avoids brittle checksum pins. The safer alternative, if fail-closed behavior is too strict, is a provider-specific compatibility allowance with tests.
