## What This Changes

This PR adds `validate_downloaded_script()` to `scripts/install.sh`, calls it before Homebrew and NodeSource downloaded setup scripts are executed, and adds `scripts/test-install-validation.sh`.

## Best Possible Solution

Land the scoped validator if maintainers accept fail-closed installer behavior, coordinate it with the adjacent NodeSource installer PR, and treat signed or checksum-pinned third-party provenance as separate trust-model work.

## Reproduction Assessment

Yes, by source inspection. Current main downloads the Homebrew and NodeSource setup scripts to temp files and executes them with bash without structural validation.

## Solution Assessment

Yes, with maintainer acceptance of the compatibility tradeoff. A shared non-empty/shebang validator with sanitized diagnostics is a narrow hardening path; full signing or checksum pinning would be broader trust-model work.
