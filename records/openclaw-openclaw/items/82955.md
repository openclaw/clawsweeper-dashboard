## Label Justifications

- `P2`: This is a normal-priority installer security hardening with limited surface area but meaningful compatibility review needed.
- `merge-risk: 🚨 compatibility`: The new first-line shebang requirement can reject future upstream setup scripts that bash might otherwise run successfully.
- `merge-risk: 🚨 availability`: A rejected Homebrew or NodeSource setup script would stop first-time installs or Node bootstrapping before OpenClaw is installed.

## Summary

Keep open. Current main still executes the downloaded installer payloads directly, and the PR’s terminal proof plus source inspection support the hardening; the remaining blocker is maintainer acceptance of the stricter install failure contract.

## What This Changes

Adds `validate_downloaded_script()` to `scripts/install.sh` before Homebrew and NodeSource setup-script execution, plus a standalone shell validation harness.

## Best Possible Solution

Land centralized downloaded-script validation once installer owners accept the strict shebang contract, or narrow it to explicitly reviewed safe prefixes while still rejecting empty, HTML, JSON, binary, and diagnostic-injection payloads.

## Reproduction Assessment

Yes. Source inspection of current main shows the Homebrew and NodeSource paths download into temp files and invoke Bash directly; the PR body supplies after-fix terminal proof against malformed local HTTP and trusted self-signed HTTPS responses.

## Solution Assessment

Yes, with maintainer acceptance of the strict fail-closed contract. A centralized helper is the narrow maintainable fix; the safer alternative is to explicitly allow reviewed benign prefixes before merge.
