## Summary

Keep open: the PR hardens the Homebrew `run_remote_bash` path and includes sufficient terminal proof, but Linux NodeSource setup-script downloads are still executed with `bash` without the same validation, leaving the installer security boundary incomplete.

## What This Changes

The PR adds non-empty/shebang validation and sanitized first-line diagnostics to `scripts/install.sh`'s `run_remote_bash`, plus a standalone installer validation harness.

## Best Possible Solution

Extract a reusable downloaded-shell-script validator and call it before every installer path that executes a downloaded setup script, preserving current sudo/root behavior and avoiding checksum pinning.

## Reproduction Assessment

Yes, by source inspection. Current main downloads remote setup scripts to temp files and executes them with bash, and the PR body includes terminal proof that the patched helper rejects empty, HTML, and escape-payload responses.

## Solution Assessment

No. Validating `run_remote_bash` is the right direction for Homebrew, but the maintainable fix should reuse the same validator for the NodeSource apt/dnf/yum setup-script paths too.
