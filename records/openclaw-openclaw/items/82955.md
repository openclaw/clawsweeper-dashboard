## Label Justifications

- `P2`: This is a normal-priority installer security hardening change with limited surface but real fresh-install impact.
- `merge-risk: 🚨 compatibility`: The new `#!`-at-byte-zero rule can reject future third-party setup scripts whose format is still executable by Bash but no longer matches this stricter contract.
- `merge-risk: 🚨 availability`: A rejected setup script would make the installer fail before Homebrew or NodeSource configuration, affecting fresh installs and Node auto-install flows.

## Summary

Keep open for maintainer review. The PR addresses a real installer hardening gap with sufficient terminal proof, but changing remote setup-script execution to strict shebang validation is compatibility- and availability-sensitive and overlaps the open NodeSource installer work at https://github.com/openclaw/openclaw/pull/82926.

## What This Changes

Adds a reusable downloaded-shell-script validator to `scripts/install.sh` before Homebrew and NodeSource setup-script execution, plus a standalone shell harness for malformed-download cases.

## Best Possible Solution

Land centralized downloaded-script validation after installer/security owners accept the strict shebang contract, reconcile the NodeSource overlap, and preferably carry a small CI-visible regression check for the validator behavior.

## Reproduction Assessment

Yes. Current main and the latest release both show downloaded Homebrew and NodeSource setup scripts being executed with Bash before content validation, and the PR body supplies after-fix terminal proof against malformed HTTP/HTTPS responses.

## Solution Assessment

Yes, if maintainers accept the strict fail-closed script-format contract. Centralizing validation before every downloaded-script execution point is the narrow maintainable fix; the safer alternative is to tolerate reviewed harmless prefixes and add integrated test coverage before landing.
