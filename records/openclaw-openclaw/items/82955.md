## Summary

Keep this PR open for maintainer review. The patch is a focused installer hardening change with sufficient terminal/live-output proof and no blocking code findings, but it intentionally changes installer failure behavior from executing whatever was downloaded to failing closed on empty or non-shebang setup scripts.

## What This Changes

The PR adds structural validation for downloaded shell scripts in `scripts/install.sh` before Homebrew and NodeSource setup execution, plus a shell harness covering malformed downloads and sanitized diagnostics.

## Best Possible Solution

Land the bounded validation if maintainers accept fail-closed installer hardening, keeping the check local to the download-then-execute paths and preserving the current Homebrew/NodeSource URLs.

## Reproduction Assessment

Yes. Current main clearly downloads Homebrew and NodeSource setup scripts and executes the temp file with bash without validating content first; I did not run the failing current-main path during this read-only review.

## Solution Assessment

Yes, with a maintainer decision. Non-empty plus raw shebang validation is a narrow, maintainable hardening step for these download-then-execute paths, but the fail-closed behavior is an intentional compatibility/availability tradeoff.

## Risks / Open Questions

- Merging this changes the installer to fail closed before Homebrew or NodeSource setup execution when the downloaded file is empty, malformed, or does not start with a shebang; that is the security-hardening goal, but it can make installs stop in cases that previously continued into bash.
- The fail-closed behavior depends on the third-party setup-script shape remaining shebang-first; current Homebrew and NodeSource scripts match that contract, but future upstream drift or transient CDN/proxy responses would become immediate installer failures.
