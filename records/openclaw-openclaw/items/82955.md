## Summary

The PR is useful installer hardening, but it still needs one narrow correctness repair before merge: the Homebrew `run_remote_bash` path validates after an unchecked downloader failure, so a shebang-prefixed partial download can still be executed.

## What This Changes

The PR adds structural validation for downloaded Homebrew and NodeSource installer scripts before execution and adds a shell validation harness.

## Best Possible Solution

Ship the structural validation after downloader failures short-circuit before validation or execution, with focused coverage for that failed-partial-download case and maintainer acceptance of the fail-closed installer behavior.

No. The structural validator is the right direction, but the branch should preserve downloader failure semantics before validation or execution.
