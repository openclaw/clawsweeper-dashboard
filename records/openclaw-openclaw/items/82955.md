## Summary

Keep open: current main and the latest release still execute the downloaded Homebrew and NodeSource setup scripts without structural validation, while this PR is a focused hardening patch with sufficient terminal proof. The remaining issues are a small portability defect in the supplemental harness and maintainer acceptance of the stricter fail-closed installer behavior.

## What This Changes

Adds `validate_downloaded_script()` to `scripts/install.sh`, calls it before Homebrew and NodeSource downloaded setup scripts are executed, and adds a Bash validation harness.

## Best Possible Solution

Keep the downloaded-script validation and diagnostic sanitization, make the supplemental harness portable, and have maintainers explicitly accept the stricter installer policy before merge.

## Reproduction Assessment

Yes from source inspection: current main downloads the Homebrew and NodeSource setup scripts and executes the temp file without checking for empty or non-shebang content. I did not execute the full installer path in this read-only review.

## Solution Assessment

Mostly yes: non-empty and shebang validation is a narrow hardening of the implicated paths, and the PR sanitizes rejected diagnostics. The best merge path is to make the harness portable and confirm the intended fail-closed installer policy.
