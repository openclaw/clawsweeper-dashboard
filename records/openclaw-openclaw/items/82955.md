## What This Changes

The branch adds `validate_downloaded_script()` to `scripts/install.sh`, calls it before Homebrew and NodeSource setup scripts execute, and adds `scripts/test-install-validation.sh`.

## Best Possible Solution

Keep the validation localized to the four remote-script execution paths, make the harness portable, and have install owners explicitly accept the fail-closed behavior before merge.

## Reproduction Assessment

Yes. Current main and `v2026.5.22` show the Homebrew and NodeSource download paths running temp files with bash without validating the downloaded content first.

## Solution Assessment

Yes, with a caveat. Non-empty plus shebang validation is a bounded hardening step without brittle checksum pins, but maintainers still need to accept the fail-closed installer policy and the harness should avoid GNU-only `grep -P`.
