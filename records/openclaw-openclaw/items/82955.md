## Summary

Keep open: the installer hardening is focused and mostly correct, but the added harness still has a low-probability nondeterministic binary-fixture flake and the stricter fail-closed installer behavior remains maintainer-visible merge risk.

## What This Changes

The PR adds structural validation and failed-download guards before executing Homebrew and NodeSource downloaded installer scripts, plus a shell harness for bad-download cases.

## Best Possible Solution

Keep the validation and failed-download guards, make the binary fixture deterministic, and land only after maintainers accept the stricter fail-closed installer policy.

## Reproduction Assessment

Yes. Source inspection shows current main downloads Homebrew and NodeSource setup scripts to temp files and executes them without validating the downloaded content; the PR body supplies terminal proof against local bad-download fixtures.

## Solution Assessment

Mostly yes. The narrowest maintainable fix is to reject failed or non-script downloads before bash execution, but the harness should use deterministic bytes and maintainers should explicitly accept the fail-closed installer behavior.
