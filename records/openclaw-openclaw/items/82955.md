## Summary

Keep open for maintainer merge review: the current head addresses the earlier mechanical blockers and I found no blocking correctness defect, but this intentionally changes installer behavior to fail closed around third-party downloaded scripts.

## What This Changes

The PR adds `validate_downloaded_script()` to `scripts/install.sh` before Homebrew and NodeSource downloaded-script execution, plus a shell harness for malformed downloads and sanitized diagnostics.

## Best Possible Solution

Land the validator if installer owners accept fail-closed structural validation for these four remote-script paths, with broader platform install smoke as the final confidence check if desired.

## Reproduction Assessment

Yes. Current main has a clear source-reproducible path: `download_file` writes remote Homebrew/NodeSource bodies to temp files and the installer then invokes `bash` on those files without validating the content.

## Solution Assessment

Yes, with a maintainer policy caveat. Raw-byte shebang plus non-empty validation is a narrow hardening step for frequently changing third-party scripts where checksum pinning would be brittle, but installer owners still need to accept the fail-closed behavior.
