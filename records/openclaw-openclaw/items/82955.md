## Summary

Keep this PR open for changes before merge. The hardening target is real on current main and the proof is useful, but the proposed validator still checks a shell string after Bash command substitution, so malformed raw bytes can bypass the proposed shebang gate; the added harness also has one portability issue for macOS validation.

## What This Changes

The PR adds downloaded-script validation before the Homebrew and NodeSource installer setup scripts execute, plus a shell harness for invalid downloads and sanitized diagnostics.

## Best Possible Solution

Keep the installer hardening direction, but repair the validator to check raw leading bytes before shell string handling and make the harness portable before maintainers decide whether to accept the stricter fail-closed installer policy.

## Reproduction Assessment

Yes for the PR blocker: a focused shell probe shows Bash command substitution drops a leading NUL before the proposed `#!` comparison. Current main and the latest release also show the unvalidated download-then-execute paths this PR is trying to harden.

## Solution Assessment

Not yet. The shared validator is the right maintainable shape, but it should check raw bytes before shell string handling and avoid GNU-only test assertions before the fail-closed installer policy is accepted.

  - body: The validator reads the downloaded header through Bash command substitution before checking for `#!`.
