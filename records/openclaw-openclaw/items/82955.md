## Summary

Keep this PR open: current main still executes downloaded Homebrew and NodeSource setup scripts without structural validation, while the PR adds a focused guard and proof. I found no blocking patch defect, but the fail-closed installer behavior is compatibility/availability-sensitive and should remain a maintainer decision before merge.

## What This Changes

The PR adds `validate_downloaded_script()` before executing downloaded Homebrew and NodeSource setup scripts and adds a standalone shell validation harness.

## Best Possible Solution

Land the focused validation if maintainers accept the fail-closed installer policy, ideally after a current-base smoke that confirms Homebrew and NodeSource live scripts still pass and malformed responses fail clearly.

## Reproduction Assessment

Yes from source: current main downloads remote installer scripts and executes them without validating that the downloaded file is non-empty shell-script content. I did not run the installer because this review was read-only, but the affected execution path is direct.

## Solution Assessment

Yes, with maintainer policy acceptance: the raw-byte non-empty shebang check and sanitized diagnostics are a narrow guard on the implicated execution paths. A checksum pin would be stronger but is not practical for frequently changing third-party setup scripts.
