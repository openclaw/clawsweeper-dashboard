## Summary

Keep open for maintainer review: current main still executes downloaded Homebrew and NodeSource scripts without structural validation, and the PR is focused with sufficient terminal proof, but the strict shebang fail-closed contract can block installs if upstream script formats change.

## What This Changes

The PR adds structural validation for downloaded installer scripts in `scripts/install.sh` and adds a standalone shell harness for malformed-download cases.

## Best Possible Solution

Land centralized downloaded-script validation after owner acceptance of the strict shebang policy, or narrowly widen the validator for reviewed harmless prefixes while keeping malformed-download rejection.

## Reproduction Assessment

Yes. Current main is source-reproducible because `scripts/install.sh` downloads temp files and runs `/bin/bash`, `bash "$tmp"`, or `sudo bash "$tmp"` on the Homebrew and NodeSource paths; the PR body includes after-fix terminal proof against malformed HTTP and HTTPS responses.

## Solution Assessment

Yes, with maintainer acceptance. The centralized validator is a narrow maintainable fix; if owners dislike strict first-byte shebang validation, the safer alternative is tolerating explicitly reviewed harmless prefixes while continuing to reject empty and non-script payloads.

## Risks / Open Questions

- The validator intentionally aborts unless the first line starts with `#!`; if Homebrew or NodeSource later publish a Bash-compatible setup script with a UTF-8 BOM, leading comments, or another harmless prefix, OpenClaw installs can fail before package setup.
- This changes installer behavior from permissive execution to fail-closed validation for third-party setup downloads, so maintainers should explicitly accept the availability tradeoff.
- The malformed-download harness demonstrates the behavior but is not wired into the maintained Vitest, Docker, or installer CI lanes, so long-term regression coverage depends on maintainer choice.
