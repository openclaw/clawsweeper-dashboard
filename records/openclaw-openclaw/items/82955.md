## Summary

Keep open: this is useful installer hardening, but the Homebrew `run_remote_bash` path still validates after an unchecked `download_file` call, so a failed transfer that leaves shebang-prefixed partial content can still reach bash.

## What This Changes

The PR adds structural validation for downloaded installer scripts in `scripts/install.sh` and adds a standalone shell harness for validation cases.

## Best Possible Solution

Land the structural validator only after `run_remote_bash` checks `download_file` failure before any validation or execution and the harness covers a shebang-prefixed partial failed download.

## Reproduction Assessment

Yes, source-level. Current main and the PR head show the download-then-bash path, and a focused Bash check confirms `errexit` does not protect commands inside a function invoked from `if "$@"` in `run_quiet_step`.

## Solution Assessment

No, not yet. The structural validator is the right narrow direction, but failed downloads must become terminal before checking script shape or executing the temporary file.
