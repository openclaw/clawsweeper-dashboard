## Summary

This PR is still useful and not obsolete, but it should stay open because the current head still validates the Homebrew temp file after an unchecked downloader failure; a failed partial transfer that begins with a shebang can still reach execution.

## What This Changes

The branch adds `validate_downloaded_script()` to installer remote-script execution paths and adds a shell harness for malformed downloaded-script cases.

## Best Possible Solution

Keep the structural validation hardening, but return immediately on failed downloads before validating or executing temp files and cover that path in the harness.

## Reproduction Assessment

Yes. Source inspection plus a small shell-semantics probe gives a high-confidence path for the remaining issue: `run_remote_bash` can continue after a nonzero `download_file` when invoked through `run_quiet_step`.

## Solution Assessment

No, not yet. The validation direction is maintainable, but the downloader exit status must be honored as the first failure boundary before validating or executing temp-file content.

- **[P2] Honor failed downloads before validation:** `scripts/install.sh:122`
