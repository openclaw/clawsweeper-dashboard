## Summary

Keep this PR open for maintainer review: the hardening is focused and source-backed, with no blocking correctness finding, but it intentionally makes installer script downloads fail closed when the response lacks a first-line shebang.

## What This Changes

The PR adds non-empty and first-line-shebang validation before executing Homebrew and NodeSource downloaded installer scripts, plus a standalone shell validation harness.

## Best Possible Solution

Land the hardening after installer owners accept the first-line-shebang fail-closed policy and decide whether the malformed-download cases should move into the existing installer CI lane.

## Reproduction Assessment

Yes. Current main shows the Homebrew and NodeSource setup scripts are downloaded to temp files and executed by bash or sudo bash without structural validation; I verified that by source inspection rather than running the full install path.

## Solution Assessment

Yes, with a maintainer policy call. Non-empty plus first-line shebang validation is a narrow hardening path and live upstream scripts currently satisfy it, but unconditional fail-closed behavior needs installer-owner acceptance.
