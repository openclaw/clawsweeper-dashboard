## Summary

Keep open: the focused installer hardening is still useful and is not present on current `main` or `v2026.5.22`, but the added harness still has one portable-shell validation defect and the fail-closed installer behavior needs maintainer acceptance before merge.

## What This Changes

The branch adds `validate_downloaded_script` to `scripts/install.sh`, calls it before Homebrew and NodeSource setup script execution, and adds a standalone shell harness for malformed-download validation.

## Best Possible Solution

Land the localized validation after replacing `grep -P` in the harness and after installer owners explicitly accept fail-closed malformed-download behavior, with live macOS/Linux install smoke proof if availability remains the blocker.

## Reproduction Assessment

Yes, from source inspection. Current `main` and `v2026.5.22` download Homebrew and NodeSource setup scripts into temp files and execute them with bash without checking non-empty or shebang content first.

## Solution Assessment

Yes, with caveats. Non-empty plus shebang validation is bounded hardening without brittle checksum pins, but the harness should avoid GNU-only `grep -P` and maintainers must accept the fail-closed installer policy.
