## What This Changes

The PR adds downloaded-shell-script validation to `scripts/install.sh` for Homebrew and NodeSource setup downloads, plus a standalone shell harness covering malformed and valid downloads.

## Best Possible Solution

Land centralized downloaded-script validation once installer/security owners accept the strict shebang contract, or narrowly allow reviewed harmless prefixes while still rejecting empty, HTML, JSON, binary, and diagnostic-injection payloads.

## Solution Assessment

## Risks / Open Questions

- The validator intentionally aborts unless the downloaded file starts with `#!`; if Homebrew or NodeSource later publish a Bash-compatible setup script with a UTF-8 BOM, leading comments, or another harmless prefix, OpenClaw installs could fail before package setup.
- This changes installer behavior from permissive execution to fail-closed validation for third-party setup downloads, so maintainers should explicitly accept the availability tradeoff.
- The malformed-download harness demonstrates the behavior but is not wired into the maintained Vitest, Docker, or installer CI lanes, so long-term regression coverage depends on maintainer choice.
