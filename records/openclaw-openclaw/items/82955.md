## Summary

Keep this PR open for maintainer review: the patch is a focused installer hardening change with sufficient terminal proof and no blocking diff findings, but it intentionally changes malformed third-party setup-script responses from “execute anyway” to fail-closed.

## What This Changes

The PR adds a Bash validator for downloaded installer scripts, wires it into the Homebrew and NodeSource setup-script execution paths, and adds a standalone validation harness.

## Best Possible Solution

Land the scoped validator if maintainers accept fail-closed installer behavior, coordinate it with the related NodeSource installer PR, and treat signed or checksum-pinned third-party provenance as separate trust-model work.

## Reproduction Assessment

Yes, by source inspection. Current main downloads the Homebrew and NodeSource setup scripts to temp files and executes them with bash without structural validation.

## Solution Assessment

Yes, with maintainer acceptance of the compatibility tradeoff. A shared non-empty/shebang validator with sanitized diagnostics is a narrow hardening path; full signing or checksum pinning would be broader trust-model work.
