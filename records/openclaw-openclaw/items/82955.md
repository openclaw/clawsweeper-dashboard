## What This Changes

The branch adds `validate_downloaded_script()` to `scripts/install.sh`, calls it before Homebrew and NodeSource remote setup scripts run, and adds `scripts/test-install-validation.sh`.

## Best Possible Solution

Land the validation after fixing the portable harness check, with explicit maintainer acceptance that malformed or no-shebang third-party setup downloads should fail closed.

## Reproduction Assessment

## Solution Assessment

Mostly yes: a non-empty/shebang check with sanitized diagnostics is a narrow hardening path that avoids brittle checksum pins for changing third-party scripts. The harness should use a portable byte check before merge, and maintainers need to accept the fail-closed compatibility boundary.
