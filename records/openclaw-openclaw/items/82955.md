## Summary

Keep open for maintainer review: the patch is a focused installer hardening with no blocking code-review findings and sufficient terminal/live-output proof, but it intentionally changes Homebrew and NodeSource setup downloads to fail closed before execution.

## What This Changes

The PR adds `validate_downloaded_script()` to `scripts/install.sh` for Homebrew and NodeSource downloaded setup scripts, plus a standalone shell harness for malformed-download validation.

## Best Possible Solution

Land the hardening only after maintainers accept the fail-closed installer behavior and verify it against the latest `main` installer flow.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current `main` downloads remote setup scripts to temp files and executes them with bash before validating content; serving an empty or HTML 200 response would exercise the reported failure mode.

## Solution Assessment

Yes, with maintainer acceptance. Raw-byte shebang plus non-empty validation is a narrow hardening step and checksum pinning would be brittle for frequently updated third-party setup scripts, but the fail-closed behavior needs an explicit owner decision.
