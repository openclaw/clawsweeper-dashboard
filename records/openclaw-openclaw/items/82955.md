## What This Changes

The PR adds `validate_downloaded_script` to `scripts/install.sh` for Homebrew and NodeSource downloaded setup scripts, plus a shell validation harness.

## Best Possible Solution

Land the localized validation after making the harness portable and after installer owners explicitly accept fail-closed malformed-download behavior, with live install smoke proof if they need more availability assurance.

## Reproduction Assessment

Yes. Current main and `v2026.5.22` download Homebrew and NodeSource setup scripts into temp files and execute them with bash without validating the body first; the PR body supplies after-fix terminal proof for malformed and valid responses.

## Solution Assessment

Yes, with caveats. Non-empty plus shebang validation is bounded hardening without brittle checksum pins, but the harness should avoid GNU-only `grep -P` and maintainers must accept the fail-closed installer policy.
