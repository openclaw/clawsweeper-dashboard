## Summary

Keep open: the installer hardening is still useful and not on current main, and the PR has sufficient terminal proof. The remaining actionable issue is a small portability bug in the added harness, while the fail-closed installer behavior needs maintainer acceptance before merge.

## What This Changes

The PR adds structural validation before executing downloaded Homebrew and NodeSource setup scripts and adds a shell harness for malformed-download cases.

## Best Possible Solution

Keep the validation at the download-execute boundary, make the harness portable across macOS and Linux, and have maintainers explicitly accept the fail-closed installer behavior before merge.

## Reproduction Assessment

Yes from source inspection: current main downloads Homebrew and NodeSource setup scripts to temp files and executes them without checking for empty or non-shell content first. I did not run the installer or the added harness because this review was required to keep the checkout read-only.

## Solution Assessment

Mostly yes: validating immediately before execution is the narrow maintainable boundary for malformed downloads. The production direction is sound, but the test harness should be portable and the fail-closed policy needs maintainer acceptance.
