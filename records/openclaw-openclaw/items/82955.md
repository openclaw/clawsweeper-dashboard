## Summary

Keep open for maintainer review: the patch is a focused installer hardening change with sufficient terminal proof and no blocking correctness findings, but it intentionally makes malformed or non-shebang upstream installer downloads fail before execution.

## What This Changes

The branch adds a shared shell-script download validator to `scripts/install.sh`, runs it before Homebrew and NodeSource setup scripts execute, and adds a standalone validation harness.

## Best Possible Solution

Land the scoped validator if maintainers accept fail-closed installer behavior, coordinate it with the related NodeSource installer PR, and treat signed or checksum-pinned provenance as separate trust-model work.

## Reproduction Assessment

Yes, by source inspection. Current main downloads the Homebrew and NodeSource setup scripts to temp files and executes them with bash without checking that the body is non-empty or script-shaped.

## Solution Assessment

Yes, with maintainer acceptance of the compatibility tradeoff. A shared non-empty/shebang validator with sanitized diagnostics is a narrow hardening path; full signing or checksum pinning would be broader trust-model work.
