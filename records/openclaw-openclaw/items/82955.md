## Summary

Keep open for maintainer review: the patch appears correct and well-proven, but it intentionally changes installer behavior to fail closed when a downloaded setup script is empty or lacks a first-line shebang, so installer owners should accept that compatibility contract before merge.

## What This Changes

The branch adds `validate_downloaded_script` to `scripts/install.sh`, calls it before executing Homebrew and NodeSource setup downloads, and adds a standalone shell validation harness.

## Best Possible Solution

Land the structural validation once installer owners accept the fail-closed shebang contract, with durable installer-lane coverage added first if maintainers want regression proof in CI.

## Reproduction Assessment

Yes, at source level: current main downloads Homebrew and NodeSource setup scripts into temp files and executes them with bash without validating downloaded content. I did not run a full installer repro in this read-only review.

## Solution Assessment

Yes, with a maintainer policy caveat: non-empty plus first-line shebang validation and sanitized diagnostics is a narrow hardening layer, but unconditional fail-closed installer behavior needs owner acceptance.
