## Summary

Keep this PR open for maintainer review: current main still executes the Homebrew and NodeSource downloaded scripts without structural validation, and the PR’s terminal proof shows the patched path rejecting malformed downloads while accepting valid scripts. I found no blocking code-review findings, but the fail-closed shebang requirement is an installer compatibility and availability policy choice before merge.

## What This Changes

The PR adds `validate_downloaded_script()` to `scripts/install.sh`, calls it before Homebrew and NodeSource setup-script execution, and adds a standalone shell validation harness.

## Best Possible Solution

Keep validation before every remote setup-script execution, then land after installer owners accept the fail-closed shebang contract or request a compatibility-preserving rule with wired regression coverage.

## Reproduction Assessment

Yes. Current-main source inspection shows the Homebrew and NodeSource paths download temp files and execute them with bash without structural validation, and the PR body supplies after-fix terminal proof for malformed responses.

## Solution Assessment

Yes, if maintainers accept the fail-closed installer contract. Otherwise the safer variant is to keep empty and HTML rejection while defining a narrow compatibility rule for harmless script prefixes and moving the regression cases into the existing installer test lane.
