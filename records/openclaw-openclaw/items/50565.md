## Summary

Keep open. Current main no longer has the old Feishu-specific tool-call wording, but embedded tool lifecycle logs still inherit the static `agent/embedded` logger because the subscription layer is not given the originating channel identity.

## What This Changes

The issue asks for tool lifecycle log entries to use the originating channel subsystem instead of a hardcoded or static logger.

## Best Possible Solution

Thread a typed, normalized channel-origin or logger from `runEmbeddedAttempt` into `subscribeEmbeddedPiSession`, use `gateway/channels/<channel>` for valid message-channel runs, keep `agent/embedded` for channel-less runs, and add focused regression coverage.

## Reproduction Assessment

Yes, at source level. Current main carries channel identity on the runner params, omits it from subscription params, installs a static `agent/embedded` logger, and uses that logger for tool start/end debug lines.

## Solution Assessment

Yes. The maintainable fix is a narrow typed channel-origin or logger seam between the runner and subscriber, not per-channel branches inside tool handlers.
