## Summary

Keep open. The old Feishu-specific log text is gone, but current main still routes embedded tool lifecycle debug logging through a static `agent/embedded` logger instead of the originating channel subsystem.

## What This Changes

Fix tool lifecycle log attribution so channel-originated tool calls are logged under the originating channel subsystem.

## Best Possible Solution

Thread a typed normalized channel origin or logger from `runEmbeddedAttempt` into `subscribeEmbeddedPiSession`, use `gateway/channels/<channel>` for valid channel-originated runs, and keep `agent/embedded` for channel-less runs.

## Reproduction Assessment

Yes, at source level. Current main carries channel identity on the runner params, omits it from subscription params, installs a static `agent/embedded` logger, and uses that logger for tool start/end debug lines.

## Solution Assessment

Yes. The narrow maintainable fix is to pass a typed channel-origin or logger seam into the subscriber; per-channel branches inside tool handlers would be the wrong ownership boundary.
