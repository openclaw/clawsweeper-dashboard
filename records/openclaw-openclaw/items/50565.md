## Summary

Keep open. The old Feishu-specific `tool call` text is gone, but current `main` and the latest release still attribute embedded tool lifecycle debug logs to a static `agent/embedded` logger because the originating channel is not passed into the subscription layer.

## What This Changes

Request that tool call lifecycle logs use the originating channel subsystem instead of being attributed to Feishu or a static embedded logger.

## Best Possible Solution

Thread a typed, normalized channel origin or logger from `runEmbeddedAttempt` into `subscribeEmbeddedPiSession`, use `gateway/channels/<channel>` for valid message-channel tool lifecycle logs, and keep `agent/embedded` as the fallback for channel-less runs.

## Reproduction Assessment

Yes, at source level. Current main carries `messageChannel`/`messageProvider` on the run params, omits them from subscription params, installs a static `agent/embedded` logger, and uses that logger for tool start/end debug lines.

## Solution Assessment

Yes. The narrow maintainable fix is a typed channel-origin or logger seam at the runner-to-subscriber boundary, not per-channel branches inside the tool handlers.
