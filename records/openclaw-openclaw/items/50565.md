## Summary

Keep open. The original Feishu-specific log text is gone, but current main still routes embedded tool lifecycle debug logs through a static `agent/embedded` logger instead of the originating channel subsystem.

## What This Changes

The issue requests correct channel attribution for embedded tool lifecycle logs such as tool start/end/failure entries.

## Best Possible Solution

Thread a typed normalized channel origin or logger seam from `runEmbeddedAttempt` into `subscribeEmbeddedPiSession`, use `gateway/channels/<channel>` for valid channel-originated runs, and keep `agent/embedded` for channel-less runs.

## Reproduction Assessment

Yes, at source level. Current main carries channel identity in the runner, omits it from subscription params, installs a static `agent/embedded` logger, and uses that logger for tool lifecycle debug lines.

## Solution Assessment

Yes. The narrow maintainable fix is to pass a typed channel-origin or logger seam into the embedded subscriber; per-channel branches inside tool handlers would be the wrong ownership boundary.
