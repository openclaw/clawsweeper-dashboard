## Label Justifications

- `P3`: The issue is a low-risk logging attribution bug with no reported functional impact, but it harms multi-channel debugging.

## Summary

Keep open. Current main has removed the old Feishu-specific tool-call wording, but the embedded tool lifecycle logs still inherit a static `agent/embedded` logger instead of the originating channel logger.

## What This Changes

Tool lifecycle logs should be attributed to the channel that originated the run, such as Telegram or Discord, instead of a Feishu or static embedded subsystem.

## Best Possible Solution

Thread a typed, normalized channel-origin or logger from `runEmbeddedAttempt` into `subscribeEmbeddedPiSession`, use `gateway/channels/<channel>` for message-channel runs, and keep `agent/embedded` for channel-less runs.

## Reproduction Assessment

Yes, at source level. Current main carries channel identity on the runner params, omits it from subscription params, installs a static `agent/embedded` logger, and uses that logger for tool start/end debug lines.

## Solution Assessment

Yes. The maintainable fix is a narrow typed channel-origin or logger seam between the runner and subscriber, not per-channel branches inside tool handlers.
