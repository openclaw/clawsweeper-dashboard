## What This Changes

The issue requests a per-provider config option that forces OpenAI-compatible upstream requests to use non-streaming JSON responses instead of SSE.

## Best Possible Solution

Maintainers should decide whether to add a narrow provider transport opt-out while keeping the Feishu/SSE duplication repair tracked on the canonical symptom issue.

## Reproduction Assessment

No, not as a strict bug repro in this review. Source inspection does confirm the missing capability: current config rejects unknown provider fields and `openai-completions` always builds requests with `stream: true`.

## Solution Assessment

Unclear. A provider-level `stream: false` opt-out is plausible, but the best fix depends on whether maintainers want a core config escape hatch or prefer repairing the malformed-SSE/channel duplication path directly.
