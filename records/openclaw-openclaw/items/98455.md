## Summary

Keep open: current main still buffers non-429 browser-control HTTP error bodies with `res.text()`, and this PR is a focused fix with sufficient proof and no blocking findings.

## What This Changes

The PR caps browser-control HTTP error body reads at 16 KiB through the shared bounded response reader and adds a loopback streaming error-body boundary test.

## Best Possible Solution

Land the bounded read plus regression test after ordinary maintainer and CI review, keeping the dispatcher path unchanged.

## Reproduction Assessment

Yes. Current main has a clear source-level reproduction path: return a non-429 HTTP error with a large streaming body to `fetchBrowserJson`, and `fetchHttpJson` reaches an unbounded `res.text()` call.

## Solution Assessment

Yes. Reusing the existing SDK `readResponseTextLimited` helper is the narrowest maintainable fix; a browser-local reader would duplicate shared behavior, and dropping all error text would lose useful diagnostics.
