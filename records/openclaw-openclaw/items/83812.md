## Summary

Keep open: current main has partial OpenCode Go/Kimi reasoning cleanup, but the no-reasoning replay contract requested here is not fully enforced for replayed assistant payloads. The exact `reasoning_details` field is now sanitized in the OpenAI-completions builder, but `reasoning_content` can still survive for `kimi-k2.6`, and the existing OpenCode Go wrapper only strips root reasoning parameters.

## What This Changes

The issue requests OpenCode Go Kimi K2.5/K2.6 request sanitization so unsupported reasoning fields are stripped from root payloads and replayed messages before provider calls.

## Best Possible Solution

Add a provider-owned OpenCode Go Kimi no-reasoning sanitizer that strips all unsupported reasoning fields from root payloads and replayed `messages[]`/`input[]` for `kimi-k2.5` and `kimi-k2.6`, while preserving existing DeepSeek V4 and other provider replay contracts.

## Reproduction Assessment

Yes, at source level: current main can still preserve replayed string `reasoning_content` for Kimi K2.6 while the OpenCode Go wrapper only strips root reasoning params. I did not run a live OpenCode Go provider call in this read-only review.

## Solution Assessment

Yes: a narrow provider-owned sanitizer is the best fit because the incompatibility is specific to OpenCode Go Kimi, while other Kimi-style routes still need preserved reasoning replay.
