## Summary

Keep open: the narrowed Discord timing bug is still present on current `main`; Discord status reactions are wired, but final cleanup still ignores configured `doneHoldMs` and `errorHoldMs`.

## What This Changes

The issue requests that Discord honor configured `messages.statusReactions` final done/error hold timing before reaction cleanup.

## Best Possible Solution

Use a merged Discord status-reaction timing object in the finalizer, delay cleanup by the configured done/error hold values, and add focused Discord regression tests for both success and failure cleanup timing.

## Reproduction Assessment

Yes from source inspection and the reporter’s live follow-up, but not from a fresh live Discord run in this review. The current Discord finalizer deterministically uses `DEFAULT_TIMING` for final cleanup, so configured `doneHoldMs`/`errorHoldMs` cannot affect that path.

## Solution Assessment

Yes. The narrow maintainable fix is to mirror Telegram’s merged timing pattern inside the existing Discord finalization path, without changing `removeAckAfterReply: false` semantics or broadening status reaction behavior.
