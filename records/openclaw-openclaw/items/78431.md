## Summary

Keep open: current main wires Discord lifecycle reactions, but Discord final cleanup still uses default hold values, so configured `doneHoldMs` and `errorHoldMs` are ignored despite the documented channel support and the reporter’s narrowed live repro.

## What This Changes

Honor configured Discord `messages.statusReactions.timing` final hold values before clearing done/error status reactions.

## Best Possible Solution

Use a merged Discord `statusReactionTiming` object for done/error final cleanup, add regression tests for configured hold timing, and leave `removeAckAfterReply: false` restore behavior unchanged.

## Reproduction Assessment

Yes, at source level and with reporter live data. Current main still reaches `setDone()` and then clears after `DEFAULT_TIMING.doneHoldMs`, so any configured Discord `doneHoldMs` override is ignored.

## Solution Assessment

Yes. The narrow maintainable fix is to mirror Telegram’s merged timing usage in Discord finalization and cover configured done/error hold timing in the Discord process tests.
