## What This Changes

The issue asks Discord status reactions to honor configured `messages.statusReactions.timing.doneHoldMs` and `errorHoldMs` during terminal cleanup.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/94736 or a narrow replacement that uses configured Discord terminal hold timing, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes. Current main passes configured Discord status-reaction timing into the controller, then ignores that config for terminal cleanup by sleeping hardcoded default done/error hold values.

## Solution Assessment

Yes. The best fix is narrow: resolve configured status-reaction timing in the Discord finalizer and use it for terminal cleanup, matching Slack and Telegram behavior.
