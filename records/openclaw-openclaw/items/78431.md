## Summary

Keep open: current main wires Discord lifecycle status reactions, but the narrowed timing bug remains because the Discord final cleanup still ignores configured `doneHoldMs` and `errorHoldMs`.

## What This Changes

Request to complete Discord `messages.statusReactions` parity by honoring configured terminal hold timing before cleaning up done/error reactions.

## Best Possible Solution

Apply the Telegram-style merged timing object in the Discord finalizer, use it for done/error cleanup delays, add focused Discord regression tests, and include a changelog fix entry.

## Reproduction Assessment

Yes. The high-confidence source path is `removeAckAfterReply: true` with a custom `messages.statusReactions.timing.doneHoldMs`: current Discord code reaches `setDone()` and then sleeps `DEFAULT_TIMING.doneHoldMs`, while the reporter also observed the done emoji flash briefly in a real Discord setup.

## Solution Assessment

Yes. The narrow maintainable fix is to mirror Telegram's merged timing handling in Discord final cleanup and cover configured `doneHoldMs`/`errorHoldMs`; changing persistence semantics for `removeAckAfterReply: false` should stay out of this bug fix.
