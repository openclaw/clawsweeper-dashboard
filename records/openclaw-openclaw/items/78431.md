## Summary

Keep open: current main still wires Discord lifecycle reactions but final cleanup ignores configured status-reaction hold timing, so the narrowed `doneHoldMs` bug remains source-reproducible.

## What This Changes

The issue asks Discord status reactions to honor the documented lifecycle timing, especially preserving the done emoji for configured `doneHoldMs`.

## Best Possible Solution

Apply the Telegram-style merged timing object in the Discord finalizer, use it for done/error cleanup delays, and add focused Discord regression coverage for configured hold timing.

## Reproduction Assessment

Yes at source level: with `removeAckAfterReply: true` and custom `messages.statusReactions.timing.doneHoldMs`, current main reaches `setDone()` and then sleeps with `DEFAULT_TIMING.doneHoldMs` before clearing. I did not run a live Discord gateway in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is to mirror Telegram's merged timing handling in Discord final cleanup and cover configured `doneHoldMs`/`errorHoldMs`; changing `removeAckAfterReply: false` persistence semantics should stay out of this bug fix.
