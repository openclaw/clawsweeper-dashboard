## Summary

Keep open: current main no longer looks Telegram-only for Discord status reactions, but the reported timing/persistence part is still source-reproducible because Discord final cleanup ignores configured `doneHoldMs` and `errorHoldMs`.

## What This Changes

Request Discord parity for `messages.statusReactions`, including lifecycle emoji transitions and configured done/error hold timing.

## Best Possible Solution

Complete Discord parity by applying configured status reaction hold timing to Discord final done/error cleanup and covering it with focused Discord tests.

## Reproduction Assessment

Yes from source inspection, but not from a live Discord run. Discord final cleanup still uses `DEFAULT_TIMING` directly, so `messages.statusReactions.timing.doneHoldMs` and `errorHoldMs` are ignored for the final clear path.

## Solution Assessment

Yes for the narrow timing gap: mirror Telegram’s merged timing object in the Discord finalizer and add regression coverage. Changing `removeAckAfterReply: false` to persist the done emoji would be a separate product decision rather than a bug fix.
