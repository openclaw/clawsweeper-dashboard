## Summary

Keep open: the bug is valid and current main still lets the legacy global block-streaming default suppress explicit Telegram partial streaming, but this PR is not merge-ready because its helper also disables that global default for unconfigured Telegram accounts and it lacks real Telegram proof.

## What This Changes

The branch adds a Telegram-local block-streaming resolver and updates Telegram dispatch, doctor warnings, and tests so explicit Telegram partial/progress streaming can override the legacy global block-streaming default.

## Best Possible Solution

Keep the Telegram-local resolver, but only let explicitly configured Telegram partial/progress mode override the global block default, preserve implicit global-default behavior, and require redacted real Telegram proof before merge.

## Reproduction Assessment

Yes, source-reproducible: current main applies the global block default before Telegram draft eligibility, so an explicit partial Telegram config can still be blocked. The PR-head regression is also source-clear because an unset Telegram config resolves to the default `partial` mode and disables the global default.

## Solution Assessment

No; the Telegram-owned helper is the right layer, but the fallback is too broad because it treats Telegram's implicit default partial mode as explicit user intent. The safer fix is to override the global default only when partial/progress was explicitly configured for Telegram.
