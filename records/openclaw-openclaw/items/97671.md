## Summary

This PR should stay open: it targets a real Telegram streaming bug, but the current helper also changes existing Telegram accounts that rely on the global block-streaming default with no explicit Telegram streaming mode.

## What This Changes

The branch adds a Telegram-owned block-streaming resolver and uses it from Telegram runtime and doctor checks so explicit partial/progress preview modes can override the legacy global block-streaming default.

## Best Possible Solution

Keep the Telegram-local helper, but make the global default yield only when Telegram partial/progress mode is explicitly configured and add a regression test for the implicit-default case.

## Reproduction Assessment

Yes, source-reproducible: current main lets the global block default make `accountBlockStreamingEnabled` true, which disables Telegram draft streaming even when the account is configured for partial mode.

## Solution Assessment

No; the PR fixes the explicit partial/progress case but is too broad because it also treats Telegram's implicit default partial mode as a reason to disable the legacy global block default.
