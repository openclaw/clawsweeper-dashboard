Full review comments:

- **[P2] Wire stripping into the reported send path:** `src/shared/text/strip-markdown.ts:10`
  - body: The PR says it fixes iMessage approval messages, but current `sendMessageIMessage` does not call `stripMarkdown`, and the iMessage adapter sanitizer remains HTML-focused. Changing only this helper leaves the reported delivery path unchanged, so either wire the actual plain-text/iMessage path or narrow the PR claim to helper consumers that use it.
  - confidence: 0.9
- **[P2] Require fenced blocks to start on their own line:** `src/shared/text/strip-markdown.ts:10`
  - body: The regex also matches same-line text such as `Run ```npm i``` now`, treats `npm i` as the info string, and replaces the whole span with an empty capture. Require a newline-delimited block fence so plain-text output does not drop user-visible inline content.
  - confidence: 0.93
- **[P2] Match the closing fence delimiter:** `src/shared/text/strip-markdown.ts:10`
  - body: `buildFence` lengthens the delimiter when command text already contains triple backticks, but this regex always closes on three backticks. Four-backtick approval blocks can leave stray markers or lose content, so capture the opening delimiter of three or more backticks and close on the same delimiter.
  - confidence: 0.9
