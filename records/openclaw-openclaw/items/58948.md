## Best Possible Solution

Update the PR for current main by using newline-delimited 3+ backtick fence handling that preserves inline spans, adding regression coverage and a changelog entry, and either wiring the current affected plain-text delivery path or explicitly leaving that broader pipeline change to https://github.com/openclaw/openclaw/pull/56283.

## Solution Assessment

No. `stripMarkdown` is a reasonable narrow helper boundary for TTS/LINE/speech consumers, but the proposed regex is too broad and the current iMessage path no longer uses this helper for the reported surface.
