## What This Changes

The PR adds fenced-code-block stripping to `stripMarkdown` and adds Vitest coverage for fenced approval-message output.

## Best Possible Solution

Use delimiter-aware, newline-delimited fenced-code stripping and connect the actual plain-text/iMessage delivery path, or fold the helper fix into the broader capability-aware outbound Markdown policy.

## Reproduction Assessment

Yes for source-level helper behavior: current main leaves fenced markers, and a PR-regex probe reproduces content loss for same-line triple backticks and longer approval fences. I did not establish a live iMessage send reproduction.

## Solution Assessment

  - confidence: 0.9
