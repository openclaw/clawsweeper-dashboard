## What This Changes

This PR adds an `adaptive` markdown table mode for Microsoft Teams, builds Adaptive Card table attachments, changes the Teams default table mode, and adds focused tests.

## Best Possible Solution

Keep Teams table rendering on the existing default until the adaptive path preserves card messages end-to-end, shared markdown table types are aligned, and maintainers have enough desktop/mobile proof to decide whether schema 1.5 cards should be default or opt-in.

## Reproduction Assessment

## Solution Assessment

No. Native Adaptive Card tables are a plausible Teams-specific solution, but this implementation needs end-to-end send support, aligned shared types, and an explicit default/upgrade decision before it is the best fix.

Full review comments:

- **[P1] Keep adaptive card messages in the send batch:** `extensions/msteams/src/messenger.ts:241`
  - body: `renderAdaptiveTablePayload` and the proactive path create `{ adaptiveCard }`-only messages, but `sendMSTeamsMessages` still filters the batch to messages with non-empty `text` or `mediaUrl`. The table card is removed before `buildActivity` runs, so mixed replies send only the surrounding text and table-only replies can produce no sent message. Include `adaptiveCard` in the send filter and add coverage through the actual send batching path.
  - confidence: 0.94
- **[P1] Update the markdown-core table mode type:** `packages/markdown-core/src/tables.ts:15`
  - body: `convertMarkdownTables` now checks for `mode === "adaptive"`, but its `MarkdownTableMode` import comes from `packages/markdown-core/src/types.ts`, which still only allows `"off" | "bullets" | "code" | "block"`. That leaves the new branch outside the helper's own type contract and should fail type checking; add `adaptive` to the markdown-core type contract or keep the adaptive value out of this shared helper.
  - confidence: 0.9
