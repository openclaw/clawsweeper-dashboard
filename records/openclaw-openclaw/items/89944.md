## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep adaptive cards in the send batch:** `extensions/msteams/src/messenger.ts:239`
  - body: `renderAdaptiveTablePayload` now creates table-only messages shaped as `{ adaptiveCard: ... }`, but `sendMSTeamsMessages` filters its batch to messages with non-empty `text` or `mediaUrl`. Those card-only messages are dropped before `buildActivity` can attach them, so table-only replies and the table portion of mixed replies can silently disappear. Include `adaptiveCard` in the predicate and add a send-path regression test.
  - confidence: 0.95
- **[P1] Keep adaptive tables opt-in until fallback is proven:** `extensions/msteams/src/channel.ts:510`
  - body: Setting the msteams default to `adaptive` changes every existing install that has not configured `channels.msteams.markdown.tables`. The generated cards use Adaptive Card schema 1.5 `Table`, while Microsoft documents Teams mobile support as safest up to Adaptive Card 1.2. Keep the current default and make `adaptive` explicit opt-in unless the PR includes desktop/web/mobile fallback proof and upgrade docs.
  - confidence: 0.9
- **[P2] Preserve all adaptive split-send receipt IDs:** `extensions/msteams/src/send.ts:426-430`
  - body: `sendAdaptiveTableSegments` gets every platform ID back from `sendMSTeamsMessages`, but this return path only records the first ID in the result receipt. Once adaptive card messages are actually delivered, mixed text/table/text sends will lose receipt tracking for the card and trailing text parts. Pass `platformMessageIds` into `createMSTeamsSendResult` or build the receipt the same way text sends do.
  - confidence: 0.84
- **[P2] Document the new table mode where the config is exposed:** `src/config/types.base.ts:121`
  - body: This adds `adaptive` to the shared markdown table mode type/schema, but the inline config comment and public markdown formatting docs still list only the older modes and do not explain the Teams opt-in/opt-out behavior. Align the config docs and schema coverage when exposing the new value.
  - confidence: 0.8
