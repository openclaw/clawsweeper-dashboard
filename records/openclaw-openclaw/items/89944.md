  - body: `renderAdaptiveTablePayload` and `sendAdaptiveTableSegments` now create table-only messages shaped as `{ adaptiveCard: ... }`, but this filter keeps only messages with `text` or `mediaUrl`. Please keep the current default and make `adaptive` an explicit opt-in unless the PR includes desktop/web/mobile fallback proof and upgrade docs.
  - confidence: 0.9
