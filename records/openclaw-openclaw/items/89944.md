## Summary

Keep open: this is a useful Microsoft Teams rendering feature with sufficient screenshot proof, but it is not merge-ready because the shared adaptive table mode can drop table content outside Teams, the Teams split path loses markdown/link metadata around tables, and the branch currently fails test typechecking.

## What This Changes

The PR adds an opt-in Microsoft Teams markdown table mode that sends final and proactive table replies as Adaptive Card Table attachments, updates shared markdown config/types/docs, and adds focused tests.

## Best Possible Solution

Land a Teams-owned adaptive table renderer that preserves markdown/link rendering for non-table slices, keeps non-Teams table rendering safe, fixes the test type error, and leaves the Teams schema 1.5 compatibility boundary explicit and opt-in.

## Reproduction Assessment

Yes, from source inspection: an adaptive-mode Teams reply with a markdown link or emphasis around a table will split and send plain `ir.text`, losing URL/style metadata. A non-Teams channel configured with `markdown.tables: adaptive` can also route that value into IR table parsing without a rendering branch.

## Solution Assessment

No, not yet. The Teams renderer is a reasonable location, but the implementation needs to render non-table IR slices through existing marker renderers, make the shared `adaptive` mode safe or Teams-scoped, and clear the typecheck failure before merge.
