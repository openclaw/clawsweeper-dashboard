## Summary

Keep open: current main and the latest release still allow the shared automatic progress-label pool to select `Lobstering`, and three open linked PRs are candidate fixes rather than merged provenance. The remaining maintainer decision is the desired shared default tone.

## What This Changes

The issue asks OpenClaw to stop using mascot-themed wording such as `Lobstering` in the default automatic progress-draft label pool unless a user opts in with progress-label configuration.

## Best Possible Solution

Choose the shared neutral default wording, land one linked fix with docs and regression coverage, then close this issue after the merge.

## Reproduction Assessment

Yes at source level: leaving `streaming.progress.label` and `streaming.progress.labels` unset or `auto` uses the built-in pool that still contains `Lobstering`, and the formatter renders that chosen label as the draft header. I did not run a live Telegram session in this read-only review.

## Solution Assessment

Unclear until maintainers choose the desired default tone. If accepted, the best implementation is a shared default-pool update with docs and regression tests, not a Telegram-only override.
