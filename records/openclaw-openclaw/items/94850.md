## What This Changes

The issue requests neutral, technical default progress-draft labels so Telegram/shared progress mode cannot show `Lobstering` unless the user configures it explicitly.

## Best Possible Solution

Choose one canonical neutral default-label policy for the shared progress-draft pool, update docs and focused tests with that policy, and close this issue after the selected PR lands.

## Reproduction Assessment

Yes, at source level: leaving `streaming.progress.label` and `streaming.progress.labels` unset or `auto` uses the built-in pool that still contains `Lobstering`, and the formatter renders that chosen label as the draft header. I did not run a live Telegram session in this read-only review.

## Solution Assessment

Unclear until maintainers choose the desired default tone. If accepted, the maintainable layer is the shared default-pool/docs/test path, not a Telegram-only override.
