## Summary

Keep open: current main and v2026.6.8 still allow the shared automatic progress-label pool to select `Lobstering`, and multiple open closing PRs already target the fix. The remaining decision is which shared neutral default wording maintainers want to land.

## What This Changes

The issue asks to make built-in progress draft labels neutral by default and keep mascot-themed labels available only through explicit `streaming.progress.label` or `streaming.progress.labels` configuration.

## Best Possible Solution

Land one maintainer-approved shared default-label patch with matching docs and regression coverage, while preserving explicit custom labels as the opt-in path for themed wording.

## Reproduction Assessment

Yes, at source level: leave `streaming.progress.label` and `streaming.progress.labels` unset or `auto`, enable progress streaming, and the shared resolver can select `Lobstering` before the formatter makes it the draft header. I did not run a live Telegram session in this read-only review.

## Solution Assessment

Unclear until maintainers choose the default tone. If accepted, the maintainable fix is changing the shared default label pool with docs and regression coverage, not adding a Telegram-only override.
