## Summary

Keep open: current main and the latest release still allow the automatic progress-label pool to select `Lobstering`, and three linked closing PRs remain open candidate fixes rather than merged provenance.

## What This Changes

The issue asks OpenClaw to stop using mascot-themed progress draft labels such as `Lobstering` as default Telegram-visible progress headings without explicit user opt-in.

## Best Possible Solution

Choose one canonical neutral default-label policy, land one linked PR with runtime, docs, tests, and visible Telegram or equivalent runtime proof, then close this issue after merge.

## Reproduction Assessment

Yes. Source inspection shows unset or `auto` progress labels fall back to the default pool containing `Lobstering`, and the shared formatter renders that selected label as the draft heading.

## Solution Assessment

Unclear until maintainers choose the desired default wording. If accepted, the maintainable fix belongs in the shared default-pool/docs/test path rather than a Telegram-only override.
