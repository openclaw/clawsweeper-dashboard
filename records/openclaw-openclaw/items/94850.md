## Summary

Keep open: current main and the latest release still allow the built-in automatic progress-label pool to select `Lobstering`, and Telegram progress mode uses that shared formatter for visible status drafts. The behavior was originally documented and described as intentional, so the remaining decision is a maintainer UX/defaults call rather than a cleanup close.

## What This Changes

Request neutral default progress-draft labels so Telegram does not show themed headers like `Lobstering` unless the operator configures them.

## Best Possible Solution

If maintainers accept the UX direction, make the shared automatic progress-label defaults neutral, update docs/config hints/tests, and keep custom label or label-pool configuration as the opt-in path for themed wording.

## Reproduction Assessment

Yes, at source level: leave `streaming.progress.label` and `streaming.progress.labels` unset or `auto`, enable Telegram `streaming.mode: "progress"`, and the shared resolver can select `Lobstering` from the built-in pool before formatting it as the draft header. I did not run a live Telegram session in this read-only review.

## Solution Assessment

Unclear until maintainers confirm the desired default tone. If accepted, the narrowest maintainable fix is a shared default-label-pool change with aligned docs and tests, not a Telegram-only override.
