## Summary

Keep open: current main and the latest release still allow the shared automatic progress-label pool to select `Lobstering`, and the open linked PRs are candidate fixes rather than merged provenance.

## What This Changes

The issue requests neutral default progress-draft labels so Telegram/channel users do not see mascot-themed headings such as `Lobstering` unless they configure them explicitly.

## Best Possible Solution

Choose the shared neutral default-label policy, land one linked PR with docs, regression coverage, and visible runtime proof, then close this issue and supersede the remaining candidate PRs.

## Reproduction Assessment

Yes, at source level: leaving `streaming.progress.label` and `streaming.progress.labels` unset or `auto` uses the built-in pool that still contains `Lobstering`, and the formatter renders that label as the draft header. I did not run a live Telegram session in this read-only review.

## Solution Assessment

Unclear until maintainers choose the default tone. If accepted, the right layer is the shared progress-draft default pool plus matching docs/tests, not a Telegram-only override.
