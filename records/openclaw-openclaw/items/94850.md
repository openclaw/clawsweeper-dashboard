## Summary

Keep open: current main and `v2026.6.9` still allow the shared automatic progress-label pool to select `Lobstering`, and the open linked PRs are candidate fixes rather than merged provenance. The remaining maintainer decision is the shared default tone and which linked PR should become canonical.

## What This Changes

The issue requests neutral default channel progress-draft labels so Telegram progress drafts cannot show `Lobstering` unless a user explicitly configures themed labels.

## Best Possible Solution

Choose one canonical neutral default-label policy for the shared channel helper, land a single linked PR with aligned docs, focused coverage, and real Telegram or equivalent channel proof, then close this issue and supersede the remaining candidate PRs.

## Reproduction Assessment

Yes. Source inspection shows that unset or `auto` progress labels use the built-in pool containing `Lobstering`, and the formatter renders the selected label as the draft header; I did not run a live Telegram session in this read-only review.

## Solution Assessment

Yes on owner boundary, but not yet on product wording. If maintainers approve neutral defaults, changing the shared default pool plus docs/tests is the right layer; a Telegram-only override would leave sibling channels inconsistent.
