## Summary

Keep open. Current main has partial source-level handling for length-limited terminal turns, but the requested `mode:"run"` fail/continue policy and config surface are still absent, and the one-shot subagent lifecycle still only converts `blocked` liveness into an error completion.

## What This Changes

Request configurable hard-fail and optional continuation behavior for provider `stopReason:"length"` in one-shot `mode:"run"` agent sessions.

## Best Possible Solution

Keep the issue open for a maintainer-approved runtime policy that first guarantees caller-visible failure for length truncation, then considers opt-in continuation with explicit replay and side-effect safety rules.

## Reproduction Assessment

Yes, at source level: current main recognizes `length` as a provider stop reason and tests length-limited turns as non-deliverable/abandoned, but it does not provide the requested configured continuation or fail policy. I did not run a live provider cutoff replay in this read-only review.

## Solution Assessment

Unclear for the full catch-and-continue proposal; a hard-fail signal is the safest first increment, while continuation needs explicit replay and side-effect policy before implementation.
