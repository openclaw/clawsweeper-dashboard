## Summary

Keep open. Current main preserves provider `stopReason: "length"` and propagates it in metadata, but embedded `mode:"run"` sessions still have no configurable fail/continue policy and can reach the normal terminal path with partial visible output.

## What This Changes

Request a configurable embedded-run policy for provider `stopReason:"length"`, including explicit fail or bounded continuation behavior.

## Best Possible Solution

Add a documented fail-mode first for embedded-run length stops, then evaluate opt-in continuation separately once replay and side-effect rules are explicit.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: mock an embedded attempt whose last assistant has `stopReason:"length"` and partial visible text, then the current guard returns null and `run.ts` reaches the normal terminal path. I did not run tests because this review is read-only.

## Solution Assessment

Unclear for full auto-continue, because replay and side-effect policy need maintainer judgment. The narrowest maintainable first step is the requested fail-mode subset: surface length-truncated terminal runs as explicit errors without replaying work.
