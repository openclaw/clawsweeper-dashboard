## Summary

Keep open. Current main preserves provider `stopReason: "length"` metadata, but it still has no configurable fail/continue policy for embedded `mode:"run"` subagent sessions, and the normal terminal path can complete with partial visible output instead of surfacing a hard failure.

## What This Changes

Request a configurable embedded agent runtime policy for provider output-limit truncation in `mode:"run"` sessions, including fail/continue behavior and a continuation cap.

## Best Possible Solution

Add a conservative fail-mode first for `stopReason:"length"` in embedded run/subagent sessions, then consider opt-in continuation after the replay-safety and side-effect rules are explicit.

## Reproduction Assessment

Yes. A focused source-level reproduction can mock an embedded attempt whose last assistant has `stopReason:"length"` and partial visible text; current main will propagate the stop reason as metadata but follow the normal terminal success path.

## Solution Assessment

Unclear for the full requested continue behavior, because continuation needs replay and side-effect safeguards. The safest first implementation is the requested `fail` subset: turn length-truncated terminal runs into an explicit error signal without replaying work.
