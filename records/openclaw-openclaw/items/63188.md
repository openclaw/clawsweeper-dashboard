## Summary

Keep open: current main now preserves and surfaces some `stopReason:"length"` incomplete-turn failures, but the requested `mode:"run"` configurable fail/continue runtime policy and config surface are still absent and need maintainer design approval.

## What This Changes

Request to add configurable `mode:"run"` handling for provider output-limit `stopReason:"length"`, including hard-fail and catch-and-continue policies.

## Best Possible Solution

Adopt a maintainer-approved fail-first runtime policy for length truncation, then consider opt-in continuation with explicit replay-safety, side-effect, retry-budget, schema, docs, and tests.

## Reproduction Assessment

Yes, at source level: current main explicitly treats `stopReason:"length"` as incomplete in some terminal paths, while the requested config and continuation behavior are absent. I did not run a live provider cutoff replay in this read-only review.

## Solution Assessment

Unclear for the full catch-and-continue proposal; a hard-fail signal is the safer first increment, while automatic continuation needs explicit replay and side-effect policy before implementation.
