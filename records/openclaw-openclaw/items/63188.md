## Summary

Keep open. Current main now prevents some length-truncated turns from silently succeeding and has a fresh fix for truncated tool-call execution, but the requested mode:"run" configurable hard-fail/continue policy and config surface are still absent and need maintainer runtime-policy design.

## What This Changes

Request configurable mode:"run" handling for provider stopReason:"length", including hard-fail and optional continuation policies with retry caps.

## Best Possible Solution

Adopt a maintainer-approved fail-first policy for length truncation in mode:"run", then add opt-in continuation only with explicit replay-safety, side-effect, retry-budget, schema, docs, and tests.

## Reproduction Assessment

Yes, at source level: current main has explicit code and tests for stopReason:"length" as incomplete/non-deliverable, and the requested config/continuation behavior is absent. I did not run a live provider cutoff replay in this read-only review.

## Solution Assessment

Unclear for the full catch-and-continue proposal; hard-fail signaling is the safer first increment, while automatic continuation needs maintainer policy for replay safety and side effects.
