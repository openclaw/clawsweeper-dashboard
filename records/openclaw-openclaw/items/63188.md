## Summary

Keep open. Current main partially detects `stopReason:"length"` terminal text as incomplete/abandoned, but it still lacks the requested configurable fail/continue policy, and the one-shot subagent completion path does not clearly turn abandoned length termination into a hard caller-visible failure; the latest release lacks the partial length-specific guard.

## What This Changes

Request to add configurable handling for provider `stopReason:"length"` in one-shot subagent run sessions, including hard-fail signaling and optional continuation behavior.

## Best Possible Solution

Keep the current fail-safe length detection, then define an explicit subagent run truncation policy that first guarantees caller-visible hard failure and only adds opt-in continuation after replay and side-effect rules are settled.

## Reproduction Assessment

Yes, at source level: current main receives `length`, classifies some length-limited turns as incomplete, but still has no configured continuation policy and no explicit abandoned-to-error subagent completion rule. I did not run a live provider cutoff replay because this review is read-only.

## Solution Assessment

Unclear for the full catch-and-continue proposal; a hard-fail signal is the safest first step, while continuation needs explicit replay and side-effect policy before implementation.
