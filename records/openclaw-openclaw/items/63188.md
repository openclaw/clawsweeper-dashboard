## Summary

Keep open. Current main still lacks the requested configurable embedded `mode:"run"` handling for provider `stopReason:"length"`; reporter evidence plus a later local-model comment show the same truncation family across providers, and source inspection shows visible partial length-terminated runs still reach the normal terminal path.

## What This Changes

The issue requests per-agent configurable handling for `mode:"run"` sessions when a provider returns `stopReason:"length"`, including fail-fast or bounded continuation behavior.

## Best Possible Solution

Define a documented embedded-run length-truncation policy that first supports explicit failure signaling, then only adds opt-in continuation once replay and side-effect rules are settled.

## Reproduction Assessment

Yes, at source level: a visible partial assistant turn with `stopReason:"length"` bypasses the incomplete-turn error path and reaches the normal terminal path. I did not run a live provider truncation replay because this review is read-only.

## Solution Assessment

Unclear for the full catch-and-continue proposal; the narrow fail-mode subset is the safest first step, while continuation needs explicit replay and side-effect policy before implementation.
