## Summary

Keep open. The checked-out main, the local `origin/main` ref, and the latest release tag still lack the requested run-mode `stopReason:"length"` policy: visible partial length-terminated output can still pass through the normal terminal path, and the proposed config surface does not exist. The report remains actionable but needs maintainer runtime-policy judgment before implementation.

## What This Changes

The issue requests a per-agent runtime policy for embedded run sessions that fail or continue when a provider returns `stopReason:"length"`, with an optional continuation cap.

## Best Possible Solution

Define a documented embedded-run length-truncation policy that first supports explicit failure signaling for unsafe partial `length` stops, then adds opt-in continuation only after replay and side-effect rules are settled.

## Reproduction Assessment

Yes, at source level: a visible partial assistant turn with `stopReason:"length"` and payloads returns null from the incomplete-turn detector and reaches the normal terminal path. I did not run a live provider cutoff replay because this review is read-only.

## Solution Assessment

Unclear for the full catch-and-continue proposal; the narrow fail-mode subset is the safest first step, while continuation needs explicit replay and side-effect policy before implementation.
