## Summary

Keep open. Current main now partially handles length-limited assistant turns by preserving `stopReason:"length"` and surfacing incomplete-turn errors, but the requested `mode:"run"` configurable fail/continue policy and config surface are still absent and need maintainer runtime-policy judgment.

## What This Changes

Request for configurable `stopReason:"length"` handling in `mode:"run"` sessions, including hard-fail signaling and optional bounded continuation.

## Best Possible Solution

Adopt a maintainer-approved fail-first runtime policy for length truncation, then consider opt-in continuation with explicit replay-safety, side-effect, retry-budget, schema, and test coverage.

## Reproduction Assessment

Yes, at source level: current main has explicit code and tests for `stopReason:"length"` as an incomplete/non-deliverable terminal turn, while the requested config and continuation behavior are absent. I did not run a live provider cutoff replay in this read-only review.

## Solution Assessment

Unclear for the full catch-and-continue proposal; a hard-fail signal is the safer first increment, while automatic continuation needs explicit replay and side-effect policy before implementation.
