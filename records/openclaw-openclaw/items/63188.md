## Summary

Keep open. Current main has source-level handling for some length-limited terminal turns, but it still lacks the requested `mode:"run"` fail/continue policy and configurable continuation limits, so the central subagent runtime behavior remains unresolved.

## What This Changes

The issue requests configurable `mode:"run"` handling for provider `stopReason:"length"`, including fail signaling, optional continuation, and retry limits.

## Best Possible Solution

Define a maintainer-approved policy that guarantees caller-visible failure for `mode:"run"` length truncation first, then add continuation only with explicit replay safety and bounded retry semantics.

## Reproduction Assessment

Yes at source level, but not live-reproduced here. Current source shows length-limited turns can be detected as incomplete, while the requested configurable fail/continue policy is absent and the subagent completion path still only hard-fails blocked liveness.

## Solution Assessment

The safest first increment is hard-fail signaling to the orchestrator; catch-and-continue is not clearly the best fix until maintainers decide replay safety, side-effect handling, and config shape.
