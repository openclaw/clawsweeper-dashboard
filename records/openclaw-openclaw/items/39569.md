## Summary

Keep open. Current main still has the Feishu reply-mention drop path, but this PR is not safe to merge as written because it can treat non-bot orphan placeholders as bot mentions, clears topic roots before scope is known, and lacks inspectable real behavior proof.

## What This Changes

The PR changes Feishu mention parsing to treat orphaned reply placeholders as bot mentions, clears `rootId` on mentioned replies, and adds parser tests for those cases.

## Best Possible Solution

Rework or replace this with a Feishu plugin patch that uses the strongest bot-specific dropped-mention signal available, preserves topic and explicit thread scope, and adds focused regression tests.

## Reproduction Assessment

Yes, by source inspection rather than a live run: a Feishu group reply with a non-empty `mentions` array that omits the bot leaves `mentionedBot=false`, then the mention-required group ingress drops it.

## Solution Assessment

No. The fix belongs in the Feishu plugin, but this PR's any-orphan heuristic can bypass mention-required dispatch and its parse-time root clearing can break topic scopes; the safer path is bot-specific detection with scope-aware reply targeting.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve topic roots before clearing reply routing:** `extensions/feishu/src/bot.ts:823`
  - body: Clearing `rootId` during parsing happens before `groupSessionScope` is known. In `group_topic` and `group_topic_sender` modes, `resolveFeishuGroupSession()` uses that root as the topic key, so mentioned replies without `thread_id` can fall back to the chat-level session and merge unrelated topic context.
  - confidence: 0.91
- **[P2] Require bot-specific evidence for orphan placeholders:** `extensions/feishu/src/bot.ts:484`
  - body: This treats any unmatched `@_..._N` token in reply text as a bot mention. A literal placeholder or dropped non-bot mention can flip `mentionedBot` to true, bypass `requireMention`, and trigger the root-clearing path without evidence that the bot was addressed.
  - confidence: 0.88
