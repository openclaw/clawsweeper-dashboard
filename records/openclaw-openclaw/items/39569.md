## Summary

## What This Changes

The PR changes Feishu mention parsing to treat orphan reply placeholders as bot mentions, clears reply roots for mentioned replies, and adds parser tests.

## Best Possible Solution

Rework the Feishu-local fix around current bot-content.ts, use bot-specific dropped-mention evidence, preserve root context until scope-aware reply routing runs, and attach redacted live Feishu proof.

## Reproduction Assessment

Yes, at source level. A Feishu group reply whose non-empty mentions array omits botOpenId leaves mentionedBot=false, then mention-required group ingress can return before dispatch.

## Solution Assessment

No. The fix belongs in the Feishu plugin, but this PR’s any-orphan placeholder heuristic and parse-time rootId clearing are broader than the narrow maintainable repair.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve topic roots before clearing reply routing:** `extensions/feishu/src/bot.ts:808-812`
  - body: Clearing `rootId` in `parseFeishuMessageEvent` runs before `groupSessionScope` is resolved. For `group_topic` and `group_topic_sender`, `rootId` is the topic/session key, so addressed replies with `root_id` can fall back to the chat-level session and merge unrelated topic context.
  - confidence: 0.91
- **[P2] Require bot-specific evidence for orphan placeholders:** `extensions/feishu/src/bot.ts:481`
  - body: This fallback marks any unmatched placeholder token in reply text as a bot mention. A literal token or dropped non-bot mention can bypass `requireMention` and trigger root clearing without evidence that the bot was addressed.
  - confidence: 0.88
