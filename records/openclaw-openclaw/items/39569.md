## What This Changes

The PR updates Feishu mention parsing to treat orphaned reply placeholders as bot mentions, clears `rootId` for mentioned replies, and adds parser tests.

## Best Possible Solution

Rework the Feishu-local fix on current main by adding bot-specific dropped-mention detection in `bot-content.ts`, preserving root/thread context until scope-aware routing, and adding focused tests plus redacted Feishu proof.

## Reproduction Assessment

Yes, at source level. A Feishu group reply whose structured `mentions` array omits the bot open_id leaves `mentionedBot=false`, and mention-required group ingress can return before dispatch.

## Solution Assessment

No. The fix belongs in the Feishu plugin, but this implementation uses non-bot orphan placeholders and parse-time root clearing; the safer solution is bot-specific mention evidence plus scope-aware routing.
