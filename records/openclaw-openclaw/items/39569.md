## Summary

Keep open. The paired Feishu bug is still present on current main, but this PR should not merge as-is because it can bypass Feishu group mention gating and it conflicts with current topic/thread routing semantics.

## What This Changes

This PR changes Feishu mention detection to treat orphaned reply placeholders as bot mentions, clears reply root metadata for mentioned replies, and adds parser regression tests.

## Best Possible Solution

Rework the fix in the Feishu plugin by adding reply-placeholder handling in `bot-content.ts`, preserving root/thread metadata for routing, using the strongest bot-specific signal Feishu exposes, and covering mention detection plus reply-target behavior with focused regression tests and a changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence unit-level path: construct a Feishu group text reply with `parent_id`, text containing an `@_user_N` placeholder, and `mentions` missing the bot; current parsing leaves `mentionedBot=false`, and the group `requireMention` gate drops it.

## Solution Assessment

No. The PR addresses the right user-visible bug, but clearing `rootId` before session scope is known and accepting any orphan placeholder are not the narrowest maintainable or safest fixes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve topic roots for mentioned replies:** `extensions/feishu/src/bot.ts:815`
  - body: Clearing `rootId` when `mentionedBot && parent_id` runs before the group session scope is known. Topic modes use `ctx.rootId` as the topic key and reply target, so addressed replies can be collapsed into the chat-level session or wrong reply target instead of preserving topic context.
  - confidence: 0.88
- **[P2] Require bot-specific evidence for orphan placeholders:** `extensions/feishu/src/bot.ts:481`
  - body: The fallback marks any unmatched `@_...` token in reply text as `mentionedBot=true`. A literal placeholder or dropped non-bot mention can satisfy `requireMention` and trigger processing even though the bot was not identified; narrow this to a stronger bot-specific signal or explicit Feishu payload condition.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: No supply-chain files are changed, but the diff broadens Feishu group activation in a way that can bypass mention gating without confirmed bot mention evidence.

Concerns:

- **[medium] Broad orphan placeholder can bypass mention gating:** `extensions/feishu/src/bot.ts:481`
  - body: The fallback treats any unmatched reply placeholder as `mentionedBot=true`. In groups using `requireMention`, crafted placeholder-looking text or a dropped non-bot mention could trigger OpenClaw processing and responses when the bot was not actually addressed.
