## Summary

Keep this PR open. Current main still lacks the Feishu reply-placeholder mention fix, the PR is paired with open issue #39568, and the proposed patch has blocking mention-gating and topic-routing concerns before it can merge.

## What This Changes

The PR changes Feishu mention parsing to treat orphaned reply placeholders as bot mentions, clears reply root metadata for mentioned replies, and adds parser regression tests.

## Best Possible Solution

Rework the Feishu plugin fix in the current `bot-content.ts`/`bot.ts` split, preserve topic root metadata for routing, use the strongest bot-specific signal Feishu exposes, and add focused tests plus a user-facing changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: parse a Feishu group text reply with `parent_id` or `root_id`, text containing an `@_user_N` placeholder, and `mentions` missing the bot; current main leaves `mentionedBot=false`, and the group `requireMention` gate drops it.

## Solution Assessment

No. The PR addresses the right user-visible bug, but clearing `rootId` before session scope is known and treating any orphan placeholder as a bot mention are not the narrowest maintainable or safest fixes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve topic roots for mentioned replies:** `extensions/feishu/src/bot.ts:815`
  - body: Clearing `rootId` when `mentionedBot && parent_id` runs before the group session scope is known. Topic modes use `ctx.rootId` as the topic key and reply target, so addressed replies can be collapsed into the chat-level session or wrong reply target instead of preserving topic context.
  - confidence: 0.88
- **[P2] Require bot-specific evidence for orphan placeholders:** `extensions/feishu/src/bot.ts:481`
  - body: The fallback marks any unmatched `@_...` token in reply text as `mentionedBot=true`. A literal placeholder or dropped non-bot mention can satisfy `requireMention` and trigger processing even though the bot was not identified; narrow this to a stronger bot-specific signal or explicit Feishu payload condition.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: No supply-chain files are changed, but the diff broadens Feishu group activation in a way that can bypass mention gating without confirmed bot mention evidence.

Concerns:
