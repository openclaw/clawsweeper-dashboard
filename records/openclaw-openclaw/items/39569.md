## Summary

Keep open: current main still appears to miss the reported Feishu reply-mention path, but this PR is not merge-ready because the proposed fallback is too broad, the root clearing conflicts with current scope-aware thread routing, and contributor-visible real behavior proof is missing.

## What This Changes

The PR changes Feishu mention parsing to infer reply mentions from orphaned placeholders, clears `rootId` for mentioned replies, and adds parser regression tests.

## Best Possible Solution

Rework the Feishu plugin fix in current `bot-content.ts`/routing structure: add the strongest available dropped-bot-mention detection, preserve topic and configured thread roots, add focused regression coverage, then land it to close the linked issue.

## Reproduction Assessment

Yes, by source inspection. A Feishu group text reply with `parent_id`/`root_id`, text containing a bot placeholder, and a non-empty `mentions` array that omits the bot leaves `mentionedBot=false`, then the `requireMention` gate returns early.

## Solution Assessment

No. The PR targets the right bug, but any-orphan placeholder detection is too broad and parse-time `rootId` clearing is not the narrowest maintainable routing fix on current main.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve topic roots for mentioned replies:** `extensions/feishu/src/bot.ts:823`
  - body: Clearing `rootId` during parsing for every mentioned reply happens before `groupSessionScope` and `replyInThread` are known. Current topic and configured-thread modes rely on that root as the session key and reply target, so addressed replies can collapse to the chat-level session or wrong target.
  - confidence: 0.88
- **[P2] Require bot-specific evidence for orphan placeholders:** `extensions/feishu/src/bot.ts:484`
  - body: The fallback treats any unmatched `@_...` token in reply text as a bot mention. A literal placeholder or dropped non-bot mention can satisfy `requireMention` and cause OpenClaw to respond without proof the bot was addressed.
  - confidence: 0.86
