## Summary

Keep open. Current main still appears to miss the Feishu reply-mention path, but this PR should not merge as-is because orphan placeholders can bypass mention gating, parse-time root clearing can break topic/configured thread routing, and after-fix real behavior proof is absent.

## What This Changes

The PR changes Feishu mention parsing to treat orphaned reply placeholders as bot mentions, clears `rootId` for mentioned replies, and adds parser regression tests.

## Best Possible Solution

Rework the Feishu plugin fix to detect dropped bot mentions with the strongest bot-specific evidence available, preserve topic/configured thread roots, and land with focused regression tests plus redacted real Feishu proof.

## Reproduction Assessment

Yes, by source inspection. A Feishu group reply with `parent_id`/`root_id`, text containing the bot placeholder, and a non-empty `mentions` array that omits the bot leaves `mentionedBot=false` on current main and is dropped by the `requireMention` gate.

## Solution Assessment

No. The PR targets the right Feishu-owned bug, but broad orphan-placeholder detection and parse-time `rootId` clearing are not the narrowest maintainable fix against current main.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve topic roots for mentioned replies:** `extensions/feishu/src/bot.ts:823`
  - body: Clearing `rootId` during parsing for every mentioned reply happens before `groupSessionScope` and `replyInThread` are known. Topic-mode and explicitly threaded groups use that root as the session key and reply target, so addressed replies can collapse to the chat-level session or wrong target.
  - confidence: 0.88
- **[P2] Require bot-specific evidence for orphan placeholders:** `extensions/feishu/src/bot.ts:484`
  - body: This fallback treats any unmatched `@_...` token in a reply as a bot mention. A literal placeholder or dropped non-bot mention can satisfy `requireMention` and cause OpenClaw to respond without proof the bot was addressed.
  - confidence: 0.86
