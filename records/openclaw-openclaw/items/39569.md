## Summary

Keep this PR open. Current main still lacks Feishu reply-message orphan-placeholder mention detection, so the paired Feishu bug remains live; however, the proposed patch needs rework because it broadly treats any orphan placeholder as a bot mention and clears `rootId` before the current topic/session routing logic can use it.

## What This Changes

This PR changes the Feishu channel parser to treat orphaned `@_user_N`/`@_bot_N` placeholders in text replies as bot mentions, clears `rootId` for mentioned replies, and adds parser tests.

## Best Possible Solution

Rework the fix on current main by adding reply-placeholder detection in `extensions/feishu/src/bot-content.ts`, covering the dropped-bot reply payload with regression tests, using the strongest bot-specific signal Feishu exposes, and preserving root/thread metadata so `bot.ts` can choose the correct reply target from group/session scope.

## Reproduction Assessment

Yes. A unit-level reproduction can construct a Feishu group text event with `parent_id`, text containing an `@_user_N` placeholder, and `mentions` empty or missing the bot; current `parseFeishuMessageEvent()` leaves `mentionedBot=false`, and `handleFeishuMessage()` drops it when `requireMention` is true. I did not run tests because this review was read-only, but the source path is direct.

## Solution Assessment

No. The PR targets the right bug, but clearing `rootId` before group scope resolution and accepting any orphan placeholder is not the narrowest maintainable fix; the safer approach is a current-main rework that keeps routing metadata intact and tightens the fallback as much as the Feishu payload allows.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve topic roots for mentioned replies:** `extensions/feishu/src/bot.ts:815`
  - body: Clearing `rootId` when `mentionedBot && parent_id` runs before the group session scope is known. In topic or explicit thread modes, current routing uses `ctx.rootId` to choose the topic session and reply target, so this can collapse an addressed reply into the chat-level session or wrong reply target instead of preserving the topic context.
  - confidence: 0.88
- **[P2] Require bot-specific evidence for orphan placeholders:** `extensions/feishu/src/bot.ts:481`
  - body: The fallback marks any unmatched `@_...` token in reply text as a bot mention. That means a literal placeholder or a dropped non-bot mention can satisfy `requireMention` and trigger bot processing even though the bot was not identified; the fallback needs a stronger bot-specific signal or a narrower Feishu payload check.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: No supply-chain files are changed, but the PR broadens Feishu group mention-gate behavior in a way that can trigger bot responses without confirmed bot mention evidence.

Concerns:

- **[medium] Broad orphan placeholder can bypass mention gating:** `extensions/feishu/src/bot.ts:481`
  - body: The new fallback treats any unmatched placeholder token in a reply as `mentionedBot=true`. In groups where `requireMention` is intended to limit bot activation, crafted placeholder-looking text or a dropped non-bot mention could cause OpenClaw to process and respond when the bot was not actually addressed.
  - confidence: 0.82
