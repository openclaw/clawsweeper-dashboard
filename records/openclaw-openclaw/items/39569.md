## Summary

Keep open. The Feishu reply-mention bug is still source-reproducible on current main, but this conflicted PR is not merge-ready because it can bypass mention gating with any orphan placeholder and clears reply roots before topic routing is known.

## What This Changes

The PR modifies Feishu mention parsing to infer bot mentions from orphaned reply placeholders, clears rootId for mentioned replies, and adds parser regression tests.

## Best Possible Solution

Rework the Feishu plugin fix in the current parser/routing split: add the strongest available bot-specific dropped-mention detection, preserve topic and explicit thread roots, add focused regression coverage, and close the linked issue through the landed implementation.

## Reproduction Assessment

Yes, by source inspection. A Feishu group text reply with `parent_id`/`root_id`, text containing a bot placeholder, and `mentions` that omit the bot still leaves `mentionedBot=false` on current main, so the `requireMention` gate returns early.

## Solution Assessment

No. The PR targets the right bug, but any-orphan placeholder detection is too broad and parse-time `rootId` clearing is not the narrowest maintainable routing fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve topic roots for mentioned replies:** `extensions/feishu/src/bot.ts:823`
  - body: Clearing `rootId` during parse for every mentioned reply happens before `groupSessionScope` and `replyInThread` are known. Topic and explicitly threaded groups rely on that root as the session key and reply target, so addressed replies can collapse to the chat-level session or wrong target.
  - confidence: 0.88
- **[P2] Require bot-specific evidence for orphan placeholders:** `extensions/feishu/src/bot.ts:484`
  - body: The fallback treats any unmatched `@_...` token in reply text as a bot mention. A literal placeholder or dropped non-bot mention can satisfy `requireMention` and cause OpenClaw to respond without proof the bot was addressed.
  - confidence: 0.86
