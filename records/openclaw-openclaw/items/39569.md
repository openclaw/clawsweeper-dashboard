## Summary

Keep open. The linked Feishu bug is still valid on current main, but this PR should not merge as-is because it clears `rootId` before topic scope is known, uses an over-broad orphan-placeholder heuristic, is currently conflicting, and lacks inspectable real behavior proof.

## What This Changes

This PR changes Feishu mention parsing to treat orphaned reply placeholders as bot mentions, clears `rootId` on mentioned replies, and adds parser tests.

## Best Possible Solution

Patch the Feishu plugin with the strongest available bot-specific dropped-mention detection and keep reply target selection scope-aware so normal groups can reply in the main chat without breaking topic sessions.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a Feishu group reply with a non-empty `mentions` array that omits the bot leaves `mentionedBot=false`, then the group `requireMention` gate drops the message.

## Solution Assessment

No. The direction is right, but this PR's broad orphan-key signal and parse-time `rootId` clearing are not the narrowest maintainable fix; detection and reply routing need to stay scope-aware.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve topic roots before clearing reply routing:** `extensions/feishu/src/bot.ts:823`
  - body: Clearing `rootId` whenever a mentioned reply is parsed happens before `groupSessionScope` is known. In `group_topic` and `group_topic_sender` modes, `resolveFeishuGroupSession()` uses that root as the topic key, so reply events without `thread_id` fall back to the chat-level session and can merge unrelated topic context.
  - confidence: 0.91
- **[P2] Require bot-specific evidence for orphan placeholders:** `extensions/feishu/src/bot.ts:481`
  - body: This treats any unmatched `@_user_N` token in reply text as a bot mention. A literal placeholder or a dropped non-bot mention can flip `mentionedBot` to true, bypass `requireMention`, and trigger the new root-clearing path without evidence that the bot was addressed.
  - confidence: 0.88
