## Summary

Keep open: the Feishu bug is still valid, but this PR is not merge-ready because the fallback can treat non-bot placeholders as bot mentions, parse-time root clearing conflicts with topic routing, the branch is conflicting against current main, and the claimed live Feishu verification is not inspectable.

## What This Changes

The branch modifies Feishu mention parsing to treat orphan reply placeholders as bot mentions, clears `rootId` on mentioned replies, and adds parser tests.

## Best Possible Solution

Rework the Feishu-local fix against the current `bot-content.ts` and `bot.ts` split: require bot-specific dropped-mention evidence, preserve root context through session routing, and let scope-aware reply-target logic choose main-chat versus topic replies.

## Reproduction Assessment

Yes, at source level. A Feishu group reply whose non-empty `mentions` array omits the bot open_id leaves `mentionedBot=false`, and mention-required group ingress can return before dispatch.

## Solution Assessment

No. The fix belongs in the Feishu plugin, but this implementation uses non-bot orphan placeholders and parse-time root clearing; the safer solution is bot-specific mention evidence plus scope-aware routing.
