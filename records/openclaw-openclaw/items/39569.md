## Summary

Keep open: the linked Feishu bug is still valid on current main, but this branch is not merge-ready because its fallback can admit unaddressed messages, its root clearing breaks topic scopes, and no inspectable live proof is attached.

## What This Changes

The branch changes Feishu mention parsing to treat orphaned reply placeholders as bot mentions, clears rootId for mentioned replies, and adds parser tests.

## Best Possible Solution

Rework or replace this with a Feishu plugin patch that uses the strongest bot-specific dropped-mention signal available, keeps topic/session root handling scope-aware, and includes focused regression tests plus redacted live proof.

## Reproduction Assessment

Yes. By source inspection, a Feishu group reply with text containing the bot placeholder but a non-empty mentions array that omits the bot returns mentionedBot=false and is dropped by the requireMention group ingress gate.

## Solution Assessment

No. The fix belongs in the Feishu plugin, but this PR's any-orphan heuristic and parse-time root clearing are broader than the safe behavior; the maintainable fix should be bot-specific and scope-aware.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve topic roots before clearing reply routing:** `extensions/feishu/src/bot.ts:823`
  - body: Clearing rootId during parsing happens before groupSessionScope is known. In group_topic and group_topic_sender modes, resolveFeishuGroupSession uses that root as the topic key, so mentioned replies without thread_id can fall back to the chat-level session and merge unrelated topic context.
  - confidence: 0.91
- **[P2] Require bot-specific evidence for orphan placeholders:** `extensions/feishu/src/bot.ts:484`
  - body: This treats any unmatched @_..._N token in reply text as a bot mention. A literal placeholder or dropped non-bot mention can flip mentionedBot to true, bypass requireMention, and trigger root clearing without evidence that the bot was addressed.
  - confidence: 0.88
