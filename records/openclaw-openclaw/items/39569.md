## Summary

Keep open. Current main has not implemented the Feishu reply-mention fix, and this linked PR has useful source work, but it should not merge as-is because it broadens mention detection, can break topic-scoped routing, and lacks inspectable real behavior proof.

## What This Changes

The PR changes Feishu mention parsing so reply messages with orphaned mention placeholders can mark the bot as mentioned and clear reply-thread routing, with regression tests.

## Best Possible Solution

Land a current-main Feishu plugin fix that detects dropped bot mentions with the strongest available bot-specific signal while preserving existing topic-session and explicit thread routing contracts.

## Reproduction Assessment

Yes from source inspection. A Feishu group reply with a non-empty `mentions` array that omits the bot will leave `mentionedBot=false` on current main, which then fails the documented group @mention gate when mention-required policy applies.

## Solution Assessment

No. The fix belongs in the Feishu plugin, but this PR's root clearing is too broad for topic-scoped sessions and the orphan-placeholder fallback needs a stronger bot-specific guard.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P1] Preserve topic roots before clearing reply routing:** `extensions/feishu/src/bot.ts:823`
  - body: Clearing `rootId` for every mentioned reply also affects groups configured with `groupSessionScope: "group_topic"` or `"group_topic_sender"`. When a reply has `root_id` but no `thread_id`, current routing uses that root as the topic key; removing it collapses the turn to chat-level routing and can merge unrelated topic context.
  - confidence: 0.88
- **[P2] Require stronger evidence before marking the bot mentioned:** `extensions/feishu/src/bot.ts:485`
  - body: The orphan-key fallback treats any unmatched `@_user_N` token in a reply as a bot mention, including literal user text or a dropped non-bot mention. That can bypass `requireMention` and trigger the new routing behavior without evidence that the bot was addressed.
  - confidence: 0.8
