## Summary

This PR should stay open because current main still appears to miss the reported Feishu reply-mention shape, but the proposed patch is too broad: it treats any orphan placeholder in a reply as a bot mention and clears reply roots before session scope is known.

## What This Changes

The branch changes Feishu mention parsing to infer reply mentions from orphaned text placeholders, clears `rootId` for mentioned replies, and adds parser tests for that behavior.

## Best Possible Solution

Keep the fix in the Feishu plugin, but make mention recovery bot-specific or payload-backed and defer root/reply routing decisions until the existing group session-scope logic can apply.

## Reproduction Assessment

Yes, by source inspection. A Feishu group text reply with a bot placeholder left in `content.text` but no bot entry in `message.mentions` remains `mentionedBot=false`, so mention-required group ingress drops it before dispatch.

## Solution Assessment

No. The PR targets the right plugin surface, but the any-orphan heuristic and parse-time root clearing are broader than the existing mention-gating and topic-routing contracts allow.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve topic roots before clearing reply routing:** `extensions/feishu/src/bot.ts:823`
  - body: Clearing `rootId` during parsing happens before `groupSessionScope` is known. In `group_topic` and `group_topic_sender` modes, routing uses that root as the topic key, so mentioned replies without `thread_id` can fall back to the chat-level session and merge unrelated topic context.
  - confidence: 0.91
- **[P2] Require bot-specific evidence for orphan placeholders:** `extensions/feishu/src/bot.ts:484`
  - body: This treats any unmatched `@_..._N` token in reply text as a bot mention. A literal placeholder or dropped non-bot mention can flip `mentionedBot` to true, bypass `requireMention`, and trigger root clearing without evidence that the bot was addressed.
  - confidence: 0.88
