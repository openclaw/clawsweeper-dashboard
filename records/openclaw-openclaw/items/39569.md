## Summary

Keep open. The linked Feishu bug is still source-reproducible on current main, but this PR is not ready to merge because it can mark non-bot placeholder text as a bot mention and clears `rootId` before preserving topic/thread routing.

## What This Changes

The PR changes Feishu mention parsing to treat orphaned reply placeholders as bot mentions, clears `rootId` for mentioned replies, and adds focused parser tests.

## Best Possible Solution

Rework the Feishu plugin fix on current main so dropped bot mentions are detected with the strongest available bot-specific signal, while preserving scope-aware topic and explicit-thread routing.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a Feishu text reply with `parent_id`, a non-empty `mentions` array that omits the bot, and text containing the dropped bot placeholder leaves `mentionedBot=false` and can be dropped by the group mention gate.

## Solution Assessment

No. The direction belongs in the Feishu plugin, but this branch should not merge while any orphan placeholder can mark the bot mentioned and while `rootId` is cleared before topic/thread scope is considered.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve topic roots for mentioned reply events:** `extensions/feishu/src/bot.ts:823`
  - body: Clearing `rootId` during parsing removes the only topic anchor before group session scope is known. In `group_topic`, `group_topic_sender`, or explicit `replyInThread` configurations, current routing needs `rootId` to keep the session and reply in the expected topic; this change can collapse topic conversations back to the chat-level reply target.
  - confidence: 0.88
- **[P2] Require a bot-specific signal before treating orphan keys as mentions:** `extensions/feishu/src/bot.ts:486`
  - body: This fallback turns any unmatched `@_user_N` token in reply text into `mentionedBot=true`. A literal placeholder or a dropped non-bot mention can bypass `requireMention` and trigger the routing changes even though the bot was not addressed, so the heuristic needs a stronger bot-specific guard.
  - confidence: 0.84
