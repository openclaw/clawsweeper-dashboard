## Summary

## What This Changes

The PR changes Feishu mention parsing to infer dropped bot mentions from orphan reply placeholders, clears `rootId` for mentioned replies, and adds parser tests.

## Best Possible Solution

Rework the Feishu-local fix against current main: require bot-specific dropped-mention evidence, preserve root context through session routing, and let scope-aware reply target logic choose main-chat versus topic replies.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes, at source level: a Feishu group reply whose non-empty `mentions` array omits the bot open_id leaves `mentionedBot=false`, and mention-required group ingress can return before dispatch.

## Solution Assessment

Is this the best way to solve the issue? No. The fix belongs in the Feishu plugin, but this implementation uses non-bot orphan placeholders and parse-time root clearing; the safer solution is bot-specific mention evidence plus scope-aware routing.
