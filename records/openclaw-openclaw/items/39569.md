## Summary

Keep open. The Feishu reply-mention bug is still source-reproducible on current main, but this PR's broad orphan-placeholder fallback can bypass mention-required group gating and its parse-time `rootId` clearing can break topic-scoped routing; external real behavior proof is also missing.

## What This Changes

The PR changes Feishu mention parsing to treat orphaned reply placeholders as bot mentions, clears `rootId` on mentioned replies, and adds parser tests.

## Best Possible Solution

Rework the Feishu plugin to detect dropped bot mentions with the strongest available bot-specific signal and choose reply targets after group session scope is known, preserving topic roots while keeping normal group replies visible.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a Feishu group reply with a non-empty `mentions` array that omits the bot leaves `mentionedBot=false`, then the group `requireMention` gate drops the message.

## Solution Assessment

No. The direction is right, but the current PR is not the narrowest maintainable fix because mention detection and reply routing need to stay bot-specific and scope-aware.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve topic roots before clearing reply routing:** `extensions/feishu/src/bot.ts:823`
  - body: Clearing `rootId` whenever a mentioned reply is parsed happens before `groupSessionScope` is known. In `group_topic` and `group_topic_sender` modes, `resolveFeishuGroupSession()` uses that root as the topic key, so reply events without `thread_id` can fall back to the chat-level session and merge unrelated topic context.
  - confidence: 0.91
- **[P2] Require bot-specific evidence for orphan placeholders:** `extensions/feishu/src/bot.ts:481`
  - body: This treats any unmatched `@_..._N` token in reply text as a bot mention. A literal placeholder or dropped non-bot mention can flip `mentionedBot` to true, bypass `requireMention`, and trigger the new root-clearing path without evidence that the bot was addressed.
  - confidence: 0.88
