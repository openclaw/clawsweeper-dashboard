## Summary

Keep open: current main still has the Feishu dropped-mention path, and this external PR is not merge-ready because inspectable real behavior proof is missing and the patch has correctness blockers around mention gating and topic routing.

## What This Changes

The PR updates Feishu mention parsing to treat orphaned reply placeholders as bot mentions, clears `rootId` for mentioned replies, and adds parser tests.

## Best Possible Solution

Rebase or replace this with a Feishu plugin fix that detects dropped bot mentions using the strongest bot-specific signal available, keeps reply target selection after group session scope is known, and adds focused regression tests.

## Reproduction Assessment

Yes. Source inspection on current main shows a group reply with a non-empty `mentions` array that omits the bot returns `mentionedBot=false` in `checkBotMentioned()`, then the group `requireMention` ingress gate drops it.

## Solution Assessment

No. The fix belongs in the Feishu plugin, but this PR's any-orphan heuristic can bypass mention-required dispatch and parse-time root clearing can break topic scopes; the solution should be bot-specific and scope-aware.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve topic roots before clearing reply routing:** `extensions/feishu/src/bot.ts:823`
  - body: Clearing `rootId` during parsing happens before `groupSessionScope` is known. In `group_topic` and `group_topic_sender` modes, the group session resolver uses that root as the topic key, so mentioned replies without `thread_id` can fall back to the chat-level session and merge unrelated topic context.
  - confidence: 0.91
- **[P2] Require bot-specific evidence for orphan placeholders:** `extensions/feishu/src/bot.ts:484`
  - body: This treats any unmatched `@_..._N` token in reply text as a bot mention. A literal placeholder or dropped non-bot mention can flip `mentionedBot` to true, bypass `requireMention`, and trigger the root-clearing path without evidence that the bot was addressed.
  - confidence: 0.88
