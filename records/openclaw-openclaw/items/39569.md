## Summary

Keep this PR open: it targets a valid Feishu mention-gating bug, but the current patch still has blocking routing and activation-safety problems and also needs rework against the current `bot-content.ts` split.

## What This Changes

The PR updates Feishu mention parsing to treat orphaned reply placeholders as bot mentions, clears reply root metadata for mentioned replies, and adds parser regression tests.

## Best Possible Solution

Rework the fix in the current `bot-content.ts`/`bot.ts` split, preserve root/thread metadata for topic scopes, only loosen mention gating with narrowly bounded Feishu evidence, and add focused tests plus a changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: parse a Feishu group text reply with `parent_id` or `root_id`, text containing an `@_user_N` placeholder, and `mentions` missing the bot; current main leaves `mentionedBot=false`, and the group `requireMention` gate drops it.

## Solution Assessment

No. The PR addresses the right symptom, but clearing `rootId` during parsing and treating any orphan placeholder as a bot mention are not the narrowest maintainable or safest fixes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve topic roots for mentioned replies:** `extensions/feishu/src/bot.ts:816-820`
  - body: Clearing `rootId` whenever `mentionedBot && parent_id` runs before the group session scope is known. Topic modes use `rootId` as the topic key and reply target, so addressed replies can collapse into the chat-level session or wrong reply target instead of preserving topic context.
  - confidence: 0.88
- **[P2] Require bot-specific evidence for orphan placeholders:** `extensions/feishu/src/bot.ts:481`
  - body: The fallback marks any unmatched `@_...` token in reply text as `mentionedBot=true`. A literal placeholder or dropped non-bot mention can satisfy `requireMention` and trigger processing even though the bot was not identified; narrow this to a stronger bot-specific signal or explicit Feishu payload condition.
  - confidence: 0.86

## Security Review
