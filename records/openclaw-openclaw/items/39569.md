## Summary

Keep open. Current main still misses the reported Feishu reply-mention path, but this PR remains dirty, lacks inspectable real-behavior proof, and introduces mention-gating/thread-routing concerns that should block merge as written.

## What This Changes

The PR changes Feishu mention parsing to infer bot mentions from orphaned reply placeholders, clears reply roots for mentioned replies, and adds parser tests.

## Best Possible Solution

Rework the Feishu plugin fix in the current parser/routing split: use the strongest available bot-specific dropped-mention detection, preserve topic and explicit thread roots, add focused regression coverage, and land it through this PR or a replacement tied to the linked issue.

## Reproduction Assessment

Yes, by source inspection. A Feishu group text reply with parent_id/root_id, text containing a bot placeholder, and mentions that omit the bot still leaves mentionedBot=false on current main, so the requireMention gate returns early.

## Solution Assessment

No. The PR targets the right bug, but any-orphan placeholder detection is too broad and parse-time rootId clearing is not the narrowest maintainable routing fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve topic roots for mentioned replies:** `extensions/feishu/src/bot.ts:823`
  - body: Clearing rootId during parse for every mentioned reply happens before groupSessionScope and replyInThread are known. Topic and explicitly threaded groups rely on that root as the session key and reply target, so addressed replies can collapse to the chat-level session or wrong target.
  - confidence: 0.88
- **[P2] Require bot-specific evidence for orphan placeholders:** `extensions/feishu/src/bot.ts:484`
  - body: The fallback treats any unmatched placeholder token in reply text as a bot mention. A literal placeholder or dropped non-bot mention can satisfy requireMention and cause OpenClaw to respond without proof the bot was addressed.
  - confidence: 0.86

## Security Review
