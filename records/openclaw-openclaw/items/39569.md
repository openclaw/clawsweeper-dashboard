## Summary

Keep open: the linked Feishu bug is still source-reproducible on current main, but this PR is not merge-ready because it lacks inspectable real Feishu proof, conflicts with main, and broadens mention/root routing in ways that can break gating and topic sessions.

## What This Changes

The PR adds a Feishu reply-mention fallback that treats orphaned text placeholders as bot mentions, clears `rootId` for mentioned replies, and adds parser tests.

## Best Possible Solution

Rework or replace this PR with a Feishu-local fix that uses bot-specific dropped-mention evidence and selects reply targets after group session scope is known, then attach redacted live Feishu proof.

## Reproduction Assessment

Yes, source-level. A Feishu group reply whose non-empty `mentions` array omits `botOpenId` leaves `mentionedBot=false`, then mention-required group ingress can return before dispatch.

## Solution Assessment

No. The fix belongs in the Feishu plugin, but this PR’s any-orphan placeholder heuristic and parse-time `rootId` clearing are broader than the narrow maintainable repair.
