## Summary

Keep open. Current main still lacks Feishu reply-placeholder mention detection and the paired bug report remains open, but this patch should not merge as-is because it clears topic root metadata and treats any orphan placeholder as a bot mention.

## What This Changes

The PR adds a Feishu reply-message orphan-placeholder mention fallback, clears `rootId` for mentioned replies, and adds parser tests for reply mention/root behavior.

## Best Possible Solution

Rework the Feishu plugin fix on current main by adding narrowly scoped reply-placeholder handling in `bot-content.ts`, preserving topic root metadata for topic sessions, and covering mention gating plus reply target behavior with focused regression tests and a changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a Feishu group text reply with `parent_id`, an `@_user_N` placeholder in text, and `mentions` missing the bot leaves `mentionedBot=false`, then the group `requireMention` gate drops it.

## Solution Assessment

No. The PR targets the right bug, but clearing `rootId` before session routing and accepting any orphan placeholder are not the narrowest maintainable or safest solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve topic roots for mentioned replies:** `extensions/feishu/src/bot.ts:816-820`
  - body: Clearing `rootId` when `mentionedBot && parent_id` runs before the group session scope is known. Topic modes use `ctx.rootId` as the topic key and reply target, so addressed replies can collapse into the chat-level session or wrong reply target instead of preserving topic context.
  - confidence: 0.88
- **[P2] Require bot-specific evidence for orphan placeholders:** `extensions/feishu/src/bot.ts:481`
  - body: The fallback marks any unmatched `@_...` token in reply text as `mentionedBot=true`. A literal placeholder or dropped non-bot mention can satisfy `requireMention` and trigger processing even though the bot was not identified; narrow this to a stronger bot-specific signal or explicit Feishu payload condition.
  - confidence: 0.86

## Security Review

Status: needs_attention

Summary: No supply-chain files are changed, but the diff broadens Feishu group activation in a way that can bypass mention gating without confirmed bot mention evidence.

Concerns:

- **[medium] Broad orphan placeholder can bypass mention gating:** `extensions/feishu/src/bot.ts:481`
  - body: The fallback treats any unmatched reply placeholder as `mentionedBot=true`. In groups using `requireMention`, crafted placeholder-looking text or a dropped non-bot mention could trigger OpenClaw processing and responses when the bot was not actually addressed.
  - confidence: 0.82
