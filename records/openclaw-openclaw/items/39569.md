## Summary

Keep open: this targets a valid Feishu bug, but the branch is stale against the current Feishu split and the patch still has blocking correctness and mention-gating concerns around topic roots and orphan placeholders.

## What This Changes

The PR changes Feishu mention parsing to infer bot mentions from orphaned reply placeholders, clears `rootId` on mentioned replies, and adds parser tests for mention/root handling.

## Best Possible Solution

Rework the Feishu fix in the current `bot-content.ts` / `bot.ts` split, preserve topic roots for topic scopes, require bot-specific mention evidence or an explicitly approved heuristic, and add focused parser/dispatch tests plus a changelog entry.

## Reproduction Assessment

Yes. Source inspection shows a current-main path: parse a Feishu group text reply with `parent_id` or `root_id`, an `@_user_N` placeholder, and `mentions` missing the bot; `mentionedBot` remains false and the group `requireMention` gate drops it.

## Solution Assessment

No. The PR addresses the right user-visible bug, but clearing `rootId` before session scope is known and treating any orphan placeholder as a bot mention are not the narrowest maintainable or safest fixes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve topic roots for mentioned replies:** `extensions/feishu/src/bot.ts:823`
  - body: Clearing `rootId` when `mentionedBot && parent_id` runs before the group session scope is known. Topic modes use `ctx.rootId` as the topic key and reply target, so addressed replies can collapse into the chat-level session or wrong reply target instead of preserving topic context.
  - confidence: 0.88
- **[P2] Require bot-specific evidence for orphan placeholders:** `extensions/feishu/src/bot.ts:481`
  - body: The fallback marks any unmatched `@_...` token in reply text as `mentionedBot=true`. A literal placeholder or dropped non-bot mention can satisfy `requireMention` and trigger processing even though the bot was not identified.
  - confidence: 0.86
