## Summary

Keep open: the PR targets a real Feishu reply-mention bug that is still source-reproducible on current main, but the branch is conflicting, lacks inspectable real Feishu proof, and introduces blocking mention-gating and topic/session routing risks.

## What This Changes

The branch changes Feishu mention parsing in the older `bot.ts` shape to infer reply mentions from orphan placeholders, clears `rootId` for mentioned replies, and adds parser tests.

## Best Possible Solution

Rework the Feishu-local fix against the current `bot-content.ts` and `bot.ts` split: require bot-specific dropped-mention evidence, preserve root context through session routing, let scope-aware reply targeting choose the destination, and attach redacted live Feishu proof.

## Reproduction Assessment

Yes, at source level. A Feishu group reply whose non-empty `mentions` array omits the bot open_id leaves `mentionedBot=false`, and mention-required group ingress can return before dispatch.

## Solution Assessment

No. The fix belongs in the Feishu plugin, but this implementation uses non-bot orphan placeholders and parse-time root clearing; the safer solution is bot-specific mention evidence plus scope-aware routing.
