## Summary

Keep open: the underlying Feishu bug is still source-reproducible on current main, but this PR is not merge-ready because it needs real behavior proof, a current-main rework, and narrower mention/session routing semantics before it can safely land.

## What This Changes

The PR changes Feishu mention parsing to infer dropped reply mentions from orphan placeholders, clears `rootId` for mentioned replies, and adds parser tests.

## Best Possible Solution

Rework the Feishu-local fix against the current `bot-content.ts` and `bot.ts` split: require bot-specific dropped-mention evidence, preserve root context through session routing, and let the existing scope-aware reply target logic choose main-chat versus topic replies.

## Reproduction Assessment

Yes, at source level. A Feishu group reply whose non-empty `mentions` array omits the bot open_id leaves `mentionedBot=false`, and mention-required group ingress can return before dispatch.

## Solution Assessment

No. The fix belongs in the Feishu plugin, but this implementation uses non-bot orphan placeholders and parse-time root clearing; the safer solution is bot-specific mention evidence plus scope-aware routing.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve topic roots before choosing reply targets:** `extensions/feishu/src/bot.ts:823`
  - body: Clearing `rootId` in `parseFeishuMessageEvent` runs before `groupSessionScope` is resolved. For `group_topic` and `group_topic_sender`, `rootId` is the topic/session key, so addressed replies with `root_id` can fall back to the chat-level session and merge unrelated topic context.
  - confidence: 0.91
- **[P2] Require bot-specific evidence for orphan placeholders:** `extensions/feishu/src/bot.ts:484`
  - body: This fallback marks any unmatched placeholder token in reply text as a bot mention. A literal token or dropped non-bot mention can bypass `requireMention` and trigger root clearing without evidence that the bot was addressed.
  - confidence: 0.88
