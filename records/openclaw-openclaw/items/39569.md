## Summary

Keep open. The Feishu reply-mention bug still exists on current main, but this PR is not merge-ready because it broadens mention gating, clears thread roots before routing scope is known, and lacks after-fix real behavior proof.

## What This Changes

The PR changes Feishu message parsing to infer bot mentions from orphaned reply placeholders, clears `rootId` for mentioned replies, and adds parser tests for those cases.

## Best Possible Solution

Rework the Feishu plugin fix in the current parser/routing split: add the strongest available bot-specific dropped-mention detection, keep topic and explicit thread roots intact, and cover both mention detection and reply targeting with focused tests.

## Reproduction Assessment

Yes, by source inspection. A Feishu group text reply with `parent_id` or `root_id`, text containing the bot placeholder, and a `mentions` array that omits the bot still leaves `mentionedBot=false` on current main and is dropped by the group mention gate.

## Solution Assessment

No. The Feishu plugin is the right boundary, but the patch should avoid any-orphan activation and adjust reply targeting after session scope is known instead of clearing `rootId` during parse.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve topic roots for mentioned replies:** `extensions/feishu/src/bot.ts:823`
  - body: Clearing `rootId` in `parseFeishuMessageEvent` runs before `groupSessionScope` or explicit `replyInThread` is resolved. Topic-mode and configured-thread groups use that root as the session and reply target, so addressed replies can collapse into the chat-level session or reply to the wrong place.
  - confidence: 0.88
- **[P2] Require bot-specific evidence for orphan placeholders:** `extensions/feishu/src/bot.ts:484`
  - body: This fallback accepts any unmatched `@_...` token as proof that the bot was mentioned. A literal placeholder or dropped non-bot mention can pass `requireMention` and trigger a reply without bot-specific evidence.
  - confidence: 0.86
