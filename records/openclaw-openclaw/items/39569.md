## Summary

Keep open. The PR targets a real Feishu reply-mention bug that still exists on current main, but the proposed patch has correctness risks around topic routing and broad orphan-placeholder mention detection, and the external contributor has not provided inspectable real behavior proof.

## What This Changes

The PR changes Feishu mention parsing so reply messages with orphaned mention placeholders can count as bot mentions and clears reply root routing when the bot is mentioned.

## Best Possible Solution

Rebase or replace this PR with a Feishu plugin patch that implements dropped bot-mention detection in the current `bot-content.ts` path, preserves scope-aware topic/reply routing, adds focused regressions, and includes redacted real Feishu proof.

## Reproduction Assessment

Yes, from source inspection for the reported payload shape: a group reply with `mentions` present but missing the bot returns false before checking the text placeholder, then the group `requireMention` gate can drop it. I did not run a live Feishu reproduction in this read-only review.

## Solution Assessment

No for the current PR shape. The fix belongs in the Feishu plugin, but it should use a narrower bot-specific dropped-mention path and defer any main-chat reply choice until after topic/session scope is known.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve topic roots when clearing reply routing:** `extensions/feishu/src/bot.ts:812`
  - body: Clearing `rootId` here happens before `resolveFeishuGroupSession` knows whether the group is using `group_topic` or `group_topic_sender`. For reply events that have `root_id` but no `thread_id`, this removes the only topic key, so topic-scoped sessions fall back to chat/sender routing and replies no longer stay on the configured topic root.
  - confidence: 0.87
- **[P2] Require bot-specific evidence before orphan-key mentions:** `extensions/feishu/src/bot.ts:481`
  - body: This fallback treats any unmatched `@_..._N` token in a reply as a bot mention. A literal placeholder or a non-bot mention omitted from Feishu metadata can bypass `requireMention` and trigger the new root-clearing path, so the fallback needs a stronger bot-specific signal or a narrower guard.
  - confidence: 0.82
