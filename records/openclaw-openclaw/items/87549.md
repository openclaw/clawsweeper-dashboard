## Summary

Keep open for normal maintainer review. ## What This Changes

The PR adds runtime validation and tests for `assembleHarnessContextEngine` so malformed context-engine `assemble()` results without a `messages` array throw before runner state is corrupted.

## Best Possible Solution

Land the narrow boundary guard after maintainers accept the `messages`-only validation scope, keeping the broader context-engine RFC separate.

## Reproduction Assessment

Yes from source inspection: current main returns a malformed `assemble()` result unchanged, the runner then assigns `assembled.messages`, and later runner code reads message array lengths. The PR body also supplies before/after terminal output for the malformed lossless-claw-shaped payload.

## Solution Assessment

Yes with a contract-scope caveat: validating the required `messages` array at the shared harness boundary is the narrowest maintainable fix and both embedded and Codex callers already recover from thrown assemble errors. The only maintainer choice is whether to also validate `estimatedTokens` before merge.
