## Summary

Keep open for maintainer review. The patch is a focused fix for the reported malformed context-engine assemble result crash and I found no blocking code-review findings, but it intentionally tightens the plugin/harness runtime contract and the reviewed head still has a failing `check-test-types` check to clear before merge.

## What This Changes

The PR adds a shared harness validator for context engine `assemble()` results and regression tests for malformed `messages` payloads before runner state consumption.

## Best Possible Solution

The shared harness should reject malformed `AssembleResult.messages` before session state mutation, keep well-formed engines unchanged, and leave broader context-engine metadata/API expansion to the separate RFC.

## Reproduction Assessment

Yes. By source inspection on current main, a context engine returning `{ estimatedTokens: 0 }` passes the truthy result check, can assign `undefined` into session messages, and later reaches `.length` reads in the runner.

## Solution Assessment

Yes, with one maintainer scope choice. A shared harness guard is the narrow maintainable fix for malformed assemble output; validating `estimatedTokens` too is optional broader contract hardening, not required for the reported crash.
