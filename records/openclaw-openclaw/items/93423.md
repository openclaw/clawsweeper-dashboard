## Summary

Keep open: current main and v2026.6.8 still deliver forwarded exec approval prompts immediately and expose no hold/retract API. The request is a product/security-sensitive feature because it would add approval config plus an agent-facing cancellation path on the execution approval boundary.

## What This Changes

The issue requests an opt-in hold-and-retract window for forwarded exec approval prompts so agents can cancel unintended approval requests before they reach a human.

## Best Possible Solution

Keep this open for maintainer review of a narrow opt-in design that preserves immediate delivery by default, fail-opens delivery on buffering errors, and maps retraction to a denial or abort result without executing the command.
