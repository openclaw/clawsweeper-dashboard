## Summary

Keep open: current main and the latest release still deliver forwarded exec approvals immediately and expose no hold/retract config or agent-accessible retraction RPC. This is a real approval-ergonomics feature, but it needs maintainer design because it changes approval timing, config, Gateway protocol, audit, and Codex/native approval semantics.

## What This Changes

The issue requests an opt-in exec approval hold window plus agent-side retraction so forwarded approval prompts can be cancelled before human delivery.

## Best Possible Solution

Keep this open for maintainer design of an opt-in approval hold/retract mechanism that updates config, Gateway protocol/schema/docs, forwarding tests, audit logging, and native/Codex approval handling while preserving the stated safety constraints.

## Reproduction Assessment

Not applicable as a feature request. Source inspection shows current main and `v2026.6.9` lack the proposed `holdMs` config and agent-side retraction RPC.

## Solution Assessment

Unclear until maintainers choose the product/API boundary. A default-zero hold window plus retraction RPC is plausible, but it changes approval semantics across OpenClaw and Codex rather than repairing a narrow broken path.
