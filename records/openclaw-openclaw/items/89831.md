## What This Changes

The PR gates the diagnostic stuck-session fallback so sessions with fresh progress but no active work kind classify as long-running instead of recoverable stuck, with two regression cases for queued and non-queued snapshots.

## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

Yes, this is the narrowest maintainable fix for the source-confirmed classifier bug because diagnostic activity already treats progress and active-work ownership as separate facts. Registering CLI sessions as embedded active work would be a broader runtime taxonomy change and is not required to stop false stuck recovery on fresh progress.
