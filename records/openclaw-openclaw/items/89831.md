## Solution Assessment

Yes for the source-confirmed classifier bug: the PR gates only the no-active-work fallback on fresh progress and preserves stale or missing-progress recovery. A broader CLI active-work taxonomy could be considered later, but it is not required to stop false recovery while progress is fresh.
