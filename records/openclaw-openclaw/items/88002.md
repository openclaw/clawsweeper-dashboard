## Summary

Keep this open: current main and v2026.6.9 still have the cron run-history session-key mismatch, and the direct fix PR was closed unmerged without becoming a safe canonical fix.

## What This Changes

The issue reports that cron run history's Open Chat button redirects to the wrong conversation for runs in OpenClaw 2026.5.6 through 2026.5.27.

## Best Possible Solution

Resolve cron run-history Open Chat to a session-store entry containing the selected run transcript, either with a confirmed runSessionKey alias or a canonical deep-link key, plus focused regression coverage and real Control UI proof.

## Reproduction Assessment

Yes at source level: current main records and opens a run-specific cron session key while persistence stores only the stable cron session key. A live Control UI click-through reproduction was not run in this read-only review.

## Solution Assessment

Unclear as the final code path: the closed alias-persistence PR is the narrowest known repair, but maintainers should confirm that this is the intended session-state model before reviving it.
