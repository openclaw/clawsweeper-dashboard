- `impact:session-state`: Splitting same-action WhatsApp messages into separate agent turns can lose conversational context and alter the session transcript.

## Summary

Keep open. Current main and v2026.5.12 still use receive-time WhatsApp inbound debouncing; Baileys `messageTimestamp` is preserved as message metadata but not used as a coalescing key, timer input, or documented config surface, and the adjacent composing-aware debounce PR does not implement timestamp coalescing.

## What This Changes

Requests WhatsApp inbound batching that coalesces same-chat messages by Baileys `messageTimestamp` instead of only by receive-time debounce windows.

## Best Possible Solution

Add a WhatsApp-owned timestamp coalescing path for text-only same-account, same-conversation, same-sender messages with conservative tolerance, schema/types/docs, fake-timer coverage, and coordination with the composing-aware debounce PR.

## Reproduction Assessment

Yes, source-level. Two same-sender WhatsApp text messages with the same `messageTimestamp` but delivered after `debounceMs` expires will flush separately because the receive-time timer deletes the first buffer before the later message arrives.

## Solution Assessment

Unclear for the exact proposed config shape. The maintainable direction is a WhatsApp-owned timestamp coalescing path with conservative defaults, not a broad shared debouncer semantic change.
