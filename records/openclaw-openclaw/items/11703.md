- `impact:session-state`: Splitting same-action WhatsApp messages into separate agent turns can lose conversational context and alter the session transcript.

## Summary

Keep open. Current main and the latest release still batch WhatsApp inbound messages by receive-time debounce; `messageTimestamp` is preserved as message metadata but is not used in the debounce key, timer, schema, or docs, and the adjacent composing-aware debounce PR does not implement timestamp coalescing.

## What This Changes

Request new WhatsApp inbound batching that groups same-action messages by Baileys `messageTimestamp` rather than only by receive-time debounce.

## Best Possible Solution

Implement an opt-in or otherwise backwards-compatible WhatsApp-owned timestamp coalescing mode for text-only same-account, same-conversation, same-sender messages, with conservative tolerance, schema/types/docs, and focused fake-timer coverage.

## Reproduction Assessment

Yes, source-level. Two same-sender WhatsApp text messages with the same `messageTimestamp` but delivered after `debounceMs` expires will flush separately because the receive-time timer deletes the first buffer before the later message arrives.

## Solution Assessment

Unclear for the exact proposed config shape. The maintainable direction is a WhatsApp-owned timestamp coalescing path with conservative defaults/tolerance, not a broad shared debouncer semantic change.
