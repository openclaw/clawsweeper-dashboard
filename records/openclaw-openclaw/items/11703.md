## Summary

Source inspection confirms the requested WhatsApp timestamp-based coalescing is still missing on current main: `messageTimestamp` is retained as metadata, while batching is still keyed only by account/conversation/sender and flushed by receive-time timers. This should stay open because the useful fix requires a new WhatsApp-owned behavior/config decision rather than conservative cleanup closure.

## What This Changes

Add WhatsApp inbound batching that can group same-sender messages by WhatsApp `messageTimestamp` instead of receive time alone.

## Best Possible Solution

Add WhatsApp plugin-level timestamp coalescing for text-only same account/conversation/sender messages, with conservative tolerance/defaults, schema/types/docs, and focused regression coverage.

## Reproduction Assessment

Yes, source-level. Two same-sender WhatsApp text messages with the same `messageTimestamp` but delivered after `debounceMs` has elapsed will flush separately because the current buffer is deleted by the receive-time timer before the later message arrives.

## Solution Assessment

Unclear for the exact proposed config shape. The maintainable direction is WhatsApp-owned timestamp coalescing, not a broad shared debouncer semantic change unless maintainers intentionally want this behavior across channels.
