The PR replaces both `FileAuthStorageBackend` `auth.json` persistence writes with durable `replaceFileAtomicSync` calls and adds a Vitest regression for interrupted writes preserving credentials.

## Best Possible Solution

Land the atomic credential write after secops confirms the durability behavior and either accepts the `0o700` parent-directory tightening or adjusts the patch to preserve existing parent directory mode.

Yes, with one merge-safety choice left. Durable sibling-temp replacement is the right in-tree fix for the interrupted write, but maintainers should decide whether the parent-directory mode should be preserved or tightened.
