## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Persist feedback only after reaction authorization:** `extensions/discord/src/monitor/listeners.reactions.ts:470-480`
  - body: For guild and thread reactions, this capture runs before `resolveThreadChannelAccess` or `authorizeReactionIngressForChannel` applies channel allowlist and member-role checks. That means reactions from denied channels or disallowed members can still be written to durable feedback state; move capture behind the same authorized path or pass an already-authorized result into it.
  - confidence: 0.9
- **[P2] Use plugin state instead of a JSONL sidecar:** `extensions/discord/src/monitor/reaction-feedback.ts:36`
  - body: This creates a new durable `OPENCLAW_STATE_DIR/discord/*-reaction-feedback.jsonl` file, but OpenClaw runtime state is database-first and Discord already uses `runtime.state.openKeyedStore`/`openSyncKeyedStore` for plugin-owned persistence. Store these records in a Discord plugin-state namespace or an explicitly approved SQLite/product artifact so doctor, limits, backup, and cleanup behavior do not drift.
  - confidence: 0.88
