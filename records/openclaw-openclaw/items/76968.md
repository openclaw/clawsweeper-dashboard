## What This Changes

The issue asks OpenClaw to persist orphaned trailing user leaf removal so file-backed session transcripts stay aligned with the active in-memory branch.

## Best Possible Solution

Land PR #76969 or an equivalent narrow fix that persists the orphan leaf removal, keeps the non-file-backed fallback, and covers both plain text and tool-result-adjacent transcript flows.

## Solution Assessment

Yes. Persisting the removal through the file-backed SessionManager state or rewrite hook, while retaining the current branch/reset fallback, is the narrow maintainable fix for the reported invariant break.
