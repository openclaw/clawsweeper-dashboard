## Summary

Current main still has the source-level transcript persistence bug, and this issue should remain open because PR #76969 is open, unmerged, and directly targets the same orphaned-user repair path with closing syntax.

## What This Changes

Persist orphaned trailing user leaf removal when the embedded runner merges that leaf into the next prompt, keeping file-backed JSONL transcripts aligned with the active branch.

## Best Possible Solution

Land PR #76969 or an equivalent narrow fix that removes the persisted orphan leaf and rewrites the file-backed transcript under the existing session lock.

## Reproduction Assessment

Yes. Source inspection shows a file-backed session whose leaf is a user message reaches the repair block, which updates only the active branch pointer and in-memory messages without rewriting the JSONL transcript.

## Solution Assessment

Yes. Persisting the leaf removal through the file-backed SessionManager internals or rewrite helper, while keeping the current branch/reset fallback for nonstandard managers, is the narrow maintainable fix.
