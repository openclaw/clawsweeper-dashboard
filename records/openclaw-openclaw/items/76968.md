## What This Changes

Persist orphaned trailing user leaf removal in file-backed embedded-runner sessions so the active branch and JSONL transcript stay in sync.

## Best Possible Solution

Land PR #76969 or an equivalent narrow patch that persists file-backed orphan-leaf removal and adds regression coverage for both text-only and tool-use transcript flows.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a file-backed session whose current leaf is a `user` message enters the orphan repair block, moves the active leaf in memory, and leaves the stale JSONL entry untouched.

## Solution Assessment

Yes. Persisting the leaf removal through the file-backed manager while retaining the branch/reset fallback is the narrow maintainable fix direction; the linked PR should also be validated against the tool-use flow from the later comment.
