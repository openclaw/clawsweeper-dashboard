## Summary

Current main still has the reported persistence gap: the orphaned trailing user repair moves the active branch in memory but does not remove or rewrite the persisted transcript entry. This should stay open because PR #76969 is an open, unmerged fix candidate with closing syntax, and #76990 adds related transcript-consistency evidence that should be considered before landing.

## What This Changes

The issue asks for orphaned trailing user leaf repair to keep file-backed JSONL transcript state in sync when it removes the leaf from the active branch.

## Best Possible Solution

Land PR #76969 or an equivalent narrow fix that removes and rewrites the persisted orphan leaf, then add regression coverage for both text-only and tool-use transcript flows.

## Reproduction Assessment

Yes. Source inspection shows a file-backed session with a trailing user leaf reaches the repair block, which only branches/resets the in-memory leaf while leaving the JSONL entry intact.

## Solution Assessment

Yes. Persisting the leaf removal through the file-backed manager while retaining the existing branch/reset fallback is the narrow maintainable fix direction; the linked PR remains the candidate to review.
