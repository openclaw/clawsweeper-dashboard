## What This Changes

The issue reports that orphaned trailing user-message repair updates the in-memory session branch but does not persist removal to the file-backed transcript, leaving JSONL session state out of sync.

## Best Possible Solution

Land PR #76969, or an equivalent narrow fix, so file-backed sessions persist orphan-leaf removal and add regression coverage for both text-only and tool-use transcript flows.

## Reproduction Assessment

Yes, source-reproducible. A file-backed session whose current leaf is a user message reaches the current repair block, which moves the active leaf in memory without removing or rewriting the persisted JSONL entry.

## Solution Assessment

Yes. Persisting the leaf removal through the file-backed manager while retaining the existing branch/reset fallback is the narrow maintainable fix direction; the linked PR should be reviewed for the added tool-use flow before merge.
