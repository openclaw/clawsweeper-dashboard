## Summary

Keep open: the PR addresses a real stale `/codex bind` thread failure, but the latest diff still drops the persisted Codex auth profile during recovery, which can retry and rewrite the binding under different credentials.

## What This Changes

The PR adds one-time Codex bound-thread recreation after `thread not found`, regression coverage, and a changelog entry.

## Best Possible Solution

Preserve `authProfileId` through Codex thread recreation and assert the recovered binding keeps the profile before merging.

## Reproduction Assessment

Yes, by source inspection: current main sends bound turns to the saved app-server thread and reports a thrown stale-thread error as a failed reply. The PR's new test also models the `thread not found` path, though this read-only review did not execute tests.

## Solution Assessment

No. Recreate-and-retry once is the right narrow shape, but the implementation must carry `authProfileId` alongside the other persisted binding fields.
