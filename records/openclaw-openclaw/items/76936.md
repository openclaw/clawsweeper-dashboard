## Summary

Keep open: the PR targets a real `/codex bind` restart recovery bug, but the latest head still drops persisted `authProfileId` when recreating the missing app-server thread, so a retry can run under the wrong Codex credential profile.

## What This Changes

The PR adds one-time stale `/codex bind` app-server thread recreation and retry, regression coverage, and a changelog entry.

## Best Possible Solution

Merge the recovery approach only after thread recreation carries `authProfileId` through the recreated thread path and the regression test asserts the client and rewritten binding preserve it.

## Reproduction Assessment

Yes, source-reproducible. Current main sends bound turns straight to `turn/start` and reports stale-thread errors as failed replies, while the PR test demonstrates the intended retry path; I did not execute tests because this review is read-only and `pnpm` is unavailable in the checkout.

## Solution Assessment

No. Recreating and retrying once is the right narrow shape, but the implementation must also preserve the saved `authProfileId` alongside model, provider, approval, sandbox, and service-tier metadata.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve the binding auth profile during recovery:** `extensions/codex/src/conversation-binding.ts:372-381`
  - body: The recovery path recreates the app-server thread from the saved binding but never passes `binding.authProfileId` into `startCodexConversationThread`/`createThread`. Bound turns already start the shared client with that field and binding files persist it, so a Codex OAuth/profile-specific chat can retry under default credentials and then rewrite the binding without the original profile.
  - confidence: 0.89

## Security Review

Status: needs_attention

Summary: Missing-thread recovery can drop a persisted Codex auth profile and route the retry through different credentials.

Concerns:

- **[medium] Recovery can switch auth profiles:** `extensions/codex/src/conversation-binding.ts:372`
  - body: `authProfileId` selects the Codex app-server credential/profile for bound turns. Recreating the missing thread without carrying that field forward can route the retry through another profile and persist the replacement binding without the original profile.
  - confidence: 0.84
