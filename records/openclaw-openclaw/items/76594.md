## Summary

Keep open. The PR is useful source work, but the provider-hook cache is placed around a temporary reentrancy guard and can persist an empty hook list after provider loading completes.

## What This Changes

The PR adds config-scoped caches for effective tool policy resolution and provider hook plugin discovery, plus small comment/spacing changes in `src/agents/openclaw-tools.ts`.

## Best Possible Solution

Keep the performance optimization only if it caches stable runtime-load results without storing transient reentrancy-guard empties, with focused regression coverage for that path.

## Reproduction Assessment

Yes. By source inspection, a config-scoped `resolveProviderPluginsForHooks` call made while provider loading is in flight would cache the temporary `[]`, then a later identical call after loading completes would still receive no hook plugins.

## Solution Assessment

No. The provider hook cache should keep the in-flight guard outside the cached load or avoid storing guard-only empty results; otherwise it changes provider hook discovery semantics.

## Review Findings
