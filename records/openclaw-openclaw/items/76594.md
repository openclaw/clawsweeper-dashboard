## Summary

Keep open. The PR introduces a real functional regression: provider-hook discovery can cache the temporary empty result used only as an in-flight reentry guard, so later calls can keep missing provider hooks after loading completes.

## What This Changes

The PR adds config-scoped caches for effective tool policy resolution and provider hook plugin discovery, plus comment/spacing changes in `src/agents/openclaw-tools.ts`.

## Best Possible Solution

Keep only caching that stores stable runtime-load results, and add regression coverage proving nested provider-load guard empties are not reused after loading completes.

## Reproduction Assessment

Yes. By source inspection, a config-scoped `resolveProviderPluginsForHooks` call made while provider loading is in flight would cache `[]`, and a later identical call after loading completes would still receive no hook plugins.

## Solution Assessment

No. Caching provider-hook results above the in-flight guard is not the narrow maintainable fix; the cache must avoid storing guard-only empty results or sit only around stable provider-load results.
