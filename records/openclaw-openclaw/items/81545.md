## Summary

Keep open. Current main already has adjacent Task Flow, approval policy, progress streaming, liveness diagnostics, and restart recovery, but the requested opt-in Tenacity mode spans core execution policy, checkpoint/replay, auto-resume, progress state, and approval semantics; related open trackers cover checkpoint/restart pieces, but no single shipped or canonical item fully resolves this request.

## What This Changes

Request an optional Tenacity execution mode that keeps an agent running toward a goal with checkpoints, auto-resume, liveness detection, and visible progress.

## Best Possible Solution

Define a versioned opt-in autonomy policy that composes Task Flow, native checkpoint/replay, restart auto-resume, liveness diagnostics, and approval policy; keep workflow templates in skills/plugins unless a missing core API is identified.
