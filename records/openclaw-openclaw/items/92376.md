## Summary

Keep open: the fix direction is useful, but this PR changes the approval terminal-error contract without migrating existing Telegram, WhatsApp, Signal, and plugin SDK consumers, and the new terminal archive retains full approval payloads longer than needed.

## What This Changes

The PR adds bounded terminal approval-id classification, distinct expired/already-resolved gateway responses, new approval error detectors, and focused tests.

## Best Possible Solution

Keep the finer gateway taxonomy, expose one canonical stale-terminal approval classifier through core/plugin SDK, migrate Telegram/WhatsApp/Signal cleanup while preserving unknown-only plugin fallback, and store only minimal terminal archive metadata.

## Reproduction Assessment

Yes. On the PR head, a Telegram, WhatsApp, or Signal approval callback whose gateway resolve returns `APPROVAL_EXPIRED` or `APPROVAL_ALREADY_RESOLVED` bypasses the current `isApprovalNotFoundError` cleanup branches.

## Solution Assessment

No, not at the current scope. Distinct gateway reasons are the right direction, but the maintainable fix must migrate every stale-approval consumer and avoid retaining full approval payloads in the terminal archive.
