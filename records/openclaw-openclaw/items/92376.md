## What This Changes

The PR adds a bounded archive of terminated approval records, distinct expired/already-resolved/unknown gateway errors, error detectors, and focused lifecycle tests.

## Best Possible Solution

Keep the distinct terminal reasons, add one canonical stale-terminal classifier through the supported SDK surface, migrate every channel cleanup path to it, and preserve plugin fallback only for genuinely unknown IDs.

## Solution Assessment

No, not in its current scope. Distinct gateway reasons are the right direction, but the narrowest maintainable fix must migrate all consumers of the prior stale-approval contract.
