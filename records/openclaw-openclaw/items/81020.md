## Summary

Keep open: the report points at a real Windows service-supervision gap, but current main and the latest release do not contain a `P1D` creation path. Current main does preserve existing Scheduled Task trigger/settings state when updating an existing task, so a stale finite repetition setting would survive reinstall/update and needs an explicit maintainer design decision.

## What This Changes

The issue requests durable Windows Scheduled Task watchdog behavior so the OpenClaw Gateway keeps auto-restarting after crashes instead of losing repetition when a one-day duration expires.

## Best Possible Solution

Decide the Windows Scheduled Task contract, then either safely repair/recreate stale triggers or implement a dedicated durable watchdog task with tests and docs while preserving intentional user setting customization where appropriate.

## Reproduction Assessment

Unclear for the exact `P1D` creation path because current main and the latest release do not create that value. The stale-task repair gap is source-reproducible: existing tasks are updated with `/Change /TR` only, so finite trigger repetition would survive reinstall/update.

## Solution Assessment

Unclear. A blind `P1D` to `P9999D` edit is not available in current source; the maintainable fix needs a maintainer choice between trigger repair/recreation, preserving customized Scheduler settings, or adding a separate watchdog task.
