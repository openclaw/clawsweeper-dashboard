## Summary

Keep this open: the report points at a real Windows daemon-supervision gap, but current main does not contain the reported `P1D` creation path, so this should get maintainer design review rather than an auto-close or blind one-line fix.

## What This Changes

The issue asks for the Windows installed Gateway watchdog to keep auto-restarting beyond the first day instead of expiring its Scheduled Task repetition window.

## Best Possible Solution

Decide the Windows service contract, then make install/update either recreate or repair a durable Gateway watchdog task, or explicitly document that native Windows Scheduled Tasks are logon-only and require a separate watchdog.

## Reproduction Assessment

Unclear for the exact `P1D` origin: current main and v2026.5.7 do not create that value. The source does show a reproducible repair gap because existing tasks are updated with `/Change /TR` only, so stale finite repetition settings would not be corrected.

## Solution Assessment

Unclear. Changing `P1D` to `P9999D` would be narrow if that creation path exists, but current main uses `schtasks /SC ONLOGON`, so the maintainable fix likely needs an owner decision between trigger recreation and a dedicated watchdog task.
