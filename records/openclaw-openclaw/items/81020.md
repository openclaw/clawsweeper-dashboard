## Summary

Keep this open: current main still does not contain the reported `P1D` creation path, but the Windows install/update path would preserve stale Scheduled Task trigger settings instead of repairing them, so this remains a plausible Windows daemon-supervision gap rather than a cleanup close.

## What This Changes

The issue asks for Windows gateway Scheduled Task supervision to keep repeating beyond the first day, or to use a durable watchdog task.

## Best Possible Solution

Define the native Windows daemon contract, then make install/update either repair or recreate durable Scheduled Task triggers, or add a separate watchdog task with focused Windows tests and docs.

## Reproduction Assessment

Unclear for the exact `P1D` origin: current main does not create that value. Source inspection does show a reproducible repair gap because an existing task with stale finite repetition settings would be updated with `/Change /TR` only.

## Solution Assessment

Unclear. Changing `P1D` to `P9999D` is not directly applicable to the current `/Create /SC ONLOGON` path; the maintainable solution needs an owner decision between trigger normalization, task recreation, or a dedicated watchdog task.
