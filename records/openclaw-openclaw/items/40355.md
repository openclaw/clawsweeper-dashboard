## Summary

Keep open: current main still lacks the requested generic doctor/config warning, and the PR remains a plausible implementation path, but the live diff still has blocking correctness findings and missing real behavior proof before merge.

## What This Changes

The PR adds a config-validation helper, a doctor health contribution, and tests to warn when configured default or per-agent primary model refs are missing from the loaded model catalog.

## Best Possible Solution

Land or replace this with a focused diagnostic that uses runtime-equivalent alias/model resolution, distinguishes trusted catalog evidence from user-declared config rows, sanitizes terminal output, and keeps broader fallback visibility in its separate tracking path.

## Reproduction Assessment

Yes. Source inspection shows current main structurally accepts configured model refs and lacks a generic doctor/config warning for unknown default or per-agent model refs; the PR defects are also visible directly in the live diff.

## Solution Assessment

No. Advisory doctor validation is a reasonable direction, but this patch should not trust config-populated catalog rows, should resolve aliases like runtime does, and should sanitize config-derived terminal output.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Validate against catalog rows that do not come from config:** `src/flows/doctor-health-contributions.ts:407`
  - body: `runModelConfigHealth` loads `loadModelCatalog({ config: ctx.cfg })`, but that catalog appends `models.providers.*.models` from the same config. A bogus model declared under `models.providers.openai.models` can therefore appear present and skip the warning even though provider/runtime calls still fail. Use trusted provider/static rows or carry origin metadata into the check.
  - confidence: 0.88
- **[P2] Resolve aliases before checking configured primaries:** `src/config/validation.ts:1341`
  - body: `collectModelConfigWarnings` calls `resolveModelRefFromString` without `cfg` or a `ModelAliasIndex`, so a valid alias such as `fast` can be parsed as `anthropic/fast` and warned as missing. Runtime paths build aliases from `agents.defaults.models` before resolving configured refs, so doctor should use equivalent resolution.
  - confidence: 0.9
- **[P3] Sanitize model warning text before printing it:** `src/flows/doctor-health-contributions.ts:411`
  - body: The warning path and message include model identifiers derived from local config and are passed directly to `note()`. Route them through the existing terminal sanitizer or config issue formatter so control characters cannot spoof or hide doctor output.
  - confidence: 0.82
