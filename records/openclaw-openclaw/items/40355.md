## Summary

Keep open: current main has not implemented the requested generic configured-model warning, but the live PR head still has blocking false-negative, false-positive, and terminal-output issues, so it needs maintainer review and contributor follow-up rather than cleanup closure.

## What This Changes

Adds `collectModelConfigWarnings`, registers a `doctor:model-config` health contribution, and adds targeted tests for unknown default/per-agent model primaries.

## Best Possible Solution

Land a focused doctor or shared diagnostic that resolves model refs the same way runtime does, separates trusted provider/catalog evidence from self-declared config rows, sanitizes terminal output, and leaves broader fallback-user-notification behavior to its separate tracker.

## Reproduction Assessment

Yes. Source inspection shows current main lacks a generic `agents.defaults.model`/per-agent doctor warning, and the PR defects are reproducible from the live diff without running the branch.

## Solution Assessment

No. A warning-only diagnostic is the right direction, but this implementation is not the best fix because it can miss self-declared bogus models, warn on valid aliases, and render config-derived text unsafely.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Validate against trusted catalog rows:** `src/flows/doctor-health-contributions.ts:408`
  - body: The new doctor pass validates against `loadModelCatalog({ config: ctx.cfg })`, but current catalog loading appends `models.providers.*.models` from the same config being checked. The linked issue's reproduction starts by declaring the bogus model there, so this can suppress the warning this PR is meant to produce.
  - confidence: 0.88
- **[P2] Resolve aliases before warning:** `src/config/validation.ts:1344`
  - body: `resolveModelRefFromString` is called without `cfg` or an alias index. A valid `agents.defaults.models` alias such as `fast` resolves at runtime, but this helper can check it as a default-provider model and emit a false warning.
  - confidence: 0.9
- **[P3] Sanitize model warning text before printing:** `src/flows/doctor-health-contributions.ts:411`
  - body: The warning path and message include config-derived model text and are passed directly to `note()`. A model id with ANSI or control characters can spoof doctor output, so these values should go through the existing terminal sanitizer before rendering.
  - confidence: 0.82
