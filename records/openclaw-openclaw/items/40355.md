## Summary

Keep open. Current main still does not add the generic doctor warning this PR targets, but the PR is not merge-ready because it can miss the linked misconfiguration, can warn on valid aliases, and prints config-derived warning text unsanitized.

## What This Changes

The PR adds a config-validation helper, a doctor health contribution, and tests to warn when default or per-agent model primaries are missing from the loaded model catalog.

## Best Possible Solution

Land a focused doctor/config diagnostic that uses runtime-equivalent model-ref resolution, distinguishes provider/runtime/static catalog evidence from self-declared config rows, sanitizes terminal output, covers aliases plus the linked model-config shapes, and records the user-facing fix in the changelog.

## Reproduction Assessment

Yes. Source inspection reproduces the PR defects: a bogus model declared under models.providers.*.models is appended to the same catalog being checked, and alias-based configs are resolved differently unless the runtime alias index is used.

## Solution Assessment

No. A warning-only doctor diagnostic is the right direction, but this implementation is not the best fix until it matches runtime resolution semantics and avoids trusting catalog rows supplied by the config value under review.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Validate against non-config catalog evidence:** `src/flows/doctor-health-contributions.ts:407`
  - body: `loadModelCatalog({ config: ctx.cfg })` appends `models.providers.*.models` from the same config being validated. In the linked reproduction, a bogus model listed there is treated as present, so doctor suppresses the warning even though provider/runtime calls can still fail.
  - confidence: 0.88
- **[P2] Resolve aliases before warning:** `src/config/validation.ts:1341`
  - body: `collectModelConfigWarnings` calls `resolveModelRefFromString` without the config alias index used by runtime resolution. A valid alias such as `fast` can be parsed as a default-provider model and warned as missing even though OpenClaw can run it.
  - confidence: 0.9
- **[P3] Sanitize model warnings before note output:** `src/flows/doctor-health-contributions.ts:411`
  - body: The warning path and message include model refs derived from local config and are passed directly to `note()`. A model id with ANSI or control characters can spoof doctor output; route this text through the existing terminal sanitizer before display.
  - confidence: 0.82
- **[P3] Add the changelog entry:** `src/flows/doctor-health-contributions.ts:681-685`
  - body: This adds a user-visible `openclaw doctor` warning for config mistakes, but the PR does not update `CHANGELOG.md`. OpenClaw policy requires user-facing fixes to add a changelog entry before merge.
  - confidence: 0.86
