## Summary

Keep this PR open: current main still lacks a generic doctor check for default/per-agent model config, but the proposed patch has blocking correctness and security issues before it can land.

## What This Changes

The PR adds a config-validation helper, a doctor health contribution, and tests to warn when configured default or per-agent model primaries are missing from the loaded model catalog.

## Best Possible Solution

Land a focused diagnostic that matches runtime model-ref resolution, distinguishes provider/runtime catalog evidence from self-declared config rows, sanitizes terminal output, and records the user-facing doctor warning in the changelog.

## Reproduction Assessment

Yes. Source inspection shows the false-negative path through config-inclusive catalog rows and the false-positive alias path through missing alias-index resolution without needing to run the branch.

## Solution Assessment

No. A non-blocking doctor warning is the right direction, but this implementation is not yet the best fix because it does not use runtime-equivalent resolution or a trustworthy catalog source for this diagnostic.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Validate against catalog rows that do not come from config:** `src/flows/doctor-health-contributions.ts:407`
  - body: `loadModelCatalog({ config: ctx.cfg })` appends `models.providers.*.models` from the same config being validated. In the linked reproduction, adding a bogus model under `models.providers.openai.models` makes it appear present here, so doctor suppresses the warning even though provider/runtime calls can still fail.
  - confidence: 0.88
- **[P2] Resolve aliases before checking configured primaries:** `src/config/validation.ts:1341`
  - body: `collectModelConfigWarnings` resolves the raw string without the config alias index used by runtime model resolution. A valid alias from `agents.defaults.models` can therefore be warned as missing even though OpenClaw can run it.
  - confidence: 0.9
- **[P3] Sanitize model warning text before printing it:** `src/flows/doctor-health-contributions.ts:411`
  - body: The warning path and message include model refs from local config and are passed directly to `note()`. A model id with ANSI/control characters can spoof or obscure doctor output, so route this text through the existing terminal sanitizer before display.
  - confidence: 0.82
- **[P3] Add the required changelog entry:** `src/flows/doctor-health-contributions.ts:681-685`
  - body: This adds a user-facing `openclaw doctor` warning for config mistakes, but the PR does not update `CHANGELOG.md`. OpenClaw policy requires a changelog entry for user-visible fixes before merge.
  - confidence: 0.86
