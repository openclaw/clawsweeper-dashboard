## Summary

Keep open: current main still lacks a generic configured-model warning, but this PR is not ready because the live diff still has catalog-authority, alias-resolution, terminal-output hygiene, and real-behavior-proof blockers.

## What This Changes

The PR adds `collectModelConfigWarnings`, registers a doctor health contribution, and adds tests to warn when configured default or per-agent primary models are absent from the loaded model catalog.

## Best Possible Solution

Land or replace this with a focused doctor/config diagnostic that uses runtime-equivalent model resolution, trusted catalog evidence, sanitized output, and regression coverage for the linked misconfiguration path.

## Reproduction Assessment

Yes, by source inspection. Current main lacks a generic configured-model doctor/config warning, and the PR defects are visible in the live diff at `dc0edbcb29eaa42faec61b0f00d7852ba77a56ab`.

## Solution Assessment

No. Advisory doctor validation is a reasonable direction, but this implementation trusts config-derived catalog rows, diverges from runtime alias resolution, and renders unsanitized config-derived text.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Validate against trusted catalog rows:** `src/flows/doctor-health-contributions.ts:408`
  - body: `loadModelCatalog({ config: ctx.cfg })` appends models from the same user config being validated. In https://github.com/openclaw/openclaw/issues/39811, the bogus model is first declared under `models.providers.openai.models`, so this diagnostic can treat it as present and skip the warning even though runtime/provider calls can still fail.
  - confidence: 0.88
- **[P2] Resolve aliases before checking configured primaries:** `src/config/validation.ts:1344`
  - body: `collectModelConfigWarnings` calls `resolveModelRefFromString` without `cfg` or an alias index, so a valid configured alias such as `fast` is parsed as a default-provider model and warned as missing. Runtime model selection builds and applies aliases from config before resolving model refs.
  - confidence: 0.9
- **[P3] Sanitize model warning text before printing it:** `src/flows/doctor-health-contributions.ts:411`
  - body: The warning path and message include model identifiers derived from local config and are passed straight to `note()`. Route the path/message through the existing terminal sanitizer before display so control characters or ANSI escapes cannot spoof doctor output.
  - confidence: 0.82
