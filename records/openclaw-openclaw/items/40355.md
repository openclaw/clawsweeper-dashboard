## Summary

Keep open: current main still does not provide a generic doctor warning for default/per-agent unknown model refs, and this PR is not merge-ready because its diagnostic can miss the reported misconfiguration, produce alias false positives, and render config-derived text without sanitization.

## What This Changes

The PR adds `collectModelConfigWarnings`, wires it into a new `doctor:model-config` health contribution, and adds tests for default/per-agent unknown model warnings.

## Best Possible Solution

Land a focused diagnostic that uses runtime-equivalent model ref resolution, separates provider/manifest evidence from self-declared config rows, sanitizes terminal output, and keeps #39811 as the canonical issue until the implementation merges.

## Reproduction Assessment

Yes. Source inspection of the PR diff shows the false-negative path through config-inclusive catalog rows and the alias false-positive path through missing alias-index resolution without running the branch.

## Solution Assessment

No. A non-blocking doctor warning is the right direction, but this implementation is not yet the best fix because it does not match runtime model resolution or distinguish trustworthy catalog rows from config-declared rows.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Validate against a catalog that excludes config-only rows:** `src/flows/doctor-health-contributions.ts:405`
  - body: `loadModelCatalog({ config: ctx.cfg })` appends `models.providers.*.models` from the same config being checked. In the linked reproduction, a bogus model declared there is treated as present, so doctor suppresses the warning even though provider/runtime calls can still fail.
  - confidence: 0.88
- **[P2] Resolve aliases before checking configured primaries:** `src/config/validation.ts:1341`
  - body: `collectModelConfigWarnings` calls `resolveModelRefFromString` without the alias index used by runtime/model-picker paths. A valid configured alias can therefore be reported as missing even though OpenClaw can run it.
  - confidence: 0.9
- **[P3] Sanitize model warnings before rendering them:** `src/flows/doctor-health-contributions.ts:408`
  - body: The warning path and message include model refs from local config and are passed directly to `note()`. Route them through the existing terminal sanitizer so control characters cannot spoof or obscure doctor output.
  - confidence: 0.82
- **[P3] Add the required changelog entry:** `src/flows/doctor-health-contributions.ts:681-685`
  - body: This adds a user-facing `openclaw doctor` warning for config mistakes, but the PR does not update `CHANGELOG.md`. OpenClaw policy requires a changelog entry for user-visible fixes before merge.
  - confidence: 0.86
