## Summary

Keep this PR open: current main still lacks the proposed generic model-config doctor warning, but the live PR diff still has correctness blockers around catalog authority and alias resolution, plus missing contributor real-behavior proof.

## What This Changes

The PR adds `collectModelConfigWarnings`, a new doctor health contribution, and targeted tests to warn when default or per-agent primary model refs are absent from the loaded model catalog.

## Best Possible Solution

Land or replace this with a focused shared diagnostic that uses runtime-equivalent model resolution, distinguishes trusted catalog evidence from user-declared config rows, sanitizes terminal output, and includes real doctor proof.

## Reproduction Assessment

Yes, by source inspection: current main accepts configured model strings structurally and lacks a generic doctor/config diagnostic for unknown default or per-agent model refs. The PR defects are also visible in the live diff against current catalog and alias-resolution contracts.

## Solution Assessment

No. Advisory doctor validation is a reasonable direction, but this patch should use runtime-equivalent alias/model resolution, avoid treating user-declared catalog rows as provider proof, sanitize output, and include real behavior proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Validate against non-config catalog rows:** `src/flows/doctor-health-contributions.ts:405`
  - body: `runModelConfigHealth` loads `loadModelCatalog({ config: ctx.cfg })`, but that catalog appends `models.providers.*.models` from the same config. In the linked reproduction, the bogus model is first added under `models.providers.openai.models`, so it can appear present and skip the warning even though runtime/provider calls still fail.
  - confidence: 0.88
- **[P2] Resolve aliases before checking primaries:** `src/config/validation.ts:1341`
  - body: `collectModelConfigWarnings` calls `resolveModelRefFromString` without `cfg` or a `ModelAliasIndex`, so a valid alias like `fast` can be parsed as `anthropic/fast` and warned as missing. Runtime model selection builds aliases from `agents.defaults.models` before resolving configured refs.
  - confidence: 0.9
- **[P3] Sanitize model warning text before printing:** `src/flows/doctor-health-contributions.ts:408`
  - body: The warning output includes config-derived model identifiers and is passed directly to `note()`. Route the path and message through the existing terminal sanitizer so ANSI/control characters cannot spoof or hide doctor output.
  - confidence: 0.82
