## Summary

Keep open: current main still lacks a generic doctor/config warning for invalid default and per-agent model refs, and this PR is the active closing implementation path, but the live patch still has blocking catalog and alias-resolution defects plus missing real behavior proof.

## What This Changes

The PR adds `collectModelConfigWarnings`, wires it into doctor health, and adds targeted tests for default and per-agent primary model catalog warnings.

## Best Possible Solution

Land or replace this with a focused diagnostic that uses runtime-equivalent model resolution, distinguishes trusted catalog evidence from user-declared config rows, sanitizes terminal output, and keeps broader fallback notification behavior on its separate tracker.

## Reproduction Assessment

Yes. Source inspection on current main shows arbitrary configured default/per-agent model refs still lack a generic doctor/config catalog warning, and the PR defects are visible directly in the live diff.

## Solution Assessment

No. Advisory doctor validation is a reasonable direction, but this patch should use trusted catalog semantics, runtime-equivalent alias resolution, and sanitized terminal rendering before merge.

Overall correctness: patch is incorrect

  - confidence: 0.9

  - body: The warning path and message include model identifiers from local config and are passed straight to `note()`. Route both through the existing terminal sanitizer before display so control characters cannot spoof doctor diagnostics.
  - confidence: 0.82
