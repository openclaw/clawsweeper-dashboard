## Summary

Keep this PR open: the requested diagnostic is still useful and not implemented on current main, but the patch has actionable correctness issues and lacks required real behavior proof before merge.

## What This Changes

The branch adds a config validation helper, doctor health contribution, and tests to warn when configured default or per-agent primary model refs are absent from the loaded catalog.

## Best Possible Solution

Land or replace this with a focused doctor/config diagnostic that uses runtime-equivalent model resolution, distinguishes user-configured catalog rows from trusted provider/static evidence, sanitizes terminal output, and keeps the linked issue open until the fix merges.

## Reproduction Assessment

Yes, by source inspection. Current main lacks a generic configured-model doctor warning, and the PR’s false-negative, false-positive, and unsanitized-output paths are visible in the live head diff.

## Solution Assessment

No. Advisory doctor validation is a reasonable direction, but this implementation should match runtime alias/model resolution, avoid trusting config-authored catalog rows, and sanitize config-derived terminal output.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Validate against catalog rows that do not come from config:** `src/flows/doctor-health-contributions.ts:407`
  - body: `loadModelCatalog({ config: ctx.cfg })` appends `models.providers.*.models` from the same config being checked. A bogus model declared there will be treated as present, so the linked misconfiguration path can still skip the new warning even though runtime/provider calls may fail.
  - confidence: 0.88
- **[P2] Resolve aliases before checking configured primaries:** `src/config/validation.ts:1342`
  - body: `collectModelConfigWarnings` calls `resolveModelRefFromString` without `cfg` or an alias index. Valid aliases from `agents.defaults.models[*].alias` can be parsed as default-provider model IDs and warned as missing even though runtime accepts them.
  - confidence: 0.9
- **[P3] Sanitize model warning text before printing it:** `src/flows/doctor-health-contributions.ts:411`
  - body: The warning message includes model identifiers derived from local config and passes them directly to `note()`. Route the path and message through the existing terminal sanitizer so control characters or ANSI escapes cannot spoof doctor output.
  - confidence: 0.82
