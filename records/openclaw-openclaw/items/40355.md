## Summary

Keep open: current main still lacks the generic configured-model doctor/config warning this PR is trying to add, but the live PR diff still has correctness and terminal-output hygiene blockers, and the contributor has not supplied real after-fix doctor output.

## What This Changes

The PR adds a config-validation helper, doctor health contribution, and tests to warn when default or per-agent primary model refs are absent from the loaded model catalog.

## Best Possible Solution

Land or replace this with a focused diagnostic that uses runtime-equivalent model resolution, distinguishes trusted catalog evidence from user-declared config rows, sanitizes terminal output, and leaves broader fallback-notification behavior to its separate tracker.

## Reproduction Assessment

Yes. Source inspection on current main shows arbitrary configured default/per-agent model refs still lack a generic doctor/config catalog warning, and the PR defects are visible directly in the live diff.

## Solution Assessment

No. Advisory doctor validation is a reasonable direction, but this patch should use trusted catalog semantics, runtime-equivalent alias resolution, and sanitized terminal rendering before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Validate against trusted catalog rows:** `src/flows/doctor-health-contributions.ts:407`
  - body: `runModelConfigHealth` passes `loadModelCatalog({ config: ctx.cfg })` into the warning helper, but that catalog appends `models.providers.*.models` from the same config. In the linked canonical issue reproduction, a bogus model declared under `models.providers.openai.models` would therefore appear present and skip the warning even though runtime/provider calls can still fail.
  - confidence: 0.88
- **[P2] Resolve aliases before warning on model primaries:** `src/config/validation.ts:1341`
  - body: `collectModelConfigWarnings` calls `resolveModelRefFromString` without config or a `ModelAliasIndex`, so a valid configured alias such as `fast` can be parsed as `anthropic/fast` and warned as missing. Runtime/model-picker paths build aliases from `agents.defaults.models` before resolving configured refs, so doctor would produce false positives for supported configs.
  - confidence: 0.9
- **[P3] Sanitize model warning text before rendering:** `src/flows/doctor-health-contributions.ts:411`
  - body: The warning path and message include model identifiers derived from local config and are passed directly to `note()`. Route both through the existing terminal sanitizer or config issue formatter so control characters cannot spoof or hide doctor output.
  - confidence: 0.82
