## Summary

Keep open: current main still lacks a generic warning for invalid configured agent model refs, and this focused PR remains useful, but the current head is not merge-ready because it can miss the linked bad-config case, warn on valid alias configs, omit fallbacks, print unsanitized config text, and lacks real doctor output proof.

## What This Changes

The PR adds a `collectModelConfigWarnings` helper, a doctor health contribution, and targeted tests to warn when configured default or per-agent primary model refs are absent from the loaded model catalog.

## Best Possible Solution

Land or replace this with a focused doctor/config diagnostic that uses runtime-equivalent model resolution, distinguishes trusted catalog evidence from user-declared rows, covers primary and fallback refs, sanitizes terminal output, and is proven with redacted doctor output.

## Reproduction Assessment

Yes, source-reproducible. Current main accepts agent model refs structurally and only has catalog-backed doctor validation for `hooks.gmail.model`, while the linked issue shows the invalid-model failure mode remains active.

## Solution Assessment

No, not as written. Advisory validation is the right direction, but this implementation needs runtime-equivalent resolution, trusted catalog semantics, fallback coverage, sanitizer handling, and real doctor-output proof.
