## Summary

Keep open: current main still lacks a generic default/per-agent model-config warning, and this PR remains the active implementation path but has blocking correctness/security findings plus missing real behavior proof.

## What This Changes

The PR adds a config-validation helper, a doctor health contribution, and tests to warn when default or per-agent primary model refs are absent from the loaded model catalog.

## Best Possible Solution

Land or replace this with a focused doctor/config diagnostic that uses runtime-equivalent model resolution, separates trusted catalog evidence from user-declared rows, sanitizes terminal output, and leaves broader fallback-user notification to its related tracker.

## Reproduction Assessment

Yes, by source inspection. Current main structurally lacks a generic doctor/config warning for unknown default and per-agent model refs, and the PR defects are visible directly in the live diff.

## Solution Assessment

No. Advisory doctor validation is a reasonable direction, but this patch trusts config-injected catalog rows, diverges from runtime alias resolution, and renders unsanitized text; the safer path is a smaller diagnostic using trusted catalog semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Use trusted catalog evidence for warnings:** `src/flows/doctor-health-contributions.ts:408`
  - body: `runModelConfigHealth` loads `loadModelCatalog({ config: ctx.cfg })`, but that catalog appends `models.providers.*.models` from the same config. In https://github.com/openclaw/openclaw/issues/39811, the bogus model is first declared under `models.providers.openai.models`, so this check can treat it as present and skip the warning while runtime/provider calls still fail.
  - confidence: 0.88
- **[P2] Resolve aliases before checking primaries:** `src/config/validation.ts:1344`
  - body: `collectModelConfigWarnings` calls `resolveModelRefFromString` without `cfg` or an alias index, so a valid configured alias such as `fast` is parsed as a default-provider model and warned as missing. Runtime model selection builds and applies aliases from `agents.defaults.models`, so doctor would produce false positives for supported configs.
  - confidence: 0.9
- **[P3] Sanitize model warnings before rendering:** `src/flows/doctor-health-contributions.ts:411`
  - body: The warning path and message include model identifiers derived from local config and are passed directly to `note()`. Route the path/message through the existing terminal sanitizer before display so control characters or ANSI escapes cannot spoof doctor output.
  - confidence: 0.82
