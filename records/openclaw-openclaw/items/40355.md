## Summary

Keep open: current main still lacks the requested generic doctor warning, but the PR is not merge-ready because the proposed check can miss the linked misconfiguration case and can warn on valid alias-based configs.

## What This Changes

The PR adds a catalog-aware config-validation helper, wires a new `openclaw doctor` health contribution for default/per-agent model primaries, and adds tests for unknown model warnings.

## Best Possible Solution

Land a focused doctor/config diagnostic that uses runtime-equivalent model-ref resolution, distinguishes provider/static discovery from self-declared config rows, sanitizes note output, and records the user-facing warning in the changelog.

## Reproduction Assessment

Yes. Source inspection shows the false-negative path through config-inclusive catalog rows and the false-positive alias path through missing alias-index resolution without needing to run the branch.

## Solution Assessment

No. A non-blocking doctor warning is the right direction, but this implementation is not yet the best fix because it does not match runtime resolution or a trustworthy catalog source for this diagnostic.

## Review Findings

Full review comments:

- **[P2] Validate against catalog rows outside local config:** `src/flows/doctor-health-contributions.ts:407`
  - body: `loadModelCatalog({ config: ctx.cfg })` appends `models.providers.*.models` from the same config. In the linked #39811 reproduction, the bogus model is first added under `models.providers.openai.models`, so doctor will see it as present and suppress the warning even though provider/runtime calls can still fail.
  - confidence: 0.88
- **[P2] Resolve aliases before warning about configured primaries:** `src/config/validation.ts:1341`
  - body: `collectModelConfigWarnings` resolves raw model strings without the alias index built from `agents.defaults.models`. A valid alias such as `fast` can therefore be warned as `anthropic/fast` even though runtime model selection supports it.
  - confidence: 0.9
- **[P3] Sanitize model warning text before printing:** `src/flows/doctor-health-contributions.ts:411`
  - body: The warning path and message include config-derived model refs and are passed directly to `note()`. A model id containing ANSI or control characters can spoof doctor output, so route this text through the existing terminal sanitizer before display.
  - confidence: 0.82
- **[P3] Add the required changelog entry:** `src/flows/doctor-health-contributions.ts:681-685`
  - body: This adds a user-facing `openclaw doctor` warning for config mistakes, but the PR does not update `CHANGELOG.md`. OpenClaw policy requires user-facing fixes to be recorded before merge.
