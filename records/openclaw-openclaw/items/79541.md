## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Force the delegate past stale ACPX records:** `extensions/acpx/src/runtime.ts:706-707`
  - body: Returning false here only enables OpenClaw's launch lease wrapper. The upstream acpx runtime still loads the same existing record and returns it when cwd, command, and resumeSessionId match, because its reuse predicate ignores PID liveness. Mark or hide the stale record before calling the delegate, otherwise a dead-PID record can still be reused instead of starting a real load/fresh session.
  - confidence: 0.9
- **[P1] Reject missing PID records during resume:** `extensions/acpx/src/runtime.ts:706-707`
  - body: The guard only rejects a record when `readRecordAgentPid` returns a PID and that PID is dead. The PR's own live repro shows `agent_pid: null`, so that case still returns true and disables the fresh launch path; treat missing PID as stale for resume/warm-restore unless a live lease proves reuse is safe.
  - confidence: 0.9
- **[P1] Limit warm restore to persistent sessions:** `src/acp/control-plane/manager.core.ts:262-265`
  - body: This resolved-identity branch does not check `session.acp.mode`, so resolved one-shot records can be included in the startup scan. The manager only passes `resumeSessionId` for persistent sessions, so a one-shot row can be re-entered as fresh work; keep warm restore limited to persistent ACP sessions.
  - confidence: 0.88
