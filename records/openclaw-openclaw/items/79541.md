Full review comments:

- **[P1] Force stale ACPX records past delegate reuse:** `extensions/acpx/src/runtime.ts:706-707`
  - body: Returning `false` from the OpenClaw launch-lease predicate is not enough here: `acpx@0.10.0` still returns the existing persistent record when cwd, command, and resume id match, before starting a client. Mark, hide, or reset the stale record before calling the delegate so the patched path actually launches or loads instead of reusing the dead record.
  - confidence: 0.9
- **[P1] Treat missing PID as stale during resume:** `extensions/acpx/src/runtime.ts:706-707`
  - body: The live repro in the PR body shows `agent_pid: null`, but this guard only rejects records when a parsed PID exists and is dead. For warm restore, a missing PID should not disable the fresh-launch path unless another live lease proves the session is reusable.
  - confidence: 0.9
- **[P1] Limit warm restore to persistent sessions:** `src/acp/control-plane/manager.core.ts:262-265`
  - body: This resolved-identity branch does not check `session.acp.mode`, so resolved one-shot rows can enter the startup scan. Current main explicitly avoids passing persisted resume ids for one-shot sessions after restart, so the startup restore predicate should preserve that invariant.
  - confidence: 0.88
