## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Limit warm restore to persistent ACP sessions:** `src/acp/control-plane/manager.core.ts:260-263`
  - body: This new `isResumableResolved` branch accepts any resolved identity with a stable id and non-error state, including one-shot records. `ensureManagerRuntimeHandle` only threads `resumeSessionId` for persistent sessions, so a resolved one-shot row that survives in the store can enter startup reconciliation and launch fresh work instead of being skipped; keep the warm-restore path limited to `mode === "persistent"` and preserve the one-shot skip behavior.
  - confidence: 0.9
- **[P1] Reject reuse when the persisted PID is missing:** `extensions/acpx/src/runtime.ts:705-709`
  - body: The liveness guard only rejects a record when `readRecordAgentPid` returns a PID and that PID is dead. ACPX records make `pid` optional, and the PR's live repro describes a post-restart record with no usable PID, so this path would still return true, disable the launch lease, and never issue the fresh `session/load`/spawn the fix relies on. Treat a missing PID as non-reusable for resume/warm-restore records or prove another live lease identity first.
  - confidence: 0.88
