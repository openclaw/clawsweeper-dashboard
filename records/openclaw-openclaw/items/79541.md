Full review comments:

- **[P1] Force stale records past the ACPX delegate:** `extensions/acpx/src/runtime.ts:706-707`
  - body: Returning `false` here only enables OpenClaw's launch lease wrapper; the real `acpx@0.10.0` delegate immediately loads the same session record and returns it when cwd, command, and `resumeSessionId` match. Mark, hide, close, or otherwise exclude the stale record before calling the delegate, or a dead-PID record can still be reused without a fresh launch.
  - confidence: 0.92
- **[P1] Treat missing PIDs as stale during resume:** `extensions/acpx/src/runtime.ts:706-707`
  - body: The guard only rejects records when a parsed PID exists and is dead. The PR's own live repro shows `agent_pid: null`, so that case still returns true and disables the fresh-launch path; missing PID should be stale for warm-restore unless another live-process signal proves reuse is safe.
  - confidence: 0.93
- **[P1] Keep startup warm restore persistent-only:** `src/acp/control-plane/manager.core.ts:262-265`
  - body: This resolved-identity branch does not check `session.acp.mode`, so resolved one-shot rows can enter the startup scan and call `ensureSession` with a persisted resume id. Current manager coverage pins that one-shot sessions after restart must not reuse persisted ACP identities, so the startup path should preserve that invariant.
  - confidence: 0.89
