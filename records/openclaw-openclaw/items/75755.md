## Summary

Keep open: the PR is aimed at the right user-visible privacy bug, but the latest diff still misses generated runtime-context shapes, so internal exec, async, and metadata scaffolding can remain visible.

## What This Changes

The PR adds modern runtime-context wrapper stripping, sanitizer regression tests, and a changelog entry for the linked runtime-context leak.

## Best Possible Solution

Make the stripper generator-aligned: consume runtime sections line-by-line or from shared sentinels, cover exec-state, async, location, and structured metadata variants, and keep the JSON/fenced false-positive protections.

## Reproduction Assessment

Yes. Build a modern wrapper using current `buildExecOverridePromptHint` output or a missing-output `buildExecEventPrompt` body, then pass it through the PR sanitizer; source inspection shows the parser stops before consuming those generated sections.

## Solution Assessment

No. Section-aware stripping is the right direction, but this implementation should align with actual generator output rather than blank-line fixture paragraphs and a partial heading allowlist.
