## Summary

Keep open. The PR is a plausible narrow sanitizer fix for a real runtime-context leak, but the current diff still misses generated exec-state, failed/missing-output async completion, and several metadata wrapper shapes, and the external PR lacks after-fix real behavior proof.

## What This Changes

The PR replaces the modern runtime-context preface stripper with a section-aware sanitizer and adds regression tests plus a changelog entry for echoed runtime-context wrappers.

## Best Possible Solution

Align the sanitizer with the actual generator contracts, preferably through shared sentinels or generator-backed fixtures, and require redacted real output proof before merge.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to wrap current outputs from `buildExecOverridePromptHint`, `buildExecEventPrompt`, and `buildInboundUserContextPrefix` with the runtime-context header and pass them through the PR sanitizer; several current generated shapes remain unmatched.

## Solution Assessment

No. Section-aware stripping is the right direction, but this implementation should be generator-aligned before merge so current OpenClaw scaffolding variants are not preserved in outbound replies.
