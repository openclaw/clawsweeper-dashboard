## Best Possible Solution

Finish the shared sanitizer so it consumes the exact sections emitted by the current generators while preserving standalone JSON and fenced visible replies.

Yes. Feed `stripInternalRuntimeContext` the modern header and notice followed by the exact single-newline `buildExecOverridePromptHint` output, or a `buildInboundUserContextPrefix` block such as `Thread starter`, then visible reply text; the PR parser stops before the real reply.

No. A section-aware sanitizer is the right direction, but this patch is still fixture-shaped; the safer solution is to parse generated sections line-by-line or share sentinel definitions with generator-backed regression tests.
