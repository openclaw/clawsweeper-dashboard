## Summary

Keep open. The PR is aimed at the right sanitizer path and has sufficient terminal proof for one Telegram-shaped wrapper, but the branch still misses current generated async failure/missing-output, exec-state, and inbound metadata shapes, so internal runtime context can still leak to user-visible output.

## What This Changes

The PR replaces modern runtime-context preface stripping with section-aware wrapper stripping, adds sanitizer regression coverage, and adds a changelog fix entry.

## Best Possible Solution

Align the wrapper stripper with the actual generator contracts, ideally through shared labels or generator-backed fixtures, while preserving legitimate JSON and fenced visible replies.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to wrap the current outputs from `buildExecEventPrompt`, `buildExecOverridePromptHint`, and `buildInboundUserContextPrefix` in the modern runtime-context header and pass them through the branch sanitizer; several generated sections remain visible.

## Solution Assessment

No. Section-aware stripping is the right direction, but this implementation is not generator-aligned enough to be the narrow maintainable fix yet.
