## Summary

Keep open. The PR is useful and points at the right sanitizer boundary, but it still misses current generated wrapper shapes for exec-state sections, metadata labels, and failed or missing-output async completion prompts.

## What This Changes

The PR expands `stripInternalRuntimeContext` to strip modern runtime-context wrapper blocks, adds sanitizer regression tests, and adds a changelog fix entry for #75726.

## Best Possible Solution

Align the stripper and regression tests with the actual generators, ideally with builder-shaped fixtures or shared sentinels for exec state, inbound metadata labels, and async completion variants.

## Reproduction Assessment

Yes. Build copied wrapper strings from `buildExecOverridePromptHint`, `buildInboundUserContextPrefix` with Location or structured metadata, and `buildExecEventPrompt` failed or missing-output variants; source inspection shows the PR parser stops before consuming those sections.

## Solution Assessment

No. Section-aware stripping is the right approach, but this implementation is not yet generator-aligned and should be repaired before merge.
