## Summary

Keep open. The PR moves in the right direction, but the current branch still leaves generated exec-state, failed async-completion, and several inbound metadata wrapper shapes able to survive into user-facing text.

## What This Changes

The PR replaces the modern runtime-context preface stripper with a section-aware sanitizer, adds regression tests, and adds a changelog fix entry for #75726.

## Best Possible Solution

Align the sanitizer and regression tests with the actual runtime-context builders, preferably by reusing shared sentinel lists or builder-backed fixtures for inbound metadata, exec state, and async success/failure/missing-output variants.

Yes. Source inspection gives a high-confidence path by wrapping output from buildExecOverridePromptHint, buildExecEventPrompt, and buildInboundUserContextPrefix; the PR parser leaves several current generated shapes unmatched.

## Solution Assessment

No. Section-aware stripping is the right approach, but this branch is not yet the best fix because it duplicates incomplete sentinel knowledge instead of matching the current generators.
