## Summary

Keep open: the PR is aimed at a real sanitizer bug and the direction is useful, but the current branch still misses generated exec-state, failed/missing async completion, and several current inbound-metadata wrapper shapes, so it should not merge as-is.

## What This Changes

The PR replaces modern runtime-context preface stripping with section-aware wrapper stripping, adds sanitizer regression coverage, and adds a changelog fix entry.

## Best Possible Solution

Align the sanitizer with the actual generator contracts, preferably with generator-backed fixtures or shared labels, so every current runtime-context section is consumed while visible JSON and fenced replies remain intact.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to wrap the current outputs from buildExecOverridePromptHint, buildExecEventPrompt, and buildInboundUserContextPrefix in the modern runtime-context header and pass them through the branch sanitizer; several generated sections still remain visible.

## Solution Assessment

No. Section-aware stripping is the right direction, but this implementation is not yet generator-aligned enough to be the narrow maintainable fix.
