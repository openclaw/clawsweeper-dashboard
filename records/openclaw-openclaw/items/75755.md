## Summary

Keep open: the PR targets the right leak, but latest head still does not align with current generator output for exec-state, metadata, and async-event variants, so internal runtime context can still surface to users.

## What This Changes

The PR adds modern runtime-context wrapper stripping, sanitizer regression tests, and a changelog fix entry for the linked runtime-context leak.

## Best Possible Solution

Align the sanitizer and regression tests with the actual generator outputs, preferably by sharing section sentinels or exhaustively testing generated fixtures while preserving JSON and fenced visible replies.

## Reproduction Assessment

Yes. Compose the modern runtime-context header and notice with current `buildExecOverridePromptHint`, `buildExecEventPrompt` failure/no-output variants, or location/structured metadata from `buildInboundUserContextPrefix`; source inspection shows the PR parser stops before consuming those generated sections.

## Solution Assessment

No. Section-aware stripping is the right direction, but this implementation is still a partial hand-match; the safer fix is to consume the generator-owned section shapes or share a narrow sentinel table with those generators.
