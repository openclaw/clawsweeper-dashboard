## Summary

Keep open. The PR is aimed at a real sanitizer leak and includes useful coverage, but the latest head still misses generated exec-state and inbound-metadata wrapper shapes that can reach user-facing channel output.

## What This Changes

The branch expands runtime-context stripping, adds sanitizer regression tests, and adds a changelog fix entry for echoed modern runtime-context wrappers.

## Best Possible Solution

Finish the shared sanitizer against the actual generator outputs, preferably by deriving regression fixtures from `buildExecOverridePromptHint`, `buildInboundMetaSystemPrompt`, and `buildInboundUserContextPrefix` while preserving standalone JSON and fenced visible replies.

## Reproduction Assessment

Yes. Use the modern runtime header and notice followed by the exact single-newline `buildExecOverridePromptHint` output and then visible reply text; the PR parser treats the heading plus body as one paragraph and fails to consume it.

## Solution Assessment

No. A section-aware sanitizer is the right direction, but this implementation remains aligned to hand-shaped fixtures rather than the current generators; the safer fix is to consume generated sections line-by-line or centralize the generated sentinels.
