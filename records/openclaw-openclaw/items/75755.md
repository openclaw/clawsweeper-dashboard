## Summary

Keep open. The PR is aimed at the right privacy-sensitive sanitizer, but latest head still misses actual generator-backed exec-state and metadata shapes, so echoed runtime scaffolding can still reach user-visible replies.

## What This Changes

The PR expands `stripInternalRuntimeContext` to strip echoed modern runtime-context, metadata, and exec-state wrappers, adds sanitizer regression tests, and records a changelog fix for #75726.

## Best Possible Solution

Update the shared sanitizer to consume the exact exec-state and metadata sections emitted by current generators while preserving standalone JSON and fenced user-visible replies.

## Reproduction Assessment

Yes. Feed the PR sanitizer the modern header and notice followed by current `buildExecOverridePromptHint` output joined with single newlines, or a generated location/structured metadata block from `buildInboundUserContextPrefix`, then visible reply text; the parser stops before the real reply.

## Solution Assessment

No. Section-aware stripping is the right direction, but this patch is still fixture-shaped; the safer repair is line-oriented or generator-aligned parsing with regression tests built from the current generator outputs.
