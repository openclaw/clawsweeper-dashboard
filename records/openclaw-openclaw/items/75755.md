## Summary

Keep open. Current main still has the reported runtime-context leak class, but this PR is not yet safe to merge because the sanitizer still misses generated exec-state, failed/missing-output async, and several inbound metadata wrapper shapes, and the external contributor has not supplied real behavior proof.

## What This Changes

The PR replaces modern runtime-context preface stripping with section-aware wrapper stripping, adds sanitizer regression tests, and adds a changelog fix entry.

## Best Possible Solution

Align the sanitizer with the actual generator contracts, preferably through shared labels or generator-backed fixtures, so all current runtime-context sections are stripped while JSON and fenced visible replies are preserved.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to wrap outputs from buildExecOverridePromptHint, buildExecEventPrompt, and buildInboundUserContextPrefix in the modern runtime-context header and pass them through the PR sanitizer; several generated sections remain visible.

## Solution Assessment

No. Section-aware stripping is the right direction, but this implementation is still not generator-aligned enough to be the narrow maintainable fix.

## Review Findings
