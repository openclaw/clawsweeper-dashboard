## Summary

Keep open. The PR targets the right sanitizer path and includes useful regression coverage, but latest head still misses current generated exec-state, failed/no-output async completion, and location/structured metadata wrapper shapes, so runtime context can still reach user-visible replies.

## What This Changes

The PR adds modern runtime-context stripping in `src/agents/internal-runtime-context.ts`, sanitizer regression tests, and a changelog fix entry for #75726.

## Best Possible Solution

Align the stripper and tests with the actual runtime-context generators, preferably by sharing sentinels or builder-based fixtures for exec state, async completion variants, and all generated metadata labels.

## Reproduction Assessment

Yes. Build wrappers from `buildExecOverridePromptHint`, `buildExecEventPrompt`, or inbound metadata with `Location`/structured context, then compare those generated paragraphs with the PR parser; source inspection shows the unmatched sections remain after stripping.

## Solution Assessment

No. Section-aware stripping is the right direction, but this implementation should be generator-aligned before merge so it does not preserve internal scaffolding variants that current main can emit.
