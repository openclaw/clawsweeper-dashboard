## Summary

Keep open. The PR is aimed at the right bug class, but the current patch still mismatches generated runtime-context shapes, so exec-state guidance, some inbound metadata blocks, and failed or missing-output async event wrappers can still leak into user-facing replies.

## What This Changes

The PR adds a modern runtime-context wrapper stripper, sanitizer regression tests, and a changelog fix entry for #75726.

## Best Possible Solution

Align the stripper and tests with the actual runtime-context generators, preferably through shared sentinels or builder-backed fixtures for exec state, async success/failure/missing-output variants, and all generated inbound metadata labels.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path by wrapping output from `buildExecOverridePromptHint`, `buildExecEventPrompt`, and `buildInboundUserContextPrefix`; the PR parser leaves several current generated shapes unmatched.

## Solution Assessment

No. Section-aware stripping is the right direction, but this implementation should be generator-aligned before merge so current OpenClaw scaffolding variants are not preserved in outbound replies.
