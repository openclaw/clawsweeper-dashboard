## Summary

Keep open. The PR is in the right area for the linked runtime-context leak, but the latest head still misses several wrapper shapes generated by current main, so it does not fully prevent internal runtime context from reaching user-visible replies.

## What This Changes

The PR adds modern runtime-context wrapper stripping, sanitizer regression tests, and a changelog entry for the linked leak report.

## Best Possible Solution

Align the stripper and tests with the actual generator outputs for exec state, inbound metadata labels, and async completion variants while preserving JSON and fenced visible replies.

## Reproduction Assessment

Yes. Build current-main wrappers from `buildExecOverridePromptHint`, `buildExecEventPrompt`, or inbound metadata with location/structured context, then inspect the PR parser paths; the source shows those generated sections are not fully consumed.

## Solution Assessment

No. Section-aware stripping is the right direction, but this implementation should be generator-aligned or share sentinels with the generators instead of matching a partial set of hand-shaped paragraphs.
