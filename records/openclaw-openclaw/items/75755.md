## Summary

Keep open. The PR targets a valid runtime-context leak, but the latest head still misses generated exec-state formatting and several generated metadata headings, so runtime scaffolding can still reach user-facing output.

## What This Changes

The PR adds a section-aware modern runtime-context stripper, regression tests, and a changelog entry for echoed OpenClaw runtime-context wrappers.

## Best Possible Solution

Finish the shared sanitizer so it consumes the exact wrapper sections produced by current runtime-context builders, preferably by reusing or centralizing the generated metadata sentinel list and testing against real builder output rather than hand-shaped fixtures.

Yes. Feed stripInternalRuntimeContext the modern header and notice followed by the exact buildExecOverridePromptHint output from current main and a visible reply; the PR parser rejects the exec-state block because it reads the generated single-newline section as one paragraph.

## Solution Assessment
