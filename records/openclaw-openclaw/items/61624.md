## What This Changes

The PR adds a WhatsApp `dmRequireMention` config flag, schema/generated metadata, account resolution, and direct-message mention gating before authorized DMs are processed.

## Best Possible Solution

Land a rebased WhatsApp-owned implementation that makes `dmRequireMention` a documented config contract, enforces it after DM authorization and before processing, preserves the empty-pattern guard, and covers account inheritance plus enabled/disabled gate behavior with focused tests.

Yes: current main's code path is enough to reproduce the gap. An authorized direct message passes `checkInboundAccessControl`, reaches `createWebOnMessageHandler`, and the direct branch proceeds toward `processMessage` without any mention-pattern gate.

## Solution Assessment
