## What This Changes

The PR adds a WhatsApp `dmRequireMention` config flag that gates authorized direct messages behind mention-pattern matching.

## Best Possible Solution

Land a rebased WhatsApp-owned implementation that enforces `dmRequireMention` after DM authorization and before processing, preserves the empty-pattern guard, uses the current runtime-config and mention-decision APIs, and documents the config contract.

Yes. Current main shows admitted WhatsApp DMs pass `checkInboundAccessControl`, then the direct branch proceeds toward preflight/broadcast/`processMessage` without any mention-pattern gate.
