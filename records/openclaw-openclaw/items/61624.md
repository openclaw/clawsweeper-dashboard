## What This Changes

The PR adds a WhatsApp `dmRequireMention` config flag, generated config metadata, account resolution plumbing, and a direct-message mention gate that reuses existing mention patterns.

## Best Possible Solution

Land a rebased WhatsApp-owned change that documents `dmRequireMention`, enforces it after DM authorization using current per-message config/account resolution and `resolveInboundMentionDecision`, and includes real WhatsApp DM proof.

## Reproduction Assessment

## Solution Assessment

No for this branch as-is. The product direction is coherent, but the patch must be rebased onto current WhatsApp runtime/config APIs, documented, and proven with real WhatsApp behavior before merge.
