## What This Changes

Request support for MCP servers to return a consent envelope that routes mutating tool calls through channel-mediated `/approve` before OpenClaw re-calls the tool with a confirmation token.

## Best Possible Solution

Review the linked implementation PR for the MCP consent-envelope contract, approval routing, token redaction, `/approve` provenance, opt-out configuration, and relationship to MCP elicitations, then close this issue only after that PR or an equivalent maintainer-approved implementation lands.

Yes, source inspection and the reporter's live vanilla run give a high-signal reproduction path for the missing enforcement: a third-party MCP tool can return a `requires_confirmation` envelope and current main has no envelope interception before returning the result to the agent. ## Solution Assessment
