## Summary

## What This Changes

Request runtime-enforced, channel-mediated approval for bundle-MCP tool calls that return a consent envelope, with an optional deployment disable switch.

## Best Possible Solution

Review and land, or replace, https://github.com/openclaw/openclaw/pull/78303 with a security-approved bundle-MCP approval design that documents the server contract and its relationship to MCP-native elicitations.

## Reproduction Assessment

Yes for the current-main baseline by source inspection: bundle-MCP tool results are returned directly to agent-visible output and the requested config switch is absent. This is a feature/security-hardening request rather than a broken documented behavior bug.

## Solution Assessment

Unclear: materializer-level consent-envelope interception and reuse of plugin approvals are plausible, but the final shape needs maintainer security/product review against MCP-native elicitations and channel-provenance requirements.
