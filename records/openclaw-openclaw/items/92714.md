Keep open. Current main still has the hardcoded node.invoke fallback, but this PR is not merge-ready because the new documented config key is not accepted by OpenClaw's strict config schema and the configured timeout is not sent to node clients.

## What This Changes

The PR adds `gateway.nodes.invokeTimeoutMs` docs, config typing, startup wiring, and NodeRegistry tests so node.invoke calls without a per-call timeout can use a configurable default.

## Best Possible Solution

Implement one validated resolved-timeout path: accept a positive integer in the canonical config schema, document the strict validation behavior, and send the resolved timeout in the node.invoke request payload while preserving per-call overrides.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.96

Full review comments:

- **[P1] Add the new config key to the canonical schema:** `src/gateway/server.impl.ts:943`
  - body: `cfgAtStart.gateway?.nodes?.invokeTimeoutMs` is read here, but `OpenClawSchema` still defines `gateway.nodes` as a strict object with only `browser`, `pairing`, `allowCommands`, and `denyCommands`. Because config validation uses that schema and the docs state unknown keys refuse startup, the documented `gateway.nodes.invokeTimeoutMs` example will be rejected before this runtime wiring can ever use it. Add the key to `src/config/zod-schema.ts` with matching help/label/docs metadata and a config validation test.
  - confidence: 0.97
- **[P1] Send the resolved timeout to node clients:** `src/gateway/node-registry.ts:493`
  - body: This resolved timeout is only used for the gateway-side timer; the payload sent earlier still contains `timeoutMs: params.timeoutMs`. When callers omit a per-call timeout, Swift node sessions receive no timeout and fall back to their own 30000ms budget, so long-running node commands can still time out before the configured gateway default. Compute the resolved timeout before building the payload and send that value to the node request while preserving explicit per-call overrides.
  - confidence: 0.93
