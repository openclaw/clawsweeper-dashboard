## Summary

Keep open: current main and the latest release still have a source-visible gap for timed-out or unsigned thinking-only turns, but the reporter’s follow-up and DeepSeek docs show the normal `reasoning_content` path should produce a signed `thinkingSignature`, so this needs focused DeepSeek trace proof before a replay-safety change.

## What This Changes

The issue reports that DeepSeek V4 reasoning-only turns with unsigned thinking blocks skip the visible-answer continuation retry and fall back to idle-timeout retry.

## Best Possible Solution

Capture a real DeepSeek V4 trace or focused fixture for the reported non-visible turn shape, then add a provider/attempt-scoped continuation path for verified replay-safe DeepSeek V4 partials without globally relaxing unsigned-thinking safety.

## Reproduction Assessment

No high-confidence live reproduction yet. Source inspection gives a medium-confidence path: an unsigned thinking-only assistant turn is classified as `incomplete-thinking`, and if the attempt is timed out both reasoning-only and empty-response continuation instructions are skipped before the same-model idle retry.

## Solution Assessment

Unclear. Broadening `isSignedThinkingBlock` or `isReasoningOnlyAssistantTurn` globally is not the safest fix; the maintainable path is a trace-backed DeepSeek V4-specific or attempt-scoped recovery that preserves replay-safety guards.
