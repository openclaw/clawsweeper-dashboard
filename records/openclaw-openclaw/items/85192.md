## What This Changes

The issue reports that DeepSeek V4 thinking-only turns without signature metadata skip visible-answer continuation recovery and fall through to idle-timeout retry behavior.

## Best Possible Solution

Add a narrow regression for the observed DeepSeek V4 non-visible turn shape, then route replay-safe unsigned or idle-timeout partial DeepSeek V4 turns through a visible-answer continuation without changing generic signed-thinking replay rules.

## Reproduction Assessment

Yes, source inspection gives a medium-confidence reproduction path for an assistant turn containing only unsigned `type: "thinking"` content: it is classified as `incomplete-thinking`, not `incomplete-text`, so reasoning-only retry is skipped. The live DeepSeek V4 claim still needs a real provider trace or a transport-level fixture because official docs describe `reasoning_content`, which OpenClaw normally signs as `thinkingSignature`.

## Solution Assessment

Unclear: broadening `isSignedThinkingBlock` globally is not the best first move because unsigned thinking also protects replay safety. The safer fix is provider/attempt-scoped continuation for the verified DeepSeek V4 shape, with tests covering signed, unsigned, timeout, and side-effect cases.
