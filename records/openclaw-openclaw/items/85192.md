## Summary

Keep open: current main still has a source-visible split where genuinely unsigned thinking blocks are not treated as reasoning-only retry candidates, while the normal DeepSeek `reasoning_content` stream path should be signed, so the remaining work needs a sanitized DeepSeek/ModelGate trace before changing provider behavior.

## What This Changes

The issue reports that DeepSeek V4 reasoning-only turns can be classified as unsigned thinking, preventing the visible-answer continuation retry and leading to idle-timeout retry behavior.

## Best Possible Solution

Capture a sanitized raw DeepSeek V4/ModelGate SSE plus persisted session fixture, then add a provider-scoped regression that ensures thinking-only no-visible turns get a visible-answer continuation before idle/fallback while preserving DeepSeek replay signatures.

## Reproduction Assessment

No high-confidence live current-main reproduction is established. Source inspection shows the unsigned-thinking branch bypasses the reasoning-only resolver, but the normal DeepSeek `reasoning_content` path is signed, so a sanitized SSE/session fixture is still the missing proof.

## Solution Assessment

Unclear. Broadening DeepSeek unsigned thinking into the reasoning-only retry may be right if the live trace proves signatureless blocks, but the safer fix is trace-first and provider-scoped rather than changing all thinking-block recovery.
