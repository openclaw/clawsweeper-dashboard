## Summary

Keep open: current main still has the source-visible skip path for unsigned or timed-out thinking-only assistant turns, but the normal DeepSeek V4 `reasoning_content` stream path should create `thinkingSignature`, so this needs a real DeepSeek trace before changing replay safety.

## What This Changes

Reports that DeepSeek V4 reasoning-only turns can miss the visible-answer continuation retry when thinking blocks lack a recognized signature or arrive through the idle-timeout path.

## Best Possible Solution

Capture a redacted DeepSeek V4 thinking-mode SSE or focused fixture for the exact non-visible turn shape, then add a provider- or attempt-scoped continuation path only for verified replay-safe partials.

## Reproduction Assessment

No high-confidence live reproduction yet. Source inspection shows the unsigned-thinking and timed-out skip paths, while the normal DeepSeek `reasoning_content` path should produce a signed `thinkingSignature`.

## Solution Assessment

Unclear. Broadening `isSignedThinkingBlock` or `isReasoningOnlyAssistantTurn` globally is not the safest fix; the safer direction is trace-backed DeepSeek-specific or attempt-scoped recovery that preserves replay-safety guards.
