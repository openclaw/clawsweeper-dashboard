## Summary

Keep open: current main still has a source-visible gap for unsigned or timed-out thinking-only turns, but the reporter's follow-up and current DeepSeek stream handling show the normal `reasoning_content` path should produce a signed `thinkingSignature`; the missing proof is a real DeepSeek V4 trace for the reported shape.

## What This Changes

The issue reports that DeepSeek V4 thinking-only turns without recognized thinking signatures can miss the visible-answer continuation retry and fall back to idle-timeout retry behavior.

## Best Possible Solution

Capture a real DeepSeek V4 trace or focused fixture for the reported non-visible turn shape, then add a provider- or attempt-scoped continuation path for verified replay-safe DeepSeek V4 partials without globally relaxing unsigned-thinking safety.

## Reproduction Assessment

No high-confidence live reproduction yet. Source inspection gives a medium-confidence path: an unsigned thinking-only assistant turn is classified as `incomplete-thinking`, and if the attempt timed out both visible-answer continuation helpers are skipped before same-model idle retry.

## Solution Assessment

Unclear. Broadening `isSignedThinkingBlock` or `isReasoningOnlyAssistantTurn` globally is not the safest fix; the maintainable path is a trace-backed DeepSeek V4-specific or attempt-scoped recovery that preserves replay-safety guards.
