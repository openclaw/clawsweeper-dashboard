## Summary

Keep open: current main still has a plausible source-level gap for unsigned thinking-only turns and timed-out reasoning-only partials, but the reporter's follow-up correctly narrows the DeepSeek V4 claim because the normal `reasoning_content` path should produce `thinkingSignature` metadata.

## What This Changes

The issue reports that DeepSeek V4 reasoning-only turns can miss the visible-answer continuation retry when thinking blocks are unsigned or only discovered after an idle timeout.

## Best Possible Solution

Capture a raw DeepSeek V4 thinking-mode stream or focused transport fixture, then add provider-scoped regression coverage and route only replay-safe DeepSeek non-visible/timed-out partial turns through the visible-answer continuation path.

## Reproduction Assessment

No high-confidence live reproduction yet. Source inspection gives a credible synthetic path for unsigned thinking-only or timed-out partial turns, but the actual DeepSeek V4 raw stream shape still needs provider trace evidence.

## Solution Assessment

Unclear: broadening `isSignedThinkingBlock` globally is not the best fix because unsigned thinking is also a replay-safety signal. The safer path is a verified DeepSeek-specific fixture plus a narrow continuation rule that preserves generic unsigned-thinking safeguards.
