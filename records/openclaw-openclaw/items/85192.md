## Summary

Keep open: current main and the latest release still have the source-visible retry gap for timed-out or truly signatureless thinking-only turns, while the normal DeepSeek `reasoning_content` path should produce a signed `thinkingSignature`. The new production comment adds useful signal, but its shown session block includes `thinkingSignature: "reasoning_content"`, so a raw DeepSeek/proxy trace is still needed before choosing the safe fix boundary.

## What This Changes

The issue asks OpenClaw to recover DeepSeek V4 reasoning-only turns when thinking blocks are treated as unsigned and the visible-answer continuation retry is skipped.

## Best Possible Solution

Capture a sanitized DeepSeek V4-Pro SSE/request/session fixture for the reported failure, then add a narrow provider- or attempt-scoped recovery that preserves replay-safety instead of globally relaxing unsigned-thinking classification.

## Reproduction Assessment

No high-confidence live reproduction yet. Source inspection shows a clear current-main path where signatureless thinking-only turns are not treated as reasoning-only continuation candidates, but the reported DeepSeek production trace still needs raw SSE or sanitized session evidence because the normal `reasoning_content` path should be signed.

## Solution Assessment

Unclear. A global `isSignedThinkingBlock` relaxation is not the safest fix; the better direction is a trace-backed DeepSeek V4 or attempt-scoped replay recovery that keeps side-effect and provider replay guards intact.
