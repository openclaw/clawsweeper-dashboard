Keep open: Docker `--init` is the right direction for the PID 1 reaping problem, but this PR only affects newly created containers and still lacks after-fix real Docker proof. ## What This Changes

Adds Docker `--init` to sandbox container creation and updates create-args tests to assert the flag is emitted.

## Best Possible Solution

Land Docker `--init` together with a create-args epoch bump for normal and browser sandbox hashes, plus redacted live Docker proof that existing long-lived containers recreate and new containers run with init.

## Reproduction Assessment

Yes, source-level: https://github.com/openclaw/openclaw/issues/68691 includes concrete PPID=1 zombie evidence, and current main still creates sandbox containers as `sleep infinity` without Docker `--init`. I did not run a live stress workload in this read-only review.

No, not as written. Docker `--init` is the right mechanism for newly created containers, but this PR needs the create-args epoch rollout from https://github.com/openclaw/openclaw/pull/76590 or an equivalent integrated change so existing long-lived containers are recreated.
