## Summary

Keep open. The PR uses the right Docker mechanism for newly created sandboxes, but as written it leaves existing hash-matching Docker and sandbox-browser containers on the old PID 1 behavior and lacks the required changelog entry.

## What This Changes

The PR adds Docker `--init` to sandbox container creation and updates create-argument tests to assert that flag is present after `docker create`.

## Best Possible Solution

Enable `--init`, add an intentional create-args/hash epoch that covers both normal and browser sandbox containers, add focused recreate/hash coverage, and include a single-line Unreleased changelog fix entry.

## Reproduction Assessment

Yes for the source-level behavior: current `main` creates Docker sandboxes without `--init`, and #68691 provides observed PPID=1 zombie evidence. The rollout bug in this PR is reproducible by an existing container whose unchanged `openclaw.configHash` still matches after the patch.

## Solution Assessment

No, not as written. `--init` is the right Docker flag for newly created containers, but the maintainable fix must also force affected existing normal and browser sandboxes through the existing hash/recreate mechanism.
