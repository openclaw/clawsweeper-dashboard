## Summary

The PR adds Docker `--init` to sandbox container creation and updates create-args tests to assert the flag.

## Best Possible Solution

Land `--init` only with a same-release create-args hash epoch bump, focused hash/recreate tests for normal and browser sandboxes, branch refresh, and real Docker proof of the upgrade path.

Yes for the upgrade gap via source inspection: the PR adds a create flag, while the reuse hash on the PR branch lacks any create-args epoch and existing containers only recreate on hash mismatch. No live Docker zombie reproduction was run in this read-only review.

## Solution Assessment

No. Adding `--init` is the right Docker-level direction for new containers, but the best fix must also bump a create-args epoch in the sandbox config hash and prove the existing-container upgrade path.
