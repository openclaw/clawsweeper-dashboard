## Summary

Keep this PR open because the underlying sandbox zombie-process bug is real, but the patch is not merge-ready: it adds the right Docker flag for newly created containers without forcing existing long-lived sandboxes through the rollout, and the contributor has not posted real Docker behavior proof.

## What This Changes

This PR adds Docker `--init` to sandbox container create arguments and updates create-args tests to assert the flag is present.

## Best Possible Solution

Land Docker `--init` together with a create-args epoch bump in the normal and browser sandbox hashes, then verify with redacted live Docker proof that existing long-lived containers are recreated.

## Reproduction Assessment

Yes, source-level. Current main creates long-lived Docker sandboxes as `sleep infinity` without `--init`, and the linked report shows PPID=1 zombie evidence; I did not run a live stress workload in this read-only review.

## Solution Assessment

No, not as written. Docker `--init` is the right mechanism for new containers, but this PR needs the create-args epoch rollout, likely coordinated with https://github.com/openclaw/openclaw/pull/76590, before it solves existing long-lived sandboxes.
