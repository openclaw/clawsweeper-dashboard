## Summary

Keep open: the `--init` direction is valid for the sandbox zombie-process bug, but the patch does not force reused normal/browser sandboxes through recreation and lacks the required changelog entry.

## What This Changes

The PR adds Docker `--init` to sandbox container creation args and extends create-args tests to assert the flag appears after `docker create`.

## Best Possible Solution

Ship Docker `--init` with a create-args epoch/hash rollout for both normal and browser sandboxes, focused recreate coverage, and a single-line changelog fix entry.

## Reproduction Assessment

Yes for source-level reproduction: current main creates long-lived Docker sandboxes without `--init`, and #68691 provides PPID=1 zombie evidence from production. I did not run a live zombie workload in this read-only pass.

## Solution Assessment

No, not as written. Docker `--init` is the right mechanism for newly created containers, but the patch also needs a create-args hash/epoch rollout so existing hash-matching sandboxes are recreated.
