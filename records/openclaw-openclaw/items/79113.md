Keep open: the PR is on the right path and has sufficient terminal proof, but latest head still silently ignores invalid `--port` values for health/probe and includes stale help/changelog text.

## What This Changes

The PR routes gateway health/probe port targeting through inherited/config-backed gateway options, updates gateway CLI docs/changelog, and adds CLI coverage.

## Best Possible Solution

Keep the config-backed port override design, but share strict port validation with `gateway run` and leave one accurate changelog/doc description of the supported flag placement.

## Reproduction Assessment

Yes, by source inspection and the linked released-version output. Current main registers a gateway parent `--port` but the health/probe query paths do not carry it into target resolution, matching the reported non-default-port failure.

## Solution Assessment

No. The config-backed inheritance path is maintainable, but this patch still needs strict invalid-port rejection and help/changelog cleanup before merge.
