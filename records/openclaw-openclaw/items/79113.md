## Summary

Keep this PR open: the linked bug is valid and the PR is focused, but the current patch still regresses gateway target resolution for configured auth/TLS and does not reliably route the documented parent-level probe port.

## What This Changes

The PR adds inherited `--port` handling for gateway health/probe, updates gateway CLI docs and changelog, and adds gateway CLI regression coverage.

## Best Possible Solution

Carry a typed local port override through the existing config-backed gateway target resolution path so `--url` precedence, configured auth, TLS scheme, and local probe semantics are preserved.

## Reproduction Assessment

Yes. Current main source shows `gateway run` has `--port` while `gateway health` and `gateway probe` do not expose an equivalent working query path, and the linked issue provides released-version unknown-option output.

## Solution Assessment

No. Inheriting the parent option is a reasonable direction, but this implementation replaces loaded config on health and routes probe port through an explicit plaintext URL instead of the existing local target resolver.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Preserve config when applying --port:** `src/cli/gateway-cli/call.ts:48`
  - body: When `--port` is supplied, this passes a synthetic config into `callGateway`. Normal CLI opts do not carry the loaded config, and `callGateway` treats any provided config as complete, so configured auth, TLS, mode, and remote settings are dropped. Merge the real config or pass a typed port override without replacing config.
  - confidence: 0.94
- **[P2] Honor inherited --port in probe:** `src/cli/gateway-cli/register.ts:619`
  - body: The documented `openclaw gateway --port 18789 probe` path stores the inherited parent value on `rpcOpts.port`, but this URL override checks `opts.port`, so the parent-level port is ignored and the probe falls back to the configured/default target. Build the probe target from the inherited value or route it through status config resolution.
  - confidence: 0.91
- **[P2] Keep probe --port on the local target path:** `src/cli/gateway-cli/register.ts:619`
  - body: For child-level `gateway probe --port`, this converts the port to `ws://127.0.0.1:<port>` before status target resolution. Existing local probe resolution uses `wss://` when gateway TLS is enabled, so TLS-enabled local gateways on a non-default port will be probed over plaintext and fail. Route the port through the config-backed local target path and add a TLS regression test.
  - confidence: 0.88
