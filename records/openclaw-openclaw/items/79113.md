## Summary

Keep open. The latest PR head addresses the earlier config-preservation and TLS-routing concerns, but it still accepts or silently ignores invalid `--port` values on the new health/probe paths and has one stale probe help string.

## What This Changes

The PR adds parent-inherited `--port` handling for gateway RPC/probe commands, routes the override through gateway config, updates gateway CLI docs/changelog, and adds gateway CLI coverage.

## Best Possible Solution

Keep the config-backed port override design, but validate query-command ports with the same strict semantics as `gateway run` and keep CLI help aligned with the TLS-preserving implementation.

## Reproduction Assessment

Yes. Current main source shows the gateway parent has a `--port` option while the health/probe query paths do not carry it into their target resolution, and the linked report provides released-version unknown-option output.

## Solution Assessment

No. Routing `--port` through config-backed gateway target resolution is the right direction, but this patch should reject invalid port values instead of falling back or creating invalid local URLs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Reject invalid RPC port overrides:** `src/cli/gateway-cli/call.ts:35-38`
  - body: The new health/RPC port path accepts `Number(port)` and returns the base config for invalid values, so `openclaw gateway --port abc health` silently checks the configured/default gateway and `--port 18789.5` writes a decimal into `config.gateway.port`. Use the existing strict port parser and fail the command when a supplied port is invalid.
  - confidence: 0.88
- **[P2] Reject invalid probe port overrides:** `src/cli/gateway-cli/register.ts:617-624`
  - body: The probe path has the same fallback: nonnumeric or non-positive `--port` becomes `undefined`, so the command probes the configured/default target rather than reporting a bad flag. Validate with the same strict port semantics before calling `gatewayStatusCommand`.
  - confidence: 0.86
- **[P3] Correct the probe port help text:** `src/cli/gateway-cli/register.ts:606`
  - body: The added help says `--port` builds `ws://127.0.0.1:<port>`, but the implementation intentionally passes a typed port so status resolution can preserve `wss://` for TLS. Update the help text to describe the config-backed local port override.
  - confidence: 0.9
