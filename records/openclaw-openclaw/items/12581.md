  - body: `ExtensionContext` does not declare `sessionKey` or `sessionId`, so these casts can stay undefined in the real extension path and silently suppress every `session:prune` emission. The implementation needs a typed identity source and an integration test through the extension handler.
  - confidence: 0.9

  - body: `triggerInternalHook` awaits registered handlers without a timeout. Awaiting it from the context-pruning path means a slow or stuck hook can block model-context preparation for any request that prunes.
  - confidence: 0.91
- **[P2] Sanitize or omit raw tool names in hook metadata:** `src/agents/pi-extensions/context-pruning/pruner.ts:288-290`
  - body: `toolName` is copied directly into the hook payload. Hook handlers commonly log or forward context fields, so the new contract should either omit tool names by default or normalize, cap, and document them as untrusted metadata.
  - confidence: 0.84
- **[P3] Document the prune event payload contract:** `docs/automation/hooks.md:235`
  - body: The docs add the event name but not the context fields, privacy limits, delivery timing, or handler expectations. Consumers cannot safely write handlers against an undocumented payload shape.
