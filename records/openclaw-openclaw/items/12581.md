- **[P1] Plumb session identity through a supported contract:** `src/agents/pi-extensions/context-pruning/extension.ts:29-30`
  - body: The normal extension path reads `sessionKey` and `sessionId` through casts, but the current `ExtensionContext` contract and `createContext()` do not expose those fields.   - confidence: 0.88
- **[P1] Keep pruning from waiting on unbounded hooks:** `src/agents/pi-extensions/context-pruning/pruner.ts:293`
  - body: `triggerInternalHook` awaits each registered handler sequentially with no timeout, and this call now runs inside context pruning. A slow or hung hook can block prompt construction for pruned turns; use bounded fire-and-forget delivery or a clear timeout instead.
  - confidence: 0.91
- **[P2] Normalize or omit tool names before hook emission:** `src/agents/pi-extensions/context-pruning/pruner.ts:289`
  - body: The new payload forwards `ToolResultMessage.toolName` values directly into hook context. Those values can come from repaired or replayed transcript state and are commonly logged by hooks, so the event should normalize/cap names or omit them by default unless maintainers approve the exposure.
  - confidence: 0.76
- **[P2] Document the prune payload contract:** `docs/automation/hooks.md:235`
