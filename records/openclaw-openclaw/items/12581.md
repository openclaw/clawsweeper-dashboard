  - body: These casts read `sessionKey` and `sessionId` from fields that are not declared on the extension context and are not produced by the current context factory. If both are absent, `emitPruneHook` silently returns and the new event never fires through the normal runtime path.
  - confidence: 0.9

  - body: `triggerInternalHook` awaits registered handlers sequentially and has no timeout. Awaiting it inside context pruning means a slow or hanging hook handler can stall prompt construction and agent turns even though the comment says pruning should never fail due to hooks.
  - confidence: 0.92
- **[P1] Define a safe prune payload contract:** `src/agents/pi-extensions/context-pruning/pruner.ts:331-332`
  - body: The PR forwards raw `toolName` values and `sessionId` into hook context without sanitization, allowlisting, size limits, or documentation that the fields are untrusted. Hook consumers may log, index, or automate from this payload, so the metadata boundary needs to be explicit before merge.
  - confidence: 0.88

  - body: The docs entry names `session:prune` but does not describe the context payload, delivery semantics, or trust boundary, leaving hook authors without the contract this new event is meant to provide.
  - confidence: 0.84
