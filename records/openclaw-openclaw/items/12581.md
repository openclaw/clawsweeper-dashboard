  - body: The hook only emits when `sessionKey` or `sessionId` is present, but the extension reads both via casts from fields that are not on the current `ExtensionContext` and are not populated by `createContext()`.   - confidence: 0.9
- **[P1] Bound hook delivery before awaiting handlers:** `src/agents/pi-extensions/context-pruning/pruner.ts:293`
  - body: `triggerInternalHook` awaits registered handlers sequentially and has no timeout. Awaiting it from context pruning means one slow or never-settling hook handler can block model-context construction and stall the agent turn, so prune hook delivery needs a timeout or non-blocking policy.
  - confidence: 0.93
- **[P1] Define a safe prune payload contract:** `src/agents/pi-extensions/context-pruning/pruner.ts:286-290`
  - body: The new payload forwards `sessionId` and raw `toolNames` to installed hook handlers without a sanitizer, length cap, or documented trust boundary. Tool names can come from transcript/tool-call state, so hook logs or indexers can receive untrusted or sensitive metadata unless the payload is normalized and explicitly bounded.
  - confidence: 0.88
- **[P2] Document the emitted context fields:** `docs/automation/hooks.md:235`
  - body: The docs add `session:prune` but only say when it fires; they do not list `softTrimmedCount`, `hardClearedCount`, `toolNames`, `sessionId`, or `prunedAt`. Hook consumers need the payload shape and privacy notes to use the event correctly.
  - confidence: 0.86
