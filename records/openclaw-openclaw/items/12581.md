  - body: `ExtensionContext` does not declare or construct `sessionKey` or `sessionId`, so these casts read undocumented fields that are absent on the current context object. In the normal extension path that leaves `hookSessionKey` undefined and silently suppresses the new `session:prune` event.

  - body: This awaits `triggerInternalHook` from context pruning, but the shared hook dispatcher awaits each handler sequentially without a timeout. A slow or hung installed hook can stall context assembly and block an agent turn whenever pruning runs.
  - confidence: 0.9
- **[P1] Define a safe prune payload contract:** `src/agents/pi-extensions/context-pruning/pruner.ts:330-333`
  - body: The new event forwards raw `toolName` values and `sessionId` to installed hook handlers without a documented trust, escaping, or length contract. Hook consumers commonly log or index these fields, so the payload should be sanitized, bounded, or intentionally documented before exposing it as a lifecycle surface.
  - confidence: 0.84
- **[P2] Document the emitted context fields:** `docs/automation/hooks.md:235`
  - body: The docs entry names `session:prune` but does not describe the context payload that hook authors can rely on. Since the event's value is the metadata it emits, the docs need the field names, types, and trust notes before this becomes public hook behavior.
  - confidence: 0.88
