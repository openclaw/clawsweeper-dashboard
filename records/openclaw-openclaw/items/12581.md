- **[P1] Pass a real session identity into prune hooks:** `src/agents/pi-extensions/context-pruning/extension.ts:29-30`
  - body: The production extension context contract does not expose `sessionKey` or `sessionId`, and current `createContext()` does not populate those props.   - body: `triggerInternalHook` awaits matching handlers sequentially and has no timeout here. Awaiting it from context pruning means any slow or hung `session:prune` handler can stall prompt construction and the model turn; make this emission bounded or nonblocking.
  - confidence: 0.88
- **[P2] Bound and sanitize prune hook tool names:** `src/agents/pi-extensions/context-pruning/pruner.ts:289`
  - body: The PR forwards raw `toolName` strings as supposedly safe summary metadata. Because these values can flow into operator hooks, logs, or indexers, the payload should cap and sanitize names or omit them unless maintainers explicitly approve the exposure.
  - confidence: 0.76
- **[P2] Document the prune hook payload:** `docs/automation/hooks.md:235`
  - body: The docs add the event name but not the context fields that hook authors are expected to consume. Document `softTrimmedCount`, `hardClearedCount`, `toolNames`, `sessionId`, and `prunedAt`, or avoid exposing them as a contract.
