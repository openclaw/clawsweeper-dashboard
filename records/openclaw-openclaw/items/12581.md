  - body: The PR reads `sessionKey` and `sessionId` through casts on `ExtensionContext`, but the current `ExtensionContext` contract does not declare those fields.   - body: `triggerInternalHook` awaits every registered handler sequentially and has no timeout. Awaiting it from context pruning means a slow or stuck hook handler can stall context preparation for the model turn, so delivery needs to be fire-and-forget or time-bounded before this runs in the pruning path.
  - confidence: 0.9
- **[P1] Define a safe prune payload contract:** `src/agents/pi-extensions/context-pruning/pruner.ts:285-290`
  - body: The new event forwards `sessionId`, counts, `toolNames`, and `prunedAt` directly to installed hook handlers, but `toolName` can come from repaired/model-supplied tool calls. Please sanitize or drop that field, cap payload sizes, and make the trusted/untrusted context contract explicit before documenting the event.
  - confidence: 0.82
- **[P2] Document the emitted context fields:** `docs/automation/hooks.md:235`
  - body: The docs entry says when `session:prune` fires but does not list the context fields or trust/escaping expectations hook authors can rely on. Once the payload is finalized, document the fields and any untrusted-string handling requirements here.
  - confidence: 0.78
