  - body: `ExtensionContext` does not define `sessionKey` or `sessionId`, and both the current OpenClaw `createContext()` implementation and the upstream 0.52.9 package context omit those fields.   - confidence: 0.9
- **[P1] Bound or decouple prune hook delivery:** `src/agents/pi-extensions/context-pruning/pruner.ts:293`
  - body: This awaits `triggerInternalHook()` inside context pruning, but the hook dispatcher awaits each handler without a timeout. A hook that never settles will block pruning and therefore model request preparation; make this fire-and-forget with bounded logging or race it against a short timeout before returning pruned messages.
  - confidence: 0.88
- **[P3] Document the prune payload contract:** `docs/automation/hooks.md:235`
  - body: The docs add `session:prune` but do not list the context fields or their trust boundary. Since the event’s value is the metadata payload, document fields such as counts, optional session id, optional tool names, timestamp semantics, and that hook consumers must treat metadata as untrusted before logging or exporting it.
  - confidence: 0.78
