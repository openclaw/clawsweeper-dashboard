- **[P1] Port the hook to the active pruning path:** `src/agents/pi-extensions/context-pruning/extension.ts:7`
  - body: Current main no longer tracks `src/agents/pi-extensions/context-pruning/*`; the embedded runtime imports context pruning from `src/agents/agent-hooks/context-pruning/*`. As written, this branch adds the hook to the old path, so after resolving conflicts it will not wire `session:prune` into the live runtime unless the change is ported.
  - confidence: 0.94
- **[P1] Use a typed session identity for prune events:** `src/agents/pi-extensions/context-pruning/extension.ts:29-30`
  - body: `ExtensionContext` does not define `sessionKey` or `sessionId`, and the runner-created context does not populate them. These casts can leave `hookSessionKey` undefined in the normal extension path, causing the new hook to be silently suppressed even when pruning occurred.
  - confidence: 0.88
- **[P1] Bound prune hook delivery:** `src/agents/pi-extensions/context-pruning/pruner.ts:292`
  - body: `triggerInternalHook` awaits each handler serially with no timeout, and this code awaits it from the pruning path that runs before model calls. A slow or hung hook handler can therefore stall context construction and block agent turns.
  - confidence: 0.9
- **[P1] Define a safe metadata payload before emitting it:** `src/agents/pi-extensions/context-pruning/pruner.ts:287-290`
  - body: The new event forwards `sessionId` and raw `toolNames` to hook handlers. Before this becomes an observable contract, the payload should be bounded, sanitized or reduced to counts by default, and documented so hooks cannot accidentally leak or log-inject tool/session metadata.
  - confidence: 0.86
- **[P2] Document the prune event context fields:** `docs/automation/hooks.md:235`
  - body: The docs entry says when `session:prune` fires but not which context fields are stable, sanitized, optional, or intentionally omitted. Hook authors need that payload contract to write correct handlers.
  - confidence: 0.82

## Security Review

Status: needs_attention
