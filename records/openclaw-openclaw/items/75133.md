This PR exports manifest activation normalization, reads `activation` from Codex, Claude, and Cursor bundle manifests, copies it into bundle registry records, adds Claude bundle coverage, and adjusts gateway run option-collision test environment isolation.

Yes. A Claude bundle on current main with `.claude-plugin/plugin.json` containing `activation.onStartup: false` is parsed into a bundle manifest with no activation field, and `buildBundleRecord` then creates a registry record with activation still undefined.

## Solution Assessment

Yes. Reusing the existing native manifest activation normalizer and threading the normalized value through bundle parsing and registry assembly is the narrowest maintainable fix; the remaining work is documentation, changelog, rebase, and validation.
