## Summary

Keep open, but the PR is not merge-ready: it addresses a real Codex message-loss bug, yet the snapshot-loop change can record stale prior-turn native tool failures as the current attempt's lastToolError.

## What This Changes

The PR records non-success Codex app-server native tool items as attempt-level lastToolError and adds projector regression tests.

## Best Possible Solution

Apply the existing current-turn snapshot boundary to native error recording, keep the item/completed path, add a prior-turn declined or failed snapshot regression, and land one canonical fix for https://github.com/openclaw/openclaw/issues/83093.

## Reproduction Assessment

Yes. Source inspection and the PR proof show declined native commandExecution currently does not reach lastToolError, while the existing payload builder only emits the visible warning when that field is present.

## Solution Assessment

No as written. Reusing lastToolError is the right narrow direction, but the snapshot-loop implementation must not record non-current-turn native tool failures.
