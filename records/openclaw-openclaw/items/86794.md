- **[P2] Invalidate cached candidates after store writes:** `src/gateway/session-utils.ts:495-496`
  - body: This reuses the child-candidate index whenever the store object is the same, but existing writer paths can take the cached store, mutate it, and write it back with the same object. Please key this by a store generation/serialized version or clear it from the store write/invalidation path and add that regression.
