## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Persist subscriptions in plugin state:** `extensions/webhooks/src/subscriptions.ts:105-107`
  - body: Dynamic subscriptions are durable OpenClaw runtime state and include route HMAC secrets, but this writes them to `webhooks/subscriptions.json`. The repo already exposes a SQLite-backed plugin keyed store with `entries()`, `register()`, `delete()`, and `update()`, so this should use that canonical state path instead of adding a new JSON sidecar and migration surface.
  - confidence: 0.9
- **[P1] Preserve the bundled-only scheduler contract:** `src/plugins/host-hook-scheduled-turns.ts:209-210`
  - body: Current SDK docs say `scheduleSessionTurn` and `unscheduleSessionTurnsByTag` are bundled-only, but this change lets every trusted official installed plugin schedule and unschedule Cron-backed agent turns. Keep the existing gate unless maintainers explicitly approve the broader API and the SDK docs/contracts are updated with the new security boundary.
  - confidence: 0.87
- **[P1] Correlate completion delivery to the accepted run:** `extensions/webhooks/src/dispatch.ts:54-60`
  - body: `onAgentCompletionDispatch` runs before the scheduler accepts the turn, while completion lookup later matches only by `sessionKey`. If scheduling fails or two webhook-triggered runs share a session, a later lifecycle event can consume the wrong pending context and deliver another run's output to this route's completion target.
  - confidence: 0.9
- **[P2] Keep dynamic target refresh out of delivery:** `extensions/webhooks/index.ts:76-79`
  - body: Every webhook request can call `subscriptionStore.loadTargets()`, which stats and may reread the subscription state file before dispatch. Gateway/plugin hot paths should use a lifecycle-owned snapshot updated by subscribe/remove/startup load instead of doing filesystem freshness work on inbound delivery.
  - confidence: 0.84
- **[P2] Keep the shipped Gmail commands documented:** `docs/cli/webhooks.md:24-38`
  - body: The PR still registers `openclaw webhooks gmail setup|run` before loading plugin CLI commands, and the automation docs still point to that flow, but the changed CLI page lists only the new dynamic subscription commands. Preserve the Gmail reference alongside the new commands so a shipped CLI surface does not disappear from docs.
  - confidence: 0.88
