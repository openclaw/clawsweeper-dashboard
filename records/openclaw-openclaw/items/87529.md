- **[P1] Correlate completion delivery to the accepted run:** `extensions/webhooks/src/dispatch.ts:54-60`
  - body: `onAgentCompletionDispatch` runs before the scheduler accepts the turn, and completion lookup later starts from sessionKey. If scheduling fails or two webhooks share a session concurrently, a later run can consume the wrong completion context and deliver the wrong assistant output.
  - confidence: 0.9
- **[P2] Move dynamic subscriptions out of JSON sidecars:** `extensions/webhooks/src/subscriptions.ts:105-107`
  - body: Dynamic subscriptions are OpenClaw-owned runtime state and include generated HMAC secrets, but this stores them under `webhooks/subscriptions.json`. Use the shared/plugin SQLite-backed state path or an approved plugin KV owner instead of adding a JSON sidecar.
  - confidence: 0.86
- **[P2] Avoid reloading subscription files on webhook delivery:** `extensions/webhooks/index.ts:76-79`
  - body: The HTTP handler resolves dynamic targets for every request, and that path stats/reads the subscription file and rebuilds target maps. Gateway/plugin hot paths should use a lifecycle-owned snapshot updated by subscribe/remove/startup instead of filesystem freshness checks per delivery.
  - confidence: 0.84
- **[P2] Keep the shipped Gmail webhook commands documented:** `docs/cli/webhooks.md:24-38`
  - body: The branch still registers `openclaw webhooks gmail setup|run` before loading plugin CLI commands, but this page now lists only the new dynamic subscription commands. Keep the Gmail reference alongside the new commands so shipped CLI behavior does not disappear from docs.
