## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Correlate completion delivery to the accepted run:** `extensions/webhooks/src/dispatch.ts:54-60`
  - body: `onAgentCompletionDispatch` runs before the scheduler accepts the turn, and completion lookup later matches only by `sessionKey`. If scheduling fails or two webhooks share a session concurrently, a later run can pick up the wrong completion context and deliver the wrong agent output.
  - confidence: 0.9
- **[P2] Move dynamic subscriptions out of JSON sidecars:** `extensions/webhooks/src/subscriptions.ts:105-107`
  - body: Dynamic subscriptions are OpenClaw-owned runtime state and include generated HMAC secrets, but this stores them in `webhooks/subscriptions.json` under the state directory. Use the shared/plugin SQLite-backed state path or an approved plugin KV owner instead of adding a JSON sidecar for persistent runtime state.
  - confidence: 0.86
- **[P2] Avoid polling subscription files from webhook delivery:** `extensions/webhooks/src/subscriptions.ts:239-254`
  - body: Every inbound webhook calls `resolveTargetsByPath()`, which calls `loadTargets()`, stats/reads the subscription file, reparses routes, and rebuilds targets. Gateway/plugin hot paths should reuse a lifecycle-owned snapshot updated by subscribe/remove/startup instead of doing filesystem freshness checks per delivery.
  - confidence: 0.84
- **[P2] Keep the shipped Gmail webhook commands documented:** `docs/cli/webhooks.md:24-38`
  - body: The PR still registers `openclaw webhooks gmail setup|run` before loading plugin CLI commands, but this page now lists only the new dynamic subscription commands. Keep the Gmail reference alongside the new commands so shipped CLI behavior does not disappear from docs.
  - confidence: 0.87
