## Summary

Keep open: the requested direction is plausible, but the current PR is not merge-ready because it leaves the status/log-file helper on the old fallback path and lacks contributor-provided real behavior proof.

## What This Changes

The branch updates `src/logging/logger.ts` to read `OPENCLAW_LOG_DIR` for default log paths and `src/infra/tmp-openclaw-dir.ts` to read and trust-check `OPENCLAW_TMP_DIR` before default temp-dir selection.

## Best Possible Solution

Rebase and rework the change so one default log-file resolver backs both logger and status output, temp overrides keep the existing trust checks, and the new env vars are documented and covered by focused tests.

## Reproduction Assessment

Yes, source-reproducible: current main's Node logger and temp-dir resolver do not read these env vars, and the proposed diff leaves status log-file resolution on the old tmp-dir fallback. I did not run the gateway because this review is read-only.

## Solution Assessment

No as-is. The direction is narrow, but the maintainable fix needs to centralize default log path resolution for logger and status surfaces, preserve `logging.file` precedence, document the env vars, and include real behavior proof.
