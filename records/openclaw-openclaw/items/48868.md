## Summary

Keep open: current main still ignores the requested log/temp directory env vars, but this PR is not ready because it leaves the status log-file helper on the old fallback, lacks real gateway proof, and adds an undocumented public env-var surface.

## What This Changes

The PR adds process-environment overrides for the default log directory and preferred OpenClaw temp directory in `src/logging/logger.ts` and `src/infra/tmp-openclaw-dir.ts`.

## Best Possible Solution

Rebase and implement the overrides through shared log-file and temp-dir resolvers, preserve `logging.file` precedence and temp-dir trust checks, document the public env vars, add focused tests, and attach redacted real gateway proof.

## Reproduction Assessment

Yes, source-reproducible from current main: the logger and temp-dir resolver do not read these env vars, and the PR diff leaves the status helper on the old temp fallback. I did not run the gateway because this review is read-only.

## Solution Assessment

No as-is. The direction is narrow, but the maintainable fix should centralize default log path resolution for logger and status surfaces, preserve existing config precedence, document the env vars, and include real behavior proof.
