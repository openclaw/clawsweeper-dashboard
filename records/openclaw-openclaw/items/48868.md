## Best Possible Solution

Rebase and rework the change so env overrides feed the logger, status/logs fallback path, tmp-dir trust checks, docs, and focused tests while preserving the existing `logging.file` contract.

Yes. Source inspection shows current main has no `OPENCLAW_LOG_DIR` or `OPENCLAW_TMP_DIR` handling, and the PR diff shows the remaining status/fallback log-path mismatch without needing a full gateway run.

## Solution Assessment

No as-is. The direction is narrow, but the maintainable fix should centralize default log-directory/file resolution for both logger and status surfaces, keep the temp-dir trust checks, then add docs, tests, and real behavior proof.
