## Summary

Keep this PR open. Current main still lacks the TypeScript Gateway env-var support the PR targets, and the latest PR head remains a plausible implementation candidate, but it is not merge-ready because the active logger override would diverge from the separate default log-path helper used by status surfaces.

## What This Changes

The PR adds `OPENCLAW_LOG_DIR` handling to the Gateway logger and `OPENCLAW_TMP_DIR` handling to the trusted temporary-directory resolver.

## Best Possible Solution

Land a refreshed implementation that uses one shared default log-directory/file helper across the logger, status, and fallback surfaces; reads `OPENCLAW_LOG_DIR` after CLI dotenv hydration; applies the existing tmp-root trust checks to `OPENCLAW_TMP_DIR`; and adds focused tests plus docs for the accepted operator contract.

## Reproduction Assessment

Yes for the current-main gap: source inspection shows the TypeScript logger and temp resolver never read `OPENCLAW_LOG_DIR` or `OPENCLAW_TMP_DIR`. Yes for the PR integration concern: the PR changes the active logger default, while `src/logging/log-file-path.ts` would still compute status defaults from `/tmp/openclaw`.

## Solution Assessment

No as-is. The direction is narrow and fits the infra surface, but the best fix should share the log default resolver with `log-file-path.ts`, add regression coverage for both env vars and tmp-dir trust cases, and document the new env vars before merge.
