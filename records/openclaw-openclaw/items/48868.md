## Summary

Keep open: current main still lacks the Node-side log and temp directory env overrides, and this PR addresses a plausible operator need, but it is not mergeable as-is because it lacks real behavior proof, is conflicting, and leaves status log-file reporting on the old fallback path.

## What This Changes

The branch adds `OPENCLAW_LOG_DIR` fallback handling in the logger and `OPENCLAW_TMP_DIR` handling in the temp-dir resolver.

## Best Possible Solution

Rebase and implement the overrides through shared log-file and temp-dir resolvers, with trust checks, documentation, focused tests, and redacted real gateway proof.

## Reproduction Assessment

Yes, source-reproducible from code inspection: current main's Node logger and temp-dir resolver ignore these vars, and the PR diff leaves the status helper on the old temp fallback. I did not run the gateway because this review is read-only.

## Solution Assessment

No as-is. The direction is narrow, but the maintainable fix should centralize default log path resolution for logger and status surfaces, preserve `logging.file` precedence, document the env vars, and include real behavior proof.
