## Summary

Keep open: the reported Telegram/Discord reasoning-loss problem is valid, but this patch is too broad because it removes the shared guard that protects channels without reasoning lanes from receiving thinking text.

## What This Changes

The PR removes two `isReasoning` suppression guards from shared dispatch and changes dispatch tests to expect reasoning block/final payload delivery.

## Best Possible Solution

Keep default suppression for generic dispatch and add a narrow channel-owned opt-in or dispatcher contract so Telegram/Discord reasoning lanes receive `isReasoning` while unsupported channels continue dropping it.

## Reproduction Assessment

Yes, source-level: current main suppresses `isReasoning` in shared dispatch, while the PR deletes those guards and changes WhatsApp/generic tests to expect delivery. Signal's shared dispatcher sends payload text without checking `isReasoning`; I did not run live channel proof in this read-only review.

## Solution Assessment

No. The best fix is a narrow channel-owned reasoning lane opt-in; broad generic delivery reopens the class of channel reasoning leaks fixed earlier.
