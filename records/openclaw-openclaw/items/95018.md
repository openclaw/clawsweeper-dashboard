## Summary

Keep open: the reported Telegram durable-reasoning bug is source-reproducible, but this PR is too broad because it routes raw reasoning through generic shared dispatch and into shared TTS paths instead of preserving channel-owned reasoning boundaries.

## What This Changes

The PR removes shared dispatch guards so `isReasoning` block and final payloads are delivered through the generic dispatch path, and updates WhatsApp/generic tests to expect reasoning text delivery.

## Best Possible Solution

Use a Telegram-owned durable reasoning path that preserves generic-channel suppression, keeps reasoning out of shared TTS, strips `isReasoning` before outbound normalization, and leaves Discord to its already-merged fix.

## Reproduction Assessment

Yes, source-level: current main suppresses both block and final `isReasoning` payloads in shared dispatch before Telegram durable delivery can handle them. I did not run a live Telegram reasoning-model session in this read-only review.

## Solution Assessment

No. Removing the guards is too broad; the maintainable fix should allow only channel-owned reasoning lanes through, preserve generic suppression, and keep reasoning out of shared TTS.
