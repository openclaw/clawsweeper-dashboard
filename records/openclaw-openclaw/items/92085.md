## Summary

Keep open: current main still lacks a Slack progress visibility policy, and source inspection shows Slack progress/native task-card paths can still expose command text, local paths, tool args, approval text, and patch filenames. The protected maintainer label plus security/default/config compatibility choices make this a maintainer product/security item, not a cleanup close.

## What This Changes

Add a Slack progress visibility setting and sanitized display policy so Slack progress and native task cards can hide commands, paths, tool args, approval text, and patch details by default.

## Best Possible Solution

Implement a Slack-owned visibility policy at the structured progress formatter boundary, default Slack to safe output, keep explicit debug/detail opt-in, and cover native task cards plus rich/text fallback with schema, docs, upgrade, and formatter tests.

## Reproduction Assessment

Yes. Source inspection on current main shows no Slack visibility config and shows tool args, approval commands, and patch paths can be normalized into ChannelProgressDraftLine detail/text and rendered by Slack task-card, rich, and text progress paths.

## Solution Assessment

Yes for the broad fix shape: the clean solution is Slack-owned config/defaulting with enforcement at the structured progress formatter/display boundary. Maintainers still need to approve the exact enum, default, and legacy commandText compatibility mapping.
