## Summary

## What This Changes

The issue requests a Slack-safe progress visibility mode so Slack progress drafts and native task cards can hide raw commands, paths, tool args, approval strings, and other operational details by default.

## Best Possible Solution

Define one typed disclosure policy at the structured progress boundary, choose a conservative Slack default with explicit trusted debug opt-in, and cover native task-card plus text/rich Slack progress with fresh-install and upgrade tests.

## Reproduction Assessment

Yes, at source level: current main routes progress detail and fallback text into Slack-visible task titles, and the config schema has no visibility mode. I did not run a live Slack workspace reproduction.

## Solution Assessment

No, not as a ready implementation plan: the secure-by-default direction is right, but maintainers still need to choose the smallest durable disclosure model, default behavior, and backward mapping for `commandText`.
