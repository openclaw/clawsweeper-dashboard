## What This Changes

## Best Possible Solution

Have maintainers choose the durable disclosure model, then implement a Slack-scoped policy at the structured progress formatter boundary with schema, UI hint, docs, upgrade/default, and renderer/dispatch coverage.

## Reproduction Assessment

Yes, at source level: current main routes structured progress detail into Slack task-card titles and rich/text progress while config lacks a visibility setting. I did not run a live Slack workspace reproduction in this read-only review.

Unclear as a final implementation: the secure-by-default direction is aligned, but maintainers still need to choose the config enum, Slack default, and backward mapping for existing commandText behavior.
