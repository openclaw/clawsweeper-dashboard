## Summary

Keep open: current main and the latest release still lack a Slack-safe progress visibility mode, and source inspection confirms Slack progress/native task rendering can still display tool, approval, command-output, and patch details. The protected `maintainer` label plus the security-sensitive Slack default/config choice make this a maintainer product/security decision, not cleanup-closeable.

## What This Changes

The issue requests a Slack-specific progress visibility mode that hides raw commands, paths, tool args, approval text, and patch details by default while retaining opt-in detailed debug output.

## Best Possible Solution

Define a typed Slack disclosure policy at the structured progress formatting boundary, default Slack to safe output, keep explicit debug/detail opt-in, and cover native task cards plus rich/text fallback with schema, docs, upgrade, and formatter tests.

## Reproduction Assessment

Yes, at source level: current main has no Slack progress visibility setting, and progress details from tools, approvals, and patches still feed Slack task titles and rich/text details. I did not run a live Slack workspace in this read-only review.

## Solution Assessment

Yes as a product direction, but maintainers still need to approve the exact default and compatibility mapping. The clean implementation belongs at the shared progress-line formatter boundary with Slack-specific config/defaulting, not in Slack-only regex redaction.
