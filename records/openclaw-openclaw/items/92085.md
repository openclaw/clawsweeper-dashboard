Keep open: current main and v2026.6.8 still lack a Slack progress visibility policy, and source inspection shows Slack native task cards plus rich/text progress can still display tool details, approval commands, patch/file paths, and other operational detail. The protected maintainer label and security-sensitive default/config choice make this a maintainer product/security item, not cleanup-closeable.

## What This Changes

Request to add a Slack progress visibility mode that defaults shared Slack progress/task-card UI to safe output while preserving an explicit detailed/debug opt-in.

## Best Possible Solution

Implement a Slack-owned visibility policy at the structured progress formatter boundary, default Slack to safe output, keep explicit debug/detail opt-in, and cover native task cards plus rich/text fallback with schema, docs, upgrade, and formatter tests.

## Reproduction Assessment

Yes. Source inspection shows no Slack progress visibility setting, while tool args, approval commands, patch file lists, and progress details still feed Slack-visible native task-card titles and rich/text progress details.

## Solution Assessment

Yes for the broad direction, but maintainers still need to approve the exact safety levels, Slack default, and legacy commandText compatibility mapping. The clean fix belongs at the shared progress-line formatter boundary with Slack-specific config/defaulting, not in Slack-only regex redaction.
