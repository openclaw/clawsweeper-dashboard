## Summary

Keep open: current main and v2026.6.9 still lack a Slack progress visibility policy, and source inspection shows Slack native task cards plus rich/text progress can still display command text, tool args, approval text, patch/file paths, and other operational detail. The protected maintainer label and security-sensitive config/default choice make this a maintainer product/security item, not a cleanup close.

## What This Changes

Request a Slack-owned progress visibility setting/default that keeps Slack progress and native task-card surfaces from showing commands, paths, tool args, approval text, and patch details unless explicitly opted into debug output.

## Best Possible Solution

Implement a Slack-owned visibility policy at the structured progress formatter boundary, default Slack to safe output, keep explicit debug/detail opt-in, and cover native task cards plus rich/text fallback with schema, docs, upgrade, and formatter tests.

## Reproduction Assessment

Yes. Source inspection shows no Slack progress visibility setting, while tool args, approval commands, patch file lists, and progress details still feed Slack-visible native task-card titles and rich/text progress details.

## Solution Assessment

Yes for the broad direction, but maintainers still need to approve the exact safety levels, Slack default, and legacy commandText compatibility mapping. The maintainable fix belongs at the shared progress-line formatter boundary with Slack-specific config/defaulting, not in Slack-only string redaction.
