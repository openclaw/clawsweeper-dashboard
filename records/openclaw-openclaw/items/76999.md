## What This Changes

The branch adds a Telegram pre-IR list-boundary spacing normalizer and regression tests for numbered-section boundaries and fenced code.

## Best Possible Solution

Keep the Telegram-local spacing fix, but make the normalizer leave all code blocks unchanged, add indented-code regression coverage, and add a single-line Unreleased/Fixes changelog entry.

## Reproduction Assessment

Yes. The linked issue gives exact input, expected output, actual output, and current main routes Telegram markdown through `markdownToTelegramHtml`/`markdownToTelegramChunks`; I verified that path from source inspection.

## Solution Assessment

No. A Telegram-local pre-IR normalization is a reasonable boundary, but this implementation should skip indented code blocks before mutating Markdown and include the required changelog entry.
