## Summary

Keep this PR open: it targets a real Telegram formatting regression, but the patch is not merge-ready because the pre-parser can alter indented code blocks and the user-facing fix lacks the required changelog entry.

## What This Changes

The PR adds Telegram-local pre-IR list-boundary spacing normalization to markdown HTML/chunk rendering and adds formatter regression tests for numbered-section spacing and fenced code.

## Best Possible Solution

Keep the Telegram-local spacing fix, but make the normalizer skip both fenced and indented code blocks, cover that regression, and add a single-line Unreleased/Fixes changelog entry.

## Reproduction Assessment

Yes. The linked issue gives exact input and expected output, and source inspection shows outbound Telegram markdown routes through markdownToTelegramHtml and markdownToTelegramChunks; I did not execute Vitest in this read-only review.

## Solution Assessment

No. A Telegram-local pre-IR normalization is a reasonable boundary, but this implementation is not syntax-aware enough because it can alter indented code blocks before Markdown parsing.
