## Summary

Keep open: this PR targets a valid Telegram formatting bug, but the current diff can change literal indented code block content before Markdown parsing and is missing the required changelog entry.

## What This Changes

The PR adds a Telegram-local pre-IR markdown normalizer and formatter tests to preserve blank lines between indented bullet blocks and following numbered sections.

## Best Possible Solution

Keep the Telegram-local spacing repair, but make it syntax-aware for both fenced and indented code blocks, add regression coverage for the indented-code case, and add one Unreleased/Fixes changelog line.

## Reproduction Assessment

Yes. The linked issue provides exact input and expected output, and current-main source shows Telegram outbound markdown goes through the affected formatter path; I did not execute Vitest in this read-only checkout.

## Solution Assessment

No. A Telegram-local pre-IR normalization is a reasonable boundary, but this implementation is not yet syntax-aware enough because it can alter indented code blocks before Markdown parsing.
