## Summary

Keep open: this is a focused Telegram rendering bug fix with sufficient real Telegram proof, but the PR still has a correctness blocker because the pre-IR normalizer can modify literal indented code blocks before Markdown parsing.

## What This Changes

The PR adds a Telegram formatter pre-normalizer and regression tests so manually indented bullet blocks keep a blank line before following numbered sections or headings in normal and chunked rendering.

## Best Possible Solution

Keep the Telegram-local spacing fix, but make the normalizer skip both fenced and indented code blocks before Markdown IR parsing and add a regression test for the indented-code case.

## Reproduction Assessment

Yes. The linked issue gives exact input, current main source routes Telegram markdown through markdownToTelegramHtml/markdownToTelegramChunks, and the May 10 Crabbox Telegram proof compares main against the PR head with the missing versus preserved blank line.

## Solution Assessment

No. A Telegram-local pre-IR normalizer is a reasonable boundary, but this implementation is not yet the best fix because it is not syntax-aware for indented code blocks.

## Review Findings
