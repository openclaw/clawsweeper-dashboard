## Summary

The patch is a focused and technically sound fix for the linked released regression: OpenClaw currently removes its shell tools when the Codex native surface is selected, while Codex 0.139.0 exposes no native shell with zero turn environments. The PR should remain open because its supplied evidence is targeted test output rather than the required after-fix execution in a real Codex app-server or isolated-cron setup.

## What This Changes

The PR restores policy-filtered OpenClaw `exec` and `process` tools for sandbox-off Codex native-surface turns and adds regression coverage for direct and sandbox-backed paths.

## Best Possible Solution

Keep the narrow plugin-local restoration as proposed, then merge after a redacted live OpenAI/Codex direct-runtime turn shows `exec` successfully executes while a sandbox-backed turn still uses Codex native shell.

## Reproduction Assessment

Yes, source-reproducible with high confidence: current OpenClaw removes its shell tools for the native surface, and pinned Codex 0.139.0 adds no native shell when the turn has zero environments. The linked issue also supplies a concrete released-version cron reproduction and a Google-provider control.

## Solution Assessment

Yes; restoring the existing policy-aware OpenClaw shell tools only when no Codex environment exists is narrower than changing provider routing, disabling the native surface, or fabricating an environment. The branch preserves exclusions, allowlists, memory-flush behavior, and sandbox-native execution.
