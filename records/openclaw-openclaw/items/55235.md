## Summary

Keep open. Current main still only exposes manual schema printing and preserves existing root `$schema` values; it does not create or refresh `openclaw_schema.json`, default a missing schema reference, or define the requested `update.maintain_config_json_schema` opt-out. The issue is also paired with open PR #55390, so it should stay open until that implementation candidate is reviewed, landed, replaced, or closed.

## What This Changes

The issue requests automatic local generation and maintenance of `openclaw_schema.json` beside `openclaw.json`, including default `$schema` wiring, schema version metadata, an update opt-out, docs, and tests.

## Best Possible Solution

Implement a narrow, idempotent schema-maintenance path in the CLI/config update flow that writes `openclaw_schema.json` beside the active config, adds version metadata, sets a default local `$schema` only when absent or empty, honors custom schema URIs, exposes `update.maintain_config_json_schema`, and documents/tests the default, opt-out, rerun, and version-refresh behavior.
