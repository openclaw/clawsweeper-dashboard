## Summary

Keep open: current main and the latest release still provide the manual `openclaw config schema` export and preserve existing root `$schema` values, but they do not auto-generate or refresh a local schema sidecar. ## What This Changes

The issue requests automatic local generation and refresh of an `openclaw.json` JSON schema sidecar plus default `$schema` stamping during OpenClaw bootstrap/update or CLI config writes.

## Best Possible Solution

Review and refresh https://github.com/openclaw/openclaw/pull/89992 as the likely landing path if maintainers accept default-on local editor-schema breadcrumbs; otherwise choose an explicit opt-in/manual schema workflow and close or supersede the implementation PRs accordingly.
