## Summary

Keep open: current main and v2026.6.8 still provide manual `openclaw config schema` output and preservation of an existing root `$schema`, but they do not create or refresh a local schema sidecar or stamp a missing `$schema` during ordinary CLI/config writes. The remaining work is already represented by open implementation PRs, especially https://github.com/openclaw/openclaw/pull/89992, so the issue should stay open for maintainer product and compatibility review.

## What This Changes

The issue requests automatic generation and refresh of a local `openclaw.json` JSON schema sidecar, default `$schema` stamping, an opt-out update setting, and brief docs/tests.

## Best Possible Solution

Review and resolve https://github.com/openclaw/openclaw/pull/89992 as the likely landing path if maintainers accept default-on local editor-schema breadcrumbs; otherwise choose an explicit opt-in/manual schema workflow and close or supersede the stale implementation PRs.
