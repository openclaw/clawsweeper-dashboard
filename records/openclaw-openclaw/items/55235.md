## Summary

Keep open. Current main still only exposes manual schema printing and schema-preservation behavior; it does not create or refresh `openclaw_schema.json`, default a missing root `$schema`, or expose the requested update opt-out. The issue is also paired with open PR #55390 using closing syntax, so it should remain open until that implementation candidate is reviewed, landed, replaced, or closed.

## What This Changes

The issue requests automatic local generation and refresh of `openclaw_schema.json` beside `openclaw.json`, defaulting `$schema` when absent, and adding an opt-out config setting plus docs and tests.

## Best Possible Solution

Implement a narrow, idempotent schema-maintenance path that writes `openclaw_schema.json` beside the active config, records schema version metadata, adds `update.maintain_config_json_schema` with a default-on behavior, preserves custom schema URIs, and documents/tests defaulting, opt-out, rerun, and version-refresh cases.

## Reproduction Assessment

Not applicable as a feature request. Current-main source inspection is enough to verify the missing behavior: schema output is manual-only, the requested opt-out key and artifact name are absent, and bootstrap/update paths do not maintain a local schema file.

## Solution Assessment

Yes, the requested local artifact maintenance is a reasonable narrow solution for zero-configuration editor support, provided it is implemented as an idempotent config/schema helper that respects user-authored `$schema` values. The open PR should be reviewed against that boundary before this issue is resolved.
