## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Use parsed subcommand options for sections:** `src/cli/program/register.configure.ts:13`
  - body: `extractSurfaceSectionsFromRawArgs()` anchors on the first raw token equal to `configure`, so a valid invocation such as `openclaw --profile configure configure surface --json-out out.json --section channels` points at the profile value, never finds the real subcommand, and line 95 passes `[]`. That silently exports both default sections instead of the requested subset; use the parsed subcommand option or Commander command context instead of scanning all raw argv.
  - confidence: 0.86
- **[P2] Preserve disabled state for setup-wizard channels:** `src/setup/surface-export.ts:121-125`
  - body: When a channel has a setup adapter, this branch uses `adapter.getStatus()` and then copies the status unchanged even though the registry entry carries `enabled`. The common setup-wizard adapter does not know plugin disablement, so a disabled installed plugin can be exported as a normal configured/unconfigured channel rather than `plugin disabled`.
  - confidence: 0.84
- **[P2] Preserve installed catalog state:** `src/setup/surface-export.ts:267`
  - body: This catalog-only path filters out only loaded `registry.channelSetups` ids and then hardcodes every remaining catalog entry as `installed: false`. Current setup discovery distinguishes manifest-installed catalog channels that are not in `channelSetups`, so external UIs can be told to reinstall an already installed but not loaded channel instead of showing the installed/broken state.
  - confidence: 0.8
