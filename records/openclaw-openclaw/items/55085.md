## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Use parsed surface options instead of raw argv search:** `src/cli/program/register.configure.ts:95-99`
  - body: The action ignores Commander’s parsed `opts.section` and reparses `program.rawArgs`, so valid invocations with earlier option values like `configure` or later values like `--json-out surface` can anchor on the wrong token and silently export the default sections instead of the requested subset. A machine-readable export should use the parsed subcommand options, not raw token search.
  - confidence: 0.88
- **[P2] Preserve catalog channel installed state:** `src/setup/surface-export.ts:267-268`
  - body: Every catalog-only entry is serialized with `installed: false`, but catalog discovery includes manifest-backed plugin entries while `registry.channelSetups` only includes successfully registered setup surfaces. If an installed plugin is present but its setup entry is missing or failed to load, the export tells downstream UIs to install it instead of showing an installed-but-broken state.
  - confidence: 0.82
- **[P2] Expose provider configured state in the surface contract:** `src/setup/surface-types.ts:47-61`
  - body: The PR says external UIs need to know whether providers are configured, but `SetupSurfaceProviderEntry` has no configured/status field, so a UI cannot distinguish a ready provider from one that still needs auth. Add a provider state descriptor before making this a stable v1 contract.
  - confidence: 0.86
