Full review comments:

- **[P2] Preserve installed catalog state in surface exports:** `src/setup/surface-export.ts:267-268`
  - body: This catalog-only branch hardcodes `installed: false` for every entry not loaded through `registry.channelSetups`. Current setup discovery can distinguish packages already installed on disk but unavailable/not loaded, so this export can send UIs toward reinstalling an already-present channel instead of showing installed-but-unavailable state.
  - confidence: 0.86
- **[P2] Expose provider configured state:** `src/setup/surface-types.ts:47-61`
  - body: The PR advertises that external UIs can tell whether providers are configured, but provider entries only expose methods/env vars and no configured/status field. Consumers would have to reimplement provider auth detection outside the stable surface or treat already-ready providers as unauthenticated.
  - confidence: 0.78
- **[P3] Document the stable setup surface:** `src/cli/program/register.configure.ts:83-93`
  - body: This registers a stable CLI command and v1 machine-readable contract without public command or schema docs. External UI authors should not have to infer flags, output shape, and compatibility expectations from source before depending on the surface.
  - confidence: 0.9
