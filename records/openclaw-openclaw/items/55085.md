## Review Findings

Full review comments:

- **[P2] Stop parsing surface sections from raw argv:** `src/cli/program/register.configure.ts:13`
  - body: `extractSurfaceSectionsFromRawArgs()` anchors on the first raw token equal to `configure`, so a valid global option value such as `--profile configure` can make the parser miss the real subcommand and silently export default sections instead of the requested subset. Use Commander-parsed subcommand options rather than rescanning raw argv.
  - confidence: 0.9
- **[P2] Reflect disabled channels in adapter-backed status:** `src/setup/surface-export.ts:121-125`
  - body: When a setup adapter exists, this path ignores `entry.enabled` and passes through the adapter status unchanged. Disabled-but-configured channels can therefore look active to external UIs, while the fallback branch explicitly marks disabled channels.
  - confidence: 0.84
- **[P2] Keep setup discovery off broad runtime activation:** `src/setup/surface-export.ts:291-298`
  - body: The read-only export loads plugins with `activate: true`, which can execute plugin registration/runtime side effects just to generate metadata. This conflicts with the control-plane setup boundary and can make external UI discovery slower or behaviorally different from manifest/light-artifact discovery.
  - confidence: 0.78
