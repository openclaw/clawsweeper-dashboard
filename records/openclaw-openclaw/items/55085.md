## Summary

Keep this PR open for maintainer/API review: it proposes a new stable CLI JSON export, while current main already ships a Gateway wizard/session RPC that covers much of the GUI setup use case but does not implement this exact `configure surface --json-out` command.

## What This Changes

The PR adds a `configure surface` CLI subcommand, setup-surface JSON types/export logic, provider auth surface metadata, tests, and generated SDK baseline updates for provider/channel setup descriptors.

## Best Possible Solution

Decide whether the shipped Gateway wizard/session RPC is the canonical dynamic setup surface; if a separate offline CLI discovery export is still wanted, rebase this PR onto current main and document/version the contract before merge.
