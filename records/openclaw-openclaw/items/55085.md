## Summary

Keep open: current main still lacks the offline `configure surface` export, but this PR adds a new stable feature/API surface that needs maintainer direction, state fixes, public docs, and contributor real behavior proof before merge.

## What This Changes

The PR adds a new `openclaw configure surface --json-out` subcommand, setup-surface v1 JSON types/export logic, provider API-key surface metadata, tests, and generated Plugin SDK API baselines.

## Best Possible Solution

Decide whether a file-backed offline discovery export should exist alongside Gateway wizard RPC; if accepted, preserve real installed/configured/error states, document the v1 schema publicly, and require a real redacted command run before merge.

## Reproduction Assessment

Not applicable; this is a feature/API-surface PR rather than a bug report. Source inspection shows current main has Gateway wizard RPC but no `configure surface` exporter.

## Solution Assessment

Unclear. A file-backed export is plausible, but it overlaps with the documented Gateway wizard RPC GUI path and the current v1 schema loses important setup state, so the contract should be settled before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Preserve discovered plugin state in catalog exports:** `src/setup/surface-export.ts:263-268`
  - body: This path marks every catalog-only channel as not installed, but the catalog can include discovered plugin manifests that failed to register a setup entry because they are disabled or errored. External UIs would be told to install an already-present plugin instead of showing its disabled or broken state.
  - confidence: 0.86
- **[P2] Expose provider configured state:** `src/setup/surface-export.ts:57-80`
  - body: Provider entries only include labels, aliases, and auth methods, with no configured/status signal. A UI consuming the new stable surface cannot tell whether a provider already has auth via profile, config, or environment without reimplementing provider auth detection, which undermines the command's setup-discovery contract.
  - confidence: 0.78
- **[P3] Document the stable setup surface:** `src/cli/program/register.configure.ts:83-93`
  - body: This adds a stable CLI command and v1 JSON contract, but the public configure docs still only describe the interactive command. External UI authors should not have to infer `configure surface`, `--json-out`, allowed sections, and output shape from source before depending on it.
  - confidence: 0.9
