## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.83

Full review comments:

- **[P2] Avoid activating plugins during surface export:** `src/setup/surface-export.ts:297`
  - body: This command is described as a read-only setup-surface export, but it loads plugins with `activate: true`. In current loader behavior, activation clears/restores active plugin runtime state and enables activation-only registrations, so an external UI discovery command can execute broader plugin side effects just to write JSON. Use a non-activating discovery/setup load path for this surface.
  - confidence: 0.86
- **[P2] Parse surface sections from Commander options:** `src/cli/program/register.configure.ts:13`
  - body: `rawArgs.indexOf("configure")` can anchor to an earlier global option value instead of the real command. Since OpenClaw has `--profile <name>`, a profile named `configure` makes `openclaw --profile configure configure surface --json-out out --section channels` pass `section: []`, so the command exports the default provider+channel surface instead of the requested subset. Rely on the parsed subcommand options or Commander command context instead of searching raw argv.
  - confidence: 0.84
- **[P3] Document the new configure surface:** `src/cli/program/register.configure.ts:83-93`
  - body: This adds a stable user-facing CLI/API contract, but the PR does not add `docs/cli/configure.md` or `CHANGELOG.md` coverage. Without docs and a changelog entry, the new supported JSON surface is not discoverable and misses the repo policy for user-facing CLI changes.
  - confidence: 0.88
