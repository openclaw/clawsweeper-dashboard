## Summary

Keep open: current main has the related Gateway wizard RPC, but not this PR's file-backed `configure surface` export. The PR adds a useful new API surface, but it is not ready because the proposed stable schema loses installed/configured state, lacks public docs, needs maintainer product/API direction, and has no contributor real behavior proof.

## What This Changes

The PR adds `openclaw configure surface --json-out` plus setup-surface JSON types, provider/channel exporters, provider API-key surface metadata, CLI registration, tests, and generated Plugin SDK API baselines.

## Best Possible Solution

Decide the canonical public setup-discovery contract; if this CLI export is accepted, derive state from existing setup discovery/auth status, preserve installed/configured/disabled/error semantics, document the v1 schema publicly, and require redacted real command proof before merge.

## Reproduction Assessment

Not applicable; this is a feature/API-surface PR rather than a bug report. Source inspection shows current main has Gateway wizard RPC but no `configure surface` exporter.

## Solution Assessment

Unclear; a file-backed discovery export is plausible, but it overlaps with the documented Gateway wizard RPC path and the current schema loses important setup state. The safer path is to settle the public contract first, then preserve status semantics and docs in the implementation.
