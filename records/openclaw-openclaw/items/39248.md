## What This Changes

The issue asks for `sessions_spawn` child sessions to initialize, or fail visibly, when `agents.defaults.sandbox.mode` is `"non-main"` instead of being accepted and then stalling with no tokens or transcript.

## Best Possible Solution

Fix the native subagent startup path so `non-main` sandboxed child runs either initialize promptly or record/log a terminal failed state with actionable Docker startup details, including timeout/abort handling and Docker-backed regression proof.

## Reproduction Assessment

## Solution Assessment

Not applicable as a patch review. The narrow maintainable fix is to handle sandbox startup in the child lifecycle path and surface failures, not to disable `non-main` sandboxing or rely on a Docker-only diagnostics patch that leaves sessions_spawn untouched.
