## Summary

Keep open. Current main still routes native `sessions_spawn` child sessions through the `non-main` sandbox path before ordinary run progress, and the related startup/sandbox lifecycle fixes are still open or unmerged.

## What This Changes

The issue reports that `agents.defaults.sandbox.mode: "non-main"` causes native `sessions_spawn` subagents to be accepted but never start, with no transcript, tokens, or gateway error.

## Best Possible Solution

Add focused sandboxed-subagent startup/error handling plus a Docker-backed regression proving `non-main` native subagents either make transcript/token progress or transition to a visible actionable terminal failure.

## Reproduction Assessment

Yes. The report gives a concrete config and command matrix, and current main still classifies spawned child keys as non-main sandboxed before entering Docker sandbox setup on the pre-progress embedded-run path.

## Solution Assessment

Not applicable as a patch review. The narrow maintainable path is to coordinate with the open sandbox/lifecycle cluster, then add Docker-backed proof that the child starts or fails visibly.
