- `P1`: The issue blocks core subagent orchestration for sandboxed non-main sessions and leaves accepted child runs with no progress.

## Summary

Keep open: current main still supports the reported `non-main` sandboxed native `sessions_spawn` path, and the source still dispatches the child through the gateway before sandbox context startup is proven complete. The open lifecycle issue and candidate PR are relevant, but they only partially cover this sandbox-triggered initialization failure and no merged change or release evidence shows the exact accepted-but-stalled child run is fixed.

## What This Changes

The issue reports that `agents.defaults.sandbox.mode: "non-main"` makes `sessions_spawn` accept child runs that never initialize, emit messages, write transcripts, or surface gateway errors.

## Best Possible Solution

Keep this as the canonical core bug until sandboxed native child startup either completes normally or records/logs an actionable terminal failure when Docker or sandbox initialization fails.
