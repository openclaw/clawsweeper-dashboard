## Repair Work Prompt

Fix the canonical sandboxed sessions_spawn startup bug tracked at https://github.com/openclaw/openclaw/issues/39248. Current main creates generated child session keys that are sandboxed under `agents.defaults.sandbox.mode: "non-main"`, returns the gateway `agent` accepted frame before `resolveSandboxContext()`/Docker startup, and can leave the subagent run showing as running with zero messages/tokens when startup fails before prompt progress. Add focused regression coverage around the native child run lifecycle with a forced sandbox context/backend failure, plus Docker-backed proof for `non-main` sessions_spawn and a forced Docker create/start or setupCommand failure path.
