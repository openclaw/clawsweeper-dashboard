Keep open. Current main still routes native `sessions_spawn` children through the `non-main` sandbox path before normal run progress, and the related sandbox/lifecycle work remains open or unmerged, so there is not enough evidence to close this as fixed or safely duplicate it away.

## What This Changes

The issue reports that `agents.defaults.sandbox.mode: "non-main"` causes spawned subagents to be accepted but never initialize, log errors, create transcripts, or consume tokens.

## Best Possible Solution

Add focused startup lifecycle/error handling plus a Docker-backed regression proving `non-main` native subagents either make transcript/token progress or transition to a visible actionable terminal error.
