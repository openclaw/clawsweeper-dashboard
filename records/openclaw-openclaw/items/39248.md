## Summary

Keep open. Current main still treats spawned child sessions as `non-main` sandboxed runs, accepts the spawn before the child runner resolves Docker sandbox setup, and has no merged PR closing the broader startup-stall behavior; the overlapping Docker failure report at https://github.com/openclaw/openclaw/issues/57992 is still open.

## What This Changes

The issue reports that `agents.defaults.sandbox.mode: "non-main"` causes native `sessions_spawn` child sessions to be accepted but never initialize or surface errors.

## Best Possible Solution

Add focused sandboxed-subagent startup handling so Docker setup failures before first prompt progress are logged and reflected as terminal subagent/session state, while preserving `non-main` sandboxing and covering the path with lifecycle tests plus Docker-backed proof.

## Reproduction Assessment
