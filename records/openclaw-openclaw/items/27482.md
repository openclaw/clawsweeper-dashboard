## Summary

Current main still does not implement direct chat video upload or provider-native video delivery. The Control UI explicitly rejects video files, the UI test suite locks that rejection in, and the Gateway attachment contract still exposes native prompt images plus generic managed-media offloads rather than a native video chat content path. Open PR #74082 directly targets this feature, so the issue should remain open while maintainers review or land that PR instead of queueing duplicate repair work.

## What This Changes

The issue requests direct MP4/WebM/MOV video uploads in chat and native delivery to video-capable LLMs, following the existing image upload pattern.

## Best Possible Solution

Keep this issue open as the product tracker and land a maintainer-reviewed implementation that adds video-aware UI acceptance and preview, Gateway validation, model capability routing, provider-native video dispatch where supported, managed-media fallback behavior, docs, and regression tests; #74082 is the active candidate.
