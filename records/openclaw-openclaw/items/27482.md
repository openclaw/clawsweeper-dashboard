## Summary

Keep open: current main and the latest release still block Control UI video uploads, and the Gateway chat attachment contract has native image content plus generic media staging but no native chat-video/provider delivery path. Open implementation PRs already target the feature, so this should stay as the product tracker rather than spawn a duplicate fix branch.

## What This Changes

The issue requests direct video file upload support for video-capable LLM chat/model inputs, similar to existing image attachment handling.

## Best Possible Solution

Keep this issue as the canonical tracker and land or replace the open implementation with one end-to-end video-aware chat contract covering UI previews, Gateway validation/staging, model capability metadata, provider-native dispatch, fallback behavior, docs, and regression proof.
