## Summary

Keep this issue open: current main and v2026.6.8 still do not support direct WebChat video upload to native LLM video input, and the related implementation attempts are closed unmerged or still review-blocked rather than safe canonical replacements.

## What This Changes

Request direct upload of video files to LLM chat models, including UI validation/preview, provider-native delivery, and support for common formats such as MP4, WebM, and MOV.

## Best Possible Solution

Keep this issue as the canonical video tracker until maintainers approve and land one coherent video-aware chat/media contract across WebChat, Gateway validation/staging, model catalog and LLM content types, provider serializers, fallback behavior, docs, and tests.

## Reproduction Assessment

Yes, at source level: current main filters MP4/WebM/MOV/M4V before WebChat send, and the Gateway/LLM runtime contracts only provide native image prompt content plus generic offloaded media. I did not run a live browser upload in this read-only review.
