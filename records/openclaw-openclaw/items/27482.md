## Summary

Current main still does not provide direct video upload/native LLM delivery from chat: Control UI filters videos, docs describe chat uploads as images plus non-video files, and the Gateway/agent prompt path remains native-image plus generic media staging. Open PR #74082 is the active implementation candidate, so this issue should stay open while maintainers review or land that work.

## What This Changes

Request direct video file uploads in the chat/message path so video-capable LLMs receive MP4/WebM/MOV inputs natively with previews and validation.

## Best Possible Solution

Keep this issue open as the product tracker until https://github.com/openclaw/openclaw/pull/74082, or a maintainer-approved equivalent, lands with end-to-end UI acceptance/preview, Gateway validation, native/fallback routing, model capability metadata, provider transport support, docs/changelog, and regression tests.

## Reproduction Assessment

Yes. The observable gap is source-reproducible on current main: selecting an MP4 in Control UI chat is filtered by isSupportedChatAttachmentFile, and the existing test asserts that video/mp4 is rejected.

## Solution Assessment

Yes. Keeping this issue open while reviewing the active implementation PR is the best path because current main only has image-native prompt content plus generic managed-media staging, not a video-aware chat/provider contract.
