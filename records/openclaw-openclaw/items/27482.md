## Summary

Keep open: current main and v2026.5.12 still reject direct WebChat video uploads and do not expose a chat-native `video` model-catalog/provider contract. The active implementation work remains in open PRs with review/proof/product blockers, so this issue should remain the tracker rather than being closed or duplicated.

## What This Changes

Request support for direct video uploads to LLM chat models, including validation, MP4/WebM/MOV handling, preview UI, storage or base64 delivery, and provider-native video input routing.

## Best Possible Solution

Keep this issue as the canonical tracker and land or replace https://github.com/openclaw/openclaw/pull/74082 with an end-to-end, maintainer-approved video-aware chat contract covering UI previews, Gateway validation/staging, model video capability metadata, provider-native dispatch, fallback notes, docs, and regression tests.

## Reproduction Assessment

Yes, for the current feature gap: source inspection shows WebChat filters video before send, tests assert MP4 rejection, and Gateway/model-catalog contracts still lack a native chat video path. This is a feature request rather than a broken existing behavior bug.

## Solution Assessment

Yes: keeping this issue open while maintainers review, repair, or replace https://github.com/openclaw/openclaw/pull/74082 is the narrowest maintainable path. A duplicate repair branch would split the same UI, Gateway, provider, docs, and test work.
