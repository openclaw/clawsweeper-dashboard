## Summary

Keep open: current main still blocks direct Control UI video uploads and the chat/provider runtime only has native image delivery plus generic managed-media staging, not a native chat video contract. Open implementation work, especially https://github.com/openclaw/openclaw/pull/74082, is already targeting the gap, so this should remain the canonical product tracker rather than spawning a duplicate repair branch.

## What This Changes

The issue requests direct video upload support for LLM chat models, including UI acceptance/previews, validation, native provider delivery, and fallback handling.

## Best Possible Solution

Keep this issue as the canonical tracker and converge the open implementations into one end-to-end video-aware chat contract covering UI previews, Gateway validation/staging, model capability metadata, provider-native dispatch, fallback notes, docs, and regression proof.

## Reproduction Assessment

Yes, source-level: current main filters video before chat.send, the UI test suite asserts MP4 rejection, and the Gateway/provider paths only provide native image plus generic MediaPaths handling. I did not run runtime tests because this was a read-only review.
