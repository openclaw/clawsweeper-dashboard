## Summary

Keep open: current main still filters video files out of WebChat before `chat.send`, documents uploads as images plus non-video files, and lacks a native chat video attachment/model-capability contract. The open implementation work at https://github.com/openclaw/openclaw/pull/74082 is the right place to converge this feature instead of opening a duplicate repair branch.

## What This Changes

The request is to add direct video uploads for video-capable LLM chat models, including file validation, upload/encoding, native provider delivery, and chat UI preview support.

## Best Possible Solution

Keep this issue as the canonical tracker and land or replace the open implementation with one end-to-end video-aware chat contract covering UI previews, Gateway validation/staging, model capability metadata, provider-native dispatch, fallback notes, docs, and regression proof.

## Reproduction Assessment

Yes, source-level: current main filters video before `chat.send`, the UI test suite asserts MP4 rejection, and Gateway/model catalog contracts do not expose native chat video inputs. I did not run runtime tests because this was a read-only review.
