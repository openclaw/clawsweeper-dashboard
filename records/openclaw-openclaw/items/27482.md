## Summary

Keep open: current main still rejects WebChat video uploads before send and has no native chat-video content path for LLM delivery. The active implementation candidate, https://github.com/openclaw/openclaw/pull/74082, remains open and directly targets this request, with review feedback still pending.

## What This Changes

The issue requests chat support for uploading video files and delivering them directly to LLM models that advertise native video understanding.

## Best Possible Solution

Land or replace the active implementation with an end-to-end video-aware contract covering UI previews, Gateway validation/staging, model video capability metadata, provider-native dispatch, fallback notes, docs, changelog, and regression tests.

## Reproduction Assessment

Yes for the missing capability: source inspection shows video uploads are filtered before send, tests assert MP4 rejection, and the Gateway/model contract has image-native support but no chat video input path. I did not run tests because this review is read-only.

## Solution Assessment

Yes: keeping this issue open while maintainers review https://github.com/openclaw/openclaw/pull/74082 is the narrowest maintainable path. A duplicate repair lane would split review across the same UI, Gateway, model-catalog, provider, docs, and test surfaces.
