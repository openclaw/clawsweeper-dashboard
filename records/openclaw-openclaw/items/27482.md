## Summary

Keep open: current main still rejects direct WebChat video uploads and has no native chat video dispatch path. The active implementation candidate, https://github.com/openclaw/openclaw/pull/74082, remains open and needs review/repair before this issue can be considered implemented.

## What This Changes

The issue requests direct video upload support in chat/WebChat so video-capable LLMs can receive MP4/WebM/MOV-like media natively instead of requiring manual frame extraction.

## Best Possible Solution

Keep this issue open until https://github.com/openclaw/openclaw/pull/74082, or a maintainer-approved equivalent, lands end-to-end video-aware UI, Gateway validation/staging, model capability routing, provider transport, docs/changelog, and regression coverage.

## Reproduction Assessment

Yes for the current missing capability: source inspection shows MP4/WebM/MOV/M4V are filtered by Control UI, current tests assert video rejection, and `chat.send` only resolves image-native model support. I did not run tests because this review is read-only.

## Solution Assessment

Yes: keeping this issue open while maintainers review the active implementation PR is the narrowest maintainable path. Current main only has image-native prompt content plus generic managed-media staging, not direct video upload to LLM chat models.
