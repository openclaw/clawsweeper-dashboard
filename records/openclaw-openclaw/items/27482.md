## Summary

Keep open: current main still filters WebChat video uploads and only routes native prompt media as images plus generic managed-media staging. The active implementation candidate, https://github.com/openclaw/openclaw/pull/74082, remains open and directly targets this request.

## What This Changes

Request direct MP4/WebM/MOV-style video uploads from WebChat/message tooling to LLM models that support native video understanding.

## Best Possible Solution

Land or replace the active implementation with an end-to-end video-aware contract covering UI previews, Gateway validation/staging, model video capability metadata, provider-native dispatch, fallback notes, docs, changelog, and regression tests.

## Reproduction Assessment

Yes for the current missing capability: source inspection shows video uploads are filtered before send, tests assert MP4 rejection, and the Gateway/model contract has image-native support but no chat video input path. I did not run tests because this review is read-only.

## Solution Assessment

Yes: keeping this issue open while maintainers review https://github.com/openclaw/openclaw/pull/74082 is the narrowest maintainable path. The requested behavior spans UI, Gateway, model capability, provider transport, docs, and tests, so it should not be closed or replaced by a duplicate repair lane yet.
