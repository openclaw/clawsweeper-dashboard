## Summary

Current main and v2026.6.6 still reject chat video uploads, while existing video media understanding does not provide direct native chat delivery. Two conflicting open PRs propose overlapping architectures, so the issue remains the appropriate product tracker pending a maintainer contract decision.

## What This Changes

Support direct video uploads in chat with preview, validation, native delivery to compatible models, and fallback for unsupported models.

## Best Possible Solution

Define one typed text/image/audio/video capability contract shared by model catalogs and provider transports, then add validated Control UI video previews and native dispatch with managed-media fallback when the final model lacks support.

## Reproduction Assessment

Yes, at source level: Control UI explicitly rejects `video/*` and common video extensions, and the central native model-input type lacks video. A live upload was unnecessary for establishing the missing current-main contract in this read-only review.

## Solution Assessment

No, following only the existing image-upload pattern is incomplete. The best solution must reconcile chat content blocks with existing media-understanding capabilities and define provider-native delivery, validation, and fallback together.
