## Summary

Current main still does not implement direct WebChat video upload or native video delivery to chat LLMs: the UI rejects video files, docs describe uploads as images plus non-video files, and Gateway routing remains image-native plus generic managed-media staging. Open PR #74082 is the active implementation candidate, so this should stay open for maintainer review rather than duplicate repair work.

## What This Changes

The issue requests direct MP4/WebM/MOV video upload from chat and native delivery to LLM models that support video input.

## Best Possible Solution

Keep this issue open until #74082, or a maintainer-approved equivalent, lands end-to-end video-aware UI, Gateway validation/staging, model capability metadata, provider transport, docs/changelog, and regression coverage.

## Reproduction Assessment

Yes. Source inspection of current main shows an MP4 is filtered by `isSupportedChatAttachmentFile`, and the existing UI test explicitly locks in `video/mp4` rejection; I did not run tests because this review is read-only.

## Solution Assessment

Yes. Keeping the issue open while maintainers review the active implementation PR is the best path because current main only has image-native prompt content plus generic managed-media staging, not a video-aware chat/provider contract.
