## Summary

Keep open: current main and the latest shipped release still reject WebChat video attachments and lack a chat-native `video` model-catalog/provider contract, while the direct implementation work remains open with review blockers.

## What This Changes

Request support for direct video uploads to video-capable LLMs, including file detection, validation, preview, upload/offload, and provider-native delivery.

## Best Possible Solution

Keep this issue as the canonical tracker and land or replace https://github.com/openclaw/openclaw/pull/74082 with an end-to-end video-aware chat contract covering UI previews, Gateway validation/staging, model video capability metadata, provider-native dispatch, fallback notes, docs, and regression tests.

## Reproduction Assessment

Yes for the current feature gap: source inspection shows WebChat filters videos before send, tests assert MP4 rejection, and Gateway/model-catalog contracts still have no native chat video path. This is a feature request, not a broken existing behavior bug.

## Solution Assessment

Yes: keeping this issue open while maintainers review, repair, or replace https://github.com/openclaw/openclaw/pull/74082 is the narrowest maintainable path. A duplicate repair branch would split the same UI, Gateway, provider, docs, and test work.
