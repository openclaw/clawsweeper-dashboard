## Summary

Keep open: current main and the latest release still do not support direct video upload to LLM chat models, and the related implementation attempts are either closed unmerged or still blocked on native media/provider contract decisions.

## What This Changes

The issue requests direct video upload support for LLM chat/model inputs, including UI acceptance, validation, previews, provider-native delivery, and MP4/WebM/MOV support.

## Best Possible Solution

Keep this issue open as the canonical direct-video-upload tracker, and converge any future implementation on a maintainer-approved typed media/provider contract that covers UI, Gateway validation, model capability metadata, provider serialization, fallback behavior, docs, tests, and raw-media policy.

## Reproduction Assessment

Yes, at source level: current main filters video files in the Control UI and lacks a native video model-input contract for chat routing. I did not run a live WebChat upload because the current UI and tests already lock the rejection path.

## Solution Assessment

No, not as a simple image-upload-pattern extension. The maintainable solution needs a shared typed media/provider contract, UI/Gateway validation and previews, fallback behavior, provider serialization, and explicit raw-media policy.
