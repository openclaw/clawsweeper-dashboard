## Summary

Keep open: current main and the latest release still reject Control UI video attachments and lack a chat-native video prompt/provider contract. The active work is already represented by open implementation PRs, especially https://github.com/openclaw/openclaw/pull/74082, so this should remain the canonical product tracker rather than spawning duplicate repair work.

## What This Changes

The issue requests direct MP4/WebM/MOV-style video uploads from chat to LLM models that support native video input.

## Best Possible Solution

Keep this issue as the canonical tracker and converge the open implementations into an end-to-end video-aware chat contract covering UI previews, Gateway validation/staging, model capability metadata, provider-native dispatch, fallback notes, docs, and regression proof.

## Reproduction Assessment

Yes for the current feature gap: source inspection shows WebChat filters video before send, tests assert MP4 rejection, and the Gateway/provider/model-catalog paths still have no native chat video delivery. This is a feature request, not a broken existing behavior bug.

## Solution Assessment

Yes: keeping this issue open while maintainers review, repair, or replace https://github.com/openclaw/openclaw/pull/74082 is the narrowest maintainable path. A duplicate repair branch would split the same UI, Gateway, model-catalog, provider, docs, and test work.
