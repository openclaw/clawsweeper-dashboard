## Summary

Current main still does not implement direct video upload/native LLM delivery from chat: the Control UI rejects video files, docs describe uploads as images plus non-video files, and Gateway/provider routing remains image-native plus generic media staging. Open PR #74082 is the active implementation candidate, so this issue should stay open for review or landing rather than spawning duplicate repair work.

## What This Changes

Add direct MP4/WebM/MOV-style video upload support in chat, including validation, preview, model capability routing, provider-native delivery, and fallback behavior.

## Best Possible Solution

Keep this issue open as the product tracker until #74082, or a maintainer-approved equivalent, lands end-to-end video-aware UI, Gateway, model metadata, provider transport, docs/changelog, and regression coverage.

## Reproduction Assessment

Yes. The observable gap is source-reproducible on current main: selecting an MP4 in Control UI chat is filtered by `isSupportedChatAttachmentFile`, and the existing test asserts that `video/mp4` is rejected.

## Solution Assessment

Yes. Keeping this issue open while maintainers review the active implementation PR is the best path because current main only has image-native prompt content plus generic managed-media staging, not a video-aware chat/provider contract.
