## Summary

Keep open: current main still blocks WebChat video attachments and lacks a chat-native video/model-catalog contract, while the direct implementation candidate remains open with review blockers.

## What This Changes

The issue requests direct video upload support for LLM chat, including UI acceptance and previews, Gateway validation, model/provider video capability routing, and MP4/WebM/MOV-style formats.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/74082 with an end-to-end video-aware chat contract covering UI previews, Gateway validation/staging, model video capability metadata, provider-native dispatch, fallback notes, docs, and regression tests.

## Reproduction Assessment

Yes for the current feature gap: source inspection shows videos are filtered before send, tests assert MP4 rejection, and Gateway/model catalog chat contracts still have no native video path. This is a feature request rather than a broken existing behavior bug.

## Solution Assessment

Yes: keeping this issue open while maintainers review, repair, or replace https://github.com/openclaw/openclaw/pull/74082 is the narrowest maintainable path. A duplicate repair branch would split the same UI, Gateway, provider, docs, and tests work.
