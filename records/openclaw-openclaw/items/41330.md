## Summary

Keep open. Current main has substantial iMessage echo-loop hardening, but the sent-message cache still intentionally lets late text-only reflected replies miss after the 4-second fallback window, and that residual path is present in the latest release tag checked.

## What This Changes

The issue requests that iMessage inbound watching ignore or dedupe agent-sent replies so they are not re-ingested as new inbound messages.

## Best Possible Solution

Keep the shipped ID/from-me/prefix/reflection guards, then add a precise late-echo signal or narrowly scoped fallback that catches delayed text-only agent echoes without suppressing legitimate repeated self-chat messages.

## Reproduction Assessment

Yes, at source level: current tests and code show a text-only echo after 4.5 to 5 seconds is not matched and can continue toward dispatch. I did not run live iMessage traffic in this read-only review.

## Solution Assessment

Unclear. The current partial implementation is safer than broad long-TTL text filtering, but the remaining fix needs owner judgment because the obvious TTL increase conflicts with known self-chat false-positive risk.
