## Summary

Keep open. Current main has substantial iMessage echo-loop hardening, but it still codifies a text-only reflected-reply path that is allowed to miss after the 4-second fallback window, which matches the delayed echo reports in the discussion.

## What This Changes

The issue reports that the legacy iMessage `imsg` channel can re-ingest agent-sent messages as inbound traffic, causing duplicate replies or loops.

## Best Possible Solution

Keep the shipped ID/from-me/self-chat/reflection guards, then add a precise late text-only echo signal tied to agent-initiated sends or reliable `imsg` metadata without broadening suppression enough to drop legitimate self-chat messages.

## Reproduction Assessment

Yes at source level. Current code and tests show that a text-only reflected reply after 4.5 to 5 seconds is not matched by the echo cache and can dispatch as a duplicate when no ID or reflection marker catches it.

## Solution Assessment

Unclear. The current implementation is a safer partial fix than broad `is_from_me` or long-TTL filtering, but the remaining repair needs maintainer judgment to avoid reintroducing self-chat message loss.
