## Summary

Keep open. Current main and v2026.5.3 include several iMessage echo-loop guards, but the sent-message cache still intentionally misses delayed text-only echoes after the 4-second fallback window, leaving a source-reproducible duplicate-delivery path that matches the remaining reports.

## What This Changes

The issue requests that iMessage agent-sent replies be ignored or deduped by the inbound watcher so they are not re-ingested as new user messages.

## Best Possible Solution

Keep the shipped ID, from-me, self-chat, corruption-prefix, and reflection guards, then add a precise late text-only echo signal that catches delayed agent reflections without dropping legitimate repeated self-chat messages.

## Reproduction Assessment

Yes, at source level. Current tests assert that a text-only sent-message echo after 4.5 to 5 seconds is not matched, and the inbound code dispatches messages that miss all echo guards.

## Solution Assessment

Unclear. The current partial implementation is safer than broad long-TTL text filtering, but the remaining fix needs maintainer judgment on a more precise late-echo heuristic.
