## Summary

Keep open: current main and the latest tagged release still delete the Twitch handler map entry unconditionally, and the linked fixing PR is open but unmerged, so this issue should close only after that fix or an equivalent one lands.

## What This Changes

The issue requests that Twitch `onMessage` cleanup remove only the handler registered by that cleanup closure, not any newer handler for the same account key.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/84195 or an equivalent identity-guard fix with regression coverage, then close this issue with the merge commit and release provenance.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: register handler A, register handler B for the same account key, then call cleanup A and current main deletes handler B from the map.

## Solution Assessment

Yes. The narrow maintainable fix is to guard cleanup with handler identity and add regression coverage for stale cleanup after handler replacement.
