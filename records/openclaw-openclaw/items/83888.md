## Summary

Keep open: current main and v2026.5.19 still have the unconditional Twitch handler cleanup, and the linked closing PR is open but not merged.

## What This Changes

The issue requests guarding TwitchClientManager.onMessage cleanup so a stale cleanup cannot delete a newer handler for the same Twitch account key.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/84195 or an equivalent identity guard plus regression tests, then close this issue as implemented.

## Reproduction Assessment

Yes, from source inspection. Register handler A, register handler B for the same account key, then call cleanup A; current main deletes the shared key and inbound dispatch only runs when the key still has a handler.

## Solution Assessment

Yes. The narrow maintainable fix is a referential guard in the cleanup closure plus regression coverage; the open linked PR implements that shape and should be reviewed before this issue closes.
