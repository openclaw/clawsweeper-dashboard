## What This Changes

The issue requests guarding `TwitchClientManager.onMessage` cleanup so a stale unregister callback cannot delete a newer handler for the same Twitch account.

## Solution Assessment

Yes. The narrow maintainable fix is an identity check in the cleanup closure plus regression tests, which is exactly the direction of the open linked PR.
