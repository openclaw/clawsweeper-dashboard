## Summary

Keep open: current main and the latest release still have the unconditional Twitch handler cleanup, and the live closing PR is open but not merged. The issue should remain the tracker until https://github.com/openclaw/openclaw/pull/84195 lands or maintainers choose a different fix.

## What This Changes

The issue reports that Twitch `onMessage` cleanup can remove a newer handler for the same account key, causing subsequent inbound Twitch messages to be dropped.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/84195 if its guard and regression coverage hold up, or replace it with the same narrow handler-identity cleanup fix.

## Reproduction Assessment

Yes. From current source, registering a second handler for the same account key replaces the first, and calling the first cleanup then deletes the shared key without checking handler identity.

## Solution Assessment

Yes. The narrow maintainable fix is to delete only when `messageHandlers.get(key) === handler`, with a regression test proving stale cleanup does not remove the newer handler.
