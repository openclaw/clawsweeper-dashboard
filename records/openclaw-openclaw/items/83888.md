- `impact:message-loss`: A stale cleanup can remove the active Twitch inbound handler, causing later chat messages for that account to be dropped.

## Summary

Keep open: current `main` still has the unconditional Twitch handler cleanup, and https://github.com/openclaw/openclaw/pull/84195 is the open implementation candidate that should resolve this when merged.

## What This Changes

The issue reports that stale Twitch `onMessage` cleanup can delete a newer active handler for the same account key and drop later inbound messages.

## Best Possible Solution

Review and land or close https://github.com/openclaw/openclaw/pull/84195; the desired end state is an identity-guarded cleanup plus regression coverage for stale cleanup after handler replacement.

## Reproduction Assessment

Yes, from source inspection. Registering handler B for the same account key after handler A, then calling A's cleanup, deletes the shared key because cleanup does not compare handler identity.

## Solution Assessment

Yes. The narrow maintainable fix is to keep the one-handler-per-account model but guard deletion with `this.messageHandlers.get(key) === handler` and add focused regression coverage.
