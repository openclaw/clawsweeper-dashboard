## Summary

Keep open: current `main` and the latest release still have the unconditional Twitch handler cleanup, and the linked fix PR is open but not merged, so the issue should remain open until that PR lands or is replaced.

## What This Changes

The issue reports that Twitch `onMessage` cleanup removes whatever handler is currently stored for an account key instead of only removing the handler registered by that cleanup.

## Best Possible Solution

Land or replace the focused identity-guard fix with regression coverage, then close this issue as fixed by the merged PR.

## Reproduction Assessment

Yes, from source inspection. Register handler A, register handler B for the same Twitch account key, then call cleanup A; current `main` deletes the shared key and removes handler B.

## Solution Assessment

Yes. The narrow maintainable solution is to delete only when `messageHandlers.get(key) === handler`, preserving single-handler cleanup while making stale cleanup a no-op.
