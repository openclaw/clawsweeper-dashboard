## Label Justifications

- `P2`: The bug can silently drop Twitch inbound messages, but the affected surface is plugin-scoped and the fix path is narrow.
- `impact:message-loss`: A stale cleanup can remove the active Twitch message handler and cause later chat messages for that account to be lost.

## Summary

Keep open: current `main` and the latest release still contain the unconditional Twitch handler cleanup, and the issue is already paired with an open linked fix PR that should be reviewed or landed before this issue closes.

## What This Changes

The issue requests that Twitch `onMessage` cleanup remove only the handler registered by that cleanup closure, not whichever handler is currently stored for the same account key.

## Best Possible Solution

Review and land the linked Twitch fix PR if it remains correct, then let the merged commit close this issue with regression coverage.

## Reproduction Assessment

Yes, from source inspection. Register handler A, register handler B for the same account key, then call A's cleanup; current `main` deletes the shared map entry that now points at B.

## Solution Assessment

Yes. The narrow maintainable fix is a handler-identity guard in the cleanup closure plus regression coverage for stale cleanup after replacement and ordinary single-handler cleanup.
