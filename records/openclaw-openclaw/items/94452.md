## What This Changes

The PR changes `openclaw nodes approve <requestId>` to translate the specific stale node-pairing request error into an actionable message with current pending request IDs, plus CLI coverage.

## Best Possible Solution

Land the focused CLI rendering fix after normal maintainer/automerge gates, and keep broader gateway idempotency behavior in separate maintainer review.

The broader idempotent gateway-approval idea in https://github.com/openclaw/openclaw/pull/94532 should remain separate unless maintainers choose that semantic change.
