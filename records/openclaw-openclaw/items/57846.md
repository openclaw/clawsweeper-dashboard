## Summary

Keep open. Current main and v2026.5.19 still use one Mattermost mention gate for top-level channel posts and thread replies, and the related implementation PR remains open with correctness and proof blockers.

## What This Changes

Request a Mattermost thread-specific mention policy or config so top-level channel messages can require @mention while bot-participated thread replies do not.

## Best Possible Solution

Implement a Mattermost-local implicit mention policy that preserves top-level `requireMention`, accepts unmentioned thread replies only with bot participation or root-post evidence, and aligns schema/docs/tests after the policy decision.

## Reproduction Assessment

Yes, by source inspection. A non-direct Mattermost thread reply with `threadRootId`, `requireMention=true`, `canDetectMention=true`, and no effective mention reaches `evaluateMattermostMentionGate` and returns `dropReason: "missing-mention"`.

## Solution Assessment

Unclear. The plugin-local direction is right, but maintainers still need to decide whether this should be a new `requireMentionInThread` config knob, a default participated-thread implicit mention policy, or another compatibility-preserving shape.
