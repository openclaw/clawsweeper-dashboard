## Summary

Keep open. Current main and the latest release still use the same Mattermost channel/group `requireMention` gate for threaded replies, while the related reply-to-bot report was closed as superseded by this canonical issue and the open implementation PR at https://github.com/openclaw/openclaw/pull/83446 still needs review changes and proof.

## What This Changes

Mattermost should support separate mention-gating behavior for top-level channel messages and thread replies.

## Best Possible Solution

Implement a Mattermost-local participated-thread implicit mention policy that preserves top-level `requireMention`, only accepts unmentioned replies when bot participation or root evidence proves the thread is directed at the bot, and aligns schema, docs, changelog, and regression tests.

## Reproduction Assessment

Yes, by source inspection. A non-direct Mattermost thread reply with `threadRootId`, `requireMention=true`, `canDetectMention=true`, and no effective mention reaches `evaluateMattermostMentionGate` and returns `dropReason: "missing-mention"`.

## Solution Assessment

Unclear. The plugin-local direction is right, but maintainers should decide the exact Mattermost policy/default/config shape before accepting an implementation that changes thread wake behavior.
