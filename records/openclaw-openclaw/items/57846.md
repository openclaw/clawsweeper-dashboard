## Summary

Keep open. Current main and v2026.5.19 still use one Mattermost `requireMention` gate for channel and thread replies, and the related implementation PR remains open with review/proof blockers rather than merged.

## What This Changes

The issue requests separate Mattermost mention gating so top-level channel messages can require @mentions while thread follow-ups after bot participation can continue without repeated mentions.

## Best Possible Solution

Implement a Mattermost-local participated-thread implicit mention policy that preserves top-level `requireMention`, only accepts unmentioned thread replies when bot participation or root evidence proves the thread is directed at the bot, and aligns schema, docs, and regression tests.

## Reproduction Assessment

Yes, by source inspection. A non-direct Mattermost thread reply with `threadRootId`, `requireMention=true`, `canDetectMention=true`, and no effective mention reaches `evaluateMattermostMentionGate` and returns `dropReason: "missing-mention"`.

## Solution Assessment

Unclear. The plugin-local direction is right, but maintainers still need to decide the exact Mattermost default/config policy before accepting a change that wakes thread replies without repeated mentions.
