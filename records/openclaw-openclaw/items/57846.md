## Summary

Keep open. Current main still applies the same Mattermost mention gate to top-level channel messages and thread replies, so this remains a real Mattermost plugin feature request rather than implemented cleanup.

## What This Changes

Add a Mattermost thread-level mention policy so top-level channel messages can require @mention while bot-participated thread replies do not.

## Best Possible Solution

Implement a Mattermost plugin-local implicit mention policy that preserves top-level `requireMention` while allowing unmentioned replies only in bot-participated or active Mattermost threads, with schema, docs, changelog, and regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a non-direct Mattermost message with `threadRootId` set, `requireMention` true, `wasMentioned` false, and `canDetectMention` true reaches `evaluateMattermostMentionGate` and returns `dropReason: "missing-mention"`.

## Solution Assessment

Unclear. The plugin-local direction is maintainable, but the exact Mattermost config name, default, and bot-participation semantics need maintainer product judgment before implementation.
