## Summary

Keep open. Current main still has one Mattermost mention-gating decision for channel/group traffic: `threadRootId` is carried to the gate, but the gate ignores it, and neither schema nor docs expose a thread-specific mention policy.

## What This Changes

Add Mattermost behavior/configuration that can require mentions for top-level channel messages while allowing unmentioned replies in an already bot-participated thread.

## Best Possible Solution

Implement this in the Mattermost plugin as a thread-aware implicit-mention policy that preserves top-level `requireMention` while allowing unmentioned replies only in bot-participated or active Mattermost thread sessions, with schema, docs, changelog, and regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a non-direct Mattermost message with `threadRootId` set, `requireMention` true, `wasMentioned` false, and `canDetectMention` true reaches `evaluateMattermostMentionGate` and returns `dropReason: "missing-mention"`.

## Solution Assessment

Yes. The narrow maintainable solution is plugin-local: use the existing Mattermost thread root/session facts, preferably through the shared implicit mention helper, instead of changing core channel behavior.
