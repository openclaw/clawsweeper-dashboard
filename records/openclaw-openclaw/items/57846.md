## Summary

Keep open. Current main and the latest release still gate Mattermost thread replies with the same top-level `requireMention` policy, and the related reply-to-bot issue was closed as superseded by this open canonical issue rather than as implemented.

## What This Changes

The issue requests a Mattermost mention policy split so channel messages can require an @mention while thread replies can continue without one after the bot has joined the conversation.

## Best Possible Solution

Implement a Mattermost-local thread implicit-mention policy that preserves top-level channel mention gating while allowing unmentioned replies only in bot-participated or active Mattermost thread sessions, with schema, docs, changelog, and regression tests aligned.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a non-direct Mattermost message with `threadRootId` set, `requireMention` true, `wasMentioned` false, and `canDetectMention` true reaches `evaluateMattermostMentionGate` and returns `dropReason: "missing-mention"`.

## Solution Assessment

Unclear. The plugin-local direction is right, but maintainers should decide the exact Mattermost config shape, default, and bot-participation semantics before implementation.
