## Summary

Keep open. Current main and v2026.5.19 still use the same Mattermost mention gate for top-level channel posts and thread replies, the related reply-to-bot report was closed in favor of this issue, and the open implementation attempt at https://github.com/openclaw/openclaw/pull/83446 is not merge-ready.

## What This Changes

Request Mattermost policy or config separation so top-level channel messages can require @mention while bot-participated thread replies can continue without repeated mentions.

## Best Possible Solution

Implement a Mattermost-local implicit mention policy that preserves top-level requireMention, accepts unmentioned thread replies only with bot participation or root-post evidence, and aligns schema, docs, and focused tests after the policy decision.

## Reproduction Assessment

Yes, by source inspection. A non-direct Mattermost thread reply with threadRootId, requireMention=true, canDetectMention=true, and no effective mention reaches evaluateMattermostMentionGate and returns dropReason "missing-mention".

## Solution Assessment

Unclear. The plugin-local direction is right, but maintainers still need to choose the policy/config shape and avoid a blanket bypass for unrelated followed threads.
