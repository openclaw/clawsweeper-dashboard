## Summary

Keep open. Current main still drops unmentioned Mattermost thread replies when `requireMention` is true, and the remaining work is a real Mattermost plugin feature rather than an implemented or duplicate cleanup item.

## What This Changes

The request is to separate Mattermost mention requirements for top-level channel messages from thread replies.

## Best Possible Solution

Implement a plugin-local Mattermost thread implicit-mention policy that preserves top-level `requireMention` while allowing unmentioned replies only in bot-participated or active Mattermost thread sessions, with schema, docs, changelog, and regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a non-direct Mattermost message with `threadRootId` set, `requireMention` true, `wasMentioned` false, and `canDetectMention` true reaches `evaluateMattermostMentionGate` and returns `dropReason: "missing-mention"`.

## Solution Assessment

Unclear. The plugin-local direction is right, but maintainers should decide the exact Mattermost config shape and default semantics before implementing it.
