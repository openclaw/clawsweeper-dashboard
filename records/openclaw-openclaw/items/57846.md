## Summary

Keep open. Current main still has one Mattermost mention-gating decision for channel/group messages: `threadRootId` is passed into the gate, but the gate does not use it to relax `requireMention`, and the config schema/docs expose no thread-specific mention policy.

## What This Changes

Request to let Mattermost require mentions in top-level channels while allowing unmentioned follow-up replies in bot-participated CRT threads.

## Best Possible Solution

Add a Mattermost-owned thread mention policy that preserves top-level `requireMention` while allowing unmentioned replies only in bot-participated or active Mattermost thread sessions, with schema, docs, and regression tests.

## Reproduction Assessment

Yes. A direct static reproduction is `evaluateMattermostMentionGate` with `kind: "channel"`, `requireMentionOverride: true`, `threadRootId` set, `wasMentioned: false`, and `canDetectMention: true`; current code returns `dropReason: "missing-mention"`.

## Solution Assessment

Yes, this is best solved inside the Mattermost plugin using the existing `threadRootId` and a participation/session check; a core change is not required unless maintainers decide to reuse the shared implicit-mention helper across channels.
