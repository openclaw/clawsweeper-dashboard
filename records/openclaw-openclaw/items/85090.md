## Summary

Keep open: the patch is a focused and plausible fix for the linked node-exec bug, and I found no blocking code-review defect, but it still lacks real multi-node behavior proof and changes a shell execution boundary that needs maintainer/security confirmation before merge.

## What This Changes

This PR disables Codex app-server native tool surfaces when the effective exec host is `node`, re-adds OpenClaw `exec`/`process` dynamic tools for that case, and adds focused `run-attempt` tests.

## Best Possible Solution

Land a version that makes effective `host=node` a single shell-routing contract for Codex app-server: native shell disabled or explicitly unsupported, OpenClaw node `exec`/`process` available, focused regression tests in place, and redacted multi-node proof attached.

Yes, at source level: current main enables Codex native Code Mode without checking `execOverrides` or configured `tools.exec.host=node`, while OpenClaw `exec` itself has documented node routing. I did not run the live Linux-gateway plus macOS-node reproduction in this read-only review.

## Solution Assessment

Unclear until maintainer/security signoff and live proof: the patch is narrow and aligns with the documented exec contract, but disabling native Code Mode for node-targeted Codex sessions is a product/security policy choice that should be confirmed before merge.
