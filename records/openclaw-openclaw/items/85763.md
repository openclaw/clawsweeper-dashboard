- **[P1] Persist automatic mode for queued automatic sends:** `src/infra/outbound/deliver.ts:1256`
  - body: Automatic durable sends rely on `undefined` meaning automatic in the live path, but this line persists that same `undefined` into the queue. Recovery now treats missing `sendPolicyMode` as `explicit`, so any new automatic delivery that is queued and later recovered will bypass `session.sendPolicy`, including the wrong-peer guard this PR is adding. Please persist an explicit `automatic` mode for new automatic entries while keeping the missing-field fallback only for legacy queue files.
  - confidence: 0.94
- **[P1] Preserve session sendPolicy overrides in delivery checks:** `src/infra/outbound/deliver.ts:1203-1214`
  - body: This new lower-level policy check re-runs `resolveSendPolicyDetailed` without the session entry. The resolver still gives `entry.sendPolicy` first priority, and `dispatch-from-config.ts` still passes that entry, so a stored per-session `allow` under a global default `deny` can pass the dispatch check and then be denied here during durable delivery. Please thread the entry override or the already-resolved policy contract into this path so existing session settings keep their upgrade behavior.
  - confidence: 0.9
