## Summary

This PR should stay open because the main predicate path is well developed, but current head still has a blocking replay-mode bug: new automatic queued deliveries can be written without `sendPolicyMode` and later recover as explicit, bypassing the policy boundary the PR is adding.

## What This Changes

The PR adds `session.sendPolicy.rules[].match.peerEquals: "inboundPeer"`, threads inbound peer context through automatic durable reply delivery, and documents/tests explicit-send compatibility.

## Best Possible Solution

Persist an explicit `automatic` mode for every new non-explicit queued delivery, keep missing-mode recovery as legacy explicit, and cover both replay paths with focused regressions.

## Reproduction Assessment

Yes. Source inspection shows an automatic delivery can enqueue with no mode and then recover through the missing-field `explicit` default; I did not run tests because this was a read-only review.

## Solution Assessment

No. The approach is close, but the queue contract must normalize new automatic entries before persistence so recovery cannot confuse them with legacy explicit entries.
