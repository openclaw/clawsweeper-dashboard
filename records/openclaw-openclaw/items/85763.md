## Label Justifications

- `P1`: The PR changes automatic outbound policy enforcement for real channel delivery and can break or bypass existing session-level send controls.
- `merge-risk: 🚨 compatibility`: The new delivery-layer check can change behavior for existing persisted `sendPolicy` overrides during normal and recovered delivery.
- `merge-risk: 🚨 message-delivery`: A policy mismatch between dispatch preflight and durable delivery can suppress messages that the existing session policy allows, or allow recovered sends that a session policy denies.
- `merge-risk: 🚨 security-boundary`: This PR is a privacy-routing guard, and missing session-row deny enforcement can weaken an operator’s configured outbound boundary.

## Summary

Keep open; the capability is useful and not already on main, but the current branch still has a blocking compatibility/security defect because the new delivery-layer check ignores persisted per-session `sendPolicy` overrides that existing code treats as authoritative.

## What This Changes

The PR adds `session.sendPolicy.rules[].match.peerEquals: "inboundPeer"`, threads inbound peer context through automatic/durable reply delivery, stores replay mode for queued deliveries, and documents the new predicate.

## Best Possible Solution

Land the relational predicate only after automatic delivery and queued replay carry the resolved per-session `sendPolicy` override into the delivery-layer policy check while explicit sends continue to bypass `session.sendPolicy`.

## Reproduction Assessment

Yes, source-reproducible: current main documents and tests that `entry.sendPolicy` overrides global config, while the PR’s new delivery-layer check calls `resolveSendPolicyDetailed` without any entry or override. The failing path is a global deny with a session allow, or recovered automatic delivery with a session deny.

## Solution Assessment

No, not yet. The predicate direction is maintainable, but the durable/recovery enforcement must use the same session-policy precedence as dispatch preflight before this is the best fix.
