Full review comments:

- **[P1] Honor per-session sendPolicy in durable delivery:** `src/infra/outbound/deliver.ts:1203-1214`
  - body: The existing `/send on|off|inherit` flow stores an override on `SessionEntry.sendPolicy`, and `resolveSendPolicyDetailed` only honors that override when an entry is supplied. This new delivery-time gate calls it without the entry or an equivalent copied override, so a session explicitly allowed under global `default: "deny"` can still be blocked when the durable path reaches delivery. Carry the effective session override into `OutboundSessionContext` or pass the already-resolved policy decision into delivery.
  - confidence: 0.9
- **[P1] Persist automatic replay mode before queueing:** `src/infra/outbound/deliver.ts:1256`
  - body: Automatic sends use an omitted `sendPolicyMode` in the live path, but this queue write stores that omission; recovery then treats a missing mode as `explicit` for legacy compatibility. Any new automatic delivery that survives to recovery can therefore bypass `session.sendPolicy`, contrary to the PR's recovered-automatic-delivery contract. Persist `params.sendPolicyMode ?? "automatic"` for new entries and add a recovery regression that still keeps true legacy entries explicit.
  - confidence: 0.92
