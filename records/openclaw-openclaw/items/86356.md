The issue requests safer inferred-commitment delivery by revalidating current source/thread context before sending visible open-loop follow-ups.

## Best Possible Solution

Keep this issue as the canonical report until a reviewed fix lands, preferably via https://github.com/openclaw/openclaw/pull/81382 or a narrower replacement that proves completed or superseded open-loop commitments are dismissed before visible delivery.
