## Summary

Keep this open: current main and v2026.6.8 still deliver due inferred commitments from stored metadata without source-context replay or tools, and the direct candidate fix is still open and conflicting.

## What This Changes

The issue asks OpenClaw to revalidate source/session context before sending inferred open-loop commitment follow-ups so completed or superseded items are suppressed.

## Best Possible Solution

Land or replace the open candidate with a maintainer-approved revalidation path that dismisses completed or superseded commitments before visible delivery and preserves enough reply context for any follow-up that is sent.

## Reproduction Assessment

Yes, source-reproducible. Current main and v2026.6.8 build commitment delivery from stored metadata, omit source conversation replay, and disable tools; the issue comments add live Slack and Telegram recurrences.

## Solution Assessment

Yes as a direction, but not yet landed. Pre-send revalidation and suppression are the right boundary, while https://github.com/openclaw/openclaw/pull/81382 remains the visible candidate and still needs refresh plus maintainer review.
