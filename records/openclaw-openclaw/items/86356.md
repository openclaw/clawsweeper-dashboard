## Summary

Keep this open: current main and the latest release still deliver due inferred commitments from stored metadata with tools disabled and without replaying source context, while the direct implementation candidate remains open and conflicting.

## What This Changes

The issue asks for inferred commitment delivery to revalidate source/session context before sending stale open-loop follow-ups.

## Best Possible Solution

Land a maintainer-approved revalidation path that suppresses completed or superseded inferred follow-ups before visible delivery and preserves enough session context for any follow-up that is sent.
