## Summary

Keep open: current main and v2026.6.9 still deliver due inferred commitments from stored metadata without source-context replay or tools, and the direct candidate fix is still open and conflicting.

## What This Changes

Commitment delivery should revalidate source/session context and suppress stale inferred open-loop follow-ups before visible heartbeat sends.

## Best Possible Solution

Land or replace the open candidate with a maintainer-approved revalidation path that dismisses completed or superseded commitments before visible delivery and preserves enough context for any follow-up that is sent.

## Reproduction Assessment

Yes, source-reproducible. Current main and v2026.6.9 build commitment delivery from stored metadata, omit source conversation replay, and disable tools; the issue comments add live Slack and Telegram recurrences.

## Solution Assessment

Yes as a direction, but not yet landed. Pre-send revalidation and suppression are the right boundary, while the open candidate still needs refresh plus maintainer review.
