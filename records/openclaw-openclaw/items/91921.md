- **[P2] Keep exec completions out of generic heartbeat-only repair:** `src/auto-reply/heartbeat-filter.ts:328-329`
  - body: This makes every exec-completion user body count as a heartbeat user message. `doctor-heartbeat-main-session-repair` uses that same classifier to decide that a session with no human user messages is safe to repair/archive, and it does not require assistant output to be only `HEARTBEAT_OK`. That means a transcript containing `[OpenClaw exec completion]` plus a substantive assistant result can be treated as disposable heartbeat-only state, which conflicts with the PR's goal of preserving used exec results for future context. Please either keep exec completion matching out of the generic heartbeat classifier or update the doctor/repair path to reject substantive exec-completion transcripts.
