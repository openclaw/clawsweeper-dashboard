## Label Justifications

- `P1`: The underlying bug breaks agent continuation after background commands, a primary asynchronous execution workflow.
- `merge-risk: 🚨 compatibility`: The patch changes established internal-only output-reuse behavior and the effective durability of completion context.
- `merge-risk: 🚨 message-delivery`: The change alters how completion content is surfaced when external user relay is disabled.
- `merge-risk: 🚨 session-state`: Inline command details are retained in transcript history when the assistant produces a substantive response.

## Summary

The bug is valid and the supplied live logs show the desired current-turn behavior, but the patch places raw exec output in `transcriptCommandBody`. Existing heartbeat filtering retains that user message whenever the assistant responds substantively, so the change silently converts ephemeral completion context into durable session history and is not merge-ready.

## What This Changes

The PR adds a dedicated `[OpenClaw exec completion]` transcript marker with inline command details and changes internal-only exec completion prompts to let the model continue from the result while external relay remains gated by `canRelayToUser`.

## Best Possible Solution

Keep a dedicated typed exec-completion event, but pass its details through a runtime-only input visible to both `before_agent_run` hooks and the current model; preserve transcript durability and external-delivery behavior unless maintainers explicitly approve and test those contract changes.

## Reproduction Assessment

Yes, at source level with high confidence. Current main routes exec completion through a heartbeat turn with the generic heartbeat transcript marker, and the supplied live before/after logs show the model acknowledging the heartbeat before the patch and consuming the result afterward.

## Solution Assessment

No. The dedicated event marker is a sound direction, but embedding raw completion details in `transcriptCommandBody` silently changes session persistence, and the broader internal-only output-reuse change lacks explicit contract proof.
