## Summary

Keep open: current main and v2026.6.10 have progress drafts and adjacent task/session status surfaces, but they still do not provide the requested persistent per-channel-turn status object with durable terminal unknown/interrupted semantics.

## What This Changes

Request a persistent, session/thread-scoped channel status surface for long-running turns, initially useful for Discord, with best-effort send/edit behavior and explicit terminal states.

## Best Possible Solution

Define an additive runtime-owned channel-turn status contract that reuses progress/session/task signals, persists only ownership needed to fail closed to unknown or interrupted, and projects first to Discord as best-effort transport.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and docs inspection show current main has transient progress/status drafts and task/session status rows, but not the requested persistent channel-turn status contract.

## Solution Assessment

Unclear as a concrete implementation plan. The one-message status model is plausible, but maintainers need to choose ownership, persistence, restart semantics, and rollout behavior before code work is safe.
