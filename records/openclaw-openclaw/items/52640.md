## Summary

Keep open: current main has adjacent progress drafts, task-ledger terminal states, session lifecycle rows, and channel busy snapshots, but it still lacks the requested persistent per-channel-turn status surface with explicit done, failed, unknown, or interrupted semantics.

## What This Changes

Issue #52640 requests a first-class editable status surface for long-running channel turns, starting with Discord and later generalizing across channels.

## Best Possible Solution

Define an additive runtime-owned channel-turn status contract that reuses existing progress/session/task signals, persists only the ownership needed to fail closed to unknown/interrupted, and projects first to Discord as best-effort opt-in transport.
