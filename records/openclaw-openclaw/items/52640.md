## Summary

Keep open: current main now has adjacent progress drafts, Web UI run-status chips, task-ledger status records, and Discord message editing, but it still does not provide the requested durable channel-turn status surface with explicit terminal/unknown ownership semantics after restart or uncertain continuity.

## What This Changes

Add a durable, session/thread-scoped status surface for long-running channel turns, starting with Discord editable messages and explicit terminal or unknown states.

## Best Possible Solution

Define one canonical runtime/channel status contract that reuses progress drafts where possible, persists only the ownership needed to fail closed, and projects explicit done/failed/unknown/interrupted state into editable channel surfaces without creating a second task ledger.

## Reproduction Assessment

Not applicable: this is a feature/product request, not a single broken current-main behavior with a repro path. The supporting evidence is source/docs comparison plus adjacent stale-running reports.

## Solution Assessment

No, not as an automated repair yet. The best path is a maintainer-owned design that extends the existing progress-draft/task-status surfaces instead of adding a Discord-only parallel status mechanism.
