## Summary

Keep open: current main and the latest release have adjacent progress drafts, Discord editable previews, task status records, and a Control UI run chip, but they do not provide the requested durable channel-turn status surface with persisted ownership and fail-closed unknown/interrupted semantics.

## What This Changes

Add a persistent, editable, per-run channel status surface for long-running turns with explicit terminal and unknown/interrupted states.

## Best Possible Solution

Define one canonical runtime/channel status contract that reuses progress drafts and task/session lifecycle state where possible, persists only the ownership/message identity needed to fail closed, and keeps status send/edit best-effort.

## Reproduction Assessment

Not applicable: this is a feature/product request, not a broken existing behavior with a single failing reproduction path; source inspection confirms only adjacent partial surfaces.

## Solution Assessment

Unclear: the durable status model is plausible, but maintainers need to choose the boundary so it reuses progress drafts, task records, and session lifecycle state instead of creating a competing source of truth.
