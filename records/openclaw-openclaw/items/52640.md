## Summary

Keep open: current main and v2026.6.8 have adjacent progress drafts, task-ledger terminal/lost states, session lifecycle projection, and channel busy patches, but they do not provide the requested first-class persisted channel-turn status message with ownership and fail-closed unknown/interrupted semantics.

## What This Changes

Requested a durable, per-run status surface for long-running channel turns, especially an editable Discord status message with explicit terminal and unknown/interrupted states.

## Best Possible Solution

Define one canonical runtime/channel status contract that reuses progress drafts, task records, and session lifecycle state where appropriate, persists only the ownership needed to fail closed, and keeps status transport best-effort.

## Reproduction Assessment

Not applicable: this is a feature request, not a broken existing-behavior report with a current reproduction path. Source, docs, and release inspection show adjacent partial surfaces but no current-main or shipped implementation of the requested durable channel-turn status contract.

## Solution Assessment
