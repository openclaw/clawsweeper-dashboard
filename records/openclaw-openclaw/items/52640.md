## Summary

Keep open. Current main has shipped a strong partial answer through Discord progress drafts, but it still does not provide the requested authoritative persistent status lifecycle with explicit failed/unknown/interrupted semantics for normal channel turns.

## What This Changes

Request to add one editable, session/thread-scoped status surface for long-running channel turns, with delayed start, milestone updates, terminal states, restart-safe unknown/interrupted behavior, and best-effort transport failures.

## Best Possible Solution

Keep the current progress-draft implementation as the partial channel UX, then design a narrower additive lifecycle/status layer for durable terminal and unknown/interrupted state, or split those leftovers into canonical follow-up issues.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection verify the current progress-draft behavior and the missing durable restart/unknown semantics; no failing runtime bug was reproduced.

## Solution Assessment

Unclear. Progress drafts are the right partial direction for visible long-running work, but the persistent terminal/unknown lifecycle still needs maintainer design before a narrow implementation is safe.
