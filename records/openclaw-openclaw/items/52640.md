## Summary

Keep open: current main and the latest release have adjacent progress drafts, Discord editable preview transport, task terminal/lost states, and Control UI run indicators, but they still do not provide the requested first-class persistent channel-turn status surface with durable ownership and fail-closed unknown/interrupted semantics.

## What This Changes

Request to add a durable, session/thread-scoped editable status surface for long-running channel turns, with Discord-first rollout and explicit terminal or unknown states.

## Best Possible Solution

Define one canonical runtime/channel status contract that reuses progress drafts and task/run state where possible, persists only the ownership/message identity needed to fail closed, and keeps send/edit failures best-effort.
