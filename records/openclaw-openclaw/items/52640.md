## Summary

Keep open: current main has progress drafts, Discord editable preview transport, task terminal/lost states, and internal channel busy snapshots, but it still lacks the requested first-class persistent channel-turn status surface with explicit done, failed, unknown, and interrupted ownership semantics after restart or uncertain continuity.

## What This Changes

Request a durable editable per-run/channel status surface for long-running chat turns, Discord first, with explicit terminal and unknown/interrupted states.

## Best Possible Solution

Define one canonical runtime/channel status contract that reuses progress drafts and task/run-state where possible, persists only the ownership/message identity needed to fail closed, and keeps send/edit failures best-effort.

## Solution Assessment

No automated fix path yet: progress drafts and task records solve adjacent slices, but durable terminal/unknown channel-turn semantics need a maintainer-owned product/API design to avoid duplicate ledgers.
