## Summary

Keep open. Current main now has substantial progress-draft support for Discord and other editable channels, but it is still a work-in-progress draft/final-answer transport rather than the requested durable per-turn status surface with explicit done, failed, unknown, or interrupted semantics and restart ownership.

## What This Changes

The issue requests a first-class persistent status surface for long-running channel turns, starting with Discord, using one editable status message with explicit lifecycle states and best-effort failure handling.

## Best Possible Solution

Keep the shipped progress-draft transport, then design an additive Discord-first per-turn lifecycle/status contract that records ownership and can fail closed to unknown or interrupted after restart, disconnect, or uncertain continuity.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives a high-confidence verification path: current main has editable progress drafts and internal busy/task state, but not the requested durable terminal/unknown status surface for normal channel turns.

## Solution Assessment

No. Current progress drafts are the right partial direction, but they are not the complete solution because they finalize into normal answer delivery and do not preserve explicit terminal or interrupted state after uncertain continuity.
