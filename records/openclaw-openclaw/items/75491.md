## Summary

Keep open. Current main still has the reported helper behavior, and the issue is now paired with an open closing PR, so the issue should remain open until that PR lands or is replaced.

## What This Changes

Fix the managed-markdown writer so inline dreaming blocks are replaced in place and duplicate managed blocks collapse to one block despite CRLF or whitespace differences.

## Best Possible Solution

Land a focused helper-level fix that tolerates CRLF/nearby marker whitespace, removes all existing blocks for the same marker pair while leaving one canonical block, and adds SDK helper plus memory-core inline regression coverage and a user-facing changelog entry.

## Reproduction Assessment

Yes. Source inspection and the no-write regex check reproduce both reported paths: CRLF after the heading fails to match, and duplicate managed blocks are not collapsed by the current single replacement.

## Solution Assessment

Yes. A narrow shared-helper fix with targeted memory-core coverage is the maintainable path; adding only a `g` flag would be weaker unless the implementation also collapses duplicates to one canonical block.
