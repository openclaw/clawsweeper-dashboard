## Summary

Keep open: current main still reproduces the managed-markdown duplication, and the linked closing PRs are open implementation candidates rather than a completed fix.

## What This Changes

Fix the shared managed-markdown writer so inline Dreaming and memory-wiki files replace one managed block per marker instead of appending or preserving duplicates when line endings or duplicate blocks already exist.

## Best Possible Solution

Land one focused helper-level fix through #75495, #75810, or a replacement that matches CRLF line endings, collapses pre-existing duplicate managed blocks, preserves unrelated markdown, adds regression tests, and includes the required changelog entry.

## Reproduction Assessment

Yes. Current source inspection plus a no-write Node mirror of the helper reproduce both reported failure modes: CRLF input appends a second block, and duplicate input keeps a stale second block.

## Solution Assessment

Yes. A shared `replaceManagedMarkdownBlock` fix is the narrow maintainable solution because Dreaming and memory-wiki callers already route through that helper; caller-specific cleanup would leave the shared defect in place.
