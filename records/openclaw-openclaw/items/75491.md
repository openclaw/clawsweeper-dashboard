## Summary

Keep open. Current main still has the reported managed-markdown replacement behavior, and the issue is paired with open PR #75495, so the right cleanup path is to review and land or replace that PR rather than close this issue early.

## What This Changes

The issue requests making managed markdown replacement for inline Dreaming blocks tolerate line-ending mismatches and remove duplicate marker blocks instead of appending more copies.

## Best Possible Solution

Land a focused fix through PR #75495 or a replacement that updates the shared helper to match managed blocks across CRLF/whitespace variations, collapse all existing blocks for the same marker pair to one canonical block, add helper and memory-core inline regression tests, and include the required user-facing changelog entry.

## Reproduction Assessment

Yes. Source inspection plus the no-write Node reproduction provide a high-confidence path: CRLF after the heading produces two start markers, and an input with two existing blocks still has two after replacement.

## Solution Assessment

Yes at the issue level. The maintainable fix belongs in the shared managed-markdown helper used by Dreaming and other memory surfaces; a bare global flag alone is not enough unless the implementation also handles CRLF/heading separation and self-heals existing duplicates.
