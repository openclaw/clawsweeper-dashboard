## Summary

Keep open. Current main has made meaningful progress on plugin externalization, dependency-state reporting, and scoped runtime/plugin loading, but this issue's central selective installation request is still not implemented: there is no setup-time channel/plugin picker, no `plugins.strictMode`, no strict discovery-root mode, no user-selected package composition, and bundled skills are still loaded then filtered for eligibility.

## What This Changes

Request selective install and load controls for bundled plugins, skills, and channels so unused integrations are not installed, scanned, loaded, or repaired.

## Best Possible Solution

Keep this as a maintainer design item and split it into focused tracks for package composition, setup selection UX, strict discovery policy, plugin inventory/status reporting, and bundled skill policy.

## Reproduction Assessment

Not applicable as a feature request. The remaining gap is high-confidence by source inspection: setup lacks a selector, plugin config lacks `strictMode`, discovery still scans standard roots, and bundled skills are still load-time filtered.

## Solution Assessment

Unclear as one implementation. The current externalization and scoped-load work is the right baseline, but the rest needs maintainer-owned design before narrow code changes are safe.
