## Summary

Keep open. Current main has important partial fixes for the runtime-dependency part of the report, but it still does not implement install-time selection, a setup channel/plugin picker, strict discovery-root filtering, or package-level slimming for bundled plugins, bundled skills, and channels.

## What This Changes

The issue requests selective installation and loading controls for plugins, skills, and channels to reduce OpenClaw install size, startup work, log noise, dependency conflicts, and security surface.

## Best Possible Solution

Keep the issue open as a maintainer design item, then split it into narrower tracks: slim-core or on-demand bundled-plugin packaging, setup/onboarding selection UX, any strict discovery mode, and separate skill-loading/install policy. Preserve the shipped runtime-deps scoping instead of reworking it as if no progress exists.

## Reproduction Assessment

Not applicable as a feature request. The current-main gap is reproducible by source/docs inspection: package.json still ships dist and skills, setup has no selector, PluginsConfig has no strictMode, and discovery still scans bundled/global roots.

## Solution Assessment

No single broad implementation is the best path yet. The maintainable direction is a product/design decision followed by small PRs with focused validation for packaging, setup UX, plugin discovery, and skills behavior.
