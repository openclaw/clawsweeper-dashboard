## Summary

Keep open. Current main has meaningful scoped runtime-loading, plugin allowlist/bundledDiscovery, plugin install/list progress, and safer postinstall behavior, but it still lacks the central setup-time selection, strict discovery policy, and user-composed plugin/skill/channel install flow requested here.

## What This Changes

Request selective setup/install and runtime loading for plugins, skills, and channels to reduce package size, startup work, dependency surface, and log noise.

## Best Possible Solution

Keep this as a maintainer design item and split it into focused tracks for package composition, setup selection UX, strict discovery semantics, plugin inventory/status reporting, and skill/channel installation policy.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives high confidence that current main still lacks setup-time channel/plugin selection, plugins.strictMode, and user-composed install selection while partial runtime scoping exists.

## Solution Assessment

Unclear as one patch. The existing allowlist, bundledDiscovery, scoped runtime-loading, and explicit plugin install/update flows are the right baseline, but the remaining install-time and strict-discovery behavior needs maintainer-owned product and packaging design.
