## Summary

Keep open. Current main has reduced several underlying costs with plugin allowlists, explicit install/update flows, channel setup helpers, skill allowlists, and postinstall cleanup, but it still lacks the requested first-class selective install/setup policy across bundled plugins, skills, and channels.

## What This Changes

The issue requests selective installation and loading controls for plugins, skills, and channels to reduce install size, startup work, dependency conflicts, and log noise.

## Best Possible Solution

Define a maintainer-approved selective install/setup policy that reuses existing plugin allowlists, install records, channel setup selection, and skill allowlists before adding new config or package-selection behavior.
