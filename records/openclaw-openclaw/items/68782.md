## Summary

Keep this open for maintainer product review: current main already has runtime plugin allowlisting and lazy manifest-based activation, but the broader install-time/package selection and skills/channels setup model is not implemented and is too compatibility-sensitive for an automatic fix PR.

## What This Changes

The issue requests install-time and runtime selection for plugins, skills, and channels to reduce package size, startup work, dependency conflicts, log noise, and unused code surface.

## Best Possible Solution

Define a maintainer-approved selective install/loading model that builds on the existing plugin allowlist and manifest-first activation plan, with explicit upgrade semantics for bundled plugins, skills, and channels.
