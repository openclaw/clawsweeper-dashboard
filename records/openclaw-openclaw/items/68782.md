## Summary

Keep open. Current main has addressed several underlying costs with plugin allowlists, scoped startup metadata reads, explicit plugin/skill install flows, channel setup install flows, and postinstall dependency cleanup, but it still does not provide the requested first-class selective install/setup policy across bundled plugins, skills, and channels.

## What This Changes

The issue requests selective installation and loading for plugins, skills, and channels to reduce install size, startup overhead, dependency conflicts, log noise, and attack surface.

## Best Possible Solution

Define a maintainer-approved selective install/setup policy that reuses existing plugin allowlists, install/index metadata, channel setup selection, and skill install flows, with docs and migration strategy before adding new config or package-selection behavior.

Not applicable as a bug reproduction: this is a feature/product request. Source and docs inspection show partial existing mechanisms but no first-class selective install/setup policy covering plugins, skills, and channels together.
