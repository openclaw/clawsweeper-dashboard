## Summary

Keep open. Current main has fixed several underlying pain points with plugin allowlists, explicit plugin and skill install flows, selected channel setup, scoped startup metadata reads, and postinstall dependency cleanup, but it still does not provide the requested first-class selective install/setup policy across bundled plugins, skills, and channels. That remaining work changes public setup, config, packaging, and plugin/channel behavior, so it needs maintainer product/API direction before implementation.

## What This Changes

Request first-class selective install, setup, and loading controls for plugins, skills, and channels to reduce install size, startup cost, dependency surface, log noise, and loaded code surface.

## Best Possible Solution

Define a maintainer-approved selective install/setup policy that reuses existing plugin allowlists, install/index metadata, channel setup selection, and skill install flows, with docs and migration strategy before adding new config or package-selection behavior.

## Reproduction Assessment

Not applicable as a bug reproduction: source inspection shows the requested first-class selective install/setup surfaces are still absent while several narrower runtime and setup alternatives already exist.

## Solution Assessment

No final implementation path is established; the best path is a maintainer-owned product/API decision that chooses the package, setup, config, and migration boundary before any code change.
