## Summary

Keep open. Current main has made real progress on scoped bundled runtime-dependency repair, including work related to the closed runtime-deps reports #71818 and #73002, but the central request is still not implemented: there is no install-time channel/plugin picker, no setup `--channels` flow, no strict discovery-root mode, no package-level selective bundle slimming, and only eligibility filtering for bundled skills.

## What This Changes

Request selective install and load controls for bundled plugins, skills, and channels so unused integrations are not installed, scanned, loaded, or repaired.

## Best Possible Solution

Keep this as a maintainer design item and split it into narrower tracks: package/bundle composition, setup or onboarding selection UX, any strict discovery mode, plugin status/deps reporting, and bundled skill policy. The shipped runtime-deps scoping should remain the baseline for the dependency-repair track.

## Reproduction Assessment

Not applicable as a feature request. The remaining gap is reproducible by source/docs inspection on current main: packaged files still include bundled `dist/` and `skills/`, setup has no channel selector, plugin config has no `strictMode`, and discovery still scans bundled/global roots.

## Solution Assessment

Unclear as a single implementation. The phased proposal points at the right product pain, but the maintainable path is a maintainer-owned design split into focused PRs rather than one broad automated change.
