## Summary

Keep open. Current main and v2026.6.8 already cover important pieces of the pain point, but the central request for first-class install-time selection across plugins, skills, and channels plus a stricter discovery/install policy still needs maintainer product/API scoping.

## What This Changes

Request first-class selective installation and loading controls for plugins, skills, and channels to reduce installed footprint, startup work, log noise, and unused-code exposure.

## Best Possible Solution

Define one maintainer-approved selective-install design that maps channels to plugin ids through manifests/catalogs, reuses existing `plugins.allow` and `skills.allowBundled` where possible, and documents upgrade/doctor behavior before adding any new config surface.
