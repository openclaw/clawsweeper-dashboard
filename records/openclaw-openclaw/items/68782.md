## Summary

Keep open. Current main has useful partial runtime work, including plugin allow/deny config, bundledDiscovery policy, scoped plugin loads, lazy runtime initialization, plugin listing/install commands, and cleanup of legacy runtime-deps, but it still does not implement the requested install-time/setup selection, strict discovery mode, or skills/channel composition policy. Related runtime-deps stall reports such as https://github.com/openclaw/openclaw/issues/71818, https://github.com/openclaw/openclaw/issues/73002, and https://github.com/openclaw/openclaw/issues/72846 were closed after narrower runtime/startup fixes, but they do not supersede this broader packaging/product request.

## What This Changes

Request install-time and runtime selection for plugins, skills, and channels to reduce package size, startup work, dependency conflicts, warnings, and loaded attack surface.

## Best Possible Solution

Keep this as a maintainer design item and split it into focused tracks for package composition, setup selection UX, strict discovery semantics, plugin inventory/status reporting, and skill/channel installation policy.

## Reproduction Assessment

Not applicable as a feature request. Source inspection against current main gives high confidence that setup-time channel/plugin selection, plugins.strictMode, and user-composed install selection are still missing while partial runtime allowlist/scoping pieces exist.

## Solution Assessment

Unclear as one implementation. The current allowlist, bundledDiscovery, scoped runtime-loading, and postinstall cleanup work are a useful baseline, but install-time selection and strict root/discovery policy need maintainer-owned design before a narrow code change is safe.
