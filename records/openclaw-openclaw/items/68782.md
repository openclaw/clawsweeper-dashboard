## Summary

Keep open. Current main covers several subproblems with runtime plugin allowlists, explicit plugin/skill install flows, channel setup selection, postinstall dependency cleanup, and scoped startup metadata reads, but the requested first-class selective install/setup policy across plugins, skills, and channels is still not implemented and needs maintainer product/API direction.

The issue requests selective installation and loading controls for plugins, skills, and channels to reduce install size and startup overhead.

## Best Possible Solution

Define a maintainer-approved selective install/setup design that builds on current plugin allowlists, plugin/skill install commands, and channel setup selection without regressing scoped startup metadata.
