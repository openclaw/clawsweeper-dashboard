## Summary

Keep open. Current main and the latest release still preserve the Telegram guarded-lane timeout behavior, while the open linked fix PR specifically owns this issue but has not merged yet.

## What This Changes

The issue requests durable Telegram DM/topic recovery so outbound send timeouts or stuck reply handlers cannot leave later direct messages blocked behind a guarded lane.

## Best Possible Solution

Review and land a narrow Telegram polling-session fix that fails the timed-out update, releases the lane guard safely after abort grace, and proves later same-lane updates drain while leaving the separate persistent topic black-hole work open.

## Reproduction Assessment

Yes at source level: current main has a focused test where a timed-out Telegram handler ignores abort and the later same-lane update remains pending. The issue comments also add a live 2026.5.28 repro, but I did not run live Telegram or local Vitest in this read-only review.

## Solution Assessment

Yes for the central in-process guarded-lane bug: the maintainable direction is to fail the timed-out update, release the active guard after abort grace, and restart ingress so later updates drain. Making sendChatAction best-effort alone would be too narrow, and the broader persistent forum-topic black-hole should stay tracked separately.
