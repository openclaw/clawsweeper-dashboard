## Summary

Keep open: current main still lacks the requested global default-model selector, and this PR’s patch is useful but stale against the current split Agents overview, so it is not merge-ready as written.

## What This Changes

The PR adds a Control UI Agents overview Active Primary Model selector, config write helpers/callbacks for `agents.defaults.model`, and focused model-state tests.

## Best Possible Solution

Port the selector and helper behavior into `ui/src/ui/views/agents-panels-overview.ts`, thread an explicit default-model callback through `renderAgents` and `renderApp`, update `agents.defaults.model` while preserving canonical `fallbacks` and supported fields such as `timeoutMs`, keep per-agent overrides on `agents.list[].model`, and cover default-vs-override state resolution in focused UI tests.

## Reproduction Assessment

Yes. Source inspection on current main is enough: the visible overview select calls `onModelChange(agent.id, ...)`, and that handler writes `agents.list[index].model` rather than `agents.defaults.model`.

## Solution Assessment

No for the current patch. The requested product behavior is narrow and reasonable, but the implementation should target the current split overview component and preserve the full supported model-object shape before merge.
