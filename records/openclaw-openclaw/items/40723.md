## Summary

Keep open. This is a plausible Control UI improvement, but the current branch is conflicting, lacks required contributor real behavior proof, and still has source-backed correctness issues around the rendered overview component and model-object preservation.

## What This Changes

The PR adds a global Active Primary Model selector to the Agents overview, writes `agents.defaults.model` from Control UI, extracts model-state/update helpers, and adds focused tests.

## Best Possible Solution

Land one coherent default-model editing path in the split Agents overview that writes defaults when appropriate, preserves all model object fields including `timeoutMs`, and keeps per-agent overrides intact.

## Reproduction Assessment

Yes for the review findings: source inspection on current main shows the rendered overview comes from `agents-panels-overview.ts` while the PR inserts the selector into the older `agents.ts` overview body, and the schema shows `timeoutMs` is a valid model object field. The requested behavior itself is a new UI capability, so bug reproduction is not applicable.

## Solution Assessment

No. The branch should target `agents-panels-overview.ts`, preserve full `AgentModelSchema` object fields including `timeoutMs`, and be reconciled with https://github.com/openclaw/openclaw/pull/40052 before merge.
