## Summary

Keep open. The feature is not implemented on current main, and the PR still has blocking review issues: the selector targets an obsolete overview path, the helper drops a supported model field, real behavior proof is missing, and the UX overlaps with the narrower open fix at https://github.com/openclaw/openclaw/pull/40052.

## What This Changes

This PR adds a Control UI Agents overview global Active Primary Model selector wired to `agents.defaults.model`, extracts model-state/update helpers, and adds focused helper tests.

## Best Possible Solution

Decide whether OpenClaw wants a separate global default selector beyond the default-agent editor fix, then implement it in `agents-panels-overview.ts` with a defaults-only handler that preserves the complete `AgentModelSchema` object and leaves per-agent overrides intact.

## Reproduction Assessment

Yes for the review findings. Source inspection on current main plus the PR diff shows the selector was added to the obsolete overview path, and the `timeoutMs` loss follows directly from `AgentModelSchema` versus `buildPrimaryModelConfig`.

## Solution Assessment

No. The feature direction may be useful, but this branch should be rebased into the current split overview, reconciled with https://github.com/openclaw/openclaw/pull/40052, preserve the complete model object contract, and include real UI proof before merge.
