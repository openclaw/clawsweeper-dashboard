## Summary

Keep open: the requested UI feature is still not on current main, and the PR remains useful, but the branch targets stale Agents overview code, drops a supported model object field, is currently conflicting, and lacks after-fix real behavior proof.

## What This Changes

Adds an Agents overview global Active Primary Model selector wired to `agents.defaults.model`, plus helper and test coverage for default-vs-override model state.

## Best Possible Solution

Port the selector into `agents-panels-overview.ts` with a defaults-only callback, preserve all supported model object fields such as `timeoutMs`, keep per-agent overrides unchanged, and add focused tests plus redacted runtime proof.

## Reproduction Assessment

Yes. Source inspection on current main is enough: the visible Agents overview is rendered from `agents-panels-overview.ts` and its select calls `onModelChange(agent.id, ...)`, while `app-render.ts` writes `agents.list[index].model`.

## Solution Assessment

No. The feature direction is reasonable, but this branch should be rebased and ported to the current split component with a defaults-only write path and full preservation of the strict model object contract.
