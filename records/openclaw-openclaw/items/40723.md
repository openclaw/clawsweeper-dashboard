## Summary

Keep this PR open, but it is not ready to merge: the feature direction is useful, yet the branch is conflicting, misses the current split Agents overview path, drops a supported model config field, and still lacks inspectable real behavior proof.

## What This Changes

The PR adds a global Active Primary Model selector for `agents.defaults.model` in the Agents overview, plus helper and test coverage for default-versus-override model resolution.

## Best Possible Solution

Port the selector into `agents-panels-overview.ts` with a defaults-only callback, preserve all supported `AgentModelConfig` fields such as `timeoutMs`, keep per-agent overrides unchanged, and add focused tests plus redacted runtime proof.

## Reproduction Assessment

Yes. Source inspection on current main is enough: the visible overview is `agents-panels-overview.ts`, its selector calls `onModelChange(agent.id, ...)`, and `app-render.ts` writes the agent model path.

## Solution Assessment

No. The feature direction is reasonable, but this branch should be rebased and moved to the current split overview with a defaults-only write path that preserves the full model object contract.
