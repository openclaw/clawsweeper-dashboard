## Summary

Keep this PR open for maintainer handling rather than cleanup-close: current main does not implement the separate global Active Primary Model selector, but the branch is conflicting, targets an obsolete overview component path, drops a supported model field, and still lacks external after-fix real behavior proof.

## What This Changes

This PR adds a Control UI Agents overview global Active Primary Model selector wired to `agents.defaults.model`, extracts model-state/update helpers, and adds focused helper tests.

## Best Possible Solution

Decide whether OpenClaw wants a separate global default selector beyond the default-agent editor fix, then implement it in `agents-panels-overview.ts` with a defaults-only handler that preserves all `AgentModelSchema` fields and leaves per-agent overrides intact.

## Reproduction Assessment

Yes for the review findings. Source inspection on current main plus the PR diff shows the selector was added to an obsolete overview path, and the `timeoutMs` loss follows directly from `AgentModelSchema` versus `buildPrimaryModelConfig`.

## Solution Assessment

No. The feature direction may be useful, but this branch should be rebased into the current split overview, reconciled with https://github.com/openclaw/openclaw/pull/40052, preserve the complete model object contract, and include real UI proof before merge.
