The PR forwards embedded-run `toolsAllow` into `createOpenClawCodingTools`, adds a force-allow merge helper for known core tools, and covers restrictive allowlist and deny-precedence regressions.

## Best Possible Solution

Land a rebased version that re-exposes explicitly requested known core tools inside the policy pipeline while preserving deny, sandbox, subagent, owner-only, provider, and runtime capability gates, with focused regression tests and a changelog entry.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainer policy review: the patch is technically narrow and deny-preserving, but it deliberately grants a per-run exception to restrictive allowlists for command/data-access tools.
