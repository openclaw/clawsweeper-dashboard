## Summary

Keep open, but do not merge as-is: the PR bypasses the current missing-device policy for Tailscale-only operator sessions and preserves self-declared scopes across a security-sensitive gateway auth boundary.

## What This Changes

The PR adds an early allow path for device-less WebSocket operator sessions authenticated by Tailscale or trusted-proxy when `sharedAuthOk` is false, plus a connect-policy test documenting that mismatch.

## Best Possible Solution

Resolve the gateway trust policy first, then encode any approved exception in the shared connect policy, docs, and end-to-end coverage for connect plus an `operator.write` RPC.

## Reproduction Assessment

Yes. From source and checked-in tests, a Tailscale-only WebSocket operator with no device identity reaches the PR's early return while current main expects `device identity required`; I did not run a live gateway in this read-only review.

## Solution Assessment

No. The patch bypasses the centralized missing-device decision in `message-handler.ts`; the safer path is an explicit, reviewed gateway auth policy change with aligned code, tests, and docs.
