## Summary

Keep open: the reported gateway scope problem is source-reproducible, but this PR is not merge-ready because the patch bypasses the centralized missing-device policy, lacks real gateway proof, and overlaps the open maintainer token-scope policy path.

## What This Changes

The PR adds an early WebSocket connect allow path for device-less Tailscale/trusted-proxy operator sessions and a connect-policy test documenting the bypass.

## Best Possible Solution

Resolve device-less operator scopes through a centralized connect-policy or explicit configured token-scope contract with docs, security tests, and redacted real gateway proof, then supersede this handler-local bypass if that policy lands elsewhere.

## Reproduction Assessment

Yes, at source level. Current main can produce Tailscale primary auth success with `sharedAuthOk=false` and no device, and current policy then rejects or clears scopes for device-less operator paths; I did not run a live gateway scenario in this read-only review.

## Solution Assessment

No. Returning early in `message-handler.ts` bypasses centralized missing-device policy; the safer solution is an explicit policy/config contract or connect-policy change aligned with docs, tests, and real transport proof.
