## Summary

Keep this PR open for review, but the current patch is not merge-ready. The final guard now targets the tokenless Tailscale Serve path, not explicit token/password auth, and it bypasses the documented no-device rejection while preserving self-declared operator scopes.

## What This Changes

The PR adds a WebSocket handler early-allow path plus a connect-policy test for device-less Tailscale/trusted-proxy operator sessions when primary transport auth succeeds but sharedAuthOk is false.

## Best Possible Solution

Keep device-less Tailscale Serve fail-closed by default; if maintainers want this capability, add it as a centralized connect-policy or configured-scope contract with explicit security tests, docs, and real gateway proof.

## Reproduction Assessment

Yes, at source level. Current main has tests and docs requiring tokenless Tailscale no-device connects to be rejected, while the PR's early return would allow that path before the decision is enforced.

## Solution Assessment

No. Returning early in message-handler.ts bypasses centralized missing-device policy; the safer fix is a connect-policy change or explicit configured-scope contract aligned with tests, docs, and real transport proof.
