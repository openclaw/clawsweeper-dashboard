## Summary

Keep this PR open, but it is not safe to merge as written: the early return bypasses the current missing-device policy for transport-authenticated operator sessions and the external PR still lacks real gateway proof.

## What This Changes

The PR adds a WebSocket connect early-allow path for device-less operator sessions authenticated by Tailscale or trusted-proxy when the shared-auth probe fails, plus a policy test documenting that bypass.

## Best Possible Solution

Keep the device-identity boundary in place until maintainers approve a narrow transport-auth contract, then encode that decision in shared connect policy with regression tests and live gateway proof.

## Reproduction Assessment

Yes. Source inspection and checked-in tests give a high-confidence path: current main rejects device-less Tailscale-only operators and clears scopes for Tailscale-routed shared-token sessions, while the PR returns before that policy path.

## Solution Assessment

No. The patch bypasses the centralized missing-device policy inside message-handler.ts; any approved exception should be modeled in connect-policy with explicit trust semantics, tests, and real gateway proof.
