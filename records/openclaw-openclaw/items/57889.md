## Summary

Keep open. The final PR diff is still security-sensitive and currently incorrect because it allows a device-less Tailscale-authenticated operator to skip the centralized missing-device policy with requested scopes intact, and the external contributor has not provided after-fix real gateway proof.

## What This Changes

The PR adds an early WebSocket connect allow path for device-less operator sessions authenticated by Tailscale or trusted-proxy when the shared-auth probe fails, plus a connect-policy test documenting that bypass.

## Best Possible Solution

Use a maintainer-approved bounded token/transport scope policy, likely aligned with https://github.com/openclaw/openclaw/pull/81792, with connect-policy, scope clearing, docs, regression tests, and real gateway proof kept in sync.

## Reproduction Assessment

Yes, at source level. Current main tests require device identity for Tailscale-only operator auth and expect a Tailscale-routed shared-token device-less operator to connect with cleared scopes before scoped RPCs fail; I did not run a live remote gateway.

## Solution Assessment

No. Returning early in `message-handler.ts` bypasses the centralized missing-device policy and scope-clearing contract; any approved exception should be modeled as an explicit auth policy/config path with aligned tests and docs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep transport auth on the missing-device policy path:** `src/gateway/server/ws-connection/message-handler.ts:568`
