## Summary

Keep open: current main and the latest release still allow non-finite `click-coords` coordinates into the browser action body, and https://github.com/openclaw/openclaw/pull/84208 is the active open fix that closes this issue.

## What This Changes

The issue asks for browser CLI positional numeric arguments to reject `NaN` before sending click-coordinate or resize requests.

## Best Possible Solution

Main should reject non-finite `click-coords` coordinates before constructing the `/act` body, keep resize validation centralized, and add focused CLI regression coverage.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: `browser click-coords abc 200` flows through `Number("abc")` without a guard before the `/act` body is built, and JavaScript serialization turns that `NaN` into JSON `null`. I did not run the full CLI because this review is read-only.

## Solution Assessment

Yes. The narrow maintainable fix is a local finite-number guard for `click-coords` plus regression coverage; the open linked PR is the current implementation path.
