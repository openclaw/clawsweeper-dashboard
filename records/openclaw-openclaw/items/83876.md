## Summary

Keep open: current main and the latest release still let `browser click-coords` build a `clickCoords` body from non-finite coordinates, and the issue is paired with the open closing fix at https://github.com/openclaw/openclaw/pull/84208.

## What This Changes

Reject non-finite numeric input for the browser CLI `click-coords` command before sending the browser action request.

## Best Possible Solution

Review and land the linked browser CLI validation fix, or an equivalent main-branch patch, then close this issue as implemented once the guard is on `main`.

## Reproduction Assessment

Yes. Source inspection on current main shows `browser click-coords abc 200` flows through `Number("abc")` without a finite guard before the `/act` body is built, and a Node probe confirms that `NaN` serializes to JSON `null`.

## Solution Assessment

Yes. The narrow maintainable solution is a CLI-boundary finite-number guard for `click-coords`, while keeping resize validation centralized in the existing helper; the open linked PR is the current implementation path.
