## Summary

Keep open: current main and v2026.6.8 still clear the suspended final-delivery payload and completion fallback on expiry, and the protected `maintainer` label plus open closing PR make this a maintainer-review item rather than a cleanup-close candidate.

## What This Changes

The issue reports that suspended subagent final-delivery output can expire during terminal reconciliation and be discarded instead of preserved for the requester.

## Best Possible Solution

Ship or sponsor an equivalent narrow repair that preserves final subagent text in durable completion fallback state before compacting expired suspended delivery payloads, while keeping broader lock-contention follow-up under the related lock issue.

## Reproduction Assessment

Yes, at source level. A suspended final-delivery record with final text can age into `discardSuspendedPendingFinalDelivery`, which clears both payload and completion fallback text on current main; this read-only review did not execute the live gateway race.

## Solution Assessment

Yes for the narrow fix direction. Preserving final text in the existing completion fallback field is the smallest maintainable repair, but the open PR still needs maintainer rebase/review because it also touches lock diagnostics.
