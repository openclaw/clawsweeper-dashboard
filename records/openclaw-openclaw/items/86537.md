Keep open: current main and latest release v2026.6.9 still clear the durable fallback during suspended final-delivery discard, and the protected maintainer label plus open closing PR make this a maintainer-review item rather than cleanup-close material.

## What This Changes

The issue reports that suspended subagent final-delivery expiry can discard a completed child result after terminal reconciliation timeouts instead of preserving it for the requester.

## Best Possible Solution

Review, rebase, and land https://github.com/openclaw/openclaw/pull/86540 or an equivalent narrow fix that preserves final text in durable completion fallback before compacting expired suspended delivery payloads.

## Reproduction Assessment

Yes, at source level: the wait-timeout path can defer terminal state to reconciliation, and the suspended-delivery expiry path then clears both payload and completion fallback text. I did not run the timing-sensitive live gateway race in this read-only review.

## Solution Assessment

Yes, the narrow fix direction is to preserve final text in the existing completion fallback field because current completion selection already reads that field. The open linked PR remains the candidate implementation and needs maintainer review before this issue can close.
