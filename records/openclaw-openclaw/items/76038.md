## Summary

## What This Changes

The issue asks OpenClaw to recover stuck `processing` sessions, reduce repeated slow session-prep work, and preserve Feishu systemd override configuration across upgrades.

## Best Possible Solution

Land a focused recovery change that distinguishes real active handles from stale reply-run flags, handles the no-active-work lane state, and keeps the already-landed prep caching; split Feishu/systemd override preservation into a separate installer reproducer if confirmed.
