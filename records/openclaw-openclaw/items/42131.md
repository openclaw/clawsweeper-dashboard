## Summary

Keep open. Current main still has a real safe-bin trusted-dir case mismatch, but this PR’s latest head uses a single `os.tmpdir()` filesystem probe for an exec trust-boundary decision, which can mis-handle trusted dirs on another case-sensitive volume; it also lacks the required changelog entry.

## What This Changes

The PR lowercases safe-bin trusted-dir keys and resolved executable directories when a runtime filesystem case probe reports case-insensitive behavior, and updates safe-bin trust/runtime-policy tests.

## Best Possible Solution

Revise the PR so case folding is determined for the actual trusted path or comparison volume, keep case-sensitive sibling directories distinct, add focused regression coverage, and include the required changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: current main lowercases configured safe-bin entries, preserves trusted-dir case, and performs exact Set membership on the resolved executable directory; I did not run live macOS/Windows filesystem validation in this read-only review.

## Solution Assessment

No. The PR targets the correct mismatch, but the global `os.tmpdir()` probe is not the best fix for an exec trust boundary because configured trusted dirs may live on filesystems with different case-sensitivity semantics.

- **[P1] Probe case sensitivity for the compared trusted path:** `src/infra/exec-safe-bin-trust.ts:68`

Gate folding on the directory being normalized or compared.
