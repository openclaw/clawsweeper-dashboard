## Summary

Keep open. Current main still has the safe-bin casing mismatch this PR targets, but the proposed branch still needs revision because it bases a trust-boundary path comparison on one process-wide temp-directory case-sensitivity probe.

## What This Changes

The PR lowercases trusted safe-bin directories and resolved executable directories when an `os.tmpdir()` inode probe indicates a case-insensitive filesystem, and updates safe-bin trust/runtime policy tests.

## Best Possible Solution

Add a shared safe-bin trust-path comparison helper that normalizes both sides consistently only for the actual path being compared, then cover mixed-case matches on a genuinely case-insensitive location and non-trust of case-distinct siblings on a case-sensitive location.

## Reproduction Assessment

Yes. The current-main code path is enough to reproduce logically: doctor lowercases configured safeBins, trusted dirs keep original case, and the final membership test is exact `Set.has`; on a case-insensitive filesystem with mixed casing this yields the reported false warning.

## Solution Assessment

No. The PR addresses the right bug, but a process-wide `os.tmpdir()` probe is not the narrowest maintainable trust-boundary fix because configured safe-bin directories can live on volumes with different case-sensitivity behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Probe case sensitivity for each trusted path:** `src/infra/exec-safe-bin-trust.ts:68`
  - body: `CASE_INSENSITIVE_FS` is computed from `os.tmpdir()`, but `safeBinTrustedDirs` can point at a different volume. If temp storage is case-insensitive while a trusted dir is on a case-sensitive mount, lowercasing collapses case-distinct sibling directories and can make `isTrustedSafeBinPath` accept an executable outside the explicitly trusted path. Gate folding on the directory being compared instead of one process-wide probe.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The diff changes exec safe-bin trust-path normalization and can collapse case-distinct directories across volumes because case sensitivity is inferred from `os.tmpdir()`.

  - body: Using a single temp-directory probe to decide whether every trusted safe-bin path should be lowercased can treat paths on a different case-sensitive mount as equivalent, allowing an untrusted case-distinct sibling directory to inherit trust.
