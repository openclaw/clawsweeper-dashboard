## Summary

Keep open. The PR targets a real current-main safe-bin doctor/runtime bug, but the proposed branch still changes an exec trust boundary by folding all trusted paths based on a single `os.tmpdir()` filesystem probe, which can be wrong for configured directories on other volumes.

## What This Changes

The PR lowercases safe-bin trusted directories and resolved executable directories when an `os.tmpdir()` case-sensitivity probe reports a case-insensitive filesystem, and updates safe-bin trust/runtime policy tests.

## Best Possible Solution

Revise the safe-bin trust comparison so case folding is decided for the actual trusted path being compared, not once per process from the temp directory, and add regression coverage for both mixed-case matches on a case-insensitive location and non-trust of case-distinct siblings on a case-sensitive location. Include the required single-line changelog entry before merge.

## Reproduction Assessment

Yes. Current main can be reproduced logically from source: doctor lowercases configured safe-bin entries, trusted dirs preserve configured case, and the final trust check is exact `Set.has`, so a mixed-case trusted dir can produce a false warning on a case-insensitive filesystem.

## Solution Assessment

No. The branch fixes the right mismatch, but a process-wide temp-directory probe is not the narrowest maintainable solution for an exec trust boundary because configured safe-bin directories may live on volumes with different case-sensitivity behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Probe case sensitivity for each trusted path:** `src/infra/exec-safe-bin-trust.ts:68`
  - body: `CASE_INSENSITIVE_FS` is computed from `os.tmpdir()`, but `safeBinTrustedDirs` can point to a different volume. If temp storage is case-insensitive while a trusted dir is on a case-sensitive mount, lowercasing collapses case-distinct sibling directories and can make `isTrustedSafeBinPath` accept an executable outside the explicitly trusted path. Gate folding on the directory being compared instead of one process-wide probe.
  - confidence: 0.88
- **[P3] Add the required changelog entry:** `src/infra/exec-safe-bin-trust.ts:68`

  - confidence: 0.82
