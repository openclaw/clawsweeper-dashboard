## Summary

Keep open. Current main still has the reported mixed-case safe-bin trust mismatch, but this PR's current head changes an exec trust boundary by folding all trusted dirs from one `os.tmpdir()` filesystem probe, which can be wrong for configured dirs on a different case-sensitivity volume.

## What This Changes

The PR adds a global filesystem case-sensitivity probe and uses it to lowercase safe-bin trusted directories and resolved executable directories, with updated safe-bin trust and runtime-policy tests.

## Best Possible Solution

Revise the PR so case folding is decided from the actual trusted path or comparison volume, preserve case-sensitive sibling separation, add regression coverage for both filesystem modes, and include the required changelog entry.

## Reproduction Assessment

Yes. Current main is source-reproducible: doctor lowercases safe-bin entries, trusted dirs keep configured case, and `isTrustedSafeBinPath()` performs an exact `Set.has` comparison, so mixed-case trusted-dir configurations can false-warn on a case-insensitive filesystem.

## Solution Assessment

No. The PR fixes the right mismatch, but a single `os.tmpdir()` case probe is not the narrowest maintainable solution for an exec trust boundary because trusted dirs may live on volumes with different case-sensitivity behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Probe case sensitivity for each trusted path:** `src/infra/exec-safe-bin-trust.ts:68`
  - body: `CASE_INSENSITIVE_FS` is derived from `os.tmpdir()`, but `safeBinTrustedDirs` can point at another volume. If temp storage is case-insensitive while a trusted dir is on a case-sensitive mount, lowercasing collapses case-distinct sibling directories and can make an executable outside the explicitly trusted path look trusted. Gate folding on the directory being compared.
  - confidence: 0.88
- **[P3] Add the required changelog entry:** `src/infra/exec-safe-bin-trust.ts:68`
  - body: This is a user-facing doctor/exec-policy fix and the PR does not touch `CHANGELOG.md`, so release notes and credit audit coverage would miss the behavior change. Add a single-line Fixes entry under the active version with appropriate non-bot attribution.
  - confidence: 0.82
