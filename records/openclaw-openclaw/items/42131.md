## Summary

Keep open: current main still has the reported safe-bin case mismatch, but this PR needs changes before merge because the latest patch uses a global temp-directory case probe for a trust-boundary comparison and is missing the required changelog entry.

## What This Changes

The PR lowercases safe-bin trusted directory keys and resolved executable directory keys when a runtime filesystem case-sensitivity probe reports case-insensitive behavior, with updated safe-bin trust and runtime-policy tests.

## Best Possible Solution

Use path-specific case-sensitivity handling for the trusted or resolved directory being compared, preserve case-sensitive volumes, add focused regression coverage, and add a single active-version changelog Fixes entry.

## Reproduction Assessment

Yes. Source inspection on current main shows doctor lowercases configured safeBins, trusted dirs remain resolved with original case, and the exact Set.has() comparison can fail on case-insensitive filesystems; I did not run live macOS or Windows filesystem validation in this read-only review.

## Solution Assessment

No. The PR targets the right mismatch, but a single os.tmpdir() probe is not the narrowest maintainable solution for a trust boundary because configured trusted dirs can live on filesystems with different case-sensitivity semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Probe case sensitivity for the compared path:** `src/infra/exec-safe-bin-trust.ts:68`
  - body: `CASE_INSENSITIVE_FS` is derived from `os.tmpdir()`, but `safeBinTrustedDirs` can point at another volume. If temp storage is case-insensitive while a trusted dir is on a case-sensitive mount, lowercasing collapses case-distinct sibling directories and can make an executable outside the explicitly trusted path look trusted. Gate folding on the directory being normalized or compared.
  - confidence: 0.88
- **[P3] Add the required changelog entry:** `src/infra/exec-safe-bin-trust.ts:68`
  - body: This is a user-facing doctor/exec-policy fix and the PR does not touch `CHANGELOG.md`, so release notes and credit-audit coverage would miss the behavior change. Add a single-line Fixes entry under the active version with appropriate non-bot attribution.
  - confidence: 0.82
