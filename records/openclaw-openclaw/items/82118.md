## Summary

Keep open. Current main still maps `reasoning_content` to thinking-only output, the Xiaomi plugin has no DeepSeek-style text/replay wrapper, and the related Xiaomi reasoning PRs are open rather than merged.

## What This Changes

Custom Xiaomi/MiMo OpenAI-compatible reasoning models should treat DeepSeek-style `reasoning_content` as usable output/replay data instead of failing as reasoning-only.

## Best Possible Solution

Complete the existing OpenAI-compatible compat path so MiMo/Xiaomi DeepSeek-style `reasoning_content` works for both bundled and custom-host providers, with focused regression coverage for first-turn reasoning-only output and multi-turn replay.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not with a live MiMo credential in this review, but source inspection gives a clear current-main path: `reasoning_content` becomes thinking-only and custom MiMo models do not get a Xiaomi/DeepSeek wrapper.

## Solution Assessment

Is this the best way to solve the issue? The requested direction is broadly right, but the exact endpoint-detection choice should be folded into the provider compatibility seam rather than another isolated retry path.
