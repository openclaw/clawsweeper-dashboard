## Summary

Close as implemented: the v2026.5.12 Telegram forum-topic serialization path is fixed on current main by the merged isolated-polling lane-drain change, and that fix is shipped in v2026.5.14-beta.1.

## What This Changes

The issue requests that simultaneous messages in different Telegram forum topics run as separate sessions in parallel up to agents.defaults.maxConcurrent instead of serializing behind one topic.

## Best Possible Solution

Keep the current lane-keyed isolated-polling implementation and direct affected v2026.5.12 users to v2026.5.14-beta.1 or the next stable release containing the same fix.

## Reproduction Assessment

Yes for the affected stable release: the related live proof and merged PR provenance isolate the serialization to the pre-fix isolated-polling spool drain. No failing current-main reproduction remains because current main contains the lane-drain fix and regression coverage.

## Solution Assessment

Yes. Reusing the existing Telegram sequential key for isolated polling and preserving the existing main-lane maxConcurrent cap is the narrow maintainable fix; adding another concurrency setting would be unnecessary.
