Keep open: the docs and current source define per-topic Telegram sessions with shared main-lane concurrency, but the reported live serialization on v2026.5.12 is not explained by source inspection and no targeted merged fix or current-main live proof was found.

## What This Changes

Reports that Telegram forum-topic messages in one supergroup run serially despite `agents.defaults.maxConcurrent=4`, although separate topics should run as separate sessions in parallel up to the configured cap.

## Best Possible Solution

Keep this open until a focused live Telegram multi-topic run or integration test isolates the bottleneck, then patch only that path so forum-topic sessions can occupy distinct session lanes and share the main lane up to `agents.defaults.maxConcurrent`.
