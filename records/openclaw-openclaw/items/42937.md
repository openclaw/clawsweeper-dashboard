## Summary

Keep open: current main still lacks `agents.list[].dmScope`, so the central feature is not implemented, but this PR cannot merge as-is because it is outdated against merged binding-level `dmScope` work and would not preserve binding precedence after rebase.

## What This Changes

This PR adds `agents.list[].dmScope` to config types, schema, help, labels, route resolution, and tests so an agent can override global `session.dmScope`.

## Best Possible Solution

Land this feature only if maintainers want the config surface, with precedence `binding session override > agent default > global session default > main`, aligned schema/help/tests, and redacted mixed-agent runtime proof.

## Reproduction Assessment

Yes, for PR review: source inspection shows current main preserves matched binding `session.dmScope`, while the PR head's effective scope ignores any binding override. A focused test combining `agents.list[].dmScope` with `bindings[].session.dmScope` would exercise the regression.

## Solution Assessment

No, not as-is. The per-agent default direction is plausible, but binding-level overrides must remain higher precedence and the new config option needs maintainer approval before merge.
