## Summary

Keep open: current main still lacks per-agent dmScope support, but the PR is not merge-ready because it would need to preserve newer binding-level dmScope precedence and it lacks real mixed-agent runtime proof.

## What This Changes

The PR adds agents.list[].dmScope to config types/schema/help/labels, uses it when building route session keys, shares the DM scope zod schema, and adds a routing unit test.

## Best Possible Solution

Land a rebased implementation where bindings[].session.dmScope remains highest precedence, agents.list[].dmScope is a lower-precedence agent default, config docs/schema/tests are aligned, and redacted mixed-agent runtime proof is attached.

## Reproduction Assessment

Yes for the PR regression path by source inspection: current main uses sessionOverride?.dmScope before the global default and passes matched.binding.session, while the PR head computes the effective scope from agent config/global default only. I did not run tests because this was a read-only review.

## Solution Assessment

No, not as-is. The per-agent default can be useful, but the maintainable precedence is binding override > agent default > global default, with aligned docs/tests and real behavior proof.
