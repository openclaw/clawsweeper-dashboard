## Summary

Keep open: current main still does not have per-agent `dmScope`, but this PR is not safe as-is because the routing diff can bypass the existing binding-level `dmScope` override after rebase, and the external PR still lacks after-fix real behavior proof.

## What This Changes

The PR adds `agents.list[].dmScope` to config types/schema/help/labels and uses it during route session-key resolution, with a focused routing unit test.

## Best Possible Solution

Land a rebased implementation with precedence ordered as `bindings[].session.dmScope` > `agents.list[].dmScope` > `session.dmScope`, aligned docs/help/tests, and redacted mixed-agent runtime proof.

## Reproduction Assessment

Yes for the blocking PR regression path by source inspection: current main passes `matched.binding.session` into `choose()`, while the PR head computes effective scope from agent config/global default only. I did not run tests because this is a read-only review.

## Solution Assessment

No, not as-is. The maintainable solution is to add the agent default below binding overrides, not replace the current binding override path.
