## Summary

Keep open for maintainer review: current main still lacks per-agent `dmScope`, but the PR as written would regress the newer binding-level `dmScope` precedence and lacks real runtime proof.

## What This Changes

The PR adds `agents.list[].dmScope` config type/schema/help/label support, uses it during route session-key selection, shares a `DmScopeSchema`, and adds a routing unit test.

## Best Possible Solution

Land only a rebased maintainer-approved implementation with precedence `binding session override > agent default > global default > main`, aligned schema/help/docs/tests, and redacted mixed-agent runtime proof.

## Reproduction Assessment

Yes for the PR regression path: source inspection shows current main preserves `bindings[].session.dmScope`, while the PR head derives scope from agent config or global config only. I did not run tests because this was a read-only review.

## Solution Assessment

No, not as-is. The per-agent default may be useful, but the maintainable solution must preserve binding overrides above agent defaults and get maintainer approval for the new config surface.
