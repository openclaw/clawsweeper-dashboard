## Summary

Keep this PR open, but it is not merge-ready: current main does not implement the unique per-agent `dmScope` config layer, while the stale branch conflicts with and can bypass the merged binding-level `dmScope` precedence unless it is rebased carefully.

## What This Changes

The PR adds `agents.list[].dmScope` config typing, validation, help/label metadata, routing logic, and a focused routing test so an agent can override global `session.dmScope`.

## Best Possible Solution

If maintainers want this config surface, land a rebased implementation with precedence `bindings[].session.dmScope` > `agents.list[].dmScope` > `session.dmScope`, aligned docs/tests, and redacted mixed-agent runtime proof; otherwise use the merged binding-level override path.
