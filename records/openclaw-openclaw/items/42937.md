## Summary

Keep this PR open for maintainer review: current main still lacks the requested per-agent dmScope config surface, but the head is not mergeable as-is because it would drop newer binding-level dmScope precedence and lacks real behavior proof.

## What This Changes

The PR adds `agents.list[].dmScope` to config types/schema/help/labels, uses it during route session-key selection, shares a `DmScopeSchema`, and adds a routing unit test.

## Best Possible Solution

Land a rebased, maintainer-approved implementation where precedence is binding override > agent default > global default > main, with aligned config docs/schema/tests and redacted mixed-agent runtime proof.

## Reproduction Assessment

Yes for the PR regression path: current main routes matched bindings through `matched.binding.session` and uses `sessionOverride?.dmScope ?? dmScope`, while the PR head computes `agentCfg?.dmScope ?? dmScope`. I did not run tests because this is a read-only review.

## Solution Assessment

No, not as-is. The per-agent default may be useful, but the patch must preserve binding overrides above agent defaults and get maintainer approval for the new config surface.
