## Summary

Keep this PR open. Current main still lacks per-agent `dmScope`, so the feature is not obsolete, but the PR needs a rebase/repair before merge because current main now has binding-level `session.dmScope` precedence that the proposed routing change would bypass if applied as-is.

## What This Changes

The PR adds an optional `agents.list[].dmScope` config field across agent types, zod validation, config help/labels, route session-key resolution, and routing tests.

## Best Possible Solution

Add `agents.list[].dmScope` as an additive config contract with routing precedence `binding session override > agent default > global session default > main`, then align strict schemas, generated config metadata, docs/help, changelog, and regression tests for per-agent override, global fallback, binding-over-agent behavior, groups, and route-cache behavior.

## Reproduction Assessment

Yes. The missing feature is reproducible by inspection on current main because the strict agent schema rejects `dmScope` and route resolution only considers binding/global scope; the precedence regression is reproducible from the existing binding-level `dmScope` test and resolver path.

## Solution Assessment

No, not as-is. The feature direction is narrow and maintainable, but the patch should be rebased so binding-level `session.dmScope` stays highest precedence and generated metadata/changelog are included before merge.
