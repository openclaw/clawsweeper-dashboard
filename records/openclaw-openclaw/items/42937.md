## Summary

Keep open for maintainer review. Current main does not implement `agents.list[].dmScope`, and this stale branch now needs to compose with the merged binding-level `dmScope` override, refresh generated config schema, and provide real behavior proof before it is mergeable.

## What This Changes

The PR adds an optional `dmScope` field on agent entries, validates and labels it, and changes route resolution so an agent setting can override global `session.dmScope` when computing DM session keys.

## Best Possible Solution

Either keep `bindings[].session.dmScope` as the supported override seam, or explicitly approve an agent-level default and implement it as `binding session.dmScope > agent dmScope > global session.dmScope` with generated schema, docs/help, and focused routing tests updated together.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an additive config feature. Source inspection confirms current main supports global `session.dmScope` and binding-level `bindings[].session.dmScope`, but not agent-level `agents.list[].dmScope`.

## Solution Assessment

No, not as-is. The concept is plausible, but the branch must be rebased onto current routing semantics, preserve binding-level overrides, refresh generated config schema, and show real behavior proof before merge.
