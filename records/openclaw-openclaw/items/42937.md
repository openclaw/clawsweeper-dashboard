## Summary

Keep this PR open, but it is not merge-ready: the per-agent `dmScope` feature is still unique on current `main`, while the branch predates the merged binding-level override path and must preserve binding precedence before it can safely land.

## What This Changes

The branch adds `agents.list[].dmScope` config support, validation/help/label metadata, routing logic that prefers it over global `session.dmScope`, and a focused routing test.

## Best Possible Solution

Land a rebased implementation with precedence `bindings[].session.dmScope` > `agents.list[].dmScope` > `session.dmScope`, aligned help/tests, and redacted mixed-agent runtime proof.

## Reproduction Assessment

Yes, at source level for the blocking PR regression path: current main passes `matched.binding.session` into `choose()`, while the PR branch computes effective scope from agent config and the global default only. No live transport reproduction was run in this read-only review.

## Solution Assessment

No, not as-is. The maintainable version is to add the agent-level default below binding overrides, document that precedence, and prove mixed-agent behavior in a real setup.
