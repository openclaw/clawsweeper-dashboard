## Summary

Keep open. The central feature is not on current main, but the branch should not merge as-is because current main now has binding-level `dmScope` overrides that must keep precedence over any new agent default, and the PR still lacks real mixed-agent behavior proof.

## What This Changes

The PR adds an optional per-agent `dmScope` config field with schema/help/label support, routing logic that uses it for DM session keys, and a focused routing test.

## Best Possible Solution

Land this only if maintainers want the config surface, with precedence `binding session override > agent default > global session default > main`, aligned schema/help/docs/tests, and redacted real mixed-agent proof.

## Reproduction Assessment

Yes, for source-level PR review: current main gives matched route bindings a `session.dmScope` override, while the PR branch computes the effective scope without that override. A regression test combining `agents.list[].dmScope` with `bindings[].session.dmScope` would reproduce the failure.

## Solution Assessment

No, not as-is. The feature direction is narrow, but the implementation must be rebased to preserve binding-first precedence before it is a maintainable solution.
