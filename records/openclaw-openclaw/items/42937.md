## Summary

Keep open. Current main still has no per-agent `dmScope`, so the feature is not implemented, but the PR is not mergeable as-is because the routing change would drop existing binding-level `dmScope` precedence and there is no after-fix real behavior proof from a mixed-agent setup.

## What This Changes

The branch adds `agents.list[].dmScope` to config types/schema/help/labels, shares a `DmScopeSchema`, uses the agent setting in route session-key resolution, and adds a routing unit test.

## Best Possible Solution

Land only a rebased maintainer-approved version with precedence `binding session override > agent default > global default > main`, aligned config docs/schema/tests, and redacted real mixed-agent runtime proof.

## Reproduction Assessment

Yes for the PR regression path: source inspection shows current main preserves `bindings[].session.dmScope`, while the PR head derives scope from agent config or global config only. I did not run tests because this was a read-only review.

## Solution Assessment

No, not as-is. The per-agent default may be useful, but the maintainable solution must preserve binding overrides above agent defaults and get maintainer approval for the new config surface.
