## Summary

Keep open: current main still lacks an agent-level `dmScope`, but this PR is not merge-ready because it must preserve existing binding-level precedence and still needs after-fix real behavior proof.

## What This Changes

The PR adds `agents.list[].dmScope` to config typing/schema/help/labels and uses it when resolving direct-message session keys.

## Best Possible Solution

Land a rebased implementation with precedence `bindings[].session.dmScope` > `agents.list[].dmScope` > `session.dmScope`, aligned docs/help/tests, and redacted mixed-agent runtime proof.

## Reproduction Assessment

Yes, at source level for the blocking PR regression path: current main passes `matched.binding.session` into `choose()`, while the PR branch computes effective scope from agent config/global default only. No live transport reproduction was run in this read-only review.

## Solution Assessment

No, not as-is. The maintainable solution is to add the agent default below binding overrides, document that precedence, and prove the mixed-agent behavior in a real setup.
