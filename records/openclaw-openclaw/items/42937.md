## Summary

Keep open: current `main` still lacks per-agent `dmScope`, but this PR cannot merge as-is because it would let agent/global scope bypass the existing binding-level DM isolation override and it has no real mixed-agent behavior proof.

## What This Changes

The PR adds `agents.list[].dmScope` to config types/schema/help/labels and uses it during route session-key resolution with a focused routing unit test.

## Best Possible Solution

Land a rebased implementation with precedence ordered as `bindings[].session.dmScope` > `agents.list[].dmScope` > `session.dmScope`, aligned schema/help/docs/tests, and redacted mixed-agent runtime proof.

## Reproduction Assessment

Yes for the blocking PR regression path by source inspection: current `main` passes `matched.binding.session` into `choose()`, while the PR head computes effective scope from agent config/global default only. I did not run tests because this was a read-only review.

## Solution Assessment

No, not as-is. The maintainable solution is to add the agent default below binding overrides, not replace the current binding override path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve binding dmScope precedence:** `src/routing/resolve-route.ts:666`
  - body: Current `main` passes `matched.binding.session` into `choose()` and uses `sessionOverride?.dmScope`, so route bindings can isolate selected DMs. This branch removes that override input and uses only `agentCfg?.dmScope ?? dmScope`, so a binding with its own `session.dmScope` would be ignored after rebase; keep precedence as binding override, then agent override, then global default.
  - confidence: 0.9
- **[P3] Document the new dmScope precedence:** `src/config/types.agents.ts:82`
  - body: This adds a public `agents.list[].dmScope` config key, but the docs still only describe global `session.dmScope` and binding-level scope. Add the per-agent field and its precedence so users can configure mixed-agent isolation without guessing which override wins.
  - confidence: 0.78
