## Summary

Keep open, but this PR is not merge-ready: the per-agent `dmScope` feature is still unique, while current main now has binding-level `dmScope` precedence that this stale conflicting branch must preserve during rebase.

## What This Changes

The PR adds optional `dmScope` to `agents.list[]`, validates and labels it, and uses it during route session-key resolution with focused unit coverage.

## Best Possible Solution

Add per-agent `dmScope` only as a lower-precedence default beneath `bindings[].session.dmScope`, document the full precedence, and prove a mixed-agent DM setup in a real run.

## Reproduction Assessment

Yes, at source level for the blocking merge path: current main passes `matched.binding.session` into `choose()`, while the PR branch computes effective scope from agent config and the global default only. No tests or live transport proof were run because this was a read-only review.

## Solution Assessment

No, not as submitted. The maintainable implementation is binding `dmScope` > agent `dmScope` > global `session.dmScope`, with docs and tests that make that precedence explicit.
