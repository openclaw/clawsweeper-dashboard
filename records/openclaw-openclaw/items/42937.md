## Summary

## What This Changes

The branch adds an optional `agents.list[].dmScope` config field, validation/help/label metadata, routing logic to use it ahead of global `session.dmScope`, and a focused route test.

## Best Possible Solution

Land a rebased implementation with precedence `bindings[].session.dmScope` > `agents.list[].dmScope` > `session.dmScope`, aligned help/docs/tests, and redacted mixed-agent runtime proof.

## Reproduction Assessment

Yes, at source level for the blocking PR regression path: current main passes `matched.binding.session` into `choose()`, while the PR branch computes effective scope from agent config/global default only. No live transport reproduction was run in this read-only review.

## Solution Assessment

No, not as-is. The maintainable solution is to add the agent-level default below binding overrides, document that precedence, and prove the mixed-agent behavior in a real setup.
