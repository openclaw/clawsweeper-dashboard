## Summary

Keep this PR open. Current main still only supports global `session.dmScope` and route-binding `bindings[].session.dmScope`; it does not accept or apply the requested per-agent default. The PR is useful, but it needs a focused rebase so the newer binding-level override keeps precedence over any agent-level fallback.

## What This Changes

The PR adds an optional `agents.list[].dmScope` config field across types, zod schema, labels/help, routing session-key resolution, and routing tests.

## Best Possible Solution

Add per-agent `agents.list[].dmScope` as an additive config contract while preserving precedence as `binding session override > agent default > global session default > main`. The implementation should update strict schema, generated config metadata, public docs/help, and routing tests for per-agent override, global fallback, binding interaction, groups, and cache behavior.

## Reproduction Assessment

Yes. The missing behavior is reproducible by inspection on current main: `AgentEntrySchema` is strict without `dmScope`, and `resolveAgentRoute` only reads binding/global dmScope when building the session key.

## Solution Assessment

No, not as-is. The feature direction is narrow and maintainable, but the current PR patch must be rebased so binding-level `session.dmScope` remains the highest-precedence override before agent/global fallback.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve binding-level dmScope precedence:** `src/routing/resolve-route.ts:663`
  - body: Current main already supports `bindings[].session.dmScope` and passes the matched binding session into route selection.
