## Summary

Keep open: the feature is plausible, but the PR is not mergeable as-is because it adds a new security-sensitive config precedence layer without preserving the merged binding-level `dmScope` override contract, and contributor real behavior proof is still missing.

## What This Changes

The PR adds an `agents.list[].dmScope` config field, schema/help labels, routing logic, and a test so individual agents can default DM session scoping differently from global `session.dmScope`.

## Best Possible Solution

Land the feature only if maintainers want per-agent default DM scope, with precedence `binding session override > agent default > global default > main`, aligned schema/help/tests, and redacted real mixed-agent runtime proof.

## Reproduction Assessment

Yes, for PR review. Source inspection shows current main preserves `bindings[].session.dmScope` through `choose(..., matched.binding.session)`, while the PR head only derives effective scope from agent config or global config; a focused test combining per-agent and per-binding scopes would reproduce the regression.

## Solution Assessment

No, not as-is. The per-agent default direction may be useful, but the maintainable solution must preserve binding overrides above agent defaults and needs maintainer approval for the new config surface.
