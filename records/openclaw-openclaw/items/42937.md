## Summary

Keep open: current main does not implement per-agent `dmScope`, but this PR needs changes before merge because its stale routing change must preserve existing binding-level session isolation and it is missing generated config metadata plus a changelog entry.

## What This Changes

The PR adds an optional `agents.list[].dmScope` config field with type/schema/help/label support, route session-key resolution changes, and a routing unit test for per-agent DM scope overrides.

## Best Possible Solution

Land an additive config contract with precedence `binding session override > agent default > global session default > main`, refreshed generated config metadata, changelog/docs coverage, and regression tests for both agent override and binding-over-agent behavior.

## Reproduction Assessment

Yes for the requested capability. Source inspection on current main shows `agents.list[].dmScope` is absent from the strict agent config schema and resolver, while existing tests and docs show binding-level `dmScope` precedence that a fix must preserve.

## Solution Assessment

No, not as-is. The feature direction is narrow, but the PR needs to be rebased around current binding precedence, regenerate config schema metadata, and add release documentation before merge.
