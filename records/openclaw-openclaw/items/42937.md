## Summary

Keep open: the requested per-agent `dmScope` surface is not on current `main`, but this PR cannot merge as-is because it lacks real behavior proof and would need to preserve binding-level `dmScope` precedence after rebase.

## What This Changes

The PR adds `agents.list[].dmScope` to config types/schema/help/labels, uses it when resolving DM session keys, shares a `DmScopeSchema`, and adds a routing unit test.

## Best Possible Solution

Land a rebased implementation with `bindings[].session.dmScope` as highest precedence, `agents.list[].dmScope` as a lower-precedence agent default, aligned config docs/schema/tests, and redacted mixed-agent runtime proof.

## Reproduction Assessment

Yes for the blocking PR regression path by source inspection: current main passes `matched.binding.session` into `choose()` and uses `sessionOverride?.dmScope`, while the PR head computes the effective scope from agent config/global default only. I did not run tests because this was a read-only review.

## Solution Assessment

No, not as-is. The useful feature should be implemented as binding override > agent default > global default, with schema/docs/tests and real behavior proof kept in sync.
