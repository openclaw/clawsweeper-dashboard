## Summary

Keep open: current main still lacks a per-agent `dmScope`, so the feature is not obsolete, but the PR is not merge-ready because it needs to preserve current binding-level DM-scope precedence, add real behavior proof, and get maintainer approval for the new config surface.

## What This Changes

The PR adds `agents.list[].dmScope` to config types/schema/help/labels and route resolution/tests so an agent can override global `session.dmScope`.

## Best Possible Solution

Land a per-agent DM-scope override only if maintainers want this config surface, with precedence `binding session override > agent default > global session default > main`, aligned schema/help/tests, and redacted mixed-agent runtime proof.

## Reproduction Assessment

Yes, for PR review: source inspection shows current main preserves `bindings[].session.dmScope`, while the PR diff computes scope from only agent config or global default. A focused regression combining `agents.list[].dmScope` with `bindings[].session.dmScope` would exercise the failure.

## Solution Assessment

No, not as-is. The feature direction is plausible, but the implementation must preserve binding-level overrides and the new config precedence needs maintainer approval before merge.
