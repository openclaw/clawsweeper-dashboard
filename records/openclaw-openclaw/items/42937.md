## Summary

Keep open. Current main does not implement per-agent `dmScope`, and the PR still targets a real feature gap, but it is not merge-ready because it is stale/conflicting against binding-level `dmScope` precedence and lacks contributor real behavior proof.

## What This Changes

The branch adds `agents.list[].dmScope` to config types/schema/help/labels, resolves routes with an agent-level DM scope default, reuses a shared zod schema, and adds a routing unit test.

## Best Possible Solution

Land a rebased version only if maintainers want the config surface, with precedence `binding session override > agent default > global default > main`, aligned schema/help/tests, and redacted real mixed-agent runtime proof.

## Reproduction Assessment

Yes, for PR review. Source inspection shows current main preserves `bindings[].session.dmScope`, while the PR head derives scope from agent config or global config only; a focused test combining per-agent and per-binding scopes would reproduce the regression.

## Solution Assessment

No, not as-is. The per-agent default direction may be useful, but the maintainable solution must preserve binding overrides above agent defaults and needs maintainer approval for the new config surface.
