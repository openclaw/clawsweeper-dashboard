## Summary

Keep open: current main still lacks per-agent `dmScope`, but this PR is not mergeable as-is because the branch would let agent/global DM scope outrank existing binding-level isolation and the contributor has not supplied real behavior proof.

## What This Changes

Adds `agents.list[].dmScope` type/schema/help/label support, reuses a shared DM-scope Zod schema, and changes route resolution/tests to prefer per-agent DM scope over global `session.dmScope`.

## Best Possible Solution

Land this only if maintainers want the config surface, with precedence `binding session override > agent default > global session default > main`, aligned metadata/docs/tests, and redacted mixed-agent runtime proof.

## Reproduction Assessment

Yes, for PR review: source inspection shows current main preserves `bindings[].session.dmScope`, while the PR diff computes the scope from only agent config or global default. A focused regression combining `agents.list[].dmScope` with `bindings[].session.dmScope` would exercise the failure.

## Solution Assessment

No, not as-is. The feature direction is plausible, but the implementation must preserve binding-level overrides before this is a maintainable solution.
