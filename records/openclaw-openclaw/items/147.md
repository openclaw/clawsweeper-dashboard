## Summary

Keep open. Current main has generic Gateway, node-event, Voice Wake, and OpenAI-compatible HTTP primitives that make an external Raspberry Pi voice bridge feasible, and the discussion includes a working external implementation, but OpenClaw still does not ship or document a supported Brabble/Raspberry Pi distributed voice-wake node or minimal external voice-node contract.

## What This Changes

Add support for running Brabble or a Raspberry Pi voice pipeline as a distributed OpenClaw voice-wake node.

## Best Possible Solution

Maintainers should choose the supported boundary first: document a minimal external voice-node contract and promote or link the Raspberry Pi companion through ClawHub/docs, or add a small generic node/plugin API only if current Gateway surfaces are insufficient.

## Reproduction Assessment

Not applicable as a feature/product request rather than a bug. The verification path is high-confidence: current-main source and docs show generic voice/node/Gateway primitives but no first-class Brabble or Raspberry Pi distributed voice-wake node.

## Solution Assessment

Unclear. Direct native Brabble integration is not proven to be the best path; current evidence favors defining the supported external-node contract and trust boundary first, then adding core or plugin API only if there is a real gap.
