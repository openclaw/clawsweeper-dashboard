## Summary

Keep open. Current main has generic Gateway, node, Voice Wake, and voice transcript primitives, and the discussion includes a working external Pi companion, but OpenClaw still does not ship or document a supported Brabble/Raspberry Pi distributed voice node or minimal external voice-node contract; prior triage left the product and security boundary unresolved.

## What This Changes

Request to run Brabble as an OpenClaw node for distributed wake-word voice activation, STT/TTS, and Gateway-routed agent responses.

## Best Possible Solution

Maintainers should define the supported boundary first: document a minimal external voice-node contract and promote/link the Pi companion through ClawHub/docs, or add a narrow generic node/plugin API only if current Gateway surfaces are insufficient.

## Reproduction Assessment

Not applicable; this is a feature/product request rather than a bug. The verification path is source-backed: current main has voice/node primitives but no first-class Brabble/Raspberry Pi node or supported external voice-node contract.

## Solution Assessment

Unclear. The posted external companion suggests /v1/chat/completions plus streaming can work without core changes, so native Brabble integration is not proven to be the narrowest maintainable path until maintainers define the support and trust boundary.
