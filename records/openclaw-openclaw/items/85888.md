## Summary

Keep open. Current main and the latest release still use the later natural cron slot as the floor for recurring cron error retries, the exact linked fix at https://github.com/openclaw/openclaw/pull/85896 was closed unmerged, and the remaining open adjacent PR does not change the live scheduled-run path that matches this report.

## What This Changes

Scheduled MiniMax cron runs should recover from retryable overload failures through the scheduled execution path instead of waiting until the next natural cron slot while manual reruns succeed.

## Best Possible Solution

Land a narrow cron scheduler fix that schedules retryable recurring cron execution errors at the backoff slot while preserving high-frequency floors, exhausted-budget behavior, and unresolved-next-run guards.
