## Summary

Keep open. The PR addresses a real media/audio configuration gap, but it should not merge as-is because it exposes a new private-network audio opt-in before the related SSRF hostname hardening is resolved and the branch is stale against current main’s audio JSON error handling.

## What This Changes

The PR adds audio-scoped `allowPrivateNetwork` request config, generated/docs/changelog updates, tests, and local OpenAI-compatible audio synthetic-auth handling.

## Best Possible Solution

Rebase the useful audio config/auth work onto current main, preserve the JSON error wrapper and lazy auth boundary, and land it only after SSRF hardening lands or maintainer security approval explicitly accepts the sequencing.

## Reproduction Assessment

Yes, source-level. Current main rejects or strips audio-scoped `allowPrivateNetwork` through the shared media schema and request sanitizers; I did not run a live transcription repro in this read-only review.

## Solution Assessment

No, not as-is. The direction is useful, but the branch needs SSRF sequencing, a rebase that preserves `readProviderJsonResponse`, and a fix for the eager `model-auth` import before it is the narrow maintainable solution.
