## What This Changes

The PR routes Google prompt-cache create and TTL-refresh response parsing through the shared byte-capped response reader and adds oversized streamed-response regressions.

## Best Possible Solution

Land the focused bounded-reader change after required exact-head CI passes, keeping the shared media-core limiter as the single bounded response-read path.

## Reproduction Assessment

I did not run the Vitest proof locally in this read-only review.

## Solution Assessment

Yes; replacing both successful cachedContents JSON reads with the existing shared byte limiter is the narrowest maintainable fix, and Google's documented create/patch responses are small `CachedContent` metadata objects. ([ai.google.dev](https://ai.google.dev/api/caching))
