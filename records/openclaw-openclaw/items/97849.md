## Summary

Keep open: current main still has the reported cross-turn leak because `annotateToolTurnOutcome` carries a later assistant reply backward until a user group is seen, and the existing regression tests only cover user-bounded turns.

## What This Changes

The issue asks Control UI/WebChat to keep failed cron or autonomous tool-error banners visible unless that same agent turn produced an assistant reply.

## Best Possible Solution

Add turn/run-aware outcome annotation in the shared chat projection, with a regression test for back-to-back agent turns and preservation of the merged within-turn collapse behavior.

## Reproduction Assessment

Yes, source-reproducible. I did not run the reporter's repro in this read-only pass, but current source confirms the backward pass can leak a later assistant reply across agent-initiated turns with no user boundary.

## Solution Assessment

No, current main is not the best state because it relies on user groups as the only turn boundary. The maintainable fix should use turn/run-aware metadata in the shared projection instead of a channel-specific renderer tweak.
