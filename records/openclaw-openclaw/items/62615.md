## Summary

Keep open. Current main has targeted retry and loop safeguards, but it still does not provide the requested first-class, configurable session-health circuit breaker with failure receipts, unhealthy-session state, and explicit recovery actions.

## What This Changes

The issue requests a gateway/runtime circuit breaker that stops retrying unhealthy sessions after bounded consecutive failures and surfaces operator-visible recovery state.

## Best Possible Solution

Design one owner-approved session-health path that records structured failure receipts, gates retry admission, and exposes resume/start-fresh recovery without duplicating the existing idle-timeout, post-compaction, tool-loop, and failover-suspension guards.
