## Summary

Keep open. Current main has several localized guardrails for retries, fallback, lane suspension, idle timeouts, and tool loops, but it still lacks the requested durable, configurable gateway/session health breaker with rolling failure tracking, operator-visible unhealthy state, resume/start-fresh controls, and token-spend policy.

## What This Changes

Add a configurable gateway-side session circuit breaker that marks repeated-failure sessions unhealthy and bounds or rotates retries with visible recovery state.

## Best Possible Solution

Define one durable gateway/session-health contract that records repeated same-session failure signals, suppresses automatic same-session retry when tripped, resets on success or explicit operator action, and exposes recovery state through session APIs, logs, and UI while coordinating with model fallback and provider cooldowns.

## Reproduction Assessment

Not applicable as a bug reproduction path; this is a feature request. Source inspection gives high-confidence proof that current main has partial mitigations but no durable configurable session-level breaker surface.

## Solution Assessment

Unclear until maintainers choose the session-health contract. A session-level breaker is directionally sound, but the maintainable path should integrate with quotaSuspension, model fallback, runRetries, session-store, logs, and UI recovery surfaces rather than adding another isolated retry guard.
