## Summary

Keep open. Current main has targeted retry guards, idle-timeout protection, post-compaction loop protection, and failover suspension, but it still lacks the requested general session-health circuit breaker with durable failure receipts, configurable recovery policy, and operator-facing resume/start-fresh semantics.

## What This Changes

Request a gateway/runtime circuit breaker that marks unhealthy sessions after repeated failures and stops bounded retry storms while preserving recovery metadata.

## Best Possible Solution

Define one owner-approved core session-health breaker that records durable failure receipts, applies bounded default policy, and exposes explicit resume/start-fresh recovery without duplicating the existing idle-timeout, post-compaction, tool-loop, or failover guards.

## Reproduction Assessment

No live current-main retry storm was reproduced in this read-only pass. Source inspection does show the requested general session-health breaker is still missing because current guards cover narrower idle-timeout, post-compaction, tool-loop, retry-cap, and failover-suspension paths.

## Solution Assessment

Unclear. The comment-proposed helper is a plausible starting point, but the best fix needs maintainer decisions on durable receipt shape, thresholds, defaults, UI/log surfacing, and resume versus fresh-session recovery semantics.
