## Summary

Keep open. Current main and the latest release have narrower retry, idle-timeout, tool-loop, and compaction guards, but not the requested first-class session-health breaker with persisted unhealthy-session state, receipt fields, recovery policy, auto-rotation, or UI/operator surfacing.

## What This Changes

The issue requests a gateway/runtime session-level circuit breaker that stops repeated unhealthy-session retries, records unhealthy state, surfaces operator alerts or UI, and optionally starts a fresh session.

## Best Possible Solution

Design one maintainer-approved session-health contract that composes with existing retry, failover, idle-timeout, tool-loop, and compaction guards before adding the minimal runtime, persistence, config, and operator surfaces.

## Reproduction Assessment

Yes at source level for the missing capability, but no live current-main retry storm was reproduced in this read-only pass. Current main lacks the requested general session-health breaker and receipt state even though it has narrower guards.

## Solution Assessment

No, the current narrower protections are only partial. The best fix is a unified session-health design rather than treating an idle-timeout subset, a broad retry cap, or tool-loop detectors as complete coverage.
