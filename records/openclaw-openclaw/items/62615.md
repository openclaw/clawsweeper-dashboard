## Summary

Keep open. Current main has partial retry-cost mitigations and lane suspension after model-fallback exhaustion, but it still lacks the requested durable, configurable session-level circuit breaker with consecutive failure/signature tracking, operator-visible unhealthy state, resume/start-fresh controls, and token-spend guardrails.

## What This Changes

Request a gateway-side circuit breaker that stops, pauses, or rotates unhealthy persistent sessions after repeated failures.

## Best Possible Solution

Define a durable gateway/session-health contract that records repeated same-session failure signals, suppresses automatic same-session retry when tripped, resets on success or explicit operator action, and exposes recovery state through session APIs, logs, and UI while coordinating with provider cooldown and model fallback.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a bug repro; this is a feature request, and source inspection confirms the full requested breaker surface is not present on current main.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers choose the session-health contract; a session-level breaker is directionally sound, but the safe path is a narrower design that fits existing fallback, suspension, session-store, and operator surfaces.
