## Summary

Keep open. Current main and the latest release have targeted retry and loop guards, but they still do not provide the requested first-class gateway/runtime session-health circuit breaker with durable unhealthy-session state, receipts, recovery actions, and configurable policy.

## What This Changes

Request a gateway/runtime session-level circuit breaker that stops retrying unhealthy persistent sessions after bounded consecutive failures and exposes clear recovery state.

## Best Possible Solution

Define one owner-approved session-health circuit breaker path in gateway/runtime with durable receipt fields, bounded retry admission, clear log/UI/API surfacing, and explicit resume-versus-start-fresh recovery semantics.

## Reproduction Assessment

Yes for the missing capability by source inspection: current main has narrower guards but no broad first-class unhealthy-session breaker. No live retry storm was reproduced in this read-only review.

## Solution Assessment

Unclear. A counter beside the idle-timeout breaker is plausible, but the best fix needs maintainer decisions on durable receipt fields, default thresholds, config shape, UI/log surfacing, and resume versus fresh-session recovery semantics.
