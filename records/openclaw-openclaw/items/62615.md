## Summary

Keep open. Current main has useful narrower guards for broad retry exhaustion, zero-progress idle timeouts, tool loops, and post-compaction loops, but it still lacks the requested first-class session-health breaker with failure receipts, unhealthy-session state, configurable policy, recovery actions, and UI/operator surfacing.

## What This Changes

The issue requests a gateway/runtime session-level circuit breaker that stops retrying unhealthy sessions after consecutive failures and exposes recovery state/actions.

## Best Possible Solution

Design one maintainer-approved session-health contract that composes with the existing retry, failover, idle-timeout, tool-loop, and compaction guards, then implement the minimal runtime/persistence/config/UI surfaces needed to expose safe recovery actions.

## Reproduction Assessment

Yes by source inspection for the missing capability, but no live current-main retry storm was reproduced in this read-only pass. Current code has narrower guards and lacks the requested general session-health breaker and receipt fields.

## Solution Assessment

No, current main is only a partial solution. The best path is a unified session-health design rather than treating the idle-timeout subset, tool-loop subset, or broad retry cap as complete coverage.
