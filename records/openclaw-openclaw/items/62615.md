## Summary

Keep open. Current main has partial cost and retry guardrails, including idle-timeout breaker behavior, configurable embedded-run retry limits, and fallback-exhaustion lane suspension, but it still lacks the requested durable, configurable session-level health breaker with rolling failure/signature tracking, operator-visible unhealthy state, resume/start-fresh controls, and token-spend policy.

## What This Changes

Request a configurable gateway/session circuit breaker that stops or redirects automatic retries after repeated same-session failures.

## Best Possible Solution

Define one durable gateway/session-health contract that records repeated same-session failure signals, suppresses automatic same-session retry when tripped, resets on success or explicit operator action, and exposes recovery state through session APIs, logs, and UI while coordinating with model fallback and provider cooldowns.

## Reproduction Assessment

Not applicable as a bug reproduction path; this is a feature request. Source inspection gives a high-confidence gap check that current main has partial mitigations but no durable configurable session-level breaker surface.

## Solution Assessment

Unclear until maintainers choose the session-health contract. A session-level breaker is directionally sound, but the maintainable path should integrate with existing fallback, quotaSuspension, runRetries, session-store, logs, and UI recovery surfaces rather than adding another isolated retry guard.
