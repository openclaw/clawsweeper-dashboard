## Summary

Keep open. Current main and the latest release have targeted retry guards for idle timeouts, post-compaction tool loops, broad run-iteration caps, and rate-limit failover, but they still do not provide the requested first-class, configurable session-health circuit breaker with failure receipts, unhealthy-session state, recovery recommendations, and resume/start-fresh controls.

## What This Changes

Request a gateway/runtime session-level circuit breaker that bounds retries for unhealthy sessions, marks session health, emits operator-visible state, and optionally rotates or resumes safely.

## Best Possible Solution

Design one owner-approved session-health path that reuses existing retry/failover signals, emits a structured health receipt, and adds config or UI surface only after default policy and upgrade behavior are agreed.
