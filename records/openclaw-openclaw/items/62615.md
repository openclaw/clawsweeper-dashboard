## Summary

Keep open. Current main added partial quota/session suspension after model-fallback exhaustion, but it still does not provide the requested configurable gateway/session circuit breaker with consecutive-failure thresholds, failure signatures, durable unhealthy-session state exposed to operators, resume/start-fresh controls, or token-spend guardrails.

## What This Changes

The issue requests a configurable gateway/runtime circuit breaker that stops retrying unhealthy sessions after repeated consecutive failures.

## Best Possible Solution

Define a durable session-health contract that records repeated same-session failure signals, suppresses automatic same-session retry when tripped, resets on success or explicit operator action, and exposes recovery state through session APIs, logs, and UI while coordinating with provider cooldown and model fallback.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not applicable as a bug repro; this is a feature request, and source inspection confirms the full requested breaker surface is not present on current main.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers choose scope; a session-level breaker is directionally sound, but the maintainable path should either fix and narrow https://github.com/openclaw/openclaw/pull/42933 or replace it with a smaller gateway/session-health design.
