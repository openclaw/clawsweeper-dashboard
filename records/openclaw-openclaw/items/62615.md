## Summary

Keep open. Current main has provider/profile cooldowns, model fallback, and a narrow subagent orphan-recovery tombstone, but it still lacks the requested durable gateway/session-level breaker that records consecutive unhealthy-session failures, marks a session unhealthy, suppresses same-session retries after a threshold, and exposes operator recovery controls.

## What This Changes

The issue requests a configurable gateway/session-level circuit breaker that stops retrying an unhealthy persistent session after repeated failures, marks it unhealthy, surfaces operator recovery state, and optionally starts a fresh session.

## Best Possible Solution

Define a durable session-health contract in the gateway/session lifecycle: record repeated failure signatures and retry exposure per session, reset on a successful model output or explicit operator action, suppress automatic same-session retry/resume when tripped, and surface recovery state through sessions RPC, status/UI, and logs while coordinating with provider cooldown/model fallback.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? No; this is primarily a feature request and the linked retry-storm report does not provide a runnable trace here, though source inspection does show the requested persistent session breaker is absent on current main.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers choose scope; a session-level breaker is directionally sound, but the maintainable path should reuse session lifecycle and fallback telemetry and decide whether to land, split, or replace #42933.
