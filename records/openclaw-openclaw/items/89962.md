## What This Changes

The PR wraps Discord `audioAsVoice` final payload delivery so rejected voice sends fall back to visible text or TTS supplement text, with regression tests for fallback text, reply targeting, and duplicate suppression.

## Best Possible Solution

Land this shape if maintainers want Discord final replies to prefer visible text over total loss after voice-send rejection; otherwise narrow the catch to failure classes known to happen before Discord accepts the voice message.
