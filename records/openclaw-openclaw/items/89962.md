## Summary

## What This Changes

The PR changes Discord outbound payload delivery so rejected voice-note sends can fall back to visible text or hidden TTS supplement text, with regression coverage.

## Best Possible Solution

Land this Discord-owned fallback if maintainers accept visible text as the recovery behavior for rejected voice-note final replies; otherwise narrow fallback to known pre-send voice failures.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.6 await `sendVoice` before sending visible text in the Discord `audioAsVoice` payload branch, so a rejected voice send exits before text delivery. The PR body also includes after-fix live Discord output for the patched path, but I did not run a live current-main repro in this read-only review.

## Solution Assessment

Yes, if maintainers accept the fallback policy; the change is in the Discord outbound payload owner and reuses the existing reply-payload TTS supplement contract. The safer alternative is to limit fallback to known pre-send failures if duplicate delivery after ambiguous voice-send failures is unacceptable.
