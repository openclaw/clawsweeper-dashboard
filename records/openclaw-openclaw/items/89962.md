## Summary

## What This Changes

The PR adds Discord `audioAsVoice` fallback logic so rejected voice-note sends can fall back to visible text or TTS supplement text, with regression tests for fallback text, reply targeting, and duplicate suppression.

## Best Possible Solution

Land this shape if maintainers want Discord final replies to prefer visible text over total loss after voice-send rejection; otherwise narrow the catch to failure classes that are known to occur before Discord accepts the voice message.
