The PR changes Discord audioAsVoice outbound payload delivery to send visible text or hidden TTS supplement text when voice delivery rejects.

## Best Possible Solution

Land this Discord-owned fallback after resolving conflicts if maintainers accept visible text recovery for rejected voice-note final replies; otherwise narrow the catch to known pre-send failures.

## Reproduction Assessment

## Solution Assessment

Yes if maintainers accept the fallback policy; the change lives in the Discord outbound payload owner and reuses the reply-payload TTS supplement contract. The safer alternative is to catch only known pre-send failures if ambiguous duplicate delivery is unacceptable.
