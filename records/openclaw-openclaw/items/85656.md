## Best Possible Solution

Normalize legacy group targets for every Telegram outbound platform call derived from recovered queue entries, including pinning, while keeping the generic durable queue unchanged.

Mostly. Outbound-boundary normalization is the maintainable plugin-owned fix, but this patch should include `pinDeliveredMessage` before merge so required pinned retries do not fail after sending.
