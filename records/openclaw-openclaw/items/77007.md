## Summary

Keep open. Current main still routes non-verbose external-chat CLI timeout failures through the generic fallback, and #77015 is an open closing PR rather than a merged fix, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue asks for Telegram and diagnostics to distinguish Claude CLI no-output and overall timeouts from generic gateway, provider, and channel failures.

## Best Possible Solution

Review and land #77015 or an equivalent narrow classifier in buildExternalRunFailureReply with regression coverage, then split any timeout-default or broader diagnostics policy into separate follow-up work.

## Solution Assessment
