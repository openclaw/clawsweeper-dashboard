## Summary

Keep open. Current main still preserves Claude CLI timeouts as structured timeout failures, but the non-verbose external-chat reply path collapses them into the generic Telegram-style failure copy; the linked fix PR is open and unmerged.

## What This Changes

The issue requests Telegram and diagnostics copy that distinguishes Claude CLI no-output or turn timeouts from generic gateway, provider, auth, billing, or channel failures.

## Best Possible Solution

Review and land a narrow external-chat timeout classifier for gateway-generated CLI timeout messages with regression coverage, then close this issue through the linked PR.

## Reproduction Assessment

Yes. A focused current-main unit test can reject the agent run with `CLI exceeded timeout (300s) and was terminated.` or `CLI produced no output...` in a non-verbose external-chat context and observe the generic failure copy.

## Solution Assessment

Yes for the narrow bug fix. Matching OpenClaw-generated CLI timeout templates before the generic fallback is the smallest maintainable path; timeout default changes and broader diagnostics should remain separate follow-ups.
