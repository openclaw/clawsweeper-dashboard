## Summary

Keep this PR open. Current main still classifies only numeric `sessions.json.bak.<digits>` store backups, while disk-budget cleanup depends on that classifier before evicting real session entries; the PR is a narrow fix with targeted tests and no blocking review finding, but it is currently dirty against main and needs maintainer review/rebase before merge.

## What This Changes

The PR broadens session store backup artifact detection from numeric `sessions.json.bak.<digits>` names to any non-empty `sessions.json.bak.*` suffix and adds regression assertions for incident backup names.

## Best Possible Solution

Rebase this PR onto current main while preserving the central classifier change and regression tests, then run the targeted artifact helper test and the changed-session lane before merging if CI stays green.
