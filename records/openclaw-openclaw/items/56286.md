## Summary

Keep open: current main and the latest release still resolve Telegram approval prompts with `clear-actions`, which clears buttons but leaves the approval message content in chat; the same-repo fix PRs are closed unmerged, and the remaining fork PR is conflicting and targets an obsolete handler path.

## What This Changes

The report asks for resolved Telegram exec approval prompts to disappear after approval instead of leaving a stale approval message in chat.

## Best Possible Solution

Fix the active Telegram native approval runtime so resolved approvals use the shared `delete` final-action path, expired approvals keep the button-only clear path, and the fix includes focused tests plus redacted real Telegram proof.

## Reproduction Assessment

Yes, source-level: current main resolves Telegram approval finalization with `clear-actions`, and the shared runtime leaves delivered content in place for that action. I did not run a live Telegram bot flow in this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is to use the existing shared `delete` final-action seam in `extensions/telegram/src/approval-handler.runtime.ts`, not to revive the removed delayed-delete handler branch.
