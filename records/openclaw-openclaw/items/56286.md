## Summary

Keep this issue open. Current `main` still resolves Telegram approval clicks by clearing inline buttons instead of deleting the approval message, and the provided GitHub context shows open PR #56319 already uses closing syntax for this bug.

## What This Changes

The issue requests that Telegram native exec approval messages disappear after an approve or deny click instead of remaining in the chat.

## Best Possible Solution

Review and either land PR #56319 or replace it with a narrow Telegram plugin change that removes inline actions immediately and then deletes the pending native approval message through the existing final-action/delete-message seam, with regression coverage for approve and deny resolution.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction exists on current `main`: resolving a Telegram approval click reaches `clearCallbackButtons()`, and the native resolved path returns `clear-actions`, both of which leave the approval message present.

## Solution Assessment

Yes for keeping this issue open. The narrow maintainable fix belongs in the Telegram plugin approval runtime and tests, using the existing shared delete final-action seam rather than a core migration or unrelated channel behavior change.
