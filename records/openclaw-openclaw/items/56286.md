## Summary

Keep open: current main still leaves the Telegram approval message in place by clearing only inline buttons, and open PR #56319 is the active implementation candidate.

## What This Changes

The issue asks Telegram to delete or dismiss exec approval messages after an approver clicks approve, instead of leaving the approval bubble in chat.

## Best Possible Solution

Review and land or replace PR #56319 with a narrow Telegram plugin change that deletes pending approval messages after approve/deny while preserving immediate button removal and regression coverage.

## Reproduction Assessment

Yes, at source level. The callback path resolves the approval and then clears only the inline keyboard, while the native runtime also resolves to `clear-actions` rather than deleting the pending message.

## Solution Assessment

Yes for keeping this issue open. Because an open PR already uses closing syntax for the same Telegram bug, the narrowest maintainable path is to review, update, or replace that PR rather than queue duplicate work from this issue.
