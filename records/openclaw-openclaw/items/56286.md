## Summary

Keep open. Current main and the latest release still clear Telegram approval buttons instead of deleting the approval message, and open PR #56319 already uses closing syntax for this bug, so this issue should wait for that PR or a maintainer-approved replacement to merge.

## What This Changes

The issue asks for Telegram exec approval prompt messages to disappear from chat after the user clicks approve.

## Best Possible Solution

Review and either land PR #56319 or replace it with a narrow Telegram plugin change that removes inline actions immediately and then deletes the resolved approval message through the existing final-action/delete-message seam, with regression coverage for approve and deny.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction exists on current main: resolve a Telegram native approval callback and the active paths reach `clearCallbackButtons()` or `clear-actions`, both of which clear markup rather than deleting the message.

## Solution Assessment

Yes for keeping this issue open. The maintainable fix belongs in the Telegram approval native runtime and tests, using the existing shared delete final-action seam rather than a core migration or unrelated channel behavior change.
