## Summary

Keep open: current main still clears Telegram approval inline buttons instead of deleting the approval message, and open PR #56319 explicitly targets this issue.

## What This Changes

The issue asks for Telegram exec approval messages to disappear after the user approves an exec request.

## Best Possible Solution

Review and either land PR #56319 or replace it with a narrow Telegram plugin change that removes inline actions immediately and deletes the pending approval message through the existing delete final-action/message-delete seam, with regression coverage for approve and deny.

## Reproduction Assessment

Yes, at source level. Resolving a Telegram approval on current main reaches `clear-actions`/`clearCallbackButtons()`, which only clears inline buttons and leaves the approval message in chat.

## Solution Assessment

Yes for keeping this issue open. Because an open PR already uses closing syntax for the same bug, the best path is to review, update, or replace that PR rather than queue a duplicate repair from this issue.
