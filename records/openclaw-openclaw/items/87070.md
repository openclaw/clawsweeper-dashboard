## What This Changes

The PR updates the Codex app-server run attempt to treat queued terminal turn notifications and enqueue-time active-turn notifications as idle-watch activity.

## Best Possible Solution

Land the focused Codex app-server fix after maintainer availability review, ideally with a targeted queue-delay regression test or explicit acceptance that the provided live proof is enough for this race.

## Reproduction Assessment

## Solution Assessment

Yes, the proposed path is a narrow owner-boundary fix for the race: it uses the existing terminalTurnNotificationQueued state and updates active-turn receipt timestamps instead of changing provider routing or fallback policy. A focused regression test would make the solution easier to land with confidence.
