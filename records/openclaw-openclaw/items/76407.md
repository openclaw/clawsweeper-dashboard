## Summary

Keep open. Current main still has unbounded cache-trace and Anthropic payload JSONL sinks, and this PR addresses the right narrow surface, but the patch should bound the new archive-count input before merge.

## What This Changes

The PR adds bounded rotation and archive retention for cache-trace and Anthropic payload JSONL diagnostics, with config/env/schema/docs/changelog updates and focused tests.

## Best Possible Solution

Land the narrow diagnostic JSONL rotation fix after bounding archive-count inputs and passing the normal targeted checks, leaving gateway.err.log, journald, and session JSONL starvation to their separate reports.

## Reproduction Assessment

Yes. Source inspection of current main shows both diagnostic sinks append through `getQueuedFileWriter` without rotation options, while the primary logger has an existing rotate-and-continue model.

## Solution Assessment

Yes, with one small hardening fix. Extending the shared queued JSONL writer and wiring the two diagnostic sinks is the narrow maintainable solution, but the new public `maxArchives` value should be capped before it drives filesystem loops.
