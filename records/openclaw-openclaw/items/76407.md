## Summary

Keep open: current main still lacks bounded rotation for these diagnostic JSONL sinks, and this PR is a useful narrow fix, but it needs a small hardening pass before merge.

## What This Changes

The PR adds queued-writer rotation/archive support for cache-trace and Anthropic payload diagnostic JSONL logs, plus config/env/schema/docs/changelog updates and focused tests.

## Best Possible Solution

Land the diagnostic JSONL rotation after bounding archive counts and preserving redaction and file-safety checks, leaving gateway.err.log, journald, and session JSONL starvation to their separate reports.

## Reproduction Assessment

Yes. Source inspection on current main shows both diagnostic sinks append through getQueuedFileWriter without rotation options, and the queued writer has only cap-and-drop behavior.

## Solution Assessment

Yes, with one hardening fix. Extending the shared queued writer and wiring both diagnostic sinks is the narrow maintainable solution, but public maxArchives input should be capped or rejected before it drives rotation loops.
