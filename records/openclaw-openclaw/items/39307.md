## Summary

Keep open: current main and v2026.5.22 still make built-in compaction notices opt-in and route post-compaction recovery through internal system context, so the requested default user-visible reset signal and explicit assistant disclosure remain unresolved product work.

## What This Changes

Requested user-visible signaling and explicit assistant recovery behavior when a session compacts or resets mid-thread.

## Best Possible Solution

Define a maintainer-owned compaction/reset disclosure policy, then implement consistent visible signaling and assistant recovery wording across the reply runner, supported chat surfaces, docs, and focused tests.

## Reproduction Assessment

Yes, at source level. Current main gates built-in chat notices behind agents.defaults.compaction.notifyUser === true, documents compaction as silent by default, and injects post-compaction recovery as internal prompt context rather than guaranteed user-visible disclosure.

## Solution Assessment

No current implementation fully solves this issue. Existing opt-in notices, hooks, WebChat history dividers, and post-compaction context refresh are useful partial pieces, but the default visibility and recovery-language policy still need maintainer product judgment.
