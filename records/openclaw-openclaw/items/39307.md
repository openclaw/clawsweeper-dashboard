## Summary

Keep open: current main and v2026.6.8 still make channel compaction notices opt-in, while the requested default user-visible disclosure plus assistant re-grounding behavior remains an audience-sensitive product decision.

## What This Changes

The issue requests a visible user signal and explicit assistant recovery disclosure when mid-thread compaction or context reset changes working context.

## Best Possible Solution

Define one typed, audience-aware compaction lifecycle disclosure policy that coordinates channel notices, Control UI indicators, and optional assistant re-grounding language without hardcoding internal recovery wording into every persona.

## Reproduction Assessment

Yes, at source level: emit a compaction event with `agents.defaults.compaction.notifyUser` unset and the runner keeps `onBlockReply` silent while internal callbacks can still fire. This proves the default user-visible signal gap without needing a live transport run.

## Solution Assessment

Unclear until maintainers choose the product contract. The existing opt-in notices are useful but not the requested default disclosure, and related customer-facing notice concerns show that the best fix must be audience-aware rather than an unconditional self-announcement.
