## Summary

Keep open: current main has partial coverage through opt-in compaction notices, transient Control UI status, channel callbacks, and internal post-compaction context injection, but it still lacks the requested default user-visible reset signal plus assistant-side disclosure and reload offer. Related #72043 also shows the default visibility and suppression policy needs maintainer product direction before a broad cross-channel implementation.

## What This Changes

The issue requests a default-visible mid-thread compaction/context-reset signal plus assistant-side disclosure and re-grounding behavior across supported chat surfaces.

## Best Possible Solution

Define a durable compaction/reset event contract with default visibility and suppression semantics, then implement user-visible disclosure and assistant re-grounding/reload-offer behavior through the reply runner, post-compaction recovery, Control UI, supported chat surfaces, docs, and focused tests.

## Reproduction Assessment

Yes for the requested current gap. Source inspection and existing tests show that default-config compaction events do not send a fallback user-facing onBlockReply notice, while docs explicitly document silent-by-default behavior.

## Solution Assessment

No current implementation is not the complete solution requested here. The existing opt-in notices and internal post-compaction grounding are useful partial pieces, but changing default visibility safely needs a maintainer-owned product contract coordinated with #72043.
