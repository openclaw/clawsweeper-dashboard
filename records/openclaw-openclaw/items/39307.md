## Summary

Keep open: current main has partial compaction visibility through Control UI indicators, compacted-history dividers, opt-in chat notices, and internal post-compaction context injection, but the requested default visible reset signal plus assistant-side disclosure/reload offer is still not implemented and needs maintainer policy direction.

## What This Changes

The issue asks OpenClaw to visibly signal mid-thread compaction/context reset and have the assistant explicitly disclose and re-ground after compaction.

## Best Possible Solution

Define a durable compaction/reset event and disclosure policy, then implement consistent user-visible reset signaling and assistant re-grounding/reload-offer behavior across the reply runner, post-compaction recovery, Control UI, supported chat surfaces, docs, and focused tests.

## Reproduction Assessment

Yes for the observable gap: source inspection shows default chat compaction notices remain silent, tests lock that behavior, and searches found no assistant user-facing reset/reload disclosure. No live channel run was needed for this source-reproducible feature review.

## Solution Assessment

No current implementation is not the best complete solution for the issue. The existing opt-in notices, Control UI status, history divider, and internal post-compaction grounding are useful partial pieces, but default visibility and assistant disclosure copy need maintainer policy direction before a cross-channel implementation.
