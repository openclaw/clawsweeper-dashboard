Request a hard-to-miss user-visible signal and assistant-side disclosure/re-grounding behavior when a session compacts or resets mid-thread.

## Best Possible Solution

Define a maintainer-owned disclosure policy, then implement consistent compaction/reset signaling and recovery wording across the reply runner, supported chat surfaces, docs, and focused tests.

## Reproduction Assessment

Yes, at source level. Current main gates built-in chat notices behind agents.defaults.compaction.notifyUser === true, and post-compaction recovery is queued as trusted System prompt context rather than guaranteed user-visible disclosure.

## Solution Assessment

No current implementation fully solves the issue. Existing opt-in notices, hooks, and post-compaction context refresh are useful partial pieces, but default visibility and recovery-language policy need maintainer product judgment first.
