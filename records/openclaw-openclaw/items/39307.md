## Summary

Keep open: current main and the latest shipped release still make compaction notices opt-in and keep post-compaction recovery as internal system-event context, so the requested default user-visible disclosure and assistant recovery behavior remain unresolved product work.

## What This Changes

The issue requests visible mid-thread compaction/reset signaling plus explicit assistant recovery wording when working context has been reduced or refreshed.

## Best Possible Solution

Define a maintainer-owned compaction/reset disclosure policy, then implement consistent visible signaling and assistant recovery wording across the reply runner, supported chat surfaces, docs, and focused tests.

## Reproduction Assessment

Yes, at source level. Current main and v2026.5.20 gate built-in chat notices behind agents.defaults.compaction.notifyUser === true, document compaction as silent by default, and queue post-compaction recovery as system-event context rather than guaranteed user-visible disclosure.

## Solution Assessment

No current implementation fully solves this issue. Existing opt-in notices, hooks, status/log surfaces, and post-compaction context refresh are useful partial pieces, but default visibility, noise level, persona impact, and recovery wording need maintainer product judgment first.
