## Summary

Keep open: current main and the latest release only partially address this. Compaction notices are still silent by default unless the opt-in notice path, callbacks, or hook messages are configured, and the post-compaction recovery prompt is internal model context rather than a guaranteed user-visible disclosure plus reload offer.

## What This Changes

The issue requests a visible mid-thread compaction/reset signal plus explicit assistant recovery disclosure and reload offer.

## Best Possible Solution

Define a durable disclosure policy, then implement consistent compaction/reset signaling and assistant re-grounding/reload-offer behavior across the reply runner, recovery prompt, Control UI, supported chat surfaces, docs, and focused tests.

## Reproduction Assessment

Yes, at source level. Current main leaves default chat compaction notices silent unless notifyUser, hook messages, or callbacks are configured, and post-compaction recovery is queued as internal system context rather than guaranteed visible disclosure.

## Solution Assessment

No current implementation is the full solution. Existing opt-in notices, the bundled hook, Control UI indicator, and post-compaction context refresh are useful partial pieces, but the default disclosure and recovery wording need maintainer product direction.
