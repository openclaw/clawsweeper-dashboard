## Summary

Keep open: current main and v2026.5.22 still make built-in compaction notices opt-in, document silent default behavior, and inject post-compaction recovery as internal system context rather than guaranteed user-visible disclosure.

## What This Changes

Request a hard-to-miss user-visible signal and explicit assistant recovery disclosure when a session compacts or resets mid-thread.

## Best Possible Solution

Define a maintainer-owned compaction/reset disclosure policy, then implement consistent visible signaling and assistant recovery wording across the reply runner, supported chat surfaces, docs, and focused tests.

## Reproduction Assessment

Yes, at source level. Current main gates built-in chat notices behind agents.defaults.compaction.notifyUser === true, documents compaction as silent by default, and injects post-compaction recovery as internal prompt context rather than guaranteed user-visible disclosure.

## Solution Assessment

No current implementation fully solves this issue. Existing opt-in notices, hooks, WebChat status indicators, and post-compaction context refresh are useful partial pieces, but the default visibility and recovery-language policy still need maintainer product judgment.
