## Summary

Keep open: current main has useful partial compaction visibility, but default chat notices remain opt-in and post-compaction recovery is still an internal system-event prompt rather than a guaranteed user-facing reset disclosure plus reload offer.

## What This Changes

This issue requests a user-visible signal plus assistant recovery disclosure when mid-thread compaction or context reset changes the agent's working context.

## Best Possible Solution

Define a durable compaction/reset disclosure policy, then implement consistent reset signaling and assistant re-grounding/reload-offer behavior across the reply runner, post-compaction recovery, Control UI, supported chat surfaces, docs, and focused tests.

## Reproduction Assessment

Yes. Source inspection against current main shows the default path leaves fallback notices gated behind notifyUser while post-compaction recovery is injected as internal system context rather than a guaranteed visible assistant disclosure.

## Solution Assessment

No current implementation is the full solution. The existing opt-in notices, bundled hook, Control UI indicator, and Discord reaction are useful partial pieces, but default visibility, suppression, and recovery wording need maintainer direction.
