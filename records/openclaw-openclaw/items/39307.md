## Summary

Keep open: current main and the latest release have useful partial compaction visibility, but the default chat path remains silent unless explicitly opted in or a hook supplies messages, and post-compaction recovery is still injected as internal model context rather than a guaranteed user-facing reset disclosure plus reload offer.

## What This Changes

Surface a visible user signal and assistant recovery disclosure when a session compacts or resets mid-thread.

## Best Possible Solution

Define a durable disclosure policy, then implement consistent compaction/reset signaling and assistant re-grounding/reload-offer behavior across reply runner, recovery prompts, Control UI, supported chat surfaces, docs, and focused tests.

## Reproduction Assessment

Yes. Source inspection and focused tests show the default path leaves chat notices silent unless notifyUser or hook messages are configured, while post-compaction recovery is internal system context rather than a guaranteed visible assistant disclosure.

## Solution Assessment

No current implementation is the full solution. Existing opt-in notices, the bundled hook, Control UI indicator, Discord reaction handling, and post-compaction context refresh are useful partial pieces, but default visibility, suppression, and assistant recovery wording need maintainer direction.
