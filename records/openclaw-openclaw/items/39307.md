## Summary

Keep open: current main still treats chat-visible compaction notices as opt-in and queues post-compaction recovery as internal system-event context, so the requested default visibility and assistant-side recovery disclosure remain unresolved product policy.

## What This Changes

The issue asks OpenClaw to visibly signal mid-thread compaction/reset and have the assistant disclose and re-ground afterward.

## Best Possible Solution

Define a maintainer-owned compaction/reset disclosure policy, then implement consistent visible signaling and assistant recovery wording across the reply runner, supported chat surfaces, docs, and focused tests.

## Reproduction Assessment

Yes, at source level. Current main gates built-in chat notices behind agents.defaults.compaction.notifyUser === true, documents compaction as silent by default, and queues post-compaction recovery as internal system-event context rather than guaranteed user-visible disclosure.

## Solution Assessment

No current implementation fully solves this issue. Existing opt-in notices, hooks, Control UI status, verbose status, and post-compaction context refresh are useful partial pieces, but the default visibility and recovery-language policy still need maintainer product judgment.
