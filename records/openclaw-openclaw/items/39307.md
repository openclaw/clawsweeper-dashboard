## Summary

Keep open: current main partially covers the trust gap with opt-in compaction notices and post-compaction system-event reinjection, but it still defaults to silent compaction and does not guarantee assistant self-disclosure or a reload/re-grounding offer after context is reduced.

## What This Changes

Request a visible user signal and explicit assistant recovery behavior when mid-thread compaction or context reset changes the agent's working context.

## Best Possible Solution

Define the default/opt-in disclosure policy first, then implement one compaction lifecycle contract that drives supported channel/UI notices and assistant recovery wording with focused tests and docs.

## Reproduction Assessment

Yes, at source level. Current main explicitly keeps compaction notices silent unless notifyUser is enabled, and the recovery path injects internal system context rather than guaranteed user-visible disclosure plus a reload offer.

## Solution Assessment

No, current main is only a partial solution. The best path needs a maintainer product decision on default visibility and audience policy before wiring a consistent channel/UI and assistant recovery contract.
