## Summary

Keep open: current main has partial compaction visibility through opt-in chat notices, Control UI status, a bundled hook, and internal post-compaction context injection, but the requested default visible reset signal plus assistant-side disclosure/reload offer is still not implemented and needs maintainer policy direction.

## What This Changes

Request to surface mid-thread compaction/context resets to users and make assistant recovery behavior explicit across supported chat surfaces.

## Best Possible Solution

Define a durable compaction/reset event contract with default and suppression semantics, then implement user-visible disclosure and assistant re-grounding/reload-offer behavior through the reply runner, post-compaction recovery, Control UI, supported chat surfaces, docs, and focused tests.

## Reproduction Assessment

Yes for the observable gap: source inspection shows default compaction notices are silent, tests lock in that default, and docs document the opt-in notifyUser behavior. No live channel run was needed for this read-only feature review.

## Solution Assessment

No current implementation is not the best complete solution for this request. The existing opt-in notices, Control UI status, bundled hook, and internal post-compaction grounding are useful partial pieces, but changing default visibility safely should be coordinated with the related recovery-notice policy thread.
