## Summary

Keep open. Current main has partial compaction visibility, but it intentionally keeps chat notices silent by default and the post-compaction prompt is agent-internal rather than a required user disclosure or reload offer.

## What This Changes

The issue requests a visible mid-thread compaction/context-reset signal plus explicit assistant-side post-compaction disclosure and re-grounding behavior.

## Best Possible Solution

Define the intended disclosure policy first, then make compaction/reset events consistently surface through existing UI, status-reaction, hook, and `notifyUser` paths while adding an assistant recovery prompt only where maintainers want that user-facing behavior.

## Reproduction Assessment

Not applicable as a bug reproduction. Source and docs inspection give a high-confidence current-behavior picture: compaction notices are silent by default, optional surfaces exist, and post-compaction context refresh is internal to the agent prompt.

## Solution Assessment

Unclear. Current main is a partial solution, but making disclosure hard to miss and requiring assistant self-announcement is a product decision because current policy deliberately avoids noisy default chat messages.
