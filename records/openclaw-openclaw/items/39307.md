## Summary

Keep open: current main still defaults compaction to silent and only exposes hidden prompt/runtime context unless operators opt in, while https://github.com/openclaw/openclaw/pull/82520 is an open, not-ready partial fix for hidden summary injection rather than the requested visible disclosure policy.

## What This Changes

Request a visible user signal and assistant recovery disclosure when an OpenClaw session compacts or resets mid-thread.

## Best Possible Solution

Define a core compaction/reset disclosure policy that reuses existing compaction notice and system-event plumbing, keeps channel wording concise, and specifies when the assistant must self-disclose and offer to re-ground.

## Reproduction Assessment

Not applicable as a failing bug, but there is a high-confidence source check for current behavior: compaction notices are silent by default and post-compaction refresh text is hidden and opt-in. I did not run a live channel compaction scenario.

## Solution Assessment

Unclear: the requested direction is plausible, but the best fix depends on maintainer policy for default visibility, channel noise, and assistant self-disclosure wording. The existing notifyUser and post-compaction prompt hooks are likely the right starting points.
