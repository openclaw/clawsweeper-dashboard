Keep open: current main has useful partial pieces, but built-in chat compaction notices are still default-silent and the recovery prompt is model-facing system context rather than guaranteed user-visible disclosure.

## What This Changes

Request a visible mid-thread compaction/context-reset signal plus explicit assistant recovery wording that tells users the working context changed and offers re-grounding help.

## Best Possible Solution

Define a maintainer-owned default disclosure and recovery-language policy, then implement consistent compaction/reset signaling and assistant re-grounding behavior across reply runner, supported chat surfaces, docs, and focused tests.

## Reproduction Assessment

Yes, at source level. Current main gates built-in chat notices behind `agents.defaults.compaction.notifyUser === true`, and post-compaction recovery is queued as trusted system context rather than guaranteed user-visible disclosure.

## Solution Assessment

No current implementation fully solves this issue. Existing opt-in notices, hooks, and post-compaction context refresh are useful partial pieces, but the requested default visibility and reload-offer behavior need a maintainer product decision first.
