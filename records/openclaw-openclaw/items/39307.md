## Summary

Keep open. Current main still keeps compaction notices silent by default and only provides opt-in notices, internal post-compaction guidance, transient Control UI status, and channel-specific callbacks; that does not satisfy the requested default user-visible reset signal plus assistant-side disclosure/reload-offer behavior. Related #72043 also shows the default visibility and suppression policy needs maintainer product judgment.

## What This Changes

Request a visible mid-thread compaction/context-reset signal and assistant recovery disclosure/reload-offer behavior across supported chat surfaces.

## Best Possible Solution

Define a durable compaction/reset event contract with explicit default and suppression semantics, then implement user-visible disclosure plus assistant re-grounding/reload-offer behavior through the reply runner, post-compaction recovery, Control UI, and supported chat surfaces, with docs and focused tests updated together.

## Reproduction Assessment

Yes. Current-main source and tests provide a high-confidence reproduction path: simulate a compaction event with default config and observe that no fallback user-facing notice is sent, while docs confirm the default remains silent.

## Solution Assessment

No current implementation is not the best complete solution for this request. The existing opt-in notice and internal prompt are useful partial pieces, but the maintainable fix needs an explicit product contract coordinated with #72043 before changing defaults.
