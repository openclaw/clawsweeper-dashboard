## Summary

Keep open: current main and v2026.5.12 only partially cover this. Users can opt into compaction notices and Control UI has an indicator, but default chat compaction remains silent and the recovery prompt is model-facing context rather than a guaranteed user-visible disclosure with a reload offer.

## What This Changes

Request a visible mid-thread compaction/context-reset signal plus explicit assistant recovery disclosure and reload offer after compaction.

## Best Possible Solution

Define a durable disclosure policy, then implement default-safe compaction/reset signaling and explicit re-grounding/reload-offer behavior across the reply runner, recovery prompt, Control UI, supported chat surfaces, docs, and focused tests.

## Reproduction Assessment

Yes, at source level. Current main gates chat compaction notices behind agents.defaults.compaction.notifyUser === true, and post-compaction recovery is queued as trusted system context rather than guaranteed visible disclosure plus reload offer.

## Solution Assessment

No current implementation is the full solution. Existing notifyUser notices, callbacks/hooks, Control UI status, and post-compaction context refresh are useful partial pieces, but maintainers still need to decide the default visibility/noise policy and user-facing recovery wording.
