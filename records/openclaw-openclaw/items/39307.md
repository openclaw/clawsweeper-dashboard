## Summary

Current `main` has more partial implementation than the earlier review noted: a merged compaction visibility PR added a bundled `compaction-notifier` hook and continuation retry path. It still does not satisfy the issue’s default cross-channel visibility plus assistant self-disclosure/reload-offer criteria, so the item should stay open for product-level design.

## What This Changes

Request a visible mid-thread compaction/reset signal and explicit assistant recovery disclosure with a reload/context re-grounding offer.

## Best Possible Solution

Define a durable compaction/reset event and disclosure policy, then implement consistent reset signaling plus assistant re-grounding/reload-offer behavior across the reply runner, post-compaction recovery, Control UI, supported chat surfaces, docs, and focused tests.

## Reproduction Assessment

Yes for the observable gap: current source and tests show default chat notices remain silent unless notifyUser or an enabled hook supplies messages, and no reload-offer implementation was found. This is a feature request rather than a broken documented behavior.

## Solution Assessment

No current implementation is not yet the full solution. The opt-in notices, bundled hook, Control UI indicators, compacted-history divider, and internal post-compaction context are useful partial pieces, but the default visibility and assistant disclosure policy still need maintainer design before a focused implementation is safe.
