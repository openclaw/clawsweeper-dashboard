## Summary

Keep open: current main still treats user-visible compaction notices as opt-in and queues post-compaction recovery as internal system-event context, so the requested default visibility and assistant recovery disclosure remain unresolved product policy rather than conservative cleanup.

## What This Changes

Request a visible user signal and assistant recovery disclosure when compaction or context reset changes working context mid-conversation.

## Best Possible Solution

Define a maintainer-owned compaction/reset disclosure policy, then implement consistent visible signaling and assistant recovery wording across the reply runner, supported chat surfaces, docs, and focused tests.

## Reproduction Assessment

Yes, at source level. Current main gates built-in chat notices behind agents.defaults.compaction.notifyUser === true, documents compaction as silent by default, and queues post-compaction recovery as internal system-event context rather than guaranteed user-visible disclosure.

## Solution Assessment

No current implementation fully solves this issue. Existing opt-in notices, hooks, verbose status, and post-compaction context refresh are useful partial pieces, but the default visibility and recovery-language policy still need maintainer product judgment.
