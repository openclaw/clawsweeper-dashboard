## Summary

Keep open: current main has partial compaction visibility through Control UI indicators, compacted-history dividers, opt-in chat notices, and internal post-compaction context injection, but the requested default visible reset signal plus assistant-side disclosure/reload offer is still not implemented and needs maintainer policy direction.

## What This Changes

Request default user-visible signals and assistant recovery disclosure/reload offers when mid-thread compaction or context reset changes working context.

## Best Possible Solution

Define a durable compaction/reset event and disclosure policy, then implement consistent user-visible reset signaling and assistant re-grounding/reload-offer behavior across the reply runner, post-compaction recovery, Control UI, supported chat surfaces, docs, and focused tests.

Yes for the observable gap: source inspection shows default chat compaction notices are silent, tests enforce that default, and searches found no assistant user-facing reset/reload disclosure. No live channel run was needed for this source-reproducible feature review.

## Solution Assessment

No current implementation is not the best complete solution for the issue.
