## Summary

Keep open: current main and the latest release only partially address compaction visibility. Chat notices remain silent by default unless `notifyUser`, hooks, or callbacks are configured, and the post-compaction recovery text is model-facing context rather than a guaranteed user-visible disclosure plus reload offer.

## What This Changes

The issue requests a visible mid-thread compaction/reset signal and explicit assistant recovery disclosure after working context is refreshed.

## Best Possible Solution

Define a durable disclosure policy, then implement consistent compaction/reset signaling and assistant re-grounding/reload-offer behavior across the reply runner, recovery prompts, Control UI, supported chat surfaces, docs, and focused tests.

## Reproduction Assessment

Yes, at source level. Current main leaves default chat notices silent unless `notifyUser`, hook messages, or callbacks are configured, and post-compaction recovery is queued as internal system context rather than a guaranteed visible disclosure.

## Solution Assessment

No current implementation is the full solution. Existing opt-in notices, the bundled hook, Control UI indicator, Discord reaction handling, and post-compaction context refresh are useful partial pieces, but the default disclosure and recovery wording need maintainer product direction.
