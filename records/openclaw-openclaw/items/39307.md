## Summary

Keep open: current main and the latest release only provide opt-in channel notices plus partial Control UI visibility, while the requested default user-visible disclosure and assistant re-grounding policy remains a maintainer product decision.

## What This Changes

The issue requests a visible signal and explicit assistant recovery behavior when a session compacts or resets mid-thread.

## Best Possible Solution

Define one typed compaction lifecycle disclosure policy that consistently drives channel notices, Control UI markers, and optional assistant re-grounding language without forcing internal wording into every persona.

## Reproduction Assessment

Yes, at source level: trigger compaction with `agents.defaults.compaction.notifyUser` unset and the channel path sends no user notice, while internal callbacks and Control UI markers may still exist. This proves the central default-visibility gap without needing a live transport run.

## Solution Assessment

Unclear until maintainers choose the product contract. The safest direction is a shared lifecycle disclosure policy with explicit default, audience, and surface semantics rather than an unconditional assistant self-announcement or another one-off config branch.
