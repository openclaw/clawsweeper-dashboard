## Summary

Keep open: current main has useful partial coverage through opt-in compaction notices, transient Control UI status, channel callbacks, and internal post-compaction context injection, but it still does not provide the requested default user-visible reset signal plus assistant-side disclosure and reload offer. Related #72043 also shows the default visibility and suppression policy needs maintainer product direction before a broad implementation.

## What This Changes

Request a user-visible mid-thread compaction/reset signal and assistant recovery disclosure with a reload-context offer across supported chat surfaces.

## Best Possible Solution

Define a durable compaction/reset event contract with default and suppression semantics, then implement user-visible disclosure and assistant re-grounding/reload-offer behavior through the reply runner, post-compaction recovery, Control UI, supported chat surfaces, docs, and focused tests.

## Reproduction Assessment

Yes. Source inspection and existing tests give a high-confidence path: simulate a compaction event with default config and observe that no fallback user-facing `onBlockReply` notice is sent, while docs confirm the silent default.

## Solution Assessment

No current implementation is not the best complete solution for this request. The existing opt-in notices and internal post-compaction grounding are useful partial pieces, but changing default visibility safely needs an explicit product contract coordinated with #72043.
