## Summary

Keep open: current main partially addresses visibility through opt-in channel notices and a Control UI checkpoint divider, but the default channel experience remains silent and no runtime contract guarantees assistant disclosure or an offer to re-ground after compaction.

## What This Changes

Request visible user notification and explicit assistant re-grounding behavior whenever conversational context is compacted mid-thread.

## Best Possible Solution

Keep the Control UI checkpoint boundary, then define one typed compaction lifecycle disclosure policy that consistently drives channel notices and optional assistant re-grounding language without forcing internal wording into every persona.

## Reproduction Assessment

Yes, at source level: trigger automatic compaction with `agents.defaults.compaction.notifyUser` unset and channel users receive no notice, while Control UI history alone receives an unconditional checkpoint divider.

## Solution Assessment

No, unconditional assistant self-announcement is not yet the best universal solution; the narrow maintainable path is a shared lifecycle event with explicit default, audience, and surface semantics.
