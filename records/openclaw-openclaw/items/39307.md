## Summary

Keep open. Current main has a recent partial implementation with opt-in compaction notices and opt-in internal post-compaction prompt reinjection, but it still documents compaction as silent by default and does not guarantee the visible disclosure plus assistant self-announcement fallback requested here.

## What This Changes

The issue requests a visible user-facing signal and explicit assistant recovery behavior when mid-thread context compaction or reset changes the agent’s working context.

## Best Possible Solution

Keep this as the disclosure-policy tracker and have maintainers decide the default/noise policy, then implement it through the existing compaction notice and system-event seams with tests across preflight and runtime compaction paths.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes by source inspection: current defaults keep compaction silent, and post-compaction prompt injection is opt-in/internal rather than a guaranteed user-visible disclosure path.

## Solution Assessment

Is this the best way to solve the issue? Yes as a product tracker; the concrete implementation should reuse existing compaction notice/system-event seams and avoid a separate channel-specific mechanism.
