## Summary

Keep open: the reported stream bug is valid, but this patch is too broad because it would execute stop-finished `tool_calls` even when the stream already contains visible assistant text, contradicting current regression coverage for spurious tool calls.

## What This Changes

The PR adds a Chat Completions stream finalizer guard that promotes `finish_reason: stop` to `toolUse` when structured tool-call blocks were assembled, plus a mocked regression test for that stream shape.

## Best Possible Solution

Land a narrowed finalizer that promotes silent stop-finished structured tool calls to `toolUse` while preserving the existing stop-plus-visible-text and length/error safeguards, with regression coverage for both cases.

## Reproduction Assessment

Yes, source-reproducible: current main maps `finish_reason: stop` to `stop` and then drops tool-call blocks, matching the linked silent stop-plus-tool-call report. The PR-introduced regression is also source-reproducible because the new guard would make the existing stop-plus-visible-text spurious-tool-call test fail.

## Solution Assessment

No: the symmetric guard is a plausible fix for the silent case, but it is not the best fix because it ignores the existing visible-text spurious-tool-call safeguard. The narrower maintainable solution is to promote only the silent structured-tool-call case while preserving the current stop-plus-text behavior.
