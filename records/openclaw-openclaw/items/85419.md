## Summary

Keep open: this is a recent Telegram delivery regression report against v2026.5.20, and current docs/source say normal group requests should still use automatic final delivery unless the turn is configured as message-tool-only or ambient room context.

## What This Changes

The issue reports that Telegram group messages on v2026.5.20 process successfully but their automatic final replies are not sent after an upgrade from v2026.5.18.

## Best Possible Solution

Confirm the redacted v2026.5.20 Telegram log/config path first; if a normal `user_request` group turn resolves to message-tool-only or skips automatic final delivery, fix that delivery-mode handoff and add focused Telegram/source-delivery regression coverage.

## Reproduction Assessment

No high-confidence exact reproduction yet. Source inspection shows the no-send shape is expected for `message_tool` or `room_event` turns, but the reported minimal `requireMention: false` path should remain a normal `user_request` with automatic delivery unless additional config is present.

## Solution Assessment

Unclear until the delivery mode for the reported turn is proven. The maintainable fix is not to remove message-tool-only handling, but to ensure ordinary Telegram group user requests keep automatic final delivery and only ambient/opt-in rooms suppress finals.
