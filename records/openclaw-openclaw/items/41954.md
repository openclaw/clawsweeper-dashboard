This PR should stay open, but not merge as-is: current main has not adopted the module split, and the stale extraction would drop current cron UI payload, markdown, and run-navigation behavior.

## What This Changes

The PR extracts cron view helpers from `ui/src/ui/views/cron.ts` into `filters.ts`, `helpers.ts`, `jobs.ts`, and `runs.ts` under `ui/src/ui/views/cron/`.

## Best Possible Solution

Re-extract the current cron view while preserving guarded payload handling, sanitized markdown rendering, run body/error rendering, and in-app chat navigation.
