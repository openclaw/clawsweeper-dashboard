## Summary

Keep this PR open, but do not merge it as-is. The extraction was reported as faithful to its old base, but current main now has guarded payload handling, sanitized markdown rendering, error-body behavior, and in-app run-chat navigation that the extracted modules would drop.

## What This Changes

This PR splits the Control UI cron view helpers from `ui/src/ui/views/cron.ts` into `filters.ts`, `helpers.ts`, `jobs.ts`, and `runs.ts` under `ui/src/ui/views/cron/`.

## Best Possible Solution

Re-extract the current `ui/src/ui/views/cron.ts` implementation into focused modules while preserving payload validation, sanitized markdown rendering, run error/body behavior, and in-app chat navigation.

## Reproduction Assessment

Yes for PR correctness: static comparison of the PR patch against current main shows clear stale-payload, markdown, run-body, and navigation regressions. Current cron view tests cover those behaviors.

## Solution Assessment

No. The PR’s extraction is not the best mergeable solution anymore; the safer path is a current-main extraction that preserves the covered behavior, or closing the cleanup if maintainers decide the churn is not worth carrying forward.
