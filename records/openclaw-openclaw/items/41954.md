## Summary

Keep open: this is a reasonable cleanup idea, but the branch is stale against current cron UI behavior and would drop payload guards, sanitized markdown rendering, run body/error layout, and in-app chat navigation; it also lacks real behavior proof.

## What This Changes

This PR extracts cron view filters, helpers, job rendering, and run rendering from `ui/src/ui/views/cron.ts` into sibling modules under `ui/src/ui/views/cron/`.

## Best Possible Solution

Re-extract the current cron view, preserving payload validation, sanitized markdown rendering, run body/error behavior, and in-app chat navigation, then provide real Control UI proof.

## Reproduction Assessment

Yes for PR correctness: static comparison of the PR diff with current main shows the dropped behavior, and `ui/src/ui/views/cron.test.ts` has focused coverage for those cases. I did not run tests because this was a read-only review.

## Solution Assessment
