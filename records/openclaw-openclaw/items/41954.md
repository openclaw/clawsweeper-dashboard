## Summary

Keep open, but the patch is not mergeable as-is: current main still has the monolithic cron view, while this branch was extracted from older code and drops covered cron UI behavior added after the PR was opened.

## What This Changes

The PR extracts the Control UI cron view filters, helpers, job rendering, and run rendering from `ui/src/ui/views/cron.ts` into sibling modules under `ui/src/ui/views/cron/`.

## Best Possible Solution

Refresh the extraction from current `ui/src/ui/views/cron.ts`, preserve the payload guards, sanitized markdown rendering, in-app run navigation, and run-body layout, then provide real Control UI proof.

## Reproduction Assessment

Yes for PR correctness: source comparison against current main shows the extracted modules omit the current payload guard, sanitized markdown paths, `onNavigateToChat`, and run-body rendering. Current focused tests cover those paths, though they were not executed in this read-only review.

## Solution Assessment

No. The module split is a reasonable cleanup direction, but the branch was extracted from older cron UI code; the maintainable path is a fresh extraction from current main rather than merging the stale modules.
