Keep open: current main and the latest release still lack serialized commitments writers, and the active fix path is the open canonical PR with closing syntax, so this issue should remain open until that PR or an equivalent fix merges.

## What This Changes

Serialize commitments-store load/modify/save mutations so concurrent status, attempt, upsert, and expiry writes cannot overwrite each other.

## Best Possible Solution

Review and land the canonical commitments-store serialization PR, using a commitments-local per-path queue plus file lock and regression tests, then close this issue from the merge.
