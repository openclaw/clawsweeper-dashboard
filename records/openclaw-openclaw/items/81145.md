## Summary

Keep open: current main still has unsynchronized commitments-store whole-file mutation paths, and the issue is already paired with an open closing PR that should be reviewed and landed before the issue closes.

## What This Changes

The issue requests serialized commitments-store writes so concurrent status, attempt, inferred-upsert, and expiry mutations cannot silently overwrite each other.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81153, or an equivalent commitments-local queue plus file-lock fix with concurrency regressions, then close this issue after the fix reaches main.

## Reproduction Assessment

Yes. The provided `Promise.all` repro targets exported store functions, and current main still shows unsynchronized load-modify-save paths that can read the same snapshot before either save completes.

## Solution Assessment

Yes.
