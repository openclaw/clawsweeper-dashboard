## Summary

Keep open: current main and the latest stable source still have the unsynchronized commitments-store writes, and an open closing PR already targets the narrow queue-plus-file-lock fix.

## What This Changes

The issue reports lost updates in commitments-store mutators because concurrent load-modify-save calls overwrite each other.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81153 or an equivalent narrow commitments-store writer gate, then close this issue from the merged fix.

## Reproduction Assessment

Yes. The provided `Promise.all` repro directly exercises exported commitments-store mutators, and current main source confirms both calls can read stale snapshots and later overwrite whole-file state; I did not execute it in this read-only review.

## Solution Assessment

Yes. A commitments-local per-path mutation gate with an in-process queue plus `withFileLock` is the narrow maintainable fix, and the linked PR implements that direction with focused regression tests.
