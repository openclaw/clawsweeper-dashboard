## Summary

Keep open: current main still has the unsynchronized commitments store load-modify-save paths, and the open linked fix PR closes this issue but has not merged yet.

## What This Changes

The issue asks for commitments store writes to be serialized so concurrent status, attempt, and upsert mutations cannot lose each other.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81153 if CI and maintainer review stay clean, then close this issue through that merged fix.

## Solution Assessment

Yes. A commitments-local mutation gate using a per-path queue plus the existing file-lock contract is the narrow maintainable fix, and the linked PR is already aimed at that boundary.
