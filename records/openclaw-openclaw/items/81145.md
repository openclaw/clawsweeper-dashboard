## Summary

Keep open: current main still has unsynchronized whole-file commitments writes, and live GitHub shows https://github.com/openclaw/openclaw/pull/81153 is an open closing PR with the proposed serialization fix, so this issue should stay open until that PR lands or is superseded.

## What This Changes

Serialize commitments-store load-modify-save mutations so concurrent status, attempt, and inferred-upsert writes preserve independent updates.

## Best Possible Solution

Review and land a commitments-specific mutation gate that combines per-store in-process serialization with a cross-process file lock, plus focused concurrent-writer regression coverage.

## Reproduction Assessment

Yes. The issue includes a deterministic `Promise.all` repro against exported store functions, and current main source still shows the same unguarded load-modify-save sequence for status, attempt, and upsert paths; I did not execute it in this read-only pass.

## Solution Assessment

Yes. A commitments-specific queue plus file lock is the narrow maintainable fix because an in-process-only queue would not cover CLI and gateway writers in separate processes.
