## Summary

Keep open: current main still has unsynchronized commitments-store read-modify-save paths, and the open closing PR has not merged yet.

## What This Changes

The issue asks for serialized commitments-store write mutations so concurrent status, attempt, and inferred upsert calls cannot lose independent updates.

## Best Possible Solution

Review and land the canonical serialization fix in https://github.com/openclaw/openclaw/pull/81153, then close this issue as implemented with regression coverage.

## Reproduction Assessment

Yes. The provided `Promise.all` repro targets exported store functions, and current main still has the same unsynchronized load-modify-save paths; I did not run it in this read-only review.

## Solution Assessment

Yes. A commitments-local writer gate using both an in-process per-path queue and file locking is the narrow maintainable fix; an in-process-only session-store queue would miss CLI-versus-gateway writers.
