## Summary

Keep open: the retry direction is useful, but the patch is not merge-ready because APT's `--fix-missing` can silently hold back required packages and the contributor has not provided inspectable real build proof.

## What This Changes

The PR adds `Acquire::Retries=5` to sandbox Dockerfile apt update/install commands and adds `--fix-missing` to the install commands in the base and common sandbox images.

## Best Possible Solution

Use bounded apt retry handling that remains fail-closed for all required packages, then provide redacted build output proving both sandbox images still contain the expected tools.

## Reproduction Assessment

Yes for source-level review: the PR adds `--fix-missing` to required package installs, and apt-get documents that command-line packages that cannot be downloaded can be silently held back. I did not reproduce the intermittent bad-network Docker failure live.

## Solution Assessment

No: bounded apt retries are the right direction, but `--fix-missing` is not the best repair because it can hide missing required packages. The safer solution is fail-closed retries, with explicit verification only if maintainers keep `--fix-missing`.
