## Summary

This PR should stay open because the retry-only part is a useful candidate for the linked sandbox build reliability issue, but the current patch is not merge-ready: `--fix-missing` changes APT installs from fail-closed to potentially incomplete builds, and the external PR still lacks inspectable after-fix build proof.

## What This Changes

The PR adds `Acquire::Retries=5` to APT update/install commands and `--fix-missing` to APT installs in both sandbox Dockerfiles.

## Best Possible Solution

Land a retry-only sandbox Dockerfile change that uses `Acquire::Retries` on both APT update/install commands, keeps package installs fail-closed, and includes real build plus package-presence proof for both images.

## Reproduction Assessment

Yes for the introduced risk: the PR diff and APT manual show `--fix-missing` can hold back requested packages. No for the original intermittent network failure itself; there is no controlled failing build log or network-failure repro in the PR.

## Solution Assessment

No. `Acquire::Retries` is the narrow maintainable fix for transient fetch failures, but `--fix-missing` is not the best path because it can hide missing explicitly requested packages unless maintainers intentionally accept partial-install semantics with verification.
