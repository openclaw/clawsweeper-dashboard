## What This Changes

The PR adds APT retry options to `apt-get update` and `apt-get install`, and adds `--fix-missing` to installs, in both sandbox Dockerfiles.

## Best Possible Solution

Land a retry-only sandbox Dockerfile change that keeps installs fail-closed, then require redacted build logs or terminal proof showing both images build and expected packages are present.

## Reproduction Assessment

No for the intermittent network failure itself; the PR does not provide a high-confidence way to reproduce that condition. Yes for the introduced risk: the diff and APT manual make the `--fix-missing` fail-open behavior source-reproducible.

## Solution Assessment

No. `Acquire::Retries` is the narrow maintainable fix for transient fetch failures, but `--fix-missing` is the wrong addition because it can hold back command-line selected packages; retry-only plus real build proof is safer.
