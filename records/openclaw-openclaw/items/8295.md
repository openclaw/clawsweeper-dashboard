Keep open: current main still lacks Telegram `allowBots` in config types, strict schema, UI/help metadata, docs, and group/topic inbound policy. The Plaiground ClawHub skill from the discussion is a useful Discord workaround, but the Telegram PR attempts are closed unmerged and do not ship this bundled Telegram behavior.

## What This Changes

Add Telegram group/topic `allowBots` support so bot-authored messages can be accepted under explicit config, matching Discord/Slack bot-to-bot behavior.

## Best Possible Solution

Keep this open until maintainers either validate and add scoped Telegram group/topic `allowBots` config, runtime filtering, docs, generated metadata, and regression tests, or document the Telegram platform boundary and recommend Telegram channels or the existing Discord Plaiground workaround.
