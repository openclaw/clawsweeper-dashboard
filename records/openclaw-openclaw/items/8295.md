## Summary

Keep open: current main still has no Telegram `allowBots` config contract, and the newer Telegram Bot-to-Bot/Guest Mode path adds update-shape, routing, authorization, and loop-prevention questions that need maintainer direction.

## What This Changes

Add a Telegram group bot-to-bot opt-in comparable to Discord/Slack `allowBots` so configured Telegram bots can process bot-authored group/topic messages.

## Best Possible Solution

Define one Telegram bot-to-bot contract, then implement scoped account/group/topic opt-in plus Guest Mode support, explicit sender/account authorization, loop safeguards, docs, and live QA coverage.

## Reproduction Assessment

Yes for the repository gap: source inspection shows current main still has no accepted Telegram `allowBots` field and no Guest Mode handler, while the runtime relies on sender allowlists. I did not run the live two-bot Telegram setup in this read-only review.

## Solution Assessment

Unclear. The requested `allowBots` knob is plausible, but Telegram's Bot-to-Bot and Guest Mode API now requires a maintainer choice about config shape, multi-account routing, authorization, and loop prevention before implementation.
