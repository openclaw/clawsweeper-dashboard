## Summary

This PR remains useful and is not obsolete: current main still reads Mantis Discord smoke API bodies with unbounded `response.text()`, while the branch switches that single read point to the existing bounded SDK helper. I found no concrete patch defect, but the intentional fail-closed 16 MiB cap and currently failing required checks leave this as maintainer-review work rather than an auto-close or repair item.

## What This Changes

The branch replaces the Mantis Discord smoke API response text read with `readResponseWithLimit` and adds regression coverage for oversized, under-cap, and malformed Discord responses.

## Best Possible Solution

Land the bounded read with the existing SDK helper once maintainers accept the 16 MiB fail-closed policy and required CI is green or explicitly waived.

## Solution Assessment
