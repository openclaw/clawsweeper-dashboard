## Summary

Keep open: current main still strips or omits Gemini thought signatures on cross-provider/API/model replay paths, and the overlapping open PR is not merged and appears to cover only part of the reported gap.

## What This Changes

The issue requests fixing Gemini 3.x subagent and cross-provider replay so `thoughtSignature` is preserved or a Google-documented fallback is attached instead of hanging on tool-call continuations.

## Best Possible Solution

Land one coordinated fix that preserves real Gemini signatures through replay boundaries, uses Google’s dummy signature only for unsigned transferred function-call history, and covers native Google plus OpenAI-compatible replay with regression tests.

## Reproduction Assessment

Yes, by source inspection rather than live execution: current main can serialize a Gemini-targeted assistant `toolCall` without a `thoughtSignature` after a provider/API/model switch. That matches Google’s documented Gemini 3 validation failure mode and the reporter’s timeout transcripts.

## Solution Assessment

Yes on direction: fixing the replay serialization and sanitization boundary is the narrow maintainable solution. The repair should preserve valid signatures and only use the Google sentinel when no captured signature exists for transferred history.
