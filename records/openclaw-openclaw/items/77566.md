## Summary

Keep open: current main still strips or fails to synthesize Gemini function-call thought signatures across provider/model replay paths, matching the reporter's subagent hang and the upstream 400-contract failure mode.

## What This Changes

The issue asks OpenClaw to preserve or safely synthesize Gemini `thoughtSignature` values during cross-provider subagent/tool-call replay and to make missing-signature failures visible.

## Best Possible Solution

Land one focused repair that preserves real Gemini signatures through replay boundaries, uses Google's sentinel only for unsigned transferred function-call history, and adds regression coverage for native Google plus OpenAI-compatible cross-provider replay.

## Reproduction Assessment

Yes, by source inspection rather than live execution: current main can serialize Gemini-targeted assistant tool calls without a `thoughtSignature` after cross-provider/model replay, matching the reporter's timeout transcripts and Google's documented 400 path.

## Solution Assessment

Yes on direction: the narrow maintainable fix is at the replay serialization/sanitization boundary, but it should be coordinated with #75713 because that open PR appears to cover only part of the gap.
