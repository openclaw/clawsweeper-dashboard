## Summary

Keep open: current main still has source-level paths that drop or omit Gemini `thoughtSignature` during cross-provider/native replay, and the overlapping open PR #75713 is not merged and appears to cover only part of the gap reported here.

## What This Changes

The issue asks OpenClaw to preserve or synthesize Gemini 3.x `thoughtSignature` values across subagent/model-switch replay and to surface the failure instead of timing out silently.

## Best Possible Solution

Land one focused repair that preserves real Gemini signatures through replay boundaries, uses Google's sentinel only for unsigned transferred function-call history, and adds regression coverage for native Google plus OpenAI-compatible cross-provider replay.

## Reproduction Assessment

Yes, by source inspection rather than live execution: current main can carry a Gemini-targeted assistant `toolCall` to native or OpenAI-compatible Google replay without a `thoughtSignature`, matching Google's documented 400 path and the reporter's timeout transcripts.

## Solution Assessment

Yes on direction: fixing the replay serialization/sanitization boundary is the narrow maintainable solution, but the open PR should be reviewed against this issue because it appears incomplete for the cross-provider gates.
