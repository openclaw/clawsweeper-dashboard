## What This Changes

The issue asks OpenClaw to preserve or provide Gemini `thoughtSignature` values during cross-provider subagent/tool replay so Gemini 3.x tool-call turns do not hang or return missing-signature errors.

## Best Possible Solution

Keep #72879 as the canonical bug thread and review or land #75713, or an equivalent maintainer-cleaned fix, with regression coverage for native Google and OpenAI-compatible cross-provider replay.

I did not run a live Gemini call in this read-only review, but #72879 and this issue provide matching runtime evidence.
