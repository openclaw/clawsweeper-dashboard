## Summary

Keep open. The PR targets a real, recent Windows/Telegram image-stall bug and includes credible Windows after-fix proof, but it has a protected `maintainer` label and introduces a security-sensitive media-root regression that should be fixed before merge.

## What This Changes

The PR adds a fast static image-model resolution path, threads `workspaceDir` through media-understanding/provider-auth paths, updates provider stream lookup plumbing, adds regression tests, and adds a changelog entry for the Windows Telegram stall.

## Best Possible Solution

Keep the fast image-model resolution path, but keep provider/auth `workspaceDir` separate from media attachment authorization so only `MediaWorkspaceDir`, default media roots, and channel-configured roots permit local media reads.

## Reproduction Assessment

Yes, enough for review: the linked issue gives concrete Windows Telegram steps and event-loop starvation logs, and the PR body includes an after-fix Windows packaged-install probe. I did not rerun the live VM probe in this read-only pass.

## Solution Assessment

No, not as-is. The fast static image-model path is the right direction, but the agent workspace must not become a fallback media attachment root when `MediaWorkspaceDir` is absent.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Keep agent workspace out of media attachment roots:** `src/media-understanding/runner.ts:282`
  - body: This fallback now treats the agent workspace as an allowed local media root whenever a message has a relative `MediaPath` but no `MediaWorkspaceDir`. Since the new callers pass `workspaceDir` for provider/auth scoping, a channel/plugin-provided relative media path can be resolved and read from the agent workspace instead of being limited to the media store or channel-configured inbound roots. Keep `workspaceDir` threaded to provider/auth resolution, but do not use it as a media root unless it came from `ctx.MediaWorkspaceDir`.
  - confidence: 0.78

## Security Review

Status: needs_attention
