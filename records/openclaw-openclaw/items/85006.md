## Summary

Keep open: current main no longer shows the suspected 20-entry Telegram topic cache and preserves topic IDs through the main send/session paths, but this recent, detailed report still lacks current-main live proof with 21+ forum topics and is already marked for maintainer review.

## What This Changes

The issue asks OpenClaw to support bidirectional Telegram forum-topic replies in groups with more than 20 active topics without losing `message_thread_id`.

## Best Possible Solution

Run a real Telegram supergroup proof with 21+ forum topics; if it reproduces, patch the narrow session/thread-state boundary, and if it does not, close with exact proof and version provenance.

## Reproduction Assessment

No high-confidence current-main reproduction is available. The issue has detailed live observations, but this review did not run a 21+ topic Telegram supergroup and current source inspection did not find the claimed 20-entry cap.

## Solution Assessment

Unclear. Current main has the right topic-routing shape, but the best fix cannot be chosen until live proof isolates whether a topic/session cache, async route, account route, or Telegram-side state is still failing at 21+ topics.
