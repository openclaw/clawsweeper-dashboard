## Summary

Close as implemented: current `main` and the `v2026.5.4` release tree already pin terminal mirrored TaskFlow timestamps to completion time and include maintenance to reconcile stale mirrored flow records.

## What This Changes

Report that `openclaw tasks audit` warns about succeeded TaskFlows whose `endedAt` is earlier than `updatedAt`.

## Best Possible Solution

Keep the shipped write-time timestamp pinning and use `openclaw tasks maintenance --apply` for existing stale mirrored TaskFlow records.

## Reproduction Assessment

No high-confidence live reproduction against current main. Source fixtures show the stale-record audit warning shape, while the current mirrored-flow update path and regression test show newly synced terminal mirrored flows keep `updatedAt` equal to `endedAt`.

## Solution Assessment

Yes. Pinning terminal mirrored flow `updatedAt` to the task completion timestamp and leaving old-state cleanup to `tasks maintenance --apply` is the narrowest maintainable fix.
