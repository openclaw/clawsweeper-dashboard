## Summary

Keep open: current main and the latest shipped tag still allow managed TaskFlow terminal writes where caller-supplied `endedAt` can be earlier than `updatedAt`, and the linked closing PR is still open rather than merged.

## What This Changes

The issue asks for TaskFlow terminal writes to preserve `endedAt >= updatedAt` so `openclaw tasks audit` stops warning on succeeded flows.

## Best Possible Solution

Land a focused TaskFlow finish/fail timestamp guard with regression coverage and after-fix audit proof or maintainer override, then let the merged PR close this issue.

## Reproduction Assessment

Yes. Source inspection shows `finishFlow` and `failFlow` can persist `endedAt < updatedAt`, and TaskFlow audit reports that exact state as `inconsistent_timestamps`; I did not run a live CLI repro in this read-only review.

## Solution Assessment

Yes in direction. Normalizing terminal timestamps in the TaskFlow finish/fail write path is the narrow maintainable fix, but the linked PR should be reviewed, trimmed if needed, and landed before this issue closes.
