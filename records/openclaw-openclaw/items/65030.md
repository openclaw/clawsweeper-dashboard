## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Pass discovery config into profile fallback:** `extensions/amazon-bedrock/discovery.ts:293`
  - body: When an inference profile has no matching foundation model, this fallback reads `config?.defaultContextWindow`, but `config` is not defined inside `resolveInferenceProfiles`. TypeScript should reject this, and the fallback path would throw at runtime; thread the discovery config into this helper instead.
  - confidence: 0.98
- **[P1] Declare provenance on ProviderRuntimeModel:** `src/agents/pi-embedded-runner/run/setup.ts:115`
  - body: The new access on `params.runtimeModel` uses `ProviderRuntimeModel`, but that type does not declare `contextWindowIsLiteralDefault`. The same applies to the other runtime call sites, so the shared runtime model type needs the optional field or the typecheck lane will fail.
  - confidence: 0.95
- **[P2] Preserve discovered contextWindow in inline layering:** `src/agents/pi-embedded-runner/model.ts:377`
  - body: The layering path spreads `...inlineMatch` after `...discoveredModel`; partial inline entries can carry `contextWindow: undefined` and wipe out the discovered context window while preserving the new flag. Restore the discovered `contextWindow` when the inline model did not explicitly set it.
  - confidence: 0.9
- **[P2] Add provenance to the discovery cache key:** `extensions/amazon-bedrock/discovery.ts:165-167`
  - body: The new flag depends on whether `defaultContextWindow` was omitted versus explicitly set to the same number, but the cache key still only contains the resolved numeric window. With refresh caching enabled, one provenance mode can reuse the other's cached discovery result until expiry.
  - confidence: 0.89
- **[P2] Make default-source windows trigger the promised warning:** `src/agents/context-window-guard.ts:51-53`
  - body: Marking the fallback value as source `default` does not currently make the guard warn, because the guard still only warns when tokens are missing or below threshold. Add the default-source warning path or coordinate this with the related warning PR before this delivers the promised warning.
  - confidence: 0.88
