Keep open: current main and the latest shipped Feishu plugin still replace the SDK request interceptor by assigning to Axios `handlers`, and the dependency contract shows a public interceptor API is available, so this remains a narrow maintainability cleanup rather than a close candidate.

The issue asks OpenClaw to stop clearing Feishu's User-Agent interceptor through Axios `handlers` internals and cover the real request-interceptor behavior.

## Best Possible Solution

Refactor Feishu User-Agent handling to use the public Axios/Lark request contract or the existing HTTP wrapper path, preserving the current OpenClaw UA behavior and adding focused regression coverage for the actual interceptor or outgoing request options.

## Reproduction Assessment

No, there is not a high-confidence current-main runtime failure path. Source inspection does confirm the direct `handlers = []` mutation and the missing focused UA coverage, but the broken behavior depends on a dependency representation change or upgrade scenario.

## Solution Assessment

Yes, the requested direction is the maintainable path. The narrowest fix is to stop assigning `handlers` directly, keep the Feishu OpenClaw User-Agent behavior unchanged, and cover it with a focused regression test.
