## What This Changes

The issue asks to stop replacing Feishu's Axios request interceptors by assigning to Axios' internal `handlers` array.

## Best Possible Solution

Refactor Feishu HTTP User-Agent handling onto public Lark/Axios or `HttpInstance` contracts, remove the direct `handlers` assignment, and add focused coverage that exercises the actual interceptor or wrapper behavior.

## Reproduction Assessment

No high-confidence runtime failure path is established on current main. Source inspection does confirm the direct `handlers = []` mutation and the missing test coverage for that module-load behavior.

## Solution Assessment

Yes, the requested direction is the maintainable fix. The narrow path is to stop assigning `handlers` directly, preserve the OpenClaw Feishu User-Agent through a public request wrapper or typed interceptor API, and cover it in `client.test.ts`.
