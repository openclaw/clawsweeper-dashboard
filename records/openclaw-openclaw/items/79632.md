## Summary

Keep open. Current main has the focused Brave/global web_search owner fix, but both source inspection and the latest maintainer comment show the broader external npm plugin registry/runtime inconsistency remains for stale openclaw.extension.json stubs, workspace node_modules packages, plugins list/inspect, and non-web plugins such as guardrail-bridge.

## What This Changes

External npm plugins installed through OpenClaw should be consistently visible to plugin diagnostics, plugins list/inspect, and runtime provider resolution instead of existing as packages plus stubs while registry/runtime paths report them missing.

## Best Possible Solution

Keep this issue open until OpenClaw has one canonical install/registry record for external npm plugins and either repairs stale npm declaration stubs into that record or reports a clear doctor/install error when the package is only present in an unsupported location.
