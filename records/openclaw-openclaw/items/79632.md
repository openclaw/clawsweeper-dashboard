## Summary

Keep open. Current main still has source-backed gaps for this report: explicit `web_search` owner lookup remains bundled-origin filtered, and the reported stub plus workspace `node_modules` layout is outside the managed npm install/recovery contract; related repair PRs cover slices but are still open.

## What This Changes

The issue asks OpenClaw to make external npm plugins consistently visible to plugin listing, inspection, diagnostics, and runtime provider resolution after install/configuration.

## Best Possible Solution

Land a coordinated fix that preserves one canonical durable install record for npm plugins, repairs or clearly rejects stale npm declaration stubs, and removes the bundled-only explicit web_search owner restriction without weakening sandboxed fetch boundaries.

## Solution Assessment

Is this the best way to solve the issue? No; closing or treating one Brave-specific PR as complete would miss the broader external npm registry/runtime inconsistency. The maintainable path is coordinated owner-boundary and install-record repair with targeted regression coverage.
