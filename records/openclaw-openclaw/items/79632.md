## Summary

Keep open. Current main still has source-backed gaps for globally installed web-search providers and for the reported npm declaration/workspace install layout, while the related Brave owner-resolution work is still open and does not cover the broader registry/runtime consistency problem.

## What This Changes

External npm plugins installed and configured through OpenClaw should be consistently visible to plugin diagnostics, registry lookup, and runtime provider selection.

## Best Possible Solution

Land a coordinated fix that removes the incorrect bundled-only owner lookup for explicit installed web_search providers, preserves sandboxed web_fetch boundaries, and defines one durable recovery or diagnostic path for stale npm declaration stubs and install records.

## Reproduction Assessment

Yes at source level, but not with a live current-main setup in this review. Current main still passes origin: "bundled" into explicit web_search owner lookup while the resolver filters by origin, and the reported stub-only layout is not discoverable by the current plugin discovery paths.

## Solution Assessment

No single currently linked patch is the full solution. The best path is to review the global web provider PR for the Brave runtime slice and make a maintainer policy decision for stale npm declaration/workspace install recovery before treating the broader issue as fixed.
