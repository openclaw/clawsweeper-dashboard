## Summary

Keep open. Current main and v2026.6.10 still treat the managed macOS gateway as a gui-domain LaunchAgent, while system LaunchDaemon discovery is only an advisory deep scan; the reporter's live follow-up adds concrete evidence for the duplicate-manager restart storm.

## What This Changes

The issue asks macOS gateway install/status/doctor paths to detect an existing same-label system LaunchDaemon and avoid installing a duplicate gui-domain LaunchAgent that restart-loops the gateway.

## Best Possible Solution

Add a focused pre-install and repair guard that detects a same-label system LaunchDaemon and refuses with clear recovery guidance, while status surfaces the system owner instead of telling the operator to install a second service.

## Reproduction Assessment

Yes, at source level and with reporter live evidence. Current main still writes/checks the managed macOS service as a user LaunchAgent while the reporter showed a healthy system LaunchDaemon being missed and a duplicate gui-domain install causing the storm.

## Solution Assessment

Yes for the narrow bug. A conflict guard before writing or bootstrapping the LaunchAgent is the smallest maintainable fix; a full `--system` LaunchDaemon mode should remain with the broader companion issues.

## Repair Work Prompt

Fix the macOS gateway duplicate-manager bug reported in https://github.com/openclaw/openclaw/issues/97178. Keep the scope narrow: before `openclaw gateway install`, forced install, or doctor repair writes/bootstraps a gui-domain LaunchAgent, detect an existing same-label system-domain LaunchDaemon or `/Library/LaunchDaemons/ai.openclaw.gateway.plist` that appears to run OpenClaw gateway on the target port, refuse to install a second manager, and print clear recovery/external-owner guidance. Make `gateway status`/doctor diagnostics avoid saying the service is simply not installed when a same-label system daemon is found; surface it as an external/system owner. Do not add a full `--system` LaunchDaemon install mode, sudo/root plist writes, new config/env options, or broad lifecycle support; leave that to https://github.com/openclaw/openclaw/issues/51860 and https://github.com/openclaw/openclaw/issues/14251.
