- **[P1] Preserve fallback content when markdown Flex is too large:** `extensions/line/src/auto-reply-delivery.ts:138`
  - body: The new guard skips an oversized markdown-generated Flex message with `continue`, but the table/code block has already been removed from `processed.text`. A large extracted table or code card can therefore disappear instead of falling back to text or a bounded summary.
  - confidence: 0.91
- **[P1] Keep pipe stripping out of shared markdown cleanup:** `src/shared/text/strip-markdown.ts:19`
  - body: `stripMarkdown` is shared by TTS and channel fallback text. Removing every literal `|` here corrupts ordinary non-LINE text such as `A | B`, so table-remnant cleanup needs to stay in a LINE-owned fallback path.
  - confidence: 0.94
- **[P2] Use LINE bubble limits in Flex guards:** `extensions/line/src/auto-reply-delivery.ts:130`
  - body: LINE's current Flex spec caps a bubble JSON at 30 KB, but this guard allows contents up to `32768 - 200` bytes and the directive guard uses `32768`. Generated table/code bubbles above 30 KB can still be sent to LINE and rejected instead of using the safe fallback.
  - confidence: 0.84
