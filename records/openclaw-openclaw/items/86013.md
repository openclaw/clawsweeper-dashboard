Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Keep pipe cleanup out of shared Markdown stripping:** `src/shared/text/strip-markdown.ts:19`
  - body: This helper is shared by SMS, TTS, and fallback text, and LINE calls it before the later LINE-specific cleanup. Adding `result.replace(/\|+/g, "")` means ordinary text like `cat file | grep foo` loses the pipe in LINE and non-LINE callers; keep this cleanup in a LINE-local missed-table fallback instead.
  - confidence: 0.96
- **[P1] Use LINE's documented 30 KB Flex bubble cap:** `extensions/line/src/reply-payload-transform.ts:312`
  - body: LINE documents the maximum JSON data defining a Flex bubble as 30 KB, but this guard allows contents up to 32768 bytes. Flex bubbles between 30 KB and this threshold will still pass validation and can be rejected, so the payload-safety fix needs the documented cap applied consistently. ([developers.line.biz](https://developers.line.biz/en/reference/messaging-api/index.html.md))
  - confidence: 0.92
- **[P2] Preserve oversized markdown Flex content:** `extensions/line/src/auto-reply-delivery.ts:138`
  - body: When a markdown table or code block is converted to Flex, the source text has already been removed. This `continue` skips an oversized Flex with only a warning, so the user still loses the table/code content; emit a text fallback or preserve the original content instead of silently dropping it.
  - confidence: 0.9
