## What This Changes

The PR adds `agents.defaults.model.fallbacksFromModels` plus schema/docs/tests so default model fallbacks can be derived from `agents.defaults.models` when explicit fallbacks are omitted.

## Best Possible Solution

Rebase the opt-in onto current main, preserve timeout and normalization contracts, wire all failover gates to derived fallbacks, and add focused tests plus redacted real runtime proof.

Yes. By source inspection, current main only reads explicit `agents.defaults.model.fallbacks` in the fallback resolver and has no `fallbacksFromModels` config key; I did not run tests because this was a read-only review.

## Solution Assessment

No. The opt-in direction is plausible, but this branch needs rebase fixes for `timeoutMs`, model-ref normalization, and the shared fallback-configured gate before it is the narrow maintainable solution.
