## Summary

Keep open. Current main still lacks catalog-derived fallback behavior, but this PR is not merge-ready because it has correctness blockers against current main and is missing contributor-provided real behavior proof.

## What This Changes

The PR adds an opt-in `agents.defaults.model.fallbacksFromModels` config/schema/docs path that derives default model fallback candidates from `agents.defaults.models` when explicit fallbacks are omitted.

## Best Possible Solution

Rebase and land the opt-in only after preserving current config contracts, syncing every failover gate with catalog-derived fallback state, and adding focused tests plus redacted runtime proof.

## Reproduction Assessment

Yes, by source inspection: current main only reads explicit `agents.defaults.model.fallbacks` and has no `fallbacksFromModels` key. I did not run tests because this was a read-only review.

## Solution Assessment

No. The opt-in direction is plausible, but this patch needs rebase fixes for `timeoutMs`, model-ref normalization, and the shared fallback-configured gate before it is a maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve timeoutMs in the shared model schema:** `src/config/zod-schema.agent-model.ts:3-7`
  - body: `AgentModelObjectSchema` is reused by default, image, PDF, and media model schemas, but this extraction keeps only `primary` and `fallbacks`. Current main accepts `timeoutMs` on the shared model shape, so carrying this diff forward would reject existing timeout-bearing configs.
  - confidence: 0.9

Retry-limit, prompt, and assistant paths can still surface the primary error instead of escalating to the derived candidate.
  - confidence: 0.92

  - confidence: 0.82
