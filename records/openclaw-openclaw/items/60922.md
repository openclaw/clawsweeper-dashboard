## Summary

Keep open. Current main does not implement `fallbacksFromModels`, and this PR is useful but still not mergeable: it addresses the earlier `applyPrimaryModel` data-loss review, but it drops existing `timeoutMs` support from the shared model schema and leaves embedded-runner failover gated on explicit fallback arrays.

## What This Changes

The PR adds an opt-in `agents.defaults.model.fallbacksFromModels` config flag with schema/docs/tests and catalog-derived fallback candidates, plus preservation of model config fields during primary model updates.

## Best Possible Solution

Keep the feature direction, but update the branch so catalog-derived fallbacks participate in the shared fallback-configured decision path, preserve the existing shared model `timeoutMs` contract, and add the required changelog entry.

Yes. Configure `agents.defaults.model.fallbacksFromModels: true` with multiple `agents.defaults.models` entries and no explicit `fallbacks`, then trigger a retry-limit, prompt, or assistant failover path that consults `fallbackConfigured`.
