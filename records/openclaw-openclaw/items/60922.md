## Summary

Keep this PR open: the feature is plausible, but the current branch is conflicting with main, still has blocking correctness regressions around schema compatibility, fallback gating, and model-ref normalization, and the external contributor proof gate is not met.

## What This Changes

The PR adds an opt-in `agents.defaults.model.fallbacksFromModels` config flag with schema/docs/generated updates, fallback-candidate derivation, and unit coverage for deriving fallbacks from `agents.defaults.models`.

## Best Possible Solution

Keep the explicit opt-in design only if it is wired through the existing fallback resolver and shared failover gate, preserves current schema/config-write contracts, and lands with focused tests plus redacted real runtime proof.

## Reproduction Assessment

Yes, for the blocking PR defects by source inspection: the PR head derives catalog candidates, but the shared fallback gate remains explicit-array-only and the extracted model schema omits current-main `timeoutMs`. I did not run tests because this was a read-only review.

## Solution Assessment

No. The feature direction is plausible, but this implementation needs a rebase that preserves `timeoutMs`, keeps model-ref normalization, and makes catalog-derived candidates count as configured fallbacks wherever runtime failover decisions are gated.

- **[P1] Preserve timeoutMs in the shared model schema:** `src/config/zod-schema.agent-model.ts:3-7`
  - body: `AgentModelObjectSchema` is reused by the default, image, PDF, and media model schemas, but this extracted object keeps only `primary` and `fallbacks`. Current main accepts `timeoutMs` on this shared shape, so carrying this diff forward would reject existing timeout-bearing configs.

  - confidence: 0.92

  - confidence: 0.82
