## Summary

Keep open. Current main still has no `fallbacksFromModels` support, and the PR branch is not mergeable as-is because it drops existing shared model `timeoutMs` schema/type support and leaves embedded-runner failover gating tied to explicit fallback arrays only.

## What This Changes

The PR adds an opt-in `agents.defaults.model.fallbacksFromModels` config flag, schema/types/docs/tests, catalog-derived fallback candidate construction, and preservation of default model fallback settings during primary model updates.

## Best Possible Solution

Keep the feature direction, but update the branch so `fallbacksFromModels` is part of the shared fallback-configured decision path and the extracted model schema/type preserves existing `timeoutMs` behavior, then rerun targeted agent/config tests and the changed gate.

## Reproduction Assessment

Yes. The code path is reproducible by enabling catalog-derived fallbacks with no explicit `fallbacks`, then hitting an embedded-runner prompt, assistant, or retry-limit failover path that consults `fallbackConfigured`.

## Solution Assessment
