## Summary

Keep open. Current main still has the shared transport sanitizer dereferencing `text.replace` with no runtime guard, and embedded end diagnostics still only log message/raw-error previews; the shipped Bedrock empty-assistant fix addresses an important side effect but not this remaining failure class.

## What This Changes

The issue asks OpenClaw to stop embedded agent `.replace()` crashes from malformed transport text and add stack detail to `embedded run agent end` diagnostics.

## Best Possible Solution

Add a shared transport text-coercion guard plus redacted, bounded stack/cause capture in embedded lifecycle diagnostics, with regression coverage for malformed assistant/user/tool text blocks.

## Reproduction Assessment

Yes. A focused source-level reproduction can call `sanitizeTransportPayloadText(undefined as unknown as string)` or feed a malformed text block into the Anthropic/OpenAI conversion path; the full live Bedrock failover chain is not required to prove the missing guard.

## Solution Assessment

No, current main is not complete. #71627 is the right adjacent fix for empty assistant replay poisoning, but the narrow maintainable solution still needs shared null-safe text serialization and diagnostic stack observation.
