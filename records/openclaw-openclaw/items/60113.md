Keep open. Current main has adjacent Bedrock replay repair, but it still does not fully satisfy the report: the shared transport sanitizer can still dereference malformed runtime text, and the embedded run-end warning still logs derived error metadata rather than a bounded stack or cause.

The issue asks OpenClaw to stop undefined `.replace()` crashes during embedded agent error/failover runs and add stack-aware diagnostics to `embedded run agent end` logs.

## Best Possible Solution

Land a focused agent transport hardening patch that coerces or rejects malformed text before shared transport serialization, preserves valid string behavior and existing Bedrock sentinel semantics, and carries a redacted, bounded stack/cause observation into embedded run-end diagnostics.
