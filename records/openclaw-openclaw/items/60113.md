## Summary

Keep open. Current main and the latest release still leave the shared transport text sanitizer dereferencing `.replace` after replay/session-repair paths can preserve non-string text blocks, and `embedded_run_agent_end` still lacks bounded stack/cause diagnostics.

## What This Changes

The issue asks OpenClaw to stop embedded agent `.replace` TypeErrors during failover/tool-error replay and add stack-aware embedded run diagnostics.

## Best Possible Solution

Coerce or drop malformed replay text at the shared transport/replay boundary and add bounded redacted stack/cause diagnostics with regression coverage across Anthropic/Bedrock and OpenAI transport paths.

## Reproduction Assessment

Yes, at source level: a persisted text block with `text: undefined` can survive replay normalization and then reach `sanitizeTransportPayloadText(item.text)`, which dereferences `.replace`. I did not live-run the full Bedrock timeout chain.

## Solution Assessment

Yes. The narrow maintainable fix is shared transport/replay coercion plus bounded redacted diagnostics; patching individual provider call sites would leave the same sanitizer reachable elsewhere.
