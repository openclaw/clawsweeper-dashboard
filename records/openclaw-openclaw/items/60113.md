## Summary

Keep open. Current main still has a source-reachable path where malformed replay text can reach a shared transport sanitizer that calls `.replace` directly, and the embedded lifecycle log still records bounded previews but no stack/cause detail for this crash family.

## What This Changes

The issue asks OpenClaw to stop embedded-agent `.replace` crashes from malformed replay/error content and improve diagnostics with stack/cause logging.

## Best Possible Solution

Coerce or drop malformed replay text at the shared replay/transport boundary and add bounded redacted stack/cause diagnostics for embedded agent failures, with regression coverage for Anthropic/Bedrock and OpenAI-compatible transport paths.

## Reproduction Assessment

Yes, at source level: a persisted replay text block with non-string or undefined `text` can survive normalization and reach `sanitizeTransportPayloadText(item.text)`, which calls `.replace`. I did not live-run the full Bedrock timeout chain.

## Solution Assessment

Yes. The narrow maintainable fix is shared replay/transport coercion plus bounded redacted diagnostics; patching only one provider call site would leave the same sanitizer reachable elsewhere.
