## Summary

Keep open. Current main and v2026.5.19 still leave malformed replay text able to reach the shared transport sanitizer’s direct `.replace` call, while embedded-run error logging has redacted raw-error previews but still no bounded stack/cause diagnostic for this crash family.

## What This Changes

The issue requests null-safe embedded-agent transport/error handling and better stack diagnostics for intermittent `.replace` crashes that can poison session transcripts.

## Best Possible Solution

Coerce or drop malformed replay text at the shared replay/transport boundary and add bounded redacted stack/cause diagnostics for embedded agent failures, with regression coverage for Anthropic/Bedrock and OpenAI-compatible transport paths.

## Reproduction Assessment

Yes, at source level: a persisted text block with non-string or undefined `text` can survive replay normalization and then reach `sanitizeTransportPayloadText(item.text)`, which calls `.replace`. I did not live-run the full Bedrock timeout chain.

## Solution Assessment

Yes. The narrow maintainable fix is shared replay/transport coercion plus bounded redacted diagnostics; patching one provider call site would leave the same sanitizer reachable elsewhere.
