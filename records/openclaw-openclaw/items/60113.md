## Summary

Keep open. Current main still has the shared transport sanitizer calling `.replace()` directly on its input, and the embedded run-end diagnostic path still carries redacted message/raw-error previews but no bounded stack/cause observation; the merged Bedrock replay fix addressed the empty-assistant death loop, not this remaining failure class.

## What This Changes

The issue asks OpenClaw to stop embedded agent `.replace()` crashes on malformed transport/error text and improve run-end diagnostics beyond `error.message`.

## Best Possible Solution

Land a focused runtime hardening patch that coerces malformed transport text at the shared sanitizer boundary and adds bounded, redacted stack/cause observation to embedded lifecycle diagnostics without replaying raw diagnostic text to providers.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction remains on current main: call `sanitizeTransportPayloadText(undefined as unknown as string)` or pass a malformed text block with `text: undefined` into the Anthropic/OpenAI transport conversion path.

## Solution Assessment

Yes, the narrow maintainable fix is shared transport text coercion plus bounded diagnostic observation. A one-off Bedrock-only guard or replay-only repair would miss other provider paths that use the same sanitizer.
