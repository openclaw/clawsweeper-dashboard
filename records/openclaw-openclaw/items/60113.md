## Summary

Keep open. Current main still has a shared transport sanitizer that directly calls `.replace` on its argument and embedded lifecycle diagnostics still do not preserve a bounded stack/cause field; the merged Bedrock replay work fixed the empty-assistant death loop side effect, but not this remaining runtime-hardening gap.

## What This Changes

The issue reports embedded-agent `.replace` crashes during failover/tool-error handling and asks for null-safe text/error handling plus better embedded-run diagnostics.

## Best Possible Solution

Add a shared transport text-coercion guard and redacted bounded stack/cause capture for embedded lifecycle errors, with regression coverage around malformed replay text blocks and the existing Bedrock sentinel behavior.
