Keep open. Current main still has the shared transport sanitizer dereferencing `text.replace` without a runtime guard, and embedded lifecycle diagnostics still do not capture bounded stack/cause fields, so the remaining crash/observability request is not implemented by the Bedrock replay fixes.

## What This Changes

Prevent undefined `.replace` crashes during embedded agent error/replay serialization and improve embedded run-end diagnostics with stack-aware logging.

## Best Possible Solution

Add a shared transport text-coercion guard plus redacted, bounded stack/cause capture in embedded lifecycle diagnostics, with focused regression coverage for malformed assistant/user/tool text blocks.
