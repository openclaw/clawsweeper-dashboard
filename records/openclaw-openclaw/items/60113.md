## Summary

Keep open. Current main and the latest shipped release still have a source-reachable malformed replay path into the shared transport sanitizer, and the embedded lifecycle log still lacks stack/cause detail for this crash family.

## What This Changes

The issue requests null-safe embedded-agent replay/error handling for `.replace` crashes and bounded stack diagnostics in `embedded_run_agent_end` logs.

## Best Possible Solution

Coerce or drop malformed replay text at the shared replay/transport boundary, then add bounded redacted stack/cause diagnostics for embedded agent failures with regression coverage across Anthropic/Bedrock and OpenAI-compatible transports.

Yes. The narrow maintainable fix is shared replay/transport coercion plus bounded redacted diagnostics; patching only one provider call site would leave the same sanitizer reachable elsewhere.
