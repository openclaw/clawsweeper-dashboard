## Summary

Keep open. Current main still has the shared transport sanitizer dereferencing `text.replace` without a runtime guard, and embedded lifecycle diagnostics still log bounded message/raw-error previews rather than stack/cause details; the merged Bedrock replay fixes address an important side effect but not the remaining malformed-text serialization crash class.

## What This Changes

The issue asks OpenClaw to stop embedded agent `.replace` TypeErrors during malformed/error replay paths and improve embedded run-end diagnostics with stack information.

## Best Possible Solution

Add a shared transport text-coercion guard plus redacted, bounded stack/cause capture in embedded lifecycle diagnostics, with focused regression coverage for malformed assistant/user/tool text blocks and no change to the existing Bedrock sentinel replay policy.
