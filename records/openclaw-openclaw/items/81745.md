## Summary

Keep open. The earlier validation is a useful direction for the invalid Anthropic runtime report, but this PR currently rejects an already accepted Codex runtime alias and lacks required real behavior proof, so it should not merge as-is.

## What This Changes

The PR adds plugin-aware validation for configured provider/model-scoped `agentRuntime.id` values and regression tests for invalid provider or unknown runtime ids.

## Best Possible Solution

Keep the early config validation boundary, but normalize runtime ids with the same helper used by harness selection and add alias coverage before merge.

## Reproduction Assessment

Yes. From source inspection, current schema accepts arbitrary `agentRuntime.id` strings while harness selection later fails closed for unknown explicit runtimes; the PR regression is also source-reproducible because `codex-app-server` normalizes to `codex` at runtime but is absent from the PR's raw known-runtime set.

## Solution Assessment

No as submitted. The validation boundary is maintainable, but it must share runtime-id normalization with selection so diagnostics do not reject configs that current main accepts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Normalize runtime ids before validating them:** `src/config/validation.ts:641-642`

Run the same runtime-id normalizer before comparing, or include supported aliases.
