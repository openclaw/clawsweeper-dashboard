The PR adds plugin-aware validation for configured provider/model `agentRuntime.id` values and regression tests for provider ids such as `anthropic` and unknown runtime ids.

## Best Possible Solution

Keep the early plugin-aware validation, but normalize configured runtime ids through the same runtime-selection helper, add alias coverage such as `codex-app-server`, and include real CLI validation proof.
