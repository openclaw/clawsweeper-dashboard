## Summary

Keep open: current main and the latest release still only suppress startup stdout for JSON output mode, while this PR moves `openclaw mcp serve` into the existing startup suppression path before config/doctor preflight can write to stdout. The patch is focused and I found no blocking code defect; exact-head CI/CodeQL was still queued at review time.

## What This Changes

The PR adds an `mcp serve` startup-policy condition that suppresses doctor/config stdout and extends the startup-policy regression test.

## Best Possible Solution

Merge the startup-policy fix and regression test after exact-head checks pass, so `openclaw mcp serve` keeps stdout JSON-RPC-only before config/doctor preflight.

## Reproduction Assessment

Yes, source-reproducible: current main and `v2026.6.8` compute `suppressDoctorStdout` only from `jsonOutputMode`, so non-JSON `openclaw mcp serve` can run pre-action config/doctor output without stdout suppression before entering the stdio server. I did not execute the smoke because this review was read-only.

## Solution Assessment

Yes. The startup policy is the narrowest maintainable layer because it already feeds both early stderr routing and config preflight note suppression; the older server-level routeLogsToStderr approach runs too late for pre-action startup output.
