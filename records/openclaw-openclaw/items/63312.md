## Summary

Keep open. The PR is a plausible narrow code change, but current main now documents and implements an explicit `skills.load.allowSymlinkTargets` trust model, while this branch would add an automatic personal/project `.agents/skills` exception that needs maintainer approval, docs alignment, and real behavior proof before merge.

## What This Changes

The PR adds a logical-path containment fallback for personal and project `.agents/skills` symlinked directories, plus tests that accept those symlinks while still rejecting managed-dir external symlinks.

## Best Possible Solution

Keep the explicit configured-target contract unless maintainers intentionally approve automatic personal/project `.agents` symlink escapes; any approved version should update security docs and keep focused regression coverage.

## Reproduction Assessment

Yes, source-reproducible. Current main follows symlinked child directories, then rejects realpaths outside the source root unless the target is listed in `skills.load.allowSymlinkTargets`; I did not run tests because this was a read-only review.

## Solution Assessment

Unclear. The patch is mechanically narrow, but current main already has a documented explicit-target solution, so the automatic `.agents` fallback is not clearly the best maintainable fix without maintainer policy approval.
