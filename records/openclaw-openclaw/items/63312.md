## Summary

Keep open. The PR targets a real current-main behavior, but it intentionally relaxes a skill-loading containment boundary for personal and project `.agents/skills` roots and still needs maintainer policy approval plus docs/changelog follow-through before merge.

## What This Changes

The PR adds a logical-path containment fallback for personal/project `.agents/skills` sources and regression tests for symlinked personal/project skills plus managed-source rejection.

## Best Possible Solution

Land this PR or an equivalent narrow patch only if maintainers want personal and project `.agents/skills` symlinked directories to be supported even when their real targets live outside configured roots; keep strict physical containment for managed, bundled, workspace, extra, and plugin-owned roots, and document the exception.

## Reproduction Assessment

Yes. Current main discovers symlinked child directories, then rejects their skill directory or `SKILL.md` when `resolveContainedSkillPath` sees a realpath outside the configured root; a symlink from personal/project `.agents/skills` to an external temp skill reproduces the skip path from source inspection.

## Solution Assessment

Unclear. The implementation is narrow for the requested `.agents/skills` workflow, but it changes a trust boundary; the safer alternative is #45912's configured-root-only symlink model unless maintainers explicitly accept external targets for user-managed `.agents` roots.
