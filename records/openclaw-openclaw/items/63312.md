## Summary

Keep open. Current main still rejects symlinked personal/project `.agents/skills` directories that resolve outside their configured root, so this PR is not obsolete; however, the patch changes a skill-loading containment boundary and still needs maintainer approval plus documentation and changelog follow-through before merge.

## What This Changes

The PR adds a logical-path containment fallback for personal and project `.agents/skills` sources, with regression tests for personal/project symlink loading and managed-source rejection.

## Best Possible Solution

Land this PR or an equivalent narrow patch only if maintainers explicitly want personal and project `.agents/skills` symlinked directories to be supported even when their real targets live outside the configured root. The landed version should preserve strict realpath containment for managed, bundled, workspace, extra, and plugin-owned roots; document the exception in public Skills security docs; keep focused regression coverage; and add the required changelog entry.

## Reproduction Assessment

Yes. Current main discovers symlinked child directories via `listChildDirectories`, then rejects their `SKILL.md` files when `resolveContainedSkillPath` sees a realpath outside the configured root.

## Solution Assessment

Unclear. The implementation is narrow for the requested `.agents/skills` workflow, but it relaxes a containment boundary and is broader than the configured-root-only approach in #45912, so it needs maintainer policy approval before it is the best maintainable fix.
