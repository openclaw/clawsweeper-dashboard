## Summary

Keep open: the PR is focused, but it changes a documented skill-loading trust boundary. Current main already supports intentional symlinked skill roots through explicit `skills.load.allowSymlinkTargets`, while this branch accepts unconfigured personal/project `.agents/skills` symlinks and still needs maintainer policy approval, docs alignment, and real behavior proof.

## What This Changes

The PR adds a logical-path containment fallback for personal and project `.agents/skills` symlinked directories and tests personal/project loading plus managed-source rejection.

## Best Possible Solution

Land this PR or an equivalent narrow patch only if maintainers want unconfigured personal/project `.agents/skills` symlinked directories to be supported; keep strict physical containment for other roots and update public Skills docs and regression coverage to match.

## Reproduction Assessment

Yes, source-reproducible. Current main follows symlinked child directories, then rejects realpaths outside the skill root unless the target is listed in `skills.load.allowSymlinkTargets`; I did not run live tests in this read-only review.

## Solution Assessment

Unclear. The patch is mechanically narrow, but it changes a documented trust model instead of using the existing explicit-target contract, so maintainer approval plus docs and real behavior proof are needed before merge.
