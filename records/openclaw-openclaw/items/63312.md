## Summary

Keep this PR open. Current main still rejects the symlinked `.agents/skills` workflow the PR targets, but the patch intentionally relaxes a skill-loading containment boundary and needs maintainer approval plus two small follow-ups before it is merge-ready.

## What This Changes

The PR adds a logical-path containment fallback for personal and project `.agents/skills` sources, while keeping realpath containment for other skill sources and adding three symlink regression tests.

## Best Possible Solution

Land this PR or an equivalent narrow patch only if maintainers want personal and project `.agents/skills` symlinked directories to be supported. Keep strict physical containment for managed, bundled, workspace, extra, and plugin-owned skill roots; gate or make the symlink tests cross-platform; and update the public Skills security docs at `https://docs.openclaw.ai/tools/skills` to describe the exception.

## Reproduction Assessment

Yes. Current main discovers symlinked child directories, then rejects their `SKILL.md` path when the real target is outside the root; the source trace through `listChildDirectories`, `loadSkills`, and `resolveContainedSkillPath` gives a high-confidence reproduction path without running a mutating test.

## Solution Assessment

Unclear until maintainer approval. The code change is narrow and scoped to personal/project `.agents` roots, but because it intentionally relaxes containment, the maintainable solution needs explicit docs and cross-platform test handling before merge.
