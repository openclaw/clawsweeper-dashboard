## Summary

Keep open. Current main and the latest v2026.5.3 release still leave Discord REST and gateway socket startup without a Discord-specific DNS/IPv4 fallback, and #77449 is an open unmerged PR with closing syntax for this issue.

## What This Changes

The issue reports that Discord inbound gateway startup fails on IPv4-only DNS networks after the 2026.5.x networking changes and asks for Discord networking to prefer or fall back to IPv4.

## Best Possible Solution

Review and land #77449 or an equivalent Discord plugin-owned REST and WebSocket IPv4 preference/fallback fix, without setting a process-wide launcher DNS option.

## Reproduction Assessment

Yes. Source inspection plus the report’s two-machine Windows workaround gives a high-confidence path: run the Discord channel on an affected IPv4-only DNS network, observe no READY, then verify `NODE_OPTIONS=--dns-result-order=ipv4first` restores READY.

## Solution Assessment

Yes for the linked PR direction: the maintainable fix belongs in Discord’s REST and gateway transport wiring. The process-wide `NODE_OPTIONS` workaround is useful proof, but it is broader than the repository fix should be.
