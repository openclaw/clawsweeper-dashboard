## Summary

Keep open. Current main still lacks Discord-owned IPv4 preference or fallback on the default REST and gateway WebSocket paths, and an open linked PR explicitly targets this regression with closing syntax, so this issue should remain open until that PR is reviewed and merged or replaced.

## What This Changes

The issue reports that Discord inbound fails on IPv4-only networks after the 2026.5 transport change unless the gateway process is forced to prefer IPv4.

## Best Possible Solution

Land a Discord plugin-owned IPv4-first or fallback transport fix for REST and gateway WebSocket startup with focused regression coverage and a changelog entry, then close this issue through the merged PR.

## Reproduction Assessment

Yes. The issue gives two-machine Windows reproduction plus an ipv4first workaround, and current main source inspection shows the default Discord REST and gateway WebSocket paths lack a Discord-owned DNS/family fallback.

## Solution Assessment

Yes in direction. A Discord-plugin-owned REST and gateway transport fix is narrower and more maintainable than setting process-wide NODE_OPTIONS, but the open linked PR should be reviewed and completed before this issue closes.
