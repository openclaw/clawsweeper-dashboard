## Summary

Keep open. Current main still lacks a Discord-owned IPv4/DNS fallback in the normal Discord REST and gateway WebSocket paths, and open PR #77449 explicitly targets this issue but has not merged.

## What This Changes

The issue requests a fix for Discord inbound failures on IPv4-only Windows networks after 2026.5.x by adding a Discord-owned IPv4/DNS fallback for REST and gateway WebSocket startup.

## Best Possible Solution

Review and land #77449, or an equivalent Discord plugin-owned REST and gateway transport fix, then close this issue after the fix is on main.

## Reproduction Assessment

Yes, source-level. The issue reports two affected Windows machines and a working ipv4first workaround, and current main still has no Discord-owned lookup/family fallback on the default REST or gateway WebSocket paths.

## Solution Assessment

Yes for the owner-local direction represented by #77449; no for setting process-wide NODE_OPTIONS as the repository fix. The maintainable fix should live in the Discord plugin transport layer with REST and gateway regression coverage.
