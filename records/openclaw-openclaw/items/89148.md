## Summary

Keep open: the shipped crash is real and the refreshed terminal proof is sufficient, but the branch still changes a plugin-SDK-visible dispatcher method from required to optional without maintainer API approval.

## What This Changes

The PR guards missing `getFailedCounts` reads in auto-reply dispatcher accounting and Feishu fallback paths, makes `ReplyDispatcher.getFailedCounts` optional, and adds focused regression coverage.

## Best Possible Solution

Guard all runtime failed-count reads for the shipped crash while preserving the public dispatcher type, unless maintainers explicitly decide that missing failed-count hooks are part of the supported plugin API.

## Reproduction Assessment

Yes. Current main and `v2026.5.28` call `dispatcher.getFailedCounts().final` unconditionally in the final-outcome helper, so a dispatcher without that method will throw.

## Solution Assessment

No as-is. The runtime guards are the right direction, but changing the exported dispatcher type is not the narrowest compatibility-safe fix without explicit maintainer API approval.
