## Summary

Keep open. Current main still batch-resolves stored auth-profile SecretRefs in the strict startup secret path, and the open closing PR has not merged.

## What This Changes

Gateway should stay online when a non-default stored model auth profile contains a stale SecretRef provider, degrading only that profile instead of aborting startup.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/77213 or an equivalent targeted patch that splits config SecretRef resolution from stored auth-profile degradation with regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows a stored `api_key.keyRef` or `token.tokenRef` pointing to an absent provider is collected into the startup-wide strict resolver and can be wrapped as a startup failure.

## Solution Assessment

Yes. The narrow maintainable fix is to keep config and reload SecretRefs strict while degrading only stale stored auth-profile refs during startup preflight.
