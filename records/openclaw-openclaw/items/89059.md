## Summary

Keep open: current main and v2026.6.8 still reject gateway-mode TUI active-run sends before Gateway queue handling, and the linked implementation PRs are open rather than merged fixes.

## What This Changes

The issue asks for TUI active-run sends to respect `/queue` mode instead of being rejected locally before Gateway queue handling.

## Best Possible Solution

Land or replace a single queue-mode-aware TUI/Gateway fix that forwards allowed active-run sends to Gateway policy while preserving stop/Esc and duplicate-submit safeguards.

## Reproduction Assessment

Yes. Source inspection shows current main rejects gateway active-run messages in both the typed-submit helper and `sendMessage` before `client.sendChat` reaches Gateway queue policy.

## Solution Assessment

Yes for the requested direction, but no merged fix exists yet. The maintainable solution is to propagate effective queue mode to the TUI and locally block only steer-mode or duplicate in-flight sends.
