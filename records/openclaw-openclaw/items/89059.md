## What This Changes

The issue reports that TUI active-run sends are rejected locally with an "agent is busy" message instead of being forwarded to Gateway queue-mode handling.

## Best Possible Solution

Review and land one queue-mode-aware TUI/Gateway fix with real TUI-to-Gateway proof, then close this issue and consolidate the newer duplicate at https://github.com/openclaw/openclaw/issues/90012 if needed.

## Reproduction Assessment

Yes from source inspection: typed submits and command sends both still block gateway-mode active-run messages before `client.sendChat` can reach Gateway queue policy. I did not run a live TUI/Gateway session in this read-only review.

## Solution Assessment

Yes for the requested direction, but no merged implementation exists yet. The maintainable fix should pass the effective Gateway queue mode into TUI state and keep client-side blocking only for steer-mode or in-flight optimistic sends.
