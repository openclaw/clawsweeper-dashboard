## Summary

Keep open: current main and the latest release still block gateway-mode TUI active-run sends before Gateway queue handling, and the linked fixing PRs are open/unmerged.

## What This Changes

The issue asks for TUI sends during active runs to respect `/queue` modes by reaching Gateway queue policy instead of being rejected locally with `agent is busy`.

## Best Possible Solution

Land or replace the linked queue-mode-aware TUI/Gateway fix, preserve stop/Esc and duplicate-submit safeguards, then close this issue and consolidate the duplicate report.

## Reproduction Assessment

Yes, source-level: current main rejects gateway-mode active-run sends in both the typed-submit helper and `sendMessage` before `client.sendChat` can reach Gateway queue policy. I did not run a live TUI/Gateway reproduction.

## Solution Assessment

Yes for the requested direction, but no merged fix exists yet. The maintainable solution is to propagate effective queue mode to the TUI and locally block only steer-mode or duplicate in-flight sends.
