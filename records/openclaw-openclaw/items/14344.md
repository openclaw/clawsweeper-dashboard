## Summary

Keep open. Current main still does not expose WhatsApp delete/revoke through the message tool, and the linked implementation PR is open with closing metadata, so this issue remains the canonical tracker for a security-sensitive WhatsApp message-mutation feature.

## What This Changes

Request to add a WhatsApp `message delete` or revoke action that lets agents retract their own sent WhatsApp messages.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/70813, or an equivalent WhatsApp plugin-owned implementation, with agreed action naming, own-message authorization, LID-aware routing, group key handling, stale revoke errors, focused tests, docs, changelog, and redacted live WhatsApp proof.

## Reproduction Assessment

Yes. Source inspection on current main shows WhatsApp delete is not advertised, is declined by the plugin action adapter, and would hit the unsupported-action path; I did not run a live WhatsApp account replay.

## Solution Assessment

Yes for the owner boundary, unclear on final semantics. A WhatsApp plugin-owned implementation through the existing message action, gateway, and listener seams is the narrow maintainable path, but naming and mutation-safety behavior need maintainer approval.
