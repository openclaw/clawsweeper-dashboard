## Summary

Keep open. Current main still lacks WhatsApp delete/revoke support, and the open implementation PR references this issue with closing syntax, so this should remain open pending maintainer review.

## What This Changes

Add a WhatsApp message delete/revoke action so agents can retract their own sent messages.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/70813 or an equivalent WhatsApp-owned implementation that advertises the agreed action, routes through the listener/runtime, builds only bot-authored delete keys, handles group and age-limit behavior, and updates tests, docs, and changelog.

## Reproduction Assessment

Yes for the unsupported current behavior by source inspection: WhatsApp delete is not advertised, the adapter supports only react, and the runtime rejects non-react actions. No live WhatsApp account replay was run.

## Solution Assessment

Yes, keeping this issue open for the linked WhatsApp plugin implementation is the right path. Closing now would race an open closing PR, and a separate repair lane would duplicate active maintainer review of naming and safety constraints.
