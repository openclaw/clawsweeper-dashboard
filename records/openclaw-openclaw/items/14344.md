## Summary

Keep open. Current main still lacks WhatsApp delete/revoke support, and the linked implementation PR with closing syntax is open and unmerged, so this issue should remain the canonical request until that PR or an equivalent maintainer-reviewed change lands.

## What This Changes

Request adds a WhatsApp `message delete`/revoke action so agents can retract their own sent WhatsApp messages.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/70813, or an equivalent WhatsApp plugin-owned implementation with explicit naming, own-message authorization, group handling, stale-message errors, docs, changelog, and focused tests.

Yes. Source inspection on current main confirms WhatsApp message actions advertise only react/poll, route only react, and throw for delete/revoke; I did not run a live WhatsApp account replay.

## Solution Assessment

Yes. A WhatsApp plugin-owned implementation through the existing message/gateway seams is the narrowest maintainable path, and the open linked PR is already the focused candidate pending maintainer review.
