## Summary

Keep open. Current main and the latest release still do not expose WhatsApp delete/revoke through the message tool, and the linked implementation PR remains open and unmerged, so this issue should stay as the canonical tracker pending maintainer review.

## What This Changes

The issue requests a WhatsApp message delete/revoke action so agents can retract their own sent WhatsApp messages through the message tool.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/70813, or an equivalent WhatsApp plugin-owned implementation, with explicit naming, own-message authorization, group handling, stale-message errors, docs, changelog, and focused tests.

## Reproduction Assessment

Yes. Source inspection on current main shows WhatsApp delete is not advertised, is declined by the plugin action adapter, and would produce the shared unsupported-action error; I did not run a live WhatsApp account replay.

## Solution Assessment

Yes, with maintainer review. A WhatsApp plugin-owned implementation through the existing message action, gateway, and listener seams is the narrow maintainable path, but naming and mutation-safety semantics need explicit approval.
