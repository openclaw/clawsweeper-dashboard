## Summary

Keep open. Current main and the latest release still do not expose WhatsApp delete/revoke through the message action surface, and the linked implementation PR with closing syntax is still open and unmerged.

## What This Changes

Add a WhatsApp message delete/revoke action so agents can retract their own sent WhatsApp messages.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/70813, or an equivalent WhatsApp plugin-owned implementation with explicit naming, own-message authorization, group handling, stale-message errors, docs, changelog, and focused tests.

## Reproduction Assessment

Yes. Source inspection on current main shows WhatsApp advertises only react/poll, accepts only react through the adapter, and throws for non-react actions; I did not run a live WhatsApp account replay.

## Solution Assessment

Yes. A WhatsApp plugin-owned implementation through the existing message/gateway seams is the narrowest maintainable path, and the open linked PR is already the focused candidate pending maintainer review.
