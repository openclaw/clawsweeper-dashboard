## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/70813, or an equivalent WhatsApp plugin-owned implementation, with agreed naming, bot-owned authorization, group key handling, LID-safe routing, stale revoke errors, focused tests, docs, changelog, and redacted live WhatsApp proof.

## Solution Assessment

Yes for the owner boundary, unclear for final semantics. The maintainable path is a WhatsApp plugin-owned action over the existing gateway/listener/Baileys seams, but naming and destructive-mutation safety need maintainer approval.
