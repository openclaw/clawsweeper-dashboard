## Summary

Keep open. Current main and the latest release still do not expose WhatsApp delete/revoke through the message action path, and the linked implementation at https://github.com/openclaw/openclaw/pull/70813 remains open and conflicting.

## What This Changes

Request to add a WhatsApp message delete/revoke action so agents can retract their own sent messages through the message tool.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/70813, or an equivalent WhatsApp plugin-owned implementation, with agreed naming, bot-owned authorization, group key handling, stale revoke errors, focused tests, docs, changelog, and redacted live WhatsApp proof.

## Reproduction Assessment

Yes. Source inspection on current main shows WhatsApp delete/revoke is not advertised, is declined by the plugin action adapter, and would hit the unsupported-action runtime path; I did not run a live WhatsApp replay.

## Solution Assessment

Yes for the owner boundary, unclear on final semantics. The maintainable path is a WhatsApp plugin-owned action over the existing gateway/listener/Baileys seams, but naming and mutation-safety behavior need maintainer approval.
