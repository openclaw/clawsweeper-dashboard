## What This Changes

Request to add a WhatsApp message delete/revoke action so agents can retract their own sent WhatsApp messages by target and message id.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/70813, or an equivalent WhatsApp plugin-owned implementation, with agreed naming, bot-owned authorization, group key handling, stale revoke errors, focused tests, docs, changelog, and redacted live WhatsApp proof.

## Reproduction Assessment

Yes. Source inspection on current main shows WhatsApp delete/revoke is not advertised, is declined by the action adapter, and would hit the unsupported-action runtime path; I did not run a live WhatsApp replay.

## Solution Assessment

Yes for the owner boundary, unclear for final semantics. The maintainable path is a WhatsApp plugin-owned action over the existing gateway/listener/Baileys seams, but naming and destructive-mutation safety need maintainer approval.
