## Summary

Keep open. Current main and the latest stable release still do not expose WhatsApp delete/revoke through the message tool, and the linked implementation at https://github.com/openclaw/openclaw/pull/70813 is open, conflicting, and not merged.

## What This Changes

Add a WhatsApp message delete/revoke action so agents can retract their own previously sent WhatsApp messages.

## Best Possible Solution

Keep this issue open until https://github.com/openclaw/openclaw/pull/70813 or an equivalent WhatsApp-plugin implementation lands with an agreed mutation contract, focused tests/docs, and live WhatsApp proof.

## Reproduction Assessment

Yes at source level: current main advertises WhatsApp react/poll/upload-file only and handleWhatsAppAction throws for non-react actions such as delete or unsend. I did not run a live WhatsApp account because the missing support is clear in dispatch/runtime source.

## Solution Assessment

Unclear as a final product shape: the WhatsApp plugin and Baileys send path are the right implementation boundary, but maintainers still need to settle the action naming and destructive-action authorization contract. A safe implementation should avoid letting existing send-enabled WhatsApp configs silently gain broader mutation authority unless that is explicitly accepted.
