## Summary

Keep this PR open: it fixes a real current-main restart loop, but the current diff regresses the existing repair-on-start path for loaded services with stale, temporary, or missing service definitions.

## What This Changes

The PR adds an `already-running` gateway service start result, CLI output, tests, and a changelog entry so `openclaw gateway start` does not restart an already running managed gateway.

## Best Possible Solution

Keep the idempotent-start behavior for healthy running services, but preserve the existing repair-required branch before returning `already-running`.

## Reproduction Assessment

Yes, by source inspection: current main runs login-shell env probes, `gateway start` reaches `service.restart()` even when runtime state is already running, and systemd restart sends the reported SIGTERM. I did not need a live systemd reproduction to establish the code path.

## Solution Assessment

No. The proposed direction is right, but the guard should run after the existing repair checks, or only return `already-running` when the loaded service definition has no repair issues.
