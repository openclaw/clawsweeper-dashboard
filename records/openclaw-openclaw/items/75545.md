## Summary

Keep this PR open for normal maintainer review. Current main still routes `openclaw gateway start` through the service restart path even when runtime state is already running, and the proposed service-layer idempotency fix matches the linked crash report without adding broad workaround logic; I found no blocking review findings.

## What This Changes

The PR adds an `already-running` gateway service start outcome, skips platform restart when the managed service runtime is already running, updates CLI output and lifecycle tests, and adds a changelog fix entry for #60087.

## Best Possible Solution

Land this service-layer idempotency change after maintainer review and changed-gate validation, keeping `gateway restart` as the explicit forced-cycle command and `gateway start` as an ensure-running operation.

## Reproduction Assessment

Yes. The reproduction path is source-backed: the exec login-shell env probe can source a profile that runs `openclaw gateway start`, and current main then reaches `service.restart` even when service runtime state is already `running`.

## Solution Assessment

Yes. The PR solves the issue at the shared gateway service start contract, which is narrower and more maintainable than adding exec-tool or shell-profile special cases, while preserving the explicit restart command.
