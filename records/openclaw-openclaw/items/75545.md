## Summary

Keep this PR open. Current main still makes `openclaw gateway start` route through the platform restart path for an installed managed service, while this PR is a narrow implementation of the #60087 fix and I did not find a blocking correctness or security issue in the proposed diff.

## What This Changes

Adds an `already-running` gateway service start outcome, skips the service-manager restart when the managed runtime is already running, surfaces that result in the CLI, updates lifecycle tests, and adds a changelog fix entry for #60087.

## Best Possible Solution

Land the narrow service-layer idempotency change with the CLI result branch, regression tests, and changelog entry, while preserving `openclaw gateway restart` as the explicit forced-cycle command.

## Reproduction Assessment

Yes. The #60087 reproduction is high-confidence from current main source: a login-shell env probe can source a profile that runs `openclaw gateway start`, and current main then reaches the platform restart action even when runtime state is already running.

## Solution Assessment

Yes. The PR fixes the command contract at the gateway service layer, which is narrower and more maintainable than special-casing exec shell startup or adding profile-specific workarounds.
