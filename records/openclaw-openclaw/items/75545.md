## What This Changes

The PR adds an `already-running` gateway service start outcome, skips service-manager restart when the resolved service runtime is already running, updates CLI output and focused tests, and adds a changelog fix entry for #60087.

## Best Possible Solution

## Solution Assessment

Yes. Fixing the shared `startGatewayService` path is the narrowest maintainable solution because all CLI/service entrypoints inherit ensure-running semantics without exec-tool or shell-profile special cases.
