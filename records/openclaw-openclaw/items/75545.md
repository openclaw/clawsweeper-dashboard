## Summary

Keep this PR open for normal maintainer review. Current main still has the non-idempotent `gateway start` path, the linked issue has a source-backed reproduction, and the PR is a narrow service-layer fix with no concrete correctness finding from this review.

## What This Changes

The PR adds an `already-running` gateway service start outcome, skips the platform restart when the managed service is already running, updates CLI output and lifecycle tests, and adds a changelog fix entry for #60087.

## Best Possible Solution

Land this idempotency change after maintainer review and changed-gate validation, preserving `gateway restart` as the explicit forced-cycle command while making `gateway start` safe to call from profile-driven ensure-running paths.

## Reproduction Assessment

Yes. The high-confidence reproduction path is source-backed: an exec login-shell env probe can source a profile that runs `openclaw gateway start`, and current main then reaches `service.restart` even when the managed runtime is already `running`.

## Solution Assessment

Yes. Solving this at `startGatewayService` is the narrowest maintainable fix because every CLI/service entrypoint gets the ensure-running semantics without adding exec-tool or shell-profile special cases.
