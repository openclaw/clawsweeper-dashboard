## What This Changes

The PR adds an `already-running` gateway service start outcome, skips service-manager restart when the gateway service state is already running, updates CLI output, adjusts lifecycle tests, and adds a changelog fix entry for #60087.

## Best Possible Solution

Land the idempotent start behavior after maintainer review and changed-gate validation, preserving `openclaw gateway restart` as the explicit forced-cycle command while making profile-driven `gateway start` safe as an ensure-running operation.

## Reproduction Assessment

Yes. The reproduction path is high-confidence and source-backed: a login-shell env probe can source `~/.profile`, invoke `openclaw gateway start`, and current main then reaches the service-manager restart path even when the runtime state is already `running`.

## Solution Assessment

Yes. Fixing this in `startGatewayService` is the narrowest maintainable solution because all CLI/service entrypoints inherit ensure-running semantics without adding exec-tool or shell-profile special cases.
