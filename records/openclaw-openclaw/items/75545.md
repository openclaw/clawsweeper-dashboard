## Summary

Keep this PR open for normal maintainer review. Current main still makes `openclaw gateway start` non-idempotent by calling the service restart path even when the managed gateway is already running, while the proposed branch is a narrow code/test/changelog fix for the linked exec-triggered SIGTERM path.

## What This Changes

The PR adds an `already-running` gateway start outcome, returns early from `startGatewayService` when runtime state is already running, updates CLI output, adjusts lifecycle tests, and adds a changelog fix entry for #60087.

## Best Possible Solution

Land this idempotency change after maintainer review and changed-gate validation, preserving `gateway restart` as the explicit forced-cycle command while making `gateway start` safe for profile-driven ensure-running paths.

## Reproduction Assessment

Yes. The high-confidence reproduction path is source-backed: a login-shell env probe can source a profile that runs `openclaw gateway start`, and current main then reaches `service.restart` even when service runtime is already `running`.

## Solution Assessment

Yes. Solving this inside `startGatewayService` is the narrowest maintainable fix because every CLI/service entrypoint gets ensure-running semantics without adding exec-tool or shell-profile special cases.
