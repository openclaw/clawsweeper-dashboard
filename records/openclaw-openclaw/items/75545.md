## Summary

Keep open: the PR addresses a real current-main gateway restart bug, but the latest diff still returns `already-running` before the existing repair-required path, which would skip stale service definition repair.

## What This Changes

The PR adds an `already-running` gateway service start outcome, CLI handling, tests, and a changelog entry so `openclaw gateway start` no-ops when the managed service is already running.

## Best Possible Solution

Keep the idempotent start behavior, but only return `already-running` after repair issues have been checked or when the loaded service definition is already healthy.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: exec uses a login-shell env probe, current main routes `gateway start` to a platform restart even when the service runtime is running, and systemd restart matches the SIGTERM report.

## Solution Assessment

No. The PR's direction is right, but the guard must preserve current repair-on-start behavior for stale, missing-binary, or temporary-path service definitions.

## Review Findings

Overall confidence: 0.92
