Keep open for maintainer review. Current main still has the reported non-idempotent start path, and the PR targets the shared lifecycle contract with focused coverage rather than being obsolete or duplicative.

## What This Changes

The PR makes `openclaw gateway start` report an already-running outcome instead of restarting an already running managed gateway, and updates CLI output, tests, and the changelog.

## Best Possible Solution

Land the idempotent start behavior after maintainer review, mergeability cleanup, and Testbox changed-gate validation, preserving `openclaw gateway restart` as the explicit forced-cycle command.

## Reproduction Assessment

Yes. Current main has a source-backed path where a login-shell env probe can source `~/.profile`, invoke `openclaw gateway start`, and then reach the service-manager restart path even when the runtime is already running.

## Solution Assessment

Yes. Fixing the shared `startGatewayService` path is the narrowest maintainable solution because all CLI/service entrypoints inherit ensure-running semantics without exec-tool or shell-profile special cases.
