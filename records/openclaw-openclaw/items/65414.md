## Summary

Keep open. Current main and the latest release still route promptless Android ACTION_ASSIST launches to Chat, and the requested Voice-tab plus one-shot mic activation behavior needs Android UX/privacy approval before code automation.

## What This Changes

Route promptless Android ACTION_ASSIST launches to the Voice tab with one-shot mic activation while leaving ASK_OPENCLAW text prompts in Chat.

## Best Possible Solution

Approve the Android assistant UX/privacy policy first, then implement promptless ACTION_ASSIST to select Voice with one-shot mic activation and permission handling while preserving App Actions draft-to-Chat behavior.

## Reproduction Assessment

Yes. Current-main source shows ACTION_ASSIST parsing as a promptless assist request, MainActivity forwarding it, and MainViewModel selecting Chat; I did not run a live Android device or emulator flow.

## Solution Assessment

Unclear until maintainers approve the Android UX/privacy direction. The proposed file boundary is narrow, but automatic mic activation from the assistant gesture must be carefully scoped and must preserve ASK_OPENCLAW draft-only handling.
