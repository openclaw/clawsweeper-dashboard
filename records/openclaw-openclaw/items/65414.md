## Summary

Keep open. Current main and the latest release still route promptless Android ACTION_ASSIST launches to Chat, while routing the assistant gesture to Voice with microphone activation is an Android UX/privacy decision rather than a narrow existing-contract bug fix.

## What This Changes

Route promptless Android ACTION_ASSIST launches to the Voice tab with one-shot microphone activation while keeping ASK_OPENCLAW text prompts in Chat.

## Best Possible Solution

Decide the Android assistant UX/privacy policy, then implement promptless ACTION_ASSIST to Voice with one-shot mic activation, focused Android coverage, docs if behavior becomes user-visible, and real assistant/App Actions proof.

## Reproduction Assessment

Yes, from current-main source: ACTION_ASSIST parses as a promptless assist request, MainActivity forwards it, MainViewModel selects Chat, and VoiceTabScreen lacks assistant-origin mic activation. I did not run a live Android device or emulator flow in this read-only review.

## Solution Assessment

Unclear until maintainers approve the Android UX/privacy direction. The proposed file boundary is narrow, but auto-opening Voice and enabling the mic from the assistant gesture must be scoped carefully and must preserve ASK_OPENCLAW draft-only handling.
