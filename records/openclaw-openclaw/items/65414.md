## Summary

Keep open. Current main and v2026.5.18 still route promptless Android ACTION_ASSIST launches to Chat, and changing that to Voice with one-shot microphone activation needs Android UX/privacy approval before it is a safe cleanup item.

## What This Changes

Route promptless Android ACTION_ASSIST assistant launches to the Voice tab with one-shot microphone activation while preserving ASK_OPENCLAW text prompts as Chat drafts.

## Best Possible Solution

Approve the scoped Android assistant UX/privacy policy, then implement promptless ACTION_ASSIST-to-Voice with one-shot mic permission handling, regression tests, and real Android proof while keeping text App Actions draft-only.

## Reproduction Assessment

Yes, by source inspection: ACTION_ASSIST parses as a promptless assist request, MainActivity forwards it to MainViewModel, and MainViewModel requests Chat. I did not run a live Android device or emulator assistant gesture flow.

## Solution Assessment

Unclear until Android owners approve the UX/privacy direction. The proposed files are the right narrow surface, but auto-opening Voice and activating or requesting the mic must be scoped carefully and preserve ASK_OPENCLAW draft-only behavior.
