## Summary

Keep open. Current main and the latest release still route promptless Android assistant launches to Chat, while the requested Voice-tab mic activation is a user-facing Android UX/security decision rather than a current-contract bug fix.

## What This Changes

Request promptless Android ACTION_ASSIST launches to open Voice and activate the mic once, while prompt-bearing ASK_OPENCLAW launches continue to Chat.

## Best Possible Solution

Decide the Android assistant UX policy, then implement promptless ACTION_ASSIST to Voice with one-shot mic activation while preserving draft-only ASK_OPENCLAW Chat behavior and adding focused Android tests/docs.

## Reproduction Assessment

Yes for the current behavior from source, but not from a live device run. ACTION_ASSIST parses to a promptless assist request, MainActivity forwards it, MainViewModel selects Chat, and VoiceTabScreen has no assistant-origin mic activation branch.

## Solution Assessment

Unclear until maintainers approve the Android UX/security direction. The proposed code boundary is narrow, but routing assistant gestures to Voice and auto-enabling mic changes documented behavior and must preserve the draft-only ASK_OPENCLAW policy from https://github.com/openclaw/openclaw/pull/70714.
