## Summary

Keep open. Current main and the latest release still route promptless Android ACTION_ASSIST launches to Chat, and changing that path to one-shot microphone activation is a product/privacy decision rather than conservative cleanup.

## What This Changes

Route Android promptless ACTION_ASSIST launches to the Voice tab with one-shot mic activation while keeping ASK_OPENCLAW text prompts on Chat.

## Best Possible Solution

Decide the Android assistant UX/privacy policy, then implement promptless ACTION_ASSIST to Voice with one-shot mic activation, focused Android coverage, and real assistant/App Actions proof while preserving ASK_OPENCLAW draft-only handling.

## Reproduction Assessment

Yes, from current-main source: ACTION_ASSIST parses as a promptless assist request, MainActivity forwards it, and MainViewModel selects Chat. I did not run a live Android device or emulator flow in this read-only review.

## Solution Assessment

Unclear until maintainers approve the Android UX/privacy direction. The proposed file boundary is narrow, but auto-opening Voice and enabling the mic from the assistant gesture must be scoped carefully and must preserve ASK_OPENCLAW draft-only handling.
