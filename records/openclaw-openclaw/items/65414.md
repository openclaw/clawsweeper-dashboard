## Repair Work Prompt

Fix Android assistant gesture routing for https://github.com/openclaw/openclaw/issues/65414. Current main parses `Intent.ACTION_ASSIST` as `AssistantLaunchRequest(source = "assist", prompt = null, autoSend = false)`, but `MainViewModel.handleAssistantLaunch` routes all assistant launches to Chat. Implement the narrow Android-side fix: when `request.source == "assist" && request.prompt == null`, request `HomeDestination.Voice`, clear stale chat launch state, and set a one-shot pending voice auto-activation flag. Expose that flag from `MainViewModel` and add a consume method. In `VoiceTabScreen`, observe the flag with a `LaunchedEffect`, consume it exactly once, request `RECORD_AUDIO` if needed, and call `setMicEnabled(true)` only after permission is already granted or newly granted. Do not change the manifest, App Actions metadata, gateway config, or schema. Keep prompt-bearing `ASK_OPENCLAW`/App Actions behavior routed to Chat. Add focused Android tests where feasible, update `docs/platforms/android.md`, and add a user-facing `CHANGELOG.md` fix entry crediting the reporter if appropriate. Run `pnpm android:test`, `pnpm android:lint`, `pnpm android:lint:android`, then `pnpm check:changed` in Testbox before handoff.
