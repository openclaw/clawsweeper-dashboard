## Summary

Keep open. Current main still leaves Feishu tenant-token/API HTTP calls on the Lark SDK default Axios-backed HTTP instance with only timeout/User-Agent wrapping, and the related open proxy PR still needs rework/proof rather than serving as an implemented fix.

The issue requests a Feishu HTTP client fix for tenant_access_token/API failures when HTTP(S) proxy environment variables are configured.

## Best Possible Solution
