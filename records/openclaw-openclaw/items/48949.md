## Summary

Keep open: current main and the latest release still leave Feishu SDK REST/token HTTP on a timeout-only Lark SDK wrapper, and the linked fix PR is open but not merged. The issue should stay open while maintainers review the Feishu proxy/security policy and the open implementation path.

## What This Changes

The issue reports Feishu/Lark token and message delivery failing when the gateway inherits HTTP proxy environment variables and the Lark SDK axios request to `tenant_access_token` is misrouted.

## Best Possible Solution

Review and either land https://github.com/openclaw/openclaw/pull/86386 or a maintainer-approved replacement, coordinating the Feishu SDK REST, WebSocket, guarded-fetch, docs, tests, and managed-proxy semantics with the related proxy-policy issues.
