## Summary

Keep open: current main still uses a timeout-only Lark SDK HTTP wrapper for Feishu REST/token calls, and the linked closing PR is open and conflicting rather than merged. The issue remains the canonical user report for Feishu proxy/token failures until that fix or a replacement lands.

## What This Changes

Feishu channel should continue token/API and message delivery when HTTP proxy environment or managed proxy settings are present, instead of failing with tenant_access_token errors.
