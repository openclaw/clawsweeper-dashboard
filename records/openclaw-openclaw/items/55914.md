## Summary

Keep open: current main and v2026.6.8 still have QR/setup-code mobile pairing plus device approval, but not the requested short cloud invite-code resolver. This remains a maintainer product and security decision because it adds cloud-mediated Gateway discovery, new auth-adjacent configuration, and likely needs to be designed with the related Cloud Relay request.

## What This Changes

The issue requests a new `openclaw invite` command that creates short cloud-backed invite codes for mobile Gateway pairing without QR scanning or manual connection entry.
