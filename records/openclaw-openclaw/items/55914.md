## Summary

Keep open. Current main still has QR/setup-code and `/pair` plugin flows, but no `openclaw invite`, short `CLAW-xxxx` discovery codes, cloud invite register/resolve config, or mobile invite resolver; related #55915 remains an open design companion rather than a superseding implementation.

## What This Changes

Add an `openclaw invite` command that creates short mobile pairing invite codes through a configurable cloud backend.

## Best Possible Solution

Design this together with #55915, then either implement a coherent invite/relay flow that reuses bootstrap tokens and device approval or explicitly document that setup codes are the supported non-QR pairing path.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing behavior. Current-main source and docs inspection confirms the requested command/config/mobile resolver are absent.

## Solution Assessment

Unclear; the requested direction is plausible but not yet proven to be the narrowest maintainable solution. Maintainers should decide the invite-code/cloud-relay trust model with #55915 before implementation.
