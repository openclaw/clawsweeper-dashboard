## Summary

Keep open. Current main and the latest shipped tag still provide QR/setup-code mobile pairing with direct Gateway URL plus bootstrap token, but they do not include the requested `openclaw invite` command, short cloud invite register/resolve flow, or `cloud.enabled`/`cloud.apiUrl` Gateway config; the linked cloud relay design remains open product/security context.

## What This Changes

The issue requests an `openclaw invite` command that creates short cloud-backed invite codes for mobile Gateway discovery and pairing.

## Best Possible Solution

Keep this open until maintainers choose a coherent cloud pairing design that defines the CLI, Gateway config/schema, backend invite protocol, mobile resolve flow, privacy/rate-limit/replay rules, device-approval invariants, docs, and tests alongside https://github.com/openclaw/openclaw/issues/55915.
