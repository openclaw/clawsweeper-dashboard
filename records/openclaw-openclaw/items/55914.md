## Summary

Keep open. Current main still has QR/setup-code mobile pairing and direct URL/bootstrap-token payloads, but it does not include the requested `openclaw invite` command, short cloud-backed invite register/resolve flow, or `cloud` gateway configuration; the linked cloud relay request is still open design context rather than a merged implementation.

## What This Changes

The issue requests a new `openclaw invite` command that creates short shareable invite codes backed by configurable cloud register/resolve endpoints for mobile pairing.

## Best Possible Solution

Design invite codes together with the cloud relay trust model, then either implement a coherent opt-in cloud pairing flow that preserves bootstrap-token and device-approval guarantees or document setup codes as the supported non-QR path.
