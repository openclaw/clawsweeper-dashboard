## Summary

Keep open. Current main and the latest shipped tag still provide QR/setup-code mobile pairing with direct Gateway URL plus bootstrap token, but they do not include the requested short cloud invite command, invite register/resolve flow, or `cloud.enabled`/`cloud.apiUrl` Gateway config.

## What This Changes

Add an `openclaw invite` command that creates short mobile pairing invite codes backed by a configurable cloud register/resolve API.

## Best Possible Solution

Keep this open until maintainers choose a coherent cloud pairing design that defines the CLI, Gateway config/schema, backend invite protocol, mobile resolve flow, privacy/rate-limit/replay rules, device-approval invariants, docs, and tests alongside https://github.com/openclaw/openclaw/issues/55915.
