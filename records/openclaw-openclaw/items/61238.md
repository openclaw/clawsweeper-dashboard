## Summary

Keep open. Current main and the latest release still keep ordinary sessions on the implicit daily 4:00 AM rollover path without a pre-reset warning, cancel/postpone control, first-class manual-only reset mode, or reset-archive restore flow; related fixes only cover system-event/provider-owned-session edge cases, and https://github.com/openclaw/openclaw/issues/57369 tracks only the narrower manual-only mode.

## What This Changes

The issue asks OpenClaw to add warning/control/restore behavior for silent automatic daily session rollover and make automatic reset opt-out discoverable.

## Best Possible Solution

Define an explicit reset lifecycle contract that combines a supported manual-only mode, visible setup/status disclosure, channel-capable warning or confirmation before automatic rollover, and safe reset-archive restore or branch semantics.

## Reproduction Assessment

Yes, for ordinary sessions by source inspection: omitted reset config resolves to daily at 4:00 AM, and initSessionState creates a new session on the next real message after the boundary. Current main has partial exceptions for system events and provider-owned CLI sessions, but those do not cover the full reported warning/control/restore gap.

## Solution Assessment

Unclear as an implementation plan because no patch is under review and the requested behavior spans defaults, config, channel UX, and archive recovery. The maintainable path is a coordinated product/API decision rather than a narrow bug-fix automation pass.
