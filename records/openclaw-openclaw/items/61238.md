## Summary

Keep open. Current main and v2026.5.18 still implement implicit daily 4:00 AM ordinary-session rollover without a pre-reset warning, cancel/postpone path, first-class manual-only reset mode, or reset-archive restore; related open issues and PRs cover pieces, but none has merged a complete lifecycle fix.

## What This Changes

The issue asks OpenClaw to stop silent daily session rollover by adding warning/cancel/save controls, a discoverable manual-only reset mode or default change, reset-archive restore, and setup documentation.

## Best Possible Solution

Define an explicit reset lifecycle contract that combines a supported manual-only mode, visible setup/status disclosure, channel-capable warning or confirmation before automatic rollover, and safe reset-archive restore or branch semantics.

## Reproduction Assessment

Yes. By source inspection, omitted reset config resolves to daily at 4:00 AM and `initSessionState` creates a new session after the boundary; the missing warning/control/restore pieces are also visible in schema, docs, and RPC inspection.

## Solution Assessment

Unclear as a concrete implementation because the request spans defaults, public config semantics, channel UX, and archive recovery. The maintainable path is a coordinated product/API decision before a narrow implementation.
