## Summary

Keep open. Current main and v2026.5.19 still have the source-backed silent daily rollover, no first-class manual-only reset mode, and no active-session restore path for reset archives; related open work covers useful pieces but not the full reset lifecycle.

## What This Changes

The issue asks OpenClaw to add warning/cancel/postpone controls, a discoverable manual-only reset mode, active restore from reset archives, and stronger setup disclosure for automatic daily session resets.

## Best Possible Solution

Define one reset lifecycle contract that combines prominent setup/status disclosure, the manual-only mode tracked by https://github.com/openclaw/openclaw/issues/57369, channel-appropriate warning or confirmation behavior, and safe read/restore semantics for reset archives.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes. Source inspection shows omitted reset config resolves to daily at 4:00 AM and initSessionState starts a new session for stale ordinary sessions, while schema/docs/commands show the missing manual-only, warning, and reset-archive restore controls.

## Solution Assessment

Is this the best way to solve the issue? Unclear as a direct implementation request because it spans defaults, public config semantics, channel UX, and archive recovery. The maintainable path is a maintainer-owned reset lifecycle contract before a narrow code change.
