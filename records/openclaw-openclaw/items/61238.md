## Summary

Keep open. Current main still has the reported core behavior for ordinary sessions: an implicit daily 4 AM reset policy rolls stale sessions without a warning/cancel path, and the public config/API surface still lacks a supported manual-only mode or general reset-archive restore flow.

## What This Changes

The issue asks OpenClaw to warn or give users control before automatic daily session resets, support disabling automatic resets, and provide a way to restore reset archives.

## Best Possible Solution

Define an explicit session reset contract: add a supported manual-only/no-auto-reset policy, expose the active policy in setup/config/status, add channel-capable warning or confirmation before automatic rollover, and document a safe branch-or-restore path for reset archives.

## Reproduction Assessment

Yes. A high-confidence source/test reproduction exists: with default config and an existing ordinary session before the 4 AM boundary, `initSessionState` evaluates the default daily policy and creates a new session on the next user message after the boundary.

## Solution Assessment

No patch is under review, and current main is only a partial mitigation. The maintainable fix should coordinate config semantics, user-facing disclosure/control, and reset archive recovery instead of treating this as a docs-only cleanup.
