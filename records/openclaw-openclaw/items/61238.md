## Summary

Keep open. Current main still applies an implicit daily 4:00 AM rollover to ordinary sessions without a warning, cancel/postpone path, supported manual-only reset mode, or reset-archive restore flow.

## What This Changes

The issue asks OpenClaw to warn or give users control before automatic daily session resets, support disabling automatic resets, and provide a way to restore reset archives.

## Best Possible Solution

Define an explicit reset contract that combines a supported manual-only policy, visible setup/status/config disclosure, channel-capable warning or confirmation before automatic rollover, and a safe restore-or-branch path for reset archives.

## Reproduction Assessment

Yes. With default config and an ordinary existing session that started before the 4 AM boundary, `initSessionState` follows the tested reset-policy path and creates a new session on the next real message after the boundary.

## Solution Assessment

No patch is under review, and current main is only a partial mitigation. The maintainable fix needs coordinated config semantics, warning/control UX, and archive recovery rather than a docs-only change.
