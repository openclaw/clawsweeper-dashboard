## Summary

Keep open: this PR is a plausible provider-routing fix, but it is not merge-ready because contributor real behavior proof is still absent and source review found the env-aware default is only applied on the dynamic resolver path, not static catalog rows that can win first.

## What This Changes

The branch adds an OpenAI default base URL helper that reads non-empty `OPENAI_BASE_URL`, uses it for GPT dynamic fallback models, adds helper tests, and updates the changelog.

## Best Possible Solution

Use one source-aware OpenAI default base URL path across dynamic fallbacks and static/catalog model resolution, preserving explicit provider config precedence, then land after redacted real proxy proof and targeted checks are green.

## Reproduction Assessment

Yes, source-level: current main hardcodes the OpenAI dynamic fallback URL and the resolver uses discovered/static `baseUrl` when provider config has no `baseUrl`. I did not run a live proxy request in this read-only review.

## Solution Assessment

No, not yet. The helper is useful for dynamic fallback metadata, but the best fix should also cover static/model-registry rows or prove by resolver tests that the intended env-only model refs use the env URL.
