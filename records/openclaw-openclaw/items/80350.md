## Summary

Keep open. Current main has stop/abort triggers and cleanup improvements, but it still relies on cooperative AbortController and Pi AgentSession abort behavior rather than a hard runtime stop contract that prevents later tool dispatch.

## What This Changes

Requested a hard emergency-stop mechanism that can terminate in-flight agent turns before additional tool dispatch.

## Best Possible Solution

Define an owner-approved emergency-stop contract that cancels active providers/tools where possible, prevents later tool dispatch after a stop signal, clears queued work, records a stopped outcome, and documents authorized channel behavior.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection shows current main uses cooperative AbortController and AgentSession.abort paths rather than an explicit hard-stop state that blocks future Pi tool dispatch after cancellation.

## Solution Assessment

Not as a small direct patch. The requested direction is valid, but the best implementation needs a core cancellation contract across gateway, embedded runner, Pi loop, tools, subagents, and transcript state rather than only adding aliases or an early return in runEmbeddedAttempt.
