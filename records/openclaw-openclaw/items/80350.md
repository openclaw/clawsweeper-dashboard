## Summary

Keep open. Current main already has stop/abort triggers, but they flow through cooperative `AbortController` and `AgentSession.abort()` paths; upstream loop behavior still depends on signal-aware model/tool execution rather than an explicit hard runtime stop before future tool dispatch.

## What This Changes

Request an authorized immediate-stop command/opcode that can terminate in-flight OpenClaw agent turns, cancel pending/future tool calls, and leave the session halted.

## Best Possible Solution

Define an owner-approved emergency-stop contract that cancels active providers/tools where possible, prevents any later tool dispatch after the stop signal, clears queued work, records a stopped outcome, and documents authorized channel behavior.

## Reproduction Assessment

No high-confidence live reproduction was established. Source inspection does show the current stop path is cooperative cancellation, not an explicit hard IMST-style runtime state machine.

## Solution Assessment

Not as a small direct patch. The requested direction is valid, but the best implementation needs a core cancellation contract rather than only adding aliases or another early return in `runEmbeddedAttempt`.
