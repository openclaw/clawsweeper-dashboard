## Summary

Keep open. Current main recognizes stop phrases and has cooperative abort paths, plus recent abort-settle cleanup, but it still lacks the requested hard IMST-style contract that prevents any later tool dispatch after an in-flight stop signal.

## What This Changes

Request a hard emergency-stop opcode for in-flight agent turns that terminates the runtime turn, clears queued work, and replies with an explicit stopped state.

## Best Possible Solution

Define and implement a documented emergency-stop contract that cancels active work where possible, prevents any later tool dispatch after the stop signal, clears queued work, records a stopped outcome, and preserves authorized channel boundaries.

## Reproduction Assessment

No high-confidence live reproduction was established. Source and dependency inspection do show the current stop path is AbortSignal-based cooperative cancellation rather than a hard IMST-style runtime state machine.

## Solution Assessment

No, this is not best solved as a small alias or early-return patch. The maintainable direction is an owner-approved core cancellation contract spanning active provider calls, tool execution, subagents, queues, and transcript cleanup.
