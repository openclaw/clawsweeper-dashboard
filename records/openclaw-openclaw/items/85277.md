## What This Changes

The PR adds a Responses replay-id helper, uses it in the OpenAI Responses transport, and adds regression coverage for omitting signed assistant message IDs when paired reasoning was pruned.

## Best Possible Solution

Land one coherent Responses replay policy that avoids provider-rejected stale item IDs while preserving safe store-enabled continuity, with this regression coverage and coordinated merge order across adjacent replay PRs.

Yes at source level: current main replays `textSignature` IDs whenever Responses item-id replay is enabled, so a pruned-reasoning transcript can send a signed message ID without its paired reasoning item. I did not run a live OpenAI failure in this read-only review.

## Solution Assessment

Yes, with maintainer coordination: the helper localizes the item-id decision and preserves signed IDs only when a replayed reasoning item immediately precedes the message. The remaining question is final replay-policy coordination with adjacent Responses PRs, not a discrete code defect in this patch.
