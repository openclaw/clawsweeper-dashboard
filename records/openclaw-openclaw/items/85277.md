## Summary

Keep open: current main partially covers the observed model-switch/orphaned `msg_*` case through the merged sanitizer fix, but this PR still proposes a broader converter-level replay guard for raw signed assistant message ids that can reach the Responses payload without paired reasoning.

## What This Changes

The PR adds a Responses replay-id policy helper, applies it to the embedded OpenAI Responses transport and shared Responses converter, and adds regression coverage for pruned reasoning plus phase-tagged assistant message ids.

## Best Possible Solution

Keep one coherent Responses replay contract: retain the sanitizer fix already on main, then either land or fold this lower-layer converter guard so store-disabled replay, pruned reasoning, phase metadata, and recovery retries all agree.
