## Summary

Keep this PR open, but the current patch should not merge as-is: the new replay policy still preserves phase-tagged raw `msg_*` ids without paired reasoning, which conflicts with the merged orphan-id sanitizer contract and the live Azure proof from the related fix.

## What This Changes

The PR adds a Responses replay-id helper, applies it to the embedded OpenAI Responses transport and shared Responses converter, and adds replay tests for pruned reasoning and phase-tagged assistant message ids.

## Best Possible Solution

Align the PR with the current sanitizer contract: preserve `phase` metadata, but replay a raw assistant `msg_*` id only when the paired reasoning item was actually replayed immediately before it.

## Reproduction Assessment

Yes, source and related live proof give a high-confidence path: a phase-bearing `msg_*` replayed without its paired `rs_*` reasoning is the provider-rejected shape documented by the merged Azure proof, and this PR preserves that id when `phase` exists.

## Solution Assessment

No, not yet. The right fix is narrower: keep phase metadata for OpenAI Responses follow-up quality, but do not preserve a raw provider message id unless its paired reasoning item was replayed.
