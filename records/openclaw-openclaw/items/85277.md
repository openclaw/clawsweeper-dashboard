## What This Changes

The branch adds a Responses replay-id helper, wires it into assistant text conversion, and extends reasoning replay coverage for pruned-reasoning signed message ids.

## Best Possible Solution

## Solution Assessment

Yes, with maintainer coordination: the helper localizes the item-id decision and preserves signed ids only when replayed reasoning immediately precedes the message. The remaining question is the broader store/retry/session-reset policy across adjacent PRs, not a concrete defect in this patch.
