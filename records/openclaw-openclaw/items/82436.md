## Summary

Keep open for maintainer product/API follow-up. Current main has reliable model-switch persistence and some `session:patch` hook coverage, but it still lacks the requested opt-in Discord channel rename feature and a clearly supported public plugin path for safely renaming Discord channels after every model-switch route.

## What This Changes

Request an opt-in Discord feature that renames a bound Discord channel with the active session model whenever the model is changed.

## Best Possible Solution

Add a Discord-owned opt-in implementation or a stable plugin/API seam that observes effective session model changes and safely, idempotently renames Discord channels with documented rate-limit and permission behavior.

## Reproduction Assessment

Not applicable as a bug reproduction. Source and docs inspection confirm model switching exists and the requested Discord channel rename config is absent on current main.

## Solution Assessment

Unclear until maintainers choose the product boundary. The requested behavior is plausible, but it should be solved through a Discord-owned opt-in feature or stable SDK/plugin seam rather than by relying on `message:received` parsing or session-store internals.
