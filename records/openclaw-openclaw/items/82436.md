## Summary

This is a useful Discord automation idea, but it is optional channel-specific behavior and current main has the extension points needed to build it outside OpenClaw core: model changes can be observed through `session:patch`, and Discord channel names can already be edited through the Discord plugin action/runtime surface.

## What This Changes

Add an opt-in Discord feature that renames the bound channel with the active model when `/model` or the native Discord model picker changes the session model.

## Best Possible Solution

Publish this as a ClawHub code plugin or hook pack that listens for model-related `session:patch` events, derives the Discord channel from the session route, and calls the existing Discord channel-edit surface with its own template and rate-limit handling.

## Reproduction Assessment

Not applicable; this is a feature request for new Discord channel-name automation, not a report of existing documented behavior failing. Source inspection confirms current docs describe the model picker but not automatic channel renaming.

## Solution Assessment

No for OpenClaw core; the narrower maintainable path is a ClawHub plugin/hook using `session:patch` plus Discord `channelEdit`. A built-in channel-renaming config would add optional Discord policy, template, permission, and rate-limit behavior to core without a missing extension API.
