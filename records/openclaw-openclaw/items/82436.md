## Summary

Keep open. This is a concrete Discord enhancement: current main persists model switches and can edit Discord channels, but it lacks an opt-in channel rename setting or stable model-changed event covering both text and native `/model` paths.

## What This Changes

Request an opt-in Discord configuration that renames a bound channel with the active session model when the model changes.

## Best Possible Solution

Design this either as a Discord plugin opt-in with template, alias, idempotence, permission, and rate-limit handling, or as a generic model-change event/API that a Discord plugin can consume safely.

## Reproduction Assessment

Not applicable: this is an enhancement request, not a broken documented behavior. Source inspection confirms the current model-switch and Discord channel-edit primitives exist, but no automatic channel-title model indicator is defined.

## Solution Assessment

Unclear until maintainer product/API review. The proposed direction is plausible, but the durable solution should decide first between a Discord-specific opt-in feature and a generic model-change hook that plugins can rely on.
