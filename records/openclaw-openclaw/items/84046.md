## Summary

Keep open: the fix direction and real-world proof are strong, but the current patch is not merge-ready because the append-only slow path can advance the session fence over non-canonical message entries that the session parser would reject.

## What This Changes

The PR adds a prefix-hash session-file fence snapshot that tolerates append-only runner-owned assistant/tool transcript entries and exports `isAgentMessage` for tail validation.

## Best Possible Solution

Accept append-only tail bytes only when every line is a canonical persisted message entry with a valid outer base and a runner-owned role, while leaving coordinated co-tenant ownership handling to the separate shared-session issue.

## Reproduction Assessment

Yes. Current main's stat-only fence path and the supplied gateway logs establish the false-positive behavior, and the remaining PR defect is source-reproducible by appending a `type: "message"` assistant tail without the required outer `id`.

## Solution Assessment

No. The patch is the right general shape, but the slow path must validate the full persisted session-entry base before it is the safest maintainable fix.
