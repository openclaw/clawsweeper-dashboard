## Summary

This PR is a strong, well-evidenced fix for the embedded session-lock false positive, but the latest slow-path validator still accepts malformed runner-owned message content that the persisted transcript parser would reject, so it should stay open for one more tightening pass.

## What This Changes

The PR replaces the embedded attempt session-file stat-only takeover fence with a snapshot plus append-only verification path and adds regression tests for runner-owned transcript appends, user-turn takeovers, rewrites, replacements, non-message tails, and malformed tool results.

## Best Possible Solution

Merge a corrected version that tolerates only append-only entries matching the full persisted runner-owned message contract, while continuing to reject user turns, non-message entries, rewrites, replacements, and malformed tails.

## Reproduction Assessment

Yes. Current main throws on any session-file stat mismatch after prompt-lock release, and the runner's own reentrant transcript append path writes the same JSONL during that window; the PR body also supplies before/after live Telegram logs.

## Solution Assessment

No. The approach is close, but the local runner-owned validator must actually mirror the persisted transcript message contract before this is the maintainable session-state fix.
