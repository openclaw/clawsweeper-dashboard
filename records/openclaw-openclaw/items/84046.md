## Summary

Keep this PR open: current main still has a source-clear path where Pi SessionManager prompt-time appends can trip the stat fence, and the PR has useful real-runtime proof plus focused regression coverage. It should not be closed as superseded by the landed delivery-mirror fix, but the latest discussion correctly needs a maintainer choice on whether to continue with append-only fence tolerance or extend the newer owned-write context approach.

## What This Changes

The PR changes the embedded runner session-file fence to accept append-only canonical assistant/tool transcript tails, exports the canonical session-entry validator, and adds session-lock regression coverage for accepted runner writes and rejected foreign or malformed writes.

## Best Possible Solution

Land one coherent session-lock fix on top of the current owned-write context: either route Pi SessionManager prompt-time appends through a refreshable owned path, or explicitly accept and test the append-only fence tolerance for runner-owned roles while keeping user, compaction, replacement, and malformed writes fail-closed.
