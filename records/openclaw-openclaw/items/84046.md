## Summary

Keep this PR open: it has strong real-world proof and the solution direction is plausible, but the latest slow path still has a concrete malformed-tail bug that should be fixed before merging a relaxed session-state fence.

## What This Changes

The PR changes the embedded runner session-file fence to accept append-only canonical assistant/tool transcript entries written during the released prompt window while preserving takeover detection for user turns, rewrites, replacements, and malformed tails.

## Best Possible Solution

Keep the append-only canonical runner-owned tail approach, but make every malformed or non-object tail value fail closed with EmbeddedAttemptSessionTakeoverError before relaxing the fence.

## Reproduction Assessment

Yes. Current-main source shows any runner-owned transcript append during the released prompt window changes the file fingerprint, and the PR body plus production comment provide real logs of the false takeover on long tool-using channel turns.

## Solution Assessment

No, not as-is. The append-only tail verification is the right narrow shape, but it must reject non-object parsed JSON before delegating to isSessionEntry so malformed tails preserve the current fail-closed takeover path.
