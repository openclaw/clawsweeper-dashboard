## Summary

Keep open: the reported false-positive takeover path is plausible, but this PR is not merge-ready because the count-only fence can miss real same-user-count transcript mutations and the PR only provides unit-test proof.

## What This Changes

The PR changes embedded attempt session-lock fencing to count user-role JSONL messages on stat mismatch and adds a unit test for assistant/tool appends during the released prompt window.

## Best Possible Solution

Tolerate only known self-owned assistant/tool transcript appends while preserving takeover detection for structural rewrites, file replacements, unexpected same-count mutations, and genuine new user turns; track broader per-session turn queue or retry behavior separately.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: current main snapshots a stat fingerprint before prompt I/O, while direct transcript appends can change size/mtime without adding a user turn. I did not run a live channel reproduction in this read-only review.

## Solution Assessment

No, not yet. Counting user entries addresses the likely false-positive shape, but it is too broad because same-count transcript rewrites or replacements can still change session ownership and should not be silently accepted.
