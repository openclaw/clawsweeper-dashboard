## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Reject non-object JSON tail entries:** `src/agents/pi-embedded-runner/run/attempt.session-lock.ts:254`
  - body: `JSON.parse` can return syntactically valid non-objects such as `null`, and this code casts that value to `FileEntry` before calling `isSessionEntry`. Since `isSessionEntry` dereferences `entry.type`, a `null` tail line throws `TypeError` instead of returning `ok: false`, so current main's fail-closed takeover behavior is not preserved for malformed appended entries. Please guard parsed tail values as non-null records before delegating to `isSessionEntry`, and add a regression for a primitive JSONL tail.
  - confidence: 0.86
