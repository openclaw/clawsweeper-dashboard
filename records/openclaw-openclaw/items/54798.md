## Summary

Keep open. Current `main` still lacks the requested atomic counter, `sessions kv incr` CLI/Gateway surface, `sessions_spawn` mutex/exclusive parameter, or agent-facing workspace lock primitive; the remaining decision is whether this belongs in core or as a plugin/ClawHub capability.

## What This Changes

Request an atomic workspace/session counter or equivalent mutex/lock primitive so concurrent cron-spawned sub-agents can share daily rate limits without file-level CAS workarounds.

## Best Possible Solution

Maintainers should first choose the API boundary; if core is desired, add a small atomic workspace/session state primitive with CLI, Gateway RPC, agent-tool coverage, docs, and cross-process regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. The missing-feature check is high-confidence from current source and docs: no `sessions kv incr`, no shared counter method, and no `exclusive`/mutex parameter in `sessions_spawn`.

## Solution Assessment

Unclear. A core counter directly matches the requested API, while the plugin SDK could host a tool/Gateway/CLI implementation; maintainers need to choose the intended boundary before implementation.
