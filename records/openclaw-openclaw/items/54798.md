## Summary

Close as ClawHub scope: the shared counter use case is useful, but current plugin surfaces can provide an agent-callable tool and optional CLI backed by SDK file locks/json storage without adding a new core sessions KV or spawn mutex API.

## What This Changes

Issue requests a shared atomic counter, spawn mutex, or file-lock primitive for cron sub-agents updating daily quota state concurrently.

## Best Possible Solution

Publish an installable ClawHub tool/CLI plugin that owns the workspace-scoped atomic counter using SDK file locks/json storage; add core API only if that plugin proves a missing extension seam.

Not applicable as a strict bug reproduction because this is a feature/API request. Source inspection confirms current core lacks the proposed primitive, while plugin APIs provide enough surface to implement it outside core.

Yes for the ClawHub/plugin direction. Adding a core sessions KV or spawn mutex would be heavier than needed unless a plugin implementation proves a missing core extension API.
