Add an agent/subagent-safe atomic shared counter or lock primitive for concurrent cron subagents so daily counters cannot race.

## Best Possible Solution

Define one narrow, documented atomic state contract for agent/subagent callers, reuse existing plugin file-lock or detached-runtime seams only where they fit, and cover it with cross-process regression proof before layering daily-limit workflows on top.
