## Summary

Current main already routes successful structured tool media into the final reply path and Telegram delivers `mediaUrls`, so the reported text-only Telegram completion is fixed on main even though I could not prove it shipped in v2026.5.3.

## What This Changes

The issue asks for generated image media from `image_generate` to be delivered in Telegram auto-replies instead of allowing a text-only completion.

## Best Possible Solution

Keep the current structured media fallback on main and let the next release carry it to users affected on v2026.5.3-era builds.

## Reproduction Assessment

No high-confidence failing reproduction remains on current main. The reported v2026.5.3-1 transcript is plausible, but current main source and regression tests show generated tool media is attached to the final reply payload.

## Solution Assessment

Yes. The current implementation is the narrow maintainable fix: keep media structured in the tool result, queue it as fallback tool media, merge it into the final assistant reply, and reuse Telegram's existing `mediaUrls` delivery path.
