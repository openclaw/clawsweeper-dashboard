## Summary

Current main already implements the reported delivery path: generated image tool results carry structured media, pending tool media is queued/merged into the visible reply or emitted as tool media, and Telegram sends the resulting media payload. The proof commit is on main and is not contained in a release tag I could identify, so this is fixed on main after v2026.5.3 rather than in the latest release.

## What This Changes

Ensure Telegram auto-reply sessions deliver successful `image_generate` MEDIA outputs instead of finishing with a text-only success reply.

## Best Possible Solution

Keep the current structured `details.media` plus pending-tool-media fallback so generated media is delivered by the runtime even when the assistant's final text omits MEDIA.

## Reproduction Assessment

No live current-main reproduction was run. The issue provides a concrete released-build Telegram repro, and current source/tests cover the exact missing delivery path rather than preserving a current failing path.

## Solution Assessment
