- `impact:message-loss`: The reported failure mode is delayed or missing delivery of a queued plugin system event to the agent.

## Summary

Keep open: current main still has a plausible plugin heartbeat wake gap. The direct plugin `runHeartbeatOnce` wrapper cannot forward the wake metadata that current runner logic uses, and the reported bare `reason: "hook"` call is not classified as a wake payload.

## What This Changes

A plugin enqueues a system event and calls `runHeartbeatOnce`, expecting either a real agent turn or a clear skipped result instead of a fast successful result with the event still queued.

## Best Possible Solution

Define the supported plugin wake contract, then make direct plugin heartbeat runs either carry explicit wake metadata into the runner or return an explicit skipped reason when no model turn consumed the queued event, with regression coverage for queued system events.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection does show that the reported plugin call cannot express the wake metadata current `runHeartbeatOnce` uses, and bare `reason: "hook"` is not classified as a wake payload.

## Solution Assessment

Unclear until maintainers choose the public plugin API direction. The maintainable fix should align the direct helper with the runner's wake contract or make the no-turn result explicit rather than adding another ad hoc heartbeat path.
