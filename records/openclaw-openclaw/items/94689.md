## Summary

Close as ClawHub/plugin-owned: current main and the latest release do not contain a bundled Stoat/Revolt channel plugin, and OpenClaw policy/docs route optional channel-specific REST/auth behavior to plugin packages or ClawHub when the existing channel plugin seams can own it.

## What This Changes

The issue asks OpenClaw to make a Stoat/Revolt channel plugin send REST requests with `x-bot-token`, or offer a header override, so self-hosted Stoat replies are delivered.

## Best Possible Solution

Keep OpenClaw core unchanged; fix the Stoat/Revolt REST auth header in the owning channel plugin package or ClawHub listing, and open a new OpenClaw issue only if that plugin needs a missing SDK seam.

## Reproduction Assessment

No for OpenClaw main: current main and v2026.6.8 do not include a bundled Stoat/Revolt plugin path to exercise. The reporter supplied useful external raw API evidence, but that applies to the owning plugin/server setup rather than a source path in this repository.

## Solution Assessment

No for core: the narrow maintainable fix is in the Stoat/Revolt channel plugin REST client or its plugin-owned config, not in OpenClaw core. A core change becomes appropriate only if the plugin cannot express the behavior through the existing channel plugin outbound/auth seams.
