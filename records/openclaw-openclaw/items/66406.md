## Summary

Keep open because current main and the latest release still pass raw Unicode emoji into Feishu reaction APIs, while the canonical fixing PR remains open with closing linkage for this issue.

## What This Changes

The issue reports Feishu `react` message actions failing with 400/231001 when Unicode emoji are sent where Feishu expects documented `emoji_type` strings.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/66478 with a Feishu-local Unicode-to-`emoji_type` normalizer, documented-value mapping, and focused regression coverage, then close this issue after merge.

## Reproduction Assessment

Yes, at source level. The issue gives concrete `message(action="react", emoji="👍")` steps, current main passes that raw value into Feishu `reaction_type.emoji_type`, and Feishu documents 231001 for invalid reaction types.

## Solution Assessment

Yes, the right direction is a Feishu-owned normalization layer at the reaction API boundary. This issue should stay open while the linked canonical PR is reviewed, repaired if needed, and landed.
