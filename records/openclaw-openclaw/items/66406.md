## Summary

Keep open because current main and the latest release still pass raw Unicode emoji into Feishu reaction API calls, while the focused fixing PR remains open with closing linkage for this issue.

## What This Changes

The issue asks OpenClaw to make Feishu `message(action="react")` accept common Unicode emoji by sending Feishu-supported `emoji_type` values to the reactions API.

## Best Possible Solution

Normalize Feishu reaction inputs at the Feishu API boundary, keep documented Feishu type strings working, add focused regression tests, and close this issue after the canonical fix merges.

## Reproduction Assessment

Yes, at source level. The reported `message(action="react", emoji="👍")` path still passes the raw value into `reaction_type.emoji_type`, while Feishu documents named `emoji_type` values such as `THUMBSUP`.

## Solution Assessment

Yes, the best fix is a Feishu-local normalizer at the reaction API boundary with regression tests. The open canonical PR already targets that shape, while the newer Feishu reaction-tool PR is complementary rather than a reason to close this issue.
