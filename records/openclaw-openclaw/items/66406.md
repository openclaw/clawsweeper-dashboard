## Summary

Keep open. Current main still trims the user-provided Feishu reaction emoji and passes it unchanged into the Feishu reaction API, while open PR #66478 remains the focused implementation candidate with closing syntax for this issue.

## What This Changes

The issue requests fixing Feishu reaction calls so Unicode emoji such as thumbs-up are accepted instead of returning Feishu error 231001, `reaction type is invalid`.

## Best Possible Solution

Keep this issue open and land PR #66478 or an equivalent Feishu plugin fix that normalizes user-facing Unicode emoji and accepted Feishu type strings before add, remove, and filtered list reaction API calls. The fix should include a focused regression test proving `emoji="👍"` sends a Feishu type such as `THUMBSUP`.

## Reproduction Assessment

Yes. The issue gives concrete configuration and action steps, and current-main source inspection shows `emoji="👍"` would be trimmed and sent unchanged as Feishu `emoji_type`; no live Feishu mutation was run in this read-only review.

## Solution Assessment

Yes. The narrow maintainable direction is a Feishu-owned normalization boundary around the reaction helper/action path, not a core message-tool change; the issue should close only after that implementation or an equivalent fix merges.
