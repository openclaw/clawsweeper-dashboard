## Summary

Keep open because current main still passes the user-provided Feishu reaction emoji through unchanged, and open PR #66478 remains the focused implementation candidate with closing syntax for this issue.

## What This Changes

The issue asks OpenClaw to make Feishu `message action=react` work when users provide normal Unicode emoji such as thumbs-up.

## Best Possible Solution

Land PR #66478 or an equivalent Feishu plugin fix that normalizes user-facing Unicode emoji and accepted Feishu type strings before add, remove, and filtered list reaction API calls, with a regression test proving thumbs-up sends `THUMBSUP`.

## Reproduction Assessment

Yes. The issue gives concrete Feishu config and tool-call steps, and current main shows `emoji="👍"` would be sent unchanged as Feishu `emoji_type`, which the Feishu API documents as a typed string field rather than raw Unicode. citeturn1view1turn1view2

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Feishu plugin reaction boundary, not in the core message tool, because Feishu owns the platform-specific emoji vocabulary and API contract.
