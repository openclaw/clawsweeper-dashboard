## Summary

Keep open because current main and the latest release still pass raw Unicode emoji into the Feishu reaction API, and the open canonical fixing PR still has closing linkage for this issue. The source path and Feishu API contract both support the reported 231001 failure.

## What This Changes

The issue reports that Feishu `message(action="react", emoji="👍")` fails with Feishu API code 231001 because Unicode emoji are sent as reaction types.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/66478 with a Feishu-local Unicode-to-documented-`emoji_type` normalizer and focused regression tests, then close this issue after merge.

## Reproduction Assessment

Yes, at source level. The issue gives concrete `message(action="react", emoji="👍")` steps, current main passes that raw value into Feishu `reaction_type.emoji_type`, and Feishu documents code 231001 for invalid reaction types.

## Solution Assessment

Yes, the maintainable fix is to normalize Feishu reaction inputs at the Feishu API boundary using documented emoji_type values. Closing this issue now would be premature because the canonical fixing PR has not merged.
