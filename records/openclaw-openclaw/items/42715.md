## Summary

Current main no longer matches the reported silent fallback path: explicit per-agent model selections are strict unless fallbacks are configured, missing provider auth fails before request dispatch, and fallback attempts are observable when fallback is intentionally enabled.

## What This Changes

Ensure Cloudflare AI Gateway custom providers without an apiKey fail clearly instead of silently falling back to the default model, with clearer credential/fallback behavior.

## Best Possible Solution

Keep the current fail-fast and explicit-fallback behavior; track gateway-level credential inheritance separately only if maintainers want that additional configuration feature.

## Reproduction Assessment

No. I did not run a live Cloudflare Gateway scenario, but current-main source and regression tests show the reported per-agent missing-apiKey path now fails instead of inheriting the default model fallback.

## Solution Assessment

Yes. The current implementation is the narrowest maintainable fix for the silent fallback bug because it makes agent/user model selections exact by default and rejects missing provider auth before provider dispatch; gateway-level credential sharing would be a separate feature.
