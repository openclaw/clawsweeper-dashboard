## Summary

Keep open. Current main addresses the dangerous silent fallback path for explicit per-agent model selections and now fails remote custom providers without credentials loudly, but the remaining Cloudflare AI Gateway multi-provider credential-sharing and documentation request is still not implemented.

## What This Changes

The issue requests gateway-level credential sharing, loud auth-error handling, and onboarding documentation for Cloudflare AI Gateway setups with multiple configured providers.

## Best Possible Solution

Keep the strict missing-auth failure behavior, document the per-provider apiKey requirement for additional Cloudflare AI Gateway custom providers, and add shared credentials only through an explicit non-leaky config design if maintainers choose that product direction.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction for the remaining behavior: configure a remote custom provider without apiKey and resolveApiKeyForProvider rejects with No API key found, while no sibling provider credential is consulted.

## Solution Assessment

No, current main is only a partial solution. The loud-failure change is the right direction, but the shared-credential and docs portions still need an explicit maintainer decision and implementation path.
