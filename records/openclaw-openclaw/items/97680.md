## Summary

Keep open: current main still treats post-core official plugin convergence as channel-based beta sync with fallback to the recorded default/latest spec, while this report asks for exact alignment with an explicitly requested beta release.

## What This Changes

The issue reports that an explicit beta OpenClaw update can upgrade core to the requested beta while leaving official external plugins resolved to the stable latest dist-tag.

## Best Possible Solution

Define the explicit-beta update contract, then make official external plugin sync prefer the requested OpenClaw prerelease version when available and surface a clear warning or failure when it cannot be used.

## Reproduction Assessment

Yes, source-reproducible at medium confidence: explicit core tags are reduced to a channel for plugin sync, and beta default plugin specs resolve through @beta with a default/latest fallback. I did not run the packaged update flow live in this read-only review.

## Solution Assessment

Unclear until maintainers choose the compatibility policy. Exact official plugin alignment looks like the right repair direction, but current docs describe fallback-as-warning behavior that may need to change for explicit beta tags.
