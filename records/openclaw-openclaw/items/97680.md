## Summary

Keep open: current main still treats an explicit update tag as a core package target while post-core official plugin convergence only receives the update channel, so the reported beta-core plus latest-plugin drift is not solved.

## What This Changes

The issue asks for explicit beta-tagged OpenClaw updates to align official external plugins to the same requested beta version when available, or warn/fail clearly when not.

## Best Possible Solution

Define the exact-tag convergence policy, then make trusted official external plugins use the resolved explicit OpenClaw version when available or report a clear missing-version warning/error while preserving ordinary beta-channel fallback behavior.

## Reproduction Assessment

Yes for source-level reproduction, but no live end-to-end replay was run. Current source shows the explicit package tag is resolved for core while plugin convergence receives only channel context and can fall back from @beta to default/latest.

## Solution Assessment

Unclear until maintainers accept the exact-tag plugin convergence policy. The narrowest maintainable fix appears to pass the resolved explicit package version into trusted official plugin sync for exact prerelease tags while preserving normal beta-channel fallback.
