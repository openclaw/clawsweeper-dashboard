## Summary

Keep open. Current main handles the ordinary Feishu chat happy path with native cards, but the text-only degradation path still loses approval-specific fallback guidance and the Feishu docs still do not explain how to finish approvals when buttons are unavailable.

## What This Changes

Request clearer Feishu approval fallback copy and documentation when interactive approval buttons are unavailable or degrade to plain text.

## Best Possible Solution

Keep the native Feishu card path, but add a narrow text-only fallback and docs update so Feishu downgrades explain why buttons are unavailable and show exact allowed `/approve <id> <decision>` commands without changing approval authorization or transport semantics.

## Reproduction Assessment

Yes. A code-level reproduction exists by sending an interactive approval-style payload to a Feishu document-comment target: the current test asserts label-only fallback text, and the forwarded approval test asserts placeholder `/approve <id>` copy.

## Solution Assessment

Yes. A focused fallback-copy and Feishu docs repair is the narrowest maintainable fix; a new core approval API or Feishu native approval client is not required for the reported UX gap.
