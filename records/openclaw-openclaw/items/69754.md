## Best Possible Solution

Land a narrow shared/Feishu fallback repair that preserves exact typed `/approve` commands when buttons cannot render, explains why manual approval is needed on that surface, and leaves native Feishu exec cards to separate maintainer review.

## Solution Assessment

Yes. The narrowest maintainable fix is fallback rendering/copy that preserves typed command actions and explains the manual path; the broader native Feishu card implementation should remain separate from this issue.
