## Summary

Keep open. Current main still has a Feishu text-only degradation path that drops approval-specific fallback guidance, while the adjacent native-approval fallback improvements do not cover Feishu document-comment or generic forwarded-payload degradation.

## What This Changes

The issue asks for clearer Feishu approval fallback guidance when interactive approval buttons are unavailable or approval messages degrade to text.

## Best Possible Solution

Keep Feishu native cards for supported chats, but add approval-aware text fallback copy and docs so degraded Feishu surfaces explain why buttons are unavailable and show exact allowed `/approve <id> <decision>` commands.

## Reproduction Assessment

Yes, at source/test level: send an interactive approval-style payload to a Feishu document-comment target and current code/test expectations produce label-only text without approval guidance. I did not run the test because this was a read-only review, but the deterministic path is clear in current main.

## Solution Assessment

Yes. A focused Feishu fallback-copy, shared renderer, and docs repair is the narrowest maintainable solution; it should not add a new approval transport, config knob, or authorization model.

## Repair Work Prompt

Fix the Feishu approval fallback UX from https://github.com/openclaw/openclaw/issues/69754. Preserve the existing native Feishu card happy path, but when approval payloads degrade to text, especially document-comment targets, the fallback should clearly state that interactive approval buttons are unavailable on that surface and include exact allowed `/approve <id> <decision>` commands. Update the generic forwarded exec approval fallback so it does not rely on `/approve <id>` placeholder copy where the request id or short id is known. Consider whether `renderMessagePresentationFallbackText` should include safe non-URL button command values for approval buttons or whether Feishu should add approval-specific degraded copy before calling it. Update Feishu and advanced exec-approval docs for the user-visible behavior. Add focused regression coverage in the Feishu outbound and approval forwarder/shared renderer tests. Do not change approval authorization, native delivery routing, Feishu credentials, or add new config options. Add a changelog entry because this is user-visible approval UX.
