## Summary

Keep open: the update-hint CLI helper is useful, but the PR's central delivery path is not merge-ready because the added “preambles” are Markdown skill-body snippets, not a supported runtime hook that automatically runs on every skill invocation.

## What This Changes

The PR adds an `openclaw update hint` subcommand that reads cached update-check state, tests it, and inserts shell snippets into selected `.agents/skills` files.

## Best Possible Solution

Use a runtime-owned, security-reviewed update-hint delivery path that covers the active skill snapshot, or narrow this PR to a manual `openclaw update hint` helper and remove the per-skill shell blocks.

## Reproduction Assessment

Yes from source inspection: current main treats SKILL.md files as Markdown instructions and the Claude CLI path only materializes them via `--plugin-dir`, so the added fenced bash blocks are not an automatic preamble runner. The PR body has no captured real run proving otherwise.

## Solution Assessment

No: this is a plausible idea but not the best implementation layer because per-file Markdown snippets are incomplete, stale, and security-sensitive. A central runtime-owned hint path or a manual CLI helper is the maintainable direction.
