## Summary

Keep open: the CLI helper is plausible, but the PR’s advertised automatic skill update notice does not work under the current skill-loading contract because Markdown code fences in `SKILL.md` are read as instructions, not executed as preambles. The PR also still lacks inspectable real behavior proof for the claimed skill-invocation path.

## What This Changes

Adds `openclaw update hint`, tests for cached update-state parsing/output, and Markdown bash preamble blocks to seven bundled skill files.

## Best Possible Solution

Expose update notices through one supported executed runtime/prelude hook, and keep the cached hint helper only if it is part of that documented contract with focused tests and real invocation proof.

## Reproduction Assessment

Yes for the PR defect: source inspection shows current main only lists skills and instructs the model to read `SKILL.md`, while the Claude CLI path only materializes skill files as a plugin. There is no current-main bug to reproduce because this is a feature PR.

## Solution Assessment

No. The cached CLI helper may be useful, but automatic notices should be delivered through a supported executed runtime/prelude hook rather than repeated Markdown code fences.
