## Summary

Keep open: the cached `update hint` helper is plausible, but the PR's automatic skill notification is not wired into an executed path. Current skill delivery advertises `SKILL.md` files for the model to read, and the Claude CLI path materializes skills as plugin directories; neither path runs fenced Markdown before a skill invocation.

## What This Changes

The PR registers a new `openclaw update hint` subcommand with cached update-state parsing/tests and adds shell-fenced update-hint snippets to bundled maintainer skill files.

## Best Possible Solution

If maintainers want this feature, expose update hints through one shared executed runtime/prelude hook and keep the cached-state CLI helper only with focused tests and real behavior proof.

## Reproduction Assessment

Yes, for the PR defect from source inspection. The current prompt and Claude CLI paths expose skills as files/directories to read, while the PR relies on fenced Markdown being executed before skill use.

## Solution Assessment

No. The cached-state helper may be useful, but per-skill Markdown fences are not the narrowest maintainable delivery mechanism for an automatic runtime notice.
