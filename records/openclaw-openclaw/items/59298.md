## Summary

Keep this PR open, but it is not merge-ready: the cached CLI helper is plausible, while the advertised automatic skill notice is implemented as fenced Markdown that current skill loading/materialization does not execute.

## What This Changes

This PR adds an `openclaw update hint` subcommand, unit coverage for cached update-state parsing/output, and fenced shell update checks to seven `.agents/skills/*/SKILL.md` files.

## Best Possible Solution

Implement automatic notices through a single documented runtime-owned skill prelude or update-notice hook with tests and real invocation proof, or narrow this PR to the cached CLI helper if maintainers do not want executable skill metadata.

## Reproduction Assessment

Yes for the PR defect by source inspection: the branch adds fenced bash to `SKILL.md`, while current main exposes skills through read-based prompt loading and Claude plugin materialization rather than a prelude executor.

## Solution Assessment

No. The cached CLI helper may be useful, but repeated Markdown shell fences are not a maintainable automatic execution path; a runtime-owned hook or helper-only scope is safer.
