## Summary

Keep this PR open, but it is not merge-ready: the cached update-hint CLI helper is plausible, while the promised automatic skill notification is currently implemented as fenced Markdown in skill files, which current OpenClaw skill loading advertises or materializes but does not execute as a prelude.

## What This Changes

The PR adds an `openclaw update hint` CLI subcommand, unit tests for cached update-state parsing/output, and fenced shell update checks to seven bundled `.agents/skills/*/SKILL.md` files.

## Best Possible Solution

Implement automatic notices through a single documented runtime-owned skill prelude or update-notice hook with tests and real invocation proof, or narrow this PR to the cached CLI helper if maintainers do not want executable skill metadata.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the PR defect by source inspection: the branch adds fenced bash to `SKILL.md`, while current main exposes skills through read-based prompt loading and Claude plugin materialization rather than a prelude executor.

## Solution Assessment

Is this the best way to solve the issue? No. The cached CLI helper may be useful, but repeated Markdown shell fences are not a maintainable automatic execution path; a runtime-owned hook or helper-only scope is safer.
