## Summary

This PR should stay open because current `main` does not already implement the requested update hint, and the branch still has a correctness blocker: adding fenced bash blocks to `SKILL.md` files does not make them run before skill invocation in OpenClaw's current skill delivery paths.

## What This Changes

The PR adds an `openclaw update hint` CLI command, tests for cached update-state parsing/output, and fenced bash update-hint snippets to bundled skill Markdown files.

## Best Possible Solution

If maintainers want this feature, keep the cached-state CLI helper only if the notice is surfaced through a shared executed skill/runtime prelude with tests and contributor-provided real behavior proof.

## Reproduction Assessment

Yes for the PR defect from source inspection. The update notice is added as fenced Markdown in skill files, while current `main` reads skill files as instructions and materializes Claude plugin files without executing those fences.

## Solution Assessment

No. A cached update-hint helper may be a reasonable component, but per-skill Markdown fences are not the narrowest maintainable delivery mechanism for automatic runtime notices.
