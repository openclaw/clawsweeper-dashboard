## Summary

Keep open: the useful update-helper idea is still unmerged, and the proposed automatic skill notification path is not wired into an executed skill hook on current main.

## What This Changes

The branch adds an `openclaw update hint` CLI subcommand with tests and inserts fenced bash update-check snippets into seven maintainer `SKILL.md` files.

## Best Possible Solution

If maintainers want this feature, keep the cached-state helper but surface notices through one shared executed runtime or skill-prelude hook with coverage instead of per-skill fenced Markdown snippets.

## Reproduction Assessment

Yes for the PR defect from source inspection. Current main exposes skills by prompting the model to read `SKILL.md` and by materializing Claude plugin files, so the added fenced bash block is not an executed preamble.

## Solution Assessment

No. The cached CLI helper may be useful, but per-skill fenced Markdown is not an executed or maintainable delivery mechanism; a shared runtime hook or explicit maintainer-approved prelude is safer.
