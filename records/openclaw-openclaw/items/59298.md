## Summary

Keep open. Current main has not implemented this feature, and the PR still has a blocking correctness issue: the added skill preambles are Markdown fences, while OpenClaw currently exposes skills by prompting the model to read SKILL.md files or by materializing them as Claude Code plugin files, not by executing fenced shell before skill use.

## What This Changes

The PR adds an `openclaw update hint` subcommand, tests, and per-skill Markdown preambles intended to surface cached update availability to agents.

## Best Possible Solution

If maintainers want this feature, keep the cached-state helper but surface update notices through one shared executed skill/runtime prelude with coverage rather than per-skill Markdown snippets.

## Reproduction Assessment

Yes for the PR defect from source inspection. The added bash block is fenced Markdown in SKILL.md, while current main tells agents to read skill files and materializes Claude skill directories without executing fences.

## Solution Assessment

No. A cached CLI helper may be useful, but per-skill Markdown preambles are not an executed or maintainable delivery mechanism; a shared runtime hook or explicit product decision is the safer direction.
