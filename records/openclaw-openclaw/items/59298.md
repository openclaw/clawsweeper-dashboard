## Summary

This PR should stay open because the central promised behavior, automatic update notification through skill preambles, is not implemented by the current approach: the added Markdown code fences are read as skill instructions, not executed before skill invocation. The cached CLI helper itself is plausible and the earlier JSON/Zod review issues appear addressed, but the branch still needs a runtime/product decision and real behavior proof before merge.

## What This Changes

The PR adds an `openclaw update hint` subcommand that reads cached update-check state, adds tests for its output and validation paths, and inserts shell-code preamble blocks into bundled `.agents/skills/*/SKILL.md` files.

## Best Possible Solution

Expose any update notice through a single supported executed runtime/prelude hook, and keep the cached `openclaw update hint` helper only if it is part of that supported hook contract.

## Reproduction Assessment

Yes for the PR defect: source inspection gives a high-confidence path because current main loads skills by making `SKILL.md` available for reading, while the branch only adds fenced shell blocks to that Markdown. There is no current-main bug reproduction because the PR is adding a new feature.

## Solution Assessment

No. A cached hint command may be a useful building block, but per-skill Markdown fences are not the best way to deliver automatic runtime notifications because they are not executed by the documented skill-loading paths.
