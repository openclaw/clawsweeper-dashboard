## Summary

Keep open. This is an active contributor feature PR and current main does not implement `openclaw update hint`; however, the advertised automatic skill notification is wired as fenced Markdown in `SKILL.md`, while current skill delivery only advertises or materializes skill files and does not execute Markdown code blocks before a skill run.

## What This Changes

The PR adds an `openclaw update hint` CLI helper with tests and inserts identical cached-update bash snippets into seven maintainer skill Markdown files.

## Best Possible Solution

If maintainers want this feature, keep the small cached-state CLI helper but surface update availability through a shared executed runtime hook or documented generic skill prelude, with tests proving the hint appears for eligible skill invocations without hand-editing individual skill files.

## Reproduction Assessment

Yes for the PR defect. Comparing the PR diff with current main shows the added block is a Markdown fence, while the skill pipeline only prompts the model to read `SKILL.md` or passes a Claude plugin directory, so there is no pre-run execution path for the block.

## Solution Assessment

No. The cached CLI helper is a reasonable building block, but shell snippets embedded in individual `SKILL.md` files are not an executed or maintainable hook for automatic notices.
