## What This Changes

Adds a 30-second unref'ed timeout around sandbox media `scpFile` transfers so a hung `scp` child is killed and the staging promise rejects.

## Best Possible Solution

Land a bounded SCP timeout only after real timeout proof and maintainer-visible acceptance or adjustment of the remote-staging timeout policy while preserving the existing SCP validation and staging contracts.

## Reproduction Assessment

Yes, from source inspection: current main only settles `scpFile` on child `error` or `exit`, so a spawned `scp` process that never exits leaves the Promise pending. I did not run a live hanging SCP setup in this read-only review.

## Solution Assessment

Unclear. A bounded timeout is the right layer for the hang, but the best final shape still needs real behavior proof and an owner decision that a hard-coded 30-second transfer cap is acceptable.
