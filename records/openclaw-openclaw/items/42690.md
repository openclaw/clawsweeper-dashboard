## Summary

Keep open: current main still only renders a runtime model row for persisted fallback notice state, so ordinary routed-model drift can leave `session_status` showing the selected/default model without a separate active-model line. The open implementation candidate at https://github.com/openclaw/openclaw/pull/75505 is not merged, so this issue should remain open until that path lands or is replaced.

## What This Changes

The issue requests that `session_status` distinguish the session selected/default model from the routed active model for the current run.

## Best Possible Solution

Land one focused status fix that keeps fallback-specific wording for real fallback notices, adds a general `Active:` line for non-fallback runtime drift, and covers both the shared renderer and `session_status` bridge with regression tests.

## Reproduction Assessment

Yes, source-level. A `buildStatusMessage` input with a selected model and a different active runtime model, without matching persisted fallback notice refs, still renders the selected `Model:` line and no general `Active:` line on current main.

## Solution Assessment

Yes. The requested selected/default-versus-active split is the narrow maintainable fix; mutating the stored session default or replacing the selected line with the active model would lose one of the facts needed for routing diagnostics.
