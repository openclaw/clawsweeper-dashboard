## Summary

Keep open: current `main` has partial selected/active model resolution, but the shared status renderer still only emits a general `Model:` line plus fallback-specific output, so non-fallback runtime routing drift is not shown as an explicit active model line. Open #75505 remains the implementation lane rather than a reason to close this issue before merge.

## What This Changes

The issue requests that `session_status` distinguish the selected/session-default model from the routed active model when they differ.

## Best Possible Solution

Land a focused status-rendering fix that keeps `Model:` as the selected/default identity, emits `Active:` when the runtime model differs without fallback notice state, keeps `Fallback:` for true fallback notices, and covers both shared renderer and `session_status`.

## Reproduction Assessment

Yes. A source-level reproduction is to call the shared status renderer with a selected/default model and a session entry whose runtime `modelProvider`/`model` differ without matching `fallbackNotice*`; current `main` resolves both identities but renders no non-fallback `Active:` line.

## Solution Assessment

Yes. The narrow maintainable solution is a display-layer split in `src/status` and the `session_status` bridge, without changing model selection, routing, fallback persistence, or session defaults.
