## Summary

Keep open: current main still has selected-versus-active model data in the shared status path, but non-fallback runtime drift is not rendered as an explicit active model line, and open #75505 is the active implementation lane rather than a reason to close this issue.

## What This Changes

The issue asks `session_status` to show the selected/session-default model separately from the routed active model when they differ.

## Best Possible Solution

Keep this issue open until a fix lands that preserves `Model:` as the selected/default model, renders a separate `Active:` line for non-fallback runtime drift, preserves `Fallback:` for real fallback notices, and adds focused renderer plus `session_status` coverage.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to call the shared status renderer with a selected/default model and a session entry whose runtime `modelProvider`/`model` differ without matching fallback notice state; current main resolves both identities but has no non-fallback `Active:` render path.

## Solution Assessment

Yes. The narrow maintainable fix is a display-layer split in `src/status` and the `session_status` bridge, without mutating session defaults, routing, fallback persistence, or model selection semantics.
