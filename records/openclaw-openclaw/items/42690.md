## Summary

Current main still does not implement the requested selected-versus-active model split. The status renderer can resolve selected and active identities, but it renders only the selected `Model:` line plus fallback-specific output, and the open implementation PR is still unmerged and conflicting with prior review blockers.

## What This Changes

The issue requests that `session_status` keep the selected/default model visible and add an explicit active-model line when runtime routing uses a different model.

## Best Possible Solution

Land a focused status-rendering fix that preserves `Model:` as the selected/default identity, emits `Active:` for non-fallback runtime drift, keeps `Fallback:` for true fallback notices, and covers both the shared renderer and `session_status` bridge.
