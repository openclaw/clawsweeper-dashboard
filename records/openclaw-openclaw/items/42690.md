## Summary

Keep open: current main still has the selected-versus-active model data in the shared status path, but it only renders `Model:` plus a fallback-specific `Fallback:` line, and the related open #75505 PR is the active implementation lane rather than a reason to close this issue.

## What This Changes

The issue asks for `session_status` to keep the selected/session-default model visible while also rendering a separate active-runtime model line when routing changes the model for the current run.

## Best Possible Solution

Keep this canonical issue open until a fix lands that preserves `Model:` as the selected/default model, renders `Active:` for non-fallback runtime drift, preserves `Fallback:` for real fallback notices, and adds focused renderer plus `session_status` coverage.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to call the shared status renderer with a selected/default model and a session entry whose `modelProvider`/`model` differ without matching fallback notice state; current main has no non-fallback `Active:` render path.

## Solution Assessment

Yes. The narrow maintainable fix is a display-layer split in `src/status` and the `session_status` bridge, without mutating session defaults, routing, fallback persistence, or model selection semantics.
