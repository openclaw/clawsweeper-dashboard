## Summary

Keep open: current main and the latest release still have a source-reproducible selected/default-versus-active model display gap in session_status. The related open PRs are unmerged and currently need maintainer handling, so this issue should remain the canonical tracker rather than close as implemented.

## What This Changes

The issue asks for session_status to distinguish the selected session/default model from the active routed model for the current run.

## Best Possible Solution

Land one focused shared-status fix that preserves the selected/default model line, adds a general Active line when the runtime model differs without fallback notice state, and covers both the renderer and session_status bridge with regression tests.

## Reproduction Assessment

Yes, source-level. A status-renderer input with a selected model and a different runtime active model, without matching fallback notice fields, still renders the selected Model line and no general Active line on current main.

## Solution Assessment

Yes. The requested selected/default-versus-active split is the narrow maintainable fix; mutating the stored session default or replacing the selected line with only the active model would lose useful routing diagnostics.
