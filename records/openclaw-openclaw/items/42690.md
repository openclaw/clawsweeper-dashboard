## Summary

Keep open: current main still does not render a general selected/default-versus-active model split for non-fallback runtime routing, and the related implementation candidate remains open rather than merged.

## What This Changes

Make `session_status` distinguish the selected/session-default model from the active routed runtime model when they differ.

## Best Possible Solution

Land a focused fix in `src/status` and the `session_status` bridge that keeps `Model:` for selected/default identity, emits `Active:` for non-fallback runtime drift, preserves `Fallback:` for real fallback notices, and adds renderer plus bridge regression coverage.

## Reproduction Assessment

Yes, source-level. A `buildStatusMessage` input with selected/default `openai/gpt-4.1-mini` and runtime `anthropic/claude-haiku-4-5` without matching fallback notice renders `Model:` and no `Active:` line on current main.

## Solution Assessment

Yes, the requested selected-versus-active split is the narrow maintainable direction. It should be implemented in the shared status renderer plus `session_status` bridge inputs, not by mutating the session default or overloading fallback state.
