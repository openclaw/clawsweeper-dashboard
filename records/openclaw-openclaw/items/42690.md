## Summary

Keep open: current main has partial active-model plumbing for semantic current requests, but the shared status renderer still emits only `Model:` plus fallback-specific output, and it does not show a non-fallback `Active:` line while preserving the selected/default model. The open implementation candidate at https://github.com/openclaw/openclaw/pull/75505 is still the review lane, not a reason to close this issue before merge.

## What This Changes

Make `session_status` distinguish the selected/default model from the active routed model when runtime routing uses a different model.

## Best Possible Solution

Land a focused status-rendering fix that keeps `Model:` as the selected/default identity, emits `Active:` when the runtime model differs without fallback notice state, preserves `Fallback:` for true fallback notices, and covers both shared renderer and `session_status`.

## Reproduction Assessment

Yes. A source-level reproduction is to call the shared status renderer with a selected/default model and a session entry whose runtime `modelProvider`/`model` differ without matching `fallbackNotice*`; current main resolves the identities but renders no non-fallback `Active:` line.

## Solution Assessment

Yes. The narrow maintainable solution is a display-layer split in `src/status` plus the `session_status` bridge, without changing routing, fallback persistence, or session defaults; current main's active-as-primary path is only partial.
