## Summary

Keep open: current main and the latest release still show only the selected/default model unless persisted fallback notice state matches, so ordinary routed active-model drift can remain invisible in session_status. The related open PRs are unmerged and need maintainer coordination rather than closing this tracker as implemented.

## What This Changes

Request that session_status distinguish the selected/default model from the routed active model when runtime routing changes the active model without mutating the session default.

## Best Possible Solution

Land one focused shared-status fix that preserves the selected/default model line, adds a general active-runtime model line when the active model differs without fallback notice state, and covers the renderer plus session_status bridge with regression tests.

## Reproduction Assessment

Yes, source-level. A status-renderer input with a selected model and a different active runtime model, without matching fallback notice fields, still renders the selected Model line and no general Active line on current main.

## Solution Assessment

Yes for the requested direction. The maintainable fix is to display selected/default and active runtime identities side by side without mutating stored session defaults; the current open PRs need rework or maintainer coordination before they can carry that fix.
