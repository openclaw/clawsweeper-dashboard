## Summary

Keep open: current `main` still has only partial active-model plumbing. The shared status renderer can resolve selected and active model identities, but it renders `Model:` plus fallback-only output and still does not show a non-fallback `Active:` line for routed runtime drift; the related open implementation PR remains the review lane, not a reason to close this issue before merge.

## What This Changes

The issue asks `session_status` to keep the selected/default model visible while also showing the active routed model when the active run uses a different model.

## Best Possible Solution

Land a focused status-rendering fix that preserves `Model:` as the selected/default identity, emits `Active:` for non-fallback runtime drift, preserves `Fallback:` for true fallback notices, and covers both the shared renderer and `session_status` bridge.

## Reproduction Assessment

Yes. A source-level reproduction is to call `buildStatusMessage` with a selected/default model and a session entry whose runtime `modelProvider`/`model` differ without matching `fallbackNotice*`; current `main` resolves the identities but renders no non-fallback `Active:` line.

## Solution Assessment

Yes. The narrow maintainable solution is a display-layer split in `src/status` plus the `session_status` bridge, without changing routing, fallback persistence, or session defaults.
