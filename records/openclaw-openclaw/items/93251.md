## Summary

Keep open: current main has partial Telegram/cron recovery for plain tool warnings, but the remaining cron failure-alert path can still deliver raw provider/runtime error text into Telegram and the final copy policy is security-sensitive.

## What This Changes

The issue asks OpenClaw to stop sending raw automated-run tool and runtime failure details as standalone Telegram chat messages by default.

## Best Possible Solution

Centralize channel-safe automated-run failure policy so recovered tool warnings stay internal, fatal failures use concise redacted actionable summaries, and cron alerts keep raw details in diagnostics rather than chat.

## Reproduction Assessment

Yes, at source level: current tests and source show recovered plain tool warnings are partially handled, while cron failure alerts still preserve raw Last error text for Telegram-targeted delivery. I did not run a live Telegram chat reproduction.

## Solution Assessment

No, closing is not the best path yet; the Feishu-only merged fix and current partial Telegram recovery do not settle channel-safe cron alert copy. The maintainable fix is a central policy with redaction and tests across recovered and fatal automated-run failures.
