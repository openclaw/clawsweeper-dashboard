## Summary

Keep open: current main and the latest release still contain source-backed paths where automated-run tool warnings and cron failure alerts can become user-visible chat text, while the open related PR only covers the exec-warning slice and has not merged.

## What This Changes

The issue reports Telegram direct chats receiving raw internal tool, command, and cron failure messages from scheduled or automated Codex-backed agent runs.

## Best Possible Solution

Keep this issue open, review the exec-warning PR as a partial fix, and land a maintainer-approved channel-safe failure policy that keeps raw details in diagnostics while sending only concise actionable chat copy.

## Reproduction Assessment

Yes, at source level: current payload code/tests still show exec and mutating tool failures becoming visible error payloads, and cron alerts still include Last error text. I did not run a live Telegram reproduction in this read-only review.

## Solution Assessment

No current complete fix is present. The open PR is a plausible partial repair for exec warnings, but the best solution still needs maintainer policy for cron alert thresholds/copy and for which automated failures are diagnostic-only versus user-actionable.
