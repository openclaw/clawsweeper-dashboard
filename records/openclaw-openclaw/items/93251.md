## Summary

Keep open: current main still has source-reproducible paths that can surface raw exec/tool/provider failure text into Telegram-visible delivery, and the open related PR only covers the narrower exec-warning-after-reply branch rather than the broader cron/channel-safe failure policy requested here.

## What This Changes

The issue asks OpenClaw to stop sending raw Telegram-visible tool and cron failure details as standalone chat messages from automated agent runs.

## Best Possible Solution

Keep this issue open as the broader channel-safe failure policy item: raw details should stay in diagnostics/session state, recovered final assistant text should win, and chat notifications should use concise redacted actionable summaries with maintainer-approved cron alert thresholds.

## Reproduction Assessment

Yes, at source level: current main still shows exec failure warnings after assistant replies and still appends raw cron errors to channel-bound failure alerts. I did not run a live Telegram reproduction in this read-only review.

## Solution Assessment

No, closing is not the best way to solve this issue yet. The narrow open PR is useful for one exec-warning branch, but the maintainable fix needs a central channel-safe failure policy for tool warnings, fatal run failures, and cron alerts.
