## Summary

Keep open: current main has a partial recovery path for recovered Telegram cron tool warnings, but source and tests still show fatal Codex/tool errors and cron failure alerts can deliver raw command, provider, or runtime failure text into Telegram.

## What This Changes

## Best Possible Solution

Centralize channel-safe failure policy so raw failure details stay in diagnostics/session state, recovered final assistant text wins, and only concise redacted actionable failure summaries or thresholded alerts reach chat.
