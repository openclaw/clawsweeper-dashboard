## Summary

Keep open: current main partially recovers plain Telegram cron tool-warning output, but cron failure alerts still build user-visible Telegram-targeted text with raw `Last error` details, so the central security and channel-copy problem remains.

## What This Changes

Suppress or redact raw tool/runtime failure details from Telegram automated-agent delivery and keep detailed diagnostics internal or agent-owned.

## Best Possible Solution

Centralize channel-safe automated-run failure policy so recovered tool warnings stay internal, fatal failures use concise redacted actionable summaries, and raw details remain in diagnostics rather than chat.

## Reproduction Assessment

Yes, at source level: current main builds Telegram-targeted cron failure alerts with raw `Last error` text, and tests assert that output. I did not run a live Telegram chat reproduction.

## Solution Assessment

No, closing is not the best path yet; Feishu-only fixes and current recovered-warning handling do not settle channel-safe cron alert copy. The maintainable fix is a central redaction/suppression policy with tests across recovered and fatal automated-run failures.
