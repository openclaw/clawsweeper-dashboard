## Summary

Keep open: current main still has source-backed paths where Telegram-visible automated runs can emit standalone tool/exec failure warnings and cron alerts containing raw command, path, provider, or runtime details. The related Feishu-only work does not close the Telegram/core failure-policy gap.

## What This Changes

Request to stop Telegram automated runs from delivering raw tool, command, provider, or cron failure text as standalone chat messages by default.

## Best Possible Solution

Centralize channel-safe failure delivery so raw details stay in diagnostics/session state, agent-owned final summaries win for recovered runs, and only concise redacted actionable summaries or thresholded cron alerts reach chat.

## Reproduction Assessment

Yes, at source level: current payload and cron alert tests still assert user-visible tool, exec, and cron failure messages. I did not run a live Telegram recording in this read-only review.

## Solution Assessment

No, a Telegram-only sanitizer or one more config flag is not the best fix. The maintainable fix should be a central channel-safe failure policy shared by agent payloads and cron failure notifications.
