## Summary

Keep open: current main and the latest release still have source-backed paths that can turn Codex/native tool failures and cron failure-alert errors into channel-visible text containing command summaries, paths, or provider failure details.

## What This Changes

The issue asks OpenClaw to stop delivering raw Telegram-visible tool and cron failure messages from automated Codex-backed agent runs by default.

## Best Possible Solution

Add one channel-safe failure-notification boundary for automated agent runs: keep raw tool/provider details in diagnostics and run history, deliver final assistant text when available, and use concise redacted actionable summaries for configured chat alerts.

## Reproduction Assessment

Yes, at source level: current main still constructs visible failed-tool payloads and cron failure-alert bodies that can include raw failure text. I did not run a live Telegram replay or inspect the private diagnostics bundle.

## Solution Assessment

No, closing this issue is not the best way to solve it. The maintainable path is a shared failure-notification policy across Codex projection, embedded-run payloads, cron alerts, and channel delivery rather than a Telegram-only string filter.
