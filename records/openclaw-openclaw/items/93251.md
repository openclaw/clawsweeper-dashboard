## Summary

Keep open: current main has partial recovered-tool-warning handling, but cron failure-alert and terminal warning paths can still put raw or near-raw failure text into Telegram, so the security-sensitive channel-copy policy is not fully solved.

## What This Changes

The issue asks OpenClaw to stop delivering raw tool, command, provider, and cron failure details as standalone Telegram messages for automated agent runs.

## Best Possible Solution

Centralize channel-safe automated-run failure policy so recovered tool warnings stay internal, fatal failures use concise redacted actionable summaries, and raw details remain in diagnostics or logs.

## Reproduction Assessment

Yes, at source level: current main still builds Telegram-capable cron alerts from raw `Last error` text, and tests assert that behavior. I did not perform a live Telegram chat reproduction in this read-only review.

## Solution Assessment

No, current main is not the full solution: it partially recovers plain tool-warning payloads, but failure-alert copy still exposes raw error text. The maintainable fix is a central redaction/suppression policy with focused cron and channel-delivery tests.
