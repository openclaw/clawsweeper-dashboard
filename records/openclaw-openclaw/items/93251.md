## Summary

Keep open: current main has partial recovery for successful cron runs, but source still shows standalone/fatal Codex tool errors and cron failure alerts can be delivered with raw command, output, or provider error text, so the report remains actionable and security-sensitive.

## What This Changes

Request to stop Telegram direct chats from receiving raw tool, command, and cron failure text from automated Codex-backed agent runs.

## Best Possible Solution

Keep detailed tool/provider failures in diagnostics and session state, deliver final assistant text when available, and route only concise redacted actionable failure summaries through cron and Telegram channel alerts.

## Reproduction Assessment

Yes, at source level. Current helpers, tests, Codex projector code, and cron failure-alert code show raw tool or failure text can still become a channel-delivery payload; I did not run a live Telegram repro.

## Solution Assessment

No, closing this issue is not the best way to solve it. The maintainable fix is a centralized channel-safe failure policy across cron isolated-agent outcomes and failure alerts, not a broad verbose config flag as the first move.
