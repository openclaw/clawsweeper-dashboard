## Summary

Keep open: current main still has source-level paths that can turn internal tool and cron failures into user-visible channel messages, so the Telegram raw-failure leak is not fixed and needs maintainer policy for redaction and notification defaults.

## What This Changes

The issue requests a default delivery guard so Telegram automated runs do not send raw tool, command, path, or model failure details as standalone chat messages.

## Best Possible Solution

Add a channel-safe failure-notification boundary for automated agent runs: keep detailed tool/provider errors in diagnostics and cron records, deliver final assistant text when available, and send only concise redacted actionable summaries to chat alerts by default.

## Reproduction Assessment

Yes, at source level. Current main still contains tool-warning payload and cron failure-alert paths that can include raw failure text; I did not run a live Telegram Desktop reproduction or inspect the private diagnostics bundle.

## Solution Assessment

No, closing this issue is not the best solution. The maintainable fix should be a central channel-safe failure policy across tool-warning and cron-alert delivery rather than a Telegram-only string scrub.
