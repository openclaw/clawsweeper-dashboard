## Summary

Keep open: current main partially reduces Telegram tool-warning noise after recovered final answers, but both current main and the latest release still build cron failure alerts with raw `Last error` text that can be delivered to Telegram. The remaining work is a security-sensitive channel-copy policy decision, not conservative cleanup to close.

## What This Changes

Issue reports raw Telegram-visible tool and cron failure details leaking from automated Codex-backed agent runs instead of being kept internal or summarized safely.

## Best Possible Solution

Centralize channel-safe automated-run failure policy so recovered tool warnings stay internal, fatal failures use concise redacted actionable summaries, and cron alerts keep raw details in diagnostics rather than chat.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.9 still construct Telegram-targeted cron failure alerts with raw `Last error` text, and current tests assert that behavior. I did not run a live Telegram chat reproduction.

## Solution Assessment

No, closing is not the best way to solve the issue yet; current main only partially mitigates recovered tool warnings and does not settle cron alert copy. The safer solution is a central redaction/suppression policy with regression coverage across recovered and fatal automated-run failures.
