## Summary

Keep open: current main still has source-backed paths that can send compact but user-visible tool-failure metadata and cron last-error text to Telegram; the merged Feishu fix only covers Feishu streaming-card composition, not this Telegram/cron failure-copy policy.

## What This Changes

The issue asks OpenClaw to stop delivering raw or low-level Telegram tool, command, cron, provider, and runtime failure messages from automated Codex-backed agent runs by default.

## Best Possible Solution

Centralize a channel-safe failure policy so raw commands, paths, provider bodies, and runtime diagnostics stay in logs/session state, recovered assistant text wins, and chat receives only concise redacted actionable summaries or thresholded cron alerts.

## Reproduction Assessment

Yes. Current source and tests show user-visible exec/mutating-tool warning payloads and cron failure alerts that include raw last-error text; I did not run a live Telegram chat reproduction in this read-only pass.

## Solution Assessment

No. Current main is not complete for this issue: the Feishu-only landed fix preserves Feishu answers, while Telegram/core payload and cron alert copy still need a channel-safe policy decision.
