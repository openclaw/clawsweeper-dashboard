## Summary

Keep open: current main and the latest release only partially recover Telegram cron output, while source and tests still allow fatal tool/provider errors and cron failure-alert text to reach chat with raw commands, paths, or provider messages.

## What This Changes

The issue asks OpenClaw to stop delivering raw Telegram-visible tool and cron failure details from automated agent runs by default.

## Best Possible Solution

Centralize a channel-safe failure policy so successful recovered runs deliver the agent-owned final text, diagnostics keep raw details internally, and chat only receives concise redacted actionable fatal summaries or thresholded cron alerts.

## Reproduction Assessment

Yes, at source level: current main tests and source still show ordinary exec/fatal/provider failure payloads and cron failure alerts can be delivered into Telegram-visible paths. I did not run a live Telegram chat reproduction.

## Solution Assessment

No, current main is only a partial solution; the maintainable fix should be a central channel-safe failure policy rather than a Telegram-only sanitizer or a broad verbose-mode toggle.
