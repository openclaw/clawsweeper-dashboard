## Summary

Keep open: current main and the latest release still have channel-delivered tool-warning and cron failure-alert paths that can expose raw command, path, and runtime error text, so the central Telegram security/UX problem remains.

## What This Changes

The issue asks OpenClaw to stop delivering raw tool and runtime failure details as standalone Telegram messages from automated Codex-backed agent runs.

## Best Possible Solution

Define a channel-safe failure disclosure policy for automated runs, then update embedded run payloads, cron failure alerts, and Telegram delivery tests so chat receives only actionable summaries while raw command/error details stay in diagnostics or agent-visible state.

## Reproduction Assessment

Yes from source, but not from a live Telegram run in this read-only review. Current code still formats tool metadata and cron failure errors into channel-delivered warning/alert text, while tests show only a partial recovered-warning guard.

## Solution Assessment

Unclear until maintainers set the disclosure policy. The best fix is not blanket suppression; it should preserve actionable user alerts while moving raw tool/runtime detail into diagnostics or agent-visible state.
