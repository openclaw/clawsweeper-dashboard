## Summary

Keep open: current main still has source-backed paths that can turn raw failed-tool and cron failure text into Telegram-visible delivery, and the related direct-chat suppression PR is still open and does not settle the cron alert policy.

## What This Changes

The issue requests suppressing raw Telegram-visible tool/runtime failure messages from automated agent runs and replacing them with internal diagnostics or concise safe summaries.

## Best Possible Solution

Keep this open until maintainers land a security-reviewed failure projection policy that keeps raw details internal, preserves actionable terminal failures, and uses redacted summaries for Telegram and cron alerts.
