## Summary

Keep open: current main has partial recovery for recovered cron tool warnings, but still intentionally emits visible exec/mutating-tool warnings, fatal cron error payloads, and cron failure-alert text that can include raw command, provider, path, or runtime details.

## What This Changes

The issue requests that Telegram delivery stop exposing raw tool, command, cron, and runtime failure details from automated Codex-backed agent runs as standalone chat messages.

## Best Possible Solution

Adopt one central channel-safe failure policy for agent and cron delivery that keeps raw diagnostics in logs/session state, sends only bounded actionable summaries to chat, and proves the Telegram path with focused tests plus real Telegram proof.

## Reproduction Assessment

Yes, by source inspection: current main still has tests and code paths that produce visible exec/tool/run failure payloads and Telegram-routed cron failure alerts. I did not run a live Telegram reproduction in this read-only review.

## Solution Assessment

Unclear as a final implementation: the requested direction is right, but the best fix needs maintainer product/security choice for which fatal or mutating failures remain chat-visible and how their fallback text is sanitized.
