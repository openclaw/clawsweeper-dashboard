## Summary

Keep open: current main and the latest release still rewrite the GA `gemini-3.1-flash-lite` model to `gemini-3.1-flash-lite-preview` across the Google manifest, helper, SDK normalizer, default alias, docs, and tests, while Google's deprecation table now recommends the GA ID. The open fix PR at https://github.com/openclaw/openclaw/pull/85276 is a candidate path but is not merged and was reviewed as incomplete.

## What This Changes

Request to stop OpenClaw from automatically using `gemini-3.1-flash-lite-preview` when users configure the GA `gemini-3.1-flash-lite` model.

## Best Possible Solution

Ship a focused Google provider/model lifecycle fix that makes `gemini-3.1-flash-lite` canonical across manifest, helper, SDK, defaults, docs, and tests while preserving any intentional preview-ID compatibility.

Yes. Source inspection on current main shows the manifest, helper, SDK normalizer, default alias, docs, and tests still rewrite or expect `gemini-3.1-flash-lite-preview`; I did not run the live CLI because this review is read-only.
