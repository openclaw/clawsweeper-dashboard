## Summary

Keep open: current main and the latest release still rewrite the GA `gemini-3.1-flash-lite` model to the preview ID, while Google lists `gemini-3.1-flash-lite-preview` as shutting down on May 25, 2026 with `gemini-3.1-flash-lite` as the replacement. A focused open PR already targets this exact provider/model lifecycle update, so this issue should stay open until that fix lands. ([ai.google.dev](https://ai.google.dev/gemini-api/docs/deprecations))

## What This Changes

The issue requests that OpenClaw stop routing configured `gemini-3.1-flash-lite` Google models to `gemini-3.1-flash-lite-preview`.

## Best Possible Solution

Land or otherwise implement the focused Google provider lifecycle update so `gemini-3.1-flash-lite` is canonical and the preview ID remains only a compatibility alias.

## Reproduction Assessment

Yes. Source inspection on current main shows multiple normalization and alias paths still rewriting `gemini-3.1-flash-lite` to `gemini-3.1-flash-lite-preview`, and existing tests assert that behavior.

## Solution Assessment

Yes. The narrow maintainable fix is to update the Google-owned normalization, manifest/default aliases, docs, and tests without adding a new config option or moving provider policy into core.
