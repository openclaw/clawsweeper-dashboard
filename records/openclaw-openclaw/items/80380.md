## Summary

Keep open: current main and the latest release still rewrite the requested bare Gemini Flash-Lite model to the preview ID. A related open PR now tracks the focused fix, so the issue should stay open until that PR lands or is replaced.

## What This Changes

Request to stop rewriting `google/gemini-3.1-flash-lite` to `gemini-3.1-flash-lite-preview` after the GA model release and preview deprecation.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81831, or an equivalent focused patch, while keeping preview IDs accepted as compatibility aliases to the bare GA ID.

## Reproduction Assessment

Yes. Source inspection on current main shows multiple model-normalization paths rewrite `gemini-3.1-flash-lite` to the preview ID, and a commenter reports the same behavior from a real `openclaw infer model run` on v2026.4.23.

## Solution Assessment

Yes. The narrow maintainable fix is to update the Google-owned normalizers, manifest/default aliases, provider-template fallback, Gemini CLI shorthand, docs, and focused tests; closing this issue before the open fix PR merges would be premature.
