## Summary

Keep open: current `main` and the latest shipped tag still rewrite configured `gemini-3.1-flash-lite` to `gemini-3.1-flash-lite-preview`, while Google’s deprecation table lists the bare GA model as the replacement. The focused open implementation candidate is https://github.com/openclaw/openclaw/pull/81831, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

Update OpenClaw’s Google model normalization and aliases so configured `gemini-3.1-flash-lite` resolves to the GA ID instead of `gemini-3.1-flash-lite-preview`.

## Best Possible Solution

Land the focused Google provider and CLI alias update in https://github.com/openclaw/openclaw/pull/81831 or an equivalent patch, preserving preview-ID compatibility aliases and updating docs/tests together.

## Reproduction Assessment

Yes. Source inspection on current main shows the Google normalizers, manifest aliases, Gemini CLI shorthand, and existing tests all route bare `gemini-3.1-flash-lite` to the preview ID; I did not run runtime commands in this read-only sweep.

## Solution Assessment

Yes. The narrow maintainable fix is to update the Google plugin-owned model normalization, manifest/default aliases, Gemini CLI shorthand, docs, and focused tests without adding a new config option or moving provider policy into core.
