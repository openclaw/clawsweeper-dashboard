## Summary

Keep open: current main still rewrites the requested GA Google Flash-Lite model to the preview id, and the matching fix is already being reviewed in https://github.com/openclaw/openclaw/pull/81831. This issue should close after that PR or an equivalent fix lands, not before.

## What This Changes

The issue asks OpenClaw to treat `gemini-3.1-flash-lite` as the canonical Google Flash-Lite model instead of rewriting it to `gemini-3.1-flash-lite-preview`.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/81831 or an equivalent focused Google provider update that makes the GA id canonical, keeps preview ids as compatibility aliases, and updates docs/tests together.

## Solution Assessment

Yes. The narrow maintainable solution is to update the Google plugin-owned normalization, default aliases, Gemini CLI shorthand, docs, and focused tests while preserving compatibility for explicit preview ids.
