## Summary

Keep open: current main and v2026.5.20 still rewrite GA `gemini-3.1-flash-lite` to deprecated `gemini-3.1-flash-lite-preview` across Google runtime normalization, SDK normalization, manifest aliases, default aliases, docs, and tests. ## What This Changes

Request that OpenClaw preserve or canonicalize GA `gemini-3.1-flash-lite` instead of automatically rewriting it to `gemini-3.1-flash-lite-preview`.

## Best Possible Solution

Ship a focused Google provider/model lifecycle update that treats `gemini-3.1-flash-lite` as the canonical Flash-Lite ID, keeps explicit preview compatibility or retirement handling where appropriate, and updates docs/tests with the same provider-scope decision.
