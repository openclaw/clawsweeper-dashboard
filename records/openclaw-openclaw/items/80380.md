## Summary

Keep open: current main and the latest shipped release still rewrite the GA `gemini-3.1-flash-lite` model ID to `gemini-3.1-flash-lite-preview`, while Google now lists the GA ID as the replacement for the preview endpoint. The related report at https://github.com/openclaw/openclaw/issues/81556 was already closed as a duplicate of this issue, so this remains the canonical tracker.

## What This Changes

Update OpenClaw’s Google model handling so `gemini-3.1-flash-lite` is not automatically routed to `gemini-3.1-flash-lite-preview`.

## Best Possible Solution

Land one focused Google provider model-lifecycle update that treats `gemini-3.1-flash-lite` as the canonical Flash-Lite ID, keeps any necessary compatibility handling for explicit preview IDs, and updates docs/tests/changelog together.

## Reproduction Assessment

Yes. Source inspection on current main shows the GA model ID is rewritten to the preview model ID in the Google helper, SDK normalizer, manifest aliases, and default alias surfaces; existing tests assert the preview result. I did not run tests because this review is read-only.

## Solution Assessment

Yes. The narrow maintainable fix is to update the existing Google provider/model normalization and alias surfaces plus their tests/docs, without adding a config option or moving provider policy out of the Google-owned paths.
