## Summary

Keep open: current main and the latest shipped release still rewrite configured Gemini 3.1 Flash-Lite GA IDs to the preview ID, while Google's Gemini API release notes say `gemini-3.1-flash-lite` is GA and `gemini-3.1-flash-lite-preview` is shutting down on May 25, 2026. ([ai.google.dev](https://ai.google.dev/gemini-api/docs/changelog?hl=en))

## What This Changes

Request to stop OpenClaw from automatically rewriting `gemini-3.1-flash-lite` to `gemini-3.1-flash-lite-preview`.

## Best Possible Solution

Ship a focused Google provider lifecycle update that treats `gemini-3.1-flash-lite` as the canonical Flash-Lite ID, keeps only intentional compatibility paths, and updates aliases, docs, and regression tests together.

## Reproduction Assessment

Yes. Source inspection shows current main rewrites `gemini-3.1-flash-lite` to `gemini-3.1-flash-lite-preview` in multiple routing surfaces, and tests currently assert that behavior; I did not run a live Google call in this read-only review.

## Solution Assessment

Yes. The maintainable fix is a narrow Google model ID lifecycle update in the provider-owned normalization/catalog/default-alias surfaces, not a new config option or broader product decision.
