## Summary

Keep open: current main and v2026.5.18 still rewrite the configured GA `gemini-3.1-flash-lite` model to the deprecated preview ID across Google normalization/default-alias surfaces, while Google's docs list the preview shutdown on May 25, 2026. An open focused fix PR exists, but it has not merged, so this issue should stay open until that or an equivalent fix lands.

## What This Changes

Request to stop OpenClaw from automatically rewriting Google Gemini 3.1 Flash-Lite GA model IDs to `gemini-3.1-flash-lite-preview`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81831 or an equivalent focused Google provider update, then close this issue after the fix is merged and released.

## Reproduction Assessment

Yes, at source level. Current main maps `gemini-3.1-flash-lite` to `gemini-3.1-flash-lite-preview` in the Google/plugin-sdk normalizers and tests assert that rewrite; I did not run a live Google API call in this read-only review.

## Solution Assessment

Yes. The narrow maintainable solution is to update the Google-owned normalization, manifest/default alias, docs, CLI shorthand, and tests so the GA ID is canonical while preview IDs remain compatibility aliases; current main should not be closed as fixed before that lands.
