## Summary

Keep open: current main still rewrites the requested stable `gemini-3.1-flash-lite` model to `gemini-3.1-flash-lite-preview`, and Google's deprecation table now names the bare GA id as the replacement. An open implementation PR, https://github.com/openclaw/openclaw/pull/81831, already targets this exact fix, so the issue should stay open until that PR lands or is superseded.

## What This Changes

Request to stop normalizing Gemini 3.1 Flash-Lite stable model IDs to the preview model and update the related Google aliases, defaults, docs, and tests.

## Best Possible Solution

Review and land, or intentionally supersede, https://github.com/openclaw/openclaw/pull/81831 so the Google plugin-owned normalizers, manifest aliases, default alias docs, Gemini CLI shorthand, and tests consistently treat `gemini-3.1-flash-lite` as canonical while preserving preview-id compatibility.

## Reproduction Assessment

Yes. Source inspection on current main shows the bare `gemini-3.1-flash-lite` id is rewritten to `gemini-3.1-flash-lite-preview` in multiple runtime/config paths, and existing tests assert the preview result.

## Solution Assessment

Yes. The narrow maintainable solution is the Google provider/model lifecycle update already proposed in https://github.com/openclaw/openclaw/pull/81831, not a new config option or a core policy change.
