## What This Changes

This PR changes the shared scripts bounded-response reader to treat unsafe decimal Content-Length values as over-limit in both TS and MJS exports and adds TS/MJS regression coverage.

## Best Possible Solution

Land the narrow scripts-reader hardening after the draft is ready and exact-head validation covers the bounded-response regression.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes from source and the PR body: on current main an unsafe decimal header parses to undefined and falls into reader.read(), so a never-completing body can hang instead of failing before read; I did not execute it in this read-only review.

## Solution Assessment

Is this the best way to solve the issue? Yes: the patch keeps the fix local to the shared scripts reader, reuses the existing over-limit cancellation branch, preserves the non-decimal streaming path, and matches the sibling media-core unsafe-length invariant.
