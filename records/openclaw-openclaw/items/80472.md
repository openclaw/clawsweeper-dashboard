## Summary

Close as duplicate/superseded: the same docs fix for the linked issue is already tracked by the earlier open PR at https://github.com/openclaw/openclaw/pull/70884, and this branch adds conflicting protocol and trusted-proxy auth guidance.

## What This Changes

The PR updates gateway protocol and trusted-proxy auth docs to show `minProtocol: 4`, describe device-less trusted-proxy WebSocket scope reduction, mark `x-openclaw-scopes` as HTTP-only, and add missing-scope troubleshooting.

## Best Possible Solution

Keep one canonical docs path open, preferably https://github.com/openclaw/openclaw/pull/70884 or a maintainer replacement, and make it match the current gateway protocol and trusted-proxy WebSocket scope contract.

## Reproduction Assessment

Yes, source-reproducible: current main protocol constants, client call sites, WebSocket negotiation, and trusted-proxy scope tests show the PR's changed statements conflict with the implemented contract. I did not run a live gateway because this was a read-only review.

## Solution Assessment

No. A docs-only correction is the right surface, but this branch duplicates the already-open canonical PR and its current wording conflicts with current source and tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P2] Keep gateway examples on the v3 lower bound:** `docs/gateway/protocol.md:47`
  - body: Current main exports `MIN_CLIENT_PROTOCOL_VERSION = 3`, native clients send `min=3/max=4`, and the server accepts ranges containing protocol 4. Changing examples to `minProtocol: 4` and saying the server rejects mismatches tells clients to drop supported compatibility.
  - confidence: 0.96
- **[P2] Do not mark WebSocket scope headers as HTTP-only:** `docs/gateway/trusted-proxy-auth.md:326-329`
  - body: The WebSocket connect handler reads `x-openclaw-scopes` from the upgrade request and uses it to bound trusted-proxy Control UI scopes. Documenting it as HTTP-only would mislead operator auth troubleshooting.
  - confidence: 0.94
- **[P2] Narrow the reduced-scope guidance to real clearing paths:** `docs/gateway/trusted-proxy-auth.md:67`
  - body: Successful trusted-proxy Control UI auth is passed as `trustedProxyAuthOk` and excluded from missing-device scope clearing. A blanket reduced read/write default for device-less trusted-proxy sessions does not match current behavior.
  - confidence: 0.9
