## Best Possible Solution

Use a policy-aware fallback that acknowledges only non-visible tool-only completion, avoids duplicate side-effect/source delivery, preserves the `message_tool_only` contract unless maintainers explicitly change it, and includes focused tests plus real Telegram proof.
