## Summary

Keep open: the linked Telegram silent-delivery bug is still source-reproducible on current main, but the PR’s `toolSummary.calls > 0` guard is too broad and can send an extra `Done.` after message tools have already delivered content; it also lacks required real behavior proof.

## What This Changes

The PR changes `src/auto-reply/reply/agent-runner.ts` to return `Done.` when an agent run has no reply payloads but reports one or more tool calls.

## Best Possible Solution

A maintainable fix should acknowledge local side-effect-only turns only when no message-tool, block-stream, or other outbound delivery already occurred, add regression coverage for both acknowledgement and non-duplication cases, and include redacted Telegram/Gateway proof.

## Reproduction Assessment

Yes, source-level: current main drains queued followups and returns `undefined` when `payloadArray.length === 0`, which matches the linked Telegram no-message report. I did not run a live Telegram/Gateway reproduction in this read-only review.

## Solution Assessment

No: `toolSummary.calls > 0` is not the narrowest safe condition because message-send tools are also tool calls and already carry delivery evidence. The safer fix is to gate the acknowledgement on the absence of committed message-tool or other outbound delivery evidence, or route it through the existing payload-building semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Skip the fallback after message-tool deliveries:** `src/auto-reply/reply/agent-runner.ts:1461-1465`
  - body: `hadToolCalls` is true for message tools as well as local side-effect tools. In this empty-payload branch the new return sends `Done.` before `buildReplyPayloads` can inspect `messagingToolSentTexts`, media URLs, or targets, so a run that already delivered content through a message tool can get an extra source-channel reply. Gate the fallback on no committed message-tool delivery evidence, or route the acknowledgement through the existing delivery path.
