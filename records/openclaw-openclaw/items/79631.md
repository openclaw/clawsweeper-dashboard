## Summary

Keep open: this PR targets a real source-level silent-delivery path, but the patch is too broad and would send an extra source reply after message-tool deliveries; it also lacks required after-fix real Telegram/Gateway proof.

## What This Changes

The PR adds a `toolSummary.calls > 0` check in `runReplyAgent` and returns `Done.` when the agent run has no reply payloads.

## Best Possible Solution

A maintainable fix should acknowledge only successful local side-effect turns with no other outbound delivery, preserve message-tool and failure suppression evidence, add focused regression coverage, and include real Telegram/Gateway proof.

## Reproduction Assessment

Yes, source-level: current main returns `undefined` for an empty `payloadArray`, matching the linked Telegram no-message report. I did not establish a live Telegram/Gateway reproduction in this read-only review.

## Solution Assessment

No: `toolSummary.calls > 0` is too broad because it also covers message-tool sends and failed/suppressed tool paths. The safer fix is to gate the acknowledgement on successful local side effects and absence of committed outbound delivery evidence.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Skip the fallback after message-tool deliveries:** `src/auto-reply/reply/agent-runner.ts:1461-1465`
  - body: `hadToolCalls` is true for message-send tools too, and this branch returns before message-tool delivery evidence can be considered. A run that already sent its user-visible output through the message tool but has no final payload would get an extra `Done.` source reply; gate the fallback on no committed message-tool or other outbound delivery evidence.
  - confidence: 0.87
- **[P2] Do not acknowledge failed tool-only turns as done:** `src/auto-reply/reply/agent-runner.ts:1461-1465`
  - body: `toolSummary.calls` counts tool calls, not successful local side effects. Current payload behavior can intentionally produce no reply for recoverable or suppressed tool errors, and this new branch would turn those empty failed runs into a success-looking `Done.` acknowledgement; include failure/success evidence in the guard.
  - confidence: 0.8
