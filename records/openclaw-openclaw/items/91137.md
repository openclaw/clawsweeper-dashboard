## Summary

Keep open: this is a focused regression fix for the shared channel message tool, and source plus TypeBox contract checks support fixing the schema merge seam rather than chasing each plugin. The only material pre-merge question is accepting the plugin API invariant that contributed message-tool properties cannot become object-level required fields.

## What This Changes

The PR wraps channel `describeMessageTool` schema contributions as TypeBox optional properties before assembling the shared `message` tool schema and adds regression coverage for markerless or cloned contributions.

## Best Possible Solution

Land the centralized optional-contribution invariant once maintainers accept the plugin API semantics and checks pass; avoid making individual plugin optionality patches the permanent fix.
