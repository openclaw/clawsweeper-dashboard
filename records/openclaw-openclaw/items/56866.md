Full review comments:

- **[P1] Preserve plugin-owned ACP binding channels:** `src/config/zod-schema.agents.ts:74-84`
  - body: This hard-coded channel allowlist would reject any third-party or future channel that supports the configured-binding provider contract. Current main only requires a concrete `match.peer.id`, so keep channel support delegated to the plugin binding compiler instead of baking a core allowlist here.
  - confidence: 0.91
- **[P1] Use the canonical media attachment reader:** `src/auto-reply/reply/dispatch-acp.ts:116-144`
  - body: The new ACP media path reads validate the raw path and final component before `fs.readFile`, but a symlinked directory under an allowed root can still point outside the root. Reuse the current `resolveAgentTurnAttachments`/`MediaAttachmentCache` path, which opens files safely and rechecks canonical real paths before reading.
  - confidence: 0.93
- **[P1] Do not unbind shared sessions on generic ACP failures:** `src/auto-reply/reply/dispatch-acp.ts:593-600`
  - body: On every `ACP_TURN_FAILED` or `ACP_SESSION_INIT_FAILED`, this removes bindings by `targetSessionKey`; that operation removes all conversations bound to the same ACP session, so one failing inbound turn can disconnect unrelated bound chats. Limit cleanup to confirmed stale-session metadata failures or the current conversation binding only.
  - confidence: 0.9
- **[P1] Use the resolved ACP session key for binding cleanup:** `src/auto-reply/reply/dispatch-acp.ts:398-400`
  - body: `resolveSession()` returns the canonical session key, but the branch still checks and unbinds using the raw inbound key in several binding calls. If a user routes through an alias such as `main`, binding lookup/cleanup can miss the canonical binding and leave stale routes behind.
  - confidence: 0.86
- **[P2] Redact local media paths from ACP prompts:** `src/auto-reply/reply/inbound-meta.ts:253-255`
  - body: Embedding `entry.mediaPath` into the prompt sends host filesystem paths to the ACP backend and hook handlers. Keep media paths in the attachment channel and use a redacted media hint in prompt text.
  - confidence: 0.88
