## Summary

Keep open: the WhatsApp ACP feature is real and not implemented on current main, but the PR still introduces source-backed security and availability regressions and lacks required real WhatsApp ACP behavior proof.

## What This Changes

The PR wires WhatsApp configured ACP session routing, media/history threading into ACP prompts, ACP lifecycle hook dispatch, WhatsApp block-delivery visibility handling, and WhatsApp ACP binding schema validation.

## Best Possible Solution

Rework the feature on current main by using the hardened attachment cache, keeping ACP binding cleanup scoped to confirmed stale sessions or the current binding, avoiding raw host paths in prompts, and then posting real WhatsApp ACP proof.

## Reproduction Assessment

Yes. Source inspection gives high-confidence reproduction paths for the introduced defects: an allowed-root path through an intermediate symlink reaches the PR's fs.readFile path, and ACP_TURN_FAILED unbinds by targetSessionKey across matching bindings.

## Solution Assessment

No. The feature direction is useful, but this patch bypasses current main's hardened attachment cache and broadens failure cleanup too much; a smaller current-main rework with explicit WhatsApp binding/media seams is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Recheck real paths before reading ACP attachments:** `src/auto-reply/reply/dispatch-acp.ts:117-139`
  - body: The PR validates the raw path and rejects only a final symlink before fs.readFile. A path under an allowed root that crosses an intermediate symlink can still resolve outside the root and be attached to the ACP turn; reuse MediaAttachmentCache or compare canonical file and root realpaths before opening the file.
  - confidence: 0.93
- **[P1] Limit ACP failure unbinding to stale bindings:** `src/auto-reply/reply/dispatch-acp.ts:595-600`
  - body: The catch path unbinds by targetSessionKey for every ACP_TURN_FAILED. The binding service removes all bindings with that target key, so one failed inbound turn can detach other conversations sharing the same ACP session; keep cleanup to confirmed stale-session errors or unbind only the current binding id.
  - confidence: 0.91
- **[P2] Remove local media paths from prompt history:** `src/auto-reply/reply/inbound-meta.ts:255`
  - body: This writes the host mediaPath into the chat-history prompt sent to ACP harnesses and prompt-build hooks. Since image bytes should be attached separately, expose an opaque placeholder or media reference instead of leaking local filesystem layout.
  - confidence: 0.82
