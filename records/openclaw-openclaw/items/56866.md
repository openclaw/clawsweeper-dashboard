## Summary

Keep open: current main still lacks the WhatsApp configured ACP inbound routing/media work, but this branch is not safe to merge as-is because it introduces security-sensitive attachment reading, broad binding cleanup, host-path disclosure, merge conflicts, and no real behavior proof.

## What This Changes

The PR wires WhatsApp messages to configured ACP bindings, threads WhatsApp media/history into ACP prompts, adds ACP hook dispatch, treats WhatsApp ACP block replies as visible, and expands related schema/tests.

## Best Possible Solution

Rework this on current main as a smaller design: land WhatsApp binding-provider/inbound routing deliberately, use MediaAttachmentCache plus opaque media references for media, and require redacted live WhatsApp ACP proof.

## Reproduction Assessment

Yes. Source inspection of the PR head gives a high-confidence path for the review findings: raw path validation before fs.readFile, target-session unbind on generic ACP errors, and mediaPath serialization into prompt text.

## Solution Assessment

No. The feature direction is useful, but this patch bypasses current main's hardened media cache, broadens cleanup semantics, and mixes routing, hooks, media, and schema changes in a way that should be split or rebased.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Recheck real paths before reading ACP attachments:** `src/auto-reply/reply/dispatch-acp.ts:117-139`
  - body: The PR checks only the raw path with isInboundPathAllowed and rejects only the final symlink before fs.readFile. A path under an allowed root that crosses an intermediate symlink can still attach a file outside configured roots; reuse MediaAttachmentCache or compare canonical file/root realpaths before opening.
  - confidence: 0.93
- **[P1] Limit ACP failure unbinding to stale bindings:** `src/auto-reply/reply/dispatch-acp.ts:595-600`
  - body: The catch path unbinds by targetSessionKey for every ACP_SESSION_INIT_FAILED or ACP_TURN_FAILED. The generic binding store removes every binding for that target session, so one failed inbound turn can detach other conversations sharing the ACP session; limit this to confirmed stale sessions or the current binding id.
  - confidence: 0.91
- **[P2] Remove local media paths from prompt history:** `src/auto-reply/reply/inbound-meta.ts:255`
  - body: This writes host mediaPath values into the chat-history prompt sent to ACP harnesses and prompt-build hooks. Image bytes should travel through the attachment channel; prompt history should use an opaque media reference or redacted media hint instead of exposing local filesystem layout.
  - confidence: 0.82
