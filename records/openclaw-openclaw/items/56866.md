## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Recheck ACP attachment paths after canonicalization:** `src/auto-reply/reply/dispatch-acp.ts:117-139`
  - body: `isInboundPathAllowed` checks the raw string path, then `fs.lstat`/`readFile` can still follow symlinked parent directories. A path under an allowed root that crosses a symlinked directory can read outside the root and send that file as base64 to ACP; reuse current main's `MediaAttachmentCache`/safe-open path before reading.
  - confidence: 0.93
- **[P1] Do not unbind shared sessions on generic turn failures:** `src/auto-reply/reply/dispatch-acp.ts:595-600`
  - body: The catch path unbinds by `targetSessionKey` for any `ACP_TURN_FAILED`, which can remove every conversation bound to that target session after one failed inbound turn. Restrict cleanup to confirmed stale/init failures and the current binding instead of session-wide generic failures.
  - confidence: 0.91
- **[P1] Run ACP turns with the canonical session key:** `src/auto-reply/reply/dispatch-acp.ts:514-517`
  - body: `resolveSession()` can return a canonical session key, but this branch passes the raw inbound `sessionKey` into `runTurn` and cleanup. Alias-bound conversations can miss the canonical transcript/progress/unbind path and keep failing under the original binding.
  - confidence: 0.88
- **[P2] Keep host media paths out of prompts and hooks:** `src/auto-reply/reply/inbound-meta.ts:253-256`
  - body: Replacing `<media:image>` with a literal `entry.mediaPath` sends local filesystem paths to ACP models and prompt lifecycle hooks. The attachment can be included without revealing host paths; use a stable redacted media hint instead.
  - confidence: 0.9
