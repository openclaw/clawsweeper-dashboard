## Summary

Keep this PR open: current main still does not wire WhatsApp inbound messages through configured ACP bindings, so the branch contains unique feature work, but the current patch has source-reproducible security/correctness blockers and lacks required real WhatsApp ACP proof.

## What This Changes

The PR adds WhatsApp configured ACP routing, media/history threading into ACP prompts, ACP lifecycle hook dispatch, block-delivery visibility handling, and WhatsApp ACP binding schema validation.

## Best Possible Solution

Rework the feature on current main using the existing configured-binding/runtime seams, MediaAttachmentCache/canonical roots, scoped stale-binding cleanup, opaque media references, and redacted real WhatsApp ACP proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a path under an allowed root that traverses an intermediate symlink reaches fs.readFile, generic ACP_TURN_FAILED unbinds by targetSessionKey, and mediaPath is serialized directly into prompt history.

## Solution Assessment

No. The feature direction is useful, but the current patch bypasses current main's hardened attachment cache, broadens failure cleanup, and exposes host paths; a smaller rebase onto current main is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Recheck real paths before reading ACP attachments:** `src/auto-reply/reply/dispatch-acp.ts:117-139`
  - body: The PR validates the raw attachment path and rejects only a final symlink before fs.readFile. A path under an allowed root that crosses an intermediate symlink can still resolve outside the root and be attached to the ACP turn; reuse MediaAttachmentCache or compare canonical file and root realpaths before opening the file.
  - confidence: 0.93
- **[P1] Limit ACP failure unbinding to stale bindings:** `src/auto-reply/reply/dispatch-acp.ts:595-600`
  - body: The catch path unbinds by targetSessionKey for every ACP_SESSION_INIT_FAILED or ACP_TURN_FAILED. The binding service removes all bindings with that target key, so one failed inbound turn can detach other conversations sharing the same ACP session; keep cleanup to confirmed stale-session errors or unbind only the current binding id.
  - confidence: 0.91
- **[P2] Remove local media paths from prompt history:** `src/auto-reply/reply/inbound-meta.ts:253-255`
  - body: This writes host mediaPath values into the chat-history prompt sent to ACP harnesses and prompt-build hooks. Since image bytes should be attached separately, expose an opaque placeholder or media reference instead of leaking local filesystem layout.
  - confidence: 0.82

## Security Review
