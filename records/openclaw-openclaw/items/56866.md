## Summary

Keep this PR open because the requested WhatsApp ACP binding capability is still absent from current main, but the branch is not safe to merge as-is: it reintroduces direct ACP file reads without current main's canonical containment checks and broadens ACP failure cleanup to session-wide unbinding.

## What This Changes

The PR adds WhatsApp configured ACP binding resolution, WhatsApp media/history threading into ACP turns, ACP prompt lifecycle hooks, WhatsApp ACP block-delivery visibility handling, and WhatsApp ACP binding schema validation.

## Best Possible Solution

Rework the WhatsApp ACP feature on current main: keep the WhatsApp configured-binding seam in the plugin/on-message path, reuse dispatch-acp-attachments.ts and MediaAttachmentCache for all media bytes, keep prompts free of host file paths, and limit binding cleanup to confirmed stale sessions or the current binding record.

## Reproduction Assessment

Yes. The blocking issues have high-confidence static reproduction paths: an allowed-root path through an intermediate symlink reaches fs.readFile without canonical containment, and an ACP_TURN_FAILED path unbinds by targetSessionKey, which removes all bindings for that target.

## Solution Assessment

No. The feature direction is useful, but this implementation is not the best fix because it bypasses current main's hardened media cache and expands ACP cleanup too broadly; the safer path is a smaller rebase that adds WhatsApp binding/media seams while preserving shared ACP security contracts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Recheck real paths before reading ACP attachments:** `src/auto-reply/reply/dispatch-acp.ts:117-139`
  - body: `isInboundPathAllowed` checks the raw path and `lstat` rejects only the final path component before `fs.readFile`. A path under an allowed root that crosses an intermediate symlink can still resolve outside the root and be attached to the ACP turn. Reuse `MediaAttachmentCache` or canonicalize both file and roots with `realpath` before opening the file.
  - confidence: 0.93
- **[P1] Limit ACP failure unbinding to stale bindings:** `src/auto-reply/reply/dispatch-acp.ts:595-600`
  - body: The catch path unbinds by `targetSessionKey` for every `ACP_TURN_FAILED`. The binding service removes all bindings with that target key, so one failed inbound turn can detach other conversations that intentionally share the same ACP session. Keep the stale-init guard or resolve and unbind only the current conversation's binding id.
  - confidence: 0.91
- **[P2] Remove local media paths from prompt history:** `src/auto-reply/reply/inbound-meta.ts:249-255`
  - body: This writes the host `mediaPath` into the untrusted chat-history prompt sent to ACP harnesses and plugin hooks. Since the image bytes are attached separately, expose an opaque media reference or keep the placeholder instead of leaking local filesystem layout.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The PR introduces concrete security and availability regressions in ACP local-file reads and session-binding cleanup.

Concerns:

- **[high] ACP attachment path traversal via intermediate symlinks:** `src/auto-reply/reply/dispatch-acp.ts:139`
  - body: The branch validates the raw path and rejects only final symlinks before `fs.readFile`, so an allowed-root path through a symlinked directory can read files outside the allowed roots.
  - confidence: 0.93
- **[high] Generic ACP failures can unbind shared conversations:** `src/auto-reply/reply/dispatch-acp.ts:599`
  - body: Unbinding by `targetSessionKey` on `ACP_TURN_FAILED` removes every binding for that session key, making a single failed message capable of detaching unrelated conversations that share the ACP session.
  - confidence: 0.91
- **[medium] Local media paths leak into ACP prompts:** `src/auto-reply/reply/inbound-meta.ts:255`
  - body: The history prompt includes local `mediaPath` values, exposing host filesystem details to the ACP harness and prompt-build hooks when an opaque placeholder would be enough.
  - confidence: 0.82
