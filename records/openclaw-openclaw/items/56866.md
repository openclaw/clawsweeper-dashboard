- **[P1] Recheck ACP attachment paths after canonicalization:** `src/auto-reply/reply/dispatch-acp.ts:117-139`
  - body: This validates only the raw path string before `fs.readFile`; `lstat()` rejects a final symlink but still allows symlinked intermediate directories under an allowed root to escape the root. Current main already routes ACP attachments through `MediaAttachmentCache`, which safely opens the file, canonicalizes it, and rechecks allowed roots, so this branch should reuse that path before sending base64 data to ACP.
  - confidence: 0.93
- **[P1] Do not unbind shared sessions on generic turn failures:** `src/auto-reply/reply/dispatch-acp.ts:595-600`
  - body: `ACP_TURN_FAILED` is the fallback for many runtime exceptions, but this cleanup removes bindings by `targetSessionKey`, which deletes every conversation bound to that session. One failed inbound WhatsApp turn can detach unrelated conversations that share the ACP session; cleanup should be scoped to the current binding or restricted to confirmed stale-session cases.
  - confidence: 0.91
- **[P1] Keep using the resolved canonical ACP session key:** `src/auto-reply/reply/dispatch-acp.ts:516-517`
  - body: `resolveSession()` can canonicalize aliases, but this branch passes the raw inbound `sessionKey` into `runTurn` and cleanup paths. Current main carries `canonicalSessionKey` forward so aliases map to the existing ACP session and binding records; dropping that can miss stale bindings or run against the wrong key.
  - confidence: 0.86
- **[P2] Remove local filesystem paths from prompt history:** `src/auto-reply/reply/inbound-meta.ts:253-256`
  - body: The prompt history now includes `entry.mediaPath` verbatim, so ACP providers and prompt hooks receive host directory details such as user names, state paths, or mount layout. Keep file paths internal to the attachment channel and render only a non-sensitive media hint or attachment reference in prompt text.
  - confidence: 0.9
