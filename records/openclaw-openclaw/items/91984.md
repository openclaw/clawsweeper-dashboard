## What This Changes

## Best Possible Solution

Keep the Telegram plugin-owned local Bot API mapping, but preserve local read failure semantics and add focused regression coverage before merge.

## Solution Assessment

No, not as currently written. The resolver is the right owner boundary, but the new candidate loop must distinguish not-found misses from real trusted-root read failures before merge.
