## Summary

Keep open. Current main does not implement the requested short-click auto-send plus 3-second hold-for-continuous dictation UX; the browser SpeechRecognition button/state were deliberately removed and the remaining Talk control is a separate realtime voice surface, so this needs maintainer product and accessibility direction before implementation.

## What This Changes

Add a single Control UI microphone button where a short click records and auto-sends on pause, while a 3-second hold enters continuous auto-send voice mode with interim transcript feedback.

## Best Possible Solution

Keep this open for maintainer product/design review. If accepted, implement it in Control UI source rather than generated dist output, with explicit dictation modes, accessible pointer/touch/keyboard behavior, no window-global recognition state, clear coexistence with Talk and the server-side STT thread, plus focused tests and docs updates.

## Reproduction Assessment

Not applicable as a feature request. The current-main verification path is high-confidence source/test inspection: the requested Voice input button and STT state are absent, so the short-click and hold-for-continuous paths cannot be exercised.

## Solution Assessment

No. The issue’s proposed generated-dist patch and window-global restart approach are not the best maintainable solution; a source-level design that either reintroduces dictation cleanly beside Talk or explicitly documents Talk/server-side STT as the replacement would be safer.
