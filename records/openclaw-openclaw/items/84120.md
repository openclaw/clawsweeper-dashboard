## Summary

Keep this open: current main and v2026.6.8 still delegate Signal typing to signal-cli `sendTyping`, and the latest inspected signal-cli v0.14.5 dependency still passes `online=true` for 1:1 typing, so the reported user-visible liveness bug is not implemented on main.

## What This Changes

The issue asks OpenClaw to restore visible native Signal typing indicators when long Signal agent turns already call signal-cli `sendTyping` successfully.

## Best Possible Solution

Adopt or wait for a fixed signal-cli/Turasa release, update the Signal runtime/install path if needed, and validate 1:1 and group typing separately before closing.

## Reproduction Assessment

Yes for source-level reproduction: current OpenClaw still calls signal-cli `sendTyping`, signal-cli v0.14.5 routes that into its typing send path, and the inspected Turasa dependency still passes `online=true` for 1:1 typing. No live official-client repro was run in this read-only review, but the reporter supplied patched-runtime live proof.

## Solution Assessment

Yes on direction, but not as an OpenClaw-only TypeScript patch: the reporter's preferred upstream dependency fix targets the layer that still owns the `online` delivery flag. Maintainers need to choose whether to wait for upstream, consume a fixed signal-cli release, or explicitly carry a patched runtime.
