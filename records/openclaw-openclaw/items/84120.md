## Summary

Keep open. Current main and v2026.5.20 still send Signal typing through signal-cli `sendTyping`, and the upstream Turasa dependency used by tested signal-cli releases still passes `online=true` for 1:1 typing; the reporter's live patched-runtime validation is directly relevant. The related open Signal PR covers early group timing, not the upstream 1:1 delivery flag.

## What This Changes

The issue asks OpenClaw to restore visible native Signal typing indicators when official clients ignore successful signal-cli `sendTyping` calls.

## Best Possible Solution

Patch or obtain an upstream signal-cli/Turasa release for 1:1 typing delivery, update OpenClaw's Signal runtime/install path once available, and verify official-client 1:1 plus separate group typing behavior.

## Reproduction Assessment

Yes for a source-level and reporter-backed live path: current main calls signal-cli `sendTyping`, the tested Turasa dependency uses `online=true`, and the reporter showed a patched `online=false` runtime makes official clients display typing. I did not personally run the official-client before/after setup in this review.

## Solution Assessment

Unclear as an OpenClaw-only fix. The narrow durable solution appears to be upstream signal-cli/Turasa behavior plus an OpenClaw runtime update, while carrying a patched runtime locally needs maintainer approval.
