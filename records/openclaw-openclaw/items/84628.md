## Summary

Keep this PR open: the scoped metadata direction is useful and the supplied live-output proof is relevant, but the patch still needs maintainer attention because upgraded installs with pre-patch persisted indexes may keep the unscoped manifest path until the registry is refreshed.

## What This Changes

The PR scopes gateway startup metadata snapshot manifest reconstruction to installed-index-derived plugin ID sets, carries scoped snapshot metadata, persists config-path startup hints, and adds focused regression coverage.

## Best Possible Solution

Land the scoped startup optimization only with an explicit upgrade story that refreshes or migrates legacy installed indexes and proves scoped, unscoped, config-path, and legacy fallback behavior.

## Reproduction Assessment

Yes, at source level. Current main reconstructs installed-index manifests without a pluginIds scope, and the PR body includes before/after live output showing 96 manifests before the patch versus scoped manifest counts after it.

## Solution Assessment

Not yet. The scoped Installed Plugin Index approach is the right boundary, but the patch should prove or implement the legacy persisted-index upgrade path before it can safely close the startup bug.
