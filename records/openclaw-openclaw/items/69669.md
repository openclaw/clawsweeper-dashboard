## Summary

Keep open: current main and the latest release still define thread-bound ACP follow-ups as direct prompt text to the bound ACP session, so changing the default to parent rewriting is a product/session contract decision rather than cleanup-closeable implemented work.

## What This Changes

Thread-bound ACP follow-ups should stay parent-orchestrated by default, with raw relay only when the user explicitly asks for exact forwarding.

## Best Possible Solution

ACP/session owners should choose the follow-up contract, then update runtime dispatch, spawn accepted-note text, public docs, and regression tests together.

## Reproduction Assessment

Yes, at source level. Current main and v2026.6.8 route a bound ACP conversation directly to the ACP session and tests assert raw follow-up text reaches the ACP runtime while parent reply resolution is skipped.

## Solution Assessment

No, not as an autonomous fix yet. The requested parent-rewrite default may be useful, but the maintainable path is an owner-approved runtime/docs/test contract change rather than a one-off pass-through patch.
