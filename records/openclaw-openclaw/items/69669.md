## Summary

Keep open. Current main and the latest release still define bound ACP follow-ups as direct child-session prompt text, and the open related delivery PR addresses suppressed visible replies rather than this issue's requested parent-rewrite default.

## What This Changes

The issue asks for thread-bound ACP follow-ups to be interpreted and rewritten by the parent assistant by default instead of being sent verbatim to the child ACP harness.

## Best Possible Solution

ACP/session owners should decide the follow-up contract, then update runtime dispatch, accepted-note wording, public ACP docs, and regression tests together.

## Reproduction Assessment

Yes, at source level. Current main retargets a bound ACP conversation to the child session and the regression test asserts BodyForAgent "continue" reaches runTurn unchanged while the parent replyResolver is skipped.

## Solution Assessment

Unclear until maintainers choose the product contract. The requested parent-rewrite default conflicts with current docs/tests, so the maintainable path is a coordinated runtime/docs/test change after owner decision.
