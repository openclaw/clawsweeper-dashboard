## Summary

Keep open. Current main still treats thread-bound ACP follow-ups as direct prompt text to the bound ACP session, and the docs plus regression coverage encode that behavior; the same-author related PR is open and docs-only, so this needs an ACP contract decision rather than cleanup closure.

## What This Changes

The issue asks OpenClaw to keep thread-bound ACP session continuity while rewriting shorthand follow-ups through the parent assistant by default, with verbatim relay only on explicit request.

## Best Possible Solution

Settle the ACP follow-up contract, then update runtime dispatch, accepted-note wording, public docs, and regression tests together.

## Reproduction Assessment

Yes. Source inspection shows current main sends a bound ACP follow-up prompt through the ACP path unchanged, and the regression test asserts runTurn receives exactly "continue" while replyResolver is skipped; tests were not run because this review is read-only.

## Solution Assessment

Unclear. Parent-orchestrated rewriting may be the desired UX, but it conflicts with current documented and tested direct ACP routing, so the maintainable solution needs an ACP contract decision before implementation.
