## Summary

Keep open. Current main still treats bound ACP follow-ups as direct prompt text to the ACP session, and the paired same-author PR is still open and docs-only, so the requested parent-orchestrated default needs an ACP contract decision before code or docs change.

## What This Changes

Request changes ACP thread-bound follow-ups so the parent rewrites shorthand into explicit child tasks by default, with raw relay only on explicit request.

## Best Possible Solution

Have ACP maintainers decide the follow-up contract; if parent orchestration is accepted, update runtime dispatch, accepted-note wording, docs, and regression tests together, otherwise keep docs aligned with direct bound routing.

## Reproduction Assessment

Yes. A source-level reproduction exists on current main: the bound ACP regression sends BodyForAgent "continue", asserts runTurn receives exactly "continue", and verifies the parent replyResolver is skipped; tests were not run because this review is read-only.

## Solution Assessment

Unclear. The requested behavior may be the desired orchestration UX, but it conflicts with the current documented and tested ACP routing contract, so it should be a maintainer ACP contract decision rather than an automated bug fix.
