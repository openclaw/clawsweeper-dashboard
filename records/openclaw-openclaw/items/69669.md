## Summary

Keep open: current main and v2026.6.10 still implement thread-bound ACP follow-ups as direct prompt text to the bound ACP session, while this request asks for a new parent-rewrite default that needs ACP/session product approval.

## What This Changes

The issue requests that thread-bound ACP shorthand follow-ups be interpreted and rewritten by the parent by default, with verbatim child relay only on explicit request.

## Best Possible Solution

ACP/session owners should decide the follow-up contract, then update dispatch, accepted-note text, public docs, and regression tests together while keeping the current direct contract intact until then.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.10 route a bound ACP conversation directly to the ACP session and test that raw follow-up text reaches the ACP runtime while parent reply resolution is skipped. I did not run a live ACP/Discord reproduction in this read-only review.

## Solution Assessment

No, not as an autonomous fix yet. The requested default may be useful, but the maintainable path is an owner-approved runtime/docs/test contract change rather than a one-off pass-through patch.
