## Summary

Keep open. Current `main` and the latest release still document and test direct bound-ACP follow-up routing, while the related docs PR was closed unmerged because this changes prompt/session semantics and needs owner review.

## What This Changes

Request asks to make thread-bound ACP follow-up messages parent-interpreted and rewritten by default, with verbatim relay only on explicit request.

## Best Possible Solution

ACP/session owners should decide the direct-routing versus parent-rewrite contract, then update runtime dispatch, accepted-note wording, public docs, and regression tests in one coordinated change.

## Reproduction Assessment

Yes, at source level. Current `main` routes a bound ACP follow-up with `BodyForAgent: "continue"` into `runTurn` unchanged and skips the parent `replyResolver`.

## Solution Assessment

Unclear until owners decide the product contract. Parent rewriting may be the desired UX, but current docs and tests establish direct routing, so a docs-only or stale-cleanup path is not the best solution.
