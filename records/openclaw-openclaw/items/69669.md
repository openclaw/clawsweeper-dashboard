## Summary

Keep open. Current main and the latest release still define bound ACP follow-ups as direct ACP prompt text, while this issue asks for a parent-rewrite default that needs an ACP/session product decision; the open related PR only addresses visible delivery under the current direct-bound-session contract.

## What This Changes

Requests changing thread-bound ACP follow-ups so the parent assistant rewrites shorthand into explicit child tasks by default, with verbatim relay only on explicit user request.

## Best Possible Solution

ACP/session owners should decide the follow-up contract, then update runtime dispatch, spawn accepted-note wording, public ACP docs, and regression tests together.

## Reproduction Assessment

Yes, at source level. Current main retargets a bound conversation to the ACP session, passes BodyForAgent to ACP runTurn, and the regression test expects "continue" to reach runTurn unchanged while the parent replyResolver is skipped.

## Solution Assessment

Unclear until maintainers choose the product contract. The requested parent-rewrite default conflicts with current docs/tests, so the maintainable path is a coordinated runtime/docs/test change after owner decision.
