## Summary

Keep open: current main still uses the old Xiaomi TTS request shape, while this PR directly fixes the linked Token Plan failure with focused provider-owned code, tests, docs, changelog, and sufficient redacted live terminal proof.

## What This Changes

The PR changes Xiaomi TTS to use Bearer auth, inherit `models.providers.xiaomi.baseUrl` when no speech-specific base URL is configured, and updates docs, changelog, and regression tests for Token Plan endpoints.

## Best Possible Solution

Land the focused Xiaomi plugin fix after normal maintainer review and CI, keeping endpoint selection provider-owned and documenting Token Plan configuration without adding a new config knob.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes at source level: current main sends `api-key` for Xiaomi TTS and the linked issue provides live Token Plan 401/success endpoint evidence, though I did not rerun a live key in this review.

## Solution Assessment

Is this the best way to solve the issue? Yes: the patch stays inside the Xiaomi plugin, reuses the existing `apiKey` and `baseUrl` surfaces, preserves explicit TTS overrides, and avoids a core API or product-policy change.
