## Summary

Keep open: the PR targets a real SecretRef/runtime-snapshot bug class and a useful Codex metadata correction, but the proposed SecretRef fix collapses configured raw SecretRefs to missing strings and can bypass existing fail-fast/fail-closed behavior.

## What This Changes

The PR replaces strict SecretRef normalization with non-throwing string normalization in skill env, web_search, memory-host, and Ollama embedding credential paths, exports the memory-host normalizer, and updates Codex fallback context metadata.

## Best Possible Solution

Preserve the Codex catalog metadata update, but repair SecretRef handling by selecting/threading the resolved runtime snapshot or carrying configured_unavailable status so active configured refs resolve or fail closed with focused regressions.

## Reproduction Assessment

Yes. Source inspection and existing tests show the current strict resolver throws for raw active SecretRefs, and extensions/ollama/src/embedding-provider.test.ts has a direct fail-fast case for memorySearch.remote.apiKey.

## Solution Assessment

No. The non-throwing normalizer avoids the throw by treating raw SecretRefs as absent; the safer narrow fix is runtime-snapshot selection or explicit configured_unavailable handling, matching the related Discord fix.
