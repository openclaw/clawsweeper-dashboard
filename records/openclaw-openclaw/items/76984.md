## Summary

Keep open: the PR targets a real SecretRef/runtime-snapshot bug class and a useful Codex catalog update, but the patch is not mergeable because several changed paths turn configured raw SecretRefs into missing credentials, which can silently skip intended auth or fall through to fallback credentials.

## What This Changes

The PR changes SecretRef credential reads in skills, web_search, memory-host, and Ollama embedding code to non-throwing normalization, exports the memory normalizer, and updates the Codex provider fallback context metadata.

## Best Possible Solution

Keep the Codex catalog metadata update, but solve SecretRef hydration by selecting the matching resolved runtime snapshot or carrying configured_unavailable status through each surface without env/provider fallback after an explicit unresolved SecretRef.

## Reproduction Assessment

Yes from source inspection and existing tests. Raw SecretRef objects passed to normalizeSecretInputString become undefined, while current tests require configured SecretRefs to resolve through runtime snapshots or fail closed instead of falling back.

## Solution Assessment

No. Replacing strict resolution with normalizeSecretInputString avoids the throw by erasing the configured SecretRef, but the narrower maintainable fix is to thread the resolved runtime snapshot or preserve configured_unavailable semantics.
