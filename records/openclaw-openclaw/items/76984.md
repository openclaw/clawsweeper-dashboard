## Summary

This PR should stay open because the SecretRef changes avoid the startup throw by dropping raw SecretRef state, which can silently select ambient/provider credentials instead of the configured credential; the Codex metadata part is useful but needs to be separated from a fail-closed SecretRef repair.

## What This Changes

The PR replaces strict SecretRef reads in skill, web-search, memory-host, and Ollama embedding paths with non-throwing normalization and updates Codex fallback context metadata.

## Best Possible Solution

Keep the Codex catalog metadata update, but repair SecretRef handling with runtime-snapshot selection or configured_unavailable status that preserves fail-closed behavior, plus focused tests and a changelog entry.

## Reproduction Assessment

Yes. Source inspection and existing tests show raw SecretRefs are distinct from missing credentials, and the PR diff changes several raw refs into undefined before fallback selection.

## Solution Assessment

No. The non-throwing string normalizer is not the best fix because it loses configured credential state; the safer approach is the related runtime-snapshot/status pattern used by the Discord fix.
