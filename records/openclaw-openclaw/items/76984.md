## Summary

Keep this PR open because the Codex metadata change is useful, but the SecretRef fix drops configured raw refs into `undefined` in credential paths that can then use fallback credentials instead of resolving through the runtime snapshot or failing closed.

## What This Changes

The PR switches several SecretRef credential reads to non-throwing string normalization, exports the memory-host normalizer, and updates Codex fallback model metadata to a 400k native window with a 272k runtime cap.

## Best Possible Solution

Keep the Codex metadata update, but repair the SecretRef paths with runtime-snapshot selection or explicit configured-unavailable status so configured credentials resolve when available and fail closed when not, with focused tests and a changelog entry.

## Reproduction Assessment

Yes. Source inspection and existing tests show raw SecretRefs are distinct from missing credentials, and the proposed patch changes several raw refs into `undefined` before fallback selection.

## Solution Assessment

No. The non-throwing string normalizer avoids the immediate throw by erasing raw SecretRef state; the narrower maintainable fix is to choose the resolved runtime snapshot or carry configured-unavailable status through each credential decision.
