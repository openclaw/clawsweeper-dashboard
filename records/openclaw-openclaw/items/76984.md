## Summary

Keep open: current main still has the reported strict SecretRef reads, and the Codex context catalog update is directionally supported, but the PR is not merge-ready because the proposed non-throwing replacements can drop raw SecretRefs instead of hydrating them.

## What This Changes

The PR switches several SecretRef credential reads in skills, web_search, memory host, and Ollama embedding paths to non-throwing string normalization, exports the memory normalizer, and updates Codex fallback context metadata.

## Best Possible Solution

Thread the runtime-resolved snapshot into these credential-read paths, preserve explicit unresolved-ref diagnostics when no resolved value exists, keep the Codex catalog metadata update, and add focused regressions for each affected surface.

## Reproduction Assessment

Yes from source inspection: strict normalizeResolvedSecretInputString throws on raw SecretRef objects in current main, and the PR's replacement normalizer returns undefined for those same objects. I did not run tests because this review was read-only.

## Solution Assessment

No. Avoiding the throw by calling normalizeSecretInputString is not the narrowest maintainable fix because it drops raw SecretRefs instead of resolving them or carrying configured-unavailable state to the runtime snapshot path.
