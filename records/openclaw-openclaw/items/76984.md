## Summary

Keep this PR open, but do not merge it as-is. The reported SecretRef hydration class is valid on current main, and the Codex fallback metadata update is directionally supported, but the proposed SecretRef fix erases raw configured SecretRefs as `undefined`, which can fall through to ambient or provider fallback credentials instead of selecting the runtime snapshot or failing closed.

## What This Changes

The PR replaces several strict SecretRef credential reads with non-throwing string normalization, exports the memory-host string normalizer, and updates Codex fallback model metadata to a 400k context window with a 272k runtime cap.

## Best Possible Solution

Keep the Codex metadata update, but repair the SecretRef paths with runtime-snapshot selection or explicit `configured_unavailable` status so configured credentials resolve when available and fail closed when not, with focused tests and a changelog entry.

## Reproduction Assessment

Yes. Source inspection and existing tests show current strict readers throw on raw active SecretRefs, and the PR diff clearly changes those raw refs into `undefined` in several credential paths.

## Solution Assessment

No. The non-throwing string normalizer avoids the immediate throw by dropping raw SecretRef state; the narrower maintainable fix is to select the resolved runtime snapshot or carry an inspect-mode configured-unavailable status, matching the related Discord fix.
