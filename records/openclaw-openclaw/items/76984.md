## Summary

Keep open: the PR is addressing a real SecretRef hydration class and a useful Codex metadata cleanup, but the current patch drops configured raw SecretRefs to `undefined` in credential paths that can then use fallback credentials.

## What This Changes

The branch replaces strict SecretRef string reads in skill, web-search, memory-host, and Ollama embedding paths with non-throwing normalization, exports the memory-host string normalizer, and updates Codex fallback model context metadata.

## Best Possible Solution

Keep the Codex metadata update, but repair the SecretRef paths with active runtime-snapshot selection or explicit configured-unavailable status so configured credentials resolve when available and fail closed when not, with focused regression tests and a changelog entry.

## Reproduction Assessment

Yes. Source inspection shows the PR turns raw configured SecretRefs into `undefined` before fallback selection, and current tests already define fail-closed behavior for web-search and Ollama remote key paths.

## Solution Assessment

No. Dropping raw SecretRefs is not the best fix; the narrower maintainable path is to select the resolved runtime snapshot or carry configured-unavailable status, matching the linked Discord fix.
