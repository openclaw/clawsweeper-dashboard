## Summary

Keep this PR open, but it needs repair before merge: the report is valid and related to the same SecretRef runtime-status class as merged #76987, but the current patch avoids the throw by turning raw SecretRefs into undefined in paths that then lose or fall back from the configured credential.

## What This Changes

The PR switches several skill, web-search, memory, and Ollama SecretRef credential reads to non-throwing normalization and updates the Codex fallback model context metadata.

## Best Possible Solution

Preserve the valid PR intent by resolving these paths through the active runtime snapshot or carrying configured-unavailable status, while keeping selected-provider fail-closed behavior and adding focused regression coverage plus a changelog entry.

## Reproduction Assessment

Yes from source inspection: current main throws on raw unresolved SecretRefs in these strict readers, and the PR replacement normalizer returns undefined for those same raw SecretRef objects. I did not run tests because this review was read-only.

## Solution Assessment

No. The PR's direct substitution to normalizeSecretInputString avoids the immediate throw but is not the narrowest maintainable fix because it drops the configured SecretRef instead of resolving it from the runtime snapshot or preserving explicit unavailable status.
