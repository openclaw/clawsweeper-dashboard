## Summary

Keep open. Current main still looks partial for the reported regression: native Vertex URL/Bearer plumbing exists, but the default/forward-compatible `google-vertex/*` path is still not clearly connected to `api: "google-vertex"`, runtime ADC is still `authorized_user`-only, and open PR https://github.com/openclaw/openclaw/pull/60860 still uses closing syntax for this issue.

## What This Changes

The issue reports that `google-vertex` model refs route through the Google AI transport/auth path instead of Vertex AI transport and ADC auth.

## Best Possible Solution

Review, land, or supersede the open Google Vertex implementation so ordinary `google-vertex/*` refs select the native Vertex transport, build project/location endpoints, and use documented ADC/Bearer auth with focused regression coverage.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: resolve a forward-compatible `google-vertex` Gemini model and it can still carry `api: "google-gemini-cli"`, while the native Vertex stream only activates for `api: "google-vertex"` plus authorized-user ADC.

## Solution Assessment

No. Current main is useful partial progress, but the best fix is to finish the Google-plugin-owned native Vertex routing/auth path in https://github.com/openclaw/openclaw/pull/60860 or a narrower replacement, then close this after merge.
