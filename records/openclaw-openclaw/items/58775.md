## Best Possible Solution

Keep the shipped native Google Vertex routing/auth implementation and leave the broader setup/onboarding decisions to https://github.com/openclaw/openclaw/issues/60736 and its active PR cluster.

Yes. The implemented fix keeps model-id normalization shared while preserving Google Vertex transport, endpoint construction, and ADC bearer auth in the owning Google plugin boundary.
