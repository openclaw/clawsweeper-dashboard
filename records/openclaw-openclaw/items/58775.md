## Summary

Current main and shipped releases restore `google-vertex` as a native Vertex transport with Vertex URL construction and ADC/Bearer auth; the broader onboarding/setup work remains tracked separately and does not keep this narrower regression open.

## What This Changes

The issue reports that `google-vertex` was normalized into the generic Google AI transport, causing Vertex models to use the wrong endpoint and auth path.

## Best Possible Solution

Keep the shipped native Google Vertex transport/ADC implementation and leave broader onboarding/setup decisions to the existing Google Vertex follow-up issue and PR.

Yes for this issue's narrow regression. The maintainable fix is in the Google plugin/provider boundary: share model ID normalization while preserving native Vertex transport, endpoint routing, and ADC auth.
