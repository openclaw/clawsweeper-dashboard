## Summary

Keep open. Current main has partial Google Vertex support, but native Vertex routing still depends on `model.api === "google-vertex"` while current config and forward-compatible model paths can fail to preserve that API, and an open implementation PR still explicitly closes this issue.

## What This Changes

The issue reports a regression where `google-vertex` is routed through the regular Google AI Studio transport/auth path instead of Vertex AI ADC and project/location routing.

## Best Possible Solution

Finish or supersede the linked Google Vertex work so `google-vertex/*` refs consistently select native Vertex project/location routing, ADC auth, config/schema support, and focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a forward-compatible `google-vertex` model can still resolve with `api: "google-gemini-cli"`, while the native Vertex stream only activates for `api: "google-vertex"` with ADC available.

## Solution Assessment

No for the current state. The maintainable path is to review, land, or replace the open Google Vertex implementation and include the native config/schema fix rather than closing this before the implementation lands.
