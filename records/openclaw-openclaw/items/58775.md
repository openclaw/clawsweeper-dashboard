## Summary

Current main and the latest release now solve the reported `google-vertex` regression: `google-vertex` remains a distinct provider, routes through the Vertex transport, builds Vertex `/projects/.../locations/.../publishers/google/models/...` URLs, and uses ADC/Bearer auth instead of collapsing into the Google AI Studio API-key path.

## What This Changes

This issue reports that `google-vertex` was treated like Google AI Studio transport/auth instead of Vertex AI transport/auth in the 2026.3.28 release.

## Best Possible Solution

Keep the shipped Google plugin implementation that gives `google-vertex` its own catalog, Vertex request URL path, and ADC/Bearer auth while sharing only Gemini model-id normalization.

## Reproduction Assessment

## Solution Assessment

Yes. The shipped fix is in the Google plugin provider boundary and keeps transport/auth separate from model-id normalization, which is the narrow maintainable solution for the reported regression.
