## Summary

Keep open. Current main has partial Google Vertex support, but the ordinary `google-vertex` path still is not fully proven to select the native Vertex transport/auth path, and the open implementation at https://github.com/openclaw/openclaw/pull/60860 still uses closing syntax for this issue.

## What This Changes

The issue reports a regression where `google-vertex` model refs are normalized into the non-Vertex Google transport/auth path instead of using Vertex AI ADC and project/location endpoints.

## Best Possible Solution

Review, land, or supersede the open Google Vertex implementation so `google-vertex/*` refs select native Vertex routing, project/location endpoints, and documented ADC auth with focused regression coverage.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: a forward-compatible `google-vertex` model can still resolve with `api: "google-gemini-cli"`, while the native Vertex stream only activates for `api: "google-vertex"` plus ADC availability.

## Solution Assessment

No. Current main is useful partial progress, but the maintainable fix is to finish or replace the open Google Vertex PR and then close this after the implementation lands.
