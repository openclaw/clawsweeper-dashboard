## Summary

Keep open. Current main still has a source-reproducible precision-loss path where numeric-looking Feishu wiki `space_id` values can be rounded before being forwarded to Lark, and there is still no Feishu-local guard or stronger model guidance.

## What This Changes

The issue reports that Feishu/Lark wiki `space_id` values are rounded or modified when passed into knowledge-base tools.

## Best Possible Solution

Treat Feishu wiki IDs as opaque strings end to end, reject unsafe number-typed IDs at the Feishu tool boundary with clear retry guidance, and strengthen schema and skill wording so models keep long numeric-looking IDs quoted.

## Reproduction Assessment

Yes, at source level. A focused JSON parse probe reproduces the exact suffix mutation, and current-main source shows the rounded value can flow into Feishu wiki Lark request fields.

## Solution Assessment

Yes. The narrow maintainable fix is Feishu-local validation plus stronger schema/skill guidance, with any shared parser changes limited to preserving raw tool arguments rather than adding new product policy.
