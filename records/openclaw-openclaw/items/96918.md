  - body: This guard returns `terminate: true` only after `message.send` has already executed a second time. In the reported cascade, that still delivers a second visible self-reply before the loop can stop, and mixed batches can avoid batch termination unless every result terminates; prevent the duplicate before delivery or get explicit maintainer approval that a two-message cap is intended.
  - confidence: 0.86
- **[P2] Update the CLI memoryFlush contract docs:** `src/auto-reply/reply/agent-runner-memory.ts:1097`
  - body: Removing the CLI exclusions makes `memoryFlush` run for CLI-backed sessions, but the session-management docs still tell operators that CLI backends skip it. Please update the compaction and CLI-backend docs with the new handoff contract, upgrade behavior, and proof expectations so users are not surprised by embedded housekeeping turns in native-harness sessions.
  - confidence: 0.84
