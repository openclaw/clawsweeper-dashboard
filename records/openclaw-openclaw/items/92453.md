## Summary

Current main has a source-reproducible prompt-context defect: the embedded run owns the canonical session key and ID, but Runtime metadata omits them. The issue should remain open because its clean, proof-supplied closing PR is still awaiting review and merge.

## What This Changes

Add the active session key and stable session ID to the agent Runtime prompt header so the model can identify its current session without inference.

## Best Possible Solution

Land one canonical prompt-plumbing path that carries the runner’s existing session key and ID into Runtime metadata with focused assembly and formatter coverage, without adding inference, alternate identity sources, or routing/storage changes.

## Reproduction Assessment

Yes. On current main, the embedded attempt has the canonical session key and ID, while the prompt metadata types and Runtime renderer omit both, providing a direct high-confidence source reproduction.

## Solution Assessment

Yes in principle. Extending the existing Runtime metadata pipeline is the narrowest maintainable fix; the linked PR still needs normal review of whether both identifiers belong in model-visible prompt text.
