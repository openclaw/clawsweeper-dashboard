## Summary

Current main still omits the concrete session key and stable session ID from the model-visible Runtime header even though the embedded run already carries both values. The report is valid, but it must remain open because the clean, focused closing PR at https://github.com/openclaw/openclaw/pull/92468 is still open and should be reviewed or closed before this issue is resolved.

## What This Changes

Expose the current session key and stable session ID in the agent Runtime header so the agent does not infer its own session identity.

## Best Possible Solution

Land one focused change that projects the already-canonical run `sessionKey` and `sessionId` through `RuntimeInfoInput`, sanitizes them in `buildRuntimeLine`, and locks the behavior with prompt-parameter and formatter tests without changing session storage, routing, or configuration.

## Reproduction Assessment

Yes. Source inspection shows that current main already has `sessionKey` and `sessionId` in the embedded run, but the runtime prompt type and formatter omit them; inspecting a generated Runtime line is the focused verification path.

## Solution Assessment

Yes. Projecting the existing canonical identifiers through the current runtime-info pipeline is narrower and more maintainable than adding a parallel self-inspection tool or making the model infer identity from `agentId`.
