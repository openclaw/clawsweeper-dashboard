Full review comments:

- **[P1] Keep reranker registration behind the plugin API guard:** `src/plugin-sdk/memory-core-host-engine-reranker.ts:47-54`
  - body: This new public subpath exports `registerMemoryReranker` directly, but only `api.registerMemoryReranker` in `src/plugins/registry.ts` enforces `contracts.memoryRerankers` ownership and duplicate-id checks. A plugin importing the subpath can bypass the manifest gate and overwrite a reranker id, so keep the registrar behind the plugin API path or make this subpath type/lookup-only.
  - confidence: 0.88
- **[P1] Raise the external reranker host compatibility floor:** `extensions/memory-external-reranker/package.json:22-25`
  - body: The package declares `minHostVersion` and `pluginApi` as `>=2026.6.2`, but it imports the new `memory-core-host-engine-reranker` SDK subpath that is absent from the latest v2026.6.8 release. Released hosts in that range can accept the plugin and then fail to load it, so set the floor to the first release that actually contains this SDK surface or keep the package unpublished until that version is known.
  - confidence: 0.91
- **[P1] Remove the unrelated dependency graph churn:** `extensions/acpx/npm-shrinkwrap.json:199-214`
  - body: This PR still changes unrelated extension shrinkwraps, for example downgrading `@clack/core`/`@clack/prompts` in the ACPX shrinkwrap, and live dependency-guard is failing for the current head SHA. External PRs cannot carry broad resolved dependency graph changes without explicit admin/secops authorization, so restore unrelated lockfiles/shrinkwraps or get that approval before merge.
  - confidence: 0.93
- **[P2] Fix the private-network opt-in docs:** `docs/plugins/reference/external-reranker.md:61-70`
  - body: The runtime requires private or loopback reranker endpoints to opt in via `models.providers.<provider>.request.allowPrivateNetwork`, but this reference example puts `allowPrivateNetwork` under the plugin config where the code never reads it. Users following the docs would still get the fail-fast private-network error, so move the example and field text to the provider `request` block.
  - confidence: 0.9
