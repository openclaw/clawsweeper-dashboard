## Summary

Keep open. Current main still prints `/model` `Available:` values from bare catalog IDs, while backend parsing only infers a provider for inputs without `/`; nested provider-native IDs and ambiguous bare IDs therefore remain not reliably copy-paste round-trippable. An open PR now overlaps this surface, so the issue should stay open until that PR is reviewed and merged or replaced.

## What This Changes

The issue requests that Control UI `/model` status output show values that can be copied back into `/model`, or that every value shown there be accepted without provider-prefix drift.

## Best Possible Solution

Make the Control UI slash-command status path format `Current model` and every `Available:` entry with the canonical value accepted by `sessions.patch`, reusing the existing catalog/provider helper where possible, while keeping backend bare-ID inference for compatibility and adding regression coverage for copying an `Available:` value back into `/model`.

## Reproduction Assessment

Yes. Source and test inspection on current main gives a high-confidence reproduction path: no-argument `/model` still prints bare `m.id` values, and slash-containing values copied back are parsed as provider/model without unique-provider inference.

## Solution Assessment

Yes. The narrow maintainable fix is in the Control UI display/selection layer, not a broad backend parsing change: print canonical provider-qualified values from the catalog, then verify the session patch path accepts exactly those values.
