## Summary

Keep open: current main still prints bare `/model` `Available:` values even though the catalog carries provider data, and the backend only applies unique-provider inference to no-slash inputs; open PR #50141 remains the overlapping fix candidate.

## What This Changes

Make `/model` `Available:` values copy-paste round-trippable by accepting displayed values directly or showing canonical accepted model refs.

## Best Possible Solution

Land or replace the overlapping Control UI fix so `/model` status prints canonical provider-qualified values that `sessions.patch` accepts, with regression coverage for copying an `Available:` value back into `/model`.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: no-argument `/model` renders bare `m.id` values, while the backend treats no-slash and slash-containing model refs differently during allowlist resolution.

## Solution Assessment

Yes. The narrow maintainable fix is in the Control UI display/selection layer using the existing catalog qualification helper, while preserving backend bare-name compatibility.
