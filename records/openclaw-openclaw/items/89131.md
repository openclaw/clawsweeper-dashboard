## Summary

Keep open: current main and the latest release still expose realtime Talk relay as audio/tool transport plus capability metadata, but not the requested provider/gateway video-frame contract. The open related PRs are partial implementation candidates and still need product/security/API direction before this issue can close.

## What This Changes

Request a provider/gateway-level realtime video-frame transport so relay sessions and realtime providers can receive one-shot visual frames outside browser-direct WebRTC paths.

## Best Possible Solution

Settle the product/security/API shape first, then implement a narrow additive one-shot VideoFrame contract with provider mappings, gateway protocol validation, docs, and focused relay/provider tests.

## Reproduction Assessment

## Solution Assessment

Unclear until maintainers decide the product/security direction. A provider/gateway seam is plausible for relay clients, but the permanent SDK/RPC contract should be approved before implementation.
