## Summary

Keep open. The worker-state observer builds on a real task-registry observer seam, but the PR still contains blocking doctor regressions and an operator-facing state-file contract that needs docs/changelog/schema review before merge.

## What This Changes

The PR adds a default task-registry observer that writes `worker-state.json` snapshots with tests, and also adds `openclaw doctor --json` plumbing.

## Best Possible Solution

Land the worker-state snapshot only after the schema and public contract are approved and documented, and repair or split the doctor JSON changes so live Gateway validation and machine-readable output are preserved.

## Reproduction Assessment

Yes. The regressions are reproducible from source: the PR replaces the current live `callGateway(status)` path with `getHealthSnapshot`, and the existing `updateResult.handled` return remains before the new JSON output block.

## Solution Assessment

No. The observer seam is a reasonable implementation point, but mixing it with broken doctor JSON behavior and weaker live Gateway validation is not the narrowest maintainable path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Restore the live Gateway health probe:** `src/commands/doctor-gateway-health.ts:27`
  - body: Replacing the live Gateway RPC with `getHealthSnapshot` makes doctor build a local snapshot instead of contacting the configured Gateway with its auth/config. A stopped Gateway or bad token can now set `healthOk` true and skip the existing failure path.
  - confidence: 0.92
- **[P2] Emit JSON before every --json return:** `src/flows/doctor-health.ts:81-82`
  - body: When the update preflight handles the run, this early return exits before the later JSON write block. Interactive `openclaw doctor --json` consumers can then receive no machine-readable payload at all.
  - confidence: 0.84
- **[P3] Add the required changelog entry:** `src/tasks/task-registry.store.ts:67-68`
  - body: This user-facing feature adds `worker-state.json` and `doctor --json` behavior, but the PR does not update `CHANGELOG.md`. Repo policy requires a changelog entry for user-facing feature changes before merge.
  - confidence: 0.82
