## Summary

Keep this PR open, but it still needs changes before merge: the live head is unchanged after the prior review, lacks real behavior proof, weakens doctor Gateway validation, adds a default runtime sidecar state file after the SQLite runtime-state refactor, and mixes in a partial doctor JSON contract.

## What This Changes

## Best Possible Solution

Expose worker/task state through the SQLite-backed task/runtime-state contract or an explicit debug/export/API surface, and split any complete doctor JSON contract into a separate PR that preserves live Gateway validation.
