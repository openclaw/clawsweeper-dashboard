## Summary

Keep open: current main does not implement the requested worker-state snapshot, and the PR diff still contains blocking doctor health and JSON-output regressions that make it unsafe to merge as-is.

## What This Changes

The PR adds a task-registry observer that writes `worker-state.json` snapshots and also adds a `doctor --json` output path with tests.

## Best Possible Solution

Repair or split the doctor `--json` work so live Gateway RPC validation is preserved and JSON is emitted on every `--json` exit, then land the worker-state observer only with maintainer-approved schema, docs, and changelog coverage.

## Reproduction Assessment

Yes. The Gateway health regression is reproducible from the diff by comparing current main’s live `callGateway` path with the PR’s local `getHealthSnapshot` path, and the JSON-output bug follows from the `updateResult.handled` early return before the new JSON write block.

## Solution Assessment

No. The observer seam is a reasonable place for a snapshot writer, but mixing in a broken doctor JSON change and weakening live Gateway validation is not the narrowest maintainable solution.
