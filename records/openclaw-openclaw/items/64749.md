## Summary

Keep open: current main still lacks the requested `worker-state.json` snapshot, but the current PR head is not mergeable as-is because it still weakens live Gateway health validation and can skip `--json` output on an existing early-return path.

## What This Changes

The PR adds an observable task-registry writer for `worker-state.json` and also introduces a separate `openclaw doctor --json` mode with tests.

## Best Possible Solution

Split or repair the doctor JSON work so live Gateway RPC validation is preserved and JSON is emitted on every JSON-mode exit, then land the worker-state observer only with an agreed schema, docs, tests, and changelog entry.

## Reproduction Assessment

Yes. The regressions are reproducible from static diff review: the PR swaps the configured Gateway call for local `getHealthSnapshot`, and the existing `updateResult.handled` return remains before the new JSON write block.

## Solution Assessment

No. The task-registry observer seam is a reasonable implementation point, but the proposed branch is not the narrowest maintainable solution while it changes doctor behavior and weakens health validation.

## Review Findings
