## Summary

Keep open: the PR still contains source-reproducible doctor regressions, lacks after-fix real behavior proof, and needs maintainer direction on whether a new state-file contract belongs beside the existing task JSON/RPC surfaces.

## What This Changes

The PR adds a task-registry observer that writes `worker-state.json` by default, and also adds a new `doctor --json` output path with tests.

## Best Possible Solution

Split or drop the doctor JSON work, then route task observability through the existing task JSON/RPC surfaces or a maintainer-approved state contract aligned with the runtime-state direction.

## Reproduction Assessment

Yes for the PR defects: source inspection of PR head `77645ab64c1cb4c9eb34da0a7c1c2a27fccb61ca` shows the Gateway check replaced by a local snapshot and the JSON writer placed after an early-return path. I did not run tests because this was a read-only review.

## Solution Assessment

No. The task-registry observer seam may be useful, but this branch is not the best merge path while it weakens doctor health validation, adds a partial CLI JSON contract, and needs maintainer agreement on the new state-file surface.
