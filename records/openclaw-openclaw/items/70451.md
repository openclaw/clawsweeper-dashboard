## Summary

Keep open. Current main and the latest release still route the hooks CLI report path through full plugin diagnostics, so the hooks-specific timeout remains source-visible even though the broader bootstrap hang in https://github.com/openclaw/openclaw/issues/64306 was closed.

## What This Changes

The issue reports Windows timeout/SIGKILL for `openclaw hooks enable/list` and asks whether hook registration can be managed without the hanging CLI.

## Best Possible Solution

Make hooks CLI reporting metadata-only for plugin-managed hooks while preserving plugin hook visibility and the existing plugin-managed enable/disable restriction.

## Reproduction Assessment

No for a high-confidence live reproduction: this read-only review did not run Windows with Node 24. The source-level path is clear on current main: hooks commands call `buildHooksReport`, which calls plugin diagnostics and can reach runtime plugin loading.

## Solution Assessment

Yes for the repair direction: the narrow maintainable fix is to build hook reports from workspace hooks plus plugin metadata or snapshots instead of full diagnostics. A broad Windows loader speedup would not remove this unnecessary runtime dependency from a control-plane command.
