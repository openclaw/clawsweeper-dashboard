## Summary

Keep open. Current main and the latest release still route hooks list/info/check/enable/disable through plugin diagnostics/runtime registration, and the linked fix PR remains open and conflicting rather than merged.

## What This Changes

The issue reports Windows OpenClaw CLI hooks commands hanging until timeout or SIGKILL when listing or enabling hooks.

## Best Possible Solution

Land or replace the narrow hooks one-shot success-exit fix, then validate it with Windows or a temporary plugin that leaves an open handle; use a metadata-only hooks report only if the hang occurs before output.

## Reproduction Assessment

Yes by source inspection: current main hooks commands load plugin diagnostics, diagnostics can execute plugin register handlers, and successful hooks actions do not force the process to exit. I did not establish a live Windows/Node 24 failing run.

## Solution Assessment

Unclear until the active PR has stronger proof: a success-exit wrapper is the narrow fix for lingering handles after output, while a metadata-only report would be safer if the timeout happens before hooks output is produced.
