## Summary

Keep open. Current main still sends the hooks CLI report path through full plugin diagnostics, so the Windows timeout/SIGKILL class remains source-backed and has not been implemented or superseded.

## What This Changes

The issue reports Windows CLI hook commands hanging or being SIGKILLed after entering plugin registry loading.

## Best Possible Solution

Make hooks CLI reporting metadata-only for plugin-managed hooks, while keeping full runtime plugin diagnostics reserved for explicit plugin inspection and gateway execution paths.

## Reproduction Assessment

Unclear for a high-confidence live reproduction because this review did not run Windows with Node 24. The source-level path is clear on current main: hooks commands call `buildHooksReport`, which calls runtime plugin diagnostics and can reach `loadOpenClawPlugins`.

## Solution Assessment

Yes, the narrow maintainable fix is to build hooks CLI list/info/check/toggle decisions from workspace hooks plus manifest or metadata snapshot hook data. A broader Windows loader speedup would not remove the unnecessary runtime dependency from this control-plane command.

## Repair Work Prompt

Fix the hooks CLI timeout path reported in https://github.com/openclaw/openclaw/issues/70451. On current main, `src/cli/hooks-cli.ts` builds hook reports by calling `buildPluginDiagnosticsReport`, which can import runtime plugin modules; make `openclaw hooks list`, `info`, `check`, `enable`, `disable`, and bare `hooks` use workspace hook entries plus cold manifest/metadata-backed plugin hook entries instead. Preserve the documented behavior that plugin-managed hooks appear as `plugin:<id>` and cannot be enabled or disabled from the hooks CLI. Do not change gateway runtime hook loading, plugin activation policy, provider behavior, or broad Windows loader policy. Add regression coverage proving the hooks CLI report path does not call runtime diagnostics/module loading and still displays plugin-managed hooks. Add a user-facing changelog fix entry unless maintainers classify the final patch as internal-only.
