## Summary

Keep open. Current main and the latest release still route the hooks CLI report/toggle path through full plugin diagnostics and runtime plugin loading; the broader bootstrap hang was closed separately, but that closure explicitly left this hooks-specific path to track here.

## What This Changes

The issue asks for Windows `openclaw hooks` commands to return normally instead of hanging during plugin registry/runtime loading.

## Best Possible Solution

Make hooks CLI reporting metadata/snapshot-based for plugin-managed hook visibility while preserving workspace hook toggling and the plugin-managed enable/disable restriction.

## Reproduction Assessment

No high-confidence live reproduction was established because this review did not run Windows with Node 24. The current-main source path is clear: hooks commands call `buildHooksReport`, which calls plugin diagnostics and can reach runtime plugin loading.

## Solution Assessment

Yes for the repair direction: the narrow maintainable fix is to remove full runtime plugin diagnostics from the hooks CLI report path. A broader Windows loader cleanup would not remove this avoidable control-plane dependency.
