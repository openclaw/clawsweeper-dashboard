## Summary

Keep open. Current main still sends the hooks CLI report path through plugin diagnostics, so the hooks commands named in the Windows timeout report can still reach runtime plugin loading after bootstrap.

## What This Changes

The issue asks for `openclaw hooks list/info/check/enable/disable` on Windows to complete without hanging or being SIGKILLed by plugin-registry/runtime loading.

## Best Possible Solution

Make hooks CLI reporting metadata-only for plugin-managed hooks, reserving full plugin runtime diagnostics for explicit runtime inspection and gateway execution paths.

## Reproduction Assessment

Unclear for the live SIGKILL because this review did not run Windows with Node 24. The source-level path is reproducible on current main: hooks commands call `buildHooksReport`, which calls plugin diagnostics and can reach `loadOpenClawPlugins` with module loading enabled.

## Solution Assessment

Yes, the narrow maintainable fix is to build hooks CLI list/info/check/toggle decisions from workspace hook entries plus manifest or metadata snapshot hook data. A broader Windows loader speedup would not remove the unnecessary runtime dependency from this control-plane command.
