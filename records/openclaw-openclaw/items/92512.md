- `P1`: The regression repeatedly disrupts reading WebChat while an agent is actively streaming, and the proposed fix leaves a common input pattern uncovered.

## Summary

The PR targets a valid current WebChat regression at the central scroll-state boundary, but its user-intent check can still suppress small upward scroll events during the programmatic guard. The supplied validation is test-only and does not demonstrate the changed interaction in a real browser, so the PR is not ready to merge.

## What This Changes

The PR adds a WebChat follow lock so deliberate upward scrolling during streaming is preserved until the user returns to the bottom.

## Best Possible Solution

Treat any genuine upward movement away from the active programmatic target as an interruption, retain bottom-only follow reacquisition, and prove gradual and larger wheel or trackpad input in a live WebChat stream.

## Reproduction Assessment

Yes. Current source shows that the programmatic-scroll guard can preserve near-bottom follow after upward user movement, and the linked release report supplies concrete WebChat streaming steps.

## Solution Assessment

No, not yet. An explicit follow lock is the right central mechanism, but interruption detection should not depend on the 12-pixel header-animation threshold because gradual wheel or trackpad events can be smaller.
