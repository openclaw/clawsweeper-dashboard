## Summary

The revised head is a focused, source-consistent fix for the current WebChat streaming backscroll race, with no discrete correctness or security finding; keep it open because the external contributor has supplied only automated test/check output and explicitly has not demonstrated the changed interaction in a real browser.

## What This Changes

The PR adds an explicit WebChat follow lock that preserves upward backscroll during streaming until the user returns to the bottom, including sub-12px wheel or trackpad movement, with focused regression tests.

## Best Possible Solution

Keep the explicit central follow-lock design and existing three-mode contract, then require a redacted live WebChat stream recording that shows small and larger upward backscroll staying fixed and follow resuming only after returning to the bottom.

## Reproduction Assessment

Yes from current-main source inspection and the linked OpenClaw 2026.6.5 steps: upward input can escape the programmatic target while remaining inside the 450px follow threshold, after which another streaming update scrolls down. This review did not execute the browser path.

## Solution Assessment

Yes in design: explicit follow state in the central scroll helper is narrower and more maintainable than changing the broad near-bottom threshold, and the revised head preserves the manual, always, and near-bottom modes. Merge readiness still depends on real browser proof.
