## Summary

Keep open. Current main still sends ACP live block output through normal routed or direct block-message delivery instead of a channel draft preview, and the remaining work needs a channel live-preview contract decision rather than conservative cleanup.

## What This Changes

ACP live progress for bound sessions should update an editable channel draft instead of arriving as separate block messages or only a final response.

## Best Possible Solution

Bridge ACP live block output into the existing channel message live-preview/finalization contract where possible, or add the smallest missing typed channel capability, then implement Telegram first with documented fallback for unsupported channels.

## Reproduction Assessment

Yes for a source-level current-main reproduction: ACP live chunks are projected as `block` deliveries and then routed through `routeReply` or `sendBlockReply`. I did not run a live Telegram ACP harness, so the transport behavior still needs implementation proof.

## Solution Assessment

Unclear as an exact fix. `streamTo: "parent"` is not the right solution for channel draft editing; the maintainable path is a typed channel live-preview integration, preferably reusing the existing `message.live` contract if it fits.
