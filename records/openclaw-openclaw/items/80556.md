## Summary

Keep open. Current main still routes ACP `deliveryMode: "live"` output as normal block deliveries, while Telegram draft editing exists only in the native channel delivery path; the separate `streamTo: "parent"` ACP relay is already implemented for parent-session system events and does not solve channel draft previews.

## What This Changes

Request ACP bound sessions to update a live channel draft under `acp.stream.deliveryMode: "live"` instead of emitting separate block messages.

## Best Possible Solution

Add an additive, typed channel live-preview/draft contract that ACP dispatch can use under `acp.stream.deliveryMode: "live"`, implement Telegram first, and document fallback semantics for channels without the capability.

## Reproduction Assessment

Yes for a source-level reproduction: current main projects ACP live text into `block` deliveries and routes those through normal message delivery. I did not run a live Telegram/ACP harness, so this is not a full end-to-end reproduction.

## Solution Assessment

Unclear as an exact implementation, but the direction is plausible. The safest solution should reuse or extend the existing channel message/live-preview contracts rather than add a parallel ad hoc method if the existing abstraction can support ACP.
