## Summary

Keep open. Current main still routes ACP `deliveryMode: "live"` output as normal block deliveries, while Telegram draft editing is only wired through the native Telegram dispatch path, so the requested bound-session draft preview remains a real API/product gap.

## What This Changes

Request live ACP bound-session progress to update a channel draft preview under `acp.stream.deliveryMode: "live"`, separate from `streamTo: "parent"` parent-session relay.

## Best Possible Solution

Add a backward-compatible channel live-preview/draft contract that ACP dispatch can use under `acp.stream.deliveryMode: "live"`, implement Telegram first, and document fallback behavior for channels without the capability.

## Reproduction Assessment

Yes, for source-level current behavior: ACP live text enters the block delivery pipeline and reaches `routeReply` or `sendBlockReply`. I did not establish a full live Telegram/ACP E2E reproduction.

## Solution Assessment

Unclear as an exact API shape, but the direction is plausible. The maintainable fix should extend a generic channel streaming/draft contract rather than add a Telegram-only ACP shortcut if the existing abstractions can support it.
