## Summary

Keep open. Current main and the latest shipped tag still route ACP live block output through normal message delivery, while draft stream helpers remain native channel paths; fixing this needs an additive channel/plugin draft capability decision.

## What This Changes

Request ACP live-mode output for bound sessions to update an editable channel draft instead of emitting separate block messages, using the existing `acp.stream.deliveryMode: "live"` gate.

## Best Possible Solution

Add an additive typed channel live-preview/draft capability that ACP dispatch can use when `acp.stream.deliveryMode` is `"live"`, implement Telegram first, and document fallback behavior for unsupported channels.

## Reproduction Assessment

Yes for a source-level current-main reproduction: ACP live chunks are projected as `block` deliveries and then routed through `routeReply` or `sendBlockReply`. I did not run a live Telegram/ACP harness, so this is not a full end-to-end reproduction.

## Solution Assessment

Unclear as an exact implementation, but the direction is plausible. A typed channel live-preview contract is the maintainable path; exposing only `streamTo: "parent"` would not solve channel draft editing because that relay targets parent-session context.
