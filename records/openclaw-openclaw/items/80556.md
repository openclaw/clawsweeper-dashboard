## Summary

Keep open. Current main still turns ACP live projector chunks into block deliveries that route as normal channel messages, while Telegram draft preview streaming remains in the native Telegram delivery path; the remaining work needs a channel/plugin live-preview contract decision.

## What This Changes

Request that ACP bound-session live output feed a channel draft/preview stream instead of per-block messages when `acp.stream.deliveryMode` is `"live"`.

## Best Possible Solution

Add an additive typed channel live-preview/draft capability, preferably aligned with the existing channel-message live-preview model, wire ACP `deliveryMode: "live"` through it for Telegram first, and define fallback behavior for unsupported channels.

## Reproduction Assessment

Yes for a source-level current-main reproduction: ACP live chunks are projected as `block` deliveries and then routed through `routeReply` or `sendBlockReply`. I did not run a live Telegram ACP harness, so this is not a full end-to-end reproduction.

## Solution Assessment

Unclear as an exact implementation. The request correctly avoids using `streamTo: "parent"` for channel drafts, but the maintainable fix should extend the generic channel live-preview contract rather than add a Telegram-only ACP shortcut.
