## Summary

Keep open. Current main still sends ACP `deliveryMode: "live"` block output through normal reply delivery, while Telegram draft editing remains wired inside the native Telegram message dispatch path; solving the requested single editable draft behavior needs an additive channel/plugin live-preview contract decision.

## What This Changes

Request ACP bound-session live output to update a single channel draft or preview message instead of sending separate block messages.

## Best Possible Solution

Expose ACP live output through the existing channel-message finalizable live-preview model where possible, or add a small additive SDK seam if needed; implement Telegram first and keep normal block delivery as the fallback for unsupported channels.

## Reproduction Assessment

Yes for a source-level reproduction: ACP live text is projected into `block` deliveries and the current Telegram live-mode test asserts `sendBlockReply`. I did not run a live Telegram/ACP harness, so this is not a full end-to-end reproduction.

## Solution Assessment

Unclear as the exact proposed method. The direction is sound, but the narrowest maintainable solution should reuse the existing channel-message live-preview/finalizer contract if it fits instead of adding a parallel ACP-only method.
