## Summary

Keep open: current main and the latest release still project ACP live output as block deliveries routed through normal reply delivery, while Telegram's editable draft stream and the channel live-preview adapter capabilities are not wired into ACP bound-session delivery.

## What This Changes

The issue requests editable live channel draft/progress streaming for bound ACP sessions, using the existing ACP live delivery gate rather than separate per-block messages or final-only replies.

## Best Possible Solution

Define the ACP live-draft integration around the existing channel `message.live` preview/finalizer contract if it is sufficient, then wire `acp.stream.deliveryMode="live"` to one editable draft with fallback/finalization coverage for Telegram and at least one sibling channel.

## Reproduction Assessment

Yes, at source level: current main and `v2026.6.10` project ACP live deltas into block deliveries that route through `routeReply` or `sendBlockReply`, while Telegram's editable draft stream is only created inside the native Telegram dispatch path. I did not run a live Telegram ACP session in this read-only review.

## Solution Assessment

Unclear as proposed: a new `createAcpLiveDraftStream` method is plausible, but current main already has `message.live` draft preview/finalizer capabilities that should be reused or explicitly extended before adding a parallel channel API.
