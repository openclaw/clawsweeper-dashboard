## Summary

Keep open. Current main and the latest release still route ACP live blocks through normal reply delivery, while Telegram draft editing remains in the native Telegram dispatch path. The request is valid, but it needs a generic channel/plugin live-preview seam rather than a narrow bugfix.

## What This Changes

Request live edited channel draft updates for ACP bound sessions when `acp.stream.deliveryMode: "live"`, instead of sending incremental ACP blocks as separate channel messages.

## Best Possible Solution

Add a generic optional channel live-preview contract for ACP delivery, implement Telegram first, preserve per-block fallback for channels without the capability, and document the SDK behavior.

## Reproduction Assessment

Yes for a source-level reproduction: current main projects ACP live output into `block` deliveries and then sends those blocks through normal reply delivery. I did not establish an end-to-end live Telegram harness run.

## Solution Assessment

Unclear as an exact API, but the requested direction is plausible. The maintainable solution should extend the channel/plugin live-preview contract rather than hard-code Telegram-specific ACP draft behavior.
