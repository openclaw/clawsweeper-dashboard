## Summary

Keep open. Current main has ACP progressive block delivery, but ACP live output still does not feed a channel-owned edited draft; Telegram draft editing remains wired to the native channel path, so the requested single live draft behavior still needs design and implementation.

## What This Changes

The issue requests live channel draft updates for ACP bound sessions using `acp.stream.deliveryMode: "live"` rather than routing each ACP block as a separate message.

## Best Possible Solution

Add an additive typed channel live-preview contract that ACP dispatch can use under `acp.stream.deliveryMode: "live"`, implement Telegram first, and define documented fallback semantics for channels without draft capability.

## Reproduction Assessment

Yes for a source-level reproduction: current main projects ACP live text into block deliveries and those blocks go through normal message delivery. I did not run a live Telegram ACP bound-session test in this read-only review.

## Solution Assessment

Unclear as a concrete API shape. The maintainable direction is to reuse or extend the existing channel message/live-preview contracts if they fit, otherwise add a narrow backwards-compatible channel plugin seam for ACP draft updates.
