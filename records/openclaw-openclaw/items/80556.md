## Summary

Keep this open: current main still sends ACP live output as block/final channel deliveries and has no channel live-draft SDK seam that can feed Telegram's editable draft stream for bound ACP sessions.

## What This Changes

Request ACP bound-session progress events to update a single channel draft, using the existing ACP live delivery mode rather than exposing streamTo:"parent" for Telegram draft editing.

## Best Possible Solution

Add a narrow optional channel live-draft/progress contract for ACP live delivery, implement and prove Telegram first, and document how acp.stream.deliveryMode:"live" differs from streamTo:"parent".

## Reproduction Assessment

Yes. The current source path shows ACP live output becoming block deliveries through dispatch-acp-delivery, while Telegram's editable draft stream remains in a separate native Telegram pipeline; I did not run a live Telegram ACP turn.

## Solution Assessment

Unclear until maintainers choose the API shape. The proposed optional channel live-draft seam is the likely owner boundary, but the final design needs an explicit plugin SDK scope and channel rollout decision.
