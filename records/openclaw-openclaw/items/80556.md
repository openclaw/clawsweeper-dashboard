## Summary

Keep open. Current main still projects ACP live output into block deliveries and routes those through normal channel delivery, while Telegram’s edited draft stream remains confined to the native Telegram message-dispatch path.

## What This Changes

ACP bound sessions should relay live ACP progress into a channel draft/preview stream instead of producing separate block messages or only a final reply.

## Best Possible Solution

Add an additive, typed channel live-draft/preview contract that ACP dispatch can use under existing `acp.stream.deliveryMode: "live"`, implement Telegram first, and document fallback behavior for channels without the capability.

## Reproduction Assessment

Yes, for a source-level reproduction: current main projects ACP live text into block deliveries and routes those through normal message delivery. I did not run a live Telegram ACP bound session, so this is not a full end-to-end reproduction.

## Solution Assessment

Unclear as an exact API, but the requested direction is plausible. The maintainable path should reuse or extend the existing channel lifecycle/streaming contracts rather than add a Telegram-only shortcut.
