## Summary

Keep open. Current `main` still turns ACP live output into block deliveries and sends those blocks through normal reply delivery, while Telegram draft editing remains scoped to the native Telegram message pipeline; solving this needs an additive channel/plugin draft capability decision rather than cleanup closure.

## What This Changes

Request wiring ACP live-mode output from bound sessions into a channel draft stream instead of sending each live block as a separate channel message, with `streamTo: "parent"` treated as a separate parent-session relay feature.

## Best Possible Solution

Add an additive typed channel live-preview/draft capability that ACP dispatch can use when `acp.stream.deliveryMode` is `"live"`, implement Telegram first, and document fallback behavior for channels that do not support live draft updates.

## Reproduction Assessment

Yes for a source-level reproduction: ACP live chunks are projected as `block` deliveries, and the delivery coordinator routes those blocks through `routeReply` or `sendBlockReply`. I did not establish a full live Telegram/ACP end-to-end reproduction.

## Solution Assessment

Unclear as an exact API, but the direction is sound. A typed channel draft/live-preview capability is the maintainable path; exposing only `streamTo: "parent"` would solve parent-session relay, not channel draft editing.
