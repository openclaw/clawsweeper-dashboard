## Summary

Keep open. Current main and the latest release still route ACP `deliveryMode: "live"` text through block-message delivery, while Telegram draft editing remains in the Telegram-native dispatch path; solving this needs a maintained channel live-preview contract decision rather than cleanup closure.

## What This Changes

Request to make ACP bound-session live output update a channel draft, especially Telegram, under `acp.stream.deliveryMode: "live"` instead of sending separate block messages.

## Best Possible Solution

Route ACP live block delivery through the existing channel message live-preview/finalizer contract, or a documented additive extension of it, with Telegram implemented first and clear fallback semantics for channels without draft support.

## Reproduction Assessment

Yes for a source-level reproduction: current main projects ACP live deltas into block deliveries and sends those through normal route/sendBlockReply delivery. I did not perform a live Telegram plus ACP harness run.

## Solution Assessment

Unclear as an exact implementation. The requested direction is valid, but adding `createAcpLiveDraftStream` as a parallel method may be less maintainable than reusing or extending the existing channel-message live-preview/finalizer contract.
