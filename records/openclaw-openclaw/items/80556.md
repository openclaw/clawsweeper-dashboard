## Summary

Keep this open: current main supports explicit `sessions_spawn` parent streaming, but the configured ACP binding path has no public `streamTo` config field and no source-visible bridge from parent-stream system events into channel progress drafts. The remaining work is a real user-facing config/runtime feature, not an auto-close cleanup item.

## What This Changes

Request ACP conversation bindings to opt into `streamTo:"parent"` and surface ACP progress as live channel draft edits.

## Best Possible Solution

Define a documented ACP binding streaming contract, update types/schema/help/docs/tests, propagate it through configured binding specs/session setup, and add a deliberate bridge from parent-stream progress events into the channel progress draft mechanism.

## Reproduction Assessment

Yes, source-reproducible: current main has no `streamTo` field in ACP runtime or binding config, and parent streaming is only wired through explicit `sessions_spawn`. I did not establish a live Telegram/ACP harness reproduction.

## Solution Assessment

Unclear as a final implementation until maintainers choose the public config and UX boundary. The proposed `streamTo:"parent"` direction is plausible, but the maintainable fix should treat channel draft updates as an explicit channel/runtime contract rather than an incidental system-event side effect.
