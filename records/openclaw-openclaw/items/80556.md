## Summary

Keep open. Current main still projects ACP live output into normal block deliveries, while Telegram draft editing exists only in the native channel delivery path; the remaining fix is an additive channel/plugin live-preview contract, not already implemented cleanup.

## What This Changes

Request ACP bound-session live progress to update an editable channel draft instead of posting separate block messages, with `streamTo: "parent"` treated as a separate parent-session relay feature.

## Best Possible Solution

Add a generic typed live-preview/draft contract that ACP dispatch can use under `acp.stream.deliveryMode: "live"`, implement Telegram first, and document fallback behavior for channels without the capability.

## Reproduction Assessment

Yes, at source level: current main projects ACP live text into `block` deliveries and routes those through normal message delivery. I did not run a live Telegram/ACP harness, so this is not a full end-to-end reproduction.

## Solution Assessment

Unclear as an exact method, but the direction is plausible. The safest solution should extend the existing channel message/live-preview contract if it can support ACP, rather than adding a parallel ACP-only draft API by default.
