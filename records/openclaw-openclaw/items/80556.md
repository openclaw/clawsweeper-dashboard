## Summary

Keep this open: current main still routes ACP live projected output through block/final channel delivery, while Telegram's edit-in-place draft stream remains a separate native channel pipeline. The remaining work is a compatibility-sensitive channel/plugin API and product-scope decision, not stale cleanup or an already-implemented fix.

## What This Changes

Request live ACP progress for bound channel sessions by feeding ACP live deliveries into an edit-in-place channel draft stream instead of separate block messages.

## Best Possible Solution

Add a narrow optional channel live-draft/progress contract used by ACP live delivery, implement and prove it for Telegram first, and document how acp.stream.deliveryMode:"live" differs from streamTo:"parent".

## Reproduction Assessment

Yes, this is source-reproducible from current main: ACP live mode produces block deliveries and ACP dispatch has no channel draft stream handle, while Telegram draft editing lives in a separate native pipeline. I did not run a live Telegram ACP turn.

## Solution Assessment

Unclear until maintainers choose the API shape: the proposed optional channel draft-stream seam is directionally aligned, but the best fix needs an explicit plugin SDK scope and channel rollout decision before implementation.
