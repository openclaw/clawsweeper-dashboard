## Summary

Keep open. Current main still projects ACP `deliveryMode: "live"` output as ordinary block deliveries, while Telegram draft editing is implemented only in the native Telegram streaming path; the already-implemented `streamTo: "parent"` mechanism is a separate parent-session relay and does not update channel drafts.

## What This Changes

ACP bound sessions using live delivery should update the channel's draft or preview stream instead of emitting separate block messages.

## Best Possible Solution

Add an additive, typed channel live-preview/draft contract that ACP live delivery can use, implement Telegram first, and define fallback semantics for channels without the capability.

## Reproduction Assessment

Yes for a source-level reproduction: `acp.stream.deliveryMode: "live"` feeds ACP deltas into block deliveries that route through `routeReply` or `sendBlockReply`, and the Telegram live-mode test asserts those block sends. I did not perform a live Telegram/ACP harness run.

## Solution Assessment

Unclear as an exact API shape, but the direction is plausible. The maintainable fix should reuse or extend the channel/plugin live-preview contract under the existing ACP live config gate, rather than treating `streamTo: "parent"` as the channel draft mechanism.
