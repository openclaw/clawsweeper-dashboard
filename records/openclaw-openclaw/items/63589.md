## Summary

Keep open. Current `main` still omits the requested WhatsApp participant identity and quoted-reply fields from the inbound gateway log payload, while the same function already computes and queues those fields for message delivery; the directly related implementation PR was closed unmerged.

## What This Changes

Include WhatsApp participant identity and quoted reply context fields in gateway inbound log entries.

## Best Possible Solution

Add the already-computed sender and quoted-reply scalar fields to the existing WhatsApp `inboundLogger.info()` payload and cover that payload with a focused monitor-inbox regression test.

## Reproduction Assessment

Yes. Source inspection on current `main` reproduces the omission, and a focused monitor-inbox test can send a WhatsApp group quoted reply and assert the logger payload includes `senderJid`, `senderE164`, `senderName`, and `replyTo*`.

## Solution Assessment

Yes. The narrowest maintainable solution is to enrich the existing log object from `inbound.participantJid`, `inbound.senderE164`, `msg.pushName`, and `enriched.replyContext` without changing mention gating, access control, delivery, or dependencies.
