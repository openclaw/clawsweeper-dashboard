Keep open. ## What This Changes

The issue requests adding already-computed WhatsApp participant identity and quoted reply context fields to gateway inbound log entries.

## Best Possible Solution

Enrich only the WhatsApp inbound logger payload with existing scalar sender and reply-context fields, add focused regression coverage, and leave mention gating, delivery, debouncing, and prompt formatting unchanged.

## Reproduction Assessment

Yes. Source inspection on current main shows the logger object lacks the requested fields while the same enqueue path builds them for `QueuedInboundMessage`; a mocked WhatsApp upsert can assert the missing `inboundLogger.info()` payload without a live session.

## Solution Assessment

Yes. The narrowest maintainable fix is additive logger payload enrichment from `inbound.participantJid`, `inbound.senderE164`, `senderName`, and `enriched.replyContext`, plus regression coverage that avoids raw message logging.
