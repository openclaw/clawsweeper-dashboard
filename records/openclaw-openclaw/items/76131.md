## Summary

Keep open: the PR needs a small correctness repair before merge because the shortcut is broader than the stated configured-model optimization and can bypass provider-owned dynamic resolution for provider-only configs; it also needs the required changelog entry.

## What This Changes

The PR adds a `skipPiDiscovery` fast path in `resolveModelAsync` to return a configured fallback model before dynamic provider preparation and adds regression coverage for that path.

## Best Possible Solution

Keep the first-turn optimization only for exact configured model entries with usable transport, preserve dynamic provider resolution for provider-only configs, and add a changelog plus a regression that fails on current `main`.

## Reproduction Assessment

Yes. A static reproduction path is `skipPiDiscovery` with a configured dynamic provider such as OpenRouter/Ollama/LM Studio where the requested model is not an exact configured model row: the new branch returns a synthetic configured fallback before `prepareProviderDynamicModel` and `runProviderDynamicModel`.

## Solution Assessment

No. The exact configured-model case is already handled by explicit model resolution, while the proposed fallback runs only when no explicit model was found and is therefore broader than the stated performance fix.
