## Summary

Keep open. The latest patch still adds a skipPiDiscovery fallback after explicit configured-model resolution has missed, which can bypass provider dynamic hooks for provider-only OpenRouter, Ollama, or LM Studio configs; the user-facing perf/tool change also still lacks a changelog entry.

## What This Changes

The PR changes first-turn model resolution, PDF tool registration, media-tool factory policy handling, the core tool catalog, and focused agent/tool tests.

## Best Possible Solution

Keep the fast path limited to exact configured model rows, preserve provider dynamic hooks for provider-only or dynamically discovered refs, keep the PDF deferral with focused coverage, and add the required changelog entry.

## Reproduction Assessment

Yes. A high-confidence static reproduction is skipPiDiscovery with a provider-only dynamic config such as OpenRouter, Ollama, or LM Studio where the requested model is not an exact configured row; the PR branch can return the configured fallback before provider dynamic hooks run.

## Solution Assessment

No. The current shortcut runs only after exact configured-model resolution misses; the safer repair is to shortcut only exact configured rows or remove this fallback branch and leave provider-only configs on the dynamic path.
