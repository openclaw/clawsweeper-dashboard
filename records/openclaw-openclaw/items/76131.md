## Summary

Keep open: the latest PR still moves provider-only `skipPiDiscovery` misses ahead of provider dynamic preparation, so dynamic providers can lose provider-owned discovery and metadata; it also still lacks the required changelog entry.

## What This Changes

The PR adds a `skipPiDiscovery` configured-model fallback in `resolveModelAsync` and a DeepSeek regression test for configured model resolution without PI discovery/runtime hooks.

## Best Possible Solution

Keep the fast path limited to exact configured model rows, preserve provider dynamic hooks for provider-only or dynamically resolved model ids, and add focused regression coverage plus a changelog entry.

## Reproduction Assessment

Yes. A high-confidence static reproduction is `skipPiDiscovery` with a configured dynamic provider such as OpenRouter, Ollama, or LM Studio where the requested model is not an exact `models.providers.<id>.models` row; the PR branch can return a synthetic configured fallback before provider dynamic hooks run.

## Solution Assessment

No. The proposed branch changes the no-explicit-model miss path, while the stated configured-row optimization is already handled by current `main`; the safer fix is to shortcut only exact configured rows or reuse the existing explicit-model path.
