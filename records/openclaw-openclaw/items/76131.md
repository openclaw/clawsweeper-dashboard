## Summary

Keep open. The latest head still has functional model-resolution blockers: provider-only configs can bypass provider dynamic discovery, while exact configured model rows still reach provider runtime normalization before the intended fast path can return.

## What This Changes

The PR adds a skipPiDiscovery fast path for configured models, defers PDF auto-model resolution behind tool policy, registers PDF in the media tool catalog/group, adds coverage, and loosens two script declaration signal types.

## Best Possible Solution

Land a focused fast path only for exact configured provider model rows, preserve provider dynamic hooks for provider-level or dynamically discoverable configs, keep the PDF/tool-policy deferral, and add the required changelog/test coverage.

## Reproduction Assessment

Yes. Static review gives a high-confidence reproduction path: call resolveModelAsync with skipPiDiscovery and a provider config that lacks an exact models[] row, then observe the new branch returning generic fallback metadata before prepareProviderDynamicModel can run.

## Solution Assessment

No. The safer implementation is to detect exact configured model rows before provider runtime normalization, while leaving provider-only configs on the existing dynamic provider path.
