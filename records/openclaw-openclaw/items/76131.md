## Summary

Keep open. The performance target is plausible, but the patch moves generic provider-config fallback ahead of provider dynamic model hooks on the first-turn path, which can drop provider-owned metadata for dynamic providers; the added test also does not exercise the new branch, and the user-facing perf change needs a changelog entry.

## What This Changes

The PR adds a `skipPiDiscovery` fast path in `resolveModelAsync` that returns configured provider fallback models before dynamic provider preparation, plus a regression test for a configured DeepSeek model.

## Best Possible Solution

Preserve dynamic-before-generic fallback semantics for provider plugins, narrow any skip path to truly explicit configured model transports, add a regression that fails on current main, and include the required changelog entry.

## Reproduction Assessment

Yes. A source-level reproduction is clear: configure a dynamic provider such as OpenRouter with provider-level transport but no inline model metadata, then call `resolveModelAsync(..., { skipPiDiscovery: true })`; the PR returns the generic fallback before `prepareDynamicModel` can load provider capabilities.

## Solution Assessment

No as written. The optimization should not bypass provider dynamic metadata or reverse the plugin hook contract; a safer fix would gate the fast path more narrowly or add an explicit seam that proves no dynamic model hook is needed.
