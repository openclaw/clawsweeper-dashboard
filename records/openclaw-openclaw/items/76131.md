## Summary

Keep open: the performance direction is useful, but the current diff should not merge because the new model shortcut can bypass provider-owned dynamic resolution and the sessions.list change drops existing default transcript usage fallback behavior.

## What This Changes

This PR changes first-turn model resolution, PDF/media tool registration policy, tool catalog coverage, and sessions.list caching/transcript hydration behavior with regression tests.

## Best Possible Solution

Keep the performance work, but limit any model shortcut to exact configured rows, preserve provider-only refs on the dynamic hook path, keep default session usage fallback semantics unless a separate lightweight API flag is added, and add the required changelog entry.

## Reproduction Assessment

Yes. A static reproduction is skipPiDiscovery with a configured provider but no exact configured model row; the PR can return a synthetic fallback before prepareProviderDynamicModel/runProviderDynamicModel. The sessions.list regression is covered by existing opts:{} transcript fallback tests in src/gateway/session-utils.search.test.ts.

## Solution Assessment

No. The PR broadens the model fast path beyond exact configured rows and changes default sessions.list usage semantics; the safer approach is an exact-row shortcut plus an explicit lightweight session-list path if callers need one.
