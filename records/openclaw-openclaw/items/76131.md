## Summary

Keep open. The PR is directionally useful, but the current diff still introduces blocking regressions in provider dynamic model resolution and cached session listing, plus it changes default session usage behavior and lacks the required changelog entry.

## What This Changes

The branch adds a first-turn configured-model fast path, defers PDF auto model setup, updates media/PDF tool policy coverage, and caches lightweight `sessions.list` results.

## Best Possible Solution

Keep the configured-model fast path limited to exact configured model rows, share session-list cache invalidation across all session-change emitters, preserve transcript usage fallback unless an explicit lightweight mode is added, and add one Unreleased changelog line.

## Reproduction Assessment

Yes. A static reproduction is `skipPiDiscovery` with a configured provider but no exact model row, plus a cached `sessions.list` followed by `agent.ts` create/send events; current tests also prove default `sessions.list` transcript usage fallback.

## Solution Assessment

No. The proposed direction is useful, but the implementation is overbroad and caches mutable session rows without shared invalidation; the safer solution is a narrower fast path and cache invalidation seam that preserves existing session usage semantics.

Existing gateway coverage proves that default list rows currently surface this fallback, so this should remain default behavior or move behind an explicit lightweight flag.
  - confidence: 0.9
