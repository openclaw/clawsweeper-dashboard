## Summary

Keep open. The latest PR no longer contains the earlier gateway session-list cache changes, but the model-resolution change still needs narrowing: it can bypass provider-owned dynamic resolution for provider-only configs, and the exact configured-model case still goes through provider normalization hooks before the new shortcut.

## What This Changes

The PR adds a skipPiDiscovery configured-model fast path, defers PDF auto-model setup behind media tool policy, registers PDF in the media tool catalog/group, and adjusts two script declaration signal types.

## Best Possible Solution

Land a narrower fast path that only short-circuits exact configured model rows before expensive provider hooks, leaves provider-only or dynamically discoverable refs on provider-owned dynamic resolution, keeps the PDF policy changes if tests stay green, and records the performance change in the changelog.

## Reproduction Assessment

Yes. Static reproduction is enough: with skipPiDiscovery and a configured provider that lacks an exact model row, the new branch returns resolveConfiguredFallbackModel before prepareProviderDynamicModel; with an exact configured row, the code still resolves through explicit-model provider hooks before the new shortcut.

## Solution Assessment

No. The safer fix is an exact configured-model shortcut before provider runtime normalization, not a provider-wide generic fallback that can skip provider dynamic discovery and metadata.
