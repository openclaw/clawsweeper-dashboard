## Summary

Keep open; the model-resolution shortcut is still not safe to merge because it can return a provider-wide configured fallback before dynamic provider hooks run. The PDF/tool-policy additions look separable and did not raise a security concern, but the model blocker and missing changelog entry remain.

## What This Changes

The PR adds a skipPiDiscovery fast path for configured first-turn models, defers PDF auto-model resolution during tool registration, threads a media-tool denylist check, adds pdf to the core tool catalog, and adds focused tests.

## Best Possible Solution

Keep the fast path limited to exact configured model rows, preserve dynamic provider hooks for provider-only or dynamically resolved model IDs, keep the PDF deferral only with focused coverage, and add the changelog entry.

## Reproduction Assessment

Yes. A high-confidence static reproduction is skipPiDiscovery with a provider-only dynamic config such as OpenRouter, Ollama, or LM Studio where the requested model is not an exact configured model row; the PR branch can return a synthetic fallback before provider dynamic hooks run.

## Solution Assessment

No. The proposed model shortcut runs on the explicit-model miss path, while current main already handles exact configured rows; the safer fix is to shortcut only exact configured rows or remove this branch and leave provider-only configs on the dynamic path.
