## Summary

Keep open: the latest head still short-circuits provider-only configs before provider dynamic discovery, does not actually avoid provider runtime normalization for exact configured rows, and lacks the required changelog entry.

## What This Changes

The PR adds a skipPiDiscovery model fast path, defers PDF auto-model resolution behind tool policy, registers PDF in the media catalog/group, updates coverage, and loosens two script signal declaration types.

## Best Possible Solution

Land a narrower fast path for exact configured provider model rows before provider runtime hooks, keep provider-only or dynamically discoverable refs on the existing dynamic path, and record the performance/tool-policy change in the changelog.

## Reproduction Assessment

Yes. Static review is enough: the inserted branch returns a generic configured fallback for provider configs without an exact model row, while exact configured rows are resolved earlier through provider runtime normalization before the new shortcut can run.

## Solution Assessment

No. The safer solution is an exact-row shortcut before runtime normalization plus a regression test that provider-only configs still run prepareDynamicModel/resolveDynamicModel before generic fallback.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep provider-only configs on the dynamic path:** `src/agents/pi-embedded-runner/model.ts:1079-1090`
  - body: This shortcut calls `resolveConfiguredFallbackModel` for any configured provider before `prepareProviderDynamicModel`. Configs that only set provider-level transport/auth and rely on dynamic discovery, such as local or OpenAI-compatible providers, can now return generic synthetic metadata and skip provider-owned discovery/cache behavior; restrict the shortcut to an exact configured `models[]` row.

That can load the provider runtime during `skipPiDiscovery`, missing the intended fast path for configured rows.

  - body: This changes user-visible first-turn model and media-tool startup behavior, but the PR does not add a single-line `CHANGELOG.md` entry under Unreleased. Repo policy requires that entry before landing.
  - confidence: 0.9
