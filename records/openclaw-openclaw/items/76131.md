## Summary

Keep open. The PR has useful performance work, but it still introduces blocking regressions in provider-owned dynamic model resolution, session-list freshness/usage semantics, and lacks the required changelog entry.

## What This Changes

The branch adds a first-turn configured-model shortcut, defers PDF auto model prep, updates media tool policy/catalog coverage, and caches lightweight `sessions.list` results.

## Best Possible Solution

Keep the exact configured-model fast path only for real configured model rows, preserve provider-owned dynamic resolution for provider-only refs, make session-list caching invalidation shared across all session-change emitters, retain transcript usage semantics or add an explicit lightweight API flag, and add a single-line changelog entry.

## Reproduction Assessment

Yes. A static reproduction is `skipPiDiscovery` with a configured provider but no exact configured model row, plus a cached `sessions.list` followed by an agent send path that emits from `server-methods/agent.ts`; current tests also show `opts: {}` transcript usage fallback as an existing behavior.

## Solution Assessment

No. The PR is directionally useful, but the current implementation broadens the fast path beyond the configured-model contract and caches full session rows without shared invalidation; the safer fix is narrower and keeps existing usage fallback behavior unless a new lightweight mode is explicit.

Full review comments:

- **[P2] Keep provider-only configs on the dynamic path:** `src/agents/pi-embedded-runner/model.ts:1077-1088`
  - body: This branch runs after exact configured-model resolution misses, but `resolveConfiguredFallbackModel` can synthesize a model for any configured provider. With `skipPiDiscovery`, provider-only OpenRouter/Ollama/LM Studio-style configs can return before provider dynamic hooks run, dropping provider-owned discovery, caches, and capability metadata.
  - confidence: 0.92
- **[P2] Invalidate cached session lists from agent events:** `src/gateway/server-methods/sessions.ts:768-779`
  - body: The new cache is cleared only by the local `emitSessionsChanged` helper in `server-methods/sessions.ts`, but agent sends and creates broadcast `sessions.changed` through a separate helper in `server-methods/agent.ts`. After a cached `sessions.list`, those agent updates can leave the UI and RPC callers seeing stale rows for up to 60 seconds.
  - confidence: 0.88
- **[P2] Preserve transcript-backed usage in default session lists:** `src/gateway/session-utils.ts:1919-1920`
  - body: Setting `transcriptUsageMaxBytes` to `0` unless title or last-message fields are requested makes default `sessions.list` rows stop recovering stale `totalTokens` and `estimatedCostUsd` from transcripts. Existing session-utils coverage and docs describe that fallback as part of the current sparse-session behavior, so this should be an explicit lightweight mode or remain enabled by default.
  - confidence: 0.9

  - body: This PR changes user-visible first-turn model/tool/session-list performance behavior, so repository policy requires a single-line `CHANGELOG.md` entry under Unreleased before landing.
