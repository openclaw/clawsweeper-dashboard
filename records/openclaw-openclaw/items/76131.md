## Summary

Keep open: the performance goal is valid, but the PR still broadens the first-turn shortcut to provider-only configs after exact configured-model resolution has already failed, which can bypass provider dynamic preparation and metadata for OpenRouter, Ollama, and LM Studio; it also lacks the required changelog entry.

## What This Changes

The PR adds a `skipPiDiscovery` fast path in `resolveModelAsync` to return a configured fallback model before dynamic provider preparation, plus coverage for configured DeepSeek model resolution without PI discovery/runtime hooks.

## Best Possible Solution

Keep the fast path limited to exact configured model rows with resolved transport, preserve provider-only configs on dynamic provider hooks, and add focused regression coverage plus a changelog entry.

## Reproduction Assessment

Yes. A high-confidence static reproduction is `skipPiDiscovery` with a configured dynamic provider such as OpenRouter, Ollama, or LM Studio where the requested model is not an exact configured model row; the PR branch returns a synthetic configured fallback before provider dynamic hooks run.

## Solution Assessment

No. The proposed branch uses the configured fallback only after explicit configured-model resolution misses, so it is broader than the stated configured-model optimization; the safer fix is to shortcut only exact configured rows or reuse the existing explicit-model path.
