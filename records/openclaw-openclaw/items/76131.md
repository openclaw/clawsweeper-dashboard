## Summary

Keep open and do not merge as-is. The performance goal is valid, but the model shortcut is too broad: current main already skips dynamic prep for exact configured rows, while the PR can bypass provider dynamic hooks for provider-only dynamic model ids; the PDF/tool-policy behavior also needs docs and changelog follow-through.

## What This Changes

This PR adds a skipPiDiscovery configured-model fallback before dynamic provider preparation, defers PDF tool auto model resolution, makes optional media factory planning honor tool deny policy, adds pdf to the core media catalog, and adds regression tests.

## Best Possible Solution

Repair the PR by limiting the model fast path to exact configured rows, preserving provider dynamic hooks on misses, and documenting/changelogging the PDF/tool-policy behavior.

## Reproduction Assessment

Yes. A high-confidence static reproduction is `skipPiDiscovery` with a configured dynamic provider but no exact `models.providers.<id>.models` row for the requested model; the proposed branch returns a synthetic fallback before dynamic provider hooks can run.

## Solution Assessment

No. The safer solution is to keep current main's exact-row shortcut and only add any new optimization where `findConfiguredProviderModel` actually matched, leaving provider-only dynamic ids on the existing dynamic path.
