## Summary

Keep open. Current main still prebuilds the gateway agent session patch from cached `entry` values and then merges that patch over the fresh store row, so the delivery/channel race described here remains source-provable.

## What This Changes

The issue asks to stop the gateway agent session patch from carrying stale cached `lastChannel`/route/channel fields over fresh session-store values.

## Best Possible Solution

Remove cached delivery/channel fallbacks from the agent handler writeback patch, preserve request-derived delivery seeding, and add stale cached-entry versus fresh-store regression coverage.

## Reproduction Assessment

Yes, at source level. Have `loadSessionEntry` return a cached row without delivery/channel fields, have the writer see a fresh row with those fields, and the current `mergeSessionEntry(store[primaryKey], nextEntryPatch)` path can overwrite the fresh values.

## Solution Assessment

Yes. The narrow maintainable fix is to include only request-derived non-undefined delivery/channel values in the patch, or compute any fallbacks from the fresh row inside `updateSessionStore`.
