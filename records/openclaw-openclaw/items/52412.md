## Summary

Keep open. Current main has gateway-level abort plumbing and hook run IDs, but the requested plugin runtime method is still absent from the public runtime type, runtime builder, SDK test mock, and SDK docs. The related implementation PRs are closed, and #66532 is adjacent rather than an exact replacement.

## What This Changes

Add `api.runtime.agent.abort({ runId })` so plugins can cancel in-progress agent runs by run ID.

## Best Possible Solution

Add an additive, documented plugin runtime cancellation seam in the plugin runtime layer, backed by the existing gateway abort infrastructure with explicit session/run ownership semantics, and update the runtime mock, SDK docs, API/baseline coverage, and targeted gateway/runtime tests. Keep it distinct from #66532 unless maintainers deliberately combine the two cancellation surfaces.

## Reproduction Assessment

Yes. A static current-main check reproduces the missing API: the type, builder, mock, and docs all lack `api.runtime.agent.abort`, while gateway tests show the lower-level abort path exists.

## Solution Assessment

Unclear. The need is valid, but the runId-only/admin-scope shape is not obviously the safest API; a sessionKey-scoped wrapper over the existing abort helper may be the narrower maintainable contract.
