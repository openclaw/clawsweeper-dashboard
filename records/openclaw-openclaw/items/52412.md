## Summary

Keep open: current main still lacks the requested public plugin runtime `api.runtime.agent.abort` API. Hook `runId` values and lower-level gateway cancellation now exist, but exposing cancellation to plugins still needs a public API and ownership/authorization decision.

## What This Changes

The issue requests `api.runtime.agent.abort({ runId })` so plugins can cancel in-progress agent runs by run ID.

## Best Possible Solution

Add an additive, documented plugin runtime cancellation seam backed by existing gateway abort infrastructure, with explicit session/run ownership rules, runtime mock support, SDK docs, API coverage, and targeted runtime/gateway tests.

## Reproduction Assessment

Yes. Static source inspection on current main shows the public runtime type, runtime builder, SDK mock, and docs all lack `api.runtime.agent.abort`, while hook run IDs and lower-level gateway abort plumbing exist.

## Solution Assessment

Unclear. The capability belongs in the plugin runtime layer, but the proposed runId-only/admin-scope shape is not obviously the safest maintainable contract compared with a session-key or ownership-scoped wrapper over the existing abort helper.
