## Summary

Keep open: current main still lacks the requested public plugin runtime cancellation method, even though hook run IDs and gateway-level abort plumbing now exist. The remaining work is a public API and authorization design decision, not a cleanup close.

## What This Changes

Request to add `api.runtime.agent.abort({ runId })` so plugins can cancel in-progress agent runs.

## Best Possible Solution

Add an additive, documented plugin runtime cancellation seam backed by existing gateway abort infrastructure, with explicit session/run ownership rules, runtime mock support, SDK docs, API coverage, and targeted gateway/runtime tests.

## Reproduction Assessment

Yes for the missing API surface: static source inspection on current main shows the public runtime type, builder, SDK mock, and docs all lack `api.runtime.agent.abort`. The lower-level gateway abort path and hook run IDs are present, so the gap is isolated to the plugin runtime contract.

## Solution Assessment

Unclear. The capability belongs in the plugin runtime layer, but the proposed runId-only/admin-scope shape is not obviously the safest maintainable contract compared with a session-key or ownership-scoped wrapper over the existing abort helper.
