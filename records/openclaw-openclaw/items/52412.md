## Summary

Keep open: current main and v2026.6.8 still do not expose the requested injected `api.runtime.agent.abort({ runId })` plugin runtime method. Adjacent public agent-harness SDK abort helpers and Gateway `chat.abort` support cover different cancellation surfaces, and the remaining runId-based plugin authority model needs maintainer product/security review.

## What This Changes

Request to add a plugin runtime `api.runtime.agent.abort({ runId })` API for cancelling in-progress agent runs from plugins.

## Best Possible Solution

Decide the plugin authority model first; if approved, add a narrow gated runtime abort method backed by shared abort logic with SDK docs, API baseline coverage, and focused tests, otherwise document the current Gateway and agent-harness alternatives as the supported boundary.

## Reproduction Assessment

Yes. Source inspection reproduces the API gap: current `api.runtime.agent` types, facade, tests, and docs omit `abort`, while runId hook context and adjacent Gateway cancellation exist separately.

## Solution Assessment

Unclear. The requested shape is plausible, but the best permanent fix needs maintainer judgment around plugin authority, run ownership, session scoping, and whether the existing SDK/Gateway alternatives are the intended contract.
