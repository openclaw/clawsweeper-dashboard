## Summary

Keep open: current main still lacks the requested injected `api.runtime.agent.abort({ runId })` plugin runtime API, and the existing Gateway/session and agent-harness cancellation surfaces do not settle the plugin authority model.

## What This Changes

Request to add `api.runtime.agent.abort({ runId })` so plugins can cancel in-progress agent runs.

## Best Possible Solution

Decide the plugin authority model first; if approved, add a narrow gated runtime abort method backed by shared abort/session logic with SDK docs, API baseline coverage, and focused tests, otherwise document the current supported alternatives.

## Reproduction Assessment

Yes. Source inspection shows current `api.runtime.agent` types, facade, tests, and docs omit `abort`, while hook `runId` and adjacent Gateway cancellation exist separately.

## Solution Assessment

Unclear. The requested shape is plausible, but the best permanent fix needs maintainer judgment around plugin authority, run ownership, session scoping, and whether existing SDK/Gateway alternatives are the intended contract.
