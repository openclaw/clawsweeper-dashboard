- `impact:session-state`: The request is about cancelling stale or redundant in-progress agent runs to avoid stale session behavior.

## Summary

Keep open: current main and v2026.6.8 still do not expose the requested plugin-runtime `api.runtime.agent.abort` method. Adjacent Gateway `chat.abort` and public agent-harness SDK abort helpers now cover some cancellation paths, but they do not settle the requested runId-based injected runtime API or its security model.

## What This Changes

Add a plugin runtime `api.runtime.agent.abort({ runId })` API so plugins can cancel in-progress agent runs.

## Best Possible Solution

Decide the plugin authority model first; if approved, add a narrow gated `api.runtime.agent.abort` backed by the shared abort path with SDK docs, API baseline coverage, and focused tests, or document the current SDK/Gateway alternatives as the supported boundary.

## Reproduction Assessment

Yes. Source inspection reproduces the API gap: current `api.runtime.agent` types, facade, tests, and docs do not expose an abort method, while adjacent Gateway and SDK cancellation paths exist separately.

## Solution Assessment

Unclear. The requested shape is plausible, but the best permanent solution needs maintainer security/product judgment around plugin authority, run ownership, session scoping, and whether the existing SDK/Gateway alternatives are the intended contract.
