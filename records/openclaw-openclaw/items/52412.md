## Summary

Keep open: current main has adjacent gateway/client cancellation through `chat.abort` and public agent-harness abort helpers, but the requested injected `api.runtime.agent.abort` plugin runtime method is still absent and needs a maintainer API/security decision.

## What This Changes

Request to add `api.runtime.agent.abort({ runId })` so plugins can cancel in-progress agent runs by run id.

## Best Possible Solution

Make a maintainer decision: add a narrow, gated runtime abort API with docs, SDK baseline, focused tests, and security review, or document the supported existing cancellation surfaces and then close with that explicit decision.
