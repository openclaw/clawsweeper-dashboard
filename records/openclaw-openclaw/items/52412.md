## Summary

Keep open: current main still lacks the requested injected `api.runtime.agent.abort({ runId })` plugin runtime API. Gateway `sessions.abort`/`chat.abort`, SDK client abort helpers, and agent-harness abort helpers are adjacent cancellation paths, but they do not settle the plugin runtime authority and security model for this new method.

## What This Changes

Add a plugin runtime API that lets trusted plugins abort an in-progress agent run by `runId`.

## Best Possible Solution

Decide the plugin authority model first; if approved, add a narrow gated runtime abort method backed by shared abort logic with SDK docs, API baseline coverage, and focused tests, otherwise document the current Gateway and agent-harness alternatives as the supported boundary.

## Reproduction Assessment

Yes. Source inspection reproduces the API gap: current `api.runtime.agent` types and facade omit `abort`, while runId hook context and adjacent Gateway cancellation exist separately.

## Solution Assessment

Unclear. The requested shape is plausible, but the best permanent fix needs maintainer judgment around plugin authority, run ownership, session scoping, and whether existing Gateway/SDK alternatives are the intended contract.
