## Summary

Keep open: current main and the latest release still lack the requested public plugin runtime `api.runtime.agent.abort` API. Hook `runId` values and lower-level gateway cancellation exist, so the remaining work is an API/ownership decision rather than an already-implemented fix.

## What This Changes

Request to add `api.runtime.agent.abort({ runId })` so plugins can cancel in-progress agent runs.

## Best Possible Solution

Add an additive, documented plugin runtime cancellation seam backed by existing gateway abort infrastructure, with explicit session/run ownership rules, SDK mock support, docs, API coverage, and targeted runtime/gateway tests.

## Reproduction Assessment

Yes. Static source inspection of current main shows the public runtime type, runtime builder, SDK mock, and docs all lack `api.runtime.agent.abort`, while hook run IDs and lower-level gateway abort plumbing exist.

## Solution Assessment

Unclear. The capability belongs in the plugin runtime layer, but the proposed runId-only/admin-scope shape needs maintainer judgment on ownership and authorization before implementation.
