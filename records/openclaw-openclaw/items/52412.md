## Summary

Keep open: current main and the latest release still do not expose the requested public plugin runtime `api.runtime.agent.abort` API. Run IDs and lower-level gateway cancellation are present, so the remaining work is a public API and authorization decision rather than an already-implemented fix.

## What This Changes

The issue requests an additive `api.runtime.agent.abort({ runId })` plugin runtime API so plugins can cancel in-progress agent runs.

## Best Possible Solution

Add a documented, additive plugin runtime cancellation seam backed by the existing abort registry, with explicit run/session ownership rules, SDK mock support, docs, API coverage, and focused runtime/gateway tests.

## Reproduction Assessment

Yes. Static source inspection of current main shows the runtime type, runtime builder, SDK mock, and docs lack `api.runtime.agent.abort`, while hook run IDs and gateway abort plumbing exist.

## Solution Assessment

Unclear. The capability belongs in the plugin runtime layer, but the proposed runId-only/admin-scope shape needs maintainer judgment on ownership and authorization before implementation.
