## Summary

Keep open: current `main` and the latest release still do not expose `api.runtime.agent.abort` on the public plugin runtime. The supporting pieces now exist, including hook `runId` context and gateway abort plumbing, but this remains an additive public plugin API and authorization/session-ownership decision.

## What This Changes

The issue requests an additive public plugin runtime API, `api.runtime.agent.abort({ runId })`, for cancelling in-progress agent runs from plugins.

## Best Possible Solution

Add a documented, additive plugin runtime cancellation seam backed by existing abort plumbing, with explicit run/session ownership rules, SDK mock support, docs, API coverage, and focused runtime/gateway tests.

## Reproduction Assessment

Yes. Static source inspection of current main and `v2026.5.20` shows the runtime type, runtime builder, SDK mock, and docs lack `api.runtime.agent.abort`, while hook run IDs and gateway abort plumbing exist.

## Solution Assessment

Unclear. The capability belongs in the plugin runtime layer, but the proposed runId-only/admin-scope shape needs maintainer judgment on ownership and authorization before implementation.
