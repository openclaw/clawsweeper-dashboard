## Summary

Keep open: current main still lacks the requested public plugin runtime cancellation method. Hook run IDs and gateway-level abort plumbing now exist, but the remaining public API and authorization semantics are still a maintainer product decision.

## What This Changes

The issue requests a public `api.runtime.agent.abort({ runId })` plugin runtime API so plugins can cancel in-progress agent runs.

## Best Possible Solution

Add an additive, documented plugin runtime cancellation seam backed by existing abort infrastructure, with explicit run/session ownership rules, SDK mock support, docs, API coverage, and targeted gateway/runtime tests.

## Reproduction Assessment

Yes for the missing API surface: static source inspection on current main shows the public runtime type, builder, SDK mock, and docs still lack `api.runtime.agent.abort`. The lower-level gateway abort path and hook run IDs are present, so the gap is isolated to the plugin runtime contract.

## Solution Assessment

Unclear. The capability belongs in the plugin runtime layer, but the proposed runId-only/admin-scope shape is not obviously the safest maintainable contract compared with a session-key or ownership-scoped wrapper over the existing abort helper.
