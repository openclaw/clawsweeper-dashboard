## Summary

Keep open: current main and the latest release still do not expose the requested plugin-runtime `api.runtime.agent.abort` method. Adjacent Gateway cancellation now works through `chat.abort`, but adding an in-process plugin abort API remains a product/security decision.

## What This Changes

Request to add `api.runtime.agent.abort({ runId })` so plugins can cancel in-progress agent runs by run id.

## Best Possible Solution

Make a maintainer API/security decision; if approved, add a narrow gated `api.runtime.agent.abort` backed by the shared abort path with SDK docs, API baseline coverage, and focused tests.

## Reproduction Assessment

Yes. Source inspection reproduces the API gap: current `api.runtime.agent` types, facade, tests, and docs do not expose an abort method, while the adjacent Gateway abort path exists separately.

## Solution Assessment

Unclear. The requested shape is plausible, but the best permanent solution needs maintainer security/product judgment around plugin authority, session scope, and gating.
