## Summary

Keep open: current main and the latest release still lack the requested `api.runtime.agent.abort` plugin runtime helper. The lower-level gateway abort path and plugin hook `runId` context exist, but exposing a plugin-facing abort method still needs an API and authorization decision.

## What This Changes

The issue requests an additive plugin runtime API, `api.runtime.agent.abort({ runId })`, so plugins can cancel in-progress agent runs.

## Best Possible Solution

Add an additive, documented plugin runtime cancellation seam backed by the existing abort infrastructure, with explicit run/session ownership rules, SDK mock support, API coverage, docs, and focused runtime/gateway tests.

## Reproduction Assessment

Yes. Static source inspection shows the public runtime type, runtime builder, SDK mock, and docs lack `api.runtime.agent.abort`, while hook `runId` values and lower-level gateway cancellation plumbing exist.

## Solution Assessment

Unclear. The plugin runtime layer is the right boundary for this capability, but the proposed runId-only/admin-scope shape needs maintainer judgment on ownership and authorization before implementation.
