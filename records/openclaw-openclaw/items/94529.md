## Summary

Keep open: current main and the latest release still omit derived-title consumption in the Control UI sidebar/session picker, and the linked implementation PR is open and unmerged.

## What This Changes

Request Control UI sidebar/session picker rows to request and display derived session titles for unlabeled WebChat sessions.

## Best Possible Solution

Land a corrected version of https://github.com/openclaw/openclaw/pull/94544, or an equivalent narrow UI patch, that requests existing Gateway derived titles and renders them in every affected Control UI label path.

## Reproduction Assessment

Yes. Source inspection of current main and v2026.6.8 shows the Control UI does not request includeDerivedTitles and does not render derivedTitle in the relevant label paths, while the Gateway can provide it.

## Solution Assessment

Yes for the narrow direction. The maintainable solution is to consume the existing Gateway includeDerivedTitles contract in Control UI callers and label helpers, not add a separate title-generation backend.
