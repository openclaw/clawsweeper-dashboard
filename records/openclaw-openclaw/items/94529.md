## Summary

Keep open: current main and v2026.6.8 still do not request or render derived titles in the Control UI sidebar/session picker, and the directly linked open PR is the candidate implementation that should be reviewed before the issue closes.

## What This Changes

Control UI should use existing derived session titles so multiple new WebChat sessions are distinguishable in the sidebar and session picker.

## Best Possible Solution

Land or replace the open candidate PR with a narrow UI change that requests `includeDerivedTitles`, preserves label/displayName precedence, updates both sidebar and picker paths, and includes real Control UI proof.

## Reproduction Assessment

Yes at source level: current main exposes Gateway derived titles, but the Control UI sidebar and picker do not request or render them. I did not run a live browser reproduction because this review is read-only.

## Solution Assessment

Yes for the narrow direction: reusing the existing Gateway `includeDerivedTitles` contract in the UI is the maintainable solution. Broader AI summarization and manual renaming should remain separate issues.
