## What This Changes

The issue requests that Control UI session entries use derived conversation titles in the sidebar instead of repeated generic `New Session` labels.

## Best Possible Solution

Land or replace the open candidate PR with a small UI change that requests `includeDerivedTitles`, preserves `label`/`displayName` precedence, updates both sidebar and picker label paths, and includes real Control UI proof.

## Reproduction Assessment

Yes, from source inspection. Current `main` exposes the Gateway option but the Control UI sidebar and picker do not request derived titles, and the shared display resolver does not render `derivedTitle`.

## Solution Assessment

Yes for the narrow direction, but it is not implemented yet. Reusing the existing Gateway `includeDerivedTitles` contract is the maintainable fix; manual renaming and AI title-generation policy should remain separate.
