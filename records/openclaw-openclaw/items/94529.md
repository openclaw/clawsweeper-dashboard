## Summary

Keep open: current main and the latest release still do not wire derived titles into the Control UI sidebar or chat picker, and an open PR already uses closing syntax for this issue.

## What This Changes

Request for the Control UI sidebar/session picker to use existing derived session titles so multiple sessions are distinguishable without manual renaming.

## Best Possible Solution

Review and land or replace the linked Control UI PR, keeping this issue open until derived-title UI wiring is merged.

## Reproduction Assessment

Yes. Source inspection shows the Gateway can return derived titles, but current main does not request them from the Control UI sidebar or chat picker paths and the shared display helper does not render `derivedTitle`.

## Solution Assessment

Yes for the narrow direction. Reusing the existing Gateway `includeDerivedTitles` contract in the Control UI, while preserving explicit label/displayName precedence, is the maintainable fix; broader AI summarization should stay separate.
