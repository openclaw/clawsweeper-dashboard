## Summary

Keep open: current main and v2026.6.10 still do not request or render derived titles in the Control UI sidebar/picker, and https://github.com/openclaw/openclaw/pull/94544 is the open closing PR that should resolve it. The shared SkillHub hook from https://github.com/woohahahaaa/openclaw-auto-rename-session-label is useful as a workaround, but it does not replace the core Control UI wiring.

## What This Changes

The issue requests readable auto-derived session titles in the Control UI sidebar and chat session picker instead of repeated generic session labels.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/94544 with the narrow UI wiring that requests includeDerivedTitles, preserves label/displayName precedence, updates both sidebar and picker label paths, and includes real Control UI proof.

## Reproduction Assessment

Yes, at source level. Current main exposes the Gateway includeDerivedTitles option, but the Control UI sidebar and picker do not request it and the shared display helper does not render derivedTitle.

## Solution Assessment

Yes for the narrow direction, but it is not on main yet. Reusing the existing Gateway includeDerivedTitles contract is the maintainable fix; broader AI title-generation or new config policy should remain separate.
