## Summary

Keep open: current main and the latest release still do not wire existing Gateway derived-title support into the Control UI sidebar or picker, and https://github.com/openclaw/openclaw/pull/94544 is an open closing PR that should be reviewed rather than closing this issue early.

## What This Changes

The issue asks Control UI session tabs to show auto-derived conversation titles instead of repeated generic `New Session` labels until manual rename.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/94544 with a small UI change that requests `includeDerivedTitles`, preserves `label` and `displayName` precedence, updates both sidebar and picker label paths, and includes real Control UI proof.

## Reproduction Assessment

Yes. Source inspection of current main shows the Control UI sidebar and picker do not request `includeDerivedTitles`, while the shared display helper ignores `derivedTitle`; no live UI run was performed in this read-only review.

## Solution Assessment

Yes for the narrow direction. Reusing the existing Gateway `includeDerivedTitles` contract and rendering `derivedTitle` through shared UI label helpers is the maintainable fix; broader AI summarization and settings belong in separate product work.
