## Summary

Keep open: current main and the latest release still do not wire derived titles into the Control UI session sidebar/picker, and https://github.com/openclaw/openclaw/pull/94544 is an open candidate fix using closing syntax for this issue.

## What This Changes

The issue requests that Control UI session tabs use existing derived session titles so multiple WebChat sessions are distinguishable without manual renaming.

## Best Possible Solution

Land or replace the linked PR with a small UI change that requests includeDerivedTitles, renders derivedTitle after label/displayName in both sidebar and picker label paths, and includes real Control UI proof.

## Reproduction Assessment

Yes. Source inspection shows current main’s Control UI loader and picker request sessions.list without includeDerivedTitles, and the shared display helper only considers label/displayName before fallback.

## Solution Assessment

Yes for the narrow direction, but it is not implemented on main yet. Reusing the existing Gateway includeDerivedTitles contract is the maintainable fix; broader AI-generated title policy should remain separate.
