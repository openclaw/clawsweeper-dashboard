## Summary

Keep open: current main and the latest release still do not request or render derived titles in the Control UI sidebar or chat picker, and the open closing PR at https://github.com/openclaw/openclaw/pull/94544 should be reviewed, fixed, and proven before this issue closes.

## What This Changes

The issue requests Control UI sidebar and WebChat session entries to show a short derived session title instead of repeated generic labels like `New Session`.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/94544 with narrow Control UI wiring that requests `includeDerivedTitles`, preserves `label`/`displayName` precedence, updates both sidebar and picker label paths, and includes real UI proof.

## Reproduction Assessment

Yes, at source level. Current main exposes `includeDerivedTitles` in the Gateway, but the Control UI sidebar and chat picker do not request it and the shared display helper does not render `derivedTitle`.

## Solution Assessment

Yes for the narrow direction, but not by closing yet. Reusing the existing Gateway `includeDerivedTitles` contract is the maintainable fix; broader AI title generation or new config should stay separate.
