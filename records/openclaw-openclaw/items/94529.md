## What This Changes

Request the Control UI sidebar and chat session picker to request and display derived session titles instead of generic or raw session labels.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/94544 with the narrow Control UI wiring that requests includeDerivedTitles, preserves label/displayName precedence, and proves both sidebar and picker rendering.

## Reproduction Assessment

Yes, at source level. Current main exposes includeDerivedTitles in the gateway, but the Control UI sidebar and picker do not request it and the shared display helper does not render derivedTitle.

## Solution Assessment

Yes for the narrow direction. Reusing the existing gateway-derived title contract in Control UI is the maintainable fix; broader AI summarization or new configuration should remain separate.
