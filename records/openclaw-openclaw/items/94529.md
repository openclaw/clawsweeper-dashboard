## What This Changes

The issue asks Control UI to use existing Gateway-derived session titles for sidebar/WebChat session labels instead of indistinguishable default names.

## Best Possible Solution

Land or replace the open candidate PR with a small Control UI change that requests `includeDerivedTitles`, preserves label/displayName precedence, renders `derivedTitle` in both sidebar and picker labels, and includes real UI proof.

## Reproduction Assessment

Yes. Source inspection shows current `main` and `v2026.6.8` expose the Gateway derived-title option, but the Control UI sidebar/picker do not request it and the shared display helper does not render `derivedTitle`.

## Solution Assessment

Yes for the narrow direction. Reusing the existing Gateway `includeDerivedTitles` contract and rendering `derivedTitle` after explicit labels is the maintainable fix; broader AI title-generation policy should stay separate.
