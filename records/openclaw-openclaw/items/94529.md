## What This Changes

Request that Control UI session lists show existing derived session titles for unlabeled WebChat sessions instead of repeated generic labels.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/94544 with narrow Control UI wiring that requests includeDerivedTitles, renders derivedTitle after label/displayName, covers both sidebar and picker paths, and includes real UI proof.

## Reproduction Assessment

Yes, at source level. Current main's Control UI session loader and chat picker request sessions.list without includeDerivedTitles, and resolveSessionDisplayName only considers label/displayName before falling back to key-derived text.

## Solution Assessment

Yes for the narrow direction. Reusing the existing Gateway includeDerivedTitles contract in Control UI is the maintainable fix; AI summarization or new config should stay separate.
