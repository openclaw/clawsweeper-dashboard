## What This Changes

The issue asks the Control UI sidebar and session picker to show readable auto-derived session titles instead of generic New Session labels.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/94544 with the narrow Control UI wiring that requests includeDerivedTitles, preserves label/displayName precedence, updates both sidebar and picker label paths, and includes real UI proof.

## Reproduction Assessment

Yes, at source level. Current main exposes the Gateway includeDerivedTitles option, but the Control UI sidebar and picker do not request it and the shared display helper does not render derivedTitle.

## Solution Assessment

Yes for the narrow direction, but it is not implemented on main yet. Reusing the existing Gateway includeDerivedTitles contract is the maintainable fix; broader AI summarization or new config should stay separate.
