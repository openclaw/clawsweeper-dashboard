## Summary

Keep this PR open, but it is not merge-ready: the patch updates secret metadata to auto-select a keyless search provider, while the actual `web_search` runtime still filters auto-detected keyless providers out, so the central no-key `web_search` behavior remains broken.

## What This Changes

The PR makes Firecrawl `web_search` keyless when selected, marks Firecrawl search credentials optional, enables keyless web-search auto-selection metadata, expands `firecrawl_search` options, and updates docs/tests.

## Best Possible Solution

Align the product policy and runtime path before merge: either keep keyless Firecrawl search explicit/opt-in, or intentionally wire and document a no-key default provider order with real OpenClaw behavior proof.
