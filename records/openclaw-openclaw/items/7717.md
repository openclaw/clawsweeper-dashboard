- `impact:message-loss`: Role-only Discord messages that users intend to address to agents are currently dropped by mention gating instead of being delivered to the agent workflow.

## Summary

Keep open. Current main still has no Discord role-mention trigger config and still treats `mentionedRoles` as generic mention metadata rather than comparing mentioned role IDs to configured trigger roles before mention-gated messages are dropped.

## What This Changes

Request to add Discord role-mention triggers, such as a `triggerOnRoles` role-ID config, so role mentions can wake bots under Discord mention gating.

## Best Possible Solution

Add an explicit Discord plugin role-trigger surface after maintainer approval, compare mentioned role IDs before mention gating, preserve sender `roles` allowlist semantics, and cover matched, unmatched, and `ignoreOtherMentions` cases in tests and docs.

## Reproduction Assessment

Yes, at source level: current preflight tracks `mentionedRoles` but never compares those IDs with configured trigger roles before `requireMention` and `ignoreOtherMentions` gates. I did not run live Discord proof because this is a read-only review and the source path is clear.

## Solution Assessment

Unclear until maintainers choose the public semantics. The narrow maintainable direction is an additive role-trigger config that does not overload the existing sender `roles` allowlist.
