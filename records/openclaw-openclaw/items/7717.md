## Summary

Keep open. Current main still has no Discord role-trigger config/schema/docs surface, and preflight records role mentions only as generic mention metadata without comparing them to configured trigger roles before mention-required messages can be dropped.

## What This Changes

The issue requests support for Discord role mentions, such as a configured `triggerOnRoles` role-ID list, to wake mention-gated agents.

## Best Possible Solution

Add an explicit Discord plugin role-trigger config after maintainer approval, compare mentioned role IDs in preflight, preserve existing sender `roles` allowlist semantics, and cover matched, unmatched, bot-managed-role, and `ignoreOtherMentions` cases in tests and docs.

## Reproduction Assessment

Yes, at source level: current preflight tracks `mentionedRoles` but never compares them with configured trigger roles before the mention-required and `ignoreOtherMentions` gates. I did not run a live Discord repro in this read-only pass.

## Solution Assessment

Unclear until maintainers approve the public config and default behavior. The narrow maintainable direction is an additive role-trigger field that does not reuse the existing sender `roles` allowlist.
