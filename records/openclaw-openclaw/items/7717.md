## Summary

Keep open. Current main and v2026.5.20 still lack a Discord role-mention wake trigger; role mentions are recorded as generic mention metadata, while the remaining public config semantics need maintainer product review.

## What This Changes

Request support for Discord role mentions, such as a configured role ID list, to wake mention-gated bots instead of requiring direct bot user mentions.

## Best Possible Solution

Add an explicit Discord plugin role-trigger config after maintainer approval, compare mentioned role IDs before mention gating, preserve sender roles allowlist semantics, and update types, schema, docs, tests, and live Discord proof.

## Reproduction Assessment

Yes, at source level: current preflight tracks mentionedRoles but never compares those IDs with configured trigger roles before requireMention and ignoreOtherMentions gates. I did not run live Discord proof in this read-only review.

## Solution Assessment

Unclear until maintainers choose the public semantics. The narrow maintainable direction is an additive role-trigger config that does not overload the existing sender roles allowlist.
