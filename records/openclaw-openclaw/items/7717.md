## Summary

Keep open. Current main still records Discord role mentions as metadata but does not expose a role-trigger config or promote matching role mentions into the explicit wake mention state.

## What This Changes

Request Discord role mentions such as @AI to wake Discord bots, preferably through a configured role-ID trigger list.

## Best Possible Solution

Add an explicit Discord role-trigger config surface in the bundled Discord plugin, then wire it through types, strict schema, metadata/help, docs, preflight, and focused tests while preserving the existing sender roles allowlist contract.

## Reproduction Assessment

Yes, at source level. A guild message with requireMention true, no bot user mention, and only mentionedRoles populated has no current path to become wasMentioned, then reaches the no-mention or other-mention drop path.

## Solution Assessment

Unclear until maintainers choose the public Discord semantics. The maintainable direction is an additive role-trigger option; overloading roles would conflict with the current sender allowlist contract.
