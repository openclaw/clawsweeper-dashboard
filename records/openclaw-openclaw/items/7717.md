## Summary

Keep open: current main still lacks a Discord role-mention wake trigger. Role IDs are preserved as mention metadata and existing `roles` config is a sender allowlist, but no configured mentioned-role IDs are accepted or fed into `wasMentioned`; prior related reports route the remaining narrow work here.

## What This Changes

Request support for Discord role mentions to wake agents, likely through a role-ID config such as `triggerOnRoles`.

## Best Possible Solution

Add a maintainer-approved Discord plugin role-trigger config, wire mentioned-role ID matching into preflight wake detection, document it, and preserve current sender `roles` allowlist semantics.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source inspection gives a high-confidence current behavior path: role IDs are preserved as mention metadata, but no configured trigger role IDs are accepted or fed into `wasMentioned`.

## Solution Assessment

Yes in direction, but the exact public config name, defaults, inheritance, and bot-managed-role behavior need maintainer choice before implementation. The narrow maintainable path is an additive Discord plugin config plus preflight role-ID matching, not reuse of the existing sender allowlist key.
