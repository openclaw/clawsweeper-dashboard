## Summary

Keep open: current main still lacks configurable Discord role-mention triggers. The related reports at https://github.com/openclaw/openclaw/issues/59151 and https://github.com/openclaw/openclaw/issues/74967 were closed in favor of this canonical tracker, and source inspection still shows role mentions are metadata/other-mention signals rather than wake triggers.

## What This Changes

Request support for Discord role mentions to wake OpenClaw bots, likely via an additive Discord guild/channel configuration option.

## Best Possible Solution

Add an explicit Discord plugin guild/channel role-trigger setting, keep sender roles as allowlists, and cover matched, unmatched, bot-managed-role, and ignoreOtherMentions role-only cases in config and preflight tests.

## Reproduction Assessment

Not applicable: this is a feature request for a new Discord config surface, not a regression against an established contract. Source inspection still gives high-confidence behavior evidence that role-only mentions populate mentionedRoles but are not converted into wasMentioned.

## Solution Assessment

Yes in direction: an additive Discord-plugin config and preflight change is the narrow maintainable path. The exact field name, default, guild/channel inheritance, and bot-managed-role behavior need maintainer judgment before implementation.
