## Summary

Current main still does not implement Discord role-mention triggers; role mentions are recorded as generic mention metadata and can participate in other-mention filtering, but they are not a configured wake condition.

## What This Changes

Add Discord role mentions, such as configured role IDs, as triggers that wake mention-gated Discord bots.

## Best Possible Solution

Add a separate Discord guild/channel role-trigger config surface, validate and document it, and update preflight so matching `message.mentionedRoles` counts as an explicit wake mention while preserving `requireMention`, `ignoreOtherMentions`, sender allowlists, and existing `@everyone` behavior.

## Reproduction Assessment

Yes, from current-main source inspection: a role-only Discord mention can populate `message.mentionedRoles`, but that data is not passed into `wasMentioned`, and the strict schema has no role-trigger config key. I did not run tests because this was a read-only review and `pnpm` is unavailable in this environment.

## Solution Assessment

No patch is under review. The requested config-based direction is plausible, but the maintainable fix should add a distinct trigger-role surface instead of changing the existing sender allowlist `roles` semantics.
