## Summary

Keep open: current main still lacks Discord role-mention wake triggers, and the narrower role-mention reports were closed in favor of this issue rather than implementing the behavior. This is a Discord plugin feature/config request that still needs maintainer product decisions before repair automation.

## What This Changes

Request support for configured Discord role mentions, such as shared agent roles, to wake mention-gated Discord bots.

## Best Possible Solution

Add a maintainer-approved Discord plugin role-trigger config, wire it through schema/docs/generated config metadata, and cover matched, unmatched, bot-managed-role, and `ignoreOtherMentions` role-only cases in preflight tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source inspection gives high-confidence current-main evidence that role-only mentions are tracked as generic mentions but do not set `wasMentioned`.

## Solution Assessment

Yes in direction: an additive Discord plugin config plus preflight comparison against mentioned role IDs is the narrow maintainable path. The exact field name, defaults, inheritance, and bot-managed-role behavior should be chosen by maintainers first.
