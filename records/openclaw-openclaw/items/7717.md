## Summary

Keep open: current main still lacks a Discord role-mention wake config and only treats role mentions as metadata for generic mention detection and ignore-other-mentions filtering. Prior reviews and related closed reports route the narrow role-trigger work here, while the open access-control redesign is broader and not a replacement.

## What This Changes

Request to add Discord role-mention triggers so configured role IDs can wake mention-gated Discord bots.

## Best Possible Solution

Add a maintainer-approved Discord plugin role-trigger config and wire role-ID matching through config types, strict schema, generated metadata/docs, preflight, and focused tests while preserving sender roles allowlists.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source inspection gives a high-confidence current behavior path: role IDs are preserved as mention metadata, but no configured trigger role IDs are accepted or fed into wasMentioned.

## Solution Assessment

Yes in direction: an additive Discord plugin config plus preflight role-ID matching is the narrow maintainable path. The exact field name, defaults, inheritance, and bot-managed-role behavior need maintainer choice before implementation.
