## Label Justifications

- `P2`: This is a bounded Discord workflow enhancement with meaningful multi-agent impact but no emergency or core-runtime outage.
- `impact:message-loss`: Role-addressed Discord messages are suppressed by mention gating before they can reach the intended agents.

## Summary

Current main and v2026.6.5 both lack configurable role-trigger activation, and related reports explicitly preserve this issue as the canonical tracker. The remaining work is valid but requires a maintainer product decision before implementation.

## What This Changes

Add guild/channel Discord role-ID triggers that treat matching role mentions as explicit agent wake mentions.

## Best Possible Solution

Add a Discord-owned, opt-in selected-role trigger at guild and channel scope, feed matching mentioned role IDs into the canonical mention-state decision, and preserve sender `roles`, direct mentions, patterns, replies, @everyone/@here, routing, and `ignoreOtherMentions` semantics with focused schema, preflight, and docs coverage.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: send a guild message containing only a role mention with `requireMention` enabled; current preflight records the role as a mention but cannot turn it into `wasMentioned`.

## Solution Assessment

Yes in direction, but not yet in final contract: a selected-role opt-in integrated into the existing mention-state path is the narrowest maintainable solution, while reusing sender `roles` or enabling all bot roles implicitly would conflate policy and activation.
