## Summary

Keep open. Current main and the latest shipped tag still lack a Discord role-mention trigger surface, and role mentions are still only generic mention metadata rather than configured wake signals.

## What This Changes

Request support for Discord role mentions so configured role IDs can wake agents in guild channels.

## Best Possible Solution

Add an explicit Discord plugin role-trigger config after maintainer approval, compare mentioned role IDs before mention gating, preserve sender roles allowlist semantics, and cover matched, unmatched, and ignoreOtherMentions cases in tests and docs.

## Reproduction Assessment

Yes, at source level: current preflight tracks mentionedRoles but never compares those IDs with configured trigger roles before requireMention and ignoreOtherMentions gates. I did not run live Discord proof in this read-only review.

## Solution Assessment

Unclear until maintainers choose the public semantics. The narrow maintainable direction is an additive role-trigger config that does not overload the existing sender roles allowlist.
