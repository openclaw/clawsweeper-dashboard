## Summary

Keep open: current main and v2026.6.8 still do not provide configurable Discord role-mention activation, and the related closed role-mention reports point back to this issue as the canonical tracker.

## What This Changes

The issue requests configurable Discord role-mention triggers so role mentions can wake matching bot instances instead of requiring direct bot user mentions.

## Best Possible Solution

Add a Discord-owned, opt-in selected-role trigger at guild and channel scope, feed matching mentioned role IDs into the existing mention-state path, and cover schema, docs, preflight, and `ignoreOtherMentions` behavior with focused tests after maintainers confirm the contract.

## Reproduction Assessment

Yes, at source level: Discord role mentions are preserved as `mentionedRoles`/`mention_roles`, but current config does not accept trigger-role IDs and preflight never compares mentioned role IDs before `requireMention` can skip the message. I did not run a live Discord replay in this read-only review.

## Solution Assessment

Unclear as a finalized product contract: a selected-role opt-in integrated into the current Discord mention-state path is the narrowest maintainable direction, but maintainers still need to choose naming, inheritance, and fan-out behavior. Reusing sender `roles` would be the wrong layer because that key already means sender access control.
