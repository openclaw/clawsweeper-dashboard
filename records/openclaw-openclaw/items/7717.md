## Label Justifications

- `impact:message-loss`: Messages addressed to agents through Discord role mentions can be suppressed by mention gating before agent processing.

## Summary

Current main and v2026.6.5 still treat Discord role mentions only as generic mention metadata, not as configurable agent wake triggers. This remains the canonical narrow tracker because the closer role-mention reports were closed in its favor, while the broader access-control proposal does not yet define or implement addressed-role activation.

## What This Changes

Add configurable Discord role IDs whose mentions wake mention-gated agents in guild channels.

## Best Possible Solution

Define an additive Discord guild/channel trigger-role list with explicit inheritance, compare it with `message.mentionedRoles` in the existing mention-decision path, and preserve the current sender `roles`, @everyone/@here, reply, routing, and `ignoreOtherMentions` semantics.

## Reproduction Assessment

Yes, from source with high confidence: send a guild message containing only a role mention while mention gating is enabled; current main records the role mention but has no role-ID activation input, so it cannot become `wasMentioned`.

## Solution Assessment

Yes at the implementation boundary, but the public API still needs a decision: extending the single existing mention-decision path is the narrowest maintainable fix, while maintainers must choose the config name, inheritance, and safe defaults.
