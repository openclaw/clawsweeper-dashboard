## Label Justifications

- `P2`: This is a normal-priority Discord plugin enhancement with a clear source-level gap and limited blast radius, but it needs product semantics before implementation.
- `impact:message-loss`: Role-only Discord messages that users intend to address to agents are currently dropped by mention gating instead of reaching the agent workflow.

## Summary

Keep open. Current main still treats Discord role mentions as generic mention metadata rather than a configurable wake trigger, and the linked closed reports explicitly point back to this issue as the canonical tracker.

## What This Changes

Request support for Discord role mentions to wake mention-gated agents, likely through an explicit Discord role-ID trigger config.

## Best Possible Solution

Keep this as the canonical Discord role-mention trigger tracker and implement an additive, explicitly named role-trigger config in the bundled Discord plugin after maintainers choose the public semantics.

## Reproduction Assessment

Yes, at source level: a role-only Discord guild message can populate `mentionedRoles`, but current preflight never compares those role IDs with configured trigger roles before `requireMention` and `ignoreOtherMentions` gates. I did not run live Discord proof in this read-only review.

## Solution Assessment

Unclear until maintainers choose the public config name, defaults, and relationship to the broader Discord access-control redesign. The narrow maintainable direction is an additive role-trigger config that does not overload the existing sender `roles` allowlist.
