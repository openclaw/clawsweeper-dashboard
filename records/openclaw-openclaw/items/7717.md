## Summary

Keep open. Current main still lacks a Discord role-mention trigger surface, and role mentions are only treated as generic mention metadata before mention gating can drop role-only messages.

## What This Changes

Add Discord role-mention triggers so configured role mentions can wake agents in mention-gated guild channels.

## Best Possible Solution

Add an explicit Discord plugin role-trigger config after maintainer approval, wire it through schema/docs/preflight, and preserve existing sender `roles` allowlist behavior.

## Reproduction Assessment

Yes, at source level: current preflight tracks `mentionedRoles` but never compares those IDs with configured trigger roles before `requireMention` and `ignoreOtherMentions` gates. I did not run live Discord proof in this read-only review.

## Solution Assessment

Unclear until maintainers choose the public semantics. The narrow maintainable direction is an additive role-trigger config that does not overload the existing sender `roles` allowlist.
