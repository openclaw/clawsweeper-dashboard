## Summary

Keep open: current main and v2026.6.10 still lack a Discord role-mention trigger surface, and preflight still treats role mentions as generic mention metadata rather than a configured wake signal. This issue remains the narrow canonical tracker; related closed reports point back here, while broader Discord routing/access-control issues only partially overlap.

## What This Changes

The issue requests Discord guild role mentions to act as configured wake triggers for agents, likely through an additive guild/channel configuration surface.

## Best Possible Solution

Approve a narrow Discord-owned selected-role trigger contract, then wire it through config types, strict schema, docs/metadata, preflight, and regression coverage while preserving current sender role allowlists and mass-mention behavior.

## Reproduction Assessment

Yes, at source level: with `requireMention` true and only `mentionedRoles` present, `explicitlyMentioned` remains false and the mention-state helper has no role-ID input before the no-mention drop path. I did not run a live Discord replay in this read-only review.

## Solution Assessment

Unclear as a finalized product contract: selected configured role IDs appear safer than automatically waking on every bot-held role, but maintainers need to choose naming, inheritance, and fan-out behavior before implementation.
