## Summary

Keep open. Current main still lacks a Discord role-mention trigger surface, and role IDs are only treated as generic mention metadata before mention gating.

## What This Changes

Add support for configured Discord role mentions to wake agents, so role-only mentions can trigger responses like direct bot mentions.

## Best Possible Solution

Keep this as the canonical Discord role-mention trigger tracker and implement an additive, explicitly named role-trigger config in the Discord plugin after maintainers choose the public semantics.

## Reproduction Assessment

Yes, at source level: a role-only Discord guild message can populate mentionedRoles, but current preflight never compares those role IDs with configured trigger roles before requireMention and ignoreOtherMentions gates. I did not run live Discord proof in this read-only review.

## Solution Assessment

Unclear until maintainers choose the public config name, defaults, and relationship to the broader Discord access-control redesign. The narrow maintainable direction is an additive role-trigger config that does not overload the existing sender roles allowlist.
