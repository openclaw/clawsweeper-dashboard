## Summary

Keep open: current main still lacks a configurable Discord role-mention trigger; role IDs are only sender allowlists/routing metadata, and adding this requires a public config decision.

## What This Changes

Request support for Discord role mentions, via a guild/channel role-ID config, to wake mention-gated bots.

## Best Possible Solution

Add an explicit Discord guild/channel role-trigger setting in the bundled Discord plugin and wire it through schema, metadata/help, docs, and focused preflight tests while preserving existing mention-gating semantics.

## Reproduction Assessment

Not applicable: this is a feature request for a new Discord config surface, not a regression against an established contract. Source inspection gives high confidence that current main does not implement the requested role-trigger behavior.

## Solution Assessment

Yes: the maintainable direction is an additive Discord-plugin config and preflight change, not reusing the existing sender roles allowlist. The remaining decision is the public config name and defaults.
