## Summary

Keep open. Current main still records Discord role mentions as generic mention metadata, but there is no role-trigger config and role IDs are not passed into the mention-state path that decides whether a guild message wakes the bot.

## What This Changes

Requests an opt-in Discord guild/channel config so selected mentioned role IDs wake the bot like direct mentions.

## Best Possible Solution

Add an opt-in Discord guild/channel role-trigger list in the bundled Discord plugin and wire it through config, schema, docs, metadata, preflight, and regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a guild message with requireMention true, no bot/user mention, and a matching ID in mentionedRoles will leave wasMentioned false and be skipped as no-mention.

## Solution Assessment

Yes. The narrow maintainable solution is an additive trigger-role config plus preflight wiring; overloading the existing roles key would conflate wake triggers with sender access control.
