## Summary

Keep open: current main and v2026.6.8 still lack configurable Discord role-mention wake behavior. The closed role-mention reports point back here, while the broader open Discord access-control redesign only partially overlaps and does not replace this selected addressed-role trigger request.

## What This Changes

This issue requests Discord role mentions, such as a configured `@AI` role, to wake bots instead of requiring direct bot user mentions.

## Best Possible Solution

Add an opt-in selected-role trigger in the Discord plugin, wire it through schema/docs/preflight, and test matched, unmatched, and `ignoreOtherMentions` role-only messages.

## Reproduction Assessment

Yes, at source level: a Discord guild message containing only a role mention while `requireMention` is true leaves `wasMentioned` false because mentioned role IDs are never compared with configured trigger role IDs. I did not run a live Discord replay in this read-only review.

## Solution Assessment

Yes in direction, but the final contract is not settled: selected role IDs integrated into the existing mention-state path are the narrowest maintainable shape. Reusing sender `roles` or enabling all bot roles implicitly would mix access control with activation and could over-trigger multi-agent servers.
