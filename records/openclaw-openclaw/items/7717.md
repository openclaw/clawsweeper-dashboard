## Summary

Keep open. Current main still lacks a Discord role-mention trigger config or runtime path; the existing `roles` fields are sender allowlists, and the related broader access-list proposal remains a separate product/design discussion rather than a replacement for this specific request.

## What This Changes

Request Discord role mentions such as `@AI` to wake agents, likely via an opt-in role-ID config like `triggerOnRoles`.

## Best Possible Solution

Add an opt-in Discord plugin role-trigger config and wire it through types, strict schema, generated docs/help, preflight, and focused tests while preserving existing sender `roles` allowlist semantics.

## Reproduction Assessment

Yes, by source inspection. A guild message with `requireMention: true`, no bot user mention, and `mentionedRoles` populated counts as generic mention metadata but never becomes `wasMentioned`, so the current preflight reaches the no-mention drop path.

## Solution Assessment

Unclear until maintainers decide the public Discord config semantics. The narrowest maintainable direction appears to be an additive `triggerOnRoles`-style option rather than overloading the existing sender `roles` allowlist.
