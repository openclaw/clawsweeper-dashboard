## Summary

Keep open. Current main still lacks a Discord role-mention trigger config or runtime path; related role-mention reports were closed in favor of this canonical tracker, and the broader access-list proposal is not a replacement for this narrower feature.

## What This Changes

Request support for Discord role mentions, such as configured role IDs, to wake bots in mention-gated guild channels.

## Best Possible Solution

Add an opt-in Discord plugin role-trigger config and wire it through types, strict schema, generated docs/help, preflight, and focused tests while preserving existing sender roles allowlist semantics.

## Reproduction Assessment

Yes, by source inspection. A guild message with requireMention true, no bot user mention, and mentionedRoles populated remains generic mention metadata and reaches the no-mention drop path.

## Solution Assessment

Unclear until maintainers decide the public Discord config semantics. The narrowest maintainable direction appears to be an additive triggerOnRoles-style option rather than overloading the existing sender roles allowlist.
