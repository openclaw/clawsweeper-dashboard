## Summary

Keep open. Current main and the latest v2026.5.22 release still do not implement Discord role-mention wake triggers; the closed related reports point back to this issue, while the open broader Discord access-list proposal is adjacent but not a direct replacement for the requested mentioned-role trigger behavior.

## What This Changes

Request support for Discord role mentions such as `@AI` or `@bots` to wake OpenClaw bots, ideally through configurable trigger role IDs.

## Best Possible Solution

Add an explicit bundled Discord plugin role-trigger surface, or consciously fold that behavior into the approved Discord access-control redesign, with config/schema/docs and preflight tests for matched, unmatched, and `ignoreOtherMentions` role-only messages.

## Reproduction Assessment

Yes for the reported current behavior via source inspection: a role-only Discord message can populate `mentionedRoles`, but explicit wake detection only checks `mentionedUsers`, so `requireMention` can still drop it as `no-mention`. I did not run a live Discord repro in this read-only review.

## Solution Assessment

Unclear until maintainers choose the public config and default semantics. The safest maintainable direction is an additive trigger-role field separate from existing sender `roles`, or an explicit decision to fold it into the broader Discord access-list design.
