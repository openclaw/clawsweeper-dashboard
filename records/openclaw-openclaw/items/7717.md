## Summary

Keep open: current main still lacks a configurable Discord role-mention trigger path, and this is a valid enhancement that needs maintainer agreement on the public config shape before implementation.

## What This Changes

Request configurable Discord role mentions, such as `triggerOnRoles`, that wake the bot when a guild message mentions one of the configured role IDs.

## Best Possible Solution

Add an explicit Discord guild/channel trigger-role setting, feed mentioned role IDs into mention resolution, and update schema, metadata/help, docs, and focused tests while preserving existing sender `roles`, @everyone/@here, `requireMention`, and `ignoreOtherMentions` semantics.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is an enhancement for a new Discord config surface. Source inspection gives high confidence that current main does not implement the requested role-trigger behavior.

## Solution Assessment

Yes: the narrow maintainable direction is an additive Discord-plugin config and preflight change, not reusing the existing sender `roles` allowlist or moving this to ClawHub. Maintainers still need to choose the public field name and defaults.
