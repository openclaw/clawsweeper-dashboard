## Summary

Keep open. Current main still has no Discord role-mention trigger config or runtime path, and the closed related reports point back to this issue as the canonical tracker rather than resolving it.

## What This Changes

The issue requests Discord role mentions, such as `@AI`, to wake agents via an opt-in role-ID trigger configuration instead of requiring direct bot user mentions.

## Best Possible Solution

Add an explicit Discord role-trigger config and preflight behavior after maintainers confirm the public semantics, preserving current sender `roles` allowlists and documenting the interaction with `requireMention` and `ignoreOtherMentions`.

## Reproduction Assessment

Yes. Source inspection shows role-only Discord mentions populate `mentionedRoles`, but they are not passed into the mention-state helper as explicit wake triggers, so a guild message with `requireMention: true` and no bot user mention reaches the no-mention drop path.

## Solution Assessment

Unclear until maintainers decide the public config semantics. The likely maintainable direction is an additive role-ID trigger option, not overloading existing sender `roles` allowlists.
