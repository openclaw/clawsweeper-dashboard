## Summary

Keep open: current main still lacks a Discord role-mention wake config and only treats role mentions as generic mention metadata. Prior review comments and closed related reports route the remaining role-trigger work here, while https://github.com/openclaw/openclaw/issues/69748 is broader Discord access-control design rather than a replacement.

## What This Changes

Request Discord role mentions such as @AI to wake agents through a configurable guild/channel role-ID trigger surface.

## Best Possible Solution

Add a maintainer-approved Discord plugin role-trigger config and wire role-ID matching through config types, strict schema, generated metadata/docs, preflight, and focused tests while preserving sender roles allowlists.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source inspection gives a high-confidence current behavior path: mentionedRoles is preserved as metadata, but no configured trigger role IDs are accepted or fed into wasMentioned.

## Solution Assessment

Yes in direction: an additive Discord plugin config plus preflight role-ID matching is the narrow maintainable path. The exact field name, defaults, inheritance, and bot-managed-role behavior need maintainer choice before implementation.
