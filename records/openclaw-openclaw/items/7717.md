## Summary

Keep open. Current `main` and the latest release still lack a Discord role-mention trigger surface; role mentions are captured as metadata but are never compared with configured trigger role IDs before mention gating can drop role-only messages.

## What This Changes

Request support for Discord role mentions to wake mention-gated Discord bots, likely through an additive role-ID trigger configuration.

## Best Possible Solution

Keep this as the canonical Discord role-mention trigger tracker and implement an additive Discord plugin config only after maintainers settle its relationship with the broader access-control design at https://github.com/openclaw/openclaw/issues/69748.

## Reproduction Assessment

Yes, at source level: a role-only Discord guild message can populate `mentionedRoles`, but current preflight never compares those role IDs with configured trigger roles before `requireMention` and `ignoreOtherMentions` gates. I did not run live Discord proof in this read-only review.

## Solution Assessment

Unclear until maintainers choose the public config name, defaults, and relationship to broader Discord access-control work. The narrow maintainable direction is an additive role-trigger config that does not overload the existing sender `roles` allowlist.
