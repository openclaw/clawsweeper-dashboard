## Summary

Keep open. Current main still lacks an opt-in Discord role-mention trigger path, and the linked broader access-list proposal does not clearly supersede this narrower request because it changes sender access semantics rather than role-mention wake semantics.

## What This Changes

The issue asks for Discord guild/channel configuration that lets selected role mentions wake the bot the way direct user mentions do.

## Best Possible Solution

Add a Discord plugin role-trigger configuration surface and wire it through config types, strict schema, docs/help metadata, preflight, and focused tests without overloading the existing sender roles allowlist.

## Reproduction Assessment

Yes, by source inspection. A guild message with requireMention true, no bot user mention, and mentionedRoles populated remains generic mention metadata and reaches the no-mention path instead of becoming wasMentioned.

## Solution Assessment

Unclear until maintainers decide the public Discord semantics. The narrow maintainable direction is an additive role-trigger option, while reusing the existing roles key would conflict with sender allowlist behavior.
