## Summary

Keep open. Current main still records Discord role mentions only as generic mention metadata; there is no role-trigger config/schema/docs surface, and the mention gate never compares mentioned role IDs with configured trigger roles before dropping mention-required messages.

## What This Changes

Request adding Discord role-mention trigger support so role-only mentions can wake mention-gated Discord bots, likely through a guild/channel role-ID config.

## Best Possible Solution

Add an explicit Discord plugin role-trigger surface after maintainer approval, compare mentioned role IDs in preflight, preserve sender roles allowlist semantics, and cover matched, unmatched, and ignoreOtherMentions cases in tests and docs.

## Reproduction Assessment

Yes, at source level: current preflight tracks mentionedRoles but never compares them with configured trigger roles before mention-required and ignoreOtherMentions gates. I did not run a live Discord repro.

## Solution Assessment

Unclear until maintainers approve the public config and default behavior. The narrow maintainable direction is an additive role-trigger config that does not overload the existing sender roles allowlist.
