## Summary

Keep open. Current main still has no Discord role-mention trigger config or runtime path: role mentions are recorded only as generic mention metadata, while the mention gate still wakes on explicit bot/user mentions, configured mention patterns, @everyone/@here, or reply-to-bot state; related closed reports were already redirected here rather than resolving the remaining role-trigger request.

## What This Changes

Request Discord role mentions for configured role IDs to wake mention-gated bots, likely via an additive guild/channel config option such as `triggerOnRoles`.

## Best Possible Solution

Add an explicit Discord guild/channel role-trigger surface in the bundled Discord plugin after maintainer approval, preserving existing sender `roles` allowlist semantics and covering role-only preflight/drop cases in tests and docs.

## Reproduction Assessment

Yes, at source level: current preflight counts role mentions as generic mention metadata but never compares mentioned roles with configured trigger roles before the mention gates. I did not run a live Discord repro.

## Solution Assessment

Unclear until maintainers decide the public config and default behavior. The narrow maintainable direction is an additive role-trigger config that does not overload the existing sender `roles` allowlist.
