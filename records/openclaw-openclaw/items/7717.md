## Label Justifications

- `impact:message-loss`: Role-only Discord messages that users intend to address to agents are currently dropped by mention gating instead of reaching the agent workflow.

## Summary

Keep open. Current main and the latest shipped release still treat Discord role mentions as generic mention metadata, not as a configurable wake trigger, and the related closed reports point back to this issue as the canonical role-mention tracker.

## What This Changes

The issue requests Discord role-mention triggers so configured Discord role mentions can wake agents instead of requiring direct bot mentions.

## Best Possible Solution

Add an explicit Discord plugin role-trigger config after maintainer approval, compare mentioned role IDs before mention gating, preserve sender `roles` allowlists, and update types, strict schema, docs, focused preflight tests, and live Discord proof.

## Reproduction Assessment

Yes, at source level: a role-only Discord guild message can populate `mentionedRoles` without `mentionedUsers`, and current preflight never compares those role IDs with configured trigger roles before `requireMention` and `ignoreOtherMentions` gates. I did not run live Discord proof in this read-only review.

## Solution Assessment

Unclear until maintainers choose the public semantics. The narrow maintainable direction is an additive role-trigger config that does not overload the existing sender `roles` allowlist.
