## Summary

Keep open: current main still lacks configurable Discord role-mention triggers. Role mentions are tracked as generic mention metadata and for other-mention filtering, while the wake decision still comes from explicit bot mentions, mention patterns, @everyone/@here, or reply-to-bot state.

## What This Changes

The issue requests a Discord guild/channel config option that lets selected role mentions wake the bot like a direct mention.

## Best Possible Solution

Add an explicit Discord guild/channel trigger-role config in the bundled Discord plugin, feed mentioned role IDs into preflight mention resolution, and update schema, generated metadata/help, docs, and focused tests while preserving existing sender roles, @everyone/@here, requireMention, and ignoreOtherMentions behavior.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is an enhancement request for a new config surface. Source inspection still gives high confidence that current main does not implement the requested role-trigger behavior.

## Solution Assessment

Yes, the narrow maintainable direction is an additive Discord-plugin config and preflight change, not reusing the existing roles allowlist or moving this to ClawHub. The exact field name and default behavior need maintainer product judgment first.
