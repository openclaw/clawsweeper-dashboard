## Summary

Current main still lacks any configured-role activation path: Discord records role mentions only as generic mention metadata, while explicit wake state remains limited to direct bot mentions, mention patterns, @everyone/@here, and reply context. This issue is the canonical narrow tracker; the broader open access-control proposal does not yet implement or safely supersede it, and maintainers still need to choose the permanent config semantics.

## What This Changes

Add a Discord guild/channel configuration that lets selected role mentions wake mention-gated agents.

## Best Possible Solution

Keep this issue as the canonical role-activation tracker, then add one clearly named additive Discord guild/channel role-ID setting with schema, help/docs, and focused preflight coverage after maintainers approve its interaction with sender access controls.

## Reproduction Assessment

Yes, from source with high confidence: role mentions enter generic mention detection, but no current config or mention-resolution path compares those roles with an activation list. A live Discord reproduction was not run because the missing path is directly observable in current main.

## Solution Assessment

Unclear until maintainers choose the contract. A dedicated additive trigger-role list is narrower and safer than reusing sender `roles`, but it must be coordinated with the open broader Discord access-control design.
