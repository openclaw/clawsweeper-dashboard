Keep open. Current main and v2026.6.5 still lack a Discord role-mention trigger surface, and this remains the focused tracker while the broader Discord access-control design stays unresolved.

## What This Changes

Requests an additive Discord role-mention trigger so configured role mentions such as @AI can wake mention-gated agents.

## Best Possible Solution

Keep this as the focused Discord role-mention trigger tracker and land an additive bundled Discord plugin config only after maintainers settle semantics with the broader access-control design.

## Reproduction Assessment

Yes, at source level: a role-only Discord guild message can populate `mentionedRoles`, but current preflight never compares those IDs with bot or configured trigger roles before mention gating can skip the message.

## Solution Assessment

Unclear until maintainers choose the public config name, defaults, and relationship to broader Discord access-control work. The narrow maintainable direction is an additive role-trigger config that does not overload the existing sender `roles` allowlist.
