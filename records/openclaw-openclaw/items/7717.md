## Summary

Keep open. Current main still lacks a Discord role-mention trigger config and wake path: role mentions are recorded as mention metadata, but they are not matched against configured trigger role IDs or promoted into the explicit mention state.

## What This Changes

Requested Discord support for configured role mentions, such as an @AI role, to wake agents in addition to direct bot mentions.

## Best Possible Solution

Add an explicit Discord role-trigger config surface in the bundled Discord plugin and wire it through config types, strict schema, generated metadata/help, docs, preflight, and focused tests while preserving the existing sender roles allowlist contract.

## Reproduction Assessment

Yes, at source level. A guild message with requireMention true, no bot user mention, and only mentionedRoles populated has no current path to become wasMentioned, then reaches the no-mention drop path.

## Solution Assessment

Unclear until maintainers choose the public Discord semantics. The maintainable direction is an additive role-trigger option; overloading roles would conflict with the current sender allowlist contract.
