Keep open: current main and v2026.6.10 still lack a selected Discord role-mention trigger config and preflight path, and this issue remains the narrow canonical tracker while broader Discord access-control and routing issues only partially overlap.

## What This Changes

Add Discord role-mention triggers so configured role IDs can wake mention-gated bots in guild channels.

## Best Possible Solution

Approve a narrow Discord-owned selected-role trigger contract, then wire it through config, schema, metadata/docs, preflight, and focused tests without reusing the existing sender roles allowlist.

## Reproduction Assessment

Yes, at source level: with requireMention true and only mentionedRoles present, explicitlyMentioned is false and role IDs are not passed into resolveDiscordMentionState, so the no-mention or ignoreOtherMentions drop paths can run. No live Discord replay was run.

## Solution Assessment

Unclear as a final product contract: configured selected role IDs appear safer than auto-triggering on every bot-held role, but maintainers still need to decide the exact config name, inheritance, and fan-out behavior.
