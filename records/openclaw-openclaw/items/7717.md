## Summary

Keep open. Current main still treats Discord role mentions as generic mention metadata, with no public role-trigger config and no comparison between mentioned role IDs and configured trigger roles before mention-gated messages are dropped.

## What This Changes

The issue requests Discord role mentions to act as configurable wake triggers for agents, rather than only direct bot user mentions.

## Best Possible Solution

Add an explicit Discord plugin role-trigger surface after maintainer approval, compare mentioned role IDs in preflight, preserve sender `roles` allowlist semantics, and cover matched, unmatched, and `ignoreOtherMentions` cases in tests and docs.

## Reproduction Assessment

Yes, at source level: current preflight tracks `mentionedRoles` but never compares those IDs with configured trigger roles before `requireMention` and `ignoreOtherMentions` gates. I did not run live Discord proof because the source path is clear and this review is read-only.

## Solution Assessment

Unclear until maintainers approve the public config and default behavior. The narrow maintainable direction is an additive role-trigger config that does not overload the existing sender `roles` allowlist.
