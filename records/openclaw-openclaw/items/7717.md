## Summary

Keep open: current main still lacks Discord role-mention wake triggers. The existing Discord `roles` config is sender access control, role mentions are only generic mention metadata, and related role-mention reports were closed in favor of this canonical tracker.

## What This Changes

Request support for Discord role mentions, optionally configured by role ID, to wake OpenClaw bots in guild channels.

## Best Possible Solution

Add a maintainer-approved Discord plugin role-trigger config, preserve sender `roles` as access control, and cover matched, unmatched, bot-managed-role, and `ignoreOtherMentions` role-only cases in schema, docs, and preflight tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request for a new Discord config and activation surface. Source inspection gives high-confidence behavior evidence that role-only mentions populate `mentionedRoles` but are not converted into `wasMentioned`.

## Solution Assessment

Yes in direction: an additive Discord-plugin config plus preflight comparison against mentioned role IDs is the narrow maintainable path. The exact field name, defaults, guild/channel inheritance, and bot-managed-role behavior need maintainer decision before implementation.
