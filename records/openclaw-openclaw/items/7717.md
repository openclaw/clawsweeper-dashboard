## Summary

Keep open: current main still lacks a configurable Discord role-mention trigger path, and the existing role surfaces are sender allowlists/routing metadata rather than wake triggers.

Add a Discord guild/channel setting that lets configured role mentions trigger bot responses.

## Best Possible Solution

Add an explicit Discord plugin trigger-role config surface, pass mentioned role IDs into preflight mention resolution, and update schema, generated config metadata, docs, and focused regression tests while preserving existing sender roles semantics.
