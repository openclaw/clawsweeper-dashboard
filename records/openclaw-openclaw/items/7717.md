## Summary

Keep open. Current main still has no Discord role-trigger config, and the Discord preflight treats role mentions as generic mention metadata rather than comparing mentioned role IDs to configured trigger roles before mention-gated messages are dropped.

## What This Changes

The issue requests Discord role mentions such as `@AI` to wake bot instances, likely through a per-guild/channel role ID trigger config.

## Best Possible Solution

Keep this issue open as the canonical role-mention trigger tracker, approve the Discord config semantics, then implement an additive Discord plugin role-trigger surface with schema, docs, and focused preflight coverage.

## Reproduction Assessment

Yes, at source level: current preflight tracks `mentionedRoles` but never compares those IDs with configured trigger roles before `requireMention` and `ignoreOtherMentions` gates. I did not run live Discord proof because this review is read-only and the source path is clear.

## Solution Assessment

Unclear until maintainers approve the public config and default behavior. The narrow maintainable direction is an additive role-trigger config that does not overload the existing sender `roles` allowlist and coordinates with the broader access-list proposal.
