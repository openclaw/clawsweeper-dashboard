## Summary

Keep open: current main and v2026.6.10 still lack a Discord role-mention trigger config or preflight path, and this issue remains the narrow canonical tracker while broader Discord routing/access-control issues only partially overlap.

## What This Changes

Requested support for Discord role mentions, configured by role ID, to wake mention-gated Discord agents when a message mentions a role assigned to the bot.

## Best Possible Solution

Approve a narrow Discord-owned selected-role trigger contract, then wire it through config, schema, docs/metadata, preflight, and focused regression tests without reusing the existing sender `roles` allowlist.

## Reproduction Assessment

Yes, at source level: with `requireMention` true and only `mentionedRoles` present, the explicit mention check remains false and the no-mention drop path can run before any role-trigger policy exists. I did not run a live Discord replay in this read-only review.

## Solution Assessment

Unclear as a final product contract: configured selected role IDs appear safer than auto-triggering on every bot-held role, but maintainers still need to decide the exact config name, inheritance, and fan-out behavior.
