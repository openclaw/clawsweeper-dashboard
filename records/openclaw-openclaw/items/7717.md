## Summary

Keep open: current main and v2026.6.10 still lack a Discord role-mention trigger surface, and Discord preflight still treats role mentions as generic mention metadata rather than a configured wake signal. This issue remains the narrow canonical tracker; related closed reports point back here, while broader Discord routing/access-control issues only partially overlap.

## What This Changes

Request to let configured Discord role mentions wake OpenClaw agents, likely through a new guild/channel role-ID trigger option.

## Best Possible Solution

Approve a narrow Discord-owned selected-role trigger contract, then wire it through config types, strict schema, generated metadata/help, docs, preflight, and focused tests while preserving current direct mentions, @everyone/@here, sender allowlists, and `ignoreOtherMentions` behavior.

## Reproduction Assessment

Yes, at source level: with `requireMention` true and only `mentionedRoles` present, `explicitlyMentioned` remains false and `resolveDiscordMentionState` has no role-ID input, so the message can reach the no-mention drop path. I did not run a live Discord replay in this read-only review.

## Solution Assessment

Unclear as a finalized contract: selected configured role IDs appear safer than automatically waking on every bot-held role, but maintainers still need to choose naming, inheritance, and fan-out behavior before implementation.
