## Summary

Keep open. Current main still lacks the requested Discord role-mention trigger config and wake path: role mentions are captured as generic mention metadata, but they are not compared to configured trigger role IDs or promoted into explicit mention state.

## What This Changes

The issue requests an additive Discord guild/channel configuration option that lets selected role mentions wake an agent, in addition to direct bot mentions.

## Best Possible Solution

Add an explicit Discord role-trigger config surface and wire it through config types, schema/help metadata, docs, preflight, and tests without reusing the existing sender roles allowlist key.

## Reproduction Assessment

Yes, by source inspection. A guild message with requireMention true, no bot user mention, and only mentionedRoles populated still has no current path to become wasMentioned.

## Solution Assessment

Unclear until maintainers choose the public Discord semantics. The maintainable direction is an additive role-trigger option; overloading roles would conflict with the current sender allowlist contract.
