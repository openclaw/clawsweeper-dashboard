## Summary

Keep open: current main still lacks a Discord role-mention trigger surface, and the related broader Discord access-control proposal is open product work rather than a merged replacement for this narrower request.

## What This Changes

The issue requests an additive Discord configuration option so role mentions such as `@AI` can wake agents that have the matching Discord role.

## Best Possible Solution

Define an additive Discord role-mention trigger contract in the bundled Discord plugin, keep it separate from sender role allowlists, and align it with the broader access-control proposal before implementation.

## Reproduction Assessment

Yes, from source. A role-only Discord guild message can populate `mentionedRoles`, but current preflight never compares those IDs with configured trigger roles before `requireMention` and `ignoreOtherMentions` gates.

## Solution Assessment

Unclear until maintainers choose the config contract. A separate additive trigger-role field is safer than reusing `roles`, but it should be reconciled with the broader Discord access-control redesign before code changes.
