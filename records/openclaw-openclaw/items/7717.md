## Summary

Keep open: current main still lacks configurable Discord role-mention triggers, and the remaining work is an additive Discord plugin config/product decision rather than cleanup that can be closed.

## What This Changes

Request support for Discord role mentions, likely via a `triggerOnRoles`-style guild/channel config array, so selected role pings can wake bot instances.

## Best Possible Solution

Add an explicit Discord guild/channel role-trigger setting in the bundled Discord plugin, wire it through config types, strict schema, metadata/help, docs, and preflight tests, and keep existing sender `roles` allowlist semantics separate.

## Reproduction Assessment

Not applicable: this is a feature request for a new Discord config surface, not a regression against an established contract. Source inspection on current main shows role-only mentions are tracked as `mentionedRoles` but are not converted into `wasMentioned`.

## Solution Assessment

Yes in direction: an additive Discord-plugin config and preflight change is the narrow maintainable path. The exact field name, default, and whether bot-managed roles should wake automatically still need maintainer judgment.
