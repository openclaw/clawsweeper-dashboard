## Summary

Keep open. Current main still lacks a Discord role-mention trigger config or preflight path; role mentions are tracked as generic mention metadata and can still be dropped by mention gating instead of becoming an explicit wake signal.

## What This Changes

The issue requests configurable Discord role-mention triggers so messages mentioning selected Discord role IDs can wake mention-gated agents.

## Best Possible Solution

Add an additive Discord guild/channel config surface such as triggerOnRoles?: string[] in the bundled Discord plugin, wire it through config types, strict schema, generated metadata/help, docs, and preflight, and treat matching message.mentionedRoles values as explicit wake mentions while preserving existing sender allowlists and mention-gating behavior.

## Reproduction Assessment

Yes. A current-main code-level reproduction is a guild message with requireMention: true, no mentionedUsers, and mentionedRoles containing the desired role ID: preflight will set explicitlyMentioned false, pass no role IDs into resolveDiscordMentionState, and then skip as no-mention.

## Solution Assessment

Yes. The narrow maintainable solution is an opt-in Discord plugin config plus preflight wiring; reusing the existing roles key would be wrong because it already means sender role allowlist.
