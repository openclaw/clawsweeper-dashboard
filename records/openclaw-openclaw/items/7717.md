## Summary

Keep open: current main still lacks configurable Discord role-mention triggers, and the request remains a Discord plugin feature/config decision rather than an already implemented cleanup candidate.

## What This Changes

Request support for Discord role mentions, such as configured role IDs, to wake agents in addition to direct bot mentions.

## Best Possible Solution

Add an explicit Discord guild/channel role-trigger setting in the bundled Discord plugin and wire it through config types, strict schema, metadata/help, docs, and focused preflight tests while preserving existing sender role allowlists.

## Reproduction Assessment

Not applicable: this is a feature request for a new Discord config surface, not a regression against an established contract. Source inspection gives high confidence that current main does not implement the requested role-trigger behavior.

## Solution Assessment

Yes in direction: an additive Discord-plugin config and preflight change is the narrow maintainable path. The exact public field name and defaults still need maintainer judgment.
