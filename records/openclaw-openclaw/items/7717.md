## Summary

Keep open. Current main and the latest shipped tag still lack a Discord role-mention trigger config or role-ID comparison in mention gating; related narrower reports were closed in favor of this issue, while https://github.com/openclaw/openclaw/issues/69748 is a broader access-control redesign rather than a complete replacement for this specific tracker.

## What This Changes

Request support for configured Discord role mentions, such as @AI, to wake agents in mention-gated guild channels.

## Best Possible Solution

Add an explicit Discord plugin role-trigger config after maintainer approval, compare mentioned role IDs before mention gating, preserve sender roles allowlist semantics, and cover matched, unmatched, ignoreOtherMentions, schema, docs, and live Discord proof.

## Reproduction Assessment

Yes, at source level: current preflight tracks mentionedRoles but never compares those IDs with configured trigger roles before requireMention and ignoreOtherMentions gates. I did not run live Discord proof in this read-only review.

## Solution Assessment

Unclear until maintainers choose the public semantics. The narrow maintainable direction is an additive role-trigger config that does not overload the existing sender roles allowlist.
