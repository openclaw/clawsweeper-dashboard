## Label Justifications

- `P2`: This is a normal-priority Discord plugin enhancement with a clear source-level gap and limited blast radius, but it needs maintainer product semantics before implementation.
- `impact:message-loss`: Role-only Discord messages that users intend to address to agents are currently dropped by mention gating instead of reaching the agent workflow.

## Summary

Keep open. Current main and v2026.5.22 still lack a Discord role-mention trigger config or runtime path; role mentions are captured as message metadata but are not compared with configured wake role IDs before mention gating can drop the message.

## What This Changes

Request adding Discord role-mention triggers so configured role mentions can wake the bot or agents instead of requiring individual user mentions.

## Best Possible Solution

Keep this as the canonical Discord role-mention trigger tracker and implement an additive Discord plugin config only after maintainers settle its relationship with the broader Discord access-control design at https://github.com/openclaw/openclaw/issues/69748.

## Reproduction Assessment

Yes, at source level: a role-only Discord guild message can populate mentionedRoles, but current preflight never compares those IDs with configured trigger roles before requireMention and ignoreOtherMentions gates. I did not run live Discord proof in this read-only review.

## Solution Assessment

Unclear until maintainers choose the public config name, defaults, and relationship to broader Discord access-control work. The narrow maintainable direction is an additive role-trigger config that does not overload the existing sender roles allowlist.
