## Summary

Keep open: current main still lacks a Discord role-mention wake config and does not compare mentioned role IDs against configured trigger roles before mention gating. This remains the canonical tracker for that feature rather than a cleanup-close candidate.

## What This Changes

Requested support for configured Discord role mentions to wake mention-gated Discord agents.

## Best Possible Solution

Add a maintainer-approved Discord plugin role-trigger config and wire role-ID matching through config types, strict schema, generated metadata, docs, hydration/preflight, and focused tests while preserving sender `roles` allowlists.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source inspection gives a high-confidence current behavior path: role IDs are preserved as mention metadata but never compared with configured trigger IDs before `wasMentioned`, and no such config field exists.

## Solution Assessment

Yes in direction: an additive Discord plugin config plus preflight role-ID matching is the narrow maintainable path. The exact field name, defaults, inheritance, and bot-managed-role behavior need maintainer choice before implementation.
