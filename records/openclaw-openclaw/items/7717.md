## Summary

Keep open. Current main still lacks a Discord role-mention trigger surface and still computes wake mentions from user mentions, mention text patterns, non-bot @everyone/@here, and reply-to-bot state, while the related closed reports point back to this issue as the canonical tracker rather than proving it fixed.

## What This Changes

Add support for configured Discord role mentions to wake mention-gated bots when a message mentions one of those roles.

## Best Possible Solution

Keep this as an open Discord plugin design task and add an explicit, documented role-trigger config only after maintainers settle the public schema and interaction with requireMention, ignoreOtherMentions, sender allowlists, and the broader access-list proposal.

## Reproduction Assessment

Not applicable as a bug reproduction because this is an additive feature/config request. Source inspection gives high confidence that current main lacks the trigger config and does not feed mentioned role IDs into wasMentioned.

## Solution Assessment

Unclear until maintainers choose the config contract. A narrow triggerOnRoles-style field is plausible, but the safer implementation must avoid overloading existing sender roles and coordinate with the broader Discord access-control proposal.
