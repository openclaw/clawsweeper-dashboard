## Summary

Keep open: current main and v2026.6.6 still do not provide configurable Discord role-mention activation, while related reports preserve this issue as the canonical tracker for the remaining role-trigger behavior.

## What This Changes

Request support for Discord role mentions to wake mention-gated agents, likely through an additive guild/channel role-trigger configuration surface.

## Best Possible Solution

Add a Discord-owned, opt-in selected-role trigger at guild and channel scope, feed matching mentioned role IDs into the existing mention-state path, and cover schema, docs, preflight, and ignoreOtherMentions behavior with focused tests.

## Reproduction Assessment

Yes, at source level: send a Discord guild message containing only a role mention while requireMention is true; preflight records mentionedRoles but never turns a matching role into wasMentioned. I did not run a live Discord replay in this read-only review.

## Solution Assessment

Yes in direction, but not yet as a finalized contract: a selected-role opt-in integrated into the existing mention-state path is the narrowest maintainable shape. Reusing sender roles or enabling all bot roles implicitly would mix access control with activation and could over-trigger multi-agent servers.
