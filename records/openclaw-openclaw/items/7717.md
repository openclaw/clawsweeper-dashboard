## Summary

Keep open: current main still lacks a Discord role-mention wake config and does not feed mentioned role IDs into the mention gate. The existing related closed reports point back to this issue as the canonical tracker, while the broader open Discord access-control redesign does not replace this narrower role-trigger request.

## What This Changes

The issue requests Discord role mentions to trigger bot responses, likely through an additive role-ID config such as `triggerOnRoles`.

## Best Possible Solution

Add a maintainer-approved Discord plugin role-trigger config and wire role-ID matching through config types, strict schema, docs/metadata, preflight, and focused tests while preserving sender `roles` allowlists.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. Source inspection gives a high-confidence current behavior path: role IDs are preserved as metadata, but no configured trigger role IDs are accepted or fed into `wasMentioned`.

## Solution Assessment

Yes in direction: an additive Discord plugin config plus preflight role-ID matching is the narrow maintainable path. The exact field name, defaults, inheritance, and bot-managed-role behavior need maintainer approval before implementation.
