## Summary

Keep open: current main and v2026.6.6 still lack configurable Discord role-mention wake activation, while related closed reports preserve this issue as the canonical tracker and the broader access-control proposal only partially overlaps.

## What This Changes

Feature request for configurable Discord role mentions to wake agents when a message mentions a selected role assigned to the bot or agent group.

## Best Possible Solution

Add a Discord-owned opt-in selected-role trigger surface after maintainer approval, wiring it into the existing mention-state decision with schema, docs, and focused preflight tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: send a guild message containing only a role mention with `requireMention` enabled; current preflight records the role as mention metadata but cannot turn it into `wasMentioned`, so the mention gate can drop it.

## Solution Assessment

Yes in direction, but not yet as an automation task. A selected-role opt-in integrated into the existing mention-state path is the narrowest maintainable solution; reusing sender `roles` or enabling all bot-held roles implicitly would conflate access control with wake activation.
