## Label Justifications

- `P2`: This is a normal-priority Discord plugin feature with a clear source-level gap, limited blast radius, and required product decision.
- `impact:message-loss`: Role-only Discord messages that users intend to address to agents are currently dropped by mention gating instead of reaching the agent workflow.

## Summary

Keep open. Current main still has no Discord role-mention trigger surface, and role mentions are still only generic mention metadata before the mention gate drops role-only messages in mention-gated guild channels.

## What This Changes

The issue requests Discord role mentions such as `@AI` or `@bots` to wake agents, likely through a new per-guild or per-channel role-trigger configuration.

## Best Possible Solution

Add an explicit Discord plugin role-trigger surface after maintainer approval, compare mentioned role IDs before mention gating, preserve sender `roles` allowlist semantics, and cover matched, unmatched, and `ignoreOtherMentions` cases in tests and docs.

## Reproduction Assessment

Yes, at source level: current preflight tracks `mentionedRoles` but never compares those IDs with configured trigger roles before `requireMention` and `ignoreOtherMentions` gates. I did not run live Discord proof in this read-only review.

## Solution Assessment

Unclear until maintainers choose the public semantics. The narrow maintainable direction is an additive role-trigger config that does not overload the existing sender `roles` allowlist.
