## Summary

Keep open: current main and the latest release still do not provide configurable Discord role-mention activation, and the related closed reports point back to this issue as the canonical tracker for the remaining role-trigger behavior.

## What This Changes

Request to add Discord role-mention wake triggers, likely through guild/channel role-ID config, so messages that mention selected roles can wake agents.

## Best Possible Solution

Add a Discord-owned, opt-in selected-role trigger at guild and channel scope, feed matching mentioned role IDs into the existing mention-state path, and cover schema, docs, preflight, and ignoreOtherMentions behavior with focused tests.

## Reproduction Assessment

Yes, at source level: a Discord guild message containing only a role mention while requireMention is true leaves wasMentioned false because mentionedRoles is never compared with configured trigger role IDs. I did not run a live Discord replay in this read-only review.

## Solution Assessment

Yes in direction, but not as a finalized contract: a selected-role opt-in integrated into the existing mention-state path is the narrowest maintainable shape. Reusing sender roles or enabling all bot roles implicitly would mix access control with activation and could over-trigger multi-agent servers.
