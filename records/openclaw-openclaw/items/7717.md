## Summary

Keep open: current `origin/main` still lacks a Discord role-mention wake config, and preflight does not compare mentioned role IDs against configured trigger roles before `wasMentioned`. The narrower role-mention reports were closed in favor of this issue, while https://github.com/openclaw/openclaw/issues/69748 is broader access-control design work rather than a replacement.

## What This Changes

Request adding Discord role-mention wake triggers, likely via guild/channel role-ID config, so messages mentioning configured roles can activate agents.

## Best Possible Solution

Add a maintainer-approved Discord plugin role-trigger config and wire it through types, strict schema, generated config metadata, docs, hydration/preflight handling, and focused tests while preserving sender `roles` allowlists.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature/config request. Source inspection gives a high-confidence current behavior path: role IDs are tracked as mention metadata but never compared with configured trigger IDs before `wasMentioned`, and no such config field exists.

## Solution Assessment

Yes in direction: an additive Discord plugin config plus preflight role-ID matching is the narrow maintainable path. The exact field name, defaults, inheritance, and managed-role behavior need maintainer choice before implementation.
