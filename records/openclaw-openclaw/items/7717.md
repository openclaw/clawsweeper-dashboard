## Summary

Keep open. Current main still lacks a Discord role-mention trigger config or preflight path; role mentions are tracked as generic mention metadata but do not become an explicit wake signal, and the related closed issues do not implement the remaining requested behavior.

## What This Changes

Requests an additive Discord guild/channel option to wake the bot when messages mention configured Discord role IDs, supporting multi-bot role pings.

## Best Possible Solution

Add an opt-in Discord guild/channel role-trigger config, wire it through config/schema/docs/metadata and preflight, and treat matching mentioned role IDs as explicit mentions while preserving existing mention-gating and sender allowlist behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a guild message with `requireMention: true`, empty `mentionedUsers`, and a matching role ID in `mentionedRoles` will not set `wasMentioned`, so the shared mention gate skips it as no-mention.

## Solution Assessment

Yes. The narrow maintainable solution is an additive Discord plugin config plus preflight wiring; overloading the existing `roles` key would conflate trigger behavior with sender access control.
