## Summary

Keep open: current main and v2026.6.5 still lack a Discord role-mention trigger surface, and role mentions are only tracked as generic mention metadata or sender-role allowlists rather than wake triggers. The broader Discord access-control proposal at https://github.com/openclaw/openclaw/issues/69748 is related but does not replace this narrower role-trigger tracker.

## What This Changes

Request to let configured Discord role mentions such as @AI wake agents in guild channels instead of only direct bot mentions.

## Best Possible Solution

Keep this as the canonical Discord role-trigger tracker and implement an additive Discord plugin config only after maintainers settle its relationship with the broader access-control design.

## Reproduction Assessment

Yes, at source level: a role-only Discord guild message can populate mentionedRoles, but current preflight never compares those role IDs with configured trigger roles before requireMention and ignoreOtherMentions gates. I did not run live Discord proof in this read-only review.

## Solution Assessment

Unclear; the requested direction is useful, but the public config name, defaults, and relationship to the broader access-control proposal need maintainer decision. The narrow maintainable path is an additive role-trigger surface that does not reuse the existing sender roles allowlist.
