## Summary

Keep open. The PR is aimed at a real session_status selected-override display gap, but the current diff still regresses existing runtime identity behavior, adds a test assertion that does not match the local mock payload, and leaves an accidental root-level patch artifact in the branch.

## What This Changes

The PR changes session_status model resolution to prefer session override and runtime model fields, adds one regression test, updates CHANGELOG.md, and still adds a root-level patch-fragment file.

## Best Possible Solution

Keep selected/default identity and active runtime identity separated: pass explicit session overrides as the selected model when present, preserve the current runtime identity resolver and providerless legacy synthesis when no override exists, remove the artifact file, and make the focused regression test assert the actual status-builder contract.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists from current-main tests for providerful, custom-provider, and providerless runtime sessions, plus the new test assertion mismatch against the unchanged mock payload.

## Solution Assessment

No. The current PR direction is too broad because it fixes explicit overrides by replacing the runtime resolver; the safer fix is a narrow precedence adjustment that keeps the existing runtime-resolution behavior intact.
