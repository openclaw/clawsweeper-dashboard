## Summary

Keep open. Current main has adjacent multi-agent routing docs but no equivalent standalone orchestration-patterns guide, so the PR is still useful as the docs follow-up to #43034. The branch is not merge-ready because its copyable examples conflict with current agent-to-agent/session visibility semantics, and the new page is not connected to docs navigation.

## What This Changes

The PR adds `docs/concepts/multi-agent-patterns.md`, a standalone guide for hub-and-spoke routing, cost-tiered routing, channel-bound specialists, heartbeat monitoring, pipeline processing, and anti-patterns.

## Best Possible Solution

Keep the PR as the canonical docs follow-up, but revise the guide so every copyable example matches current A2A allowlist and session visibility requirements, then either add the page to the Multi-agent docs navigation or intentionally fold the material into the existing multi-agent guide.

Yes. The docs issues reproduce by comparing the PR's copyable examples with `createAgentToAgentPolicy` and `createSessionVisibilityChecker`: the omitted `engineer` target fails the allow policy, and cross-agent `sessions_*` calls require `tools.sessions.visibility: "all"`.

## Solution Assessment

No. The standalone guide is a reasonable direction, but the current branch is not the best merge-ready solution until it fixes the A2A/session visibility examples and makes the new page discoverable in the docs site.
