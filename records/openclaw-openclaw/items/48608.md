## Summary

Keep open. Current main has adjacent multi-agent docs, including a newer specialist-lanes page, but it does not fully replace the proposed six-pattern guide, and this PR still has copyable A2A/session examples that conflict with current session visibility enforcement.

## What This Changes

The PR adds `docs/concepts/multi-agent-patterns.md`, a standalone guide for hub-and-spoke routing, cost-tiered routing, channel-bound specialists, heartbeat monitoring, pipeline processing, and anti-patterns.

## Best Possible Solution

Repair this PR or a replacement branch so the examples match current A2A/session visibility contracts, then either register the page in Multi-agent navigation or fold the surviving material into the existing multi-agent docs.

## Reproduction Assessment

Yes. The docs defects reproduce by comparing the PR's hub-and-spoke config with `createAgentToAgentPolicy` and `createSessionVisibilityChecker`: coordinator-to-engineer is denied by the allowlist, and cross-agent `sessions_*` calls require `tools.sessions.visibility: "all"`.

## Solution Assessment

No. The standalone guide is a reasonable direction, but this branch is not merge-ready until its examples match runtime policy and its docs-site placement is resolved.
