Keep open. Current main does not match the report’s stated root cause because the loader cache key is no longer derived from the full root config, but the issue is fresh, includes concrete latency evidence, and is paired with open related issue #76269 from the same author.

## What This Changes

The issue reports that request-time core plugin tool setup reloads plugins after bookkeeping writes because volatile config fields change the plugin registry cache key.

## Best Possible Solution

Keep the structural plugin cache key design, then either confirm a current-main reproduction for the request-time reload or route the remaining work to the gateway config-write reload path tracked by #76269.
