## Summary

This PR should stay open: it addresses a source-reproducible Google/Gemini tool-conversion crash on current main, but the current patch is still not merge-ready because it can drop named parameter-free tools that Google and OpenClaw’s provider-normalizable path allow.

## What This Changes

The PR changes Google shared tool conversion and prompt-cache creation to snapshot/filter invalid tool descriptors, gate `toolConfig` on converted tools, and add targeted regression tests.

## Best Possible Solution

Keep the shared Google converter and prompt-cache reuse, but preserve named parameter-free tools while still quarantining unreadable descriptors and truly unsupported runtime schemas.

## Reproduction Assessment

Yes, by source inspection: current main's Google converter directly reads tool list and descriptor fields before payload construction, so a throwing getter can abort request construction. I did not run the repro or tests because this review was constrained to read-only inspection.

## Solution Assessment

No. The shared converter is the right owner boundary, but the patch should preserve parameter-free Google tools instead of treating every missing parameter schema as invalid.
