## Summary

Keep open: the parser change is useful and the follow-up copied output is acceptable proof for the exercised slugified path, but the patch still misses the actual exported archive Markdown filename form that QMD can surface from the sessions collection.

## What This Changes

Adds QMD archived Markdown transcript stem restoration in the session transcript hit parser plus SDK parser and memory-core visibility regression tests for slugified reset/deleted session paths.

## Best Possible Solution

Teach the `.md` branch to recognize both actual exported `.jsonl.reset|deleted.<timestamp>.md` names and QMD-normalized slug names, then keep regression coverage for both parser and visibility behavior.

## Reproduction Assessment

Yes from source inspection: current main parses archived `.md` session paths as non-archived plain stems, and the visibility filter drops session hits whose stems do not resolve to session keys. I did not run tests because this review is read-only.

## Solution Assessment

No as-is: restoring only the hyphenated slug form is not broad enough for the exported archive Markdown names OpenClaw itself writes. The narrow maintainable fix is to parse the stripped `.md` stem through the existing archive filename parser before or alongside the QMD-normalized restore path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Handle exported archive Markdown names:** `src/plugin-sdk/session-transcript-hit.ts:67-68`
  - body: `exportSessions()` writes archived QMD exports as `<id>.jsonl.reset.<timestamp>.md` because it appends `.md` to `path.basename(sessionFile, ".jsonl")`, and `qmd-manager` can surface that indexed collection-relative path. This branch only restores the hyphenated `-jsonl-reset-` form, so a returned `qmd/sessions-main/abc.jsonl.reset.2026-...Z.md` still falls through as a non-archived stem and the visibility filter can drop it. Please parse the stripped `.md` stem with the existing archive filename parser before or alongside the QMD-normalized form, and add regression coverage for the dot-form export.
  - confidence: 0.86
