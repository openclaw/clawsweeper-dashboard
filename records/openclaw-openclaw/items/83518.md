The PR adds QMD-normalized archived `.md` session transcript parsing in `src/plugin-sdk/session-transcript-hit.ts` and regression tests for reset/deleted slugified archive stems.

## Best Possible Solution

Normalize archived Markdown stems for both canonical `<id>.jsonl.reset|deleted.<ts>.md` exports and QMD slugified `-jsonl-reset|deleted-` hints in the shared session-hit identity layer, then add parser and visibility-filter regressions that exercise the QMD session hit through key resolution.

Yes. Source inspection gives a high-confidence path: current and PR-head parsing still treat canonical `qmd/sessions-main/<id>.jsonl.reset.<ts>.md` as a non-archived stem, and the visibility filter drops session hits when that stem cannot resolve to a session key.

## Solution Assessment

No. The PR changes the right shared parser, but the implementation is incomplete because it handles only slugified archive Markdown names and not the canonical exported archive Markdown names that QMD indexed-path recovery can return.
