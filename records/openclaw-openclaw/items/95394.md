## Summary

Keep open: current main and the latest release still send the LM Studio preload/chat model key without variant-to-canonical resolution, and the active fix is still an open pull request rather than merged code.

## What This Changes

The issue reports that OpenClaw’s LM Studio retry/load path can send a quant-suffixed model identifier, causing false assistant-turn errors and split usage/session accounting.

## Best Possible Solution

Review and land or explicitly replace https://github.com/openclaw/openclaw/pull/95401, keeping the fix provider-local: map advertised LM Studio variants to the canonical key while preserving exact suffixed keys when LM Studio advertises them as canonical.

## Reproduction Assessment

Partially. The reporter supplied concrete live logs, and current source shows suffix-bearing LM Studio ids pass unchanged into preload and chat, but this review did not run a live LM Studio Desktop repro of the clean-id-to-variant retry path.

## Solution Assessment

Yes. The best fix shape is the open provider-local canonicalization PR; blanket-stripping all quant suffixes would risk regressing the intentionally supported explicit suffix refs from earlier merged work.
