## Summary

## What This Changes

The issue reports that LM Studio retry/preload can use a quant-suffixed model identifier, causing invalid-model errors, error stop reasons, and usage/session-model drift.

## Best Possible Solution

Land or revise https://github.com/openclaw/openclaw/pull/95401, or an equivalent provider-local canonicalization that maps advertised variants to loadable LM Studio keys while preserving exact suffixed keys when the server advertises them as real keys.

## Reproduction Assessment

No live reproduction was run here. The reporter provided detailed real-session logs, and current source shows suffix-bearing model ids can pass unchanged into LM Studio preload and chat-completions requests, so the bug is source-reproducible but still needs live confirmation on the reported LM Studio build.

## Solution Assessment

Yes, the best direction is provider-local key canonicalization using LM Studio discovery data. A broad suffix-strip fix would be less safe because current merged behavior intentionally supports explicit quant-suffixed model refs.
