## Summary

Keep open: current main and v2026.6.9 still pass LM Studio variant-style model keys through the preload/chat path, and https://github.com/openclaw/openclaw/pull/95401 is the open candidate fix rather than a merged implementation.

## What This Changes

Fix LM Studio preload/retry so quantized variant identifiers are canonicalized before load/chat and assistant turns are not falsely recorded as errors.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/95401 if maintainers accept the LM Studio key/variant contract, preserving exact suffixed keys only when LM Studio advertises them as canonical.

## Reproduction Assessment

Yes for a source-level reproduction path: current main passes suffix-bearing LM Studio ids unchanged into preload and chat, and the issue includes real LM Studio/session logs. I did not run the live LM Studio Desktop repro in this read-only review.

## Solution Assessment

Yes, provider-local canonicalization from LM Studio discovery data is the best fix shape. A blanket suffix strip would be riskier because OpenClaw already preserves explicitly suffixed local model refs when they are legitimate ids.
