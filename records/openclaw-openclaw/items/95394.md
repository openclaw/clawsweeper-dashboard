## Summary

Keep open: current main and the latest release still pass LM Studio model keys through preload and chat without canonicalizing advertised variant ids, and https://github.com/openclaw/openclaw/pull/95401 is the active candidate fix rather than a merged implementation.

## What This Changes

The issue asks OpenClaw to stop appending or carrying LM Studio GGUF quant suffixes into retry/load/chat requests when LM Studio expects the logical model key.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/95401, preserving exact suffixed keys that LM Studio advertises while canonicalizing advertised variants for native load and chat.

## Reproduction Assessment

No high-confidence live reproduction was established in this review. The report includes concrete same-window OpenClaw and LM Studio logs, and current source shows suffix-bearing model keys would flow unchanged into load and chat requests, but the LM Studio Desktop retry append still needs live confirmation.

## Solution Assessment

Yes, the best direction is provider-local canonicalization from LM Studio discovery data, not blanket suffix stripping. The open candidate PR follows that shape and preserves explicit suffixed keys when they are advertised as canonical.
