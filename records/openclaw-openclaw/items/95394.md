## Summary

Keep open: current main still lacks LM Studio variant-key canonicalization on the preload and chat paths, and the open candidate fix has not merged.

## What This Changes

The issue reports that LM Studio retry/load handling can send a quant-suffixed logical model id, causing false assistant-turn errors and split session/usage model state.

## Best Possible Solution

Land a narrow LM Studio provider fix that maps advertised variant ids to the canonical loadable key while preserving suffixed ids when LM Studio advertises them as canonical.

## Reproduction Assessment

Yes for source reproduction: current main passes the requested LM Studio model id into preload/load and then streams with the original model id, so the reported server shape can still hit the invalid-id path. A live Desktop repro was not run in this read-only review.

## Solution Assessment

Yes, provider-local canonicalization is the best fix shape; blanket-stripping quant suffixes would regress explicit suffix refs that prior merged parsing fixes intentionally preserved.
