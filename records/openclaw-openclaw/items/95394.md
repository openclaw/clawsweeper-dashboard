## Summary

## What This Changes

The issue reports that LM Studio retry/preload paths can send quantized variant suffixes in model identifiers, causing false assistant-turn errors and polluted usage/session state.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/95401 or an equivalent provider-local fix that resolves LM Studio variants from discovery data while preserving exact advertised canonical keys.

## Reproduction Assessment

Yes for a source-level reproduction path: current main passes suffix-bearing LM Studio ids unchanged into preload and chat, and the issue includes real LM Studio/session logs. I did not run the live LM Studio Desktop repro in this read-only review.

## Solution Assessment

Yes, the best fix shape is provider-local canonicalization from LM Studio discovery data rather than a blanket quant-suffix strip. The active PR follows that direction and should remain the review focus.
