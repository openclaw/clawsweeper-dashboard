## Summary

Keep open: the patch is plausible and targets the linked P1 macOS node regression, but the PR still lacks convincing real macOS after-fix proof and the shared connect-timeout change needs maintainer validation before merge.

## What This Changes

The PR serializes node-pairing read helpers under the existing pairing mutex and raises `GatewayNodeSession`'s post-connect snapshot wait from 500 ms to 5000 ms.

## Best Possible Solution

Land a narrow availability fix that serializes node-pairing reads and either proves the 5s snapshot wait in the real macOS node path or replaces the heuristic wait with deterministic snapshot delivery before connected notification.

## Reproduction Assessment

No high-confidence live reproduction was run in this read-only review. The linked issue has concrete commands/logs and current source shows the implicated node-pairing reads outside the existing mutex, so the race is source-reproducible but not locally reproduced.

## Solution Assessment

Unclear as a complete solution: serializing node-pairing reads is the narrow fix for the file race, but the 5s snapshot timeout is a heuristic that needs real macOS proof or a more deterministic snapshot-before-connected path.
