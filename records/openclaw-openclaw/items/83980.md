## Review Findings

Full review comments:

- **[P1] Preserve registered maintenance keys:** `src/config/sessions/store.ts:555-563`
  - body: This replaces the existing `collectSessionMaintenancePreserveKeys([opts?.activeSessionKey])` path with a set built only from the active key plus the new config values. Provider-registered runtime sessions from `store-maintenance-preserve.ts` would no longer be protected in this save path, so prune/cap maintenance can evict sessions the runtime explicitly registered to preserve. Keep the collector and merge any future config keys through that seam.
  - confidence: 0.9
- **[P2] Keep git status inside the best-effort catch:** `scripts/install.ps1:1151-1152`
  - body: The current installer catches `git status` because PowerShell 7 can surface native command failures as terminating errors under `$ErrorActionPreference=Stop`. Moving `git status` into the `if` condition puts it outside that catch, so a status failure can abort the source install before the non-critical pull fallback runs.
  - confidence: 0.84
