## Summary

Keep open: current main and the latest release still have the `formatTokens` rounding path that can emit `1000k`, and this PR is a focused fix with no blocking review findings. Merged sibling formatter fixes support the same rollover behavior rather than superseding this exact helper change.

## What This Changes

The PR changes the Control UI `formatTokens` helper and its tests so near-million counts roll from `k` into `M` instead of displaying `1000k`.

## Best Possible Solution

Land the narrow formatter rollover fix with its regression test once exact-head checks are acceptable.

## Reproduction Assessment

Yes. Current `ui/src/ui/format.ts` checks the raw value before rounding thousands, so values from 999,500 through 999,999 can source-reproduce `1000k`; the PR body also provides before/after formatter output.

## Solution Assessment

Yes. Falling through to the existing `M` branch when the rounded kilo value reaches 1000 is the narrowest maintainable fix for this helper, and sibling UI formatters already use the same guard pattern.
