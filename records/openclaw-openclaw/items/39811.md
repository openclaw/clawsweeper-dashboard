## Summary

Keep open: current main still accepts arbitrary configured model refs unless they match a known static suppression, and the remaining fix needs maintainer policy on trusted catalog evidence so custom and forward-compatible providers do not get false warnings.

## What This Changes

The issue asks OpenClaw to warn or fail early when configured model refs cannot be verified against trusted provider/catalog data, instead of allowing runtime fallback or failure to hide the misconfiguration.

## Best Possible Solution

Add shared advisory validation for config validate and doctor that resolves configured refs with runtime-equivalent alias/provider logic, distinguishes trusted provider/runtime/static catalog evidence from user-declared rows, covers defaults, per-agent refs, fallbacks, hooks, and channel overrides, and preserves intentional custom provider setups.

## Reproduction Assessment

Yes, by source inspection. Current main accepts agent model refs structurally and config validation only rejects static manifest suppressions, not arbitrary provider/model strings against trusted catalogs.

## Solution Assessment

No, current main is only a partial mitigation through fallback visibility and targeted suppressed-model validation. The maintainable fix needs maintainer-approved catalog authority and warning-scope semantics before implementation.
