## Summary

Keep open: current main still lacks a generic config/doctor validation path for arbitrary default and per-agent model refs, and the remaining work is already paired with an open closing PR. Recent comments add fresh real-world evidence for the same class of misconfiguration, while the separate fallback-visibility issue has already been closed as implemented.

## What This Changes

The issue asks OpenClaw to detect invalid configured model refs before runtime and make fallback or misconfiguration visible to users.

## Best Possible Solution

Keep this as the canonical tracker until https://github.com/openclaw/openclaw/pull/40355 or a narrower replacement lands a focused diagnostic that uses runtime-equivalent model resolution, trusted catalog evidence, sanitized output, and clear custom-provider semantics.

## Reproduction Assessment

Yes, by source inspection rather than a live command: configured model strings are collected but arbitrary unknown refs are not checked against trusted catalog evidence, and doctor only checks `hooks.gmail.model`. The May 7, 2026 user comment also provides fresh real-world evidence for a bad per-agent `"model": "default"` override.

## Solution Assessment

No, current `main` is not enough. The open PR is directionally useful, but it needs maintainer decisions around catalog authority and must match runtime alias resolution before merge.
