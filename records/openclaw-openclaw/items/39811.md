## Summary

Keep open: current main still lacks a generic config/doctor validation path for `agents.defaults.model` and per-agent model refs, while an open closing PR already targets that gap. The fallback-notification and startup-warning portions have been handled in related closed issues, and the `openclaw-extensions` workaround shared in the discussion is useful context but does not close the upstream validation issue.

## What This Changes

The issue asks OpenClaw to flag unverified configured model names and make fallback from an invalid primary model visible to users.

## Best Possible Solution

Land or replace the open PR with a focused diagnostic that uses runtime-equivalent alias/model resolution, distinguishes trusted catalog rows from user-declared config rows, sanitizes terminal output, and leaves fallback visibility to its already-fixed path.

## Reproduction Assessment

Yes, by source inspection rather than a live run. Current main structurally accepts configured model strings and only catches static suppressions, with no generic default/per-agent model catalog diagnostic.

## Solution Assessment

No, current main is not sufficient. The open PR is the right direction, but the best fix must avoid trusting config-declared catalog rows and must match runtime alias resolution before merge.
