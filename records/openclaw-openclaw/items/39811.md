## Summary

Keep open: current main still lacks a generic config/doctor validation path for default and per-agent model refs, and the remaining work is already paired with an open closing PR. The downstream fallback visibility and startup-warning pieces were addressed in related closed issues, but the config-validation gap remains.

## What This Changes

The issue asks OpenClaw to warn or validate when configured model refs do not match a trustworthy provider/model catalog and to make fallback use visible to users.

## Best Possible Solution

Keep this issue as the canonical tracker until a focused diagnostic lands that uses runtime-equivalent model resolution, trusted catalog evidence, safe terminal output, and clear behavior for custom/forward-compatible providers.

## Reproduction Assessment

Yes, by source inspection rather than a live run. Current main structurally accepts model strings and only checks static manifest suppressions, so a config-declared unknown provider/model can still pass config validation outside the narrow suppressed-model path.

## Solution Assessment

No, current main is not enough. The open PR is directionally useful, but prior review found it must separate config-declared catalog rows from trusted catalog rows and match runtime alias resolution before merge.
