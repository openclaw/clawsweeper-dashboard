## Summary

Keep open: current main and v2026.5.19 still accept configured model refs structurally without a generic catalog-backed config or doctor warning, and the open PR that explicitly closes this issue is unmerged with review concerns.

## What This Changes

Request an early warning or validation path for configured model refs so unknown primary or fallback model names do not survive until runtime fallback or failure.

## Best Possible Solution

Land or replace the linked diagnostic with a focused warning path that uses runtime-equivalent model resolution, separates trusted catalog evidence from user-declared rows, and covers defaults, per-agent overrides, and fallbacks.

## Solution Assessment

No, current main is not sufficient. Advisory validation is the right direction, but the final shape needs maintainer-approved catalog-authority semantics and should be coordinated through the linked PR or a focused replacement.
