## Summary

Keep open: current main and v2026.6.10 still do not provide the requested generic configured-model validation against trusted catalogs; they only reject statically suppressed model pairs and log a fallback warning after a `model_not_found` fallback succeeds. The open fork PR that closes this issue is still conflicting, so this issue should remain the canonical tracker until maintainers land or replace that path.

## What This Changes

The issue asks OpenClaw to warn or validate when configured model references do not match a provider/runtime catalog, preventing silent invalid-model misconfiguration and unclear fallback behavior.

## Best Possible Solution

Keep this issue open as the canonical invalid-configured-model tracker until maintainers land a narrow doctor/config validation path, or explicitly replace it with a policy-approved broader model-routing consistency fix.

## Reproduction Assessment

Yes, source-reproducible: current config validation collects the relevant model refs but only rejects manifest-suppressed pairs, so arbitrary unknown model refs are not generically checked against a trusted catalog. I did not run a live config command because this review is read-only.

## Solution Assessment

Unclear: the linked doctor-warning PR is a plausible mitigation, but the best fix needs maintainer policy on catalog trust, custom providers, forward-compatible names, and whether save/reload should warn or hard-fail.
