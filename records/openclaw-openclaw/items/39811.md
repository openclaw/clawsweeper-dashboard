## Summary

Keep open: current main still lacks generic trusted-catalog validation for configured model refs. It has targeted static suppression checks and user-visible fallback notices, but arbitrary invalid primary/fallback/per-agent refs can still pass config validation unless they match a known manifest suppression.

## What This Changes

The issue requests upfront warnings or validation for configured model refs that do not resolve to provider-backed catalog entries.

## Best Possible Solution

Add shared advisory validation for config validate and doctor that resolves configured refs with runtime-equivalent alias/provider logic, distinguishes trusted provider/runtime/static catalog evidence from user-declared rows, covers defaults, per-agent refs, fallbacks, hooks, and channel overrides, and preserves custom provider setups.

## Reproduction Assessment

Yes, by source inspection. Current main accepts agent model refs structurally and config validation only rejects static manifest suppressions, not arbitrary provider/model strings against trusted catalogs.

## Solution Assessment

No, current main is only a partial mitigation through fallback visibility and targeted suppressed-model validation. The maintainable fix needs maintainer-approved catalog authority and warning-scope semantics before implementation.
