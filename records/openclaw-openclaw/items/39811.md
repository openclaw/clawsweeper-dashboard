## Summary

Keep open: current main and v2026.6.10 still lack generic validation for invalid configured default, per-agent, and fallback model refs; the remaining fix is tied to an open linked fork PR and needs maintainer policy on catalog authority and warning-versus-error behavior.

## What This Changes

The issue asks OpenClaw to detect unverified or nonexistent configured model names before they degrade into runtime failure or silent fallback.

## Best Possible Solution

Add one shared model-ref validation analyzer for config validate and doctor that resolves refs like runtime does, distinguishes trusted catalog evidence from user-declared rows, and covers defaults, per-agent refs, fallbacks, aliases, and legacy default-style entries.

## Reproduction Assessment

Yes. Source inspection shows current main accepts arbitrary model strings structurally and only rejects statically suppressed refs, so an unsuppressed nonexistent configured model can still pass upfront validation.

## Solution Assessment

No complete solution is on main. The best fix needs maintainer-approved catalog authority and warning/error semantics before another automated patch is safe.
