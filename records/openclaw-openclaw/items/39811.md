## Summary

Keep open: current main still does not validate arbitrary configured model names against the provider/runtime catalog in `openclaw config validate`; only static suppressions and limited onboarding/startup warnings exist, and PR #40355 is still open with closing syntax for this issue.

## What This Changes

Request catalog-aware diagnostics for configured model primaries/fallbacks and visible fallback reporting when an invalid primary falls back.

## Best Possible Solution

Review and land PR #40355 or an equivalent non-blocking catalog-aware warning that covers default and per-agent model refs while leaving fallback visibility on the separate fallback-status path.

## Reproduction Assessment

Yes, from source inspection. A config with an unknown ref satisfies the model schemas, `openclaw config validate` only reports snapshot validity, and current generic validation only rejects manifest-suppressed known-bad refs.

## Solution Assessment

Unclear. A warning-based catalog check is the safer direction than hard rejection, but the open PR still needs maintainer review for full fallback-ref coverage and the exact CLI/doctor warning surface.
