## Summary

Keep open: current main has partial model-ref validation for manifest-suppressed models, but it does not generically validate configured primary/fallback refs against the loaded provider catalog, and an open PR already uses closing syntax for this issue.

## What This Changes

The issue requests validation or warnings for configured model refs that are absent from provider catalogs, plus visible notification when a failed primary model falls back.

## Best Possible Solution

Land or revise the linked fix so configured primary, fallback, default, and per-agent model refs produce clear catalog-mismatch diagnostics without breaking intentional forward-compatible custom model configuration.

## Reproduction Assessment

Yes, from source inspection: current validation only rejects manifest-suppressed refs, while model selection can synthesize allowed entries for configured refs missing from the catalog. I did not execute the CLI path in this read-only review.

## Solution Assessment

No current-main close is warranted. The best route is to review the linked PR and settle whether the diagnostic belongs in config validation, doctor warnings, runtime fallback messaging, or a combination.
