## Summary

Keep open: current main still has only structural/static-suppression model-ref validation and hook-specific doctor catalog checks, while the linked closing PR remains open and recent user comments add real-world workaround evidence.

## What This Changes

The issue asks OpenClaw to warn or validate when configured model refs are not trusted upstream/catalog models, and to make fallback behavior visible instead of silently masking bad primary models.

## Best Possible Solution

Land or replace the linked diagnostic with runtime-equivalent advisory validation that distinguishes trusted catalog evidence from user-declared rows, covers primary and fallback refs, and leaves broader fallback notification behavior to its separate tracker.

## Reproduction Assessment

Yes, by source inspection. Current main accepts configured model refs structurally, only rejects manifest-suppressed refs, and does not provide a generic catalog-backed `config validate` or doctor warning for configured primary/fallback refs.

## Solution Assessment

No, current main is not sufficient. Advisory validation is the right direction, but the final shape needs maintainer-approved catalog-authority semantics and should be coordinated through the linked PR or a focused replacement.
