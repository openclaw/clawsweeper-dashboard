## Summary

Keep open: current main partially mitigates the user impact with startup primary-model prewarm and fallback notices, but it still lacks generic catalog-backed validation for arbitrary configured primary, fallback, and per-agent model refs. The earlier closing PR is now closed unmerged, and recent comments add in-the-wild evidence plus a downstream doctor workaround.

## What This Changes

The issue requests validation or warnings for configured model refs that cannot be verified against provider catalogs, plus visible notification when runtime model fallback occurs.

## Best Possible Solution

Add shared advisory validation for config validate and doctor that resolves refs like runtime, distinguishes trusted provider/runtime/static catalog evidence from user-declared rows, covers defaults, per-agent models, and fallbacks, and preserves custom/forward-compatible provider setups.

## Reproduction Assessment

Yes, by source inspection. Current main accepts arbitrary agent model strings structurally, validates only static manifest suppressions, and lets `openclaw config validate` succeed without catalog-backed checks.

## Solution Assessment

No, current main is only a partial mitigation. The best fix needs maintainer-approved catalog authority semantics before adding broad advisory validation to config validate and doctor.
