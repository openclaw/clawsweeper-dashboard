## What This Changes

The issue asks OpenClaw to warn when configured primary, fallback, or per-agent model refs cannot be verified against provider/catalog data, and to avoid silent fallback after a bad primary model.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/40355 with a focused diagnostic that uses runtime-equivalent model resolution and trusted catalog evidence, while leaving the already-implemented fallback visibility behavior to its separate tracker.

## Reproduction Assessment

## Solution Assessment

No, current main is not sufficient. The open PR is the active direction, but the durable fix needs trusted catalog semantics and runtime-equivalent alias/model resolution before this issue should close.
