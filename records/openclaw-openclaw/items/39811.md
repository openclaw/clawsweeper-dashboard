## Summary

Keep open: current main has partial mitigations for primary startup warmup and fallback notices, but it still does not provide generic catalog-backed validation for arbitrary configured primary, fallback, and per-agent model refs; the related OpenClaw PR was closed unmerged, and the discussion adds real user evidence plus a downstream doctor workaround.

## What This Changes

The issue requests warnings or validation for invalid configured model refs, plus visible notice when fallback hides a failed primary model.

## Best Possible Solution

Add shared advisory validation for `openclaw config validate` and `openclaw doctor` that resolves refs like runtime, distinguishes trusted catalog evidence from user-declared rows, covers primaries/fallbacks/per-agent overrides, and preserves custom provider setups.

## Reproduction Assessment

Yes, by source inspection. Current main accepts arbitrary model strings structurally and `openclaw config validate` only applies static suppression checks, not trusted catalog-backed validation for arbitrary configured refs.

## Solution Assessment

No, current main is only a partial mitigation. The best fix needs maintainer-approved catalog authority and warning-scope semantics before broad advisory validation is added.
