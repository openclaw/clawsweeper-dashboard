## Summary

Keep open: current main only covers the runtime fallback-visibility part and static retired-model validation; it does not yet add the requested catalog-backed config/doctor warning for configured primary models. Open PR #40355 is already the active implementation path, while related #39818 and #45294 cover adjacent symptoms rather than superseding this remaining validation surface.

## What This Changes

The issue requests catalog-backed validation or warnings for configured model names, plus user-visible notice when fallback hides a missing primary model.

## Best Possible Solution

Review and land PR #40355 or an equivalent warning-only implementation that checks configured primary/per-agent model refs against the loaded catalog while preserving forward-compatible custom provider IDs.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection shows `config validate` and the generic config validator do not perform arbitrary provider-catalog checks, while doctor only performs a catalog check for `hooks.gmail.model`.

## Solution Assessment

Unclear pending PR review. The warning-only doctor/config direction in PR #40355 appears narrower and safer than hard config rejection because provider docs allow dynamic model IDs, but maintainers should confirm the intended surface.
