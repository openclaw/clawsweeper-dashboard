## What This Changes

The issue requests model-config validation and user-visible diagnostics when configured model refs are unknown or trigger runtime fallback.

## Best Possible Solution

Keep this issue as the canonical tracker until a focused diagnostic lands that uses runtime-equivalent model resolution, trusted catalog evidence, sanitized terminal output, and clear behavior for custom providers.

## Solution Assessment

No, current main is not sufficient. The best path is to review and fix the linked closing PR or land an equivalent diagnostic that separates provider/catalog evidence from local config rows and matches runtime alias resolution.
