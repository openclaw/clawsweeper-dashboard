## Summary

Keep open: current main still does not provide a generic `config validate` or `doctor` warning for arbitrary unknown configured model refs, and https://github.com/openclaw/openclaw/pull/40355 remains the open implementation path rather than a merged fix. The silent fallback portion has adjacent shipped coverage, but the canonical configured-model validation gap still has recent field evidence and should remain active until the open PR lands or is replaced.

## What This Changes

The issue asks OpenClaw to warn when configured model references cannot be verified against provider/catalog data, so bad primary or fallback model names do not survive until runtime.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/40355 with a focused diagnostic that uses runtime-equivalent model resolution, distinguishes trusted catalog evidence from user-declared rows, and leaves broader fallback-user notification to its already handled tracker.

## Reproduction Assessment

Yes, by source inspection. Current main accepts structural model strings, config validation only rejects manifest-suppressed refs, and doctor only checks `hooks.gmail.model` rather than generic configured default, per-agent, or fallback model refs.

## Solution Assessment

No, current main is not sufficient. The active PR is a reasonable direction, but the durable solution needs trusted catalog semantics and runtime-equivalent alias/model resolution before this issue should close.
