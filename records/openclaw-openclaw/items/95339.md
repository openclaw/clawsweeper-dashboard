Full review comments:

- **[P2] Use the canonical config resolver:** `scripts/control-director-readiness.mjs:44`
  - body: The readiness script falls back to `~/.openclaw/openclaw.director.json`, but OpenClaw's canonical config path is `OPENCLAW_CONFIG_PATH` or the active state dir's `openclaw.json`. Running the script without `--config` will inspect the wrong file on normal installs, so reuse the existing resolver/default.
  - confidence: 0.93
- **[P2] Read Ollama env from the right platform source:** `scripts/control-director-readiness.mjs:657`
  - body: Runtime env checks always call `readOllamaEnvFromLaunchctl()`, and missing values become critical scorecard failures. On Linux or other non-launchd setups this reports a critical failure even when Ollama is correctly configured, so gate launchctl to macOS or read a platform-appropriate environment source.
  - confidence: 0.9
- **[P2] Return summaries from taskFlows.list:** `src/gateway/server-methods/tasks.ts:292`
  - body: `TaskFlowsListResultSchema` declares `flows` as `TaskFlowSummary[]`, but the handler returns `mapTaskFlowDetail()` entries that include detail-only `tasks` and `taskSummary` fields. Strict generated clients or response validators can reject the list result; use a summary mapper for list and keep details on get/create/cancel.
  - confidence: 0.87
