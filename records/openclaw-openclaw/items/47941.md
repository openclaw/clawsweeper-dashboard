## Summary

Keep open. Current main still lets a hook-provided bootstrap file with a valid path but missing name reach the context report as `name: file.name`, and the `/context` renderer prints that raw name; open PR #48021 is already the linked implementation candidate.

## What This Changes

The issue asks `/context` reporting to label hook-injected virtual bootstrap files from a non-empty name, path, basename, or placeholder instead of rendering literal `undefined`.

## Best Possible Solution

Review and land #48021 or an equivalent narrow fix that normalizes the stat label at report construction time and adds regression coverage for path-only hook-injected bootstrap files.

## Reproduction Assessment

Yes. A source-level reproduction is clear: a bootstrap hook can append a path/content object without `name`, sanitization preserves it, `buildBootstrapInjectionStats` returns `name: file.name`, and `/context` renders `f.name`.

## Solution Assessment

Yes. Normalizing the label in `buildBootstrapInjectionStats` or the report-construction layer, with a focused regression test, is the narrowest maintainable fix; closing should wait until #48021 or an equivalent patch lands.
