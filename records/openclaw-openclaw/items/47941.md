## Summary

Keep open: current main still carries a path-only bootstrap file’s raw `file.name` into `/context` output, and the open closing PR is the right implementation candidate to review, land, or replace.

## What This Changes

The issue requests that context breakdown labels for hook-injected virtual bootstrap files fall back from missing `name` to `path` or basename instead of rendering literal `undefined`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/48021 or an equivalent narrow fix that derives a nonempty bootstrap report label and adds regression coverage.

## Reproduction Assessment

Yes, at source level. `agent:bootstrap` exposes mutable bootstrap files, sanitization preserves path-only entries, `buildBootstrapInjectionStats` returns `file.name`, and `/context` renders that name directly.

## Solution Assessment

Yes. Normalizing the label in `buildBootstrapInjectionStats` or the report-construction layer is the narrowest maintainable fix; this issue should stay open until the linked PR or an equivalent patch lands.
