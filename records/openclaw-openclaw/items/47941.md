## Summary

Keep open. Current main still preserves a path-only hook-injected bootstrap file without synthesizing a display name, and `/context` renders that raw stat name directly; an open linked PR is the right implementation candidate to review or replace.

## What This Changes

Ensure `/context` reporting labels path-only virtual bootstrap files with a stable name/path/basename fallback instead of literal `undefined`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/48021 or an equivalent narrow fix that derives a nonempty bootstrap report label and adds regression coverage for path-only hook-injected files.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `agent:bootstrap` can mutate `context.bootstrapFiles`, sanitization preserves a path-only object, the stat builder returns `name: file.name`, and `/context` renders `${f.name}`.

## Solution Assessment

Yes. Normalizing the label in `buildBootstrapInjectionStats` or the report-construction layer is the narrowest maintainable fix; closure should wait until the linked PR or an equivalent patch lands.
