## Summary

Keep open: current `main` and the latest release still let the generic context report copy `file.name` into the displayed bootstrap-file label, so a path-only hook entry can still surface as `undefined`. The open linked fork PR is the right current fix candidate, so this issue should wait for that PR to be reviewed, adopted, or replaced rather than being closed or queued twice.

## What This Changes

The issue asks for context reporting to fall back from a missing bootstrap file name to the file path or basename instead of rendering `undefined`.

## Best Possible Solution

Land or replace the linked generic bootstrap-budget fix so `buildBootstrapInjectionStats` derives a stable label from non-empty `name`, path, basename, and finally a placeholder, with a focused regression test for path-only hook entries.

## Reproduction Assessment

Yes, from source: a path-only bootstrap entry with content can flow into `buildBootstrapInjectionStats`, whose returned display name is `file.name`; `commands-context-report.ts` then renders `f.name` directly. I did not run a live `/context` command because this review was required to keep the checkout read-only.

## Solution Assessment

Yes, the best fix is the narrow generic stats/reporting fallback in `src/agents/bootstrap-budget.ts`, not a display-only workaround, because the same report payload feeds run, estimate, and context command output.
