## Summary

Keep open: current main and the latest release still expose the generic `/context` label path where a path-only virtual bootstrap file can render as `undefined`. The open linked fork PR is the current fix candidate, so this issue should wait for that PR to be reviewed, adopted, or replaced rather than closed.

## What This Changes

The issue asks for context reporting to derive a stable bootstrap-file label from `name`, `path`, basename, or a placeholder when hooks inject virtual files without `name`.

## Best Possible Solution

Land or adopt the generic fallback in `buildBootstrapInjectionStats`, with regression coverage that a path-only bootstrap file reports the basename and preserves raw/injected accounting.

## Reproduction Assessment

Yes at source level: hooks can return path-only bootstrap entries, current `buildBootstrapInjectionStats` copies `file.name`, and `/context` renders that value directly. I did not run a live `/context` command because this review was read-only.

## Solution Assessment

Yes: normalizing the displayed stat label in the shared bootstrap report builder is the narrowest maintainable fix because both stored system-prompt reports and `/context` output consume that stat.
