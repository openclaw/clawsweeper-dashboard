## Summary

Keep open. Current main still lets path-only hook-injected bootstrap files propagate an undefined report name, and the linked implementation candidate at https://github.com/openclaw/openclaw/pull/48021 is open and uses closing syntax for this issue.

## What This Changes

The issue requests a fallback label for virtual bootstrap files so `/context` reporting does not render literal `undefined` when a path or basename is available.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/48021 or an equivalent narrow fix that normalizes the bootstrap report label and adds regression coverage for path-only hook-injected files.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a hook can append a bootstrap file with `path` and no `name`, sanitization preserves it, the stat builder returns `name: file.name`, and `/context` renders `f.name`.

## Solution Assessment

Yes. Normalizing the label in `buildBootstrapInjectionStats` or the report-construction layer is the narrowest maintainable fix; closing should wait until the linked PR or an equivalent patch lands.
