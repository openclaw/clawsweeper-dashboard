## Summary

Keep open: current main and the latest release still allow a path-only bootstrap file to keep `name: undefined` in the shared context report, and `/context` renders that value directly. The open fork PR at https://github.com/BingqingLyu/openclaw/pull/829 is the current narrow fix candidate, so maintainers should review, adopt, or replace that patch instead of closing this issue or queueing another duplicate fix.

## What This Changes

This issue requests a fallback label for virtual/path-only bootstrap files so `/context` reporting never displays literal `undefined`.

## Best Possible Solution

Land or adopt the shared `buildBootstrapInjectionStats` label fallback with focused regression coverage for path-only and empty-name bootstrap files while preserving raw/injected accounting.

## Reproduction Assessment

Yes, at source level: hook-mutated bootstrap files can retain path-only metadata, `buildBootstrapInjectionStats` keeps `name: file.name`, and `/context` interpolates that name directly. I did not run a live `/context` command because this review is read-only.

## Solution Assessment

Yes: normalizing the label in the shared bootstrap stats builder is the narrowest maintainable fix because stored reports and `/context` output consume that same stat. A renderer-only fallback would leave JSON/report metadata unstable.
