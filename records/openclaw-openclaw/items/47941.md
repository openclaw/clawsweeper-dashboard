## Summary

Keep open: current main and the latest release still let path-only bootstrap files produce a raw `file.name` label in the generic `/context` report path, and the open linked PR is still the right fix candidate.

## What This Changes

The issue requests that `/context` reporting derive a nonempty label for hook-injected virtual bootstrap files that have a path but no name.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/48021 or an equivalent narrow stat-builder fallback with regression coverage for missing and empty bootstrap file names.

## Reproduction Assessment

Yes, at source level. Path-only hook entries can survive sanitization, `buildBootstrapInjectionStats` returns the raw `file.name`, and `/context` renders that name directly; I did not run a live scenario because this review is read-only.

## Solution Assessment

Yes. Normalizing the bootstrap report label in the stat builder is the narrowest maintainable generic fix; the merged Codex-specific fix does not replace this path.
