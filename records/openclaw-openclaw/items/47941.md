## Summary

Keep open. Current main still allows a path-only hook-injected bootstrap file to produce an undefined injected-workspace display name, and an open linked PR with closing syntax is the right implementation candidate to review, land, or replace.

## What This Changes

The issue requests a fallback label for virtual bootstrap files so `/context` reporting does not render literal `undefined` when `path` is available but `name` is missing.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/48021 or an equivalent narrow fix that derives a nonempty bootstrap report label and adds regression coverage for path-only hook-injected files.

## Reproduction Assessment

Yes, at source level. `agent:bootstrap` can leave a path-only bootstrap file, sanitization preserves the missing `name`, the stat builder returns `file.name`, and `/context` renders `f.name` directly.

## Solution Assessment

Yes. Normalizing the label in `buildBootstrapInjectionStats` or the report-construction layer is the narrowest maintainable fix; closure should wait until the linked PR or an equivalent patch lands.
