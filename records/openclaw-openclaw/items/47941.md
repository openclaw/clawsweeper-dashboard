## What This Changes

Make `/context` labels for hook-injected path-only virtual bootstrap files fall back to the path or basename instead of rendering `undefined`.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/48021 or an equivalent narrow fallback in `buildBootstrapInjectionStats` with regression coverage for missing and empty bootstrap file names.

Yes. Normalizing the bootstrap report label in the stat builder is the narrowest maintainable generic fix; the merged Codex-specific fix does not replace this path.
