## Repair Work Prompt

Fix the generic `/context` undefined bootstrap-file label bug for https://github.com/openclaw/openclaw/issues/47941. In the current OpenClaw source, hook-injected bootstrap entries can have `path` and content but no usable `name`; `buildBootstrapInjectionStats` currently returns `name: file.name`, and `commands-context-report.ts` renders `${f.name}` directly. Add a central fallback label derivation in `src/agents/bootstrap-budget.ts` or the shared report builder: prefer a non-empty `file.name`, then a non-empty `file.path` basename using slash/backslash-safe path handling, then a clear placeholder such as `(virtual bootstrap file)`. Preserve current path-first injected-content lookup and the undefined-name crash guards from PR 85615. Add regression coverage for undefined and empty names with valid paths, including injected char/truncation behavior. If touching the renderer, add or update `commands-context-report.test.ts`; if normalizing at the sanitizer boundary, add `bootstrap-files.test.ts` coverage. Check the open fork PR https://github.com/BingqingLyu/openclaw/pull/829 and the closed target PR https://github.com/openclaw/openclaw/pull/48021 for useful source work, but keep the final patch narrow and target-repo owned. Add a single-line changelog fix entry because this is user-visible. Do not add config, new plugin API, broad fallback stacks, or unrelated bootstrap policy changes.
