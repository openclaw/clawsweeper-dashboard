## Summary

Keep open. Current main still has the display-layer path where a path-only bootstrap file can become an undefined context label, and the linked implementation candidate remains open with closing syntax for this issue.

## What This Changes

The issue requests stable context-report labels for virtual bootstrap files by falling back from name to path/basename instead of rendering literal `undefined`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/48021 or an equivalent narrow fix that normalizes the bootstrap report label and adds regression coverage for path-only hook-injected files.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a bootstrap hook can append a file with `path` and no `name`, sanitization preserves it, the stat builder returns `name: file.name`, and `/context` renders `f.name`.

## Solution Assessment

Yes. Normalizing the label in `buildBootstrapInjectionStats` or the report-construction layer is the narrowest maintainable fix; closure should wait until the linked PR or an equivalent patch lands.
