## Summary

Keep open: current main and latest release still let a path-only bootstrap file carry `name: undefined` into the shared context report, and `/context` renders that value directly. The open fork PR at https://github.com/BingqingLyu/openclaw/pull/829 is the current narrow fix candidate, so this issue should wait for maintainer review, adoption, or replacement rather than being closed or queued again.

## What This Changes

The issue asks context reporting to label path-only virtual bootstrap files with a stable name from `name`, `path`, basename, or a placeholder instead of rendering literal `undefined`.

## Best Possible Solution

Land or adopt the shared `buildBootstrapInjectionStats` fallback with focused regression coverage for path-only and empty-name bootstrap files while preserving raw/injected accounting.

## Reproduction Assessment

Yes, at source level: a path-only bootstrap hook entry can pass through sanitization, `buildBootstrapInjectionStats` keeps `name: file.name`, and `/context` interpolates that name directly. I did not run a live `/context` command because this review was read-only.

## Solution Assessment

Yes: normalizing the display label in the shared bootstrap stats builder is the narrowest maintainable fix because stored reports and `/context` output consume that same stat.
