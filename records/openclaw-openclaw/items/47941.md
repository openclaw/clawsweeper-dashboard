## Summary

Keep open: current `main` and the latest release still copy the generic bootstrap report label from `file.name`, so a path-only hook file can still surface as `undefined` in `/context`; the merged Codex fix only covered the Codex app-server path, and the generic fix remains unmerged in https://github.com/BingqingLyu/openclaw/pull/829.

## What This Changes

The issue requests that context reporting label path-only virtual bootstrap files with a stable file name instead of rendering literal `undefined`.

## Best Possible Solution

Land or replace a narrow generic core fix that normalizes the report label in `buildBootstrapInjectionStats` from non-empty `file.name`, path basename, then a placeholder before `/context` renders it.

## Reproduction Assessment

Yes, source-level: `buildBootstrapInjectionStats` still copies `file.name` into the report name, and `/context` renders that field directly. A path-only hook entry is enough to make the bad label visible.

## Solution Assessment

Yes: the best fix is the narrow generic stats fallback in `src/agents/bootstrap-budget.ts`. The merged Codex-only guard is useful but intentionally does not cover the generic `/context` path.
