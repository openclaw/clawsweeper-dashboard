## What This Changes

The issue asks `/context` breakdown reporting to label unnamed virtual bootstrap files with a path or basename fallback instead of rendering literal `undefined`.

## Best Possible Solution

## Solution Assessment

Yes. Normalizing the bootstrap report label in the stat/report construction path is the narrowest maintainable fix; this issue should stay open until the linked PR or an equivalent patch lands.
