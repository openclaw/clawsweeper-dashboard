## Summary

Keep open: current main and v2026.6.10 still copy `file.name` into context report rows and `/context` renders that value directly, so path-only virtual bootstrap files can still display as `undefined`. The open fork PR at https://github.com/BingqingLyu/openclaw/pull/829 is a narrow candidate fix, but it has not landed.

## What This Changes

The issue asks OpenClaw to label virtual/path-only bootstrap files in `/context` reports with a stable name instead of rendering literal `undefined`.

## Best Possible Solution

Normalize the bootstrap report label in the shared stats/reporting path with `name -> path -> basename -> placeholder`, add regression coverage for path-only virtual files, and land or replace the open candidate fix.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: a hook-adjusted bootstrap file with `path` and content but no `name` can survive into `buildBootstrapInjectionStats`, which returns `name: file.name`, and `/context` renders that value directly. I did not run a live `/context` command in this read-only review.

## Solution Assessment

Yes, the best fix is in the shared bootstrap stats/reporting layer rather than only the command renderer, because run reports, estimate reports, JSON output, and `/context` rows all consume `injectedWorkspaceFiles`. A rendering-only fallback would leave stale structured data behind.
