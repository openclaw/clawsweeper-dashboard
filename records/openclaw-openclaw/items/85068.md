## Summary

Keep open: current main has adjacent foreground freshness fencing, abort cleanup, and detached-task tracking, but it still lacks a foreground partial-WIP incident/finalizer or TaskFlow/Taskdash state for direct chat turns that time out or are superseded after write/tool activity.

## What This Changes

The issue requests a foreground-run lifecycle guard that reports partial WIP and records state when Telegram/direct chat work times out or is superseded after file or tool activity.

## Best Possible Solution

Add a generic foreground-run lifecycle incident hook that records write/tool activity, detects timeout or supersession without a terminal report, emits a best-effort partial-WIP notice, and stores minimal redacted metadata without changing detached-task semantics.
