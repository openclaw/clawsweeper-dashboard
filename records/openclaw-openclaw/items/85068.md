## Summary

Keep open: current main and the latest release still have foreground freshness fencing and generic timeout/killed/lost states, but no privacy-safe partial-WIP/action-debt finalizer after a foreground timeout or supersession.

## What This Changes

The issue asks OpenClaw to detect foreground channel turns that end after write/tool activity without a terminal report and surface partial-WIP state to the user and task/session surfaces.

## Best Possible Solution

Add a privacy-safe foreground-run incident/finalizer that records partial-WIP state and emits a bounded user notice without weakening foreground freshness suppression.

## Reproduction Assessment

No high-confidence live Telegram reproduction was run. Source inspection is enough to show current main has no partial-WIP/action-debt state or foreground finalizer, but the original private dirty-worktree scenario still needs a redacted live repro or focused integration test.

Unclear. A foreground finalizer is plausible, but maintainers should choose the durable owner boundary across reply delivery, task/session status, privacy-safe dirty metadata, and detached-controller guidance before implementation.
