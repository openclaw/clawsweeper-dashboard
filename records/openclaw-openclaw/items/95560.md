## Summary

Keep open: current main and v2026.6.9 still keep `web_fetch` trusted-env proxy as an explicit opt-in with no startup warning, and auto-enabling it from ambient proxy env is a security/product default decision rather than a narrow bug fix.

## What This Changes

The issue requests auto-enabling or warning about `tools.web.fetch.useTrustedEnvProxy` when `HTTP_PROXY` or `HTTPS_PROXY` is present.

## Best Possible Solution

Keep explicit `web_fetch` proxy routing off by default until the NO_PROXY/security policy is settled, and consider a maintainer-approved startup or doctor diagnostic that points users to the documented opt-in.

## Reproduction Assessment

Yes for the current behavior: source and tests show `web_fetch` remains direct when `HTTP_PROXY` is present unless `useTrustedEnvProxy` is explicitly true. I did not run a live blocked-network reproduction in this read-only review.

## Solution Assessment

No for unconditional auto-enable as stated; the safer path is a maintainer-approved warning or doctor diagnostic while keeping proxy routing explicit until NO_PROXY/security behavior is decided.
