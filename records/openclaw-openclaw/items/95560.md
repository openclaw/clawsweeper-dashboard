## What This Changes

## Best Possible Solution

Keep web_fetch proxy routing explicit by default, then add a maintainer-approved startup or doctor diagnostic if maintainers accept the warning policy and coordinate it with the open NO_PROXY security decision.

## Reproduction Assessment

Yes. Source and tests show current web_fetch remains direct when HTTP_PROXY is present unless tools.web.fetch.useTrustedEnvProxy is explicitly true; I did not run a live blocked-network reproduction in this read-only review.

## Solution Assessment

No for unconditional auto-enable as stated. A warning or doctor diagnostic is likely safer, but the exact placement and wording still need maintainer product/security approval.
