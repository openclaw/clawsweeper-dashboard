## Summary

Keep open: the new PR-body proof is strong enough for the changed two-port macOS path, but the patch is not the best merge shape yet because it also changes every existing single-port Unix listener inspection from targeted `lsof -iTCP:<port>` to an all-LISTEN snapshot without reducing spawn count.

## What This Changes

The PR adds a multi-port Unix listener inspection helper that snapshots LISTEN sockets once, exports it, wires daemon status to batch distinct daemon/CLI port checks, and adds focused regression tests.

## Best Possible Solution

Reserve the all-LISTEN snapshot for true multi-port same-cycle inspection, preserve targeted `lsof -iTCP:<port>` for single-port `inspectPortUsage()`, and keep https://github.com/openclaw/openclaw/issues/90548 open for restart-health/backoff work.

## Reproduction Assessment

Yes for the spawn-count behavior: current main source shows per-port `lsof -iTCP:<port>` calls and the PR proof shows one live macOS all-LISTEN spawn for two ports. The full WindowServer watchdog failure was not reproduced, and the PR correctly treats that as out of scope.

## Solution Assessment

No, not yet: batching daemon/CLI status through a shared helper is the right direction, but changing ordinary single-port callers to all-LISTEN scans is broader than the narrow mitigation. The safer fix keeps single-port reads targeted and batches only multi-port inspections.
