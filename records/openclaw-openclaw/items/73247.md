## Summary

Keep open. Current main still has the Slack reply hook coverage gap this PR targets, but the combined patch is not merge-ready because the new bundled guard has correctness, credential-scoping, metadata, and product-scope blockers.

## What This Changes

The PR wires Slack `deliverReplies()` through `message_sending` hooks and adds a bundled `slack-addressee-guard` plugin with manifest and tests.

## Best Possible Solution

Land the Slack reply hook wiring as a focused, tested fix, and split or rework the addressee guard as an account-aware plugin with maintainer-approved scope, docs, changelog, and label routing.

## Reproduction Assessment

Yes. Source inspection on current main shows Slack event replies reach `deliverReplies()` and call `sendMessageSlack()` directly, while `message_sending` hook wiring lives in other outbound or generic dispatcher paths.

## Solution Assessment

No. The hook wiring is a narrow maintainable fix, but the bundled guard is not the best complete solution until it is account-scoped, limited to leading target mentions, supports Slack conversation ID variants, and has required metadata/changelog coverage.

## Security Review

Status: needs_attention
