## What This Changes

The PR adds an ACP projector token-usage callback, wires ACP `usage_update` events into `persistSessionUsageUpdate`, and adds projector-level regression tests.

## Best Possible Solution

Land a narrow ACP-owned fix that persists numeric ACP `usage_update` snapshots into the existing session `totalTokens` fields, with regression coverage and redacted real ACP session-list proof.

Yes, source-reproducible: current main consumes numeric ACP `usage_update` events for display/dedupe only, and session listing reads `entry.totalTokens`, which ACP dispatch never writes. I did not run the Docker live repro in this read-only review.

## Solution Assessment
