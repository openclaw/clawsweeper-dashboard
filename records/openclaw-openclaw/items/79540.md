## Summary

Keep open: the PR targets a valid, source-reproducible ACP usage accounting bug, but the dispatch integration currently fire-and-forgets the session-store write and the PR still lacks after-fix real behavior proof.

## What This Changes

The PR adds an ACP projector token-usage callback, wires numeric ACP `usage_update` events into session usage persistence, and adds projector regression tests.

## Best Possible Solution

Land an ACP-owned fix that awaits persisting numeric `usage_update` snapshots into the existing session `totalTokens` fields, with regression coverage, a changelog entry, and redacted after-fix ACP session-list proof.

## Reproduction Assessment

Yes, source-reproducible: current main consumes numeric ACP `usage_update` events for display/dedupe only, while session listing reads entry-level `totalTokens` and ACP dispatch never writes it. I did not run the Docker live repro in this read-only review.

## Solution Assessment

No, not yet: the callback seam and existing `persistSessionUsageUpdate` helper are the right direction, but the dispatcher should await the write and the PR still needs changelog and real behavior proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Await the ACP usage persistence callback:** `src/auto-reply/reply/dispatch-acp.ts:427-428`
  - body: The projector now awaits `onTokenUsage`, but this dispatcher callback discards the `persistSessionUsageUpdate` promise. That lets `runTurn`/`flush` finish and `openclaw sessions` be queried before `entry.totalTokens` is actually written, so the reported bug can still surface as `null` immediately after a completed ACP turn. Make the callback await the existing helper, with dispatch-level coverage for the store update.
  - confidence: 0.87

  - body: This is a user-visible fix to `openclaw sessions` token reporting and budget/cost surfaces, but the PR only changes ACP code and tests. Add a single-line `CHANGELOG.md` Fixes entry under the active version before merge.
  - confidence: 0.9
