## Summary

Keep open: current main has not implemented default heartbeat isolation, and this PR is not merge-ready because it changes only the run-session default while leaving preflight semantics, current-main docs, changelog, and the product decision unresolved.

## What This Changes

This PR makes heartbeat runs with omitted `isolatedSession` use a `:heartbeat` session by default, updates heartbeat docs/type comments, and adds model-override regression coverage.

## Best Possible Solution

If maintainers want default isolation, resolve the default once and apply it consistently to preflight, run-session selection, current docs, tests, and changelog; otherwise keep the existing opt-in default and close this PR.

## Reproduction Assessment

Yes for the review findings by source inspection. The PR changes only the final run-session condition, while current preflight still uses true-only isolation semantics and production cron wake callers omit `isolatedSession`.

## Solution Assessment

No. The proposed one-line runtime change is not complete without sharing the new default with preflight, updating current-main docs/changelog, and first confirming the default-policy change with maintainers.
