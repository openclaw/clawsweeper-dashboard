## Summary

Keep open: current main has not implemented default-isolated heartbeat runs, and this PR still has blocking runtime/docs findings plus missing real behavior proof for an external contributor PR.

## What This Changes

This PR changes heartbeat runs to default to an isolated `:heartbeat` session, keeps `isolatedSession: false` as an opt-out, updates docs/type comments, and adds regression tests.

## Best Possible Solution

Compute one effective heartbeat isolation mode before preflight and execution, update all public config docs, and land only after maintainer default-policy approval plus real heartbeat proof.

## Reproduction Assessment

Yes, for the review findings by source inspection: the PR flips execution to default isolated while preflight and one public docs surface still follow explicit-true/default-false semantics. No live heartbeat setup was run in this read-only review.

## Solution Assessment

No as written. The direction may be acceptable, but the implementation should compute one effective isolation value before both preflight and run selection, and the docs need to be aligned.
