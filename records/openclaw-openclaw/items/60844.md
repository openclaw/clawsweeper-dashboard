## Summary

Keep open: the PR is useful but not merge-ready. The previous ClawSweeper finding still holds on current main: the branch flips only the final run-session choice, while heartbeat preflight still uses explicit-true isolation semantics; the branch also leaves one public config doc inconsistent, lacks real behavior proof, and is currently conflicting.

## What This Changes

The PR changes heartbeat runs with omitted `isolatedSession` to use a `:heartbeat` session by default, updates heartbeat docs/type comments, and adds regression tests for default isolation and abort-state preservation.

## Best Possible Solution

If maintainers want default isolation, use one shared isolation-default resolver for preflight and run-session selection, update all heartbeat config docs, and add omitted-flag wake/cron regression coverage.

## Reproduction Assessment

Yes for the review findings by source inspection. The PR changes only the final run-session selection while current preflight still branches on explicit `isolatedSession: true`, and existing tests define the explicit isolated wake/event boundary.

## Solution Assessment

No as written. Default isolation may be the right product direction after maintainer approval, but this patch must apply the default before preflight and keep all public heartbeat config docs aligned.
