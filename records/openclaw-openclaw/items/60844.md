## Summary

Keep open: the direction may be useful, but the patch still changes only the later run-session choice, leaving heartbeat preflight on the old omitted-as-shared semantics, and it has conflicting public docs plus no real heartbeat setup proof from the contributor.

## What This Changes

The PR makes omitted `heartbeat.isolatedSession` use an isolated `:heartbeat` session by default, keeps `isolatedSession: false` as the shared-session opt-out, updates heartbeat docs/type comments, and adds regression tests.

## Best Possible Solution

If maintainers approve default-on heartbeat isolation, resolve the isolation default once before preflight and run-session selection, keep `isolatedSession: false` as the opt-out, update every public config reference, and add omitted-flag wake/cron regression coverage.

## Reproduction Assessment

Yes for the review findings by source inspection. Current main still has explicit-true preflight semantics while the PR flips only the later run-session selection, and current tests define the explicit isolated wake boundary that omitted default isolation should match.

## Solution Assessment

No as written. The requested default may be acceptable after maintainer approval, but the patch should apply the default before heartbeat preflight and keep all public heartbeat config docs aligned.
