## Summary

Keep open: current main still has opt-in heartbeat isolation, and this PR is not merge-ready because the new default is applied only at run-session selection while preflight and one public config doc still follow the old default-false contract.

## What This Changes

The PR changes heartbeat runs to use an isolated `:heartbeat` session by default, keeps `isolatedSession: false` as an opt-out, updates some docs/type comments, and adds regression tests.

## Best Possible Solution

Resolve one effective heartbeat isolation value before preflight and execution, update every public heartbeat config reference, and land only after maintainer default-policy approval plus real heartbeat proof.

## Reproduction Assessment

Yes for the review findings by source inspection: the PR flips default isolation only at run-session selection, while preflight and one public docs surface still follow the old explicit-true/default-false model. No live heartbeat setup was run in this read-only review.

## Solution Assessment

No as written. The direction may be acceptable after maintainer approval, but the implementation should compute one effective isolation value before preflight and execution, keep docs aligned, and include real runtime proof.
