## Summary

Keep open: the default-isolation change is not ready to merge because the PR flips the final run-session decision without applying the same default before heartbeat preflight, and it leaves public config-agent docs plus changelog coverage incomplete.

## What This Changes

The PR changes heartbeat runs with omitted `isolatedSession` to use a fresh `:heartbeat` session by default, updates heartbeat docs/type comments, and adds model-override regression tests.

## Best Possible Solution

If maintainers want default isolation, use one shared isolation-default resolver for both preflight and run-session selection, update every heartbeat config doc plus the changelog, and add omitted-flag wake/system-event regression coverage.

## Reproduction Assessment

Yes for the review findings by source inspection: the PR diff changes only the later `useIsolatedSession` condition, while current preflight still branches on true-only isolation semantics before that line runs. Existing explicit-isolated tests show the wake/system-event behavior that omitted default isolation should match.

## Solution Assessment

No. The one-line condition is the right narrow shape for selecting the final run session, but it is not a complete solution unless the same default is applied to preflight, all public docs, changelog, and omitted-flag regression coverage.
