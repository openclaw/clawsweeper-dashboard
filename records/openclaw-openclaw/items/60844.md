## Summary

Keep open. The PR is useful but not merge-ready against current main: the default-isolation flip is applied only at run-session selection, while heartbeat preflight still has explicit-true isolation semantics; the current docs surface also has a stale config-agent reference, and the user-visible default change needs maintainer approval plus a changelog entry.

## What This Changes

The PR changes heartbeat runs with omitted `isolatedSession` to use a `:heartbeat` session by default, updates heartbeat docs/type comments, and adds model-override regression tests.

## Best Possible Solution

If maintainers approve default isolation, use one shared isolation-default resolver for preflight and run-session selection, update every heartbeat config doc plus the changelog, and add omitted-flag regression coverage for hook/wake and event paths.

## Reproduction Assessment

Yes for the review findings by source inspection. Current main's preflight still branches on explicit `isolatedSession === true`, while the PR changes only the later run-session default and existing explicit-isolated tests show the omitted default should match isolated wake behavior.

## Solution Assessment

No. The direction may be acceptable after maintainer approval, but the current patch is incomplete unless the default is resolved before preflight, all current public docs are aligned, and the changelog records the behavior change.

  - body: Changing only this later `useIsolatedSession` condition means omitted `isolatedSession` runs become isolated after preflight has already used shared-session semantics. Use one shared isolation-default resolver before preflight and add omitted-flag wake coverage.

Rebase and update that page too so users do not see conflicting config guidance.
  - confidence: 0.86
