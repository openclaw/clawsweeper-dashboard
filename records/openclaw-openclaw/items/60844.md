## Summary

Keep open: the PR is a plausible fix path for the heartbeat contamination/default-isolation cluster, but it still has a preflight consistency bug, one stale public docs page, and no real heartbeat runtime proof.

## What This Changes

The PR makes heartbeat runs default to a `:heartbeat` isolated session, keeps `isolatedSession: false` as the opt-out, updates docs/type comments, and adds heartbeat regression tests.

## Best Possible Solution

Resolve the effective heartbeat isolation mode once before preflight and execution, update every public heartbeat config reference, and land only after maintainer default-policy approval plus real heartbeat proof.

## Reproduction Assessment

Yes for the review findings by source inspection: the PR flips default isolation only at run-session selection, while preflight and remaining docs still follow the old explicit-true/default-false model. No live heartbeat setup was run in this read-only review.

## Solution Assessment

No as written. The direction may be acceptable after maintainer approval, but the implementation should compute one effective isolation value before preflight and execution, keep docs aligned, and include real runtime proof.
