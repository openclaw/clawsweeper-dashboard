## Summary

Keep open. Current main has not implemented default-on heartbeat isolation, and the PR still has correctness and docs blockers plus missing real behavior proof, so it is neither obsolete nor ready to merge.

## What This Changes

The PR changes heartbeat runs to use an isolated `:heartbeat` session by default, keeps `isolatedSession: false` as an opt-out, updates heartbeat docs/type comments, and adds regression tests.

## Best Possible Solution

Compute one effective heartbeat isolation value before preflight and execution, update every public config reference, and land only after maintainer default-policy approval plus real heartbeat proof.

## Reproduction Assessment

Yes, for the review findings by source inspection: the PR flips run-session selection to default isolated while preflight and one docs surface still follow the old explicit-true/default-false model. No live heartbeat setup was run in this read-only review.

## Solution Assessment

No as written. The direction may be acceptable after maintainer approval, but the implementation should compute one effective isolation mode before preflight/run selection and keep the public docs aligned.
