## Summary

Keep open: the PR is a plausible gateway heartbeat default-change, but it is not implemented on main and still has blocking correctness, documentation, conflict, and real-behavior-proof gaps before merge.

## What This Changes

The PR makes heartbeat runs default to an isolated `:heartbeat` session, keeps `isolatedSession: false` as the opt-out, updates heartbeat docs/type comments, and adds regression coverage.

## Best Possible Solution

If maintainers approve default-on isolation, resolve the effective isolation mode once before preflight and run-session selection, update all public heartbeat config docs, rebase onto current main, and add real heartbeat proof.

## Reproduction Assessment

Yes for the review findings by source inspection: the PR flips the later run-session selection while current preflight still uses explicit-true isolation checks, and docs search shows remaining default-false guidance outside the diff. No live heartbeat setup was run in this read-only review.

## Solution Assessment

No as written. The direction may be acceptable after maintainer approval, but the implementation should apply one resolved isolation default before preflight and execution, keep docs aligned, and provide real runtime proof.
