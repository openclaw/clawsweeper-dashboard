## Summary

Keep open. The PR is still useful, but the default flip is incomplete against current main: it changes the later run-session decision while heartbeat preflight still treats an omitted `isolatedSession` as shared, and one current public config doc would still advertise the old default.

## What This Changes

The PR makes heartbeat runs default to an isolated `:heartbeat` session, keeps `isolatedSession: false` as an opt-out, updates heartbeat docs/type comments, and adds regression tests for default isolation, explicit sharing, and main-session abort state.

## Best Possible Solution

If maintainers want this default change, use one shared isolation-default decision before preflight and run-session selection, preserve the current event-consumption and delivery-routing safeguards, update all heartbeat config docs, and add omitted-flag wake coverage alongside the existing explicit-isolated tests.

## Reproduction Assessment

Yes for the review findings. Static inspection shows the PR flips omitted `isolatedSession` only at run-session selection while current preflight still uses true-only semantics, and existing explicit-isolated tests show the wake/event behavior that omitted default isolation should match.

## Solution Assessment

No as currently staged. The one-line condition is the right narrow shape for the final run-session decision, but it is not the best complete fix unless preflight, docs, and regression coverage are updated consistently.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Apply default isolation before heartbeat preflight:** `src/infra/heartbeat-runner.ts:607`
  - body: The PR only flips `useIsolatedSession` after `resolveHeartbeatPreflight()` has already run. Current preflight still treats omitted `isolatedSession` as shared (`params.heartbeat?.isolatedSession !== true`), so default-isolated wake runs can inspect base-session events even though explicit isolated runs intentionally avoid base-event classification/routing. Make preflight use the same opt-out default, ideally through a shared resolver, and add omitted-flag wake coverage.
  - confidence: 0.91
- **[P2] Update the config-agent heartbeat reference:** `docs/gateway/heartbeat.md:102`
  - body: The PR updates `heartbeat.md` and `configuration-reference.md` to say the default is true, but the current config-agent page still documents `isolatedSession: false // default: false` and describes isolation as only active when true. With this default flip, users would see conflicting public docs unless `docs/gateway/config-agents.md` is updated too.
  - confidence: 0.87
