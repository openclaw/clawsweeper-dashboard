## Summary

Keep open. Current main still persists heartbeat runtime metadata into shared session state, but this PR is not ready because it can cross-pair a preserved model with the heartbeat provider and it deletes unrelated changelog history.

## What This Changes

The PR adds a heartbeat-only `preserveRuntimeModel` option to session persistence, wires it from heartbeat runs, adds session-store regression tests, and changes `CHANGELOG.md`.

## Best Possible Solution

Consolidate on one branch and land a narrow heartbeat persistence fix that preserves valid prior model, provider, and context metadata without cross-pairing providers, while keeping the changelog intact.

## Reproduction Assessment

Yes. Configure shared-session `heartbeat.model` differently from the primary model, run a heartbeat, then inspect `session_status` or the session store; current main selects the heartbeat model and persists that runtime identity back into the shared session.

## Solution Assessment

No. A heartbeat-specific preserve path is the right direction, but this implementation mishandles model-only session entries and carries a destructive changelog rewrite.
