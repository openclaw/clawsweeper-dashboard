## Summary

Keep open. The PR is still useful and the latest diff no longer deletes unrelated changelog entries, but the preserve path can still persist an invalid provider/model pair for model-only session entries.

## What This Changes

The PR adds heartbeat-only runtime-model preservation plumbing, wires it from heartbeat agent runs, adds session-store regression tests, and records the fix in the changelog.

## Best Possible Solution

Land one consolidated heartbeat persistence fix that preserves an existing model/provider/context tuple together, leaves model-only entries without the heartbeat provider, covers that edge case, and closes the duplicate branch after merge.

## Reproduction Assessment

Yes. Configure a shared-session heartbeat model different from the primary model, run a heartbeat, and inspect session_status or the session store; current main persists the heartbeat run provider/model into the shared session entry.

## Solution Assessment

No. The heartbeat-specific preservation hook is the right direction, but the implementation must not synthesize a provider from the heartbeat run when preserving an older model-only session entry.
