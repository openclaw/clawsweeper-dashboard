## Summary

Keep open. The bug is valid on current main, but the latest PR head is still merge-blocked by a provider/model persistence edge case and a destructive changelog rewrite; the open duplicate PR #75567 should be consolidated rather than used to close this PR now.

## What This Changes

The PR adds heartbeat `preserveRuntimeModel` plumbing in session persistence, wires it from agent command runs, adds session-store regression tests, and updates `CHANGELOG.md`.

## Best Possible Solution

Land one consolidated heartbeat persistence fix that treats `heartbeat.model` as run-scoped, preserves valid prior runtime identity without inventing provider/model pairs, and restores the changelog against current main.

## Reproduction Assessment

Yes. Configure `agents.defaults.heartbeat.model` differently from the primary model, let a heartbeat run return `agentMeta.provider/model`, then inspect the session store or `session_status`; current main writes the heartbeat runtime model into the shared session entry.

## Solution Assessment

No. The PR’s direction is appropriate, but the implementation should preserve the prior runtime model/provider as a valid tuple or leave provider unset for model-only entries, and the changelog must be rebased/restored.

## Review Findings
