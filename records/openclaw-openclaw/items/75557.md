## Summary

Keep open. The PR is directionally useful, but it should not merge as-is because the preserve path can pair a preserved model with the heartbeat provider when older session metadata has no `modelProvider`, and the current head also deletes hundreds of unrelated changelog lines; same-author #75567 is still open as the cleaner competing branch.

## What This Changes

The PR adds a heartbeat-only `preserveRuntimeModel` path in agent session persistence, wires it from heartbeat runs, adds session-store regression tests, and rewrites `CHANGELOG.md`.

## Best Possible Solution

Consolidate on one branch and land a narrow heartbeat persistence fix that preserves valid prior model, provider, and context metadata without cross-pairing providers, while keeping the changelog intact.

## Reproduction Assessment

Yes. Configure shared-session `heartbeat.model` differently from the primary model, run a heartbeat, and inspect `session_status` or the session store; current main still writes the heartbeat run's runtime model/context into shared session state.

## Solution Assessment

No. A heartbeat-specific preserve path is the right direction, but this implementation mishandles model-only session entries and carries a destructive changelog rewrite; #75567 or a repaired version of this branch is the safer path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Preserve model-only entries without using the heartbeat provider:** `src/agents/command/session-store.ts:133-135`
  - body: When a stored session has `model` but no `modelProvider`, this branch preserves the old model but fills the provider from the heartbeat run. That can turn a valid inferred/default model such as `claude-opus-4-6` into a persisted `ollama/claude-opus-4-6` pairing after a heartbeat. Leave the provider unset or resolve it from the preserved model/config instead of using `providerUsed` from the heartbeat run.
  - confidence: 0.86

Keep the heartbeat fix entry, but rebase/restore the changelog so the branch does not erase unrelated release history.
