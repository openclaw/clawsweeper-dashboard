## Summary

Keep open. The PR targets a real current-main heartbeat persistence bug, but the latest diff still leaves two merge-blocking preservation defects in the session-store path and should be consolidated with the duplicate open implementation branch.

## What This Changes

The PR adds a heartbeat-only `preserveRuntimeModel` option to session persistence, wires it from heartbeat agent runs, adds session-store regression tests, and updates the changelog.

## Best Possible Solution

Land one consolidated heartbeat persistence fix that treats heartbeat model selection as run-scoped, preserves only valid prior runtime state including model-only entries, and leaves empty prior runtime fields unset.

## Reproduction Assessment

Yes. Configure `agents.defaults.heartbeat.model` differently from the primary model, run a heartbeat, and inspect `session_status` or the session store; current main persists the heartbeat run provider/model into shared runtime fields.

## Solution Assessment

No. The heartbeat-specific preservation boundary is appropriate, but the implementation must avoid writing heartbeat runtime state when there is no prior runtime model and must preserve model-only entries without inventing a provider.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep heartbeat runs from seeding empty runtime state:** `src/agents/command/session-store.ts:128-134`
  - body: When `preserveRuntimeModel` is true and `entry.model` is missing, this branch records the heartbeat run's provider/model/context window in the shared session. A first heartbeat for a session with no stored runtime model would still make status prefer the heartbeat model instead of falling back to the configured primary.
  - confidence: 0.88
- **[P2] Preserve model-only entries without the heartbeat provider:** `src/agents/command/session-store.ts:132-134`
  - body: Session entries may legitimately have `model` without `modelProvider`; status can infer a provider or display the bare model. Passing `entry.modelProvider ?? providerUsed` to `setSessionRuntimeModel` combines the old model with the heartbeat provider, producing invalid pairs such as `ollama/claude-opus-4-6`.
  - confidence: 0.9
