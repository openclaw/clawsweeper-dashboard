## Summary

Keeping this open: source and release evidence confirm the v3-to-v4 pairing mismatch, and the existing open protocol-skew thread does not fully cover the Android Play/beta release-gating and advertised-feature availability asks.

## What This Changes

This issue asks OpenClaw to restore pairing for released Android app v2026.4.5 against v4 gateways and align Android Play/beta release availability with gateway changelog claims.

## Best Possible Solution

Publish or otherwise make available an Android build that speaks gateway protocol v4, and separately decide whether gateways need a narrowly scoped v3 maintenance/upgrade path before advertising Android-only stable features.

## Reproduction Assessment

Yes at source level: a client advertising minProtocol=maxProtocol=3 against current protocol 4 reaches the protocol-mismatch branch before pairing. I did not run the Play Store Android binary or a real phone setup in this read-only sweep.

## Solution Assessment

Unclear as an implementation path: current Android source already speaks v4, so the maintainable fix is likely release/publishing coordination or a deliberately scoped compatibility mode, not an unconditional gateway protocol rollback.
