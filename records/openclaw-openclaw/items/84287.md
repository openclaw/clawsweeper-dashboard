## Summary

Keep open: the report is a plausible current release-skew bug, not stale cleanup. Source and tag inspection show Android v2026.4.5 speaks only Gateway protocol v3 while v2026.5.18/current Gateway requires v4, and the remaining fix needs Android release or compatibility-owner judgment.

## What This Changes

The issue requests a way for the released Android app v2026.4.5, which speaks Gateway protocol v3, to pair with stable gateways that now require protocol v4.

## Best Possible Solution

Publish or otherwise make available a v4-compatible Android build before relying on stable v4 Gateway features, or deliberately restore a tested v3-to-v4 compatibility path and gate future Android changelog claims on published-client availability.

## Reproduction Assessment

Yes, from source and tag inspection: Android v2026.4.5 sends protocol 3 as both min and max, while v2026.5.18/current Gateway requires protocol 4 and closes mismatched clients before pairing. I did not live-run the Play Store app, so this is source-reproducible rather than live reproduced.

## Solution Assessment

Unclear: the source tree already has a v4 Android app, so a code-only update may not solve the reported published-client skew. The best path needs an Android/release owner decision between Play/beta rollout, a tested Gateway compatibility shim, and release-note gating.
