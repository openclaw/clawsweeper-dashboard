## Summary

Keep open: current main still lacks the requested default-off synthetic `<think>` prepend option across config, schema, docs, runner/subscriber params, and the parser path; the linked implementation PR is closed unmerged and no open canonical replacement owns the work.

## What This Changes

The issue asks for a default-off option to synthesize an opening `<think>` tag for local reasoning models before OpenClaw parses or displays assistant output.

## Best Possible Solution

Add a documented default-off global/per-model config field, resolve it before embedded Pi runs, thread it into subscriber params, and apply it exactly once before existing tag parsing when no native opening tag is present.

## Reproduction Assessment

Yes for the feature gap: current-main source inspection shows no accepted config key, runtime schema field, runner/subscriber parameter, docs entry, or parser state for synthetic opening-tag insertion. A live local-model run is not needed to prove OpenClaw cannot enable the requested behavior today.

## Solution Assessment

Yes. A default-off threaded config option with an exactly-once native-tag guard is the narrow maintainable path; prompt-only workarounds remain model-specific and the linked implementation PR did not land.
