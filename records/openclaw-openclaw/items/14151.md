## Summary

Keep open: current main still lacks the requested default-off synthetic `<think>` prepend option, and the related implementation PR is closed unmerged. The current code accepts and documents reasoning visibility, parses existing reasoning tags, and enforces final tags, but it does not expose or thread a config flag that injects an opening tag before parsing.

## What This Changes

Request a default-off option to synthesize opening `<think>` delimiters for local reasoning models that omit them, covering streaming and non-streaming replies.

## Best Possible Solution

Add a documented default-off global and per-model config contract, resolve it before embedded Pi runs, pass it through runner/subscriber params, and inject the opening tag exactly once before existing reasoning extraction when no native opening tag is present.

## Reproduction Assessment

Yes for the feature gap: current-main source inspection shows no accepted config key, schema field, runner/subscriber parameter, docs entry, or parser state for synthetic opening-tag insertion. No live local-model run is needed to verify that OpenClaw cannot enable this behavior today.

## Solution Assessment

Yes. A default-off threaded config option with an exactly-once native-tag guard is the narrow maintainable direction; prompt-only workarounds remain model-specific, and the related implementation PR did not land.
