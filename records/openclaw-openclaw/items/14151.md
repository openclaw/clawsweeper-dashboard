## Summary

Keep open: current main still lacks the requested default-off synthetic `<think>` prepend option. The related implementation PR was closed unmerged, and the current config, schema, runner, subscriber, and message parsing paths still only support parsing or stripping existing tags.

## What This Changes

The issue requests a default-off global or per-model config option to synthesize an opening `<think>` tag for local reasoning models that omit native think delimiters.

## Best Possible Solution

The desired end state is a documented default-off global and per-model setting resolved before embedded Pi runs and applied once before tag parsing when no native opening tag is present.

## Reproduction Assessment

Yes for the feature gap: current-main source inspection shows no accepted config key, schema field, runner/subscriber parameter, docs entry, or parser state for synthetic opening-tag insertion. No live local-model run is needed to verify that OpenClaw cannot enable this behavior today.

## Solution Assessment

Yes. A default-off threaded config option with an exactly-once native-tag guard is the narrow maintainable direction; prompt-only workarounds remain model-specific, and the related implementation PR did not land.
