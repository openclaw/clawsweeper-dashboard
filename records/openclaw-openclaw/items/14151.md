## Summary

Keep open: current main still lacks a default-off synthetic `<think>` opening-tag option, and the directly related implementation PR is closed unmerged. The current source still only parses and strips existing reasoning tags; it does not expose a config key or runner/subscriber flag to inject one.

## What This Changes

Request to add a user-configurable option that prepends an opening `<think>` tag for local reasoning models that do not emit reasoning delimiters reliably.

## Best Possible Solution

Add a documented default-off global and per-model config contract, resolve it before embedded Pi runs, and inject the opening tag exactly once before existing reasoning extraction when no native opening tag is present.

## Reproduction Assessment

Yes for the feature gap: current-main source inspection shows no accepted config key, schema field, runner/subscriber parameter, docs entry, or parser state for synthetic opening-tag insertion. No live local model is needed to verify that OpenClaw cannot enable this behavior today.

## Solution Assessment

Yes. A default-off threaded config option with an exactly-once native-tag guard is the narrow maintainable direction; prompt templates remain model-specific and the related implementation PR did not land.
