## Summary

Keep open: current main still lacks the requested default-off synthetic `<think>` opening-tag option, and the related implementation PR is closed unmerged. Source inspection shows existing code only parses and strips native reasoning tags; it does not expose a config key or runner/subscriber flag to inject one.

## What This Changes

Request a default-off config option that injects an opening `<think>` tag for selected local reasoning model outputs before existing reasoning parsing.

## Best Possible Solution

Add a documented default-off global and per-model config contract, resolve it before embedded Pi runs, and inject the opening tag exactly once before existing reasoning extraction when no native opening tag is present.

## Reproduction Assessment

Yes for the feature gap: current-main source inspection shows no accepted config key, schema field, runner/subscriber parameter, docs entry, or parser state for synthetic opening-tag insertion. No local model run is needed to verify that OpenClaw cannot enable this behavior today.

## Solution Assessment

Yes. A default-off threaded config option with an exactly-once native-tag guard is the narrow maintainable direction; prompt templates remain model-specific and the related implementation PR did not land.
