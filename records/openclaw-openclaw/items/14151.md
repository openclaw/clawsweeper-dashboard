## Summary

Keep open: current main still lacks the requested default-off synthetic `<think>` opening-tag option in the config contract, runner/subscriber parameters, parser state, and docs, while the related implementation PR is closed unmerged.

## What This Changes

Add a default-off option to synthesize an opening `<think>` delimiter for selected local reasoning model outputs before existing reasoning parsing.

## Best Possible Solution

Add a documented global and per-model default-off config contract, thread it through embedded Pi run and subscriber params, and inject once before existing reasoning extraction only when no native opening tag is present.

## Reproduction Assessment

Yes for the feature gap: source inspection on current main shows no accepted config key, schema field, runner/subscriber parameter, docs entry, or parser state for synthetic opening-tag insertion. No live GLM, MiniMax, or step model is needed to verify that OpenClaw cannot enable this behavior today.

## Solution Assessment

Yes. A default-off threaded config option with an exactly-once native-tag guard is the narrow maintainable direction; prompt-template workarounds remain provider/model-specific and the related closed PR did not land the needed production path.
