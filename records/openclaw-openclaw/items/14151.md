## What This Changes

The issue requests a user-configurable OpenClaw option to synthesize an opening `<think>` tag for local reasoning model outputs before reasoning parsing and display.

## Best Possible Solution

Add a documented, default-off config contract at global and per-model scope, thread it through embedded Pi run/subscriber params, and inject exactly once before existing reasoning extraction when no native opening tag is present.
