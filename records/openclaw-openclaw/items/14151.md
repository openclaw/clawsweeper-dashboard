## Summary

Keep open: current main still lacks the requested default-off synthetic `<think>` opening-tag option across config, schema, runner/subscriber parameters, docs, and parser state. The related implementation PR #14323 is closed unmerged, so there is no landed fix to close against.

## What This Changes

Request a default-off configurable option to prepend an opening `<think>` tag for local reasoning models that omit native reasoning delimiters.

## Best Possible Solution

Add a documented global and per-model default-off config contract, thread it through embedded Pi run/subscriber params, and inject once before existing reasoning extraction only when no native opening tag is present.

## Solution Assessment

Yes. A default-off threaded config option with an exactly-once native-tag guard is the narrow maintainable direction; prompt-template workarounds remain provider/model-specific and do not satisfy the requested orchestrator boundary.
