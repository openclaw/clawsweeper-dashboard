## What This Changes

The issue requests a user-configurable option to prepend an opening `<think>` tag for local reasoning models that do not emit reasoning delimiters reliably.

## Best Possible Solution

Land a maintainer-approved, default-off config contract that threads through the embedded runner and subscriber, applies once before reasoning extraction, and preserves native `<think>` output without double tagging.

## Reproduction Assessment

Yes for the missing capability: current-main source inspection shows reasoning visibility and tag parsing exist, but no accepted config or runtime parameter force-prepends an opening `<think>` tag. No live model stream was reproduced, so this is a source-proven feature gap rather than a current-main bug repro.

## Solution Assessment

Unclear until maintainers choose the config shape. A default-off option is a plausible narrow solution, but the best implementation must define precedence, avoid double-prepending, and prove both streaming and final-text parsing paths.
