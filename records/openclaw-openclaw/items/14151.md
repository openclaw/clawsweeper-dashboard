## Summary

Keep open: current main still lacks a default-off synthetic `<think>` prepend option in config, runner parameters, subscriber parameters, and the message parsing path; the only direct implementation PR is closed unmerged, and adjacent enforce-final-tag work does not solve this request.

## What This Changes

Requests a user-configurable option to prepend an opening `<think>` tag for local reasoning models that do not emit reasoning delimiters consistently.

## Best Possible Solution

Add a documented default-off global and per-model config contract, resolve precedence before embedded Pi runs, thread it through runner/subscriber params, and inject the opening tag exactly once before existing reasoning extraction when no native opening tag is present.

## Reproduction Assessment

Yes for the feature gap: current-main source inspection shows no accepted config key, schema field, runner/subscriber parameter, docs entry, or parser state for synthetic opening-tag insertion. No live local-model run is needed to verify that OpenClaw cannot enable this behavior today.

## Solution Assessment

Yes. A default-off threaded config option with an exactly-once native-tag guard is the narrow maintainable direction; prompt-only workarounds and final-tag enforcement do not provide the requested synthetic opening-tag boundary.
