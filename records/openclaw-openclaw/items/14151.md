## What This Changes

The issue requests a user-configurable option to prepend an opening `<think>` tag for local reasoning models that omit reasoning delimiters.

## Best Possible Solution

Add a documented default-off config field, resolve global and per-model precedence before embedded Pi runs, thread it through runner and subscriber params, and apply it exactly once before existing tag parsing only when no native opening tag is present.

## Reproduction Assessment

Yes for the feature gap: source inspection shows no accepted config key, schema field, runner/subscriber parameter, docs entry, or parser state for synthetic opening-tag insertion. A live local-model run is not needed to prove OpenClaw cannot enable the requested behavior today.

## Solution Assessment

Yes. A default-off threaded config option with an exactly-once native-tag guard is the narrow maintainable path; prompt-only workarounds remain model-specific, and the linked implementation PR did not land.
