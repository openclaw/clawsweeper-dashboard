## Summary

Keep open: current main still lacks the requested default-off synthetic `<think>` opening-tag option across typed config, strict schema, runner/subscriber parameters, docs, and parsing. The directly related implementation PR #14323 is closed unmerged, so there is no landed fix to close against.

## What This Changes

The issue requests a configurable default-off option to inject an opening `<think>` tag for local reasoning models that omit reasoning delimiters.

## Best Possible Solution

Add a documented global and per-model config contract, thread it through embedded Pi run/subscriber params, and inject exactly once before existing reasoning extraction when no native opening tag is present.

## Solution Assessment

Yes. A default-off threaded config option is the narrow maintainable direction; prompt-template workarounds remain model-specific and would not satisfy the requested orchestrator boundary.
