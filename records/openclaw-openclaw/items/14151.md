The issue requests a default-off option to prepend an opening `<think>` tag for local reasoning models whose output lacks reasoning delimiters.

## Best Possible Solution

If maintainers accept the direction, add a narrow default-off config/runtime contract that injects an opening tag exactly once before existing reasoning parsing, with explicit precedence and streaming plus final-text coverage.

## Solution Assessment
