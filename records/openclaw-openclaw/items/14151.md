Keep open. Current main and the latest release still lack a first-class/default-off force-prepend `<think>` option, and the only linked implementation is closed unmerged, so the requested capability still needs maintainer product review.

## What This Changes

The issue requests a default-off config/runtime option to synthesize opening `<think>` delimiters for local reasoning models that do not emit native think tags.

## Best Possible Solution

Maintainers should first decide whether this new core config surface is acceptable; if accepted, implement a default-off option with explicit precedence and exactly-once streaming/final-text parser guards.

## Reproduction Assessment

Yes from source, not from a live model: current config and subscriber types have no force-prepend setting, and the message path only parses existing `<think>` tags. No live GLM/MiniMax/step stream was reproduced in this read-only review.

## Solution Assessment

Unclear until product direction is confirmed. A default-off, exactly-once parser/config option is plausible, but OpenClaw's high bar for new config means maintainer approval should come before implementation.
