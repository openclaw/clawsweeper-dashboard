## Summary

Keep open: current main still lacks the requested default-off synthetic `<think>` opening-tag option across config, schema, runner/subscriber parameters, docs, and parsing. The directly related PR #14323 is closed unmerged, so there is no landed implementation to close against.

## What This Changes

Request a default-off global or per-model option to prepend an opening `<think>` tag to local reasoning-model output before existing reasoning parsing, including streaming support.

## Best Possible Solution

Add a documented, default-off config contract at the global and per-model scopes, resolve it before embedded Pi runs, pass it through runner and subscriber params, and inject exactly once before existing reasoning extraction when no native opening tag is present.

## Reproduction Assessment

Yes. The feature gap is reproducible by static inspection on current main: the proposed key is absent from exported types, strict schema, docs, run params, subscribe params, and parser state, so no live GLM, MiniMax, or step model is required.

## Solution Assessment

Yes. A default-off threaded option is the narrow maintainable solution; prompt-template-only workarounds would remain model-specific and would not satisfy the requested orchestrator boundary.
