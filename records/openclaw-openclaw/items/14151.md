## Summary

Keep open. Current main still lacks the requested default-off synthetic `<think>` opening-tag option across config, strict schema validation, embedded runner parameters, subscriber parameters, docs, and the message parsing path; the directly related implementation PR was closed unmerged and no open replacement PR appears to supersede this issue.

## What This Changes

The issue requests a default-off OpenClaw option to synthesize an opening `<think>` tag for local reasoning-model outputs before existing reasoning parsing and display paths run.

## Best Possible Solution

Implement a documented, default-off config contract at the appropriate global and per-model scopes, resolve it before embedded Pi runs, pass it through runner and subscriber parameters, and inject exactly once before existing reasoning extraction only when the assistant text does not already start with a native reasoning tag.

## Reproduction Assessment

Yes. The feature gap is reproducible by static inspection on current main: the proposed config key is absent from exported types and strict Zod schema, and no runner/subscriber/message-handler parameter exists to exercise, so no live GLM, MiniMax, or step model is required to verify the missing OpenClaw control surface.

## Solution Assessment

Yes. A default-off option threaded through agent defaults, embedded runner params, subscriber params, and the existing parser is the narrow maintainable solution; prompt-template-only workarounds would remain model-specific and would not satisfy the requested orchestrator boundary.
