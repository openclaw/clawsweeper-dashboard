## Summary

Keep open: current main and the latest shipped tag still lack the requested default-off synthetic `<think>` prepending option, and the paired implementation PR at https://github.com/openclaw/openclaw/pull/14323 closed unmerged rather than landing the feature.

## What This Changes

The issue requests a user-configurable, default-off way to prepend a synthetic opening `<think>` tag for local reasoning models that omit native reasoning delimiters.

## Best Possible Solution

Add a documented default-off config field, resolve global/per-model precedence before embedded Pi runs, thread it through runner/subscriber params, and apply it exactly once before existing tag parsing only when no native opening tag is present.

## Reproduction Assessment

Yes for the feature gap: current-main source inspection shows no accepted config key, schema field, runner/subscriber parameter, docs entry, or parser state for synthetic opening-tag insertion. A live local-model run is not needed to prove OpenClaw cannot enable the requested behavior today.

## Solution Assessment

Yes. A default-off threaded config option with an exactly-once native-tag guard is the narrow maintainable path; prompt-only workarounds remain model-specific and the linked implementation PR did not land.
