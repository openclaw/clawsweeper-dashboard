Request for a default-off option to synthesize `<think>` reasoning delimiters for local reasoning models that omit them.

## Best Possible Solution

If maintainers accept the new config surface, implement one canonical default-off setting in agent config, thread it through the embedded runner and subscriber, apply it once before existing reasoning extraction with a native-tag guard, and document the precedence.

## Solution Assessment
