## Summary

Keep open: current main still lacks a default-off synthetic `<think>` opening-tag option, and the directly related implementation PR is closed unmerged. The existing maintainer and ClawSweeper comments remain consistent with current source inspection.

## What This Changes

The issue requests a configurable way to prepend an opening `<think>` tag for local reasoning models that do not reliably emit reasoning delimiters.

## Best Possible Solution

Add a documented default-off global and per-model config contract, resolve it before embedded Pi runs, and inject the opening tag exactly once before existing reasoning extraction when no native opening tag is present.

## Reproduction Assessment

Yes for the feature gap: source inspection on current main shows no accepted config key, schema field, runner/subscriber parameter, docs entry, or parser state for synthetic opening-tag insertion. No live local model is needed to verify that OpenClaw cannot enable this behavior today.

## Solution Assessment

Yes. A default-off, threaded config option with an exactly-once native-tag guard is the narrow maintainable direction; prompt templates would remain model-specific and the related implementation PR did not land.
