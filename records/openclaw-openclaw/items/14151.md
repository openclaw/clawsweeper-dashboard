## Summary

Keep open: current main still lacks a default-off synthetic `<think>` prepend option, and the only linked implementation PR is closed unmerged. The source still has reasoning visibility, final-tag enforcement, and malformed-tag cleanup, but no accepted config key or runner/subscriber/parser flag that can prepend an opening tag before existing reasoning parsing.

## What This Changes

The issue requests a configurable default-off option to prepend an opening `<think>` tag for local reasoning models that do not reliably emit reasoning delimiters.

## Best Possible Solution

Add a documented default-off global and per-model config contract, resolve precedence before embedded Pi runs, thread it through runner and subscriber params, and inject the opening tag exactly once before existing reasoning extraction when no native opening tag is present.

## Reproduction Assessment

Yes for the feature gap: current-main source inspection shows no accepted config key, schema field, runner/subscriber parameter, docs entry, or parser state for synthetic opening-tag insertion. No live local-model run is needed to verify that OpenClaw cannot enable this behavior today.

## Solution Assessment

Yes. A default-off threaded config option with an exactly-once native-tag guard is the narrow maintainable direction; prompt-only workarounds remain model-specific, and the related implementation PR did not land.
