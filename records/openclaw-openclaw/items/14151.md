## What This Changes

Requests a configurable default-off way to synthesize an opening `<think>` tag for local reasoning model outputs before existing parsing/stripping runs.

## Best Possible Solution

Add a documented default-off config field, resolve global/per-model precedence before embedded Pi runs, thread it through runner and subscriber params, and apply it exactly once before existing tag parsing only when no native opening tag is present.

## Reproduction Assessment

Yes for the feature gap: current-main source inspection shows no accepted config key, schema field, runner/subscriber parameter, docs entry, or parser state for synthetic opening-tag insertion. A live local-model run is not needed to prove OpenClaw cannot enable the requested behavior today.

## Solution Assessment

Yes. A default-off threaded config option with an exactly-once native-tag guard is the narrow maintainable path; prompt-only workarounds remain model-specific, and the linked implementation PR did not land.
