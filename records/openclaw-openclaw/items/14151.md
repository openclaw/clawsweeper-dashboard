## Summary

Keep open: current main still does not expose a default-off synthetic `<think>` prepend setting in config, schemas, runner/subscriber params, docs, or message parsing, and the linked implementation PR is closed unmerged.

## What This Changes

Requested a configurable way to prepend an opening `<think>` tag for local reasoning models that omit native reasoning delimiters.

## Best Possible Solution

Add a documented default-off global and per-model config field, resolve it before embedded Pi runs, thread it into runner/subscriber params, and apply it exactly once before existing tag parsing when no native opening tag is present.

## Reproduction Assessment

Yes for the feature gap: current-main source inspection shows no accepted config key, runtime schema field, runner/subscriber parameter, docs entry, or parser state for synthetic opening-tag insertion. A live local-model run is not needed to prove OpenClaw cannot enable the requested behavior today.

## Solution Assessment

Yes. A default-off threaded config option with an exactly-once native-tag guard is the narrow maintainable path; prompt-only workarounds remain model-specific, and the linked implementation PR did not land.
