## Summary

No duplicate or landed generic implementation exists, and current main still cannot infer reasoning boundaries from arbitrary untagged plain text. However, the original force-prepend setting is no longer clearly the best fix because current architecture uses provider/model response compatibility and structured thinking events.

## What This Changes

Add an opt-in way to normalize reasoning boundaries for local models that emit reasoning without `<think>` tags.

## Best Possible Solution

Define a model/provider response-compatibility contract from real captured payloads, then normalize that shape before the current subscriber while preserving native tags, structured thinking, streaming boundaries, and final answers.

## Reproduction Assessment

Yes at source level: current main has no generic path that treats wholly untagged plain text as reasoning. A high-confidence model-specific reproduction still needs redacted raw streaming and final payloads from the named models.

## Solution Assessment

No, the original generic force-prepend boolean is not yet proven to be the best solution; current architecture favors a typed provider/model response-compatibility rule based on the actual emitted payload dialect.
