## Summary

Keep open: current main still lacks the requested default-off synthetic `<think>` opening-tag option in the typed config, schema, embedded runner/subscriber parameters, parsing state, and docs; the only directly related implementation PR is closed unmerged.

## What This Changes

The issue requests a configurable default-off way to prepend an opening `<think>` tag for reasoning models that omit native thinking delimiters.

## Best Possible Solution

Add a documented global and per-model default-off config contract, resolve it before embedded Pi runs, thread it through runner/subscriber params, and inject once before existing reasoning extraction only when no native opening tag is present.

## Reproduction Assessment

Yes for the feature gap: source inspection on current main shows no accepted config key, schema field, runner/subscriber parameter, docs entry, or parser state for synthetic opening-tag insertion. No live GLM, MiniMax, or step model is needed to verify that OpenClaw cannot enable this behavior today.

## Solution Assessment

Yes. A default-off, threaded config option with an exactly-once native-tag guard is the narrow maintainable direction; prompt-template workarounds remain provider/model-specific and the related implementation PR did not land.
