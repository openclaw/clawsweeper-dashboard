## Summary

Keep open: current main still lacks the requested default-off synthetic `<think>` prepend option, and the related implementation PR is closed unmerged. The current code only parses and strips native reasoning tags; it does not expose a config key or runner/subscriber flag to inject one.

## What This Changes

The issue requests a user-configurable way to prepend an opening `<think>` tag before OpenClaw parses reasoning output from local reasoning models.

## Best Possible Solution

Add a documented default-off global and per-model config contract, resolve it before embedded Pi runs, pass it through runner/subscriber params, and inject the opening tag exactly once before existing reasoning extraction when no native opening tag is present.

## Reproduction Assessment

Yes for the feature gap: current-main source inspection shows no accepted config key, schema field, runner/subscriber parameter, docs entry, or parser state for synthetic opening-tag insertion. No live local-model run is needed to verify that OpenClaw cannot enable this behavior today.

## Solution Assessment

Yes. A default-off threaded config option with an exactly-once native-tag guard is the narrow maintainable direction; prompt templates remain model-specific and the related implementation PR did not land.
