## Summary

Current main still treats wholly untagged assistant output as visible text and has no setting or runtime signal that enables synthetic reasoning boundaries; the linked implementation PR closed unmerged, so the requested interoperability feature remains necessary.

## What This Changes

Add a default-off global or per-model option that prepends an opening reasoning delimiter before current streaming and final-output parsing.

## Best Possible Solution

Add one documented, default-off canonical setting with per-model override resolution, thread the resolved value through the embedded runner, and normalize exactly once before the existing partitioner while preserving native tags and a deterministic final-answer boundary.

## Reproduction Assessment

Yes at source level: current main emits a wholly untagged assistant stream as visible text because the canonical partitioner enters reasoning only after a recognized opening tag. A live run with the named local models was not performed.

## Solution Assessment

Yes, with safeguards: a default-off model-resolved setting threaded into the canonical subscriber path is the narrowest implementation already endorsed in the issue discussion, provided it prevents double tags and defines the final-answer transition.
