## Label Justifications

- `P3`: This is a bounded opt-in enhancement for a limited group of local reasoning models, with no evidence of an urgent core regression.
- `impact:session-state`: Absent reasoning boundaries can cause reasoning text to be misclassified in streamed output and assistant transcript state.

## Summary

Current main still lacks any config, runner parameter, or subscriber state that can classify a wholly untagged assistant text stream as reasoning; native reasoning events, ordinary tagged streams, and orphan closing tags are handled, but they do not cover the reported no-delimiter behavior. The only linked implementation PR closed unmerged, so the feature remains necessary and should stay open as a focused, low-priority enhancement.

## What This Changes

Add a default-off global or per-model option that treats the beginning of an untagged local reasoning-model response as reasoning before the existing streaming parser runs.

## Best Possible Solution

Add one default-off canonical config contract with documented global/per-model precedence, resolve it before the embedded run, and normalize only wholly untagged text before the existing reasoning state machine while leaving native structured reasoning and native opening tags untouched.

## Reproduction Assessment

Yes at the source level: an assistant text stream containing reasoning and an answer but no native thinking event and no reasoning delimiter follows the ordinary visible-text path, and current main has no setting that changes that classification. This review did not run the named live models.

## Solution Assessment

Yes, if implemented through the current canonical embedded-agent pipeline rather than copied from the closed PR: a single default-off setting should be fully wired and guarded against native reasoning events and native tags.
