## Summary

Close as support/config: current source and docs already define group/channel turns as message-tool-only by default, so typing or processing with no visible room message is expected when the model answers privately instead of calling the message tool.

## What This Changes

The report asks OpenClaw to deliver visible replies back into Discord, Slack, and Telegram group/channel contexts when a mentioned message is processed there.

## Best Possible Solution

Keep the documented message-tool-only default for group/channel rooms and direct users who want legacy room auto-posting to set `messages.groupChat.visibleReplies: "automatic"`; a new bug should include logs showing an attempted message-tool send that failed.

## Reproduction Assessment

Yes for the reported symptom class: source inspection and docs show that group/channel turns process while automatic visible replies are suppressed by default. The reporter’s exact provider setup was not live-reproduced, and a distinct failed message-tool send would need logs.

## Solution Assessment

Yes for cleanup: the current config knob and docs are the narrow maintainable answer, while changing code to auto-post all group replies would reverse an intentional safety default.
