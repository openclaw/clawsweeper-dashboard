## Summary

Keep open: the trusted-exec policy may be acceptable, but the patch currently bases `apply_patch` inheritance only on `security` and `ask`, so node or unavailable-sandbox exec configurations can gain local outside-workspace writes without equivalent exec authority. The docs also remain internally inconsistent.

## What This Changes

The PR lets `apply_patch` inherit outside-workspace write access when `tools.exec.security=full` and `tools.exec.ask=off`, with related tests and docs updates.

## Best Possible Solution

If maintainers accept the trusted-exec policy, derive inheritance from effective exec authority including host/target and sandbox availability, keep explicit `applyPatch.workspaceOnly=true` and `tools.fs.workspaceOnly=true` as hard containment overrides, and update all duplicated docs/tests.

## Reproduction Assessment

Yes, source-reproducible: current main keeps `apply_patch` workspace-contained unless `tools.exec.applyPatch.workspaceOnly` is explicitly false, while the PR body supplies after-fix live output for explicit full/off config.

## Solution Assessment

No as written: matching `apply_patch` to trusted exec is plausible if maintainers accept the policy, but the fallback must use effective exec authority instead of only `security` and `ask`, and the duplicate docs must be fixed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Gate apply_patch inheritance on the effective exec host:** `src/agents/pi-tools.ts:607`
  - body: The new fallback checks only `security=full` and `ask=off`. With `tools.exec.host="node"`, or `host="sandbox"` when no sandbox runtime is available, exec does not have equivalent local gateway write authority, but `apply_patch` will still run locally with `workspaceOnly:false` and can write outside the workspace. Derive this from the same effective exec target/host before dropping the workspace guard.
  - confidence: 0.9
- **[P3] Update the duplicate apply_patch default docs:** `docs/tools/exec.md:110`
  - body: This adds the inheritance note in the config section, but the same page's `apply_patch` subsection still says `tools.exec.applyPatch.workspaceOnly` defaults to true unconditionally. Readers who jump to that section get the wrong security behavior after this change, so update the duplicate bullet too.
  - confidence: 0.86
