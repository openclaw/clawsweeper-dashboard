## Summary

Keep open for maintainer review. The patch is narrow and has after-fix real behavior proof, but it intentionally broadens the `apply_patch` write boundary under trusted full/off exec and is not already on `main`.

## What This Changes

This PR lets `apply_patch` inherit configured `tools.exec.security=full` plus `tools.exec.ask=off` when `applyPatch.workspaceOnly` is unset, with focused tests and docs/schema help updates.

## Best Possible Solution

If maintainers accept the trusted-exec policy, merge the narrow inheritance with the included regression tests and docs; otherwise keep the explicit `tools.exec.applyPatch.workspaceOnly=false` requirement.

## Reproduction Assessment

Yes, source-reproducible rather than locally executed: current `main` forces `apply_patch` workspace containment unless `applyPatch.workspaceOnly=false`, so full/off exec with the setting unset remains blocked today.

## Solution Assessment

Likely yes if the policy is accepted: the PR changes the single tool-construction decision and keeps approval-required exec, `tools.fs.workspaceOnly`, and explicit `applyPatch.workspaceOnly=true` contained.
