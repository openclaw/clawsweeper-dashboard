## What This Changes

The issue reports severe gateway slowdown when a very large `commands.ownerAllowFrom` list is normalized and checked during inbound command authorization.

## Best Possible Solution

Land or replace the linked PR with a narrow `command-auth` fix that caches or compiles owner authorization lookups while preserving provider-prefix filtering, wildcard behavior, owner-only tool security, and config-reload invalidation.

## Reproduction Assessment

Yes, source-level. Configure a large `commands.ownerAllowFrom` list and send an inbound message through command/directive handling; current main resolves owner allowlists, rebuilds owner arrays, and performs linear membership checks on that path.

## Solution Assessment

Yes, directionally. A narrow cached or Set-based command-auth lookup is the maintainable fix; wildcard owner access changes security semantics, and broader config include/per-tool ACL ideas are separate feature work.
