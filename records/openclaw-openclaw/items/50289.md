## Summary

Keep open: current main still has the reported command-authorization hot path, and an open PR with closing syntax already targets the issue.

## What This Changes

The issue asks to remove severe gateway latency caused by large `commands.ownerAllowFrom` lists by caching or compiling owner authorization lookups.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/73440 after repair, or an equivalent narrow command-auth change that caches compiled owner authorization state while preserving provider-prefix filtering and fail-closed owner-only behavior.

## Reproduction Assessment

Yes, from source inspection. Configure a large `commands.ownerAllowFrom` list and send an inbound message; current main builds command context, resolves owner allowlists, and scans owner membership linearly.

## Solution Assessment

Yes, directionally. A narrow cache or compiled lookup inside `src/auto-reply/command-auth.ts` is the maintainable fix boundary; wildcard owner access or broader access-control redesign would change security semantics.
