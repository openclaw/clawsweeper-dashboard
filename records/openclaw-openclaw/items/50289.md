## What This Changes

Optimize command authorization so large `commands.ownerAllowFrom` lists do not cause severe per-message gateway slowdown.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/73440 after any needed repair, or an equivalent narrow command-auth change that caches compiled owner authorization state while preserving provider-prefix filtering and fail-closed owner-only behavior.

Yes, from source inspection. Configure a large `commands.ownerAllowFrom` list and send an inbound message; current main builds command context, resolves owner allowlists, and performs linear membership checks on that path.

## Solution Assessment

Yes, directionally. A narrow command-auth cache or compiled lookup is the maintainable fix boundary; wildcard owner access or a broad access-control redesign would change security semantics.
