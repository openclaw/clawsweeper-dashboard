## What This Changes

The issue requests a performance fix for large commands.ownerAllowFrom allowlists that slow gateway command authorization on inbound messages.

## Best Possible Solution

Land a narrow command-auth fix that caches or compiles owner authorization lookups while preserving provider-prefix filtering, wildcard behavior, owner-only tool security, and config-reload invalidation.

Yes, source-level. Configure a large commands.ownerAllowFrom list and send an inbound message; current main still builds command context, resolves owner allowlists, rebuilds owner arrays, and performs linear membership checks on that path.

## Solution Assessment

Yes, directionally. A narrow cached or Set-based command-auth lookup is the maintainable fix; wildcard owner access would change security semantics and should remain only a workaround.
