## Summary

Keep open: current main still has the reported command-authorization hot path, and an open linked PR is already the implementation path for the issue.

## What This Changes

Large commands.ownerAllowFrom lists should stop causing per-message linear owner-authorization work and gateway latency.

## Best Possible Solution

Land a narrow command-auth fix that caches compiled owner authorization lookups while preserving provider-prefix filtering, wildcard semantics, and config-reload invalidation.

## Reproduction Assessment

Yes, source-level. Configure a large commands.ownerAllowFrom list and send an inbound message; current main builds command context, resolves owner allowlists, rebuilds owner arrays, and performs linear membership checks on that path.

## Solution Assessment

Yes, directionally. A narrow compiled cache or Set-based lookup in command-auth is the maintainable fix; wildcard owner access would change security semantics and should remain only a workaround.
