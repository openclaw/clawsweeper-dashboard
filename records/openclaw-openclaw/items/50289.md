## What This Changes

The issue asks OpenClaw to avoid severe gateway latency when commands.ownerAllowFrom contains thousands of owner IDs.

## Best Possible Solution

Review and land or replace the linked PR with a narrow command-auth fix that caches or compiles owner authorization lookups while preserving provider-prefix filtering, wildcard semantics, owner-only tool security, and config-reload invalidation.

## Reproduction Assessment

Yes, source-level. Configure a large commands.ownerAllowFrom list and send an inbound command or directive; current main reaches resolveCommandAuthorization, resolves owner allowlists twice, rebuilds owner arrays, and performs linear sender membership checks.

## Solution Assessment

Yes, directionally. A narrow cached or Set-backed owner authorization lookup is the maintainable fix; wildcard owner access changes security semantics, and broader config include or per-tool ACL ideas are separate feature work.
