## Summary

Keep open: current main and v2026.6.10 still perform raw per-authorization `ownerAllowFrom` list walking and linear owner membership checks, and the linked closing PR remains open and unmerged.

## What This Changes

The issue reports severe Gateway slowdown when `commands.ownerAllowFrom` contains thousands of entries because owner authorization normalizes and searches large allowlists on inbound command paths.

## Best Possible Solution

Keep this issue as the canonical report until the open linked PR lands or is replaced by a narrow command-auth fix that caches compiled owner authorization lookups and proves the large-list Gateway path.
