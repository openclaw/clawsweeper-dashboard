## Summary

Keep open. Current main still lacks a cross-HOME managed gateway service-root safeguard; adjacent update, doctor, and config-drop hardening do not find the old dedicated-user systemd unit before a root-invoked refresh can use root's config/state.

## What This Changes

The issue asks OpenClaw update to preserve, migrate, or warn before abandoning gateway config/state when the managed service HOME changes.

## Best Possible Solution

Add a Linux package-update guard that identifies the previously installed managed gateway root before rewriting it, compares HOME and OPENCLAW_* roots, and safely preserves the prior root or blocks with explicit migration guidance.

## Reproduction Assessment

Yes, as a source-reproducible path: install the user systemd gateway under HOME=/home/openclaw, then invoke the package update with HOME=/root and observe service discovery using root's HOME-derived unit/config paths. I did not run the live cross-user systemd flow in this read-only review.

## Solution Assessment

Not applicable because there is no patch on this issue. The requested preserve/migrate/warn direction is appropriate, but the exact service-root policy should be chosen by maintainers because it intersects stale service-env and credential-root hardening.
