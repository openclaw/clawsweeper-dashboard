## Summary

Close: current main now blocks root/euid-root OpenClaw CLI invocations before state, config, or service mutation, which prevents the reported `/home/openclaw` to `/root` silent config split. The fix is current-main-only after `v2026.5.7`.

## What This Changes

The issue asks OpenClaw updates to preserve, migrate, or clearly warn when a service `HOME` change would abandon an existing config/state directory.

## Best Possible Solution

Keep the root guard and non-root server setup guidance on main, and handle already-split installations through recovery/manual cleanup rather than startup-time config migration.

## Reproduction Assessment

No failing current-main reproduction is expected: the reported root/HOME path is now blocked at CLI startup before state, config, or service writes. The historical failure remains well supported by the issue discussion and the related root-state issue that the guard closed.

## Solution Assessment

Yes. Refusing root/euid-root CLI execution before mutable startup work is the narrow preventive fix for this reported `/root` split, while the explicit override preserves intentional root/container cases.
