## What This Changes

The issue reports that the 2026.3.24 update changed the systemd service HOME from `/home/openclaw` to `/root`, abandoned the prior config/state root, and left the gateway broken.

## Best Possible Solution

Add a Linux update/service-refresh guard that identifies the previously installed gateway service root before rewrite and either preserves it safely or blocks with doctor-backed migration guidance.

## Reproduction Assessment

Yes, source-level: current main still resolves the user-systemd unit from the current HOME and service update reads service state from the updater environment. I did not run a live cross-user systemd upgrade in this read-only review.

## Solution Assessment

Unclear until maintainers choose the compatibility and privacy policy. The durable fix should handle cross-HOME service roots before update or reinstall writes new config/service state, without reintroducing unsafe stale service-env reuse.
