## Summary

Keep open. Current main has adjacent update, doctor, and managed-env hardening, but package update still discovers and refreshes the managed gateway service from the updater process environment, so a root-invoked refresh can miss a dedicated-user service root instead of preserving, migrating, warning, or blocking.

## What This Changes

Ensure updates that change HOME preserve, migrate, or warn before abandoning the existing OpenClaw config and state.

## Best Possible Solution

Add a Linux package-update/service-refresh guard that finds the previously installed managed gateway root before rewriting service metadata, then safely preserves it or blocks with explicit migration guidance.

## Reproduction Assessment

Yes. The report gives concrete update steps, and current source inspection gives a high-confidence static path: service discovery is HOME-keyed and both pre- and post-update reads start from process.env.

## Solution Assessment

Not applicable because there is no patch on this issue. The requested preserve/migrate/warn direction is right, but the exact policy needs maintainer/security judgment because service roots may contain credentials and may also be intentionally changed.
