## Summary

The PR is a narrow, correct fix for a current CLI parsing bug: inline root option values currently truncate after an embedded `=`, while the branch switches to slicing after the first separator and adds focused regression coverage. It should stay open for the normal automerge/maintainer path rather than be closed, since current `main` does not yet contain the fix.

## What This Changes

This PR updates CLI root option parsing to preserve embedded equals signs, adds focused Vitest coverage for inline and space-separated values, and records the fix in the changelog.

## Best Possible Solution

Land the narrow parser fix with its regression test so inline root option values treat only the first `=` as the separator.

## Reproduction Assessment

Yes, by source inspection: current main uses `raw.split("=", 2)`, so `--token=abc=def` returns only `abc`; the PR body also supplies after-fix live output for the same path.

## Solution Assessment

Yes, slicing after the first `=` is the narrowest maintainable fix because inline parsing only needs the first separator and the tests preserve existing empty and space-separated behavior.
