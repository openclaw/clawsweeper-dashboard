## Summary

Keep open: current main still treats any non-empty cron `payload.model` as an explicit model ref, and the active implementation path remains an open, conflicting PR that needs maintainer agreement on the sentinel contract.

## What This Changes

Request to let cron `agentTurn` jobs store a special default model value that resolves to the runtime default model instead of a pinned model ref.

## Best Possible Solution

Approve one sentinel contract, update cron selection, fallback policy, docs, and tests consistently, then land or replace the related implementation PR after real runtime proof.

## Reproduction Assessment

Yes. Source inspection on current main shows `payload.model` is trimmed and every non-empty string flows into explicit model resolution, so `default` or `@default` would be treated as a literal model value.

## Solution Assessment

No for the original one-line guard alone. The maintainable solution needs a settled sentinel contract and matching handling in cron model selection, fallback policy, documentation, and regression tests.
