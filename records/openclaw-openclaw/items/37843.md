## Summary

Keep open: current main still treats cron `payload.model: "default"` as an explicit model reference, and the remaining work needs a maintainer decision on the public sentinel contract rather than cleanup closure.

## What This Changes

Request support for a cron `payload.model` sentinel so stored jobs can inherit the runtime default model instead of pinning a provider/model string.

## Best Possible Solution

Approve one public cron model-inheritance contract, normalize it across model selection and fallback policy, document precedence, and land it through the active or replacement implementation with regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: set an agent-turn cron `payload.model` to `default`, and the code treats it as a non-empty explicit model ref before fallback policy can behave like an omitted model.

## Solution Assessment

No, not as the original one-line guard alone. A maintainable fix needs a settled sentinel spelling plus shared normalization in cron model selection, fallback policy, docs, and tests.
