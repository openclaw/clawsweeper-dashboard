## Summary

Keep open: current main still treats a non-empty cron `payload.model` such as `default` as an explicit model reference, and the active implementation path is the open, conflicting https://github.com/openclaw/openclaw/pull/73727 with a different `@default` contract and prior review blockers.

## What This Changes

Allow cron `agentTurn` jobs to store `payload.model: "default"` so the model is resolved from the current default at runtime instead of pinned as a literal model ref.

## Best Possible Solution

Settle one public sentinel contract, normalize it consistently in cron model selection and fallback policy, document it, add regression coverage, then land or replace the active related PR.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: set cron `payload.model` to `default` and the code treats it as a non-empty explicit model ref before fallback policy runs.

## Solution Assessment

No, not as the original one-line guard alone. A maintainable fix needs a settled sentinel spelling plus aligned handling in model selection, fallback policy, docs, and tests.
