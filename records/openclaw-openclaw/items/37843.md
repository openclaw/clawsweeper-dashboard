## Summary

Keep open: current main still treats every non-empty cron `payload.model` as an explicit model reference, so the requested runtime-default sentinel is not implemented. The active implementation path is open, but it still needs maintainer agreement on the public sentinel contract and has review/CI blockers.

## What This Changes

Request runtime-default sentinel support for cron `agentTurn` `payload.model` so stored jobs can follow the current system default model.

## Best Possible Solution

Agree on one public sentinel contract, then implement it consistently across cron model selection, fallback policy, docs, tests, and any accepted subagent override paths.

## Reproduction Assessment

Yes. Source inspection on current main shows cron `payload.model` is trimmed and every non-empty value is resolved as a literal allowed model, so `default` or `@default` would not fall through to defaults.

## Solution Assessment

No for the original one-line guard by itself. The maintainable solution should normalize the accepted sentinel everywhere stored model overrides affect selection and fallback policy, then document and test the contract.
