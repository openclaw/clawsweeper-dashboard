## Summary

Keep open: current main still treats non-empty cron `payload.model` values as explicit model references, so the requested runtime-default sentinel is not implemented. The active implementation path is https://github.com/openclaw/openclaw/pull/73727, but maintainers still need to settle the public sentinel spelling and scope.

## What This Changes

Request support for a stored cron `payload.model` sentinel that resolves to the runtime default model instead of a literal provider/model reference.

## Best Possible Solution

Agree on one public sentinel contract, then implement it consistently across cron model selection, fallback policy, docs, tests, and any accepted subagent override paths.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: create or edit an isolated agent-turn cron with `--model default` or `--model @default`, and current main resolves the non-empty value as a literal model ref instead of falling through to defaults.

## Solution Assessment

No for the original one-line guard by itself. The maintainable solution should normalize the accepted sentinel wherever stored cron model overrides affect both model selection and fallback policy, then document and test the public contract.
