## Summary

Keep open because current main still treats a non-empty cron `payload.model` such as `default` as an explicit model ref, while the remaining runtime-sentinel work is in an open implementation PR that still needs maintainer judgment.

## What This Changes

Request support for a cron job model sentinel that resolves to the configured default model at runtime instead of storing a concrete model ref.

## Best Possible Solution

Land or revise https://github.com/openclaw/openclaw/pull/73727, or an equivalent successor, with the accepted sentinel contract applied consistently across cron model selection, fallback policy, docs, tests, and changelog.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: an isolated agent-turn cron with `payload.model: "default"` is treated as a non-empty override and passed to model resolution instead of falling through to defaults.

## Solution Assessment

No for the original one-line guard by itself. The maintainable solution should normalize the accepted sentinel everywhere stored cron model overrides affect selection and fallback policy, then document and test the public spelling.
