## Summary

Keep open because current main still treats every non-empty cron `payload.model` as a concrete model reference, and the runtime sentinel behavior remains unmerged. The active implementation candidate is https://github.com/openclaw/openclaw/pull/73727, but it still needs maintainer decision on sentinel spelling and scope.

## What This Changes

The issue requests allowing cron agent-turn `payload.model` to use `default` as a special value that resolves to the live default model at runtime.

## Best Possible Solution

Review and land or revise https://github.com/openclaw/openclaw/pull/73727, or an equivalent successor, with the accepted sentinel contract applied consistently across cron model selection, fallback policy, docs, tests, and changelog.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: create or edit an isolated agent-turn cron with `--model default`, and the stored non-empty value is resolved as a literal model ref instead of falling through to defaults.

## Solution Assessment

No for the original one-line guard by itself. The maintainable solution should normalize the accepted sentinel wherever stored cron model overrides affect selection and fallback policy, then document and test the public spelling.
