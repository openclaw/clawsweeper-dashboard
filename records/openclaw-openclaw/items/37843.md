## Summary

Keep open: current main still validates every non-empty cron `payload.model` as a concrete model reference, so the requested runtime-default sentinel is not implemented. There is an open implementation candidate at https://github.com/openclaw/openclaw/pull/73727, but it still needs maintainer judgment on sentinel spelling and scope.

## What This Changes

The issue requests treating cron agent-turn `payload.model: "default"` as runtime inheritance from the configured default model instead of as a stored literal model ref.

## Best Possible Solution

Land or revise the active sentinel implementation, or an equivalent successor, with one accepted public spelling applied consistently across cron model selection, fallback policy, docs, tests, and changelog.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence current-main path: create or edit an isolated agent-turn cron with `--model default`, and the stored non-empty value is resolved as a literal model ref instead of falling through to defaults.

## Solution Assessment

No for the original one-line guard by itself. The maintainable solution should normalize the accepted sentinel wherever stored cron model overrides affect model selection and fallback policy, then document and test the public contract.
