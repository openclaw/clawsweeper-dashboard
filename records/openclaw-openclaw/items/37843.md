## Summary

Keep open. Current main still treats cron `payload.model: "default"` as a concrete model override, and the remaining work is already represented by open PR #73727 rather than a shipped implementation.

## What This Changes

Allow cron agent-turn jobs to store a default-model sentinel in `payload.model` so runs resolve against the live agent/system default instead of a hardcoded model.

## Best Possible Solution

Land or revise the active implementation so stored cron model overrides are normalized through a shared helper, one sentinel spelling is reserved and documented, fallback policy treats the sentinel as unset, CLI/docs/tests/changelog are aligned, and explicit real model pins keep their current behavior.

## Reproduction Assessment

Yes. Current-main source inspection gives a high-confidence reproduction path: create or edit an agent-turn cron with `--model default`, then `resolveCronModelSelection` will treat the trimmed string as a model ref instead of falling through to the configured default.

## Solution Assessment

No, the original one-line guard is not the best complete fix by itself. The maintainable solution should share sentinel normalization across model selection and fallback policy, then document and test the chosen public sentinel; PR #73727 appears to be the active broader candidate for that direction.
