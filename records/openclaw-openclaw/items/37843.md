## Summary

Keep open: current main still treats a non-empty cron `payload.model` as an explicit model reference, so the requested runtime-default sentinel is not implemented. The active implementation path is the open PR at https://github.com/openclaw/openclaw/pull/73727, but it still needs maintainer judgment on the accepted sentinel spelling and scope.

## What This Changes

Request support for a cron job `payload.model` sentinel that resolves to the current agent/default model at run time instead of storing a fixed model ref.

## Best Possible Solution

Accept one public sentinel spelling and apply it consistently across cron model selection, cron fallback policy, docs, tests, and changelog, either by revising or replacing the active implementation PR.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence current-main path: create or edit an isolated agent-turn cron with `--model default`, and the stored non-empty value is resolved as a literal model ref instead of falling through to defaults.

## Solution Assessment

No for the original one-line guard by itself. The maintainable solution should normalize the accepted sentinel wherever stored cron model overrides affect model selection and fallback policy, then document and test the public contract.
