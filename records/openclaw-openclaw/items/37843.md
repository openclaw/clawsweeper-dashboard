## Summary

Keep open. Current main still resolves any trimmed non-empty cron `payload.model` as an explicit model reference, so `default` is not a runtime-default sentinel. The fallback-policy path, CLI add/edit behavior, docs, and tests are still aligned around concrete model refs, and the related fix PRs are closed unmerged.

## What This Changes

Support cron agentTurn `payload.model: "default"` and `openclaw cron add|edit --model default` as an explicit request to use the agent/system default model at run time.

## Best Possible Solution

Add a cron-scoped payload-model normalization helper that returns unset for exact trimmed `default`, share it between runtime model selection and fallback override policy, and document/test the CLI/API contract without changing generic model-ref parsing.
