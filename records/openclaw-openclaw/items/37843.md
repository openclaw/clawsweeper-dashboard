## Summary

Keep open: current main still treats every non-empty cron `payload.model` as an explicit model ref, the public cron docs still describe strict explicit-model behavior, and the active related implementation uses a different `@default` contract with open review blockers.

## What This Changes

Request to let agent-turn cron jobs store `payload.model: "default"` as an explicit runtime-inherit sentinel instead of resolving it as a literal model ref.

## Best Possible Solution

Define one public cron model-inheritance contract, normalize it consistently in model selection and fallback policy, document precedence, and either land or replace the active related PR after maintainer API-contract review.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: set an agent-turn cron `payload.model` to `default`, and the code treats it as a non-empty explicit model ref while fallback policy also treats it as an override.

## Solution Assessment

No, not as the original one-line guard alone. A maintainable fix needs a settled sentinel spelling plus shared normalization in cron model selection, fallback policy, docs, and regression tests.
