## Summary

Keep open: current main and v2026.5.19 still treat non-empty cron `payload.model` values as explicit model refs, while the related open work shows the remaining question is the public sentinel/defaults contract.

## What This Changes

Request to let cron agent-turn `payload.model` use a special `default` value that resolves the active default model at runtime.

## Best Possible Solution

Settle one documented sentinel/defaults contract, then centralize cron model-override normalization across model selection and fallback policy with docs and tests while preserving strict errors for real explicit models.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: store an isolated agent-turn cron with `payload.model: "default"`, and current main treats it as a non-empty explicit model override rather than inherited default resolution.

## Solution Assessment

No, not as the original one-line guard alone. A maintainable fix needs a settled public sentinel spelling and shared normalization in model selection, fallback policy, docs, and regression tests.
