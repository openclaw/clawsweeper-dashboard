## Summary

Keep open because current main still treats a cron `payload.model` value like `default` as a concrete model override, while the active implementation remains open in #73727 rather than shipped.

## What This Changes

Request support an explicit cron `payload.model` sentinel, originally `default`, that resolves to the live system default model at runtime instead of pinning a stored model string.

## Best Possible Solution

Land or revise #73727, or an equivalent shared helper, so stored cron model overrides can opt into live default resolution while fallback policy, docs, tests, changelog, and explicit real model pins stay aligned.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: create or edit an agent-turn cron with `--model default`, and the stored value is treated as a literal model ref rather than falling through to defaults.

## Solution Assessment

No for the original one-line guard by itself. The narrower maintainable path should normalize the sentinel anywhere stored model overrides affect selection or fallback policy, then document and test the chosen public spelling.
