## Summary

Keep open: current main still treats a non-empty cron `payload.model` as an explicit model reference, so the requested runtime-default sentinel is not implemented. The active path is the open implementation PR at https://github.com/openclaw/openclaw/pull/73727, but maintainers still need to choose the public sentinel spelling and scope.

## What This Changes

Allow cron `agentTurn` `payload.model` to use a runtime-default sentinel instead of pinning a provider/model string.

## Best Possible Solution

Agree on one public sentinel contract, then implement it consistently across cron model selection, cron fallback policy, docs, tests, and any accepted subagent override paths.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: create or edit an isolated agent-turn cron with `--model default`, and current main resolves the non-empty value as a literal model ref instead of falling through to defaults.

## Solution Assessment

No for the original one-line guard by itself. The maintainable solution should normalize the accepted sentinel wherever stored cron model overrides affect both model selection and fallback policy, then document and test the public contract.
