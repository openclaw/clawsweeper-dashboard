## Summary

Keep open: current main and v2026.5.20 still treat every non-empty cron `payload.model` as an explicit model ref, so the requested runtime-default sentinel is not implemented and still needs maintainer agreement on sentinel spelling and scope.

## What This Changes

The issue asks for cron agent-turn `payload.model` to support a special runtime-default value instead of always storing a fixed model ref.

## Best Possible Solution

Settle one documented model-inheritance sentinel, centralize normalization across cron model selection and fallback policy, preserve strict errors for real explicit model refs, and land it with focused docs and regression coverage.

## Reproduction Assessment

Yes for the current behavior: source inspection shows `payload.model: "default"` is treated as a non-empty explicit model ref. The requested inherited-default semantics are a new public contract rather than already-defined expected behavior.

## Solution Assessment

No for the original one-line guard alone. A maintainable fix needs a settled sentinel spelling plus shared normalization across cron model selection, fallback policy, docs, and regression tests.
