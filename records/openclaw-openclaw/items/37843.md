## Summary

Keep open because current main still treats every non-empty cron `payload.model` as a concrete model reference, and the runtime sentinel behavior remains unmerged; https://github.com/openclaw/openclaw/pull/73727 is the active implementation candidate but still needs maintainer decision on spelling and scope.

## What This Changes

The issue requests a cron `payload.model` value such as `default` that resolves to the live agent/default model at runtime instead of freezing a concrete model ref.

## Best Possible Solution

Review and land or revise https://github.com/openclaw/openclaw/pull/73727, or an equivalent implementation, with model selection, fallback policy, docs, tests, and changelog aligned to the accepted sentinel contract.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: create or edit an agent-turn cron with `--model default`, and the stored non-empty value is resolved as a literal model ref instead of falling through to defaults.

## Solution Assessment

No for the original one-line guard by itself. The maintainable solution should normalize the accepted sentinel wherever stored cron model overrides affect selection and fallback policy, then document and test the public spelling.
