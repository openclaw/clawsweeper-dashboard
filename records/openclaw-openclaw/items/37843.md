## Summary

Keep open because current main still treats every non-empty cron `payload.model` as an explicit model reference, while the runtime-default sentinel remains unresolved and is currently represented by the open implementation candidate at https://github.com/openclaw/openclaw/pull/73727.

## What This Changes

Allow cron jobs to store a special `payload.model` value, proposed as `default`, that follows the system default model at runtime.

## Best Possible Solution

Land or revise the active sentinel implementation so stored cron model overrides can intentionally follow runtime defaults, with one accepted spelling handled consistently in model selection, fallback policy, tests, docs, and changelog.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: create or edit an isolated agent-turn cron with `--model default`, and the stored non-empty value is resolved as a literal model reference instead of falling through to defaults.

## Solution Assessment

No for the original one-line guard alone. The maintainable fix should normalize the accepted sentinel anywhere stored cron model overrides affect selection and fallback policy, then document and test the public spelling.
