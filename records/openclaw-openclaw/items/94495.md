## What This Changes

The PR changes the Ollama native stream timeout resolver so missing explicit `requestTimeoutMs`/`timeoutMs` now becomes a fixed 300000 ms fallback before guarded fetch.

## Best Possible Solution

Preserve current timeout inheritance when no explicit provider timeout exists, then fix the configured remote-Ollama stream boundary with focused regression coverage and redacted live remote proof.

## Reproduction Assessment

No high-confidence live remote Ollama reproduction was run here. The PR-introduced timeout regression is source-reproducible because the new fallback is passed as an explicit guarded-fetch timeout and prevents the current global timeout fallback from being used.

## Solution Assessment

No. This is a wrong-layer fallback for the reported case because the reporter already configured provider and run timeouts; the safer solution is to preserve inherited timeout behavior and fix the configured-timeout stream boundary.
