## Summary

Keep open. The PR targets a real Slack A2A session-routing bug that current main and v2026.6.6 still do not fully fix, but this branch is conflicting, lacks real Slack proof, and still has a source-backed parser regression.

## What This Changes

The PR changes session-key parsing and restart-sentinel delivery-context merging so A2A Slack thread sessions preserve thread routing instead of falling back toward webchat or a base session.

## Best Possible Solution

Rebase the useful parser fix onto current main, preserve the newer restart-sentinel route-recovery behavior, and only skip `agent:<id>:` when an explicit group/channel route follows the encoded provider.

## Reproduction Assessment

Yes, by source inspection. On the PR branch, `agent:main:webchat` with channel chat type sets `startIndex` to 2, treats `webchat` as a surface, derives an empty id, and returns `null`; current main also still source-reproduces the original Slack parser bug.

## Solution Assessment

No, not as submitted. The parser direction is useful, but the implementation needs to distinguish `agent:<id>:<mainKey>` from `agent:<id>:<provider>:<group|channel>:...`, and the restart-sentinel portion must be rebased over current main's newer route-recovery code.
