## Summary

Keep open: this PR targets a real source-visible Slack/A2A session-routing bug, but the current patch still introduces a parser regression and lacks required real Slack/A2A behavior proof.

## What This Changes

The PR changes group-session key parsing, restart-sentinel delivery-context merging, and regression tests to preserve Slack A2A thread routing.

## Best Possible Solution

A rebased fix should use guarded shared session-key parsing, keep or deliberately update the agreed restart routing policy, add focused regression tests, and include redacted Slack/A2A runtime proof.

## Reproduction Assessment

Yes, at source level. Current main can read From=agent:builder:slack:channel:C123:thread:T with Provider=webchat, miss the encoded Slack provider, and persist webchat through group session metadata.

## Solution Assessment

No, not as-is. The parser should only skip agent:<id> when the remaining body has provider plus explicit group/channel shape, preferably by reusing or mirroring parseRawSessionConversationRef(), and restart fallback precedence needs maintainer review.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9
