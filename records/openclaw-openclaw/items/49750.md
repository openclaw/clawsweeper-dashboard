## Summary

Keep open: current main still has the source-visible Slack/A2A session-routing bug, but this PR is not merge-ready because the branch introduces a parser regression, lacks real Slack/A2A proof, and is conflicting with current main.

## What This Changes

The branch changes group-session key parsing, restart-sentinel delivery-context fallback, and regression tests to preserve Slack A2A thread routing.

## Best Possible Solution

Rebase or replace the branch with guarded shared session-key parsing, deliberate restart fallback precedence, focused regression tests, and redacted live Slack/A2A proof.

## Reproduction Assessment

Yes, at source level. Current main can read From=agent:builder:slack:channel:C123:thread:T with Provider=webchat, miss the encoded Slack provider, and persist webchat through group session metadata.

## Solution Assessment

No, not as-is. The parser should only skip agent:<id> when the remaining body has provider plus explicit group/channel shape, preferably by reusing or mirroring parseRawSessionConversationRef(), and the restart fallback precedence should be reviewed after rebase.

## Review Findings
