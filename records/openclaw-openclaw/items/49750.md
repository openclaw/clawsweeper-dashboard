## Summary

Keep open. Current main still does not implement the central agent-prefixed Slack thread parser fix, and the paired same-author issue remains open. This PR is a relevant implementation candidate, but the patch still has a blocking parser edge case and is missing the required user-facing changelog entry.

## What This Changes

The PR changes session group-key parsing to skip agent prefixes, adjusts restart-sentinel delivery-context fallback precedence, and adds regression tests for agent-prefixed Slack thread routing plus incomplete delivery contexts.

## Best Possible Solution

Revise the PR so resolveGroupSessionKey() reuses or mirrors parseRawSessionConversationRef(): agent-prefixed keys should be treated as provider-encoded conversation keys only when the post-agent body includes an explicit group/channel kind. Keep the restart-sentinel intent of preserving explicit/current/parsed target hints ahead of base-session fallback, add the missing changelog entry, then run the focused session and gateway tests.

## Reproduction Assessment

Yes. Static reproduction is high confidence: current main with From=agent:builder:slack:channel:C123:thread:T, Provider=webchat, and ChatType=channel reads head=agent and can normalize the group key as webchat. The PR's remaining defect reproduces with From=agent:main:webchat, where the proposed startIndex makes webchat the head but leaves the id empty.

## Solution Assessment

No, not as-is. The PR targets the right surfaces, but the parser change must require an explicit group/channel segment or delegate to parseRawSessionConversationRef(); the patch also needs the required user-facing changelog entry.
