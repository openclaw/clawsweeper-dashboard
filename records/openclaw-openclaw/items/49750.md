## What This Changes

The branch changes group-session key parsing, restart-sentinel delivery-context merging, and related Vitest coverage to preserve Slack thread routing for agent-prefixed A2A sessions.

## Best Possible Solution

Land a rebased narrow fix that uses guarded shared session-key parsing, keeps explicit/current routing facts ahead of base fallback, preserves current thread-target policy unless intentionally changed, and includes focused regression tests plus redacted Slack runtime proof.

## Reproduction Assessment

Yes, at source level. Current main can read agent:builder:slack:channel:C123:thread:T with head=agent and fall back to Provider=webchat, while the PR diff introduces a separate agent:main:webchat parser edge case.

## Solution Assessment

No, not as-is. The parser should only skip an agent prefix when the remaining key has provider plus explicit group/channel shape, or should reuse parseRawSessionConversationRef(), and the branch still needs rebase plus real Slack proof.
