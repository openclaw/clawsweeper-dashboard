## Summary

Keep open: this PR targets a valid source-visible Slack/A2A session-routing bug, but the current patch can drop channel-scoped routing for some agent-prefixed keys, lacks real Slack/A2A after-fix proof, and is dirty against current main.

## What This Changes

The PR changes group-session key parsing, restart-sentinel delivery-context merge order, and regression tests to preserve Slack A2A thread routing.

## Best Possible Solution

Rebase or replace the branch with guarded shared session-key parsing, deliberate restart fallback precedence, focused regression tests, and redacted Slack/A2A runtime proof.

## Reproduction Assessment

Yes, at source level. Current main can read From=agent:builder:slack:channel:C123:thread:T with Provider=webchat, miss the encoded Slack provider, and persist webchat through group session metadata; I did not run a live Slack repro in this read-only review.

## Solution Assessment

No, not as-is. The parser should only skip agent:<id> when the remaining body has provider plus explicit group/channel shape, preferably by reusing or mirroring parseRawSessionConversationRef(), and restart fallback precedence needs maintainer review.

## Review Findings
