## What This Changes

The PR changes group-session parsing for agent-prefixed Slack thread keys, adjusts restart-sentinel delivery-context fallback precedence, and adds Vitest coverage.

## Best Possible Solution

Land a rebased fix that uses the shared guarded parser or an equivalent explicit-kind check, keeps explicit/current routes ahead of base fallback, and preserves thread-target rejection.

## Reproduction Assessment

Yes. Source inspection shows current main can mis-normalize From=agent:builder:slack:channel:C123:thread:T with Provider=webchat, and the PR diff has a separate source-visible edge case for From=agent:main:webchat.

## Solution Assessment

No, not as-is. The parser should skip the agent prefix only when the remaining key has an explicit group/channel kind or reuse parseRawSessionConversationRef(), and the rebase should preserve current sessions_send thread-target rejection.
