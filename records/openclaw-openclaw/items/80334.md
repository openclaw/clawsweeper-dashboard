## Summary

Keep this PR open, but the current patch should not merge as written: it fixes a source-provable Ollama failure path by moving all `think` values into `options.think`, while upstream Ollama still defines thinking control as top-level `ChatRequest.Think` and does not define `think` as an option.

## What This Changes

The PR routes configured and runtime Ollama `think` values into `options.think` instead of top-level `/api/chat` request fields, then updates stream tests to expect that shape.

## Best Possible Solution

Preserve top-level Ollama `think` for reasoning-capable native models, and harden OpenClaw so non-reasoning models cannot receive a truthy top-level `think` unless live upstream proof supports a different contract.

## Reproduction Assessment

Yes, partially from source: current `main` can emit a truthy top-level native Ollama `think`, and upstream Ollama rejects truthy `req.Think` for non-thinking models. The exact default-thinking route still needs live confirmation because current OpenClaw also has supported-level validation.

## Solution Assessment

No. Moving all native Ollama thinking control into `options.think` is not the narrowest maintainable fix because upstream Ollama reads thinking from top-level `ChatRequest.Think`; a capability-aware guard is safer.
