## Summary

Keep this PR open, but do not merge it as written: it moves Ollama `think` into `options.think`, while upstream Ollama defines and uses top-level `ChatRequest.Think` and current OpenClaw tests intentionally preserve that request shape. The patch also lacks live Ollama proof for either non-thinking or reasoning-capable models.

## What This Changes

The PR moves Ollama native `think` values from the top-level `/api/chat` payload into `options.think` and updates Ollama stream tests to expect that shape.

## Best Possible Solution

Preserve top-level `think` for reasoning-capable Ollama models, and harden OpenClaw so non-reasoning models cannot receive a truthy top-level `think` unless live upstream proof supports a different contract.

## Reproduction Assessment

Yes, partially from source: current main can emit top-level truthy `think`, and upstream Ollama rejects truthy `req.Think` for non-thinking models. The exact default-thinking route still needs live confirmation because current OpenClaw also has supported-level validation.

## Solution Assessment

No. Moving all native Ollama thinking control into `options.think` is not the narrowest maintainable fix because upstream Ollama reads `think` from top-level `ChatRequest.Think`; a capability-aware guard is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9
