## Summary

Close as implemented: current main wires Telegram `sendChatAction` into the shared typing lifecycle, tool events start or refresh the typing loop before tool progress is emitted, and the behavior is present in the shipped `v2026.5.3` tree.

## What This Changes

Request that Telegram keep its typing indicator active from inbound message receipt through tool execution until final reply delivery.

## Best Possible Solution

Keep the shared typing lifecycle and Telegram `sendChatAction` wiring; reopen only with a fresh live Telegram trace showing a platform-specific gap outside this source path.

## Reproduction Assessment

No current-main failing reproduction is available from the source path. Current main shows Telegram tool events call the shared typing signaler, and tests cover both tool-start typing and long-run keepalive ticks.

## Solution Assessment

Yes. The existing shared typing lifecycle is the maintainable fix because Telegram only owns the `sendChatAction` adapter while model/tool lifecycle policy stays channel-generic.
