## Summary

Keep open: current main still lacks both the reasoning-prefaced silent-reply classifier and the runner fallback to raw last-assistant text, so the PR addresses a real leak/retry bug. The branch is not mergeable as-is because it references an undefined `AssistantMessage` type and the external PR still needs inspectable after-fix real behavior proof.

## What This Changes

The PR adds shared detection for reasoning-prefaced `NO_REPLY` payloads and updates embedded-runner incomplete-turn handling so intentional silent replies do not trigger retries or a user-visible failure message.

## Best Possible Solution

Repair this branch narrowly by importing or aliasing the assistant-message type, add inspectable redacted after-fix proof, then land the two-layer fix if focused checks pass; leave broader trailing-token policy to the existing related PRs.

## Reproduction Assessment

Yes. Source inspection on current main shows the outbound classifier misses `think\n...NO_REPLY`, and the runner still only checks `assistantTexts` before retry/warning classification when the raw last assistant content contains the silent payload.

## Solution Assessment

No, not as written. The two-layer approach is the narrow maintainable direction for this leak class, but the undefined `AssistantMessage` reference blocks typecheck and the external proof gate is still unmet.
