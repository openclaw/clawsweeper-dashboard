## Summary

Keep open. Current main and the latest release still preserve browser screenshots for the model while also exposing the screenshot path to the trusted tool-result media queue, and the Pinchy comment corroborates real shared-channel impact; the related open PR moves in the opposite direction by adding another explicit media hint rather than a suppression control.

## What This Changes

Add a way to keep browser screenshot results model-visible while suppressing automatic chat/channel media delivery.

## Best Possible Solution

Introduce an explicit tool-result media visibility gate before queued media reaches channel replies, keeping browser screenshots available to the model while letting maintainers choose the default and control surface.

## Reproduction Assessment

Yes, by source inspection. A browser screenshot result writes details.media.mediaUrl, browser is trusted for local tool-result media, and the subscribe pipeline queues that media into visible replies without a privacy gate.

## Solution Assessment

Unclear on exact product shape. The right code boundary is before queuePendingToolMedia or reply consumption, but maintainers need to choose whether the control is a screenshot parameter, tool-level policy, channel policy, or global default.
