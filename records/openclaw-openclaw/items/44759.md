## Summary

Keep open. Current main still preserves browser screenshots for model vision while also treating the screenshot path as trusted tool-result media that can be queued for chat delivery; the Pinchy comment confirms this affects real shared-channel automation workflows.

## What This Changes

The issue asks for a way to keep browser screenshots visible to the model without automatically delivering the image to chat channels.

## Best Possible Solution

Add a tool-result media visibility gate that can suppress channel delivery while preserving model-visible screenshot content, with browser screenshot coverage and queue/flush regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: browser screenshot returns imageResultFromFile, imageResult records details.media.mediaUrl, browser is trusted for local media, and handleToolExecutionEnd queues the path for outbound reply media.

## Solution Assessment

Yes for the direction. The maintainable fix is to gate visibility before queuePendingToolMedia rather than removing model-visible screenshot content; the exact defaults and UX need maintainer/security decision.
