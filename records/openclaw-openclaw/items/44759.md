## Summary

Keep open. Current main still has a source-backed sensitive-media exposure path: browser screenshots remain model-visible and are also converted into trusted pending tool media for channel delivery, and the corroborating Pinchy comment shows this affects real shared-channel automation workflows.

## What This Changes

The issue requests a way to keep browser screenshots visible to the model without automatically broadcasting them as chat-channel media.

## Best Possible Solution

Add an explicit tool-result media visibility policy that can suppress channel delivery while preserving model-visible content, with a narrow browser screenshot private path and regression tests covering the queue and reply flush behavior.

## Reproduction Assessment

Yes. A high-confidence source-backed reproduction is to invoke browser screenshot in a channel session: current main returns model image content plus details.media.mediaUrl, trusts browser media, queues the path, and attaches or flushes it to outbound replies.

## Solution Assessment

Yes, the reporter's per-call private screenshot option is the narrowest concrete direction, but it should be implemented through the shared tool-result media queue so visibility is decided before queuePendingToolMedia runs.
