## Summary

Keep open. Current main still keeps browser screenshots available to the model while also exposing the screenshot path as trusted tool-result media that can be queued for visible channel delivery; the Pinchy comment corroborates real shared-channel impact.

## What This Changes

The issue asks for a way to keep browser screenshots visible to the model without automatically delivering them to chat channels.

## Best Possible Solution

Introduce an explicit tool-result media visibility gate before queuePendingToolMedia so browser screenshots can stay in model context without channel delivery, with regression coverage for browser screenshots and trusted media queue behavior.

## Reproduction Assessment

Yes. Source inspection against current main shows browser screenshot returns imageResultFromFile, imageResult writes details.media.mediaUrl, browser is trusted for local media, and emitToolResultOutput queues the filtered URL; tests also assert trusted media queueing when verbose output is off.

## Solution Assessment

Unclear on the exact product shape. A suppression gate before queueing is the right code boundary, but the default behavior and whether this is a tool parameter, config policy, or channel policy needs maintainer/security direction.
