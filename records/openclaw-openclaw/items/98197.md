## Summary

Keep open: current main still has the 64 KiB sanitizer tail cap that can reject a valid split SSE event before its boundary arrives, and the issue is already paired with an open closing PR that should be reviewed and landed or rejected before cleanup.

## What This Changes

The issue reports that the OpenAI ChatGPT Responses SSE sanitizer rejects legitimate gpt-5.5 events larger than 64 KiB, breaking the subscription-backed model path.

## Best Possible Solution

Review and land the linked fix PR or an equivalent narrow change that decouples the tight non-OK error-body cap from a larger bounded inter-event sanitizer cap.

## Reproduction Assessment

Yes, source-reproducible. A sanitized `text/event-stream` response whose first chunk contains more than 64 KiB of one event before the `\n\n` boundary reaches the throw path in `src/agents/provider-transport-fetch.ts`.

## Solution Assessment

Yes, likely. Separating the non-OK body cap from the SSE inter-event sanitizer cap is the narrowest maintainable fix shape; the exact threshold and live proof should be finalized in the linked PR review.
