## Summary

Keep open: current main still has the 64 KiB successful-SSE sanitizer cap, and the open linked PR is the active candidate fix that should land or be rejected before cleanup.

## What This Changes

The issue reports a main-only regression where OpenAI ChatGPT Responses SSE sanitization rejects legitimate large gpt-5.5 events because the inter-event buffer is capped at 64 KiB.

## Best Possible Solution

Review and either land or reject https://github.com/openclaw/openclaw/pull/98198, preserving a tight non-OK error-body cap while allowing legitimate large SSE events on the OpenAI ChatGPT Responses path.

## Reproduction Assessment

Yes, source-reproducible: current main throws when a sanitized SSE response buffers more than 64 KiB before an event boundary, matching the reported error path. I did not run a live provider call in this read-only review.

## Solution Assessment

Yes, likely: separating the non-OK error-body cap from the successful SSE inter-event sanitizer cap is the narrow owner-path fix. The open linked PR carries that candidate implementation, so this issue should remain open until that PR lands or is rejected.
