## Summary

Keep open: current `main` still has the 64 KiB OpenAI SSE sanitizer inter-event cap and the issue is paired with an open closing PR that should be reviewed and landed or rejected before cleanup.

## What This Changes

The issue reports that `gpt-5.5` via `openai-chatgpt-responses` fails because the OpenAI SSE sanitizer rejects legitimate single events larger than 64 KiB.

## Best Possible Solution

Review and either land or reject https://github.com/openclaw/openclaw/pull/98198, preserving a tight non-OK error-body cap while allowing legitimate large SSE events on the OpenAI ChatGPT Responses path.

## Reproduction Assessment

Yes, source-reproducible: current `main` throws when a sanitized SSE response buffers more than 64 KiB before an event boundary, which matches the reported failure path. I did not run a live provider call in this read-only review.

## Solution Assessment

Yes, likely: separating the non-OK body cap from the inter-event sanitizer cap is the narrowest owner-path fix. The open linked PR carries that implementation candidate and should be reviewed on its own merge-risk tradeoff.
