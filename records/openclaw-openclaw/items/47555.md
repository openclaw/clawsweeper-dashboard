## Summary

Keep open. Current main still lets an explicit root `--profile` coexist with an inherited `OPENCLAW_LAUNCHD_LABEL`, and the macOS launchd restart path resolves that inherited label before the profile-derived label, so the reported wrong-gateway restart remains possible. PR #66000 is an open implementation candidate for the same stale-label root cause, so this should stay open until that or an equivalent fix lands.

## What This Changes

The issue requests that `openclaw --profile <name> gateway restart` target the requested profile instead of an inherited agent-session gateway label.

## Best Possible Solution

Explicit root profile selection should override inherited gateway service identity for lifecycle and status resolution, while preserving custom launchd labels in managed contexts that did not request a different profile.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists with `OPENCLAW_PROFILE=rescue` and inherited `OPENCLAW_LAUNCHD_LABEL=ai.openclaw.gateway`: current label resolution returns the inherited label and `restartLaunchAgent` targets that service.

## Solution Assessment

Yes. The narrowest maintainable direction is to clear or re-derive stale launchd service identity when an explicit root `--profile` is accepted, then cover restart/status/plist resolution with focused tests.
