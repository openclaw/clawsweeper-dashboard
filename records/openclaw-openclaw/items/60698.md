## What This Changes

The PR adds automatic per-agent Bedrock `requestMetadata` injection and also adds a Feishu outbound media MIME-sniffing fallback.

## Best Possible Solution

Rework this as a narrow Bedrock change that keeps current wrapper precedence, safely handles empty params, validates or omits AWS-invalid metadata, records the agent-name privacy choice, adds focused regression coverage, and splits the Feishu change out.

## Reproduction Assessment

Yes for the PR blockers: source inspection of the PR head shows the zero-config Bedrock path can dereference undefined `extraParams`, raw config strings flow into AWS metadata, and wrapper precedence changes. The user-facing AWS cost-attribution path still lacks live Bedrock or CloudTrail proof.

## Solution Assessment

No. Automatic Bedrock metadata is a useful direction, but this branch should not merge until it is rebased, narrowed, validated against AWS constraints, covered by focused tests, and backed by real Bedrock proof.

Full review comments:

- **[P1] Keep the zero-config Bedrock path from dereferencing extraParams:** `src/agents/pi-embedded-runner/extra-params.ts:241-242`
  - body: When no params are configured, `hasBedrockMetadata` lets execution continue with `extraParams` still undefined, but this function later reads `extraParams.cachedContent`. A no-config Bedrock agent will throw before `requestMetadata` is added, so default `extraParams` to an object or optional-chain every read before enabling the wrapper.
  - confidence: 0.96
- **[P1] Validate Bedrock metadata before injection:** `src/agents/pi-embedded-runner/extra-params.ts:282-284`
  - body: `agentId` and `agentName` come from config fields that OpenClaw accepts as arbitrary strings, but Bedrock constrains requestMetadata key/value length and characters. A valid OpenClaw name with an emoji or apostrophe can make every Bedrock stream fail with a validation error, so normalize, validate, or omit invalid entries before sending them.
  - confidence: 0.94
- **[P2] Preserve provider wrapper precedence:** `src/agents/pi-embedded-runner/extra-params.ts:510`
  - body: Moving `applyPrePluginStreamWrappers` before `wrapProviderStreamFn` makes extra params inner to provider wrappers. Current main applies extra params outside provider defaults so explicit config can win; keep that order or add Bedrock metadata without changing global wrapper precedence.
  - confidence: 0.9
