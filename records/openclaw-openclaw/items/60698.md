## Summary

This external PR is useful but not ready to merge: current main still lacks automatic per-agent Bedrock `requestMetadata`, while the branch has concrete blockers around raw metadata validation, wrapper precedence, unrelated Feishu scope, conflicts, and missing live Bedrock proof.

## What This Changes

The PR adds automatic Bedrock `requestMetadata` injection for agent id/name in the embedded runner and also adds a Feishu outbound media MIME-sniffing fallback with tests.

## Best Possible Solution

Land or replace this with a narrow Bedrock-owned implementation that validates or omits invalid metadata, preserves existing wrapper precedence, adds focused regression coverage, and leaves arbitrary configured metadata to https://github.com/openclaw/openclaw/issues/57360.

## Reproduction Assessment

Yes for the PR blockers: source inspection shows raw config strings flow into Bedrock metadata and the wrapper order changes from current main. The user-facing AWS behavior still lacks live Bedrock or CloudTrail proof.

## Solution Assessment

No. Automatic per-agent Bedrock metadata is a useful direction, but this branch should not merge until metadata is validated, wrapper precedence is preserved, unrelated Feishu changes are split, and real Bedrock proof is added.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Sanitize Bedrock metadata before injection:** `src/agents/pi-embedded-runner/extra-params.ts:282-284`
  - body: `agentId` and `agentName` come from config strings that OpenClaw does not constrain, while Bedrock `requestMetadata` values have character and length restrictions. A valid OpenClaw agent name such as `Support 🤖` or `O'Reilly` can make Bedrock reject every stream for that agent, so normalize, validate, or omit invalid metadata before forwarding it.
  - confidence: 0.92
- **[P2] Keep extra params outside provider wrappers:** `src/agents/pi-embedded-runner/extra-params.ts:510`
  - body: Moving the extra-param wrapper before provider wrapping reverses the current precedence contract. Provider wrappers can now pass default options into the inner extra-param wrapper, and because that wrapper spreads `...streamParams, ...options`, those defaults can overwrite explicit config values; keep the old order or add Bedrock metadata without changing global wrapper precedence.
  - confidence: 0.86
