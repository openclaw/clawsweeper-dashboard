## Summary

Keep open. The Bedrock cost-attribution request is still valid and current main does not inject `requestMetadata`, but this PR is not safe to merge as written because it forwards unconstrained agent strings to an AWS field with a strict value pattern and it reorders the existing stream-wrapper precedence from current main.

## What This Changes

The PR adds automatic Amazon Bedrock `requestMetadata` injection from per-agent `agentId`/`agentName` in the embedded Pi runner and also includes unrelated Feishu media MIME-sniffing changes/tests.

## Best Possible Solution

Land a narrow current-main implementation that injects per-agent Bedrock metadata only after validating or normalizing values to the AWS contract, preserves existing provider-wrapper precedence, includes focused unit coverage, and leaves broader configurable metadata for #57360.

## Reproduction Assessment

Yes for the patch defects: configure a Bedrock agent with an allowed OpenClaw `name` that violates AWS `requestMetadata` value pattern, such as an emoji or apostrophe, and the PR will forward it to `ConverseStream`. The wrapper-order regression can be covered with a focused unit test where provider wrapper defaults collide with explicit extra params.

## Solution Assessment

No. The requested direction is useful, but this branch is not the best implementation because it bypasses AWS metadata sanitization, reorders current wrapper composition, and carries unrelated Feishu changes; a narrow rebased patch in the runner/provider seam is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Sanitize Bedrock metadata before injection:** `src/agents/pi-embedded-runner/extra-params.ts:282-284`
  - body: `agentId` and `agentName` are plain config strings, but Bedrock `requestMetadata` values only accept a limited character pattern and length. Names such as `Support 🤖` or `O'Reilly` pass OpenClaw config validation and would make every Bedrock stream for that agent fail with a validation error, so normalize, skip, or reject invalid metadata before adding it to the request.
  - confidence: 0.91
- **[P2] Keep extra-param wrappers outside provider wrappers:** `src/agents/pi-embedded-runner/extra-params.ts:510`
  - body: Moving `applyPrePluginStreamWrappers` before `wrapProviderStreamFn` reverses the current composition contract. Provider wrappers can pass defaults as `options`, and the inner extra-param wrapper spreads `...options` after `streamParams`, so provider defaults can override explicit config such as `openaiWsWarmup: false`; keep the current wrapper order and add the Bedrock metadata path without changing precedence.
  - confidence: 0.82
