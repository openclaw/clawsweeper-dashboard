## Summary

This PR should stay open: current main still lacks the Bedrock cost-attribution feature, but the branch has correctness blockers around raw Bedrock metadata values and provider wrapper precedence, and it still needs real Bedrock proof before merge.

## What This Changes

The branch injects Bedrock `requestMetadata` with agent id/name in the embedded runner and also adds Feishu outbound media MIME-sniffing fallback code and tests.

## Best Possible Solution

Land or replace this with a narrow Bedrock implementation that validates or omits invalid metadata values, preserves existing wrapper precedence, includes focused regression coverage, and leaves arbitrary configured metadata to https://github.com/openclaw/openclaw/issues/57360.

## Reproduction Assessment

Yes for the PR blockers: source inspection reproduces the raw metadata path from unconstrained agent config and the wrapper-order regression against current main. The user-facing AWS behavior still lacks live Bedrock or CloudTrail proof.

## Solution Assessment

No. Automatic per-agent Bedrock metadata is a useful direction, but this branch should not merge until metadata is validated, wrapper precedence is preserved, unrelated Feishu changes are split, and real Bedrock proof is added.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Sanitize Bedrock metadata before injection:** `src/agents/pi-embedded-runner/extra-params.ts:282-284`
  - body: `agentId` and `agentName` come from config strings that OpenClaw does not constrain, but Bedrock request metadata values have length and pattern restrictions. A valid agent name such as `Support 🤖` or `O'Reilly` can make Bedrock reject every stream for that agent, so normalize, validate, or omit invalid metadata before forwarding it.
  - confidence: 0.92
- **[P2] Preserve extra-param precedence over provider defaults:** `src/agents/pi-embedded-runner/extra-params.ts:510`
  - body: Moving the extra-param wrapper inside provider wrapping reverses the current contract that configured extra params sit outside provider defaults. Because the extra-param wrapper applies `...streamParams, ...options`, provider-supplied option defaults can now win over explicit config values; keep the old order or add Bedrock metadata without changing global wrapper precedence.
  - confidence: 0.86
