## Summary

Keep open: the feature direction is useful, but this PR should not merge as-is because the branch injects raw Bedrock metadata, reverses current stream-wrapper precedence, lacks real Bedrock/CloudTrail proof, and mixes unrelated Feishu media behavior into the same branch.

## What This Changes

The PR adds automatic per-agent Bedrock requestMetadata injection and also changes Feishu outbound media routing to sniff MIME type from buffers.

## Best Possible Solution

Replace this with a narrow Bedrock path that validates or omits invalid metadata, preserves current wrapper precedence, adds focused regression coverage, and leaves arbitrary configurable metadata to https://github.com/openclaw/openclaw/issues/57360.

## Reproduction Assessment

Yes for the PR blockers: source inspection of the head commit shows raw config strings flow into Bedrock requestMetadata and extra-param wrapping moved before provider wrapping. The user-facing AWS cost-attribution path still lacks live Bedrock or CloudTrail proof.

## Solution Assessment

No. Automatic per-agent Bedrock metadata is a useful direction, but this branch should not merge until metadata validation, privacy policy, wrapper precedence, tests, real proof, and Feishu split/rebase are handled.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Validate Bedrock metadata before injection:** `src/agents/pi-embedded-runner/extra-params.ts:282-284`
  - body: `agentId` and `agentName` come from config fields that OpenClaw accepts as arbitrary strings, but Bedrock requestMetadata has length and character restrictions. A valid OpenClaw name containing an emoji or apostrophe can make every Bedrock stream for that agent fail with validation errors, so validate, normalize, or omit invalid entries before adding them to the payload.
  - confidence: 0.93
- **[P2] Preserve provider wrapper precedence:** `src/agents/pi-embedded-runner/extra-params.ts:510`
  - body: Moving `applyPrePluginStreamWrappers` before `wrapProviderStreamFn` makes the extra-param wrapper inner to provider wrappers. Provider defaults can then arrive as `options` and overwrite explicit config values because the wrapper spreads `...streamParams, ...options`; keep the current order or add Bedrock metadata without changing global precedence.
  - confidence: 0.88

## Security Review

Status: needs_attention
