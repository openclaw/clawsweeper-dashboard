## Summary

Keep open: current main still lacks the requested Bedrock per-agent metadata path, but this PR is not merge-ready because it forwards unconstrained agent strings into AWS-constrained metadata, reverses the current provider/extra-param wrapper precedence, and carries unrelated Feishu behavior changes.

## What This Changes

The PR adds per-agent Bedrock `requestMetadata` injection in the embedded Pi runner and also changes Feishu outbound media MIME sniffing/tests.

## Best Possible Solution

Land a narrow current-main implementation that injects sanitized per-agent Bedrock metadata, preserves existing wrapper precedence, adds focused coverage and a changelog entry, and leaves configurable arbitrary metadata to #57360.

## Reproduction Assessment

Yes. Current main can be verified by source search as lacking any `requestMetadata` path, and the PR defects are reproducible by using a Bedrock agent name with unsupported metadata characters plus a provider wrapper that supplies default options after configured extra params.

## Solution Assessment

No. The requested direction is useful, but this branch is not the best implementation until it validates or normalizes metadata values, preserves current wrapper composition, removes the unrelated Feishu change, and documents the user-facing feature.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Sanitize Bedrock metadata before injection:** `src/agents/pi-embedded-runner/extra-params.ts:282-284`
  - body: `agentId` and `agentName` are plain config strings, but Bedrock `requestMetadata` values only allow a limited character pattern and length. Names such as `Support 🤖` or `O'Reilly` pass OpenClaw config and would make every Bedrock stream for that agent fail with a validation error, so normalize, skip, or reject invalid metadata before forwarding it.
  - confidence: 0.92
- **[P2] Preserve provider wrapper precedence:** `src/agents/pi-embedded-runner/extra-params.ts:510`
  - body: Moving `applyPrePluginStreamWrappers` before `wrapProviderStreamFn` reverses the current composition contract. Provider wrappers can pass defaults as `options`, and the inner extra-param wrapper spreads `...options` after `streamParams`, so provider defaults can override explicit config such as `openaiWsWarmup: false`.
  - confidence: 0.86
- **[P3] Add the required changelog entry:** `src/agents/pi-embedded-runner/extra-params.ts:282-284`
  - body: This is a user-facing Bedrock feature, but the PR only changes runner and Feishu files. Repo policy requires user-facing `feat` changes to add a single-line `CHANGELOG.md` entry with real human attribution when known.
  - confidence: 0.9
