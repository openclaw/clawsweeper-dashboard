## Summary

Keep open: current `main` still fails the `/active-memory` command gate when `config.agents` is omitted or empty, and the only linked fix PR was closed unmerged. The remaining maintainer choice is whether empty agent config should mean the default `main` agent or remain an explicit fail-closed allowlist.

## What This Changes

The issue requests consistent Active Memory slash-command behavior for single-agent setups where `plugins.entries.active-memory.config.agents` is omitted or empty.

## Best Possible Solution

Maintainers should decide the Active Memory targeting contract, then either land a replacement for the closed fallback PR with docs and regression coverage, or keep fail-closed targeting and improve setup diagnostics/guidance.

## Reproduction Assessment

Yes for the slash-command symptom by source inspection: omitted `agents` normalizes to `[]`, `isEnabledForAgent` rejects `main`, and the command handler returns `Active Memory: off for this session.` before status/on/off. I did not run a live channel or Vitest reproduction in this read-only review.

## Solution Assessment

Unclear. The default-agent fallback is a narrow code fix, but it changes the current explicit-agent targeting contract in docs and manifest, so maintainer product approval should come before repair work.
