## Summary

Keep open: current main still rejects `/active-memory` session commands for the default `main` agent when `config.agents` is omitted or empty, and the only directly related fix PR is closed unmerged.

## What This Changes

The issue asks for Active Memory slash commands to honor the default `main` agent when `config.agents` is omitted or empty in a single-agent setup.

## Best Possible Solution

Align the shared Active Memory enablement gate, docs, and regression tests so missing or empty `config.agents` selects the default `main` agent while explicit non-empty allowlists still fail closed.

## Reproduction Assessment

Yes at source level. Current main normalizes omitted `config.agents` to `[]`, resolves a `main` session, and then returns `Active Memory: off for this session.` before the command action runs.

## Solution Assessment

Yes, the narrow maintainable fix is to make `isEnabledForAgent` use the same effective-agent fallback already used by command session resolution, with docs and tests updated to preserve explicit allowlist behavior.
