## Summary

Keep open: current main still has the empty-`agents` slash-command failure, and the only direct fix PR is closed and unmerged. The remaining question is whether maintainers want missing or empty `config.agents` to mean the default `main` agent despite the current explicit-targeting docs.

## What This Changes

The issue requests that Active Memory treat missing or empty `config.agents` as the default `main` agent so `/active-memory status`, `/active-memory on`, and `/active-memory off` work in single-agent setups.

## Best Possible Solution

Decide the empty-agent contract; if default-main is accepted, land a narrow code, test, and docs fix that preserves explicit non-empty allowlists and proves the real `/active-memory` command path.

## Reproduction Assessment

Yes, by source inspection: omitted `config.agents` normalizes to `[]`, then the `/active-memory` command gate rejects `main` and returns `Active Memory: off for this session.` I did not run a live channel or Vitest repro in this read-only review.

## Solution Assessment

Unclear until maintainers decide the contract. The closed fallback PR is mechanically narrow, but the best landing shape also needs docs/tests for the new empty-agent default and proof through the real command path.
