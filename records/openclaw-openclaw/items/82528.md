## Summary

Keep open: current main still has the active-memory empty-agent allowlist mismatch, and an open linked PR already targets the same helper, so this issue should stay open until that PR lands or is rejected.

## What This Changes

Report that active-memory session slash commands cannot be toggled when the plugin config omits `config.agents` in a single-agent `main` setup.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/82529; the intended end state is a shared default-agent fallback in `isEnabledForAgent` with regression coverage for omitted `config.agents`.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: omitted `config.agents` normalizes to `[]`, the command resolves `main`, and `isEnabledForAgent` checks `[].includes("main")` before returning the reported off response.

## Solution Assessment

Yes. Applying the existing `DEFAULT_AGENT_ID` fallback inside `isEnabledForAgent` is the narrowest maintainable fix because both slash commands and the prompt-build gate share that helper while explicit non-empty allowlists remain unchanged.
