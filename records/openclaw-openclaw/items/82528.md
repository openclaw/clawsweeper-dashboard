## Summary

Keep open: current main and the latest release still treat omitted Active Memory `config.agents` as an empty target list, so the reported session command problem is not implemented on main. The open candidate fix at https://github.com/openclaw/openclaw/pull/92253 owns the remaining code path, but it still needs maintainer default-policy review because it changes omitted-agent targeting semantics.

## What This Changes

The issue requests consistent Active Memory session status/on/off behavior when the enabled plugin config omits `config.agents` in a single-agent default setup.

## Best Possible Solution

Review and either land or reject https://github.com/openclaw/openclaw/pull/92253; if maintainers reject the broader omitted-agents default, choose a narrower default-main, setup, docs, or doctor repair for single-agent users.

## Reproduction Assessment

Yes, at source level. Current main normalizes omitted Active Memory `config.agents` to `[]`, then the command gate requires `config.agents.includes(agentId)`, so a default `main` session reaches the reported `off for this session` response.

## Solution Assessment

Unclear as a final product decision. A central normalization fix is clean if maintainers want omission to mean configured agents, but a narrower default-main, setup, docs, or doctor path would have less upgrade blast radius.
