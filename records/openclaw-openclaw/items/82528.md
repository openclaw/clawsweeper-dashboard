## Summary

Keep open: current `main` and the latest release still normalize omitted Active Memory `config.agents` to an empty list and gate `/active-memory` through that list, so the reported command behavior remains source-reproducible. The broader omitted-agent default is now owned by an open related PR, but that PR still needs maintainer compatibility approval before this issue can close.

## What This Changes

The issue asks for Active Memory session commands to work when the plugin is enabled but `plugins.entries.active-memory.config.agents` is omitted in a single-agent setup.

## Best Possible Solution

Review and either land or reject the open Active Memory targeting PR; if maintainers reject the broader default change, update setup/docs or add a narrower config repair path so users do not enable Active Memory without an explicit target list.

## Reproduction Assessment

Yes. Source inspection on current `main` shows omitted plugin `agents` normalizes to `[]`, `isEnabledForAgent` checks that list directly, and `/active-memory` returns `Active Memory: off for this session.` when the agent is not listed.

## Solution Assessment

Unclear as a final product decision. A helper-level fallback would fix the reported command symptom, but the open PR proposes a broader omitted-field default that changes operator targeting semantics and needs maintainer approval.
