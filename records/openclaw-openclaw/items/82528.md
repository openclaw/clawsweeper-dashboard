## Summary

Keep open for maintainer judgment: the reported `off for this session` command path is clear in current source when `agents` is omitted, but current docs and manifest describe `config.agents` as an explicit target allowlist, and the per-turn hook also fails closed through the same enablement gate.

## What This Changes

The issue requests that active-memory treat omitted or empty `config.agents` as the default `main` agent so `/active-memory status|on|off` work in a single-agent setup.

## Best Possible Solution

Maintainers should either preserve the explicit-targeting contract and improve setup guidance, or deliberately change Active Memory so an empty target list means the configured default agent and update docs/tests accordingly.

## Reproduction Assessment

Partly. Source inspection reproduces the command response for `config.enabled: true`, empty `config.agents`, and agent `main`, but current source does not reproduce the claimed mismatch where per-turn recall keeps running under that same config.

## Solution Assessment

Unclear. Applying a `[DEFAULT_AGENT_ID]` fallback is narrow mechanically, but it changes the documented fail-closed agent-targeting contract, so it needs maintainer product approval first.
