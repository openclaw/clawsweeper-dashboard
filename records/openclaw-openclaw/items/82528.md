## Summary

Keep open: current main and the latest stable tag still fail the `/active-memory` command gate when `config.agents` is omitted, but the correct empty-agent semantics need maintainer judgment because current docs and manifest describe `config.agents` as an explicit allowlist.

## What This Changes

The issue asks for Active Memory slash commands to treat missing or empty `config.agents` in a single-agent setup as the default `main` agent instead of always reporting off.

## Best Possible Solution

Maintainers should decide the Active Memory targeting contract, then either preserve explicit `config.agents` with clearer setup/errors or land a default-`main` fallback with docs, command tests, and real command-path proof.

## Reproduction Assessment

Yes for the slash-command symptom by source inspection: omitted `config.agents` becomes `[]`, `isEnabledForAgent` rejects `main`, and the handler returns the reported off message. The per-turn mismatch was not reproduced because current `before_prompt_build` uses the same gate.

## Solution Assessment

Unclear. The fallback proposed in the closed PR is mechanically narrow, but it changes the documented explicit-targeting contract, so the best fix depends on maintainer product judgment.
