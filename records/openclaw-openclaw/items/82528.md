## Summary

Keep open: current main still has the `/active-memory` slash-command fail-closed path when `config.agents` is omitted, and https://github.com/openclaw/openclaw/pull/82529 is an open closing PR for the same behavior. This issue should remain the tracker until that PR lands or maintainers choose a different Active Memory targeting contract.

## What This Changes

Make `/active-memory status`, `on`, and `off` work for single-agent Active Memory configs that omit or leave `config.agents` empty instead of always reporting off.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/82529 with docs alignment and real behavior proof, or explicitly preserve the current allowlist contract and clarify onboarding/docs instead.

## Reproduction Assessment

Yes for the source-level slash-command failure: omitted `config.agents` becomes `[]`, then `isEnabledForAgent` rejects `main` and the handler returns the reported off response before processing the command action. I did not run a live Telegram command in this read-only review.

## Solution Assessment

Yes if maintainers accept empty `agents` as the single-agent default: the linked PR touches the shared gate and preserves explicit non-empty allowlists. If that contract is rejected, the safer solution is onboarding/docs clarification rather than changing runtime semantics.
