## Summary

Keep open: current main and v2026.6.8 still fail closed when Active Memory `config.agents` is omitted, and the open candidate PR has not merged because the omitted-agent default needs maintainer approval.

## What This Changes

The issue reports that `/active-memory status`, `/active-memory on`, and `/active-memory off` all report session-off when Active Memory config omits `config.agents` in a single-agent setup.

## Best Possible Solution

Review and either land or reject the open Active Memory targeting PR; if maintainers reject the broader default change, use a narrower default-main, setup, docs, or doctor repair so single-agent users do not enable Active Memory without an explicit target.

## Reproduction Assessment

Yes. Source inspection on current `main` and v2026.6.8 shows omitted plugin `agents` normalizes to `[]`, `isEnabledForAgent` checks that list directly, and `/active-memory` returns `Active Memory: off for this session.` when the resolved agent is not listed.

## Solution Assessment

Unclear as a final product decision. A helper-level default-main fallback would fix the single-agent command symptom, but the open PR proposes a broader omitted-field default that changes operator targeting semantics and needs maintainer approval.
