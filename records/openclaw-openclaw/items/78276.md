- `impact:security`: The reported race crosses the bootstrap-token authentication boundary and can revive a pairing credential.

## Summary

Keep open: current main and latest release v2026.6.10 still persist bootstrap-token state through process-local locking plus whole-file JSON writes, and the linked closing PR remains open and unmerged. The issue has a high-confidence source reproduction path and should stay open until the linked PR or a replacement security fix lands.

## What This Changes

Fix setup-code races that can let concurrent bootstrap-token state writers revive consumed tokens or erase newly issued setup codes.

## Best Possible Solution

Land the linked PR or a narrow replacement that serializes every mutating bootstrap-token state read-modify-write across processes and proves stale snapshots cannot revive consumed tokens.

## Reproduction Assessment

Yes. Source inspection shows independent processes can load stale devices/bootstrap.json snapshots and later overwrite bootstrap-token binding or revocation because current main uses only process-local serialization.

## Solution Assessment

Yes for the issue direction: the narrow maintainable solution is cross-process serialization or an equivalent transactional owner for every mutating bootstrap-token state path. The open linked PR is the current candidate, but the issue should stay open until a reviewed fix lands.
