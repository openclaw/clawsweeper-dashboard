## Summary

Keep open: current main and v2026.6.10 still mutate bootstrap-token state through a process-local lock plus whole-file JSON writes, and the linked closing PR is still open, conflicting, and unmerged.

## What This Changes

Prevent concurrent setup-code issuance and bootstrap verification from reviving consumed bootstrap tokens in `devices/bootstrap.json`.

## Best Possible Solution

Land the linked PR or a narrow replacement that serializes every mutating bootstrap-token read-modify-write across processes and proves stale snapshots cannot revive consumed setup codes.

## Reproduction Assessment

Yes. Source inspection shows independent processes can load stale `devices/bootstrap.json` snapshots and later overwrite bootstrap-token binding or revocation because current main uses only process-local serialization.

## Solution Assessment

Yes for the issue direction. The narrow maintainable fix is cross-process serialization or an equivalent transactional owner for every mutating bootstrap-token state path, but the issue should stay open until a reviewed fix lands.
