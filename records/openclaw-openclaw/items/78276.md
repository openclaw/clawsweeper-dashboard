- `impact:security`: The reported race crosses the bootstrap-token authentication boundary and can revive a pairing credential.

## Summary

Keep open: current main and latest release v2026.6.8 still persist bootstrap-token state through whole-file JSON writes guarded by a process-local lock, and the linked closing PR remains open and unmerged.

## What This Changes

The issue reports a security bug where concurrent setup-code issuance and bootstrap verification can write stale `devices/bootstrap.json` snapshots that revive consumed bootstrap tokens.

## Best Possible Solution

Land the linked PR or a replacement security fix that serializes every mutating bootstrap-token state read-modify-write across processes and proves the stale-snapshot race cannot revive consumed tokens.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path where independent Gateway/setup-code processes can load stale `devices/bootstrap.json` snapshots and later overwrite bootstrap-token binding or revocation under only process-local serialization.

## Solution Assessment

Yes. The narrow maintainable solution is cross-process serialization or an equivalent transactional owner for every mutating bootstrap-token state path; the open linked PR is only an implementation candidate until it lands cleanly.
