## Summary

Current OpenClaw plugin hooks already provide the generic outbound content and normalized-payload rewrite seams needed for this optional, deployment-specific footer; adding a global gateway template with service-specific variables would expand core configuration at the wrong ownership boundary.

## What This Changes

Requests a gateway configuration template that appends identity and diagnostic links to group-memory messages.

## Best Possible Solution

Implement and publish a ClawHub code plugin that appends the deployment-specific footer through `reply_payload_sending` or `message_sending`, keeping endpoint URLs, agent identity lookup, board IDs, and template policy with the integration that owns them.

## Reproduction Assessment

Not applicable; this requests a new automatic footer capability rather than reporting broken existing behavior.

## Solution Assessment

No—the proposed global `agents.defaults.messageTemplate.footer` setting is broader than necessary because existing plugin hooks can own outbound footer injection and the named endpoint variables belong to the external integration.
