## Summary

Current OpenClaw already provides a public prompt-build plugin hook that can append per-run model context with agent, session, provider, trigger, and channel metadata. Because the named group-memory and token-info services are external and team-specific, a new global gateway template setting would duplicate an existing extension seam and put integration policy in core.

## What This Changes

Requests a gateway configuration template that automatically appends identity and operational links to group-memory messages.

## Best Possible Solution

Keep core unchanged and publish an external plugin that uses `before_prompt_build` to append the footer, with the service base URL, board identity, and template policy stored in plugin-owned configuration.

Not applicable; this issue requests new templating behavior rather than reporting a failure of documented or tested OpenClaw behavior.

## Solution Assessment

No. A core `agents.defaults.messageTemplate.footer` setting would duplicate the existing public prompt-mutation hook; a plugin is the narrower maintainable solution.
