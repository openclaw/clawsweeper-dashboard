## Summary

Keep open. The zh-CN UI proof and localization direction are useful, but the PR still has a source-reproducible generated i18n metadata defect: non-target locale files contain English `chat.slashCommands` values that are not recorded as fallbacks, so future syncs can preserve them as if translated.

## What This Changes

The PR routes chat slash-command, composer, Talk, and command-palette text through Control UI i18n, adds zh-CN translations, updates generated locale outputs, and adjusts focused UI tests.

## Best Possible Solution

Keep the localization approach, then regenerate or repair the generated i18n outputs so every non-target English `chat.slashCommands` fallback is tracked truthfully before validation.

## Reproduction Assessment

Yes. The remaining issue is source-reproducible by comparing the PR-head non-target locale bundles, their `.i18n` metadata, and the current fallback-refresh logic in the i18n generator.

## Solution Assessment

No. Routing the UI through i18n is the right direction, but merging generated locale outputs with inaccurate fallback metadata would make future translation refreshes unreliable.
