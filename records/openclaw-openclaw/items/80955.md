## What This Changes

This PR routes Control UI chat slash-command, command-palette, search, and Talk status strings through i18n and adds zh-CN plus generated locale updates.

## Best Possible Solution

Keep the localization direction, but land only after the PR preserves the guarded slash-command import boundary, records truthful generated locale state, and covers the visible Talk controls in zh-CN.

## Reproduction Assessment

Yes for the review blockers: PR-head source and current CI show the extra i18n import fails the slash-command browser import guard, and source inspection shows the non-target fallback metadata and Talk-options gaps.

## Solution Assessment

No; routing the UI through i18n is the right direction, but this patch should keep localization out of the slash-command registry module, regenerate or mark fallbacks truthfully, and translate the visible Talk options form before merge.
