This PR should stay open because the latest branch still has two concrete i18n correctness blockers: generated non-target locale bundles contain English fallback text while reporting no fallbacks, and the visible Talk options form remains English despite the PR's stated zh-CN coverage. The PR routes Control UI chat slash-command and composer strings through i18n, adds zh-CN translations plus generated locale outputs, and localizes slash-command descriptions and argument choices at render time.

## Best Possible Solution
