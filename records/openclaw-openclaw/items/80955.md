## Summary

Keep open: the PR is useful, but the latest head still has two blocking i18n correctness issues. Non-target locale bundles record English fallback text as translated output, and the visible Talk options form still renders English labels under zh-CN.

## What This Changes

The PR routes Control UI chat slash-command and composer text through i18n, adds zh-CN translations and generated locale metadata, and localizes slash-command descriptions and argument choices at render time.

## Best Possible Solution

Keep the localization approach, but repair the generated i18n outputs so fallback metadata is truthful and localize the visible Talk options panel before merge.

## Reproduction Assessment

Yes for the review blockers by source inspection: latest head has English additions in non-target locale bundles with empty fallback metadata, and the Talk options form still renders English visible labels. The supplied screenshots also verify the improved slash menu and arg picker in a real browser UI.

## Solution Assessment

No; routing the chat UI through i18n is the right direction, but this patch should not merge until the generated locale metadata is accurate and the stated Talk options surface is actually localized.
