## Summary

Keep open: the zh-CN UI proof is credible, but the latest PR head still has inaccurate generated i18n metadata for non-target locale English slash-command fallbacks, which can make future syncs preserve those English strings as translated.

## What This Changes

The branch routes Control UI chat slash-command, composer, Talk, and command-palette text through i18n and adds zh-CN translations plus regenerated locale output.

## Best Possible Solution

Keep the i18n rendering approach, but regenerate or repair generated fallback metadata so non-target English slash-command fallbacks are tracked before merge.

## Reproduction Assessment

Yes. At the latest PR head, the German locale contains English chat.slashCommands values while de.meta.json records only chat.talk.* fallback keys; the current generator path shows why that metadata matters.

## Solution Assessment

No. The runtime localization approach is sound, but the generated locale metadata should be corrected before merging so future translation syncs can refresh those fallback strings correctly.
