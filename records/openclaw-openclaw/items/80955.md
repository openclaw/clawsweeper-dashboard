## Summary

Keep open: the latest PR addresses the earlier generated-output drift, but it now treats English fallback text in non-target locale bundles as fully translated, and it leaves the visible Talk options form in English despite the stated zh-CN scope.

## What This Changes

The PR moves Control UI chat slash-command, command-palette, Talk, and search text through i18n keys and adds zh-CN translations plus generated locale bundle updates.

## Best Possible Solution

Keep the zh-CN localization, but regenerate or repair the Control UI i18n outputs so non-target locale fallback state is truthful and the visible Talk options UI matches the stated localized behavior.

## Reproduction Assessment

Yes for the review blockers by source inspection: the PR head contains English additions in non-target locale bundles while metadata reports zero fallbacks, and the Talk options renderer still has visible English labels under zh-CN.

## Solution Assessment

No; routing the chat UI through i18n is the right direction, but the current patch should not merge until generated fallback metadata is truthful and the stated Talk options surface is actually localized.
