## Summary

Keep open: the PR fixes the right update-cli path for true bundle payloads, but the new early bundle-record branch can bypass native package validation for dual-format ClawHub installs that the installer and docs treat as native.

## What This Changes

The PR changes update-time plugin payload smoke checks and missing-payload collection to validate bundle install records through the shared bundle manifest detector/loader, with regression tests.

## Best Possible Solution

Keep the shared-loader validation for true bundle payloads, but preserve native-first classification so valid native package payloads remain subject to package validation even when bundle markers or ClawHub bundle metadata are present.

## Reproduction Assessment

## Solution Assessment

No. The PR is in the right updater boundary and uses the right shared bundle loader for true bundles, but the best fix must preserve native-first package validation before treating bundle metadata as sufficient.
