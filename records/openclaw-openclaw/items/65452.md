## Summary

Keep open. The current-main vulnerability is real, but this PR is not merge-ready because it is dirty against current main, conflicts with the package-owned memory SDK bridge, and changes SQLite identifier behavior for valid quoted dotted table names.

## What This Changes

The PR adds SQL identifier quoting helpers to memory schema management and applies them to cache table, FTS table, and ALTER TABLE interpolation in both the package file and the source-side mirror file.

## Best Possible Solution

Land a revised security fix on current main that changes only the package-owned memory schema implementation, keeps the source-side file as a bridge, defines and enforces the accepted SQLite identifier forms, and adds focused regression tests for malicious identifiers, schema-qualified names, and quoted dotted-name behavior.

## Reproduction Assessment

Yes. A high-confidence source-level and in-memory SQLite reproduction exists: feeding a crafted table identifier into the current helper's DDL path drops an existing `files` table through `db.exec`.

## Solution Assessment

No, not as-is. Identifier quoting is the right direction, but the current patch should be updated for the package/bridge split and for an explicit SQLite table identifier contract before merge.

## Review Findings

Overall correctness: patch is incorrect

  - confidence: 0.86
