## Summary

Keep open: the security issue is still present on current main, but this PR lacks auditable real behavior proof and has blocking correctness/security review findings around stale source-side code and SQLite identifier compatibility.

## What This Changes

The PR adds SQLite identifier-quoting helpers to memory schema DDL in the package implementation and a now-stale source-side memory schema file.

## Best Possible Solution

Land a revised package-owned SQL identifier hardening that defines preserve-or-reject behavior for SQLite identifiers, adds focused regression tests, and keeps the current SDK bridge layout intact.

## Reproduction Assessment

Yes. Source inspection on current main shows ensureMemoryIndexSchema interpolates embeddingCacheTable and ftsTable into db.exec DDL, and that helper is exported through the memory-core host SDK facade.

## Solution Assessment

No. Identifier quoting is the right mitigation direction, but this PR should be rebased onto the package-only layout and should parse/preserve or explicitly reject quoted dotted SQLite identifiers with tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

- **[P2] Do not restore the deleted source-side schema file:** `src/memory-host-sdk/host/memory-schema.ts:1-10`

Reintroducing a full implementation here will conflict or drift on rebase, so apply the fix in the package-owned file and current export surface instead.
  - confidence: 0.93
