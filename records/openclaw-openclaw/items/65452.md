## Summary

Keep open: current main and the latest release still interpolate dynamic memory schema table identifiers, but this PR is not merge-ready because it restores a deleted source-side copy, mishandles dotted quoted identifiers, lacks contributor real behavior proof, and needs security/owner agreement on the SQLite identifier contract.

## What This Changes

The branch adds SQLite identifier-quoting helpers to memory schema DDL in the package memory host SDK and an obsolete source-side copy.

## Best Possible Solution

Land a narrow package-owned hardening that validates or quotes table identifiers under a documented SQLite contract, adds malicious and qualified-name regression coverage, and leaves the deleted source bridge removed.

## Reproduction Assessment

Yes. A read-only in-memory node:sqlite command against current-main-style DDL showed a crafted table identifier executing a second DROP TABLE statement, and current source still has the unquoted interpolation path.

## Solution Assessment

No. Identifier hardening is the right direction, but this branch restores obsolete source-side code and its quoteTable helper rewrites valid quoted dotted identifiers instead of preserving or explicitly rejecting them.
