## Summary

Keep open: current main and the latest release still have unquoted memory schema DDL, so the security problem is real, but this PR is not correct as-is because it restores a deleted source-side file, rewrites already-quoted dotted identifiers, and lacks auditable real behavior proof.

## What This Changes

The PR adds SQLite identifier-quoting helpers to memory schema DDL in the package memory host SDK and an old source-side copy.

## Best Possible Solution

Land a revised package-owned SQL identifier hardening that preserves or rejects supported SQLite table identifier forms, adds focused regression tests, and leaves the deleted source bridge deleted.

## Reproduction Assessment

Yes. Current main has unquoted table-name DDL, and a read-only in-memory node:sqlite check confirmed a crafted identifier can terminate the statement and drop the files table.

## Solution Assessment

No. Identifier hardening is the right direction, but this branch should be rebased onto the package-owned layout and should define preserve-or-reject behavior for dotted quoted identifiers with tests.
