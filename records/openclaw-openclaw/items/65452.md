The PR adds SQLite identifier-quoting helpers to memory schema DDL in `packages/memory-host-sdk` and the old `src/memory-host-sdk` copy.

## Best Possible Solution

Land package-owned memory schema hardening that quotes or validates identifiers under an explicit SQLite contract, adds malicious and compatibility regression tests, and leaves the stale source bridge deleted.

## Reproduction Assessment

Yes. A read-only in-memory `node:sqlite` command against current-main-style unquoted DDL showed a crafted table identifier executing a second `DROP TABLE`, and current main still has the same interpolation path.

## Solution Assessment

No. Identifier hardening is the right direction, but this PR restores a deleted source-side copy and its dot-splitting helper breaks valid quoted dotted SQLite identifiers instead of preserving or explicitly rejecting them.
