## Summary

Keep open. The reported SQL identifier injection is still present on current main through the exported memory host schema helper, but the proposed patch is not merge-ready because it reintroduces source-side implementation drift and changes valid dotted quoted table identifiers.

## What This Changes

The PR adds SQL identifier quoting helpers and applies them to memory schema DDL in the package implementation and the older source-side memory schema file.

## Best Possible Solution

Land a revised security fix in the package-owned memory schema implementation, keep the source-side file as a bridge, define the accepted SQLite identifier forms, quote or validate identifiers consistently, add focused regression tests for malicious identifiers and schema-qualified names, and include the required user-facing changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to call the exported ensureMemoryIndexSchema with cacheEnabled true and a crafted embeddingCacheTable containing SQL statement separators; current main passes that string directly into db.exec DDL.

## Solution Assessment

No, not as-is. Identifier quoting is the right direction, but this PR should be revised for the current package/bridge layout and for a documented quoted-table contract with tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

  - confidence: 0.91

That regresses current pass-through behavior; either parse and preserve quoted forms or reject them explicitly with tests.
  - confidence: 0.86
