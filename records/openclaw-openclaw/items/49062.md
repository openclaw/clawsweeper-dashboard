## What This Changes

The issue asks OpenClaw to define and document how Telegram `channel_post` updates should interact with sender-scoped per-chat `groups[chatId].allowFrom` authorization.

## Best Possible Solution

Choose one Telegram `channel_post` authorization policy, then update the Telegram plugin code, focused auth tests, public docs, and live allowed/denied Telegram proof to match it.

## Reproduction Assessment

Source-reproducible only: current code and docs show no explicit `channel_post` plus per-chat `allowFrom` policy, but I did not run a live Telegram channel reproduction.

No implementation path is best until maintainers choose the security semantics; the maintainable path is a narrow policy decision followed by code, tests, docs, and live Telegram proof.
