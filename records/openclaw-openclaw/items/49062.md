## Summary

Keep open: current main and v2026.6.8 still do not define how Telegram `channel_post` commands should handle sender-scoped per-chat `allowFrom`, and the linked open PR explicitly leaves that policy question unresolved.

## What This Changes

Define and document how Telegram `channel_post` native commands interact with sender-scoped per-chat `groups[chatId].allowFrom` when no human sender identity is available.

## Best Possible Solution

Make the Telegram plugin policy explicit, add focused tests for explicit channel allow plus configured per-chat `allowFrom` on `channel_post`, and document the chosen rule in the public Telegram docs.

## Reproduction Assessment

Partly: source inspection shows the current command auth path depends on `msg.from` and docs do not define a channel_post allowFrom rule. I did not establish a live Telegram reproduction in this read-only review.
