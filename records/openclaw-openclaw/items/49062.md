## What This Changes

Define and document Telegram channel_post authorization behavior when sender-scoped groups[chatId].allowFrom is configured.

## Best Possible Solution

Choose one maintainer-approved channel_post authorization rule, implement it in the Telegram plugin, document it at https://docs.openclaw.ai/channels/telegram, and cover allowed and denied behavior with focused tests plus live Telegram proof.

Yes, source-level. Current main still routes Telegram native command auth through ctx.message and msg.from while channel_post surfaces use channelPost/sender_chat, and docs do not define the sender-scoped allowFrom exception.

No current patch or shipped behavior is the best solution yet. The maintainable path is a maintainer-approved auth policy with code, docs, regression tests, and live Telegram verification.
