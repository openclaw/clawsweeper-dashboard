## What This Changes

Define and document how Telegram channel_post native commands should behave when sender-scoped groups[chatId].allowFrom is configured and no human sender ID is available.

## Best Possible Solution

Maintainers should choose the channel_post authorization policy first, then implement and document native/plugin command, callback, and test behavior around that policy.
