## What This Changes

The issue asks for the host `write` tool to deterministically follow in-workspace symlink directory components when writing files.

## Best Possible Solution

Canonicalize in-root symlink parents for host workspace-only write/edit before pinned write operations, preserve outside-root symlink rejection, and add focused regression coverage for the OpenClaw tool path plus the fs-safe dependency-facing behavior.

## Reproduction Assessment

Yes, by source inspection: a workspace-only host write through an in-root symlink parent reaches `root.write` with the lexical relative path, and the pinned fs-safe write path can reject that symlink parent. I did not run a live intermittent session reproduction in this read-only review.

## Solution Assessment

Yes, a narrow canonicalization fix is the maintainable direction, provided it keeps the existing workspace escape protections and covers both write and edit host paths with regression tests.
