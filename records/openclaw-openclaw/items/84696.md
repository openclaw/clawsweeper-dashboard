## What This Changes

The issue requests deterministic write-tool support when a target file path includes an in-workspace symlink directory component.

## Best Possible Solution

Canonicalize in-root symlink parents before the fs-safe-backed host write/edit operation reaches pinned writes, while keeping symlink escapes outside the workspace rejected and adding focused regression coverage.

## Solution Assessment

Yes. A narrow canonicalization fix on the fs-safe-backed host write/edit path is the maintainable solution; a broad symlink policy change or exec-style fallback would be riskier than needed.
