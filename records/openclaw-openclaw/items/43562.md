## Summary

Keep open. Current main still puts the hardcoded personal-assistant identity before SOUL.md Project Context, and the supported config surface still only exposes full prompt replacement; #48442 remains the active implementation path.

## What This Changes

Request to make the agent system prompt's first identity line configurable or neutral when SOUL.md defines the agent persona.

## Best Possible Solution

Review and land #48442 or a narrower successor that adds a documented default and per-agent identity-line override while preserving full `systemPromptOverride` behavior.

## Reproduction Assessment

Yes. Source inspection gives a deterministic path: build or inspect `buildAgentSystemPrompt` with a SOUL.md context file and the personal-assistant line appears before Project Context.

## Solution Assessment

Unclear until maintainers choose the final API shape. A focused identity-line or preamble setting looks like the narrowest maintainable direction; broad section disabling needs product review.
