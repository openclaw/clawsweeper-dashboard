## Summary

Keep open: current main and the latest release still put the hardcoded personal-assistant identity line first, SOUL.md remains lower-priority Project Context, and the available config contract does not expose a narrow identity-line setting. The related implementation attempts are unmerged, and the remaining decision is the maintainer-approved prompt/config boundary rather than cleanup closure.

## What This Changes

The issue requests a configurable or neutral first system-prompt identity line so SOUL.md-defined agent identities are not overridden by the default personal-assistant preamble.

## Best Possible Solution

Approve a narrow identity-line customization contract, then implement schema, types, docs/config metadata, resolver wiring, and runtime coverage across all prompt assembly paths while treating broad section disabling as a separate decision.

## Reproduction Assessment

Yes for the source condition: current main still places the personal-assistant line before Project Context, and SOUL.md is injected later with an explicit higher-priority caveat. I did not run a live model session to prove exact response wording.

## Solution Assessment

Unclear. A narrow configurable identity line is plausible, but the repo still needs a maintainer-approved public config shape and safety boundary before accepting either the old `preamble` approach or the broader `identityLine` plus `disabledSections` proposal.
