## Summary

Keep open. Current main still emits the hardcoded personal-assistant identity before SOUL.md Project Context, the supported config surface still only exposes full prompt replacement, and the latest release remains affected. The open related PR #48442 is the active implementation candidate, so this should stay with maintainer review rather than a separate repair branch.

## What This Changes

The issue requests a configurable agent identity preamble so custom SOUL.md personas are not overridden by the hardcoded personal-assistant line.

## Best Possible Solution

Review and land #48442 or a narrow successor that adds a documented default and per-agent identity-line/preamble configuration path, preserves the existing default output and `systemPromptOverride` behavior, and covers SOUL.md ordering plus prompt-mode behavior in tests and docs/schema.

## Reproduction Assessment

Yes. The prompt construction path is deterministic: inspecting `buildAgentSystemPrompt` with a SOUL.md context file shows the personal-assistant line emitted before the SOUL.md Project Context guidance.

## Solution Assessment

Unclear until maintainers choose the final config shape. A focused identity-line or preamble setting is more maintainable than requiring full prompt replacement, while the active PR's broader section-disabling surface needs review.
