## Label Justifications

- `P2`: This is a normal-priority hook observability feature with limited blast radius but real contract, privacy, and runtime-delivery design work.
- `impact:session-state`: The request concerns context pruning, active session memory visibility, and downstream indexing after session context changes.

## Summary

Keep open: current main and v2026.5.22 still prune by returning rewritten messages without a pruning lifecycle hook or metadata payload, and https://github.com/openclaw/openclaw/pull/12581 remains the open direct implementation path rather than a merged fix.

## What This Changes

Expose a context/session pruning hook with metadata so handlers can react after tool-output pruning.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/12581 with a maintainer-approved session-pruning lifecycle event that emits bounded sanitized summary metadata after successful pruning, documents visibility, and covers pruning plus hook-delivery tests.

## Reproduction Assessment

Yes, at source level. Current main rewrites and returns pruned messages without hook emission or pruning metadata, and the hook catalogs/docs still omit a pruning lifecycle event.

## Solution Assessment

Unclear until maintainers choose the event shape and visibility. The narrow maintainable path is a safe summary-metadata hook in the existing context-pruning path, with docs stating whether it is internal-only or plugin-visible.
