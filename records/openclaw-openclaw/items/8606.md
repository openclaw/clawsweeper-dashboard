## Summary

Keep open. Current main still has pruning behavior but no lifecycle hook or pruning metadata surface, and the open implementation candidate https://github.com/openclaw/openclaw/pull/12581 still explicitly targets this issue but is conflicting and aimed at the old `pi-extensions` path.

## What This Changes

The issue requests a context-pruning lifecycle hook that emits safe metadata when old tool outputs are trimmed from active context.

## Best Possible Solution

Review, rebase, or replace https://github.com/openclaw/openclaw/pull/12581 with a current `pi-hooks` implementation that emits a privacy-safe pruning lifecycle event after successful pruning, with focused tests and docs.

## Reproduction Assessment

Yes, at source level. Current main’s pruning extension returns only rewritten messages, the pruner returns no metadata payload, and hook catalogs/docs omit `session:prune` or `context:prune`.

## Solution Assessment

Unclear. A pruning lifecycle hook is a plausible solution, but the maintainable implementation should align with the current `pi-hooks` path, define a privacy-safe payload, and resolve the stale linked PR before landing.
