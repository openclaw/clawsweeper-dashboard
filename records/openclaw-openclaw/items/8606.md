## Summary

Keep open because current main still lacks the requested context-pruning lifecycle hook, and an open implementation PR with closing syntax directly targets this issue. This is a new hook/product-contract feature with privacy-sensitive payload choices, not a stale cleanup close.

## What This Changes

Request a hook event that fires when context pruning trims old tool outputs so external handlers can index or act on pruning metadata.

## Best Possible Solution

Add a narrow, summary-only pruning lifecycle event after successful pruning in the current pi-hooks path, with focused regression tests and docs; review, rebase, replace, or close https://github.com/openclaw/openclaw/pull/12581 as the implementation path.

## Reproduction Assessment

Yes, at source level for the feature gap: current main prunes and returns rewritten messages, while source, hook catalogs, tests, and docs expose no session:prune or context:prune event. This is a new hook capability rather than broken existing behavior.

## Solution Assessment

Unclear until maintainers settle the hook contract and payload privacy boundary. A narrow session:prune-style event with safe summary metadata appears maintainable, but the open implementation candidate needs current-main review, rebase, or replacement.

## Security Review

Status: needs_attention

Summary: The feature request is security-sensitive because pruning hooks could expose sensitive tool output unless the payload is intentionally redacted or summary-only.

Concerns:
