## Summary

Keep open: current main still prunes context without emitting a pruning lifecycle hook, and https://github.com/openclaw/openclaw/pull/12581 is an open implementation candidate with closing syntax. This is a new hook contract with privacy-sensitive payload choices, so it needs maintainer review rather than cleanup closure.

## What This Changes

The issue requests a lifecycle hook such as session:prune or context:prune that fires after context pruning and exposes safe metadata about trimmed content.

## Best Possible Solution

Add a narrow, summary-only pruning lifecycle event after successful pruning in the current pi-hooks path, with focused regression coverage and public hook documentation; review, rebase, replace, or close https://github.com/openclaw/openclaw/pull/12581 as the implementation path.

## Reproduction Assessment

Yes, at source level: current main prunes and returns rewritten messages, while source, hook catalogs, tests, and docs expose no session:prune or context:prune event. This is a feature gap rather than broken existing behavior.

## Solution Assessment

Unclear until maintainers settle the hook contract and payload privacy boundary. A narrow session:prune-style event with safe summary metadata appears maintainable, but the open implementation candidate needs current-main review, rebase, or replacement.
