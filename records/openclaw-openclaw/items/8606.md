## Summary

Keep open because current main still lacks the requested context-pruning lifecycle hook, and the direct implementation candidate is open with a maintainer label.

## What This Changes

Request to emit a hook event when context pruning trims tool outputs so handlers can observe or index pruning metadata.

## Best Possible Solution

Add a narrow, summary-only pruning lifecycle event after successful pruning in the current pi-hooks path, with focused tests and public hook/session-pruning docs.

## Reproduction Assessment

Yes, at source level for the feature gap: current main prunes and returns rewritten messages, while source, hook catalogs, tests, and docs expose no session:prune or context:prune event. This is a new hook capability rather than a broken existing-behavior bug.

## Solution Assessment

Unclear until maintainers settle the hook contract. A narrow session:prune-style event with safe summary metadata appears maintainable, and https://github.com/openclaw/openclaw/pull/12581 is the active implementation candidate but needs current-main review or replacement.
