## Summary

Keep open: current main and v2026.6.10 still prune context without emitting a prune lifecycle hook, and https://github.com/openclaw/openclaw/pull/12581 is an open conflicting implementation candidate that needs maintainer product and security contract review.

## What This Changes

Request a lifecycle hook such as `session:prune` or `context:prune` after context pruning, with metadata about trimmed content for event-driven indexing.

## Best Possible Solution

Keep this as the canonical request and resolve it through the linked PR or a replacement that emits bounded, sanitized pruning metadata after successful pruning, with focused tests and public docs.

## Reproduction Assessment

Yes, at source level. Current main prunes through the context extension and pruner without emitting a pruning hook or returning hook metadata, and the hook catalogs and docs omit a pruning event.

## Solution Assessment

Unclear until maintainers choose the hook contract. A metadata-only `session:prune` lifecycle event looks like the narrowest maintainable direction, while raw pruned-content indexing needs explicit privacy, trust, timeout, and size-limit decisions.
