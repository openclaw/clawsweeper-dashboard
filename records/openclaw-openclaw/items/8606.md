## Summary

Keep this issue open. Current `main` still lacks a context-pruning lifecycle hook, and an open implementation pull request already closes this issue, so the right cleanup path is to review, rebase, replace, or close that linked pull request rather than close this report early.

## What This Changes

The issue requests a hook event that fires after context pruning with metadata about what was trimmed so external indexing can run event-driven.

## Best Possible Solution

Keep this issue open and use https://github.com/openclaw/openclaw/pull/12581, or a maintainer-approved replacement, to add a safe pruning lifecycle event in `src/agents/pi-hooks/context-pruning/*` with focused tests and docs.

Yes. Source inspection on current `main` shows the pruning path rewrites and returns messages without emitting a hook, and the hook catalog/docs have no pruning event.

## Solution Assessment
