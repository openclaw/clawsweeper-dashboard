## Summary

Keep open. Current main still lacks a context-pruning lifecycle hook or pruning metadata for hook consumers, and open PR #12581 explicitly targets this feature with closing syntax, so the issue should remain open until that PR or a maintainer-approved replacement lands.

## What This Changes

Add a context-pruning lifecycle hook that fires with pruning metadata so handlers can index or otherwise preserve access to pruned content.

## Best Possible Solution

Add a narrow, documented pruning lifecycle event from the context-pruning owner path after successful pruning, with safe summary metadata, focused tests, and public hook documentation.

## Reproduction Assessment

Yes, from source inspection rather than a failing runtime test: current main rewrites and returns pruned messages but exposes no `session:prune` or `context:prune` event in source or docs.

## Solution Assessment

Unclear until maintainers review the open implementation PR. A `session:prune` lifecycle event with safe summary metadata is the narrow direction, but the internal/plugin boundary and payload contract still need approval.
