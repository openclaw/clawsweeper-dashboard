## Summary

A hook fits OpenClaw’s plugin-extension direction, but maintainers must decide the permanent public surface, payload privacy, and non-blocking delivery semantics before landing it.

## What This Changes

Request a lifecycle hook that emits metadata when context pruning trims tool outputs so external indexing can react immediately.

## Best Possible Solution

Define one documented plugin lifecycle hook emitted after successful pruning, with privacy-safe counts and bounded identifiers, best-effort non-blocking delivery, and the preserved transcript remaining the supported source for indexing full content.

## Reproduction Assessment

Yes at source level: current main’s pruning extension returns rewritten messages without emitting any pruning lifecycle event. This is a clearly verifiable feature gap rather than a failing existing contract.

## Solution Assessment

A hook is the appropriate extension seam, but the linked implementation is not yet proven to be the best final design because maintainers must settle plugin exposure, payload privacy, and failure-isolation semantics.
