## Summary

Keep open because current main and the latest release still lack the requested pruning lifecycle hook, and the direct implementation candidate remains open, conflicting, and unmerged.

## What This Changes

The issue requests a context-pruning lifecycle hook that exposes safe metadata about pruned tool output for event-driven indexing or memory workflows.

## Best Possible Solution

Add a summary-only pruning lifecycle event after successful pruning in the current pi-hooks path, with focused regression tests and public hook/session-pruning docs, then resolve the existing implementation candidate or replace it with a current-main patch.

## Reproduction Assessment

Yes, at source level for the feature gap: current main prunes and returns rewritten messages, while source, hook catalogs, tests, and docs expose no session:prune or context:prune event. This is not a broken existing-behavior bug.

## Solution Assessment

Unclear until maintainers finalize the hook contract. A narrow session:prune-style lifecycle event with safe summary metadata appears maintainable, and https://github.com/openclaw/openclaw/pull/12581 is the active implementation candidate but needs current-main review.
