## Summary

Keep this open: current `main` still has no context-pruning lifecycle hook or pruning metadata payload, and the direct implementation PR is still open and conflicting.

## What This Changes

The issue requests a hook event that fires after context pruning trims tool outputs, with enough metadata for downstream indexing/search workflows.

## Best Possible Solution

Land or replace the open pruning-hook PR with a maintainer-approved event contract that emits safe summary metadata after successful pruning and documents the lifecycle surface.

## Reproduction Assessment

Yes. Source inspection on current `main` shows the pruning path rewrites and returns messages without emitting a hook or exposing pruning metadata; this is a missing lifecycle surface rather than a runtime crash.

## Solution Assessment

Unclear until maintainers choose the public shape of the event. The narrowest maintainable solution is to add safe summary metadata in the existing context-pruning path and document whether the event is internal-only or also available to plugins.
