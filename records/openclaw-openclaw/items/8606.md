## Summary

## What This Changes

The issue requests a context/session pruning lifecycle hook that fires after pruning with metadata for indexing pruned content.

## Best Possible Solution

Land or replace the linked pruning-hook PR with a maintainer-approved event contract that emits safe summary metadata after successful pruning and documents the lifecycle surface.

## Reproduction Assessment

Yes. Source inspection on current main shows the pruning path rewrites messages and returns them without hook emission or pruning metadata; this is a missing lifecycle surface rather than a runtime crash.

## Solution Assessment

Unclear until maintainers choose the public shape of the event. The narrowest maintainable path is a safe summary-metadata hook in the existing context-pruning path, with docs stating whether it is internal-only or also a plugin hook.
