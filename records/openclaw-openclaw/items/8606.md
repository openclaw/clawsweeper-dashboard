## Summary

Keep open because current main still lacks the requested context-pruning lifecycle hook, and the open implementation candidate at https://github.com/openclaw/openclaw/pull/12581 still directly targets this issue.

## What This Changes

The issue requests a hook event fired after context pruning, with safe metadata about pruned content so downstream handlers can index or otherwise process what left active context.

## Best Possible Solution

Add a narrow summary-only pruning lifecycle event after successful pruning in the current pi-hooks path, with focused regression tests and public hooks/session-pruning docs once the contract is approved.

## Reproduction Assessment

Yes, at source level for the feature gap: current main prunes and returns rewritten messages, while source, hook catalogs, tests, and docs expose no session:prune or context:prune event. This is not a broken existing-behavior bug.

## Solution Assessment

Unclear until maintainers settle the hook contract. A narrow session:prune-style event with safe summary metadata appears maintainable, and https://github.com/openclaw/openclaw/pull/12581 is the active implementation candidate but needs current-main review.
