## Summary

Keep open because current main still lacks the requested context-pruning lifecycle hook, and an open implementation PR explicitly targets this issue but is unmerged and currently conflicting.

## What This Changes

Expose a context-pruning lifecycle hook with safe pruning metadata so custom handlers can react when tool-result content is pruned from active context.

## Best Possible Solution

Add a narrow summary-only pruning lifecycle event after successful pruning in the current pi-hooks path, with focused regression tests and public hook/session-pruning docs.

## Reproduction Assessment

Yes, at source level for the feature gap: current main prunes and returns rewritten messages, while source, hook catalogs, tests, and docs expose no session:prune or context:prune event. This is a new hook capability rather than a broken existing-behavior bug.

## Solution Assessment

Unclear until maintainers settle the hook contract. A narrow session:prune-style event with safe summary metadata appears maintainable, and https://github.com/openclaw/openclaw/pull/12581 is the active implementation candidate but needs current-main review or replacement.
