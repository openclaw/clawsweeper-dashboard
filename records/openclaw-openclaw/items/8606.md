## Summary

Keep open: current main and v2026.5.18 still lack a context-pruning lifecycle hook or pruning metadata payload, and https://github.com/openclaw/openclaw/pull/12581 is the active open implementation candidate rather than a reason to close this issue early.

## What This Changes

Add a context/session pruning lifecycle hook with metadata so handlers can react when context pruning trims tool outputs.

## Best Possible Solution

Land or replace the linked pruning-hook PR with a maintainer-approved event contract that emits bounded summary metadata after successful pruning and documents the lifecycle surface clearly.

## Reproduction Assessment

Yes, at source level. Current main rewrites and returns pruned messages without hook emission or pruning metadata, and the hook catalog/docs still omit a pruning lifecycle event.

## Solution Assessment

Unclear until maintainers choose the event shape. The narrow maintainable path is a safe summary-metadata hook in the existing context-pruning path, with docs stating whether it is internal-only or plugin-visible.
