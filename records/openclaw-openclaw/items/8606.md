## Summary

Keep open: current main and the latest release still have no context-pruning lifecycle hook or pruning metadata payload, and the linked open PR with closing syntax is the active implementation candidate rather than a reason to close the issue early.

## What This Changes

Request a lifecycle hook such as `session:prune` or `context:prune` after context pruning, with metadata so follow-up indexing can react to trimmed content.

## Best Possible Solution

Land or replace the linked pruning-hook PR with a maintainer-approved event contract that emits safe summary metadata after successful pruning and documents whether the lifecycle surface is internal-only or plugin-visible.

## Reproduction Assessment

Yes, at source level. Current main rewrites and returns pruned messages without hook emission or pruning metadata, and the hook catalog/docs still omit a pruning lifecycle event.

## Solution Assessment

Unclear until maintainers choose the public event shape. The narrowest maintainable path is a safe summary-metadata hook in the existing context-pruning path, with docs stating whether it is internal-only or also a plugin hook.
