## Summary

Keep open because current main still lacks the requested context-pruning lifecycle hook, and an open maintainer-labeled implementation PR explicitly targets the issue but has not merged.

## What This Changes

The issue requests a new `session:prune` or `context:prune` lifecycle hook after context pruning, with metadata downstream handlers can use for indexing.

## Best Possible Solution

Review, rebase, land, or replace the open implementation PR with a narrow summary-only prune lifecycle event in the current `pi-hooks` path, with focused tests and public docs.

## Reproduction Assessment

Yes, at source level: current main prunes and returns rewritten messages, while source, hook catalogs, tests, and docs expose no pruning event. This is a feature gap rather than a broken existing behavior bug.

## Solution Assessment

Unclear until maintainers settle the hook contract. A narrow `session:prune` event with safe summary metadata appears maintainable, and https://github.com/openclaw/openclaw/pull/12581 is the active implementation candidate but needs current-main review or replacement.
