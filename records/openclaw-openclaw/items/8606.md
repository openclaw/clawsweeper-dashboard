## Summary

Keep open: current main still prunes tool-result context without emitting a pruning lifecycle hook, and the direct implementation candidate remains open and conflicting rather than merged.

## What This Changes

The issue requests a context/session pruning lifecycle hook with safe metadata so external memory or indexing handlers can react after pruning.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/12581 or a maintainer-approved replacement that emits a privacy-safe pruning lifecycle event after successful pruning, with focused source tests and public docs.

## Reproduction Assessment

Yes, source-level: current main rewrites pruned context messages without emitting an event, and the hook catalogs plus docs contain no pruning lifecycle hook.

## Solution Assessment

Yes in direction, but the final payload contract needs maintainer judgment. A generic summary-only lifecycle hook is the maintainable seam; raw pruned content should not be exposed without an explicit security decision.
