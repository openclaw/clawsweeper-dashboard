## Summary

Keep open because current main still lacks the requested pruning lifecycle hook, and the direct implementation candidate at https://github.com/openclaw/openclaw/pull/12581 is open, conflicting, and unmerged.

## What This Changes

Expose a context/session pruning lifecycle hook with metadata about trimmed tool output so handlers can index pruned content.

## Best Possible Solution

Add a summary-only pruning lifecycle event after successful pruning in the current pi-hooks path, with focused regression tests and public hooks/session-pruning docs.

## Reproduction Assessment

Yes, at source level for the feature gap: current main prunes and returns rewritten messages, while source, hook catalogs, tests, and docs expose no session:prune or context:prune event. This is not a broken existing-behavior bug.

## Solution Assessment

Unclear until maintainers settle the hook contract. A narrow session:prune-style event with safe summary metadata appears maintainable, and https://github.com/openclaw/openclaw/pull/12581 is the active implementation candidate but needs current-main review.
