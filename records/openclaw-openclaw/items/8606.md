## Summary

Keep open: current main and v2026.5.22 still prune by returning rewritten messages without a pruning lifecycle hook or metadata payload, and the direct implementation PR is open rather than merged.

## What This Changes

The issue requests a context-pruning lifecycle hook that fires after pruning and exposes safe metadata about what was trimmed.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/12581 with a maintainer-approved pruning lifecycle event that emits bounded sanitized summary metadata after successful pruning, documents visibility, and covers pruning plus hook-delivery tests.

## Reproduction Assessment

Yes, at source level. Current main rewrites and returns pruned messages without hook emission or pruning metadata, and the hook catalogs/docs still omit a pruning lifecycle event.

## Solution Assessment

Unclear until maintainers choose the event shape and visibility. The narrow maintainable path is a safe summary-metadata hook in the existing context-pruning path, with docs stating whether it is internal-only or plugin-visible.
