## Summary

Current main still has no pruning lifecycle event, and the open maintainer-authored PR explicitly targets this issue; keep the issue open until that PR or an approved replacement lands. The main unresolved product question is whether consumers receive safe summary metadata only or any raw pruned tool content.

## What This Changes

Request a lifecycle hook that reports what context pruning removed so external indexing can react without scheduled polling.

## Best Possible Solution

Expose a generic post-success pruning lifecycle event at the pruning owner boundary with stable safe metadata, focused tests, and public hook documentation; keep raw pruned content out of the default payload unless maintainers approve an explicit sensitive-data contract.

## Reproduction Assessment

Yes at source level: current main's context handler calls the pruner and returns rewritten messages without emitting a lifecycle event, while the hook catalog and docs contain no prune hook.

## Solution Assessment

Yes for a safe-metadata lifecycle event at the pruning boundary; exposing raw removed content by default is not the best solution because it creates a sensitive-data contract that requires explicit maintainer approval.
