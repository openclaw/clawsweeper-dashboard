## Summary

Keep open: current main and v2026.5.20 still prune by returning rewritten messages without a pruning lifecycle hook or metadata payload, and https://github.com/openclaw/openclaw/pull/12581 remains the open implementation path that needs maintainer review.

## What This Changes

Request a context-pruning lifecycle hook that fires after pruning with metadata about what was trimmed so external indexing can react without polling.

## Best Possible Solution

Land or replace the linked pruning-hook PR with a maintainer-approved session:prune-style event that emits bounded summary metadata after successful pruning, documents the internal/plugin visibility boundary, and covers pruning plus hook-delivery tests.

## Reproduction Assessment

Yes, at source level. Current main and v2026.5.20 rewrite and return pruned messages without hook emission or pruning metadata, and the hook catalogs/docs still omit a pruning lifecycle event.

## Solution Assessment

Unclear until maintainers choose the event shape and visibility. The narrow maintainable path is a safe summary-metadata hook in the existing context-pruning path, with docs stating whether it is internal-only or plugin-visible.
