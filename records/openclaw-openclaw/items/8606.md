Keep open. Current main still lacks a context-pruning lifecycle hook or pruning metadata contract, and the open linked implementation PR should be reviewed, rebased, or replaced before this issue closes.

## What This Changes

The issue requests a hook event such as `session:prune` or `context:prune` that fires after context pruning with metadata about what was trimmed.

## Best Possible Solution

Land a current `pi-hooks` implementation that emits a privacy-safe pruning lifecycle event only after successful pruning, with focused tests and public hook documentation.

Yes, at source level. Current main’s pruning path returns only rewritten messages, the pruner exposes no metadata payload, and hook catalogs/docs omit a prune lifecycle event.

## Solution Assessment

Unclear. A pruning lifecycle hook is plausible, but the best implementation needs maintainer agreement on event naming and privacy-safe payload shape before the current `pi-hooks` path changes.
