## Summary

Current main still performs context pruning without emitting a pruning lifecycle event, and the open closing pull request https://github.com/openclaw/openclaw/pull/12581 remains the canonical implementation path. Keep this issue open until that PR or a maintainer-approved replacement lands, with explicit review of the plugin boundary and payload privacy.

## What This Changes

Add a lifecycle hook that reports safe metadata when context pruning trims tool outputs.

## Best Possible Solution

Land one canonical, documented plugin-observable pruning event after successful pruning, expose bounded summary metadata by default, and leave content indexing and long-term memory storage to an external memory plugin.

## Reproduction Assessment

Yes at source level: current main’s pruning handler rewrites messages without emitting a lifecycle event, and the hook catalogs and relevant docs expose no pruning event. This is a missing-capability check rather than a failing runtime bug reproduction.

## Solution Assessment

Yes directionally: a reusable lifecycle hook is the narrowest maintainable solution. The best permanent form is a documented plugin-facing contract with metadata-only defaults, rather than core-owned indexing or unrestricted raw-content exposure.
