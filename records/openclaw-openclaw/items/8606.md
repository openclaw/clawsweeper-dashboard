## Summary

Current main still lacks the requested pruning lifecycle event, and the open linked PR remains the explicit implementation candidate. The issue should stay open until maintainers approve a safe metadata contract and either repair and land that PR or replace it.

## What This Changes

Add a lifecycle hook that reports what context pruning trimmed so external indexing or memory workflows can react without polling.

## Best Possible Solution

Return minimal pruning summary facts from the pure pruner, emit one owner-approved lifecycle event only after a successful rewrite, and keep raw pruned content out of the default payload unless separately security-reviewed.

## Reproduction Assessment

Yes. Current-main source provides a high-confidence reproduction: the context-pruning handler rewrites messages but neither it nor the hook catalogs expose a pruning lifecycle event.

## Solution Assessment

The lifecycle point is appropriate, but the best solution is a metadata-only result and event contract at the pruning owner boundary, not default exposure of the removed content itself.
