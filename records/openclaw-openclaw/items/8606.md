## Summary

Keep open. Current main still lacks a context-pruning lifecycle hook, and https://github.com/openclaw/openclaw/pull/12581 is a live implementation candidate with closing syntax, so the issue should wait for that PR or a maintainer-approved replacement to land.

## What This Changes

Request to add a hook event such as `context:prune` or `session:prune` after context pruning, with metadata about trimmed tool outputs.

## Best Possible Solution

Review, rebase, or replace https://github.com/openclaw/openclaw/pull/12581 with a current `pi-hooks` implementation that emits a privacy-safe `session:prune` event after successful pruning, with focused tests and docs.

## Reproduction Assessment

Yes, at source level. Current main’s pruning extension returns rewritten messages without hook emission, the pruner exposes no pruning metadata payload, and hook docs/catalogs omit `session:prune` or `context:prune`.

## Solution Assessment

Unclear. A pruning lifecycle hook is a plausible direction, but the maintainable solution should use summary-only metadata, align with the current `pi-hooks` paths, and resolve the stale/conflicting linked PR before landing.
