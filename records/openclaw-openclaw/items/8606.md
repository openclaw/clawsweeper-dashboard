## Summary

Keep open because current main still has no context-pruning lifecycle hook or pruning metadata for hook consumers, and open PR #12581 explicitly targets this feature.

## What This Changes

Request a new context-pruning hook event with metadata so handlers can index content after it is pruned from active context.

## Best Possible Solution

Land #12581 or a maintainer-approved replacement that emits a safe summary-only pruning lifecycle event after successful pruning, with focused tests and docs.

## Reproduction Assessment

Yes, at source level: current main rewrites and returns pruned messages but exposes no session:prune or context:prune event in source, hook catalogs, tests, or docs. This is a feature gap rather than broken existing behavior.

## Solution Assessment

Unclear until maintainers review the event contract. A narrow session:prune-style lifecycle event with safe summary metadata appears to be the maintainable direction, and #12581 is the active candidate.
