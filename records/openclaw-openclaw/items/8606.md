- `impact:session-state`: The request concerns context pruning, active session memory visibility, and downstream indexing after context changes.

## Summary

Keep open: current main and v2026.5.22 still prune by returning rewritten messages without a pruning lifecycle hook or payload metadata, and https://github.com/openclaw/openclaw/pull/12581 remains the open implementation path rather than a merged fix.

## What This Changes

Expose a context-pruning lifecycle hook, such as `session:prune` or `context:prune`, with metadata about trimmed tool output so external indexing can react when pruning occurs.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/12581 with a maintainer-approved `session:prune`-style event that emits bounded sanitized summary metadata after successful pruning, documents internal/plugin visibility, and covers pruning plus hook-delivery tests.

## Reproduction Assessment

Yes, at source level. Current main rewrites and returns pruned messages without hook emission or pruning metadata, and the hook catalogs/docs still omit a pruning lifecycle event.

## Solution Assessment

Unclear until maintainers choose the event shape and visibility. The narrow maintainable path is a safe summary-metadata hook in the existing context-pruning path, with docs stating whether it is internal-only or plugin-visible.
