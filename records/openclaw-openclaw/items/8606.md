## Summary

Keep open. Current main still lacks a context-pruning lifecycle hook or pruning metadata surface, and the open linked implementation PR uses closing syntax for this issue but is conflicting and targets the old pruning path.

## What This Changes

Add a context/session pruning lifecycle hook that fires after context pruning with metadata about what was trimmed, so custom handlers can index pruned content.

## Best Possible Solution

Review, rebase, or replace https://github.com/openclaw/openclaw/pull/12581 with a current `pi-hooks` implementation that emits a privacy-safe pruning lifecycle event after successful pruning, with focused tests and docs.

## Reproduction Assessment

Yes, at source level. Current main’s pruning extension returns only rewritten messages, the pruner returns no metadata payload, and hook catalogs/docs omit `session:prune` or `context:prune`.

## Solution Assessment

Unclear. A pruning lifecycle hook is plausible, but the maintainable solution needs a current `pi-hooks` implementation, a privacy-safe payload contract, and maintainer agreement on the event surface before landing.
