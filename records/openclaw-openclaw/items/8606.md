## Summary

Keep open: current `main` and latest release still lack a context-pruning lifecycle hook or pruning metadata payload, and the linked implementation candidate remains open and conflicting rather than merged.

## What This Changes

The issue requests a `context:prune` or `session:prune` lifecycle hook with metadata so handlers can react after context pruning without polling.

## Best Possible Solution

Land or replace the linked pruning-hook PR with a maintainer-approved event contract that emits bounded summary metadata only after successful pruning and documents the lifecycle surface clearly.

## Reproduction Assessment

Yes, at source level. Current `main` rewrites and returns pruned messages without hook emission or pruning metadata, and the hook catalog/docs still omit a pruning lifecycle event.

## Solution Assessment

Unclear until maintainers choose the event shape. The narrow maintainable path is a safe summary-metadata hook in the existing context-pruning path, with docs stating whether it is internal-only or plugin-visible.
