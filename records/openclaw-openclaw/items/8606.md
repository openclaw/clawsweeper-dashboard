## Summary

Keep open because current main still lacks the requested pruning lifecycle hook, and #12581 explicitly targets this issue but remains open and unmerged.

## What This Changes

The issue requests a context-pruning hook event with metadata so custom handlers can process content after it is trimmed from active context.

## Best Possible Solution

Add a summary-only pruning lifecycle event after successful pruning, with focused regression tests and public docs, while preserving full raw transcript data only in existing session storage.

## Reproduction Assessment

Yes, at source level for the feature gap: current main prunes and returns rewritten messages, while source, hook catalogs, tests, and docs expose no session:prune or context:prune event. This is not a broken existing-behavior bug.

## Solution Assessment

Unclear until maintainers finalize the hook contract. A narrow session:prune-style lifecycle event with safe summary metadata appears maintainable, and #12581 is the active implementation candidate but needs review against the current pi-hooks path.
