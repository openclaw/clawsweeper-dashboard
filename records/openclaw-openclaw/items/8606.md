## Summary

Keep open because current main still lacks a context-pruning lifecycle hook or pruning metadata for hook consumers, and open PR #12581 explicitly targets this gap.

## What This Changes

Add a hook event after context pruning with metadata about trimmed tool results so downstream handlers can index or search pruned content.

## Best Possible Solution

Land #12581 or a maintainer-approved replacement that emits a safe summary-only pruning lifecycle event after successful pruning, with focused tests and public hook docs.

## Reproduction Assessment

Yes, at source level: current main rewrites and returns pruned messages but exposes no session:prune or context:prune event in source, hook catalogs, tests, or docs. This is a feature gap rather than broken existing behavior.

## Solution Assessment

Unclear until maintainers finalize the event contract. A narrow session:prune-style lifecycle event with safe summary metadata appears maintainable, and #12581 is the active candidate.
