## Summary

Keep open because current main still lacks a context-pruning lifecycle hook or pruning metadata contract, and open PR #12581 explicitly targets this issue but has not merged.

## What This Changes

Add a hook event that fires after context pruning so custom handlers can index safe metadata about trimmed content.

## Best Possible Solution

Land #12581 or a maintainer-approved replacement that emits a safe summary-only pruning lifecycle event after successful pruning, with focused tests and public hook docs.

## Reproduction Assessment

Yes, at source level for the feature gap: current main rewrites and returns pruned messages, while source, hook catalogs, tests, and docs expose no session:prune or context:prune event. This is not a broken existing-behavior bug.

## Solution Assessment

Unclear until maintainers finalize the event contract. A narrow session:prune-style lifecycle event with safe summary metadata appears maintainable, and #12581 is the active implementation candidate.
