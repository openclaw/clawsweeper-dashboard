## Summary

Keep open because current main still lacks the requested context-pruning lifecycle hook and pruning metadata contract, while open PR #12581 explicitly targets this issue and has not merged.

## What This Changes

The issue requests a context/session pruning hook that fires after old tool outputs are pruned so downstream handlers can index or otherwise process the pruned content.

## Best Possible Solution

Land #12581 or a maintainer-approved replacement that emits a safe summary-only pruning lifecycle event after successful pruning, with focused tests and public hook docs.

## Reproduction Assessment

Yes, at source level for the feature gap: current main rewrites and returns pruned messages, while source, hook catalogs, tests, and docs expose no session:prune or context:prune event. This is not a broken existing-behavior bug.

## Solution Assessment

Unclear until maintainers finalize the hook contract. A narrow session:prune-style lifecycle event with safe summary metadata appears maintainable, and #12581 is the active implementation candidate.
