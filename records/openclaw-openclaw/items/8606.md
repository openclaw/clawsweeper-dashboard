## Summary

Keep open. Current main still has no context-pruning lifecycle hook or pruning metadata for hook consumers, and open PR #12581 explicitly targets this feature, so this issue should remain open until that PR or a maintainer-approved replacement lands.

## What This Changes

Expose a context-pruning lifecycle hook with metadata about trimmed tool-output content so handlers can react after pruning runs.

## Best Possible Solution

Add a narrow, documented pruning lifecycle event from the context-pruning owner path after successful pruning, with safe summary metadata and focused regression coverage.

## Reproduction Assessment

Yes, at source level: current main rewrites and returns pruned messages but exposes no session:prune or context:prune event in source, tests, or docs. This is a feature gap rather than a broken existing contract.

## Solution Assessment

Unclear until maintainers review #12581. A session:prune lifecycle event with safe summary metadata is the narrow maintainable direction, but the event boundary and payload contract still need approval.
