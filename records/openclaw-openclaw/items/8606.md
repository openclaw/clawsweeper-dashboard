## Summary

Keep open: current main still prunes tool-result context without emitting or documenting a context-pruning lifecycle hook, and the direct implementation candidate remains open and conflicting rather than merged.

## What This Changes

The issue requests a context-pruning lifecycle hook that fires after pruning and exposes safe metadata for follow-up indexing or memory workflows.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/12581 or a maintainer-approved replacement that emits a privacy-safe pruning lifecycle event only after successful pruning, with focused tests and docs for the supported payload.

## Reproduction Assessment

Yes, source-level: current main rewrites pruned context messages without emitting an event, and the hook catalogs plus docs contain no pruning lifecycle hook.

## Solution Assessment

Yes in direction, but the final event name and payload contract need maintainer judgment. A summary-only lifecycle hook is the maintainable seam; raw pruned content should not be exposed without an explicit security decision.
