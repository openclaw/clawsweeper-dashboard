## Summary

Keep open: current main still lacks a context-pruning lifecycle hook, and an open implementation PR with closing syntax already targets this issue. The remaining work is a new hook contract with privacy-sensitive payload choices, so it needs maintainer review rather than cleanup closure.

## What This Changes

Request to emit a hook event when context pruning trims tool outputs so external handlers can index or otherwise react to pruned content.

## Best Possible Solution

Add a narrow, summary-only pruning lifecycle event after successful pruning in src/agents/pi-hooks/context-pruning/*, with focused tests and public hook documentation; review, rebase, replace, or close https://github.com/openclaw/openclaw/pull/12581 as the implementation path.

## Reproduction Assessment

Yes, at source level: current main prunes and returns rewritten messages, while source, hook catalogs, tests, and docs expose no session:prune or context:prune event. This is a feature gap rather than broken existing behavior.

## Solution Assessment

Unclear until maintainers settle the hook contract and payload privacy boundary. A narrow session:prune-style event with safe summary metadata appears maintainable, but the open implementation candidate needs current-main review, rebase, or replacement.

## Security Review

Status: needs_attention

Summary: Needs attention because a pruning hook can persist transcript or tool-output details unless its payload is intentionally summary-only.

Concerns:

- **[medium] Keep pruned-content payloads summary-only:** `src/agents/pi-hooks/context-pruning/pruner.ts:287`
  - body: A pruning hook that forwards raw pruned tool output could persist secrets, local paths, private endpoints, or other sensitive transcript data into logs, indexes, or third-party handlers. The contract should default to safe summary metadata and require explicit maintainer approval for raw-content access.
