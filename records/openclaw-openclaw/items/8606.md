## Summary

Keep open because current main still lacks the requested context-pruning lifecycle hook, and the open implementation candidate at https://github.com/openclaw/openclaw/pull/12581 directly targets this issue.

## What This Changes

Request to add a context-pruning hook event with metadata so pruned content can be indexed or handled after trimming.

## Best Possible Solution

Add a narrow summary-only pruning lifecycle event after successful pruning in the current `pi-hooks` path, with focused regression tests and public hooks/session-pruning docs.

## Reproduction Assessment

Yes, at source level for the feature gap: current main prunes and returns rewritten messages, while source, hook catalogs, tests, and docs expose no `session:prune` or `context:prune` event. This is not a broken existing-behavior bug.

## Solution Assessment

Unclear until maintainers settle the hook contract. A narrow `session:prune`-style event with safe summary metadata appears maintainable, and https://github.com/openclaw/openclaw/pull/12581 is the active implementation candidate but needs current-main review.
