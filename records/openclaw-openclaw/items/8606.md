## Summary

Keep open: current main and v2026.5.22 still prune by returning rewritten messages without a pruning lifecycle hook or payload metadata, and the linked implementation PR remains open and unresolved.

## What This Changes

Add a context-pruning lifecycle hook that fires after pruning and exposes safe metadata about what was trimmed for downstream indexing.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/12581 with a maintainer-approved session:prune-style event that emits bounded sanitized summary metadata after successful pruning, documents the internal/plugin visibility boundary, and covers pruning plus hook-delivery tests.

Yes, at source level. Current main rewrites and returns pruned messages without hook emission or pruning metadata, and the hook catalogs/docs still omit a pruning lifecycle event.

## Solution Assessment

Unclear until maintainers choose the event shape and visibility. The narrow maintainable path is a safe summary-metadata hook in the existing context-pruning path, with docs stating whether it is internal-only or plugin-visible.
