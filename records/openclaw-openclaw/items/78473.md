Keep open: current main still contains the reported watchdog suppression path, and the post-stall rebuild path needs better bounded diagnostics before this can be considered fixed.

## What This Changes

The issue asks OpenClaw to fix Telegram polling watchdog recovery so stale `getUpdates` is not masked by outbound API activity and transport-rebuild failures are logged or handled.

## Best Possible Solution

Keep the issue open and land a narrow Telegram plugin fix that makes the watchdog depend on completed `getUpdates` liveness, then adds bounded logging or timeout coverage for each post-stall rebuild/startup step.

## Solution Assessment
