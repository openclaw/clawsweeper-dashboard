## Label Justifications

- `P1`: The issue tracks a real provider-routing credential misroute that can break agent runs and expose provider-specific API keys to the wrong endpoint.

## Summary

Keep open: main has fixed the standard Google omitted-api route and the remediation hint, but current source still allows endpoint-less configured provider/model rows to resolve to an OpenAI-family transport under a non-OpenAI provider id.

## What This Changes

The issue asks OpenClaw to prevent provider credentials from being sent to the wrong OpenAI-compatible endpoint when provider or model API/base URL metadata is incomplete.

## Best Possible Solution

Define and enforce a provider/transport coherence rule before dispatch, preserve only explicitly intentional custom OpenAI-compatible behavior, and keep provider-owned catalog gaps such as OpenCode Zen in the owning plugin.

Yes from source inspection for endpoint-less configured provider/model rows; no fresh live request was performed during this read-only review.

## Solution Assessment

Unclear until maintainers choose the compatibility boundary; the standard Google fix was appropriate, but the generic fail-closed rule must preserve explicit custom OpenAI-compatible provider contracts.
