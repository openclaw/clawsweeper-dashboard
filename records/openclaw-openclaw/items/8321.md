## Summary

Keep this open. Current main still lacks a per-cron-job cache/context-pruning override, the prior implementation PR is closed and unmerged, and the direct-script cron work is complementary rather than a replacement for agent-run cron jobs.

## What This Changes

Add per-cron-job cache/context-pruning control so isolated cron jobs can disable or tune prompt cache pruning independently of global agent defaults.

## Best Possible Solution

Implement an additive top-level contextPruning?: AgentContextPruningConfig cron job override using the existing mode: "off" | "cache-ttl" contract, and merge it into the effective agent defaults before isolated cron execution.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives high confidence that current main lacks the requested job-level field and still reads pruning from effective agent defaults.

## Solution Assessment

Yes. A top-level contextPruning override is the narrow maintainable direction because it reuses the existing pruning config type; a disableCache boolean would solve only one mode and create a second API shape.

## Repair Work Prompt

Implement issue #8321 as a focused feature PR. Add an additive top-level `contextPruning?: AgentContextPruningConfig` field to cron jobs, not a one-off `disableCache` boolean. Thread it through shared cron types and persistence, cron normalization/create/patch/list behavior, gateway protocol schemas/types, agent cron tool schemas, CLI add/edit flags or structured input, Control UI types/form state, docs, and CHANGELOG.md. In isolated cron execution, merge `job.contextPruning` into the effective `cfgWithAgentDefaults.agents.defaults.contextPruning` before the embedded runner sees the config, so `mode: "off"` on a job overrides a global/per-agent `mode: "cache-ttl"`. Keep #24597 direct script/no-LLM execution separate and do not change elevated exec policy or unrelated cron behavior. Add regression coverage for create/update round-trip and runner override precedence.
