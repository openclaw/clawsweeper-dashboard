## Summary

Current main addresses the issue’s deterministic shell-script examples with direct command cron payloads and reduces command-like agent prompts through lightweight bootstrap context, but model-backed cron jobs still cannot override inherited context-pruning policy per job. The prior direct implementation at https://github.com/openclaw/openclaw/pull/8341 closed without merging, and no viable replacement currently owns the remaining agent-turn behavior, so this issue is still necessary but needs a maintainer product decision on adding another persisted cron setting.

## What This Changes

Add optional per-job cache or context-pruning control so isolated model-backed cron jobs can override agent-level pruning defaults.

## Best Possible Solution

Keep command payloads as the canonical zero-LLM path, and if maintainers approve the remaining use case, add one additive top-level `contextPruning` override for model-backed cron jobs with explicit job-over-agent-over-global precedence and complete storage, protocol, client, UI, docs, and runtime coverage.

## Reproduction Assessment

Yes, at source level. Current main clearly provides command jobs and lightweight bootstrap context, while the model-backed cron contract, persisted job shape, and protocol schemas still contain no per-job context-pruning override.

## Solution Assessment

Yes, with a narrower boundary than the original examples imply: deterministic scripts should use command payloads, while a typed pruning override should apply only to cron jobs that still require an agent turn. A standalone `disableCache` boolean would duplicate the existing pruning contract and be less extensible.
