## Summary

Keep open. Current main still has no per-job cache/context-pruning override for cron jobs, the prior implementation PR is closed unmerged, and the direct-exec cron thread is complementary rather than a replacement for agent-run cron jobs that need per-job pruning policy.

## What This Changes

The issue requests a per-cron-job setting to disable or override prompt cache/context pruning for isolated cron jobs.

## Best Possible Solution

Add an additive job-level contextPruning?: AgentContextPruningConfig override, round-trip it through cron storage, gateway, CLI, UI, tool, and docs surfaces, and merge it into the effective agent defaults for that scheduled run.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not as a strict bug repro, but yes at source level: current main has no disableCache or contextPruning path in cron job types, persistence, gateway add/update/list schemas, CLI/UI surfaces, or isolated execution.

## Solution Assessment

Is this the best way to solve the issue? Yes directionally, but the maintainable API should reuse AgentContextPruningConfig mode off/cache-ttl unless maintainers explicitly choose a simpler boolean surface.
