## Summary

Keep open. Current main and the latest release still lack a per-cron-job cache or context-pruning override; the prior implementation pull request was closed unmerged, and related direct-exec or per-agent context-pruning work does not replace per-job control for agent-run cron jobs.

## What This Changes

Request an optional cron-job-level cache/context-pruning setting so isolated scheduled jobs can disable prompt-cache pruning independently of global agent defaults.

## Best Possible Solution

Add an additive top-level cron `contextPruning?: AgentContextPruningConfig` override, thread it through persistence, gateway protocol, CLI, Control UI, docs, cron tool surfaces, and isolated-run config composition, with regression coverage for job-level `mode: "off"` overriding defaults-level `cache-ttl`.

## Reproduction Assessment

Yes, at source level. Current main still has no job-level `disableCache` or `contextPruning` field in cron types, persistence, gateway schema, or isolated-run config composition.

## Solution Assessment

Unclear. The additive `contextPruning` direction looks more maintainable than a one-off boolean, but maintainers still need to choose the public cron API and merge semantics.
