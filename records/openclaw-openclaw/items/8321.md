## Summary

Keep this issue open. Current main still has only payload-level `lightContext` for cron jobs, while cron storage, Gateway schemas, CLI/UI/docs surfaces, and the isolated runner do not expose or apply a per-job cache/context-pruning override; the prior implementation PR is closed unmerged.

## What This Changes

The issue requests per-cron-job cache/context-pruning control so simple isolated cron jobs can avoid unnecessary prompt-cache/context costs.

## Best Possible Solution

Add a top-level `contextPruning?: AgentContextPruningConfig` cron job override, round-trip it through cron APIs and clients, and merge it into the effective agent defaults before isolated cron invokes the embedded runner.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: Gateway create/update schemas reject the proposed top-level fields, persisted cron types do not store them, and isolated cron reads pruning only from effective agent defaults.

## Solution Assessment

Yes. The narrowest maintainable direction is the maintainer-discussed `contextPruning` override using the existing `mode: "off" | "cache-ttl"` contract rather than a separate boolean `disableCache`.

## Repair Work Prompt

Implement per-job cron context-pruning control for openclaw/openclaw#8321. Add an additive top-level `contextPruning?: AgentContextPruningConfig` field to cron jobs using the existing `mode: "off" | "cache-ttl"` contract; do not add a separate `disableCache` boolean unless maintainers ask. Thread the field through cron types, shared persistence, normalization/create/update patch handling, Gateway TypeBox schemas, CLI add/edit surfaces, agent cron tool schema/help, Control UI types/controllers/views as needed, docs, and CHANGELOG.md. Apply the job override by merging it into the effective `cfgWithAgentDefaults.agents.defaults.contextPruning` before isolated cron invokes the embedded runner, while leaving global and per-agent defaults unchanged for jobs without the override. Add regression coverage proving a job-level `mode: "off"` overrides global/per-agent `mode: "cache-ttl"` for an isolated cron run. Treat #8341 as useful prior source context only; #24597 direct script execution and #41484 elevated exec scoping are separate and should not be folded into this PR.
