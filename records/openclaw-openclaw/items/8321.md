## Summary

Keep open. Current main still lacks per-cron-job cache or context-pruning control, and the remaining work is a public cron API/config decision rather than conservative cleanup.

## What This Changes

The issue requests a per-cron-job option to disable or override prompt caching/context pruning for isolated scheduled jobs.

## Best Possible Solution

Add an additive job-level `contextPruning?: AgentContextPruningConfig` override, round-trip it through cron API/storage/UI/tool surfaces, and merge it into effective agent defaults only for that scheduled run.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high-confidence proof that current main lacks job-level `disableCache` or `contextPruning` fields while runtime pruning still reads effective agent defaults.

## Solution Assessment

Yes directionally, but the maintainable solution should reuse the existing `AgentContextPruningConfig` contract unless maintainers explicitly prefer a separate boolean API.
