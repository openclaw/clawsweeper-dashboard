## Summary

Keep this open. Current main still lacks per-cron-job cache/context-pruning control, the prior implementation PR is closed and unmerged, and the related direct-script cron discussion does not replace per-job pruning for agent-run cron jobs.

## What This Changes

Add per-job cache or context-pruning control to cron job definitions so isolated cron runs can override prompt caching independently of global agent defaults.

## Best Possible Solution

Add a top-level `contextPruning?: AgentContextPruningConfig` cron job override using the existing `mode: "off" | "cache-ttl"` contract, and apply it only to that job's isolated agent run.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection gives high confidence that current main lacks the requested job-level field and still reads pruning from effective agent defaults.

## Solution Assessment

Yes. Reusing the existing `contextPruning` shape is the narrowest maintainable direction; a separate `disableCache` boolean would create a second API for only one pruning mode.

## Repair Work Prompt

Implement issue #8321 as an additive per-cron-job `contextPruning?: AgentContextPruningConfig` override. Reuse the existing `AgentContextPruningConfig` contract (`mode: "off" | "cache-ttl"`) rather than adding a separate `disableCache` boolean unless a maintainer explicitly asks. Thread the field through cron persisted/shared types, normalization/create/update/patch paths, gateway cron schemas, cron tool schemas, CLI add/edit, Control UI types/controllers/forms, docs, and changelog. In isolated cron execution, merge the job-level override into the effective agent defaults before the embedded runner receives config, while preserving global defaults for jobs without the field. Add regression coverage proving a job-level `mode: "off"` overrides global or per-agent `agents.defaults.contextPruning.mode: "cache-ttl"`. Do not implement direct script/no-LLM cron execution, elevated exec scoping, or unrelated pruning policy changes in this PR.
