## Summary

Keep open: current main still lacks the requested per-job cache/context-pruning override, and the prior implementation PR is closed unmerged; direct exec work is complementary rather than a replacement.

## What This Changes

Request adds per-cron-job cache/context-pruning control so isolated cron jobs can opt out of global prompt-cache pruning behavior.

## Best Possible Solution

Add a top-level cron contextPruning override using the existing AgentContextPruningConfig contract, round-trip it through all cron APIs and clients, and apply it only to that job's effective isolated-run agent config.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection on current main gives high-confidence evidence that the requested job-level cache/pruning field is absent and pruning remains driven by agents.defaults.contextPruning.

## Solution Assessment

Yes. Reusing the existing contextPruning object is the narrowest maintainable direction because it avoids a one-off disableCache boolean and preserves the current off/cache-ttl pruning contract.
