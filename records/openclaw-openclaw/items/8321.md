## Summary

Keep open: current main still lacks a per-cron-job cache/context-pruning override, and the related direct-exec cron work is complementary rather than a replacement for agent-run cron jobs.

## What This Changes

Requested an additive per-cron-job cache/context-pruning override so isolated scheduled jobs can disable prompt caching independently from global agent defaults.

## Best Possible Solution

Add a top-level cron contextPruning?: AgentContextPruningConfig override using the existing off/cache-ttl contract, round-trip it through cron APIs and clients, and apply it to the effective isolated-run agent config for that job only.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source inspection on current main gives high-confidence evidence that the requested job-level cache/pruning field is absent and pruning remains driven by agent defaults.

## Solution Assessment

Yes. Reusing the existing contextPruning object is the narrowest maintainable direction because it avoids a one-off disableCache boolean and preserves the existing off/cache-ttl pruning contract.
