## Summary

Keep open. Current main still lacks a per-cron-job cache or context-pruning override: the cron job contract, Gateway protocol, CLI/UI surfaces, and isolated runner only expose payload-level `lightContext`, while prompt-cache pruning is still controlled through `agents.defaults.contextPruning`; the prior implementation PR is closed unmerged and the direct-exec cron tracker is complementary rather than a replacement.

## What This Changes

The issue requests an optional per-cron-job cache/context-pruning override so isolated cron jobs can avoid prompt caching when they do not need LLM continuity.

## Best Possible Solution

Add an additive top-level `contextPruning?: AgentContextPruningConfig` cron job override using the existing `mode: "off" | "cache-ttl"` contract, thread it through cron types, normalization, persistence, Gateway schema, CLI, Control UI, docs, and merge it into `cfgWithAgentDefaults` before isolated cron runs.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection and the v2026.5.12 release check give high-confidence proof that current OpenClaw only supports payload-level `lightContext` plus defaults-level `contextPruning`, not per-job cache control.

## Solution Assessment

Yes directionally, but the maintainable form should reuse the existing `AgentContextPruningConfig` and `mode: "off" | "cache-ttl"` contract rather than inventing a separate `disableCache` boolean unless maintainers explicitly prefer that API.
