## Summary

Keep open: current main and the latest release support command cron jobs and payload-level `lightContext`, but they still do not provide per-job cache/context-pruning control for model-backed cron jobs.

## What This Changes

The issue requests per-cron-job control over prompt/context caching so simple isolated cron jobs can opt out of global context pruning behavior.

## Best Possible Solution

Keep this issue open for a maintainer-reviewed additive `contextPruning` cron-job override for agent-turn jobs, while treating command cron jobs as a complementary zero-LLM path for deterministic scripts.

## Reproduction Assessment

Yes, at source level. Current main exposes command cron payloads and `lightContext`, but the cron job contract, protocol schemas, storage codec, CLI, docs, and isolated runner still lack a per-job context-pruning override for model-backed cron jobs.

## Solution Assessment

Partly. The user-visible need is valid, but the best maintainable shape is likely a job-level `contextPruning` override using the existing `mode: "off" | "cache-ttl"` contract rather than a second boolean named `disableCache`.
