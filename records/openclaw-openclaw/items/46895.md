## What This Changes

The PR changes the Control UI slash-command executor so `/model status` and `/model list` return model information instead of patching the session model, with regression tests for those tokens and normal model selection.

## Best Possible Solution

Land or replace one narrow Control UI fix that routes `/model list` to the documented picker/list behavior, routes `/model status` to detailed status/auth/endpoint output, preserves normal model selection, and adds focused regression coverage.

## Reproduction Assessment

## Solution Assessment

No. Avoiding the bad `sessions.patch` call is necessary, but this PR collapses documented `list` and `status` modes into the bare summary response instead of preserving the shared command contract.
