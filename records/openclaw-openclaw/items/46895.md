## Summary

Keep open: this PR targets a real Control UI `/model` bug, but the patch is not the best mergeable fix because it collapses documented `list` and `status` modes into the bare summary response and still lacks contributor-run real behavior proof.

## What This Changes

The branch adds Control UI guards for `/model status` and `/model list` so they return model info instead of calling `sessions.patch`, with regression tests for those tokens and the normal model-setting path.

## Best Possible Solution

Revise or replace this with one narrow Control UI fix that routes `/model list` to the compact picker/list behavior, routes `/model status` to detailed provider/auth/endpoint output, preserves normal model selection, and adds focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `/model status` parses to `args: "status"`, Control UI dispatch passes it unchanged, and current `executeModel` calls `sessions.patch` for every non-empty argument.

## Solution Assessment

No. Avoiding the bad `sessions.patch` call is necessary, but this implementation routes both documented subcommands to the bare summary response instead of preserving the shared list/status behavior.
