## Summary

Keep open. The reported Control UI bug is still source-reproducible on current main, but this PR is not ready to merge because it routes documented `/model status` and `/model list` modes into the bare summary response and still lacks after-fix real behavior proof.

## What This Changes

The PR changes the Control UI `/model` executor so `status` and `list` use the local model-info branch and adds executor tests for those cases.

## Best Possible Solution

Land or replace one narrow Control UI fix that routes `/model list` and `/model status` to the documented informational behavior, preserves normal model selection, and includes focused regression coverage plus redacted real Control UI proof.

## Reproduction Assessment

Yes. Source inspection on current main shows `/model status` parses to `args: "status"`, Control UI dispatch passes that string unchanged, and `executeModel` sends every non-empty `/model` arg to `sessions.patch`; I did not run a live Control UI session.

## Solution Assessment

No. Avoiding the bad `sessions.patch` call is necessary, but this PR collapses `/model status` and `/model list` into the bare summary response instead of using the documented detailed status and list/picker behaviors.
