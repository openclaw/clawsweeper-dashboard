Keep open. ## What This Changes

The PR changes Control UI `/model` slash-command handling so `status` and `list` return model information instead of calling `sessions.patch`, with executor regression tests.

## Best Possible Solution

Land or replace one canonical Control UI fix that routes `/model list` and `/model status` to the documented flows before model mutation, preserves normal model-selection compatibility, and includes focused tests plus redacted UI proof.

## Reproduction Assessment

Yes, at source level. Current main parses `/model status` as `args: "status"`, Control UI preserves that argument, and `executeModel` sends every non-empty `/model` arg to `sessions.patch`; I did not run a live Control UI session in this read-only review.

## Solution Assessment

No, not as-is. Avoiding the bad `sessions.patch` call is necessary, but this PR collapses documented `list` and `status` modes into the bare summary branch and unconditionally reserves keywords that may also be valid model IDs.
