## Summary

Keep open. The PR addresses the bad `sessions.patch` path for `/model status` and `/model list`, but it collapses both documented modes into the bare `/model` summary and still lacks after-fix real Control UI proof.

## What This Changes

The PR changes the Control UI local `/model` slash command so `status` and `list` use the existing model-info branch instead of `sessions.patch`, with regression tests.

## Best Possible Solution

Align the Control UI local `/model` handling with the documented/shared directive contract: keep normal model selection, make `list` use the model list or picker path, make `status` show detailed status information, and cover those paths with focused tests.

## Solution Assessment
