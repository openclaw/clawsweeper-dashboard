## What This Changes

The PR changes Control UI slash-command handling so `/model status` and `/model list` return model information instead of calling `sessions.patch`, with regression tests for those paths.

## Best Possible Solution

Land or replace one canonical Control UI fix that routes `/model list` and `/model status` to their documented flows before model mutation, preserves normal model selection compatibility, and includes focused tests plus redacted UI proof.

## Reproduction Assessment

Yes. Source inspection on current main shows `/model status` parses to `args: "status"`, Control UI preserves it, and `executeModel` sends every non-empty `/model` arg to `sessions.patch`; I did not run a live Control UI session in this read-only review.

## Solution Assessment

No, not as-is. Avoiding the bad `sessions.patch` call is necessary, but this PR collapses documented `list` and `status` modes into the bare summary branch and unconditionally reserves keywords that may also be valid model IDs.
