## Summary

Keep open. The underlying Control UI bug is source-reproducible on current main, but this PR is not merge-ready because it routes `list` and `status` into the bare summary instead of the documented/shared modes, and the external PR has no after-fix real behavior proof.

## What This Changes

The PR changes the Control UI local `/model` slash-command executor so `status` and `list` route to the existing model-info response instead of `sessions.patch`, with regression tests.

## Best Possible Solution

Align the Control UI local `/model` handling with the shared directive/docs contract: preserve normal model selection, make `list` use the model-list/picker path, make `status` show detailed status information, and cover all paths with focused tests.

## Reproduction Assessment

Yes, by source inspection. On current main, `/model status` parses to command `model` with args `status`, app-chat forwards that arg unchanged, and `executeModel` calls `sessions.patch` for every non-empty arg.

## Solution Assessment

No. The PR prevents the bad patch call, but it does so by collapsing documented `list` and `status` behavior into the bare `/model` summary instead of matching the shared directive contract.
