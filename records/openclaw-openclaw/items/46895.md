## Summary

Keep open. The underlying Control UI bug still exists on current main, and this PR points at the right behavior, but its head is stale/dirty and routes the aliases through an outdated model-info path that bypasses current configured-catalog handling.

## What This Changes

The PR changes the Control UI slash-command executor and tests so `/model status` and `/model list` display model information instead of sending those arguments to `sessions.patch`.

## Best Possible Solution

Land one rebased Control UI fix that recognizes `/model status` and `/model list` before the set-model branch while reusing current main's `SlashCommandContext`, configured catalog loading, provider-qualified override resolution, and focused regression tests for both aliases.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: call `executeSlashCommand(client, "main", "model", "status", context)` on current main and the non-empty argument path reaches `sessions.patch` instead of model-info output.

## Solution Assessment

No. The behavioral direction is right, but this PR is not the best mergeable fix as-is because it routes aliases through an older `models.list {}` path instead of current main's configured-catalog model-info path.
