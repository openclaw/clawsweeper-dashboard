## Summary

Keep open. This PR targets a valid, source-reproducible Control UI bug, but it is not ready to merge because it collapses `/model status` into the compact `/model`/`/model list` output instead of the documented detailed status view, and the external contributor has not posted real after-fix behavior proof.

## What This Changes

The PR changes the Control UI slash-command executor so `/model status` and `/model list` return model information instead of calling `sessions.patch`, with regression tests for both keywords.

## Best Possible Solution

Land or replace one narrow Control UI fix that routes `/model list` to compact model browsing, routes `/model status` to detailed status-equivalent output, preserves normal `/model <ref>` selection, and adds focused regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows `/model status` is parsed as a non-empty `args` value and then sent to `sessions.patch` as a model value.

## Solution Assessment

No. The branch fixes the accidental patch call, but it treats `status` and `list` as the same compact info query even though current docs and the shared directive handler define different behavior.
