This PR updates the Control UI `/model` executor to treat `status` and `list` arguments as info-only requests and adds executor tests for those arguments plus the normal model-set path.

## Best Possible Solution

Land one refreshed Control UI fix that handles `/model status` and `/model list` before model mutation while preserving the documented status/list behaviors, normal model selection, focused regression tests, and redacted after-fix UI proof.

## Reproduction Assessment

## Solution Assessment

No. Avoiding the bad `sessions.patch` call is necessary, but the best fix is to preserve the documented `/model list` picker/list flow and `/model status` detailed provider/auth flow instead of treating both as bare model info.
