## Summary

Keep open. Current main still has the reported Control UI `/model status` and `/model list` failure path, but this PR is not merge-ready because it collapses the documented list/status behaviors into the bare summary path and lacks required after-fix real behavior proof.

## What This Changes

The PR adds `status` and `list` guards to the Control UI local `/model` executor and tests that those tokens return model info instead of calling `sessions.patch`.

## Best Possible Solution

Land or replace one narrow Control UI fix that routes `/model list` to the compact list/picker behavior, routes `/model status` to detailed provider/auth/endpoint output, preserves normal model selection, and adds focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `/model status` parses to `args: "status"`, Control UI dispatch passes it unchanged, and current `executeModel` calls `sessions.patch` for every non-empty argument.

## Solution Assessment

No. Avoiding the bad `sessions.patch` call is necessary, but this implementation routes both documented subcommands to the bare summary response instead of preserving the shared list/status handlers.
