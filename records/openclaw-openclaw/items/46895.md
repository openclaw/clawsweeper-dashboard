## What This Changes

The PR changes the Control UI `/model` slash-command executor so `status` and `list` avoid `sessions.patch`, and adds executor tests for those tokens.

## Best Possible Solution

Land or replace one narrow Control UI fix that routes `/model list` and `/model status` through the documented informational behavior, preserves normal model selection, and adds focused regression coverage plus real Control UI proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `/model status` parses to `args: "status"`, Control UI dispatch passes it unchanged, and current main sends every non-empty `/model` argument to `sessions.patch`.

## Solution Assessment

No. Avoiding the bad `sessions.patch` call is necessary, but this guard sends `/model status` and `/model list` into the bare summary branch instead of the documented detailed status and list/picker behaviors.
