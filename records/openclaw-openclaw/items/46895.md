## Summary

Keep open. The current-main bug is valid, but this PR is not merge-ready because it collapses the documented `/model list` and `/model status` paths into the bare summary response and lacks required after-fix real behavior proof.

## What This Changes

The PR changes the Control UI local `/model` executor to treat `status` and `list` as informational arguments and adds regression tests for those arguments.

## Best Possible Solution

Land or replace one narrow Control UI fix that routes `/model list` to the compact list/picker behavior, routes `/model status` to detailed status output, preserves normal model selection, and adds focused regression coverage.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: `/model status` parses to `args: "status"`, Control UI dispatch passes it unchanged, and current `executeModel` calls `sessions.patch` for every non-empty argument. I did not run a live UI flow in this read-only review.

## Solution Assessment

No. Avoiding the bad patch call is the right direction, but this PR returns the bare `/model` summary for both documented subcommands instead of preserving the shared list/status behaviors.
