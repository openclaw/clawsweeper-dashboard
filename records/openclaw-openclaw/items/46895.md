## What This Changes

The branch changes the Control UI local `/model` slash command so `status` and `list` avoid `sessions.patch` and adds regression tests for those arguments.

## Best Possible Solution

Land or replace one narrow Control UI fix that routes `/model list` to the documented list/picker behavior, routes `/model status` to detailed status output, preserves normal model selection, and includes focused tests plus real Control UI proof.

## Reproduction Assessment

Yes, at source level. Current main parses `/model status` into `args: "status"`, passes it to `executeModel`, and the executor calls `sessions.patch` for every non-empty argument.

## Solution Assessment

No. This patch prevents the bad patch call, but it collapses documented `list` and `status` modes into the bare `/model` summary instead of preserving the shared command contract.
