## Summary

Keep open: current main still has a source-reproducible Control UI `/model status` and `/model list` bug, but this PR only routes those tokens to the bare `/model` summary instead of the documented list/status responses, and the external contributor has not provided real behavior proof.

## What This Changes

The PR changes the Control UI `/model` slash-command executor so `status` and `list` avoid `sessions.patch`, and adds executor tests for those tokens.

## Best Possible Solution

Land or replace one narrow Control UI fix that routes `/model list` and `/model status` through the documented informational behavior, preserves normal model selection, and adds focused regression coverage plus real Control UI proof.

## Reproduction Assessment

Yes. Source inspection on current main shows `/model status` parses to `args: "status"`, Control UI dispatch passes it unchanged, and `executeModel` calls `sessions.patch` for every non-empty `/model` argument.

## Solution Assessment

No. Avoiding the bad `sessions.patch` call is necessary, but this PR should preserve the documented `list` picker/list behavior and the detailed `status` provider/auth/endpoint output instead of returning the bare summary for both.
