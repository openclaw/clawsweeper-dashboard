## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Keep the callback session through token exchange:** `src/agents/mcp-oauth-provider.ts:124-126`
  - body: The SDK's `finishAuth` calls `auth()`/`fetchToken()` after `waitForAuthorizationCode()` returns, and `fetchToken()` requires `provider.redirectUrl` for authorization-code exchange. This `finally` clears `pendingAuthorizationCode` before the caller invokes `finishAuth`, so first-time OAuth can fail after receiving a valid callback.
  - confidence: 0.91
- **[P1] Create a callback session for cached OAuth clients:** `src/agents/mcp-oauth-provider.ts:155-159`
  - body: When persisted `clientInfo` exists, `clientInformation()` skips `ensureCallbackSession()`, leaving `redirectUrl` undefined. The SDK checks `!provider.redirectUrl` before loading or refreshing tokens, so expired or missing cached tokens can take the non-interactive path instead of opening the authorization browser.
  - confidence: 0.88
- **[P1] Reconnect with fresh MCP client and transport after OAuth:** `src/agents/pi-bundle-mcp-runtime.ts:227`
  - body: After `finishAuth`, this retries `connectWithTimeout` on the same client and transport. The SDK sets protocol transport before initialization and the HTTP/SSE transports retain started state, so the retry can fail as already connected or already started instead of completing the post-auth connection.
  - confidence: 0.86
- **[P1] Preserve redirect-scrubbing fetch for streamable HTTP:** `src/agents/mcp-transport.ts:105-108`
  - body: Current main passes `fetchStreamableHttpWithRedirectScrub` to streamable HTTP transports so configured headers are stripped on cross-origin redirects. This replacement omits the custom fetch, reopening the header-leak boundary for redirected MCP requests.
  - confidence: 0.84
- **[P1] Write MCP OAuth tokens with private permissions:** `src/agents/mcp-oauth-provider.ts:77-82`
  - body: Access and refresh tokens are written using default `mkdirSync` and `writeFileSync` modes, so a permissive umask can leave `~/.openclaw/mcp-auth/*.json` readable by other local users. Token directories and files should be created or chmodded with restrictive modes.
  - confidence: 0.87
- **[P1] Key OAuth state by endpoint identity, not server name:** `src/agents/mcp-oauth-provider.ts:61-62`
  - body: The persisted state path only uses the display server key, while OpenClaw MCP server keys can be reused across workspaces. Reusing a key like `github` or `figma` for different URLs can cause tokens and client info from one endpoint to be sent to another.
  - confidence: 0.83

## Security Review
