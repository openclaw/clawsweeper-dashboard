Full review comments:

- **[P1] Preserve the streamable HTTP redirect-scrub fetch:** `src/agents/mcp-transport.ts:105-108`
  - body: Current main passes `fetchStreamableHttpWithRedirectScrub` into `StreamableHTTPClientTransport` so configured headers are scrubbed on cross-origin redirects. This branch constructs the OAuth streamable transport without that custom fetch, so resolving the stale branch literally would downgrade the existing header/token redirect boundary.
  - confidence: 0.91
- **[P1] Scope OAuth state by server identity:** `src/agents/pi-bundle-mcp-runtime.ts:173-176`
  - body: The new persistence path is keyed only by `serverName`, and the runtime calls `loadMcpOAuthState(serverName)` / `saveMcpOAuthState(serverName, ...)`. Reusing a name like `github` or `figma` in another workspace or for another endpoint can share tokens/client registration state with the wrong server, so the key needs to include a stable endpoint/workspace identity.
  - confidence: 0.9
- **[P2] Write OAuth token files with private permissions:** `src/agents/mcp-oauth-provider.ts:79-82`
  - body: `saveMcpOAuthState` creates the directory and token JSON with default filesystem modes. On common umask settings, access and refresh tokens can become readable by other local users; create/chmod the directory as `0700` and token files as `0600`.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The patch introduces bearer-token persistence and localhost OAuth callback handling with concrete credential-isolation, filesystem-permission, and redirect-boundary concerns.

Concerns:

- **[high] Token state is scoped only by display name:** `src/agents/pi-bundle-mcp-runtime.ts:174`
  - body: Persisting OAuth state by `serverName` can reuse tokens/client registration across different endpoints or workspaces with the same key, which risks sending credentials to the wrong MCP server.
  - confidence: 0.9
- **[medium] Token files rely on default filesystem modes:** `src/agents/mcp-oauth-provider.ts:82`
  - body: Access and refresh tokens are written with default `mkdirSync`/`writeFileSync` modes, so local file readability depends on umask instead of OpenClaw's credential-file boundary.
  - confidence: 0.88
- **[high] Branch omits current redirect header scrubbing:** `src/agents/mcp-transport.ts:105`
  - body: The branch's streamable HTTP transport no longer wires the current custom fetch that drops unsafe headers across cross-origin redirects, which can expose Authorization or configured MCP headers if not preserved during rebase.
  - confidence: 0.9
