## What This Changes

Adds opt-in `auth: "oauth"` for remote MCP HTTP/SSE servers, an MCP OAuth provider and localhost callback server, token persistence under `~/.openclaw/mcp-auth`, SDK `authProvider` transport wiring, UnauthorizedError reconnect handling, and focused tests.

## Best Possible Solution

Land a rebased, endpoint-scoped MCP OAuth implementation that preserves current transport hardening, keeps redirect state through token exchange and reauth, writes credential files privately, documents the config contract, and is proven against a real OAuth MCP server.

## Reproduction Assessment

Yes for the source-level PR defects: MCP SDK v1.29.0 requires a live `redirectUrl` for authorization-code exchange, while this branch clears or never initializes it in key paths. No live OAuth MCP end-to-end reproduction was provided by the contributor.

## Solution Assessment

No, not as currently proposed. SDK `authProvider` wiring is the right general direction, but this branch needs rebase, auth lifecycle fixes, endpoint-scoped private storage, docs/contract review, and real behavior proof before it is the maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Keep redirect URL alive through token exchange:** `src/agents/mcp-oauth-provider.ts:124-126`
  - body: `pendingAuthorizationCode.promise` clears `pendingAuthorizationCode` as soon as the callback code resolves. Runtime then calls `finishAuth(code)`, and the MCP SDK's authorization-code token exchange requires `provider.redirectUrl`; with this cleared, first-time OAuth fails before tokens are saved.
  - confidence: 0.92
- **[P1] Prepare the callback session for cached-client reauth:** `src/agents/mcp-oauth-provider.ts:155-159`
  - body: When persisted `clientInfo` exists, `clientInformation()` returns without creating a callback session, leaving `redirectUrl` undefined. The MCP SDK treats that as a non-interactive flow, so expired or invalid cached tokens cannot start the browser authorization path again.
  - confidence: 0.9
- **[P1] Scope persisted OAuth state to the server endpoint:** `src/agents/pi-bundle-mcp-runtime.ts:174-176`
  - body: The credential store key uses only `serverName`, so a `github` or `figma` entry in two workspaces or for two different URLs can share the same client info and bearer tokens. Key the state by stable endpoint identity, not display name alone.
  - confidence: 0.87
- **[P2] Write token state with private permissions:** `src/agents/mcp-oauth-provider.ts:80-82`
  - body: This creates `~/.openclaw/mcp-auth` and writes access/refresh token JSON with default permissions. On common umask settings the file can be readable by other local users, so the directory and file should be explicitly restricted.
  - confidence: 0.86
