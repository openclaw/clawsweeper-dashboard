## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve streamable redirect scrubbing:** `src/agents/mcp-transport.ts:105-108`
  - body: Current main passes `fetchStreamableHttpWithRedirectScrub` into `StreamableHTTPClientTransport`, but this constructor drops that fetch hook while adding OAuth. That regresses the redirect header-scrubbing boundary for configured headers or bearer tokens on cross-origin redirects.
  - confidence: 0.93
- **[P1] Keep the callback session through finishAuth:** `src/agents/mcp-oauth-provider.ts:122-126`
  - body: `waitForAuthorizationCode()` awaits the promise that clears `pendingAuthorizationCode` in `finally`, so `provider.redirectUrl` is gone before `finishAuth(code)` exchanges the token. The MCP SDK requires `redirectUrl` for the default authorization-code token request, so first-time OAuth can fail after a successful callback.
  - confidence: 0.94
- **[P1] Create a callback session for cached clients:** `src/agents/mcp-oauth-provider.ts:155-159`
  - body: When persisted `clientInfo` exists, `clientInformation()` returns without preparing a callback session. If tokens are missing or expired, the SDK sees no `redirectUrl`, treats the flow as non-interactive, and cannot restart the browser authorization flow.
  - confidence: 0.9
- **[P1] Reconnect with a fresh SSE transport after auth:** `src/agents/pi-bundle-mcp-runtime.ts:224-227`
  - body: After an OAuth 401, the retry reuses the same SSE transport instance. In the MCP SDK, `SSEClientTransport.close()` closes the EventSource but does not clear `_eventSource`, so a second `start()` on the same transport can throw `SSEClientTransport already started` instead of completing first-time OAuth.
  - confidence: 0.88
- **[P1] Scope OAuth state by endpoint identity:** `src/agents/mcp-oauth-provider.ts:61-62`
  - body: The token file path is derived only from `serverName`, so two workspaces or configs using the same key for different MCP URLs can share and overwrite OAuth tokens. Key the persisted state by a stable endpoint/workspace identity instead of the display name alone.
  - confidence: 0.89
- **[P2] Write token state with private permissions:** `src/agents/mcp-oauth-provider.ts:77-82`
  - body: `mkdirSync` and `writeFileSync` use default permissions, so on common umasks the access and refresh token JSON can be readable by other local users. Create the directory and file with restrictive modes, and repair existing files if needed.
  - confidence: 0.86
