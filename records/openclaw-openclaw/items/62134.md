Full review comments:

- **[P1] Preserve streamable redirect scrubbing:** `src/agents/mcp-transport.ts:105-108`
  - body: Current main sends streamable HTTP requests through `fetchStreamableHttpWithRedirectScrub` so configured secrets are not forwarded across cross-origin redirects. This constructor adds `authProvider` but drops that `fetch` hook, which would regress an existing security boundary for static headers and OAuth bearer traffic.
  - confidence: 0.93
- **[P1] Keep the callback session through token exchange:** `src/agents/mcp-oauth-provider.ts:122-126`
  - body: `waitForAuthorizationCode()` awaits a promise whose `finally` clears `pendingAuthorizationCode`, so `provider.redirectUrl` can be undefined by the time `finishAuth(code)` calls the SDK token exchange. The MCP SDK requires `redirectUrl` for authorization-code exchange, so first-time OAuth can fail after a successful callback.
  - confidence: 0.88
- **[P1] Namespace OAuth state by server identity:** `src/agents/pi-bundle-mcp-runtime.ts:173-176`
  - body: Persisting tokens only under `serverName` lets two workspaces or configs that reuse a key like `figma` share one token file even when they point at different endpoints. Key the store by a stable server identity, at least the normalized URL plus any needed scope, so one server's token cannot be sent to another.
  - confidence: 0.9
- **[P2] Write token files with private permissions:** `src/agents/mcp-oauth-provider.ts:77-82`
  - body: `mkdirSync` and `writeFileSync` use the process umask, so access and refresh tokens can be written world-readable on common setups. Create the auth directory with `0o700` and state files with `0o600`, and repair existing modes where practical.
  - confidence: 0.86
