## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P1] Keep the callback session through finishAuth:** `src/agents/mcp-oauth-provider.ts:124-126`
  - body: `waitForAuthorizationCode()` awaits `pendingAuthorizationCode.promise`, whose `finally` clears `pendingAuthorizationCode` before the runtime calls `finishAuth(code)`. The MCP SDK then sees `provider.redirectUrl` as undefined during the authorization-code token exchange and throws `redirectUrl is required for authorization_code flow`, so first-time OAuth cannot complete after the callback.
  - confidence: 0.96
- **[P1] Prepare a redirect URL for cached clients:** `src/agents/mcp-oauth-provider.ts:155-159`
  - body: When persisted `clientInfo` exists, `clientInformation()` returns without `ensureCallbackSession()`, leaving `redirectUrl` undefined. After tokens expire or are absent, the SDK treats this as a non-interactive flow and tries token exchange without an authorization code instead of opening the browser, so previously registered servers cannot re-authorize.
  - confidence: 0.94
- **[P1] Preserve streamable HTTP redirect scrubbing:** `src/agents/mcp-transport.ts:105-108`
  - body: Current main passes a redirect-scrubbing fetch to `StreamableHTTPClientTransport` so configured auth headers are stripped on cross-origin redirects. This constructor drops that fetch while adding `authProvider`, which would let static headers or OAuth bearer tokens follow default redirect behavior and regress the current security hardening.
  - confidence: 0.93
- **[P1] Scope OAuth state to the endpoint identity:** `src/agents/pi-bundle-mcp-runtime.ts:175-176`
  - body: The runtime persists OAuth state by `serverName` only, so two workspaces or configs that both use a key like `figma` or `github` but point at different URLs will share tokens, client info, verifier, and discovery state. Key this store by a stable server identity such as normalized URL plus workspace/agent scope to avoid sending one server's bearer token to another.
  - confidence: 0.9
- **[P2] Write token state with private permissions:** `src/agents/mcp-oauth-provider.ts:80-82`
  - body: `mkdirSync` and `writeFileSync` rely on the process umask, so on common `022` setups `~/.openclaw/mcp-auth/*.json` can be readable by other local users. Access and refresh tokens should be written under an explicit private directory/file mode, with existing files repaired when possible.
  - confidence: 0.88
