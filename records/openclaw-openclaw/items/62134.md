## Summary

Keep open. Current main still lacks remote MCP OAuth support and the PR addresses a real feature gap, but this branch is not mergeable because the OAuth callback, cached-client reauth, and reconnect paths are source-broken, token persistence has security issues, and the contributor has not provided real end-to-end OAuth proof.

## What This Changes

The PR adds opt-in `auth: "oauth"` for remote MCP servers, wires an OAuth provider into HTTP/SSE transports, adds a loopback callback/token persistence layer, and adds focused tests.

## Best Possible Solution

Merge a rebased OAuth implementation only after fixing the SDK lifecycle defects, endpoint-scoping and hardening token storage, and attaching redacted real OAuth MCP proof.

## Reproduction Assessment

Yes for the blocking PR defects: the failing paths are source-reproducible from the PR head plus the MCP SDK 1.29.0 auth/transport contract. No live end-to-end OAuth MCP reproduction or after-fix proof was provided by the contributor.

## Solution Assessment

No. Passing the SDK `authProvider` is the right broad direction, but this branch needs a safer OAuth lifecycle, fresh client/transport reconnect handling, endpoint-scoped private credential storage, and real behavior proof before it is the best mergeable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Keep the callback session through finishAuth:** `src/agents/mcp-oauth-provider.ts:124-126`
  - body: `waitForAuthorizationCode()` awaits a promise whose `finally` clears `pendingAuthorizationCode`, so `provider.redirectUrl` is undefined by the time `finishAuth(code)` calls the SDK token exchange. SDK 1.29.0 requires `redirectUrl` for authorization-code token exchange, so first-time OAuth can fail before tokens save.
  - confidence: 0.95
- **[P1] Create callback state for cached client reauth:** `src/agents/mcp-oauth-provider.ts:155-159`
  - body: When persisted `clientInfo` exists, `clientInformation()` returns without creating a callback session, leaving `redirectUrl` undefined. The SDK treats that as a non-interactive flow before token refresh and tries token fetch without an authorization code, so reauthorization after missing or expired tokens cannot start the browser flow.
  - confidence: 0.9
- **[P1] Reconnect with a fresh MCP client and transport:** `src/agents/pi-bundle-mcp-runtime.ts:227`
  - body: After `finishAuth`, this retries `connectWithTimeout` with the same SDK client and transport. The SDK attaches transports before initialization and HTTP/SSE transports retain started state after failed auth, so the retry can fail as already connected or already started even with valid tokens.
  - confidence: 0.9
- **[P1] Namespace OAuth state by endpoint identity:** `src/agents/mcp-oauth-provider.ts:61-62`
  - body: The persisted OAuth state path is based only on `serverName`, so different workspaces or remote MCP URLs using the same key can share one token file. That can send credentials minted for one MCP server to another before auth has a chance to correct it.
  - confidence: 0.88
- **[P2] Write OAuth state files with restrictive permissions:** `src/agents/mcp-oauth-provider.ts:79-82`
  - body: This writes access and refresh tokens with default directory and file modes, which depend on the process umask. Create the directory and token file with restrictive permissions and repair loose existing modes before saving credential state.
  - confidence: 0.9

## Security Review

Status: needs_attention

Summary: Needs attention because the diff persists remote MCP bearer tokens without endpoint-scoped keys or explicit private filesystem modes.

Concerns:

- **[high] OAuth tokens are keyed only by display name:** `src/agents/mcp-oauth-provider.ts:61`
  - body: A global token file derived only from `serverName` can mix credentials across workspaces or different MCP URLs that reuse a key like `github`, risking bearer tokens being sent to the wrong remote server.
  - confidence: 0.88
- **[medium] OAuth token files can be too broadly readable:** `src/agents/mcp-oauth-provider.ts:82`
  - body: The new token persistence path writes access and refresh tokens with default mode bits; on permissive umasks, other local users may be able to read the credential file.
  - confidence: 0.9
