## Summary

Keep open: the diagnostic fix addresses a source-reproducible current-main bug, but this contributor PR has a protected security label, an unverified OpenGrep binary execution change, stale recovery docs, and incomplete real behavior proof for the non-diagnostic changes.

## What This Changes

This PR clears `noop/no_active_work` stuck-session diagnostic state, adds regression coverage and changelog entries, changes `models.json` cost lookup to use the active/default agent dir, and rewires OpenGrep workflows to download the release asset directly.

## Best Possible Solution

Land the diagnostic fix with regression coverage and docs, and split, remove, or harden the usage-cost and OpenGrep workflow changes with focused proof before merge.

## Reproduction Assessment

Yes, from source inspection. Current main can return `noop/no_active_work`, but only `aborted` and `released` are treated as mutating, so the coordinator returns before clearing stale diagnostic state.

## Solution Assessment

No, not as currently packaged. The diagnostic code path is a narrow maintainable fix, but the bundled OpenGrep installer and usage-cost changes need hardening, docs alignment, and their own real behavior proof.
