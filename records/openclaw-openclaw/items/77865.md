## Summary

Keep open: the diagnostic fix is source-backed and has terminal proof, but the PR also changes a security-sensitive OpenGrep workflow without verifying the downloaded executable and adds usage-cost behavior without after-fix real behavior proof.

## What This Changes

The PR clears diagnostic queue state for `noop/no_active_work`, adds regression coverage and changelog entries, switches OpenGrep workflows to direct release-asset downloads, and makes usage-cost lookup read `models.json` from the active/default agent directory.

## Best Possible Solution

Land a narrowed patch that keeps the diagnostic and pricing fixes with matching proof/tests, hardens OpenGrep installation with digest or cosign verification, and updates the diagnostic liveness docs after maintainer/security review.

## Reproduction Assessment

Yes. Source inspection on current main shows `noop/no_active_work` is returned when no active work remains, but the recovery predicate rejects `noop`, so the coordinator returns before clearing diagnostic state; the PR body also reports a before-fix failing regression test.

## Solution Assessment

No, not as packaged. The diagnostic state-clearing change is the narrow fix, but the CI installer must retain asset integrity verification and the added pricing behavior needs its own proof before merge.
