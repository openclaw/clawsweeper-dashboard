## Summary

Keep open: current main still has the diagnostic state-clearing bug and the PR's diagnostic fix is plausible, but the security-labeled external PR still has supply-chain, docs, and proof blockers before merge.

## What This Changes

This PR changes stuck-session diagnostic recovery to clear queued state for `noop/no_active_work`, adds regression coverage and changelog entries, adjusts `models.json` cost lookup for active/default agent dirs, and rewires OpenGrep workflows to download the pinned release asset directly.

## Best Possible Solution

Land the diagnostic fix with matching docs and regression coverage, and either split the usage-cost and OpenGrep workflow changes into separately proven PRs or harden them with focused proof and maintainer/security review.

## Reproduction Assessment

Yes, from source inspection. Current main can return `noop/no_active_work`, but the coordinator returns before clearing diagnostic state because only `aborted` and `released` are treated as mutating outcomes.

## Solution Assessment

No, not as currently packaged. The diagnostic code path is narrow, but the PR also adds an unverified CI binary download and a usage-cost behavior change without complete real behavior proof or docs alignment.
