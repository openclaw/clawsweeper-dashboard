## Summary

Keep open: the diagnostic fix is narrow and backed by source/proof, but the PR also changes security-scanning workflow installation to execute an OpenGrep release asset without signature or digest verification, and the protected security label requires maintainer handling before merge.

## What This Changes

The PR treats `noop/no_active_work` diagnostic recovery as state-clearing, adds regression coverage and a changelog entry, and switches OpenGrep workflows from a pinned installer script to direct release asset download.

## Best Possible Solution

Merge the diagnostic state-clearing fix after restoring verifiable OpenGrep asset integrity, such as cosign signature verification or a pinned digest, in both OpenGrep workflows.

## Reproduction Assessment

Yes. Source inspection on current main shows `noop/no_active_work` is currently non-mutating, so the coordinator emits completion and returns before clearing queued diagnostic state; the PR body also reports a before-fix reproduction with the new regression test.

## Solution Assessment

No, not as currently packaged. The diagnostic code path is the narrow fix, but the OpenGrep workflow repair should preserve signature or digest verification instead of replacing the pinned installer with an unverified executable download.
