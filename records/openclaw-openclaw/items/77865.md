## Summary

This PR should stay open for maintainer/security handling: the diagnostic fix is source-backed, but the OpenGrep workflow change still executes a downloaded release asset without signature or digest verification, and the later usage-cost change lacks after-fix real behavior proof.

## What This Changes

The PR changes stuck-session diagnostic recovery to clear queued state on `noop/no_active_work`, adds regression coverage and changelog entries, switches OpenGrep workflows to direct release-asset downloads, and changes usage-cost lookup to read `models.json` from the active/default agent directory.

## Best Possible Solution

Keep the diagnostic and pricing fixes narrow, split or harden the OpenGrep installer change with signature/digest verification, and add real behavior proof for every user-visible behavior before maintainer security review.

## Reproduction Assessment

Yes. Current main's recovery predicate only treats `aborted` and `released` as mutating while the runtime returns `noop/no_active_work` when no active work remains, so source inspection shows the stale diagnostic state path; the PR body also reports a before-fix failing regression test.

## Solution Assessment

No, not as packaged. The diagnostic code change is the narrow fix, but the OpenGrep workflow should retain verifiable asset integrity and the later usage-cost change needs real behavior proof before merge.
