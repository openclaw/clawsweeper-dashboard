## Summary

Keep open. Current main still has the real media-understanding provider/model telemetry available, but the assistant-visible inbound body is formatted without that telemetry, so the reported guessed-model failure remains a narrow unresolved core gap.

## What This Changes

Expose the actual current-turn media-understanding provider/model in the inbound body so assistant answers do not guess which model analyzed media.

## Best Possible Solution

Render an `Analysis model:` line from real successful `MediaUnderstandingOutput` telemetry in the formatter/apply path, with explicit handling for missing model values and synthetic placeholders.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: pass an output with `provider` and `model` into `formatMediaUnderstandingBody`, then observe that the returned body lacks `Analysis model:` before `applyMediaUnderstanding` promotes it to `BodyForAgent`. I did not run tests because this review is read-only.

## Solution Assessment

Yes. The narrow maintainable fix is to use the existing typed media-understanding output telemetry while formatting the current turn, without changing provider selection, config, or fallback policy.
