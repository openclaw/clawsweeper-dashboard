## Summary

Keep open. Current main still has media-understanding provider/model telemetry, but the assistant-visible media body is formatted without that telemetry, so the reporter’s guessed-model scenario remains a narrow unresolved formatter/apply-path gap.

## What This Changes

Expose actual media-understanding provider/model telemetry in the assistant-visible inbound media body so agents do not infer the analysis model.

## Best Possible Solution

Render `Analysis model:` from real successful `MediaUnderstandingOutput.provider` and `model` values in the media body, skip or explicitly handle synthetic placeholders and missing model values, and cover the behavior with focused tests plus a changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level path exists: pass an output with `provider` and `model` into `formatMediaUnderstandingBody`, then observe that the returned body lacks `Analysis model:` and is promoted to `BodyForAgent` by `applyMediaUnderstanding`.

## Solution Assessment

Yes. The narrowest maintainable fix is in the media-understanding formatter/apply path using existing typed telemetry; changing conversation-model fallback behavior would not expose the actual current-turn media analyzer.

## Repair Work Prompt

Fix this issue by adding actual media-understanding analysis telemetry to the assistant-visible inbound media body. Update `formatMediaUnderstandingBody` / nearby helpers so real successful media outputs include an `Analysis model:` line derived only from that output's `provider` and `model` values, for example `ollama/qwen3.5:397b-cloud`; handle missing model values explicitly without guessing. Do not use conversation model, session fallback, stale state, or configured defaults as a substitute for current-turn output telemetry. Do not present synthetic skipped-audio placeholder outputs such as `openclaw/synthetic-empty-audio` / `synthetic-empty-audio` as real analysis models. Add focused regression coverage in formatter and apply-path tests, update docs if the visible body format is documented, and add a user-facing `CHANGELOG.md` fix entry. Do not add a config option or broaden media selection behavior.
