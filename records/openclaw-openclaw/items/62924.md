## Summary

Keep open. Current main still preserves media-understanding provider/model telemetry internally but drops it when formatting the assistant-visible body, so the reporter’s guessed-model failure remains a valid narrow code gap.

## What This Changes

Expose actual current-turn media-understanding provider/model telemetry in the assistant-visible inbound media body so agents do not guess which media model analyzed attachments.

## Best Possible Solution

Render an `Analysis model:` line from real successful `MediaUnderstandingOutput` telemetry in the media-understanding body, cover provider-only and synthetic-placeholder cases, and add focused formatter/apply tests plus a user-facing changelog entry.

Yes. Source-level reproduction is high-confidence: pass a `MediaUnderstandingOutput` with `provider` and `model` into `formatMediaUnderstandingBody`, then observe the returned body lacks `Analysis model:` and is promoted to `BodyForAgent` by `applyMediaUnderstanding`.

## Solution Assessment

Yes. The narrowest maintainable solution is in the media-understanding formatter/apply path using existing typed telemetry, with explicit handling for missing model values and synthetic placeholders.
