## Summary

Keep open. Current main and the latest shipped release still preserve media-understanding provider/model telemetry internally but omit it from the assistant-visible inbound body, so the reporter's guessed-model failure remains source-reproducible and narrowly fixable.

## What This Changes

Expose the actual media-understanding provider/model used for the current turn in the inbound media body so assistants do not report guessed analysis models.

## Best Possible Solution

Render an `Analysis model:` line from real current-turn media-understanding output telemetry in the formatter/apply path, with explicit handling for missing models and synthetic placeholders.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: media outputs carry provider/model, `formatMediaUnderstandingBody` drops that telemetry, and `applyMediaUnderstanding` forces the formatted body into `BodyForAgent`.

## Solution Assessment

Yes. The narrow maintainable fix is to render existing current-turn telemetry in the media formatter/apply path while avoiding guessed fallback state and synthetic placeholder telemetry.
