## Summary

Keep open. Current main still carries media-understanding provider/model telemetry internally, but the formatter drops it before `BodyForAgent`, so the reported guessed-model behavior remains a narrow unresolved bug.

## What This Changes

Expose the actual current-turn media-understanding provider/model in the assistant-visible inbound media body so the assistant does not guess which model analyzed media.

## Best Possible Solution

Render `Analysis model:` from real successful `MediaUnderstandingOutput` telemetry in the media-understanding body, explicitly handling provider-only and synthetic-placeholder cases with focused tests and a changelog entry.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: pass a `MediaUnderstandingOutput` with `provider` and `model` into `formatMediaUnderstandingBody`, then observe the returned body lacks `Analysis model:` and is promoted to `BodyForAgent` by `applyMediaUnderstanding`.

## Solution Assessment

Yes. The narrowest maintainable solution is to format existing current-turn media telemetry in `src/media-understanding/format.ts` while avoiding synthetic placeholders and any guessed fallback.
