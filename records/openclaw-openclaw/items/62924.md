## Summary

Keep open. Current `main` and the latest release still carry media-understanding provider/model telemetry internally, but the assistant-visible inbound body is formatted without that telemetry, so the guessed-model failure remains source-reproducible and narrowly fixable.

## What This Changes

Expose the actual media-understanding provider/model used for each inbound media description or transcript to the assistant-visible turn body.

## Best Possible Solution

Render an `Analysis model:` line from real current-turn `MediaUnderstandingOutput.provider` and `model` values, with explicit handling for missing model values and synthetic placeholders.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: outputs carry provider/model, the formatter drops those fields, and the apply path forces the formatted text into `BodyForAgent`. I did not rerun the live Discord scenario.

## Solution Assessment

Yes. The narrow maintainable fix is to render existing current-turn telemetry from the media-understanding output contract, not infer it from conversation defaults or broader provider-routing state.
