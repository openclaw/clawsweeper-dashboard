## Summary

Keep open. Current main and v2026.5.12 still carry media-understanding provider/model telemetry internally, but the assistant-visible media body is formatted without that telemetry, so the guessed-model failure remains source-reproducible and narrowly fixable.

## What This Changes

Expose the actual current-turn media-understanding provider/model in the inbound assistant body so the assistant does not guess which model analyzed media.

## Best Possible Solution

Render an `Analysis model:` line from real current-turn `MediaUnderstandingOutput` provider/model telemetry in the formatter/apply path, while suppressing synthetic placeholders and avoiding any guessed fallback state.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: media outputs carry provider/model, `formatMediaUnderstandingBody` drops that telemetry, and `applyMediaUnderstanding` forces the formatted body into `BodyForAgent`.

## Solution Assessment

Yes. The narrow maintainable fix is to render existing current-turn telemetry without falling back to the conversation model, stale session state, defaults, or guesses; https://github.com/openclaw/openclaw/issues/81525 is adjacent but separate.
