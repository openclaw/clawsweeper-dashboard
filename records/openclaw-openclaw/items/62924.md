## Summary

Keep open. Current main and the latest release still carry media-understanding provider/model telemetry internally but omit it from the assistant-visible inbound body, so the reporter's guessed-model failure remains source-reproducible and narrowly fixable.

## What This Changes

The issue requests that inbound media-understanding bodies expose the actual provider/model used for image, audio, or video analysis so assistants do not guess.

## Best Possible Solution

Render an `Analysis model:` line from real `MediaUnderstandingOutput.provider` and `model` values in the current-turn formatter/apply path, with explicit handling for missing model values and synthetic placeholders.

## Reproduction Assessment

Yes for a source-level current-main path: outputs carry provider/model, the formatter drops those fields, and the apply path forces the formatted text into `BodyForAgent`. I did not rerun the live Discord scenario.

## Solution Assessment

Yes. The narrow maintainable fix is to render existing current-turn telemetry from the media-understanding output contract, not infer it from conversation defaults or broader provider-routing state.
