## Summary

Keep open. Current main and the latest release still carry media-understanding provider/model telemetry internally, but the assistant-visible inbound body is formatted from transcript/description text only, so the reported guessed-model failure remains source-reproducible and narrowly fixable.

## What This Changes

The issue requests exposing the actual media-understanding provider/model in the assistant-visible inbound body so the assistant does not guess which model analyzed media.

## Best Possible Solution

Render an `Analysis model:` line from real `MediaUnderstandingOutput.provider` and `model` values in the media-understanding formatter/apply path, with explicit handling for missing model values and synthetic placeholders.

## Reproduction Assessment

Yes for a source-level current-main path: outputs carry provider/model, the formatter drops those fields, and the apply path forces the formatted text into `BodyForAgent`. I did not rerun the live Discord scenario.

## Solution Assessment

Yes. The narrow maintainable fix is to render existing current-turn telemetry from the media-understanding output contract, not infer it from conversation defaults or broader provider-routing state.
