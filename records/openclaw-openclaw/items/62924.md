## Summary

Keep open. Current main still carries media-understanding provider/model telemetry internally, but the assistant-visible inbound body is still formatted from only the media title, optional user text, and transcript/description text, so the guessed-model failure remains source-reproducible.

## What This Changes

The issue asks OpenClaw to expose the actual media-understanding provider/model used for the current inbound media turn in the assistant-visible body so the assistant does not guess.

## Best Possible Solution

Render an `Analysis model:` line from real current-turn `MediaUnderstandingOutput.provider` and `model` values in the formatter/apply path, with explicit handling for missing model values and synthetic placeholders.

## Reproduction Assessment

Yes for a source-level current-main path: successful outputs carry provider/model, the formatter drops those fields, and the apply path forces the formatted text into `BodyForAgent`. I did not rerun the live Discord scenario.

## Solution Assessment

Yes. The narrow maintainable fix is to render existing current-turn telemetry from the media-understanding output contract, not infer it from conversation defaults or broader provider-routing state.
