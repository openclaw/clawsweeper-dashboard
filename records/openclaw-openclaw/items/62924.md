Keep open. Current main and shipped v2026.5.12 still carry media-understanding provider/model telemetry internally but omit it from the assistant-visible media body, so the guessed-model failure remains source-reproducible and narrowly fixable.

## What This Changes

Expose the actual current-turn media-understanding provider/model in the inbound assistant body so the assistant does not guess which media model analyzed an attachment.

## Best Possible Solution

Render `Analysis model:` from real current-turn `MediaUnderstandingOutput` telemetry in the formatter/apply path, suppress synthetic placeholders, and add focused regression coverage.
