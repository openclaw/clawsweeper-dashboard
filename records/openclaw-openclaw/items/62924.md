## Summary

Keep open. Current main still carries the actual media-understanding provider/model on typed outputs, but the assistant-visible inbound body is rendered from title, optional user text, and transcript/description only, so the reporter’s guessed media-model problem remains unresolved.

## What This Changes

Expose the actual media-understanding provider/model used for each current-turn media analysis in the assistant-visible inbound body.

## Best Possible Solution

Add a narrow formatter/apply update that renders `Analysis model:` from real successful `MediaUnderstandingOutput.provider` and `model` values, handles provider-only and synthetic outputs explicitly, and updates focused tests plus changelog.

## Reproduction Assessment

Yes. A source-level reproduction is available on current main: pass an output with `provider` and `model` into `formatMediaUnderstandingBody`, then observe that the returned body lacks `Analysis model:` and is promoted into `BodyForAgent` by `applyMediaUnderstanding`.

## Solution Assessment

Yes. The narrowest maintainable fix is in the media-understanding formatter/apply path using existing typed telemetry; changing conversation-model fallback behavior would not expose the actual current-turn media analyzer.
