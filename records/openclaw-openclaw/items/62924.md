## Summary

Keep open. Current main and v2026.5.12 still carry media-understanding provider/model telemetry internally, but the assistant-visible media body is formatted without that telemetry, so the guessed-model failure remains source-reproducible and narrowly fixable.

## What This Changes

Expose the actual current-turn media-understanding provider/model in the assistant-visible inbound body so the assistant does not guess the analysis model.

## Best Possible Solution

Render an `Analysis model:` line from real current-turn `MediaUnderstandingOutput.provider` and `model` telemetry in the formatter/apply path, without falling back to conversation models, stale session state, defaults, or guesses.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: media outputs carry provider/model, `formatMediaUnderstandingBody` drops that telemetry, and `applyMediaUnderstanding` forces the formatted body into `BodyForAgent`.

## Solution Assessment

Yes. The narrow maintainable fix is to render existing current-turn telemetry in the media formatter/apply path while suppressing synthetic placeholders and avoiding any guessed fallback state.
