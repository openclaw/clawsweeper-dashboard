## Summary

Keep open. Current main still carries actual media-understanding provider/model telemetry internally, but the assistant-visible inbound body is still formatted from title, optional user text, and transcript/description only, so the reporter’s guessed-model problem remains unresolved.

## What This Changes

Expose the actual media-understanding provider/model used for the current inbound media turn in the assistant-visible body, or otherwise make the assistant say it cannot confirm the analysis model.

## Best Possible Solution

Add a narrow media-understanding formatter/apply behavior that renders `Analysis model:` from real successful `MediaUnderstandingOutput.provider` and `model` values, handles provider-only and synthetic-placeholder outputs explicitly, and covers the assistant-visible body with focused formatter/apply tests plus a user-facing changelog entry.

## Reproduction Assessment

Yes. A source-level reproduction is available on current main: pass an output with `provider` and `model` into `formatMediaUnderstandingBody`, and the returned body contains only the transcript/description; `applyMediaUnderstanding` then promotes that body into `BodyForAgent`.

## Solution Assessment

Yes. The proposed formatter/apply-path fix is the narrowest maintainable solution because the runtime already has typed current-turn telemetry; changing conversation-model fallback behavior would not expose the actual media analyzer used for the inbound media turn.
