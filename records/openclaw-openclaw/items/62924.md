## Summary

Keep open. Current main still carries actual media-understanding provider/model telemetry internally, but the assistant-visible inbound body is formatted without that telemetry, so the reported guessed media-model behavior remains unresolved.

## What This Changes

Expose the actual media-understanding provider/model in the assistant-visible inbound body so the assistant does not guess which media model analyzed the current turn.

## Best Possible Solution

Render analysis telemetry from real successful `MediaUnderstandingOutput.provider` and `model` values in the media-understanding body, with explicit handling for missing model values and synthetic placeholder outputs.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: pass an output with `provider` and `model` into `formatMediaUnderstandingBody`, then observe the returned body lacks `Analysis model:` and is promoted to `BodyForAgent` by `applyMediaUnderstanding`.

## Solution Assessment

Yes. The narrowest maintainable fix is in the media-understanding formatter/apply path using existing typed telemetry; changing conversation-model fallback behavior would not expose the actual current-turn media analyzer.
