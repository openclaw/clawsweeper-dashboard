## Summary

Keep open. Current main still preserves media-understanding provider/model telemetry internally but drops it from the assistant-visible body, so the reported guessed-model behavior remains a narrow unresolved formatter/apply-path bug.

## What This Changes

Expose the actual media-understanding provider/model used for the current turn in the assistant-visible inbound media body so replies do not guess the analysis model.

## Best Possible Solution

Render the real current-turn media-understanding provider/model from typed output telemetry in the formatter/apply path and explicitly suppress synthetic placeholder telemetry.

## Reproduction Assessment

Yes, by source inspection: a `MediaUnderstandingOutput` with `provider` and `model` reaches `formatMediaUnderstandingBody`, and the returned body omits any `Analysis model:` line before becoming `BodyForAgent`. I did not run tests because this review must keep the checkout read-only.

## Solution Assessment

Yes. The narrow fix belongs in the media-understanding formatter/apply path using existing typed telemetry, with a guard so synthetic placeholders are not reported as real model analysis.
