## Summary

Keep open. Current main and the latest release still carry media-understanding provider/model telemetry internally but omit it from the assistant-visible `BodyForAgent` block, so the guessed-model failure remains source-reproducible and narrowly fixable.

## What This Changes

Expose the actual provider/model chosen for current-turn media understanding in the inbound assistant-visible body so the assistant does not guess.

## Best Possible Solution

Render an `Analysis model:` line from real current-turn `MediaUnderstandingOutput.provider` and `model`, skip or clearly handle synthetic placeholders, and cover the formatter/apply path with regression tests.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: media execution returns provider/model telemetry, `formatMediaUnderstandingBody` drops it, and `forceBodyForAgent` exposes the telemetry-less body to the assistant. I did not run live Discord or tests because this review is read-only.

## Solution Assessment

Yes. The narrow maintainable fix is to render already-carried current-turn telemetry in the media formatter/apply path, while avoiding synthetic placeholders and any guessed fallback state.
