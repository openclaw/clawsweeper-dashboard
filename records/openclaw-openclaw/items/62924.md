## Summary

Keep open. Current main and the latest release still carry media-understanding provider/model telemetry internally, but the assistant-visible inbound body is still formatted without that telemetry, so the reported guessed-model failure remains source-reproducible and narrowly fixable.

## What This Changes

The issue requests assistant-visible current-turn telemetry for the actual media-understanding provider/model so the assistant does not guess which model analyzed media.

## Best Possible Solution

Render an `Analysis model:` line from real current-turn `MediaUnderstandingOutput.provider` and `model` values, with explicit handling for missing model values and synthetic placeholders.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence current-main path: outputs carry provider/model, the formatter drops those fields, and the apply path forces the formatted text into `BodyForAgent`.

## Solution Assessment

Yes. The narrow maintainable fix is to render existing current-turn telemetry from the media-understanding output contract, not infer it from conversation defaults or broader provider-routing state.
