## Summary

Keep open. Current main still carries media-understanding provider/model telemetry internally, but the assistant-visible inbound body drops it before `BodyForAgent`, so the reporter's guessed-model failure remains a valid narrow core gap.

## What This Changes

The issue asks OpenClaw to include the actual current-turn media-understanding provider/model in the assistant-visible inbound media body, or otherwise avoid confirming a guessed analysis model.

## Best Possible Solution

Render an `Analysis model:` line from real successful `MediaUnderstandingOutput` telemetry in the media-understanding body, with explicit handling for missing model values and synthetic placeholders.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: pass a `MediaUnderstandingOutput` with `provider` and `model` into `formatMediaUnderstandingBody`, then observe the returned body lacks `Analysis model:` and is promoted to `BodyForAgent` by `applyMediaUnderstanding`.

## Solution Assessment

Yes. The narrowest maintainable solution is to use the existing typed telemetry in the media-understanding formatter/apply path, while guarding synthetic and missing-model cases.

## Repair Work Prompt

Fix the media-understanding inbound body so the assistant can see the actual current-turn analysis backend. In `src/media-understanding/format.ts`, render an `Analysis model:` line from real `MediaUnderstandingOutput.provider` and `model` telemetry for successful outputs. Use a canonical provider/model value when both are present, handle provider-only or missing-model cases explicitly, and do not expose synthetic placeholder outputs such as `provider: "openclaw", model: "synthetic-empty-audio"` as real analysis telemetry. Keep the behavior current-turn scoped; do not infer from conversation model, session state, fallback history, or stale config. Update focused tests in `src/media-understanding/format.test.ts` and `src/media-understanding/apply.test.ts`, including image/audio/video, provider-only, multiple-output formatting, and synthetic skipped-audio cases. Add a user-facing `CHANGELOG.md` fix entry for this issue. Do not change provider selection, media config schema, or broad model fallback behavior.
