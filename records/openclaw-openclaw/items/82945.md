## Summary

Keep this PR open: it fixes the right Telegram raw-update logging path, but the current redactor still misses sensitive fields from default Telegram update families, so it is not merge-ready as the security fix.

## What This Changes

This PR moves Telegram verbose raw-update serialization into a new redacting formatter, wires bot-core logging through it, adds formatter tests, and records a changelog entry.

## Best Possible Solution

Land a conservative raw-update formatter that redacts sensitive fields across the default Telegram update set, with focused regression cases for message/callback, business/join/reaction, payment/shipping, paid media, and passport payloads.

## Reproduction Assessment

Yes. Source inspection shows current main serializes `ctx.update` directly on the verbose Telegram raw-update log path, and the default allowed update list plus grammY types show concrete sensitive update shapes that can reach it.

## Solution Assessment

No. Extracting a formatter is the right repair shape, but the current implementation is too narrow for Telegram’s default payment, shipping, paid-media, and passport payload surfaces.
