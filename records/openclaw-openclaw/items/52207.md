## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Map Tavily freshness to the current time_range contract:** `extensions/tavily/src/tavily-search-provider.ts:73-76`
  - body: The shared `web_search` contract uses `day`, `week`, `month`, or `year`, and Tavily's current API supports those through `time_range`; this branch only accepts `pd`/`pw`/`pm`/`py` and sends `days`, so valid calls such as `{ freshness: "week" }` are ignored or sent on the wrong API field. ([docs.tavily.com](https://docs.tavily.com/documentation/api-reference/endpoint/search?utm_source=openai))
  - confidence: 0.88
- **[P1] Preserve custom OpenRouter completion endpoints:** `src/agents/pi-embedded-runner/model.provider-normalization.ts:31-37`
  - body: The added `isOpenRouter` branch rewrites every OpenRouter `openai-completions` model to `openai-responses`, even when the user configured a custom or proxied `baseUrl`. That can break deployments that intentionally point OpenRouter-compatible traffic at a Chat Completions-only endpoint, and it is unrelated to the web-search feature.
  - confidence: 0.86
- **[P2] Do not accept Tavily keys on an unread config path:** `src/config/zod-schema.agent-runtime.ts:319-325`
  - body: This schema addition makes `tools.web.search.tavily.apiKey` validate, but the Tavily resolver still reads `plugins.entries.tavily.config.webSearch.apiKey` or `TAVILY_API_KEY`. Users can follow the newly accepted config shape and still get missing-key errors at runtime.
  - confidence: 0.82
- **[P2] Prompt SearXNG setup for the base URL:** `extensions/searxng/src/searxng-search-provider.ts:55-69`
  - body: The provider advertises an optional SearXNG API key as the onboarding credential, but SearXNG search needs the self-hosted `baseUrl`. Selecting SearXNG can store the user's setup input on the wrong field and leave the provider pointed at the fallback URL.
  - confidence: 0.78
- **[P2] Count keyless SearXNG search in exposure audits:** `src/security/audit-extra.sync.ts:335-339`
  - body: SearXNG can run with a configured base URL and no API key, but this audit heuristic checks only key-based setup. A keyless SearXNG configuration can therefore run web search while the exposure audit treats web search as disabled.
  - confidence: 0.78
