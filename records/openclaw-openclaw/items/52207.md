  - body: This branch rewrites every `openrouter` model using `openai-completions` to `openai-responses`, even when the user configured a custom or proxied `baseUrl`. Existing deployments that only implement Chat Completions would stop working despite explicitly configuring that API surface.

  - body: The generic `web_search` tool advertises freshness as `day`, `week`, `month`, or `year`, and Tavily documents those values under `time_range`; this mapping only accepts `pd`/`pw`/`pm`/`py` and forwards `days`, which is news-topic-specific, so normal generic freshness calls can silently lose the recency filter.   - confidence: 0.84

  - body: Adding `tools.web.search.tavily.apiKey` to the strict runtime schema makes that config validate, but the Tavily resolver still reads only `plugins.entries.tavily.config.webSearch.apiKey` or `TAVILY_API_KEY`. Users can follow the accepted config and still get missing-key errors at runtime.
  - confidence: 0.8
- **[P2] Count keyless SearXNG setups in web-search audits:** `src/security/audit-extra.sync.ts:335-339`
  - body: SearXNG is intentionally usable without an API key, but this audit heuristic only counts SearXNG when `search.searxng.apiKey` or `SEARXNG_API_KEY` is present. A selected provider with a configured base URL can therefore be active while the exposure audit reports web search as absent.
