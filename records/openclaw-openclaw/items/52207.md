## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep OpenRouter custom endpoints on the configured API:** `src/agents/pi-embedded-runner/model.provider-normalization.ts:31-37`
  - body: The new `isOpenRouter` branch rewrites every OpenRouter `openai-completions` model to `openai-responses`, even when the user configured a custom or proxied `baseUrl`. That can break deployments that intentionally point OpenRouter-compatible traffic at a Chat Completions-only endpoint, and it is unrelated to the web-search feature.
  - confidence: 0.86
- **[P2] Map Tavily freshness from the shared contract:** `extensions/tavily/src/tavily-search-provider.ts:73-76`
  - body: The shared `web_search` contract uses `day`, `week`, `month`, or `year`, but this branch only recognizes `pd`/`pw`/`pm`/`py`. A valid call like `{ freshness: "week" }` will not be applied for Tavily, so the new option does not satisfy the current tool contract.
  - confidence: 0.88
- **[P2] Read Tavily keys from any accepted config path:** `src/config/zod-schema.agent-runtime.ts:318-325`
  - body: Adding `tools.web.search.tavily.apiKey` to the strict runtime schema makes that config validate, but the Tavily resolver still reads only `plugins.entries.tavily.config.webSearch.apiKey` or `TAVILY_API_KEY`. Users can follow the accepted config shape and still get missing-key errors at runtime.
  - confidence: 0.82
- **[P2] Prompt SearXNG setup for the base URL:** `extensions/searxng/src/searxng-search-provider.ts:55-69`
  - body: The provider advertises and stores an optional SearXNG API key as the onboarding credential, but SearXNG search needs the self-hosted `baseUrl`. Selecting SearXNG can save setup input on the wrong field and leave the provider pointed at a fallback URL.
  - confidence: 0.78
- **[P2] Count keyless SearXNG setups in exposure audits:** `src/security/audit-extra.sync.ts:335-339`
  - body: SearXNG can run with a configured base URL and no API key, but this audit heuristic checks only key-based setup. A keyless SearXNG configuration can therefore run web search while the exposure audit treats web search as disabled.
  - confidence: 0.78
