  - body: The new `isOpenRouter` branch rewrites every OpenRouter `openai-completions` model to `openai-responses`, including custom or proxied `baseUrl` deployments that may only implement Chat Completions. This unrelated change can break configurations that explicitly selected that API surface.

  - body: Adding `tools.web.search.tavily.apiKey` to the strict runtime schema makes that config validate, but the Tavily resolver still reads only `plugins.entries.tavily.config.webSearch.apiKey` or `TAVILY_API_KEY`. Users can follow the accepted config and still hit missing-key errors at runtime.
  - confidence: 0.8
- **[P2] Count keyless SearXNG search in exposure audits:** `src/security/audit-extra.sync.ts:335-339`
  - body: SearXNG can run with a configured base URL and no API key, but this heuristic only counts SearXNG when an API key or `SEARXNG_API_KEY` exists. A keyless SearXNG configuration can run web search while the audit reports it disabled.
  - confidence: 0.78
