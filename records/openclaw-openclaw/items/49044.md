## Summary

Keep open: current main still lacks the feature, and this PR is not merge-ready because it persists an account-level `soulFile` value that current schema validation rejects and current bootstrap loading never consumes. The earlier https://github.com/openclaw/openclaw/pull/42861 discussion is useful context, but that PR was closed as superseded by this one rather than shipped.

## What This Changes

The branch adds `channels add --soul`, onboarding prompts, an account-scoped SOUL allowlist, and setup-snapshot tests that store `soulFile` under channel account config.

## Best Possible Solution

Rework this as a complete account-scoped SOUL contract: schema/type support, CLI and onboarding writes on current main, channel/account-aware bootstrap resolution and cache keys, safe path fallback behavior, focused tests, and redacted live channel proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: the PR writes `soulFile`, current main validates strict account schemas without that key, and bootstrap loading still uses the canonical workspace `SOUL.md`.

## Solution Assessment

No. This PR is not the best merge shape because it adds write surfaces without the matching current-main schema and runtime reader; the safer solution is a complete implementation of the account-scoped SOUL contract.
