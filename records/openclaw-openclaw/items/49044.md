## Summary

Keep open. Current main still does not implement per-account SOUL files, and this PR is not a safe merge shape because it writes a new account-level `soulFile` key without the matching current-main schema/runtime reader and without real behavior proof.

This PR adds `channels add --soul`, account-scoped `soulFile` writes, onboarding SOUL prompts, and channel setup snapshot coverage for per-account SOUL files.

## Best Possible Solution
