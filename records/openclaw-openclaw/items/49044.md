## Summary

Keep open: the branch now wires the CLI flag and fixes the setup snapshot issue, but the central feature is not end-to-end because `soulFile` is neither accepted by the current strict channel account schemas nor consumed by bootstrap/system-prompt resolution, and the PR still lacks real behavior proof.

## What This Changes

The PR adds `--soul` handling to `channels add` and onboarding, stores `soulFile` on selected channel accounts, and adjusts channel setup snapshot tests/types.

## Best Possible Solution

Implement the feature end-to-end with a documented account-level `soulFile` contract, schema support for the chosen channels/plugins, account-aware bootstrap resolution, workspace-scoped filename validation, regression tests, and real channel proof.

## Reproduction Assessment

Yes for the review blockers by source inspection: the PR writes `soulFile`, current config writes validate strict channel account schemas that do not define it, and current bootstrap reads only the fixed workspace `SOUL.md`. I did not run commands or tests because this review was required to keep the checkout read-only.

## Solution Assessment

No. A CLI/onboarding-only config write is not sufficient; the maintainable fix needs schema and runtime bootstrap changes in the shared account/session path, plus proof that replies actually use the selected account SOUL file.
