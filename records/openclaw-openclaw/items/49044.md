## Summary

Keep open: the requested feature is not implemented on current main, but this PR is not merge-ready because it writes a new `soulFile` config key without adding validated channel schema support or any runtime bootstrap path that reads that key.

## What This Changes

This PR adds a `--soul` option and onboarding prompts that store `soulFile` under channel account config, plus channel setup snapshot compatibility tests.

## Best Possible Solution

Implement per-account SOUL as a complete config and runtime contract: validate `soulFile` for supported account schemas, resolve it during channel/account-aware bootstrap loading, preserve default `SOUL.md` fallback, update cache/tests/docs, and rebase onto current main.

## Reproduction Assessment

Yes, the PR defects are source-reproducible: the patch writes `channels.*.accounts.*.soulFile`, current write validation uses strict schemas that lack that key, and current bootstrap loading only selects fixed workspace bootstrap filenames. I did not run tests because this review is read-only and `pnpm` is unavailable.

## Solution Assessment

No. A command-only config write is not the best fix; the maintainable solution must add the validated config contract and thread channel/account context into bootstrap SOUL file resolution before exposing the user-facing flag and onboarding prompt.
