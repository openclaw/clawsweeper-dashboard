## Summary

Keep open. Current main has active config validation, doctor unknown-key stripping, current-schema output, and Control UI pending-change diffs, but it still does not provide the requested historical `openclaw doctor --config-diff` migration-guidance workflow; the related items cover adjacent safety or UI flows rather than superseding this feature.

## What This Changes

Request a new `openclaw doctor --config-diff` mode that compares a config's last-touched version against the current schema, reports removed/renamed/new keys and enum changes, and can apply safe migrations.

## Best Possible Solution

Define a maintainer-approved doctor upgrade-diff design for schema baselines, migration-suggestion ownership, and safe auto-fix boundaries, then implement the CLI, docs, and tests around that contract.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. The missing CLI surface is high-confidence from command registration, option types, docs, source search, and current-schema-only repair code.

## Solution Assessment

Unclear until maintainers settle the design. A doctor CLI workflow is plausible, but schema snapshot storage, migration advice ownership, and safe auto-fix rules need product and code-owner decisions first.
