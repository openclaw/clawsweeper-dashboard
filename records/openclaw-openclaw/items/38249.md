## What This Changes

The issue requests an `openclaw doctor --config-diff` flag that compares a user's last-touched config version with the current schema and suggests safe migrations.

## Best Possible Solution

Define a versioned, redacted doctor config-diff contract, then implement the CLI flag, safe migration ownership, docs, and focused tests around that contract.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence that current main lacks the requested `doctor --config-diff` mode while related commands only validate or lint current config state.

## Solution Assessment

Unclear until maintainers settle the design. A doctor CLI workflow is plausible, but historical schema storage, migration-advice ownership, redacted output, and automatic rewrite policy need product decisions first.
