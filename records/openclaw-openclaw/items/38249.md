## Summary

Keep open: current main and the latest release still do not provide the requested historical config schema-diff workflow; doctor covers current-schema validation, known legacy migrations, and unknown-key cleanup, but not version-to-version migration guidance. The related reports cover adjacent Control UI apply flows or specific validation bugs, so they do not supersede this CLI feature request.

## What This Changes

Add `openclaw doctor --config-diff` to compare the config last-touched version with the current schema, report stale or invalid keys/values, and optionally apply safe migrations.

## Best Possible Solution

Define a versioned doctor config-diff contract first, then implement redacted comparison output, curated migration suggestions, safe auto-fix rules, docs, and focused regression coverage.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source and docs inspection give high confidence the requested CLI mode is absent on current main and the latest release.

## Solution Assessment

Unclear until maintainers settle the product contract. A doctor CLI workflow is plausible, but historical schema storage, migration-advice ownership, redaction, and automatic rewrites need explicit design first.
