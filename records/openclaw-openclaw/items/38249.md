## Summary

Keep open. Current main has active-schema validation, doctor repair for unknown keys, and Control UI pending-change diffs, but it still lacks the requested `openclaw doctor --config-diff` CLI workflow and version-to-version migration report.

## What This Changes

Add `openclaw doctor --config-diff` and optional fix mode to compare a user's last-touched config version with the current schema and provide migration guidance.

## Best Possible Solution

Implement a doctor CLI upgrade-diff workflow, or document a chosen alternate UX, that surfaces stale/renamed config keys with maintainable migration guidance.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. The missing CLI surface is high-confidence from command registration, option types, docs, and source search on current main.

## Solution Assessment

Unclear until maintainer design is settled. The CLI direction is plausible, but historical schema storage, migration-suggestion ownership, and safe auto-fix boundaries need product/code-owner decisions first.
