## Summary

Keep open: current main and the latest release still lack the requested historical `openclaw doctor --config-diff` workflow; existing support covers live-schema validation, known legacy migrations, and current-schema unknown-key cleanup instead.

## What This Changes

Request to add `openclaw doctor --config-diff` and safe `--fix` migration guidance for configs written by older OpenClaw versions.

## Best Possible Solution

Define a versioned config-diff contract for doctor, then implement the CLI flag, redacted comparison output, safe migration ownership, docs, and focused tests around that contract.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source inspection gives high confidence the requested CLI mode is absent because doctor registration/options have no config-diff path and only current-schema cleanup plus registered legacy migrations are implemented.

## Solution Assessment

Unclear until maintainers settle the design. A doctor CLI workflow is plausible, but per-release schema storage, migration-advice ownership, redaction, and automatic enum/key rewrites are product and code-owner decisions.
