## Summary

Keep open: current `main` and shipped `v2026.5.12` still lack the requested historical `openclaw doctor --config-diff` workflow. Existing code covers current-schema validation, doctor repair, and unknown-key cleanup, but not version-to-version schema comparison with migration guidance.

## What This Changes

The issue requests an `openclaw doctor --config-diff` flag that compares a config's last-touched version with the current schema, reports stale keys or enum values, and can apply safe migrations.

## Best Possible Solution

Define a versioned doctor config-diff contract, then implement redacted comparison output, safe migration ownership, docs, and focused tests around that contract.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source, docs, and release inspection give high confidence that the requested CLI mode is absent on current `main` and in `v2026.5.12`.

## Solution Assessment

Unclear until maintainers settle the design. A doctor CLI workflow is plausible, but historical schema retention, migration-advice ownership, redaction, and automatic rewrites need product and code-owner decisions first.
