## Summary

Keep open: current main and the latest release still provide current-schema validation, doctor repair, unknown-key stripping, and config dry-run/schema helpers, but they do not implement the requested historical `doctor --config-diff` workflow or migration-guidance output.

## What This Changes

Request adds `openclaw doctor --config-diff` to compare a config's last-touched version against the current schema and offer migration guidance plus safe fixes.

## Best Possible Solution

Define a doctor upgrade-diff contract for versioned schema baselines, redacted historical comparison, migration-suggestion ownership, and safe `--fix` boundaries, then implement the CLI, docs, and focused tests around that contract.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source and docs checks give a high-confidence current-main gap: no `--config-diff` option, no historical schema snapshots, and current doctor repair only uses live-schema cleanup plus registered legacy migrations.

## Solution Assessment

Unclear until maintainers settle the design. A doctor CLI workflow is plausible, but per-release schema storage, migration-advice ownership, redaction, and automatic enum/key rewrites are product and code-owner decisions.
