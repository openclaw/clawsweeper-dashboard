## Summary

Keep open. Current main has active-schema validation, doctor unknown-key stripping, current Control UI pending-change diffs, and adjacent recovery fixes, but it still does not provide the requested `openclaw doctor --config-diff` historical schema-diff and migration-guidance workflow; the related open/closed issues cover adjacent safety or UI flows rather than superseding this feature.

## What This Changes

Add an `openclaw doctor --config-diff` workflow that compares a config's last-touched version with the current schema, reports stale or invalid config keys and enum values, and optionally applies approved safe migrations.

## Best Possible Solution

Define a maintainer-approved doctor upgrade-diff design for schema baselines, migration-suggestion ownership, and safe auto-fix boundaries, then implement the CLI/docs/tests around that contract.
