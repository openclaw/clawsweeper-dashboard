## Summary

Keep open. Current main has active-schema validation, doctor repair for unknown keys, and Control UI pending-change diffs, but it still lacks the requested `openclaw doctor --config-diff` CLI workflow and historical version-to-version migration report; the linked issues cover adjacent UX and validator-recovery problems rather than superseding this feature.

## What This Changes

Add a `doctor --config-diff` CLI mode that reports stale, renamed, removed, or invalid config keys across OpenClaw versions and can optionally apply safe migrations.

## Best Possible Solution

Define a maintainer-approved doctor upgrade-diff design for historical schema baselines, migration suggestion ownership, and safe auto-fix boundaries, then implement it with CLI docs and focused tests.
