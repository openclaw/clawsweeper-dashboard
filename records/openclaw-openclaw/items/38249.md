## Summary

Keep open. Current main has current-schema validation, doctor legacy migrations, unknown-key cleanup, and adjacent dry-run work in flight, but it still lacks the requested historical `openclaw doctor --config-diff` workflow, per-release schema comparison, and migration-guidance output.

## What This Changes

Request `openclaw doctor --config-diff` to compare a config's last-touched version with the current schema, report removed/renamed/new/invalid fields, and optionally apply safe migrations.

## Best Possible Solution

Define a doctor upgrade-diff contract for schema baselines, version comparison, redacted output, migration-suggestion ownership, and safe auto-fix boundaries, then implement the CLI, docs, and focused tests around that contract.

## Reproduction Assessment

Not applicable as a runtime bug; this is a feature request. Source and docs inspection give a high-confidence current-main gap: there is no doctor config-diff option, typed mode, historical schema store, or migration-guidance diff.

## Solution Assessment

Unclear until maintainers settle the design. A doctor CLI workflow is plausible, but the historical-schema source, migration-advice ownership, redaction, and safe-fix rules need product and code-owner decisions first.
