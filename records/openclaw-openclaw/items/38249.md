## Summary

Keep open: current main still has current-schema validation, unknown-key cleanup, and doctor lint/repair paths, but it does not provide the requested historical `openclaw doctor --config-diff` workflow or migration-advice contract.

## What This Changes

Add an `openclaw doctor --config-diff` mode, with optional fixing, to compare a config’s last-touched version against the current schema and show upgrade-safe migration guidance.

## Best Possible Solution

Define a versioned, redacted doctor config-diff contract, then implement the CLI flag, migration-advice ownership, safe fix policy, docs, and focused tests around that contract.

## Reproduction Assessment

## Solution Assessment

Unclear until maintainers settle the design. A doctor CLI workflow is plausible, but historical schema storage, migration-advice ownership, redacted output, and automatic rewrite policy need product decisions first.
