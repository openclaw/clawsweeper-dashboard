  - body: `tags` are arbitrary plugin/user metadata, but this path emits accepted values directly into Prometheus labels. That bypasses the existing `lowCardinalityLabel` redaction boundary and can leak secret-like values, hostnames, or paths despite the Prometheus docs saying those never appear.

  - body: The OTel exporter has the same raw tag path: accepted tag values become `openclaw.task.flow.tag.*` attributes without going through `lowCardinalityAttr` or another redaction helper. Please keep the tag values on the same redacted low-cardinality path as the rest of diagnostic telemetry.
  - confidence: 0.9
- **[P2] Keep generated Prometheus tag labels valid:** `extensions/diagnostics-prometheus/src/service.ts:506`
  - body: This builds label names from raw tag keys after `LOW_CARDINALITY_VALUE_RE`, which allows characters such as `.` and `-`; `formatLabels` later prints label names raw.
