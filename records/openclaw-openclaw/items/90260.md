## What This Changes

The PR adds a decode-once guard to the shared HTML-entity tool-call argument decoder, removes the duplicate xAI provider decoder, and adds focused regression coverage for core and xAI wrapper behavior.

## Best Possible Solution

Land the narrow core decode-once guard and xAI duplicate-removal after normal checks, keeping `html-entities` handling owned by model compat plus the shared core wrapper.

## Reproduction Assessment

Yes. Source inspection shows current main can decode the same final tool-call arguments through partial/message/result and xAI stacks a plugin-local decoder with the core compat decoder; the PR body also includes a focused before/after wrapper-chain reproduction.

## Solution Assessment

Yes. The best fix is to make the shared core decoder idempotent per arguments object and delete the xAI-local duplicate; keeping a provider-local patch would leave Venice dependent on the same core bug and preserve duplicated policy.
