## Summary

Keep open: current main still collapses a zero-byte or invalid session store to an empty store, and the relevant recovery work is in an open closing PR rather than merged.

## What This Changes

The issue requests corruption detection and automatic recovery for empty or invalid `sessions.json`, plus doctor visibility for that failure mode.

## Best Possible Solution

Review and land the focused session-store recovery PR, or replace it with an equivalent loader/doctor fix that detects corrupt stores and safely self-heals from trusted artifacts.

## Reproduction Assessment

Yes. Source inspection shows a zero-byte `sessions.json` makes `JSON.parse` throw, the loader returns the default `{}`, and `sessions.list` then aggregates no rows; I did not run a mutating filesystem repro in this read-only review.

## Solution Assessment

No, current main is not sufficient because it lacks corrupt-store recovery. The narrow path is to review/land https://github.com/openclaw/openclaw/pull/79910 or an equivalent loader/doctor fix; https://github.com/openclaw/openclaw/pull/81692 is not the same solution.
