## Summary

Keep open: current main still treats a zero-byte or invalid `sessions.json` as an empty store, and the focused closing recovery PR remains open rather than merged.

## What This Changes

The issue requests automatic recovery and diagnostics when `sessions.json` becomes empty or corrupt so session aggregation does not silently collapse.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/79910, or replace it with an equivalent loader and doctor recovery fix that safely validates corrupt stores and self-heals from trusted artifacts.

## Reproduction Assessment

Yes, source-reproducible: a zero-byte `sessions.json` makes `JSON.parse` throw, `loadSessionStore` catches the final failure and returns `{}`, and `sessions.list` then aggregates that empty store. I did not run a mutating filesystem repro in this read-only review.

## Solution Assessment

No, current main is not sufficient because it lacks corrupt-store recovery. The narrow maintainable path is to land the open focused recovery PR or an equivalent fix in the same loader and doctor surface.
