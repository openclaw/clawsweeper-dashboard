The PR changes Control UI approval resolution by renaming approval buttons, removing queue entries with the returned id, adding a resolved-event log entry, dispatching a UI event, and reloading Overview after a decision.

## Best Possible Solution

Land only a rebased version that updates approval labels through the Control UI i18n source, uses the event buffer or canonical gateway resolved events for durable history, and demonstrates the approval flow in a running UI.
