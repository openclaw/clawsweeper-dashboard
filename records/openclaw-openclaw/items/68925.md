## What This Changes

The PR branch renames Control UI approval actions, removes resolved approvals after a resolve request, adds a synthetic approval-resolved log/DOM event, and refreshes overview state after approval resolution.

## Best Possible Solution

Keep the approval-flow polish, but rebase it onto current main, either remove the synthetic local log and rely on canonical gateway resolved broadcasts or write through the same capped event-buffer path, and route any button-copy change through the existing locale keys with focused UI coverage.

## Reproduction Assessment

Yes. With the PR patch, resolve an approval and then trigger an event-log sync such as Debug refresh; the new synthetic `approval-resolved` entry is absent from `eventLogBuffer` and is overwritten from the buffer.

## Solution Assessment

No. The current patch writes to only one visible state array and bypasses the established gateway-event history path; the narrower maintainable fix is to use the canonical `*.approval.resolved` events or a shared buffer append helper, plus a clean i18n-aware rebase.
