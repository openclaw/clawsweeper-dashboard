## Summary

Keep open: the PR targets a plausible source-level post-compaction context gap, but it lacks after-fix real behavior proof and introduces blocking correctness risks around stale transcript selection and unbounded transcript reads.

## What This Changes

The PR adds transcript compaction-summary extraction, injects that summary into post-compaction context, threads `sessionFile` through two runner paths, and adds focused tests.

## Best Possible Solution

Read the latest compaction summary from the active post-compaction transcript with a bounded/tail parser, preserve the AGENTS.md refresh contract, add regression coverage for rotated transcripts, and attach redacted real-run proof.

## Reproduction Assessment

No. Source inspection confirms current main does not pass a session transcript into `readPostCompactionContext()`, but there is no live reproduction or after-fix runtime artifact showing agent restart before and continuation after.

## Solution Assessment

No. Injecting the summary is a reasonable direction, but this diff should use the refreshed active session file and a bounded transcript scan before it is a maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Read the refreshed compaction transcript:** `src/auto-reply/reply/agent-runner.ts:1915`
  - body: After auto-compaction, the code above persists `runResult.meta.agentMeta.sessionFile` into the session store and refreshes queued runs, but this new argument still points at `followupRun.run.sessionFile`. When compaction rotates or delegates to a new active transcript, the post-compaction context can read the pre-compaction archive and miss or stale-read the summary it is supposed to inject. Use the refreshed session entry or `runResult.meta.agentMeta.sessionFile` for this call.
  - confidence: 0.83
- **[P2] Avoid reading the entire transcript:** `src/auto-reply/reply/post-compaction-context.ts:77`
  - body: `readLatestCompactionSummary()` loads the whole JSONL transcript into memory even though it only needs the latest compaction entry near the tail. This runs specifically after sessions have grown large enough to compact, so tool-heavy transcripts can stall the reply path or spike memory; use a bounded reverse/tail scan or existing transcript reader instead.
  - confidence: 0.78
