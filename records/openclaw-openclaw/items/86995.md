## Summary

Keep open: the bugfix direction is valid and the PR now has terminal proof, but the prompt-local insertion still misplaces runtime context for non-empty Pi sessions, so the central stale-reply fix can fail on existing conversations.

## What This Changes

The PR replaces queued next-turn Pi runtime context with prompt-local hidden custom messages, tightens LLM-boundary cleanup, adjusts late Codex app-server steering, and updates related tests and the runtime-context E2E harness.

## Best Possible Solution

Keep runtime context prompt-local by appending it to pre-prompt state for normal submissions and reinserting before an already-present active user only during retry/continue, with regression coverage for non-empty sessions and overflow retry.

## Reproduction Assessment

Yes, from source inspection: use a Pi session with prior user/assistant history and current-turn runtime context, then the PR helper inserts before the historical user before AgentSession.prompt appends the active user. The LLM-boundary cleanup then strips that context because it is not immediately before the last user.

## Solution Assessment

No. The intended prompt-local approach is right, but the implementation needs separate handling for pre-prompt insertion versus retry state where the active user is already present.

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Do not insert prompt context before historical users:** `src/agents/pi-embedded-runner/run/attempt.ts:1018-1025`
  - body: In the normal prompt path the active user message does not exist in `activeSession.messages` yet: `AgentSession.prompt()` builds it later and pi-agent-core sends `context.messages + prompts`. With existing history, this helper finds the previous user and inserts the runtime context before that historical turn. When the active prompt user is appended, `normalizeMessagesForLlmBoundary()` keeps only runtime context immediately before the last user, so this new context is stripped and non-empty sessions lose the very prompt-local context this PR is meant to preserve. Append for the pre-prompt case, and keep the before-user insertion only for retry/continue states where the active user is already present.
  - confidence: 0.93
