The branch changes the gateway approval-runtime token helper to prefer an existing `exec-approvals.json` socket token, keeps a process-local fallback, and updates approval client tests for loopback versus remote token attachment.

## Best Possible Solution

Land the narrow shared-token path only after maintainers explicitly accept same-`OPENCLAW_HOME` socket-token trust for local approval-runtime clients, while preserving remote token omission and fallback coverage.

## Solution Assessment

Yes, with maintainer sign-off: reusing the existing exec-approvals socket token is a narrow fix that avoids creating new config and keeps remote URLs token-free. The remaining question is whether maintainers accept that token as approval-runtime authority.
