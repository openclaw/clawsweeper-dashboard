## Summary

This should stay open: current main still hides configured-agent subagent rows from the normal Control UI sidebar/session selectors, and only part of the live tool-progress path appears to have been addressed after the reported release.

## What This Changes

The issue reports WebChat dashboard regressions after upgrading to 2026.5.18: configured-agent subagent sessions disappear from Control UI navigation and Codex app-server progress stops streaming live.

## Best Possible Solution

Define the Control UI UX for configured-agent subagent sessions, then restore their discoverability without reintroducing disk-only stores, and prove live Codex app-server tool progress through the session.tool/WebChat path while preserving hidden runtime-context filtering.

## Solution Assessment

Unclear because there is no patch. The likely fix is narrow, but maintainers need to decide how subagent sessions should surface after prior intentional picker hiding and how live progress should coexist with display:false privacy filtering.
