## Summary

Keep open: the underlying Tavily SecretRef fallback bug is valid, but this patch is not merge-ready because it turns every unavailable configured SecretRef into an ambient `TAVILY_API_KEY` fallback instead of preserving the configured credential boundary.

## What This Changes

The PR changes Tavily API-key normalization to use SecretInput inspect mode so unresolved configured secrets return `undefined` and allow the existing `TAVILY_API_KEY` environment fallback.

## Best Possible Solution

Use a guarded Tavily resolver matching the Firecrawl pattern: classify inspect-mode results, block nonmatching configured refs, add regression tests, and require redacted current-head runtime proof before merge.

Yes, source-reproducible. Current main calls strict Tavily SecretRef normalization before the documented `TAVILY_API_KEY` fallback, and the linked issue includes matching runtime logs; I did not run a live Tavily command in this read-only review.

## Solution Assessment

No. Inspect mode is the right primitive, but this implementation discards the `configured_unavailable` ref metadata needed to block unsafe ambient env fallback.
