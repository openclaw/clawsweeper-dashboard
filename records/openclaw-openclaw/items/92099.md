## Best Possible Solution

## Reproduction Assessment

Yes. Current main passes `event.prompt` into Active Memory message-mode recall unchanged, and the PR regression is source-reproducible because the hook contract and tests show `prompt` is the current turn while `messages` can be historical context.

## Solution Assessment

No. The truncation helper is a plausible part of the fix, but selecting the latest message from `event.messages` is not the right current-turn boundary; the safer fix is to cap `event.prompt` or add an explicit slim-intent hook field.
