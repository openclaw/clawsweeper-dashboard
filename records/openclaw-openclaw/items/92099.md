## Summary

## What This Changes

The branch adds an Active Memory `messageMaxChars` config/schema/docs field, truncates message-mode `buildQuery()` output, and changes recall/search input from `event.prompt` to the latest user entry in `event.messages`.

## Best Possible Solution

Use the current-turn source, either `event.prompt` with a maintainer-approved cap or an explicit slim-intent hook field, then add redacted real-runtime proof for the large-envelope path before merge.

## Reproduction Assessment

Yes, source-reproducible. Current main passes `event.prompt` into Active Memory message-mode recall unchanged, and current hook tests document `messages` as history while `prompt` is the latest turn.

## Solution Assessment

No. The truncation helper may be part of the fix, but selecting the latest message from `event.messages` is not the right current-turn contract; the safer path is to cap `event.prompt` or first add an explicit slim-intent hook field.
