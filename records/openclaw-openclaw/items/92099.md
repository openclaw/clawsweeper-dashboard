## Best Possible Solution

Use a current-turn source, either `event.prompt` with a maintainer-approved cap or an explicit slim-intent hook field, then prove the long Telegram/envelope path with redacted real behavior logs before merge.

## Solution Assessment

No. The truncation helper is a plausible part of the fix, but selecting the latest message from `event.messages` is the wrong layer for the current-turn contract; the safer fix is to cap `event.prompt` or first add an explicit slim-intent hook field.
