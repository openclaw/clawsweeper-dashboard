## What This Changes

The branch adds an Active Memory `messageMaxChars` config/schema/docs entry, truncates message-mode `buildQuery()`, and switches recall/search input from `event.prompt` to the latest user item in `event.messages`.

## Best Possible Solution

Use `event.prompt` or a maintainer-approved slim current-intent hook field as the current message source, define the cap/default contract, and prove the long-envelope path with redacted real logs before merge.

## Reproduction Assessment

Yes, source-reproducible. Current main passes `event.prompt` into Active Memory message-mode recall unchanged, and the PR regression is source-reproducible because hook tests document `messages` as history while `prompt` is the latest turn.

## Solution Assessment

No. The truncation helper is a plausible part of the fix, but selecting the latest message from `event.messages` is the wrong layer for the current-turn contract; the safer fix is to cap `event.prompt` or first add an explicit slim current-intent hook field.
