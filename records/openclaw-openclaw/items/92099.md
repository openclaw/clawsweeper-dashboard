## Summary

Keep open. The branch is not merge-ready because it derives Active Memory's message-mode query from historical hook messages instead of the current prompt, adds a new config contract that still needs maintainer agreement, lacks real after-fix runtime proof, and is currently conflicting against main.

## What This Changes

The PR adds an optional Active Memory `messageMaxChars` config/schema/docs field, truncates message-mode query text when configured, and changes recall/search input from `event.prompt` to the latest user entry in `event.messages`.

## Best Possible Solution

Use the current-turn source, either `event.prompt` with a maintainer-approved cap or a new slim-intent hook field, then add redacted real-runtime proof for the large-envelope path before merge.

## Reproduction Assessment

Yes. Current main passes `event.prompt` into Active Memory message-mode recall unchanged, and hook source/tests show `messages` is historical context while `prompt` carries the current user prompt.

## Solution Assessment

No. The truncation helper may be part of the fix, but selecting the latest message from `event.messages` is not the right current-turn contract; the safer path is to cap `event.prompt` or first add an explicit slim-intent hook field.
