## Summary

Keep open: current main still has no supported memory-core config that disables Dream Diary prose while preserving the machine dreaming pipeline, and this is a new public config/product decision rather than conservative cleanup.

## What This Changes

The issue requests a memory-core dreaming option to skip human-readable Dream Diary generation and `DREAMS.md` writes while keeping managed dreaming phase artifacts and promotion outputs enabled.

## Best Possible Solution

Add a memory-core-owned output toggle with schema, docs, UI/status expectations, and focused tests so narrative subagents and `DREAMS.md` writes are skipped while phase reports, events, promotion, and current defaults continue unchanged.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing contract. Source inspection confirms current main has no supported way to keep dreaming enabled while disabling Dream Diary prose.

## Solution Assessment

Yes on the owner boundary: memory-core plugin config is the right place if maintainers accept the feature. The exact option name and downstream Dreams UI/status behavior need product judgment before implementation.
