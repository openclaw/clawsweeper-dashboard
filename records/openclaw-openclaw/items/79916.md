## Summary

Keep open: current main still has no supported way to keep memory-core dreaming enabled while disabling Dream Diary prose, and adding that boundary is a new public config/product decision rather than conservative cleanup.

## What This Changes

Request a memory-core dreaming config option that preserves machine-readable dreaming phases and artifacts while skipping Dream Diary narrative subagents and `DREAMS.md` writes.

## Best Possible Solution

Add a memory-core-owned diary/output toggle with schema, docs, UI/status handling, and focused tests while keeping current Dream Diary behavior as the default.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing contract. Source, schema, docs, and tests confirm current main has no supported way to keep dreaming enabled while disabling Dream Diary prose.

## Solution Assessment

Yes on the owner boundary: memory-core plugin config is the right place if maintainers accept the feature. The exact option name and visible Dreams/doctor semantics need maintainer product judgment before implementation.
