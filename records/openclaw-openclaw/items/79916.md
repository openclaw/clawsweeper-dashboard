## Summary

Keep open: current main still has no supported Dream Diary or human-readable output opt-out, and adding one is a new public memory-core config/product decision rather than conservative cleanup.

## What This Changes

Request a memory-core dreaming option that keeps machine-readable dreaming phases and promotion artifacts enabled while disabling Dream Diary prose and `DREAMS.md` writes.

## Best Possible Solution

Add a memory-core-owned diary/output toggle with schema, docs, UI/status handling, and focused tests that skips narrative subagents and `DREAMS.md` writes while preserving phase reports, events, promotion, and current defaults.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing contract. Source, manifest, docs, and tests confirm current main has no supported way to keep dreaming enabled while disabling Dream Diary prose.

## Solution Assessment

Yes on the owner boundary: memory-core plugin config is the right place if maintainers accept the feature. The exact option name and Dreams UI/doctor semantics need product judgment before a fix PR is safe.
