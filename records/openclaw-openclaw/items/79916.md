This is a valid feature request, but it adds a new public memory-core config contract that needs maintainer product approval before implementation.

## What This Changes

Request to add memory-core dreaming config that keeps machine dreaming artifacts enabled while disabling Dream Diary narrative generation and `DREAMS.md` writes.

## Best Possible Solution

Add a memory-core-owned diary/output toggle with schema, docs, UI/status handling, and focused tests that skips narrative subagents and `DREAMS.md` writes while preserving phase reports, events, promotion, and current defaults.
