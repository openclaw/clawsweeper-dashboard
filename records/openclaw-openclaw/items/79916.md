## Summary

Keep open: current main still does not provide a supported Dream Diary or human-readable output opt-out, and the prior ClawSweeper review still matches the current source state. This is a new public memory-core configuration feature rather than a conservative cleanup close.

## What This Changes

Request a memory-core dreaming option that keeps machine-readable dreaming phases and promotion artifacts enabled while disabling generated Dream Diary prose and `DREAMS.md` writes.

## Best Possible Solution

Add a memory-core-owned diary/output toggle with schema, docs, UI/status handling, and focused tests that skips narrative subagents and `DREAMS.md` writes while leaving phase reports, events, promotion, and current defaults intact.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing contract. Source inspection confirms current main has no supported way to keep dreaming enabled while disabling Dream Diary prose.

## Solution Assessment

Yes on the owner boundary: memory-core plugin config is the right place if maintainers accept the feature. The exact naming and Dreams UI semantics need maintainer product judgment before implementation.
