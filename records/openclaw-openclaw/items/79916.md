## Summary

Keep open: current main does not provide a Dream Diary or human-readable output opt-out, and adding one is a new public memory-core config decision rather than conservative cleanup.

## What This Changes

The issue requests a memory-core dreaming option that disables Dream Diary prose while preserving machine-readable dreaming artifacts and promotion outputs.

## Best Possible Solution

Add a memory-core-owned diary/output toggle with schema, docs, UI/status handling, and focused tests that skips narrative subagents and `DREAMS.md` writes while leaving phase reports, events, promotion, and current defaults intact.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing contract. Source inspection confirms the current config surface has no supported way to keep dreaming enabled while disabling Dream Diary prose.

## Solution Assessment

Yes on the owner boundary: memory-core plugin config is the right place if maintainers accept the feature. The exact naming and Dreams UI semantics need maintainer product judgment before a fix PR should be queued.
