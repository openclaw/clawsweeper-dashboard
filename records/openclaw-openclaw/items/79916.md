## Summary

Keep open: current main still has no supported memory-core config boundary between machine dreaming artifacts and Dream Diary prose, and adding one is a new public config/product decision rather than conservative cleanup.

## What This Changes

Add a memory-core dreaming option that keeps phase reports, promotion artifacts, and machine-readable memory outputs enabled while disabling Dream Diary narrative generation and DREAMS.md writes.

## Best Possible Solution

Add a memory-core-owned diary/output toggle with schema, docs, status/UI handling, and focused tests that skips narrative subagents and DREAMS.md writes while leaving phase reports, events, promotion, and current defaults intact.

## Reproduction Assessment

Not applicable; this is a feature request rather than broken existing behavior. Source inspection confirms current main has no supported way to keep dreaming enabled while disabling Dream Diary prose.

## Solution Assessment

Yes on owner boundary if maintainers accept the feature; memory-core plugin config is the right surface. The exact option name and Dreams UI/status semantics need maintainer product judgment before a fix PR is safe.
