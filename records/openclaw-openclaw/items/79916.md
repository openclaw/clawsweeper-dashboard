## Summary

Keep open: current main still does not provide a supported way to keep memory-core dreaming enabled while disabling Dream Diary prose, and adding that switch is a new public config/product decision rather than conservative cleanup.

## What This Changes

Add a memory-core dreaming configuration option that disables human-readable Dream Diary narrative output while preserving machine-readable phase artifacts and promotion behavior.

## Best Possible Solution

Add a memory-core-owned diary/output toggle with schema, docs, UI/status handling, and focused tests that skips narrative subagents and `DREAMS.md` writes while preserving phase reports, events, promotions, and current defaults.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing contract. Source inspection of current main confirms there is no supported config path to keep dreaming enabled while disabling Dream Diary prose.

## Solution Assessment

Yes on the owner boundary: memory-core plugin config is the right place if maintainers accept the feature. The exact naming, defaults, and Dreams UI semantics need maintainer product judgment before implementation is safe.
