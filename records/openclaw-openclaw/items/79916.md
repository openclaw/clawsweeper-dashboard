## Summary

Keep open: current main still has no supported way to keep memory-core dreaming enabled while suppressing Dream Diary prose, and the requested split is a new public config/product decision rather than conservative cleanup.

## What This Changes

Request a memory-core dreaming config option that preserves machine phase/promotion artifacts while disabling Dream Diary narrative subagents and `DREAMS.md` writes.

## Best Possible Solution

Add a memory-core-owned diary/output toggle with schema, docs, UI/status handling, and focused tests that skips narrative subagents and `DREAMS.md` writes while leaving phase reports, events, promotion, and current defaults intact.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing contract. Source inspection gives high confidence that current main has no supported way to keep dreaming enabled while disabling Dream Diary prose.

## Solution Assessment

Yes on ownership boundary: memory-core plugin config is the right place if maintainers accept the feature. The exact option name and user-facing Dreams semantics need product approval before implementation.
