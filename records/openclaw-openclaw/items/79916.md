## Summary

Keep open: current main still has no supported way to disable Dream Diary prose while leaving memory-core dreaming’s machine pipeline enabled, and adding that split is a new public config/product decision rather than conservative cleanup.

## What This Changes

Request a memory-core dreaming option that suppresses Dream Diary prose and `DREAMS.md` writes while preserving phase artifacts, events, and promotions.

## Best Possible Solution

Add a memory-core-owned diary/output toggle with schema, docs, UI/status handling, and focused tests that skips narrative subagents and `DREAMS.md` writes while preserving machine artifacts and current defaults.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing contract. Source inspection against current main confirms there is no supported config gate for keeping dreaming enabled while disabling Dream Diary prose.

## Solution Assessment

Yes, if maintainers accept the product direction: memory-core plugin config is the right ownership boundary. The exact naming, docs, and Dreams UI semantics need maintainer approval before implementation should be queued.
