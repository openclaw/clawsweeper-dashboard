## Summary

Keep open: current main and v2026.6.10 still lack a supported machine-only memory-core dreaming mode, and the remaining work is a public plugin config/product decision rather than conservative cleanup.

## What This Changes

Request a memory-core dreaming option that keeps phase reports, promotion decisions, and machine-readable artifacts enabled while skipping Dream Diary narrative generation and DREAMS.md writes.

## Best Possible Solution

Approve one plugin-local Dream Diary output contract that defaults to current behavior, skips narrative subagents/fallback prose and managed DREAMS.md writes when explicitly disabled, preserves phase reports/signals/promotions, and documents the UI/status behavior.

## Reproduction Assessment

## Solution Assessment

Unclear until maintainers choose the public contract. A backward-compatible memory-core opt-out is plausible, but the exact key name and Dreams UI/status semantics should be approved before implementation.
