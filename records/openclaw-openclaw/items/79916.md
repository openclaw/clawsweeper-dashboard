## Summary

Keep open: current main still couples enabled memory-core dreaming to Dream Diary narrative generation and `DREAMS.md` writes, and the requested machine-only mode is a new public config/product decision rather than conservative cleanup.

## What This Changes

The issue requests a memory-core dreaming option that keeps machine-readable phase artifacts and promotion behavior enabled while disabling human-readable Dream Diary prose and `DREAMS.md` writes.

## Best Possible Solution

Add a memory-core-owned output toggle with schema, docs, UI/status handling, and focused tests that skips narrative subagents and `DREAMS.md` writes while preserving phase reports, events, promotion, and current defaults.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing contract. Source inspection gives high confidence that current main has no supported way to keep dreaming enabled while disabling Dream Diary prose.

## Solution Assessment

Unclear until maintainers decide the public config name and UI/status semantics. The narrow maintainable direction is a memory-core config/schema toggle that gates only narrative launch and `DREAMS.md` writes, not the machine dreaming pipeline.
