## Summary

Keep open: current main still couples enabled memory-core dreaming to Dream Diary narrative output, and the related open work covers adjacent prompt/timeout behavior rather than this machine-only opt-out. This is a new public config/product decision, not conservative cleanup.

## What This Changes

Request to add an optional memory-core dreaming config that suppresses Dream Diary prose while preserving machine-readable dreaming artifacts and promotion flow.

## Best Possible Solution

Add a memory-core-owned diary/narrative output toggle with schema, docs, UI/status handling, and focused tests that skips narrative subagents and `DREAMS.md` writes while preserving phase reports, events, promotion, and current defaults.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing contract. Source inspection confirms current main has no supported way to keep dreaming enabled while disabling Dream Diary prose.

## Solution Assessment

Yes on direction, but not implementation-ready for automation. Memory-core config is the right owner boundary, while naming, defaults, and Dreams UI/doctor semantics need maintainer approval first.
