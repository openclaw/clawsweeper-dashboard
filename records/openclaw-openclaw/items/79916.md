## Summary

Keep open: current main still has no supported way to keep memory-core dreaming machine artifacts enabled while disabling Dream Diary prose, and the adjacent language/prompt PR does not provide a machine-only output mode. This remains a new public config/product decision rather than conservative cleanup.

## What This Changes

Add a memory-core dreaming config option that disables Dream Diary prose while preserving machine-readable phase reports, events, and promotion artifacts.

## Best Possible Solution

Add a memory-core-owned, schema-backed output toggle that skips Dream Diary narrative subagents and DREAMS.md writes while preserving machine reports, events, promotion behavior, and current defaults.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing contract. Source inspection confirms current main has no supported way to keep dreaming enabled while disabling Dream Diary prose.

## Solution Assessment

Yes on owner boundary if maintainers accept the feature: memory-core config is the right place for this split. The exact field name, UI/status semantics, and docs need maintainer product judgment.
