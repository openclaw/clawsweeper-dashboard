## Summary

Keep open: current main still has no supported way to keep managed `memory-core` dreaming artifacts enabled while suppressing Dream Diary prose, and the requested split is a new public config/product decision rather than conservative cleanup.

## What This Changes

The issue requests a backward-compatible `memory-core` dreaming option to skip Dream Diary narrative generation and `DREAMS.md` writes while preserving machine-readable dreaming phases, promotion, events, and artifacts.

## Best Possible Solution

Add a memory-core-owned diary/output toggle with schema, docs, UI/status handling, and focused tests that skips narrative subagents and `DREAMS.md` writes while leaving phase reports, events, promotion flow, and current defaults intact.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing contract. Source inspection confirms current main has no supported way to enable managed dreaming while disabling Dream Diary prose.

## Solution Assessment

Yes on ownership boundary: a `memory-core` plugin config option is the maintainable place if maintainers accept the feature. The exact naming and Dreams UI/status semantics need product judgment before implementation.
