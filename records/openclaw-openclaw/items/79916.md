## Summary

Keep open: current main and the latest release still couple enabled memory-core dreaming to Dream Diary prose, and the request is a new public config feature that needs maintainer vocabulary and UI/status semantics rather than conservative cleanup.

## What This Changes

Request adds a memory-core dreaming option to disable Dream Diary prose while keeping machine dreaming phases, reports, events, and promotion enabled.

## Best Possible Solution

Add a memory-core-owned diary/output toggle with schema, docs, UI/status handling, and focused tests that skips narrative subagents and DREAMS.md writes while preserving phase reports, events, promotion, and current defaults.

## Reproduction Assessment

Not applicable; this is a feature/config request, not a broken existing contract. Source inspection has high confidence that current main has no supported config path to run memory-core dreaming while disabling Dream Diary prose.

## Solution Assessment

Yes on owner boundary if maintainers accept the feature: memory-core plugin config is the right place for this split. The exact option name, default semantics, and Dreams UI/doctor behavior need maintainer product approval before implementation.
