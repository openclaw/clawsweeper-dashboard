## Summary

Keep open: current main still has no supported way to keep memory-core dreaming machine artifacts enabled while disabling Dream Diary prose, and adding that switch is a new public config/product decision rather than conservative cleanup.

## What This Changes

Add a memory-core dreaming config option that skips human-readable Dream Diary generation and DREAMS.md writes while preserving managed dreaming phases, promotion, reports, and current defaults.

## Best Possible Solution

Add a memory-core-owned diary/output toggle with schema, docs, status/UI handling, and focused tests that skips narrative subagents and DREAMS.md writes while leaving phase reports, events, promotion, and existing defaults intact.

## Reproduction Assessment

Not applicable; this is a feature/config request rather than a broken existing contract. Source inspection gives high confidence that current main has no supported way to keep dreaming enabled while disabling Dream Diary prose.

## Solution Assessment

Yes on the owner boundary: memory-core plugin config is the maintainable place for this capability if maintainers accept it. The exact option name and Dreams UI/status semantics need product approval before implementation.
