Keep open: current main still couples enabled memory-core dreaming to Dream Diary narrative output, and the requested opt-out is a new public config/product decision rather than conservative cleanup.

## What This Changes

Add a memory-core dreaming option to keep machine phase artifacts and promotion flow enabled while suppressing Dream Diary prose and `DREAMS.md` writes.

## Best Possible Solution

Add a memory-core-owned diary/output toggle with schema, docs, UI/status handling, and focused tests that skips narrative subagents and `DREAMS.md` writes while leaving phase reports, events, promotion, and current defaults intact.

## Solution Assessment

Yes on owner boundary, if maintainers accept the feature: memory-core plugin config is the right surface. The exact option name and Dreams UI semantics need maintainer product judgment before implementation.
