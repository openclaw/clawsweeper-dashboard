## Summary

Keep open: current main still has no supported way to keep memory-core dreaming enabled while disabling Dream Diary prose, and the request adds a new public config option plus UI/status semantics that need maintainer product judgment.

## What This Changes

Request a memory-core dreaming config option that keeps machine-readable dreaming phases and artifacts enabled while disabling Dream Diary narrative generation and `DREAMS.md` writes.

## Best Possible Solution

Add a memory-core-owned diary/output toggle that defaults to today’s behavior, skips narrative subagents and `DREAMS.md` writes when disabled, preserves phase reports/events/promotion, and updates schema, docs, UI/status handling, and focused tests together.

## Solution Assessment
