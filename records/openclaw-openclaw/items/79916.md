## Summary

Keep open: current main still couples enabled memory-core dreaming to Dream Diary narrative output, and the requested split is a new public config surface that needs maintainer product approval.

## What This Changes

Request adds a memory-core dreaming option to disable Dream Diary narrative generation while keeping machine phase reports, events, and promotions enabled.

## Best Possible Solution

Add a memory-core-owned diary/output toggle with schema, docs, status/UI handling, and focused tests that skips narrative subagents and `DREAMS.md` writes while preserving phase reports, events, promotion, and current defaults.

## Reproduction Assessment

Not applicable; this is a feature request rather than a broken existing contract. Source review confirms current main has no supported way to keep dreaming enabled while disabling Dream Diary prose.

## Solution Assessment

Unclear until maintainers choose the public vocabulary and UX semantics. The memory-core plugin config boundary is the right place if the feature is accepted, but it should not be queued as a narrow bug fix yet.
