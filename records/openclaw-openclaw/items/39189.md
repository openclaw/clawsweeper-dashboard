## Summary

Keep this PR open, but do not merge it as-is: it targets plausible bugs, yet the branch still has two blocking correctness problems and no after-fix real behavior proof for the messaging scenario.

## What This Changes

The branch adds Anthropic usage-limit billing matchers, moves billing checks ahead of rate-limit handling, and enables final-reply text dedupe whenever a messaging tool sent text.

## Best Possible Solution

Land a narrowed replacement: add a precise Anthropic specified-API-usage-limits billing matcher with regression coverage, and solve duplicate final replies through route- or alias-aware matching backed by redacted real messaging proof.

## Reproduction Assessment

Partly yes. The billing misclassification is source-reproducible from current main's matcher and classifier order, while the multi-identifier duplicate-reply case still needs redacted live iMessage or alias evidence.

## Solution Assessment

No. The PR touches the right surfaces, but the billing matcher needs narrowing and the dedupe fix should preserve route-scoped or alias-aware suppression instead of global same-text suppression.
