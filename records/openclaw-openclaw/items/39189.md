## Summary

Keep open: the PR targets real failover and duplicate-reply surfaces, but the current branch is not mergeable as-is because it lacks after-fix real behavior proof and introduces two correctness regressions in billing classification and cross-target text dedupe.

## What This Changes

The PR adds Anthropic usage-limit billing matchers and reorders billing checks, then enables final-reply text dedupe whenever the messaging tool sent any text.

## Best Possible Solution

Replace this with a rebased, narrow fix that preserves route-scoped dedupe, handles same-person multi-identifier evidence without suppressing cross-target replies, and classifies only truly billing-specific Anthropic usage-limit messages with focused tests and redacted real behavior proof.

## Reproduction Assessment

Partly. The billing-classifier path is source-reproducible from the matcher/order, but the iMessage multi-handle duplicate requires real setup proof that is not present in the PR body or comments.

## Solution Assessment

No. The current diff fixes in the right area, but the broad usage-limit billing matcher and target-agnostic text dedupe are not the narrow maintainable solution.

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Restore route scoping before text dedupe:** `src/auto-reply/reply/agent-runner-payloads.ts:188-190`
  - body: This enables final-payload text dedupe whenever any messaging-tool text exists, even when the target comparison rejected the send as cross-target. Because `filterMessagingToolDuplicates` removes by text alone, a same-text send to another channel or user can incorrectly suppress the originating final reply.
  - confidence: 0.91
- **[P1] Narrow the usage-limit billing matcher:** `src/agents/pi-embedded-helpers/failover-matches.ts:53`
  - body: The new `/reached your.*usage limits?/i` billing pattern is broad enough to match ordinary resettable usage-limit wording. Since this PR checks billing before rate limits, those messages can move into long billing-disable behavior instead of the short rate-limit cooldown path.
  - confidence: 0.87
