## Summary

Keep open: the PR still targets a real billing-classification gap, but the branch is not mergeable because its billing matcher is too broad, its reply-dedupe change regresses cross-target same-text replies, and it still lacks after-fix real behavior proof.

## What This Changes

The branch adds Anthropic usage-limit billing patterns/tests, moves billing classification ahead of rate-limit handling, and changes final-reply dedupe to drop same text whenever the messaging tool sent text.

## Best Possible Solution

Land a narrow Anthropic specified-usage-limits billing matcher with regression coverage while keeping the route-scoped, alias-aware reply dedupe design already on main.

## Reproduction Assessment

Partly yes. The billing misclassification is source-reproducible from current main; the multi-handle duplicate-reply scenario still lacks live iMessage/channel-alias proof.

## Solution Assessment

No. The safer fix is a narrower billing matcher that preserves periodic rate-limit semantics plus the route-scoped dedupe design already on current main, not the PR's broad matcher and global same-text suppression.
