## Summary

Keep open: the PR targets a real source-level billing classification gap, but the current diff is not mergeable because the billing matcher is too broad and the reply dedupe change bypasses route matching that current main now preserves.

## What This Changes

The PR adds Anthropic usage-limit billing matchers, moves billing classification ahead of rate-limit handling, and changes messaging-tool reply dedupe to run whenever text was sent.

## Best Possible Solution

Split or rework the patch: add a precise Anthropic billing matcher for the specified-usage-limits/regain-access wording, and solve duplicate replies through route- or alias-aware channel suppression while preserving cross-target same-text delivery.

## Reproduction Assessment

Partly yes. The billing case is source-reproducible from current main because generic "usage limit" text is classified before billing; the multi-handle duplicate reply still needs channel-specific alias or live reproduction proof.

## Solution Assessment

No. The billing direction is valid but needs narrower matching, and the reply fix should stay behind route/alias-aware suppression instead of global same-text dedupe.
