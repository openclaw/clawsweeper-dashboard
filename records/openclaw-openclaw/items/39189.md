## What This Changes

The PR adds Anthropic usage-limit billing patterns, moves billing classification ahead of rate-limit handling, and enables text dedupe whenever the messaging tool recorded sent text.

## Best Possible Solution

Replace this with a narrow repair that classifies only high-confidence Anthropic billing text and adds alias- or route-aware same-person dedupe while preserving cross-target delivery, backed by redacted real iMessage proof and sanitized history.

## Reproduction Assessment

No for the full reported multi-handle iMessage scenario: the source path is plausible, but the PR has no redacted live proof. Yes for the patch regressions: current main source and tests show cross-target same-text replies are intentionally preserved and periodic usage limits remain retryable.

## Solution Assessment

No: this patch is not the best way to solve the issue because it removes route-scoped delivery protection and broadens billing classification. A safer fix should be provider-specific for billing and route- or alias-aware for same-person dedupe.
