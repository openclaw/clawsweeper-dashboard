Keep this PR open but do not merge it as-is: current main still shows the Anthropic billing classification gap, while the branch broadens usage-limit billing matching, weakens route-scoped final-reply dedupe, and lacks required after-fix real behavior proof.

## Best Possible Solution

## Solution Assessment

No. The PR touches the right surfaces, but the billing matcher needs narrowing and the dedupe fix should preserve route-scoped or alias-aware suppression instead of global same-text suppression.
