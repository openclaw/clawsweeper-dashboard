## Summary

This PR should stay open but not merge as-is. The billing half points at a real remaining gap in current main, but the proposed matcher is too broad, and the reply-dedupe half bypasses the existing same-target guard that current tests explicitly protect.

## What This Changes

The PR adds Anthropic usage-limit billing classification coverage and changes messaging-tool final-reply dedupe so sent text can suppress final replies even when the message-tool target differs from the originating target.

## Best Possible Solution

Split or rework the PR: add a narrow Anthropic billing matcher tied to the exact `specified API usage limits` plus `regain access` wording, and solve duplicate replies with identity-aware target matching through the existing channel suppression hook or target normalization while preserving cross-target same-text delivery tests.

## Reproduction Assessment

Partly yes. The billing misclassification has a high-confidence source-level reproduction through `classifyFailoverReason` on the exact Anthropic wording, while the multi-handle duplicate reply needs a live or channel-specific alias reproduction beyond the current read-only source inspection.

## Solution Assessment

No. The billing direction is useful but needs a narrower matcher, and the reply-dedupe fix should not bypass the same-target guard; identity-aware target normalization is the safer maintainable path.

  - confidence: 0.9
