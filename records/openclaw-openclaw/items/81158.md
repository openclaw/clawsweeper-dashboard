## What This Changes

The PR removes workspace-scoped current metadata snapshot reuse from unscoped plugin auto-enable and adds a regression test for a workspace Telegram channel shadow.

## Best Possible Solution

Land this focused resolver and test change after normal PR gates, keeping workspace-aware flows on explicit manifest registries instead of broad current-snapshot reuse.

## Solution Assessment

Yes. Removing the workspace-snapshot opt-in from the unscoped resolver is the narrowest maintainable fix because explicit workspace-aware callers can still pass the manifest registry they intentionally resolved.
