## Summary

Keep open: current main still allows unscoped plugin auto-enable to reuse workspace-scoped current metadata snapshots, and this PR is a focused fix with regression coverage. The remaining blocker is maintainer acceptance of the compatibility and plugin trust-boundary policy before merge.

## What This Changes

The PR removes two workspace-scoped current-snapshot opt-ins from plugin auto-enable manifest registry resolution and adds a regression test for a workspace Telegram shadow falling back to the bundled channel.

## Best Possible Solution

Land the narrow resolver and regression-test change if maintainers agree that unscoped auto-enable must only use unscoped/default snapshots; otherwise update any proven workspace-aware caller to pass an explicit manifestRegistry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: seed a workspace-scoped current metadata snapshot with a workspace channel shadow, then call applyPluginAutoEnable without manifestRegistry or workspaceDir; I did not execute the repro in this read-only review.

## Solution Assessment

Yes, subject to maintainer acceptance of the policy change. Removing the two resolver opt-ins is narrower than changing the snapshot helper or channel claimant ordering, while explicit workspace-aware flows can still pass manifestRegistry.
