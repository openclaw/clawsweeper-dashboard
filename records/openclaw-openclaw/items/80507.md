## Summary

Keep open. Current main and the latest release still have the dry-run pretty-text bug, and live GitHub shows an open closing PR that should be reviewed, landed, or replaced before this issue closes.

## What This Changes

The issue reports that `openclaw message send --dry-run` and the analogous poll path can print success-style output even though delivery is correctly skipped.

## Best Possible Solution

Land or replace the open formatter fix so send and poll dry runs format from `dryRun` before success output, while preserving the existing no-delivery guard.

## Reproduction Assessment

Yes. The current-main source path is high-confidence: send and poll can return `dryRun: true` with `handledBy: "core"`, while the formatter only checks `handledBy === "dry-run"` before success formatting.

## Solution Assessment

Yes. The narrow maintainable fix is to branch on the existing `dryRun` result flag before send/poll success formatting; the open linked PR takes that direction with focused formatter tests.
