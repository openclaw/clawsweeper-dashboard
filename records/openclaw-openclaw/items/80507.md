## Summary

Keep open. Current main still has the dry-run text bug: send/poll dry-run results carry `dryRun: true` while `handledBy` stays `core`/`plugin`, and an open closing PR already targets this issue.

## What This Changes

The issue requests that `openclaw message send --dry-run` and related poll output render an explicit dry-run line instead of success-style `Sent via` text.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/80604 after real CLI proof, so dry-run send/poll formatting reads dry-run state before success formatting while preserving the no-delivery guard.

## Reproduction Assessment

Yes. Current-main source shows a dry-run send can reach the formatter with `dryRun: true` and `handledBy: "core"`, which skips the dry-run branch and falls through to success-style text.

## Solution Assessment

Yes. The narrow maintainable fix is to check `result.dryRun` before send/poll success formatting, and the open linked PR is already aimed at that path.
