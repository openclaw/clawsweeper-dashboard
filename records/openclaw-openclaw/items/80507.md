Keep open. Current main and the latest release still have the dry-run text bug, and an open linked fix PR with closing syntax exists, so this issue should remain open until that PR lands or is replaced.

## What This Changes

The issue reports that `openclaw message send --dry-run` renders normal success text even though the send result carries `dryRun: true`.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/80604 so send and poll dry-runs are recognized before success formatting, with focused regression coverage and real CLI proof.

## Reproduction Assessment

Yes. Current-main source shows a dry-run send can reach the formatter with `dryRun: true` and `handledBy: "core"`, which skips the dry-run branch and reaches success formatting.

## Solution Assessment

Yes. The narrow maintainable fix is to key send/poll display on the explicit `dryRun` state before normal success formatting; the open linked PR is the current implementation candidate.
