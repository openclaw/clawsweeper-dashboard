## Summary

Keep open. Current main still has the dry-run text bug, and the latest release tag shows the same formatter path; an open closing PR already targets the issue, so this should stay open until that PR lands or is replaced.

## What This Changes

The issue asks for `openclaw message send --dry-run` and related poll dry-runs to print explicit dry-run text instead of success-style send/poll output.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/80604 so send and poll dry-runs are recognized before success formatting, with focused text-mode regression coverage.

## Reproduction Assessment

Yes. Current main can produce a send or poll result with `dryRun: true` while `handledBy` remains `core`, and the formatter checks only `handledBy` before reaching success output.

## Solution Assessment

Yes. The narrow maintainable fix is to key send/poll dry-run formatting off the stable `dryRun` state, or otherwise make the result contract expose dry-run before success rendering.
