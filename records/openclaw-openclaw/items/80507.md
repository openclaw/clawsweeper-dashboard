## Summary

Keep open. Current main and the latest release still have the dry-run text bug, and the remaining fix path is already represented by the open closing PR at https://github.com/openclaw/openclaw/pull/80604.

## What This Changes

The issue reports that `openclaw message send --dry-run` can print success-style pretty text even though delivery is skipped and JSON shows `dryRun: true`.

## Best Possible Solution

Land or replace the open formatter fix so send and poll dry-runs are detected from `dryRun` before success formatting, with focused regression coverage and contributor-owned real CLI proof on the PR.

## Reproduction Assessment

Yes. Current-main source shows a dry-run send or poll can reach `formatMessageCliText` with `dryRun: true` and `handledBy: "core"`, which skips the dry-run branch and reaches success formatting.

## Solution Assessment

Yes. The narrow maintainable fix is to make the formatter, or the result contract feeding it, recognize the existing `dryRun` state before rendering send/poll success output; the open linked PR is the current implementation candidate.
