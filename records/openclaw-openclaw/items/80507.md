## Summary

Keep open: the report matches current main source. Dry-run delivery is skipped, but pretty-text formatting still keys the dry-run message on `handledBy: "dry-run"`, while send/poll dry-run results carry `dryRun: true` with `handledBy: "core"`.

## What This Changes

The issue asks for `openclaw message send --dry-run` pretty-text output to show a dry-run preview instead of the normal sent-success line.

## Best Possible Solution

Update the message CLI text formatter to honor `dryRun` for send and poll before rendering success summaries, preserve JSON output and delivery behavior, and add focused formatter regression coverage.
