## Summary

Keep open. Current `origin/main` still has the dry-run text bug, and two open PRs already reference this issue with closing syntax, so the issue should stay open until one fix lands or is replaced.

## What This Changes

Fix misleading pretty-text output for `openclaw message send --dry-run` and `message poll --dry-run` so dry-runs do not render as successful real sends.

## Best Possible Solution

Land one narrow formatter/test fix that checks the stable `dryRun` flag before send/poll success formatting while preserving the existing no-delivery guard.

## Reproduction Assessment

Yes. Current-main source shows a dry-run send or poll can reach the formatter with `dryRun: true` and `handledBy: "core"`, skipping the dry-run branch and falling through to success formatting.

## Solution Assessment

Yes. The narrow maintainable fix is to key message send/poll pretty-text formatting on `result.dryRun` before rendering success output; the open linked PRs are implementation candidates for that direction.
