## What This Changes

The issue asks for `openclaw message send --dry-run` and poll dry-runs to print explicit dry-run text instead of success-style send output.

## Best Possible Solution

Land or replace the open formatter fix so dry-run send and poll results are recognized before success formatting, while preserving the existing no-delivery guard and focused regression coverage.

## Reproduction Assessment

Yes. The current-main source path is enough to reproduce with high confidence: dry-run send and poll results can carry `dryRun: true` while `handledBy` remains `core`, so the formatter skips its dry-run branch.

## Solution Assessment

Yes. The narrow maintainable solution is to key the formatter on `result.dryRun` before send/poll success formatting; the open linked PR takes that direction with focused tests.
