## What This Changes

The issue requests that `openclaw message send --dry-run` render explicit dry-run output instead of a success-style sent message.

## Best Possible Solution

Land a narrow formatter fix that branches on the existing `dryRun` result flag for send and poll, with focused tests for both actions.

## Reproduction Assessment

## Solution Assessment

Yes. The narrow formatter/test direction is the best current fix; changing outbound delivery semantics is unnecessary because `sendMessage()` already short-circuits dry-runs before delivery.
