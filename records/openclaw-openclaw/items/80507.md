## Summary

Keep open. Current `main` and the latest release still have the dry-run formatter mismatch, and live GitHub shows an open closing PR, so this issue should remain open until that PR lands or is replaced.

## What This Changes

Fix pretty-text CLI output for `openclaw message send --dry-run` and related poll dry-runs so they show dry-run text instead of success-style send text.

## Best Possible Solution

Review and land or replace the open linked fix so dry-run send and poll results are recognized before success-style formatting while preserving the no-delivery dry-run guard.

## Reproduction Assessment

Yes. The current-main source path is enough to reproduce with high confidence: dry-run sends carry `dryRun: true` separately from `handledBy: "core"`, so the formatter skips its dry-run branch and can render success-style output.

## Solution Assessment

Yes. The narrow maintainable fix is to key send and poll pretty-text formatting on `dryRun` before the success branches; the open linked PR takes that direction with focused formatter coverage.
