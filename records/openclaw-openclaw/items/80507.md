## What This Changes

The issue asks for `openclaw message send --dry-run` and related dry-run message output to show explicit `[dry-run]` text instead of the normal sent success line.

## Best Possible Solution

Land or replace the open formatter fix so send and poll dry-run results are recognized before success formatting, with focused regression coverage and the existing no-delivery guard preserved.

## Reproduction Assessment

Yes. Current-main source is enough to reproduce with high confidence: dry-run state reaches the formatter as `dryRun: true` while send/poll `handledBy` remains `core` or `plugin`, so the existing guard can miss it.

## Solution Assessment

Yes. The narrow maintainable fix is to check `result.dryRun` before send/poll success formatting, which is exactly the direction of the open linked PR.
