## Summary

Keep open. Current main still formats dry-run send and poll results by checking `handledBy === "dry-run"`, while the send path keeps `dryRun` separate from `handledBy`, and an open closing PR at https://github.com/openclaw/openclaw/pull/80604 should be reviewed, landed, or replaced before this issue closes.

## What This Changes

The issue requests distinct dry-run CLI text for `openclaw message send --dry-run` instead of the normal success-style `Sent via` output.

## Best Possible Solution

Land or replace the open closing PR so send and poll dry-run results are recognized before success formatting, with focused formatter coverage and the no-delivery guard preserved.

## Reproduction Assessment

Yes. The current-main source path is high-confidence: the CLI logs `formatMessageCliText(result)`, dry-run sends carry `dryRun: true` separately from `handledBy: "core"`, and the formatter only checks `handledBy === "dry-run"` before falling through to success output.

## Solution Assessment

Yes. The narrow maintainable fix is to key formatter behavior on dry-run state, or otherwise make the result contract expose dry-run before send/poll success formatting; the open linked PR follows that direction with focused tests.
