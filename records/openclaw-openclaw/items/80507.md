## Summary

Keep open. Current main still has the dry-run text bug: send and poll results can carry `dryRun: true` while `handledBy` remains `core`, so the formatter's `handledBy === "dry-run"` branch is skipped and the normal success text can render. Two open PRs already use closing syntax for this issue, so the issue should stay open until one lands or is replaced.

## What This Changes

The issue asks OpenClaw to render explicit dry-run output for `openclaw message send --dry-run` instead of success-style `Sent via` text.

## Best Possible Solution

Land or replace the linked fix with a narrow formatter or result-contract change that recognizes `dryRun` before success formatting for send and poll, with focused regression coverage and no change to the existing no-delivery guard.

## Reproduction Assessment

Yes. The current-main source path is enough to reproduce with high confidence: a dry-run send or poll can reach the formatter with `dryRun: true` and `handledBy: "core"`, so the dry-run text branch is skipped.

## Solution Assessment

Yes. The narrow maintainable solution is to render dry-run state from the existing `dryRun` flag before success formatting, or consistently map dry-run core results to the existing dry-run handledBy branch.
