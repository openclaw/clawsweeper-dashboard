## Summary

Keep open. Current main still has the reported dry-run text bug, and the open linked fix at https://github.com/openclaw/openclaw/pull/80604 uses closing syntax, so this issue should stay open until that PR lands or is replaced.

## What This Changes

The issue reports that `openclaw message send --dry-run` renders a success-style `Sent via` line even though the send result is a dry-run.

## Best Possible Solution

Review and land or replace the linked formatter fix so send and poll dry-runs render explicit `[dry-run]` text before success formatting, while preserving the existing no-delivery guard.

## Reproduction Assessment

Yes. Current-main source shows dry-run send and poll results can carry `dryRun: true` while `handledBy` remains `core`, so the formatter's dry-run branch is skipped before normal success formatting.

## Solution Assessment

Yes. The narrow maintainable solution is to key the pretty-text formatter off the existing `dryRun` result state before rendering send/poll success output; the open linked PR is the active implementation candidate.
