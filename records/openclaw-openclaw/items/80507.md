## What This Changes

The issue reports that `openclaw message send --dry-run` can print success-style output because pretty-text formatting checks `handledBy === "dry-run"` instead of the result `dryRun` flag.

## Best Possible Solution

Land or replace the canonical formatter/test fix so send and poll dry-run results branch on `dryRun` before success formatting, while preserving the existing no-delivery guard.

## Reproduction Assessment

Yes. Current-main source shows dry-run send/poll results can carry `dryRun: true` while `handledBy` remains `core` or `plugin`, and the reporter included live CLI output from v2026.5.7 showing the misleading success line.

## Solution Assessment

Yes. The narrow maintainable fix is to check `result.dryRun` before success formatting; the open linked PR takes that direction with focused formatter coverage.
