## Summary

Keep open. Current main still has the dry-run text bug, and an open closing PR already targets the issue, so this should stay open until that fix lands or is replaced.

## What This Changes

The issue asks for `openclaw message send --dry-run` and related dry-run message actions to render explicit dry-run text instead of successful-send output.

## Best Possible Solution

The CLI formatter should recognize `dryRun` before send/poll success formatting while preserving the existing no-delivery guard and adding focused regression coverage.

## Reproduction Assessment

Yes. Current main can produce a send result with `dryRun: true` and `handledBy: "core"`, which skips the formatter's only dry-run branch and reaches the normal send success text.

## Solution Assessment

Yes. The narrow maintainable fix is to key send/poll formatting off the stable dry-run state, or otherwise align the result contract so dry-runs are recognized before success output.
