## Summary

Keep open. Current main still carries `dryRun: true` separately from `handledBy: "core"` for core send/poll paths, so the pretty-text formatter misses its dry-run branch; the open linked fix PR should be reviewed or closed before this issue is closed.

## What This Changes

Requests that message send and poll dry-runs render a distinct dry-run line instead of normal success text.

## Best Possible Solution

Make send and poll dry-run results reach the formatter's dry-run branch, add focused text-mode regression coverage, and close this issue only after the linked PR or a replacement lands.

## Reproduction Assessment

Yes. The current source path is high-confidence: dry-run sends return `dryRun: true`, but `executeSendAction` reports `handledBy: "core"`, so `formatMessageCliText` falls through to success output.

## Solution Assessment

Yes. The narrow maintainable fix is to key send and poll dry-run formatting from the result contract or `dryRun: true` before success summaries; keep this issue open while the linked implementation is still unmerged.
