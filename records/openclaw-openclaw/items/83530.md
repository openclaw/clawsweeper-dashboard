## Summary

Keep open: current main only repairs Telegram final text when the shorter final has an ellipsis-style truncation marker, while this report is a stricter prefix-only final-payload mismatch with complete canonical assistant text available.

## What This Changes

Telegram final reply delivery can send a strict prefix payload even when run metadata or the session transcript has the complete assistant answer.

## Best Possible Solution

Add a narrow canonical-final reconciliation step for single text-only final payloads where the delivered payload is a strict prefix of the current turn's final assistant text, while preserving existing behavior for arbitrary longer stream previews.

## Reproduction Assessment

Yes, at source level. Current main only consults longer candidates for ellipsis-marked finals, and an existing test confirms no-marker shorter finals are delivered unchanged even when longer text exists.

## Solution Assessment

Yes, if kept narrow. The safest fix is not a general longer-preview override, but a canonical run/transcript-backed strict-prefix repair for single visible text finals with no media, reasoning, or error payloads.
