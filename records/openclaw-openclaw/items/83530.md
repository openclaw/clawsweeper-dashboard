## Summary

Keep open. Current main only reconciles transcript-backed Telegram finals when the shorter final has an ellipsis-style truncation marker, but this report is about markerless exact-prefix payloads, so the central failure mode is still plausible from source inspection.

## What This Changes

The issue asks Telegram final delivery to replace a stale prefix-only final payload with the canonical completed assistant text when the replacement is safely detectable.

## Best Possible Solution

Add a narrow guarded reconciliation at the canonical final/payload handoff so single text-only, non-error, non-reasoning markerless prefix payloads use the canonical final assistant text, with regression coverage for normal and progress Telegram final delivery.

## Reproduction Assessment

Yes, source-level. Current main returns and sends a markerless strict-prefix final unchanged because the transcript-backed longer-final path is gated on an ellipsis-style truncation marker.

## Solution Assessment

Yes, the proposed direction is the right narrow fix if it stays guarded to one text-only final payload and uses the canonical final assistant text; a broad chunk-size or Telegram-limit workaround would miss the actual handoff bug.
