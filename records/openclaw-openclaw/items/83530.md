## Summary

Current main already fixes the stale-prefix final payload path by preferring the structured current-turn final assistant answer over accumulated streamed preview text; that fix is shipped in v2026.5.18, while the separate Telegram HTML parse-mode issue remains open.

## What This Changes

The issue asks Telegram final delivery to use the complete canonical assistant output instead of an intermediate streaming prefix.

## Best Possible Solution

Keep the shipped canonical-final-answer payload assembly and leave the separate Telegram HTML parse-mode truncation work tracked by its own open issue.

## Reproduction Assessment

Yes, source-level. The merged regression models the reported prefix-only payload shape and proves payload assembly selects the complete structured final assistant answer instead.

## Solution Assessment

Yes. Fixing final payload assembly at the embedded-runner boundary is narrower and more maintainable than a Telegram-only prefix replacement because every channel receives the canonical final payload before delivery.
