## Summary

This should stay open: current main has protections for ellipsis-style Telegram preview truncation, but the reported clean prefix-only final payload is still not guarded in the normal agent-turn delivery path.

## What This Changes

The issue reports Telegram final delivery using a stale streaming prefix instead of the complete assistant final text already present in run metadata.

## Best Possible Solution

Add a narrow final-payload repair that promotes canonical final assistant text only for one non-error, non-reasoning, text-only payload that is a strict prefix of a longer canonical final, while preserving media, errors, reasoning, chunking, and multi-payload behavior.

## Reproduction Assessment

Yes at source level, but not via a live Telegram run in this review. Current code leaves a clean prefix-only final payload unchanged unless it has an ellipsis truncation marker, even when canonical final text exists in run metadata.

## Solution Assessment

Yes, the bounded strict-prefix repair direction is the best fit. It avoids changing Telegram limits or HTML formatting and should only apply when the canonical final text proves the delivered payload is stale.
