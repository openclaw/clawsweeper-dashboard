## Summary

Keep open. Current main only repairs transcript-backed Telegram final text when the final payload looks ellipsis-truncated; the reported strict-prefix, non-ellipsis final payload can still pass through unchanged, and there is no open PR tied to this issue.

## What This Changes

The issue requests Telegram final reply delivery to replace stale strict-prefix final payloads with the canonical final assistant text already recorded in run/session metadata.

## Best Possible Solution

Extend transcript-backed final selection with a conservative strict-prefix detector backed by current-turn transcript/run metadata and Telegram regression coverage, while preserving media, error, reasoning, and multi-payload behavior.

## Reproduction Assessment

Yes, source-level. A focused test can stub the current-turn transcript candidate to a full answer while final delivery receives a strict prefix without ellipsis; current main returns the prefix because the helper only recovers ellipsis-truncated finals.

## Solution Assessment

Yes, the reported mitigation shape is directionally right, but the maintainable fix should live in the shared transcript-backed final selection or Telegram final-delivery seam with strict safety checks and regression tests.
