## Summary

Keep this open: current main and the latest release still use the reported same-target foreground fence, and the only linked fix PRs are closed unmerged.

## What This Changes

The issue reports WhatsApp automatic foreground replies being silently suppressed for earlier concurrent group messages because the foreground reply fence is scoped per group target instead of per inbound message or participant.

## Best Possible Solution

Define a participant- or reply-lineage-aware foreground fence that isolates independent WhatsApp group replies while preserving direct-chat and same-participant freshness, then prove it with focused regression coverage and redacted live WhatsApp evidence.

## Reproduction Assessment

Yes at source level: two dispatches sharing the same group target can complete out of order, and the newer visible delivery causes the older generation to cancel before transport delivery. I did not run a live WhatsApp group reproduction in this read-only review.

## Solution Assessment

Not exactly as a broad per-message key: the fix belongs in the foreground fence identity, but it should preserve direct-chat and same-participant freshness rather than disabling supersession for every distinct message.
