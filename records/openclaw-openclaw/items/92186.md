## Summary

Keep open: current main and v2026.6.6 still key the foreground reply fence by group target, so a newer visible WhatsApp delivery can cancel an older completed same-group reply before transport delivery. The open linked PR is the active fix path, but it still needs maintainer review of the fence boundary and real WhatsApp proof.

## What This Changes

Fix silent suppression of older completed WhatsApp group automatic replies when concurrent mentions share a foreground reply fence.

## Best Possible Solution

Use a narrow participant- or reply-lineage-aware fence for group conversations while retaining target-level freshness where prompts genuinely supersede each other, then prove it with focused regression tests and a redacted live WhatsApp run.

## Reproduction Assessment

Yes. Source inspection shows two same-target foreground dispatches can share one fence, and a newer visible WhatsApp delivery can make the older completed reply return null before delivery.

## Solution Assessment

Unclear as proposed. Per-message isolation addresses the reported loss, but the safer fix is a narrower participant or reply-lineage boundary that preserves existing freshness for genuinely superseding prompts.
