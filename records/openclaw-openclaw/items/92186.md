## Summary

Keep this open: current main and v2026.6.9 still use a target-scoped foreground reply fence, so a newer visible WhatsApp group delivery can cancel an older completed reply for a different participant; both linked fix PRs are closed unmerged and no owner-approved fence boundary has replaced them.

## What This Changes

The issue reports silent WhatsApp group answer loss when automatic visible replies suppress an earlier completed concurrent reply in the same group target.

## Best Possible Solution

Define a participant- or reply-lineage-aware fence for group replies that preserves direct-chat and same-participant freshness, add an info-level cancellation diagnostic, and prove both independent delivery and stale suppression.

## Reproduction Assessment

Yes at source level: same-target foreground generations can complete out of order, and a newer visible WhatsApp delivery can cancel an older completed reply before transport delivery. I did not run a live WhatsApp group reproduction in this read-only review.

## Solution Assessment

No complete fix is merged yet. The right fix is in the foreground fence identity, but maintainers should choose the participant or reply-lineage boundary so the fix does not disable legitimate stale-reply suppression.
