## Summary

Keep open: the CJK-aware estimator direction is useful and the real-behavior proof is sufficient, but current PR head still underestimates recoverable CJK tool-result truncation during preemptive routing, so it can compact unnecessarily instead of using truncate-only recovery.

## What This Changes

The PR adds a CJK-aware token-estimator wrapper, routes compaction and preemptive estimates through it, adds focused estimator tests, and updates the changelog.

## Best Possible Solution

Keep the estimator wrapper, but make preemptive tool-result recovery use the same script-aware unit as overflow pressure and cover that route with a CJK-heavy regression test.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: use a CJK-heavy toolResult where weighted truncation recovery exceeds overflow plus buffer while raw chars/4 does not, then assert the route should be truncate_tool_results_only.

## Solution Assessment

No. The wrapper is the right narrow fix for the main compaction undercount, but preemptive route recovery must be script-aware before this is merge-ready.
