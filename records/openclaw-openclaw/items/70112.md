## Summary

Keep this PR open: current main still has the CJK-blind compaction estimator, and the closed linked bug points here as the canonical fix path. The patch is useful but still has a source-proven preemptive-routing defect because CJK-aware overflow is compared against raw tool-result recovery divided by four.

## What This Changes

The PR adds a CJK-aware token-estimation wrapper, routes agent compaction estimates through it, adds focused tests, and updates the changelog.

## Best Possible Solution

Land a rebased patch that keeps the CJK-aware estimator direction, routes all compaction estimates through it, and compares preemptive truncation recovery in the same weighted token units with focused regressions.

## Reproduction Assessment

Yes. Source inspection shows current main still uses the dependency `chars / 4` estimator in compaction, and PR head still underestimates CJK-heavy tool-result recovery by converting raw reducible characters with `/ 4`.

## Solution Assessment

No, not yet. The wrapper is the right narrow fix for the main undercount, but the preemptive truncation route must also compute recoverable tool-result tokens in CJK-aware units before this is merge-ready.
