This PR adds a CJK-aware token-estimation wrapper, routes agent compaction estimates through it, updates preemptive compaction accounting, adds focused regression tests, and adds a changelog fix entry.

## Best Possible Solution

Land a rebased patch that keeps the CJK-aware estimator direction, routes all compaction estimates through it, and compares preemptive tool-result recovery in weighted token units with focused CJK regressions.

## Solution Assessment

No, not yet. The wrapper is the right narrow fix for the main undercount, but the preemptive truncation route must compute recoverable tool-result tokens in CJK-aware units before this is merge-ready.
