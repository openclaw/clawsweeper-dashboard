## Summary

Keep open: the CJK-aware estimator is the right direction, but the current PR head is stale and incomplete because it drops current-main runtime-context stripping and makes preemptive truncation routing compare CJK-aware token overflow against raw character recovery.

## What This Changes

The branch adds a CJK-aware wrapper around `estimateTokens`, routes compaction/preemptive estimates through it, and adds focused compaction regression tests.

## Best Possible Solution

Rebase the CJK estimator work onto current main, preserve current sanitizers and pressure safeguards, make truncation-route recovery use consistent token/script-aware units, and land it with regression coverage plus a changelog entry.

## Reproduction Assessment

Yes, from source inspection. Current main still uses the upstream chars/4 estimator in compaction, and raw PR head `f78619b3` shows the stale sanitizer gap plus the token/character unit mismatch in the preemptive route.

## Solution Assessment

No. The wrapper is a maintainable fix direction, but this PR should not merge until it preserves current-main runtime-context/unwindowed safeguards and compares preemptive truncation recovery in consistent units.
