## Summary

Keep open: current main still has the CJK-blind compaction estimator, but this PR is not merge-ready because its head is stale against current runtime-context sanitization and still mixes CJK-aware overflow tokens with raw character truncation thresholds.

## What This Changes

The PR adds an `estimateTokensCjkAware` wrapper, routes compaction and pre-prompt token estimates through it, and adds CJK/ASCII/toolCall regression tests.

## Best Possible Solution

Rebase this work onto current main, preserve runtime-context stripping and current preemptive pressure behavior, make truncation-route comparisons token-aware or consistently script-aware, then land with focused regression tests and a changelog entry.

## Reproduction Assessment

Yes. A deterministic review reproduction is a CJK-heavy prompt/tool-result case where the PR makes `overflowTokens` script-aware while `shouldPreemptivelyCompactBeforePrompt()` still multiplies by four and compares against raw `maxReducibleChars`; comparing PR head `f78619b3` with current main also reproduces the stale runtime-context sanitizer regression.

## Solution Assessment

No. The CJK-aware estimator is the right direction, but the current branch is incomplete until it preserves current-main sanitization and compares preemptive truncation recovery in consistent units.
