## Summary

Keep open. The PR fixes the observable MS Teams win32 launcher bug and has focused tests/changelog, but the changed Windows process-launch path still diverges from current main's hardened browser-open contract, so it needs explicit maintainer review before merge.

## What This Changes

The PR changes MS Teams delegated OAuth launch to use `explorer.exe` on win32, accepts explorer exit code 1 as success, adds setup-surface regression tests, and updates the changelog.

## Best Possible Solution

Keep the MS Teams delegated OAuth fix, but route win32 URL opening through a plugin-safe shared browser-open seam or mirror the current `%SystemRoot%\System32\rundll32.exe url.dll,FileProtocolHandler` plus HTTP(S) normalization locally.

## Reproduction Assessment

Yes. Stub `process.platform` to `win32` and call `openDelegatedOAuthUrl`; current main spawns `xdg-open`, while the PR tests cover the intended Windows branch and exit-code behavior.

## Solution Assessment

No. The PR is a narrow functional fix, but the win32 launcher should be aligned with the current hardened browser-open contract or explicitly accepted by maintainers as a deliberate exception.
