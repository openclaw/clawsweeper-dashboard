## Summary

Keep open as the canonical scoped fix for the heartbeat privacy leak; current source, docs, Codex contracts, related PR review context, and the PR proof all support suppressing only explicit `message_tool` no-tool final replies while preserving fallback and notice paths.

## What This Changes

The PR adds a heartbeat-runner guard and regression test so explicit `message_tool` heartbeat final text without `heartbeat_respond` is treated as a silent ack instead of being delivered.

## Best Possible Solution

Land this scoped runner guard with the regression test, then close #94053 and supersede the duplicate fix PRs.

## Reproduction Assessment

Yes. Current main's runner source and #94053 logs show `message_tool_only` can still fall through to durable delivery for no-tool text payloads, and the PR body includes before/after runner proof.

## Solution Assessment

Yes. The heartbeat runner is the right boundary because it sees forced heartbeat-tool mode, explicit visibility config, reply payload metadata, exec-completion state, and the final durable delivery decision; broader suppression on all forced heartbeat-tool runs would break Codex fallback behavior.
