## Best Possible Solution

## Reproduction Assessment

Yes. Source inspection on current main shows forced `message_tool_only` heartbeat runs with non-empty no-tool payloads can still normalize and deliver text, and the linked issue includes matching v2026.6.8 Telegram logs; I did not run a live Telegram API repro.

## Solution Assessment

Yes. The heartbeat runner is the right boundary because it has the forced heartbeat-tool flag, explicit visibility config, reply metadata, exec-completion state, and final delivery decision; broader suppression would risk Codex fallback behavior.
