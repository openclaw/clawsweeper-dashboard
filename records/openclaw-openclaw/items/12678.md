## Summary

Keep open: current main and the latest release still have adjacent controls, but not the requested capability manifest, explicit grant/revoke state, default-deny pre-exposure filtering, cross-surface execution checks, or durable capability audit trail for skills/tools.

## What This Changes

The issue requests a capability-based permission model for skills and tools with default-deny handling for high-risk actions.

## Best Possible Solution

Keep this as the broad core/plugin-API security hardening umbrella, with narrower RFCs or PRs landing schema visibility first and later adding grants, filtering, execution checks, and durable audit records.

## Reproduction Assessment

## Solution Assessment

Unclear until maintainers decide the capability taxonomy, grant storage, and enforcement boundary. The issue’s phased schema, grant, filter, enforce, and audit shape is the maintainable direction if core owns it.
