## Summary

Keep open. ## What This Changes

Add a capability-based permission model for skills/tools with declared sensitive capabilities, explicit grant/revoke state, default-deny filtering, execution-boundary enforcement, and audit logs.

## Best Possible Solution

Keep this as a core/plugin-API security design item split into manifest schema/diagnostics, persisted grants/revocations, pre-model filtering after tool-policy resolution, execution-boundary checks, and durable audit receipts.
