## Summary

Current main has substantial adjacent controls, but it still lacks a unified manifest-declared capability system with persisted operator grants, consistent pre-exposure and execution-boundary enforcement, and durable capability decision records. The governance plugin and narrower related issues provide useful prior art, while the remaining core/plugin-API architecture needs explicit security and product decisions.

## What This Changes

Add capability-based authorization for skills and tools with manifest declarations, explicit grants and revocations, default-deny enforcement, and durable audit records.

## Best Possible Solution

Define one versioned capability decision contract spanning manifest declarations, canonical persisted grants, pre-model filtering, centralized execution-boundary authorization, and durable decision records, while leaving contextual trust scoring and dynamic governance policy to plugins.

## Reproduction Assessment

No high-confidence bug reproduction applies because this is a feature request. Source inspection does clearly verify the architectural gap: current skill metadata and tool policies do not express persisted, scoped capability grants.

## Solution Assessment

Yes at the architectural level: a single grant, enforcement, and audit model is more maintainable than adding further tool-name lists or relying only on hooks. The public contract, persistence model, compatibility behavior, and staged rollout still require maintainer decisions.
