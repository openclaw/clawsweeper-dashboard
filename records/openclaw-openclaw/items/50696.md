## What This Changes

The PR changes the Overview recent-session label fallback to prefer a non-key `label` before `displayName` or `key`.

## Best Possible Solution

Land label-first Overview recent-session rendering, ideally keeping it aligned with the shared session display resolver so Overview and the session selector preserve the same non-key fallback semantics.
