## What This Changes

The PR changes the Overview recent sessions renderer to prefer a non-key session label before falling back to displayName or key.

## Best Possible Solution

Yes for the code direction: the current one-line patch is a narrow fix for the observable fallback bug and now avoids labels that exactly duplicate the key. Reusing resolveSessionDisplayName would reduce drift, but it is not a correctness blocker for this PR.
