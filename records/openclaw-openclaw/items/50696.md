## Summary

Keep this PR open. Current main still renders Overview recent sessions with `displayName` before `label`, so the intended label-first UI fix is not implemented on main; the provided PR diff remains a relevant narrow fix, and I found no blocking correctness or security issue in the current proposed one-line change.

## What This Changes

This PR changes the Overview recent-session display fallback in `ui/src/ui/views/overview-cards.ts` to prefer a usable session `label` before `displayName` or `key`.

## Best Possible Solution

Land a label-first Overview recent-session display fix, ideally by reusing `resolveSessionDisplayName` or extracting a shared helper so Overview, the chat dropdown, trimming, key-equality, typed-prefix, and parsed fallback behavior stay aligned; add a focused render/unit test for `label > displayName > key` and the `label === key` fallback case.

## Reproduction Assessment

Yes. A static current-main check is enough: a `GatewaySessionRow` with `key`, `label: "General"`, and `displayName: "from:123456"` will render `displayName` first because `overview-cards.ts:247` evaluates `s.displayName || s.label || s.key`.

## Solution Assessment

Partly. The PR’s inline expression fixes the observable label-first bug and handles labels that duplicate the key, but the most maintainable final shape would share the existing session display helper and add Overview-specific coverage.
