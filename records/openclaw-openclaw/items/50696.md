## Summary

Keep open. Current main still has the reported Overview recent-session fallback order, and the PR is a narrow implementation candidate with no blocking review findings found in the one-line diff.

## What This Changes

This PR changes the Overview recent-session display fallback in `ui/src/ui/views/overview-cards.ts` to prefer a non-key session `label` before falling back to `displayName` or `key`.

## Best Possible Solution

Land a label-first Overview recent-session display fix, ideally by reusing `resolveSessionDisplayName` or extracting a shared helper so Overview and the chat/session dropdown stay aligned, with a focused regression test for label-over-displayName and label-equals-key fallback behavior.

## Reproduction Assessment

Yes. A static current-main check is enough: a `GatewaySessionRow` with `key`, `label: "General"`, and `displayName: "from:123456"` renders `displayName` first at `ui/src/ui/views/overview-cards.ts:247`.

## Solution Assessment

Partly. The PR’s expression fixes the observable bug and handles the prior `label === key` review case, but the more maintainable solution would share the existing display-name helper and add Overview-specific coverage.
