## Summary

Keep open: current main still renders Overview recent-session text with `displayName` before `label`, and this PR is a narrow fix for that UI bug. ## What This Changes

The PR changes the Overview recent-session card to prefer a session `label` over `displayName` or `key` when rendering the visible session text.

## Best Possible Solution

Land label-first Overview recent-session rendering, ideally staying aligned with `resolveSessionDisplayName` semantics, after redacted real behavior proof and focused validation are refreshed.

## Reproduction Assessment

Yes. Source inspection shows current main renders `displayName` before `label` in the Overview recent sessions list while valid session rows can contain both fields and the shared helper/tests establish label-first behavior.

## Solution Assessment

Yes. The PR direction is the narrow maintainable fix for the UI priority bug; using the shared resolver directly would reduce future drift, but the current diff handles the known `label === key` edge case and is not a blocking correctness issue.
