## Summary

Current main has mature opt-in agent tool sandbox backends and a public backend SDK, but it still does not implement the central request: sandboxing defaults off, native plugin runtime is not moved behind a process/IPC isolation boundary, and there are no tiered secure-by-default presets or per-plugin environment, filesystem, network, credential, and resource policies. The existing maintainer discussion also explicitly retains this as the canonical architecture tracker, so the item remains necessary and needs product, security, compatibility, and milestone decisions rather than one automated fix.

## What This Changes

Add secure-by-default tiered sandbox presets with cross-platform process isolation and per-plugin credential, filesystem, network, tool, and resource capability enforcement.

## Best Possible Solution

Keep this as the canonical owner-reviewed sandbox/isolation architecture tracker, then split it into narrow milestones for threat model and defaults, plugin process/IPC boundaries, manifest capability contracts, credential scoping, platform backend guarantees, resource controls, and tested upgrade migration while preserving the existing Docker, SSH, OpenShell, and backend-SDK groundwork.

## Risks / Open Questions

- Changing the default from off to an enforced preset could break existing plugins, tools, credentials, network access, setup flows, and upgrades without a staged compatibility and migration policy.
- A plugin process boundary and proxy SDK would change a large public and internal contract surface, including lifecycle, callback, streaming, error, performance, and shutdown semantics.
- The proposed Linux, macOS, and Windows mechanisms do not provide identical guarantees, so named presets could overstate security unless each platform's enforceable contract and fallback behavior are explicit.
- Credential filtering and manifest auto-generation are security-sensitive: permissive learning or fallback behavior could preserve ambient secret exposure, while fail-closed enforcement could make existing installations unusable.
