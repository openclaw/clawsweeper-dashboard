## Summary

Keep open. Current `main` still has useful opt-in tool sandbox backends and a plugin-facing sandbox backend seam, but the central request remains unresolved: secure-by-default tiered presets, native plugin/process isolation, IPC-proxied plugin execution, manifest-enforced env/fs/net/resource policy, credential scoping, and resource limits are not implemented.

## What This Changes

The issue requests a unified secure-by-default sandbox architecture with multi-platform support, tiered presets, plugin/process isolation, IPC-proxied SDK execution, manifest policy enforcement, credential scoping, and resource controls.

## Best Possible Solution

Keep this as the canonical owner-reviewed sandbox/isolation tracker, or split it into reviewed milestones for secure-by-default presets, plugin/process isolation, IPC/proxy SDK enforcement, manifest policy, credential scoping, resource controls, and migration docs.

## Reproduction Assessment

Not applicable as a normal bug reproduction. Source and docs give high-confidence feature-gap verification: sandboxing is optional tool isolation, native plugins remain in-process, and config/manifest contracts lack the requested preset and policy surfaces.

## Solution Assessment

No for a one-shot automated fix. The maintainable path is security/product design followed by staged changes across config/schema, plugin loader, SDK IPC/proxying, sandbox backends, manifest policy, credentials, resource limits, and migration tests.

## Security Review

Status: needs_attention

Summary: This security-sensitive feature gap remains open because current main lacks secure-by-default plugin/process isolation and manifest-enforced sandbox policy.

- **[medium] Native plugins still run in the Gateway process:** `docs/plugins/architecture.md:444`
  - body: The documented native plugin execution model keeps plugin code in-process with the Gateway, so plugin bugs or malicious code still share the Gateway trust boundary rather than a process or memory isolation boundary.
  - confidence: 0.94

  - confidence: 0.93

  - body: The native plugin manifest surface does not define per-plugin environment, filesystem, network, syscall, or resource requirements that could be enforced at a host boundary.
