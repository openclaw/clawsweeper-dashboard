## Summary

Keep open: current main has useful opt-in tool sandbox backends and a sandbox backend registry, but it still does not implement the central request for secure-by-default tiered presets, plugin/process isolation, IPC-proxied plugin execution, per-plugin env/fs/net/resource manifest policy, or default credential scoping.

## What This Changes

The issue requests a unified multi-platform sandbox architecture with tiered presets, plugin capability manifests, credential scoping, filesystem/network/resource controls, and a staged migration plan.

## Best Possible Solution

Keep this as the canonical owner-reviewed sandbox/isolation tracker, then split implementation into reviewed milestones for defaults, plugin process/IPC boundaries, manifest policy, credential scoping, resource controls, and migration compatibility.
