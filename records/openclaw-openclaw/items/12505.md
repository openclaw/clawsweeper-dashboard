## Summary

Keep open. Current main still provides optional agent tool sandbox backends and hardening, but it does not implement the central secure-by-default plugin/process isolation architecture; recent maintainer and ClawSweeper comments also already treat this as unresolved canonical sandbox work.

## What This Changes

Request secure-by-default tiered sandbox presets with multi-platform process, filesystem, network, credential, resource, and manifest-based policy enforcement for plugins and agent execution.

## Best Possible Solution

Keep this as the canonical owner-reviewed sandbox/isolation tracker unless maintainers split it into narrower milestones for defaults, plugin process boundaries, manifest policy, credential scoping, backend enforcement, and migration compatibility.

## Reproduction Assessment

Not applicable as a normal bug reproduction. There is a high-confidence verification path for the feature gap: inspect current config, schema, docs, plugin manifest, and plugin loader behavior to confirm sandboxing remains optional tool isolation while native plugins still run in-process.

## Solution Assessment

No for an automated one-shot implementation; the requested direction is security-relevant and broad. The maintainable path is maintainer design review followed by separate milestones with targeted config, plugin-loader, SDK, backend, and migration tests.

Status: needs_attention

Summary: This issue is security-sensitive, and current main still lacks the requested secure-by-default native-plugin isolation boundaries.

Concerns:

- **[medium] Native plugins still run in the Gateway process:** `docs/plugins/architecture.md:126`
  - body: The documented and implemented native plugin model loads plugin modules and calls register(api) in-process, so plugin code still shares the Gateway process trust boundary.
  - confidence: 0.94
- **[medium] Sandboxing remains opt-in instead of secure by default:** `src/agents/sandbox/config.ts:246`
  - body: Sandbox config resolution defaults to mode "off", and public docs describe sandboxing as optional tool isolation rather than a default preset system.

  - body: The native plugin manifest surface does not declare per-plugin environment, filesystem, network, syscall, or resource policy requirements for enforcement.
