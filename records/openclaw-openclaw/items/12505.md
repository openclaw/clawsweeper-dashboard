## What This Changes

The issue requests a unified built-in sandbox architecture with multi-platform backend support, named security presets, plugin/process isolation, IPC proxying, manifest-scoped env/fs/net/resource policies, and default credential scoping.

## Best Possible Solution

Keep this as the canonical owner-reviewed sandbox/isolation tracker unless maintainers split it into narrower milestones for preset defaults, plugin/process isolation, IPC/proxy SDK enforcement, manifest policy, credential scoping, resource controls, and migration guidance.
