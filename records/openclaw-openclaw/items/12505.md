## Summary

Keep open. Current main and the latest shipped release still provide optional tool sandbox backends, while the requested secure-by-default tiered presets, native plugin/process isolation, IPC-proxied plugin execution, manifest-enforced env/fs/net/resource policy, and default credential scoping remain unimplemented.

## What This Changes

Request a unified secure-by-default sandbox architecture with multi-platform backends, tiered presets, IPC-proxied plugin isolation, manifest policy, credential scoping, and resource limits.

## Best Possible Solution

Keep this as the canonical owner-reviewed sandbox/isolation tracker, or split it into reviewed milestones for presets, plugin isolation, IPC/proxy SDK enforcement, manifest policy, credential scoping, resource limits, and migration docs.

## Reproduction Assessment

Not applicable as a normal bug reproduction. Source and docs provide high-confidence feature-gap verification: sandboxing is optional tool isolation, native plugins remain in-process, and config/manifest contracts lack the requested preset and policy surfaces.

## Solution Assessment

No for a one-shot automated fix. The maintainable path is security/product design followed by staged changes across config/schema, plugin loader, SDK IPC/proxying, sandbox backends, manifest policy, credentials, resource limits, and migration tests.

## Security Review

Status: needs_attention

Summary: This security-sensitive feature gap remains open because current main lacks secure-by-default plugin/process isolation and manifest-enforced sandbox policy.

Concerns:

- **[medium] Native plugins still run in the Gateway process:** `docs/plugins/architecture.md:444`
  - body: The documented native plugin execution model keeps plugin code in-process with the Gateway, so plugin bugs or malicious code still share the Gateway trust boundary rather than a process or memory isolation boundary.
  - confidence: 0.94
- **[medium] Sandboxing remains opt-in instead of secure by default:** `src/agents/sandbox/config.ts:246`
  - body: Sandbox config resolution defaults to mode "off", and public docs describe sandboxing as optional tool isolation rather than a secure-by-default preset system.
  - confidence: 0.93

  - body: The native plugin manifest surface does not define per-plugin environment, filesystem, network, syscall, or resource requirements that could be enforced at a host boundary.
