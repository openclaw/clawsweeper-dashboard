## Summary

Keep open: current main lacks this new Gateway contract, but the PR branch still misses archived family session IDs and has no visible after-fix Gateway proof, so it needs revision rather than cleanup closure.

## What This Changes

The PR adds an optional `includeLineage` parameter to `sessions.resolve`, serializes lineage through Gateway RPC and the embedded stub, and updates docs, Swift protocol models, tests, and changelog.

## Best Possible Solution

Revise this into a narrow Gateway contract that resolves both current and archived family session IDs while preserving visibility and ambiguity rules, then require real Gateway output proof before merge.

## Reproduction Assessment

Yes for the PR defect by source inspection: create a row whose current `sessionId` is `sess-current` and whose `usageFamilySessionIds` includes `sess-old`, then call `sessions.resolve({ sessionId: "sess-old", includeLineage: true })`. The PR selector never includes that row, so the request returns `No session found`.

## Solution Assessment

No. Adding `includeLineage` to `sessions.resolve` is a plausible narrow surface, but this implementation must match archived family IDs and be coordinated with https://github.com/openclaw/openclaw/issues/79903 and https://github.com/openclaw/openclaw/pull/79970 before the public contract lands.
