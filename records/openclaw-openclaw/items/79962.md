## What This Changes

The PR adds an optional `includeLineage` flag to `sessions.resolve`, returns lineage through Gateway RPC and the embedded stub, and updates protocol docs, Swift models, and resolver tests.

## Best Possible Solution

Land a narrow versioned Gateway resolver surface only after archived family IDs participate in sessionId selection, ambiguity and visibility rules stay intact, and real Gateway or companion output proves the contract.

## Reproduction Assessment

Not applicable: this is a feature/API PR, not a broken existing contract. Source inspection verifies current main lacks `includeLineage`, and the patch finding is provable from the resolver selecting rows before lineage serialization.

## Solution Assessment

No. Adding `includeLineage` to `sessions.resolve` is a reasonable narrow direction, but this implementation should resolve archived family session IDs before it satisfies durable rotated-family discovery.
