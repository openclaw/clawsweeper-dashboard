## Summary

Keep open: the PR is a plausible additive Gateway feature, but the patch does not resolve predecessor sessionIds stored in usageFamilySessionIds, the external proof is test-only, and the public protocol shape still needs maintainer API review.

## What This Changes

The PR adds includeLineage to Gateway sessions.resolve, serializes lineage metadata through the Gateway and embedded stub, and updates Swift protocol models, docs, changelog, and focused tests.

## Best Possible Solution

Extend the sessionId match path to include usageFamilySessionIds while preserving existing visibility and ambiguity rules, add predecessor-id regression coverage, and get maintainer approval for the additive protocol shape.

## Reproduction Assessment

Not applicable as a current-main bug because this is a new feature request. The blocking patch defect is source-reproducible: a predecessor sessionId in usageFamilySessionIds is never included in the match set before lineage serialization.

## Solution Assessment

No. Adding includeLineage to sessions.resolve is a reasonable surface, but the implementation should resolve archived family ids as well as the current id before it can satisfy the rotated-lineage contract.
