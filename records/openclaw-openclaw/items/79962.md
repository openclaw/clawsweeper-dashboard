## Summary

Keep open: the feature is still absent from current main and the PR is a plausible narrow Gateway addition, but the patch has a correctness defect because archived family session IDs stored only in `usageFamilySessionIds` still cannot resolve to canonical lineage.

## What This Changes

Adds an opt-in `includeLineage` parameter to Gateway `sessions.resolve`, returning lineage through Gateway RPC and the embedded stub while updating Swift protocol models, docs, tests, and changelog.

## Best Possible Solution

Revise the Gateway contract so `sessions.resolve` can resolve both current and archived family session IDs with explicit ambiguity and visibility semantics, then require redacted real Gateway output proof before merge.

## Reproduction Assessment

Yes for the PR defect by source inspection: create a row with current `sessionId: "sess-current"` and `usageFamilySessionIds: ["sess-old", "sess-current"]`, then call `sessions.resolve({ sessionId: "sess-old", includeLineage: true })`. The PR selector never includes that row, so the request returns `No session found`.

## Solution Assessment

No. Adding `includeLineage` to `sessions.resolve` is a plausible narrow surface, but the implementation must match archived family IDs before selection and coordinate the public contract with https://github.com/openclaw/openclaw/issues/79903 and https://github.com/openclaw/openclaw/pull/79970.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Resolve archived family session IDs before returning lineage:** `src/gateway/sessions-resolve.ts:274`
  - body: `sessions.resolve({ sessionId, includeLineage: true })` only reaches this return after the row has already matched the current `entry.sessionId` or store key. After reset or rotation, predecessor IDs can live only in `usageFamilySessionIds`, so resolving one of those IDs returns `No session found` instead of the canonical key and lineage. Include family IDs in the match path and add a predecessor-ID regression test.
