## Summary

Keep open: the patch is a narrow plausible fix for the Windows duplicate scheduled-task `/Run` race, and I found no blocking code-review finding, but the external PR still lacks after-fix native Windows behavior proof.

## What This Changes

This PR adds a PowerShell `Get-ScheduledTask` running-state guard before the Windows scheduled-task self-restart helper retries `schtasks /Run`, plus focused tests and a changelog entry.

## Best Possible Solution

Land this patch or an equivalent current-main port after redacted native Windows restart proof shows one gateway instance/window remains, while preserving the fallback path and regression coverage.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes at source level: current main still has both the CLI `schtasks /Run` restart path and the self-restart helper retrying `/Run`, and the linked Windows report gives a concrete native `openclaw gateway restart` path.

## Solution Assessment

Is this the best way to solve the issue? Yes for the code direction: guarding the self-restart helper when the task is already running is a narrow fix that keeps config-watcher fallback behavior, but the PR is not merge-ready without real behavior proof.
