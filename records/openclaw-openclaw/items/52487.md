## Summary

Keep open: the PR is a narrow, plausible fix for the remaining Windows self-restart `/Run` race, but this external PR still lacks after-fix native Windows behavior proof, and current main does not already contain the proposed running-task guard.

## What This Changes

The PR adds a PowerShell `Get-ScheduledTask` running-state probe before the Windows self-restart helper retries `schtasks /Run`, plus focused tests and a changelog entry.

## Best Possible Solution

Land this patch or an equivalent current-main port after adding redacted native Windows restart proof that the patched flow leaves one gateway instance and preserves the existing fallback behavior.

## Reproduction Assessment

Yes, at source level: current main still has a self-restart helper that retries `schtasks /Run` while the CLI restart path can also start the scheduled task. I did not perform a live Windows Scheduled Task reproduction in this read-only review.

## Solution Assessment

Yes for the code direction: guarding the self-restart helper before retrying `/Run` is a narrow maintainable fix, and the refreshed patch addresses the earlier locale, quoting, and PowerShell profile review gaps. It is not merge-ready without real behavior proof.
