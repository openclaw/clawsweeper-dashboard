## Summary

Keep this PR open, but do not merge it as-is. Current main still has the Windows restart race that the PR targets, and the author addressed the earlier locale-sensitive `schtasks /Query` review by switching to `Get-ScheduledTask`; however, the current patch adds a PowerShell-in-batch probe that is not cmd-safe for task-name interpolation and runs without profile isolation.

## What This Changes

The PR adds a PowerShell `Get-ScheduledTask` running-state probe to `src/infra/windows-task-restart.ts` before the generated Windows scheduled-task self-restart helper retries `schtasks /Run`.

## Best Possible Solution

Keep the narrow Windows duplicate-start fix, but revise the generated helper to use a profile-free, noninteractive PowerShell state check and pass the task name through a cmd-safe mechanism or validate it consistently with the update restart helper. Add focused tests for guard ordering, default and custom task names, unsafe/custom task names, profile-free invocation, fallback, and cleanup before merging.

## Reproduction Assessment

Yes. The issue has a high-confidence code-level reproduction path: on Windows, `openclaw gateway restart` can trigger the CLI `/End` then `/Run` path while the shutdown self-restart helper independently retries `/Run`; current main still contains both paths.

## Solution Assessment

No, not as currently proposed. The direction is narrow and appropriate, but the implementation should harden the PowerShell probe against cmd parsing and profile side effects before it is the best maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Quote or validate the task name before the probe:** `src/infra/windows-task-restart.ts:38`
  - body: The new probe embeds `taskName` directly in a `.cmd` `powershell -Command` line and only doubles PowerShell single quotes. Values from `OPENCLAW_WINDOWS_TASK_NAME` containing cmd metacharacters or `%` expansion can be parsed before PowerShell sees them, so the guard can miss an already-running task or execute unintended batch text. Pass the value through a cmd-safe assignment or reject unsafe names before embedding it.
  - confidence: 0.88
- **[P2] Run the probe without PowerShell profiles:** `src/infra/windows-task-restart.ts:38`
  - body: `powershell -Command` can load user or system profiles before `Get-ScheduledTask`. Profile output containing `Running` can make `findstr` take the cleanup path when the task is not running, and slow or interactive profile code can block the helper before it reaches `/Run`. Add `-NoProfile` and preferably `-NonInteractive` for this generated restart probe.
  - confidence: 0.82
- **[P3] Add the required changelog entry:** `src/infra/windows-task-restart.ts:38`
  - body: This is a user-facing Windows restart fix, but the PR only changes `src/infra/windows-task-restart.ts` and does not add a `CHANGELOG.md` entry. Add a single-line fix entry with contributor credit before merge.
  - confidence: 0.78
