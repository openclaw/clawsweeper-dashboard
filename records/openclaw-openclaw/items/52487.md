## Summary

Keep open: current main still has the duplicate Windows scheduled-task start paths, and this PR targets the right bug, but the proposed probe needs command-generation hardening, profile-free PowerShell, a changelog entry, and a current-main port before merge.

## What This Changes

The PR adds a PowerShell `Get-ScheduledTask` running-state probe to the Windows self-restart `.cmd` helper before retrying `schtasks /Run`.

## Best Possible Solution

Land a current-main port that preserves restart logging, detects an already-running scheduled task through a locale-independent and profile-free path, passes or validates task names safely, and adds focused tests plus a changelog entry.

## Reproduction Assessment

Yes. Source inspection on current main shows the CLI restart path starts the scheduled task and the shutdown self-restart helper separately retries `schtasks /Run`, which matches the reported duplicate-window race.

## Solution Assessment

No, not as currently proposed. The running-state guard is the right narrow direction, but it should avoid cmd-unsafe task-name interpolation, run PowerShell without profiles, include tests and changelog, and be ported onto current main.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Quote or validate the task name before the probe:** `src/infra/windows-task-restart.ts:38`
  - body: The new `powershell -Command` line embeds `taskName` in a batch command after only PowerShell single-quote escaping. `%` expansion, double quotes, and other cmd parsing can happen before PowerShell sees the value, so custom task names can make the probe miss the running task or alter the generated command.
  - confidence: 0.88
- **[P2] Run the probe without PowerShell profiles:** `src/infra/windows-task-restart.ts:38`
  - body: `powershell -Command` can load user or system profiles before `Get-ScheduledTask`. Profile output containing `Running`, slow startup code, or interactive profile logic can make the helper skip or block the restart; add `-NoProfile` and preferably `-NonInteractive`.
  - confidence: 0.82
- **[P3] Add the required changelog entry:** `src/infra/windows-task-restart.ts:38`
  - body: This is a user-facing Windows restart fix, but the PR only changes the restart helper. Add a single-line `CHANGELOG.md` fix entry with appropriate contributor credit before merge.
  - confidence: 0.78
