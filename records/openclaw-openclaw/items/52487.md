## Summary

Keep open: current main still lacks the running-task guard in the Windows self-restart helper, while this focused PR addresses the duplicate `/Run` race with tests and sufficient native Windows after-fix proof. This should proceed through maintainer landing flow rather than cleanup close.

## What This Changes

The PR adds a PowerShell `Get-ScheduledTask` running-state guard before the Windows self-restart helper retries `schtasks /Run`, plus regression tests and a changelog entry.

## Best Possible Solution

Land this guard or an equivalent current-main patch while preserving the fallback helper and focused Windows restart coverage.

## Reproduction Assessment

Yes. Current main has both the CLI `/End` then `/Run` path and the self-restart helper's retrying `/Run` path without a running-task guard, and the PR discussion includes native Windows before/after proof; I did not run Windows locally in this checkout.

## Solution Assessment

Yes. Guarding the self-restart helper before retrying `/Run` is the narrowest maintainable fix for this race, and the refreshed diff addresses the earlier locale, task-name quoting, and PowerShell profile-startup concerns.
