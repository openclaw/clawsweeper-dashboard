## Summary

Keep open: the Windows duplicate-start race is still present on current main, and this PR is a useful narrow fix, but the current patch adds a PowerShell-in-batch probe that is not cmd-safe, runs profiles, and lacks the required changelog entry.

## What This Changes

The PR adds a running-state check to the Windows scheduled-task self-restart helper before retrying `schtasks /Run`.

## Best Possible Solution

Land a current-main port that preserves restart logging and fallback behavior while adding a locale-independent, profile-free, cmd-safe already-running guard with focused tests and a changelog entry.

## Reproduction Assessment

Yes. Source inspection on current main shows the CLI restart path starts the scheduled task and the shutdown self-restart helper separately retries `schtasks /Run`, which is the reported duplicate-window race.

## Solution Assessment

No, not as currently proposed. The running-state guard is the right narrow direction, but the generated command needs cmd-safe task-name passing, `-NoProfile`/noninteractive PowerShell invocation, regression tests, and changelog coverage.
