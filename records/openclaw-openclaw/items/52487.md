## Summary

Keep open. The Windows restart race still exists on current main, but this PR needs hardening before merge because the new probe introduces cmd-unsafe task-name interpolation, uses profile-loading PowerShell, and lacks the required changelog entry.

## What This Changes

Adds a PowerShell `Get-ScheduledTask` running-state check to the generated Windows scheduled-task self-restart batch helper before retrying `schtasks /Run`.

## Best Possible Solution

Keep the narrow duplicate-start guard, but implement it with cmd-safe task-name handling or validation, profile-free noninteractive PowerShell, focused helper tests, and a changelog entry against current main's logged handoff script.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: current main still has a CLI restart `/Run` path and a shutdown self-restart helper that retries `/Run` without checking whether the task is already running.

## Solution Assessment

No, not as currently proposed. The approach is narrow and useful, but the implementation should harden generated command execution and PowerShell invocation before it is the best maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Quote or validate the task name before the probe:** `src/infra/windows-task-restart.ts:38`
  - body: The new probe embeds `taskName` in a `.cmd` `powershell -Command` line and only doubles PowerShell single quotes. Values from `OPENCLAW_WINDOWS_TASK_NAME` containing cmd metacharacters or `%` expansion can be parsed before PowerShell sees them, so pass the value through a cmd-safe boundary or reject unsafe names before embedding it.
  - confidence: 0.88
- **[P2] Run the probe without PowerShell profiles:** `src/infra/windows-task-restart.ts:38`
  - body: `powershell -Command` can load user or system profiles before `Get-ScheduledTask`. Profile output containing `Running` can make `findstr` skip the restart, and slow or interactive profile code can block the helper before it reaches `/Run`; add `-NoProfile` and preferably `-NonInteractive`.
  - confidence: 0.82
- **[P3] Add the required changelog entry:** `src/infra/windows-task-restart.ts:38`
  - body: This is a user-facing Windows restart fix, but the PR only changes `src/infra/windows-task-restart.ts` and does not add a `CHANGELOG.md` entry. Add a single-line fix entry with contributor credit before merge.
  - confidence: 0.78
