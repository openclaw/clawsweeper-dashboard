## Summary

Keep open. The Windows duplicate-start race is still present on current main, and this PR is directionally useful, but the patch should not merge until the new PowerShell-in-batch probe is hardened and ported onto the current helper shape.

## What This Changes

The PR adds a PowerShell `Get-ScheduledTask` running-state check to the generated Windows scheduled-task self-restart helper before retrying `schtasks /Run`.

## Best Possible Solution

Keep the narrow duplicate-start fix, but pass or validate the task name in a cmd-safe way, run the probe with profile-free noninteractive PowerShell, add focused helper tests and a changelog entry, and port it onto current main's logged restart script.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: current main still has one `/Run` path in CLI restart and a second `/Run` retry path in the shutdown self-restart helper, matching the Windows report linked by the PR.

## Solution Assessment

No, not as currently proposed. The approach is narrow, but the implementation should harden command generation and PowerShell invocation before it is the best maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Quote or validate the task name before the probe:** `src/infra/windows-task-restart.ts:38`
  - body: The new probe embeds `taskName` directly in a `.cmd` `powershell -Command` line and only doubles PowerShell single quotes. Values from `OPENCLAW_WINDOWS_TASK_NAME` or profile-derived names containing cmd metacharacters or `%` expansion can be parsed before PowerShell sees them, so pass the value through a cmd-safe mechanism or reject unsafe names before embedding it.
  - confidence: 0.88
- **[P2] Run the probe without PowerShell profiles:** `src/infra/windows-task-restart.ts:38`
  - body: `powershell -Command` can load user or system profiles before `Get-ScheduledTask`. Profile output containing `Running` can make `findstr` skip the restart, and slow or interactive profile code can block the helper before it reaches `/Run`; add `-NoProfile` and preferably `-NonInteractive`.
  - confidence: 0.82
- **[P3] Add the required changelog entry:** `src/infra/windows-task-restart.ts:38`
  - body: This is a user-facing Windows restart fix, but the PR only changes `src/infra/windows-task-restart.ts` and does not add a `CHANGELOG.md` entry. Add a single-line fix entry with contributor credit before merge.
  - confidence: 0.78
