## Summary

Keep open. Current main still matches the reported lifecycle gaps: clean gateway close 1000 is formatted by the gateway but not treated as recoverable for waits, pending `agent.wait` results drop supervision, and stale sweep can still fail an unended run as lost even when the backing session row is still `running`.

## What This Changes

The issue requests more resilient subagent supervision after clean gateway closes, pending waits, or missing in-memory run context while the backing child session is still running.

## Best Possible Solution

Implement a focused agents-runtime fix in the wait classifier, subagent wait manager, and stale sweep path, with regression tests that preserve fail-closed behavior for missing, terminal, drifted, or truly stale backing sessions.

## Reproduction Assessment

Yes. There is a high-confidence source-level reproduction path on current main: close 1000 is not recoverable, pending waits return without a follow-up waiter, and a still-running backing session can fall through to the lost-context finalization path.

## Solution Assessment

Yes. The narrowest maintainable fix is to extend the existing recoverable-wait and subagent supervision seams rather than rewriting gateway lifecycle handling.
