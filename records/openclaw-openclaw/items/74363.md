## Summary

Keep open. Current main still matches the reported lifecycle gaps: clean gateway close 1000 is formatted by the gateway but not classified as recoverable for waits, `agent.wait` pending results return without a follow-up waiter, and the stale sweep path can still mark an unended run lost even when the backing session row is still `running`.

## What This Changes

The issue requests hardening subagent lifecycle recovery for clean gateway close 1000, pending `agent.wait`, and missing active run context while the backing session remains running.

## Best Possible Solution

Implement a focused agents-runtime fix in the wait classifier, subagent wait manager, and stale sweep reattach path, while preserving fail-closed behavior for missing, terminal, drifted, or truly stale backing sessions.

## Reproduction Assessment

Yes. There is a high-confidence source-level reproduction path on current main: clean close 1000 is not recoverable, pending waits return without re-arming, and a still-running backing session can fall through to the lost-context finalization path.

## Solution Assessment

Yes. The narrowest maintainable solution is to extend the existing recoverable-wait and subagent supervision seams rather than redesigning gateway lifecycle handling.
