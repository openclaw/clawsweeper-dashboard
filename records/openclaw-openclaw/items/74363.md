## Summary

Keep open. Current main still has the reported lifecycle gaps: clean gateway close 1000 is formatted as a normal close but is not classified as recoverable, pending waits drop supervision, and a stale unended run with a still-running backing session can fall through to lost-context completion.

## What This Changes

Subagent lifecycle recovery should avoid falsely terminalizing runs after clean gateway closure, pending waits, or missing in-memory context while the backing session is still running.

## Best Possible Solution

Land a focused agents-runtime fix that treats clean gateway close 1000 as recoverable, re-arms pending waits, and reattaches stale unended runs when the backing session is still running while preserving fail-closed behavior for missing, terminal, stale, or drifted sessions.

## Reproduction Assessment

Yes. There is a high-confidence source-level reproduction path on current main: a clean close 1000 is not recoverable, a pending wait returns without a follow-up waiter, and a stale unended run with a running backing session can reach the lost-context completion path.

## Solution Assessment

Yes. The narrowest maintainable fix is to extend the existing wait classifier, wait supervision retry seam, and sweeper reconciliation path rather than redesigning gateway lifecycle handling.
