## Summary

Keep open. Current `main` still has the three reported lifecycle gaps, and the two related open PRs are adjacent rather than complete fixes for clean close 1000 handling, pending wait rearming, and running-session reattachment.

## What This Changes

The issue asks OpenClaw to avoid falsely failing or losing subagent runs after clean gateway close 1000, `agent.wait` pending results, or missing in-memory context while the backing session is still running.

## Best Possible Solution

Implement a focused agents-runtime fix in the existing recoverable wait classifier, subagent wait supervision, and stale sweep reattach path while preserving fail-closed behavior for missing, terminal, stale, drifted, or truly orphaned sessions.

## Reproduction Assessment

Yes. There is a high-confidence source-level reproduction path on current `main`: close 1000 is not recoverable, pending waits return without rearming, and a present running backing session can still fall through to lost-context finalization.

## Solution Assessment

Yes. The narrowest maintainable repair is to extend the existing wait classifier, reuse the existing delayed wait retry pattern for pending waits, and add a running-session reattach/re-wait guard before lost-context finalization.
