## Summary

Keep open: current main and v2026.6.10 still contain the clean-close 1000, pending wait, and running-session lost-context gaps, while the related open PRs only cover adjacent lifecycle hardening or one lost-context facet.

## What This Changes

The issue asks OpenClaw to recover subagent lifecycle supervision after clean gateway close, pending waits, or missing in-memory run context when the backing session is still running.

## Best Possible Solution

Land or adapt a focused agents lifecycle fix that treats clean close 1000 as recoverable, re-arms pending waits, and reattaches or re-waits still-running backing sessions while preserving fail-closed behavior for missing, terminal, stale, or drifted sessions.

## Reproduction Assessment

Yes, at source level: current main lacks the clean-close 1000 recoverable pattern, returns from pending waits without re-arming, and lets a running backing session fall through to the lost-context fallback. I did not run a live gateway restart reproduction in this read-only review.

## Solution Assessment

No, current main is not the best solution because it only handles adjacent timeout, 1006 transport, and aborted-session edges. The maintainable fix should explicitly separate recoverable lifecycle states from permanent failures and add regression coverage around each branch.
