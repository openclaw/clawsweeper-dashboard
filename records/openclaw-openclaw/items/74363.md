## What This Changes

The issue asks OpenClaw to avoid falsely terminalizing subagent runs after clean gateway close, pending wait results, or missing in-memory run context while the backing session is still running.

## Best Possible Solution

Land one focused agents lifecycle fix that adds clean-close 1000 recovery, re-arms pending waits, reattaches or re-waits still-running backing sessions, and proves the path with targeted tests plus loopback gateway restart/reconnect proof.

## Reproduction Assessment

Yes at source level. Current main still lacks clean-close 1000 recovery, returns from pending waits without re-arming, and lets still-running backing sessions fall through to lost-context completion.

## Solution Assessment

No, current main is not the best end state. The maintainable fix should retry only explicitly recoverable lifecycle states while keeping missing, stale, terminal, auth, and permanent failures fail-closed.
