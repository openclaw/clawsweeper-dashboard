## What This Changes

The issue requests subagent lifecycle recovery for clean gateway close 1000, pending agent.wait results, and missing in-memory run context when the backing session store still says running.

## Best Possible Solution

Keep this issue open until the agents lifecycle path treats clean close 1000 as recoverable, re-arms pending waits, and reattaches or re-waits running backing sessions with focused regression coverage and loopback gateway proof.

## Reproduction Assessment

Yes at source level. Current main still lacks gateway close 1000 in the recoverable classifier, returns from pending waits without re-arming, and lets a running backing session fall through to lost-context completion.

## Solution Assessment

No, current main is not the best end state because it only handles some timeout, 1006, and aborted-session edges. The maintainable fix should keep permanent failures fail-closed while rearming or reattaching only for explicitly recoverable lifecycle states.
