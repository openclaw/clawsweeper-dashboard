## Summary

Keep open. Current `main` still matches the three reported lifecycle gaps: clean gateway close 1000 is not classified as recoverable, pending waits drop supervision, and a still-running backing session can fall through to the lost-context finalizer.

## What This Changes

The issue asks OpenClaw to stop falsely failing or losing subagent runs after clean gateway close, pending waits, or missing in-memory context while the backing session is still running.

## Best Possible Solution

Implement a focused agents-runtime fix in the existing wait classifier, subagent wait manager, and stale sweep reattach path while preserving fail-closed behavior for missing, terminal, stale, drifted, or truly orphaned backing sessions.

## Reproduction Assessment

Yes. There is a high-confidence source-level reproduction path on current `main`: close 1000 is not recoverable, pending waits return without re-arming, and a present `running` backing session can still fall through to lost-context finalization.

## Solution Assessment

Yes. The narrowest maintainable solution is to extend the existing recoverable-wait and subagent supervision seams rather than redesigning gateway lifecycle handling.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/74363 with a narrow agents-runtime patch. In `src/agents/run-wait.ts`, classify clean gateway close 1000 from `src/gateway/call.ts` as recoverable for agent waits without turning real gateway timeouts into recoverable errors. In `src/agents/subagent-registry-run-manager.ts`, keep supervising a subagent when `agent.wait` returns `pending` by scheduling a guarded follow-up wait similar to the recoverable-error branch. In `src/agents/subagent-registry.ts`, when the stale sweep has lost in-memory active context but the backing child session row is still present with `status: "running"`, reattach/re-wait via `resumeSubagentRun` or the existing wait manager instead of terminally failing as `subagent run lost active execution context`. Preserve existing fail-closed behavior for missing session rows, missing session ids, terminal statuses, stale unended runs, aborted-last-run recovery, and genuinely drifted/orphaned state. Add focused regression tests for close 1000 recovery, pending re-wait, running-session reattach, and the negative missing/terminal/stale cases. Update `CHANGELOG.md` under Unreleased Fixes with a user-facing single-line entry for this bug, crediting @nicolasdmolina if appropriate. Coordinate with related open PRs #74131 and #75462; do not broaden into gateway readiness redesign or unrelated subagent cleanup policy.
