## Summary

Keep open. Current main still matches the reported lifecycle gaps: clean gateway close 1000 is not recoverable for waits, pending wait results drop supervision, and the stale sweep can still mark a run lost even when the backing session row is present and running.

## What This Changes

The issue asks OpenClaw to keep subagent lifecycle supervision alive across clean gateway close, pending `agent.wait`, and still-running backing-session recovery edges.

## Best Possible Solution

Implement a focused agents-runtime fix in the wait classifier, subagent wait manager, and stale sweep path, with regression tests that preserve fail-closed behavior for missing, terminal, drifted, or truly stale backing sessions.

## Reproduction Assessment

Yes. There is a high-confidence source-level reproduction path on current main: close 1000 is not recoverable, pending waits return without a follow-up waiter, and an ordinary still-running backing session can fall through to the lost-context finalization path.

## Solution Assessment

Yes. The narrowest maintainable fix is to extend the existing recoverable-wait and subagent supervision seams rather than rewriting gateway lifecycle handling.
