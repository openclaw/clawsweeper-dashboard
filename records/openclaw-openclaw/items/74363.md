Fix the subagent lifecycle false-failure issue tracked by https://github.com/openclaw/openclaw/issues/74363. Keep the patch focused to agents runtime code: add `gateway closed (1000...)` to the recoverable wait classifier, make `waitForSubagentCompletion()` keep supervising when `agent.wait` returns `pending`, and make the stale active-run sweeper reattach/re-wait when the backing session store entry is still `status: "running"`. Preserve existing fail-closed behavior for missing sessions, terminal session states, stale/drifted entries, aborted-last-run recovery, and truly orphaned runs.
