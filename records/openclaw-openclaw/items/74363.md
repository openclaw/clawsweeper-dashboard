## Summary

Keep open. Current main still has the reported source-level gaps: clean gateway close 1000 is not classified as recoverable, pending waits return without rearming supervision, and stale active subagent runs can still be completed as lost context even when the backing session is still running.

## What This Changes

The issue asks OpenClaw to keep supervising subagent runs across clean gateway close, pending wait, and stale in-memory context cases instead of falsely marking them failed or lost.

## Best Possible Solution

Implement a focused agents-runtime fix in the existing wait classifier, subagent wait supervision, and stale sweep recovery path while preserving fail-closed behavior for missing, terminal, stale, drifted, or truly orphaned sessions.

## Reproduction Assessment

Yes, at source level. Current main still lacks 1000 clean-close recovery, returns from pending wait without scheduling another waiter, and can fail a stale active run even when its backing session entry remains running.

## Solution Assessment

Yes. The narrow maintainable repair is to extend the existing recoverable wait classifier, rearm the guarded wait retry for pending waits, and add a running-session reattach or re-wait guard before lost-context finalization.
