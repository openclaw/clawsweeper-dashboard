## Summary

Keep open. Current main still has the reported source-level gaps: clean gateway close 1000 is not classified as recoverable for subagent waits, pending waits return without rearming supervision, and a stale active run with a still-running backing session can still be finalized as lost context.

## What This Changes

The issue requests resilient subagent lifecycle recovery for clean gateway close, pending wait results, and missing in-memory run context while the backing session remains running.

## Best Possible Solution

Implement a focused agents-runtime fix in the existing wait classifier, subagent wait supervision, and stale sweep reattach path while preserving fail-closed behavior for missing, terminal, stale, drifted, or truly orphaned sessions.

## Reproduction Assessment

Yes, at source level. Current main still lacks 1000 clean-close recovery, returns from pending wait without scheduling another waiter, and can fail a stale active run even when its backing session entry remains running.

## Solution Assessment

Yes. The narrow maintainable repair is to extend the existing recoverable wait classifier, reuse the existing guarded delayed wait retry pattern for pending waits, and add a running-session reattach or re-wait guard before lost-context finalization.
