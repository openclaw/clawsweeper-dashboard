## Summary

Keep open. Current main still matches the reported lifecycle gaps: clean gateway close 1000 is formatted by the gateway but not classified as recoverable for waits, pending agent waits return without re-arming supervision, and stale subagent sweep can still finalize a run as lost while its backing session row is present and running.

## What This Changes

The issue requests subagent lifecycle recovery for clean gateway close 1000, pending agent.wait results, and stale sweep cases where the child session is still running.

## Best Possible Solution

Implement a focused agents-runtime fix in the wait classifier, subagent wait manager, and stale sweep reattach path, preserving fail-closed behavior for missing, terminal, drifted, or truly stale backing sessions.

## Reproduction Assessment

Yes. There is a high-confidence source-level reproduction path on current main: close 1000 is not recoverable, pending waits return without a follow-up waiter, and a present running backing session can fall through to lost-context finalization.

## Solution Assessment

Yes. The narrowest maintainable solution is to extend the existing recoverable-wait and subagent supervision seams, with regression tests, rather than redesigning gateway lifecycle handling.
