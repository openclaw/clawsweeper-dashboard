## Summary

Keep this open: current `main` and latest release `v2026.6.10` still miss the reported clean-close 1000 recovery, pending wait re-arm, and running backing-session reattach behavior. The open lifecycle hardening PR overlaps the area, but it is not merged, does not formally close this issue, and is currently conflicting.

## What This Changes

The issue requests source fixes so subagent lifecycle supervision treats clean gateway close, `agent.wait` pending results, and still-running backing sessions as recoverable instead of falsely failing or losing the run.

## Best Possible Solution

Land one focused agents lifecycle fix that adds clean-close 1000 recovery, re-arms pending waits, reattaches or re-waits still-running backing sessions, and proves the path with targeted tests plus loopback gateway restart/reconnect proof.

## Reproduction Assessment

Yes at source level. Current `main` lacks clean-close 1000 recovery, returns from pending waits without re-arming, and lets still-running backing sessions fall through to lost-context completion.

## Solution Assessment

No, current `main` is not the best end state. The maintainable fix should retry only explicitly recoverable lifecycle states while keeping missing, stale, terminal, or auth/permanent failures fail-closed.
