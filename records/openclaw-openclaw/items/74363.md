## Summary

Keep open. Current main still matches the reported subagent lifecycle gaps: clean gateway close code 1000 is not classified as recoverable, pending waits still return without rearming supervision, and a stale in-memory run can still be completed as lost even when the persisted child session is still running.

## What This Changes

The issue requests a subagent lifecycle fix so clean gateway close, pending wait results, and still-running backing sessions do not falsely terminalize child runs as failed or lost.

## Best Possible Solution

Implement a focused agents-runtime fix that treats clean gateway-close wait interruptions as recoverable for subagent supervision, schedules another guarded wait after pending results, and re-waits or reattaches when stale sweep finds a nonterminal running backing session while preserving missing, terminal, drifted, aborted, and true stale failure semantics.

## Reproduction Assessment

Yes. There is a high-confidence source-level reproduction path on current main: a clean gateway close 1000 reaches the wait classifier but is not recoverable, pending wait exits supervision, and a still-running session entry can fall through to the lost-context completion path.

## Solution Assessment

Yes. The narrowest maintainable solution is in the agents runtime wait, run-manager, and registry sweep paths with focused regression coverage, rather than a broad gateway lifecycle rewrite.
