## Summary

Keep open. Current main still matches the reported subagent lifecycle gaps: clean gateway close 1000 is formatted by the gateway but not classified recoverable for waits, `waitForSubagentCompletion()` still drops supervision on `pending`, and the stale sweeper can still finalize an unended run as lost when the backing session is present and still `running`.

## What This Changes

The issue requests hardened subagent lifecycle supervision so clean gateway close 1000, pending wait results, and missing in-memory context with a still-running backing session do not falsely finalize subagent runs as failed or lost.

## Best Possible Solution

Implement a focused agents-runtime fix that treats clean gateway-close wait interruptions as recoverable for subagent supervision, re-arms pending waits, and re-waits or reattaches when stale sweep finds a non-terminal running backing session, while preserving fail-closed behavior for missing, terminal, drifted, or stale sessions.

## Reproduction Assessment

Yes. There is a high-confidence source-level reproduction path on current main: close 1000 is not classified as recoverable, pending wait exits supervision, and a still-running session can fall through to the lost-context completion path.

## Solution Assessment

Yes. The narrowest maintainable solution is in the wait classifier, subagent wait manager, and stale sweep path with regression tests, not a broad gateway lifecycle rewrite.
