## Summary

Keep open. Current main still matches the reported subagent lifecycle gaps: clean gateway close 1000 is formatted by the gateway but not classified as recoverable for agent waits, pending subagent waits return without re-arming supervision, and the stale sweep path can still finalize a run as lost after finding a non-terminal backing session entry.

## What This Changes

The issue requests recoverable handling for clean gateway-close wait interruptions, pending `agent.wait` results, and missing in-memory subagent run context when the backing child session is still running.

## Best Possible Solution

Subagent lifecycle supervision should keep authoritative active child work alive across clean gateway reconnects and pending wait edges, while still failing closed for genuinely missing, terminal, stale, or inconsistent backing session state.

## Reproduction Assessment

Yes. There is a high-confidence source-level reproduction path on current main: clean close 1000 is not recoverable in the wait classifier, pending waits return without follow-up supervision, and a present running child-session entry can fall through to the lost-context path.

## Solution Assessment

Yes. A focused agents-runtime fix in the wait classifier, subagent wait manager, and stale sweep reconciliation path is the narrowest maintainable solution; a broader gateway lifecycle rewrite is not needed for the reported failure modes.
