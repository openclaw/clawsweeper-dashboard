- `impact:session-state`: The issue is about subagent run/session lifecycle state drifting from the backing session's actual running state.

## Summary

Keep open: current main and the latest release still contain the reported source-level gaps, and the open related lifecycle-hardening PR is unmerged and does not formally close this issue.

## What This Changes

The issue asks OpenClaw to stop falsely terminalizing subagent runs after clean gateway close, pending `agent.wait`, or missing in-memory run context while the backing session is still running.

## Best Possible Solution

Land one canonical subagent lifecycle fix that re-arms pending waits, handles clean reconnect closes as bounded recoverable interruptions where appropriate, and reattaches or re-waits still-running backing sessions while preserving fail-closed behavior for missing, terminal, stale, or drifted sessions.

## Reproduction Assessment

Yes for source-level reproduction: current main still lacks clean close 1000 recovery, returns on pending waits, and can lost-context a still-running backing session without `abortedLastRun`. I did not run a live gateway restart reproduction in this read-only review.

## Solution Assessment

Yes, the requested fix direction is the narrow maintainable repair, provided it is coordinated with the open related lifecycle-hardening PR. The safer implementation should keep retry windows bounded and retain fail-closed handling for missing, terminal, stale, or drifted sessions.
