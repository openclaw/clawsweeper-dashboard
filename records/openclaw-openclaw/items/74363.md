## Summary

Keep open. Current main still matches the reported subagent lifecycle failure modes, and the related open PRs are adjacent context rather than a merged or closing fix for the three exact gaps.

## What This Changes

The issue asks subagent lifecycle recovery to treat clean gateway close 1000 and pending waits as recoverable, and to reattach or re-wait when the backing child session is still running.

## Best Possible Solution

Implement a focused agents-runtime fix that treats clean gateway-close wait interruptions as recoverable for subagent supervision, re-arms pending waits, and re-waits or reattaches when stale sweep finds a non-terminal running backing session, while preserving fail-closed behavior for missing, terminal, drifted, or truly stale sessions.

## Reproduction Assessment

Yes. Current main gives a high-confidence source-level reproduction: clean close 1000 is not recoverable, pending wait exits supervision, and a still-running session can fall through to lost-context completion.

## Solution Assessment

Yes. The narrowest maintainable path is to fix the wait classifier, subagent wait manager, and stale sweep path with regression tests, not to rewrite gateway lifecycle handling or patch bundled dist files.
