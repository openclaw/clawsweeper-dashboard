## Summary

Keep this open: current main and v2026.6.8 still have the three reported source-level gaps, and the open related lifecycle-hardening PR only partially overlaps rather than closing the clean-close, pending-wait, and running-session reattach behavior.

## What This Changes

Fix subagent lifecycle recovery so clean gateway closes, pending agent waits, and missing in-memory context with a running backing session do not falsely fail or lose subagent runs.

## Best Possible Solution

Land or request a targeted agents lifecycle fix that covers all three gaps: classify clean close 1000 appropriately, re-arm pending waits, and reattach/re-wait when the backing session is still running, coordinated with the existing lifecycle-hardening PR.
