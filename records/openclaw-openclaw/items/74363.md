In `src/agents/run-wait.ts`, treat clean `gateway closed (1000...)` agent.wait failures as recoverable without making `gateway timeout` recoverable. In `src/agents/subagent-registry-run-manager.ts`, when `waitForAgentRun()` returns `pending`, keep lifecycle supervision attached by scheduling a bounded follow-up wait using the same entry identity and endedAt guards as the recoverable-error retry path. Add focused regression coverage for close 1000 recovery, pending wait rearming, and running-session reattachment, plus negative coverage for terminal/missing sessions. Add a single-line `CHANGELOG.md` fix entry because this is user-facing runtime behavior. Do not change plugin APIs, session store migrations, cleanup deletion policy, or unrelated subagent announce behavior. Re-check overlap with open PRs #74131 and #75462 before editing.
