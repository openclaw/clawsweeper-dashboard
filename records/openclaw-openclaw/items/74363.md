## Summary

Keep open. Current main still matches the reported lifecycle gaps: clean gateway close code 1000 is not recoverable for agent wait supervision, a pending subagent wait returns without rearming supervision, and stale active sweep can still mark a non-terminal running child session as lost. The related closed #73064 and open #74131 are useful context, but neither is a merged/current-main fix for this exact issue.

## What This Changes

This issue asks for subagent lifecycle recovery to treat clean gateway-close and pending/running-session wait edges as recoverable instead of terminal failed or lost outcomes.

## Best Possible Solution

Implement a focused src/agents fix that expands recoverable wait classification for clean gateway-close interruptions, rearms subagent completion waits after pending results, and re-waits or reattaches when stale sweep finds a still-running backing session while preserving terminal, missing-session, stale, and aborted-recovery semantics.

## Reproduction Assessment

Yes. There is a high-confidence source-level reproduction path on current main: the 1000 close string misses the recoverable classifier, pending wait returns without scheduling another waiter, and an ordinary running backing session can fall through to the lost-context terminal branch during stale sweep.

## Solution Assessment

Yes. The narrowest maintainable solution is to patch the existing wait classifier, subagent wait manager, and stale sweep behavior with regression tests, without broad gateway protocol or product-policy changes.
