## Summary

Keep open: current main and v2026.6.10 still contain the reported clean-close 1000, pending-wait no-rearm, and running-session lost-context gaps. The open lifecycle hardening PR overlaps the area, but it does not currently own all three requested branches.

## What This Changes

The issue asks OpenClaw to avoid false failed/lost subagent outcomes after clean gateway close, pending agent.wait results, or missing in-memory run context while the backing session is still running.

## Best Possible Solution

Land a focused agents lifecycle fix that treats clean close 1000 as recoverable, re-arms pending waits, and reattaches or re-waits still-running backing sessions with regression coverage and loopback gateway proof.

## Reproduction Assessment

Yes, at source level. Current main still lacks gateway close 1000 in the recoverable classifier, returns from pending waits without re-arming, and lets a running backing session fall through to lost-context completion.

## Solution Assessment

No, current main is not the best end state. The maintainable fix should reuse the existing retry/reattach seams while preserving fail-closed behavior for missing, terminal, stale, unauthorized, or drifted sessions.
