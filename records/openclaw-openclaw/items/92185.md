## Summary

This PR remains necessary because current main still reduces approved async gateway exec output to a whitespace-normalized 400-character tail. The patch cleanly separates agent-resume output from compact background notifications, propagates the existing capture-limit state through all runtime completion paths, and has no actionable correctness finding.

## What This Changes

The PR preserves captured multiline output in async gateway exec approval follow-ups, propagates capture-truncation metadata, and adds focused regression coverage.

## Best Possible Solution

Land the dedicated approval-resume formatter so agents receive all output retained by the existing exec capture layer, with an explicit notice only when that layer actually truncates, while leaving ordinary background notifications compact.

## Reproduction Assessment

Yes. Current main deterministically tails approval-completion output to 400 characters and normalizes whitespace, matching the linked issue's repeatable Discord scenario and the contributor's before/after runtime evidence.

## Solution Assessment

Yes. A dedicated approval-resume formatter is the narrowest maintainable fix because it restores the captured tool result without broadening ordinary channel notification output or introducing another configurable limit.
