## What This Changes

The PR replaces compact 400-character normalized async gateway approval follow-up output with full captured output, threads a truncation flag through exec outcomes, and adds multiline approval-output regression coverage.

## Best Possible Solution

Land the approval-completion formatter split after separating aggregate capture truncation from pending-buffer trimming and adding focused regression coverage for that boundary.

## Solution Assessment

No as written. The formatter boundary is the right fix shape, but the PR needs a separate aggregate truncation signal so it does not add false capture-limit notices when only the pending poll buffer was capped.
