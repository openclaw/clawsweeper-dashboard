## Summary

Keep open. Current main still has the gateway default mismatch, but this branch remains incomplete because the live async `sessions.list` path and agent-specific context/thinking defaults can still report mixed global metadata.

## What This Changes

The PR changes gateway session defaults to resolve the model from the selected/default agent, adds gateway regression tests, and also includes unrelated bundle-hook path canonicalization changes.

## Best Possible Solution

Land a rebased gateway-only fix that resolves model, context window, thinking levels, and thinking default from the selected/default agent for both sync and async `sessions.list`, with focused regression coverage.

## Reproduction Assessment

Yes, by source inspection. Current main still resolves `sessions.list.defaults` through global configured defaults, and the live RPC goes through the async list helper that does not pass the requested agent into default resolution.

## Solution Assessment

No. The agent-aware resolver direction is right, but this PR only partially applies it and leaves live async results plus context/thinking metadata inconsistent; it should be rebased, narrowed to gateway files, and completed.

## Review Findings

- **[P2] Thread agent defaults through the async list path:** `src/gateway/session-utils.ts:1286`

  - confidence: 0.9

  - confidence: 0.88
