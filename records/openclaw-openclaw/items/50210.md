## Summary

Keep open: this PR targets a real gateway bug, but the current branch is still incorrect against the live `sessions.list` path, mixes selected-agent model defaults with global metadata, lacks real behavior proof, is merge-conflicting, and includes unrelated hook-loader changes.

## What This Changes

The PR changes gateway session defaults to resolve the model from the default/requested agent and adds regression tests, while also adding an unrelated hook path canonicalization change.

## Best Possible Solution

Land a gateway-focused fix that derives model, context-token, thinking-level, and thinking-default metadata from one effective selected/default agent in both sync and async session listing, with focused regression coverage and redacted real Gateway or Control UI proof.

## Reproduction Assessment

Yes, by source inspection: current main's live `sessions.list` handler calls `listSessionsFromStoreAsync()`, which still builds unscoped defaults, and the Control UI renders its Default option from those defaults. I did not run a live Control UI repro in this read-only review.

## Solution Assessment

No. `resolveDefaultModelForAgent()` is the right direction for the model field, but this patch needs to cover the async path, keep context/thinking metadata scoped to the same agent, rebase cleanly, and remove the unrelated hook change.
