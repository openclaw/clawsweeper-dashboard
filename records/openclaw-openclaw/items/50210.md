## Summary

Keep this PR open: the linked Control UI default-model bug is still source-reproducible on current main, and this PR is the canonical thread for the closed reporter issue, but the branch is not merge-ready because it is conflicting, partial against the current async sessions path, and still lacks real behavior proof.

## What This Changes

The branch changes gateway session default resolution and regression tests, and also adds an unrelated bundle hook path canonicalization change under `src/hooks`.

## Best Possible Solution

Land a rebased gateway-focused fix that derives model, context-token, thinking-level, and thinking-default metadata from the same selected/default agent in both sync and async `sessions.list`, with focused tests and redacted real gateway or Control UI proof; split the hook change if it is still wanted.

## Reproduction Assessment

Yes. Current main still computes `sessions.list.defaults` without an agent-scoped resolver, the live RPC uses the async list helper, and the Control UI consumes those gateway defaults directly.

## Solution Assessment

No. Using `resolveDefaultModelForAgent()` is the right direction for the model field, but this PR does not cover the current async path or keep model, context, and thinking defaults scoped to the same agent, and it includes unrelated hook work.
