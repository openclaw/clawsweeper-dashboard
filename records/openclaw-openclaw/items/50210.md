## Summary

Keep open. The reported gateway/default-model bug is still valid on current main, and this PR moves in the right direction, but the patch is incomplete for current `sessions.list` behavior because the live async path still returns global defaults and the new agent-aware model default is not matched by agent-aware context/thinking defaults.

## What This Changes

The PR changes gateway session defaults to resolve the default or filtered agent model through `resolveDefaultModelForAgent`, adds session utility regression tests, and also adds bundle hook path canonicalization code and tests.

## Best Possible Solution

Gateway session defaults should derive model, context window, thinking options, and thinking default from one normalized selected/default agent source, with hook canonicalization handled separately if maintainers still want it.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: a config with only `agents.list[].model.primary` for the default or filtered agent still reaches global `getSessionDefaults()` on current main. I did not run the test suite in this read-only review.

## Solution Assessment

No. The PR chooses the right model resolver direction, but it does not update the live async list defaults or derive context/thinking metadata from the same selected agent, and it mixes in unrelated hook-loader work.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Pass the filtered agent into async defaults:** `src/gateway/session-utils.ts:1286`
  - body: The PR updates the synchronous list return to pass the filtered agent into `getSessionDefaults()`, but the live `sessions.list` RPC on current main calls `listSessionsFromStoreAsync()`. That async return still computes defaults without `opts.agentId`, so the actual gateway response can still advertise the repo/global default model for a filtered agent view.
  - confidence: 0.89
- **[P2] Resolve all default metadata from the selected agent:** `src/gateway/session-utils.ts:899-900`
  - body: This makes `defaults.model` come from the requested/default agent, but `contextTokens`, `thinkingLevels`, and `thinkingDefault` still come from the global/default metadata path. A selected agent can therefore advertise its model with the wrong context window or thinking default in Control UI/TUI; derive those fields from the same effective agent config.
  - confidence: 0.88
