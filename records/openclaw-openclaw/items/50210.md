## Summary

Keep this PR open. Current main still does not implement the intended `sessions.list.defaults` model behavior, and the PR is a useful source patch, but it is not merge-ready because it leaves default context metadata mixed with global defaults and includes an unrelated hook-loader path change.

## What This Changes

The PR changes gateway session defaults to resolve the model from the default or filtered agent, adds regression tests for default-agent and filtered-agent model defaults, and also adds hook path canonicalization code and tests.

## Best Possible Solution

Land a focused gateway session-defaults fix: normalize the selected/default agent once, resolve model, context tokens, thinking options, and thinking default from that same agent-aware source, pass filtered `agentId` through `listSessionsFromStore()`, and keep regressions for default-agent and filtered-agent defaults. Split the hook path canonicalization into a separate review if maintainers want that behavior.

## Reproduction Assessment

Yes. A source-level fixture with only `agents.list[].model.primary` and no `agents.defaults.model.primary` reproduces the current-main mismatch because `getSessionDefaults()` never consults the default agent; the PR's own tests encode that default-agent and filtered-agent path. The remaining PR bug is also reproducible from the diff by giving the selected agent a distinct `contextTokens` value or model context window.

## Solution Assessment

No. The PR uses the right model resolver direction, but the current patch is incomplete because related default metadata is not derived from the same selected agent and the branch includes unrelated hook-loader changes. A narrower rebase that keeps model/context/thinking defaults together is the safer maintainable fix.
