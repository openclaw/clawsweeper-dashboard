## Summary

Keep open: the underlying gateway bug is still present on current main, but this PR is incomplete for the live `sessions.list` path, mixes only part of the default metadata with agent-aware model resolution, and lacks required real behavior proof from a running setup.

## What This Changes

The branch changes gateway session defaults to resolve agent model defaults, adds gateway regression tests, and also adds bundle hook path canonicalization code/tests.

## Best Possible Solution

Land a rebased gateway-only fix that resolves model, context window, thinking levels, and thinking default from the selected/default agent for both sync and async `sessions.list`, with focused regression coverage.

## Reproduction Assessment

Yes, by source inspection. Current main still resolves `sessions.list.defaults` through global configured defaults, and the live RPC path uses the async list helper that does not pass the requested agent into default resolution.

## Solution Assessment

No. The agent-aware resolver direction is right, but this PR only partially applies it and leaves live async results plus context/thinking metadata inconsistent; the hook-loader change should be split from this gateway bug fix.
