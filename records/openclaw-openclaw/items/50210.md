## Summary

Keep open: the linked Control UI/gateway bug is still source-reproducible on current main, but this PR is now partial against the live async `sessions.list` path, misses selected-agent default metadata, lacks after-fix real behavior proof, and includes an unrelated hook-loader change.

## What This Changes

This PR changes gateway session defaults to resolve the default/requested agent model and adds regression tests, plus a separate bundle hook path canonicalization change.

## Best Possible Solution

Land a rebased gateway-only fix that resolves model, context tokens, thinking levels, and thinking defaults from the selected/default agent for both sync and async `sessions.list`, with focused regression coverage and redacted real behavior proof.

## Reproduction Assessment

Yes, source-reproducible: current main still reports `sessions.list.defaults` from global configured defaults, while the live RPC path goes through the async helper that does not pass the requested agent into default resolution.

## Solution Assessment

No. The agent-aware resolver is the right direction, but this branch only covers a stale/sync model path, leaves selected-agent context/thinking defaults inconsistent, and mixes in unrelated hook work.
