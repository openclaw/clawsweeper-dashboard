## Summary

## What This Changes

The PR changes gateway session default model resolution and tests, plus an unrelated bundle hook path canonicalization change under `src/hooks`.

## Best Possible Solution

Land a gateway-focused fix that derives model, context-token, thinking-level, and thinking-default metadata from one effective selected/default agent in both sync and async session listing, with focused regression tests and redacted real gateway or Control UI proof.

## Reproduction Assessment

Yes, by source inspection: current main's live `sessions.list` path calls `listSessionsFromStoreAsync()`, which still returns unscoped `getSessionDefaults()`, and the Control UI renders its Default option from those defaults. I did not run a live Control UI repro in this read-only review.

## Solution Assessment

No. `resolveDefaultModelForAgent()` is the right direction for the model field, but the PR needs to rebase to current main, cover the async path, keep context/thinking metadata scoped to the same agent, and split the unrelated hook change.
