## Summary

Keep open: current main still has the gateway defaults bug and the branch is useful, but the patch is incomplete for the live async `sessions.list` path, leaves default metadata mixed across agents, lacks the required changelog entry, and is currently dirty against main.

## What This Changes

The branch changes gateway session defaults to use agent-aware model resolution, adds session default regression tests, and adds bundle hook path canonicalization code/tests.

## Best Possible Solution

Gateway session defaults should derive model, context window, thinking levels, and thinking default from one selected/default agent source for both sync and async `sessions.list`, with unrelated hook work split out.

## Reproduction Assessment

Yes. Source inspection shows a config with only `agents.list[].model.primary` still reaches `getSessionDefaults()` on current main, which uses the global configured fallback, and the live RPC path uses the async helper that ignores `opts.agentId`.

## Solution Assessment

No. The resolver direction is right, but this patch is not the best complete fix because it misses the live async path, leaves context/thinking metadata on a different default source, and needs a changelog entry plus rebase/update against main.
