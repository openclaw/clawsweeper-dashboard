## Summary

Keep open: the gateway default-model bug is still present on current main, and this PR is useful source work, but it is not mergeable as-is because the live async `sessions.list` path and selected-agent context/thinking metadata remain incomplete.

## What This Changes

The PR changes gateway session defaults to resolve through the effective default/requested agent model, adds sync regression tests, and also adds bundle-hook path canonicalization code/tests.

## Best Possible Solution

Land a rebased gateway-only fix that derives model, context window, thinking levels, and thinking default from the selected/default agent for both sync and async `sessions.list`, with regression coverage and a changelog entry.

## Reproduction Assessment

Yes. Source inspection on current main shows `getSessionDefaults()` still uses the global configured fallback, and the live `sessions.list` RPC calls the async helper that does not pass `opts.agentId` into default resolution.

## Solution Assessment

No. The agent-aware resolver direction is right, but this PR only partially applies it and leaves live async results plus context/thinking metadata inconsistent; the safer fix is a rebased gateway-only repair.
