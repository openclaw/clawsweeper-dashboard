## Summary

Keep open: the linked Control UI default-model bug still appears valid, but this PR is not merge-ready because it is stale against the live async `sessions.list` path, leaves default metadata mixed, lacks real behavior proof, and includes unrelated hook-loader changes.

## What This Changes

The PR changes gateway session defaults to use the default/requested agent model and adds session-utils regression tests, while also adding a bundle hook path canonicalization change.

## Best Possible Solution

Land a rebased gateway-focused fix that resolves all session default metadata from the selected or default agent in both sync and async `sessions.list`, with focused regression tests and redacted real Control UI or gateway output proof.

## Reproduction Assessment

Yes, source-reproducible: current main still derives `sessions.list.defaults` globally while the Control UI consumes those defaults directly. The live async listing path also omits the requested agent when building defaults.

## Solution Assessment

No. The agent-aware model resolver is the right direction, but this branch is partial because it misses the async path, leaves context/thinking defaults mixed, and includes unrelated hook work.
