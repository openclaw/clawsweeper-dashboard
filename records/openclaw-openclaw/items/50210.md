## Summary

Keep open: the underlying gateway bug is still present on current main and the PR takes the right model-resolver direction, but it is not merge-ready because the live async `sessions.list` path and default context/thinking metadata are still not selected-agent aware, and the required changelog entry is missing.

## What This Changes

The PR changes gateway session defaults to use the default or requested agent model, adds gateway regression tests, and also adds bundle hook path canonicalization code/tests.

## Best Possible Solution

Derive model, context tokens, thinking options, and thinking default from one normalized selected/default agent source in both sync and async session-list helpers, with focused gateway tests and a single-line changelog entry.

## Reproduction Assessment

Yes, by source inspection. On current main, a config with only a default agent `model.primary` still reaches `getSessionDefaults()` through the live async `sessions.list` path, which resolves the global default model rather than the selected agent model.

## Solution Assessment

No. The PR uses the right resolver direction for the model field, but it does not update the live async defaults path or derive context/thinking metadata from the same selected agent, and it mixes in unrelated hook-loader work.
