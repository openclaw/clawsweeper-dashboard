## Summary

Keep open. Current main still exposes the gateway defaults mismatch that this PR targets, and the PR is useful source work, but the patch is not merge-ready because it makes the default model agent-aware while leaving related default metadata on the global path and also bundles an unrelated hook-loader change.

## What This Changes

The PR changes gateway session defaults to use the default or filtered agent model, adds regression tests for those model defaults, and also adds hook path canonicalization code and tests.

## Best Possible Solution

Land a focused gateway-only fix that normalizes the selected/default agent once, derives model, context tokens, thinking options, and thinking default from that same effective agent source, passes filtered `agentId` through sync and async session listing, and keeps hook canonicalization in a separate PR if maintainers want it.

## Reproduction Assessment

Yes. A source-level fixture with a single default agent whose `model.primary` is set only under `agents.list[]` reproduces the current-main mismatch because `getSessionDefaults()` never consults the default agent; a filtered `sessions.list` view has the same issue for `opts.agentId`.

## Solution Assessment

No. The PR uses the right model resolver direction, but it is incomplete because `contextTokens` and `thinkingDefault` remain derived from global/default metadata instead of the same selected agent, and the branch includes unrelated hook-loader changes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Resolve all session defaults from the selected agent:** `src/gateway/session-utils.ts:899-900`
  - body: This makes `defaults.model` come from the requested/default agent, but the rest of the default metadata still comes from the global path: `contextTokens` uses `cfg.agents?.defaults?.contextTokens`/model lookup, and `resolveGatewaySessionThinkingDefault` is still called without the selected `agentId`. A filtered agent or single configured agent can therefore advertise that agent's model with the wrong context window or thinking default in Control UI/TUI. Derive those fields from the same normalized agent config used for the default model.
  - confidence: 0.88
