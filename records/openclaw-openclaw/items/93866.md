Full review comments:

- **[P1] Preserve protected skill mounts over conflicting binds:** `src/agents/sandbox/workspace-mounts.ts:155-156`
  - body: This skips the managed read-only mount for `/workspace/skills` or `/workspace/.agents/skills` whenever a user bind claims the same path. Current main protects checked-in skill instructions by keeping that overlay authoritative, so filter or reject the custom bind instead of removing the protection.
  - confidence: 0.94
- **[P2] Reuse splitSandboxBindSpec for bind parsing:** `src/agents/sandbox/workspace-mounts.ts:133-138`
  - body: Splitting from the first colon treats a Windows drive-letter prefix as the host/container separator, so `C:\repo:/workspace/skills:rw` is not recognized and the duplicate target can still reach Docker. The existing `splitSandboxBindSpec` helper already handles this case.
  - confidence: 0.88
- **[P1] Align Docker mounts with the filesystem bridge:** `src/agents/sandbox/docker.ts:599`
  - body: Passing `skipContainerPaths` here makes Docker see the user bind at a protected skill path, but `buildSandboxFsMounts` still adds and prioritizes the `protectedSkill` mount. Tool path resolution can then disagree with the actual container mount table unless the same effective mount policy is shared.
  - confidence: 0.86
