## What This Changes

The PR hardens `openclaw/plugin-sdk/provider-tools` so unreadable plugin-provided tool schemas return diagnostics or remain unchanged instead of throwing during provider normalization and inspection.

## Best Possible Solution

Keep the guarded traversal centralized in `provider-tools`, then merge after maintainer SDK-contract confirmation and current-head validation; keep adjacent malformed-tool hardening PRs separate unless maintainers choose to batch them.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes from source: current main directly traverses plugin-provided schema objects and arrays and reads tool fields, so hostile getters or proxy array reads can throw before provider diagnostics are returned.

## Solution Assessment

Is this the best way to solve the issue? Yes, if maintainers accept the SDK contract: guarding traversal in the shared provider-tools helper is narrower than scattering provider-local catches and keeps DeepSeek, Gemini, and OpenAI-family hooks aligned.
