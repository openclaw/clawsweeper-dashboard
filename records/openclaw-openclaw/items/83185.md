## Summary

Keep open: current main mirrors Codex-native subagents only from Codex app-server notifications, and task maintenance explicitly preserves `codex-native` subagent rows that lack an OpenClaw child session. That makes the reported stale-running registry state plausible and distinct from the existing result-payload issue.

## What This Changes

The issue requests reliable finalization or reconciliation of Codex-native subagent task rows when completion/status events are missed.

## Best Possible Solution

Add Codex-owned stale-task recovery that checks app-server thread state when possible, and otherwise records a clear terminal/lost diagnostic without weakening ordinary subagent maintenance semantics.
