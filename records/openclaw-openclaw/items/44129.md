## Summary

Keep open. The linked WSL managed-skills bug still appears unresolved on current main, but this PR should not merge as-is because it broadens a security-sensitive skill containment guard, changes sandbox remove failure semantics, and carries a Feishu hunk that is stale against current main.

## What This Changes

The PR adds an `openclaw-managed` bypass to skill path containment, changes sandbox mutation helper permission/error handling, and serializes Feishu streaming-card `close()` calls.

## Best Possible Solution

Rework the skills fix so it distinguishes the WSL alias/realpath false positive from true managed-root escapes, preserving fail-closed containment for untrusted symlinked skill directories and `SKILL.md` files. Land sandbox permission preservation and Feishu close serialization only as separate current-main patches with tests if those bugs still need fixes.

## Reproduction Assessment

Unclear for the original WSL issue: #44051 gives a concrete and plausible path using a freshly installed ClawHub skill on WSL2, but this review did not execute WSL. The PR-introduced regressions are reproducible from diff and source inspection.

## Solution Assessment

No. The managed-skills bypass is broader than the WSL problem, the sandbox exception changes alter force-remove semantics, and the Feishu change needs a current-main rebase that preserves later final text and final-note behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep managed skills inside the configured root:** `src/agents/skills/workspace.ts:216-224`
  - body: This returns `candidateRealPath` for every `openclaw-managed` escape after `isPathInside()` fails. A symlinked managed skill directory or `SKILL.md` under the managed root would now be loaded from outside that root, so the WSL tolerance needs to distinguish the WSL alias false positive from a real escape.
  - confidence: 0.93
- **[P2] Only ignore missing targets on forced remove:** `src/agents/sandbox/fs-bridge-mutation-helper.ts:236`
  - body: Changing the remove handler from `FileNotFoundError` to `OSError` means the default forced remove path can report success for failures like `ENOTEMPTY`, `EACCES`, or `EPERM` while leaving the entry behind. Force should ignore missing targets only and propagate other removal failures.
  - confidence: 0.9
- **[P2] Preserve later final text during concurrent close:** `extensions/feishu/src/streaming-card.ts:326-329`
  - body: When `closePromise` already exists, a concurrent `close(finalText)` returns the first promise and drops the later final text. If the first close was started without the final response, the card never receives that content; merge or retain the latest non-empty final text before closing.
  - confidence: 0.78
