## Summary

Keep open. The linked WSL managed-skills problem is not implemented on current main, but this PR should not merge as-is because it weakens skill path containment for all managed skills, changes sandbox remove failure semantics, and carries unrelated Feishu changes that are stale against current main.

## What This Changes

The PR adds an `openclaw-managed` bypass to skill path containment, preserves sandbox write permissions while broadening several Python helper exception handlers, and serializes Feishu streaming-card `close()` calls.

## Best Possible Solution

Rework the WSL skills fix so it distinguishes the WSL alias/realpath false positive from a true managed-root escape, with regression coverage around managed skill symlinks. Keep sandbox permission preservation and Feishu close serialization as separate, current-main patches with focused tests if those issues are still needed.

Unclear for the original WSL issue: #44051 gives a plausible path using a freshly installed ClawHub skill on WSL2, but I did not live-run WSL in this read-only review. The PR-introduced regressions are high-confidence from source and patch inspection.

## Solution Assessment

No. The proposed managed-skills bypass is broader than the WSL problem, and the sandbox exception changes alter unrelated force-remove behavior; a narrower WSL-aware containment comparison or same-target proof is the safer direction.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Keep managed skills inside the configured root:** `src/agents/skills/workspace.ts:216-224`
  - body: This returns `candidateRealPath` for every `openclaw-managed` escape after `isPathInside()` fails. A symlinked skill directory or `SKILL.md` under the managed root would now be loaded from outside that root, so the WSL tolerance needs to distinguish the WSL alias false positive from a real escape.
  - confidence: 0.93
- **[P2] Only ignore missing targets on forced remove:** `src/agents/sandbox/fs-bridge-mutation-helper.ts:236`
  - body: Changing the remove handler from `FileNotFoundError` to `OSError` means the default forced remove path reports success for failures like `ENOTEMPTY`, `EACCES`, or `EPERM` while leaving the entry behind. Force should ignore missing targets only and propagate other removal failures.
  - confidence: 0.9
- **[P2] Preserve later final text during concurrent close:** `extensions/feishu/src/streaming-card.ts:326-329`
  - body: When `closePromise` already exists, a concurrent `close(finalText)` returns the first promise and drops the later final text. If the first close was started without the final response, the card never receives that content; merge or retain the latest non-empty final text before closing.
  - confidence: 0.78
