## What This Changes

The PR adds optional model availability metadata to `models.list`, updates generated Swift protocol output, and changes Android provider status surfaces to distinguish available, expiring, setup-required, and catalog-only provider rows.

## Best Possible Solution

Land the PR after CI and maintainer review if they accept the additive availability contract, keeping `available` optional and missing availability legacy-compatible.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction shape: current main has no `models.list` availability field and Android counts provider readiness from auth status alone. I did not run the emulator against current main in this read-only review.

## Solution Assessment

Yes, this is the best inspected fix shape: the gateway owns model usability knowledge and Android preserves older-gateway behavior by treating missing availability as unknown. An Android-only inference would keep catalog-only rows ambiguous.
