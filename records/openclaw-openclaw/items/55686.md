## Summary

Keep this issue open. Current main still produces the reported plain `/reasoning` option text, while Feishu's native button/card infrastructure exists only on lower-level presentation paths and is not wired into the normal directive reply flow.

## What This Changes

The issue requests rendering Feishu command option choices such as `/reasoning` `on/off/stream` as clickable interactive-card buttons instead of requiring typed replies.

## Best Possible Solution

Add a focused command-option presentation path for known directive/menu replies and have Feishu's normal inbound reply pipeline render those buttons as native cards through the existing quick-action callback path, while retaining readable text fallback for channels or contexts without button support.

## Reproduction Assessment

Yes. The current `/reasoning` status path and tests reproduce the issue as a plain `Options: on, off, stream.` text payload; Feishu's direct dispatcher then sends text/media rather than rendering presentation buttons.

## Solution Assessment

Yes. The best fix is to reuse the existing shared `presentation`/quick-action contract rather than adding Feishu-specific command parsing or folding in the separate #42754/#42908 callback-field work.
