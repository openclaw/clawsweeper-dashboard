## Summary

Keep open: current main still returns `/reasoning` choices as plain `Options:` text, while Feishu can already render presentation buttons but does not have a channel-owned transform for command option replies.

## What This Changes

Request Feishu command option replies such as `/reasoning` to render clickable interactive card buttons instead of requiring users to type option values.

## Best Possible Solution

Add a Feishu-owned option presentation path that converts trusted simple command-option replies into shared presentation button payloads for Feishu cards, while preserving the existing plain text fallback.

## Reproduction Assessment

Yes, source-level. `/reasoning` with no argument still builds a plain `Options: on, off, stream` text reply, current tests assert that line, and Feishu has no outbound transform equivalent to Slack's option-line control generation.

## Solution Assessment

Yes, directionally. The maintainable path is a Feishu plugin/channel transform or explicit presentation helper using the existing shared presentation contract, not Feishu-specific UI logic embedded in shared directive handling.
