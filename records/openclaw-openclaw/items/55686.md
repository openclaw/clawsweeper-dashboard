Keep open: current main and latest release still return `/reasoning` options as plain `Options:` text, while Feishu can already render presentation buttons and Slack has a channel transform for simple option lines. The adjacent Feishu callback-field PRs are useful context, but they address inbound card-action parsing rather than this outbound command-option UI.

## What This Changes

Request to render Feishu command option replies such as `/reasoning` as clickable card buttons instead of plain `Options:` text.

## Best Possible Solution

Add a Feishu-owned command-option presentation path that reuses existing reply transform/presentation contracts and Feishu card button envelopes, while preserving plain text fallback.

## Reproduction Assessment

Yes, source-level. `/reasoning` with no argument returns a plain `Options: on, off, stream` text reply, current tests assert that text, and Feishu has no reply transform to turn it into buttons.

## Solution Assessment

Yes, directionally. The maintainable path is a Feishu plugin transform or presentation helper using existing shared payload contracts, not hardcoding Feishu UI into core directives.
