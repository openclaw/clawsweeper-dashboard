## What This Changes

Request clickable Feishu card buttons for simple command choices such as `/reasoning` instead of requiring users to type option values.

## Best Possible Solution

Add a Feishu-owned presentation/normalization path that converts trusted simple command-option replies into card buttons carrying safe command values, while keeping the existing text output as fallback.

## Solution Assessment

Yes, directionally. The best fit is a Feishu adapter presentation path or shared option-presentation producer, not Feishu-specific UI code inside the shared directive handler.
