## Summary

Keep open: current main and the latest release still produce the no-argument `/reasoning` response as text-only options, while Feishu only renders clickable buttons when a reply carries typed `presentation` button blocks. The related native args-menu issue and open PR overlap, but they do not currently provide the Feishu command-status card/button path requested here.

## What This Changes

Requested Feishu command option replies, especially `/reasoning`, to render selectable options as clickable card buttons instead of plain text.

## Best Possible Solution

Define command-status option controls as typed presentation command actions, let Feishu render them through its existing card renderer, preserve the text fallback, and coordinate the shared behavior with https://github.com/openclaw/openclaw/issues/85088.

## Reproduction Assessment

Yes. Source inspection shows no-argument `/reasoning` returns only a text `Options:` reply, while Feishu clickable controls require a typed `presentation` button payload or native card data.

## Solution Assessment

Unclear as filed. Feishu card buttons are the right user outcome, but the maintainable fix should produce typed command-action presentation rather than adding a Feishu-only parser for `Options:` text.
