## Summary

Keep open: current main and v2026.6.8 still return the no-argument `/reasoning` status as text-only, while Feishu only gets clickable buttons when a reply carries typed `presentation` button blocks. The related native args-menu issue and PR overlap, but they do not replace the Feishu command-status card presentation request.

## What This Changes

Request to render Feishu command options such as `/reasoning` as clickable interactive card buttons instead of plain text choices.

## Best Possible Solution

Emit typed portable presentation buttons for selectable command-status replies, let Feishu render them through its existing card renderer, and preserve the current text fallback.

## Reproduction Assessment

Yes. Source inspection shows no-argument `/reasoning` returns only a text `Options:` reply, while Feishu renders clickable controls only when a payload carries typed `presentation` button blocks.

## Solution Assessment

Yes, if solved at the shared typed-presentation boundary. A Feishu-only parser that infers buttons from `Options:` text or an args-menu-only change would be the wrong layer and would conflict with the repository's typed action guidance.
