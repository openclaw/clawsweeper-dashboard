## Summary

Keep open: current main and v2026.6.10 still return no-argument `/reasoning` as a text-only `Options:` reply, while Feishu can render typed presentation buttons only when the reply carries a `presentation` button block. The related native args-menu issue and open PR overlap, but they do not currently give Feishu a command-status card/button path.

## What This Changes

The issue requests Feishu command options such as `/reasoning` to render as clickable interactive-card buttons instead of plain text choices.

## Best Possible Solution

Define the command-status UX at the shared typed-presentation boundary, then let Feishu render those command-action buttons through its existing card renderer while preserving the current text fallback and coordinating with https://github.com/openclaw/openclaw/issues/85088.

## Reproduction Assessment

Yes. Source inspection shows no-argument `/reasoning` returns only a text `Options:` reply, while Feishu clickable controls require a typed `presentation` button payload or native card data.

## Solution Assessment

Unclear as filed. Feishu card buttons are the right user outcome, but the maintainable fix should produce typed command-action presentation rather than adding a Feishu-only parser for `Options:` text.
