## Summary

Keep open: current main still has the Feishu-specific UX gap. `/reasoning` has shared `argsMenu: "auto"`, but Feishu outbound does not normalize the plain `Options:` reply into presentation buttons the way Slack does, and the latest release still has the same behavior.

## What This Changes

The issue requests Feishu clickable card buttons for command option replies such as `/reasoning` instead of requiring users to type `on`, `off`, or `stream`.

## Best Possible Solution

Add a Feishu-owned outbound normalization path that turns safe command option replies into `MessagePresentation` buttons, while preserving the existing text fallback and keeping broader callback-field work separate.

## Reproduction Assessment

Yes, source-level. Current main still emits a plain `/reasoning` `Options:` text line, and Feishu outbound has no `normalizePayload` consumer comparable to Slack's options-line transform.

## Solution Assessment

Yes directionally. The narrow maintainable fix is a Feishu adapter normalization/presentation path, not Feishu-specific UI logic inside the shared directive handler.
