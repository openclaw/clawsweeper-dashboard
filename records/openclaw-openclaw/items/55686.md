## Summary

Keep open: current main and the latest release still make the no-argument `/reasoning` status reply text-only in Feishu. Feishu can render typed presentation buttons now, but the command-status path does not emit the presentation payload needed for clickable card options.

## What This Changes

Request Feishu to show selectable command options such as `/reasoning` as clickable card buttons instead of requiring typed option text.

## Best Possible Solution

Emit typed portable presentation buttons for selectable command-status replies, let Feishu render them through its existing card renderer, and preserve the current text fallback.

## Reproduction Assessment

Yes. Source inspection shows no-argument `/reasoning` returns only a text `Options:` reply, while Feishu renders clickable controls only when a payload carries typed `presentation` button blocks.

## Solution Assessment

Yes, if solved at the shared typed-presentation boundary. A Feishu-only parser that infers buttons from `Options:` text would be the wrong layer and would conflict with the repository's typed action guidance.
