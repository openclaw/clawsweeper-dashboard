## Summary

Keep open: current main has the Feishu primitives to render and dispatch presentation buttons, but `/reasoning` with no argument still produces only a text reply with an `Options:` line, so the reported Feishu command-option UX remains unimplemented.

## What This Changes

The issue requests clickable Feishu card buttons for command options such as `/reasoning` instead of plain text option lists.

## Best Possible Solution

Emit typed portable presentation command actions for no-argument command option replies, then let Feishu render those as native card buttons while preserving the current text fallback and current-level status.

## Reproduction Assessment

Yes, at source level: no-argument `/reasoning` returns only text with an `Options:` line, while Feishu only renders buttons when a reply already carries structured presentation blocks. I did not run a live Feishu chat reproduction in this read-only review.

## Solution Assessment

Yes, if solved through the existing typed presentation-action path rather than Feishu string parsing. The narrow maintainable fix is to preserve text fallback and add structured command buttons where the command status reply is produced.
