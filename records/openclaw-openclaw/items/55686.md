## Summary

Keep open: current main still emits command-option replies such as `/reasoning` as plain `Options:` text, and Feishu's normal reply path does not transform those lines into clickable card buttons. Existing Feishu presentation-button and quick-action plumbing makes this a narrow feature candidate, while the adjacent callback-field work at https://github.com/openclaw/openclaw/pull/42908 does not implement this UX path.

## What This Changes

Render Feishu command-option replies such as `/reasoning` as clickable card buttons instead of plain option text.

## Best Possible Solution

Add a focused Feishu command-option presentation path that reuses the shared `presentation` or interactive-reply contract and existing `ocf1` quick-action callbacks, while preserving plain text fallback for unsupported contexts.

## Reproduction Assessment

Yes, source-level: `/reasoning` without an argument returns a plain `Options: on, off, stream.` text payload, and the Feishu normal reply path currently does not transform that payload into native buttons. I did not run a live Feishu replay in this read-only review.

## Solution Assessment

Yes at the direction level: the narrow maintainable solution is to reuse shared presentation or channel-owned transform plumbing plus Feishu quick-action envelopes, not Feishu-specific markdown scraping in unrelated paths. The repair should stay limited to validated command-option/menu replies and preserve text fallback.
