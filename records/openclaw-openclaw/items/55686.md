## Summary

Keep open: current main and v2026.6.8 still return the no-argument `/reasoning` status as text-only, while Feishu only renders clickable card buttons when a reply payload carries typed `presentation` button blocks. The issue is still necessary, and the related args-menu/card-action issues are adjacent rather than a complete replacement.

## What This Changes

The issue requests clickable Feishu card buttons for command options such as `/reasoning` instead of plain-text option lists.

## Best Possible Solution

Emit typed portable presentation buttons for selectable command-status replies, let Feishu render them through its existing card renderer, and preserve the current text fallback while coordinating with the open native args-menu work.

## Reproduction Assessment

Yes. Source inspection of current main shows no-argument `/reasoning` returns only a text `Options:` reply, while Feishu clickable controls require typed `presentation` button blocks.

## Solution Assessment

Yes, if solved at the shared typed-presentation boundary. A Feishu-only text parser or relying only on `argsMenu` would be too narrow because Feishu does not currently consume that native-menu path.
