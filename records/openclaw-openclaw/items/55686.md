## Summary

Keep open: current main and the latest release still render `/reasoning` option choices as plain `Options:` text in Feishu. Feishu has native card/button delivery plumbing, and Slack already uses the shared reply-transform seam for this pattern, but Feishu does not yet transform command-option replies into clickable buttons; the related Feishu callback-field PR is adjacent and does not implement this UI.

## What This Changes

The issue requests rendering Feishu command option choices such as `/reasoning` as clickable interactive card buttons instead of plain text.

## Best Possible Solution

Add a Feishu-owned command-option presentation path that reuses the existing reply transform/presentation contracts and Feishu structured card button envelopes, while preserving plain text fallback.

## Reproduction Assessment

Yes, source-level. The `/reasoning` no-argument path returns `text: withOptions(..., "on, off, stream")`, current tests assert the plain `Options:` line, and Feishu has no `transformReplyPayload` hook to convert that reply before delivery.

## Solution Assessment

Yes, directionally. A Feishu messaging transform or outbound presentation path using existing `ReplyPayload` interactive/presentation contracts is narrower than core hardcoding, as long as it is bounded to validated command-option replies rather than arbitrary text.
