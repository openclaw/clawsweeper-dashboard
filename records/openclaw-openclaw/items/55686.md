## Summary

Keep open: current main and v2026.6.10 still return no-argument `/reasoning` as a text-only `Options:` reply, while Feishu can render typed presentation buttons only when a reply carries presentation/card data. The related native args-menu issue and open PR overlap, but they do not currently provide the Feishu command-status card-button path.

## What This Changes

The issue requests rendering Feishu command options such as `/reasoning` modes as clickable Feishu card buttons instead of plain text choices.

## Best Possible Solution

Define selectable command-status replies as typed command-action presentation and let Feishu render them through its existing card renderer while preserving text fallback and coordinating with the native args-menu work.

## Reproduction Assessment

Yes. Source inspection shows no-argument `/reasoning` still returns a text-only `Options:` reply, while Feishu clickable controls require a typed presentation/card payload.

## Solution Assessment

Unclear as filed. Feishu card buttons are the right user outcome, but the maintainable fix should produce typed command-action presentation rather than adding a Feishu-only parser for `Options:` text.
