## Summary

Keep open: current main still returns command-option status replies such as `/reasoning` as plain `Options:` text, and Feishu does not transform normal replies into clickable card buttons even though it has presentation/card plumbing available.

## What This Changes

The issue requests rendering Feishu command option replies such as `/reasoning` as clickable interactive-card buttons instead of plain `Options:` text.

## Best Possible Solution

Add a focused Feishu-owned command-option presentation path that reuses the shared reply transform/presentation contracts and Feishu structured interaction envelopes, while preserving plain text fallback.

## Reproduction Assessment

Yes, source-level. `/reasoning` without an argument returns a plain `Options: on, off, stream.` text payload, and Feishu currently has no normal-reply transform that converts those options into native buttons.

## Solution Assessment

Yes at the direction level. The narrow maintainable fix is a Feishu channel transform or presentation renderer using existing structured button envelopes, not core hardcoding or broad markdown scraping.
