## What This Changes

The branch adds `/resume [session-label]` with docs, registry/dispatcher entries, a new resume action handler, and focused tests for named-session listing and binding.

## Best Possible Solution

Land the feature only after `/resume` runs one shared current-binding eligibility check before any session lookup, adds the missing no-argument regression coverage, and includes redacted live Gateway/Telegram proof.

## Reproduction Assessment

Yes, from source for the review findings: PR head still calls `sessions.list` and `sessions.resolve` before the current-placement/capability guard. Real Gateway/Telegram behavior was not reproduced because after-fix real-environment proof is missing.

## Solution Assessment

No as written. A label-only `/resume` command is a plausible direction, but lookup must be behind the same current-binding eligibility boundary before this can merge.
