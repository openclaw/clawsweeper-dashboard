## Summary

Keep this PR open: the feature is plausible, but the no-argument `/resume` path has a blocking access-control issue and the external PR still lacks after-fix real behavior proof.

## What This Changes

The PR adds `/resume [session-label]` to Gateway chat command routing, factors `/focus` binding into a shared helper, documents the command, and adds focused routing/binding tests.

## Best Possible Solution

Keep the shared binding direction, but gate both `/resume` listing and binding through the same controller, conversation, and visibility checks before accepting the new command surface.

## Reproduction Assessment

Yes, from source: a no-argument `/resume` reaches `sessions.list` before the existing leaf-subagent control-scope check. The labeled path rejects leaf subagents later, which shows the intended guard is not applied uniformly.

## Solution Assessment

No as written. Reusing the existing session label resolver and binding helper is a good direction, but the list path needs the same access-control boundary before `sessions.list` and the PR needs real behavior proof.
