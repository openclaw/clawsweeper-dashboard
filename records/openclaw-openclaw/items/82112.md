## Summary

Keep open. This is a useful new command surface, but the current PR still performs named-session lookups outside the existing subagent visibility boundary and lacks after-fix real behavior proof, so it should not merge as-is.

## What This Changes

The PR adds a `/resume [session-label]` Gateway chat command with registry wiring, a new handler, docs, and focused routing/binding tests.

## Best Possible Solution

Keep the new command direction only if every list, resolve, and bind path goes through the existing controller and Gateway visibility scopes, then require redacted real Gateway or Telegram proof before merge.

## Reproduction Assessment

Yes, from source. A no-argument `/resume` reaches `sessions.list` before the controller check, and the labeled path calls `sessions.resolve` without the `spawnedBy` scope that current `/focus` uses for subagent callers.

## Solution Assessment

No as written. Reusing session labels for a focused resume command is maintainable, but the implementation must preserve the existing subagent visibility boundary before it is safe to merge.
