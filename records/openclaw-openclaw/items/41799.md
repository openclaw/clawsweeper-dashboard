The PR adds a legacy `{ ok, to }` extraction path in outbound target normalization, resolver/send regression tests, and a secret-baseline line update.

## Best Possible Solution

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: a plugin normalizer returning `{ ok: true, to: "qqbot:..." }` is passed to `normalizeOptionalString`, which rejects non-strings before `.to` can be used.

## Solution Assessment
