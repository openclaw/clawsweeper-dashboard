## Summary

Keep open: current main still lacks compatibility for object-shaped legacy normalizer results, and the fix direction is useful, but this branch is stale against the current split outbound tests and needs porting before merge.

## What This Changes

The branch adds a runtime shim that extracts a trimmed string `to` from legacy object-shaped channel `normalizeTarget` results, updates outbound regression coverage, and refreshes secrets-baseline line numbers.

## Best Possible Solution

Land a rebased version of the shim in `target-normalization.ts` with focused target-normalization and full send-path coverage in the current split tests, plus a single-line changelog entry.

## Reproduction Assessment

Yes, at source level. A channel plugin whose `messaging.normalizeTarget` returns `{ ok: true, to: "qqbot:..." }` reaches `normalizeTargetForProvider`, where current main ignores the object `to` value because only strings survive `normalizeOptionalString`.

## Solution Assessment

Yes on the core approach, but no on the current branch shape. The narrow maintainable solution is runtime tolerance in the core normalization helper, with the regression tests ported to current main's split test files and no SDK contract change.
