## Summary

Keep open: this PR targets a real Control UI false-busy bug, but the current patch is not merge-ready because it can hide Stop/Steer controls for a still-active local run and its proof is mocked rather than real Control UI behavior.

## What This Changes

The branch narrows Control UI webchat busy/abort rendering to local send/stream state and adds unit plus mocked Chromium regression coverage for stale session-list active-run rows.

## Best Possible Solution

Land a repaired Control UI fix that separates stale session-list rows from authoritative local active-run state, with real browser/gateway proof for both idle-after-completion and active streamless tool-run controls.

## Reproduction Assessment

Yes, at source level: current main can have chatRunId set while chatStream is null, and the PR's new gate would hide controls in that state. I did not run a live browser/gateway reproduction in this read-only review.

## Solution Assessment

No. The stale-session split is the right direction, but sending || stream !== null is too narrow; the fix should preserve an explicit local active-run signal while excluding session-list-only stale rows.
