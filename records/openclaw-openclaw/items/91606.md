## Summary

## What This Changes

The PR narrows Control UI WebChat busy/abort UI in `ui/src/ui/views/chat.ts` to local sending/stream state and adds unit plus mocked Chromium tests for stale active session rows.

## Best Possible Solution

Rebase and split abortability so session-list-only stale rows do not drive visible composer busy UI, while local active `chatRunId` runs still keep Stop, Queue, and Steer available; then add real browser/gateway proof.

## Reproduction Assessment

Yes, source-level: current main combines local `chatRunId` abortability with session-list active rows, and the PR diff narrows the visible controls to `sending || stream !== null`. I did not run a live browser/gateway reproduction in this read-only review.

## Solution Assessment

No. Splitting stale session-list abortability from visible composer state is the right direction, but `sending || stream !== null` is too narrow; the fix should preserve a local active-run signal while excluding session-list-only stale rows.
