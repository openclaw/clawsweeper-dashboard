## Best Possible Solution

Rebase and repair the Control UI split so session-list-only stale rows do not drive composer busy UI, while a local active run id still keeps Stop/Queue/Steer available; then provide redacted real browser/gateway proof.

## Reproduction Assessment

Yes, at source level: current main passes `hasAbortableSessionRun(state)` into visible chat controls, and current source shows `chatRunId` can remain active while `chatStream` is null after tool-stream reconciliation. I did not run a live browser/gateway reproduction in this read-only review.

## Solution Assessment

No. Splitting session-list-only abortability from visible composer state is the right direction, but `sending || stream !== null` is too narrow; the fix should preserve a local active-run signal while excluding session-list-only stale rows.
