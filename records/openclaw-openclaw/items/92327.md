## Summary

Keep this issue open with the paired implementation PR. OpenClaw v2026.6.5 intentionally renders `sessions_send` handoffs as left-side assistant groups rather than right-side user bubbles, but the reported loss of `Forwarded from <agent>` attribution remains plausible and the linked patch does not yet address its likely source.

## What This Changes

Restore visible source attribution for inter-session WebChat messages while preserving their intended assistant-side presentation.

## Best Possible Solution

Keep forwarded inter-session messages assistant-side, reproduce the missing label in a real two-agent WebChat session, and repair the earliest history or live-delivery boundary that drops provenance, with focused gateway and browser regression coverage.

## Reproduction Assessment

Partly yes. The report provides a concrete two-agent `sessions_send` path and source/tests define the forwarded-label contract, but this read-only review did not independently reproduce the missing label in a live v2026.6.5 or current-main WebChat session.

## Solution Assessment

No for the linked patch as currently scoped. Converting a blank label to null only selects the generic assistant fallback; the best fix must identify why the gateway-generated forwarded label is absent from the observed WebChat path.
