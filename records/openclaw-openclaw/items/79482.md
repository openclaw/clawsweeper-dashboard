## Summary

This PR is useful and pointed at the right NVIDIA-owned surface, but it should not merge yet because the changed hook does not actually preserve NVIDIA's ranked order in the setup/model-picker path, and the supplied real behavior proof only shows the upstream endpoint response.

## What This Changes

The PR adds a live NVIDIA featured-model fetcher with validation, 24-hour caching, fallback to the bundled catalog, provider-local tests, docs, and a changelog entry.

## Best Possible Solution

Keep the provider-local fetch/cache/fallback design, but route the live ranked rows through the actual onboarding/model-picker catalog path or narrow the claim/docs to the surfaces that really consume the live provider catalog.

## Reproduction Assessment

Yes, source inspection gives a clear reproduction path for the ordering gap: NVIDIA onboarding uses promptDefaultModel, which consumes loadModelCatalog rows that are appended and sorted by provider/name rather than NVIDIA feed rank. I did not live-run the PR branch in this read-only review.

## Solution Assessment

No; the provider-local fetch/cache/fallback helper is a good building block, but routing ranked rows through augmentModelCatalog is not sufficient for the advertised setup/model-picker promotion because the shared catalog path reorders those rows.
