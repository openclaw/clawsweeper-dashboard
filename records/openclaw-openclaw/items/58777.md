## Summary

Keep open. ## What This Changes

Request additive telemetry and lifecycle/session state that records requested versus effective provider, model, and thinking for explicitly pinned sub-agent runs after fallback or runtime routing.

## Best Possible Solution

Define one selected/requested/effective provider-model-thinking contract and thread it through fallback results, sub-agent completion/lifecycle metadata, session/status surfaces, docs, and tests without overwriting the user's selected model state.
