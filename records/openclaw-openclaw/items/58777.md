Keep open. Current main and latest release have partial winner/request-shaping metadata, but they still do not provide one durable requested-versus-effective provider/model/thinking contract for completed sub-agent runs, lifecycle metadata, and session telemetry.

## Best Possible Solution

Define one selected/requested/effective provider-model-thinking contract and thread it through fallback results, sub-agent completion metadata, lifecycle/session/status surfaces, docs, and focused tests without overwriting user-selected model state.
