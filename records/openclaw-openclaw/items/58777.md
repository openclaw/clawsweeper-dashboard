## Summary

Keep open. Current main has fallback-step logging and spawn-time resolved sub-agent model metadata, but it still lacks one durable requested-versus-effective provider/model/thinking contract for completed sub-agent runs, lifecycle metadata, and session telemetry.

## What This Changes

The issue requests durable requested-versus-effective model/provider/thinking telemetry and reconciliation for sub-agent runs when fallback or routing changes the executed model.

## Best Possible Solution

Define one selected/requested/effective provider-model-thinking contract and thread it through fallback results, sub-agent lifecycle/completion metadata, session/status surfaces, docs, and tests without overwriting the user's selected model state.

## Reproduction Assessment

Yes at source level: current main identifies fallback winners internally and records attempts, but the completed embedded/sub-agent metadata path still lacks durable requested-versus-effective provider/model/thinking fields. I did not run a live pinned fallback scenario.

## Solution Assessment

Unclear pending maintainer direction; the requested direction is valid, but the best fix should be a single requested/effective contract across fallback, lifecycle, session, and status surfaces rather than local fields in one path.
