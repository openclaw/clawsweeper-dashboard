## Summary

Keep open. Current main has partial fallback observability and shipped fallback-state persistence, but it still does not provide the requested sub-agent closed-loop `effective_model` / `effective_provider` / `effective_thinking` completion telemetry or a settled product contract for explicit pin fidelity across fallback/runtime changes.

## What This Changes

Request to add sub-agent requested-vs-effective model/thinking telemetry and reconcile fallback/session state when explicit model pins are bypassed.

## Best Possible Solution

Define one durable effective-model/effective-thinking contract for fallback-selected runs, then wire it through sub-agent completion/lifecycle telemetry with focused fallback and subagent tests.

## Reproduction Assessment

Yes at source level: current main exposes selected fallback provider/model in some logs/callbacks but does not publish the requested effective completion fields for sub-agent pin verification. I did not run a live Pi/Codex fallback scenario in this read-only review.

## Solution Assessment

Unclear. The requested telemetry direction is valid, but maintainers should first choose the durable effective-model contract and whether thinking/session reconciliation belongs in lifecycle events, sub-agent completion payloads, or a shared runtime result type.
