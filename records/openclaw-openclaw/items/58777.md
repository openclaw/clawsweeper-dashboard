## Summary

Keep open. Current main has spawn-time resolved model metadata and per-run fallback traces, but it still lacks a durable completed-run requested-versus-effective provider/model/thinking contract for sub-agent lifecycle, session, and status surfaces.

## What This Changes

Request durable telemetry for explicit sub-agent model pins so parents can compare requested and effective provider, model, and thinking after fallback or routing changes.

## Best Possible Solution

Define one selected/requested/effective provider-model-thinking contract and thread it through sub-agent spawn/completion, fallback results, lifecycle, session/status surfaces, docs, and tests in coordination with #51441.

## Reproduction Assessment

Yes at source level: current main exposes the internal fallback winner and spawn-time resolved model, but completed sub-agent lifecycle/session/status paths still lack the requested effective provider/model/thinking contract.

## Solution Assessment

Unclear pending maintainer direction; the telemetry direction is valid, but the best fix should be one shared contract across fallback, lifecycle, session, status, and the overlapping resolved-backend work.
