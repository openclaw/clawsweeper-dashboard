## Summary

Keep open: this PR is an active fix candidate for the open Telegram callback timeout issue, but it is not merge-ready because the early acknowledgement is not marked to prevent the existing downstream acknowledgement from firing, and the PR still lacks live Telegram proof.

## What This Changes

The PR adds a pre-sequentialize Telegram middleware that fire-and-forgets `answerCallbackQuery` before queued callback processing reaches the existing handler.

## Best Possible Solution

Keep the pre-sequentialize acknowledgement shape, but mark successful early acknowledgements and skip the handler-level fallback before merging, with focused regression coverage and redacted live Telegram proof.

## Reproduction Assessment

Yes, source-reproducible: current main registers `sequentialize` before the callback handler, ordinary callbacks share the chat/topic sequential key, and grammY runner waits before invoking downstream middleware. I did not run a live Telegram client reproduction in this read-only review.

## Solution Assessment

No for this branch as written. Pre-sequentialize acknowledgement is the right layer, but the branch should also prevent the downstream duplicate answer and supply live Telegram proof before merge.
