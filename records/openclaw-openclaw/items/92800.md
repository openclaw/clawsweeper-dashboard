## Summary

Keep open: this is the right Telegram timing layer and the later Mantis proof now covers the visible callback behavior, but the patch still leaves the downstream handler acknowledgement active, so ordinary callbacks can issue a second Bot API answer request after the early acknowledgement.

## What This Changes

Adds a Telegram middleware before grammY `sequentialize` that fire-and-forgets `answerCallbackQuery` for callback updates before continuing to the existing handler chain.

## Best Possible Solution

Keep the pre-sequentialize acknowledgement, mark successful early answers on the context, skip the handler fallback when that marker is present, and add focused middleware-order coverage.

## Reproduction Assessment

Yes. Current main and the latest release put `sequentialize` before the callback handler, ordinary callbacks share the chat/topic key, and the runner waits before invoking downstream middleware; Mantis also exercised the visible Telegram path.

## Solution Assessment

No, not as written. Pre-sequentialize acknowledgement is the right layer, but the patch should suppress the downstream duplicate acknowledgement after a successful early answer.
