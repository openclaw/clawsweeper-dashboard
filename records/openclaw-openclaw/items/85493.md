## Summary

Keep open: the PR fixes the central Telegram menu-slot problem in the right area, but it currently drops hidden aliases from the shared native-spec surface without preserving them as reserved command names, and the external PR still lacks live Telegram proof.

## What This Changes

The PR changes the shared command registry so `nativeAliases` remain valid for lookup but are no longer emitted by `listNativeCommandSpecs*` for native command menus.

## Best Possible Solution

Hide aliases from Telegram menu registration while keeping them reserved for conflict checks and callable alias lookup, with focused regression coverage and live Telegram proof.

## Reproduction Assessment

Yes for source-level reproduction: current main expands `/side` into native specs, and the PR diff removes that expansion while alias lookup still resolves `/side` to `/btw`. I did not run a live Telegram bot in this read-only review.

## Solution Assessment

No, not as written: hiding aliases from the menu is the right fix direction, but aliases still need to be reserved for command-conflict checks and the shared SDK behavior needs an intentional compatibility shape.
