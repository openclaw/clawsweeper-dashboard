## Label Justifications

- `P2`: This is a normal Telegram command-menu bugfix with limited blast radius, but the current patch has compatibility risk before merge.
- `merge-risk: 🚨 compatibility`: The diff changes a shared plugin SDK/native command helper and can remove existing native aliases from Discord and Slack slash-command surfaces.
- `merge-risk: 🚨 message-delivery`: Dropping native alias specs can prevent alias slash commands from being registered or handled, so user command invocations may no longer reach the intended handler.

## Summary

Keep open. The bug is valid, but this PR is not merge-ready because it fixes the Telegram menu count by changing the shared native command-spec registry, which can drop native aliases from non-Telegram slash registration and Telegram hidden native handlers; proof is also unit-test-only.

## What This Changes

The PR changes native command spec generation so `nativeAliases` remain lookup aliases but are no longer emitted as separate native command menu specs.

## Best Possible Solution

Preserve alias-aware native command catalogs and handlers by default, then filter native aliases only from Telegram menu registration and prove `/btw` is menu-visible, `/side` is not, and typed `/side` still routes correctly.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: current main expands `nativeAliases` into native command specs, and Telegram feeds those specs into a 100-command capped menu payload. I did not run live Telegram proof in this read-only review.

## Solution Assessment

No, this PR's current approach is broader than the Telegram bug. The safer fix is to keep shared alias catalogs intact and suppress aliases only from Telegram menu registration.
