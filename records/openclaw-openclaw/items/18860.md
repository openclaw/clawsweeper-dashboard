## Summary

Keep this PR open: current main still lacks a plugin-facing resolved-tools/schema API, and the linked feature issue was closed in favor of this PR. The PR is still not merge-ready because the new hook builds `ctx.channelId` differently from the shared agent hook context path.

## What This Changes

The PR adds an observation-only `after_tools_resolved` plugin hook with resolved tool metadata, docs, SDK baseline updates, tests, and a changelog entry.

## Best Possible Solution

Land the additive resolved-tools hook after reusing the shared agent hook context helper and preserving schema snapshots, or explicitly supersede it with a maintainer-approved pull-based resolved-tools API.

## Reproduction Assessment

Not applicable for the feature request itself. The PR-head defect is source-reproducible: for `agent:main:discord:channel:c1`, the shared helper returns `ctx.channelId` as `c1` while the new hook would report `discord`.

## Solution Assessment

No, not as-is. The hook direction is plausible, but the emission should reuse `buildAgentHookContextChannelFields(params)` so plugins receive consistent routing metadata across hook phases.
