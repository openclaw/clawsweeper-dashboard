## Summary

Keep this PR open: current main still lacks an equivalent resolved-tools plugin API, the PR has real runtime log proof, but it still needs changes before merge for mutable schema exposure and a missing changelog entry.

## What This Changes

The PR adds an `after_tools_resolved` plugin hook, embedded-runner emission of final tool metadata, SDK hook typings/baseline hashes, docs, and focused hook tests.

## Best Possible Solution

Land an additive resolved-tools hook or pull API that publishes cloned or immutable final per-attempt schema snapshots, updates SDK/docs/changelog/tests, and has maintainer approval for the final semantics.

## Reproduction Assessment

Yes for the review concern. Current main lacks the hook/API, and the PR defect is source-reproducible because it publishes `tool.parameters` by reference before the same tool definitions are used for provider/session setup.

## Solution Assessment

No, not yet. The hook is a plausible narrow feature direction, but an observation-only API should publish cloned or immutable schema snapshots and include the required changelog entry before merge.
