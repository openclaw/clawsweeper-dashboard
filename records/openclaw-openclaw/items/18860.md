## Summary

Keep this PR open: current main still lacks a plugin-facing resolved-tools/schema API and this PR remains the canonical implementation path, but the current patch has a build-blocking type import error that must be fixed before merge.

## What This Changes

The PR adds an `after_tools_resolved` plugin hook, documentation, changelog entry, SDK baseline hash update, and focused tests so plugins can observe the final per-attempt tool list and JSON Schema snapshots.

## Best Possible Solution

Fix the type import/export mismatch, keep the cloned-schema contract and docs aligned, then have maintainers decide whether this additive hook is the preferred plugin API over a pull-based resolved-tools surface.

## Reproduction Assessment

Yes. Source inspection of the PR head shows a type import from `src/plugins/types.ts` while that module does not export `PluginHookAfterToolsResolvedEvent`; the event is exported from `src/plugins/hook-types.ts`.

## Solution Assessment

No. The overall hook direction is plausible and supported by the linked feature request, but this patch should not merge until the import/export mismatch is fixed and the public plugin API boundary is approved.
