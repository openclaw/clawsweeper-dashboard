## What This Changes

The PR adds an `after_tools_resolved` plugin hook, embedded-runner emission, SDK API baseline updates, docs, and focused tests for resolved tool metadata.

## Best Possible Solution

Land the additive resolved-tools hook after schema snapshots are cloned or frozen, retry/snapshot semantics are documented, SDK baseline and changelog are aligned, and maintainers approve the public API shape.

## Reproduction Assessment

Yes. Current main source lacks the hook/API, and the PR defect is source-reproducible by registering an `after_tools_resolved` handler that mutates `event.tools[0].parameters` synchronously before session creation consumes the same schema object.

## Solution Assessment

No, not as proposed yet. The hook is a narrow maintainable direction, but an observation-only plugin API should expose cloned or immutable schema snapshots and carry the required changelog entry before merge.
