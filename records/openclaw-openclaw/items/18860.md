## Summary

The branch adds an `after_tools_resolved` plugin hook, emits it during embedded attempts with resolved tool metadata, documents it, refreshes SDK baseline hashes, adds focused tests, and updates the changelog.

## Best Possible Solution

Land the additive resolved-tools hook only after it reuses the shared agent hook context contract and maintainers approve the plugin API boundary, or supersede it with a pull-based resolved-tools API.

## Reproduction Assessment

Not applicable for the feature request itself. Yes for the PR-head defect: source inspection shows `agent:main:discord:channel:c1` would report `ctx.channelId` as `discord` in the new hook while existing shared hook context reports `c1`.

## Solution Assessment

No, not as-is. The hook direction is plausible, but the emission should reuse `buildAgentHookContextChannelFields(params)` so plugins receive consistent routing metadata across hook phases.
