## Summary

Keep open. Current main still lacks the proposed resolved-tools hook or an equivalent plugin-facing API, and the latest maintainer discussion explicitly keeps this PR open for follow-up. The PR remains a plausible implementation path, but the schema immutability/read-only contract needs maintainer attention before merge.

## What This Changes

The PR adds an `after_tools_resolved` plugin hook, embedded-runner emission, hook contract/docs updates, SDK API baseline hashes, and focused tests for resolved tool metadata.

## Best Possible Solution

Keep this PR open as the active design/implementation path, but require a maintainer-quality resolved-tools contract: immutable or cloned schema snapshots, clear trusted-plugin/security semantics, retry/dedupe guidance, public docs, SDK baseline updates, and focused tests for normal attempts, compaction exclusion, and session active-tool ordering.

## Reproduction Assessment

Yes for the current-main gap. Source inspection on main shows `after_tools_resolved` is absent from hook names, handler types, hook runner methods, plugin API alternatives, docs, and the embedded attempt tool-resolution path.

## Solution Assessment

No, not yet. The additive hook is a narrow and maintainable direction, but the proposed read-only observation contract should not publish mutable provider-facing schema references without cloning, freezing, or otherwise snapshotting them.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.78

Full review comments:

- **[P2] Clone or freeze resolved tool schemas before publishing:** `src/agents/pi-embedded-runner/run/attempt.ts:396`
  - body: The new hook is documented as observation-only, but this assigns the live `parameters` object into the event. A handler can mutate that object before the provider attempt consumes it, changing tool schemas from a hook that is supposed to observe only; publish a cloned/frozen snapshot or emit after an immutable schema snapshot exists.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: No third-party dependency or CI supply-chain change is visible, but the diff exposes mutable provider-facing tool schemas through a new plugin hook.

Concerns:

- **[medium] Mutable schema references escape through observation hook:** `src/agents/pi-embedded-runner/run/attempt.ts:396`
  - body: Passing `parameters` by reference gives hook handlers write access to live tool schema objects even though the hook is described as read-only. In the trusted-plugin model this is not a new arbitrary-code boundary, but it can silently alter tool validation or provider payloads and should be hardened before merge.
