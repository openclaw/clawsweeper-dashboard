## Summary

Keep open. Current main still lacks an `after_tools_resolved` hook or equivalent plugin-facing resolved-tool/schema API, and this PR is the active implementation path, but the proposed patch is not ready because it exposes mutable provider-facing schema objects through an observation hook.

## What This Changes

The PR adds an `after_tools_resolved` plugin hook, embedded-runner emission, hook contract/docs updates, SDK API baseline hashes, and focused tests for resolved tool metadata.

## Best Possible Solution

Keep this PR open as the active resolved-tools API path, but require an immutable or cloned schema snapshot, clear trusted-plugin semantics, retry/dedupe guidance, public docs, SDK baseline consistency, and focused tests for normal attempts, compaction exclusion, and session active-tool ordering before merge.

## Reproduction Assessment

Yes for the current-main gap. Source inspection at `53593f0683fa28960f487b5e10c2bf9a4e2ed16e` shows the hook name, event type, runner method, plugin API alternative, docs entry, and embedded-attempt emission point are all absent.

## Solution Assessment

No, not as proposed yet. The additive hook is a narrow maintainable direction, but an observation-only contract should publish immutable or cloned schema snapshots rather than live provider-facing objects.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.78

Full review comments:

- **[P2] Clone or freeze resolved tool schemas before publishing:** `src/agents/pi-embedded-runner/run/attempt.ts:396`
  - body: The new hook is documented as observation-only, but this publishes the live `parameters` object in the event. A handler can mutate that object before or during provider/session consumption, changing tool schemas from a hook that should only observe; publish a cloned or frozen snapshot instead.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The diff introduces no dependency or CI supply-chain change, but it exposes mutable provider-facing tool schemas through a new plugin hook.

Concerns:

- **[medium] Mutable schema references escape through observation hook:** `src/agents/pi-embedded-runner/run/attempt.ts:396`
  - body: Passing `parameters` by reference gives hook handlers write access to live tool schema objects even though the hook is described as read-only. In the trusted-plugin model this is not a new arbitrary-code boundary, but it can silently alter tool validation or provider payloads and should be hardened before merge.
