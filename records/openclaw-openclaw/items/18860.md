## Summary

Keep open: current main still lacks any plugin-facing resolved-tools hook or pull API, and the linked feature request was closed as superseded by this PR. The branch is still useful, but PR head has a concrete context-consistency bug that should be fixed before merge.

## What This Changes

The PR adds an `after_tools_resolved` plugin hook, emits resolved tool metadata during embedded agent attempts, documents it, updates SDK baseline/changelog metadata, and adds focused tests.

## Best Possible Solution

Land the additive resolved-tools hook after it uses the standard hook context contract and the API boundary is accepted, or explicitly supersede it with an equivalent pull-based resolved-tools API.

## Reproduction Assessment

Not applicable for the feature request itself. For the PR-head defect, yes: source inspection shows a session key like `agent:main:discord:channel:c1` would report `ctx.channelId` as `discord` in the new hook while existing shared hook context reports `c1`.

## Solution Assessment

No, not as-is. The hook direction is plausible, but the emission should reuse `buildAgentHookContextChannelFields(params)` so plugins receive consistent routing metadata across hook phases.
