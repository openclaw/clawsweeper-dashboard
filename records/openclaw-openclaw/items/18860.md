## Summary

Keep this PR open. Current main still lacks `after_tools_resolved` or an equivalent resolved-tool discovery API, and the PR remains useful, but the current patch still has a source-reproducible context mismatch in the new hook emission.

## What This Changes

The PR adds an observation-only `after_tools_resolved` plugin hook, docs, SDK baseline updates, tests, and a changelog entry for resolved tool/schema metadata.

## Best Possible Solution

Land the additive resolved-tools hook after reusing the shared agent hook context helper and maintainer approval of the API boundary, or explicitly supersede it with a pull-based resolved-tools API.

## Reproduction Assessment

Not applicable for the feature request itself. The PR-head defect is source-reproducible: for `agent:main:discord:channel:c1`, the shared helper returns `ctx.channelId` as `c1` while the new hook reports `discord`.

## Solution Assessment

No, not as-is. The hook direction is plausible, but the emission should reuse `buildAgentHookContextChannelFields(params)` so plugins receive consistent routing metadata across hook phases.
