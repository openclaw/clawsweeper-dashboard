## Summary

Keep this PR open: current main still lacks an equivalent resolved-tools plugin API, and the branch is still the active implementation path, but PR head has a concrete context-contract bug that should be repaired before merge.

## What This Changes

The PR adds an `after_tools_resolved` plugin hook, emits resolved tool metadata during embedded agent attempts, documents the hook, refreshes SDK baselines, adds tests, and updates the changelog.

## Best Possible Solution

Land the additive resolved-tools hook after it reuses the shared agent hook context contract and the plugin API boundary is explicitly accepted, or supersede it with a pull-based resolved-tools API.

## Reproduction Assessment

Not applicable for the feature request itself. For the PR-head defect, yes: source inspection shows `agent:main:discord:channel:c1` would report `ctx.channelId` as `discord` in the new hook while current shared hook context reports `c1`.

## Solution Assessment

No, not as-is. The hook direction is plausible, but the emission should reuse `buildAgentHookContextChannelFields(params)` so plugins receive consistent routing metadata across hook phases.
