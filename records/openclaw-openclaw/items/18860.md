## Summary

Keep this PR open: current main still lacks a dedicated `after_tools_resolved` hook or equivalent normalized post-policy resolved-tools API, and this PR remains the canonical implementation path, but the PR head still has a source-reproducible channel-context bug that should be fixed before merge.

## What This Changes

The PR adds an observation-only plugin hook that emits resolved tool metadata and JSON Schema snapshots from embedded agent attempts, with docs, SDK baseline updates, tests, and a changelog entry.

## Best Possible Solution

Land the additive resolved-tools hook after reusing the shared agent hook context helper and preserving schema snapshots, or explicitly supersede it with a maintainer-approved pull-based resolved-tools API.

## Reproduction Assessment

Not applicable for the feature request itself. The blocking PR-head defect is source-reproducible: the new hook hand-builds `ctx.channelId` while adjacent agent hooks use `buildAgentHookContextChannelFields(params)`.

## Solution Assessment

No, not as-is. The hook direction is plausible, but the emission should reuse the shared channel context helper so plugins receive consistent routing metadata across hook phases.
