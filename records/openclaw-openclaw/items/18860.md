## Summary

## What This Changes

The PR adds an `after_tools_resolved` plugin hook, embedded-runner emission, hook/event types, docs, plugin SDK API baseline hashes, focused tests, and a changelog entry.

## Best Possible Solution

Land a maintainer-approved resolved-tools plugin surface after the hook context is consistent with existing agent hooks and the emission is placed at the current post-policy session-tool boundary.

## Reproduction Assessment

Yes for the PR-head defect: source inspection shows the new hook hand-builds `ctx.channelId`, while the existing helper/test resolves channel session keys to conversation ids. Reproduction is not applicable to the underlying request because it is a new plugin API surface.

## Solution Assessment

No, not as-is. The additive hook is plausible, but it should reuse `buildAgentHookContextChannelFields(params)` and be rebased into the current post-policy session-tool path before maintainer API review.
