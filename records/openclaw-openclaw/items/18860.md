## Best Possible Solution

Land an additive resolved-tools plugin API after the hook reuses the shared agent context contract, keeps cloned schema snapshots documented and tested, and gets maintainer approval for the push-hook design.

## Reproduction Assessment

## Solution Assessment

No, not as-is. The additive hook is plausible, but the new hook should reuse `buildAgentHookContextChannelFields(params)` so its context matches existing agent hooks before maintainers decide the final API shape.
