## Summary

Keep this PR open, but it still needs a small correctness fix before merge: the new hook does not use the shared channel-context helper, so it can report different `ctx.channelId` values than the existing agent hooks.

## What This Changes

The PR adds an observation-only `after_tools_resolved` plugin hook, SDK baseline/docs updates, focused tests, and a changelog entry for resolved tool metadata.

## Best Possible Solution

Land the additive resolved-tools hook after reusing the shared agent hook context helper and preserving schema snapshots, or explicitly supersede it with a maintainer-approved pull-based resolved-tools API.

## Reproduction Assessment

Not applicable for the feature request itself. The PR-head defect is source-reproducible: for a session key like `agent:main:discord:channel:c1`, existing hooks derive `ctx.channelId` as `c1`, while the new hook would report `discord`.

## Solution Assessment

No, not as-is. The hook direction is plausible, but the emission should reuse `buildAgentHookContextChannelFields(params)` so plugins receive consistent routing metadata across hook phases.
