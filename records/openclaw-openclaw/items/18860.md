## Summary

Keep this PR open: current main still lacks a dedicated resolved-tools plugin hook, but PR head has a blocking context bug where the new hook hand-builds `channelId` instead of using the shared agent hook context helper.

## What This Changes

The PR adds an `after_tools_resolved` plugin hook, event and runner types, embedded-runner emission, docs, SDK API baseline hashes, focused tests, and a changelog entry.

## Best Possible Solution

Land a maintainer-approved resolved-tools plugin surface after the new hook uses standard agent hook context and the final docs clarify read-only schema snapshots plus retry/dedupe expectations.

## Reproduction Assessment

Yes for the PR-head defect: source inspection shows the new hook reports provider/channel text while the existing helper and tests resolve channel session keys to conversation ids. Reproduction is not applicable to the underlying request because it is a new plugin API surface.

## Solution Assessment

No, not as-is. The additive hook is plausible, but it should reuse `buildAgentHookContextChannelFields(params)` and add focused regression coverage before maintainer API review.
