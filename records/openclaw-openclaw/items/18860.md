## Summary

Keep this PR open: current main still lacks an equivalent resolved-tools API, and the linked feature issue was closed as superseded by this PR. The branch is still not merge-ready because the new hook reports channel context differently from existing agent hooks.

## What This Changes

The branch adds an `after_tools_resolved` plugin hook, emits it from embedded agent attempts, documents it, refreshes SDK baseline hashes, adds tests, and adds a changelog entry.

## Best Possible Solution

Land the additive resolved-tools hook only after it reuses the shared hook context contract and maintainers approve the plugin API boundary, or explicitly supersede it with a pull-based resolved-tools API.

## Reproduction Assessment

Not applicable for the feature request itself. The PR-head defect is source-reproducible: for a session key like agent:main:discord:channel:c1, the shared helper reports channelId c1 while the new hook would report discord.

## Solution Assessment

No, not as-is. The hook direction is plausible, but the emission should reuse buildAgentHookContextChannelFields(params) so plugins receive consistent routing metadata across hook phases.
