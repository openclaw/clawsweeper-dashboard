## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Use the shared hook channel context fields:** `src/agents/pi-embedded-runner/run/attempt.ts:1693`
  - body: This hook builds `ctx.channelId` from `messageChannel ?? messageProvider`, while the other agent hooks use `buildAgentHookContextChannelFields`, which derives the actual conversation id from `sessionKey`, `currentChannelId`, and `messageTo`. For channel sessions where `messageChannel` is only the provider or a prefixed target, `after_tools_resolved` will report a different channel context than `llm_input`/`agent_end`, so plugins cannot reliably correlate the resolved tool snapshot with the rest of the run.
