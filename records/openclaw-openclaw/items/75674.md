## Summary

Keep open. Current main still passes MCP EmbeddedResource blocks through the bundle-MCP boundary unchanged while provider and replay paths consume only Pi text/image content; #75714 is an open closing PR, so this should remain open until that PR or an equivalent fix lands.

## What This Changes

MCP tool results containing `type: "resource"` should pass `resource.text` to the model instead of becoming an image placeholder.

## Best Possible Solution

Normalize MCP `EmbeddedResource` blocks once at the bundle-MCP to Pi `AgentToolResult` boundary, add focused regression coverage, and land #75714 or an equivalent narrow replacement.

## Reproduction Assessment

Yes. A source-level reproduction is clear: have a bundle MCP tool return `content: [{ type: "resource", resource: { text: "..." } }]`; current main passes that block through unchanged and downstream provider conversion finds no top-level text.

## Solution Assessment

Yes. Boundary normalization is the narrowest maintainable fix because Pi/provider transports already expect text/image content, while patching each provider separately would leave future transports exposed to the same MCP shape mismatch.
