## What This Changes

The issue requests extracting `resource.text` from MCP `type: "resource"` tool-result blocks instead of sending a `(see attached image)` placeholder to the model.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/75714 so bundle-MCP normalizes MCP resource text into Pi text content before provider transports; keep broader resource variants as a follow-up if needed.

## Solution Assessment
