## Summary

Keep open. Current main still lets MCP embedded resource blocks cross the bundle-MCP tool boundary without converting `resource.text` to normal tool-result text, and the OpenAI-compatible replay path still only reads top-level text or image blocks before using the image placeholder. Related #68037/#68046 cover the HTTP MCP gateway response path, not this agent-side bundle-MCP/provider replay path.

## What This Changes

The issue asks OpenClaw to preserve text from MCP `EmbeddedResource` tool results in tool output and model context instead of replacing it with a placeholder.

## Best Possible Solution

Normalize MCP tool-result content at the bundle-MCP boundary so text embedded resources become normal OpenClaw text tool-result blocks before persistence/replay, while keeping image blocks intact and using explicit, bounded text fallbacks for unsupported MCP block variants.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: an MCP `CallToolResult` with `content: [{ type: "resource", resource: { text: "..." } }]` passes through the current materializer and then the OpenAI-compatible conversion reads no top-level text, producing the reported placeholder path.

## Solution Assessment

Yes. Extracting `resource.resource.text` is the right behavior, but the narrowest maintainable seam is the MCP-to-OpenClaw tool-result conversion boundary rather than duplicating resource handling separately in every provider transport.

## Repair Work Prompt

Fix the agent-side MCP embedded-resource tool-result bug reported in https://github.com/openclaw/openclaw/issues/75674. Current main casts MCP `CallToolResult.content` directly into `AgentToolResult.content` in `src/agents/pi-bundle-mcp-materialize.ts`, even though the Pi agent contract only supports text/image blocks. Add a focused normalization step for bundle-MCP tool results: preserve MCP text blocks as text, preserve supported image blocks, convert embedded text resources (`type: "resource"` with `resource.text`) into normal text blocks, and use an explicit bounded text fallback for unsupported MCP blocks such as resource links, audio, or blob resources. Do not duplicate or replace the HTTP MCP gateway work in #68046 unless a tiny shared helper is clearly safer. Add regression coverage for `resource.text` in `src/agents/pi-bundle-mcp-tools.materialize.test.ts`, and add provider replay coverage only if the fix still allows non-text/image blocks to reach provider conversion. Add a user-facing `CHANGELOG.md` fix entry with reporter credit. Do not change dependency versions or unrelated MCP schema behavior.
