## Summary

Keep open. Current main still passes MCP `resource` tool-result blocks through unchanged while provider conversion paths only extract text/image blocks, and the open closing PR #75714 is unmerged.

## What This Changes

The issue requests that MCP `EmbeddedResource` tool-result blocks expose `resource.text` to the model instead of becoming an image placeholder.

## Best Possible Solution

Normalize MCP `EmbeddedResource` blocks at the bundle-MCP to Pi `AgentToolResult` boundary, add focused regression coverage, and leave provider transports generic.

## Reproduction Assessment

Yes. Return `content: [{ type: "resource", resource: { text: "..." } }]` from a bundle MCP tool; current main passes it through unchanged and the downstream provider paths ignore `resource.text`.

## Solution Assessment

Yes. Boundary normalization is the narrowest maintainable fix because the provider transports already expect Pi text/image result blocks; #75714 follows that direction but has not landed.
