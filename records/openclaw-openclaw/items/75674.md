## Summary

Keep open. Current main still preserves MCP EmbeddedResource blocks at the bundle-MCP boundary and the provider/replay paths only extract top-level text or image content; the focused closing PR #75714 is still open and unmerged.

## What This Changes

MCP tool results with EmbeddedResource text should reach the model as extracted markdown/text instead of an image placeholder.

## Best Possible Solution

Normalize MCP EmbeddedResource text into Pi text content at the bundle-MCP boundary with regression coverage, then land or replace #75714 as the focused implementation path.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: return a bundle MCP `CallToolResult.content` array containing `{ type: "resource", resource: { text: "..." } }`, and current main preserves that block until provider paths that only read top-level text/image content.

## Solution Assessment

Yes. Boundary normalization in `src/agents/pi-bundle-mcp-materialize.ts` is the narrowest maintainable fix because downstream transports already expect Pi text/image tool-result content.
