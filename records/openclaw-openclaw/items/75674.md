## Summary

Keep open. Current main still lets MCP EmbeddedResource blocks enter agent tool-result content without normalization, and the OpenAI-compatible, OpenAI replay, and Anthropic conversion paths only extract top-level text/image blocks. The open PR #75714 explicitly targets this issue, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

MCP tool results containing `type: "resource"` should pass `resource.text` through as tool-result text instead of becoming an image placeholder or empty content.

## Best Possible Solution

Land #75714 or an equivalent narrow fix that normalizes MCP `EmbeddedResource` blocks at the bundle-MCP to Pi `AgentToolResult` boundary, with focused regression coverage for text resources and bounded behavior for non-text resources.

## Reproduction Assessment

Yes. A source-level reproduction is clear: return `content: [{ type: "resource", resource: { text: "..." } }]` from a bundle MCP tool, then observe that current main passes it through unchanged and downstream provider conversion finds no top-level text.

## Solution Assessment

Yes. Boundary normalization is the narrowest maintainable direction because it converts MCP shapes once before provider-specific transport code sees the result; patching each downstream transport separately would leave the same class of bug available to future providers.
