## Summary

Keep open. Current main still casts bundle MCP tool-result content into Pi `AgentToolResult` without unwrapping MCP `resource` blocks, downstream provider/replay paths only consume top-level text/image blocks, and #75714 is an open closing PR that should land or be replaced before this issue closes.

## What This Changes

The issue asks OpenClaw to pass MCP `EmbeddedResource.resource.text` tool results through to the model instead of converting them to the image placeholder.

## Best Possible Solution

Normalize MCP `EmbeddedResource` blocks at the bundle-MCP to Pi `AgentToolResult` boundary, extracting `resource.text` into a text block and adding focused regression coverage; land #75714 or an equivalent narrow replacement.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a bundle MCP tool returning `content: [{ type: "resource", resource: { text: "..." } }]`; current main preserves that block unchanged and downstream provider paths do not extract its text.

## Solution Assessment

Yes. Boundary normalization is the narrowest maintainable direction because provider transports and replay helpers already assume Pi text/image content; patching each transport separately would leave the MCP shape mismatch exposed elsewhere.
