## Summary

Keep open. Current main and the latest release still lack MCP resource-to-text normalization at the bundle-MCP result boundary, and the linked closing PR remains open and unmerged, so the reported model-visible placeholder bug still needs the active fix path.

## What This Changes

The issue asks OpenClaw to preserve MCP `type: "resource"` tool-result text instead of converting it into an image placeholder.

## Best Possible Solution

Land or replace the linked fix by normalizing MCP resource blocks to Pi text content at the bundle-MCP materialization boundary, with regression tests for text and non-text resources.

## Reproduction Assessment

Yes, source-reproducible: an MCP `resource` content block reaches `toAgentToolResult`, remains non-text/non-image, and OpenAI/Anthropic replay code either drops it or falls back to the placeholder. I did not run a live QMD server in this read-only sweep.

## Solution Assessment

Yes, normalizing at the bundle-MCP materialization boundary is the narrowest maintainable fix because provider transports already consume only Pi text/image content. Patching each downstream transport separately would duplicate the same MCP-to-Pi conversion policy.
