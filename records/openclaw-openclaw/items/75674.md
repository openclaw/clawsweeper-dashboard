## Summary

Keep open. Current main still preserves MCP `resource` blocks at the bundle-MCP boundary, while the provider/replay paths only consume text and image-shaped content; the linked closing PR #75714 is open and should be reviewed or replaced before this issue closes.

## What This Changes

Handle MCP tool results whose content contains `type: "resource"` so `resource.text` reaches the model instead of an image placeholder.

## Best Possible Solution

Normalize MCP `EmbeddedResource` blocks once at the bundle-MCP to Pi `AgentToolResult` boundary, cover text resources with regression tests, and land #75714 or an equivalent narrow replacement.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: return `content: [{ type: "resource", resource: { text: "..." } }]` from a bundle MCP tool and current main passes it through unchanged, leaving provider conversion with no top-level text.

## Solution Assessment

Yes. Boundary normalization is the narrowest maintainable direction because provider transports already expect Pi text/image content; fixing each transport separately would leave the same MCP shape mismatch for future providers.
