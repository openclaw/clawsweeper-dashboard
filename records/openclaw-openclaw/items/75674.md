## Summary

Keep open. Current main still passes MCP `resource` tool-result blocks through the bundle-MCP boundary unchanged, while downstream provider/replay conversion still consumes only text/image-shaped content; the open closing PR #75714 should be reviewed, landed, or replaced before this issue closes.

## What This Changes

MCP text `EmbeddedResource` tool results should reach the model as their `resource.text` content instead of an image placeholder.

## Best Possible Solution

Normalize MCP `EmbeddedResource` blocks once at the bundle-MCP to Pi `AgentToolResult` boundary, cover text resources with regression tests, and land #75714 or an equivalent narrow replacement.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to return `content: [{ type: "resource", resource: { text: "..." } }]` from a bundle MCP tool; current main passes it through unchanged, leaving provider conversion with no top-level text.

## Solution Assessment

Yes. Boundary normalization is the narrowest maintainable fix because Pi/provider transports already expect text/image content; patching each transport separately would leave the same MCP shape mismatch for future providers.
