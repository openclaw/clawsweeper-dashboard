## Summary

Keep open. Current main still passes MCP `resource` content through the bundle-MCP boundary without converting `resource.text`, and the provider/replay paths only consume top-level text/image blocks; #75714 is an open closing PR, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

MCP `EmbeddedResource` tool-result blocks should expose `resource.text` to the model instead of becoming placeholder text.

## Best Possible Solution

Normalize MCP `EmbeddedResource` blocks once at the bundle-MCP to Pi `AgentToolResult` boundary, add focused regression coverage, and land #75714 or an equivalent narrow replacement.

## Reproduction Assessment

Yes. A source-level reproduction is clear: return `content: [{ type: "resource", resource: { text: "..." } }]` from a bundle MCP tool and current main passes it through unchanged, leaving downstream provider conversion with no top-level text.

## Solution Assessment

Yes. Boundary normalization is the narrowest maintainable fix because Pi/provider transports already expect text/image content; patching each transport separately would leave future providers exposed to the same MCP shape mismatch.
