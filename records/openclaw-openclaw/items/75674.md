## Summary

Keep open. Current main still passes MCP `resource` tool-result blocks through the bundle-MCP boundary unchanged, while downstream provider conversion only extracts top-level text/image blocks; the linked closing PR remains the right item to review or replace before this issue closes.

## What This Changes

The issue asks OpenClaw to preserve MCP `EmbeddedResource.resource.text` from bundled MCP tool results instead of sending an image-placeholder result to the model.

## Best Possible Solution

Normalize MCP `EmbeddedResource` blocks once at the bundle-MCP to Pi `AgentToolResult` boundary, add regression coverage for text and non-text resources, and keep provider transports generic.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to return `content: [{ type: "resource", resource: { text: "..." } }]` from a bundled MCP tool and follow the current code path into provider conversion that only reads top-level text/image blocks.

## Solution Assessment

Yes. Boundary normalization is the narrowest maintainable fix because Pi/provider transports already expect text/image content; patching each provider separately would leave the MCP-to-Pi shape mismatch in place.
