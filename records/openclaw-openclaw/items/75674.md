## Summary

Keep open. Current main still forwards MCP EmbeddedResource blocks through the bundle-MCP boundary unchanged while provider and replay paths consume only top-level text/image blocks, and #75714 is an open closing PR rather than merged code.

## What This Changes

The issue asks OpenClaw to extract text from MCP tool-result EmbeddedResource blocks instead of converting them to an image placeholder.

## Best Possible Solution

Normalize MCP EmbeddedResource blocks once at the bundle-MCP to AgentToolResult boundary, add focused regression coverage, and land #75714 or an equivalent narrow replacement.

## Reproduction Assessment

Yes. A bundle MCP tool returning content with { type: "resource", resource: { text: "..." } } reaches toAgentToolResult unchanged, and the OpenAI-compatible provider path deterministically falls back when it sees no top-level text block.

## Solution Assessment

Yes. Boundary normalization is the narrowest maintainable fix because Pi/provider transports already operate on text/image content, and patching each transport separately would leave future providers exposed.
