## Summary

Keep open: the force-pushed head fixes the earlier missing native draft payload and final-send downgrade, but it is still not merge-ready because native-capable streams can claim a temporary preview before any native draft exists, and the PR still lacks real Telegram proof.

## What This Changes

The PR adds Telegram Bot API `sendRichMessageDraft` support for private-chat live previews, keeps final replies on the rich-message send path, updates progress HTML line breaks, and adds focused tests.

## Best Possible Solution

Keep final replies on `sendRichMessage`, report temporary state only after a native draft is active, preserve fallback delivery for no-send previews, and attach redacted real Telegram proof before merge.

## Reproduction Assessment

Yes for the PR finding by source inspection: a native-capable stream can return `temporary()` before any native draft is active, and the lane deliverer then skips fallback delivery when there is no message id. I did not run a live Telegram session, so the user-visible preview behavior still needs real proof.

## Solution Assessment

No: native rich drafts are a plausible fix direction and the revised Bot API payload is much closer, but the current implementation conflates capability with active delivery. The safer fix is to track active native draft state precisely and keep fallback behavior whenever no draft actually landed.
