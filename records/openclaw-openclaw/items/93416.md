## Summary

Keep open: this PR is not merge-ready because the native draft path does not satisfy Telegram's Bot API contract, durable final replies are downgraded from rich messages, and there is no after-fix real Telegram proof.

## What This Changes

The PR switches Telegram live previews toward `sendRichMessageDraft`, changes progress draft HTML line breaks, changes durable Telegram text sends to `sendMessage` HTML, and updates the Mantis Telegram workflow secret fallback.

## Best Possible Solution

Keep durable final replies on `sendRichMessage`, add native drafts only with the required Bot API fields and private-chat gating plus fallback behavior, then verify with real Telegram proof.

## Reproduction Assessment

Yes for the review blockers: source inspection plus the Bot API contract show the native draft call is invalid, and current-main tests/source show durable rich final sends are being replaced. I did not run a live Telegram session, so the original visual preview symptom still needs real behavior proof.

## Solution Assessment

No: the current PR is not the best fix because it changes final delivery and does not implement the native draft API contract. The safer path is a narrowly scoped native-draft implementation that preserves rich final sends and falls back cleanly outside supported private-chat cases.
