## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Pass a real draft_id to native draft sends:** `extensions/telegram/src/draft-stream.ts:182-185`
  - body: Telegram's Bot API requires `draft_id` for `sendRichMessageDraft`, and it must be non-zero; this call only sends `chat_id` and `rich_message`. Because `canUseNativeDraft()` then bypasses the legacy send/edit preview path, Bot API 10.1 clients will reject every native preview update instead of showing a temporary draft. ([core.telegram.org](https://core.telegram.org/bots/api))
  - confidence: 0.93
- **[P1] Preserve reply mode across inner sendMessage chunks:** `extensions/telegram/src/bot/delivery.send.ts:160-167`
  - body: This branch inherits the legacy sendMessage loop that reuses `messageParams` for every 4096-character inner chunk. `deliverReplies` applies `replyToMode: "first"` before this helper, so one long outer chunk can now produce multiple visible Telegram messages that all reply to the same source instead of only the first chunk.
  - confidence: 0.88
