## Label Justifications

- `P1`: The PR touches Telegram live preview delivery and currently has a source-level path that can mark private-chat content delivered without any message being sent.
- `merge-risk: 🚨 compatibility`: Switching private-chat previews from persistent rich send/edit messages to ephemeral native drafts changes existing Telegram user-visible behavior and finalization expectations.
- `merge-risk: 🚨 message-delivery`: The new `temporary()` signal can suppress the fallback send path when no draft/message was actually delivered.

## Summary

Keep open: the revised PR fixes the earlier missing `draft_id` and final-send downgrade, but it still has a blocking source-level delivery regression and no real Telegram proof for the new native draft transport.

## What This Changes

The PR adds native Telegram rich-message draft previews for eligible private chats, keeps unsupported targets on existing rich preview messages, adjusts progress HTML layout, and adds unit coverage for the draft path.

## Best Possible Solution

Expose actual active native-draft state, preserve the existing fallback when no preview was sent, and merge the native draft path only with maintainer-approved live Telegram proof for preview and final persistence.

## Reproduction Assessment

Yes for the review blocker: source inspection shows the non-final lane path trusts `stream.temporary?.()` after a flush with no message id, while the PR head stream reports true whenever native drafts are available even if the debounce prevented a draft send. I did not run a live Telegram session, so the user-visible native draft behavior still needs real proof.

## Solution Assessment

No: the native draft direction is plausible, but this implementation is not the best fix until it reports actual draft activity rather than capability and has live proof for the transport change. A safer version preserves current fallback delivery whenever no native draft was posted.
