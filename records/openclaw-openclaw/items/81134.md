## Summary

## What This Changes

The issue reports that OpenClaw v2026.5.12-beta.1 failed to load Control UI Debug, AI & Agents, and infrastructure pages because a dynamically imported JavaScript asset could not be fetched.

## Best Possible Solution

Keep the build-scoped Control UI service-worker/cache implementation and have affected beta users update to v2026.5.14-beta.2 or newer.

## Reproduction Assessment

Source-reproducible, not live-reproduced. The reported beta source used a fixed service-worker cache and unversioned registration, while current main and the shipped fix rotate the worker/cache per build to avoid stale hashed chunk imports.

## Solution Assessment

Yes. The merged fix targets the update/cache layer that can leave lazy Control UI chunks stale, without changing panel code or Gateway routing unnecessarily.
