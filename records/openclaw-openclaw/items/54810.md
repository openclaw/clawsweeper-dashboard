## Summary

Keep this issue open. Current main still renders active Control UI stream text through the markdown renderer without any unclosed-fence preprocessing, and the linked fix PR #73485 is closed without being merged, so the reported live-stream indentation problem is still actionable.

## What This Changes

The issue asks for Control UI streaming code blocks to preserve indentation before a page refresh/history reload restores the final rendered message.

## Best Possible Solution

Add a narrow render-only normalization in `ui/src/ui/markdown.ts` that temporarily closes currently unclosed backtick and tilde fenced code blocks before `md.render`, then prove with colocated tests that streaming/unclosed fences render as code blocks and preserve leading indentation without changing persisted transcript text.

## Reproduction Assessment

Yes. A focused unit reproduction should call `toSanitizedMarkdownHtml` with an unclosed fenced code block containing indented lines and assert that the output is a code block with the indentation preserved; current main has the streaming render path and lacks that regression test.

## Solution Assessment

Yes. The narrowest maintainable solution is in the Control UI markdown renderer, not the provider, Gateway, or transcript storage layer, because the symptom occurs only during partial streamed rendering and refresh/history can already display the final content correctly.
