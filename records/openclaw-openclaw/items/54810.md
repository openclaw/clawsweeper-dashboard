## Summary

Current main already handles the reported live Control UI code-indentation case through the markdown-it renderer: streaming assistant text is routed through the same sanitizer, and markdown-it 14.1.1 auto-closes unclosed fenced code blocks at end of document while preserving fenced content indentation. ## What This Changes

The issue reports that the local web Control UI temporarily shows streamed code blocks with missing indentation until a page refresh restores formatting.

## Best Possible Solution

Keep the current markdown-it-based renderer as the implementation for this case; if this surface changes again, add a focused regression test for streaming unclosed fenced code so the dependency-backed behavior remains covered.
