## Summary

Keep open: current main still ignores `connectAuth.token` in password mode, and this PR is the active narrow fix for that gateway auth bug. I found no blocking code-review or security finding, but this external PR still needs redacted real Rabbit/Funnel or equivalent gateway handshake proof before merge.

## What This Changes

The PR makes password-mode gateway auth fall back from `connectAuth.password` to `connectAuth.token`, adds Rabbit/Funnel regression edge tests, and adds a changelog fix entry.

## Best Possible Solution

Land this or an equivalent refreshed narrow branch after the contributor adds real handshake proof; keep device-specific Rabbit bootstrap tokens as a separate follow-up.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: on current main, password mode passes only `connectAuth.password`, so `connectAuth: { token: "secret" }` deterministically reaches `password_missing` before the PR fallback.

## Solution Assessment

Yes. The proposed `connectAuth.password ?? connectAuth.token` fallback is the narrowest maintainable fix because it stays inside password mode, preserves password precedence, and still compares against the configured shared password.
