## What This Changes

The branch changes gateway password-mode auth to use `connectAuth.password ?? connectAuth.token`, adds Rabbit/Funnel regression tests, and adds a changelog fix entry.

## Best Possible Solution

Land the narrow password-mode token fallback with regression tests after redacted real handshake proof and fresh validation.

## Reproduction Assessment

Yes. Source inspection on current main shows WebSocket auth preserves `connectAuth.token`, but password mode passes only `connectAuth.password` to the validator, so a token-only shared-secret handshake reaches `password_missing`.

## Solution Assessment

Yes. The PR keeps the fallback inside password mode, preserves explicit password precedence, and still compares the supplied value against the configured shared password; the added tests cover the important edge cases.
