## Summary

Keep open: current main and the latest release still lack the password-mode token fallback, and this PR is a focused fix with targeted tests. It should not merge yet because contributor real behavior proof is still absent and the branch is currently conflicting.

## What This Changes

The PR changes gateway password-mode auth to use `connectAuth.password ?? connectAuth.token`, adds Rabbit/Funnel regression tests, and adds a changelog fix entry.

## Best Possible Solution

Land the narrow password-mode token fallback with its regression tests after branch repair and redacted real Rabbit/Funnel or equivalent gateway handshake proof.

## Reproduction Assessment

Yes. Source inspection on current main shows password mode passes only `connectAuth?.password` while the WebSocket auth context preserves `connectAuth.token`, so a token-only shared-secret handshake deterministically returns `password_missing`; I did not run a live Rabbit/Funnel device path.

## Solution Assessment

Yes. The proposed `connectAuth.password ?? connectAuth.token` fallback is the narrowest maintainable fix because it stays inside password mode, preserves password precedence, and still compares against the configured shared password.
