## What This Changes

The PR changes gateway password-mode auth to fall back from `connectAuth.password` to `connectAuth.token`, adds Rabbit/Funnel regression and edge-case tests, and adds a changelog fix entry.

## Best Possible Solution

Rebase or repair this PR against current main, keep the password-mode fallback and regression tests narrow, preserve the contributor changelog credit, then land after the changed gateway checks are green; any move toward device-specific Rabbit bootstrap tokens should be handled as a separate follow-up.

## Reproduction Assessment

Yes. A password-mode `authorizeGatewayConnect` call with `connectAuth: { token: "secret" }` deterministically reaches `authorizePasswordAuth` with an undefined `connectPassword` on current main and returns `password_missing`; the PR adds the missing unit coverage for that path.

## Solution Assessment

Yes. The proposed `connectAuth.password ?? connectAuth.token` fallback is the narrowest maintainable fix for Rabbit/Funnel compatibility because it stays inside password mode, preserves password precedence, and still compares the same configured shared secret.
