## Summary

Keep this PR open: current main and the latest release still reject token-only password-mode handshakes, and the PR’s fallback/tests look narrowly correct, but it still needs contributor real behavior proof and a branch repair/rebase before merge.

## What This Changes

The PR changes gateway password-mode auth to fall back from `connectAuth.password` to `connectAuth.token`, adds Rabbit/Funnel regression coverage, and adds a changelog fix entry.

## Best Possible Solution

Land the narrow password-mode token fallback with its regression tests after the branch is repaired and redacted real Rabbit/Funnel or equivalent gateway handshake proof is provided.

## Reproduction Assessment

Yes. Source inspection on current main shows password mode passes only `connectAuth?.password` while the WebSocket auth context preserves `connectAuth.token`, so a token-only shared-secret handshake deterministically returns `password_missing`; I did not run a real Rabbit device.

## Solution Assessment

Yes. The proposed `connectAuth.password ?? connectAuth.token` fallback is the narrowest maintainable fix because it stays inside password mode, preserves password precedence, and still compares against the configured shared password.
