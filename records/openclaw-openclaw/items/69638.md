## Summary

Keep open. Current main still rejects the token-only password-mode handshake described by the Rabbit/Funnel report, while the PR is a narrow and appropriate fix that appears to need branch repair/rebase before it can merge.

## What This Changes

The PR updates gateway password-mode auth to fall back to `connectAuth.token` when `connectAuth.password` is absent, adds Rabbit/Funnel regression and edge-case tests, and adds a changelog fix entry.

## Best Possible Solution

Refresh the branch against current main, keep the fallback and regression coverage narrow, place the changelog entry in the active section with contributor credit preserved, and land after the gateway changed checks pass; device-specific Rabbit bootstrap tokens should remain a separate follow-up.

## Reproduction Assessment

Yes. On current main, `authorizeGatewayConnect` in password mode with `connectAuth: { token: "secret" }` reaches `authorizePasswordAuth` with `connectPassword` undefined and deterministically returns `password_missing`.

## Solution Assessment

Yes. The proposed `connectAuth.password ?? connectAuth.token` fallback is the narrowest maintainable fix because it stays inside password mode, preserves password precedence, and still compares against the configured shared password.
