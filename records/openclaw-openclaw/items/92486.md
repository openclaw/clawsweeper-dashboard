## Summary

Current main still has the reported source-level mismatch, and this issue has two open closing PRs that specifically target the fix, so it should stay open until one implementation lands.

## What This Changes

The issue reports that config-driven subagent model overrides are resolved during spawn planning but discarded before the child agent run starts.

## Best Possible Solution

Land one focused fix that preserves config-selected subagent model override provenance through session creation while keeping legacy stale auto-fallback cleanup constrained to old fallback state.

## Reproduction Assessment

Yes from source: current main creates config-driven subagent model overrides as `auto` without origin fields, and the runtime then treats that shape as legacy stale fallback state before applying stored overrides. I did not run a live spawn because this read-only review may not create test or runtime artifacts.

## Solution Assessment

Yes, the best fix direction is to preserve provenance for config-selected subagent model overrides or otherwise make the legacy cleanup exclude fresh spawn patches. Because two open closing PRs already target that path, this issue should remain open until one lands.
