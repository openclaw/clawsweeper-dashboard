## Summary

Keep this issue open: current main and the latest release still create an originless auto model override for config-selected subagent models, and two open PRs already reference this issue with closing syntax, so it should close only after one fix lands.

## What This Changes

The issue reports that `agents.defaults.subagents.model` is resolved for `sessions_spawn` but then discarded because the persisted `modelOverrideSource:"auto"` entry lacks fallback-origin metadata.

## Best Possible Solution

Land one focused provenance fix that sets and preserves origin metadata for config-selected subagent model overrides, then close this issue from the merged PR.

## Reproduction Assessment

Yes. Source inspection on current main shows the spawn plan writes an originless automatic model override, the legacy cleanup predicate matches it, and the child run suppresses the stored override before model selection.

## Solution Assessment

Yes. The narrowest maintainable fix is to complete or preserve provenance at spawn/session-patch time for config-selected subagent models, rather than weakening cleanup for genuinely stale legacy fallback entries.
