The issue asks OpenClaw to preserve `agents.defaults.subagents.model` selections through `sessions_spawn` child startup instead of silently falling back to the agent default model.

## Best Possible Solution

Land one narrow fix that adds provenance for config-selected auto model overrides at spawn planning and direct child persistence while preserving legacy cleanup for true fallback pins.

## Solution Assessment

Yes: the best narrow fix is to set and preserve origin metadata where the config-selected override is created, rather than weakening cleanup for genuinely stale legacy auto-fallback entries.
