## What This Changes

## Best Possible Solution

Land this fallback only after maintainers explicitly accept degraded keyword fallback semantics; otherwise keep the narrower unavailable-message path in https://github.com/openclaw/openclaw/pull/69199 as the supported behavior.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes at source/proof level: current main returns unavailable manager errors when `MemoryIndexManager.get` fails, and the PR body shows the production CLI path with `node:sqlite` blocked at the loader seam.

## Solution Assessment

Is this the best way to solve the issue? Yes as a technical mitigation: the manager-resolution seam is the narrow place to handle a missing builtin backend while leaving SQLite/vector search canonical, but the fallback policy and scan tradeoff still require maintainer acceptance.
