## Summary

## What This Changes

Adds a memory-core keyword fallback manager for missing node:sqlite errors and a regression test that searches markdown memory files when the builtin SQLite manager cannot load.

## Best Possible Solution

Land the fallback only after maintainers explicitly accept degraded keyword-fallback semantics and scan bounds; otherwise keep the narrower unavailable-message path in https://github.com/openclaw/openclaw/pull/69199 as the supported behavior.

## Reproduction Assessment

I did not run a separate physical Node build without node:sqlite in this read-only checkout.

## Solution Assessment

Yes as an acceptable mitigation if maintainers accept the product tradeoff: the patch lives at the manager-resolution seam, preserves the canonical SQLite path, and reuses the memory-file boundary. The safer alternative, if fallback policy is rejected, is to keep the unavailable-message improvement in https://github.com/openclaw/openclaw/pull/69199.
