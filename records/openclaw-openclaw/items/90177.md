## Summary

Keep open: current main and the latest release still run the Feishu empty-message guard before fetching quoted content, so a reply whose visible body is only the bot mention can be dropped before the parent message is read.

## What This Changes

The issue requests that Feishu group replies containing only an @bot mention still fetch and dispatch quoted parent content instead of being discarded as empty.

## Best Possible Solution

Keep the true blank-message skip, but compute quoted parent content first and only drop the event when current text, media, audio transcript, and allowed quoted content are all empty.

## Reproduction Assessment

Yes from source, but not from a fresh live run: a group @bot-only reply can normalize to empty content, then `handleFeishuMessage` returns before `getMessageFeishu` reads `ctx.parentId`. The reporter's log matches that current-main ordering.

## Solution Assessment

Yes, the suggested direction is the right fix shape: move or factor quote fetching before the effective-empty decision while preserving the existing blank-turn protection for genuinely empty messages.
