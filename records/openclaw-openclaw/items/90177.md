## Summary

Keep open: current main and the latest release still run the Feishu empty-message guard before fetching quoted parent content, and the open linked fix PR is the active implementation path rather than a reason to close this issue before merge.

## What This Changes

The issue asks Feishu to dispatch @bot-only group replies that quote a parent message by treating the quoted parent as effective input instead of dropping the event as blank.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/90192 or a narrow equivalent that fetches allowed quoted content before the effective-empty decision while preserving true blank-message skips and group mention gating.

## Reproduction Assessment

Yes from source, but not from a fresh live run: an @bot-only Feishu reply can normalize to empty content, then the handler returns before reading `ctx.parentId`. The reporter's runtime log matches that current-main path.

## Solution Assessment

Yes, the best fix shape is narrow: keep the Feishu plugin's blank-turn protection, but compute allowed quoted parent content before deciding the effective user input is empty. The exact PR should still be reviewed for live/group proof before this issue closes.
