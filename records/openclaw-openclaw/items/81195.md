## Summary

Keep open: the PR targets a valid Telegram bug, but the patch still misses inherited/override requireMention configurations and over-suppresses failures for messages that actually mention the bot. The supplied proof is a mocked Vitest run rather than the real Telegram or equivalent bot-to-bot proof expected for visible reply behavior.

## What This Changes

The PR adds Telegram media-fetch error gating for requireMention groups, a regression test, and a changelog entry for https://github.com/openclaw/openclaw/issues/81181.

## Best Possible Solution

Use the same effective requireMention and mention facts as the inbound context to skip only unmentioned non-audio media in mention-only groups, while preserving DM and explicitly mentioned media-failure replies.

## Reproduction Assessment

Yes by source inspection: current main resolves Telegram media and can send the download-failure reply before the downstream no-mention gate runs. I did not run a live Telegram/WSL reproduction in this read-only review, so this remains source_reproducible rather than live reproduced.

## Solution Assessment

No: this patch gates on group requireMention configuration rather than the actual mention decision and does not reuse the canonical requireMention resolution. The narrower maintainable fix is to carry or compute canonical mention eligibility before non-audio media download failure replies are emitted.
