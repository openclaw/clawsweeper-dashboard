## Summary

Keep open: the latest head fixes part of the earlier review by using the Telegram group-policy resolver, but the patch still disagrees with session activation and suppresses feedback for explicitly addressed media failures; the proof is still mock-only.

## What This Changes

The PR passes group mention policy into Telegram media processing, suppresses media-download failure replies in mention-only groups, adds a regression test, and adds a changelog entry.

## Best Possible Solution

Carry the same effective mention decision facts early enough to suppress only unmentioned non-audio media-fetch replies, while preserving DMs, /activation overrides, explicit mentions, reply-to-bot handling, and audio preflight.

## Reproduction Assessment

Yes by source inspection: current main resolves Telegram media and can send the download-failure reply before the downstream requireMention no-mention gate runs. I did not run a live Telegram/WSL reproduction, so this is source_reproducible rather than live reproduced.

## Solution Assessment

No. The latest head fixed the wildcard/topic config lookup problem, but the suppression still does not reuse session activation or actual mention/reply facts, so it can both miss mention-only sessions and over-suppress addressed requests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Honor session activation in the suppression gate:** `extensions/telegram/src/bot-handlers.runtime.ts:2411-2420`
  - body: This recomputes requireMention from config/group policy only, but the existing context path gives `/activation always|mention` session state precedence via `resolveGroupActivation`. A session toggled to mention can still emit the spam reply, while a session toggled to always can lose a useful media-failure reply, so the gate can disagree with the message that would actually be handled.
  - confidence: 0.87
- **[P2] Preserve failure feedback for addressed group media:** `extensions/telegram/src/bot-handlers.runtime.ts:1467-1468`
  - body: The early return runs for every requireMention group before mention facts are computed. If a user explicitly mentions the bot or replies to the bot with a photo and the download fails, the bot now only logs and says nothing even though those messages are valid group interactions; the gate needs the actual mention decision, not only the group policy.
  - confidence: 0.86
