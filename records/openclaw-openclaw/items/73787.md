## Summary

Keep open; current main does not already implement the branch, and the proposed filter still misses current inbound metadata blocks, leaving the privacy/prompt-contamination failure mode partially unsolved.

## What This Changes

The PR adds memory-lancedb envelope metadata detection and sanitization, filters recalled memories after overfetching, strips media annotations from recalled memory text, expands tests, and adds a changelog entry.

## Best Possible Solution

Keep the plugin-level memory boundary, but align detection/sanitization with the full current inbound metadata shape and add location plus structured-context regression tests before merge.

## Reproduction Assessment

Yes. A focused PR-branch unit test with `Location (untrusted metadata):` or `WhatsApp contact (untrusted metadata):` followed by fenced JSON containing a memory trigger would show the block bypasses the new sanitizer/detector.

## Solution Assessment

No. Filtering inside memory-lancedb is the right narrow direction, but the implementation should match `buildInboundUserContextPrefix`’s generic untrusted-metadata JSON block shape rather than a stale fixed sentinel list.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Cover all untrusted metadata blocks:** `extensions/memory-lancedb/index.ts:546-553`
  - body: `buildInboundUserContextPrefix` can emit `Location (untrusted metadata):`, `Structured object (untrusted metadata):`, and caller-provided `<label> (untrusted metadata):` fenced JSON blocks, but this new list only detects six fixed sentinels. Those blocks bypass both detection and sanitization, so a location/contact metadata block with a trigger word can still be stored and recalled. Please cover the generic line-start untrusted-metadata JSON block shape or add the current block types with tests.
  - confidence: 0.92

  - body: When any media marker is present, this replacement collapses all `\s{2,}` in the memory; `\s` includes newlines, so unrelated multi-line memory text is flattened before recall. Limit cleanup to spaces/tabs around removed markers or otherwise preserve line breaks.
