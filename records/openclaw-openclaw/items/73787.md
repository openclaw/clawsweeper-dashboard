## Summary

Keep open: the latest head improves the memory-lancedb boundary, but it still loses valid captures when current inbound metadata uses generic Location or structured labels because those blocks are detected but not stripped before the capture gate.

## What This Changes

The PR adds memory-lancedb envelope metadata detection and sanitization, filters contaminated recall results, strips media annotations during memory prompt escaping, expands tests, and adds a changelog entry.

## Best Possible Solution

Update sanitizeForMemoryCapture to remove the same generic fenced inbound metadata/context block shape that buildInboundUserContextPrefix emits, including Location and structured labels, before the shouldCapture gate.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a Location or structured metadata block prepended to 'I always prefer dark mode' survives the PR sanitizer, then shouldCapture rejects the candidate because the generic label remains.

## Solution Assessment

No. The plugin-level filtering boundary is the right place, but the sanitizer needs to strip the generic inbound metadata block shape that the detector now recognizes instead of only the stale fixed sentinel list.
