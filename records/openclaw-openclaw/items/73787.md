## Summary

Keep this PR open. The direction is right and the PR supersedes noisy closed PR #67475, but the current patch still misses inbound metadata blocks that current main can emit, so the central contamination fix is incomplete.

## What This Changes

The PR adds memory-lancedb envelope-sludge detection, capture sanitization, recall filtering/overfetching, media-annotation prompt escaping, regression tests, and a changelog entry.

## Best Possible Solution

Keep the plugin-scoped fix, but make memory-lancedb detect and strip every current inbound metadata JSON block shape without importing core internals, then add regression tests for Location and generic structured context.

## Reproduction Assessment

Yes. A user message containing a current-main `Location (untrusted metadata):` fenced JSON block plus text like `I always prefer this cafe` is left intact by the PR sanitizer and still satisfies the memory trigger path.

## Solution Assessment

No. The proposed extension-local sanitizer is the right boundary, but it should match the producer's full untrusted metadata contract rather than a partial duplicated sentinel list.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Cover all inbound metadata blocks before capture:** `extensions/memory-lancedb/index.ts:546-553`
  - body: `buildInboundUserContextPrefix` now emits `Location (untrusted metadata):` and arbitrary structured labels such as `Structured object (untrusted metadata):`, but this list only covers the older six labels. Since `sanitizeForMemoryCapture()` iterates this list, a location/structured block followed by a trigger phrase remains in the text that gets embedded and stored. Please detect line-anchored untrusted-metadata JSON blocks generically, or add all current producer labels with regression coverage.
  - confidence: 0.91
- **[P3] Preserve newlines when removing media annotations:** `extensions/memory-lancedb/index.ts:531-533`
  - body: When a recalled memory contains any `[media attached: ...]` annotation, this cleanup runs `/\s{2,}/g` across the whole memory, flattening newlines and indentation that may carry useful context. Limit the normalization to horizontal whitespace around the removed annotation or preserve existing line breaks.
  - confidence: 0.78
