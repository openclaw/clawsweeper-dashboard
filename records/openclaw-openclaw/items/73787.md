## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Handle chat-window structured context labels:** `extensions/memory-lancedb/index.ts:619-620`
  - body: Current main can emit chat-window context headers such as `Chat window (untrusted, before current message):`, but this matcher only accepts `(untrusted metadata)` and `(untrusted, for context)`. Those blocks are plain transcript lines rather than fenced JSON, so a trigger word inside the transcript can still pass the capture gate and be stored as memory sludge. Extend the detector/sanitizer to cover the current chat-window qualifier shapes or reuse an equivalent inbound-meta stripping contract.
  - confidence: 0.9
- **[P2] Strip current metadata blocks before capture:** `extensions/memory-lancedb/index.ts:593-600`
  - body: `sanitizeForMemoryCapture` still iterates only this fixed sentinel list, which has the old `Replied message` label and omits current `Reply target`, `Reply chain`, `Location`, and custom structured JSON labels. In the auto-capture path, the leftover metadata then goes into `shouldCapture`, so valid user preferences paired with those blocks are dropped instead of stored cleanly. Strip generic `formatUntrustedJsonBlock` shapes before calling `shouldCapture`, or align the list with the full current `buildInboundUserContextPrefix` contract.
  - confidence: 0.88
