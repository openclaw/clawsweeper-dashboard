## Summary

The PR is useful but not merge-ready: source inspection shows current inbound-meta still emits reply-chain and chat-window untrusted context headers that the new matcher and sanitizer do not cover, so the contaminated-memory failure can persist.

## What This Changes

The branch adds envelope-sludge detection, capture sanitization, recall filtering, media-annotation stripping, regression tests, and a changelog entry for the memory-lancedb plugin.

## Best Possible Solution

Extend the plugin-local detector, sanitizer, and regression tests to cover every current `buildInboundUserContextPrefix` untrusted block shape, including reply-chain `nearest first` and chat-window `untrusted, ...` labels, without importing core internals into the plugin.

## Reproduction Assessment

Yes. Source inspection plus a direct regex probe shows the PR head does not recognize current reply-chain and chat-window untrusted context headers, so trigger words inside those blocks can still satisfy `shouldCapture`.

## Solution Assessment

No. A plugin-local sanitizer and recall filter is the right boundary, but this implementation needs to match the full current inbound metadata contract before it is the narrow maintainable fix.
