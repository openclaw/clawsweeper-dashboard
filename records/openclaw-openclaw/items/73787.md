## Summary

Keep open: the PR is aimed at a real memory-lancedb contamination bug, but current head still misses current inbound context shapes and the real-behavior proof does not show an actual contaminated capture/recall rejection.

## What This Changes

The PR adds memory-lancedb envelope metadata detection and sanitization, filters auto-recall results, strips media annotations from recalled memories, and adds regression tests plus a changelog entry.

## Best Possible Solution

Keep the plugin-level fix, but make detection and sanitization follow the full current inbound metadata contract before requiring redacted real capture/recall proof.

## Reproduction Assessment

Yes. Source inspection shows current main emits chat-window, reply-chain, reply-target, location, and generic structured-context labels that PR head does not fully strip or detect; I did not run a live repro because this review is read-only.

## Solution Assessment

No. The memory-lancedb boundary is the right place to fix polluted long-term memory, but this patch should parse or strip the full current inbound metadata contract instead of mixing a partial fixed list with a narrower detector.
