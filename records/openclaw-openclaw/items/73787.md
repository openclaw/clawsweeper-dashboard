## Summary

Keep this PR open. Current main still has the memory-lancedb envelope-metadata capture problem and this PR is scoped to the right plugin boundary, but the latest head still drops valid auto-capture text when core prepends generic `Location` or structured untrusted metadata because the sanitizer only strips the fixed sentinel list.

## What This Changes

The PR adds memory-lancedb envelope metadata detection and sanitization, filters contaminated recall results, stores sanitized auto-capture text, strips recalled media annotations, expands tests, and updates the changelog.

## Best Possible Solution

Broaden the plugin sanitizer to strip generic line-start inbound metadata fenced blocks emitted by current core while preserving following user text, then keep the recall filtering and changelog coverage from this PR.

## Reproduction Assessment

Yes, from source inspection. A user message containing a generic `Location (untrusted metadata):` fenced block followed by `I always prefer dark mode` reaches auto-capture; the PR sanitizer leaves the block, then shouldCapture rejects the whole candidate because the generic header still matches the new detector.

## Solution Assessment

No. The plugin-level filtering boundary is the right maintainable approach, but the current patch must strip generic inbound metadata block shapes before capture gating instead of only detecting and rejecting them.
