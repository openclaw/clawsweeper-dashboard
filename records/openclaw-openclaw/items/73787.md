## What This Changes

The PR adds memory-lancedb envelope metadata detection and sanitization, filters contaminated recall results, strips recalled media annotations, expands regression tests, and updates CHANGELOG.md.

## Best Possible Solution

Broaden sanitizeForMemoryCapture to strip every inbound block emitted by buildInboundUserContextPrefix, including generic JSON labels and chat-window/reply-chain context, then add focused auto-capture regression coverage for preserving clean user memory after those blocks.

## Reproduction Assessment

Yes, from source inspection. A current-main Location or structured inbound-context block followed by `I always prefer dark mode` survives the PR sanitizer; shouldCapture then sees remaining metadata on the sanitized candidate and rejects or contaminates the capture path.

## Solution Assessment

No. The plugin-level filtering boundary is the right place, but the sanitizer must match the current generic inbound metadata contract instead of relying on a stale fixed sentinel list.
