Keep this PR open: it targets a real memory-lancedb contamination bug and is the active clean reimplementation after https://github.com/openclaw/openclaw/pull/67475, but the current head would regress current-main capture-trigger behavior and still does not sanitize every inbound metadata shape emitted by core. ## What This Changes

The PR adds memory-lancedb envelope metadata detection and sanitization, filters contaminated recall results, strips recalled media annotations, overfetches recall candidates, and updates tests plus the changelog.

## Best Possible Solution

Keep the plugin-level fix direction, rebase/reconcile it with current main, preserve the current trigger contract, strip every buildInboundUserContextPrefix block shape before capture, and attach redacted real behavior proof.

## Reproduction Assessment

Yes, source-reproducible. Current main supports custom/CJK capture triggers and emits inbound metadata labels that the PR head either drops or does not strip before capture.

## Solution Assessment

No. Filtering in memory-lancedb is the right boundary, but this branch needs to preserve current shouldCapture semantics and use a generic sanitizer that matches the current inbound metadata contract.
