## Summary

Keep this PR open; the plugin-scoped direction is right, but the current patch still misses inbound `Location` and arbitrary structured untrusted metadata blocks, so contaminated memories can still be captured and recalled.

## What This Changes

The PR adds memory-lancedb envelope-sludge detection and sanitization, filters contaminated recall results, strips media annotations from recalled memories, expands tests, and adds a changelog entry.

## Best Possible Solution

Keep the memory-lancedb boundary, but make detection/sanitization cover the full current inbound metadata shape, including generic line-start `<label> (untrusted metadata):` JSON blocks, with Location and structured-context regression tests.

## Reproduction Assessment

Yes. A focused unit test using `Location (untrusted metadata):` or `Structured object (untrusted metadata):` followed by fenced JSON plus a memory trigger would show the PR leaves the metadata block in sanitized capture.

## Solution Assessment

No. Filtering in `memory-lancedb` is the right narrow solution, but the implementation should align with `buildInboundUserContextPrefix` instead of duplicating a stale six-sentinel list.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Cover all inbound metadata blocks:** `extensions/memory-lancedb/index.ts:545-552`
  - body: `buildInboundUserContextPrefix` now emits `Location (untrusted metadata):` and arbitrary structured `<label> (untrusted metadata):` blocks, but this new sentinel list only covers six older headers. Those current core-injected blocks bypass both `looksLikeEnvelopeSludge()` and `sanitizeForMemoryCapture()`, so a location or structured block with a memory trigger can still be embedded and recalled as long-term memory. Please cover the generic line-start untrusted-metadata JSON block shape, or add the missing current block types with tests.
  - confidence: 0.92
- **[P3] Preserve newlines when stripping media annotations:** `extensions/memory-lancedb/index.ts:532`
  - body: When a memory contains any `[media attached: ...]` marker, this cleanup runs `/\s{2,}/g` across the entire memory. Because `\s` includes newlines, one media marker flattens unrelated multi-line memory text before recall; restrict normalization to spaces/tabs around the removed marker or otherwise preserve line breaks.
  - confidence: 0.78
