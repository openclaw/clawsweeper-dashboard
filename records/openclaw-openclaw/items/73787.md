## Summary

Keep this PR open, but do not merge it as-is: the extension-local direction is right, yet PR head still misses indented metadata headers and generic untrusted-metadata blocks that current main can emit.

## What This Changes

The PR adds memory-lancedb envelope-sludge detection/sanitization, applies it to auto-capture and auto-recall, strips recalled media annotations, overfetches recall candidates, adds tests, and updates CHANGELOG.md.

## Best Possible Solution

Keep the fix in `extensions/memory-lancedb`, but mirror the current inbound metadata producer/stripper contract with optional leading whitespace and generic fenced `(untrusted metadata):` blocks, then add focused regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction: feed PR-head capture/recall filtering an indented `Sender (untrusted metadata):` plain-text block or a `Location (untrusted metadata):` fenced JSON block containing `always`; those forms bypass the new filters and can satisfy `MEMORY_TRIGGERS`.

## Solution Assessment

No, not as-is. The extension boundary and sanitize-before-store approach are maintainable, but the concrete matching rules are narrower than current main's metadata contract.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Accept indented metadata headers:** `extensions/memory-lancedb/index.ts:581`
  - body: The new detector and sanitizer only match metadata headers at column zero, while current main's metadata stripper compares `line.trim()` to the same sentinels. An indented `Sender (untrusted metadata):` or `Untrusted context` block can bypass filtering and still be captured or recalled, so allow leading whitespace and cover that fixture.
  - confidence: 0.9
- **[P2] Detect generic untrusted metadata blocks:** `extensions/memory-lancedb/index.ts:546-565`
  - body: Current main can emit `Location (untrusted metadata):` and arbitrary `<label> (untrusted metadata):` JSON blocks, but the PR only lists six fixed sentinels plus a narrow JSON-key fallback. Those blocks can contain trigger words and survive capture/recall filtering, so detect generic line-start untrusted metadata blocks followed by a JSON fence.
  - confidence: 0.89
- **[P3] Preserve newlines after media annotation stripping:** `extensions/memory-lancedb/index.ts:532`
  - body: When any `[media attached: ...]` annotation is present, this collapses `\s{2,}` across the whole memory, which also flattens newlines and indentation unrelated to the removed marker. Limit cleanup to spaces/tabs around the removed annotation so multi-line memories keep their formatting.
  - confidence: 0.82
