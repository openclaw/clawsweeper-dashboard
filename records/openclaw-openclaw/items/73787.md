## Summary

Keep this PR open. Current main still lacks envelope-sludge filtering, and the PR is the right extension-local direction, but the force-pushed head still misses privacy-sensitive current metadata forms during capture/recall filtering.

## What This Changes

The PR adds memory-lancedb envelope metadata detection and sanitization, filters auto-capture and auto-recall results, strips recalled media annotations, expands tests, and adds a changelog entry.

## Best Possible Solution

Keep the fix in memory-lancedb, but make the filter mirror current inbound metadata contracts: optional indentation and any fenced line-label ending in (untrusted metadata):, with focused regression tests.

## Reproduction Assessment

Yes. Feed the PR head detector an indented Sender block, a Location block, or a WhatsApp contact structured metadata block; source inspection and a regex replay show those cases are not classified as sludge.

## Solution Assessment

No, not as-is. The extension-local boundary is right, but the detector and sanitizer need to track all current producer forms before this is the narrowest maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Accept indented metadata sentinels:** `extensions/memory-lancedb/index.ts:581`
  - body: The detector requires known metadata headers to start at column zero, while current core stripping treats trimmed lines as sentinel lines. A stored memory beginning with an indented `Sender (untrusted metadata):` block is not filtered during recall, so allow optional leading whitespace wherever sentinel headers are detected and add regression coverage.
  - confidence: 0.9
- **[P2] Reject every untrusted metadata label:** `extensions/memory-lancedb/index.ts:546-565`
  - body: Current prompt assembly also emits `Location (untrusted metadata):` and arbitrary structured labels such as `WhatsApp contact (untrusted metadata):`. Because this list only covers six labels and the JSON fallback only checks a narrow key set, those current metadata blocks can survive capture or recall filtering; use a generic line-anchored untrusted-metadata fenced-block rule or otherwise sync the full producer contract.
  - confidence: 0.88
