## Summary

Keep this PR open. The latest head fixes several earlier review comments, but the capture sanitizer still does not strip current generic inbound metadata blocks, so valid user memory text can be dropped after Location or structured-context metadata is prepended.

## What This Changes

The PR adds memory-lancedb envelope metadata detection and sanitization, filters contaminated recall results, strips recalled media annotations, expands regression tests, and updates the changelog.

## Best Possible Solution

Broaden sanitizeForMemoryCapture to remove generic line-start inbound metadata fenced blocks emitted by buildInboundUserContextPrefix, then add Location and structured-context auto-capture regression tests.

## Reproduction Assessment

Yes from source inspection. A Location or structured untrusted metadata block prepended before `I always prefer dark mode` survives sanitization, then shouldCapture rejects the candidate because the generic metadata label is still present.

## Solution Assessment

No. The plugin-level filtering boundary is right, but the sanitizer must match the current generic inbound metadata block shape instead of relying on a stale fixed sentinel list.
