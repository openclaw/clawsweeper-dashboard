## Summary

Keep open. The PR addresses a real memory-lancedb contamination path, but it is not ready to merge because the new filter does not match current inbound metadata contracts for indentation, Location, and structured untrusted metadata; those gaps leave the same privacy and prompt-poisoning class partially open.

## What This Changes

The PR adds memory-lancedb envelope metadata detection and sanitization for auto-capture and recall, strips recalled media annotations, overfetches then filters recall results, and expands extension tests.

## Best Possible Solution

Revise the extension-local filter so it mirrors the current inbound metadata producer contract: accept optional leading whitespace, cover Location and generic structured '(untrusted metadata):' fenced JSON blocks without importing core internals, keep capture/recall filtering in memory-lancedb, add focused regression tests, and add the required changelog entry before rerunning the extension test and changed gate.

Yes. ## Solution Assessment

No, not as-is. The extension-local boundary is right, but the proposed detector is narrower than the current inbound metadata contract, so the maintainable fix needs broader contract-matching filters plus regression tests and changelog coverage.
