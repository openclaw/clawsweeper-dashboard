## Summary

Keep open. The PR is a useful plugin-scoped fix, but latest head still misses current `Location` and arbitrary structured untrusted metadata blocks emitted by main, so envelope metadata can still be captured and recalled; it also flattens multi-line memory text when removing media annotations.

## What This Changes

The branch adds memory-lancedb envelope-sludge detection, capture sanitization, recall filtering/overfetching, media-annotation stripping, tests, and a changelog entry.

## Best Possible Solution

Keep the fix plugin-scoped, but make detection and sanitization cover the current generic line-start untrusted-metadata fenced JSON shape, with Location and structured-context regression tests.

## Reproduction Assessment

Yes. A focused unit test with `Location (untrusted metadata):` or `WhatsApp contact (untrusted metadata):` followed by fenced JSON containing a trigger such as `always` would show the PR leaves the metadata block capturable.

## Solution Assessment

No. Filtering in `memory-lancedb` is the right narrow boundary, but the implementation should align with the current `buildInboundUserContextPrefix` output instead of duplicating a stale six-sentinel list.
