## What This Changes

The PR adds memory-lancedb envelope metadata detection and sanitization, filters recalled memories before prompt injection, strips media annotations before prompt escaping, updates tests, and adds a changelog entry.

## Best Possible Solution

Keep the plugin-level memory boundary, but make sanitizer and detector coverage derive from the full current inbound metadata contract, then require direct redacted capture and recall proof before merge.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: current main emits chat-window and generic inbound metadata shapes that the PR head either misses or leaves in sanitized text before capture gating. I did not run the extension test suite because this review is read-only and tests may write artifacts in this checkout.

## Solution Assessment

No. The memory-lancedb plugin is the right boundary for protecting long-term memory, but the proposed implementation should strip the full current metadata contract instead of combining a partial fixed list with a narrower detector.
