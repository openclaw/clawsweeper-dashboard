## Summary

This PR should close because the same default-agent auth upgrade regression is already owned by an open, mergeable, proof-positive doctor-migration PR, while this branch adds a runtime fallback that conflicts with the current SQLite auth-store boundary and can mark credential-less metadata as eligible.

## What This Changes

The PR adds a runtime auth fallback that treats config-defined api_key/token profiles as eligible and injects metadata-only config profiles into an empty auth store.
