## Summary

Keep open: current main and the latest release still support active-turn image understanding and media path persistence, but they do not implement durable image-summary/media-memory reuse for later compacted or pruned agent context.

## What This Changes

Request durable persisted metadata and image-understanding summaries for agent image attachments so later compacted or raw-replay-disabled context can reuse them.

## Best Possible Solution

Define a generic persisted media-memory/session contract that stores bounded metadata plus summaries in canonical state and exposes summary stubs to context assembly while preserving native image use for fresh turns.

## Reproduction Assessment

Yes, source-reproducible: current main persists only MediaPath/MediaPaths/MediaType/MediaTypes, applies image descriptions only to active ctx.Body/ctx.MediaUnderstanding, and prunes old image history to generic markers.

## Solution Assessment

Unclear as originally scoped: the direction is valid, but the maintainable solution is a generic session/media-memory and context-engine contract rather than a lossless-claw-only handoff.
