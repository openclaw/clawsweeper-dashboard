## Summary

Keep open: current main and the latest release still keep image-understanding output in transient reply context and persist only basic media path/type fields, so the requested durable image-summary memory remains real OpenClaw work.

## What This Changes

The issue requests durable storage and reuse of image attachment metadata plus image-understanding summaries in agent context, instead of relying on transient native image bytes or later assistant prose.

## Best Possible Solution

Add a generic media-memory contract that persists bounded image metadata and summaries in canonical session state, reuses existing media-understanding output, and lets context engines assemble summary stubs without replaying raw image bytes by default.

## Reproduction Assessment

Yes, source-reproducible for the current gap: current main persists only basic media path/type fields, keeps image descriptions on transient context, and prunes older image data to generic markers. I did not run a live image session because this review is read-only.

## Solution Assessment

Unclear that the exact requested budgets and replay controls are the final best API. The maintainable path is a generic core/plugin media-memory seam with canonical storage and context-engine reuse, not a one-off lossless-claw handoff.
