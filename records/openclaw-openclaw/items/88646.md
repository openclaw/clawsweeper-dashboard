## Summary

Keep open: current main still has transient image-understanding output and raw/native image forwarding, but no durable media-memory record that stores reusable image summaries, metadata, or summary stubs for later compacted/session context.

## What This Changes

Issue requests persisted image attachment metadata and image-understanding summaries so later agent turns can reuse image memory without replaying raw base64.

## Best Possible Solution

Add a generic media-memory contract that persists bounded image metadata and summaries in canonical session state, reuses existing MediaUnderstanding output, and lets context engines assemble summary stubs without raw image replay by default.

## Reproduction Assessment

Yes, source-reproducible. Current main stores only basic MediaPath/MediaType fields, keeps image.description output on transient context, and prunes older image replay to generic markers rather than a durable summary stub.

## Solution Assessment

Unclear as a final product/API design. The maintainable direction is a generic core/plugin media-memory seam with canonical storage and context-engine reuse, not a one-off lossless-claw handoff or a hot-path image.describe rerun.
