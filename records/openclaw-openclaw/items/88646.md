## Summary

Keep open: current main supports native image forwarding and transient image-description output, but it does not persist durable attachment metadata or image summaries for context-engine reuse after raw image replay is unavailable.

## What This Changes

Request durable media memory for agent image attachments, including metadata, reusable image-description/OCR summaries, and bounded fallback stubs for compacted or replay-disabled context.

## Best Possible Solution

Add a generic ingest/maintenance media-memory contract that stores metadata plus bounded image summaries in canonical session state, reuses existing MediaUnderstanding output, and lets context engines assemble summary stubs without replaying raw image bytes by default.

## Reproduction Assessment

Yes, source-reproducible. Current main persists only basic media path/type fields, keeps image.description on transient ctx, and later prunes old image data to a generic marker rather than a reusable summary.

## Solution Assessment

Unclear whether the exact requested shape is the best final API. The maintainable direction is a generic core/SDK media-memory seam with canonical storage and context-engine reuse, not a one-off lossless-claw handoff.
