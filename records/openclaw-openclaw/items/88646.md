## Summary

Keep open: current main and the latest release still support active-turn image routing and media-understanding output, but they do not persist durable image metadata plus summaries for later compacted or pruned agent context.

## What This Changes

Request persistent agent attachment media memory: metadata plus reusable image-understanding summaries for follow-up and compacted context without normal raw image replay.

## Best Possible Solution

Define a generic persisted media-memory contract that stores bounded metadata plus summaries in canonical state and exposes summary stubs during context assembly while preserving native image use for fresh turns.

## Reproduction Assessment

Yes, source-reproducible: current main persists only MediaPath(s)/MediaType(s), applies image descriptions only to the current MsgContext, and prunes older image history to generic markers. I did not run a live session because this review was read-only.

## Solution Assessment

Unclear as originally scoped: the direction is valid, but the maintainable solution is a generic session/media-memory and context-engine contract rather than a lossless-claw-only handoff.
