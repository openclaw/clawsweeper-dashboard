## Summary

Keep open: current main still has active-turn media-understanding and media path persistence, but it does not implement the requested durable image-summary/media-memory contract for later compacted or pruned agent context.

## What This Changes

The issue asks OpenClaw to persist image attachment metadata plus reusable image.description/OCR summaries for later agent context without replaying raw base64.

## Best Possible Solution

Define a generic persisted media-memory/session contract that stores bounded metadata plus summaries in canonical state and exposes summary stubs to context assembly while preserving native image use for fresh turns.

## Reproduction Assessment

Yes, source-reproducible: current code writes only MediaPath(s)/MediaType(s), applies image descriptions only into current ctx.Body/ctx.MediaUnderstanding, and prunes older image history to generic markers.

## Solution Assessment

Unclear as originally scoped: the direction is valid, but the maintainable solution is a generic session/media-memory and context-engine contract rather than a lossless-claw-only handoff.
