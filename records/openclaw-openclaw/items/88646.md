## Summary

Keep open: current main has native image turns, image-understanding text injection, and persisted media paths, but it still lacks the requested durable image-summary/media-memory contract for later compacted or pruned context.

## What This Changes

The issue requests persisted image attachment metadata plus reusable image-understanding summaries so later agent turns can use bounded media-memory stubs instead of raw image replay or assistant prose.

## Best Possible Solution

Define a generic persisted media-memory contract that stores bounded image metadata plus summary text in canonical session state and lets context assembly use summary stubs without replaying raw image bytes by default.

## Reproduction Assessment

Yes, source-reproducible: current main persists image paths and can inject image descriptions into the active prompt, but native vision turns still have no durable image.description/media-memory stub and old replay prunes image context to generic markers.

## Solution Assessment

Unclear as proposed: the direction is valid, but the maintainable solution is a generic session/media-memory contract with context-engine reuse rather than a one-off lossless-claw handoff.
