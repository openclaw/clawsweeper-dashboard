## Summary

Keep open: current main still has active-turn image understanding and managed image file metadata, but it does not persist reusable image-understanding summaries or a durable media-memory stub for later compacted/session context.

## What This Changes

The issue requests durable persisted image attachment metadata and image-understanding summaries for agent context reuse.

## Best Possible Solution

Design a generic persisted media-memory contract that stores bounded image metadata plus summary text in canonical session state and lets context assembly use summary stubs without replaying raw image bytes by default.

## Reproduction Assessment

Yes, source-reproducible: current main has active-turn image descriptions and raw/native image replay, but persisted user-turn and managed-image records do not carry reusable image summary text.

## Solution Assessment

Unclear whether the requested token budgets and replay controls are the final best API. The maintainable path is a generic media-memory/session contract with context-engine reuse, not a one-off lossless-claw handoff.
