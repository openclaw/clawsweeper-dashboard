## Summary

Keep open: current main still forwards native image bytes and keeps image-description output in transient context, but it does not persist reusable image summaries or rich attachment metadata for later compacted/session context.

## What This Changes

Request durable media memory for agent image attachments, including metadata, OCR/description summaries, summary reuse, bounded budgets, and metadata-only fallback when image summary generation fails.

## Best Possible Solution

Add a generic media-memory contract that persists bounded image metadata and summaries in canonical session state, reuses existing MediaUnderstanding output, and lets context engines assemble summary stubs without replaying raw image bytes by default.

## Reproduction Assessment

Yes, source-reproducible. Current main persists only basic media path/type fields, keeps image.description on transient ctx, and later prunes old image data to generic markers rather than a reusable summary.

## Solution Assessment

Unclear whether the exact requested budgets and replay controls are the final best API. The maintainable path is a generic core/plugin media-memory seam with canonical storage and context-engine reuse, not a one-off lossless-claw handoff.
