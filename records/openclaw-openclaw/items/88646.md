## Summary

Keep open: current main and v2026.6.9 still have active-turn media understanding plus media path persistence, but not the requested durable per-attachment image-summary/media-memory contract for compacted or pruned context.

## What This Changes

Request durable persisted media-memory summaries and metadata for agent image attachments, reusing or generating image descriptions without replaying raw image data by default.

## Best Possible Solution

Define a generic persisted media-memory contract in canonical session/state storage that exposes bounded metadata plus summaries to context assembly while keeping native image use for fresh turns explicit and owner-controlled.

## Reproduction Assessment

Yes. Source inspection shows current main persists only Media* references, applies image descriptions only to the active inbound context, and later prunes old image history to generic markers rather than reusable summary stubs.

## Solution Assessment

Unclear as an implementation proposal. The desired behavior is valid, but the best fix is a generic OpenClaw media-memory/session contract rather than a lossless-claw-only shortcut.
