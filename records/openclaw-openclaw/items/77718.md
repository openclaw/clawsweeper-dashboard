## Summary

Keep open: current `main` already handles an omitted per-agent `workspace`, but an explicit `null` is still source-reproducible as invalid config and the doctor repair sequence has no targeted repair for it.

## What This Changes

The issue asks OpenClaw to recover from `agents.list[].workspace` being null or missing instead of leaving the gateway in an invalid-config restart loop.

## Best Possible Solution

Doctor should normalize explicit null per-agent workspace values to the same fallback semantics as omission while keeping startup fail-closed for other invalid config.

## Reproduction Assessment

Yes for the null case: `workspace: null` fails `z.string().optional()` and startup rejects invalid snapshots before proceeding. No for the omitted-field example, because current `main` accepts omission and resolves a fallback workspace.

## Solution Assessment

Yes, a targeted doctor repair is the narrowest maintainable fix. Relaxing startup to run with partially invalid agent config would weaken the existing fail-closed config contract.
