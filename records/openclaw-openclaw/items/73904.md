## What This Changes

The PR wires built-in context-engine initialization into subagent spawn preparation before resolving the configured engine and adds a cold-registry ordering regression test.

## Best Possible Solution

Yes. A cold cliBackend-only process can invoke sessions_spawn before any built-in context engine registration, and current main then resolves the default legacy engine against an empty registry; the PR's test models that ordering directly.

## Solution Assessment

Yes, with the changelog addition. Calling the idempotent built-in initializer immediately before spawn-prep resolution is the narrowest maintainable fix for this endpoint and matches adjacent context-engine resolution paths.
