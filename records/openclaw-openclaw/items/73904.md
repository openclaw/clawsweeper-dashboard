## Summary

Keep open. Current main still has the cold-registry failure path, while this PR is a focused and source-backed fix with regression coverage; the remaining blocker I found is the missing user-facing changelog entry before merge.

## What This Changes

The PR adds an injected `ensureContextEnginesInitialized()` dependency and call before subagent spawn context-engine resolution, plus a cold-registry ordering regression test.

## Best Possible Solution

Land this PR, or an equivalent initializer-before-resolve patch with the same regression test, after adding the required Unreleased Fixes changelog entry and letting the normal changed gate validate the final head. Keep #75129 as the separate CLI compaction follow-up unless a maintainer explicitly wants both fixes combined.

## Reproduction Assessment

Yes. A cold process using cliBackend-routed chat can reach native `sessions_spawn` before context engines are initialized, causing default `legacy` resolution against an empty registry and the reported `(none)` error.

## Solution Assessment

Yes. Calling the idempotent built-in initializer at the spawn-side resolution boundary is the narrowest maintainable fix; making registration a top-level side effect would have broader bundling and runtime implications, and the CLI compaction counterpart is already split into #75129.
