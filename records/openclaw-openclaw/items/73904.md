## What This Changes

The PR initializes built-in context engines before native `sessions_spawn` spawn preparation resolves the configured context engine, exposes that dependency through the spawn runtime seam, and adds a cold-registry regression test.

## Best Possible Solution

Land this focused spawn-prep initializer fix with its regression test after adding the missing release-note entry; keep #75129 as the separate CLI compaction follow-up unless a maintainer explicitly chooses to combine them.

## Reproduction Assessment

Yes. A cold cliBackend-only process can invoke native `sessions_spawn` before any built-in context engine registration, and current main then resolves the default `legacy` engine against an empty registry; the PR's new test models that ordering directly.

## Solution Assessment

Yes for the code change. Calling the idempotent built-in initializer immediately before spawn-prep resolution is the narrowest maintainable fix for this endpoint and avoids broader top-level registration side effects; the remaining issue is release-note compliance, not implementation design.
