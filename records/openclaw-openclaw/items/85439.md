## What This Changes

The issue asks for Feishu final interactive cards to optionally retain intermediate tool-call progress text after streaming completes, restoring the older visible progress trail behind a channel-level setting.

## Best Possible Solution

Add a Feishu-owned opt-in final-card progress transcript only if maintainers accept the UX and privacy tradeoff, with focused tests for live status, final card content, and the existing clean-final default.

## Solution Assessment

Is this the best way to solve the issue? Unclear until maintainers decide the product contract; a Feishu-scoped opt-in config is safer than a global final-delivery revert because current tests and docs preserve clean final replies.
