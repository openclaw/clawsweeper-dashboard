## Summary

Keep open. Current main and the latest release still use clean-final Feishu streaming cards: tool progress is visible as transient streaming status, but final card close removes it, and adding a persisted tool trail needs Feishu owner/product judgment rather than cleanup closure.

## What This Changes

The issue requests restoring or adding a Feishu option so final delivered cards can retain intermediate tool-call progress text after streaming completes.

## Best Possible Solution

Define a Feishu-owned opt-in final progress transcript or delivery mode, keep the clean-final default unless maintainers choose otherwise, and cover live tool status plus final card content with focused tests and redacted Feishu proof.

## Reproduction Assessment

No high-confidence live reproduction is available. Source inspection does show current main and v2026.5.20 render tool status as transient streaming-card state, clear it before final close, and lack a final-progress persistence option.

## Solution Assessment

Unclear until maintainers choose the desired Feishu UX. A hard revert risks duplicate/noisy final delivery, while a Feishu-scoped opt-in mode with tests and live proof is the safer maintainable path if the behavior is accepted.
