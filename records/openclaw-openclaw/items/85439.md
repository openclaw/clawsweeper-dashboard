## Summary

Keep open: current main and v2026.5.20 still make Feishu tool-call status transient in the live streaming card and explicitly omit it from the final close, with no Feishu config knob for persisting a final progress trail. That makes this a real remaining Feishu UX/product decision rather than a cleanup close.

## What This Changes

Request configurable Feishu final-card delivery that can include tool-call progress text after a turn completes.

## Best Possible Solution

Make a Feishu-owned product decision: either add an opt-in final-card progress transcript with focused tests and docs, or document the clean-final behavior while routing ordered-progress needs to an accepted segment/status mode.

## Reproduction Assessment

No high-confidence live reproduction was established in this read-only review. Source and shipped-release inspection do show the path: Feishu renders tool status as transient streaming-card state, then clears it before final close.

## Solution Assessment

Unclear until maintainers decide the desired Feishu UX. A hard revert risks duplicate/noisy final delivery, while a Feishu-scoped opt-in config or accepted segment mode is the safer maintainable direction.
