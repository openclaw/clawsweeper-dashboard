## Summary

Keep open. Current main and the latest release still make Feishu tool-start progress transient: the live streaming card can show a status line, but final close clears it, the tests assert that behavior, and there is no Feishu config option for a persistent final-card progress trail. Related open issues track overlapping Feishu text-loss bugs, but they do not replace the product decision requested here.

## What This Changes

Request Feishu/Lark final delivered cards to optionally retain tool-call progress that is currently only shown transiently during streaming.

## Best Possible Solution

Make a Feishu plugin product decision first: either document the current clean-final behavior as intended, or add a Feishu-owned opt-in final progress transcript with focused privacy, config, streaming, and final-card tests.

## Reproduction Assessment

Yes, at source level: current Feishu onToolStart writes a transient statusLine, the final close clears it, and the regression test asserts the final card omits it. I did not run a live credentialed Feishu card reproduction.

## Solution Assessment

Unclear until maintainers choose the UX and privacy policy. A Feishu-scoped opt-in is the safer implementation if this behavior is accepted; a hard revert would risk reintroducing noisy or duplicate final delivery.
