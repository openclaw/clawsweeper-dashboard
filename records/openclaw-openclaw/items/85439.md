## Summary

Keep open. Current main and v2026.6.8 still make Feishu tool-progress status transient: live streaming can show a tool line, but final card close clears it and the current test now asserts the final close contains only the answer. The overlapping Feishu pre-tool text-loss trackers are real, but they do not replace this issue's product/privacy decision about persistent final-card tool progress.

## What This Changes

The issue asks OpenClaw to restore or add a Feishu option for preserving tool-call progress and pre-tool assistant text in the final delivered interactive card.

## Best Possible Solution

Maintainers should choose the Feishu final-card semantics first, then either document clean-final behavior as intentional or add a Feishu-owned opt-in progress transcript with focused config, privacy, streaming, final-card, and docs coverage.

## Reproduction Assessment

Yes, at source level: Feishu onToolStart writes a live statusLine, closeStreaming clears that statusLine before final close, and the current test asserts the final card omits the tool line. I did not run a live credentialed Feishu card reproduction.

## Solution Assessment

Unclear until maintainers choose the UX and privacy policy. If this behavior is accepted, a Feishu-scoped opt-in is safer than an unconditional hard revert because tool progress may include durable diagnostic or command detail.
