## Summary

Keep open. Current main and v2026.5.20 still keep Feishu tool-call progress as transient streaming-card status, then clear it before final card close; adding persisted final-card progress needs a Feishu UX/privacy decision and likely an opt-in config rather than cleanup closure.

## What This Changes

The issue asks OpenClaw to let Feishu final interactive cards retain intermediate tool-call progress text that was visible before the v2026.5.19 final-delivery changes.

## Best Possible Solution

Add a Feishu-owned opt-in final-card progress transcript only if maintainers accept the UX/privacy tradeoff, with focused coverage for live status, final card content, and the clean-final default; otherwise document the current transient progress behavior.

## Reproduction Assessment

No high-confidence live reproduction yet. Source inspection does show the current-main and v2026.5.20 path: Feishu renders tool progress as transient streaming-card status, clears that status on close, and has no config option to persist it in the final card.

## Solution Assessment

Unclear until maintainers decide the desired Feishu UX. A hard revert risks reintroducing duplicate or noisy final delivery, while a Feishu-scoped opt-in config with regression tests is the safer maintainable direction if the behavior is accepted.
