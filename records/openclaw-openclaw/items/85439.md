## Summary

Keep open. Current main and the latest release still make Feishu tool status a transient streaming-card line, clear it before final close, and expose no Feishu config for persisting that trail in the final delivered card; this needs Feishu owner/product judgment rather than cleanup closure.

## What This Changes

Request Feishu to preserve, or optionally include, tool-call progress in final delivered cards instead of showing it only during live streaming.

## Best Possible Solution

Feishu plugin owners should decide the UX/privacy policy, then implement an opt-in Feishu-owned final progress transcript or segment-progress mode with focused dispatcher, config, docs, and live Feishu proof.

## Reproduction Assessment

Yes at source level: the existing Feishu test invokes onToolStart, sees the streaming update include Web Search, and asserts the final close contains only final answer. No live Feishu card repro was run.

## Solution Assessment

Unclear until maintainers decide the desired Feishu UX. A hard revert risks duplicate/noisy final delivery; a Feishu-scoped opt-in config or segment mode with regression tests is the safer direction if the behavior is accepted.
