## Summary

Keep open. Current main still has clean-final Feishu behavior: tool-start progress updates a transient streaming-card status line, final close clears that line, and duplicate final delivery is skipped after a streaming card is closed. The requested 5.12-style persisted progress trail is plausible, but it needs a Feishu/product decision and likely an opt-in config rather than cleanup closure.

## What This Changes

The issue asks OpenClaw to preserve or optionally include Feishu streamed tool-call progress in the final interactive card instead of delivering only the final assistant reply.

## Best Possible Solution

Add a Feishu-owned opt-in final-card progress transcript only if maintainers accept the UX/privacy tradeoff, with focused tests for live tool status, final card content, and the clean-final default; otherwise document the current clean-final behavior clearly.

## Reproduction Assessment

No high-confidence live reproduction yet. Source inspection does show the current-main path: Feishu renders live tool status as transient streaming-card state, clears it before final close, and lacks a config option to persist that progress in the final card.

## Solution Assessment

Unclear until maintainers decide the desired UX. A hard revert would risk reintroducing duplicate or noisy final delivery, while a Feishu-scoped opt-in config with regression tests is the safer maintainable direction if the behavior is accepted.
