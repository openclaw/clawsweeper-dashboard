## Summary

Keep open: current main and v2026.6.8 still make Feishu tool-progress/status content transient in final cards, while the remaining request needs a Feishu UX/privacy decision rather than duplicate closure into the narrower text-loss trackers.

## What This Changes

The issue asks OpenClaw to restore or add a Feishu-scoped option for final interactive cards to retain tool-call progress and pre-tool visible text instead of closing with only the final answer.

## Best Possible Solution

Make a Feishu plugin product decision first, then either document clean-final cards as intentional or implement a Feishu-owned opt-in/fix that preserves final progress while coordinating with the open streaming-card bug trackers.

## Reproduction Assessment

Yes, at source level: current Feishu streaming cards can show statusLine during live updates, then clear it before final close, and the final payload path can overwrite earlier streamed text. I did not run a live credentialed Feishu reproduction.

## Solution Assessment

Unclear until maintainers choose the final-card progress policy. A Feishu-scoped opt-in or documented intentional clean-final behavior is safer than a hard revert that could reintroduce noisy or duplicate final delivery.
