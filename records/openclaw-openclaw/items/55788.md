## Summary

Keep open. Current main still lacks Codex audio, but this PR remains incorrect because it bypasses configured audio endpoints and misses the manifest/provider metadata surfaces now used for audio defaults and auto selection.

## What This Changes

The PR adds audio support to the OpenAI Codex media-understanding provider by wiring a transcription wrapper, adding a default audio model entry, and extending the OpenAI plugin test.

## Best Possible Solution

Refresh the branch against current main, add Codex audio consistently to runtime provider and manifest metadata/default/auto-priority paths, preserve explicit OpenAI-compatible audio endpoint overrides while stripping only known ChatGPT backend URLs, and add focused tests plus a changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure `openai-codex` for audio or rely on auto audio selection, and current main cannot select or execute it because Codex is image-only in metadata and has no `transcribeAudio` handler.

## Solution Assessment

No. The PR fixes part of the runtime gap, but it is not the best current solution because it drops endpoint policy and edits an obsolete defaults table instead of the current manifest/provider metadata path.
