## Summary

Keep open. Current main still lacks `openai-codex` audio transcription support, but this PR is not merge-ready because it discards explicit audio endpoint overrides and updates stale default-model plumbing instead of the current manifest-backed metadata path.

## What This Changes

The PR adds audio capability and a transcription wrapper for the `openai-codex` media-understanding provider, adds an `openai-codex` default audio model entry, and extends the OpenAI plugin test assertion.

## Best Possible Solution

Update the current OpenAI plugin runtime and manifest metadata together: add Codex audio capability, default model, auto priority, and a transcription handler that strips only known ChatGPT backend URLs while preserving explicit OpenAI-compatible audio endpoints, with focused tests and changelog coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure or select `openai-codex` for audio media understanding, and current main cannot choose or execute it because the provider is image-only and lacks `transcribeAudio`.

## Solution Assessment

No. The PR moves in the right direction, but the current patch is incomplete because it uses obsolete defaults plumbing and handles the Codex chat-backend routing problem by discarding all configured audio endpoints.
