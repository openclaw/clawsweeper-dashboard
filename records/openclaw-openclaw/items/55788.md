## Summary

Keep open. Current main still lacks OpenAI Codex audio transcription support, but this PR is not merge-ready because it updates an obsolete defaults path and the proposed wrapper drops configured audio endpoint overrides.

## What This Changes

The PR adds audio capability and a transcription wrapper for the `openai-codex` media-understanding provider, adds an `openai-codex` default audio model entry, and extends the OpenAI plugin test assertion.

## Best Possible Solution

Refresh the PR against current main, add Codex audio consistently to the runtime provider and plugin manifest metadata/default model/auto-priority paths, preserve explicit OpenAI-compatible audio endpoint overrides while stripping only known ChatGPT backend URLs, and update focused tests plus docs if the support matrix changes.

## Reproduction Assessment

Yes. Current-main source inspection gives a high-confidence path: configure `openai-codex` for media audio or use it as the active model, and the audio runner cannot select or execute it because current provider metadata is image-only and the runtime provider has no `transcribeAudio` handler.

## Solution Assessment

No. The PR addresses part of the runtime gap, but the current approach is not the best fix because it drops endpoint policy and updates an obsolete defaults table instead of the current manifest/provider metadata path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve explicit Codex audio endpoints:** `extensions/openai/media-understanding-provider.ts:29`
  - body: `transcribeOpenAiCodexAudio` overwrites every caller-supplied `baseUrl` with `undefined`. The runner already passes configured provider/media endpoints into `transcribeAudio`, and the shared OpenAI-compatible helper honors them, so this would force user audio to the default OpenAI host even when an operator configured a relay or proxy. Strip only known ChatGPT backend URLs and keep explicit audio overrides.
  - confidence: 0.9
- **[P2] Wire Codex audio through current metadata:** `extensions/openai/media-understanding-provider.ts:36`
  - body: Adding `"audio"` to the runtime provider is not enough on current main. Default model and auto-provider selection now come from provider `defaultModels`/`autoPriority` and manifest metadata, while this branch only edits the removed defaults table, so Codex-authenticated auto audio paths still will not be selected correctly.
  - confidence: 0.86
