## Summary

Keep this PR open: the underlying `openai-codex` audio gap is still present on current `main`, but the proposed patch is stale and incomplete for the current media-understanding metadata path and introduces an endpoint-routing regression.

## What This Changes

The PR adds audio capability, a transcription handler, a default audio model entry, and a handler assertion for the `openai-codex` media-understanding provider.

## Best Possible Solution

Update the OpenAI plugin runtime and manifest metadata together so `openai-codex` has audio capability, default model, auto priority, and a transcription wrapper that strips only known ChatGPT backend URLs while preserving explicit audio endpoints, with focused regression tests and changelog coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows `openai-codex` is image-only and lacks `transcribeAudio`, while the runner skips audio providers without that handler.

## Solution Assessment

No. The PR moves in the right direction, but it is not the best current fix because it discards configured audio base URLs and wires defaults through a removed/stale path instead of provider and manifest metadata.
