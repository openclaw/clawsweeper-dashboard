## Summary

Keep open. The underlying Codex audio gap is still real on current main, but this PR is not merge-ready because it drops explicit Codex audio endpoint overrides and updates an obsolete defaults path instead of the current runtime and manifest metadata surfaces.

## What This Changes

The PR adds audio capability and a `transcribeAudio` wrapper for the `openai-codex` media-understanding provider, adds an `openai-codex` default audio model entry, and extends the OpenAI plugin test assertion.

## Best Possible Solution

Add Codex audio consistently to the runtime provider, manifest metadata, default model, auto-priority, tests, and changelog while preserving explicit OpenAI-compatible audio endpoints and stripping only known ChatGPT backend URLs.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure or select `openai-codex` for audio media understanding, and current main cannot choose or execute it because the provider is image-only and lacks `transcribeAudio`.

## Solution Assessment

No. The PR fixes part of the runtime gap, but it is not the best solution because it drops configured endpoint routing and misses the current metadata surfaces that drive default model and auto-provider selection.

## Review Findings
