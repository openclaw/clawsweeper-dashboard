## Summary

Keep this PR open: current main still lacks `openai-codex` audio support, but the submitted patch is stale against the current manifest-backed media metadata and drops explicit audio endpoint overrides.

## What This Changes

The PR adds audio capability and a transcription handler to the `openai-codex` media-understanding provider, adds a default audio-model entry, and updates an OpenAI plugin test assertion.

## Best Possible Solution

Update the OpenAI plugin runtime and manifest metadata together so `openai-codex` has audio capability, default model, auto priority, and a transcription wrapper that strips only known ChatGPT backend URLs while preserving explicit audio endpoint overrides.

## Reproduction Assessment

Yes. Source inspection on current main shows `openai-codex` is image-only and lacks `transcribeAudio`, while the audio runner skips providers without that handler.

## Solution Assessment

No. The PR direction is useful, but this patch is not the best current fix because it updates a stale defaults path and unconditionally discards configured audio base URLs.
