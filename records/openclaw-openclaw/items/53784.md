## Summary

Keep open. Current main still preserves the documented Feishu behavior where ordinary MP3-style audio attachments are sent as files unless explicit voice delivery is requested, while this PR changes that default and still has merge blockers around docs, lockfile alignment, contributor proof, and Feishu audio policy.

## What This Changes

The PR changes Feishu outbound media routing so common audio formats and audio MIME types upload as Feishu audio messages with optional `music-metadata` duration parsing.

## Best Possible Solution

Maintainers should decide the Feishu common-audio default explicitly; if desired, land a rebased change that updates routing, duration handling, docs, package metadata, lockfile, and proof together.

Yes, source inspection gives a high-confidence current-main path: `song.mp3` with `audio/mpeg` routes to `stream`/`file`, and the colocated test asserts that behavior. I did not run a live Feishu send in this read-only review.

## Solution Assessment
