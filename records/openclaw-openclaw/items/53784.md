## Summary

Keep open. Current main still implements the documented explicit-voice policy, while this PR proposes a useful but user-facing Feishu default change that needs maintainer approval plus rebase, docs, changelog, and lockfile alignment before merge.

## What This Changes

The PR broadens Feishu outbound audio routing so common audio extensions and audio MIME types send as native audio messages, adds duration parsing with `music-metadata`, and updates Feishu media tests/package metadata.

## Best Possible Solution

Decide the Feishu audio contract explicitly; if direct playable bubbles for common formats are desired, land a rebased narrow change that updates routing, duration handling, tests, docs, changelog, package metadata, and lockfile together.

## Reproduction Assessment

Yes. Source and test evidence on current main show `song.mp3` with `audio/mpeg` uploads as `stream` and sends as `msg_type: "file"` unless `audioAsVoice` requests transcoding.

## Solution Assessment

Unclear as-is. The approach is plausible and backed by useful Feishu/community-plugin context, but changing ordinary audio attachments into native audio bubbles is a product contract decision and the branch is missing required merge-alignment work.
