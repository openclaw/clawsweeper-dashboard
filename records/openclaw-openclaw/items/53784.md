## Summary

Close as superseded: current main settled Feishu common-audio voice delivery through explicit voice-intent transcoding, while docs and tests intentionally keep ordinary MP3-style attachments as files. The PR routes common Feishu audio extensions and audio MIME types directly as `msg_type: "audio"`, maps them to `file_type: "opus"`, parses duration with `music-metadata`, and updates Feishu media tests/package metadata.
