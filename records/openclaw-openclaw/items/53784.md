## Summary

Keep open for maintainer review: current main deliberately documents and tests ordinary MP3-style Feishu audio as file delivery unless `asVoice`/`audioAsVoice` is requested, while this PR changes that default and still needs docs, changelog, package, and lockfile alignment.

## What This Changes

The PR broadens Feishu outbound audio routing to send common audio extensions and audio MIME types as native audio messages, adds `music-metadata` duration parsing, and updates Feishu media tests/package metadata.

## Best Possible Solution

Decide the Feishu audio contract explicitly; if direct playable bubbles for common formats are desired, land a rebased narrow change that updates routing, duration handling, tests, docs, package metadata, lockfile, and changelog together.

## Reproduction Assessment

Yes. Source and existing test coverage on current main show `song.mp3` with `audio/mpeg` routes as `stream`/`file` unless explicit voice delivery is requested.

## Solution Assessment

No as-is. The PR direction is plausible, but it is not the best merge shape until the default-audio policy is chosen and the docs, changelog, package metadata, and lockfile are aligned.
