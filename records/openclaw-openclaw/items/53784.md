## Summary

Keep open. Current main and the latest shipped release still intentionally route common Feishu audio formats such as MP3 as files unless explicit voice delivery is requested, while this PR changes that default and still needs maintainer policy approval, lockfile/docs alignment, and inspectable real behavior proof.

## What This Changes

The branch broadens Feishu outbound routing so common audio extensions and audio MIME types upload as Feishu audio messages, adds `music-metadata` duration parsing, and updates Feishu media tests/package metadata.

## Best Possible Solution

Decide the Feishu audio contract explicitly; if direct playable bubbles for common formats are desired, land a rebased narrow change that updates routing, duration handling, tests, docs, package metadata, lockfile, and real proof together.

Yes. Source inspection gives a high-confidence current-main path: `song.mp3` with `audio/mpeg` routes to `stream`/`file`, and the colocated test asserts that behavior; I did not perform a live Feishu send.
