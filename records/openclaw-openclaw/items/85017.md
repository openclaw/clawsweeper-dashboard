## Summary

Keep open: current main still has source-visible Matrix DM classification paths that can route a configured two-person room as a DM before room `requireMention` policy is applied, and the linked fix PR is still open rather than merged.

## What This Changes

The issue asks OpenClaw to treat explicit Matrix non-DM signals and configured Matrix groups as room traffic so `requireMention` is honored in two-person rooms.

## Best Possible Solution

Review and land a fix that makes explicit local non-DM and configured-room signals veto strict-DM routing before room policy is skipped, while preserving intentional fresh unnamed DM recovery.

## Reproduction Assessment

Yes, at source level. The current handler computes DM routing before room config, and a strict two-member `m.direct` path can return direct-message handling before `requireMention` can apply.

## Solution Assessment

Yes in direction, but not complete while the linked PR is still open. The maintainable fix is to apply explicit non-DM and configured-room vetoes before every inbound strict-DM branch, with regression coverage for the fresh-DM fallback.
