## Summary

This Matrix routing report should stay open because current main still has the strict two-member direct-room path and an open linked fix PR explicitly closes this issue, so the issue should track that implementation until it merges or is rejected.

## What This Changes

The issue asks Matrix routing to stop treating explicitly non-direct or group-configured two-person rooms as DMs so `requireMention` remains enforced.

## Best Possible Solution

Review and land or revise the linked Matrix PR so explicit non-DM signals are honored before DM routing, then decide whether configured `groups[roomId]` should also be a first-class veto with regression coverage.

## Reproduction Assessment

Yes from source inspection, though I did not run a live Matrix homeserver in this read-only pass. Current main computes direct-message status before room config and still returns strict two-member evidence without an explicit-false veto in the shared helper.

## Solution Assessment

Yes in direction: fix the routing boundary rather than weakening `requireMention`. The safest implementation is to make explicit non-DM and configured-room signals veto strict-DM routing while preserving the recent-invite path for true DMs.
