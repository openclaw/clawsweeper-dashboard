## What This Changes

The PR changes agent-core compaction token estimation so image content blocks in user messages are charged like image blocks in tool/custom messages, with regression coverage for image-heavy cut-point selection.

## Best Possible Solution

Land the focused estimator and regression-test change after exact-head checks pass; keep historical image pruning as the separate already-shipped behavior.

## Reproduction Assessment

Yes, by source inspection: current main accepts user content arrays with image blocks, but estimateTokens only counts text in the user branch, and findCutPoint directly consumes that estimate. I did not run tests because this review was required to keep the checkout read-only.

## Solution Assessment

Yes. The cut-point bug is owned by estimateTokens, and sharing the existing image heuristic across user and tool/custom content arrays is the narrowest maintainable fix; changing history pruning would not cover recent retained image turns.
