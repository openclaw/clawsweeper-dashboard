## What This Changes

The PR makes agent-core compaction token estimation count image content blocks in user messages with the same helper used for tool/custom array content and adds Vitest coverage for image-heavy cut-point selection.

## Best Possible Solution

Land the focused estimator and regression-test change after normal maintainer review and repository gates; keep historical image pruning as the separate already-shipped behavior.

## Reproduction Assessment

Yes, by source inspection: current main allows user messages to carry ImageContent arrays, but estimateTokens only sums text in the user branch and findCutPoint directly consumes that estimate. I did not run tests because this review was required to keep the checkout read-only.

## Solution Assessment

Yes. The cut-point estimator owns this heuristic, and sharing one image-aware content-block counter across user and tool/custom array content is the narrowest maintainable fix; old-history pruning solves a different older-image bloat case.
