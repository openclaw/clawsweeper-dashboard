## Summary

Keep open: current main still undercounts user-message image blocks in the compaction cut-point estimator, and the PR’s shared helper plus focused tests address that path cleanly. I found no blocking code findings, but the external-PR real-behavior proof gate is still unit-test-only.

## What This Changes

The PR makes `estimateTokens` count image blocks in user-message array content using the same image heuristic already used for tool/custom content, and adds regression tests for image-heavy cut-point selection.

## Best Possible Solution

Land this focused estimator fix once the contributor adds acceptable real behavior proof and required checks pass, keeping the broader history-pruning behavior separate.

## Reproduction Assessment

Yes. Source inspection shows current main counts only text in user-message array content while `ImageContent` is a valid user block, so a user-image-only message estimates to zero tokens; I did not run tests because this review was read-only.

## Solution Assessment

Yes. `estimateTokens` is the local owner used by `findCutPoint`, and sharing the content-block helper with the existing tool/custom image path is the narrowest maintainable fix I found.
