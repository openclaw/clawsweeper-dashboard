Keep open: current main still undercounts image blocks in user-message arrays during compaction cut-point estimation, and this PR is a focused bug fix with sufficient before/after planner proof and no blocking review findings.

## What This Changes

The PR adds shared image-block accounting to agent-core compaction token estimation for user/tool/custom content arrays and adds regression tests for image-heavy cut-point selection.

## Best Possible Solution

Land the focused estimator and regression-test fix after normal maintainer review; keep historical image pruning as the separate already-implemented behavior.

## Reproduction Assessment

Yes. Current main allows user content arrays with image blocks, but estimateTokens only counts text in the user branch and findCutPoint consumes that estimate; I did not run tests because this review had to keep the checkout read-only.

## Solution Assessment

Yes. The cut-point estimator owns this heuristic, and sharing the existing image charge between user and tool/custom content arrays is narrower than changing history pruning, which only handles older processed images.
