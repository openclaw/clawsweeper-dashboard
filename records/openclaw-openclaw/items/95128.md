Keep this PR open for normal review and CI gating. Current main and the latest release still under-count user-message image blocks, while the proposed helper centralizes the existing image cost used by tool/custom content and the added tests cover the cut-point behavior.

## What This Changes

The PR updates the compaction token estimator so image blocks in user-message content count like image blocks in tool/custom content, and adds regression tests for image-heavy cut-point behavior.

## Best Possible Solution

Merge the focused estimator and regression-test fix after normal CI, keeping image accounting centralized in the shared compaction planner.

## Solution Assessment

Yes. The estimator is the shared owner used by `estimateContextTokens`, `findCutPoint`, branch summarization, and session compaction planning, so centralizing user/tool image block accounting there is the narrow maintainable fix.
