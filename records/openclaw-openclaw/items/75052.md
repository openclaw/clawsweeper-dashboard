## Summary

Keep open: current main still has the data-loss path, and an open closing PR is already the canonical fix candidate. The source path matches the report: state cleanup recursively removes the state directory before workspace-scope deletion is considered, while the default workspace lives under that same directory.

## What This Changes

Preserve workspaces stored under `.openclaw` when uninstall removes config/state but workspace deletion is not selected.

## Best Possible Solution

Land #75061 or an equivalent narrow patch that keeps state/config cleanup independent from workspace deletion by preserving or warning about configured nested workspaces, with focused regression coverage, docs updates, and a user-facing changelog entry when merged.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: create the default workspace under `.openclaw`, select state/config removal without workspace removal, and the current state cleanup recursively removes the parent before workspace preservation can occur.

## Solution Assessment

Yes. The narrow maintainable solution is to guard the uninstall cleanup path against deleting a parent state directory when configured workspaces must be preserved; #75061 is already the active implementation candidate.
