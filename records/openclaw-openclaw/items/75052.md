## What This Changes

The issue reports that state/config-only uninstall removes workspaces nested under the `.openclaw` state directory even when workspace deletion is not selected.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/75061 or an equivalent narrow fix that preserves configured nested workspaces during state-only cleanup while keeping explicit `--workspace` and `--all` deletion behavior destructive.

## Reproduction Assessment

Yes. Source inspection on current main shows the default workspace under `$HOME/.openclaw/workspace`, state-only uninstall recursively removes the state directory, and workspace cleanup only runs when the workspace scope is selected.

## Solution Assessment

Yes. A preservation/pruning guard in state cleanup is the narrow maintainable fix; no new option or broader uninstall policy change is needed.
