## What This Changes

The issue asks uninstall to preserve workspace files when state/config deletion is selected but workspace deletion is not selected.

## Best Possible Solution

Land PR #75061 or an equivalent canonical fix that preserves configured nested workspaces during state-only uninstall, with regression coverage, docs, and a changelog entry.

Yes. A high-confidence source-level reproduction exists: the default workspace is under `~/.openclaw`, and state-only uninstall recursively removes that parent before any workspace-specific deletion decision can preserve it.
