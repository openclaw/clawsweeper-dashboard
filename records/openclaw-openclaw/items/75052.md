## Summary

Keep open: current main still has the reported state-only uninstall data-loss path, and the open closing-syntax repair PR #75061 remains the right implementation candidate.

## What This Changes

State-only uninstall should preserve workspace directories when workspace deletion is not selected, including workspaces nested under the default state directory.

## Best Possible Solution

Land PR #75061 or an equivalent narrow fix that skips or preserves configured nested workspaces during state cleanup unless the workspace scope is explicitly selected.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence current-main path: the default workspace is inside ~/.openclaw, state cleanup recursively removes ~/.openclaw, and workspace cleanup is skipped when the workspace scope is unchecked.

## Solution Assessment

Yes. The narrow maintainable fix is a preservation guard for configured nested workspaces during state-only uninstall while keeping explicit --workspace and --all deletion behavior destructive.
