## What This Changes

The issue asks OpenClaw uninstall to preserve workspace directories when state/config deletion is selected but workspace deletion is not, including workspaces nested under `.openclaw`.

## Best Possible Solution

Land one canonical uninstall fix, preferably by reviewing and landing #75061 or an equivalent patch, that keeps state/config deletion independent from workspace deletion and includes focused regression coverage, updated uninstall docs, and a user-facing changelog entry.

## Reproduction Assessment

Yes. A destructive live uninstall was not run during this read-only review, but the source path is high-confidence: default workspaces are inside `.openclaw`, and state-only uninstall recursively removes that parent directory before workspace scope is considered.
