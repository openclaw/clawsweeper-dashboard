## What This Changes

Preserve workspace directories nested under `.openclaw` when uninstall removes state/config but workspace deletion is not selected.

## Best Possible Solution

Land PR #75061 or an equivalent canonical fix that preserves configured workspaces during state-only uninstall, with regression tests, docs, and a changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: the default workspace is `~/.openclaw/workspace`, and state-only uninstall recursively removes `~/.openclaw` before any workspace-specific deletion decision can preserve that nested path.

## Solution Assessment

Yes. A narrow preservation guard in the uninstall cleanup path is the maintainable fix, and the linked PR is the right implementation path to review rather than starting another duplicate repair.
