## Summary

Keep open: current main still has the state-only uninstall data-loss path, and #75061 is open but unmerged as the closing-syntax implementation candidate.

## What This Changes

State/config uninstall should preserve workspace directories when the workspace deletion scope is not selected, including workspaces nested under the OpenClaw state directory.

## Best Possible Solution

Preserve configured workspace directories nested under the state directory during state-only cleanup, add regression coverage, and keep explicit `--workspace` and `--all` deletion destructive.

## Reproduction Assessment

Yes. Source inspection on current main shows the default workspace is inside `~/.openclaw`, state cleanup recursively removes `~/.openclaw`, and workspace cleanup only runs when the workspace scope is selected.

## Solution Assessment

Yes. The narrow maintainable fix is a preservation guard for configured nested workspaces during state-only cleanup; #75061 appears to be the active implementation path for that approach.
