## Summary

Keep open: current main still has the state-only uninstall data-loss path, and the open closing-syntax repair PR #75061 has not merged.

## What This Changes

The issue requests preserving workspaces located under `.openclaw` when uninstall removes state/config but the workspace scope is not selected.

## Best Possible Solution

Land PR #75061 or an equivalent narrow guard that preserves configured nested workspace directories during state cleanup unless `--workspace` or `--all` is selected, with regression coverage and docs.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: the default workspace is under `~/.openclaw`, state-only uninstall recursively removes that parent directory, and workspace cleanup is skipped when the workspace scope is unchecked.

## Solution Assessment

Yes. The narrow maintainable fix is to preserve configured workspace dirs during state cleanup when they are nested under the state dir; the open linked PR is the right implementation path to review.
