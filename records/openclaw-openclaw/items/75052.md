## Summary

Keep open. Current main still removes the whole state directory recursively when the state/config scope is selected, while the default workspace is nested under that same state directory; open PR #75061 is the active closing fix candidate and should be reviewed or landed before this issue closes.

## What This Changes

Preserve workspace directories nested under `.openclaw` when uninstall removes state/config but workspace deletion is not selected.

## Best Possible Solution

Land one canonical uninstall fix that keeps state/config deletion and workspace deletion scopes independent: state-only uninstall must not delete configured or default workspaces nested under the state directory, and the behavior should be covered by focused regression tests plus uninstall docs.

## Reproduction Assessment

Yes. The source-level path is high confidence: default workspaces resolve under `.openclaw`, and state-only uninstall recursively removes that parent before the workspace scope is checked.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the uninstall cleanup path, with tests and docs; because PR #75061 already targets the issue, the next step is to review or adjust that PR rather than open another repair branch.
