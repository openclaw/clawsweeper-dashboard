## Summary

Keep open: current main still has the reported state-only uninstall data-loss path, and PR #75061 is an open unmerged closing-syntax fix.

## What This Changes

Preserve workspaces stored under `.openclaw` when uninstall removes state/config but workspace deletion is not selected.

## Best Possible Solution

Review and land PR #75061 or an equivalent narrow fix that preserves configured nested workspaces unless `--workspace` or `--all` is selected.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: default workspace is under `~/.openclaw`, state cleanup recursively removes `~/.openclaw`, and workspace cleanup is skipped when the workspace scope is unchecked.

## Solution Assessment

Yes. The narrow maintainable solution is to preserve configured nested workspaces during state cleanup while keeping explicit workspace or all-scope deletion destructive.
