## Summary

Keep open: current main still has the source-reproducible state-only uninstall data-loss path, and the open closing-syntax repair PR #75061 has not merged.

## What This Changes

The issue asks `openclaw uninstall` to preserve workspace directories nested under `.openclaw` when state/config cleanup is selected but workspace deletion is not.

## Best Possible Solution

Land PR #75061 or an equivalent narrow fix that skips or preserves configured nested workspaces during state cleanup while keeping explicit `--workspace` and `--all` deletion behavior destructive.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: default state is `~/.openclaw`, default workspace is `~/.openclaw/workspace`, and state cleanup recursively removes the parent before workspace deletion is considered.

## Solution Assessment

Yes. The linked repair direction is the narrow maintainable fix: preserve configured nested workspaces only when the workspace scope is not selected, without changing explicit destructive scopes.
