## What This Changes

The issue reports that state/config uninstall deletes workspaces nested under the OpenClaw state directory even when workspace deletion is not selected.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/75061 or an equivalent narrow fix, then close this issue after main contains the preservation behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows the default workspace under the state directory, state cleanup recursively removes the state directory, and workspace cleanup only runs when the workspace scope is selected; I did not run a destructive filesystem repro in this read-only review.

## Solution Assessment

Yes. Preserving configured workspace directories while pruning state cleanup around them is the narrow maintainable fix, and explicit workspace or all-scope deletion should remain destructive.
