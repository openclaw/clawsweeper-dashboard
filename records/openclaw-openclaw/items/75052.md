## Summary

Keep open: current main still has the reported state-only uninstall data-loss path, and #75061 is an open but unmerged closing-syntax fix.

## What This Changes

State-only uninstall should preserve workspaces nested under `~/.openclaw` when workspace deletion is not selected.

## Best Possible Solution

Land #75061 or an equivalent narrow fix that preserves configured nested workspaces during state cleanup unless workspace deletion is explicitly selected, with regression coverage and uninstall docs aligned.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: default state is `~/.openclaw`, default workspace is `~/.openclaw/workspace`, and state-only uninstall recursively removes the state directory while skipping workspace cleanup.

## Solution Assessment

Yes. The narrow maintainable solution is a preservation guard for configured nested workspaces during state cleanup while keeping explicit `--workspace` and `--all` deletion destructive.
