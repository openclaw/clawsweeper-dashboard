## Summary

Keep open: current main still has the WhatsApp source path that can transcribe a voice note but cannot send the configured transcript echo, and an open linked PR is already the implementation path that should be reviewed or landed before this issue closes.

## What This Changes

The issue reports that WhatsApp voice-note transcription reaches the agent, but `tools.media.audio.echoTranscript` no longer sends the transcript echo after the WhatsApp plugin was externalized.

## Best Possible Solution

Land a focused WhatsApp plugin fix that passes the existing route/account/target metadata into `transcribeFirstAudio` and adds regression coverage for direct and group/broadcast echo routing.

## Reproduction Assessment

Yes from source inspection, though not from a live WhatsApp run. Current main routes WhatsApp voice-note preflight through `transcribeFirstAudio` without the channel and target fields that `sendTranscriptEcho` requires.

## Solution Assessment

Yes. Passing the existing WhatsApp routing context into the shared preflight/echo path is the narrow maintainable fix; the issue should remain open until the linked PR or an equivalent fix merges.
