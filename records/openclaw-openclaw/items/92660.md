## Summary

Keep open: the reported dry-run false negative is source-reproducible, but this branch is not merge-ready because it silently changes the documented exact-version pin contract and broadens pinned npm plugin updates to the registry default without real CLI proof.

## What This Changes

The PR changes `src/plugins/update.ts` so `resolveNpmUpdateSpecs` strips exact-version selectors from npm install specs before update probes.

## Best Possible Solution

Preserve exact-version pin semantics by default, and solve the false negative through the existing explicit unpinned npm-spec path or a maintainer-approved latest-advisory mode with aligned docs, tests, and real CLI proof.

## Reproduction Assessment

Yes, at source level: current main passes the recorded exact npm spec into the update probe, and live `npm view` shows exact specs resolve to the pinned version while bare package names resolve to the current default. I did not run a full CLI reproduction because this review is read-only.

## Solution Assessment

No. The patch is a plausible root-cause fix, but it is not the best solution because the helper feeds dry-run and live updates and the change silently unpins stored exact versions despite current docs and tests saying those pins remain sticky.
