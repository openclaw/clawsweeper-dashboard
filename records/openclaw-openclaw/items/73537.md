## Summary

Keep open. Current main has stable/beta/dev release lanes, release validation profiles, staged update hardening, and rollback/update documentation, but it still does not expose an official per-release Production-ready, Early adopter, Known issues, or equivalent readiness marker; related release-stability threads explicitly route the broader policy question here.

## What This Changes

The issue requests an official per-release production-readiness or stability marker for OpenClaw releases.

## Best Possible Solution

Define a maintainer-owned readiness taxonomy and source-of-truth release metadata, require release notes to display it, and optionally expose the same value to update CLI/UI consumers.

## Reproduction Assessment

Not applicable: this is a release-policy feature request, not a broken runtime behavior. The current-main check is source, docs, workflow, release, and related-thread inspection, which shows no official per-release readiness marker.

## Solution Assessment

Unclear until release owners define the public criteria. The requested direction is plausible, but the maintainable implementation should be an owner-set release metadata field rather than heuristic risk inference alone.
