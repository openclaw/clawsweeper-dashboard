## Summary

Keep this open. Current main has stable/beta/dev lanes, beta-first validation, release checks, package-update safeguards, and rollback docs, but it still does not expose an owner-set per-release Production-ready, Early adopter, Known issues, or equivalent readiness marker.

## What This Changes

Request a release-visible stability/readiness indicator, such as Production-ready, Early adopter, or Known issues, for each OpenClaw release.

## Best Possible Solution

Define a release-owned readiness taxonomy with a required metadata value in release notes, then optionally expose the same value to update CLI/UI consumers.

## Reproduction Assessment

Not applicable as a feature request. The current-main verification path is high confidence: targeted searches plus release docs, release workflows, update docs, and update CLI registration show no visible per-release readiness label.

## Solution Assessment

Unclear until release owners choose the public taxonomy and criteria. The requested direction is plausible, but the maintainable solution should start with owner-set release metadata rather than inferred update-review heuristics alone.
