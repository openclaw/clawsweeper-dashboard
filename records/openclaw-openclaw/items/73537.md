## Summary

Keep open. Current main has stable/beta/dev lanes, beta-first validation, release checks, update safeguards, and rollback docs, but it still does not expose an official per-release Production-ready, Early adopter, Known issues, or equivalent readiness marker. The Agent Watch dashboard linked in the discussion is useful community visibility, but it does not replace the requested OpenClaw-owned release signal.

## What This Changes

The issue requests a visible production-readiness or stability indicator on OpenClaw releases.

## Best Possible Solution

Define an owner-set release readiness taxonomy and required release-note or release-metadata value, then optionally expose that same source through update CLI and Control UI surfaces.

## Reproduction Assessment

Not applicable: this is a release-policy/product feature request, not a runtime bug. The gap is high-confidence source-reproducible by inspecting release docs, release workflows, update docs, and update CLI registration on current main.

## Solution Assessment

Unclear until release owners choose the public promise and criteria. The requested direction is plausible, but the maintainable implementation should start with authoritative release metadata rather than inferred update-risk heuristics alone.
