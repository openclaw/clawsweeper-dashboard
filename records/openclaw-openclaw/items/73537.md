## Summary

Keep this open. Current main has stable/beta/dev lanes, release validation, update hardening, and rollback guidance, but it still does not expose an owner-set per-release readiness marker; related maintainer comments also point the broader release-readiness discussion back to this issue. The Agent Watch dashboard linked in the thread is useful community visibility, but it does not replace an official OpenClaw release label.

## What This Changes

Request a visible production-readiness or stability marker on OpenClaw releases.

## Best Possible Solution

Release owners should define a required readiness taxonomy and criteria in the release process, publish it in release notes, and optionally expose the same metadata through update CLI/UI surfaces.

## Reproduction Assessment

Not applicable as a feature request. The verification path is source-backed: current release docs, workflows, and update CLI surfaces do not expose a per-release readiness marker.

## Solution Assessment

Unclear until release owners choose the public taxonomy and criteria. The maintainable direction is an owner-set release metadata field or release-note requirement, not inferred heuristics or a community dashboard alone.
