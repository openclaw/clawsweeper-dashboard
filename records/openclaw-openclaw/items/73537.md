## Summary

Keep open. Current main has release lanes, validation profiles, staged update hardening, and rollback/update docs, and Agent Watch is a useful community workaround, but OpenClaw still does not expose an official per-release Production-ready, Early adopter, Known issues, or equivalent readiness marker; related release-stability threads explicitly route the broader policy discussion here.

## What This Changes

The issue requests an official production-readiness/stability marker in OpenClaw release notes or release metadata.

## Best Possible Solution

Define a maintainer-owned readiness taxonomy and source-of-truth release metadata, require release notes to carry it, and expose the same value to update CLI/UI consumers where useful.

## Reproduction Assessment

Not applicable as a feature request. Source, docs, workflow, latest-release, and related-thread inspection gives high-confidence verification that current main lacks the requested official per-release readiness label.

## Solution Assessment

Unclear until release owners define the public criteria. The requested direction is plausible, but the maintainable implementation should be an owner-set release metadata field rather than heuristic risk inference alone.
