## Summary

Keep open. Current main still lacks an official per-release readiness marker; Agent Watch is useful community visibility, but maintainers have explicitly routed adjacent release-policy discussions here and the official taxonomy remains undecided.

## What This Changes

Request to add an official per-release production-readiness or stability marker so operators know whether a release is safe to upgrade to.

## Best Possible Solution

Define a release-owned readiness taxonomy and canonical metadata or release-note field, then optionally expose the same source through update CLI and Control UI surfaces.

## Reproduction Assessment

Not applicable as a feature request. The current-main verification path is source-backed: release docs, release workflow inputs, latest release notes, and update CLI registration show no official per-release readiness marker.

## Solution Assessment

Unclear until release owners choose the public taxonomy and criteria. The requested direction is plausible, but the maintainable solution should start with an owner-set release metadata or release-note source rather than inferred heuristics alone.
