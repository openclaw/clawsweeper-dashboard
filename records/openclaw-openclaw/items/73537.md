## Summary

Keep this open. Current main has stable/beta/dev lanes, release validation profiles, safer update mechanics, and rollback guidance, but it still does not expose an owner-set per-release Production-ready, Early adopter, Known issues, or equivalent readiness marker.

## What This Changes

The issue requests a visible production-readiness stability marker on each release, such as Production-ready, Early adopter, or Known issues.

## Best Possible Solution

Define a release-owned readiness taxonomy and require one readiness value in release notes or release metadata, then let updater CLI/UI surfaces consume that same source if maintainers want client-side display.

## Reproduction Assessment

Not applicable as a feature request. Source and docs inspection give high confidence that current main has no visible per-release readiness label or metadata field.

## Solution Assessment

Unclear until release owners choose the public taxonomy and criteria. The maintainable direction is an explicit owner-set release metadata or release-note field, not inferred heuristics alone.
