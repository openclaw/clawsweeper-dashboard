## Summary

Keep this PR open: the central change is not on current main, the patch is source-consistent and proof-backed, but it changes hosted Ollama discovery behavior enough that maintainer compatibility acceptance is still needed before merge.

## What This Changes

The PR changes the bundled Ollama plugin so hosted `*.ollama.com` configs without explicit provider model rows skip `/api/tags` live discovery, keeps local/self-hosted and explicit-model paths covered, adds discovery tests, widens local Ollama config typing, and updates Knip dependency metadata.

## Best Possible Solution

Land only after maintainers accept that hosted `models.providers.ollama` configs without explicit provider model rows no longer discover `/api/tags`, with `ollama-cloud` remaining the first-class hosted live-catalog path.

## Reproduction Assessment

Yes, source-level: current main reaches `buildProvider` for configured hosted Ollama base URLs, and a live unauthenticated probe confirms the hosted `/api/tags` endpoint returns a catalog. I did not run an authenticated OpenClaw model-list flow in this read-only review.

## Solution Assessment

Yes, with maintainer acceptance of the compatibility tradeoff: the guard is hosted-Ollama-specific, keeps local and remote self-hosted discovery, and preserves explicit model rows. The safer alternative is an upgrade note or explicit compatibility path if maintainers want hosted `ollama` auto-discovery preserved.
