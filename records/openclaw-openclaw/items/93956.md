## Summary

Keep this PR open: the hosted Ollama Cloud discovery problem still exists on current main and in the latest release, while the PR head is a focused fix with no blocking correctness finding. The remaining blocker is maintainer acceptance of the compatibility change for existing hosted `models.providers.ollama` setups.

## What This Changes

The PR changes Ollama discovery so hosted Ollama Cloud base URLs skip broad `/api/tags` catalog discovery unless explicit models are configured, while preserving local and remote self-hosted discovery.

## Best Possible Solution

Land the narrow hosted-cloud guard only after maintainers accept the hosted `models.providers.ollama` upgrade behavior and keep `ollama-cloud` as the live hosted catalog path.
