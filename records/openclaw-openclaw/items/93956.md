## Summary

Keep open: this is a focused Ollama discovery bug-fix PR, but it is not merge-ready because the guard is too broad for existing remote self-hosted Ollama discovery, the added tests do not typecheck, lint fails, and the external PR only has mock/unit proof.

## What This Changes

The PR adds a remote-base-url guard to Ollama discovery so non-local configured base URLs skip automatic model discovery unless explicit models are configured, plus tests for remote and local cases.

## Best Possible Solution

Preserve the current remote self-hosted discovery contract, add a cloud-specific skip for `ollama.com` or `ollama-cloud` only if that is the intended product behavior, and back it with typed tests plus redacted real behavior proof.

## Reproduction Assessment

Yes for the PR defects: CI reproduces the typecheck and lint failures, and source inspection shows the new guard returns before `buildProvider` for any non-local configured Ollama base URL. I did not run a live Ollama Cloud scenario in this read-only review.

## Solution Assessment

No. The idea is plausible for Ollama Cloud, but using `isLocalOllamaBaseUrl` as the cloud guard is too broad; the safer fix is cloud-host-specific and preserves authenticated remote self-hosted discovery.
