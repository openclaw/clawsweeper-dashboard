## Review Findings

Overall confidence: 0.91

Full review comments:

- **[P1] Render a bounded preview before parsing long messages:** `ui/src/ui/chat/grouped-render.ts:650`
  - body: For completed messages above the threshold, this still computes and inserts `toSanitizedMarkdownHtml(markdown)` for the entire payload before CSS hides the tail. Since the same PR raises the hard and parse limits, the large messages this feature targets can still freeze the chat view; render a capped preview until expansion and keep the cheap fallback for oversized content.
  - confidence: 0.93
- **[P2] Preserve the cheap fallback for streaming replies:** `ui/src/ui/markdown.ts:60-61`
  - body: Raising `MARKDOWN_PARSE_LIMIT` from 40k to 100k makes in-progress long replies keep going through markdown parsing for an extra 60k characters. Streaming inputs change every chunk, so the cache does not help; keep the current fallback behavior unless maintainers explicitly accept the availability tradeoff with browser proof.
  - confidence: 0.88
- **[P2] Drive the toggle from current state:** `ui/src/ui/chat/grouped-render.ts:659-674`
  - body: `isExpanded` is captured when the template is rendered, but the click handler then mutates DOM classes without updating that captured value or triggering a stable owner-state rerender. A long message can expand once while the button text and later clicks still follow the original branch until some unrelated rerender occurs.
  - confidence: 0.9
- **[P2] Keep the expanded button anchored in the message:** `ui/src/styles/chat/text.css:153-157`
  - body: `.chat-text--expanded` removes the `.chat-text--long` class that provides `position: relative`, while `.chat-text__show-more` stays absolutely positioned. After expansion the Show less button can anchor to an outer positioned ancestor instead of the message container.
  - confidence: 0.84
- **[P3] Keep the control outside the fade mask:** `ui/src/styles/chat/text.css:145-151`
  - body: The collapsed mask is applied to the same `.chat-text` element that contains the absolutely positioned button, so the Show more control is faded with the message tail. Put the control in an unmasked wrapper or apply the fade only to the preview content.
  - confidence: 0.78
