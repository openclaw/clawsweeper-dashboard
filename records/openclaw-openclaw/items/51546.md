Revise or replace the branch so collapsed long messages render only a bounded preview until expansion, preserve the current oversized and streaming fallback behavior, use stable reactive expand/collapse state, and include real Control UI proof.

## Review Findings

Overall confidence: 0.91

Full review comments:

- **[P1] Render a bounded preview before raising parse limits:** `ui/src/ui/chat/grouped-render.ts:650`
  - body: For completed messages above the 5k threshold, this still computes and inserts `toSanitizedMarkdownHtml(markdown)` for the entire payload before CSS hides the tail. Since the same PR raises the markdown hard and parse limits, the large messages this feature targets can still freeze the chat view, now with larger inputs; render only a capped preview until expansion and keep the cheap fallback for oversized content.
  - confidence: 0.93
- **[P2] Keep the streaming oversized fallback cheap:** `ui/src/ui/markdown.ts:60`
  - body: Raising `MARKDOWN_PARSE_LIMIT` from 40k to 100k makes in-progress long replies keep going through markdown parsing for an extra 60k characters, and cache hits do not help while the stream changes on every chunk. Preserve the current streaming/oversized fallback unless expansion is explicitly requested.
  - confidence: 0.88
- **[P2] Derive toggle behavior from current state:** `ui/src/ui/chat/grouped-render.ts:663-675`
  - body: The click handler closes over the render-time `isExpanded`, so after the first click it can keep taking the same branch and the button label remains bound to the stale value until an unrelated render. Store the state in reactive owner state or read the current state and request a re-render/text update on each toggle.
  - confidence: 0.9
- **[P2] Keep the disclosure button anchored and visible:** `ui/src/styles/chat/text.css:153-160`
  - body: `.chat-text__show-more` is absolutely positioned, but `.chat-text--expanded` does not provide a positioned containing block, so the Show less button can escape the message when expanded. The collapsed control also sits inside the masked element, which can fade the affordance with the message tail; keep the control outside the masked content or give both states a stable positioning wrapper.
  - confidence: 0.86
