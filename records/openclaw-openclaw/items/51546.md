## Summary

Keep open: current main has not implemented long-message disclosure, and the PR head still has blocking source-level performance and UI-state defects that need changes before merge.

## What This Changes

The PR adds Control UI Show more/Show less rendering for long markdown messages, lowers JSON auto-parse size, and raises markdown truncation, parse, and cache limits.

## Best Possible Solution

Ship a revised Control UI implementation that renders only a bounded preview until expansion, preserves cheap markdown fallback behavior during streaming, stores disclosure state reactively, keeps the control outside masked content, and adds regression tests plus a changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main has no disclosure UI and the PR diff exposes the expensive parse path, stale toggle state, and CSS positioning issue without needing a live browser run.

## Solution Assessment

No. Progressive disclosure is a reasonable direction, but this implementation should keep parse guards cheap and use bounded preview rendering with stable reactive state before merge.

## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P1] Keep the large-message parse guard cheap:** `ui/src/ui/markdown.ts:67`
  - body: Raising MARKDOWN_PARSE_LIMIT to 100k makes 40k-100k replies hit full markdown parse/sanitize again, while the new UI still renders the complete long message before CSS hides it. That reopens the large-payload freeze path this feature is meant to avoid; keep the cheap fallback or render only a bounded preview until expansion.
  - confidence: 0.91
- **[P2] Read the disclosure state at click time:** `ui/src/ui/chat/grouped-render.ts:659-673`
  - body: The click handler closes over isExpanded from the render that created the button. After the first click, later clicks on the same DOM node can keep taking the same branch, leaving labels/classes stale and making Show less unreliable until an unrelated rerender replaces the template.
  - confidence: 0.9
- **[P2] Keep the expanded control positioned inside the message:** `ui/src/styles/chat/text.css:153-157`
  - body: .chat-text__show-more is absolutely positioned, but .chat-text--expanded removes the positioning context supplied by .chat-text--long. When expanded, the Show less button can be positioned relative to an ancestor instead of the message container.
  - confidence: 0.86
- **[P3] Move the disclosure control outside the fade mask:** `ui/src/styles/chat/text.css:146-147`
  - body: The fade mask is applied to the same .chat-text--long element that contains the button, so the reveal control can be faded with the message tail. Mask only the content body or render the control outside the masked element.
  - confidence: 0.78
- **[P3] Add the required changelog entry:** `ui/src/ui/chat/grouped-render.ts:638`
  - body: This is a user-facing Control UI feature, but the diff only changes UI source and CSS files. Add an Unreleased changelog entry before merge.
  - confidence: 0.82
