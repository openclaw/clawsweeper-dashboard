## Summary

Keep open: current main still does not make the explicit `context1m` opt-in effective for GitHub Copilot or other proxied Anthropic providers, and this PR is the canonical linked implementation path; it still needs real behavior proof from the contributor before merge.

## What This Changes

The branch removes the provider-id guard from Anthropic 1M context-token resolution, adds GitHub Copilot proxied-provider regression tests, and adds a changelog entry.

## Best Possible Solution

Land or replace this with a current-main-compatible resolver/test update that honors explicit `params.context1m: true` for Claude Opus/Sonnet model ids behind proxy providers while preserving existing direct Anthropic and Claude CLI behavior.

## Reproduction Assessment

Yes. ## Solution Assessment

Yes in direction, with merge-time caveats. Removing the provider guard for the explicit opt-in path while keeping model-prefix gating is the narrow maintainable fix, but the branch should be reconciled with current-main direct Anthropic/Claude CLI 1M behavior and backed by real proof.
