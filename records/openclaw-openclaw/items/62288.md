## Summary

Keep open. Current main has partial browser-lane fixes, but it still falls short of the maintainer-defined canonical acceptance bar: profile diagnostics lose existing-session failure reasons, `browser doctor` only reports a coarse profile count, and default tab selection can still choose internal Chrome MCP pages before user tabs.

## What This Changes

The issue asks the browser plugin to prefer the managed OpenClaw browser lane while making explicit existing-session attach failures more reliable, better diagnosed, and safer to fall back from.

## Best Possible Solution

Finish this as a narrow browser-plugin fix: explicit existing-session failures should carry stable reason codes through status/profiles/doctor, and default tab selection should filter Chrome/DevTools/internal pages unless explicitly targeted.

## Reproduction Assessment

Yes. There is a high-confidence source-level reproduction: mock Chrome MCP `list_pages` with `chrome://inspect` before a real tab and verify default selection, then mock DevToolsActivePort/transport failures and verify `/profiles` plus doctor preserve the reason.

## Solution Assessment

No. Current main is only a partial solution; the maintainable fix belongs in the browser plugin with focused regression tests, not in core startup migrations or a broader browser redesign.
