## Summary

Keep open. Current main has the managed `openclaw` default and better DevToolsActivePort attach wording, but the canonical acceptance bar is still unmet: profile/status diagnostics collapse existing-session failure reasons, `browser doctor` only reports coarse profile state, and default tab selection can still pick Chrome/DevTools internal pages.

## What This Changes

Improve the browser plugin’s existing-session `profile="user"` attach path, fallback messaging, diagnostics, and default tab selection while keeping the managed `openclaw` lane preferred.

## Best Possible Solution

Finish this as a narrow browser-plugin fix: carry stable existing-session failure reasons through status/profiles/doctor and filter Chrome/DevTools/internal pages from default target selection unless explicitly requested.

## Reproduction Assessment

Yes. There is a high-confidence source-level reproduction: mock Chrome MCP `list_pages` with `chrome://inspect` before a real tab and verify default selection, then mock DevToolsActivePort/transport failures and verify `/profiles`, status, and doctor preserve actionable reason codes.

## Solution Assessment

No. Current main is only a partial solution; the maintainable fix belongs in the browser plugin with focused regression tests, not in core startup migrations or a broader browser redesign.
