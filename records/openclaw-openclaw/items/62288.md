## Summary

Keep open. Current main covers the managed default and has improved direct Chrome MCP attach errors, but the canonical diagnostics work is still incomplete: profile listing still drops Chrome MCP failure reasons, and Chrome MCP tab selection can still treat browser-internal pages as normal tabs because internal-target filtering only applies to CDP paths.

The issue requests a more reliable browser existing-session attach path with managed `openclaw` as the default, actionable fallback diagnostics, and clearer profile/status health separation.

## Best Possible Solution

Finish a focused browser-plugin repair that preserves Chrome MCP attach/list failure reasons through status, profiles, doctor, and CLI output, filters browser-internal Chrome MCP pages before tab listing/default selection, and leaves FSB-style extension bridge ideas as separate prior art.
