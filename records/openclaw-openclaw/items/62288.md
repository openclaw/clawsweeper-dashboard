## Summary

Keep open. Current main already prefers the managed `openclaw` lane and has better DevToolsActivePort failure text, but it still does not satisfy the maintainer-defined canonical acceptance bar for existing-session reason codes, profile/doctor diagnostics, and internal-tab avoidance.

The issue asks the browser plugin to keep the managed OpenClaw browser as the preferred lane while making `profile="user"` existing-session attach fallback and diagnostics less brittle.

Finish this as a narrow browser-plugin fix: preserve explicit existing-session attach failure reason codes through status/profiles/doctor, and make default tab selection skip Chrome/DevTools/internal pages unless explicitly targeted.
