## Best Possible Solution

Fix the core reply payload or delivery-policy contract so automatic visible-reply turns return a final payload whenever the completed assistant final is the user-visible answer, while preserving intentional `message_tool_only` private finals.

## Solution Assessment

No, the proposed transcript fallback is not the best long-term fix. The safer fix is to preserve the final reply through the existing embedded-run payload and source-delivery policy boundary, not recover it by reading transcript files during dispatch.
