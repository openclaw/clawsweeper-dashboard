## Summary

Keep open: this PR is not obsolete and has useful proof, but it is not merge-ready because it adds a new wildcard config/default tier without maintainer config direction, still has two `check-test-types` failures, and leaves sibling channel-key scanners that can treat `*` as a real channel id.

## What This Changes

This PR adds an ackReaction-only `channels["*"]` wildcard config tier, updates `resolveAckReaction` precedence, reserves the wildcard in several channel scanners, documents the fallback, and adds regression tests.

## Best Possible Solution

Have a maintainer decide whether OpenClaw should sponsor `channels["*"].ackReaction`; if approved, fix the type errors and reserve `*` across every channel-key consumer, otherwise keep `messages.ackReaction` as the supported cross-channel default.

## Solution Assessment
