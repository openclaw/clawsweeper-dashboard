## Summary

This PR still addresses behavior that is missing from current main and the latest release, but it is not merge-ready because the runtime fallback is still object-level, the new config tier lacks complete scanner/docs follow-through, and the contributor proof only shows validation passing rather than a real channel using the wildcard reaction.

## What This Changes

The PR accepts `channels["*"].ackReaction` in config validation, applies it in `resolveAckReaction()`, excludes `*` from several configured-channel scanners, and adds focused tests.

## Best Possible Solution

If maintainers want this wildcard tier, land it as a complete ackReaction-only contract: field-level fallback that preserves explicit empty values, all channel-id scanners reserving `*`, docs updated, focused regression tests, and redacted real channel proof.

## Solution Assessment

No. The direction is plausible, but the patch is not the best complete fix yet because it uses object-level fallback, leaves one metadata scanner and docs behind, and lacks runtime proof.
