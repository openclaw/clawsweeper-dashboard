Keep open: this branch accepts a new wildcard channel config namespace before the runtime contract and sibling scanners are aligned. Current main and the latest release still ignore the accepted wildcard fields at runtime, the companion ackReaction runtime PR is still open, `responsePrefix` has no wildcard runtime/docs contract, and additional channel-config scanners still treat only `defaults` and `modelByChannel` as reserved keys.

## What This Changes

This PR adds `channels["*"]` as a reserved accepted channel config key, validates only `ackReaction` and `responsePrefix` under it, and teaches two channel enumeration helpers to ignore the wildcard.

## Best Possible Solution

Define one wildcard channel-config contract before merge: reserve `channels["*"]` across all channel-config scanners, land only fields with matching runtime/docs/tests, and either combine or correctly sequence the ackReaction runtime fallback with this validation change.
