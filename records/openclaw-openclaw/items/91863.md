## Summary

## What This Changes

## Best Possible Solution

Define `channels["*"]` as a reserved wildcard contract across validation, schema/docs, runtime fallback, and channel-presence/status/route consumers, or keep validation limited to the exact wildcard field maintainers approve.

## Reproduction Assessment

Yes, at source level: current main's validator does not include `*` in the allowed channel keys, so `channels["*"]` reaches the `unknown channel id` path. I did not run tests because this review is read-only.

## Solution Assessment

No. Adding `*` only to the validator is too broad unless the same wildcard contract is reserved and enforced across schema/docs, runtime lookup, and sibling channel-key consumers.
