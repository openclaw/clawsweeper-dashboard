## Best Possible Solution

Full review comments:

- **[P1] Treat `*` as a reserved wildcard, not a real channel:** `src/config/validation.ts:1503`
  - body: Adding `*` here lets any `channels["*"]` object pass validation, but sibling code still treats every non-reserved `channels` key as a concrete channel; for example presence, status, and route-target scans only skip `defaults` and `modelByChannel`. Please either implement/schema/document the wildcard contract across those consumers or keep this validator narrow to the exact supported setting.
