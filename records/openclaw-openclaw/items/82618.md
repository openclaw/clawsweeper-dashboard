## Summary

Keep open: the recovery-only direction matches current main and the upstream edit-tool contract, and contributor proof is now present, but the patch still has two merge-blocking correctness issues around diff size bounding and cross-platform tests.

## What This Changes

The PR adds a recovery-branch unified-diff helper, threads its output into recovered edit success details, and adds focused agent tests.

## Best Possible Solution

Keep the recovery-only diff attachment, add a character or byte budget before returning diff text, make the wrapper tests platform-native, then merge through the paired issue path.

## Reproduction Assessment

Yes, at source level: current main returns an empty diff from recovered edit success while the HTML exporter only renders the diff block when `result.details.diff` is truthy. I did not run a full recovered session export in this read-only review.

## Solution Assessment

No: the recovery-only design is the right boundary, but this implementation should bound diff bytes/characters and use platform-native paths in the new tests before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Bound diff output by characters as well as lines:** `src/agents/pi-tools.host-edit-diff.ts:182`
  - body: The new guard only counts lines, so a generated or minified file with one very long changed line stays under both line caps and can still attach a multi-megabyte `details.diff` to the recovered result and HTML export. Add a byte or character budget before computing or returning the diff text.
  - confidence: 0.88
- **[P2] Use native paths in recovery wrapper tests:** `src/agents/pi-tools.host-edit-diff.test.ts:147`
  - body: These tests seed `/root/...` map keys while `resolveEditPath` uses Node's platform `path` module. On Windows the resolved path will not match these POSIX keys, so readback can fail before the recovery diff assertion exercises the intended branch; build the root and file keys with `path.resolve`/`path.join`.
  - confidence: 0.9
