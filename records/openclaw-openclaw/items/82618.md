## Summary

Keep open: the PR addresses a real edit-diff presentation gap, especially on recovery/export paths, but the patch is not ready because it adds unbounded duplicate diff generation, misses the upstream normal-success shape, lacks real behavior proof, and still needs UX/privacy direction for displaying file diffs.

## What This Changes

The PR adds a custom unified-diff builder to `src/agents/pi-tools.host-edit.ts`, threads generated edit diffs into edit success/recovery results, and adds focused host-edit diff tests.

## Best Possible Solution

Reuse the upstream edit result diff on normal success, add a size-bounded recovery-only fallback if needed, and prove the selected UX with a real exported-session artifact.

## Reproduction Assessment

Partly: source inspection shows the recovery result has an empty diff and the HTML renderer can consume `details.diff`. The broad normal-success claim is not high-confidence because the pinned upstream edit tool already returns `details.diff`.

## Solution Assessment

No: duplicating diff generation with an unbounded LCS is riskier than reusing the upstream edit diff contract and adding a size-bounded recovery fallback after maintainer UX direction.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Bound diff generation before allocating LCS:** `src/agents/pi-tools.host-edit.ts:107`
  - body: The output cap is applied only after `buildLineOps` has allocated the `(oldLines + 1) * (newLines + 1)` DP matrix. A large edit or recovery readback can therefore allocate millions of entries and hang or OOM after the file was written, so the edit tool can fail instead of returning the intended best-effort success.
  - confidence: 0.86
- **[P3] Match the real edit success shape:** `src/agents/pi-tools.host-edit.ts:338`
  - body: The pinned upstream `createEditTool` success result contains `content` and `details` but does not set `isError: false`; this guard only accepts explicit false. Real wrapped edit successes with missing or empty diffs skip the new attachment branch.
  - confidence: 0.78
