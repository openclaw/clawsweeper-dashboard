## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve Undici no_proxy precedence in the wrapper:** `src/infra/net/undici-global-dispatcher.ts:174-175`
  - body: Undici checks `process.env.no_proxy ?? process.env.NO_PROXY`, so an empty lowercase `no_proxy` intentionally shadows uppercase `NO_PROXY` and keeps requests proxied. Calling OpenClaw `matchesNoProxy()` here changes that behavior because blank lowercase values fall through to uppercase, so an existing setup can start bypassing the operator proxy after upgrade.
  - confidence: 0.9
- **[P2] Pass bypass connect options under connect:** `src/infra/net/undici-global-dispatcher.ts:58`
  - body: `params.connect` is the Undici connect policy, but spreading it into Agent options drops the `connect` nesting that `createHttp1Agent` and Undici expect. Bypassed NO_PROXY requests therefore lose the same WSL2/autoSelectFamily and connection policy that proxied requests receive.
  - confidence: 0.87
