## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Call the lifecycle callback only after both dispatchers settle:** `src/infra/net/undici-global-dispatcher.ts:198-199`
  - body: The shared `onSettled` invokes the caller callback as soon as `settledCount >= expectedSettled`, but `expectedSettled` is still being incremented while the underlying lifecycle calls are started. A synchronous callback can notify before the proxy dispatcher is even closed, and if one dispatcher later reports a different result the caller can be notified twice or with the wrong error. Collect both outcomes first and call the original callback exactly once, preserving the first error.
  - confidence: 0.82
