## Summary

Keep open. Current main still has the reported Control UI/config validation mismatch: array rows render with one-based labels while validation summaries preserve raw zero-based schema path segments, so the fourth visible model row can still be reported as index 3.

## What This Changes

Align Control UI array row labels and config validation error paths so a visible fourth model row is not reported to users as index 3.

## Best Possible Solution

Human-facing config validation messages should display array positions consistently with the Control UI labels, while structured RPC/JSON issue details retain zero-based schema paths.

## Reproduction Assessment

Yes. A source-level reproduction is clear: make the fourth item in models.providers.<provider>.models invalid through the Control UI form and save/apply; the visible badge is #4 while the validation summary formats the raw path segment as 3.

## Solution Assessment

Yes. The narrow maintainable fix is a display-layer formatter for human-facing config save/apply errors, not changing Zod path mapping or the shared ConfigValidationIssue contract globally.
