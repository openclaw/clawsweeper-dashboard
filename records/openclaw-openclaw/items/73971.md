## Best Possible Solution

Human-facing config validation summaries should translate array positions to the same one-based row labels shown in the Control UI, while structured RPC issue details keep raw schema paths.

## Solution Assessment

Yes. The narrow maintainable direction is a presentation-layer or schema-aware display formatter for config save/apply errors, not changing Zod path mapping or the shared machine-readable issue contract globally.
