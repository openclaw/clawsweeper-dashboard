## Summary

Keep open. Current main still has the reported Control UI/config validation mismatch: array rows are displayed one-based while human-facing config validation summaries still print raw zero-based schema path segments.

## What This Changes

The issue requests aligning Control UI model row numbering with config validation error paths so users do not confuse the fourth visible row with raw index 3.

## Best Possible Solution

Add a schema-aware display formatter for human-facing config validation summaries that renders array positions consistently with Control UI row labels while preserving raw ConfigValidationIssue.path for RPC clients.

## Reproduction Assessment

Yes. A source-level reproduction is clear: make the fourth item in models.providers.<provider>.models invalid through the Control UI form and save/apply; the visible badge is #4 while the validation summary is still derived from raw index 3.

## Solution Assessment

Yes. The narrow maintainable fix is presentation-layer formatting for human-facing config validation messages, not changing Zod path mapping or the shared machine-readable issue contract globally.
