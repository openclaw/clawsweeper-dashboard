## Summary

Keep open. Current main still has the reported Control UI/config validation mismatch: array rows render one-based labels while human-facing config validation messages still stringify raw zero-based path segments.

## What This Changes

Align Control UI model-row labels and human-facing config validation errors so they identify the same configured model row.

## Best Possible Solution

Human-facing config save/apply messages should use a schema-aware display path for array positions that matches Control UI labels while keeping ConfigValidationIssue.path raw for RPC clients.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to make the fourth item in models.providers.<provider>.models invalid through the Control UI form and save/apply; the visible badge is #4 while the validation summary is still derived from raw index 3.

## Solution Assessment

Yes. The narrow maintainable fix is a presentation-layer formatter for human-facing config validation summaries, not changing Zod path mapping or the shared machine-readable issue contract globally.
