## Summary

Keep open. Current main still has the reported Control UI/config validation mismatch: the form displays array rows as one-based labels, while backend validation paths and human-facing summaries still expose raw zero-based numeric path segments.

## What This Changes

The issue requests matching numbering between the model configuration UI and backend validation errors so users can identify the same model row in both places.

## Best Possible Solution

Add display-only, schema-aware issue path formatting for human-facing config summaries so array entries use the same one-based row labels as the Control UI, while preserving raw validation paths for machine consumers.

## Reproduction Assessment

Yes, from source inspection. Current main displays config array rows as #${idx + 1}, while validation and formatter paths still print raw numeric path segments such as models.providers.openrouter.models.0.api.

## Solution Assessment

Yes. The narrow maintainable fix is presentation-layer formatting for human-facing config validation summaries, not changing the canonical ConfigValidationIssue.path contract globally.
