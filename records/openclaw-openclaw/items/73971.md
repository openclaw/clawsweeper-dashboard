## Repair Work Prompt

Fix the Control UI/config validation numbering mismatch reported in https://github.com/openclaw/openclaw/issues/73971. When a user edits models.providers.<provider>.models in the Control UI, array rows are displayed as one-based badges such as #4, but config.set/config.apply validation summaries still show the raw zero-based path segment such as models.providers.<provider>.models.3.<field>. Add a narrow human-facing display formatter so config save/apply error messages use array positions consistent with the Control UI labels, while keeping structured RPC issue details and ConfigValidationIssue.path machine-readable/raw. Prefer src/config/issue-format.ts or the gateway/UI presentation layer; do not change Zod issue path mapping, config schema semantics, or raw persisted config paths. Add focused tests for a models.providers.*.models fourth-row failure and preserve existing root normalization/sanitization behavior. Add a CHANGELOG.md fix entry because this is user-facing.
