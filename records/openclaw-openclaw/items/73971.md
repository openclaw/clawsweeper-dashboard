Keep open. Current main still has the reported Control UI/config validation mismatch: array rows render one-based badges, while the human-facing config save/apply error summary still prints raw zero-based validation paths.

The issue requests consistent numbering between Control UI model-array row labels and backend validation errors for additional configured model rows.

## Best Possible Solution

Add a human-facing, schema-aware config issue display formatter that renders array positions consistently with Control UI row labels while preserving raw validation paths for machine consumers.

## Solution Assessment

Yes. The narrow maintainable fix is presentation-layer formatting for human-facing config validation summaries, not changing Zod path mapping or the structured ConfigValidationIssue contract globally.
