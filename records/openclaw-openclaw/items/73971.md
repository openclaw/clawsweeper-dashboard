## Summary

Keep open. Current main still has the reported Control UI/config-validation mismatch: array rows render one-based labels, while human-facing save/apply errors still print raw zero-based validation path segments.

## What This Changes

The issue asks for frontend row numbering and backend validation error numbering to be displayed consistently for configured model rows.

## Best Possible Solution

Add a schema-aware human-display formatter for config validation paths so array positions match Control UI row labels while raw machine-readable issue paths remain unchanged.

## Solution Assessment
