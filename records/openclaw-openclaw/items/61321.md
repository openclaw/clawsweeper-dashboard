## Summary

Keep this issue open. Current main still preserves structurally valid old-root same-agent sessionFile paths, and https://github.com/openclaw/openclaw/pull/61322 is an open implementation PR with closing syntax for this issue.

## What This Changes

The issue asks OpenClaw to remap stale same-agent absolute sessionFile paths from an old state root into the current agent sessions directory before write locking or transcript writes.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/61322, or an equivalent narrow resolver fix, after retargeting same-agent remaps to the active sessionsDir and adding default-store plus custom session.store regression coverage.

## Reproduction Assessment

Yes, at source level. The issue gives concrete stale metadata and CLI steps, and current main can still return the old-root structural path before the write lock creates that path's parent directory.

## Solution Assessment

Yes for the issue direction. Session path resolution before persistence and write locking is the right boundary, but the linked PR needs the remap target adjusted to the active sessionsDir/custom-store contract.
