## Summary

Keep this issue open. Current main still preserves the reported same-agent old-root absolute sessionFile path, and the provided GitHub context shows an open same-author implementation PR with closing syntax that should be reviewed rather than closing this issue early.

## What This Changes

The issue asks OpenClaw to remap stale absolute same-agent sessionFile paths from an old state root into the current state directory before session writes.

## Best Possible Solution

Land PR #61322 or an equivalent narrow resolver change that remaps only same-agent canonical old-root `agents/<agent>/sessions/<file>` paths into the current sessions directory, while preserving existing cross-agent and unsafe-path fallback behavior and adding focused regression coverage.

## Reproduction Assessment

Yes. The issue provides concrete CLI/state steps, and current main statically reproduces the failure path: the resolver returns the stale absolute path unchanged and the write lock then creates that path's directory.

## Solution Assessment

Yes. Resolving this in the session path resolver before persistence/write-lock code sees the path is the narrowest maintainable fix; a broad startup migration would miss runtime resolver entry points and has a larger compatibility surface.
