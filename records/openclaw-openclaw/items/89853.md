The PR adds post-write verification helpers for delegated agent write/edit paths and regression tests for missing-file and verification-error recovery cases.

## Best Possible Solution

Verify readback content whenever the active write/edit operations expose readFile, and keep stat/type/size checks only as a fallback for stat-only callers.

## Reproduction Assessment

Yes from source inspection. Current main can report write success once an injected writeFile resolves, and this PR's remaining gap is also source-reproducible when statFile reports a same-size stale file before readback is attempted.

## Solution Assessment

No. The PR is in the right owner boundary, but the best fix must use content readback when available so same-size wrong-content writes do not pass verification.
