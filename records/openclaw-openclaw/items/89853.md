## Summary

## What This Changes

The PR adds post-write stat/readback verification to delegated session, sandbox, host, and workspace write paths and prevents edit recovery from masking verification failures.

## Best Possible Solution

Make WriteVerificationError terminal in both write and edit recovery paths, retain exact-content readback verification, and add a regression proving write recovery cannot turn a verification failure into success.

## Solution Assessment

Not yet: post-write verification is the correct boundary, but write recovery must explicitly reject WriteVerificationError just as edit recovery does.
