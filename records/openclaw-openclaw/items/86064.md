- **[P1] Recognize the cron CLI JSON success output:** `src/agents/embedded-agent-subscribe.handlers.tools.ts:1305`
  - body: The new detector only counts output containing standalone `created` or `scheduled`, but current `openclaw cron add` writes the gateway job response as JSON via `printCronJson`; a successful job has fields such as `createdAtMs`, not a standalone receipt word. That means the reported shell path can still leave `successfulCronAdds` at zero and append the false reminder note, so please key this on the actual CLI success contract and cover that output in tests.
  - confidence: 0.91
- **[P1] Require a zero exit before counting shell cron adds:** `src/agents/embedded-agent-subscribe.handlers.tools.ts:1305-1306`
  - body: Normal exec exits with nonzero codes are still returned as `status: "completed"` and may have `isError: false`; this branch only checks for receipt words. A failed command whose output says something like `not created` or echoes `scheduled` could increment the counter and suppress the guard note, so require completed zero-exit details before incrementing and add a nonzero-with-receipt regression test.
  - confidence: 0.86
