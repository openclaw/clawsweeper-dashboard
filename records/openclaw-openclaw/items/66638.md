## Summary

Keep open. Current main has useful completion-oriented delivery, retry, pending-final-delivery state, spawn hooks, and sessions.changed broadcasts, but it still does not provide the requested first-class per-event spawn/progress/terminal notification stream with durable requester-channel delivery independent of the parent session lane.

## What This Changes

Requests framework-owned subagent lifecycle notifications for spawn, progress, completion, failure, and cancellation that are delivered promptly and durably outside parent session-lane batching.

## Best Possible Solution

Define a versioned runtime lifecycle delivery contract with per-event receipts, durable idempotent retry, per-run ordering, and safe channel-direct policy, then coordinate it with the narrower completion-spool and session-lane fairness trackers.

## Reproduction Assessment

No high-confidence live channel reproduction is available from this review. Source inspection does reproduce the current-main implementation gap: completion-only announce delivery, internal hooks, and drop-if-slow sessions.changed broadcasts do not satisfy the requested per-event durable requester-channel lifecycle stream.

## Solution Assessment

Unclear at detailed design level, but runtime-owned lifecycle delivery is the right class of solution. Prompt guidance, completion-only delivery, or per-channel sidebands would not consistently cover ordering, durability, receipts, and lane-decoupling semantics.
