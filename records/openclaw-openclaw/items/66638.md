## Summary

Keep open. Current main has partial completion-delivery improvements, but it still lacks the requested runtime-owned spawn/progress lifecycle announcement stream and a durable restart-safe announce queue decoupled from the parent/session lane.

## What This Changes

The issue requests immediate per-run subagent lifecycle notifications, including spawn ACKs, progress and terminal events, channel-direct fallback, and durable retryable announce delivery.

## Best Possible Solution

Keep this as an umbrella architecture issue for a versioned runtime-owned lifecycle delivery contract with immediate requester-channel spawn ACK, per-run progress and terminal events, safe direct delivery, and durable idempotent retry storage.

## Reproduction Assessment

Yes, for the implementation gap: current source shows completion-only internal events, plugin hooks, and a process-local announce queue rather than the requested requester-channel lifecycle stream. I did not establish a live WhatsApp timing reproduction in this read-only review.

## Solution Assessment

Unclear at the detailed design level, but the requested direction is the right class of fix. Current main is partial; prompt guidance, completion-only delivery, or per-channel sidebands would not cover spawn acknowledgement, progress, and restart-resilient lifecycle delivery consistently.
