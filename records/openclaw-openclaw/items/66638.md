## Summary

Keep open. Current main has useful partial plumbing for spawn hooks, session lifecycle events, final completion handoff, and pending final-delivery retry, but it still does not implement the requested first-class per-run spawn/progress/terminal notification stream decoupled from the parent session lane.

## What This Changes

Request a runtime-owned subagent lifecycle notification contract with immediate spawn, progress, completion, failure, and cancellation delivery outside parent session-lane batching.

## Best Possible Solution

Define a versioned runtime-owned lifecycle delivery contract that covers spawn, progress, terminal states, durable idempotent retry, and safe channel-direct policy without making parent session turns the implicit transport.

## Reproduction Assessment

Yes for the current-main implementation gap: source and docs show hooks/SSE, final task_completion delivery, and pending final-delivery retry, but no requester-channel spawn/progress lifecycle stream. I did not establish a live WhatsApp starvation repro.

## Solution Assessment

Unclear at detailed design level, but a runtime-owned lifecycle event contract is the right class of solution. Completion-only delivery, prompt guidance, or per-channel sidebands would not consistently cover spawn acknowledgement, progress, and restart-resilient delivery.
