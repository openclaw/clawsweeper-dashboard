## Summary

Keep open. Current main has useful completion delivery, pending-final-delivery retry state, spawn hooks, and sessions.changed broadcasts, but it still lacks the requested first-class spawn/progress/terminal notification stream with durable requester-channel delivery independent of the parent session lane.

## What This Changes

The issue requests runtime-owned subagent lifecycle notifications that deliver spawn, progress, terminal, and retryable announce events independently of the parent session lane.

## Best Possible Solution

Define a versioned runtime lifecycle delivery contract that covers immediate spawn acknowledgement, progress and terminal event delivery, durable idempotent retry, and safe channel-direct policy while coordinating with the narrower completion-spool and lane-fairness trackers.

## Reproduction Assessment

Yes for the current-main implementation gap: source and docs show final completion delivery, internal hooks, sessions.changed broadcasts, and pending-final-delivery retry, but no first-class requester-channel spawn/progress/terminal lifecycle stream. I did not run a live WhatsApp starvation reproduction.

## Solution Assessment

Unclear at detailed design level, but a runtime-owned lifecycle delivery contract is the right class of solution. Prompt guidance, completion-only delivery, or per-channel sidebands would not cover the requested ordering, durability, and lane-decoupling semantics consistently.
