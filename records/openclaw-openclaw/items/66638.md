## Summary

Keep open. Current main has useful partial plumbing for subagent spawn hooks, sessions.changed lifecycle broadcasts, final completion delivery, and pending final-delivery retry, but it still does not provide the requested first-class per-run spawn/progress/terminal notification stream with durable delivery independent of the parent session lane.

## What This Changes

Request immediate, per-run subagent lifecycle notifications and durable delivery that are not batched behind the parent session lane.

## Best Possible Solution

Define a versioned runtime-owned lifecycle delivery contract covering spawn, progress, terminal states, durable idempotent retry, and safe channel-direct policy without using parent session turns as the implicit transport.

## Reproduction Assessment

Yes for the current-main implementation gap: source and docs show internal hooks/SSE, final task_completion delivery, and pending final-delivery retry, but no requester-channel spawn/progress/terminal stream. I did not establish a live WhatsApp starvation repro.

## Solution Assessment

Unclear at detailed design level, but a runtime-owned lifecycle event delivery contract is the right class of solution. Prompt guidance, completion-only delivery, or per-channel sidebands would not cover the requested ordering, durability, and lane-decoupling semantics consistently.
