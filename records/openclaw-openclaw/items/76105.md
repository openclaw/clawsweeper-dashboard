## Summary

Keep open. Current main still drops hidden runtime-context rows before history readers can recover `senderLabel`, while the latest PR diff adds the missing recovery and has addressed the prior changelog, cached-payload, and streaming-visitor concerns.

## What This Changes

The PR recovers sender-label display metadata from hidden `openclaw.runtime-context` transcript rows in gateway history readers/indexes, adds regression coverage, and updates the changelog.

## Best Possible Solution

Land this PR after normal maintainer review, preserving hidden runtime-context rows while recovering only minimal `senderLabel` display metadata for history surfaces.

## Reproduction Assessment

Yes. A current-main transcript with a visible user message followed by a hidden `openclaw.runtime-context` custom row containing `Sender (untrusted metadata)` is read without `senderLabel` because both the sync converter and async index skip hidden custom rows.

## Solution Assessment

Yes. The PR fixes the recovery at the gateway history/index boundary, keeps runtime-context rows hidden, avoids caching full hidden payloads, and preserves async visitor streaming semantics.
