## Summary

Keep open. The PR fixes a real sender-label recovery gap, but the proposed implementation regresses memory behavior by caching full hidden transcript records and materializing async visitor output before visiting; it also still needs the required changelog entry.

## What This Changes

The PR updates gateway session history readers and transcript-index tests to recover `senderLabel` from hidden runtime-context custom rows while keeping those rows hidden.

## Best Possible Solution

Recover only minimal runtime-context sender metadata while preserving hidden-row privacy, cached-index memory bounds, async visitor iteration behavior, and a single-line Unreleased fix entry.

## Reproduction Assessment

Yes. A transcript with a visible Discord-origin user message followed by a hidden `openclaw.runtime-context` row containing `Sender (untrusted metadata)` currently returns the user message without `senderLabel` because main skips hidden custom rows.

## Solution Assessment

No, not yet. Recovering the label in the gateway history reader is the right direction, but the safer implementation should avoid caching full hidden records and keep `visitSessionMessagesAsync` incremental.
