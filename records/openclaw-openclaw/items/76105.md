## Summary

Keep open. Current main still drops hidden runtime-context sender metadata, and this PR is a focused fix with regression coverage and green exact-head checks; the remaining blocker is that GitHub currently reports the branch dirty against main.

## What This Changes

The PR updates gateway transcript indexing and session-history readers to recover `senderLabel` from hidden `openclaw.runtime-context` rows, adds regression tests, and adds a changelog entry.

## Best Possible Solution

Update the branch against current main, then land this gateway-owned recovery path once normal review and mergeability remain clean.

## Reproduction Assessment

Yes. On current main, a visible user transcript row followed by a hidden `openclaw.runtime-context` row with sender metadata loses `senderLabel` because the sync and async history readers only emit visible messages.

## Solution Assessment

Yes. The PR keeps runtime-context rows hidden, indexes only minimal sender-label metadata for async reads, and restores labels in the gateway history layer without changing channel storage or exposing hidden payloads.
