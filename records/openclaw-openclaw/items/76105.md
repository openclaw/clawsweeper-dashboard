## Summary

Keep open. Current main still loses sender labels from hidden runtime-context rows, and this PR is aimed at the right gateway surface, but the submitted patch introduces blocking transcript memory/latency regressions and still needs the required changelog entry.

## What This Changes

The PR changes gateway session transcript readers/indexing to recover `senderLabel` from hidden `openclaw.runtime-context` rows while keeping those rows out of visible history output, with session-utils regression tests.

## Best Possible Solution

Revise the gateway transcript reader/index so it carries only minimal runtime-context sender metadata, preserves async visitor streaming semantics, keeps runtime-context rows hidden, adds one `Unreleased` fix entry, and then lands with targeted gateway validation.

## Reproduction Assessment

Yes. A transcript with a visible user message followed by a hidden `openclaw.runtime-context` row containing `Sender (untrusted metadata)` reads back without `senderLabel` on current main because both the sync converter and async index ignore custom hidden rows.

## Solution Assessment

No, not as submitted. Recovering labels in the gateway history path is the right direction, but the implementation should avoid caching all hidden payloads and should keep `visitSessionMessagesAsync` streaming while applying a pending runtime-context sender label.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

  - body: This stores every non-visible record body in the cached transcript index. Hidden custom rows can be large, and the cache keeps up to 256 session indexes; sender recovery only needs minimal runtime-context sender metadata, so this can reintroduce gateway RSS pressure on busy installs.
  - confidence: 0.84

  - confidence: 0.86

  - body: This restores user-visible gateway history labels, but the PR does not add the required single-line `Unreleased` fix entry. Please add one before merge.
