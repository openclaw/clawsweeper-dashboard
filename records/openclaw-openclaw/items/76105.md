## Summary

Keep open. The PR addresses a real gateway history bug and supersedes the older #72951 branch, but the current diff should not merge as-is because it caches full hidden transcript payloads, regresses the async visitor into building a full message list, and lacks the required changelog entry.

## What This Changes

The PR changes gateway session-history parsing and indexing to recover `senderLabel` from hidden `openclaw.runtime-context` rows for visible user messages, with regression tests.

## Best Possible Solution

Recover sender labels while keeping runtime-context rows hidden, carry only minimal hidden-label state through the cached async index, preserve incremental visitor behavior, and add a single-line `Unreleased` fix entry.

## Reproduction Assessment

Yes. On current main, a transcript with a visible user message followed by a hidden `openclaw.runtime-context` row containing `Sender (untrusted metadata)` loses `senderLabel` because hidden custom rows are skipped by both the sync message conversion and async transcript index.

## Solution Assessment

No, not as written. The recovery belongs in the gateway history reader/index path, but the implementation should avoid caching full hidden payloads and should keep `visitSessionMessagesAsync` from materializing all messages before visiting.

## Review Findings

Overall correctness: patch is correct

Overall confidence: 0.84

Full review comments:

- **[P2] Avoid caching full hidden transcript payloads:** `src/gateway/session-transcript-index.fs.ts:180`
  - body: The new index stores `record: entry.record` for every non-visible row and then caches that index. Hidden custom/runtime rows can contain large payloads, so busy gateways may retain much more transcript data in memory; sender-label recovery should cache only minimal runtime-context metadata needed for ordering and label extraction.
  - confidence: 0.86

Artifact scans use this visitor, so long sessions allocate the full message list and delay processing; keep the visitor incremental while still allowing a small pending-message window for runtime-context label recovery.
  - confidence: 0.82
