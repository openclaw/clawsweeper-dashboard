## Summary

Keep open: current main and v2026.6.8 still skip Control UI device identity when `crypto.subtle` is unavailable, so the central HTTP LAN/reverse-proxy request is not implemented. The bypass-scope and implied-scope subissues are already fixed, but enabling persistent browser device identity in non-secure contexts is a security/product decision rather than conservative cleanup.

## What This Changes

The issue requests Control UI device identity generation without depending on browser Secure Context, plus related fixes for device-auth bypass scope handling.

## Best Possible Solution

Settle the security contract first; if maintainers approve plaintext HTTP device identity, implement the narrow UI hash/identity change with an explicit dependency contract, focused UI/auth tests, docs updates, and real LAN/reverse-proxy browser proof while keeping the already-fixed bypass-scope behavior.
