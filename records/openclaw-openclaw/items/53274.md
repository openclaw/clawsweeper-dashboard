## Summary

Keep open: current main and the latest release still skip browser device identity when `crypto.subtle` is unavailable, so the central LAN HTTP/reverse-proxy request remains unresolved. The bypass-scope and implied-scope subissues are already fixed, but enabling persistent Control UI device identity in plaintext browser contexts needs maintainer security/product approval.

## What This Changes

The issue requests generating Control UI device identity outside browser Secure Contexts for LAN HTTP/reverse-proxy deployments, with related fixes for bypass scope preservation and implied operator scopes.

## Best Possible Solution

Keep the secure default until maintainers explicitly decide the plaintext browser identity policy; if approved, land a narrow UI identity/hash change with docs, dependency updates, focused auth tests, and real LAN/reverse-proxy browser proof.

## Solution Assessment
