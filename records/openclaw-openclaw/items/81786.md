## Summary

Keep open: this broad agents/scripts guardrail port is not obsolete on current main and has sufficient terminal proof, but the cron guard currently leaves a supported packet alias in the forwarded cron payload.

## What This Changes

The PR adds runtime execution-packet guardrails for `sessions_spawn` and cron `agentTurn`, plus LOC, tsgo, dependency-ownership, script, and test hardening.

## Best Possible Solution

Keep the guardrail design, but strip every accepted packet alias before forwarding cron add/update params and add focused regression coverage for alias cleanup.

## Reproduction Assessment

Not applicable. This is an internal guardrail port rather than a bug report; review is based on current-main comparison, PR-head source inspection, contributor terminal proof, and discussion context.

## Solution Assessment

Mostly yes, but not yet mergeable as-is. The guard belongs at the agent tool boundary, but the cron path should remove every packet alias it accepts so governance metadata is not persisted or forwarded.
