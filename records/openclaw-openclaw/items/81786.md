## Summary

Keep open: the current head appears to address the prior cron alias-stripping blocker and includes real terminal proof, but the new runtime-packet heuristic would block common read-only delegated source-review tasks without an execution packet.

## What This Changes

This PR adds runtime execution-packet guardrails for side-effectful `sessions_spawn` and cron `agentTurn` work, plus related runtime-packet linting, LOC baseline handling, tsgo/local-check hardening, dependency ownership metadata, and focused tests.

## Best Possible Solution

Keep the guarded tool-boundary design, but narrow side-effect detection so read-only source review, inspection, audit, and research tasks do not require packets while mutation prompts still do and cron packet aliases stay stripped before storage.

## Reproduction Assessment

Yes for the review finding. Source inspection shows `taskTextRequiresRuntimeExecutionPacket("review source code")` would require a packet because `source` matches the side-effect regex and the read-only exemption requires an explicit no-side-effects phrase.

## Solution Assessment

No, not quite. The PR is aiming at the right tool boundary, but the classifier should avoid noun-like false positives or treat clear read-only review/inspect/audit wording as non-mutating unless stronger mutation verbs are present.
