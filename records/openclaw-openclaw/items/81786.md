## Summary

Keep open: the guardrail work is useful and not implemented on current main, but the latest head still has two source-reproducible blockers: read-only source-review tasks can be rejected as side-effectful, and the dynamic prompt snapshot fixtures were not regenerated for the new tool schemas.

This PR adds runtime execution-packet guardrails for `sessions_spawn` and cron `agentTurn` work, LOC/tsgo audit tooling hardening, dependency ownership metadata, and targeted tests.

## Best Possible Solution

Land the guardrails after narrowing the side-effect classifier for read-only source review/audit tasks and regenerating the committed prompt snapshot artifacts, while preserving the cron alias stripping fix.

## Reproduction Assessment

Yes. The remaining blockers are source-reproducible from the PR-head regex/enforcement path and the unchanged prompt-snapshot contract; I did not run tests because this was a read-only review.

## Solution Assessment

No, not yet. The overall guardrail direction is maintainable, but the classifier should avoid read-only source-review false positives and the generated prompt artifacts must match the new cron and `sessions_spawn` schemas.
