## Summary

Keep open: current main and the latest release still use `.jsonl.deleted.*` / `.jsonl.reset.*` archive names and still physically remove archived transcript artifacts through retention and disk-budget paths. Related open issues cover overlapping retention/search semantics, but none replaces this issue's core durable-archive product decision.

## What This Changes

The issue asks OpenClaw to stop using deletion-like archive suffixes for retained session transcripts and to avoid code paths that physically remove session files.

## Best Possible Solution

Maintainers should define a durable archive contract, keep old `.deleted.*` and `.reset.*` files readable, then update naming, retention cleanup, docs, and tests together.

## Reproduction Assessment

Yes at source level: current main still produces `.jsonl.deleted.*` and `.jsonl.reset.*` archive names and has cleanup paths that physically remove archive artifacts. I did not run a live `/new` or `/reset` flow because this review is read-only.

## Solution Assessment

No for a blind rename-only fix: current parsers, docs, tests, cleanup, disk-budget behavior, and related memory/session paths depend on the shipped suffixes. The best fix is a maintainer-approved retention/migration design that changes new archive semantics without losing old archives.
