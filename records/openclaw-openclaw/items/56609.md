## Summary

Keep open. Current main and the latest release still write `.jsonl.reset.*` and `.jsonl.deleted.*` transcript archives and still physically remove retained archives through retention and disk-budget cleanup; related open items overlap but do not supersede this issue's stronger never-delete/data-loss policy request.

## What This Changes

Requests changing session transcript archive naming/storage so reset/delete transcripts use archived terminology or an archive directory and are never physically removed by OpenClaw.

## Best Possible Solution

Define a compatibility-preserving transcript archive contract: keep reading legacy suffixes, write the approved non-destructive naming or archive directory going forward, make hard deletion explicit, and update cleanup, indexing, docs, config, and tests together.

## Reproduction Assessment

Yes for the current behavior. Source inspection on current main shows reset/delete paths still emit `.jsonl.reset.*` and `.jsonl.deleted.*`, and maintenance plus disk-budget paths can physically remove retained archive files.

## Solution Assessment

Unclear as a direct implementation. The preservation direction is plausible, but a rename-only change is not the best fix until maintainers choose the archive layout, legacy compatibility, indexing, privacy, and hard-delete policy.
