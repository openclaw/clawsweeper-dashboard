## Summary

Keep open. Current main and the latest shipped tag still use `.jsonl.reset.*` / `.jsonl.deleted.*` archive names and still contain retention and disk-budget paths that physically remove archive files, so the requested durable transcript policy is not implemented or superseded.

## What This Changes

The issue requests renaming or relocating session transcript archives and preventing OpenClaw from deleting session `.jsonl` history files.

## Best Possible Solution

Define a compatibility-preserving transcript archive lifecycle: keep reading legacy suffixes, write the approved non-destructive name or archive directory going forward, make hard deletion explicit, and update cleanup, indexing, docs, config, and tests together.

## Reproduction Assessment

Yes. Source inspection on current main shows reset/delete paths still emit `.jsonl.reset.*` and `.jsonl.deleted.*`, and maintenance plus disk-budget paths can physically remove retained archives; this read-only review did not run a mutating repro.

## Solution Assessment

Unclear for a rename-only change. The durable-preservation direction is sound, but maintainers need to choose archive layout, legacy compatibility, indexing, privacy, and hard-delete semantics before implementation.
