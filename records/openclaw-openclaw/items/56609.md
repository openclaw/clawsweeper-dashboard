## Summary

Keep open. Current main and the latest release still write `.jsonl.reset.*` and `.jsonl.deleted.*` transcript archives and still physically remove retained archive files through retention and disk-budget cleanup; related open work overlaps but does not supersede this issue's stronger preservation policy request.

## What This Changes

Request to rename or move session transcript archives into durable archive storage and prevent OpenClaw code paths from hard-deleting session files.

## Best Possible Solution

Define a compatibility-preserving transcript archive contract: keep reading legacy suffixes, write the approved non-destructive naming or archive directory going forward, make hard deletion explicit, and update cleanup, indexing, docs, config, and tests together.

## Reproduction Assessment

Yes. Source inspection on current main shows reset/delete paths still emit `.jsonl.reset.*` and `.jsonl.deleted.*`, and maintenance plus disk-budget paths can physically remove retained archive files; this read-only review did not execute a mutating repro.

## Solution Assessment

Unclear for a rename-only implementation. The durable-preservation direction is plausible, but maintainers need to choose archive layout, legacy compatibility, indexing, privacy, and hard-delete semantics before code changes.
