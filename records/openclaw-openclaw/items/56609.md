## Label Justifications

- `P1`: The report describes real session transcript loss risk tied to current lifecycle naming and cleanup behavior.
- `impact:data-loss`: The affected artifacts are the only retained transcript copies after reset/delete flows and can be physically removed or treated as disposable.

## Summary

Keep open. Current main still uses `.deleted.*` and `.reset.*` retained-transcript names and still has retention and disk-budget paths that can physically remove archived session artifacts; the closest open related issue covers terminology but not this issue's stronger no-delete policy.

## What This Changes

The issue asks OpenClaw to replace destructive-looking session transcript archive names with non-destructive archive semantics and avoid physical session-file deletion paths.

## Best Possible Solution

Define one maintainer-approved session transcript lifecycle: non-destructive archive naming for retained transcripts, compatibility for existing archive names, and explicit retention/deletion policy that does not imply recoverable history is trash.

## Reproduction Assessment

Yes. Source inspection on current main shows `/reset` and delete/maintenance paths still create `.jsonl.reset.*` and `.jsonl.deleted.*` archive artifacts, and maintenance/disk-budget code can physically remove archived session files.

## Solution Assessment

No single blind rename is enough. The maintainable solution needs an explicit product decision on archive naming, old-artifact compatibility, retention config semantics, and whether physical removal remains allowed.
