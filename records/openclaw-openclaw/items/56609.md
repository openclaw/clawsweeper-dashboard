## Summary

Keep open: current main and v2026.6.10 still create `.jsonl.deleted.*` and `.jsonl.reset.*` archive names, and current retention plus disk-budget paths can still physically remove those transcript artifacts. This remains a real session-durability product decision rather than stale cleanup or an implemented request.

## What This Changes

The issue requests replacing destructive-looking retained session transcript names such as `.deleted.*` and `.reset.*` with durable archive semantics or storage so session transcripts are not treated as disposable.

## Best Possible Solution

Define a maintainer-approved durable transcript archive contract, keep existing `.deleted.*` and `.reset.*` files readable or migratable, then update new archive naming, cleanup defaults, disk-budget behavior, docs, and tests together.

## Reproduction Assessment

Yes, at source level. Current main still produces `.jsonl.deleted.*` and `.jsonl.reset.*` archive names and has retention/disk-budget paths that physically remove archive artifacts; this read-only review did not run a live Gateway flow.

## Solution Assessment

No final fix is selected yet. A rename-only change would be too narrow because current parsers, tests, docs, memory/session tooling, cleanup, disk-budget behavior, and upgrade expectations depend on the shipped suffixes.
