## Summary

Keep open: current main and v2026.6.6 still create `.jsonl.deleted.*` and `.jsonl.reset.*` archive names, and both retention and disk-budget paths can still physically remove archive artifacts. The remaining work is a maintainer retention-contract decision, not a stale or already-implemented cleanup item.

## What This Changes

The issue requests clearer permanent session archive semantics by renaming or moving retained session transcripts and preventing OpenClaw from physically deleting them.

## Best Possible Solution

Maintainers should decide the durable session-archive contract first, then update archive naming/location, cleanup defaults, disk-budget behavior, migration/doctor handling, docs, and regression tests together while keeping existing `.deleted.*` and `.reset.*` archives readable.

## Solution Assessment
