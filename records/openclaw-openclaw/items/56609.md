## Summary

Keep open: current main and v2026.6.6 still use `.jsonl.reset.*` and `.jsonl.deleted.*` archive suffixes, and current retention/disk-budget paths still physically remove archive artifacts, so the central data-durability request remains unresolved.

## What This Changes

The issue requests replacing destructive-looking session archive names and preventing OpenClaw code paths from deleting session transcript files.

## Best Possible Solution

Decide the durable archive contract, then update archive naming/storage, retention behavior, legacy archive recognition, docs, and regression tests together.

## Reproduction Assessment

Yes, at source level: current main still creates `.jsonl.reset.*` and `.jsonl.deleted.*` archives and still has retention and disk-budget paths that physically remove archive files. I did not run a live `/new` or `sessions.delete` flow because this review is read-only.

## Solution Assessment

No for a blind rename-only fix: the suffixes are part of current parsers, docs, tests, and cleanup behavior. The maintainable path is a product-approved archive/retention migration that preserves old archives while changing new durable-archive semantics.
