## Summary

Keep open. Current main still writes retained session transcripts as `.jsonl.deleted.<timestamp>` and `.jsonl.reset.<timestamp>`, and it still has retention and disk-budget paths that can physically remove those archive artifacts; the closest open related issue covers much of the terminology problem but does not fully settle this issue's stronger no-delete policy request.

## What This Changes

The issue asks OpenClaw to replace destructive retained-session filename semantics such as `.deleted.*` and `.reset.*` with archive semantics, preferably an archive directory, and to avoid removing session transcripts outright.

## Best Possible Solution

Define one maintainer-approved session transcript lifecycle: non-destructive archive naming for retained transcripts, explicit compatibility for old archive names, and a clear retention/deletion policy that does not imply recoverable history is trash.

## Reproduction Assessment

Yes, at source level. Current main still produces `.reset.*` and `.deleted.*` retained transcript names and still has archive cleanup paths that can remove them after retention or disk-budget pressure.

## Solution Assessment

No single blind rename is enough. The maintainable solution needs an explicit product decision on archive naming, old-artifact compatibility, retention config semantics, and whether physical removal remains allowed.
