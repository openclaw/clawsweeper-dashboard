## Summary

Keep open: current main and the latest release still use `.jsonl.deleted.*` and `.jsonl.reset.*` archive artifacts, and current cleanup paths can physically remove expired archived transcripts. The related retention-semantics issue is useful context, but it is not a merged fix or maintainer-designated replacement for this data-loss-focused report.

## What This Changes

The issue asks OpenClaw to stop naming retained session transcript files as deleted/reset artifacts and instead use archive semantics or a permanent archive directory.

## Best Possible Solution

Keep this as the data-loss-focused anchor unless maintainers explicitly designate the retention-semantics issue as canonical, then implement a single archive lifecycle with migration, docs, and tests.
