## Summary

Keep open. Current main and the latest release still use `.jsonl.reset.*` and `.jsonl.deleted.*` transcript archives, and current maintenance/disk-budget code can still physically remove retained archive files, so the durable transcript policy requested here is not implemented or superseded.

## What This Changes

The issue asks OpenClaw to stop implying or performing session transcript deletion by replacing `.deleted.*`/`.reset.*` archive naming with clearer archive semantics and avoiding hard removal of session files.

## Best Possible Solution

Define a compatibility-preserving session transcript lifecycle that keeps reading legacy suffixes, writes the approved archive name or archive directory going forward, makes physical deletion explicit, and aligns cleanup, indexing, docs, config, and tests.

## Reproduction Assessment

Yes. Source inspection on current main shows reset/delete paths still emit `.jsonl.reset.*` and `.jsonl.deleted.*`, and maintenance plus disk-budget paths can physically remove retained archives; this read-only review did not run a mutating repro.

## Solution Assessment

Unclear for a rename-only change. The durable-preservation direction is sound, but maintainers need to choose archive layout, legacy compatibility, indexing, privacy, retention, and hard-delete semantics before implementation.
