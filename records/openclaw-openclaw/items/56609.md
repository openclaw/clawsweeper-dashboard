## Summary

Keep open. Current main and v2026.5.19 still write `.jsonl.reset.*` and `.jsonl.deleted.*` transcript archives, and retained archives can still be physically removed by retention cleanup and disk-budget enforcement, so the reported preservation policy is not implemented or superseded.

## What This Changes

The issue requests replacing `.jsonl.deleted.*` and `.jsonl.reset.*` transcript archive naming with non-destructive archive semantics and preventing OpenClaw code from physically deleting session transcript files.

## Best Possible Solution

Define a compatibility-preserving transcript archive lifecycle: keep reading legacy suffixes, write the approved non-destructive name or archive directory going forward, make hard deletion explicit, and update cleanup, indexing, docs, config, and tests together.

## Reproduction Assessment

Yes. Source inspection on current main shows reset/delete paths still emit `.jsonl.reset.*` and `.jsonl.deleted.*`, and maintenance plus disk-budget paths can physically remove retained archive files; this read-only review did not run a mutating repro.

## Solution Assessment

Unclear for a rename-only change. The durable-preservation direction is plausible, but maintainers need to choose archive layout, legacy compatibility, indexing, privacy, and hard-delete semantics before implementation.
