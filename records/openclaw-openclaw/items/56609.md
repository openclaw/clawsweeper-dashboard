## Summary

Keep open. Current main still writes `.jsonl.reset.*` and `.jsonl.deleted.*` archives and still hard-removes retained transcript artifacts under maintenance or disk-budget pressure; the newer https://github.com/openclaw/openclaw/issues/80263 overlaps, but it does not fully replace this issue’s stronger never-delete/data-loss policy request.

## What This Changes

Request to rename or relocate session transcript archives away from `.deleted.*`/`.reset.*` and prevent OpenClaw from hard-deleting session files.

## Best Possible Solution

Define a compatibility-preserving transcript archive contract: keep reading legacy suffixes, write the approved non-destructive naming or archive directory going forward, make hard deletion explicit, and update cleanup, indexing, docs, and tests together.

## Reproduction Assessment

Yes for the current behavior. Source inspection shows reset/delete/archive helpers still emit `.jsonl.reset.*` and `.jsonl.deleted.*`, while maintenance and disk-budget paths can physically remove retained archive files.

## Solution Assessment

Unclear. The data-preservation direction is plausible, but a rename-only implementation is not the best fix until maintainers choose the legacy compatibility, archive layout, and hard-delete policy.
