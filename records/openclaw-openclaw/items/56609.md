## Label Justifications

- `P1`: The issue reports real transcript loss risk and current shipped code can still physically remove retained session archives, but the fix needs retention-policy judgment rather than an emergency rollback.
- `impact:data-loss`: The request is about preventing loss or accidental deletion of irreplaceable session transcript files.
- `impact:session-state`: The affected artifacts are session transcripts used for conversation history, memory, recovery, and reset/delete lifecycle state.

## Summary

Keep open. Current main and v2026.5.22 still use `.jsonl.reset.*` and `.jsonl.deleted.*` transcript archive names, and current maintenance plus disk-budget paths can physically remove those retained transcript artifacts, so the durable session-file policy requested here remains unresolved.

## What This Changes

The issue asks OpenClaw to stop treating session transcript archives as deletable trash by renaming `.deleted.*` and `.reset.*` artifacts or moving them into a durable archive area, while avoiding hard deletes of session files.

## Best Possible Solution

Define a compatibility-preserving session transcript lifecycle that keeps reading legacy suffixes, writes the approved archive name or archive directory going forward, makes physical deletion explicit, and aligns cleanup, indexing, docs, config, and tests.

## Reproduction Assessment

Yes. Source inspection on current main shows reset/delete still produce `.jsonl.reset.*` and `.jsonl.deleted.*`, and maintenance plus disk-budget paths can physically remove retained archive artifacts; this review did not run a mutating filesystem repro.

## Solution Assessment

Unclear. The durable-preservation direction is sound, but a rename-only change is not enough; maintainers need to settle archive layout, legacy compatibility, indexing, privacy, retention, and hard-delete semantics first.
