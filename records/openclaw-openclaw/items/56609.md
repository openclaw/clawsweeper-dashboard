## Label Justifications

- `impact:data-loss`: The report is about retained transcript artifacts being treated as disposable and later lost or physically removed.

## Summary

Keep open: current main and the latest release still use `.jsonl.deleted.*` / `.jsonl.reset.*` archive names, and retention plus disk-budget paths can still physically remove those transcript artifacts. This remains a real session-retention product decision, not a stale or implemented item.

## What This Changes

The issue requests replacing destructive-looking session archive suffixes such as `.deleted.*` and `.reset.*` with non-destructive archive terminology or an archive directory, and asks that OpenClaw never physically removes session files.

## Best Possible Solution

Define a maintainer-approved durable transcript archive contract, keep existing `.deleted.*` and `.reset.*` files readable or migratable, then update new archive naming, cleanup defaults, disk-budget behavior, docs, and tests together.

## Reproduction Assessment

Yes. Source inspection shows current main still produces `.jsonl.deleted.*` and `.jsonl.reset.*` archive names and has retention/disk-budget paths that physically remove archive artifacts; I did not run a live `/new` or `/reset` flow because this review is read-only.

## Solution Assessment

No final fix is selected yet. A rename-only patch would be too narrow because current parsers, docs, tests, memory/session tooling, cleanup, disk-budget behavior, and upgrade expectations depend on the shipped suffixes.
