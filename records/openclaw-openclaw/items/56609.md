## Label Justifications

- `P1`: The current shipped behavior can physically remove retained session transcript archives, and the report ties the confusing archive naming to real irreversible transcript loss.
- `impact:data-loss`: The issue is about preventing session transcript archives from being treated as disposable and later lost or deleted.
- `impact:session-state`: The affected files are session transcripts and reset/delete archive state that determine recoverable conversation history.

## Summary

Keep open. Current main and the latest release still use `.jsonl.deleted.<timestamp>` and `.jsonl.reset.<timestamp>` for session transcript archives, and maintenance still physically removes old archive artifacts; the remaining work is a maintainer product decision around durable archive semantics, compatibility, and retention defaults.

## What This Changes

Request to rename session transcript archive files away from `.deleted.*`/`.reset.*`, preferably move them into a durable archive directory, and ensure OpenClaw never physically removes session transcript files by default.

## Best Possible Solution

Define one canonical non-destructive session archive lifecycle: new writes use a clear archive name/location, shipped `.deleted.*`/`.reset.*` files are migrated or recognized through an explicit upgrade path, retention defaults/docs/tests align, and physical transcript deletion is opt-in or separately approved.

## Reproduction Assessment

Yes, at source level: current main still creates `.jsonl.deleted.*` and `.jsonl.reset.*` archives and cleanup still removes old archive files. I did not run a live `/new` or `/reset` flow because the read-only review already found direct source and test coverage for the behavior.

## Solution Assessment

Unclear. The requested direction is plausible and aligned with data-safety goals, but maintainers need to choose the canonical archive name/location, retention semantics, and migration strategy before an implementation should land.
