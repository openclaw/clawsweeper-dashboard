## Summary

Keep open. Current main still writes `.jsonl.reset.*` and `.jsonl.deleted.*` transcript archives and still has maintenance paths that remove old archive artifacts; the related archive visibility fixes make those files searchable/countable but do not satisfy this retention and naming request. The open #65445 report asks for cleanup of the same suffix family, so maintainers should reconcile the transcript retention contract before an automated code change.

## What This Changes

The issue requests replacing `.deleted.*` and `.reset.*` session transcript archive naming with archival wording or an archive directory, and asks OpenClaw to avoid hard-deleting session transcript files.

## Best Possible Solution

Define the transcript retention contract first, then implement it compatibly: continue reading legacy `.deleted.*` and `.reset.*` archives everywhere, write the chosen `.archived.*` or archive-directory convention going forward, make any hard deletion explicit and documented, and update session reset/delete, maintenance, disk-budget, usage, memory indexing, tests, config help, and public docs together.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction exists: `sessions.reset` passes `reason: "reset"`, `sessions.delete` passes `reason: "deleted"`, the archive helper embeds that reason in the filename, and tests assert those suffixes while maintenance tests assert old archives are removed.

## Solution Assessment

Unclear. The requested direction is plausible for data preservation, but a rename-only fix is not enough because cleanup, disk-budget behavior, docs, and legacy archive readers all need a compatibility-preserving policy decision.
