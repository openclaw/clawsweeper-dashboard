## Summary

Keep open: current main and the latest release still use `.jsonl.deleted.*` and `.jsonl.reset.*` for retained transcripts, and current maintenance/disk-budget paths can still physically remove those artifacts. The remaining work is a maintainer product and upgrade decision, not stale cleanup.

## What This Changes

The issue requests replacing destructive-looking session archive names with durable archive semantics and preventing OpenClaw code paths from deleting session transcript files.

## Best Possible Solution

Maintainers should define a durable archive contract, keep existing `.deleted.*` and `.reset.*` files readable through migration/doctor guidance, then update naming, cleanup defaults, docs, and tests together.

## Reproduction Assessment

Yes at source level: current main still produces `.jsonl.deleted.*` and `.jsonl.reset.*` archive names and has cleanup paths that physically remove archive artifacts. I did not run a live `/new` or `/reset` flow because this review is read-only.

## Solution Assessment

No for a rename-only fix: current parsers, docs, tests, cleanup, disk-budget behavior, and related memory/session paths depend on the shipped suffixes. The best fix is a maintainer-approved retention and migration design that changes new archive semantics without losing old archives.
