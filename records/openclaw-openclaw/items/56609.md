## Summary

Keep open: current main and the latest release still use `.jsonl.deleted.*` and `.jsonl.reset.*` archive names, and retention/disk-budget paths can still physically remove those transcript artifacts. The related retention-semantics issue overlaps, but it does not supersede this issue's data-loss and never-delete product decision.

## What This Changes

The issue asks OpenClaw to stop using deletion-oriented session transcript archive names and to ensure retained session files are archived rather than physically deleted.

## Best Possible Solution

Define a maintainer-approved durable transcript archive contract, keep existing `.deleted.*` and `.reset.*` files readable or migratable, then update new archive naming, cleanup defaults, disk-budget behavior, docs, and tests together.

## Reproduction Assessment

Yes. Source inspection shows current main still produces `.jsonl.deleted.*` and `.jsonl.reset.*` archive names and has retention/disk-budget paths that physically remove archive artifacts; I did not run a live `/new` or `/reset` flow because this review is read-only.

## Solution Assessment

No for a rename-only patch. The best solution needs a maintainer-approved retention and migration design because current parsers, docs, tests, cleanup, disk-budget behavior, and memory/session paths depend on the shipped suffixes.
