## What This Changes

The issue reports that the in-loop context-overflow guard sizes `bashExecution`, summary, and custom harness roles as a flat 256 chars even though they render into provider-visible user text.

## Best Possible Solution

Update `tool-result-char-estimator.ts` to estimate harness roles from the exact provider-rendered text used by `convertToLlm`, including an `excludeFromContext` zero path for bash records, and add focused guard/estimator regression coverage.

## Reproduction Assessment

Yes from source inspection, but not from an executed repro in this review. Current `main` still counts these harness roles as `256` before `convertToLlm` expands them into provider-visible text, which matches the reporter's described failure path.

## Solution Assessment

Yes, the proposed direction is the narrowest maintainable fix. The estimator should share the rendered-role accounting already used by the pre-prompt estimator rather than adding a new config option or broader compaction policy.
