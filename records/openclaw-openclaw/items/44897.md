## Summary

Keep open: the reported loading hang is still source-reproducible on current main, but this branch is not merge-ready because it lacks real behavior proof, conflicts with current main, and the timeout path does not clean up the underlying pending Gateway RPC.

## What This Changes

The PR modifies `ui/src/ui/controllers/chat.ts` to race `loadChatHistory`'s `chat.history` request against a 10-second timeout, conditionally apply late successful responses, and invalidate late history on chat send.

## Best Possible Solution

Rebase onto current main and implement bounded history-timeout recovery through a cancellable or cleanup-capable Gateway request path, preserving the newer history guards and adding focused regression coverage plus real Control UI proof.

## Reproduction Assessment

Yes, source-reproducible for the narrow bug: current `loadChatHistory` awaits `chat.history`, and the await-history refresh test expects the call to stay pending until history resolves. I did not run a live browser/Gateway reproduction in this read-only review.

## Solution Assessment

No. The UI timeout direction is reasonable, but this patch needs request cleanup or cancellation and a rebase that preserves current-main history safeguards.
