## Summary

Keep open: the underlying Control UI hang is still present on current main, but this PR needs changes because the timeout leaves browser RPCs pending and late history success bypasses current history reconciliation; it also needs the required changelog entry.

## What This Changes

The PR adds a 10-second timeout around Control UI `chat.history` loading, applies some late successful responses, clears the timeout timer, and bumps a freshness token when sending chat.

## Best Possible Solution

Land a current-main-native timeout path that exits empty chat loading, removes or cancels timed-out browser RPCs, and applies any safe late success through the shared history reconciliation path.

## Reproduction Assessment

Yes. Mock `state.client.request("chat.history")` with a never-settling Promise; current main keeps `loadChatHistory` pending with `chatLoading` true, and the empty chat renderer shows the loading skeleton for that state.

## Solution Assessment

No. A bounded UI timeout is the right direction, but this patch is not the best fix until it cancels or removes timed-out RPCs and reuses the existing history filtering/reconciliation logic.
