## Summary

Keep open: current main still has a source-reproducible empty-chat loading hang when `chat.history` never settles, but this PR is not merge-ready because its timeout leaves the browser RPC pending and its late-success path bypasses current history reconciliation.

## What This Changes

The PR wraps Control UI `loadChatHistory`'s `chat.history` request in a 10-second `Promise.race`, clears the timeout timer, and adds a freshness-guarded late-success handler.

## Best Possible Solution

Land a current-main-native Control UI fix that bounds `chat.history` stalls, cancels or forgets timed-out browser RPCs, applies late successes through shared history reconciliation, and adds regression tests plus a changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: mock `state.client.request("chat.history")` with a never-settling Promise, and `loadChatHistory` never reaches the `finally` block that clears `chatLoading`.

## Solution Assessment

No. A bounded UI timeout is the right direction, but the PR is not the best fix until the timed-out browser RPC is cancellable or forgotten and late successes reuse the same reconciliation path as normal history loads.
