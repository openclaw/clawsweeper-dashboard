## Summary

Keep open: the PR targets a plausible Control UI hang, but the current patch still leaves timed-out Gateway RPCs alive and has no after-fix real behavior proof. Current main has newer responsiveness and history-guard work, but it still does not provide a bounded or cancellable `chat.history` load path.

## What This Changes

The PR wraps Control UI `loadChatHistory` around a 10s timeout, clears the timer, and applies late successful `chat.history` responses behind freshness guards.

## Best Possible Solution

Implement bounded `chat.history` recovery with request cancellation or pending-entry cleanup while preserving current main's retry, session, history-limit, and optimistic-tail behavior, then add focused regression coverage and redacted real behavior proof.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection shows a never-settling `chat.history` request leaves `loadChatHistory` awaiting with `chatLoading` true, and current tests confirm unresolved history can remain pending.

## Solution Assessment
