## Summary

Keep open: current main still has an unbounded startup history await, and the PR addresses the right symptom but still leaves timed-out RPCs pending and bypasses current history reconciliation.

## What This Changes

The PR wraps Control UI `loadChatHistory`'s `chat.history` request in a 10-second `Promise.race`, clears the timeout timer, and applies late successful responses with freshness/session guards.

## Best Possible Solution

Land a current-main-native Control UI fix that bounds `chat.history` stalls, cancels or forgets timed-out browser RPCs, reconciles late successes through the shared history path, and adds regression coverage plus a changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: make `state.client.request("chat.history")` return a never-settling Promise, and `loadChatHistory` never reaches the `finally` block that clears `chatLoading`.

## Solution Assessment

No. A bounded UI timeout is the right direction, but this PR should also cancel or forget the underlying pending browser RPC and reuse current history reconciliation for late successes.
