## Summary

Keep this PR open: it targets a real source-reproducible Control UI hang, but the branch is not merge-ready because the timeout only races the UI promise and leaves the underlying Gateway RPC pending, and it still lacks after-fix real behavior proof.

## What This Changes

The PR changes `ui/src/ui/controllers/chat.ts` to race `chat.history` against a 10s timeout, clear the timer, and apply late successful history responses behind freshness guards.

## Best Possible Solution

Port bounded history recovery onto current main with request cancellation or explicit pending-entry cleanup, preserving the current history guards, limits, startup retry, optimistic-tail behavior, and focused regression coverage.

## Reproduction Assessment

Yes, source-reproducible: a never-settling `chat.history` promise keeps `loadChatHistory` awaiting, and current tests deliberately expect an `awaitHistory` refresh to remain pending until the promise resolves. No live browser/Gateway reproduction was run in this read-only pass.

## Solution Assessment

No. A UI-level `Promise.race` mitigates the spinner, but the mergeable fix needs cancellation or cleanup of the underlying Gateway request and must be ported onto current main's newer history behavior.
