## Summary

Keep open: current main still has a source-reproducible path where a never-settling `chat.history` leaves the Control UI loading, but this PR is not mergeable as-is because the timeout does not clean up the underlying Gateway RPC and it lacks real behavior proof.

## What This Changes

The PR changes Control UI `loadChatHistory` to race `chat.history` against a 10s timeout and apply late successful responses behind freshness guards.

## Best Possible Solution

Implement bounded `chat.history` recovery with Gateway request cleanup or cancellation while preserving current main's retry, session, history-limit, and optimistic-tail behavior, then add focused regression coverage and redacted real behavior proof.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection gives a clear current-main path: a never-settling `chat.history` request leaves `loadChatHistory` awaiting with `chatLoading` true, and the chat view renders the loading skeleton while no messages exist.

## Solution Assessment

No. A UI-level `Promise.race` mitigates the spinner, but the mergeable fix needs to clean up or cancel the underlying Gateway request and be ported onto current main's newer history behavior.
