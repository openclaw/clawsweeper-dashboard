## Best Possible Solution

Implement a current-main chat-history timeout that actively cancels or removes the pending Gateway RPC, preserves existing session/history guards, and proves the Control UI exits loading without losing chat state.

## Reproduction Assessment

## Solution Assessment

No. The timeout direction is useful, but the best fix must be rebased onto current main and cleanly cancel or remove the pending RPC while preserving current history/session guards.
