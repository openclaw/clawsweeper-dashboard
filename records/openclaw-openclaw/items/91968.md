## Summary

## What This Changes

This PR changes `cron.status().storePath` from the configured logical cron store key/import path to the shared SQLite database path and updates service and gateway assertions.

## Best Possible Solution

Preserve the shipped logical-key field and add a clearly named canonical SQLite database-path field through the protocol, CLI, tests, and documentation.

## Reproduction Assessment

Yes at source level. Current source and release `v2026.6.5` return the logical `jobs.json` key, and the linked issue provides corresponding real CLI output after persistence moved to SQLite.

## Solution Assessment

No. Reporting the SQLite location is useful, but silently repurposing a shipped field is less safe and maintainable than adding an explicitly named database-path field.
