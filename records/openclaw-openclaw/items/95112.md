## Summary

Keep open: the Tavily bug is valid, but this patch is not merge-ready because it lets any unresolved configured SecretRef fall through to ambient `TAVILY_API_KEY`, and the submitted proof only verifies source text rather than the real runtime behavior.

## What This Changes

This PR replaces Tavily API-key strict SecretRef resolution with inspect-mode resolution so unresolved refs fall through to the existing `TAVILY_API_KEY` environment fallback.

## Best Possible Solution

Land one canonical Tavily resolver fix that keeps inspect mode, only allows fallback for the matching permitted `TAVILY_API_KEY` env ref, adds focused credential-boundary tests, and includes redacted source-build runtime proof.

## Reproduction Assessment

Yes. Source inspection shows current main resolves Tavily's configured API-key SecretRef through strict `normalizeResolvedSecretInputString` before reaching the existing `TAVILY_API_KEY` fallback; I did not run the live Tavily command in this read-only review.

## Solution Assessment

No, not as submitted. Inspect mode is the right primitive, but the patch needs Firecrawl-style guardrails so only the intended env SecretRef can fall through to ambient `TAVILY_API_KEY`.
