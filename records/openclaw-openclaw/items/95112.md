## Summary

Keep open: the bug is real, but this patch is too broad because it treats every unresolved configured SecretRef as missing and then falls through to ambient `TAVILY_API_KEY`, which can bypass an explicit credential boundary.

## What This Changes

The PR changes Tavily API-key resolution from strict SecretRef normalization to inspect-mode resolution so unresolved refs can fall through to the existing environment fallback.

## Best Possible Solution

Use inspect mode only with guardrails: allow the intended `env:default:TAVILY_API_KEY` fallback, fail closed for other unresolved refs, and add regression tests plus redacted runtime proof.

## Reproduction Assessment

Yes. Source inspection shows the current Tavily path calls strict SecretRef normalization before the documented `TAVILY_API_KEY` fallback; I did not run the live Tavily command in this read-only review.

## Solution Assessment

No. Inspect mode is the right building block, but this implementation is too broad because it erases the difference between missing config and an explicit unresolved SecretRef.
