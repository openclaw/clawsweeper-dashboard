## Summary

Keep open: the PR targets a real Tavily SecretRef/env fallback bug, but the patch is not merge-ready because it treats every configured-unavailable SecretRef as absent and can silently substitute ambient `TAVILY_API_KEY` for explicit file, exec, custom-provider, or unrelated env refs.

## What This Changes

The PR changes Tavily API-key normalization to use SecretRef inspect mode so unresolved configured SecretRefs fall through to the existing `TAVILY_API_KEY` environment fallback.

## Best Possible Solution

Keep the Tavily env fallback, but implement it with Firecrawl-style SecretRef-aware guardrails and add focused tests plus redacted runtime proof from the PR branch.

## Reproduction Assessment

Yes, from source inspection: current main and `v2026.6.8` strictly resolve the configured Tavily SecretRef before evaluating the existing `TAVILY_API_KEY` fallback. I did not run the live CLI command in this read-only review.

## Solution Assessment

No. Inspect mode is the right primitive, but this implementation is too broad because it collapses all unavailable configured refs into a missing value instead of gating fallback to the matching Tavily env ref.
