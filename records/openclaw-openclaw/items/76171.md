## Summary

Keep open. Current main and the latest release still leave the isolated cron CLI branch without run-end cleanup flags, while nearby embedded and one-shot CLI paths already have cleanup controls; related open PRs only cover overlapping pieces, not the full reported stale-worker lifecycle.

## What This Changes

The issue asks OpenClaw to prevent completed cron/agent worker processes from accumulating and slowing hosts under frequent isolated cron runs.

## Best Possible Solution

Reuse the existing run-end cleanup contract for isolated cron CLI runs, with focused regression coverage and a Docker/Railway-style stress smoke that records worker command lines, counts, load, and cleanup behavior.

## Solution Assessment

Yes. The narrow maintainable fix is to pass the existing cleanup flags for isolated cron CLI runs and preserve session reuse for non-isolated targets, while coordinating with the open one-shot cleanup PR overlap.
