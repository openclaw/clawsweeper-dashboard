## Summary

Keep open. Current main and v2026.6.6 still route the isolated cron CLI provider branch through `runCliAgent` without the existing run-end cleanup flags, while adjacent MCP cleanup work remains open or closed unmerged rather than a shipped fix.

## What This Changes

The issue reports stale OpenClaw worker processes accumulating after cron/agent turns, causing high host load and slow or stalled responses.

## Best Possible Solution

Make isolated cron CLI executions true one-shot runs by propagating the existing live-session and bundled-MCP cleanup contract through the CLI branch, then cover CLI and embedded cron cleanup paths with focused regression tests while coordinating with the open MCP cleanup PR.

## Reproduction Assessment

Yes at source level, but not by a live stress run in this review. Current main and v2026.6.6 show the reported cleanup contract exists but is not wired into the isolated cron CLI branch.

## Solution Assessment

No, current main is not yet the best fix because it leaves the CLI cron branch without the cleanup flags. The best path is a narrow lifecycle fix or coordinated landing of the open MCP cleanup work if it covers this exact branch.
