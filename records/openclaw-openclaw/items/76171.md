## Repair Work Prompt

Fix the stale worker accumulation reported at https://github.com/openclaw/openclaw/issues/76171. In current main, the isolated cron CLI-provider path in src/cron/isolated-agent/run-executor.ts calls runCliAgent without cleanupCliLiveSessionOnRunEnd or cleanupBundleMcpOnRunEnd, while runCliAgent already closes Claude live sessions and MCP loopback resources when those one-shot flags are true. Wire isolated cron CLI runs through the existing cleanup flags, add focused regression coverage proving the flags are passed for isolated cron CLI jobs and not applied to persistent/non-isolated semantics, and avoid broad pkill, age-based cleanup scripts, Docker entrypoint changes, or service-marker policy changes. Add a user-facing changelog fix entry unless maintainers decide this is purely internal. Validate with targeted cron/CLI tests and, if available, a Docker/Railway-style stress smoke that records process command lines, worker counts, load average, and cleanup behavior over repeated isolated cron runs.
