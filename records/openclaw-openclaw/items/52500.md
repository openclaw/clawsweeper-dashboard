## Summary

Keep open. Current main still maps bounded Zod config failures through the generic message path without a normalized maximum/minimum hint, and the open closing PR has not merged.

## What This Changes

Request schema-validation diagnostics for ceiling-constrained config values to explicitly include the allowed maximum value.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/73220 or an equivalent central mapper change that adds accurate numeric bound hints with focused regression coverage.

## Reproduction Assessment

Yes by source inspection. Values above `session.agentToAgent.maxPingPongTurns`'s `.max(20)` ceiling reach `mapZodIssueToConfigIssue()`, which currently returns the raw Zod message without normalized maximum-bound guidance.

## Solution Assessment

Yes, the central mapper is the right surface because it solves schema-generated config errors once instead of adding per-field messages. The linked PR is the right direction but needs to match current main and handle Zod origin/inclusive details before merge.
