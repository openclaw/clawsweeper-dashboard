## Summary

Keep open. Current main still maps bounded Zod config failures through the generic message path without a normalized maximum/minimum hint, and both prior closing PRs for this issue are closed unmerged.

## What This Changes

The issue requests that schema-validated numeric ceiling and floor rejections state the maximum or minimum allowed value explicitly in operator-facing config errors.

## Best Possible Solution

Add a central validation-mapper helper that appends accurate numeric max/min hints for Zod numeric-origin bound issues, respects inclusive versus exclusive bounds, preserves allowed-values behavior, and adds focused regression coverage.

## Reproduction Assessment

Yes by source inspection. Values above `session.agentToAgent.maxPingPongTurns`'s `.max(20)` ceiling reach `mapZodIssueToConfigIssue()`, which currently returns the raw Zod message without normalized maximum-bound guidance.

## Solution Assessment

Yes, the central mapper is the right surface because it fixes schema-generated config diagnostics once. The previously proposed PR direction was close, but the helper should be numeric-origin aware and handle exclusive bounds before landing.
