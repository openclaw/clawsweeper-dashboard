## Summary

Keep open. Current main still has no central `too_big`/`too_small` bound-hint enrichment in config validation, and open PR #73220 explicitly targets this issue with closing syntax, so the issue should remain open until that PR lands or is closed.

## What This Changes

Config validation errors for schema-enforced numeric ceilings should explicitly state the maximum allowed bound.

## Best Possible Solution

Keep this issue open until PR #73220 lands or an equivalent central mapper change adds generic numeric maximum/minimum hints with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection shows `session.agentToAgent.maxPingPongTurns` rejects values above `5`, while the central config mapper has no `too_big`/`too_small` branch to normalize maximum or minimum bound guidance.

## Solution Assessment

Yes. A generic `mapZodIssueToConfigIssue()` enhancement using Zod’s structured bound fields is the narrowest maintainable fix; per-field custom messages would miss other schema-enforced bounds.
