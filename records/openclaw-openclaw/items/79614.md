## Summary

Keep open. Current main has active-run steering and a bounded `before_agent_finalize` hook, but source and docs do not show the requested host-owned latest-user snapshot, complaint flag, or built-in stale-reply re-anchor contract.

## What This Changes

The issue requests a built-in stale-reply recheck that re-reads the latest user message at finalization, detects stale-reply complaints, and allows one configured re-anchor pass before sending.

## Best Possible Solution

Define an explicit host-owned freshness contract around active-run finalization, then add focused Pi/Codex regression tests, config/schema docs, and observability for any bounded re-anchor pass.
