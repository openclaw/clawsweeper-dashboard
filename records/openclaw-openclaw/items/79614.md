## Summary

Keep open. Current main has active-run steering and a bounded `before_agent_finalize` revision path, but it still lacks the requested host-owned latest-user snapshot, complaint flag, and built-in stale-reply re-anchor contract; the remaining choice is API/product design rather than cleanup.

## What This Changes

Request a host-owned stale-reply recheck that snapshots the newest inbound message at finalization, detects stale-reply complaints, and optionally triggers one bounded re-anchor pass.

## Best Possible Solution

Define an explicit host-owned freshness contract around finalization, then add focused Pi/Codex regression coverage, config/schema docs, and observability for any bounded re-anchor pass.

## Solution Assessment
