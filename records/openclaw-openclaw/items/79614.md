## Summary

Keep open. Current main has a foreground stale-final delivery fence and a bounded `before_agent_finalize` hook, but it still does not provide the requested host-owned latest-user snapshot, built-in complaint signal, or configurable re-anchor contract; related open issues track narrower stale-reply root causes rather than replacing this product/runtime contract request.

## What This Changes

Add a host-owned finalize-time latest-user recheck and complaint-triggered re-anchor path so stale assistant drafts are revised or explicitly deferred without a workspace-local plugin.

## Best Possible Solution

Define a host-owned freshness and re-anchor contract that composes with the existing foreground freshness fence, `before_agent_finalize`, retry prompts, and queue/orphan repair paths, then document the config and add focused Pi/Codex regression coverage.
