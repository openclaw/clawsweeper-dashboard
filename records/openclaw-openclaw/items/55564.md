## Summary

Keep open. Current main still runs startup outbound delivery recovery without an age or restart-generation cutoff for first-attempt queued entries, and the focused fix path remains the open, unmerged https://github.com/openclaw/openclaw/pull/54900.

## What This Changes

The issue requests preventing stale outbound delivery-queue entries from being replayed in a batch after gateway restart, potentially through TTL or discard behavior.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/54900 with a shared outbound recovery freshness fix that skips or quarantines old never-attempted queued replies while preserving legitimate recent crash recovery.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to seed an old pending delivery entry with retryCount 0 and no lastAttemptAt, then run recoverPendingDeliveries; current main treats it as eligible and the gateway startup path calls recovery with the real outbound delivery function.

## Solution Assessment

Unclear as proposed. The shared outbound recovery boundary is the right place to solve this, but the exact TTL/default/quarantine policy and the open PR's attempted-send predicate need maintainer review before merge.
