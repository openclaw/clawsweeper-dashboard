## Summary

Keep open. Current main still replays pending outbound delivery entries on gateway startup without an age or restart-generation cutoff, and the focused implementation candidate at https://github.com/openclaw/openclaw/pull/54900 is still open and unmerged.

## What This Changes

The issue requests preventing stale queued outbound messages from flooding channels after a gateway restart, with TTL, discard, or configurable max-age behavior.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/54900 with a shared outbound recovery freshness fix that skips or quarantines old never-attempted queue entries while preserving legitimate recent crash recovery.

## Reproduction Assessment

Yes, at source level. Seed an old pending delivery entry with retryCount 0 and no lastAttemptAt, then run recoverPendingDeliveries; current main treats it as eligible and gateway startup calls recovery with the real outbound delivery function.

## Solution Assessment

Unclear as proposed. The shared outbound recovery boundary is the right place, but the TTL/default/quarantine policy and the open PR's edge cases need maintainer review before merge.
