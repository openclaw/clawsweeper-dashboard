## Summary

Keep open. Current main and v2026.5.18 still recover old pending outbound delivery entries without a queued-age or expiry gate, and the focused implementation candidate is still open and unmerged.

## What This Changes

The issue requests preventing stale outbound delivery queue entries from replaying in bulk after gateway restart, potentially through a TTL or configurable recovery policy.

## Best Possible Solution

Land or replace a focused outbound delivery expiry policy that quarantines stale queued replies while preserving recent crash recovery, reconnect drains, budget deferrals, and uncertain-send reconciliation.

## Reproduction Assessment

## Solution Assessment

Unclear as proposed. The outbound recovery boundary is right, but the TTL default, expiry action, config surface, and the open PR's stale predicate still need maintainer alignment before this can close.
