## Summary

Keep open. Current main still starts delivery recovery without an age or restart-generation cutoff for stale pending outbound entries, and the focused implementation path remains the open, unmerged https://github.com/openclaw/openclaw/pull/54900.

## What This Changes

The issue requests stale outbound delivery entries to be dropped, quarantined, or bounded by a TTL after gateway restart so old channel messages are not replayed in a flood.

## Best Possible Solution

Land or replace the focused outbound recovery fix so old never-attempted queued replies are skipped or quarantined while legitimate recent crash recovery remains intact.

## Reproduction Assessment

Yes, at source level. Seed an old pending delivery entry with retryCount 0 and no lastAttemptAt, then run recoverPendingDeliveries; current main treats it as eligible and gateway startup calls that recovery path.

## Solution Assessment

Unclear as proposed. The shared outbound recovery boundary is the right place to solve this, but the TTL default/action, config exposure, and the open PR's attempted-send predicate need maintainer alignment before merge.
