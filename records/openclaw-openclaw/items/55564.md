## Summary

Keep open. Current `main` still recovers first-attempt queued outbound deliveries on gateway startup without any age, expiry, or restart-generation cutoff, and the focused fix remains open and unmerged.

## What This Changes

The issue asks for stale outbound delivery-queue messages from before a gateway restart to be dropped, expired, or controlled by a TTL instead of replayed in a batch.

## Best Possible Solution

Land or replace the focused outbound recovery fix so stale never-attempted queued replies are skipped or quarantined while legitimate recent crash recovery remains intact, with any operator-facing TTL semantics coordinated through https://github.com/openclaw/openclaw/issues/16555.

## Reproduction Assessment

Yes, at source level. Seed an old pending delivery entry with `retryCount: 0` and no `lastAttemptAt`, then run `recoverPendingDeliveries`; current `main` treats it as eligible and gateway startup calls that recovery path.

## Solution Assessment

Unclear as proposed. The shared outbound recovery boundary is the right place, but the TTL default/action and the open PR's attempted-send predicate need maintainer alignment and repair before merge.
