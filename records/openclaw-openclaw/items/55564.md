## Summary

Keep open. Current main still makes old first-attempt pending outbound deliveries eligible for gateway startup recovery without any queued-age or expiry cutoff, and the focused implementation remains open and unmerged.

## What This Changes

Request that gateway restart recovery drop, expire, or age-limit stale queued outbound messages instead of replaying them as a batch.

## Best Possible Solution

Land or replace the focused outbound recovery change so stale pending replies are skipped or quarantined on startup while legitimate recent crash recovery remains intact; coordinate any operator-facing TTL config through https://github.com/openclaw/openclaw/issues/16555.

## Reproduction Assessment

Yes, at source level. Seed an old pending delivery entry with retryCount: 0 and no lastAttemptAt, then run recoverPendingDeliveries; current main treats it as eligible and gateway startup calls that path.

## Solution Assessment

Unclear as proposed. The shared outbound recovery boundary is the right place, but the TTL default/action/config policy and the open PR's stale predicate still need maintainer alignment before this can close.
