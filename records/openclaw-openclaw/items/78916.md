## What This Changes

The branch adds a QQBot explicit-target parser, registers it on the QQBot channel, makes cron delivery previews avoid re-adding an already-present provider prefix, adds parser/preview tests, and updates the changelog.

## Best Possible Solution

Land this focused parser and preview fix after maintainer merge review, then close https://github.com/openclaw/openclaw/issues/78893 as fixed by the merge.

## Reproduction Assessment

Yes. Current main is source-reproducible because cron preview formatting blindly prepends the channel and QQBot lacks a channel-owned explicit parser, so `delivery.channel=qqbot` with `delivery.to=qqbot:c2c:...` can display a duplicated provider prefix.

## Solution Assessment

Yes. Registering QQBot's parser through the existing channel-owned seam and making preview formatting prefix-aware is the narrow maintainable fix without hard-coding QQBot-specific policy into generic cron storage.
