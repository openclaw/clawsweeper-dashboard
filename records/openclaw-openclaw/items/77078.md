## Summary

Keep this PR open: it addresses a real current-main Talk transport gap and the diff is directionally sound, but the user-facing docs/changelog still describe the old browser-direct OpenAI/Google behavior and one exact-head CI check was still running when inspected.

## What This Changes

The PR routes browser Talk realtime sessions through the Gateway relay, removes Google Live browser-direct session creation, updates relay rebinding/ready replay behavior, and adjusts Google PCM16 silence handling and tests.

## Best Possible Solution

Land the Gateway-relay Talk behavior after updating the public docs and changelog, then let exact-head CI finish before merge.

## Solution Assessment
