## Review Findings

Overall confidence: 0.88

Full review comments:

- **[P1] Keep startup memory out of raw model probes:** `src/gateway/server-methods/agent.ts:1528`
  - body: This new first-turn branch runs for any new Gateway session. `openclaw infer model run --gateway` uses the raw model path and the docs promise it sends only the supplied prompt without transcript/bootstrap context; on a fresh or rotated session this condition can still resolve and prepend daily memory. Gate this path out for `isRawModelRun` and the existing internal non-user sources before resolving or prepending startup context.
  - confidence: 0.87
- **[P1] Do not gate reset prompt resolution on startup context:** `src/auto-reply/reply/get-reply-run.ts:619-621`
  - body: Including `shouldApplyStartupContext` in `shouldResolveStartupContext` makes `startupPromptState` null when a user has `startupContext.enabled=false` or `applyOn` excludes the reset action. Bare `/new` or `/reset` then gets an empty `baseBodyFinal` and can fall through as an empty message instead of the established reset greeting. Resolve the bare reset prompt state independently, and only gate the daily-memory prelude on the startupContext setting.
  - confidence: 0.9
