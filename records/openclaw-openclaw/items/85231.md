## Summary

Keep open. The direction is useful, but the patch is not merge-ready because source inspection shows the new recorder only covers direct sends and preview-finalized drafts while other successful Telegram final delivery modes still bypass prompt-context caching, and the supplied proof is synthetic rather than real Telegram reply-context proof.

## What This Changes

The PR adds a Telegram outbound prompt-context recorder, calls it from direct Telegram sends, media sends, and preview-finalized streamed finals, and adds focused regression tests.

## Best Possible Solution

Record outbound prompt context at a common successful visible Telegram delivery boundary that can return message ids and visible text/media for preview, progress/off, media, and follow-up deliveries, then add redacted live Telegram proof.

## Reproduction Assessment

Yes, at source level: current main selects Telegram prompt context from the message cache, while outbound assistant replies are not recorded into that cache before this PR. I did not run a live Telegram reproduction in this read-only review.

## Solution Assessment

No; the current implementation records at a few call sites rather than at the common successful delivery boundary. A narrower maintainable fix should cover every visible final delivery path or explicitly document and test any excluded modes.
