## Summary

This PR is a focused fix for a real Telegram/Codex context-retention gap, and I found no discrete blocking code-review finding. It should stay open for maintainer signoff because it intentionally expands selected tool-result content into Codex prompt context and changes live Telegram follow-up context behavior.

## What This Changes

The PR caches successful Telegram outbound sends and preview-finalized streamed finals into Telegram prompt-context storage, and changes Codex per-turn context-engine projections to preserve redacted tool-result evidence.

## Best Possible Solution

Land the combined fix only after maintainer acceptance of the selected tool-result prompt boundary; otherwise split and land the Telegram outbound-context cache fix separately.

## Reproduction Assessment

Yes at source level: current main records inbound Telegram messages but not outbound sends or preview-finalized replies, and active per-turn Codex context projections still elide tool payloads without a bootstrap projection. I did not run a live Telegram reproduction in this read-only review.

## Solution Assessment

Yes for the Telegram cache gap, and conditionally for the Codex projection gap if maintainers accept the prompt-boundary tradeoff. If that boundary is not accepted, the safer solution is to split the Codex projection change from the Telegram outbound-context fix.
