## Summary

The source path confirms the current PR does not cover the observed normal streamed Telegram final: `sendMessageTelegram` would cache direct sends, but streamed assistant finals are finalized through the draft-stream and `editMessageTelegram` path, which only emits hooks/transcript mirroring today. Codex tool evidence preservation is plausible, but it intentionally broadens selected model context and needs maintainer acceptance plus after-fix real behavior proof.

## What This Changes

The branch changes Codex context-engine projection to preserve redacted tool evidence and records `sendMessageTelegram` outbound sends into the Telegram prompt-context cache, with focused regression tests.

## Best Possible Solution

Record preview-finalized streamed Telegram finals into the same prompt-context cache, keep or split the Codex tool-evidence change after maintainer boundary acceptance, and prove the patched behavior with redacted Telegram context diagnostics.

## Reproduction Assessment

Yes at source level: current main builds Telegram prompt context from `messageCache`, inbound messages are recorded there, and streamed assistant finals are finalized through a draft-stream path that does not record to that cache. I did not run a live Telegram repro in this read-only review.

## Solution Assessment
