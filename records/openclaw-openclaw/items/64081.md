## Summary

Keep open. Current main still lacks a typed, schema-backed, documented config-level default for task notification policy; adjacent ACP/subagent delivery work reduces some banner symptoms but does not implement the requested config surface.

## What This Changes

The issue requests a config-level default task notify policy for background task records, especially ACP and subagent completions.

## Best Possible Solution

Add one backward-compatible task notify default contract, likely under agents.defaults.tasks, resolve it before ACP/subagent/Task Flow records are created, preserve existing behavior when unset, let explicit task or flow policy win, and update schema, exported types, docs, and focused regression coverage together.

## Reproduction Assessment

Yes for the configuration gap: current-main source inspection shows hardcoded task notify defaults and no accepted agent-defaults task notify key. This is source-reproducible feature work rather than a failing bug reproduction.

## Solution Assessment

Unclear because there is no patch and the main question is the public config API. The maintainable direction is a single coherent config contract across task registry, Task Flow, ACP, and subagent creation rather than another one-path runtime knob.
