## Summary

Keep open. Current main still lacks a typed, schema-backed, documented config-level default task notification policy, and the remaining work is a user-facing config/API decision rather than stale cleanup.

## What This Changes

The issue requests a global or per-runtime config default for background task notification policy so ACP/subagent completions can be silent by default.

## Best Possible Solution

Add one backward-compatible task notify default contract, likely under agents.defaults.tasks, resolve it before ACP/subagent/Task Flow records are created, preserve existing behavior when unset, and let explicit task/flow policy win.

## Reproduction Assessment

Yes for the configuration gap. Current-main source inspection shows hardcoded notify defaults and no accepted agent-defaults task notify key; this is source-reproducible feature work rather than a failing bug reproduction.

## Solution Assessment

Unclear because there is no patch and the public config API still needs maintainer direction. The maintainable path is one coherent config contract across task registry, Task Flow, ACP, and subagent creation rather than another runtime-specific knob.
