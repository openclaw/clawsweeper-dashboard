## Summary

Keep this open. Current main still has hardcoded task notify fallbacks and no schema-backed config default, and choosing the public config shape needs maintainer API direction.

## What This Changes

Request a config-level default notification policy for ACP, subagent, and related background task records while preserving existing defaults when unset.

## Best Possible Solution

Add a backwards-compatible `agents.defaults.tasks`-style config contract that resolves the effective notify policy before ACP, subagent, and Task Flow records are created; explicit per-task policies should win and existing defaults should remain when unset.

## Reproduction Assessment

Yes, through source inspection. Current main has no accepted config key in the strict agent-defaults schema/types/docs, and the main creation paths still omit `notifyPolicy`, so deliverable records inherit `done_only`.

## Solution Assessment

Unclear because there is no patch and the API shape is the unresolved part. The maintainable direction is one shared config-backed policy resolution path rather than runtime-specific after-the-fact mutation.
