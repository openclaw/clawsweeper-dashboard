## Summary

Keep open. Current main still hardcodes task notify defaults and exposes no typed, schema-backed, or documented config key for choosing a default policy; the related open PR only addresses a narrower ACP child-session banner symptom.

## What This Changes

Request a config-level default task notification policy so users can make ACP and subagent task completions silent by default while preserving explicit per-task policies.

## Best Possible Solution

Add one backward-compatible agents.defaults.tasks config contract that resolves the effective notify policy before ACP, subagent, and Task Flow records are created, keeps current behavior when unset, lets explicit task or flow policy win, and updates schema, exported types, docs, and regression coverage together.

## Reproduction Assessment

Yes for the configuration gap: current-main source inspection shows fixed task notify defaults and no accepted agent-defaults config key. This is a source-reproducible feature gap rather than a failing bug reproduction.

## Solution Assessment

Unclear because there is no patch and the central decision is the public config API. The maintainable direction is a single coherent config contract across task registry, Task Flow, ACP, and subagent creation rather than a one-path runtime knob.
