## Summary

Keep this issue open. Current main still has no typed, schema-backed, or documented config-level default for task notify policy, and the central ACP, subagent, and Task Flow creation paths still fall back to `done_only` unless a caller supplies an explicit policy.

## What This Changes

Add a user configuration surface for the default background-task notify policy, potentially globally or per runtime, while preserving current defaults when unset.

## Best Possible Solution

Add a backwards-compatible `agents.defaults.tasks`-style config contract that resolves the effective policy before task and flow records are created, with explicit per-task policy still taking precedence.

## Reproduction Assessment

Yes for the configuration gap. Source inspection on current main shows no accepted config key in the strict agent-defaults schema/types/docs, while ACP, subagent, and Task Flow creation still default deliverable records to `done_only`.

## Solution Assessment

Unclear because there is no patch and the unresolved part is the public API shape. The maintainable direction is one coherent config contract across task registry, ACP, subagent, and Task Flow creation rather than separate runtime-specific ad hoc knobs.
