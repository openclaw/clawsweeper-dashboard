## Summary

Keep this issue open: current main and the latest release still remove the session row and archive transcripts without deleting trajectory sidecars, and the only direct fix is the still-open, dirty linked PR.

## What This Changes

Deleting a dashboard or `sessions.delete` session should remove the session transcript plus its `.trajectory.jsonl` and `.trajectory-path.json` artifacts.

## Best Possible Solution

Land or replace the linked delete-path fix so session deletion removes unreferenced trajectory runtime and pointer artifacts with regression coverage and before-after filesystem proof.

## Reproduction Assessment

Yes, at source level: create a session with transcript and trajectory sidecars, call `sessions.delete`, and current main deletes the store row plus transcript archive without calling trajectory cleanup. I did not run a live filesystem repro because this review is read-only.

## Solution Assessment

Yes for issue routing: the best fix is to reuse the existing store-dir-restricted trajectory cleanup helper from the delete lifecycle path while preserving still-referenced session IDs. The open linked PR is the implementation candidate, but it must be refreshed and made merge-ready before this issue can close.
