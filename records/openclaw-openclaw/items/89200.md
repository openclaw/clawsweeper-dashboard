## Summary

Keep this issue open: current main and v2026.6.8 still delete the session row and archive the transcript without removing trajectory sidecars, and the direct fix is still an open conflicting PR.

## What This Changes

Deleting a session through the dashboard or `sessions.delete` should also remove that session's `.trajectory.jsonl` and `.trajectory-path.json` files.

## Best Possible Solution

Land or replace the linked delete-path fix so `sessions.delete` removes unreferenced trajectory runtime and pointer artifacts with regression coverage and redacted filesystem or dashboard proof.

Yes, source-level reproduction is high confidence: create a session with transcript and trajectory sidecars, call `sessions.delete`, and current main removes the row and archives the transcript without invoking trajectory cleanup. I did not run a live filesystem repro because this review is read-only.
