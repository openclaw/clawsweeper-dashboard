## Summary

Keep this issue open: current main and v2026.6.9 still delete the session row and archive transcripts without invoking trajectory sidecar cleanup, while the direct closing PR remains open and unmerged.

## What This Changes

Deleting a session via `sessions.delete` or the dashboard should remove the deleted session's trajectory runtime and pointer sidecars instead of leaving orphan files behind.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/89209 with lifecycle-bound cleanup that removes unreferenced trajectory sidecars on delete while preserving documented transcript-retention semantics.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to create a non-main session with transcript, trajectory runtime, and trajectory pointer files, then call `sessions.delete`; current main removes the store row and archives transcripts without calling trajectory cleanup.

## Solution Assessment

Yes for issue routing. The narrow maintainable solution is to reuse the referenced-session-aware trajectory cleanup at the delete lifecycle boundary; the open linked PR is the current candidate but must land or be replaced before this issue closes.
