Keep this issue open because current main and the latest release still leave the delete path without trajectory sidecar cleanup, while the linked implementation PR at https://github.com/openclaw/openclaw/pull/89209 is still open and not mergeable.

## What This Changes

Deleting a session via `sessions.delete` or the dashboard should remove the transcript and associated trajectory runtime/pointer files instead of leaving unreferenced sidecars.

## Best Possible Solution

Finish, land, or replace the linked delete-path fix so `sessions.delete` reuses the existing trajectory cleanup helper while preserving still-referenced session IDs and lifecycle/session-change behavior.

Yes, at source level: create or select a session with transcript plus trajectory sidecars, call the dashboard/`sessions.delete` path, and current main archives only the primary transcript. I did not run a live filesystem repro because this review is read-only.
