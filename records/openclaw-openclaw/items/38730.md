## Summary

Keep open. Current main has meaningful cron run-session retention cleanup, but the high-frequency `every` schedule advisory requested by the issue is still absent from the main creation and update paths, and open PR #38731 is a same-author implementation candidate with closing syntax for this issue.

## What This Changes

The issue requests guardrails for cron session accumulation, including cron run-session TTL cleanup, session/context pressure warnings, high-frequency schedule warnings, and stronger cleanup behavior.

## Best Possible Solution

Land or replace #38731 with a shared, non-blocking advisory for high-frequency `every` schedules that is applied consistently across CLI add/edit, Gateway add/update, Control UI add/edit, and agent-tool-created cron jobs, while preserving valid short intervals and the existing cron retention cleanup.
