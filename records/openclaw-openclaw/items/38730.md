## Summary

Keep open: current main includes cron run-session retention cleanup, but the requested high-frequency `every` schedule warning is still absent from the checked creation/update paths, and open replacement PR #76473 is the active closing implementation candidate.

## What This Changes

Request guardrails for high-frequency cron schedules, context/session pressure warnings, and cleanup of accumulated cron run sessions.

## Best Possible Solution

Keep the shipped cron run-session pruning and add one shared, non-blocking advisory for short `every` schedules across CLI, Gateway, Control UI, and agent-tool creation/update paths; split broader context-usage diagnostics into a separate follow-up if maintainers still want them.

## Solution Assessment
