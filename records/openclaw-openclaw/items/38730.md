## Summary

Keep open: current main already has cron run-session retention and reaper cleanup, but the requested high-frequency schedule advisory is still absent in current cron creation/update paths, and open replacement PR #76473 uses closing syntax for this issue.

## What This Changes

Requests guardrails for high-frequency cron jobs, including run-session cleanup, context/session warnings, and schedule-frequency warnings to prevent silent agent non-response.

## Best Possible Solution

Keep the shipped cron run-session retention cleanup and land or replace #76473 with one shared, non-blocking advisory for short `every` schedules; handle broader context-threshold diagnostics separately if maintainers still want them.

## Reproduction Assessment

## Solution Assessment

Yes for the current direction: keeping the issue open while #76473 is reviewed is the narrowest maintainer path. Closing before that PR lands would lose the remaining warning work that current main still lacks.
