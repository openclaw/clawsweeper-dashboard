## Summary

Keep open. Current main fixes the simplest bare `/new` transcript-marker case and skips blank visible prompts, but it still intentionally submits runtime-only turns with an empty visible prompt, and the Anthropic Messages transport still builds `messages: []` when converted replay content is empty.

## What This Changes

The issue reports that `/new` or `/reset` can race with active-memory/runtime context and submit an empty prompt plus empty message array to Anthropic-compatible providers.

## Best Possible Solution

Land or adapt a provider-boundary fallback for Anthropic-compatible empty message arrays, with runner coverage proving runtime-only reset/startup context reaches strict providers without becoming visible user transcript text.

## Reproduction Assessment

Yes. A source-level reproduction remains: runtime context plus a blank transcript prompt becomes `runtimeOnly` with `prompt: ""`, bypasses the skip guard, and can reach Anthropic transport as an empty converted `messages` array.

## Solution Assessment

No for the local workaround as written. Injecting a visible greeting changes transcript semantics; the narrower maintainable fix is a transport/runner boundary guard plus regression coverage.
