## Summary

Close as implemented on current main. The bare reset path now records a non-empty session marker, blank visible prompts are skipped before provider submission, and runtime-only context turns submit a non-empty runtime-event marker instead of an empty Anthropic/MiniMax message payload; the proof SHA is on main after the latest release tag.

## What This Changes

A bare `/new` or `/reset` with active-memory races should not dispatch an empty prompt/messages payload to Anthropic-compatible providers or leave the session stuck.

## Best Possible Solution

Keep the current main implementation: use neutral internal markers for reset/runtime-only turns and skip truly blank visible submissions rather than injecting user-visible greeting text into transcripts.

## Reproduction Assessment

Yes. The source-level reproduction is covered by current tests for bare `/new`, blank replay-history prompts, and runtime-only context turns; those paths no longer produce an empty provider submission on main.

## Solution Assessment

Yes. The current approach is narrower than the local workaround because it preserves transcript semantics while preventing empty dispatch at the embedded-runner boundary.
