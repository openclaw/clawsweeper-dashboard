## Summary

Keep open. Current main fixed the bare `/new` transcript marker and blank visible-prompt cases, and Codex Responses already has a system-only fallback, but the Anthropic/MiniMax path on main can still receive an empty `messages` payload from runtime-only turns; related PR #74731 is promising but remains unmerged and explicitly scopes itself to #74589.

## What This Changes

The issue asks OpenClaw to prevent `/new` or `/reset` races from submitting empty prompt/messages payloads to strict providers such as Anthropic Messages and MiniMax.

## Best Possible Solution

Land or adapt a provider-boundary fallback for Anthropic-compatible empty message arrays, then add an embedded-runner regression proving runtime-only reset/startup context reaches strict providers without persisting internal context as a visible user turn.

## Reproduction Assessment

Yes. A source-level reproduction is available: non-empty runtime context plus blank transcript prompt compiles to `prompt: ""`, bypasses the runtime-only skip guard, and is submitted while Anthropic params on main can still contain an empty converted message list.

## Solution Assessment

No for the local workaround as written; injecting a visible greeting at the runner risks transcript semantics. A narrow provider fallback like #74731, paired with runtime-only embedded-runner coverage, is the safer direction.
