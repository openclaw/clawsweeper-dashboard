## Summary

Keep open. Current main has fixed the narrower bare `/new` transcript-marker path and blank visible-prompt guard, but it still deliberately allows runtime-only hidden-context turns to submit an empty visible prompt; with empty replay history, strict Anthropic-compatible transports can still build `messages: []`.

## What This Changes

The issue asks OpenClaw to prevent `/new` or `/reset` races with active-memory/runtime-only context from dispatching empty prompt and empty message payloads to strict provider APIs.

## Best Possible Solution

Define a provider-safe runtime-only input contract at the embedded runner or provider-adapter boundary: hidden runtime context should remain out of persisted visible user turns, but every strict provider request must contain non-empty model-visible input. Cover bare session reset, active-memory memory flush, context-engine runtime-only events, and Anthropic/MiniMax-compatible transports with focused regression tests.

## Reproduction Assessment

Yes. A source-level reproduction is clear: pass a non-empty `effectivePrompt` with `transcriptPrompt: ""`, no replay messages, and no images; current helpers produce `runtimeOnly: true`, bypass the empty-submission guard, and submit `activeSession.prompt("")`.

## Solution Assessment

No. Current main is only a partial fix because it skips blank non-runtime prompts while deliberately allowing runtime-only empty prompts; the local workaround's broad visible greeting injection is less maintainable than a contract-level sentinel, skip, or adapter fallback chosen by maintainers.
