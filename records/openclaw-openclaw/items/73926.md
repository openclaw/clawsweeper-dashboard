## Summary

Keep open. Current main fixed the narrower bare `/new` transcript-marker path and skips blank non-runtime visible prompts, but it still preserves a runtime-only hidden-context path that can submit `prompt: ""` with an empty replay list to strict Anthropic-compatible transports.

## What This Changes

The issue asks OpenClaw to prevent `/new` or `/reset` races with active-memory/runtime-only context from sending empty prompt and empty messages payloads to strict provider APIs.

## Best Possible Solution

Define a provider-safe runtime-only input contract at the embedded runner or provider-adapter boundary: hidden runtime context should stay out of persisted visible user turns, but every strict provider request must contain non-empty model-visible input. Cover bare session reset, active-memory memory flush, context-engine runtime-only events, and Anthropic/MiniMax-compatible transports with focused regression tests.

## Reproduction Assessment

Yes. At source level, a non-empty `effectivePrompt` with blank `transcriptPrompt` becomes `prompt: ""` plus `runtimeOnly: true`, bypasses the empty-submission guard, and is submitted to the session prompt path; existing tests assert that shape.

## Solution Assessment

No. Current main is only a partial fix because it skips blank non-runtime prompts but deliberately allows runtime-only empty prompts; a provider-visible sentinel or adapter-level guard that preserves hidden transcript semantics is safer than the local workaround's broad visible greeting injection.
