## Summary

Keep open. Current main has fixed the simple bare `/new` transcript marker path, but it still allows runtime-only hidden context to produce an empty visible prompt, bypass the empty-submission guard, and reach Anthropic-compatible transports with no model-visible messages.

## What This Changes

Reports that `/new` or `/reset`, especially during active-memory rebuild races, can dispatch an embedded agent run with no model-visible prompt or messages to Anthropic-compatible providers.

## Best Possible Solution

Define a provider-safe runtime-only input contract at the embedded runner or transport boundary: hidden runtime context should stay out of visible user turns, while provider calls must never contain an empty model-visible message set. Cover active-memory startup, memory flush, context-engine runtime context, and Anthropic/MiniMax-compatible transports with focused regressions.

## Reproduction Assessment

Yes. At source level, a non-empty runtime prompt with blank `transcriptPrompt` becomes `prompt: ""` plus `runtimeOnly: true`, the guard allows it, and current tests assert the empty runtime-only submission shape.

## Solution Assessment

No. Current main is a partial fix: the bare `/new` transcript marker is covered, but runtime-only hidden context still needs a provider-safe non-empty input contract; a broad visible greeting injection would be riskier because it can change transcript semantics.
