## Summary

Keep open. Current main covers the simplest bare `/new`/`/reset` and blank visible-prompt cases, but the source still allows runtime-only turns with empty visible prompt/history to reach Anthropic-compatible dispatch; the related provider-boundary fix remains unmerged in the provided context.

## What This Changes

The issue requests preventing `/new`/`/reset` and active-memory reset races from dispatching empty prompt/messages to strict Anthropic-compatible providers.

## Best Possible Solution

Add a provider-boundary fallback or structured no-op for Anthropic-compatible empty message arrays, with regression coverage that preserves hidden runtime context semantics.

## Reproduction Assessment

Yes. A source-level reproduction remains: blank transcript prompt plus non-empty runtime context produces a runtime-only empty visible prompt, bypasses the skip guard, and can reach Anthropic transport with an empty converted message list.

## Solution Assessment

No for the local workaround as written; injecting a visible greeting at the runner entry risks transcript semantics. A narrow Anthropic-compatible transport fallback, paired with runner coverage for runtime-only context, is the safer maintainable path.
