## Summary

Keep open. Current main still matches the report: Mattermost docs advertise `channels.mattermost.streaming`, but the Mattermost schema/types do not accept that field and the inbound monitor still creates and updates the draft preview unconditionally while PR #73252 remains open.

## What This Changes

Mattermost should honor the documented `channels.mattermost.streaming` opt-out for draft preview streaming instead of unconditionally creating and editing preview posts.

## Best Possible Solution

Land or replace PR #73252 with a Mattermost-local implementation of the existing canonical `streaming.mode` contract at channel and account scope, preserving the current default while disabling draft callbacks and preview finalization when mode is `off`.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: configure the documented `channels.mattermost.streaming` value and current strict Mattermost schema/types do not accept it, while the monitor still unconditionally creates and updates a draft preview.

## Solution Assessment

Yes. Using the existing shared `streaming.mode` contract is the narrowest maintainable fix; adding a separate Mattermost-only `draftPreview` boolean would create a second public model for the same behavior.
