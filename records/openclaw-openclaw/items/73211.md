## Summary

Keep open. Current main still has the Mattermost preview-streaming mismatch, and open PR #73252 is already the direct closing implementation candidate, so the issue should stay open until that PR or an equivalent fix lands.

## What This Changes

Mattermost should let users opt out of draft-preview streaming and keep docs, schema, account resolution, and runtime delivery behavior aligned.

## Best Possible Solution

Land PR #73252 or an equivalent Mattermost-local fix that accepts canonical streaming.mode at channel and account scope, preserves the current default, and disables draft preview callbacks plus in-place finalization when mode is off.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: configure channels.mattermost.streaming, then inspect the strict schema/type that do not accept it and the monitor path that unconditionally creates and updates the preview stream.

## Solution Assessment

Yes. Using the existing shared streaming.mode contract is the narrowest maintainable fix; a Mattermost-only draftPreview boolean would create a second public model for the same behavior.
