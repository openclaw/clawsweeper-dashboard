## Summary

Keep open. Current main still has the Mattermost preview-streaming config mismatch: docs advertise `channels.mattermost.streaming`, but the strict Mattermost schema/runtime type do not accept it and the inbound monitor still creates and updates the draft preview unconditionally; open PR #73252 is the direct closing implementation candidate.

## What This Changes

Request to let users opt out of Mattermost draft-preview streaming through config and align the schema, docs, and runtime behavior.

## Best Possible Solution

Land PR #73252 or an equivalent Mattermost-local change that accepts canonical `streaming.mode` at channel/account scope and skips preview creation, update callbacks, and preview finalization when mode is `off`.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: configure `channels.mattermost.streaming`, then inspect the strict schema/type that do not accept it and the monitor path that unconditionally creates and updates the draft preview.

## Solution Assessment

Yes. The existing shared `streaming.mode` contract is the right public surface; adding a new Mattermost-only `draftPreview` boolean would split the channel streaming model.
