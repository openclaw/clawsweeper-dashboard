## Summary

Keep open. Current main still has the Mattermost preview-streaming config mismatch: docs advertise `channels.mattermost.streaming` and `off`, but the strict Mattermost schema/generated metadata do not expose that key and the inbound runtime still creates and updates the draft-preview path without a mode gate. Open PR #73252 is the active closing implementation path, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

Request a Mattermost configuration path that lets users opt out of draft-preview streaming instead of always creating and editing a placeholder post in place.

## Best Possible Solution

Land PR #73252 or an equivalent Mattermost-local fix that accepts canonical `streaming.mode` at channel and account scope, preserves today’s preview default when omitted, disables draft-preview callbacks and in-place finalization when mode is `off`, and keeps tests/docs/generated metadata aligned.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: current main’s strict schema/generated metadata do not accept Mattermost `streaming`, while the inbound monitor unconditionally creates the draft stream and wires partial/tool/reasoning updates into it.

## Solution Assessment

Yes. Honoring the existing canonical channel `streaming.mode` contract is the narrowest maintainable solution; adding a Mattermost-only `draftPreview` boolean would create a second public config shape for the same behavior.
