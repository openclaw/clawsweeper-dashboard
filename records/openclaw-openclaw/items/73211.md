## Summary

Keep open. Current main still has the reported Mattermost mismatch: docs describe `channels.mattermost.streaming`, but the strict schema/type/account resolver do not expose a `streaming` object and the inbound monitor still creates and updates the draft-preview path unconditionally. Open PR #73252 uses closing syntax for the same bug, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue requests a Mattermost configuration switch to disable draft-preview streaming instead of hardcoded preview POST/PUT updates.

## Best Possible Solution

Land PR #73252 or an equivalent Mattermost-local fix that accepts channel/account `streaming.mode`, resolves preview mode through the shared helper, skips draft updates/finalization when `off`, and keeps tests, docs, changelog, and generated config metadata aligned.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: configure `channels.mattermost.streaming`, then inspect the strict schema/type resolver that does not accept or resolve it and the monitor path that always creates and updates the draft preview.

## Solution Assessment

Yes. Honoring the existing canonical `streaming.mode` contract is the narrowest maintainable fix; adding a Mattermost-only `draftPreview` boolean would create a second public streaming model for the same behavior.
