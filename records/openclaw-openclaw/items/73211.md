## Summary

Keep open. Current main still has the Mattermost draft-preview streaming mismatch: docs describe `channels.mattermost.streaming` and `off` mode, but the Mattermost schema/type surface does not accept the canonical streaming object and the inbound monitor still creates and updates the draft preview path unconditionally. Open PR #73252 uses closing syntax for the same bug, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

Mattermost draft-preview streaming should be configurable instead of hardcoded, matching documented `channels.mattermost.streaming` behavior.

## Best Possible Solution

Land PR #73252 or an equivalent Mattermost-local fix that accepts canonical `streaming.mode` at channel and account scope, preserves the current preview default when omitted, disables draft-preview callbacks and in-place finalization when mode is `off`, and keeps tests, docs, changelog, and generated config metadata aligned.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: configure `channels.mattermost.streaming.mode = "off"`, observe the strict Mattermost schema/type surface does not accept that key, then inspect the inbound monitor path that still creates and updates the draft preview unconditionally.

## Solution Assessment

Yes. Honoring the existing shared `streaming.mode` contract is the narrowest maintainable fix; adding a Mattermost-only `draftPreview` boolean would diverge from the documented channel streaming model.
