## Summary

Keep open. Current main still has the Mattermost preview-streaming config mismatch, and open PR #73252 is the active closing implementation path, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue asks for Mattermost draft-preview streaming to be configurable so users can opt out instead of having the behavior hardcoded by release.

## Best Possible Solution

Mattermost should accept the canonical channel streaming config at channel and account scope, preserve draft preview as the default when omitted, disable draft-preview creation/callbacks/finalization when preview mode is `off`, and keep docs, generated config metadata, types, and regression tests aligned.

## Reproduction Assessment

Yes. Current main has a high-confidence source-level reproduction: the strict Mattermost schema/generated metadata do not accept `streaming`, while the inbound monitor still creates and updates the draft preview path without a preview-mode gate.

## Solution Assessment

Yes, with one config-shape caveat. The narrow maintainable fix is to honor the existing `streaming.mode` channel contract rather than add a Mattermost-only `draftPreview` boolean for the same mode decision.
