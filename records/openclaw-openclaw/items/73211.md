## Summary

Keep open. Current main still has the reported Mattermost preview-streaming mismatch: docs now describe `channels.mattermost.streaming`, but the strict Mattermost schema/runtime type still do not expose it, and the inbound monitor still creates and updates the draft-preview path unconditionally. The supplied timeline also shows open PR #73252 uses closing syntax for the same bug, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue asks for Mattermost draft-preview streaming to be configurable so users can opt out instead of having the feature hardcoded by version.

## Best Possible Solution

Land PR #73252 or an equivalent Mattermost-local fix that accepts the canonical `streaming.mode` shape at channel and account scope, defaults to current preview behavior when omitted, disables draft-preview callbacks and in-place finalization when mode is `off`, and keeps tests, docs, changelog, and generated config metadata aligned.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: add `channels.mattermost.streaming` to config and inspect the strict schema/type surface that rejects or drops it, then inspect the inbound monitor path that unconditionally creates and updates the draft preview.

## Solution Assessment

Yes. Honoring the existing shared `streaming.mode` contract is the narrowest maintainable solution; adding a Mattermost-only `draftPreview` boolean would create a second public streaming model for the same behavior.
