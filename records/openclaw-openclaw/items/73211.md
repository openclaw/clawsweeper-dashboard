## Summary

Keep open. Current main still has the reported Mattermost preview-streaming mismatch: docs advertise `channels.mattermost.streaming`, but the strict Mattermost schema/runtime type do not accept it and the inbound monitor still creates the draft-preview stream unconditionally; open PR #73252 is the direct closing implementation candidate.

## What This Changes

Users want Mattermost draft-preview streaming to be configurable so `streaming.mode: "off"` disables the placeholder-post and in-place-update path.

## Best Possible Solution

Land PR #73252 or an equivalent Mattermost-local fix that accepts canonical `streaming.mode` at channel and account scope, preserves the current default, and disables preview callbacks plus in-place finalization when mode is `off`.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: add `channels.mattermost.streaming` to config, then inspect the strict schema/type that do not accept it and the monitor path that unconditionally creates and updates the draft preview.

## Solution Assessment

Yes. Honoring the existing shared `streaming.mode` contract is the narrowest maintainable solution; adding a Mattermost-only `draftPreview` boolean would create a second public streaming model for the same behavior.
