## Summary

Keep open. Current main still has the Mattermost draft-preview streaming mismatch: public docs describe configurable `channels.mattermost.streaming`, but the strict Mattermost schema/types/generated metadata do not expose that key and the inbound runtime still creates/updates the draft preview without a preview-mode gate. Open PR #73252 uses closing syntax and is the active implementation path, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

Users request a Mattermost configuration path to opt out of draft-preview streaming instead of hardcoded preview behavior.

## Best Possible Solution

Land PR #73252 or an equivalent Mattermost-local fix that accepts the canonical `streaming.mode` shape at channel and account scope, preserves today’s preview default when omitted, disables draft-preview callbacks and in-place finalization when mode is `off`, and keeps tests, docs, changelog, and generated config metadata aligned.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: the strict Mattermost schema/generated metadata reject `streaming`, while the monitor unconditionally creates and updates the draft preview stream.

## Solution Assessment

Yes for the active direction. Honoring the existing channel `streaming.mode` contract is narrower and more maintainable than adding a Mattermost-only `draftPreview` boolean for the same mode decision.
