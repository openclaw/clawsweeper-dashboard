## Summary

Keep open. Current main still has the Mattermost draft-preview streaming mismatch: docs advertise `channels.mattermost.streaming` including `off`, but the strict Mattermost schema/types/generated metadata do not expose that config and the inbound runtime still creates and updates a draft preview without a streaming-mode gate. Open PR #73252 uses closing syntax and is the active implementation path, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue requests configurable Mattermost draft-preview streaming so users can opt out and have schema, runtime, generated metadata, and docs agree.

## Best Possible Solution

Land PR #73252 or an equivalent Mattermost-local fix that accepts canonical `streaming.mode` at channel and account scope, preserves today’s preview default when omitted, disables draft-preview callbacks and in-place finalization when mode is `off`, and updates tests, docs, changelog, and generated config metadata together.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: the strict Mattermost schema/generated metadata reject or omit `streaming`, while the monitor unconditionally creates and updates the draft preview stream.

## Solution Assessment

Yes. Honoring the existing channel `streaming.mode` contract is narrower and more maintainable than adding a Mattermost-only `draftPreview` boolean for the same mode decision.
