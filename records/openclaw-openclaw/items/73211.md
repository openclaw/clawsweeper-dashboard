## Summary

Keep open. Current main still has the Mattermost draft-preview streaming mismatch: the strict Mattermost schema and types do not accept the documented `streaming` control, while inbound replies still create and update a draft preview without checking preview-stream mode. Open PR #73252 is the linked implementation path, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

Mattermost draft-preview streaming should be configurable through the documented channel streaming mode so users can opt out instead of having preview streaming hardcoded by version.

## Best Possible Solution

Land PR #73252 or an equivalent Mattermost-local fix that accepts the canonical `streaming.mode` shape at channel and account scope, preserves the current preview default when omitted, disables draft-preview callbacks and in-place finalization when mode is `off`, and keeps tests, docs, changelog, and generated config metadata aligned.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: validate Mattermost config containing `streaming` against the strict schema, then inspect the inbound monitor path that unconditionally creates and updates the draft preview.

## Solution Assessment

Yes. Honoring the existing shared channel `streaming.mode` contract is the narrowest maintainable direction; adding a Mattermost-only `draftPreview` boolean would diverge from the documented streaming model.
