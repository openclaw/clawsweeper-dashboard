## Summary

Keep open. Current main still has the Mattermost preview-streaming mismatch: docs advertise `channels.mattermost.streaming` and `off`, while the strict Mattermost config schema/generated metadata do not expose that key and the inbound runtime still creates and updates the draft preview without a preview-mode gate. Open PR #73252 uses closing syntax for this issue, so the issue should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue requests a Mattermost config/runtime fix so users can disable draft-preview streaming through the documented `channels.mattermost.streaming` contract.

## Best Possible Solution

Land PR #73252 or an equivalent Mattermost-local fix that accepts the canonical `streaming.mode` shape at channel and account scope, preserves the current preview default when omitted, disables draft-preview callbacks and in-place finalization when mode is `off`, and keeps tests, docs, changelog, and generated config metadata aligned.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: parse or validate Mattermost config containing `streaming` against the strict schema/generated metadata and inspect the inbound monitor path that unconditionally creates and updates the draft preview.

## Solution Assessment

Yes. Honoring the existing channel `streaming.mode` contract is the narrowest maintainable fix; adding a Mattermost-only `draftPreview` boolean would diverge from the documented shared streaming model.
