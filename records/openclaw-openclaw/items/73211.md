## Summary

Keep open. Current main still has the Mattermost draft-preview streaming mismatch: the docs advertise `channels.mattermost.streaming` and `off`, but the strict Mattermost config schema/type surface does not accept that key and the inbound monitor still creates and updates the draft preview without a preview-mode gate. Open PR #73252 uses closing syntax for this issue, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

Mattermost draft-preview streaming should have a supported opt-out/configuration path instead of being hardcoded across versions.

## Best Possible Solution

Land PR #73252 or an equivalent Mattermost-local fix that accepts the canonical `streaming.mode` shape at channel and account scope, preserves the current preview default when omitted, disables draft-preview callbacks and in-place finalization when mode is `off`, and keeps tests, docs, changelog, and config metadata aligned.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: compare the documented `channels.mattermost.streaming` examples with the strict Mattermost schema/type surface, then inspect the monitor path that unconditionally creates and updates the draft stream.

## Solution Assessment

Yes. Honoring the existing shared `streaming.mode` contract is the narrowest maintainable fix; adding a Mattermost-only `draftPreview` boolean would diverge from the documented shared channel streaming model.
