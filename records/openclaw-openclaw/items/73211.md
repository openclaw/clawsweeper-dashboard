## Summary

Keep open. Current main still has the Mattermost mismatch: docs advertise `channels.mattermost.streaming`, the strict Mattermost schema/types do not accept it, and the inbound monitor still creates and updates the draft preview unconditionally; open PR #73252 remains the direct unmerged implementation candidate.

## What This Changes

Allow Mattermost draft-preview streaming to be controlled by the documented channel streaming configuration instead of being hardcoded on.

## Best Possible Solution

Land a Mattermost-local fix that accepts the existing canonical `streaming.mode` contract at channel and account scope, preserves today’s default, and gates draft callbacks plus preview finalization when mode is `off`.

## Reproduction Assessment

Yes. Source-level reproduction is high-confidence: configure `channels.mattermost.streaming` as documented, then the strict Mattermost schema/type reject or omit that field while the monitor still creates and updates the draft preview path.

## Solution Assessment

Yes. Using the existing shared `streaming.mode` contract is the narrowest maintainable solution; adding a separate Mattermost-only `draftPreview` boolean would create a second public model for the same behavior.
