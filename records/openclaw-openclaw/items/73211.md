## Summary

Keep open. Current main still has the Mattermost preview-streaming mismatch: the strict Mattermost schema and type surface do not expose `streaming`, while the inbound monitor still creates and updates the draft preview path unconditionally; open PR #73252 uses closing syntax for the same bug, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

Users want Mattermost draft-preview streaming to be configurable and honored by schema, docs, and runtime instead of being hardcoded on or unavailable across versions.

## Best Possible Solution

Land PR #73252 or an equivalent Mattermost-local fix that accepts the canonical `streaming.mode` shape at channel and account scope, preserves the current preview default when omitted, disables draft-preview callbacks and in-place finalization when mode is `off`, and keeps tests, docs, changelog, and generated config metadata aligned.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists on current main: put `channels.mattermost.streaming` in config and inspect the strict schema/type surface that lacks the key, then inspect the inbound monitor path that unconditionally creates and updates the draft preview.

## Solution Assessment

Yes. Honoring the existing channel `streaming.mode` contract is the narrowest maintainable fix; adding a Mattermost-only `draftPreview` boolean would diverge from the shared documented streaming model.
