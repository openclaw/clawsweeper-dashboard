## Summary

Current main and shipped releases now implement the Mattermost draft-preview opt-out: the documented `channels.mattermost.streaming` config is accepted, resolved through the shared channel streaming contract, and `streaming: "off"` disables the draft preview path. The still-open implementation PR is now superseded by the shipped direct fix.

## What This Changes

The issue asks for Mattermost draft-preview streaming to be configurable instead of hardcoded, especially allowing users to disable the placeholder-post plus in-place edit behavior.

## Best Possible Solution

Keep the shipped canonical `channels.mattermost.streaming` implementation; broader block-streaming or channel A2A concerns should remain on the separate canonical follow-up at https://github.com/openclaw/openclaw/issues/70253.

## Reproduction Assessment

No. Current main no longer reproduces by source inspection: the schema accepts `channels.mattermost.streaming`, account resolution maps `streaming: "off"` to `streamingMode: "off"`, and the monitor switches to a disabled draft stream in that mode.

## Solution Assessment

Yes. The shipped fix uses the existing canonical channel streaming contract rather than adding a Mattermost-only `draftPreview` boolean, and it includes focused schema/runtime tests plus a changelog entry.
