## Summary

Keep open. Current main still has the Mattermost draft-preview config mismatch: docs advertise `channels.mattermost.streaming` with `off`, but the strict Mattermost schema/types/generated metadata do not expose that key and the inbound runtime still wires draft-preview creation and updates without a streaming-mode gate. Open PR #73252 is the active closing implementation path, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

Allow Mattermost users to opt out of draft-preview streaming via documented config instead of having the schema/runtime force the behavior on.

## Best Possible Solution

Land PR #73252 or an equivalent Mattermost-local fix that accepts the canonical `streaming.mode` config at channel and account scope, preserves today’s preview default when omitted, disables draft-preview callbacks and in-place finalization when mode is `off`, and keeps tests/docs/generated config metadata aligned.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: current main’s strict Mattermost schema and generated metadata do not accept `streaming`, while the inbound monitor unconditionally creates the draft stream and wires partial/tool/reasoning updates into it.

## Solution Assessment

Yes. Honoring the existing channel `streaming.mode` contract is the narrowest maintainable solution; adding a Mattermost-only `draftPreview` boolean would create a second public config shape for the same preview-streaming decision.
