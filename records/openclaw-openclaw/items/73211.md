## Summary

Keep open: current main still has the Mattermost draft-preview streaming config mismatch, and open PR #73252 is the active closing implementation path rather than a reason to close this before merge.

## What This Changes

The issue asks for Mattermost draft-preview streaming to be configurable so users can opt out and the schema, runtime, generated metadata, and docs agree.

## Best Possible Solution

Land PR #73252 or an equivalent Mattermost-local fix that accepts canonical streaming.mode at channel and account scope, preserves current draft-preview default behavior when omitted, disables draft-preview callbacks and in-place finalization when mode is off, and regenerates/tests the config metadata and docs contract.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: current main’s strict Mattermost schema and generated metadata do not expose streaming, while the inbound monitor still creates the draft preview stream without a mode gate.

## Solution Assessment

Yes. Reusing the existing channel streaming.mode contract is the narrowest maintainable direction; adding a Mattermost-only draftPreview boolean would create a second public config shape for the same preview-streaming behavior.
