## Summary

Keep open. Current main still matches the report: Mattermost docs advertise `channels.mattermost.streaming` and `off`, but the Mattermost schema/types do not accept that config and the inbound monitor still creates and updates a draft preview unconditionally; open PR #73252 is the direct closing implementation candidate and is still unmerged.

## What This Changes

Mattermost draft-preview streaming should honor the documented `channels.mattermost.streaming` opt-out instead of being hardcoded on for inbound replies.

## Best Possible Solution

Review and land PR #73252, or an equivalent Mattermost-local fix, so the documented canonical `streaming.mode` contract is accepted at channel/account scope and gates draft-preview callbacks plus finalization.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: configure `channels.mattermost.streaming` as the docs show, and current Mattermost schema/types reject or omit it while the monitor still creates and updates the draft preview path.

## Solution Assessment

Yes. Honoring the existing canonical `streaming.mode` contract is the narrowest maintainable fix; adding a Mattermost-only `draftPreview` boolean would create a second public model for the same behavior.
