## Summary

Close as implemented. Current main accepts the documented Mattermost `channels.mattermost.streaming` config, resolves it to a preview-streaming mode, and uses a disabled draft stream when the mode is `off`; the fix shipped in `v2026.5.7`.

## What This Changes

The issue asks for Mattermost draft-preview streaming to be configurable instead of hardcoded across versions.

## Best Possible Solution

Keep the shipped shared `streaming` contract and handle overlapping stale Mattermost streaming PRs as separate cleanup rather than adding a Mattermost-only `draftPreview` option.

## Reproduction Assessment

No. Current main no longer reproduces the reported source-level path: the schema accepts `channels.mattermost.streaming`, account resolution maps it to `streamingMode`, and `off` uses a disabled draft stream.

## Solution Assessment

Yes. The shipped fix uses the existing shared `streaming` shape and preserves the default `partial` behavior while allowing users to disable draft-preview streaming with `off`.
