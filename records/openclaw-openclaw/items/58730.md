## What This Changes

The issue requests broad safety and lifecycle improvements: OS-level exec sandbox isolation, granular tool permissions, context compaction, session persistence/resume, and a general hooks system.

## Best Possible Solution

Keep this open as a security/product design item, or split it into narrower tracked issues: cross-platform exec isolation/default policy, exec approval posture, and any remaining session-resume UX. Any implementation should fit the existing sandbox backend and exec approval architecture instead of treating a deprecated macOS wrapper as the complete fix.

No, the currently linked implementation path is not the complete solution for this issue. A macOS sandbox-exec wrapper may be useful source work, but the maintainable fix needs a cross-platform policy decision aligned with sandbox backends, host approvals, and plugin hook approval surfaces.
