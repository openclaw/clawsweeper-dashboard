## Summary

Keep open. Current main still forwards positive out-of-range read offsets into the upstream read implementation without EOF recovery, and the open closing PR #75536 has not merged yet.

## What This Changes

The issue requests graceful read-tool behavior when callers pass an offset beyond the file's line count instead of surfacing the upstream EOF error.

## Best Possible Solution

Review and land or replace #75536 with a narrow wrapper-level EOF recovery plus regression tests for explicit-limit, adaptive, newline-terminated, far-past-EOF, and empty-file reads.

## Reproduction Assessment

Yes. A small text file read through `createSandboxedReadTool` or `createOpenClawReadTool` with a positive `offset` greater than the file line count will reach the upstream read tool path that throws on `startLine >= allLines.length`.

## Solution Assessment

Yes. The best fix is a focused OpenClaw wrapper recovery that catches only the upstream EOF-offset error and preserves current sandboxing, path validation, image normalization, and dependency version behavior.
