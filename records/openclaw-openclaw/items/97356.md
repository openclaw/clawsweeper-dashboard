## What This Changes

The PR makes `serializeJsonlLine` throw when root values stringify to `undefined` and adds unit tests for the failing and preserved JSONL serialization cases.

## Best Possible Solution

Land this narrow serializer guard with its regression tests after normal maintainer review and merge gates, while keeping read-side tolerance for already-corrupt transcript lines.

## Reproduction Assessment

Yes, at source level: current main interpolates the raw `JSON.stringify` result into JSONL, Node returns `undefined` for the reported root values, and the reader skips malformed JSONL lines.

## Solution Assessment

Yes. Guarding `serializeJsonlLine` is the narrowest maintainable fix because all JSONL entry write helpers pass through it, and caller-specific checks would duplicate the same invariant.
