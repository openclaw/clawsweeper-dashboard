The PR wraps `MatrixAuthedHttpClient.requestJson` JSON parsing in `try/catch` for successful JSON responses and adds a colocated regression test for malformed homeserver JSON.

## Best Possible Solution

Land the Matrix HTTP client parse guard with its colocated regression test after ordinary maintainer review; no core API, config, or migration change is needed.

## Solution Assessment

Yes. Catching malformed JSON inside `MatrixAuthedHttpClient.requestJson` is the narrow owner-boundary fix; pushing this into each caller would duplicate policy, and changing transport parsing would blur raw/text response handling.
