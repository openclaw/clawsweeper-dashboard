## What This Changes

The PR updates the macOS Voice Wake settings view to show an unsupported-state panel below macOS 26, avoid starting preview/meter work in that state, and align the macOS Voice Wake docs with the support floor and Right Option shortcut.

## Best Possible Solution

Land the bounded settings/docs fix after normal maintainer approval so unsupported macOS shows one clear non-actionable state and does not start Voice Wake preview or meter work.

## Reproduction Assessment

Yes, source-reproducible: on macOS below 26, voiceWakeSupported is false, and current main/v2026.6.5 still render the Voice Wake subcontrols and activate the preview path. The linked macOS 15 report matches that unsupported-state confusion.

## Solution Assessment

Yes, this is the best bounded fix: the settings owner view now hides unsupported controls and guards preview/meter startup while the docs state the existing macOS 26 requirement. A lower-level runtime-only fix would leave the misleading UI in place.
