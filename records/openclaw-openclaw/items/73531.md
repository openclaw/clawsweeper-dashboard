## Summary

Keep open. Current main still builds status task and audit summaries through two independent helper calls, and the issue is already paired with open unmerged PR #73650, so it should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue asks OpenClaw to remove redundant task reconciliation from `openclaw status` by reusing a reconciled task list between registry and audit summary generation.

## Best Possible Solution

Review and land PR #73650 if its per-status-call shared snapshot is correct, or merge an equivalent narrow fix that reconciles tasks once inside `getStatusSummary()` and derives both summaries from that snapshot while preserving existing zero-argument helper behavior for other call sites.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: current main calls both status task helpers in one summary build, and each helper independently calls `reconcileInspectableTasks()` before reaching the clone-and-sort task list path.

## Solution Assessment

No for the suggested module-level process cache as the first fix. A per-status-call shared snapshot is narrower, avoids invalidation and shared-reference hazards, and matches the existing open PR direction.
