## Summary

Keep open. Current main still builds the status task registry summary and task audit summary through two independent helper calls, and each helper still reconciles the inspectable task list internally. The issue is also paired with open PR #73650 using closing syntax, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

Request to remove duplicate `reconcileInspectableTasks()` work from `openclaw status`, preferably by sharing or caching the reconciled task snapshot for the registry and audit summaries.

## Best Possible Solution

Land PR #73650 after review if its per-status-call shared snapshot is correct, or merge an equivalent narrow fix that reconciles inspectable tasks once inside the status summary path while preserving the zero-argument helper behavior for other callers.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: `getStatusSummary()` calls both task summary helpers in one invocation, and each helper independently calls `reconcileInspectableTasks()`, which reaches the clone-and-sort `listTaskRecords()` path.

## Solution Assessment

No for the issue's suggested module-level process cache as the first fix. A per-status-call shared snapshot is narrower, avoids invalidation and shared-reference hazards, and matches the linked PR direction.
