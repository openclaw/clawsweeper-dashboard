## Summary

Keep open. Current main still has the duplicate status-source path, and open PR #73650 already targets this issue with closing syntax, so this issue should remain open until that PR lands, is replaced, or is closed.

## What This Changes

The issue requests eliminating duplicated `reconcileInspectableTasks()` work during `openclaw status` by sharing or caching the reconciled task snapshot.

## Best Possible Solution

Review and land PR #73650, or an equivalent narrow change that reconciles once in the status summary path while preserving zero-argument helper behavior for other callers.

## Reproduction Assessment

Yes. Source inspection on current main shows `getStatusSummary()` calls both task helpers, each helper calls `reconcileInspectableTasks()`, and reconciliation reaches the clone-and-sort `listTaskRecords()` path.

## Solution Assessment

Yes for the linked PR direction, not necessarily for the issue's process-wide cache suggestion. A per-status shared snapshot is the narrower maintainable fix because it removes duplicated work without adding module-level invalidation or shared-reference hazards.
