## What This Changes

The issue requests eliminating redundant task reconciliation during `openclaw status` so registry and audit summaries share one per-invocation reconciled task snapshot.

## Best Possible Solution

Implement the shared-snapshot shape on current main: compute the reconciled task list once per `getStatusSummary()` call and pass it to registry/audit helpers while leaving other zero-argument callers intact.
