## Summary

Keep open. Current main still has the duplicate status task-reconciliation path, and open PR #73650 already references this issue with closing syntax, so this should stay open until that PR lands or is rejected.

## What This Changes

This issue asks to eliminate duplicate task reconciliation in `openclaw status` by sharing or caching the reconciled task list between registry and audit summaries.

## Best Possible Solution

Review and land PR #73650, or an equivalent narrow fix that reconciles once per status summary while preserving existing zero-argument helper behavior for other callers.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: one `getStatusSummary()` invocation calls both task summary helpers, each helper independently calls `reconcileInspectableTasks()`, and reconciliation reaches the full `listTaskRecords()` clone-and-sort path.

## Solution Assessment

Yes for the linked per-status-call shared snapshot direction. It is narrower than a process-lifetime module cache because it avoids invalidation and shared-reference hazards outside the status command.
