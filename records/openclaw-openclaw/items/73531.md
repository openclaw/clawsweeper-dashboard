## Summary

Keep open. Current main still has the duplicate status path, and open PR #73650 already targets this exact bug with closing syntax, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue requests that `openclaw status` avoid reconciling the inspectable task list twice by sharing or caching a reconciled task snapshot across the registry and audit summaries.

## Best Possible Solution

Review and land PR #73650 if its per-status-call shared snapshot is correct, or merge an equivalent narrow fix that reconciles inspectable tasks once in the `getStatusSummary()` path while preserving the zero-argument helper behavior for other callers.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: current `main` calls both task summary helpers in one `getStatusSummary()` invocation, each helper calls `reconcileInspectableTasks()`, and reconciliation reaches the clone-and-sort `listTaskRecords()` path.

## Solution Assessment

No for the issue's suggested module-level process cache as the first fix. A per-status-call shared snapshot, as proposed by PR #73650, is narrower and avoids invalidation and shared-reference hazards while keeping the helper fallback contract intact.
