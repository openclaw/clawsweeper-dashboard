## Summary

Keep open. Current main still performs the duplicate status-task reconciliation path, and open PR #73650 is already linked as the implementation path, so this issue should remain open for review, revision, landing, or closure of that PR.

## What This Changes

The issue asks to remove redundant `openclaw status` task reconciliation by reusing one reconciled task snapshot or cache across the registry and audit summaries.

## Best Possible Solution

Review and land PR #73650 if its per-status-call shared snapshot is correct, or merge an equivalent narrow fix that reconciles tasks once inside `getStatusSummary()` and derives both summaries from that snapshot while preserving zero-argument helper behavior for other call sites.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: current main calls both status task helpers in one summary build, and each helper independently calls `reconcileInspectableTasks()` before reaching the clone-and-sort task list path.

## Solution Assessment
