## Summary

Keep open. Current main still executes duplicate task reconciliation in the status path, and open PR #73650 is already linked with closing syntax for this bug.

## What This Changes

Request to remove duplicate task reconciliation from `openclaw status` so registry and audit summaries reuse one reconciled task snapshot or cache per invocation.

## Best Possible Solution

Review and land PR #73650, or an equivalent narrow fix that shares one reconciled task snapshot inside the status summary path while preserving existing zero-argument helper behavior.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: one `getStatusSummary()` invocation calls both task summary helpers, and each helper independently reaches `reconcileInspectableTasks()` and the clone-and-sort `listTaskRecords()` path.

## Solution Assessment

Yes for the linked per-status-call shared snapshot direction. It is narrower than a module-level process cache because it avoids invalidation complexity and shared-reference hazards outside the status command.
