## Summary

Keep open. Current `main` still takes the duplicate source path, and open PR #73650 already targets the issue with closing syntax, so this should remain open until that PR lands, is replaced, or is closed.

## What This Changes

The issue requests eliminating redundant `reconcileInspectableTasks()` work during `openclaw status` by sharing or caching the reconciled task list for registry and audit summaries.

## Best Possible Solution

Review and land PR #73650, or an equivalent narrow change that reconciles once in `getStatusSummary()` while preserving zero-argument helper behavior for other callers.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: one `getStatusSummary()` call invokes two helpers, and each helper independently reaches `reconcileInspectableTasks()` and the clone-and-sort `listTaskRecords()` path.

## Solution Assessment

No, not as the issue's module-level process cache suggestion. A per-status shared reconciled snapshot is the narrower maintainable fix because it avoids global invalidation and shared-reference hazards while removing the duplicate work in the reported command.
