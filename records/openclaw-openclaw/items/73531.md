## Summary

Keep open. Current main still has the duplicated status task reconciliation path, and the linked implementation PR is open and unmerged, so this issue should remain available for that PR to land or be closed.

## What This Changes

The issue requests eliminating redundant `reconcileInspectableTasks()` work in `openclaw status` by sharing or caching the reconciled task list across registry and audit summaries.

## Best Possible Solution

Review and land PR #73650 if its per-status-call shared snapshot is correct, or merge an equivalent narrow fix that reconciles tasks once inside `getStatusSummary()` while preserving zero-argument helper behavior for other call sites.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: current main calls both status task helpers during one summary build, and each helper independently calls `reconcileInspectableTasks()` before reaching the clone-and-sort task list path.

## Solution Assessment

No for the issue's module-level process cache as the first fix. A per-call shared reconciled snapshot is narrower, avoids invalidation and shared-reference hazards, and matches the open PR direction.
