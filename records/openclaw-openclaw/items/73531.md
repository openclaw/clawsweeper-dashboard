## Summary

Keep open. Current main still takes the duplicate reconciliation path, and open PR #73650 already targets this issue with closing syntax, so the issue should remain open until that PR lands or is replaced.

## What This Changes

The issue requests avoiding duplicate `reconcileInspectableTasks()` work during `openclaw status` by sharing or caching the reconciled task snapshot.

## Best Possible Solution

Review and land PR #73650, or an equivalent narrow change that reconciles once in the status summary path while keeping zero-argument helper behavior for other callers.

## Reproduction Assessment

Yes. A source-level reproduction is high-confidence: one `getStatusSummary()` call invokes two helpers, and each helper independently reaches `reconcileInspectableTasks()` and the clone-and-sort `listTaskRecords()` path.

## Solution Assessment

Not exactly as the issue’s module-level cache suggestion. A per-status shared snapshot is the narrower maintainable fix because it avoids process-wide invalidation and shared-reference hazards while eliminating the duplicate work in the reported command.
