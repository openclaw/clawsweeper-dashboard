## Summary

Keep open. Current main still takes the duplicate status reconciliation path, and open PR #73650 is already the linked implementation candidate for the same bug.

## What This Changes

The issue asks OpenClaw to avoid reconciling the full task registry twice during one `openclaw status` run.

## Best Possible Solution

Review and land PR #73650, or an equivalent narrow fix that reconciles once in the status summary path while preserving zero-argument helper behavior for other callers.

## Reproduction Assessment

Yes. Current main provides a high-confidence source-level reproduction: one `getStatusSummary()` call invokes two helpers, and each helper independently reaches `reconcileInspectableTasks()` and `listTaskRecords()`.

## Solution Assessment

Yes for the linked per-status shared snapshot direction. It is narrower than a module-level process cache because it avoids global invalidation and shared-reference hazards while keeping existing helper callers compatible.
