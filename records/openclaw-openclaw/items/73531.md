## Summary

Keep open. Current main still has the duplicate `openclaw status` task reconciliation path, and open PR #73650 is already the linked implementation candidate with closing syntax.

## What This Changes

Optimize `openclaw status` so registry and audit summaries share one reconciled task snapshot instead of reconciling the task list twice.

## Best Possible Solution

Review and land PR #73650, or an equivalent narrow shared-snapshot change, while preserving zero-argument helper behavior for existing callers.

## Reproduction Assessment

Yes. Current main provides a high-confidence source-level reproduction: one `getStatusSummary()` call invokes two helpers, and each helper independently reaches `reconcileInspectableTasks()` and `listTaskRecords()`.

## Solution Assessment

Yes for the linked shared-snapshot direction. It is narrower than a module-level process cache because it avoids global invalidation and shared-reference hazards while keeping existing helper callers compatible.
