## Summary

Keep open. Current main still has the duplicate status reconciliation path, and open PR #73650 is already the linked implementation candidate for this exact issue.

## What This Changes

The issue requests eliminating duplicate task reconciliation in `openclaw status` when building task registry and audit summaries.

## Best Possible Solution

Review and land PR #73650, or an equivalent narrow change that shares one reconciled snapshot within `getStatusSummary()` while preserving zero-argument helper behavior for other callers.

## Reproduction Assessment

Yes. Current main provides a high-confidence source-level reproduction: one `getStatusSummary()` invocation reaches both helpers, and each helper independently calls `reconcileInspectableTasks()` and `listTaskRecords()`.

## Solution Assessment

Yes. The linked per-status shared snapshot approach is narrower than a module-level process cache because it avoids global invalidation and shared-reference hazards while keeping existing helper callers compatible.
