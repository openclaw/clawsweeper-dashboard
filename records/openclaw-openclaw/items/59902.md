## Summary

Keep open: current main and v2026.6.10 still make the `/cron` History action only load runs without moving the viewport, while the open linked PR is the active closing fix path.

## What This Changes

The issue requests clearer visible feedback when the `/cron` History button updates a run-history panel that may be below the visible viewport.

## Best Possible Solution

Review and land the narrow linked PR so clicking History keeps loading the selected job runs and scrolls the existing Run history card into view.

## Reproduction Assessment

Yes, at source level: current main renders Run history below the jobs list and the History button only calls `onLoadRuns`, so a long `/cron` page can leave the viewport unchanged. I did not run a live browser reproduction in this read-only review.

## Solution Assessment

Yes. Scrolling or focusing the existing Run history card is the narrowest maintainable solution for the reported no-feedback behavior; a modal or drawer would add more UI surface than this issue needs.
