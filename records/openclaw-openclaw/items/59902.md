## Summary

Keep open. Current main and the latest release still make the `/cron` History button select/load the lower in-page Run history card without any scroll, focus, modal, drawer, or other attention shift; the related cron history and cron refactor threads do not cover this UX gap.

## What This Changes

Request that the `/cron` History action visibly focus or open the run-history surface instead of silently updating a lower in-page panel.

## Best Possible Solution

Keep the existing Run history card and add accessible scroll/focus attention on explicit History clicks, with focused UI coverage; avoid a broader modal or drawer redesign unless maintainers choose a new surface.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: render `/cron` with the Run history card below the fold, click a job History button, and current main only changes run scope/loads runs while leaving viewport and focus unchanged. I did not run a browser because this review was read-only.

## Solution Assessment

Yes. Scrolling/focusing the existing Run history card is the narrowest maintainable fix because the data flow and card already exist; a modal or drawer would be a larger UI redesign.
