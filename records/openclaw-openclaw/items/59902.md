## Summary

Keep open. Current main and v2026.5.18 still make the `/cron` History action update the lower in-page Run history card without moving attention to it, and the related larger inline-history PR was closed not planned rather than superseding this narrower UX gap.

## What This Changes

The issue asks for clearer feedback when the `/cron` History button updates an in-page run-history section that may be below the visible viewport.

## Best Possible Solution

Keep the existing Run history card and add accessible scroll/focus attention on explicit History clicks, with focused UI coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: render `/cron` with enough jobs for the Run history card to sit below the fold, click a job History button, and current main only loads job runs without moving viewport or focus.

## Solution Assessment

Yes. Scrolling/focusing the existing Run history card is the narrowest maintainable solution; a modal, drawer, or inline per-job history would be a larger product/UI redesign.
