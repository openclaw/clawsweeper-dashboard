## Summary

## What This Changes

The issue requests clearer visible feedback when clicking a cron job's History button, such as scrolling or focusing the existing Run history section.

## Best Possible Solution

Land a narrow fix that keeps the existing run-loading behavior and visibly brings the existing Run history card into view, with focused UI coverage and browser proof.

## Reproduction Assessment

Yes at source level: current main renders Run history below the jobs list and the History button only calls onLoadRuns, so a long /cron page can leave the visible viewport unchanged. I did not run a live browser reproduction in this read-only review.

## Solution Assessment

Yes: scrolling or focusing the existing Run history card is the narrowest maintainable solution for this report. A modal or drawer would add more UI surface than the issue needs unless maintainers choose a broader redesign.
