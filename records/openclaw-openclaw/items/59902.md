## Summary

Keep open. Current main still makes the `/cron` History action update the lower in-page Run history card without a scroll, focus, modal, drawer, or other attention shift, and I found no duplicate or open fix PR covering this UX gap.

## What This Changes

The issue asks for the `/cron` History button to give clearer feedback, such as scrolling/focusing the Run history section or opening a dedicated history surface.

## Best Possible Solution

Keep the existing cron run-history card and data flow, but make the explicit History action move attention to that card with a stable scroll/focus target and focused UI coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: render `/cron` with the Run history card below the fold, click a job’s History button, and current main only updates the run filter and loads runs while leaving viewport and focus unchanged.

## Solution Assessment

Yes. A scroll/focus attention shift is the narrowest maintainable solution because the Run history surface already exists; a modal or drawer would be a broader Control UI redesign.
