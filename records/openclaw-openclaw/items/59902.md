## Summary

Keep open: current `main` and the latest shipped tag still make the `/cron` History button only load the lower in-page Run history card, with no scroll, focus, drawer, modal, or visible attention transfer.

## What This Changes

The issue requests clearer feedback when clicking the `/cron` History button because the lower run-history panel can update outside the visible viewport.

## Best Possible Solution

Keep the existing Run history card and add an accessible scroll/focus attention shift on explicit History clicks, with focused coverage proving the card is revealed without redesigning the page into a modal or drawer.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: render `/cron` with enough jobs for Run history to be below the fold, click a job History button, and current main only loads runs without moving viewport or focus.

## Solution Assessment

Yes. Scrolling or focusing the existing Run history card is the narrowest maintainable solution; a modal or drawer would be a larger product redesign for the same usability gap.
