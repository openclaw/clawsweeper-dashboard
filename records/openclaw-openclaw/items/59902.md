## Summary

Keep open. Current main and the latest release still make the `/cron` History action update the lower in-page Run history card without moving attention to it, and the related inline-history proposal was closed not planned rather than superseding this narrower UX gap.

## What This Changes

The issue requests clearer feedback when clicking a `/cron` History button, such as scrolling or focusing the existing Run history section instead of silently updating a lower in-page panel.

## Best Possible Solution

Keep the existing Run history card and add accessible scroll/focus attention on explicit History clicks, with focused UI coverage rather than a modal, drawer, or inline-history redesign.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: render `/cron` with enough jobs for the Run history card to sit below the fold, click a job History button, and current main only loads job runs without moving viewport or focus.

## Solution Assessment

Yes. Scrolling/focusing the existing Run history card is the narrowest maintainable solution; the larger inline-history direction was already closed not planned in https://github.com/openclaw/openclaw/pull/22103.
