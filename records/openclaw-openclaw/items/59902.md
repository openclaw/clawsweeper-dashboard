## Summary

Keep open: current main and latest release still make the /cron History action load runs without visibly moving the user to the Run history card, and both linked fix attempts are closed unmerged.

## What This Changes

The issue requests clearer feedback for the /cron History button, such as scrolling or focusing the existing Run history section after the click.

## Best Possible Solution

Add a narrow scroll or focus affordance to the existing Run history card after an explicit History click, with focused UI coverage and visual proof, while preserving the current run-loading and filter behavior.

## Reproduction Assessment

Yes at source level: current main renders Run history lower on the page and the History button only calls onLoadRuns, so a long job list can leave the visible viewport unchanged. I did not run a live browser reproduction in this read-only review.

## Solution Assessment

Yes: a guarded scroll or focus to the existing Run history card is the narrowest maintainable fix. A modal or drawer would add more product surface than this report needs unless maintainers choose a larger UX redesign.
