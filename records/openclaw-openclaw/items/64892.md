## Summary

Keep open. Current main and the latest release still lack an exec-side guard for raw global OpenClaw package-manager self-updates, and this maintainer-labeled issue is already paired with an open, unmerged closing PR.

## What This Changes

The issue asks OpenClaw to prevent agent self-update requests from bypassing `update.run`/`openclaw update` with raw global package-manager installs that leave the gateway running stale code.

## Best Possible Solution

Land or replace the linked fix with a shared exec-control guard that blocks raw global OpenClaw package-manager upgrades, keeps `update.run`/`openclaw update` as the supported self-update path, and includes focused parser/tool tests plus real behavior proof.

## Reproduction Assessment

Yes. The issue provides concrete service logs for raw `npm install -g openclaw@latest` followed by stale chunk failures, and current source shows the exec guard still does not block that command class.

## Solution Assessment

Yes. The maintainable fix belongs in the shared exec-control guard before process launch, while `update.run`/`openclaw update` remains the restart-owning update path.
