Keep open. Current main and v2026.6.10 still lack an exec-side guard for raw global OpenClaw package-manager self-updates, and the maintainer-protected issue is paired with an open, unmerged, currently conflicting closing PR.

## What This Changes

The issue reports that agent self-update requests can use raw global package-manager installs instead of the supported update.run/openclaw update path, leaving a live gateway on stale code.

## Best Possible Solution

Land or replace the linked fix with a shared exec-control guard that blocks raw global OpenClaw package-manager upgrades, keeps update.run/openclaw update as the supported self-update path, and includes focused parser/tool tests plus real behavior proof.
