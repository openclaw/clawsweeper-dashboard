## Label Justifications

- `P1`: The issue reports a real agent/channel workflow break where raw self-update can leave a running gateway unable to lazy-import runtime chunks until restart.
- `impact:message-loss`: The issue and follow-up comment include Telegram message failures and silent hangs after the stale package swap.
- `impact:crash-loop`: The stale gateway process remains alive but degraded with module-load failures, which is a process-availability failure mode.

## Summary

This issue should stay open: current main and v2026.6.6 have safer `update.run` restart/handoff handling, but the exec guard still does not block raw global OpenClaw package-manager upgrades, and an open closing PR is already the concrete fix path. The `maintainer` label and MEMBER author also make this a human-maintainer item rather than an auto-close candidate.

## What This Changes

Ensure explicit agent self-update requests use `update.run` / `openclaw update`, or otherwise guarantee a gateway restart instead of allowing raw global package-manager installs.

## Best Possible Solution

Review and either land/update https://github.com/openclaw/openclaw/pull/78696 by moving the raw OpenClaw package-manager guard into the current exec control-command guard, or choose an equivalent maintainer-approved enforcement point that preserves `update.run` / `openclaw update` as the self-update path.

## Reproduction Assessment

Yes, by source and report evidence: a service-managed global install can be changed with raw `npm install -g openclaw@latest` through exec, then stale runtime chunks fail before a gateway restart. I did not run the live host scenario, but current main's exec guard lacks the raw package-manager block and the issue/comment logs provide concrete failure signatures.

## Solution Assessment

Yes, the best narrow fix is to enforce the boundary at exec command preflight while keeping `update.run` / `openclaw update` as the supported restart-owning path. The open PR targets that behavior, but it likely needs review against the current extracted guard module.
