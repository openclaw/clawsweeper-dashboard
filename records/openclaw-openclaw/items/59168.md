## Summary

Keep open. Current main still lacks provider/name keying or an equivalent selectable model-profile to provider-facing model-id contract; the narrow suffix-stripping follow-up was closed as duplicate to this canonical issue, while the related `routeTo` PR remains open and unmerged.

## What This Changes

Request to decouple OpenClaw's internal/selectable model refs from provider-facing API model IDs, using `provider/name` keys and model-profile reasoning levels.

## Best Possible Solution

Define one backward-compatible model-profile routing contract, then update schema, runtime resolution, params/context budgeting, picker/status surfaces, sessions, fallbacks, docs, and migration together.

## Reproduction Assessment

## Solution Assessment

Unclear. Reusing `name` as the internal key is plausible, but a dedicated `routeTo` or `apiModelId` style contract may be safer and needs maintainer product judgment before implementation.
