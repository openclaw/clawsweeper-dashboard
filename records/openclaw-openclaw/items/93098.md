This PR adds an optional `encoding` argument to the built-in session `read` tool, decodes text buffers through `TextDecoder` or a Buffer fallback, updates `ReadToolInput`, and adds GBK/default UTF-8 tests.

## Best Possible Solution

Pick one canonical GBK read-tool fix: either bless explicit `read.encoding` after removing or implementing `auto`, or land the narrower Windows codepage fallback and close duplicate branches after the chosen fix lands.

## Solution Assessment
