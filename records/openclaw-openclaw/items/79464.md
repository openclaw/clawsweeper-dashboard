## What This Changes

The PR makes elevated provider-key resolution skip system-event providers and fall back to `OriginatingChannel`, then adds a regression test and changelog entry.

## Best Possible Solution

Keep the `OriginatingChannel` fallback, repair the regression test so it checks the call made by this scenario, then rebase if needed and land with focused auto-reply/elevation validation.

## Solution Assessment

No for the submitted patch as a whole: the runtime fallback is the narrow maintainable fix, but the added regression test must assert the specific resolver call instead of accumulated mock history.
