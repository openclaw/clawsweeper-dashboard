## Summary

Keep open. Current main still compares OpenRouter base URLs case-sensitively in the OpenRouter provider, Arcee OpenRouter path, and embedded-runner support; the PR is a narrow comparison-only hardening patch with no blocking review or security findings found.

## What This Changes

The PR lowercases OpenRouter base-URL values before matching in the OpenRouter provider, Arcee OpenRouter path, and embedded-runner test support, and adds OpenRouter hook regression coverage for uppercase legacy URLs, canonical no-op behavior, and custom-proxy passthrough.

## Best Possible Solution

Land this PR or an equivalent narrow patch that lowercases only the OpenRouter base-URL match key in the provider-owned helpers, while preserving canonical return values and leaving non-OpenRouter custom proxies untouched.
