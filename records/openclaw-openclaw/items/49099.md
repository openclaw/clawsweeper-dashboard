## What This Changes

Extract source URL, title, and description metadata from inbound WhatsApp GIF/video media so agents can understand picker GIFs without video processing.

## Best Possible Solution

Implement a focused WhatsApp plugin repair that classifies `gifPlayback` videos as GIFs and forwards `externalAdReply` fields through the existing untrusted structured context path.

## Reproduction Assessment

Yes, at source level. A `videoMessage` currently maps to `<media:video>` regardless of `gifPlayback` or `contextInfo.externalAdReply`, and the pinned Baileys proto exposes those fields.

## Solution Assessment

No for the exact inline-attribute proposal. The better fix is to keep the placeholder compact and route provider-sourced GIF metadata through `untrustedStructuredContext` with prompt-rendering tests.
