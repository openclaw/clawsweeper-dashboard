## Summary

Keep open: this PR is the right fix direction for https://github.com/openclaw/openclaw/issues/80667, but the updated branch can still skip trajectory flushing when a cleanup step rejects, and the submitted proof is still mock-only.

## What This Changes

The PR wires trajectory lifecycle recording into `executePreparedCliRun` and adds focused lifecycle tests for setup-failure handling.

## Best Possible Solution

Land a narrow CLI-runner recorder integration after moving trajectory flush into a guaranteed cleanup path, adding regression coverage for cleanup rejection, and collecting redacted real pure `claude-cli` sidecar proof.

## Reproduction Assessment

Yes, source-reproducible: current main has no recorder calls in `executePreparedCliRun`, while the docs and embedded path establish default runtime trajectory events. I did not run a live Claude CLI session in this read-only review.

## Solution Assessment

No, not as currently proposed. The recorder seam is appropriate, but the cleanup/flush ordering still lets fallible cleanup skip flushing, and the PR still needs real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Flush before fallible cleanup:** `src/agents/cli-runner/execute.ts:818-826`
  - body: The trajectory flush block runs after `claudeSkillsPlugin.cleanup()`, `systemPromptFile.cleanup()`, and `cleanupImages()` in the same `finally`. If any of those awaits rejects, execution leaves the `finally` before `trajectoryRecorder.flush()` runs, so a successful run can still leave the sidecar unflushed even though `session.ended` was recorded. Move the flush into its own protected path before or around fallible cleanups, or wrap each cleanup so recorder flush is always reached.
  - confidence: 0.94
- **[P3] Remove the unnecessary result assertion:** `src/agents/cli-runner/execute.ts:842`
  - body: The current PR head fails `check-lint` because `return runResult as CliOutput` is flagged as an unnecessary assertion. Drop the assertion or adjust the control flow so the lint gate passes before merge.
  - confidence: 0.9
