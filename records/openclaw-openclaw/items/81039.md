## Summary

Keep open: this targets a source-proven trajectory parity bug, but the external PR is missing after-fix real behavior proof and the patch should cover setup-failure paths before it merges.

## What This Changes

The PR adds trajectory runtime recorder calls to `executePreparedCliRun` so pure CLI executor runs emit session, prompt, model, and end events to the session trajectory sidecar.

## Best Possible Solution

Land a narrow CLI-runner recorder integration that covers live-session and spawn paths, records and flushes terminal events for every thrown path, adds focused regression coverage, and includes real `claude-cli` after-fix proof.

## Reproduction Assessment

Yes, from current-main source inspection: the embedded/fallback paths have trajectory recorder calls, while the CLI executor path has none. I did not run a live `claude-cli` session in this read-only review.

## Solution Assessment

No, not as currently proposed. The recorder seam is the right place, but error/end/flush handling should cover setup and cleanup failures, and the PR still needs real behavior proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Wrap setup failures in trajectory cleanup:** `src/agents/cli-runner/execute.ts:394`
  - body: The recorder is created before system-prompt, image, and skills prep, but this `try` only wraps `enqueueCliRun`. If any awaited prep step throws after `session.started` or `prompt.submitted`, the new sidecar can miss `session.ended` and this recorder's flush. Move the error/end/flush handling so it covers the whole prepared CLI run after recorder creation.
