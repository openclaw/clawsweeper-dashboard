## Summary

Keep open: the PR targets a real source-proven CLI trajectory gap, but the current patch still misses terminal recorder cleanup for setup/cleanup failures and the supplied proof exercises the recorder directly rather than the patched CLI executor path.

## What This Changes

The branch imports the trajectory runtime recorder in `src/agents/cli-runner/execute.ts` and records CLI-run lifecycle events around `executePreparedCliRun`.

## Best Possible Solution

Land a narrow CLI-runner recorder integration that covers the whole prepared run lifecycle, includes focused regression coverage, and is backed by redacted real CLI executor proof.

## Reproduction Assessment

Yes, source-reproducible: current main has no recorder calls in `executePreparedCliRun`, while docs and the embedded path establish default runtime trajectory events. I did not run a live claude-cli session in this read-only review.

## Solution Assessment

No, not as currently proposed. The recorder seam is the right fix direction, but terminal error/end/flush handling should cover setup and cleanup failures, and the proof should exercise the patched CLI executor path.
