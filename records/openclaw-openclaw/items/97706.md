The PR switches Discord PluralKit message lookup and voice upload URL success responses from `response.json()` to `readProviderJsonResponse`.

## Best Possible Solution

## Reproduction Assessment

## Solution Assessment

Yes. Reusing the existing plugin SDK provider JSON helper is the narrowest maintainable fix for these two paths; a custom Discord-only reader would duplicate the shared contract.
