## Summary

Keep open. Current main still has absolute-token-only compaction config, and the linked implementation PR is open and unmerged, so the feature still needs maintainer API, runtime, docs, and validation follow-through.

## What This Changes

Add percentage or context-window-relative compaction budgets so `reserveTokensFloor` and `keepRecentTokens` scale across models instead of requiring absolute token counts.

## Best Possible Solution

Land one backward-compatible relative-budget config API via the active PR or a successor, covering types, schema, runtime consumers, docs/help/baselines, and focused tests before closing this issue.

## Solution Assessment
