## Summary

Keep open. Current main still has context-window-aware capping for the numeric reserve floor, but it does not implement percentage string values for compaction token settings in the public config type, Zod schema, generated schema, docs, or Pi runtime resolver. The prior ClawSweeper comment matches the current checkout: this remains a narrow, valid enhancement rather than cleanup-close material.

## What This Changes

Add percentage-string support for context-aware compaction token settings such as `reserveTokensFloor` and `keepRecentTokens`.

## Best Possible Solution

Add a shared typed token-budget value contract that preserves existing numeric inputs, accepts documented percentage strings, resolves them against the effective model context budget in the Pi runtime path, and updates generated schema, docs, tests, and changelog together.

## Reproduction Assessment

Not applicable as a feature request, but there is a high-confidence static verification path. Current main's config type, Zod schema, generated schema, docs, and runtime resolver all accept numbers only for these token settings.

## Solution Assessment

Yes. Explicit percentage-string support is the narrowest maintainable direction because it keeps existing absolute numeric configs stable; silently auto-scaling existing numbers would change established behavior.
