## Summary

Keep open. Current main still exposes the requested compaction token-budget settings as numeric-only in public types, Zod validation, generated schema, docs, and runtime resolution, so percentage strings like `"10%"` remain unimplemented.

## What This Changes

Allow context-aware compaction token settings such as `reserveTokensFloor` and `keepRecentTokens` to be configured as percentages of the active model context window.

## Best Possible Solution

Add a shared token-budget value type and resolver, preserve existing numeric semantics, and resolve percent strings against the effective model context window wherever compaction thresholds are applied.

## Reproduction Assessment

Yes, source-level. Current main validates these compaction knobs as numbers and runtime readers ignore non-number values, so a config using `"10%"` cannot reach context-aware resolution.

## Solution Assessment

Yes. Explicit percent-string support resolved against the effective context window is narrower and safer than auto-scaling existing numeric values, which would change current user behavior.
