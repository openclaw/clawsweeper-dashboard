## Summary

Keep open. Current main still treats the requested compaction token-budget settings as numeric-only in public types, validation, generated schema, docs, and runtime resolution, so percentage strings remain unimplemented and there is no linked open implementation PR in the reviewed context.

## What This Changes

Request support for percentage values such as `reserveTokensFloor: "10%"` and `keepRecentTokens: "15%"` so compaction budgets scale with the active model context window.

## Best Possible Solution

Add a shared token-budget value type and resolver for these compaction fields, preserve existing numeric semantics, update schema/docs/tests/changelog, and resolve percent strings against the effective model context window.

## Reproduction Assessment

Yes for source-level reproduction. Current main validates the knobs as numbers and runtime readers ignore non-number values, so a config using `"10%"` cannot reach context-aware resolution.

## Solution Assessment

Yes. Explicit percent-string support resolved against the effective context window is narrower and safer than auto-scaling existing numeric values, which would change current user behavior.
