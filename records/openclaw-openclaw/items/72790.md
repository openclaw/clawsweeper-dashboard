## Summary

Keep open. Current main and the latest release still expose absolute-token-only compaction budgets, and the remaining implementation path is the open feature PR at https://github.com/openclaw/openclaw/pull/81176 rather than a merged fix.

## What This Changes

Request support for context-window-relative compaction budgets so reserve and keep token settings can be configured proportionally instead of as fixed token counts.

## Best Possible Solution

Land one backward-compatible relative-budget config API, covering schema/types, Pi settings, memory preflight/flush thresholds, docs/help/baselines, and focused tests while preserving existing numeric config semantics.

## Reproduction Assessment

Not applicable: this is a feature request, not a bug report. Source inspection gives high confidence that current main has no percentage or share-based compaction budget config surface yet.

## Solution Assessment

Unclear until maintainers settle the public config API. Additive numeric share fields look safer than overloading existing numeric keys with percent strings, but the final solution needs every compaction threshold consumer and public config doc updated together.
