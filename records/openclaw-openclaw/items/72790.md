## Summary

Keep open. Current main still publishes and resolves the relevant compaction token-budget knobs as numeric-only values, so percentage strings such as `"10%"` remain unimplemented rather than obsolete or already fixed.

## What This Changes

Request support for percentage values in compaction token-budget settings so the same config scales across model context windows.

## Best Possible Solution

Add a shared absolute-or-percentage token-budget value type and resolver for these compaction fields, resolve percentages against the effective model context window, preserve current numeric behavior, and update schema, docs, tests, and changelog together.

## Reproduction Assessment

Yes, source-level. Current main validates these settings as numbers and the runtime readers also require numbers, so a `"10%"` value cannot reach context-aware resolution.

## Solution Assessment

Yes. Explicit percent-string support resolved against the effective context window is narrower and safer than auto-scaling existing numeric values, which would change current user behavior.
