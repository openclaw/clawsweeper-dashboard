## What This Changes

Request support for percentage-string compaction token settings so values can scale with the effective model context window.

## Best Possible Solution

Add a shared token-budget value contract that preserves existing numeric configs, accepts documented percentage strings, resolves them against the effective model context budget, and updates schema, docs, tests, and changelog together.

## Solution Assessment

Yes. Explicit percentage-string support is the narrow maintainable path because it keeps existing absolute numeric configs stable; silently auto-scaling existing numbers would change user behavior.
