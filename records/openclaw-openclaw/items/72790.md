## Summary

Keep open. Current main still treats the relevant compaction token settings as numeric-only across the public type, Zod validation, generated schema, docs, and runtime resolver, so percentage string values remain unimplemented. The request is narrow enough for a focused fix PR.

## What This Changes

Add percentage string support for compaction token budget settings such as reserveTokensFloor and keepRecentTokens so values scale with the active model context window.

## Best Possible Solution

Add a shared typed token-budget value contract that preserves existing numeric configs, accepts documented percentage strings, resolves them against the effective model context budget at runtime boundaries, and updates generated schema, docs, tests, and changelog together.

## Reproduction Assessment

Not applicable as a runtime bug report, but there is a high-confidence static reproduction path: current main only types, validates, documents, and resolves these compaction token settings as numbers.

## Solution Assessment

Yes. Explicit percentage-string support is the narrowest maintainable fix because it keeps existing absolute numeric configs stable; auto-scaling already configured numbers would silently change established behavior.
