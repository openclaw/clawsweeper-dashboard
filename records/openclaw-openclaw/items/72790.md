## Summary

Keep open. Current main still accepts and resolves the relevant compaction token budgets as absolute numbers only, and the active related PR is unmerged with API/review blockers, so this feature is neither implemented nor safely superseded yet.

## What This Changes

The issue requests context-window-relative compaction token budgets, such as percentage values for `reserveTokensFloor` and `keepRecentTokens`.

## Best Possible Solution

Settle one backward-compatible public config API for context-window-relative compaction budgets, then implement shared resolution across schema, runtime consumers, generated metadata, and docs while preserving current numeric semantics.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence that current main only accepts numeric absolute token budgets and lacks percent or share-based compaction budget support.

## Solution Assessment

Unclear until maintainers choose the public API shape. Explicit relative-budget support is the right direction, but additive share fields may be safer than overloading existing numeric keys with percent strings, and silently auto-scaling existing numbers would risk changing current user configs.
