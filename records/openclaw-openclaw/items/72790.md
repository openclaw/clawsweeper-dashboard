## Summary

Keep open. Current main still does not implement percentage string support for compaction token budgets: the public config surface, validation, generated schema, runtime readers, and docs remain numeric-only.

## What This Changes

Allow compaction token-budget settings such as reserveTokensFloor and keepRecentTokens to accept percentage values resolved against the effective model context window.

## Best Possible Solution

Add a shared token-budget value type and resolver for the compaction fields, preserve existing numeric semantics, resolve percent strings against effective context tokens, and update schema, docs, tests, and changelog together.

## Reproduction Assessment

Yes, for source-level reproduction. Current main types and validates the requested knobs as numbers, and runtime readers require numeric inputs before doing context-aware resolution.

## Solution Assessment

Yes. Explicit percent-string support resolved against the effective context window is narrower than auto-scaling existing numeric configs, because it preserves current user behavior.
