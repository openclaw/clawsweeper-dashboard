## Summary

Keep open. Current main still has context-window-aware capping for the numeric reserve floor, but it does not implement percentage string values for compaction token settings in the public config type, Zod schema, generated schema, docs, or Pi runtime resolver.

## What This Changes

Request support for percentage-based compaction token settings so values such as `reserveTokensFloor: "10%"` and `keepRecentTokens: "15%"` resolve against the active model context window.

## Best Possible Solution

Add a shared typed token-budget value contract for compaction settings that preserves numeric inputs, accepts documented percentage strings, resolves them against the effective model context budget in Pi runtime paths, and updates schema generation, docs, tests, and changelog together.

## Reproduction Assessment

Not applicable as a feature request, but there is a high-confidence static verification path. Current main's public type, Zod schema, generated schema, docs, and runtime resolver all accept numbers only for the relevant compaction token settings.

## Solution Assessment

Yes. Explicit percentage-string support is the narrowest maintainable solution because it keeps existing absolute numeric configs stable; silently auto-scaling existing numbers would change established behavior.
