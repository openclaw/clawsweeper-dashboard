## Summary

Keep open. Current main still exposes the requested compaction token knobs as numeric-only config, schema, docs, and runtime math, so percentage-based values remain unimplemented.

## What This Changes

The issue requests percentage-string support for context-aware compaction settings such as `reserveTokensFloor` and `keepRecentTokens`.

## Best Possible Solution

Add a shared token-budget value contract that preserves existing numeric configs, accepts documented percentage strings, resolves them against the effective model context window, and updates schema generation, docs, tests, and changelog together.

## Reproduction Assessment

Yes. Static inspection of current main shows percentage strings would be rejected by public validation/schema and not resolved by the runtime compaction settings path.

## Solution Assessment

Yes. Explicit number-or-percentage support is the narrow maintainable path because it keeps existing absolute numeric configs stable; silently auto-scaling existing numbers would change user behavior.
