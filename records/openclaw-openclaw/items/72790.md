## Summary

Keep open. Current main still defines, validates, publishes, and reads the relevant compaction token-budget settings as numeric-only, so percentage strings remain unimplemented; this is a focused config/runtime/docs feature rather than an obsolete report.

## What This Changes

Allow compaction token-budget settings such as `reserveTokensFloor` and `keepRecentTokens` to be configured as percentages of the effective model context window.

## Best Possible Solution

Extend the existing compaction token-budget contract with a shared number-or-percent resolver against the effective model context window, preserve numeric semantics, and update schema, docs, and focused regression tests.

## Reproduction Assessment

Yes. Source inspection on current main shows percent strings would fail schema validation and would not be consumed by the runtime readers, so the requested config form is still absent.

## Solution Assessment

Yes. Explicit percent-string support resolved against the effective context window is narrower and safer than auto-scaling existing numeric values, which would silently change current user configs.
