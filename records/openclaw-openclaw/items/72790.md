## Summary

Keep open. Current main still has numeric-only compaction token budgets, and the same feature is being explored in open PRs that have not landed and still need maintainer API/review handling.

## What This Changes

Request support for percentage or context-window-relative compaction settings for `reserveTokensFloor` and `keepRecentTokens`.

## Best Possible Solution

Choose one backward-compatible relative-budget config API, then land it consistently across schema, runtime compaction readers, memory preflight, generated config metadata, tests, and docs while preserving current numeric semantics.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence that current main only accepts numeric absolute compaction budgets and lacks percent or share-based budget resolution.

## Solution Assessment

Unclear until maintainers choose the public config API shape. Additive share fields look safer than percent-string overloading, but the implementation must cover all compaction and preflight consumers before this issue is resolved.
