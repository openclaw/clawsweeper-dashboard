## Summary

Keep open. Current main still exposes, validates, documents, and resolves the affected compaction token-budget settings as numeric-only, so percentage strings remain unimplemented and the provided context does not show a linked implementation PR.

## What This Changes

Allow compaction token-budget settings such as `reserveTokensFloor` and `keepRecentTokens` to be expressed as percentages of the effective context window.

## Best Possible Solution

Add a shared token-budget value type and resolver, preserve existing numeric behavior, resolve percent strings against the effective context window, and update schema, docs, tests, and changelog together.

## Reproduction Assessment

Yes for source-level reproduction. Current main types and validates these knobs as numbers, and the runtime readers require `typeof value === "number"`, so a config using `"10%"` cannot reach context-aware resolution.

## Solution Assessment

Yes. Explicit percentage-string support resolved against the effective context window is narrower than auto-scaling existing numeric values because it preserves current user configs.
