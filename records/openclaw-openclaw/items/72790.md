## Summary

Keep open. Current main still treats the requested compaction token settings as numeric-only in public config types, validation, generated schema, docs, and runtime/preflight math, so the enhancement remains valid and unimplemented.

## What This Changes

The issue requests percentage-string support for compaction token budgets so the same config scales across models with different context windows.

## Best Possible Solution

Add a shared token-budget value contract that preserves existing numeric configs, accepts documented percentage strings, resolves them against the effective model context budget, and updates schema generation, docs, tests, and changelog together.

## Reproduction Assessment

Not applicable as a runtime bug report, but yes for static reproduction: current source, validation, generated schema, docs, and runtime consumers all show numeric-only handling for the requested settings.

## Solution Assessment

Yes. Explicit percentage-string support is the narrow maintainable path because it keeps existing absolute numeric configs stable; silently auto-scaling existing numbers would change user behavior.
