## Summary

Keep open. Current main still exposes the requested compaction token-budget settings as numeric-only in public config types, validation, generated schema, docs, and runtime/preflight math, so percentage-based values are not implemented.

## What This Changes

Add support for percentage-string compaction token settings so configs scale across model context windows.

## Best Possible Solution

Add a shared token-budget value contract that preserves existing numeric configs, accepts documented percentage strings, resolves them against the effective model context window, and updates schema, docs, tests, and changelog together.

## Reproduction Assessment

Yes for static reproduction, though this is a feature request rather than a runtime bug. Current source, validation, generated schema, docs, and runtime consumers all show numeric-only handling for the requested settings.

## Solution Assessment

Yes. Explicit percentage-string support is the narrow maintainable path because it keeps existing absolute numeric configs stable; auto-scaling existing numbers would silently change user behavior.

## Repair Work Prompt

Implement percentage-based compaction token config for https://github.com/openclaw/openclaw/issues/72790. Preserve all existing numeric behavior, but allow documented percentage strings such as `"10%"` for context-window-relative compaction token settings. Add a shared resolver for token-budget values and use it where Pi compaction settings and preflight/memory-flush compaction math resolve `reserveTokensFloor`, `keepRecentTokens`, and, if the local design treats the three Pi token knobs consistently, `reserveTokens`. Resolve percentages against the effective model context window already available to those paths; do not add model-specific defaults, silently reinterpret existing numbers, or move owner-specific behavior into core. Update `AgentCompactionConfig`, Zod validation, generated schema/help metadata, docs, focused tests, and `CHANGELOG.md`. Include tests for valid percentages, invalid strings, numeric backward compatibility, small-context capping, and preflight threshold math.
