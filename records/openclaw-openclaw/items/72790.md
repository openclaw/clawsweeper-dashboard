## Summary

Keep open. Current main still defines, validates, publishes, and consumes the relevant compaction token-budget settings as numeric-only, so percentage strings remain a focused config/runtime/docs feature rather than implemented behavior.

## What This Changes

Request support for percentage values such as "10%" in compaction token-budget settings so they resolve against the effective model context window.

## Best Possible Solution

Add a shared number-or-percent token-budget resolver against the effective context window, preserve existing numeric semantics, and update config schema, generated docs, runtime paths, and regression tests together.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection confirms current main's config contract and runtime paths are numeric-only, so the requested percent values cannot be used today.

## Solution Assessment

Yes. Explicit percent-string support resolved against the effective context window is narrower and safer than auto-scaling existing numeric values, which would silently change current user configs.
