## Summary

Keep open. Current main still accepts only absolute numeric compaction token settings in the public config type, validation schema, generated schema, docs, and Pi runtime resolver, so the requested percentage-based config is not implemented.

## What This Changes

Allow compaction token budget settings to accept percentage values such as "10%" and resolve them against the effective model context window.

## Best Possible Solution

Add a shared typed token-budget value contract that preserves numeric inputs, accepts documented percentage strings, resolves them against the effective contextTokenBudget in Pi compaction paths, and keeps the public type, Zod schema, generated schema, docs, tests, and changelog aligned.

## Reproduction Assessment

Not applicable as a feature request, but there is a high-confidence static verification path. Current main's type declarations, Zod schema, generated schema, docs, and runtime resolver all accept numbers only for the relevant token-budget settings.

## Solution Assessment

Yes. Explicit percentage-string support is the narrowest maintainable direction because it keeps existing absolute numeric configs stable; silently auto-scaling existing numbers would change established behavior.
