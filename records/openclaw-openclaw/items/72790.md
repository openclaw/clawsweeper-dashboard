## Summary

Keep open. Current main still treats the requested compaction token-budget settings as numeric-only in public types, validation, generated schema, docs, and runtime resolution, so percentage strings remain unimplemented; the request is a narrow feature/config addition rather than a stale cleanup close.

## What This Changes

Request percentage string values for compaction token-budget settings so they scale with model context windows.

## Best Possible Solution

Add a shared token-budget value type and resolver for these compaction fields, preserve existing numeric semantics, document percentage syntax, and resolve percentages against the effective model context window wherever the fields are consumed.

## Reproduction Assessment

Yes, for source-level reproduction. A config using `"10%"` would be rejected by the Zod schema, and the current runtime readers require numbers even if validation were bypassed.

## Solution Assessment

Yes. Explicit percentage-string support resolved against the effective context window is the narrowest maintainable direction; auto-scaling existing absolute numbers would change current user semantics.
