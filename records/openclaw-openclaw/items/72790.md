## Summary

Keep open. Current main still declares, validates, publishes, and reads the relevant compaction token-budget settings as numeric-only, so percentage strings remain unimplemented and this is a focused config/runtime/docs feature rather than a closeable cleanup item.

## What This Changes

Allow existing compaction token-budget settings such as `reserveTokensFloor` and `keepRecentTokens` to accept percentages resolved against the effective model context window.

## Best Possible Solution

Add a shared number-or-percent token-budget resolver for the existing compaction knobs, resolve percentages against the effective context window, preserve current numeric semantics, and update schema/types/docs/generated outputs together.

## Reproduction Assessment

Yes, source-level. Current main validates these knobs as numbers and the runtime readers ignore non-number values, so a config value like `"10%"` cannot validate or reach context-aware resolution.

## Solution Assessment

Yes. Explicit percent-string support resolved against the effective context window is narrower than auto-scaling existing numeric values, which would silently change current user configs.
