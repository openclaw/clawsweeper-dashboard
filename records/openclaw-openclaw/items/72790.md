## Summary

Keep open. Current main still publishes and reads the relevant compaction token-budget settings as numeric-only, so percentage strings remain unimplemented and the remaining work is a focused config/runtime/docs feature.

## What This Changes

Request support for percentage string values in compaction token-budget config such as `reserveTokensFloor` and `keepRecentTokens`, resolved relative to model context windows.

## Best Possible Solution

Extend the existing compaction token-budget config contract with a shared number-or-percent value resolver, resolved against the effective model context window, while preserving current numeric semantics.

## Reproduction Assessment

Yes, via source-level reproduction. Current main validates these knobs as numbers and the runtime readers ignore non-number values, so a config like `"10%"` cannot validate or reach context-aware resolution.

## Solution Assessment

Yes. Explicit percent-string support resolved against the effective context window is narrower and safer than auto-scaling existing numeric values, which would silently change current user configs.
