## Summary

Keep open. Current main still exposes the relevant compaction token settings as numeric-only across the public type, Zod validation, generated schema, docs, and runtime/preflight math, so the enhancement remains valid and unimplemented; the scope is narrow enough for a focused repair PR.

## What This Changes

Support percentage string values for compaction token-budget settings such as `reserveTokensFloor` and `keepRecentTokens` so configs scale with the effective model context window.

## Best Possible Solution

Add a shared typed token-budget value contract that preserves existing numeric configs, accepts documented percentage strings, resolves them against the effective model context budget at runtime, and uses that resolver consistently in Pi settings plus preflight compaction/memory-flush threshold paths. Update schema generation, docs, focused tests, and the changelog in the same PR.

## Reproduction Assessment

Not applicable as a runtime bug report, but there is a high-confidence static reproduction path: current type definitions, Zod validation, generated schema, docs, and runtime resolvers all accept or consume numeric token settings only.

## Solution Assessment

Yes. Explicit percentage-string support is the narrow maintainable direction because it keeps existing absolute numeric configs stable; auto-scaling existing numbers would silently change established behavior.
