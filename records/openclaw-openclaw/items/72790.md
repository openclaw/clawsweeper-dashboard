## Summary

Keep open. Current main still treats `reserveTokens`, `keepRecentTokens`, and `reserveTokensFloor` as integer-only config and runtime values, so percentage strings such as `"10%"` remain unsupported across validation, generated schema, docs, and compaction resolution.

## What This Changes

Request support for percentage-based compaction token budgets so compaction settings scale with each model's context window.

## Best Possible Solution

Add a shared token-budget value contract that preserves existing numeric semantics, accepts documented percentage strings, resolves them against the effective context window, and updates schema, docs, tests, and changelog together.

## Reproduction Assessment

Yes. Static reproduction is high-confidence: the checked-in Zod schema rejects `"10%"`, and the runtime compaction resolvers only consume finite numbers before applying token budgets.

## Solution Assessment

Yes. Explicit percentage-string support resolved against the effective model context budget is the narrow maintainable path; silently auto-scaling existing numeric values would change established user behavior.
