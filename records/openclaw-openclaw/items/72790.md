## Summary

Keep open. Current main still exposes the relevant compaction knobs as numeric-only config and resolves them with number-only runtime paths, so percentage strings remain unimplemented.

## What This Changes

Add support for percentage-based compaction token settings so reserve and recent-history budgets can scale with the active model context window.

## Best Possible Solution

Add a shared token-budget value contract that preserves existing numeric behavior, accepts documented percentage strings, resolves them against the effective context window, and updates schema, docs, tests, and changelog together.

## Reproduction Assessment

Yes. Static reproduction is high confidence: a config value such as reserveTokensFloor: "10%" is rejected by validation and would not be consumed by the runtime numeric resolvers.

## Solution Assessment

Yes. Explicit percentage-string support resolved against contextTokenBudget is the narrow maintainable path; auto-scaling existing numeric values would change established user behavior.
