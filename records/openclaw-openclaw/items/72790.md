Choose one backward-compatible relative-budget API, document precedence and ranges, then implement shared resolution across schema, runtime compaction paths, memory-flush thresholds, generated metadata, and docs while preserving absolute-number semantics.

## Solution Assessment

Unclear until maintainers choose the public API shape. Additive relative-budget fields look safer than silently changing absolute-number semantics, but the open PR needs API approval and fixes before it can be the solution.
