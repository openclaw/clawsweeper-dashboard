## Summary

Keep open. Current main still treats the requested compaction token settings as numeric-only in public types, validation, generated schema, docs, and runtime resolution, so percentage strings like "10%" remain unimplemented.

## What This Changes

Support percentage values such as "10%" for compaction token budgets so settings scale with the active model context window.

## Best Possible Solution

Add a shared token-budget value contract that preserves existing numeric configs, accepts documented percentage strings, resolves them against the effective context window, and updates schema generation, docs, tests, and changelog together.

## Reproduction Assessment

Yes for static reproduction. Current source inspection shows percentage strings are rejected by config validation and would be ignored by runtime numeric resolvers before they could scale against the model context window.

## Solution Assessment

Yes. Explicit percentage-string support resolved against the effective context budget is the narrow maintainable path; silently auto-scaling existing numeric values would change established user behavior.
