## Summary

Keep open. Current main still exposes and validates the relevant compaction token-budget knobs as numeric-only, and the provided discussion has no linked implementation PR that would make this obsolete.

## What This Changes

The issue requests percent-based values for compaction token-budget settings so the same config scales across model context windows.

## Best Possible Solution

Add a shared number-or-percent token budget resolver for compaction settings, resolve percentages against the effective model context window, and update config schema/docs while preserving numeric semantics.

## Reproduction Assessment

Yes, via source-level reproduction: current main's config type and Zod schema require numbers, and the Pi runtime helpers drop non-number values before applying overrides. A value like "10%" cannot validate or resolve today.

## Solution Assessment

Yes. Explicit percent-string support resolved against the effective context window is the narrow maintainable path; auto-scaling existing numeric values would silently change current user configs.
