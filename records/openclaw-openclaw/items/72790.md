## Summary

Keep open. Current main still only accepts absolute numeric compaction token budgets, and the related share/percentage work is still in open PRs that need maintainer API and review handling before this issue can close.

## What This Changes

Add percentage or context-window-relative compaction settings so token budgets like reserve floor and keep-recent scale across models with different context windows.

## Best Possible Solution

Choose one backward-compatible relative-budget config API, then land it consistently across schema, runtime compaction readers, memory preflight/flush planning, generated metadata, docs, and focused tests while preserving current numeric semantics.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence that current main only accepts absolute numeric compaction budgets and lacks percent or share-based budget resolution.

## Solution Assessment

Unclear until maintainers choose the public config API shape. Additive share fields look safer than overloading existing numeric keys with percent strings, but the implementation must cover all compaction and preflight consumers before this issue is resolved.
