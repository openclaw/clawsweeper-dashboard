## Summary

Keep open. Current main still exposes the compaction token-budget knobs as numeric-only in the public config contract, schema, runtime readers, generated schema, and docs, so percentage values remain unimplemented.

## What This Changes

Request to allow percentage values such as `"10%"` for compaction token-budget settings so configs scale with model context windows.

## Best Possible Solution

Extend the existing compaction config contract with a shared number-or-percent token-budget resolver against the effective model context window, while preserving current numeric semantics.

## Reproduction Assessment

Yes, via source-level reproduction. Current main validates these knobs as numbers and runtime readers require numeric values, so a config value like `"10%"` cannot validate or resolve into a context-aware token budget.

## Solution Assessment

Yes. Explicit percent-string support resolved against the effective context window is narrower and safer than auto-scaling existing numeric values, which would silently change current user configs.
