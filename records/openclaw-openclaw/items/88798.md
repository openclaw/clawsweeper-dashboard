## What This Changes

This PR removes QuickStart's automatic `gateway.controlUi.allowInsecureAuth=true` write for loopback setup and updates wizard tests so the flag stays unset unless explicitly configured.

## Best Possible Solution

Keep explicit `allowInsecureAuth` support, but ship this default change only after maintainers choose secure-by-default QuickStart or request an explicit setup opt-in/warning for local HTTP compatibility.

## Reproduction Assessment

Yes. Current main and `v2026.6.6` write `allowInsecureAuth=true` for QuickStart loopback setup when unset, and the PR body includes terminal proof that the patched interactive QuickStart leaves `gateway.controlUi` empty.

## Solution Assessment

Unclear until maintainer decision. The code change is the narrowest way to stop silently writing a warning-class flag and it preserves explicit opt-in, but an explicit onboarding opt-in or warning may be better if local HTTP compatibility must remain part of setup.
