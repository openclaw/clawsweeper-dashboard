## Summary

Keep open. Current main still has the chat reminder delivery gap, and the PR is not merge-ready because its promotion path bypasses current delivery-context routing while the branch is draft and dirty against current main.

## What This Changes

The PR changes cron-tool add handling to promote implicit active-chat systemEvent reminders into isolated agentTurn announce jobs and adds cron-tool tests for promotion, explicit main reminders, and context history.

## Best Possible Solution

Implement the narrow cron-tool promotion for active-chat implicit reminder systemEvents while preserving explicit sessionTarget, delivery.mode="none", webhook delivery, explicit targets, accountId/threadId, and currentDeliveryContext, with focused regression coverage and a changelog entry.

## Reproduction Assessment

Yes. ## Solution Assessment

No, not as written. The direction is plausible and narrow, but the implementation should rebase onto current main and compute inferred delivery from inferDeliveryFromContext(opts.currentDeliveryContext) before falling back to inferDeliveryFromSessionKey(...).
