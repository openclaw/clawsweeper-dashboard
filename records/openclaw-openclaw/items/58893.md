## Summary

Keep open: this is a focused cron reminder delivery fix, but the PR still needs real behavior proof and its current patch would regress delivery routing and explicit internal-job preservation.

## What This Changes

The branch promotes implicit chat reminder `systemEvent` cron jobs into isolated `agentTurn` announce jobs and adds cron-tool regression tests.

## Best Possible Solution

Land a narrow cron-tool fix that promotes only implicit active-chat reminders into isolated announce jobs while preserving explicit internal or no-delivery jobs, supported wrappers, and currentDeliveryContext routing.

## Reproduction Assessment

Yes, source-level. Current main normalizes implicit `systemEvent` cron jobs to `sessionTarget="main"` and only infers announce delivery for `agentTurn` jobs, so an active-chat reminder created as a system event can remain internal with `deliveryStatus="not-requested"`.

## Solution Assessment

No, not as written. The fix direction is narrow, but the patch should reuse currentDeliveryContext before session-key fallback and should detect explicit fields after unwrapping supported legacy job wrappers.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve current delivery context for promoted reminders:** `src/agents/tools/cron-tool.ts:451`
  - body: The promotion path builds `inferredDelivery` only from `agentSessionKey`. Current main intentionally prefers `currentDeliveryContext` so Matrix/Telegram targets keep provider casing, accountId, and threadId, and so delivery still infers when no session key is present; promoted reminders should use that same context-first inference and add coverage for context-only/context-preferred promotion.
  - confidence: 0.87
- **[P2] Respect explicit nested delivery before promotion:** `src/agents/tools/cron-tool.ts:239-240`
  - body: Current normalization accepts wrapped inputs like `{ data: { ... } }`, but this guard only checks the outer raw job for `sessionTarget` or `delivery`. A wrapped job with nested `sessionTarget: "main"` or `delivery: { mode: "none" }` and reminder text would be promoted into a user-visible isolated `agentTurn`; check the same unwrapped input used by normalization and cover that case.
  - confidence: 0.84
