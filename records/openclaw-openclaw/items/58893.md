## Summary

Keep open: current main still leaves implicit active-chat systemEvent reminder cron jobs on the internal main-session path, so the PR targets a real bug, but the proposed promotion drops currentDeliveryContext routing and the contributor has not provided real after-fix behavior proof.

## What This Changes

The PR promotes implicit active-chat systemEvent reminder cron jobs into isolated agentTurn announce jobs and adds cron-tool tests for promotion, explicit main reminders, and context history.

## Best Possible Solution

Land a narrow cron-tool fix that promotes only implicit active-chat reminder systemEvents while preserving explicit sessionTarget, delivery.mode="none", webhook/explicit delivery targets, currentDeliveryContext, accountId/threadId, and focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: create cron.add from an active chat with payload.kind="systemEvent" and no explicit delivery; current main normalizes it to sessionTarget="main" and never enters the agentTurn delivery inference block.

## Solution Assessment

No, not as written. The direction is narrow, but the promotion should infer delivery from inferDeliveryFromContext(opts.currentDeliveryContext) before falling back to inferDeliveryFromSessionKey(...), with regression coverage for that route.
