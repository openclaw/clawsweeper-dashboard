## Summary

Keep open: current main still has a source-reproducible chat-reminder delivery gap, but this draft and conflicting PR is not mergeable as-is because its promotion path ignores currentDeliveryContext and it lacks after-fix real behavior proof.

## What This Changes

The PR promotes implicit active-chat systemEvent reminder cron jobs into isolated agentTurn announce jobs and adds cron-tool regression tests.

## Best Possible Solution

Land a narrow cron-tool fix that promotes only implicit active-chat systemEvent reminders into isolated announce agentTurn jobs while reusing current-main delivery inference and preserving explicit internal or no-delivery jobs.

## Reproduction Assessment

Yes, source-level. On current main, an active-chat cron.add with payload.kind="systemEvent" and no explicit delivery/sessionTarget normalizes to sessionTarget="main" and never reaches the agentTurn delivery inference block.

## Solution Assessment

No, not as written. The direction is narrow, but the promotion path should infer from currentDeliveryContext before falling back to the session key and needs matching regression coverage plus real behavior proof.
