## Summary

Keep open for maintainer review: current main still lacks Telegram subagent lifecycle hook registration and the explicit per-kind thread-spawn capability, while this PR provides a focused implementation plus now-sufficient redacted live Telegram proof. I found no blocking correctness or security finding, but the new SDK/channel capability seam still needs owner review before landing.

## What This Changes

The PR adds per-kind thread-bound spawn capability plumbing, Telegram subagent lifecycle hooks, docs, tests, prompt snapshots, and a changelog entry for binding spawned subagents to child forum topics.

## Best Possible Solution

Land or replace this canonical Telegram implementation so normal top-level bindings stay current, explicit subagent thread spawns create child forum topics, and ACP child-topic support remains disabled unless separately designed.

## Reproduction Assessment

Yes at source level: current main has Telegram child-topic binding primitives but no Telegram subagent hook registration, and automatic thread spawn support is still tied to child top-level placement. The PR body also supplies after-fix live Telegram proof for the intended path.

## Solution Assessment

Yes, with maintainer approval for the new seam. The per-kind capability is narrower than changing Telegram's default top-level placement and preserves ACP behavior while enabling only explicit subagent thread spawns.
