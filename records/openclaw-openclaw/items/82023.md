## Summary

Keep this PR open as the current implementation candidate for Telegram thread-bound subagent spawns, but do not merge it yet. Current main still lacks Telegram subagent hook registration, while this branch still needs live Telegram forum proof and is currently conflicting against main.

## What This Changes

The PR adds per-kind automatic thread-spawn capability metadata plus Telegram subagent lifecycle hooks, docs, tests, and prompt snapshots so `sessions_spawn(thread:true)` can bind spawned subagents to child forum topics while ACP thread spawns stay disabled.

## Best Possible Solution

Land or replace one current-boundary Telegram implementation that keeps normal Telegram top-level bindings as `current`, advertises only supported per-kind thread spawns, and includes redacted live Telegram forum proof.

## Reproduction Assessment

Yes at source level: current main requires a `subagent_spawning` hook for thread-bound native subagent sessions, while Telegram has child topic binding primitives but no hook registration and no separate automatic-spawn capability. No live Telegram reproduction was run in this read-only review.

## Solution Assessment

Unclear for merge until live proof and rebase review are complete. The per-kind capability is a narrower approach than changing Telegram’s default placement, but it needs maintainer review as the canonical implementation path.
