## Summary

This should stay open: current main has opt-in mitigations for large tool catalogs, but it does not resolve the reported default behavior where ordinary embedded PI turns can still carry the enabled tool surface unless tools are disabled, allowlisted, or Tool Search is explicitly enabled.

## What This Changes

Request to reduce prompt-assembly latency by avoiding full tool-schema injection for trivial agent turns.

## Best Possible Solution

Define an owner-approved lightweight-turn policy that preserves tool safety and replay semantics, likely by formalizing when to use Tool Search, tool allowlists, raw model mode, or an explicit lite/no-tools mode, then benchmark the chosen path.

## Reproduction Assessment

Yes, source-level rather than live: current embedded PI runs construct and pass enabled tools unless disabled, allowlisted, or Tool Search is configured, and the issue includes concrete prompt-length timing data. I did not run an Ollama benchmark in this read-only review.

## Solution Assessment

Unclear. The suggested directions are plausible, but automatic schema skipping or default catalog compaction changes model capability and fallback behavior, so maintainers should choose the product policy before implementation.
