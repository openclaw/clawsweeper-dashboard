## Summary

Keep open. Current main has opt-in ways to reduce the tool surface, but ordinary agent turns with tool-capable models still build and pass the default OpenClaw tool set; automatic trivial-turn schema skipping, lazy default registration, or an explicit lightweight user-facing mode remains a product/architecture decision.

## What This Changes

The issue asks OpenClaw to avoid charging trivial agent turns for the full system prompt and tool-schema surface, or otherwise provide a lighter prompt path.

## Best Possible Solution

Choose and document a maintainable lightweight prompt/tool strategy that cuts default schema cost for trivial or explicitly lightweight turns while preserving a safe path back to full tools when needed.

## Reproduction Assessment

Source-reproducible but not live-reproduced here: current main still constructs and passes the default tool set for ordinary tool-capable agent turns without toolsAllow. The downstream issues provide timing and token evidence, but this review did not run a current-main model benchmark.

## Solution Assessment

Unclear. The requested directions are plausible, but choosing between automatic schema-skip, lazy/default Tool Search, an explicit lightweight mode, or schema compression needs maintainer product and safety judgment.
