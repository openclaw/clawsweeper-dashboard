## Summary

Keep open. Current main still has only DM-specific session scoping, group/channel peers remain isolated in source and docs, and the focused implementation at https://github.com/openclaw/openclaw/pull/47234 is still open rather than merged.

## What This Changes

Request an opt-in `groupScope`-style configuration so selected group chats can share the agent main session while other groups remain isolated.

## Best Possible Solution

Define an additive opt-in routing/config contract that preserves default group isolation, lets selected group/channel peers use main or a named shared session, and updates schemas, generated config metadata, docs, routing/reply tests, and privacy guidance.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main still isolates group/channel peers and has no group-to-main config surface.

## Solution Assessment

Unclear whether the literal `groupScope` field is the best final API. The maintainable direction is an explicit opt-in per-binding or named shared-session contract aligned with privacy, sandbox, and reply-routing boundaries.

## Security Review

Status: needs_attention

Summary: The requested feature is privacy-sensitive because it intentionally merges selected group/channel context into a main session used by other interactions.

Concerns:

- **[medium] Document merged group context exposure:** `docs/channels/groups.md:144`
  - body: Any implementation should keep default isolation, require explicit opt-in, and document that messages from a merged group or channel can become context for main-session interactions and tool or sandbox decisions.
  - confidence: 0.9
- **[medium] Preserve non-main runtime boundaries:** `docs/channels/groups.md:144`
  - body: Current docs describe groups as non-main sessions with different sandbox posture, so a group-to-main route needs an explicit decision on whether the group turn inherits main-session trust or retains group-origin safeguards.
