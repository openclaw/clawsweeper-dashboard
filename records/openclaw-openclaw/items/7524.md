## Summary

Keep open: current main and the latest shipped release still keep non-direct peers isolated, and the remaining work is a new opt-in session-routing contract that needs maintainer product and security review rather than cleanup closure.

## What This Changes

The issue requests a `groupScope`-style option so selected group or channel conversations can use an agent's main session instead of their default isolated session.

## Best Possible Solution

Define an additive opt-in session-target contract that preserves default group isolation, carries origin metadata through routing and replies, documents privacy behavior, and adds focused core plus representative channel regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config gap. Source and docs inspection gives high confidence that current non-direct peers remain isolated and no group-to-main option exists.

## Solution Assessment

Unclear as a final implementation shape. A per-group or per-binding opt-in is plausible, but maintainers still need to choose the public config contract, privacy semantics, outbound routing behavior, and relationship to broader shared-session work.
