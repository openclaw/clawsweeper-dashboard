## Summary

Keep this open: current main still has only DM-specific session scoping, and the live linked implementation PR is open and unmerged. The request is valid but product/security-sensitive because it adds config surface and intentionally collapses selected group sessions into the main context.

## What This Changes

The issue requests a `groupScope` option so selected group chats can use the agent main session instead of always receiving isolated group session keys.

## Best Possible Solution

Keep this issue open as the focused tracker, review https://github.com/openclaw/openclaw/pull/91702 against the broader shared-session/security direction, and close it only after an accepted implementation lands.

## Reproduction Assessment

Yes. Source inspection shows direct peers can use `dmScope`, while group/channel peers still resolve to isolated channel/group session keys and the strict config schemas have no `groupScope`.
