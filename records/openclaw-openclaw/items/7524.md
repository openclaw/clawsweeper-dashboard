## Summary

Keep open. Current main still has only DM-specific session consolidation, group/channel peers still resolve to isolated session keys, and the focused implementation candidate is still an open PR rather than merged behavior.

## What This Changes

The issue requests an opt-in `groupScope`-style configuration that routes selected group or channel chats into the agent main session while preserving isolated defaults for other groups.

## Best Possible Solution

Land #47234 or an equivalent additive, opt-in route-binding/session contract that preserves default isolation and documents the trust boundary.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main verifies the limitation: group/channel peers still build isolated session keys and config surfaces expose only DM scoping.

## Solution Assessment

Unclear whether the literal `groupScope` shape is final. The maintainable direction is a backwards-compatible per-binding or shared-session contract coordinated with #19929, with privacy and queueing behavior documented.
