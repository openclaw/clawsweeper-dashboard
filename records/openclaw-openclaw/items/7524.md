## Summary

Keep open: current main and the latest release still isolate non-direct group/channel peers, while the requested group-to-main behavior remains an unlanded public config and trust-boundary decision with a related open implementation PR under review.

## What This Changes

Add an opt-in groupScope-style session routing option so selected group or channel chats can use an agent's main session instead of isolated per-group keys.

## Best Possible Solution

Define an additive opt-in session-target contract that preserves default isolation, records origin metadata for reply routing, documents privacy and sandbox behavior, and lands with focused routing/config/docs regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config gap. Source, docs, release inspection, and regression tests give high confidence that current main still isolates non-direct peers and exposes no group-to-main option.

## Solution Assessment

Unclear as a final implementation shape. A per-group or per-binding opt-in is plausible, but maintainers still need to choose the public config contract, privacy semantics, outbound routing behavior, and relationship to https://github.com/openclaw/openclaw/issues/19929 and https://github.com/openclaw/openclaw/pull/47234.
