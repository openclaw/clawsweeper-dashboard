## Summary

Keep open. Current main still only has DM-specific session scoping, while group/channel peers resolve to isolated session keys and the strict config/types/docs expose no group-to-main option; open PR #47234 remains an implementation candidate rather than a reason to close before merge.

## What This Changes

The issue requests an opt-in groupScope-style setting so selected group chats can consolidate into the agent main session while isolated group sessions remain the default.

## Best Possible Solution

Keep this as the focused tracker and resolve it by reviewing/landing #47234 or an equivalent additive routing/config contract coordinated with #19929.

## Reproduction Assessment

Yes for the current limitation. Source inspection on current main shows non-direct peers still produce agent:<agentId>:<channel>:<peerKind>:<peerId>, and the schemas/docs have no group-to-main option.

## Solution Assessment

No current-main solution exists yet. The best direction is a narrow, backwards-compatible opt-in session routing contract for selected group/channel peers, with #47234 treated as a review candidate and #19929 as broader design context.
