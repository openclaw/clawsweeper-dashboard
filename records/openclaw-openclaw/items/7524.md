## Summary

Keep open. Current main and the latest release still only support DM-specific session scoping, while group/channel peers remain isolated; prior discussion also treats this issue as the focused tracker, with a related open PR still needing product/security review rather than cleanup closure.

## What This Changes

Request a `groupScope`-style config option so selected group/channel chats can route into an agent main session instead of isolated peer sessions.

## Best Possible Solution

Keep this as the focused tracker and design an additive, opt-in session target contract for selected group/channel peers, coordinated with https://github.com/openclaw/openclaw/issues/19929 and the review path for https://github.com/openclaw/openclaw/pull/47234.
