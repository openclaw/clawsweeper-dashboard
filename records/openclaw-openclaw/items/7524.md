## Summary

Keep open: current main and v2026.5.22 still only support DM-specific session consolidation, while selected group/channel-to-main routing remains an unresolved public config and trust-boundary decision.

## What This Changes

The issue requests an opt-in `groupScope`-style setting so selected group chats can share the agent main session instead of always using isolated group session keys.

## Best Possible Solution

Keep this focused tracker open and design an additive opt-in shared-session routing contract that preserves isolated defaults, carries origin metadata, documents privacy behavior, and includes routing/config/docs/channel regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config gap. Source, docs, tests, and latest-release inspection give high confidence that current OpenClaw still isolates non-direct peers and exposes no group-to-main option.

## Solution Assessment

Unclear as a final implementation shape. A selective opt-in is directionally right, but maintainers still need to choose the public config contract, privacy semantics, reply-routing behavior, and relationship to https://github.com/openclaw/openclaw/issues/19929 and https://github.com/openclaw/openclaw/issues/51805.
