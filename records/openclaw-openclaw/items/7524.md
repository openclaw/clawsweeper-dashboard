## Summary

Keep open. Current main and the latest release still expose only DM-specific session scoping, while group/channel peers remain isolated and the focused implementation PR is still open rather than merged.

## What This Changes

The issue requests an opt-in `groupScope`-style config to route selected group or channel chats into the agent main session instead of isolated per-peer sessions.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/47234 or an equivalent additive, opt-in routing/session contract that preserves default isolation and documents privacy, queueing, and sandbox behavior.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that the limitation still exists because group/channel peers build isolated session keys and config surfaces expose only DM scoping.

## Solution Assessment

Unclear whether the literal `groupScope` API is the final shape. The maintainable direction is a backwards-compatible per-binding or shared-session contract coordinated with https://github.com/openclaw/openclaw/issues/19929 and documented as a trust-boundary change.
