## Summary

Keep open. Current main still has only DM-specific session scoping, group/channel peers still resolve to isolated session keys, and the focused implementation candidate is still an open PR rather than merged behavior.

## What This Changes

Request adds an opt-in group/session scoping option so selected trusted group or channel chats can share an agent's main session context.

## Best Possible Solution

Land #47234 or an equivalent additive, opt-in route-binding/session contract that preserves default isolation while documenting the trust boundary.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main verifies the limitation: group/channel peers still build isolated session keys and config surfaces expose only DM scoping.

## Solution Assessment

Unclear whether the literal `groupScope` shape is final. The maintainable direction is a backwards-compatible per-binding or shared-session contract coordinated with #19929, with privacy and sandbox behavior documented.
