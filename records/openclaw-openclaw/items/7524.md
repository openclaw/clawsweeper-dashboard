## Summary

Keep open: current main and the latest release still isolate group/channel peers, strict config/docs expose no groupScope-style option, and the linked implementation PR is open rather than merged.

## What This Changes

The issue requests an opt-in groupScope-style routing config so selected group chats can share the agent main session while other groups remain isolated.

## Best Possible Solution

Land or replace the linked implementation with an additive, isolated-by-default group session-scope contract that covers inbound/outbound routing, schemas/types/help, docs, security audit posture, and real behavior proof.

## Reproduction Assessment

Yes at source level: current main deterministically routes non-direct peers to isolated channel/peer session keys and the strict schemas expose no groupScope. This is a feature gap rather than a failing bug reproduction.
