## Summary

Keep open. Current main still only supports DM-specific session scoping, group/channel peers remain isolated by source, and the focused implementation at https://github.com/openclaw/openclaw/pull/47234 is still open rather than merged; related cleanup has treated this issue as the canonical focused tracker rather than superseding it with broader shared-session requests.

## What This Changes

The issue requests an opt-in groupScope-style option so selected group or channel conversations can use the agent main session instead of an isolated group/channel session.

## Best Possible Solution

Define an additive, opt-in routing/config contract that keeps default group isolation, lets selected group/channel peers use main or a named shared session, and updates schemas, generated metadata, docs, and regression coverage across routing and reply delivery.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection confirms current main keeps group/channel peers on isolated keys and has no group-to-main config surface.

## Solution Assessment

Unclear whether the literal groupScope shape is the best final API. The maintainable direction is an opt-in per-binding or named shared-session contract aligned with privacy and runtime-boundary docs.
