## Summary

Keep open: current main and v2026.5.20 still isolate non-direct group/channel peers and expose no selected group-to-main session option. The issue remains the focused tracker, while the related implementation PR is still open and needs product/security review.

## What This Changes

Request an opt-in groupScope or equivalent session-routing option so selected group chats can use the agent main session instead of isolated group session keys.

## Best Possible Solution

Define an additive opt-in session-target contract that preserves default group isolation, carries the chosen target through inbound and outbound routing, documents privacy and sandbox semantics, and adds focused core plus representative channel regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config gap. Source, docs, tests, and v2026.5.20 inspection give high confidence that current non-direct peers remain isolated and no group-to-main option exists.

## Solution Assessment

Unclear as a final implementation shape. A per-group or per-binding opt-in is plausible, but maintainers still need to choose the public config contract, privacy semantics, outbound routing behavior, and relationship to broader shared-session work.
