- `impact:session-state`: The request changes which conversations share transcript context, session keys, queue lanes, and main-session state.

## Summary

Keep open: current main and v2026.5.20 still provide only DM-specific session scoping, while group/channel peers remain isolated and strict config/docs expose no group-to-main option. The open implementation PR and broader shared-session issues are useful follow-up paths, not shipped behavior or a clear superseding canonical item.

## What This Changes

Add an opt-in `groupScope` or equivalent session target so selected group chats can share an agent's main session instead of isolated group sessions.

## Best Possible Solution

Define an additive opt-in session-target contract that preserves default group isolation, carries the selected target through inbound and outbound routing, documents privacy/sandbox semantics, and adds focused routing plus representative channel regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config gap. Source, docs, tests, and v2026.5.20 inspection give high confidence that current non-direct peers remain isolated and no group-to-main option exists.

## Solution Assessment

Unclear as a final implementation shape. A per-group or per-binding opt-in is plausible, but maintainers still need to choose the public config contract, privacy semantics, outbound routing behavior, and relationship to broader shared-session work.
