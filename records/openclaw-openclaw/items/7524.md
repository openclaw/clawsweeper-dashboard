## Summary

Keep open. Current main still isolates group/channel peers, strict config exposes only DM scoping, docs/tests preserve group isolation, and the focused implementation at https://github.com/openclaw/openclaw/pull/47234 remains open and conflicting rather than merged.

## What This Changes

Request an opt-in groupScope/shared-session config so selected group/channel chats can route into the agent main session while other groups remain isolated.

## Best Possible Solution

Define an additive opt-in routing contract that keeps default group isolation, lets selected group/channel peers use main or a named shared session, and updates schemas/help/docs plus routing, outbound, and channel regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an enhancement. Source inspection gives high confidence current main still lacks the requested group-to-main session scope and preserves group/channel isolation.

## Solution Assessment

Unclear whether the literal groupScope shape is the final best API. The maintainable direction is an opt-in per-binding or shared-session contract aligned with existing routing config and backed by privacy documentation.
