## Summary

Keep open. Current main and the latest release still only expose DM-specific session scoping; group/channel peers remain isolated, strict config cannot express group-to-main routing, and https://github.com/openclaw/openclaw/pull/47234 is still open rather than merged.

## What This Changes

The issue requests an opt-in groupScope-style configuration that routes selected group/channel conversations into the agent main session instead of always isolating them.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/47234 or an equivalent additive, opt-in routing/session contract that preserves default isolation and documents privacy, queueing, and sandbox behavior.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main gives high confidence that the limitation still exists: group/channel peers build isolated session keys and config surfaces expose only DM scoping.

## Solution Assessment

Unclear whether the literal groupScope API is final. The maintainable direction is a backwards-compatible per-binding or shared-session contract coordinated with https://github.com/openclaw/openclaw/issues/19929 and documented as a trust-boundary change.
