The issue requests a configurable `groupScope`-style option so selected group chats can share an agent's main session instead of always using isolated group session keys.

## Best Possible Solution

Keep this as the focused tracker until maintainers either land https://github.com/openclaw/openclaw/pull/91702 or an equivalent backwards-compatible, opt-in group session routing contract with inbound/outbound parity and security guidance.

## Reproduction Assessment

Yes, at source level. Current main deterministically derives isolated keys for non-direct peers and the strict schemas cannot express a group-to-main option.

## Solution Assessment

Unclear as a final product path. `groupScope` is a plausible additive fix with an open implementation PR, but maintainers still need to settle security posture, config scope, and PR readiness before this issue can close.
