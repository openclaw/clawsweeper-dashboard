## Summary

Current main and release v2026.6.5 still isolate every non-direct peer. The open closing PR at https://github.com/openclaw/openclaw/pull/91702 owns the focused implementation but still needs maintainer resolution of its security contract, outbound-routing finding, and real behavior proof, so this issue should remain open until that PR lands or another direction is chosen.

## What This Changes

Add an opt-in global and per-binding group session scope that routes selected group conversations into an agent's main session.

## Best Possible Solution

Keep group isolation as the default, then land the linked PR only after narrowing or fixing outbound binding matching, documenting the context and sandbox boundary, and proving selected-group main folding plus per-binding opt-out in a real channel setup.

## Reproduction Assessment

Yes at source level: current main deterministically routes every non-direct peer to an isolated group/channel key and exposes no groupScope configuration. This is a missing capability rather than broken existing behavior.

## Solution Assessment

Unclear. A per-binding opt-in is the narrowest match for selected trusted groups, but maintainers still need to decide whether a global default belongs in the contract and require consistent outbound matching, thread delivery, sandbox behavior, and security guidance.

## Risks / Open Questions

- Folding a group into the main session intentionally carries multi-user group content into later main-session interactions and removes the current non-main sandbox boundary for that conversation.
- A global groupScope default has a wider misconfiguration blast radius than the selected-group use case; maintainers must decide whether the permanent contract needs both global and per-binding controls.
- The linked implementation must preserve identical inbound and outbound session resolution for bindings that match guild, team, parent, or role metadata, while retaining correct reply and thread/topic targeting.
