## Summary

Keep open: current main and v2026.6.6 still only support DM-specific session scoping, while group/channel peers remain isolated. The open closing PR is the active implementation candidate, so this issue should stay open until that PR lands or maintainers choose a broader canonical direction.

## What This Changes

The issue requests a `groupScope`-style session routing option so selected group chats can use the agent main session instead of isolated per-group session keys.

## Best Possible Solution

Keep this issue open while maintainers review https://github.com/openclaw/openclaw/pull/91702; the end state should be an additive, default-isolated routing/config contract with docs and tests for session-state, reply routing, and trust-boundary behavior.

Unclear as a final product direction. The linked PR is a plausible focused implementation, but the best solution still needs maintainer review of naming, defaults, docs, tests, and the group-to-main security boundary.
