## Summary

## What This Changes

Request adding an opt-in `groupScope` session setting so selected group conversations can share the agent main session instead of always using isolated group session keys.

## Best Possible Solution

Review https://github.com/openclaw/openclaw/pull/91702 to either land a secure opt-in group-scope contract or reject that approach and keep this issue as the focused product decision point.

## Reproduction Assessment

Yes for the source-level gap: current main applies `dmScope` only to direct peers, and group/channel peers still resolve to isolated `agent:<agentId>:<channel>:<peerKind>:<peerId>` keys. This is a missing feature rather than a broken existing contract.

## Solution Assessment

Unclear as a final implementation shape. A selective opt-in group-to-main route is plausible, but maintainers still need to settle public config semantics, privacy posture, and the relationship to broader shared-session trackers.
