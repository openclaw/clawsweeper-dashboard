## Summary

Keep open. Current main still only supports DM-specific session scoping; group/channel peers remain isolated in source, strict config schemas and public docs expose no group-to-main option, and the focused implementation at https://github.com/openclaw/openclaw/pull/47234 is still open rather than merged.

## What This Changes

Add an opt-in groupScope-style configuration so selected group/channel chats can share the agent main session while other groups remain isolated.

## Best Possible Solution

Define an additive opt-in routing/config contract that preserves default group isolation, lets selected group/channel peers use main or named shared sessions, and updates schemas, generated config metadata, docs, privacy guidance, routing/reply tests, and channel-specific paths.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main still isolates non-DM peers and has no group-to-main config surface.

## Solution Assessment

Unclear whether the literal `groupScope` field is the best final API. The maintainable direction is an explicit opt-in per-binding or named shared-session contract aligned with privacy, sandbox, queue, and reply-routing boundaries.
