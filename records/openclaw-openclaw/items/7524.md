## Summary

Current main still isolates every non-direct peer, so the requested behavior remains missing. The open implementation at https://github.com/openclaw/openclaw/pull/91702 explicitly closes this issue; keep the issue open while maintainers resolve that PR’s routing-parity, security-posture, and real-behavior-proof requirements.

## What This Changes

Add global and per-binding group session scoping so selected group conversations can share an agent’s main session.

## Best Possible Solution

Land an isolated-by-default, explicit opt-in group-session contract through the linked PR after inbound and outbound binding matching are consistent, shared-context exposure is documented and audited, and redacted real group-chat proof demonstrates main, isolated, and per-binding override behavior.

## Reproduction Assessment

Yes, at source level. Current main deterministically allows direct peers to fold through `dmScope` while every non-direct group or channel peer receives an isolated session key.

## Solution Assessment

The isolated-by-default global plus per-binding option is a reasonable direction, but it is not merge-ready until the linked PR proves inbound/outbound routing parity and establishes the security, audit, and documentation posture for shared group/main context.
