## Summary

Keep open. Current main still only supports DM session scoping; group/channel peers remain isolated, schemas/types/docs expose no group-to-main option, and #47234 is an open implementation candidate rather than a merged fix.

## What This Changes

Request a groupScope-style config option so selected group/channel chats can share an agent's main session instead of always using isolated group/channel session keys.

## Best Possible Solution

Land #47234 or an equivalent additive route-binding/session contract with schema/types/docs and regression coverage that preserves default isolation while allowing selected trusted group/channel peers to share main context.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main verifies the limitation: group/channel peers still build isolated session keys and config surfaces expose only DM scoping.

## Solution Assessment

Unclear for the exact groupScope shape. The maintainable direction is an additive, backwards-compatible routing/config contract coordinated with broader shared-session work in #19929.
