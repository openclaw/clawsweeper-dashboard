## Summary

Keep open. Current main still only supports DM-specific session scoping, while group/channel peers continue to resolve to isolated session keys; open PR #47234 is an implementation candidate, not a reason to close the issue before merge.

## What This Changes

The issue requests an opt-in `groupScope`-style configuration so selected group/channel conversations can use the agent main session instead of isolated group sessions.

## Best Possible Solution

Keep this as the focused tracker and resolve it by reviewing/landing #47234 or an equivalent additive route-binding/session contract coordinated with #19929.

## Reproduction Assessment

Yes for the current limitation. Source inspection on current main shows non-direct peers still produce isolated group/channel session keys, and the schemas/types/docs expose no group-to-main option.

## Solution Assessment

No current-main solution exists yet. The narrowest maintainable direction is a backward-compatible, opt-in routing/config contract for selected group/channel peers, with docs and regression coverage for privacy and reply-routing behavior.
