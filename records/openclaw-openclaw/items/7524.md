## Summary

Keep open. Current main and the latest release still only expose DM-specific session scoping; group/channel peers remain isolated, the focused implementation PR is still open and conflicting, and related shared-session issues are design inputs rather than a merged replacement.

## What This Changes

Request an opt-in `groupScope` or equivalent config so selected group/channel conversations can route into an agent's main session while other groups remain isolated.

## Best Possible Solution

Define an additive opt-in routing/config contract that preserves default group isolation, lets selected group/channel peers target main or a named shared session, and updates schemas, generated config metadata, docs, routing/reply tests, queue behavior, and sandbox guidance.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection on current main gives high confidence that group/channel peers still isolate and no group-to-main config surface exists.

## Solution Assessment

Unclear whether the literal `groupScope` field is the best final API. The maintainable direction is an explicit opt-in shared-session contract aligned with routing, privacy, sandbox posture, queue serialization, and reply delivery.
