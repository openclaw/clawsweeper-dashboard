## Summary

Keep open: current main and the latest release still lack a `groupScope` or equivalent group-to-main session routing surface, and https://github.com/openclaw/openclaw/pull/91702 is an open, unmerged implementation candidate rather than a landed fix.

## What This Changes

The request is to add an opt-in `groupScope` session setting so selected group chats can share the agent main session while other groups remain isolated.

## Best Possible Solution

Keep this open until a maintainer-approved, opt-in group-to-main session contract lands with isolated defaults, routing/outbound parity, schema/types/help/docs, security-audit posture, and regression coverage.

## Reproduction Assessment

Yes, at source level. Current main deterministically routes non-direct group/channel peers to isolated keys, and strict config schemas cannot express `groupScope`.

## Solution Assessment

Unclear as a final product shape. The linked PR is a plausible implementation path, but the best solution still needs maintainer agreement on the security/audit posture and exact config contract.
