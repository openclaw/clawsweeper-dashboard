## Summary

Keep open. Current main still has only DM-specific session scoping, group/channel peers remain isolated, and the active #47234 implementation candidate has not merged; broader shared-session design in #19929 also leaves naming and privacy boundaries as maintainer decisions.

## What This Changes

Request an opt-in `groupScope` or equivalent config path so selected group/channel chats can share the agent main session instead of always using isolated session keys.

## Best Possible Solution

Land #47234 or an equivalent additive route-binding/session contract that preserves default isolation while allowing selected trusted group/channel peers to share main context with tests and docs.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main verifies the limitation: group/channel peers still build isolated session keys and config surfaces expose only DM scoping.

## Solution Assessment

Unclear for the exact `groupScope` shape. The maintainable direction is a backwards-compatible, opt-in per-binding or shared-session contract coordinated with #19929 and documented privacy boundaries.
