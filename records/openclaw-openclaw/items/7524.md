## Summary

Keep open. Current main still lacks any group/channel-to-main session scope: non-direct peers remain isolated, schemas and public types are DM-only, docs still document group isolation, and open PR #47234 is an implementation candidate rather than a reason to close before merge.

## What This Changes

Request an opt-in `groupScope`-style configuration that lets selected group or channel chats route into an agent's main session while leaving other groups isolated.

## Best Possible Solution

Keep this as the focused tracker and resolve it by landing #47234 or an equivalent additive route-binding/session contract with docs and regression coverage for privacy, reply routing, and default isolation.

## Reproduction Assessment

Not applicable as a feature request. The current limitation is source-verifiable on main: group/channel peers still build isolated keys, and schemas/types/docs expose no group-to-main option.

## Solution Assessment

Unclear for the exact `groupScope` shape. The maintainable direction is an additive, backwards-compatible routing/config contract coordinated with broader shared-session work in #19929.
