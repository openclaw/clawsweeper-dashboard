## Summary

Keep open: current main and v2026.5.12 still isolate group/channel peers, the focused implementation PR remains open and unmerged, and related cleanup treats this issue as the focused tracker rather than superseding it with the broader shared-session design issue.

## What This Changes

The issue requests an opt-in groupScope-style config so selected group/channel chats can share an agent’s main session while other groups remain isolated.

## Best Possible Solution

Define an additive opt-in routing/config contract that preserves default group isolation, lets selected group/channel peers target main or a named shared session, and updates schemas, generated metadata, docs, routing/reply/outbound tests, queue behavior, and sandbox guidance.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source, docs, tests, and v2026.5.12 inspection give high confidence that group/channel peers still isolate and no group-to-main config surface exists.

## Solution Assessment

Unclear whether the literal groupScope field is the best final API. The maintainable direction is an explicit opt-in shared-session contract aligned with routing, reply delivery, privacy, sandbox posture, and queue serialization.
