## Summary

Keep open: current main still lacks a group/channel-to-main session option, and prior maintainer-facing discussion treats this issue as the focused tracker for the narrower groupScope request rather than something superseded by the broader shared-session issue. The request changes session-state and privacy boundaries, so it should remain a product/security-reviewed feature instead of an automated bug fix.

## What This Changes

Request an opt-in `groupScope` or equivalent config so selected group chats can share the agent main session instead of always using isolated group sessions.

## Best Possible Solution

Define an additive opt-in session-target contract that preserves default group isolation, carries the selected target through inbound and outbound routing, documents privacy and sandbox semantics, and adds focused core plus representative channel regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection gives high confidence that current main and the latest release still isolate non-direct peers and expose no group-to-main option.

## Solution Assessment

Unclear as a final implementation shape. A per-group or per-binding opt-in is plausible, but maintainers still need to choose the public config contract, privacy semantics, outbound routing behavior, and relationship to broader shared-session work.
