## Summary

Keep open: current main and v2026.5.20 still isolate group/channel peers, and the linked implementation remains an open unmerged PR. This is a product/security-sensitive session-routing feature, not a cleanup-close candidate.

## What This Changes

The issue requests an opt-in `groupScope` or equivalent per-group setting so selected group or channel conversations can share an agent's main session while others remain isolated.

## Best Possible Solution

Define an additive opt-in session-target contract that preserves default group isolation, carries the chosen target through inbound and outbound routing, documents privacy and sandbox semantics, and adds focused core plus representative channel regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config gap. Source, docs, and v2026.5.20 inspection give high confidence that current non-direct peers remain isolated and no group-to-main option exists.

## Solution Assessment

Unclear as a final implementation shape. A per-group or per-binding opt-in is plausible, but maintainers still need to choose the public config contract, privacy semantics, outbound routing behavior, and relationship to broader shared-session work.
