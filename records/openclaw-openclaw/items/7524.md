## Summary

Keep open. Current main still has only DM-specific session scoping; group/channel peers remain isolated by default, there is no shipped `groupScope` or equivalent config surface, and the focused implementation in https://github.com/openclaw/openclaw/pull/47234 is still an open PR rather than a merged fix.

## What This Changes

The issue requests an opt-in `groupScope`-style setting so selected group chats can use the agent main session instead of isolated per-group session keys.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/47234 or an equivalent additive, opt-in routing/session contract that preserves default isolation and documents privacy, queueing, sandbox, outbound, and channel-specific behavior.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence the limitation still exists because group/channel peers build isolated keys and the public config surfaces expose only DM scoping.

## Solution Assessment

Unclear whether the literal `groupScope` field is the final best shape. The maintainable direction is a backwards-compatible, opt-in per-binding or shared-session contract coordinated with https://github.com/openclaw/openclaw/issues/19929 and backed by privacy documentation and regression coverage.
