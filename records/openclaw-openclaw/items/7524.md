## Summary

Keep open. Current main still lacks opt-in group/channel-to-main session scoping: routing, config schemas/types, docs, and continuity tests all preserve isolated non-direct sessions, and open PR #47234 is still an implementation candidate rather than a merged fix.

## What This Changes

Request an opt-in groupScope-style config that lets selected group/channel chats share the agent main session while leaving other groups isolated.

## Best Possible Solution

Keep this as the focused tracker and resolve it by landing #47234 or an equivalent additive route-binding/session contract with docs and regression coverage for default isolation, selected shared context, reply delivery, and privacy boundaries.

## Reproduction Assessment

Not applicable as a feature request. The current limitation is source-verifiable on main: group/channel peers still build isolated keys and schemas/types/docs expose no group-to-main option.

## Solution Assessment

Unclear for the exact groupScope shape. The maintainable direction is an additive, backwards-compatible routing/config contract coordinated with broader shared-session work in #19929.
