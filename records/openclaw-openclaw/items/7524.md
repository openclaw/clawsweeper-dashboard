## Summary

Keep open. Current main still isolates group/channel peers, the route/config/docs surface remains DM-only, and the related issue/PR context shows this is still the focused tracker with an open but incomplete implementation candidate rather than implemented or superseded work.

## What This Changes

The issue requests an opt-in group/session scoping option so selected group or channel chats can share the agent main session instead of always using isolated group/channel session keys.

## Best Possible Solution

Define an additive opt-in routing/config contract that preserves default group/channel isolation, lets selected group/channel peers target main or a named shared session, and updates schemas, generated metadata, docs, routing/reply tests, queue behavior, and sandbox guidance.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection on current main gives high confidence that group/channel peers still isolate and no group-to-main config surface exists.

## Solution Assessment

Unclear whether the literal groupScope field is the best final API. The maintainable direction is an explicit opt-in shared-session target aligned with routing, privacy, sandbox posture, queue serialization, reply delivery, and the open implementation PR review.
