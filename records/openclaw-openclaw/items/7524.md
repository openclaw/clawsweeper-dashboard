## Summary

Keep open: current main and v2026.5.12 still have only DM-specific session scoping, group/channel peers still resolve to isolated session keys, and the closest implementation PR remains open with unresolved product/security review.

## What This Changes

Request an opt-in `groupScope` or equivalent session-routing option so selected group chats can share an agent's main session while other groups remain isolated.

## Best Possible Solution

Define an additive opt-in session-target contract that preserves default group/channel isolation, carries the chosen session target through inbound and outbound routing, documents privacy and sandbox semantics, and adds focused core plus representative channel regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection gives high confidence that current main and v2026.5.12 still isolate non-direct peers and expose no group-to-main option.

## Solution Assessment

Unclear as a concrete implementation. The per-group or per-binding opt-in direction is plausible, but maintainers still need to choose the public config shape, privacy semantics, sandbox posture, outbound routing behavior, and representative channel coverage.
