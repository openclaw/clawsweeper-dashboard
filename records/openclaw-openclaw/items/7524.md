## Summary

Keep open: current main still only supports DM-specific session scoping while group/channel peers resolve to isolated session keys, and the focused implementation path is represented by an open, unmerged PR rather than shipped code. The request is a valid feature/config gap, but it changes session-state and privacy boundaries, so it needs maintainer product and security review before implementation.

## What This Changes

The issue requests an opt-in `groupScope` or equivalent config so selected group or channel chats can share the agent's main session instead of always using isolated group/channel session keys.

## Best Possible Solution

Define an additive opt-in session-target contract that preserves default group isolation, carries the chosen target through inbound and outbound routing, documents privacy and sandbox semantics, and adds focused core plus representative channel regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection gives high confidence that current main still isolates non-direct peers and exposes no group-to-main option.

## Solution Assessment

Unclear as a final implementation shape. A per-group or per-binding opt-in is plausible, but maintainers still need to choose the config contract, privacy semantics, outbound routing behavior, and relationship to broader shared-session work.
