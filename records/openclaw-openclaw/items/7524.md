## Summary

Keep this issue open: current main and the latest v2026.6.8 release still only expose DM-specific session scoping, while group and channel peers remain isolated. The linked implementation PR at https://github.com/openclaw/openclaw/pull/91702 explicitly closes this issue but is still open, so the issue should remain the focused tracker until that PR lands or maintainers choose a different product direction.

## What This Changes

Requests an opt-in group session scope so selected group chats can share the agent main session while isolated group sessions remain the default.

## Best Possible Solution

Land or deliberately reject a backwards-compatible, opt-in group session routing contract with inbound/outbound parity, docs, security guidance, and regression coverage; otherwise keep this focused tracker open behind the linked PR.

## Reproduction Assessment

Yes, at source level. Current main deterministically derives isolated keys for non-direct group/channel peers and the strict schemas cannot express a group-to-main option.

## Solution Assessment

Unclear as a final product path. groupScope is a plausible additive fix and has an open implementation PR, but maintainers still need to settle the security posture and exact config scope before this issue can close.
