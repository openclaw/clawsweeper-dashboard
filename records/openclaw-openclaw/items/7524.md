## Summary

Keep open: current main and the latest release still route non-direct group/channel peers to isolated session keys, while the requested group-to-main behavior remains an unlanded, security-sensitive config/product decision with an open implementation PR still under review.

## What This Changes

The issue requests an opt-in groupScope-style config so selected group chats can share an agent's main session instead of always using isolated group session keys.

## Best Possible Solution

Define an additive opt-in session-target contract that preserves default group isolation, carries origin metadata through replies, documents privacy behavior, and adds focused routing/config/docs tests before landing or replacing the open implementation PR.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config gap. Source, docs, and tests give high confidence that current main still isolates non-direct peers and exposes no group-to-main option.

## Solution Assessment

Unclear as a final implementation shape. A per-group or per-binding opt-in is plausible, but maintainers still need to choose the public config contract, privacy semantics, outbound routing behavior, and relationship to https://github.com/openclaw/openclaw/issues/19929 and https://github.com/openclaw/openclaw/pull/47234.
