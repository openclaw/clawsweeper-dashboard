## Summary

Keep open. Current main and the latest release still only support DM-specific session scoping; group/channel peers remain isolated, while the linked implementation PR is still open, conflicting, and awaiting proof/security/product review.

## What This Changes

The issue requests an opt-in group session scope so selected group chats can share the agent main session instead of always using isolated group session keys.

## Best Possible Solution

Keep this issue open as the focused tracker, and resolve it by landing https://github.com/openclaw/openclaw/pull/91702 or a successor that adds an opt-in, backward-compatible group/shared-session contract with routing, outbound, config, docs, tests, and security-audit coverage.
