## Summary

Keep open. Current main and the latest release still have only DM-specific session scoping, group/channel peers still resolve to isolated session keys, and the related implementation PR is open but unmerged with product/security questions still unresolved.

## What This Changes

Request an opt-in groupScope-style config so selected group or channel chats can share an agent's main session while default group isolation remains unchanged.

## Best Possible Solution

Keep this as the focused tracker and design an additive, backward-compatible session target contract for selected group/channel peers, coordinating with https://github.com/openclaw/openclaw/issues/19929 and reviewing https://github.com/openclaw/openclaw/pull/47234 before landing.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source and latest-release inspection give high confidence that current behavior still lacks the requested group-to-main option.

## Solution Assessment

Unclear. The per-selected-group direction is plausible, but the best maintainable solution needs maintainer agreement on the config contract, privacy boundaries, sandbox behavior, and reply-routing semantics.
