## Summary

Keep open. Current main and the latest release still have only DM-specific session scoping; group/channel peers resolve to isolated session keys, strict config schemas/types expose no group-to-main option, and the focused maintainer review plus related open PR show this still needs product/security follow-up rather than cleanup closure.

## What This Changes

Request an opt-in `groupScope` or equivalent config that lets selected group chats share the agent main session while leaving other groups isolated.

## Best Possible Solution

Keep this as the focused tracker and design an additive, backward-compatible session target contract for selected group/channel peers, with schemas/types/docs, inbound/outbound/topic routing, queue/runtime-policy semantics, and regression/security coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source and latest-release inspection give high confidence that current behavior still lacks the requested group-to-main option.

## Solution Assessment

Unclear. The per-selected-group direction is plausible and supported by user comments, but the best maintainable solution needs maintainer product/security agreement on the session API and trust-boundary semantics before implementation lands.
