## Summary

Keep open: current main still only exposes DM-specific session scoping, while group/channel peers continue to resolve to isolated session keys. The related implementation PR is open and unmerged, and this remains a product/security-sensitive config feature rather than shipped behavior.

## What This Changes

The issue requests an opt-in `groupScope`-style config so selected group or channel chats can share the agent main session instead of always using isolated peer sessions.

## Best Possible Solution

Define an additive opt-in session-target contract that preserves default group isolation, carries the chosen target through inbound and outbound routing, documents privacy/sandbox semantics, and adds focused core plus representative channel regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config gap. Source inspection gives high confidence that current main still isolates non-direct peers and exposes no group-to-main option.

## Solution Assessment

Unclear as a final implementation shape. A per-group or per-binding opt-in is plausible, but maintainers still need to choose the public config contract, privacy semantics, outbound routing behavior, and relationship to broader shared-session work.
