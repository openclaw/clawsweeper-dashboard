## Summary

Keep this issue open: current `main` and the latest release still expose only DM-specific session consolidation, while the live implementation path is the open closing PR at https://github.com/openclaw/openclaw/pull/91702.

## What This Changes

The issue requests a `groupScope`-style session option so selected group chats can route into an agent's main session instead of always using isolated group session keys.

## Best Possible Solution

Review https://github.com/openclaw/openclaw/pull/91702 to either land a secure opt-in group-scope contract or reject it and keep this issue as the product decision point.

## Reproduction Assessment

Yes for the source-level gap: current `main` applies `dmScope` only to direct peers, and group/channel peers still resolve to isolated `agent:<agentId>:<channel>:<peerKind>:<peerId>` keys. This is a missing feature rather than a broken existing contract.

## Solution Assessment

Unclear as a final implementation shape. A selective opt-in group-to-main route is plausible, but maintainers still need to settle the public config semantics, privacy posture, and relationship to the broader shared-session trackers.
