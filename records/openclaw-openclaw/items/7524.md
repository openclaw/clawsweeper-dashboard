Keep open: current main and v2026.5.12 still only support DM-specific session scoping, while group/channel peers remain isolated and the related implementation PR is open with product/security concerns unresolved.

## What This Changes

The issue requests an opt-in config surface so selected group chats can route into an agent's main session instead of always using isolated group sessions.

## Best Possible Solution

Define an additive opt-in session-target contract that preserves default group/channel isolation, carries the selected target through routing and outbound paths, documents privacy and sandbox semantics, and adds focused config/routing/channel regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source inspection gives high confidence that current main and v2026.5.12 lack a group-to-main option and still isolate non-direct peers.

## Solution Assessment
