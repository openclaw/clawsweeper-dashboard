## Summary

Keep open. Current main still has only DM-specific session scoping, group/channel peers remain isolated, the docs/tests preserve that behavior, and the related implementation at https://github.com/openclaw/openclaw/pull/47234 is still open rather than merged. The discussion also treats this issue as the focused tracker: https://github.com/openclaw/openclaw/issues/59739 was closed as a duplicate of this request, while https://github.com/openclaw/openclaw/issues/19929 and https://github.com/openclaw/openclaw/issues/51805 cover broader or distinct shared-session variants.

## What This Changes

Add an opt-in group/session scope so selected group or channel chats can route into the agent main session instead of isolated group/channel session keys.

## Best Possible Solution

Define an additive, opt-in routing/config contract that keeps default group isolation, lets selected group/channel peers use main or a named shared session, updates schemas/help/docs, and adds regression coverage across routing, outbound delivery, channel overrides, queueing, and privacy guidance.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an enhancement. Source inspection gives high confidence that current main still lacks a group-to-main session scope and continues to isolate group/channel peers.

## Solution Assessment

Unclear whether the literal groupScope shape is the best final API. The maintainable direction is a documented opt-in per-binding or shared-session contract aligned with existing routing config and privacy boundaries.
