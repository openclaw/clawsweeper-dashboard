## Summary

Keep this issue open: current main and the latest release still only support DM-specific session consolidation, while selected group-to-main routing remains an unlanded public config and trust-boundary decision.

## What This Changes

The issue requests a `groupScope`-style option that lets selected group chats route into an agent's main session instead of always using isolated group session keys.

## Best Possible Solution

Design an additive opt-in session-routing contract that preserves isolated defaults, carries origin metadata for correct replies, documents privacy behavior, and adds routing/config/docs regression coverage before landing.

## Reproduction Assessment

Yes for the source-level gap: any non-direct group/channel peer still resolves to an isolated `agent:<agentId>:<channel>:<peerKind>:<peerId>` key, and the strict schema/types/docs expose no `groupScope`-style option. This is a missing feature rather than a broken existing contract.

## Solution Assessment

Unclear as a final implementation shape. A selective opt-in group-to-main route is plausible, but maintainers still need to choose the public config contract, privacy semantics, outbound routing behavior, and relationship to the broader shared-session issues.
