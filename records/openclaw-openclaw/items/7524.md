Keep this issue open: current main and the latest release still only support DM-specific session scoping, while the linked implementation pull request is open and unmerged. ## What This Changes

The issue requests a `groupScope`-style session-routing option so selected group chats can use the agent main session instead of always using isolated group session keys.

## Best Possible Solution

Keep this issue as the focused tracker and resolve it by landing or revising the linked groupScope implementation with default isolation, explicit opt-in config, security/audit docs, and inbound/outbound routing proof.

## Reproduction Assessment

Yes, at source level. Current main deterministically derives isolated keys for non-direct group/channel peers and the strict config schemas cannot express a group-to-main option.

## Solution Assessment

Unclear as a final product shape. `groupScope` is a plausible narrow solution, but the permanent contract still needs maintainer review alongside the broader shared/canonical-session proposals and the open implementation PR.
