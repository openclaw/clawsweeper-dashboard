## Summary

Keep open: current main and the latest shipped release still have only DM-specific session scoping, while selected group/channel-to-main routing remains an unlanded, security-sensitive product decision with an open implementation PR under review.

## What This Changes

Request an opt-in groupScope or equivalent session-routing option so selected group chats can use an agent's main session while other groups stay isolated.

## Best Possible Solution

Define an additive opt-in session-target contract that preserves default group isolation, carries origin metadata through routing and replies, documents privacy behavior, and adds focused core plus representative channel regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config gap. Source, docs, release, and schema inspection give high confidence that current non-direct peers remain isolated and no group-to-main option exists.

## Solution Assessment

Unclear as a final implementation shape. A per-group or per-binding opt-in is plausible, but maintainers still need to choose the public config contract, privacy semantics, outbound routing behavior, and relationship to broader shared-session work.
