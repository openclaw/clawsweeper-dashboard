## Summary

Current main and release v2026.6.5 still isolate group peers, while the open closing PR at https://github.com/openclaw/openclaw/pull/91702 owns the focused implementation; keep this issue open until that PR lands or maintainers choose a different security and product contract.

## What This Changes

Add an opt-in session scope that routes selected group chats into an agent’s main session instead of isolated group sessions.

## Best Possible Solution

Review and either land or revise https://github.com/openclaw/openclaw/pull/91702 as an additive, default-isolated per-binding capability with explicit context-sharing warnings and consistent inbound, outbound, queue, thread, and sandbox behavior; leave broader named shared sessions to https://github.com/openclaw/openclaw/issues/19929.

## Reproduction Assessment

Yes, at source level: current main deterministically folds only direct peers according to `dmScope`, while group and channel peers remain isolated and strict config exposes no group equivalent.

## Solution Assessment

Unclear pending maintainer review: per-binding opt-in is the safest apparent direction, but the linked PR’s global option, trust-boundary documentation, and relationship to broader shared sessions require a product decision.
