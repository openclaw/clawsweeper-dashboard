## Summary

This PR should stay open because the intent is still useful, but the patch is not merge-ready: it broadens cron error suppression from typed non-terminal metadata to a rendered-text heuristic and still lacks real after-fix cron/scanner proof.

## What This Changes

The PR changes cron isolated-agent payload classification so trailing rendered search-failure payloads can be treated as nonfatal after prior deliverable cron output, with helper tests for search-miss and non-search-error cases.

## Best Possible Solution

Carry optional or non-terminal search-miss intent from the producer into payload metadata, add a regression for partial/progress text followed by a required search failure, and require real cron-run proof before merge.

## Reproduction Assessment

Yes, source inspection gives a high-confidence helper-level reproduction: current main marks a final error payload fatal, while the PR changes success text followed by a matching search-failure payload to nonfatal. I did not run tests because this review is read-only and test commands can write artifacts.

## Solution Assessment

No, this is not the best fix yet because it infers optional scanner intent from rendered text plus any prior deliverable payload. The safer fix is to preserve typed optional or non-terminal metadata from the producer and test the required-search failure case.
