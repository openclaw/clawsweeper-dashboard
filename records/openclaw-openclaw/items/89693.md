## Summary

This is a focused cron bug-fix PR, but it should stay open because the new rendered-text heuristic can downgrade required terminal search failures after partial output and the PR still lacks real cron/scanner proof.

## What This Changes

The PR changes isolated cron payload outcome handling so trailing rendered scanner search-miss error payloads can be treated as nonfatal after earlier deliverable output.

## Best Possible Solution

Carry typed optional or non-terminal search-miss evidence from the producer into cron outcome resolution, cover required-search-after-progress, and add redacted real cron/scanner proof before merge.

## Reproduction Assessment

Yes, at source level. A payload list with earlier partial text followed by a required rendered search failure would match the new predicate and clear fatal structured-error handling without typed optionality proof.

## Solution Assessment

No. The text regex is only a plausible mitigation; the safer fix is to carry typed optional or non-terminal failure metadata from the producer and keep required trailing search failures fatal.
