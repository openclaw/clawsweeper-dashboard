## Best Possible Solution

Use a typed optional/non-terminal search-miss signal from the producer, or align with the broader cron recovered-error contract, then add regression coverage and redacted real cron/scanner proof.

## Reproduction Assessment

Yes, at source level. Current main marks a final error payload fatal, while the PR's proposed rule has a clear counterexample: progress or partial text followed by a required matching search failure would be downgraded.

## Solution Assessment

No. The proposed text regex plus any prior deliverable output is not the best fix; the safer boundary is typed optional/non-terminal metadata or a broader resolver contract that proves recovery.
