## Summary

Keep open: the linked gateway compaction crash-loop is plausible and freshly documented, but this PR does not implement the advertised 60-second post-compaction hard timeout and lacks after-fix real behavior proof.

## What This Changes

The branch adds `setImmediate` yields before two post-compaction retry `continue` paths and raises the supervised gateway lock retry timeout from 30s to 120s.

## Best Possible Solution

Implement a source-proven bounded post-compaction retry or cancellation path, or another narrow fix chosen after reproducing the managed Debian/systemd hang, with regression coverage and live proof.

## Reproduction Assessment

No high-confidence live reproduction for the full linked issue in this review; the source path and logs make the failure plausible, but the Debian/systemd/MiniMax long-session scenario was not run. For the PR defect, source inspection is high-confidence: the diff does not add the advertised abort timer.

## Solution Assessment

No: this is not the best way to solve the issue because the patch only yields once before retrying and does not bound the retry attempt it claims to hard-timeout. The safer path is an explicit, tested first-post-compaction retry abort or a reproduced alternative fix at the real hang boundary.
