## Summary

Keep open: current main still has a credible source path where channel-routed non-default agents enter the embedded runner per turn, and the reporter added current-release Gateway timing plus trace-stage evidence after the prior review.

## What This Changes

The issue asks OpenClaw to avoid per-message embedded_run setup for channel-routed non-default agents by reusing a warm runtime or using a direct session path.

## Best Possible Solution

Define a supported warm-runtime contract for channel-routed non-default agents and amortize model, tool, prompt, and session setup without weakening agent isolation.

## Reproduction Assessment

Yes for the runtime path: reporter logs show repeated non-default Gateway embedded-run setup, and current main still routes generic non-CLI replies through runEmbeddedPiAgent. The exact Feishu/current-main 10-17s magnitude was not live-reproduced here.

## Solution Assessment

Unclear. Direct-session parity is plausible, but the maintainable solution needs an explicit runtime/session reuse boundary rather than a Feishu-specific shortcut or broad cache.
