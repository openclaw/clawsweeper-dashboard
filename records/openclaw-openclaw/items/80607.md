## Summary

Current main still appears to have the reported performance boundary: channel-routed non-default sessions can resolve to their agent/workspace and then go through the generic embedded runner, where model resolution and tool construction happen per invocation. The reporter’s fresh Gateway timings make this worth keeping open, but the fix is not a safe one-shot repair because the correct warm-session or ACP/direct-session contract needs maintainer design.

## What This Changes

The issue asks for channel-routed non-default agents to avoid per-message embedded-run setup and get runtime/session reuse comparable to the default agent path.

## Best Possible Solution

Define a supported warm-runtime or direct-dispatch contract for channel-routed non-default agents that amortizes model, tool, prompt, and resource setup while preserving per-agent isolation and policy boundaries.

## Reproduction Assessment

Yes at source/log level, but not as a live current-main benchmark. The reporter supplied Gateway timing and trace-stage evidence, and current source still shows non-ACP, non-CLI channel replies flowing into per-call embedded-runner setup.

## Solution Assessment

Unclear. Direct-session parity is plausible, but the maintainable solution needs a maintainer decision on whether to route these turns through ACP dispatch, a generic warm embedded-runtime cache, or another prepared-runtime contract.
