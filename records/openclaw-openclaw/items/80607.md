## Summary

Keep open: current main still routes the reported non-default agent shape through the embedded runner, and the reporter has since supplied gateway timing plus trace-stage evidence. This is not cleanup-closeable because the right fix needs runtime reuse and isolation design.

## What This Changes

Request to remove repeated embedded-run setup latency for non-default multi-agent sessions by reusing warm runtime/session state or using the default agent's direct session path.

## Best Possible Solution

Define a supported warm-session or runtime-reuse contract for channel/CLI-routed non-default agents, with explicit invalidation and isolation semantics, then optimize the measured embedded startup stages without regressing per-agent state separation.

## Reproduction Assessment

No high-confidence live reproduction was run here. The issue now has reporter-supplied gateway timings and trace logs, and current source confirms routed non-default OpenAI-compatible runs still enter the per-run embedded startup/tool path, so the behavior is source-reproducible enough to keep open.

## Solution Assessment

Unclear. Reusing the default agent's direct session path may be attractive, but the safer maintainable solution is a deliberate non-default-agent runtime reuse design that preserves workspace, auth, session, plugin, and delivery isolation.
