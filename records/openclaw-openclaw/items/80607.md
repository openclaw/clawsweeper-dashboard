## Summary

Keep open: current source still supports the reported channel-routed non-default-agent path into per-turn embedded-run setup, and the reporter added current release gateway/trace timing that narrows the hot stages rather than making this cleanup-closeable.

## What This Changes

The issue asks for non-default agent turns to reuse a warm runtime or direct session path instead of paying repeated embedded-run setup latency on each message.

## Best Possible Solution

Define the supported runtime contract for channel-routed non-default agents, then remove repeated warm-turn embedded startup/prep work while preserving per-agent workspace, auth, session, and delivery isolation.

## Reproduction Assessment

Yes for a maintainer with a two-agent gateway setup: run the same model through default and non-default agents with trace logging and compare `[trace:embedded-run]` stages. I did not execute that live setup here, but current source and the reporter's v2026.5.7 traces provide a concrete reproduction path.

## Solution Assessment

Unclear: direct-session parity may be the best end state, but a safer first maintainer step is to measure warm non-default turns and choose between runtime reuse, embedded startup caching, or a documented policy change.
