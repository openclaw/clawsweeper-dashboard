## Summary

Keep open: current main still routes Feishu-bound non-default agent turns through the generic reply path and embedded runner, and the reporter added a live v2026.5.7 gateway benchmark showing an 8-11s non-default-agent gap. This is a plausible runtime performance bug, but the fix direction needs maintainer design judgment rather than cleanup closure.

## What This Changes

The issue asks OpenClaw to avoid per-message embedded_run setup for non-default multi-agent sessions by reusing a warm runtime path or matching the default agent's direct-session behavior.

## Best Possible Solution

Define the supported runtime contract for channel-routed non-default agents, then remove repeated warm-turn embedded setup while preserving per-agent workspace, auth, sandbox, and session isolation.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? No, not from this review alone: the reporter supplied a live v2026.5.7 gateway benchmark and current source reproduces the routed embedded-run path, but the exact latency still needs a current-main Feishu or gateway trace run.

## Solution Assessment

Is this the best way to solve the issue? Unclear: direct-session parity may be right, but the safer maintainer path is to capture embedded-run and plugin-tool trace timings first, then optimize the canonical prepared runtime/session path rather than adding ad hoc caches or a policy knob prematurely.
