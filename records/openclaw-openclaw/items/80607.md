## Summary

Keep open: the reporter added gateway-mode and trace-level timing data after the prior review, and current main still routes the relevant non-default Feishu turn through per-run embedded startup, model resolution, auth, and tool construction rather than a proven warm direct-session path.

## What This Changes

The issue asks OpenClaw to avoid per-message embedded-run setup for channel-routed non-default agents, either by reusing runtime/session state or using the same warm direct path as the default agent.

## Best Possible Solution

Define the non-default channel-session runtime contract and remove repeated warm-turn embedded startup work while preserving per-agent workspace, auth, tool, transcript, and channel isolation.

## Reproduction Assessment

No, not as a live current-main repro in this review. Source inspection shows the routed non-default Feishu path still reaches per-run embedded startup, and the reporter supplied gateway/trace evidence on 2026.5.7, but a maintainer should confirm the exact latency on current main with the same Feishu setup.

## Solution Assessment

Unclear. Direct-session parity may be correct, but the safer maintainable path is to first decide the runtime reuse/prepared-runtime boundary and then optimize the measured per-run stages without leaking state across agents or sessions.
