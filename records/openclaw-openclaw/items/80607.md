## Summary

Keep open: current main still routes generic channel replies for non-CLI, non-ACP runtimes through the embedded runner, and the reporter added fresh Gateway and trace timings showing the non-default agent pays embedded-run setup on each turn.

## What This Changes

The issue asks OpenClaw to avoid per-message `embedded_run` setup for channel-routed non-default agents by reusing a warm runtime/session or matching the default agent's direct path.

## Best Possible Solution

Define a supported warm-runtime/session reuse path for channel-routed non-default agents that amortizes setup while preserving per-agent workspace, auth, tool policy, delivery context, and isolation.

## Reproduction Assessment

Yes, source/log-reproducible: the issue provides Gateway-mode default-vs-non-default timings and trace-stage logs, and current source still routes generic non-CLI replies into `runEmbeddedPiAgent`. I did not run a live Feishu/Gateway reproduction on current main.

## Solution Assessment

Unclear. Direct-session parity is plausible, but the maintainable fix should first decide whether this belongs in ACP dispatch, prepared runtime reuse, scoped caching, or another generic runtime contract.
