## Summary

Keep open: current main still has a credible source path where routed non-default, non-ACP, non-CLI Gateway replies enter embedded-run setup, and the reporter added fresh Gateway timing plus trace-stage evidence after the earlier review.

## What This Changes

The issue asks for non-default channel-routed agents to avoid per-message embedded-run setup latency by reusing a warm runtime/session path comparable to the default agent.

## Best Possible Solution

Define a supported warm-runtime contract for channel-routed non-default agents and amortize model, tool, and prompt setup while preserving per-agent isolation and routing semantics.

## Reproduction Assessment

Yes, source/log-reproducible: the reporter supplied Gateway-mode default-vs-non-default timings and trace stages, and current source still shows generic non-ACP, non-CLI routed replies entering runEmbeddedPiAgent. I did not run a live Feishu/Gateway reproduction here.

## Solution Assessment

Unclear. Direct-session parity is plausible, but the maintainable fix should first decide whether this belongs in ACP dispatch, prepared embedded-runtime reuse, scoped caching, or another runtime contract.
