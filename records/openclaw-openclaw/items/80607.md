## Summary

Keep open: the updated reporter benchmark gives useful live v2026.5.7 timing evidence, and current main still routes Feishu-bound non-default agent turns through the shared dispatch path into embedded-run tool preparation. This needs maintainer runtime/performance design rather than cleanup closure.

## What This Changes

The issue reports that channel-routed non-default agents repeatedly use the embedded run path and add roughly 8-17 seconds of per-message latency compared with the default agent.

## Best Possible Solution

Define the supported runtime contract for channel-routed non-default agents, profile current main with embedded-run and plugin-tool traces, then remove repeated warm-turn setup while preserving per-agent workspace, auth, tool, and session isolation.

## Reproduction Assessment

No, not as a high-confidence current-main live repro. Source inspection confirms the routed non-default Feishu path still enters embedded-run preparation, and the reporter added v2026.5.7 Gateway timing data, but current-main timing needs a real trace run.

## Solution Assessment

Unclear. Direct-session parity may be right, but maintainers should first measure the routed session's embedded prep stages and decide whether to optimize plugin/tool warm paths, runtime reuse, or the channel-routed runtime contract.
