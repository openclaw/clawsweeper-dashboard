## Summary

Keep open: current main still has the same credible path where channel-routed non-default agents resolve to the routed agent/workspace but non-CLI execution enters a fresh embedded run, and the reporter added recent Gateway timing plus trace-stage evidence after the prior review.

## What This Changes

Reduce per-message latency for channel-routed non-default agents by avoiding fresh embedded-run setup and reusing or directly dispatching a warm session like the default agent path.

## Best Possible Solution

Define a supported warm-runtime contract for channel-routed non-default agents and amortize model, tool, and prompt setup without weakening per-agent isolation.

## Reproduction Assessment

Yes, source/log-reproducible: the reporter supplied Gateway-mode default-vs-non-default timings and trace stages, and current source still shows generic non-CLI reply runs entering runEmbeddedPiAgent. I did not run a live Feishu/Gateway reproduction against current main.

## Solution Assessment

Unclear. Direct-session parity is plausible, but the maintainable fix should first decide whether this belongs in ACP dispatch, prepared runtime reuse, scoped caching, or another runtime contract rather than a Feishu-specific shortcut.
