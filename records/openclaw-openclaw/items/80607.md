## Summary

Keep open. Current main still routes Feishu configured/bound non-default sessions through the generic reply runner and embedded PI attempt path, while the adjacent open perf trackers do not terminally own this Feishu/non-default-agent routing parity case.

## What This Changes

The issue asks OpenClaw to avoid repeated embedded-run initialization for non-default Feishu-bound agents by reusing the same warm/direct session path as the default agent or otherwise amortizing warm turns.

## Best Possible Solution

Define and implement one canonical warm runtime path for Feishu bound non-default channel sessions that preserves binding, reply delivery, tool, and security contracts while avoiding repeated warm-turn prep, then prove it with Feishu multi-agent regression coverage and timing evidence.

## Reproduction Assessment

No high-confidence live reproduction is available from this review. Source inspection does show the reported Feishu configured-binding path reaching the generic reply runner and embedded PI attempt, but the latency requires a live Feishu multi-agent setup or a targeted harness with timing logs.

## Solution Assessment

Unclear. Reusing a direct session path may be right, but the safer maintainer direction is to first decide whether the fix belongs in Feishu binding routing, reply-run lifecycle reuse, or embedded tool-prep caching so channel and plugin contracts are not bypassed.
