## Summary

Keep open: current main still has a credible path where channel-routed non-default agents fall through to embedded-run setup, and the reporter added fresh Gateway timing plus trace-stage evidence after the prior review.

## What This Changes

The issue requests removing repeated embedded-run setup latency for channel-routed non-default agents by reusing a warm runtime or direct session path.

## Best Possible Solution

Define a supported warm-runtime contract for channel-routed non-default agents and amortize model, tool, and prompt setup without weakening agent isolation.

Yes, source/log-reproducible: the reporter supplied Gateway-mode default-vs-non-default timings and trace stages, and current source still shows generic non-ACP, non-CLI reply runs entering runEmbeddedPiAgent. I did not run a live Feishu/Gateway reproduction here.

## Solution Assessment

Unclear. Direct-session parity is plausible, but the maintainable fix should first decide whether this belongs in ACP dispatch, prepared runtime reuse, scoped caching, or another runtime contract rather than a Feishu-specific shortcut.
