## What This Changes

Fix the preflight compaction estimator so stale pre-compaction transcript usage cannot trigger repeated compactions after the active replay is small.

## Best Possible Solution

Correct and land the linked estimator fix, preserving fresh usage and post-usage tail pressure while excluding known-stale post-compaction usage, then close this issue as implemented.

## Reproduction Assessment

Yes, at source level: current main reads raw last transcript usage, uses it as the preflight fallback when token freshness is stale or missing, and projects that value into the compaction threshold. The linked PR also provides a focused runtime proof using a temporary transcript with stale 240k usage.

## Solution Assessment

Yes, the narrow estimator-and-regression-test path is the right fix, but the open PR needs the stale output-token edge case addressed before merge.
