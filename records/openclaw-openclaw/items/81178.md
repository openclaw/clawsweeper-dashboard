## Summary

Keep open: current main still has the source path where stale transcript usage can dominate the preflight compaction estimate, and the linked fixing PR is still open with review/proof blockers rather than merged.

## What This Changes

Prevent repeated early preflight compactions after a successful compaction by basing preflight estimates on active post-compaction context instead of stale transcript usage records.

## Best Possible Solution

Revise and land https://github.com/openclaw/openclaw/pull/81231 or a replacement that caps stale pre-compaction usage without dropping legitimate latest-usage plus tail pressure, with focused regression coverage and real runtime proof.

## Reproduction Assessment

Yes at source level: current main still reads the last nonzero transcript usage and treats it as a prompt-token floor when totalTokensFresh is stale. I did not run a live Discord/Pi Default session reproduction.

## Solution Assessment

The bounded stale-usage direction is appropriate, but the currently linked PR is not yet the best final patch because review found it can discard legitimate post-usage tail pressure.
