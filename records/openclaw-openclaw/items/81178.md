## What This Changes

The issue reports repeated safeguard preflight compactions after a successful compaction because stale transcript usage can dominate the active replay estimate.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81916 or an equivalent bounded-estimator fix with regression coverage, then close this issue from the merged PR.

## Solution Assessment

Yes, bounding stale raw transcript usage against the active replay estimate while preserving real post-usage tail pressure is the narrow maintainable fix direction. The issue should stay open until the linked PR or an equivalent patch merges.
