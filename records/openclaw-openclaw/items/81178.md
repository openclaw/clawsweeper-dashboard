## Summary

Keep open: current main still has the reported source path where raw transcript usage can be used as the preflight token floor after compaction, so the issue is actionable and not safely closeable.

## What This Changes

Preflight compaction should use the active post-compaction context rather than stale raw transcript usage when deciding whether to compact again.

## Best Possible Solution

Patch the preflight token estimator to prefer active post-compaction replay/current prompt estimates over stale raw transcript usage, keep byte-size triggers gated by explicit transcript rotation settings, and add a regression test for the stale-usage-after-compaction path.

## Reproduction Assessment

No live reproduction was run, but the source-level path is clear: stale total-token state re-enters transcript fallback, the fallback reads the last raw usage record, and the estimator treats that record as a floor over the active replay estimate.

## Solution Assessment

Yes, the reporter's proposed bounded-usage direction is the narrowest maintainable fix if it preserves existing conservative tail-pressure behavior and keeps explicit transcript-size compaction separate.
