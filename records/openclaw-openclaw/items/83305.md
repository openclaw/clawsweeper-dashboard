## Summary

Keep open: the proposed CI policy gate is bypassable because it runs from the PR-controlled checkout, and the market-sensitive matcher would block ordinary template-based bug-fix PRs behind an undefined approval label.

## What This Changes

The PR adds a CI regression detector gate and script that require bug-fix evidence and market-sensitive approval before pull requests proceed.

## Best Possible Solution

Rework this as a trusted base-branch metadata gate, add focused tests for bypass and template-collision cases, and define the approval-label policy before making it required.

## Reproduction Assessment

Yes for the review findings by source inspection: the workflow runs policy code from the PR checkout, and the script's bare risk matcher collides with the repository PR template. I did not run CI because this was a read-only review.

## Solution Assessment

No. A metadata enforcement gate should run trusted base code and use a narrow documented approval signal; this PR currently uses PR-controlled code and an undefined broad label policy.
