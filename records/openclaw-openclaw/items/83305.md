## Summary

Keep open: this is a focused CI policy PR with sufficient terminal proof, but it should not merge as-is because the new gate is bypassable from PR-controlled code and its approval heuristic would block ordinary template-based bug-fix PRs.

## What This Changes

The PR adds a CI regression detector gate workflow job and a Node script that require bug-fix evidence and a special approval label for market-sensitive fixes.

## Best Possible Solution

Implement this as a trusted metadata-only gate using base-branch code, then define or reuse a documented approval-label contract with a narrow market-sensitive matcher.

## Reproduction Assessment

Yes for the review findings by source inspection: the proposed workflow runs PR-controlled gate code, and the script's bare risk keyword matches the current PR template. I did not run the workflow because this was a read-only review.

## Solution Assessment

No. A PR metadata enforcement gate should run trusted base code and use a narrow, documented approval signal; the current implementation is bypassable and overbroad.
