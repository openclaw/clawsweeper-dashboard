## Summary

Keep open: the reported timeout/error visibility bug is real, but this PR still returns and caches a pending lifecycle error before the intentional retry grace window can be cancelled, so it should not merge as-is.

## What This Changes

The PR changes `waitForAgentJob` so its outer timeout checks `pendingAgentRunErrors` and returns that snapshot, with a fake-timer regression test.

## Best Possible Solution

Surface diagnostic context for short `agent.wait` or `sessions_send` timeouts without recording a pending lifecycle error as terminal before its retry grace window expires.

## Reproduction Assessment

Yes, from source inspection: current main defers lifecycle errors for 15 seconds but the outer wait timeout can resolve `null` earlier. I did not run tests because this review is read-only.

## Solution Assessment

No. The PR addresses the right race, but the implementation bypasses the existing grace-window contract; the safer fix should preserve retry cancellation while carrying diagnostic context to the timeout result.
