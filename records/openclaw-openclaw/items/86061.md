## Summary

## What This Changes

The PR changes `waitForAgentJob` so its outer timeout returns a pending lifecycle error snapshot, and adds a fake-timer regression test for that race.

## Best Possible Solution

Preserve the transient-error grace contract while adding a timeout-safe, bounded way for `agent.wait` and `sessions_send` to include recent model/provider failure context.

## Reproduction Assessment

Yes, at source level: current main queues lifecycle errors behind a 15s grace timer while the outer `waitForAgentJob` timeout can still resolve `null` first. I did not run tests because this was a read-only review.

## Solution Assessment

No: the proposed patch bypasses the current grace-window contract by returning the pending error as terminal when `timeoutMs` fires. The safer fix is to keep the grace semantics and expose failure context through a timeout-safe response path.
