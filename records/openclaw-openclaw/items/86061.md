## Summary

Keep open. The underlying bug is source-reproducible and this PR has a useful narrow fix idea, but the current branch is dirty against main, lacks real behavior proof, and caches the pending error before the existing retry-grace window has expired.

## What This Changes

The branch changes `waitForAgentJob` so the outer timeout returns a queued pending error snapshot instead of `null`, and adds a fake-timer regression test.

## Best Possible Solution

Port the useful pending-error return onto current main’s `setSafeTimeout` wait path without caching the error before grace expiry, preserve the newer wait lifecycle behavior, and require redacted runtime proof before merge.

## Reproduction Assessment

Yes, at source level: current main can observe a lifecycle error, defer its snapshot, and let the shorter outer wait timer return `null`, which becomes a generic `agent.wait` timeout. I did not run live proof because this review is read-only and tests would write artifacts.

## Solution Assessment

No for the submitted implementation as-is: checking the pending error is the right seam, but it should not record the snapshot as cached terminal state before retry grace expires, and the stale branch must be ported to current main.
