## Best Possible Solution

Preserve the transient lifecycle-error grace contract while exposing bounded, sanitized recent model/provider failure context through `agent.wait` and `sessions_send` timeout responses.

## Reproduction Assessment

Yes, source-reproducible: current main queues lifecycle errors behind a 15-second grace timer while the outer `waitForAgentJob` timeout can still resolve `null` first. Current gateway coverage also emits a lifecycle error during a short `agent.wait` and expects a timeout, matching the hidden-error failure shape.

## Solution Assessment

No: this PR fixes the symptom by returning and caching the pending error before the grace window expires. The safer solution is to keep the grace semantics and carry bounded error context through the timeout response path.
