## What This Changes

The PR changes waitForAgentJob to return a queued pending agent-run error snapshot when its outer timeout fires before the error grace timer, and adds a fake-timer regression test.

## Best Possible Solution

Preserve the transient-error grace contract while adding a narrow, tested way for timeout responses to include recent model/provider failure context when that context is safe to expose.

## Reproduction Assessment

Yes from source inspection: on current main, waitForAgentJob queues lifecycle errors in pendingAgentRunErrors, but the outer timeout still resolves null before the 15s grace timer records the snapshot. I did not run tests because this was a read-only review.

## Solution Assessment

No: the proposed fix surfaces the pending error immediately when timeoutMs fires, which bypasses the current grace-window contract meant to avoid premature terminal snapshots during auth/model failover. A safer fix should keep the grace semantics and expose failure context through a timeout-safe response path.
