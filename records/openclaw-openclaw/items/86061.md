## Summary

Keep open: the linked diagnosis is useful, but this PR should not merge as written because it turns pending lifecycle errors into terminal `agent.wait` errors before the existing retry-grace window expires, and the proof is still fake-timer test output only.

## What This Changes

The PR changes `waitForAgentJob` so the outer wait timeout returns a pending lifecycle error snapshot, and updates gateway tests to expect an error instead of a timeout in that race.

## Best Possible Solution

Preserve the existing retry-grace semantics and surface root-cause context only after the error is terminal, or add explicitly non-terminal timeout diagnostics with maintainer-approved API semantics and real gateway proof.

## Reproduction Assessment

Yes, from source inspection: current main queues lifecycle errors for the 15-second grace and the outer wait timeout returns null/timeout if it fires first. The branch's fake-timer test exercises the race, but no live gateway run was verified.

## Solution Assessment

No, not as written; it solves opacity by making pending errors terminal before the grace window. A safer fix should preserve recovery semantics and surface diagnostic context only as terminal error data or explicitly non-terminal timeout metadata.
