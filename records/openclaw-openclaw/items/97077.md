## Best Possible Solution

Normalize null and undefined ask as the same missing-context fallback across the new copy checks, add null regression coverage, and land one canonical tested fix for https://github.com/openclaw/openclaw/issues/97069.

## Reproduction Assessment

Yes. From source, current main collapses `ask=always` and one-shot persistence into the same missing-Allow-Always decision set, and the PR-introduced null bug is source-reproducible with `ask: null` plus filtered decisions.

## Solution Assessment

No, not as submitted. The ask-aware copy direction is a plausible narrow fix, but the implementation needs a consistent missing-ask helper or check so null protocol values keep the safe policy fallback.
