## Summary

This is a credible v2026.6.5 regression: OpenClaw still treats `google/gemini-3.1-pro-preview` as canonical across Google runtime code, tests, docs, and upgrade guidance, yet the shipped native Google catalog can omit it and leave a retained default unusable. No merged fix or canonical replacement item currently owns this mismatch.

## What This Changes

Fix upgrades that retain `google/gemini-3.1-pro-preview` as the resolved default even when the native Google runtime catalog cannot inspect or execute it.

## Best Possible Solution

Keep one canonical Google Gemini Pro identifier across plugin metadata, catalog listing, inspection, and execution; if Google has retired that identifier, add a focused `openclaw doctor --fix` migration and clear validation to a verified supported replacement instead of a runtime alias or silent fallback.
