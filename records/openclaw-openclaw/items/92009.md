## What This Changes

The issue asks OpenClaw to stop retaining a configured `google/gemini-3.1-pro-preview` default that cannot be found or executed after upgrading to v2026.6.5.

## Best Possible Solution

Keep the merged planner/catalog fix on main and let the next release carry it; v2026.6.5 users can switch the default model to a listed Google model until then.

## Reproduction Assessment

Yes. The reporter supplied exact v2026.6.5 CLI output, and source inspection shows the inspect/run failures follow when the generated catalog lacks the configured default model.

## Solution Assessment

Yes. Treating blank provider `baseUrl` as omitted before discovery and merge preserves bundled provider catalog rows without adding a runtime shim or new config surface.
