## Summary

Keep this PR open as the canonical fix: current main still has the Telegram fallback behavior, the patch is narrowly targeted and has real Telegram proof, but the new public command-result flag still needs a small SDK docs/scope clarification before landing.

## What This Changes

Adds a `suppressReply` plugin command result flag, makes Telegram native plugin commands skip the empty fallback when it is set, and adds a focused regression test.

## Best Possible Solution

Land this canonical PR after documenting the command-result contract and confirming maintainer approval for its SDK/channel scope, then close https://github.com/openclaw/openclaw/issues/80756.

## Reproduction Assessment

Yes. Mantis Telegram Desktop proof reproduces the current-main symptom, and source inspection shows undefined or non-renderable Telegram plugin command results flow into the empty-response fallback.

## Solution Assessment

Mostly yes. An explicit `suppressReply` result is a narrow maintainable fix for plugin-owned delivery, but the public SDK docs should define its intended channel scope before landing.
