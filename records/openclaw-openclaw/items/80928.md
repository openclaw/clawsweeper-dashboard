## What This Changes

The branch adds `suppressReply?: boolean` to plugin command results, makes Telegram native commands skip fallback delivery when that flag is returned, documents it, and adds regression coverage.

## Best Possible Solution

Maintainers should choose the command-result suppression contract, then rebase the PR and land the focused Telegram-native suppression fix with SDK docs and regression coverage.

## Reproduction Assessment

Yes. Source inspection shows current main normalizes an empty plugin command result to `{}` and then Telegram native commands fall through to `EMPTY_RESPONSE_FALLBACK`; Mantis also captured baseline/candidate Telegram Desktop proof.

## Solution Assessment

Unclear until maintainers choose the API shape. A first-class suppression result is a narrow fix for the native Telegram fallback, but exporting a Telegram-only field on generic `PluginCommandResult` needs an explicit SDK decision and a conflict-free branch.
