## Summary

Keep this PR open: the reported path is source-reproducible and the patch direction is plausible, but the external PR still lacks real Telegram proof and leaves low-priority SDK docs/test follow-ups.

## What This Changes

The PR adds a `suppressReply` command-result flag, skips Telegram's empty-response fallback when that flag is set, and replaces one Telegram native-command test with suppression coverage.

## Best Possible Solution

Land a documented no-reply command result contract with Telegram suppression, preserved DM binding coverage, and live Telegram before/after proof or an explicit maintainer proof override.

## Reproduction Assessment

Yes, source-reproducible: current main normalizes an undefined plugin command result to `{}` and Telegram native commands turn non-renderable results into the empty-response fallback. I did not establish a live Telegram reproduction in this read-only review.

## Solution Assessment

Mostly yes: an explicit result flag is a narrow way to distinguish out-of-band delivery from accidental empty output. The safer completion is to document the public contract, preserve existing binding-context coverage, and provide live Telegram proof or a maintainer override.
