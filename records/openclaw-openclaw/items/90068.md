The branch is currently conflicting and needs maintainer review/CI rather than cleanup closure.

## What This Changes

The PR snapshots bundled capability provider/backend/harness ids and tool names per captured row, skips malformed rows with diagnostics, preserves manifest tool contracts, and adds a focused bundled-runtime regression test.

## Best Possible Solution

Keep the per-row bundled capability projection fix, resolve the current conflict, and land after maintainer CI or changed-gate proof covers the plugin runtime boundary.

## Reproduction Assessment

Yes, by source inspection. Current main reads captured capability ids and tool names inside one enclosing try block, so a throwing captured getter can fail the whole bundled runtime snapshot before healthy siblings are registered.

## Solution Assessment

Yes. Snapshotting the captured identity fields once at the bundled capability boundary and reusing the manifest tool contract is narrower and better aligned with the normal registry path than plugin-specific shims.
