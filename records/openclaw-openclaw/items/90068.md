Keep open: current main and the latest release still have the old bundled capability projection path, while this protected maintainer draft contains a focused hardening fix that still needs rebase and maintainer CI review rather than cleanup closure.

## What This Changes

The branch snapshots captured bundled capability ids/tool names per row, preserves manifest contracts on synthesized plugin records, and adds a bundled-runtime regression test.

## Best Possible Solution

Rebase the draft onto current main, keep the per-row registration isolation plus manifest-contract carry, and land after maintainer CI or changed-gate proof covers the bundled plugin runtime boundary.

## Reproduction Assessment

Yes by source inspection: current main reads captured identity fields inside one enclosing try block, so a throwing captured `tool.name` or provider `id` getter can fail the whole bundled runtime snapshot. I did not run tests because this cleanup review is read-only.

## Solution Assessment

Yes: snapshotting captured string identity fields once per row fixes the boundary in one place, and carrying manifest contracts aligns the bundled path with the normal plugin registry. The related open PR at https://github.com/openclaw/openclaw/pull/81312 covers contract preservation only, not the malformed-row isolation.
