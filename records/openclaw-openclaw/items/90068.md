## What This Changes

The branch changes bundled capability runtime registration to snapshot provider/backend/harness ids and tool names per row, skip malformed captured rows with diagnostics, carry manifest tool contracts into bundled plugin records, and add a focused regression test.

## Best Possible Solution

Keep the per-row bundled capability projection fix, resolve the current main conflict, and land only after maintainer CI or changed-gate proof covers the plugin runtime boundary.

## Reproduction Assessment

Yes by source inspection. Current main reads captured ids/names directly inside one outer try in loadBundledCapabilityRuntimeRegistry, so a throwing captured tool.name or provider id getter trips the plugin-level catch; I did not run tests because this review is read-only.

## Solution Assessment

Yes, this appears to be the right layer: the bundled runtime capture boundary is where one bad captured row currently poisons the registry, and carrying manifest contracts aligns it with normal registry enforcement. The remaining blocker is conflict resolution and maintainer validation, not a concrete patch defect I found.
