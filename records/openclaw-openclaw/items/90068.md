## What This Changes

The PR snapshots bundled captured provider/backend/harness ids and tool names per row, skips malformed captured rows with diagnostics, preserves manifest contracts on the synthesized bundled PluginRecord, and adds a focused regression test.

## Best Possible Solution

Refresh the draft against current main, choose sequencing with the focused contract PR, and land the scoped per-row capture hardening after plugin-boundary CI or changed-gate proof.

## Reproduction Assessment

Yes. Current main reads captured ids and tool names directly inside one enclosing try block in `src/plugins/bundled-capability-runtime.ts`, so a throwing captured field can fail the whole bundled runtime record.

## Solution Assessment

Yes, with normal maintainer proof. Snapshotting captured identity fields once per row at the bundled capability boundary is narrower than plugin-specific shims and preserves healthy siblings without adding public API surface.
