## What This Changes

The PR changes bundled capability runtime registry loading to snapshot captured ids and tool names per row, skip malformed captured rows with diagnostics, preserve manifest contracts, and add a regression test.

## Best Possible Solution

Rebase on current main, reconcile the contract-preservation overlap with https://github.com/openclaw/openclaw/pull/81312, and land the per-row capture isolation with focused runtime and plugin-boundary proof.

## Reproduction Assessment

Yes, by source inspection and the PR's focused regression path: current main reads captured identity fields inside one outer try block, so a throwing tool name or provider id getter can fail the whole bundled runtime snapshot. I did not run tests because this review is read-only.

## Solution Assessment

Yes for the unique isolation fix: snapshotting captured string fields per row at the bundled runtime boundary is narrower than plugin-specific shims and aligns with normal registry contract enforcement. The merge path still needs a rebase and reconciliation with the overlapping contract-preservation PR.
