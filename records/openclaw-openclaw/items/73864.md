## Summary

Keep open. Current main still lacks a user-facing scope or mode selector for `openclaw mcp serve`, the bridge still requests the fixed read/write/approvals scope set, and #74428 is the active implementation candidate that should be reviewed rather than duplicating the work.

## What This Changes

Request a supported `openclaw mcp serve` scope or mode selector so MCP bridge identities can ask for narrower operator scopes.

## Best Possible Solution

Review and land #74428 or an equivalent narrow change that validates MCP bridge-relevant scopes, preserves the current default when omitted, and updates focused CLI/server/docs coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows no `--scope` or mode option, no serve-options field for scopes, and fixed scopes passed to `GatewayClient`; no live gateway is needed to verify the gap.

## Solution Assessment

Yes, with maintainer review. A validated repeatable `--scope` option limited to the MCP bridge scope set is the narrowest maintainable direction; dynamic `tools/list` filtering can remain a separate follow-up.
