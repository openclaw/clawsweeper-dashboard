## Summary

Keep open. Current main still lacks a user-facing scope or mode selector for `openclaw mcp serve`, and the bridge still requests the fixed read/write/approvals scope set; open PR #74428 is the active implementation candidate while #74030 was closed unmerged.

## What This Changes

Request a supported way for `openclaw mcp serve` to request narrower operator scopes, such as repeatable `--scope` flags or documented modes/config.

## Best Possible Solution

Review and land #74428 or an equivalent narrow change that validates MCP bridge-relevant scopes, preserves the current default when omitted, and updates focused CLI/server/docs coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows no `--scope`/mode option, no scopes field in `OpenClawMcpServeOptions`, and fixed scopes passed to `GatewayClient`; no live gateway is needed to verify this gap.

## Solution Assessment

Yes, with maintainer review. A validated repeatable `--scope` option limited to `operator.read`, `operator.write`, and `operator.approvals` is the narrow maintainable fix; dynamic `tools/list` filtering can follow separately if desired.
