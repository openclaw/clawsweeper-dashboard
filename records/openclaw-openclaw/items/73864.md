## Summary

Keep open. Current main and v2026.5.12 still lack a configurable `openclaw mcp serve` operator-scope selector, while https://github.com/openclaw/openclaw/pull/74428 remains the active implementation candidate for this exact feature.

## What This Changes

Request a supported way for `openclaw mcp serve` to request narrower operator scopes via a flag, config/env setting, or explicit mode.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/74428 or an equivalent narrow implementation that validates MCP bridge scopes, preserves current defaults when omitted, documents cached-token behavior, and adds focused CLI/server coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows no `--scope` or mode option, no scope field in `OpenClawMcpServeOptions`, and a fixed MCP bridge scope list passed to `GatewayClient`.

## Solution Assessment

Yes, directionally. A repeatable, allowlisted `--scope` flag threaded through MCP serve options is the narrow maintainable path, but the active PR still needs maintainer review for allowlist and cached-token semantics.
