## Summary

Keep open. Current main and v2026.5.12 still lack a configurable `openclaw mcp serve` operator-scope selector, while https://github.com/openclaw/openclaw/pull/74428 is the active implementation candidate and https://github.com/openclaw/openclaw/pull/74030 was closed as the broader superseded attempt.

## What This Changes

Add a supported way for `openclaw mcp serve` to request narrower operator scopes, such as a repeatable `--scope` flag or equivalent mode/config selector.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/74428 or an equivalent narrow implementation that validates MCP bridge scopes, preserves current defaults when omitted, documents the contract, and adds focused CLI/server coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows no `--scope` or mode option, no scope field in `OpenClawMcpServeOptions`, and a fixed MCP bridge scope list passed to `GatewayClient`.

## Solution Assessment

Yes, with maintainer review. A repeatable, allowlisted `--scope` flag threaded through MCP serve options to `GatewayClient` is the narrow maintainable path; the broader operator-scope-to-agent-tool-policy work remains separate in https://github.com/openclaw/openclaw/issues/49183.
