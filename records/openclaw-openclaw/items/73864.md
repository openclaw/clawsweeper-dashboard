## Summary

Keep open. Current main still lacks a user-facing scope or mode selector for `openclaw mcp serve`, the MCP bridge still requests a fixed read/write/approvals scope set, and open PR #74428 remains the active implementation candidate while #74030 was closed unmerged.

## What This Changes

Add a supported way for `openclaw mcp serve` to request narrower operator scopes, such as repeatable `--scope` flags or equivalent mode/config controls.

## Best Possible Solution

Land #74428 or an equivalent narrow change that validates MCP bridge-relevant scopes, preserves the current default when omitted, and updates focused CLI/server/docs coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows no `--scope` or mode option, no scopes field in `OpenClawMcpServeOptions`, and fixed scopes passed to `GatewayClient`; no live gateway is needed to verify the gap.

## Solution Assessment

Yes, with maintainer review. A validated repeatable `--scope` option limited to the MCP bridge scopes is the narrow maintainable direction, while dynamic `tools/list` filtering can remain a separate follow-up.
