## Summary

Keep open: current main still has the 64 KiB inter-event SSE sanitizer cap, and this PR is a focused candidate fix for the paired provider regression with no blocking correctness finding found. The remaining maintainer decision is accepting the larger bounded sanitizer memory ceiling before merge.

## What This Changes

The PR raises the OpenAI-compatible SSE sanitizer inter-event buffer from 64 KiB to 16 MiB while keeping non-OK error-body truncation at 64 KiB, and updates the oversized-stream regression test threshold.

## Best Possible Solution

Land a bounded provider-runtime fix that accepts legitimate large SSE events while preserving a tight non-OK error-body cap and the boundary-less stream guard.

## Reproduction Assessment

Yes from source: current main throws when an SSE response exceeds 64 KiB before an event boundary, and the paired report describes live `gpt-5.5` streams doing exactly that. I did not run a live provider repro in this read-only review.

## Solution Assessment

Yes, this is a narrow fix in the owner runtime path: it keeps the non-OK cap tight and only raises the successful SSE inter-event guard. The main alternative would be provider-specific bypassing, which would be a worse owner-boundary fit than a bounded generic sanitizer ceiling.
