## Summary

Keep open for maintainer merge review: no blocking patch defect was found, and the remaining decision is whether the 16 MiB successful-SSE sanitizer ceiling is the right availability budget.

## What This Changes

The PR raises the successful SSE sanitizer inter-event buffer cap to 16 MiB chars, keeps non-OK response bodies capped at 64 KiB, and adds provider-transport tests for large valid events and boundary-less overflow.

## Best Possible Solution

Land this focused fix, or an equivalent tuned ceiling, after maintainers accept the successful-SSE sanitizer memory budget while keeping non-OK error bodies tightly capped.

## Reproduction Assessment

Yes. Current main source rejects sanitized SSE streams after 64 KiB without a boundary, and the PR body includes live before/after `gpt-5.5` output showing repeated failures before the cap change and a clean stream afterward.

## Solution Assessment

Yes, with maintainer threshold acceptance. Separating successful-stream buffering from non-OK error-body truncation fixes the false positive without removing the OOM guard; unbounding or bypassing the sanitizer would be worse.
