## Summary

Keep open: this is a focused candidate fix for a source-reproducible scanned-PDF fallback bug, and current main plus the latest release still use the batch image fallback. The patch appears functionally correct, but maintainers should explicitly accept or document that `maxPixels` now acts as a per-page render budget rather than an aggregate PDF fallback budget.

## What This Changes

The PR changes the document-extract PDF image fallback to render selected pages one page at a time and updates tests to expect per-page image extraction calls.

## Best Possible Solution

Land the plugin-owner fix after maintainers accept and document the per-page `maxPixels` semantics for PDF fallback rendering.

## Reproduction Assessment

Yes at source level: current main batches fallback image extraction, and clawpdf 0.3.0's `renderImages` uses one shared `remainingPixels` counter across selected pages. The PR body also includes before/after real output from a 3-page PDF render against clawpdf 0.3.0.

## Solution Assessment

Yes, with a merge-safety caveat: per-page `extract({ mode: "images", pages: [n] })` keeps the fix in the plugin call shape and lets clawpdf own render sizing, page validation, and forms handling, but the changed budget semantics should be accepted or documented.
