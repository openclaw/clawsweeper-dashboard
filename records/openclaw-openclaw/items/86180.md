## Summary

Close as a config/support issue: `voice` is not a supported ElevenLabs TTS selector in OpenClaw; current main, v2026.5.22, tests, and docs consistently use `voiceId`, while provider config intentionally accepts unknown provider-owned keys.

## What This Changes

The issue asks OpenClaw to honor `talk.providers.elevenlabs.voice` as the ElevenLabs TTS voice selector.

## Best Possible Solution

Keep the existing `voiceId` contract for ElevenLabs TTS and route this report to the documented config shape rather than adding a new undocumented `voice` alias.

## Reproduction Assessment

Yes. Source inspection shows the exact reported key, `talk.providers.elevenlabs.voice`, is accepted as an unknown provider-owned field but is not read by the ElevenLabs resolver; the supported key is `voiceId`.

## Solution Assessment

No. The maintainable solution is to use the documented `voiceId` key; making `voice` an ElevenLabs alias would add a new public config compatibility path and needs product judgment rather than a narrow bug fix.
