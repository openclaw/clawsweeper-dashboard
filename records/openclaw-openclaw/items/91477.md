## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve estimated usage through session state:** `src/agents/cli-runner/usage-estimate.ts:44`
  - body: `estimated: true` is added to `CliOutput.usage`, but the real session/status path drops that marker: `SessionEntry` only stores numeric token/cost fields and status/TUI usage renders from those fields. Once a backend uses this hook, heuristic counts can appear in status, usage, and cost surfaces as if they were provider-supplied exact usage, which is the product concern this discriminator was meant to avoid. Carry an estimate/source flag through the persisted/session usage contract and update the consumers/tests, or keep estimated values out of those surfaces until they can be labeled.
  - confidence: 0.9
- **[P2] Expose the estimate types on the SDK subpath:** `src/plugins/cli-backend.types.ts:68-86`
  - body: The new hook contract adds `CliBackendUsageEstimate` and `CliBackendEstimateUsageContext`, but the documented import path for CLI backend authors is `openclaw/plugin-sdk/cli-backend`, and that subpath still re-exports only the old type list. External backend authors cannot import the new contract types from the public subpath, and the SDK docs/baseline remain stale. Add the exports to `src/plugin-sdk/cli-backend.ts` and refresh the SDK contract docs/baseline with the same change.
  - confidence: 0.86
