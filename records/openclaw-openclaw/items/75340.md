## Summary

Keep open. Current main still routes plain text status, including `--usage`, through the fast metadata path with setup/runtime fallback disabled, while bundled channel discovery deliberately skips staged runtime-deps registration. The report remains a valid packaged CLI/status regression, and related #75685 is broader and open rather than a superseding fix.

## What This Changes

The issue asks packaged `openclaw status --usage` to make configured bundled channel rows reliable by registering or reusing staged bundled runtime dependencies before channel status/setup modules need them.

## Best Possible Solution

Keep status read-only and fast, but make the configured bundled channel table reliable for packaged installs by either using manifest/setup metadata that avoids runtime imports or by preparing only the active staged runtime-deps root before the existing bundled setup/channel import path that can require external packages.

## Reproduction Assessment

Yes. The source-level reproduction is high-confidence: run packaged `openclaw status --usage` with configured bundled channels whose dependencies exist only under the managed runtime-deps root; current main keeps the fast path off setup fallback and skips runtime-deps preparation on bundled discovery/setup loads.

## Solution Assessment

No current-main solution is sufficient yet. The safest fix is a narrow status/read-only-channel change, not an unconditional full plugin preload or live channel probe.
