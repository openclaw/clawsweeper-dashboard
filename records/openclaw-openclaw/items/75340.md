## Summary

Keep open. Current main still matches the reported regression: plain text status, including `--usage`, uses the fast channel-table path with setup/runtime fallback disabled, while the bundled channel/setup loaders can still import channel modules with runtime-deps registration explicitly skipped.

## What This Changes

Make packaged `openclaw status --usage` render configured bundled channel rows by preparing the active managed runtime-deps path, or avoiding runtime imports, before bundled channel status/setup modules load.

## Best Possible Solution

Keep plain status read-only and reasonably fast, but make configured bundled channel rows reliable in packaged installs by using manifest/setup metadata where possible and narrowly registering the active staged runtime-deps root before any unavoidable bundled setup import.

## Reproduction Assessment

Yes. A high-confidence reproduction path is a packaged 2026.4.29-style install with configured bundled channels whose dependencies exist only under the managed `plugin-runtime-deps` root, then run `openclaw status --usage`; current main's source path still disables setup fallback and skips runtime-deps registration before those imports.

## Solution Assessment

Unclear. The report's suggested registration path is plausible, but the most maintainable fix should first avoid runtime imports for the status table when metadata can answer the question, and only register staged deps for the narrow setup/import path that remains necessary.
