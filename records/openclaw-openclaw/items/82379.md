## Summary

Keep open: the docs-only change addresses a real Lobster documentation gap, but the proposed env-var table does not match the pinned Lobster runtime because it omits `LOBSTER_ARGS_JSON` and documents `LOBSTER_FINAL_STATUS` without source support.

## What This Changes

This PR adds a Lobster docs subsection that enumerates injected step environment variables, warns against `LOBSTER_STEP_*` and `LOBSTER_OUTPUT_*`, and points users to supported step-output access patterns.

## Best Possible Solution

Document the exact upstream workflow env contract and supported step-output mechanisms, with no runtime changes unless maintainers decide to add new env vars.

## Reproduction Assessment

Yes. The docs gap and PR mismatch are source-reproducible: current docs lack the env-var contract, OpenClaw passes `ctx.env` through, and `@clawdbot/lobster@2026.4.6` injects `LOBSTER_ARGS_JSON` plus `LOBSTER_ARG_*`.

## Solution Assessment

No. A targeted docs clarification is the right direction, but this PR should first align the table with the pinned runtime source by adding `LOBSTER_ARGS_JSON` and removing or proving `LOBSTER_FINAL_STATUS`.
