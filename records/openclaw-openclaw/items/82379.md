## Best Possible Solution

Update the docs to reflect the exact `@clawdbot/lobster@2026.4.6` env contract and supported output paths, leaving runtime behavior unchanged.

## Reproduction Assessment

Yes. The mismatch is source-reproducible by comparing the PR table with the pinned Lobster runtime: OpenClaw forwards `process.env`, while upstream `mergeEnv` injects `LOBSTER_ARGS_JSON` and `LOBSTER_ARG_*` with no `LOBSTER_FINAL_STATUS` hit.

## Solution Assessment

No. The docs-only direction is right, but this PR should first add `LOBSTER_ARGS_JSON` and remove or prove `LOBSTER_FINAL_STATUS` against the pinned runtime source.
