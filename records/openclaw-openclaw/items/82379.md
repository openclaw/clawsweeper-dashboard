## Summary

Keep open: the docs-only PR addresses a real Lobster docs gap, but its claimed complete env-var table is inaccurate for the pinned runtime because it omits `LOBSTER_ARGS_JSON` and documents `LOBSTER_FINAL_STATUS` without source support; a follow-up comment already called out the same doc drift as blocking.

## What This Changes

The PR adds a Lobster docs section listing injected step env vars, warning against `LOBSTER_STEP_*` and `LOBSTER_OUTPUT_*`, and pointing to `stdin:`, template substitution, and `condition`/`when` output access.

## Best Possible Solution

Update the docs to match the pinned Lobster runtime contract: list `LOBSTER_ARGS_JSON` and `LOBSTER_ARG_<NORMALIZED_NAME>`, remove or prove `LOBSTER_FINAL_STATUS`, and keep the step-output guidance docs-only.

## Solution Assessment

No. A targeted docs clarification is the right direction, but this PR should first align the table with the pinned runtime instead of documenting an unsupported final-status variable.
