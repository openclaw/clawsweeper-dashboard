## What This Changes

The PR changes Feishu reply dispatch so `blockStreaming: true` can send block payloads as independent Feishu messages when card streaming is inactive, with tests for duplicate-final suppression.

## Best Possible Solution

Merge a narrow Feishu dispatcher repair that sends non-card block chunks through the existing message path while preserving reply, thread, and mention metadata, after redacted live Feishu proof shows independent blocks and no duplicate final.

## Reproduction Assessment

Yes, source-reproducible: current main and v2026.6.8 return from Feishu block delivery when `useStreamingCard` is false, so `streaming: false` plus `blockStreaming: true` loses block text. I did not run a live Feishu tenant in this read-only review.

## Solution Assessment

No, not yet. The dispatcher is the right layer for the narrow fix, but the PR should preserve the existing mention-forward metadata on the first independent block and then show real Feishu behavior before merge.
