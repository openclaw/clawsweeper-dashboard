The PR changes Feishu reply dispatch and tests so `blockStreaming: true` can send block payloads as standalone Feishu messages when card streaming is inactive, then skip a matching final reply.

## Best Possible Solution

## Reproduction Assessment

Yes, at source level: current main returns from Feishu block delivery when `useStreamingCard` is false, so `streaming: false` plus `blockStreaming: true` loses block text. I did not run a live Feishu tenant in this read-only review.

## Solution Assessment
