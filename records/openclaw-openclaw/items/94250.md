## What This Changes

## Best Possible Solution

Preserve reply, thread, and mention metadata on the first independent block send, add focused regression coverage, then attach redacted live Feishu proof showing separate block messages and no duplicate final.

## Reproduction Assessment

Yes, at source level: current main returns from Feishu block delivery when `useStreamingCard` is false, so `streaming: false` plus `blockStreaming: true` loses block text. I did not run a live Feishu tenant in this read-only review.

## Solution Assessment

No, not as submitted. The dispatcher is the right layer for this narrow bug, but the independent block path must preserve existing mention metadata and show real Feishu behavior before merge.
