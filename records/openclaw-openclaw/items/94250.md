## Summary

Keep open: the narrow Feishu block-streaming fix is useful, but the PR is not merge-ready because the new independent block path can drop mention-target metadata and the supplied proof is only mocked dispatcher test output.

## What This Changes

The branch changes Feishu reply dispatch and tests so `blockStreaming: true` can send block payloads as standalone Feishu messages when card streaming is inactive, then suppress a matching final reply.

## Best Possible Solution

Preserve reply, thread, and mention metadata on the first independent block send, then attach redacted live Feishu proof showing separate block messages and no duplicate final before merge.

## Reproduction Assessment

Yes, at source level: current main returns from Feishu block delivery when `useStreamingCard` is false, so `streaming: false` plus `blockStreaming: true` loses block text. I did not run a live Feishu tenant in this read-only review.

## Solution Assessment

No, not as submitted. The dispatcher is the right layer for this narrow bug, but the independent block path must preserve existing mention metadata and show real Feishu behavior before merge.
