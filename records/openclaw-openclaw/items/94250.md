## What This Changes

The PR changes Feishu reply dispatch so `blockStreaming: true` sends block payloads as independent messages when no streaming card path is active, and adds dispatcher regression tests for block sends and duplicate-final suppression.

## Best Possible Solution

Merge this narrow Feishu dispatcher repair after redacted live Feishu proof shows `streaming: false` plus `blockStreaming: true` sends completed blocks as independent messages without duplicate finals or broken reply/thread fallback.

## Reproduction Assessment

Yes, at source level: current main returns from Feishu block delivery when `useStreamingCard` is false, so `streaming: false` plus `blockStreaming: true` drops block text. I did not run a live Feishu tenant in this read-only review.

## Solution Assessment

Yes for the narrow code shape: repairing the existing Feishu `blockStreaming` contract in the dispatcher is more maintainable than adding a parallel config mode. It is not proof-sufficient for merge until live Feishu behavior is shown.
