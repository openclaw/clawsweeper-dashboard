## Summary

Keep open: current main and the latest shipped release still default omitted cron jobs to enabled, let cron agent turns inherit agent/default model selection when `payload.model` is absent, and only apply generic recurring error backoff. The related open reports cover cron defaults, guardrails, and provenance slices, but they do not fully replace this broader consent, cost, and quota policy issue.

## What This Changes

Request to make externally registered recurring cron jobs opt-in and add model-cost, doctor/update visibility, and quota circuit-breaker guardrails before they run.

## Best Possible Solution

Approve a cron registration and provenance contract that separates operator-created jobs from external or agent/plugin registrations, then implement opt-in activation, cron-specific model/default policy, doctor/update visibility, reason-aware guardrails, docs, and regression coverage together.
