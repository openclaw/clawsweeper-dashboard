## What This Changes

The issue asks OpenClaw to stop externally or plugin-registered cron jobs from auto-enabling without confirmation, inheriting expensive agent defaults, and running without usage circuit breakers.

## Best Possible Solution

Approve a cron provenance and consent contract that distinguishes operator-created schedules from plugin or agent-created schedules, then implement opt-in activation, cron-specific model defaults, doctor/update visibility, and reason-aware quota guardrails with docs and regression coverage.
