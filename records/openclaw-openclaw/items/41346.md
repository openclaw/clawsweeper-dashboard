## Summary

Keep open. Current main and latest release still contain the core behavior: cron create defaults enable jobs, bundled plugin scheduled turns explicitly add enabled agentTurn jobs, agentTurn jobs without payload.model can inherit the configured default model, and no single related item owns the full consent, provenance, model-cost, and quota circuit-breaker scope.

## What This Changes

The issue asks for externally registered cron jobs to require user activation and visible diffs, avoid silent expensive model inheritance, and gain usage circuit breakers.

## Best Possible Solution

Define one maintainer-approved cron safety policy for external/plugin registration provenance, default activation, explicit model or cost warning behavior, doctor/update visibility, and quota-aware pause/notification semantics.

Yes at source level: current main still defaults new cron jobs to enabled, plugin scheduled turns explicitly register enabled agentTurn jobs, and agentTurn jobs without payload.model can inherit agent/default model selection. I did not run a live quota-burning provider scenario.

## Solution Assessment

Unclear as a single patch: the requested safeguards are directionally aligned, but the best fix needs maintainer decisions across operator cron compatibility, plugin or agent scheduling provenance, model-cost policy, and quota circuit-breaking.
