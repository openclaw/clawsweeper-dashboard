## Summary

Keep open: current main and the latest release still allow omitted cron `enabled` to become active, allow agent-turn cron jobs without `payload.model` to inherit agent/default model selection, and provide only generic recurring backoff rather than consent, cost, or quota circuit-breaker policy.

## What This Changes

The issue requests opt-in activation, update/doctor visibility, cron-specific model-cost defaults or warnings, and usage circuit breakers for externally registered polling cron jobs.

## Best Possible Solution

Approve a cron registration and provenance contract that separates operator-created jobs from external, agent, plugin, or synced registrations, then implement opt-in activation, cron-specific model/default policy, doctor/update visibility, reason-aware guardrails, docs, and regression coverage together.
