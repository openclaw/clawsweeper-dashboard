## Summary

Keep open: current main and the latest release still allow plugin/external cron registrations to create active recurring agent-turn jobs, and missing per-job models still fall through to agent/default model selection; the remaining fix needs maintainer policy and security review.

Requests cron safeguards so externally registered recurring jobs do not auto-enable, silently inherit expensive default models, or continue consuming quota without guardrails.

Approve a cron registration/provenance contract that separates operator-created jobs from external, plugin, and agent-created schedules, then add opt-in activation, cron-specific model/default policy, doctor/update visibility, quota-aware guardrails, docs, and regression coverage together.
