## Summary

Keep open. Current main and the latest release still default new cron jobs to enabled, plugin scheduled turns still create enabled agentTurn jobs, and missing per-job models still resolve through agent/default model selection; related open work covers pieces but not the full consent, model-cost, and usage-circuit-breaker problem.

## What This Changes

The issue asks OpenClaw to add consent, safer defaults, model-cost guardrails, dry-run visibility, and usage circuit breakers for externally registered polling cron jobs.

## Best Possible Solution

Keep this issue open until maintainers define one coherent cron safety policy for external or plugin registration provenance, activation consent, explicit model selection or cost warnings, and quota-aware pause/notification behavior with docs and regression coverage.

## Reproduction Assessment

Yes at source level. Current main defaults missing cron enabled to true, plugin scheduled turns pass enabled true, and agentTurn cron jobs without payload.model resolve through agent/default model selection; I did not run a live quota-burning scenario.

## Solution Assessment

Unclear as a single implementation. The requested safeguards are directionally aligned, but the best fix needs a maintainer-chosen split between operator cron, plugin or agent scheduling provenance, default model policy, and usage or reason-aware circuit breaking.
