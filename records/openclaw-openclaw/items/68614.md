## Summary

Keep open. The timeout-fallback direction is useful because current main still has no timer-driven Google Chat deferred-response expiry path, but this PR head still leaves the new timer armed when dispatch exits without delivery, lacks contributor real behavior proof, and is conflicting with current main's split reply-delivery structure.

## What This Changes

The PR adds a 25-second Google Chat typing-message timeout fallback, threads an expired-deferred flag into reply delivery, and logs/falls back when deferred message updates fail.

## Best Possible Solution

Port the timeout fallback into current main's Google Chat monitor/reply-delivery split, cancel the timer on every dispatch exit path, add focused fake-timer coverage, and verify with redacted real Google Chat evidence.

## Reproduction Assessment

Yes for the PR blocker. Source inspection of PR head shows the 25-second timer is started and cleared only from `onDeliver`, while the shared dispatch path has no-delivery returns; I did not run live Google Chat in this read-only pass.

## Solution Assessment

No. The direction is right, but the mergeable solution should clear the timer in a dispatch `finally` path and be ported to current main's split helper structure before live proof.
