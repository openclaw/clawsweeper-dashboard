## Summary

Keep open. Current main still lacks a deferred-response timeout for Google Chat typing-message updates, and this PR is useful source work, but the branch is not merge-ready because the timeout can still fire after dispatch exits without delivering a reply.

## What This Changes

The PR changes Google Chat reply handling to start a 25-second timer after creating a typing placeholder, send an interim fallback, delete the placeholder, and route the eventual agent reply as a new message when the deferred update window expires.

## Best Possible Solution

Port the timeout behavior to current main's split Google Chat code while preserving the existing typingIndicator default, reaction fallback, text/media fallback helper behavior, and adding unconditional timer cleanup around turn dispatch completion.

## Reproduction Assessment

Yes. A focused unit reproduction can fake timers, create a message-mode typing placeholder, make dispatch return without invoking deliver, advance 25 seconds, and observe the PR's spurious fallback; current main also lacks any timer path for long-running first replies.

## Solution Assessment

No. The PR is directionally correct, but as-is it is not the best solution because it misses cleanup when no delivery callback runs and needs to be adapted to current main's monitor plus reply-delivery split.

## Review Findings
