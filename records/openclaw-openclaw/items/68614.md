## Summary

Keep open: current main still lacks the Google Chat timeout fallback, but this PR is not mergeable as-is because the timeout is only cleared from the delivery callback, contributor real behavior proof is missing, and the branch conflicts with current main's reply-delivery split.

## What This Changes

The PR changes Google Chat reply handling to add a 25-second typing-message timeout fallback and logging when deferred message updates fail.

## Best Possible Solution

Port the timeout fallback into current main's Google Chat monitor/reply-delivery split with unconditional dispatch-exit cleanup, focused fake-timer coverage, and redacted live Google Chat proof.

## Reproduction Assessment

Yes for the PR blocker: source inspection shows PR head starts a 25-second timer after typing-message creation and clears it only inside `onDeliver`, while dispatch can exit without delivery. I did not run live Google Chat or tests in this read-only pass.

## Solution Assessment

No. The direction is right, but the mergeable solution should clear the timer in a dispatch `finally` path and be ported into current main's split helper structure before live proof.
