## Summary

Keep open: current main still lacks the timer-driven Google Chat deferred-response fallback, but this PR is not ready because it leaves a timeout side effect when dispatch exits without delivery and lacks required real behavior proof.

## What This Changes

The PR changes `extensions/googlechat/src/monitor.ts` to add a 25-second Google Chat typing-message timeout, a still-working fallback message, a deferred-expired final-delivery path, and extra logging around deferred update failures.

## Best Possible Solution

Port the timeout fallback into current main's split Google Chat monitor/reply-delivery flow, clear the timer on dispatch exit, remove the unrelated issue reference, add focused fake-timer coverage, and collect redacted live Google Chat proof.

## Reproduction Assessment

Yes for the PR blocker: source inspection shows PR head arms a 25-second timer and clears it only inside `onDeliver`, while the dispatcher has duplicate/inflight returns before delivery. The original Google Chat delivery-loss path is source-plausible from current main plus Google's 30-second interaction contract, but I did not run live Google Chat.

## Solution Assessment

No. The direction is right, but the mergeable solution should fit current main's split helper structure, clear the timeout in a dispatch `finally` path, remove the unrelated issue reference, and include real Google Chat proof before merge.
