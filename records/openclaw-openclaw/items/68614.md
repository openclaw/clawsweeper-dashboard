## Summary

Keep open: current main still lacks the Google Chat deferred-response timeout, and this PR's useful timeout path is not mergeable as-is because it still leaves a timer-cleanup edge case and lacks required real behavior proof.

## What This Changes

The branch modifies Google Chat monitor reply handling to add a 25-second typing-message timeout, a user-facing fallback message, an expired-deferred delivery flag, and update-failure logging.

## Best Possible Solution

Port the timeout fallback into current main's monitor/reply-delivery split with unconditional timer cleanup around dispatch, focused fake-timer coverage, and redacted live Google Chat proof.

## Reproduction Assessment

Yes. Source inspection shows PR head starts a 25-second timer after the typing placeholder and clears it only from `onDeliver`, so a dispatch path that throws or exits before delivery leaves the timer active; I did not run tests because this pass is read-only.

## Solution Assessment

No. The direction is right, but the patch is not the best mergeable solution as-is because it needs unconditional dispatch-exit cleanup and a port to current main's split reply-delivery helper.
