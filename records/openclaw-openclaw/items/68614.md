## Summary

Keep open: the PR targets a real Google Chat delivery gap that current main still lacks, but the branch is not mergeable as-is because the new timeout is only cleared from the delivery callback and contributor real behavior proof is still missing.

## What This Changes

The PR modifies Google Chat reply handling to start a 25-second typing-message timer, send a still-working fallback, and route final replies through a new-message path after deferred expiry.

## Best Possible Solution

Port the timeout fallback into current main's split Google Chat monitor/reply-delivery flow, clear the timer on dispatch exit, add focused fake-timer coverage, and collect redacted live Google Chat proof.

## Reproduction Assessment

Yes for the PR blocker: source inspection shows PR head arms a 25-second timer and only clears it inside `onDeliver`, while the dispatcher has duplicate/inflight return paths that do not deliver. I did not run live Google Chat in this read-only pass.

## Solution Assessment

No. The direction is right, but the mergeable solution should clear the timer in a dispatch `finally` path and be ported into current main's split helper structure before live proof.
