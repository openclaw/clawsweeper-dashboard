## Summary

Keep open: current main still lacks a timer-driven Google Chat fallback for expired typing-message updates, and this PR targets a plausible message-delivery bug, but the branch is not merge-ready because it leaves timeout side effects on no-delivery exits, needs porting to the current split helper/media API, and has no real Google Chat proof.

## What This Changes

The PR adds a 25-second Google Chat typing-message timeout and fallback path so long-running replies can send an interim message and then deliver the final response as a new message.

## Best Possible Solution

Port the timeout fallback into current main's split Google Chat monitor/reply-delivery flow, clear pending timeout state from `onFinalize` or an equivalent dispatch-exit path, add focused fake-timer coverage, and require redacted live Google Chat proof.

## Reproduction Assessment

Yes for the PR blockers: source inspection shows the PR clears the timeout only from `onDeliver`, while the shared dispatcher can return before delivery. The original silent-drop symptom is source-plausible on current main, but I did not run live Google Chat.

## Solution Assessment

No. The direction is right, but the mergeable solution should fit current main's split helper structure, clear timeout state on every dispatch exit, remove the unrelated issue reference, and include real Google Chat proof.
