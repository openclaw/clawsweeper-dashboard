## Summary

Keep open: the Google Chat timeout problem is plausible and still not implemented on current main, but this PR is not merge-ready because it lacks real Google Chat proof, conflicts with current main's split reply-delivery structure, and leaves the timeout armed when dispatch exits without delivery.

## What This Changes

The PR adds a 25-second Google Chat typing-message timeout that sends a still-working fallback and routes the eventual reply through a new-message path.

## Best Possible Solution

Port the timeout fallback into current main's split Google Chat monitor/reply-delivery flow, cancel the timer on dispatch exit, add focused fake-timer coverage, and collect redacted live Google Chat proof.

## Reproduction Assessment

Yes for the PR blocker: source inspection shows PR head arms a 25-second timer and clears it only in `onDeliver`, while dispatch can return without delivery. The original user-visible Google Chat timeout path is source-plausible from Google's 30-second contract and current main's missing timer path, but I did not run live Google Chat.

## Solution Assessment

No. The direction is right, but the mergeable solution should be ported to current main's split helper structure, clear the timer in a dispatch `finally` path, and include real Google Chat proof before merge.
