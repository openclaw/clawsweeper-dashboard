The PR adds an `openclaw-managed` non-secret marker, sanitizes generated `models.json` provider `apiKey` values, preserves models.json-only plaintext credentials, and adds planner plus registry regression coverage.

## Best Possible Solution

Land one canonical Layer 1 fix that removes source-managed plaintext from generated `models.json`, preserves pi registry compatibility, and keeps models.json-only credentials unless maintainers choose a stricter migration.
