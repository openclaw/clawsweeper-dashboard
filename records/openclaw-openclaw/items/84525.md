Keep open for maintainer review: the patch looks correct for the Layer 1 plaintext `models.json` leak, but it changes the generated auth contract and overlaps with several open Layer 1 PRs, so maintainers should choose the canonical implementation before merge.

The PR adds an `openclaw-managed` auth marker, sanitizes generated `models.json` provider `apiKey` values for source-managed credentials, preserves models.json-only plaintext credentials, and adds regression coverage.

## Best Possible Solution

Land one canonical Layer 1 fix that removes source-managed plaintext from generated `models.json`, preserves pi registry compatibility, and keeps models.json-only credentials unless maintainers choose a stricter migration.
