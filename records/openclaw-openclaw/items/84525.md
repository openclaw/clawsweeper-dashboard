## Summary

Keep open: this is a useful security fix candidate, but it is not merge-ready because the new persisted `apiKey` marker can still cross into runtime request authentication, and the branch is conflicting with current main's root-plus-plugin catalog writer shape.

## What This Changes

The PR adds an `openclaw-managed` non-secret auth marker, rewrites source-managed plaintext provider `apiKey` values during generated `models.json` planning, preserves models.json-only plaintext keys, and adds planner/registry regression tests.

## Best Possible Solution

Land one maintainer-selected Layer 1 implementation that keeps plaintext out of every generated catalog, keeps real credentials in runtime-only auth storage, and preserves existing models.json-only credentials through an explicit upgrade contract.

## Solution Assessment

No. The redaction direction is right, but this marker-based patch is not the best fix until the marker cannot be consumed as request auth and the current root-plus-plugin catalog writer is covered after rebase.
