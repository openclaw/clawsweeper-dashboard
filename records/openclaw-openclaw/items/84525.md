## Summary

Keep this PR open: the direction and proof are useful, but the new sanitizer still misses a supported normalized provider-key path that can leave a plaintext `apiKey` in generated `models.json`.

## What This Changes

The PR adds an `openclaw-managed` non-secret auth marker, sanitizes source-managed plaintext provider `apiKey` values before serializing generated `models.json`, preserves models.json-only plaintext keys, and adds focused regression coverage.

## Best Possible Solution

Normalize source-provider ownership in the sanitizer, add the merge-mode normalized-key regression, then land the marker-based redaction if maintainers accept that generated source-managed `models.json` entries are no longer standalone secret-bearing pi configs.
