## What This Changes

The branch adds an OpenClaw-managed auth marker, sanitizes generated models.json provider apiKey values before serialization, and adds regression and registry tests for plaintext, marker, merge, and custom-model discovery paths.

## Best Possible Solution

Rebase or replace this with a current-main fix that separates validation-only catalog metadata from runtime request auth, redacts source-managed plaintext from both root and plugin catalog writes, and preserves models.json-only credentials deliberately.

## Reproduction Assessment

## Solution Assessment

No. Redacting source-managed plaintext is the right direction, but putting a new validation marker in the runtime apiKey field is not the narrowest safe fix because that field is also a request-auth source.
