## Summary

Keep this PR open, but it is not merge-ready: the redaction direction is useful and the supplied terminal proof covers the serialization/registry boundary, while the new `openclaw-managed` marker still lacks a proven runtime credential path and the branch is conflicting against current main's catalog split.

## Best Possible Solution

Land one maintainer-selected Layer 1 implementation that redacts plaintext generated catalogs for root and plugin-owned outputs while preserving existing models.json-only auth and supplying a real runtime credential path for source-managed secrets.
