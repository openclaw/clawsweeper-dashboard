## Summary

Keep this PR open for maintainer review: the patch appears correct and has sufficient real-behavior proof for the planner/registry boundary, but it deliberately changes the generated `models.json` auth contract for source-managed plaintext keys.

## What This Changes

The PR adds an `openclaw-managed` non-secret marker, sanitizes generated `models.json` provider `apiKey` values for source-managed plaintext config, preserves `models.json`-only plaintext keys, and adds agent/auth regression tests.

## Best Possible Solution

Land the marker-based Layer 1 redaction if maintainers accept the auth contract, and keep https://github.com/openclaw/openclaw/issues/11829 open for the remaining masked-secrets and upstream hardening work.

## Reproduction Assessment

Yes. Source inspection shows current main serializes `finalProviders` directly into generated `models.json`, and the normalization path can carry plaintext provider `apiKey` values into that object.

## Solution Assessment

Yes for the Layer 1 persistence fix. The marker preserves pi's non-empty `apiKey` registry contract while removing plaintext from generated config, but it should not be treated as completing the broader secrets roadmap.
