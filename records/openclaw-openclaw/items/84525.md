## Summary

Keep this PR open for maintainer review: it targets a real generated-model-catalog credential exposure, but the current patch writes a new `openclaw-managed` value into the same `apiKey` field that runtime request auth can later consume, and the branch is conflicting with current main's root plus plugin catalog writer.

The PR adds an `openclaw-managed` non-secret marker, sanitizes source-managed plaintext provider `apiKey` values during generated `models.json` planning, preserves models.json-only plaintext keys, and adds planner/registry regression tests.

Land one maintainer-selected Layer 1 implementation that redacts plaintext from root and plugin-generated catalogs while keeping real credentials in runtime-only auth storage and preserving existing models.json-only credentials through an explicit upgrade contract.
