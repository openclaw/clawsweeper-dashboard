## Summary

Keep open: current main still has a real product/documentation gap around Pro-tier Codex refs. Spark is suppressed in provider docs and runtime tests, but the Codex harness snapshot still lists it, and `gpt-5.5-pro`/`gpt-5.4-pro` remain cataloged/resolvable without an account-scoped runnable contract.

## What This Changes

Clarify whether ChatGPT Pro/Codex users can run `gpt-5.5-pro`, `gpt-5.4-pro`, or retired Spark through OpenClaw, then align docs, model listing, picker, fallback, probing, or runtime support with that contract.

## Best Possible Solution

Maintainers should choose the product contract first, then either document Pro/Spark as unsupported and suppress them from default picker/fallback flows, or add account-scoped live probing so Pro refs are surfaced only when the selected account can actually run them.

## Solution Assessment
