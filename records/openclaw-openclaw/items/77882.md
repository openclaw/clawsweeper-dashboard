- **[P1] Enforce the selected account's Bitable gate:** `extensions/feishu/src/bitable.ts:589-592`
  - body: This only checks whether any enabled Feishu account allows Bitable before registering the family. If account `a` has `tools.bitable: false` and account `b` has it enabled, the tools still register, and execution with `agentAccountId: "a"` or `accountId: "a"` reaches `createFeishuToolClient` without rechecking account `a`'s tools config. That defeats the new per-account Bitable/Base gate for the sensitive create/update tools.
  - confidence: 0.9
- **[P2] Preserve account-level base aliases over inherited bitable:** `extensions/feishu/src/tools-config.ts:26-27`
  - body: After `tools` is deep-merged, an account with `tools.base: false` can inherit top-level `tools.bitable: true`, producing `{ bitable: true, base: false }`. `resolveToolsConfig` then treats the inherited `bitable` value as explicit and mirrors `base` back to true, so the documented per-account `base` alias cannot disable Bitable in that common inherited case.
  - confidence: 0.84
