Overall correctness: patch is incorrect

Full review comments:

- **[P1] Normalize aliases before merging account tools:** `extensions/feishu/src/accounts.ts:195`
  - body: With `channels.feishu.tools.bitable=true` and `channels.feishu.accounts.a.tools.base=false`, the new nested merge leaves both keys on the account tools object, and `resolveToolsConfig` makes `bitable` win. That means the documented per-account `base` alias cannot disable Bitable for that account, so sensitive `feishu_bitable_*` tools can remain available despite the operator using the documented disable key. Normalize the alias at each scope before merging, or make account-level `base` override inherited `bitable` when account-level `bitable` is absent.
  - confidence: 0.88
