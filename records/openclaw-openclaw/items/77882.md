- **[P1] Avoid deep-merging every Feishu account tools config:** `extensions/feishu/src/accounts.ts:195`
  - body: Adding `nestedObjectKeys: ["tools"]` changes shipped account semantics for all Feishu tool families, not only Bitable/Base.   - confidence: 0.87
- **[P2] Preserve account-level base alias overrides:** `extensions/feishu/src/tools-config.ts:26`
  - body: The alias resolver runs after top-level and account `tools` are merged, so top-level `bitable: true` plus account-level `base: false` becomes `{ bitable: true, base: false }` and resolves back to enabled. Since the docs add `accounts.<id>.tools.base` as a per-account alias, normalize aliases before cross-level merging or add a regression where account `base: false` overrides top-level `bitable: true`.
  - confidence: 0.75
