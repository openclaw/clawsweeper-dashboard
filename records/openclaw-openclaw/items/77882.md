## Summary

Keep this PR open for revision: the Feishu Bitable aggregate registration gate is useful, but the patch still lets a disabled selected account execute Bitable tools when another account keeps the tool family registered.

## What This Changes

The branch adds Feishu `tools.bitable`/`tools.base` schema, defaults, docs, and tests, then gates `feishu_bitable_*` registration through resolved Feishu tools config.

## Best Possible Solution

Enforce the Bitable/Base gate after resolving the target account, keep alias docs and tests aligned with the approved config contract, rebase the branch, and provide redacted proof that disabled config hides or rejects the Bitable tools.

## Reproduction Assessment

Yes. Source inspection shows a mixed-account path where one account enables Bitable, another disables it, the global Bitable tools register, and execution can still resolve the disabled selected account without a Bitable/Base gate check.

## Solution Assessment

No. A registration-only aggregate gate is not sufficient for the documented per-account tool gate; the selected account must be checked at execution time or the feature must be documented as global-registration-only.
