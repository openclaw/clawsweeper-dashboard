## Summary

Keep open: current main still omits the existing dmPolicy as the wizard selection default, and this PR is not merge-ready because the stated fix is stale, compile-broken, and mixed with unrelated security and dependency changes.

## What This Changes

The branch adds `initialValue: current` to the legacy channel onboarding DM policy prompt and bundles gateway/channel security patches, dependency and lockfile updates, tests, changelog churn, and a repo-root GHSA note.

## Best Possible Solution

Move the preservation fix to the current flow by computing the current policy before the select prompt and passing it as `initialValue`, with focused regression coverage; keep GHSA and dependency work in separate maintainer-reviewed PRs.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists: set a channel dmPolicy to a non-default value, rerun setup/update and opt into DM policy configuration; current main's prompt has no `initialValue`, so it can default to the first option before comparing with the existing value.

## Solution Assessment

No. The branch is not the best fix because it edits the stale command path, has a TypeScript scoping error, and bundles unrelated security/supply-chain changes; the safer solution is a narrow current-main patch plus tests.
