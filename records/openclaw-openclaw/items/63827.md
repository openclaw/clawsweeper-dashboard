## What This Changes

The PR adds an initial DM-policy selection value to onboarding code and also changes gateway, channel, dependency, lockfile, changelog, and security-advisory surfaces.

## Best Possible Solution

Land a narrow current-main fix in src/flows/channel-setup.prompts.ts that reads the current DM policy before selection, passes it as initialValue, and adds focused regression coverage; split unrelated security and dependency work into separate owner-reviewed PRs.

## Reproduction Assessment

Yes for the configure path by source inspection: current main reaches the DM-policy select prompt and reads the existing policy only after selection. No for the update path; the inspected update wizard does not invoke configure or channel setup.

## Solution Assessment

No. Preserving the current policy as the default selection is the right direction, but this PR applies it to stale code with an out-of-scope variable and bundles unrelated security/dependency work instead of the narrow current-main fix.
