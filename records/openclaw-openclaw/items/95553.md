## Summary

Keep open: current main and v2026.6.9 still pass the reply-operation abort signal into required budget/preflight compaction, so an earlier reply abort can beat the configured compaction timeout; several linked PRs are open but none has merged.

## What This Changes

The issue asks OpenClaw to make budget/preflight compaction honor `agents.defaults.compaction.timeoutSeconds` instead of being capped by the reply operation abort path.

## Best Possible Solution

Define a cancellation-only preflight compaction signal that excludes the normal reply lifecycle timeout, preserves explicit stop/restart/shutdown cancellation, adds focused regression tests, and lands one canonical linked PR before closing this issue.

## Reproduction Assessment

Yes at source level: current main and v2026.6.9 pass the reply-operation abort signal into required budget/preflight compaction, and the safety wrapper lets that abort race the configured timeout. I did not live-run the reporter's exact slow-provider environment.

## Solution Assessment

Partly: honoring the existing compaction timeout without adding new config is the narrowest direction. A new preflight-disable knob should remain a fallback only if maintainers decide the cancellation semantics cannot be fixed cleanly with the current config surface.
