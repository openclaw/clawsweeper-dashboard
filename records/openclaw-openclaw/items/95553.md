## Summary

Keep open: current main and the latest release still pass the reply-operation abort signal into required budget/preflight compaction, and the focused linked fix is still open rather than merged.

## What This Changes

The issue reports that budget/preflight compaction can be capped by the reply-operation abort signal instead of honoring `agents.defaults.compaction.timeoutSeconds`.

## Best Possible Solution

Review and land the focused fix in https://github.com/openclaw/openclaw/pull/95561, or an equivalent narrow change, while preserving deliberate user/restart cancellation semantics.

## Reproduction Assessment

Yes, at source level: current main still passes the reply-operation abort signal into required budget/preflight compaction, and the timeout wrapper lets an external abort win before the configured compaction timeout. I did not run a live slow-backend reproduction.

## Solution Assessment

The narrow best direction is to make preflight compaction use the configured compaction timeout for timing, while retaining explicit cancellation where maintainers want it. Adding a new disable toggle should be secondary unless maintainers decide the timing contract alone is not enough.
