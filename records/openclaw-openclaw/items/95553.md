## Label Justifications

- `P1`: The issue describes repeated pre-turn delays and lock contention in real long-running agent sessions when context is near the compaction threshold.
- `impact:session-state`: The affected path is pre-turn transcript/context compaction and can leave sessions un-compacted near overflow.
- `impact:crash-loop`: The reported failure can recur on every large turn, making the session repeatedly spend the preflight timeout window and remain effectively stuck.

## Summary

Keep open: current main and the latest release still pass the reply-operation abort signal into required budget/preflight compaction, so an earlier reply abort can beat the configured compaction timeout; open PRs now target this issue but none has merged.

## What This Changes

The issue asks OpenClaw to stop hard-capping budget/preflight compaction with the reply-operation abort signal and to honor the existing compaction timeout, with an optional preflight-disable knob proposed as a fallback.

## Best Possible Solution

Review and land one narrow PR that gives required preflight compaction the configured compaction timeout while deliberately preserving the intended explicit cancellation paths.

## Reproduction Assessment

Yes at source level: current main passes the reply-operation abort signal into required budget/preflight compaction, and the timeout wrapper races that external abort against the configured compaction timeout. I did not live-run the reporter's slow-backend timing path.

## Solution Assessment

Unclear until maintainers choose the exact cancellation contract. Honoring the existing compaction timeout at the preflight call site looks narrower than adding a new config toggle, but explicit abort and restart semantics need to be preserved deliberately.
