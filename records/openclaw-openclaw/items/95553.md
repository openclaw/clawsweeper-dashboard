- `impact:session-state`: The affected path is transcript/session compaction before the next agent turn and can leave context near overflow.

## Summary

Keep open: current `main` and `v2026.6.10` still pass the raw reply-operation abort signal into required budget/preflight compaction, and open linked PRs are candidate fixes rather than merged fixes.

## What This Changes

The issue reports that budget-triggered preflight compaction can be cancelled by the reply operation abort lifecycle instead of being governed by `agents.defaults.compaction.timeoutSeconds`.

## Best Possible Solution

Keep this issue as the canonical tracker, land one narrow fix that prevents normal reply lifecycle timeout aborts from capping required preflight compaction while preserving explicit cancellation, then decide separately whether to accept the additive config gate.

## Reproduction Assessment

Yes at source level: current `main` and `v2026.6.10` still pass the reply operation abort signal into required budget compaction, and the compaction wrapper races caller aborts before the configured timeout. I did not run the reporter's slow vLLM/Qwen setup, so the exact duration remains live-proof pending.

## Solution Assessment

Unclear as a maintainer decision: honoring the existing compaction timeout is the right end state, but maintainers need to choose the cancellation shape and whether the separate preflight-disable config gate should be accepted.
