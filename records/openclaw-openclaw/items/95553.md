## Summary

Keep open: current `main` and the latest release still pass the reply-operation abort signal into required budget/preflight compaction, and the open linked PRs are unmerged candidate fixes rather than completed work.

## What This Changes

The issue reports that budget/preflight compaction is cancelled by the reply-operation abort signal around 60 seconds instead of honoring `agents.defaults.compaction.timeoutSeconds`, and asks for either timeout ownership to be fixed or a preflight-disable config gate.

## Best Possible Solution

Keep this issue open until maintainers choose and land one narrow cancellation-contract fix; prefer using the existing compaction timeout contract before adding new config surface.

## Reproduction Assessment

Yes at source level: current `main` still passes the reply-operation abort signal into budget/preflight compaction, and the timeout wrapper lets that external abort win. I did not run the reporter's slow vLLM/Qwen live setup in this read-only review.

## Solution Assessment

No final implementation is on `main` yet. The likely best solution is to define the preflight cancellation contract around the existing compaction timeout; adding a preflight-disable config gate is an alternative that needs maintainer product approval.
