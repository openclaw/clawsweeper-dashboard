## Summary

Keep this PR open. The Instances localization work is still relevant because current main only localizes part of the screen, but the branch is stale and not merge-ready against the current generated-locale pipeline.

## What This Changes

The PR adds Instances locale keys to English and pt-BR and updates the Instances view to use translation lookups for labels, fallbacks, scope metadata, last-input text, and reason text with an empty-reason guard.

## Best Possible Solution

Finish this as a rebased or replacement Instances-only localization patch: keep current shared loading/refresh and `common.secondsAgo` behavior, move the remaining unknown host/mode and scope labels through English source keys, guard empty reasons or deliberately harden interpolation with a focused test, and regenerate all shipped locale bundles plus `.i18n` metadata through the repo pipeline.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: under a non-English locale, `renderEntry` still emits literal English fallback/scope labels, and an empty `entry.reason` can flow through translation interpolation into a raw placeholder.

## Solution Assessment

No, not as currently proposed. The direction is correct, but the branch needs to be rebased onto current main, preserve the existing shared age helper, and commit generated locale output from `pnpm ui:i18n:sync`.
