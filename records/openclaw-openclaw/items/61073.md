## Summary

Keep this PR open. The Instances localization work is still relevant because current main only localizes part of the screen, but the branch is not merge-ready against the current locale pipeline: it adds new keys only beside English and pt-BR, while generated locale parity now covers every shipped locale.

## What This Changes

The PR adds Instances locale keys to English and pt-BR and updates the Instances view to call `t(...)` for labels, fallbacks, scope metadata, last-input text, and reason text with an empty-reason guard.

## Best Possible Solution

Finish this as a rebased or replacement Instances-only localization patch: keep current `common.secondsAgo` and shared loading/refresh behavior, add locale-backed unknown host/mode and scope labels through the English source, guard empty reasons or harden interpolation deliberately, and regenerate all shipped locale bundles plus `.i18n` metadata through the repo pipeline.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: under a non-English locale, `renderEntry` still emits literal English fallback/scope labels, and an empty `entry.reason` can flow through `t(...)` into a raw `{reason}` placeholder.

## Solution Assessment

No, not as currently proposed. The direction is correct, but the branch needs to be rebased onto current main and regenerated through `pnpm ui:i18n:sync` rather than hand-editing only pt-BR beside English.
