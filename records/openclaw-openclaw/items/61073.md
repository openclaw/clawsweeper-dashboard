## What This Changes

The PR adds Instances locale keys and rewires the Control UI Instances view to use translation lookups for labels, fallback text, scope chips, and reason metadata.

## Best Possible Solution

Finish this as a fresh Instances-only localization repair on current main: add the missing fallback and scope labels through the English source, regenerate all shipped locale bundles and `.i18n` metadata, preserve `common.secondsAgo`, and guard empty reasons.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: a non-English locale with missing host, scope, or reason data still reaches literal English fallback strings or the raw interpolation placeholder path.

## Solution Assessment

No, not as currently proposed. The direction is right, but the branch needs current-main regeneration, the shared `common.secondsAgo` helper, and real UI proof before it is a maintainable merge candidate.
