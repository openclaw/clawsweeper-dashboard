Keep open: the PR is useful, but it still needs contributor real UI proof and a focused rebase because current main already changed the Instances i18n surface while this branch adds keys only to English and pt-BR and regresses last-input localization.

## What This Changes

This PR adds `instances.*` locale keys in English and pt-BR and rewires the Control UI Instances view to use `t(...)` for labels, fallbacks, scopes, last-input, and reason metadata.

## Best Possible Solution

Land a rebased Instances i18n cleanup that updates the English source of truth, regenerates all shipped locale bundles and `.i18n` metadata, preserves shared `common.*` helpers, and includes redacted UI proof.

## Reproduction Assessment

## Solution Assessment

No. The direction is right, but the branch should rebase onto current i18n keys, preserve `common.secondsAgo`, regenerate every shipped locale bundle through the pipeline, and include real UI proof.
