## Summary

Keep open: this is a new native subagent progress-stream feature request, not stale or already implemented. Current main supports completion handoff, active subagent status context, and ACP-only parent streaming, but native `runtime="subagent"` still has no live AsyncGenerator/callback stream for tool/progress events.

## What This Changes

Expose live structured subagent progress and tool-call events to the parent agent instead of only final completion events.

## Best Possible Solution

Define a single native subagent progress-event contract that reuses existing lifecycle/transcript/queue machinery, clearly separates progress from final completion, and then update the tool schema, runtime SDK types, docs, and focused tests together.

## Reproduction Assessment

Not applicable: this is a feature request, and source inspection confirms the requested native live progress stream is not an existing documented contract. Current behavior can be verified from the native spawn schema, runtime types, docs, and tests listed above.

## Solution Assessment

Unclear: AsyncGenerator is a plausible API shape, but maintainers should first choose the event contract and ownership boundary. A one-off `streamTo` extension for native subagents would be riskier than a designed progress-event surface that fits existing completion and ACP stream paths.
