Full review comments:

- **[P2] Thread preserveKeys through every removal path:** `src/config/zod-schema.session.ts:96`
  - body: This schema makes `preserveKeys` valid config, but the configured set is only used by save-time stale pruning and count capping. `loadSessionStore({ runMaintenance })`, `openclaw sessions cleanup`, and disk-budget eviction still remove entries without checking the configured keys, so a supposedly preserved main session can still disappear through those paths.   - confidence: 0.9

  - body: Catching every `git pull --rebase` failure and continuing installation means a real conflict, network, auth, or repository error leaves the user installing a stale checkout after only a warning. If this installer change stays, handle the PowerShell stderr behavior without swallowing nonzero update failures.
  - confidence: 0.82
