Full review comments:

- **[P1] Keep cron key recovery out of the shared parser:** `src/llm/utils/json-parse.ts:151`
  - body: The reported failure is in cron argument canonicalization, but this assignment rewrites every parsed object key for all parser consumers. If a tool payload contains both a canonical key and a padded duplicate, the duplicate is silently merged before strict gateway validation can reject the conflict; move this recovery to cron's known-key canonicalizer path instead.
  - confidence: 0.9
- **[P1] Avoid assigning prototype keys into a plain object:** `src/llm/utils/json-parse.ts:151`
  - body: A model-supplied key like `__proto__ ` is trimmed to `__proto__` and assigned into a plain object, which can change the returned object's prototype. Downstream code uses membership checks on parsed tool payloads, so parser-level key repair must reject prototype-sensitive keys or avoid this path entirely.
  - confidence: 0.88
