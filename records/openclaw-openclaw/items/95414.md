- **[P1] Keep key repair out of the global parser:** `src/llm/utils/json-parse.ts:110-112`
  - body: This makes every parseJsonWithRepair caller receive renamed keys, including provider SSE events and all streaming tool arguments, even though the reported corruption is a cron job shape.   - confidence: 0.9
- **[P1] Preserve padded duplicates for strict validation:** `src/llm/utils/json-parse.ts:149`
  - body: `result[trimmed] = ...` collapses canonical and padded keys before cron validation can reject the conflict. For example, a payload containing `allowUnsafeExternalContent: false` plus `allowUnsafeExternalContent : true` can become a single true field, silently changing a security-sensitive setting instead of surfacing an additional property error.
  - confidence: 0.91
- **[P1] Block unsafe prototype keys while normalizing:** `src/llm/utils/json-parse.ts:146-149`
  - body: The helper accumulates into `{}` and then assigns attacker/model-controlled trimmed keys. If a key like `__proto__ ` is trimmed, that assignment can mutate the returned object's prototype; use an unsafe-key guard or a null-prototype accumulator at the narrow owner boundary.
  - confidence: 0.82
