Full review comments:

- **[P1] Move key recovery back to the cron boundary:** `src/llm/utils/json-parse.ts:149`
  - body: This rewrites every model-controlled object key before the owning tool schema sees it. If input contains both a canonical key and a padded duplicate, the last assignment silently wins, so strict validation cannot reject the ambiguity; keep recovery in cron canonicalization and restrict it to known cron fields.
  - confidence: 0.94
- **[P1] Do not assign trimmed __proto__ keys into plain objects:** `src/llm/utils/json-parse.ts:149`
  - body: A key like `__proto__ ` trims to `__proto__`, and assigning that into `{}` mutates the returned object's prototype instead of preserving or rejecting the JSON property. If any global normalization remains, it needs a null-prototype/defineProperty design and explicit security tests.
  - confidence: 0.82
