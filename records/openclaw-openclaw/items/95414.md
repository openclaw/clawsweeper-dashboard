Full review comments:

- **[P1] Keep padded-key recovery out of the global JSON parser:** `src/llm/utils/json-parse.ts:149`
  - body: This helper rewrites every parsed object key with `trimEnd()`. If a tool call includes both a canonical key and a padded duplicate, the latter silently overwrites the former before strict validation can reject the conflict; for example `allowUnsafeExternalContent: false` can become `true`, and `__proto__ ` becomes a prototype mutation on the returned object. This recovery should stay in the cron canonicalizer for known cron fields and preserve duplicate conflicts for schema rejection.
  - confidence: 0.92
