## Summary

Keep open: current main still matches the reported config UX behavior, and the requested changes affect invalid-config policy plus an agent-facing persistent config write boundary, so this needs maintainer product/security direction rather than cleanup closure.

## What This Changes

The issue asks OpenClaw to warn or migrate removed config keys, let read-only CLI commands degrade gracefully on non-fatal config errors, and document or relax the protected `agents.defaults.heartbeat.timeoutSeconds` gateway patch path.

## Best Possible Solution

Define a maintainer-approved config UX policy that keeps doctor as the canonical writer, explicitly chooses which read-only commands can bypass non-fatal invalid config, and either documents heartbeat timeout editing or safely expands the gateway patch allowlist with focused tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence current-main path: the legacy `fallback` key fails the strict schema, `sessions` is not an invalid-config bypass command, and `config.patch` rejects heartbeat timeout because it is outside the protected allowlist.

## Solution Assessment

Unclear. The requested direction is plausible, but the narrowest maintainable fix needs maintainer decisions across startup warnings, CLI degradation semantics, and agent-facing config write permissions.
