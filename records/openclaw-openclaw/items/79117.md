## Summary

Keep open: current main and v2026.6.8 partially address legacy `agentRuntime` detection and heartbeat config validity, but `openclaw sessions` is still not allowed to run under invalid config and agent-facing `config.patch` still protects `agents.defaults.heartbeat.timeoutSeconds`.

## What This Changes

Requests safer config-upgrade UX for deprecated keys, read-only CLI behavior under invalid config, and a documented or allowlisted path for changing heartbeat timeout through gateway config patching.

## Best Possible Solution

Keep strict runtime config validation, preserve doctor migration for legacy `agentRuntime`, decide a narrow safe invalid-config policy for `sessions`, and either document heartbeat timeout as operator-only or deliberately add it to the agent-facing allowlist with focused tests.

## Reproduction Assessment

Yes, source-level. Current main excludes `sessions` from invalid-config allowances and the agent-facing gateway tool rejects config.patch changes outside its allowlist, including heartbeat timeout; I did not execute the CLI because this review is read-only.

## Solution Assessment

Unclear as one bundled change. Doctor-based migration matches repository policy, but invalid-config read-only allowances and gateway protected-path changes need maintainer scoping before implementation.
