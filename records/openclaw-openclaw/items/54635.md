## Summary

Keep open. Current main and v2026.5.3 still route mutating tool errors through a warning branch that does not apply recoverable-error suppression, the regression suite still expects recoverable mutating errors to warn, and the related PR is closed unmerged.

## What This Changes

The issue asks OpenClaw to stop showing mutating tool failure notifications when the underlying write has recovered or already succeeded.

## Best Possible Solution

Fix the embedded runner/tool-error path so recovered or cleared mutating write failures do not render a final failure notification, while unresolved mutating failures still warn and have focused regression coverage.

## Reproduction Assessment

Yes, at source level. Passing a recoverable mutating lastToolError into buildEmbeddedRunPayloads still reaches the mutating warning branch, and the existing payload test suite explicitly expects that warning today.

## Solution Assessment

No, the one-line proposed recoverable-error check is not clearly the safest complete fix. The maintainable path should suppress only when recovery or a later matching mutation proves the write state is good, while preserving warnings for real write/edit failures.
