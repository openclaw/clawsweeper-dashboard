## Summary

Keep open. Current main and the latest v2026.5.18 release still keep recoverable-looking mutating tool errors visible, while the merged same-target edit/write recovery only handles proven recovery state and does not settle the broader warning-policy question.

## What This Changes

The issue asks OpenClaw to stop showing false failed edit/write notifications when a mutating tool error was recoverable or the write actually succeeded.

## Best Possible Solution

Adopt a maintainer-approved warning policy that clears or suppresses mutating failure notifications only when recovery is proven, while preserving visible warnings for unresolved mutating failures.

## Reproduction Assessment

Yes, at source level. Passing a recoverable mutating `lastToolError` into `buildEmbeddedRunPayloads` still reaches the mutating warning branch, and current tests expect the warning.

## Solution Assessment

No, the proposed one-line recoverable-keyword suppression is not clearly the safest fix. The maintainable direction is proof-based state clearing or a maintainer-approved delivery policy that does not hide real unresolved mutating failures.
