## What This Changes

The issue reports that an installed Weixin channel plugin loads but Gateway reports it as OFF/not configured and returns before calling the plugin's `startAccount` hook.

## Best Possible Solution

Keep this issue open until maintainers converge the related Weixin startup/account PRs and prove, with a real external Weixin account, that the loaded plugin reaches `startAccount` and reports running on current main.

## Reproduction Assessment

Source-reproducible, but not with a high-confidence live current-main run: Gateway still has pre-`startAccount` returns for empty account ids or unconfigured accounts, and the published Weixin plugin depends on its account index plus token state. The reporter's exact Windows state still needs proof of whether `accounts.json` contains the saved account id.

## Solution Assessment

Unclear as a core-only fix. The safest path is maintainer convergence across the external plugin account index, OpenClaw startup planning, and status diagnostics rather than a new Weixin-specific core workaround.
