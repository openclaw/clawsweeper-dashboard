The recent field report reinforces the same unmet scheduling need, but the proposed condition language and state source need maintainer product/API direction before implementation.

## What This Changes

Request to add deadline- or state-aware conditional cron/heartbeat intervals so scheduled work can change cadence based on runtime conditions.

## Best Possible Solution

Maintainers should choose a narrow scheduler contract or plugin API for state-aware next-run calculation, then update cron persistence, protocol/tool/CLI/docs, and focused scheduler tests together.
