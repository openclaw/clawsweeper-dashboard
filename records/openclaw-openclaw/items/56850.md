## Summary

Keep open. Current main and v2026.5.22 still expose only static `at`, `every`, and `cron` schedule shapes, while the linked zeroclaw prior-art PR is closed unmerged and OpenClaw has no canonical issue or PR that now owns this exact conditional-cadence contract.

## What This Changes

Requested support for conditional cron intervals that shorten the next heartbeat cadence when a state predicate, such as an approaching deadline, is true.

## Best Possible Solution

Define a maintainer-approved conditional scheduling contract, preferably with restricted predicates or a plugin scheduler hook, then align cron types, Gateway schema, persistence, CLI/tool input, docs, and focused next-run tests.

## Reproduction Assessment

Not applicable as a feature request. Source inspection shows current main only supports static `at`, `every`, and `cron` schedules, so there is no broken existing behavior to reproduce.

## Solution Assessment

No for the literal free-form YAML predicate as written. The maintainable path needs a restricted predicate contract or a scheduler extension hook with clear security and compatibility boundaries.
