## Summary

Keep open. Current main and the latest release still only support static `at`, `every`, and `cron` schedules; the useful zeroclaw prior-art PR is closed unmerged in another repo, and no canonical OpenClaw item now owns the exact conditional-cadence contract.

## What This Changes

Allow cron jobs to define a conditional shorter interval based on scheduler-visible state so deadline-sensitive workflows can run sooner than their base cadence.

## Best Possible Solution

Define a backward-compatible conditional scheduling contract, preferably with restricted predicates or an explicit scheduler hook, then align cron types, Gateway schema, CLI input, persistence, docs, and next-run tests.

## Reproduction Assessment

Not applicable as a feature request. Source inspection shows current main only supports static `at`, `every`, and `cron` schedules, so there is no broken existing behavior to reproduce.

## Solution Assessment

No for the literal free-form YAML expression as written. The maintainable path is a maintainer-approved predicate contract or scheduler hook with clear security and compatibility boundaries.
