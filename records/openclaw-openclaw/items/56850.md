## Summary

Keep open. Current main and the latest shipped release still expose only static cron schedules, while the linked zeroclaw implementation is useful prior art but is closed unmerged in another repo and does not implement this OpenClaw feature.

## What This Changes

This issue requests deadline-aware cron scheduling so a job can switch to a shorter interval when scheduler-visible state conditions are met.

## Best Possible Solution

Define a backward-compatible conditional scheduling contract with restricted predicates or an approved scheduler hook, then align cron types, gateway schema, CLI/tool inputs, persistence, docs, and scheduler coverage.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main and v2026.5.19 shows only static `at`, `every`, and `cron` schedules, with no conditional cadence contract to reproduce as broken.

## Solution Assessment

No for the literal free-form YAML expression as written. The maintainable path is a maintainer-approved predicate contract or scheduler hook with clear security boundaries, not arbitrary state-expression evaluation in cron config.
