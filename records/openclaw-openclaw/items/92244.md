## Summary

Keep open: current main and the latest release still contain the reported picker call shape, and https://github.com/openclaw/openclaw/pull/92254 is the open closing PR for this bug rather than a merged fix.

## What This Changes

The issue reports that Telegram model picker selection can silently clear session auth-profile override fields because its `applyModelOverrideToSessionEntry` call omits the preserve flag.

## Best Possible Solution

Land or replace the linked picker fix so interactive model selection preserves auth-profile overrides it does not own, with focused regression coverage for the helper/caller invariant.

## Reproduction Assessment

Yes from source inspection: Telegram's picker call omits `preserveAuthProfileOverride`, and the helper deletes auth-profile override fields when that flag and `profileOverride` are absent. I did not run a live Telegram reproduction in this read-only review.

## Solution Assessment

Yes, the active linked PR's caller-intent fix is a narrow maintainable path because pickers choose models and do not own auth-profile overrides. A helper-level source-aware guard remains an alternative if maintainers want broader defense for future callers.
