## Summary

Keep open: current main and latest release still have model-picker callers that omit the auth-profile preservation flag, and the linked closing pull request is open but unmerged.

## What This Changes

The issue reports that Telegram model picker selection can silently clear a session entry's existing auth profile override.

## Best Possible Solution

Review and land a narrow picker fix that preserves compatible existing auth profile overrides across Telegram and sibling native pickers, with focused regression coverage, then close this issue after the fix merges.

## Reproduction Assessment

Yes, by source inspection. Pre-seeding a session entry with `authProfileOverride` and then using the Telegram picker reaches a helper branch that clears those fields because the picker does not pass `preserveAuthProfileOverride`.

## Solution Assessment

Yes, the best fix belongs at the picker caller boundary and sibling native pickers using the existing preservation contract. The linked PR is the active implementation candidate, but the issue should remain open until it lands or is replaced.
