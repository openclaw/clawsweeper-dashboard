## Label Justifications

- `P1`: A routine model-picker action can silently erase persisted auth-routing state and cause subsequent agent turns to fail or use the wrong credentials.
- `impact:session-state`: The issue concerns destructive mutation of persisted session-entry override fields.
- `impact:auth-provider`: The erased override determines which authentication profile subsequent provider calls use.

## Summary

Codex review confirms the reported current-main bug: Telegram and Discord interactive model pickers omit the existing preservation intent, so selecting a model can silently clear unrelated persisted auth-profile state. Keep this issue open because the focused fixing PR is still open and explicitly owns the implementation.

## What This Changes

Prevent interactive native model pickers from clearing auth profile overrides they did not establish.

## Best Possible Solution

Land the linked caller-intent fix for every interactive native picker with persisted-session regression coverage, while retaining the shared helper's explicit clearing behavior for callers that own the auth-profile transition.

## Reproduction Assessment

Yes, from source with high confidence: seed a session entry with a user-sourced auth profile override, invoke either native picker on current main, and the helper's default-clear branch removes the override fields. The linked PR also reports a focused before/after regression test, although this read-only issue review did not execute it.

## Solution Assessment

Yes, the linked PR's caller-intent approach is the narrowest maintainable fix because interactive pickers do not own the existing auth-profile binding. The issue's suggested source-aware shared-helper rewrite is broader than the demonstrated failure and could alter intentional clearing semantics for other callers.
