## Label Justifications

- `P1`: The report describes a real Telegram model-picker workflow that can silently lose auth-profile session state and affect subsequent provider calls.
- `impact:data-loss`: The affected path deletes persisted auth override fields from the session entry without warning.
- `impact:session-state`: The lost fields are per-session model/auth routing state used by later turns.
- `impact:auth-provider`: The deleted authProfileOverride controls which provider credentials or profile subsequent model calls use.

## Summary

Keep open: current main and v2026.6.9 still contain the destructive helper path, and the Telegram picker still calls it without auth-profile preservation. The linked closing pull request is open and unmerged, so this issue remains the canonical tracker until that fix lands or is replaced.

## What This Changes

The issue requests that Telegram model picker model changes stop silently clearing auth profile override fields set by non-picker sources.

## Best Possible Solution

Land or replace the linked provider-aware preservation PR, with regression coverage proving picker model changes preserve compatible auth profile overrides and clear only intentionally incompatible ones.

## Reproduction Assessment

Yes. Source inspection shows an entry with authProfileOverride reaches the Telegram picker applyModelOverrideToSessionEntry call without preserveAuthProfileOverride, and the shared helper then deletes the auth-profile fields when preservation is absent.

## Solution Assessment

Yes. The narrow maintainable direction is a shared provider-aware preservation rule reused by Telegram, Discord, and sessions.patch; the open linked PR is the active candidate but must merge before this issue can close.
