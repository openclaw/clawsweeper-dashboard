## Summary

Keep open: current main and the latest release still have the destructive helper behavior, and the linked open PR is the active fix path but has not merged.

## What This Changes

The issue reports that Telegram model-picker selection can clear an existing session auth profile override because the picker applies a model override without requesting auth-profile preservation.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/92254 or an equivalent narrow patch that preserves auth profile overrides for interactive model-picker call sites and keeps regression coverage for the shared picker invariant.

## Reproduction Assessment

Yes. A source-level reproduction is clear: seed a session entry with `authProfileOverride`, take the Telegram picker callback path, and the helper clears the fields because the caller omits `preserveAuthProfileOverride`.

## Solution Assessment

Yes. The best current path is the linked caller-intent fix using the existing preserve flag on interactive pickers; changing the helper's deletion semantics can be a separate maintainer decision if desired.
