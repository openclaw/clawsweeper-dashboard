## Summary

Keep this issue open: current main still has the source-level failure path, the latest release also carries the destructive helper behavior, and the linked closing PR is still open and unmerged.

## What This Changes

This issue reports that Telegram model-picker selections clear stored auth profile override fields because the picker calls `applyModelOverrideToSessionEntry` without `preserveAuthProfileOverride`.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/92254 or an equivalent focused fix that preserves auth profile overrides for native picker call sites with regression coverage, then close this issue as implemented.

## Reproduction Assessment

Yes at source level: current main's Telegram picker passes no preserve flag, and the helper then deletes auth profile override fields. I did not run a live Telegram interaction in this read-only review.

## Solution Assessment

Yes, the narrow maintainable path is a caller-intent fix across native picker surfaces using the helper's existing preserve contract rather than adding a broad new runtime fallback.
