## Summary

Keep open. Current main still has source-reproducible Gemini signature replay and tagged/native reasoning conflicts, while related open trackers cover only parts of the broader failure and do not close the separate textSignature bloat path.

## What This Changes

The issue reports Gemini 2.5 Pro session failures from retained text signatures, tagged `<think>/<final>` handling, and mixed text plus tool-call delivery behavior.

## Best Possible Solution

Resolve this at the Google provider and replay boundary: preserve required current-turn Gemini function/native-thinking signatures, bound or avoid replaying stale visible-text signatures, and align Google/Gemini reasoning output with native thinking while coordinating with the open tagged-reasoning and inter-tool-delivery trackers.

## Reproduction Assessment

Yes for the core source paths: current main stores and replays Gemini visible text signatures and still combines tagged reasoning mode with native Gemini thinking payloads. No fresh live 5-10 turn Telegram/Gemini run was performed, so the end-to-end timeout/silent-delivery symptom remains unproven in this pass.

## Solution Assessment

Unclear for automation. The affected files are narrow, but the correct signature-retention rule depends on the Gemini thought-signature contract and should be coordinated with https://github.com/openclaw/openclaw/issues/69220 and https://github.com/openclaw/openclaw/issues/25592.
