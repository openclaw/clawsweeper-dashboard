## Summary

Keep open: current main and v2026.6.10 only partially address the report. Direct Google now uses native reasoning, but visible Gemini text signatures are still stored and replayed as `thoughtSignature`, so the session-bloat failure remains unresolved.

## What This Changes

The issue asks OpenClaw to prevent Gemini session bloat and dropped replies caused by large text signatures, tagged reasoning text, and mixed text/tool turns.

## Best Possible Solution

Define a Google-provider replay policy that preserves required function-call and native-thinking signatures while bounding, externalizing, or dropping optional visible-text signatures, then prove it with focused replay tests and live Gemini coverage.
