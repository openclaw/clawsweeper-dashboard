## Summary

Keep open. Current main and the latest shipped release still preserve and replay Gemini visible-text signatures, still configure Google/Gemini for tagged reasoning while also sending native Gemini thinking payloads, and still suppress final-less tagged output; the related open reasoning issue covers only part of the report, not the visible `textSignature` bloat path.

## What This Changes

The issue requests fixes for Gemini session bloat from persisted `textSignature`, suppressed `<think>`/final-tag output, and dropped user-visible text around mixed text plus tool-call turns.

## Best Possible Solution

Fix this at the Google provider and replay boundary: preserve required function-call/native-thinking signatures, bound or avoid replaying stale visible-text signatures, and align Gemini reasoning delivery with native thinking so malformed tagged output cannot suppress replies.

## Reproduction Assessment

Yes, at source level: current main deterministically stores and replays same-route Gemini visible-text signatures, keeps Google/Gemini on tagged reasoning while sending native thinking payloads, and suppresses final-less tagged text. I did not run a paid multi-turn Gemini/Telegram loop.

## Solution Assessment

Unclear for automation. The issue direction is valid, but the best fix needs maintainer judgment on required versus optional Gemini signature replay and on whether Gemini should remain tagged or move to native reasoning output.
