## Summary

Keep open. Current main and the latest release still retain and replay Gemini text signatures, and Google/Gemini still combines tagged reasoning enforcement with native Gemini thinking payloads, so the report is not implemented or stale.

## What This Changes

The issue reports Gemini 2.5 Pro session failures from stored and replayed text signatures, embedded reasoning tags, and mixed text/tool responses.

## Best Possible Solution

Fix this at the Google provider replay boundary: preserve required Gemini function-call/native-thinking signatures, bound or avoid stale visible-text signature replay, and align Gemini reasoning delivery with native thinking while coordinating the narrower open trackers.

## Reproduction Assessment

Yes, at source level: current main deterministically stores/replays Gemini visible text signatures and still combines tagged reasoning mode with native Gemini thinking payloads. No fresh live 5-10 turn Telegram/Gemini run was performed, so the full timeout/silent-delivery symptom is not end-to-end reproduced here.

## Solution Assessment

No for automatic cleanup. The affected files are narrow, but the correct fix depends on Gemini signature-retention semantics and should be coordinated with https://github.com/openclaw/openclaw/issues/69220 plus the broader delivery semantics in https://github.com/openclaw/openclaw/issues/25592.
