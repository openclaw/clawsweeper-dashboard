## Summary

Keep open. Current main still persists Gemini visible-text signatures and replays them as Gemini thought signatures, while the replay sanitizer only targets thought-signature fields. The mixed text-before-tool part now has flush protection, but the signature bloat and Gemini tagged-reasoning/final-tag suppression parts remain unresolved.

## What This Changes

Fix Gemini 2.5 Pro sessions so text signatures do not bloat/replay history, tagged thinking does not suppress replies, and text emitted before tool calls is delivered.

## Best Possible Solution

Handle this at the Google provider and replay boundary: retain only Gemini signatures required for valid native-thinking/tool continuation, avoid replaying or bound stale visible-text signatures, and align Gemini reasoning delivery with native thinking or a fallback that cannot silently suppress final-less visible replies.

## Reproduction Assessment

Yes. There is a high-confidence source-level reproduction path: a same-provider Google assistant text block with textSignature is replayed as thoughtSignature, and enforced final-tag delivery suppresses text without a <final> start tag; the live Gemini/Telegram loop was not run.

## Solution Assessment

No current-main solution is complete. The text-before-tool flush path is partially addressed, but signature persistence/replay and Gemini tagged-versus-native reasoning still need a provider-owner decision before a narrow implementation is safe.
