## Summary

Keep open: current main and v2026.6.6 fix the direct-Gemini tagged/native reasoning conflict, and pre-tool text flushing now has coverage, but the separate Gemini visible-text `textSignature` storage and same-route replay path still exists, so the core session-bloat failure is not fully implemented or superseded.

## What This Changes

The issue asks OpenClaw to prevent Gemini text/thinking signature bloat and mixed text/tool replies from corrupting session replay or suppressing Telegram delivery.

## Best Possible Solution

Define a Google-owned replay policy that preserves required tool-call/native-thinking signatures while bounding, externalizing, or avoiding replay of optional visible-text signatures from Gemini responses.

Yes, at source level: current Google streaming stores visible text signatures and same-route replay sends them back as `thoughtSignature`. I did not establish a fresh paid Gemini plus Telegram failure in this read-only review.

## Solution Assessment

No, current main is only a partial fix. The tagged reasoning symptom is fixed, but the remaining session-bloat fix needs a provider-owned signature retention and replay contract rather than a blanket strip.
