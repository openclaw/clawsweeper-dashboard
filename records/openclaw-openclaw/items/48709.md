## Summary

Keep open. Current main still has the central Gemini signature-bloat path, and the Gemini tagged-reasoning/final-tag suppression path remains unresolved; only the mixed text-before-tool delivery path appears partially covered now.

## What This Changes

The issue requests fixes for Gemini 2.5 Pro session bloat from stored text signatures, tagged reasoning suppression, and dropped text around mixed text/tool turns.

## Best Possible Solution

Handle this at the Google provider and replay boundary: preserve only Gemini signatures required for valid continuation, bound or drop stale visible-text signatures, and remove the conflict between Gemini native thinking and tagged final-tag delivery.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: same-provider Google textSignature values are persisted and replayed as thoughtSignature, while enforced final-tag delivery suppresses final-less visible text. I did not run the live Gemini/Telegram loop.

## Solution Assessment

No current-main solution is complete. The text-before-tool flush path is partially protected, but signature retention/replay and Gemini tagged-versus-native reasoning need a provider-owner decision before a narrow implementation is safe.
