## Summary

Keep open. Current main and latest release still retain and replay valid same-route Gemini text signatures and still wire Google/Gemini to tagged reasoning while native Gemini thinking is enabled; the merged signature hardening only drops malformed/truncated signatures, and the mixed text-before-tool path is only partially covered by flush behavior.

## What This Changes

The issue asks OpenClaw to stop Gemini 2.5 Pro session bloat and reply loss caused by persisted text signatures, embedded reasoning tags, and mixed text plus tool-call turns.

## Best Possible Solution

Fix this at the Google provider and replay boundary: bound or avoid replaying stale visible-text signatures, preserve required Gemini function-call/native-thinking signatures, and align Google/Gemini reasoning delivery with the native-thinking decision tracked by the related Gemini reasoning issue.

## Reproduction Assessment

Yes, at source level: current main deterministically stores and replays same-route Gemini text signatures, keeps Google/Gemini on tagged reasoning, enables native Gemini thinking payloads, and suppresses final-less tagged output. I did not run the paid live Gemini plus Telegram loop.

## Solution Assessment

Unclear for automation. The requested direction is valid, but maintainers need to choose which Gemini signatures must be retained or replayed and whether Google/Gemini should continue using tagged reasoning when native thinking is enabled.
