## Summary

Keep open. Current main and the latest stable release still preserve and replay Gemini visible-text `textSignature` values, and the Google provider still combines tagged reasoning mode with native Gemini thinking; related issue https://github.com/openclaw/openclaw/issues/69220 covers the tagged/native part but not the separate signature-bloat path.

## What This Changes

The issue reports Gemini session failures caused by stored/replayed `textSignature` bloat, `<think>`/`<final>` tagged reasoning conflicts, and mixed text/tool turns dropping user-visible output.

## Best Possible Solution

Fix this at the Google provider and replay boundary: preserve required function-call/native-thinking signatures, bound or stop replaying non-required visible-text signatures, and align Gemini reasoning delivery with native thinking while coordinating the related tagged/native tracker.

## Reproduction Assessment

Yes, at source level: current main deterministically stores/replays same-route Gemini visible text signatures and still combines Google tagged reasoning with native `thinkingConfig`. No fresh paid 5-10 turn Gemini/Telegram loop was run, so the full timeout and silent-delivery symptom remains user-reported.

## Solution Assessment

Unclear for automation. The issue direction is valid, but the best fix needs maintainer judgment on required versus optional Gemini signature replay and on whether Gemini should remain tagged or move to native reasoning output.
