## Summary

Keep this PR open: the Mattermost ack feature is useful and not already on current main, but the current patch can remove an ack after a suppressed reasoning-only final where no visible Mattermost reply was delivered.

## What This Changes

The PR adds Mattermost automatic ack reactions, channel/account ackReaction overrides, emoji-name normalization, docs/generated config metadata, and focused unit coverage.

## Best Possible Solution

Keep the Mattermost-owned ack feature, but make ack cleanup depend on actual visible reply delivery or preview finalization and add a regression for suppressed reasoning-only replies.

## Reproduction Assessment

Yes for the review finding: source inspection shows the helper intentionally suppresses reasoning-prefixed Mattermost finals, while the PR marks delivery after that helper returns. I did not execute tests because this was a read-only review.

## Solution Assessment

No; the current patch should not treat every successful helper return as a delivered reply. The narrower maintainable fix is to have the delivery path return or record whether a visible Mattermost reply was actually sent or preview-finalized before removing the ack.
