## Summary

This issue is the surviving canonical report after the author closed the earlier identical issue as its duplicate. Current main still hardcodes the five directory groups across scanning, compilation, generated indexes, and page-kind inference, so the requested capability remains missing; however, the proposed global-plus-vault configuration precedence is a product and storage-contract decision rather than a safe autonomous fix.

## What This Changes

Add configurable memory-wiki page groups, custom directory and heading mappings, optional vault-local overrides, and recursive Markdown scanning.

## Best Possible Solution

If maintainers accept custom layouts, define one shared plugin-owned page-group registry consumed by configuration, querying, creation routing, kind inference, compilation, and index generation; prefer the existing plugin configuration as the single canonical source unless a vault-local override contract is explicitly approved.

## Solution Assessment

No. The central capability is reasonable, but adding recursive query scanning and two layered configuration sources without first centralizing the shared page-group contract would create competing layout behavior across the plugin.
