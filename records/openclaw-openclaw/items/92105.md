## Summary

The configurable page-group capability remains absent from current main, and no viable open PR owns that unique work. The former implementation PR closed unmerged because its final branch no longer contained the memory-wiki changes; the open recursion PR addresses only nested-page discovery, so this issue should remain the canonical place for the configuration product decision.

## What This Changes

Add configurable memory-wiki page groups, custom vault directories and headings, vault-local overrides, and recursive Markdown discovery.

## Best Possible Solution

Land or replace the narrow recursive-scanning fix independently, then approve one canonical plugin-owned page-group configuration authority and invite a fresh focused PR covering reads, writes, generated reports, docs, defaults, and upgrade behavior.

## Reproduction Assessment

Yes at source level: current main fixes the query surface to five named directories and exposes no page-group configuration. The recursive-scanning portion also has a concrete current-main reproduction in https://github.com/openclaw/openclaw/issues/90869.

## Solution Assessment

No, not as one bundled implementation. Recursive scanning is a narrow existing-behavior repair, while custom directories, multiple directories per kind, and global-versus-vault precedence are a separate product and compatibility contract.
