## Summary

Keep open: the static bundled-channel fallback is useful, but this PR does not fix the reported bare `telegram:slash:<id>` path because that session key never reaches the patched inference branch.

## What This Changes

The PR imports `normalizeChatChannelId` in `src/commands/sandbox-explain.ts` and uses it as a fallback after runtime channel-registry normalization during session-key channel inference.

## Best Possible Solution

Land one consolidated sandbox-explain fix that wraps non-agent-scoped colon session keys, applies bundled-channel fallback for inferred and stored channel metadata, and covers bare plus agent-scoped Telegram slash sessions with an empty runtime channel registry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: pass `--session telegram:slash:<id>`, observe that it remains raw, fails `parseAgentSessionKey`, and never reaches the fallback added by this PR.

## Solution Assessment

No. The proposed fallback is a necessary piece, but it is not sufficient without the non-agent colon-key normalization tracked by #51286 or an equivalent inference path.
