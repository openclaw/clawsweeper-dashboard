## Summary

Keep open: the static fallback is useful for agent-scoped keys, but this PR still misses the stated bare `telegram:slash:<id>` CLI path because unscoped colon keys fail agent-session parsing before the new fallback can run. It also needs real behavior proof before merge.

## What This Changes

The PR imports `normalizeChatChannelId` and uses it as a static fallback after `normalizeAnyChannelId` in `sandbox explain` channel inference.

## Best Possible Solution

Land one consolidated fix that normalizes non-agent-scoped channel session keys, falls back to the bundled channel catalog when plugins are unloaded, and covers `sandbox explain --session telegram:slash:<id>` elevated output.

## Reproduction Assessment

Yes, by source inspection. Current main leaves `telegram:slash:<id>` unscoped, `parseAgentSessionKey` rejects it, and `sandbox explain` therefore keeps `channel` undefined and `allowedByConfig` false.

## Solution Assessment

No. The static fallback is a necessary piece for no-registry CLI inference, but the reported bare session key also needs normalization before this fallback is reachable.

## Review Findings
