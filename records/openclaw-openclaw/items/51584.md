## Best Possible Solution

Land one reconciled fix that both wraps raw channel-prefixed session keys and falls back to bundled channel IDs when plugins are unloaded, with focused regression coverage and redacted real CLI or Telegram proof.

## Reproduction Assessment

Yes, source-level. Current main leaves `telegram:slash:<id>` unscoped, `parseAgentSessionKey` rejects it, and unresolved channel identity keeps `allowedByConfig` false.

## Solution Assessment

No as a standalone PR. The static fallback is necessary for agent-scoped keys when plugins are unloaded, but the best fix also needs raw session-key normalization and real behavior proof.
