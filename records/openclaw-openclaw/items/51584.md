## Summary

Keep this PR open: the static channel fallback is a useful half of the fix, but it is unreachable for the stated bare `telegram:slash:<id>` command and still lacks real behavior proof.

## What This Changes

The PR imports `normalizeChatChannelId` and uses it as a fallback after `normalizeAnyChannelId` when `sandbox explain` infers a channel from a session key.

## Best Possible Solution

Land one reconciled fix that normalizes bare channel session keys, resolves bundled channels when plugins are unloaded, and includes regression coverage plus redacted real CLI proof.

## Reproduction Assessment

Yes, by source inspection. The stated `telegram:slash:<id>` input remains non-agent-scoped, `parseAgentSessionKey` rejects it, and this PR's fallback is not reached for that command.

## Solution Assessment

No. The static fallback is necessary for unloaded plugin registries, but the stated path also needs bare session-key normalization before that fallback can run.
