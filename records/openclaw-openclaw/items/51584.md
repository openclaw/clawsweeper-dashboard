## Summary

Keep open. The fallback is a useful half of the fix, but the reported bare `telegram:slash:<id>` key still stays unscoped and exits channel inference before the new fallback can run; the external PR also lacks after-fix real behavior proof.

## What This Changes

The PR imports `normalizeChatChannelId` and uses it as a fallback after `normalizeAnyChannelId` in `sandbox explain` channel inference.

## Best Possible Solution

Land one reconciled fix that normalizes non-agent-scoped channel session keys, falls back to bundled channel ids when plugins are unloaded, and covers elevated `sandbox explain --session telegram:slash:<id>` diagnostics with regression coverage and redacted real CLI proof.

## Reproduction Assessment

Yes, by source inspection. The reported `telegram:slash:<id>` input remains unscoped on current main, fails agent-session parsing, leaves `channel` undefined, and keeps `allowedByConfig` false.

## Solution Assessment

No. The static fallback is necessary for the no-registry CLI path, but the reported bare session key also needs to be normalized or wrapped before this fallback is reachable.
