## What This Changes

The PR makes sandbox workspace setup fall back to the configured agent workspace when an explicit workspace path is omitted or blank, with regression coverage for both cases.

## Best Possible Solution

Land the canonical resolver fallback and its omitted/blank-input regressions while continuing to treat every nonblank caller-provided workspace path as authoritative.

## Reproduction Assessment

Yes. Current main has a direct omitted/blank fallback to the environment-only default, and the supplied real-filesystem production-function run demonstrates the old path mismatch and successful configured-workspace skill synchronization after the patch.

## Solution Assessment

Yes. Reusing `resolveAgentWorkspaceDir` at the existing fallback is the narrowest maintainable fix because it preserves explicit workspace paths and introduces neither a new configuration surface nor a parallel workspace policy.
