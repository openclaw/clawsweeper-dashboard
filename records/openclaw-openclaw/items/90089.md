## Summary

The PR is still necessary: current main falls back to the global default workspace for omitted or blank sandbox workspace inputs, while the established resolver honors configured default and named-agent workspaces. The proposed boundary is correct, the live filesystem proof is convincing, and no duplicate or current-main implementation was found; ordinary maintainer review and the failing relevant CI lanes remain before merge.

## What This Changes

The PR changes sandbox skill synchronization to use the config-resolved agent workspace when `workspaceDir` is omitted or blank and adds focused regression tests for both inputs.

## Best Possible Solution

Land one canonical workspace policy: preserve caller-supplied nonblank paths, otherwise resolve the current agent’s configured workspace, with the omitted and blank regressions retained and relevant CI green or proven unrelated.

## Reproduction Assessment

Yes. On current main, omit `workspaceDir` or pass an empty string while configuring a different `agents.defaults.workspace`; source inspection shows sandbox skill sync selects `DEFAULT_AGENT_WORKSPACE_DIR` instead of the configured path.

## Solution Assessment

Yes. Reusing `resolveAgentWorkspaceDir` at the existing fallback boundary is the narrowest maintainable fix because it preserves explicit inputs and removes duplicate workspace-default policy.
