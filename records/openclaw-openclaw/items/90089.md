## Summary

Keep open: current main still falls back to the hardcoded default workspace when sandbox layout is called without `workspaceDir`, while the PR narrowly switches that fallback to the existing config-aware resolver and adds focused coverage. ## What This Changes

The PR changes sandbox skill sync fallback to use the config-resolved agent workspace when no explicit `workspaceDir` is provided and adds a regression test for that fallback.

## Best Possible Solution

Merge the narrow resolver alignment once maintainers accept the omitted-workspace fallback semantics; optional extra tests for blank input and per-agent workspaces would reduce review uncertainty but are not required to fix the reported path.

## Reproduction Assessment

## Solution Assessment

Yes. Reusing `resolveAgentWorkspaceDir` is the narrow canonical fix and avoids duplicating workspace config parsing; the main alternative, changing skill sync itself, would be a broader layer for a caller fallback bug.
