## Summary

Keep open: current main and v2026.6.9 already have a browser-local tool-call visibility toggle, an Activity tab, and collapsed tool summaries for many cases, but the requested default/config decision and non-terminal error de-promotion are not fully implemented.

## What This Changes

The issue requests Control UI chat changes to collapse activity/tool blocks by default, suppress partial assistant fragments during tool activity, add an activity default config option, and hide transient error activity after successful retry.

## Best Possible Solution

Make a maintainer UX decision first, reuse the existing local tool-call visibility setting and Activity tab where possible, and coordinate any non-terminal error fix with the open render-boundary PR.

## Reproduction Assessment

Partially yes: source inspection confirms the missing config key and remaining error auto-expansion path. I did not establish a live browser reproduction for the exact screenshot state, and one earlier pre-tool text race already has gateway flush coverage.

## Solution Assessment

Unclear: collapsing noisy activity is aligned with the UX goal, but the requested gateway config key is not proven to be the narrowest layer compared with a local preference or default-only UI change.
