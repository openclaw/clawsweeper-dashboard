## Summary

Keep open: current main and v2026.6.10 have the collapsed non-error Activity summary and a browser-local tool-call visibility toggle, but the remaining request still includes stream-fragment suppression, error de-promotion after successful turns, and a new config/default decision that is not fully implemented.

## What This Changes

The issue requests Control UI chat readability changes: collapse tool activity by default, suppress partial assistant fragments between activity blocks, and auto-collapse retry-recovered error blocks.

## Best Possible Solution

Define the desired Control UI chat default first, then implement it at the chat projection/rendering boundary while reusing the existing Activity tab and local tool-call visibility preference unless maintainers explicitly want a gateway config surface.

## Reproduction Assessment

Yes, at source level for the remaining behavior: current main still interleaves stream segments with tool messages and still promotes tool-card errors, while the requested config option is absent. I did not run a live browser reproduction or inspect an actual screenshot.

## Solution Assessment

Unclear. Collapsing noisy activity by default is plausible, but the requested `gateway.controlUi.activityDefault` is not obviously the best layer until maintainers choose between a default-only change, browser-local preference, or server config.
