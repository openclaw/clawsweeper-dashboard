## Summary

Keep open: current main and v2026.6.10 already have a compact Activity summary, an Activity tab, and a persisted tool-call visibility toggle, but they still render stream segments around tool messages and still promote failed tool cards unless the open non-terminal error PR lands.

## What This Changes

The issue requests a Control UI chat UX change: collapse tool activity by default, suppress partial streamed answer fragments around tool blocks, and hide non-terminal retry errors after success.

## Best Possible Solution

Define the desired Control UI chat projection policy first, reuse the existing local tool-call preference and Activity tab where possible, and land https://github.com/openclaw/openclaw/pull/90122 or an equivalent render-boundary fix for non-terminal failed tools.

## Reproduction Assessment

Yes, at source level for the remaining behavior: current main still interleaves stream segments with tool messages and promotes error tool cards, while the requested gateway config option is absent. I did not run a fresh live Dashboard browser reproduction.

## Solution Assessment

Unclear. Collapsing noisy activity may be the right UX, but the proposed gateway config option is not proven to be the best layer compared with an adjusted default, the existing browser-local preference, and the linked non-terminal error fix.
