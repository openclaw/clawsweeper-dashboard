## Summary

Keep open: current main partially addresses the clutter with collapsed activity summaries, an Activity tab, and a local show/hide tool-calls preference, but it still intentionally renders stream fragments around tool activity, defaults tool calls to visible, and leaves the non-terminal error subproblem to an open PR.

## What This Changes

The issue requests a cleaner Control UI chat transcript by collapsing tool activity by default, suppressing partial assistant fragments around tool calls, and hiding non-terminal tool errors after successful retry.

## Best Possible Solution

Define the desired Control UI chat transcript policy first, then implement it at the chat projection/rendering boundary while reusing the existing Activity tab and browser-local tool-call preference unless maintainers explicitly choose a gateway config surface.

## Reproduction Assessment

Yes, at source level for the remaining behavior: current main still interleaves stream segments with tool messages, defaults tool-call visibility to on, and lacks the requested gateway config option. I did not run a live browser reproduction in this read-only review.

## Solution Assessment

Unclear. Collapsing noisy activity is plausible, but the proposed `gateway.controlUi.activityDefault` option is not obviously the best layer while the Control UI already has a browser-local tool-call preference and a separate Activity tab.
