## Summary

Keep open: current main and latest release already have a compact multi-tool Activity summary and a separate Activity tab, but the issue's remaining behavior is not fully solved because stream fragments are still intentionally interleaved with tool cards, error activity can remain prominent, and the requested Control UI config/default needs maintainer UX direction.

## What This Changes

The issue asks for Control UI chat activity blocks to default collapsed, partial streaming fragments to be hidden or separated, non-terminal error activity to auto-collapse after retry success, and a new Control UI activity default config option.

## Best Possible Solution

Define the desired Control UI chat default and config scope, then implement the UX at the chat projection/rendering boundary while reusing existing Activity and local tool-call visibility surfaces when possible.

## Reproduction Assessment

Yes at source level for the remaining pieces: current code still interleaves stream segments with tool messages and has no requested server-side `activityDefault` config key. I did not run a live browser reproduction.

## Solution Assessment

Unclear. Collapsing noisy activity may be right, but adding a gateway config key is not clearly better than using the existing local tool-call visibility setting or a maintainer-chosen default-only change.
