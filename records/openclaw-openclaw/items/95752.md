## Summary

Keep open: current main has a tool-call visibility toggle and grouped Activity summaries, but it still shows tool activity by default, still renders completed stream fragments around tool messages, and does not include the requested Control UI activity-default config or product decision.

## What This Changes

The issue requests quieter Control UI chat rendering by collapsing tool activity by default, suppressing partial assistant fragments around tool calls, and de-promoting retried or non-terminal tool-error activity.

## Best Possible Solution

Define the Control UI transcript visibility contract, then implement the default at the chat projection/rendering boundary while reusing the existing Activity tab and browser-local tool-call visibility preference unless maintainers explicitly approve a gateway config key.

## Reproduction Assessment

Yes at source level: current main still renders stream segments and tool activity by default, and the proposed gateway activity-default setting is absent. No live browser reproduction was run in this read-only pass.

## Solution Assessment

Unclear. A quieter default is plausible, but adding gateway.controlUi.activityDefault is not obviously the best layer while a browser-local chatShowToolCalls preference and an Activity tab already exist.
