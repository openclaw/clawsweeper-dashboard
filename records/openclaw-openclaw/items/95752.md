## Summary

Keep open: current main already has the broad collapsed Activity summary and an Activity tab, but the remaining request mixes chat-stream suppression, non-terminal error de-promotion, and a new Control UI config/default decision that is not fully solved on main.

## What This Changes

Request to make Control UI chat tool activity less noisy by default, suppress in-progress assistant fragments around tool blocks, auto-collapse non-terminal error activity, and optionally add a `gateway.controlUi.activityDefault` setting.

## Best Possible Solution

Define the desired Control UI chat default first, then implement it at the chat projection/rendering boundary while reusing the existing Activity tab and local tool-call visibility preference unless maintainers explicitly want a gateway config surface.

## Reproduction Assessment

Yes, at source level for the remaining behavior: current main still interleaves stream segments with tool messages and promotes error tool cards, while the requested config option is absent. I did not run a live browser reproduction or inspect an actual screenshot.

## Solution Assessment

Unclear. Collapsing noisy activity by default is likely right, but adding `gateway.controlUi.activityDefault` is not obviously the best fix until maintainers choose between a default-only change, browser-local preference, or server config.
