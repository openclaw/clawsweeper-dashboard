## Summary

Keep open: current main already has collapsed tool summaries and a separate Activity tab, but the remaining request still spans default tool visibility, stream-fragment policy, non-terminal error de-promotion, and a new Control UI config/default decision.

## What This Changes

The issue asks Control UI chat to collapse tool activity by default, suppress partial assistant fragments around tool activity, auto-collapse non-terminal errors after successful retry, and optionally expose a gateway controlUi activity default.

## Best Possible Solution

Define the normal/debug Control UI chat transcript contract first, then implement the chosen behavior at the chat projection/render boundary while reusing the Activity tab and browser-local tool-call preference unless maintainers explicitly want a gateway config default.

## Reproduction Assessment

Yes, at source level for the remaining behavior: current main still defaults tool calls visible, interleaves stream segments with tool messages, promotes failed tool cards, and lacks the requested config key. I did not run a live Control UI browser reproduction.

## Solution Assessment

Unclear. Collapsing noisy activity is plausible, but the proposed gateway config option is not clearly the best layer until maintainers choose between a default change, browser-local preference, or server-side config.
