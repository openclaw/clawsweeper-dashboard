## Summary

Keep open: current main still maps fully permissive OpenClaw exec policy to Claude CLI bypass mode without a root-aware branch, and the remaining behavior needs maintainer security/product judgment rather than conservative cleanup.

## What This Changes

The issue requests root-aware Claude CLI permission handling, clearer docs/diagnostics, watchdog guidance, and possibly higher-level backend presets for root gateway operators.

## Best Possible Solution

Add a security-reviewed root policy for the Anthropic CLI backend, document supported root workarounds and watchdog knobs, preserve bundled argv defaults, and improve permission-stall diagnostics without changing unrelated CLI backends.

## Reproduction Assessment

No, not as a live high-confidence reproduction in this review. Source inspection gives a credible current-main path for root-unaware bypass inference and generic permission-stall timeouts, but a real root Claude CLI run was not rerun here.

## Solution Assessment

Unclear. acceptEdits is a plausible workaround, but choosing fail-fast, warn, automatic downgrade, or a new permissionMode/rootBehavior setting changes permission semantics and should be decided by maintainers first.
