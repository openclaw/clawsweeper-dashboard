## Summary

Keep open: current main still maps OpenClaw YOLO exec policy to Claude CLI `bypassPermissions` without a root-aware branch, and the remaining fix needs maintainer security/product judgment around fail-fast, `acceptEdits`, native approval bridging, docs, and possible config surface.

## What This Changes

Requests root-aware Claude CLI permission-mode handling, docs and diagnostics for `acceptEdits`/watchdog stalls, and easier backend presets for root gateway operators.

## Best Possible Solution

Add a security-reviewed root policy in the Anthropic CLI backend, document the supported workaround and watchdog knobs, preserve bundled argv defaults when overriding, and improve permission-stall diagnostics without changing unrelated CLI backends.

## Reproduction Assessment

No, not as a live high-confidence reproduction in this review. Source inspection gives a credible current-main path: YOLO exec policy still normalizes Claude args to `bypassPermissions` with no root guard, and Claude live permission prompts can still fall through to watchdog behavior, but a real root Claude CLI run was not rerun here.

## Solution Assessment

Unclear. `acceptEdits` is a plausible workaround, but choosing fail-fast, warn, automatic downgrade, or a new `permissionMode`/`rootBehavior` setting changes permission semantics and should be decided by maintainers first.
