## Summary

Keep open: current main and the latest release still derive Claude CLI `bypassPermissions` from YOLO exec policy without a root-aware branch, and the requested fail-fast/warn/`acceptEdits`/preset behavior needs maintainer security and product judgment.

## What This Changes

The issue requests root-aware Claude CLI permission handling, clearer docs and diagnostics, watchdog guidance, and a higher-level preset so root users do not have to replace raw backend args.

## Best Possible Solution

Define a maintainer-approved root policy for the Anthropic CLI backend, document the supported workaround and watchdog knobs, and improve permission-stall diagnostics while preserving bundled Claude transport defaults.

## Reproduction Assessment

No, not as a live high-confidence reproduction in this review. Source inspection gives a credible current-main path: YOLO exec policy still normalizes Claude args to `bypassPermissions` with no root-aware guard, but the root Claude CLI failure was not rerun here.

## Solution Assessment

Unclear. `acceptEdits` is a plausible root workaround, but fail-fast, warn, automatic downgrade, or a new preset/config knob each changes permission semantics and should be chosen by maintainers.
