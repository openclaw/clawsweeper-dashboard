## Summary

Keep open: current main still maps default OpenClaw YOLO exec policy to Claude CLI `bypassPermissions` with no root-aware branch, and the remaining root behavior, diagnostics, and preset/config asks require explicit maintainer security/product judgment.

## What This Changes

The issue requests root-aware Claude CLI permission handling, clearer docs/diagnostics for blocked turns, watchdog workaround guidance, and easier Claude backend permission presets.

## Best Possible Solution

Define a security-reviewed root policy for the Anthropic CLI backend, document the supported root workaround and watchdog knobs, preserve bundled argv defaults when users override Claude mode, and route permission-prompt stalls through clear diagnostics or a tested control-response bridge.

No, not as a live high-confidence reproduction in this review. Source inspection does show current main still infers Claude `bypassPermissions` from YOLO policy with no root-aware guard, and the issue provides a concrete root-host report plus workaround evidence.

Unclear. `acceptEdits` is a plausible workaround, but choosing automatic downgrade, fail-fast, warning-only, or a new preset/config surface changes permission semantics and should be decided by maintainers before implementation.
