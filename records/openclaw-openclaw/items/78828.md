Keep open: current main still maps OpenClaw YOLO exec policy to Claude CLI `bypassPermissions` without a root-aware branch, and the requested fallback, diagnostics, and preset behavior need maintainer security/product judgment before a fix is safe.

## What This Changes

Request root-aware Claude CLI permission-mode handling, docs, blocked-turn diagnostics, and easier presets/watchdog guidance for OpenClaw running as root.

## Best Possible Solution

Add a security-reviewed root policy for the Anthropic CLI backend, document the supported workaround and watchdog knobs, preserve bundled argv defaults when overriding, and improve no-output permission-stall diagnostics without changing unrelated CLI backends.
