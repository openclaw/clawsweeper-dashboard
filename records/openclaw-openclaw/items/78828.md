## Summary

Keep open: current main still has the reported OpenClaw-side Claude permission mapping and docs/diagnostic gaps, while the root `acceptEdits` behavior and possible preset remain a security/product decision not fully covered by related open items.

## What This Changes

The issue asks OpenClaw to handle Claude CLI permission modes more safely under root, document the `acceptEdits` workaround and watchdog knobs, improve blocked-turn diagnostics, and add a higher-level root/permission preset.

## Best Possible Solution

Have maintainers choose the supported root Claude CLI policy, then implement that boundary in the Anthropic plugin and docs while letting the separate control-request issue/PR handle live-session permission stalls.

## Reproduction Assessment

No, not as a live end-to-end root Claude reproduction in this read-only review. Yes at source level for OpenClaw's side: current main still infers `bypassPermissions` from YOLO exec policy and has no root-aware branch.

## Solution Assessment

Unclear. `acceptEdits` is a plausible workaround, but fail-fast, warning, automatic downgrade, or a new preset/config knob each changes permission semantics and should be chosen by maintainers before implementation.
