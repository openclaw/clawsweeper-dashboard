## Summary

Keep open: current main and v2026.6.6 still map full/no-ask Claude CLI execution to `bypassPermissions` with no root-aware branch. The `control_request` dead-air subset is mostly handled by the merged live-session fix, but the root gateway permission-mode policy and documentation remain unresolved.

## What This Changes

The issue asks OpenClaw to make Claude CLI permission-mode handling root-aware, document root workarounds, and surface blocked permission turns more clearly.

## Best Possible Solution

Make Claude CLI root behavior explicit in the Anthropic CLI boundary: either fail fast with a clear diagnostic or route through a maintainer-approved permission-control path, then document the root/container workaround and keep exec-policy semantics intentional.
