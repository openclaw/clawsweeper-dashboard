## Summary

Current main still has the reported OpenClaw-side behavior: YOLO exec policy maps to Claude CLI `bypassPermissions` with no root-aware branch, and timeout diagnostics remain generic. The issue should stay open because the right root behavior is a security/product decision, not a stale cleanup close.

## What This Changes

The issue requests root-aware Claude CLI permission handling, clearer root/systemd docs and diagnostics, watchdog guidance, and easier permission-mode presets.

## Best Possible Solution

Add a security-reviewed root policy for the Anthropic CLI backend, document the supported root workaround and watchdog knobs, preserve bundled argv defaults when overriding, and improve permission-stall diagnostics without changing unrelated CLI backends.

## Reproduction Assessment

No, not as a live high-confidence reproduction in this review. Source inspection gives a credible current-main path for OpenClaw adding `bypassPermissions` under YOLO with no root guard, but the root Claude CLI rejection and blocked-turn stall were not rerun here.

## Solution Assessment

Unclear. `acceptEdits` is a plausible workaround, but choosing fail-fast, warn, automatic downgrade, or a new preset/config surface changes permission semantics and should be decided by maintainers first.
