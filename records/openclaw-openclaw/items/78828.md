Keep open: current main and the latest release still have root-unaware Claude CLI permission-mode inference, and the blocked-turn timeout path is only partly covered by a separate open control-request PR. The remaining work needs maintainer security/product judgment, not conservative cleanup.

## What This Changes

## Best Possible Solution

Define a maintainer-approved Anthropic CLI root policy, document supported root/systemd workarounds and watchdog knobs, preserve bundled argv defaults, and handle permission stalls through the Claude live-session bridge or a clear fail-fast diagnostic.
