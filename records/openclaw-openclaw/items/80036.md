Keep open. Current `main` still has the readiness gap described in the report: Chrome MCP status is based on transport attach, while the documented success path and the failing operations require page-level Chrome MCP tools; the recent status-probe fix in https://github.com/openclaw/openclaw/pull/77473 does not cover the signed-in user-profile page-tool timeout.

The issue reports that Chrome MCP existing-session `profile=user` on macOS reports transport readiness while page-level tools such as tabs, snapshot, screenshot, and evaluate time out.
