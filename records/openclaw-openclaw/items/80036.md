## Summary

Keep open. Current main still has the reported Chrome MCP readiness split: `profile=user` status can be marked running from a transport-only attach probe, while the documented success path and affected operations require page-level Chrome MCP tools that may still time out in this macOS Chrome 148 scenario. The related merged status timeout fix covers slow status attach, not page-tool hangs after ready status.

## What This Changes

The issue requests investigation and a fix for Chrome MCP `profile=user` reporting ready while `list_pages`, snapshots, screenshots, and script evaluation time out on macOS Chrome 148.

## Best Possible Solution

Live-reproduce on macOS Chrome 148 against current main, then fix the browser plugin page discovery/tool path or explicitly split transport readiness from page readiness with actionable diagnostics and focused regression coverage.

## Solution Assessment
