## Summary

Keep open. Current main still supports Chrome MCP existing-session profiles, and source/tests show `status` can report `running/cdpReady=true` from the MCP transport handshake while page reachability can still fail; the reported macOS page-tool timeouts are plausible and not covered by the older closed attach-failure issue.

## What This Changes

Investigate and fix `profile=user` Chrome MCP existing-session readiness and page-tool timeouts on macOS so status and browser operations produce accurate, actionable results.

## Best Possible Solution

The browser plugin should distinguish Chrome MCP transport attach from page/tool readiness, provide actionable timeout causes, and restore reliable `tabs`, snapshot, screenshot, and evaluate behavior for supported `user` profiles.

## Reproduction Assessment

No high-confidence end-to-end reproduction was established in this review. Source and tests do show the readiness gap, and the reporter supplied concrete macOS commands/logs for the page-tool timeouts.

## Solution Assessment

Not applicable because there is no proposed patch. The narrowest maintainable direction is to live-reproduce on macOS Chrome 148, then fix the Chrome MCP page discovery/tool path or split status into transport readiness versus page readiness with clear errors.
