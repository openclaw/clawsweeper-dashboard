## Summary

Keep open. Current main still documents existing-session `user` profiles as supporting tabs and snapshots, while source and tests show `status` can report `running`/`cdpReady` from a Chrome MCP transport attach without proving page-tool reachability; the reporter's macOS timeout logs match the current Chrome MCP tool timeout path and are not covered by the older closed loopback attach issues or the merged status-timeout change.

## What This Changes

Investigate and fix Chrome MCP existing-session `profile=user` on macOS where status reports ready but page-level tools time out.

## Best Possible Solution

Live-reproduce on macOS Chrome 148, then fix the Chrome MCP page discovery/tool path or explicitly split transport readiness from page readiness with actionable diagnostics while preserving existing-session safety.
