## Summary

Keep open. Current main still ships the bundled `summarize` skill with the deleted Homebrew formula and still reports generic enable success while requirements keep the skill ineligible; https://github.com/openclaw/openclaw/issues/74380 and https://github.com/openclaw/openclaw/pull/74408 cover the installer half but not the Control UI enable-feedback gap.

## What This Changes

The issue requests a portable or clearly unsupported `summarize` skill installer and user-visible feedback when enabling the skill fails due to a missing binary.

## Best Possible Solution

Land or replace the installer-source fix, then add requirement-aware Control UI enable feedback that names the missing binary and available install action.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with no `summarize` on `PATH`, `skills.update` can return success while `requirementsSatisfied` remains false and the UI reports generic enable success.

## Solution Assessment

Unclear. Requirement-aware enable feedback is a narrow maintainable fix, but the supported installer source for bundled `summarize` needs maintainer package-source direction and the active installer PR still needs revision.
