## Summary

Keep open. Current main and `v2026.5.20` still ship the bundled `summarize` skill with the removed Homebrew formula, and the Control UI enable path still reports success instead of validating missing required binaries; the related open PR at https://github.com/openclaw/openclaw/pull/74408 covers the package-source side for https://github.com/openclaw/openclaw/issues/74380 but not this issue's feedback/platform decision.

## What This Changes

The issue asks OpenClaw to make the bundled `summarize` skill installable or clearly platform-scoped, and to show missing-binary feedback when users enable it in Control UI.

## Best Possible Solution

Confirm the supported `summarize` install/platform policy, land or replace the related installer-source PR, and add requirement-aware Control UI enable feedback that names missing binaries and shows supported install or platform guidance.

## Reproduction Assessment

Yes, at source level. Current main still points the bundled `summarize` skill at a removed Homebrew formula, and the enable path can report success without validating that the required `summarize` binary is present.

## Solution Assessment

Unclear. Replacing the deleted formula is necessary, but maintainers still need to confirm the supported installer source/platform policy and pair it with a focused missing-requirement UI message.
