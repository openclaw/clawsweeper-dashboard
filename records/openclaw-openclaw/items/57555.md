## What This Changes

The issue asks OpenClaw to make the bundled `summarize` skill installable or clearly unsupported on Linux/Docker and to show actionable feedback when enabling fails because the required binary is missing.

## Best Possible Solution

Land or replace the related installer-source PR after confirming the supported package source, then add requirement-aware Control UI enable feedback that names the missing `summarize` binary and shows the supported install or platform guidance.

## Reproduction Assessment

Yes, at source level. Current main still points the bundled `summarize` skill at a Homebrew formula removed upstream, and the enable flow can report success while refreshed status keeps the skill unavailable when the required binary is missing.

## Solution Assessment

Unclear. Replacing the deleted formula is necessary, but the exact supported installer source or platform gating needs maintainer confirmation, and the UI fix should be a focused missing-requirement message rather than a broad install-policy change.
