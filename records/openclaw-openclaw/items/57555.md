## Summary

Keep open: current main and v2026.6.6 still ship the bundled `summarize` skill with only the deleted `steipete/tap/summarize` Homebrew formula, and the current UI/status path only partially improves feedback instead of resolving the broken installer source.

## What This Changes

The issue asks OpenClaw to make the bundled `summarize` skill installable or clearly unsupported on Linux/Docker, and to show clear feedback when enable/install cannot satisfy the required binary.

## Best Possible Solution

Choose one honest bundled-skill path: use a verified maintained installer for `summarize`, or stop presenting it as installable where its required binary cannot be provided, with clear UI/install output for that decision.

## Reproduction Assessment

Yes, at source level. Current main and v2026.6.6 still declare only `steipete/tap/summarize`, the installer runs `brew install` for that formula, and the upstream tap no longer contains the formula.

## Solution Assessment

Unclear. The issue identifies a real broken path, but the best fix depends on whether maintainers want to keep the bundled skill with a maintained installer, mark it unsupported, or remove/de-bundle it.
