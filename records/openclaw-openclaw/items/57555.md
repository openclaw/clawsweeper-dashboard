## Summary

Keep open. Current main and the latest release still ship the bundled `summarize` skill with the removed Homebrew formula, and the Control UI enable path can still report generic success while refreshed skill status keeps the skill ineligible; https://github.com/openclaw/openclaw/pull/74408 covers only the installer-source half and is still open.

## What This Changes

The issue asks OpenClaw to make the bundled `summarize` skill usable or clearly platform-gated outside Homebrew/macOS, and to show clear Control UI feedback when enabling fails because the required binary is missing.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/74408 for the deleted installer source after confirming the supported package path, then add requirement-aware Control UI enable feedback that names the missing `summarize` binary and available install or platform guidance.

## Reproduction Assessment

Yes at source level. Without `summarize` on `PATH`, `skills.update` can return success while refreshed skill status keeps the skill ineligible, and the bundled installer still points to the removed Homebrew formula.

## Solution Assessment

Unclear. Replacing the deleted formula is necessary, but the exact installer source or platform gating needs maintainer confirmation, and the UI feedback fix should be a narrow requirement-aware message rather than a broad install policy change.
