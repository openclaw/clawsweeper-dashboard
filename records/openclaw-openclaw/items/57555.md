## Summary

Keep open. Current `main` still ships the bundled `summarize` skill with a hard `summarize` binary requirement and only the deleted Homebrew formula, and the Control UI enable path still reports generic success instead of surfacing missing requirements; https://github.com/openclaw/openclaw/issues/74380 and https://github.com/openclaw/openclaw/pull/74408 cover only the installer portion.

## What This Changes

The issue asks OpenClaw to provide a supported `summarize` install path or platform indication and show clear Control UI feedback when enabling fails because the binary is missing.

## Best Possible Solution

Land or replace the installer-source fix for `summarize`, then add requirement-aware Control UI feedback so enabling an ineligible skill names the missing binary and available install action instead of reporting generic success.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with no `summarize` binary on `PATH`, `skills.update` can write `enabled: true` and return success while status remains ineligible because `missing.bins` contains `summarize`.

## Solution Assessment

Unclear. Requirement-aware enable feedback is a narrow maintainable fix, but the supported installer/platform answer for the bundled `summarize` skill needs maintainer package-source and product direction.
