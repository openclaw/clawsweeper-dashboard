## Summary

Keep open. Current main still bundles `summarize` with a hard `summarize` binary requirement and only the deleted Homebrew tap as its install option, while Control UI enablement still writes config and reports generic success instead of validating missing requirements. Open PRs now cover parts of the installer/onboarding problem, but they are not merged and do not fully cover the requested Control UI enable-feedback behavior.

## What This Changes

The issue asks for the bundled `summarize` skill to be installable or clearly platform-scoped outside Homebrew, and for Control UI to show a clear error when enabling fails because required binaries are missing.

## Best Possible Solution

Keep the issue open until current main ships a maintained installer or accurate platform/removal decision for the bundled `summarize` skill, and until enabling an ineligible skill in Control UI produces clear requirement-specific feedback instead of a silent blocked state.

## Reproduction Assessment

Yes. Against current main, the source path is enough to reproduce with high confidence: `summarize` still has a missing-binary gate plus brew-only installer metadata, and enabling through Control UI still calls `skills.update` without requirement validation.

## Solution Assessment

Unclear. Open PR #74408 looks like a plausible installer fix for `summarize`, but it does not solve the Control UI enable-feedback request; the best path is a coordinated maintainer decision plus a focused server/UI feedback change.
