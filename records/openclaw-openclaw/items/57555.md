## Summary

Keep open. Current main still ships the bundled `summarize` skill with a hard `summarize` binary requirement and the deleted Homebrew formula, and the Control UI enable path still reports generic success instead of surfacing missing requirements; the related open issue and PR cover the installer portion but not the enable-feedback gap.

## What This Changes

The issue requests a maintained non-brew or platform-aware install path for the bundled `summarize` skill and visible Control UI feedback when enablement is blocked by missing requirements.

## Best Possible Solution

Land or replace the installer-source fix for `summarize`, then add requirement-aware Control UI feedback so enabling an ineligible skill names the missing binary and available install action instead of reporting generic success.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with no `summarize` binary on `PATH`, enabling the skill can write config and return success while status remains blocked because requirements are unsatisfied.

## Solution Assessment

Unclear. Requirement-aware enable feedback is a narrow maintainable fix, but the supported installer/platform answer for bundled `summarize` needs maintainer package-source and product direction.
