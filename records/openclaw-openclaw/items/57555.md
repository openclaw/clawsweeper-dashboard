## Summary

Keep open: current main and the latest release still ship the bundled `summarize` skill with only the removed Homebrew formula. Current UI code now has partial missing-requirement/message feedback, but the broken installer source and support/removal decision remain unresolved.

## What This Changes

The issue asks OpenClaw to stop silently failing when enabling the bundled `summarize` skill by providing a valid install path or clearly marking the skill unsupported where its binary cannot be installed.

## Best Possible Solution

Make the bundled skill truthful: point it at a verified maintained installer, mark or hide it where unsupported, or move/remove it from bundled skills with matching UI/status coverage.

Yes, source-level reproduction is high-confidence: current main and v2026.6.6 still declare only `steipete/tap/summarize`, and the upstream tap no longer has that formula. I did not run a live Docker UI toggle in this read-only review.
