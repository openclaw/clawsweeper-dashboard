## Summary

Keep open. Current main and v2026.5.18 still ship the bundled `summarize` skill with the deleted Homebrew formula, and the Control UI enable path still reports generic success while missing requirements keep the skill ineligible; the open installer PR covers only part of this issue.

## What This Changes

The issue requests a valid or clearly unsupported install path for the bundled `summarize` skill and visible Control UI feedback when enabling fails because the required binary is missing.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/74408 for the installer-source failure, then add requirement-aware Control UI enable feedback that names the missing binary and available install action.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: without `summarize` on `PATH`, `skills.update` can return success while `buildWorkspaceSkillStatus` keeps the skill ineligible and the UI reports generic enable success.

## Solution Assessment

Unclear. Requirement-aware enable feedback is a narrow maintainable fix, but the supported installer source for bundled `summarize` should be settled through the active installer PR or an equivalent maintainer-approved replacement.
