## Summary

Keep open. Current `main` still ships the bundled `summarize` skill with a hard `summarize` binary requirement and only a `steipete/tap/summarize` Homebrew installer, while the Control UI enable path still reports a generic success after writing config instead of requirement-specific missing-binary feedback.

## What This Changes

Request to make bundled `summarize` usable or clearly platform-scoped outside Homebrew and show feedback when enabling fails because the required binary is missing.

## Best Possible Solution

Choose the supported ownership path for `summarize`, then make skill enablement/reporting requirement-aware so users see missing binaries and valid install options where they toggle the skill.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with no `summarize` binary on `PATH`, enabling the skill through `skills.update` writes config and succeeds, while status remains ineligible because `missing.bins` still contains `summarize`.

## Solution Assessment

Unclear. Generic enable feedback is a narrow maintainable fix, but the installer/platform part needs maintainer direction because the only bundled installer is reported deleted upstream.
