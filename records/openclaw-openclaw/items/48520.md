## Summary

Keep open. Current main still does not provide the SEA packaging surface, and the PR remains useful but not mergeable because the generated trampoline mishandles the existing compile-cache re-entry path and misses required docs/changelog integration.

## What This Changes

The PR adds Node SEA sidecar packaging commands, build and Node-fetch scripts, installation docs, checksum hardening, and focused respawn/runtime-dependency tests.

## Best Possible Solution

Land a revised SEA sidecar package that handles compile-cache re-entry correctly, remains discoverable in docs and changelog, and preserves the Node archive checksum verification.

## Reproduction Assessment

Yes. With Node compile cache active, current `openclaw.mjs` respawns the SEA binary with `openclaw.mjs` as argv[1], and the generated trampoline only strips binary basenames before forwarding user arguments.

## Solution Assessment

No. The sidecar design and checksum hardening are plausible, but the current patch needs the wrapper re-entry fix plus docs navigation and changelog work before it is the narrow maintainable solution.
