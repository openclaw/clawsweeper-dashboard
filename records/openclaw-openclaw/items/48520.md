Keep open: current main still lacks the SEA build surface, and the branch remains useful, but it is not merge-ready because the SEA wrapper can forward a stray `openclaw.mjs` argument during compile-cache respawn and no real build/run proof is posted.

## What This Changes

The PR adds Node SEA sidecar packaging scripts, `build:sea` package commands, install documentation, checksum-verified Node archive fetching, and focused launcher/runtime-dependency tests.

## Best Possible Solution

Land a revised SEA sidecar build after fixing SEA re-entry argument handling, adding the page to docs navigation, and proving the packaged binary in a real container or matching host build.

## Reproduction Assessment

Yes for the blocking PR bug from source inspection: current `openclaw.mjs` can respawn `process.execPath` with `openclaw.mjs` as the first script argument, and the PR trampoline only strips `openclaw` or `openclaw.exe`. I did not run the SEA build in this read-only review.

## Solution Assessment

No. The SEA sidecar design is plausible and the checksum hardening is sound, but the current branch needs the wrapper fix, docs navigation follow-through, and real runtime proof before it is the maintainable merge path.
