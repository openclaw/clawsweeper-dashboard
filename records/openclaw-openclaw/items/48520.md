Full review comments:

- **[P1] Align the SEA Node ABI with sidecar dependencies:** `scripts/fetch-node-for-sea.mjs:10`
  - body: `pnpm build:sea` always packages a Node 25.9.0 executable but stages the checkout's existing `node_modules`, which may have been installed under the repo-supported Node 22/24 runtime. Node refuses C++ addons compiled for a different module ABI, and OpenClaw ships native dependencies such as `@lydell/node-pty`, so the generated package can fail at runtime unless the builder requires matching Node-sidecar deps or reinstalls/proves them for the packaged Node.
  - confidence: 0.86
- **[P1] Preserve the caller working directory:** `scripts/build-sea.mjs:228`
  - body: The generated SEA bootstrap calls `process.chdir(packageRoot)` before loading `openclaw.mjs`, which changes normal CLI cwd semantics for every command run from another workspace. Keep package-root resolution explicit for sidecar files while leaving `process.cwd()` as the caller's directory so relative paths, workspace fallback, and cwd-loaded configuration behave like the npm CLI.
  - confidence: 0.93
- **[P1] Preserve interactive TLS respawn:** `src/entry.respawn.ts:70`
  - body: This top-level guard uses `shouldSkipRespawnForArgv`, but current main split startup-environment respawn from warning suppression so `tui`, `terminal`, and `chat` still receive automatic `NODE_EXTRA_CA_CERTS` repair. With the PR branch, those interactive commands return `null` before the CA repair path, regressing nvm/managed-CA setups that current tests cover.
  - confidence: 0.95
