## Summary

Keep open: the Slack `file_shared` fallback remains useful and current main still lacks that event handler/subscription, but this branch targets the removed legacy `src/slack` path and needs real Slack proof plus an active-plugin implementation before merge.

## What This Changes

The PR adds a legacy Slack `file_shared` handler, dedupe, attachment-note handling, and Slack attachment types under `src/slack`.

## Best Possible Solution

Implement the fallback in `extensions/slack`, update the Slack manifest/docs/tests for `file_shared`, use a timestamp-bounded or paginated lookup, preserve existing media and dedupe safeguards, and require redacted real Slack proof.

## Reproduction Assessment

Yes, from source inspection: current main has no active `file_shared` handler or setup subscription. A live repro would require Slack Socket Mode with a large or multi-file upload that emits `file_shared` without a normal message event, which was not run here.

## Solution Assessment

No. The requested fallback is useful, but this patch is not the best implementation because it targets the removed legacy path and leaves setup/docs plus the history lookup incomplete.
