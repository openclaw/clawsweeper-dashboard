## Summary

Keep this PR open, but do not merge it as-is: the missing Slack `file_shared` fallback is still real on current main, while this branch targets the removed legacy `src/slack` runtime and lacks real Slack behavior proof.

## What This Changes

The PR adds a Slack `file_shared` large-upload fallback, file/message dedupe, attachment-note typing, and message preparation changes under legacy `src/slack` paths.

## Best Possible Solution

Port a focused `file_shared` fallback into `extensions/slack`, update Slack setup/docs/tests for the event subscription, use timestamp-bounded history hydration with dedupe, and require redacted live Slack proof.

## Reproduction Assessment

Yes for the source-level path: current main registers no active `file_shared` Slack handler and the setup manifest omits that event, so a delivered `file_shared` event cannot reach the inbound pipeline. I did not run a live Slack workspace reproduction.

## Solution Assessment

No. The PR points at the right missing capability, but the maintainable fix needs to live in `extensions/slack`, update setup surfaces, and anchor the history lookup instead of merging this legacy-path diff.
