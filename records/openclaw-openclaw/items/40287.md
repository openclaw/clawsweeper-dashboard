## Summary

Keep open: the underlying Slack large-upload gap still appears present on current main and the related issue was closed in favor of this PR, but the branch is not mergeable as-is because it targets removed legacy paths, misses setup/docs subscription updates, uses an unanchored history lookup, and lacks live proof.

## What This Changes

The PR adds a Slack `file_shared` fallback under the legacy `src/slack` tree, plus file/message dedupe and Slack attachment note/type handling.

## Best Possible Solution

Port a focused `file_shared` fallback into `extensions/slack`, update Slack setup/docs/tests for the event subscription, use timestamp-bounded hydration with dedupe, and require redacted live Slack proof.

## Reproduction Assessment

Yes for the source-level path: current main registers no active `file_shared` handler and does not subscribe the event in Slack setup surfaces, so a delivered `file_shared` event cannot reach the inbound pipeline. I did not run a live Slack workspace reproduction.

## Solution Assessment

No. The PR points at the right missing capability, but the maintainable solution needs to live in `extensions/slack`, update setup/docs/tests, and anchor the Slack history lookup instead of merging this stale legacy-path diff.
