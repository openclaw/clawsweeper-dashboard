## Summary

Keep open: current main still lacks the Slack `file_shared` fallback and setup subscription, and the closed companion issue points to this PR as the canonical remaining item. The branch is not mergeable as-is because it targets removed legacy `src/slack` code, lacks real Slack proof, and still needs active-plugin setup/docs/history-lookup work.

## What This Changes

The PR adds a Slack `file_shared` fallback for large uploads, Slack attachment-note handling, and local Slack attachment typing under `src/slack`.

## Best Possible Solution

Port a focused `file_shared` fallback into `extensions/slack`, add `file_shared` to setup docs/tests, hydrate the matching Slack message with timestamp-bounded or paginated lookup, dedupe against normal delivery, and require redacted live Slack proof.

## Reproduction Assessment

Yes for the source-level failure path: a delivered Slack `file_shared` event cannot reach the active plugin because current main registers no handler and does not subscribe that event. A live Slack large-upload run was not performed, so real behavior proof is still needed before merge.

## Solution Assessment

No. The fallback is useful, but this branch is not the best implementation because it edits removed legacy code and omits setup/docs plus robust history anchoring; the maintainable path is an active Slack plugin implementation or replacement PR.
