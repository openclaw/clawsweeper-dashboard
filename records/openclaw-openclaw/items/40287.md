## Summary

Keep open: the Slack `file_shared` gap still exists on current main, but this branch is not mergeable as-is because it edits removed legacy `src/slack` paths, misses setup subscription changes, uses a sliding history lookup, and lacks real Slack proof.

## What This Changes

The PR adds a legacy Slack `file_shared` event fallback, file/message dedupe, attachment-note typing/helper code, and message preparation changes for large file uploads.

## Best Possible Solution

Port a focused fallback into `extensions/slack`, add `file_shared` to setup/docs/tests, hydrate with timestamp-bounded history and dedupe, and prove it with redacted Slack runtime output.

## Reproduction Assessment

Yes for the source-level path: current main registers no active `file_shared` Slack handler and the generated manifest omits that event, so a delivered `file_shared` event cannot reach the inbound pipeline. I did not run a live Slack workspace reproduction.

## Solution Assessment

No. The PR points at the right Slack event, but the maintainable fix should live in `extensions/slack`, update setup surfaces, and anchor history hydration around the event timestamp instead of merging this legacy-path diff.
