## Summary

Keep open. The PR targets a real missing Slack `file_shared` fallback, but the patch is not merge-ready because it edits the legacy `src/slack` path instead of the active Slack plugin, misses setup/docs event subscriptions, uses an unanchored history lookup, and lacks real after-fix Slack proof.

## What This Changes

The PR adds a legacy Slack `file_shared` fallback handler, file-message deduplication, attachment-note handling, and Slack attachment typings under `src/slack`.

## Best Possible Solution

Port the fallback to `extensions/slack`, add the `file_shared` subscription to setup/docs, hydrate with a timestamp-bounded lookup, preserve existing media safety limits and dedupe behavior, and prove it with a real Slack run.

## Reproduction Assessment

Yes. Source inspection is high-confidence: current main registers only `message` and `app_mention` for inbound Slack messages and setup/docs omit `file_shared`; a live repro would share a large or multi-file upload where Slack emits only `file_shared` and verify no inbound dispatch occurs.

## Solution Assessment

No. The PR identifies the right gap, but the current patch is not the best implementation because it edits the removed legacy Slack path, omits subscription setup/docs, and reads a sliding recent-history window instead of anchoring around the file event.

## Review Findings

- **[P2] Port the handler into the active Slack plugin:** `src/slack/monitor/events.ts:5`
  - body: Current main no longer runs the Slack runtime from `src/slack`; the active bundled plugin lives under `extensions/slack/src`. Registering the new fallback here will not wire the handler into the runtime users actually run, so the implementation needs to move to the active Slack plugin.
  - confidence: 0.97

  - confidence: 0.9

  - body: The fallback sleeps and then reads a recent-message window using only `limit`, so a busy channel can still push the upload out of the lookup before it runs. Bound the history call around `fileEvent.event_ts` with `oldest`/`latest` or an equivalent targeted lookup.
  - confidence: 0.86
