## Summary

Keep open: current main still lacks the Slack `file_shared` fallback this PR is trying to add, and the duplicate issue was already closed in favor of this PR. The branch should not merge as-is because it edits the legacy `src/slack` path, does not update Slack setup subscriptions, uses an unanchored history lookup, and still lacks real after-fix Slack proof.

## What This Changes

The PR adds a legacy `src/slack` `file_shared` event handler with file/message dedupe, Slack history hydration, and attachment-note typing for Slack uploads or forwarded content.

## Best Possible Solution

Port a focused `file_shared` fallback into `extensions/slack`, update setup/docs/tests for the event subscription, use timestamp-bounded hydration with dedupe, and require redacted live Slack proof.

## Reproduction Assessment

Yes for the source-level path: current main registers no active `file_shared` handler and the Slack setup manifest omits that event, so a delivered `file_shared` event cannot reach the inbound pipeline. I did not run a live Slack workspace reproduction.

## Solution Assessment

No. The PR points at the right missing capability, but the maintainable solution needs to live in `extensions/slack`, update Slack setup/docs/tests, and anchor the history lookup instead of merging this stale legacy-path diff.
