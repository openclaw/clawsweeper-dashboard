## Summary

Keep open. The Slack large-file fallback gap still appears real on current main, but this PR is not merge-ready because it edits the removed legacy `src/slack` tree while the active Slack plugin lives under `extensions/slack`, and it still misses setup/docs subscription work plus a timestamp-anchored history lookup.

## What This Changes

The PR adds a legacy Slack `file_shared` fallback handler under `src/slack`, marks normal file messages for deduplication, appends forwarded attachment notes to prepared Slack messages, and extends Slack attachment typing.

## Best Possible Solution

Port the useful `file_shared` fallback into the active `extensions/slack` plugin: register and test a file-shared handler, update generated setup manifest and Slack docs, hydrate the matching Slack message with a bounded timestamp-anchored history lookup, dedupe Slack retries plus normal `message`/`file_share` delivery, preserve existing Slack media security and size limits, and add focused Slack plugin tests.

## Reproduction Assessment

Yes. A high-confidence code-level reproduction exists: current main registers only `message` and `app_mention` as inbound message events and the Slack manifest omits `file_shared`; a live check would share a large or multi-file upload in Slack Socket Mode and observe no OpenClaw inbound dispatch when Slack sends only `file_shared`.

## Solution Assessment

No. The PR addresses the right Slack fallback problem, but the submitted patch is not the best solution because it targets the removed legacy Slack tree and omits required setup/docs plus a history lookup anchored to the file event timestamp.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Port the handler into the active Slack plugin:** `src/slack/monitor/events.ts:3`
  - body: Current main no longer has the `src/slack` runtime this patch edits; the active Slack channel is under `extensions/slack/src`. As written, the fallback handler is not wired into the runtime that ships on main and needs to be ported instead of recreating legacy Slack files.
  - confidence: 0.97
- **[P2] Subscribe Slack apps to file_shared events:** `src/slack/monitor/events.ts:21-24`
  - body: Registering a handler is not enough for installed Slack apps to receive these events. The generated Slack manifest and public Slack docs still omit `file_shared`, so users following setup will not get the fallback event that this handler depends on.
  - confidence: 0.9
- **[P2] Anchor the history lookup to the file event timestamp:** `src/slack/monitor/events/file-shared.ts:84-87`
  - body: After the fixed delay this still fetches only recent channel history without `oldest` or `latest` tied to the `file_shared` event. Busy channels can push the upload outside that window, making the fallback silently drop the large-file message it is meant to recover.
  - confidence: 0.85
