## Summary

Keep open: the Slack `file_shared` gap still exists on current main, but this branch is not mergeable as-is because it edits the removed legacy Slack path, omits setup/docs subscription changes, uses an unanchored history lookup, and lacks real behavior proof.

## What This Changes

The PR adds a Slack `file_shared` fallback handler, file-message deduplication, attachment-note handling, and Slack attachment typings under `src/slack`.

## Best Possible Solution

Port the fallback into `extensions/slack`, add `file_shared` to setup manifest/docs/tests, anchor history retrieval around the file event timestamp, preserve existing media safety limits and dedupe behavior, and require redacted real Slack proof.

## Reproduction Assessment

Yes. Source inspection is high-confidence: current main registers only `message` and `app_mention` for inbound Slack message handling and setup/docs omit `file_shared`; a live repro would share a large or multi-file upload where Slack emits only `file_shared`.

## Solution Assessment

No. The PR identifies the right gap, but the patch is not the best implementation because it targets the removed legacy Slack path, omits setup/docs subscription changes, and still relies on a sliding recent-history window.
