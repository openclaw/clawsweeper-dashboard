## Summary

Keep this PR open: the Slack `file_shared` fallback is still missing on current main, but this branch cannot merge as-is because it targets the removed legacy Slack path and still needs real Slack behavior proof.

## What This Changes

The branch adds a legacy `src/slack` `file_shared` fallback handler, file-message deduplication, attachment-note handling, and Slack attachment typings.

## Best Possible Solution

Port the fallback into `extensions/slack`, subscribe setup/docs manifests to `file_shared`, hydrate by timestamp-bounded history lookup, preserve existing media safety limits, and add focused Slack tests.

## Reproduction Assessment

Yes. Source inspection is high-confidence: current main registers only `message` and `app_mention` for inbound Slack messages and setup/docs omit `file_shared`; a live repro would share a large or multi-file upload and verify no dispatch occurs when Slack emits only `file_shared`.

## Solution Assessment

No. The PR points at a real gap, but the current patch is not the best fix because it edits the removed `src/slack` runtime, misses setup/docs subscriptions, and uses an unanchored recent-history lookup.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Port the handler into the active Slack plugin:** `src/slack/monitor/events.ts:5`
  - body: Current main no longer ships `src/slack`; the active Slack runtime is under `extensions/slack/src`. Registering the new fallback in this legacy path will not wire it into the runtime users run, so the handler needs to be ported to the active plugin.
  - confidence: 0.97
- **[P2] Subscribe Slack apps to file_shared:** `src/slack/monitor/events.ts:24-27`
  - body: The handler depends on Slack delivering `file_shared`, but the branch does not update generated setup manifests or docs to include that bot event. Slack apps created or repaired from current setup guidance will still never send the fallback event.
  - confidence: 0.9
- **[P2] Anchor the history lookup to the file event:** `src/slack/monitor/events/file-shared.ts:82-85`
  - body: The fallback sleeps and then reads a sliding recent-message window with only `limit`, so a busy channel can push the upload out of the result before lookup. Bound the history call around the `file_shared` event timestamp with `oldest`/`latest` or an equivalent targeted lookup.
  - confidence: 0.86
