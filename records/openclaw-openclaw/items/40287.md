The PR adds a legacy `src/slack` `file_shared` handler, message dedupe, Slack attachment typing, and an attachment-note helper for large or forwarded Slack file content.

## Best Possible Solution

Port a focused fallback into `extensions/slack`, update Slack setup/docs/tests for `file_shared`, use timestamp-bounded history plus dedupe, drop the duplicate attachment-note path, and require redacted Slack runtime proof before merge.

## Reproduction Assessment

## Solution Assessment

No. The PR points at the right Slack event, but the maintainable fix should live in `extensions/slack`, update setup surfaces, reuse current attachment handling, and anchor history hydration around the event timestamp.
