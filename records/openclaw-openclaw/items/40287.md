## Summary

Keep open: current main still lacks the Slack `file_shared` fallback, but this PR is not mergeable as-is because it targets the removed legacy Slack path, omits the setup/docs event subscription, and has no real behavior proof.

## What This Changes

The PR adds a legacy Slack `file_shared` event handler, dedupe state, attachment-note handling, and Slack attachment typings under `src/slack`.

## Best Possible Solution

Port the fallback into `extensions/slack`, add `file_shared` to Slack setup/docs/tests, hydrate and dedupe the matching message with a timestamp-bounded or paginated lookup, and preserve the active attachment/media handling.

## Reproduction Assessment

Yes, from source inspection: current main has no `file_shared` handler or setup subscription in the active Slack plugin. A live Slack Socket Mode large or multi-file upload was not run.

## Solution Assessment

No. The fallback is useful, but this PR is not the best solution because it targets removed `src/slack` code and leaves setup/docs plus the history lookup incomplete; the safer path is a focused active-plugin implementation.
