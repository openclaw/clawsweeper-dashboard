## Summary

Keep open: current main does not implement skill `references/` auto-loading, and the PR still has a blocking read-path defect because it appends references only during read-only sandbox skill sync while normal and `workspaceAccess: rw` runs continue to read the bare `SKILL.md`.

## What This Changes

The PR adds `metadata.openclaw.references` parsing, loads skill-local `references/*.md` into `referencesContent`, appends it to sandbox-copied `SKILL.md`, and documents/changelogs the new skill references feature with focused tests.

## Best Possible Solution

Materialize one supported merged skill-read contract so selected skills resolve to `SKILL.md` plus allowed reference markdown across normal runs, skill snapshots, read-only sandbox sync, and `workspaceAccess: rw`.

## Reproduction Assessment

Yes. A static reproduction is clear from the diff: create a skill with `references/guide.md`; the PR computes `referencesContent`, but normal and `workspaceAccess: rw` runs still advertise and read the original `entry.skill.filePath`, so the supplemental markdown is missed.

## Solution Assessment

No. The sync-only append is not the best fix because it makes the feature depend on one workspace-copy path; the safer repair is to make the advertised skill location or read contract itself resolve to merged content in every supported runtime mode.
