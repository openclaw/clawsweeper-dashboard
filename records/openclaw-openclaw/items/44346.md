## What This Changes

This PR adds `metadata.openclaw.references` parsing, skill-local markdown reference loading, docs, changelog, and tests for auto-loading `references/` files alongside `SKILL.md`.

## Best Possible Solution

Define one merged skill read/materialization contract so selected references are visible consistently in normal runs, snapshots, read-only sandbox sync, and `workspaceAccess: "rw"`, with docs and tests covering those paths.

## Reproduction Assessment

Yes. Source inspection of the PR head shows references are appended only during sandbox skill sync, while current prompt and rw workspace paths still read the original `SKILL.md` location.

## Solution Assessment

No. A sync-only append is not the narrow maintainable solution because it makes the feature depend on one workspace-copy path instead of the common skill read/materialization contract.
