## Summary

Keep this PR open. Current main still lacks references auto-load support, and the proposed branch only makes merged reference content visible after sandbox skill sync, while the normal prompted read path and `workspaceAccess: rw` path still point agents at the original `SKILL.md`.

## What This Changes

This PR adds skill metadata parsing for `references`, loads skill-local `references/*.md` content into `SkillEntry`, appends it to synced sandbox `SKILL.md` files, and adds docs, changelog, and tests for that behavior.

## Best Possible Solution

Materialize one supported merged skill-read contract: when a selected skill has auto-loaded references, every advertised `<location>` should resolve to content that includes `SKILL.md` plus the allowed reference markdown, across normal runs, skill snapshots, read-only sandbox sync, and `workspaceAccess: rw`. Keep the `.md`, realpath containment, and aggregate size guards, then update docs and changelog only after those paths are covered by tests.

## Reproduction Assessment

Yes. A static reproduction is clear: create a skill with `references/guide.md`, run a non-sandbox or `workspaceAccess: rw` agent turn, and the prompt still tells the agent to read the original `SKILL.md` path while the PR only appends references in `syncSkillsToWorkspace`.

## Solution Assessment

No. The PR's current approach is not the best fix because it stores merged content on `SkillEntry` but only exposes it through sandbox sync; a safer solution is to make the prompt/read location or read contract itself point to the same merged content in every supported runtime path.
