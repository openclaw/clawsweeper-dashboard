## Summary

## What This Changes

The PR adds `references` skill metadata/frontmatter parsing, loads skill-local markdown reference files into `SkillEntry`, appends them during skill sync, and documents/tests the new `references/` directory behavior.

## Best Possible Solution

Define one shared materialized skill content contract used by prompt, snapshot, normal local, read-write sandbox, and copied-sandbox paths, with the same size/security guards and focused real-run proof.

## Reproduction Assessment

Yes for the PR defect: source inspection shows `referencesContent` is created but only appended inside `syncSkillsToWorkspace`, while normal and read-write paths still read the original `SKILL.md` location. I did not run tests because this review was required to keep the checkout read-only.

## Solution Assessment

No. A sync-only append is not the best implementation for a public skill content feature; the safer path is one shared materialized read contract with tests for every runtime path it promises to support.
