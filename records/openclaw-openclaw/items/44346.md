## Summary

Keep open: the feature is not already on current main, and the PR head still has a blocking contract bug because reference content is only appended during sandbox skill sync while common skill reads still use the original `SKILL.md` path.

## What This Changes

The PR adds `metadata.openclaw.references` parsing, skill-local markdown reference loading, sync-time `SKILL.md` appending, docs, changelog, and skill tests.

## Best Possible Solution

Introduce one documented merged skill-read/materialization contract that exposes selected references consistently across normal runs, snapshots, read-only sandbox sync, and `workspaceAccess: "rw"`, with real proof from an actual agent run.

## Reproduction Assessment

Yes. Source inspection of the PR head shows `entry.referencesContent` is appended only in `syncSkillsToWorkspace`, while current prompt paths advertise `entry.skill.filePath` and rw sandbox setup skips sync.

## Solution Assessment

No. A sync-only append is not the best fix because it makes the feature depend on one workspace-copy path instead of a common read/materialization contract.
