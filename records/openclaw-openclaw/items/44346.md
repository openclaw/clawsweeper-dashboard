## Summary

Keep open: current main still lacks the requested skill references capability, but this PR should not merge as-is because the new reference content is only appended in the sandbox sync path and is not available through the normal skill read contract. The PR also still lacks after-fix real behavior proof from the contributor.

## What This Changes

Adds skill metadata parsing, docs, changelog, loader logic, and tests for auto-loading `references/*.md` and appending that content during skill sync.

## Best Possible Solution

Define one shared skill content materialization contract so selected references are visible consistently in normal runs, snapshots, sandbox sync, and read-write workspaces, then land it with focused tests, docs, and redacted real-run proof.

## Reproduction Assessment

Yes. Source inspection of PR head shows `referencesContent` is appended only in `syncSkillsToWorkspace`, while current main prompts agents to read the original `skill.filePath` and read-write or non-sandbox runs do not rely on that sync copy.

## Solution Assessment

No. A sync-only append is not the narrow maintainable solution because reference availability should not depend on one sandbox-copy path; a shared read/materialization contract is safer.
