## What This Changes

The PR adds `references` skill metadata, `referencesContent` on `SkillEntry`, skill-local markdown reference loading, sync-time `SKILL.md` appending, docs, changelog text, and parser/loader tests.

## Best Possible Solution

Define one shared materialized skill content contract so selected references are visible consistently in prompt, snapshot, normal local, sandbox-copy, and read-write workspace paths, then land it with focused regression tests and redacted real-run proof.

## Reproduction Assessment

Yes. Source inspection of the PR head shows `referencesContent` is appended only in `syncSkillsToWorkspace`, while current prompt and sandbox read-write paths still direct agents to the original `SKILL.md` location.

## Solution Assessment

No. A sync-only append is not the best implementation; the maintainable path is a shared materialized skill-read contract used by prompt, snapshot, runtime, and sandbox consumers.
