## Summary

Keep open: current main still lacks skill reference auto-load support, and the PR head has a blocking read-path bug where references are only merged for copied read-only sandbox skills. The external PR also has only test claims, not after-fix real behavior proof from an actual run.

## What This Changes

The PR adds `metadata.openclaw.references` parsing and skill-local `references/*.md` loading, appends selected reference content to synced sandbox `SKILL.md` files, and updates docs, changelog, and skill tests.

## Best Possible Solution

Materialize one supported merged skill-read contract so selected skills resolve to `SKILL.md` plus allowed reference markdown across normal runs, skill snapshots, read-only sandbox sync, and `workspaceAccess: rw`.

## Reproduction Assessment

Yes. Source inspection of the PR head shows `referencesContent` is only appended during `syncSkillsToWorkspace`, while the prompt still tells agents to read the original `<location>` and rw sandbox runs skip sync.

## Solution Assessment

No. A sync-only append is not the best fix because it makes the feature depend on one workspace-copy path; the safer repair is a common read/materialization path or advertised location that always resolves to merged content.

## Review Findings
