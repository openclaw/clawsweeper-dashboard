## Summary

Keep open: current main still lacks `references/` auto-load support, but this PR is not merge-ready because the branch only materializes references during sandbox skill sync while common skill-read paths still point at the original `SKILL.md`. It also lacks contributor real-behavior proof and needs maintainer agreement on the core skill materialization contract.

## What This Changes

The PR adds metadata parsing, docs/changelog entries, tests, and workspace loading for skill-local `references/` markdown files, appending selected content to synced sandbox copies of `SKILL.md`.

## Best Possible Solution

Define one merged skill read/materialization contract so selected references are visible consistently in normal runs, snapshots, read-only sandbox sync, and `workspaceAccess: "rw"`, then document and test those paths.

## Reproduction Assessment

Yes. Source inspection of the PR head shows `referencesContent` is populated on `SkillEntry` but appended only during sandbox sync, while prompt and `workspaceAccess: "rw"` paths still read the original `SKILL.md` location.

## Solution Assessment

No. A sync-only append is not the narrow maintainable solution because it makes the feature depend on one workspace-copy path instead of a common skill read/materialization contract.
