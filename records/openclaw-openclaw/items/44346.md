## Summary

Keep open: current main does not implement skill `references/` support, and the PR still has a blocking read-contract bug because merged reference content is only appended in one sandbox sync path.

## What This Changes

The PR adds `metadata.openclaw.references` parsing, loads skill-local `references/*.md` content, appends it during sandbox skill sync, documents the feature, adds a changelog entry, and adds focused frontmatter/workspace tests.

## Best Possible Solution

Materialize one supported merged skill-read contract so selected skills with auto-loaded references resolve to `SKILL.md` plus allowed reference markdown across normal runs, skill snapshots, read-only sandbox sync, and `workspaceAccess: rw`, then document that behavior with tests.

## Reproduction Assessment

Yes. A static reproduction is clear: create a skill with `references/guide.md`, run a normal or `workspaceAccess: rw` agent turn, and the prompt still sends the agent to the original `SKILL.md` while the PR only appends references during sandbox sync.

## Solution Assessment

No. The PR's sync-only append is not the best fix because it makes the feature depend on one workspace-copy path; the safer repair is to make the advertised skill location or read contract itself resolve to merged content in every supported runtime mode.
