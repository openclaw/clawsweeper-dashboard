## Summary

Keep open: current main does not implement skill `references/` metadata or content materialization, and the PR still has a blocking design gap because references are only appended during sandbox skill sync while common skill-read paths keep reading the original `SKILL.md`. The PR also still lacks after-fix real behavior proof from a real run.

## What This Changes

The PR adds skill `references/` auto-loading metadata, loader logic, docs, changelog text, and tests for appending selected markdown reference files to skill content.

## Best Possible Solution

Define a shared materialized skill content contract so selected references are visible consistently in normal runs, snapshots, sandbox sync, and read-write workspaces, then land it with focused regression tests, docs, and redacted real-run proof.

## Reproduction Assessment

Yes. Source inspection of the PR head shows reference content is appended only in `syncSkillsToWorkspace`, while current prompt and runtime paths still direct agents to read the original `SKILL.md` path in common runs.

## Solution Assessment

No. A sync-only append is not the narrowest maintainable solution; the safer direction is one shared materialized skill-read contract used by prompt, snapshot, runtime, and sandbox consumers.
