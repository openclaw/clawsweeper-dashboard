## What This Changes

This PR adds skill `references` metadata parsing, loads skill-local `references/*.md` into `SkillEntry.referencesContent`, appends it during skill sync, documents the behavior, and adds tests.

## Best Possible Solution

Define a shared materialized skill-content contract used by every skill read path, make reference auto-loading opt-in or explicitly migrate intended bundled skills, document the exact semantics, and prove it with a real run.

## Reproduction Assessment

Yes for the PR defects: source inspection shows reference content is only appended in `syncSkillsToWorkspace`, while current main still tells agents to read the original `SKILL.md` location. I did not run tests because this review is read-only.

## Solution Assessment

No: the current implementation is not the best way to solve the issue because it changes existing `references/` semantics and does not centralize materialized skill content. A safer solution is shared content resolution plus explicit opt-in for auto-loaded references.
