The PR adds skill `references` metadata, loads skill-local `references/*.md`, appends that content during skill sync, updates skills docs/changelog, and adds parser/workspace tests.

## Best Possible Solution

Define a shared materialized skill-content contract used by prompt, snapshot, local, read-write, and sandbox reads; make reference loading opt-in; document supported metadata syntax; and prove the behavior with a real run.

## Reproduction Assessment

Yes for the PR defects: source inspection shows the default loader would consume existing `references/*.md` directories and the merged content is only appended in `syncSkillsToWorkspace`. I did not run tests because this review is read-only.

## Solution Assessment

No: the current branch is not the best way to solve the issue because it changes existing reference semantics and does not centralize materialized skill content. A safer path is explicit opt-in plus one shared content/path contract across all skill read modes.
