## Summary

Keep open. Current main still lacks the requested skill references contract, and the PR head still has a blocking design bug: references are appended only during sandbox skill sync, so normal skill reads and `workspaceAccess: "rw"` keep reading the original `SKILL.md` without the referenced content.

## What This Changes

The PR adds skill `references.autoLoad` metadata parsing, loads selected `references/*.md` files, appends them to copied sandbox skill files, and documents/tests the behavior.

## Best Possible Solution

Define one merged skill content/materialization contract used consistently by prompt reads, snapshots, sandbox sync, and rw workspaces, then document and test those paths.

## Reproduction Assessment

Yes. Source inspection at PR head shows `referencesContent` is populated on `SkillEntry` but appended only in `syncSkillsToWorkspace`, while the normal prompt/read contract still points agents at the original `skill.filePath` and `workspaceAccess: "rw"` skips sync.

## Solution Assessment

No. A sync-only append is not the narrow maintainable solution because reference availability should not depend on one workspace-copy path; a shared skill content materialization contract is safer.
