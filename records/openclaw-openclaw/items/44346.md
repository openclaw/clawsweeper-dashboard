## What This Changes

The PR adds skill `references/` metadata parsing, docs, tests, and workspace-loader logic to append selected reference markdown to synced `SKILL.md` files.

## Best Possible Solution

Define one shared skill content materialization contract so selected references are visible consistently in prompt, snapshot, normal read, sandbox sync, and read-write workspace flows, then land it with focused regression coverage and redacted real-run proof.

## Reproduction Assessment

Yes. Source inspection of the PR head shows reference content is appended only in `syncSkillsToWorkspace`, while current-main prompt and runtime paths continue to advertise and read the original `skill.filePath`.

## Solution Assessment

No. A sync-only append is not the best fix because reference availability should not depend on one sandbox-copy path; the safer direction is a shared materialized skill-read contract.
