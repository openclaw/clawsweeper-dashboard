## Summary

Keep this PR open. Current main still lacks the feature, and the PR head still misses the common runtime read contract: normal runs and `workspaceAccess: rw` runs advertise the original `SKILL.md` path, so appended reference content is only available after sandbox sync. The latest PR commit appears to address the earlier realpath and aggregate-size comments, but not the remaining P1 behavior gap.

## What This Changes

This PR adds `metadata.openclaw.references` parsing, loads skill-local `references/*.md` content into skill entries, appends it to sandbox-synced `SKILL.md` files, and adds docs, changelog, and tests.

## Best Possible Solution

Materialize one supported merged skill-read contract: when a selected skill has auto-loaded references, every advertised `<location>` should resolve to content that includes `SKILL.md` plus allowed reference markdown across normal runs, skill snapshots, read-only sandbox sync, and `workspaceAccess: rw`. Keep the `.md`, realpath containment, per-file, and aggregate size guards, then document the behavior once those paths are covered by tests.

## Reproduction Assessment

Yes. A static reproduction is clear: create a skill with `references/guide.md`, run a non-sandbox or `workspaceAccess: rw` agent turn, and the prompt still sends the agent to the original `SKILL.md` while the PR only appends references during sandbox sync.

## Solution Assessment

No. The PR's sync-only append is not the best fix because it makes the feature depend on one workspace-copy path; the safer repair is to make the advertised skill location or read contract itself resolve to merged content in every supported runtime mode.
