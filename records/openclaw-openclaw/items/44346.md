## Summary

Keep open: current main still lacks a `references/` auto-load contract, but this PR should not merge as-is because it only materializes references during sandbox skill sync, misses common read paths and `workspaceAccess: "rw"`, and has no external real-behavior proof.

## What This Changes

Adds skill metadata, docs, loader logic, and tests for auto-loading selected `references/*.md` files and appending them during sandbox skill sync.

## Best Possible Solution

Define one shared skill content materialization contract so selected references are visible consistently in normal runs, snapshots, sandbox sync, and `workspaceAccess: "rw"`, then land it with focused tests, docs, and redacted real-run proof.

## Reproduction Assessment

Yes. Source inspection of PR head shows `referencesContent` is only appended in `syncSkillsToWorkspace`, while current prompt/system-prompt contracts direct agents to read `skill.filePath` and `workspaceAccess: "rw"` skips skill sync.

## Solution Assessment

No. A sync-only append is not the narrow maintainable solution because reference availability should not depend on one sandbox-copy path; a shared materialization/read contract is safer.
