## Summary

Keep open: current main still lacks this feature, but the PR cannot merge as-is because the PR head only materializes references through sandbox skill sync, lacks contributor real-behavior proof, and changes the core skill read contract.

## What This Changes

The branch adds `metadata.openclaw.references` parsing, skill-local `references/*.md` loading, sync-time `SKILL.md` appending, docs, changelog, and skill tests.

## Best Possible Solution

Introduce one documented merged skill-read/materialization contract that exposes selected references consistently across normal runs, snapshots, read-only sandbox sync, and `workspaceAccess: "rw"`, with proof from an actual agent run.

## Reproduction Assessment

Yes. Source inspection of the PR head shows `entry.referencesContent` is appended only in `syncSkillsToWorkspace`, while current prompt paths advertise `entry.skill.filePath` and rw sandbox setup skips sync.

## Solution Assessment

No. A sync-only append is not the best fix because it makes the feature depend on one workspace-copy path; a common read/materialization path is safer.
