## Summary

Keep open: current main does not implement skill `references/` auto-load, but this PR should not merge as-is because it only materializes reference content through sandbox skill sync and lacks after-fix real behavior proof.

## What This Changes

The PR adds skill frontmatter metadata for `references.autoLoad` and `onDemand`, loads selected `references/*.md` files, appends them during sandbox skill sync, and documents/tests the behavior.

## Best Possible Solution

Define one merged skill read/materialization contract so selected references are visible consistently in normal runs, snapshots, sandbox sync, and `workspaceAccess: "rw"`, then land it with docs, focused tests, and redacted real-run proof.

## Reproduction Assessment

Yes. Source inspection at PR head shows `referencesContent` is populated on `SkillEntry` but appended only in `syncSkillsToWorkspace`, while the prompt/read contract still points agents at the original `skill.filePath` and `workspaceAccess: "rw"` skips sync.

## Solution Assessment

No. A sync-only append is not the narrow maintainable solution because reference availability should not depend on one workspace-copy path; a shared skill content materialization contract is safer.
