## Summary

Keep open: current main does not implement skill references, and this PR still exposes them only through sandbox skill sync while normal prompt reads and `workspaceAccess: "rw"` paths continue to use the original `SKILL.md`. The PR also still lacks external contributor real-behavior proof from an actual agent run.

## What This Changes

The PR adds skill frontmatter metadata for `references.autoLoad` and `onDemand`, loads selected `references/*.md` content, appends it during sandbox skill sync, and documents/tests the behavior.

## Best Possible Solution

Define one merged skill read/materialization contract so selected references are visible consistently in normal runs, snapshots, sandbox sync, and `workspaceAccess: "rw"`, then document and test those paths.

## Reproduction Assessment

Yes. Source inspection at PR head shows `referencesContent` is populated on `SkillEntry` but appended only in `syncSkillsToWorkspace`, while the current prompt/read contract still points agents at the original `skill.filePath` and `workspaceAccess: "rw"` skips sync.

## Solution Assessment

No. A sync-only append is not the narrow maintainable solution because the feature should not depend on one workspace-copy path; the safer direction is a shared skill content materialization contract used by every read path.
