## Summary

Keep open: the feature is not implemented on current main, and this PR still has a blocking behavior/design issue plus missing real behavior proof because references are appended only during sandbox skill sync while common prompt/runtime paths read the original SKILL.md.

## What This Changes

The PR adds skill `references/` metadata parsing, docs, tests, and workspace-loader logic to append selected markdown reference files to synced SKILL.md content.

## Best Possible Solution

Define one shared skill content materialization contract so selected references are visible consistently in normal runs, snapshots, sandbox sync, and read-write workspaces, then land it with focused regression tests, docs, and redacted real-run proof.

## Reproduction Assessment

Yes. Source inspection of the PR head shows reference content is appended only in `syncSkillsToWorkspace`, while current-main prompt and runtime paths still direct agents to read the original SKILL.md path.

## Solution Assessment

No. A sync-only append is not the best fix because reference availability should not depend on one sandbox-copy path; the safer direction is a shared materialized skill-read contract used by prompt, snapshot, and sync consumers.
