## Summary

Keep open. Current main does not implement skill `references/` auto-loading, and the latest PR head still has a blocking correctness gap: reference content is only appended during sandbox skill sync, while common prompt-read paths still point agents at the original `SKILL.md`; the external PR also lacks after-fix real behavior proof.

## What This Changes

The PR adds skill metadata parsing, docs, tests, and workspace-loader logic to collect `references/*.md` and append selected reference content to synced `SKILL.md` files.

## Best Possible Solution

Define one shared skill content materialization contract so selected references are visible consistently in normal runs, snapshots, sandbox sync, and read-write workspaces, then land it with focused regression tests, docs, and redacted real-run proof.

## Reproduction Assessment

Yes. Source inspection of the PR head shows reference content is appended only in `syncSkillsToWorkspace`, while current-main prompt and runtime paths continue to advertise/read the original `skill.filePath`.

## Solution Assessment

No. A sync-only append is not the best fix because reference availability should not depend on one sandbox-copy path; the safer direction is a shared materialized skill-read contract used by prompt, snapshot, and sync consumers.
