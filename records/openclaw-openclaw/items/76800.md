## Summary

Keep this PR open: it moves the fix in the right direction, but the new skip-plugin-validation flag still does not reliably reach the final persistence path, so the reported doctor write can still fail.

## What This Changes

The PR returns partially valid migrated legacy configs, threads a doctor write flag for skipping plugin validation, adds regression coverage, and updates the changelog.

## Best Possible Solution

Complete the write path so known-safe legacy migrations can be persisted while base schema validation and destructive-write guards still run, with end-to-end doctor coverage through `replaceConfigFile`.

## Reproduction Assessment

Yes. Source inspection shows current main returns `config: null` after post-migration plugin validation fails, keeps the unmigrated candidate, and validates with plugins before writing.

## Solution Assessment

No. The approach is narrow, but the current PR does not fully wire the skip flag through `replaceConfigFile` and should also account for the include-write fast path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Forward the skip flag through the public writer:** `src/config/io.ts:2068`
  - body: `runWriteConfigHealth` passes `skipPluginValidation` through `replaceConfigFile`, but `replaceConfigFile` ultimately calls the exported `writeConfigFile` wrapper in `src/config/io.ts`. That wrapper still omits `skipPluginValidation` when delegating to `createConfigIO().writeConfigFile`, so the added validation branch never sees the flag and the #76798 write still throws on the stale provider.
  - confidence: 0.93
- **[P2] Honor the skip flag before include writes:** `src/flows/doctor-health-contributions.ts:568`
  - body: Even after the wrapper forwards the option, `replaceConfigFile` first tries the single top-level include writer, whose validation still calls `validateConfigObjectWithPlugins(nextConfig)` without write options. For configs that keep the migrated top-level section in an include, `doctor --fix` will still fail before reaching the normal writer.
  - confidence: 0.84
