## Summary

Current main already implements and documents the third-party provider plugin onboarding path the issue asks for: provider ids, env vars, auth choices, model catalog rows, install-on-demand metadata, model picker setup entries, and ClawHub publishing are all covered by generic plugin surfaces and shipped in v2026.6.10.

## What This Changes

The issue asks OpenClaw to confirm and document how a third-party AI model provider plugin can register provider metadata, appear in onboarding/model selection, and publish through ClawHub.

## Best Possible Solution

Keep this as the generic plugin-first contract: AI/ML API or similar providers should publish an external ClawHub provider plugin with manifest auth, setup, install, and modelCatalog metadata.
