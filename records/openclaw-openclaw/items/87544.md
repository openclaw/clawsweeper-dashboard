## Summary

Keep open. Current main has durable outbound receipts and a Slack-only transcript mirror, but it still lacks the requested generic transcript-entry-to-provider-message ledger, Discord reconciliation semantics, Control UI delivery-state surface, and idempotent redelivery path; the linked mitigation PR is still draft/partial.

## What This Changes

The issue requests durable delivery-state tracking and reconciliation between channel-scoped assistant transcript entries and their expected Discord/client-visible messages.

## Best Possible Solution

Design one generic delivery ledger keyed by transcript entry and channel target, feed it from the durable delivery commit/reconciliation lifecycle, expose it through session APIs and Control UI, and add an idempotent operator-triggered reconciler.

## Reproduction Assessment

No high-confidence live Discord reproduction is present for the intermittent drop. Source inspection does confirm current main lacks the requested transcript-entry to provider-message delivery ledger, Discord reconciler, and UI/API delivery-state surface.

## Solution Assessment

Unclear as proposed because the goal is right but the durable implementation should extend the existing delivery queue, receipt, and commit-hook lifecycle. A parallel transcript sidecar would be riskier than a single source of delivery truth.
