## Summary

Keep open: current main supports typed `/model <provider/model>@<profileId>` and shows auth labels in `/models` text headers, but the interactive `/models` picker contract and channel callbacks still carry provider/model/runtime state only, with no selectable auth-profile/account choice.

## What This Changes

Request to let `/models` interactive buttons show and switch provider auth profiles/accounts for GPT/OpenAI and Claude/Anthropic-backed models.

## Best Possible Solution

Add provider-generic, privacy-safe auth-profile choices to the shared `/models` picker contract and persist selections through the existing `authProfileOverride` session path.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Yes as a source-level feature gap: current `/models` picker data and channel callbacks lack selectable auth-profile/account choices.

## Solution Assessment

Unclear until maintainers settle account-label privacy and account-switch session semantics. The maintainable direction is a provider-generic picker contract backed by existing auth-profile/session override plumbing, not GPT/Claude-specific picker state.
