## Summary

Keep open: current main and v2026.5.22 support typed per-session auth-profile pinning with `/model <model>@<profileId>`, but the shared `/models` interactive picker still exposes provider/model/runtime data only and has no account/auth-profile choices to render or submit.

## What This Changes

Requested behavior is to let users choose and see the backing GPT or Claude account directly from the `/models` interactive picker.

## Best Possible Solution

Add provider-generic, privacy-safe auth-profile choices to the shared `/models` picker contract and channel renderers, then apply selections through the existing `authProfileOverride` session path.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main has typed auth-profile switching but lacks selectable account/auth-profile choices in the interactive `/models` picker.

## Solution Assessment

Unclear until maintainers settle the account-label privacy and cross-channel UX contract. The maintainable direction is an additive provider-generic picker contract backed by existing session auth-profile override plumbing, not GPT/Claude-specific picker state.
