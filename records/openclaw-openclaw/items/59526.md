## Summary

Keep open: current main documents and implements typed `/model ...@<profileId>` auth-profile overrides, but the `/models` picker contracts and channel callbacks still expose provider/model/runtime selection only. Adding account switching in chat pickers needs a generic, privacy-aware UX/API decision across shared channel surfaces.

## What This Changes

Request account/profile selection directly inside the `/models` picker for GPT and Claude-style account-backed providers.

## Best Possible Solution

Add a generic, privacy-aware auth-profile choice model to shared `/models` picker data and channel contracts, then apply selected profiles through the existing `authProfileOverride` session pipeline.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that typed `/model ...@<profileId>` works while picker payloads and callbacks still carry only provider/model/runtime selections.

## Solution Assessment

Unclear until maintainers settle chat-surface UX and privacy policy for account labels. The narrow maintainable direction is a provider-generic picker contract backed by existing auth-profile/session override plumbing, not GPT/Claude-specific state.
