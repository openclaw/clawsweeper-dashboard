## Summary

Keep open: current main supports typed auth-profile selection through `/model <model>@<profileId>` and shows an auth label in `/models` text headers, but the shared interactive `/models` picker contract still has no selectable account/auth-profile choices.

## What This Changes

Allow `/models` buttons or menus to show and switch the backing GPT/Claude provider account without manually typing raw model/profile strings.

## Best Possible Solution

Add provider-generic, privacy-safe auth-profile choices to the shared `/models` picker/channel SDK contract and apply selections through the existing `authProfileOverride` session path.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current `/models` picker payloads and callbacks still lack selectable account/auth-profile choices.

## Solution Assessment

Unclear until maintainers settle account-label privacy and cross-channel UX. The maintainable direction is a provider-generic picker contract backed by existing auth-profile/session override plumbing, not GPT/Claude-specific picker state.
