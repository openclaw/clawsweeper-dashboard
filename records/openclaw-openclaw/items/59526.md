## Summary

Keep open: current main supports typed `/model ...@<profileId>` and visible auth labels, but the shared `/models` picker data and channel callbacks still only carry provider/model/runtime choices, not selectable account or auth-profile choices.

## What This Changes

Request to let `/models` picker/buttons switch both the model and the backing GPT/Claude account while showing the active account.

## Best Possible Solution

Add provider-generic, privacy-aware auth-profile choice data to the shared `/models` picker/channel SDK contract and apply selections through the existing `authProfileOverride` path.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current `/models` picker payloads and callbacks still lack selectable account/auth-profile choices.

## Solution Assessment

Unclear until maintainers settle account-label privacy and cross-channel UX. The maintainable direction is a provider-generic picker contract backed by existing auth-profile/session override plumbing, not GPT/Claude-specific picker state.
