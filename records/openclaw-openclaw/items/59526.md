## Summary

Keep open: current main supports typed `/model <model>@<profile>` and lower-level profile override persistence, but the interactive `/models` picker paths still expose provider/model/runtime selection without account/profile choices.

## What This Changes

Add account/profile selection and active-account visibility to `/models` picker buttons for GPT/OpenAI and Claude/Anthropic-backed models.

## Best Possible Solution

Extend the existing picker/directive flow so provider, model, runtime, and auth profile are selected and displayed together through the shared session override and profile validation contracts.

## Reproduction Assessment

Yes, at source level: the picker paths persist provider/model and sometimes runtime, while typed `/model ...@profile` is the path that resolves and persists auth profiles. I did not run a live chat flow because the missing capability is visible in the current source.

## Solution Assessment

Unclear until maintainers choose the UX shape. The maintainable direction is to extend the existing picker/directive contracts rather than add a separate raw model-string or account-editing path.
