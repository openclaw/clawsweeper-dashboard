## Summary

Keep open: current main supports typed `/model ...@<profileId>` and shows an active auth label, but the shared `/models` picker contracts still expose provider/model/runtime choices only, with no selectable account or auth-profile choices.

## What This Changes

Request to add account/auth-profile selection and active account visibility directly inside the `/models` UI/buttons for GPT- and Claude-style account-backed models.

## Best Possible Solution

Add provider-generic, privacy-aware auth-profile choice data to shared `/models` picker data and channel SDK contracts, then apply selections through the existing `authProfileOverride` persistence path.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current `/models` picker payloads and callbacks still lack selectable account/auth-profile choices.

## Solution Assessment

Unclear until maintainers settle account-label privacy and cross-channel UX. The maintainable direction is a provider-generic picker contract backed by existing auth-profile/session override plumbing, not GPT/Claude-specific picker state.
