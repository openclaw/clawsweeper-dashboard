## Summary

Keep open: current main supports typed `/model ...@<profileId>` overrides and shows the active auth label in some provider-list text, but the shared `/models` picker data and channel interaction paths still do not expose selectable auth-profile/account choices.

## What This Changes

Allow the `/models` picker/buttons to choose and display provider auth profiles/accounts for GPT/OpenAI and Claude/Anthropic models.

## Best Possible Solution

Add a generic, privacy-aware auth-profile choice model to shared `/models` picker data and channel contracts, then apply selected profiles through the existing `authProfileOverride` session field.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives a high-confidence current-main gap: typed `/model ...@<profileId>` works, while `/models` channel picker payloads and callbacks only carry provider/model/runtime selections.

## Solution Assessment

Unclear until maintainers settle the UX/privacy policy for account labels in chat. The narrow maintainable direction is provider-generic picker data backed by the existing auth-profile store and session override pipeline, not GPT/Claude-specific state.
