## Summary

Keep open: current main still lacks first-class account/profile choices in the interactive `/models` picker paths; typed `/model ...@profile` support and the related preservation PR do not satisfy the requested picker UX.

## What This Changes

Request to let users choose and see the provider account/auth profile directly from `/models` buttons or menus for GPT/OpenAI and Claude/Anthropic models.

## Best Possible Solution

Define one cross-channel picker UX that lists compatible auth profiles for the selected provider/model and persists through the existing session override and profile validation contracts.

## Reproduction Assessment

Yes, at source level: current picker paths carry provider/model/runtime state only, while typed `/model ...@profile` resolves and persists auth profiles. I did not run a live chat flow because the missing capability is directly visible in current source.

## Solution Assessment

Unclear until maintainers choose the UX shape. The maintainable direction is to extend the existing picker/directive/profile contracts rather than add raw model-string editing or channel-specific account hacks.
