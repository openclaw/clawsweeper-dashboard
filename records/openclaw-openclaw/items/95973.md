## Summary

## What This Changes

The PR adds plugin-specific approval setup guidance plumbing, wires Telegram to that guidance, documents the new channel approval hook, and adds focused approval regression tests.

## Best Possible Solution

Carry gateway delivery/no-route state through the plugin approval request result so setup guidance appears for undelivered Telegram native approvals, while actually delivered plugin routes keep plain timeout behavior and the SDK hook lands only with maintainer acceptance.

## Reproduction Assessment

Yes. Source inspection shows the gateway can accept a plugin approval without actual delivery via `hasApprovalTurnSourceRoute`, and the PR head then returns plain `Approval timed out` after `plugin.approval.waitDecision`.

## Solution Assessment

No. The immediate no-route fix is narrow, but the patch needs route/delivery state for the timeout branch so it does not miss accepted-but-undelivered Telegram native approval failures.
