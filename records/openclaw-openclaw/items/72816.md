## Summary

Keep this PR open, but it needs changes before merge: the core idea is still useful and not on current main, while the patch introduces a narrow mobile layout regression and still lacks real UI/runtime proof.

## What This Changes

The PR adds a pre-stream assistant activity indicator in Control UI chat, styles and tests it, and makes a small Codex plugin test cleanup.

## Best Possible Solution

Land a focused UI change that shows pre-stream assistant activity, preserves the existing streaming avatar behavior, hides cleanly in narrow mobile layouts, and includes contributor-provided real browser proof.

## Reproduction Assessment

Not applicable for the feature request itself. The mobile regression is source-reproducible from the PR wrapper plus the current `max-width: 400px` avatar-hide rule; I did not run browser E2E in this read-only review.

## Solution Assessment

No, not yet. Passing an active-run flag into chat item building is a reasonable implementation direction, but the patch should first cover the mobile avatar-hide case and provide real UI proof.

Overall correctness: patch is incorrect

- **[P2] Hide the activity wrapper with mobile avatars:** `ui/src/styles/components.css:3516-3522`
