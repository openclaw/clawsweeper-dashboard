## What This Changes

The PR changes reply and gateway subagent model resolution so spawned subagent sessions prefer configured subagent defaults over the parent primary or parent stored `/model` override, with focused regression tests and a changelog entry.

## Best Possible Solution

Resolve the effective subagent model before all reply preprocessing that depends on activeModel, keep heartbeat and explicit child overrides ahead of defaults, and land with one coordinated subagent precedence contract across reply, gateway, spawn, and cron surfaces.
