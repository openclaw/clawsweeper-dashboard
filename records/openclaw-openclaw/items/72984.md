## Summary

Keep open: this PR addresses a real subagent model-routing bug, but the latest head still has one correctness blocker where alias resolution can use the parent default alias map after the runtime default is switched to the subagent model.

## What This Changes

The PR changes subagent runtime model resolution in agent, gateway, reply, stored-override, and cron paths so spawned subagent sessions use configured subagent defaults instead of parent primary models, with tests and a changelog entry.

## Best Possible Solution

Land this after the effective alias index is updated together with the subagent default switch, with regression coverage for reset/channel/directive alias resolution in subagent sessions.

## Reproduction Assessment

Yes, at source level. The docs define the subagent default contract, current main still resolves reply defaults before loading subagent session state, and the PR body provides a real macOS gateway proof of the wrong persisted model before the fix.

## Solution Assessment

No as-is. The fix direction is narrow and maintainable, but the PR must keep the alias index synchronized when it changes the effective default provider/model for subagent sessions.
