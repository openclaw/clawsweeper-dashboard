- **[P2] Keep subagent alias resolution runtime-free:** `src/auto-reply/reply/directive-handling.defaults.ts:66-75`
  - body: `resolveDefaultModel` above deliberately disables plugin runtime normalization because this lookup runs on every reply. This new subagent path builds and resolves aliases without that flag, so a configured subagent alias can cold-load plugin runtime hooks or normalize differently on the reply hot path. Pass `allowPluginNormalization: false` through both calls.
  - confidence: 0.86
- **[P2] Rebuild subagent alias index without runtime hooks:** `src/auto-reply/reply/get-reply.ts:579`
  - body: After switching `defaultProvider` to the subagent provider, this rebuild drops the no-plugin normalization policy used by the reply path. Downstream bare model overrides in the same subagent reply can now invoke plugin runtime normalization; rebuild this alias index with the same runtime-free option.
  - confidence: 0.82
- **[P2] Honor gateway no-plugin normalization in subagent fallback:** `src/gateway/session-utils.ts:1767-1776`
  - body: `resolveSessionModelRef` accepts `allowPluginNormalization`, and gateway search/list callers pass `false` for static display paths. The new subagent fallback ignores that option when building and resolving aliases, so a subagent alias can still trigger plugin runtime normalization in gateway status/listing. Forward `options?.allowPluginNormalization` here too.
  - confidence: 0.87
