## Summary

Keep open: the branch addresses the documented subagent default precedence bug and supplies real behavior proof, but the latest stored-override guard overcorrects by blocking parent `/model` inheritance for all subagent entries, including the documented caller-inheritance case when no subagent default is configured.

## What This Changes

The PR changes subagent model selection in gateway, reply-runtime, and cron paths so configured subagent defaults beat parent agent primary models, with alias-aware fallbacks, regression tests, and a changelog entry.

## Best Possible Solution

Land one canonical fix that preserves the documented order: explicit child model, per-agent/global subagent defaults, then caller inheritance when no subagent model is configured, consistently across spawn, reply runtime, gateway session resolution, and cron.

## Reproduction Assessment

Yes, at source level. The docs define caller inheritance unless subagent defaults are configured, while PR head returns null for parent overrides on any subagent entry without a direct override, which reproduces the regression with a stored parent `/model` override and no subagent default.

## Solution Assessment

No, not yet. The selector and runtime fallback direction is maintainable, but the stored parent-override guard should be conditional on an actual configured subagent default so caller inheritance still works when no subagent model is set.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Preserve parent /model inheritance without subagent defaults:** `src/auto-reply/reply/stored-model-override.ts:59-60`
  - body: This guard now disables parent override inheritance for every subagent child that lacks its own override. The docs say `sessions_spawn` inherits the caller unless a per-agent or global subagent model is configured, so an older or racy child entry with no subagent default will fall back to the agent primary instead of the parent's selected `/model`. Please only skip parent inheritance when an actual subagent default is configured, and add that regression case.
