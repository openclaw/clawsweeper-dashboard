## Summary

Keep open: the PR is directionally correct and has supplied real behavior proof, but the latest head still lets an inherited parent stored model override replace the configured subagent default in one source-reproducible path.

## What This Changes

The PR changes subagent model-selection precedence and adds gateway, reply-runtime, isolated-cron, tests, and changelog coverage so configured subagent defaults can beat parent agent primary models.

## Best Possible Solution

Keep the unified selector, but make configured subagent defaults outrank inherited parent stored overrides while preserving explicit child overrides, then add a focused regression for that session-store shape.

## Reproduction Assessment

Yes. Current main docs and source show the reported precedence bug, and the remaining PR blocker is source-reproducible from the PR head ordering: the subagent default is applied before inherited parent stored overrides are resolved.

## Solution Assessment

No, not fully yet. The selector, alias-aware fallback, and cron direction are maintainable, but the reply runtime must either skip parent override inheritance when a configured subagent default applies or apply the subagent default after that parent-inheritance pass.
