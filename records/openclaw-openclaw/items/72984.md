## Summary

Keep open: the reported subagent model-resolution bug is real and the branch is close, but the latest patch can overwrite an already-selected one-turn image model override when the session is a subagent, so image-bearing subagent turns may stop using the configured image model.

## What This Changes

The branch changes subagent runtime model resolution in session fallback, reply defaults, stored parent override inheritance, alias handling, tests, and changelog text so spawned subagents use configured subagent defaults instead of parent primary models.

## Best Possible Solution

Preserve explicit and one-turn model overrides ahead of subagent default fallback, then land after maintainers accept the compatibility semantics and coordinate the remaining precedence work with the related open PR.

## Reproduction Assessment

Yes, at source level. Current main resolves reply runtime defaults before loading sessionEntry and resolveSessionModelRef falls back to the parent agent default when a subagent entry has no runtime model.

## Solution Assessment

No for the current patch shape because the new subagent default block can replace an already-selected one-turn image model override. The narrow fix is to preserve explicit/one-turn overrides and apply the subagent default only as a fallback.
