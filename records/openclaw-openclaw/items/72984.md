## Summary

Keep this PR open: it addresses a real documented subagent model-selection bug and has sufficient real behavior proof, but the current head still lets an inherited parent session model override replace the configured subagent default in one reply-runtime path.

## What This Changes

The PR changes subagent model selection so configured subagent defaults are used across spawn, gateway session fallback, reply runtime, and isolated cron paths instead of falling back to the parent agent primary model.

## Best Possible Solution

Keep the unified subagent default precedence, but make the reply-runtime selection order ensure configured subagent defaults outrank inherited parent stored overrides while preserving explicit child overrides.

## Reproduction Assessment

Yes, at source level. Current main docs define the subagent default contract, current main code and tests still prefer the agent primary over the global subagent default, and the PR body supplies live sessions_spawn proof for the reported normal path.

## Solution Assessment

No, not fully yet. The shared selector, cron alignment, and alias-aware gateway/reply fallback are the right direction, but the reply runtime must either apply the configured subagent default after parent override inheritance or skip parent inheritance when that configured subagent default applies.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep parent overrides from replacing subagent defaults:** `src/auto-reply/reply/get-reply.ts:404-416`
  - body: The new subagent default is applied before `resolveStoredModelOverride()`. When a child subagent entry has no direct override but its parent session does, the later stored-override block still takes the parent override and switches the run back to the parent model, which is the reported priority-chain bug. Move the subagent-default resolution after parent inheritance or skip parent inheritance for configured subagent-default sessions, and cover that session-store shape.
  - confidence: 0.86
