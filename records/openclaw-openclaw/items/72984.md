## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Keep reply subagent alias resolution runtime-free:** `src/auto-reply/reply/directive-handling.defaults.ts:66-75`
  - body: `resolveDefaultModel` deliberately disables plugin normalization because this lookup is on every reply and plugin runtime normalization can cold-load plugins. The new subagent helper builds and resolves aliases without that flag, so alias-configured subagent replies can enter the broader runtime normalization path on the hot reply path; pass the same static policy through both calls.
  - confidence: 0.86
- **[P2] Preserve the static alias policy after switching defaults:** `src/auto-reply/reply/get-reply.ts:582`
  - body: After the subagent default replaces `defaultProvider`, this alias-index rebuild omits the same `allowPluginNormalization: false` policy used by the original reply defaults. Downstream channel, reset, stored, and dispatch overrides should keep the static alias policy after the provider switch.
  - confidence: 0.84
- **[P2] Forward gateway normalization options into the fallback:** `src/gateway/session-utils.ts:1678-1687`
  - body: `resolveSessionModelRef` accepts `allowPluginNormalization`, but the new subagent fallback builds and resolves aliases without passing that option. Gateway callers that deliberately disable plugin normalization can still hit broader normalization through this fallback.
  - confidence: 0.85
