## Summary

Keep open. The PR is still useful because current main has not fully adopted the requested subagent-default precedence or the proposed empty-entry fallback, but the current patch needs changes: both new fallback paths bypass the alias-aware model resolution that current main already uses for subagent spawn selection.

## What This Changes

The PR reorders subagent configured model precedence, adds gateway and reply-runtime subagent default fallbacks with tests, adjusts config-validation code, and changes the qa-lab Command re-export to type-only.

## Best Possible Solution

Revise the PR so reply-time and gateway subagent fallbacks call the existing alias-aware subagent spawn resolver or a shared alias-aware helper, add alias regression coverage for both fallback paths, and keep the documented precedence change coordinated with tests and changelog before merge.

## Reproduction Assessment

Yes. Configure agents.defaults.models with an alias such as gpt, set agents.defaults.subagents.model to that alias, then exercise the PR's reply helper or an empty-model subagent resolveSessionModelRef path; the PR parses the alias as defaultProvider/gpt instead of the configured provider/model.

## Solution Assessment

No. The direction is useful, but this patch is not the best fix until the new fallback paths reuse the same alias-aware resolver as sessions_spawn and the user-visible precedence change is documented for release.
