## Summary

Keep this PR open, but it still needs a narrow correctness repair: the latest head applies the configured subagent default before stored parent override inheritance, so a child session without its own persisted override can still be switched back to the parent model.

## What This Changes

The PR changes gateway, reply runtime, isolated cron model selection, tests, and changelog text so configured subagent model defaults beat parent agent primary models.

## Best Possible Solution

Keep the unified subagent default precedence, but make reply-runtime stored override resolution skip parent inheritance when a configured subagent default applies, or apply that default after parent override resolution, with a focused regression for the edge case.

## Reproduction Assessment

Yes, at source level. On the PR head, create a subagent session with `spawnDepth >= 1` or `subagentRole`, no child override, and a parent session override; `resolveSubagentSessionDefaultModel` sets the configured default first, then `resolveStoredModelOverride` can return the parent override and replace it.

## Solution Assessment

No, not fully yet. The shared selector and alias-aware gateway/reply fallback direction is maintainable, but the reply-runtime ordering must prevent parent stored overrides from beating configured subagent defaults.
