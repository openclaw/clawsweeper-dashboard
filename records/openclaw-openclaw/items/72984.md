## Summary

Keep open: this PR addresses a real current-main subagent model-routing bug and still carries useful runtime/gateway/stored-override work, but latest head still has a blocking alias-index mismatch after replacing the active subagent defaults.

## What This Changes

The PR changes subagent model selection across gateway session resolution, reply startup, stored parent override handling, isolated cron selection, tests, and changelog so spawned subagents use configured subagent defaults instead of the parent primary.

## Best Possible Solution

Land the broader subagent runtime fix after keeping active default provider/model and alias index together whenever subagent session defaults replace parent defaults.

## Reproduction Assessment

Yes, source-reproducible. Current main still has reply startup and stored parent override paths that can select parent/default models for subagent sessions, and PR head still leaves a stale alias index after switching defaults.

## Solution Assessment

No as-is. The direction is maintainable, but the PR should rebuild or carry the alias index with the updated subagent default provider/model before reset, channel, and inline directive model paths are safe.
