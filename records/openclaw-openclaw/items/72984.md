## Summary

Keep open: the PR addresses a real documented subagent model-selection bug and the latest head has green checks, but the new reply-runtime fallback can still be replaced by an inherited parent session model override in the no-direct-child-override path.

## What This Changes

The branch changes subagent model precedence and adds gateway, reply-runtime, cron, changelog, and regression-test coverage so configured subagent defaults beat parent agent primary models.

## Best Possible Solution

Keep the unified subagent default precedence, but make the reply-runtime fallback immune to parent stored overrides when a configured subagent default applies, then add a focused regression for that session-store shape.

## Reproduction Assessment

Yes. Current main docs and source show the precedence bug, and the PR body supplies a concrete local `sessions_spawn` proof; the remaining blocker is source-reproducible from the new fallback running before parent stored override inheritance.

## Solution Assessment

No, not fully yet. The shared selector and alias-aware fallback direction is maintainable, but the reply-runtime fallback must either run after parent override inheritance or skip parent inheritance for configured subagent-default sessions.
