## Summary

Keep this PR open, but it still needs one narrow repair before merge: the branch fixes real subagent model-selection gaps, yet the reply path leaves the alias index tied to the old parent default after switching to the subagent default.

## What This Changes

The PR changes subagent model resolution across agent model selection, gateway session resolution, reply startup, stored override handling, isolated cron selection, tests, and changelog text.

## Best Possible Solution

Land the broader subagent runtime fix after tying the active default provider/model and alias index together whenever subagent session defaults replace the parent defaults.

## Reproduction Assessment

Yes, at source level. Current main still has reply startup, stored parent override, and gateway fallback paths that can select the parent/default model for subagent sessions despite the documented subagent default contract.

## Solution Assessment

No as-is. The direction is the maintainable fix, but the current PR must rebuild or carry the alias index with the updated subagent default provider before reset, channel, and inline model directive paths are safe.

- **[P2] Rebuild alias index after changing subagent defaults:** `src/auto-reply/reply/get-reply.ts:473-477`
