## Summary

Keep this PR open: it fixes a real subagent model-routing bug and has sufficient live proof, but the current patch still overwrites one-turn image model overrides for subagent sessions, which can route image turns to a non-vision subagent default.

## What This Changes

The PR changes subagent model resolution across session fallback, reply defaults, stored parent override inheritance, cron exports, tests, and changelog text so spawned subagents use configured subagent defaults instead of parent primary/session models.

## Best Possible Solution

Preserve explicit and one-turn image model overrides ahead of subagent default fallback, then land only after maintainers accept the compatibility semantics and coordinate the overlapping precedence PR.

## Reproduction Assessment

Yes, at source level. Current main resolves one-turn image model overrides before session state and protects them from later model selection, while the PR adds a later subagent-default assignment that does not check hasAppliedImageModelOverride.

## Solution Assessment

No, not in the current patch shape. The maintainable fix is to apply the subagent default only as a fallback and preserve explicit one-turn image overrides, while keeping the broader precedence change as a maintainer-approved compatibility decision.
