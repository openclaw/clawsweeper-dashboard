## What This Changes

This PR adds subagent-aware model default resolution in reply and gateway session paths, skips parent model override inheritance for spawned subagents, and adds regression tests.

## Best Possible Solution

Keep the runtime fallback direction, thread no-plugin normalization through the new alias paths, and coordinate the compatibility-sensitive precedence decision with https://github.com/openclaw/openclaw/pull/58823 before merge.

## Reproduction Assessment

Yes, at source level. Current main resolves reply defaults from the parent agent before session-entry-specific subagent state is considered, and parent stored model overrides can still be inherited by spawned subagent child sessions.

## Solution Assessment

Partly, but not merge-ready. The runtime fallback is a plausible owner-boundary fix for raced or clobbered subagent entries, but the new alias resolution must preserve the no-plugin hot-path contract and the broader precedence work should be sequenced with the related PR.

- **[P2] Keep subagent alias resolution runtime-free:** `src/auto-reply/reply/directive-handling.defaults.ts:66-75`
  - body: `resolveDefaultModel` above deliberately disables plugin runtime normalization for every-reply default and alias lookup. This new helper builds and resolves the subagent alias index without `allowPluginNormalization: false`, so a configured subagent alias can cold-load plugin runtime or normalize differently on the reply hot path; pass the flag through both calls.
  - confidence: 0.87
- **[P2] Rebuild subagent alias index without runtime hooks:** `src/auto-reply/reply/get-reply.ts:579`
  - body: After switching `defaultProvider` to the subagent provider, this rebuild omits the same no-plugin normalization flag used by the original reply alias index. Downstream bare channel, reset, or dispatch overrides can therefore invoke plugin runtime normalization on every subagent reply; rebuild with `allowPluginNormalization: false`.
  - confidence: 0.84
- **[P2] Honor gateway no-plugin normalization in the subagent fallback:** `src/gateway/session-utils.ts:1767-1776`
  - body: `resolveSessionModelRef` already accepts `allowPluginNormalization` and session display callers pass `false`, but this new subagent fallback builds and resolves the alias index without forwarding that option. A subagent alias can still invoke plugin runtime normalization in gateway status/list paths; pass `options?.allowPluginNormalization` through both calls.
