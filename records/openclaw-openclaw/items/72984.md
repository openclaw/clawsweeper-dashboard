## Summary

Keep open. Current main already fixes the write-side modelOverride clobber for the original sessions_spawn failure, but this PR still carries useful remaining work around subagent-default precedence and defensive fallbacks; the proposed fallback helpers are not merge-ready because they bypass the existing alias-aware model resolver.

## What This Changes

The PR changes subagent model precedence, adds gateway and reply-runtime fallback helpers/tests for subagent default model resolution, and includes small config-validation and qa-lab export adjustments.

## Best Possible Solution

Keep the current-main modelOverride persistence fix, repair or replace this PR so any remaining defensive fallback reuses the alias-aware resolver, and coordinate the subagent-default precedence change with #58822/#58823 plus changelog coverage.

## Reproduction Assessment

Yes. Configure agents.defaults.models with an alias such as gpt, set a subagent default to that alias, and exercise the PR's new reply helper or empty-model gateway fallback; the PR parses it as <defaultProvider>/gpt instead of the configured alias target.

## Solution Assessment

No. The direction is useful, but the current patch is not the best fix because it bypasses the existing alias-aware subagent spawn resolver and leaves overlapping precedence work uncoordinated with #58822/#58823.
