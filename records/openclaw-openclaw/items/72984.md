## Summary

Keep open. The PR targets a real subagent model-resolution bug, but the current head still keeps the alias index from the parent defaults after switching a subagent session to its configured default, so later reset/channel/directive model resolution can still use stale parent-provider aliases.

## What This Changes

The PR updates subagent model resolution in agent, gateway, reply, stored-override, and cron paths, with tests and changelog coverage, so spawned subagent sessions use configured subagent defaults instead of parent primary models.

## Best Possible Solution

Land this fix after the active default provider/model and alias index are updated together, with regression coverage for bare aliases used after a subagent default override.

## Reproduction Assessment

Yes, at source level. The docs define the subagent default contract, and current main still has reply-startup and parent-override paths that can select parent/default models for subagent sessions.

## Solution Assessment

No as-is. The proposed direction is the right narrow fix, but the PR must rebuild or carry the alias index whenever it replaces the active subagent default provider/model.
