## Summary

Keep open: this is a valid subagent model-routing bug fix with sufficient real behavior proof, but the latest PR head still keeps the alias index built from the parent/default model after switching to the subagent default. Reset, channel, and directive model handling can therefore resolve bare aliases under the wrong provider and move a subagent back off its configured default.

## What This Changes

The PR updates subagent model resolution across agent selection, gateway session display, reply startup, stored overrides, cron runtime exports, tests, and changelog so spawned subagents use configured subagent defaults instead of parent primary models.

## Best Possible Solution

Land this after rebuilding or carrying the alias index whenever the active subagent default provider/model changes, with regression coverage for bare aliases used after the subagent default override.

## Reproduction Assessment

Yes, at source level. The docs define the subagent model-default contract, current main still resolves reply defaults before session state and inherits parent model overrides for child sessions, and the PR body supplies live before/after proof from a local gateway run.

## Solution Assessment

No as-is. The proposed direction is the narrow maintainable fix, but the active default provider/model and alias index need to move together before reset, channel, and directive model paths run.
