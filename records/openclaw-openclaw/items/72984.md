- **[P2] Preserve no-plugin alias resolution in subagent defaults:** `src/auto-reply/reply/directive-handling.defaults.ts:66-75`
  - body: `resolveDefaultModel` disables plugin normalization because this path runs on every reply. The new subagent helper builds an alias index and resolves the configured model without that flag, so a configured alias can cold-load plugin/provider normalization on subagent replies. Pass `allowPluginNormalization: false` through both calls here.
  - confidence: 0.88
- **[P2] Keep the rebuilt reply alias index runtime-free:** `src/auto-reply/reply/get-reply.ts:579`
  - body: After switching `defaultProvider` to the subagent provider, this rebuild drops the same no-plugin guard used by the original reply alias index. Subagent replies with configured aliases can therefore reintroduce plugin runtime normalization after the default resolver intentionally avoided it. Include `allowPluginNormalization: false` here too.
  - confidence: 0.86
- **[P2] Honor lightweight session-list normalization:** `src/gateway/session-utils.ts:1767-1776`
  - body: Session list/search callers pass `allowPluginNormalization: false` into `resolveSessionModelRef`, but the new subagent fallback ignores `options` when it builds and resolves aliases. A lightweight session row for a subagent with an empty runtime model can now load plugin-backed normalization despite the caller opting out. Thread `options?.allowPluginNormalization` through these calls.
