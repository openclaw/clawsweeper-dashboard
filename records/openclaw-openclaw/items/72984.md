## Summary

Keep open: the PR targets a real documented subagent model-default bug and has sufficient live proof, but the latest head still lets an inherited parent session override replace the configured subagent default in the reply runtime path.

## What This Changes

The PR changes gateway, reply-runtime, and cron model selection so spawned subagents use configured subagent defaults before falling back to the parent agent primary model.

## Best Possible Solution

Keep the unified precedence, but make reply runtime model selection order explicit: child direct overrides win, configured subagent defaults beat inherited parent overrides, and parent inheritance remains the fallback only when no subagent default applies.

## Reproduction Assessment

Yes, at source level. Current main contradicts the documented subagent default contract, and the remaining PR blocker is visible on the PR head because `resolveStoredModelOverride` runs after the new subagent default and can overwrite it with a parent override.

## Solution Assessment

No, not fully yet. The shared selector and gateway fallback direction are maintainable, but the reply-runtime fallback must be moved after stored override resolution or parent override inheritance must be skipped when a configured subagent default applies.
