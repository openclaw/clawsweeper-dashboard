Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Preserve no-plugin alias resolution on subagent replies:** `src/auto-reply/reply/directive-handling.defaults.ts:65-73`
  - body: `resolveDefaultModel` deliberately disables plugin normalization because this path runs on every reply. The new subagent default helper rebuilds the alias index and resolves the model without that flag, so any configured alias on a subagent turn can load plugin/manifest metadata and undo the hot-path guard. Pass `allowPluginNormalization: false` through both calls here, or thread an explicit option from the caller.
  - confidence: 0.86
- **[P2] Honor lightweight session-list normalization:** `src/gateway/session-utils.ts:1749-1760`
  - body: Session list/search callers pass `allowPluginNormalization: false` into `resolveSessionModelRef`, but this new subagent fallback ignores `options` when it builds the alias index and resolves the configured subagent model. In lightweight session listing, a subagent entry with an empty runtime model and an alias can now load plugin-backed normalization or return a different normalized ref than the caller requested. Pass `options?.allowPluginNormalization` through the new alias-resolution calls.
  - confidence: 0.83
