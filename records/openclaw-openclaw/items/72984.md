## Best Possible Solution

Land a narrow fix that treats `defaultProvider`, `defaultModel`, and `aliasIndex` as one effective defaults bundle for subagent sessions, with focused coverage for aliases and parent override inheritance.

## Reproduction Assessment

Yes, at source level. Current main resolves reply defaults before session state and allows parent stored overrides to apply to spawned subagent children, while the docs define `agents.defaults.subagents.model` as the default when configured.

## Solution Assessment

No, not yet. The direction is the right maintainable fix, but the effective subagent default must carry a matching alias index before reset, channel override, and directive resolution use it.
