Keep open: the PR addresses real subagent runtime model clobbering paths that current main still has, but the latest head still changes the active subagent defaults without rebuilding the alias index, so aliases can continue resolving under the parent provider.

## What This Changes

The PR updates subagent model resolution across gateway session fallback, reply startup, stored parent overrides, isolated cron selection, tests, and changelog text.

## Best Possible Solution

Keep the subagent default and parent-override fixes, but carry the active defaults and alias index together so every later model parser sees the same provider/model context.

## Reproduction Assessment

Yes, at source level. Current main still has reply-startup, gateway fallback, and stored-parent-override paths that can use the parent model for spawned subagent sessions despite the documented subagent default contract.

## Solution Assessment

No as-is. The direction is the right maintainable fix, but the PR must rebuild or replace the alias index after switching to the subagent default before it is safe to merge.
