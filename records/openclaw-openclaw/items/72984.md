## Summary

Keep this PR open: it fixes a real subagent model-resolution gap, but the current patch applies the subagent default after media understanding has already decided based on the parent model, which can drop image-description handling for subagent runs.

## What This Changes

The PR changes subagent model resolution in reply/session fallback paths so spawned subagent sessions use configured subagent defaults, skip parent `/model` inheritance, add regression tests, and add a changelog line.

## Best Possible Solution

Land a version that makes media handling use the same provider/model the subagent run will use, preserves direct and heartbeat overrides, leaves release-owned changelog generation to release tooling, and coordinates remaining precedence semantics with the related PR.

## Reproduction Assessment

Yes, at source level. Current main selects reply defaults before loading the subagent session entry and `resolveSessionModelRef` falls back to the parent agent default when the subagent entry lacks runtime model fields.

## Solution Assessment

No for the current patch shape. The proposed fix addresses a real gap, but the subagent default is applied too late for media understanding; the safer shape makes media decisions see the final subagent run model while preserving explicit overrides.
