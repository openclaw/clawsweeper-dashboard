## Summary

Current main and the shipped v2026.4.29 source already cover the reported session path: default/simple embedded sessions now route supported streamSimple fallbacks through the boundary-aware OpenAI completions transport, that transport sends streaming usage requests, and it preserves usage-only final chunks before skipping empty choices. The broader missing-telemetry fallback in #73990 remains separate and open.

## What This Changes

Request that openai-completions sessions against llama.cpp/OpenAI-compatible endpoints request streaming usage and persist real token counts instead of zero usage.

## Best Possible Solution

Keep the shipped v2026.4.29 implementation for usage-capable OpenAI-compatible streaming endpoints, and leave #73990 to track conservative transcript-derived fallback estimates when a provider genuinely omits usage telemetry.

## Reproduction Assessment

Yes. The original report gives a concrete llama.cpp curl reproduction for OpenClaw 2026.4.27, and current-main verification can be done by inspecting the outbound streaming payload or the focused transport tests; no live endpoint was run in this read-only pass.

## Solution Assessment

Yes. Reusing the existing boundary-aware OpenAI completions transport for default session streams is the narrow maintainable solution because it centralizes stream_options injection and final usage-chunk parsing instead of adding provider-specific session-log repair.
