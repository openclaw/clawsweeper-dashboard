## Summary

Close as implemented on current main. The reported session JSONL path now routes default embedded session streams through OpenClaw's boundary-aware completions transport, which always requests streaming usage and parses usage-only chunks before looking at choices; that implementation is present in `v2026.4.29`.

## What This Changes

The issue asks OpenClaw to record real token usage for no-proxy/no-TLS `openai-completions` sessions against OpenAI-compatible endpoints that return streaming usage when `stream_options.include_usage=true` is sent.

## Best Possible Solution

Keep the shipped session transport behavior. If non-session utility calls such as thread-title generation or capability CLI simple completions need the same unconditional usage request, track that as a separate narrow follow-up rather than keeping this session JSONL report open.

## Reproduction Assessment

Yes. The issue gives concrete session and curl steps, and the current source provides a high-confidence static verification path: a normal embedded session with a default stream now reaches OpenClaw's completions transport, which requests and parses streaming usage.

## Solution Assessment

Yes for the reported session surface. Routing sessions through the existing OpenClaw transport is narrower than adding session-log repair or requiring fake proxy/TLS settings, and it preserves the existing usage parser for usage-only final chunks.
