## Summary

Current main and the v2026.4.29 tagged source already cover the reported no-proxy/no-TLS `openai-completions` session path: embedded agent fallback streams route through OpenClaw's boundary-aware completions transport, that transport always requests streaming usage, and existing tests cover final usage-only chunks. The linked fork PR proposes an extra wrapper for the same request flag, but the shipped core route already solves the observable session-usage problem.

## What This Changes

Request that no-proxy/no-TLS `openai-completions` agent sessions request streaming usage from OpenAI-compatible endpoints and persist non-zero token usage in session JSONL.

## Best Possible Solution

Keep the shipped boundary-aware transport route for normal agent sessions and leave the broader missing-telemetry fallback problem to #73990, which covers endpoints that truly do not return usage.

## Reproduction Assessment

Yes. The report gives a concrete llama.cpp curl reproduction for OpenClaw 2026.4.27, and the source-level route explains how that version could record zero usage; current main/v2026.4.29 should not reproduce that path because it routes through the usage-aware transport.

## Solution Assessment

Yes. The current implementation solves this at the request/stream boundary by reusing the existing OpenClaw completions transport, which is narrower and more maintainable than repairing zero usage after session persistence.
