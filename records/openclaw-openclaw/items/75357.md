## Summary

Close as implemented. Current main routes default embedded OpenAI-compatible session streams through OpenClaw's boundary-aware completions transport, and that transport always requests and parses streaming usage; the same source is present in the shipped v2026.4.29 tag. The open fork PR is useful context, but the target repository already has the needed core behavior for this issue's session path.

## What This Changes

Ensure `openai-completions` sessions against llama.cpp/OpenAI-compatible endpoints request streaming usage and persist real token counts instead of zero usage.

## Best Possible Solution

Keep the shipped boundary-aware embedded session route and leave broader fallback-estimation work, where providers do not return usage at all, to the related telemetry issue rather than adding session-log repair for this usage-capable endpoint.

## Reproduction Assessment

Yes. The old-version reproduction is clear from the reporter's llama.cpp/proxy curl checks, and current-main source verification shows that the no-proxy/no-TLS embedded session path now requests and consumes the same final usage chunk.

## Solution Assessment

Yes. Reusing the existing OpenClaw completions transport for default embedded session streams is the narrow maintainable fix; it avoids fake proxy/TLS configuration and preserves the explicit per-model `compat.supportsUsageInStreaming` workaround as a fallback rather than making it mandatory.
