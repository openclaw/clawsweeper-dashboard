## Summary

Current main and the shipped `v2026.4.29` tag already cover the reported session path: default embedded sessions now resolve supported `streamSimple` OpenAI-compatible models through OpenClaw's boundary-aware completions transport, and that transport always requests streaming usage. The open fork PR is useful context, but its proposed wrapper is redundant for the session JSONL behavior reported here.

## What This Changes

The issue asks OpenClaw to request streaming usage for no-proxy/no-TLS `openai-completions` sessions and persist nonzero token usage in session JSONL.

## Best Possible Solution

Keep the released boundary-aware session transport path and avoid adding a second session-log repair path. If a separate non-session `completeSimple` utility still needs this behavior later, track it as a narrower follow-up instead of reopening this session JSONL report.

## Reproduction Assessment

Yes. The issue includes a concrete llama.cpp curl reproduction for the old release, and current-main source shows the session route now requests streaming usage before persistence; I did not need a live endpoint to verify the fixed routing contract.

## Solution Assessment

Yes. Reusing OpenClaw's existing OpenAI completions transport at the embedded session boundary is the narrow maintainable fix because it preserves normal usage parsing and avoids post-hoc session JSONL patching.
