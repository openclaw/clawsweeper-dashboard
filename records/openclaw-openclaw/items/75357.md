## Summary

Keep open: current main still preserves the reported route gap for generic custom OpenAI-compatible completions providers, and open PR #75837 references this issue but needs review or adjustment before closure.

## What This Changes

The issue reports that simple custom `openai-completions` sessions omit `stream_options.include_usage=true` unless `compat.supportsUsageInStreaming` is manually set, causing zero token usage in session JSONL for llama.cpp-style endpoints.

## Best Possible Solution

Review #75837 against the custom llama.cpp-style reproduction and land either a safe generic `openai-completions` streaming-usage injection or a manifest/API default that covers custom local providers without fake proxy/TLS settings.

## Reproduction Assessment

Yes. The issue gives curl-level endpoint proof, the reporter confirmed the compat override causes `stream_options.include_usage=true` and real token usage, and current source/tests still default generic custom `openai-completions` endpoints to streaming usage off.

## Solution Assessment

Unclear. The manifest seam and explicit workaround are proven, but the open PR should be checked for coverage of custom-named llama.cpp providers rather than only bundled Ollama defaults.
