## Summary

Current main and the shipped v2026.4.29 source already route normal embedded session `streamSimple` fallbacks through the boundary-aware OpenAI-completions transport, which requests streaming usage and records final usage-only chunks. The linked external fork PR is useful context, but the core behavior is already present in the released source.

## What This Changes

The issue requests that no-proxy/no-TLS `openai-completions` sessions request streaming usage from OpenAI-compatible endpoints and persist non-zero token usage in session JSONL.

## Best Possible Solution

Keep the shipped boundary-aware embedded-session transport path, close this report as fixed in v2026.4.29, and leave the broader missing-telemetry fallback work to #73990.

## Reproduction Assessment

Yes. The current-main source path is deterministic: an embedded session with no provider/custom stream falls back to the boundary-aware OpenAI-completions transport, which sends `stream_options.include_usage` and parses final usage-only chunks; no live llama.cpp endpoint was available in this read-only pass.

## Solution Assessment

Yes. Reusing the existing boundary-aware OpenAI-completions transport is the narrow maintainable fix for session JSONL usage, avoiding fake proxy/TLS configuration or downstream session-log repair.
