## Summary

Keep open: current main still has the reported simple `openai-completions` route where generic custom endpoints default streaming usage off unless the user sets the compat override, and the issue is paired with open fixing PRs that need review rather than pre-merge cleanup.

## What This Changes

Reports that custom/local `openai-completions` streaming sessions omit `stream_options.include_usage` unless compat is set, causing session JSONL usage to stay zero.

## Best Possible Solution

Land a reviewed fix that covers custom llama.cpp-style `openai-completions` simple runs without fake proxy/TLS settings, either by safely injecting streaming usage for completions requests or by adding a generic manifest/API path that preserves explicit opt-outs.

## Reproduction Assessment

Yes. The issue provides curl-level proof that the endpoint returns streaming usage only when `stream_options.include_usage=true`, and current main still has source/test coverage showing generic custom simple completions default that compat flag off unless explicitly overridden.

## Solution Assessment

Unclear. The confirmed per-model compat override is a valid workaround and #75837 may improve bundled Ollama defaults, but the reported custom provider-id path likely needs a broader defaulting or request-injection fix.
