## Summary

Keep open: the report is still valid against current main and now has an open in-repo fixing PR (#75837), while the confirmed workaround only helps users who set `compat.supportsUsageInStreaming: true` manually.

## What This Changes

The issue requests real token usage for simple `openai-completions` streaming sessions against llama.cpp/OpenAI-compatible endpoints without requiring per-model compat overrides.

## Best Possible Solution

Review #75837 against the reported custom/local OpenAI-compatible reproduction, and land either that manifest fix or a generic streaming-usage injection that covers custom llama.cpp-style providers without fake proxy/TLS settings.

## Reproduction Assessment

Yes. The issue gives curl-level reproduction steps, the reporter confirmed that `compat.supportsUsageInStreaming: true` adds `stream_options.include_usage=true` and restores token usage, and current source/tests still show generic custom endpoints default that flag off.

## Solution Assessment

Unclear. The workaround is proven but not a product fix, and the open PR needs maintainer review to ensure it solves the custom llama.cpp path rather than only the bundled Ollama manifest default.
