## Summary

Keep open for maintainer review: the patch addresses a source-reproducible gateway bug and I found no concrete code-review blocker, but it intentionally expands node removal into device-role credential revocation and session invalidation, which needs maintainer acceptance before merge.

## What This Changes

The PR updates `node.pair.remove` to remove device-backed node roles, preserve unrelated mixed-role credentials, clear matching legacy node records, add a credential-mutation queue barrier, and document/test the behavior.

## Best Possible Solution

Land this gateway-layer fix after maintainers accept that removing a displayed node should revoke the matching device-backed `node` role while preserving unrelated roles and tokens.

## Reproduction Assessment

Yes, source-reproducible. The reporter's output plus current source show `node.list` can display a device-backed Android node row while `node.pair.remove` only deletes legacy node-pairing records and returns `unknown nodeId`.

## Solution Assessment

Yes, pending maintainer acceptance of the credential semantics. The fix belongs in the gateway node removal handler plus a device-pairing role helper because the bug is a store-boundary mismatch between listing and removal.
