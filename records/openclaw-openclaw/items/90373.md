## Summary

Keep open: this is still the active fix candidate for the Android device-backed node removal bug, and current main plus v2026.6.8 still have the list/remove store mismatch. The patch looks correct, but it intentionally makes `node.pair.remove` revoke device-backed node credentials, so maintainer acceptance of that credential semantics change is still needed before merge.

## What This Changes

The PR updates gateway node removal to remove device-backed node roles, preserve mixed-role devices, clear matching legacy node records, add credential-mutation queue protection, and document/test the new behavior.

## Best Possible Solution

Land this gateway-layer fix after maintainers accept that removing a displayed node should revoke the matching device-backed `node` role while preserving unrelated roles and tokens.

## Reproduction Assessment

Yes, source-reproducible. The linked report plus current source show `node.list` can display a device-backed Android node row while current `node.pair.remove` only deletes legacy node-pairing records and returns `unknown nodeId`.

## Solution Assessment

Yes, pending maintainer acceptance of the credential semantics. The fix belongs in the gateway node removal handler plus a device-pairing role helper because the bug is a store-boundary mismatch between listing and removal.
