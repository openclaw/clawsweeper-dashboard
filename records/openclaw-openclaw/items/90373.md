## Summary

Keep open: this PR is still the active fix candidate for the Android device-backed node removal bug, and current main plus v2026.6.8 still have the list/remove store mismatch. ## What This Changes

The branch expands `node.pair.remove`/`openclaw nodes remove` to revoke device-backed `node` roles, preserve mixed-role devices, clear matching legacy nodes, invalidate/disconnect node sessions, emit redacted security events, add focused tests, and update docs.

## Best Possible Solution

Land this gateway-layer fix once maintainers accept node removal as node-role credential revocation and exact-head checks pass; otherwise split an explicit device-role revocation flow instead of changing `nodes remove` semantics implicitly.

## Reproduction Assessment

## Solution Assessment

Yes, pending maintainer acceptance of the credential semantics. The maintainable fix belongs in the gateway node-removal handler plus a device-pairing role helper, not in a CLI-only fallback or full-device deletion workaround.
