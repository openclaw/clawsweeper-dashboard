## Summary

Keep open for maintainer review: current main and v2026.6.8 still have the Android/device-backed node removal mismatch, and this PR is the active focused fix candidate with sufficient terminal proof and no blocking code-review findings. ## What This Changes

The branch changes gateway node removal to revoke device-backed `node` roles, preserve mixed-role device credentials, clear matching legacy node records, add a credential-mutation queue barrier, and update docs/tests.

## Best Possible Solution

Land this gateway-layer fix after maintainers explicitly accept that removing a displayed node should revoke the matching device-backed `node` role while preserving unrelated roles and tokens.

## Reproduction Assessment

Yes, source-reproducible. The reporter's output plus current source show `node.list` can display a device-backed Android node row while `node.pair.remove` only deletes legacy node-pairing records and returns `unknown nodeId`.

## Solution Assessment

Yes, pending maintainer acceptance of the credential semantics. The fix belongs in the gateway node removal handler plus a device-pairing role helper because the bug is a store-boundary mismatch between listing and removal.
