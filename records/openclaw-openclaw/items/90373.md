## Summary

Keep open for maintainer review: the PR is a focused and well-proven fix for the linked Android device-backed node removal bug, but it intentionally expands `node.pair.remove` into persisted device-role credential revocation and node-role session invalidation, so that command-boundary decision should remain explicit before merge.

## What This Changes

The PR changes gateway node removal to revoke device-backed `node` roles from `devices/paired.json`, preserve mixed-role device rows, clear matching legacy node rows/runtime state, add credential invalidation and security-event coverage, and update node-removal docs.

## Best Possible Solution

Land this gateway-layer fix after maintainers accept node removal as node-role credential revocation and current-head checks pass; otherwise split an explicit device-role revocation flow instead of changing `nodes remove` semantics implicitly.

## Reproduction Assessment

Yes. Current main lists device-backed nodes from device pairing while `node.pair.remove` deletes only legacy node-pairing records, matching the linked Android report's `unknown nodeId` failure.

## Solution Assessment

Yes, pending maintainer acceptance of the credential semantics. The maintainable fix belongs in the gateway node-removal handler plus a device-pairing role helper, not in a CLI-only fallback or full-device deletion workaround.
