## Best Possible Solution

## Reproduction Assessment

Yes. Source inspection shows current main lists device-backed node-role pairings but `node.pair.remove` only deletes legacy node-pairing records, matching the linked `unknown nodeId` failure for Android rows.

## Solution Assessment

Yes, with one docs gap. Removing only the `node` role in the device-pairing owner is the right boundary because `node.list` already treats node-role device pairings as node rows, but the public node-removal docs need to be updated with that new behavior.
