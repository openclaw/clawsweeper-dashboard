## Summary

Keep open: this PR is the current focused fix candidate for the Android device-backed node removal bug, and I found no discrete blocking patch defect. ## What This Changes

The PR changes gateway node removal to revoke device-backed `node` roles, preserve mixed-role device rows, clear matching legacy node rows/runtime state, add credential-invalidation/security-event coverage, and update node pairing docs.

## Best Possible Solution

Land this gateway-layer fix after maintainers accept node removal as device-role credential revocation and current-head merge checks pass; otherwise split an explicit device-role revocation path before changing `nodes remove` semantics.

## Reproduction Assessment

## Solution Assessment

Yes, pending maintainer acceptance of the credential semantics. The maintainable fix belongs in the gateway node-removal handler plus a device-pairing role helper, not in a CLI-only fallback or full-device deletion workaround.
