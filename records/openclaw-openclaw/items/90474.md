## What This Changes

The PR adds a guarded legacy install-index migration path that archives older legacy npm records when newer current SQLite install metadata proves the same package superseded them.

## Best Possible Solution

Land this as the guarded older-legacy supersession rule if maintainers accept that the current SQLite npm record is authoritative under the package/version/install-path guard, while coordinating separately with the same-version artifact PR if desired.

## Reproduction Assessment

## Solution Assessment

Yes, with maintainer acceptance of the compatibility tradeoff. The patch is in the state-migration owner and is narrower than archive-all or timestamp-only alternatives because it requires package identity, a managed current install path, and a strictly newer current version.
