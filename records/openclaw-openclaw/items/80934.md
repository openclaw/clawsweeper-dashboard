## Summary

Keep open: the narrow direction is useful, but enabling the Claude opt-in currently exposes a combined missing-transcript plus auth-change path that can raw-reseed across an auth boundary, and the supplied proof does not exercise live Claude session-expired recovery.

## What This Changes

The PR enables bounded raw transcript reseed for the bundled Anthropic Claude CLI backend and adds a regression asserting native resume remains configured.

## Best Possible Solution

Preserve auth-profile and auth-epoch no-raw semantics before enabling the Claude backend opt-in, add focused regression coverage, then land after live Claude CLI session-expired proof or maintainer proof override.

## Reproduction Assessment

Yes, from source: current main lacks the Claude backend opt-in while the shared prepare path only reseeds session-expired retries when that boolean is true. I did not establish a live Claude CLI rotation in this read-only review.

## Solution Assessment

No as currently patched. The backend opt-in is the right owner-boundary shape, but auth-profile and auth-epoch invalidation must win over missing-transcript classification before raw history is allowed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve auth boundaries before enabling raw reseed:** `extensions/anthropic/cli-backend.ts:66`
  - body: This opt-in applies to every allowed raw-reseed reason, including `missing-transcript`. In current prepare, a missing Claude transcript is classified before `resolveCliSessionReuse()` can return `auth-profile` or `auth-epoch`, so a credential/profile change plus missing Claude transcript can replay prior raw OpenClaw history into a fresh Claude run. Resolve auth reuse first or suppress raw reseed in that combined case before enabling the bundled backend.
  - confidence: 0.86
