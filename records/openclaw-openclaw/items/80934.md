## Summary

Keep open: the intended Claude `session_expired` recovery is a narrow use of existing reseed machinery, but this backend-wide opt-in also enables raw missing-transcript reseed where current prepare ordering can bypass auth-boundary invalidation. The submitted proof is useful test output, but it does not show the changed Claude recovery behavior in a real run.

## What This Changes

The PR adds `reseedFromRawTranscriptWhenUncompacted: true` to the bundled Anthropic Claude CLI backend and adds an extension test asserting the opt-in preserves native resume args.

## Best Possible Solution

Enable the bundled Claude reseed opt-in only after prepare resolves auth-profile and credential-epoch invalidations before any raw missing-transcript reseed, with regression coverage for the combined auth-change plus missing-transcript case and the intended `session_expired` fresh retry.

## Reproduction Assessment

No high-confidence live reproduction was run for an authenticated Claude CLI rotation. Source inspection and existing tests do show the intended `session_expired` retry path and the auth-boundary precedence problem introduced by enabling the backend opt-in.

## Solution Assessment

No, not as patched. The one-line backend opt-in is the narrow mechanism for `session_expired`, but it also activates raw missing-transcript reseed before auth-boundary classification is protected.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve auth boundaries before enabling raw Claude reseed:** `extensions/anthropic/cli-backend.ts:66`
  - body: This opt-in enables raw reseed for all allowed invalidations, including `missing-transcript`. In current `prepareCliRunContext`, Claude transcript-missing is chosen before `resolveCliSessionReuse` can return `auth-profile` or `auth-epoch`, so a missing Claude transcript plus changed auth identity can send raw OpenClaw history into a fresh Claude run under the new credentials. Resolve auth invalidation first or suppress raw reseed for that combined case before turning the bundled backend on.
  - confidence: 0.86
