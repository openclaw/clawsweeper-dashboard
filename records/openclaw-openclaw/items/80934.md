## Summary

Keep open: the one-line Anthropic opt-in is not on current main and is the right owner boundary for the reported Claude continuity gap, but the patch should not merge while it activates a raw-reseed path that can bypass auth-boundary invalidation and still lacks live behavior proof.

## What This Changes

The PR enables raw transcript reseed for the bundled Anthropic Claude CLI backend and adds a regression asserting native resume remains configured.

## Best Possible Solution

Keep the owner-plugin opt-in shape, but make auth-profile and auth-epoch invalidations win before missing-transcript raw reseed, add focused prepare-level coverage, then land with live Claude proof or an explicit proof override.

## Reproduction Assessment

Yes, from source: current main's Anthropic backend lacks the opt-in, and the shared runner only prepares raw reseed history when that backend flag is true. I did not establish a live Claude CLI rotation in this read-only review.

## Solution Assessment

No as currently patched. The owner-plugin opt-in is narrow and maintainable, but it needs an auth-boundary precedence guard before enabling raw transcript reseed for the default Claude backend.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Guard auth resets before enabling Claude raw reseed:** `extensions/anthropic/cli-backend.ts:66`
  - body: Enabling raw reseed for the bundled Claude backend makes the `missing-transcript` path active, but `prepareCliRunContext` chooses that reason before `resolveCliSessionReuse` can report an `auth-profile` or `auth-epoch` invalidation. If the Claude transcript is missing after the auth identity changes, this can send prior raw OpenClaw transcript history into a fresh Claude run under the new credentials, contradicting the documented no-raw auth boundary. Please make auth-boundary invalidations win before raw missing-transcript reseed is allowed and add coverage for that combined case.
  - confidence: 0.86
