## Summary

Keep open. Current main and the latest release preserve Slack ID mention tokens but do not resolve or rewrite bare `@name` text before posting, so the reported user-visible behavior remains and needs a Slack-specific product decision rather than cleanup closure.

## What This Changes

The issue requests Slack outbound responses to use Slack mention syntax such as `<@USER_ID>` instead of plain `@name` text.

## Best Possible Solution

Keep existing `<@USER_ID>` preservation and add a Slack-local policy for either stronger agent guidance or a constrained resolver that only mentions unambiguous known users.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path for the reported symptom: bare `@Emre` text passes through outbound Slack formatting unchanged while existing `<@...>` tokens are preserved. I did not run a live Slack workspace reproduction in this read-only review.

## Solution Assessment

Unclear. A blanket display-name auto-conversion is not the best narrow fix because Slack mentions are ID-based and display names can be ambiguous; prompt/context improvements or a constrained resolver are safer.
