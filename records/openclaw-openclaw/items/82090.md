## Summary

This should stay open: current Slack outbound formatting preserves existing `<@USER_ID>` tokens but does not rewrite plain `@name` text into Slack mention tokens, and there is no direct duplicate or implemented fix on current main.

## What This Changes

The issue requests Slack bot responses to turn plain `@name` references into Slack mention markup so users are notified and mentions are clickable.

## Best Possible Solution

Add a Slack-owned outbound mention policy, likely mirroring Discord’s configured alias rewrite while preserving existing `<@USER_ID>` tokens and documenting how users obtain stable Slack IDs.

## Reproduction Assessment

Yes for the observable source path: `@Emre` is plain text to the Slack formatter and is posted as text unless the model already emitted `<@USER_ID>`. I did not run a live Slack workspace reproduction.

## Solution Assessment

Unclear: automatically resolving every bare `@name` at send time could be ambiguous and expensive, while configured aliases are deterministic but add config surface. Maintainers should choose the Slack plugin policy before implementation.
