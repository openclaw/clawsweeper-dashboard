## Summary

Keep open. Current main still preserves explicit Slack mention tokens but leaves bare output such as `@Emre` unchanged, and the safe fix needs a Slack-local policy decision rather than automatic cleanup closure.

## What This Changes

The issue asks the Slack plugin to send user mentions as Slack mention tokens instead of plain `@name` text.

## Best Possible Solution

Keep explicit `<@USER_ID>` preservation and choose a Slack-local policy for either stronger agent guidance or a constrained resolver that only mentions unambiguous known users.

## Reproduction Assessment

Yes, at source level: `@Emre` has no Slack angle token, so the formatter returns it unchanged and the send path posts it without a user lookup or automatic mention parsing. I did not run a live Slack workspace reproduction.

## Solution Assessment

No, blanket automatic conversion is not clearly the best solution because Slack names are ambiguous and automatic parsing can create unwanted notifications. A safer fix should use explicit Slack IDs from context or a constrained Slack-plugin resolver.
