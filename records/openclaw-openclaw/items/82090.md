## Summary

Keep open. Current main preserves Slack `<@...>` mention tokens but sends plain `@Emre` text unchanged, and Slack’s developer docs favor ID-based mention syntax while warning against broad automatic parsing. ([docs.slack.dev](https://docs.slack.dev/messaging/formatting-message-text))

## What This Changes

The issue requests Slack replies to render user mentions as Slack mention tokens instead of plain @display-name text.

## Best Possible Solution

Keep Slack’s existing `<@USER_ID>` preservation and decide a Slack-local policy for either stronger agent guidance or a constrained resolver that only mentions unambiguous known users.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: `sendMessageSlack` sends `@Emre` through Slack mrkdwn conversion unchanged because only `<@...>` tokens are recognized or preserved. I did not run a live Slack workspace reproduction in this read-only review.

## Solution Assessment

No blanket auto-conversion is not clearly the best fix because Slack display names are ambiguous and automatic parsing can create unwanted notifications. A safer solution should use explicit Slack user IDs from context or a constrained Slack-plugin resolver.
