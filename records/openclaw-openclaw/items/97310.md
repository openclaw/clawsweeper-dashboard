## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep the shared Feishu chunker raw:** `extensions/feishu/src/outbound.ts:517`
  - body: `feishuOutbound.chunker` is used by core delivery planning before `sendText` decides whether text should become a card or a post. Pointing it at the post newline normalizer means table markdown is rewritten before the card renderer sees it, so table replies can lose their intended card/table formatting; keep this shared chunker raw and normalize only in the post delivery path after routing.
  - confidence: 0.87
- **[P2] Track the opening fence length before toggling:** `extensions/feishu/src/send.ts:588-590`
  - body: The normalizer stores only the fence character, so a three-backtick line inside a four-backtick fence closes the fence early even though Markdown requires a closing fence to be at least as long as the opener. That lets later code-block lines receive added blank lines; store the opening delimiter length and close only on a matching marker of sufficient length.
  - confidence: 0.82
