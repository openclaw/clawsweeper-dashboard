## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Keep post normalization after table/card routing:** `extensions/feishu/src/outbound.ts:93-94`
  - body: `sendTextMediaPayload` calls the adapter chunker before `sendText`, so this normalizes raw markdown before the Feishu path can detect tables and choose the card renderer. A normal table like `| A | B |\n|---|---|` gets blank lines inserted before `shouldUseCard` or `convertMarkdownTables` sees it, which can silently turn existing table replies into altered post text; normalize only after the post-vs-card decision or use a post-only rechunking path.
  - confidence: 0.89
- **[P2] Track the opening fence length before toggling:** `extensions/feishu/src/send.ts:586-590`
  - body: The normalizer stores only the fence character, so a three-backtick line inside a four-backtick fence closes the fence early even though Markdown requires the closing fence to be at least as long as the opener. That lets later code-block lines receive added blank lines; store the opening delimiter length and only close on a matching marker of sufficient length.
  - confidence: 0.82
