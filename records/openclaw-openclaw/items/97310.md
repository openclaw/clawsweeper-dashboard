Full review comments:

- **[P1] Keep post normalization after table/card routing:** `extensions/feishu/src/outbound.ts:93-94`
  - body: `sendTextMediaPayload` chunks before calling the adapter `sendText`, so this helper now inserts blank lines before Feishu can detect markdown tables and choose the card path. A table like `| A | B |\n|---|---|` can lose the adjacent header/separator pattern and be sent as altered post text; normalize only after post-vs-card routing or use a post-only rechunking path.
  - confidence: 0.89
- **[P2] Track the opening fence length before toggling:** `extensions/feishu/src/send.ts:586-590`
  - body: The normalizer stores only the fence character, so a three-backtick line inside a four-backtick fence closes the fence early even though Markdown requires a closing fence at least as long as the opener. Store the opening delimiter length and only close on a matching marker of sufficient length.
  - confidence: 0.82
