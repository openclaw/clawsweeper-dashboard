## Summary

Keep open. Current main has adjacent transcript indexing, Active Memory timeout partial recall, timeout compaction, and `/new`/`/reset` session-memory capture, but it still lacks repeated model-timeout pending session capture followed by later recovery into daily memory; that persistence path needs maintainer design before implementation.

## What This Changes

The issue requests temporary session persistence and later memory recovery when repeated model timeouts prevent normal memory maintenance.

## Best Possible Solution

Design a generic timeout-recovery lifecycle that stores bounded private pending transcript artifacts and drains them idempotently into memory with dedupe, retention, cleanup, docs, and regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: repeated model timeout or memory-flush failure reaches current timeout/error handling without a pending memory artifact, while existing session-memory capture only runs for `/new` or `/reset`.

## Solution Assessment

No, not as a mechanical one-file fix. The proposed `memory/pending` direction is plausible, but the maintainable solution needs cross-runner and memory-plugin persistence semantics rather than ad hoc transcript writes.
