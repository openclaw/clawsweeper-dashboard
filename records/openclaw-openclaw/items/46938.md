## Summary

Keep open. Current main has adjacent session transcript indexing, Active Memory timeout partial recall, timeout compaction, and `/new`/`/reset` session-memory handling, but it still does not implement repeated model-timeout pending session capture followed by later recovery into daily memory.

## What This Changes

Request to add timeout-triggered pending session storage and later recovery into memory when repeated model timeouts prevent normal memory maintenance.

## Best Possible Solution

Design a generic timeout-recovery lifecycle that stores bounded private pending transcript artifacts and drains them idempotently into memory with dedupe, retention, cleanup, docs, and regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: repeated model timeout or memory-flush failure reaches blocked/error handling or a logged flush failure, with no pending memory artifact or later recovery job in current main.

## Solution Assessment

No, not as a mechanical one-file fix. The proposed `memory/pending` direction is plausible, but the maintainable solution needs a cross-runner and memory-plugin persistence policy rather than ad hoc transcript writes.
