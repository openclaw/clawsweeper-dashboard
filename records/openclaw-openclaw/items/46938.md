## Summary

Keep open. Current main has adjacent session transcript indexing, Active Memory timeout partial recall, and `/new`/`/reset` session-memory capture, but it still lacks repeated model-timeout pending-session capture followed by later recovery into daily memory; that persistence path needs maintainer design before implementation.

## What This Changes

The issue requests staging session contents after repeated model timeouts and recovering them into memory after the model becomes available.

## Best Possible Solution

Design a generic timeout-recovery lifecycle that stores bounded private pending transcript artifacts and drains them idempotently into memory with dedupe, retention, cleanup, docs, and regression coverage.

## Reproduction Assessment

Yes, by source inspection rather than a live timeout run. Memory flush failures are caught and reported without a pending artifact, while session-memory only writes for `/new` or `/reset`, which is enough to show current main lacks the requested recovery path.

## Solution Assessment

No, not as a mechanical one-file fix. The proposed `memory/pending` direction is plausible, but the durable fix should go through the memory/session contracts with retention, privacy, dedupe, cleanup, and docs rather than ad hoc transcript writes.
