## Summary

Keep this PR open for maintainer review: current main still has observation-only `llm_input`, `llm_output`, and `after_tool_call` hooks, but this patch is not merge-ready because it combines mutable LLM hooks with adapter-side tool-result mutation and leaves security and dispatch-contract gaps.

## What This Changes

The PR changes plugin hook contracts, embedded runner call sites, adapter-side tool execution, and hook tests so `llm_input`, `llm_output`, and `after_tool_call` can return blocking or payload-modifying results.

## Best Possible Solution

Ship a focused, hardened mutable `llm_input`/`llm_output` contract, then handle pre-model tool-result mutation as a separate plugin API design with exact-once dispatch, validation, docs, changelog, and regression coverage.

## Reproduction Assessment

Yes for the review findings: static inspection of current main plus the PR diff shows the hooks are still void today and the proposed mutation paths bypass prompt-policy, derived prompt artifacts, output cleanup, and exact-once tool dispatch guarantees. I did not run tests because this task is read-only.

## Solution Assessment

No. The direction is useful, but this PR is broader than the safer implementation path because it couples LLM prompt/output mutation with adapter-side `after_tool_call` result mutation before the policy and dispatch contracts are settled.

## Security Review

Status: needs_attention

Summary: The diff introduces prompt and response mutation surfaces without fully carrying existing prompt-injection and redaction safeguards forward.

Concerns:

- **[high] Mutable llm_input bypasses prompt-injection policy:** `src/plugins/types.ts:1563`
  - body: `llm_input` gains `prompt` and `systemPrompt` result fields, but current policy only treats the existing prompt-building hooks as prompt-injection surfaces. Without matching registry constraints, a plugin configured with `allowPromptInjection=false` can still rewrite model instructions.
  - confidence: 0.9
- **[high] Prompt override can retain stale prompt artifacts:** `src/agents/pi-embedded-runner/run/attempt.ts:2840`
  - body: The PR applies prompt overrides after prompt-local image detection and related prompt state are computed. For a redaction plugin, stale image artifacts can still be sent unless the implementation moves the hook earlier or recomputes all derived prompt payload state.
  - confidence: 0.9
- **[medium] Output override can leave original assistant text reachable:** `src/agents/pi-embedded-runner/run/attempt.ts:3177`
  - body: The proposed `llm_output` override changes `assistantTexts` but leaves the original `lastAssistant` object and message snapshots intact, so sensitive text that a plugin intended to redact can remain available to downstream handling.
  - confidence: 0.78
