## Summary

Keep this PR open, but do not merge it as-is: the core LLM-hook direction is useful, yet the diff still has security and correctness blockers, and the `llm_input`/`llm_output` subset now overlaps the narrower open PR #61485.

## What This Changes

The PR changes plugin hook contracts and embedded/tool call sites so `llm_input`, `llm_output`, and adapter-side `after_tool_call` can block or modify prompt, assistant-response, and tool-result payloads.

## Best Possible Solution

Ship a focused, hardened mutable `llm_input`/`llm_output` contract and split pre-model tool-result mutation into a separate plugin API design with exact-once dispatch, result validation, docs, changelog, and regression tests.

## Reproduction Assessment

Yes. Static inspection of current main shows the hooks are still void observation hooks, and PR diff/context inspection gives high-confidence paths for stale prompt images, prompt-policy bypass, output replacement leakage, and duplicate `after_tool_call` dispatch.

## Solution Assessment

No. The direction is useful, but this PR combines LLM prompt/output mutation with adapter-side tool-result mutation before policy enforcement, derived prompt state, output cleanup, and dispatch ownership are resolved.

## Security Review

Status: needs_attention

Summary: The diff introduces prompt and response mutation surfaces without fully carrying existing prompt-injection and redaction safeguards forward.

Concerns:

- **[high] Mutable llm_input bypasses prompt-injection policy:** `src/plugins/types.ts:1563`
  - body: `llm_input` gains `prompt` and `systemPrompt` result fields, but current policy only treats the existing prompt-building hooks as prompt-injection surfaces. Without matching registry constraints, a plugin configured with `allowPromptInjection=false` can still rewrite model instructions.
  - confidence: 0.9
- **[high] Prompt override can retain stale prompt artifacts:** `src/agents/pi-embedded-runner/run/attempt.ts:2840`
  - body: The PR applies prompt overrides after prompt-local image detection. For a redaction plugin, stale image artifacts can still be sent unless the implementation moves the hook earlier or recomputes all derived prompt payload state.
  - confidence: 0.9
- **[medium] Output override can leave original assistant text reachable:** `src/agents/pi-embedded-runner/run/attempt.ts:3177`
  - body: The proposed `llm_output` override changes `assistantTexts` but leaves the original `lastAssistant` object and message snapshots intact, so sensitive text that a plugin intended to redact can remain available to downstream handling.
  - confidence: 0.78
