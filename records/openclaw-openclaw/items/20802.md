## Summary

Keep this PR open for maintainer/API/security review. Current main still has observe-only `llm_input`, `llm_output`, and `after_tool_call` hooks, but this branch is merge-conflicting, still has concrete correctness/security blockers, lacks real behavior proof for this PR, and overlaps with the focused mutable LLM-hook work in https://github.com/openclaw/openclaw/pull/61485.

## What This Changes

The PR changes plugin hook contracts, hook runners, embedded agent handling, adapter-side tool execution, and hook tests so `llm_input`, `llm_output`, and `after_tool_call` can return blocking or payload-modifying results.

## Best Possible Solution

Land or supersede this with a focused, hardened mutable `llm_input`/`llm_output` contract, then handle pre-model tool-result mutation as a separate plugin API design with exact-once dispatch, result validation, docs, changelog, and live proof.

## Reproduction Assessment

Yes, for the review findings by source inspection: the PR head applies prompt rewrites after image detection, uses truthy override checks, leaves original assistant data reachable after output replacement, and changes `after_tool_call` dispatch ownership against current-main guards.

## Solution Assessment

No. The feature direction is useful, but this PR bundles mutable LLM hooks with adapter-side tool-result mutation before prompt-policy, prompt-artifact, result-validation, exact-once dispatch, docs/changelog, conflict resolution, and live proof concerns are settled.

## Security Review

Status: needs_attention

Summary: The diff grants installed plugins new authority over model prompts, system prompts, responses, and tool results without carrying existing policy and redaction guarantees forward.

Concerns:

- **[high] Mutable llm_input bypasses prompt policy:** `src/plugins/types.ts:1563`
  - body: `llm_input` gains `prompt` and `systemPrompt` result fields, but the prompt-injection policy is not extended to this new mutation path, so `allowPromptInjection=false` would not prevent these rewrites.
  - confidence: 0.9
- **[high] Prompt override can retain stale image artifacts:** `src/agents/pi-embedded-runner/run/attempt.ts:2840`
  - body: Prompt rewrites happen after prompt-local image detection, so a redaction plugin can change the prompt while images from the original prompt still go to the model.
  - confidence: 0.91
- **[medium] Output override can leave original assistant data reachable:** `src/agents/pi-embedded-runner/run/attempt.ts:3197`
  - body: The PR replaces `assistantTexts` but returns the original `lastAssistant`, which can preserve content, reasoning, or error metadata that a redaction plugin intended to remove.
  - confidence: 0.84
- **[medium] Tool-result replacement lacks result validation:** `src/agents/pi-tool-definition-adapter.ts:185`
  - body: `after_tool_call` can return an arbitrary `unknown` value that is cast to `AgentToolResult`, so malformed plugin output can enter the model-facing tool result path without a boundary check.
  - confidence: 0.75
