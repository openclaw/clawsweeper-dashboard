## Summary

Keep this PR open for maintainer review: current main still lacks modifying `llm_input`/`llm_output`, but this branch is broader than the focused LLM-hook request and has concrete prompt-policy, redaction, and `after_tool_call` ownership blockers.

## What This Changes

This PR changes plugin hook types/runners and embedded agent/tool paths so `llm_input`, `llm_output`, and adapter-side `after_tool_call` can return blocking or payload-modifying results.

## Best Possible Solution

Land or supersede this with a focused, hardened mutable `llm_input`/`llm_output` contract, then handle pre-model tool-result mutation as a separate plugin API design with exact-once dispatch, validation, docs, changelog, and live proof.

## Reproduction Assessment

Yes for the review findings by source inspection: the branch applies `llm_input` rewrites after prompt image detection, adds prompt/systemPrompt result fields without prompt-policy enforcement, and changes `after_tool_call` dispatch ownership while an exact-once regression test remains on the old runner shape.

## Solution Assessment

No. The direction is useful, but the current PR couples a public mutable LLM-hook contract with adapter-side tool-result mutation before prompt policy, redaction cleanup, result validation, and dispatch ownership are settled.

Full review comments:

- **[P1] Recompute prompt artifacts after llm_input rewrites:** `src/agents/pi-embedded-runner/run/attempt.ts:2840`
  - body: `llm_input` prompt overrides are applied after `detectAndLoadPromptImages` has already built `imageResult`, but the later model call still sends `imageResult.images`. If a plugin removes or changes prompt-local image references, stale images from the original prompt can still reach the model; move the hook earlier or recompute and roll back all derived prompt state after applying the override.
  - confidence: 0.9

  - body: The new result type lets `llm_input` handlers return `prompt` and `systemPrompt`, but the registry only constrains the existing prompt-injection hook names when `allowPromptInjection=false`. A plugin without prompt-injection permission can still rewrite model instructions through this new path.
  - confidence: 0.9

  - body: When `llm_output` replaces `assistantTexts`, the returned attempt still exposes the original `lastAssistant`. That can leave unredacted assistant text, reasoning, or error metadata reachable after a plugin requested response replacement.
  - confidence: 0.82

  - confidence: 0.78
