The PR adds Linux systemd gateway-scope detection for canonical and marker-owned system units, then routes status, stop, and restart through the detected unit name with focused tests.

## Best Possible Solution

Land this PR after maintainers accept the Linux system-scope control semantics; keep the canonical and marker-owned detection while avoiding unmanaged signals for systemd-owned gateways.
