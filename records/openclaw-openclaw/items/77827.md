## Summary

Keep open: this is a valid focused bug fix with sufficient real LM Studio proof, but the current PR head still drops active Responses reasoning-item state before `response.output_item.done` can attach replay metadata.

## What This Changes

Adds `response.reasoning_text.done` handling and regression tests in the OpenAI Responses stream parser so LM Studio done-only reasoning text becomes a thinking block.

## Best Possible Solution

Merge a refreshed branch that supports LM Studio done-only reasoning text while preserving replayable reasoning-item signatures for normal Responses streams.

## Reproduction Assessment

Yes. Current main has no `response.reasoning_text.done` branch, the upstream event is documented, and the PR includes inspected before/after LM Studio screenshots showing the missing and restored thinking block.

## Solution Assessment

No, not yet. The direction is right, but the standard Responses path should keep the active reasoning block alive until `response.output_item.done` attaches `thinkingSignature`, or attach equivalent metadata before clearing state.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep reasoning item state until output_item.done:** `src/agents/openai-transport-stream.ts:507`
