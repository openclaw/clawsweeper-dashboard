Close as duplicate/superseded: the requested group-to-DM, per-identity shared-session behavior is still missing on current main, but the remaining product/security design is already covered by active shared-session and identity-routing trackers.

## What This Changes

The issue requests an opt-in way to share live agent session context between group chats and DMs for the same user identity while keeping other group participants isolated.

## Best Possible Solution

Track the remaining work in the canonical shared-session and identity-routing issues, with isolated defaults, explicit trust boundaries, origin-preserving delivery, and per-user group filtering.

## Reproduction Assessment

Yes at source level: current routing only applies direct-message scoping and identity links to direct peers, while group/channel peers always receive isolated channel/group session keys. This is a feature gap rather than a broken existing contract.

## Solution Assessment

No as a separate tracker: the best path is to fold this use case into the active shared-session and identity-routing design work instead of adding another competing session-routing config surface.
