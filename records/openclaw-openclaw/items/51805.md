## Summary

Keep open: current main and the latest release still isolate group/channel sessions from direct-message identity sessions, and the related trackers/closed PRs do not implement this sender-filtered group-to-DM identity merge.

## What This Changes

Requested an opt-in session-routing mode that lets a linked user continue one agent session across group chats and DMs while excluding unrelated group participants.

## Best Possible Solution

Design an additive opt-in core session-routing contract that maps authorized group senders and direct peers to a canonical identity session, filters unrelated group participants, preserves replies to the originating chat, and updates schemas, docs, and representative routing/channel tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/config request. Source, docs, tests, and latest-release inspection give high confidence that current main still isolates non-direct peers and only applies identityLinks to direct-message scopes.

## Solution Assessment

Unclear. The proposed examples are useful directionally, but the maintainable solution needs maintainer-approved privacy-filtered identity merging plus runtime, reply-routing, queueing, and config semantics before implementation.
