## Summary

Keep open. Current main still isolates group/channel session keys and applies identity linking only to direct-message scopes, while the related shared-session trackers do not cover this issue's per-identity group-to-DM filtering and privacy requirement.

## What This Changes

Request an opt-in session configuration that shares context between a linked sender's group/channel turns and DMs while excluding other group participants.

## Best Possible Solution

Design this as an additive, opt-in core session-routing contract that maps authorized group/channel senders and direct peers to a canonical identity session, filters out other participants, preserves replies to the originating chat, and updates config schema, docs, and focused routing tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main still keeps group/channel keys separate and applies identityLinks only to direct-message scopes.

## Solution Assessment

Unclear. The proposed config examples are directionally useful, but the best maintainable solution needs an explicit privacy-filtered identity merge model and runtime/reply-routing safeguards coordinated with the existing shared-session trackers.
