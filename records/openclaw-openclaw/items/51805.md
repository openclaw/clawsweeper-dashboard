## Summary

Keep open. Current main still applies identity linking only to direct-message session scopes and keeps group/channel sessions isolated, while this request adds a new privacy-sensitive group-to-DM identity merge policy that related shared-session trackers do not fully cover.

## What This Changes

The issue requests an opt-in config surface for sharing conversation context between group chats and DMs for the same linked identity while excluding other group participants.

## Best Possible Solution

Design an additive, opt-in core session-routing contract that maps authorized group/channel sender turns and direct peers to a canonical identity session, filters other participants, preserves replies to the originating chat, and updates config schema, docs, and routing tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main still separates group/channel session keys from DM identity-linked session keys.

## Solution Assessment

Unclear as a final implementation. The proposed config examples are useful directionally, but the best solution needs maintainer design for privacy-filtered identity merging, reply routing, and runtime-boundary semantics.
