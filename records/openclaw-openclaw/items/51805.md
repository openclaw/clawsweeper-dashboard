## Summary

Keep open. Current main and v2026.5.18 still route group/channel peers to isolated non-DM session keys, and the related shared-session issues and open groupScope PR do not cover this issue's per-identity group-to-DM filtering and privacy model.

## What This Changes

Request an opt-in session-routing/config option that shares context between DMs and group chats for the same identity while filtering unrelated group participants.

## Best Possible Solution

Design an additive opt-in core session-routing contract that maps authorized group senders and direct peers to a canonical identity session, filters unrelated group participants, preserves replies to the originating chat, and updates schemas, docs, and routing/channel tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, tests, and release inspection give high confidence that current main still isolates non-direct peers and only applies identityLinks to direct-message scopes.

## Solution Assessment

Unclear. The proposed config examples are useful directionally, but the maintainable solution needs maintainer-approved privacy-filtered identity merging plus runtime and reply-routing safeguards before implementation.
