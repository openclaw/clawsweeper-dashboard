## Summary

Keep open: current main still isolates group/channel sessions from direct-message identity sessions, and the related shared-session trackers and groupScope PR do not cover this issue's sender-filtered group-to-DM privacy model.

## What This Changes

Request an opt-in way for a user's group-chat turns and DMs to share one session while excluding unrelated group participants.

## Best Possible Solution

Design an additive opt-in core session-routing contract that maps authorized group senders and direct peers to a canonical identity session, filters unrelated group participants, preserves replies to the originating chat, and updates schemas, docs, and representative routing/channel tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, and tests give high confidence that current main still isolates non-direct peers and only applies identityLinks to direct-message scopes.

## Solution Assessment

Unclear. The proposed examples are useful directionally, but the maintainable solution needs maintainer-approved privacy-filtered identity merging plus runtime, reply-routing, and config semantics before implementation.
