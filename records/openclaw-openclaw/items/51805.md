## Summary

Keep open: current main still isolates group/channel sessions from direct-message identity sessions, and the adjacent open trackers/PR cover group-to-main or channel grouping but not this issue's sender-filtered group-to-DM identity merge.

## What This Changes

The issue requests an opt-in way to share session context between group chats and direct messages for the same user identity while filtering unrelated group participants.

## Best Possible Solution

Design an additive opt-in core session-routing contract that maps authorized group senders and direct peers to a canonical identity session, filters unrelated group participants, preserves replies to the originating chat, and updates schemas, docs, and representative routing/channel tests.

Not applicable as a bug reproduction; this is a feature/config request. Source, docs, and tests give high confidence that current main still isolates non-direct peers and only applies identityLinks to direct-message scopes.
