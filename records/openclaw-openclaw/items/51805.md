## Summary

Keep open. Current main still isolates group/channel session keys and only applies `identityLinks` to direct-message scopes, while the related shared-session issues and open groupScope PR do not yet solve this issue's per-identity group-to-DM filtering and privacy model.

## What This Changes

Request an opt-in session merge mode so a user's group-chat turns and direct messages can share context across channels without leaking other group participants.

## Best Possible Solution

Design an additive, opt-in core session-routing contract that maps authorized group/chat senders and direct peers to a canonical identity session, filters unrelated group participants, preserves replies to the originating chat, and updates schema, docs, and routing tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main still isolates non-direct peers and only applies `identityLinks` to direct-message scopes.

## Solution Assessment

Unclear. The proposed config examples are useful directionally, but the best maintainable solution needs explicit privacy-filtered identity merging plus runtime and reply-routing safeguards before implementation.
