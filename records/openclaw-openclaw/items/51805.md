## Summary

Keep open. Current main still isolates group/channel sessions from DMs and only applies identity linking inside direct-message session scopes, while the related open shared-session trackers and groupScope PR do not cover this issue's per-identity group-to-DM filtering and privacy model.

## What This Changes

Request an opt-in session-routing mode that lets a user's group-chat turns and DMs share conversation context without including other group participants.

## Best Possible Solution

Design an additive opt-in core session-routing contract that maps authorized group senders and direct peers to a canonical identity session, filters unrelated group participants, preserves replies to the originating chat, and updates schemas, docs, routing tests, and sandbox guidance.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection gives high confidence that current main still isolates non-direct peers and only applies `identityLinks` to direct-message scopes.

## Solution Assessment

Unclear. The proposed config examples are useful directionally, but the best maintainable solution needs explicit privacy-filtered identity merging plus runtime and reply-routing safeguards before implementation.
