## Summary

Keep this PR open. Current main still has the CJK auto-capture gap, and the PR is the right plugin-scoped implementation path, but it broadens CJK capture without matching localized prompt-injection guard coverage.

## What This Changes

The PR expands memory-lancedb auto-capture for short CJK phrases, adds validated literal customTriggers config, wires it into the live capture path, and updates tests, docs, manifest metadata, and changelog.

## Best Possible Solution

Land the plugin-scoped trigger/config fix with localized prompt-injection guard coverage if maintainers want parity with the existing English safety filter; leave LLM intent detection as separate product work.

## Reproduction Assessment

Yes. On current main, the reported short CJK phrases fail deterministically through shouldCapture's length gate or missing trigger coverage before any embedding or LanceDB dependency is involved.

## Solution Assessment

Mostly yes. Keeping the fix inside memory-lancedb with literal custom triggers is the narrow maintainable approach, but the safest version should add CJK prompt-injection tests or filters alongside the new language triggers.
