## Summary

This PR should stay open: it adds useful cron fallback observability and has real runtime proof, but the current head still withholds the drain-continuation flag from the first embedded attempt, so an already-admitted cron/manual run can fail or skip its primary model if Gateway drain starts before the first queue enqueue.

## What This Changes

The PR adds cron task fallback progress, queue pressure fields, gateway-drain continuation plumbing for admitted fallback work, terminal-control sanitization, focused tests, a proof script, and related model-failover/governance docs.

## Best Possible Solution

Keep the cron fallback progress feature, but first preserve the continuation contract for the first already-admitted embedded attempt, add a regression for drain before the first session/global enqueue, and rebase the branch cleanly.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: start an admitted cron/manual fallback run, flip Gateway drain before the first embedded session/global enqueue, and the primary attempt lacks the continuation flag so the queue rejects it.

## Solution Assessment

No, not yet. The proposed direction is maintainable after the prior repairs, but the first-attempt drain edge means the implementation still does not fully preserve the admitted-run contract it is adding.
