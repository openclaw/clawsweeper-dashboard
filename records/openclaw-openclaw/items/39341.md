## What This Changes

The issue asks the Discord security audit to stop warning about missing slash-command allowlists when top-level `channels.discord.allowFrom` is configured.

## Best Possible Solution

## Reproduction Assessment

Yes, at source level. With native commands enabled, a configured guild, no guild/channel users allowlist, no credential-store or legacy allowlist, and only top-level `channels.discord.allowFrom`, current main leaves `ownerAllowFromConfigured` false and still emits the warning.

## Solution Assessment

Yes, the requested direction is the right narrow fix boundary. The implementation should use the existing effective Discord allowFrom resolver so the audit matches runtime account precedence.
