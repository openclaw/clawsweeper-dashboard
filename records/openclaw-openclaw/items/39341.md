## Summary

Keep open. Current main still computes the Discord native slash-command no-allowlists audit from legacy `dm.allowFrom` plus the credential-store allowlist, while top-level `channels.discord.allowFrom` is schema-supported, documented, and used by runtime authorization. There is already an open exact-scope PR, #72467, with closing syntax for this issue, so the maintainer path is to review, update, or land that PR rather than close early or queue duplicate automation.

## What This Changes

The issue asks for the Discord security audit to treat top-level `channels.discord.allowFrom` as an owner allowlist for native slash-command audit warnings.

## Best Possible Solution

Land a narrow Discord plugin fix that normalizes top-level `discordCfg.allowFrom` alongside legacy `dm.allowFrom` and pairing-store allowlists for `ownerAllowFromConfigured`, with the regression test retained. No core API change appears necessary.
