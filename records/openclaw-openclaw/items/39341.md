## Summary

Keep open: current main and v2026.5.2 still omit top-level `channels.discord.allowFrom` from the Discord `no_allowlists` audit check, and open PR #72467 explicitly targets this bug with closing syntax.

## What This Changes

The issue asks for the Discord security audit to treat top-level `channels.discord.allowFrom` as an owner allowlist for native slash-command audit warnings.

## Best Possible Solution

Review and land PR #72467 or an equivalent narrow change so the Discord audit uses the same effective account `allowFrom` source as runtime authorization, with regression coverage retained.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: enable native Discord commands with a configured guild, leave guild/channel users and legacy/store allowlists empty, and set only top-level `channels.discord.allowFrom`; `ownerAllowFromConfigured` remains false and the warning is emitted.

## Solution Assessment

Yes. The narrow maintainable fix is to include effective top-level/account `allowFrom` in `ownerAllowFromConfigured` and cover that case in `extensions/discord/src/security-audit.test.ts`; no new config option or product decision is needed.
