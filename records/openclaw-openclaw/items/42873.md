## Summary

Keep open: current main still has the reported source-level leak, but this branch is conflicting, incomplete for the broader delivery surface, and lacks after-fix real behavior proof. The linked issue and follow-up PRs need maintainer policy review before one implementation is landed.

## What This Changes

The branch adds an own-line `NO_REPLY` detector, uses it in reply normalization, and updates token and normalizer tests for that behavior.

## Best Possible Solution

Define one shared, documented, case-insensitive silent-reply classifier and land a rebased implementation that covers normalization, directive/outbound planning, cron/subagent paths, gateway projection/history, and real delivery proof.

## Reproduction Assessment

Yes, at source level. Current `origin/main` strips a trailing own-line `NO_REPLY` and returns the non-empty preamble from `normalizeReplyPayload`, and the current normalizer test asserts that strip-and-deliver behavior.

## Solution Assessment

No, not as-is. The branch covers only one normalizer branch and is case-sensitive; the safer path is a shared classifier and maintainer-selected policy applied consistently across delivery surfaces.
