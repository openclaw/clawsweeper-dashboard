## Summary

Keep open for maintainer review: the leak is source-reproducible, but this PR only handles one normalizer path, misses the case-insensitive silent-token contract, lacks after-fix real behavior proof, and overlaps open follow-up work that has not merged.

## What This Changes

The PR adds an own-line `NO_REPLY` detector, uses it in `normalizeReplyPayload`, and updates token/normalizer tests so own-line trailing `NO_REPLY` suppresses the whole reply while inline tokens are stripped.

## Best Possible Solution

Agree on one shared case-insensitive silent-token policy, document it, then land a rebased canonical fix across normalization, directive parsing, dispatcher/cron/subagent handling, and gateway display/history with regression tests and redacted real channel or runtime proof.

## Reproduction Assessment

Yes, source-reproducible. Current main and `v2026.5.12` strip trailing own-line `NO_REPLY` in `normalizeReplyPayload` and return the preamble, and the current test suite asserts that behavior.

## Solution Assessment

No, not as-is. The direction may be useful, but the current patch is too narrow, misses lowercase `no_reply`, and needs maintainer policy alignment plus real behavior proof before merge.
