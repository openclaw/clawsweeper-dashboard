## Summary

Keep open: current main and v2026.5.12 still strip a trailing own-line `NO_REPLY` and deliver the preamble, so the request is not implemented. The branch is not merge-ready because it only covers the normalizer, misses lower-case `no_reply`, overlaps linked work, and lacks inspectable real behavior proof.

## What This Changes

The PR adds an own-line silent-token helper, uses it in reply normalization, and updates token/normalizer tests to suppress payloads ending with a line containing `NO_REPLY` while preserving inline token stripping.

## Best Possible Solution

Choose and document one shared case-insensitive silent-token policy, then land or replace the open implementations with coverage across normalization, directives/outbound, cron/subagent handling, gateway display/history, and tests.
