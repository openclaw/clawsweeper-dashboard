## Summary

Keep open: current main still has the shared normalizer behavior this PR targets, so it is not obsolete. The branch should not merge as-is because it only covers one path, is case-sensitive, lacks contributor proof, and overlaps broader silent-token policy work.

## What This Changes

This PR adds an own-line `NO_REPLY` helper, uses it in `normalizeReplyPayload`, and updates token/normalizer tests to suppress an entire reply when `NO_REPLY` appears on its own trailing line.

## Best Possible Solution

Choose and document one case-insensitive shared silent-token classifier, then land a rebased implementation across normalization, directives/outbound, cron/subagent, gateway display, and tests.

## Reproduction Assessment

Yes, source-reproducible. Current main strips a trailing own-line `NO_REPLY` in `normalizeReplyPayload` and returns the preamble, and the current test suite asserts that behavior; I did not run tests because this review is read-only.

## Solution Assessment

No, not as-is. The direction may be useful, but this branch needs a shared case-insensitive policy across all delivery/display paths plus contributor real behavior proof.
