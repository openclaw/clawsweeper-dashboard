## Summary

Keep open: this PR targets a real silent-reply leak class, but the patch is incomplete for the claimed behavior and still needs contributor real-behavior proof plus maintainer selection among overlapping silent-token policies.

## What This Changes

The PR adds an own-line `NO_REPLY` detector and updates reply normalization/tests so a trailing own-line token suppresses the whole payload while inline tokens continue to be stripped.

## Best Possible Solution

Define one shared, cached, case-insensitive silent-token classifier and apply it consistently through normalization, directive parsing, outbound planning, cron/subagent handling, and gateway live/history projection after maintainers choose the trailing/own-line policy.

## Reproduction Assessment

Yes, source-reproducible. Current main strips trailing `NO_REPLY` in the normalizer and the checked-in test asserts that the preceding preamble remains deliverable.

## Solution Assessment

No. This is a useful narrow step, but it is not the best complete fix because it only updates normalization and leaves shared delivery/projection gates and mixed-case own-line matching unresolved.
