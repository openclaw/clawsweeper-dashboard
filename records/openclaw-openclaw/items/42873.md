## Summary

Keep open. The leak report is source-reproducible, but this PR is not mergeable as-is: it changes only the normalizer path, adds a case-sensitive own-line matcher, lacks inspectable real behavior proof, and overlaps with active silent-token policy alternatives.

## What This Changes

This PR adds an own-line `NO_REPLY` detector, applies it in `normalizeReplyPayload`, and updates token/normalizer tests so trailing own-line `NO_REPLY` suppresses the whole reply while inline tokens are stripped.

## Best Possible Solution

Choose and document one case-insensitive silent-token policy shared across normalization, outbound planning, directives, cron/subagent handling, and gateway history, then land one replacement with regression tests and redacted real channel or runtime proof.

## Reproduction Assessment

Yes, source-reproducible. Current main and `v2026.5.12` strip trailing `NO_REPLY` in `normalizeReplyPayload` and return the preamble, and the current test suite asserts that behavior.

## Solution Assessment

No, not as-is. The direction may be useful, but this branch needs a shared case-insensitive classifier across delivery/display paths plus maintainer policy selection and real behavior proof.
