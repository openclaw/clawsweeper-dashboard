## Summary

Keep open: the PR targets a real source-reproducible leak shape, but the branch is not merge-ready because it only covers one normalization path, misses case-insensitive own-line tokens, lacks real behavior proof, and overlaps active PRs with different silent-token policies.

## What This Changes

This PR adds an own-line `NO_REPLY` detector, applies it in `normalizeReplyPayload`, and updates token and normalizer tests to suppress own-line silent replies while preserving inline stripping.

## Best Possible Solution

Adopt one shared case-insensitive silent-token classifier and apply it consistently across normalization, directive parsing, outbound planning, cron/subagent delivery, gateway display/history, and tests while preserving media, channelData, and legitimate inline mentions.

## Reproduction Assessment

Yes. Source inspection shows current main strips trailing `NO_REPLY` but keeps a non-empty preamble, and the current normalizer test asserts that strip-and-deliver behavior.

## Solution Assessment

No, not as-is. Own-line suppression may be useful, but this patch needs a shared case-insensitive policy across final delivery/display paths plus contributor real behavior proof before merge.
