## Summary

Keep open: current main and the latest release still preserve preamble text before trailing own-line `NO_REPLY`, but this PR is not merge-ready because it only covers one normalizer path, misses lower-case silent tokens, and lacks inspectable real behavior proof.

## What This Changes

This PR adds an own-line `NO_REPLY` detector, applies it in `normalizeReplyPayload`, and updates token/normalizer tests so own-line trailing `NO_REPLY` suppresses the whole reply while inline tokens are stripped.

## Best Possible Solution

Land or replace this with one shared, documented, case-insensitive silent-token classifier across normalization, directives, dispatch, cron/subagent, and gateway paths while preserving media/channelData and legitimate token mentions.

## Reproduction Assessment

Yes, source-reproducible. Current main and `v2026.5.12` strip trailing `NO_REPLY` in `normalizeReplyPayload` and return the preamble, and current tests assert that behavior.

## Solution Assessment

No, not as-is. The direction may be useful, but this branch needs a shared case-insensitive classifier across delivery/display paths plus maintainer policy selection and real behavior proof.
