## Summary

Keep open. Current main and the latest shipped release still build both the SSH tunnel and configured remote targets, then emit `multiple_gateways` from reachable target count alone, so the reported false positive remains a narrow CLI warning bug.

## What This Changes

Request to stop `openclaw gateway probe` from treating an SSH tunnel and configured remote URL to the same gateway as multiple gateways.

## Best Possible Solution

Group reachable probe results by non-null gateway self identity before warning, suppress or downgrade the warning only when all reachable transports identify the same gateway, and keep the existing warning for missing or distinct identities.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: configure both `gateway.remote.sshTarget` and `gateway.remote.url`, let both probes connect, and current main will warn solely because two targets are reachable.

## Solution Assessment

Yes. Identity-aware warning logic is the narrowest maintainable fix because it preserves the direct-path probe while avoiding the misleading multiple-gateway warning for two transports to one gateway.
