## Summary

Keep open. Current main still emits the warning from the count of reachable probe targets, so an SSH tunnel plus configured direct URL can still be misclassified as multiple gateways even when both probes identify the same logical gateway.

## What This Changes

The issue requests identity-aware `gateway probe` warnings when SSH tunnel and configured direct URL transports reach one logical gateway.

## Best Possible Solution

Classify reachable targets by a conservative non-null gateway `self` identity, suppress or downgrade same-identity multi-transport results, and preserve `multiple_gateways` for distinct or unknown identities.

## Reproduction Assessment

Yes, at source level. Construct two reachable `GatewayStatusProbedTarget` entries with the same non-null `self`; current `buildGatewayStatusWarnings` still emits `multiple_gateways` because it only checks `reachable.length > 1`.

## Solution Assessment

Yes. Identity-aware warning classification is the narrowest maintainable fix because it keeps the useful direct-path probe while avoiding a false multi-gateway warning; skipping `configRemote` when a tunnel is active would lose useful reachability data.
