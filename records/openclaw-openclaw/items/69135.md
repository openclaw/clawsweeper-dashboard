## Summary

Keep open. Current main still warns on the number of reachable probe targets rather than distinct gateway identity, so an SSH tunnel plus direct `gateway.remote.url` can still be misclassified as multiple gateways when both transports reach the same logical gateway.

## What This Changes

The issue requests identity-aware `gateway probe` warnings so duplicate transports to one gateway are not reported as multiple reachable gateways.

## Best Possible Solution

Classify reachable targets by a conservative non-null gateway `self` identity, suppress or downgrade same-identity multi-transport results, and preserve `multiple_gateways` for distinct or unknown identities.

## Reproduction Assessment

Yes, at source level. Construct two reachable `GatewayStatusProbedTarget` entries with the same non-null `self`; current `buildGatewayStatusWarnings` still emits `multiple_gateways` because it only checks `reachable.length > 1`.

## Solution Assessment

Yes. Identity-aware warning classification is the narrowest maintainable fix because it keeps the useful direct-path probe while avoiding a false multi-gateway warning; skipping the configured remote target would lose reachability data.
