## Summary

Keep open. Current main still warns on `reachable.length > 1`, so an SSH tunnel target plus a configured remote URL to the same logical gateway can still trigger the reported false `multiple_gateways` warning.

## What This Changes

Requests that `openclaw gateway probe` avoid or clarify the `multiple_gateways` warning when an SSH tunnel and `gateway.remote.url` reach the same logical gateway.

## Best Possible Solution

Classify reachable targets by a conservative non-null gateway self identity, suppress or downgrade the warning when all reachable targets share that identity, preserve `multiple_gateways` for distinct or unknown/mixed identities, and update focused tests, docs, and changelog copy.

## Reproduction Assessment

Yes. A high-confidence source reproduction is to pass two reachable `GatewayStatusProbedTarget` entries with the same non-null `self` into `buildGatewayStatusWarnings`; current main still emits `multiple_gateways` from the target count.

## Solution Assessment

Yes. Identity-aware warning classification is the narrowest maintainable fix because it keeps the useful direct-path probe while avoiding a false multi-gateway warning; skipping the configured remote target would remove useful reachability information.
