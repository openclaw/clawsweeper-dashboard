## Summary

Keep open. Current main still emits `multiple_gateways` from the count of reachable probe targets, so an SSH tunnel plus configured direct URL can still warn about multiple gateways even when both probes report the same logical gateway identity.

## What This Changes

The issue requests identity-aware `gateway probe` warnings so two transports to the same gateway are not reported as multiple distinct gateways.

## Best Possible Solution

Classify reachable targets by a conservative non-null gateway self identity, suppress or downgrade the warning when all reachable targets share that identity, and preserve `multiple_gateways` for distinct or unknown identities.

## Reproduction Assessment

Yes, at source level. Construct two reachable `GatewayStatusProbedTarget` entries with the same non-null `self`; current `buildGatewayStatusWarnings` still emits `multiple_gateways` because it only checks `reachable.length > 1`.

## Solution Assessment

Yes. Identity-aware warning classification is the narrowest maintainable fix because it keeps the useful direct-path probe while avoiding a false multi-gateway warning; skipping `configRemote` when a tunnel is active would hide useful reachability data.
