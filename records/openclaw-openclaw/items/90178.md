## Summary

Keep open: current main still has a source-reproducible retry-limit/expiry give-up path that marks delivery failed or suspended without delivering a requester-visible wake, and the direct candidate fix remains open and unmerged.

## What This Changes

The issue asks for subagent announce retry-limit/expiry give-up to wake a parent waiting in `sessions_yield`, ideally with a synthetic failure/completion event and matching tool result, plus configurable retry/expiry limits.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/95080 or an equivalent narrow recovery that wakes the requester and closes the tool-result pairing, while tracking retry/expiry configurability separately.

## Reproduction Assessment

Yes, at source level: current main reaches retry-limit or expiry give-up, marks delivery failed/blocked, and the generic task wake path still suppresses non-cancelled subagent terminal updates. I did not run the timing-sensitive live Signal/K8s scenario.

## Solution Assessment

Yes for the narrow wake-up bug: terminal announce give-up needs a bounded requester-visible failure/completion result. No for bundling retry/expiry configurability without maintainer direction, because that adds public config surface.
