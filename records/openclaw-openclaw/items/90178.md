## Label Justifications

- `P1`: The report describes a reproducible agent/session deadlock where queued user messages can remain blocked until a gateway restart.
- `impact:session-state`: The failure leaves the parent session waiting for child completion state that current cleanup deletes without replacement.
- `impact:message-loss`: Lost subagent completion delivery can suppress the result the parent was waiting to synthesize or send back to the user.

## Summary

Current main still drops a subagent completion when announce delivery gives up: the cleanup path logs and deletes the pending run without emitting a parent-visible failure event, so the reported parent wait/deadlock risk remains valid. The configurable retry/expiry part needs product judgment, but the missing synthetic wake-up is a concrete session-state/message-delivery bug.

## What This Changes

The issue asks OpenClaw to wake a parent session waiting in `sessions_yield` when a child completion announce reaches retry-limit or expiry, and also to consider making announce retry/expiry timings configurable.

## Best Possible Solution

Keep this open for a narrow core fix that records/delivers a canonical child failure result to the requester session when announce retry-limit or expiry is reached, while treating retry timing configurability as optional follow-up product work.

## Reproduction Assessment

Yes for source-level reproduction: the current cleanup branch reaches retry-limit or expiry, logs give-up, deletes the pending run, and has no parent notification call. I did not run a live Signal/K8s reproduction, so this is source-reproducible rather than live-reproduced.

## Solution Assessment

The synthetic parent-visible failure event is the best narrow fix for the deadlock because it preserves the existing retry path while making the terminal failure observable to `sessions_yield`. Making the retry constants configurable is not automatically the best first fix and needs maintainer product judgment on config surface.
