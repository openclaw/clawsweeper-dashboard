## Summary

Keep open: current main and latest release still have retry-limit/expiry give-up paths that mark subagent delivery failed or blocked without a requester-visible wake event. The newly linked PR is the right candidate to review, but it is still open and unmerged.

## What This Changes

The issue reports that subagent announce retry-limit or expiry give-up discards the completion without waking a parent session blocked in sessions_yield.

## Best Possible Solution

Review and land the narrow blocked-delivery parent wake fix, then leave retry/expiry configurability to a separate maintainer product decision.

## Reproduction Assessment

Yes, at source level: current main reaches retry-limit or expiry give-up and records failed or suspended delivery while non-cancelled subagent terminal task updates are not auto-delivered to the requester. I did not run the timing-sensitive live Signal/K8s scenario.

## Solution Assessment

Yes for the narrow wake-up bug: terminal announce give-up should produce a bounded requester-visible failure/completion event. No for bundling retry/expiry config knobs automatically, because that adds public config surface and needs maintainer direction.
