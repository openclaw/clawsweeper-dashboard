## Summary

Keep open. Current main still does not show a hostname in the TUI footer/status bar, and the request is a narrow, valid multi-host usability improvement with clear implementation and validation paths.

## What This Changes

The issue requests adding a compact hostname label to the TUI status bar/footer, especially for users connected to multiple machines over Tailscale.

## Best Possible Solution

Add a TUI-only host label after the agent label, deriving Gateway mode from the parsed `connection.url.hostname` and handling embedded mode with a deliberate local-host fallback, while avoiding credentials/full URLs and leaving gateway protocol unchanged.

## Reproduction Assessment

Yes. Current-main source inspection is a high-confidence reproduction path: the footer renderer has no host part, while the docs and adjacent header behavior confirm the missing field is specific to the requested footer/status bar location.

## Solution Assessment

Yes. The narrow maintainable path is a local TUI formatter/rendering change plus tests and docs; a new gateway protocol field would be unnecessary unless maintainers want server-reported identity rather than the resolved connection/local host label.
