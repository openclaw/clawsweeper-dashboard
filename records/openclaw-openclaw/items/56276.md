## Summary

Keep open. Current `main` still does not show a hostname in the TUI footer/status bar, and the request remains a narrow multi-host usability feature with clear source, docs, and test touch points.

## What This Changes

Request to add a hostname segment to the OpenClaw TUI footer/status bar so Tailscale and other multi-host sessions are easier to distinguish.

## Best Possible Solution

Add a TUI footer `host <hostname>` segment derived from parsed gateway `connection.url.hostname`, deliberately handle non-URL local mode, update focused tests/docs/changelog, and avoid protocol or config changes.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection on current `main` gives high-confidence proof that the requested footer hostname is absent.

## Solution Assessment

Yes. The narrow maintainable solution is a local TUI formatter/rendering change using the existing backend connection URL, with regression coverage and docs, rather than adding a gateway protocol field.
