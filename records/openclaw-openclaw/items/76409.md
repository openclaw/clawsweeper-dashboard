## Summary

Current main already contains the same short tweakcn ID validator relaxation with focused regression tests, and the fix is present in shipped releases, so this PR is now redundant rather than a useful landing branch.

## What This Changes

The PR relaxes the Control UI tweakcn theme ID regex from an 8-character minimum to a 1-character minimum so short IDs like `twitter` can normalize before fetch.

## Best Possible Solution

Keep the shipped main implementation with its regression tests and close this older branch as redundant.

## Reproduction Assessment

Yes. The old failing path is source-proven from the PR base and linked reports: the `{7,127}` validator rejected short IDs before fetch, while live tweakcn endpoints for `claude` and `twitter` returned valid JSON; current main no longer reproduces because it has the fix.

## Solution Assessment

Yes. The best solution is already on main: relax the shared validator lower bound and cover short IDs in `ui/src/ui/custom-theme.test.ts`, rather than merging this older branch without the landed test coverage and release proof.
