## Summary

The reported published-package shape does not match the current npm registry package: the Feishu tarball retains TypeScript source entries but also includes the runtime metadata and compiled files that OpenClaw expects.

## What This Changes

The issue reports that `@openclaw/feishu@2026.5.6` points plugin runtime metadata at TypeScript source files instead of compiled JavaScript files.

## Best Possible Solution

Keep the current source-entry plus runtime-entry package contract; if the gateway log still appears, handle it as a stale local install or cache-specific support report with the installed package.json attached.

## Reproduction Assessment

No. A registry/package reproduction against `@openclaw/feishu@2026.5.6` did not match the report because the package includes `runtimeExtensions`, `runtimeSetupEntry`, and the compiled `dist` files.

## Solution Assessment

Yes. The current design keeps source entries for source checkouts and uses `runtimeExtensions`/`runtimeSetupEntry` for npm runtime files, which is narrower than rewriting `openclaw.extensions` to dist paths.
