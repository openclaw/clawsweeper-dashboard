## Summary

Keep open. Current main still leaves non-`--all` provider-filtered model listing on the configured-only row path, the checked-in test still asserts `No models found.` for an unconfigured provider filter, and the issue is paired with open fixing PR #75615, so it should remain open until that PR lands or is replaced.

## What This Changes

The issue reports that `openclaw models list --provider google` returns `No models found.` in version 2026.4.27 instead of listing provider models.

## Best Possible Solution

Review and land PR #75615, or a narrow replacement with the same boundary, so non-`--all` provider-filtered listings use the existing catalog/source-plan cascade while preserving the configured-only fast path for unfiltered `models list`.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: current `main` calls `appendConfiguredModelRowSources` for `modelsListCommand({ provider: "moonshot" })`, and the checked-in test asserts the user-visible `No models found.` output.

## Solution Assessment

Yes. The maintainable fix belongs in CLI model-list source selection and row-source tests, because the failure is generic provider-filter routing rather than Google-specific provider behavior or docs-only cleanup.
