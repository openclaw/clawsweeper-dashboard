## Summary

Keep open. Current main still routes non-`--all` provider-filtered model listings through the configured-only row path, and open PR #75615 already targets this regression with closing syntax, so this issue should remain open until that PR lands or is replaced.

## What This Changes

Restore provider-filtered model listing so `openclaw models list --provider google` returns catalog or discovered models instead of `No models found.`

## Best Possible Solution

Review and land or replace #75615 so non-`--all` provider filters use the existing catalog fallback cascade while preserving the unfiltered configured-list fast path.

Yes. A high-confidence source-level reproduction exists: current main only initializes the source-plan path for `--all`, then sends plain provider filters through configured-only rows; the current test also asserts `No models found.` for an unconfigured provider filter.

## Solution Assessment

Yes. The narrow maintainable fix belongs in CLI model-list source selection and row-source regression coverage, and #75615 is already the active implementation candidate for that path.
