## Summary

Keep open. Current main still routes provider-filtered `models list` without `--all` through the configured-only row path, and the issue is already paired with open fixing PR #75615, so it should remain open until that PR lands, is replaced, or is closed.

## What This Changes

The issue asks for `openclaw models list --provider google` to return the provider’s model catalog again instead of `No models found.`

## Best Possible Solution

Review and land #75615, or a replacement with the same narrow boundary, so provider-filtered non-`--all` listings use the catalog/source-plan cascade while preserving the default configured-list fast path.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: current main loads the source-plan module only for `--all`, routes `modelsListCommand({ provider: "moonshot" })` through configured-only rows, and the existing test explicitly expects `No models found.`

## Solution Assessment

Yes. The maintainable fix belongs in the CLI model-list source selection and row-source tests, not in Google-specific provider code or docs-only cleanup; the linked PR targets that boundary.
