## Summary

Keep open. Current main still has the provider-filtered non-`--all` path on the configured-only row flow that can print `No models found.`, and the issue is now paired with open fixing PR #75615, so it should not be closed before that PR lands or is replaced.

## What This Changes

The issue requests restoring `openclaw models list --provider google` so it returns provider catalog models instead of `No models found.`

## Best Possible Solution

Review and land #75615, or a replacement with the same narrow boundary, so provider-filtered non-`--all` listings use the catalog/source-plan cascade while preserving the no-filter configured-list fast path; then close this issue as implemented after merge.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: on current main, plain `--provider` skips the source-plan catalog cascade and the existing forward-compat test explicitly expects `No models found.` for a provider with no configured rows.

## Solution Assessment

Yes. The maintainable fix belongs in the CLI models-list source selection and row-source tests, not in provider-specific Google code or docs-only cleanup; #75615 appears to target that exact boundary and should be reviewed before this issue is closed.
