## Summary

Keep open. Current main still routes non-`--all` provider-filtered model listings through the configured-only row path, and open PR #75615 already uses closing syntax for this regression, so this issue should remain open until that PR lands or is replaced.

## What This Changes

Restore `openclaw models list --provider <id>` so unconfigured providers show catalog or registry rows instead of `No models found.`

## Best Possible Solution

Provider-filtered list should use the same manifest, provider-index, static catalog, and registry fallback cascade that `--all --provider` uses, with regression coverage for unconfigured Google-like providers.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: current main only initializes the source-plan path for `--all`, and the existing `provider: "moonshot"` test explicitly expects `No models found.` without loading the registry.

## Solution Assessment

Yes. The maintainable fix belongs in the CLI model-list source selection and row-source tests, preserving the default configured-list fast path while restoring catalog fallback for provider-filtered queries.
