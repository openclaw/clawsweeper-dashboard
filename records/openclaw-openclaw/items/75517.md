## Summary

Keep open. Current main still routes plain `models list --provider <id>` through the configured-only row path, and open PR #75615 explicitly references this report as the fix candidate, so this issue should stay open until that PR lands, is revised, or is replaced.

## What This Changes

Restore `openclaw models list --provider google` so provider-filtered model listing returns provider model rows instead of `No models found` for users without manually configured provider models.

## Best Possible Solution

Review PR #75615 against the intended CLI contract. The end state should explicitly define plain `models list --provider <id>` behavior, update focused tests/docs/changelog if catalog rows are restored, and preserve the no-flag configured-only fast path.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: current main only builds the source plan under `--all`, then the non-`--all` provider-filtered path appends configured rows and has an existing test expecting `No models found.` for a provider with no configured rows.

## Solution Assessment

Unclear. Restoring the provider catalog cascade for non-`--all --provider` matches the reporter's expected UX, but current docs and tests reserve full catalog discovery for `--all --provider`, so maintainers should validate that contract before merge.
