## Summary

Keep open. Current main still leaves non-`--all` provider-filtered model listing on the configured-only row path, and the issue is paired with open fixing PR #75615, so it should remain open until that PR lands or is replaced.

## What This Changes

The issue asks for `openclaw models list --provider google` to return provider catalog models again instead of `No models found.`

## Best Possible Solution

Review and land #75615, or a replacement with the same narrow boundary, so provider-filtered non-`--all` listings use the catalog/source-plan cascade while preserving the default configured-list fast path.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: current main routes `modelsListCommand({ provider: "moonshot" })` through the configured-only path, and the existing test explicitly expects `No models found.`

## Solution Assessment

Yes. The maintainable fix belongs in the CLI model-list source selection and row-source tests, not in Google-specific provider code or docs-only cleanup; the linked PR targets that boundary.
