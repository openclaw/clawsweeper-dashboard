## Summary

Keep this PR open: current main still lacks the inbound Feishu interactive-card hydration/raw-card context path, but the proposed patch is not mergeable as-is because it uses an unvalidated card-content query shape, changes the RawBody contract, and still lacks real Feishu after-fix proof.

## What This Changes

The PR adds Feishu inbound interactive-card parsing, richer message fetching, raw card JSON propagation, shared parser tests, and optional rawContent fields in Feishu message context types.

## Best Possible Solution

Keep the fix Feishu-local: request the validated card-content format through a compatible adapter, parse cards into normalized user text, expose raw card data without changing RawBody alias semantics, and require redacted live Feishu proof before merge.

## Reproduction Assessment

Yes, source-reproducible. Current main has no direct inbound interactive-card parsing branch and does not request full card content in Feishu message reads, so placeholder-only card payloads can remain unreadable without a live tenant repro here.

## Solution Assessment

No, not as currently patched. The Feishu plugin boundary is right, but the patch should use the validated card-content query shape and preserve RawBody as normalized user text.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Request the validated Feishu card-content format:** `extensions/feishu/src/send.ts:274`
  - body: The new fetch sends `params: { raw_card_content: true }`, but the related Feishu card-content work uses `card_msg_content_type=raw_card_content` for full card JSON. With this request shape, the follow-up fetch can still return placeholder content while the mock passes; use a compatible adapter for the validated query and assert it in tests.
  - confidence: 0.9
- **[P2] Preserve RawBody as normalized user text:** `extensions/feishu/src/bot.ts:942`
  - body: `parseFeishuMessageEvent` now records `ctx.rawContent` for every message type, and this line makes `RawBody` prefer that raw Feishu JSON envelope. Regular text messages would expose values like `{"text":"hello"}` instead of the documented legacy `CommandBody` alias, breaking hooks or templates that match plain user text.
  - confidence: 0.88
