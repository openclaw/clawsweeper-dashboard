## Summary

Keep this PR open because current main still does not implement the broader inbound Feishu interactive-card raw fetch/context path. The proposed branch is useful but not merge-ready: it requests raw card content with the wrong Feishu query shape and regresses the documented `RawBody` compatibility behavior for ordinary messages.

## What This Changes

The PR adds a shared Feishu interactive-card parser, fetches richer inbound interactive message content, propagates optional raw card JSON through Feishu message context fields, appends that raw JSON to the agent prompt for interactive cards, and adds focused Feishu tests.

## Best Possible Solution

Land a narrow Feishu plugin repair that preserves current-main quoted-card extraction, fetches inbound interactive-card payloads with Feishu’s raw-card message query shape, keeps `RawBody` and `CommandBody` normalized for ordinary messages, exposes raw card JSON only through an explicitly scoped interactive-card context path, and covers direct inbound cards plus existing quoted/thread card tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is an inbound Feishu `interactive` event whose event content is only `[Interactive Card]`: current main does not fetch a richer body for that path, and this PR’s mock would still pass even if the real API query does not request raw card content.

## Solution Assessment

No. The PR is aimed at the right Feishu-owned boundary, but the raw-card fetch parameter and `RawBody` compatibility change need correction before this is the narrowest maintainable fix.
