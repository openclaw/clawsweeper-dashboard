## Summary

Keep this PR open. Current main still does not implement the inbound Feishu interactive-card fetch path, but the proposed patch is not mergeable as-is because it uses an unverified raw-card query key, changes the documented RawBody compatibility behavior, and would drop current-main interactive-card parser fallbacks added after the PR branch diverged.

## What This Changes

The PR adds inbound Feishu interactive card parsing, a shared card parser, follow-up message fetch/context propagation for raw card JSON, and focused Feishu tests.

## Best Possible Solution

Land a narrow Feishu plugin repair that keeps current-main quoted/thread card parsing behavior, adds direct inbound interactive-card fetch/parsing with the correct Feishu raw-card query, exposes raw card JSON only through a card-specific context/prompt path, and preserves `RawBody`/`CommandBody` compatibility for ordinary messages.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is an inbound Feishu event with `message_type: "interactive"` and placeholder event content; current main has no interactive branch or follow-up fetch for that path, while the PR’s current mocks do not prove the real raw-card query works.

## Solution Assessment

No. The PR targets the right Feishu-owned boundary, but it needs the raw-card query, RawBody compatibility, and current-main parser fallback preservation fixed before it is the narrowest maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Use Feishu's raw-card message query:** `extensions/feishu/src/send.ts:274`
  - body: The new fetch sends `raw_card_content=true`, but the pinned SDK type surface for `im.message.get` only exposes `user_id_type`, and the related raw-card work identifies the query shape as `card_msg_content_type=raw_card_content`. With this key, the follow-up fetch can keep returning placeholder content while the mock still passes, so assert and send the query Feishu actually expects.
  - confidence: 0.9
- **[P2] Preserve RawBody for normal messages:** `extensions/feishu/src/bot.ts:942`
  - body: `parseFeishuMessageEvent` now stores `event.message.content` in `ctx.rawContent` for every message type, and this line makes `RawBody` prefer that raw envelope. Regular text messages would expose values like `{"text":"hello"}` instead of the documented legacy alias of `CommandBody`, breaking hooks/templates that use `RawBody` for plain user text.
  - confidence: 0.9
- **[P2] Preserve current card parser fallbacks:** `extensions/feishu/src/interactive-card.ts:151-159`
  - body: This new shared parser replaces the current-main `send.ts` interactive-card parser, but it does not carry forward current tests for template-variable substitution, `body.i18n_elements`/locale fallback, and post-format fallback. Rebasing or merging this PR as-is can regress quoted/thread interactive cards that current main already parses.
  - confidence: 0.86
