## Summary

Keep open: current main still lacks the direct inbound Feishu full-card parsing that this PR targets, but the patch is not merge-ready because the fetch uses the wrong card-content query shape and changes `RawBody` compatibility for ordinary messages.

## What This Changes

The PR adds Feishu inbound interactive-card parsing, a shared parser, raw card JSON propagation, and focused Feishu tests.

## Best Possible Solution

Keep the fix inside the Feishu plugin: fetch full card JSON with Feishu's message-content query, parse it through a bounded shared parser, expose interactive-card metadata without changing `RawBody`/`CommandBody`, and add focused tests plus a changelog entry.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: a direct inbound Feishu `interactive` event whose callback content is only a placeholder remains un-enriched on current main because the direct inbound parser does not fetch full card content.

## Solution Assessment

No. The PR is in the right Feishu-owned boundary, but it is not the best merge shape until the Feishu query parameter matches the message API behavior and `RawBody` remains compatible for ordinary messages.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Request Feishu's raw card content format:** `extensions/feishu/src/send.ts:274`
  - body: The new fetch sends `params: { raw_card_content: true }`, but the related Feishu raw-card work and reviewed API shape use `card_msg_content_type: "raw_card_content"` and return the richer card as `json_card`. With this request shape, live Feishu can still return placeholder content while the mock passes, so inbound cards remain unreadable.
  - confidence: 0.91
- **[P2] Keep RawBody on normalized message text:** `extensions/feishu/src/bot.ts:942`
  - body: `parseFeishuMessageEvent` now stores the raw Feishu envelope for every message type, and the context builder prefers it for `RawBody`. Regular text messages would expose JSON like `{"text":"hello"}` instead of the documented `CommandBody` alias, breaking hooks/templates that match plain user text.
  - confidence: 0.88
