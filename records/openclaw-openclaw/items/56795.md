## Summary

Keep this PR open. Current main still lacks the broader inbound Feishu interactive-card fetch/raw payload path, so the PR is not obsolete, but the proposed patch has two merge-blocking functional issues: it requests the richer card payload with the wrong query shape and it changes `RawBody` compatibility for ordinary Feishu messages.

## What This Changes

The PR adds a shared Feishu interactive-card parser, fetches richer inbound interactive message content, propagates optional raw card JSON through Feishu message context fields, appends that raw JSON to the agent prompt for interactive cards, and adds focused Feishu tests.

## Best Possible Solution

Land a narrow Feishu plugin repair that preserves current-main quoted-card extraction, fetches inbound interactive-card payloads with Feishu’s raw-card message query shape, keeps `RawBody` and `CommandBody` normalized for ordinary messages, exposes raw card JSON only for interactive-card context, and covers direct inbound cards plus existing quoted/thread card tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is an inbound Feishu `interactive` event whose event content is only the `[Interactive Card]` placeholder: current main does not fetch the full card for that path, while this PR’s current fetch mock would still pass even if the real API query does not request raw card content.

## Solution Assessment

No. The PR is aimed at the right Feishu-owned boundary, but the fetch parameter and `RawBody` compatibility change need correction before it is the narrowest maintainable solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Use Feishu's raw-card message parameter:** `extensions/feishu/src/send.ts:274`
  - body: The new fetch sends `raw_card_content=true`, but the inspected SDK type surface for `im.message.get` only exposes `user_id_type`, and the related raw-card work uses `card_msg_content_type=raw_card_content`. With this key, the follow-up fetch can keep returning placeholder content while the mock still passes. Send the raw-card query key Feishu expects and assert it in the mock.
  - confidence: 0.9
- **[P2] Preserve RawBody for normal messages:** `extensions/feishu/src/bot.ts:942`
  - body: `parseFeishuMessageEvent` now stores `event.message.content` in `ctx.rawContent` for every message type, and this line makes `RawBody` prefer that raw envelope. Regular text messages would expose values like `{"text":"hello"}` instead of the legacy clean body, breaking hooks/templates that use `RawBody` as the `CommandBody` alias. Keep `RawBody` on normalized content except for an explicitly scoped interactive-card raw field.
  - confidence: 0.88
