## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Use the Feishu raw-card content parameter:** `extensions/feishu/src/send.ts:274`
  - body: The branch calls `client.im.message.get` through a cast with `params: { raw_card_content: true }`, but the SDK version in this repo exposes `card_msg_content_type?: string`. Without the supported request parameter, rich interactive cards can still come back as placeholder content.
  - confidence: 0.9
- **[P1] Keep RawBody as normalized user text:** `extensions/feishu/src/bot.ts:942`
  - body: `parseFeishuMessageEvent` now stores the raw Feishu event payload for every message type, and the final context prefers it for `RawBody`. For normal text messages that turns `RawBody` into a JSON envelope while `CommandBody` remains text, breaking the documented compatibility alias.
  - confidence: 0.94
- **[P1] Preserve current parser fallbacks:** `extensions/feishu/src/interactive-card.ts:211-215`
  - body: Current main already handles interactive-card template variables, locale fallback arrays, and post-format fallback in the Feishu readback parser. Replacing that path with this new parser during rebase would drop those cases, so the branch should extend the current parser instead.
  - confidence: 0.82
