## What This Changes

The PR adds Feishu inbound interactive-card parsing, fetches fuller card content during message handling, propagates raw card JSON, and adds focused parser/message tests.

## Best Possible Solution

Consolidate the Feishu plugin fix around one shared parser/readback path, use a verified full-card request shape, preserve RawBody and CommandBody text compatibility, expose raw card data through a separate safe field, and require focused tests plus redacted live Feishu proof.

## Reproduction Assessment

Yes, at source level. Current main routes direct inbound Feishu interactive events through parseMessageContent without an interactive branch, and readback still does not request full card payloads.

## Solution Assessment

No, not as currently patched. The Feishu plugin is the right boundary, but the implementation should use a verified full-card request shape and preserve RawBody compatibility instead of overloading it with raw JSON.
