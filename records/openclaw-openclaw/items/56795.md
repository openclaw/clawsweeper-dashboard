- **[P1] Preserve RawBody for regular messages:** `extensions/feishu/src/bot.ts:942`
  - body: `parseFeishuMessageEvent` now stores the raw Feishu JSON envelope in `ctx.rawContent` for every message type, and this line makes `RawBody` prefer it.   - body: Feishu's message get/list docs expose `card_msg_content_type` as the card return-format parameter, but this sends `raw_card_content` as its own boolean parameter. With that request shape the follow-up fetch can keep returning the placeholder card body, so the new inbound interactive path still fails to obtain the raw card JSON it depends on.
