## Summary

Keep open. Current main has improved quoted/fetched Feishu card parsing, but it does not yet fetch full payloads for direct inbound interactive-card events, and this PR still has merge-blocking Feishu query and RawBody compatibility issues.

## What This Changes

This PR adds Feishu inbound interactive-card parsing, fetches richer message content, carries optional raw card JSON through Feishu message context, and adds focused Feishu tests.

## Best Possible Solution

Keep the fix inside the Feishu plugin by fetching card JSON through Feishu's message-content query, preserving the normalized body contract, and reusing current-main interactive-card parser coverage.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is an inbound Feishu event with `message_type: "interactive"` whose event content is only placeholder/fallback text; current main does not fetch richer card content for that direct inbound path, and the PR adds a mocked test for it.

## Solution Assessment

No. The PR targets the right Feishu-owned boundary, but it is not the best merge shape until the Feishu query parameter is corrected, RawBody remains compatible for ordinary messages, and the branch is reconciled with current-main parser fixes.

## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P1] Request Feishu's card content format:** `extensions/feishu/src/send.ts:274`
  - body: The new fetch sends `params: { raw_card_content: true }`, but the inspected Feishu message API uses `card_msg_content_type` to control card return format and the SDK type surface does not include this boolean key. With this request shape, the follow-up fetch can still return placeholder card content while the mock passes. Use the Feishu query shape and assert it in the test.
  - confidence: 0.91
- **[P2] Keep RawBody on normalized message text:** `extensions/feishu/src/bot.ts:942`
  - body: `parseFeishuMessageEvent` now populates `ctx.rawContent` for every message type, and this line makes `RawBody` prefer that raw Feishu envelope. Regular text messages would expose values like `{"text":"hello"}` instead of the documented legacy `CommandBody` alias, breaking hooks/templates that match plain user text. Scope raw card JSON to interactive-card metadata instead of replacing `RawBody` globally.
  - confidence: 0.88
