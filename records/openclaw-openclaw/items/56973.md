## Summary

Keep open. Current main still has the two ambiguous exported wrapper names and the reported aliased imports, and the plugin-facing wrapper is part of a published SDK subpath that contract tests still expect, so this needs maintainer compatibility direction rather than conservative cleanup closure.

## What This Changes

The issue requests replacing ambiguous exported normalizeChannelId wrapper names with the canonical normalizeChatChannelId and normalizeAnyChannelId names.

## Best Possible Solution

Use canonical internal imports for normalizeAnyChannelId and normalizeChatChannelId, while retaining or formally deprecating the public SDK normalizeChannelId alias until maintainers choose a compatible removal window.

## Reproduction Assessment

Yes. The issue is source-reproducible on current main by reading the two exported wrappers and the aliased imports in sessions-send-helpers.ts; no runtime path is needed for this naming collision.

## Solution Assessment

Unclear. The internal canonical-import cleanup is reasonable, but removing the exported plugin wrapper outright is not the safest solution while SDK subpaths still publish and test that symbol.
