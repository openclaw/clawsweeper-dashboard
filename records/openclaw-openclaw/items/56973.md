## Summary

Keep open. Current main still has the reported naming collision, but the exact removal request crosses public Plugin SDK compatibility surfaces, so this needs maintainer direction before a safe cleanup can proceed.

## What This Changes

Request to remove two exported `normalizeChannelId` wrapper functions and update call sites to import `normalizeChatChannelId` and `normalizeAnyChannelId` directly.

## Best Possible Solution

Stage this as an internal clarity cleanup first: use canonical helper names in repo-internal imports, keep or explicitly deprecate public SDK compatibility aliases, and only remove public aliases through a documented versioned SDK transition.

## Reproduction Assessment

Yes. A static source check on current main reproduces the issue: both exported wrappers still exist, and at least `sessions-send-helpers.ts` and `session-conversation.ts` must alias the two same-named imports.

## Solution Assessment

No for the exact suggested removal. The internal rename direction is reasonable, but deleting the plugin-side export now is not the narrowest maintainable fix while public SDK subpaths and contract tests still expose `normalizeChannelId`.
