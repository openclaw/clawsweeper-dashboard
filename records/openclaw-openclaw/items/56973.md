## Summary

Keep open. Current main still has the ambiguous exported wrapper names and the exact aliased import pattern described by the issue, but outright removal crosses the published plugin SDK surface, so this needs maintainer compatibility judgment rather than automatic cleanup.

## What This Changes

The issue requests removing ambiguous exported normalizeChannelId wrapper functions and updating callers to use canonical channel normalizer names directly.

## Best Possible Solution

Adopt canonical internal imports for `normalizeAnyChannelId` and `normalizeChatChannelId`, while preserving or formally deprecating the public SDK `normalizeChannelId` alias until maintainers choose a breaking-change window and update SDK docs/contracts.

## Reproduction Assessment

Yes. The issue is statically reproducible on current main by reading the two wrapper exports and the aliased imports in `sessions-send-helpers.ts`; no runtime reproduction is needed for this naming collision.

## Solution Assessment

Unclear. The requested internal rename is reasonable, but removing the exported plugin wrapper outright is not the safest solution unless maintainers approve the SDK compatibility break or keep a public alias.
