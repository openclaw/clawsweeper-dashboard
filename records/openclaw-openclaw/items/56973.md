## Summary

Keep open. Current main still has both ambiguous exported wrapper names and the reported aliased imports, but removing the plugin wrapper outright would cross a published plugin SDK surface and needs maintainer compatibility direction.

## What This Changes

Request to remove or deprecate two exported `normalizeChannelId` wrappers and update callers to use `normalizeChatChannelId` and `normalizeAnyChannelId` directly.

## Best Possible Solution

Use canonical internal imports for `normalizeAnyChannelId` and `normalizeChatChannelId`, while retaining or formally deprecating the public SDK `normalizeChannelId` alias until maintainers choose a compatible removal window.

## Reproduction Assessment

Yes. The issue is statically reproducible on current main by reading the two wrapper exports and the aliased imports in `sessions-send-helpers.ts`; no runtime path is needed for this naming collision.

## Solution Assessment

Unclear. The internal canonical-import cleanup is reasonable, but removing the exported plugin wrapper outright is not the safest solution while the SDK subpaths still publish and test that symbol.
