## What This Changes

The issue requests fixing iOS LAN gateway onboarding by allowing intended plaintext local WebSocket connections, avoiding bootstrap-only setup-code stalls, and preferring explicit token/password auth over bootstrap when both are present.

## Best Possible Solution

Keep this issue open while maintainers review PR #65185 or a successor. The end state should align iOS URL/TLS handling, Swift and TypeScript auth precedence, setup-code generation, docs, and regression tests around a clear private-LAN versus public/Tailscale policy.
