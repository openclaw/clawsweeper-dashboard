## Summary

Keep this PR open. It targets a real current-main GPT-5.4/5.5 default-parameter failure, but the branch still has two concrete correctness blockers: the poll send-validation tests are stale against the new contract, and the Discord no-op predicate drops meaningful component options.

## What This Changes

The PR changes poll-intent detection, treats empty Discord component defaults as absent, updates poll helper tests, and adds a changelog entry for the GPT-5.4/5.5 default-parameter issue.

## Best Possible Solution

Land the normalization after updating every affected send-path regression test and narrowing Discord default-object detection to the exact empty model-filled skeleton while preserving meaningful component fields.

## Reproduction Assessment

Yes. Static source review gives a high-confidence path: the PR guard makes current send-validation duration/public-only cases stop throwing, and a Discord payload with content plus `components.container.accentColor` would fall through to plain send instead of Components v2.

## Solution Assessment

No. The direction is maintainable, but this branch is not the best fix until it aligns runner tests and makes the Discord no-op predicate precise enough to preserve component-only options.
