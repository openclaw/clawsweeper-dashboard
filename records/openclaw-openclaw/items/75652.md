## Summary

Keep this PR open, but it still needs changes before merge: the normalization direction matches the reported GPT-5.4/5.5 default-parameter bug, while the branch currently weakens send-path poll validation and can drop meaningful Discord component options.

## What This Changes

The PR changes poll-intent detection, Discord component parsing for empty default-filled objects, poll helper tests, and the changelog entry for the GPT-5.4/5.5 message-tool compatibility issue.

## Best Possible Solution

Land a narrow default-normalization fix that preserves invalid-poll detection where intended, treats only true model-filled empty Discord component skeletons as absent, and leaves the broader #52757 media-path work open.

## Reproduction Assessment

Yes. Static review gives a high-confidence reproduction path: with the PR diff, existing duration/public-only send-validation cases stop throwing, and a Discord payload with message content plus `components.container.accentColor` can be downgraded to plain send.

## Solution Assessment

No. The approach is maintainable, but this branch is not the best fix until the poll contract/tests are aligned and the Discord no-op predicate accounts for meaningful component fields.
