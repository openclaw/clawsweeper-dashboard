## Summary

Keep open: the PR targets a real GPT-5.4/5.5 default-parameter bug, but the current patch still leaves two merge-blocking issues in poll send validation and Discord component normalization.

## What This Changes

The PR changes poll intent detection to require a non-empty poll question or option, treats empty Discord component/default modal objects as absent, updates poll helper tests, and adds a changelog entry.

## Best Possible Solution

Land the normalization after updating send-path regression coverage, narrowing Discord default-object detection to the exact empty model-filled skeleton, and keeping #52757 open for the remaining media-path work.

## Reproduction Assessment

Yes. Source review shows current main treats non-zero poll durations and true booleans as poll intent, while this PR's guard would make existing duration/public-only send-validation cases stop rejecting; the Discord parser/runtime path also shows how a container/reusable-only component payload can be dropped by the PR's early return.

## Solution Assessment

No. Normalizing default-filled optional params is the right direction, but this branch needs narrower Discord no-op detection and aligned send-path tests before it is the maintainable fix.
