## Summary

Keep open. The bug remains source-reproducible on current main and this PR targets the right area, but the branch still has two P2 correctness blockers and lacks after-fix real behavior proof.

## What This Changes

The PR changes poll intent detection, Discord component parsing, poll helper tests, and the changelog to tolerate GPT-5.4/5.5 default-filled `message.send` parameters.

## Best Possible Solution

Land a corrected normalization that treats model-filled no-op defaults as absent without dropping intentional component metadata, updates send-path regression coverage, and leaves media-root policy to a separate review.

## Reproduction Assessment

Yes, source-reproducible. Current main still throws for `action="send"` when default-filled poll params make `hasPollCreationParams` true, and empty Discord modal fields still reach the parser throw path.

## Solution Assessment

No. The normalization direction is sound, but this branch is not the best fix until it reconciles the send-validation poll contract, preserves component-only metadata, and includes real behavior proof.
