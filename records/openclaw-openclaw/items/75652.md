## Summary

Keep this PR open for changes. The PR is aimed at a real current-main bug, and the changelog/closing-syntax follow-up is now addressed, but the branch still leaves existing send-validation tests inconsistent with the new poll-intent contract and its Discord empty-default shortcut can drop meaningful component-only options.

## What This Changes

The PR changes poll-intent detection to require a non-empty poll question or option, treats all-empty Discord component/modal defaults as absent, updates poll helper tests, and adds a changelog entry for the GPT-5.4/GPT-5.5 default-parameter issue.

## Best Possible Solution

Land a narrow normalization fix that preserves the semantic poll-intent direction, updates every affected send-path regression test, and narrows Discord default-object detection so only the exact empty model-filled skeleton is ignored while meaningful component options still use the component path.
