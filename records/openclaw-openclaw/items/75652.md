The PR changes poll creation detection and Discord component parsing to treat model-filled default parameters as absent, updates poll helper tests, and adds a changelog entry.

## Best Possible Solution

Land the normalization after aligning send-path regression coverage, preserving meaningful Discord component fields, and keeping #52757 open for the remaining media-path work.
