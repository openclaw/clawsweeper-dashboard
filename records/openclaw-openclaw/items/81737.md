## Summary

Keep open: current main and the latest shipped release still reserve stdout for the final `--json` response and do not expose an agent stdout NDJSON event stream. This is a real public CLI/API feature request that needs maintainer design for the flag and event schema rather than cleanup closure.

## What This Changes

Request an optional NDJSON stdout event stream for `openclaw agent --json` and `openclaw agent --local --json` so scripts can observe agent progress before the final result.

## Best Possible Solution

Add a documented core CLI `--stream` or `--event-stream` contract that emits sanitized stable NDJSON for both Gateway and local runs while preserving plain `--json` final-only behavior.

## Reproduction Assessment

Not applicable as a bug reproduction. Source and release inspection confirm the current behavior: `openclaw agent --json` has no stream flag and emits only the final JSON response.

## Solution Assessment

Unclear until maintainers settle the public event schema. A core CLI flag is the right surface if OpenClaw wants scriptable progress streaming, but it should project sanitized stable events rather than pipe raw internal event shapes.
