## Summary

Keep open: current main preserves `openclaw agent --json` as a final-result JSON output path, while the requested stdout NDJSON stream would add a new CLI flag and public output contract.

## What This Changes

Add an optional stdout NDJSON event stream for `openclaw agent --json`, especially for local embedded agent runs.

## Best Possible Solution

Define an opt-in CLI streaming contract that reuses redacted agent events, preserves final-only `--json` by default, and appends a final result event compatible with existing JSON consumers.

## Reproduction Assessment

Not applicable for a feature request. Source, docs, and tests confirm current main has a final-result `--json` path and no streaming flag.

## Solution Assessment

Unclear until maintainers choose the public NDJSON schema, flag name, and privacy contract. The narrow maintainable path would be an opt-in stream derived from the existing agent event bus, not a change to default `--json` behavior.
