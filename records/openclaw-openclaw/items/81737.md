## Summary

Keep open: current main and v2026.6.8 still expose final-only `openclaw agent --json` output with no stdout NDJSON/event-stream flag. The related duplicate issues are already closed to this canonical design item, and the remaining work is a public CLI/API contract that needs maintainer direction.

## What This Changes

Requests an opt-in `openclaw agent --json` stdout NDJSON event stream for live agent progress events plus a final result payload.

## Best Possible Solution

Define one opt-in streaming CLI contract that preserves today’s final-only `--json` behavior, then implement the agreed event schema across Gateway and local runs with docs and focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction. Source and docs verification show current main and v2026.6.8 register only final `--json` output for `openclaw agent`, with no stdout event-stream flag.

## Solution Assessment

Unclear until maintainers choose the public contract. NDJSON is plausible, but the best implementation needs decisions on flag naming, event types, final-envelope compatibility, redaction, and local/Gateway parity.
