## Summary

Keep open: current `main` has internal agent event streams and Gateway SSE surfaces, but no `openclaw agent --json` stdout NDJSON event-stream mode. This is a legitimate new CLI/API feature request, not an already-implemented behavior or cleanup-close candidate.

## What This Changes

The issue requests an optional NDJSON stdout event stream for `openclaw agent --json` so scripts can consume real-time agent progress before the final result.

## Best Possible Solution

Design a documented `--stream` or `--event-stream` CLI contract that emits sanitized NDJSON events for both Gateway and `--local` runs while preserving current `--json` final-output behavior when streaming is not requested.

## Reproduction Assessment

Yes for the current behavior by source inspection: `openclaw agent --json` has no stream flag and writes final JSON only. This is not a bug reproduction; it is a confirmed missing feature surface.

## Solution Assessment

No immediate close is appropriate. The requested direction is plausible, but maintainers should first settle the CLI event schema and compatibility contract before implementation.
