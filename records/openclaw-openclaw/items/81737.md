## Summary

Keep open: current main still lacks a stdout NDJSON event-stream mode for `openclaw agent --json`; this is a legitimate new public CLI/API feature request that needs maintainer schema and safety decisions.

## What This Changes

The issue requests a new `openclaw agent --json` streaming flag that emits real-time NDJSON progress events on stdout while preserving the existing final JSON behavior.

## Best Possible Solution

Design a documented, sanitized NDJSON event-stream contract for `openclaw agent` that works in both Gateway and local modes while leaving plain `--json` unchanged.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection confirms current main has no stream flag and only emits final JSON for `--json`.

## Solution Assessment

Unclear until maintainers settle the public event schema. A new flag is plausible, but directly exposing every internal event shape would not be the safest contract.
