## Summary

Keep open: current main still has internal agent event streams, Gateway/App SDK streaming surfaces, and final JSON output for `openclaw agent --json`, but no stdout NDJSON event-stream mode for the CLI. This is a legitimate new public CLI/API feature request that needs schema, redaction, and compatibility design rather than conservative cleanup.

## What This Changes

The issue requests an optional `--stream` or `--event-stream` mode that emits real-time NDJSON events on stdout for `openclaw agent --json`.

## Best Possible Solution

Define a documented `--stream` or `--event-stream` CLI contract that emits sanitized NDJSON events for both Gateway and `--local` runs while preserving the current final-only `--json` behavior when streaming is not requested.

## Reproduction Assessment

Yes for the current behavior by source inspection: `openclaw agent --json` has no stream flag and writes final JSON only. This is not a bug reproduction; it confirms a missing feature surface.

## Solution Assessment

Unclear as an immediate implementation. The requested direction is plausible, but maintainers should first settle the stdout NDJSON schema, redaction rules, and Gateway/local compatibility contract.
