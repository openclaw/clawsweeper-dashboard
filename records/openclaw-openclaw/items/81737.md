## Summary

Keep open: current main and v2026.6.6 still expose `openclaw agent --json` as final-only JSON, and the closed related reports point back here as the canonical CLI stdout streaming design request.

## What This Changes

Request an opt-in stdout NDJSON/event stream for `openclaw agent --json` so automation can observe tool, thinking, and text progress before the final JSON result.

## Best Possible Solution

Define an explicit opt-in CLI streaming contract that preserves today’s final-only `--json` behavior, then implement the agreed event schema for both local and Gateway-backed runs with focused docs and tests.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request. Source and docs confirm current main lacks a streaming stdout flag and keeps `--json` final-envelope-only.

## Solution Assessment

Unclear until maintainers choose the public contract. The requested NDJSON stream is plausible, but the best implementation needs a decision on flag naming, event types, final-envelope compatibility, and local/Gateway parity.
