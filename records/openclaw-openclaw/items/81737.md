## Summary

Keep open: current main still has no stdout NDJSON event-stream mode for `openclaw agent --json`. The request is a legitimate new public CLI/API feature, not an implemented behavior or cleanup-close candidate.

## What This Changes

The issue requests a new `--stream` or `--event-stream` mode that emits real-time NDJSON agent events to stdout while preserving existing `--json` final-result behavior.

## Best Possible Solution

Design and document a sanitized `openclaw agent --json --stream` contract that emits stable NDJSON events for Gateway and local runs while keeping plain `--json` as a single final response.

## Reproduction Assessment

Yes for the current behavior by source inspection: the CLI has no stream flag and both Gateway and embedded JSON paths write only a final JSON envelope. This is a confirmed feature gap, not a failing existing bug contract.

## Solution Assessment

Unclear: a stream flag is a plausible direction, but the best implementation needs maintainer agreement on a sanitized public event schema and compatibility rules before code is written.
