## Summary

Keep open: current main still has final-only JSON output for `openclaw agent --json`, while live agent events are available only through internal/Gateway/plugin/UI surfaces. The requested stdout NDJSON mode is a legitimate new public CLI/API feature, not an implemented-on-main cleanup close.

## What This Changes

Request a `--stream` or `--event-stream` mode for `openclaw agent --json` that emits real-time NDJSON progress events to stdout while preserving the final JSON payload.

## Best Possible Solution

Design a documented, versioned `--stream` or `--event-stream` contract that emits sanitized NDJSON for both Gateway and local runs while leaving current `--json` final-output behavior unchanged by default.

## Reproduction Assessment

Yes for the current behavior by source inspection: `openclaw agent` has no stream flag, Gateway mode waits for a final response, and both Gateway/local `--json` paths write one final JSON envelope. This is a confirmed missing feature surface, not a broken existing contract.

## Solution Assessment

Unclear for the exact implementation. The requested direction is plausible, but maintainers should first settle the public event schema, redaction policy, and local/Gateway parity before implementation.
