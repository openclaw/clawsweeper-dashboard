## Summary

Keep open: current main still lacks a stdout NDJSON event-stream mode for `openclaw agent --json`. The prior ClawSweeper review reached the same conclusion, and this remains a new public CLI/API feature rather than a cleanup-close candidate.

## What This Changes

Request to add an optional `--stream` or `--event-stream` mode that emits real-time agent events to stdout while preserving the current final JSON output behavior.

## Best Possible Solution

Design a documented `--stream` or `--event-stream` CLI contract that emits sanitized NDJSON for both Gateway and `--local` runs while preserving the current non-streaming `--json` behavior.

## Reproduction Assessment

Not applicable as a bug reproduction; source inspection gives high confidence in the current behavior. `openclaw agent` has no stream flag, and both Gateway and embedded JSON paths emit a final JSON payload rather than incremental stdout events.

## Solution Assessment

Unclear until maintainer design review. A stdout NDJSON mode is plausible, but piping raw internal events directly would be a weak public contract; the safer solution is a stable, documented, sanitized event schema.
