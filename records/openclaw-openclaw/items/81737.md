## Summary

Keep open: current main has internal agent event streams and Gateway/session event surfaces, but no stdout NDJSON event-stream mode for `openclaw agent --json`. This is a legitimate new public CLI/API feature request, not an already-implemented behavior or cleanup-close candidate.

## What This Changes

Add an optional `--stream` or `--event-stream` mode to `openclaw agent --json` that emits real-time NDJSON events on stdout while preserving existing final JSON behavior.

## Best Possible Solution

Define and document a sanitized NDJSON CLI event contract for local and Gateway-backed agent runs, then implement it behind an explicit stream flag with focused tests for stdout/stderr behavior and final-result compatibility.

## Reproduction Assessment

Yes for the current behavior by source inspection: the agent command has no stream flag, and both Gateway and embedded `--json` paths emit only terminal JSON. This is a confirmed missing feature surface, not a bug in an existing documented contract.

## Solution Assessment

Unclear until maintainers settle the public event schema and redaction policy. The requested direction is plausible, but piping raw internal events directly to stdout would be a risky contract for a public CLI surface.
