## Summary

Keep open: current main still has internal agent events and Gateway streaming surfaces, but no stdout NDJSON event-stream mode for `openclaw agent --json`; this is a legitimate new CLI/API feature request that needs maintainer design before implementation.

## What This Changes

Add an optional `--stream` or `--event-stream` mode to `openclaw agent --json` that emits real-time NDJSON agent events on stdout while preserving existing final JSON behavior when streaming is not requested.

## Best Possible Solution

Define and document a stable, sanitized `openclaw agent --json --stream` NDJSON contract, then implement it across Gateway and local delivery paths without changing current non-streaming `--json` output.

## Reproduction Assessment

Yes for the current behavior by source inspection: `openclaw agent --json` has no stream flag and both Gateway and local paths emit final JSON only. This is a missing feature surface rather than a bug reproduction.

## Solution Assessment

Unclear as an immediate implementation: a stdout event stream is plausible, but maintainers should first settle the public NDJSON schema and filtering rules instead of exposing internal events directly.
