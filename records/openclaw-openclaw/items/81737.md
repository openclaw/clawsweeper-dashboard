## Summary

Keep open because current main has internal and Gateway event surfaces but no stdout NDJSON stream option for `openclaw agent --json`; this is a fresh public CLI/API feature request that needs schema and sanitization decisions, not a cleanup-close candidate.

## What This Changes

Add a streaming stdout NDJSON event mode to `openclaw agent --json` for real-time agent progress while preserving existing final JSON behavior.

## Best Possible Solution

Design and document a sanitized NDJSON event-stream flag for `openclaw agent`, sharing a stable event schema across local and Gateway runs and preserving the current `--json` stdout contract when streaming is absent.

## Reproduction Assessment

Yes, by source inspection: current main has no `--stream` or `--event-stream` CLI option and both Gateway and local JSON paths emit only final JSON. This is not a broken-behavior reproduction; it is a confirmed missing feature surface.

## Solution Assessment

Unclear as an implementation plan: the requested direction is plausible, but maintainers should first define the public event schema, redaction rules, and local/Gateway parity instead of piping raw internal events directly.
