## Summary

Keep open: current main and the latest release still expose final-only `openclaw agent --json` output, while duplicate reports have already been closed to this canonical design item. The remaining work is a public CLI/API streaming contract that needs maintainer direction before implementation.

## What This Changes

The issue requests an opt-in NDJSON stdout event stream for `openclaw agent --json` that emits live progress events and a final result for external consumers.

## Best Possible Solution

Define one opt-in stdout event contract that preserves final-only `--json`, then implement it for Gateway and local runs with docs and regression coverage.

## Reproduction Assessment

Not applicable as a bug reproduction. Source and docs verification show current main registers only final `--json` for `openclaw agent` and no `--stream` or `--json-stream` stdout event mode.

## Solution Assessment

Unclear until maintainers choose the public contract. NDJSON is plausible, but flag naming, event taxonomy, final envelope, redaction, and local/Gateway parity need a product/API decision.
