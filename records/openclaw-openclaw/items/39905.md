## Summary

Keep open. The requested Telegram override behavior is still plausible and not fully implemented on current main, but this branch is not merge-ready because it edits removed `src/telegram` runtime files, uses a stale streaming config shape, bundles unrelated secrets changes, and lacks required real Telegram proof.

## What This Changes

The PR adds Telegram direct/group/topic `replyToMode` and `streaming` overrides, resolves them before normal/native dispatch, adjusts quoted-DM preview streaming, updates docs/tests, and changes secrets baseline/test files.

## Best Possible Solution

Ship a focused Telegram plugin change in `extensions/telegram/src` that layers scoped `replyToMode` and nested `streaming` overrides over account defaults for normal/native replies, updates docs/tests, omits unrelated secrets changes, and includes redacted real Telegram proof.

## Reproduction Assessment

Yes at source level for the current gap: current main's Telegram schemas/types do not accept scoped `replyToMode` or `streaming`, and normal/native dispatch still receives account-level values only. Live Telegram behavior proof is still missing.

## Solution Assessment

No. The desired behavior should be implemented in `extensions/telegram/src` with the nested streaming config contract and separated from secrets changes; live Telegram proof is required before merge.
