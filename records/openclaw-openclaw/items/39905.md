## Summary

Keep open: the per-conversation Telegram override request is still not implemented on current main, but this PR is not correct as-is because it patches the deleted `src/telegram` layout, uses the old scalar streaming shape, lacks live Telegram proof, and mixes in unrelated secrets/CI changes.

## What This Changes

The branch adds Telegram direct/group/topic `replyToMode` and `streaming` overrides in old `src/telegram` runtime/config/docs/tests, plus unrelated secrets baseline and Windows secrets-test changes.

## Best Possible Solution

Rework this as an active Telegram plugin change under `extensions/telegram/src/**`, use the nested streaming config contract with clear topic > direct/group > account precedence, split unrelated secrets changes, and provide redacted live Telegram proof.

## Reproduction Assessment

Yes, at source level: current main resolves Telegram `replyToMode` and `streamMode` from account-level config and passes them unchanged through normal and native-command reply paths, while strict scoped schemas lack those override fields. I did not establish a live Telegram reproduction.

## Solution Assessment

No. The direction is useful, but the branch patches obsolete files and the old scalar streaming shape; the maintainable solution is a fresh plugin-side implementation with nested streaming config and real Telegram proof.
