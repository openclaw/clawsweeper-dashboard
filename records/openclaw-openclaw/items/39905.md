## Summary

Keep open, but do not merge as-is. The per-conversation Telegram override is still not implemented on current main, but this branch targets obsolete `src/telegram/**` runtime files, uses the old scalar streaming config shape, includes unrelated secrets churn, and still lacks real Telegram proof.

The PR adds Telegram direct/group/topic `replyToMode` and `streaming` overrides, wires them into normal and native command replies, updates docs/tests, and also changes secrets baseline/test files.

## Best Possible Solution

Implement scoped overrides in the active Telegram plugin, using the nested streaming config contract and topic > direct/group > account precedence for normal and native replies, then update docs/tests and provide redacted real Telegram proof.

## Reproduction Assessment

Yes, at source level: current main lacks scoped Telegram `replyToMode`/`streaming` fields and the active dispatch paths still use account-level values only. I did not run a live Telegram reproduction.

## Solution Assessment

No. The proposed branch is not the best solution because it targets obsolete files and the old scalar streaming shape; the maintainable fix belongs in `extensions/telegram/src/**` with the current nested config contract.
