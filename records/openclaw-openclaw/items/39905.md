## Summary

Keep open. ## What This Changes

This PR adds Telegram direct/group/topic `replyToMode` and preview `streaming` overrides, docs/tests, and a quoted-DM streaming behavior change.

## Best Possible Solution

Rework the feature in the active Telegram plugin with the nested streaming config contract, split unrelated security-test churn, and prove the behavior with redacted live Telegram evidence.

## Reproduction Assessment

Yes, at source level. Current main resolves Telegram `replyToMode` and `streamMode` only from account config, and strict direct/group/topic schemas still reject the requested override fields.

## Solution Assessment

No. The feature direction is reasonable, but this branch targets obsolete runtime paths and a scalar streaming contract; the safer solution is a fresh active-plugin implementation using the current nested config shape.
