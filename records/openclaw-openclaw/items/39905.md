Keep open. The scoped Telegram override idea is still not implemented on current main, but this PR is not mergeable as-is because it targets removed `src/telegram/**` files, uses the stale scalar streaming schema, includes unrelated secrets changes, and lacks required real Telegram proof.

## What This Changes

The branch adds Telegram direct/group/topic reply and streaming override types, schemas, docs, dispatch resolution, tests, a quoted-DM streaming subfix, and unrelated secrets/formatting changes.

## Best Possible Solution

Ship a focused current-main Telegram plugin port that adds scoped reply and streaming overrides through `extensions/telegram/src`, reuses the nested streaming contract, preserves the existing quoted-reply streaming fixes, and includes live Telegram proof.

## Reproduction Assessment

Yes, at source level. Current main's strict topic/group/direct schemas omit the scoped keys and the active Telegram plugin still dispatches normal and native command replies with account-level reply and streaming settings only.

## Solution Assessment

No. The feature direction is useful, but this branch should not merge without a current-plugin port, nested streaming schema alignment, unrelated-file split, and real Telegram proof.
