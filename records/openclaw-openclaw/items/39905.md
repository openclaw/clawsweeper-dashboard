## Summary

Keep open. The feature direction is useful, but this branch targets the old `src/telegram/**` tree instead of the active Telegram plugin, uses the wrong scoped streaming contract, bundles unrelated secrets changes, and still lacks real Telegram proof for a reply-context and streaming change.

## What This Changes

This PR adds Telegram direct/group/topic `replyToMode` and preview streaming overrides, dispatch resolution, docs/tests, and a quoted-DM streaming subfix.

## Best Possible Solution

Ship a focused current-main Telegram plugin port under `extensions/telegram/src`, reuse the nested streaming contract, keep the separately fixed quoted-reply streaming behavior, split unrelated secrets changes, and require live Telegram proof.

## Reproduction Assessment

Yes, at source level. Current main lacks the scoped config fields in strict schemas and still dispatches Telegram replies with account-level reply/stream settings only.

## Solution Assessment

No. The useful feature should be ported to the active Telegram plugin, aligned with the nested streaming schema, separated from unrelated secrets changes, and backed by real Telegram proof.
