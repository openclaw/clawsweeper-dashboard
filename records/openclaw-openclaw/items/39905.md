## Summary

Keep this PR open as useful but not merge-ready. Current main still lacks conversation-scoped Telegram reply/streaming overrides, but the PR targets the removed `src/telegram` runtime path, uses the old scalar streaming config shape for scoped overrides, and mixes unrelated secret-scanning/test churn into a Telegram feature branch.

## What This Changes

The PR adds Telegram per-direct/group/topic `replyToMode` and preview `streaming` overrides, wires normal and native-command dispatch to use scoped settings, updates Telegram docs/tests, and also changes secrets and CLI test artifacts.

## Best Possible Solution

Port the useful feature into the active Telegram plugin: add typed and strict-schema scoped overrides under direct chats, groups, and topics using the current nested streaming config shape; resolve topic > direct/group > account precedence in normal and native-command delivery; update Telegram docs/config metadata; cover scoped reply/streaming plus the existing quoted/plain reply streaming behavior; and leave unrelated secrets, CI, or baseline churn out of the patch.

## Reproduction Assessment

Yes. A source-level reproduction is enough: current main's strict Telegram topic/group/direct schemas omit the scoped keys, and runtime dispatch still uses only account-level `replyToMode` and `streamMode`. I did not run tests because this review was explicitly read-only.

## Solution Assessment

No, not as written. The product direction is reasonable, but the PR needs to be ported from the removed `src/telegram` tree to `extensions/telegram/src`, updated to the nested streaming config contract, and split from unrelated security/test changes.
