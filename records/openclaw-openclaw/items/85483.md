## What This Changes

The branch adds an in-memory image-generation usage recorder, records success/failure metadata from image and shared media completion paths, adds unit tests, and adds a root `_provenance.json` artifact.

## Best Possible Solution

Record only image-generation completions through an image-owned hook or typed media usage contract, keep fields bounded and redacted, remove the provenance artifact, and require redacted real-generation proof.

## Reproduction Assessment

Yes for the review findings by source inspection: the PR adds image usage recording to the shared media lifecycle used by music and video, and the new store is an unbounded process-lifetime array. I did not run tests because this review is read-only.

## Solution Assessment

No: the current implementation is not the best way to solve image-generation analytics because it crosses the media ownership boundary and stores raw metadata without policy. A safer solution is an image-specific, bounded, redacted usage hook or an extension of the existing diagnostics/session-usage contract.
