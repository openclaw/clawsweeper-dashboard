## What This Changes

## Best Possible Solution

Land a narrow local-provider availability fix only after live CLI proof and focused regression coverage confirm the intended configured-local semantics.

## Reproduction Assessment

## Solution Assessment

Yes, the row-level direction is a narrow fix for configured local providers because docs already describe the column as provider-level auth/availability metadata rather than a live per-model execution probe. It still needs live CLI proof and focused regression coverage before merge.
