The PR changes `toModelRow` so rows with a local `baseUrl` report `available: true` before falling back to registry availability or provider-auth checks.

## Best Possible Solution

Land a narrow row-builder fix that treats configured local/no-auth providers as auth-available for list output, with focused regression coverage and redacted live CLI proof from a local provider setup.

## Reproduction Assessment

No high-confidence live reproduction was run in this read-only review. Source inspection shows current main can keep local configured rows tied to registry/auth availability rather than the documented local no-auth signal, matching the linked report.

## Solution Assessment

Mostly yes: the shared row builder is the right narrow layer because it already computes both `local` and `available`. The mergeable final state still needs focused regression coverage and real CLI proof before maintainers accept the output-semantic change.
