## Summary

This PR addresses a real auth-fallback mismatch, but it is not merge-ready: it overrides authoritative model-discovery results for every local-looking URL, so stopped endpoints or absent models can be reported available, and the submitted evidence does not exercise the changed CLI against a live provider.

## What This Changes

The PR changes model-list row construction so models with loopback, wildcard, or `.local` base URLs are always reported as available before registry or provider-auth checks.

## Best Possible Solution

Keep explicit model-registry membership authoritative and apply the local-provider exception only in the existing no-discovery or explicitly enabled fallback path, with positive and negative tests plus redacted live CLI proof.

## Reproduction Assessment

Yes, at source level: an auth-free local model can fall through to false when registry availability is unavailable, and the proposed regression is reproducible by supplying a local model with an explicit empty `availableKeys` set. The original live Ollama CLI path has not been demonstrated.

## Solution Assessment

No. Skipping provider-auth fallback for local providers is directionally appropriate, but unconditional `true` bypasses the stronger model-level discovery result; the exception should be limited to the existing fallback branch.
