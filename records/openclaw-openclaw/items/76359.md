## Summary

Keep open. Current main still skips mtime/size-stable daily notes after the first Dreaming ingestion, while this draft PR targets that remaining checkpoint bug with a narrow per-dreaming-day state field and no blocking review findings from source inspection.

## What This Changes

The PR adds a per-daily-note `lastDreamingDayIngested` checkpoint, passes the current dreaming day into daily ingestion collection, strengthens the regression test, and adds a changelog entry.

## Best Possible Solution

Land a polished version of the per-dreaming-day daily-ingestion checkpoint with the strengthened regression coverage after the normal memory-core changed checks pass.

## Reproduction Assessment

Yes from source inspection. Current main's regression test only gets `dailyCount` to 2 after deleting `daily-ingestion.json`, and the production collector otherwise skips unchanged mtime/size daily files before they can contribute a later-day signal.

## Solution Assessment

Yes. Tracking the last dreaming calendar day per daily note is narrower than lowering promotion thresholds or changing recall scoring, and it preserves the existing same-day dedupe contract in `recordShortTermRecalls`.
