## Summary

Keep open as a CLI cleanup/design item. Current main and the latest release still mutate Commander's readonly `commands` array directly, but Commander 14.0.3 currently dispatches through that same array, so the reported stale name-map runtime failure is not proven enough for an automated bug-fix lane.

## What This Changes

The issue requests replacing or avoiding direct mutation of Commander's readonly `commands` array in CLI command-tree removal helpers.

## Best Possible Solution

Have a CLI owner either refactor lazy command replacement so OpenClaw no longer mutates Commander's readonly command list, or explicitly keep the helper with tests proving removed names cannot dispatch and detached command parent state is harmless.

## Reproduction Assessment

No high-confidence runtime reproduction is established for the claimed stale routing behavior. Source inspection does reproduce the direct mutation of Commander's readonly `commands` array on current main.

## Solution Assessment

Unclear. Commander 14.0.3 does not provide a public removal API, so the best fix is likely a small owner-approved lazy-registration refactor or a tested helper contract rather than a simple API swap.
