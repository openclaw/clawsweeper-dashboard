## Summary

Keep open as a low-priority CLI maintainability cleanup: current main and the latest release still splice Commander's readonly command list, but the reported stale dispatch failure remains unproven for the pinned Commander version.

## What This Changes

The issue asks OpenClaw to stop removing lazy CLI commands by directly mutating Commander's internal `commands` array.

## Best Possible Solution

CLI owners should choose a supported lazy command replacement design, or explicitly keep this helper as a tested internal Commander contract with parse-dispatch and parent-state coverage.

## Reproduction Assessment

No high-confidence runtime failure is reproduced here. Source inspection confirms the direct readonly-array mutation remains, while Commander 14.0.3 source indicates dispatch lookup reads `this.commands`, not a separate name map.

## Solution Assessment

Unclear. Commander 14.0.3 does not appear to expose a public removal API, so the best fix is likely a small owner-reviewed lazy-registration redesign or explicit tests around the current internal helper rather than a mechanical API swap.
