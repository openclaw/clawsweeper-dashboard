## Summary

Keep open. Current main and the latest release still expose only `daily` and `idle` session reset modes; the related `off` request was closed as a duplicate of this canonical manual-only reset request, while the broader silent-reset issue does not replace the focused config/API decision here.

## What This Changes

The issue requests an additive `session.reset.mode: "never"` manual-only mode for base, type, and channel reset policies.

## Best Possible Solution

Add one explicit manual-only mode across public types, Zod validation, runtime freshness, SDK exports, config help/docs, focused tests, and changelog while preserving existing reset behavior.

## Reproduction Assessment

Yes, by source inspection. A config using `mode: "never"` reaches a schema that accepts only `daily` and `idle`, and the runtime freshness policy has no manual-only branch.

## Solution Assessment

Yes in principle: a first-class manual-only reset mode is clearer than giant idle timeout sentinels. The exact public literal and strictness around unused fields need maintainer API/product direction before implementation.
