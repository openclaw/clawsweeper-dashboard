## Summary

Keep open. Current main at 5ecd01ff still rejects and omits a public `mode: "never"` session reset policy across types, zod validation, generated schema/help, runtime reset policy, tests, and docs; the earlier Codex keep-open assessment remains valid.

## What This Changes

Add a first-class `mode: "never"` session reset policy for base, type, and channel overrides so sessions reset only through explicit operator action.

## Best Possible Solution

Implement `never` as an additive session reset mode across public config, validation, generated schema/help, runtime policy resolution, freshness evaluation, docs, changelog, and focused precedence tests while preserving existing daily, idle, legacy idle, `/new`, and `/reset` behavior.

## Reproduction Assessment

Yes. A config with `session.reset.mode: "never"`, or the same value under `resetByType` or `resetByChannel`, deterministically reaches `SessionResetConfigSchema`, which currently accepts only `daily` and `idle`.

## Solution Assessment

Yes. A first-class additive union member and explicit no-expiry policy branch is the narrowest maintainable solution; the giant idle-timeout workaround is semantically wrong and is not represented in validation or docs.
