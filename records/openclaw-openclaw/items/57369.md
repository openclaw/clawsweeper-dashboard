## Summary

Keep open. Current `main` and the latest shipped tag still expose only `daily` and `idle` session reset modes, and the related `off` request was closed as a duplicate of this canonical manual-only reset request rather than replacing it.

## What This Changes

The issue asks OpenClaw to add a first-class `session.reset.mode: "never"` option for manual-only session reset behavior across base, type, and channel reset configuration.

## Best Possible Solution

Add one explicit manual-only reset mode across exported types, Zod validation, runtime freshness, config help/docs, focused session tests, and release notes while preserving existing `daily` and `idle` behavior.

## Reproduction Assessment

Yes, by source inspection. A config using `mode: "never"` reaches a strict schema that accepts only `daily` and `idle`, and the runtime freshness policy has no manual-only branch.

## Solution Assessment

Yes in principle: a first-class manual-only reset mode is clearer than giant idle timeout sentinels. The exact public literal and strictness around unused fields need maintainer API/product direction before implementation.
