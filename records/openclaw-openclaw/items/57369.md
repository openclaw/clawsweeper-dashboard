## Summary

Keep open. Current main and the latest shipped tag still expose only `daily` and `idle` reset modes, so the requested manual-only reset behavior remains unimplemented; the related `off` request was closed in favor of this canonical thread at https://github.com/openclaw/openclaw/issues/76534.

## What This Changes

Request adding a first-class `session.reset.mode: "never"` manual-only reset mode for base, type, and channel reset policies.

## Best Possible Solution

Add one explicit manual-only reset mode across exported types, Zod validation, runtime freshness evaluation, config help/docs, focused session tests, and changelog while preserving existing `daily` and `idle` behavior.

## Reproduction Assessment

Yes, by source inspection. A config using `mode: "never"` reaches a strict schema that accepts only `daily` and `idle`, and the runtime freshness policy has no manual-only branch.

## Solution Assessment

Yes in principle: a first-class manual-only reset mode is clearer than giant idle timeout sentinels. The exact public literal and validation semantics need maintainer API/product direction before implementation.
