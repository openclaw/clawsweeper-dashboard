## Summary

Keep open: current main still permits an internal WebChat Codex direct turn with unset visibleReplies to resolve the Codex harness default to message_tool_only, suppressing automatic final delivery. The issue is paired with an open closing PR and an open alternative PR, so it should remain open until maintainers land or reject a canonical fix.

## What This Changes

Ensure direct internal WebChat Codex turns do not let the Codex harness message-tool default suppress the normal visible final reply.

## Best Possible Solution

Land one canonical WebChat/Codex delivery fix that keeps internal WebChat replies visible while preserving message-tool semantics for explicit external origins.

## Reproduction Assessment

Yes, by source inspection: unset direct visibleReplies plus the Codex harness default resolves to message_tool_only, while WebChat chat.send builds an internal webchat source context. I did not execute a live transcript in this read-only review.

## Solution Assessment

Yes for the narrow safety boundary: internal WebChat should stay on automatic final delivery while explicit external origins keep message-tool delivery. The longer-term renderer alternative remains a maintainer product choice.
