## What This Changes

Ensure internal WebChat Codex direct replies stay on automatic final-reply delivery instead of being suppressed by the Codex harness message_tool default.

## Best Possible Solution

Review and land the linked WebChat-specific policy fix, or an equivalent narrow change, then track any intentional WebChat message-tool rendering semantics separately.

## Reproduction Assessment

Yes. Source inspection on origin/main gives a high-confidence path: unset visibleReplies plus the Codex harness default resolves direct delivery to message_tool_only, while WebChat is rejected as an outbound target.

## Solution Assessment

Yes for the immediate bug. The narrow maintainable fix is to skip the harness message_tool default only for internal WebChat source contexts while preserving external direct-chat behavior.
