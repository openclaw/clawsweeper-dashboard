## Summary

Keep this PR open: it addresses a plausible task persistence failure, but the patch normalizes only at the SQLite bind layer while the current task registry contract remains string-only and multiple live consumers still assume strings.

## What This Changes

The branch stringifies non-string `TaskRecord.task` values before SQLite writes and adds a regression test that inserts an object-shaped task payload.

## Best Possible Solution

Enforce the task value contract before records enter the registry: either reject/normalize non-string values at the create/upsert boundary, or deliberately widen `TaskRecord.task` and update every string-consuming caller.

## Reproduction Assessment

No high-confidence real OpenClaw reproduction is available yet. Source inspection shows a direct invalid-record upsert can exercise the SQLite bind path, but current registry and runtime contracts still type `task` as a string.

## Solution Assessment

No. The patch is a plausible persistence guard, but the best fix is to normalize or reject before records enter the registry, or deliberately widen the task contract and update string-assuming consumers.
