## Summary

Keep open: the low-level SQLite bind failure is real, but this patch normalizes only at the persistence layer, leaving the string-only task contract and downstream consumers exposed; the added test also needs a rebase/import fix and contributor real behavior proof is still missing.

## What This Changes

This PR adds SQLite-side serialization for non-string `TaskRecord.task` values and a regression test that inserts an object payload into `task_runs`.

## Best Possible Solution

Keep `TaskRecord.task` string-only by normalizing untrusted task input before record creation, then cover persistence plus status and flow consumers.

## Reproduction Assessment

Partially: yes for the low-level SQLite bind failure, because current main binds `record.task` directly and `node:sqlite` rejects object parameters. No high-confidence end-to-end OpenClaw path for structured task ingress is shown yet.

## Solution Assessment

No. SQLite-only serialization fixes one symptom too late in the data path; the maintainable fix is to normalize before `TaskRecord` creation, or deliberately widen the contract and update all string consumers.
