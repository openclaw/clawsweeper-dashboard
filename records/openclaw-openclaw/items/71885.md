## What This Changes

The PR adds JSON serialization for non-string TaskRecord.task values at the SQLite binding layer and adds a regression test that stores an object task payload as JSON.

## Best Possible Solution

Normalize non-string task input before TaskRecord creation or deliberately widen the contract across all consumers, then rebase and include redacted after-fix terminal/log proof for the SQLite/OpenClaw path.

## Reproduction Assessment

No high-confidence end-to-end OpenClaw reproduction path is shown. Source inspection does show that a non-string task value reaching the current SQLite store would be bound directly, but the real structured-task ingress path is still missing.

## Solution Assessment

No. SQLite-only serialization is too late for the current TaskRecord string contract; the safer fix is boundary normalization before records enter memory and persistence, or an explicit contract widening with all consumers updated.
