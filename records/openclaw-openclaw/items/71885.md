## Summary

Keep open: this is a plausible bug-fix PR, but the current patch is not mergeable because it serializes after the string-only task-record boundary, leaves other task consumers unsafe for structured values, adds a test with missing imports, and lacks required real behavior proof.

## What This Changes

This PR adds SQLite task-value serialization for structured `TaskRecord.task` payloads and a regression test that writes an object task payload to the task registry store.

## Best Possible Solution

Keep `TaskRecord.task` string-only by normalizing untrusted task input before record creation, then cover create, persistence, status, and flow consumers with focused regression proof.

## Reproduction Assessment

Partially: the low-level SQLite failure is source-reproducible because current main binds `record.task` directly and `node:sqlite` rejects object parameters. There is not yet a high-confidence end-to-end OpenClaw reproduction path showing structured task ingress.

## Solution Assessment

No. SQLite-only serialization fixes one symptom after the task has already crossed the string-only registry boundary; the safer fix is to normalize before `TaskRecord` creation or deliberately widen the task contract and update every consumer.
