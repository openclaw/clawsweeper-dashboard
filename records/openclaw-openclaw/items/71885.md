## Summary

Keep open: this targets a real SQLite persistence failure, but the current patch normalizes too late in the task lifecycle and is stale against current main tests/changelog policy.

## What This Changes

The PR adds a SQLite bind helper that JSON-stringifies non-string `TaskRecord.task` values and a store-level regression test for object task payloads.

## Best Possible Solution

Keep `TaskRecord.task` string-only, normalize untrusted task inputs at registry or detached-runtime ingress, and cover SQLite plus status/flow consumers with a changelog entry.

## Reproduction Assessment

Yes. The in-memory SQLite bind check reproduces object bind failure, and source inspection shows current SQLite persistence binds `record.task` directly.

## Solution Assessment

No. The PR's SQLite-only serialization is too late; the safer fix is to normalize before `TaskRecord` construction or deliberately widen the full task contract and update all consumers.
