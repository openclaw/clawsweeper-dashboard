## Summary

Keep open. The PR addresses the SQLite bind failure class, but it normalizes too late: a structured task value can still live in the in-memory `TaskRecord` and reach flow, status, and delivery code that assumes `task` is a string.

## What This Changes

The PR adds bind-time serialization for non-string task values in the SQLite task registry store and a regression test that inserts a cast object-valued task record directly into SQLite.

## Best Possible Solution

Preserve `TaskRecord.task` as the durable string contract and normalize unknown runtime task inputs before lookup, merge, storage, flow sync, observer emission, or delivery formatting. SQLite should then receive an already-normalized string rather than being the first normalization boundary.

## Reproduction Assessment

Yes. A high-confidence path is to pass a structured value through task creation or merge and then hit current status/flow/delivery code that calls `task.task.trim()`; the PR’s own regression also demonstrates the direct SQLite bind failure for a cast object-valued record.

## Solution Assessment

No. The proposed bind helper is a partial fix; the narrow maintainable solution is to coerce at registry ingress while leaving downstream task semantics string-based.

## Review Findings
