## Summary

Keep open: this is a plausible SQLite bind hardening fix, but the patch normalizes too late in the task data path, its added test is missing imports, and the contributor still has not provided after-fix real behavior proof.

## What This Changes

This PR serializes non-string `TaskRecord.task` values before SQLite binding and adds a regression test for object task payload storage.

## Best Possible Solution

Keep `TaskRecord.task` string-only by normalizing untrusted task input before record creation, then cover persistence plus flow/status consumers.

## Reproduction Assessment

Partially: the low-level SQLite bind failure is source-reproducible because current main binds `record.task` directly and `node:sqlite` rejects object parameters. No high-confidence end-to-end OpenClaw path for structured task ingress is shown yet.

## Solution Assessment

No. SQLite-only serialization fixes one symptom too late; the maintainable fix is to normalize before `TaskRecord` creation, or intentionally widen the contract and update all string consumers.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88
