## Summary

Keep open. The SQLite binding failure is real, but this PR is not ready for merge because it serializes only at the persistence bind point while the in-memory task record still violates the existing string contract and can still break status, flow, and delivery consumers; the test also needs to be refreshed against current main's test setup.

## What This Changes

The PR adds a SQLite bind helper that JSON-stringifies non-string `TaskRecord.task` values and adds a regression test for object-valued task payload persistence.

## Best Possible Solution

Keep `TaskRecord.task` as the durable string contract and normalize unknown runtime task inputs before task lookup, merge, record construction, persistence, observer emission, and flow/status delivery; persistence should then bind an already-normalized string, with regression coverage through the registry ingress path.

## Reproduction Assessment

Yes. The focused in-memory SQLite bind check reproduces object and array bind failures, and current-main source inspection shows a structured value reaching `createTaskRecord` would still remain non-string in memory under this PR.

## Solution Assessment

No. Binding-time serialization is too late for the current contract; the narrower maintainable fix is to coerce the task value at task-registry ingress and refresh the regression test against current main.
