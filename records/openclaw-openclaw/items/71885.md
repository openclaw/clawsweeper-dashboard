## Summary

Keep open: this PR is a focused attempt at a real task-persistence edge, but the SQLite-only guard is the wrong layer while `TaskRecord.task` remains a string contract and live in-memory consumers can still see a structured value before restore; the external PR also lacks real behavior proof and is currently conflicting.

## What This Changes

The PR JSON-stringifies non-string `TaskRecord.task` values at SQLite bind time and adds a regression test that inserts an object-shaped task directly into the SQLite store.

## Best Possible Solution

Normalize or reject non-string task values at the task registry entry boundary, or explicitly widen the `TaskRecord.task` contract and update consumers and persistence together, then refresh the branch and add real behavior proof.

## Reproduction Assessment

No high-confidence real OpenClaw reproduction is present. Source inspection shows a non-string value would hit a text/string SQLite binding path if it reached the store, but the PR does not prove that ingress path in a real setup.

## Solution Assessment

No. The patch is a plausible persistence guard, but the better fix is to normalize or reject before `TaskRecord` enters memory, or deliberately widen the task contract and update all consumers.
