## Summary

Keep open. Current main still binds `TaskRecord.task` directly into SQLite, but this PR is not merge-ready because it only serializes at the SQLite edge, lacks real behavior proof, and leaves the central task-record contract unresolved.

## What This Changes

The PR adds SQLite bind-time serialization for non-string task values and a regression test that directly upserts an object-shaped task payload.

## Best Possible Solution

Normalize or reject non-string task values at the task registry/API boundary, or explicitly widen the `TaskRecord.task` contract and update all consumers before merging.

## Reproduction Assessment

No high-confidence real OpenClaw reproduction is present. Source inspection shows a direct invalid-record upsert path can exercise the SQLite bind behavior, but current tool and registry entry contracts still require `task` to be a string.

## Solution Assessment

No. The patch is a plausible persistence guard, but the best fix is to normalize or reject before records enter the registry, or deliberately widen the task contract and update all string-assuming consumers.
