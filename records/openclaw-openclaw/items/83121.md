## What This Changes

Make Telegram `/models` expose a discoverable button path for selecting Claude CLI runtime-backed models after the provider/runtime split.

## Best Possible Solution

Keep the merged `/models` CLI-runtime provider implementation on main and let the next release carry it; a two-level runtime selector can be tracked separately if maintainers want that broader UX.

## Reproduction Assessment

## Solution Assessment

Yes. The implemented sibling-provider picker is the narrowest current fix for the broken discoverability and button-selection path; the larger runtime-subpicker idea is optional follow-up UX rather than required for this issue.
