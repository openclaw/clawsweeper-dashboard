## Summary

Keep open: current main and the latest release still intentionally prefer Homebrew for gateway-backed skill dependency installs when `skills.install.preferBrew` is true and `brew` is available. The requested change would alter documented installer policy across skills, not repair a proven regression from the supplied report.

## What This Changes

The issue requests making Homebrew a backup rather than the default/preferred installer for OpenClaw plugin or skill dependency installs, especially in root/container environments.

## Best Possible Solution

Make an explicit installer-policy decision for gateway-backed skill dependencies: either keep Homebrew preferred and document container workarounds, or make the selection root-aware/default-non-brew with updated docs and focused coverage.

## Reproduction Assessment

No, not for the claimed failure. Source inspection gives a high-confidence path for the documented brew-preferred behavior, but the issue lacks a concrete current-main command, affected skill/plugin, container image, and log to reproduce the root/container install failure.

## Solution Assessment

Unclear. Making Homebrew only a fallback may improve container ergonomics, but the maintainable fix depends on a product decision about the default skill dependency installer order rather than a narrow bug repair.
