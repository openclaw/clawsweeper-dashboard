## Summary

Keep open: current main still intentionally prefers Homebrew for gateway-backed skill dependency installs when available, and changing Homebrew to fallback-only is a cross-surface installer policy decision rather than a confirmed regression from this report.

## What This Changes

The issue requests making Homebrew a fallback rather than the default/preferred installer for OpenClaw skill dependency installs, especially in root/container environments.

## Best Possible Solution

Decide the skill installer policy, then either make `preferBrew` default false or root-aware and add non-brew installers where possible, or keep Homebrew preferred and document/configure the container workaround clearly.

## Reproduction Assessment

No. Source inspection confirms Homebrew is still the documented preferred skill dependency installer when available, but the report does not provide a concrete current-main failing command, affected skill/plugin id, container image, OS image, or log to reproduce the claimed root/container install failure.

## Solution Assessment

Unclear. Making Homebrew fallback-only may improve container/root ergonomics, but current docs and code intentionally prefer Homebrew when available, so maintainers should choose between flipping the default, adding root/container detection, expanding non-brew installers, or keeping the documented policy.
