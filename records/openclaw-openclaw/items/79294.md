## Summary

Keep open: current main still intentionally prefers Homebrew for gateway-backed skill dependency installs when `skills.install.preferBrew` is true and `brew` is available, while plugin package installs use ClawHub/npm/git rather than Homebrew. The root/container concern is plausible in some environments, but the report does not include a concrete failing command or affected skill, and changing the default installer policy needs maintainer product judgment rather than cleanup close.

## What This Changes

The issue asks OpenClaw to stop using Homebrew as the default installer for plugin or skill dependencies and only use it as a fallback.

## Best Possible Solution

Make the installer policy explicit: either keep Homebrew preferred and document root/container opt-outs, or change skill installer selection to be root/container-aware and prefer non-brew specs when available, with matching docs and tests.

## Reproduction Assessment

No. Source inspection confirms current main will select Homebrew first for skill installs when `preferBrew` is true and `brew` is available, but the issue does not provide a concrete failing skill/plugin, command, container image, or log to reproduce the claimed install failure.

## Solution Assessment

Unclear. Making Homebrew only a fallback may be a valid root/container reliability improvement, but the blanket default change is a cross-surface installer policy decision rather than the narrowest proven bug fix.
