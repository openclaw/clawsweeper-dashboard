## Summary

Keep open: current main and the latest release still intentionally prefer Homebrew for gateway-backed skill installs when `skills.install.preferBrew` is true and `brew` is available, while plugin package installs use ClawHub/npm/git rather than Homebrew. The report does not include a concrete failing command or environment, and making Homebrew a fallback everywhere is an installer policy decision rather than a proven regression fix.

## What This Changes

Request to stop using Homebrew as the default preferred installer for OpenClaw plugin or skill dependencies, especially in root/container environments.

## Best Possible Solution

Maintainers should decide the desired default installer policy, then either make skill installs root-aware and provide non-brew defaults where needed, or keep Homebrew preferred and document the container-specific path clearly.

## Reproduction Assessment

No. Source inspection confirms Homebrew is the current preferred skill installer when available, but the issue provides no concrete skill or plugin id, command, container image, OS, or logs to reproduce a current-main failure.

## Solution Assessment

Unclear. Making Homebrew only a fallback may help some container setups, but a blanket default change would alter shipped installer policy and should be decided by maintainers rather than handled as a narrow bug fix.
