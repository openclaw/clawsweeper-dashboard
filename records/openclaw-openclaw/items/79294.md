## Summary

Keep open: current main intentionally documents and implements Homebrew as the preferred gateway-backed skill dependency installer when `skills.install.preferBrew` is true and `brew` is available, while plugin package installs use ClawHub/npm/git rather than Homebrew. Changing Homebrew from preferred installer to fallback is a cross-surface installer policy decision, not a confirmed regression from the supplied report.

## What This Changes

Request to stop using Homebrew as the default installer for plugin or skill dependencies and make it a fallback for root/container-friendly installs.

## Best Possible Solution

Decide the desired installer policy for gateway-backed skill dependency installs, then either make `preferBrew` default false or root-aware and add non-brew bundled skill installers, or keep Homebrew preferred and document the container-specific workaround.

## Reproduction Assessment

No. Source inspection confirms Homebrew is a documented preferred skill dependency installer when available, but the issue does not provide a concrete current-main failing command, skill/plugin id, container image, or log to reproduce the claimed root/container failure.

## Solution Assessment

Unclear. Making Homebrew only a fallback may be a valid setup reliability improvement, but it changes documented installer defaults across bundled skills and should be decided as installer policy rather than handled as a narrow bug fix.
