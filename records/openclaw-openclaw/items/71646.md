## Summary

Keep open. Current main still has the reported MCP channel bridge retention pattern, and the linked same-author closing PR #71648 is still open and unmerged, so this issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue requests lifecycle bounds for MCP channel bridge pending Claude permission and approval state so stale entries do not accumulate in long-running `openclaw mcp serve` sessions.

## Best Possible Solution

Land or replace #71648 with an equivalent focused MCP bridge fix that clears both pending maps on close, prevents post-close writes, expires stale Claude permission and approval entries, and either adds a bounded cap or records that cap as a deliberate follow-up coordinated with #56420.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: add unique Claude permission requests without matching `yes/no` replies, or approval requested events without matching resolved frames, and the corresponding maps retain entries; `close()` also leaves both maps untouched.

## Solution Assessment

Yes for keeping this issue open. Because current main is still affected and #71648 is an open closing PR for the same report, the maintainable path is PR review, landing, or replacement rather than closing this issue before the fix reaches main.
