## Summary

Keep open. Current main still has the reported MCP channel bridge retention pattern: both pending maps are plain instance maps, close only clears waiters, Claude permissions are deleted only by a matching reply, and approvals are deleted only by a matching resolved event. The same-author closing PR #71648 remains open/unmerged, so this issue should stay open until that PR or an equivalent fix lands.

## What This Changes

This issue reports unbounded MCP channel bridge pending state for Claude permission requests and exec/plugin approvals because entries lack TTL/sweeper cleanup, close-time clearing, and caps.

## Best Possible Solution

Review, rebase, land, or replace #71648 with an equivalent MCP bridge fix that clears both pending maps on close, prevents post-close writes, expires stale Claude permission and approval entries, and adds focused regression tests without changing the public MCP protocol surface.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: insert unique Claude permission requests without matching replies, or approval requested events without matching resolved frames, and the corresponding maps retain entries; `close()` also leaves both maps untouched.
