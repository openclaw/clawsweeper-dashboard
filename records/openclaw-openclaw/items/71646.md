## Summary

Keep open: current main still has the MCP bridge pending-map lifecycle gap, and the same-author closing PR at https://github.com/openclaw/openclaw/pull/71648 is open and unmerged.

## What This Changes

The issue reports that MCP channel bridge pending Claude permission and approval maps can grow without TTL, close-time clearing, or hard caps in long-running `openclaw mcp serve` sessions.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/71648 with equivalent pending-map lifecycle cleanup, then track cap/FIFO and session-binding hardening as coordinated follow-ups.

## Reproduction Assessment

Yes, source-reproducible with high confidence. Unique Claude permission requests without matching replies and approval-requested events without matching resolved frames remain in the current maps, and `close()` leaves both maps untouched.

## Solution Assessment

Yes, the narrow maintainable fix is to bound the two maps inside `src/mcp/channel-bridge.ts` with expiry/sweeping and close-time clearing. The linked open PR is the implementation candidate rather than a reason to close the issue before merge.
