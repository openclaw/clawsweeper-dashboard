## Summary

Keep open: current main still forwards blank select/multiselect labels into Clack, so the adapter-level hardening is useful, but this branch is not mergeable yet because it predates current wizard behavior and lacks contributor real behavior proof.

## What This Changes

The PR adds wizard option-label normalization for Clack select/multiselect prompts and unit coverage for blank-label fallbacks.

## Best Possible Solution

Rebase the label fallback onto current main's shared option mapping, preserve searchable/sensitive/plain wizard behavior, add searchable-select coverage, and include redacted real wizard output before merge.

## Reproduction Assessment

Yes for the missing fallback behavior: current main directly forwards blank labels through `createClackPrompter().select()` and `.multiselect()`. The claimed Clack crash is less certain because Clack 1.3.0 treats a blank string label as present rather than automatically falling back to the value.

## Solution Assessment

No, not yet. The prompter adapter is the right boundary for malformed labels, but this branch needs to preserve current main's searchable select and sensitive prompt paths and provide real behavior proof before it is the best mergeable solution.
