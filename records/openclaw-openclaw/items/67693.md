## Summary

Keep open: current main and the latest release still forward blank option labels unchanged, so the central hardening is not implemented, but this PR should not merge as-is because it is behind current searchable-select behavior and lacks after-fix real behavior proof.

## What This Changes

The PR normalizes blank Clack select and multiselect option labels in the wizard prompter and adds unit tests for those fallbacks.

## Best Possible Solution

Keep the adapter-level fallback, rebase it onto current main while preserving searchable select, add searchable-select coverage, and require redacted real setup proof before merge.

## Reproduction Assessment

Yes for the missing fallback behavior: current main directly forwards blank labels through `createClackPrompter().select()` and `.multiselect()`. The claimed crash itself is unclear because Clack 1.3.0 treats a blank string label as present and does not obviously throw from that path.

## Solution Assessment

No, not yet. The prompter adapter is the right place to normalize malformed labels, but this branch needs to preserve current main's searchable-select path and add proof before it is the best mergeable solution.
