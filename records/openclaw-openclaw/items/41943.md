## Summary

Keep open: the Feishu proxy bug is still source-reproducible on current main, but this PR is not mergeable as submitted because it unconditionally forces proxy routing, crosses the Lark SDK type boundary, lacks real behavior proof, and is stale/conflicting against the current proxy helper and dependency surface.

## What This Changes

The PR adds an `HttpsProxyAgent`-backed proxy path to Feishu's Lark SDK HTTP wrapper when proxy environment variables are set.

## Best Possible Solution

Replace or rebase the branch with a target-aware Feishu HTTP adapter at the Lark `httpInstance` seam that preserves timeout/User-Agent behavior, honors proxy eligibility, uses the current helper/dependency surface, adds focused tests, and includes redacted real proxy proof.

## Reproduction Assessment

Yes, source-backed: current main sends Feishu probe/token/API HTTP calls through the Lark Axios-backed SDK path without a Feishu HTTP proxy policy, and linked reports provide concrete proxy-env steps plus `400 The plain HTTP request was sent to HTTPS port` and ECONNRESET failure modes.

## Solution Assessment

No. The PR targets the right Feishu-owned seam, but the submitted patch unconditionally forces proxy routing, bypasses target/NO_PROXY decisions, crosses the SDK type contract, and needs a current-main rebase plus real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Honor NO_PROXY before forcing proxy routing:** `extensions/feishu/src/client.ts:57-60`
  - body: When `proxyAgent` exists, every Lark HTTP request gets `httpsAgent` and `proxy: false`, so Axios no longer applies target-based `NO_PROXY` exclusions. Resolve the effective Feishu/Lark target URL and only attach proxy routing when that request should use it.
  - confidence: 0.93
- **[P2] Keep Axios-only options behind a typed adapter:** `extensions/feishu/src/client.ts:58-60`
  - body: `Lark.HttpRequestOptions` does not declare `httpsAgent` or `proxy`, so assigning those fields directly crosses the SDK type contract and should fail strict TypeScript. Keep the Lark request options typed and add a narrow Axios-backed handoff type or adapter.
  - confidence: 0.9
- **[P2] Use the current proxy helper surface:** `extensions/feishu/src/client.ts:10-17`
  - body: The branch constructs `HttpsProxyAgent` directly, but current main moved Feishu proxy construction to the shared proxyline-backed helper and `@openclaw/feishu` no longer declares `https-proxy-agent`. Rebase onto the current helper seam or add valid plugin-owned runtime dependency metadata.
  - confidence: 0.82
- **[P3] Refresh proxy routing when reusing cached clients:** `extensions/feishu/src/client.ts:52`
  - body: `proxyAgent` is captured when the HTTP instance is constructed, while cached clients are reused based on credentials, domain, and timeout only. If proxy env changes during the process lifetime, cached clients keep stale routing; resolve eligibility per request or include proxy state in the cache comparison.
  - confidence: 0.76
