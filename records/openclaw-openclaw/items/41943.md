## Summary

Keep open: current main still does not fix the Feishu SDK HTTP proxy failure and the PR targets the right seam, but the branch is conflicting and blocked by correctness, egress-policy, dependency, and real-behavior-proof issues.

## What This Changes

The branch changes the Feishu client HTTP wrapper to create an HttpsProxyAgent from proxy environment variables and pass `httpsAgent` plus `proxy: false` into Lark SDK HTTP calls.

## Best Possible Solution

Rebase or replace this with target-aware Feishu HTTP proxy handling at the Lark SDK `httpInstance` seam, honoring `NO_PROXY`, preserving timeout/User-Agent behavior, using the current dependency surface, and adding focused tests plus real proxy proof.

## Reproduction Assessment

Yes, source-backed rather than live-run: current main sends Feishu probe/token/API HTTP calls through the Lark Axios-backed SDK path without an HTTP proxy policy, and the linked reports provide concrete proxy env steps plus the `400 The plain HTTP request was sent to HTTPS port` failure mode.

## Solution Assessment

No. The PR targets the right Feishu-owned seam, but the maintainable fix should be target-aware, honor `NO_PROXY`, keep Axios-only options behind a typed adapter, and align with current plugin dependency ownership.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Honor NO_PROXY before forcing the proxy agent:** `extensions/feishu/src/client.ts:57-60`
  - body: When `proxyAgent` exists, every Lark HTTP request gets `httpsAgent` and `proxy: false`, bypassing Axios' normal `proxy-from-env`/`NO_PROXY` selection. Resolve the effective Feishu/Lark target URL and only attach the agent when that request should actually use the proxy.
  - confidence: 0.93
- **[P2] Keep Axios options out of Lark request types:** `extensions/feishu/src/client.ts:58-60`
  - body: `Lark.HttpRequestOptions` does not declare `httpsAgent` or `proxy`, so assigning those fields directly to `result` crosses the SDK type contract and should fail strict TypeScript. Keep the Lark options typed and add a narrow Axios-backed adapter or explicit intersection at the handoff.
  - confidence: 0.9
- **[P2] Align the proxy dependency with current plugin packaging:** `extensions/feishu/src/client.ts:2`
  - body: This import relies on `https-proxy-agent`, but current main no longer declares that dependency in `@openclaw/feishu`. A rebased fix should either use the shared `proxy-agent` helper seam or add plugin-owned runtime dependency metadata so standalone installs do not rely on root hoisting.
  - confidence: 0.82
- **[P3] Refresh proxy routing when the cached client is reused:** `extensions/feishu/src/client.ts:52`
  - body: `proxyAgent` is captured when the HTTP instance is constructed, while the cached client key tracks credentials, domain, and timeout only. If proxy env changes during the process lifetime, cached clients keep stale routing; resolve eligibility per request or include proxy state in the cache comparison.
  - confidence: 0.76
