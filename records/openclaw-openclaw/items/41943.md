## Summary

Keep open, but do not merge as-is. Current main still lacks HTTPS CONNECT proxy handling for Feishu/Lark SDK HTTP calls, and this PR targets the right owner file, but the proposed implementation bypasses NO_PROXY and writes Axios-only options onto the narrower Lark request type.

## What This Changes

The PR changes the Feishu client HTTP wrapper to create an HTTPS proxy agent from proxy environment variables and pass it through Lark SDK Axios calls.

## Best Possible Solution

Keep the fix in the Feishu plugin: add target-aware Axios proxy injection for Feishu/Lark HTTP calls, preserve NO_PROXY behavior, use a narrow Axios adapter type, add regression tests, and include the required changelog entry.

## Reproduction Assessment

Yes, source-backed. Set HTTP_PROXY/HTTPS_PROXY and run `openclaw channels status --probe`; current main routes the Feishu probe through the timeout-only Lark SDK HTTP wrapper that lacks an HTTPS CONNECT agent.

## Solution Assessment

No. The PR is in the right owner file, but the maintainable fix should resolve the effective Feishu/Lark target URL, gate proxy use through the existing NO_PROXY-aware helper, and bridge Axios-specific options with a narrow adapter type.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Respect NO_PROXY before adding Axios proxy agents:** `extensions/feishu/src/client.ts:58-60`
  - body: When a proxy env var exists, this patch always sets `httpsAgent` and `proxy = false`. That bypasses normal NO_PROXY/no_proxy exclusion decisions, so destinations explicitly excluded by operators would still tunnel through the proxy.
  - confidence: 0.93
- **[P2] Use an Axios-compatible request option type:** `extensions/feishu/src/client.ts:58-60`
  - body: `Lark.HttpRequestOptions` does not declare Axios fields such as `httpsAgent` or `proxy`. Keep the public Lark type at the boundary and use a local Axios-backed intersection or adapter before passing options to the SDK's Axios instance.
  - confidence: 0.86
- **[P3] Recompute proxy state for cached clients:** `extensions/feishu/src/client.ts:52`
  - body: The proxy agent is captured when the cached Feishu client is created, but the cache decision only compares credentials, domain, and timeout. If proxy env changes during process lifetime, the reused client keeps stale routing; include proxy state in the cache key or resolve per request.
  - confidence: 0.78
