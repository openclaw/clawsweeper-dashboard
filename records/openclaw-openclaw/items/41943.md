## Summary

Keep open. The PR targets a real Feishu proxy failure, but the branch is now conflicting with current main, lacks required real behavior proof, and forces Axios proxy behavior without a settled Feishu egress policy.

## What This Changes

The PR adds an `HttpsProxyAgent` from proxy environment variables to the Feishu Lark SDK HTTP wrapper and sets Axios `proxy: false` on wrapped requests.

## Best Possible Solution

Rework the Feishu-owned Lark HTTP adapter so it prevents Axios proxy misrouting while preserving timeout/User-Agent behavior, managed Proxyline policy, focused proxy tests, and redacted real proxy proof.

## Reproduction Assessment

Yes at source level: Feishu startup probing reaches `probeFeishu` -> `createFeishuClient` -> the Lark Axios-backed HTTP wrapper while proxy env can be active. I did not run live Feishu credentials through a local proxy in this read-only review.

## Solution Assessment

No. The PR targets the right seam, but it unconditionally forces proxy routing, crosses the Lark SDK option type, conflicts with current Proxyline helper work, and lacks real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Honor proxy eligibility before forcing the agent:** `extensions/feishu/src/client.ts:57-60`
  - body: When any proxy env var exists, every wrapped Lark HTTP request receives `httpsAgent` and `proxy: false`. That bypasses Axios/proxy-from-env target eligibility and can ignore NO_PROXY or managed-proxy policy, so resolve the effective Feishu/Lark URL before setting these options.
  - confidence: 0.93
- **[P2] Keep Axios-only options behind a typed adapter:** `extensions/feishu/src/client.ts:58-60`
  - body: `Lark.HttpRequestOptions` does not declare `httpsAgent` or `proxy`, so assigning those fields directly crosses the SDK type contract. Keep the Lark request options typed and add a narrow Axios-backed handoff type or adapter for Axios-only fields.
  - confidence: 0.9
- **[P2] Use the current proxy helper surface:** `extensions/feishu/src/client.ts:10-17`
  - body: The branch constructs proxy behavior from env directly, but current main routes plugin proxy construction through the shared Proxyline-backed helper and `@openclaw/feishu` no longer owns `https-proxy-agent` in its package metadata. Rebase onto the current helper seam or add valid plugin-owned dependency metadata.
  - confidence: 0.82
- **[P3] Refresh proxy routing when reusing cached clients:** `extensions/feishu/src/client.ts:52`
  - body: `proxyAgent` is captured when the HTTP instance is constructed, while cached clients are reused based on credentials, domain, and timeout only. If proxy env changes during the process lifetime, cached clients keep stale routing; resolve eligibility per request or include proxy state in the cache comparison.
  - confidence: 0.76
