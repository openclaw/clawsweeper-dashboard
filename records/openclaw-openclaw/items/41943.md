## Summary

Keep open: current main still lacks Feishu HTTP HTTPS_PROXY support, but this PR needs changes before merge because it force-adds Axios proxy options without NO_PROXY gating, uses fields outside the Lark request option type, and needs package/test/changelog follow-through.

## What This Changes

This PR imports `https-proxy-agent` into the Feishu client and injects `httpsAgent` plus `proxy: false` into Lark SDK HTTP requests when proxy environment variables are present.

## Best Possible Solution

Port the proxy fix inside the Feishu plugin with target-aware Axios proxy injection, NO_PROXY preservation, focused regression tests, dependency metadata if a direct agent import remains, and the required changelog entry.

## Reproduction Assessment

Yes, source-backed. `openclaw channels status --probe` reaches `probeFeishu`, which calls the Feishu client HTTP wrapper; current main only injects timeouts there, leaving the Axios HTTPS proxy behavior reported in #41820 unhandled.

## Solution Assessment

No. The PR is in the right owner area, but the maintainable fix should resolve the effective Feishu/Lark URL, honor NO_PROXY via the existing plugin-facing helper, and bridge Axios-specific options with a narrow typed adapter.
