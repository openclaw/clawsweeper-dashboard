## Summary

Keep open: the PR targets a real unresolved Feishu HTTP proxy bug, but the current branch is conflicting and still has blocking correctness, type-contract, security-policy, and real-proof gaps.

## What This Changes

The PR modifies `extensions/feishu/src/client.ts` to attach an HTTPS proxy agent and disable Axios proxy handling for Feishu Lark SDK HTTP requests when proxy environment variables are set.

## Best Possible Solution

Rework the Feishu HTTP adapter at the Lark `httpInstance` seam so it preserves timeout/User-Agent behavior, honors target-aware proxy eligibility, aligns with Proxyline-managed egress, adds focused tests, and includes redacted real proxy proof.

## Reproduction Assessment

Yes, source-backed: Feishu probe and token/API calls still flow through the Lark Axios-backed HTTP wrapper on current main, and linked reports provide proxy-env steps plus 400/plain-HTTP-to-HTTPS-port and ECONNRESET failures. I did not run live Feishu credentials through a local proxy in this read-only review.

## Solution Assessment

No. The PR targets the right Feishu-owned seam, but the submitted patch unconditionally forces proxy routing, bypasses target/NO_PROXY decisions, crosses the Lark SDK type contract, and needs a current-main rebase plus real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Honor NO_PROXY before forcing proxy routing:** `extensions/feishu/src/client.ts:57-60`
  - body: When `proxyAgent` exists, every Lark HTTP request gets `httpsAgent` and `proxy: false`, so Axios no longer applies target-based `NO_PROXY` exclusions. Resolve the effective Feishu/Lark target URL and only attach proxy routing when that request should use it.
  - confidence: 0.93
- **[P2] Keep Axios-only options behind a typed adapter:** `extensions/feishu/src/client.ts:58-60`
  - body: `Lark.HttpRequestOptions` does not declare `httpsAgent` or `proxy`, so assigning those fields directly crosses the SDK type contract. Keep the Lark request options typed and add a narrow Axios-backed handoff type or adapter.
  - confidence: 0.9
- **[P2] Use the current proxy helper surface:** `extensions/feishu/src/client.ts:10-17`
  - body: The branch constructs proxy behavior from env directly, but current main routes Feishu proxy construction through the shared Proxyline-backed helper and `@openclaw/feishu` no longer owns an `https-proxy-agent` dependency. Rebase onto the current helper seam or add valid plugin-owned dependency metadata.
  - confidence: 0.82
- **[P3] Refresh proxy routing when reusing cached clients:** `extensions/feishu/src/client.ts:52`
  - body: `proxyAgent` is captured when the HTTP instance is constructed, while cached clients are reused based on credentials, domain, and timeout only. If proxy env changes during the process lifetime, cached clients keep stale routing; resolve eligibility per request or include proxy state in the cache comparison.
  - confidence: 0.76
