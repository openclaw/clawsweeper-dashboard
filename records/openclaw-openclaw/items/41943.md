## Summary

Keep open: the Feishu proxy bug is still source-reproducible on current main, but this stale PR is not merge-ready because it conflicts with current main, lacks real Feishu/proxy proof, and changes egress routing in a way that needs target-aware proxy policy.

## What This Changes

The PR modifies the Feishu client HTTP wrapper to create an HTTPS proxy agent from proxy environment variables and attach `httpsAgent` plus `proxy: false` to Lark SDK HTTP calls.

## Best Possible Solution

Rework the Feishu-owned Lark HTTP adapter so it preserves timeout/User-Agent behavior, avoids Axios double proxy handling, honors the managed Proxyline contract or explicit target eligibility, adds focused tests, and includes redacted real proxy proof.

## Reproduction Assessment

Yes at source level: Feishu startup probe and token/API calls still flow through the Lark Axios-backed HTTP wrapper while proxy env can be active. I did not run live Feishu credentials through a local proxy in this read-only review.

## Solution Assessment

No. The PR targets the right seam, but it unconditionally forces proxy routing, crosses the Lark SDK option type, conflicts with current Proxyline helper work, and lacks real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Honor proxy eligibility before forcing the agent:** `extensions/feishu/src/client.ts:57-60`
  - body: When a proxy env var exists, every Lark HTTP request gets `httpsAgent` and `proxy: false`, so Axios no longer applies target-based `NO_PROXY` exclusions. Resolve the effective Feishu/Lark URL and attach proxy routing only when that request should use it.
  - confidence: 0.93
- **[P2] Keep Axios-only options behind a typed adapter:** `extensions/feishu/src/client.ts:58-60`
  - body: `Lark.HttpRequestOptions` does not declare `httpsAgent` or `proxy`, so assigning those fields directly crosses the SDK type contract and should not pass strict TypeScript unchanged. Keep Lark request options typed and add a narrow Axios-backed handoff type or adapter.
  - confidence: 0.9
- **[P2] Use the current proxy helper surface:** `extensions/feishu/src/client.ts:10-17`
  - body: The branch constructs proxy behavior from env directly, but current main routes plugin proxy construction through the shared Proxyline-backed helper and `@openclaw/feishu` no longer owns an `https-proxy-agent` runtime dependency. Rebase onto the current helper seam or add valid plugin-owned dependency metadata.
  - confidence: 0.82
- **[P3] Refresh proxy routing when reusing cached clients:** `extensions/feishu/src/client.ts:52`
  - body: `proxyAgent` is captured when the HTTP instance is constructed, while cached clients are reused based on credentials, domain, and timeout only. If proxy env changes during the process lifetime, cached clients keep stale routing; resolve eligibility per request or include proxy state in the cache comparison.
  - confidence: 0.76

## Security Review
