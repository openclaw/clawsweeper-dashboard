## What This Changes

The PR clamps memory-core's sqlite-vec widen-fallback KNN `k` value to 4096 and adds regression tests for above- and below-ceiling widened queries.

## Best Possible Solution

Land the narrow clamp with the new regression tests after the contributor adds redacted terminal output, logs, or a linked artifact proving a real >4096-vector sqlite-vec-backed memory_search succeeds after the patch.

## Reproduction Assessment

Yes from source inspection: current main can widen sqlite-vec KNN from `limit * 8` to the full vector table count, and sqlite-vec v0.1.9 rejects k values above 4096. I did not run a live >4096-vector store reproduction in this read-only review.

## Solution Assessment

Yes, the proposed `Math.min` clamp is the narrowest maintainable fix for the dependency ceiling because it preserves the existing fast path and only bounds an invalid sqlite-vec argument. A broader full-scan fallback could be considered later for recall beyond sqlite-vec's ceiling, but it is not required to stop the current failure.
