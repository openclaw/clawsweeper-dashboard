## Summary

Keep open. Current main retries the reported fetch/socket-style embedding transport errors, but it still lacks the exhausted multi-item batch split fallback, and open PR #44167 explicitly closes this issue.

## What This Changes

The issue requests memory reindex embedding batches to retry transient transport errors and split multi-item batches after retry exhaustion instead of aborting the whole sync.

## Best Possible Solution

Land or replace the focused memory-core fix that keeps the broadened transport retry classifier and adds exhausted multi-item split fallback for text and structured embedding batches with regression coverage.

## Reproduction Assessment

Yes. A high-confidence source-level unit reproduction remains: mock a provider so multi-item text or structured batches keep throwing `TypeError("fetch failed")` after retries, and current main still propagates the exhausted error instead of splitting the batch.

## Solution Assessment

Yes. The requested retry-plus-split fallback is the narrowest maintainable repair; current main has the retry-classifier half, so the remaining work should stay in memory-core rather than adding provider branching or config.
