## Summary

Keep open. Current main still has the narrow memory embedding retry classifier and direct throw-on-failure batch path described in the report, and this issue is paired with open PR #44167 using closing syntax, so it should not be closed before that PR or an equivalent fix lands.

## What This Changes

The issue asks memory-core embedding batch retry logic to treat transient transport failures as retryable and recursively split exhausted multi-item batches instead of aborting an entire reindex.

## Best Possible Solution

Review, update, and land #44167 or an equivalent focused memory-core change that classifies narrow transient embedding transport failures as retryable, reuses the existing backoff loop, recursively splits exhausted multi-item text and structured embedding batches while preserving order, and adds targeted regression tests; keep broader rollback/cache recovery in #59137 separate.
