## Summary

Keep open. Current main still loads and watches the personal `~/.agents/skills` source unconditionally, and the strict public `skills.load` config has no source-disable or ignore-dir setting; the closed related issue points back here as the canonical tracker.

## What This Changes

The issue requests a config option to disable scanning `~/.agents/skills` so Claude Code personal skills do not automatically enter OpenClaw skill discovery.

## Best Possible Solution

Add a backward-compatible `skills.load` source opt-out that skips disabled built-in skill roots before discovery and watcher registration while preserving current defaults and precedence when unset.

## Reproduction Assessment

Yes for the described current behavior: source inspection on current main shows personal `~/.agents/skills` is loaded and watched without a source-specific guard, and docs/tests assert that default. This is a feature/config gap rather than a broken existing contract.

## Solution Assessment

Yes in direction, but the exact public config API still needs maintainer choice. A source-level opt-out under `skills.load` is the narrow maintainable path; broader on-demand skill routing should stay separate.
