## What This Changes

The issue requests a config option to disable scanning of the personal Claude Code `~/.agents/skills` directory.

## Best Possible Solution

Add a backward-compatible `skills.load` source opt-out that skips disabled built-in roots before discovery and watcher registration while preserving current defaults and precedence when unset.

## Reproduction Assessment

Yes for the described current behavior: source inspection shows personal `~/.agents/skills` is loaded and watched without a source-specific guard, and tests/docs assert that default. This is a feature/config gap rather than a broken existing contract.

## Solution Assessment

Yes directionally: a source-level opt-out under `skills.load` is the narrow maintainable path. The exact public field shape and disabled-source semantics still need maintainer choice before implementation.
