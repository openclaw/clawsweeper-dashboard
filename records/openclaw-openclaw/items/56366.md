## Summary

Keep open. Current main and v2026.6.1 still load and watch the user-level `.agents/skills` source without a source-disable option, and the closely matching later report was already closed back to this as the canonical tracker.

## What This Changes

Request to add a core configuration mechanism to disable scanning the user-level `.agents/skills` root, and possibly other skill source directories, to avoid cross-tool skill conflicts and prompt bloat.

## Best Possible Solution

Add a narrow, documented `skills.load` source-control mechanism that preserves current defaults, applies consistently to loading and watching, and includes focused schema, docs, and regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows the personal `.agents/skills` source is loaded and watched, while the strict `skills.load` schema has no source-disable option.

## Solution Assessment

Unclear. The requested direction is valid, but maintainers still need to choose the public config shape and how it should interact with existing agent skill allowlists.
