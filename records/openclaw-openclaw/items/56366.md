## Summary

Keep open. Current main and the latest v2026.5.22 release still unconditionally load and watch the personal `~/.agents/skills` source, and the strict public `skills.load` config contract has no source-level opt-out; the closely matching later report was closed in favor of this issue as the canonical tracker.

## What This Changes

The issue requests a configuration option to disable or ignore the personal `~/.agents/skills` skill source to avoid Claude Code skill conflicts and prompt bloat.

## Best Possible Solution

Add a backward-compatible `skills.load` source opt-out that skips disabled built-in roots before discovery and watcher registration, with docs, schema, and tests while preserving current defaults when unset.

## Reproduction Assessment

Yes. Source inspection against current main shows `~/.agents/skills` is loaded and watched unconditionally, and the strict config type/schema has no source-disable field; this reproduces the reported configuration gap without needing a live machine.

## Solution Assessment

Unclear until maintainers choose the public config shape. A source-level opt-out is the narrow maintainable direction because it can prevent discovery, watcher registration, and prompt bloat together.
