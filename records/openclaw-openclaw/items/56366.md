## Summary

Keep open. Current main still loads and watches `~/.agents/skills` without a source-specific opt-out, and the closed related issue explicitly points back here as the canonical tracker; the remaining work is a new public config surface that needs maintainer API judgment.

## What This Changes

Request a config option to stop OpenClaw from scanning `~/.agents/skills` so Claude Code personal skills do not bloat or override OpenClaw skill prompts.

## Best Possible Solution

Add a backward-compatible `skills.load` source opt-out that skips disabled built-in skill roots before discovery and watcher registration while preserving current defaults and precedence when unset.

## Reproduction Assessment

Yes for the current behavior: source inspection on current main shows personal `~/.agents/skills` is loaded and watched without a source-specific guard, and docs/tests assert that default. This is a feature/config gap rather than a broken existing contract.

## Solution Assessment

Yes directionally, but the exact public config API still needs maintainer choice. A source-level opt-out under `skills.load` is the narrow maintainable path; broader on-demand skill routing should stay separate.
