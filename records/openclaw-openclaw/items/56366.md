## Summary

Keep open. Current main still loads and watches the personal `~/.agents/skills` source unconditionally, the strict public `skills.load` config has no source-disable option, and the closed related issue points back here as the canonical tracker.

## What This Changes

The issue requests a configuration option to disable scanning `~/.agents/skills` so Claude Code skill directories do not always get loaded into OpenClaw prompts.

## Best Possible Solution

Add a backward-compatible `skills.load` source opt-out that skips disabled built-in roots before discovery and watcher registration, with schema/help/docs/changelog and focused tests while preserving current defaults.

## Reproduction Assessment

Yes for the reported current behavior: source inspection on current main shows personal `~/.agents/skills` is loaded and watched unconditionally, and tests assert that default. This is a feature/config gap rather than a broken existing contract.

## Solution Assessment

Yes in direction, but the exact public config API still needs maintainer choice. A source-level opt-out under `skills.load` is the narrow maintainable path; broader on-demand skill routing should stay separate.
