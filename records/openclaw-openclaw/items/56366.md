## Summary

Keep open. Current main still loads and watches the personal `~/.agents/skills` source unconditionally, and the public `skills.load` config/schema/docs still lack a source-level opt-out; the closed #64639 report remains a duplicate rather than a fix.

## What This Changes

Request adds a `skills.load` opt-out or ignore mechanism for `~/.agents/skills` scanning so OpenClaw can coexist with Claude Code skill directories.

## Best Possible Solution

Add a backwards-compatible `skills.load` source opt-out, preserve current defaults when unset, and document it with loader, watcher, schema, and regression-test coverage.

## Reproduction Assessment

Yes, by source inspection rather than command execution. Current main has a direct path where a home `.agents/skills` skill is loaded into the prompt and the watcher registers that path without any source-specific config guard.

## Solution Assessment

Yes in direction. A source-level opt-out under `skills.load` is the narrow maintainable solution; broader on-demand skill routing or MCP-router integration would be separate product work.
