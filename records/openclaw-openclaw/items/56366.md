## Summary

Keep open. Current main and the latest release still load and watch the personal `~/.agents/skills` root unconditionally, and the public `skills.load` contract has no source-disable or ignore-dir setting; the related duplicate was closed in favor of this tracker.

## What This Changes

The issue requests a configuration option to disable scanning `~/.agents/skills` so OpenClaw can coexist with Claude Code skill directories without prompt bloat or name conflicts.

## Best Possible Solution

Add a backward-compatible `skills.load` source opt-out that skips disabled built-in skill roots before discovery and watcher registration while preserving current defaults and precedence when unset.

## Reproduction Assessment

Yes for the described current behavior: source inspection shows personal `~/.agents/skills` is loaded and watched without a source-specific guard, and docs/tests assert that default. This is a feature/config gap rather than a broken existing contract.

## Solution Assessment

Yes directionally, but the exact public config API still needs maintainer choice. A source-level opt-out under `skills.load` is the narrow maintainable path; broader on-demand skill routing should remain separate.
