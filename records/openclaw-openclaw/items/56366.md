Keep this open: current main and the latest release still load and watch the personal `.agents/skills` source, and the strict config/docs surface still has no source-level opt-out. ## What This Changes

The issue requests a public option to disable scanning Claude Code's personal `~/.agents/skills` directory so OpenClaw can coexist without skill prompt bloat or source conflicts.

## Best Possible Solution

Choose a documented source-disable policy, then implement it consistently across discovery, watcher registration, status/snapshots, schema/help docs, and focused regression tests while leaving on-demand skill selection as separate design work.

## Reproduction Assessment

Yes. At source level, current `main` still loads and watches `~/.agents/skills`, and the strict `skills.load` config has no source-disable field; I did not run a live filesystem repro because this was a read-only review.

## Solution Assessment

Unclear until maintainers choose the public config/API shape. A backwards-compatible source-disable setting is plausible, but repository policy sets a high bar for new config surfaces and existing allowlists/prompt limits are only partial mitigations.
