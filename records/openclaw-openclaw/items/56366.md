## Summary

Keep open. Current main still loads and watches `~/.agents/skills` unconditionally, the public `skills.load` config has no source-disable option, and the related duplicate at https://github.com/openclaw/openclaw/issues/64639 is closed back to this issue as the canonical request.

## What This Changes

The issue requests a configuration option to skip loading and watching the personal `~/.agents/skills` source, and possibly project `.agents/skills`, to avoid Claude Code skill conflicts and prompt bloat.

## Best Possible Solution

Add a backwards-compatible `skills.load` source opt-out that skips disabled built-in roots before discovery and watcher registration while preserving current defaults and precedence when unset.

## Reproduction Assessment

Yes for the reported current behavior: source inspection on current main shows personal `~/.agents/skills` is loaded and watched unconditionally, and tests assert that default. This is a feature/config gap rather than a broken existing contract.

## Solution Assessment

Yes in direction: a backward-compatible source-level opt-out under `skills.load` is the narrow maintainable fix. The broader on-demand skill-selection idea should remain separate.
