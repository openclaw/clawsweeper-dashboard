## Summary

Keep open. Current main still loads and watches the personal `~/.agents/skills` source unconditionally, while the public `skills.load` config only exposes `extraDirs`, `watch`, and `watchDebounceMs`; the closed #64639 thread is a duplicate, so this remains the canonical narrow feature/config request.

## What This Changes

Request adding a config option to disable default `.agents/skills` source scanning, especially `~/.agents/skills`, to avoid Claude Code coexistence conflicts and prompt bloat.

## Best Possible Solution

Add a backwards-compatible source opt-out under `skills.load`, apply it before filesystem discovery and watcher registration, preserve current defaults when unset, and document the new behavior with targeted regression coverage.

## Reproduction Assessment

Yes. Source inspection and existing tests give a high-confidence reproduction path: a mocked home `.agents/skills` skill is loaded into the prompt, and the watcher registers the home `.agents/skills` path without a source-specific config guard.

## Solution Assessment

Yes in direction. A source-level opt-out is the narrow maintainable fix for this coexistence problem, and it avoids folding in broader on-demand skill routing or MCP skill-router design work.
