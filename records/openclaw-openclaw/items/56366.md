## Summary

Keep open. Current main still hardcodes the personal `~/.agents/skills` source in both skill discovery and watcher setup, while the strict public config/docs have no source-level opt-out; the closely matching later report was closed in favor of this issue as the canonical tracker.

## What This Changes

Requests a `skills.load` option to disable or ignore the personal `~/.agents/skills` scan for Claude Code coexistence and prompt-context control.

## Best Possible Solution

Add a backward-compatible `skills.load` source opt-out that skips disabled built-in skill roots before discovery and watcher registration, with docs/schema/tests and current defaults unchanged when unset.

## Reproduction Assessment

Yes for the current behavior: source inspection against current main shows `~/.agents/skills` is loaded and watched unconditionally, and docs/tests assert it as a built-in root. This reproduces the configuration gap, not a broken existing contract.

## Solution Assessment

Yes directionally: a source-level `skills.load` opt-out is the narrow maintainable fix. The exact public field shape and disabled-source semantics still need maintainer choice before implementation.
