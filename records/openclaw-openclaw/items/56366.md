## Summary

Keep open. Current main still loads and watches `~/.agents/skills` without a source-specific opt-out, and the public `skills.load` contract has no disable/ignore setting; the closed related issue points back here as the canonical tracker. The remaining work is a new public config option, so maintainers should choose the API shape and semantics before implementation.

## What This Changes

The issue requests a `skills.load` option to disable automatic scanning of `~/.agents/skills` for users who share that directory with Claude Code.

## Best Possible Solution

Add a backward-compatible `skills.load` source opt-out that skips disabled built-in skill roots before discovery and watcher registration while preserving current defaults and precedence when unset.

## Reproduction Assessment

Yes for the described current behavior: source inspection on current main shows personal `~/.agents/skills` is loaded and watched without a source-specific guard, and docs/tests assert that default. This is a feature/config gap rather than a broken existing contract.

## Solution Assessment

Yes directionally, but the exact public config API still needs maintainer choice. A source-level opt-out under `skills.load` is the narrow maintainable path; broader on-demand skill routing should remain separate.
