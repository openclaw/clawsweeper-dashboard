## What This Changes

Request an OpenClaw configuration or source-level opt-out to stop scanning and watching `~/.agents/skills` for users who share a machine with Claude Code skills.

## Best Possible Solution

Maintainers should choose a backward-compatible skill-source opt-out contract, then implement it consistently across discovery, watcher registration, config schema/help, docs, status behavior, and focused regression tests.

## Reproduction Assessment

## Solution Assessment

Unclear until maintainers choose the public API shape. A generic source-disable setting is plausible, but OpenClaw policy sets a high bar for new config surface and the implementation must cover discovery, watching, docs, and tests together.
