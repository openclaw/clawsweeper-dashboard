## Summary

Keep open. Current main still loads and watches the personal `~/.agents/skills` source unconditionally, the public `skills.load` schema has no source-disable option, and the closed duplicate at https://github.com/openclaw/openclaw/issues/64639 points back to this issue as the canonical request.

## What This Changes

Request a backwards-compatible `skills.load` option to disable scanning and watching `~/.agents/skills`, and possibly other built-in skill source directories.

## Best Possible Solution

Add a backwards-compatible source opt-out under `skills.load`, apply it before filesystem discovery and watcher registration, preserve current defaults when unset, and keep native on-demand skill selection as a separate design item.

## Reproduction Assessment

Yes for the reported current behavior: source inspection and existing tests show the home `.agents/skills` root is loaded into the prompt and watched without a source-specific config guard. This is a feature/config gap rather than a broken existing contract.

## Solution Assessment

Yes in direction. A backwards-compatible source-level opt-out under `skills.load` is the narrow maintainable fix for this coexistence problem; native on-demand skill routing should remain separate.
