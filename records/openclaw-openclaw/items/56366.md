## Summary

Keep open. Current main still unconditionally loads and watches the personal `~/.agents/skills` source, the public `skills.load` schema has no source opt-out, and the closed #64639 thread is a duplicate rather than a replacement fix.

## What This Changes

Add a backwards-compatible `skills.load` option to disable or ignore `~/.agents/skills` scanning for Claude Code coexistence.

## Best Possible Solution

Add a source-level opt-out under `skills.load`, apply it before filesystem discovery and watcher registration, preserve current defaults when unset, and document it with focused regression coverage.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: current main always resolves and loads home `.agents/skills`, and the watcher always registers that path. Existing tests also assert that a mocked personal AgentSkills directory is loaded into the prompt.

## Solution Assessment

Yes in direction. A source-level `skills.load` opt-out applied in both the loader and watcher is the narrow maintainable fix, and it avoids broad on-demand skill routing or per-skill disable churn.
