## Summary

Keep open. Current main still loads and watches personal AgentSkills from the home `.agents/skills` directory without a source-specific opt-out, while the public config schema only supports extra directories and watcher settings; the closed #64639 thread is duplicate context, so this remains the canonical narrow request.

## What This Changes

The issue requests a `skills.load` option to disable scanning personal AgentSkills directories such as `~/.agents/skills` for Claude Code coexistence and prompt-size control.

## Best Possible Solution

Add a backward-compatible `skills.load` source opt-out, apply it before filesystem discovery and watcher registration, keep default scanning unchanged, and update schema/help/docs/changelog with regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: create a home `.agents/skills/<name>/SKILL.md` and build a workspace skills prompt, or start the skills watcher, and current main will include/watch that source without a source-specific guard.

## Solution Assessment

Yes in direction. A source-level opt-out under `skills.load` is the narrow maintainable fix; broader on-demand skill routing or MCP skill-router behavior should not be folded into this issue.
