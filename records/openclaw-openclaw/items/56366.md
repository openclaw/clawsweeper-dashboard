## Summary

Keep open. Current main still loads and watches the personal `~/.agents/skills` source unconditionally, while `skills.load` only exposes `extraDirs`, `watch`, and `watchDebounceMs`; the closed #64639 item is duplicate context, so this remains the canonical narrow config request.

## What This Changes

Add a configuration option to disable scanning personal or project `.agents/skills` sources so coexistence with Claude Code skill directories does not bloat or conflict with OpenClaw prompts.

## Best Possible Solution

Add a backwards-compatible source opt-out under `skills.load`, apply it before both filesystem discovery and watcher registration, preserve default behavior when unset, and cover it with focused docs and regression tests.

## Reproduction Assessment

Yes for the current limitation. Source inspection on current main shows the personal `.agents/skills` directory is loaded and watched without a source-specific guard, and existing tests assert that behavior.

## Solution Assessment

Yes in direction. A source-level opt-out is the narrow maintainable fix; broader on-demand skill routing or MCP skill-router behavior should stay out of this focused change.
