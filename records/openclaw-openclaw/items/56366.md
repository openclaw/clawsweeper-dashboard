## Summary

Keep open. Current main and the latest release still load and watch the personal `~/.agents/skills` root unconditionally, and the related later report was closed in favor of this canonical tracker; the remaining work is a public skills-load config decision, not stale cleanup.

## What This Changes

Request a config option to disable scanning `~/.agents/skills` so OpenClaw can coexist with Claude Code or other local skill managers without prompt bloat or source conflicts.

## Best Possible Solution

Add a backward-compatible `skills.load` source opt-out that skips disabled built-in roots before discovery and watcher registration while preserving current defaults and precedence when unset.

## Reproduction Assessment

Yes for the described current behavior: source inspection shows personal `~/.agents/skills` is loaded and watched without a source-specific guard, and docs/tests assert that default. This is a feature/config gap rather than a broken existing contract.

## Solution Assessment

Yes, directionally: a source-level opt-out under `skills.load` is the narrow maintainable path. The exact public field shape and disabled-source semantics still need maintainer choice before implementation.
