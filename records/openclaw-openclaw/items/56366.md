## Summary

Keep open. Current main still loads and watches the personal `~/.agents/skills` source unconditionally, the strict public `skills.load` config has no source-disable option, and the closed duplicate at https://github.com/openclaw/openclaw/issues/64639 points back to this issue as the canonical narrow request.

## What This Changes

Request a `skills.load` option to disable scanning the personal `~/.agents/skills` AgentSkills root so Claude Code skills are not automatically loaded into OpenClaw prompts.

## Best Possible Solution

Add a backwards-compatible source opt-out under `skills.load`, apply it before filesystem discovery and watcher registration, preserve the current defaults when unset, and update schema, help, docs, changelog, and focused regression tests.

## Reproduction Assessment

Yes for the reported current behavior: source inspection and existing tests show the home `.agents/skills` root is loaded into the prompt and watched without a source-specific config guard. This is a feature/config gap rather than a broken existing contract.

## Solution Assessment

Yes in direction. A backward-compatible source-level opt-out under `skills.load` is the narrow maintainable fix for this coexistence problem; native on-demand skill routing should remain a separate design item.
