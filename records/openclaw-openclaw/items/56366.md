## Summary

Keep open. Current main still hardcodes personal `~/.agents/skills` discovery and watcher registration, and the strict public `skills.load` config has no source-disable or ignore-dir setting; the related closed issue points back here as the canonical tracker.

## What This Changes

Request to add a configuration option that disables or ignores `~/.agents/skills` scanning so OpenClaw can coexist with Claude Code skill directories without prompt bloat or skill conflicts.

## Best Possible Solution

Add a backward-compatible `skills.load` source opt-out that skips disabled built-in skill roots before discovery and watcher registration while preserving current defaults and precedence when unset.

## Reproduction Assessment

Yes for the described current behavior: source inspection on current main shows personal `~/.agents/skills` is loaded and watched without a source-specific guard, and tests/docs assert that default. This is a feature/config gap rather than a broken existing contract.

## Solution Assessment

Yes in direction, but the exact public config API still needs maintainer choice. A source-level opt-out under `skills.load` is the narrow maintainable path; broader on-demand skill routing should stay separate.
