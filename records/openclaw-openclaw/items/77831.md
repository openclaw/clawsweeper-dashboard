## What This Changes

The issue asks memory-core dreaming to stop treating internal session-corpus files as promotable memory and retrieval input.

## Best Possible Solution

Land or replace the focused fix in https://github.com/openclaw/openclaw/pull/78130 so session-corpus remains internal dreaming evidence and cannot enter recall ranking, promotion, or memory-search pollution.

## Reproduction Assessment

Yes, source-reproducible: current main writes `memory/.dreams/session-corpus/<day>.txt`, records collected batch results with `recordShortTermRecalls()`, and treats those paths as short-term memory. I did not run a multi-day live dreaming cron reproduction.

## Solution Assessment

Yes for the direction: the narrow maintainable fix is to prevent session-corpus artifacts from entering or ranking as promotion candidates while preserving them as internal phase evidence. A new user exclude-path config would be a weaker workaround for a plugin-owned invariant.
