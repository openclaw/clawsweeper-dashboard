## Summary

Keep open: current main still feeds dreaming session-corpus paths into short-term recall and promotion candidates, while the existing contamination filters only cover some raw REM/narrative noise patterns.

## What This Changes

The issue asks OpenClaw to stop memory-core dreaming session-corpus artifacts from polluting short-term recall, vector search, and `MEMORY.md` promotions.

## Best Possible Solution

Define session-corpus as internal dreaming evidence unless explicitly promoted through a grounded, non-derived path, then prune existing session-corpus short-term entries and cover the boundary in memory-core tests and docs.

## Reproduction Assessment

Yes from source inspection, but not by running a multi-day live cron: current main writes session corpus under `memory/.dreams`, records it into short-term recall, and tests expect it in promotion candidates.

## Solution Assessment

Unclear. Excluding all session-corpus paths is a plausible fix for the pollution, but it conflicts with current docs/tests that treat redacted transcript ingestion as part of dreaming, so the maintainer decision should come first.
