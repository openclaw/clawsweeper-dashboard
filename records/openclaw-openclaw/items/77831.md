## What This Changes

The issue asks memory-core dreaming to keep generated session-corpus artifacts out of memory indexing, short-term recall promotion, and durable `MEMORY.md` output.

## Best Possible Solution

Review and revise or land https://github.com/openclaw/openclaw/pull/78130, or a replacement fix, so session-corpus can support internal dreaming analysis without entering promotion candidates, durable memory, or user-facing recall indexes.
