## Summary

Keep open: current main still filters the sidebar Recent list to the selected agent and excludes `subagent`/`spawnedBy` rows, while the overlapping open issue and PR do not safely cover the distinct cross-agent sidebar discovery request. The remaining work is valid but needs maintainer UX/security direction because exposing internal subagent transcripts changes the current hidden-by-default behavior.

## What This Changes

Request Web UI discovery of existing child-spawned subagent sessions across agent scopes in the Recent Sessions sidebar.

## Best Possible Solution

Add an explicit, default-safe cross-agent/subagent visibility path for operators, preserving current hidden-by-default internal rows and covering both gateway row selection and sidebar/chat picker presentation.

## Reproduction Assessment

Yes, at source level: scoped UI refreshes call `sessions.list` with `agentId`, `filterSessionEntries` keeps only matching agent ids, and the sidebar also drops subagent or `spawnedBy` rows. I did not run a live browser session.

## Solution Assessment

Unclear pending maintainer product judgment. A default-off all-agent/show-subagents mode is a plausible narrow fix; unconditionally removing the current filters would be the wrong shape because tests and prior PRs intentionally hide internal sessions by default.
