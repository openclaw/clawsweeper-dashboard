## Summary

Keep open: current main and the latest release still scope the sidebar/session data to the selected agent and hide `subagent`/`spawnedBy` rows. The related open operator-reveal issue and session-kind-filter PR only partially cover this distinct cross-agent sidebar discovery request, and the remaining fix needs maintainer UX/security direction.

## What This Changes

Request to make child-spawned subagent sessions discoverable in the Web UI sidebar across agent scopes, likely through an All Agents or Show spawned sessions control.

## Best Possible Solution

Add a default-safe, explicit cross-agent/subagent visibility path for operators that preserves hidden-by-default internal rows, distinguishes spawned sessions clearly, and covers both sidebar Recent Sessions and picker navigation semantics.

## Reproduction Assessment

Yes, at source level: current main scopes UI `sessions.list` requests by agent, the gateway filters rows by that agent id, and the sidebar also filters out subagent/spawned rows. I did not run a live browser session.

## Solution Assessment

Unclear pending maintainer product judgment. A default-off All Agents or Show spawned sessions mode is plausible; unconditionally removing the filters would conflict with current hidden-by-default tests and safe-default expectations.
