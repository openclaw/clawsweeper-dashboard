## Summary

Keep open: current main still excludes subagent/spawned rows from the Web UI Recent Sessions sidebar and also scopes chat session loads to the active agent, so the reported cross-agent child-session discovery gap remains. The earlier merged work intentionally fixed the opposite problem by filtering Recent Sessions to the selected agent, so this needs a maintainer UX decision rather than cleanup closure.

## What This Changes

The issue requests a Web UI way to discover existing child-spawned subagent sessions across agent scopes, such as an All Agents or Show spawned sessions mode.

## Best Possible Solution

Preserve the selected-agent Recent Sessions default, then add a deliberate cross-agent spawned-session discovery affordance with focused UI and gateway-session tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: the UI requests an active-agent session list, the gateway filters by that agent id, and the sidebar separately filters out subagent/spawned rows.

## Solution Assessment

No, not as a simple removal of filtering. The maintainable direction is an explicit All Agents or spawned-sessions view so current selected-agent behavior does not regress.
