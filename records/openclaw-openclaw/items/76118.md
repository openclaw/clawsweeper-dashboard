## Summary

Keep open. Current main still batches config and auth-store SecretRef assignments together, while the latest PR head provides the focused runtime split, regression coverage, changelog entry, and green exact-head checks needed for review/merge.

## What This Changes

The PR separates config SecretRef resolution from auth-store SecretRef resolution so stale auth-profile provider references degrade to profile-unavailable warnings without aborting gateway startup.

## Best Possible Solution

Land this focused PR after required owner review, keeping config SecretRef failures fail-fast while degrading only auth-store provider-resolution failures to targeted profile-unavailable warnings.

## Reproduction Assessment

Yes. The linked issue steps map directly to current main's shared resolver batch for config and auth-store assignments, and the PR adds focused tests for the stale-provider and valid-sibling cases.

## Solution Assessment

Yes. Splitting the assignment groups inside `prepareSecretsRuntimeSnapshot` is the narrowest maintainable fix because it preserves required config failure behavior while isolating generated auth-store provider failures.
