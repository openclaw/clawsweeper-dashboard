## Summary

Keep open. Current main still has the startup-abort behavior, while the latest PR head addresses the earlier review blockers by catching only auth-profile provider-resolution failures and adding the required changelog entry.

## What This Changes

The PR splits config SecretRef resolution from auth-profile SecretRef resolution, downgrades stale auth-profile provider refs to a warning, adds regression coverage, and documents the user-facing gateway startup fix in the changelog.

## Best Possible Solution

Land the scoped PR after exact-head CI and maintainer review, keeping stale auth-profile provider refs profile-scoped while preserving fail-fast behavior for config SecretRefs and missing credential values.

## Reproduction Assessment

Yes. Current main can reproduce the linked bug by preparing a startup secrets snapshot with an auth-store api_key or token profile whose keyRef/tokenRef points at an unconfigured exec provider; the batch resolver throws before activation can continue.

## Solution Assessment

Yes, based on the latest diff. Splitting config and auth-store resolution is a narrow maintainable fix, and narrowing the catch to SecretProviderResolutionError avoids hiding missing env/file secret values that should still fail visibly.
