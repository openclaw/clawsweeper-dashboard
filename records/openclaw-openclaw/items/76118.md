## Summary

Keep open for normal PR review and landing. The reported startup-abort path is present on current main, and this PR is a narrow fix with focused regression coverage, a changelog entry, and no blocking correctness or security findings found in review.

## What This Changes

The PR separates config and auth-store SecretRef resolution so stale auth-profile provider refs become warnings, while config SecretRef failures still abort startup.

## Best Possible Solution

Land a scoped fix that preserves strict startup failure for required config secrets while isolating stale generated auth-profile provider refs to the affected profile with an operator-visible warning.

## Reproduction Assessment

Yes. The linked issue’s steps map directly to current main: a stale auth-profile exec provider is collected into the same resolver batch as config assignments, and the startup activator wraps the thrown provider error into the required-secrets abort.

## Solution Assessment

Yes. The proposed split is the narrow maintainable fix because it changes only auth-store resolution error handling, keeps config assignments strict, preserves the lazy prepare boundary, and adds regression tests for both degraded and still-valid sibling profiles.
