## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Clear stale multipart headers after conversion:** `src/infra/net/proxy-fetch.ts:45`
  - body: `normalizeInitForUndici()` rebuilds the multipart encoder but returns the caller's headers unchanged. If `content-type` or `content-length` came from the original FormData body, Undici will serialize the new body with a different boundary while sending stale framing headers, so multipart endpoints can still reject the upload. Clear those headers when the body is replaced, matching the existing runtime normalizer.
  - confidence: 0.82
- **[P3] Preserve explicit empty filenames:** `src/infra/net/proxy-fetch.ts:42`
  - body: `blobValue.name || undefined` drops an intentional empty filename when copying `File` entries. Use nullish handling or the same filename rule as the shared runtime normalizer so the conversion does not silently change a valid `File.name` value.
  - confidence: 0.76
