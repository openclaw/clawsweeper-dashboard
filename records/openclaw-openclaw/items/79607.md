## Summary

Keep open: current main still lacks the user-facing `openclaw agent --session-key` selector and has no identity-anchor config that can route voice, hooks, CLI injection, and channel messages into one canonical user session. The open CLI PR at https://github.com/openclaw/openclaw/pull/35241 covers only the first slice and does not close or implement the broader identity-routing request.

## What This Changes

The issue requests identity-based session unification across channels and programmatic inputs, including a CLI session-key override and a config-backed identity-to-session anchor.

## Best Possible Solution

Design an additive identity/session-alias contract that extends current `dmScope` and `identityLinks`, exposes a validated CLI/API selector, and preserves isolated defaults unless the operator explicitly anchors an identity.

## Reproduction Assessment

Not applicable: this is a feature request rather than a bug against an existing contract. Source and docs inspection on current main gives high confidence that the requested CLI selector and identity-anchor config do not exist today.

## Solution Assessment

Unclear as a final implementation: a CLI `--session-key` flag is a narrow useful slice, but identity-wide routing needs a maintainer-approved contract for allowed targets, validation, privacy documentation, and how voice, hooks, CLI injection, and channel routing interact.
