## Summary

Keep open: current main and v2026.6.10 still probe the deterministic Claude project transcript before persisting a claude-cli binding and clear the binding on a miss, so the reported warm-stdin continuity failure is not fixed or superseded by the related open fingerprint PR.

## What This Changes

Fix claude-cli warm-stdin continuity so a post-turn transcript probe miss does not wipe valid session bindings and prevent OpenClaw transcript reseed.

## Best Possible Solution

Keep this issue open and fix the claude-cli live-session binding boundary so warm-stdin validates against the artifact that actually proves continuity, or makes pure flush-probe misses non-destructive while preserving real invalidation clears.

## Reproduction Assessment

## Solution Assessment

Yes, the best fix is in the claude-cli session-binding/probe boundary rather than broad channel history or model fallback changes. The implementation should be backend-aware or non-destructive for pure probe misses without weakening explicit invalidation boundaries.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/96564. Current main treats a claude-cli post-turn transcript probe miss as an unflushed session and emits clearCliSessionBinding, which breaks warm-stdin claude-cli continuity when no native ~/.claude/projects/<workspace>/<sid>.jsonl is written. Keep the fix narrow to the claude-cli binding/probe/reseed boundary: either make the warm-stdin/live-session probe validate the OpenClaw transcript artifact that actually proves the turn was captured, or make a pure flush-probe miss non-destructive while preserving explicit auth-profile, auth-epoch, system-prompt, cwd, mcp, session-expired, and orphan-tool-use invalidation behavior. Do not add config flags, do not broaden raw reseed across auth boundaries, and do not change unrelated channel/session lifecycle policy such as https://github.com/openclaw/openclaw/issues/84593. Add focused regression coverage for a claude-cli live-session turn whose native Claude transcript is absent but OpenClaw has captured the assistant turn, and include live/package proof or clearly documented closest proof for Claude CLI warm stdin.
