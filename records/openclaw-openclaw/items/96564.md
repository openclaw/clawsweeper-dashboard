## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/96841 as the canonical repair, preserving safe invalidation while keeping warm-stdin continuity through raw transcript reseed.

## Reproduction Assessment

Yes, at source level: current main clears the binding on bindingFlushOk=false and then returns undefined on the next missing-transcript lookup, which prevents raw transcript reseed from seeing the candidate session id. I did not run a live claude-cli warm-stdin session in this read-only review.

## Solution Assessment

Yes: the best current path is a two-part fix that preserves the warm-stdin binding after a post-turn probe miss and still forwards the missing-transcript candidate to prepare so reseed can recover. Binding-persistence-only PRs appear too narrow because the read-side candidate can still be dropped.
