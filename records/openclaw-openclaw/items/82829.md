## Summary

Keep open: the reported generated-file indentation root cause is not supported, but current main has a narrow source-level path that can still treat `queuedFinal` as enough to suppress Telegram group/forum fallback even when no delivery was observed.

## What This Changes

The issue reports Telegram forum-topic replies being processed by the agent but never sent back to Telegram because the dispatcher `deliver` callback is allegedly not wired in the generated bot bundle.

## Best Possible Solution

Restore source-level forum/group no-visible-delivery handling so `queuedFinal` alone does not suppress fallback or count as a visible final response, and add the focused forum-topic regression test back under Telegram dispatch tests.

## Reproduction Assessment

Yes at source level: a dispatcher mock that returns `queuedFinal: true` without invoking `dispatcherOptions.deliver` in a forum-topic group context follows current main into the `!queuedFinal` fallback guard and `queuedFinal` final-response accounting. I did not run a live Telegram forum-topic proof.

## Solution Assessment

No: editing generated `dist/bot-*.js` indentation is not the best fix. The maintainable fix belongs in `extensions/telegram/src/bot-message-dispatch.ts`, with regression coverage that proves a queued final without observed Telegram delivery still produces a visible forum-topic fallback or an explicit suppression path.
