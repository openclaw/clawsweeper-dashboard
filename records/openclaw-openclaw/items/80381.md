## What This Changes

The issue reports that Docker `scripts/docker/setup.sh` fails on Windows/WSL with `EACCES: permission denied, mkdir '/mnt/c'` during setup-time gateway config sync on v2026.5.3 through v2026.5.7.

## Best Possible Solution

Make setup-time Docker config commands explicitly resolve container-local config/state/home paths while leaving `/mnt/c/...` only as Compose bind-mount sources, then cover that with WSL-style regression proof.

## Reproduction Assessment

No, not from a live current-main run. Source inspection still matches the reported failure point: setup reaches `sync_gateway_config`, invokes container CLI config writes before startup, and config writes mkdir the resolved config directory, so a leaked `/mnt/c/...` path would produce the observed error.

## Solution Assessment

No; the current implementation is not the best final state because setup-time config writes still rely on the prestart container's resolved environment. The safer fix is explicit container-local path handling for those commands plus WSL regression coverage.
