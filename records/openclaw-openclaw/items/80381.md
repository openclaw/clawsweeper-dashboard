## What This Changes

Report that Docker `scripts/docker/setup.sh` fails on Windows/WSL with `EACCES: permission denied, mkdir '/mnt/c'` during setup-time gateway config sync when config/workspace dirs are under `/mnt/c/...`.

## Best Possible Solution

Make setup-time Docker config commands invariant to host-style mount paths by forcing container-local config/state/home values for prestart CLI invocations, then cover the WSL `/mnt/c` case with focused regression proof.

## Reproduction Assessment

No, not with high-confidence live reproduction. Source inspection matches the reported failure point and explains how a leaked `/mnt/c` config path would fail, but a real Windows/WSL Docker setup smoke is still needed.

## Solution Assessment

No; current main is not enough to close because setup-time config writes still need runtime proof that host mount paths cannot reach the in-container config writer. The maintainable fix is a narrow prestart environment/path contract plus WSL-style regression coverage.
