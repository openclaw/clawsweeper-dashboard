## What This Changes

The issue reports a Windows/WSL Docker setup regression where `scripts/docker/setup.sh` fails during gateway config sync with `EACCES: permission denied, mkdir '/mnt/c'`.

## Best Possible Solution

Make setup-time Docker config commands explicitly use container-local config/state/home paths while leaving `/mnt/c/...` only as the Compose bind-mount source, then cover that with a WSL-style regression test.

## Reproduction Assessment

No, not with high confidence from a live run. Source inspection matches the reported failure point and shows how a leaked `/mnt/c` config path would fail, but a real WSL/Docker setup smoke is still needed.

## Solution Assessment

No; the current implementation is not the best final state because setup-time config writes still need to be invariant to host-style mount paths. The safer fix is an explicit prestart container environment plus WSL-style regression coverage.

## Repair Work Prompt

Fix the Windows/WSL Docker setup regression reported at https://github.com/openclaw/openclaw/issues/80381. The failure happens after `==> Docker gateway defaults` when `scripts/docker/setup.sh` runs prestart `openclaw-gateway` config get/set and the in-container config path appears to resolve to `/mnt/c/...`, producing `EACCES: permission denied, mkdir '/mnt/c'`. Keep the existing Docker/compose host-path isolation intent from https://github.com/openclaw/openclaw/issues/77436: host `OPENCLAW_CONFIG_DIR` and `OPENCLAW_WORKSPACE_DIR` may be Compose bind-mount sources, but runtime/prestart OpenClaw commands inside the container must use `/home/node/.openclaw` and `/home/node/.openclaw/workspace` or an explicit container-local config path/state dir. Add focused regression coverage in `src/docker-setup.e2e.test.ts` using WSL-style `/mnt/c/...` host paths and ensuring setup-time config commands cannot inherit host-style config/state paths. Do not change package manager/runtime policy, Podman behavior, or unrelated Docker security settings. Add a changelog entry if the fix is user-visible. Validate with the Docker setup test and a WSL/Docker setup smoke when available.
