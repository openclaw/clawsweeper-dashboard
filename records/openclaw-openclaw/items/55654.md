## Summary

Keep open. Current main still lacks an agent-facing `feishu_bitable_delete_record` tool; the only record-delete call is internal cleanup after creating a new Bitable app, and related implementation PRs are open or closed-unmerged rather than landed.

## What This Changes

Add an agent-facing `feishu_bitable_delete_record` tool to the bundled Feishu Bitable tool set for deleting a record by `app_token`, `table_id`, and `record_id`.

## Best Possible Solution

Keep this as the canonical tracker until a Feishu-owned implementation lands: add a typed delete-record helper in `extensions/feishu/src/bitable.ts`, register it through the existing Bitable tool helper, preserve account selection, return a structured success result, and cover registration/routing/error behavior with focused Feishu plugin tests plus a user-facing changelog entry.

## Reproduction Assessment

Yes. On current main, source inspection shows no `feishu_bitable_delete_record` registration, while `appTableRecord.delete` exists only inside create-app cleanup fallback logic; no live Feishu account is needed to reproduce the missing tool surface.

Yes. The narrow maintainable fix is an additive Feishu Bitable plugin tool following the existing get/create/update patterns; because #40697 and #41368 are already open implementation candidates, the best path is maintainer review or consolidation of those PRs rather than a new repair lane.
