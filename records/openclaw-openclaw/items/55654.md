## Summary

Close as ClawHub/plugin-surface work. The requested delete-record capability is still absent from current main and the latest release, but it is optional Feishu plugin tool expansion that current OpenClaw plugin APIs and the @openclaw/feishu npm/ClawHub path can carry without a missing core API.

## What This Changes

Add a `feishu_bitable_delete_record` agent tool for deleting one Feishu Bitable record through the Feishu record-delete API.

## Best Possible Solution

Move delete-record support to a Feishu/ClawHub plugin or a maintainer-sponsored official plugin update with explicit irreversible-delete policy and proof, leaving OpenClaw core closed unless a missing plugin API is identified.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for source-level verification: current main and `v2026.6.8` enumerate the Feishu Bitable tools and omit `feishu_bitable_delete_record`, while delete calls are internal cleanup only.

## Solution Assessment

Is this the best way to solve the issue? No as an OpenClaw core tracker; the maintainable path is Feishu plugin/ClawHub ownership unless maintainers explicitly sponsor the destructive tool in the official plugin.
