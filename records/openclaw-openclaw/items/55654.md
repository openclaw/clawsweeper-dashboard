## Summary

Keep open. Current main still does not expose an agent-facing `feishu_bitable_delete_record` tool, and the only Bitable record deletion calls are internal cleanup after creating a new Bitable app. The related implementation PRs remain open, so this issue should stay as the tracker while maintainers choose the canonical Feishu plugin path.

## What This Changes

The issue requests adding a Feishu Bitable `delete_record` tool that deletes one record through Feishu's record-delete API.

## Best Possible Solution

Add the delete-record capability in the bundled Feishu plugin using the existing Bitable registration and account-routing helper, call the typed Lark SDK delete method with `app_token`, `table_id`, and `record_id`, return a structured success result, and add focused Feishu plugin tests for registration, routing, schema, SDK parameters, and error propagation.
