## Summary

Keep open: current main and latest v2026.6.6 still reject per-session Talk realtime instructions, and the issue discussion includes live installed-package validation; this remains an additive Talk API/product-boundary decision rather than stale cleanup.

## What This Changes

Request to let `talk.client.create` and `talk.session.create` accept bounded per-session realtime context/instructions and merge it with the Gateway-owned realtime prompt.

## Best Possible Solution

Add an additive, bounded per-session context field only after maintainers decide the caller trust model, preserving OpenClaw's base realtime prompt and configured operator instructions.

## Reproduction Assessment

Yes. Current source validation rejects request-time `instructions` because both Talk create schemas use `additionalProperties: false` and omit the requested field; the reporter also live-validated the same failure on 2026.5.22.

## Solution Assessment

Unclear. The requested direction is plausible, but the best fix needs a maintainer decision on whether this is caller-supplied instructions or bounded session context and how it is merged safely.
