## Summary

Keep open: current main and v2026.5.18 still reject per-session Talk realtime instructions, and this is a real additive protocol capability that needs a maintainer decision before implementation.

## What This Changes

Request to add a bounded per-session instructions or context parameter to Talk client/session creation and merge it into realtime gateway/browser session instructions.

## Best Possible Solution

Define an additive bounded per-session context field, append or quote it after OpenClaw's built-in and configured realtime instructions, and update protocol docs plus focused schema/handler tests.

## Reproduction Assessment

Yes, from source inspection: the create schemas disallow additional properties and tests pin request-time instruction override rejection. I did not run a live Gateway RPC because the validator and shipped-tag evidence are direct.

## Solution Assessment

Unclear until maintainers decide the prompt-policy boundary. The safer path is a bounded context field appended to the built-in plus configured realtime prompt, not replacement of configured instructions.
