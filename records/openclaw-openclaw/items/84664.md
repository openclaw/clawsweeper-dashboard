## What This Changes

The issue requests an additive Talk API field for per-session realtime context on gateway relay and browser/client Talk session creation.

## Best Possible Solution

Add a bounded, documented per-session context/instructions field only after maintainers decide how it composes with configured realtime instructions and what callers are trusted to supply.

## Reproduction Assessment

Yes. Source inspection on current main shows both Talk create schemas use additionalProperties: false and omit a request-time instructions field, matching the reporter's live gateway validation.

## Solution Assessment

Unclear. A bounded per-session context field is plausible, but maintainers need to decide the prompt trust boundary, length limits, and merge order before this is the best durable API shape.
