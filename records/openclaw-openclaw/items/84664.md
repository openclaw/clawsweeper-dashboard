## Summary

Keep open: current main and latest v2026.6.8 still reject request-time Talk `instructions`, while the realtime handlers assemble provider instructions only from configured Gateway state. This remains a focused additive Talk API request that needs maintainer product and security decisions before implementation.

## What This Changes

Allow `talk.client.create` and `talk.session.create` to accept bounded per-session realtime context or instructions for Gateway relay sessions.

## Best Possible Solution

Add an additive, bounded per-session context contract at the Talk protocol boundary after maintainers decide trust, merge order, validation, redaction, and how it combines with Gateway-owned base/configured realtime instructions.

## Reproduction Assessment

Yes. Source inspection shows current main rejects the requested fields at schema validation and the reporter provided live Gateway failures on installed releases.

## Solution Assessment

Unclear pending maintainer decision. The requested direction is plausible, but the maintainable fix should be a bounded context/merge contract rather than an unreviewed raw instruction override.
