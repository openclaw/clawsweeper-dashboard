## Summary

Keep open: the patch is a narrow, plausible fix for the Anthropic OAuth callback-host mismatch, but it changes auth-provider compatibility behavior and still lacks proof of the real Anthropic login/callback/token-exchange path.

## What This Changes

The PR restricts Anthropic OAuth callback hosts to loopback values, derives the redirect URI from the selected host, changes the default callback host to localhost, and adds focused callback-host tests.

## Best Possible Solution

Land the loopback allowlist and derived redirect URI after redacted real Anthropic auth-flow proof and maintainer acceptance of the fail-closed env behavior, or deliberately retire the legacy direct Anthropic OAuth helper if maintainers choose that direction.

## Reproduction Assessment

Yes, at source level. Current main and v2026.6.10 can bind the callback server to OPENCLAW_OAUTH_CALLBACK_HOST while sending a hardcoded localhost redirect URI; I did not run a live Anthropic OAuth flow.

## Solution Assessment

Yes, as a narrow code repair, but not yet proof-sufficient. The loopback allowlist and derived redirect URI match the sibling OpenAI pattern; removing the env override or retiring direct Anthropic OAuth remains a maintainer security/product choice.
