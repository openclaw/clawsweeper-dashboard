## Summary

Keep open: the PR targets the right Anthropic OAuth security issue, but it changes the default provider-facing redirect URI from localhost to 127.0.0.1 without live Anthropic proof, and the submitted proof is only unit/typecheck output.

## What This Changes

The PR restricts Anthropic OAuth callback hosts to a loopback allowlist, derives the redirect URI from the resolved host, and adds regression tests for host validation and URI formatting.

## Best Possible Solution

Keep the loopback restriction, preserve the shipped localhost default redirect unless live Anthropic proof confirms alternatives, and land with focused tests plus real OAuth-flow evidence.

## Reproduction Assessment

Yes. Source inspection on current main and v2026.6.10 shows CALLBACK_HOST can diverge from the hardcoded localhost REDIRECT_URI; I did not run a full Anthropic login because it requires an interactive provider account.

## Solution Assessment

No. The loopback allowlist is the right direction, but deriving from a 127.0.0.1 default changes the registered-looking redirect URI without proof; the safer fix keeps localhost as the default or otherwise proves Anthropic accepts the new URI.
