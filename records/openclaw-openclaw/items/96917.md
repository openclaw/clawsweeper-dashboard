## Summary

Keep open: the PR is a plausible narrow fix for the Anthropic OAuth callback-host/redirect mismatch, but it still needs maintainer security/compatibility acceptance and stronger real behavior proof for the provider auth flow before merge.

## What This Changes

The PR allowlists Anthropic OAuth callback hosts to loopback values, derives the redirect URI from the validated host, changes the default bind host to localhost, and adds focused callback-host tests.

## Best Possible Solution

Land the loopback allowlist and derived redirect URI after redacted real Anthropic auth proof and maintainer acceptance of the fail-closed env behavior, or deliberately retire the legacy direct Anthropic OAuth helper if maintainers choose that direction.

## Reproduction Assessment

Yes, at source level. Current main and v2026.6.10 can bind the callback server to OPENCLAW_OAUTH_CALLBACK_HOST while sending a hardcoded localhost redirect URI; I did not run a live Anthropic OAuth flow.

## Solution Assessment

Yes, as a narrow repair, but not yet proof-sufficient. The loopback allowlist and derived redirect URI match the sibling OpenAI pattern; removing the env override or retiring direct Anthropic OAuth remains a maintainer security/product choice.
