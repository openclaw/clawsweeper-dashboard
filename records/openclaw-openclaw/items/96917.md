## Summary

Keep open: current main still has the Anthropic OAuth callback-host/redirect mismatch, but this PR is not merge-ready because it validates the env var at module import time and the proof still does not exercise the real provider auth flow.

## What This Changes

The branch restricts Anthropic OAuth callback hosts to loopback values, derives redirect URIs from the selected host, changes the default callback host to localhost, and adds callback-host tests.

## Best Possible Solution

Scope callback-host validation to the Anthropic OAuth login/callback-server path, keep the derived redirect URI and loopback allowlist, add import-safety coverage, and require redacted provider-flow proof plus maintainer security acceptance before merge.

## Reproduction Assessment

Yes, at source level. Current main and v2026.6.10 can bind the callback server to OPENCLAW_OAUTH_CALLBACK_HOST while sending a hardcoded localhost redirect URI; I did not run a live Anthropic OAuth flow.

## Solution Assessment

No, not as currently implemented. The loopback allowlist and derived redirect URI are the right repair shape, but validation should be scoped to Anthropic login instead of module import and still needs real-flow proof.
