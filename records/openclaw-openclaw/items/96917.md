## Summary

Keep open for maintainer review: this is a focused candidate fix for Anthropic OAuth callback-host hardening, but it intentionally tightens an existing auth env surface and affects credential acquisition, so the security and compatibility tradeoff still needs maintainer acceptance.

## What This Changes

The PR restricts Anthropic OAuth callback hosts to loopback values, derives the redirect URI from the resolved host, threads it through authorization/token exchange, and adds focused regression tests.

## Best Possible Solution

Land the loopback allowlist and derived redirect URI after maintainer acceptance of the env compatibility and security tradeoff, or pause for a broader decision on whether to deprecate direct Anthropic OAuth.

## Reproduction Assessment

Yes. Source inspection on current main and v2026.6.10 shows the callback server can bind to OPENCLAW_OAUTH_CALLBACK_HOST while authorization and token exchange use a fixed localhost redirect URI.

## Solution Assessment

Yes, pending maintainer acceptance. The loopback allowlist plus derived redirect URI is the narrowest maintainable repair and matches the sibling OpenAI OAuth pattern better than preserving arbitrary callback hosts.
