## Summary

Keep open: the PR addresses a real model-selection area, but it should not merge as-is because the branch introduces fuzzy alias routing and raw alias hint risks, lacks real behavior proof, and current main already fixed the linked Control UI provider-drift issue through the shared resolver/UI path.

## What This Changes

The branch folds spaces and underscores in alias lookup, adds configured-provider inference and alias hints to model errors, updates gateway session model identity resolution, and expands model-selection tests.

## Best Possible Solution

Rebase any desired fuzzy matching into model-selection-shared.ts with exact-ID precedence, collision rejection, sanitized hints, focused tests, and real Control UI or gateway proof; otherwise keep the current-main fix for the linked issue.

## Reproduction Assessment

Yes for the PR-head defects by source inspection: alias lookup happens before exact model parsing, same-provider normalized matches overwrite matchedModelId, and raw aliases flow into errors. I did not run tests because this is a read-only review.

## Solution Assessment

No. The direction may be useful, but this implementation is not the narrowest maintainable fix until maintainers decide the fuzzy separator policy and the branch is rebased into the shared resolver with safer precedence and output handling.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve exact model IDs before fuzzy alias fallback:** `src/agents/model-selection.ts:361-364`
  - body: Folding spaces and underscores before alias lookup lets a providerless exact model ID such as `my_model` match an alias like `my-model` before the normal model path. With `anthropic/my_model` allowlisted and `my-model` pointing elsewhere, `/model my_model` can silently switch providers.
  - confidence: 0.92
- **[P2] Reject same-provider normalized model collisions:** `src/agents/model-selection.ts:236-237`
  - body: `inferUniqueProviderFromConfiguredModels` overwrites `matchedModelId` for every normalized match but only treats multiple providers as ambiguous. If one provider configures both `gpt_5.4` and `gpt-5.4`, input like `gpt 5.4` resolves to whichever entry is visited last.
  - confidence: 0.84
- **[P2] Sanitize aliases before adding error hints:** `src/agents/model-selection.ts:765-766`
  - body: The new available-alias hint joins configured aliases directly into `model not allowed` errors. Aliases can contain control characters or ANSI escapes, so each alias needs to be sanitized or escaped before it is returned to terminal or UI surfaces.
  - confidence: 0.88

## Security Review
