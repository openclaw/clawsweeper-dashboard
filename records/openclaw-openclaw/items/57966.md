## Summary

Keep open. Current main already addresses the linked Control UI provider-drift problem, but this PR still proposes distinct fuzzy alias matching and error-message behavior, and the proposed patch has blocking routing and sanitization regressions plus no real behavior proof.

## What This Changes

The PR broadens model alias matching by normalizing spaces and underscores, adds configured-allowlist provider inference and alias hints in errors, and updates model-selection/session tests.

## Best Possible Solution

Keep the shipped UI/catalog and backend provider-inference fixes, and only merge a smaller rebased PR if it defines safe fuzzy-alias precedence, ambiguity handling, sanitized errors, and real behavior proof.

## Reproduction Assessment

Yes for the patch-level regressions: source inspection shows exact `my_model` inputs hit normalized alias lookup before exact model parsing, and configured aliases flow raw into the new error hint. I did not run tests because this review is read-only.

## Solution Assessment

No. The direction is plausible, but the patch needs exact-model precedence, ambiguity rules for normalized collisions, and sanitized alias hints before it is a maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Preserve exact model IDs before fuzzy aliases:** `src/agents/model-selection.ts:361-364`
  - body: This normalized alias lookup runs before exact/default-provider model parsing, so a config with `anthropic/my_model` and an alias `my-model -> openai/gpt-4o` will route `/model my_model` to the alias target instead of the exact allowlisted model. Check exact model IDs before accepting fuzzy alias matches.
  - confidence: 0.89
- **[P2] Reject ambiguous normalized model matches:** `src/agents/model-selection.ts:218-220`
  - body: `inferUniqueProviderFromConfiguredModels` only treats multiple providers as ambiguous. If the same provider has both `gpt_5.4` and `gpt-5.4`, input like `gpt 5.4` overwrites `matchedModelId` and silently chooses whichever config entry is visited last.
  - confidence: 0.82
- **[P2] Sanitize aliases before returning error hints:** `src/agents/model-selection.ts:765-766`
  - body: Aliases come from config, but the new `available aliases` hint joins them raw into the error returned by callers. A configured alias containing a newline or ANSI escape can forge terminal/chat output; sanitize or escape aliases before appending them.
  - confidence: 0.91
