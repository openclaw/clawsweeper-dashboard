## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve exact model IDs before fuzzy aliases:** `src/agents/model-selection.ts:361-364`
  - body: The new separator-folded alias lookup runs before normal model parsing. With allowlisted `anthropic/my_model` and alias `my-model -> openai/gpt-4o`, `/model my_model` now hits the alias first and silently switches providers; exact providerless model IDs should win before fuzzy aliases.
  - confidence: 0.92
- **[P2] Reject same-provider normalized model collisions:** `src/agents/model-selection.ts:236-237`
  - body: `inferUniqueProviderFromConfiguredModels` tracks ambiguity only by provider while `matchedModelId` is overwritten for each normalized match. If `openai/gpt_5.4` and `openai/gpt-5.4` are both configured, input like `gpt 5.4` routes to iteration order instead of rejecting an ambiguous match.
  - confidence: 0.86
- **[P2] Sanitize alias hints before returning errors:** `src/agents/model-selection.ts:765-766`
  - body: The available-alias hint joins configured alias strings directly into the `model not allowed` error. Aliases can contain newlines, control characters, or ANSI escapes, so each alias needs sanitizing or escaping before reaching terminal, chat, or log surfaces.
  - confidence: 0.9
