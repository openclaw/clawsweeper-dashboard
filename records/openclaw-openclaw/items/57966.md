## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve exact model IDs before fuzzy aliases:** `src/agents/model-selection.ts:361-364`
  - body: Folding spaces and underscores before normal model parsing lets a providerless exact model ID such as `my_model` match an alias like `my-model` first. With `anthropic/my_model` configured and `my-model` pointing elsewhere, `/model my_model` can silently switch to the alias target; preserve exact model IDs before accepting a fuzzy alias hit.
  - confidence: 0.92
- **[P2] Treat normalized same-provider matches as ambiguous:** `src/agents/model-selection.ts:236-237`
  - body: `inferUniqueProviderFromConfiguredModels` overwrites `matchedModelId` for every normalized match and only rejects multiple providers. If one provider configures both `gpt_5.4` and `gpt-5.4`, input like `gpt 5.4` resolves to whichever entry is visited last instead of rejecting the ambiguous match.
  - confidence: 0.86
- **[P2] Sanitize aliases before appending error hints:** `src/agents/model-selection.ts:765-766`
  - body: The available-alias hint joins configured alias strings directly into `model not allowed` errors. Aliases can contain newlines, control characters, or ANSI escapes, so each alias needs sanitizing or escaping before it reaches terminal, log, or UI surfaces.
  - confidence: 0.9

## Security Review
