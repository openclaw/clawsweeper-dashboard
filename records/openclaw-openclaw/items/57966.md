## Summary

Keep open: the linked user issue is closed as implemented on current main, but this PR still carries unique resolver changes against a stale code path and has blocking correctness, security, and proof gaps before it could be considered.

## What This Changes

The branch changes model-selection and gateway session identity logic to fold spaces and underscores in model aliases, infer providers with matched model IDs, add alias hints to disallowed-model errors, and expand resolver tests.

## Best Possible Solution

Either retire this branch in favor of the current-main fix for the linked issue, or land a fresh narrow PR against the shared resolver that preserves exact IDs, rejects ambiguous normalized matches, sanitizes hints, and proves the real UI/gateway flow.

## Reproduction Assessment

Yes. Source inspection of the PR head gives high-confidence resolver-level reproductions for exact-model rerouting, same-provider normalized collisions, and raw alias text flowing into errors.

## Solution Assessment

No. The direction is plausible, but this branch is not the best implementation until it is rebased onto the shared resolver and the separator policy, ambiguity handling, and output sanitization are fixed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve exact model IDs before fuzzy alias fallback:** `src/agents/model-selection.ts:360-364`
  - body: Folding spaces and underscores before alias lookup lets a providerless exact model ID such as `my_model` match an alias like `my-model` before the normal model path. With `anthropic/my_model` allowlisted and `my-model` pointing elsewhere, `/model my_model` can silently switch providers.
  - confidence: 0.92
- **[P2] Sanitize aliases before adding error hints:** `src/agents/model-selection.ts:765-767`
  - body: The new available-alias hint joins configured aliases directly into `model not allowed` errors. Aliases can contain control characters or ANSI escapes, so each alias needs to be sanitized or escaped before it is returned to terminal or UI surfaces.
  - confidence: 0.88
- **[P2] Reject normalized same-provider model collisions:** `src/agents/model-selection.ts:236-237`
  - body: `inferUniqueProviderFromConfiguredModels` overwrites `matchedModelId` for every normalized match but only treats multiple providers as ambiguous. If one provider configures both `gpt_5.4` and `gpt-5.4`, input like `gpt 5.4` resolves to whichever entry is visited last.
  - confidence: 0.84

## Security Review
