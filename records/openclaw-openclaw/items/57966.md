## Summary

Keep open: the branch is a plausible resolver improvement, but it still has blocking correctness and security issues, lacks real behavior proof, and needs maintainer judgment now that the linked Control UI bug was fixed on current main.

## What This Changes

The PR folds spaces and underscores in model alias lookup, adds configured-provider inference and available-alias error hints, updates gateway session model identity resolution, and expands model-selection tests.

## Best Possible Solution

If maintainers still want fuzzy separator matching, rebase it into the shared resolver with exact-ID precedence, collision rejection, sanitized hints, focused regression tests, and real Control UI or gateway proof; otherwise keep the current-main UI fix as the solution.

## Reproduction Assessment

Yes for the PR-head defects: source inspection shows alias lookup before exact model parsing, same-provider normalized matches overwriting matchedModelId, and raw aliases flowing into errors. I did not run tests because this review is read-only.

## Solution Assessment

No. The direction may be useful, but this branch is not the best implementation until it is rebased onto the shared resolver and the separator policy, ambiguity handling, output sanitization, and real behavior proof are addressed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve exact model IDs before fuzzy alias fallback:** `src/agents/model-selection.ts:360-364`
  - body: Folding spaces and underscores before alias lookup lets a providerless exact model ID such as `my_model` match an alias like `my-model` before the normal model path. With `anthropic/my_model` allowlisted and `my-model` pointing elsewhere, `/model my_model` can silently switch providers.
  - confidence: 0.92
- **[P2] Reject normalized same-provider model collisions:** `src/agents/model-selection.ts:236-237`
  - body: `inferUniqueProviderFromConfiguredModels` overwrites `matchedModelId` for every normalized match but only treats multiple providers as ambiguous. If one provider configures both `gpt_5.4` and `gpt-5.4`, input like `gpt 5.4` resolves to whichever entry is visited last.
  - confidence: 0.84
- **[P2] Sanitize aliases before adding error hints:** `src/agents/model-selection.ts:765-767`
  - body: The new available-alias hint joins configured aliases directly into `model not allowed` errors. Aliases can contain control characters or ANSI escapes, so each alias needs to be sanitized or escaped before it is returned to terminal or UI surfaces.
  - confidence: 0.88

## Security Review
