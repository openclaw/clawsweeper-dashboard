## Summary

Keep this PR open, but it is not currently mergeable: the linked Control UI issue was separately fixed on current main, while this branch still proposes unique fuzzy alias and error-hint behavior that has correctness and security blockers, a conflicting rebase, and no real after-fix behavior proof.

## What This Changes

This PR changes model alias lookup to fold spaces and underscores, adds configured-provider inference and available-alias hints, adjusts gateway session model identity resolution, and expands model-selection tests.

## Best Possible Solution

Rebase only the desired separator-tolerant behavior into the shared resolver with exact-ID precedence, normalized collision rejection, sanitized alias hints, focused tests, and redacted real behavior proof.

## Reproduction Assessment

Yes for the PR-head defects by source inspection: alias lookup runs before exact model parsing, same-provider normalized matches overwrite matchedModelId, and raw aliases flow into error hints. I did not run tests because this is a read-only review.

## Solution Assessment

No. The direction may be useful, but this patch is not the best merge path until the fuzzy separator policy is settled and the implementation is rebased into the shared resolver with safer precedence and output handling.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve exact model IDs before fuzzy aliases:** `src/agents/model-selection.ts:361-364`
  - body: Folding spaces and underscores before normal model parsing lets a providerless exact model ID such as `my_model` match an alias like `my-model` first. With `anthropic/my_model` configured and `my-model` pointing elsewhere, `/model my_model` can silently switch to the alias target; the resolver should preserve exact model IDs before accepting a fuzzy alias hit.
  - confidence: 0.92
- **[P2] Treat normalized same-provider matches as ambiguous:** `src/agents/model-selection.ts:236-237`
  - body: `inferUniqueProviderFromConfiguredModels` overwrites `matchedModelId` for every normalized match and only rejects multiple providers. If one provider configures both `gpt_5.4` and `gpt-5.4`, input like `gpt 5.4` resolves to whichever entry is visited last instead of rejecting the ambiguous match.
  - confidence: 0.86
- **[P2] Sanitize aliases before appending error hints:** `src/agents/model-selection.ts:765-766`
  - body: The new available-alias hint joins configured alias strings directly into `model not allowed` errors. Aliases can contain newlines, control characters, or ANSI escapes, so each alias needs sanitizing or escaping before it reaches terminal, log, or UI surfaces.
  - confidence: 0.9
