## Summary

Keep this PR open for maintainer/contributor follow-up: the linked Control UI report was fixed separately on current main, but this branch still proposes distinct fuzzy alias and error-hint behavior with blocking correctness/security findings and no real behavior proof.

## What This Changes

The PR changes model alias lookup to fold spaces and underscores, adds configured-provider inference with canonical model IDs, includes available-alias hints in not-allowed errors, adjusts session model identity resolution, and expands model-selection tests.

## Best Possible Solution

Rebase only the desired model-selection behavior into src/agents/model-selection-shared.ts with exact-ID precedence, normalized collision rejection, sanitized alias hints, focused tests, and redacted real behavior proof.

## Reproduction Assessment

Yes for the PR-head defects by source inspection: alias lookup runs before exact model parsing, same-provider normalized matches overwrite matchedModelId, and raw aliases flow into error hints. I did not run tests because this is a read-only review.

## Solution Assessment

No. The direction may be useful, but this patch is not the best merge path until separator-folding policy, exact-ID precedence, collision handling, and output sanitization are settled in the shared resolver.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve exact model IDs before fuzzy aliases:** `src/agents/model-selection.ts:361-364`
  - body: Folding spaces and underscores before normal model parsing lets a providerless exact model ID such as `my_model` match an alias like `my-model` first. With `anthropic/my_model` configured and `my-model` pointing elsewhere, `/model my_model` can silently switch to the alias target; preserve exact model IDs before accepting a fuzzy alias hit.
  - confidence: 0.92
- **[P2] Treat normalized same-provider matches as ambiguous:** `src/agents/model-selection.ts:236-237`
  - body: `inferUniqueProviderFromConfiguredModels` overwrites `matchedModelId` for every normalized match and only rejects multiple providers. If one provider configures both `gpt_5.4` and `gpt-5.4`, input like `gpt 5.4` resolves to whichever entry is visited last instead of rejecting the ambiguous match.
  - confidence: 0.86
- **[P2] Sanitize aliases before appending error hints:** `src/agents/model-selection.ts:765-766`
  - body: The new available-alias hint joins configured alias strings directly into `model not allowed` errors. Aliases can contain newlines, control characters, or ANSI escapes, so each alias needs sanitizing or escaping before it reaches terminal, log, or UI surfaces.
  - confidence: 0.9

## Security Review
