## Summary

Keep open: the linked Control UI bug is already addressed on current main, but this PR still proposes unique fuzzy alias matching and alias-hint behavior with blocking routing/security issues and no real after-fix proof.

## What This Changes

The branch changes model selection to fold spaces/underscores in alias lookup, infer providers from configured models, append alias hints to disallowed-model errors, update gateway session identity resolution, and expand tests.

## Best Possible Solution

If maintainers still want this behavior, rebase it onto the shared resolver with exact-ID precedence, collision rejection, sanitized hints, focused regression tests, and redacted real runtime proof.

## Reproduction Assessment

Yes for the PR-head defects: source inspection of 9f4f7dafd5e4e4c2c43c385f36de3409db60fe4d shows fuzzy alias lookup before exact model parsing, same-provider match overwrite, and raw alias hints. I did not run tests because this review is read-only.

## Solution Assessment

No. The direction may be useful, but this branch is not the best implementation until the fuzzy matching policy, ambiguity handling, output sanitization, rebase, and real behavior proof are addressed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve exact runtime model IDs before fuzzy aliases:** `src/agents/model-selection.ts:360-364`
  - body: `resolveModelRefFromString` now normalizes a providerless input and checks aliases before parsing the model ref. With `anthropic/my_model` allowlisted and alias `my-model` pointing elsewhere, `/model my_model` can silently switch to the alias target instead of the exact model ID.
  - confidence: 0.92
- **[P1] Preserve exact default model IDs before fuzzy aliases:** `src/agents/model-selection.ts:391-394`
  - body: `resolveConfiguredModelRef` uses the same fuzzy alias lookup for `model.primary` before providerless fallback/inference. Existing configs that intentionally use `model.primary: "my_model"` can be rerouted to an unrelated `my-model` alias at startup.
  - confidence: 0.9
- **[P2] Reject same-provider normalized model collisions:** `src/agents/model-selection.ts:236-237`
  - body: `inferUniqueProviderFromConfiguredModels` overwrites `matchedModelId` for each normalized match and only treats multiple providers as ambiguous. If one provider configures both `gpt_5.4` and `gpt-5.4`, input like `gpt 5.4` resolves to whichever entry is visited last.
  - confidence: 0.84
- **[P2] Sanitize aliases before adding error hints:** `src/agents/model-selection.ts:765-766`
  - body: The available-alias hint joins configured aliases directly into `model not allowed` errors. Aliases can contain newlines, control characters, or ANSI escapes, so each alias should be sanitized or escaped before returning it to terminal or UI surfaces.
  - confidence: 0.88
