## Summary

Keep open: current main fixed the linked Control UI provider-drift problem, but this PR still carries unique backend alias-normalization and error-hint behavior that is not on main, and the current PR head has blocking routing and raw-alias output defects plus no real behavior proof.

## What This Changes

The PR changes model alias lookup, configured-provider inference, model-not-allowed error hints, gateway session model identity resolution, and model-selection tests.

## Best Possible Solution

Rebase only the desired separator-tolerant alias behavior into the shared resolver with exact-ID precedence, normalized collision rejection, sanitized alias hints, focused tests, and redacted real behavior proof.

## Reproduction Assessment

Yes for the PR-head defects by source inspection: alias lookup runs before exact model parsing, same-provider normalized matches overwrite matchedModelId, and raw aliases flow into error hints. I did not run tests because this is a read-only review.

## Solution Assessment

No. The direction may be useful, but this patch is not the best merge path until the fuzzy separator policy is settled and the implementation is rebased into the shared resolver with safer precedence and output handling.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve exact model IDs before fuzzy alias fallback:** `src/agents/model-selection.ts:361-364`
  - body: Folding spaces and underscores before alias lookup lets a providerless exact model ID such as my_model match an alias like my-model before the normal model path. With anthropic/my_model allowlisted and my-model pointing elsewhere, /model my_model can silently switch providers.
  - confidence: 0.92
- **[P2] Reject same-provider normalized model collisions:** `src/agents/model-selection.ts:236-237`
  - body: inferUniqueProviderFromConfiguredModels overwrites matchedModelId for every normalized match but only treats multiple providers as ambiguous. If one provider configures both gpt_5.4 and gpt-5.4, input like gpt 5.4 resolves to whichever entry is visited last.
  - confidence: 0.84
- **[P2] Sanitize aliases before adding error hints:** `src/agents/model-selection.ts:765-766`
  - body: The new available-alias hint joins configured aliases directly into model-not-allowed errors. Aliases can contain control characters or ANSI escapes, so each alias needs to be sanitized or escaped before it reaches terminal or UI surfaces.
  - confidence: 0.88

## Security Review

Status: needs_attention
