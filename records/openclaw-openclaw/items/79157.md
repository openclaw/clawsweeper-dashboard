## Summary

Keep open: current main has deterministic exec approvals, safeBins, and a Codex-specific guardian reviewer, but not a generic LLM-assisted exec approval mode. This would change core host-exec safety policy and needs maintainer product/security design rather than cleanup or an automated fix.

## What This Changes

The issue requests an LLM-assisted exec approval mode that can classify command intent and risk before auto-allowing, asking, or blocking.

## Best Possible Solution

Keep this open for a maintainer-owned design that either improves deterministic safe-bin/allowlist ergonomics or defines a bounded approval-reviewer contract with hard blocks, auditability, docs, and regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction. The reported friction is source-backed by the strictInlineEval and safeBins policy, but the requested outcome is a new adaptive approval mode rather than broken documented behavior.
