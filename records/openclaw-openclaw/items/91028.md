Full review comments:

- **[P1] Mask credentials through Lobster's final env merge:** `extensions/lobster/src/lobster-runner.ts:313-315`
  - body: Deleting these keys from `ctx.env` leaves them absent, and `@clawdbot/lobster` creates the actual env as `{ ...process.env, ...ctx.env }`.   - confidence: 0.93
- **[P1] Preserve host LLM auth and model policy:** `extensions/lobster/src/lobster-tool.ts:277-288`
  - body: This adapter calls `runEmbeddedAgent` directly after local model parsing, skipping `llm-task` plugin config for auth profile, allowed models, thinking validation, temperature, maxTokens, and `authProfileIdSource`. Embedded workflow LLM calls should reuse or share the host-owned policy path before invoking the model.
  - confidence: 0.9
- **[P1] Avoid shadowing Lobster's HTTP openclaw provider:** `extensions/lobster/src/lobster-tool.ts:323-327`
  - body: Lobster documents `openclaw` as its HTTP/env provider and then resolves a direct adapter for the selected provider before the built-in branch. Adding `llmAdapters.openclaw` therefore changes `llm.invoke --provider openclaw` to this in-process adapter unless maintainers intentionally approve and prove that compatibility change.
  - confidence: 0.86
