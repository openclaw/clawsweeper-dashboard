Full review comments:

- **[P1] Mask credentials through Lobster's final env merge:** `extensions/lobster/src/lobster-runner.ts:313-315`
  - body: Deleting the gateway keys from `ctx.env` leaves them absent, and Lobster then creates the actual command env as `{ ...process.env, ...ctx.env }`.   - confidence: 0.93
- **[P1] Preserve host LLM auth and model policy:** `extensions/lobster/src/lobster-tool.ts:277-288`
  - body: This adapter calls `runEmbeddedAgent` directly after local model parsing, skipping `llm-task` plugin config for auth profile, allowed models, thinking validation, temperature, max tokens, and auth-profile source. Embedded workflow LLM calls should reuse or share the host-owned policy path before invoking the model.
  - confidence: 0.9
- **[P1] Avoid shadowing Lobster's HTTP openclaw provider:** `extensions/lobster/src/lobster-tool.ts:323-327`
  - body: Lobster resolves `ctx.llmAdapters` before built-in providers, so adding `llmAdapters.openclaw` changes `llm.invoke --provider openclaw` from Lobster's HTTP/env provider to this in-process adapter. Use a maintainer-approved distinct name or document and prove the compatibility change.
  - confidence: 0.86
