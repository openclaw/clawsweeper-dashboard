## Summary

Keep open: current main and v2026.6.11 still lack the embedded Lobster in-process LLM bridge, but this PR is not merge-ready because the new adapter has credential-masking, provider-contract, policy-preservation, and live-proof blockers.

## What This Changes

The branch adds host-provided Lobster embedded runtime bridges, registers an in-process `llmAdapters.openclaw` adapter backed by `runEmbeddedAgent`, and adds runner tests for adapter injection and credential stripping.

## Best Possible Solution

Land a maintainer-approved embedded adapter contract that masks gateway credentials through the real Lobster runtime env, preserves or explicitly replaces `llm-task` policy, settles the provider id, updates docs, and includes redacted live configured-model proof.

## Solution Assessment

No, not yet. The in-process bridge direction is plausible and safer than injecting Gateway tokens, but this patch must fix the final env masking hole, preserve or intentionally replace `llm-task` policy, settle provider naming, and add live proof.

- **[P1] Mask gateway credentials through Lobster's final env merge:** `extensions/lobster/src/lobster-runner.ts:313-314`
  - body: This deletes gateway credential keys from `ctx.env`, but `@clawdbot/lobster` later builds the actual runtime env as `{ ...process.env, ...ctx.env }`. If the Gateway process has `OPENCLAW_URL`, `OPENCLAW_TOKEN`, `CLAWD_URL`, or `CLAWD_TOKEN`, those values are restored before workflow shell steps run; pass explicit masking values or sanitize through the real runtime path and test that path.
  - confidence: 0.92
- **[P1] Reuse the llm-task policy path for the direct adapter:** `extensions/lobster/src/lobster-tool.ts:277-288`
  - body: The new adapter calls `runEmbeddedAgent` directly with only provider/model/timeout, so embedded `llm.invoke --provider openclaw` bypasses existing `llm-task` handling for plugin defaults, `allowedModels`, `authProfileId`, thinking policy, temperature, max token limits, and schema validation. Share that policy path unless maintainers explicitly define a different contract.
  - confidence: 0.88
- **[P1] Avoid shadowing Lobster's built-in openclaw adapter:** `extensions/lobster/src/lobster-tool.ts:324-326`
  - body: Registering the direct adapter under `openclaw` changes embedded `llm.invoke --provider openclaw` because Lobster resolves `ctx.llmAdapters[provider]` before its built-in HTTP `openclaw` provider. Use a distinct provider name or get maintainer approval and docs for the compatibility change.
  - confidence: 0.86
