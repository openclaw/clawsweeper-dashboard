## Summary

Keep open: the in-process Lobster LLM bridge is a useful direction, but this patch is not merge-ready because the security boundary is incomplete, the direct adapter breaks normal `llm.invoke` model semantics, transient LLM transcripts are not cleaned up, and the external PR has only unit/type proof.

## What This Changes

The PR adds a Lobster embedded-runner bridge that passes host-provided `llmAdapters` into the runtime and implements an `openclaw` in-process LLM adapter backed by `api.runtime.agent.runEmbeddedAgent`, with one runner unit test.

## Best Possible Solution

Land a narrow embedded bridge only after maintainers approve the provider contract, credential env keys are scrubbed, model/auth resolution matches the supported OpenClaw path, temp transcripts are cleaned, and a redacted live embedded Lobster proof is supplied.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Strip gateway credentials from the embedded env:** `extensions/lobster/src/lobster-runner.ts:286-289`
  - body: `createEmbeddedToolContext` still starts with `{ ...process.env }`, so a gateway launched with `OPENCLAW_URL`, `OPENCLAW_TOKEN`, `CLAWD_URL`, or `CLAWD_TOKEN` will pass those values to Lobster shell steps. Since this PR's security model is to avoid exposing gateway auth, delete those keys from `ctx.env` and seed them in the regression test before asserting they are absent.
  - confidence: 0.93
- **[P2] Preserve bare model resolution for llm.invoke:** `extensions/lobster/src/lobster-tool.ts:242-243`
  - body: `@clawdbot/lobster` resolves the provider outside the payload and only puts the CLI `--model` value in `payload.model`; its own examples use `llm.invoke --provider openclaw --model claude-3-sonnet`. With this code, a bare model lacks a slash, `provider` becomes `undefined`, and the new embedded path throws instead of using the selected/default provider and OpenClaw alias resolution like `llm-task` does.
  - confidence: 0.9
- **[P2] Clean up embedded LLM session files:** `extensions/lobster/src/lobster-tool.ts:261-264`
  - body: The adapter creates a random session file under `os.tmpdir()` and never removes it, while `runEmbeddedAgent` persists the prompt/transcript to the session file. Use the OpenClaw temp workspace helper pattern from `llm-task` so workflow inputs and model output do not remain as orphaned transcript files after each embedded LLM step.
  - confidence: 0.82
