## Summary

## What This Changes

The PR adds host-provided in-process LLM adapter bridges to the embedded Lobster runner, wiring an `openclaw` adapter through `runEmbeddedAgent` and adding tests for adapter injection and gateway credential stripping.

## Best Possible Solution

Land a maintainer-approved in-process adapter that keeps Gateway credentials out of the final Lobster runtime env, preserves `llm-task` auth/model policy, chooses and documents the provider name, and includes redacted live embedded invocation proof.

## Reproduction Assessment

## Solution Assessment

No, not yet. The in-process bridge direction is plausible and safer than injecting Gateway tokens, but this patch must fix the final env masking hole, preserve `llm-task` policy, settle provider naming, and add live proof.

  - confidence: 0.86

  - body: Registering the direct adapter under `openclaw` changes `llm.invoke --provider openclaw` in embedded runs because Lobster resolves `ctx.llmAdapters[provider]` before its built-in HTTP `openclaw` adapter.
