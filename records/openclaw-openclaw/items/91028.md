Full review comments:

- **[P1] Keep gateway credentials out of the final Lobster env:** `extensions/lobster/src/lobster-runner.ts:293-294`
  - body: This deletes the secret keys from the copied env, but the real `@clawdbot/lobster` runtime later builds `env` as `{ ...process.env, ...ctx.env }`. Because the deleted keys are absent rather than explicitly masked, any `OPENCLAW_URL`/token still present in the gateway process env is reintroduced before pipeline shell steps run.
  - confidence: 0.93
- **[P1] Route embedded LLM calls through the host policy path:** `extensions/lobster/src/lobster-tool.ts:277-288`
  - body: The adapter accepts `payload.model` and calls `runEmbeddedAgent` directly, so embedded workflows can choose provider/model outside the existing plugin LLM policy checks and outside `llm-task`'s `allowedModels`/`defaultAuthProfileId` behavior. This should preserve the host-owned `runtime.llm.complete` policy path or add an explicitly equivalent allowlist/auth contract before exposing the adapter.
  - confidence: 0.88
- **[P2] Forward llm.invoke sampling limits to the model run:** `extensions/lobster/src/lobster-tool.ts:277-288`
  - body: Lobster's `llm.invoke` payload includes `temperature` and `maxOutputTokens`, but this adapter only reads `timeoutMs` and never forwards sampling limits to `runEmbeddedAgent`. Users can pass the documented options and get silently different model behavior.
  - confidence: 0.82
