## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Mask credentials through Lobster's final env merge:** `extensions/lobster/src/lobster-runner.ts:313-314`
  - body: This deletes Gateway credential keys from `ctx.env`, but Lobster builds the actual runtime env as `{ ...process.env, ...ctx.env }` before running workflows. If the Gateway process has these keys, they are restored and can reach workflow shell steps; mask them through the final runtime env path and test via `createToolContext`.
  - confidence: 0.92
- **[P2] Preserve llm-task auth and model policy:** `extensions/lobster/src/lobster-tool.ts:277-288`
  - body: The adapter calls `runEmbeddedAgent` directly with only provider/model/timeout, bypassing `llm-task` defaults, allowed-model checks, auth-profile selection, thinking validation, stream params, and schema validation. Reuse that policy path or document an approved replacement before exposing this provider.
  - confidence: 0.87
- **[P1] Avoid shadowing Lobster's HTTP openclaw provider:** `extensions/lobster/src/lobster-tool.ts:324-326`
  - body: Lobster resolves `ctx.llmAdapters[provider]` before its built-in HTTP providers, so registering a direct adapter as `openclaw` changes `llm.invoke --provider openclaw` in embedded runs. Use a distinct provider id or get maintainer approval and docs for that compatibility change.
  - confidence: 0.84
