## Summary

The PR adds Lobster embedded runtime bridges, registers an in-process `openclaw` LLM adapter backed by `runEmbeddedAgent`, and adds runner tests for adapter injection and credential stripping.

## Best Possible Solution

Land a maintained embedded LLM bridge only after it masks secrets through the real Lobster runtime env path, preserves or explicitly replaces llm-task policy, uses a maintainer-approved provider name, and includes redacted live invocation proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Mask credentials after Lobster's final env merge:** `extensions/lobster/src/lobster-runner.ts:293-294`
  - body: Deleting these keys from `ctx.env` is not enough because `@clawdbot/lobster` creates the real runtime env as `{ ...process.env, ...ctx.env }`. If the gateway process has `OPENCLAW_URL` or token values, omitted keys are repopulated and workflow shell steps can still inherit them, so the fix needs to mask through the actual dependency path and test that path.
  - confidence: 0.93
- **[P1] Preserve llm-task auth and model policy:** `extensions/lobster/src/lobster-tool.ts:277-288`
  - body: This adapter calls `runEmbeddedAgent` directly after local provider/model parsing, so embedded `llm.invoke` does not honor llm-task controls such as `defaultAuthProfileId`, `allowedModels`, thinking validation, and stream params. That lets Lobster workflow LLM calls route differently from the configured llm-task contract; reuse/share that path or carry the same policy with focused tests.
  - confidence: 0.9
- **[P1] Avoid shadowing Lobster's HTTP openclaw provider:** `extensions/lobster/src/lobster-tool.ts:323-327`
  - body: Lobster resolves direct `ctx.llmAdapters` before built-in providers, so registering this adapter as `openclaw` changes explicit `llm.invoke --provider openclaw` embedded runs from the HTTP/env provider to the new in-process path. Use a maintainer-approved distinct provider name or add the documented migration/proof for intentionally taking over `openclaw`.
  - confidence: 0.86
