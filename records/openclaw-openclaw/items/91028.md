## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Mask credentials after Lobster's final env merge:** `extensions/lobster/src/lobster-runner.ts:293-294`
  - body: Deleting these keys from `ctx.env` is not enough because `@clawdbot/lobster` builds the real runtime env as `{ ...process.env, ...ctx.env }`. If the gateway process has `OPENCLAW_URL` or token values, omitted keys are repopulated and workflow shell steps can still inherit them, so the fix needs to mask through the actual dependency path and test that path.
  - confidence: 0.93
- **[P1] Preserve host LLM auth and model policy:** `extensions/lobster/src/lobster-tool.ts:277-288`
  - body: This adapter calls `runEmbeddedAgent` directly after local provider/model parsing, so embedded `llm.invoke` does not honor llm-task/runtime controls such as auth profile, allowed models, thinking validation, timeout defaults, and stream params. That lets Lobster workflow LLM calls route differently from operator configuration; reuse the host-owned policy path or carry the same checks with focused tests.
  - confidence: 0.9
- **[P1] Avoid shadowing Lobster's HTTP openclaw provider:** `extensions/lobster/src/lobster-tool.ts:323-327`
  - body: Lobster resolves direct `ctx.llmAdapters` before built-in providers, so registering this adapter as `openclaw` changes explicit `llm.invoke --provider openclaw` embedded runs from the HTTP/env provider to the new in-process path. Use a maintainer-approved distinct provider name or add the documented migration and compatibility proof for intentionally taking over `openclaw`.
  - confidence: 0.86
