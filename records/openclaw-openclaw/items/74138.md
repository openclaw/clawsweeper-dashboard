## Summary

Keep open. Current main still lacks the legacy `openclaw/plugin-sdk/feishu` SDK facade needed for the reported `createScopedPairingAccess` failure, and open PR #74223 already targets this issue with closing syntax, so the issue should stay open until that PR or a replacement merges.

## What This Changes

A Feishu/Lark regression report asks OpenClaw to restore compatibility for legacy Feishu channel packages that call `createScopedPairingAccess` through `openclaw/plugin-sdk/feishu` after upgrading and configuring multiple accounts.

## Best Possible Solution

Land one narrow backward-compatible `openclaw/plugin-sdk/feishu` facade with aligned export map, SDK entrypoint/API contract coverage, and any required changelog/docs; keep Feishu runtime, auth, config, and pairing storage behavior unchanged for this fix.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: a legacy Feishu package importing `openclaw/plugin-sdk/feishu` cannot receive `createScopedPairingAccess` from current main because that subpath is absent from the package exports and SDK entrypoint wiring.

## Solution Assessment

Yes. The narrow maintainable solution is a compatibility SDK facade plus contract coverage; current bundled Feishu already uses the generic `channel-pairing` helper, so a Feishu runtime rewrite is not needed.
