## Summary

Keep open. Current main still lacks the legacy `openclaw/plugin-sdk/feishu` SDK subpath needed for the reported `createScopedPairingAccess` failure, and open PR #74223 already targets this issue with closing syntax, so this should remain open until that fix or a consolidated replacement merges.

## What This Changes

The issue reports that upgrading to OpenClaw 2026.4.26 broke Feishu dispatch for multi-account setups because `createScopedPairingAccess` is missing from the Feishu SDK facade.

## Best Possible Solution

Restore a narrow backward-compatible `openclaw/plugin-sdk/feishu` facade, align the SDK entrypoint/export/API contract surfaces, and land PR #74223 or a consolidated replacement before closing this issue as implemented.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: a legacy Feishu package importing `openclaw/plugin-sdk/feishu` cannot get `createScopedPairingAccess` from current main because that subpath is absent from package exports and SDK entrypoint wiring.

## Solution Assessment

Yes. The best fix direction is the narrow compatibility facade plus SDK contract coverage described by PR #74223; closing this issue before that PR or an equivalent fix merges would hide a still-present regression.
