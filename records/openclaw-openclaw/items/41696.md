## Summary

Keep open. The behavioral fix is valid, but the branch edits the old `src/discord` resolver while current `main` routes Discord allowlist resolution through the active plugin, where the bare-numeric guild regression still exists.

## What This Changes

The PR changes the old Discord resolver and tests so bare numeric allowlist entries prefer guild IDs before channel lookup.

## Best Possible Solution

Port the resolver and regression coverage into `extensions/discord`, keep explicit prefix semantics plus channel fallback for non-guild bare numerics, and add the required user-facing changelog entry.

## Reproduction Assessment

Yes. Current `main` has a static and test-backed reproduction: the active parser classifies bare `999` as `channelId`, and the active regression test expects that unresolved result even when guild `999` exists.

## Solution Assessment

No. The ordering proposed by the PR is the right narrow behavior, but applying it to deleted/inactive `src/discord` files is not the maintainable fix for current `main`.
