## Summary

Keep open: the Discord allowlist bug still exists on current main, but this PR patches removed legacy files and would not change the active Discord plugin resolver. It also still needs contributor-supplied real behavior proof before merge.

## What This Changes

The PR changes legacy `src/discord` resolver code and tests so bare numeric Discord allowlist entries prefer guild resolution before channel lookup.

## Best Possible Solution

Port guild-first handling for bare numeric Discord group allowlist entries to `extensions/discord/src/resolve-channels.ts` and its tests, while keeping explicit prefixes authoritative and preserving channel fallback.

## Reproduction Assessment

Yes. Source inspection of current main shows `parseDiscordChannelInput` returns `channelId` for a bare numeric entry before the guild path, and the active test asserts `entries: ["999"]` is unresolved even when guild `999` exists.

## Solution Assessment

No. The intended guild-first logic is narrow and maintainable, but this branch applies it to removed `src/discord` files instead of the active `extensions/discord` plugin files.
