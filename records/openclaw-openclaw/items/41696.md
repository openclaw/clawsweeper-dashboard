## What This Changes

The PR changes legacy `src/discord/resolve-channels.*` so bare numeric Discord allowlist entries are checked against joined guild IDs before falling back to channel lookup.

## Best Possible Solution

Move the guild-first parser change and regression coverage to `extensions/discord/src/resolve-channels.*`, preserve explicit prefixes and bare-channel fallback, and require redacted resolver or live Discord proof before merge.

## Reproduction Assessment

Yes, by source inspection: the active parser returns `channelId` for a bare numeric entry, and the active regression test expects `999` to remain unresolved while `guild:999` resolves.

## Solution Assessment

No. The intended guild-first algorithm is plausible, but this branch applies it to removed legacy files; the maintainable fix is the same behavior in the active Discord extension resolver with proof.
