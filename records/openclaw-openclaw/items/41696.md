## Summary

Keep open: the reported bug is still present on current main, but this PR changes the removed legacy `src/discord` resolver instead of the active Discord plugin path, so merging it would not fix runtime behavior.

## What This Changes

The PR changes Discord allowlist parsing so bare numeric entries prefer matching guild IDs before falling back to channel lookup.

## Best Possible Solution

Port the guild-first bare numeric handling and fallback regression coverage into `extensions/discord/src/resolve-channels.ts` and its active tests, preserving explicit `channel:` and `guild:` semantics.

## Reproduction Assessment

Yes, source-reproducible. Current-main source parses a bare numeric input as `channelId`, and the active test still expects `entries: ["999"]` to remain unresolved even when guild `999` exists.

## Solution Assessment

No. The intended behavior is the narrow maintainable fix, but this PR applies it to inactive legacy files; the same logic needs to be moved to the active Discord plugin resolver.
