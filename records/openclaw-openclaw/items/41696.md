## Summary

Keep open: the reported Discord allowlist bug is still valid, but this PR is not correct as-is because it edits removed legacy `src/discord` files while current main and the latest release use the active Discord plugin resolver under `extensions/discord`.

## What This Changes

The PR changes the legacy Discord channel allowlist resolver and tests so bare numeric entries prefer guild resolution before channel lookup.

## Best Possible Solution

Port guild-first handling for bare numeric Discord group allowlist entries to `extensions/discord/src/resolve-channels.ts`, update the active resolver tests for guild match and channel fallback, and keep explicit `channel:`/`guild:` prefixes authoritative.

## Solution Assessment
