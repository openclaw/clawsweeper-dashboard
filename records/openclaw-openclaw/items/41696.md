## What This Changes

The branch changes the legacy `src/discord` allowlist resolver and test so bare numeric IDs resolve against guilds before falling back to channel lookup.

## Best Possible Solution

Implement guild-first handling for bare numeric group allowlist entries in the active Discord plugin resolver and add regression tests for both guild match and channel fallback.

No. The intended behavior change is the narrow fix, but this PR applies it to obsolete legacy files; the maintainable solution is to port the same logic into `extensions/discord/src/resolve-channels.ts` and its tests.
