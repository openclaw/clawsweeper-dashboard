## Summary

Keep open: current main still has the reported external channel secret-contract loader bug, and this PR targets the right code path, but it needs repair before merge because the proposed search order can let `dist/` override an existing root-level fallback and the changelog points at unrelated #77249.

## What This Changes

The PR adds a `dist/` fallback to external channel secret-contract API resolution, a regression test for dist-only npm channel plugins, and a changelog entry for #77241.

## Best Possible Solution

Land one canonical root-first `dist/` fallback for external channel contract APIs with regression coverage and a corrected changelog entry for #77241.

## Reproduction Assessment

Yes. Source inspection shows current main only probes root-level contract files while the npm plugin runtime build emits package entries under `dist/`, and a focused regression test can reproduce the missing `dist/secret-contract-api.cjs` path.

## Solution Assessment

Not yet. The `dist/` fallback is the right narrow fix, but the implementation should preserve root-level resolution precedence across both contract names and fix the changelog reference before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Preserve root-level contract precedence:** `src/secrets/channel-contract-api.ts:90-91`
  - body: The new nested search loop checks `rootDir/dist/secret-contract-api.*` before it ever checks `rootDir/contract-api.*`. That makes a compiled `dist` secret contract override an existing root-level fallback, even though the current resolver can load root `contract-api.*` and the PR says `dist/` is only a fallback. Build the candidate list root-first across both names, then fall back to `dist/`, and cover that precedence with a test.
  - confidence: 0.86

  - body: The new changelog entry ends with `(#77249)`, which points to an unrelated Slack reconnect issue. That would send release-note readers to the wrong discussion, so update the reference for this fix before merge.
