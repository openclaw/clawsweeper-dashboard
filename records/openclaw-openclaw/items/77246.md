## Summary

Keep open: current main still misses `dist/` contract discovery, and this PR addresses the right bug, but the patch should not merge as written because it can let `dist/secret-contract-api.*` override a root-level `contract-api.*` even though `dist/` is meant to be fallback-only.

## What This Changes

The PR adds `rootDir/dist` probing for external channel secret contract files, a dist-only regression test, and a changelog entry for #77241.

## Best Possible Solution

Land one canonical fix that preserves all root-level contract precedence, adds `dist/` only as a fallback, and keeps regression coverage for npm-compiled channel plugins.

## Reproduction Assessment

Yes. Source inspection shows current main only probes root-level contract files, while the npm plugin runtime emits and packages public surfaces under `dist`, so a dist-only external channel contract is missed.

## Solution Assessment

No as written. The fix direction is right, but the implementation should exhaust both root-level basenames before probing `rootDir/dist` so `dist/` remains a true fallback.

## Review Findings

Overall correctness: patch is incorrect
