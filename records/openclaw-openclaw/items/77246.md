## Summary

Keep this PR open, but the current diff should not land as-is because its fallback search order can let a built `dist/secret-contract-api` override an existing root-level `contract-api` even though the PR promises root-level files remain preferred.

## What This Changes

The PR adds a `dist/` fallback for external channel secret-contract API resolution, adds a dist-only regression test, and records the Discord/npm plugin fix in the changelog.

## Best Possible Solution

Land one canonical fix that searches root-level `secret-contract-api` and `contract-api` candidates before falling back to the same basenames under `dist/`, with regression coverage for both dist-only discovery and root-over-dist precedence.

## Reproduction Assessment

Yes. Current main is source-reproducible: the resolver only checks the plugin root, while OpenClaw npm plugin builds and packages public surfaces under `dist/`, including Discord's `secret-contract-api` surface.

## Solution Assessment

No. Adding a `dist/` fallback is the right direction, but this implementation should preserve existing root-level precedence by exhausting root `secret-contract-api` and `contract-api` candidates before probing `dist/`.

## Review Findings
