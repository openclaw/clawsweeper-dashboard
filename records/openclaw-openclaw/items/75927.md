## What This Changes

The PR adds a module-level per-version warning cache in `src/config/io.ts` and an Unreleased changelog entry so the newer-OpenClaw config warning logs once per process.

## Best Possible Solution

Land the narrow per-process future-version warning dedupe with the changelog entry after maintainer review and CI, leaving cross-process coordination out of scope.

## Reproduction Assessment

Yes. A config whose `meta.lastTouchedVersion` is newer than `VERSION` passes `shouldWarnOnTouchedVersion`, and current main can call the same warning helper through both `loadConfig` and `readConfigFileSnapshotInternal` in one process.

## Solution Assessment

Yes. A module-level `Set` keyed by touched version is the narrowest maintainable in-process fix and matches the nearby in-memory logging-dedupe style; cross-process dedupe would need shared state and is not part of this bug.
