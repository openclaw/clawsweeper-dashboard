## Summary

Keep open. The PR addresses a valid source-level WebChat auto-TTS media trust gap, but the patch is incomplete because later TTS-only dispatch paths still reconstruct local audio payloads without the trust flag, and the external real behavior proof is prose-only.

## What This Changes

The PR adds `trustedLocalMedia: true` to auto-TTS payloads returned by `maybeApplyTtsToPayload`.

## Best Possible Solution

Carry the internal local-media trust signal through every OpenClaw-generated auto-TTS local audio payload, including reconstructed final-TTS dispatch payloads, with focused regression coverage and redacted real WebChat playback proof.

## Reproduction Assessment

Yes, at source level. Current main returns auto-TTS local media without `trustedLocalMedia`, while the WebChat local-audio embedding helper drops local audio unless that flag is true; I did not run live WebChat in this read-only review.

## Solution Assessment

No. Adding the flag on the direct `maybeApplyTtsToPayload` return is necessary, but it is not complete because `dispatch-from-config.ts` and `dispatch-acp.ts` rebuild TTS-only media payloads without carrying the flag.
