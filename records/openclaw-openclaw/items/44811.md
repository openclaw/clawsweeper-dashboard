## What This Changes

The PR changes runtime config writeback to project runtime-derived writes onto a fresh disk snapshot and adds a regression test for preserving disk-added `session.dmScope`.

## Best Possible Solution

Land a rebased writer-level fix that uses one current readable existing disk snapshot for both projection and persistence, preserves disk content even when validation fails, and keeps focused regression coverage for the stale runtime snapshot case.

## Reproduction Assessment

Yes, source-level. Set a stale runtime/source snapshot, add `session.dmScope` on disk, then call `writeConfigFile(loadConfig())`; current main projects onto the stale source snapshot and can emit `null` merge-patch deletes for disk-only fields.

## Solution Assessment

No as currently patched. The PR targets the right writer path, but the maintainable fix should use a readable existing disk snapshot even when validation fails and should reuse that snapshot for the inner persistence write.
