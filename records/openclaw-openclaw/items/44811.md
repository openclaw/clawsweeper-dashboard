## Summary

Keep open: current main still has the stale runtime-source projection bug, but this PR is not merge-ready because it lacks real behavior proof, is dirty/conflicting, and leaves writer correctness gaps around invalid-readable disk configs and double-read consistency.

## What This Changes

The PR changes `writeConfigFile` runtime-snapshot projection to use the latest valid on-disk config as the merge base and adds a regression test for preserving disk-added `session.dmScope`.

## Best Possible Solution

Land a rebased writer-level fix that uses one current readable existing disk snapshot for both projection and persistence, preserves disk content even after validation failures, and keeps focused stale-runtime-snapshot regression coverage.

## Reproduction Assessment

Yes, source-level: set a stale runtime/source snapshot, add `session.dmScope` on disk, then call `writeConfigFile(loadConfig())`; current main projects onto the stale source snapshot and can emit `null` merge-patch deletes for disk-only fields.

## Solution Assessment

No as currently patched. The PR targets the right writer path, but the maintainable fix should use a readable existing disk snapshot even when validation fails and should reuse that snapshot for the inner persistence write.

  - confidence: 0.74
