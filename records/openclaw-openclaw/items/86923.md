## Summary

Keep open: the Rastermill direction is plausible, but this PR currently removes an existing image backend override and optimizer-unavailable media passthrough while switching Rastermill to an unreleased GitHub tarball, so it needs compatibility, supply-chain, and real-behavior proof before merge.

The PR replaces OpenClaw-local image resize/encode/probe code with Rastermill `encodeBest`, removes the Photon-local implementation and fallback tests, and updates package locks to a Rastermill GitHub tarball.

## Best Possible Solution

Land the Rastermill-backed refactor only after preserving or explicitly migrating the backend override and unavailable-optimizer passthrough, using a reviewed Rastermill release artifact or approved temporary pin, and adding real behavior proof.

- **[P1] Preserve the image backend override:** `src/media/image-ops.ts:49-61`
  - body: Current main passes `OPENCLAW_IMAGE_BACKEND` into Rastermill and uses it to avoid forced Photon selection when an operator has chosen a backend.   - confidence: 0.9
- **[P1] Keep the unavailable-optimizer passthrough:** `src/media/web-media.ts:766-771`
  - body: The `loadWebMedia` path previously caught image-processor unavailability and returned the original non-HEIC image when it still satisfied the cap and model dimensions. This direct call now propagates backend outages, so a valid image can fail to send on machines without a working Rastermill backend; preserve the fallback or deliberately migrate it with upgrade proof.
  - confidence: 0.86
- **[P1] Use a reviewed Rastermill release artifact:** `package.json:1841`
  - body: This changes the shipped root dependency from the npm `rastermill` release to a GitHub codeload tarball while the package version still says `0.2.0`. That bypasses normal npm release provenance and makes the same version name resolve to different contents, so this needs a published Rastermill version or explicit supply-chain/release approval before merge.
  - confidence: 0.87
