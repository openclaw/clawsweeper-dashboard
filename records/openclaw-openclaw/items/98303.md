## Summary

Keep open: the PR addresses a real, source-reproducible Telegram picker bug, but the current index-only callback can apply the wrong model when an old keyboard is clicked after the provider list shifts, and completed real Telegram proof is still missing.

## What This Changes

The PR changes Telegram model-selection buttons to emit provider/page/index callback data and updates parsing, resolution, and model-button tests for that new callback shape.

## Best Possible Solution

Use a length-safe callback that also binds to the intended model identity or an opaque snapshot key, then require redacted real Telegram proof that an over-limit model remains visible and selectable.

## Reproduction Assessment

Yes. Current main is source-reproducible: an over-limit compact mdl_sel/{model} payload returns null in buildModelSelectionCallbackData, and buildModelsKeyboard skips that row while the handler reports the full model count.

## Solution Assessment

No. The length-safe index direction is plausible, but provider/page/index alone is not the best fix because stale callbacks can apply a shifted model; the safer shape is identity-checked or opaque and proven in real Telegram.
