## Summary

Keep open. This PR addresses the real Telegram model-picker bug tracked by https://github.com/openclaw/openclaw/issues/98221, but it is not merge-ready because the new index-only callback can apply a different model from a stale in-range keyboard and the PR has no after-fix real Telegram proof.

## What This Changes

This PR changes Telegram model-selection buttons to use provider/page/index callback data and updates parsing, resolution, and focused tests for that new callback shape.

## Best Possible Solution

Use a length-safe callback that also binds to the intended model identity or an opaque snapshot key, then prove the over-limit model remains visible and selectable in real Telegram before merge.

## Reproduction Assessment

Yes. Current main is source-reproducible: an over-limit compact mdl_sel/{model} payload returns null in buildModelSelectionCallbackData, and buildModelsKeyboard skips that row while the handler reports the full model count.

## Solution Assessment

No. The length-safe index direction is plausible, but storing only provider/page/index is not the best fix because stale callbacks can apply a shifted model; the safer shape is identity-checked or opaque and proven in real Telegram.
