## Summary

Keep open. The PR now has real Telegram proof for the visible button-list fix, but the new index callback should not merge until it is bound to model identity because a stale in-range keyboard can apply a shifted model.

## What This Changes

The PR changes Telegram model-picker callbacks to use provider/page/index callback data and updates parsing, resolution, and model-button tests.

## Best Possible Solution

Keep the length-safe Telegram callback direction, but add a compact model-identity or opaque-snapshot guard so stale buttons refresh instead of selecting a shifted model.

## Reproduction Assessment

Yes. Current main is source-reproducible: an over-limit compact callback returns null, and the model keyboard skips that row while the caller reports the full model count.

## Solution Assessment

No. Provider/page/index is a plausible length-safe fix, but it is not the best final shape until it also verifies model identity or uses an opaque snapshot key.
