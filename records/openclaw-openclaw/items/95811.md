## What This Changes

Request to add an optional heartbeat transcript artifact mode so HEARTBEAT_OK delivery suppression can stay enabled while model context keeps a compact heartbeat result instead of stripping the whole span.

## Best Possible Solution

Review the linked PR as the canonical implementation path, keeping strip-all as the default and landing only if schema, docs, tests, and upgrade behavior stay aligned.

## Reproduction Assessment

No high-confidence failing bug repro was run, because this is a feature request. Source inspection does confirm the current path: ackMaxChars controls whether a final HEARTBEAT_OK response is treated as an artifact, and recognized heartbeat spans are stripped from prompt history.

## Solution Assessment

Unclear pending maintainer review: decoupling delivery suppression from transcript filtering is the right product target, but adding a new heartbeat config option must pass OpenClaw's config-surface bar. The open linked PR is the concrete path to review rather than closing this issue early.
