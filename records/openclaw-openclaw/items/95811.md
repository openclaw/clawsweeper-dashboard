## Summary

Keep open: current main and v2026.6.9 still feed `ackMaxChars` into both heartbeat delivery suppression and heartbeat transcript artifact filtering, and there is no current `keep-result` style heartbeat transcript policy. This is a real, narrow feature request, but it needs maintainer product direction because it adds compatibility-sensitive heartbeat config behavior.

## What This Changes

The issue requests an optional heartbeat transcript artifact mode that preserves a compact final heartbeat result in model history while stripping poll/tool noise and keeping current behavior as the default.

## Best Possible Solution

Choose a default-preserving heartbeat transcript policy that separates model-history filtering from delivery suppression, then implement it with focused filter, runner, config, docs, and regression coverage.

## Reproduction Assessment

Yes at source level. A heartbeat span whose final assistant text is treated as an ack within `ackMaxChars` is removed by `filterHeartbeatTranscriptArtifacts` before later model requests, while the same threshold is also used for delivery suppression.

## Solution Assessment

Unclear until maintainers choose the product shape. A config-gated mode is plausible, but the repo's config-surface policy means maintainers should first decide whether a new option is warranted or whether structured heartbeat summaries should solve this without another knob.
