## What This Changes

Request to add configurable heartbeat transcript filtering so silent heartbeat runs can keep a compact final result in model context while still suppressing delivery and stripping poll/tool noise.

## Best Possible Solution

Use https://github.com/openclaw/openclaw/pull/95838 as the canonical implementation path, keep the current strip-all behavior as default, and land only after maintainers accept the new heartbeat transcript-mode contract.

## Reproduction Assessment

No high-confidence failing bug reproduction was run because this is a feature request, not a broken existing contract. Source inspection does confirm the current behavior: recognized heartbeat spans are stripped from prompt history based on the same ackMaxChars threshold used for delivery suppression.

## Solution Assessment

Unclear pending maintainer review. Decoupling delivery suppression from transcript/context filtering is the right target if maintainers want this workflow, and the linked PR is the concrete path, but a new heartbeat config contract needs explicit product acceptance.
