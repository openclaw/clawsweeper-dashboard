## What This Changes

This PR removes the hardcoded `fastMode: true` from hidden commitment extraction and adds a regression assertion that the embedded-agent request omits `fastMode`.

## Best Possible Solution

Resolve commitment extraction fast mode through `resolveFastModeState`, pass `fastMode` and the auto cutoff into `runEmbeddedAgent`, keep the default off, and add regression tests for default-off plus explicit opt-ins.

## Reproduction Assessment

Yes, at source level: current main passes `fastMode: true` from hidden commitment extraction, and the MiniMax wrapper rewrites `MiniMax-M2.7` to `MiniMax-M2.7-highspeed` when fast mode is true. I did not run a live MiniMax Coding Plan account in this read-only review.

## Solution Assessment

No. Removing the hardcoded field is directionally right, but the best fix is to resolve and pass the shared fast-mode state so default-off and documented explicit opt-ins both work.
