## What This Changes

The PR removes the hardcoded `fastMode: true` from hidden commitments extraction and adds a regression assertion that the embedded-agent request omits `fastMode`.

## Best Possible Solution

Resolve hidden commitment extraction fast mode through the shared resolver after selecting the commitment model, pass the effective `fastMode` and auto cutoff into `runEmbeddedAgent`, add default-off and configured-opt-in regression coverage, then provide current-head or live after-fix proof.

## Reproduction Assessment

## Solution Assessment

No. Removing forced `true` is directionally right, but the best fix should use the shared resolver so default-off MiniMax plans and explicit configured fast-mode opt-ins both behave as documented.
