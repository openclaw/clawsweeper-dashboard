## What This Changes

The PR removes `fastMode: true` from the commitments embedded-agent payload, adds a regression assertion that the payload omits `fastMode`, and adds a changelog fix entry for the MiniMax non-highspeed plan failure.

## Best Possible Solution

Pass `resolveFastModeState({ cfg, provider: modelRef.provider, model: modelRef.model, agentId: first.agentId }).enabled` into commitment extraction, cover default-false and configured-fast cases, then merge after redacted real extraction proof is added.

## Reproduction Assessment

## Solution Assessment

No. Removing the key fixes the default non-highspeed symptom, but the maintainable fix is to pass the existing fast-mode resolver output so configured-fast agents keep their documented behavior.
