The PR removes the hardcoded `fastMode: true` from hidden commitment extraction and adds a regression assertion that the embedded-agent request omits `fastMode`.

## Best Possible Solution

Commitment extraction should remove the unconditional true default, pass the shared resolved fast-mode value for the selected provider/model/agent, and then merge with exact-head redacted runtime proof.

## Reproduction Assessment

Yes, at source level. Current main hardcodes `fastMode: true` in hidden commitment extraction, and the MiniMax wrapper rewrites `MiniMax-M2.7` to `MiniMax-M2.7-highspeed` when that flag is true.

## Solution Assessment

No. The default-off direction is right, but the patch should use the shared fast-mode resolver so explicit per-agent and per-model opt-ins are preserved.
