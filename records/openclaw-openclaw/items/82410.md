## Summary

Keep open, but the current patch is not mergeable: it reads `reasoning_content` from `AssistantMessage`, which current types and CI reject, while the replay contract stores reasoning as `thinking` content blocks with `thinkingSignature`. The PR also lacks required after-fix real behavior proof for the external proxy path.

## What This Changes

The PR changes the Anthropic messages transport to copy a non-empty `reasoning_content` property from assistant messages into serialized assistant payloads.

## Best Possible Solution

Rewrite the fix against the existing reasoning-block replay contract, add focused Anthropic transport regression coverage, and include redacted real provider proof before merge.

## Reproduction Assessment

Yes for the PR defect: the changed lines fail `pnpm tsgo:prod` in CI with TS2339, and source inspection shows the field is not part of `AssistantMessage`. No real setup proof is available for the original proxy failure after this patch.

## Solution Assessment

No. The current patch reads a non-existent top-level field; the safer fix should convert or preserve replay reasoning from `content` thinking blocks, with provider-aware tests and real proxy evidence.
