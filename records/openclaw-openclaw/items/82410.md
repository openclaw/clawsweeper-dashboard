## Summary

Keep open: the provider compatibility problem is plausible and related DeepSeek/Xiaomi reasoning-content work exists, but this patch is not ready because it lacks after-fix real behavior proof and appears to read the wrong replay representation while changing the generic Anthropic payload shape.

## What This Changes

The PR changes `src/agents/anthropic-transport-stream.ts` to include a top-level `reasoning_content` value on serialized assistant messages when one is present on the source message object.

## Best Possible Solution

Add a provider/model-scoped Anthropic-compatible proxy replay path that captures and round-trips the provider's actual reasoning format while leaving stock Anthropic on native thinking blocks, with focused tests and redacted live proof.

## Reproduction Assessment

No high-confidence live reproduction is present. Source inspection shows the current Anthropic converter only emits assistant `role` and `content` from content blocks, but the PR does not prove the failing Xiaomi/MiMo proxy path after the fix.

## Solution Assessment

No. The patch preserves a top-level field that current OpenClaw does not normally produce on Anthropic assistant messages and does so without provider gating; a safer fix would translate the existing `thinkingSignature: "reasoning_content"` block only for affected proxy models and verify it with tests plus real provider proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Read replayed reasoning from content blocks:** `src/agents/anthropic-transport-stream.ts:390-391`
  - body: OpenClaw stores replayed provider reasoning in assistant `content` blocks, for example as a `thinking` block with `thinkingSignature: "reasoning_content"`; the transport output and replay transform do not promote that value to `msg.reasoning_content`. This branch therefore will not preserve the value the failing proxy needs on normal replayed turns.
  - confidence: 0.88
- **[P2] Gate proxy-only fields away from stock Anthropic:** `src/agents/anthropic-transport-stream.ts:390-391`
  - body: This adds `reasoning_content` to every Anthropic assistant payload whenever that extra property exists, but the public Anthropic contract uses `role`/`content` messages and native `thinking` blocks for reasoning replay. A proxy-specific field should be gated to the affected provider/model so stale or proxy-shaped transcripts cannot change direct Anthropic requests.
  - confidence: 0.78
