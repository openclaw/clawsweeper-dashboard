## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P2] Keep the Node floor at 22.19+:** `docs/start/getting-started.md:21`
  - body: The new prerequisite line says Node 22.14+ is supported, but current main documents Node 22.19+ and `package.json` enforces `>=22.19.0`. Merging this would send first-time users to an unsupported Node floor.
  - confidence: 0.96
- **[P2] Scope the installer bootstrap claim:** `docs/start/getting-started.md:38`
  - body: This says the install script auto-installs anything missing, but the POSIX installer still needs an existing downloader and only remediates specific setup pieces. The quickstart should not imply every listed prerequisite can be skipped.
  - confidence: 0.94
- **[P2] Check command status before piping to head:** `docs/start/getting-started.md:46`
  - body: The fallback is attached to `head`, so a missing command prints the shell error but never emits the promised `MISSING` marker. Capture the command output and status before truncating to the first line.
  - confidence: 0.98
- **[P2] Probe Windows tools before running cmd:** `docs/start/getting-started.md:65`
  - body: Wrapping `cmd /c` in `catch` does not reliably report missing or failing native tools because nonzero external processes can return a status without throwing. Use `Get-Command` or explicit status handling first.
  - confidence: 0.91
- **[P3] Do not require Bash 4 on macOS:** `docs/start/getting-started.md:25`
  - body: The installer intentionally preserves macOS Bash 3.2 compatibility, so the quickstart should not add a stricter Bash 4+ prerequisite for the installer path.
  - confidence: 0.87
