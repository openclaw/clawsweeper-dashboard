Full review comments:

- **[P2] Keep the Node floor at 22.19+:** `docs/start/getting-started.md:21`
  - body: The new prerequisite line says Node 22.14+ is supported, but current main's package engine and public install docs require Node 22.19+. Merging this would point first-time users at an unsupported runtime.
  - confidence: 0.97
- **[P2] Scope the installer auto-install claim:** `docs/start/getting-started.md:17-18`
  - body: The added text says the installer auto-installs anything missing, but the POSIX installer still needs an existing downloader and only remediates specific dependencies such as Node and Git. Please narrow the claim so users do not skip prerequisites the installer cannot bootstrap.
  - confidence: 0.94
- **[P2] Check command status before piping to head:** `docs/start/getting-started.md:46`
  - body: The `|| echo "MISSING"` fallback is attached to `head`, so a missing command prints the shell error and never emits the promised `MISSING` marker. Capture the command output/status before truncating it to the first line.
  - confidence: 0.98
- **[P2] Probe Windows tools before invoking cmd:** `docs/start/getting-started.md:65`
  - body: Wrapping `cmd /c` in `catch` does not reliably classify missing external tools because failed external processes can return a nonzero status without throwing. Use `Get-Command` or explicit status handling before running each command.
  - confidence: 0.9
- **[P3] Do not require Bash 4 for the quickstart:** `docs/start/getting-started.md:25`
  - body: The installer is launched with `/bin/bash` and current guidance supports macOS, where the system Bash is still 3.2. Requiring Bash 4 here makes the docs stricter than the installer path.
  - confidence: 0.84
