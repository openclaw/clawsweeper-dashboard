## Summary

Keep this PR open. Current main still lacks the requested Step 0 prerequisite guidance, but the draft is not merge-ready because its runnable Bash and PowerShell snippets do not reliably mark missing commands and its installer wording overstates what the installers can auto-remediate.

## What This Changes

The PR modifies `docs/start/getting-started.md` to add a prerequisite list, Step 0 macOS/Linux and Windows verification snippets, per-platform install hints, and a note that `pnpm` is only needed for source builds.

## Best Possible Solution

Keep the Step 0 docs direction, but update this draft so it states the actual installer contract, uses command-existence/status checks before formatting output in both shells, and validates the final snippets plus Mintlify syntax before merge.

## Reproduction Assessment

Yes for the blocking review findings. The Bash failure reproduces locally with the PR pipeline shape, the PowerShell issue is supported by shell-wrapper reproduction and existing review comments, and the installer-scope claim is checked directly against current installer source.

## Solution Assessment

No, not as written. A narrow docs-only correction is the best path: keep the prerequisite guidance, but align the claims and snippets with the current POSIX and Windows installer contracts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Check command status before piping to head:** `docs/start/getting-started.md:46`
  - body: The fallback is attached after `head`, so a missing command prints the shell error and never emits the promised `MISSING` marker. This makes the diagnostic misleading for the exact prerequisite failure it is supposed to clarify.
  - confidence: 0.96
- **[P2] Probe executables before using cmd:** `docs/start/getting-started.md:65`
  - body: `cmd /c` can return a nonzero status for a missing tool without entering PowerShell `catch`, so Windows users also do not get the promised `MISSING` marker. Use `Get-Command` or explicit exit-status handling before running each command.
  - confidence: 0.89
- **[P2] Scope auto-install wording to supported dependencies:** `docs/start/getting-started.md:36`
  - body: The docs say the installer auto-installs anything missing, but the current installers only remediate specific setup pieces such as Node, Git, and source-install `pnpm`; bootstrap tools like a downloader and shell still need to be present. This overpromises first-run behavior.
  - confidence: 0.93
