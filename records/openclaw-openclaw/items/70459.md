## Summary

Keep this draft PR open. The documentation direction is useful and current main still lacks the proposed Step 0 prerequisite checker, but the current patch is not merge-ready because both runnable checker snippets can misreport missing tools and the installer auto-install wording overstates the actual POSIX and Windows installer contracts.

## What This Changes

The PR modifies `docs/start/getting-started.md` to add prerequisite checks, platform-specific install hints, Step 0 verification snippets, and a note that `pnpm` is only needed for source builds.

## Best Possible Solution

Update the draft in place so the getting-started docs accurately describe installer-scoped remediation, use explicit command availability/status checks for both shells, and publish after target-shell and Mintlify rendering validation.

## Reproduction Assessment

Yes for the blocking review findings. The Bash failure reproduces locally with the proposed pipeline and a missing command, and a PowerShell probe confirms external nonzero exits do not enter `catch`, though native Windows validation should still be completed.

## Solution Assessment

No, not as written. The proposed documentation direction is appropriate, but the current snippets and auto-install wording are not the narrowest maintainable solution until they match the actual installer contracts and shell behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Check command status before piping to head:** `docs/start/getting-started.md:46`
  - body: The fallback is attached to `head`, not the checked command. A missing command prints the shell error and the loop never emits the promised `MISSING` marker, so the diagnostic snippet gives unclear results for the exact case it is meant to handle.
  - confidence: 0.96
- **[P2] Probe executables before using cmd:** `docs/start/getting-started.md:65`
  - body: `cmd /c` can return a nonzero status for a missing tool without entering PowerShell `catch`, so Windows users also do not get the promised `MISSING` marker. Use `Get-Command` or explicit status handling before running each command.
  - confidence: 0.88
- **[P2] Scope the auto-install claim to supported dependencies:** `docs/start/getting-started.md:38`
  - body: The docs say the installer auto-installs anything missing, but the installers only remediate specific setup pieces such as Node, Git, and source-install pnpm; bootstrap dependencies like curl/wget still have to exist. This overpromises first-run behavior.
  - confidence: 0.93
