## Summary

This draft PR should stay open, but it is not merge-ready: the proposed docs still conflict with current Node requirements, overstate installer remediation, and include runnable prerequisite snippets that fail to report missing tools reliably.

## What This Changes

The PR expands `docs/start/getting-started.md` with prerequisite checks, Step 0 macOS/Linux and Windows verification snippets, install hints, and a note that `pnpm` is source-build only.

## Best Possible Solution

Keep the Step 0 docs direction, but rebase it onto current main, align the wording with the Node 22.16+ package/docs contract and installer remediation scope, fix the snippets, and attach rendered docs plus redacted terminal proof before merge.

## Reproduction Assessment

Yes for the blocking PR findings: the Bash missing-tool failure reproduces with the PR's pipeline shape, and the PowerShell issue is source-backed by the proposed `cmd /c` try/catch and external-process status behavior. This is a PR patch review, not a current-main product regression.

## Solution Assessment

No, the PR is not the best solution as written. The narrow maintainable path is to preserve the prerequisite-check docs idea while correcting the contracts, command snippets, and validation proof.

- **[P2] Keep the Node floor at 22.16+:** `docs/start/getting-started.md:21`
  - body: The new prerequisite line says Node 22.14+ is supported, but current main's package engine and public docs require Node 22.16+. Merging this would send first-time users to an unsupported Node floor.
  - confidence: 0.94
- **[P2] Scope the installer auto-install claim:** `docs/start/getting-started.md:38`
  - body: The new text says the install script auto-installs anything missing, but the POSIX installer still needs an existing downloader and only remediates specific setup pieces such as Node, Git, and source-install pnpm. This overpromises the first-run path.
  - confidence: 0.93
- **[P2] Check command status before piping to head:** `docs/start/getting-started.md:46`
  - body: The `|| echo "MISSING"` fallback is attached to `head`, so a missing command prints the shell error and never emits the promised `MISSING` marker. Capture the command output/status before truncating to the first line.
  - confidence: 0.98
- **[P2] Probe executables before calling cmd:** `docs/start/getting-started.md:65`
  - body: On Windows, this relies on `catch` around `cmd /c` to report missing tools, but failed external processes do not reliably enter that branch. Use `Get-Command` or explicit status handling before running each command.
  - confidence: 0.9
- **[P3] Do not require Bash 4 on macOS:** `docs/start/getting-started.md:25`
  - body: The new prerequisite says macOS/Linux users need Bash 4+, but the current installer deliberately keeps compatibility with macOS's Bash 3.2. This would make the quick-start prerequisites stricter than the installer actually requires.
  - confidence: 0.86
