## Summary

Keep open. The docs direction is useful, but this draft/conflicting branch currently adds misleading prerequisite claims and runnable snippets that fail their own missing-dependency goal, and the external PR still lacks after-fix real behavior proof.

## What This Changes

The PR adds dependency prerequisites, copy-paste version checks, install hints, and a pnpm note to `docs/start/getting-started.md`.

## Best Possible Solution

Rebase onto current main, keep the Node 22.16+ contract, scope installer auto-install wording to the real installer behavior, fix missing-command detection in both snippets, and verify the rendered page plus the commands.

## Reproduction Assessment

Yes for the review defects: the Bash one-liner can be run with a missing command and fails to print `MISSING`, and source inspection confirms the auto-install wording overstates `install.sh`. The PowerShell issue is source-reproducible from native-command failure behavior and matches the existing review comment.

## Solution Assessment

No, not in the current form. A narrower maintainable solution should align this page with `docs/install/*`, `package.json`/runtime Node requirements, and snippets that check command presence before invocation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep the Node floor at 22.16+:** `docs/start/getting-started.md:21`
  - body: This new prerequisite line says Node 22.14+ is supported, but current main and the package/runtime guard require Node >=22.16.0. Rebase and keep the current 22.16+ wording so first-time users are not sent to an unsupported runtime.
  - confidence: 0.93
- **[P2] Scope the installer auto-install claim:** `docs/start/getting-started.md:38`
  - body: `install.sh` must already have `curl` or `wget` before it can fetch anything, and its main flow only remediates Node and Git. Saying it auto-installs anything missing overpromises for curl, bash, openssl, and pnpm, so the Step 0 text should describe the narrower supported behavior.
  - confidence: 0.91
- **[P2] Make the Bash missing check test eval:** `docs/start/getting-started.md:46`
  - body: Here `|| echo "MISSING"` is attached to the pipeline after `head`, so a missing command prints the shell error and still exits through `head` without the promised marker. Capture the command output/status before piping or check command presence first.
  - confidence: 0.95
- **[P2] Check PowerShell commands before invoking them:** `docs/start/getting-started.md:65`
  - body: `cmd /c` returning a native nonzero status does not make this `catch` block report a missing inner executable on Windows. Use `Get-Command` for the executable before running it so missing Node/Git/OpenSSL produce the documented `MISSING` output.
  - confidence: 0.88
