## Summary

Keep this PR open, but it is not merge-ready: the intended zod bundling fix is plausible, but the current merge result overwrites an existing `alwaysBundle` rule and the PR lacks after-fix proof from a real pnpm global install.

## What This Changes

The PR adds `deps.alwaysBundle: ["zod"]` to `tsdown.config.ts` so `openclaw/plugin-sdk/zod` is bundled instead of leaving a bare runtime `zod` import.

## Best Possible Solution

Fold zod into the existing `shouldAlwaysBundleDependency` path, add focused bundle coverage that rejects a bare `zod` import in `plugin-sdk/zod.js`, add the required changelog entry, and verify a packed/global pnpm install without a symlink workaround.

## Reproduction Assessment

Yes from source inspection, but not from a live run in this review. `plugin-sdk/zod.ts` re-exports `zod`, package dependencies are external by default, and the linked issue reports the published `dist/plugin-sdk/zod.js` failing under pnpm global installs.

## Solution Assessment

No, not as currently merged. Bundling zod is the right direction, but the patch must preserve the existing fs-safe `alwaysBundle` predicate and include real install proof.
