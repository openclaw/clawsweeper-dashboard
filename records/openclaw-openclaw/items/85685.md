## Summary

Close as duplicate: the same zero-LLM cron direct-exec/fixed-wrapper request is already the open canonical tracker at https://github.com/openclaw/openclaw/issues/18160, and this issue adds no separate remaining behavior that needs its own thread.

## What This Changes

The issue requests a deterministic cron payload that runs an approved fixed local wrapper directly, captures output and exit status, and uses cron delivery/run history without invoking a model.

## Best Possible Solution

Keep https://github.com/openclaw/openclaw/issues/18160 as the canonical tracker and settle the fixed-wrapper/direct-exec cron design there with security-reviewed approval policy, run-history, and delivery semantics.

## Reproduction Assessment

Yes, for source-level capability absence rather than a bug repro. Current main only accepts systemEvent or agentTurn cron payloads, and isolated/current/session jobs are validated and executed through agentTurn.

## Solution Assessment

No, not as a separate tracker. The maintainable path is to consolidate this narrower fixed-wrapper wording into the already-open direct-exec cron design at https://github.com/openclaw/openclaw/issues/18160.
