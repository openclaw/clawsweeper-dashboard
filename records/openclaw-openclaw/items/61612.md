## Summary

Keep open. The performance direction is useful, but the current patch still has compatibility blockers around plugin CLI help fallback and pairing help channel data, and the branch is now conflicting with current main after substantial CLI/plugin-loader refactors.

## What This Changes

The PR adds a CLI help fast path for built-in and plugin subcommands and switches help-only plugin CLI loading toward metadata registration with focused tests.

## Best Possible Solution

Rebuild the fast path on current main using the existing descriptor and activation-planning loaders while retaining runtime fallback and pairing channel help semantics.

## Reproduction Assessment

Yes for the review findings by source inspection: the PR routes fallback help registration through metadata-only loading, and pairing help registers before channel-providing plugins can populate `listPairingChannels()`. I did not run the CLI because this review is read-only.

## Solution Assessment

No. The fast-path goal is reasonable, but this patch should be repaired or replaced on current main so it does not drop runtime-backed plugin help or pairing help channel data.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve runtime plugin help fallback:** `src/cli/run-main.ts:345`
  - body: Passing `helpOnly` for every help invocation routes fallback plugin command registration through metadata-only entries. Runtime-only third-party plugin CLIs that do not expose complete `cli-metadata` can lose their help surface, so keep the full runtime fallback unless the primary command is known to be descriptor-backed and safely materialized from metadata.
  - confidence: 0.88
- **[P2] Keep pairing help channel data:** `src/cli/program/register.subclis.ts:232-237`
  - body: This early return registers `pairing` help before plugin CLI registration, but `registerPairingCli()` snapshots `listPairingChannels()` into option and argument help. Plugin-backed channels can disappear from fresh help output, so preserve the registration or metadata source needed before rendering pairing help.
  - confidence: 0.84
