## Summary

Keep open. The performance goal is useful, but the current PR head still has two compatibility regressions: plugin help fallback can drop runtime-only plugin CLIs, and pairing help can lose plugin-provided channel data.

## What This Changes

The PR adds faster CLI help handling for built-in and plugin subcommands by adding early help paths and metadata-only plugin CLI registration.

## Best Possible Solution

Rebuild the help performance work on current main using manifest/descriptor-owned command lookup and activation-planning loaders while preserving runtime plugin fallback and pairing channel help semantics.

## Reproduction Assessment

Yes for the review findings by source inspection. The PR routes fallback plugin help registration through metadata-only loading, and pairing help returns before plugin-backed channel data can be populated for registerPairingCli().

## Solution Assessment

No. The hardcoded plugin map and broad helpOnly switch are not the safest implementation; the repair should use current-main owner/descriptor resolution and only take metadata-only paths when compatibility is known.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve runtime plugin help fallback:** `src/cli/run-main.ts:345`
  - body: Passing `helpOnly: hasHelpOrVersion(parseArgv)` makes fallback help invocations use the metadata-only loader. Runtime-only third-party plugin CLIs, or plugins with incomplete CLI metadata, can lose their `openclaw <plugin> --help` command where the full runtime registry path worked before, so keep the runtime fallback except for roots proven safe from metadata.
  - confidence: 0.88
- **[P2] Keep pairing help channel data:** `src/cli/program/register.subclis.ts:235-237`
  - body: This returns before the plugin registration that populates channel plugins, but `registerPairingCli()` snapshots `listPairingChannels()` into option and argument help. Fresh pairing help can render empty or incomplete channel lists for plugin-backed pairing channels, so preserve a channel metadata source before registering the help command.
  - confidence: 0.8
