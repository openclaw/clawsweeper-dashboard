## Summary

Keep open. The performance direction is useful and current main still does not have a general built-in/plugin subcommand help fast path, but the proposed patch regresses supported plugin CLI fallback behavior and can make pairing help omit plugin-backed channel data.

## What This Changes

The PR adds subcommand help fast-path helpers, a plugin CLI `helpOnly` metadata-loader path, special pairing/plugins help short-circuits, and focused CLI/plugin tests.

## Best Possible Solution

Keep the fast-path direction, but make it descriptor-aware and conservative: use metadata-only loading only for known descriptor-backed plugin commands, preserve the full runtime CLI fallback for commands-only plugins, and keep pairing help populated from plugin-backed channel metadata.

## Reproduction Assessment

Yes for the review findings: the relevant paths are traceable in source and tests. Live confirmation would run `openclaw pairing --help` with a plugin-backed pairing channel and `openclaw <commands-only-plugin> --help`, but this review stayed read-only.

## Solution Assessment

No. The proposed approach is narrower than the old full bootstrap for known descriptor-backed commands, but the global `helpOnly` switch and pairing short-circuit are broader than the documented compatibility contract allows.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve runtime fallback for plugin help:** `src/cli/run-main.ts:345`
  - body: Passing `helpOnly` for every help invocation forces plugin CLI registration onto the metadata-only registry. Plugins that expose CLI commands only from their normal runtime entry remain supported by current docs/tests, so those commands can disappear for `openclaw <plugin-command> --help`. Keep the runtime loader fallback unless the command is known to be descriptor-backed.
  - confidence: 0.88
- **[P2] Keep pairing help channel data:** `src/cli/program/register.subclis.ts:232-235`
  - body: `registerPairingCli()` snapshots `listPairingChannels()` when the command is registered and uses it in help text. Returning before plugin-backed channel registration means valid plugin channels can be omitted from `openclaw pairing --help`, producing incomplete channel choices.
  - confidence: 0.86
- **[P3] Add the required changelog entry:** `src/cli/run-main.ts:88`
  - body: This is a user-facing CLI performance change, but the PR does not include a `CHANGELOG.md` entry. OpenClaw policy requires one for `perf` changes before merge, with appropriate contributor attribution.
  - confidence: 0.78
