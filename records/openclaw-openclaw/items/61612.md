## Summary

Keep open. The performance goal is useful, but the latest diff still has source-reproducible compatibility blockers around runtime-only plugin CLI help and pairing help channel data, and current main has newer activation-aware plugin CLI loading that the replacement should build on.

## What This Changes

The PR adds CLI subcommand help fast paths, metadata-only plugin help loading, pairing/plugins help shortcuts, and focused CLI/plugin tests.

## Best Possible Solution

Implement a descriptor- and activation-aware help fast path on current main that avoids heavy bootstrap only for known safe command surfaces while preserving runtime fallback and pairing channel registry population.

## Reproduction Assessment

Yes for the review findings: source inspection of the PR diff and current contracts shows runtime-only plugin CLI help can be skipped, and pairing help snapshots channel registry data during registration.

## Solution Assessment

No. The fast-path direction is sound, but this diff is not the safest implementation on current main; it should build on the activation-aware plugin CLI loader and keep fallback/runtime-owned help paths intact.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve runtime plugin help fallback:** `src/cli/run-main.ts:345`
  - body: Passing `helpOnly` for every help invocation sends fallback plugin command registration through metadata-only entries. Plugins that expose CLI commands only from their runtime entry can lose `openclaw <plugin> --help`, so keep the full runtime fallback unless the command is known descriptor-backed.
  - confidence: 0.88
- **[P2] Keep pairing help channel data:** `src/cli/program/register.subclis.ts:232-235`
  - body: This short-circuits `pairing --help` before plugin CLI registration, but `registerPairingCli()` snapshots `listPairingChannels()` into option and argument help. Plugin-backed channels can disappear from help text, so preserve the registration needed for pairing help.
  - confidence: 0.86
