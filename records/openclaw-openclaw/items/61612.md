## Summary

Keep open: the performance goal is useful, but the PR still has a compatibility regression in the plugin help fallback. Current main preserves command-only plugin CLI registration through the full loader, while this PR switches all fallback help registration to metadata-only loading.

## What This Changes

The PR adds CLI subcommand help fast paths and a `helpOnly` plugin CLI mode that uses metadata-only plugin loading for help output.

## Best Possible Solution

Rebuild the help performance work on current main using manifest/descriptor-owned command lookup while preserving the full-loader fallback for supported command-only plugin CLIs.

## Reproduction Assessment

Yes for the review finding by source inspection. The PR routes help-time fallback registration through metadata-only loading even though current docs and tests preserve command-only plugin CLI registrations through the full-loader compatibility path.

## Solution Assessment

No. The performance approach is reasonable, but metadata-only help should be used only when descriptor or manifest ownership proves it will not drop supported command-only plugin CLIs.
