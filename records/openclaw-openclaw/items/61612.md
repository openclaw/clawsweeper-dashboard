## Summary

Keep open: the help-performance direction is useful and the PR includes after-fix benchmark evidence, but the current patch still has a plugin CLI compatibility regression and needs to be rebuilt on top of current main's registration path.

## What This Changes

This PR adds CLI help fast paths for built-in and plugin subcommands and changes plugin CLI help registration to use metadata-only loading.

## Best Possible Solution

Rebuild the help fast path on current main using manifest/descriptor-owned metadata paths while preserving the full-loader fallback for supported command-only plugin CLIs.

## Reproduction Assessment

Yes for the review finding by source inspection. The PR routes help-time fallback registration through metadata-only loading even though current docs and tests preserve command-only plugin CLI registrations through the full loader compatibility path.

## Solution Assessment

No. The performance approach is reasonable, but the implementation should only use metadata-only help paths when descriptor or manifest ownership proves the command is metadata-backed.

## Review Findings

Overall confidence: 0.84
