## Summary

Keep open: the performance goal is plausible, but the PR is not safe as-is because the new fast path can misread pass-through `--help` arguments and the metadata-only plugin help fallback can drop supported command-only plugin CLIs.

## What This Changes

The PR adds CLI subcommand help fast paths plus a help-only metadata plugin CLI registration mode for built-in and plugin-owned commands.

## Best Possible Solution

Rebuild the help performance work on current main using invocation-aware help detection and manifest/descriptor-owned lookup, while preserving the full non-activating loader path for supported command-only plugin CLIs.

## Reproduction Assessment

Yes for the review findings by source inspection. The PR uses a terminator-insensitive help helper in a parse-path decision and routes fallback plugin help through metadata-only loading despite the documented commands-only compatibility path.

## Solution Assessment

No. The fast-path direction is reasonable, but the implementation should use `resolveCliArgvInvocation`/`isHelpOrVersionInvocation` semantics and only skip the full plugin CLI loader when manifest or descriptor ownership proves metadata-only help is sufficient.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Honor -- before taking help fast paths:** `src/cli/run-main.ts:91-92`
  - body: `shouldUseSubcommandHelpFastPath` calls `hasHelpOrVersion`, which treats `--help` after `--` as a help invocation. That lets commands such as `openclaw <subcommand> -- --help` get intercepted by the minimal help parser instead of reaching the command; base this decision on the invocation-aware help parser so pass-through args remain data.
  - confidence: 0.87
- **[P2] Preserve command-only plugin help fallback:** `src/cli/run-main.ts:345`
  - body: Passing `helpOnly` for every fallback help invocation sends plugin registration through metadata-only discovery. The SDK docs and current tests keep commands-only `api.registerCli` as a supported eager compatibility path, so plugin CLIs without descriptors can disappear from help fallback; only use metadata-only after manifest/descriptor ownership proves it is safe, or keep the full non-activating loader fallback.
  - confidence: 0.84
