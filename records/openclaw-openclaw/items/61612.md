## Summary

Keep open. The performance direction is useful, but the current diff changes help invocations to metadata-only plugin loading and skips pairing plugin setup in ways that can remove existing plugin CLI and channel help behavior.

## What This Changes

Adds early CLI subcommand help fast paths plus metadata-only plugin CLI help registration and focused CLI/plugin tests.

## Best Possible Solution

Keep the optimization, but rebase it onto current main as a descriptor-aware fast path that preserves full-loader fallback and complete pairing help.

## Reproduction Assessment

Yes. Static review reproduces the blockers: the PR diff routes all plugin help through `helpOnly`, while current tests/docs preserve full-loader CLI compatibility and pairing help snapshots channel data at registration time.

## Solution Assessment

No. The fast path is the right direction, but the current implementation is too broad; it should only use metadata-only loading when the command is descriptor-backed and should not drop pairing channel registration data.
