## Summary

Keep open: the performance direction is useful and the PR body includes after-fix benchmark proof, but the latest diff is not safe to merge because the fallback help path can drop supported command-only plugin CLIs, and the branch now overlaps/conflicts with newer main-line CLI registration work.

## What This Changes

The PR adds fast paths for built-in and plugin subcommand help and changes plugin CLI help registration to use metadata-only loading.

## Best Possible Solution

Rebuild the help performance work on current main using manifest/descriptor-owned command lookup while preserving the full-loader fallback for supported command-only plugin CLIs.

## Reproduction Assessment

Yes for the review finding by source inspection. The PR routes help-time fallback registration through metadata-only loading even though current docs and tests preserve command-only plugin CLI registrations through the full loader compatibility path.

## Solution Assessment

No. The performance approach is reasonable, but the implementation should only use metadata-only help paths when descriptor/manifest ownership proves that doing so will not drop supported command-only plugin CLIs.
