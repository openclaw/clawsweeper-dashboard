## Summary

Keep open. Current main has some newer lazy CLI routing, but it does not fully implement this PR's built-in and plugin subcommand help fast path; the PR is still relevant, but the latest diff has merge-blocking correctness issues around plugin help fallback and pairing help data.

## What This Changes

The PR adds early CLI help fast-path helpers, a plugin CLI `helpOnly` metadata-loader option, special handling for `pairing` and `plugins` help, and focused unit tests for those paths.

## Best Possible Solution

Keep the fast-path direction, but make it descriptor-aware and conservative: use metadata-only registration only when the target command is known to be represented by CLI metadata, preserve full runtime fallback for compatibility commands, and keep pairing help populated from plugin-backed channel metadata without loading unnecessary runtime modules.

## Reproduction Assessment

Yes for the review findings: the code path is traceable from the diff and current contracts without running tests. A live reproduction would be `openclaw pairing --help` with a plugin-backed pairing channel, plus a plugin command that registers only through the normal runtime entry, but this read-only review did not execute commands.

## Solution Assessment

No. The proposed approach is promising for descriptor-backed commands, but the global `helpOnly` switch and pairing help short-circuit are broader than the compatibility contract allows; a narrower metadata fast path with runtime fallback is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve runtime fallback for plugin help:** `src/cli/run-main.ts:345`
  - body: Passing `helpOnly` for every help invocation forces plugin CLI registration onto the metadata-only registry. Plugins that expose CLI commands only from their normal runtime entry, which current docs/tests keep as supported eager compatibility, will disappear for `openclaw <plugin-command> --help` and can become unknown commands. Keep the runtime loader fallback unless the command is known to have metadata descriptors.
  - confidence: 0.88
- **[P2] Keep pairing help channel data:** `src/cli/program/register.subclis.ts:232-235`
  - body: `registerPairingCli()` builds its channel option/help text from `listPairingChannels()` at registration time. Returning before plugin-backed channel registration means valid plugin channels can be omitted from `openclaw pairing --help`, producing incomplete help for users who rely on those channels.
  - confidence: 0.84
