## Summary

Keep open: the PR targets a useful CLI help performance gap that is not fully solved on current main, but the branch still needs repair because its help-only plugin registration can drop command-only plugin CLI surfaces.

## What This Changes

The PR adds fast paths for built-in and plugin subcommand help and routes help-time plugin CLI registration through metadata-only loading with focused tests.

## Best Possible Solution

Rebuild the help performance work on current main using manifest/descriptor-owned command lookup while preserving the documented eager compatibility path for command-only plugin CLI registrations.

## Reproduction Assessment

Yes for the review finding by source inspection. The PR routes help-time plugin registration through metadata-only loading even though current docs and tests still preserve command-only plugin CLI registrations through the full loader compatibility path.

## Solution Assessment

No. The performance direction is reasonable, but the implementation should use current main's plugin-owner resolution and only take metadata-only paths when that does not drop supported command-only plugin CLIs.
