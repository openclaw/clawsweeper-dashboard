## Summary

Keep open: the performance direction is useful, but the branch still violates current CLI contracts by using terminator-insensitive help detection and by routing fallback plugin help through metadata-only registration that drops supported commands-only plugin CLIs.

## What This Changes

The PR adds subcommand help fast paths plus a `helpOnly` metadata-only plugin CLI registration mode, with focused tests for built-in and plugin-owned help paths.

## Best Possible Solution

Rework the help performance path on current main using invocation-aware argv semantics and descriptor/manifest-owned plugin CLI lookup, while preserving the full non-activating fallback for commands-only plugin CLIs.

## Reproduction Assessment

Yes for the review findings by source inspection. The PR uses a broad help/version scan where current main has an invocation-aware parser, and it forces fallback plugin help through metadata-only discovery despite the documented commands-only compatibility path.

## Solution Assessment

No. The fast-path direction is reasonable, but this implementation should be rebuilt around `resolveCliArgvInvocation` semantics and should only skip the full plugin CLI loader when manifest or descriptor ownership proves metadata-only help is sufficient.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Honor -- before taking help fast paths:** `src/cli/run-main.ts:91-92`
  - body: `shouldUseSubcommandHelpFastPath` calls `hasHelpOrVersion`, which also matches flags after `--`. That can intercept `openclaw <subcommand> -- --help` with the minimal help parser instead of leaving `--help` as command data; base this decision on the invocation-aware parser used by current CLI policy.
  - confidence: 0.87
- **[P2] Preserve command-only plugin help fallback:** `src/cli/run-main.ts:345`
  - body: Passing `helpOnly` for every fallback help invocation sends plugin registration through metadata-only discovery. Commands-only `api.registerCli` remains a supported eager compatibility path, so plugin CLIs without descriptors can disappear from `openclaw <plugin> --help`; keep the full non-activating loader unless manifest or descriptor ownership proves metadata-only loading is safe.
  - confidence: 0.84
