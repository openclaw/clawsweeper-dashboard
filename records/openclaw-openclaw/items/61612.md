## Summary

Keep open: the performance goal is useful, and the PR includes real benchmark evidence, but the patch still has correctness blockers in CLI/plugin help registration. Source review confirms the broad `helpOnly` metadata path can drop runtime-only plugin help, and the pairing help path can register before channel plugin data is available.

## What This Changes

The PR adds early CLI help fast-path functions, help-only plugin CLI registration, and focused tests so built-in and plugin subcommand help can avoid full bootstrap.

## Best Possible Solution

Rebuild the performance fast path on current main using manifest/descriptor-owned command lookup and activation-planning loaders, while preserving runtime plugin CLI fallback and pairing channel help semantics.

## Reproduction Assessment

Yes for the review findings by source inspection. The PR routes fallback plugin help registration through metadata-only loading, and pairing help returns before plugin-backed channel data can be populated for `registerPairingCli()`.

## Solution Assessment

No. The broad `helpOnly` switch and hardcoded plugin map are not the safest implementation; the repair should use current-main owner/descriptor resolution and only take metadata-only paths when compatibility is known.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve runtime plugin help fallback:** `src/cli/run-main.ts:345`
  - body: Passing `helpOnly: hasHelpOrVersion(parseArgv)` makes every fallback help invocation use the metadata-only loader. Runtime-only third-party plugin CLIs, or plugins with incomplete CLI metadata, can lose their `openclaw <plugin> --help` command where the full runtime registry path worked before, so keep the runtime fallback except for roots proven safe from metadata.
  - confidence: 0.88
- **[P2] Keep pairing help channel data:** `src/cli/program/register.subclis.ts:235-237`
  - body: This returns before the plugin registration that populated channel plugins, but `registerPairingCli()` snapshots `listPairingChannels()` into option and argument help. Fresh pairing help can render empty or incomplete channel lists for plugin-backed pairing channels, so preserve a channel metadata source before registering the help command.
  - confidence: 0.8
