## Summary

Keep open: the performance goal is useful, but the current patch would break supported command-only plugin CLI help by routing fallback help registration through metadata-only discovery. The branch is also conflicting with current main, which has overlapping CLI/plugin-registration changes that should be incorporated in a rebased repair.

## What This Changes

The PR adds early CLI help paths and a `helpOnly` metadata-loader mode so selected built-in and plugin subcommand help can avoid full plugin bootstrap.

## Best Possible Solution

Rebuild the help performance work on current main using manifest/descriptor-owned command lookup while preserving the full-loader fallback for supported command-only plugin CLIs.

## Reproduction Assessment

Yes for the review finding by source inspection. The PR routes help-time fallback registration through metadata-only loading even though current docs and tests preserve command-only plugin CLI registrations through the full-loader compatibility path.

## Solution Assessment

No. The performance approach is reasonable, but metadata-only help paths should be used only when descriptor or manifest ownership proves that supported command-only plugin CLIs will not be dropped.
