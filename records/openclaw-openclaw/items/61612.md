## Summary

Keep open. The performance goal is useful and not fully replaced on current main, but this stale branch would reintroduce config/plugin metadata loading on help paths that current main now keeps configless, so it needs a current-main repair rather than a direct merge.

## What This Changes

The PR adds CLI subcommand help fast paths and metadata-only plugin help registration for built-in and plugin-owned commands.

## Best Possible Solution

Rework the fast path on current main as configless descriptor-backed help for known command descriptors, while preserving the current plugin-registration skip policy for generic help paths and using activation-aware plugin CLI narrowing for non-help execution.

## Reproduction Assessment

Yes for the review findings: source inspection shows the PR calls validated config and plugin metadata registration on help paths, while current main explicitly skips plugin registration for help. I did not run live CLI commands because this was a read-only review.

## Solution Assessment

No. The performance direction is sound, but the branch should be rewritten on current main to keep help configless and bounded instead of adding a config-reading plugin help path and global `helpOnly` fallback.
