## What This Changes

The branch aggregates active session transcripts with same-stem reset/deleted archives for per-session summary, cache-backed summary, time-series, and logs, with focused regression tests.

## Best Possible Solution

Land this branch or an equivalent canonical same-stem lineage resolver after maintainer review accepts the accounting change and bounded request-path scan cost, while keeping broader chat-history, trajectory, topic, and checkpoint lineage work separate.

## Reproduction Assessment

Yes, source-reproducible: create an active `<id>.jsonl` beside same-stem `.jsonl.reset.*` or `.jsonl.deleted.*` archives and request per-session usage detail. Current main and v2026.6.6 resolve and scan only the active file for summary, time-series, and logs.

## Solution Assessment

Yes for the narrow same-stem reset/deleted cost-detail bug. Reusing the canonical usage-counted filename parser inside the session-cost loader/cache paths is better than broad prefix matching; broader artifact families should stay separate unless maintainers expand scope.
