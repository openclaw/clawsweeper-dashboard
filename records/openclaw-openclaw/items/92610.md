## Best Possible Solution

Land this branch or an equivalent canonical same-stem lineage resolver after maintainer review accepts the accounting and request-path scan cost, while keeping broader chat-history, trajectory, topic, and checkpoint lineage work separate.

## Reproduction Assessment

## Solution Assessment

Yes for the narrow same-stem reset/deleted bug. Reusing the canonical usage-counted filename parser in the session-cost loader/cache paths is more maintainable than another caller-specific prefix glob; broader artifact families should remain separate unless maintainers expand scope.
