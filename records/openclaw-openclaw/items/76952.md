## Summary

Keep open. Current main partially documents browser realtime Talk, OpenAI realtime setup, Voice Call, mobile nodes, and the Clawdia phone-bridge example, but it still lacks the requested unified end-user Talk guide and leaves entitlement plus mobile/phone bridge scope as maintainer product decisions.

## What This Changes

The issue asks for clearer user-facing guidance on Realtime Talk billing, realtime voices, the voice-agent mental model, and phone/mobile bridge options.

## Best Possible Solution

Publish a scoped Talk guide or FAQ that states the current OpenAI Realtime billing route, lists realtime voice/provider limits, explains direct realtime versus consult-agent behavior, and links browser, mobile, Voice Call, and external bridge options while splitting new entitlement or bridge behavior into separate maintainer-approved items.
