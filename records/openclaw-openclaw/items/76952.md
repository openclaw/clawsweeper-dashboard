## Summary

Keep open. Current main documents browser realtime Talk, OpenAI realtime config, mobile Talk basics, and the Voice Call plugin, but it still lacks the requested unified user-facing explanation for billing/auth, realtime voices and styles, the voice-agent consult model, and mobile/phone bridge choices.

## What This Changes

Request clearer Realtime Talk docs for OpenAI billing/auth, realtime voices/styles, the voice-agent consult model, and mobile/phone bridge options.

## Best Possible Solution

A dedicated Realtime Talk guide should tie together OpenAI Platform billing, realtime provider voices, voice-agent consult behavior, mobile Talk limitations, and Voice Call as the phone bridge, with entitlement or new bridge feature changes split into separate tracked decisions.
