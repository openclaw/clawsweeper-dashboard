## What This Changes

Adds gateway session-store alias helpers and wires session, compaction, node-event, resolver, and plugin host-hook writes/tests to keep raw external keys synchronized with canonical agent keys.

## Best Possible Solution

Keep the shared alias-helper direction, but choose the freshest entry across each canonical/raw alias group before migration, mutation, filtering, search, count, ordering, and active checks, with raw-newer regressions.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: store a canonical key and hidden raw alias with the same sessionId but a newer raw entry, then mutate the visible canonical key or list/search/filter sessions; the PR excludes that raw alias from freshness selection or drops it before filters.

## Solution Assessment

No. The shared alias-helper direction is maintainable, but the implementation still treats the canonical row as authoritative before considering a newer preserved raw alias; the safer repair is alias-group freshness selection before writes and list filtering.
