## Label Justifications

## Summary

This PR is a plausible fix for the reported raw external session-key split and the contributor supplied useful terminal proof, but the current patch is not merge-ready because alias preservation is added inside a shared migration helper without updating callers that delete or replace the canonical entry.

## What This Changes

The PR changes gateway session-key migration to copy raw structured external keys onto canonical agent-scoped keys while preserving the raw alias, with a focused regression test.

## Best Possible Solution

Preserve raw external aliases for canonicalization while making alias lifecycle explicit: every mutation that replaces or deletes the canonical primary key must update or remove the preserved raw aliases, with regression coverage for migrate, reset, delete, and a normal write path.

## Reproduction Assessment

Yes, from source inspection. With the PR diff applied, the shared migration helper preserves the raw alias before callers delete or replace only `primaryKey`, leaving a stale raw entry behind in those mutation paths.

## Solution Assessment

No. Preserving raw aliases during canonicalization is the right direction, but doing it inside the shared helper without a synchronization contract for delete/reset/write callers is not the safest implementation.
