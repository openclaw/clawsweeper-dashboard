## Summary

The latest PR head fixes the earlier raw/canonical alias gaps for the covered gateway mutation and compaction paths, but it still leaves plugin host-hook session-state writes outside the alias-sync contract. That can make a hidden raw alias stale again after callers use the canonical key returned by `sessions.list`, so this PR needs one more focused repair before merge.

## What This Changes

The PR adds gateway session-store alias preservation and synchronization across migration, patch/delete/reset/writeback/resolve, compaction/checkpoint writes, and list filtering, with targeted regression tests.

## Best Possible Solution

Keep the alias-aware gateway direction, but route plugin host-hook session-state writes through the same preserved-alias synchronization contract before maintainer landing.

## Reproduction Assessment

Yes, at source level. Current main prunes raw external aliases during canonicalization, and the PR-head gap is traceable because `sessions.list` hides raw aliases while plugin host-hook writers still mutate only `loaded.storeKey`.

## Solution Assessment

No, not quite yet. The central preserved-alias helper approach is the right shape, but the best fix must include plugin host-hook session-state writers so the raw/canonical alias invariant is not broken by another session-key mutation path.
