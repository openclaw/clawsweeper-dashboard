## Summary

Keep open: the branch is a focused session-state fix with sufficient backend proof, but two P1 blockers remain around fresher hidden raw aliases being ignored by canonical-key mutation and list/filter paths.

## What This Changes

The PR adds shared gateway session-store alias helpers, routes gateway/session/plugin mutations through them, and expands regressions for raw external session aliases.

## Best Possible Solution

Keep the shared alias-helper direction, but choose the freshest entry across each canonical/raw alias group before migration, mutation, filtering, search, count, ordering, and active checks, with raw-newer regressions.

## Reproduction Assessment

Yes. Source inspection shows a high-confidence reproduction with a store containing a canonical key plus a fresher hidden raw alias: migration selects only `target.storeKeys`, and list filtering drops the raw alias before freshness-sensitive filters and counts.

## Solution Assessment

No. The alias-helper approach is maintainable, but the current PR must make canonical/raw alias groups choose one freshest entry before sync and list filtering to avoid stale state loss.
