## Summary

Keep open. The PR targets a valid gateway session-continuity bug and has useful proof, but it is not merge-ready because the new preserved raw aliases can still appear as duplicate visible session rows and counts.

## What This Changes

The PR preserves raw external gateway session aliases during canonicalization, synchronizes or deletes them across session mutations, and adds gateway regression tests.

## Best Possible Solution

Keep raw aliases as internal lookup aliases, sync and delete them across mutations, and collapse them from list/search/count output before landing.

## Reproduction Assessment

Yes, at source level. Current main canonicalizes a raw structured key into an agent-scoped key and prunes the raw candidate, while the PR preserves both keys and the remaining list path still enumerates every store entry.

## Solution Assessment

No, not yet. The alias-sync boundary is the right direction, but aliases need to stay internal to lookup/mutation paths so one logical session is not exposed twice.
