## Summary

Keep open. The PR fixes the main raw-alias lifecycle for migration, patch/delete/reset, node-event touches, and list visibility, with strong terminal proof, but checkpoint restore and compaction/checkpoint writers can still update only the canonical key and leave preserved raw aliases stale.

## What This Changes

The PR adds preserved raw external session alias tracking to gateway session-store canonicalization, sync/delete helpers for aliases, and regression coverage for patch/delete/reset/list/resolve/node-event paths.

## Best Possible Solution

Keep the alias-preservation design, but thread preservedAliasKeys through every session-store mutation that can replace or mutate the canonical entry before merge.

## Reproduction Assessment

Yes, at source level. A store with both a raw external key and its canonical agent key will still diverge when checkpoint restore or compaction checkpoint metadata writes update only the canonical entry.

## Solution Assessment

No, not yet. The alias-aware helpers are the right direction, but the implementation needs to cover the remaining compaction/checkpoint mutation paths before this is the narrow maintainable fix.
