## Summary

Keep this PR open: the shared alias-helper direction and proof are useful, but the current head still has two P1 session-state freshness bugs that can overwrite or hide newer raw-alias state.

## What This Changes

The PR adds gateway session-store alias helpers and routes gateway, compaction, resolver, node-event, and plugin host-hook write paths through raw/canonical alias synchronization.

## Best Possible Solution

Treat each canonical key and its preserved raw aliases as one logical freshness group before every mutation and before list/search/filter/count/order/limit behavior, with raw-newer regressions.

## Reproduction Assessment

Yes. Seed a store with an older canonical entry and a newer hidden raw alias for the same external key, then mutate by the canonical visible key or list/search/filter the store; source inspection shows the PR head still does not choose the raw alias as the freshest logical row first.

## Solution Assessment

No. The shared alias-helper direction is maintainable, but the fix is incomplete until canonical and preserved raw keys are collapsed to one freshness group before writes and before list selection.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Include raw aliases in freshness selection:** `src/gateway/session-utils.ts:928-930`
  - body: `preservedAliasKeys` is computed here, but the freshest match is selected only from `target.storeKeys`. If `sessions.list` returns the canonical key while the hidden raw alias is newer, this keeps the stale canonical entry and the later alias sync copies it over the raw alias before the write, losing newer session or plugin state. Include preserved raw aliases in the freshness set before syncing.
  - confidence: 0.93
- **[P1] Collapse alias groups before filtering session lists:** `src/gateway/session-utils.ts:2153-2154`
  - body: This filters raw aliases out before choosing the logical row. When the raw alias is fresher than the canonical row, list/search/active filters, totals, ordering, and limits are computed from stale canonical data, so a current external conversation can appear old, fail search, or be ordered incorrectly. Collapse each alias group to the freshest canonical-visible row first.
  - confidence: 0.91
