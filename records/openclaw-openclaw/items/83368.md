## Summary

This PR targets a real gateway session-continuity bug, but the current patch only preserves the raw alias during migration and leaves later mutation/delete paths able to stale or revive that alias.

## What This Changes

The PR changes `migrateAndPruneGatewaySessionStoreKey` to preserve raw structured external keys as aliases of canonical agent-scoped keys and adds a focused regression test.

## Best Possible Solution

Keep the raw external alias and canonical entry synchronized in the same store transaction for migrate, patch, agent writeback, reset, and delete, with regressions proving stale aliases cannot revive old session state.

## Reproduction Assessment

Yes, at source level. Current main prunes the raw structured key during canonicalization, while PR-head inspection shows the alias is preserved only at migration time and later writers still operate only on `primaryKey`.

## Solution Assessment

No, not as currently patched. Preserving the raw alias is the right direction, but the maintainable fix has to make that alias follow canonical writes and deletes instead of becoming a second stale source of truth.
