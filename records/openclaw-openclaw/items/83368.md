## Summary

This PR should stay open: the alias-preserving direction is right and the supplied proof is solid, but the new raw/canonical alias lifecycle still misses checkpoint restore/persistence writers that can leave the raw alias stale.

## What This Changes

The PR adds preserved raw external session alias tracking to gateway session-store migration, syncs aliases through several write/delete paths, hides alias duplicates from session lists, and adds gateway regression tests.

## Best Possible Solution

Make every gateway session-store replacement/delete path that can touch canonicalized raw external keys use the alias-aware helpers, including compaction checkpoint restore and checkpoint metadata persistence, with focused regression coverage.

## Reproduction Assessment

Yes at source level. Current main prunes raw structured aliases during canonicalization and later mutation paths write or delete only the canonical primary key; I did not run a live external-integration round trip in this read-only review.

## Solution Assessment

No, not yet. The central alias-helper approach is maintainable, but it needs to cover checkpoint restore and checkpoint metadata persistence before it fully solves the alias lifecycle.
