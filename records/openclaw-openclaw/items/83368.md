## Best Possible Solution

Keep the shared alias contract, but choose the freshest entry across canonical target keys and preserved raw aliases before any alias sync, replacement write, prune, reset, patch, or compaction follow-up.

## Solution Assessment

No, not yet: the shared alias helper is the right maintainable shape, but the freshness decision must include preserved aliases before any sync or write path relies on the primary entry.
