## Summary

Keep open: the PR addresses the raw-alias promotion path, but the helper it changes is also used before reset and delete mutations. Preserving the raw alias before callers mutate only the canonical primary key would leave raw external aliases undeleted or stale, so the patch needs a focused lifecycle fix before merge.

## What This Changes

The PR updates gateway session-key migration to copy raw colon-delimited external keys to canonical agent keys while preserving the raw alias, with a regression test in `src/gateway/session-utils.test.ts`.

## Best Possible Solution

Preserve raw aliases for canonicalization while ensuring every mutation that replaces or deletes the canonical primary key also updates or removes preserved raw aliases, with regression coverage for migrate, reset, and delete.

## Reproduction Assessment

Yes from source inspection. After the PR change, the shared helper can preserve a raw alias before `sessions.delete` deletes only the canonical key and before reset overwrites only the canonical key; I did not run tests because this review is read-only.

## Solution Assessment

No. Preserving raw aliases during canonicalization is the right direction, but putting it in the shared helper without updating destructive callers is not the narrowest maintainable fix.
