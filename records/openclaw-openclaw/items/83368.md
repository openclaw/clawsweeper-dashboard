## Summary

Keep open: the root fix direction and current-head terminal proof are useful, but the patch still risks exposing one logical session twice because it persists raw and canonical aliases while the session list path enumerates every store key without alias collapse.

## What This Changes

The PR adds preserved alias tracking to gateway session-store migration helpers and uses shared write/delete/sync helpers across agent, sessions patch/delete/reset, node-event touch, and resolve paths with regression tests.

## Best Possible Solution

Keep raw aliases as internal lookup aliases, sync/delete them across mutations, and collapse them out of user-facing session list/search/count surfaces before landing the linked issue fix.

## Reproduction Assessment

Yes, from source inspection. Current main canonicalizes a raw structured key into an agent-scoped key and prunes target store keys, matching the linked external-session split path without needing a new feature contract.

## Solution Assessment

No, not yet. The alias-sync approach is the right fix boundary, but the PR also needs to keep those aliases internal to lookup/mutation paths so one logical session does not appear twice in listing surfaces.
