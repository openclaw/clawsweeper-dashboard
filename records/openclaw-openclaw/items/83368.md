Needs changes before merge. The shared alias-helper direction is useful and the PR supplies terminal proof, but source review still shows two P1 session-state gaps where a fresher hidden raw alias can be overwritten or hidden by stale canonical data.

## What This Changes

The PR adds gateway session-store alias helpers and routes session, compaction, resolver, node-event, and plugin host-hook writes through raw/canonical alias synchronization.

## Best Possible Solution

Keep the shared alias-helper approach, but select the freshest entry across each canonical/raw alias group before mutation and list filtering, then cover raw-newer canonical-visible flows with focused regressions.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: store a canonical key and hidden raw alias with the same `sessionId` but newer raw metadata, then mutate or list by the visible canonical key.

## Solution Assessment

No, not yet. The shared helper belongs in the right area, but it must collapse alias groups by freshness before writes and list filtering to avoid stale canonical state winning.
