Full review comments:

- **[P1] Include raw aliases in freshness selection:** `src/gateway/session-utils.ts:928-930`
  - body: When a caller passes the visible canonical key, `preservedAliasKeys` can contain an existing hidden raw alias, but the freshness scan only checks `target.storeKeys`. If the raw alias has newer metadata or plugin state, the following alias sync copies the older canonical entry over it, so canonical-key patch/reset/compact flows can still lose state. Include preserved aliases in the freshest-entry selection before writing or syncing.
  - confidence: 0.93
- **[P1] Collapse alias groups before filtering session lists:** `src/gateway/session-utils.ts:2153-2154`
  - body: This filters out the raw alias before search, activeMinutes, count, ordering, and limit, but it keeps the canonical row even when the hidden raw alias is newer or has the matching subject/updatedAt. A raw-newer external session can therefore disappear from search/active results or sort as stale. Collapse each raw/canonical alias group to its freshest entry before applying list filters.
  - confidence: 0.9
