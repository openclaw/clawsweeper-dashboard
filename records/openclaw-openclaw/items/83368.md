## Summary

Keep open. This PR addresses a valid session-state bug and has strong backend proof, but source inspection of the current head still finds blocking alias freshness gaps that can lose or hide newer raw-alias state.

## What This Changes

The PR adds gateway raw external session-alias helpers and routes session mutation, compaction, resolver, node-event, and plugin host-hook writes through raw/canonical alias synchronization with regression tests.

## Best Possible Solution

Keep the shared alias-helper direction, but collapse canonical and preserved raw aliases into one logical freshness group before every mutation and before list/search/count/order/limit behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: seed raw and canonical entries for the same logical session with the hidden raw alias newer, then mutate by the visible canonical key or list with search/active filters.

## Solution Assessment

No. The shared helper direction is maintainable, but the implementation is not the best fix until raw/canonical aliases are treated as one freshness group before mutation and list selection.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Include raw aliases in freshness selection:** `src/gateway/session-utils.ts:928-930`
  - body: Here `preservedAliasKeys` is computed, but the freshest entry is still chosen only from `target.storeKeys`. In the list-to-patch/list-to-delete flow the caller passes the visible canonical key, so a newer hidden raw alias is not considered and alias sync can copy the older canonical entry over it. Include preserved raw aliases in the freshness set before syncing.
  - confidence: 0.9
- **[P1] Collapse alias groups before filtering session lists:** `src/gateway/session-utils.ts:2153-2154`
  - body: This drops the raw alias before search, active filtering, totalCount, sort, and limit choose a row. If the raw alias has a later `updatedAt` or fresher plugin/session metadata with the same `sessionId`, the visible canonical row can be stale or excluded. Collapse each alias group to the freshest entry before filtering/listing.
  - confidence: 0.88
