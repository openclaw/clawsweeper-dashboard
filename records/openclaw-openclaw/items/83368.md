## Summary

Keep open: the latest head still has source-proven session-state blockers. Canonical-key mutation can overwrite a fresher hidden raw alias, and list/search/active/order/limit can hide the fresher raw alias while surfacing the older canonical row.

## What This Changes

Adds gateway session-store alias helpers and routes raw/canonical alias synchronization through session mutation, compaction/checkpoint, resolver, node-event, and plugin host-hook paths.

## Best Possible Solution

Keep the shared alias-helper direction, but collapse canonical and preserved raw aliases into one logical freshness group before every mutation and before list/search/count/order/limit behavior.

## Reproduction Assessment

Yes. Source inspection on the PR head gives a high-confidence path: create canonical and raw alias entries for the same logical session with the raw alias newer, then mutate or list by the visible canonical key.

## Solution Assessment

No, not yet. The shared alias helper is the right direction, but the implementation must choose the freshest alias-group entry before syncing or hiding raw aliases.
