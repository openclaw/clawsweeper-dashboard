## Summary

Keep open: the shared alias-helper direction and terminal proof are strong, but the current patch can still lose or hide fresher hidden raw-alias state when follow-up operations use the canonical visible key.

## What This Changes

The branch adds gateway raw/canonical session-store alias helpers and routes session mutation, compaction, resolver, node-event, and plugin host-hook writes through alias synchronization with regression tests.

## Best Possible Solution

Keep the shared alias-helper direction, but treat each canonical key and its preserved raw aliases as one freshness group before every mutation and before list/search/count/order/limit behavior.

## Reproduction Assessment

Yes. Seed a store with an older canonical entry and a newer hidden raw alias for the same external key, then mutate by the canonical visible key or list/search/count the store; source inspection shows the raw alias is not selected as the freshest row first.

## Solution Assessment

No. The helper-based approach is maintainable, but it is incomplete until alias groups select the freshest row before mutation and visible listing behavior.
