## Summary

Keep open. The PR has strong terminal proof and the shared alias-helper direction is good, but current head still has two blocking session-state freshness bugs: canonical-key mutation can overwrite a newer hidden raw alias, and session listing can hide fresher raw-alias data behind an older canonical row.

## What This Changes

The PR adds shared gateway raw-alias helpers and routes session mutation, compaction, resolver, node-event, and plugin host-hook writes through raw/canonical alias synchronization.

## Best Possible Solution

Keep the shared alias-helper direction, but treat each canonical key and its preserved raw aliases as one freshness group before every mutation and before list/search/count/order/limit behavior.

## Reproduction Assessment

Yes. Seed a store with an older canonical entry and a newer hidden raw alias for the same external key, then mutate by the canonical visible key or list/search/count the store; source inspection shows the raw alias is not selected as the freshest row first.

## Solution Assessment

No. The shared alias contract is the right direction, but the current implementation still syncs and lists from the canonical row before collapsing the alias group to its freshest entry.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Include raw aliases in freshness selection:** `src/gateway/session-utils.ts:928-930`
  - body: When a caller uses the canonical key returned by `sessions.list`, `migrateAndPruneGatewaySessionStoreKey` discovers preserved raw aliases but selects the freshest entry only from `target.storeKeys`. If the hidden raw alias is newer, the later alias sync copies the older canonical entry over it before patch/reset/delete/compaction/plugin follow-up work can see the fresh state. Include the preserved alias keys in the freshness selection before syncing or writing.
  - confidence: 0.93
- **[P1] Collapse alias groups before filtering session lists:** `src/gateway/session-utils.ts:2153-2154`
  - body: `sessions.list` filters preserved raw aliases out before search, active filters, totals, ordering, and limit are computed. If the raw alias has the fresher `updatedAt`, label, subject, or plugin-visible state, users see or search the older canonical row instead. Build a logical raw/canonical alias group and expose the freshest canonical-visible row before applying list filters.
  - confidence: 0.9
