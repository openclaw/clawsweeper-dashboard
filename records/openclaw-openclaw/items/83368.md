## Summary

This PR is a useful fix for the linked gateway session-state bug, but it is not merge-ready because canonical-key follow-up paths can still overwrite a newer hidden raw alias and session listing can still hide the freshest logical row before filtering, ordering, and counting.

## What This Changes

The PR adds gateway session-store alias helpers and routes gateway, compaction, resolver, node-event, and plugin host-hook writes through raw/canonical alias synchronization.

## Best Possible Solution

Treat each canonical key and its preserved raw aliases as one logical freshness group before every mutation and before list/search/filter/count/order/limit behavior, with raw-newer regressions.

## Reproduction Assessment

Yes. Seed an older canonical entry plus a newer hidden raw alias, then mutate or list by the canonical visible key; source inspection shows the PR head still ignores or drops the raw alias before freshness selection.

## Solution Assessment

No. The shared alias-helper direction is maintainable, but the current implementation must first make alias-group freshness the source of truth for mutation and listing paths.
