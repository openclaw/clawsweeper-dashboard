## Summary

Keep open: the shared alias direction is right, but PR head still lets stale canonical entries win over newer hidden raw aliases in mutation and list paths, so it remains session-state unsafe to merge.

## What This Changes

The branch adds shared gateway session-store alias helpers and routes gateway, session, compaction, resolver, node-event, and plugin host-hook writes through raw/canonical alias synchronization with regression coverage.

## Best Possible Solution

Treat each canonical key and its preserved raw aliases as one logical freshness group before every mutation and before session list search, filter, count, order, and limit behavior, with raw-newer regression coverage.

## Reproduction Assessment

Yes. Seed an older canonical row plus a newer hidden raw alias for the same external key, then mutate or list by the canonical visible key; source inspection shows the PR head still ignores or drops the raw alias before freshness selection.

## Solution Assessment

No. The shared alias helper direction is maintainable, but the implementation must make alias-group freshness the unit of selection before mutation and list filtering to avoid stale canonical rows winning.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Include raw aliases in freshness selection:** `src/gateway/session-utils.ts:928-930`
  - body: When the caller uses the canonical visible key returned by `sessions.list`, `target.storeKeys` contains the canonical key but not the hidden raw alias discovered in `preservedAliasKeys`. If that raw alias is newer, this path syncs or patches from the stale canonical entry and can overwrite newer session/plugin state; include the preserved aliases in freshest-entry selection and add raw-newer canonical-key regressions.
  - confidence: 0.92
- **[P1] Collapse alias groups before filtering session lists:** `src/gateway/session-utils.ts:2153-2154`
  - body: This filter drops the raw row as soon as a same-session canonical row exists, before search, active filters, totals, ordering, and limits. A newer raw alias with the matching subject, label, or `updatedAt` can therefore be hidden behind stale canonical fields; collapse each alias group to the freshest canonical-visible row before applying list semantics.
  - confidence: 0.88
