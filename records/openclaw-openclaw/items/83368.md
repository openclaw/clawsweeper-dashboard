## Summary

Keep open. The PR addresses a real raw-vs-canonical gateway session split at migration time, and the terminal proof is useful, but the preserved raw alias is not kept in sync when later callers patch, reset, delete, or write back the canonical entry.

## What This Changes

The PR adds gateway session-store migration logic and regression coverage to preserve structured raw external session keys as aliases of canonical agent-scoped keys.

## Best Possible Solution

Keep raw external aliases tied to the canonical key through migrate, patch, agent writeback, reset, and delete, with regression tests proving stale aliases cannot revive old session state.

## Reproduction Assessment

Yes, at source level. Current main canonicalizes a structured raw key to an agent-scoped key, includes the raw key in the candidate set, and prunes or mutates only the canonical key depending on the gateway path.

## Solution Assessment

No, not as currently patched. Preserving the raw alias is the right direction, but the alias needs a lifecycle contract so primary-entry replacement and deletion cannot leave stale session state behind.
