## Summary

This PR addresses a real gateway session-state bug and the direct helper proof is sufficient, but it is not merge-ready because the preserved raw alias is only refreshed during migration. Later primary-key mutations and deletes can leave the raw alias stale or surviving, which keeps the session-splitting/deleted-session problem alive.

## What This Changes

The PR changes `migrateAndPruneGatewaySessionStoreKey` to keep structured raw external session keys as aliases of canonical agent keys and adds a focused regression test.

## Best Possible Solution

Keep the raw external alias and canonical entry synchronized for every gateway session-store mutation and deletion, with regression coverage proving stale aliases cannot revive old session state.

## Reproduction Assessment

Yes, at source level. Current main prunes raw structured keys during canonicalization, and PR-head proof shows the helper preserves the alias immediately while source inspection shows later writers still mutate only `primaryKey`.

## Solution Assessment

No, not as currently patched. Preserving the raw alias is the right direction, but the implementation needs to make that alias part of the same mutation/delete contract as the canonical entry.
