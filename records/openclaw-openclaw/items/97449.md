## Summary

## Best Possible Solution

Normalize Runtime prompt identity only for canonical isolated-cron run keys, omit the duplicated run `sessionId`, and preserve non-cron session keys byte-for-byte.

- **[P2] Drop the duplicated run sessionId from Runtime:** `src/agents/system-prompt.ts:1404`
  - body: `runEmbeddedAgent` passes both the run-scoped `sessionKey` and the per-run `sessionId`, and `buildRuntimeLine` renders `sessionId` immediately after `session`.   - confidence: 0.92
- **[P2] Gate run stripping to cron-run keys:** `src/agents/system-prompt.ts:1404`
  - body: The replacement removes any `:run:` segment from any session key, but existing session-key helpers only classify `agent:<id>:cron:<job>:run:<id>` as a cron run and explicitly leave cron-like channel keys alone. A normal channel or externally owned id containing `:run:` would now be misrepresented in the model-visible Runtime line. Please parse the canonical cron-run shape before stripping and leave other keys unchanged.
  - confidence: 0.84
