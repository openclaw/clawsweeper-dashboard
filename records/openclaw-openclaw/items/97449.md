## What This Changes

The PR strips the terminal `:run:` suffix from cron-run Runtime prompt session keys, omits `sessionId` for cron-run prompts, and adds focused tests plus a standalone proof script.

## Best Possible Solution

Parse the terminal cron run id, render the stable cron base key, omit `sessionId` only when it equals that parsed run id, and add redacted after-fix proof from the isolated-cron runtime path.

## Solution Assessment

No. Prompt-facing normalization is the right narrow direction, but the implementation should omit `sessionId` only when it equals the parsed run id; this PR also needs real isolated-cron runtime proof before merge.
