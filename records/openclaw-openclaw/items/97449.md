## Summary

Keep open: this is a useful candidate fix, but it only stabilizes the rendered `session=` field while real isolated cron runs still render a volatile `sessionId=` and the broad regex can rewrite non-cron session keys.

## What This Changes

The PR changes `buildRuntimeLine` to strip any `:run:` segment from the rendered Runtime `session=` field and adds two runtime-line tests plus a standalone proof script.

## Best Possible Solution

Normalize prompt-facing identity only for canonical isolated-cron run keys, omit the duplicated run `sessionId`, and preserve all non-cron session keys unchanged.

## Reproduction Assessment

Yes at source level: current main builds a `:run:` isolated-cron key, passes both `sessionKey` and `sessionId` into embedded execution, and `buildRuntimeLine` renders both. I did not run a live metered-provider token trace.

## Solution Assessment

No. The patch is a plausible partial fix, but the best fix must also handle the duplicated run `sessionId` and must scope stripping to canonical cron-run keys instead of every `:run:` substring.

Please parse the canonical cron-run shape before stripping and leave other keys unchanged.
