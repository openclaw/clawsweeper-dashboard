## Summary

Keep open. Current main still lacks the Slack inbound files.info hydration, media-read fallback token path, and cross-turn staged filename nonce this PR is trying to add, but the PR head is dirty against main and would need to preserve newer Slack media timeout, abort, and thread-context behavior before it can land.

## What This Changes

The PR hardens Slack inbound file ingestion with files.info hydration, media-read token fallback plumbing, thread-root file and attachment hydration, nonce-based sandbox staging filenames, and focused tests.

## Best Possible Solution

Rebase the useful Slack media hydration work onto current main's split Slack media/thread modules, keep fallback downloads routed through the existing bounded fetchSlackMedia path, preserve current thread-context filtering, keep the sandbox filename nonce change with focused tests, and add the required changelog entry.

## Reproduction Assessment

Yes. The PR body gives a live Slack thread image repro, and current main source shows id-only Slack files without private URLs still return null in resolveSlackMedia; repeated inbound basenames also still map to the original basename in the sandbox staging helper.

## Solution Assessment

No as currently proposed. The product direction is right, but the branch needs a rebase and adaptation to current main's Slack media wrapper and thread-context behavior before it is the narrowest maintainable fix.
