## Summary

Keep this open: current main still lets native Moonshot miss the skip-PI model fast path and fall back to broad models.json/provider discovery before dispatch, matching the reporter's repeated 30s model-resolution traces and the reported explicit-config workaround.

## What This Changes

Native Moonshot model refs should resolve quickly for Discord and TUI runs without requiring users to add explicit models.providers.moonshot metadata.

## Best Possible Solution

Resolve bundled/native static provider model refs from the manifest or a provider-owned hook during the embedded-runner skip-PI path, then keep broad models.json generation only as a later fallback.

## Reproduction Assessment

No full live reproduction was run in this read-only pass. The reporter supplied repeated shipped-release traces, and current source still shows the same skip-PI miss into broad models.json discovery path.

## Solution Assessment

Yes in direction: the maintainable fix is to use the existing bundled static catalog or a cheap Moonshot-owned resolver before broad discovery, not to require users to add explicit provider config.
