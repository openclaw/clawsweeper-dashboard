## Summary

Keep this open: current main and the latest release still gate bundled static catalog fallback to plugin-owned harness transport, so native Moonshot can still miss the lean skip-discovery path and fall into broad model discovery. The linked fix attempts are closed unmerged, and the remaining repair needs maintainer choice on a narrow provider-safe fast path.

## What This Changes

The issue reports slow native Moonshot Discord/TUI runs spending 10-30 seconds in model resolution before dispatch when `moonshot/kimi-k2.6` is configured without explicit inline provider model metadata.

## Best Possible Solution

Land a narrow fix that lets native Moonshot resolve bundled static catalog rows before broad `models.json` generation, while preserving provider dynamic/runtime metadata precedence for other providers unless they explicitly opt into a static fast path, and include redacted before/after dispatch timing proof.

## Reproduction Assessment

No high-confidence live reproduction was run in this read-only review. The reporter provided concrete shipped-release timing logs, and current source still shows native Moonshot can miss the skip-discovery fast path and fall into broad model discovery.

## Solution Assessment

No for the previously linked broad PR: it changed provider precedence globally and was closed unmerged. A narrower Moonshot-owned or explicit capability-gated fast path is the safer maintainable solution.
