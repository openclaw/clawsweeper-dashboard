## Summary

Keep open: current main still has a source-level path where native Moonshot misses the skip-PI fast path and falls into broad models.json generation before dispatch, matching the reporter's 10-30s model-resolution traces and their explicit-config workaround.

## What This Changes

The issue requests fixing native Moonshot/Kimi embedded-run model resolution so Discord and TUI dispatch do not wait 10-30s before the model starts when Moonshot has no explicit models.providers config.

## Best Possible Solution

Keep Moonshot as a bundled provider, but make embedded model resolution use the existing plugin/static catalog metadata for moonshot/kimi-k2.6 before broad models.json generation, preserving broad discovery for setup and catalog flows.

## Reproduction Assessment

Yes at source level, but not as a live Moonshot/Discord run. Current main can miss skip-PI model resolution for bundled Moonshot without explicit provider metadata and then call ensureOpenClawModelsJson before dispatch, which matches the reporter's traces and workaround.

## Solution Assessment

Yes in direction. The narrow maintainable fix is to reuse existing provider-owned static catalog or a Moonshot resolveDynamicModel hook on the embedded-run fast path, not to require users to add config or introduce a new latency knob.
