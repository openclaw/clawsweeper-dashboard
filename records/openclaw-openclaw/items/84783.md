## Summary

Keep this open: current `main` still has a source-level path where native Moonshot can fall from the intended skip-PI model resolution into broad `models.json` provider discovery before Discord dispatch, matching the reporter’s 33.7s model-resolution trace.

## What This Changes

Native Moonshot Discord messages are reported to spend 10-30 seconds in embedded-run model resolution before dispatch when no explicit `models.providers.moonshot` config exists.

## Best Possible Solution

Resolve bundled/native provider model refs from manifest or targeted owner discovery fast enough for Discord dispatch, with broad `models.json` preparation kept as a later fallback.

## Reproduction Assessment

No, not as a full live high-confidence repro in this read-only pass. Source inspection and the reporter’s shipped-release trace strongly match the fallback from skip-PI model resolution into broad provider discovery.

## Solution Assessment

Yes in direction: the best fix is to use the existing fast provider-owned/static-catalog path or scoped discovery before broad `models.json` generation, without requiring users to add a new Moonshot config option.
