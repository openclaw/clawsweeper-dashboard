## Summary

Keep this open: current main and the latest release still route native Moonshot embedded runs through the broad model-discovery fallback when the skip-discovery pass misses, and an open draft PR explicitly owns the fix path.

## What This Changes

The issue reports 10-30s model-resolution delay before Discord and TUI dispatch for native `moonshot/kimi-k2.6` without explicit `models.providers.moonshot` metadata.

## Best Possible Solution

Review and land the linked fix PR or a narrow replacement that resolves bundled Moonshot model refs before broad `models.json` discovery, with same-environment timing proof and preserved runtime-preferred provider metadata.

## Reproduction Assessment

No high-confidence live reproduction was run here. The reporter provided concrete shipped-release timing logs, and current source still shows the native Moonshot path can miss the skip-discovery fast path and fall into broad model discovery.

## Solution Assessment

Yes in direction: the maintainable fix is to use the existing bundled static catalog or another targeted provider-owned path instead of requiring users to add explicit Moonshot config. The open PR is the current implementation candidate but still needs proof and maintainer review before this issue closes.
