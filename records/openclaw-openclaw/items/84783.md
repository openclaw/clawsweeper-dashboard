## Summary

Keep this open: current main still has a source-level path where native Moonshot can miss the skip-discovery fast path and fall back into broad models.json/provider discovery before dispatch, matching the reporter's 30s model-resolution traces.

## What This Changes

The issue reports slow embedded-run model resolution for native Moonshot models before Discord and TUI dispatch when no explicit Moonshot provider metadata is configured.

## Best Possible Solution

Resolve bundled Moonshot model refs from the existing provider-owned static catalog or an equivalent targeted Moonshot dynamic hook during the embedded runner's skip-discovery first pass, leaving broad models.json generation as a later fallback.

## Reproduction Assessment

No live high-confidence reproduction was run in this read-only review. Source inspection plus the reporter's shipped-release traces provide a high-confidence source-level reproduction path: native Moonshot misses the skip-discovery model result and falls back through broad models.json/provider discovery.

## Solution Assessment

Yes in direction: the narrow maintainable fix is to use the existing bundled static-catalog fast path, or an equivalent provider-owned Moonshot dynamic resolver, before broad discovery. Requiring users to add explicit models.providers.moonshot metadata should remain only a workaround.
