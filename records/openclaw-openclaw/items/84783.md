## Summary

Keep this open: current main still has a source-level path where native Moonshot can miss the fast embedded-run resolution path and fall back into `models.json` generation before dispatch, matching the reporter's logs and workaround.

## What This Changes

Native Moonshot provider runs are reported to spend 10-30 seconds in embedded-run model resolution before dispatch when no explicit `models.providers.moonshot` metadata is configured.

## Best Possible Solution

Resolve bundled Moonshot model refs from provider-owned static/dynamic metadata during the embedded runner's skip-PI path, and keep broad `models.json` generation as a later fallback instead of requiring users to add explicit Moonshot config.

## Reproduction Assessment

No, not as a full live current-main reproduction in this read-only pass. Source inspection plus the reporter's 2026.5.18/2026.5.19 logs and explicit-config workaround provide a clear reproduction path to target.

## Solution Assessment

Yes for keeping it open and routing a focused fix. The maintainable fix is to use the existing provider-owned static/dynamic metadata path, not to require a new config option or rely on shortening the global fallback timeout.
