## Summary

Keep this open: current main and the latest release still gate bundled static-catalog fallback to plugin-harness transport, while native Moonshot has no dynamic resolver and can still fall into broad model discovery before dispatch.

## What This Changes

Native Moonshot Discord/TUI runs report 10-30s pre-dispatch model-resolution latency when config lacks explicit Moonshot model metadata.

## Best Possible Solution

Land a narrowed provider-safe fast path that lets native Moonshot resolve bundled static catalog rows before broad `models.json` discovery, while preserving dynamic/runtime metadata precedence for other providers and proving before/after dispatch latency with redacted logs.

## Reproduction Assessment

No high-confidence live current-main reproduction was run. The reporter supplied concrete shipped-release timing logs and current source still shows native Moonshot can miss the skip-discovery static-catalog path and fall into broad discovery.

## Solution Assessment

No, current main is not the best solution because it still leaves native Moonshot outside the static-catalog fast path. The previously linked broad static-first PR is also not best as-is because it changes provider precedence beyond Moonshot without maintainer acceptance.
