## Summary

Keep open. Current main and the latest release still return the reported mixed-selector error before `sessions_send` can resolve or deliver by `sessionKey`, and open implementation PRs have not landed.

## What This Changes

The issue asks for `sessions_send` to stop failing when a client submits an explicit `sessionKey` along with an injected or redundant `label`.

## Best Possible Solution

Land one canonical fix that makes explicit `sessionKey` or sessionId routing robust to redundant label-style fields, preserves visibility/A2A confidentiality, updates tool guidance, and adds focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: call `createSessionsSendTool().execute` with both `sessionKey` and `label`, and current main returns the reported mutual-exclusion error before gateway resolution.

## Solution Assessment

Yes in direction, but not yet landed. Executor-side normalization is the narrow maintainable fix because schema or docs guidance alone would not protect client paths that already submit a polluted payload.
