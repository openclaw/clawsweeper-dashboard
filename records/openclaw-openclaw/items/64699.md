## Summary

Keep open. Current main and the latest release still return the reported mutual-exclusion error when `sessions_send` receives both `sessionKey` and `label`, and the active implementation path has not merged.

## What This Changes

A `sessions_send` call intended to target by `sessionKey` should not fail because a client or UI layer redundantly included `label`.

## Best Possible Solution

Land one canonical fix that makes explicit `sessionKey`/sessionId routing robust to redundant label-style fields, preserves label-only lookup and A2A/visibility checks, documents the precedence rule, and adds focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: call `sessions_send` with both `sessionKey` and `label`, and current main returns the reported mutual-exclusion error before gateway resolution.

## Solution Assessment

Yes in direction, but not landed yet. Executor-side normalization is the narrow maintainable fix because schema text alone cannot protect client paths that already emitted a polluted payload.
