## Summary

Keep open. Current main still returns the reported mutual-exclusion error whenever `sessions_send` receives both `sessionKey` and `label`, and the active implementation work has not landed.

## What This Changes

The issue asks for `sessions_send` calls routed by `sessionKey` to stop failing when a client path redundantly injects `label`.

## Best Possible Solution

Land one canonical fix that makes explicit `sessionKey` routing robust to redundant label fields, preserves label-only lookup and A2A policy checks, updates the tool description, and adds focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: call `sessions_send` with both `sessionKey` and `label`, and current main returns the reported mutual-exclusion error before any gateway resolution.

## Solution Assessment

Yes. Executor-side normalization is the narrow maintainable fix because schema or docs guidance alone would not protect client paths that already emitted a polluted payload.
