## Summary

Keep open. Current main still rejects a `sessions_send` payload containing both `sessionKey` and `label` before target resolution, the latest release source has the same guard, and open implementation PRs are still unmerged.

## What This Changes

Make `sessions_send` tolerate or normalize a redundant `label` when `sessionKey` is explicitly provided instead of failing with the mutual-exclusion error.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/74009 or an equivalent replacement that makes `sessionKey`/`sessionId` authoritative, preserves visibility and A2A checks, updates tool-facing guidance, and adds focused regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: call `sessions_send` with both `sessionKey` and `label`, and current main returns the reported mutual-exclusion error before gateway resolution.

## Solution Assessment

Yes. Executor-side normalization is the narrow maintainable fix because schema or docs guidance alone would not protect client paths that already emitted a redundant `label`.
