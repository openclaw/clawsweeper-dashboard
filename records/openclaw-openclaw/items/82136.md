## Best Possible Solution

## Reproduction Assessment

Yes, from source and the linked debug trace, though not from a live run in this read-only review. Current main stores missing WebChat `turnSourceTo` as null and then requires a nonempty value in the backend chat replay bridge, which leads to `APPROVAL_CLIENT_MISMATCH`.

## Solution Assessment

Yes. Switching only `turnSourceTo` to the existing optional matcher is the narrowest maintainable fix while the required channel/session checks and existing target-mismatch tests preserve replay binding.
