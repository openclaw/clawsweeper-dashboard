## Summary

Close as superseded: current main fixes the linked LM Studio JIT/preload problem through the canonical `models.providers.lmstudio.params.preload: false` surface, while this PR adds an older alternate top-level key.

## What This Changes

The PR adds a top-level `sendLoadApi` model-provider option and uses it to skip LM Studio `/api/v1/models/load` calls for inference and embeddings.

## Best Possible Solution

Keep the provider-owned `models.providers.lmstudio.params.preload: false` solution; handle any separately reproduced embedding warmup concern as a narrow follow-up against that same params key.

## Reproduction Assessment

Yes. The linked issue gives deterministic LM Studio JIT/TTL steps, and current main has a focused unit test proving `params.preload: false` skips inference preload.

## Solution Assessment

No for this PR. The current main solution is narrower and documented, while this PR would split configuration between `params.preload` and a second top-level `sendLoadApi` key.
