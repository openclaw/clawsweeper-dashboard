## Summary

Keep open. The PR implements the right narrow LM Studio opt-out direction for the linked JIT/TTL bug, but the patch is incomplete because it changes the public config surface without refreshing generated config artifacts and lacks the required changelog entry.

## What This Changes

This PR adds a `models.providers.lmstudio.sendLoadApi` opt-out that skips LM Studio `/api/v1/models/load` calls for inference preload and memory embeddings while preserving the default preload behavior.

## Best Possible Solution

Land the LM Studio-owned default-true `sendLoadApi` opt-out with generated config artifacts, public LM Studio docs, changelog coverage, and tests proving inference and embeddings skip the load call only when the option is false.

## Reproduction Assessment

Yes. The linked issue gives a detailed LM Studio JIT/TTL scenario, and current main deterministically posts to `/api/v1/models/load` before LM Studio inference and embeddings.

## Solution Assessment

Mostly yes. A provider-owned opt-out that preserves current preload behavior by default is the narrow maintainable fix, but this PR needs generated schema/docs, changelog, and focused tests before it is merge-ready.
