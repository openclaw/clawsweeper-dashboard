## Label Justifications

- `P1`: The report describes a real provider/session failure mode where one transcript block can make all later turns and forked subagents fail.
- `impact:session-state`: The failure is caused by persisted session transcript state being replayed in a way that makes the session unrecoverable.
- `impact:message-loss`: Once triggered, user turns receive provider errors instead of successful assistant replies.

## Summary

Keep open: current main strips missing or blank Anthropic thinking signatures, but it does not recover from a non-empty opaque signature that the direct Anthropic API later rejects. The related Bedrock report at https://github.com/openclaw/openclaw/issues/45010 is closed and covers empty signatures, not this direct-API stale-signature path.

## What This Changes

Report that direct Anthropic Claude 4.6 extended-thinking sessions can become permanently unrecoverable when a persisted thinking block signature is later rejected during history replay.

## Best Possible Solution

Add a targeted Anthropic replay recovery path that detects the provider's invalid-thinking-signature rejection, retries once with historical thinking stripped, and records focused regression coverage without changing Claude 4.6's normal replay policy.

## Reproduction Assessment

No live current-main reproduction was run. Source inspection plus the reporter's logs show the replay path: non-empty Claude thinking signatures are preserved, while current cleanup only removes missing or blank signatures.

## Solution Assessment

The blanket strip-all-old-thinking suggestion would likely recover sessions but may fight Claude 4.6 replay behavior. The better narrow fix is provider-error recovery that strips thinking only after Anthropic rejects the replay.
