The issue asks OpenClaw to allow `exec host=node` on a Companion App node when Gateway and node exec approvals are configured with `security=full` and `ask=off`.

## Best Possible Solution

Land one security-reviewed canonical fix, likely through https://github.com/openclaw/openclaw/pull/93583 or a maintainer-approved replacement, that makes full/off Gateway-to-Companion node exec work while preserving canonical migration and node approval boundaries.
