## What This Changes

The issue reports that the published v2026.6.11 npm package ships stale reply-session dist files, causing `reply session initialization conflicted` failures despite the source-side fix existing after the release tag.

## Best Possible Solution

Cut or promote a patch release from the fixed source and add a package/tarball parity guard that fails when bundled dist lacks the reentrant store-writer path.

## Reproduction Assessment

## Solution Assessment

Yes: the narrow product-correct fix is to ship a patched package built from the current fixed source, with a release tarball smoke/parity check so this source-vs-dist drift cannot recur. Editing installed dist locally is only a workaround.
