## What This Changes

The issue reports a Feishu WebSocket p2p DM regression where the bot receives messages after upgrading to 2026.5.12 but generated replies fail with `SUBSCRIPTION_NOT_FOUND`.

## Best Possible Solution

Land a narrow Feishu plugin fix that keeps typing/reply lifecycle tied to the inbound message but sends non-thread p2p visible replies through `user:<open_id>`, with focused regression tests and live Feishu proof if available.

## Reproduction Assessment

Yes, source-reproducible. A p2p event computes `user:<open_id>` for route metadata, but current main still constructs the reply dispatcher with `ctx.chatId`, and the dispatcher sends top-level visible replies to that target after DM reply metadata is suppressed.

## Solution Assessment

Yes, the current linked fix direction is the narrow maintainable path: separate the top-level delivery target from the inbound chat id for p2p DMs while preserving group/thread behavior. The issue should remain open until that fix is mergeable and reviewed or a replacement lands.
