## Summary

Keep open. Current main and the latest shipped release still compute the safer Feishu DM target as `user:<open_id>` but pass the raw p2p `oc_*` chat id into final reply dispatch after DM reply metadata is suppressed; the linked fix PR was closed unmerged, so this issue remains the canonical repair target.

## What This Changes

Feishu p2p direct-message replies fail after the 2026.5.12 upgrade because generated replies are not delivered to the user.

## Best Possible Solution

Route Feishu p2p final reply delivery through the computed `user:<senderOpenId>` target while preserving group/topic chat routing and adding focused regression coverage.

## Reproduction Assessment

Yes at source level, though not from a live tenant in this review. Current main carries p2p events as `oc_*`, suppresses DM reply metadata, and then sends final replies through the raw dispatcher `chatId`, matching the report's `replies=1` but no delivered reply.

## Solution Assessment

Yes. The narrow maintainable fix is to use the already-computed DM `user:<senderOpenId>` target for final delivery, rather than adding a retry fallback, config option, or Feishu-specific workaround elsewhere.
