## Summary

Keep open: the PR targets a real sessions.list performance hot path from the linked report, but the new CLI-provider cache can return stale classifications after plugin registry/setup metadata changes, and the required user-facing changelog entry is missing.

## What This Changes

The PR adds a process-wide isCliProvider result cache, limits Control UI chat-final session refreshes to five recent sessions, and adds a cache regression test.

## Best Possible Solution

Keep the chat refresh bounding, but make CLI-provider memoization invalidation-aware or scoped to a single session-list build, add regression coverage for runtime/setup registry changes, and add the Unreleased changelog entry.

## Reproduction Assessment

No high-confidence live reproduction was run in this read-only review. Source inspection does reproduce the relevant current-main path: chat-final refreshes call sessions.list without a limit, and the linked report supplies large-store timing data.

## Solution Assessment

No, not as written. Limiting the chat refresh is a narrow fit, but the global isCliProvider cache must account for plugin registry/setup metadata changes or be scoped so it cannot outlive the registry state it observed.
