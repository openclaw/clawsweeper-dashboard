## Summary

Keep open: the PR targets a real Control UI session-list cost path, but the proposed process-wide CLI-provider cache can return stale results after plugin registry or setup metadata changes, and the user-facing performance fix still needs a changelog entry.

## What This Changes

The branch adds an `isCliProvider` result cache, caps final chat-turn session refreshes to five rows, and adds a setup CLI backend cache regression test.

## Best Possible Solution

Keep the chat refresh bounding, but make CLI-provider memoization invalidation-aware or scoped to one session-list build, then add focused regression coverage and the Unreleased changelog entry.

## Reproduction Assessment

No live large-store reproduction was run in this read-only review. Source inspection does show the current-main chat-final refresh lacks a limit, and the linked report supplies the large-store timing evidence.

## Solution Assessment

No, not as written. Limiting the chat refresh is a narrow maintainable fix, but the global CLI-provider cache needs registry/setup invalidation or a shorter scope before merge.
