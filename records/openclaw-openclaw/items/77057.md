## Best Possible Solution

Keep the chat refresh bounding, but make CLI-provider memoization invalidation-aware or scoped to a single session-list build, add stale-cache regression coverage, and add the required Unreleased changelog entry.

No high-confidence live reproduction was run in this read-only review. Source inspection confirms current main's chat-final refresh calls sessions.list without a limit, and the linked issue supplies large-store timing data.

## Solution Assessment
