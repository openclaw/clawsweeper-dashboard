## Summary

Keep open: current main still misses documented system-scoped Linux gateway units in daemon status/control paths, but this PR should not merge as-is because it overuses sudo, misreports system-unit uninstall permission failures, and regresses newer update-restart guidance already on main.

## What This Changes

The PR adds system-vs-user systemd unit detection for Linux gateway daemon status/control and update restart script generation, with tests for system-scoped units.

## Best Possible Solution

Rebase or replace this with a focused daemon-systemd scope resolver that supports system units for managed-command discovery, status/runtime, and explicit control actions while preserving current update-helper guidance and using privilege only where needed.

## Reproduction Assessment

Yes, at source level. Current docs support /etc/systemd/system gateway units, while current main still reads the user unit path and routes enabled/runtime/stop/restart through user-scoped systemctl.

## Solution Assessment

No, not as currently implemented. The PR targets the right bug, but the safer fix should avoid sudo for read-only systemctl queries, report system-unit removal permission failures honestly, and preserve the newer update restart handoff already on main.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Reserve sudo for mutating systemd calls:** `src/daemon/systemd.ts:307-308`
  - body: `execSystemctlSystem` is used for read-only `is-enabled` and `show` paths as well as `stop` and `restart`, so `gateway status` can require passwordless sudo even though plain system `systemctl` reads are normally available. Keep sudo for explicit mutations or try direct reads before falling back.
  - confidence: 0.91
- **[P2] Handle system-unit removal permissions explicitly:** `src/daemon/systemd.ts:594-597`
  - body: When `context.unitPath` points under `/etc/systemd/system`, `fs.unlink` runs as the current user and EACCES or EPERM is caught as `not found`. Non-root system-unit uninstalls can leave the unit installed while printing a misleading message.
  - confidence: 0.9
- **[P2] Preserve the current system-unit update handoff:** `src/cli/update-cli/restart-helper.ts:88`
  - body: Current main's Linux restart helper logs restart attempts and emits explicit sudo guidance when it sees a system-scoped unit. This branch replaces that with automatic `sudo -n restart`, which can fail in root or no-sudo environments and drops the durable operator-facing handoff.
  - confidence: 0.88
- **[P2] Keep user-scope tests from selecting system scope:** `src/daemon/systemd.ts:80`
  - body: The new resolver probes `/etc/systemd/system` before the user unit path, so existing tests that mock every `fs.access` call as successful take the system-scope path and assert the wrong systemctl arguments. Make those mocks path-specific or avoid the extra access in user-scope cases.
  - confidence: 0.84
