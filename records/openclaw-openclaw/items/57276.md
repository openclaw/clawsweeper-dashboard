## Summary

Keep open: current main still leaves the daemon status/runtime/control paths tied to user-scoped systemd, while this branch is not merge-ready because it conflicts with newer update-restart behavior and still has privileged-systemd correctness issues.

## What This Changes

The PR adds Linux systemd system-unit detection for gateway daemon status/control and update restart script generation, with tests for system-scoped units.

## Best Possible Solution

Rebase or replace this with a focused daemon-systemd patch that detects system units for status/runtime/control, avoids sudo for read-only status, uses privilege only for explicit mutations, preserves current update-helper guidance, and covers both scopes.

## Reproduction Assessment

Yes, at source level. Current docs allow system-scoped gateway units, while current main's daemon managed-command, enabled, runtime, stop, and restart paths still resolve user-unit files or call user-scoped systemctl.

## Solution Assessment

No, not as currently implemented. The PR targets a real gap, but the safer path is a rebased patch that keeps current update-helper behavior, avoids sudo for read-only systemctl calls, and handles system-unit uninstall permissions explicitly.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Reserve sudo for mutating systemd calls:** `src/daemon/systemd.ts:307-308`
  - body: `execSystemctlSystem` is used for read-only `is-enabled` and `show` as well as stop/restart, so non-root `gateway status` can require passwordless sudo even though plain system `systemctl show` and `is-enabled` are normally readable. Keep sudo for explicit mutations or try direct reads first.
  - confidence: 0.91
- **[P2] Handle system-unit removal permissions explicitly:** `src/daemon/systemd.ts:590`
  - body: Once `context.unitPath` can point under `/etc/systemd/system`, this `fs.unlink` runs as the current user and the broad catch reports EACCES/EPERM as not found. Non-root system-unit uninstalls can leave the unit file installed while printing misleading output.
  - confidence: 0.9
- **[P2] Preserve the current system-unit update handoff:** `src/cli/update-cli/restart-helper.ts:88`
  - body: Current main's Linux restart helper logs restart attempts and emits explicit sudo guidance when it detects a system-scoped unit. This branch replaces that with automatic `sudo -n systemctl restart`, which can fail in root/no-sudo environments and drops the durable operator-facing handoff.
  - confidence: 0.88
- **[P2] Keep user-scope tests from selecting system scope:** `src/daemon/systemd.ts:86`
  - body: The new resolver probes `/etc/systemd/system` before the user unit path, so existing tests that mock every `fs.access` call as successful take the system-scope path and assert the wrong systemctl arguments. Make those mocks path-specific or avoid the extra access in user-scope cases.
  - confidence: 0.84
