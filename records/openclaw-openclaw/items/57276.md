## Summary

Keep open, but the PR is not mergeable as-is: current main still has the daemon-side systemd system-unit gap, while this branch conflicts with newer update-helper behavior and has privileged systemd handling blockers.

## What This Changes

The PR adds Linux systemd system-vs-user unit detection for gateway daemon status/control paths and update restart-script generation, with focused tests.

## Best Possible Solution

Rework this against current main as a focused systemd-scope fix: detect real system units, keep read-only status probes unprivileged, use explicit privilege or operator handoff for lifecycle mutations, preserve update-helper guidance, and coordinate with https://github.com/openclaw/openclaw/pull/68909.

## Reproduction Assessment

Yes, at source level: current main still routes daemon lifecycle/status through user-systemd paths while docs support /etc/systemd/system gateway units. The PR body also provides live WSL output, but I did not run a live systemd reproduction in this read-only review.

## Solution Assessment

No, not as currently implemented. The direction is valid, but the patch should avoid sudo for read-only probes, handle system-unit removal permissions honestly, and preserve current main's update-helper handoff behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Reserve sudo for mutating systemd calls:** `src/daemon/systemd.ts:307-308`
  - body: `execSystemctlSystem` wraps every system-scoped operation in `sudo -n`, including read-only `is-enabled` and `show` calls used by gateway status. On typical installs those probes can work without passwordless sudo, so status can fail where it should remain read-only.
  - confidence: 0.91
- **[P2] Handle system-unit removal permissions explicitly:** `src/daemon/systemd.ts:592-597`
  - body: When `context.unitPath` points under `/etc/systemd/system`, `fs.unlink` runs as the current user and EACCES/EPERM is caught as `not found`. Non-root system-unit uninstalls can leave the unit installed while printing a misleading message.
  - confidence: 0.9
- **[P2] Preserve the current system-unit update handoff:** `src/cli/update-cli/restart-helper.ts:88`
  - body: Current main's Linux restart helper logs restart attempts and emits explicit sudo guidance when it detects a system-scoped unit. This branch replaces that with automatic `sudo -n restart`, which can fail in root or no-sudo environments and drops the durable operator-facing handoff.
  - confidence: 0.88
- **[P2] Keep user-scope tests on the user path:** `src/daemon/systemd.ts:86`
  - body: The new resolver probes `/etc/systemd/system` before the user unit path, so existing tests that mock every `fs.access` call as successful take the system-scope path and assert the wrong `systemctl` arguments. Make those mocks path-specific or avoid the extra access in user-scope cases.
  - confidence: 0.84
