## Summary

Keep open. Current main still lacks the PR's central daemon lifecycle support for system-scoped systemd units, and this branch has correctness blockers plus conflicts with the newer update-restart diagnostics from https://github.com/openclaw/openclaw/pull/74920.

## What This Changes

The PR detects `/etc/systemd/system/<unit>.service` in the Linux daemon and update restart helper, then routes status/control operations through system-scope systemctl/sudo with focused tests.

## Best Possible Solution

Decide the supported system-unit lifecycle boundary, then revise this PR or replace it with a focused implementation that preserves current diagnostics, uses direct read-only `systemctl`, handles privileged mutations explicitly, and keeps user-unit behavior unchanged.

## Reproduction Assessment

Yes. Source inspection on current main shows the daemon lifecycle still resolves the user unit path and calls `systemctl --user`, while the PR body provides live WSL proof for the proposed system-unit behavior.

## Solution Assessment

No. The PR targets the right surface, but it is not the best fix as-is because it forces sudo for read-only checks, can hide system-unit removal permission failures, emits sudo even for root-run restart scripts, and must preserve the newer update diagnostics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Use direct systemctl for read-only system checks:** `src/daemon/systemd.ts:307-309`
  - body: `execSystemctlSystem` sends every non-root system-scope command through `sudo -n`, including read-only `is-enabled` and `show` calls. That makes `gateway status` fail on normal installs without passwordless sudo even though those queries should usually work unprivileged.
  - confidence: 0.91
- **[P2] Handle system-unit removal permission failures:** `src/daemon/systemd.ts:594-598`
  - body: After `context.unitPath` can point at `/etc/systemd/system/...`, uninstall still removes it with plain `fs.unlink` and reports every error as not found. A non-root uninstall will typically leave the system unit installed with `EACCES` hidden, so this needs privileged removal or an explicit permission error.
  - confidence: 0.9
- **[P2] Avoid sudo in root-run restart scripts:** `src/cli/update-cli/restart-helper.ts:88`
  - body: The generated Linux update restart script always emits `sudo -n systemctl restart ...` for system units. If the script is already running as root or on a minimal system without sudo, the restart can fail even though plain `systemctl restart ...` would work.
  - confidence: 0.86
- **[P2] Keep user-scope tests from matching the system probe:** `src/daemon/systemd.ts:86`
  - body: The new `/etc/systemd/system` probe runs before the user-unit check, so existing tests that mock all `fs.access` calls as successful now select the system path and assert the wrong systemctl arguments. Make those mocks path-specific or avoid the redundant system access in the user-scope cases.
  - confidence: 0.84
