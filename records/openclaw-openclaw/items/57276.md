## Summary

Keep open: current main partially supersedes the updater portion through the merged gateway recovery work, but the daemon state/runtime/control path still uses user-scoped systemd paths and this PR still has concrete correctness issues before it can merge.

## What This Changes

The PR adds Linux systemd system-unit detection for gateway daemon status/control and update restart script generation, with tests for system-scoped units.

## Best Possible Solution

Rebase or replace this with a focused systemd-scope patch that detects system units in service state/runtime/control, reads status without unnecessary sudo, uses privilege only for explicit mutations, preserves current update-helper guidance, and covers both user and system scopes.

## Reproduction Assessment

Yes, at source level. Current docs allow /etc system units, while current main's managed command, enabled check, runtime status, and control paths still flow through user-unit paths or user-scoped systemctl for the primary daemon service path.

## Solution Assessment

No, not as currently implemented. The PR targets a real gap, but it should be revised to avoid sudo for read-only status, handle system-unit uninstall permissions, preserve the current restart-helper handoff, and fix user-scope test mocks.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Reserve sudo for mutating systemd calls:** `src/daemon/systemd.ts:307-308`
  - body: `execSystemctlSystem` is used for `is-enabled` and `show` as well as restart/stop, so non-root `gateway status` would require passwordless sudo even though plain system `systemctl show` and `is-enabled` are normally readable. Keep sudo for explicit mutations or try direct reads first.
  - confidence: 0.91
- **[P2] Handle system-unit removal permissions explicitly:** `src/daemon/systemd.ts:590`
  - body: Once `context.unitPath` can point under `/etc/systemd/system`, this `fs.unlink` runs as the current user and the broad catch reports EACCES/EPERM as "not found". Non-root system-unit uninstalls can leave the unit file installed while printing misleading output.
  - confidence: 0.9
- **[P2] Preserve the current system-unit update handoff:** `src/cli/update-cli/restart-helper.ts:88`
  - body: Current main’s Linux restart helper logs restart attempts and emits explicit sudo guidance when it detects a system-scoped unit. This branch replaces that with automatic `sudo -n systemctl restart`, which can fail in root/no-sudo environments and drops the durable operator-facing handoff.
  - confidence: 0.88
- **[P2] Keep user-scope tests from selecting system scope:** `src/daemon/systemd.ts:86`
  - body: The new resolver probes `/etc/systemd/system` before the user unit path, so existing tests that mock every `fs.access` call as successful take the system-scope path and assert the wrong systemctl arguments. Make those mocks path-specific or avoid the extra access in user-scope cases.
  - confidence: 0.84
