## What This Changes

The PR adds guarded provider-contract reads in gateway startup plugin planning and a regression test for a throwing manifest `contracts` getter beside a configured Brave web-search provider.

## Best Possible Solution

Land the narrow guard after maintainer review confirms skip-on-unreadable-manifest is the intended fallback for malformed plugin metadata.

## Reproduction Assessment

Yes from source inspection: current main directly reads manifest provider contract arrays during startup planning, and a throwing `contracts` getter would abort the loop. I did not run the repro because this review is read-only.

## Solution Assessment

Yes, this appears to be an acceptable narrow fix for the demonstrated failure. Moving all startup provider ownership reads to installed-index contributions would be a broader refactor across several provider families, while this patch preserves the existing manifest-validation layer and prevents one malformed manifest record from aborting startup planning.
