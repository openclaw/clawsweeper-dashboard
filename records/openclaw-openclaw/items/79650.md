## Summary

Keep open. Current main still has post-compaction AGENTS.md section reinjection and startup daily-memory prelude as separate mechanisms, and the requested fresh-session section injection would add a new user-facing lifecycle/config contract that needs maintainer API judgment.

## What This Changes

The issue requests a fresh-session AGENTS.md section injection mechanism parallel to post-compaction section reinjection.

## Best Possible Solution

Define one canonical startup-section config surface, reuse the workspace-boundary AGENTS.md section extraction logic, and document how it interacts with startupContext and boot-md.

## Reproduction Assessment

Yes for the observed gap by source inspection. Current main injects AGENTS.md sections only from compaction paths, while fresh-session startupContext loads daily memory and the gateway only prepends that prelude for reset/new startup handling.

## Solution Assessment

Unclear until maintainers choose the contract. The requested mechanism is plausible, but the config name, trigger semantics, and boot-md/startupContext relationship should be settled before implementation.
