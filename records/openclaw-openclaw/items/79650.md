## Summary

Keep open. Current main still has post-compaction AGENTS.md section reinjection and daily-memory startupContext as separate paths, while this issue asks for a new fresh-session AGENTS.md section injection contract.

## What This Changes

Requests a fresh-session startup hook or config key that extracts configured AGENTS.md sections and injects them on first turns for parentSession-none sessions, parallel to postCompactionSections.

## Best Possible Solution

Define one canonical fresh-session startup-section config surface, reuse the workspace-boundary AGENTS.md section extraction logic, and document/test how it interacts with startupContext, boot-md, /new, /reset, and gateway-created fresh sessions.

## Reproduction Assessment

Yes, by source inspection rather than a live gateway run. Current main injects AGENTS.md sections only from post-compaction paths, while fresh-session startupContext loads daily memory and has no AGENTS.md section extraction surface.

## Solution Assessment

Unclear until maintainers choose the contract. The requested direction is plausible, but naming, trigger semantics, and interaction with startupContext and boot-md need product/API judgment before implementation.
