## Summary

Keep open. Current main has configurable post-compaction AGENTS.md section reinjection and a separate bare `/new`/`/reset` daily-memory startup prelude, but it does not have a config-backed AGENTS.md section injection path for ordinary parentless fresh sessions.

## What This Changes

The issue requests a fresh-session startup injection config that mirrors `postCompactionSections` for sessions without a parent session.

## Best Possible Solution

Define one first-turn fresh-session section-injection contract, reuse the existing AGENTS.md section extraction and context-limit patterns, and wire it through both chat and gateway first-turn paths with docs and focused regression tests.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is source inspection: current main only injects AGENTS.md sections after compaction and only preloads daily memory for bare reset/startup flows.

## Solution Assessment

Unclear until maintainers choose the public API and injection timing. Reusing `extractSections()` plus the existing first-turn prompt/system-event plumbing appears to be the narrowest maintainable direction.
