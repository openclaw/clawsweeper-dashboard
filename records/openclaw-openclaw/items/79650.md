## Summary

Keep open. Current main still separates post-compaction AGENTS.md section reinjection from fresh-session startupContext, and the requested fresh-session section injection is a new user-facing config/lifecycle contract rather than an already-defined bug fix.

## What This Changes

Request a fresh-session startup config or hook that extracts named AGENTS.md sections and injects them on the first turn for sessions without a parent session.

## Best Possible Solution

Define one canonical startup-section config surface, reuse the existing workspace-boundary AGENTS.md section extraction logic, and cover auto-reply plus gateway first-turn paths with focused tests and docs.

## Reproduction Assessment

Yes for the observed gap, by source inspection rather than a live gateway run. Current main injects AGENTS.md sections only from compaction paths, while fresh-session startupContext loads daily memory and does not extract AGENTS.md sections.

## Solution Assessment

Unclear until maintainers choose the contract. The requested mechanism is plausible, but the config name, trigger semantics, and boot-md/startupContext relationship need a product/API decision before implementation.
