## Summary

Keep open. Current main has compaction-only AGENTS.md section reinjection and a separate daily-memory startup prelude, but no configured first-turn AGENTS.md section injector for fresh parentless sessions.

## What This Changes

The issue requests a fresh-session startup section injection hook parallel to the existing post-compaction section injection.

## Best Possible Solution

Add a documented fresh-session AGENTS.md section injection surface only after maintainers choose whether it belongs under `startupContext`, compaction settings, or a shared runtime-context contract.

## Reproduction Assessment

Yes, from source inspection rather than a live gateway run: `readPostCompactionContext` is wired only from compaction paths, while `buildSessionStartupContextPrelude` only loads recent memory files. That confirms the requested fresh-session AGENTS.md section hook is absent on current main.

## Solution Assessment

Unclear. A parallel key may be workable, but the narrowest maintainable solution needs maintainer choice on config ownership, trust level, default sections, and interaction with existing `startupContext` and `boot-md`.
