## Summary

Keep open. Current main has post-compaction AGENTS.md section reinjection and a separate daily-memory startup prelude, but there is no equivalent fresh-session AGENTS.md section injection; this is a feature/config contract request rather than a broken existing contract.

## What This Changes

Request a configurable fresh-session startup injection that reads named AGENTS.md sections like postCompactionSections and applies them on parentless fresh sessions.

## Best Possible Solution

Choose a canonical config surface for fresh-session AGENTS.md section injection, reuse the existing workspace-boundary read and section extraction logic, and cover both auto-reply and gateway first-turn paths with focused tests and docs.

## Reproduction Assessment

Yes, by source inspection rather than a live gateway run. Current main injects AGENTS.md sections only from the compaction handlers, while the fresh-session startup prelude reads daily memory for bare /new and /reset and does not extract AGENTS.md sections.

## Solution Assessment

Unclear. The requested mechanism is plausible, but the key name, trigger boundary, and relationship to startupContext versus postCompactionSections are config/product decisions; boot-md only runs at gateway startup and is not an equivalent first-turn fresh-session guarantee.
