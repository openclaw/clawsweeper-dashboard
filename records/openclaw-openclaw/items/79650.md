## Summary

Keep open. Current main still has separate compaction-only AGENTS.md section reinjection and reset/startup daily-memory prelude behavior, while the requested fresh-session AGENTS.md section injection would add a new user-facing lifecycle/config contract.

## What This Changes

The issue requests an `onSessionStart`-style config surface that injects selected AGENTS.md sections into fresh sessions in parallel with `postCompactionSections`.

## Best Possible Solution

Define one canonical startup-section config surface, reuse the existing workspace-boundary AGENTS.md extraction helper, and cover chat plus gateway first-turn paths with docs and focused tests.

## Reproduction Assessment

Yes, by source inspection rather than a live gateway run. Current main invokes AGENTS.md section extraction only from compaction paths, while fresh startupContext handling loads daily memory for reset/new flows and does not extract AGENTS.md sections.

## Solution Assessment

Unclear until maintainers choose the contract. The requested mechanism is plausible, but the config name, trigger semantics, and interaction with existing startupContext and boot-md behavior should be decided before implementation.
