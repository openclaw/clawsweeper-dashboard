## Summary

Keep open. Current main still separates post-compaction AGENTS.md section reinjection from fresh-session startup context, and the requested fresh-session section injector would add a new user-facing config and lifecycle contract rather than close out as conservative cleanup.

## What This Changes

The issue requests a fresh-session startup hook such as `onSessionStart` or `startupSections` that reuses `postCompactionSections`-style AGENTS.md section extraction for sessions with no parent session.

## Best Possible Solution

Define one documented startup-section config surface, reuse the existing workspace-boundary AGENTS.md section extraction helper, and cover both auto-reply and gateway first-turn paths with focused tests.

## Reproduction Assessment

Yes for the source-level gap, not from a live gateway restart run. Current main injects configured AGENTS.md sections only after compaction, while fresh-session startup context loads daily memory prelude content and does not extract named AGENTS.md sections.

## Solution Assessment

Unclear until maintainers choose the public contract. The requested direction is plausible, but a new first-turn lifecycle hook and config key should be designed around `startupContext`, `boot-md`, and parent-session semantics.
