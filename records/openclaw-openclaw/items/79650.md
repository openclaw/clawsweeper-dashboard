## Summary

Close as ClawHub/plugin scope. Current main still does not add a built-in fresh-session AGENTS.md section config, but the requested behavior can be implemented outside core with the existing session lifecycle and prompt-injection plugin APIs, and OpenClaw's scope guidance points optional runtime policy to plugins unless a missing API is identified.

## What This Changes

Request to add a fresh-session startup hook/config that reads named AGENTS.md sections and injects them on the first turn, parallel to post-compaction section reinjection.

## Best Possible Solution

Publish this as an installable ClawHub code plugin that reads configured AGENTS.md sections for fresh session_start events and injects them with before_prompt_build, leaving core unchanged unless a concrete plugin API gap appears.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a feature request for a new core config/lifecycle behavior. Source inspection does confirm the built-in startupContext does not extract AGENTS.md sections, while current plugin APIs can implement that behavior externally.

## Solution Assessment

Yes, ClawHub is the best current solution. A code plugin can combine session_start, workspaceDir, and before_prompt_build/prependSystemContext without adding another one-off core config key or changing the post-compaction contract.
