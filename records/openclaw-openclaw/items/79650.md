## Summary

Keep open. Current main only supports configurable AGENTS.md section reinjection after compaction, while fresh-session startupContext is a daily-memory prelude; adding fresh-session section injection is new core config and product behavior.

## What This Changes

Request a fresh-session AGENTS.md section injection hook parallel to the existing post-compaction section injection.

## Best Possible Solution

Design a core startup-section config that reuses the existing AGENTS.md section extractor, documents exact fresh-session trigger semantics, and keeps trust/budget behavior aligned with startupContext and post-compaction context.
