## Summary

Keep open. Current main still has AGENTS.md section reinjection only on compaction successor paths, while fresh parentless session startup context is a separate daily-memory prelude; adding a first-turn AGENTS.md section hook would require a new or expanded config/runtime contract.

## What This Changes

The issue requests a fresh-session startup hook that extracts named AGENTS.md sections and injects them on the first turn for sessions without a parent session.

## Best Possible Solution

Add a documented fresh-session AGENTS.md section injection surface only after maintainers choose the config owner and how it composes with startupContext and compaction recovery.

## Reproduction Assessment

Yes, from source inspection: parentless or fresh startup paths run through buildSessionStartupContextPrelude, while AGENTS.md section extraction is only called from compaction successor paths. This establishes the requested fresh-session injection hook is absent on current main.

## Solution Assessment

Unclear. A parallel key may be workable, but the narrowest maintainable solution needs maintainer choice on config ownership, trust level, defaults, and interaction with startupContext and boot-md.
