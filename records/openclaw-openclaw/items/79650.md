## Summary

Keep open. Current main still has compaction-only AGENTS.md section reinjection plus a separate daily-memory startup prelude, but no first-turn AGENTS.md section injector for fresh parentless sessions.

## What This Changes

Add a fresh-session startup hook or config surface that reuses AGENTS.md H2/H3 section extraction for parentless sessions, parallel to postCompactionSections.

## Best Possible Solution

Define a documented fresh-session runtime-context surface that reuses AGENTS.md section extraction only after maintainers settle the config shape, naming, and trust semantics.

## Reproduction Assessment

Yes, from source inspection. The AGENTS.md section extractor is only wired from compaction paths, while fresh-session startup context currently loads daily memory files and reset prompt text instead of named AGENTS.md sections.

## Solution Assessment

Unclear. A parallel key may be workable, but the narrowest maintainable solution needs maintainer choice on whether this belongs under startupContext, compaction, or a shared runtime-context contract.
