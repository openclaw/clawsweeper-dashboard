## Summary

Keep open. Current main still has post-compaction AGENTS.md section reinjection and startup daily-memory prelude as separate paths, while the requested fresh-session AGENTS.md section injection would add a new lifecycle/config contract that needs maintainer API judgment.

## What This Changes

Add a fresh-session startup injection surface that reuses AGENTS.md section extraction for parentless sessions, parallel to postCompactionSections.

## Best Possible Solution

Define one canonical startup-section config surface, reuse the existing workspace-boundary AGENTS.md section extraction logic, and cover auto-reply plus gateway first-turn paths with focused tests and docs.

## Reproduction Assessment

Yes for the observed gap by source inspection. Current main injects AGENTS.md sections only from post-compaction paths, while fresh-session startupContext loads daily memory and does not extract AGENTS.md sections.

## Solution Assessment

Unclear until maintainers choose the contract. The requested mechanism is plausible, but config naming, parentless-session trigger semantics, and startupContext/boot-md interaction should be decided before code.
