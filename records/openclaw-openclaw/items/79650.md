## Summary

Keep open: current main has compaction-only AGENTS.md section reinjection and bare-reset daily-memory startup context, but no config-backed fresh-session AGENTS section injection for ordinary parentless first turns.

## What This Changes

Request a new config-backed fresh-session bootstrap that extracts named AGENTS.md sections and injects them on parentless first turns.

## Best Possible Solution

Define one explicit fresh-session context contract, likely by extending startupContext or adding a narrowly named sibling that reuses extractSections, documents parentless-session triggers, and adds schema/docs/tests without changing boot-md semantics.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection does confirm the reported gap: current main wires named AGENTS.md section extraction only to compaction paths, while startupContext is a bare /new or /reset daily-memory prelude.

## Solution Assessment

Unclear. The requested concept is plausible, but maintainers should decide whether the maintainable shape is a new key, an extension of startupContext, or a plugin/system-event recipe before implementation.
