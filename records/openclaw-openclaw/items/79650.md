## Summary

Keep open. Current main still has post-compaction AGENTS.md section reinjection, but fresh-session startup paths do not provide an equivalent section-injection contract; this is a new config and lifecycle feature that needs maintainer API/product judgment.

## What This Changes

Add a fresh-session startup injection config that reuses AGENTS.md section extraction for parentSession-null sessions, parallel to postCompactionSections.

## Best Possible Solution

Define one canonical startup-section config surface, reuse the existing workspace-boundary AGENTS.md section extraction logic, and cover auto-reply plus gateway first-turn paths with focused tests and docs.

## Reproduction Assessment

Yes by source inspection, not by a live gateway run. Current main invokes AGENTS.md section extraction only from compaction paths, while fresh-session startup paths are either bare /new/reset daily-memory preludes or normal bootstrap behavior.

## Solution Assessment

Unclear until maintainers choose the contract. The requested mechanism is plausible, but config naming, trigger semantics, and the boot-md/startupContext relationship need product/API judgment first.
