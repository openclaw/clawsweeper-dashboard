## Summary

Keep open. Current main has compaction-only AGENTS.md section reinjection plus a separate daily-memory startup prelude, but no configured first-turn AGENTS.md section injector for fresh parentless sessions.

## What This Changes

Request a fresh-session AGENTS.md section injection config surface parallel to `agents.defaults.compaction.postCompactionSections`.

## Best Possible Solution

Define a documented fresh-session AGENTS.md section injection contract, reusing the existing section extractor only after maintainers choose the config ownership and first-turn trust semantics.

## Reproduction Assessment

Yes, from source inspection rather than a live gateway run. Current main wires `readPostCompactionContext` only through compaction paths, while the first-turn startup prelude loads recent daily memory files and has no AGENTS.md section extraction for fresh parentless sessions.

## Solution Assessment

Unclear. A parallel key is plausible, but the narrowest maintainable solution needs maintainer choice on whether this belongs under `startupContext`, compaction settings, or a shared runtime-context contract.
