## Summary

Keep open. The PR still conflicts with current main's exec allowlist contract: main source, tests, and docs allow bare command-name patterns for PATH-resolved commands, while this branch adds tests/docs expecting bare names like `python3` to be ignored without a matching implementation change.

## What This Changes

The PR adds Exec V2 follow-through docs, code, and tests for Linux allowlist semantics, safe-bin trusted-dir filtering, cat/ls safe-bin rejection, effective policy merge cases, and interpreter script allowlist hardening.

## Best Possible Solution

Rebase or replace the branch, preserve current bare-command allowlist behavior unless maintainers explicitly change the matcher/docs/tests together, add the required changelog entry, then land the safe-bin and interpreter-script hardening with focused exec validation.

## Reproduction Assessment

Yes. A code-level reproduction is `rawExecutable: "python3"`, a resolved PATH location, and `allowlist: [{ pattern: "python3" }]`; current main's matcher treats that as a bare executable-name match, so the PR's added false expectation would fail.

## Solution Assessment

No as-is. The hardening direction is useful, but the narrow maintainable path is to either align the new tests/docs with the existing bare-name matcher or make an explicit maintainer-approved semantic change across code, docs, and tests.
