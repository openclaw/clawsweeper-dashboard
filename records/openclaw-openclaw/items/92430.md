Keep this PR open: the boundary guard correctly prevents the reported memory-wiki crash, but the external-contributor merge gate remains unmet because the supplied test output does not exercise missing `agentIds` in a real gateway or wiki-tool flow.

## What This Changes

The PR defaults missing memory artifact `agentIds` to an empty array while cloning and sorting public artifacts.

## Best Possible Solution

Normalize runtime-missing `agentIds` once in `listActiveMemoryPublicArtifacts`, retain the required producer type, and add focused coverage proving malformed plugin output becomes `agentIds: []` before wiki consumers run.

## Reproduction Assessment

Yes, at source level: returning an otherwise valid public artifact with a runtime-missing `agentIds` makes current code throw during cloning, and the linked issue reports the same user-visible failure on OpenClaw 2026.6.5.

## Solution Assessment

Yes in direction: normalizing at the public-artifact boundary is the narrowest maintainable fix; the comparator fallback is harmless but redundant after cloning, and focused regression coverage should accompany the change.
