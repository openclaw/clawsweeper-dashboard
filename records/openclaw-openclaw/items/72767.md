## Summary

Keep open. Current main does not implement `agent.capabilities`, and the PR is useful but not merge-ready because the new handler drops inherited default skills and the branch lacks the required changelog entry.

## What This Changes

The PR adds a READ-scoped `agent.capabilities` Gateway RPC with protocol schemas/types, Swift models, method-scope wiring, and a handler summarizing an agent's model, tools, MCP servers, skills, channels, and sessions.

## Best Possible Solution

Ship the additive Gateway RPC after it reuses the existing effective skill-filter contract, adds focused handler coverage for inherited defaults/replacement/explicit empty skills, and records the user-facing method in the changelog.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction: configure `agents.defaults.skills`, define an agent without its own `skills`, and the PR handler returns `[]` because it checks the resolved config object's own `skills` property.

## Solution Assessment

No. The RPC shape is maintainable, but the skill list should come from `resolveAgentSkillsFilter()` or `resolveEffectiveAgentSkillFilter()` rather than inspecting `resolveAgentConfig()` output.
