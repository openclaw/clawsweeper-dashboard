## Summary

Keep open. The PR is useful but not merge-ready because the new capabilities response misreports inherited default skills for configured agents, and the user-facing Gateway method still needs the required changelog entry. The remaining repair is narrow and source-backed.

## What This Changes

The PR adds a READ-scoped `agent.capabilities` Gateway RPC with protocol schemas/generated Swift models, method registration, and a handler that summarizes model, tools, MCP servers, skills, channel bindings, and active sessions.

## Best Possible Solution

Keep the additive Gateway RPC direction, but compute `skills` through the existing effective agent skill-filter contract, add focused handler coverage for inherited defaults and explicit overrides, and add a single-line changelog entry before maintainer review resumes.

## Reproduction Assessment

Yes. With `agents.defaults.skills: ["github"]` and `agents.list: [{ id: "writer" }]`, the PR's static code path returns `[]` for `writer` because `resolveAgentConfig()` returns an own `skills` property with `undefined`; the expected effective result is `github`.

## Solution Assessment

No. The RPC is a reasonable surface, but this implementation should reuse `resolveAgentSkillsFilter` or `resolveEffectiveAgentSkillFilter` instead of detecting raw overrides on a resolved config object.
