## Summary

Keep open. The PR is useful and current main does not already provide this RPC, but the branch still has a correctness bug in inherited skill reporting plus the required user-facing changelog entry is missing.

## What This Changes

The branch adds a READ-scoped `agent.capabilities` Gateway RPC with TypeBox/protocol exports, generated Swift models, method-list/scope wiring, and a server handler that summarizes an agent's model, tools, MCP servers, skills, channel bindings, and sessions.

## Best Possible Solution

Ship the additive Gateway RPC after it reports skills through the existing effective skill-filter contract, covers inherited defaults and explicit replacement in focused tests, and records the user-facing method in the Unreleased changelog.

## Reproduction Assessment

Yes. A focused unit test with `agents.defaults.skills: ["github"]` and an agent entry that omits `skills` would show the PR handler returning an empty `skills` list instead of the inherited default.

## Solution Assessment

No. The RPC direction is maintainable, but the skill portion should reuse `resolveAgentSkillsFilter` or `resolveEffectiveAgentSkillFilter` rather than inspecting the resolved config object's own properties.
