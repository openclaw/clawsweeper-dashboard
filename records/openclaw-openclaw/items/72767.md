## Best Possible Solution

## Reproduction Assessment

Yes. With `agents.defaults.skills: ["github"]` and `agents.list: [{ id: "writer" }]`, the PR handler reads an own `skills` property from the resolved config and returns `[]` instead of the inherited `github`.

## Solution Assessment

No. The RPC direction is reasonable, but the implementation should reuse `resolveAgentSkillsFilter` or the effective skill-filter helper instead of checking raw override ownership on a resolved config object.
