## What This Changes

The PR adds a READ-scoped `agent.capabilities` Gateway RPC plus protocol and Swift models that summarize an agent's model, tools, MCP servers, skills, bound channels, and active sessions.

## Best Possible Solution

Ship the additive RPC after skill reporting reuses the effective agent skill-filter contract, focused handler coverage covers inherited and explicit-empty skills, and the contributor posts redacted real Gateway proof.

## Reproduction Assessment

Yes, by source inspection. With `agents.defaults.skills` set and an agent entry that omits `skills`, the PR handler sees an owned `skills: undefined` on the resolved config object and returns `[]` instead of inherited defaults.

## Solution Assessment

No. The Gateway RPC direction is reasonable, but this implementation should reuse `resolveAgentSkillsFilter()` or `resolveEffectiveAgentSkillFilter()` for the skill list instead of inferring ownership from `resolveAgentConfig()`.
