The PR adds a READ-scoped `agent.capabilities` Gateway RPC, protocol schemas/generated Swift models, method registration, and a handler that reports model, tool, MCP, skill, channel, and session-count data for an agent.

## Best Possible Solution

Ship the additive Gateway RPC after it reports skills through the existing effective allowlist contract, includes focused handler coverage for inherited defaults and explicit replacement, and records the new user-facing method in the changelog.

## Reproduction Assessment

Yes. With `agents.defaults.skills: ["github"]` and `agents.list: [{ id: "writer" }]`, the PR handler resolves an own `skills: undefined` property and returns `[]`; current helper tests show the expected effective result is `github`.

## Solution Assessment

No. The RPC direction is maintainable, but this implementation should reuse `resolveAgentSkillsFilter` or the effective skill-filter helper instead of doing raw override detection on a resolved config object.
