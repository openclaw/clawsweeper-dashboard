## Summary

Keep open. Current main still lacks this Gateway RPC, so the PR is not obsolete, but the branch has a concrete correctness blocker: it reports no skills for agents that should inherit `agents.defaults.skills`. It also needs the required changelog entry for a user-facing Gateway feature.

## What This Changes

The PR adds a READ-scoped `agent.capabilities` Gateway RPC, protocol schemas/generated Swift models, method registration, and a handler that reports model, tool, MCP, skill, channel, and session-count data for an agent.

## Best Possible Solution

Ship the additive RPC after the handler uses the existing effective skill allowlist contract, adds focused handler coverage for inherited defaults and explicit replacement, and records the new user-facing method in the changelog.

## Reproduction Assessment

Yes. With `agents.defaults.skills: ["github"]` and `agents.list: [{ id: "writer" }]`, the PR handler resolves an own `skills: undefined` property and returns `[]`; current helper tests show the expected effective result is `github`.

## Solution Assessment

No. The RPC direction is maintainable, but the skill reporting should reuse `resolveAgentSkillsFilter` or the effective skill-filter helper instead of doing raw override detection on a resolved config object.
