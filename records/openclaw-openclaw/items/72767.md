## Summary

Keep open. The additive Gateway RPC is a plausible direction and the earlier Greptile import/export comments appear addressed, but the current PR head still drops inherited default skills from the reported capabilities and is missing the required user-facing changelog entry.

## What This Changes

The PR adds a READ-scoped `agent.capabilities` Gateway RPC plus protocol and Swift generated models summarizing an agent's model, tool counts, MCP servers, skills, bound channels, and session count.

## Best Possible Solution

Ship the additive Gateway RPC after it uses the existing effective skill-filter contract, adds focused handler coverage for inherited defaults and explicit replacement/empty skills, and records the new user-facing method in `CHANGELOG.md`.

## Reproduction Assessment

Yes. With `agents.defaults.skills` set and an agent entry that omits `skills`, the PR handler reads the resolved config object's own `skills: undefined` property and returns an empty list, while current helper tests expect inherited defaults.

## Solution Assessment

No. The RPC shape is maintainable, but the skill field should reuse `resolveAgentSkillsFilter` or `resolveEffectiveAgentSkillFilter` instead of inspecting the resolved config object's own properties.
