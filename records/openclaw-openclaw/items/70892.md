## Summary

Keep open. Current main already covers the locked-down `<available_skills>` use case through the existing agent skill allowlist, but it still lacks operator-configurable gates for the CLI quick reference, assistant output/reply directives, and messaging component guidance, and the docs still lack the requested `systemPromptOverride` cookbook.

## What This Changes

The issue requests default-preserving prompt controls to omit selected system-prompt sections, filter `<available_skills>` for single-purpose pods, and document which prompt blocks are structural versus optional for full overrides.

## Best Possible Solution

Add default-true prompt-section controls for the remaining prompt blocks, wire them through the shared prompt builder used by embedded and CLI runs, keep skills trimming on the existing `agents.defaults.skills` / `agents.list[].skills` allowlist contract, and document structural versus optional prompt pieces for override users.

## Reproduction Assessment

Not applicable as a runtime bug. Static current-main inspection is enough to reproduce the gap: the named prompt sections still render without operator gates, while skill allowlisting already trims `<available_skills>`.

## Solution Assessment

Partly yes. Per-section gates are a narrow maintainable way to reduce prompt cost, but adding a new `availableSkills.include/exclude` schema would duplicate the existing skill allowlist and should be avoided unless maintainers choose that product shape.
