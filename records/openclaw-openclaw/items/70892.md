## Summary

Keep open. Current main still lacks the requested public prompt-section gates and the systemPromptOverride cookbook; the skills include-style use case is already covered by existing agent skill allowlists, while subtractive skill filtering would be a new config/product decision.

## What This Changes

The issue requests default-on config gates for selected system-prompt blocks, filtering for the available skills prompt, and docs explaining safe systemPromptOverride building blocks.

## Best Possible Solution

Add maintainer-approved default-on prompt-section controls for the optional blocks, reuse the existing skills allowlist unless exclude semantics are intentionally accepted, and document structural versus optional prompt pieces for override users.

## Solution Assessment
