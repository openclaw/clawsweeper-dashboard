## Summary

Keep open. Current main has partially reduced the old prompt text, and existing agent skill allowlists already cover the include-style available-skills use case, but there are still no operator-configurable prompt-section gates for the remaining CLI/control, assistant-output, or messaging guidance and the requested systemPromptOverride cookbook is still missing.

## What This Changes

The issue requests default-preserving config and docs support to omit selected system-prompt sections and trim `<available_skills>` for single-purpose deployments.

## Best Possible Solution

Add default-true prompt-section controls for the remaining prompt blocks, keep skills trimming on the existing agent skill allowlist contract, and document required versus optional prompt pieces for override users.

## Reproduction Assessment

Not applicable as a runtime bug. Static source inspection gives high confidence that the requested operator gates and override cookbook are still absent, while the available-skills include case is already handled by `agents.defaults.skills` and `agents.list[].skills`.

## Solution Assessment

Partly yes. Per-section gates and documentation are the narrow maintainable direction, but a new `availableSkills.include/exclude` surface would duplicate the existing allowlist unless maintainers intentionally choose separate subtractive semantics.
