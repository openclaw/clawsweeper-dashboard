## Summary

Keep open. Current main still has no operator-configurable gates for the CLI quick reference, assistant output/reply directives, or messaging component guidance, and the docs still lack the requested `systemPromptOverride` structural-vs-optional cookbook; the `<available_skills>` include-style use case is already covered by existing agent skill allowlists.

## What This Changes

The issue requests default-preserving prompt controls to omit selected system-prompt sections, filter `<available_skills>` for single-purpose pods, and document which prompt blocks are structural versus optional for full overrides.

## Best Possible Solution

Add default-true prompt-section controls for the remaining prompt blocks, keep skills trimming on the existing `agents.defaults.skills` / `agents.list[].skills` allowlist contract, and document required versus optional prompt pieces for override users.

## Reproduction Assessment

Not applicable as a runtime bug. Static current-main inspection is enough to reproduce the gap: the named prompt sections still render without operator gates, while skill allowlisting already trims `<available_skills>`.

## Solution Assessment

Partly yes. Per-section gates are a narrow maintainable way to reduce prompt cost, but adding a new `availableSkills.include/exclude` schema would duplicate the existing skill allowlist unless maintainers explicitly want that product surface.
