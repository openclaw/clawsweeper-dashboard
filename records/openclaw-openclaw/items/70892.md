## Summary

Keep open. Current main and the latest release still lack the requested public prompt-section gates and the systemPromptOverride cookbook; the include-style skills case is already covered by agent skill allowlists, while subtractive skill filtering remains a new config/product decision.

The issue requests default-on prompt-section gates, an available-skills filter, and docs explaining safe systemPromptOverride building blocks.

## Best Possible Solution

Add maintainer-approved default-on prompt-section controls, keep existing skill allowlists for include-style filtering unless exclude semantics are deliberately accepted, and document structural versus optional prompt pieces for override users.

## Solution Assessment

Partly yes. Section gates and docs are a plausible narrow direction, but the availableSkills include/exclude namespace should not duplicate or conflict with the existing allowlist contract without maintainer approval.
