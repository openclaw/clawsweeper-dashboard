Full review comments:

- **[P2] Restore update state required by runUpdate:** `ui/src/ui/app-view-state.ts:157-158`
  - body: `runUpdate(state)` still requires `ConfigState.pendingUpdateExpectedVersion` and `updateStatusBanner`, and `app-render` still passes `AppViewState` into it. This branch removes those members from `AppViewState`/`OpenClawApp`, so the Control UI no longer type-checks and cannot persist the expected post-update version.
  - confidence: 0.88
- **[P2] Replay queued aborts after reconnect:** `ui/src/ui/app-gateway.ts:341-345`
  - body: `handleAbortChat` still queues `host.pendingAbort` when Stop is pressed during a disconnect, but this reconnect path now clears `chatRunId` and stream state without sending that queued `chat.abort`. A user who clicks Stop while the socket is down will see the local run cleared after reconnect while the gateway run keeps executing.
  - confidence: 0.9
- **[P2] Restore post-update verification on hello:** `ui/src/ui/app-gateway.ts:338-345`
  - body: After `update.run` succeeds, current Control UI polls `update.status` on the next hello and surfaces mismatched or unhealthy restarts. This branch removes that verification from the hello path, so a failed package/git update can reconnect without the error banner required by the Control UI update contract.
  - confidence: 0.86
