## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Restore update state required by runUpdate:** `ui/src/ui/app-view-state.ts:157-158`
  - body: `runUpdate(state)` still requires `pendingUpdateExpectedVersion` and `updateStatusBanner`, and this branch still calls `runUpdate(state)` from app-render. Removing those fields from `AppViewState` means the Control UI cannot type-check or persist the expected post-update version for reconnect verification.
  - confidence: 0.9
- **[P2] Replay queued aborts after reconnect:** `ui/src/ui/app-gateway.ts:341-345`
  - body: `handleAbortChat` queues `host.pendingAbort` when Stop is pressed while disconnected, but this hello path clears the local run and tool state without sending the queued `chat.abort`. A disconnected Stop can therefore make the UI look stopped while the gateway run keeps executing.
  - confidence: 0.91
- **[P2] Restore post-update verification on hello:** `ui/src/ui/app-gateway.ts:338-345`
  - body: After a successful `update.run`, current Control UI polls `update.status` on reconnect and surfaces mismatched or unhealthy restarts. This branch removes that verification from the hello path, so failed package/git updates can reconnect with no error banner.
  - confidence: 0.88
- **[P2] Show exhausted auth mismatch errors:** `ui/src/ui/app-gateway.ts:375-386`
  - body: `AUTH_TOKEN_MISMATCH` is recoverable only while the browser client's bounded device-token retry can still run. After a previous connection, this branch clears `lastError` and `lastErrorCode` for recoverable closes, so an exhausted token mismatch can be hidden behind a reconnect banner instead of telling the operator to fix credentials.
  - confidence: 0.86
- **[P3] Preserve loopback alias client-version handshakes:** `ui/src/ui/app-gateway.ts:195`
  - body: Current main treats `localhost`, `127.*`, and `::1` on the same effective port as the same Control UI endpoint so the browser sends `clientVersion`. Requiring exact `host` equality drops that version on common loopback alias launches, weakening stale-UI detection covered by existing tests.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: Needs attention: the reconnect changes can hide exhausted auth-token mismatch feedback after a prior successful connection.

Concerns:

- **[medium] Auth mismatch can be hidden behind reconnect UI:** `ui/src/ui/app-gateway.ts:375`
  - body: The PR treats recoverable closes after a prior connection as reconnecting and clears `lastError`/`lastErrorCode`. Because current Gateway auth intentionally excludes `AUTH_TOKEN_MISMATCH` from non-recoverable errors only for the browser client's bounded retry path, an exhausted mismatch can lose the security-relevant credential message operators need to fix access.
  - confidence: 0.86
