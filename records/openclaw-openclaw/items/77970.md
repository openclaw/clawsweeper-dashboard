## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Restore update state required by runUpdate:** `ui/src/ui/app-view-state.ts:157-160`
  - body: `runUpdate(state)` still requires `pendingUpdateExpectedVersion` and `updateStatusBanner`, and this branch still passes `state` to `runUpdate`. Removing those fields from `AppViewState` and `OpenClawApp` breaks the type contract and loses the expected post-update version needed for reconnect verification.
  - confidence: 0.9
- **[P2] Replay queued aborts after reconnect:** `ui/src/ui/app-gateway.ts:341-345`
  - body: `handleAbortChat` queues `host.pendingAbort` when Stop is pressed while disconnected, but this hello path clears local run/tool state without sending the queued `chat.abort`. A disconnected Stop can therefore make the UI look stopped while the gateway run keeps executing.
  - confidence: 0.91
- **[P2] Restore post-update verification on hello:** `ui/src/ui/app-gateway.ts:338-360`
  - body: After a successful `update.run`, current Control UI polls `update.status` on reconnect and surfaces mismatched or unhealthy restarts. This branch removes that verification from the hello path, so failed package/git updates can reconnect with no error banner.
  - confidence: 0.88
- **[P2] Show exhausted auth mismatch errors:** `ui/src/ui/app-gateway.ts:375-386`
  - body: `AUTH_TOKEN_MISMATCH` is recoverable only while the browser client's bounded device-token retry can still run. After a previous connection, this branch clears `lastError` and `lastErrorCode` for recoverable closes, so an exhausted token mismatch can be hidden behind a reconnect banner instead of telling the operator to fix credentials.
  - confidence: 0.86
- **[P2] Keep Stop available for active sessions:** `ui/src/ui/app-render.ts:1931`
  - body: Current main uses `hasAbortableSessionRun(state)` so Stop remains available when `sessionsResult` says the current session still has an active run after reload/reconnect. Reducing this to `Boolean(state.chatRunId)` hides Stop exactly when the browser lost the local run id but the gateway is still running the task.
  - confidence: 0.86
- **[P2] Restore session.message history reloads:** `ui/src/ui/app-gateway.ts:545`
  - body: Current main handles `session.message` by deferring history reloads while a run is active and replaying them after terminal events, including silent final payloads. Removing that branch means persisted messages for the active session can fail to appear until a manual refresh or later full reload.
  - confidence: 0.84
- **[P3] Preserve loopback alias client-version handshakes:** `ui/src/ui/app-gateway.ts:198`
  - body: Current main treats `localhost`, `127.*`, and `::1` on the same effective port as the same Control UI endpoint so the browser sends `clientVersion`. Requiring exact `host` equality drops that version on common loopback alias launches, weakening stale-UI detection covered by existing tests.
  - confidence: 0.78
