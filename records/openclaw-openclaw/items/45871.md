## Summary

Keep open. Current main still enforces and documents a 5-turn cap, so the PR is not obsolete, but the branch is not merge-ready because it updates only part of the config/docs surface, edits localized docs that this repo does not maintain, lacks the required changelog entry, and still needs maintainer product judgment on the 20-turn ceiling.

## What This Changes

The PR raises the maximum `session.agentToAgent.maxPingPongTurns` value from 5 to 20 in the runtime clamp, source Zod validation, type/help text, and selected docs while keeping the default at 5.

## Best Possible Solution

If maintainers accept the 20-turn cap, land a rebased branch that updates runtime clamping, source validation, generated base config schema/help, current English config docs, focused boundary coverage, and a changelog entry together, while removing localized-doc edits from this repository.

## Reproduction Assessment

Yes. A static current-main reproduction path is clear: values above 5 are rejected by `SessionSchema` and also clamped back to 5 by `resolvePingPongTurns`; tests were not run because this was a read-only review.

## Solution Assessment

No, not as currently proposed. Raising both the runtime clamp and source schema ceiling is the narrow technical direction if the cap is accepted, but the PR also needs generated/public config surfaces, current docs, i18n-policy cleanup, changelog coverage, and boundary proof before merge.
