## Summary

This PR is still necessary: current main and the latest release still pass build-all TypeScript script steps through `--experimental-strip-types` unchanged, while the branch adds a narrow tested fallback with real affected-host proof. There is no concrete line-level blocker, so this should stay open for maintainer review and normal merge/check gating.

## What This Changes

The branch adds a build-all helper that rewrites `node --experimental-strip-types <script>.ts` steps to `node --import tsx <script>.ts` when the current Node binary lacks native TypeScript stripping, with focused regression tests.

## Best Possible Solution

Land the narrow build-all fallback if maintainers accept the build-all-only scope, and track direct package-script compatibility separately only if that wider scope is desired.

## Reproduction Assessment

Yes. Running `pnpm build` on a Node binary where `process.features.typescript` is false is the high-confidence path; current main leaves build-all strip-types args unchanged, and the PR body supplies before/after proof from such a host.

## Solution Assessment

Yes for the reported build-all path. `resolveBuildAllStep` is the command-selection boundary for `pnpm build`, and this patch is narrower than reverting the Node cleanup or rewriting all direct package scripts in the same PR.
