## What This Changes

The PR adds a build-all node-step argument resolver that falls back from native TypeScript stripping to `node --import tsx` for `.ts` scripts when the current Node binary reports no TypeScript support, with focused tests.

## Best Possible Solution

Land the narrow build-all fallback after normal maintainer review and merge checks; handle direct package-script fallbacks separately only if maintainers want that broader scope.

## Reproduction Assessment

Yes. The current-main source path passes `--experimental-strip-types` through unchanged, and the PR body supplies real Debian/Node terminal proof where `process.features.typescript` is false; this read-only review did not rerun the build locally.

## Solution Assessment

Yes. `resolveBuildAllStep` owns build-all command selection, so a narrow node-step fallback is the maintainable fix; rewriting every direct package script would be broader than the reported `pnpm build` failure.
