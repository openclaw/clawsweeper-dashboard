## Summary

Keep open: current main still runs build-all TypeScript script steps through native `--experimental-strip-types`, while this PR adds a narrow fallback for Node builds without native TypeScript stripping and covers the routing with focused tests. I did not find a blocking correctness or security issue in the patch.

## What This Changes

The branch adds a build-all node-argument resolver that uses `node --import tsx` for `.ts` script steps when native TypeScript stripping is unavailable, plus focused build-all tests.

## Best Possible Solution

Land the narrow build-all fallback for `pnpm build`; track direct package-script aliases separately only if maintainers want repository-wide fallback behavior outside build-all.

## Reproduction Assessment

Yes, from source and submitted real-host proof. Current main unconditionally invokes `node --experimental-strip-types` for build-all TypeScript steps, and the PR body records an affected Node v24.15.0 host with `process.features.typescript=false`; I did not rerun that host in this read-only review.

## Solution Assessment

Yes for the reported `pnpm build` failure. Resolving the fallback inside `resolveBuildAllStep` is the narrow owner boundary; rewriting every package-script alias would be a broader follow-up rather than a prerequisite for this patch.
