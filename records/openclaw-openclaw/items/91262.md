## Summary

Keep open for maintainer review: current main still runs `build-all` TypeScript script steps through native stripping only, while this PR adds a narrow `tsx` fallback with focused tests and sufficient real-host proof. I found no blocking patch defects.

## What This Changes

The PR changes `scripts/build-all.mjs` to route `node --experimental-strip-types <script>.ts` build steps through `node --import tsx` when the current Node binary lacks TypeScript stripping, and adds regression coverage.

## Best Possible Solution

Land the narrow `build-all` fallback after ordinary maintainer and CI review; track broader direct package-script compatibility separately if maintainers want that scope.

## Reproduction Assessment

Yes. Current main unconditionally routes build-all `.ts` steps through native stripping, and the PR body supplies before/after proof from a real Node `v24.15.0` host where `process.features.typescript === false`; I did not run that Debian Node binary locally.

## Solution Assessment

Yes. For the reported `pnpm build` failure, the narrowest maintainable fix is to preserve native stripping where available and use the existing `tsx` dependency only for unsupported Node builds.
