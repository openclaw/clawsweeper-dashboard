## Summary

Keep open: current main and the latest release still route build-all TypeScript node steps directly through native strip-types, while this PR adds a narrow fallback with focused coverage and sufficient real-host proof. I found no blocking patch defect; the remaining decision is whether maintainers accept build-all-only scope before merge.

## What This Changes

The PR makes build-all rewrite TypeScript node steps from `--experimental-strip-types` to `--import tsx` only when the current Node binary does not report native TypeScript stripping support, and adds regression coverage for that routing.

## Best Possible Solution

Merge the build-all fallback if maintainers accept that scope; if all build entrypoints must support TypeScript-disabled Node binaries, add a follow-up shared wrapper for the direct package scripts.

## Reproduction Assessment

Yes. Current main and v2026.6.8 pass build-all TypeScript steps directly to `--experimental-strip-types`, and the PR body supplies before/after proof from a Debian Node v24.15.0 host where `process.features.typescript` was false.

## Solution Assessment

Yes for the reported `pnpm build` failure. `resolveBuildAllStep` is the build-all command-selection seam, and the patch preserves native stripping where supported while keeping direct package-script scope as a maintainer follow-up decision.
