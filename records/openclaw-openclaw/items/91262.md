## What This Changes

The PR rewrites build-all node TypeScript script steps to use `node --import tsx` when native TypeScript stripping is unavailable, with focused regression coverage.

## Best Possible Solution

Land the narrow build-all fallback after maintainer review and merge checks, and track broader direct package-script compatibility separately only if maintainers want that wider support.

## Reproduction Assessment

Yes. Source inspection shows current main passes strip-types args unchanged, and the PR body supplies before/after terminal evidence from Node v24.15.0 with `process.features.typescript=false`; I did not rerun that host in this read-only review.

## Solution Assessment

Yes for the reported build-all path. `resolveBuildAllStep` is the command-selection boundary for `pnpm build`, and the fallback keeps native stripping where available while repairing only `.ts` strip-types steps when native support is absent.
