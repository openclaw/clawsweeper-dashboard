## What This Changes

The PR changes `scripts/build-all.mjs` so eligible `.ts` build-all node steps fall back from native `--experimental-strip-types` to `node --import tsx` when the current Node binary does not report TypeScript support, with regression tests in `test/scripts/build-all.test.ts`.

## Best Possible Solution

Land the narrow build-all fallback after ordinary maintainer review and current-head validation; handle direct strict-smoke/package-script fallbacks separately if affected hosts need those surfaces too.

## Reproduction Assessment

Yes. Current main passes `--experimental-strip-types` unchanged for build-all TypeScript node steps, and the PR body supplies real affected Debian/Node proof where `process.features.typescript` is false; this read-only review did not execute the failing build locally.

## Solution Assessment

Yes for the reported `pnpm build` failure. Handling the fallback inside `resolveBuildAllStep` is the narrow build-owner boundary; rewriting all package scripts would be broader and should be a separate decision for direct strict-smoke surfaces.
