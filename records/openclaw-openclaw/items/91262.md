## Summary

Keep open: current main and v2026.6.9 still pass build-all TypeScript node steps through native `--experimental-strip-types`, while this PR narrowly fixes the reported `pnpm build` failure mode with sufficient real CLI proof and focused resolver tests.

## What This Changes

The PR updates `scripts/build-all.mjs` to fall back from native TypeScript stripping to `node --import tsx` for build-all `.ts` node steps when the current Node binary lacks native TypeScript support, with regression coverage in `test/scripts/build-all.test.ts`.

## Best Possible Solution

Land the narrow build-all fallback after maintainer review and merge checks, and track broader direct package-script compatibility separately only if maintainers want that wider scope.

## Reproduction Assessment

Yes. Source inspection shows current main passes native strip-types args unchanged, and the PR body supplies before/after proof from an affected Node v24.15.0 environment where `process.features.typescript=false`; I did not rerun that external host in this read-only review.

## Solution Assessment

Yes for the reported build-all path. `resolveBuildAllStep` is the command-selection boundary for `pnpm build`, and this is narrower than reverting the Node 26 cleanup or rewriting all direct package scripts in the same PR.
