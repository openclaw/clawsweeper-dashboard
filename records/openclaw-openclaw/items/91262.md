## What This Changes

The PR updates `scripts/build-all.mjs` to fall back from native strip-types to `node --import tsx` for build-all `.ts` node steps when the current Node binary does not report TypeScript support, with focused resolver tests.

## Best Possible Solution

Land the narrow `build-all` fallback after maintainer review and merge checks, and track broader direct package-script compatibility separately if maintainers want that scope.

## Reproduction Assessment

Yes. Source inspection shows current main passes native strip-types args unchanged, and the PR body supplies an affected Node v24.15.0 environment where `process.features.typescript=false`; I did not rerun that external host in this read-only review.

## Solution Assessment

Yes for the reported `pnpm build` path. `resolveBuildAllStep` is the command-selection boundary for build-all, and the PR is narrower than reverting the Node 26 cleanup or rewriting direct package scripts in the same change.
