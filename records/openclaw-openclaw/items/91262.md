## Summary

Keep open: current main and the latest release still pass build-all TypeScript node steps through native `--experimental-strip-types`, while this PR adds a narrow fallback with sufficient real-host proof and no blocking review findings.

## What This Changes

This PR adds a `scripts/build-all.mjs` node-step argument resolver that switches `--experimental-strip-types <script>.ts` to `--import tsx <script>.ts` when Node reports no TypeScript support, with focused regression tests.

## Best Possible Solution

Land the narrow build-all fallback after maintainer review and merge checks; consider a separate follow-up only if direct package scripts must support Node builds without native stripping.

## Reproduction Assessment

Yes. Current main and v2026.6.8 pass `--experimental-strip-types` through unchanged, and the PR body supplies terminal proof on Debian Node v24.15.0 where `process.features.typescript` is false; this read-only review did not rerun that host.

## Solution Assessment

Yes for the reported `pnpm build` failure. `resolveBuildAllStep` owns build-all command selection, so this fallback is narrower than reverting the Node 26 cleanup or editing every direct package script.
