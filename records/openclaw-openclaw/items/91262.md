## Summary

Keep this PR open: current main and the latest release still leave build-all TypeScript script steps on `node --experimental-strip-types`, while the PR adds a narrow fallback with focused tests and supplied real-host proof. I found no blocking code-review or security findings; the remaining decision is maintainer acceptance of the build-all-only scope.

## What This Changes

The branch changes `scripts/build-all.mjs` to rewrite build-all `node --experimental-strip-types <script>.ts` steps to `node --import tsx <script>.ts` when Node lacks native TypeScript support, and adds regression coverage for that routing.

## Best Possible Solution

Land the narrow build-all fallback if maintainers accept the build-all-only scope, and track direct package-script compatibility separately only if that wider source-build guarantee is desired.

## Reproduction Assessment

Yes, source inspection plus supplied proof gives a high-confidence path: on a Node binary where `process.features.typescript` is false or missing, current main still invokes `node --experimental-strip-types` for build-all `.ts` steps. I did not run the full failing build locally because this review is read-only and the local Node reports native TypeScript support.

## Solution Assessment

Yes for the reported `pnpm build` path. `resolveBuildAllStep` is the command-selection boundary, and the PR is narrower than rewriting all package scripts or reverting the current native strip-types preference.
