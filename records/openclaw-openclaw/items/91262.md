## What This Changes

The PR changes build-all node step resolution to use `tsx` for TypeScript build scripts when the current Node binary lacks native TypeScript stripping, with focused resolver tests.

## Best Possible Solution

Land the narrow build-all fallback once maintainer review and required checks are satisfied; handle direct package-script no-native-TS support separately if maintainers want that broader compatibility.

## Reproduction Assessment

Yes from source and supplied proof: current main passes native strip-types args unchanged, and the PR body reports an affected Node v24.15.0 checkout with `process.features.typescript=false`. I did not rerun the affected host in this read-only review.

## Solution Assessment

Yes for the reported `pnpm build` path: `resolveBuildAllStep` owns build-all command selection, and the fallback is narrower than reverting the Node cleanup or rewriting every direct package script in this PR.
