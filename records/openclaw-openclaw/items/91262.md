## Summary

Keep this PR open for normal maintainer review: current main still unconditionally sends build-all TypeScript steps through `--experimental-strip-types`, while the PR is a focused fallback with sufficient real-host proof and no blocking correctness findings from this review.

## What This Changes

The PR adds a build-all resolver fallback from native Node TypeScript stripping to `node --import tsx` for `.ts` node steps when `process.features.typescript` is unavailable or false, plus focused regression coverage.

## Best Possible Solution

Land the narrow build-all resolver fallback after ordinary maintainer validation, keeping native strip-types as the default path and using `tsx` only for affected Node builds.

## Reproduction Assessment

Yes. Current main passes `--experimental-strip-types` unchanged for build-all `.ts` node steps, and the PR supplies an affected Debian Node environment where `process.features.typescript` is false and the build failed at `write-plugin-sdk-entry-dts`; this read-only review did not rerun that host.

## Solution Assessment

Yes. Resolving the fallback in `resolveBuildAllStep` is the narrowest owner boundary for `pnpm build` and package-artifact builds; broad package-script rewrites or requiring a different Node build would be less targeted.
