## What This Changes

The PR changes `scripts/build-all.mjs` so build-all node steps that run `.ts` scripts with `--experimental-strip-types` fall back to `node --import tsx` when the current Node binary lacks native TypeScript stripping, and adds regression tests for that routing.

## Best Possible Solution

Land the narrow build-all resolver fallback after ordinary maintainer and current-head validation, keeping native strip-types as the default path and using `tsx` only for affected Node builds.

## Reproduction Assessment

Yes. The current main code unconditionally passes `--experimental-strip-types` to build-all TypeScript script steps, and the PR supplies an affected Debian Node environment where `process.features.typescript` is false.

## Solution Assessment

Yes. Resolving the fallback in `resolveBuildAllStep` is the narrowest owner boundary for `pnpm build`; broader package-script rewrites or requiring a different Node build would be less targeted.
