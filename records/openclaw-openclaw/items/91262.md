## Summary

Keep open: current main still runs build-all TypeScript node steps through native strip-types unchanged, while this PR adds a narrow fallback with focused regression coverage and sufficient real-host proof.

## What This Changes

The PR changes `scripts/build-all.mjs` to rewrite build-all `.ts` node steps from native strip-types to `node --import tsx` when the current Node binary reports no TypeScript support, and adds tests for that routing.

## Best Possible Solution

Land the narrow build-all fallback after maintainer review and normal merge gates; track direct package-script parity separately only if those scripts need the same Debian Node compatibility.

## Reproduction Assessment

Yes. Current main passes build-all `.ts` node steps through native strip-types unchanged, and the PR body supplies real Debian/Node 24.15.0 proof where `process.features.typescript` was false and `pnpm build` previously failed.

## Solution Assessment

Yes. `resolveBuildAllStep` owns build-all command selection, so the conditional fallback is the narrow maintainable fix for `pnpm build`; rewriting every direct package script would be broader than the reported failure.
