## Summary

Keep open. Current main has a bundled Xiaomi provider, but it still only covers model/auth catalog wiring and Xiaomi TTS; there is no Xiaomi-owned `webSearch` config or request wrapper that appends Xiaomi MiMo's native `web_search` tool. The reporter supplied concrete API behavior and the related items are for other providers or unrelated native-tool surfaces, so this remains valid narrow Xiaomi plugin work.

## What This Changes

The issue requests opt-in Xiaomi MiMo provider support for passing Xiaomi's native `web_search` tool in chat-completions requests.

## Best Possible Solution

Implement this inside the Xiaomi plugin: add validated `plugins.entries.xiaomi.config.webSearch` settings, gate native tool injection to supported MiMo chat models and official endpoint behavior, append Xiaomi's native tool object while preserving ordinary function tools and avoiding duplicates, then document activation, annotations, billing/privacy caveats, and token-plan limitations.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for an absence check: current main has no Xiaomi `webSearch` config schema, no Xiaomi native `web_search` payload hook, and no Xiaomi docs/tests for the feature; live API reproduction was not possible because no Xiaomi credential was available.

## Solution Assessment

Is this the best way to solve the issue? Yes. The narrow maintainable solution is Xiaomi-owned provider-plugin behavior through the existing request-wrapper seam; the reporter's proposed `models.providers.xiaomi.webSearch` shape should be adapted to the plugin-owned config path used elsewhere in OpenClaw.
