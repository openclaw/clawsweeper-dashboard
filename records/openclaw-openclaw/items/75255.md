## Summary

Keep open. Current main has the Xiaomi provider and TTS plugin, but it still does not expose Xiaomi MiMo native `web_search` config or inject Xiaomi's provider-native `web_search` tool into chat-completions payloads; the request is narrow, provider-owned, and not covered by the related native-search items in the supplied context.

## What This Changes

Add opt-in Xiaomi MiMo provider-native `web_search` tool support, including Xiaomi-scoped configuration and request payload injection for supported MiMo chat models.

## Best Possible Solution

Implement this inside the Xiaomi plugin: add validated plugin-scoped `webSearch` settings, gate the native tool to supported MiMo chat models and official endpoint/key behavior, append Xiaomi's `web_search` object while preserving ordinary function tools and avoiding duplicates, then document activation, billing/privacy caveats, annotations, and token-plan limitations.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for an absence check: current main has no Xiaomi `webSearch` config schema, no Xiaomi native `web_search` payload hook, and no Xiaomi docs/tests for the feature; live API reproduction is blocked only by the missing Xiaomi credential.

## Solution Assessment

Is this the best way to solve the issue? Yes. The narrow maintainable solution is Xiaomi-owned provider-plugin behavior using the existing provider request-wrapper seam; a broader core native-tool abstraction should wait until multiple providers need one.
