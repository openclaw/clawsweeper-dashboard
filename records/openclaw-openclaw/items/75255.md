## Summary

Keep open. Current main still lacks Xiaomi-owned native `web_search` config, request-payload injection, tests, and docs, while the issue discussion includes a concrete provider feature plus a reporter-tested official Xiaomi API example.

## What This Changes

Add opt-in Xiaomi MiMo native `web_search` support so OpenClaw can append Xiaomi's provider-native search tool to chat-completions requests.

## Best Possible Solution

Implement this inside the bundled Xiaomi plugin as an opt-in provider-native search wrapper, with validated plugin config, Xiaomi docs, web-search docs, focused tests, and a changelog entry while preserving managed `web_search` defaults.

## Reproduction Assessment

Yes. A repository absence check is high-confidence: current main has no Xiaomi `webSearch` schema, no Xiaomi request wrapper appending `{ type: "web_search" }`, and no Xiaomi native-search docs or tests.

## Solution Assessment

Yes. A Xiaomi-owned `wrapStreamFn`/payload-patch implementation following the OpenAI native-search pattern is the narrow maintainable path; a core hardcode or generic search-provider rewrite would cross ownership boundaries.
