## Summary

Keep open because current main still bundles Xiaomi as a model/TTS provider but has no Xiaomi-owned native `web_search` config, request-payload injection, tests, or docs, while the reporter and Xiaomi docs describe a concrete provider feature.

## What This Changes

The issue requests opt-in Xiaomi MiMo native `web_search` tool support in the bundled Xiaomi provider.

## Best Possible Solution

Implement this inside the bundled Xiaomi plugin with validated opt-in `plugins.entries.xiaomi.config.webSearch` settings, provider-owned request payload patching, docs/changelog coverage, and tests that preserve ordinary function tools while replacing the managed `web_search` function only when Xiaomi native search is enabled.

## Reproduction Assessment

Yes. A repository absence check is high-confidence: current main has no Xiaomi `webSearch` schema, no Xiaomi stream/request wrapper that appends `{type: "web_search"}`, and no Xiaomi native-search docs or tests; no Xiaomi/MiMo key was present in the environment/profile for a live API call.

## Solution Assessment

Yes. The narrow maintainable path is Xiaomi-owned plugin behavior through the existing provider hook/config surfaces, not a core hardcode or a separate ClawHub-only plugin, because OpenClaw already bundles and owns the Xiaomi provider plugin.
