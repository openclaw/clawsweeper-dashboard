## Summary

Keep open. Current main still has a bundled Xiaomi provider, but it only wires model/auth catalog behavior and Xiaomi TTS; there is no Xiaomi-owned native `web_search` config, request-body injection, tests, or docs. The reporter supplied concrete provider behavior and the related items are for other providers, so this is a valid narrow Xiaomi plugin feature.

## What This Changes

Add Xiaomi MiMo native `web_search` support so Xiaomi chat requests can opt into provider-level web search and return Xiaomi citation annotations.

## Best Possible Solution

Implement the feature inside the bundled Xiaomi plugin: add validated `plugins.entries.xiaomi.config.webSearch` settings, append Xiaomi's native `web_search` tool only when enabled and supported, preserve normal function tools without duplicates, surface citation/usage behavior as far as the transport supports it, and update Xiaomi/web-search docs plus changelog.

## Reproduction Assessment

Yes for a repository absence check: current main has no Xiaomi `webSearch` schema, no Xiaomi request wrapper that appends `{type: "web_search"}`, and no Xiaomi native-search docs or tests. Live API verification was not run because no `XIAOMI_API_KEY` was present in the environment check.

## Solution Assessment

Yes. The narrow maintainable solution is Xiaomi-owned plugin behavior through the existing provider hook/config schema surfaces, not a core hardcode or a separate ClawHub-only plugin, because OpenClaw already bundles and owns the Xiaomi provider plugin.
