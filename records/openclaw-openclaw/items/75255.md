## Summary

Keep open. Current main still has Xiaomi model and TTS support, but no Xiaomi plugin config, runtime payload hook, docs, or tests for MiMo provider-native `web_search`; the reporter also supplied concrete live API behavior rather than a vague request.

## What This Changes

Request support for Xiaomi MiMo's native `web_search` chat-completions tool by adding Xiaomi-scoped config and appending the provider-native tool when enabled.

## Best Possible Solution

Add this as Xiaomi-owned plugin behavior: declare Xiaomi `webSearch` settings, gate native tool injection to official Xiaomi MiMo chat-completions traffic and supported models, preserve ordinary function tools, document endpoint/key and billing/annotation behavior, and cover the payload path with focused Xiaomi tests.

## Reproduction Assessment

Not applicable as a runtime bug reproduction, but yes for a high-confidence absence check: current main has no Xiaomi `webSearch` config schema, no Xiaomi native `web_search` payload hook, and no Xiaomi web-search docs or tests.

## Solution Assessment

Yes. A provider-local Xiaomi plugin implementation using the existing `wrapStreamFn`/payload-patch seam is the narrowest maintainable path; a broader core abstraction should wait until multiple providers need the same new contract.
