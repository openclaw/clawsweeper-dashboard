## Summary

Keep open. Current main still lacks Xiaomi-owned native `web_search` support: the bundled Xiaomi plugin registers model/auth/catalog behavior plus TTS, but has no opt-in config, request-body wrapper, tests, or docs for Xiaomi's provider-native search tool. The reporter also added a concrete working API example, so this is a valid narrow provider-plugin feature rather than stale or duplicate cleanup.

## What This Changes

Add opt-in Xiaomi MiMo native `web_search` support that appends Xiaomi's provider-native tool to chat-completions requests and exposes related settings.

## Best Possible Solution

Implement this inside the bundled Xiaomi plugin: add a validated `plugins.entries.xiaomi.config.webSearch` surface, inject Xiaomi's native `{type: "web_search"}` tool only when enabled for supported Xiaomi models and official endpoints, preserve normal function tools without duplicates, document citation/usage behavior, and add focused unit coverage plus a changelog entry.

## Reproduction Assessment

Yes for the repository state: current main has no Xiaomi `webSearch` schema, no Xiaomi request wrapper that appends `{type: "web_search"}`, and no Xiaomi native-search docs/tests. Live API reproduction is not available in this checkout because the Xiaomi API key is absent.

## Solution Assessment

Yes. The narrow maintainable fix is Xiaomi-owned plugin behavior through the existing provider hook and manifest config surfaces, not a core hardcode or a separate ClawHub-only plugin, because OpenClaw already bundles and owns the Xiaomi provider plugin.
