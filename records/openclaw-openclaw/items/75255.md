## Summary

Keep open because current main still bundles Xiaomi as a model/TTS provider but does not expose Xiaomi native web search config, declare a web-search provider contract, or patch Xiaomi chat-completions payloads with the requested `web_search` tool.

## What This Changes

Add opt-in Xiaomi MiMo native `web_search` support so supported Xiaomi models can use provider-level search without an external OpenClaw web-search provider.

## Best Possible Solution

Implement this as Xiaomi-owned bundled plugin behavior with validated opt-in config, request-payload injection, docs, changelog, and focused tests rather than adding a core Xiaomi hardcode.

## Reproduction Assessment

Yes. A read-only source check is enough: current main has no Xiaomi `webSearch` schema, no Xiaomi web-search contract or runtime registration, and no Xiaomi payload wrapper that appends native `{ type: "web_search" }`.

## Solution Assessment

Yes. The narrow maintainable path is to use the existing provider hook surface from the Xiaomi plugin, preserving ordinary function tools and documenting the official-key requirement instead of changing generic core web-search behavior.
