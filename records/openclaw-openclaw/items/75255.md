## Repair Work Prompt

Implement Xiaomi MiMo native web_search support for this issue in the bundled Xiaomi plugin only. Add validated opt-in config under `plugins.entries.xiaomi.config.webSearch` with `enabled`, `forceSearch`, `maxKeyword` (1-50), `limit` (1-50), and optional approximate `userLocation` fields. Use the existing provider `wrapStreamFn`/payload-patch seam to affect only Xiaomi OpenAI-compatible chat-completions requests when enabled: preserve ordinary function tools, remove OpenClaw's managed `{ type: "function", name: "web_search" }` tool when replacing it, avoid duplicating an existing native `{ type: "web_search" }`, and append Xiaomi's native tool using snake_case fields (`force_search`, `max_keyword`, `limit`, `user_location`). Keep this provider-owned; do not add Xiaomi hardcodes to core or change other providers. Document that official Xiaomi API keys/base URL are required for native search and token-plan keys may reject it. Update Xiaomi/web-search docs and `CHANGELOG.md` because this is user-facing. Add focused tests for config schema/manifest acceptance and request-payload patch behavior, including preserving non-search tools and disabled config behavior.
