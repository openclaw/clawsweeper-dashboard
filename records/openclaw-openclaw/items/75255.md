## Repair Work Prompt

Implement opt-in Xiaomi MiMo native web_search support for the bundled Xiaomi plugin. Add validated `plugins.entries.xiaomi.config.webSearch` settings for at least `enabled`, `forceSearch`, `maxKeyword`, `limit`, and optional approximate `userLocation` fields; keep the feature disabled by default. In the Xiaomi provider runtime, use the existing provider `wrapStreamFn`/payload patch seam to append a Xiaomi-native `{ type: "web_search", ... }` tool when enabled, preserving existing non-web tools and avoiding duplicate native entries. Do not add Xiaomi-specific behavior to core. Account for the issue's credential/endpoint note: official `sk-...` API keys and `https://api.xiaomimimo.com/v1` support this, while token-plan credentials may reject it; document the limitation and keep failure behavior explicit. Add focused Xiaomi plugin tests for config parsing and payload injection, update Xiaomi/web-search docs, and add a user-facing changelog entry crediting the human reporter if appropriate. Do not change unrelated web-search providers or broad provider replay policy.
