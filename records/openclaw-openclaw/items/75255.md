## Summary

Keep open. Current main still bundles the Xiaomi provider, but it has no Xiaomi-owned native web_search config, request-body injection hook, docs, or tests; the reporter also added concrete tested API details, so this is a valid narrow provider-plugin feature rather than stale cleanup.

## What This Changes

Add opt-in Xiaomi MiMo native web_search support so Xiaomi chat-completions requests can append the provider-specific web_search tool with configurable parameters.

## Best Possible Solution

Implement this inside the bundled Xiaomi plugin: add validated plugins.entries.xiaomi.config.webSearch settings, append Xiaomi's native web_search tool only when explicitly enabled and appropriate for the official Xiaomi-compatible route/model, preserve existing function tools without duplicates, document citations/usage behavior and endpoint limitations, and add focused Xiaomi tests plus a changelog entry.

## Reproduction Assessment

Yes for repository behavior: current main has no Xiaomi webSearch schema, no Xiaomi request wrapper that appends {type: "web_search"}, and no Xiaomi native-search docs/tests. Live API reproduction is not available in this checkout because XIAOMI_API_KEY is unset.

## Solution Assessment

Yes, the best fix is provider-owned Xiaomi plugin behavior using the existing provider hook/config surfaces. A core hardcode or ClawHub-only plugin would be less maintainable because OpenClaw already bundles and owns the Xiaomi provider plugin.
