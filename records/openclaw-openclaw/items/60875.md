## Summary

Keep this PR open. Current main still treats manifest-load failures as registry errors without a typed failure reason, so the workspace missing-manifest diagnostic downgrade is not implemented; the proposed approach is narrow, but the branch needs mechanical rebase/adaptation and a required changelog entry before landing.

## What This Changes

The PR adds typed native and bundle manifest-load failure reasons, downgrades workspace-origin missing-manifest registry diagnostics to warnings, and adds registry tests for workspace versus global severity.

## Best Possible Solution

Land this PR or an equivalent rebased patch that carries typed failure reasons through native and bundle manifest failures, downgrades only workspace-origin not-found diagnostics, preserves errors for unsafe/parse/invalid and non-workspace missing manifests, adds the focused regression tests, and records the user-facing fix in the changelog.

## Reproduction Assessment

Yes. A `loadSingleCandidateRegistry` workspace candidate whose root lacks `openclaw.plugin.json` reaches the current unconditional registry error path, and discovery can create such candidates under `.openclaw/extensions` when an `index.*` file is present.

## Solution Assessment

Yes, conceptually. A typed failure reason is a maintainable fix because it avoids brittle error-string matching while preserving hard errors for unsafe, parse, invalid, and non-workspace missing-manifest failures.
