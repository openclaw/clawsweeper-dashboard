## Summary

Keep open. Current main still has the narrower source-level behavior this PR targets, and the patch direction is sound, but this external PR lacks after-fix real behavior proof and needs rebasing/adaptation onto the current manifest loader shape before merge.

## What This Changes

The PR adds typed native and bundle manifest-load failure reasons, downgrades workspace-origin missing-manifest diagnostics to warnings, and adds focused registry tests.

## Best Possible Solution

Land a rebased version that carries typed failure reasons through current native and bundle manifest loaders, downgrades only workspace-origin not-found diagnostics, keeps unsafe/parse/invalid and non-workspace failures as errors, and preserves focused regression coverage.

## Reproduction Assessment

Yes. From source inspection, a workspace-origin candidate under `.openclaw/extensions` can lack `openclaw.plugin.json`, current main emits the manifest-load failure as an error, and config validation turns that into a blocking issue.

## Solution Assessment

Yes, conceptually. A typed failure reason is the narrow maintainable fix because it avoids error-string matching while preserving hard errors for unsafe, parse, invalid, and non-workspace missing-manifest failures; the implementation just needs to be adapted to current main.
