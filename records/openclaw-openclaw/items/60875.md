## Summary

Keep open. Current main still turns workspace missing-manifest candidates into registry errors that feed config validation, while this PR is a narrow, useful fix that now needs rebase/adaptation plus the required changelog entry before maintainer merge review.

## What This Changes

The PR adds typed manifest-load failure reasons, downgrades workspace-origin missing-manifest diagnostics to warnings, and adds registry tests for workspace versus global severity.

## Best Possible Solution

Land a rebased version that carries typed failure reasons through native and bundle manifest failures, downgrades only workspace-origin not-found diagnostics, preserves hard errors for unsafe/parse/invalid and non-workspace missing manifests, covers the boundary with tests, and records the fix in the changelog.

## Reproduction Assessment

Yes, from source inspection. A workspace-origin candidate under `.openclaw/extensions` without `openclaw.plugin.json` reaches `loadPluginManifest`, returns an untyped missing-manifest failure, is emitted as a registry error, and then becomes a config validation issue.

## Solution Assessment

Yes, conceptually. A typed failure reason is the maintainable boundary because it avoids string matching while preserving errors for unsafe paths, parse/invalid manifests, and non-workspace missing manifests.
