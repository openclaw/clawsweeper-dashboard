## Summary

Keep open: the PR still addresses a real current-main plugin registry behavior, but it needs rebase/adaptation to the current manifest-loading code and real behavior proof before merge.

## What This Changes

The PR adds typed native and bundle manifest-load failure reasons, uses them to warn instead of error for workspace-origin missing manifests, and adds registry tests for the workspace versus non-workspace boundary.

## Best Possible Solution

Land a rebased version that carries typed failure reasons through the current native and bundle manifest loaders, downgrades only workspace-origin not-found diagnostics, preserves hard errors elsewhere, covers the boundary with tests, and includes real gateway/config proof.

## Reproduction Assessment

Yes, from source inspection. Current main can create workspace-origin candidates under `.openclaw/extensions`, missing manifests return untyped failures, the registry emits those failures as errors, and config validation turns error diagnostics into blocking issues.

## Solution Assessment

Yes, conceptually. A typed failure reason is the maintainable fix because it avoids matching human-readable error text while keeping unsafe, parse, invalid, and non-workspace missing-manifest failures as hard errors.
