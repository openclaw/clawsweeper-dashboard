## Summary

Keep open: current main still has the reported error-level path, and the PR is useful but not ready because it is dirty against main and lacks the required user-facing changelog entry.

## What This Changes

The PR adds typed native and bundle manifest-load failure reasons, downgrades workspace-origin missing-manifest diagnostics to warnings, and adds registry tests for workspace versus non-workspace origins.

## Best Possible Solution

Land a rebased version that carries typed failure reasons through all native and bundle manifest failure returns, downgrades only workspace-origin not-found diagnostics, preserves hard errors elsewhere, covers the boundary with tests, and records the fix in the changelog.

## Reproduction Assessment

Yes, from source inspection. A workspace-origin candidate under `.openclaw/extensions` can lack `openclaw.plugin.json`, `loadPluginManifest` returns an untyped missing-manifest failure, the registry emits it as an error, and config validation turns that error into a blocking issue.

## Solution Assessment

Yes, conceptually. A typed failure reason is the maintainable boundary because it avoids error-string matching while preserving hard errors for unsafe, parse, invalid, and non-workspace missing-manifest failures.
