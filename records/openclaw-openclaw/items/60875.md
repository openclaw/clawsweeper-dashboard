## Summary

Keep open: the narrower workspace-origin missing-manifest behavior is still source-reproducible on current main, and this PR addresses the prior review feedback with a typed failure reason, but it needs real behavior proof and a rebase before merge.

## What This Changes

The PR adds typed native and bundle manifest-load failure reasons and uses them to warn, not error, only for workspace-origin missing manifests, with focused registry tests.

## Best Possible Solution

Land a rebased version adapted to current manifest cache and fs-safe helpers that carries typed failure reasons through native and bundle manifest failures, downgrades only workspace-origin not-found diagnostics, and includes focused tests plus real behavior proof.

## Reproduction Assessment

Yes, from source inspection. A workspace-origin candidate under .openclaw/extensions can lack openclaw.plugin.json, current manifest loading returns an untyped missing-manifest failure, the registry emits it as an error, and config validation turns that into a blocking issue.

## Solution Assessment

Yes, conceptually. A typed failure reason is the narrow maintainable fix because it avoids coupling behavior to human-readable error text while preserving hard errors for unsafe, parse, invalid, and non-workspace missing-manifest failures.
