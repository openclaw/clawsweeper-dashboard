## What This Changes

The PR adds typed manifest-load failure reasons, uses them to warn only for workspace-origin missing manifests, and adds registry tests for workspace versus non-workspace missing-manifest diagnostics.

## Best Possible Solution

Land a rebased version that carries typed failure reasons through the current native and bundle manifest loaders, downgrades only workspace-origin missing manifests, preserves hard errors for unsafe/parse/invalid and non-workspace failures, and includes focused tests plus real behavior proof.

## Reproduction Assessment

Yes, from source inspection. A top-level `.js` file or directory with `index.js` under `<workspace>/.openclaw/extensions` is promoted as a workspace candidate, then missing `openclaw.plugin.json` becomes an error-level registry diagnostic and a config validation issue.

## Solution Assessment

Yes, conceptually. A typed failure reason is the narrow maintainable fix because it avoids coupling runtime behavior to human-readable error text while preserving strict errors for unsafe, parse, invalid, and non-workspace missing-manifest failures.
