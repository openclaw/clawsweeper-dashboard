## Summary

Keep open: current main still emits missing workspace plugin manifests as error diagnostics, while this PR’s typed failure-reason approach is the right narrow fix; it needs current-main adaptation and the required changelog entry before merge.

## What This Changes

The PR adds typed native and bundle manifest-load failure reasons, uses them to warn only for workspace-origin missing manifests, and adds registry tests for workspace versus non-workspace origins.

## Best Possible Solution

Land a current-main version that carries typed failure reasons through native and bundle manifest failures, downgrades only workspace-origin not-found diagnostics, keeps unsafe/parse/invalid and non-workspace missing manifests as errors, covers the boundary with tests, and records the user-facing fix in `CHANGELOG.md`.

## Reproduction Assessment

Yes, from source inspection. A workspace-origin candidate under `.openclaw/extensions` can lack `openclaw.plugin.json`, `loadPluginManifest` returns an untyped missing-manifest failure, the registry emits it as an error, and config validation turns that into a blocking issue.

## Solution Assessment

Yes, conceptually. A typed failure reason is the maintainable boundary because it avoids error-string matching while preserving hard errors for unsafe, parse, invalid, and non-workspace missing-manifest failures.
