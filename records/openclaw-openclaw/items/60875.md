## Summary

Keep open. Current main still emits all failed plugin manifest loads as errors and lacks typed failure reasons, so the user-visible workspace startup/config bug remains; the PR direction is right, but it needs the required changelog entry and a small rebase/adaptation to current main before merge.

## What This Changes

The PR adds typed manifest-load failure reasons for native and bundle manifests, downgrades workspace-origin missing-manifest registry diagnostics from error to warn, and adds registry tests for workspace versus global origins.

## Best Possible Solution

Land a rebased version that carries typed failure reasons through native and bundle manifest failures, warns only for workspace-origin missing manifests, preserves hard errors for unsafe/parse/invalid and non-workspace failures, covers the boundary with tests, and records the fix in `CHANGELOG.md`.

## Reproduction Assessment

Yes, from source inspection. A workspace-origin candidate under `.openclaw/extensions` without `openclaw.plugin.json` can reach manifest loading, return an untyped missing-manifest failure, be recorded as a registry error, and then become a config validation issue.

## Solution Assessment

Yes, conceptually. A typed failure reason is the narrow maintainable fix because it avoids string matching while preserving errors for unsafe paths, parse/invalid manifests, and non-workspace missing manifests.
