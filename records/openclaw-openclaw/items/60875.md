## Summary

Keep open. Current main still turns missing-manifest plugin registry failures into errors regardless of workspace origin, while this PR targets a real remaining narrow case with a reasonable typed failure-reason approach. The branch needs maintainer review and likely rebase/adaptation, not closure or a separate repair job.

## What This Changes

The PR adds typed native and bundle manifest-load failure reasons, uses them to downgrade only workspace-origin missing-manifest registry diagnostics to warnings, and adds focused registry tests for workspace versus global origins.

## Best Possible Solution

Land this PR after rebasing it onto current main, or land an equivalent narrow change that uses a stable manifest failure reason, downgrades only workspace-origin missing manifests, preserves hard errors for unsafe/parse/invalid and non-workspace missing-manifest failures, and keeps the focused regression tests.

## Reproduction Assessment

Yes. A workspace-origin candidate whose root lacks openclaw.plugin.json still flows through loadPluginManifest into an unconditional registry error on current main; loadSingleCandidateRegistry with origin "workspace" and an empty temp root is the focused reproduction path.

## Solution Assessment

Yes, conceptually. A typed failure reason is the maintainable fix because it avoids brittle error-string matching while preserving strict severity for unsafe, parse, invalid, and non-workspace failures; the branch still needs maintainer review and rebase.
