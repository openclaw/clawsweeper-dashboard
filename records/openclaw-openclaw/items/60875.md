## Summary

Keep open: the PR still addresses a real, source-reproducible plugin discovery/registry severity bug on current main, but it lacks external-contributor real behavior proof and needs rebasing onto the current manifest-loader code.

## What This Changes

The PR adds typed plugin manifest-load failure reasons and downgrades workspace-origin missing-manifest diagnostics from error to warning, with focused registry tests.

## Best Possible Solution

Land a rebased version that carries typed failure reasons through the current native and bundle manifest loaders, downgrades only workspace-origin not-found diagnostics, keeps unsafe/parse/invalid and non-workspace failures as errors, and preserves focused regression coverage.

## Reproduction Assessment

Yes, from source inspection in this read-only pass. A workspace-origin candidate inside `.openclaw/extensions` can lack `openclaw.plugin.json`, current manifest loading returns an untyped missing-manifest failure, the registry emits it as an error, and config validation turns that into a blocking issue.

## Solution Assessment

Yes, conceptually. A typed failure reason is the narrow maintainable fix because it avoids error-string matching while preserving hard errors for unsafe, parse, invalid, and non-workspace missing-manifest failures; the patch just needs current-main adaptation and real behavior proof.
