## Summary

Keep open: current main still lacks typed manifest-load failure reasons and still emits error-level diagnostics for workspace-origin missing manifests. The PR direction is narrow and useful, but it needs contributor real behavior proof and a current-main rebase before merge.

## What This Changes

The PR adds typed manifest-load failure reasons, downgrades only workspace-origin not-found manifest diagnostics to warnings, and adds focused manifest-registry tests.

## Best Possible Solution

Land a rebased version that carries typed failure reasons through the current native and bundle manifest loaders, downgrades only workspace-origin not-found diagnostics, preserves strict errors for unsafe/parse/invalid and non-workspace failures, and includes focused tests plus real gateway/config proof.

## Reproduction Assessment

Yes, from source inspection. Current main still promotes top-level files and index directories under .openclaw/extensions as workspace candidates, then an untyped missing-manifest failure becomes an error-level registry diagnostic and a config validation issue.

## Solution Assessment

Yes, conceptually. A typed failure reason is the narrow maintainable fix because it avoids coupling runtime behavior to human-readable error text while preserving hard errors for unsafe, parse, invalid, and non-workspace failures.
