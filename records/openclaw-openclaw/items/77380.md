## Summary

Keep this PR open: it targets a real OpenRouter regression, but the current diff changes the shared DeepSeek V4 wrapper and would also alter direct DeepSeek/OpenCode Go wire behavior that current tests and upstream DeepSeek docs still define as `xhigh -> max`. It also needs the required changelog entry before merge.

## What This Changes

The PR removes `max` from OpenRouter’s advertised DeepSeek V4 thinking levels and changes the shared DeepSeek V4 wire mapping plus OpenRouter tests from `max` to `xhigh`.

## Best Possible Solution

Keep the OpenRouter fix, but make the `xhigh` wire value an OpenRouter-specific mapping while preserving direct DeepSeek/OpenCode Go `xhigh -> max` behavior and documenting the user-facing fix in the changelog.

## Reproduction Assessment

Yes. Source inspection shows OpenRouter-routed DeepSeek V4 currently goes through the shared wrapper and can emit `reasoning_effort: "max"`; the linked issue provides the corresponding OpenRouter 400 response for that payload.

## Solution Assessment

No. The fix should be provider-specific, for example by adding an option to the shared wrapper or splitting the OpenRouter wrapper, so OpenRouter sends `xhigh` without changing direct DeepSeek/OpenCode Go semantics.
