## Summary

Keep open. The linked bug is real on current main, but this PR is still partial: aborts during Pi preflight before `Agent.activeRun` exists can still escape, contributor live proof is missing, and the required changelog entry is absent.

## What This Changes

The PR exports `makeAbortError`, replaces the embedded runner abort wrapper with Pi-agent cleanup, adds a pre-prompt guard/finally backstop, and adds abort-before-prompt regression tests.

## Best Possible Solution

Land an abort-aware prompt-start handoff that refuses to start Pi `agent.prompt()` when the run signal aborts at the final preflight boundary, keep regression coverage for pre-aborted, pre-activeRun, and activeRun cases, add the changelog entry, and include redacted live interrupt-mode proof.

## Reproduction Assessment

Yes. Current main is source-reproducible because `activeSession.prompt(...)` is evaluated before `abortable(...)` observes an already-aborted signal, and https://github.com/openclaw/openclaw/issues/74859 provides a deterministic pre-aborted-signal reproduction plus production log signatures.

## Solution Assessment

No. The pre-call guard and active-run cleanup are useful layers, but the PR still needs an abort-aware final Pi preflight-to-`agent.prompt()` boundary and coverage for that exact window.
