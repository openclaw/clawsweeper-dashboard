## Summary

Keep open. The PR addresses the deterministic pre-aborted case, but it still leaves the pre-activeRun race called out in review: Pi 0.71.1 performs async `AgentSession.prompt()` preflight before `agent.prompt()` creates `activeRun`, so aborts in that window can still reject the wrapper while the agent later starts uncancelled. The PR also lacks the required changelog entry for a user-impacting cost/runaway-LLM fix.

## What This Changes

The PR adds embedded runner abort guards and Pi Agent cleanup around prompt submission, extends the runner test harness, and adds a regression test for abort-before-prompt zombie Agent loops.

## Best Possible Solution

Make prompt submission abort-aware at the last boundary before Pi starts `agent.prompt()` while keeping the pre-call guard as an early fast path. The narrowest maintainable route is to use Pi's current `PromptOptions.preflightResult` hook, or an upstream AbortSignal API if added, then cover pre-aborted, pre-activeRun, and activeRun-existing abort cases and add the changelog credit.

## Reproduction Assessment

Yes. #74859 gives a deterministic pre-aborted-signal reproduction, and the remaining PR gap can be reproduced by delaying Pi `AgentSession.prompt()` preflight, aborting before it calls `Agent.prompt()`, and verifying no Agent run starts afterward.

## Solution Assessment

No. The pre-prompt guard is a useful first layer, but it is not the best complete fix because it does not observe aborts during Pi preflight; checking the signal immediately before `agent.prompt()` starts is safer and still narrow.
