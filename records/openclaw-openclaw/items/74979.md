## Summary

Keep open. The underlying bug is real on current main and this PR covers the pre-aborted and active-run abort windows, but it still misses the abort window inside pinned `AgentSession.prompt()` preflight before `Agent.activeRun` exists, and the required changelog entry is absent.

## What This Changes

This PR adds a pre-prompt abort guard, Pi agent abort/queue cleanup during abort handling and cleanup, test-harness hooks, and a new regression test for aborted embedded attempts.

## Best Possible Solution

Land a small revision that checks the abort signal both before calling `AgentSession.prompt()` and at the session preflight handoff before `Agent.prompt()`, keeps coverage for pre-aborted, pre-activeRun, and activeRun abort races, and records the user-facing fix in the changelog.

## Reproduction Assessment

Yes. The linked bug report gives a deterministic pre-aborted-signal reproduction, and current main confirms the eager `activeSession.prompt()` call path. The remaining PR gap can be reproduced by delaying Pi session preflight and aborting before `this.agent.prompt(messages)` creates `activeRun`.

## Solution Assessment

No. The pre-prompt guard and activeRun cleanup are useful layers, but the patch is not the best complete fix until the Pi session preflight-to-agent boundary is abort-aware, for example by using the existing `preflightResult` hook to throw before `Agent.prompt()` when the run signal is aborted.
