## Summary

Keep open. The PR adds a useful pre-aborted guard, but it still leaves the abort-during-`AgentSession.prompt()` preflight window before `Agent.activeRun` exists, and it is missing the required changelog entry for a user-impacting fix.

## What This Changes

This PR adds a pre-prompt abort guard, Pi agent abort/queue cleanup during abort handling and cleanup, test-harness methods, and a new abort-before-prompt regression test for the embedded attempt runner.

## Best Possible Solution

Land a boundary-aware fix that checks the run abort signal at the actual `AgentSession.prompt()` preflight-to-`Agent.prompt()` handoff, keeps coverage for pre-aborted, pre-activeRun, and activeRun-existing abort cases, and records the user-facing fix in the changelog.

## Reproduction Assessment

Yes. #74859 gives a deterministic pre-aborted-signal reproduction, and the remaining gap can be reproduced by aborting after OpenClaw’s new guard but before pinned Pi `AgentSession.prompt()` reaches `Agent.prompt()`.

## Solution Assessment

No. The pre-prompt guard is the right first layer, but the PR is still a partial fix because `agent.abort()` cannot cancel anything before `activeRun` exists; the safer fix is prompt-start-boundary cancellation plus the cleanup backstop.
