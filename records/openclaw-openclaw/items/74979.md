## Summary

Keep open: the PR is useful and addresses the deterministic pre-aborted case, but it is still a partial fix because aborts during Pi `AgentSession.prompt()` preflight before `activeRun` exists can still start a loop after attempt cleanup; it also lacks the required changelog entry and contributor real behavior proof.

## What This Changes

The PR exports the embedded-runner abort error helper, adds a local abortable wrapper with Pi agent abort/queue cleanup plus a pre-prompt guard/finally backstop in `runEmbeddedAttempt`, extends the test harness, and adds an abort-before-prompt regression test.

## Best Possible Solution

Land a boundary-aware fix that checks the run abort signal at the final Pi prompt-start handoff, keeps coverage for pre-aborted, pre-activeRun, and activeRun-existing aborts, adds the changelog entry, and includes redacted live interrupt-mode proof.

## Reproduction Assessment

Yes, from source inspection and the linked report rather than an executed local test. Current `main` still creates the `activeSession.prompt()` promise before `abortable()` can reject, and https://github.com/openclaw/openclaw/issues/74859 provides deterministic reproduction code plus production log signatures.

## Solution Assessment

No. The pre-call guard is a good first layer, but the PR still relies on `Agent.abort()` before Pi has necessarily created `activeRun`; the safer fix is to make the actual preflight-to-`agent.prompt()` boundary abort-aware, then retain cleanup as a backstop.

- **[P2] Make the prompt-start boundary abort-aware:** `src/agents/pi-embedded-runner/run/attempt.ts:2359-2360`
