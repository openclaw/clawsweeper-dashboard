## Summary

Keep open. The PR addresses the deterministic pre-aborted signal case, but it still leaves the abort-during-`AgentSession.prompt()` preflight window open and is missing the required changelog entry for a user-facing cost/runaway fix.

## What This Changes

The PR adds an embedded-attempt pre-prompt abort guard, Pi agent abort/queue cleanup backstops, test-harness hooks, and abort-before-prompt regression coverage.

## Best Possible Solution

Land a boundary-aware fix that prevents `AgentSession.prompt()` from reaching `Agent.prompt()` after the run aborts, preserves the cleanup backstop, covers pre-aborted, pre-activeRun, and activeRun-existing abort cases, and records the fix in `CHANGELOG.md`.

## Reproduction Assessment

Yes. The linked bug provides a deterministic pre-aborted-signal reproducer, and the remaining PR gap can be reproduced by making `AgentSession.prompt()` await before calling `agent.prompt()`, aborting during that await, and checking that no agent loop starts after the attempt exits.

## Solution Assessment

No. The pre-prompt guard is a useful first layer, but relying on `agent.abort()` is incomplete before Pi creates `activeRun`; the narrower maintainable fix is to make the prompt-start boundary itself abort-aware and add the missing changelog entry.
