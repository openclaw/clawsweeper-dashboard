## Summary

Keep this PR open: it targets a real source-reproducible runaway Agent-loop bug, but the current patch still leaves the Pi preflight-before-activeRun abort window unclosed, lacks the required changelog entry, and only supplies mock/unit-test behavior proof.

## What This Changes

The PR exports `makeAbortError`, replaces the local abort wrapper with Pi Agent cleanup, adds a pre-prompt guard and cleanup backstop, extends the test harness, and adds abort-before-prompt regression tests.

## Best Possible Solution

Land a complete abort-aware prompt handoff that prevents Pi `agent.prompt()` from starting after run abort across all preflight windows, keep regression coverage, add the changelog entry, and require redacted live interrupt-mode proof.

## Reproduction Assessment

Yes. Current main is source-reproducible because `activeSession.prompt(...)` is evaluated before `abortable(...)` observes an already-aborted signal, and the linked issue at https://github.com/openclaw/openclaw/issues/74859 provides a deterministic pre-aborted-signal reproduction plus production log signatures.

## Solution Assessment

No. The direction is right, but the patch still needs an abort-aware final Pi preflight-to-`agent.prompt()` handoff and coverage for aborts before `Agent.activeRun` exists.
