## Summary

Keep open: the underlying abort bug is real and still source-reproducible on current main, but this PR is not ready because it leaves the acknowledged Pi preflight window before `Agent.activeRun` exists and the supplied proof is mock/unit-test only.

## What This Changes

The PR exports `makeAbortError`, adds a pre-prompt abort guard plus Agent cleanup in the embedded runner, extends the test session harness, and adds abort-before-prompt regression tests.

## Best Possible Solution

Keep this PR and the linked bug open until the prompt boundary on current main is abort-aware before Pi can start `agent.prompt()`, with focused regression coverage and real interrupt-mode proof.

## Reproduction Assessment

Yes. Current main still creates `activeSession.prompt(...)` before `abortableWithSignal` can observe a pre-aborted signal, and the linked issue supplies a deterministic test plus production log signature; I did not execute it because this review is read-only.

## Solution Assessment

No. The direction is useful, but calling `Agent.abort()` from the wrapper is not sufficient before Pi creates `activeRun`; the safer fix needs an abort-aware final handoff before `agent.prompt()` starts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

- **[P2] Make the Pi preflight handoff abort-aware:** `src/agents/pi-embedded-runner/run/attempt.ts:2359-2360`
