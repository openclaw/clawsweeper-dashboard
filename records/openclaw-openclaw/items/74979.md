## Summary

Keep open. The PR usefully targets the reported runaway Agent loop, but the current patch still leaves the abort window between `AgentSession.prompt()` preflight and the underlying `Agent.prompt()` start, and it is missing the required changelog entry for a user-impacting fix.

## What This Changes

The PR adds an embedded-runner pre-prompt abort guard, Agent abort/queue cleanup, test-harness methods, and a new regression test file for abort-before-prompt behavior.

## Best Possible Solution

Revise this PR or replace it with a narrow current-main patch that checks the run abort signal at the actual `Agent.prompt()` boundary, preserves the extracted `abortable` memory-leak fix, adds regression coverage for pre-aborted, pre-activeRun, activeRun-existing, and normal completion paths, and records the user-facing fix in `CHANGELOG.md`.

## Reproduction Assessment

Yes. The linked bug provides a deterministic pre-aborted `AbortSignal` test shape, and the remaining gap can be reproduced by delaying `AgentSession.prompt()` preflight, aborting after the outer guard passes, and verifying no underlying `Agent.prompt()` starts afterward.

## Solution Assessment

No. The pre-prompt guard is necessary but partial because `AgentSession.prompt()` has awaited preflight before `Agent.prompt()`; the safer solution is to make that final prompt-start boundary abort-aware without changing dependency versions.
