## Summary

The Gateway API and management UI already support permanent card deletion, including in v2026.6.5, but the agent-facing Workboard tool catalog still has no delete/remove tool on current main. The issue remains useful as a narrow tool-parity feature requiring an explicit destructive-action policy.

## What This Changes

Requests a Workboard card delete/remove operation in the agent tool API and optionally the management UI, with soft-delete or hard-delete retention semantics.

## Best Possible Solution

Add a narrowly scoped plugin-owned agent deletion tool that reuses `WorkboardStore.delete`, defines safe behavior for claimed cards, clearly documents permanent deletion, and preserves archive/unarchive as the reversible retention workflow.

## Reproduction Assessment

Not applicable as a bug reproduction: source inspection confirms this is an absent agent-tool capability, while the existing Gateway and UI deletion paths work by design.

## Solution Assessment

Unclear pending product approval: reusing the existing store deletion path is the narrowest implementation, but unrestricted agent deletion is not necessarily the safest policy.
