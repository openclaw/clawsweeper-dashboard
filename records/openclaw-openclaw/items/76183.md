## Summary

Keep this PR open but do not merge the current head: the Active Memory fast-fail path is useful, but the terminal zero-hit predicate can discard usable memory results when `debug.hits` is zero and `details.results` is non-empty.

## What This Changes

This PR adds Active Memory transcript polling to fast-fail empty or unavailable recall paths, filters embedded timeout boilerplate, adds focused tests, and updates the changelog.

## Best Possible Solution

Keep the fast-fail design, but only classify zero-hit search as terminal empty when the final returned `details.results` array is actually empty, with #75860 left as separate QMD recall-quality work.

## Reproduction Assessment

Yes. A focused regression can write a `memory_search` tool result with non-empty `details.results` and `debug.hits: 0`, then verify Active Memory waits for the subagent summary instead of resolving empty.

## Solution Assessment

No. The overall PR direction is narrow and maintainable, but the current predicate is not the best implementation because it ignores the final returned results array when `debug.hits` is zero.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Do not drop non-empty search results:** `extensions/active-memory/index.ts:1601`
