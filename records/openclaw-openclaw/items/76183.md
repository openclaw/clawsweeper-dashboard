## Summary

Keep this PR open. Current main does not already implement the terminal Active Memory fast-fail path, but the proposed branch still has a P2 correctness blocker and the discussion explicitly paused automation for human review.

## What This Changes

The PR adds Active Memory transcript polling for terminal memory search/recall results, filters timeout boilerplate, adds focused regression tests, and records a changelog fix entry.

## Best Possible Solution

Keep the terminal fast-fail design, but require the returned results array to be empty before treating debug.hits === 0 as a terminal empty recall.

## Reproduction Assessment

Yes. A focused regression can write a memory_search toolResult with details.results containing a supplement/wiki hit and debug.hits set to 0; the PR watcher would currently resolve empty before the recall subagent can use the result.

## Solution Assessment

No. The overall approach is narrow, but the zero-hit predicate is not the best implementation because it trusts debug.hits even when the returned result set is non-empty.
