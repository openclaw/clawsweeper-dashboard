The PR changes `ui/src/styles/chat/grouped.css` to give `.chat-bubble-actions` padding, rounded background, shadow, blur, and a light-theme border.

## Best Possible Solution

Revise the chip styling so the full action chip border box has reserved space across desktop/mobile and light/dark themes, then attach real Control UI proof showing wrapped assistant text remains clear.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? I did not run a live browser hover session.

## Solution Assessment

Is this the best way to solve the issue? No, not as currently patched: the visual chip can be reasonable, but it needs matching layout reservation or positioning plus real UI proof before merge.
