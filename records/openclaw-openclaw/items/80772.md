## Summary

Keep open: the CSS chip styling is plausible, but this branch is not merge-ready because it does not include the `has-copy` padding restore it says it builds on, and the external PR still lacks after-fix real behavior proof.

## What This Changes

This PR adds padding, radius, a card-like backdrop, shadow, blur, and a light-mode border to `.chat-bubble-actions` in Control UI chat CSS.

## Best Possible Solution

Land one coherent Control UI chat fix that restores the action-padding class and then styles the action row as a chip, with browser proof for short and wrapped assistant messages in light and dark themes.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes from source inspection: current main can render `.chat-bubble-actions` without adding the `has-copy` class that activates the existing padding rule; I did not run a live browser hover session in this read-only review.

## Solution Assessment

Is this the best way to solve the issue? No, not as a standalone branch: the chip styling should be stacked with or include the `has-copy` restoration from https://github.com/openclaw/openclaw/pull/80767 before it is treated as the complete UI fix.
