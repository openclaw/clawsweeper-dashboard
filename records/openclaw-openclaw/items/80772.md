## Summary

Keep open: the CSS chip is plausible, but the PR is not safe as a standalone fix because current main still renders assistant action rows without the `has-copy` padding class, and the external PR still lacks after-fix real behavior proof.

## What This Changes

This PR adds a backdrop, padding, radius, shadow, blur, and light-theme border to `.chat-bubble-actions` in Control UI chat CSS.

## Best Possible Solution

Land one coherent Control UI chat fix that restores `has-copy` when bubble actions render and styles the action row as a chip, with browser proof for wrapped assistant messages in light and dark themes.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes from source inspection: current main can render `.chat-bubble-actions` for assistant text while the parent bubble never receives `has-copy`, so the existing action-padding CSS is inactive; I did not run a live browser hover session in this read-only review.

## Solution Assessment

Is this the best way to solve the issue? No, not as a standalone branch: the chip styling should be stacked with or include the `has-copy` restoration from https://github.com/openclaw/openclaw/pull/80767 before it is treated as the complete UI fix.
