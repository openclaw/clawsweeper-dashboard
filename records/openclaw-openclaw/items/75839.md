## Summary

Keep open. Current main has partial `sessions.list` and trajectory-write mitigations, but the default list path still synchronously loads and merges session stores before row selection, `pi-trajectory-flush` still inherits the 10000 ms cleanup timeout, and open PR #77133 uses closing syntax for this issue so it should not be closed before that work lands or is replaced.

## What This Changes

The issue asks for `sessions.list` to remain responsive under moderate session load and for `pi-trajectory-flush` cleanup timeouts to become configurable, adaptive, or otherwise non-disruptive.

## Best Possible Solution
