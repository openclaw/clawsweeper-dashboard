## Best Possible Solution

## Repair Work Prompt

Fix openclaw/openclaw#75839 with a focused core PR. Current main handles `sessions.list {}` by loading and merging all agent session stores before responding, and cached stores are still deep-cloned before the list path can use them. Add a narrow read-only fast path or equivalent improvement that reduces full-store clone/merge overhead for dashboard-style `sessions.list {}` calls while preserving mutation isolation for write paths and non-list callers. Also make the `pi-trajectory-flush` cleanup timeout bounded and configurable, preferably through a documented environment variable or config seam, without changing unrelated cleanup steps unless the same helper cleanly supports it. Add regression coverage for cache/no-clone behavior or large-list behavior and for the configurable cleanup timeout. Update user-facing docs and `CHANGELOG.md` if runtime behavior or configuration changes. Do not redesign the session backend, introduce SQLite, or remove existing multi-agent/disk-only session visibility semantics in this PR.
