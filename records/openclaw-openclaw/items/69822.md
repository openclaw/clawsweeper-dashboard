## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve slow-client teardown behavior:** `src/gateway/server-methods/session-change-event.ts:154`
  - body: This changes teardown `sessions.changed` from always using `dropIfSlow: true` to `false` for reset/delete/new. In the existing broadcaster, false or omitted `dropIfSlow` closes clients over the slow-consumer threshold, so existing long-lived websocket clients can be disconnected during ordinary session teardown instead of dropping one update. Please preserve the old `sessions.changed` policy unless maintainers explicitly approve a close-on-slow protocol change.
  - confidence: 0.9
- **[P1] Include selected global agent identity in drains:** `src/gateway/server-methods/session-change-event.ts:62-64`
  - body: The drain is routed to per-agent global subscribers, but the payload only says `sessionKey: "global"`. Adjacent selected-global session events carry `agentId`, and clients tracking more than one global agent cannot reliably evict the right row without it. Include `agentId` for selected-global drains and cover that shape.
  - confidence: 0.88
- **[P2] Document the advertised drain event contract:** `src/gateway/server-methods-list.ts:47`
  - body: Adding `socket.drain` to `GATEWAY_EVENTS` makes it discoverable by websocket clients, but the PR does not define its payload fields, reason values, or slow-consumer semantics in the gateway protocol docs. Add the protocol docs or keep this event internal until the contract is approved.
  - confidence: 0.86
