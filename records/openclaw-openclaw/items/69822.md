## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Preserve slow-client teardown behavior:** `src/gateway/server-methods/session-change-event.ts:59-67`
  - body: This new drain send omits delivery options, and the later teardown `sessions.changed` send uses `{ dropIfSlow: false }`. `createGatewayBroadcaster` treats false or omitted `dropIfSlow` as close-on-slow, so reset/delete/new can now disconnect existing long-lived gateway clients under backpressure instead of preserving the current drop-on-slow behavior. Keep the existing compatibility behavior by default, or make the close-on-slow teardown contract explicit and maintainer-approved before merge.
  - confidence: 0.93
- **[P1] Include agent identity in global drain events:** `src/gateway/server-methods/session-change-event.ts:62-64`
  - body: Selected global resets/deletes are routed through an agent-scoped subscription key such as `agent:work:global`, but the emitted `socket.drain` payload only says `sessionKey: "global"`. A client tracking multiple global-agent rows cannot tell whether to evict main, work, or another agent's global row. Include `agentId` whenever the source payload has one and cover that in the drain tests.
  - confidence: 0.95
- **[P2] Document the advertised drain event:** `src/gateway/server-methods-list.ts:47`
  - body: Adding `socket.drain` to `GATEWAY_EVENTS` makes it part of the discoverable gateway event surface, but the protocol docs still list only `session.message`, `session.operation`, `session.tool`, and `sessions.changed` for session events. Before advertising the event, document the payload shape, allowed reasons, selected-global identity rule, and slow-consumer delivery semantics so SDK and websocket clients do not implement incompatible guesses.
  - confidence: 0.87
