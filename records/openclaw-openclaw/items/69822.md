## Summary

Keep this PR open, but it still needs changes before merge: the core socket.drain idea is useful and has real proof, but the current branch introduces duplicate/mis-scoped reset delivery and advertises a new gateway event without the public protocol contract.

## What This Changes

The PR adds socket.drain teardown broadcasts for gateway session message subscribers, wires message-subscriber lookup into gateway request context, changes teardown slow-consumer handling, and extends gateway session event tests.

## Best Possible Solution

Land a refreshed gateway patch that emits one correctly scoped socket.drain per teardown, documents the new event contract, and either preserves existing slow-consumer compatibility or records explicit maintainer acceptance for closing slow clients.

## Reproduction Assessment

Yes, source inspection is enough for the review findings: PR head emits reset teardown once without agentId and again later with agentId, and current main lacks socket.drain delivery for message-only subscribers. I did not run a live gateway repro in this read-only review.

## Solution Assessment

No, this is not yet the best fix. The right shape is the shared session-change emitter with one selected-agent-aware teardown broadcast, matching docs, and an explicit compatibility decision for slow WebSocket consumers.
