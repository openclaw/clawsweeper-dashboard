## Summary

Keep open: the runtime direction fits the source-level gap, but the external PR is not merge-ready because proof is still test-only and the new advertised Gateway event lacks docs/changelog coverage.

## What This Changes

The PR adds read-scoped `socket.drain` teardown broadcasts for reset/delete/new session paths, routes them to session-event and per-session message subscribers through gateway context, refactors agent reset emission, and adds gateway tests.

## Best Possible Solution

Land this Gateway protocol change, or a maintainer-approved equivalent, after docs/changelog and real client proof show message-only subscribers receive read-scoped `socket.drain` on reset/delete.

## Reproduction Assessment

Yes, at source level. Current main keeps message subscribers in a separate registry while `emitSessionsChanged` only targets session-event subscribers, and `socket.drain` is absent from the event guard/discovery lists.

## Solution Assessment

Yes for the runtime direction, but not yet for merge readiness. Unioning event and message subscribers in `emitSessionsChanged` follows the existing `session.message` broadcast precedent; docs/changelog and contributor-supplied real behavior proof remain required.
