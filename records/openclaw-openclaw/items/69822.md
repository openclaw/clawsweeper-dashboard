## Summary

Keep this PR open. Current main still lacks the `socket.drain` routing, scope guard, and message-subscriber access that the PR adds; the remaining blockers are merge readiness rather than cleanup closure.

## What This Changes

The PR adds a read-scoped Gateway `socket.drain` event for session reset/delete/new, routes it to session-event and per-session message subscribers, exposes the message subscriber lookup in request context, refactors the shared session-change emitter, and extends gateway tests.

## Best Possible Solution

Land a maintainer-approved Gateway protocol change after documenting `socket.drain` and receiving redacted live WebSocket or Gateway log proof that reset/delete/new deliver the event in a real run.

## Reproduction Assessment

Yes, at source level. Current main routes session mutation broadcasts only to `sessions.subscribe` connections and rejects unregistered non-plugin events, while per-session message subscribers are tracked separately.

## Solution Assessment

Yes for the runtime direction, but not yet for merge readiness. Unioning event and message subscribers in `emitSessionsChanged` follows the existing targeted transcript-event pattern; protocol docs and contributor-supplied live proof remain missing.
