- **[P1] Preserve slow-client teardown behavior:** `src/gateway/server-methods/session-change-event.ts:154`
  - body: Changing teardown `sessions.changed` broadcasts to `dropIfSlow: false` means the gateway closes slow websocket clients during reset/delete/new instead of preserving the current drop-on-slow behavior. `broadcastToConnIds` treats false or absent `dropIfSlow` as close-on-slow, so this can intentionally disconnect existing long-lived clients during ordinary session maintenance and needs either the compatible default or an explicit maintainer-approved protocol contract.
  - confidence: 0.92
- **[P1] Include the selected global agent in drain payloads:** `src/gateway/server-methods/session-change-event.ts:62-64`
  - body: For selected-global sessions, the PR routes `socket.drain` to an agent-scoped subscription key but sends only `sessionKey: "global"`. A client subscribed to more than one global agent on the same websocket cannot tell which global session was reset or deleted, while the sibling `sessions.changed` and `session.message` paths preserve `agentId`; include the selected agent identity in the drain payload or keep using the richer existing event shape.
  - confidence: 0.9
- **[P2] Document the advertised drain event contract:** `src/gateway/server-methods-list.ts:47`
  - body: Adding `socket.drain` to `GATEWAY_EVENTS` makes it discoverable as a gateway protocol event, but the PR does not document its payload, reason values, selected-global identity rules, or slow-consumer behavior in the gateway protocol docs or SDK contract. External clients should not have to infer a new advertised event from tests and implementation details.
