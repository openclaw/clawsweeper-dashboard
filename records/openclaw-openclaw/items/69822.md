- **[P1] Preserve slow-client teardown behavior:** `src/gateway/server-methods/session-change-event.ts:154`
  - body: Current main sends `sessions.changed` with `dropIfSlow: true`, which skips frames for saturated clients.   - confidence: 0.91

  - body: The new subscriber lookup uses `agentId` to target selected global sessions, but the emitted `socket.drain` payload only contains `sessionKey: "global"`, `reason`, and `ts`. Clients tracking multiple global agents cannot tell which row to evict, unlike the sibling `session.message` and `sessions.changed` payloads that include `agentId`.
  - confidence: 0.89
- **[P2] Document the advertised drain event contract:** `src/gateway/server-methods-list.ts:47`
  - body: Adding `socket.drain` to `GATEWAY_EVENTS` makes it discoverable in the Gateway hello features, but the protocol docs do not define its payload, reason set, scoped-global semantics, or delivery guarantees. Please document that contract before exposing it to client integrations.
