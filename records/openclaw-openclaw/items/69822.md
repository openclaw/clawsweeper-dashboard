## Summary

Keep open. The runtime direction looks plausible and CI is green, but this external PR still has test-only behavior proof and it adds a discoverable Gateway event without a documented protocol contract.

## What This Changes

The PR adds read-scoped Gateway `socket.drain` broadcasts for session reset/delete/new flows, routes them to both session-event and per-session message subscribers, and adds focused Gateway tests.

## Best Possible Solution

Land a maintainer-approved Gateway protocol change that documents `socket.drain` and keeps the runtime routing centered in session mutation handling after live redacted WebSocket proof is supplied.

## Reproduction Assessment

Yes, source-level only. Current main has a clear gap: session mutation broadcasts only target all-session subscribers, while per-session message subscribers are only unioned for transcript updates and `socket.drain` is not registered as an allowed event.

## Solution Assessment

Yes for the runtime direction, but not merge-ready as-is. The narrow maintainable path is to pair the subscriber-union implementation with protocol/docs updates and redacted live Gateway proof before merge.
