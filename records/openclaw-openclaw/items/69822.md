## Summary

## What This Changes

The PR adds Gateway `socket.drain` teardown broadcasts for session reset/delete/new, exposes message-subscriber connection lookup to request handlers, registers the event, adjusts teardown slow-consumer handling, and expands session-message Gateway tests.

## Best Possible Solution

Land a documented Gateway teardown event that reaches message subscribers while preserving the existing `sessions.changed` compatibility path unless maintainers explicitly accept and document close-on-slow teardown delivery.

## Reproduction Assessment

Yes at source level: current main only sends lifecycle `sessions.changed` to event subscribers, while message-only subscribers are only included for transcript updates. I did not run a live WebSocket reproduction in this read-only review.

## Solution Assessment

No, not as written. The message-subscriber drain route is plausible, but the existing `sessions.changed` slow-consumer contract should be preserved or explicitly accepted and documented by maintainers.
