Full review comments:

- **[P1] Include selected-agent identity in drain payloads:** `src/gateway/server-methods/sessions.ts:321-323`
  - body: `sessions.reset` and `sessions.delete` pass `agentId` for selected-global sessions, and `sessions.changed` preserves it, but the new `socket.drain` payload only sends `sessionKey`, `reason`, and `ts`. A `sessions.subscribe` consumer receiving `sessionKey: "global"` cannot distinguish the `work` global session from the default global session and can evict the wrong UI/cache row; include `agentId` when present and cover this path.
  - confidence: 0.9
- **[P1] Avoid unscoped duplicate drains for bare resets:** `src/gateway/server-methods/agent.ts:1460-1463`
  - body: The bare `/new`/`/reset` path now emits `emitSessionsChanged` once immediately after `runSessionResetFromAgent`, then emits again in the no-post-message ack branch. For selected-global resets the first call also omits `agentId`, so it can target the default global message subscription before the scoped event is sent; emit exactly once with the resolved agent identity.
  - confidence: 0.88
- **[P1] Keep session-change broadcasts drop-safe by default:** `src/gateway/server-methods/sessions.ts:412`
  - body: Changing `sessions.changed` to `{ dropIfSlow: !isTeardown }` means reset/delete/new broadcasts close slow `sessions.subscribe` sockets because the broadcaster closes when `dropIfSlow` is false. That changes an existing subscriber contract from dropping one over-buffered update to disconnecting the client during session maintenance; keep `sessions.changed` at `dropIfSlow: true` unless maintainers explicitly approve and document the compatibility break.
  - confidence: 0.86
- **[P2] Document the new drain event before advertising it:** `src/gateway/server-methods-list.ts:47`
  - body: Adding `socket.drain` to `GATEWAY_EVENTS` exposes it in hello `features.events`, but the Gateway protocol docs only describe the subscribe methods and do not define this event's payload, reason values, selected-agent behavior, or slow-client delivery semantics. Add the protocol docs/schema coverage, or keep the event internal until the contract is defined.
  - confidence: 0.81
