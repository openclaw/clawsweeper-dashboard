## Summary

Keep open: current main has the Gateway subscriber gap this PR targets, but the patch is not merge-ready because it changes the existing slow-consumer behavior for `sessions.changed`, advertises an undocumented event, and still lacks direct real WebSocket receipt proof.

## What This Changes

The PR adds Gateway `socket.drain` teardown broadcasts for reset/delete/new, exposes message-subscriber connection lookup in `GatewayRequestContext`, registers the event, adjusts teardown slow-consumer handling, and expands session-message Gateway tests.

## Best Possible Solution

Land a documented Gateway teardown event that reaches message subscribers while preserving the existing `sessions.changed` compatibility path unless maintainers explicitly accept and document close-on-slow teardown delivery.

## Reproduction Assessment

Yes at source level: current main sends lifecycle `sessions.changed` only to session-event subscribers, while transcript updates union message subscribers only for `session.message`. I did not run a live WebSocket reproduction in this read-only review.

## Solution Assessment

No, not as written. The message-subscriber drain route is plausible, but the existing `sessions.changed` slow-consumer contract should be preserved or explicitly accepted and documented by maintainers.
