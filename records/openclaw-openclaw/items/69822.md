## What This Changes

The branch adds Gateway `socket.drain` teardown broadcasts for session reset/delete/new flows, routes them to event and per-session message subscribers, exposes message-subscriber lookup in request context, registers the event, and adds focused Gateway tests.

## Best Possible Solution

Land a maintainer-approved Gateway protocol update that documents `socket.drain`, keeps drain routing centered in session mutation handling, and includes redacted live WebSocket proof.

## Reproduction Assessment

Yes, at source level: current main has no `socket.drain` registration or teardown broadcast path for message-only subscribers, while transcript routing already unions message subscribers only for `session.message`. I did not run a live WebSocket repro in this read-only review.

## Solution Assessment

Yes for the runtime direction, but not merge-ready as-is. The narrow maintainable path is to pair the subscriber-union implementation with protocol docs and redacted live Gateway/WebSocket proof.
