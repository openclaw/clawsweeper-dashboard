## Summary

## What This Changes

## Best Possible Solution

Land this as a compatibility-preserving protocol change: document `socket.drain`, route it to message-only subscribers, and keep existing event-subscriber slow-reader behavior unless maintainers explicitly accept the disconnect policy.
