## Review Findings

Full review comments:

- **[P1] Preserve slow-client teardown behavior:** `src/gateway/server-methods/session-change-event.ts:154`
  - body: This changes reset/delete/new `sessions.changed` broadcasts from `{ dropIfSlow: true }` to `{ dropIfSlow: false }`. In the shared broadcaster, `false` does not force a reliable send; it closes any client whose outbound buffer is already over the limit, so an existing long-lived gateway client can be disconnected during teardown instead of merely missing that frame. Please preserve the existing drop-on-slow behavior unless maintainers explicitly approve and document close-on-slow teardown semantics.
  - confidence: 0.9
- **[P1] Include the global agent in drain payloads:** `src/gateway/server-methods/session-change-event.ts:61-65`
  - body: For selected-global sessions the routing key can be `agent:<id>:global`, and `sessions.changed` carries `agentId`, but the new `socket.drain` frame only sends `sessionKey: "global"`. A WebSocket client subscribed to more than one selected-global session cannot tell which agent's row to evict. Please include `agentId` or send an unambiguous scoped key in the drain payload.
  - confidence: 0.86
- **[P2] Document the advertised drain event contract:** `src/gateway/server-methods-list.ts:47`
  - body: Adding `socket.drain` to `GATEWAY_EVENTS` exposes it in gateway discovery, but the protocol docs and SDK event normalizer still define only the existing session event families. Downstream clients need the payload shape, reason set, selected-global identity rules, and slow-consumer delivery behavior before this becomes a public event.
  - confidence: 0.84
