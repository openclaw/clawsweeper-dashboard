## Summary

Keep open. The gateway runtime problem is source-reproducible and the PR moves in a plausible direction, but this PR is not mergeable as packaged because it adds an unrelated scheduled sync workflow with a push-capable secret and still needs public protocol/changelog coverage for the newly advertised `socket.drain` event.

## What This Changes

The PR adds `socket.drain` teardown broadcasts for session eviction/reset paths, threads session-message subscriber IDs into gateway context, registers the event for read-scoped broadcast/discovery, adds gateway tests, and also adds an upstream-sync workflow plus unrelated formatting churn.

## Best Possible Solution

Land a scoped gateway change that adds the teardown event with tests and protocol/changelog docs, without the sync workflow or unrelated churn.

## Reproduction Assessment

Yes, at source level. Current main keeps `sessions.messages.subscribe` in a separate registry while session mutation broadcasts only target session-event subscribers, and `socket.drain` is absent from the gateway event guard/discovery list.

## Solution Assessment

No, not as currently packaged. The gateway routing direction is narrow and maintainable, but the PR should remove the unrelated workflow and add protocol/changelog coverage before merge.
