  - body: This changes reset/delete/new `sessions.changed` broadcasts from the existing `{ dropIfSlow: true }` behavior to close-on-slow behavior because `server-broadcast` closes slow sockets when `dropIfSlow` is falsy. Existing long-lived websocket clients can now be disconnected during ordinary session teardown under backpressure, so keep the existing drop-on-slow path unless maintainers explicitly accept and document the disconnect contract.
  - confidence: 0.9

  - body: For selected global sessions, the PR routes by an agent-specific subscription key but emits a `socket.drain` payload containing only `sessionKey: "global"`, `reason`, and `ts`. A client subscribed to multiple agent global sessions on the same connection cannot know which global row to evict, so include the selected `agentId` or otherwise make the target unambiguous.
  - confidence: 0.88
- **[P2] Document the advertised drain event contract:** `src/gateway/server-methods-list.ts:47`
  - body: Adding `socket.drain` to `GATEWAY_EVENTS` makes it a discoverable gateway event, but the patch does not update the gateway protocol docs with payload fields, reason values, or slow-consumer behavior. Define that contract before advertising the event to SDK and external websocket clients.
