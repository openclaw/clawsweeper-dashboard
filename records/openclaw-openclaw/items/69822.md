- **[P1] Emit reset drains once with the selected agent scope:** `src/gateway/server-methods/agent.ts:1507-1510`
  - body: This new pre-ack emit does not pass `agentId`, so a selected-global reset resolves to `global` and first broadcasts an unscoped/default `socket.drain`/`sessions.changed`; the bare reset branch then emits again with the selected agent scope. That can evict the wrong downstream session and duplicate the teardown signal. Carry the selected agent through `runSessionResetFromAgent` and emit once, or make the early emit use the same selected scope and suppress the later duplicate.
  - confidence: 0.9
- **[P1] Keep teardown broadcasts from closing existing slow clients:** `src/gateway/server-methods/session-change-event.ts:154`
  - body: `broadcastToConnIds` closes a socket when it is slow and `dropIfSlow` is false or omitted. This PR makes teardown `sessions.changed` use `dropIfSlow: false` and sends `socket.drain` with no opts, so reset/delete/new can disconnect clients that current main only skipped for one slow frame. Preserve the existing default or get explicit maintainer approval plus upgrade/tests for the close-on-teardown policy.
  - confidence: 0.86
- **[P2] Document the advertised socket.drain event:** `src/gateway/server-methods-list.ts:47`
  - body: Adding `socket.drain` to `GATEWAY_EVENTS` makes it visible in the protocol discovery list, but the gateway docs still enumerate the common session events without this payload or its allowed reasons. Add the user-facing protocol docs for `sessionKey`, `reason`, and teardown semantics, or keep the event out of discovery until the contract is documented.
  - confidence: 0.82
