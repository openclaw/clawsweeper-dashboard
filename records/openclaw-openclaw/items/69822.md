## Summary

Keep this PR open: current main still lacks the proposed `socket.drain` teardown path, and the branch is an active focused implementation, but it still needs live behavior proof and maintainer approval for the new Gateway event contract before merge.

## What This Changes

The PR adds Gateway `socket.drain` broadcasts for session reset/delete/new flows, routes them to both session-event and per-session-message subscribers, registers the event for discovery/read-scope checks, and expands Gateway tests.

## Best Possible Solution

Land a maintainer-approved Gateway protocol update that documents `socket.drain`, keeps teardown routing centered in session mutation handling, and includes redacted live WebSocket proof for reset/delete delivery.

## Reproduction Assessment

Yes, at source level: current main broadcasts teardown `sessions.changed` only to session-event subscribers and lacks `socket.drain` registration, while transcript updates already union event and message subscribers. I did not run a live WebSocket repro in this read-only review.

## Solution Assessment

Mostly yes: centralizing teardown drains in session mutations and reusing subscriber mappings is the narrow maintainable runtime shape. It still needs public protocol docs and contributor-supplied live proof before merge.
