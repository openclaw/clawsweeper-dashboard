  - body: The drain routing uses `payload.agentId` to find the selected-global message subscriber, but the emitted payload only says `sessionKey: "global"`. Generic session subscribers and clients receiving this event cannot tell whether the main or work agent's global session was drained, so they can evict the wrong global state.
  - confidence: 0.93

  - body: For a bare `/reset` or `/new` without a suffix, this emits a teardown before the ack path, then the ack path emits another one below with selected-global `agentId` when present. That gives subscribers duplicate teardown frames and, for selected-global sessions, the first frame is unscoped.
  - confidence: 0.91

  - body: Changing teardown `sessions.changed` broadcasts to `dropIfSlow: false` means existing slow `sessions.subscribe` clients are closed during reset/delete/new instead of only dropping the over-buffered update. The new reliable drain should be targeted to `socket.drain` unless maintainers explicitly approve this compatibility change.
  - confidence: 0.9
