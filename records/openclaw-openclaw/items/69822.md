- **[P1] Preserve slow-client teardown behavior:** `src/gateway/server-methods/session-change-event.ts:154`
  - body: `broadcastToConnIds` closes slow sockets whenever `dropIfSlow` is false or omitted. This PR sends `socket.drain` without options and changes teardown `sessions.changed` to `dropIfSlow: false`, so reset/delete/new can disconnect existing long-lived gateway clients where current session updates only drop the frame. Keep the existing default unless maintainers explicitly choose and document close-on-drain behavior.
  - confidence: 0.91

  - body: For selected-global sessions the PR resolves subscribers through `agent:<id>:global`, but the emitted payload still only says `sessionKey: "global"`. A single WebSocket can subscribe to multiple session keys, so a message-only client subscribed to more than one global-agent stream cannot know which agent's global session was evicted.
  - confidence: 0.88

  - body: Adding `socket.drain` to `GATEWAY_EVENTS` makes it discoverable as a gateway protocol event, but the gateway docs still list the common event families without it.
