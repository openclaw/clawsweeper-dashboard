## Summary

Keep open: current main still lacks this Gateway drain path, and the PR targets the right surface, but the patch is not merge-ready because it fails type checks, exposes a new advertised event without protocol docs, and still has only test-output proof rather than live WebSocket behavior proof.

## What This Changes

The PR adds Gateway `socket.drain` teardown broadcasts for session reset/delete/new, routes them to session event and per-session message subscribers, exposes the subscriber lookup in request context, registers the event, and adds focused Gateway tests.

## Best Possible Solution

Land a documented, type-clean Gateway protocol update that routes teardown drains through the existing subscriber registries and includes live WebSocket proof for reset/delete delivery.

## Reproduction Assessment

Yes, at source level: current main emits only `sessions.changed` to session-event subscribers and does not register `socket.drain` in event discovery or scope guards. I did not run a live WebSocket reproduction in this read-only review.

## Solution Assessment

Mostly yes, but the current PR is not the best complete merge shape yet: the routing approach is narrow, but it needs the missing local context implementation, protocol docs, and live behavior proof before maintainer approval.
