## Summary

Keep open: current main still lacks the `socket.drain` teardown path for message-only session subscribers, and this PR is the active canonical implementation. It is not merge-ready until contributor-supplied live Gateway/WebSocket proof is added and the newly advertised event is documented.

## What This Changes

The PR adds Gateway `socket.drain` broadcasts for session reset/delete/new flows, routes them to event and message subscribers, registers the event with read scope and discovery, and adds focused Gateway tests.

## Best Possible Solution

Land a maintainer-approved Gateway protocol update that documents `socket.drain`, keeps teardown routing centered in session mutation handling, and includes redacted live WebSocket proof.

## Reproduction Assessment

Yes, at source level: current main sends teardown `sessions.changed` only to session-event subscribers, lacks `socket.drain` registration, and already uses event-plus-message subscriber union for transcript events. I did not run a live WebSocket repro in this read-only review.

## Solution Assessment

Mostly yes: centralizing teardown drains in session mutations and reusing subscriber mappings is the narrow maintainable runtime shape. It still needs public protocol docs and contributor-supplied live proof before merge.
