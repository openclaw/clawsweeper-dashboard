- **[P1] Preserve slow-client teardown behavior:** `src/gateway/server-methods/session-change-event.ts:154`
  - body: Before this PR, session mutation broadcasts used `{ dropIfSlow: true }`, which skips frames but keeps long-lived gateway clients connected. This line switches teardown sends to `{ dropIfSlow: !isTeardown }`; the broadcaster closes slow sockets when `dropIfSlow` is false, so reset/delete/new can disconnect existing clients under backpressure.
  - confidence: 0.91
- **[P1] Include the global agent in drain payloads:** `src/gateway/server-methods/session-change-event.ts:62-64`
  - body: For selected-global sessions, this PR routes the drain to an agent-specific subscription key but sends only `sessionKey: "global"` in the payload. Clients subscribed to multiple global-agent sessions cannot tell which agent's global session to evict, even though the following `sessions.changed` payload includes `agentId`.
  - confidence: 0.88
- **[P2] Document the advertised drain event contract:** `src/gateway/server-methods-list.ts:47`
  - body: Adding `socket.drain` to `GATEWAY_EVENTS` makes it a discoverable gateway protocol event, but the PR does not document the payload, reason values, global-agent semantics, or slow-consumer behavior.
