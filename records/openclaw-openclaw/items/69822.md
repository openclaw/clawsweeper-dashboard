- **[P1] Preserve slow-client teardown delivery semantics:** `src/gateway/server-methods/session-change-event.ts:112`
  - body: This changes teardown `sessions.changed` broadcasts to `dropIfSlow: false`, and the new `socket.drain` send also has no `dropIfSlow: true` option. In the current broadcaster, slow targeted clients are closed before any frame is sent when dropping is disabled, so existing subscribers under backpressure will be disconnected and still may not receive the drain payload. Keep the existing drop behavior unless maintainers explicitly choose a documented disconnect-on-teardown contract, or implement a path that actually delivers before closing.
  - confidence: 0.92
- **[P1] Include agentId on selected-global drain payloads:** `src/gateway/server-methods/session-change-event.ts:62-65`
  - body: The drain recipient lookup uses `payload.agentId` to target `agent:<id>:global` message subscribers, but the emitted payload only contains `sessionKey`, `reason`, and `ts`. Current selected-global `session.message` and `sessions.changed` events carry `agentId` so multi-agent clients can ignore other agents; without it, a selected-global consumer can evict the wrong cached global session.
  - confidence: 0.91

  - body: Adding `socket.drain` to `GATEWAY_EVENTS` exposes it in Gateway feature discovery, but the PR does not update the public protocol docs or SDK normalization for the event, payload fields, reason enum, scope, or slow-consumer behavior.
