## Summary

Keep open. Current main still lacks the requested `socket.drain` routing, and the PR’s runtime direction matches the source-level gap, but it is not merge-ready because the supplied proof is only scoped test output and the newly advertised gateway event is not documented.

## What This Changes

The PR adds read-scoped `socket.drain` teardown broadcasts for session reset/delete/new paths, routes them to session-event and per-session message subscribers, threads message-subscriber lookup through gateway context, refactors agent resets to shared session change emission, and adds gateway tests.

## Best Possible Solution

Land this scoped gateway protocol change, or an equivalent one, after contributor-supplied live gateway-client proof and docs/changelog describe the `socket.drain` payload and read-scoped teardown semantics.

## Reproduction Assessment

Yes, at source level. Current main keeps `sessions.messages.subscribe` in a separate subscriber registry while session mutation broadcasts only use session-event subscribers, and `socket.drain` is absent from event guards/discovery.

## Solution Assessment

Yes for the runtime shape, but not yet for merge readiness. Unioning event and message subscribers at `emitSessionsChanged` matches the existing transcript broadcast precedent; docs/changelog and contributor-supplied real behavior proof remain required.
