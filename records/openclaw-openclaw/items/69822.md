## Summary

Keep open: the source-level gap is real, but the PR is not merge-ready because it makes the request context contract incomplete, advertises a new Gateway event without a documented protocol contract, and supplies only focused test output rather than live WebSocket behavior proof.

## What This Changes

This PR adds `socket.drain` Gateway teardown events for session reset/delete/new, routes them to session-event and per-session message subscribers, advertises and scope-guards the event, refactors session change emission, and adds Gateway tests.

## Best Possible Solution

Land a type-clean, documented Gateway protocol update with local embedded context support and redacted live WebSocket proof for reset/delete drain delivery.

## Reproduction Assessment

Yes, at source level: current main routes `sessions.changed` only through session-event subscribers and the message subscription path is isolated to transcript events. I did not run a live WebSocket failure case in this read-only review.

## Solution Assessment

Mostly yes, but not complete: centralizing teardown drains in `emitSessionsChanged` is a narrow Gateway boundary, while the patch still needs the local context implementation, protocol documentation, and live behavior proof.
