## Summary

Keep open: current main does not implement the requested `socket.drain` routing, and this PR is a focused implementation, but it still needs protocol docs and a changelog entry before merge.

## What This Changes

The PR adds read-scoped `socket.drain` teardown broadcasts for session reset/delete/new paths, routes them to session-event and matching session-message subscribers, threads message-subscriber lookup through gateway context, and adds gateway tests.

## Best Possible Solution

Land the scoped gateway runtime/test changes after adding protocol quick-reference and changelog coverage for the new `socket.drain` event.

## Reproduction Assessment

Yes, at source level. Current main keeps `sessions.messages.subscribe` in a separate subscriber registry while session mutation broadcasts only target session-event subscribers, and `socket.drain` is absent from the allowed/discovered gateway events.

## Solution Assessment

Yes, with one packaging gap. Unioning event and per-session message subscribers at the mutation emitter matches existing transcript broadcast precedent, but the newly advertised event needs protocol docs and changelog coverage before merge.
