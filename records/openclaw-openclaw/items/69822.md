- **[P1] Preserve slow-client teardown behavior:** `src/gateway/server-methods/session-change-event.ts:154`
  - body: Current main sends `sessions.changed` with `{ dropIfSlow: true }`, which makes `server-broadcast.ts` drop the frame for an over-buffered socket. This PR flips teardown updates to `false` and also emits `socket.drain` without explicit options, so reset/delete/new can close existing slow clients with code 1008 under normal backpressure; keep the existing drop-on-slow default unless maintainers explicitly approve and document the incompatible close-on-slow behavior.
  - confidence: 0.94
- **[P1] Include the selected global agent in drain payloads:** `src/gateway/server-methods/session-change-event.ts:61-65`
  - body: The PR computes per-agent global subscription keys, but the emitted payload only contains `sessionKey: "global"`. Since one WebSocket connection can hold multiple session subscriptions, downstream clients need the same `agentId` carried by `sessions.changed` to know which selected-global session cache to flush.
  - confidence: 0.91
- **[P2] Document the advertised drain event contract:** `src/gateway/server-methods-list.ts:47`
