## Summary

Keep open: the PR is a focused implementation for the canonical Gateway `socket.drain` feature, but the current head does not typecheck, the newly advertised event is not documented, and the supplied proof is still test-only rather than live Gateway/WebSocket evidence.

## What This Changes

The PR adds Gateway `socket.drain` teardown broadcasts for session reset/delete/new flows, routes them to session event and per-session message subscribers, registers the event, and expands Gateway tests.

## Best Possible Solution

Land a type-clean, documented Gateway protocol update that routes reset/delete/new drains to the right subscribers and includes redacted live WebSocket proof for reset and delete delivery.

## Reproduction Assessment

Yes, at source level: current main routes lifecycle changes only to session-event subscribers as `sessions.changed`, while transcript updates are the path that unions message subscribers. I did not run a live failing WebSocket reproduction in this read-only review.

## Solution Assessment

Mostly yes, but incomplete: centralizing teardown drain routing in `emitSessionsChanged` is a narrow maintainable direction. The implementation still needs the local context method, public protocol documentation, and live behavior proof before merge.
