## Summary

Keep open: the runtime direction addresses a real current-main gap, but this external PR is not merge-ready because the supplied proof is still test-only and the newly advertised Gateway event lacks docs/changelog coverage.

## What This Changes

The PR adds a read-scoped `socket.drain` Gateway event for session reset/delete/new, routes teardown broadcasts to session and per-session message subscribers, exposes the message-subscriber registry in request context, refactors agent reset emissions, and adds gateway tests.

## Best Possible Solution

Land this Gateway protocol change, or a maintainer-approved equivalent, after documenting `socket.drain` and receiving redacted live WebSocket proof that message-only subscribers receive it on reset/delete.

## Reproduction Assessment

Yes, at source level. Current main keeps session-message subscribers in a separate registry while `emitSessionsChanged` only targets all-session event subscribers, and `socket.drain` is absent from the event guard/discovery lists.

## Solution Assessment

Yes for the runtime direction, but no for merge readiness. Unioning event and message subscribers at `emitSessionsChanged` matches existing `session.message` routing; docs/changelog and contributor-supplied real behavior proof remain required.
