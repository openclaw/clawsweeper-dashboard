## Best Possible Solution

Land a maintainer-approved Gateway protocol update that documents and tests `socket.drain`, keeps teardown routing centralized in `emitSessionsChanged`, and includes redacted live WebSocket proof.

## Reproduction Assessment

Yes, source-level only. Current main keeps per-session message subscribers separate from `sessions.changed` mutation broadcasts and rejects unregistered non-plugin events, so the requested `socket.drain` path is absent without running a live websocket repro.

## Solution Assessment

Yes for the runtime direction, but not yet merge-ready. Unioning subscriber sets in `emitSessionsChanged` matches existing transcript-event routing; the safer merge path is to document the protocol event and require contributor-supplied live proof first.
