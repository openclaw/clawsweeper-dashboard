## Summary

Keep open. Current main still has the verbose-off session/channel leak, and this PR is directionally useful but not merge-ready because it misses final chat payloads, changes verbose resolution semantics, caches runtime settings too aggressively, and mixes in unrelated plugin/channel changes.

## What This Changes

The PR gates assistant delta and buffered-flush session sends behind verbose-level checks, adds gateway regression tests, caches verbose resolution, and includes unrelated plugin SDK, channel, and docs merge fixups.

## Best Possible Solution

Land a focused current-main gateway fix that suppresses assistant text to session/channel subscribers whenever effective verbose is off across delta, buffered flush, final done, and final error paths, while preserving Control UI/WebSocket broadcasts and leaving plugin/channel refactors out.

## Reproduction Assessment

Yes, from source inspection. Register a verbose-off gateway run, emit assistant text followed by a tool-start or lifecycle-end event, and current main reaches unconditional `nodeSendToSession` chat sends in the delta, flush, or final paths.

## Solution Assessment

No. The PR is useful source work, but the best fix is a narrower current-main gateway patch that gates final payloads too, defaults unresolved verbose to off, avoids stale in-flight verbose caching, and drops unrelated plugin/channel edits.

  - body: The patch guards assistant deltas and buffered flushes, but final `done` and `error` chat payloads still go to the session unconditionally. With verbose off, subscribers can still receive the complete buffered assistant text or terminal error payload at run end.
