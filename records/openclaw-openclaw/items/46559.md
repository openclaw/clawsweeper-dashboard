## Summary

Keep open. The PR targets a real verbose-off channel/session leak on current main, but the branch is incomplete: it gates assistant delta/flush sends while final/error chat payloads and stale verbose decisions can still leak or suppress text incorrectly, and the external PR still lacks real Telegram/channel proof.

## What This Changes

The branch gates gateway assistant chat delta and flush session sends on verbose state, adds agent-event tests, and carries stale docs/plugin SDK/channel conflict fixups.

## Best Possible Solution

Land a focused gateway fix that gates every session/channel assistant chat delivery path against current verbose state while preserving WS/control UI broadcasts, with lifecycle-end regression coverage and redacted real channel proof.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists: current main still sends assistant chat payloads to sessions with verbose off, and the PR head still sends the buffered final payload after lifecycle end.

## Solution Assessment

No. The gateway gating direction is useful, but this PR only gates delta/flush sends and adds a stale verbose cache; final/error sends and current-state re-resolution need to be fixed before this is the narrow maintainable solution.

  - confidence: 0.82
