## Summary

Keep open. The PR targets a real leak class that current main still has, but the branch is not merge-ready because it only gates delta/flush delivery, leaves final/error session sends exposed, freezes verbose state for an entire run, lacks real channel proof, and carries stale unrelated merge-fixup churn.

## What This Changes

The branch gates gateway assistant chat delta/flush session sends behind verbose-level resolution, adds gateway regression tests, and carries stale docs/plugin SDK/channel merge fixups.

## Best Possible Solution

Update the gateway fix to gate every session/channel chat delivery path, including final and error events, against current verbose state while preserving WS/control UI broadcasts; remove unrelated stale merge fixups and add lifecycle-end regression plus redacted real channel proof.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists on the PR head: emit assistant text with verbose off, then emit lifecycle end; deltas are gated, but emitChatFinal still sends the buffered final payload to the session subscriber.

## Solution Assessment

No. The PR's gateway gating direction is useful, but gating only delta/flush paths is incomplete; final/error sends need the same policy and the verbose decision should be refreshed or invalidated for in-flight session changes.
