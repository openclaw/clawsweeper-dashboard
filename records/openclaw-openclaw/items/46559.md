## Summary

Keep open. Current main still has the verbose-off leak, but this PR is not merge-ready because it only guards some streaming paths, freezes/defaults verbose decisions unsafely, and carries unrelated plugin/channel changes that alter WhatsApp group policy wiring.

## What This Changes

The PR gates gateway assistant delta and buffered-flush session sends by verbose level, adds gateway regression tests, and includes unrelated plugin SDK, channel plugin, and docs changes from merge fixups.

## Best Possible Solution

Land a focused current-main gateway fix that suppresses assistant text to session/channel subscribers whenever effective verbose is off across delta, buffered flush, final done, and final error paths, while preserving Control UI/WebSocket broadcasts and leaving plugin SDK/channel refactors for separate review.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: register a verbose-off gateway run, emit assistant text plus a tool or lifecycle event, and observe current `nodeSendToSession` chat payloads from the unconditional delta, flush, or final paths.

## Solution Assessment

No. The PR is directionally useful, but the best fix is a narrower current-main gateway patch that resolves verbose safely per event/default, covers final payloads, and omits unrelated plugin/API changes.

## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P1] Gate final chat payloads by verbose level:** `src/gateway/server-chat.ts:496-507`
  - body: The patch guards delta and buffered-flush session sends, but final `done` and `error` chat payloads still go to the session unconditionally. With verbose off, subscribers can still receive the complete buffered assistant text or error payload at run end.
  - confidence: 0.96
- **[P1] Default unresolved verbose to off:** `src/gateway/server-chat.ts:543-546`
  - body: `shouldStreamAssistantTextToSession` returns true when no verbose level resolves. Since the existing gateway verbose fallback is off, default or temporarily unresolved sessions can still stream assistant text unless they have an explicit off entry.
  - confidence: 0.91
- **[P1] Re-read verbose after runtime updates:** `src/gateway/server-chat.ts:511-513`
  - body: The new cache returns the first resolved verbose value for the lifetime of the run. Runtime `/verbose` or `sessions.patch` changes can then keep using a stale mode, so later assistant/tool events may leak or suppress text incorrectly.
  - confidence: 0.88
- **[P1] Preserve WhatsApp group policy registration:** `extensions/whatsapp/src/channel.ts:47-52`
  - body: The branch removes WhatsApp group adapter registration while current main uses those adapters for require-mention, tool policy, and group intro behavior. That access-control surface should not change as part of this gateway verbose fix.
  - confidence: 0.82
