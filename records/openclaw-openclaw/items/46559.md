## Summary

Keep open. This is a real verbose-off/channel leak fix attempt, but the PR is not merge-ready: the final chat event can still carry the buffered assistant text to session/channel subscribers, the new verbose cache can stale in-flight mode changes, the branch is conflicted with failing checks, and external real behavior proof is missing.

## What This Changes

The branch gates assistant chat delta and flush forwarding to session/channel subscribers on verbose level, adds gateway regression tests, and includes unrelated extension/plugin-sdk merge fixups.

## Best Possible Solution

Rebase to current main, keep the change narrowly in the gateway chat delivery path, suppress verbose-off internal assistant text through delta, flush, and final lifecycle paths, preserve Control UI visibility, and prove it with focused tests plus real Telegram/channel behavior evidence.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: on the PR head, emit assistant text with verbose off, then emit lifecycle end; the delta path is suppressed but emitChatFinal still forwards the buffered assistant text to session/channel subscribers.

## Solution Assessment

No. The direction is plausible, but the current patch is incomplete and stale; the safer fix is a narrow current-main gateway change that covers finalization and does not broaden plugin SDK surfaces.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Suppress verbose-off text in final chat events:** `src/gateway/server-chat.ts:546`
  - body: After the new delta guards, a verbose-off run that emits assistant text and then reaches lifecycle end still sends the complete buffered text to session/channel subscribers, because emitChatFinal calls nodeSendToSession unconditionally. Add coverage that emits assistant text plus lifecycle end with verbose off and remove the message content, or otherwise avoid forwarding it, on the subscriber path.
  - confidence: 0.91
- **[P2] Keep verbose changes observable during a run:** `src/gateway/server-chat.ts:511-513`
  - body: resolveVerboseLevel returns a cached value before checking the current run context or session entry. Current main re-reads session state and lets a later /verbose update affect subsequent events; this cache can keep leaking after verbose is turned off, or keep suppressing after it is turned on, for the rest of the run.
  - confidence: 0.86
- **[P2] Remove the unrelated root SDK barrel expansion:** `src/plugin-sdk/index.ts:60-61`
  - body: The gateway leak fix should not add broad runtime, config, webhook, secret, and channel helper exports to the root plugin-sdk barrel. That entrypoint is a public hot surface with dedicated subpaths for these helpers, and this unrelated conflict fixup makes the branch riskier to merge.
  - confidence: 0.82
