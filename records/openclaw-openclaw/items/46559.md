## Summary

Keep open. Current main still has a source-reproducible verbose-off assistant chat leak path, but this PR is not merge-ready because it leaves final/error chat payloads unguarded, makes verbose decisions stale during active runs, carries unrelated SDK/channel churn, and lacks real channel proof.

## What This Changes

The branch gates gateway assistant delta/flush session sends behind verbose state, adds gateway regression tests, and carries unrelated docs, channel, and Plugin SDK conflict fixups.

## Best Possible Solution

Land a focused gateway fix that gates every session/channel assistant chat delivery path on the current verbose state, preserves WS/control UI broadcasts, removes unrelated SDK/channel churn, and includes regression coverage plus redacted real channel proof.

## Reproduction Assessment

Yes. Current main has a high-confidence source reproduction: verbose-off assistant text is still sent through unguarded session chat delta, flush, final, and error paths, and existing gateway harness tests can assert those session sends.

## Solution Assessment

No. The PR's gateway gating direction is useful, but the submitted patch only gates delta/flush paths, leaves final/error delivery exposed, makes verbose state stale for a run, and broadens unrelated SDK surfaces.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Gate final chat sends when verbose is off:** `src/gateway/server-chat.ts:496-507`
  - body: The new guard only wraps delta and flush sends. `emitChatFinal` still builds a final assistant message from buffered text and sends it to the session unconditionally, so verbose-off channel/session subscribers can still receive the complete internal text at lifecycle end.
  - confidence: 0.94
- **[P2] Re-read verbose state during active runs:** `src/gateway/server-chat.ts:511-513`
  - body: The per-run `verboseLevels` cache returns the first resolved value before checking run context or session config again. If `/verbose` or session config changes during a long stream, later assistant/tool events can keep using the stale mode and leak or suppress text incorrectly.
  - confidence: 0.82
- **[P3] Remove unrelated root SDK barrel exports:** `src/plugin-sdk/index.ts:60-61`
  - body: This branch adds many runtime/config/channel helpers to the root `openclaw/plugin-sdk` barrel even though that entry is intentionally narrow and channel helpers belong on dedicated subpaths. Rebase the branch and keep this chat leak fix from broadening plugin import topology.
  - confidence: 0.76
