## Summary

Keep open. The branch addresses a real leak class, but it still lets verbose-off final/error chat payloads reach session subscribers, freezes verbose state for the run, carries unrelated SDK surface expansion, and lacks required real channel proof.

## What This Changes

The PR tries to suppress assistant chat streaming to session/channel subscribers when verbose is off while keeping WebSocket chat broadcasts visible to the Control UI.

## Best Possible Solution

Rebase or replace the PR with a narrow gateway-only fix that guards delta, flush, final, and error session chat sends consistently, preserves current dynamic verbose semantics, avoids unrelated SDK/plugin churn, and includes live channel proof plus focused regression tests.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the source-level gateway path: assistant chat payloads can be sent to session/channel subscribers while verbose is off, and the PR head still leaves final/error sends unguarded; this review did not run a live channel reproduction.

## Solution Assessment

Is this the best way to solve the issue? No. The intended guard is reasonable, but this branch is incomplete, makes verbose decisions stale for a run, and mixes unrelated public SDK changes into a bug fix.
