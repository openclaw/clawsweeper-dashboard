## Summary

Keep open. Current main still has the verbose-off session/chat leak, and this PR is directionally useful but not mergeable as-is because it misses final chat payloads, makes unresolved verbose streamable, caches verbose decisions across in-flight updates, and carries unrelated plugin/channel changes.

## What This Changes

The PR gates some gateway assistant delta and buffered-flush session sends by verbose level, adds gateway regression tests, and includes unrelated plugin SDK, channel plugin, and docs changes from merge fixups.

## Best Possible Solution

Land a focused gateway fix that suppresses assistant text to session/channel subscribers whenever effective verbose is off across delta, buffered flush, final done, and final error paths, while preserving Control UI/WebSocket broadcasts and leaving plugin SDK/channel refactors for separate review.

## Reproduction Assessment

Yes. The gateway event-handler harness can register a verbose-off run, emit assistant text plus a tool/lifecycle event, and observe `nodeSendToSession` receiving chat payloads; the current buffered-flush test already asserts that leaking behavior.

## Solution Assessment

No. The PR is directionally useful, but the best solution is a narrower current-main gateway patch that resolves verbose safely per event/default, covers final payloads, and omits unrelated plugin/API changes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Gate final chat payloads by verbose level:** `src/gateway/server-chat.ts:496-507`
  - body: The patch guards delta and buffered-flush session sends, but final `done` and `error` chat payloads still go to the session unconditionally. With verbose off, subscribers can still receive the complete buffered assistant text or error payload at run end.
  - confidence: 0.96
- **[P1] Default unresolved verbose to off:** `src/gateway/server-chat.ts:543-546`
  - body: `shouldStreamAssistantTextToSession` returns true when no verbose level resolves. Since verbose off is the documented default, default or temporarily unresolved sessions can still stream assistant text unless they have an explicit off entry.
  - confidence: 0.91
- **[P1] Re-read verbose after runtime updates:** `src/gateway/server-chat.ts:511-513`
  - body: The new cache returns the first resolved verbose value for the lifetime of the run. Runtime `/verbose` or `sessions.patch` changes are documented to affect in-flight output, so later assistant/tool events can keep using a stale mode and leak or suppress text incorrectly.
  - confidence: 0.88
- **[P1] Preserve WhatsApp group policy wiring:** `extensions/whatsapp/src/channel.ts:47-52`
  - body: The branch removes WhatsApp group adapter registration while current main uses those adapters for require-mention, tool policy, and group intro behavior. That access-control surface should not change as part of this gateway verbose fix.
  - confidence: 0.82
