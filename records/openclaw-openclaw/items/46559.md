## Summary

Keep open: current main still has a source-level verbose-off chat leak, but this PR is not merge-ready because it leaves final/error session chat payloads unguarded, treats missing verbose as stream-enabled, caches stale verbose state, and carries unrelated Plugin SDK/channel conflict churn.

## What This Changes

The branch adds verbose-based gates for gateway assistant delta/flush session sends, tests those cases, and also changes root Plugin SDK plus bundled channel setup exports/imports.

## Best Possible Solution

Land a focused gateway change that gates delta, flush, final, and error session/channel assistant text on the current effective verbose state, preserves Control UI and normal final delivery, and leaves Plugin SDK/channel churn to separate API review.

## Reproduction Assessment

Yes, at source level. Current main still calls `nodeSendToSession` for assistant chat delta, flush, final, and error payloads without a verbose assistant-text gate, and an existing verbose-off test expects session chat delivery; I did not run tests because this review is read-only.

## Solution Assessment

No. Gating delta and flush sends is useful, but the PR must also cover final/error sends, honor default-off and current session state, and drop unrelated SDK/plugin churn before it is the maintainable fix.
