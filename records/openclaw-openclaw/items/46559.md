## Summary

Keep open. The PR targets a real verbose-off privacy bug, but the patch is incomplete and unsafe to merge because final chat payloads can still reach session/channel subscribers, the new verbose cache can use the wrong effective mode, and unrelated WhatsApp group-policy wiring is removed.

## What This Changes

The PR gates assistant delta and buffered-flush session sends by resolved verbose level, adds gateway regression tests, and includes unrelated plugin SDK, channel, and docs merge fixups.

## Best Possible Solution

Land a focused current-main gateway fix that suppresses assistant text to session/channel subscribers whenever effective verbose is off across delta, buffered flush, final done, and final error paths, while preserving Control UI/WebSocket broadcasts and leaving unrelated plugin/channel refactors out.

## Reproduction Assessment

Yes, from source inspection. Register a verbose-off gateway run, emit assistant text, then trigger a tool-start or lifecycle-end event; current main reaches unconditional `nodeSendToSession` chat sends in the delta, flush, and final paths.

## Solution Assessment

No. The PR fixes part of the leak, but the best solution is a narrower current-main gateway patch that also gates final payloads, treats unresolved verbose as off, avoids stale in-flight verbose caching, and drops unrelated plugin/channel edits.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Gate final chat payloads by verbose level:** `src/gateway/server-chat.ts:496-507`
  - body: The patch guards assistant deltas and buffered flushes, but final `done` and `error` chat payloads still go to the session unconditionally. With verbose off, subscribers can still receive the complete buffered assistant text or terminal error payload at run end.
  - confidence: 0.96
- **[P1] Default unresolved verbose to off:** `src/gateway/server-chat.ts:543-546`
  - body: `shouldStreamAssistantTextToSession` returns true when no verbose value resolves. The existing default is off, so default or temporarily unresolved sessions can still stream assistant text unless they have an explicit off entry.
  - confidence: 0.91
- **[P1] Re-read verbose after runtime updates:** `src/gateway/server-chat.ts:511-513`
  - body: The new cache returns the first resolved verbose value for the lifetime of the run. Runtime `/verbose` or `sessions.patch` changes can then keep using a stale mode, so later assistant/tool events may leak or suppress text incorrectly.
  - confidence: 0.88
- **[P1] Preserve WhatsApp group policy registration:** `extensions/whatsapp/src/channel.ts:45`
  - body: The branch removes WhatsApp group adapter registration while current main uses those adapters for require-mention, tool policy, and group intro behavior. That access-control surface should not change as part of this gateway verbose fix.
  - confidence: 0.82
