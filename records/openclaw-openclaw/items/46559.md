## Summary

Keep open. The leak is still source-reproducible on current main and this PR has useful gateway direction, but the branch remains incomplete because final/error chat sends are still unguarded, verbose state can go stale during a run, unrelated SDK churn is mixed in, and external real behavior proof is missing.

## What This Changes

The PR gates gateway assistant delta/flush session sends on verbose level, adds gateway regression tests, and carries unrelated merge-fix edits across channel plugins, docs, and the Plugin SDK barrel.

## Best Possible Solution

Land a focused gateway fix that gates every session/channel assistant chat delivery path on the current verbose state, preserves WS/control UI broadcasts, drops unrelated plugin/SDK churn, adds regression coverage, and includes redacted live channel proof.

## Reproduction Assessment

Yes. Source inspection shows current main unconditionally sends assistant chat delta, flush, final, and error payloads to session subscribers, and current gateway coverage still asserts verbose-off tool-flush session delivery.

## Solution Assessment

No. Gating delta and flush sends is useful, but the PR must also cover final/error sends, preserve current verbose-state re-read behavior, and remove unrelated SDK/plugin churn before it is the maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Gate final chat sends when verbose is off:** `src/gateway/server-chat.ts:496-507`
  - body: The new verbose guard only covers delta and flush sends. `emitChatFinal` still builds a final assistant message from buffered text and sends it to session/channel subscribers unconditionally, so verbose-off users can still receive the complete internal message at lifecycle end.
  - confidence: 0.94
- **[P2] Re-read verbose state during active runs:** `src/gateway/server-chat.ts:511-535`
  - body: `resolveVerboseLevel` returns the first cached value before checking run context or session config again. If `/verbose` changes while a stream is active, later assistant/tool events keep using the old mode, so text can remain leaked or suppressed contrary to the current session setting.
  - confidence: 0.86
- **[P2] Drop the unrelated SDK barrel expansion:** `src/plugin-sdk/index.ts:60-61`
  - body: This gateway leak fix should not add a large set of public `openclaw/plugin-sdk` re-exports from config, infra, webhook, and channel runtime helpers. That turns internal implementation convenience into public plugin API surface and conflicts with the SDK boundary rules; keep only the gateway fix and remove the merge-fix barrel churn.
  - confidence: 0.84
