Full review comments:

- **[P1] Gate final chat payloads before session delivery:** `src/gateway/server-chat.ts:490-502`
  - body: The patch gates assistant deltas, but `emitChatFinal` still sends the complete final or error chat payload to the session without the same verbose-off check. A subscriber that saw no deltas can still receive the buffered assistant text at lifecycle end, which leaves the reported leak open.
  - confidence: 0.86
- **[P1] Do not cache missing verbose state for the whole run:** `src/gateway/server-chat.ts:525-529`
  - body: `resolveVerboseLevel` caches `null` when the early event has no session key or run verbose level, and later calls return that cached value instead of re-checking once the session is known. That makes verbose-off sessions leak if mapping arrives after an early event.
  - confidence: 0.78
- **[P1] Respect in-flight verbose updates:** `src/gateway/server-chat.ts:525-529`
  - body: The new per-run verbose cache makes the verbose decision immutable for the run, so a `/verbose` or session update during a long stream is ignored. The previous per-event lookup could observe the current setting, and this change can leak or suppress assistant text contrary to the user’s latest mode.
  - confidence: 0.74
- **[P1] Split unrelated Plugin SDK and channel wiring changes:** `src/plugin-sdk/index.ts:57`
  - body: This PR is a gateway leak fix, but it also expands the root Plugin SDK barrel and removes channel config/group wiring from several bundled channel plugin registrations. Those compatibility-sensitive changes are not needed for the verbose gate and should not ride with a security/message-delivery fix.
  - confidence: 0.7
