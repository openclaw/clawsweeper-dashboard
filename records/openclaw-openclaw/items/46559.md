## Summary

Keep open. The underlying verbose-off chat leak is still source-reproducible on current main, but this PR head is incomplete, has unrelated SDK/plugin churn, is conflicting, and still lacks real channel proof.

## What This Changes

The PR gates gateway assistant delta/flush session sends on verbose state, adds gateway regression tests, and includes unrelated docs, channel, and Plugin SDK conflict fixups.

## Best Possible Solution

Land a focused gateway fix that gates every session/channel assistant chat delivery path on the current verbose state, preserves WS/control UI broadcasts, removes unrelated SDK/channel churn, and includes regression coverage plus redacted real channel proof.

## Reproduction Assessment

Yes by source inspection. Current main unconditionally mirrors assistant chat delta, flush, final, and error payloads to sessions, and existing tests still assert verbose-off tool-flush session delivery; I did not run tests because this review is read-only.

## Solution Assessment

No. The PR's gateway gating direction is useful, but this branch leaves final/error delivery exposed, makes verbose state stale during active runs, and broadens unrelated SDK/plugin surfaces.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Gate final chat sends when verbose is off:** `src/gateway/server-chat.ts:496-507`
  - body: The new verbose guard only covers delta and flush sends. `emitChatFinal` still builds a final assistant message from buffered text and sends it to session/channel subscribers unconditionally, so verbose-off users can still receive the complete internal message at lifecycle end.
  - confidence: 0.94
- **[P2] Re-read verbose state during active runs:** `src/gateway/server-chat.ts:511-513`
  - body: `resolveVerboseLevel` returns the first cached value before checking run context or session config again. Current main lets newer in-flight session verbose settings override older run context; this cache can keep leaking or suppressing assistant text after `/verbose` changes during a stream.
  - confidence: 0.82
- **[P3] Remove unrelated root SDK barrel exports:** `src/plugin-sdk/index.ts:60-61`
  - body: This leak fix adds broad root `openclaw/plugin-sdk` exports for config, channel, webhook, secret, and runtime helpers. That widens a public plugin surface unrelated to gateway chat gating and conflicts with the local SDK boundary guidance to use narrow subpaths.
  - confidence: 0.76
