## Summary

Keep open. Current main still has a source-reproducible verbose-off session/chat leak, and this PR targets a real bug, but the branch is incomplete because final/error chat payloads can still reach session subscribers, verbose state can go stale during a run, and the external PR lacks real channel proof.

## What This Changes

The branch adds verbose-based gating for gateway assistant delta/flush session sends, regression tests for verbose-off/on streaming, and stale merge-fix changes across channel plugin setup and Plugin SDK exports.

## Best Possible Solution

Land a focused gateway fix that gates every session/channel assistant chat delivery path on current verbose state, preserves WS/control UI broadcasts, drops unrelated plugin/SDK churn, adds regression coverage, and includes redacted real Telegram/channel proof.

## Reproduction Assessment

Yes. Source inspection shows current main unconditionally mirrors assistant chat delta, flush, final, and error payloads to session subscribers, and current tests still assert verbose-off tool-flush session delivery.

## Solution Assessment

No. The PR's gateway gating direction is useful, but it leaves final/error delivery exposed, can use stale verbose state during active runs, and mixes unrelated plugin/SDK churn into the leak fix.
