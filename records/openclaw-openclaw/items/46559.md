## Summary

Keep open. Current main still has the verbose-off assistant-text leak, but this PR is stale, merge-conflicted, incomplete on final/default/cache paths, and carries unrelated plugin/channel changes that should not ride with the gateway privacy fix.

## What This Changes

The PR gates gateway assistant delta and buffered-flush session sends by effective verbose level, adds gateway regression tests, and carries plugin SDK/channel/docs merge-fixup changes.

## Best Possible Solution

Land a focused current-main gateway patch that suppresses assistant text to session/channel subscribers whenever effective verbose is off across delta, buffered flush, final done, and final error paths while preserving WebSocket/Control UI broadcasts and existing verbose on/full behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: register a verbose-off run, emit assistant text plus a tool-start or lifecycle end event, and current main sends `nodeSendToSession` chat payloads from the delta, flush, or final paths.

## Solution Assessment

No. The PR is directionally useful, but the safest fix is a narrower current-main patch that defaults unresolved verbose to off, avoids stale in-run cache behavior, gates final payloads, and drops unrelated plugin/channel changes.

  - body: The patch guards assistant delta and buffered-flush session sends, but final `done` and `error` chat payloads still go to session subscribers unconditionally. With verbose off, the subscriber can still receive the complete buffered assistant text or terminal error payload at run end.

  - body: The new cache makes the verbose decision immutable for the run once it resolves. Runtime `/verbose` or session patch changes can then keep using a stale mode, so later assistant/tool events may leak or suppress text contrary to the updated setting.
  - confidence: 0.88
