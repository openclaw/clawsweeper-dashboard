## Summary

Keep open. The PR is a plausible bug fix direction, but the current branch is not mergeable as-is: it leaves the final chat path leaking buffered assistant text, regresses current in-flight verbose updates, carries unrelated public SDK surface churn, and lacks required real behavior proof.

## What This Changes

The PR gates gateway assistant chat delta/flush forwarding to session subscribers on verbose level, adds agent-event regression tests, and carries plugin/channel SDK merge fixups.

## Best Possible Solution

A narrow current-main gateway patch should suppress verbose-off internal assistant text across delta, flush, and final session-subscriber paths while preserving Control UI visibility and current in-flight verbose update behavior.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists on the PR head: emit assistant text with `verboseLevel: "off"`, then emit lifecycle end; the delta path is suppressed but `emitChatFinal` still forwards the buffered assistant text to session/channel subscribers.

## Solution Assessment

No. The direction is useful, but the current patch is incomplete and stale; the safer path is a narrow gateway change on current main plus focused tests and real Telegram/channel proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Suppress verbose-off text in final chat events:** `src/gateway/server-chat.ts:496`
  - body: After the new delta guards, a verbose-off run that emits assistant text and then reaches lifecycle end still sends the complete buffered text to session/channel subscribers because `emitChatFinal` calls `nodeSendToSession` unconditionally. Add coverage for assistant text plus lifecycle end with verbose off, and avoid forwarding that final text on the subscriber path.
  - confidence: 0.91
- **[P2] Re-read verbose state for in-flight updates:** `src/gateway/server-chat.ts:511-513`
  - body: This cache return makes the verbose decision immutable for the run after the first lookup. Current main re-reads session state so a later `/verbose` update affects subsequent events; this branch can keep leaking after verbose is turned off, or keep suppressing after it is turned on.
  - confidence: 0.86
- **[P2] Remove the unrelated root SDK barrel expansion:** `src/plugin-sdk/index.ts:60-61`
  - body: The gateway leak fix should not add broad runtime, config, webhook, secret, and channel helpers to the root `openclaw/plugin-sdk` barrel. Current main keeps that entrypoint intentionally small with dedicated subpaths, so this unrelated merge fixup broadens a public API surface without being part of the bug fix.
  - confidence: 0.82
