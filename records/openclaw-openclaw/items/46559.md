## Summary

Keep open. The underlying verbose-off chat leak is still present on current main, and this PR is not mergeable as-is because it only partially gates assistant text delivery while also carrying unrelated stale plugin/channel changes.

## What This Changes

The PR adds verbose-level gating for some gateway assistant delta and buffered-flush session sends, adds gateway regression tests, and carries unrelated plugin SDK, channel plugin, and docs merge-fix changes.

## Best Possible Solution

Land a focused gateway fix that suppresses assistant text to session/channel subscribers whenever effective verbose is off across delta, buffered flush, final done, and final error paths, while preserving WebSocket/control UI broadcasts and leaving plugin SDK/channel changes for separate review.

## Reproduction Assessment

Yes. The gateway event-handler harness can register a verbose-off chat run, emit assistant text plus tool/lifecycle events, and observe nodeSendToSession receiving chat payloads on current main; the current buffered-flush test already asserts that behavior.

## Solution Assessment

No. The PR is directionally useful, but it is not the best mergeable fix because it misses final payloads, treats unresolved verbose as streamable despite the off default, caches verbose decisions across in-flight updates, and bundles unrelated channel/API churn.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Gate final chat payloads by verbose level:** `src/gateway/server-chat.ts:546-559`
  - body: The patch guards some delta/flush sends, but final done and error chat payloads are still sent to the session unconditionally. With verbose off, subscribers can still receive the complete buffered assistant text or error payload at run end, so the leak remains.
  - confidence: 0.96
- **[P1] Default unresolved verbose to off:** `src/gateway/server-chat.ts:543-547`
  - body: shouldStreamAssistantTextToSession returns true when the verbose level is unresolved. Since verbose off is the documented default, default/inherited sessions can still stream assistant text unless they have an explicit off entry.
  - confidence: 0.91
- **[P1] Re-read verbose after runtime updates:** `src/gateway/server-chat.ts:510-513`
  - body: The new cache returns the first resolved verbose value for the whole run. Runtime /verbose or sessions.patch changes are documented to affect in-flight output, so later assistant/tool events can keep using a stale mode and leak or suppress text incorrectly.
  - confidence: 0.88
- **[P1] Preserve WhatsApp group policy wiring:** `extensions/whatsapp/src/channel.ts:45`
  - body: The branch carries unrelated WhatsApp channel changes around group resolver registration. Current main uses that wiring for requireMention, tool policy, and group intro behavior, so this access-control surface should not be changed as part of the gateway verbose fix.
  - confidence: 0.82
