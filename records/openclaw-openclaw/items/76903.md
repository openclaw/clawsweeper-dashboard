## Summary

Close as implemented on main: the 2026.5.2 path fell through to the unrelated npm `discord` package, while current main resolves bare official plugin ids through the catalog to `@openclaw/discord` before raw npm, with coverage for that routing.

## What This Changes

The issue reports that `openclaw plugins install discord --force` on 2026.5.2 installs the wrong package shape instead of the official Discord plugin.

## Best Possible Solution

Keep the catalog-based official plugin id resolution on main and ship it in the next release; 2026.5.2 users can use the scoped `@openclaw/discord` package spec meanwhile.

## Reproduction Assessment

Yes. The issue log plus v2026.5.2 source show bare `discord` fell through to raw npm after bundled lookup, while current main inserts official catalog resolution before that fallback.

## Solution Assessment

Yes. The current-main fix is the narrow maintainable path: resolve only bare official plugin ids from the existing catalog, while preserving explicit `npm:` semantics for raw npm installs.
