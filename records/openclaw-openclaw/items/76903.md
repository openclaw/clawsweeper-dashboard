## Summary

Close as implemented: current main resolves bare official plugin IDs through the official catalog before generic npm fallback, so `discord` now resolves to `@openclaw/discord` instead of npm's unrelated `discord@0.8.2` package.

## What This Changes

Installing the Discord plugin with the bare id `discord` should fetch the official `@openclaw/discord` package instead of npm's unrelated unscoped `discord` package.

## Best Possible Solution

Keep the official-catalog pre-npm resolution on main and direct affected 2026.5.2 users to 2026.5.3-beta.1 or the next stable release.

## Reproduction Assessment

Yes, from source and package inspection rather than executing the installer. `openclaw@2026.5.2` lacks the official-external pre-npm resolver while current main and `2026.5.3-beta.1` contain it, and npm `discord` resolves to the reported `0.8.2` package.

## Solution Assessment

Yes. Resolving known official plugin ids before generic npm fallback is the narrow maintainable fix; explicit scoped npm specs and `npm:discord` remain available without relaxing plugin package validation.
