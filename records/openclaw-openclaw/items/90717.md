## Best Possible Solution

Land a narrow cooldown-probe fix that preserves auth fail-closed behavior, keeps multi-fallback preference intact, and proves single-provider rate/subscription caps can recover without adding config or migration surface.

## Reproduction Assessment

## Solution Assessment

Yes. Moving single-provider recovery into the shared primary probe helper is narrower than capping persisted `blockedUntil`, because it fixes recovery at the fallback decision boundary while preserving the stored provider reset data and existing auth/billing guards.
