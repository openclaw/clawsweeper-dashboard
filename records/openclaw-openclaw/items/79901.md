## Summary

Keep open: the PR is a recent focused fix for a real git plugin ref-pinning bug, and I found no blocking code-review finding. Maintainer review is still needed because PR CI was not fully green at review time.

## What This Changes

This PR changes git plugin ref checkout from `git checkout --detach <ref>` to `git switch --detach -- <ref>` and adds a regression test for option-like refs.

## Best Possible Solution

Land this PR or an equivalent narrow argv fix after CI is green, preserving the regression test for option-like refs.

## Reproduction Assessment

Yes. Current main's source path passes `parsed.ref` directly to `git checkout --detach`, and an isolated Git check reproduced the option-like ref success behavior that weakens pinning.

## Solution Assessment

Yes. Separating options from the requested ref at the git invocation boundary is the narrow maintainable fix, and the added unit test targets the exact regression point.
