## Summary

Keep open: current main still has the git ref-pinning bug and this recent PR is the active same-author fix for the linked issue; the diff looks correct, with sufficient terminal proof, but normal maintainer/CI validation still needs to finish.

## What This Changes

The PR changes git plugin ref checkout from `git checkout --detach <ref>` to `git switch --detach -- <ref>` and adds regression coverage for option-like refs.

## Best Possible Solution

Land this PR or an equivalent single canonical fix, then close https://github.com/openclaw/openclaw/issues/79898 as implemented after merge.

## Reproduction Assessment

Yes for the root cause: current main parses the selector as `parsed.ref` and invokes `git checkout --detach <ref>`, while the PR thread includes a scratch Git reproduction showing the option-like value exits successfully under checkout. I did not run a full OpenClaw plugin install during this read-only review.

## Solution Assessment

Yes; switching this one argv boundary to `git switch --detach -- <ref>` is the narrowest maintainable fix because it preserves detached ref checkout semantics while separating user ref text from Git options.
