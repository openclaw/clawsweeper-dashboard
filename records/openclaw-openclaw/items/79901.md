## Summary

Keep open: current main and the latest release still use the vulnerable git-ref checkout path, while this focused PR addresses the implicated argv boundary with regression coverage and supplied real behavior proof. I found no code-review blocker; CI is the remaining validation concern.

## What This Changes

The PR changes git plugin ref checkout from `git checkout --detach <ref>` to `git switch --detach -- <ref>` and adds a regression test for option-like refs.

## Best Possible Solution

Land this focused installer hardening after CI is green, then close https://github.com/openclaw/openclaw/issues/79898 as implemented.

## Reproduction Assessment

Yes for the root cause. Current main parses `@--ignore-skip-worktree-bits` as `parsed.ref` and sends it to `git checkout --detach`; a scratch Git 2.53.0 repo confirmed checkout exits 0 while `git switch --detach --` rejects the option-like value.

## Solution Assessment

Yes. Changing only the git checkout argv boundary to `git switch --detach -- <ref>` preserves valid detached ref checkout and forces leading-dash values into ref parsing, with a focused regression test before package install can proceed.
