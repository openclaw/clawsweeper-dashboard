## Summary

Keep open: current main still has the unsafe git plugin ref checkout argv, and this PR is a focused canonical fix with sufficient terminal proof and no blocking review findings.

## What This Changes

The PR changes git plugin ref checkout from `git checkout --detach <ref>` to `git switch --detach -- <ref>` and adds regression coverage for option-like refs.

## Best Possible Solution

Land this focused installer fix after normal validation, then let https://github.com/openclaw/openclaw/issues/79898 close as implemented.

## Reproduction Assessment

Yes for the root cause. Current main passes parsed refs to `git checkout --detach <ref>`, and the issue plus prior ClawSweeper context show Git accepting `--ignore-skip-worktree-bits` as a checkout option instead of rejecting it as a ref.

## Solution Assessment

Yes. Changing only this argv boundary to `git switch --detach -- <ref>` preserves normal detached ref checkout while making invalid option-like selectors fail before plugin install proceeds.
