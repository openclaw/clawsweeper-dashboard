## Summary

Keep this PR open: current main still uses the unsafe git checkout argv for explicit plugin refs, and this focused patch addresses the linked bug without an actionable code-review finding.

## What This Changes

This PR changes git plugin ref checkout from `git checkout --detach <ref>` to `git switch --detach -- <ref>` and adds regression coverage for option-like refs.

## Best Possible Solution

Land this focused installer fix after normal validation, then let the linked issue at https://github.com/openclaw/openclaw/issues/79898 close as implemented.

## Reproduction Assessment

Yes. Current main source passes `parsed.ref` directly to `git checkout --detach`, the linked issue and PR body provide terminal evidence for the option-like ref behavior, and local Git help confirms the option parsing boundary involved.

## Solution Assessment

Yes. Changing this single argv boundary to `git switch --detach -- <ref>` and pinning it with a focused unit test is the narrowest maintainable fix for the documented git-ref install contract.
