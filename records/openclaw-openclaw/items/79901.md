## Summary

Keep open: current main still has the reported git ref-boundary bug, and this PR is a narrow, well-supported fix with no blocking review findings.

## What This Changes

The PR changes git plugin ref checkout from `git checkout --detach <ref>` to `git switch --detach -- <ref>` and adds regression coverage for option-like refs.

## Best Possible Solution

Land this focused installer fix after normal validation, then let the linked issue at https://github.com/openclaw/openclaw/issues/79898 close as implemented.

## Reproduction Assessment

Yes for the root cause: current main passes refs to `git checkout --detach <ref>`, and a scratch Git repo reproduced the option-like value exiting successfully under checkout. I did not run the full OpenClaw CLI install on current main.

## Solution Assessment

Yes; switching this single argv boundary to `git switch --detach -- <ref>` is the narrowest maintainable fix, and `git checkout -h` confirms checkout’s `--` form is pathspec-oriented rather than the right ref separator here.
