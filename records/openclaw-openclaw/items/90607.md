## Summary

Keep open: current main still has the stale Twilio active-stream behavior, and this PR is a narrow fix that moves active-call tracking to the media stream registration path with focused regression coverage for https://github.com/openclaw/openclaw/issues/81122.

## What This Changes

The PR updates the Twilio voice-call provider so `activeStreamCalls` is set from `registerCallStream` instead of TwiML generation, and updates provider tests for connected-stream and no-stream cases.

## Best Possible Solution

Land this PR after normal checks if maintainers accept local provider proof for the no-stream path; it keeps the queue behavior for registered streams while avoiding stale TwiML-only activation.

## Reproduction Assessment

Yes. Current main can be traced from inbound stream TwiML generation to `activeStreamCalls.add`, and the PR body includes before/after terminal proof for the new regression test.

## Solution Assessment

Yes. Moving active-state ownership to `registerCallStream` is narrower than adding a pending timer because it uses the existing accepted media-start boundary and preserves queueing once a stream is actually registered.
