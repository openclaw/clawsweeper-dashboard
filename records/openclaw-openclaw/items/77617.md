## Summary

Keep open: the PR uses the right Ollama-owned provider-policy artifact, but as written it appears not to fix the reported pre-activation path because `reasoning` can be undefined and the new function would return the off-only profile.

## What This Changes

This PR adds an Ollama `resolveThinkingProfile` export, a regression test, and a changelog entry so `/think max` is intended to remain available before the full Ollama plugin runtime activates.

## Best Possible Solution

Keep the provider-policy artifact fix, but make it handle the no-catalog pre-activation Ollama model path and add a regression test for `ollama/deepseek-v4-pro:cloud` without `reasoning` metadata.

## Reproduction Assessment

Yes, by source inspection. The issue logs show the failing pre-activation path has `catalogReasoning=undefined`, and current code passes that undefined value directly into the provider thinking profile resolver.

## Solution Assessment

No, not as written. Adding the Ollama provider-policy hook is the right seam, but the implementation must cover the undefined-reasoning/no-catalog case instead of returning the off-only profile there.
