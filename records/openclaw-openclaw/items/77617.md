## Summary

Keep open: current main still lacks the Ollama provider-policy thinking export, and this PR is a narrow owner-local fix that mirrors the existing runtime Ollama thinking profile with regression coverage and a changelog entry.

## What This Changes

The branch adds an Ollama `resolveThinkingProfile` export to the provider-policy artifact, tests reasoning and non-reasoning profile output, and documents the fix in `CHANGELOG.md`.

## Best Possible Solution

Land this PR after normal CI or changed-gate proof, keeping the lightweight provider-policy profile aligned with the full Ollama runtime hook.

## Reproduction Assessment

Yes, by source inspection. Current main validates thinking levels through provider profiles and falls back to the lightweight provider-policy artifact before runtime activation, but Ollama's artifact currently has no profile while the runtime plugin exposes `max` for reasoning models.

## Solution Assessment

Yes. Adding the same profile to `extensions/ollama/provider-policy-api.ts` is the narrowest maintainable fix because the missing behavior is Ollama-owned static policy, not a new core seam or config option.
