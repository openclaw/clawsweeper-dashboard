## Summary

Keep open: the branch targets a real auto-update bug, but it should not merge because the proposed undefined env values do not delete inherited markers, and the update child would still be blocked by the gateway ancestry guard.

## What This Changes

The PR adds undefined gateway-service marker entries to the auto-update child env override, a regression test for that override shape, and a changelog entry.

## Best Possible Solution

Implement a service-safe auto-update handoff or equivalent real env deletion plus ancestry-aware routing that runs the package update outside the live gateway process tree while preserving the manual in-service package-update block.

## Reproduction Assessment

Yes. Source inspection reproduces the PR defect: inherited marker values survive the undefined overrides, and current update-cli tests show a gateway-owned updater child is blocked by process ancestry before package install.

## Solution Assessment

No. The proposed env override is not the best fix because it is ineffective under the current env merge contract and does not address the separate gateway-process-tree guard.
