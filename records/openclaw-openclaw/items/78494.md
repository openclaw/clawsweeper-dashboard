## Summary

Keep this PR open: the reported auto-update bug is valid, but the patch is still incorrect because `undefined` env overrides do not delete inherited gateway service markers, and a non-dry-run updater child can still be blocked by the gateway process-tree preflight.

## What This Changes

The PR adds undefined gateway-service marker entries to the auto-update child env override, adds a regression test for that override shape, and records the change in the changelog.

## Best Possible Solution

Implement a service-safe auto-update handoff that truly removes inherited gateway service state and runs outside the live gateway process ancestry while preserving the manual in-service package-update guard.

## Reproduction Assessment

Yes. Source inspection provides a high-confidence path: managed gateway env contains the marker keys, the auto-update child inherits through `resolveCommandEnv`, and the package update command blocks gateway-service or gateway-ancestry execution.

## Solution Assessment

No. The proposed undefined override is ineffective under the current env merge contract, and the fix also needs to account for the managed-service ancestry preflight before claiming `openclaw update --yes` can install from the gateway auto-updater.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.95

Full review comments:

- **[P1] Delete inherited service markers before spawning:** `src/infra/update-startup.ts:285-286`
  - body: Setting these keys to `undefined` does not remove inherited values. `mergeChildEnv` copies `process.env` first and `assignChildEnvValue` ignores undefined overrides, so the child still receives the gateway markers and trips the service-env guard.
  - confidence: 0.97
- **[P2] Route the package update outside gateway ancestry:** `src/infra/update-startup.ts:285-286`
  - body: Even after real marker deletion, this child is still spawned by the live gateway. Non-dry-run package updates call the managed-service preflight and block when the gateway PID is an ancestor, so the auto-updater still cannot reach package installation safely.
  - confidence: 0.9
