## Summary

Keep open: the env strip is a useful partial change, but the patch still leaves non-dry-run managed-service package updates blocked by the gateway ancestry preflight before the env-marker guard is reached.

## What This Changes

The PR clears gateway service marker env vars for the auto-update child spawn, adds a regression test, and adds a changelog note.

## Best Possible Solution

Implement a service-safe auto-update handoff or explicit trusted auto-update preflight that preserves the manual in-service package-update guard, with regression coverage for both marker env and gateway ancestry plus redacted managed-service proof.

## Reproduction Assessment

Yes, from source inspection and existing tests: a gateway-spawned non-dry-run updater enters the package preflight with the running gateway PID in its ancestor set and exits before package install. I did not run a live macOS/npm managed-service update in this read-only review.

## Solution Assessment

No. Stripping marker env vars is the right narrow fix for the later env guard, but it does not address the earlier gateway ancestry block that still applies to the auto-update child.
