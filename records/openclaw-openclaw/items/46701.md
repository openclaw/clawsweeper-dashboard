## Summary

Keep open. Current main and v2026.5.19 still silently skip allowlist-denied Telegram group senders, while the requested auto-reply is a new Telegram feature with privacy, abuse, wording, and opt-in policy decisions.

## What This Changes

Request an automatic Telegram group reply that tells non-allowlisted senders how to contact an admin and includes their Telegram user ID.

## Best Possible Solution

Add a Telegram-owned, explicit opt-in unauthorized-group-sender notice that is topic-aware, rate-limited or deduplicated, configurable for admin contact/template and ID disclosure, documented, and covered by focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence in the current behavior: allowlist-denied Telegram group messages return before visible reply dispatch, and existing tests expect zero replies.

## Solution Assessment

No for an unconditional public auto-reply. The safer maintainable shape is an explicit opt-in notice with cooldowns, topic-safe delivery, configurable wording/admin contact, and documented privacy behavior.
