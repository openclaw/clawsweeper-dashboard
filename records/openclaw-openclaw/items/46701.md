## Summary

Keep open: current main and the latest release still drop non-allowlisted Telegram group senders without the requested onboarding auto-reply, and related open issues/PRs cover audit visibility, group pairing, passive hooks, or DM pre-auth rather than this exact visible group-sender notice. The remaining blocker is maintainer product/security direction on opt-in defaults, rate limiting, user-ID disclosure, and message/admin-contact configuration.

## What This Changes

The issue requests an opt-in Telegram group auto-reply for non-allowlisted senders that explains authorization and includes the sender's Telegram user ID for admin approval.

## Best Possible Solution

Define a Telegram-owned explicit opt-in unauthorized-group-sender notice that is topic-aware, rate-limited, privacy-documented, configurable for safe wording/admin contact, and leaves silent fail-closed behavior as the default unless maintainers choose otherwise.

## Reproduction Assessment

Yes, at source level: a Telegram group sender not matching groupAllowFrom reaches group-policy-allowlist-unauthorized and the handler returns before reply dispatch. Existing tests also assert no reply for this blocked group-sender path.

## Solution Assessment

Unclear as requested: the auto-reply addresses a real onboarding cliff, but making it default would be risky. The safer direction is an explicit, rate-limited Telegram policy with documented privacy behavior, or a maintainer decision to prefer owner/audit notification instead.
