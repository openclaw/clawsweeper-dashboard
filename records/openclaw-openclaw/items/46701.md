## Summary

Keep open. Current main still silently drops ordinary Telegram group messages from senders rejected by `groupPolicy`/`groupAllowFrom`, and changing that into a visible onboarding reply needs explicit Telegram product, privacy, and spam-control decisions rather than conservative cleanup closure.

## What This Changes

Request that Telegram group messages from senders outside `groupAllowFrom` receive an authorization/admin-contact reply that includes the sender's Telegram user ID.

## Best Possible Solution

Add a Telegram-owned, explicit opt-in unauthorized-group-sender notice with configurable admin instructions, safe user-ID wording, topic-aware delivery, dedupe or rate limiting, and focused docs/tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives a high-confidence current-main behavior trace: allowlist-denied Telegram group senders are dropped before ordinary reply dispatch, and tests still expect zero replies.

## Solution Assessment

No. The requested default public reply is not yet the safest shape; the maintainable path is an explicit opt-in notice with rate limiting or dedupe, topic-safe delivery, and documented privacy behavior.
