## Summary

Keep open: current main and v2026.6.8 still silently reject ordinary Telegram group messages from non-allowlisted senders, and the useful onboarding reply needs maintainer security/product design before implementation.

## What This Changes

Requested Telegram behavior: reply to non-allowlisted group senders with authorization instructions and their Telegram user ID instead of silently dropping the message.

## Best Possible Solution

Add a Telegram-owned, explicit opt-in unauthorized-group-sender notice with rate limiting, topic-aware replies, privacy-safe sender ID handling, docs, and focused tests while preserving current fail-closed authorization defaults.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: groupPolicy allowlist rejection returns before normal inbound processing, and current tests expect zero replies for non-allowlisted group senders.

## Solution Assessment

Unclear until maintainers choose the security/product shape. The requested behavior should not be an unconditional default; a safer design is an explicit Telegram-owned opt-in with rate limiting and documented privacy tradeoffs.
