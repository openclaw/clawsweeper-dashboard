## Summary

Keep open: current main and v2026.6.8 still skip Telegram group messages from senders rejected by `groupPolicy`/`groupAllowFrom` without the requested onboarding reply, and the remaining work needs a maintainer product/security decision rather than cleanup closure.

## What This Changes

Request an automatic Telegram group reply that tells non-allowlisted senders how to contact an admin and includes their Telegram user ID.

## Best Possible Solution

Define an explicit opt-in Telegram-owned unauthorized-group-sender notice that is thread-aware, rate-limited, privacy-conscious, documented, and tested separately from existing fail-closed authorization behavior.

## Reproduction Assessment

Yes for current behavior at source level: unmatched Telegram group senders reach the allowlist-unauthorized branch and return before reply dispatch. This is a feature request rather than a broken documented reply contract.

## Solution Assessment

Unclear as requested: replacing silent drops with a default visible reply is not the safest shape. The better path is an explicit, rate-limited opt-in notice with privacy and admin-contact behavior decided by maintainers.
