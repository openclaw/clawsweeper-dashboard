## Summary

Keep open. Current main and the latest release still silently deny Telegram group messages from non-allowlisted senders, and the requested auto-reply remains a new security-sensitive Telegram feature rather than an implemented bug fix or a duplicate of the broader audit-alert request.

## What This Changes

The issue requests a Telegram group authorization notice that replies to non-allowlisted group senders with admin-contact guidance and their Telegram user ID.

## Best Possible Solution

Add a Telegram-owned, explicit opt-in unauthorized-group-sender notice with topic-safe delivery, cooldown or dedupe, configurable admin-contact/template and ID disclosure, docs, and focused tests while keeping silent denial as the default.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. The current behavior is still source-reproducible: allowlist-denied Telegram group messages return before visible reply dispatch, and the current tests expect zero replies.

## Solution Assessment

No for an unconditional public auto-reply. The maintainable shape is an explicit opt-in notice with cooldowns, topic-aware delivery, configurable wording/admin contact, and documented privacy behavior.
