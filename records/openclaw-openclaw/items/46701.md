## Summary

Keep open: current main and v2026.6.10 still drop Telegram group messages from non-allowlisted senders without the requested onboarding reply, and the related open pre-auth PR covers blocked DMs/operator notification rather than group sender auto-replies. The remaining decision is product/security policy around opt-in behavior, rate limiting, public ID disclosure, and admin-contact wording.

## What This Changes

The issue requests a Telegram group auto-reply for non-allowlisted senders that explains authorization and includes the sender's Telegram user ID for the group admin.

## Best Possible Solution

Define a Telegram-owned opt-in unauthorized-group-sender notice that is topic-aware, rate-limited, privacy-documented, configurable enough for admin-contact wording, and leaves silent fail-closed behavior as the default unless maintainers explicitly choose otherwise.

## Reproduction Assessment

Yes, at source level: a Telegram group message from a sender not matching groupAllowFrom reaches the group-policy-allowlist-unauthorized branch and exits before reply dispatch. Existing tests also assert zero replies for blocked group senders.

## Solution Assessment

Unclear as requested: a visible auto-reply solves the onboarding cliff, but making it default would be risky. The safer maintainable shape is an explicit, rate-limited Telegram policy with documented privacy behavior, or a maintainer decision to prefer audit/operator notification instead.
