## Summary

Keep open. Current main still silently skips ordinary Telegram group messages rejected by `groupPolicy`/`groupAllowFrom`, and changing that into a visible onboarding reply needs maintainer product and privacy/spam decisions rather than conservative cleanup closure.

## What This Changes

Request that Telegram group messages from non-allowlisted senders receive an onboarding reply with the sender’s Telegram user ID and admin-contact instructions.

## Best Possible Solution

Add a Telegram-owned, explicit opt-in unauthorized-group-sender notice with configurable admin instructions, safe user-ID wording, topic-aware delivery, rate limiting or dedupe, and focused docs/tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection still gives a high-confidence current-main behavior trace: allowlist-denied Telegram group senders are dropped before reply dispatch, and tests expect zero replies.

## Solution Assessment

No. The requested default public reply is not yet the safest shape; the maintainable path is an explicit opt-in notice with rate limiting, topic-safe delivery, and documented privacy behavior.
