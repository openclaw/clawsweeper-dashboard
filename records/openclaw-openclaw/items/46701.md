## Summary

Keep open. Current main still silently skips ordinary Telegram group messages from non-allowlisted senders, and the requested onboarding reply needs product and security decisions before it is a safe implementation task.

## What This Changes

Request an automatic Telegram group reply that tells unauthorized senders to contact an admin and includes their Telegram user ID.

## Best Possible Solution

Add a Telegram-owned, explicit opt-in unauthorized-group-sender notice that is topic-aware, rate-limited or deduped, configurable, privacy-documented, and covered by ordinary group-message tests.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence current-main trace: allowlist-rejected Telegram group messages return before normal processing, and tests currently expect zero replies.

## Solution Assessment

No. The requested default public auto-reply is not yet the safest shape; the maintainable path is an explicit opt-in notice with rate limiting, topic-safe delivery, and documented ID-disclosure behavior.
