## Summary

Keep open. Current main still drops ordinary Telegram group messages from senders rejected by `groupPolicy`/`groupAllowFrom`, and turning that into a visible onboarding reply needs maintainer product and privacy/spam decisions rather than conservative cleanup closure.

## What This Changes

Request Telegram group allowlist denials to reply with admin-contact instructions and the sender's Telegram user ID instead of silently dropping the message.

## Best Possible Solution

Add a Telegram-owned, explicit opt-in unauthorized-group-sender notice with configurable admin instructions, safe user-ID wording, topic-aware delivery, rate limiting or dedupe, and focused docs/tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives a high-confidence current-main behavior trace: allowlist-denied Telegram group senders are dropped before ordinary reply dispatch, and tests expect zero replies.

## Solution Assessment

No. The requested default public reply is not yet the safest shape; the maintainable path is an explicit opt-in notice with rate limiting, topic-safe delivery, and documented privacy behavior.
