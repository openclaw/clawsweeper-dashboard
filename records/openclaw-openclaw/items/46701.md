## Summary

Keep open. Current main still silently skips ordinary Telegram group messages from senders rejected by `groupPolicy`/`groupAllowFrom`, while the requested visible onboarding reply is a new Telegram behavior that needs product and security decisions before implementation.

## What This Changes

Request Telegram to send an onboarding reply with the sender's user ID when a group sender is rejected by `groupAllowFrom` or `groupPolicy`.

## Best Possible Solution

Add a Telegram-owned, explicit opt-in unauthorized group-sender notice with rate limiting, topic-aware delivery, admin-contact/template configuration, privacy documentation, and focused tests after maintainers approve the product/security contract.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence that current main still silently skips ordinary allowlist-rejected Telegram group messages before normal reply delivery.

## Solution Assessment

No, not as a default hardcoded reply. The maintainable direction is an explicit opt-in notice with rate limiting and privacy-aware configuration rather than changing every unauthorized group skip into a visible response.
