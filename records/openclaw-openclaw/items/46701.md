## Summary

Keep open. Current main still silently drops ordinary Telegram group messages from senders denied by groupAllowFrom/groupPolicy, while the request and follow-up discussion add product/security decisions around opt-in behavior, rate limiting, admin wording, and public user-ID disclosure.

## What This Changes

Request an opt-in Telegram group onboarding reply that tells allowlist-denied senders how to contact an admin with their Telegram user ID.

## Best Possible Solution

Add a Telegram-owned, explicit opt-in unauthorized-group-sender notice that is topic-aware, deduplicated or rate-limited, configurable for admin instructions and user-ID disclosure, documented, and covered by focused Telegram group-auth tests.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. Source inspection gives high-confidence current-main behavior: allowlist-denied Telegram group messages return before reply delivery, and tests still expect zero replies.

## Solution Assessment

No, not as the requested default public auto-reply. The safer maintainable path is an explicit opt-in notice with cooldowns, topic-safe delivery, configurable admin wording, and documented privacy behavior.
