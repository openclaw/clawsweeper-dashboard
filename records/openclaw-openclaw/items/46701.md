## Summary

Keep open: current main and v2026.6.6 still block ordinary Telegram group messages from senders rejected by groupPolicy/groupAllowFrom without the requested onboarding reply, and adding that reply remains a Telegram product/security decision.

## What This Changes

Request an automatic Telegram group reply that tells unauthorized senders how to contact an admin and includes their Telegram user ID.

## Best Possible Solution

Add a Telegram-owned, explicit opt-in denied-group-sender notice that is topic-aware, rate-limited, privacy-conscious about IDs, documented, and tested while keeping silent fail-closed behavior as the default.

## Reproduction Assessment

Yes, source-level: configure Telegram with groupPolicy allowlist and a non-matching groupAllowFrom/allowFrom sender, and the handler returns the group-policy-allowlist-unauthorized skip path before reply dispatch. I did not run a live Telegram scenario in this read-only review.

## Solution Assessment

Unclear as requested: a default auto-reply is not the safest solution because it can amplify spam and disclose identifiers. The maintainable path is an explicit opt-in notice with rate limiting, topic-safe delivery, and documented privacy behavior.
