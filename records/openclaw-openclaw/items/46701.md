## Summary

Keep open. Current main still drops allowlist-denied Telegram group senders before any visible reply path, and the later issue discussion supports the onboarding need while adding rate-limit and opt-out concerns that require maintainer product/security choices.

## What This Changes

Request that Telegram group messages from non-allowlisted senders receive an onboarding reply with admin contact instructions and the sender's Telegram user ID instead of being silently dropped.

## Best Possible Solution

Add a Telegram-owned, explicit opt-in unauthorized-group-sender notice with configurable admin instructions, safe user-ID wording, topic-aware delivery, cooldown or dedupe, docs, and focused regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives a high-confidence current-main trace: allowlist-denied Telegram group messages return before visible reply dispatch, and tests still expect zero replies.

## Solution Assessment

No. The requested default public reply is useful but not yet the safest maintainable shape; the better path is an explicit opt-in notice with rate limiting, topic-safe delivery, and documented privacy behavior.
