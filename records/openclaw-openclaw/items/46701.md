## Summary

Keep open. Current main still drops Telegram group messages from senders rejected by groupPolicy/groupAllowFrom before any visible reply path, and the requested auto-reply needs maintainer product/security choices around opt-in behavior, rate limiting, admin-contact wording, and public user-ID disclosure.

## What This Changes

Request auto-replies for unauthorized Telegram group senders that explain how to contact an admin and include the sender's Telegram user ID.

## Best Possible Solution

Add a Telegram-owned, explicit opt-in unauthorized-group-sender notice with admin-contact/template config, topic-aware delivery, safe ID disclosure, cooldown or dedupe, docs, and focused tests, coordinated with the broader denial-hook/audit work.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high-confidence current behavior: allowlist-denied Telegram group messages return before visible reply dispatch, and existing tests expect zero replies.

## Solution Assessment

No. The requested unconditional public auto-reply is useful for onboarding, but the safer maintainable shape is an explicit opt-in notice with cooldowns, topic-safe delivery, configurable wording/admin contact, and documented privacy behavior.
