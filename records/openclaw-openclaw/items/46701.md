## Summary

Keep open. Current main still drops ordinary Telegram group messages from senders rejected by `groupPolicy`/`groupAllowFrom` before reply delivery, and the requested auto-reply needs explicit maintainer choices on opt-in scope, public user-ID disclosure, admin-contact wording, and rate limiting.

## What This Changes

Request that Telegram group messages rejected by `groupAllowFrom`/`groupPolicy` receive a friendly admin-contact reply including the sender's Telegram user ID instead of being silently dropped.

## Best Possible Solution

Add a Telegram-owned, explicit opt-in unauthorized-group-sender notice with per-group/topic settings, privacy-safe sender-ID templating, admin-contact/message configuration, cooldowns, docs, and focused tests.

## Reproduction Assessment

Yes. A Telegram group message from sender `999999` under `groupPolicy: "allowlist"` with only `123456789` allowed reaches `group-policy-allowlist-unauthorized`, returns before delivery, and the existing table test expects zero replies; I did not run tests because this review is read-only.

## Solution Assessment

No, not as the unconditional sample behavior. The maintainable solution is an opt-in, rate-limited Telegram config surface with privacy-safe defaults rather than always posting a sender's numeric ID into the group.

## Security Review

Status: needs_attention

Summary: The requested behavior is security-sensitive because it changes a silent access-control denial into a public group reply.

Concerns:

- **[medium] Avoid public Telegram user ID disclosure by default:**
  - body: The sample reply would post the sender's numeric Telegram user ID into a group chat. That can be useful when deliberately configured, but it should not become an unconditional public default.
  - confidence: 0.88
- **[medium] Rate-limit unauthorized group notices:**
  - body: Replying to every rejected group message can spam group chats and reveal allowlist behavior to probing. A safe implementation should be opt-in and deduped or rate-limited.
  - confidence: 0.9
