## Summary

Keep open. Current main and v2026.5.20 still silently drop Telegram group messages from senders denied by groupPolicy/groupAllowFrom, and the requested auto-reply needs product and security decisions before implementation.

## What This Changes

Request an optional Telegram group denial reply that tells unauthorized senders how to contact an admin and includes their Telegram user ID.

## Best Possible Solution

Add a Telegram-owned, explicit opt-in unauthorized-group-sender notice with topic-aware delivery, rate limiting or dedupe, configurable admin-contact/template and raw-ID disclosure, docs, and focused tests while keeping silent denial as the default.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence in the current behavior: allowlist-denied Telegram group messages return before visible reply dispatch, and current tests expect zero replies.

## Solution Assessment

No for an unconditional public auto-reply. The maintainable shape is an explicit opt-in notice with cooldowns, topic-safe delivery, configurable wording/admin contact, and documented privacy behavior.
