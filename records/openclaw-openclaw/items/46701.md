## Summary

Keep open. Current main still drops Telegram group messages from senders rejected by groupPolicy/groupAllowFrom before any visible reply path, while the issue discussion confirms the onboarding need and also raises rate-limit, opt-out, and privacy concerns that need maintainer product/security decisions.

## What This Changes

The issue requests a Telegram auto-reply for group senders rejected by groupAllowFrom/groupPolicy that tells them to contact the group admin and includes their Telegram user ID.

## Best Possible Solution

Add a Telegram-owned, explicit opt-in unauthorized-group-sender notice with admin-contact/template configuration, safe user-ID disclosure, topic-aware delivery, cooldown or dedupe, docs, and focused regression tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives a high-confidence current-main trace: allowlist-denied Telegram group messages return before visible reply dispatch, and tests still expect zero replies.

## Solution Assessment

No. The requested default public auto-reply is useful but not yet the safest maintainable shape; the better path is an explicit opt-in notice with cooldowns, topic-safe delivery, and documented privacy behavior.
