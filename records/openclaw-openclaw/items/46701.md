## Summary

Keep open: current `main` and latest release still silently deny ordinary Telegram group messages from senders rejected by `groupPolicy`/`groupAllowFrom`, and a sender-facing auto-reply needs a maintainer product/security decision rather than cleanup closure.

## What This Changes

Request an opt-in Telegram group auto-reply for non-allowlisted senders with admin contact and Telegram user ID instructions.

## Best Possible Solution

Define a Telegram-owned unauthorized-group notice that is explicit opt-in, topic-aware, rate-limited, privacy-conscious, documented, and tested separately from existing fail-closed authorization behavior.

## Reproduction Assessment

Yes, at source level: an unauthorized group sender reaches `group-policy-allowlist-unauthorized`, `shouldSkipGroupMessage` returns true, and the handler exits before reply dispatch. I did not run a live Telegram deployment in this read-only review.

## Solution Assessment

No as a default replacement for silent denial: the safer solution is an explicit, rate-limited opt-in notice with privacy and admin-contact behavior chosen by maintainers.
