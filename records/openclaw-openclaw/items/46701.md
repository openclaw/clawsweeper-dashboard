## Summary

Keep open. Current main still silently skips ordinary Telegram group messages from senders rejected by `groupPolicy`/`groupAllowFrom`, and the requested auto-reply needs maintainer product/security decisions before implementation.

## What This Changes

The issue requests a Telegram group auto-reply for senders rejected by `groupAllowFrom`/`groupPolicy`, including instructions to contact a group admin with their Telegram user ID.

## Best Possible Solution

Add a Telegram-owned, explicit opt-in unauthorized-group-sender notice that preserves fail-closed trigger authorization, is topic-aware, deduped or rate-limited, has configurable admin instructions, documents the privacy tradeoff, and includes focused tests for ordinary group messages.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence current-main trace: unmatched Telegram group senders are rejected before ordinary message processing, and tests currently expect zero replies for that case.

## Solution Assessment

No. The requested default public reply is not yet the safest shape; the maintainable path is an explicit opt-in notice with rate limiting/dedupe, topic-safe delivery, and documented privacy behavior.
