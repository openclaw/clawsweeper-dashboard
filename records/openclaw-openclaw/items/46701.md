## Summary

Keep open. Current main and the latest release still silently drop allowlist-denied Telegram group messages, while the requested reply is a feature/security policy change that needs opt-in behavior, cooldowns, user-ID disclosure, and admin-contact design.

## What This Changes

Request that Telegram send an onboarding denial reply with the sender's user ID when a group message comes from a user blocked by groupAllowFrom/groupPolicy.

## Best Possible Solution

Add a Telegram-owned, explicit opt-in unauthorized-group-sender notice that is topic-aware, deduplicated or rate-limited, configurable for admin-contact/template and ID disclosure, documented, and covered by focused tests.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives high confidence in the current behavior: allowlist-denied Telegram group messages return before visible reply dispatch, and existing tests expect zero replies.

## Solution Assessment

No. The requested unconditional public auto-reply is useful for onboarding, but the safer maintainable shape is an explicit opt-in notice with cooldowns, topic-safe delivery, configurable wording/admin contact, and documented privacy behavior.
