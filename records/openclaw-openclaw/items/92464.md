## Summary

Current main still has a concrete lifecycle gap worth fixing: one-shot CLI teardown closes cached memory managers, but the reported 2026.6.1 workload reproduces after that shipped fix, and local embeddings use a separately forked child whose shutdown depends on its owning provider being closed. No canonical issue or open fix PR was found, so this remains necessary.

## What This Changes

Fix `openclaw agent --local --json` so completed turns exit promptly and do not leave embedding or agent worker descendants alive.

## Best Possible Solution

Make each one-shot local memory or harness resource owner deterministically await provider and child shutdown, while preserving cached resources for long-lived gateway operation and avoiding forced `process.exit()` or unref masking.

## Reproduction Assessment

Yes, at high confidence from source and shipped-version evidence: the report supplies an exact released Docker workload, and current code exposes a forked embedding worker whose exit depends on propagated close ownership. This review did not execute the costly 10,000-entry live workload.

## Solution Assessment

Yes in direction: repairing deterministic resource-owner teardown with focused exit and descendant checks is the narrowest maintainable fix; forcing process exit or documenting runner kill workarounds would hide leaked resources.
