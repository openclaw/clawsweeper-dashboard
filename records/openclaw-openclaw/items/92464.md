## Summary

The report targets a shipped lifecycle path that remains materially unchanged on current main: short-lived CLI cleanup calls the memory runtime, and memory managers can close local embedding workers, but existing coverage only proves mocked cleanup dispatch rather than prompt process exit with zero surviving children. No merged or open canonical fix was found, so this should remain open for a focused lifecycle regression and repair.

## What This Changes

Fix `openclaw agent --local --json` so completed local turns exit promptly and do not leave local embedding or agent worker descendants alive.

## Best Possible Solution

Keep cleanup at the existing root CLI and plugin-owned manager/provider boundaries, add a real child-process regression proving prompt post-JSON exit and zero surviving descendants, and fix only the lifecycle link that the regression exposes.
