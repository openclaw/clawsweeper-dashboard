## Label Justifications

- `P1`: On affected releases, a completed local agent turn can still occupy runner capacity and accumulate worker memory until externally terminated.
- `impact:crash-loop`: The report concerns a process hang and surviving child workers, which is a process-level availability failure.

## Summary

Current main now performs centralized short-lived CLI teardown after command parsing, closes the active memory managers, disposes registered agent harnesses, and the local embedding provider’s close path disconnects and kills its child worker. This directly addresses the v2026.6.1 behavior described here; the implementation is current-main-only and is not contained in the latest v2026.6.6 tag.

## What This Changes

Ensure `openclaw agent --local --json` exits promptly after final output and does not leave invocation-owned workers running.

## Best Possible Solution

Keep the centralized current-main CLI teardown and include process-level local-memory exit/survivor proof in release validation before shipping it.

## Reproduction Assessment

Yes at source level for the affected release shape: local memory managers can own child embedding workers, and the report provides a concrete scale-sensitive v2026.6.1 command sequence. Current main adds the missing outer CLI cleanup, so the original failure is no longer expected there.

## Solution Assessment

Yes. Centralized CLI-finally teardown is the narrowest maintainable fix because it covers success and error exits without forcing `process.exit` or duplicating cleanup inside the local agent command.
