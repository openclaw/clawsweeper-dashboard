## Summary

This issue remains necessary: the shipped QMD one-shot repair prevents CLI-mode QMD background sync, but this report exercises `agent --local --json` with large memory indexing and also reports surviving embedding workers, so neither the prior generic teardown nor the narrower QMD fix proves the current path exits cleanly.

## What This Changes

Fix local JSON agent runs that emit their final response but retain the CLI process or child workers under large memory workloads.

## Best Possible Solution

Add a process-level current-main regression that runs the local JSON agent path, captures active handles and descendant processes after final output, and repair the specific manager, embedding provider, or harness owner so normal awaited teardown exits promptly with zero survivors.

## Reproduction Assessment

No current-main live reproduction was completed in this read-only review, but the report provides a detailed v2026.6.1 procedure and current source exposes the exact cleanup and provider lifecycle boundaries to instrument. The narrower QMD reproduction does not cover this command or its embedding descendants.

## Solution Assessment

Yes, owner-driven deterministic shutdown with process and descendant proof is the best fix. A command-specific `process.exit()` would conceal leaked resources and can bypass output flushing or cleanup.
