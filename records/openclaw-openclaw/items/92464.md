## Summary

This remains a concrete P1 CLI lifecycle bug with an exact released-image reproduction, and current main still lacks provider-auth warm-worker cleanup in the full CLI finalizer. The linked open PR targets that gap, but it has not run the scale-sensitive real workload or shown prompt process exit and zero surviving descendants, so the issue should stay open until that PR is proven and merged or replaced.

## What This Changes

Ensure `openclaw agent --local --json` exits promptly after final output and does not leave agent, provider-auth, memory, or embedding workers alive.

## Best Possible Solution

Keep all process-current local-agent resource disposal in the shared full-CLI finalizer, then prove the patched real command exits within a defined grace period and leaves no surviving worker processes under the reported workload or a smaller deterministic active-worker equivalent.

## Reproduction Assessment

Yes, at high confidence from the report: run the specified 2026.6.1 Docker workload, record final JSON completion, wait the stated 15-second grace period, and inspect the process tree for survivors. This read-only review did not execute that large workload, so the status is source-reproducible rather than independently reproduced.

## Solution Assessment

The shared full-CLI finalizer is the narrowest maintainable owner boundary, but the provider-auth-only addition is not yet proven to be the complete fix because the report also implicates memory or embedding-style workers.
