The PR adds `OPENCLAW_INCLUDE_ROOTS` parsing, threads approved include roots through config include reads and write-preparation env-reference collection, documents the env var, updates `.env.example` and `CHANGELOG.md`, and adds include/path regression tests.

## Best Possible Solution

Land the feature after normal maintainer approval and exact-head merge gates, preserving the explicit env allowlist, lexical containment check, realpath recheck, and guarded include reader. External-root write-through should remain a separate product decision unless intentionally expanded later.

## Solution Assessment

Yes. Layering an operator-controlled env allowlist onto the existing include resolver and guarded file-read path is the narrowest maintainable design for this request; the PR avoids changing default confinement or broadening write-through semantics.
