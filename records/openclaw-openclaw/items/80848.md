## Summary

Keep open: this is an active, focused Telegram bug-fix PR. Current main still allows pre-session Telegram cache entries into selected prompt context, while the PR adds a narrow session-boundary filter with focused tests and sufficient real replay proof; I found no blocking functional or security finding, but merge conflict/normal maintainer checks remain.

## What This Changes

The PR filters Telegram conversation context by persisted session start and cached reset boundaries, threads that timestamp through inbound buffering, adds regression tests, and updates the changelog.

## Best Possible Solution

Land the targeted Telegram context-boundary fix after conflict resolution, CI, and maintainer review; leave broader live Telegram QA or custom reset-trigger refinements as follow-up only if maintainers want them.

## Reproduction Assessment

Yes. Source inspection shows current main selects Telegram cache entries for prompt context without a session boundary, and a focused seeded-cache replay can exercise stale pre-session topic messages reaching Conversation context; I did not execute tests in this read-only review.

## Solution Assessment

Yes. Using persisted sessionStartedAt as the primary boundary and cached /new or non-soft /reset as a secondary guard is the narrowest maintainable fix in the Telegram-owned selector and ingestion plumbing.
