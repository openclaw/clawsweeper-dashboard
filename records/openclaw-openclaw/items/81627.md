## Summary

Keep this open: current main has generic transient-network and benign-uncaught-exception handling, but the reported `ERR_HTTP2_INVALID_SESSION` signature is not covered, so an escaping uncaught instance still follows the fatal Gateway exit path.

## What This Changes

A user reports the Gateway exiting during active model calls when Node/undici raises uncaught `ERR_HTTP2_INVALID_SESSION` from a destroyed HTTP/2 session.

## Best Possible Solution

Contain destroyed HTTP/2 session failures at the OpenClaw-owned model-call/undici boundary so the Gateway stays alive and the affected call fails or retries with regression coverage.

## Reproduction Assessment

No, not as a high-confidence end-to-end Gateway crash repro. The exact Node HTTP/2 error signature is reproducible locally, and current-main source shows an uncaught instance would miss the benign classifier and take the fatal exit path.

## Solution Assessment

Unclear as a concrete patch, but the requested behavior is the right direction. The narrowest maintainable fix should first contain the error at the model-call/undici request boundary, then add a defensive classifier only if the dependency exception can still escape.
