## Summary

This should stay open: current main has generic transient-network and benign-uncaught-exception handling, but the reported `ERR_HTTP2_INVALID_SESSION` signature is not covered, and the embedded-attempt undici global dispatcher can still create a direct Agent without the repo's HTTP/1-only hardening.

## What This Changes

The issue asks for destroyed undici HTTP/2 sessions during model calls to fail or retry without crashing the gateway.

## Best Possible Solution

Keep the gateway process alive by containing destroyed HTTP/2 session failures at the OpenClaw-owned undici/model-call boundary, with regression coverage for the reported error signature.

## Reproduction Assessment

No, not as a full gateway crash repro. Source inspection plus a Node http2 one-liner reproduce the exact `ERR_HTTP2_INVALID_SESSION` signature, and current main would route an uncaught instance to the fatal exit path.

## Solution Assessment

Unclear until the exact escaping path is fixed and tested. The narrowest maintainable direction is to keep OpenClaw-owned undici model-call dispatchers off the unreliable HTTP/2 path and add a defensive uncaught-exception classifier only if the request boundary cannot catch it first.
