## What This Changes

The issue requests that Mattermost channel prompt context stop calling channel turns a `group chat` when the authoritative `chat_type` is `channel`.

## Best Possible Solution

Make the auto-reply group/shared-chat prompt ChatType-aware so `channel` turns use neutral or channel-specific wording while preserving routing and leaving the separate target-inference issue to its own fix.
