## Summary

Keep open: current `main` and latest `v2026.6.9` still send Mattermost public-channel turns through prompt wording that says `group chat` while trusted metadata can say `chat_type: "channel"`. The related Mattermost private-channel report is adjacent, but it does not replace this narrower prompt-intro fix.

## What This Changes

The issue asks OpenClaw to stop describing Mattermost public-channel prompt context as a group chat when the authoritative chat type is `channel`.

## Best Possible Solution

Make `buildGroupChatContext` or its caller use `ChatType`-aware/neutral wording for `channel` while preserving true group-chat guidance and leaving the deeper target-inference work to https://github.com/openclaw/openclaw/issues/95646.

## Reproduction Assessment

Yes, from source: Mattermost `O` channel type maps to `channel`, and `get-reply-run` routes `channel` through `buildGroupChatContext`, which emits the group-chat intro. I did not run a live Mattermost server repro.

## Solution Assessment

Yes, the narrowest maintainable fix is to make the prompt intro ChatType-aware or neutral while preserving shared-chat delivery guidance. The target-string inference problem should remain in the related issue unless maintainers choose to combine the fixes.

## Repair Work Prompt

Fix the Mattermost/public-channel prompt wording bug reported in https://github.com/openclaw/openclaw/issues/95645. Current `src/auto-reply/reply/get-reply-run.ts` treats both `ChatType: "group"` and `ChatType: "channel"` as shared-chat contexts, then `src/auto-reply/reply/groups.ts` always emits `You are in a <provider> group chat.`. Make the intro ChatType-aware or neutral so `channel` does not declare itself as `group`, while preserving existing automatic reply/message-tool guidance and keeping the separate private-channel target inference issue https://github.com/openclaw/openclaw/issues/95646 out of scope unless needed for tests. Add focused coverage for `ChatType: "channel"` and keep existing group/direct prompt behavior stable. Do not add config options, migrations, or CHANGELOG edits.
