## Summary

Keep open: current `main` and the latest release still emit the conflicting group-chat prompt intro for Mattermost channel turns, and the linked fixing PRs are open candidates rather than merged implementations.

## What This Changes

The issue asks OpenClaw to stop describing Mattermost public-channel prompt context as a group chat when trusted metadata says `chat_type: "channel"`.

## Best Possible Solution

Land one narrow prompt-builder fix that makes the opener `ChatType`-aware or neutral for `channel`, with focused prompt tests, while leaving the deeper private-channel inference work to https://github.com/openclaw/openclaw/issues/95646 unless maintainers intentionally combine them.

## Reproduction Assessment

Yes, from source: Mattermost public channels map to `ChatType: "channel"`, `get-reply-run` routes `channel` through `buildGroupChatContext`, and that helper emits the group-chat opener. I did not run a live Mattermost server in this read-only review.

## Solution Assessment

Yes, the narrow prompt wording repair is the best boundary for this issue. The private-channel target-inference problem is adjacent and should remain in https://github.com/openclaw/openclaw/issues/95646 unless maintainers choose a combined fix.
