## Summary

Keep open: current main and the latest release still put `ChatType: "channel"` turns through prompt wording that says `group chat`, and there are open candidate PRs tied to the same issue rather than a merged fix.

## What This Changes

The issue asks OpenClaw to stop describing Mattermost channel conversations as a `group chat` when trusted metadata reports `chat_type: "channel"`.

## Best Possible Solution

Land one focused ChatType-aware prompt wording fix with regression coverage for `ChatType: "channel"`, while keeping the deeper Mattermost target-inference work tracked separately in https://github.com/openclaw/openclaw/issues/95646.

## Reproduction Assessment

Yes. Source inspection shows Mattermost public channels can enter the prompt with `ChatType: "channel"`, while the injected group/channel context still declares a `group chat`; I did not run a live Mattermost server repro.

## Solution Assessment

Yes. The narrow maintainable fix is to make `buildGroupChatContext` or its prompt labels derive the venue wording from authoritative `ChatType` while leaving the deeper target-string inference issue to its own tracked item.
