## What This Changes

The issue asks OpenClaw to stop describing Mattermost public-channel prompt context as a group chat when trusted metadata says `chat_type: "channel"`.

## Best Possible Solution

Review and land one focused linked prompt-wording PR, preferably the current-base candidate at https://github.com/openclaw/openclaw/pull/96186, then close this issue from the merged fix.

## Reproduction Assessment

Yes, from source. Mattermost public channel type `O` maps to `ChatType: "channel"`, `get-reply-run` routes `channel` through `buildGroupChatContext`, and that helper still emits a group-chat opener on current `main`.

## Solution Assessment

Yes. The narrowest maintainable fix is to make the shared prompt opener `ChatType`-aware or neutral while leaving the private-channel target-inference work to https://github.com/openclaw/openclaw/issues/95646.
