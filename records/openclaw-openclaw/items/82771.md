The PR adds a setup-only, guarded, read-only Podman mount for the host Claude CLI credential directory and documents `OPENCLAW_HOST_CLAUDE_DIR`.

## Best Possible Solution

Merge the guarded setup-only credential handoff if maintainers accept the directory-level auth boundary; otherwise narrow the mount to the credential file or choose a different Podman Anthropic onboarding path and re-prove Fedora setup.

## Solution Assessment

Yes, with maintainer signoff. A setup-only, read-only mount guarded by existing private-owner checks is a focused fix for the reported Podman path; the remaining design choice is directory-level exposure versus a narrower credential-file handoff.
