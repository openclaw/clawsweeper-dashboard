## What This Changes

The PR conditionally bind-mounts host Claude CLI credentials read-only into the Podman setup container, allowlists `OPENCLAW_HOST_CLAUDE_DIR`, and documents the setup-only override.

## Best Possible Solution

Merge the guarded setup-only read-only credential handoff if maintainers accept the directory-level credential boundary; otherwise narrow the mount to the minimum credential file and re-prove the Fedora setup path.

## Reproduction Assessment

Yes, by source inspection. Current main's Podman setup container does not mount `/home/node/.claude`, while Anthropic setup reads `$HOME/.claude/.credentials.json` and throws the reported auth error when it is absent.

## Solution Assessment

Yes with maintainer signoff. A setup-only read-only mount guarded by existing private-owner checks is a narrow fix for the reported Podman setup path, while the remaining question is whether directory-level exposure should be accepted or narrowed to the credential file.
