## Summary

Keep open: this targets a valid Podman Claude CLI setup regression, but the PR still needs target-environment proof and credential-mount hardening before merge.

## What This Changes

The PR adds a setup-only Podman bind mount from the host Claude CLI credentials directory into `/home/node/.claude` when `.credentials.json` exists, using a read-only shared SELinux relabel.

## Best Possible Solution

Land the setup-only read-only credential mount after reusing the launcher’s private owner/symlink/permission guards for the Claude dir and file, plus redacted Fedora rootless Podman/SELinux proof or an explicit maintainer proof override.

## Reproduction Assessment

Yes by source inspection: current main sets `HOME=/home/node` for setup, does not mount host `~/.claude`, and the Anthropic setup path reads `HOME/.claude/.credentials.json` before throwing the reported auth error. I did not establish a live Fedora reproduction in this read-only review.

## Solution Assessment

No, not yet. The setup-only mount is the right likely boundary, but the current patch should validate the credential source with existing private-file guards and needs target-environment proof or an explicit maintainer override.
