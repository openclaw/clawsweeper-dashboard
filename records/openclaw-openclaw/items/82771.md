## Summary

Keep this PR open: it targets a real Podman onboarding auth bug and the fix is on the right surface, but it still needs credential-boundary hardening and target-environment proof before merge.

## What This Changes

The PR adds a setup-only Podman bind mount of the host Claude CLI credentials directory into `/home/node/.claude`, gated on `.credentials.json`, with an `OPENCLAW_HOST_CLAUDE_DIR` override and shared SELinux relabel.

## Best Possible Solution

Land the setup-only read-only mount after reusing the launcher's private owner, symlink, and permission checks for the Claude directory and credentials file, with redacted Fedora rootless Podman/SELinux proof or an explicit maintainer proof override.

## Reproduction Assessment

Yes, by source inspection: current main runs setup with `HOME=/home/node`, does not mount host `~/.claude`, and the Anthropic setup path reads `HOME/.claude/.credentials.json` before throwing the reported auth error. I did not run a live Fedora rootless Podman reproduction.

## Solution Assessment

Mostly yes: the Podman setup launcher is the right boundary because gateway runtime already uses migrated OpenClaw credentials, but the PR should harden the host credential source and prove the SELinux/rootless path before merge.
