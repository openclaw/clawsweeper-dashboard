## Best Possible Solution

Merge the guarded setup-only read-only credential handoff if maintainers accept the directory-level credential boundary; otherwise narrow the mount to the minimum credential file and re-prove the Fedora setup path.

## Solution Assessment

Yes with maintainer signoff. A setup-only read-only mount guarded by existing private-owner checks is a narrow fix for the reported Podman setup path, with the remaining choice being directory-level exposure versus a narrower file mount.
