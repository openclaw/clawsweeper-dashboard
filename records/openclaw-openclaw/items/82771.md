## Summary

Keep open: the Podman credential mount is focused and backed by strong Fedora/rootless Podman proof, but the newly documented `OPENCLAW_HOST_CLAUDE_DIR` override is not admitted through the launcher's `~/.openclaw/.env` allowlist, so persisted configuration would be silently ignored.

## What This Changes

This PR adds a setup-only Podman bind mount for host Claude CLI credentials with private path guards and documents the `OPENCLAW_HOST_CLAUDE_DIR` override.

## Best Possible Solution

Keep the setup-only read-only credential mount with private guards, wire the documented override into the launcher env-file contract or narrow the docs, then have maintainers accept the credential-boundary tradeoff.

## Reproduction Assessment

Yes. Current main's setup container does not mount host `~/.claude`, while Anthropic setup reads `.claude/.credentials.json` under the container home and the linked issue plus PR proof describe the Fedora rootless Podman failure path.

## Solution Assessment

Not quite yet. The setup-only read-only mount with private guards is the right shape, but the newly documented override needs to be added to the env-file allowlist or documented as process-env-only before merge.
