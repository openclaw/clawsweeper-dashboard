## Summary

Keep open: the patch is a small, source-aligned fix for the linked Podman setup regression, but it still needs target Fedora/rootless Podman/SELinux proof or an explicit maintainer proof override before merge because it intentionally exposes host Claude CLI credentials to the setup container.

## What This Changes

The PR conditionally bind-mounts the host Claude credentials directory into the Podman setup container so Anthropic Claude CLI onboarding can read existing host credentials.

## Best Possible Solution

Land the bounded setup-only read-only credentials mount after redacted Fedora rootless Podman/SELinux proof, or after an explicit maintainer proof override accepting the source-level proof for this shell-only credential-boundary change.

## Reproduction Assessment

Yes. Current main sets setup `HOME=/home/node`, does not mount `.claude`, and the Anthropic CLI setup path reads `HOME/.claude/.credentials.json` before throwing the reported error when credentials are absent; I did not run the full Fedora/rootless Podman flow live.

## Solution Assessment

Yes, the proposed setup-only read-only mount is the narrow maintainable fix for the source-proven path, and using `CLAUDE_CONFIG_DIR` is not a better fix because OpenClaw intentionally clears that env for managed Claude CLI runs.
