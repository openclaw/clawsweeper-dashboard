The PR adds a setup-only read-only mount of host Claude CLI credentials to `scripts/run-openclaw-podman.sh`, guarded by private-source checks, soft-fail warnings, SELinux `:z` handling, and an `OPENCLAW_HOST_CLAUDE_DIR` override.

## Best Possible Solution

Land the setup-only soft-fail credential mount if maintainers accept the credential exposure model, and either document the override as supported Podman launcher surface or remove it before merge.

## Reproduction Assessment

## Solution Assessment

Yes, with maintainer acceptance of the credential boundary: the patch is a narrow setup-only fix, skips unsafe sources without blocking other providers, and leaves non-setup gateway launches untouched. The safer pre-merge refinement is to decide whether the new override should be documented or removed.
