## Summary

Keep this PR open for maintainer review: the patch is focused, source-backed, and has sufficient real Fedora rootless Podman proof, with no blocking code findings found. The remaining decision is whether OpenClaw should accept a setup-only, read-only host Claude credential directory mount with shared SELinux relabeling for Anthropic Claude CLI onboarding.

## What This Changes

The branch updates the Podman launcher and Podman install docs so setup can mount a privately guarded host Claude CLI credential directory read-only into the setup container, with `OPENCLAW_HOST_CLAUDE_DIR` allowed from the launcher env file.

## Best Possible Solution

Merge the guarded setup-only credential handoff if maintainers accept directory-level exposure and shared SELinux relabeling; otherwise narrow the secret handoff and re-prove Fedora Podman setup.

## Reproduction Assessment

Yes, by source inspection and the PR's live-environment report. Current `main` does not mount `/home/node/.claude` into the setup container, while Anthropic setup reads `$HOME/.claude/.credentials.json` and errors when that credential is absent.

## Solution Assessment

Yes, with maintainer signoff. A setup-only, read-only mount guarded by existing private-owner checks is a focused fix; the remaining design choice is directory-level exposure versus a narrower credential handoff.
