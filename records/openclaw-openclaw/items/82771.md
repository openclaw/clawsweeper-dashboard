## What This Changes

The branch mounts a guarded host Claude CLI credential directory read-only into Podman setup, allows an `OPENCLAW_HOST_CLAUDE_DIR` override from the launcher env file, and documents the setup-only option.

## Best Possible Solution

Merge the guarded setup-only credential handoff if maintainers accept the directory-level auth boundary; otherwise narrow the handoff to the credential file or choose a different Podman Anthropic onboarding path and re-prove Fedora setup.

## Reproduction Assessment

Yes by source inspection. Current `main` does not mount `/home/node/.claude` into Podman setup, while Anthropic setup reads `$HOME/.claude/.credentials.json` and errors when that credential is absent.

## Solution Assessment

Yes, with maintainer signoff. A setup-only, read-only mount guarded by existing private-owner checks is a focused fix for the reported Podman path; the remaining design choice is directory-level exposure versus a narrower credential-file handoff.
