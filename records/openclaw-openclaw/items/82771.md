## Summary

Keep this PR open: the patch is focused, proof-backed, and has no blocking code finding, but it intentionally grants the setup container read-only access to host Claude CLI credentials, so a maintainer should accept or reject that auth/security boundary before merge.

## What This Changes

The branch adds `OPENCLAW_HOST_CLAUDE_DIR` to the Podman launcher env allowlist, mounts a guarded host Claude credential directory read-only during setup only, and documents the override.

## Best Possible Solution

Land the guarded setup-only mount only if maintainers accept the credential-sharing boundary; otherwise require an explicit opt-in or a different Claude CLI credential import flow.

## Reproduction Assessment

Yes. Source inspection shows current `main` runs setup with `HOME=/home/node` and no `/home/node/.claude` mount, while Anthropic CLI setup reads `.claude/.credentials.json`; the PR body also supplies live Fedora rootless Podman proof of the before/after path.

## Solution Assessment

Unclear until maintainer review. The patch is the narrowest direct fix for the container visibility bug, but the automatic setup-only credential mount is an auth/security policy choice; explicit opt-in would be safer if maintainers do not want that default boundary.
