This PR adds a setup-only Podman bind mount for the host Claude CLI credential directory, documents `OPENCLAW_HOST_CLAUDE_DIR`, and allows that override in the launcher env-file allowlist.

## Best Possible Solution

Merge the guarded setup-only credential handoff if maintainers accept the directory-level secret exposure and shared SELinux relabeling; otherwise narrow the credential source and re-prove Fedora Podman setup.

## Solution Assessment

Yes, with maintainer signoff. A setup-only, read-only mount guarded by existing private-owner checks is a focused fix; the remaining policy choice is directory-level exposure versus a narrower credential handoff.
