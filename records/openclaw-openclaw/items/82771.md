## Summary

Keep open for maintainer review: the patch appears focused and correct with sufficient Fedora rootless Podman/SELinux proof, but it intentionally changes the Anthropic Podman setup auth boundary by handing the setup container read-only access to host Claude CLI credentials.

## What This Changes

The branch conditionally mounts a privately guarded host Claude CLI credential directory read-only into the Podman setup container, allowlists `OPENCLAW_HOST_CLAUDE_DIR` in the launcher env file, and documents the setup-only override.

## Best Possible Solution

Merge the guarded setup-only credential handoff if maintainers accept directory-level credential exposure; otherwise narrow the handoff to the credential file or choose a different Anthropic Podman onboarding path and re-prove Fedora setup.

## Reproduction Assessment

Yes by source inspection. Current main does not mount `/home/node/.claude` into Podman setup, while Anthropic setup reads `$HOME/.claude/.credentials.json` and errors when that credential is absent.

## Solution Assessment

Yes, with maintainer signoff. A setup-only, read-only mount guarded by existing private-owner checks is a focused fix for the reported Podman path; the remaining design choice is directory-level exposure versus a narrower credential-file handoff.
