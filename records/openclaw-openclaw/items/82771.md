## Summary

Keep open: the PR is a focused, plausible fix for the linked Podman Claude CLI setup bug, and I found no blocking line-level defect, but the credential mount is an auth/security boundary that still needs maintainer acceptance before merge.

## What This Changes

The branch adds a setup-only, read-only Podman bind mount for host Claude CLI credentials, allows `OPENCLAW_HOST_CLAUDE_DIR` in the launcher env file, and documents the override.

## Best Possible Solution

Merge the focused script/docs fix only if Podman/auth owners accept setup-only read access to Claude credentials and the shared SELinux relabel; otherwise replace it with an explicit opt-in credential setup path.

## Reproduction Assessment

Yes. Current main sets the setup container home to `/home/node` without mounting `.claude`, while the Anthropic setup path reads `.claude/.credentials.json` from the process home and throws the reported error when absent.

## Solution Assessment

Yes, conditionally. The patch is narrow and uses existing private-source guards, but the final choice to mount host Claude credentials into setup is a maintainer auth/security policy decision.
