## Summary

Keep open: the linked root cause is well supported by current source, and the one-file setup-only mount is a plausible fix shape, but this PR still needs target Podman/SELinux proof and should harden the host credential source before merge.

## What This Changes

The PR adds a setup-only conditional Podman bind mount from the host Claude CLI credentials directory into `/home/node/.claude`, with a host-dir override and shared SELinux relabeling.

## Best Possible Solution

Land a bounded setup-only read-only credential mount after adding source safety validation and either redacted Fedora rootless Podman/SELinux proof or an explicit maintainer proof override.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes by source inspection: current main sets `HOME=/home/node` in the setup container, does not mount host `~/.claude`, and the Anthropic setup path reads `HOME/.claude/.credentials.json` before throwing the reported auth error; I did not establish a live Fedora reproduction in this read-only review.

## Solution Assessment

Is this the best way to solve the issue? Not yet: the setup-only read-only mount is the right boundary, but the patch should reuse existing private source checks for the credential directory/file and still needs target-environment proof or an explicit override.
