## Summary

Keep open for maintainer review: I found no blocking correctness defect in the current diff, and the supplied Fedora/Podman proof covers the load-bearing setup behavior, but the PR intentionally changes the auth boundary by exposing host Claude CLI credentials to the setup container.

## What This Changes

The branch conditionally bind-mounts the host Claude CLI credential directory into the Podman setup container, allowlists `OPENCLAW_HOST_CLAUDE_DIR`, and documents the override.

## Best Possible Solution

Merge the guarded setup-only credential handoff if maintainers accept the directory-level auth boundary; otherwise narrow the mount to the credential file or choose a different Podman Anthropic onboarding path and re-prove Fedora setup.

## Reproduction Assessment

Yes, by source inspection. Current `main` does not mount `/home/node/.claude` into the Podman setup container, while Anthropic setup reads Claude CLI credentials from `$HOME/.claude/.credentials.json` and errors when absent.

## Solution Assessment

Yes with maintainer signoff. A setup-only read-only mount guarded by existing private-owner checks is a narrow fix for the reported Podman setup path, with the main choice being directory-level exposure versus a narrower file mount.
