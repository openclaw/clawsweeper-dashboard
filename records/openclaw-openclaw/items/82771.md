## Summary

Keep open: the PR fixes a real Podman Anthropic setup gap and now includes adequate Fedora/SELinux proof, but the patch still runs Claude credential validation before the setup-only branch, so ordinary gateway launches can fail on a host credential file they do not mount or use.

## What This Changes

The PR adds a conditional read-only host `~/.claude` bind mount, private source validation, and an `OPENCLAW_HOST_CLAUDE_DIR` override for Podman setup onboarding.

## Best Possible Solution

Keep the setup-only read-only mount, but construct and validate Claude mount args only inside the setup path so ordinary gateway launch never inspects host Claude credentials.

## Reproduction Assessment

Yes, source-reproducible: current main runs Podman setup with `HOME=/home/node`, does not mount host `~/.claude`, and the Anthropic setup path reads `~/.claude/.credentials.json` before throwing the reported auth error. I did not run the full wizard in this read-only review.

## Solution Assessment

No, not as currently placed: the read-only setup mount is the right narrow fix, but its discovery and validation must be scoped to setup so normal gateway launch is unchanged.
