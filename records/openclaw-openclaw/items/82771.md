## Summary

Keep this PR open. The patch is a focused fix for a real Podman onboarding auth gap, and I found no blocking code-review findings, but it still needs maintainer acceptance of the setup-container credential boundary and shared SELinux relabeling before merge.

## What This Changes

The PR adds a setup-only, read-only host Claude CLI credential mount for the Podman launcher, allows `OPENCLAW_HOST_CLAUDE_DIR` from the launcher env file, and documents that override.

## Best Possible Solution

Merge the focused script/docs fix only if Podman/auth owners accept setup-only read access to Claude credentials and the shared SELinux relabel; otherwise replace it with an explicit opt-in credential setup path.

## Reproduction Assessment

Yes. The current-main setup container runs with `HOME=/home/node` and no `.claude` mount while the Anthropic setup path reads `.claude/.credentials.json`, so the reported Fedora rootless Podman failure is source-reproducible even though I did not run the live Podman flow here.

## Solution Assessment

Yes, conditionally. The patch is narrow and uses existing private-source guards, but the final choice to mount host Claude credentials into setup is a maintainer auth/security policy decision.
