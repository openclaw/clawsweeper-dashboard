## What This Changes

The PR conditionally mounts the host Claude CLI credential directory into the Podman setup container, allowlists `OPENCLAW_HOST_CLAUDE_DIR`, and documents the setup-only behavior.

## Best Possible Solution

Merge the guarded setup-only credential handoff if maintainers accept directory-level exposure and shared SELinux relabeling; otherwise narrow the handoff and re-prove Fedora Podman setup.

## Reproduction Assessment

Yes, by source inspection and the linked real-environment report. Current `main` does not mount `/home/node/.claude` into the setup container, while Anthropic setup reads `$HOME/.claude/.credentials.json` and errors when that credential is absent.

## Solution Assessment

Yes, with maintainer signoff. A setup-only, read-only mount guarded by existing private-owner checks is a focused fix; the remaining design choice is directory-level exposure versus a narrower credential-file handoff.
