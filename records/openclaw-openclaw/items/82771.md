## What This Changes

The PR adds a setup-only conditional read-only Podman bind mount from the host Claude CLI credentials directory into `/home/node/.claude`, with an `OPENCLAW_HOST_CLAUDE_DIR` override and shared SELinux relabeling when SELinux mount options are active.

## Best Possible Solution

Land the bounded setup-only read-only credentials mount after redacted Fedora rootless Podman/SELinux proof, or after a maintainer explicitly accepts the source-level proof as sufficient for this shell-only change.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes by source inspection: current main sets `HOME=/home/node` for setup, mounts only OpenClaw state, and the Anthropic setup path reads `HOME/.claude/.credentials.json` before throwing the reported auth error; I did not run Fedora rootless Podman live.

## Solution Assessment

Is this the best way to solve the issue? Yes, the setup-only read-only mount matches the existing credential reader and avoids changing runtime auth; the remaining gap is proof and maintainer acceptance of the credential boundary.
