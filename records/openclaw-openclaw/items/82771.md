## Best Possible Solution

Merge the guarded setup-only credential handoff if maintainers accept the directory-level auth boundary; otherwise narrow the mount to the credential file or choose a different Podman Anthropic onboarding path and re-prove Fedora setup.

## Reproduction Assessment

Yes by source inspection. Current main does not mount `/home/node/.claude` into the Podman setup container, while Anthropic setup reads `$HOME/.claude/.credentials.json` and errors when absent.

## Solution Assessment

Yes, with maintainer signoff. A setup-only, read-only mount guarded by existing private-owner checks is a focused fix for the reported Podman setup path; the remaining design choice is directory-level exposure versus a narrower credential-file handoff.
