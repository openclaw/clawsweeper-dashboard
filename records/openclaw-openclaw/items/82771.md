## What This Changes

The branch adds a setup-only, read-only Podman bind mount for the host Claude CLI credential directory with private-owner guards, allowlists `OPENCLAW_HOST_CLAUDE_DIR` from the launcher env file, and documents the override.

## Best Possible Solution

Merge the guarded setup-only credential handoff if maintainers accept the directory-level auth boundary; otherwise narrow the mount to the minimum credential file or choose a different Podman Anthropic onboarding path and re-prove Fedora setup.

## Reproduction Assessment

Yes, by source inspection. Current main's Podman setup container does not mount `/home/node/.claude`, while Anthropic setup reads `$HOME/.claude/.credentials.json` and throws the reported auth error when absent.

## Solution Assessment

Yes with maintainer signoff. A setup-only read-only mount guarded by existing private-owner checks is a narrow fix for the reported Podman setup path, with the remaining choice being directory-level exposure versus a narrower file mount.
