## What This Changes

The PR adds a setup-only, read-only host Claude CLI credential mount to the Podman launcher with private-source validation, soft-fail warnings, SELinux `:z` handling, and an `OPENCLAW_HOST_CLAUDE_DIR` override.

## Best Possible Solution

Land the setup-only soft-fail mount if maintainers accept the credential exposure model, and document or deliberately omit the new override from the supported Podman env-var surface.

## Reproduction Assessment

Yes, by source inspection: current main sets `HOME=/home/node` for setup, does not mount host `~/.claude`, and the Anthropic CLI setup path reads `/home/node/.claude/.credentials.json` before throwing when it is missing. I did not personally run the full Fedora wizard, but the linked issue and PR proof match this path.

## Solution Assessment

Yes, the current PR shape is a narrow maintainable fix for the reported setup gap: valid credentials are mounted read-only only for setup, unsafe sources are skipped without blocking other providers, and non-setup launches stay untouched. The remaining question is maintainer acceptance of the credential boundary and whether the override should be documented.
