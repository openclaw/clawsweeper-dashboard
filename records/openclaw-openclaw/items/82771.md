## Summary

Keep open: the PR addresses a real source-reproducible Podman setup auth gap, but the proposed script change has merge-blocking issues and lacks after-fix real Podman/SELinux proof.

## What This Changes

The PR adds a setup-only Podman bind mount for a host Claude credentials directory into `/home/node/.claude`, with an `OPENCLAW_HOST_CLAUDE_DIR` override.

## Best Possible Solution

Land a validated setup-only Podman mount for Claude CLI credentials that preserves existing home/path validation and SELinux behavior, with live rootless Podman proof.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes by source inspection: current setup uses container `HOME=/home/node` without a host `.claude` mount, while the Anthropic setup path reads `/home/node/.claude/.credentials.json` and throws the reported error when it is missing.

## Solution Assessment

Is this the best way to solve the issue? Not yet; a setup-only read-only mount is the right direction, but this patch should use the resolved home fallback, validate the new mount source, and include real rootless Podman/SELinux proof.

## Review Findings

- **[P2] Use the resolved home for Claude defaults:** `scripts/run-openclaw-podman.sh:532`
  - body: The script already handles an unset `HOME` by resolving `EFFECTIVE_HOME`, but this new default reads `$HOME` under `set -u`.   - confidence: 0.92

A value containing `:` or invalid path segments can make Podman reinterpret the volume spec or mount the wrong host path, so validate the source before adding the bind mount.
