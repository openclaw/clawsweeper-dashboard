## Summary

This PR should stay open for maintainer review: it has sufficient live Fedora/rootless Podman proof and no concrete blocking patch defect, but it deliberately gives the setup container read-only access to a host Claude credential directory and uses shared SELinux relabeling, which is a maintainer auth/security boundary decision.

## What This Changes

The PR adds a setup-only guarded `~/.claude` bind mount and `OPENCLAW_HOST_CLAUDE_DIR` override to the Podman launcher, plus Podman install docs for the override.

## Best Possible Solution

Merge the guarded setup-only credential handoff if maintainers accept directory-level exposure and shared SELinux relabeling; otherwise narrow the secret handoff and re-prove Fedora Podman setup.

Yes. Source inspection shows current main launches setup without a `/home/node/.claude` mount while Anthropic Claude CLI auth reads credentials from the container home, and the PR body includes a live Fedora/rootless Podman report for the failing class.

## Solution Assessment

Yes, with maintainer signoff. A setup-only, read-only mount guarded by existing private-owner checks is a focused fix; the remaining design choice is directory-level exposure versus a narrower credential handoff.
