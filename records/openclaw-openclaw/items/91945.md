## Summary

Keep open: current main and the latest release still expose Cloudflare AI Gateway as a gateway-domain Anthropic Messages provider, while Cloudflare’s current docs support the REST direction. The remaining work needs maintainer direction because changing the default endpoint, API family, model-id shape, and credential/header semantics is compatibility-sensitive and overlaps broader Cloudflare bridge work.

## What This Changes

The issue requests upgrading the built-in Cloudflare AI Gateway provider default from legacy gateway-domain behavior to Cloudflare’s REST `/client/v4/accounts/{ACCOUNT_ID}/ai/v1` API shape.

## Best Possible Solution

Define and implement a provider-owned Cloudflare REST bridge contract in the Cloudflare plugin, with explicit API-family defaults, `author/model` mapping, legacy passthrough preservation, auth/profile migration rules, docs, focused tests, and live Cloudflare proof.

## Reproduction Assessment

Yes, source-level: current main and v2026.6.8 still generate a single Anthropic Messages Cloudflare provider with the gateway-domain `/anthropic` base URL and no REST default. I did not run a live Cloudflare request in this read-only review.

## Solution Assessment

Unclear as proposed: REST support is the right direction, but switching specifically to an OpenAI chat-completions default is not clearly the best fix because Cloudflare REST also supports Anthropic Messages, matching OpenClaw’s current default behavior.
