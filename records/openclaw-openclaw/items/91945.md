## Summary

Keep open: current main and v2026.6.6 still configure the bundled Cloudflare AI Gateway provider as an Anthropic gateway-domain route, while Cloudflare’s REST API guidance is real and overlaps with the open Workers AI onboarding request. The remaining work needs maintainer direction on REST endpoint families, model defaults, and credential semantics before a safe implementation can be queued.

## What This Changes

Request to modernize the bundled Cloudflare AI Gateway provider from legacy gateway-domain defaults to Cloudflare REST `/ai/v1` behavior while preserving legacy configurations.

## Best Possible Solution

Define one provider-owned Cloudflare REST contract that supports the needed REST API families, preserves existing gateway-domain configs, and documents/tests fresh setup plus upgrade behavior before changing defaults.

## Reproduction Assessment

Not applicable as a feature request: source inspection shows current main and v2026.6.6 still generate the Anthropic gateway-domain provider shape, while Cloudflare docs establish the requested REST API target.

## Solution Assessment

Unclear. Moving toward REST is well supported, but defaulting specifically to OpenAI chat completions is not yet proven best because Cloudflare REST also supports Anthropic Messages and Workers AI, so maintainers need to choose the canonical API-family and credential model first.
