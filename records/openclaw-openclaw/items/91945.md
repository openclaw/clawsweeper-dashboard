## Summary

Keep open: current main and the latest release still build and document Cloudflare AI Gateway around the gateway-domain Anthropic path, while Cloudflare's current docs point users at REST `/ai/v1`; the remaining work is a real provider modernization but needs maintainer direction on API-family defaults and credential migration before implementation.

## What This Changes

The issue requests moving the built-in Cloudflare AI Gateway provider defaults from legacy gateway-domain endpoints to Cloudflare's REST `/ai/v1` API while preserving legacy configurations.

## Best Possible Solution

Maintain the issue as a provider-plugin product decision: add REST `/ai/v1` support per supported API family, keep explicit legacy gateway-domain configs working, and define the credential/header migration before changing onboarding defaults.

## Reproduction Assessment

Not applicable: this is a feature/provider-modernization request rather than a broken existing behavior report. Source and docs inspection do confirm current main still uses the legacy gateway-domain Anthropic contract the issue describes.

## Solution Assessment

No as written: moving directly to `openai-completions` is only one possible REST shape and may regress the current Anthropic Messages default. The safer solution is a maintainer-approved REST design that covers API family, model refs, auth headers, and migration behavior together.
