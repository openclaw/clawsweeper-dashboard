## Summary

Keep this PR open, but it needs changes before merge: the patch still misses the stated Mattermost DM path because direct chats pass `undefined` into the new reply hydrator, and the supplied proof shows branch hygiene rather than real DM behavior.

## What This Changes

The PR adds a Mattermost post-fetch helper, reply-context hydration/cache, and focused tests to populate `ReplyToBody` and `ReplyToSender` for replied-to Mattermost posts.

## Best Possible Solution

Keep Mattermost DMs top-level for delivery, but derive a separate context-only reply target from the actual Mattermost reply root, cover that monitor-level direct path, and require redacted real DM proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: Mattermost direct chats clear `effectiveReplyToId`, and the PR helper exits before lookup when that value is missing; I did not run a live Mattermost workspace.

## Solution Assessment

No. The helper/cache direction is reasonable, but wiring it to `effectiveReplyToId` is not the narrowest correct fix because that value intentionally suppresses DM delivery threading; use a separate context-only reply target instead.
