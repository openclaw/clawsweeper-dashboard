## What This Changes

The PR adds a Mattermost post fetch/cache and reply-context helper so inbound Mattermost contexts can include `ReplyToBody` and `ReplyToSender`, plus focused unit tests.

## Best Possible Solution

Separate reply metadata hydration from thread/session routing, keep Mattermost DMs flat by default, add monitor-level direct-message coverage, resolve the dirty branch, and include redacted live Mattermost proof of the after-fix inbound payload.

## Reproduction Assessment

Yes, source-level. Current main intentionally returns `undefined` for direct-message `effectiveReplyToId`, and the PR helper returns `{}` when that value is absent; I did not run a live Mattermost workspace in this read-only review.

## Solution Assessment

No. Wiring hydration to `effectiveReplyToId` is not the best fix because that id is intentionally suppressed for direct chats; the safer fix is a separate verified DM reply metadata target while preserving flat routing.
