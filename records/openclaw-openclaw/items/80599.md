Keep open: this is a useful narrow Mattermost bug-fix candidate, but the current patch does not actually hydrate DM reply metadata because it passes the helper an id that current main intentionally clears for direct messages.

## What This Changes

The PR adds a Mattermost post fetch/cache and reply-context helper intended to populate `ReplyToBody` and `ReplyToSender` for direct-message replies.

## Best Possible Solution

Separate reply metadata hydration from Mattermost thread/session routing, keep direct messages flat by default, add monitor-level DM regression coverage, resolve conflicts, and include redacted live Mattermost payload proof.

## Reproduction Assessment

Yes, source-level. Current main intentionally returns `undefined` for direct-message `effectiveReplyToId`, and the PR passes that value into the new hydration helper, so the advertised DM reply lookup does not run for direct chats.

## Solution Assessment

No. The helper shape is plausible, but it should receive a separate reply-target metadata id instead of reusing the direct-message thread/session routing id.
