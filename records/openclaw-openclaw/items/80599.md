## Summary

Keep open: the updated branch still routes reply-context hydration through `effectiveReplyToId`, which current Mattermost DM logic intentionally suppresses, so the target DM reply path remains broken and the external real-behavior proof gate is not satisfied.

## What This Changes

The PR adds Mattermost post fetch, a bounded post cache, a reply-context hydration helper, and focused tests to populate `ReplyToBody`/`ReplyToSender` for DM replies.

## Best Possible Solution

Derive a separate reply-context target from the actual Mattermost reply root such as `post.root_id` while keeping DM delivery/threading top-level, then prove it with monitor-level DM coverage and redacted real Mattermost evidence.

## Reproduction Assessment

Yes, by source inspection: a Mattermost direct-message event with `post.root_id` still gets `effectiveReplyToId: undefined`, and the PR helper returns `{}` before fetching. I did not run live Mattermost or tests during this read-only review.

## Solution Assessment

No. The helper/cache direction is plausible, but reusing the delivery/threading id for quote metadata is not the narrow correct fix because direct-message threading is intentionally disabled.
