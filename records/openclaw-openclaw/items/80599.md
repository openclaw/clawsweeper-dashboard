## Summary

Keep open: the branch targets a plausible Mattermost DM reply-context bug, but the current wiring still skips the DM reply lookup because direct chats intentionally have no effective reply target; the supplied proof also does not show real DM behavior.

## What This Changes

The PR adds a Mattermost post fetch, reply-context helper/cache, and tests to populate `ReplyToBody` and `ReplyToSender` for resolved reply targets.

## Best Possible Solution

Keep DM replies top-level for delivery/session routing, but hydrate context from a separate context-only target such as normalized `post.root_id`, add monitor-level DM coverage, and require redacted real DM proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: Mattermost channel type `D` becomes `direct`, direct chats force `effectiveReplyToId` to `undefined`, and the PR helper exits before lookup when that value is missing.

## Solution Assessment

No. The helper and bounded fetch direction are reasonable, but wiring it to `effectiveReplyToId` is not the narrowest correct fix because that value intentionally suppresses DM threading; use a separate context-only reply target while leaving delivery threading unchanged.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93
