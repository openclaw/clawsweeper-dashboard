## What This Changes

The PR adds a Mattermost post fetch helper, bounded in-memory post cache, reply-context helper, and tests to populate `ReplyToBody` and `ReplyToSender` for Mattermost replies.

## Best Possible Solution

Separate metadata hydration from Mattermost session/thread routing: use the real DM reply target for `ReplyToBody` and `ReplyToSender`, keep DM delivery non-threaded, and prove it with monitor coverage plus redacted live DM evidence.

## Reproduction Assessment

Yes, source-reproducible: current main clears `effectiveReplyToId` for direct chats, and the PR passes that cleared value into the new hydration helper. I did not run a live Mattermost workspace in this read-only review.

## Solution Assessment

No. The maintainable fix is to separate reply-context metadata from session/thread routing instead of reusing `effectiveReplyToId`.
