## Summary

Keep this open: the branch is a plausible narrow Mattermost fix, but it still wires the new hydrator to `effectiveReplyToId`, which is intentionally undefined for direct messages, so it misses the DM reply path it claims to repair; the supplied proof also does not show real Mattermost behavior.

## What This Changes

Adds a Mattermost post-fetch helper, a reply-context hydrator/cache, and tests intended to populate `ReplyToBody` and `ReplyToSender` for replied-to Mattermost posts.

## Best Possible Solution

Hydrate quote metadata from a separate context-only Mattermost reply target such as normalized `post.root_id`, keep DM delivery/session threading flat, add monitor-level DM coverage, and require redacted real Mattermost proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: Mattermost `D` maps to `direct`, direct chats set `effectiveReplyToId` to `undefined`, and the PR helper exits before lookup when that value is missing.

## Solution Assessment

No. The helper/cache direction is reasonable, but the current wiring reuses a delivery-threading id that is intentionally suppressed for DMs; the safer fix is a separate context-only reply target.
