## Summary

Keep this PR open. It targets a real Mattermost reply-context gap, but the current head still hydrates from `effectiveReplyToId`, which is intentionally undefined for direct chats, so the stated DM reply path remains unfixed and the supplied proof does not show real Mattermost behavior.

## What This Changes

The PR adds a Mattermost post-fetch helper, reply-context hydration/cache in the monitor, and regression tests intended to populate `ReplyToBody` and `ReplyToSender` for replied-to posts.

## Best Possible Solution

Hydrate DM reply context from a separate context-only Mattermost target such as normalized `post.root_id`, while keeping `ReplyToId` and delivery threading flat for direct messages.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: Mattermost type `D` maps to `direct`, direct chats force `effectiveReplyToId` to `undefined`, and the PR helper exits without lookup when that value is missing.

## Solution Assessment

No. The helper and bounded fetch direction are reasonable, but wiring hydration to `effectiveReplyToId` is not correct for DMs; use a separate context-only reply target and cover the monitor-level DM path.
