## Summary

Keep this PR open, but it should not merge as-is: the patch still feeds the reply-context helper from `effectiveReplyToId`, which current Mattermost direct-message handling intentionally leaves unset, so the DM reply path this PR targets still skips hydration.

## What This Changes

The branch adds a Mattermost post-fetch helper, a bounded post cache, a reply-context hydrator, and focused tests to populate `ReplyToBody` and `ReplyToSender` for replied-to Mattermost posts.

## Best Possible Solution

Keep Mattermost DMs top-level for delivery, but hydrate quote metadata from a context-only reply target such as normalized `post.root_id`, with monitor-level regression coverage and redacted live Mattermost proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: Mattermost direct chats force `effectiveReplyToId` to `undefined`, and the new helper exits before lookup when that value is missing.

## Solution Assessment

No. The helper and bounded fetch/cache direction are reasonable, but using `effectiveReplyToId` is not the right target for DM reply metadata because that value intentionally suppresses DM threading.
