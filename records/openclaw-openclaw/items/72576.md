## Summary

Keep open. Current main has the Anthropic `claude-cli` production registration, manifest metadata, direct registration coverage, and generic legacy `embeddedHarness` migration coverage, but it still lacks the requested default-on bundled-loader guardrail; the open unmerged PR #73122 is the implementation candidate and uses closing syntax for this issue.

## What This Changes

Request to add a default-on integration guardrail proving the bundled Anthropic `claude-cli` backend remains registered and legacy runtime config remains resolvable after plugin loading.

## Best Possible Solution

Land PR #73122 or an equivalent narrow test-only patch that loads the bundled Anthropic plugin through the real loader path, asserts `claude-cli` is available as a CLI backend rather than an embedded AgentHarness, and pins legacy config migration behavior without changing runtime semantics.

## Reproduction Assessment

Yes for the test-gap claim: static inspection of current main reproduces the missing guardrail because `src/plugins/loader.test.ts` has no Anthropic or `claude-cli` loader assertion. Reproducing the older runtime outage is not necessary to verify the missing CI coverage.

## Solution Assessment

Yes, the test-only approach is the best way to solve this issue. The safer path is to review and land #73122 or equivalent coverage, not to add production aliases or blur the CLI-backend versus AgentHarness split.
