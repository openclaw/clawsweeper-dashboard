## Summary

Keep open. Current main has the Anthropic production registration and some direct/migration coverage, but it still lacks the requested bundled-loader guardrail; the open closing PR #73122 is the active implementation candidate and should be reviewed or closed before this issue is resolved.

## What This Changes

The issue requests a default-on test guardrail proving the bundled Anthropic `claude-cli` backend is registered/resolvable after plugin load and that legacy `embeddedHarness.runtime: "claude-cli"` configs migrate to the current runtime shape.

## Best Possible Solution

Review and land PR #73122, or an equivalent narrow test-only change, so CI exercises the bundled Anthropic loader path, pins `claude-cli` as a CLI backend, and covers legacy runtime migration without changing production behavior.

## Reproduction Assessment

## Solution Assessment

Yes. A focused test-only loader and migration guardrail is the narrowest maintainable solution; production routing changes should stay with separate behavior bugs such as #74500 if they still reproduce.
