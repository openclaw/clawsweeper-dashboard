## Summary

Keep open: the PR targets a valid doctor warning gap for the linked report, but the implementation has a blocking false negative when a profile is combined with an explicit allowlist, and the PR body does not yet include inspectable after-fix real behavior proof.

## What This Changes

The PR adds a `doctor` security warning and tests for channel-bound agents whose explicit tool allowlist omits the `message` tool.

## Best Possible Solution

Add the doctor warning using resolved `message` availability across profile, global, and per-agent policy, with production-path regression tests and redacted real `openclaw doctor` output proof.

## Reproduction Assessment

Yes, source-reproducible: current policy filtering removes `message` when an explicit allowlist excludes it, and current main has no channel-binding doctor warning for that mismatch. I did not run a live Discord/Ollama reproduction in this read-only review.

## Solution Assessment

No. A doctor warning is the right surface for this PR's scope, but the implementation should reuse resolved tool-policy semantics instead of treating profiles as sufficient by themselves.
