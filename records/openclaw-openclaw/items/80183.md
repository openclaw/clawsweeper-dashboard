## Summary

Keep open: the PR addresses a real source-verifiable doctor gap, but the implementation has a blocking false negative when a profile that includes `message` is narrowed by an explicit allowlist, and the PR still lacks required real behavior proof.

## What This Changes

The PR adds a `doctor` security warning and tests for channel-bound agents whose explicit tool allowlist omits the `message` tool.

## Best Possible Solution

Add the doctor warning using resolved `message` availability across profile, global, and per-agent policy, with production-path regression tests and redacted real `openclaw doctor` output proof.

## Reproduction Assessment

Yes, source-reproducible: an explicit non-empty allowlist without `message` filters that tool out, and current main has no route-binding warning for explicit message actions. I did not run a live Discord/Ollama reproduction.

## Solution Assessment

No. A doctor warning is the right surface for this PR's scope, but the implementation should reuse resolved tool-policy semantics instead of treating profiles as sufficient by themselves.
