## Summary

Keep open for maintainer product/security review: current main still isolates ACPX Codex CODEX_HOME and has no plugin-inheritance, marketplace allowlist, or explicit Codex-home override on main, while the related override PR remains open.

## What This Changes

The issue requests ACPX Codex background sessions to inherit or explicitly load user-installed Codex plugins, skills, or marketplaces such as Superpowers.

## Best Possible Solution

Keep ACPX isolated by default; if accepted, add an explicit allowlisted Codex home/plugin/marketplace path with public docs, focused ACPX tests, and redacted real-runtime proof.

## Reproduction Assessment

No high-confidence live reproduction was run. Source, tests, docs, and the pinned wrapper contract confirm current main uses an isolated generated Codex home and does not apply user-installed Codex plugin or marketplace state to ACP bindings.

## Solution Assessment

No, automatic inheritance of the operator's personal Codex plugins is not clearly the safest default. A maintainable solution should be explicit and allowlisted, or use a reviewed Codex-home override path such as https://github.com/openclaw/openclaw/pull/77006 after maintainer/security review.
