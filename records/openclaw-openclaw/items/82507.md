## Summary

Keep open for maintainer product/security review: current main still intentionally isolates Codex ACPX `CODEX_HOME` and has no ACPX plugin-inheritance or marketplace allowlist path. The related open Codex-home override PR is a candidate direction, but it is not merged and does not settle the plugin execution policy.

## What This Changes

Request ACPX Codex background tasks to inherit or explicitly load user-installed Codex plugins and skills such as Superpowers from the user's normal Codex home.

## Best Possible Solution

Keep ACPX isolated by default, and if accepted, add an explicit allowlisted Codex ACP plugin/home/marketplace path with public docs, focused ACPX tests, and redacted real-runtime proof.

## Reproduction Assessment

No high-confidence live reproduction was run. Source, docs, and focused tests confirm current main uses an isolated generated Codex home and does not apply native Codex plugin config to ACP bindings.

## Solution Assessment

No, automatic inheritance of the operator's personal Codex plugins is not clearly the safest default. A maintainable solution should be explicit and allowlisted, or use a reviewed Codex-home override path after maintainer/security review.
