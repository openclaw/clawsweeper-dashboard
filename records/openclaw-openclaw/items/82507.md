## Summary

Keep open for maintainer product/security review: current main still intentionally isolates the ACPX Codex `CODEX_HOME`, and the requested plugin inheritance would change the code-execution and credential boundary rather than fix a narrowly defined bug.

## What This Changes

The issue requests ACPX Codex background tasks to inherit user-installed Codex plugins, marketplaces, or skill directories from the user's personal Codex home.

## Best Possible Solution

Keep ACPX isolated by default; if accepted, add an explicit allowlisted Codex home/plugin/marketplace path with public docs, focused ACPX tests, and redacted live ACPX/Codex proof.

## Reproduction Assessment

No high-confidence live reproduction was run. Source, tests, and docs do confirm current main uses an isolated generated Codex home and does not apply native Codex plugin config or user plugin caches to ACP bindings.

## Solution Assessment

No, implicit automatic inheritance is not clearly the safest solution. The maintainable direction is an explicit, reviewed, allowlisted Codex home or marketplace path that preserves the isolated default.
