## Summary

Keep open: the Brave Goggles feature is useful and upstream-supported, but this PR still targets the pre-provider core web-search path, so it would not implement Goggles in the active Brave provider on current main; it also lacks inspectable real behavior proof.

## What This Changes

Adds a Brave-only `goggles` argument to the legacy core `web_search` path, forwards it to Brave web and llm-context URLs, adds cache/test coverage, and updates web/Brave docs plus changelog.

## Best Possible Solution

Implement Goggles in the current generic `web_search` schema and provider-owned Brave runtime with case-preserving cache identity, canonical docs, focused tests, and redacted live Brave proof.

## Reproduction Assessment

Yes, for the review findings. Source inspection of current main shows active Brave execution under `extensions/brave` and the generic agent schema, while the PR changes old core files and current active schema/runtime/cache/docs contain no `goggles`.

## Solution Assessment

No. The feature direction is valid, but the mergeable solution should port `goggles` through current agent/provider schemas, Brave web and llm-context URL builders, cache identity, and canonical docs instead of the old core implementation.
