## Summary

Keep open. The Brave Goggles feature is still useful and not implemented on current main, but this PR is not mergeable as-is because it changes the old core web-search implementation while current Brave execution, schema, URL building, and caching live in the provider-owned Brave plugin path; it also lacks inspectable real behavior proof.

## What This Changes

The PR adds a Brave-only `goggles` parameter to `web_search`, forwards it to Brave web and llm-context requests, separates cache entries by goggles value, and updates web-search documentation.

## Best Possible Solution

Implement Goggles in the current generic tool contract and provider-owned Brave schema/runtime/cache/tests/docs path, then attach redacted live Brave proof for both web and llm-context modes.

## Reproduction Assessment

Yes, for the review findings. Source inspection of current main shows no `goggles` in the generic schema, active Brave schema, Brave URL builders, or Brave cache key, while the PR adds it to the old core implementation.

## Solution Assessment

No. The feature direction is reasonable, but this patch is not the best merge path because it must be ported to `extensions/brave` and canonical docs instead of the stale core web-search surface.
