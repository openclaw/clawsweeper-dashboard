## Summary

Keep this PR open. Current main still hides pinned write helper failures behind the generic safe-open message, and the PR is useful direction, including the follow-up fix for Greptile's macOS platform-guard comment. It is not merge-ready because missing `python3` spawn failures still bypass the new install-hint path, and the branch lacks the required user-facing changelog entry.

## What This Changes

The PR changes pinned write error handling in `src/infra/fs-safe.ts` and `src/infra/fs-pinned-write-helper.ts` to preserve helper failure details and add platform-specific Python install hints.

## Best Possible Solution

Preserve underlying helper messages through `SafeOpenError`, add explicit write-helper child spawn-error handling for missing or non-executable Python, cover the macOS/Linux diagnostic paths with focused tests, and add the required changelog credit while leaving pinned write boundary semantics unchanged.

## Reproduction Assessment

Yes. The current-main source path is high confidence: write/copy callers normalize non-`SafeOpenError` helper failures to the generic message, and a read-only Node spawn check confirmed the missing-binary path that the PR still does not format with its install hint.

## Solution Assessment

No, not as-is. The normalizer direction is right and the macOS guard follow-up is addressed, but the maintainable fix should handle child spawn errors like the pinned path helper before relying on stderr/exit-code hints, then prove the user-facing diagnostics with tests and changelog coverage.
