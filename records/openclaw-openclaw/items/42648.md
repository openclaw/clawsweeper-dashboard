## Summary

Keep open: current main still has Markdown-backed memory indexing, short-term promotion, and recall/read tooling, but not the requested durable write resolver with classification, provenance-gated writes, dedupe/merge, conflict surfacing, revision history, and entity normalization. The w1tv production-experience comment, related Memory MVP issues, and open Memory v2 PR are useful design inputs rather than evidence that this is already implemented.

## What This Changes

The issue requests a Memory MVP durable write pipeline that classifies candidate memories, attaches source/provenance, dedupes or merges near matches, and routes conflicts or low-confidence writes for review.

## Best Possible Solution

Settle one Memory MVP contract for durable records, provenance, resolver decisions, conflict/review states, revision history, and edit/forget semantics, then implement it across memory-host-sdk and memory-core with docs and focused regression coverage.
