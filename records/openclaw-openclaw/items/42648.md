## Summary

Keep this open. Current main has adjacent memory search, dreaming promotion, Memory Wiki provenance/contradiction reporting, and LanceDB duplicate checks, but it still lacks the requested durable write resolver with typed create/merge/update/conflict outcomes, required source attachment, normalized entities, and append-only revisions.

## What This Changes

The issue requests a conservative durable memory write pipeline that classifies candidates, attaches sources, dedupes/merges, handles conflicts explicitly, and preserves revision history.

## Best Possible Solution

Define one backend-honored durable memory write contract with typed canonical records, required source references, normalized subjects/entities, near-duplicate and contradiction lookup, explicit create/merge/update/conflict-for-review outcomes, and append-only revision/provenance history.
