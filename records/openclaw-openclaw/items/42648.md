## Summary

Keep open: current main still has Markdown-backed memory search plus thresholded short-term promotion into MEMORY.md, not the requested typed write resolver with classification, source-required durable writes, near-duplicate/conflict resolution, merge policy, and revision history. The linked Memory MVP issues and the smart-memory/real-world comments are useful design input, but this remains a product and schema contract that is not safe to sweep closed.

## What This Changes

Requested buildout of a conservative memory write pipeline that classifies candidates, attaches sources, deduplicates and merges records, and routes conflicts without silent overwrite.

## Best Possible Solution

Settle the v1 memory record, provenance, merge/conflict, and review contracts across the Memory MVP cluster, then implement the resolver in the active memory plugin with SQLite-backed canonical records and explicit review/conflict states.

## Reproduction Assessment

No; reproduction is not applicable because this is a requested new memory write capability, not a broken existing behavior with a failing current-main path. Source and docs show the requested resolver is absent from the current Markdown promotion model.

## Solution Assessment

No; this is not best solved as a standalone narrow fix. The maintainable path is to decide the schema, provenance, conflict policy, and review flow first, then implement the resolver at the active memory plugin boundary.
