## Summary

Keep open: current main has thresholded short-term promotion, source line markers, and exact promotion-marker dedupe, plus a separate memory-wiki belief layer, but it still lacks the requested typed durable write resolver with classification, create/merge/conflict decisions, revision history, and explicit conflict review.

## What This Changes

Build a conservative Memory MVP write pipeline that classifies candidate memories, attaches sources, dedupes or merges before durable write, and routes contradictions or low-confidence candidates to review.

## Best Possible Solution

Keep this as the write-pipeline umbrella until maintainers settle the durable record model and implement a resolver that gates candidate promotion through typed classification, source-backed dedupe/merge, explicit conflicts, and revision history.
