## Summary

Keep open. Current main has a thresholded short-term promotion path and a companion Memory Wiki layer, but it still lacks the requested typed write resolver with classification, durable source/provenance records, near-duplicate merge/update, explicit conflict review, and revision history.

## What This Changes

The issue requests a conservative Memory MVP write pipeline that classifies candidate memories, attaches sources, dedupes or merges before durable write, and routes conflicts or low-confidence writes to review.

## Best Possible Solution

Define and implement a versioned memory-core write resolver backed by a durable record/provenance model, with Memory Wiki integration used as a companion knowledge layer rather than the active write-path substitute.

## Reproduction Assessment

Not applicable. This is a Memory MVP feature/design request, and the current-main code inspection shows the requested resolver capability is not present rather than a failing existing contract.

## Solution Assessment

Unclear as an implementation path. The requested direction is coherent, but the best maintainable solution needs maintainer decisions on schema ownership, resolver semantics, and how conflict/review flows split between memory-core and memory-wiki.
