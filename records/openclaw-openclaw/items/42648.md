## Summary

Keep open: current main and v2026.5.20 still provide Markdown-backed memory indexing plus thresholded promotion, not the requested durable write resolver; the related open Memory MVP items and Memory v2 PR show this is still product and architecture work, with useful policy input from w1tv's production-experience comment.

## What This Changes

Build a Memory MVP write resolver that classifies candidate memories, attaches sources, dedupes or merges near-duplicates, records revisions, and routes conflicts or low-confidence candidates to review.

## Best Possible Solution

Define one Memory MVP contract for durable records, provenance, resolver decisions, conflict/review states, revision history, and edit/forget semantics, then implement it across memory-host-sdk and memory-core with docs and focused regression coverage.
