## Summary

Keep open: current main still relies on Markdown-backed active memory, indexing, short-term promotion, and companion wiki tooling; it does not yet provide the requested durable write resolver for classification, provenance-gated writes, dedupe/merge, conflict surfacing, revision history, and entity normalization.

## What This Changes

Build a Memory MVP write pipeline that classifies candidate memories, attaches source provenance, checks near-duplicates and conflicts, and resolves each write to create, merge/update, or review.

## Best Possible Solution

Settle one Memory MVP contract for durable records, provenance, resolver decisions, conflict/review states, revision history, and edit/forget semantics, then implement it across memory-host-sdk and memory-core with docs and focused regression coverage.
