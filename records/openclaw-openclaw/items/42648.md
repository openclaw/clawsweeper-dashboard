## Summary

Keep this open. Current main has related memory search, short-term promotion, Memory Wiki provenance/contradiction support, and optional LanceDB auto-capture, but it still lacks the requested conservative durable write resolver with typed create/merge/update/conflict outcomes, mandatory source attachment, entity normalization, and append-only revision history.

## What This Changes

Build a conservative memory write pipeline that classifies candidates, attaches sources, checks near-duplicates and conflicts, resolves create/merge/update/review outcomes, and preserves provenance plus revisions before durable writes.

## Best Possible Solution

Define one durable memory write contract that all active memory backends can honor: typed canonical records, required source references, normalized subjects/entities, near-duplicate and contradiction lookup, explicit create/merge/update/conflict-for-review outcomes, and append-only revision/provenance history, with Memory Wiki and indexing layers consuming or reflecting that contract rather than bypassing it.

## Reproduction Assessment

Not applicable as a feature architecture request. The high-confidence verification path is source and docs inspection: current main implements Markdown/index/wiki/LanceDB-adjacent memory paths, but no canonical durable write resolver or storage model.

## Solution Assessment

Unclear for a concrete patch because no implementation is under review. The requested direction is valid, but the maintainable solution needs a coordinated memory contract rather than a one-off change to promotion thresholds, wiki metadata, or LanceDB duplicate checks.
