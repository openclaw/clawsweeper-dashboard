## Summary

Keep open: current main has Markdown-backed memory, recall indexing, thresholded promotion, and a memory-wiki companion, but it does not provide the requested durable write resolver with typed classification, source-gated ingestion, near-duplicate/conflict resolution, merge/update decisions, and revision history.

## What This Changes

Build a conservative memory ingestion write pipeline that classifies candidates, attaches sources, dedupes or merges near matches, and routes conflicts before durable writes.

## Best Possible Solution

Define the v1 durable memory write contract first, then implement it in the owning memory plugin with schema/provenance and review-flow follow-through while reusing memory-wiki’s claim/evidence model where it fits.

## Solution Assessment

Unclear as a single implementation task. The requested direction is reasonable, but the maintainable path should be a maintainer-designed slice across schema, provenance, conflict UX, and memory plugin ownership.
