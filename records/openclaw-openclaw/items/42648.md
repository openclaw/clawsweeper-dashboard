## Summary

Keep open: current main still has Markdown-backed memory search, short-term promotion, and append-only memory flush behavior, but it does not have the requested durable resolver that classifies candidate records, deduplicates or merges them, preserves revisions, and routes conflicts before writing. The related Memory MVP issues remain open, and w1tv's implementation-experience comment is useful design input rather than evidence that this is already implemented.

## What This Changes

The issue requests a durable memory write pipeline that classifies candidates, attaches sources, deduplicates or merges records, routes conflicts, and preserves revision history before writing.

## Best Possible Solution

Land one coherent durable-memory record, provenance, resolver, and conflict/review lifecycle across memory-host-sdk and memory-core, then expose ingestion and operator commands on that agreed contract.
