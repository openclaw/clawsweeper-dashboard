## Summary

Keep open: current main still uses Markdown-backed memory indexing, short-term promotion, and wiki sidecar tooling, but it does not implement the requested durable write resolver with classification, provenance-gated writes, dedupe/merge, conflict surfacing, revision history, and entity normalization.

## What This Changes

The issue requests a Memory MVP write pipeline that classifies candidate memories, attaches source provenance, deduplicates or merges records, and surfaces conflicts before durable writes.

## Best Possible Solution

Settle one Memory MVP contract for durable records, provenance, resolver decisions, conflict/review states, revision history, and edit/forget semantics, then implement it across memory-host-sdk and memory-core with docs and focused regression coverage.
