## Summary

Keep this open. Current main has adjacent memory search, dreaming promotion, Memory Wiki provenance/contradiction tools, and LanceDB duplicate checks, but it still lacks the requested durable write resolver with typed create/merge/update/conflict outcomes, required source attachment, normalized entities, and append-only revisions; the w1tv comment adds useful production design context rather than making the work complete.

## What This Changes

Build a conservative durable memory ingestion write path that classifies candidates, attaches sources, resolves near-duplicates and conflicts, preserves revisions, and blocks unsafe writes.

## Best Possible Solution

Define one backend-honored durable memory write contract with typed canonical records, required source references, normalized subjects/entities, near-duplicate and contradiction lookup, explicit create/merge/update/conflict-for-review outcomes, and append-only revision/provenance history.

Not applicable; this is a feature architecture request, not a broken current-main behavior. The high-confidence check is source and docs inspection showing adjacent memory pieces but no durable write resolver.

## Solution Assessment

Unclear for a concrete patch because no implementation is under review. The requested direction is sound, but the maintainable solution needs a coordinated memory contract rather than a one-off promotion, wiki metadata, or LanceDB duplicate-check tweak.
