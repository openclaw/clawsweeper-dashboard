## Summary

Keep this open. Current main has related memory search, short-term promotion, Memory Wiki provenance/contradiction support, and LanceDB duplicate checks, but it still lacks the requested canonical durable write resolver with typed create/merge/update/conflict outcomes, required source attachment, entity normalization, and append-only revision history.

## What This Changes

The issue requests a conservative memory ingestion write pipeline with classification, source attachment, near-duplicate and conflict resolution, merge/update outcomes, revision history, and entity normalization.

## Best Possible Solution

Define one durable memory write contract that active memory backends can honor: typed canonical records, required source references, normalized subjects/entities, near-duplicate and contradiction lookup, explicit create/merge/update/conflict-for-review outcomes, and append-only revision/provenance history.

## Reproduction Assessment

Not applicable as a feature architecture request. The high-confidence verification path is source and docs inspection: current main implements Markdown/index/wiki/LanceDB-adjacent memory paths, but no canonical durable write resolver or storage model.

## Solution Assessment

Unclear for a concrete patch because no implementation is under review. The requested direction is sound, but the maintainable solution needs a coordinated memory contract rather than a one-off promotion threshold, wiki metadata, or LanceDB duplicate-check change.
