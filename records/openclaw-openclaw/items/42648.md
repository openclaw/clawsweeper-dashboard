## Summary

Keep open: current main still has useful adjacent memory promotion, search, event, and memory-wiki pieces, but it does not implement the requested typed durable memory write resolver with classification, source-required dedupe/merge/conflict decisions, revision history, and canonical conflict state.

## What This Changes

Build a Memory MVP write path that classifies candidate memories, attaches sources, dedupes or merges before durable writes, routes conflicts for review, and preserves revision/provenance history.

## Best Possible Solution

Define the Memory MVP schema/provenance/resolver contract first, then implement the write pipeline inside the owning memory plugin/SDK boundary with SQLite canonical records and conflict-review integration.

## Reproduction Assessment

Not applicable as a bug reproduction: this is a Memory MVP feature/design request. Source inspection shows current main lacks the requested typed write resolver and canonical revision/conflict schema.

## Solution Assessment

No: current Markdown promotion, memory search, and memory-wiki are useful partial mechanisms, but the maintainable solution is one owner-approved write contract rather than another ad hoc append path.
