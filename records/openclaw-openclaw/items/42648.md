## Summary

Keep this open. Current main has related memory search, short-term promotion, Memory Wiki provenance/contradiction reporting, and a LanceDB duplicate check, but it still does not provide the requested durable write resolver with typed create/merge/update/conflict outcomes, required source attachment, normalized entities, and append-only revisions.

## What This Changes

Build a conservative durable memory write pipeline that classifies candidates, attaches sources, dedupes/merges, and routes conflicts before writing.

## Best Possible Solution

Define one backend-honored durable memory write contract: typed canonical records, required source references, normalized subjects/entities, near-duplicate and contradiction lookup, explicit create/merge/update/conflict-for-review outcomes, and append-only revision/provenance history.
