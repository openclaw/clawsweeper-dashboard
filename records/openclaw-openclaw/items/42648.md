## Summary

Keep open: current main still lacks the requested typed durable memory write resolver with source-required classification, dedupe/merge decisions, conflict state, and revision history. The existing memory-core promotion path is useful but still appends Markdown, while memory-wiki is an adjacent knowledge layer and the sibling Memory MVP issues remain open.

## What This Changes

The issue requests a durable memory write pipeline that classifies candidate memories, attaches provenance, deduplicates or merges existing records, and routes conflicts to review.

## Best Possible Solution

Define the Memory MVP contract first, then implement one owner-approved active memory write path through the memory plugin/SDK boundary with SQLite canonical records, source/provenance, resolver decisions, conflict state, and revision history.

## Solution Assessment
