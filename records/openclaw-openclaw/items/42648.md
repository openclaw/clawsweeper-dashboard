## Summary

Keep open: current main and the latest release still provide Markdown-backed memory indexing plus thresholded promotion, not the requested durable write resolver with classification, source-required writes, merge/conflict decisions, and revision history. The Memory v2 PR is related but remains open and does not make this issue implemented on main; w1tv's production-experience comment is useful product input for the eventual design.

## What This Changes

The issue requests a Memory MVP write pipeline that classifies candidate memories, attaches sources, deduplicates or merges before durable writes, and routes conflicts or low-confidence candidates safely.

## Best Possible Solution

Define one Memory MVP contract for durable records, provenance, resolver decisions, conflict/review states, revision history, and edit/forget semantics, then implement it across memory-host-sdk and memory-core with docs and focused regression coverage.
