## Summary

Keep open. Current main still has thresholded short-term promotion, retrieval indexing, REM/backfill, and a memory-wiki companion layer, but it does not implement the requested durable write resolver with typed classification, source-required writes, near-duplicate/conflict resolution, merge revisions, and explicit review routing.

## What This Changes

Add a conservative Memory MVP write pipeline that classifies candidate memories, attaches provenance, dedupes or merges before durable write, and routes conflicts or low-confidence writes for review.

## Best Possible Solution

Define the durable schema, provenance, review/conflict, and operator contracts together, then implement the resolver in the active memory owner layer while keeping memory-wiki as the companion knowledge/provenance layer.

Not applicable. This is a Memory MVP feature/design request, and source review shows adjacent indexing and thresholded promotion behavior rather than a failing current-main bug path to reproduce.

## Solution Assessment

Unclear as an immediate implementation. The requested direction is reasonable, but the maintainable solution needs schema, provenance, conflict/review, and CLI decisions before adding the resolver.
