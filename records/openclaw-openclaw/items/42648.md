## Summary

Keep open: current main and v2026.5.22 still expose Markdown-backed memory indexing, search, and thresholded promotion rather than the requested durable write resolver. The related Memory MVP issues remain open, and w1tv's production-experience comment adds useful policy input for the eventual resolver design.

## What This Changes

Build a Memory MVP write path that classifies candidate memories, attaches source data, deduplicates or merges related records, preserves revisions, and routes conflicts or low-confidence writes for review.

## Best Possible Solution

Define one Memory MVP contract for durable records, provenance, resolver decisions, conflicts/review states, revision history, and edit/forget semantics, then implement it in the memory plugin/host boundary with docs and focused regression coverage.
