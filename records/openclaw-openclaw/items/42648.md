## Summary

Keep open: current main still provides Markdown-backed memory indexing, staged short-term promotion, and recall/read tooling, but it does not include the requested durable write resolver with classification, provenance-gated writes, dedupe/merge, conflict surfacing, revision history, and entity normalization. The w1tv production-experience comment and the open Memory MVP companion issues are useful design input rather than evidence that this is already implemented.

## What This Changes

Build a conservative Memory MVP write pipeline that classifies candidate memories, attaches sources, deduplicates or merges before durable writes, and routes conflicts or low-confidence entries for review.

## Best Possible Solution

Settle one Memory MVP contract for durable records, provenance, resolver decisions, conflict/review states, revision history, and edit/forget semantics, then implement it across memory-host-sdk and memory-core with docs and focused regression coverage.
