## Summary

Keep open. Current main still emits the same generic ACP parent-stream stall notice when a child run produces no assistant output, and the current bridge does not preserve prompt-submitted/no-first-output state needed to distinguish a proxy/network failure from a true interactive wait.

## What This Changes

The issue asks OpenClaw to surface ACP Codex network/proxy no-output failures as concrete runtime diagnostics instead of an interactive-wait stall.

## Best Possible Solution

Land a focused ACP diagnostics patch that carries prompt-submitted/runtime-activity signals through the ACP manager, agent command, dispatch path, task progress, and parent relay, then classifies no-first-output-after-prompt separately from interactive waits.

## Reproduction Assessment

Yes for the OpenClaw-visible failure mode: current source shows a no-output ACP child run still produces the exact generic interactive-wait notice. The proxy-stripped upstream network failure is environment-dependent and was not live-run here.

## Solution Assessment

Yes: the narrow maintainable fix is additive diagnostics/state propagation, not a broad proxy policy change or a new ACP doctor command. #44949 is useful prior art, but it closed unmerged, and #29195 is broader than this focused failure mode.

## Repair Work Prompt

Fix issue #44810 by making ACP parent-stream stalls distinguish "prompt submitted but no first assistant output" from true interactive waits. Preserve and forward prompt/runtime activity from the ACP runtime through the manager, `src/agents/agent-command.ts`, `src/auto-reply/reply/dispatch-acp.ts`, task progress, and `src/agents/acp-spawn-parent-stream.ts`; update the stall notice and stream log to avoid implying interactive input after a prompt has been submitted. Add focused tests for relay classification, ACP event forwarding, dispatch propagation, and ACPX/Codex prompt event parsing if the current `acpx` surface exposes it. Keep proxy diagnostics redacted, do not add a broad new doctor command, do not change global proxy policy or ACPX dependency pins without maintainer approval, and add a user-facing CHANGELOG.md fix entry.
