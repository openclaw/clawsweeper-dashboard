## Summary

Keep open. Current main still has the OpenClaw-visible no-output ACP stall path: the parent relay emits the generic interactive-input notice, while the current ACP command/dispatch surfaces do not carry enough prompt-submitted or runtime-activity state to classify this as no response after prompt.

## What This Changes

The issue asks OpenClaw to surface concrete ACP/Codex network or runtime diagnostics when a prompted child run produces no output, instead of reporting a generic interactive-input stall.

## Best Possible Solution

Land a focused ACP diagnostics patch that propagates prompt-submitted and runtime-activity signals through the current ACP manager, agent command/dispatch paths, task progress, and parent relay, then classifies no-first-output-after-prompt separately from true interactive waits.
