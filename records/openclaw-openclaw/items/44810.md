## Summary

Keep open. Current main still has the OpenClaw-visible failure mode: the ACP parent relay emits the generic interactive-input stall notice when no assistant output arrives, while the current ACP agent bridge still drops non-text runtime/status events that could distinguish prompt-submitted/no-first-output from a real input wait.

## What This Changes

The issue asks OpenClaw to surface network/proxy-caused Codex ACP no-output stalls as concrete runtime or no-response diagnostics instead of interactive-input waits.

## Best Possible Solution

Land a focused ACP diagnostics patch that carries prompt-submitted and runtime-activity signals through the current ACP manager, agent command/dispatch paths, task progress, and parent relay, then classifies no-first-output-after-prompt separately from interactive waits.
