## Summary

Keep open: the Discord safety direction is valid, but the current PR still exposes bot-loop protection on prepared/assembled channel turns without enforcing it there, and the supplied terminal proof is stale against current head.

## What This Changes

Adds a generic pair-loop guard, Discord bot-loop config/docs/schema, and Discord pre-dispatch suppression for accepted bot-authored messages.

## Best Possible Solution

Enforce one shared guard in every documented channel-turn entry point, keep Discord facts/config/docs aligned with that contract, and require current-head runtime proof before merge.

## Reproduction Assessment

Yes, from source inspection. Current main lets Discord bot-authored messages that pass `allowBots`, access, routing, and mention gates reach dispatch, while the replay guard only dedupes repeated delivery of the same message id.

## Solution Assessment

Unclear as submitted. A pair guard before record/dispatch is the right direction, but the PR should enforce the documented contract for prepared/assembled turns and refresh current-head proof before it is the narrow maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Guard prepared channel turns before dispatch:** `src/channels/turn/types.ts:348`
