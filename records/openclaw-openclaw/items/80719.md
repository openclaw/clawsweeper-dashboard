## What This Changes

This PR adds a Discord bot-loop tracker, `botLoopProtection` config/schema/UI metadata/docs/changelog, and preflight regression tests to suppress repeated bot-to-bot messages when `allowBots` admits bot traffic.

## Best Possible Solution

Land the Discord-owned preflight guard after maintainers approve the defaults, keeping broader semantic loop detection separate.

## Reproduction Assessment

Yes, source-reproducible. Current main lets bot-authored Discord messages that pass `allowBots` and mention/access gates reach successful preflight without any per-pair window or cooldown state.

## Solution Assessment

Yes, with maintainer policy review. A Discord-local preflight guard keyed by account, channel, and bot pair is a narrow maintainable fix for the reported runaway loop, while the exact default budget remains a product choice.
