## What This Changes

The PR adds a Discord-local bot-loop tracker, config/schema/UI metadata, docs/changelog, and regression tests to suppress repeated bot-to-bot inbound messages when `allowBots` permits bot traffic.

## Best Possible Solution

Land the Discord-owned preflight guard after maintainer review confirms the default threshold/cooldown policy and CI passes, keeping broader semantic loop scoring separate.

## Reproduction Assessment

Yes. Source inspection of current main shows bot-authored Discord messages that pass `allowBots` and mention/access gates can reach successful preflight without any bot-pair window or cooldown state.

## Solution Assessment

Yes, with maintainer policy review. A Discord-local guard keyed by account, channel, and sorted bot pair is the narrow maintainable fix for the reported runaway loop, while semantic/progress loop scoring should remain separate.
