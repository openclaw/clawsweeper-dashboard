## Summary

Keep open: the reported TUI symptom is source-reproducible, but this PR should not merge as-is because it broadens a shared assistant-visibility helper that current tests and adjacent maintainer work keep final-answer-safe outside TUI.

## What This Changes

The PR adds a commentary-phase fallback to `extractAssistantVisibleText` when no `final_answer` text is present.

## Best Possible Solution

Keep shared history, preview, transcript, and Control UI extraction final-answer-safe, then implement any fallback in a TUI-specific live-stream projection path with regression coverage and real TUI proof.

## Reproduction Assessment

Yes. Source inspection shows commentary-only phased assistant messages return no visible text from the shared helper, and the TUI stream assembler reaches that helper through `extractContentFromMessage`.

## Solution Assessment

No. The PR targets the right symptom but changes a shared visibility boundary; the narrower maintainable fix is a TUI-scoped rendering fallback that preserves existing non-TUI suppression contracts.
