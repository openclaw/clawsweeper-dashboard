## Summary

Keep open: this targets a plausible TUI visibility bug, but the patch changes a shared extraction boundary that current callers and tests use to suppress commentary-only text, and the PR still lacks after-fix TUI proof.

## What This Changes

The PR adds a commentary-phase fallback to shared assistant visible-text extraction when no final_answer text is present.

## Best Possible Solution

Keep shared history, preview, transcript, and Control UI extraction final-answer-safe, then add any fallback in a TUI-scoped rendering path with regression coverage and real TUI proof.

## Reproduction Assessment

Yes from source inspection: a commentary-only phased assistant message returns no visible shared text, and the TUI stream assembler reaches that helper through extractContentFromMessage. I did not live-run a TUI session in this read-only review.

## Solution Assessment

No. The PR targets the right symptom, but changing the shared helper is too broad; the maintainable fix is a TUI-scoped fallback or caller-scoped option that preserves non-TUI commentary suppression.
