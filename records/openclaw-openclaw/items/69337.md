## Summary

Keep open: the TUI visibility bug is source-reproducible, but this patch broadens a shared visible-text helper and can expose commentary/progress text in non-TUI surfaces that currently suppress it.

## What This Changes

The branch adds a commentary-phase fallback to `extractAssistantVisibleText` when no `final_answer` text is found.

## Best Possible Solution

Keep shared extraction final-answer-safe, add a TUI-scoped or caller-scoped opt-in commentary fallback, and cover both TUI no-final visibility and non-TUI commentary suppression.

## Reproduction Assessment

Yes at source level. A commentary-only phased assistant message returns no shared visible text on current main, and the TUI formatter reaches that helper through `extractContentFromMessage`; I did not run a live TUI session in this read-only review.

## Solution Assessment

No. The proposed global shared-helper fallback is broader than the TUI bug requires; the safer solution is a TUI-scoped or explicit caller option with regression coverage for non-TUI suppression.
