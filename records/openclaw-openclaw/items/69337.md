## Summary

Keep open: the TUI visibility symptom is source-reproducible, but this patch broadens a shared final-answer visibility helper that current Gateway preview and Control UI paths intentionally rely on to hide commentary-only text.

## What This Changes

The PR adds a commentary-phase fallback to `extractAssistantVisibleText` when no `final_answer` text is present.

## Best Possible Solution

Keep shared history, preview, and Control UI extraction final-answer-safe, then fix the TUI symptom in a TUI-specific live-stream projection or narrowly scoped output-normalization path with regression coverage and real TUI proof.

## Reproduction Assessment

Yes, by source inspection. A commentary-only phased assistant message currently returns no shared visible text, and the TUI stream assembler reaches that helper through `extractContentFromMessage`.

## Solution Assessment

No. The PR targets the right symptom, but applying the fallback in the shared helper changes Gateway preview and Control UI behavior that current tests intentionally protect.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9
