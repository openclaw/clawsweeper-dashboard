## Summary

Keep open: the TUI no-final symptom is source-reproducible, but this PR fixes it by broadening a shared visibility helper that current Gateway, Control UI, transcript, and shared tests use to suppress commentary-only assistant text, and the PR still lacks after-fix real TUI proof.

## What This Changes

The branch modifies `src/shared/chat-message-content.ts` to return commentary-phase text from `extractAssistantVisibleText` when no `final_answer` text exists.

## Best Possible Solution

Keep shared extraction final-answer-safe, add a TUI-scoped or explicit caller-option commentary fallback for streaming/no-final display, and cover both TUI visibility and non-TUI commentary suppression.

## Reproduction Assessment

Yes by source inspection, though I did not run a live TUI session. A commentary-only phased assistant message returns no shared visible text on current main, and the TUI stream path reaches that helper through `extractContentFromMessage`.

## Solution Assessment

No. The proposed global shared-helper fallback is broader than the TUI bug requires; the safer solution is a TUI-scoped fallback or explicit caller option with regression coverage for non-TUI suppression.
