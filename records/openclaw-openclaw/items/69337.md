## Summary

Keep open: the TUI no-final/commentary-only visibility problem is source-reproducible on current main, but this PR fixes it by changing a shared extraction contract that Gateway previews and Control UI currently rely on to suppress commentary-only assistant text.

## What This Changes

The branch adds a commentary-phase fallback to `extractAssistantVisibleText` when no `final_answer` text is present.

## Best Possible Solution

Keep shared extraction final-answer-safe and add a TUI-scoped or caller-scoped opt-in fallback, with regression coverage for TUI no-final visibility and non-TUI commentary suppression.

## Reproduction Assessment

Yes. A commentary-only phased assistant message returns no shared visible text on current main, and the TUI stream assembler reaches that helper through `extractContentFromMessage`; I did not run a live TUI session in this read-only review.

## Solution Assessment

No. The patch targets the right symptom, but changing the shared helper is too broad; the maintainable fix is a TUI-scoped fallback or explicit caller option that preserves non-TUI commentary suppression.
