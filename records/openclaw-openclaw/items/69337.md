## Summary

Keep open: the TUI symptom is plausible, but this patch broadens a shared assistant visibility contract and can expose commentary-only text in Gateway, Control UI, and transcript surfaces; it also lacks after-fix real TUI proof.

## What This Changes

The branch adds a commentary-phase fallback to the shared assistant visible-text helper when no `final_answer` text is present.

## Best Possible Solution

Keep shared extraction final-answer-safe, add a TUI-scoped or caller-scoped opt-in commentary fallback, and cover both TUI no-final visibility and non-TUI commentary suppression.

## Reproduction Assessment

Yes. A commentary-only phased assistant message returns no shared visible text on current main, and the TUI formatter reaches that helper through `extractContentFromMessage`; I did not run a live TUI session in this read-only review.

## Solution Assessment

No. The proposed global shared-helper fallback is broader than the TUI bug requires; the safer solution is a TUI-scoped or explicit caller option with regression coverage for non-TUI suppression.
