## Summary

Keep open, but do not merge this head as-is: the TUI no-final symptom is source-reproducible, yet the patch broadens a shared helper whose current contract intentionally suppresses commentary-only assistant text in Gateway and Control UI surfaces.

## What This Changes

The PR adds an unconditional commentary-phase fallback to `extractAssistantVisibleText` in `src/shared/chat-message-content.ts`.

## Best Possible Solution

Keep shared extraction final-answer-safe and add a TUI-scoped or explicit caller-scoped opt-in fallback, with tests proving both TUI visibility and non-TUI commentary suppression.

## Reproduction Assessment

Yes, from source inspection: a commentary-only phased assistant message returns no shared visible text, and the TUI stream assembler reaches that helper through `extractContentFromMessage`. I did not live-run a TUI session in this read-only review.

## Solution Assessment

No. The patch targets the right symptom, but changing the shared helper is too broad; the maintainable fix is a TUI-scoped fallback or caller option that preserves non-TUI commentary suppression.
