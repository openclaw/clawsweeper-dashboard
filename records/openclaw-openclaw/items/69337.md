## Summary

Keep open: the TUI symptom is source-reproducible, but this branch should not merge as-is because it broadens a shared final-answer-safe extraction helper and can expose commentary text in non-TUI surfaces that current tests intentionally keep hidden.

## What This Changes

The PR adds a commentary-phase fallback in `src/shared/chat-message-content.ts` so `extractAssistantVisibleText` returns commentary text when no `final_answer` text exists.

## Best Possible Solution

Preserve shared final-answer-only extraction and add a TUI-scoped or caller-scoped opt-in fallback, with tests proving both TUI visibility and non-TUI commentary suppression.

## Reproduction Assessment

Yes from source inspection: a commentary-only phased assistant message returns no visible shared text, and the TUI stream assembler reaches that helper through `extractContentFromMessage`. I did not live-run a TUI session in this read-only review.

## Solution Assessment

No. The patch targets the right symptom, but changing the shared helper is too broad; the maintainable fix is a TUI-scoped fallback or caller-scoped option that preserves non-TUI commentary suppression.
