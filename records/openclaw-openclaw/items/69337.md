## Summary

Keep open: the reported TUI symptom is source-reproducible, but this branch changes a shared visibility helper and would make commentary-only assistant text visible in Gateway previews and Control UI, so it is not safe to merge as written.

## What This Changes

The PR adds a commentary-phase fallback to shared `extractAssistantVisibleText` when no `final_answer` text is available.

## Best Possible Solution

Keep shared extraction final-answer/history-safe, then fix the TUI symptom in a TUI-specific live-stream projection path or by normalizing completed no-final-tag provider output before shared display extraction.

## Reproduction Assessment

Yes by source inspection. A phased assistant message with only commentary text currently returns no shared visible text, and the TUI stream assembler reaches that helper through `extractContentFromMessage`.

## Solution Assessment

No. The PR targets the right symptom but solves it at the wrong layer; a shared fallback would expose commentary-only text in non-TUI surfaces that current tests intentionally hide.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9
