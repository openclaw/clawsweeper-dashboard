## Summary

This PR should stay open because it targets a real TUI visibility symptom, but the current patch changes a shared visibility helper that other surfaces intentionally use to hide commentary-only assistant text. It also still lacks after-fix real TUI proof from the contributor.

## What This Changes

The PR adds a commentary-phase fallback to `extractAssistantVisibleText` when no `final_answer` text is present.

## Best Possible Solution

Keep shared history, preview, and Control UI extraction final-answer-safe, then fix the TUI symptom in a TUI-specific live-stream projection or narrowly scoped output-normalization path with regression coverage.

## Reproduction Assessment

Yes by source inspection. A commentary-only phased assistant message currently returns no shared visible text, and the TUI stream assembler reaches that helper through `extractContentFromMessage`.

## Solution Assessment

No. The PR targets the right symptom but applies the fallback in a shared extraction helper that current shared, gateway, and UI tests rely on to suppress commentary-only assistant text.
