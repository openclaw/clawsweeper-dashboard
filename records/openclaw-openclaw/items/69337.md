## Summary

Keep open for maintainer review. Current main still does not implement the PR's intended TUI commentary fallback, but the proposed one-line shared-helper fallback would also change gateway previews, history, live chat, and Control UI surfaces that currently and intentionally suppress commentary-only assistant text.

## What This Changes

The PR adds a commentary-phase fallback to `extractAssistantVisibleText` in `src/shared/chat-message-content.ts` when no `final_answer` text is present.

## Best Possible Solution

Keep the shared helper's commentary suppression contract for durable/history and operator UI surfaces, then solve the TUI symptom with a scoped TUI/live-stream projection path or by normalizing true completed assistant output into `final_answer` before it reaches display helpers.

## Reproduction Assessment

Yes. The source-level reproduction is a phased assistant message with only a commentary text block: current main's `extractAssistantVisibleText` returns `undefined`, and the TUI stream path reaches that helper through `extractContentFromMessage`.

## Solution Assessment

No. The PR targets the observable TUI symptom, but changing `extractAssistantVisibleText` globally is not the narrowest maintainable solution because several non-TUI surfaces rely on commentary-only suppression.
