## Summary

Keep this PR open for maintainer review. The patch is a narrow, source-backed fix for the linked P1 gateway restart/message-loss bug, but it intentionally changes auth cooldown reload behavior from restart-required fallback to managed hot reload, so the remaining question is maintainer acceptance of that compatibility-sensitive path.

## What This Changes

This PR adds an `auth.cooldowns` hot-reload rule and planner/reloader coverage so cooldown config edits refresh runtime config without scheduling a gateway restart.

## Best Possible Solution

Land the scoped hot-reload rule with the planner/reloader coverage once maintainers accept the auth reload compatibility risk, while keeping credential paths such as `gateway.auth.*` restart-required.

## Solution Assessment

Yes, this is the narrowest maintainable fix shape I found: add a hot reload prefix so the existing managed reloader refreshes the active runtime config, while credential paths remain restart-required.
