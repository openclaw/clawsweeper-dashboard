## What This Changes

The PR adds `CommandSource: "text"` for authorized iMessage inbound turns and a focused `/new` inbound-processing test.

## Best Possible Solution

Carry an actual iMessage text-command fact into the inbound decision, or emit a structured `CommandTurn`, and only mark authorized slash commands as text command turns with redacted real iMessage acknowledgement proof.

## Reproduction Assessment

Yes for source-level reproduction: current `main` sets iMessage `CommandAuthorized` without `CommandSource`/`CommandTurn`, while delivery policy only treats explicit command turns as automatic replies. I did not establish a live Messages.app failing path.

## Solution Assessment

No: `decision.commandAuthorized` is an authorization fact, not proof the inbound message was a slash command. The fix should key off actual control-command detection or pass a structured command turn from the iMessage decision.
