## Summary

Keep open: the broadened PR fixes the earlier xAI transport gap for `/think low|medium|high`, but it still mishandles explicit `/think off` for reasoning-capable Grok models by omitting the reasoning control even though xAI documents omission as defaulting to low and `none` as the disabled value.

## What This Changes

The PR adds xAI thinking-profile support plus runtime-compat and stream-wrapper changes so reasoning-capable Grok routes forward `/think` levels, with xAI tests and a changelog entry.

## Best Possible Solution

Keep the xAI-owned profile and transport changes, but make explicit `/think off` send xAI's disabled reasoning value while unsupported xAI routes continue stripping reasoning payloads.

## Reproduction Assessment

Yes. Source and dependency inspection show the PR maps xAI `off` to null/omission, and xAI docs state omitted `reasoning_effort` defaults to low while `none` disables reasoning.

## Solution Assessment

No. The policy-plus-transport direction is the right owner-boundary fix, but the implementation must preserve `/think off` semantics for reasoning-capable Grok models before merge.
