## Summary

Keep open: the xAI-owned policy and transport direction is useful, but the patch is not mergeable because it treats generic xAI `reasoning: true` as effort-configurable and still makes explicit `/think off` omit the disabled effort for `grok-4.3`.

## What This Changes

The PR makes xAI provider thinking profiles context-aware, forwards reasoning fields for reasoning-capable Grok routes, updates xAI tests, and adds a changelog entry.

## Best Possible Solution

Keep the xAI-owned policy and transport changes, but gate effort controls to documented effort-capable xAI models and send xAI's disabled `none` effort for explicit `/think off`.

## Reproduction Assessment

Yes. Source inspection plus dependency contracts reproduce both blockers: the PR maps `off` to `null`, pi-agent/pi-ai then omit disabled reasoning, and xAI docs say omission defaults to low; the same source path shows `reasoning: true` is too broad for configurable effort support.

## Solution Assessment

No. Provider-owned profile and transport handling is the right boundary, but the implementation needs a narrower effort-capability gate and an explicit disabled payload for off before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P2] Gate effort mapping by xAI's effort-capable models:** `extensions/xai/runtime-model-compat.ts:45-48`
  - body: `withCompat.reasoning === true` is broader than "accepts configurable effort": current xAI docs say Grok 4.20 and Grok 4.1 Fast reasoning routes reason automatically and reject `reasoning_effort`, while the catalog marks those families as reasoning-capable. This will forward `/think` payloads to routes that should keep stripping them, so gate the map and profile by documented effort-capable model ids instead of the generic reasoning flag.
  - confidence: 0.91
- **[P2] Send xAI's disabled effort for `/think off`:** `extensions/xai/runtime-model-compat.ts:26`
  - body: Mapping `off` to `null` makes pi-ai omit the Responses `reasoning` field, but xAI documents omitted effort as defaulting to low and `none` as the disabled value. Selecting Off on `xai/grok-4.3` would still run default reasoning, so map explicit off to xAI's disabled value and cover that outbound payload.
  - confidence: 0.95
