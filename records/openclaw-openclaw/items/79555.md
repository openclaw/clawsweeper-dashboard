## Summary

Keep open: the PR proves the xAI `/think` menu can expose levels, but it intentionally stops before the transport change needed for the linked Grok `reasoning_effort` request, so merging it would expose controls that current xAI requests still ignore.

## What This Changes

The PR changes the xAI plugin thinking profile hook to advertise off/low/medium/high for reasoning-capable xAI models and adds provider-policy tests.

## Best Possible Solution

Update the xAI provider profile, runtime mapping, and stream payload wrapper together so supported Grok routes expose and forward reasoning_effort while unsupported xAI routes remain off-only.

## Reproduction Assessment

Yes for the PR defect. Source inspection shows the branch advertises xAI low/medium/high, while current main still strips all xAI reasoning-effort fields and maps xAI thinking levels to null before dispatch.

## Solution Assessment

No. The policy-only approach is not the best fix for the linked request because the maintainable solution needs policy and transport support to agree before users see selectable levels.
