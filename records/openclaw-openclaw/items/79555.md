## Summary

Keep open: the branch proves the xAI policy/menu surface, but it intentionally leaves transport unchanged, so selectable levels still would not reach xAI and the linked provider feature is not complete.

## What This Changes

The PR wires the bundled xAI provider to return context-aware thinking profiles for reasoning-capable xAI models and adds provider-policy unit tests.

## Best Possible Solution

Update xAI provider policy, runtime compatibility, and stream payload mapping together so supported Grok 4.3 routes expose and send xAI's accepted reasoning effort while unsupported routes remain off-only.

## Reproduction Assessment

Yes for the PR defect: source inspection shows the branch advertises xAI levels while current xAI runtime code still strips reasoning controls and maps levels to null. The screenshots reproduce only the policy/menu surface, not an end-to-end xAI request.

## Solution Assessment

No. The policy-only approach is not the best fix because the user-visible selector and provider transport need to agree before users are offered levels that imply runtime effect.
