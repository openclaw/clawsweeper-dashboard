## Summary

Keep open: the PR fixes the missing non-off xAI reasoning transport path, but `/think off` still maps to omission for Grok 4.3 even though xAI documents omitted effort as `low` and `none` as disabled.

## What This Changes

The PR updates the xAI plugin thinking profile, runtime model compatibility, stream payload wrapper, xAI tests, and changelog so reasoning-capable Grok models forward `/think` levels.

## Best Possible Solution

Keep the xAI-owned policy and transport changes, but map explicit `off` to xAI's disabled `none` effort for reasoning-capable models while preserving stripping for non-reasoning routes.

## Reproduction Assessment

Yes, source-reproducible: the PR sets `off` to `null`, pi-ai omits disabled reasoning when `off` is null, and current xAI docs state omission defaults to low while `none` disables it. A live xAI call was not run.

## Solution Assessment

No. The provider-owned policy-plus-transport direction is the right boundary, but the implementation must preserve `/think off` semantics before merge.
