## Summary

Keep this draft PR open. Current main still uses legacy Slack section/fields rich progress drafts, and this PR has unique native plan/task_card plus plan_update/task_update work, but the latest patch has a source-level failure-path bug: finalization marks in-flight native progress tasks complete even when the turn is about to rethrow an error.

## What This Changes

The PR changes the Slack plugin to render progress drafts as Slack plan/task_card blocks and stream plan_update/task_update chunks through native Slack streaming, with Slack renderer, dispatch, and streaming tests.

## Best Possible Solution

Keep the plan/task_card and native chunk direction, but revise finalization so failed turns do not mark active Slack progress tasks complete, then re-run the focused Slack tests and mark the draft ready.

## Reproduction Assessment

Yes for the review finding: PR-head source shows dispatchError is captured, then native progress completion chunks are still built and stopped without consulting it. I did not run a live failing Slack turn in this read-only review.

## Solution Assessment

No, not as currently patched. The Slack-owned plan/task_card and structured chunk surfaces are the right direction, but failed-turn finalization needs to preserve an error/unfinished state instead of converting active tasks to complete.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Preserve error state when finalizing failed progress:** `extensions/slack/src/monitor/message-handler/dispatch.ts:1502-1505`
  - body: `dispatchError` can already be set here, but the finalizer still sends completion chunks for every in-flight progress line before rethrowing. A failed tool/model turn would leave Slack showing completed task cards for work that actually failed; skip completion or emit error-state chunks when `dispatchError` is present.
  - confidence: 0.84
