## Summary

Keep open. This draft PR addresses an unsolved Slack UX feature, but it still has a blocking correctness issue: failed/error runs can mark in-flight native progress task cards complete, and the supplied live proof is stale for the latest head.

## What This Changes

The PR enables Slack native progress streaming for progress mode with plan/task chunks, preserves legacy draft fallback rendering, adds progress identity metadata, and adds Slack/SDK tests plus a trajectory proof helper.

## Best Possible Solution

Revise the Slack native progress path so task finalization is success/error-aware, then provide redacted current-head Slack proof before maintainer review and landing through the linked feature issue.

## Reproduction Assessment

Yes for the review finding: PR-head source emits completion chunks from both final append and stream stop without success/error guards, so a failed run with in-flight progress can be marked complete. I did not run live Slack in this read-only review.

## Solution Assessment

No, not as currently patched. The native Slack plan/task-card direction fits the request, but finalization needs to gate completion on success or emit error-state chunks, followed by current-head real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9
