Keep this PR open: current main does not yet render Slack progress drafts as native plan/task cards, but the branch is still a draft with a concrete title-behavior regression against the latest PR discussion and no inspectable real Slack proof after the latest push.

## Best Possible Solution

Keep the Slack-owned renderer work in this PR, but revise it so hidden/no-label titles promote from meaningful progress and native streaming uses stable Slack task/plan update semantics, then attach redacted live Slack proof before merge.
