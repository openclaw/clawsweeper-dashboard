## Summary

Keep open: this is a focused Slack enhancement PR, but the current branch still regresses the hidden-progress-label contract, reuses a Slack block id across repeated draft edits, and lacks the requested redacted live Slack proof.

## What This Changes

The PR changes Slack rich progress drafts to render one Slack `plan` block with mapped task entries, preserves status-final draft/final handling, and adds renderer plus dispatch coverage.

## Best Possible Solution

Preserve `label: false`, omit or vary the Slack block id on draft edits, keep the existing fallback/status-final lifecycle, and attach redacted live Slack proof before merge.

## Reproduction Assessment

Yes for the review finding: source inspection shows `label: false` resolves to no title today, while the PR adds a default visible title. The broader Slack plan-card request is a feature, so normal bug reproduction is not applicable.

## Solution Assessment

No as written: the existing Slack rich-progress hook is the right seam, but the patch needs to preserve the hidden-title contract and avoid a stale constant `block_id` across edited draft iterations.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve hidden progress labels:** `extensions/slack/src/progress-blocks.ts:120`
  - body: When users configure `streaming.progress.label: false`, current rich rendering omits the label section while still showing progress rows. This change always sends a plan block titled `OpenClaw is working`, so hidden-label configs become visibly titled again; keep that contract, for example by falling back to the old row shape or avoiding a synthetic plan title when labels are disabled.
  - confidence: 0.91
- **[P3] Avoid reusing one Slack block id for every draft edit:** `extensions/slack/src/progress-blocks.ts:119`
  - body: Progress drafts edit the same Slack message repeatedly as task state changes, but this constant `block_id` is sent on every update. Slack documents `block_id` as an iteration-specific identifier for updated messages, so omit it here or vary it per draft update to avoid stale block identity in Slack clients.
  - confidence: 0.74
