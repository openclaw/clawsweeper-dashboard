## Summary

Keep open: this is a useful Slack-only opt-in feature with sufficient live proof, but the Slack patch-summary adapter drops the stable IDs that the PR adds, so native task cards can leave stale or duplicate patch tasks instead of updating in place.

## What This Changes

Adds an opt-in Slack `nativeTaskCards` progress mode that streams native plan/task chunks, adds Slack-only config/docs/schema metadata, preserves progress IDs in SDK events, and tests the renderer, dispatch, and stream helpers.

## Best Possible Solution

Keep the Slack-only opt-in native progress path, fix patch-summary identity propagation, and leave broader upstream identity/error-status work to explicit follow-ups if maintainers accept those limitations.

## Reproduction Assessment

Not applicable as a bug reproduction for the PR as a whole; this is an opt-in Slack feature. The blocking finding is source-reproducible because the core patch-summary callback supplies IDs while the Slack adapter omits them before building native task chunks.

## Solution Assessment

No, not quite yet. The Slack-only opt-in shape is maintainable, but the adapter must forward patch IDs before this is the best implementation of stable native progress cards.
