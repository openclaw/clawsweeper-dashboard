## Summary

Keep open: current main still lacks native Slack plan/task-card progress rendering, and this draft PR implements the right Slack-owned direction but still has two source-visible correctness blockers before it is safe to merge.

## What This Changes

The PR changes Slack progress drafts and native streaming to render plan/task_card blocks and emit plan_update/task_update chunks, with related progress-line identity tests.

## Best Possible Solution

Keep the native plan/task_card approach, but revise completion handling to preserve error state and use stable task identities before landing with focused Slack tests and repaired-path proof.

## Reproduction Assessment

Yes for the review findings: PR-head source shows completion chunks can be emitted without checking failed final/dispatch state, and task_update IDs include the current array index. I did not run a live Slack failure or pruning scenario in this read-only review.

## Solution Assessment

No, not as currently patched. The Slack-owned plan/task_card and structured chunk direction is maintainable, but the implementation needs stable task identities and success-aware finalization before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Only complete progress tasks after successful finals:** `extensions/slack/src/monitor/message-handler/dispatch.ts:831-839`
  - body: This path builds completion chunks for any streamed final, and the later stop path can do the same after `dispatchError` has already been captured. Failed turns can therefore rewrite in-flight task cards to `complete`; gate completion on a successful final/dispatch or emit error-state chunks instead.
  - confidence: 0.87
- **[P2] Use stable native task update IDs:** `extensions/slack/src/progress-blocks.ts:102`
  - body: `task_id` and `task_update.id` still include the task's current array index. Once older progress lines are pruned, a remaining task such as `cmd-2` can change from `cmd_2_2` to `cmd_2_1`, so Slack treats it as a different task and can leave stale cards; use the stable line id when present and only synthesize an index for id-less lines.
  - confidence: 0.84
