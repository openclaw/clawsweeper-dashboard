## Summary

Keep open: the PR is useful and well-proven, but current head still has a source-reproducible Slack delivery bug where opted-in progress mode can fall back into native text streaming when no progress card was started.

## What This Changes

The PR adds Slack-only opt-in native progress task cards using Slack stream chunks, plus config/schema/docs updates and focused Slack progress tests.

## Best Possible Solution

Keep the opt-in Slack-native task-card design, but fix the no-card final-delivery fallback before merge and leave broader upstream identity/error-state propagation to separate core or SDK work.

## Reproduction Assessment

Not applicable for the original feature request, but yes for the blocking review finding: source inspection shows `nativeTaskCards=true` makes progress mode use native streaming while the native progress updater does nothing with zero lines, so a final-only payload reaches the generic text stream start path.

## Solution Assessment

No for current head; the opt-in native card direction is maintainable, but the final-only/no-card fallback should keep normal progress-mode delivery instead of silently becoming partial-style native text streaming.
