## What This Changes

This draft PR changes Slack rich progress drafts to emit one Slack plan block with task entries, updates the status-final dispatch path, and adds renderer/dispatch tests.

## Best Possible Solution

Keep the existing Slack progress/final lifecycle and fallback text, but update the branch to preserve hidden labels, avoid stale block identity on edited drafts, and attach redacted live Slack proof.

## Reproduction Assessment

Not applicable for the broader request because this is a Slack presentation feature, not a broken existing behavior report. For the blocking review finding, source inspection is high-confidence: label false returns undefined on current main, while the PR converts that into a visible default title.

## Solution Assessment

No as written. The existing Slack rich progress renderer is the right narrow seam, but the patch needs to preserve the hidden-title contract and avoid a constant block_id across repeated draft edits.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve hidden progress labels:** `extensions/slack/src/progress-blocks.ts:120`
  - body: When users configure `streaming.progress.label: false`, `resolveChannelProgressDraftLabel` returns `undefined` and current rich rendering omits the title while keeping progress rows. This branch always sends a plan block titled `OpenClaw is working` whenever tasks exist, so hidden-title configs become visibly titled again; preserve that contract before merge.
  - confidence: 0.91
- **[P3] Avoid reusing one Slack block id for every draft edit:** `extensions/slack/src/progress-blocks.ts:119`
  - body: Progress drafts repeatedly edit the same Slack message, but this constant `block_id` is sent on every update. Slack documents `block_id` as unique per message iteration, so omit it or vary it per draft update to avoid stale block identity in Slack clients. ([docs.slack.dev](https://docs.slack.dev/reference/block-kit/blocks/plan-block/?utm_source=openai))
  - confidence: 0.74
