## Summary

Keep open. The PR is a plausible Slack-native progress rendering implementation, but it is still draft, current main does not already provide this behavior, and PR head still has a correctness issue where failed Slack runs can finalize in-flight native task cards as complete.

## What This Changes

This PR enables Slack native progress streaming for progress mode, maps progress lines to Slack `plan_update`/`task_update` chunks, extends progress-line identity handling, adds Slack/SDK tests, and adds a Slack trajectory proof script.

## Best Possible Solution

Land this through the paired Slack feature issue after success/error-aware finalization, maintainer UX review of the accepted scope, focused regression coverage, and fresh current-head Slack proof.

## Reproduction Assessment

Yes for the review finding: PR-head source shows native progress completion chunks are emitted from final append and stream stop paths without success/error guards. I did not run live Slack in this read-only review.

## Solution Assessment

No, not as currently patched. The native Slack plan/task-card direction fits the request, but finalization should be success/error-aware before the PR can be considered correct.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Gate native task completion on successful finals:** `extensions/slack/src/monitor/message-handler/dispatch.ts:845-858`
