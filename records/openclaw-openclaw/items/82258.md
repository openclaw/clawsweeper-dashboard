## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Gate native task completion on successful finals:** `extensions/slack/src/monitor/message-handler/dispatch.ts:845-858`
  - body: This builds completion chunks whenever final text is appended, and the stop path repeats the same behavior, without checking `payload.isError` or `dispatchError`. Failed Slack runs can show active task cards as complete; gate completion on successful finals or emit error-state chunks.
  - confidence: 0.88
- **[P2] Keep native progress task IDs stable:** `extensions/slack/src/progress-blocks.ts:128-130`
  - body: `task_id` and `task_update.id` still include the task's current sliced-array index even when `line.id` exists. When max-line pruning drops older lines, the same task changes identity and Slack can leave stale cards; use the stable line id directly when available.
  - confidence: 0.9
