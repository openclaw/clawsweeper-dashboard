## Summary

Keep open. The PR has a concrete P2 correctness issue: it turns `label: false` rich Slack progress into a visible `Thinking` plan title even though current SDK, docs, and Slack config hints say the progress title is hidden; the plan-card direction and supplied live proof are useful, so this needs contributor revision or maintainer product approval rather than cleanup close.

## What This Changes

This PR changes Slack rich progress drafts from section/field Block Kit output to native Slack `plan` blocks, preserves recent progress lines across draft boundaries, and adds focused renderer/dispatch tests.

## Best Possible Solution

Keep the Slack plan-card renderer at the existing rich progress-draft hook, but preserve the documented hidden-title behavior for `label: false` unless maintainers intentionally update the config/docs contract.

## Reproduction Assessment

Yes for the PR finding: source inspection shows `label: false` resolves to `undefined` on current main, while PR head converts that missing title into visible `Thinking`. The linked plan-card request itself is a feature request, not a current-main bug reproduction path.

## Solution Assessment

No, not as currently patched. The renderer and Slack dispatch hook are the right narrow place, but the fallback title overrides an existing hidden-label contract without a corresponding maintainer-approved product/docs change.
