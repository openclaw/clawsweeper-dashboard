## Summary

Keep open. The Slack plan/task-card direction is plausible and the PR body supplies successful-path live Slack proof, but PR head still has two source-visible correctness defects and the branch is currently draft/conflicting.

## What This Changes

The PR changes Slack progress rendering and native streaming to emit plan/task_card blocks plus plan_update/task_update chunks, with SDK progress-line identity support and focused Slack tests.

## Best Possible Solution

Keep the native plan/task-card direction, but repair success-aware finalization and stable task identities before landing with focused Slack tests and updated proof if runtime behavior changes.

## Reproduction Assessment

Yes for the PR findings via source inspection: PR head emits completion chunks without a success/error guard and derives task_update IDs from the current array index after capping. I did not run a live Slack failure or pruning case.

## Solution Assessment

No, not as currently patched. The native Slack plan/task chunk approach is the right surface, but finalization and task identity semantics need repair before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Only complete progress tasks after successful finals:** `extensions/slack/src/monitor/message-handler/dispatch.ts:831-839`
  - body: This path builds completion chunks for every streamed final, including error finals, and the stop path does the same after dispatchError can already be set. Failed turns can therefore rewrite in-flight task cards to complete; gate completion on a successful final/dispatch or emit error-state chunks instead.
  - confidence: 0.86
- **[P2] Keep native task update IDs stable:** `extensions/slack/src/progress-blocks.ts:102`
  - body: task_id/task_update.id includes the task's current array index even when the line has a stable id. Once older progress lines are pruned, a remaining task changes IDs, so Slack can treat it as a different task and leave stale cards instead of updating the original.
  - confidence: 0.84
