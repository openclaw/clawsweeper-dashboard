## Summary

Keep open. Current main does not implement native Slack progress plan/task cards, and this active draft PR is the paired implementation path, but the latest head still completes in-flight native tasks on error paths and the live Slack proof predates the current head.

## What This Changes

The draft PR enables Slack native progress streaming with plan/task update chunks, adds progress task identity support in the plugin SDK, updates Slack streaming/progress tests, and adds a Slack trajectory proof helper.

## Best Possible Solution

Finish this through the paired Slack feature issue with success/error-aware native task finalization, focused regression coverage, updated user-facing Slack help/docs, and fresh current-head Slack proof; keep the broader upstream tool identity plumbing as a separate follow-up if maintainers accept that scope split.

## Reproduction Assessment

Yes for the review finding: PR-head source shows native progress completion chunks are emitted from both final append and stream stop paths without success/error guards. I did not run live Slack in this read-only review.

## Solution Assessment

No, not as currently patched. The native Slack plan/task-card direction fits the request, but completion needs to be success/error-aware and the PR needs current-head real Slack proof before merge.
