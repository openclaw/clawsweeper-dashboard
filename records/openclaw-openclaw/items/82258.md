## Summary

Keep open: current main still lacks the requested Slack plan-card progress rendering, and this PR is the focused implementation path, but the current head is draft and still changes the documented `label: false` title-hiding contract by rendering a visible `Thinking` plan title.

## What This Changes

The PR changes Slack rich progress drafts to render existing progress lines as native Slack `plan` blocks while keeping fallback text and the one-progress-draft plus final-reply lifecycle.

## Best Possible Solution

Keep the Slack plan-card feature in the Slack plugin, but preserve `label: false` as a hidden-title contract or land an explicit maintainer-approved contract/docs change before merge.

## Reproduction Assessment

Yes for the review finding: PR-head source and tests show `label: false` becomes a `plan` titled `Thinking`, while current main source, docs, and Slack config hints define false as hiding the title. For the linked feature request, reproduction is not applicable because it is a new Slack presentation capability that current main lacks.

## Solution Assessment

No, not as currently patched; the Slack renderer and dispatch hook are the narrow maintainable place for plan-card rendering, but hidden-label behavior should be preserved unless maintainers explicitly change the public contract.

## Review Findings

Overall correctness: patch is incorrect
