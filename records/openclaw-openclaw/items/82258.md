## Summary

Keep open: this focused Slack UX PR is still draft, lacks after-fix live Slack proof, and the current head still introduces regressions around hidden progress labels and Slack block identity.

## What This Changes

The PR replaces Slack rich progress draft section-field blocks with a single Slack `plan` block, preserves recent task lines across draft boundaries, and updates focused renderer/dispatch tests.

## Best Possible Solution

Update this PR to preserve `label: false`, omit or vary Slack `block_id` on edited drafts, keep the existing fallback/lifecycle, add regression coverage, and attach redacted live Slack proof before merge.

## Reproduction Assessment

Yes for the blocking review finding: source inspection shows `label: false` resolves to `undefined` on main, while PR head line 120 substitutes a visible default plan title. The broader plan-card request is a feature, so normal bug reproduction is not applicable.

## Solution Assessment

No as written: the existing Slack rich-progress hook is the right narrow seam, but the branch needs to preserve the hidden-title contract and avoid reusing one Slack `block_id` across repeated draft edits.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve hidden progress labels:** `extensions/slack/src/progress-blocks.ts:120`
  - body: When users configure `streaming.progress.label: false`, the resolver passes `undefined` so current rich rendering omits the title while still showing progress rows. This line turns that hidden-label case into a visible `OpenClaw is working` plan title whenever tasks exist, regressing the documented config contract.
  - confidence: 0.92
- **[P3] Avoid reusing one block id for every draft edit:** `extensions/slack/src/progress-blocks.ts:119`
  - body: Progress drafts repeatedly edit the same Slack message as task state changes, but this constant `block_id` is sent on every update. Slack documents `block_id` as unique per message and per updated-message iteration, so omit it or vary it per draft update to avoid stale block identity in Slack clients.
  - confidence: 0.76
