## Summary

Keep this open: current main and the latest release include important mitigations, but neither implements default gateway-side model-tag verification nor adds authoritative provider/model identity to `message_sending` itself.

## What This Changes

The issue requests authoritative provider/model identity in outbound delivery hooks or gateway-side enforcement so agent self-tags cannot silently disagree with the runtime model.

## Best Possible Solution

Keep this open and choose one canonical model-provenance contract, likely building on `reply_payload_sending.usageState`, before adding default tag enforcement or expanding `message_sending`.
