## Summary

Keep open: current main still ignores `restart.coalesced` from a successful `update.run`, so the Update button can clear its busy state without visible feedback even though the gateway returned coalesced restart metadata. The prior ClawSweeper review remains accurate; related backend restart fixes do not add the missing Control UI handling.

## What This Changes

The issue asks the Control UI Update button to show visible feedback when `update.run` is coalesced because another update restart is already in flight.

## Best Possible Solution

Teach `runUpdate` to treat a coalesced restart as visible in-progress or informational feedback while preserving backend restart coalescing and existing version verification.

## Reproduction Assessment

Yes, source-reproducible: a controller test can mock `update.run` returning `ok: true`, `result.status: "ok"`, and `restart.coalesced: true`, then observe that current `runUpdate` leaves `updateStatusBanner` null and clears `updateRunning`. The gateway source returns that same coalesced restart shape.

## Solution Assessment

Yes: the narrowest maintainable fix is in Control UI response handling because the gateway already exposes the coalesced restart state. No backend policy, config option, or product decision is needed for basic feedback.
