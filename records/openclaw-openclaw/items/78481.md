## What This Changes

The issue asks the Control UI to show visible feedback when `update.run` coalesces because a gateway restart is already in flight.

## Best Possible Solution

Review and either land or reject https://github.com/openclaw/openclaw/pull/93082; if rejected, keep the fix as a narrow Control UI controller change that consumes existing `restart.coalesced` metadata without changing backend restart policy.

Yes, source-reproducible: the gateway returns `restart.coalesced` metadata, while current `runUpdate` returns through the `ok` path before surfacing that state. I did not run a live browser update click in this read-only review.

## Solution Assessment

Yes, the best fix is narrow UI consumption of existing restart metadata while preserving the current restart and managed-service handoff policy. Broader backend coalescing and mixed-version recovery belong to adjacent update/restart items.
