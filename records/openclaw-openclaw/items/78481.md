## Summary

Keep open: current main still drops the successful `update.run` response into the normal success path without reading `restart.coalesced`, so the Control UI can clear its busy state with no visible feedback while the gateway reports an already in-flight restart.

## What This Changes

Report that the Control UI Update button gives visible feedback when `update.run` is coalesced by an already in-flight restart.

## Best Possible Solution

Teach `runUpdate` to surface `restart.coalesced` as an in-progress or informational update state while preserving the backend restart coalescing contract and existing version verification.

## Reproduction Assessment

Yes, source-reproducible: mock `update.run` returning `ok: true`, `result.status: "ok"`, and `restart.coalesced: true`; current `runUpdate` returns through the normal success path without setting `updateStatusBanner` and then clears `updateRunning`.

## Solution Assessment

Yes: the narrowest maintainable fix is Control UI response handling because the gateway already exposes the coalesced restart state. No backend policy, config option, or product decision is needed for basic feedback.
