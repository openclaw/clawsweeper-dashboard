## Summary

Keep open: current main and v2026.6.8 still ignore `restart.coalesced` on the successful Control UI `update.run` path, and the open closing PR is the implementation candidate rather than a reason to close before merge.

## What This Changes

The issue asks for visible Control UI feedback when `update.run` returns a coalesced gateway restart instead of leaving the Update button apparently idle.

## Best Possible Solution

Review and either land or reject https://github.com/openclaw/openclaw/pull/93082; if rejected, keep the repair as a narrow Control UI controller change that renders existing `restart.coalesced` metadata without changing backend restart policy.

## Reproduction Assessment

Yes, source-reproducible: the gateway returns `restart.coalesced` metadata, while current `runUpdate` returns through the successful update path before surfacing that state. I did not run a live browser update click in this read-only review.

## Solution Assessment

Yes, the best fix is narrow UI consumption of existing restart metadata while preserving the current restart and managed-service handoff policy. Broader backend coalescing and mixed-version recovery belong to adjacent update/restart items.
