## Summary

Keep open: current main still has a real Control UI feedback gap because `runUpdate` treats a coalesced successful restart as ordinary success, clears the busy state, and leaves no visible banner even though the gateway returns `restart.coalesced`.

## What This Changes

The issue requests visible Control UI feedback when `update.run` succeeds but its restart request is coalesced because a prior restart/update is already in flight.

## Best Possible Solution

Update Control UI `runUpdate` response handling to surface coalesced restart metadata with visible in-progress or informational feedback and focused regression coverage, while leaving backend restart coalescing unchanged.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction: mock `update.run` to return `ok: true`, `result.status: "ok"`, and `restart.coalesced: true`, then observe that `runUpdate` clears `updateRunning` and leaves `updateStatusBanner` null.

## Solution Assessment

Yes. The narrowest maintainable fix is in Control UI response handling because the gateway already exposes the coalesced restart state; no backend policy, config option, or product decision is needed for basic feedback.
