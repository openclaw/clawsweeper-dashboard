## What This Changes

The issue reports that the Control UI Update button gives no visible feedback when `update.run` succeeds with an already coalesced gateway restart.

## Best Possible Solution

Refresh, prove, and land https://github.com/openclaw/openclaw/pull/93082 if maintainers accept the narrow UI fix; if not, keep the replacement fix limited to consuming existing `restart.coalesced` metadata without changing backend restart policy.

## Solution Assessment

Yes, pending PR proof: rendering the existing gateway metadata in the Control UI controller is the narrowest maintainable fix for the reported feedback gap. Backend restart scheduling and stale-token repair belong to adjacent updater work.
