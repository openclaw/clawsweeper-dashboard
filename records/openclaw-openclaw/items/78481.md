## Summary

Keep open: current main still has a source-confirmed Control UI feedback gap. The gateway returns structured `restart.coalesced` data for a coalesced `update.run`, but `runUpdate` ignores that field and clears the busy state without showing a banner.

## What This Changes

The issue requests visible Control UI feedback when an `update.run` request coalesces with an already in-flight update restart.

## Best Possible Solution

Teach the Control UI to treat `restart.coalesced` from `update.run` as a visible in-progress or informational state, with regression coverage, while leaving backend restart coalescing intact.

## Reproduction Assessment

Yes, source-reproducible: a controller test can mock `update.run` returning `{ ok: true, result: { status: "ok" }, restart: { coalesced: true, delayMs: 0 } }` and observe that current `runUpdate` sets no banner and clears `updateRunning`. The gateway path also clearly returns `coalesced: true` for already in-flight restarts.

## Solution Assessment

Yes: the narrowest maintainable fix is in the Control UI response handling because the gateway already exposes the coalesced state. No backend policy, config option, or product decision is needed for basic feedback.
