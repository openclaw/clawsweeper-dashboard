## Summary

Keep open for maintainer review: the patch looks correct and the supplied browser diagnostics are strong, but it deliberately removes service-worker handling for all top-level Control UI navigations, so maintainers need to accept the offline/PWA navigation compatibility tradeoff before merge.

## What This Changes

The PR adds an early `event.request.mode === "navigate"` return in `ui/public/sw.js`, leaving asset and API/RPC/plugin routing unchanged.

## Best Possible Solution

Land the navigation bypass if maintainers accept the offline/PWA navigation fallback removal, and keep https://github.com/openclaw/openclaw/issues/87268 as the canonical bug until the fix merges.

## Reproduction Assessment

Yes. Source inspection on current main shows same-origin non-asset navigations go through `event.respondWith(fetch(...))`, and the PR body supplies a concrete Firefox/Caddy before/after flow; I did not rerun the live proxy setup locally.

## Solution Assessment

Yes, if maintainers accept losing offline app-shell navigations. The early return restores the browser-native network path and preserves asset/API/RPC/plugin branches; a 401-only variant would be more complex and should be a maintainer-chosen refinement.
