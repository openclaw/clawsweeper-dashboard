## Summary

Keep open. Current main already has the baseline artifact RPC and SDK surface from the merged maintainer path, but this PR also proposes new cursor/type filtering and Hyprland capture work that is not fully present on main and the branch still has blocking correctness and security concerns.

## What This Changes

The PR adds artifact Gateway/SDK RPC schemas and handlers with cursor pagination/type filtering, changes artifact provenance filtering, updates SDK/docs/generated Swift/tests, and adds a Hyprland screenshot capture config path.

## Best Possible Solution

Keep the merged #74926 artifact API as the base. If cursor/type filtering is wanted, land it as a focused Gateway/SDK follow-up with a stable cursor tiebreaker such as message sequence plus content index or artifact id, preserve fail-closed provenance semantics, and split the Hyprland capture work into a browser-owned change with trusted binary resolution.

## Reproduction Assessment

Yes. The PR cursor bug reproduces with one transcript message containing two artifact blocks, then artifacts.list with limit 1 followed by the returned cursor; the second artifact is skipped because the cursor only advances by messageSeq. The provenance and Hyprland wiring issues are high-confidence static checks against the diff and current resolver behavior.

## Solution Assessment

No. The branch overlaps the merged maintainer artifact implementation and mixes a public artifact API extension with browser capture work; a narrower follow-up on top of main is the safer route.

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Keep run and task artifact queries provenance-scoped:** `src/gateway/server-methods/artifacts.ts:232-235`
  - body: This changes runId/taskId filtering to keep messages that lack explicit provenance, so a run-scoped query can return unrelated untagged artifacts from the same session. Keep the current fail-closed behavior or add a separate proven mapping before returning these artifacts.
  - confidence: 0.9
- **[P2] Preserve artifacts that share a message sequence:** `src/gateway/server-methods/artifacts.ts:388`
  - body: The cursor only encodes messageSeq. If one transcript message contains more artifacts than the page limit, the next request filters with messageSeq greater than the cursor and permanently skips remaining artifacts from that message. Include a tiebreaker such as content index or artifact id.
  - confidence: 0.95
- **[P2] Wire the Hyprland toggle through resolved config:** `extensions/browser/src/browser/routes/agent.snapshot.ts:493`
  - body: The route checks ctx.state().resolved.hyprlandCapture, but the normal browser config resolver does not copy the raw setting into ResolvedBrowserConfig. In a real gateway the condition remains falsey, so the new capture path cannot be enabled outside test setup.
  - confidence: 0.9
- **[P3] Resolve Hyprland tools from trusted paths:** `extensions/browser/src/browser/hyprland-capture.ts:8`
  - body: captureWithHyprland spawns hyprctl and grim by command name through PATH. Use resolveSystemBin before spawning so enabling this Gateway path cannot execute a shadowed binary from an untrusted PATH entry.
  - confidence: 0.78
