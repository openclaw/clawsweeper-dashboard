## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep run-scoped artifacts provenance-limited:** `src/gateway/server-methods/artifacts.ts:226-230`
  - body: The branch's provenance change filters only when a message carries `runId` or `taskId`. A run-scoped query first resolves to the owning session, so untagged artifact messages in the same session can pass and be returned under the requested run. Keep the stricter current-main behavior from #74926 or require explicit transcript provenance.
  - confidence: 0.88
- **[P2] Use a cursor with an artifact tiebreaker:** `src/gateway/server-methods/artifacts.ts:388`
  - body: The cursor encodes only `messageSeq`. If one transcript message has more artifacts than the page limit, the next request filters to later message sequences and skips the remaining artifacts from the same message. Include a stable tiebreaker such as content index or artifact id.
  - confidence: 0.95
- **[P2] Resolve Hyprland config through the browser resolver:** `extensions/browser/src/browser/routes/agent.snapshot.ts:493`
  - body: The route checks `ctx.state().resolved.hyprlandCapture`, but the branch only adds schema/type surface and does not populate the resolved browser config through `resolveBrowserConfig`. Real config will not enable this path outside test-only state injection.
  - confidence: 0.9
- **[P3] Resolve Hyprland tools from trusted system paths:** `extensions/browser/src/browser/hyprland-capture.ts:8`
  - body: The new helper spawns `hyprctl` and `grim` by command name through `PATH`. Since this is a Gateway subprocess path, resolve those binaries through the existing trusted system-bin helper before spawning.
  - confidence: 0.78
