## Summary

Keep open: current main already contains the merged #74926 artifact RPC base, but this branch still proposes additional cursor/type-filter and Hyprland behavior and remains blocked by provenance scoping, cursor correctness, SDK compatibility, and browser-config wiring issues.

## What This Changes

This PR adds Gateway/SDK artifact cursor pagination and type filters, generated protocol/docs updates, a revised run/task artifact provenance rule, and a Hyprland screenshot capture/config path.

## Best Possible Solution

Rebase on merged #74926, retain fail-closed provenance and shipped SDK signatures, then make cursor/type filtering an additive artifact API follow-up with artifact-level cursors; handle Hyprland as a separately owned browser change.

## Reproduction Assessment

Yes. The cursor bug is source-reproducible with two artifacts sharing one messageSeq and limit: 1, and the provenance issue follows directly from run/task queries retaining untagged messages that current main excludes.

## Solution Assessment

No. This is not the best merge vehicle until it preserves #74926’s scoping and SDK contract, fixes cursor tiebreaking, and splits or fully wires the Hyprland browser behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Keep scoped artifact queries fail-closed:** `src/gateway/server-methods/artifacts.ts:233-237`
  - body: This keeps messages that lack explicit run/task provenance in runId/taskId queries, so a scoped request can return unrelated untagged artifacts from the resolved session. Current main and tests require those messages to be excluded unless provenance is present.
  - confidence: 0.9
- **[P2] Add an artifact-level cursor tiebreaker:** `src/gateway/server-methods/artifacts.ts:388-397`
  - body: The cursor encodes only messageSeq and the next page filters `messageSeq > cursorSeq`. If one transcript message has multiple artifacts and the page limit splits that message, the remaining artifacts from that message are skipped.
  - confidence: 0.95
- **[P2] Preserve the shipped artifact helper signatures:** `packages/sdk/src/client.ts:756-765`
  - body: Current main documents and tests `oc.artifacts.get(id, query)` and `download(id, query)` with required query scope, but this branch switches get/download to param-object calls and allows list with no params. Keep the shipped shape or add overloads so existing SDK consumers do not break.
  - confidence: 0.86
- **[P2] Return hyprlandCapture from resolved config:** `extensions/browser/src/browser/routes/agent.snapshot.ts:496-497`
  - body: The screenshot route checks `ctx.state().resolved.hyprlandCapture`, but the normal browser config resolver is not updated to return that field. In a real gateway the new Hyprland branch stays disabled even when the raw config validates.
  - confidence: 0.88
- **[P3] Resolve Hyprland tools from trusted system paths:** `extensions/browser/src/browser/hyprland-capture.ts:9`
  - body: The helper spawns `hyprctl` and `grim` by command name through `PATH`. Resolve those binaries through the trusted system-binary helper before enabling this local capture path so a shadowed command is not executed.
  - confidence: 0.78
