Overall confidence: 0.84

Full review comments:

- **[P2] Build warning scans from resolved watch roots:** `src/config/validation.ts:905`
  - body: This always scans `<workspace>/memory`, but QMD only watches resolved `qmd.collections`, and `memory.qmd.includeDefaultMemory: false` removes the default memory collections. A gateway with QMD default memory disabled and a large stale `memory/` tree will still get told to disable `memorySearch.sync.watch` even though that tree is not watched; derive the scan roots from the same builtin/QMD watcher resolution or gate the default root on the active backend config.
  - confidence: 0.87
- **[P2] Deduplicate synchronous validation scans:** `src/config/validation.ts:930-938`
  - body: `opendirSync(..., { recursive: true })` and the read loop run inside config validation for each default/agent candidate, including shared default `extraPaths`. A multi-agent gateway with a slow shared path can repeatedly block startup or config-save validation up to the scan cap per agent; cache scan results per resolved path during one validation pass or otherwise avoid repeated synchronous walks.
  - confidence: 0.76
