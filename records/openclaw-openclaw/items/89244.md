## Summary

Keep open: the PR is a useful follow-up to reduce memory-watch warning noise, but it still scans default QMD memory roots even when QMD config disables those roots, so it can keep producing the false-positive warning it is meant to remove.

## What This Changes

The PR narrows the local gateway memory-watch config warning to large configured memory trees by adding a bounded Markdown-file scan and focused validation-policy tests.

## Best Possible Solution

Align the warning scan with the same resolved backend/watch collection roots used by the memory managers, or at least gate default roots on QMD backend and `includeDefaultMemory`, while preserving the bounded threshold scan and adding a regression test.

## Reproduction Assessment

Yes from source inspection: with `memory.backend: "qmd"`, `memory.qmd.includeDefaultMemory: false`, and more than 2,000 files under `workspace/memory`, the PR still scans that default root and can emit the warning even though QMD does not configure that collection. I did not run tests because this review is read-only.

## Solution Assessment

No. The bounded scan is a reasonable mitigation, but the maintainable fix should derive candidate roots from the resolved memory backend/watch configuration or mirror its backend and `includeDefaultMemory` gates.
