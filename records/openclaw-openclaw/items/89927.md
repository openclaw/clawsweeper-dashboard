## Label Justifications

- `P1`: The report describes a broken long-running agent workflow that can duplicate paid subagent work and wedge active sessions for real users.
- `impact:session-state`: The core problem is concurrent lanes mutating shared session state, heartbeat state, worklog state, and transcript locks.
- `impact:message-loss`: The report says subagent work can be dispatched and billed without appearing in the orchestrator worklog or producing delivered artifacts.
- `impact:crash-loop`: The reported lock contention can wedge a session until the Gateway is restarted, which is a process/session availability failure.

## Summary

Keep open: current main still has a Gateway timeout fallback path that starts a fresh `gateway-fallback-*` embedded run without first aborting or quiescing the already accepted Gateway agent run. Related stale-lock cleanup and prior ACP child cleanup exist, so the remaining issue should be narrowed to the accepted-run timeout fallback ownership decision.

## What This Changes

The issue requests single-owner handling for Gateway timeout fallback sessions so a `gateway-fallback-*` lane cannot run concurrently with the original accepted Gateway agent run or duplicate subagent work.

## Best Possible Solution

Define a single-owner Gateway timeout fallback contract: when an accepted Gateway run times out, either abort/quiesce the original run before embedded fallback starts or suppress fallback unless ownership transfer is confirmed.

## Reproduction Assessment

Yes for the central source path: current main's timeout fallback starts a fresh embedded `gateway-fallback-*` run without invoking the existing accepted-run abort path. The full billed-duplicate/wedged-session sequence was not live-reproduced from the provided report.

## Solution Assessment

Unclear until maintainers choose the timeout semantics. Aborting or quiescing the accepted Gateway run before embedded fallback appears to be the right owner-boundary fix, while stale-lock cleanup and prior ACP cleanup should stay as existing supporting mechanisms.
