## Summary

Keep open: current main still has the Discord preflight branch that drops ordinary guild text control commands before the later authorization and delivery paths can run.

## What This Changes

The issue requests that Discord guild text messages such as `/steer ...` no longer vanish when Discord sends them as ordinary messages instead of native slash-command interactions.

## Best Possible Solution

Keep the native Discord interaction echo drop, but route ordinary guild text control-command messages through the existing Discord command authorization and text-command handling path with regression coverage.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: call `preflightDiscordMessage` for a guild text message with normal message type and content like `/steer test`; current main returns `null` at the early `hasControlCommand()` guard. I did not live-run Discord in this read-only pass.

## Solution Assessment

Yes, the requested direction is the narrowest maintainable fix: remove or relocate the early silent return so the existing command gate and command reply behavior handle the message, while preserving the later `ChatInputCommand`/`ContextMenuCommand` echo drop.
