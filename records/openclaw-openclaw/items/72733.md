## Summary

Keep open for maintainer security/product review: current main does not implement this reversible guardrail, and the PR still has source-level redaction/storage defects plus missing real behavior proof.

## What This Changes

This PR adds an opt-in reversible Security Guardrail that redacts sensitive prompt values into `[VAULT_N]` tokens, restores them for tools/replies, and adds config, UI metadata, docs, and a small vault test.

## Best Possible Solution

Land only after maintainers choose the guardrail boundary and the implementation covers every model-visible prompt surface, block-secret detection, reversible storage lifecycle, docs/navigation, focused tests, and redacted live proof.

## Reproduction Assessment

Yes, at source level for the review defects: the PR forwards `transcriptPrompt` unchanged even though current main can submit it as the visible model prompt, and regex-only scanning applies multiline private-key patterns one line at a time. I did not run a live local-model/provider/tool setup in this read-only review.

## Solution Assessment

No. The direction is useful, but this patch is not the best merge shape until the security boundary, all prompt surfaces, multiline secret detection, and reversible storage semantics are reviewed and covered.

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Redact transcriptPrompt before submitting it:** `src/agents/pi-embedded-runner/run.ts:1006`
  - body: `finalPrompt` is redacted, but the attempt still receives `params.transcriptPrompt` unchanged. The attempt path can submit that transcript prompt as the visible model prompt, so secrets in that field bypass the guardrail and reach the provider.
  - confidence: 0.94
- **[P1] Scan multiline secrets before splitting lines:** `src/security/patterns.ts:97-109`
  - body: The PEM/OpenSSH/PGP patterns require full multiline blocks, but `scanSensitive` splits the input and runs each pattern on one line at a time. In regex-only mode, normal private-key blocks remain unredacted before cloud transit.
  - confidence: 0.95
- **[P2] Avoid durable raw JSON vault storage:** `src/agents/pi-embedded-runner/run.ts:925-928`
  - body: This writes the full token-to-secret map as plaintext JSON beside the session file. Owner-only permissions reduce local exposure, but the feature still leaves durable raw credentials outside a reviewed secret-store and cleanup lifecycle.
  - confidence: 0.86
