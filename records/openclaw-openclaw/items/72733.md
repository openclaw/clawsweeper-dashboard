## Summary

Keep open: this is a useful but broad security-sensitive feature PR, and the current patch still has source-level secret-leak paths, missing real behavior proof, and an unresolved core-vs-plugin guardrail boundary.

## What This Changes

The PR adds an opt-in reversible sensitive-data guardrail with local-LLM/regex scanning, `[VAULT_N]` redaction/restoration, runner/tool integration, config schema/UI metadata, docs, and a vault test.

## Best Possible Solution

Land only after maintainers choose a reviewed pre-model guardrail boundary that covers every model-visible prompt surface, block-secret detection, reversible storage cleanup/secret-store semantics, docs, UI, and targeted tests.

## Reproduction Assessment

Yes at source level: current main turns `transcriptPrompt` into the submitted `promptForModel`, while the PR forwards that field unchanged; the regex-only private-key gap follows from applying multiline patterns to one line at a time. I did not run a live local-model/tool reproduction in this read-only review.

## Solution Assessment

No. The direction is useful, but this patch is not the best merge shape because it hard-codes sensitive policy into the embedded runner while leaving model-visible prompt and secret-storage lifecycle gaps.

Full review comments:

- **[P1] Redact transcriptPrompt before submitting it:** `src/agents/pi-embedded-runner/run.ts:1006`
  - body: `finalPrompt` is redacted, but the attempt still receives `params.transcriptPrompt` unchanged. Current main can use a distinct `transcriptPrompt` as the model-visible prompt, so secrets in that field can bypass the guardrail and reach the model.
  - confidence: 0.94
- **[P1] Scan block secrets before splitting lines:** `src/security/patterns.ts:95-109`
  - body: The PEM/OpenSSH/PGP patterns now require full multiline blocks, but `scanSensitive` splits the input and runs each pattern against one line at a time. In regex-only mode, normal private-key blocks will remain unredacted before cloud transit.
  - confidence: 0.95
- **[P2] Avoid storing reversible vaults as raw sidecars:** `src/agents/pi-embedded-runner/run.ts:923-928`
  - body: This writes the full token-to-secret map as plaintext JSON beside the session file. Owner-only permissions reduce local exposure, but the feature still leaves durable raw credentials outside a reviewed secret-store and cleanup lifecycle.
  - confidence: 0.86
