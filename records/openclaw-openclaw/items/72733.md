## Summary

Keep open for maintainer/secops review: the PR is a broad security-sensitive core runner/config feature and still has blocking redaction gaps, reversible secret-storage concerns, and no after-fix real behavior proof.

## What This Changes

This PR adds an opt-in reversible Security Guardrail that scans user prompts with a local LLM or regexes, replaces sensitive values with `[VAULT_N]` tokens, restores them for tools and replies, and adds config, UI metadata, docs, and tests.

## Best Possible Solution

Land only after maintainers choose a reviewed pre-model guardrail boundary that covers every model-visible prompt surface, block-secret detection, reversible storage cleanup or secret-store semantics, docs, UI, and targeted tests.

## Reproduction Assessment

Yes at source level for the review defects: the PR forwards `params.transcriptPrompt` unchanged while the attempt path treats `transcriptPrompt` as the visible model prompt, and the multiline private-key gap follows from applying block regexes one line at a time. I did not run a live local-model/tool reproduction in this read-only review.

## Solution Assessment

No. The direction is useful, but this patch is not the best merge shape until maintainers approve the security boundary and the implementation covers all model-visible prompt paths plus reviewed vault storage semantics.
