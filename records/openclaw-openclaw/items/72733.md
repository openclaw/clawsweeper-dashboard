## Summary

Keep open: current main does not implement this guardrail, and the PR remains security-sensitive with unresolved source-level bypass/storage findings plus missing real behavior proof. The remaining path needs maintainer security/product review rather than cleanup closure.

## What This Changes

The PR adds an opt-in Security Guardrail that redacts sensitive prompt values into `[VAULT_N]` tokens via local LLM or regex scanning, restores them for tool/client-tool/output paths, and adds config, UI metadata, docs, and a vault test.

## Best Possible Solution

Land only after maintainers choose a reviewed pre-model guardrail boundary that covers every model-visible prompt surface, block-secret detection, reversible storage cleanup or secret-store semantics, docs, UI, and targeted tests.

## Reproduction Assessment

Yes at source level: the PR passes `params.transcriptPrompt` unchanged while the attempt path can submit that field as the model-visible prompt, and the multiline private-key gap follows from applying block regexes one line at a time. I did not run a live local-model/tool reproduction in this read-only review.

## Solution Assessment

No. The direction is useful, but this patch is not the best merge shape until maintainers approve the security boundary and the implementation covers all model-visible prompt paths plus reviewed vault storage semantics.

Full review comments:

- **[P1] Redact transcriptPrompt before submitting it:** `src/agents/pi-embedded-runner/run.ts:1006`
  - body: `finalPrompt` is redacted, but the attempt still receives `params.transcriptPrompt` unchanged. The attempt path can submit a distinct transcript prompt to the model, so secrets in that field bypass the guardrail and reach the provider.
  - confidence: 0.94
- **[P1] Scan block secrets before line splitting:** `src/security/patterns.ts:95-109`
  - body: The PEM/OpenSSH/PGP patterns require full multiline blocks, but `scanSensitive` splits the input and runs each pattern on one line at a time. In regex-only mode, normal private-key blocks remain unredacted before cloud transit.
  - confidence: 0.95
- **[P2] Avoid storing reversible vaults as raw sidecars:** `src/agents/pi-embedded-runner/run.ts:925-928`
  - body: This writes the full token-to-secret map as plaintext JSON beside the session file. Owner-only permissions reduce local exposure, but the feature still leaves durable raw credentials outside a reviewed secret-store and cleanup lifecycle.
  - confidence: 0.86
