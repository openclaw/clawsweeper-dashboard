## Summary

Keep open: current main does not implement this guardrail, and the latest PR head is still not safe to merge because source inspection shows model-visible bypasses, regex fallback gaps, durable reversible-secret storage, and missing real behavior proof.

## What This Changes

The branch adds an opt-in reversible Security Guardrail that redacts user prompts with a local LLM or regex vault, restores tokens for tools and replies, and adds config, UI metadata, docs, and tests.

## Best Possible Solution

Land only after maintainers choose the guardrail boundary and the implementation covers every model-visible prompt surface, regex fallback correctness, reversible storage lifecycle, docs/navigation, and focused regression plus real-behavior proof.

## Reproduction Assessment

Yes, at source level for the review defects: the PR still forwards `transcriptPrompt` unchanged while that field can become the submitted visible prompt, and multiline private-key regexes are applied one line at a time. I did not run a live local-model/provider/tool reproduction in this read-only review.

## Solution Assessment

No. The direction is useful, but this patch is not the best merge shape until the security boundary, all prompt surfaces, block-secret detection, and reversible storage semantics are reviewed and covered.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.93

Full review comments:

- **[P1] Redact transcriptPrompt before submitting it:** `src/agents/pi-embedded-runner/run.ts:1006`
  - body: `finalPrompt` is redacted, but the attempt still receives `params.transcriptPrompt` unchanged. The attempt path can submit a distinct transcript prompt as the model-visible prompt, so secrets in that field bypass the guardrail and reach the provider.
  - confidence: 0.94
- **[P1] Scan multiline secrets before splitting lines:** `src/security/patterns.ts:97-109`
  - body: The PEM/OpenSSH/PGP patterns require full multiline blocks, but `scanSensitive` splits the input and runs each pattern on one line at a time. In regex-only mode, normal private-key blocks remain unredacted before cloud transit.
  - confidence: 0.95
- **[P2] Do not persist reversible vaults as raw JSON:** `src/agents/pi-embedded-runner/run.ts:925-928`
  - body: This writes the full token-to-secret map as plaintext JSON beside the session file. Owner-only permissions reduce local exposure, but the feature still leaves durable raw credentials outside a reviewed secret-store and cleanup lifecycle.
  - confidence: 0.86
