## Summary

Keep this issue open: current main and the latest release still support WhatsApp outbound reactions and approval-specific reaction handling, but not the requested generic reaction query, historical retrieval, or broad agent/plugin-visible reaction event surface. The linked implementation attempts are now closed unmerged, while https://github.com/openclaw/openclaw/issues/79996 remains a narrower related hook-design item rather than a full replacement.

## What This Changes

The issue requests WhatsApp support for querying, persisting, and surfacing message reactions so agents can use reaction feedback.

## Best Possible Solution

Keep this open until the WhatsApp plugin has a complete, privacy-gated reaction read path with documented query/history semantics, focused tests, and docs.

## Reproduction Assessment

Yes, at source level: current main's WhatsApp action discovery and dispatch do not support the generic `reactions` action, and ordinary WhatsApp reaction events are not exposed beyond approval-target handling. I did not run a live WhatsApp account in this read-only review.

## Solution Assessment

Unclear until maintainers choose the reaction read model. A narrow implementation should define the supported query, persistence, hook, and access-control contract before turning reactions into ordinary agent-visible events.
