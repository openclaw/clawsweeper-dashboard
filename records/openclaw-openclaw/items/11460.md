## Summary

Keep open: current main and the latest tagged release still support outbound WhatsApp `react`, but not WhatsApp `reactions` listing/querying, stored historical reaction retrieval, inbound reaction exposure, or a reaction hook. The related implementation attempts are useful context, but https://github.com/openclaw/openclaw/pull/46881 and https://github.com/openclaw/openclaw/pull/48570 remain open/unmerged and https://github.com/openclaw/openclaw/pull/13641 was closed unmerged.

## What This Changes

Request WhatsApp support for listing/querying message reactions, retrieving historical reaction state, and exposing inbound reaction events to agents or plugins.

## Best Possible Solution

Keep this broad tracker open until the official WhatsApp plugin supports generic reaction list/query behavior, feasible historical retrieval or persistence, and privacy-gated inbound reaction or hook exposure with focused tests, docs, and redacted live WhatsApp proof.

## Reproduction Assessment

Yes, by source inspection for the reported unsupported query path: the generic `reactions` action exists in core, but WhatsApp only advertises and handles outbound `react`. I did not exercise a live WhatsApp account.

## Solution Assessment

Unclear until maintainers choose the API, storage, hook, and privacy model. The related PRs are useful input, but the maintainable direction needs to reconcile generic reaction querying, historical storage or retrieval, inbound event delivery, and plugin hook boundaries.
