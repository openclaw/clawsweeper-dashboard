## Summary

Keep open: current main and the latest release still support outbound WhatsApp `react`, but not the generic WhatsApp `reactions` query, historical reaction retrieval, or inbound reaction exposure. The related open pull requests and narrower hook issue are useful implementation inputs, not shipped fixes for the full request.

## What This Changes

The issue requests WhatsApp support for querying message reactions, retrieving or storing historical reaction state, and surfacing inbound reaction events to agents or plugins.

## Best Possible Solution

Keep this as the broad WhatsApp reaction-read tracker until the official WhatsApp plugin supports generic `reactions` query/list behavior, feasible historical retrieval or persistence, and privacy-gated inbound reaction exposure with tests, docs, and live proof.

## Reproduction Assessment

Yes for the reported unsupported query path by source inspection: the generic `reactions` action exists, but WhatsApp only advertises and handles `react`, so dispatch falls through to the unsupported-channel error. I did not exercise a live WhatsApp account.

## Solution Assessment

Unclear until maintainers choose the API, storage, and privacy model. The related PRs are useful input, but the maintainable direction needs to reconcile generic reaction querying, historical storage or retrieval, inbound event delivery, and plugin hook boundaries.
