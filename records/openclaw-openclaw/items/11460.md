## Summary

Keep open: current main and the latest release still support outbound WhatsApp `react` plus a narrow approval-reaction path, but they do not support the generic WhatsApp `reactions` query action, historical reaction retrieval, or a completed privacy-gated inbound reaction exposure path. The related implementation PRs are now closed unmerged, and the open hook issue is narrower than this broader query/history tracker.

## What This Changes

The issue requests WhatsApp support for querying message reactions, exposing reaction events to agents or plugins, and retrieving historical reaction feedback.

## Best Possible Solution

Keep this broad tracker open until the official WhatsApp plugin supports generic reaction list/query behavior, feasible historical retrieval or persistence, and privacy-gated inbound reaction or hook exposure with focused tests, docs, and redacted live WhatsApp proof.

## Reproduction Assessment

Yes, source-reproducible: the generic `reactions` action exists, but WhatsApp discovery, dispatch, and runtime handling only cover outbound `react`/`upload-file` and never expose a reaction query path. I did not run a live WhatsApp account or CLI command in this read-only review.

## Solution Assessment

Unclear until maintainers choose the API, storage, and privacy model. The maintainable solution should reconcile generic reaction querying, historical persistence or provider retrieval, inbound event delivery, and plugin hook boundaries instead of landing another partial competing surface.
