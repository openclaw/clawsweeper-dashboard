## Summary

Keep open: current main still supports outbound WhatsApp `react`, but not the generic WhatsApp `reactions` query, stored historical reaction retrieval, or privacy-gated inbound reaction exposure. The related open PRs and narrower hook issue are useful implementation input, not shipped fixes.

## What This Changes

The issue requests WhatsApp support for listing/querying message reactions and exposing reaction feedback to agents or plugins.

## Best Possible Solution

Keep this broad tracker open until the official WhatsApp plugin supports generic reaction list/query behavior, feasible historical retrieval or persistence, and privacy-gated inbound reaction exposure with focused tests, docs, and redacted live WhatsApp proof.

Yes by source inspection for the reported unsupported query path: the generic `reactions` action exists, but WhatsApp only advertises and handles outbound `react`, so dispatch remains unsupported. I did not exercise a live WhatsApp account.

## Solution Assessment

Unclear until maintainers choose the API, storage, and privacy model. The related PRs are useful input, but the maintainable direction needs to reconcile generic reaction querying, historical storage or retrieval, inbound event delivery, and plugin hook boundaries.
