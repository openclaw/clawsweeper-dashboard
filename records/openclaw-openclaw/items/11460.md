## Summary

Keep open: current main and the latest shipped release still implement only outbound WhatsApp `react`; the WhatsApp `reactions` query action, historical reaction retrieval or persistence, and inbound reaction exposure are still missing. The related open PRs and the narrower hook issue are useful inputs, but they are not merged fixes and do not cover the full request.

## What This Changes

Add WhatsApp support for listing/querying message reactions, exposing reaction events, and retrieving historical reactions.

## Best Possible Solution

Keep this as the broad WhatsApp reaction-read tracker until the official WhatsApp plugin supports `reactions` query/list behavior, feasible historical retrieval or persistence, and privacy-gated inbound reaction exposure with tests, docs, and live proof.

## Reproduction Assessment

Yes for the reported unsupported query path by source inspection: the CLI can dispatch `reactions`, but WhatsApp advertises and handles only `react`, and the shared runner throws the unsupported-channel error when no plugin handles the action. I did not exercise a live WhatsApp account.

## Solution Assessment

Unclear until maintainers choose the API and privacy model. The maintainable direction is a WhatsApp-owned design that reconciles generic reaction querying, historical storage or retrieval, inbound event delivery, and the overlapping open implementation PRs.
