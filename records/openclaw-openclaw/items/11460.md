## Summary

Keep this issue open: current main and the latest release still support WhatsApp outbound reactions and approval-specific reaction handling, but not the requested generic reaction query, historical retrieval, or broad inbound reaction event surface.

## What This Changes

The issue requests WhatsApp support for querying/listing message reactions, surfacing reaction events to agents or plugins, and retrieving historical reactions.

## Best Possible Solution

Keep this open until the WhatsApp plugin has a complete, privacy-gated reaction read path with documented query/history semantics, focused tests, and docs.

## Reproduction Assessment

Yes, at source level: current main's WhatsApp action discovery and dispatch do not support the generic `reactions` action, and the inbound monitor lacks a general `messages.reaction` consumer. I did not run a live WhatsApp account in this read-only review.

## Solution Assessment

Unclear until maintainers choose the reaction read model. A narrow implementation should avoid turning all reactions into ordinary agent messages by default and instead define the supported query, storage, hook, and access-control contract first.
