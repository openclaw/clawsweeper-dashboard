## Summary

Keep open. Current main and the latest release still support WhatsApp outbound `react` and approval-specific reaction handling, but they do not advertise or dispatch the generic WhatsApp `reactions` query action, do not expose ordinary inbound reaction events through a public hook/event contract, and do not provide historical reaction retrieval.

## What This Changes

The issue requests WhatsApp support for reading/querying reactions, surfacing reaction events, and retrieving historical reaction state.

## Best Possible Solution

Define a WhatsApp-plugin-owned reaction read contract that covers the generic `reactions` query, a privacy-gated inbound add/remove event surface, and any historical persistence/query limits, then implement it with focused WhatsApp tests, docs, and live proof.

## Reproduction Assessment

Yes, at source level. Current main still omits the WhatsApp `reactions` query action from discovery and dispatch, and ordinary inbound reaction events are not surfaced outside the approval-specific path.

## Solution Assessment

No current implementation is sufficient. A maintainable fix should live in the WhatsApp plugin boundary and settle query, event, persistence, and privacy-gating semantics before implementation.
