## Summary

Keep open: current main and v2026.6.8 still support WhatsApp outbound reactions plus approval-specific reaction handling, but not the requested generic `message reactions` query, historical retrieval, or broad agent/plugin-visible reaction event surface. The related implementation PRs are closed unmerged, and the remaining open hook issue is narrower than this issue.

## What This Changes

The issue requests WhatsApp support for querying, storing, and surfacing message reactions so agents can use reaction feedback.

## Best Possible Solution

Keep this open until the WhatsApp plugin has a complete, privacy-gated reaction read path with documented query/history semantics, focused tests, and docs.

## Reproduction Assessment

Yes, at source level: current WhatsApp action discovery and dispatch do not support the generic `reactions` action, and ordinary WhatsApp reaction events are not exposed beyond approval-target handling. No live WhatsApp account was exercised in this read-only review.

## Solution Assessment

Unclear until maintainers choose the reaction read model. A narrow implementation should define query, persistence, hook, and access-control semantics before turning reactions into ordinary agent-visible events.
