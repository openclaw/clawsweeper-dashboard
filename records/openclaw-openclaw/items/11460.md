## Summary

Keep open: current main and latest release still lack the requested generic WhatsApp reaction query/history/event surface. WhatsApp supports outbound `react` and approval-specific reaction handling, but not the `message reactions` list action, historical reaction retrieval, or a broad agent/plugin-visible reaction event path; the related hook issue is narrower and the implementation PRs are closed unmerged.

## What This Changes

Request support for querying and surfacing WhatsApp message reactions, including historical retrieval and agent/plugin-visible reaction events.

## Best Possible Solution

Define a privacy-gated WhatsApp reaction read model, then implement the generic `reactions` action, historical retrieval or persistence, and observation hooks in the WhatsApp plugin with focused tests and docs.

## Reproduction Assessment

Yes, at source level: current WhatsApp action discovery and dispatch do not support the generic `reactions` action, and ordinary WhatsApp reaction events are not exposed beyond approval-target handling. No live WhatsApp account was exercised in this read-only review.

## Solution Assessment

Unclear until maintainers choose the reaction read model. A maintainable fix should define query, persistence, hook, privacy, and access-control semantics before turning reactions into ordinary agent-visible events.
