## Summary

Keep open. Current main and v2026.6.6 still do not support the generic WhatsApp `message reactions` query or historical reaction retrieval; the current WhatsApp code only supports outbound `react` plus a narrow approval-reaction path. The related implementation PRs are closed unmerged, while the open hook/turn issues are adjacent but do not replace this query/history request.

## What This Changes

Add WhatsApp support for querying/listing message reactions, making reaction events searchable or stored, and retrieving historical reactions for agent feedback.

## Best Possible Solution

Define a WhatsApp-owned reaction read contract that advertises and handles `message reactions`, persists or retrieves reaction history where feasible, coordinates with the reaction hook and reaction-triggered-turn work, and ships focused docs, tests, and live WhatsApp proof.

## Reproduction Assessment

Yes, at source level. Current main exposes `reactions` as a generic action name, but WhatsApp docs, action discovery, action support, and runtime handling omit it; I did not run a live WhatsApp account in this read-only review.

## Solution Assessment

Unclear until maintainers choose the reaction read model. A maintainable fix should stay inside the WhatsApp plugin boundary, preserve privacy/access-control gates, and avoid reviving the closed unmerged PR branches as-is.
