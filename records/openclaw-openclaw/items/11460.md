## Summary

Keep open: current main and v2026.6.6 still do not advertise or handle WhatsApp `message reactions`, while newer approval-reaction code only resolves persisted approval targets. The open hook sibling is narrower and the prior PR attempts are closed unmerged, so no implemented or superseding close is supported.

## What This Changes

Request WhatsApp support for querying, storing, or surfacing message reactions so agents can use reaction feedback.

## Best Possible Solution

Keep this open until the WhatsApp plugin has a maintainer-approved reaction read path: generic `reactions` action support where feasible, reaction event persistence or hook emission as decided, focused tests/docs, and coordination with https://github.com/openclaw/openclaw/issues/79996 and https://github.com/openclaw/openclaw/issues/78963.

## Reproduction Assessment

Yes, at source level: current WhatsApp discovery and dispatch omit the generic `reactions` action, docs exclude WhatsApp from reaction queries, and ordinary inbound reactions are not surfaced outside approval-specific handling. I did not run a live WhatsApp account in this read-only review.

## Solution Assessment

No current implementation is sufficient. The best path is a WhatsApp-plugin-owned reaction read/history/event contract aligned with the narrower hook discussions, not closing this in favor of approval-only reaction handling or closed unmerged PRs.
