## Summary

Keep open. Current main and the latest release still support outbound WhatsApp `react`, but not the generic `message reactions` query/list action or general inbound reaction surfacing; the narrower open hook issue covers only part of this request.

## What This Changes

The issue requests WhatsApp support for querying historical message reactions and surfacing reaction events to agents or plugins.

## Best Possible Solution

Define the WhatsApp reaction read contract at the WhatsApp plugin/SDK boundary, then implement query and/or stored event surfacing with privacy gating, focused tests, and docs.

## Reproduction Assessment

Yes, at source level. Current main registers the generic `reactions` action, but WhatsApp does not advertise or dispatch it, and the latest release has the same unsupported shape; no live WhatsApp account was used in this read-only review.

## Solution Assessment

Unclear. The issue lists plausible implementation shapes, but maintainers still need to choose the durable product/API contract for historical query, persistence, and reaction-event surfacing before a fix PR is safe.
