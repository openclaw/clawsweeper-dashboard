## Summary

Keep open: current main still supports outbound WhatsApp `react`/status reactions, but not generic reaction listing, historical reaction storage/query, or inbound reaction event/hook delivery; the related implementation PRs remain open and unmerged with unresolved review/product scope.

## What This Changes

Add WhatsApp support for querying message reactions, surfacing reaction events, and retrieving historical reaction feedback for agent workflows.

## Best Possible Solution

Keep this as the broad WhatsApp reaction-read tracker until the WhatsApp plugin advertises and handles `reactions`, persists or retrieves historical reaction data where feasible, and emits privacy-gated inbound reaction events with focused tests, docs, and live proof.

## Reproduction Assessment

Yes for the current unsupported path. Source inspection on current main shows WhatsApp advertises and dispatches only `react`/`poll`, so `openclaw message reactions --channel whatsapp` reaches the shared unsupported-channel path and no inbound reaction listener or storage/query contract exists.

## Solution Assessment

Unclear until maintainers choose scope. The maintainable path is a single WhatsApp-owned design that reconciles generic query support, historical storage or retrieval, inbound hook/session events, privacy opt-in, and the overlapping open PRs.
