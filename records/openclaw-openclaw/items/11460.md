## Summary

Keep open: current main still supports outbound WhatsApp `react`, but not the generic `reactions` query action, historical reaction storage/query, or inbound reaction event delivery. The related implementation PRs at https://github.com/openclaw/openclaw/pull/46881 and https://github.com/openclaw/openclaw/pull/48570 remain open/unmerged with review and proof blockers, while https://github.com/openclaw/openclaw/issues/79996 is a narrower hook request rather than a replacement for this broader tracker.

## What This Changes

Request WhatsApp support for listing/querying message reactions, exposing reaction events, and retrieving historical reaction data.

## Best Possible Solution

Keep this as the broad WhatsApp reaction-read tracker until the WhatsApp plugin advertises and handles `reactions`, persists or retrieves historical reaction data where feasible, and emits privacy-gated inbound reaction events with focused tests, docs, and live proof.

## Reproduction Assessment

Yes for the unsupported query path: current main source shows WhatsApp advertises only `react`/`poll`, supports only `react`, and falls through to the shared unsupported-channel error for `reactions`. I did not exercise a live WhatsApp account.

## Solution Assessment

Unclear until maintainers choose scope. The maintainable path is a single WhatsApp-owned design that reconciles generic query support, historical storage or retrieval, inbound hook/session events, privacy opt-in, and the overlapping open PRs.
