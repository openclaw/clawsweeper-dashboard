## Summary

Keep open: current main still only supports outbound WhatsApp reaction sending and status/ack reactions, not the generic `reactions` query action, historical reaction retrieval, or inbound reaction events/hooks. The related open PRs and narrower hook issue are useful implementation input, but they are not merged and do not supersede this broader tracker.

## What This Changes

The issue asks for WhatsApp support to query message reactions and expose reaction signals to agents or plugins, including historical retrieval where feasible.

## Best Possible Solution

Keep this broad tracker open until the official WhatsApp plugin supports generic reaction list/query behavior, feasible historical retrieval or persistence, and privacy-gated inbound reaction or hook exposure with focused tests, docs, and redacted live WhatsApp proof.

## Reproduction Assessment

Yes, source-reproducible: the generic `reactions` action exists for other channels, but WhatsApp only advertises and handles outbound `react`, and the inbound listener does not attach reaction handling. I did not run a live WhatsApp account or the CLI path.

## Solution Assessment

Unclear until maintainers choose the public API, persistence, and privacy model. The maintainable path should reconcile generic reaction querying, historical storage/provider retrieval, inbound event delivery, and plugin hook boundaries instead of landing a partial competing surface.
