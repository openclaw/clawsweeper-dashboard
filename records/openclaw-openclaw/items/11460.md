## Summary

Keep open. Current main and the latest release still lack the requested WhatsApp reaction read path: WhatsApp advertises and handles outbound `react`, but not the generic `reactions` query action, and ordinary inbound reactions are only consumed by approval-specific handling rather than exposed as queryable history or plugin/agent events.

## What This Changes

Add WhatsApp support for querying message reactions, retrieving or storing historical reactions, and surfacing inbound reaction feedback for agent/plugin use.

## Best Possible Solution

Keep this issue open until the WhatsApp plugin has a maintainer-approved reaction read contract with query/history or event exposure, privacy gating, focused tests/docs, and real WhatsApp proof.

## Reproduction Assessment

Yes, at source level: the generic `reactions` action exists, but WhatsApp declines it and the runner throws the reported unsupported-channel error. Current source also shows no general WhatsApp reaction event or history surface outside approval-specific handling.

## Solution Assessment

No current implementation is sufficient. The best path is a WhatsApp-plugin-owned reaction read/event contract aligned with privacy and hook decisions, not closing this in favor of approval-only reactions or closed unmerged PRs.
