The issue requests WhatsApp support for listing/querying message reactions, surfacing reaction events, and retrieving historical reactions for agent feedback workflows.

## Best Possible Solution

Keep this broad tracker open until the official WhatsApp plugin supports generic reaction list/query behavior, feasible historical retrieval or persistence, and privacy-gated inbound reaction or hook exposure with focused tests, docs, and redacted live WhatsApp proof.

Unclear until maintainers choose the API, storage, and privacy model. The maintainable solution should reconcile generic reaction querying, historical persistence or provider retrieval, inbound event delivery, and plugin hook boundaries instead of landing a partial competing surface.
