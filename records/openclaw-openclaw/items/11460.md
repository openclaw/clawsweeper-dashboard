## Summary

Keep open: current main and the latest release still lack WhatsApp support for the generic reaction-list/query path, historical reaction retrieval, and broad reaction event exposure. The remaining work is real but needs maintainer product/API choices because current WhatsApp behavior is outbound `react` plus approval-specific reaction handling only.

## What This Changes

Add WhatsApp support for listing/querying message reactions, retrieving or storing historical reactions, and exposing reaction events to agents or plugins.

## Best Possible Solution

Define a privacy-gated WhatsApp reaction read model, then implement the generic `reactions` action, reaction persistence or retrieval, and observation hooks inside the WhatsApp plugin with focused tests, docs, and live proof.

## Reproduction Assessment

Yes, at source level: current WhatsApp action discovery and dispatch do not support the generic `reactions` action, and ordinary WhatsApp reaction events are not exposed beyond approval-target handling. No live WhatsApp account was exercised in this read-only review.

## Solution Assessment

Unclear until maintainers choose the reaction read model. A maintainable fix should define query, persistence, hook, privacy, access-control, and agent-turn semantics before surfacing reactions broadly.
