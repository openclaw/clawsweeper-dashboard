## Label Justifications

## Summary

Keep open: current main and the latest release still support outbound WhatsApp `react` only, while the requested `message reactions` query, ordinary inbound reaction surfacing, and historical reaction retrieval remain missing. The related reaction PRs are closed unmerged, and the narrower open hook issue only covers part of this broader request.

## What This Changes

The issue requests WhatsApp support for listing/querying message reactions, surfacing reaction events to agents or plugins, and retrieving historical reaction data.

## Best Possible Solution

Keep this issue open until WhatsApp has a maintainer-approved, privacy-gated reaction read path that advertises and handles `message reactions`, persists or retrieves historical reactions where feasible, and exposes inbound add/remove reaction events through the appropriate plugin/agent contract with tests and docs.

## Reproduction Assessment

Yes, at source level. Current main rejects WhatsApp `reactions`, advertises only outbound `react`, lacks a generic reaction listener/hook, and only consumes `reactionMessage` for approval-specific handling; no live WhatsApp account was used in this read-only review.

## Solution Assessment

No current implementation is sufficient. The best path is a WhatsApp-plugin-owned reaction query/history/event contract aligned with the narrower hook and hooks-only discussions, not closing this in favor of approval-only reactions or closed unmerged PRs.
