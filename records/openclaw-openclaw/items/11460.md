## Summary

Keep open. Current main and v2026.6.6 still lack WhatsApp reaction querying, generic reaction listing, historical reaction storage/query, and non-approval inbound reaction surfacing; the related implementation PRs were closed unmerged, while the open hook issue covers only a narrower slice.

## What This Changes

The issue requests WhatsApp support for querying, storing, or surfacing message reactions so agents can use reaction feedback.

## Best Possible Solution

Define one canonical WhatsApp reaction read model, then implement it in the WhatsApp plugin with generic reaction listing where feasible, SQLite-backed or provider-backed history if supported, privacy-gated add/remove event surfacing, focused tests, and public docs.

## Reproduction Assessment

Yes. Source inspection on current main shows WhatsApp does not advertise or handle the generic `reactions` action, and ordinary inbound reactions are not surfaced outside approval-specific handling; no live WhatsApp account was used in this read-only review.

## Solution Assessment

No current implementation is sufficient. The best path is a WhatsApp-plugin-owned reaction read/event contract aligned with the broader plugin-hook discussions, rather than closing this in favor of the narrower hook-only issue.
