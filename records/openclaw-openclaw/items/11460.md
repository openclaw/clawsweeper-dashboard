## Summary

Keep open: current main still supports outbound WhatsApp `react`, but not the generic WhatsApp `reactions` query action, historical reaction retrieval, inbound reaction exposure, or a documented reaction hook. The related open PRs and the narrower hook issue are useful implementation input, but none has merged or supersedes this broader tracker.

## What This Changes

The issue requests WhatsApp support for querying, storing, and exposing message reactions to agents or plugins.

## Best Possible Solution

Keep this broad tracker open until the official WhatsApp plugin supports generic reaction list/query behavior, feasible historical retrieval or persistence, and privacy-gated inbound reaction or hook exposure with focused tests, docs, and redacted live WhatsApp proof.

## Reproduction Assessment

Yes, source-reproducible: the generic `reactions` action exists for other channels, but WhatsApp only advertises and handles outbound `react`, so the unsupported-action path is clear from current main. I did not run a live WhatsApp account or the CLI path.

## Solution Assessment

Unclear until maintainers choose the public API and privacy model. The maintainable solution should reconcile generic reaction querying, historical persistence or provider retrieval, inbound event delivery, and plugin hook boundaries instead of landing a partial competing surface.
