## Summary

Keep open: current main and the latest shipped release still only cover outbound WhatsApp reactions, while WhatsApp reaction listing, historical storage/query, and inbound reaction/event exposure remain unimplemented. The linked implementation PRs are useful input but remain open and do not supersede this broader tracker.

## What This Changes

Request support for querying WhatsApp message reactions, making reaction events available to agents/plugins, and retrieving historical reactions.

## Best Possible Solution

Keep this as the broad WhatsApp reaction-read tracker until the WhatsApp plugin supports `reactions` query/list behavior, persists or retrieves historical reaction data where feasible, and emits privacy-gated inbound reaction events with focused tests, docs, and redacted live proof.

## Reproduction Assessment

Yes for the reported unsupported query path by source inspection: current main advertises only `react`/`poll`, supports only `react`, and falls through to the shared unsupported-channel error for `reactions`. I did not exercise a live WhatsApp account.

## Solution Assessment

Unclear until maintainers choose the public surface. The maintainable path should reconcile generic reaction querying, historical storage or retrieval, inbound event delivery, privacy opt-in, and the overlapping open PRs in one WhatsApp-owned design.
