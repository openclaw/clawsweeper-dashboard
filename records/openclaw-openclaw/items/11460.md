## Summary

Keep open: current `main` and `v2026.5.22` still support outbound WhatsApp `react` plus a narrow approval-reaction path, but not the generic WhatsApp `reactions` query action, historical reaction retrieval, or a completed inbound reaction exposure path. Related implementation PRs are still open/unmerged and the narrower hook issue does not supersede this broader tracker.

## What This Changes

Add WhatsApp support for querying message reactions, retrieving historical reaction signals, and exposing inbound reaction events to agents or plugins.

## Best Possible Solution

Keep this broad tracker open until the official WhatsApp plugin supports generic reaction list/query behavior, feasible historical retrieval or persistence, and privacy-gated inbound reaction or hook exposure with focused tests, docs, and redacted live WhatsApp proof.

## Reproduction Assessment

Yes, source-reproducible: the generic `reactions` action exists, but WhatsApp only advertises and handles outbound `react`/`upload-file`, so the unsupported query path is clear from current main. I did not run a live WhatsApp account or CLI command in this read-only review.

## Solution Assessment

Unclear until maintainers choose the API, storage, and privacy model. The maintainable solution should reconcile generic reaction querying, historical persistence or provider retrieval, inbound event delivery, and plugin hook boundaries instead of landing a partial competing surface.
