## Summary

Keep this issue open: current main still supports only outbound WhatsApp reactions, while reaction queries, historical retrieval, and inbound reaction delivery remain unavailable. The three related implementation PRs closed without merging, and the narrower open hook issue covers only live plugin events, not the full query and history request.

## What This Changes

Add WhatsApp support for querying message reactions, surfacing inbound add/remove events, and retrieving historical reactions.

## Best Possible Solution

Define a canonical WhatsApp reaction read model, then implement live add/remove events and any supported historical query inside the WhatsApp plugin through generic channel/plugin contracts, using the appropriate shared or agent SQLite store if persistence is required.

## Reproduction Assessment

Yes at source level. Current main deterministically omits the generic WhatsApp reaction-query action and does not subscribe to inbound reaction events, although no live WhatsApp account was exercised during this read-only review.

## Solution Assessment

No single proposed approach is yet the best complete solution. A maintainer should first define whether live hooks, agent-visible events, historical persistence, and CLI queries share one canonical contract or intentionally remain separate surfaces.
