## Summary

Current main has approval-specific reaction consumption, but arbitrary WhatsApp reactions still cannot be listed, historically retrieved, or exposed through a supported general agent/plugin path; the related open hook request covers only part of this broader capability.

## What This Changes

Add WhatsApp support for listing message reactions, retrieving historical reaction data, and exposing inbound reaction feedback to agents or plugins.

## Best Possible Solution

Define a canonical WhatsApp-plugin reaction event model, expose live add/remove events through access-controlled passive hooks or agent routing as explicitly chosen, and add SQLite-backed historical queries only after retention and identity semantics are approved.

## Reproduction Assessment

Yes, from current source: WhatsApp does not advertise or implement the generic `reactions` action, and arbitrary inbound reactions are not surfaced beyond the approval-specific consumer. A live WhatsApp account was not exercised during this read-only review.

## Solution Assessment

No single proposed option is yet the best complete solution; a canonical plugin-owned live event contract should be decided first, with historical persistence treated as a separate explicit product and storage choice.
