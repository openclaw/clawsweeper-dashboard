## Summary

Keep open: current main and the latest shipped tag still support outbound WhatsApp `react`, but not WhatsApp `reactions` listing, historical reaction storage/query, or inbound reaction delivery. The related PRs are useful implementation input, but they remain open with proof/review blockers and do not supersede this broader tracker.

## What This Changes

Request WhatsApp support for querying, storing, and surfacing message reactions, including historical reactions and inbound reaction events.

## Best Possible Solution

Keep this as the broad WhatsApp reaction-read tracker until the WhatsApp plugin advertises and handles `reactions`, persists or retrieves historical data where feasible, and emits privacy-gated inbound reaction events with tests, docs, and live proof.

## Reproduction Assessment

Yes for the unsupported query path by source inspection: current main advertises only `react`/`poll`, supports only `react`, and falls through to the shared unsupported-channel error for `reactions`. I did not exercise a live WhatsApp account.

## Solution Assessment

Unclear until maintainers choose the public surface. The maintainable path should reconcile generic query support, historical storage or retrieval, inbound hook/session events, privacy opt-in, and the overlapping open PRs in one WhatsApp-owned design.
