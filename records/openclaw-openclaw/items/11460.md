## Summary

Keep open: current main still only supports outbound WhatsApp `react`; generic reaction listing, historical reaction retrieval/storage, and inbound reaction-event routing remain missing, while the related implementation PRs are open or closed unmerged.

## What This Changes

Request WhatsApp support for listing/querying message reactions, retrieving historical reaction state, and exposing inbound reaction events to agents.

## Best Possible Solution

Keep this as the broad WhatsApp reaction-read tracker until the WhatsApp plugin advertises and handles `reactions`, persists or retrieves historical reaction data where feasible, and emits privacy-gated inbound reaction events with focused tests, docs, and live proof.

## Reproduction Assessment

Yes. Source inspection on current main shows WhatsApp advertises and dispatches only `react`/`poll`, so `openclaw message reactions --channel whatsapp` reaches the generic unsupported-channel path and no inbound reaction listener or storage/query contract exists.

## Solution Assessment

Unclear until maintainers choose scope. The maintainable path is a single WhatsApp-owned design that reconciles generic query support, historical storage/retrieval, inbound hook/session events, privacy opt-in, and the overlapping open PRs.
