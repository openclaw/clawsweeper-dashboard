## What This Changes

Ensure Claude CLI `rate_limit_event` statuses `allowed` and `allowed_warning` are treated as successful warnings, not billing failures that disable the auth profile and trigger fallback.

## Best Possible Solution

Add a focused Claude CLI stream-json fixture for `allowed`, `allowed_warning`, and denied cap events; keep warning statuses out of `markAuthProfileFailure`, map actual denial to `rate_limit`, and reserve `billing` for explicit payment or credit failures.
