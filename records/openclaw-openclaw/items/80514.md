Keep open: current main still lacks structured Claude CLI `rate_limit_event.status` handling, and the only linked fix PR was closed unmerged. ## What This Changes

Treat Claude CLI subscription `allowed`/`allowed_warning` rate-limit events as successful responses and map hard cap rejection to `rate_limit` instead of `billing`.

## Best Possible Solution

Add focused Claude CLI JSONL fixtures for `allowed`, `allowed_warning`, and `rejected` events, ignore warning statuses, classify `rejected` as `rate_limit`, and keep `billing` reserved for explicit payment or credit failures.
