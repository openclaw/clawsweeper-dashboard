## What This Changes

The issue reports Claude CLI `rate_limit_event.status: allowed_warning` being treated as a billing failure, causing a long auth-profile cooldown and unwanted fallback to OpenRouter.

## Best Possible Solution

Add a status-aware Claude CLI JSONL handler with regression fixtures: ignore `allowed` and `allowed_warning`, map `rejected` to `rate_limit`, and reserve `billing` for explicit payment or credit failures.
