## What This Changes

The issue asks OpenClaw to treat Claude CLI `allowed`/`allowed_warning` rate-limit events as successful responses, classify hard-cap rejection as `rate_limit`, and avoid long billing cooldown fallback.

## Best Possible Solution

Add a narrow Claude CLI JSONL fixture and parser branch for `rate_limit_event`: ignore `allowed` and `allowed_warning`, map `rejected` to `rate_limit`, and keep `billing` reserved for explicit billing or credit failures.
