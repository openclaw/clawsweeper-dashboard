The issue asks OpenClaw to treat Claude CLI `rate_limit_event` `allowed`/`allowed_warning` statuses as successful soft notices and hard-cap statuses as `rate_limit`, not `billing`.

## Best Possible Solution

Add a focused Claude CLI JSONL fixture and parser branch that ignores `allowed`/`allowed_warning`, maps `rejected` to `rate_limit`, and reserves `billing` for explicit billing or credit failures.
