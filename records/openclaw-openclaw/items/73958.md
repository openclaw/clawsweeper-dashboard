## Summary

Keep open. Current main still forces Feishu group replies into thread mode when only `thread_id` is present, and this PR is a focused Feishu-plugin fix for that behavior; the remaining cleanup is the now-inert dispatcher `threadReply` parameter.

## What This Changes

The branch changes Feishu reply routing so `root_id` anchors group thread replies while `thread_id`-only contexts honor `replyInThread`, updates focused Feishu tests, and adds a changelog entry.

## Best Possible Solution

Land this PR, or an equivalent Feishu-plugin patch, after removing the inert dispatcher plumbing. Keep the broader disabled-as-hard-off behavior in #39507, typing-reaction targeting in #67783, and rootless-thread fallback in #75024 as separate review decisions unless maintainers intentionally combine them.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: a Feishu group event with `thread_id`, no `root_id`, and `replyInThread: "disabled"` flows to dispatcher params with `replyInThread: true`, and the current test asserts that behavior.

## Solution Assessment

Yes. The proposed fix stays inside the Feishu plugin, preserves shared contracts, and adds targeted coverage; the safer refinement is to remove the unused `threadReply` dispatcher parameter and matching test/call-site expectations.
