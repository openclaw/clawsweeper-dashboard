## Summary

Keep open. Current main still forces Feishu/Lark `thread_id`-only group messages into threaded delivery despite `replyInThread: "disabled"`, and this PR is a narrow plugin-level fix, but it still has one mechanical cleanup before the automerge path should proceed.

## What This Changes

The PR changes Feishu/Lark reply routing so `root_id` anchors group topic replies, `thread_id`-only group events honor `replyInThread`, DMs do not force threaded delivery from `rootId`, and focused tests plus a changelog entry are updated.

## Best Possible Solution

Land this Feishu-plugin fix after removing the inert dispatcher boundary and rebasing, while keeping #39507, #67783, and #75024 as separate review decisions unless maintainers intentionally combine their semantics.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists on current main: a Feishu group event with `thread_id`, no `root_id`, and `replyInThread: "disabled"` resolves to dispatcher params with `replyInThread: true`, and the current test asserts that behavior.

## Solution Assessment

Yes. The PR’s direction is the narrowest maintainable fix because it stays inside the Feishu plugin and targeted tests; the safer refinement is to remove the now-unused `threadReply` dispatcher parameter before merge.
