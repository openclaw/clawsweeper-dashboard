## Summary

Keep open for maintainer handling: the classification concern is plausible, but this branch is not mergeable as-is because it patches stale ACPX wrapper paths, broadens error classification, and lacks after-fix real behavior proof.

## What This Changes

The PR reclassifies selected silent acpx nonzero exit paths to `ACP_BACKEND_UNAVAILABLE` and adds runtime, manager, fixture, and error-text regression tests.

## Best Possible Solution

Move silent backend-exit classification into the `acpx` runtime contract, consume the released package in the ACPX plugin, and preserve stderr, control-command, and session-recovery semantics with focused tests plus a real ACPX smoke proof.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection of current main plus `acpx` 0.7.0 shows runtime disconnect/prompt failures can still normalize through generic runtime errors that OpenClaw treats as `ACP_TURN_FAILED`.

## Solution Assessment

No. This PR patches stale OpenClaw wrapper and manager fallback paths; the narrower maintainable solution is an acpx runtime-level classification contract consumed by OpenClaw.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Guard backend-unavailable behind truly silent prompt exits:** `extensions/acpx/src/runtime.ts:679-680`
  - body: This upgrades any nonzero prompt exit without a structured ACP error event to `ACP_BACKEND_UNAVAILABLE`, even when stderr contains a concrete adapter or validation failure. That routes normal turn failures to `/acp doctor`; only genuinely silent exits should get backend-unavailable guidance.
  - confidence: 0.89
- **[P2] Preserve control fallback for stderr failures:** `extensions/acpx/src/runtime.ts:129-131`
  - body: `resolveSilentAcpxControlExitErrorCode` remaps every non-permission nonzero control exit from `ACP_TURN_FAILED` to backend-unavailable. Control commands can fail with plain stderr for input or adapter reasons, so this should keep the fallback unless the result is actually silent.
  - confidence: 0.87
- **[P2] Keep ensure-session recovery after silent ensure exits:** `extensions/acpx/src/runtime.ts:483-484`
  - body: The new early rethrow prevents `recoverEnsureFailure` from running for silent `sessions ensure` exits. That drops the existing status probe and reuse/recreate recovery path for transient ensure failures that could still be recoverable.
  - confidence: 0.84
