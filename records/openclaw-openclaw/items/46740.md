## Summary

Keep open. The user problem may still be valid, but this branch is no longer the right merge shape: current main moved reusable ACPX runtime behavior behind the published `acpx` package boundary, and the PR still broadens several OpenClaw-visible error paths beyond truly silent prompt exits.

## What This Changes

The PR changes ACPX/ACP error classification so selected non-zero `acpx` process exits surface as `ACP_BACKEND_UNAVAILABLE`, with runtime, manager, fixture, and error-text test updates.

## Best Possible Solution

Move the exit classification to the current ACPX boundary: only genuinely silent prompt/backend exits should become `ACP_BACKEND_UNAVAILABLE`, while stderr-backed failures, permission-denied exits, control commands, session initialization, and early recovery paths keep their existing semantics. OpenClaw should then consume the published or explicitly pinned ACPX change with focused wrapper, manager, and error-text coverage.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the review findings: the PR diff deterministically remaps prompt/control exits and early session-init paths, while current-main tests still cover `acpx exited with code 1` as `ACP_TURN_FAILED` after a done event and as recoverable before output. I did not execute tests because this review was read-only.

## Solution Assessment

Is this the best way to solve the issue? No. The PR targets stale in-tree ACPX runtime logic and broadens classification beyond truly silent exits; the safer solution is an ACPX package-level fix plus a narrow OpenClaw consumption/update layer.

  - body: `runTurn` now tags any non-zero prompt exit without a structured error event as `ACP_BACKEND_UNAVAILABLE`, but this path can also include concrete stderr output. - **[P2] Preserve turn-failure codes for non-silent control exits:** `extensions/acpx/src/runtime.ts:1038-1044`

Status, cancel, close, mode, or config failures can therefore be mislabeled as backend outages.
  - confidence: 0.84

  - body: The early rethrow skips `recoverEnsureFailure` for silent `sessions ensure` exits, so OpenClaw no longer runs the status probe and fallback reuse/recreate flow that can recover transient session-init failures.
