## Summary

Keep this PR open, but it is not merge-ready: the bug-fix intent is still useful and not fully implemented on current main, while the branch is conflicting, lacks real ACPX proof, and still has correctness blockers around stderr, control-command, and ensure-recovery behavior.

## What This Changes

The PR changes ACPX silent-exit handling so selected `acpx exited with code 1` paths are classified as `ACP_BACKEND_UNAVAILABLE`, with runtime, manager, and error-text regression coverage.

## Best Possible Solution

Move silent backend-exit classification into the published `acpx` runtime contract, consume that release in the ACPX plugin, preserve stderr/control/session-recovery semantics, and validate it with focused tests plus redacted real ACPX smoke proof.

## Reproduction Assessment

No, not as a high-confidence live reproduction. The source path is clear: `acpx@0.7.0` can emit runtime-level failure codes, OpenClaw maps unknown codes to `ACP_TURN_FAILED`, and current tests still expect a plain `acpx exited with code 1` turn error to stay `ACP_TURN_FAILED`.

## Solution Assessment

No. The submitted branch patches stale wrapper and manager fallback paths; the maintainable solution is a current-main `acpx/runtime` contract-level classification that preserves stderr, control-command, and session-recovery behavior.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Guard backend-unavailable behind truly silent prompt exits:** `extensions/acpx/src/runtime.ts:679`
  - body: This tags any nonzero prompt exit without a structured error event as `ACP_BACKEND_UNAVAILABLE`, even when stderr contains a concrete adapter or validation failure. That routes ordinary turn failures to `/acp doctor`; only exits with no structured error and no stderr should get backend-unavailable guidance.
  - confidence: 0.89
- **[P2] Preserve turn-failure codes for non-silent control exits:** `extensions/acpx/src/runtime.ts:129-131`
  - body: `resolveSilentAcpxControlExitErrorCode` remaps every non-permission nonzero control exit from `ACP_TURN_FAILED` to backend-unavailable without checking whether stderr carried a normal command failure. Control commands can fail for input or adapter reasons, so keep the fallback unless the control result is truly silent.
  - confidence: 0.87
- **[P2] Keep ensure-session recovery after silent ensure exits:** `extensions/acpx/src/runtime.ts:480-482`
  - body: The early rethrow prevents `recoverEnsureFailure` from running for silent `sessions ensure` exits. That drops the existing status probe and reuse/recreate recovery path for transient ensure failures that could still be recoverable.
  - confidence: 0.84
