## Summary

Keep open: the PR targets a plausible ACPX error-classification bug, but the branch now edits the old embedded ACPX runtime shape and still has blocking behavior findings, while current main treats the bundled ACPX plugin as a thin consumer of the upstream acpx runtime package.

## What This Changes

The branch remaps selected `acpx exited with code 1` failures to `ACP_BACKEND_UNAVAILABLE` in the old ACPX runtime wrapper, ACP manager fallback, fixtures, and error-text tests.

## Best Possible Solution

Move the exit classification into upstream `openclaw/acpx` so only genuinely silent backend crashes surface as backend-unavailable, then bump the bundled ACPX package and keep focused OpenClaw manager/error-text coverage.

## Reproduction Assessment

Yes for source-level review: current main delegates the runtime stream to `acpx@0.7.0`, and the current manager/test path still shows generic `acpx exited with code 1` failures flowing to `ACP_TURN_FAILED`. I did not run a live ACPX backend crash locally.

## Solution Assessment

No. The PR puts broad classification logic into the old OpenClaw wrapper and manager fallback; the maintainable path is an upstream acpx runtime classification fix consumed by OpenClaw through a package bump.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Gate backend-unavailable on truly silent prompt exits:** `extensions/acpx/src/runtime.ts:676`
  - body: `runTurn` classifies any nonzero prompt exit without a structured error event as `ACP_BACKEND_UNAVAILABLE`, but this path can also cover stderr-only prompt, auth, or validation failures. Those should keep turn-failure recovery unless the backend crash was genuinely silent.
  - confidence: 0.86
- **[P2] Preserve turn-failure codes for stderr control exits:** `extensions/acpx/src/runtime.ts:1038-1044`
  - body: `resolveSilentAcpxControlExitErrorCode` only sees the exit code and fallback code, so non-permission-denied control exits with stderr can be mislabeled as backend outages. Control/input failures from status, cancel, close, mode, or config paths should not route users to backend doctor guidance unless they are truly silent crashes.
  - confidence: 0.84
- **[P2] Keep ensure-session recovery for silent ensure exits:** `extensions/acpx/src/runtime.ts:480-482`
  - body: The early rethrow skips `recoverEnsureFailure` for silent `sessions ensure` exits, so OpenClaw no longer runs the status probe and fallback reuse/recreate flow that can recover transient session-init failures.
  - confidence: 0.83
