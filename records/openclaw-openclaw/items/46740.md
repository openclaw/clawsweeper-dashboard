Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Gate backend-unavailable on empty stderr:** `extensions/acpx/src/runtime.ts:679`
  - body: `runTurn` now tags any non-zero prompt exit without a structured error event as `ACP_BACKEND_UNAVAILABLE`, but this path still includes concrete stderr failures in `formatAcpxExitMessage`. That routes normal prompt or agent validation failures to `/acp doctor` instead of turn-failure recovery; only exits with no structured error and empty stderr should be upgraded.
  - confidence: 0.88
- **[P2] Preserve control-error codes for stderr exits:** `extensions/acpx/src/runtime.ts:129-131`
  - body: `resolveSilentAcpxControlExitErrorCode` remaps every non-permission non-zero control exit from `ACP_TURN_FAILED` to `ACP_BACKEND_UNAVAILABLE` without checking whether the command produced stderr. Control failures from `setMode`, `setConfigOption`, `status`, `cancel`, or `close` can therefore be mislabeled as backend outages.
  - confidence: 0.85
- **[P2] Keep ensure recovery for silent init exits:** `extensions/acpx/src/runtime.ts:483-484`
  - body: The early `shouldAttemptEnsureRecovery` rethrow skips `recoverEnsureFailure` for silent `sessions ensure` exits, so the manager no longer gets the status-probe and recreate/reuse recovery path after a transient ensure failure. That turns a previously recoverable init path into an immediate `ACP_SESSION_INIT_FAILED`.
  - confidence: 0.82
