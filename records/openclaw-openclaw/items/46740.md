## Summary

Keep open. The PR is useful but not safe to merge: the patch over-classifies prompt/control failures as backend unavailable, skips existing ensure-session recovery, and current main has since moved the child-process runtime behavior behind the published `acpx` package boundary.

## What This Changes

The PR changes ACPX runtime and ACP manager error handling/tests so selected `acpx exited with code 1` failures surface as `ACP_BACKEND_UNAVAILABLE` with doctor guidance.

## Best Possible Solution

Land the classification in the `openclaw/acpx` runtime package so only genuinely silent backend crashes become `ACP_BACKEND_UNAVAILABLE`, then update OpenClaw’s pinned package and add focused wrapper, manager, Discord, and error-text coverage.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for source-level review: the PR diff deterministically remaps exit-code paths without proving stderr silence and bypasses ensure recovery. I did not execute tests because this was a read-only review.

## Solution Assessment

Is this the best way to solve the issue? No. The durable fix should be upstream in the ACPX runtime package, then consumed by OpenClaw with narrow compatibility tests rather than reintroducing classification logic into the thin wrapper.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Guard backend-unavailable code behind truly silent prompt exits:** `extensions/acpx/src/runtime.ts:676`
  - body: `runTurn` now tags any non-zero prompt exit without a structured error event as `ACP_BACKEND_UNAVAILABLE`, but this path can also include concrete stderr output. Stderr-backed prompt/auth/model/validation failures should keep turn-failure recovery instead of routing users to `/acp doctor`.
  - confidence: 0.86
- **[P2] Preserve turn-failure codes for non-silent control exits:** `extensions/acpx/src/runtime.ts:1038-1044`
  - body: `resolveSilentAcpxControlExitErrorCode` only receives the exit code and fallback code, so every non-permission-denied control-command exit with an `ACP_TURN_FAILED` fallback becomes `ACP_BACKEND_UNAVAILABLE` even when the command wrote stderr. Status, cancel, close, mode, or config failures can therefore be mislabeled as backend outages.
  - confidence: 0.84
- **[P2] Keep ensure-session recovery path for silent ensure exits:** `extensions/acpx/src/runtime.ts:480-482`
  - body: The early rethrow skips `recoverEnsureFailure` for silent `sessions ensure` exits, so OpenClaw no longer runs the status probe and fallback reuse/recreate flow that can recover transient session-init failures.
  - confidence: 0.83
