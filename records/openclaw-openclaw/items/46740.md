## Summary

Keep this PR open, but do not merge it as-is: current main moved the ACPX runtime boundary to the published `acpx` package, and this branch still has blocking behavior regressions around stderr/control exits and session recovery.

## What This Changes

The PR remaps selected `acpx exited with code 1` failures to `ACP_BACKEND_UNAVAILABLE` in the ACPX runtime wrapper, ACP manager fallback, fixtures, and error-text tests.

## Best Possible Solution

Move the classification into upstream `openclaw/acpx` so only genuinely silent backend crashes emit a code OpenClaw maps to `ACP_BACKEND_UNAVAILABLE`, then bump `extensions/acpx` and keep focused OpenClaw manager/error-text coverage.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for source-level PR review: current main plus `acpx@0.6.1` inspection shows generic runtime failures still flow back to `ACP_TURN_FAILED`, and the PR diff deterministically changes that classification.

## Solution Assessment

Is this the best way to solve the issue? No. The maintainable path is an upstream ACPX runtime classification fix with an OpenClaw version bump, not reintroducing broad exit-classification logic into the thin wrapper.

  - body: `runTurn` classifies any nonzero prompt exit without a structured error event as `ACP_BACKEND_UNAVAILABLE`, but this path can also cover stderr-only prompt, auth, or validation failures. Those should keep turn-failure recovery unless the backend crash was genuinely silent.
  - confidence: 0.86
- **[P2] Preserve turn-failure codes for stderr control exits:** `extensions/acpx/src/runtime.ts:1038-1044`
  - body: `resolveSilentAcpxControlExitErrorCode` only sees the exit code and fallback code, so non-permission-denied control exits with stderr can be mislabeled as backend outages. Control/input failures from status, cancel, close, mode, or config paths should not route users to backend doctor guidance unless they are truly silent crashes.
  - confidence: 0.84

  - body: The early rethrow skips `recoverEnsureFailure` for silent `sessions ensure` exits, so OpenClaw no longer runs the status probe and fallback reuse/recreate flow that can recover transient session-init failures.
  - confidence: 0.83
