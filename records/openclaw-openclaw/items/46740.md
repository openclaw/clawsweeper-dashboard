## Summary

Keep open, but the current branch should not merge as-is. The reported classification problem may still be valid, but this PR patches the old embedded ACPX runtime that current `main` replaced with the published `acpx/runtime` package, and the diff introduces source-proven error-code and recovery regressions.

## What This Changes

The PR changes ACPX runtime, manager, fixture, and error-text tests so selected `acpx exited with code 1` failures surface as `ACP_BACKEND_UNAVAILABLE` with doctor guidance while session-init failures stay scoped to init errors.

## Best Possible Solution

Implement the classification in `openclaw/acpx` so only genuinely silent prompt/backend disconnects become `ACP_BACKEND_UNAVAILABLE`, then bump OpenClaw's ACPX package and add focused wrapper, manager, Discord, and error-text coverage.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for source-level PR review: the diff deterministically remaps exit paths without proving stderr silence and skips the existing ensure recovery path. I did not run tests in this read-only review.

## Solution Assessment

Is this the best way to solve the issue? No. The maintainable fix should live in the upstream `acpx/runtime` package now consumed by OpenClaw, with OpenClaw limited to a package bump and regression coverage.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Guard backend-unavailable code behind truly silent prompt exits:** `extensions/acpx/src/runtime.ts:676`
  - body: `runTurn` now tags any non-zero prompt exit without a structured error event as `ACP_BACKEND_UNAVAILABLE`, but this branch still includes stderr in the formatted message. Stderr-backed prompt/auth/model/validation failures should keep turn-failure recovery instead of routing users to `/acp doctor`; only exits with no structured error and empty stderr should be upgraded.
  - confidence: 0.86
- **[P2] Preserve turn-failure codes for non-silent control exits:** `extensions/acpx/src/runtime.ts:1038-1044`
  - body: `resolveSilentAcpxControlExitErrorCode` only sees the exit code and fallback code, so every non-permission-denied control-command exit with an `ACP_TURN_FAILED` fallback becomes `ACP_BACKEND_UNAVAILABLE` even when the command wrote stderr. Status, cancel, close, mode, or config failures can therefore be mislabeled as backend outages.
  - confidence: 0.84
- **[P2] Keep ensure-session recovery for silent ensure exits:** `extensions/acpx/src/runtime.ts:480-482`
  - body: The early rethrow skips `recoverEnsureFailure` for silent `sessions ensure` exits, so OpenClaw no longer runs the status probe and fallback reuse/recreate flow that can recover transient session-init failures.
  - confidence: 0.83
