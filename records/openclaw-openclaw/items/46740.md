  - body: `runTurn` now tags any non-zero prompt exit without a structured ACP error as `ACP_BACKEND_UNAVAILABLE`, but that path can still include concrete stderr failure text. Those user/config/runtime failures would be routed to `/acp doctor`; only truly silent exits should be upgraded.
  - confidence: 0.84

  - confidence: 0.82

  - body: This early rethrow skips `recoverEnsureFailure` for silent `sessions ensure` exits, so the status probe and fallback reuse/recreate flow no longer run after a transient ensure failure. That weakens the session-init recovery behavior the manager currently depends on.
  - confidence: 0.8
