## Summary

Keep open: the bug intent is plausible, but this PR should not merge as-is because current main now treats the OpenClaw acpx plugin as a thin wrapper over `acpx/runtime`, the patch broadens exit classification beyond truly silent backend exits, and contributor real behavior proof is still missing.

## What This Changes

This PR changes the acpx runtime wrapper, ACP manager fallback, fixtures, and tests so selected `acpx exited with code 1` paths report `ACP_BACKEND_UNAVAILABLE`.

## Best Possible Solution

Move silent backend-exit classification into the acpx runtime contract, update OpenClaw to the appropriate pinned acpx release, and keep OpenClaw manager fallback logic narrow enough to preserve stderr, control-command, and session-recovery behavior.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection of current main, acpx 0.7.0, and the manager tests shows the relevant current-main `acpx exited with code 1` path still normalizes to `ACP_TURN_FAILED`, so it is source-reproducible but not live-reproduced.

## Solution Assessment

No. The current PR is not the best solution because it patches stale OpenClaw wrapper and manager fallback paths instead of the acpx runtime boundary, and it broadens behavior beyond truly silent backend exits.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Guard backend-unavailable behind truly silent prompt exits:** `extensions/acpx/src/runtime.ts:679-680`
  - body: This upgrades any non-zero prompt exit without a structured ACP error event to `ACP_BACKEND_UNAVAILABLE`, even when stderr contains a concrete adapter or validation failure. That routes normal turn failures to `/acp doctor`; only genuinely silent exits should get the backend-unavailable code.
  - confidence: 0.89
- **[P2] Preserve control fallback for stderr failures:** `extensions/acpx/src/runtime.ts:1041-1044`
  - body: `runControlCommand` now maps every non-permission non-zero control exit with fallback `ACP_TURN_FAILED` to backend-unavailable. Control commands can fail with plain stderr for input or adapter reasons, so this should keep `params.fallbackCode` unless the result is actually silent.
  - confidence: 0.87
- **[P2] Keep ensure-session recovery after silent ensure exits:** `extensions/acpx/src/runtime.ts:483-484`
  - body: The new early rethrow prevents `recoverEnsureFailure` from running for silent `sessions ensure` exits. That drops the existing status probe and reuse/recreate recovery path for transient ensure failures that could still be recoverable.
  - confidence: 0.84
