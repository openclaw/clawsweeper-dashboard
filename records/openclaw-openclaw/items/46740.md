Full review comments:

- **[P2] Gate backend-unavailable on empty stderr:** `extensions/acpx/src/runtime.ts:679`
  - body: `runTurn` now tags any non-zero prompt exit without a structured error event as `ACP_BACKEND_UNAVAILABLE`, but this path also covers exits that include a concrete stderr failure message. When `acpx` reports a normal turn error only on stderr, this routes users to `/acp doctor` instead of turn-failure recovery; only genuinely silent exits should be upgraded.
  - confidence: 0.87
- **[P2] Preserve control errors when stderr is present:** `extensions/acpx/src/runtime.ts:129-131`
  - body: `resolveSilentAcpxControlExitErrorCode` remaps every non-permission-denied nonzero control exit from `ACP_TURN_FAILED` to `ACP_BACKEND_UNAVAILABLE` without checking whether the failure was actually silent. Control paths such as mode/config/status/cancel/close can return stderr-backed command errors, and those should not be mislabeled as backend outages.
  - confidence: 0.84
- **[P2] Keep ensure recovery for silent init exits:** `extensions/acpx/src/runtime.ts:483-484`
  - body: This early rethrow skips `recoverEnsureFailure` for silent `sessions ensure` exits, so the status probe and fallback reuse/recreate flow no longer run after a transient ensure failure. That is a regression in session initialization resilience for failures that were previously recoverable.
  - confidence: 0.81
