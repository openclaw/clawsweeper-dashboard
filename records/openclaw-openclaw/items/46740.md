## Summary

Keep open: current main has not implemented the requested ACPX silent-exit classification, but this PR should not merge as-is because it patches the old embedded runtime path, broadens error remapping beyond truly silent exits, and regresses ensure-session recovery.

## What This Changes

The PR changes ACPX/ACP error classification so selected `acpx exited with code 1` failures surface as `ACP_BACKEND_UNAVAILABLE` with `/acp doctor` guidance.

## Best Possible Solution

Move the classification into `openclaw/acpx` so only genuinely silent backend crashes emit a structured backend-unavailable code, publish and bump the package, then keep narrow OpenClaw coverage for manager, Discord, and user-facing error text behavior.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for source-level PR review: the supplied diff deterministically remaps the prompt, control, and ensure-session paths, while current main plus `acpx@0.6.1` source inspection shows the requested backend-unavailable classification is not implemented in the shipped dependency path.

## Solution Assessment

Is this the best way to solve the issue? No. The PR changes the old embedded runtime and over-classifies non-silent failures; the maintainable fix belongs in the upstream `acpx` runtime package with OpenClaw consuming the released package.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Gate backend-unavailable on truly silent prompt exits:** `extensions/acpx/src/runtime.ts:676`
  - body: `runTurn` now classifies any nonzero prompt exit without a structured error event as `ACP_BACKEND_UNAVAILABLE`, but this path also covers stderr-only prompt, auth, or validation failures. Those should keep turn-failure recovery unless the backend crash is genuinely silent.
  - confidence: 0.86
- **[P2] Preserve turn-failure codes for stderr control exits:** `extensions/acpx/src/runtime.ts:1038-1044`
  - body: `resolveSilentAcpxControlExitErrorCode` only sees the exit code and fallback code, so non-permission-denied control exits with stderr can be mislabeled as backend outages. Control/input failures from status, cancel, close, mode, or config paths should not be routed to `/acp doctor` unless they are truly silent backend crashes.
  - confidence: 0.84
- **[P2] Keep ensure-session recovery for silent ensure exits:** `extensions/acpx/src/runtime.ts:480-482`
  - body: The early rethrow skips `recoverEnsureFailure` for silent `sessions ensure` exits, so OpenClaw no longer runs the status probe and fallback reuse/recreate flow that can recover transient session-init failures.
  - confidence: 0.83
