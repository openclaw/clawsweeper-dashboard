## Summary

Keep open. The requested ACP error-classification fix is plausible, but this branch is stale against current main's acpx dependency split, lacks real behavior proof, and the proposed diff over-classifies non-silent stderr/control failures as backend unavailable.

## What This Changes

This PR changes the acpx runtime wrapper, ACP manager fallback, fixtures, and tests so silent non-zero acpx turn exits report `ACP_BACKEND_UNAVAILABLE` and `/acp doctor` guidance while session-init exits keep `ACP_SESSION_INIT_FAILED`.

## Best Possible Solution

Implement the classification in the acpx-owned runtime contract, distinguish truly silent backend exits from stderr or structured command failures, then update OpenClaw's acpx dependency and keep manager fallback narrow and defensive.

## Reproduction Assessment

No live high-confidence reproduction was run. Source inspection gives a clear path: current `acpx` 0.7.0 emits runtime-style disconnect errors and OpenClaw maps unknown event codes back to `ACP_TURN_FAILED`, but the PR still needs real backend proof.

## Solution Assessment

No. The branch solves the right class of problem in an outdated wrapper layer and should instead make the acpx runtime distinguish silent exits from stderr/control failures before OpenClaw consumes the result.

Full review comments:

- **[P2] Guard backend-unavailable behind silent prompt exits:** `extensions/acpx/src/runtime.ts:679-680`
  - body: This branch upgrades any non-zero prompt exit without a structured error event to `ACP_BACKEND_UNAVAILABLE`, even when stderr contains a concrete adapter or validation failure. That routes normal turn failures to `/acp doctor`; only genuinely silent exits should get the backend-unavailable code.
  - confidence: 0.89
- **[P2] Preserve control fallback for stderr failures:** `extensions/acpx/src/runtime.ts:1041-1044`
  - body: `runControlCommand` now maps every non-permission non-zero control exit with fallback `ACP_TURN_FAILED` to backend-unavailable. Control commands can fail with plain stderr for input or adapter reasons, so this should keep `params.fallbackCode` unless the result is actually silent.
  - confidence: 0.87
- **[P2] Keep ensure-session recovery after silent ensure exits:** `extensions/acpx/src/runtime.ts:482-485`
  - body: The new early rethrow prevents `recoverEnsureFailure` from running for silent `sessions ensure` exits. That drops the existing status probe and reuse/recreate recovery path for transient ensure failures that could still be recoverable.
  - confidence: 0.84
