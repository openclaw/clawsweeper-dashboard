## Summary

Keep open. Current main has not implemented the requested backend-unavailable classification, but this PR is not safe to merge as-is because ACPX runtime ownership has moved to the published `acpx/runtime` package and the branch still overclassifies non-silent/control/session-init failures.

## What This Changes

The PR branch changes ACPX/ACP error handling so selected non-zero `acpx` exits surface as `ACP_BACKEND_UNAVAILABLE`, with runtime, manager, fixture, and error-text test updates.

## Best Possible Solution

Move the exit classification into `openclaw/acpx` so only genuinely silent prompt/backend exits emit a backend-unavailable code, then consume a published or explicitly pinned ACPX update in OpenClaw with focused wrapper, manager, and error-text coverage.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes from source: `acpx@0.6.1` emits package-level runtime failure codes that OpenClaw normalizes to `ACP_TURN_FAILED`, and current manager tests still assert `acpx exited with code 1` stays turn/session-init failure in key paths.

## Solution Assessment

Is this the best way to solve the issue? No. The branch edits stale in-tree ACPX runtime logic and still misclassifies non-silent/control failures; the maintainable path is an ACPX package fix followed by an OpenClaw dependency update and narrow compatibility tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Guard backend-unavailable code behind truly silent prompt exits:** `extensions/acpx/src/runtime.ts:676`
  - body: `runTurn` now tags any non-zero prompt exit without a structured error event as `ACP_BACKEND_UNAVAILABLE`, but this path can also include concrete stderr output. Stderr-backed prompt/auth/model/validation failures should keep turn-failure recovery instead of routing users to `/acp doctor`.
  - confidence: 0.86
- **[P2] Preserve turn-failure codes for non-silent control exits:** `extensions/acpx/src/runtime.ts:1038-1044`
  - body: `resolveSilentAcpxControlExitErrorCode` only receives the exit code and fallback code, so every non-permission-denied control-command exit with an `ACP_TURN_FAILED` fallback becomes `ACP_BACKEND_UNAVAILABLE` even when the command wrote stderr. Status, cancel, close, mode, or config failures can therefore be mislabeled as backend outages.
  - confidence: 0.84
- **[P2] Keep ensure-session recovery path for silent ensure exits:** `extensions/acpx/src/runtime.ts:480-482`
  - body: The early rethrow skips `recoverEnsureFailure` for silent `sessions ensure` exits, so OpenClaw no longer runs the status probe and fallback reuse/recreate flow that can recover transient session-init failures.
  - confidence: 0.83
