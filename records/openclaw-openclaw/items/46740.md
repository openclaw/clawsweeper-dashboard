## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.8

Full review comments:

- **[P2] Gate backend-unavailable on empty stderr:** `extensions/acpx/src/runtime.ts:679`
  - body: `runTurn` now marks every non-zero prompt exit without a structured error event as `ACP_BACKEND_UNAVAILABLE`, even when acpx produced stderr. That can route ordinary prompt or agent validation failures to `/acp doctor` instead of turn-failure remediation; only genuinely silent exits should be upgraded.
  - confidence: 0.82
- **[P2] Preserve control errors when stderr is present:** `extensions/acpx/src/runtime.ts:129-131`
  - body: `resolveSilentAcpxControlExitErrorCode` remaps non-permission-denied control-command exits from `ACP_TURN_FAILED` to `ACP_BACKEND_UNAVAILABLE` without knowing whether stderr described a normal command failure. Control paths such as mode/config/status/cancel/close can therefore be mislabeled as backend outages.
  - confidence: 0.78
- **[P2] Keep ensure recovery for silent init exits:** `extensions/acpx/src/runtime.ts:483-484`
  - body: The new early rethrow before `recoverEnsureFailure` means a transient silent `sessions ensure` failure no longer gets the existing status-probe and fallback reuse/recreate flow. That is a session-initialization availability regression unless the branch preserves recovery for recoverable init exits.
  - confidence: 0.76
