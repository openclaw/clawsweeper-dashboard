## Summary

Keep open: current main does not implement the requested acpx exit classification, but this PR is not mergeable as-is because it targets outdated wrapper behavior, has blocking correctness findings, and still lacks real acpx backend proof.

## What This Changes

This PR changes the acpx runtime wrapper, ACP manager fallback, fixtures, and tests so silent non-zero acpx turn exits report `ACP_BACKEND_UNAVAILABLE` while session-init exits keep `ACP_SESSION_INIT_FAILED`.

## Best Possible Solution

Move the classification into the acpx-owned runtime contract, distinguish genuinely silent backend exits from stderr/control/session-init failures, then update OpenClaw’s pinned acpx dependency and keep any manager fallback narrow.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection gives a clear path through acpx 0.7.0 terminal error events and OpenClaw’s fallback normalization, but the PR still needs real backend smoke proof.

## Solution Assessment

No. The branch addresses the right problem class in an outdated wrapper layer and over-broadly classifies non-silent/control failures; the safer fix is in the external acpx runtime package plus a dependency bump.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Guard backend-unavailable behind silent prompt exits:** `extensions/acpx/src/runtime.ts:676-680`
  - body: This upgrades any non-zero prompt exit without a structured error event to `ACP_BACKEND_UNAVAILABLE`, even when stderr contains a concrete adapter or validation failure. That routes normal turn failures to `/acp doctor`; only genuinely silent exits should get the backend-unavailable code.
  - confidence: 0.89
- **[P2] Preserve control fallback for stderr failures:** `extensions/acpx/src/runtime.ts:1038-1044`
  - body: `runControlCommand` now maps every non-permission non-zero control exit with fallback `ACP_TURN_FAILED` to backend-unavailable. Control commands can fail with plain stderr for input or adapter reasons, so this should keep `params.fallbackCode` unless the result is actually silent.
  - confidence: 0.87
- **[P2] Keep ensure-session recovery after silent ensure exits:** `extensions/acpx/src/runtime.ts:480-482`
  - body: The new early rethrow prevents `recoverEnsureFailure` from running for silent `sessions ensure` exits. That drops the existing status probe and reuse/recreate recovery path for transient ensure failures that could still be recoverable.
  - confidence: 0.84
