## Summary

Keep open. The reported behavior still looks plausible, but this branch is not safe to merge because it patches an outdated wrapper layer, over-classifies non-silent failures, skips existing ensure recovery, and lacks real backend proof.

## What This Changes

The PR changes the acpx runtime wrapper, ACP manager fallback, fixtures, and tests so silent non-zero acpx turn exits report `ACP_BACKEND_UNAVAILABLE` and `/acp doctor` guidance while session-init exits keep `ACP_SESSION_INIT_FAILED`.

## Best Possible Solution

Move the classification into the acpx-owned runtime contract, classify only truly silent backend exits as backend unavailable, preserve stderr/control/session-init recovery behavior, then update OpenClaw's acpx dependency with focused regression coverage.

## Reproduction Assessment

No live high-confidence reproduction was run. Source inspection shows the path: acpx 0.7.0 emits runtime-style errors, and current OpenClaw maps unrecognized event codes back to `ACP_TURN_FAILED`, but the PR still needs real backend proof.

## Solution Assessment

No. The branch targets the right class of failure but implements it in the wrapper layer and broadens classification beyond truly silent backend exits; the safer fix belongs in the acpx runtime contract with narrower OpenClaw consumption.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Guard backend-unavailable behind truly silent prompt exits:** `extensions/acpx/src/runtime.ts:679-680`
  - body: This branch upgrades any non-zero prompt exit without a structured error event to `ACP_BACKEND_UNAVAILABLE`, even when stderr contains a concrete adapter or validation failure. That routes normal turn failures to `/acp doctor`; only genuinely silent exits should get the backend-unavailable code.
  - confidence: 0.89
- **[P2] Preserve control fallback for stderr failures:** `extensions/acpx/src/runtime.ts:1041-1044`
  - body: `runControlCommand` maps every non-permission non-zero control exit with fallback `ACP_TURN_FAILED` to backend-unavailable. Control commands can fail with plain stderr for input or adapter reasons, so this should keep `params.fallbackCode` unless the result is actually silent.
  - confidence: 0.87
- **[P2] Keep ensure-session recovery after silent ensure exits:** `extensions/acpx/src/runtime.ts:482-485`
  - body: The new early rethrow prevents `recoverEnsureFailure` from running for silent `sessions ensure` exits. That drops the existing status probe and reuse/recreate recovery path for transient ensure failures that could still be recoverable.
  - confidence: 0.84
