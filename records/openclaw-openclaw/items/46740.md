## Summary

Keep this PR open, but do not merge it as-is: the bug report is plausible, yet the proposed patch over-classifies non-silent/control failures, can bypass ensure-session recovery, lacks after-fix real behavior proof, and now predates the current acpx/runtime delegation on main.

## What This Changes

This PR changes acpx runtime error classification, ACP manager fallback, fixtures, and tests so silent non-zero acpx turn exits report `ACP_BACKEND_UNAVAILABLE` while session-init exits keep `ACP_SESSION_INIT_FAILED`.

## Best Possible Solution

Move the silent backend-exit classification into the acpx-owned runtime contract, consume the appropriate pinned acpx release from OpenClaw, keep any manager fallback narrow, and preserve existing stderr/control/init recovery behavior.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection shows the current acpx 0.7.0 runtime emits unclassified runtime failures that OpenClaw normalizes to `ACP_TURN_FAILED`, and current tests cover `acpx exited with code 1` remaining in that class for some paths.

## Solution Assessment

No. The patch tries to solve a plausible bug in the OpenClaw wrapper and manager fallback, but current main has moved the reusable runtime boundary into `acpx/runtime` and the proposed code over-classifies non-silent/control failures while disrupting ensure recovery.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Guard backend-unavailable behind truly silent prompt exits:** `extensions/acpx/src/runtime.ts:679-680`
  - body: This upgrades any non-zero prompt exit without a structured ACP error event to `ACP_BACKEND_UNAVAILABLE`, even when stderr contains a concrete adapter or validation failure. That routes normal turn failures to `/acp doctor`; only genuinely silent exits should get the backend-unavailable code.
  - confidence: 0.89
- **[P2] Preserve control fallback for stderr failures:** `extensions/acpx/src/runtime.ts:1041-1044`
  - body: `runControlCommand` now maps every non-permission non-zero control exit with fallback `ACP_TURN_FAILED` to backend-unavailable. Control commands can fail with plain stderr for input or adapter reasons, so this should keep `params.fallbackCode` unless the result is actually silent.
  - confidence: 0.87
- **[P2] Keep ensure-session recovery after silent ensure exits:** `extensions/acpx/src/runtime.ts:483-484`
  - body: The new early rethrow prevents `recoverEnsureFailure` from running for silent `sessions ensure` exits. That drops the existing status probe and reuse/recreate recovery path for transient ensure failures that could still be recoverable.
  - confidence: 0.84
