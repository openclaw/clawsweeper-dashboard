## Summary

Keep open: the requested classification is not implemented on current main, but this PR is not the right patch to merge because current main delegates ACPX process handling to the published `acpx` package and the supplied diff still introduces behavior regressions in exit-code classification and ensure-session recovery.

## What This Changes

The PR edits the ACPX runtime wrapper, ACP manager fallback, fixtures, and tests so selected `acpx exited with code 1` failures surface as `ACP_BACKEND_UNAVAILABLE` with `/acp doctor` guidance.

## Best Possible Solution

Move the classification into `openclaw/acpx` so only genuinely silent backend crashes become `ACP_BACKEND_UNAVAILABLE`, publish and bump the package, then keep narrow OpenClaw coverage for manager, Discord, and error-text behavior.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for source-level PR review: the supplied diff deterministically remaps exit paths, and current main plus `acpx@0.6.1` inspection shows unknown runtime event codes still fall back to `ACP_TURN_FAILED`.

## Solution Assessment

Is this the best way to solve the issue? No. The durable fix belongs in the upstream `acpx` runtime package with OpenClaw consuming the released package, not by reintroducing exit-classification logic into the thin bundled wrapper.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Gate backend-unavailable on truly silent prompt exits:** `extensions/acpx/src/runtime.ts:676`
  - body: `runTurn` classifies any nonzero prompt exit without a structured error event as `ACP_BACKEND_UNAVAILABLE`, but this path also covers stderr-only prompt, auth, or validation failures. Those should keep turn-failure recovery instead of sending users to `/acp doctor`.
  - confidence: 0.86
- **[P2] Preserve turn-failure codes for stderr control exits:** `extensions/acpx/src/runtime.ts:1038-1044`
  - body: `resolveSilentAcpxControlExitErrorCode` only sees the exit code and fallback code, so non-permission-denied control exits with stderr can be mislabeled as backend outages. Control/input failures from status, cancel, close, mode, or config paths should not be routed to backend doctor guidance unless they are truly silent backend crashes.
  - confidence: 0.84
- **[P2] Keep ensure-session recovery for silent ensure exits:** `extensions/acpx/src/runtime.ts:480-482`
  - body: The early rethrow skips `recoverEnsureFailure` for silent `sessions ensure` exits, so OpenClaw no longer runs the status probe and fallback reuse/recreate flow that can recover transient session-init failures.
  - confidence: 0.83
