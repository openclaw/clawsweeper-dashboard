## Summary

Keep open, but this PR should not merge as-is. Current main has moved ACPX runtime behavior into the published `acpx` package, the branch is dirty against main, and the proposed patch still broadens error classification in ways that can misroute non-silent failures and skip existing session recovery.

## What This Changes

The PR adds runtime, manager, and error-text handling/tests so selected `acpx exited with code 1` failures surface as `ACP_BACKEND_UNAVAILABLE` with `/acp doctor` guidance.

## Best Possible Solution

Implement the classification in `openclaw/acpx` so only genuinely silent backend crashes become `ACP_BACKEND_UNAVAILABLE`, then publish or pin that package and update OpenClaw wrapper, manager, Discord, and error-text coverage.

## Reproduction Assessment

Yes for source-level review: the PR diff deterministically remaps prompt/control exit paths without proving stderr silence and adds an early ensure-session rethrow that bypasses recovery. I did not execute tests because this was a read-only review.

## Solution Assessment

No. The maintainable fix belongs in the ACPX runtime package consumed by OpenClaw, and this branch both targets an obsolete implementation surface and introduces classification regressions.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Guard backend-unavailable code behind truly silent prompt exits:** `extensions/acpx/src/runtime.ts:676`
  - body: `runTurn` now tags any non-zero prompt exit without a structured error event as `ACP_BACKEND_UNAVAILABLE`, but this path can also include concrete stderr output. Stderr-backed prompt/auth/model/validation failures should keep turn-failure recovery instead of routing users to `/acp doctor`.
  - confidence: 0.86
- **[P2] Preserve turn-failure codes for non-silent control exits:** `extensions/acpx/src/runtime.ts:1038-1044`
  - body: `resolveSilentAcpxControlExitErrorCode` only receives the exit code and fallback code, so every non-permission-denied control-command exit with an `ACP_TURN_FAILED` fallback becomes `ACP_BACKEND_UNAVAILABLE` even when the command wrote stderr. Status, cancel, close, mode, or config failures can therefore be mislabeled as backend outages.
  - confidence: 0.84
- **[P2] Keep ensure-session recovery path for silent ensure exits:** `extensions/acpx/src/runtime.ts:480-482`
  - body: The early rethrow skips `recoverEnsureFailure` for silent `sessions ensure` exits, so OpenClaw no longer runs the status probe and fallback reuse/recreate flow that can recover transient session-init failures.
  - confidence: 0.83
