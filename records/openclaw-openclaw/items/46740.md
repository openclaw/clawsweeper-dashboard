## Summary

Keep open: the bug class is plausible, but this PR is now conflicted/outdated against current main, lacks real acpx backend proof, and still introduces over-broad error classification regressions.

## What This Changes

The PR changes the acpx runtime wrapper, ACP manager fallback, fixtures, and tests so silent non-zero acpx turn exits report `ACP_BACKEND_UNAVAILABLE` while session-init exits keep `ACP_SESSION_INIT_FAILED`.

## Best Possible Solution

Move the silent backend-exit classification into the acpx-owned runtime contract, bump OpenClaw's pinned `acpx` dependency, keep any manager fallback narrow, and validate it with a real backend smoke proof.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection shows the current `acpx` 0.7.0 runtime emits legacy terminal errors that OpenClaw normalizes to `ACP_TURN_FAILED`, and the PR's proposed paths are reviewable at the cited head lines.

## Solution Assessment

No. The requested behavior should be fixed at the acpx runtime boundary and then consumed by OpenClaw, while the current PR's wrapper changes over-classify non-silent/control failures and disrupt ensure recovery.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Guard backend-unavailable behind truly silent prompt exits:** `extensions/acpx/src/runtime.ts:679-680`
  - body: This upgrades any non-zero prompt exit without a structured error event to `ACP_BACKEND_UNAVAILABLE`, even when stderr contains a concrete adapter or validation failure. That routes normal turn failures to `/acp doctor`; only genuinely silent exits should get the backend-unavailable code.
  - confidence: 0.89
- **[P2] Preserve control fallback for stderr failures:** `extensions/acpx/src/runtime.ts:1041-1044`
  - body: `runControlCommand` now maps every non-permission non-zero control exit with fallback `ACP_TURN_FAILED` to backend-unavailable. Control commands can fail with plain stderr for input or adapter reasons, so this should keep `params.fallbackCode` unless the result is actually silent.
  - confidence: 0.87
- **[P2] Keep ensure-session recovery after silent ensure exits:** `extensions/acpx/src/runtime.ts:483-484`
  - body: The new early rethrow prevents `recoverEnsureFailure` from running for silent `sessions ensure` exits. That drops the existing status probe and reuse/recreate recovery path for transient ensure failures that could still be recoverable.
  - confidence: 0.84
