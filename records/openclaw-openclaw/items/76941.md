## What This Changes

The PR routes external streaming steer turns through the follow-up queue, preserves embedded steering for internal webchat, and adds Telegram transcript doctor repair coverage.

## Best Possible Solution

Merge after the steer gate reuses the authoritative Provider/messageProvider channel identity and the doctor repair matches metadata by stored transcript identity rather than filename basename.

## Reproduction Assessment

Yes, source-level. Current main unconditionally sends streaming steer turns to queueEmbeddedPiMessage, and the PR's added Telegram case demonstrates the intended external-channel queue path, though local test execution was unavailable.

## Solution Assessment

No. The direction is right, but this implementation should use the existing Provider-first/messageProvider contract for steer classification and resolve transcript metadata from stored session file identity or the transcript header session id.
