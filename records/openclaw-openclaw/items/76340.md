## Summary

Keep open. The report is credible and detailed for v2026.4.27/v2026.4.29, and current main contains several relevant beta/main performance fixes, but there is not yet high-confidence proof that the Rocky Linux Docker plus Feishu/runtime-deps path is fixed after those changes.

## What This Changes

The issue reports a post-v2026.4.23 performance regression where OpenClaw blocks the Node.js event loop for about 9-15 seconds, slowing status commands and causing Feishu/Lark timeouts.

## Best Possible Solution

Validate current main or the beta build in the same Docker/Feishu setup; if the delay remains, profile the dominant hot path and land one targeted regression fix there.
