## Summary

Current main and the shipped 2026.5.x releases cover the reported v2026.4.27/v2026.4.29 plugin/runtime-deps event-loop regression: the hot plugin metadata paths reuse prepared snapshots, unchanged manifests are cached, legacy plugin-runtime-deps debris is cleanup-only, and the linked trajectory-flush stall was also bounded.

## What This Changes

The issue reports a post-v2026.4.23 regression where plugin/tool scanning blocks the Node event loop, slows `openclaw status`, and causes Feishu/Lark timeouts.

## Best Possible Solution

Keep the shipped plugin metadata snapshot, manifest cache, legacy runtime-deps cleanup, and bounded trajectory implementations; use a fresh issue only if v2026.5.4 or newer still reproduces with current liveness and plugin-status logs.

## Reproduction Assessment

## Solution Assessment

Yes. The shipped fix moves plugin metadata facts earlier and reuses them, removes legacy runtime-deps debris through repair/update paths, and bounds the linked trajectory flush instead of adding a timeout knob that would mask event-loop starvation.
