Keep open. Current main has relevant mitigations for status fast paths, bundled-plugin dependency cleanup, Feishu setup/status probing, and session update churn, but those do not prove this mixed Docker/Feishu/runtime-deps event-loop regression is fixed on the reporter's upgrade path.

## Best Possible Solution

Capture a current-main or post-v2026.5.2 Docker repro with gateway stability, status timing, Feishu probe timing, and doctor cleanup output, then fix the specific remaining hot path that the trace identifies.
