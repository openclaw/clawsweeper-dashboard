## Best Possible Solution

Keep runtime initialization snapshots hydrated, compute only the concurrency revision from a persisted/unhydrated representation, and add focused regression coverage plus redacted Discord second-turn proof.

## Solution Assessment

No. Normalizing the revision to persisted shape is the right direction, but returning unhydrated entries from the runtime initialization snapshot is not the best fix because prompt assembly expects hydrated `skillsSnapshot.prompt`.
