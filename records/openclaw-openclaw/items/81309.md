## Summary

Close as plugin/ClawHub-scoped: the requested WeChat fetch-resource tool is a new channel-specific capability owned by the external `@tencent-weixin/openclaw-weixin` plugin, while OpenClaw core already provides the plugin-owned agent tool contract.

## What This Changes

Request to add a WeChat `weixin_fetch_resource` agent tool for downloading decrypted files from `openclaw-weixin` messages by message and file key.

## Best Possible Solution

Keep OpenClaw core channel-agnostic and have `@tencent-weixin/openclaw-weixin`, or a ClawHub-distributed companion plugin, publish the WeChat fetch-resource tool through the existing plugin tool contract.

## Reproduction Assessment

Not applicable; this is a feature request, not a current-main bug report. The relevant check is scope and API ownership, and the current docs/code show WeChat media runtime is external-plugin owned.

## Solution Assessment

Yes for core cleanup: the narrow maintainable path is an external WeChat plugin or ClawHub plugin registering the tool through `api.registerTool` and `contracts.tools`. Any generic plugin tool-registration failure is separately tracked at https://github.com/openclaw/openclaw/issues/77982.
