## Summary

## What This Changes

The issue reports that Windows subprocess callers get valid JSON from `openclaw gateway status --json --require-rpc --timeout 5000`, but the OpenClaw Node process stays alive until the parent kills it.

## Best Possible Solution

Capture a Windows packaged subprocess repro with parent/child exit events and active handles, then release or unref the specific OpenClaw-owned handle in the status probe or Windows respawn lifecycle instead of adding an unconditional successful-command `process.exit()`.

## Solution Assessment
