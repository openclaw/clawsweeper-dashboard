## Summary

Keep open: current main and v2026.6.10 still lack a normal-session max-lifetime or max-turn rotation policy; existing code covers daily/idle reset, compaction, and thread-binding max age only, so this remains a session-lifecycle product/config decision.

## What This Changes

The issue requests configurable session maximum age and turn-count limits that automatically rotate long-running agent sessions after persisting memory.

## Best Possible Solution

Design one core session-lifecycle policy with canonical config, durable age/turn accounting, safe pre-rotation memory or continuity behavior, docs, and focused upgrade tests.

## Reproduction Assessment

Yes, at source level: current main and v2026.6.10 have no normal-session max-age/max-turn config, no durable turn counter, and no evaluator input for this policy. I did not run a six-day fleet reproduction because the requested behavior is absent in source.

## Solution Assessment

Unclear until maintainers choose the public config/defaults and continuity semantics. The likely layer is core session lifecycle, not the existing focused-thread max-age or context-pruning controls.
