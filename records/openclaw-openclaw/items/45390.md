## Summary

Keep open. Current main covers daily/idle reset, heartbeat-aware freshness, thread-binding max-age auto-unfocus, session store maintenance, and pre-compaction memory flush, but it still does not provide configurable normal-session max-age or max-turn rotation with safe pre-rotation persistence.

## What This Changes

The issue requests configurable automatic session rotation by maximum session age and maximum turn count, with a pre-rotation memory save and rotation logging.

## Best Possible Solution

Extend the existing session reset lifecycle with documented max-age and max-turn fields, durable accounting, typed rollover reasons, and a safe pre-rotation memory hook that composes with daily/idle reset, compaction, and provider-owned sessions.

## Reproduction Assessment

Not applicable as a feature request. Static source and docs review is high-confidence because current main exposes daily/idle reset and adjacent maintenance controls, but no normal-session max-age or max-turn rotation policy.

## Solution Assessment

Unclear until maintainer API review. The requested direction is valid, but the maintainable path should likely extend `session.reset` rather than add separate `agents.defaults` aliases with independent lifecycle semantics.
