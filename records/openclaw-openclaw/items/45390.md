## Summary

Keep open. Current main has daily/idle rollover, heartbeat isolation/freshness mitigations, and lifecycle hooks, but it still lacks a configurable normal-session max-age or max-turn rotation policy; related #42867 covers part of the turn-count/heartbeat surface but not the broader max-age/API shape.

## What This Changes

Request configurable session max-age and max-turn limits that save state, rotate to a fresh session, and log automatic rollover.

## Best Possible Solution

Extend the existing session.reset lifecycle with documented max-age and max-turn limits, durable counters, typed rollover reasons, and a safe hook/persistence path for automatic rollover.

## Reproduction Assessment

Not applicable as a feature request. Static verification is high-confidence because current schema, types, reset evaluation, runtime reuse, docs, and store shape do not expose normal-session max-age or max-turn rotation.

## Solution Assessment

Unclear until maintainer API review. The requested direction is valid, but the maintainable solution should consolidate with #42867 and extend session.reset rather than adding disconnected agents.defaults aliases.
