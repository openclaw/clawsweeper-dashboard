## Summary

Keep open: current main and the latest release have heartbeat-aware daily/idle freshness and a thread-binding max-age helper, but no general session TTL or max-turn rotation for normal agent sessions.

## What This Changes

The issue requests configurable session max age/max turns that auto-flushes memory, starts a fresh session, and logs the rotation.

## Best Possible Solution

Design a core session-lifecycle feature that extends the existing reset policy with explicit max-age/max-turn semantics, durable counting, safe memory/transcript flush behavior, and documented upgrade/default behavior.

## Reproduction Assessment

Source-reproducible, not a bug repro: current main has no normal-session max-age/max-turn config, enforcement path, or durable turn counter. I did not run a six-day fleet reproduction.

## Solution Assessment

Unclear pending maintainer design: the requested direction is plausible, but the best implementation must settle config shape, turn-count semantics, and safe pre-rotation persistence.
