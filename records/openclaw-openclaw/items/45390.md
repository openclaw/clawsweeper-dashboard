## Summary

Keep open. Current main still has daily, idle, manual, heartbeat-aware, and thread-binding session lifecycle behavior, but it does not provide configurable normal-session max-age or max-turn rotation with safe pre-rotation persistence; related issues either point here as canonical or track adjacent budget/context-split work.

## What This Changes

Request an opt-in session TTL and max-turn policy that persists state and automatically rotates long-lived agent sessions.

## Best Possible Solution

Define an additive central session lifecycle policy with typed max-age and max-turn limits, durable accounting, explicit rollover reasons, safe memory-flush semantics, schema/help/docs alignment, regression coverage, and default-off upgrade compatibility.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, test, and related-issue inspection give high-confidence absence proof for normal-session max-age/max-turn rotation on current main.

## Solution Assessment

Unclear until maintainer API review. The requested direction is valid, but the maintainable solution should extend central session lifecycle/reset contracts and coordinate persistence/accounting semantics rather than add isolated aliases.
