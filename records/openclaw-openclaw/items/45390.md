## Summary

Keep open. Current main and latest shipped release still expose daily/idle reset, heartbeat-aware freshness, session maintenance, heartbeat isolated sessions, and thread-bound max-age, but not configurable normal-session max-age or max-turn rotation with safe pre-rotation persistence.

## What This Changes

The issue requests configurable maximum session age and turn-count rotation with pre-rotation persistence for long-running agent sessions.

## Best Possible Solution

Define one central session lifecycle policy with typed max-age and max-turn triggers, durable accounting, explicit rollover reasons, safe pre-rotation persistence, schema/help/docs alignment, and regression coverage across normal, cron, heartbeat, and system-event paths.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and docs inspection give a high-confidence absence check because current main exposes daily/idle reset and thread-bound max-age only, with no normal-session max-age or max-turn branch.

## Solution Assessment

Unclear until maintainer API review. The requested direction is valid, but the maintainable solution should extend the existing session reset/lifecycle contracts and coordinate persistence and accounting semantics rather than add isolated aliases.
