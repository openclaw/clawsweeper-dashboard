## Summary

Keep open. Current main still has daily/idle reset, heartbeat-aware freshness, thread-binding max-age auto-unfocus, and pre-compaction memory flush, but it does not expose a configurable normal-session max-age or max-turn rotation policy with safe pre-rotation persistence; the related turn-count issue was closed in favor of this issue, and the overlapping open maintenance PR does not close or implement this request on main.

## What This Changes

Request configurable max-age and max-turn automatic session rotation with pre-rotation memory persistence and logging.

## Best Possible Solution

Extend the existing session.reset lifecycle with documented max-age and max-turn fields, durable accounting, typed rollover reasons, and a safe pre-rotation memory hook that composes with daily/idle resets and provider-owned sessions.

## Reproduction Assessment

Not applicable as a feature request. Static verification is high-confidence because current schema, runtime reuse logic, tests, and docs expose daily/idle/manual lifecycle controls but no normal-session max-age or max-turn rotation policy.

## Solution Assessment

Unclear until maintainer API review. The requested direction is valid, but the maintainable path should likely extend session.reset rather than add agents.defaults aliases with separate lifecycle semantics.
