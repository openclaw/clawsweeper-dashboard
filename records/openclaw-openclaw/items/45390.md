## Summary

Keep open. Current main still lacks configurable normal-session max-age or max-turn rotation, even though daily/idle/manual reset, heartbeat freshness, session lifecycle hooks, and pre-compaction memory flush support are present.

## What This Changes

Requests configurable session max-age and max-turn automatic rotation that saves memory, starts a fresh session, and logs the rollover.

## Best Possible Solution

Extend the existing session.reset lifecycle with documented max-age and max-turn limits, durable counters, typed rollover reasons, and a safe hook/persistence path for automatic rollover.

## Reproduction Assessment

Not applicable as a feature request. Static verification is high-confidence because current main's schema, types, runtime reuse path, store type, and docs expose daily/idle/manual lifecycle controls but no normal-session max-age or max-turn rotation policy.

## Solution Assessment

Unclear until maintainer API review. The requested direction is valid, but the maintainable fix should consolidate with #42867 and extend the existing reset lifecycle instead of adding disconnected agent-default aliases.
