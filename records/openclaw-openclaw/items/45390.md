## Summary

Keep open. Current main fixes adjacent heartbeat/daily/idle freshness problems and exposes session lifecycle hooks, but it still lacks configurable normal-session max-age or max-turn rotation.

## What This Changes

Request configurable session TTL and turn-count limits that automatically save state, rotate to a fresh session, and log the rollover.

## Best Possible Solution

Extend the existing session.reset lifecycle with documented max-age and max-turn limits, durable turn accounting, typed rollover reasons, and a safe pre-rotation persistence path that aligns with #42867.

## Reproduction Assessment

Not applicable as a feature request. Static verification is high-confidence because current schema, runtime reuse logic, docs, and store types expose daily/idle/manual lifecycle controls but no normal-session max-age or max-turn policy.

## Solution Assessment

Unclear until maintainer API review. The direction is valid, but the maintainable fix should build on session.reset and existing session_start/session_end hooks rather than adding disconnected agents.defaults aliases.
