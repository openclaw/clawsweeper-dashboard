## Summary

Keep open. Current main has daily/idle rollover and heartbeat-aware freshness, but it still lacks configurable normal-session max-age or max-turn rotation with safe pre-rotation persistence.

## What This Changes

Request configurable session maximum age and turn-count limits that rotate long-running agent sessions after saving state.

## Best Possible Solution

Extend the existing `session.reset` lifecycle with documented max-age and max-turn fields, durable accounting, typed rollover reasons, and a safe pre-rotation memory hook that composes with daily/idle resets.

## Reproduction Assessment

Not applicable as a feature request. Static verification is high-confidence because current schema, runtime reuse logic, store types, and docs expose daily/idle/manual lifecycle controls but no normal-session max-age or max-turn rotation policy.

## Solution Assessment

Unclear until maintainer API review. The requested direction is valid, but the maintainable path should extend the existing `session.reset` lifecycle rather than introduce separate `agents.defaults` aliases with disconnected semantics.
