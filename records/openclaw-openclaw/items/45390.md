## Summary

Keep open. Current main still lacks configurable normal-session max-age or max-turn rotation; the related trackers cover heartbeat isolation, budget caps, auto-splitting, pruning, and pre-reset memory, but none supersedes this core lifecycle/API decision.

## What This Changes

Request a configurable session max-age/max-turn lifecycle that saves memory, rotates to a fresh session, and logs the rotation.

## Best Possible Solution

Define a central session lifecycle policy with typed max-age and max-turn triggers, durable accounting, explicit rollover reasons, safe pre-rotation persistence, schema/help/docs alignment, and regression coverage across normal, cron, and system-event paths.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection is high-confidence because current schema/types/docs expose daily/idle/manual reset and thread-bound max-age only, while the reuse gate and freshness evaluator have no max-age or max-turn branch.

## Solution Assessment

Unclear until maintainer API review. The requested direction is valid, but the maintainable path should extend the existing session lifecycle/reset contracts and coordinate with budget, auto-split, pruning, and pre-reset memory trackers rather than add isolated aliases with independent semantics.
