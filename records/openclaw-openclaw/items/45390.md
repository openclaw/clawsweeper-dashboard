## Summary

Keep open. Current main and v2026.5.20 still provide daily/idle/manual normal-session resets, thread-binding max-age, heartbeat isolated sessions, and compaction/memory-flush mitigations, but not configurable normal-session max-age or max-turn rotation with safe pre-rotation persistence; the related turn-count report at https://github.com/openclaw/openclaw/issues/42867 was closed back to this as the canonical design item.

## What This Changes

Request a configurable session max-age/max-turn policy that flushes or persists state, starts a fresh session, and logs rotation before long-running agent sessions exceed practical context limits.

## Best Possible Solution

Define one additive opt-in session lifecycle policy with typed max-age and max-turn triggers, durable accounting, explicit rollover reasons, safe pre-rotation persistence, schema/help/docs alignment, and regression coverage across reset paths.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, related-issue, and release-tag inspection give high-confidence absence proof for normal-session max-age/max-turn rotation on current main and v2026.5.20.

## Solution Assessment

Unclear until maintainer API review. The requested direction is valid, but the maintainable solution should extend central session lifecycle/reset contracts and coordinate persistence/accounting semantics rather than add isolated ad hoc aliases.
