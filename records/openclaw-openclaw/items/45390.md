## Summary

Keep open: current main and v2026.6.10 still lack a normal-session max-age or max-turn rotation policy; existing daily/idle reset, thread-binding max-age, compaction, and cache-TTL pruning are adjacent mechanisms but do not satisfy the requested lifecycle cap.

## What This Changes

Requested configurable maximum session age and turn-count rotation so long-running agent sessions can rotate before context quality and provider limits degrade.

## Best Possible Solution

Design one core session-lifecycle policy with canonical config, durable age/turn accounting, safe continuity or memory behavior, docs, and focused upgrade tests.

## Reproduction Assessment

Yes, at source level: current main lacks normal-session max-age/max-turn config, durable turn accounting, and reset freshness inputs for this policy. I did not run a six-day fleet reproduction because the requested policy is absent in source.

## Solution Assessment

Unclear until maintainers choose the public config, defaults, and continuity semantics. The likely owner is core session lifecycle, not thread-binding max-age or context-pruning alone.
