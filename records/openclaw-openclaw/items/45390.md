## Summary

Keep open. Current main has daily/idle rollover, heartbeat freshness fixes, and thread-bound max-age controls, but it still lacks a configurable normal-session max-age or max-turn rotation policy with safe pre-rotation persistence.

## What This Changes

The issue requests configurable session maximum age and maximum turn limits that automatically save state, rotate the session, and log the rollover.

## Best Possible Solution

Extend the existing session.reset lifecycle with documented max-age and max-turn limits, durable counters, typed rollover reasons, and a safe hook/persistence path before automatic rollover.

## Reproduction Assessment

Not applicable as a feature request. Static verification is high-confidence because current schema, types, reset policy, runtime reuse path, docs, and store shape do not expose normal-session max-age or max-turn rotation.

## Solution Assessment

Unclear until maintainer API review. The direction is valid, but the maintainable path should consolidate with #42867 and extend session.reset rather than adding disconnected agents.defaults aliases.
