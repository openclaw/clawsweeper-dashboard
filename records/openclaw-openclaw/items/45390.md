## Summary

Keep open: current main and the latest release still lack a normal-session max-age or max-turn rotation policy; existing daily/idle reset and thread-binding max-age behavior only cover adjacent lifecycle cases.

## What This Changes

Request adds configurable session max age and max turns so long-running agent sessions rotate automatically after a lifetime or turn threshold.

## Best Possible Solution

Define one session lifecycle contract for max-age and max-turn rotation, with explicit continuity/memory behavior and focused docs, schema, and reset-path tests before adding the public config.

## Reproduction Assessment

Yes at source level: current main has daily and idle freshness checks, but no normal-session max-age or max-turn policy to enforce the requested rotation. I did not run a live multi-day session repro because the missing config/runtime branch is directly visible in source and docs.

## Solution Assessment

Unclear as implementation policy: a TTL/max-turn option is plausible, but the config shape and pre-rotation memory behavior need maintainer review. A narrow code fix should wait until maintainers choose how this coordinates with reset memory flush and context-window auto-split behavior.
