## Summary

Keep open. Current main and v2026.5.19 still provide daily/idle reset, heartbeat-aware freshness, thread-binding max-age, session maintenance, and compaction memory flush, but not configurable normal-session max-age or max-turn rotation with safe pre-rotation persistence. The closed turn-count report at https://github.com/openclaw/openclaw/issues/42867 explicitly leaves this as the canonical TTL/max-turn design item, while https://github.com/openclaw/openclaw/issues/53321, https://github.com/openclaw/openclaw/issues/64463, and https://github.com/openclaw/openclaw/issues/45608 are adjacent rather than superseding it.

## What This Changes

Requested configurable session TTL and max-turn automatic rotation for long-running agent sessions.

## Best Possible Solution

Define one central session lifecycle policy with typed opt-in max-age and max-turn triggers, durable accounting, explicit rollover reasons, safe pre-rotation persistence, schema/help/docs alignment, and regression coverage across normal, cron, heartbeat, CLI, ACP, and system-event paths.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and docs inspection give high-confidence absence proof because current main and v2026.5.19 expose daily/idle reset and thread-binding max-age only, with no normal-session max-age or max-turn branch.

## Solution Assessment

Unclear until maintainer API review. The requested direction is valid, but the maintainable solution should extend the existing session lifecycle/reset contracts and coordinate persistence/accounting semantics rather than add isolated aliases.
