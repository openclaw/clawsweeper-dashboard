## Summary

Current main fixes the reported heartbeat-driven indefinite-session pattern by separating session start and real interaction timestamps from general store updates, but it still has no configurable absolute age or turn-count limit. The remaining request is a valid core session-lifecycle feature that needs maintainer product judgment, especially because safe automatic rotation must coordinate persistence, queued work, and the related token-pressure split proposal.

## What This Changes

Requests configurable maximum session age and turn limits that safely persist memory before automatically rotating long-running sessions.

## Best Possible Solution

Design one owner-approved session-rotation state machine with age, turn, and token-pressure triggers, a safe idle boundary, durable pre-rotation flush/checkpoint behavior, explicit continuity metadata, and canonical configuration under the existing session policy surface.

## Reproduction Assessment

Yes for the remaining capability gap: current-main types, strict schema, and freshness evaluation show no absolute elapsed-lifetime or turn-count trigger. The original heartbeat-maintained lifetime symptom is no longer reproducible on current main because system events do not extend reset freshness.

## Solution Assessment

No, the configuration sketch alone is not yet the best complete solution. Age and turn limits are reasonable triggers, but they should use a shared rotation state machine and explicit continuity contract with https://github.com/openclaw/openclaw/issues/53321 rather than an ad hoc system event followed by immediate reset.
