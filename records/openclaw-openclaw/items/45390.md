## Summary

Keep open. Current main and v2026.5.20 still provide daily, idle, manual, heartbeat-aware, and thread-binding session lifecycle behavior, but not configurable normal-session max age or max-turn rotation with safe pre-rotation persistence; related reports and the open session-maintenance PR do not supersede this product/API design item.

## What This Changes

The issue requests configurable maximum session age and turn-count rotation with safe pre-rotation persistence for long-running agent sessions.

## Best Possible Solution

Define one additive opt-in session lifecycle policy with typed max-age and max-turn triggers, durable accounting, explicit rollover reasons, safe pre-rotation persistence, schema/help/docs alignment, and regression coverage across reset paths.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, related-issue, open-PR, and release-tag inspection give high-confidence absence proof for normal-session max-age/max-turn rotation on current main and v2026.5.20.

## Solution Assessment

Unclear until maintainer API review. The requested direction is valid, but the maintainable solution should extend central session lifecycle/reset contracts and coordinate persistence/accounting semantics rather than add isolated ad hoc aliases.
