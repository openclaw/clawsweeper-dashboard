## Summary

Keep open. Current main has daily/idle reset, heartbeat-aware freshness, thread-binding max-age auto-unfocus, and pre-compaction memory flush, but it still lacks configurable normal-session max-age or max-turn rotation with safe pre-rotation persistence.

## What This Changes

Request configurable session max-age and max-turn automatic rotation, with pre-rotation memory persistence and rotation logging.

## Best Possible Solution

Extend the existing session.reset lifecycle with documented max-age and max-turn fields, durable accounting, typed rollover reasons, and a safe pre-rotation memory hook that composes with daily/idle resets.

## Reproduction Assessment

Not applicable as a feature request. Static verification is high-confidence because current schema, runtime reuse logic, tests, and docs expose daily/idle/manual lifecycle controls but no normal-session max-age or max-turn rotation policy.

## Solution Assessment

Unclear until maintainer API review. The requested direction is valid, but the maintainable path should likely extend session.reset rather than add disconnected agents.defaults aliases with separate semantics.
