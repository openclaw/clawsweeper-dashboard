## Summary

Keep open: current main and the latest release still lack a normal-session max-age or max-turn rotation policy, so this remains a core session-lifecycle product/config decision rather than cleanup.

## What This Changes

Request a configurable normal-session max age and max-turn limit that saves memory, starts a fresh session, and logs automatic rotation.

## Best Possible Solution

Design one core session-lifecycle policy with canonical config, durable age/turn accounting, safe pre-rotation memory or continuity behavior, docs, and focused upgrade tests.

## Reproduction Assessment

Yes at source level for the missing capability: current main and v2026.6.8 have no normal-session max-age/max-turn config, no durable turn counter, and no evaluator input for this policy. I did not run a six-day fleet reproduction because the requested behavior is absent in source.

## Solution Assessment
