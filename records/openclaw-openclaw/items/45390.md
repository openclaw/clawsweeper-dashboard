## Summary

Keep open. Current main still has daily, idle, manual reset, heartbeat isolation, lifecycle hooks, and pre-compaction memory flushes, but it does not expose configurable normal-session max-age or max-turn automatic rotation for the requested TTL behavior.

## What This Changes

Request configurable session max-age and max-turn limits that save memory, rotate to a fresh session, and log the rollover before sessions grow too large.

## Best Possible Solution

Extend the existing session reset lifecycle with documented max-age and max-turn limits, durable counters, typed rollover reasons, and a safe pre-rollover persistence hook path.

## Reproduction Assessment

Not applicable as a feature request. Static verification is high-confidence because current main's config schema, reset policy, runtime reuse gate, store type, tests, and docs expose daily/idle/manual lifecycle controls but no normal-session max-age or max-turn TTL.

## Solution Assessment

Unclear until maintainer API review. The requested direction is valid, but the maintainable fix should consolidate with #42867 and extend the existing reset lifecycle instead of adding disconnected agent-default aliases.
