## Summary

Keep open. Current main still lacks configurable normal-session max-age or max-turn rotation; adjacent open trackers cover token-pressure splitting, session budget caps, rolling pruning, and pre-reset memory, but none supersedes this TTL lifecycle decision.

## What This Changes

Add configurable session max-age and max-turn rotation that saves state, starts a fresh session, and logs the rollover.

## Best Possible Solution

Define a central session lifecycle policy with typed max-age and max-turn triggers, durable accounting, explicit rollover reasons, safe pre-rotation persistence, docs, schema/help alignment, and regression coverage across normal, cron, and system-event paths.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection is high-confidence because strict session schema/types expose daily/idle reset and thread-bound max-age only, while the reuse gate and freshness evaluator have no max-age or max-turn branch.

## Solution Assessment

Unclear until maintainer API review. The requested direction is valid, but the maintainable path should extend the existing session lifecycle/reset contracts and coordinate with budget, auto-split, pruning, and pre-reset memory trackers rather than add separate aliases with independent semantics.
