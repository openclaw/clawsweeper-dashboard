## Summary

Keep open. Current main and v2026.5.22 still support daily/idle/manual resets and thread-binding max age, but not configurable normal-session max-age or max-turn rotation with safe pre-rotation persistence.

## What This Changes

The issue requests configurable session max-age and max-turn limits that automatically save memory, rotate to a fresh session, and log the rotation.

## Best Possible Solution

Define one additive opt-in session lifecycle policy with typed max-age and max-turn triggers, durable accounting, explicit rollover reasons, safe pre-rotation persistence, schema/help/docs alignment, and regression coverage across reset paths.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, related-issue, and release-tag inspection give high-confidence absence proof for normal-session max-age and max-turn rotation on current main and v2026.5.22.

## Solution Assessment

Unclear until maintainer API review. The requested direction is valid, but the maintainable solution should extend central session lifecycle/reset contracts and coordinate persistence/accounting semantics rather than add isolated ad hoc aliases.
