## Summary

Keep open. Current main has adjacent daily/idle reset, heartbeat-aware freshness, thread-binding max-age auto-unfocus, and session-store maintenance, but it still lacks configurable normal-session max-age or max-turn rotation with a safe pre-rotation persistence hook.

## What This Changes

The issue requests configurable session maximum age and maximum turns with automatic session rotation, memory persistence before rotation, and rotation logging.

## Best Possible Solution

Extend the existing `session.reset` lifecycle with documented max-age and max-turn fields, durable accounting, typed rollover reasons, and a safe pre-rotation memory hook.

## Reproduction Assessment

Not applicable as a feature request. Static source and docs review is high-confidence because current main exposes adjacent daily/idle reset behavior but no normal-session max-age or max-turn rotation policy.

## Solution Assessment

Unclear until maintainer API review. The requested direction is valid, but the maintainable path should likely extend `session.reset` rather than add separate `agents.defaults` aliases with independent lifecycle semantics.
