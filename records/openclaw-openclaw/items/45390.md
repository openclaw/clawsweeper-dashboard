## Summary

Keep open. Current main and v2026.5.19 still provide daily/idle reset, heartbeat-aware freshness, thread-binding max-age, session maintenance, and pre-compaction memory flush, but not configurable normal-session max-age or max-turn rotation with safe pre-rotation persistence; the closed turn-count report explicitly leaves this issue as the canonical TTL/max-turn design item.

## What This Changes

Request configurable session max-age and max-turn limits that automatically save memory, rotate to a fresh session, and log the rotation.

## Best Possible Solution

Define one additive opt-in session lifecycle policy with typed max-age and max-turn triggers, durable accounting, explicit rollover reasons, safe pre-rotation persistence, schema/help/docs alignment, and regression coverage across reset paths.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and docs inspection give high-confidence absence proof because current main and v2026.5.19 expose daily/idle reset and thread-binding max-age only, with no normal-session max-age or max-turn branch.

## Solution Assessment

Unclear until maintainer API review. The requested direction is valid, but the maintainable solution should extend the central session lifecycle/reset contracts and coordinate persistence/accounting semantics rather than add isolated aliases.
