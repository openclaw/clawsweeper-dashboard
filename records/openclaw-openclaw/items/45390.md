## Summary

Keep open: current main and v2026.6.6 still lack normal-session max-age or max-turn rotation; the request is valid core session lifecycle work, but it needs maintainer product direction before implementation.

## What This Changes

Request to add configurable session max-age and max-turn rotation that flushes memory, starts a new session, and logs the rollover.

## Best Possible Solution

Define one opt-in core session rotation policy on the existing reset lifecycle, with max-age and max-turn semantics, state-preservation behavior, docs/schema updates, and coverage across chat and Gateway agent entry points.

## Reproduction Assessment

Yes, at source level: current config/schema and session freshness paths expose only daily/idle timestamp reset for normal sessions, so max-age/max-turn rotation cannot be configured or enforced.

## Solution Assessment

Unclear until maintainers choose the product policy; the maintainable path is to extend the existing reset lifecycle rather than add a separate cleanup loop that can diverge across chat, Gateway, cron, and heartbeat behavior.
