## Summary

Keep open. Current main and the latest shipped tag still lack configurable normal-session max-age or max-turn rotation; the existing surfaces cover daily/idle freshness, heartbeat isolation, session maintenance, and thread-bound maxAgeHours, but they do not satisfy this lifecycle/API request.

## What This Changes

Request configurable session max-age and max-turn limits that auto-save state, rotate to a fresh session, and log the rollover.

## Best Possible Solution

Define a central session lifecycle policy with typed max-age and max-turn triggers, durable accounting, explicit rollover reasons, safe pre-rotation persistence, schema/help/docs alignment, and regression coverage across normal, cron, heartbeat, and system-event paths.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source and docs inspection give a high-confidence absence check because current main exposes daily/idle reset and thread-bound max-age only, with no max-age or max-turn branch for normal sessions.

## Solution Assessment

Unclear until maintainer API review. The requested direction is valid, but the maintainable solution should extend the existing session lifecycle/reset contracts and coordinate persistence and accounting semantics rather than add isolated aliases.
