## Summary

Keep open. Current main has daily/idle rollover, heartbeat-aware freshness, and lifecycle hooks, but it still lacks a configurable normal-session max-age or max-turn rotation policy with safe pre-rotation persistence.

## What This Changes

Request configurable session TTL and turn-count limits that automatically save state, rotate to a fresh session, and log the rollover.

## Best Possible Solution

Keep this open as the broader max-age/max-turn TTL design and extend session.reset with documented limits, durable counters, typed rollover reasons, and a safe pre-rotation persistence path coordinated with #42867.

## Reproduction Assessment

Not applicable as a feature request. Static verification is high-confidence because current schema, runtime reuse logic, docs, and store types expose daily/idle/manual lifecycle controls but no normal-session max-age or max-turn policy.

## Solution Assessment

Unclear until maintainer API review. The requested direction is valid, but the maintainable solution should consolidate under the existing session.reset lifecycle instead of adding disconnected agents.defaults aliases.
