## Summary

Close as a support/configuration issue: current main and the v2026.5.18 release already have the supported Gateway activation path for hook-only plugins, and the reported manifest is missing the activation metadata that makes a hook-only plugin part of live startup.

## What This Changes

The issue asks how to make a hook-only plugin that registers `tool_result_persist` run in the live Gateway after CLI plugin commands show it loaded.

## Best Possible Solution

Keep the existing activation-planning contract and treat this as configuration: hook-only plugins that must run in the live Gateway should declare hook startup intent in the manifest or explicit hook policy, then restart the Gateway.

## Reproduction Assessment

Yes from source inspection: a hook-only plugin with no activation metadata will not be selected by the Gateway startup plan. The current tests cover both the selected and intentionally skipped cases.

## Solution Assessment

Yes: the narrow supported solution is to add manifest activation such as `activation.onCapabilities: ["hook"]` or `activation.onStartup: true`, not to migrate `tool_result_persist` into a fake tool capability.
