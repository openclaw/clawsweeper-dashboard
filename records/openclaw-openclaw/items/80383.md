## Summary

Keep open: current main still lacks queued-item cancellation for abort-before-start, and this PR addresses a real ACP queue bug, but the PR head still has a blocking diagnostics-retention issue that should be fixed before merge.

## What This Changes

The PR adds AbortSignal-aware cancellation to SessionActorQueue, forwards withSessionActor's signal into it, and adds focused queue regression tests.

## Best Possible Solution

Keep the queue-level cancellation design, but make the diagnostic state item-scoped and cleaned after idle, or remove the production diagnostics map if it is not needed.

## Reproduction Assessment

Yes. Source inspection shows current main increments pending at enqueue, only decrements on queue settlement, and lets withSessionActor reject the caller before the queued actor starts without telling the queue.

## Solution Assessment

Mostly yes. SessionActorQueue is the right owner boundary for cancelling an already-enqueued slot while preserving per-key serialization, but the PR is not mergeable until the diagnostics map is removed or cleaned safely.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.82

Full review comments:

- **[P2] Avoid retaining settled diagnostics forever:** `src/acp/control-plane/session-actor-queue.ts:213-216`
  - body: Every enqueue stores a `diagnosticsBySession` entry for the actor key, but `onSettle` only stamps `settleTime` and never removes the entry once the key is idle. Because actor keys are session keys in a long-running gateway, this retains one diagnostic object for every session key ever queued even after `pendingBySession` and the tail map have been cleaned up; please either drop this production map or clean/update it only when the settling item still owns the live diagnostic entry.
  - confidence: 0.84
