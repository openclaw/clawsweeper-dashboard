## Summary

Keep open: current main and the latest release still lack queued-call cancellation for SessionActorQueue, and this PR is the only focused fix I found. It still needs maintainer-aware conflict resolution against the current ACP manager refactor and a small diagnostics guard before merge.

## What This Changes

The PR adds AbortSignal-aware cancellation to ACP SessionActorQueue, forwards withSessionActor signals into the queue, and adds focused queue regression tests.

## Best Possible Solution

Land a rebased signal-aware queue cancellation fix that keeps pending accounting item-scoped and makes diagnostics either item-scoped or limited to stable per-key facts.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: enqueue a non-settling first same-key task, enqueue a second through withSessionActor, then abort the second caller before actor start; current main rejects the caller but pending only decrements from onSettle, which cannot run while the predecessor is stuck.

## Solution Assessment

Mostly yes. Queue-level cancellation is the right owner boundary because the manager cannot settle a queued item blocked behind a non-settling predecessor; the mergeable shape should be rebased and should guard diagnostic updates by queued item identity.
