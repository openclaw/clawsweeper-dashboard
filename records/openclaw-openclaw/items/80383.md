The PR adds AbortSignal-aware cancellation to ACP SessionActorQueue, forwards manager abort signals into the queue, and adds focused queue regression tests.

## Best Possible Solution

Keep queue-owned cancellation, but remove or item-scope the diagnostic map so idle actor keys are released and diagnostic callbacks only update their own queued item.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: current main increments pending on enqueue and only decrements in onSettle, while withSessionActor can reject the caller before the queued actor starts. I did not run the live repro because this was a read-only review.

## Solution Assessment
