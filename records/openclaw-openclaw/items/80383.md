## What This Changes

The PR makes ACP `SessionActorQueue` accept an abort signal so a queued same-session item can be cancelled before it starts, decrementing pending diagnostics without changing public Plugin SDK queue APIs.

## Best Possible Solution

Merge a narrow queue-level cancellation fix after the contributor adds redacted active runtime proof for the abort-before-start case, while keeping the broader lane-timeout and session-lock policy work in the related open items.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main rejects the aborted caller before start, but the enqueued same-key item remains behind the predecessor and `pendingBySession` only decrements from `onSettle`.

## Solution Assessment

Yes, mostly. Moving pre-start cancellation into `SessionActorQueue` is the narrow maintainable layer for this bug, as long as the manager keeps its caller-facing ACP abort error and the contributor supplies active runtime proof.
