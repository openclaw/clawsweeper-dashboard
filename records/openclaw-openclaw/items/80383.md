## Summary

Keep open: the PR targets a real ACP queue accounting bug and the queue-level cancellation direction is sound, but the current head introduces an unbounded diagnostics map that should be fixed before merge.

## What This Changes

The PR adds AbortSignal-aware cancellation for queued ACP SessionActorQueue items, forwards withSessionActor signals into the queue, and adds focused queue regression tests.

## Best Possible Solution

Keep the queue-owned cancellation design, but remove the production diagnostics map or make its entries item-owned and cleaned when the actor key is idle.

## Reproduction Assessment

Yes, there is a high-confidence source reproduction path: current main enqueues and increments pending immediately, abort-before-start rejects the caller without notifying the queue, and the same-key predecessor can keep onSettle from running indefinitely.

## Solution Assessment

Mostly yes: SessionActorQueue is the right owner for cancelling an already-enqueued slot while preserving same-key serialization, but the diagnostics retention must be fixed before this is mergeable.

## Review Findings

Overall correctness: patch is incorrect

- **[P2] Avoid retaining settled diagnostics forever:** `src/acp/control-plane/session-actor-queue.ts:213-216`

In a long-running gateway this retains one diagnostic object for every session key ever queued; remove the production map or clear item-owned entries when the key is idle.
