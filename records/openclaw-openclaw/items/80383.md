## Summary

Keep this PR open: it addresses a real ACP abort-before-start queue liveness bug, but the current patch still introduces a production diagnostics map that is never cleaned up for idle actor keys.

## What This Changes

The branch adds AbortSignal-aware cancellation to SessionActorQueue, passes withSessionActor's signal into it, and adds focused queue regression tests.

## Best Possible Solution

Keep the queue-owned cancellation design, but remove or clean the diagnostic state so idle actor keys are not retained and newer diagnostics cannot be mutated by older queue items.

## Reproduction Assessment

Yes. Source inspection shows current main enqueues pending work immediately, only decrements on queued-task settlement, and lets withSessionActor reject an abort-before-start caller without notifying SessionActorQueue.

## Solution Assessment

Mostly yes. SessionActorQueue is the right boundary for cancelling an already-enqueued slot while preserving per-key serialization, but the diagnostics map must be removed, item-scoped, or cleaned before the patch is mergeable.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Clean up settled queue diagnostics:** `src/acp/control-plane/session-actor-queue.ts:213-216`
  - body: Every enqueue stores a diagnosticsBySession entry for the actor key, but onSettle only stamps settleTime and never removes the entry once the key is idle. In a long-running gateway this retains one object per session key, and older item hooks can also mutate the current key snapshot; please drop this map or clean/update it only when the settling item still owns the live diagnostic entry.
  - confidence: 0.84
