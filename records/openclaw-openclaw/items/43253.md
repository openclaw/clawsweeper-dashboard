## Summary

Keep open. Current main still has the skill-creator scaffold and guidance gap, and the PR is a narrow, valid fix path, but it needs small merge polish around changelog coverage and clearer wording before it should land.

## What This Changes

The PR adds YAML quote guidance to the skill-creator guide and changes the generated skill template so the placeholder `description` value is double-quoted.

## Best Possible Solution

Land the skill-creator docs/template fix with unambiguous plain-string guidance, a single-line `CHANGELOG.md` entry crediting @parubets, and preferably a focused scaffold frontmatter regression, while leaving runtime skill loading and validation semantics unchanged.

## Reproduction Assessment

Yes. Current main can be reproduced by parsing the generated `SKILL_TEMPLATE` frontmatter with PyYAML: the unquoted placeholder becomes a `list`, and `quick_validate.validate_skill()` requires `description` to be a string.

## Solution Assessment

Yes, with minor polish. Quoting the generated `description` and documenting the YAML footgun is the narrowest maintainable fix; the safer final form is clearer wording plus changelog and focused scaffold coverage rather than runtime loader changes.
