## Summary

Keep open. Current main still has the scaffold/frontmatter problem this PR addresses, and the PR remains a narrow authoring-surface fix; the remaining work is normal maintainer/author follow-up on wording clarity and focused regression coverage, not a cleanup close.

## What This Changes

The PR adds YAML quote guidance to the skill-creator guide and changes the generated `SKILL.md` template so the description TODO value is double-quoted.

## Best Possible Solution

Land the narrow skill-creator docs/template fix after rephrasing the guidance as plain string scalars, quoted when needed or always quoted, and adding focused coverage for the generated scaffold frontmatter.

## Reproduction Assessment

Yes. Current main can be checked by parsing the scaffold's frontmatter with PyYAML: the unquoted TODO description becomes a `list`, and the existing validator rejects non-string descriptions; the quoted PR form parses as `str`.

## Solution Assessment

Yes, with minor polish. Quoting the generated description and documenting the YAML footgun is the narrowest maintainable fix; the safer final form is clearer wording plus a scaffold/validator regression test rather than runtime loader changes.
