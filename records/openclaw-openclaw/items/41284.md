## Summary

Keep open: current main and the latest release still only parse `IDEProvisioningTeams`, while this PR addresses a specific Xcode preferences layout that remains unsupported. The related merged Xcode 16 signing work added fallback/guidance but did not supersede direct `IDEProvisioningTeamByIdentifier` parsing.

## What This Changes

The PR updates `scripts/ios-team-id.sh` to read Xcode teams from `IDEProvisioningTeamByIdentifier` via `defaults export` before preserving existing fallbacks, with related test fixture updates.

## Best Possible Solution

Land the focused script fix after standard maintainer validation, preserving the fallback chain and adding stronger parser-level regression coverage if practical.

## Reproduction Assessment

Yes, at source level: create a macOS/Xcode preferences state where teams exist only under `IDEProvisioningTeamByIdentifier`; current main only asks `plutil` for `IDEProvisioningTeams`, while the PR body provides before/after live output from that setup. I did not independently run this on macOS.

## Solution Assessment

Yes. Reading `defaults export` for both known Xcode team layouts before falling back to the existing `plutil`, legacy defaults, and provisioning-profile paths is the narrowest maintainable fix for this helper.
