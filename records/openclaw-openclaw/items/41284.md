## What This Changes

This PR updates `scripts/ios-team-id.sh` to parse Xcode team data from `IDEProvisioningTeamByIdentifier` via `defaults export` while preserving the existing `IDEProvisioningTeams` fallback, and updates the related script test fixture.

## Best Possible Solution

Land the focused identifier-key support after the regression case uses a real Python binary with mocked `defaults` and `plutil`, while preserving the current selection order and fallback chain.

## Reproduction Assessment

Yes. Current main only extracts `IDEProvisioningTeams`, while documented Xcode 16 setup uses `ios-configure-signing.sh`; a preferences layout containing only `IDEProvisioningTeamByIdentifier` follows the reported warning path by source inspection.

## Solution Assessment

No, not merge-ready as proposed. The runtime approach is narrow and appropriate, but the changed smoke test does not execute the new `defaults export` / `plistlib` parser; the safer path is to keep the runtime change and fix the harness coverage.
