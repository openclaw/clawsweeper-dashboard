## Summary

Keep open: current main and the latest release still miss direct `IDEProvisioningTeamByIdentifier` parsing, so the PR remains useful, but its regression test does not exercise the new inline plist parser.

## What This Changes

The PR updates `scripts/ios-team-id.sh` to read Xcode team data from `defaults export` for both `IDEProvisioningTeamByIdentifier` and `IDEProvisioningTeams`, with matching script-test fixture changes.

## Best Possible Solution

Land the focused parser change after the smoke test runs the real inline plist parser, while preserving the existing fallback order and team-selection semantics.

## Reproduction Assessment

Yes, at source level: current main only parses `IDEProvisioningTeams`, so a preferences file with teams only under `IDEProvisioningTeamByIdentifier` can skip the Xcode preferences path and trigger the caller warning/no-op path.

## Solution Assessment

Mostly yes: the production change is a narrow script-level fix with the fallback chain preserved, but the test should exercise the real inline `plistlib` parser instead of a canned fake Python stub.
