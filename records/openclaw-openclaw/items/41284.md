## Summary

Keep open: this PR targets a real iOS signing detection gap, and the production approach looks narrow, but the regression test still bypasses the new plist parser so the patch is not ready to merge.

## What This Changes

The PR changes `scripts/ios-team-id.sh` to parse Xcode team data from `defaults export` for both `IDEProvisioningTeamByIdentifier` and `IDEProvisioningTeams`, with matching iOS team-id test fixture updates.

## Best Possible Solution

Land the focused parser change after the regression test executes the real Python plist parser and the branch is refreshed against current main.

## Reproduction Assessment

Yes, at source level: current main only parses `IDEProvisioningTeams`, so a preferences file with teams only under `IDEProvisioningTeamByIdentifier` skips the primary Xcode preferences path and can trigger the signing warning path. I did not run a live macOS/Xcode repro in this read-only Linux checkout, but the PR body includes after-fix macOS output.

## Solution Assessment

Mostly yes: the production change is a narrow script-level fix and preserves the existing fallback chain. The test change is not yet the best solution because the smoke case should run the real inline plist parser instead of a canned fake Python stub.
