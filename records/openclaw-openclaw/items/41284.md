The PR changes `scripts/ios-team-id.sh` to read Xcode teams from `IDEProvisioningTeamByIdentifier` as well as `IDEProvisioningTeams`, and updates the script test fixture.

## Best Possible Solution

Land a refreshed narrow fix that reads both Xcode preference layouts, preserves existing fallback and team-selection behavior, and adds a regression test that executes the real parser path.

## Reproduction Assessment

Yes. At source level, current main only reads `IDEProvisioningTeams`, while the documented iOS setup calls the signing wrapper that warns and exits unchanged when the helper returns no team; the PR body also includes macOS terminal output for that path.

## Solution Assessment

Mostly yes. Parsing both Xcode preference keys in `scripts/ios-team-id.sh` is the narrow maintainable fix, but the proposed regression test needs to run the real Python plist parser instead of a fake stdout script.
