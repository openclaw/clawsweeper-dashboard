## Summary

The PR updates `scripts/ios-team-id.sh` to parse Xcode team data from `IDEProvisioningTeamByIdentifier` via `defaults export` while preserving the legacy `plutil` fallback, and adjusts the iOS team-id test fixture.

## Best Possible Solution

Land the focused signing-helper support after the identifier-key regression case executes the real inline Python parser, while preserving the existing fallback and team-selection order.

## Reproduction Assessment

Yes, by source inspection rather than a live macOS run. Current main only extracts `IDEProvisioningTeams`, so preferences containing only `IDEProvisioningTeamByIdentifier` can leave no Xcode preference candidates and reach the documented signing warning path.

## Solution Assessment
