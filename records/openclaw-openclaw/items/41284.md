## Summary

Keep open: current main still only reads `IDEProvisioningTeams` from the Xcode plist, while this PR adds the missing `IDEProvisioningTeamByIdentifier` path and preserves the existing fallbacks. I found no blocking correctness or security finding in the patch, and the PR body now contains real macOS shell output for the before/after behavior, but the branch is currently conflicting with main.

## What This Changes

The PR updates `scripts/ios-team-id.sh` to parse Xcode team IDs from `IDEProvisioningTeamByIdentifier` as well as `IDEProvisioningTeams`, with focused test changes for the new layout.

## Best Possible Solution

Land a rebased version of this targeted parser change, preserving the existing team-selection and fallback order and keeping focused coverage for the identifier-key plist layout.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: on current main, a macOS Xcode preferences file that only stores teams under `IDEProvisioningTeamByIdentifier` will not be read by `scripts/ios-team-id.sh`, and the PR body includes before/after live shell output for that setup.

## Solution Assessment

Yes. Parsing `defaults export` with `plistlib` while retaining the old `plutil`, provisioning-profile, and keychain fallbacks is the narrowest maintainable fix for the reported layout; improving test coverage to execute the real inline Python parser would be useful but is not a blocking correctness issue by itself.
