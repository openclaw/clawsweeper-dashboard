## Summary

Keep open: current main still lacks direct `IDEProvisioningTeamByIdentifier` parsing, so the PR addresses a real iOS signing-helper gap, but the current patch still has a blocking test issue because its smoke case bypasses the new inline plist parser.

## What This Changes

The PR updates `scripts/ios-team-id.sh` to read Xcode teams from `IDEProvisioningTeamByIdentifier` as well as `IDEProvisioningTeams`, and adjusts the script tests.

## Best Possible Solution

Land a refreshed narrow fix that reads both Xcode preference layouts, preserves the existing fallback and team-selection order, and covers the new parser path with a regression test that actually executes it.

## Reproduction Assessment

Yes, at source level: current main only extracts `IDEProvisioningTeams`, while the documented iOS flow runs the signing wrapper that warns and exits unchanged when the helper returns no team. I did not run a live macOS/Xcode repro in this read-only Linux review.

## Solution Assessment

Mostly yes: parsing both Xcode preference keys inside `ios-team-id.sh` is the narrow maintainable fix. The proposed patch should first make its regression test execute the real inline plist parser and refresh against current main.
