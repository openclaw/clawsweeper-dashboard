## Summary

Keep open: current main still routes Telegram partial-preview tool progress through exec-derived item progress text, so the reported noisy DM behavior remains source-reproducible and narrow enough for a focused fix.

## What This Changes

The issue asks Telegram partial streaming to stop surfacing excessive transient tool/exec progress, command text, and command output bubbles in DMs before the final answer.

## Best Possible Solution

Filter or redact exec command/output details from Telegram preview progress while preserving final answers, compact non-sensitive status previews, and the existing streaming.preview.toolProgress opt-out.

## Reproduction Assessment

Yes by source inspection: partial-mode Telegram previews consume item progressText, and exec updates can populate progressText from command output. I did not live-test Telegram in this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is to sanitize or omit exec command/output details in the Telegram preview progress path, not disable partial streaming globally or add another config knob.
