## Summary

Keep open: current main still intentionally streams Telegram tool-progress lines in `partial` mode by default, while the exact excessive live DM bubble pattern was not reproduced here. The existing quiet path is documented, but changing the default or tightening what counts as safe progress needs maintainer UX judgment.

## What This Changes

The issue asks Telegram `streaming.mode: "partial"` to avoid exposing excessive transient tool/exec progress, command text, and command output before the final DM reply.

## Best Possible Solution

Decide whether Telegram `partial` should remain progress-visible by default; if quiet DMs are preferred, preserve answer previews while defaulting or strongly steering Telegram DM users to `streaming.preview.toolProgress: false`, with regression coverage for no raw exec output and no standalone chatter.

## Reproduction Assessment

Unclear for the exact live issue: source and tests show `partial` mode streams tool progress by default, but I did not reproduce the reported 10-20 transient Telegram DM bubbles or raw output exposure on current main.

## Solution Assessment

Unclear: `streaming.mode: "off"` is a valid workaround, but the narrower maintainable direction is likely a Telegram partial-mode policy decision or formatter tightening rather than disabling all streaming previews.
