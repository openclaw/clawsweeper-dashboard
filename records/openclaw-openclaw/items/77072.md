## Summary

Keep open: current main still allows Telegram preview streaming to include exec-derived progress text, which matches the reported noisy DM behavior and is broader than the documented short-status preview contract.

## What This Changes

The issue reports that Telegram partial streaming shows transient tool and exec progress bubbles in direct messages before the final response.

## Best Possible Solution

Filter or redact command result details from Telegram preview progress while preserving final answers, compact non-sensitive status previews, and the existing preview.toolProgress opt-out.

## Reproduction Assessment

Yes by source inspection: Telegram partial preview updates consume item progressText/summary, and exec tool updates can populate those fields from command output. I did not run a live Telegram DM session in this read-only review.

## Solution Assessment

Yes: the narrow maintainable fix is to sanitize or omit exec command/output details in the Telegram preview progress path, rather than turning off preview streaming globally or adding another config knob.
