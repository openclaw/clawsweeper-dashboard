## Summary

Keep open. Current main still rejects the reported nested Discord channel `agentId` shape, and the only current repair path strips unknown keys generically, which can discard the user’s intended per-channel route instead of migrating it or giving Discord-specific guidance.

## What This Changes

The issue requests support, repair, or actionable guidance for Discord channel-level `agentId` entries that currently invalidate gateway config reloads.

## Best Possible Solution

Keep channel-local `agentId` out of the Discord runtime schema unless maintainers explicitly choose that alias, and add a Discord-owned doctor compatibility path that safely migrates it to top-level route bindings or emits a targeted `bindings[]` migration hint.
