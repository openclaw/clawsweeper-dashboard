  - body: When `scheduledResetTriggered` is true, this emits `command:new` even though no user command was sent, which can run normal command:new hooks and before_reset automations on an ordinary post-expiry message while bypassing the manual sender-authorization guard.

  - body: The new loop drops a standalone `[message_id: ...]` line after inbound metadata even when that line is the first real user-authored line, which regresses transcript fidelity for lookalike user text.
  - confidence: 0.86
- **[P2] Do not drop slash-prefixed user content after stripping:** `src/hooks/bundled/session-memory/handler.ts:85`
  - body: The PR strips metadata before the existing `!text.startsWith("/")` filter, so legitimate channel messages whose real content starts with `/` are omitted from session memory after metadata removal.
  - confidence: 0.88
- **[P2] Handle timestamp-prefixed inbound metadata:** `src/hooks/bundled/session-memory/handler.ts:85`
  - body: `stripLeadingInboundMetadata` only strips when the first non-empty line is a metadata sentinel, so BodyForAgent text with an injected timestamp before the metadata block can still persist the reported metadata pollution.
  - confidence: 0.82
