## Summary

Keep open: the session-memory sanitizer goal is valid on current main, but this PR should not merge as-is because it mixes that fix with scheduled reset hook dispatch and introduces cleanup regressions. The lazy reset hook behavior is already tracked separately by #61675.

## What This Changes

The PR sanitizes session-memory user turns with inbound metadata stripping, extends message-id cleanup, adds scheduled reset hook signaling, and adds regression tests around session-memory and reset-hook fallback paths.

## Best Possible Solution

Land a narrow sanitizer-only patch in the current session-memory transcript extraction path, reuse established inbound/envelope/message-id cleanup semantics, and leave lazy daily/idle reset hook dispatch to #61675.

## Reproduction Assessment

Yes, at source level. Current main reads user JSONL text and writes it to memory without the sanitizer, so Feishu-style inbound metadata and message-id hint lines can persist in session-memory artifacts.

## Solution Assessment

No. Sanitizing session-memory user text is the right direction, but this PR is not the best form because it bundles scheduled reset hook dispatch and should instead mirror the existing display cleanup boundary in the transcript extraction path.

## Review Findings

Full review comments:

- **[P2] Keep scheduled resets out of /new hook dispatch:** `src/auto-reply/reply/get-reply.ts:356-368`
  - body: When `scheduledResetTriggered` is true, this emits `command:new` hooks for an ordinary post-stale message and bypasses the authorized-sender guard used by the manual path. Keep lazy reset hook work in #61675 instead of bundling it with the sanitizer fix.
  - confidence: 0.88
- **[P2] Filter slash commands before sanitizer cleanup:** `src/hooks/bundled/session-memory/handler.ts:83-84`
  - body: Running the slash-command filter after metadata stripping drops legitimate channel messages whose actual content starts with `/`, such as `/var/log`, even though they were previously retained. Apply command filtering to the raw command turn or otherwise distinguish commands from user text.
  - confidence: 0.86
- **[P2] Use shared message-id hint cleanup:** `src/auto-reply/reply/strip-inbound-meta.ts:240-245`
  - body: Extending `stripLeadingInboundMetadata()` with a local message-id loop duplicates policy already covered by `stripMessageIdHints()` and only works in the sentinel-block shape. Reuse the shared envelope/message-id cleanup semantics so body-only Feishu hints and existing UI behavior stay aligned.
  - confidence: 0.78
- **[P2] Strip timestamp envelopes before storing memory:** `src/hooks/bundled/session-memory/handler.ts:83-84`
  - body: The new memory cleanup only calls `stripLeadingInboundMetadata()`, so timestamp prefixes that become leading text after metadata removal can still be written to memory. Use the broader existing sanitizer path that removes timestamp/envelope metadata as well as inbound blocks.
  - confidence: 0.76
