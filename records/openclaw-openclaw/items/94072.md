## Best Possible Solution

Land this maintainer-selected narrow fix, then let the linked bug close and use this PR as the canonical path for closing the remaining duplicate one-line PRs.

## Solution Assessment

Is this the best way to solve the issue? Yes: this updates the existing payload-builder reply-evidence gate that already feeds `resolveToolErrorWarningPolicy`, without adding Discord-specific logic or duplicating the message-tool delivery predicate.
