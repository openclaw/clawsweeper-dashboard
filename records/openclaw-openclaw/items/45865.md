## Summary

Keep open. Current main still lacks an injected `api.runtime.llm` or `api.runtime.llm.complete` Plugin SDK runtime helper, and the existing embedded-agent, simple-completion, and optional `llm-task` paths do not provide the requested single-shot managed plugin runtime surface. The open #64294 PR is the active implementation candidate, so this issue should remain open until that PR lands or maintainers choose a different API shape.

## What This Changes

Request to add a Plugin SDK `api.runtime.llm()` inference helper so plugins can call OpenClaw's managed model stack instead of provider APIs directly.

## Best Possible Solution

Review #64294 or a replacement design and land a narrow, backwards-compatible SDK/runtime implementation with types, runtime wiring, test mocks, docs, and context-engine support only if maintainers agree that context engines should share the same surface.

## Reproduction Assessment

Not applicable as a feature request rather than a runtime bug. The missing capability is verifiable with high confidence by inspecting the `PluginRuntime` type, runtime builder, public docs, and targeted source search on current main.

## Solution Assessment

Unclear pending maintainer API review. A host-mediated, single-shot completion helper with no tool execution looks like the maintainable direction, but naming plus budget, cost, and permission semantics should be settled before merge.
