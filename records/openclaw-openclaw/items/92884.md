Keep open: the reporter supplied the missing redacted fixture, and current source/docs still support a focused bug in the alignment between `config schema` metadata and `config validate` for same-channel replacement plugins.

## What This Changes

Make `openclaw config validate` accept plugin-owned `channelConfigs.<id>.schema` extensions for replacement channel plugins, matching the schema surfaced by `openclaw config schema`.

## Best Possible Solution

Align `config validate` with the same active channel metadata source that `config schema` exposes for replacement channel ids, with a Slack replacement regression and no global relaxation of bundled schemas.

## Reproduction Assessment

No live current-main reproduction was run in this read-only review. The reporter supplied a redacted shipped-release fixture, and current source gives a credible reproduction path around replacement channel metadata selection for `channels.slack`.

## Solution Assessment

Yes, the best fix is a narrow validation/schema metadata alignment repair. Adding a new validation mode or relaxing Slack's bundled schema globally would create a second source of truth.
