## Summary

Keep open: current main still lacks the same-origin `preferOver` ownership guard for replacement channel schemas, and https://github.com/openclaw/openclaw/pull/92902 is the open candidate fix rather than a reason to close this issue before merge.

## What This Changes

Validate plugin-owned channel config extensions with the active plugin-provided channel schema instead of rejecting replacement-plugin fields as additional properties.

## Best Possible Solution

Review and land or explicitly reject https://github.com/openclaw/openclaw/pull/92902; if rejected, keep the end state of making `config validate` and `config schema` share deterministic active channel schema ownership.

## Reproduction Assessment

Yes, at the source level: the reporter supplied a redacted config and manifest fixture, and current main lacks the equal-origin `preferOver` schema ownership rule needed for that path. I did not run a fresh live command in this read-only review.

## Solution Assessment

Yes; the narrow fix should live in the shared channel metadata collector used by both schema generation and validation. A Slack-only relaxation or separate validation mode would create a second source of truth.
