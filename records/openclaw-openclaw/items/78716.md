## Summary

Keep open: the linked plugin dispatch bug is real, but this external PR still needs real cached-plugin runtime proof and the latest diff still invokes unrelated named factories before falling back to unnamed candidates.

## What This Changes

The PR changes `src/plugins/tools.ts` so cached descriptor execution iterates plugin tool factories and matches by the returned runtime tool name before executing.

## Best Possible Solution

Dispatch cached descriptors by actual runtime tool name while filtering explicit-name candidates before invoking factories, add regression coverage and a changelog entry, then require redacted real runtime proof before merge.

## Reproduction Assessment

Yes. Source inspection on current main shows the cached wrapper can select the first unnamed factory through shared plugin-level `declaredNames`, and https://github.com/openclaw/openclaw/issues/78671 includes sanitized live `plugin tool runtime missing` failures for that shape.

## Solution Assessment

No. Matching by returned runtime tool name is the right direction, but this PR is not the best merge state until it skips non-matching named factories, adds regression/changelog coverage, and supplies real after-fix plugin execution proof.
