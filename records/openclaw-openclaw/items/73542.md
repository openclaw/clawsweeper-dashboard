## Summary

Keep open: current main still lacks a hook-directory pre-send dispatch, and open PR #73570 references this issue with closing syntax but has not merged. Maintainers still need to settle the internal event name, ordering, and whether hook-directory pre-send handlers are observe-only or intercepting.

## What This Changes

Expose or bridge the outbound `message_sending` hook event to user-defined hook-directory handlers before channel delivery.

## Best Possible Solution

Review and land #73570 or a narrow replacement that emits and documents a pre-delivery internal hook from the shared outbound path, with tests for hook-directory registration, event-name compatibility, ordering, and guardrail semantics.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: hook-directory events are registered verbatim, internal dispatch only reaches colon-style keys, and current main never emits an internal pre-send message event in the shared outbound delivery path.

## Solution Assessment

Unclear. Bridging near `applyMessageSendingHook` is the likely seam, but maintainers should decide whether the user-facing event is `message_sending`, `message:sending`, or both, and whether hook-directory handlers should be observe-only or able to modify/cancel delivery.
