## Summary

Keep open. Current main still has the reported gap: the shared outbound delivery path runs the typed plugin `message_sending` hook before delivery, while user hook directories register internal event keys and there is no matching internal pre-send dispatch. Open PR #73570 already references this issue with closing syntax, so the issue should stay open until that PR lands, is adapted, or is replaced.

## What This Changes

User-defined hook-directory handlers registered for `message_sending` should fire before outbound messages are delivered, matching the existing plugin pre-send hook behavior.

## Best Possible Solution

Land or replace PR #73570 with a canonical bridge in the shared outbound delivery path that fires before channel delivery, clearly defines `message_sending` versus `message:sending` compatibility, preserves plugin modification/cancel semantics, and adds focused tests plus docs for the internal hook contract.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: hook-directory metadata registers literal event keys, internal dispatch reaches only `message` and `message:<action>` listeners, and the current pre-send outbound path only invokes the typed plugin `runMessageSending` hook.

## Solution Assessment

Partly. Bridging the pre-send dispatch in `deliverOutboundPayloads` is the narrow maintainable direction, but the final patch should explicitly cover event-name compatibility and pre-delivery ordering rather than adding only an undocumented notification.
