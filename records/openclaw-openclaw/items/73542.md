## Summary

Keep open: current main still has plugin-only pre-send dispatch for `message_sending`, while the internal directory-hook path only bridges `message:sent`; open PR #73570 references this issue with closing syntax but is not merged.

## What This Changes

The issue requests that directory-based user hooks registered for the outbound pre-send event fire before messages are delivered, matching the existing plugin hook surface.

## Best Possible Solution

Land PR #73570 after adapting it as needed, or merge a narrow replacement that emits a documented pre-delivery internal hook with clear ordering, spelling compatibility, and guardrail semantics.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: directory hook metadata registers event names verbatim, current internal dispatch only reaches `message` and `message:<action>`, and the shared outbound pre-send path never emits a `message/sending` internal hook on current main.

## Solution Assessment

Unclear. Bridging from `deliverOutboundPayloads` is the right location, but the final solution should explicitly handle event-name compatibility and avoid implying that directory hooks can modify or cancel messages unless that capability is implemented.
