## Summary

Keep open: current main still runs the pre-send `message_sending` path only through the plugin hook runner, and open PR #73570 references this issue with closing syntax but has not merged.

## What This Changes

User-defined hook-directory handlers registered for `message_sending` should fire before outbound delivery, matching the plugin pre-send hook path.

## Best Possible Solution

Review and land PR #73570 or a narrow replacement that emits a pre-delivery internal hook from the shared outbound delivery path with documented ordering, event-key compatibility, and guardrail semantics.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: directory hooks register event keys verbatim, internal dispatch only reaches `message` and `message:<action>` listeners, and current main never creates a `message/sending` internal event in the pre-delivery path.

## Solution Assessment

Unclear. Bridging the event near `applyMessageSendingHook` is the likely narrow location, but the merge-ready fix should settle `message_sending` versus `message:sending` compatibility and whether user-defined hooks are observe-only or can intercept delivery.
