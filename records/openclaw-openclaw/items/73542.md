## Summary

Keep open: current main still runs the pre-send path only through plugin `message_sending` hooks and never emits an internal pre-send message hook for directory hooks; PR #73570 is the active closing PR and should be reviewed before any cleanup close.

## What This Changes

User-defined hooks from hook directories should fire before outbound messages when registered for the pre-send `message_sending` event.

## Best Possible Solution

Review and adapt PR #73570, or an equivalent narrow replacement, so the shared outbound path emits a documented pre-delivery internal hook with clear event naming, ordering, and interception semantics.

## Reproduction Assessment

Yes. The source-level path is high confidence: directory hook metadata is registered verbatim, internal dispatch only targets colon-style keys, and the current pre-send delivery path never calls `triggerInternalHook` for a sending event.

## Solution Assessment

Unclear. Bridging the outbound pre-send point is the right narrow location, but the maintainable fix should settle underscore/colon compatibility and whether internal hooks can modify or cancel delivery.
