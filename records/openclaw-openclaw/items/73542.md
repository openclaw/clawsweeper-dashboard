## Summary

Keep open: current main still lacks an internal pre-send hook dispatch, and open PR #73570 references this issue with closing syntax but has not merged.

## What This Changes

User-defined hook-directory handlers registered for `message_sending` should fire before outbound channel delivery, matching the plugin pre-send hook and existing post-send internal-hook bridge.

## Best Possible Solution

Review and adapt PR #73570 or a narrow replacement so the shared outbound delivery path emits a documented pre-delivery internal hook with clear ordering, naming compatibility, and guardrail semantics.

## Reproduction Assessment

Yes. The source-level path is high confidence: hook metadata registers event keys verbatim, internal dispatch only reaches `message` and `message:<action>` listeners, and current main never creates a `message/sending` internal event in the pre-send delivery path.

## Solution Assessment

Unclear. Bridging the event in `deliverOutboundPayloads` is the right location, but the merged fix should settle event-name compatibility and whether internal hooks are observe-only or can actually intercept delivery.
