## Summary

Keep open: current `main` and the latest release still do not implement the requested standalone/internal pre-send hook behavior, and the direct fix PR was closed unmerged. The remaining question is not whether code plugins have `message_sending`, but whether OpenClaw should define and support a matching `HOOK.md`/internal pre-send contract.

## What This Changes

The issue asks for user-defined `HOOK.md` hooks registered for `message_sending` to fire before outbound channel delivery.

## Best Possible Solution

Define the standalone pre-send hook contract explicitly, then either implement and document it with correct key/timing semantics or clarify that rewrite/cancel guardrails belong in code plugins.

## Reproduction Assessment

Yes at source level: current `main` runs plugin `message_sending`, registers directory hook events verbatim, and internal dispatch only matches general type or colon-style `type:action` keys. A live hook-directory run was not performed in this read-only review.

## Solution Assessment

Unclear. A simple bridge is only a plausible partial fix; the best fix needs a maintainer decision on the standalone hook key, whether it is observation-only, and when it should fire relative to plugin rewrite/cancel decisions.
