## Summary

Keep open: current main still lacks a user-hook pre-send dispatch, and open PR #73570 references this issue with closing syntax but has not merged.

## What This Changes

User-defined hook-directory handlers registered for outbound `message_sending` should fire before channel delivery like plugin hooks do.

## Best Possible Solution

Land #73570 or a narrow replacement that emits a documented pre-delivery internal hook from the shared outbound delivery path with clear ordering, naming compatibility, and guardrail semantics.

## Reproduction Assessment

Yes. The source-level path is high confidence: hook metadata registers event keys verbatim, internal dispatch only reaches `message` and `message:<action>` listeners, and the pre-send delivery path currently never creates a `message/sending` internal event.

## Solution Assessment

Unclear. Bridging the event in `deliverOutboundPayloads` is the right surface, but the merge-ready fix should settle listener-key compatibility and whether hook-directory handlers are observe-only or can actually intercept delivery.
