## Summary

Keep open: current main still lacks an internal pre-send hook dispatch, and #73570 is open/unmerged with closing syntax, so this issue should remain tracked until a fix lands.

## What This Changes

User-defined hook-directory handlers registered for outbound pre-send events should fire before delivery, matching the plugin `message_sending` path.

## Best Possible Solution

Review and adapt #73570, or a replacement, so the shared outbound path emits a documented pre-delivery internal hook with clear naming, ordering, and modification/cancel semantics.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: hook metadata registers event keys verbatim, internal dispatch reaches only `message` and `message:<action>` listeners, and current main never creates a pre-send internal message event in `deliverOutboundPayloads`.

## Solution Assessment

Unclear. Bridging the pre-send dispatch in `deliverOutboundPayloads` is the right location, but the accepted fix should settle key compatibility and observable semantics for hook-directory users.
