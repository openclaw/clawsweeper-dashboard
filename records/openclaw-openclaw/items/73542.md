## Summary

Keep open: current main still lacks an internal pre-send hook dispatch, and open PR #73570 references this issue with closing syntax but has not merged.

## What This Changes

User-defined hook-directory handlers listening for `message_sending` are expected to fire before outbound delivery, matching the plugin pre-send hook surface.

## Best Possible Solution

Land #73570 or a narrow replacement that emits a documented pre-delivery internal hook from the shared outbound delivery path with clear ordering, naming compatibility, and guardrail semantics.

## Reproduction Assessment

Yes. The source-level path is high confidence: hook metadata registers event keys verbatim, internal dispatch only reaches `message` and `message:<action>` listeners, and current main never creates a `message/sending` internal event in the pre-send delivery path.

## Solution Assessment

Unclear. Bridging the event near `applyMessageSendingHook` is the right location, but the merge-ready fix should settle `message_sending` versus `message:sending` compatibility and whether hook-directory handlers are observe-only or can actually intercept delivery.
