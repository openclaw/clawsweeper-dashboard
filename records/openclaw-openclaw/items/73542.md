## Summary

Keep open: current main still lacks the requested hook-directory pre-send dispatch, and open PR #73570 references this issue with closing syntax but has not merged. The remaining maintainer decision is the exact internal event name, ordering, and whether hook-directory handlers are observe-only or intercepting.

## What This Changes

Request that hook-directory handlers registered for `message_sending` fire before outbound delivery, matching plugin-hook coverage and the existing `message_sent` dual dispatch pattern.

## Best Possible Solution

Review and land #73570 or a narrow replacement that emits and documents a pre-delivery internal hook from the shared outbound delivery path with clear compatibility, ordering, and guardrail semantics.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: hook-directory events are registered verbatim, internal dispatch only reaches colon-style keys, and current main never emits an internal pre-send message event in the shared outbound delivery path.

## Solution Assessment

Unclear. Bridging near `applyMessageSendingHook` is the likely seam, but the merge-ready solution needs an explicit maintainer decision on `message_sending` versus `message:sending` compatibility and observe-only versus intercepting semantics.
