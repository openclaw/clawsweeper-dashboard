## Summary

Keep open: current main and latest release still run only the typed plugin `message_sending` hook before outbound delivery, while internal/user hook dispatch remains `message:sent` only. The open candidate fix at https://github.com/openclaw/openclaw/pull/94342 should be reviewed instead of closing this issue early.

## What This Changes

The issue requests a user-defined/internal pre-send hook bridge so hooks configured as `message_sending` can observe outbound delivery before messages are sent.

## Best Possible Solution

Land one narrow, reviewed hook bridge that defines the internal event key and ordering relative to typed plugin rewrite/cancel decisions, while keeping broader channel-specific plugin hook bypasses tracked separately.

## Reproduction Assessment

Yes, at source level: current main registers directory hook event keys verbatim, dispatches internal hooks only by type or colon key, and has no internal pre-send bridge in outbound delivery. I did not run a live Gateway hook-directory reproduction in this read-only review.

## Solution Assessment

No, the issue is not solved yet. The right area is shared outbound/internal hook delivery, but the final fix must define whether this is observation-only or guardrail-capable before it is the best maintainable solution.
