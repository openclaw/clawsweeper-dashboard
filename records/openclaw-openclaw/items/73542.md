## Summary

Keep open: current main still lacks an internal pre-send dispatch path, and the requested file-based hook behavior needs maintainer direction because internal hooks use `message:...` event keys while mutable/cancel-capable `message_sending` is currently a plugin hook contract.

## What This Changes

The issue asks for user-defined hook directories to receive a pre-send outbound message hook instead of only plugin `message_sending` handlers seeing that lifecycle point.

## Best Possible Solution

Maintainers should decide the public contract: either document that mutable pre-send guardrails require plugin hooks, or add a supported file-hook `message:sending` event with explicit notification versus mutation semantics, context shape, ordering, and regression tests.

## Reproduction Assessment

Yes at source level: a file hook registered for the reported pre-send event has no current internal dispatch path in `deliverOutboundPayloads`; only plugin `message_sending` and internal `message:sent` are wired. I did not run a live gateway reproduction in this read-only pass.

## Solution Assessment

Unclear. Adding `triggerInternalHook` near `applyMessageSendingHook` is the likely notification seam, but the requested cancellation/rewrite behavior and underscore-versus-colon event naming need maintainer contract direction before implementation.
