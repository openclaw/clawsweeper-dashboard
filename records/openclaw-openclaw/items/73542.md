## Summary

Keep open. Current main still has the reported gap: the shared outbound delivery path runs the typed plugin `message_sending` hook before channel delivery, but it does not bridge that pre-send point into the internal hook registry for hook-directory/user-defined hooks. Open PR #73570 references this issue with closing syntax, so the issue should stay open until that PR lands, is adapted, or is replaced.

## What This Changes

The issue requests that user-defined hook-directory handlers registered for the pre-send outbound message hook fire consistently with the typed plugin `message_sending` hook before delivery.

## Best Possible Solution

Land or replace PR #73570 with a canonical pre-delivery bridge in `src/infra/outbound/deliver.ts` that fires user-defined hooks before channel delivery, defines `message_sending` versus `message:sending` compatibility, preserves typed plugin modification/cancel semantics, and adds focused tests plus docs for the internal hook contract.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: user hook metadata registers literal event keys, internal dispatch only reaches `message` and `message:<action>`, and the current pre-send outbound path only invokes the typed plugin `message_sending` hook.

## Solution Assessment

Yes in direction, but not complete as a close decision. Bridging the shared pre-delivery path is the narrow maintainable solution; the patch should also cover event-name compatibility and document whether internal pre-send hooks are notification-only or can block/modify delivery.
