## Summary

Keep open: current main still only runs the plugin `message_sending` hook before shared outbound delivery, and the open closing PR has not merged.

## What This Changes

User-defined hook-directory handlers that declare `message_sending` should fire before outbound messages are delivered, matching the plugin pre-send hook surface.

## Best Possible Solution

Review and adapt PR #73570 or a replacement so the shared outbound path emits a documented pre-delivery internal hook with clear naming, ordering, and notification versus mutation semantics.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: hook metadata registers event keys verbatim, internal dispatch only reaches `message` and `message:<action>` listeners, and current main never creates a `message/sending` internal event in the pre-send delivery path.

## Solution Assessment

Unclear. Bridging the pre-send dispatch in `deliverOutboundPayloads` is the right location, but the final solution should explicitly settle `message_sending` versus `message:sending` compatibility and whether user hooks can observe only or also intercept delivery.
