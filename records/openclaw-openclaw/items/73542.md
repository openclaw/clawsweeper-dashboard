## What This Changes

The issue asks for hook-directory `message_sending` handlers to fire before outbound channel delivery, matching the plugin hook path.

## Best Possible Solution

Define and document the pre-delivery internal hook contract, then emit it in shared outbound delivery only for effective deliverable payloads with compatibility tests for the intended standalone hook key.

Yes, at source level. Current main runs plugin `message_sending` before delivery, while hook-directory events are registered verbatim and internal dispatch only matches type or colon-style keys.
