## Summary

Keep open: current main and v2026.6.10 still run the typed plugin `message_sending` hook before delivery but do not bridge a pre-send event into the internal/user hook registry; the open candidate fix at https://github.com/openclaw/openclaw/pull/94342 should be reviewed rather than closing this issue early.

## What This Changes

The issue requests that user-defined workspace hooks registered for `message_sending` fire before outbound message delivery, matching the existing typed plugin hook path.
