## Summary

Keep open: current main still dispatches pre-send outbound hooks only through the plugin hook runner, while the hook-directory internal path has no equivalent pre-delivery dispatch and the linked closing PR is still open.

## What This Changes

User-defined hook-directory handlers registered for outbound `message_sending` should fire before delivery like the plugin hook path.

## Best Possible Solution

Land or adapt #73570, or a replacement, so the shared outbound path emits a documented pre-delivery internal hook with clear naming, ordering, and interception semantics.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes. The source path shows hook-directory events are registered verbatim, internal dispatch only reaches colon-keyed message actions, and the current pre-send outbound path never creates a `message/sending` internal event.

## Solution Assessment

Is this the best way to solve the issue? Unclear. Bridging the outbound pre-send path is the right location, but the maintainable fix should settle event-name compatibility and whether hook-directory handlers are notify-only or can modify/cancel delivery.
