## Summary

Keep open: current main still lacks a hook-directory pre-send dispatch, and open PR #73570 references this issue with closing syntax but has not merged. The remaining maintainer decision is the internal event key compatibility and whether directory hooks are observe-only or can participate in interception semantics.

## What This Changes

The issue asks for hook-directory handlers registered for `message_sending` to fire before outbound delivery, matching the plugin `message_sending` hook coverage.

## Best Possible Solution

Review and land #73570 or a narrow replacement that emits, tests, and documents a pre-delivery internal hook from the shared outbound delivery path with explicit event-key, ordering, and guardrail semantics.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: a hook-directory `message_sending` event is registered verbatim and can show ready, but current outbound delivery only invokes plugin `message_sending` and never emits an internal pre-send message event.

## Solution Assessment

Unclear. Bridging near `applyMessageSendingHook` is the likely seam, but the merge-ready solution needs a maintainer decision on `message_sending` versus `message:sending` compatibility and observe-only versus intercepting behavior.
