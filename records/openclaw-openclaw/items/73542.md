## Summary

Keep open: current main still lacks a hook-directory pre-send dispatch, and https://github.com/openclaw/openclaw/pull/73570 references this issue with closing syntax but remains open and reviewed as not merge-ready.

## What This Changes

User-defined hook-directory handlers registered for `message_sending` should fire before outbound delivery, matching the plugin hook lifecycle.

## Best Possible Solution

Define and document a pre-delivery internal hook contract, then update shared outbound delivery and regression tests so hook-directory handlers observe the effective deliverable payload before channel send.

## Reproduction Assessment

Yes. Source inspection on current main shows plugin `message_sending` hooks run in the shared outbound delivery path, while hook-directory dispatch has no matching pre-send internal event and only matches type or colon-style keys.

## Solution Assessment

Unclear. Bridging near `applyMessageSendingHook` is the right area, but the merge-ready solution needs maintainer decisions on event key compatibility, ordering, and whether hook-directory pre-send handlers are observe-only or intercepting.
