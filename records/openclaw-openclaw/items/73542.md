## Summary

Keep open: current main still lacks an internal pre-send hook dispatch, and open PR #73570 references this issue with closing syntax but has not merged.

## What This Changes

User-defined hook-directory handlers registered for the outbound pre-send message event are listed as ready but never invoked before delivery.

## Best Possible Solution

Review and adapt PR #73570 or a replacement so the shared outbound path emits a documented pre-delivery internal hook with clear naming compatibility, ordering after plugin modifications, and cancellation semantics.

## Reproduction Assessment

Yes. The source-level reproduction is high confidence: register a hook-directory event for `message_sending`, send outbound content through `deliverOutboundPayloads`, and current main never emits an internal pre-send message event.

## Solution Assessment

Unclear. Bridging `deliverOutboundPayloads` is the right location, but the final fix should explicitly settle `message_sending` versus `message:sending` compatibility and whether internal hooks observe only messages that will actually be delivered.
