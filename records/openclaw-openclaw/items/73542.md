## Summary

Keep open: current main still lacks an internal pre-send hook dispatch, and open PR #73570 already references this issue with closing syntax but has not merged.

## What This Changes

User-defined directory hooks registered for `message_sending` should fire before outbound channel delivery, matching the plugin pre-send hook and the existing internal post-send bridge.

## Best Possible Solution

Review and adapt PR #73570 or a narrow replacement so shared outbound delivery emits a documented pre-delivery internal hook at the correct ordering point, with explicit compatibility and modification/cancel semantics.

## Reproduction Assessment

Yes. The source-level path is high confidence: directory metadata registers `message_sending` verbatim, current outbound delivery runs only the plugin `message_sending` hook before send, and internal dispatch has no emitted `message/sending` event.

## Solution Assessment

Unclear. Bridging in `deliverOutboundPayloads` is the right narrow location, but the final fix should explicitly settle the public event name and whether user-defined internal hooks are observational or can intercept delivery.
