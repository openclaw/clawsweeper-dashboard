## Summary

Keep open: current main still lacks an internal pre-send hook dispatch, and open PR #73570 references this issue with closing syntax but has not merged.

## What This Changes

User-defined hook-directory handlers for outbound `message_sending` should fire before channel delivery.

## Best Possible Solution

Review and adapt PR #73570 or a replacement so the shared outbound path emits a documented pre-delivery internal hook with clear naming, ordering, and modification/cancel semantics.

## Reproduction Assessment

Yes. The source-level path is high confidence: hook metadata registers event keys verbatim, internal dispatch reaches only `message` and `message:<action>` listeners, and current main never creates a `message/sending` internal event in the pre-send delivery path.

## Solution Assessment

Unclear. Bridging the pre-send dispatch in `deliverOutboundPayloads` is the right location, but the final fix should explicitly settle `message_sending` versus `message:sending` compatibility and whether internal hooks are notification-only or can intercept delivery.
