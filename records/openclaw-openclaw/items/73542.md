## Summary

Keep open: current main and v2026.6.6 still do not define a standalone/internal pre-send hook for the reported `HOOK.md` use case, and the direct fix attempt closed unmerged. The remaining work is a maintainer decision about whether internal hooks should support this pre-send contract, what key it uses, and whether it is observation-only or part of guardrail timing.

## What This Changes

The issue requests a standalone `HOOK.md`/internal pre-send hook for `message_sending` so user-defined hooks can observe or guard outbound delivery.

## Best Possible Solution

Define the standalone pre-send hook contract explicitly, then either implement and document it with correct key and timing semantics or clarify that rewrite/cancel guardrails belong only in typed code plugins.

## Reproduction Assessment

Yes at source level: directory hook events are registered verbatim, internal dispatch only matches type or colon-style keys, and current outbound delivery emits only the typed plugin `message_sending` hook plus internal `message:sent`. I did not run a live hook-directory setup in this read-only review.

## Solution Assessment

Unclear. A simple bridge is only a partial fix until maintainers decide the supported standalone hook key, whether it is observation-only, and exactly when it fires relative to rewrite, cancellation, and delivery.
