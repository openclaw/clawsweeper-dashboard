Full review comments:

- **[P1] Gate compaction errors before swallowing them:** `src/agents/agent-command.ts:2239-2243`
  - body: This catch runs whenever transcript persistence returned `persisted`, but `persistTextTurnTranscript` can return `persisted` even with no prompt or reply text. That means a no-reply turn can have a compaction failure converted into a normal delivery/end path, hiding the recovery signal and leaving session state uncompacted. Please gate the catch on normalized deliverable output, keep no-reply compaction failures fatal, and log the skipped compaction.
