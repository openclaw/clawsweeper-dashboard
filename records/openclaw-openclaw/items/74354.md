## Summary

Keep open. Current main still lacks the Control UI composer safeguards and focused numeric-preservation coverage, and open PR #74369 is already the active closing implementation path for this report.

## What This Changes

The issue reports that numeric-only messages sent through Control UI webchat are rewritten with hyphen group separators instead of being sent unchanged.

## Best Possible Solution

Review and land PR #74369, or an equivalent narrow Control UI change, so the composer discourages browser/OS numeric rewriting and tests exact outgoing chat.send payload preservation without adding server-side deformatting that could corrupt intentional hyphenated text.

## Reproduction Assessment

Unclear: the report includes concrete Windows 10 Control UI steps and a screenshot, and current source inspection supports how a rewritten textarea value would be sent. This read-only review did not live-reproduce the browser/IME rewrite.

## Solution Assessment

Yes: the linked narrow Control UI composer fix with regression tests is the best-shaped solution. Repairing grouped numbers in the Gateway would be broader and could alter valid user-authored hyphenated input.
