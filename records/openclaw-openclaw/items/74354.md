## Summary

Keep open. Current main still lacks the Control UI composer safeguards and numeric-preservation coverage, and open PR #74369 already targets this issue with closing syntax, so the issue should wait for that PR to land, be replaced, or be closed.

## What This Changes

The issue requests that Control UI chat send pure numeric input unchanged instead of allowing it to be rewritten with separators such as `123-456-789`.

## Best Possible Solution

Land PR #74369, or an equivalent narrow Control UI change, that adds plain-text/autocorrect suppression attributes to the chat composer and regression tests exact numeric payload preservation without adding broad Gateway-side deformatting.

## Solution Assessment
