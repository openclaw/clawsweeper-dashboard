## Summary

Keep open. The report is still valid against current main, and PR #74369 is already the open closing implementation path for the same Control UI numeric-input bug.

## What This Changes

The issue asks for numeric-only Control UI chat input to be sent exactly as typed instead of being rewritten with separator grouping such as `123456789` becoming `123-456-789`.

## Best Possible Solution

Review and land PR #74369, or an equivalent narrow Control UI composer change, with regression coverage for exact outgoing numeric payload preservation and no Gateway-side deformatting that could alter intentional hyphenated text.

## Reproduction Assessment

Unclear: the issue has concrete Windows 10 Control UI steps and a screenshot, and current source inspection supports the missing guard. This review did not live-reproduce the browser or OS rewrite.

## Solution Assessment

Yes: fixing the Control UI composer is the narrowest maintainable direction because the send path should preserve exactly what the composer stores. Repairing grouped numbers in the Gateway would be broader and could corrupt valid user-authored hyphenated input.
