## What This Changes

The issue reports that Control UI webchat reformats numeric-only input such as `123456789` before sending it.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/74369 or an equivalent narrow Control UI change that prevents browser/OS rewriting at the composer boundary and preserves numeric payloads in tests.

## Reproduction Assessment

No, there is not a high-confidence live reproduction from this review. Source inspection shows the plausible path: the textarea value is stored and sent unchanged while current main lacks attributes that discourage browser/OS numeric rewriting.

## Solution Assessment

Yes, the linked composer-attribute fix appears to be the narrowest maintainable solution. Gateway-side deformatting would be a worse layer because it could rewrite intentionally hyphenated or grouped user text after the browser has already changed the draft.
