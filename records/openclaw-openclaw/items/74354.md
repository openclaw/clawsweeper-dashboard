## Summary

Keep open. Current main and the latest release still lack the Control UI composer safeguards for numeric-only input, and the open linked pull request is the focused fix path rather than a reason to close before merge.

## What This Changes

Control UI should preserve pure numeric chat input instead of allowing browser or OS formatting to rewrite it before `chat.send`.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/74369 or an equivalent narrow textarea-boundary fix, avoiding downstream deformatting that could corrupt intentionally hyphenated text.

## Reproduction Assessment

No high-confidence live repro was established in this read-only pass. Source inspection shows the composer lacks plain-text input safeguards and forwards the textarea value into `chat.send`, matching the reported corruption boundary.

## Solution Assessment

Yes, the linked composer-boundary fix is the narrowest maintainable direction. Server-side deformatting would be broader and could damage intentional separator text.
