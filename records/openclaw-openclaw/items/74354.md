## Summary

Keep open. Current main still lacks Control UI composer safeguards and regression coverage for numeric-only chat input, and open PR #74369 is already the active implementation path for the same bug.

## What This Changes

The issue asks Control UI webchat to preserve pure numeric input exactly instead of letting the composer/browser rewrite it with hyphen group separators.

## Best Possible Solution

Land PR #74369, or an equivalent narrow Control UI fix, so the composer discourages browser auto-formatting and tests exact numeric payload preservation.

## Reproduction Assessment

Unclear: the issue has concrete Windows 10 Control UI steps and screenshot evidence, and source inspection supports the missing composer guard. This read-only pass did not live-reproduce the browser or IME rewrite path.

## Solution Assessment

Yes: a composer-boundary fix with plain-text input attributes and UI tests is the narrow maintainable direction. Server-side deformatting would be broader and could corrupt intentionally hyphenated user text.
