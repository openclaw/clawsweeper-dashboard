## Summary

Keep open. Current main still lacks the Control UI composer safeguards and numeric-preservation regression coverage, and open PR #74369 is already the unmerged closing implementation path.

## What This Changes

The issue requests that Control UI chat preserve pure numeric input instead of sending browser/OS-formatted separator text.

## Best Possible Solution

Land PR #74369, or an equivalent narrow Control UI composer fix, with regression tests for exact numeric payload preservation and no gateway-side deformatting of user text.

## Reproduction Assessment

No: there is not a high-confidence live reproduction path from this pass. Source inspection shows the vulnerable path clearly: the textarea value becomes the draft and is forwarded to `chat.send`, but the Windows/browser/IME rewrite itself was not reproduced.

## Solution Assessment

Yes: composer-level plain-text suppression attributes plus exact payload regression tests are the narrowest maintainable fix. Server-side numeric deformatting would risk corrupting intentionally hyphenated user input.
