## Summary

Keep open. Current main still lacks the Control UI composer attributes and regression coverage proposed for numeric-only chat input, and open PR #74369 is already the active closing implementation path for this issue.

## What This Changes

The issue reports that Control UI web chat rewrites pure numeric input such as `123456789` into grouped text such as `123-456-789` before sending.

## Best Possible Solution

Land PR #74369, or an equivalent narrow Control UI change, so the composer discourages browser/OS numeric rewriting and regression tests prove exact outgoing payload preservation without adding server-side deformatting that could corrupt intentional hyphenated text.

## Reproduction Assessment

Unclear: the issue has concrete Windows 10 Control UI steps and a screenshot, and current source inspection supports the missing guard. I did not live-reproduce the browser/OS/IME rewrite against current main.

## Solution Assessment

Yes: a focused Control UI textarea attribute fix with regression tests is the narrowest maintainable path. Repairing grouped numbers in the Gateway would be broader and could change valid user-authored hyphenated input.
