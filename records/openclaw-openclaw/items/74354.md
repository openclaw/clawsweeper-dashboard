## Summary

Keep open. Current main still lacks the Control UI composer safeguards and focused regression coverage for numeric-only chat input, while open PR #74369 is already the active closing implementation path for the same report.

## What This Changes

The issue requests that Control UI webchat preserve pure numeric input exactly instead of allowing values like 123456789 to be rewritten as 123-456-789 before sending.

## Best Possible Solution

Land PR #74369, or an equivalent narrow Control UI change, so the composer discourages browser/OS rewriting of numeric drafts and tests exact outgoing payload preservation without adding server-side deformatting that could corrupt intentional hyphenated text.

## Reproduction Assessment

Unclear: the issue has concrete Windows 10 Control UI steps and a screenshot, and current source inspection supports the missing guard. This review did not live-reproduce the browser/IME rewrite.

## Solution Assessment

Yes: a focused Control UI composer fix with regression tests is the narrowest maintainable path. Repairing grouped numbers in the Gateway would be broader and could alter valid user-authored hyphenated input.
