## Summary

Keep open. Current main still lacks the Control UI composer safeguards and focused numeric-preservation coverage, while open PR #74369 is already the active closing implementation path for this report.

## What This Changes

The issue reports that Control UI chat rewrites pure numeric input such as `123456789` into grouped text such as `123-456-789`, breaking numeric IDs, arguments, and URLs.

## Best Possible Solution

Review and land PR #74369, or an equivalent narrow Control UI change, so the composer discourages browser/OS numeric rewriting and tests exact outgoing `chat.send` payload preservation without server-side deformatting that could corrupt intentional hyphenated input.

## Reproduction Assessment

Unclear: the report includes concrete Windows 10 Control UI steps and a screenshot, and current source inspection supports the missing guard. This read-only pass did not live-reproduce the browser/IME rewrite.

## Solution Assessment

Yes: the best path is a narrow Control UI composer fix with regression tests for numeric payload preservation. Repairing grouped numbers in the Gateway would be broader and could change valid user-authored hyphenated text.
