## Summary

Keep this issue open: current main and the latest release still lack the Control UI composer safeguards, and the linked open fix PR at https://github.com/openclaw/openclaw/pull/74369 is the active implementation path rather than a reason to close before merge.

## What This Changes

Control UI should preserve numeric-only chat composer input exactly as typed instead of allowing browser or OS formatting to alter it before `chat.send`.

## Best Possible Solution

Rebase, review, and land the linked focused Control UI composer fix, or land an equivalent narrow replacement, then close this issue once the change reaches main.

## Reproduction Assessment

No, I do not have a high-confidence live reproduction against current main. Source inspection shows the plausible failure boundary: the composer accepts textarea values and forwards them to `chat.send`, while current main lacks plain-text input attributes that would discourage browser or OS numeric rewriting.

## Solution Assessment

Yes, the linked PR's textarea-attribute mitigation appears to be the narrowest maintainable fix. Gateway-side deformatting would be the wrong layer because it could rewrite intentionally hyphenated input after the browser already changed the draft.
