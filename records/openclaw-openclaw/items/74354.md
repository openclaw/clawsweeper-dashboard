## Summary

Keep open. Current main still lacks the Control UI composer safeguards and regression coverage for numeric-only chat input, and open PR #74369 is already the linked implementation path.

## What This Changes

The issue asks Control UI webchat to preserve pure numeric messages exactly instead of allowing grouped or auto-formatted text to be sent.

## Best Possible Solution

Land PR #74369, or an equivalent narrow Control UI composer change, with tests that lock numeric payload preservation while avoiding gateway-side deformatting of intentional hyphenated text.

## Reproduction Assessment

Unclear: the reporter provided Windows Control UI steps and screenshot evidence, and source inspection shows the composer lacks suppression attributes while forwarding textarea values, but this pass did not live-reproduce the browser/OS rewrite that inserts separators.

## Solution Assessment

Yes: the linked composer-level plain-text attributes plus exact payload regression tests are the narrow maintainable direction. Server-side deformatting would risk changing intentionally hyphenated user content.
