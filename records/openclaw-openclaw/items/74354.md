## Summary

Keep open. Current main still lacks the Control UI composer safeguards and regression coverage for numeric-only chat input, and PR #74369 remains the open closing implementation path.

## What This Changes

Control UI should preserve pure numeric chat input exactly instead of allowing browser or OS input behavior to rewrite it with separators.

## Best Possible Solution

Land PR #74369, or an equivalent narrow Control UI composer fix, with regression coverage proving numeric-only drafts are sent unchanged and without gateway-side deformatting.

## Reproduction Assessment

Unclear: the issue gives concrete Windows 10 Control UI steps and screenshot evidence, and current source inspection supports the missing guard. This read-only review did not live-reproduce the browser or IME rewrite.

## Solution Assessment

Yes: the composer-level plain-text attribute fix with UI/send-path tests is the narrowest maintainable direction. Repairing hyphen-grouped digits in the Gateway would be broader and could corrupt intentional hyphenated input.
