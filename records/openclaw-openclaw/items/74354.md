Keep open. Current main still lacks the Control UI composer safeguards and regression coverage for numeric-only chat input, and open PR #74369 is already the active closing implementation path.

## Best Possible Solution

Land PR #74369, or an equivalent narrow composer-level fix, with tests that preserve exact numeric payloads and without server-side deformatting of intentional hyphenated text.
