Keep open. Current main still lacks the Control UI composer safeguards and regression coverage for numeric-only chat input, and open PR #74369 is already the active closing implementation path.

## What This Changes

The issue reports that numeric-only Control UI chat input can be reformatted with hyphen group separators before it is sent.

## Best Possible Solution

Land PR #74369, or an equivalent narrow Control UI composer fix, with tests proving exact numeric payload preservation and no server-side deformatting of intentional hyphenated text.

## Solution Assessment

Yes: a composer-level plain-text input fix with regression coverage is the narrowest maintainable direction; repairing grouped numbers in the Gateway could corrupt user-authored hyphenated input.
