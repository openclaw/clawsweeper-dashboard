## Summary

Keep open. Current main still lacks the Control UI composer safeguards and regression coverage for numeric-only drafts, and open PR #74369 is already the active linked implementation path with closing syntax.

## What This Changes

The issue reports that Control UI webchat can rewrite numeric-only input such as 123456789 into grouped text before sending.

## Best Possible Solution

Review and land PR #74369, or an equivalent narrow Control UI composer change, with tests proving numeric drafts are sent unchanged and without gateway-side deformatting.

## Reproduction Assessment

Unclear: the issue has concrete Windows 10 Control UI steps and screenshot evidence, and source inspection confirms the draft is forwarded as read from the textarea. This read-only pass did not live-reproduce the browser or IME rewrite path.

## Solution Assessment

Yes: composer-level suppression attributes plus exact payload tests are the narrow maintainable fix. Server-side deformatting would be broader and could corrupt intentionally hyphenated user text.
