## Summary

Keep this issue open: current main and v2026.6.6 still lack the Control UI composer safeguards for numeric-only chat input, and the open linked fix PR at https://github.com/openclaw/openclaw/pull/74369 is the active implementation path rather than a reason to close before merge.

## What This Changes

The issue asks Control UI to preserve numeric-only chat input exactly instead of allowing the webchat composer to convert it to grouped text such as `123-456-789`.

## Best Possible Solution

Rebase, review, and land https://github.com/openclaw/openclaw/pull/74369, or an equivalent narrow Control UI composer fix, then close this issue once the fix reaches main.

## Reproduction Assessment

No high-confidence live reproduction was performed. Source inspection shows the plausible current-main path: the composer accepts textarea values and forwards them to `chat.send`, while current main lacks the plain-text attributes that would discourage browser or OS numeric rewriting.

## Solution Assessment

Yes for the narrow numeric issue: fixing the composer textarea boundary is the safest path. Gateway-side deformatting would be a weaker solution because it could rewrite intentionally hyphenated or grouped user input after the browser already changed the draft.
