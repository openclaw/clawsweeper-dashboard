## Summary

Keep open. Current main still lacks the Control UI composer safeguards and regression coverage for numeric-only chat input, while open PR #74369 is already the active closing implementation path.

## What This Changes

Control UI should preserve pure numeric chat input exactly instead of letting browser or OS text handling rewrite it with separators.

## Best Possible Solution

Land PR #74369, or an equivalent narrow Control UI composer fix, with regression coverage for exact numeric payload preservation and without server-side deformatting that could alter intentional hyphenated text.

## Reproduction Assessment

## Solution Assessment

Yes: the linked composer-level fix is the narrowest maintainable direction. Repairing grouped numbers in the Gateway would be broader and could corrupt user-authored hyphenated input.
