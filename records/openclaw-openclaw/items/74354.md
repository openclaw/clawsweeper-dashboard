## Summary

Keep this issue open: current main and the latest release still lack the Control UI composer safeguards for numeric-only input, and the open linked fix PR is the active implementation path rather than a reason to close before merge.

## What This Changes

The issue asks Control UI to preserve pure numeric chat input instead of allowing it to become a hyphen-grouped string such as `123-456-789`.

## Best Possible Solution

Land or replace the narrow composer-boundary fix with regression coverage for numeric payload preservation, then close this issue once that fix reaches main.

## Reproduction Assessment

No high-confidence live reproduction was performed. Source inspection shows the plausible current-main path: the composer stores textarea values and sends them through `chat.send`, while current main lacks the plain-text textarea attributes that would discourage browser or OS numeric rewriting.

## Solution Assessment

Yes for the narrow reported bug: fixing the composer textarea boundary is the best maintainable path. Gateway-side deformatting would be riskier because it could rewrite intentional hyphenated or grouped input after the draft was already changed.
