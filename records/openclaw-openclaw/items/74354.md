## What This Changes

The issue asks for Control UI chat to preserve numeric-only input exactly instead of letting browser or OS input behavior rewrite it with separators.

## Best Possible Solution

Review and land PR #74369, or an equivalent narrow Control UI composer change, with tests that preserve numeric drafts without adding gateway-side deformatting.
