## What This Changes

Request a Telegram config toggle or limit to disable supplemental selected conversation context in direct messages.

## Best Possible Solution

If maintainers want this, add an explicit Telegram-owned DM supplemental-context limit or disable option with schema, docs, and focused tests while preserving intentional reply-context behavior.

## Reproduction Assessment

Yes for the current behavior: source inspection shows Telegram always builds a selected context window with recentLimit: 10 and injects it when non-empty. This is not a bug reproduction for an existing contract because the requested opt-out does not exist yet.

## Solution Assessment

Unclear until maintainers choose the product contract. An explicit new Telegram option is safer than silently coupling supplemental context to dmHistoryLimit.
