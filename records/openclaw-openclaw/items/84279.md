## What This Changes

The issue requests a Telegram configuration control to disable or limit the supplemental selected conversation context window in Telegram direct messages.

## Best Possible Solution

Keep the issue open for a maintainer decision on a narrow Telegram-owned DM supplemental-context setting or numeric limit, with group behavior preserved by default.

## Reproduction Assessment

Yes for the current behavior by source inspection: current main builds the selected Telegram context window with a hard-coded recent limit and no DM opt-out. This is not a reproduced bug against an existing contract because the requested config option does not exist yet.

## Solution Assessment

Unclear until maintainers choose the contract. An explicit DM supplemental-context option or limit looks safer than silently overloading `dmHistoryLimit`, because the current code and docs treat transcript history and supplemental structured context separately.
