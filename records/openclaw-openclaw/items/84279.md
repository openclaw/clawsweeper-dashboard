Keep open: current main and the latest release still build Telegram selected conversation context with no DM-only opt-out, and the requested behavior is a new Telegram config/product contract rather than conservative cleanup or an already-implemented fix.

## What This Changes

Add a Telegram DM config toggle or limit to disable the supplemental selected conversation context window.

## Best Possible Solution

If maintainers want this, add an explicit Telegram-owned DM supplemental-context option or numeric selected-context limit with config schema, docs, and focused Telegram tests instead of accidentally overloading `dmHistoryLimit`.

## Reproduction Assessment

Yes for the current behavior: source inspection shows Telegram always builds and injects the selected context window when cached context is non-empty. This is not a failing bug reproduction for an existing contract because the requested DM opt-out does not exist yet.

## Solution Assessment

No final fix path is settled. An explicit new Telegram option is safer than silently changing `dmHistoryLimit` semantics, but maintainers should choose the product contract before implementation.
