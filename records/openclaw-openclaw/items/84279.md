## Summary

Keep open: current main and the latest release still build a Telegram selected conversation context window with no DM-only opt-out. The requested change is a new config/product knob, not conservative cleanup or an already-implemented fix.

## What This Changes

The issue requests a Telegram configuration option to disable the supplemental selected conversation context window in direct messages.

## Best Possible Solution

If maintainers want this behavior, add an explicit Telegram-owned DM supplemental-context limit or disable option with schema, docs, and focused tests rather than overloading `dmHistoryLimit` by accident.

## Reproduction Assessment

Yes for the current behavior: source inspection shows Telegram always builds the selected context window with `recentLimit: 10` and injects it when non-empty. This is not a bug reproduction for an existing contract because the requested opt-out does not exist yet.

## Solution Assessment

No final fix path is settled. The explicit new option suggested in the issue is safer than silently changing `dmHistoryLimit` semantics, but maintainers should choose the product contract before implementation.
