## Summary

Keep this PR open: it is the current active Eden AI provider proposal, but the patch is not merge-ready because live model discovery does not send the required bearer auth and the manifest endpoint metadata uses an endpoint class current core will ignore. The bundled-provider direction also still needs explicit maintainer product/ownership approval.

## What This Changes

The PR adds a bundled Eden AI provider plugin with onboarding auth, a static and live model catalog, docs, labeler metadata, `.env.example`, and workspace lockfile entries.

## Best Possible Solution

If maintainers approve Eden AI as an official bundled provider, repair live catalog discovery to use authenticated shared catalog fetching, make endpoint metadata effective or remove it, and land with explicit ownership expectations; otherwise publish this as an external plugin first.

## Reproduction Assessment

For the feature request, reproduction is not applicable because this is new provider support. For the patch defects, yes: source inspection plus Eden AI's docs show live `/v3/models` needs bearer auth while the PR sends no headers, and current core skips unknown manifest endpoint classes.

## Solution Assessment

No, not as submitted. The provider-plugin shape is plausible, but the live catalog should use authenticated discovery and the endpoint metadata must either match current core policy or be removed before maintainers decide on bundled support.
