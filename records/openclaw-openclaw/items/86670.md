Full review comments:

- **[P1] Send auth to Eden AI model discovery:** `extensions/edenai/provider-catalog.ts:271-275`
  - body: `catalog.run` resolves `EDENAI_API_KEY`, but `buildEdenaiProvider()` never receives it and `discoverEdenaiModels()` calls `/v3/models` without an `Authorization: Bearer ...` header. Eden AI requires that header, so authenticated catalog discovery falls back to the static eight-model list and live-only Eden AI model refs fail the new `prepareDynamicModel` check instead of working as advertised.
  - confidence: 0.91
- **[P2] Parse the documented reasoning capability:** `extensions/edenai/provider-catalog.ts:218-220`
  - body: The live catalog parser looks for `capabilities.supports_reasoning`, but Eden AI documents this field as `capabilities.reasoning`. Once live discovery is authenticated, reasoning-capable live-only models will still be registered with `reasoning: false`, so OpenClaw will not get the metadata this PR says comes from Eden AI's source of truth.
  - confidence: 0.84
