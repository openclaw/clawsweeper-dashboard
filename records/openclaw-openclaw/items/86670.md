  - confidence: 0.94
- **[P2] Thread the API key into Eden AI discovery:** `extensions/edenai/index.ts:90`
  - body: `catalog.run` resolves the Eden AI API key but calls `buildEdenaiProvider()` without passing it to discovery, and `discoverEdenaiModels()` sends `/models` without auth headers or an auth-scoped cache key. Eden AI documents `/v3/models` with Bearer auth, so model listing and dynamic validation can drift from what the user's key can actually run.
  - confidence: 0.88
- **[P2] Read Eden AI's documented capability fields:** `extensions/edenai/provider-catalog.ts:217-220`
  - body: The live catalog parser reads `capabilities.supports_reasoning` and `input_modalities`, but Eden AI documents LLM capabilities as fields like `reasoning`, `web_search`, and `tool_calling`. Live rows outside the static fallback will lose documented reasoning metadata unless the parser is aligned with the actual Eden AI response or the PR proves the extra fields are present.
  - confidence: 0.82
