## Summary

Keep open. The PR targets a real source-reproducible gateway auth issue, but the current patch is not merge-ready because it blocks only self-declared operator.admin while preserving other self-declared operator scopes on the same device-less shared-auth backend path, and it changes a documented shared-secret trust boundary without maintainer-visible security approval or real behavior proof.

## What This Changes

The PR changes gateway WebSocket backend self-pairing so shared token/password auth no longer preserves self-declared operator.admin scopes without device identity, and adds focused gateway auth tests.

## Best Possible Solution

Have gateway security owners choose the shared-auth backend credential contract, then either clear all self-declared scopes for device-less shared token/password backend connects and update docs/tests, or explicitly document and approve the narrower admin-only split.

## Reproduction Assessment

Yes. Source inspection of current main shows a no-Origin local or shared-secret-loopback backend client with shared token/password auth sets skipLocalBackendSelfPairing, and the compat baseline currently expects operator.admin to be preserved.

## Solution Assessment

No. The direction is plausible hardening, but the PR keys the fix only on operator.admin and does not settle or document the broader shared-secret backend scope boundary.
