## Summary

Keep open. Current main still removes workspace paths only as separate backup candidates while archiving the broad state tree, and the archive filter has no branch that excludes nested workspace roots when workspace inclusion is disabled.

## What This Changes

The issue asks for `backup create --no-include-workspace` to materially exclude workspace directories even when they live under `~/.openclaw`, or to stop promising that behavior.

## Best Possible Solution

Make the existing flag archive-real by excluding canonical workspace roots during state-asset tar creation when workspace inclusion is disabled, while keeping generic exclude-pattern UX tracked separately unless maintainers intentionally land it together.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path: default workspaces live under the default state directory, the flag drops only separate workspace candidates, and the tar filter still includes state entries without a workspace check. I did not run a live backup because this was a read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is a workspace-aware state archive filter keyed off the existing `includeWorkspace` flag; a docs-only clarification would leave the CLI behavior contrary to current help and backup guidance.
