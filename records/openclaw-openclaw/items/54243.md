## Summary

Keep open. Current main still plans the broad state directory and only removes workspace candidates from the plan, so default workspaces nested under the state directory are still archived unless an archive-level filter is added or the public contract is changed.

## What This Changes

The issue asks for `openclaw backup create --no-include-workspace` to materially exclude workspace contents from backup archives, or for the CLI/docs to stop implying that it does.

## Best Possible Solution

Add canonical nested-workspace exclusions to backup archive creation when workspace inclusion is disabled, keep the manifest and verification semantics consistent, and document the exact scope of the flag after the behavior is real.
