## Label Justifications

- `P2`: This is a normal-priority correctness bug in an existing backup flag with a clear source path and limited CLI/archive blast radius.
- `impact:security`: The flag can unexpectedly include workspace/project contents in backup archives despite asking to exclude them, which can expose sensitive workspace data.

## Summary

Keep open. Current main and the latest release still promise workspace exclusion, still include the broad state directory, and the archive tar filter has no workspace-root exclusion for default workspaces under `~/.openclaw`.

## What This Changes

The issue requests that `backup create --no-include-workspace` actually omit nested agent workspaces from backup archives instead of only removing separate workspace assets from the plan.

## Best Possible Solution

Make the existing flag archive-real by excluding canonical workspace roots from state-asset tar traversal when `includeWorkspace` is false, while leaving generic exclude-pattern UX to the separate native-exclude work.

## Reproduction Assessment

Yes, from source inspection. Default workspaces live under `~/.openclaw`, the planner still includes the broad state asset, and the tar filter has no workspace-root exclusion when `includeWorkspace` is false.

## Solution Assessment

Yes. The narrow maintainable fix is to apply a workspace-aware archive filter keyed off the existing flag; a docs-only clarification would leave behavior contrary to current CLI help and backup docs.
