## Summary

Keep open. Current main and v2026.5.18 still keep the broad state directory in the backup plan and the archive filter has no workspace-root exclusion when workspace inclusion is disabled, so the existing flag remains misleading for default nested workspace layouts.

## What This Changes

`openclaw backup create --no-include-workspace` should materially exclude agent workspace directories even when they live under the OpenClaw state directory.

## Best Possible Solution

Make the existing flag archive-real by carrying canonical workspace roots into archive creation and excluding those roots from state-asset tar entries when workspace inclusion is disabled, while leaving generic exclude patterns to the broader tracked work.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: default workspaces live under the default state directory, the flag drops only separate workspace candidates, and the tar filter still includes state entries without a workspace-root check.

## Solution Assessment

Yes. The narrow maintainable fix is to make the existing flag affect archive filtering for canonical workspace roots; a docs-only clarification would leave behavior contrary to CLI help and backup guidance.
