## Summary

Keep open. Current main and the latest release still keep the broad state directory in the backup plan and the archive filter has no workspace-root exclusion when workspace inclusion is disabled, so the existing CLI flag remains misleading for default `~/.openclaw/workspace` layouts.

## What This Changes

The issue reports that `openclaw backup create --no-include-workspace` still archives nested agent workspaces through the broad state-directory backup asset.

## Best Possible Solution

Make the existing flag archive-real by excluding canonical workspace roots during state-asset tar creation when workspace inclusion is disabled, while keeping generic exclude-pattern UX tracked separately unless maintainers intentionally land it together.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: default workspaces live under the default state directory, the flag drops only separate workspace candidates, and the tar filter still includes state entries without a workspace check; I did not run a live backup in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is a workspace-aware state archive filter keyed off the existing `includeWorkspace` flag; a docs-only clarification would leave behavior contrary to CLI help and backup guidance.
