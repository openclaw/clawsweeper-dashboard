## Summary

Keep open: the PR is useful, but unset dry-run still reports success after schema-only validation without checking affected SecretRefs or provider fallout, so it does not yet match the existing config set/patch dry-run contract.

## What This Changes

The PR adds `--dry-run` and `--json` handling for `openclaw config unset`, plus focused tests for no-write behavior, JSON output, missing paths, and standalone `--json` rejection.

## Best Possible Solution

Route unset dry-run through the existing config mutation dry-run semantics, or an equivalent helper, so deletion validates schema, SecretRef/provider fallout, exec-skip or allow-exec behavior, JSON reporting, tests, and docs together.

## Reproduction Assessment

Yes. Current main is source-reproducible because `config unset` registers no `--dry-run` or `--json` options, while the linked issue’s help-output report expects those flags.

## Solution Assessment

No. The direction is right, but the patch is not yet the best fix because it schema-validates unset dry-run success without checking affected SecretRefs or provider fallout like sibling dry-run paths do.
