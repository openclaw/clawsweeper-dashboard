## Summary

Keep open: the PR addresses a real `config unset` parity gap, but the latest implementation still violates the established exec SecretRef dry-run contract and lacks visible latest-head real behavior proof.

## What This Changes

The PR adds `--dry-run` and `--json` to `openclaw config unset`, validates the post-unset config, and adds focused config CLI tests.

## Best Possible Solution

Reuse the shared config mutation dry-run semantics, or an equivalent helper, so unset deletes preserve schema/resolvability checks, exec skip or explicit allow-exec behavior, JSON result metadata, and focused regression coverage.

## Reproduction Assessment

Yes. Current main is source-reproducible because `config unset` registers no `--dry-run` or `--json` options, while the linked issue shows the missing help output and asks for parity with `config set` and `config patch`.

## Solution Assessment

No. The direction is right, but the latest PR should preserve the existing exec SecretRef dry-run opt-in contract instead of resolving affected refs directly from unset dry-run.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Skip exec SecretRefs during unset dry-runs:** `src/cli/config-cli.ts:1824-1827`
  - body: The new provider-path dry-run branch sends all affected refs to `collectDryRunResolvabilityErrors`. For exec refs, that reaches the configured exec provider command, while existing set/patch dry-runs skip exec resolution unless `--allow-exec` is set; `unset` adds no such opt-in. Dry-running a nested `secrets.providers.<alias>.*` unset can therefore execute local provider commands unexpectedly.
  - confidence: 0.87
