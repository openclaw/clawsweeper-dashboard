## Summary

Keep open: this PR addresses a real `config unset` parity gap, but the latest diff still has blocking SecretRef dry-run regressions, including a security-sensitive exec-provider path, and the supplied proof does not cover that changed behavior.

## What This Changes

The PR adds `--dry-run` and `--json` handling to `openclaw config unset`, updates its dry-run validation path, and adds focused config CLI tests.

## Best Possible Solution

Route unset dry-run through the shared config mutation dry-run semantics, or an equivalent helper, so schema validation, SecretRef selection, exec skipping, JSON metadata, and provider-map fallout stay consistent across config commands.

## Reproduction Assessment

Yes. Current main source shows `config unset` has no `--dry-run` or `--json` options, and the linked issue provides shipped v2026.5.7 help output for the same gap; the PR regressions are source-reproducible from the latest diff and current SecretRef resolver path.

## Solution Assessment

No. The requested CLI direction is reasonable, but this implementation duplicates dry-run logic and misses established exec-skip and provider-map semantics; reusing or closely mirroring the shared mutation dry-run path is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Filter exec refs before unset dry-run resolution:** `src/cli/config-cli.ts:1824-1826`
  - body: This sends provider-fallout refs directly to `collectDryRunResolvabilityErrors`, but `config unset` has no `--allow-exec` option. For a dry-run of a nested path under an exec provider, the provider remains configured and resolving the affected ref can spawn the configured command, unlike set/patch dry-runs that skip exec refs by default.
  - confidence: 0.9
- **[P2] Check whole provider-map removals for dangling refs:** `src/cli/config-cli.ts:1821-1822`
  - body: When the unset path is `secrets.providers`, `parsedPath[2]` is undefined, so no SecretRef fallout checks run. That dry-run can report success even though existing refs would point at removed providers.
  - confidence: 0.86
