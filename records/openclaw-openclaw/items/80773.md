## Summary

Keep open: this PR addresses a real current-main CLI parity gap, but the patch still has a dry-run validation parity bug and overlaps an open maintainer-labeled implementation at https://github.com/openclaw/openclaw/pull/81895.

## What This Changes

The PR adds `--dry-run`, `--json`, and `--allow-exec` support to `openclaw config unset`, with post-unset validation, SecretRef fallout checks, tests, and config CLI docs.

## Best Possible Solution

Land one canonical implementation that reuses the shared plugin-aware config mutation dry-run machinery, carries unset flags through fast routing, keeps the stronger SecretRef fallout handling, updates docs, and then closes https://github.com/openclaw/openclaw/issues/80721.

## Reproduction Assessment

Yes. Current-main source shows `config unset` lacks dry-run/json/allow-exec handling, and the linked issue includes released CLI help output showing the same gap.

## Solution Assessment

No. Adding the flags to `config unset` is the right direction, but this patch should use plugin-aware/shared dry-run validation and be reconciled with the overlapping implementation before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Use plugin-aware validation for unset dry-runs:** `src/cli/config-cli.ts:1813-1816`
  - body: `config set` and `config patch` dry-runs validate through the plugin-aware raw config path, but this new unset dry-run calls only `validateConfigObjectRaw`. Removing a plugin config field can therefore report dry-run success even when the installed plugin schema would reject the post-unset config; reuse the shared dry-run validation path or call `validateConfigObjectRawWithPlugins` here.
  - confidence: 0.84
