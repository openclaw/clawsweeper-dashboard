## Summary

Keep this PR open, but it is not ready to merge: the feature direction matches the linked parity request, while the current diff returns dry-run success before doing the post-change validation that the request and existing config mutation contract require.

## What This Changes

The PR adds `--dry-run` and `--json` options to `openclaw config unset` with three focused tests for no-write and JSON output behavior.

## Best Possible Solution

Implement `config unset --dry-run` through the same mutation and validation semantics as set/patch delete operations, with structured JSON only in dry-run mode and no write to `openclaw.json`.

## Reproduction Assessment

Yes. The missing capability is source-reproducible on current main because `config unset` registers no `--dry-run` or `--json` options, matching the linked issue's help-output report.

## Solution Assessment

No. The direction is right, but this patch is not the best implementation because it reports dry-run success before post-change validation and accepts standalone `--json`; reusing the existing validated mutation path for delete semantics is safer.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Run post-change validation before dry-run success:** `src/cli/config-cli.ts:1795-1801`
  - body: The new dry-run branch returns success immediately after the path is removed from the cloned object. That bypasses the schema/policy validation that normal writes perform and that the linked parity request expects, so dry-run can say an unset is safe even when the resulting config would be rejected. Validate the post-change config before emitting text or JSON success.
  - confidence: 0.92
- **[P2] Reject --json unless dry-run is set:** `src/cli/config-cli.ts:2061`
  - body: The added option is described as outputting a dry-run result, but `runConfigUnset` ignores `json` when `dryRun` is false. That means `openclaw config unset <path> --json` still mutates config and prints the normal human message, unlike `config patch`, which rejects this mode. Add the same `--json requires --dry-run` guard for unset.
  - confidence: 0.87
