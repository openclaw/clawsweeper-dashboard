## Summary

Keep open. The PR targets a valid CLI parity gap, but the latest patch still violates the existing dry-run exec SecretRef contract and the supplied proof does not cover the current head or the SecretRef fallout path.

## What This Changes

The branch adds `--dry-run` and `--json` to `openclaw config unset`, validates the post-unset config, checks provider-removal SecretRef fallout, and adds focused config CLI tests.

## Best Possible Solution

Reuse the shared config mutation dry-run semantics, or an equivalent helper, so unset preserves schema validation, SecretRef fallout checks, skipped exec metadata, `--allow-exec` opt-in behavior, and regression coverage.

## Reproduction Assessment

Yes. Current main is source-reproducible because `config unset` exposes no `--dry-run` or `--json` options, and the linked report includes help output showing the gap against `config set` and `config patch`.

## Solution Assessment

No. The direction is right, but the patch should preserve the existing exec SecretRef dry-run opt-in contract and cover broader provider-map removals instead of resolving all affected refs directly.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Skip exec SecretRefs during unset dry-runs:** `src/cli/config-cli.ts:1824-1827`
  - body: The provider-removal dry-run branch passes every affected ref directly to `collectDryRunResolvabilityErrors`. Exec refs then reach the configured provider command, while sibling dry-runs skip exec resolution unless `--allow-exec` is set; `unset` adds no such opt-in, so a dry-run can execute local commands unexpectedly.
  - confidence: 0.9
- **[P2] Check provider-map removals for dangling refs:** `src/cli/config-cli.ts:1821-1822`
  - body: When the unset path is `secrets.providers` or another provider subtree without a concrete alias, `providerAlias` is undefined and the dry-run skips SecretRef fallout checks entirely. Removing the provider map can leave file/exec refs dangling while still reporting success.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The patch introduces an unexpected local command execution path in a dry-run flow for exec-backed SecretRefs.

Concerns:
