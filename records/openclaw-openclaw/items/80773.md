## Summary

Keep open: this is a plausible fix for the linked config-unset parity gap, but the latest head still bypasses the established exec SecretRef dry-run opt-in and misses whole-provider-map fallout checks.

## What This Changes

The PR adds `--dry-run` and `--json` options to `openclaw config unset`, validates the post-unset config, checks SecretRef fallout for provider removal, and adds focused CLI tests.

## Best Possible Solution

Route unset through the shared config mutation dry-run semantics, or an equivalent helper, so schema validation, SecretRef selection, exec skipping/allow-exec, JSON metadata, and provider-map fallout stay consistent across config commands.

## Reproduction Assessment

Yes. Current main source shows `config unset` has no `--dry-run` or `--json` options and writes immediately, and the linked issue includes v2026.5.7 help output showing the same gap. The PR regressions are also source-reproducible from the added direct SecretRef resolution path and provider-alias-only fallout branch.

## Solution Assessment

No. The requested direction is maintainable, but this implementation duplicates part of the dry-run pipeline and misses established SecretRef execution and provider-map semantics; the safer fix is to reuse or closely mirror the shared mutation dry-run helper.

  - confidence: 0.86
