## What This Changes

The PR adds `--dry-run`, dry-run-only `--json`, and dry-run-only `--allow-exec` support to `openclaw config unset`, with post-unset schema and SecretRef validation plus config CLI tests.

## Best Possible Solution

Land one canonical `config unset` dry-run implementation after the mechanical CI blockers are fixed, preserving parity with `config set` and `config patch` for JSON, schema validation, SecretRef fallout, and explicit exec resolution.

## Reproduction Assessment

Yes. Source inspection of current main shows `config unset` lacks the requested flags while sibling mutation commands already have dry-run semantics, and CI annotations reproduce the two current PR blockers.

## Solution Assessment

No, not yet. The implementation direction is narrow and maintainable, but the current patch needs the redundant conversion removed and the queued mock snapshot leak fixed before merge.
