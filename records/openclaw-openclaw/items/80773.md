## Summary

Keep this PR open. The requested CLI parity is valid, but the current patch still has a security-sensitive dry-run execution regression, a PR-introduced test isolation failure, and no latest-head real behavior proof.

## What This Changes

This PR adds `--dry-run` and `--json` to `openclaw config unset`, validates post-unset schema and SecretRef fallout, and adds focused config CLI tests.

## Best Possible Solution

Reuse the shared config mutation dry-run semantics, or an equivalent helper, so unset dry-runs validate schema and resolvability while skipping exec refs unless explicitly opted in, with regression coverage and clean test isolation.

## Reproduction Assessment

Yes. Current main source shows `config unset` has no `--dry-run` or `--json` options while `config set` and `config patch` do, matching the linked issue's help-output report.

## Solution Assessment

No. The feature direction is right, but this implementation bypasses the existing exec SecretRef dry-run opt-in contract and leaves a new test mock queued across test cases.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Skip exec SecretRefs during unset dry-runs:** `src/cli/config-cli.ts:1824-1827`
  - body: The provider-removal branch passes all affected refs straight into `collectDryRunResolvabilityErrors`. For exec refs, that reaches the configured provider command, while sibling dry-runs skip exec resolution unless `--allow-exec` is set; `unset` adds no such opt-in, so a dry run can unexpectedly execute local commands.
  - confidence: 0.9
- **[P2] Remove the unused snapshot setup:** `src/cli/config-cli.test.ts:2382`
  - body: This test calls `setSnapshot`, but the new `--json` guard throws before `runConfigUnset` reads the config. The queued `mockResolvedValueOnce` leaks into the following `config file` test, which is why the CLI CI shard now sees `/tmp/openclaw.json` instead of the home-directory snapshot.
  - confidence: 0.92
