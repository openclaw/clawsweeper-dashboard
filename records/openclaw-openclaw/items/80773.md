## Summary

Keep open: the PR targets a real CLI parity gap, but the latest head still has a blocking exec SecretRef dry-run regression and the real behavior proof is stale relative to the SecretRef fallout change.

## What This Changes

This PR adds `--dry-run` and `--json` to `openclaw config unset`, with schema/SecretRef dry-run validation and focused config CLI tests.

## Best Possible Solution

Reuse the shared config mutation dry-run semantics, or an equivalent helper, so unset preserves schema validation, SecretRef fallout checks, skipped exec metadata, `--allow-exec` opt-in behavior, and regression coverage.

## Reproduction Assessment

Yes. Current main is source-reproducible because `config unset` registers no `--dry-run` or `--json` options, while https://github.com/openclaw/openclaw/issues/80721 shows the missing help output and asks for set/patch parity.

## Solution Assessment

No. The direction is right, but the latest patch should preserve the existing exec SecretRef dry-run opt-in contract instead of resolving affected exec refs directly from unset dry-run.
