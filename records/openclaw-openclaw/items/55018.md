## Summary

Keep open: current main still uses the repeated shallow-copy recursion, so the optimization is useful, but this PR is not merge-ready because it changes the helper’s mutation contract, lacks real behavior proof, and is currently conflicting.

## What This Changes

Refactors `src/config/schema.hints.ts` so `mapSensitivePaths` delegates to a private in-place accumulator instead of cloning hints during every recursive schema traversal.

## Best Possible Solution

Keep the in-place accumulator idea, but rebase onto current main, clone once in the public wrapper before mutating, add focused immutability/perf validation as needed, and attach redacted real benchmark or terminal proof before merge.

## Reproduction Assessment

No high-confidence measured runtime reproduction is attached. Source inspection does confirm current main recursively clones the growing hints object, but the claimed wall-clock improvement still needs redacted benchmark, terminal, or log proof.

## Solution Assessment

No as written. The private accumulator is the right fix shape, but the public wrapper should clone once before mutation so the traversal becomes O(N) without changing the helper’s observable non-mutating behavior.
