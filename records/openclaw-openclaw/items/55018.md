## Summary

Keep open: current main still has the recursive shallow-copy behavior, but this PR is not merge-ready because it changes the exported helper to mutate caller-owned hints, is currently conflicting, and still lacks real behavior proof.

## What This Changes

The branch refactors `mapSensitivePaths` in `src/config/schema.hints.ts` to use a private in-place accumulator helper and adds JSDoc documenting the mutation behavior.

## Best Possible Solution

Land a rebased patch that keeps traversal O(N) by using a private accumulator while cloning the input once at the public boundary, backed by redacted terminal benchmark or live config-schema timing proof.

## Reproduction Assessment

No measured current-main reproduction is attached. Source inspection does show the current traversal repeatedly clones a growing hints object while walking `OpenClawSchema`, so the performance claim is source-reproducible but still needs benchmark proof.

## Solution Assessment

No as written. The private accumulator is the right fix shape, but the public wrapper should clone once before mutation so the traversal becomes O(N) without changing the helper's observable non-mutating behavior.
