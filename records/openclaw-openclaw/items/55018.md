## What This Changes

The PR changes `src/config/schema.hints.ts` so `mapSensitivePaths` uses a private in-place accumulator with JSDoc instead of recursively shallow-copying the hints object.

## Best Possible Solution

Land the in-place traversal optimization after the contributor adds redacted runtime or benchmark proof and maintainers rerun focused schema-hints plus current changed-gate validation.

## Solution Assessment

Is this the best way to solve the issue? Yes for the code approach: a private in-place accumulator removes repeated shallow copies while the main production caller passes fresh hints and the exported mutation behavior is documented.
