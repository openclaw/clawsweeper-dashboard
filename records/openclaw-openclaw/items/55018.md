## Summary

Keep open: current main still has the recursive shallow-copy traversal, but this PR is not merge-ready until it preserves the exported helper's non-mutating behavior, rebases cleanly, and includes real behavior or benchmark proof.

## What This Changes

The PR changes `mapSensitivePaths` to delegate recursion to a private in-place mutator so schema-sensitive hint traversal avoids repeated shallow copies.

## Best Possible Solution

Keep the private accumulator idea, but have the exported wrapper clone `hints` once before mutation, add a focused immutability regression test, rebase, and attach redacted benchmark or terminal proof for full schema traversal.

## Reproduction Assessment

No measured runtime reproduction is attached. Source inspection does show current main repeatedly clones a growing `hints` object while traversing `OpenClawSchema`, so the performance issue is source-reproducible but the reported 30s timing still needs benchmark proof.

## Solution Assessment

No as written. The private accumulator is the right fix shape, but the exported wrapper should clone once before mutation so traversal becomes O(N) without changing the helper's observable non-mutating behavior.
