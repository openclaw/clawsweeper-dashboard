## What This Changes

The PR changes `src/config/schema.hints.ts` so `mapSensitivePaths` uses a private in-place accumulator with JSDoc instead of shallow-copying the hints object during recursive Zod schema traversal.

## Best Possible Solution

Land the in-place traversal optimization after the contributor adds redacted runtime or benchmark proof and maintainers rerun focused schema-hints plus changed-gate validation.

## Reproduction Assessment

Yes for the source-level path: current main copies the growing hints object at each recursive node in `mapSensitivePaths`. I did not benchmark the reported 30s wall-clock delay, so the runtime magnitude remains unverified.

## Solution Assessment

Yes for the code approach: a private in-place accumulator removes repeated shallow copies while the main production caller passes fresh hints and the exported mutation behavior is documented. The remaining blocker is proof, not a code-design defect.
