## What This Changes

Against current main, this PR changes `mapSensitivePaths` to clone the hints map once, use a private recursive accumulator, and test that caller-owned hints are not mutated.

## Best Possible Solution

Land the narrow accumulator refactor after exact-head merge checks finish, keeping the public copy-return behavior and the caller-immutability regression test.

## Reproduction Assessment

Yes, at source level: current main clones the growing hints map at each recursive `mapSensitivePaths` call. I did not rerun a wall-clock benchmark in this read-only checkout, but the PR includes maintainer-provided benchmark and Crabbox proof.

## Solution Assessment

Yes. Cloning once at the public boundary and mutating only a private accumulator is the narrowest maintainable fix because it removes repeated allocation without changing caller-visible copy semantics.
