## Summary

## What This Changes

## Best Possible Solution

Keep the PR open, preserve `mapSensitivePaths` clone-on-entry behavior while using a private mutating accumulator internally, rebase it, and add redacted terminal benchmark or live config initialization proof.

## Reproduction Assessment

No, not for the reported ~30s wall-clock symptom. Source inspection does show current main cloning a growing hints map on every recursive schema node, so the algorithmic issue is source-reproducible but still needs measured proof.

## Solution Assessment

No as written. The private mutating accumulator is the right fix shape, but the public wrapper should clone once before recursion so the traversal becomes O(N) without changing the helper's observable non-mutating behavior.
