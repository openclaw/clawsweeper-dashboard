## Best Possible Solution

Keep the O(N) accumulator approach, but have the public wrapper clone once before invoking the private mutating traversal, rebase onto current main, and add focused immutability coverage plus redacted benchmark or terminal proof for the config-schema path.

## Solution Assessment

No, not as written. The private mutating accumulator is the right fix shape, but the public wrapper should clone once before recursion so traversal becomes O(N) without changing caller-visible mutation behavior.
