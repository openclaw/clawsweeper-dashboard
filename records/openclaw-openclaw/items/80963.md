## Summary

Keep open: current main still lets the upstream OpenAI Codex OAuth helper expose the browser authorization URL after the localhost:1455 listener fails, so the misleading occupied-port callback path remains source-reproducible. The later targeted PR had useful proof but closed unmerged after review found it needed to preserve manual URL exposure.

## What This Changes

Make OpenAI Codex OAuth handle an occupied localhost:1455 callback port by avoiding the misleading automatic browser callback and switching immediately to clear manual-entry diagnostics.

## Best Possible Solution

Add a local-only localhost:1455 availability preflight in the OpenAI provider path, show clear conflict diagnostics and the manual authorization URL, and prompt immediately without changing the fixed redirect URI or remote/VPS behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: occupy 127.0.0.1:1455 and run `openclaw models auth login --provider openai-codex`; the upstream helper still calls `onAuth` after bind failure, so OpenClaw treats the browser flow as started and waits before manual entry.

## Solution Assessment

Yes. The narrow maintainable fix is a plugin-owned local port preflight or equivalent occupied-port branch that avoids automatic browser opening, preserves manual URL exposure, and keeps the registered redirect URI unchanged.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/80963. In the OpenAI Codex OAuth wrapper, handle local mode with 127.0.0.1:1455 already occupied before the automatic browser opener runs. Add or reuse a local port probe for localhost:1455, but do not make the callback port configurable and do not change token exchange, refresh, remote/VPS behavior, TLS preflight behavior, or the fixed OpenAI redirect URI. On EADDRINUSE, explain that the callback port is in use, include `lsof -nP -iTCP:1455 -sTCP:LISTEN`, mention that `Invalid or expired OAuth state` can be caused by the wrong local process handling the callback, do not call `openUrl`, and prompt for manual redirect/code immediately. Preserve a visible/logged authorization URL for manual opening; this was the blocker in https://github.com/openclaw/openclaw/pull/80971. Mirror the behavior in the bundled OpenAI runtime copy or refactor only if the plugin SDK boundary remains valid. Add focused regression coverage for occupied-port behavior, no openUrl call, immediate manual prompt, diagnostic text, manual URL availability, normal free-port local flow, and remote/VPS skip. Add a changelog entry because this is user-visible auth UX.
