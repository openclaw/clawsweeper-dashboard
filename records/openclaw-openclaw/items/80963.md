## Summary

Keep open: current main and the latest v2026.5.18 release still expose the OpenAI Codex browser authorization URL after the local callback bind can fail on localhost:1455, so the confusing busy-port path remains source-reproducible. The exact-scope fixing PR was closed unmerged, and the open related callback-settled issue tracks remote prompt lifecycle rather than this local-port conflict.

## What This Changes

The issue asks OpenClaw to make an occupied OpenAI Codex localhost:1455 callback port clear, avoid opening the browser flow to the wrong local listener, and prompt for manual OAuth entry immediately.

## Best Possible Solution

Add an OpenAI-provider-local occupied-port guard that logs clear diagnostics and a manual-open URL, suppresses openUrl only when localhost:1455 is busy, and prompts immediately without changing the fixed redirect URI or remote/VPS flow.

## Reproduction Assessment

Yes. Occupy 127.0.0.1:1455 and run `openclaw models auth login --provider openai-codex`; source and dependency inspection show the helper still calls onAuth after bind failure and OpenClaw then opens/logs the browser URL before delayed manual fallback.

## Solution Assessment

Yes. The narrow maintainable fix is a provider-owned busy-port preflight or onAuth guard that preserves manual URL visibility and immediate paste entry; a configurable callback port is not needed.
