## Summary

Keep open: current main still has the source-reproducible OpenAI Codex OAuth port-conflict UX bug, and the linked fix at https://github.com/openclaw/openclaw/pull/80971 references this issue but is still open and unmerged.

## What This Changes

The issue asks OpenClaw to make OpenAI Codex OAuth localhost:1455 port conflicts explicit and skip the misleading browser callback path in favor of immediate manual paste guidance.

## Best Possible Solution

Land a narrow OpenAI-plugin-owned fix that preflights localhost:1455 locally, suppresses the browser opener on EADDRINUSE, shows the diagnostic command, and prompts for manual redirect/code immediately while preserving remote and normal local OAuth behavior.

## Reproduction Assessment

Yes. Occupy 127.0.0.1:1455 and run `openclaw models auth login --provider openai-codex`; source inspection shows the pinned helper still calls onAuth after bind failure, OpenClaw opens/logs the browser URL, and manual paste waits on the 15 second fallback path.

## Solution Assessment

Yes. The narrow maintainable fix is a local port availability check before invoking the browser opener, with immediate manual paste guidance on EADDRINUSE; a configurable callback port is not needed for this report.
