## Summary

Keep this PR open, but not merge-ready: it changes a documented opt-in cron failure-alert default into default-on notifications, so maintainers need to approve that policy change and the contributor still needs real after-fix alert proof.

## What This Changes

The PR changes cron failure-alert resolution so recurring jobs alert by default when no job or global failure-alert config is set, and adds a unit test for that default path.

## Best Possible Solution

Either keep the documented opt-in default, or have maintainers explicitly approve opt-out alerts and land docs/upgrade alignment plus real cron alert proof.

## Reproduction Assessment

Yes for the source-level behavior: current main returns null for unset job/global failure-alert config, so alerts do not fire by default. The same default is documented as false, so the PR is changing policy rather than fixing a documented contract bug.

## Solution Assessment

No, not as submitted. The resolver and test are narrow, but changing default-on notifications needs maintainer approval, docs and upgrade alignment, and after-fix real behavior proof.
