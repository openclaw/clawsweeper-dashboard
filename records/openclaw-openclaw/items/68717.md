## What This Changes

The PR changes cron failure-alert resolution so unset job/global alert config alerts by default and adds a unit test for repeated failures with no failureAlert config.

## Best Possible Solution

Land a revised branch only after maintainer confirmation, keeping the narrow runtime guard and default-path test while updating public docs and changelog for the new default.

## Reproduction Assessment

Yes. On current main, the cron service failure-alert seam can run an isolated recurring job twice with no job-level failureAlert and no global enabled=true, and resolveFailureAlert returns null so sendCronFailureAlert is never reached.

## Solution Assessment

No, not as submitted. The code and unit test are the narrow runtime fix if default-on alerts are intended, but the complete solution must align docs/changelog or keep the default-off gate.
