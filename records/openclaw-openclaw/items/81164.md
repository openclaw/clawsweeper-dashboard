## Summary

This PR should stay open for normal review: the additive context-engine seam is plausible, but the current patch still has a source-backed runtime blocker, lacks public contract docs, and the supplied proof does not show the changed behavior running.

## What This Changes

The branch adds an optional `ContextEngine.interceptCompaction` contract, an `interceptsCompaction` flag, Pi `session_before_compact` hook wiring, reserve-token-floor handling, SDK exports, and focused tests.

## Best Possible Solution

Land a documented additive compaction-intercept seam that preserves default behavior while keeping Pi threshold compaction enabled for engines that opt into interception.

## Reproduction Assessment

Yes for the PR blockers: source inspection of the PR head plus `pi-coding-agent` 0.74.0 shows the both-flags intercept path can be registered while the needed auto-compaction event is disabled. It is not a current-main bug reproduction because this is an additive feature PR.

## Solution Assessment

No; the seam is plausible, but this implementation needs the both-flags guard semantics fixed and the plugin-facing contract documented before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Keep Pi auto-compaction enabled for intercepting owners:** `src/agents/pi-embedded-runner/extensions.ts:196-201`
  - body: This gate now registers engines that set both `ownsCompaction` and `interceptsCompaction`, but the active run path still passes the same engine info into `applyPiAutoCompactionGuard`, whose predicate disables Pi compaction whenever `ownsCompaction` is true. In `pi-coding-agent` 0.74.0, disabled compaction returns before `session_before_compact`, so the motivating both-flags engine never receives the threshold event. Please update the guard semantics and add active run-path coverage.
  - confidence: 0.94
- **[P2] Document the compaction intercept SDK contract:** `src/plugin-sdk/index.ts:121-122`
  - body: The branch exports plugin-facing `CompactionIntercept*` types and adds `info.interceptsCompaction`, but the context-engine docs on the PR head still describe only `ownsCompaction` and `compact()`. Plugin authors need the opt-in flag, request/result shape, fallback behavior, throw behavior, and interaction with `ownsCompaction` documented with this SDK change.
  - confidence: 0.88
