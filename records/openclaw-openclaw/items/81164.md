## Summary

Keep open. This PR adds a plausible context-engine plugin seam and is paired with the lossless-claw implementation, but the current patch has a blocking safeguard-mode ordering bug and only documents unit-test proof rather than a real runtime compaction path.

## What This Changes

The PR adds an optional `ContextEngine.interceptCompaction()` contract, wires a Pi `session_before_compact` extension for active context engines, threads engine info into compaction settings, and adds focused unit coverage.

## Best Possible Solution

Keep the PR open, compose the intercept and safeguard paths so an intercepting engine runs before any fallback summarization or cancel path, then validate a real Pi compaction with the paired lossless-claw plugin before API approval.

## Reproduction Assessment

Yes, from source inspection. Configure safeguard compaction with an intercepting context engine and a safeguard summarization failure; Pi's sequential before-event runner returns on the safeguard `{ cancel: true }`, so the intercept handler registered afterward is never called.

## Solution Assessment

No. Registering intercept after safeguard preserves last-truthy override on success, but it runs the fallback work first and lets safeguard cancellation block the new contract; a composed handler should try intercept first and only run safeguard/default when the engine declines.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Run the intercept before safeguard can cancel:** `src/agents/pi-embedded-runner/extensions.ts:189-193`
  - body: In safeguard mode this registers the intercept after `compactionSafeguardExtension`, but Pi executes before-event handlers sequentially and returns immediately when an earlier handler returns `{ cancel: true }`. The safeguard handler can return cancel when its summarization fails, so an intercepting context engine never gets a chance to provide its lossless compaction in that path; even successful safeguard runs spend the fallback summary before being overridden. Please compose these paths so intercept is attempted first, and only declined intercepts run safeguard/default fallback.
  - confidence: 0.9
