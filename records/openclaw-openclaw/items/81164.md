## Summary

Keep open: the seam is useful and current main does not have it, but this PR is not merge-ready because the advertised both-flags path still disables the Pi auto-compaction event, plugin-author docs are not updated, and the supplied proof does not show a real interception run.

## What This Changes

The PR adds an optional `ContextEngine.interceptCompaction` contract, Pi `session_before_compact` hook wiring, reserve-token floor adjustments, SDK exports, and focused tests.

## Best Possible Solution

Land a documented additive compaction-intercept seam after the both-flags guard semantics are fixed and a redacted live or equivalent paired-deploy run proves the hook replaces native compaction.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an additive plugin SDK/runtime feature PR. The main blocker is source-reproducible from the PR head plus the pi-coding-agent 0.74.0 contract: disabled compaction skips the hook before `session_before_compact` can fire.

## Solution Assessment

No; the proposed seam is a plausible place to solve the downstream need, but this implementation must keep the Pi threshold event enabled for `ownsCompaction && interceptsCompaction` engines and document the public plugin contract before merge.

- **[P2] Keep auto-compaction enabled for intercepting owners:** `src/agents/pi-embedded-runner/extensions.ts:196-201`
  - body: This registers engines that set both `ownsCompaction` and `interceptsCompaction`, but `runEmbeddedAttempt` still feeds that same info to `applyPiAutoCompactionGuard`, whose current predicate disables Pi auto-compaction whenever `ownsCompaction` is true. Please update the guard semantics and add an active run-path regression test.

Plugin authors need the opt-in flag, request/result shape, fallback behavior, and interaction with `ownsCompaction` documented with the SDK change.
  - confidence: 0.88
