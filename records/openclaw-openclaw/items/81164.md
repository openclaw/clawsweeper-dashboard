## Summary

Keep open: the proposed seam is plausible and has an active lossless-claw consumer, but the PR still disables the Pi threshold path for the motivating both-flags engine, lacks public SDK documentation, and the supplied proof does not show an actual runtime interception.

## What This Changes

The PR adds an optional `ContextEngine.interceptCompaction` contract, Pi `session_before_compact` hook wiring, reserve-token floor handling, SDK exports, and focused tests.

## Best Possible Solution

Land a documented additive compaction-intercept seam after the both-flags auto-compaction guard semantics are fixed, active run-path coverage is added, and redacted paired runtime proof shows the hook replacing native compaction.

## Reproduction Assessment

Yes for the PR blocker: source inspection of the PR head plus `pi-coding-agent` 0.74.0 shows disabled auto-compaction returns before `session_before_compact` can fire. It is not a current-main bug reproduction because this is an additive feature PR.

## Solution Assessment

No; the seam is plausible, but this implementation needs to keep the Pi threshold event reachable for `ownsCompaction && interceptsCompaction` engines and document the plugin contract before merge.
