## Summary

Keep this PR open: the additive context-engine intercept API is plausible, but the current patch still awaits third-party plugin code directly on Pi's auto-compaction path and the supplied real-environment proof has not shown a handled `session_before_compact` event.

## What This Changes

The PR adds an optional `ContextEngine.interceptCompaction` contract, wires it into Pi `session_before_compact`, updates compaction settings/docs, and adds focused tests.

## Best Possible Solution

Land the additive intercept API only after the plugin callback is bounded by the existing compaction timeout/abort policy and redacted live output shows an opted-in engine handling a real `session_before_compact` event.

## Solution Assessment

No as currently proposed: the API shape is plausible, but the implementation should bound the plugin callback with the same timeout policy used for plugin-owned compaction before merge.
