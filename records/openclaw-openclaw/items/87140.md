## Summary

Keep open: current main and latest release still hard-wire macOS Push-to-Talk to Apple Speech and only forward recognized text, while the existing `tools.media.audio` and Talk transcription surfaces remain separate Gateway/provider paths. The request is valid but needs maintainer product/API review rather than conservative cleanup or an automated fix PR.

## What This Changes

The issue requests configurable STT backends for native macOS Push-to-Talk, including Gateway media-audio or private/local ASR, while keeping Apple Speech as the default.

## Best Possible Solution

Design a narrow macOS PTT backend selector that keeps Apple Speech as the default, reuses Gateway media-understanding or Talk transcription where appropriate, and explicitly documents config, permissions, privacy, and failure-state behavior.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source, docs, and latest-release inspection confirm native macOS PTT is Apple Speech-backed and not configurable through `tools.media.audio`.

## Solution Assessment

Unclear until maintainers choose the product/API shape. Reusing existing Gateway media-understanding or Talk transcription surfaces looks preferable to a separate macOS-only ASR protocol, but config, privacy, and overlay-state behavior need owner review.
