## Summary

Keep open. Current main has channel/webhook/none cron delivery and limited main-session awareness, but it does not implement a configurable `output.targets` contract for file, channel fan-out, session injection, silent mode, or fallback targets; the related channel fan-out work in https://github.com/openclaw/openclaw/pull/44859 only covers part of the request and still needs maintainer direction.

## What This Changes

The issue requests a cron job `output` configuration that deterministically routes completed agent responses to file, channel, session, or silent targets.

## Best Possible Solution

Design one canonical cron output contract that either extends `delivery` or adds `output.targets`, with explicit semantics for file writes, channel fan-out, session injection, silent discard, fallback failure handling, status traces, CLI/UI/docs, and how it supersedes or incorporates https://github.com/openclaw/openclaw/pull/44859.

## Solution Assessment
