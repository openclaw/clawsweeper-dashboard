## Summary

Keep open. Current main and v2026.6.6 still expose only the existing single-route cron delivery contract, so deterministic file/channel/session output routing remains unimplemented and needs product/API design before a repair PR.

## What This Changes

Request a first-class cron output target config that routes completed job output to files, channels, sessions, or silent discard independent of model prompt compliance.

## Best Possible Solution

Define one canonical cron output-routing contract covering schema, persistence, file safety, channel delivery, session transcript/context semantics, fallback/status reporting, CLI/UI/docs, and regression tests.
