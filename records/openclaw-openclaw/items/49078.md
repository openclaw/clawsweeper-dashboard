## Summary

Keep open. Current main and the latest shipped release still expose only the existing cron `delivery` modes and a single delivery route; the open related PR covers channel fan-out only, while this issue asks for a broader file/channel/session output-routing contract that still needs maintainer API/product review.

## What This Changes

Request adding a first-class cron `output.targets` config for deterministic file, channel, session, and silent result routing after a cron run completes.

## Best Possible Solution

Design one canonical cron output-routing contract that keeps schema, normalization, runtime dispatch, file safety, channel delivery, session transcript behavior, fallback/status traces, CLI/UI, docs, and tests aligned.
