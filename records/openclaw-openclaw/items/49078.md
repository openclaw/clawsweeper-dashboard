## Summary

Keep open. Current main still has only the existing single-route cron delivery contract, while this request asks for a broader public output-routing API covering file targets, channel target lists, session injection, silent discard, and fallback semantics; the related open fan-out PR is narrower and does not supersede the issue.

## What This Changes

The issue requests a first-class cron output-routing config that can deterministically route completed cron output to files, channels, sessions, or silent discard targets.

## Best Possible Solution

Design one canonical cron output-routing contract that aligns schema, normalization, runtime dispatch, file safety, channel fan-out, session transcript behavior, fallback/status traces, CLI/UI, docs, and tests.

## Solution Assessment

Unclear as proposed. The user problem is valid, but maintainers need to decide whether the durable API should be a broad `output.targets` contract, an extension of `delivery`, or a smaller sequence of file/channel/session features.
