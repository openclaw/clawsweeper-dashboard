## Summary

Keep open. Current main still lacks a first-class cron output-routing contract for deterministic file writes, target lists, named-session delivery, and fallback semantics; related work covers narrower channel fan-out or awareness behavior but does not replace the broader API/product decision.

The issue requests configurable cron job output targets for files, messaging channels, sessions, silent discard, and fallback delivery independent of prompt compliance.

## Best Possible Solution

Design one canonical cron output-routing contract that aligns schema, normalization, runtime dispatch, file safety, channel fan-out, session transcript behavior, fallback/status traces, CLI/UI, docs, and tests.
