## Summary

Keep open. Current main and the latest release still expose the existing cron delivery contract, while this issue asks for a broader deterministic output API covering file writes, channel targets, session injection, silent output, fallback behavior, and status semantics.

## What This Changes

The issue requests deterministic cron output targets for files, channels, sessions, and silent runs so cron results do not rely only on prompt instructions.

## Best Possible Solution

Design one canonical cron output-routing contract that aligns schema, SQLite persistence, runtime dispatch, file path safety, channel delivery, session transcript behavior, failure/status reporting, CLI/UI, docs, and regression tests.
