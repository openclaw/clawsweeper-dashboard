## Summary

Keep open. Current main still only supports cron delivery modes for none, announce, and webhook, with one resolved chat/webhook destination plus failure notifications; the requested configurable output targets for file writes, channel fan-out, session injection, silent discard, and fallbacks remain a new product/API surface.

## What This Changes

Add a cron output routing contract so final agent responses can be deterministically routed to configured file, channel, session, or silent targets without relying on prompt instructions.

## Best Possible Solution

Design one canonical cron output-routing contract that either extends delivery or introduces output.targets, with typed semantics for file writes, channel fan-out, session injection, silent discard, fallback behavior, status traces, CLI/UI/docs, and migration from the open channel fan-out proposal.

## Reproduction Assessment

Not applicable. This is a feature/config request, and source inspection confirms the requested output-routing contract is absent rather than a broken existing behavior with an already-defined repro path.

## Solution Assessment

Unclear as proposed. The user problem is valid, but maintainers should first choose whether the durable API is output.targets or an extension of delivery, because the current open channel fan-out work covers only one slice of the request.
