## Summary

Keep open. Current main still has only a single cron delivery route and no first-class output-target contract for files, channel target lists, named sessions, silent discard, or fallback semantics; the related fan-out PR is narrower and still open, so this remains a product/API design request.

## What This Changes

The issue requests a cron job `output` config surface that deterministically routes final job output to declared file, channel, session, or silent targets.

## Best Possible Solution

Design one canonical cron output-routing contract that aligns schema, normalization, runtime dispatch, file safety, channel fan-out, session transcript behavior, fallback/status traces, CLI/UI, docs, and tests.
