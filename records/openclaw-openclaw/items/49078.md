## Summary

Keep open. Current main still lacks a first-class cron output-routing contract for deterministic file output, channel target lists, session injection, silent mode, and fallback/status semantics; the related channel fan-out PR is narrower and still open.

The issue requests a cron job output-routing config that can route completed job results to files, messaging channels, sessions, or discard them without relying on prompt instructions.

## Best Possible Solution

Design one canonical cron output-routing contract that aligns schema, normalization, runtime dispatch, file safety, channel fan-out, session transcript behavior, fallback/status traces, CLI/UI, docs, and tests.
