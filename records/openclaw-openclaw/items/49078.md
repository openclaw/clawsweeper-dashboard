## Summary

Keep open. Current main and v2026.6.6 still expose the existing single-route cron delivery contract, while this issue asks for a broader deterministic output API covering file writes, channel targets, session injection, fallback behavior, and status semantics.

## What This Changes

The issue requests a first-class cron `output.targets` contract for deterministic routing of job results to files, channels, and sessions.

## Best Possible Solution

Design one canonical cron output-routing contract that aligns schema, SQLite persistence, runtime dispatch, file path safety, channel delivery, session transcript behavior, failure/status reporting, CLI/UI, docs, and regression tests.

## Solution Assessment

Unclear as proposed. The user problem is concrete, but maintainers should first choose whether the durable API is broad `output.targets`, an extension of `delivery`, or coordinated narrower file/channel/session features.
