## Summary

Keep open: current main still has a single cron `delivery` contract with `none`, `announce`, and `webhook`, while the requested file, multi-target, and configurable session output routing remains unimplemented. A related closed fix covers only next-turn main-session awareness for successful isolated direct deliveries, not the broader `output.targets` feature requested here.

## What This Changes

Allow cron jobs to declare deterministic output targets such as files, channels, and sessions instead of relying on prompt text or a single delivery mode.

## Best Possible Solution

Keep this as a maintainer design item for a versioned cron output-routing contract that aligns schema, CLI/tool docs, delivery semantics, file safety, and session-awareness behavior.

## Reproduction Assessment

Not applicable as a bug reproduction. Source and docs inspection provide high-confidence evidence that current main lacks the requested `output.targets` feature while preserving only the existing delivery modes and a narrower awareness path.

## Solution Assessment

Unclear. A config-level output contract is plausible, but the proposed multi-target shape needs maintainer decisions around file write safety, delivery fallback semantics, and session injection before it is the best maintainable solution.
