## Summary

Keep open. Current main still exposes cron results through the existing single `delivery` route, while the requested deterministic `output.targets` fan-out to files, channels, sessions, and silent handling is not implemented and needs product/security design.

## What This Changes

Request adding a cron job output target contract for deterministic file, channel, session, and silent routing of final job results.

## Best Possible Solution

Design an additive schema-backed cron output contract that validates target types, records per-target status, defines mixed-success behavior, and updates runtime, CLI, Control UI, protocol docs, and focused tests.

## Reproduction Assessment

Not applicable as a feature request. The missing capability is source-verifiable on current main because cron schemas, runtime planning, CLI/UI, and docs expose only the existing `delivery` path.

## Solution Assessment

Unclear whether the proposed `output` field is the best shape versus extending `delivery`. The maintainable direction is a narrow additive public contract with explicit security rules rather than relying on prompt instructions.
