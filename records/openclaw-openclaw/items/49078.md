## Summary

Keep open. Current main still lacks a first-class cron output-routing contract for deterministic file writes, multiple declared targets, named-session injection, and fallback/status semantics; the related open PR only covers narrower channel fan-out and still needs maintainer API selection.

## What This Changes

Request adding a cron job `output` target contract so final cron responses can be routed deterministically to files, channels, sessions, or discarded silently.

## Best Possible Solution

Design one canonical cron output-routing contract that aligns schema, normalization, runtime dispatch, file safety, channel fan-out, session transcript behavior, fallback/status traces, CLI/UI, docs, and tests.

## Reproduction Assessment

Not applicable. This is a new public cron configuration feature, and current-main source inspection shows the requested output-routing contract is absent rather than a broken existing documented behavior.

## Solution Assessment

Unclear as proposed. The user problem is valid, but maintainers should first choose whether the durable API is a broad `output.targets` contract, an extension of `delivery`, or separate channel/file/session features.
