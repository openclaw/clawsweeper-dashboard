## Summary

Keep open: current main supports cron fallback delivery to chat and webhook callbacks, but it does not implement the requested first-class `output` field, file targets, session injection target, or multiple output targets. This is a real feature/config design request rather than a broken existing contract.

## What This Changes

Add cron job configuration for deterministic output routing to file, channel, session, or silent targets instead of relying on prompt instructions.

## Best Possible Solution

If maintainers want this capability, design it as a core cron output contract that covers schema/RPC/CLI/tool/UI/docs, safe file path handling, session injection semantics, and multi-target delivery failure behavior.

## Reproduction Assessment

Not applicable. This is a feature request for a new cron output contract, and current source/docs show the existing behavior is limited to delivery modes rather than file/session/multi-target output routing.

## Solution Assessment

Unclear. The proposed `output` field is a plausible direction, but the best implementation needs maintainer design first because it crosses scheduler config, protocol validation, outbound delivery, filesystem safety, and session persistence.
