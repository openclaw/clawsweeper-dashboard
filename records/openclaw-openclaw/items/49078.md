## Summary

Keep open. Current main still lacks a first-class cron output target contract for deterministic file writes, channel target lists, session injection, and fallback/status semantics; the open channel fan-out PR covers only one part and does not supersede the broader API/product decision.

## What This Changes

The issue requests a new cron job output-routing contract so scheduled agent results can be deterministically written to files, sent to channels, injected into sessions, or discarded independently of prompt compliance.

## Best Possible Solution

Design one canonical cron output-routing contract that aligns schema, normalization, runtime dispatch, file safety, channel fan-out, session transcript behavior, fallback/status traces, CLI/UI, docs, and tests.
