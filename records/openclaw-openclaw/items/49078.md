## Summary

Keep open. Current main and v2026.6.10 still expose only the existing single cron delivery policy (`none`, `announce`, `webhook`); related merged work improves awareness after existing channel delivery, but the requested deterministic file, channel fan-out, session, and discard output targets remain unimplemented.

## What This Changes

Add a first-class cron `output.targets` contract for file, channel, session, and silent/discard routing independent of prompt compliance.

## Best Possible Solution

Maintainers should first design one cron output-routing contract covering schema/protocol, safe workspace file writes, channel fan-out, session insertion, partial failure/fallback/status behavior, docs, tests, and upgrade behavior.
