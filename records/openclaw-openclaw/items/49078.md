## Summary

Keep open. Current main still lacks a deterministic cron `output.targets` or equivalent contract for file output, channel fan-out, session injection, silent discard, and fallback target lists; the remaining work is a new public cron API/product surface rather than conservative cleanup.

## What This Changes

The issue requests a new cron job `output` config that hard-routes final job output to files, channels, sessions, or silent discard independent of prompt compliance.

## Best Possible Solution

Design one canonical cron output-routing contract that aligns delivery or `output.targets` across schema, normalization, runtime dispatch, status traces, CLI/UI/docs, channel fan-out, file writes, session injection, silent discard, and fallback behavior.

## Reproduction Assessment

Not applicable. This is a feature/config request, and current-main source inspection shows the requested output-routing contract is absent rather than a broken existing behavior with an already-defined repro path.

## Solution Assessment

Unclear as proposed. The user problem is valid, but maintainers should first choose whether the durable API is a broader `output.targets` contract, an extension of `delivery`, or narrower linked features for channel fan-out and session delivery.
