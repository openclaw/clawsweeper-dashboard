## Summary

Keep open. Current main still has only the existing single-route cron delivery contract, while this request asks for a broader public output-routing API spanning file writes, channel delivery, session injection, fallback/status semantics, and docs; the related open fan-out PR is narrower and does not supersede the feature request.

## What This Changes

The issue requests a first-class cron output-routing configuration that can send job results to declared file, channel, and session targets instead of relying on prompt instructions.

## Best Possible Solution

Design one canonical cron output-routing contract that aligns schema, normalization, runtime dispatch, file safety, channel fan-out, session transcript behavior, fallback/status traces, CLI/UI, docs, and tests.
