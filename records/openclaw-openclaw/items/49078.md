## Summary

Keep open. Current main and the latest release still expose the existing single-route cron delivery contract, while this issue asks for a broader deterministic output contract covering files, channels, sessions, fallback behavior, and status semantics.

## What This Changes

The issue requests a new cron job `output` configuration that can route scheduled job results deterministically to files, messaging channels, and main or named sessions.

## Best Possible Solution

Design one canonical cron output-routing contract that aligns schema, normalization, runtime dispatch, file safety, channel routing, session transcript behavior, failure/status reporting, CLI/UI, docs, and regression tests.
