## Summary

Keep open. Current main has channel/webhook-oriented cron delivery but no first-class `output.targets` contract for file writes, multiple targets, session injection, fallback ordering, or output formatting; the related open session-mirroring work covers only a narrower transcript-awareness subset.

## What This Changes

Add a cron job `output` config that can route final job results to files, channels, sessions, or a silent sink without relying on prompt instructions.

## Best Possible Solution

Design a core cron output contract that reconciles `delivery`, webhook events, session mirroring, and safe workspace file writes before adding schema, CLI, docs, and focused regression coverage.

## Reproduction Assessment

Not applicable. This is a feature/config request, and source inspection confirms current cron contracts do not include the requested output target schema.

## Solution Assessment

Unclear until maintainers choose the product/API contract. The session-awareness subset should continue through https://github.com/openclaw/openclaw/pull/74913, while file and multi-target routing need a broader design decision.
