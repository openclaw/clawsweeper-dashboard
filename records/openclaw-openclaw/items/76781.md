## Summary

Keep open: current main still has the partial plugin-tool registry reuse behavior, and the open PR is the right narrow fix but needs small follow-up before merge.

## What This Changes

The PR changes plugin tool registry reuse to require tool entries for every requested plugin id and adds a wildcard allowlist regression test.

## Best Possible Solution

Merge the narrow registry-coverage fix after tightening the regression test and adding the required Unreleased/Fixes changelog entry.

## Reproduction Assessment

Yes. Source inspection on current main shows `registryHasScopedPluginTools` accepts a registry when any requested plugin id has tools, and `resolvePluginTools` then skips the cold-load path for missing selected plugin tools.

## Solution Assessment

Yes for the source fix. Requiring every selected plugin id to appear in `registry.tools` before reuse is narrow and maintainable, but the regression test should keep the compatible-registry mock partial so it fails on current main.
