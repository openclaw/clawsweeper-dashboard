## Summary

Keep this PR open. The reported plugin tool-discovery bug is valid, but the patch is likely incomplete because the standalone cold-load helper can still return the same compatible active registry based on plugin-id coverage before `loadOpenClawPlugins` runs, leaving a plugin-complete but tool-partial registry in use.

## What This Changes

The PR tightens plugin tool registry reuse from any requested plugin id to every requested plugin id and adds a regression test for wildcard-selected plugin tools with a partial active registry.

## Best Possible Solution

Teach the plugin tool resolver or standalone loader to require selected plugin-id tool-entry coverage before reusing any runtime registry, and cover the plugin-complete but tool-partial active-registry case with a focused regression test.

## Reproduction Assessment

Yes from source inspection: a compatible active registry whose `plugins` list covers all selected ids but whose `tools` list only covers one selected plugin can still be returned before a fresh tool-discovery load. I did not execute the Vitest case because the review must stay read-only and `pnpm` is unavailable.

## Solution Assessment

No. Tightening `registryHasScopedPluginTools` is directionally correct, but the cold-load path also needs to avoid reusing the same tool-partial registry solely because registry-level plugin ids match.
