## Summary

Keep open: current main still has the partial-registry reuse bug, and this PR is a narrow source/test/changelog fix for the linked plugin tool discovery regression.

## What This Changes

The PR requires reused plugin tool registries to cover every selected plugin id, force-loads incomplete standalone tool registries with cache bypass, adds a regression test, and adds a changelog entry.

## Best Possible Solution

Land this PR or an equivalent narrow fix after resolving mergeability and CI, then let the linked issue close through the PR merge.

## Reproduction Assessment

Yes. Source inspection on current main shows a multi-plugin scoped request can accept a registry once any requested plugin has tool entries, leaving another selected plugin's tools absent from discovery surfaces.

## Solution Assessment

Yes. Requiring every scoped plugin id to be represented in `registry.tools` before reuse, then force-loading when the active registry is incomplete, is the narrowest maintainable fix I found.
