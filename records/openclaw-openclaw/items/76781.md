## Summary

Keep open: current main still has the reported plugin tool discovery bug, and this PR is a narrow fix with regression coverage and no blocking review findings from this read-only pass.

## What This Changes

The PR requires reused plugin tool registries to cover every selected plugin id, force-loads incomplete standalone tool registries, adds a regression test, and adds a changelog fix entry.

## Best Possible Solution

Land this PR or an equivalent narrow fix after exact-head validation, then let the linked issue close through the merge.

## Reproduction Assessment

Yes. Source inspection on current main shows a multi-plugin scoped request can accept a registry once any selected plugin has tool entries, leaving another selected plugin's tools absent from discovery surfaces.

## Solution Assessment

Yes. Requiring every scoped plugin id to be represented in `registry.tools` before reuse, then force-loading when existing registries are incomplete, is the narrowest maintainable fix I found.
