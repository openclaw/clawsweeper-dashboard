## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve the source account and thread on requeue:** `src/auto-reply/reply/commands-session-interrupt.ts:82-90`
  - body: The requeued `agent` call sends `channel` and `to` but drops the originating `accountId` and `threadId`. Current delivery planning only gets those values from the request, so multi-account or threaded interrupts can resume through the default account or parent conversation.
  - confidence: 0.91
- **[P1] Do not force delivery for internal webchat interrupts:** `src/auto-reply/reply/commands-session-interrupt.ts:86-88`
  - body: For webchat/internal chat, this resolves the channel to the internal message channel while still setting `deliver: true`. The gateway rejects delivered requests that remain internal unless best-effort downgrade is requested, so the command can abort the run and then fail to start the redirected turn.
  - confidence: 0.84
- **[P2] Register /interrupt in the command registry:** `src/auto-reply/reply/commands-core.ts:206`
  - body: The handler is added to the text pipeline, but the builtin command registry has no `/interrupt` entry. Slack and Discord native commands are generated from that registry, so native-command installations will not expose the new command.
  - confidence: 0.88
- **[P2] Serialize thread-context store writes:** `src/sessions/thread-context-store.ts:137-141`
  - body: `saveThreadContext` performs an async read-modify-write without a per-file queue or lock. Overlapping cron deliveries can load the same JSON snapshot and the later write will drop the earlier context entry.
  - confidence: 0.86
- **[P2] Normalize Telegram chat keys before saving context:** `src/cron/isolated-agent/run.ts:942-946`
  - body: Cron saves `resolvedDelivery.to` as the chat key, but explicit Telegram targets are stored as bare chat ids while inbound replies use the `telegram:` target form, including topic routing. Those keys will not match, so prior-thread context is skipped for explicit Telegram thread deliveries.
  - confidence: 0.82
