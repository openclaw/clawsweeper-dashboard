## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Reuse the existing active-run control path:** `src/auto-reply/reply/commands-session-interrupt.ts:60-80`
  - body: Current main already has `/queue interrupt`, `/steer`, and gateway/session steering contracts for this class of behavior. This handler manually aborts and requeues through a new path, creating a second source of truth that already misses routing and delivery invariants.
  - confidence: 0.88
- **[P1] Preserve routing context when requeueing:** `src/auto-reply/reply/commands-session-interrupt.ts:80-90`
  - body: The new `/interrupt` path calls `agent` with only `channel` and `to`, dropping `accountId` and `threadId`. In multi-account or threaded channels, the redirected turn can resume from the default account or parent conversation instead of the original surface.
  - confidence: 0.91
- **[P1] Do not force delivered webchat requeues:** `src/auto-reply/reply/commands-session-interrupt.ts:86-88`
  - body: For webchat/internal sessions this always sets `deliver: true` even when the resolved channel is internal. The gateway can reject or misresolve the request after the active run has already been aborted, making webchat interrupts fail instead of redirecting the run.
  - confidence: 0.86
- **[P2] Register /interrupt in the command registry:** `src/auto-reply/reply/commands-core.ts:206`
  - body: The handler is added only to the text-command pipeline. Slack and Discord native commands are generated from the shared command registry, so installations using native commands will not expose `/interrupt` unless the registry and native-command coverage are updated too.
  - confidence: 0.88
- **[P2] Serialize thread-context store writes:** `src/sessions/thread-context-store.ts:137-141`
  - body: `saveThreadContext` performs an async read-modify-write with no per-file lock. Two overlapping cron deliveries can read the same snapshot and the later write will drop the earlier context entry.
  - confidence: 0.86
- **[P2] Canonicalize Telegram chat keys before saving context:** `src/cron/isolated-agent/run.ts:942-946`
  - body: Cron saves `resolvedDelivery.to` as the chat key, while inbound Telegram topic replies can use the in-band `telegram:<chat>:topic:<id>` route. Those keys do not match reliably, so explicit-topic follow-up replies can skip the stored prior context.
  - confidence: 0.84
