## Summary

Keep this PR open, but it is not safe to merge yet: the feature is not on current main, the branch still has source-proven delivery and persistence bugs, and the external contributor has not provided real behavior proof.

## What This Changes

The PR adds a `/interrupt` command that aborts and requeues an agent turn, plus a JSON-backed thread context store used by cron isolated-agent deliveries and new reply sessions.

## Best Possible Solution

Fix the delivery-context, registry, serialization, and Telegram keying bugs, then require real Telegram or equivalent messaging proof for `/interrupt` and cron-thread follow-up before merge.

## Reproduction Assessment

Yes for the review findings: the failing paths are source-reproducible from the PR head and current-main delivery contracts. I did not run the behavior because this was a read-only review and no real proof artifact was provided.

## Solution Assessment

No. The feature direction is plausible, but this implementation needs to preserve the exact originating delivery context, register the new command consistently, serialize store writes, and normalize provider-specific thread keys before it is the narrow maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve the source account and thread on requeue:** `src/auto-reply/reply/commands-session-interrupt.ts:82-90`
  - body: The requeued `agent` call sends `channel`/`to` but drops `params.ctx.AccountId` and `params.ctx.MessageThreadId`. The gateway only preserves those when `request.accountId` and `request.threadId` are supplied, so multi-account or threaded interrupts can resume from the default account or the parent conversation.
  - confidence: 0.91
- **[P1] Handle internal webchat without forcing delivery:** `src/auto-reply/reply/commands-session-interrupt.ts:86`
  - body: For webchat, this resolves `channel` to the internal message channel while still setting `deliver: true`. The agent handler rejects delivered requests that stay internal unless best-effort downgrade is requested, so an interrupt can abort the run and then fail to start the redirected turn.
  - confidence: 0.84
- **[P2] Register /interrupt in the command registry:** `src/auto-reply/reply/commands-core.ts:206`
  - body: The handler is added to the text pipeline, but the command is not added to `buildBuiltinChatCommands`. Slack and Discord deploy native commands from `listNativeCommandSpecsForConfig`, so native-command installations will not expose the new feature.
  - confidence: 0.88
- **[P2] Serialize thread-context store writes:** `src/sessions/thread-context-store.ts:136-141`
  - body: `saveThreadContext` reads the store, mutates the in-memory object, and writes it back without any lock or per-file queue. Concurrent cron deliveries can both read the same snapshot and the later write will drop the earlier context entry.
  - confidence: 0.86
- **[P2] Normalize Telegram chat IDs before saving context:** `src/cron/isolated-agent/run.ts:942-946`
  - body: Cron saves `resolvedDelivery.to` as the chat key, but explicit Telegram topic targets are stripped to a bare chat id while inbound replies use `OriginatingTo` with the `telegram:` prefix. The saved key will not match follow-up replies for those cron deliveries.
  - confidence: 0.82
