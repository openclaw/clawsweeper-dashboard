## Summary

Keep open: current main and the latest release still reject `timeoutSeconds: 0` in the Control UI and drop an explicit zero from the submitted cron payload, while the gateway/runtime contract accepts zero as no-timeout. The open linked PR at https://github.com/openclaw/openclaw/pull/41275 is the active closing path, so this issue should stay open until that PR lands or is replaced.

## What This Changes

Allow the cron Control UI to accept `timeoutSeconds: 0` as no-timeout and submit the explicit zero value.

## Best Possible Solution

Land or replace the linked PR with a complete Control UI fix that accepts and preserves explicit zero, rejects negative and malformed timeout text, updates the validation copy, and adds focused controller coverage.

## Reproduction Assessment

Yes, source-reproducible. Current main sends entered `0` through the `timeout <= 0` validation error path, and the payload builder only submits positive values while the gateway/runtime contract accepts zero.

## Solution Assessment

No, the validator-only proposed fix in the issue is incomplete. The maintainable fix also needs to preserve `timeoutSeconds: 0` in the submitted payload and update tests plus user-facing validation copy.
