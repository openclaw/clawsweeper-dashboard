Keep open: current main and the latest release still reject `timeoutSeconds: 0` in the Control UI cron form and omit explicit zero from the submitted payload, while the gateway schema and cron runtime already define zero as no-timeout. The same-author fix PR at https://github.com/openclaw/openclaw/pull/41275 remains open with closing syntax, so this issue should stay open until that PR lands or is replaced.

## What This Changes

The issue requests that the Control UI cron form accept `timeoutSeconds: 0` as no-timeout instead of rejecting it.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/41275 with a complete Control UI fix that accepts and preserves explicit zero, rejects invalid timeout input, and updates focused tests plus timeout copy.

## Reproduction Assessment

Yes. Source inspection against current main shows `timeoutSeconds: "0"` goes through the validation error path and explicit zero is omitted from the payload, while schema and runtime code accept zero as no-timeout.
