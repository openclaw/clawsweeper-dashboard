## What This Changes

The issue requests that cron job UI validation accept `timeoutSeconds: 0` as no-timeout mode.

## Best Possible Solution

Land or replace the linked UI fix so the form accepts zero, preserves `timeoutSeconds: 0` in the submitted agent-turn payload, and covers validation/submission with focused controller tests.

## Solution Assessment

No, the one-line validator-only fix described in the issue is not complete. The maintainable fix should also include `timeoutSeconds: 0` in `buildCronPayload` and add regression coverage for both validation and submitted payload.
