## Summary

Keep open: current main and v2026.5.20 still reject `timeoutSeconds: 0` in the Control UI cron validator and omit explicit zero from the payload, while the gateway schema and cron runtime already accept zero as the no-timeout value. The linked same-author implementation PR at https://github.com/openclaw/openclaw/pull/41275 remains open with closing syntax, so this issue should stay open until that PR lands or is replaced.

## What This Changes

Accept `timeoutSeconds: 0` in the Control UI cron form as no-timeout and submit that explicit zero in the cron payload.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/41275 with a complete Control UI fix that accepts and preserves explicit zero, rejects malformed and negative input, and updates focused tests plus timeout copy.

## Reproduction Assessment

Yes. Source inspection shows current main routes `timeoutSeconds: "0"` through the validation error path and drops explicit zero from payload construction, while schema and runtime policy already accept zero as no-timeout.

## Solution Assessment

No, not as the issue's validator-only sketch. The controller is the right boundary, but the maintainable fix must also preserve explicit zero in the payload, keep malformed input invalid, and update focused tests plus validation/help copy.
