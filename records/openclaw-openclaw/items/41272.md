## Summary

Keep open: current main and the latest release still reject `timeoutSeconds: 0` in the Control UI cron validator and omit explicit zero from the UI payload, while the gateway schema and cron runtime already treat zero as valid no-timeout. The same-author linked implementation PR is still open with closing syntax, so this issue should remain open until that PR lands or is replaced.

## What This Changes

The issue asks for the Control UI cron form to accept and submit `timeoutSeconds: 0` as no-timeout mode.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/41275 with a complete Control UI fix that accepts and preserves explicit zero, rejects malformed and negative input, and updates focused controller tests plus timeout copy if needed.

## Reproduction Assessment

Yes. Source inspection on current main shows `timeoutSeconds: "0"` reaches the validation error path and explicit zero is dropped from payload construction, while schema and runtime tests accept zero as no-timeout.

## Solution Assessment

No, not as the issue's validator-only sketch. The controller is the right boundary, but the maintainable fix must also preserve explicit zero in the payload and keep malformed or negative timeout input invalid with focused tests.
