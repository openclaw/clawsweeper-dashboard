## Summary

Keep open: current main and the latest release still reject explicit cron `timeoutSeconds: 0` in the Control UI and omit zero from the submitted payload, while the gateway schema and runtime timeout policy accept zero as no-timeout. The issue also has an open linked implementation PR, so it should remain the tracker until that PR lands or is replaced.

## What This Changes

Request that the cron Control UI accept `timeoutSeconds: 0` as no-timeout and preserve explicit zero in submitted agent-turn payloads.

## Best Possible Solution

Review, land, or replace https://github.com/openclaw/openclaw/pull/41275 with a complete controller fix that accepts and preserves explicit zero, rejects malformed and negative timeout input, and updates focused tests plus validation copy/locales.

## Reproduction Assessment

Yes, source-reproducible. Current main sends `timeoutSeconds: "0"` through the `timeout <= 0` validation error path and omits zero from payload construction, while schema and runtime policy already accept zero as no-timeout.

## Solution Assessment
