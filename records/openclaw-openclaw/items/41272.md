## Summary

Keep open: current main still rejects explicit `timeoutSeconds: 0` in the cron Control UI and omits zero from the submitted payload, while the gateway/runtime contract already accepts zero as no-timeout. The issue is paired with an open closing PR, so it should remain open until that PR lands or is replaced.

## What This Changes

The issue asks for the cron job UI to accept `timeoutSeconds: 0` as the no-timeout mode instead of showing a validation error.
