## What This Changes

Allow the cron job form to preserve and submit `timeoutSeconds: 0` as the existing no-timeout value.

## Best Possible Solution

Land a focused Control UI fix that accepts and serializes zero, continues rejecting negative and non-finite values, updates the source validation copy and generated locales, and proves the submitted payload plus no-timeout runtime behavior.

## Reproduction Assessment

Yes. Current main deterministically rejects the form value `0` in `validateCronForm` and omits it in `buildCronPayload`, while downstream source and tests establish zero as no timeout.

## Solution Assessment

Yes in direction, but the issue’s original validation-only snippet is not sufficient; the best fix must also preserve zero in the payload and update tests and UI copy.
