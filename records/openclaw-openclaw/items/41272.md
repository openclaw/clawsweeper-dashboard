## Summary

Keep open: current main and v2026.5.12 still reject or drop explicit `timeoutSeconds: 0` in the Control UI, while the gateway/runtime contract already accepts zero as no-timeout. The linked open PR at https://github.com/openclaw/openclaw/pull/41275 is still the active closing path, so this issue should remain open until that PR lands or is replaced.

## What This Changes

The issue requests that Control UI cron jobs accept `timeoutSeconds: 0` as no-timeout instead of rejecting it.

## Best Possible Solution

Review, land, or replace the open Control UI PR with a complete fix that accepts and preserves explicit zero, rejects malformed and negative timeout input, and aligns controller tests plus validation copy.

## Reproduction Assessment
