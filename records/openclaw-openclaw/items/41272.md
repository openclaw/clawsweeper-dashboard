Keep open: current main and the latest release still reject `timeoutSeconds: 0` in the Control UI cron validator and drop explicit zero from the submitted payload, while the gateway schema and cron runtime already accept zero as no-timeout. The linked same-author fix at https://github.com/openclaw/openclaw/pull/41275 is still open with closing syntax, so this issue should remain open until that PR lands or is replaced.

## What This Changes

The issue requests that the Control UI cron form accept `timeoutSeconds: 0` as the no-timeout mode for agent-turn cron jobs.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/41275 with a complete Control UI fix that accepts and preserves explicit zero, rejects malformed and negative input, and updates focused tests plus user-facing timeout copy if needed.
