## Summary

Keep open: the stale-record cleanup idea addresses a real Codex native hook outage, but the current predicate can delete live bridge records from other same-user gateway profiles, and the external PR still lacks after-fix proof from the submitted branch.

## What This Changes

The PR adds a registration-time bridge-directory scan that deletes native hook relay `.json` records whose recorded PID differs from the current process.

## Best Possible Solution

Keep the self-healing stale-record cleanup, but prune only records that are actually dead or expired, or limit repair to the current relay's own registry path, then add multi-gateway regression coverage and submitted-branch proof.

## Reproduction Assessment

Yes at source level for the introduced PR bug: the bridge directory is UID-scoped, multiple same-user gateways are documented, and the new predicate removes records from any different PID. I did not run a live two-gateway repro in this read-only review.

## Solution Assessment

No. The cleanup direction is useful, but deleting every non-current-PID record is too broad; the maintainable fix should verify dead or expired records, or limit cleanup to the current relay's own file.
