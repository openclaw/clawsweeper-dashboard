## Summary

## What This Changes

The PR moves Google Chat Zod config schemas from `zod-schema.providers-core.ts` into a new `zod-schema.providers-googlechat.ts` module and re-exports `GoogleChatConfigSchema` from the bundled channel schema SDK barrel.

## Best Possible Solution

Land the schema split only after prod typecheck stays green, the bundled-schema guardrail recognizes the new module, and the shared bot-loop schema comes from a neutral config module.

## Reproduction Assessment

Yes. Source inspection shows the guardrail omission and CI confirms the prod typecheck failure from the stale `SecretRefSchema` import; I did not run local tests because this review is read-only.

## Solution Assessment

No. The extraction direction is reasonable, but this branch needs to remove the stale import, update the guardrail, and avoid importing `providers-core` from the split GoogleChat schema.
