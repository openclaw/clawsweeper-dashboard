## Summary

Keep open. Current main still has the reported LF-only, non-global managed-block matcher, and this PR is a focused implementation with relevant regression tests; I found no functional or security blocker, but it needs the required changelog entry before merge.

## What This Changes

The PR updates `replaceManagedMarkdownBlock` to handle CRLF/lone-CR heading separators, replace all existing managed blocks, collapse adjacent duplicates, and add regression tests for CRLF, duplicate self-healing, and idempotency.

## Best Possible Solution

Land the helper and regression-test fix after adding the required changelog entry, keeping the behavior in the generic SDK helper so memory-core and memory-wiki consumers share the repair.

## Reproduction Assessment

Yes. Current main has a clear static reproduction path: a CRLF separator after the managed heading misses the LF-only regex and appends a new block, while duplicated matching blocks are only one replacement per call because the regex is not global.

## Solution Assessment

Yes for the code direction. A global, CRLF-tolerant matcher plus duplicate collapse is the narrow maintainable fix for the shared helper; the remaining safer completion is a release-note entry and normal validation.
