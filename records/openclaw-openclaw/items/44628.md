## Summary

Keep open: the underlying Linux systemd-scope bug is still visible on current main, but this PR should not merge as-is because it only changes the lower restart helper and also removes Docker mount whitespace validation.

## What This Changes

The PR removes Docker setup whitespace validation and adds synchronous user/system `systemctl is-enabled` and `is-active` probes to `triggerOpenClawRestart()` before attempting a Linux systemd restart with fallback.

## Best Possible Solution

Land a focused Linux systemd lifecycle fix in the managed daemon adapter, with regression tests for restart, stop, status/runtime, and scope reporting, while leaving Docker mount validation unchanged.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence mock reproduction: make `systemctl --user restart/show openclaw-gateway.service` return a missing-unit error while system-scope `systemctl restart/show` succeeds; current main never reaches the system-scope path in the managed daemon adapter.

## Solution Assessment

No. This PR is not the best fix because it changes `triggerOpenClawRestart()` instead of the managed Linux `src/daemon/systemd.ts` lifecycle path, can prefer enabled-only units over the active unit, and mixes in an unrelated Docker setup change.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Move scope detection into the managed systemd adapter:** `src/infra/restart.ts:309-332`
  - body: `openclaw gateway restart` and status use the Linux service registry, which calls `restartSystemdService()` and `readSystemdServiceRuntime()` in `src/daemon/systemd.ts`. Adding detection only to `triggerOpenClawRestart()` leaves the managed system-service path broken.
  - confidence: 0.91
- **[P2] Prefer active units before enabled-only scopes:** `src/infra/restart.ts:345-356`
  - body: After the enabled+active cases, this decision tree chooses enabled-only units before active-only units. If one scope is enabled but inactive while the other is currently running, the command can restart the wrong scope and leave the active gateway untouched.
  - confidence: 0.9
- **[P2] Keep Docker mount whitespace validation or quote mounts:** `docker-setup.sh:147`
  - body: This unrelated change removes the early whitespace guard even though downstream mount specs still reject spaces and generated Compose volume lines are not consistently quoted. That changes Docker host-mount parsing behavior without being part of the systemd fix.
  - confidence: 0.86
- **[P3] Avoid double-labeling fallback detail strings:** `src/infra/restart.ts:389-392`
  - body: `detectionDetail` already contains the selected scope, so formatting fallback failures as `user (${detectionDetail})` produces duplicated labels such as `user (user (enabled+active))`, making restart errors harder to read.
  - confidence: 0.78
