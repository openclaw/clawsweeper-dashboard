## Summary

This PR should stay open only as source context for a narrower replacement: current main still has the managed Linux systemd lifecycle gap, but this patch targets the older infra restart helper and carries an unrelated Docker mount validation regression.

## What This Changes

The PR adds user/system systemd scope probing to `triggerOpenClawRestart()` and removes the Docker setup whitespace guard for mount path values.

## Best Possible Solution

Land a focused managed-systemd lifecycle fix in `src/daemon/systemd.ts` with regression tests for restart, stop, status/runtime, and scope reporting, while leaving Docker mount validation unchanged unless quoting is redesigned deliberately.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence mock path: make user-scope `systemctl` report the gateway unit missing while system-scope restart/show succeeds, and current main never reaches the system-scope managed adapter path.

## Solution Assessment

No. This PR is not the best fix because it changes `triggerOpenClawRestart()` instead of the managed Linux systemd adapter, can choose an enabled-but-inactive scope ahead of the active unit, and removes unrelated Docker whitespace validation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Move scope detection into the managed systemd adapter:** `src/infra/restart.ts:309-332`
  - body: `openclaw gateway restart` routes through the daemon service registry to `restartSystemdService()`, whose action helper still calls the user-scope systemd manager only. Adding detection here leaves the managed Linux lifecycle path broken for system-level services.
  - confidence: 0.91
- **[P2] Prefer active units before enabled-only scopes:** `src/infra/restart.ts:345-356`
  - body: After the enabled+active cases, this decision tree selects enabled-only units before active-only units. If one scope is enabled but inactive while the other is currently running, the command can restart the wrong scope and leave the active gateway untouched.
  - confidence: 0.9
- **[P2] Keep Docker mount whitespace validation or quote mounts:** `docker-setup.sh:147`
  - body: This unrelated change removes the early whitespace guard even though downstream mount specs still reject spaces and generated Compose volume lines are not consistently quoted. That changes Docker host-mount parsing behavior without being part of the systemd fix.
  - confidence: 0.87
- **[P3] Avoid double-labeling fallback detail strings:** `src/infra/restart.ts:389-392`
  - body: `detectionDetail` already includes the selected scope, so formatting failures as `user (${detectionDetail})` produces labels like `user (user (enabled+active))`, making restart errors harder to read.
  - confidence: 0.78
