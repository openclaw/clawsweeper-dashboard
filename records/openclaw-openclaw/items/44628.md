## Summary

Keep open. The systemd scope problem is valid, but this PR is not mergeable as written because it fixes the lower restart helper instead of the managed Linux lifecycle adapter, can choose an enabled-but-inactive unit over the running unit, and includes an unrelated Docker mount-validation change with parsing risk.

## What This Changes

The PR removes Docker setup whitespace validation and adds synchronous user/system `systemctl is-enabled` and `is-active` probes to `triggerOpenClawRestart()` so it can select and report a restart scope with fallback.

## Best Possible Solution

Land a focused daemon/systemd lifecycle fix: restart, stop, and runtime/status should fall back to system scope only when the user unit is clearly missing, preserve user-bus and permission errors, report the chosen scope, and add regression coverage. Docker mount validation should stay out of this change.

## Reproduction Assessment

Yes. A high-confidence reproduction path is to mock `execFile` so `systemctl --user restart/show openclaw-gateway.service` reports a missing unit while `systemctl restart/show openclaw-gateway.service` succeeds; current main still fails or reports the wrong state in that path.

## Solution Assessment

No. The PR is not the best fix because it updates `triggerOpenClawRestart()` while the managed Linux CLI lifecycle routes through `src/daemon/systemd.ts`, and its decision tree can prefer enabled-only services over active services.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Move scope detection into the managed systemd adapter:** `src/infra/restart.ts:309-332`
  - body: `gateway restart` and status/runtime use the Linux service registry, which calls `restartSystemdService()` and `readSystemdServiceRuntime()` in `src/daemon/systemd.ts`. Adding detection only to `triggerOpenClawRestart()` leaves the managed system-scope install path broken.
  - confidence: 0.91
- **[P2] Prefer active units before enabled-only scopes:** `src/infra/restart.ts:345-356`
  - body: After the enabled+active cases, this decision tree chooses enabled-only units before active-only units. If one scope is enabled but inactive while the other is currently running, the command can restart the wrong service and leave the active gateway untouched.
  - confidence: 0.9
- **[P2] Keep Docker whitespace validation or quote mounts:** `docker-setup.sh:147`
  - body: This unrelated change removes the early whitespace guard even though downstream mount specs still reject spaces and generated Compose volume lines are not consistently quoted. That changes Docker host-mount parsing behavior without being part of the systemd fix.
  - confidence: 0.86
