## Summary

Keep open. The underlying system-level systemd lifecycle bug still appears valid on current main, but this PR is not mergeable as written because it changes the lower restart helper instead of the managed daemon systemd adapter and includes an unrelated Docker mount-validation regression.

## What This Changes

The PR removes Docker setup whitespace validation and adds synchronous user/system `systemctl is-enabled` and `is-active` probes to `triggerOpenClawRestart()` before restarting with fallback.

## Best Possible Solution

Land a focused daemon/systemd lifecycle fix in the managed service adapter: restart, stop, status/runtime, and enabled checks should fall back to system scope only when the user unit is clearly missing, preserve existing user-bus and permission errors, report the selected scope, and leave Docker mount validation unchanged.

## Reproduction Assessment

Yes. A high-confidence reproduction path is to mock `systemctl --user restart/show openclaw-gateway.service` returning a missing-unit/not-found error while `systemctl restart/show openclaw-gateway.service` succeeds; current main's managed adapter only calls the user-scope path in that scenario.

## Solution Assessment

No. The PR is not the best fix because it adds detection to `triggerOpenClawRestart()` while the managed CLI lifecycle routes through `src/daemon/systemd.ts`, and its priority order can select an enabled-but-inactive unit over the active unit.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Move scope detection into the managed systemd adapter:** `src/infra/restart.ts:309-332`
  - body: `openclaw gateway restart` and status use the Linux service registry, which calls `restartSystemdService()` and `readSystemdServiceRuntime()` in `src/daemon/systemd.ts`. Adding detection only to `triggerOpenClawRestart()` leaves the managed system-scope install path broken.
  - confidence: 0.91
- **[P2] Prefer active units before enabled-only scopes:** `src/infra/restart.ts:345-356`
  - body: After the enabled+active cases, this decision tree chooses enabled-only units before active-only units. If one scope is enabled but inactive while the other is currently running, the command can restart the wrong service and leave the active gateway untouched.
  - confidence: 0.9
- **[P2] Keep Docker whitespace validation or quote mounts:** `docker-setup.sh:147`
  - body: This unrelated change removes the early whitespace guard even though downstream mount specs still reject spaces and generated Compose volume lines are not consistently quoted. That changes Docker host-mount parsing behavior without being part of the systemd fix.
  - confidence: 0.86
- **[P3] Avoid double-labeling fallback detail strings:** `src/infra/restart.ts:389-392`
  - body: `detectionDetail` already contains the selected scope, so formatting fallback failures as `user (${detectionDetail})` or `system (${detectionDetail})` produces duplicated labels such as `user (user (enabled+active))`.
  - confidence: 0.78
