## Summary

Keep open: the underlying systemd bug still appears valid on current main, but this PR is not mergeable as-is because it fixes the wrong layer and includes an unrelated Docker validation regression.

## What This Changes

The PR adds user/system systemd scope probing to `triggerOpenClawRestart()` and removes Docker setup whitespace rejection for mount paths.

## Best Possible Solution

Land a focused Linux systemd lifecycle fix in `src/daemon/systemd.ts` with regression coverage for restart, stop, status/runtime, and scope reporting, while leaving Docker mount validation unchanged.

## Reproduction Assessment

Yes, by source-level reproduction: make user-scope `systemctl --user restart/show openclaw-gateway.service` report a missing unit while system-scope `systemctl restart/show` succeeds. Current main still routes managed restart/status through the user-scope adapter path.

## Solution Assessment

No. The PR is not the best fix because it changes `triggerOpenClawRestart()` instead of the managed Linux systemd lifecycle adapter, can choose an enabled-but-inactive unit ahead of an active unit, and mixes in an unrelated Docker setup validation change.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Move scope detection into the managed systemd adapter:** `src/infra/restart.ts:309-332`
  - body: `openclaw gateway restart` and status use the Linux service registry, which calls `restartSystemdService()` and `readSystemdServiceRuntime()` in `src/daemon/systemd.ts`. Adding detection only to `triggerOpenClawRestart()` leaves the managed system-service path broken.
  - confidence: 0.91
- **[P2] Prefer active units before enabled-only scopes:** `src/infra/restart.ts:345-356`
  - body: After the enabled+active cases, this decision tree chooses enabled-only units before active-only units. If one scope is enabled but inactive while the other is currently running, the command can restart the wrong scope and leave the active gateway untouched.
  - confidence: 0.9
- **[P2] Keep Docker mount whitespace validation or quote mounts:** `docker-setup.sh:147`
  - body: This unrelated change removes the early whitespace guard even though downstream mount specs still reject spaces and generated Compose volume lines are not consistently quoted. That changes Docker host-mount parsing behavior without being part of the systemd fix.
  - confidence: 0.86
- **[P3] Avoid double-labeling fallback detail strings:** `src/infra/restart.ts:389-392`
  - body: `detectionDetail` already contains the selected scope, so formatting fallback failures as `user (${detectionDetail})` produces duplicated labels such as `user (user (enabled+active))`, making restart errors harder to read.
  - confidence: 0.78
