## Summary

Keep open. The underlying Linux systemd scope bug still appears real on current main, but this PR should not merge as written because it fixes only the lower restart helper, leaves the managed daemon lifecycle/status path user-scope only, and includes an unrelated Docker mount-validation change that current main still deliberately rejects.

## What This Changes

The PR removes Docker setup whitespace validation and adds synchronous user/system `systemctl is-enabled` and `is-active` probes inside `triggerOpenClawRestart()` to choose a restart scope and report the detected scope.

## Best Possible Solution

Land a focused Linux systemd lifecycle fix in `src/daemon/systemd.ts`: restart, stop, enabled detection, and runtime/status should fall back to system scope only for well-classified missing-user-unit cases, preserve permission and bus errors, report the chosen scope, and keep Docker mount validation out of the change.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to mock the managed systemd path so `systemctl --user restart/show` reports the unit missing while the system scope would succeed; current main never tries the system scope in `restartSystemdService()` or `readSystemdServiceRuntime()`.

## Solution Assessment

No. This PR is directionally useful but not the best fix because it updates `triggerOpenClawRestart()` instead of the managed systemd adapter, prioritizes enabled-only units before active-only units, and mixes in an unrelated Docker validation change.

  - body: The Linux service registry routes gateway lifecycle restart/status through `src/daemon/systemd.ts`, but this patch adds detection only inside `triggerOpenClawRestart()`.   - confidence: 0.9
- **[P2] Keep Docker mount whitespace blocked unless mounts are quoted:** `docker-setup.sh:147`
  - body: This unrelated change removes the early whitespace rejection even though downstream mount validation still rejects spaces and generated Compose volume lines are not quoted. That turns a clear validation error into broken or ambiguous Docker volume parsing.
  - confidence: 0.86

  - body: When fallback restart attempts fail, the detail strings wrap `detectionDetail` in another scope label, producing output like `user (user (enabled+active))`. The same construction appears in the system-preferred branch and should be formatted once.
  - confidence: 0.78
