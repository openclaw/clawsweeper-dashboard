## Summary

Keep open: the PR fixes the main automatic-fallback persistence bug, but its remaining direct-agent and cron guards still infer source from final provider/model values, which misclassifies internal live-session switches handled inside `runWithModelFallback`.

## What This Changes

The PR skips persisting automatic fallback models as durable session runtime models, preserves contextTokens when skipping that write, adds direct-agent/session-store/cron tests, and updates the changelog.

## Best Possible Solution

Add explicit selection-source metadata to `runWithModelFallback` and use that contract in direct-agent and cron persistence so automatic fallbacks are transient while user-initiated live switches persist durably.

## Reproduction Assessment

Yes. Source inspection shows `runWithModelFallback` can consume `LiveSessionModelSwitchError` internally and return the switched provider/model with `attempts: []`, while the PR still infers source from final provider/model in direct-agent and cron callers.

## Solution Assessment

No. The fallback no-persist direction is correct, but final-value comparison is not the best contract; explicit fallback-result source metadata is the narrower maintainable fix.
