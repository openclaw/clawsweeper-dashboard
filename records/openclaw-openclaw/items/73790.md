## Summary

Keep open: the PR fixes the sticky automatic-fallback session model path, including context-token preservation, but latest head still infers fallback source from final provider/model equality, so live switches consumed inside the shared fallback runner can be skipped by both direct-agent and cron persistence.

## What This Changes

The PR guards direct-agent and isolated-cron runtime-model persistence against automatic fallback results, preserves contextTokens when skipping fallback persistence, adds regression tests, and updates CHANGELOG.md.

## Best Possible Solution

Add explicit selection-source metadata to runWithModelFallback's result and have both direct-agent and cron persistence use that contract, with regression coverage for internal live-switch redirects and automatic fallback no-persist behavior.

## Reproduction Assessment

Yes. Source inspection shows runWithModelFallback can consume LiveSessionModelSwitchError internally and return the switch target with attempts [], while the PR callers still infer source from final provider/model equality.

## Solution Assessment

No. The current PR direction fixes the main fallback persistence bug, but final-value comparison is not the best contract; explicit fallback-result source metadata is the narrower maintainable fix.
