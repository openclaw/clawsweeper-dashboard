## Label Justifications

- `P1`: The PR targets a gateway heap OOM path that can make long-running OpenClaw runtime unavailable for real users.
- `merge-risk: 🚨 session-state`: The patch clears run context by session id, and using the wrong session key can leave stale session/run state reachable.

## Summary

Keep open: the PR targets the right gateway heap-retention bug, but it is not merge-ready because the cleanup clears a mutable final session id instead of the originally registered run id, and the PR still has only unit-level validation for a live heap/OOM problem.

## What This Changes

The PR adds finally-block cleanup for isolated cron runs, exports `clearAgentRunContext` through the cron runtime barrel, and adds cleanup-focused unit tests.

## Best Possible Solution

Clear the exact immutable run id registered before execution, keep store disposal after final persistence and delivery, add a rotated-session regression, and attach redacted runtime or heap proof for repeated isolated cron runs.

## Reproduction Assessment

Yes, for the PR defect by source inspection: registration uses the initial session id, while finalization can rotate `sessionEntry.sessionId` before the PR clears it. I did not run a live heap or gateway reproduction.

## Solution Assessment

No. The cleanup direction is right, but the implementation should key off the immutable registered run id and prove that path with a rotated-session regression plus real runtime evidence.
