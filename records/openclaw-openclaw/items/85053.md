## Summary

Keep open: the fix targets a real P1 heap-retention path, but the cleanup uses a mutable session id after finalization can rotate it, so it can miss the registered context; the PR also has only test-level proof.

## What This Changes

The PR adds a finally-block cleanup for isolated cron runs that clears the agent run context and drops the cron session store, plus a small cleanup test file.

## Best Possible Solution

Clear the context using the immutable run id captured before execution, keep store disposal after final persistence and delivery, add a regression where finalization rotates agentMeta.sessionId, and attach redacted live or heap proof for repeated isolated cron runs.

## Reproduction Assessment

Yes, there is a high-confidence source-level reproduction for the PR defect: registration uses the initial session id, finalization can rotate sessionEntry.sessionId, and the PR clears the mutable final value. I did not run a live heap or gateway reproduction.

## Solution Assessment

No. The cleanup direction is right, but the implementation should key off the immutable registered run id and prove that path with a rotated-session regression plus real runtime evidence.
