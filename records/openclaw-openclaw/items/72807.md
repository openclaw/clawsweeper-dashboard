## What This Changes

This PR adds an optional `gitInit` parameter to `agents.create` and `ensureAgentWorkspace` so callers can skip `git init` for brand-new agent workspaces while preserving default behavior.

## Best Possible Solution

Land a rebased additive `gitInit` option through the schema, RPC handler, workspace helper, and current generated clients while preserving existing workspace defaults and adding real after-fix proof.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an additive API feature. Source inspection of current main shows `AgentsCreateParams` rejects unknown fields and `ensureAgentWorkspace` still calls `git init` for brand-new bootstrapped workspaces.

## Solution Assessment

No, not as submitted. The optional boolean defaulting to current behavior is a reasonable direction, but the branch must preserve `skipOptionalBootstrapFiles`, regenerate current protocol clients, and provide real behavior proof.
