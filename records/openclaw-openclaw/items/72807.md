## Summary

Keep open. Current main still lacks an `agents.create`/workspace opt-out for automatic git initialization, and this PR is a narrow, mostly correct additive implementation; it still needs the required changelog entry before merge.

## What This Changes

The PR adds an optional `gitInit` flag to `agents.create` and `ensureAgentWorkspace`, forwards it through the gateway handler, regenerates Swift protocol models, and adds a workspace test for skipping `.git` creation.

## Best Possible Solution

Land the additive `gitInit` option with default behavior unchanged, keep schema and generated client artifacts aligned, add release-note coverage, and avoid broad workspace policy or startup migration changes.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high-confidence behavior evidence: current main lacks `gitInit`, and brand-new bootstrapped workspaces still run `git init` when git is available.

## Solution Assessment

Yes. An optional boolean that defaults to current behavior is the narrowest maintainable API for embedders that need to suppress automatic repo creation; the remaining work is release-note/test coverage rather than a different product design.
