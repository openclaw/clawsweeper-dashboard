## Summary

Keep open. Current main still lacks a `gitInit` opt-out on `agents.create`, so this is useful source work rather than an obsolete PR, but the branch is not merge-ready because the changed `agents.create` workspace call needs to preserve current main's `skipOptionalBootstrapFiles` forwarding and the user-facing feature still needs a changelog entry.

## What This Changes

The PR adds an optional `gitInit` boolean to the `agents.create` protocol and `ensureAgentWorkspace`, threads it through the gateway handler, updates generated Swift protocol models, and adds a workspace regression test for skipping `.git` creation.

## Best Possible Solution

Land the feature as an additive protocol/workspace option with default behavior unchanged, after rebasing onto current main, preserving existing workspace bootstrap defaults, adding a single-line changelog entry with contributor credit, and validating the agents and protocol surfaces through repo scripts.

## Reproduction Assessment

Not applicable as a bug reproduction. Source inspection gives high-confidence evidence for the missing capability: current main rejects unknown `agents.create` params and still calls `git init` for brand-new bootstrapped workspaces when git is available.

## Solution Assessment

Yes, directionally. An optional boolean defaulting to current behavior is the narrowest maintainable API shape for embedders that need to suppress automatic repo creation; the merge path needs a rebase-preservation fix and release-note coverage.
