## Summary

Keep open. Current main still lacks an agent-level delivery default contract; existing routing preserves explicit or already-captured session delivery context, but fresh programmatic sessions still have no per-agent fallback target/channel.

## What This Changes

Request to add per-agent delivery defaults so targetless `message.send` calls in programmatic sessions can infer channel and target.

## Best Possible Solution

Define a typed `agents.list[].delivery` contract using `DeliveryContext`-style fields, validate and document it, resolve it with agent config, and apply it only as the lowest-precedence trusted fallback after explicit message args and run/session delivery context.

## Reproduction Assessment

Yes for the current limitation: source inspection shows the proposed agent config key is absent and targetless `message.send` still throws without an inferable tool context. I did not run a live channel reproduction because this is a new cross-channel config surface, not a regression in an existing documented contract.

## Solution Assessment

Unclear. An agent-level fallback is plausible, but the proposed two-field shape should be reconciled with the existing `DeliveryContext` model and explicit precedence rules before implementation.
