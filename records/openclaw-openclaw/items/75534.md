## Summary

Keep open. Current main still computes the config hashes but continues into writer side effects, and open PR #75543 already targets this issue with closing syntax.

## What This Changes

Request a config writer no-op guard that skips disk I/O, backup/audit/log noise, and reload-triggering writes when the final persisted config hash has not changed.

## Best Possible Solution

Add a writer-level no-op short-circuit after final serialization/hash calculation, with regression coverage for unchanged mtime/no backup/no overwrite log while preserving real-change audit, rejection, and reload behavior.

## Reproduction Assessment

Yes. A focused test can write a config, call `writeConfigFile` again with an equivalent final payload, and assert current main still proceeds to temp-write/backup/audit work because there is no early return after hash comparison.

## Solution Assessment

Yes. Keeping this issue open behind PR #75543 is the right path; the maintainable fix belongs at the config writer boundary rather than only in the `config.patch` RPC or reloader dedupe layer.
