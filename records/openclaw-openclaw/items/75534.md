## Summary

Keep open. Current main still lacks the requested `writeConfigFile` hash-equality short-circuit, and open PR #75543 already targets this issue with closing syntax, so the issue should remain open until that PR is reviewed, landed, or replaced.

## What This Changes

Request a writer-level no-op short-circuit in `writeConfigFile` when the computed output hash matches the existing config hash to avoid unnecessary disk writes and watcher-triggered gateway restarts.

## Best Possible Solution

Review and land PR #75543, or replace it with an equivalent narrow writer-boundary guard that skips disk effects only for true stable no-ops while preserving write protections for real changes.

## Reproduction Assessment

Yes. Source inspection gives a focused reproduction path: write a config, call `writeConfigFile` again with equivalent persisted output, and assert current main still reaches temp-write, backup, overwrite-log/audit, and rename paths instead of returning after hash comparison.

## Solution Assessment

Yes. The requested writer-boundary guard is the narrow maintainable fix because existing `config.patch` and reloader guards do not cover every `writeConfigFile` caller or prevent watched-path I/O.
