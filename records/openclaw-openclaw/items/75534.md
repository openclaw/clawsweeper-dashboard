## Summary

Keep open. Current main still lacks the requested writer-level no-op short-circuit: `writeConfigFile` computes the hashes and then continues into audit, temp-write, backup, and rename work, while the existing `config.patch` no-op and reload hash dedupe are narrower/downstream protections. Open PR #75543 already targets this issue with closing syntax, so this issue should stay open until maintainers review, land, or replace that PR.

## What This Changes

Add a no-op short-circuit to `writeConfigFile` so unchanged config saves avoid disk writes, audit/backup noise, and reload cascades.

## Best Possible Solution

Review and land or replace PR #75543 with a narrow `writeConfigFile` no-op guard plus writer-level regression coverage for unchanged mtime, skipped backup/audit, and preserved real-change behavior.
