## Summary

Keep open. Current main still lacks the requested writer-level hash-equality short-circuit, and open PR #75543 already targets this issue with closing syntax, so maintainers should review, land, or replace that PR rather than closing this issue early.

## What This Changes

The issue requests a no-op short-circuit in `writeConfigFile` so equal persisted and next config hashes do not trigger disk writes, audit noise, backups, or reload cascades.

## Best Possible Solution

Land or replace PR #75543 with a narrow writer-boundary guard plus regression coverage proving true no-op writes skip disk I/O while real config changes still preserve validation, audit, backup, and reload behavior.

## Reproduction Assessment

Yes. A focused repo check can write a config, call `writeConfigFile` again with an equivalent payload, and assert current main still reaches the post-hash write path instead of returning after hash comparison.

## Solution Assessment

Yes for keeping this issue open behind the linked PR. The maintainable fix belongs at the config writer boundary; the existing `config.patch` and reloader guards are useful but too narrow to solve the generic writer-side I/O problem.
