## Summary

Keep open. Current main still lacks a writer-level no-op branch, and the matching fix is already represented by open PR #75543 with closing syntax, so this issue should stay open until that PR or an equivalent replacement lands.

## What This Changes

The issue requests that `writeConfigFile` skip disk write side effects when the computed config output is unchanged or semantically unchanged.

## Best Possible Solution

Land or replace PR #75543 with a narrow writer-boundary guard that skips disk I/O for true stable no-ops while preserving audit, backup, suspicious-write rejection, runtime snapshot refresh, and reload behavior for real config changes.

## Reproduction Assessment

Yes. A focused repo reproduction can write a config, call `writeConfigFile` again with an equivalent stable payload, and assert that current main still reaches backup/temp-write/log/audit side effects instead of returning before disk I/O.

## Solution Assessment

Yes for keeping this open behind the linked PR. The best code solution belongs at the writer boundary, not only in `config.patch` or the reload watcher, and should cover the stable no-op cases without weakening real-change safety paths.
