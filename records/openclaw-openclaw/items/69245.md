## What This Changes

The PR broadens `isCacheTtlEligibleProvider` so any `modelApi === "openai-completions"` provider is eligible for cache-TTL pruning, and adds unit assertions for OpenAI, Astron Coding Plan, and DeepSeek cases.

## Best Possible Solution

## Reproduction Assessment

Yes by source inspection. Current main passes `model.api` into the eligibility helper and has no generic `openai-completions` fallback, so configured cache-TTL pruning remains disabled for those providers unless a provider hook opts in.

## Solution Assessment

No as written. The one-line fallback is mechanically simple, but the safer maintainable path is provider/capability-gated eligibility plus docs and live proof for each supported OpenAI-compatible family.
