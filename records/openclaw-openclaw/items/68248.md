## Summary

Keep open. Current main implements a substantial avatar subset through assistant identity resolution, authenticated avatar metadata, and browser-local avatar storage, but the full request is still incomplete: favicon links remain static, Quick Settings still has no display-name editor, the browser WebSocket connect payload does not send a displayName, and agent/session selectors remain native text-only controls.

## What This Changes

Request adds per-agent Control UI avatar/favicon customization plus a user-controlled display-name override for distinguishing multiple agents and sessions.

## Best Possible Solution

Keep the shipped avatar plumbing and scope a focused Control UI follow-up that updates the active favicon safely, adds a browser-local display-name editor with Gateway displayName propagation, and decides whether agent/session pickers need custom avatar-aware controls.

## Reproduction Assessment

Not applicable as a runtime bug report, but there is a high-confidence source-inspection path for the remaining gaps: static favicon links, fixed Quick Settings user title/no Name field, no browser client displayName propagation, and native text-only selectors.

## Solution Assessment

Unclear for the full request as filed because it bundles already-shipped avatar plumbing with remaining favicon, display-name, and picker-presentation work. The maintainable path is a scoped Control UI enhancement that reuses the existing avatar route and preserves the current image/CSP policy.
