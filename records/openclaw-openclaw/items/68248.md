## Summary

Keep open. Current main has substantial agent/avatar plumbing, but the full request is still incomplete: the Control UI favicon is static, the browser client does not send a user display name, Quick Settings still exposes only user-avatar controls, and the agent picker remains a native text-only select.

## What This Changes

Add Control UI customization for per-agent avatars, per-agent/browser-tab favicons, and user display-name overrides.

## Best Possible Solution

Keep the shipped avatar resolver/route model, then scope a Control UI follow-up that adds safe dynamic favicon handling, a browser-local display-name editor with Gateway displayName propagation, and a deliberate avatar-capable agent picker if native selects are no longer sufficient.

## Reproduction Assessment

Yes for the remaining gaps by source inspection rather than runtime steps: current main has static favicon links, no browser displayName connect payload, an avatar-only Quick Settings user card, and text-only native agent selection.

## Solution Assessment

Unclear for the full feature as filed because it bundles already-shipped avatar plumbing with remaining favicon, display-name, and picker-presentation work. The maintainable path is a narrower Control UI enhancement that reuses the existing avatar and CSP constraints.
