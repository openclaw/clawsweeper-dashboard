## Summary

Keep open. Current main has implemented the core assistant/agent avatar plumbing and authenticated avatar route, but the requested Control UI favicon customization, editable browser-local display name, and avatar rendering in the relevant agent/session picker surfaces remain incomplete.

## What This Changes

The issue requests per-agent avatar images, per-agent/browser-tab favicon customization, and user display-name overrides in the Control UI.

## Best Possible Solution

Keep the shipped avatar identity model, then narrow the remaining work to a focused Control UI follow-up: update the active favicon from the current agent/avatar using the existing safe avatar route or local blob/data URLs, expose a browser-local display-name editor that feeds chat rendering and Gateway client displayName metadata, and decide whether native select-based agent/session pickers should be replaced or supplemented with avatar-capable UI controls.

## Reproduction Assessment

Not applicable as a runtime failure report, but there is a high-confidence source-inspection path for the remaining gaps: static favicon links, fixed 'You' user identity UI with no name input, no browser client displayName in the connect payload, and text-only native agent/session selectors all remain on current main.

## Solution Assessment

Unclear for the full feature as requested because it bundles implemented avatar plumbing with remaining favicon, display-name, and picker presentation work. The best maintainable path is a narrowed Control UI enhancement that reuses the existing authenticated avatar route and preserves the documented image/CSP policy.
