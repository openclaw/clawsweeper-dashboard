## Summary

Keep open. Current main implements substantial avatar identity plumbing and local avatar overrides, but the filed request is still only partially covered: favicon/tab identity is static, Quick Settings has no display-name editor, Control UI chat sends no browser-local sender identity, and workspace avatar upload is tracked separately.

## What This Changes

Add Control UI support for per-agent avatars, browser-tab favicon customization, and editable display names for multi-agent workflows.

## Best Possible Solution

Keep the shipped avatar identity model, route workspace avatar upload through https://github.com/openclaw/openclaw/issues/42504, resolve tab/title or favicon policy alongside https://github.com/openclaw/openclaw/issues/80942, and separately scope a visible local display-name editor/propagation path if maintainers want it.

Not applicable as a bug reproduction; this is a feature request. Source inspection against origin/main gives a high-confidence baseline for the remaining gaps: static favicon/title links, no Quick Settings display-name editor, no local identity in chat.send, and a separate open avatar upload/write path.

## Solution Assessment

Unclear for the bundled request as filed because it mixes shipped avatar support with favicon, display-name, picker, and upload behavior. The safer maintainable path is to split those decisions rather than treat this as one narrow bug fix.
