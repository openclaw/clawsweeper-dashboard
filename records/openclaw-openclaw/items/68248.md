## Summary

Keep open. Current main has substantial agent/avatar plumbing and browser-local avatar overrides, but the filed request is still not complete: favicon links remain static, Quick Settings exposes avatar controls but no user display-name editor, `chat.send` does not carry browser-local user identity, and the agent/session pickers still render text-only native options. The upload/write portion is separately tracked in https://github.com/openclaw/openclaw/issues/42504, and browser tab disambiguation is also related to https://github.com/openclaw/openclaw/issues/80942.

## What This Changes

Request to add Control UI customization for agent avatars, per-agent/browser favicons, and user display-name overrides for multi-agent workflows.

## Best Possible Solution

Keep the shipped avatar model, split the remaining work into scoped Control UI decisions for active-tab favicon behavior and editable browser-local display names, and leave workspace avatar upload/write behavior to https://github.com/openclaw/openclaw/issues/42504.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature request. Source inspection on current main gives a high-confidence baseline for the remaining gaps: static favicon links, no visible display-name editor or chat.send field, text-only native pickers, and a separately tracked avatar upload/write path.

## Solution Assessment

Unclear for the request as filed because it bundles shipped avatar plumbing with favicon, display-name, picker, and upload behavior. The maintainable path is to scope those decisions separately instead of treating this as a narrow bug fix.
