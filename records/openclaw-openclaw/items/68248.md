## Summary

Keep open. Current main has substantial Control UI avatar plumbing and browser-local avatar overrides, but the filed request still has meaningful remaining product work: favicon links are static, Quick Settings exposes only avatar controls for the local user name, chat/agent pickers are native text-only selects, and workspace avatar upload/write behavior is tracked separately in https://github.com/openclaw/openclaw/issues/42504.

## What This Changes

Add Control UI support for per-agent avatars, per-agent favicon/tab icons, and user display-name overrides so multi-agent sessions are easier to distinguish.

## Best Possible Solution

Keep the shipped avatar model and scope narrower follow-ups for active-session favicon behavior, editable browser-local display names, and avatar-aware pickers, while leaving workspace avatar upload/write behavior to https://github.com/openclaw/openclaw/issues/42504.
