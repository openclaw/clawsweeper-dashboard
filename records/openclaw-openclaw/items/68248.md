## Summary

Keep open. Current main covers the core agent/avatar plumbing, but the full request is still incomplete: favicon links are static, Quick Settings still has no user display-name editor, the browser WebSocket client does not send `displayName`, and agent/session pickers remain text-only native selects.

## What This Changes

The issue requests per-agent avatar image support, per-agent/browser-tab favicon customization, and a Control UI user display-name override.

## Best Possible Solution

Keep the shipped agent avatar model and narrow the remaining work to a Control UI follow-up that updates the active favicon safely, restores a browser-local display-name editor with Gateway `displayName` propagation, and decides whether picker avatars need custom controls.

## Reproduction Assessment

Not applicable as a runtime failure report, but yes, there is a high-confidence source-inspection path for the remaining gaps: static favicon links, fixed `You` identity UI, no browser client `displayName`, and native text-only selectors are present on current main.

## Solution Assessment

Unclear for the full feature as filed because it bundles already-shipped avatar plumbing with remaining favicon, display-name, and picker-presentation work. The maintainable path is a scoped Control UI enhancement that reuses the existing avatar route and preserves the image/CSP policy.
