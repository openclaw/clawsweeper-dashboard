## Summary

Keep this PR open for maintainer review. Current main still lacks the PR's generic gateway colon-deferral behavior, and the proposed patch is incomplete because real embedded tool execution emits an adjacent item-start event that can still flush the colon-ending buffer. The linked Discord symptom has a shipped channel-preview mitigation, so the remaining decision is whether maintainers want this generic gateway policy at all.

## What This Changes

The PR adds a `suppressIncompleteNarration` option to gateway buffered-chat flushing, enables it for tool-start flushes when buffered assistant text ends with `:`, and adds gateway event tests for colon suppression and normal period-ending flushing.

## Best Possible Solution

Either close this PR as superseded by the shipped channel-preview behavior, or update it into an explicit generic gateway policy that defers incomplete narration across the approved tool and item start boundaries while preserving terminal final/error flushing and adding regression coverage for that full sequence.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes. A static gateway event sequence with a throttled assistant update ending in `:`, followed by tool `start` and then item `start`, shows the proposed patch can still flush the colon-ending buffer at the unchanged item boundary.

## Solution Assessment

Is this the best way to solve the issue? No. The direction is narrow, but the implementation only suppresses the tool-start flush; the safer generic fix would carry the same scoped suppression through the adjacent item-start boundary and keep terminal flushing unchanged.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Carry colon suppression through item-start flushes:** `src/gateway/server-chat.ts:761-763`
  - body: The embedded tool path emits a tool `start` and then an item `start` for the same call. This change suppresses only the tool-start flush; because that return leaves `deltaLastBroadcastLen` behind, the unchanged item-start branch can immediately flush the same colon-ending text as a chat delta. Please carry the suppression through that adjacent item boundary and cover the tool-plus-item sequence in the gateway event test.
  - confidence: 0.88
