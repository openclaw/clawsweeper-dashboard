## Summary

Keep open. Current main still lacks the PR's generic gateway colon-deferral behavior, and the proposed patch does not cover the adjacent item-start boundary that follows embedded tool-start events.

## What This Changes

The PR adds a gateway buffered-chat option to defer colon-ending assistant text before tool-start flushes and adds gateway event tests for colon suppression plus normal period-ending flushing.

## Best Possible Solution

Decide whether the shipped preview behavior is sufficient; if a generic gateway policy is accepted, scope it across approved tool and item start boundaries while preserving terminal final/error flushing and adding regression coverage.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes. A gateway event sequence with a throttled assistant update ending in `:`, followed by embedded tool `start` and then item `start`, shows the proposed patch can still flush the colon-ending buffer at the unchanged item boundary.

## Solution Assessment

Is this the best way to solve the issue? No. The PR is narrower than the gateway event boundary it targets, and the original Discord-facing symptom already has a shipped channel-preview mitigation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Carry colon suppression through item-start flushes:** `src/gateway/server-chat.ts:761-763`
  - body: The embedded tool path emits a tool `start` and then a tracked item `start` for the same call. This change only passes `suppressIncompleteNarration` on the tool-start flush; when the colon-ending text is a throttled tail, the item-start branch can immediately flush the same buffer as a chat delta. Please apply the scoped suppression to the adjacent item-start boundary and cover that sequence in the gateway event test.
  - confidence: 0.88
