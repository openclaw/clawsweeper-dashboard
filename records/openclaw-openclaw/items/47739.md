## Summary

Keep open for maintainer review: current main does not contain this gateway colon-suppression change, and the PR still has a source-reproducible item-start gap. The linked Discord symptom appears handled by shipped preview streaming, so accepting a generic gateway policy needs maintainer behavior judgment.

## What This Changes

The PR adds an optional gateway buffered-chat flush option that skips tool-start delta flushing when buffered assistant text ends with `:` and adds two gateway event tests.

## Best Possible Solution

Decide whether to close this PR as superseded by preview streaming, or accept a generic gateway policy and implement it consistently across approved tool/item boundaries while preserving terminal final and error flushing.

## Reproduction Assessment

Yes, from source inspection. A throttled assistant update ending in `:` followed by embedded tool `start` and tracked item `start` can bypass the PR's tool-start-only suppression and flush at the item-start branch.

## Solution Assessment

No. The patch is narrower than the gateway event boundary it targets, and the original Discord-facing symptom already has a shipped preview-streaming mitigation, so maintainers should first decide whether the generic gateway policy is wanted.

## Review Findings

Overall correctness: patch is incorrect

- **[P2] Carry colon suppression through item-start flushes:** `src/gateway/server-chat.ts:761-763`
  - body: This only applies the incomplete-narration guard to tool-start flushes. Apply the same scoped policy to item starts, or make the boundary policy explicit and cover that sequence.
