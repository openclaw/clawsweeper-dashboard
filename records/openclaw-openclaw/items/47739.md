## Summary

Keep open for maintainer judgment: the linked Discord symptom was closed after the shipped preview-streaming path, but current main does not implement this PR's generic gateway colon-suppression policy and the PR still misses an adjacent item-start boundary.

## What This Changes

The PR adds a scoped gateway buffered-chat option to skip pre-tool chat-delta flushing when buffered assistant text ends in a colon, plus gateway event tests for colon suppression and period-ending flushing.

## Best Possible Solution

Decide whether to close this PR as superseded by the shipped preview-streaming behavior, or explicitly accept a generic gateway policy and apply it consistently across tool and item start boundaries while preserving terminal final/error flushing.

## Reproduction Assessment

Yes, from source inspection. A throttled assistant update ending in `:` followed by embedded tool `start` and tracked item `start` can still flush the colon-ending buffer at the unchanged item-start boundary.

## Solution Assessment

No. The patch is narrower than the gateway event boundary it targets, and the original Discord-facing symptom already has a shipped channel-preview mitigation, so maintainers should first decide whether the generic policy is wanted.
