## Summary

Keep open for maintainer review: current main does not implement the generic gateway colon-suppression change, but this PR still needs item-start coverage, real channel proof, and a maintainer decision on whether the generic heuristic belongs in gateway after the linked Discord symptom was handled separately.

## What This Changes

The branch adds a `suppressIncompleteNarration` option to the gateway buffered-chat flush helper, enables it for tool-start flushes when text ends with a colon, and adds two gateway event tests.

## Best Possible Solution

Decide whether the shipped channel-preview behavior is sufficient; if not, rebase and implement an explicit gateway policy for approved tool and item boundaries while preserving final/error flushing and adding live channel proof.

## Reproduction Assessment

Yes for the generic gateway path from source: current main flushes buffered assistant text before tool and item start events, and existing tests assert pre-tool flushing. No live Discord reproduction is established in the PR.

## Solution Assessment

No as-is. The helper option is a reasonable narrow shape, but the PR only covers tool-start flushes while current main also has item-start flushes, and the remaining generic gateway policy needs maintainer approval.
