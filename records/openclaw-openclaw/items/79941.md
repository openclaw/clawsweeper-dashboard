Current main already fixes the reported whole-agent Codex pin problem by ignoring legacy whole-agent runtime pins and resolving runtime policy from the selected provider/model instead; this is main-only after v2026.5.7.

## What This Changes

The issue asks for dashboard session model switching to avoid accepting or running a Codex-runtime plus Ollama-auth mismatch, either by routing to PI/Ollama, blocking the selection, or surfacing a clear error.

## Best Possible Solution

Keep the provider/model-scoped runtime policy from current main and leave the separate explicit provider/model forced-Codex fallback semantics to https://github.com/openclaw/openclaw/issues/75739 and https://github.com/openclaw/openclaw/pull/74341.
