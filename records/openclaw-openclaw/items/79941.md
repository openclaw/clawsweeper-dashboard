## Summary

Current main already fixes the reported stale Codex-runtime routing path: user model switches are stored as provider/model overrides, stale runtime model fields are cleared, and legacy whole-agent/session runtime pins no longer select the Codex harness for non-OpenAI overrides.

## What This Changes

The issue reports that switching a Codex-runtime main chat session to an Ollama/Gemma model is accepted but the reply still runs through Codex and fails on the Ollama auth profile.

## Best Possible Solution

Keep the current main behavior: legacy whole-agent/session runtime pins stay ignored, and the distinct forced provider/model runtime support bug continues under https://github.com/openclaw/openclaw/issues/75739 and https://github.com/openclaw/openclaw/pull/74341.

## Reproduction Assessment

No, not against current main by source and regression-test inspection. The 2026.5.7 report has concrete release logs, but current main ignores `agents.defaults.agentRuntime` and stale session `agentHarnessId` when a non-OpenAI user model override is dispatched.

## Solution Assessment

Yes. The current provider/model-scoped runtime selection is the narrow maintainable fix for this report because it keeps OpenAI/Codex routing explicit while letting an Ollama session override use the normal PI/auto path.
