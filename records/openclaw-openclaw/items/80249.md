## What This Changes

Request to add a backward-compatible Feishu/TTS option that sends readable text alongside native voice-message replies instead of suppressing it.

## Best Possible Solution

Review and repair the linked PR, or choose a shared TTS text-delivery policy, while preserving the current default suppression and fallback text behavior.

Yes for the current behavior: source and tests show Feishu suppresses text whenever native voice media is delivered. This is not a strict bug repro because the request is for a new opt-in delivery policy.

## Solution Assessment

Unclear. A Feishu `tts.preserveText` option is plausible, but maintainers should decide whether this belongs in Feishu config or a generic `messages.tts` delivery policy, and the open PR needs its fallback behavior fixed first.
