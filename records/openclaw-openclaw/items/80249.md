## Summary

Keep open. Current main still suppresses Feishu text whenever native voice media is delivered, there is no preserve-text delivery option in Feishu or shared TTS config, and the exact work is still under active review in https://github.com/openclaw/openclaw/pull/81258 rather than implemented on main.

## What This Changes

Add a backward-compatible Feishu TTS option to send text alongside native voice replies instead of always suppressing the text.

## Best Possible Solution

Land a documented, backward-compatible TTS text-delivery policy that Feishu honors without duplicating fallback text, after maintainers choose Feishu-specific versus shared config scope.

## Reproduction Assessment

Yes for the current behavior: source and tests show Feishu suppresses text whenever a successful reply delivers native voice media. This is not a strict bug repro because preserving text is a new opt-in policy request.

## Solution Assessment

Unclear. A Feishu boolean would address this reporter's use case, but the maintainable solution should align with the broader TTS delivery-policy discussion and preserve existing degradation/fallback behavior.
