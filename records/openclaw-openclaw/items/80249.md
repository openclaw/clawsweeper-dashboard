## Summary

Keep open. Current main and the latest release still suppress Feishu text when native voice media is delivered, and the open linked implementation PR uses closing syntax for this issue but has not merged yet.

## What This Changes

Request adds an opt-in Feishu TTS setting to send visible text alongside native voice-message delivery.

## Best Possible Solution

Repair and review https://github.com/openclaw/openclaw/pull/81258, or replace it with an equivalent accepted TTS delivery-policy change that keeps the current default and fallback behavior correct.

## Reproduction Assessment

Yes for the current behavior: source and tests show Feishu suppresses text whenever native voice media is delivered. This is not a broken-contract bug repro; the request is for a new opt-in delivery policy.

## Solution Assessment

Unclear. The linked Feishu-only boolean is plausible, but it currently needs a fallback fix and maintainers should confirm whether this belongs as Feishu-specific config or a generic TTS delivery policy.
