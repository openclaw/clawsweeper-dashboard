## Summary

Keep open. Current main still suppresses Feishu text when native voice/TTS media is present, the requested opt-in config is not implemented, and the linked open PR still needs review/product resolution before this issue can close.

## What This Changes

Request an opt-in Feishu TTS setting that preserves visible text alongside native voice messages instead of delivering voice-only.

## Best Possible Solution

Review and either repair/land https://github.com/openclaw/openclaw/pull/81258 or replace it with a maintainer-approved generic TTS text-delivery policy that Feishu honors while preserving current defaults and fallback behavior.

## Reproduction Assessment

Yes for the current behavior: source and tests show Feishu suppresses text whenever a reply delivers native voice media. This is not a failing bug repro because the requested outcome is a new opt-in delivery policy.

## Solution Assessment

Unclear. A Feishu-only boolean may satisfy this issue, but the linked generic TTS delivery-policy request means maintainers should choose the public config shape and fallback semantics before implementation lands.
