## Summary

Keep open. Current main still suppresses Feishu text when native voice media is present, and the requested opt-in needs a new config/product policy that is not implemented or exactly covered by another canonical item.

## What This Changes

Add an opt-in Feishu or TTS setting that sends visible text alongside Feishu TTS voice messages instead of voice-only delivery.

## Best Possible Solution

Add a backward-compatible TTS text-delivery policy that Feishu honors at the maintainer-chosen scope, keeping current default suppression and fallback behavior intact.

Yes for the current behavior: source and tests show Feishu suppresses text whenever native voice media is delivered. This is not a failing bug reproduction because the requested preserve-text mode is a new opt-in policy.

## Solution Assessment

Unclear. A Feishu-only boolean would solve the reporter's use case, but a generic TTS delivery policy such as the one discussed in https://github.com/openclaw/openclaw/issues/73210 may be the cleaner product surface.
