## What This Changes

Request an opt-in Feishu TTS setting that sends readable text alongside native voice media instead of suppressing the text bubble.

## Best Possible Solution

Review and revise the linked Feishu PR or a generic TTS delivery-policy design so the opt-in preserves text without regressing fallback/degradation behavior.

Yes for the current behavior: source and tests show Feishu suppresses text whenever native voice media is delivered. This is not a strict bug reproduction because the requested behavior is a new opt-in policy.

## Solution Assessment

Unclear. The direction is useful, but maintainers need to choose Feishu-specific tts.preserveText versus a generic messages.tts delivery policy and fix fallback semantics before merge.
