## Summary

Keep open. Current main and the latest release still suppress Feishu text when native voice media is delivered, and the requested opt-in preservation is a new public delivery-policy/config decision rather than an implemented fix or exact duplicate.

## What This Changes

Add a backward-compatible Feishu or TTS configuration option that sends visible text alongside native Feishu TTS voice messages.

## Best Possible Solution

Define a backward-compatible TTS text-delivery policy that Feishu can honor per channel/account if maintainers choose that scope, preserving the current voice-only default and degraded-upload fallback behavior.

## Reproduction Assessment

Yes for the current behavior: source, tests, docs, and the latest release show Feishu suppresses text whenever a reply is delivered as native voice media. This is not a failing bug reproduction because the requested behavior is a new opt-in policy.

## Solution Assessment

Unclear. Preserving text with voice is a coherent request, but maintainers should choose between a Feishu-specific per-account option and a shared TTS delivery-policy contract before implementation.
