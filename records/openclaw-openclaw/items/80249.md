## Summary

Keep open. Current main still suppresses Feishu text whenever native voice media is delivered, and the requested opt-in is a new delivery-policy/config feature rather than an already-implemented cleanup item.

## What This Changes

Add an opt-in Feishu/TTS configuration path that sends visible text alongside TTS native voice bubbles instead of voice-only delivery.

## Best Possible Solution

Define a backward-compatible TTS text-delivery policy that Feishu can honor per the chosen scope, defaulting to current voice-only behavior and preserving existing fallback behavior.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature/config request. The current behavior is high-confidence from source and tests: Feishu suppresses visible text when a reply delivers native voice media.

## Solution Assessment

Unclear. A Feishu-only boolean may solve the reported use case, but maintainers should first choose whether this belongs in generic `messages.tts` delivery policy, Feishu channel config, or per-account config.
