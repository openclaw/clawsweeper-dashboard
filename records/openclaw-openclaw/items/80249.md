## Summary

Keep open. Current main still suppresses Feishu text when native voice media is delivered, and the requested opt-in would add a new public config/product policy rather than closeable maintainer cleanup.

## What This Changes

Request an opt-in Feishu or TTS configuration that sends text alongside TTS voice replies instead of voice-only delivery.

## Best Possible Solution

Define a backward-compatible TTS delivery policy that can preserve Feishu text beside voice when opted in, keeps the current voice-only default unless maintainers change it, and preserves existing degradation/failure text fallbacks.

## Reproduction Assessment

Yes for the current behavior: source and tests show Feishu suppresses text whenever a reply delivers native voice media. This is not a failing bug reproduction because the requested behavior is a new opt-in policy.

## Solution Assessment

Unclear. A Feishu-only boolean could satisfy this report, but maintainers should decide whether a generic TTS delivery policy coordinated with https://github.com/openclaw/openclaw/issues/73210 and https://github.com/openclaw/openclaw/issues/42539 is the maintainable API.
