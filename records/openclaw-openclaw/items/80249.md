## Summary

Keep open. Current main and v2026.5.12 still suppress Feishu text when a reply is delivered as native voice media, and the related implementation PR at https://github.com/openclaw/openclaw/pull/81258 explicitly closes this issue but remains open with a needs-changes review and an unresolved config-surface decision.

## What This Changes

Request a backward-compatible Feishu or TTS option to send visible text alongside TTS voice bubbles instead of suppressing text for voice media.

## Best Possible Solution

Review and repair the linked implementation PR, then land a maintainer-approved delivery policy with fallback/degradation coverage and docs so this issue closes after merge.

## Reproduction Assessment

Yes for the current behavior at source level: current main and tests show Feishu suppresses text whenever voice media is detected. This is not a strict bug reproduction because the requested outcome is a new opt-in delivery policy.

## Solution Assessment

Unclear. A Feishu-only boolean may satisfy this request, but the related generic TTS policy request means maintainers should confirm the config surface before merge.
