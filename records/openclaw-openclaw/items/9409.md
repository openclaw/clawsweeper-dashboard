## Summary

Codex review: keep this open. Current main still surfaces generic user-visible context-overflow text across the runner, precheck, shared formatter/sanitizer, and auto-reply fallback paths, while the requested session/channel label, token count, model limit, model identity, and overage are only partly available in logs or metadata.

## What This Changes

Improve context-overflow messages so users can see the affected session/channel, observed context size, model limit, overage, model identity, and concrete recovery actions.

## Best Possible Solution

Add a shared, privacy-conscious context-overflow diagnostic formatter and wire it through runner, precheck, formatter/sanitizer, and auto-reply paths with focused regression coverage for user-visible text and metadata preservation.

## Reproduction Assessment

Yes, from source inspection: current main has clear overflow paths that still construct fixed user-visible messages while nearby code logs or stores richer diagnostics. I did not run a live Discord overflow scenario because this review is read-only.

## Solution Assessment

Yes, a shared diagnostic formatter is the narrowest maintainable direction because patching only one branch would leave other current fallbacks generic. Maintainers should decide the safe channel/session label policy before implementation.
