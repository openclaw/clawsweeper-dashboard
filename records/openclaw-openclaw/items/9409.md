## Summary

Keep open: current main and the latest release still emit generic user-visible context-overflow copy in the runner, precheck, shared formatter/sanitizer, and auto-reply fallbacks. Diagnostics now exist in logs, status metadata, and related budget work, but they are not wired into the overflow message requested here; the direct implementation PR at https://github.com/openclaw/openclaw/pull/15749 is closed unmerged, and broader visibility or budget PRs do not supersede this narrower diagnostic request.

## What This Changes

The issue requests richer user-visible context-overflow diagnostics with session/channel identity, token counts, model limit, overage, and actionable recovery steps.

## Best Possible Solution

Define a redaction-aware shared overflow diagnostic formatter and route it through the runner, precheck/mid-turn precheck, shared formatter/sanitizer, and auto-reply fallback surfaces with safe omissions when fields are unavailable.

## Reproduction Assessment

Yes, at source level: current main still returns fixed generic overflow strings while logging or storing provider, model, token, and session diagnostics separately. I did not run a live oversized-model request in this read-only review.

## Solution Assessment

Yes, the requested direction is maintainable if it is implemented as one shared redaction-aware diagnostic formatter. A raw session-key dump or a single runner-only string edit would not be the best way to solve it.
