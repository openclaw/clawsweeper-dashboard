## Summary

Keep this issue open. Current main records detailed overflow diagnostics in logs, but the user-visible runner, precheck, sanitizer, and auto-reply paths still emit generic messages without the requested model, effective limit, token usage, overage, or privacy-safe conversation label. The merged logging work in https://github.com/openclaw/openclaw/pull/84676 improves operator observability only, while the direct implementation in https://github.com/openclaw/openclaw/pull/15749 closed unmerged and no viable open PR owns the full request.

**Likely related people**
- `steipete` — high confidence; recent adjacent owner of overflow recovery, token snapshots, and reply fallback behavior.
- Vincent Koc — medium confidence; contributor to the shared error formatter and user-facing sanitizer.
- Gio Della-Libera and Tak Hoffman — medium confidence; contributors to the preemptive overflow diagnostic and routing paths.

## What This Changes

Improve user-visible context-overflow errors with safe session context, token usage, model limits, overage, and actionable recovery guidance.

## Best Possible Solution

Introduce one typed, privacy-safe overflow diagnostic result and formatter with optional provider/model, effective context budget, observed-or-estimated usage, computed overage, and redacted conversation label, then reuse it consistently across runner, precheck, sanitizer, and auto-reply fallback paths.

## Reproduction Assessment

Yes at source level. Current main has several concrete user-visible overflow branches whose fixed strings omit the requested fields, although this read-only review did not force a live provider overflow.

## Solution Assessment

Yes in overall direction, but not as direct interpolation of raw session keys or unqualified token values. A shared typed formatter with privacy-safe labels and explicit observed-versus-estimated semantics is the narrowest maintainable solution.
