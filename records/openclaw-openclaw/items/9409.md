## What This Changes

The issue asks for context-overflow user messages to include the affected session/channel, token counts, model limit, model identity, overage, and clearer recovery actions.

## Best Possible Solution

Add one shared redaction-aware overflow diagnostic formatter and route it through runner, precheck/mid-turn precheck, shared formatter/sanitizer, and auto-reply fallbacks with safe omissions when fields are unavailable.

## Reproduction Assessment

Yes, at source level: current main still returns fixed generic overflow strings while logging or storing provider, model, token, and session diagnostics separately. I did not run a live oversized-model request in this read-only review.

## Solution Assessment

Yes, the requested direction is maintainable if implemented as one shared redaction-aware diagnostic formatter. A raw session-key dump or runner-only string edit would not be the best solution.
