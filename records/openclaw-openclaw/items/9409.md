## Summary

Current main still exposes generic context-overflow guidance to users. Richer session, model, and token facts now exist in logs and recovery internals, but no merged or viable open PR wires them safely into user-facing replies.

## What This Changes

Improve context-overflow replies with safe session, token-budget, model, overage, and actionable recovery details.

## Best Possible Solution

Define a structured overflow diagnostic carrying provider/model, effective budget, measured-or-estimated prompt size, overage, and a redacted conversation display label, then render it once at the final user-facing boundary across runner, precheck, sanitizer, and auto-reply paths.

## Solution Assessment

No, the issue’s literal raw-session-key example is not the safest final design; a shared redaction-aware formatter with explicit measured-versus-estimated token semantics is the narrowest maintainable solution.
