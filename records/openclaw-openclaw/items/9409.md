## Summary

Keep open. Current `main` still emits generic context-overflow copy in the runner, precheck, shared formatter/sanitizer, and auto-reply fallback paths; the direct implementation at https://github.com/openclaw/openclaw/pull/15749 is closed unmerged, and the broader visibility request at https://github.com/openclaw/openclaw/issues/67690 does not supersede this narrower diagnostic request.

## What This Changes

The issue requests richer user-facing context overflow diagnostics, including session/channel identity, token counts, model limit, model identity, overage, and actionable recovery steps.

## Best Possible Solution

Add a shared privacy-conscious context-overflow diagnostic formatter and wire it through runner, precheck, shared formatter/sanitizer, and auto-reply fallback paths with focused regression coverage.

## Reproduction Assessment

Yes for the current behavior: source inspection on current main shows multiple overflow paths constructing fixed user-visible messages while nearby code logs or stores richer diagnostics. I did not run a live overflow scenario because this is feature triage and the source path is sufficient to prove the gap.

## Solution Assessment

Unclear until maintainers choose the safe session/channel label policy. Once that policy is set, a shared formatter is the narrowest maintainable solution because one-off copy in a single runner branch would miss current fallback paths.
