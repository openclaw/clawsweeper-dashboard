## Summary

Current main and v2026.6.5 still normalize terminal context-overflow failures to generic recovery copy, so the requested user-visible diagnostic remains missing. The merged logging work in https://github.com/openclaw/openclaw/pull/84676 provides useful token, model, and session facts only in debug logs; the direct implementation at https://github.com/openclaw/openclaw/pull/15749 and later messaging attempt at https://github.com/openclaw/openclaw/pull/85337 both closed unmerged, while https://github.com/openclaw/openclaw/issues/67690 is a broader visibility/control request rather than a safe canonical replacement.

## What This Changes

Improve context-overflow errors with safe session, token-count, model-limit, overage, and actionable recovery details.

## Best Possible Solution

Define a shared typed overflow diagnostic that carries provider/model, clearly labeled estimated prompt size, effective usable budget, computed overage, and a redacted human display label, then render it consistently at every terminal user-facing overflow boundary without exposing raw routing identifiers.

## Reproduction Assessment

Yes—source inspection gives a high-confidence path: any exhausted embedded-runner overflow, precheck overflow, sanitized provider overflow, or external-run overflow fallback produces generic copy without the requested diagnostics. I did not execute a live provider overflow because the deterministic source paths are sufficient for this feature request.

## Solution Assessment

No, not exactly as proposed: surfacing raw session keys and an unlabeled exact “context size” would be unsafe and potentially misleading. The maintainable solution is one shared redacted formatter using authoritative or explicitly estimated budget fields across all terminal paths.
