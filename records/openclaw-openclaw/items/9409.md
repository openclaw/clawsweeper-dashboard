## Label Justifications

- `P2`: This is a normal-priority core diagnostic improvement for disruptive but recoverable context failures with a bounded blast radius.
- `impact:session-state`: The issue concerns identifying and recovering a session whose accumulated context exceeds its active model budget.

## Summary

Current main still emits generic context-overflow text across the terminal runner, prechecks, shared error normalization, and auto-reply fallbacks. Detailed budget data now exists in debug logs, but the requested safe user-visible diagnostics remain unimplemented, and no viable open PR currently owns the work.

## What This Changes

Improve context-overflow errors with safe session identification, token and model limits, overage, and actionable recovery guidance.

## Best Possible Solution

Define one structured overflow diagnostic carrying provider/model identity, estimated prompt pressure, effective budget, computed overage, and a safe conversation display label, then render it consistently at terminal runner and reply boundaries while retaining generic sanitization for untrusted upstream errors.

## Reproduction Assessment

Yes at source level. Current-main branches and regression tests deterministically produce generic overflow text after known overflow conditions, although a live provider overflow was not executed during this read-only review.

## Solution Assessment

No, not verbatim as proposed. The diagnostic goal is appropriate, but the maintainable solution should use one shared safe formatter, distinguish estimates and effective budgets from the native model limit, and avoid exposing raw session identifiers.
