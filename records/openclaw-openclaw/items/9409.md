## Summary

Keep open. Current main and the latest shipped release still surface generic user-visible context-overflow text, while the requested session/channel label, token count, model limit, model identity, and overage remain only logged or stored in metadata on some paths; the direct implementation PR is closed unmerged and the broader visibility issue does not replace this narrower diagnostic request.

## What This Changes

The issue requests a more diagnostic user-facing context overflow message that includes session/channel, token counts, model limit, model identity, overage, and recovery steps.

## Best Possible Solution

Adopt one shared, redaction-aware overflow diagnostic formatter used by the runner, precheck, shared error formatting, sanitizer, and auto-reply fallbacks, with safe fallbacks when token or model details are unavailable.

## Reproduction Assessment

Yes, at source level: current main still returns fixed generic overflow strings while logging or storing provider/model/token diagnostics separately. I did not run a live oversized-model request in this read-only review.

## Solution Assessment

No, the current implementation is not the best end state because it hides the useful diagnostics from the user. The requested direction is maintainable if implemented as a shared formatter, but the raw session/channel identifier example needs a safe display policy first.
