## Summary

Codex review: keep this open. Current main still emits generic user-facing context-overflow text in the runner, precheck, shared formatter/sanitizer, and auto-reply fallback paths; diagnostics exist in logs or metadata, but the requested session/channel label, token count, model limit, model identity, and overage are not surfaced consistently to users.

## What This Changes

Request to replace generic context-overflow text with diagnostic details showing the affected session/channel, observed context size, model limit, model identity, overage, and concrete recovery actions.

## Best Possible Solution

Add a shared, privacy-conscious context-overflow diagnostic formatter and wire it through runner, precheck, formatter/sanitizer, and auto-reply paths with focused regression tests for user-visible text and metadata preservation.

## Reproduction Assessment

## Solution Assessment

Yes, a shared formatter is the narrowest maintainable direction for this diagnostic, because patching only one overflow branch would leave other current fallbacks generic. Maintainers still need to decide the safe redaction and channel/session labeling policy before implementation.
