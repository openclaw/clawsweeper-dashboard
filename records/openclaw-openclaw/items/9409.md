## Summary

Current main still converts context-overflow failures into generic user-facing copy; detailed budget state exists only in logs/session metadata, the direct implementation PR closed unmerged, and no viable canonical item supersedes this narrower diagnostic request.

## What This Changes

Requests a more specific user-facing context-overflow diagnostic with session, token-budget, model, overage, and recovery details.

## Best Possible Solution

Carry one structured overflow diagnostic through the terminal outcome and render privacy-safe copy that clearly labels estimated prompt size, effective budget or known model limit, computed overage, model identity, a redacted session display label, and supported recovery actions.

## Reproduction Assessment

Yes at source level: current-main branches and regression tests deterministically normalize overflow failures to generic copy while detailed context-budget state is stored separately.

## Solution Assessment

No, not verbatim: the requested outcome is useful, but the maintainable solution should avoid raw identifiers and exact-token claims, using the existing structured status consistently across every user-facing overflow path.
