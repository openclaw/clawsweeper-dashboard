## Summary

Keep open. Current main still emits generic user-visible context-overflow copy in the runner, precheck, shared formatter/sanitizer, and auto-reply fallback paths; the closed direct implementation PR did not land, and the broader visibility issue does not supersede this narrower diagnostic request.

## What This Changes

The issue requests a more diagnostic context-overflow error with session/channel identity, token usage, model limit, model identity, overage, and recovery steps.

## Best Possible Solution

Add a shared privacy-conscious context-overflow diagnostic formatter, then wire it through runner, precheck, formatter/sanitizer, and auto-reply fallback paths with focused regression coverage.

## Reproduction Assessment

Yes for current behavior: source inspection on current main shows fixed user-visible overflow messages while nearby code logs or stores richer token/model/session details. I did not run a live overflow scenario because the source path is sufficient for this feature triage.

## Solution Assessment

Yes on direction, but only after maintainers choose the safe session/channel label policy. A shared formatter is the narrowest maintainable solution because one-off copy in only run.ts would miss current fallback and sanitizer paths.
