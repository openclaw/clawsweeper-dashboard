## Summary

Keep open. Current main still emits generic user-visible context-overflow messages across the runner, precheck, shared formatter/sanitizer, and auto-reply fallback paths; token/model/session diagnostics are logged or stored in metadata in places but are not surfaced in the error copy requested here.

## What This Changes

The issue asks for context-overflow errors to show session/channel identity, context size, model limit, model identity, overage, and concrete recovery actions.

## Best Possible Solution

Add one shared, redaction-aware overflow diagnostic formatter and use it consistently from the runner, precheck, shared error formatting, sanitizer, and auto-reply fallbacks with safe fallbacks when details are unavailable.

## Reproduction Assessment

Yes, at source level: current main still returns fixed generic overflow strings while nearby code logs or stores provider/model/token diagnostics separately. I did not run a live oversized request in this read-only review.

## Solution Assessment

Yes for the direction, but not as a one-off message edit. The maintainable fix is a shared diagnostic formatter with an explicit redaction policy for session/channel labels and optional token/model fields.
