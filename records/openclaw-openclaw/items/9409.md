## Summary

Current main and v2026.6.5 still expose generic recovery copy after context overflow; detailed budget, model, and session facts exist in logs or metadata but are not carried into the user-visible reply. The direct implementation attempts are closed unmerged, and the broader context-visibility issue does not replace this narrower diagnostic request.

## What This Changes

Improve context-overflow replies with safe session identity, context size, model limit and identity, overage, and actionable recovery guidance.

## Best Possible Solution

Define a typed, privacy-safe overflow diagnostic assembled at the budget or terminal decision point, with explicit measured/estimated semantics, then render it consistently through runner, precheck, shared formatter/sanitizer, and auto-reply fallback paths.

## Reproduction Assessment

Yes at source level: current main computes relevant overflow facts and then returns generic user-facing text, with tests preserving that generic behavior. This read-only review did not deliberately fill a live provider session to force an overflow.

## Solution Assessment

The user outcome is worthwhile, but the literal proposal is not yet the best implementation because raw session identity and ambiguous token-limit terminology need one maintainer-approved, privacy-safe diagnostic contract.
