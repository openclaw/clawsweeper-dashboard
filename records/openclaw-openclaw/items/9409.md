## Summary

Keep open. Current main and v2026.5.12 still expose generic user-visible context-overflow copy; the useful token/model/session diagnostics are logged or stored as metadata, and the closed direct PR plus broader context-visibility issues do not supersede this narrower diagnostic request.

## What This Changes

The issue asks for context-overflow errors to include session/channel identity, token usage, model limit, model identity, overage, and clearer recovery actions.

## Best Possible Solution

Adopt one shared, redaction-aware overflow diagnostic formatter used by runner, precheck, shared error formatting, sanitizer, and auto-reply fallbacks, with safe fallbacks when token or model details are unavailable.

## Solution Assessment
