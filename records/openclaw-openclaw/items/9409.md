## Summary

Keep open. Current main still sends generic context-overflow text on the runner, precheck, shared formatting/sanitization, and auto-reply fallback paths; diagnostics exist in logs/metadata, but the requested user-visible session label, token count, model limit, model identity, and overage are not wired through. The direct implementation PR is closed unmerged, and the broader context-visibility issue does not supersede this narrower diagnostic request.

## What This Changes

The issue requests richer, user-visible context-overflow diagnostics including the affected session/channel, token count, model limit, model identity, overage, and recovery steps.

## Best Possible Solution

Add one shared overflow-diagnostic formatter that accepts safe session display data, provider/model identity, effective context budget, observed or estimated prompt tokens, and computed overage, then reuse it from the runner, precheck/mid-turn precheck, shared formatter/sanitizer, and auto-reply fallbacks with focused regression coverage.

Yes, a shared safe formatter wired through all overflow surfaces is the narrowest maintainable fix. The closed direct PR only patched one branch and had review-noted edge cases around unsafe identifiers and missing/invalid limits, so it should be treated as source context rather than the final solution.
