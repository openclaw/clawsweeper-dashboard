## Summary

Keep open. Current main and the latest release still send generic context-overflow text to users in the runner, shared error formatting, sanitizer, and auto-reply fallback paths; newer work added logs/session metadata, but it does not surface the requested session/channel label, token count, model limit, model identity, or overage in the visible error message.

## What This Changes

The issue asks OpenClaw to replace generic context-overflow copy with diagnostic, actionable details about the affected session, context size, model limit, overage, model, and recovery actions.

## Best Possible Solution

Add one redaction-aware context-overflow diagnostic formatter for user-visible delivery and wire it through the runner, precheck/attempt handling, shared formatter/sanitizer, and auto-reply fallbacks with focused regression coverage.

## Reproduction Assessment

Yes from source inspection, though I did not run a live overflow. Current main still routes multiple context-overflow paths to generic visible copy while logging or storing richer budget facts elsewhere.

## Solution Assessment

Yes in direction, but the best fix is a shared safe formatter rather than editing one string. Maintainers still need to decide exactly which session/channel identifiers may be shown to users and how they should be redacted or labeled.
