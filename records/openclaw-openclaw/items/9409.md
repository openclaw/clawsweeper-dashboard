## Summary

Keep open. Current main still returns generic context-overflow copy in the runner, precheck, formatter/sanitizer, and auto-reply fallback paths; the direct implementation PR is closed unmerged, and the broader context-visibility issue does not supersede this narrower diagnostic request.

## What This Changes

Request richer user-visible context-overflow diagnostics that include session/channel identity, token counts, model limit, overage, and actionable recovery steps.

## Best Possible Solution

Add a shared, privacy-conscious context-overflow diagnostic formatter and wire it through runner, precheck, formatter/sanitizer, and auto-reply fallback paths with focused regression coverage.

## Reproduction Assessment

## Solution Assessment

Unclear until maintainers choose the safe session/channel label policy. Once that policy is set, a shared formatter is the narrowest maintainable solution because one-off copy in a single branch would miss current fallback paths.
