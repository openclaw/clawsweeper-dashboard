## Summary

Keep open. Current main still emits generic user-visible context-overflow messages across the runner, precheck, formatter/sanitizer, and auto-reply paths; the direct implementation PR is closed unmerged, and the broader context-visibility issue does not supersede this narrower diagnostic request.

## What This Changes

Request to make context-overflow errors show the affected session or channel, token counts, model limit and identity, overage, and concrete recovery steps.

## Best Possible Solution

Add a shared, privacy-conscious context-overflow diagnostic formatter and wire it through runner, precheck, formatter/sanitizer, and auto-reply fallback paths with focused regression coverage.

## Reproduction Assessment

Yes for the current behavior: source inspection on current main shows multiple overflow paths constructing fixed user-visible messages while nearby code logs or stores richer diagnostics. I did not run a live overflow scenario because the source path is sufficient for this feature triage.

## Solution Assessment

Yes on direction, but only after maintainers decide the safe session/channel label policy. A shared formatter is the narrowest maintainable approach because one-off copy in a single branch would miss current fallback paths.
