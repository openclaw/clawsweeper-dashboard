## Summary

Keep open. Current main still shows fixed, generic context-overflow text in the runner, precheck and mid-turn precheck paths, shared formatter/sanitizer, and auto-reply fallbacks; the richer token/model/session facts are logged or stored in metadata but are not user-visible. ## What This Changes

The issue requests a more specific user-facing context overflow error that includes session/channel context, token counts, model limit, model identity, overage, and actionable recovery guidance.

## Best Possible Solution

Add a shared privacy-conscious context-overflow diagnostic formatter and wire it through runner, precheck, formatter/sanitizer, and auto-reply fallback paths with focused regression coverage.

## Solution Assessment
