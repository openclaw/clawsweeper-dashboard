## What This Changes

The issue requests extending `openclaw models status --probe` or adding a Codex-specific status command that reports Codex app-server viability beyond generic OAuth/profile metadata.

## Best Possible Solution

Define a Codex-aware diagnostic contract that reuses the existing auth-probe and Codex harness status helpers, reports effective profile/order, refresh/app-server/trivial-turn/fallback state, and redacts sensitive auth details.
