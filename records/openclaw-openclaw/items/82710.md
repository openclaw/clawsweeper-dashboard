## Summary

Keep open: current main still has a generic `models status --probe` live auth probe plus a separate `/codex status` app-server control-plane check, but it does not provide the requested Codex viability matrix for refresh, selected profile/order, app-server startup, trivial turn completion, last runtime failure, or fallback usability. The related work covers important pieces of Codex routing, refresh classification, timeout handling, and status labels, but it does not supersede this product-facing diagnostic contract.

## What This Changes

Request to extend `openclaw models status --probe` or add a clearly named Codex diagnostic command that reports Codex OAuth, profile order, app-server startup, trivial turn, last failure, and fallback-chain viability.

## Best Possible Solution

Define a Codex-aware diagnostic contract that reuses the existing auth-probe and Codex app-server control surfaces, reports selected profile/order, refresh, startup, trivial-turn, last-failure, and fallback state, and keeps all secret/profile data redacted.

## Reproduction Assessment

Not applicable as a feature request. Source inspection gives a high-confidence absence path: current main has generic live auth probes and `/codex status`, but no combined Codex app-server/fallback viability report.

## Solution Assessment

Yes directionally, but the implementation should wait for maintainer agreement on the diagnostic contract and side effects. The maintainable path is to extend the existing probe/status schemas and Codex harness helpers rather than add an ad hoc parallel report.
