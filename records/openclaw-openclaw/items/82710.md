## Reproduction Assessment

## Solution Assessment

Yes directionally, but the best solution needs maintainer agreement on the output contract and on whether `models status` should start Codex app-server turns. A maintainable implementation should keep Codex-specific logic in the Codex plugin and expose it through a narrow status-probe seam rather than duplicating runtime behavior in the CLI.
