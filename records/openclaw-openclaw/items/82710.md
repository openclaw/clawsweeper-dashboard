## What This Changes

Request to extend `openclaw models status --probe` or a Codex-specific status command with structured Codex app-server viability diagnostics for auth, selected profile/order, token refresh, startup, trivial turns, runtime failures, and fallback state.

## Best Possible Solution

Have maintainers decide the diagnostic contract, likely by reviving or reworking https://github.com/openclaw/openclaw/pull/82996 so the generic probe surface can call a plugin-owned Codex harness status probe with secret-redacted output and focused regression coverage.

## Solution Assessment

Unclear until maintainers accept the product/API contract. The closed implementation direction looks maintainable because it keeps Codex-specific checks in the Codex plugin, but expanding the public harness status-probe surface needs owner review before automation.
