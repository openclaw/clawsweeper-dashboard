## Summary

Keep open: current main already has live `models status --probe` auth checks, but this request asks for a richer Codex-specific diagnostic that separates profile metadata, refresh viability, selected route/profile, app-server startup/turn success, last runtime failures, and fallback-chain usability in one operator-facing probe.

## What This Changes

Requested a stronger Codex/OpenAI model status probe that separates OAuth/profile health from real Codex app-server turn viability and fallback usability.

## Best Possible Solution

Extend the existing `models status --probe` path, or add a clearly named subcommand, so Codex routes report metadata health, refresh/auth viability, selected profile/order, app-server startup and trivial-turn result, recent runtime failure, and fallback-chain usability without duplicating the separate `/codex status` surface.

## Reproduction Assessment

Not applicable as a feature request; source inspection shows the current probe surface is generic live auth probing and does not expose the requested Codex-specific app-server/fallback fields.

## Solution Assessment

Yes directionally, but the maintainable path is to build on the existing probe/status schemas and Codex harness diagnostics rather than adding an ad hoc parallel probe output.
