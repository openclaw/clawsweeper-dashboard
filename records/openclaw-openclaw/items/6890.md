## Summary

Keep open: current main still lacks a Ralph-style repeated same-prompt loop and per-agent Ralph iteration default. Adjacent `runRetries`, tool-loop detection, and agent-to-agent turn caps cover different failure-retry or tool-safety paths, so this remains a product/API design item rather than conservative cleanup.

## What This Changes

The issue requests a Ralph-style agent loop setting in agent specs with a per-agent max iteration default and precedence against CLI and global configuration.

## Best Possible Solution

Define one OpenClaw-owned continuation and budget contract that separates intentional repeated-prompt workflows from accidental tool loops, then expose the smallest approved typed config surface with documented precedence and terminal-outcome semantics.

## Reproduction Assessment

Yes for the missing capability, not as a broken-behavior bug. Current source has no Ralph/repeated-prompt loop field, strict schemas reject unknown agent keys, and the resolver has no setting to propagate to the runner.

## Solution Assessment

Unclear as proposed. The user need fits agent-harness work, but copying Kimi-style field names, STOP sentinels, and unlimited-loop behavior into core should wait for an OpenClaw-native continuation and budget contract.
