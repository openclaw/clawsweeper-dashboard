## Summary

Current main still lacks deliberate same-prompt continuation until an explicit stop and lacks a per-agent iteration default. The request is distinct from the open runaway-tool-loop safeguards, but its termination signal, unlimited mode, interruption behavior, and configuration precedence establish a new core orchestration contract that requires maintainer product judgment.

## What This Changes

Add an agent-level default and precedence rules for a STOP-driven loop that repeatedly runs the same prompt up to a configurable iteration limit.

## Best Possible Solution

Design one runner-owned continuation and run-budget contract that distinguishes intentional repeated-prompt workflows from accidental tool loops, uses typed terminal outcomes rather than raw text inference, and exposes the smallest approved global, per-agent, or plugin surface with explicit override and interruption semantics.

## Reproduction Assessment

Not applicable: this issue requests a new execution mode and configuration contract rather than reporting broken behavior already defined by OpenClaw.

## Solution Assessment

No, not as currently proposed. The user need is valid, but directly copying Kimi's raw STOP sentinel, unlimited sentinel, and precedence chain would create a parallel orchestration contract before OpenClaw defines runner-owned termination and interruption semantics.
