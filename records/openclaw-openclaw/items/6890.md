## Summary

The central feature remains missing: current main supports per-agent limits for failure-recovery retries, but not intentional same-prompt reruns until an explicit STOP. This requires maintainer product judgment because loop lifecycle, interruption, safety, and core-versus-plugin ownership remain undefined.

## What This Changes

Add a per-agent default for a bounded Ralph-style repeated-prompt loop with CLI-over-agent-over-global precedence.

## Best Possible Solution

Define a generic typed intentional-repeat execution contract with finite safe defaults, explicit STOP, cancellation, interruption, and persistence semantics, and one canonical precedence path; place it in core only if existing plugin hooks cannot safely express it without duplicating orchestration.

## Reproduction Assessment

Not applicable. This issue requests a new execution mode and configuration contract; current main has no documented Ralph-style behavior whose failure can be reproduced.

## Solution Assessment

No. The Kimi-shaped field proposal is not yet the narrowest maintainable OpenClaw solution because it omits lifecycle semantics and risks creating a second loop path beside recovery retries; a generic execution contract and ownership decision should come first.
