## Summary

Keep open. Current main and v2026.6.6 still lack the requested Ralph-style repeated same-prompt mode and per-agent Ralph iteration default; adjacent `runRetries` and `tools.loopDetection` cover retry/failure recovery and accidental tool-loop detection, not an intentional STOP-driven continuation contract.

## What This Changes

The issue requests an OpenClaw-native Ralph-style repeated-prompt loop with a per-agent maximum iteration default and clear CLI/agent/global precedence.

## Best Possible Solution

Define one runner-owned continuation and budget contract that separates intentional repeated-prompt workflows from accidental tool loops, then expose the smallest approved typed config/API surface with clear precedence and terminal-outcome semantics.

## Reproduction Assessment

No for a bug reproduction; this is a new execution-mode/config request. Source inspection gives a high-confidence reproduction of the missing capability because current strict schemas, resolver, docs, and runner-adjacent surfaces have no Ralph loop field or STOP-driven continuation mode.

## Solution Assessment

Unclear as proposed. The user need is valid, but copying Kimi's raw STOP sentinel and unlimited iteration shape into core should wait for an OpenClaw-owned typed continuation, interruption, and budget contract.
