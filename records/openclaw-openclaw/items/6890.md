## Summary

Keep open: current main and v2026.6.10 still do not provide a Ralph-style repeated same-prompt loop or a per-agent Ralph iteration default, and the remaining work needs maintainer product/API design rather than conservative cleanup.

## What This Changes

The issue requests a per-agent Ralph/repeated-prompt iteration default with explicit CLI, agent, global, and built-in precedence.

## Best Possible Solution

Define one OpenClaw-owned continuation and budget contract that separates intentional repeated-prompt workflows from accidental tool loops, then expose the smallest approved typed config surface with documented precedence and terminal-outcome semantics.

## Reproduction Assessment

Yes for the missing capability, not as a bug reproduction. Current source and docs have no Ralph/repeated-prompt loop field, strict schemas reject unknown agent keys, and the resolver has no setting to propagate to the runner.

## Solution Assessment

Unclear as proposed. The user need fits agent-harness work, but copying Kimi-style field names, STOP sentinels, and unlimited-loop behavior into core should wait for an OpenClaw-native continuation and budget contract.
