## Summary

Keep open. Current main still lacks a Ralph/STOP-driven repeated-prompt loop and a per-agent max-iteration default; related #75468 is a broader safeguards request and does not supersede this narrower feature. Existing `tools.loopDetection` only covers repetitive tool-call patterns, and `max_tool_calls` in OpenResponses is currently documented as ignored.

## What This Changes

The issue requests a per-agent Ralph Loop iteration limit with CLI, agent, global, and default precedence.

## Best Possible Solution

Keep this open for an OpenClaw-native loop contract decision; if accepted, add typed config, schema/help docs, resolver/runtime wiring, and focused precedence and safety tests.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main gives high confidence that the requested per-agent Ralph loop/default surface is absent.

## Solution Assessment

Unclear until maintainers decide the product/API semantics. A per-agent default is plausible, but it should align with OpenClaw's current config model and safeguards rather than porting the Kimi names directly.
