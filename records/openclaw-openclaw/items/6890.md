## Summary

Keep open: current main and v2026.6.10 still lack a Ralph-style repeated same-prompt loop and a per-agent Ralph iteration default; the remaining work is a new agent-runtime/config contract that needs maintainer product design.

## What This Changes

The issue requests a per-agent Ralph Loop iteration limit with CLI, agent, global, and default precedence.

## Best Possible Solution

Define one OpenClaw-owned continuation and budget contract that separates intentional repeated-prompt workflows from accidental tool loops, then expose the smallest approved typed config surface with documented precedence and terminal-outcome semantics.

## Reproduction Assessment

Yes for the missing capability, not as a bug reproduction. Current source and release checks show no Ralph/repeated-prompt loop field, strict schemas reject unknown agent keys, and the resolver has no setting to propagate to runtime.

## Solution Assessment

Unclear as proposed. The user need fits agent-harness work, but copying Kimi-style field names, STOP sentinels, and unlimited-loop behavior into core should wait for an OpenClaw-native continuation and budget contract.
