## Summary

Keep open. Current main and v2026.6.6 still lack Ralph-style repeated same-prompt continuation and a per-agent Ralph iteration default; the existing `runRetries` and `tools.loopDetection` surfaces cover failure recovery and accidental tool-loop detection, not the requested intentional STOP-driven mode.

## What This Changes

The issue requests an OpenClaw-native Ralph Loop mode with a per-agent max iteration default and CLI, agent, global, and default precedence.

## Best Possible Solution

Define one runner-owned continuation and budget contract that separates intentional repeated-prompt workflows from accidental tool loops, then expose the smallest approved typed config/API surface with clear override, interruption, and terminal-outcome semantics.

## Reproduction Assessment

Not applicable: this is a request for a new execution mode and configuration contract, not broken behavior already defined by OpenClaw. Source inspection confirms the requested capability is absent from current main.

## Solution Assessment

No, not as currently proposed. The user need is valid, but copying Kimi's raw STOP sentinel, unlimited sentinel, and precedence chain would create a parallel orchestration contract before OpenClaw defines typed runner-owned termination and interruption semantics.
