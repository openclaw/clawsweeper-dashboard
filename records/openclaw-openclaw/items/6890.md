## Summary

Keep open. Current main still lacks a Ralph Loop implementation and still has no per-agent `loop.max_ralph_iterations` or equivalent iteration-limit surface; the existing loop-related code is tool-loop protection or strict-agentic retry behavior, not STOP-driven repeated same-prompt execution with the requested precedence.

## What This Changes

Request to add an OpenClaw-native Ralph Loop feature with a per-agent iteration limit and CLI > agent > global precedence.

## Best Possible Solution

Keep this open for maintainer product/API judgment. If accepted, add an OpenClaw-native typed config surface, strict schema/help/docs support, resolver propagation, and runtime wiring with explicit override precedence and safety tests; if maintainers want this outside core, first define the plugin/runtime API needed to implement it safely.

## Reproduction Assessment

Not applicable as a feature request, but there is a high-confidence source check for the missing surface: current schemas are strict, no Ralph symbols exist, and `resolveAgentConfig` has no loop setting to pass to runtime.

## Solution Assessment

Unclear until maintainers decide the product semantics. The proposed per-agent default is plausible, but the maintainable OpenClaw solution should be a native config/runtime contract with safety limits rather than a direct port of the old Kimi file names in the report.
