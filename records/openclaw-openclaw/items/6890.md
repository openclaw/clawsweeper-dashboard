## Summary

Keep open. Current main still does not provide a Ralph Loop or per-agent `loop.max_ralph_iterations` equivalent, and the newer `runRetries`, strict-agentic, and tool-loop guard surfaces are adjacent safeguards rather than a STOP-driven repeated-prompt loop with the requested precedence.

## What This Changes

Add a per-agent Ralph Loop iteration-limit default/config surface with CLI > agent > global precedence.

## Best Possible Solution

Define an OpenClaw-native loop/default contract in the agent config schema and resolver, document its precedence and safety semantics, and wire it only after maintainers decide whether Ralph-style repeated prompting belongs in core or behind a plugin/runtime API.

## Reproduction Assessment

Not applicable as a feature request. The source/docs check does show with high confidence that the requested Ralph/per-agent loop config surface is absent on current main.

## Solution Assessment

Unclear. The requested direction is plausible, but current `runRetries`, strict-agentic, and tool-loop detection are not substitutes; maintainers should choose the public config shape and runtime semantics before implementation.
