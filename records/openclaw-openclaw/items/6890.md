## Summary

Keep open. Current main and v2026.6.8 still lack a Ralph-style repeated same-prompt loop and per-agent Ralph iteration default; the remaining work is a product-sensitive agent runtime/config contract, not conservative cleanup.

## What This Changes

The issue requests a per-agent Ralph Loop iteration limit with CLI-over-agent-over-global precedence and matching docs/tests.

## Best Possible Solution

Define one typed OpenClaw continuation and budget contract that separates intentional repeated-prompt workflows from accidental tool loops, then expose the smallest approved config surface with documented precedence and terminal-outcome semantics.

## Reproduction Assessment

Yes for the missing capability, not as a bug reproduction. Current source and the latest release tag have no Ralph/repeated-prompt loop field, strict schemas reject unknown agent keys, and the resolver has no setting to propagate to the runner.

## Solution Assessment

Unclear as proposed. The user need fits OpenClaw's agent-harness direction, but copying Kimi-style field names, STOP sentinels, and unlimited-loop behavior into core should wait for a typed OpenClaw continuation and budget contract.
