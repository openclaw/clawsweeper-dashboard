## Summary

Keep open: the report describes a serious current macOS Gateway restart loop on the latest shipped release, and current `main` does not contain a clear post-release fix for the implicated restart files. Source inspection maps the logged warning to stale-listener cleanup that runs after a restart has already been requested, so the remaining work is to identify what is repeatedly requesting the restart and whether the stale-PID classifier is falsely selecting the active LaunchAgent child.

## What This Changes

The issue reports repeated macOS Gateway restarts every 3-5 minutes on OpenClaw 2026.5.12, with healthy Telegram traffic interrupted by stale-gateway cleanup.

## Best Possible Solution

Capture a macOS LaunchAgent repro with restart trace and deep status, then patch the specific repeated restart trigger or stale-PID classification path with focused regression coverage.

## Reproduction Assessment

No, not yet. The report has credible live logs on 2026.5.12, and source inspection maps the warning to cleanup-before-restart, but we do not yet have the restart requester that fires every 3-5 minutes on current main.

## Solution Assessment

Unclear. Treating this only as stale-PID detection is premature; the safer fix path is to trace the restart origin first, then change either the requester or the PID classifier that is proven wrong.
