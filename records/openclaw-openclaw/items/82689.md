## Summary

Keep open. Current main still has a source-visible path where service-mode startup runs stale-gateway cleanup before the supervised duplicate-start logic can leave an existing healthy launchd gateway in control; the adjacent Telegram wedge fix is useful context but does not cover this stale-PID kill path.

## What This Changes

Report that a macOS LaunchAgent gateway is repeatedly detected as stale and restarted every few minutes even after health-monitor settings are disabled.

## Best Possible Solution

Honor the existing supervised healthy-owner contract before service-mode stale cleanup sends SIGTERM, while keeping stale orphan cleanup for dead or unhealthy listeners.

## Reproduction Assessment

Yes, partially from source inspection: a supervised service-mode duplicate can run stale-PID cleanup before the healthy-owner lock recovery path, matching the reported SIGTERM warning sequence. The exact recurring 3-5 minute trigger still needs live macOS diagnostics.

## Solution Assessment

Yes, the narrow maintainable fix is to protect an existing healthy supervised gateway before stale cleanup kills another OpenClaw PID. Treating this only as Telegram health-monitor configuration would not address the reported restart subsystem warning.
