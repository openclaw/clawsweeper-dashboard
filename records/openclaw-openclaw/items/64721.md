## Summary

Keep open. Current main and v2026.6.6 cover `model` and `timeoutSeconds`, but `contextTokens` and `maxSpendUsd` are still absent from the cron tool, strict Gateway protocol, runtime payload type, storage codec, CLI surfaces, and execution semantics, so the remaining work needs maintainer API/product direction.

## What This Changes

The issue asks for cron add/update tooling to accept and preserve per-job model, timeoutSeconds, contextTokens, and maxSpendUsd fields.

## Best Possible Solution

Define one canonical per-job cron budget/model contract, then update gateway protocol, cron payload types, normalization, storage, runtime, agent tool schema, CLI add/edit behavior, docs, and focused tests together.

## Reproduction Assessment

Yes. Source inspection shows the remaining contextTokens/maxSpendUsd gap across the strict protocol, tool schema, payload type, storage codec, CLI surfaces, and runtime context handling.

## Solution Assessment

No. Adding fields only to the model-facing cron tool schema would be too narrow because the Gateway protocol, runtime type, storage codec, CLI surfaces, docs, and execution semantics also need a coherent contract.
