## Summary

Keep open. Current main has already added the model and timeoutSeconds pieces, but the requested per-job contextTokens and maxSpendUsd contract is still missing from the cron tool/schema, gateway protocol, runtime type surface, CLI/docs, and run-time behavior, so this is not safe cleanup closure.

## What This Changes

The issue requests cron add/update support for per-job model, timeout, context-token, and spend-limit fields across the agent-facing cron tool and related cron surfaces.

## Best Possible Solution

Define one canonical per-job cron budget/model contract, then update the gateway protocol, cron payload types, normalization, agent tool schema, CLI add/edit behavior, runtime handling, docs, and focused regression tests together.

## Reproduction Assessment

Yes, source-level: current main still omits contextTokens and maxSpendUsd from the strict cron payload contract while cron.add/update validate against that contract. I did not run a live gateway command because the read-only source path is enough to show the remaining fields are unsupported.

## Solution Assessment

No, adding fields only to the model-facing cron tool schema would be too narrow. The maintainable solution is a single cron API/runtime decision that keeps gateway protocol, CLI, docs, validation, storage, and runtime execution in sync.
