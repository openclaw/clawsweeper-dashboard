## Summary

Keep open. Current main and the latest release accept `model` and `timeoutSeconds`, but `contextTokens` and `maxSpendUsd` remain outside the cron payload, protocol, runtime, and CLI contracts, so the remaining request still needs a cron API/runtime product decision.

## What This Changes

Request cron add/update tooling and CLI/API paths to accept per-job `model`, `timeoutSeconds`, `contextTokens`, and `maxSpendUsd` fields for scheduled agent-turn jobs.

## Best Possible Solution

Define one canonical per-job cron budget/model contract, then update gateway protocol, cron payload types, normalization, agent tool schema, CLI add/edit behavior, runtime handling, docs, and focused regression tests together.

## Reproduction Assessment

Yes. Source inspection shows `contextTokens` and `maxSpendUsd` are absent from the strict cron payload/protocol/runtime contract while `cron.add` and `cron.update` validate against that contract.

## Solution Assessment

No. Adding the fields only to the model-facing cron tool schema would be too narrow; the maintainable fix is a coordinated cron API, runtime, CLI, docs, and test update.
