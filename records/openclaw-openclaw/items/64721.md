## Summary

Keep open. Current main and the latest release cover the `model` and `timeoutSeconds` cron payload paths, but `contextTokens` and `maxSpendUsd` are still absent from the strict Gateway cron payload contract, and the related CLI timeout ambiguity is still a real product/API decision rather than a cleanup close.

## What This Changes

The issue requests cron add/update tool and CLI support for agent-turn `model`, `timeoutSeconds`, `contextTokens`, and `maxSpendUsd` payload fields.

## Best Possible Solution

Define one end-to-end cron agent-turn resource-limit contract, then update the tool schema, Gateway schema, types, normalization, runtime precedence, spend-cap behavior, CLI/docs, and regression tests together.

I did not run tests because this review is read-only, but the validation path is clear from source.

## Solution Assessment

No for a schema-only pass-through. The maintainable fix needs resource-limit semantics, validation, runtime precedence, spend enforcement or explicit non-support, CLI timeout disambiguation, docs, and regression coverage.
