## Summary

Keep open. Current main and v2026.5.19 still expose only the already-covered cron `model` and `timeoutSeconds` paths; `contextTokens` and `maxSpendUsd` remain outside the cron payload contract, and the CLI timeout ambiguity from the discussion is still a related open design point.

The issue requests cron add/update tool and related cron management surfaces to accept and preserve per-job `model`, `timeoutSeconds`, `contextTokens`, and `maxSpendUsd` fields for agent-turn jobs.

## Best Possible Solution

Define one end-to-end cron agent-turn resource-limit contract, then update the tool schema, Gateway schema, normalization, runtime precedence, spend-cap enforcement or explicit non-support, CLI wording, docs, and regression tests together.
