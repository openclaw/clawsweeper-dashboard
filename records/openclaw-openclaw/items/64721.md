## Summary

Keep open. ## What This Changes

The issue asks for cron add/update surfaces to accept and persist per-job model, timeout, context-token, and spend-limit fields for scheduled agent turns.

## Best Possible Solution

Define one canonical per-job cron budget/model contract, then update the Gateway protocol, cron payload types, normalization, persistence, isolated-run behavior, agent tool schema, CLI add/edit behavior, docs, and focused regression tests together.

## Solution Assessment

No, not as a cron-tool-schema-only change. The maintainable fix is a coordinated cron API, runtime, persistence, CLI, docs, and test update after maintainers choose the context/spend semantics.
