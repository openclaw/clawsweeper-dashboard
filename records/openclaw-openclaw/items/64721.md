## Summary

Keep open. Current main has the earlier `model` and `timeoutSeconds` cron paths, but `contextTokens` and `maxSpendUsd` remain outside the strict cron payload contract and need maintainer API/product decisions before implementation.

## What This Changes

Accept cron agent-turn resource fields such as `model`, `timeoutSeconds`, `contextTokens`, and `maxSpendUsd` through the agent cron tool, Gateway cron APIs, and related CLI edit flow.

## Best Possible Solution

Define one end-to-end cron agent-turn resource-limit contract, then update the tool schema, Gateway schema, normalization, runtime precedence, spend-cap behavior or explicit non-support, CLI wording, docs, and regression tests together.

## Solution Assessment

No for a schema-only pass-through. The maintainable solution needs explicit context-budget precedence, spend-cap enforcement or documented non-support, CLI timeout disambiguation, docs, and regression coverage.
