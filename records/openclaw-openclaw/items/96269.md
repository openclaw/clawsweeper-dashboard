## Label Justifications

- `P2`: This is a normal-priority session model switching bug with real but bounded user-visible routing impact.
- `impact:session-state`: The failure is missing session switch intent when persisted model state has no stored delta from the configured default.
- `impact:auth-provider`: The affected `/model` path controls provider/model routing for the active session.

## Summary

Keep open: current main and v2026.6.10 still leave an explicit `/model <configured-default>` request without live-switch intent when the stored session entry has no non-default model delta, so the active runtime can keep using the previous model.

## What This Changes

Issue requests that explicit `/model <configured-default>` switch back to the configured default model even when the currently running model differs from that default.

## Best Possible Solution

Keep canonical default storage with no persisted default override, but make an explicit default-model selection record live switch intent when the active runtime may differ.

## Reproduction Assessment

Yes, at source level: send `sessions.patch` a model equal to the configured default while the stored entry has no override/profile delta, and the helper does not set `liveModelSwitchPending`. I did not run the live webchat/macOS reproduction in this read-only review.

## Solution Assessment

Yes, the best fix is a narrow session override/session patch repair that records explicit live switch intent for default selections without storing the configured default as a permanent override or adding config.
