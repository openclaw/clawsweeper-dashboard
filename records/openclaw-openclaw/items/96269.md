## Summary

Keep open: current main now fixes the persisted runtime-field mismatch subcase via the merged helper-level PR, but source still shows an explicit default-model selection can be a no-op when the active runtime differs and the session entry has no stored override/runtime delta to mutate.

## What This Changes

The issue asks for `/model <configured-default>` to switch an active session back to the configured default model instead of silently leaving it on another live model.

## Best Possible Solution

Preserve canonical default storage with no persisted override, but make explicit default-model selections record pending live-switch intent even when no other session-entry field changes.

## Reproduction Assessment

Yes at source level: an explicit default selection with no stored override/runtime delta still cannot set `liveModelSwitchPending`, so an active runtime on another model will not be redirected. I did not run a live webchat/provider repro in this read-only review.

## Solution Assessment

No: the merged helper fix is the right shape for the persisted runtime-field subcase, but it is not yet the complete fix for explicit default selections that produce no stored entry mutation. The safer remaining fix is to record switch intent without writing a default override.
