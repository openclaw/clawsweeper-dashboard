## Summary

Keep open: current main and v2026.6.10 mitigate stale persisted runtime model fields, but an explicit `/model <configured-default>` can still fail to record live switch intent when the session entry has no stored delta even though the active runtime model differs.

## What This Changes

The issue reports that `/model <configured-default>` can silently fail to switch a session back to the configured default model after the active model drifted to another provider/model.

## Best Possible Solution

Preserve canonical default storage with no persisted override, but make an explicit `/model <configured-default>` request create switch intent when the active runtime model may differ.

## Reproduction Assessment

Yes at source level: when `sessions.patch` receives the configured default and the entry has no override/runtime fields to change, the helper returns no update and does not set `liveModelSwitchPending`. I did not run a live webchat reproduction.

## Solution Assessment

No. Current main is only a partial mitigation; the narrower fix is to keep default override storage canonical while still recording explicit switch intent for active sessions.
