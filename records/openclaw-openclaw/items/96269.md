## Summary

Current main still has a source-level path where explicit default-model selection can clear stale runtime fields without setting the live-switch flag, so this issue remains necessary.

## What This Changes

The issue reports that `/model <configured-default>` can fail to switch back when the session is currently running a different model.

## Best Possible Solution

Keep the issue open and fix explicit default-model selections so stale runtime-only state triggers the live-switch path without adding a new config option or reviving broad default-pin semantics.

## Reproduction Assessment

Yes at source level: stale `modelProvider`/`model` fields plus an explicit configured-default selection can update the entry without setting `liveModelSwitchPending`. I did not run the full WebChat flow.

## Solution Assessment

Yes for a narrow repair: the shared session model override or sessions.patch path should mark this explicit default selection as a pending live switch when the current runtime model differs.
