## What This Changes

The branch adds a plugin-install formatter guard and regression test to suppress the hook-pack fallback suffix when a plugin install fails for missing compiled JavaScript runtime output.

## Best Possible Solution

Suppress only the missing-hook-manifest fallback tail, keep other hook validation errors visible, and require redacted real CLI proof before merge.

## Reproduction Assessment

Yes, at source level. Current main routes non-terminal plugin install failures through hook fallback formatting, and the formatter still appends `Also not a valid hook pack` outside its narrow existing special cases.

## Solution Assessment

No. The formatter is the right layer, but the implementation should suppress only the missing-hook-manifest fallback tail instead of dropping every hook error when the plugin error mentions missing compiled runtime output.
