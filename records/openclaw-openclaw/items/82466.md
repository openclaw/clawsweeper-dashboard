## Summary

Keep open: this fixes a real CLI diagnostic noise problem, but the formatter guard is too broad and can hide actionable hook-pack validation errors; contributor-supplied real CLI proof is also still missing.

## What This Changes

Adds a CLI helper guard and regression test to suppress the hook-pack fallback suffix when plugin install reports missing compiled TypeScript runtime output.

## Best Possible Solution

Narrow the suppression to compiled-runtime plugin errors paired with a missing-hook-manifest fallback, preserve other hook validation errors, add focused regression coverage, and require redacted real CLI output before merge.

## Reproduction Assessment

Yes, source-level. Current main routes compiled-runtime plugin failures through the hook fallback formatter, and existing hook tests show plugin-capable hook packages are part of the supported surface.

## Solution Assessment

No. The formatter is the right layer, but this implementation should key off the specific missing-hook-manifest fallback instead of suppressing every hook error when the plugin error mentions missing compiled runtime output.
