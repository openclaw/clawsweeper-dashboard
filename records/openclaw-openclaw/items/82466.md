## Summary

Keep open: the formatter is the right surface and current main/latest release still append the hook-pack fallback tail, but this patch suppresses all hook fallback details for compiled-runtime plugin failures and still lacks real CLI behavior proof.

## What This Changes

The PR adds a CLI plugin-install formatter guard and regression test to suppress the hook-pack fallback suffix when the plugin install error says a TypeScript entry lacks compiled runtime output.

## Best Possible Solution

Suppress only the missing-hook-manifest fallback tail, keep other hook validation errors visible, and require redacted real CLI proof before merge.

## Reproduction Assessment

Yes, at source level. Current main routes non-terminal plugin install failures through the hook fallback formatter, and the linked report shows the combined compiled-runtime plus missing-hooks diagnostic; I did not run a live install in this read-only pass.

## Solution Assessment

No. The formatter is the right layer, but the implementation should key off the specific missing-hook-manifest fallback shape instead of suppressing every hook error for compiled-runtime plugin failures.
