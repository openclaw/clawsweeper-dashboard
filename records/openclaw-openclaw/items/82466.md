## Summary

Keep open: current main still has the noisy plugin/hook fallback formatter behavior, but this PR is not merge-ready because the new guard suppresses all hook fallback details for compiled-runtime plugin errors and the external-PR real CLI proof gate is still missing.

## What This Changes

The branch adds a CLI plugin-install formatter guard and regression test to hide the hook-pack fallback suffix when plugin installation fails for missing compiled JavaScript runtime output.

## Best Possible Solution

Keep the formatter-layer fix, but suppress only the missing `openclaw.hooks` fallback shape, preserve other hook validation details, and require redacted real CLI proof before merge.

## Reproduction Assessment

Yes, at source level. Current main routes non-terminal plugin install failures through hook fallback formatting, and the formatter still appends `Also not a valid hook pack` outside its narrow existing special cases.

## Solution Assessment

No. The formatter is the right layer, but the implementation should suppress only the missing-hook-manifest fallback tail instead of dropping every hook error when the plugin error mentions missing compiled runtime output.
