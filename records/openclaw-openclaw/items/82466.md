## Summary

Keep open: current main still has the noisy combined diagnostic, but this patch is not merge-ready because it suppresses by plugin error alone and can hide actionable hook-pack validation details, and the external contributor has not supplied real CLI proof.

## What This Changes

The PR adds a CLI install formatter guard and regression test to suppress the hook-pack fallback suffix when npm plugin install reports missing compiled runtime output.

## Best Possible Solution

Keep the formatter-layer fix, but suppress only the missing-hook-manifest fallback tail for the compiled-runtime plugin error and preserve other hook validation details.

## Reproduction Assessment

Yes, source-level. Current main routes non-terminal npm plugin failures through the hook fallback formatter, and hook packages can be plugin-capable, so the PR’s plugin-error-only suppression can hide a real sibling diagnostic.

## Solution Assessment

No. The formatter is the right layer, but the implementation should key off the specific missing-hook-manifest fallback instead of suppressing every hook error for compiled-runtime plugin failures.
