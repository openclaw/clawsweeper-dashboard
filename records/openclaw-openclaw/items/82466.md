## Summary

Keep open: the PR targets a real CLI diagnostic problem, but the current guard suppresses hook fallback details based only on the plugin error and can hide actionable hook-pack validation output for plugin-capable packages; it also still lacks contributor-supplied real CLI proof.

## What This Changes

This PR changes plugin install error formatting to suppress the hook-pack fallback suffix when the plugin install failure reports missing compiled runtime output, and adds a regression test for that CLI message.

## Best Possible Solution

Keep the formatter-layer fix, but suppress only the compiled-runtime plus missing-hook-manifest fallback case, preserve other hook validation details, and require redacted real CLI output before merge.

## Reproduction Assessment

Yes, source-level. Current main routes plugin install failures through the hook fallback formatter, and existing hook install coverage proves plugin-capable hook packages are part of the supported surface.

## Solution Assessment

No. The formatter is the right layer, but this implementation should key off the specific missing-hook-manifest fallback instead of suppressing every hook error when the plugin error mentions missing compiled runtime output.
