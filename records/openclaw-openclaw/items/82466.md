## Summary

Keep open: the diagnostic cleanup is still useful and not implemented on current main, but the branch hides hook fallback details based only on the plugin error, which can suppress actionable hook-pack validation for dual packages, and external real CLI proof is still missing.

## What This Changes

The PR adds a CLI plugin-install formatter guard and regression test to suppress the hook-pack fallback suffix when missing compiled runtime output is reported.

## Best Possible Solution

Keep the formatter-layer cleanup, but suppress only the missing `openclaw.hooks` fallback shape, preserve other hook validation details, and require redacted real CLI proof before merge.

## Reproduction Assessment

Yes, at source level: current main routes non-terminal plugin install failures into hook fallback formatting and appends the fallback suffix. No live CLI run was performed because this is a read-only review.

## Solution Assessment

No. The formatter is the right layer, but the implementation should suppress only the missing-hook-manifest fallback tail instead of dropping every hook error when the plugin error mentions missing compiled runtime output.
