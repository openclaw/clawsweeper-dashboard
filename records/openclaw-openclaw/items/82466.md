## Best Possible Solution

## Reproduction Assessment

Yes, source-level. Current main routes non-terminal plugin install failures through the hook fallback formatter, and hook packages can be plugin-capable, so the PR’s plugin-error-only suppression can hide a real sibling diagnostic.

## Solution Assessment

No. The formatter is the right layer, but the implementation should key off the specific missing-hook-manifest fallback instead of suppressing every hook error for compiled-runtime plugin failures.
