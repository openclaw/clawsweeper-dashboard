## Summary

Keep open. The PR is a focused CLI diagnostic improvement, but the current guard suppresses all hook fallback details whenever the plugin error contains the compiled-runtime text, which can hide actionable hook-pack validation failures for dual plugin/hook packages; it also still lacks external contributor real CLI proof.

## What This Changes

The PR changes plugin install error formatting and adds a regression test so missing compiled runtime output does not append the `Also not a valid hook pack` suffix.

## Best Possible Solution

Keep the formatter-layer fix, but suppress only the specific missing-hook-manifest fallback tail for the compiled-runtime plugin error and preserve other hook validation details.

## Reproduction Assessment

Yes, at source level. Current main routes non-terminal plugin install failures through the hook fallback formatter, and hook packages can be plugin-capable, so the PR’s plugin-error-only suppression can hide a real sibling diagnostic.

## Solution Assessment

No. The formatter is the right layer, but the implementation should key off the specific missing-hook-manifest fallback instead of suppressing every hook error for compiled-runtime plugin failures.
