## Summary

Keep open: the diagnostic problem is real, but this patch is not merge-ready because the new formatter guard can hide actionable hook-pack validation details, and the external contributor has not supplied real CLI proof.

## What This Changes

The PR adds a CLI error-formatting guard and regression test to omit hook fallback text when plugin install fails with a missing compiled runtime output diagnostic.

## Best Possible Solution

Keep the formatter-layer fix, but suppress only the missing `openclaw.hooks` fallback tail, preserve other hook validation details, and require redacted real CLI proof before merge.

## Reproduction Assessment

Yes, source-level: current main routes non-terminal plugin install failures through hook fallback formatting, and hook packages with `openclaw.extensions` are supported. Under this patch, a dual plugin/hook package with the compiled-runtime plugin error plus a real hook validation error would lose the hook diagnostic.

## Solution Assessment

No. The formatter is the right layer, but the implementation should key off the specific missing-hook-manifest fallback instead of suppressing every hook error for compiled-runtime plugin failures.
