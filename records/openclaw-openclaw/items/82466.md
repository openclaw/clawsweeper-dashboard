## Summary

Keep open: the CLI diagnostic cleanup is still useful, but the current guard is too broad and contributor real CLI proof is still missing before merge.

## What This Changes

The branch adds a CLI formatter guard and regression test to suppress the hook-pack fallback suffix when plugin install fails because a package lacks compiled JavaScript runtime output.

## Best Possible Solution

Suppress only the missing-hook-manifest fallback tail, keep other hook validation errors visible, and require redacted real CLI proof before merge.

## Reproduction Assessment

Yes, at source level. Current main routes non-terminal plugin install failures through the hook fallback formatter, and supported plugin-capable hook packages make the PR's broad suppression path risky without running a live install.

## Solution Assessment

No. The formatter is the right layer, but the implementation should key off the specific missing-hook-manifest fallback shape instead of suppressing every hook error for compiled-runtime plugin failures.
