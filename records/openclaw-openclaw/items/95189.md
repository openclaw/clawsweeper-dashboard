Keep open: the PR fixes the linked macOS Tahoe runtime-prompt bug in the right area and includes useful real-host proof, but it adds an uncached synchronous `sw_vers` lookup to runtime prompt construction and needs a small repair before merge.

## What This Changes

The branch adds `resolveRuntimePromptOs()`, routes the embedded attempt, compaction, and CLI runtime prompt OS fields through it, and adds macOS product-version regression coverage.

## Best Possible Solution

Keep the shared Darwin product-version helper, but cache or reuse the macOS product-version lookup before landing the prompt caller changes.

## Reproduction Assessment

Yes from source inspection: current main passes `os.type()` plus `os.release()` into the three runtime prompt paths, so a Tahoe host with Darwin 25.x reaches agent context as a Darwin kernel version. The PR body also includes a macOS 26.5 / Darwin 25.5 before-and-after terminal probe.

## Solution Assessment

Mostly yes: centralizing Darwin runtime prompt OS display beside the existing OS summary helper is the right layer. The implementation should reuse cached product-version state rather than spawning `sw_vers` for each prompt build.
