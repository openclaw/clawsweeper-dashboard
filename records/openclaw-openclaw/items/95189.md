## Summary

Keep open: this is a focused fix for a real macOS 26 runtime prompt bug, but the Darwin path should not merge while it adds an uncached synchronous `sw_vers` subprocess to agent prompt construction.

## What This Changes

The PR adds a Darwin-specific runtime prompt OS helper, routes the CLI, embedded attempt, and compaction prompt callers through it, and adds focused prompt/OS-summary tests.

## Best Possible Solution

Land the runtime prompt fix with macOS product-version discovery cached or reused from the existing OS-summary owner, while keeping non-Darwin prompt output unchanged.

## Reproduction Assessment

Yes. Current main and v2026.6.8 still pass `os.type()` plus `os.release()` into the runtime prompt, so a macOS 26 host with Darwin 25.x will expose the kernel release to agent context; I did not run a live Tahoe host in this read-only review.

## Solution Assessment

Not quite as submitted. The PR fixes the right prompt surfaces, but the best fix should cache or reuse the macOS product-version lookup instead of adding an uncached synchronous subprocess to runtime prompt construction.
