## Summary

Keep open. This is an active external feature PR for a real core setup i18n gap, and current main still lacks the CLI locale surface; latest head fixes some earlier web-search review gaps but still has merge-blocking localization gaps and insufficient latest-head real setup proof.

## What This Changes

Adds an English/zh-CN CLI setup catalog, locale selection/detection/persistence via `cli.locale` and `OPENCLAW_LOCALE`, localized setup/finalization copy, config docs/schema/help labels, and a changelog entry.

## Best Possible Solution

Finish this as a core setup i18n feature by routing all setup, security, finalization, and enum-like summary values through the CLI catalog, adding focused regression coverage, and aligning the public locale config contract with docs.
