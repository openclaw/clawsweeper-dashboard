## Summary

Keep open: current main does not implement CLI onboarding locale support, and this PR still has source-reproducible blockers around reset locale preservation, docs/changelog coverage, token-storage copy, and one untranslated setup branch.

## What This Changes

The PR adds English/zh-CN setup message catalogs, setup locale selection/resolution, `cli.locale` schema support, and localized setup/finalization copy.

## Best Possible Solution

Keep this as the consolidated zh-CN setup localization candidate, but make locale resolution return the effective locale, document the config/env contract, fix token-storage wording, localize the remaining web-search note, add the required changelog entry, and rerun focused setup tests plus the changed gate.

## Solution Assessment

No. The product direction is plausible and current main does not replace it, but this patch is not the best mergeable solution until the correctness, docs, changelog, security-copy, and remaining localization gaps are fixed.
