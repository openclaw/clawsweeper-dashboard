## Summary

Keep this PR open. Current main still lacks the requested CLI zh-CN setup localization and `cli.locale` contract, but the PR head has merge-blocking correctness, docs, localization, and security-copy gaps.

## What This Changes

This PR adds English/zh-CN setup message catalog support, locale resolution from env/config/LANG/prompt, `cli.locale` schema support, and localized setup/finalization copy.

## Best Possible Solution

Keep this as the consolidated zh-CN setup localization candidate, but make locale resolution return the effective locale, document the public config/env contract, fix token-storage wording, finish the untranslated setup paths, and then rerun focused setup tests plus the changed gate.

## Reproduction Assessment

Yes. The reset bug is reproducible by inspection at PR head: set `OPENCLAW_LOCALE=zh-CN` or start with `baseConfig.cli.locale`, choose existing config `Reset`, and `applySelectedLocale({})` cannot preserve `cli.locale` because `selectedLocale` is `undefined`.

## Solution Assessment

No. The product direction is plausible and current main does not replace it, but this patch is not the best mergeable solution until the correctness, docs, security-copy, and remaining localization gaps are fixed.
