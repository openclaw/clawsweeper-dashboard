## Summary

Keep open: current main still documents and enforces process-denied `exec` as synchronous, while https://github.com/openclaw/openclaw/pull/74951 is the active implementation path and still needs maintainer policy review for the restricted-tool contract.

## What This Changes

The issue requests a design decision for finite-timeout `exec` backgrounding when the `process` follow-up tool is hidden by policy.

## Best Possible Solution

Keep this design issue open as the canonical tracker while maintainers decide whether to land or reject https://github.com/openclaw/openclaw/pull/74951; if accepted, ensure process-hidden finite-timeout completion remains observable and track true detached jobs separately.

## Reproduction Assessment

Yes for the current-main behavior: source, docs, and an existing test show that denying `process` makes `exec` run synchronously despite `yieldMs`. This is not a strict bug reproduction because current docs define that behavior today.

## Solution Assessment

Unclear until maintainer review. The linked PR is a focused implementation, but it changes a documented restricted-tool contract and should also resolve or explicitly sequence the quiet-success completion visibility concern.
