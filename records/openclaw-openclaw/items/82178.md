## Summary

Keep open: current main and the latest release still define process-denied exec as synchronous, while the linked implementation PR changes that documented restricted-tool contract and needs maintainer policy review.

## What This Changes

Request to approve and track finite-timeout exec backgrounding when the process follow-up tool is hidden by policy.

## Best Possible Solution

Have maintainers decide the restricted-tool policy on the linked PR, ensure process-hidden completions remain observable, and keep true detached jobs as separate follow-up work.

## Reproduction Assessment

Yes for the current-main behavior: source, docs, shipped docs, and tests show that denying process makes exec ignore yieldMs/background and complete synchronously. This is not a strict bug reproduction because that is the documented contract today.

## Solution Assessment

Unclear until maintainer review. The linked PR is a focused implementation for finite-timeout hidden-process backgrounding, but it changes a documented tool-policy contract and should account for quiet-success completion visibility before landing.
