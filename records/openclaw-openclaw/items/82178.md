## Summary

Keep open: current main still intentionally couples exec backgrounding to process visibility, and the linked implementation remains an open policy/contract change rather than something already implemented or safe to auto-close.

## What This Changes

The issue requests a maintainer design decision on whether finite-timeout exec backgrounding should remain available when the process follow-up tool is hidden by policy.

## Best Possible Solution

Keep this as the design anchor while maintainers decide the restricted-tool policy and resolve the linked PR or a narrower follow-up that preserves timeout and completion visibility guarantees.

## Reproduction Assessment

Yes, source-level: current main derives exec backgrounding from the effective `process` policy, and current docs/tests show process-denied exec runs synchronously despite `yieldMs`. This is not a strict bug reproduction because the current public contract defines that behavior today.

## Solution Assessment

Unclear until maintainer review: the linked PR is the focused implementation path, but it changes a documented tool-policy contract and needs the quiet-success completion visibility issue resolved before it is the best solution.
