## Summary

Keep open: current main still implements the documented process-denied synchronous behavior, and the linked PR/issue pair is an active policy decision rather than conservative cleanup.

## What This Changes

Requests maintainer design review for allowing finite-timeout `exec` backgrounding when `process` is hidden by policy.

## Best Possible Solution

Have maintainers decide the restricted-tool contract on the linked PR, preserve observable completion without `process` if accepted, and keep true detached-job behavior as separate follow-up work.

## Reproduction Assessment

Yes for the current-main behavior: source, docs, and a current test show that denying `process` makes `exec` run synchronously despite `yieldMs`. This is not a strict bug reproduction because current docs define that contract today.

## Solution Assessment

Unclear until maintainer review. The linked PR is a focused implementation for finite-timeout hidden-process backgrounding, but it changes a documented tool-policy contract and should also resolve quiet-success visibility before landing.
