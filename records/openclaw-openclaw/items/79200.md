## Summary

This PR is useful but not ready to merge: the current diff still resolves `messageFile` inside the shared outbound runner, which exposes host-file reads beyond the CLI boundary, and the external PR proof still does not exercise the actual CLI flag path.

## What This Changes

Adds `--message-file <path>` to `openclaw message send`, documents it, reads UTF-8 file content in the outbound send runner, adds send-validation tests, and updates the changelog.

## Best Possible Solution

Resolve `--message-file` only in the CLI helper path, pass only `message` into shared outbound dispatch, keep docs/tests/changelog aligned, then require real CLI proof before merge.

## Reproduction Assessment

Not applicable as a strict bug reproduction: this PR adds a new CLI input mode, and current main only supports inline `--message`, media, and presentation inputs. Source inspection confirms the flag is missing; shell expansion happens before OpenClaw receives argv.

## Solution Assessment

No. The CLI flag is a reasonable direction, but file resolution belongs at the CLI boundary before `messageCommand`/`runMessageAction`, not in the shared outbound runner used by agent/tool dispatch.
