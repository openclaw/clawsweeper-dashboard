## Summary

Keep open. Current main still blocks non-bundled agent tool-result middleware and has no trusted-plugin dispatch opt-in; the request would expand plugin authority over outbound delivery and tool-result mutation, so it needs maintainer product and security design rather than cleanup closure.

## What This Changes

The issue requests an explicit trusted-plugin opt-in so user-installed plugins can use dispatch mutation hooks and agent tool-result middleware.

## Best Possible Solution

Design a narrow per-surface trust policy under the existing plugin config and hook model, then update registry loading, SDK docs, and contract tests for approved elevated hooks.

## Reproduction Assessment

Not applicable as a bug reproduction, but source inspection gives a high-confidence reproduction of the current limitation: non-bundled middleware is rejected and no trusted dispatch opt-in exists. No live VoiceBox setup was run.

## Solution Assessment

Unclear: the requested capability is plausible, but a broad top-level trusted boolean may be too coarse for dispatch and tool-result mutation. A per-hook or per-surface trust policy would be the safer maintainable direction.
