## Label Justifications

- `P2`: The report identified a meaningful plugin workflow limitation, but current main now resolves it without an urgent runtime outage.
- `impact:security`: The issue concerns installed plugin authority over model-visible tool results and local files sent through outbound channel payloads.

## Summary

Current main solves the central plugin workflow through narrower capabilities: explicitly enabled installed plugins may register manifest-declared tool-result middleware, while `reply_payload_sending` can add, modify, or cancel normalized outbound payloads—including local media on ACP-routed replies. The proposed blanket `trusted: true` privilege is no longer necessary.

## What This Changes

Allow user-installed plugins to opt into elevated tool-result and outbound-dispatch mutation so workflows can transform results and attach media without an extra agent message call.

## Best Possible Solution

Keep the manifest-declared, explicitly enabled middleware contract and `reply_payload_sending` as the canonical solution: plugins can transform tool context with the former and add or modify outbound media through the latter without receiving blanket host trust.

## Reproduction Assessment

No on current main. Source, focused tests, docs, and ACP delivery call sites show that declared installed middleware and scoped outbound payload mutation now cover the reported workflow, including hook-added local media.

## Solution Assessment

Yes. The current manifest-scoped middleware plus transport-neutral payload hook is narrower and safer than granting plugins blanket trusted status.
