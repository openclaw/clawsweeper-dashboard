## Summary

Current main already implements the central trusted installed-plugin hook capability with manifest-declared contracts and explicit enablement, and the implementation shipped in v2026.6.6 via the merged related PR.

## What This Changes

The issue asks for user-installed plugins to opt into trusted dispatch and tool-result hooks that were unavailable to non-bundled plugins.

## Best Possible Solution

Keep the shipped manifest-declared, explicitly enabled trusted-surface contract instead of adding a broader `trusted: true` config flag.

## Reproduction Assessment

No, not as a current-main failure. Source, tests, docs, the merged implementation PR, and v2026.6.6 release notes show the old bundled-only limitation is replaced by manifest-declared explicit enablement.

## Solution Assessment

Yes. The shipped manifest-contract design is narrower and more auditable than a blanket `trusted: true` config flag while still covering the tool-result middleware and dispatch-hook use cases described here.
