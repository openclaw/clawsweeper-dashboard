## Summary

Keep open. Current main still reserves tool-result middleware for bundled plugins and has only narrow per-hook trust knobs for external plugins, so this remains a real Plugin SDK trust-boundary feature request rather than conservative cleanup or ClawHub-only work.

## What This Changes

The issue requests an explicit trusted user-plugin opt-in so non-bundled plugins can mutate outbound dispatch and tool-result paths such as `message_sending`, `before_dispatch`, and `registerAgentToolResultMiddleware`.

## Best Possible Solution

Design a narrow per-surface trust policy under the existing plugin hook/config model, then update registry, middleware loading, schema/help, public docs, and contract tests while preserving the bundled-only default.

## Reproduction Assessment

No high-confidence bug reproduction is applicable because this is a feature request. Yes for a source-level feature-gap check: current main still rejects non-bundled tool-result middleware and lacks a trusted dispatch opt-in.

## Solution Assessment

Unclear. The capability is plausible, but a broad top-level `trusted` field is probably not the best final shape without maintainer product and security approval for per-hook or per-surface authority.
