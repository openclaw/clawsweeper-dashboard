## Summary

Keep open. Current main still has no trusted opt-in for non-bundled plugins to use tool-result middleware or broader dispatch-mutation authority, and the requested capability changes plugin trust boundaries enough to need maintainer product and security design.

## What This Changes

Allow explicitly trusted user-installed plugins to mutate outgoing dispatch and tool-result flow for ACPX-style workflows such as attaching generated VoiceBox audio automatically.

## Best Possible Solution

Design a narrow trust-gated plugin policy for the approved dispatch and middleware surfaces, then update registry/loading, config schema/types, docs, and contract tests together.

## Reproduction Assessment

Not applicable as a feature request. Source inspection does confirm current main lacks the proposed trust config and still blocks non-bundled tool-result middleware.

## Solution Assessment

Unclear: the requested capability is plausible, but the proposed top-level trusted boolean needs maintainer design against narrower per-surface trust controls before implementation.
