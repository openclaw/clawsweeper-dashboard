## Summary

Keep open. Current main and the latest release still reserve the requested middleware path for bundled plugins, and the config/schema surface has no trusted dispatch opt-in, so this needs maintainer product and security design rather than cleanup closure.

## What This Changes

The issue requests an explicit trusted opt-in so user-installed plugins can mutate outgoing dispatches and register tool-result middleware currently reserved for bundled plugins.

## Best Possible Solution

Design a narrow per-surface trust policy under the existing plugin config and hook model, then update registry loading, docs, and contract tests so explicitly opted-in local plugins can use only approved dispatch and middleware hooks.

## Reproduction Assessment

Not applicable as a feature request. Source inspection on current main confirms the requested non-bundled middleware path and trusted config opt-in do not exist.

## Solution Assessment

Unclear. The capability is plausible, but a broad top-level trusted boolean may be less maintainable than explicit per-hook or per-surface trust controls using the existing plugins.entries policy shape.
