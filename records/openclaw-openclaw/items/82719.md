## Summary

Keep open. Current main still reserves the relevant tool-result middleware for bundled plugins and has no trusted dispatch opt-in for user-installed plugins; this is a new Plugin SDK/config trust boundary that needs maintainer product and security design, not conservative cleanup.

## What This Changes

Request to let explicitly trusted local user plugins use dispatch-mutation hooks and agent tool-result middleware for workflows such as automatically attaching generated VoiceBox audio.

## Best Possible Solution

Design an explicit per-surface plugin trust policy, with config schema, docs, contract tests, audit/debug behavior, and clear interaction with the existing public pre-model hook discussion.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a Plugin SDK feature request. Source inspection confirms current main still lacks the requested trusted config and keeps the pre-model middleware path bundled-only.

## Solution Assessment

Unclear: the capability is plausible, but a broad top-level `trusted` boolean is likely too coarse for dispatch and tool-result mutation authority. A per-hook or per-surface trust policy under the existing plugin config model would be safer to review and test.
