## Summary

Keep open. Current main still enforces bundled-only tool-result middleware and has no trusted dispatch opt-in for non-bundled plugins; because this expands plugin authority over outbound dispatch and tool results, it needs product and security design rather than conservative cleanup.

## What This Changes

Requests an explicit trusted user-plugin designation so non-bundled plugins can mutate dispatch and tool-result middleware for ACPX/VoiceBox-style workflows.

## Best Possible Solution

Design a narrow per-surface trust policy under the existing plugin hook/config model, then update registry, lazy loading, docs, and contract tests so explicitly opted-in local plugins can use only the approved dispatch and middleware hooks.

## Reproduction Assessment

Not applicable as a feature request, though source inspection confirms current main blocks the requested non-bundled middleware path and lacks the proposed trust config. No live VoiceBox setup was run.

## Solution Assessment

Unclear: the capability is plausible, but a broad top-level trusted boolean may be less maintainable than explicit per-hook or per-surface trust controls using the existing hooks policy shape.
