## Summary

Keep open. Current main still blocks non-bundled agent tool-result middleware and does not expose a trusted dispatch opt-in; the request would expand plugin authority over outbound messages and agent-visible tool results, so it needs maintainer product and security design rather than conservative cleanup.

## What This Changes

The issue requests a trusted plugin opt-in that lets user-installed plugins use dispatch mutation hooks and agent tool-result middleware.

## Best Possible Solution

Design a narrow per-surface trust policy for local user plugins, then update registry, lazy loading, config schema/help, docs, and contract tests around the approved dispatch and middleware hooks.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection on current main confirms the requested trusted opt-in is absent and non-bundled middleware registration is blocked.

## Solution Assessment

Unclear. The capability is plausible, but the proposed top-level trusted boolean should be reviewed against a narrower per-hook or per-surface trust policy before implementation.
