## Summary

Current main fixes the bundled-only tool-result middleware restriction, but it does not fully solve the central automatic-delivery workflow: middleware changes model-visible results, while the existing dispatch hooks cannot add the generated audio attachment. The remaining typed outbound capability requires product and security design.

## What This Changes

Allow explicitly trusted user plugins to transform ACPX tool results and automatically modify or augment final channel delivery, including attaching generated media.

## Best Possible Solution

Keep explicit enablement as the operator trust action, add separate manifest-declared capabilities for read-only tool telemetry and typed outbound reply-payload augmentation, and route media changes through canonical durable delivery rather than a blanket trusted boolean.

## Reproduction Assessment

Yes at source level: current main proves installed middleware registration now works, but the durable outbound hooks remain text-oriented and cannot add the reported .ogg attachment without a separate message call.

## Solution Assessment

No. The proposed blanket trusted boolean is broader than necessary; explicit enablement already supplies operator consent, while capability-specific manifest contracts and a typed reply-payload seam would reduce authority and delivery risk.
