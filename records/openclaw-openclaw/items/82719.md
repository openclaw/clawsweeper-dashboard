## Label Justifications

- `P2`: A concrete advanced plugin workflow remains blocked, but ordinary agent and channel operation is unaffected.
- `impact:security`: The remaining request concerns granting installed plugin code authority to mutate or cancel outbound messages.

## Summary

Current main partially addresses the report through merged PR https://github.com/openclaw/openclaw/pull/90004: explicitly enabled installed plugins can now register manifest-declared agent tool-result middleware. The original VoiceBox workflow still lacks an ACPX outbound-dispatch mutation path, so the issue remains necessary but should be narrowed to that capability and its trust model.

## What This Changes

Allow explicitly trusted user-installed plugins to transform agent tool results and mutate ACPX outbound channel delivery.

## Best Possible Solution

Extend the existing manifest-declared, explicitly enabled capability model with a narrowly scoped ACPX outbound-dispatch contract, while exposing read-only tool telemetry through a separate non-mutating event.

## Reproduction Assessment

Yes at source level: current main now permits declared installed tool-result middleware, but ACPX does not invoke the outbound message mutation hooks required for automatic attachment delivery.

## Solution Assessment

No, the original broad `trusted: true` switch is not the best solution; the newer manifest-contract and explicit-enable design should be extended with capability-specific permissions.
