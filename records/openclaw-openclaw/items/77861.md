## Summary

Keep open: current main has a source-reproducible context-window reporting bug, and this PR is directionally right, but it still leaves an uncapped directive-persistence path plus missing real behavior proof and changelog coverage.

## What This Changes

The PR caps agent-level `contextTokens` against model/catalog windows in reply model selection and status rendering, adds cold-cache catalog lookup for status, and adds focused unit tests.

## Best Possible Solution

Land a complete fix that shares downward-only cap semantics across model selection, directive persistence, and status/command context reporting, with a changelog entry and real `/status` proof.

## Reproduction Assessment

Yes, from source inspection: current main returns the agent default before model lookup while the guard contract already treats the same value as a downward-only cap. I did not run a live `/status` turn under the read-only review constraints.

## Solution Assessment

No. The code direction is right, but the PR should also carry the capped value through directive persistence or the shared resolver, then add the required changelog entry and after-fix real behavior proof.
