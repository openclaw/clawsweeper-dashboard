## Summary

Keep open: the PR addresses a real source-visible log-forging problem, but the current patch is not merge-ready because it leaves one interpolated field unsanitized, duplicates a weaker sanitizer, and lacks required real behavior proof.

## What This Changes

The PR adds channel logging sanitization and regression tests for control-character log injection in shared channel logging helpers.

## Best Possible Solution

Use a string-coercing wrapper around the shared terminal sanitizeForLog for every interpolated channel logging field, with regression coverage for channel, ANSI/OSC/C1/DEL controls, and JavaScript non-string inputs.

## Reproduction Assessment

Yes. Source inspection on current main shows caller-provided channel logging fields are directly interpolated into log lines, so newline or control-character inputs can forge output; I did not run tests because this review is read-only.

## Solution Assessment

No. The hardening direction is right, but this branch should sanitize params.channel, reuse the shared sanitizer, and preserve prior stringification behavior for loosely typed plugin callers.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Sanitize channel before emitting log lines:** `src/channels/logging.ts:16`
  - body: The patch still starts each message with raw params.channel. A channel value containing a newline or control character can forge a separate log line, so sanitize channel in all three helpers and cover it in tests.
  - confidence: 0.93
- **[P2] Coerce values before replacing characters:** `src/channels/logging.ts:4-6`
  - body: sanitizeForLog calls .replace on its input. JavaScript plugin callers that pass numeric IDs or objects for target or reason would now throw where template interpolation previously stringified them, so coerce inside the sanitizer first.
  - confidence: 0.88
- **[P2] Reuse the shared terminal sanitizer:** `src/channels/logging.ts:4-6`
  - body: This local sanitizer strips only C0 controls and DEL, while the existing terminal sanitizer also removes ANSI/OSC and C1 controls. Delegate through the shared helper so log-safety behavior stays consistent.
  - confidence: 0.84
