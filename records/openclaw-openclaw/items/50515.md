## Summary

Keep open: this is useful security hardening, but the current PR should not merge as-is because it leaves a log-forgeable field, uses a weaker duplicate sanitizer, and lacks required real behavior proof.

## What This Changes

The PR adds a local channel log sanitizer, applies it to target/reason/error fields in shared channel logging helpers, and adds Vitest coverage for control-character log injection cases.

## Best Possible Solution

Revise the PR to run every interpolated channel-log value through a string-coercing wrapper around the shared terminal sanitizer, then cover channel, ANSI/C1, DEL, and non-string runtime cases.

## Reproduction Assessment

Yes. Source inspection is enough: on the PR branch, logInboundDrop with channel "x\nFAKE" still emits params.channel directly, and current main directly interpolates the same fields.

## Solution Assessment

No. The direction is right, but the maintainable fix is to reuse the existing shared sanitizer with explicit string coercion and apply it consistently to channel, target, reason, and error values.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Sanitize the channel field before logging:** `src/channels/logging.ts:16`
  - body: The patch sanitizes reason, target, and error, but every helper still interpolates params.channel directly. A channel string containing a newline or other control character can still forge log lines, so sanitize the channel value before building each message and cover it in tests.
  - confidence: 0.93
- **[P2] Coerce values before sanitizing:** `src/channels/logging.ts:6`
  - body: sanitizeForLog calls .replace on its input. These helpers are public plugin SDK exports, and loose JavaScript plugin callers could pass numeric targets or other values that previously stringified through template literals; this patch would throw instead of logging.
  - confidence: 0.88
- **[P2] Reuse the shared log sanitizer:** `src/channels/logging.ts:4-6`
  - body: The new local sanitizer strips only C0 controls and DEL, while the existing terminal sanitizer also strips ANSI/OSC and C1 controls. Reuse the shared helper through a small channel wrapper so channel logs follow the same log-safety contract as the rest of the repo.
  - confidence: 0.84
