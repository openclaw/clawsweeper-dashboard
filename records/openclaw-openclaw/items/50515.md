## Summary

Keep open: the underlying log-forging issue is source-reproducible on current main, but this PR still leaves a channel field unsanitized, adds a weaker duplicate sanitizer, can regress loosely typed plugin callers, and lacks after-fix real behavior proof.

## What This Changes

The PR adds a channel-local log sanitizer for `reason`, `target`, and `error` values in `src/channels/logging.ts`, plus a new Vitest suite for control-character log injection cases.

## Best Possible Solution

Use a small string-coercing wrapper around the shared terminal `sanitizeForLog` for every interpolated field, with regression coverage for channel, ANSI/OSC/C1/DEL controls, and non-string JavaScript inputs.

## Reproduction Assessment

Yes. Source inspection is enough: current main interpolates raw channel logging fields, so a caller-provided newline or control character in those values can forge log output.

## Solution Assessment

No. The hardening direction is right, but this patch should reuse the shared sanitizer and sanitize all interpolated fields without breaking public plugin SDK callers.

## Review Findings

Full review comments:

- **[P2] Sanitize channel before emitting log lines:** `src/channels/logging.ts:16`
  - body: The patch still emits `params.channel` directly at the start of each log message. A channel value containing a newline or control character can still forge a separate log line, so sanitize `channel` in each helper and cover it in tests.
  - confidence: 0.93
- **[P2] Coerce log values before replacing characters:** `src/channels/logging.ts:4-6`
  - body: `sanitizeForLog()` calls `.replace` on its input. These helpers are public plugin SDK surface, and JavaScript callers that pass numeric or object `target`/`reason` values would now throw where template interpolation previously stringified them.
  - confidence: 0.88
- **[P2] Reuse the shared terminal log sanitizer:** `src/channels/logging.ts:4-6`
  - body: This adds a second sanitizer that handles only C0 controls and DEL, while the existing terminal sanitizer also strips ANSI/OSC sequences and C1 controls. Delegate through the shared helper so log-safety behavior stays consistent.
  - confidence: 0.84
