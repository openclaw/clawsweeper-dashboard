## Summary

The reported Feishu `queuedFinal=false, replies=0` signature matches the documented group/channel visible-reply policy: normal group final answers stay private unless the agent uses the message tool or `messages.groupChat.visibleReplies` is set to `automatic`.

## What This Changes

The issue reports that Feishu group mentions generate an agent response but no outbound group reply is delivered after upgrading to 2026.5.7.

## Best Possible Solution

Keep the current tool-only group default, and have affected Feishu users either set `messages.groupChat.visibleReplies: "automatic"` for legacy room replies or use a model/runtime that reliably calls `message(action=send)`.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a Feishu group turn with default or `message_tool` group visible replies and a model that returns ordinary final text without `message(action=send)` will process the agent turn but queue zero visible final replies.

## Solution Assessment

Yes. This is best handled by the existing config/model guidance rather than a Feishu code change; a code fix would be appropriate only if `visibleReplies: "automatic"` or a message-tool send still fails to deliver.
