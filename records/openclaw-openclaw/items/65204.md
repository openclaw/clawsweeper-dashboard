## What This Changes

The issue reports that `openclaw browser start` fails because a profile with `cdpUrl: "http://127.0.0.1"` and `cdpPort: 18800` resolved Chrome's CDP bind port to 80.

## Best Possible Solution

Keep the current browser profile port-precedence implementation on `main` and close this issue; affected stable-release users need the next release or a main build.

## Reproduction Assessment

No for current `main`. The prior failure path was source-reproducible, but the same profile shape now resolves `cdpPort` to 18800 and has a regression test locking that behavior.

## Solution Assessment

Yes. The fix lives in browser profile normalization and preserves explicit URL ports plus remote default-port behavior, which is the narrow maintainable place to solve the issue.
