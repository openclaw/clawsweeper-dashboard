## What This Changes

The issue asks OpenClaw to refresh or stop repeatedly bootstrapping a stale local `anthropic:claude-cli` OAuth profile from fresher Claude CLI credentials.

## Best Possible Solution

Review and land a corrected version of the linked fix, preserving read-only loads and owner-store OAuth write semantics, or deliberately choose a documented state-change/rate-limited DEBUG behavior instead.

## Reproduction Assessment

Yes. Source inspection shows an expired stored `anthropic:claude-cli` OAuth profile plus fresher Claude CLI credentials will return a runtime-only overlay and emit the reported DEBUG bootstrap message on each resolving path.

## Solution Assessment

Unclear. Persisting fresher matching external CLI credentials is a plausible narrow direction, but the linked PR review shows the implementation must avoid read-only loads and write back to the owning auth store before this issue can close.
