## What This Changes

Fix stale local `anthropic:claude-cli` OAuth profiles so fresh external CLI credentials stop retriggering per-resolution bootstrap DEBUG logs.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/80148, or a replacement that persists fresher matching non-bootstrapOnly CLI credentials under the correct auth-store lock and owner semantics while preserving read-only paths.

## Solution Assessment

Unclear as submitted. Persisting fresher matching CLI credentials is the right user-visible direction, but the write path needs to preserve read-only loads, OAuth inheritance, and auth-store ownership semantics before merge.
