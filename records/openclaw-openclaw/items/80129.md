## Summary

Keep open. Current main still has the reported source path: a stale stored `anthropic:claude-cli` OAuth profile can be overlaid from fresher Claude CLI credentials as runtime-only, which leaves the persisted profile stale and can repeat the DEBUG bootstrap message; the linked open fix PR should be reviewed or replaced rather than closing this issue early.

## What This Changes

The issue asks OpenClaw to resync local `anthropic:claude-cli` OAuth profiles from fresher Claude CLI credentials so stale stored profiles do not repeatedly trigger bootstrap DEBUG logging.

## Best Possible Solution

Land an owner-reviewed fix that persists fresher matching external CLI OAuth credentials only from write-authorized runtime paths under the owning auth-store lock, or deliberately retain runtime-only overlays and rate-limit or de-duplicate the DEBUG log with docs coverage.

## Reproduction Assessment

Yes, source-reproducible: an expired stored `anthropic:claude-cli` OAuth profile plus fresher Claude CLI credentials will enter `resolveExternalCliAuthProfiles`, emit the reported DEBUG bootstrap message, and return a runtime-only overlay. I did not run a live Windows/Claude CLI expiry cycle.

## Solution Assessment

Unclear. Persisting the fresher credential is a plausible narrow fix, but it must respect read-only loads and persisted-owner resolution; the current best path is to repair and review the linked open PR before closing this issue.
