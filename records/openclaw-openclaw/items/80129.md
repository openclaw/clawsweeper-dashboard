## Summary

Keep open. Current main still matches the report: a stale stored `anthropic:claude-cli` OAuth profile with fresher Claude CLI credentials resolves through the external CLI bootstrap path as a runtime-only overlay, so the local profile is not durably refreshed. The fix is already tracked by open draft https://github.com/openclaw/openclaw/pull/80274, so this issue should close only after that PR or another canonical fix merges.

## What This Changes

Request that stale local `anthropic:claude-cli` OAuth profiles resync from fresher Claude CLI credentials, or otherwise stop repeated bootstrap DEBUG logging.

## Best Possible Solution

Land a reviewed version of https://github.com/openclaw/openclaw/pull/80274, or an equivalent fix, that persists fresher matching non-bootstrap external CLI credentials under the auth-store lock while preserving Codex bootstrap-only, read-only, inheritance, and identity guards.

## Reproduction Assessment

Yes, source-reproducible: an expired stored `anthropic:claude-cli` OAuth profile plus fresher Claude CLI credentials reaches `resolveExternalCliAuthProfiles`, emits the reported DEBUG bootstrap message, and returns a runtime-only overlay. I did not establish a live Windows/Claude CLI rollover reproduction.

## Solution Assessment

Yes, the canonical direction in https://github.com/openclaw/openclaw/pull/80274 appears to be the narrow maintainable fix, but it still needs human owner review because it changes OAuth credential persistence semantics.
