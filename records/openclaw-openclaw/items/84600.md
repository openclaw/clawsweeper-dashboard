## Summary

Keep open. Current main and the latest release still generate the reported `powershell.exe ... | findstr /I /C:"Running"` line inside `openclaw-schtasks-restart-*.cmd`; the related visible-window items at https://github.com/openclaw/openclaw/issues/81254 and https://github.com/openclaw/openclaw/pull/81330 cover a different scheduled-task launch script, not this restart handoff helper.

## What This Changes

The issue requests that the Windows Gateway scheduled-task heartbeat/restart status check run silently instead of leaving a visible cmd/findstr window.

## Best Possible Solution

Make the generated Windows scheduled-task restart helper perform its running-state check without spawning or holding a visible console, while preserving the retry loop, race guard, logging, and startup fallback behavior.

## Reproduction Assessment

## Solution Assessment

Yes, a focused fix in `src/infra/windows-task-restart.ts` is the narrowest maintainable path. The repair should hide or replace the cmd pipeline without deleting the running-task race guard or fallback behavior.
