## Summary

Keep open. Current main still leaves the reported recovery guidance incomplete: direct Gateway startup mentions `openclaw doctor --fix` without `.bak` recovery, while daemon start/restart preflight still points to generic `openclaw doctor`.

## What This Changes

Gateway invalid-config startup errors should tell operators to run `openclaw doctor --fix` and, if still blocked, recover from `openclaw.json.bak`.

## Best Possible Solution

Add one shared invalid-config recovery hint for direct Gateway startup and daemon lifecycle preflight, covering `doctor --fix` plus cautious `.bak` restore guidance without adding automatic rollback behavior.

## Reproduction Assessment

Yes, via source-level reproduction. Current main and `v2026.5.7` still contain the incomplete direct startup and daemon preflight messages; I did not run a live Gateway process in this read-only review.

## Solution Assessment

Yes. The narrowest maintainable fix is a shared message/helper update plus focused assertions for direct startup and daemon preflight.
