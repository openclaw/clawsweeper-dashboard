## Summary

Keep open. Current main still leaves the reported recovery guidance incomplete: direct Gateway startup mentions `openclaw doctor --fix` but not `.bak` recovery, while daemon start/restart preflight still points to generic `openclaw doctor`.

## What This Changes

The issue asks for Gateway invalid-config startup errors to suggest `openclaw doctor --fix` and a backup restore path using `openclaw.json.bak`.

## Best Possible Solution

Add one platform-neutral invalid-config recovery hint used by direct Gateway startup and daemon lifecycle preflight, covering `doctor --fix` plus cautious `.bak` restore guidance without adding automatic rollback behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: invalid direct startup throws the incomplete `doctor --fix` message, and daemon start/restart invalid-config preflight still emits generic `openclaw doctor`; I did not run the Gateway in this read-only review.

## Solution Assessment

Yes. The narrowest maintainable fix is a shared or consistently duplicated recovery-message update plus focused assertions for direct startup and daemon preflight, with backup wording that avoids unsafe shell commands.
