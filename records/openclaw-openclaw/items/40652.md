Keep open. Current main still only partially satisfies the report: direct Gateway startup names `openclaw doctor --fix` but omits `.bak` recovery guidance, and daemon start/restart preflight still points users at generic `openclaw doctor`.

The issue requests Gateway invalid-config startup errors to suggest `openclaw doctor --fix` and a fallback restore from `openclaw.json.bak`.

## Best Possible Solution

Add one shared invalid-config recovery hint for direct Gateway startup and daemon lifecycle preflight, covering `doctor --fix` plus cautious `.bak` restore guidance without adding automatic rollback behavior.
