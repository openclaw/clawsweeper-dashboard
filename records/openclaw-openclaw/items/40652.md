## Summary

Keep open. Current main still only partially satisfies the report: direct Gateway startup names `openclaw doctor --fix` but omits the `openclaw.json.bak` fallback, while daemon start/restart preflight still points at generic `openclaw doctor`; the related fix PRs in the issue context are closed unmerged.

## What This Changes

Improve Gateway invalid-config startup errors to suggest `openclaw doctor --fix` and restoring `openclaw.json.bak` when repair does not unblock startup.

## Best Possible Solution

Add a shared invalid-config recovery hint used by direct Gateway startup and daemon lifecycle preflight, covering `doctor --fix` plus a cautious `openclaw.json.bak` restore fallback, with focused assertions and a changelog entry.

Yes. A high-confidence source-level reproduction exists: current direct startup code and tests omit the backup fallback, and daemon start/restart preflight still constructs the generic `openclaw doctor` hint when `readConfigFileSnapshot()` returns an invalid snapshot.

## Solution Assessment

Yes. The narrowest maintainable solution is a shared message/helper update plus targeted assertions for direct startup and daemon preflight; this should remain recovery guidance, not automatic rollback behavior.
