## Summary

Keep open. Current main and the latest release still leave the reported recovery guidance incomplete: direct Gateway startup mentions `openclaw doctor --fix` but not `.bak` recovery, while daemon start/restart preflight still points to generic `openclaw doctor`.

## What This Changes

The issue requests Gateway invalid-config startup errors to suggest `openclaw doctor --fix` and cautious `openclaw.json.bak` recovery guidance.

## Best Possible Solution

Add one shared invalid-config recovery hint for direct Gateway startup and daemon lifecycle preflight, covering `doctor --fix` plus cautious `.bak` restore guidance without adding automatic rollback behavior.

## Reproduction Assessment

Yes, via source-level reproduction. Current main and `v2026.5.7` still contain the incomplete direct startup and daemon preflight messages, though I did not run the Gateway in this read-only review.

## Solution Assessment

Yes. The narrowest maintainable fix is a shared message/helper update plus focused assertions for direct startup and daemon preflight, with platform-neutral backup wording.
