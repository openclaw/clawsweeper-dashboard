## Summary

Keep open: current main still rejects approval-bound system.run from symlinked cwd paths, and the requested opt-in escape hatch would add a security-sensitive exec-approval config mode that needs maintainer threat-model/product review rather than stale cleanup.

## What This Changes

The issue requests an opt-in exec-approval setting that allows symlinked cwd paths for approval-bound system.run while binding approval to the canonical realpath inode.

## Best Possible Solution

Keep the issue open for a maintainer-owned security/product decision on whether an opt-in symlink-cwd mode should exist; if accepted, implement it as a documented default-off config with canonical-path display, inode revalidation, and regression tests for both rejected and opted-in cases.
