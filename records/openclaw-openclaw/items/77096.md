## Summary

Keep open: current main still intentionally rejects symlinked approval cwd paths and has no allowSymlinkCwd-style policy field. The request would add a default-off security-sensitive escape hatch for shipped TOCTOU hardening, so it needs maintainer threat-model and product approval before implementation.

## What This Changes

Request to add a default-off exec approvals option that permits trusted symlink cwd paths for approval-bound node-host system.run while surfacing and pinning the canonical target.

## Best Possible Solution

Maintainers should decide the security policy first; if accepted, implement a narrow default-off opt-in that shows requested and canonical cwd, pins the canonical target identity, keeps fail-closed revalidation, and updates schema, CLI/UI, docs, changelog, and regression tests.

## Reproduction Assessment

Yes for the current behavior: source and regression tests on current main show approval-backed node-host system.run denies both direct symlink cwd paths and symlink parent components. This is not a failing existing contract; the requested escape hatch is new behavior.

## Solution Assessment

Unclear pending maintainer security review. The proposed design is plausible, but adding an allowSymlinkCwd-style config relaxes a shipped TOCTOU hardening default; clearer denial guidance is the safer fallback if the policy is not accepted.
