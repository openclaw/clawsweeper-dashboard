## Summary

Keep open: current main still intentionally rejects symlinked approval cwd paths, and the requested opt-in is a new security-sensitive config and approval UX policy rather than an implemented cleanup.

## What This Changes

Request adds a default-off exec approvals opt-in for explicitly trusted symlink cwd paths in approval-bound `system.run` while preserving canonical-target display and revalidation.

## Best Possible Solution

Adopt a default-off opt-in only if maintainers accept the threat model; otherwise improve the denial message and docs for canonical-path workarounds.

## Reproduction Assessment

Yes for the current behavior: source and regression coverage show approval-backed `system.run` rejects a symlink cwd and a symlink parent path. This is a feature request for an opt-in relaxation, not a failing existing contract.

## Solution Assessment

Unclear pending maintainer security review. The proposal is plausible if approval displays both requested and canonical cwd and revalidates the pinned target identity; the lower-risk interim is clearer denial guidance.
