## Summary

Keep open: this is a narrow, plausible fix for a real DeepSeek tool-schema bug, and current main still collapses const-literal unions to the first non-null variant. ## What This Changes

The PR changes `normalizeDeepSeekSchema` to rewrite string `const` `anyOf`/`oneOf` unions into flat string `enum` schemas and adds SDK regression coverage for nullable and non-null DeepSeek const unions.

## Best Possible Solution

Land this narrow SDK helper/test fix after the contributor adds redacted real DeepSeek or OpenAI-compatible runtime proof that enum-shaped const-union tool parameters are accepted and visible to the model.

## Reproduction Assessment

Yes, as a source-level reproduction: current main feeds `normalizeDeepSeekSchema` a string `const` union and then selects only the first non-null variant. I did not run a live DeepSeek request in this read-only review.

## Solution Assessment

Yes, the proposed shape is the narrowest maintainable fix: keep the existing DeepSeek family helper and preserve enum-like const choices there instead of adding a new SDK API or provider config.
