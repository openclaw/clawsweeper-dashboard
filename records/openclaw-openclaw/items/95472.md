## Summary

Keep open: the PR addresses a plausible false-positive restart-recovery mark, but the lifecycle-generation guard is too broad and can skip existing current-generation restart-shutdown recovery for post-terminal active runs.

## What This Changes

This PR narrows `markRestartAbortedMainSessions` so the `updatedAt < observedAt` match only applies to stale lifecycle generations and adds a focused regression test for completed current-generation sessions.

## Best Possible Solution

Refine the marker so maintenance-expired current-generation terminal candidates do not reopen completed sessions while restart-shutdown post-terminal active runs can still be marked, with regression tests for both paths.

## Reproduction Assessment

Yes at source level: current main can match a completed session when a current-generation candidate has a later `observedAt`, and the PR adds a focused test for that marker boundary. I did not run a live Telegram replay in this read-only review.

## Solution Assessment

No: lifecycle generation alone is not the best discriminator because restart shutdown also passes current-generation active post-terminal runs through the same predicate. The safer fix needs to preserve that path while excluding maintenance-expired completed-session candidates.
