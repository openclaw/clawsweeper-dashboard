## Summary

Keep open: this PR targets a real current-main Control UI listener leak, but the proposed patch still leaves the same-button toggle dismissal path removing the popover without unregistering the document click listener. Related PR #69982 is open for the same bug, so the work should be consolidated into one complete fix before merge.

## What This Changes

The PR updates `ui/src/ui/chat/grouped-render.ts` so Cancel, Delete, and outside-click delete-confirm dismissals use a shared popover cleanup helper.

## Best Possible Solution

Land one canonical narrow patch that routes every delete-confirm dismissal path through listener cleanup, adds focused listener-balance coverage, and includes the required changelog entry.

## Reproduction Assessment

Yes. Current main reproduces from source by opening a delete confirmation, waiting for `requestAnimationFrame` to register the document click listener, then dismissing via Cancel, Delete, or the same delete button toggle path.

## Solution Assessment

No. A shared cleanup helper is the right narrow approach, but this patch does not make that cleanup reachable from the existing-popover toggle branch and lacks regression coverage.
