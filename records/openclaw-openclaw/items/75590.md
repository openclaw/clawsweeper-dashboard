## Summary

Keep open. The PR addresses a real current-main listener leak for Cancel, Delete, and click-outside dismissal, but it is not merge-ready because the same-button toggle dismissal still removes the popover without unregistering the document capture listener. It also needs the required changelog entry for a user-facing UI fix.

## What This Changes

The PR adds a local `dismissPopover` helper in `ui/src/ui/chat/grouped-render.ts` and routes the Cancel, Delete, and click-outside delete-confirm popover paths through it so the document click listener is removed.

## Best Possible Solution

Keep the PR open and amend it so every delete-confirm popover removal path shares one cleanup path for the exact document listener, with focused regression coverage and a single-line Unreleased Fixes changelog entry.

## Reproduction Assessment

Yes. On current `main`, open a delete confirmation popover, wait for the `requestAnimationFrame` listener install, then click the same delete button again; the capture listener ignores its own button and the existing-popover branch removes only the DOM node.

## Solution Assessment

No. The helper is the right direction for several dismissal paths, but it is not the best complete fix until the same-button toggle path and regression coverage are included.
