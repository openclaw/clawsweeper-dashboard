## Summary

Keep open. The PR addresses a real Control UI listener leak on current main, but the patch is not merge-ready because the same-button toggle dismissal path can still remove an open delete-confirm popover without unregistering its document click listener; it also needs focused listener-balance coverage before landing.

## What This Changes

The PR updates `ui/src/ui/chat/grouped-render.ts` so delete-confirm Cancel, Delete, and outside-click dismissals call a shared helper that removes the popover and unregisters the document click listener.

## Best Possible Solution

Land one canonical narrow UI fix that makes every delete-confirm dismissal path share the same cleanup state, prevents deferred listener registration after early dismissal, adds focused Vitest coverage for listener balance, and records the user-facing fix in `CHANGELOG.md`.

## Reproduction Assessment

Yes. Current main reproduces from source by opening a delete confirmation, waiting for the `requestAnimationFrame` document listener registration, then dismissing with Cancel, Delete, or the same delete button; the listener is not removed on those non-outside-click paths.

## Solution Assessment

No. A shared cleanup helper is the right narrow direction, but this PR only wires it into the fresh-popover handlers and leaves the already-open toggle path outside the cleanup contract.

## Review Findings
