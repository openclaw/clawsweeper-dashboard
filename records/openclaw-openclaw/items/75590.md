## Summary

Keep open. Current main still has the original document click listener leak, and this PR moves the main Cancel/Delete/outside-click paths in the right direction, but the patch is not merge-ready because the same-button toggle path still removes an already-open popover without unregistering the document listener. The open predecessor #69982 tracks the same bug, so maintainers should consolidate around one corrected branch or replacement PR.

## What This Changes

The PR updates `ui/src/ui/chat/grouped-render.ts` so the delete confirmation popover uses a shared dismissal helper for Cancel, Delete, and outside-click dismissal.

## Best Possible Solution

Land one narrow corrected patch that routes every delete-confirm dismissal path through listener cleanup, including Cancel, Delete, outside-click, and clicking the delete button again while the popover is already open. Add focused listener-balance coverage and the required user-facing changelog entry before merge.

## Reproduction Assessment

Yes. From source, open a delete confirmation, let the `requestAnimationFrame` register the document click listener, then dismiss via Cancel/Delete on current main or click the same delete button again on this PR; the add/remove listener calls are not balanced.

## Solution Assessment

No. The shared helper is the right direction, but as written it is scoped below the existing-popover toggle branch, so the patch does not cover every dismissal path.
