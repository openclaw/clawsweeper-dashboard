## Summary

Keep open. Current main still has the document click-listener leak, and this PR moves the main Cancel/Delete/outside-click dismissals in the right direction, but the patch is still incomplete because the same-button toggle path removes an existing popover without unregistering the document listener. The branch also needs the usual focused regression coverage and changelog before it is merge-ready.

## What This Changes

The PR updates `ui/src/ui/chat/grouped-render.ts` so Cancel, Delete, and outside-click dismissal of the delete confirmation popover share a helper that removes the popover and unregisters the document click listener.

## Best Possible Solution

Land one canonical narrow patch that makes every delete-confirm dismissal path unregister the document listener, including Cancel, Delete, outside-click, and clicking the same delete button again while the popover is already open. Add a focused listener-balance regression test and a `CHANGELOG.md` fix entry crediting the contributor.

## Reproduction Assessment

Yes. Current main reproduces from source by opening a delete confirmation, waiting for the requestAnimationFrame listener registration, then dismissing with Cancel or Delete; this PR still reproduces by clicking the same delete button again to remove the existing popover.

## Solution Assessment

No, not as written. The shared helper is the right implementation direction, but it is scoped only to the newly-created popover branch and does not clean up the earlier existing-popover toggle branch.
