## Summary

Keep open: current main still has the delete-confirm document click listener leak, and this PR is directionally useful but incomplete because the same-button toggle dismissal path still removes the existing popover without unregistering the listener. The work also overlaps open PR #69982, so the safest path is one canonical narrow fix rather than merging diverging one-file repairs.

## What This Changes

This PR changes `ui/src/ui/chat/grouped-render.ts` to add a shared delete-confirm popover dismissal helper used by Cancel, Delete, and outside-click dismissal so the document click listener is removed.

## Best Possible Solution

Keep a single canonical patch that routes every delete-confirm dismissal path through cleanup of the exact registered document listener, then add focused listener-balance coverage and the required user-facing changelog entry.

## Reproduction Assessment

Yes. Current main reproduces from source by opening a delete confirmation, waiting for the `requestAnimationFrame` listener registration, then dismissing via Cancel, Delete, or the same delete button; those paths remove the popover without removing the document click listener.

## Solution Assessment

No, not as written. A shared cleanup helper is the right narrow approach, but it must also cover the existing-popover toggle path and should be backed by a regression test before merge.
