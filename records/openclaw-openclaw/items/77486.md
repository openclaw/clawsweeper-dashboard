## Summary

Keep open: the proposed direction matches a valid current-main bug, but the patch currently has a typed IO contract mismatch that should block strict TypeScript before merge.

## What This Changes

The PR threads persisted config write results through config mutation and Gateway config write handlers so config.set, config.apply, and config.patch responses use the post-write config.

## Best Possible Solution

Land the shared persisted-result plumbing after tightening the injected write IO type and adding the required changelog entry.

## Reproduction Assessment

Yes. Current main source shows the write RPC handlers respond with parsed/validated config while the lower write path can stamp and canonicalize a different persisted payload.

## Solution Assessment

No, not as-is. The shared write boundary is the right place to solve the bug, but the injected IO type must return the persisted write result before the new persistedConfig dereference can typecheck.
