## Summary

Keep open. The PR is a useful skill enhancement and the contributor-provided terminal proof is sufficient, but the patch still has a JSON correctness bug for CodexBar rows that expose per-model total tokens without per-model split fields.

## What This Changes

This PR updates the `model-usage` skill script, docs, and tests to add per-model token totals to `--mode all` text and JSON output.

## Best Possible Solution

Land a corrected skill enhancement that reports per-model total tokens when present, preserves unavailable split fields as unknown, and leaves cost normalization to https://github.com/openclaw/openclaw/issues/37878.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a skill feature PR. Current main source confirms the missing token summaries, and the PR body provides terminal output for the proposed behavior.

## Solution Assessment

No, not yet. Parsing token data at the `modelBreakdowns[]` boundary is the right direction, but the JSON output should not turn unavailable per-model split fields into zeroes for current CodexBar total-only rows.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve unknown per-model token splits:** `skills/model-usage/scripts/model_usage.py:320-323`
  - body: Current CodexBar per-model breakdowns can expose `totalTokens` without input/output/cache splits. This patch stores absent split fields as dataclass zeroes and emits them whenever `tokensAvailable` is true, so JSON falsely reports zero input/output/cache tokens instead of unknown. Track field availability and emit `null` for missing split fields.
  - confidence: 0.86
