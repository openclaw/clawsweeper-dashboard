## Summary

Keep open. The PR is useful and has sufficient terminal proof, but the current head still has a concrete JSON correctness bug: total-only per-model rows make unavailable split fields appear as zero.

## What This Changes

The PR updates the `model-usage` skill, script, docs, and tests so `--mode all` can include per-model token totals from CodexBar and OpenAI-style breakdown fields.

## Best Possible Solution

Land a corrected skill enhancement that reports per-model total tokens when present, preserves unknown split fields as `null`, and leaves cost normalization to https://github.com/openclaw/openclaw/issues/37878.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a skill feature PR. Current main source confirms the missing token summaries, and the PR body provides terminal proof for the proposed behavior.

## Solution Assessment

No, not yet. Parsing token data at the `modelBreakdowns[]` boundary is the right direction, but JSON output should not turn unavailable per-model split fields into zeroes for current CodexBar total-only rows.
