## Summary

Keep open: this is a useful skill enhancement, but the PR still has a blocking JSON correctness bug for current CodexBar total-only per-model token rows.

## What This Changes

The PR updates the `model-usage` skill script, docs, and tests to include per-model token totals in `--mode all` text and JSON output.

## Best Possible Solution

Land the skill enhancement after it preserves unknown per-model split fields as `null` in JSON while still reporting available per-model totals.

## Reproduction Assessment

Yes for the review finding: a CodexBar-style `modelBreakdowns[]` row with `totalTokens` but no split fields exercises the PR path, and source inspection shows the PR serializes missing split fields as zeroes.

## Solution Assessment

No, not yet. Parsing per-model token totals at the `modelBreakdowns[]` boundary is the right direction, but the JSON model needs per-field availability so unavailable splits stay unknown.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve unknown per-model token splits:** `skills/model-usage/scripts/model_usage.py:320-323`

With this patch, missing split fields stay at the dataclass default `0` and `build_json_all` serializes them whenever `tokensAvailable` is true, so real JSON output would claim zero split tokens instead of unknown; track per-field availability and emit `null` for fields that were not present.
