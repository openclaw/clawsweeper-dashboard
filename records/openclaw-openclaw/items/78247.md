## What This Changes

The branch updates the model-usage skill docs, Python script, and tests so `--mode all` per-model summaries include token fields when present and show `tokens unavailable` for cost-only rows.

## Best Possible Solution

Land the skill enhancement after preserving unknown split fields as `null`, adding totalTokens-only coverage, and refreshing the branch against current main without losing the finite-cost repair path from the related open PR.

## Reproduction Assessment

Yes, for the review finding: source inspection shows that a CodexBar model breakdown containing only `totalTokens` sets `tokens_available` while split counters remain default zeroes. The feature itself is not a bug reproduction; current main and the latest release simply do not implement token summaries in all-mode output.

## Solution Assessment

No, not yet. Parsing token data at the `modelBreakdowns[]` boundary is the right owner-boundary fix, but the JSON shape must distinguish unavailable split fields from real zero-token usage.
