## Summary

Keep open: this is a useful model-usage skill enhancement, but the current patch should not merge because it serializes unavailable per-model token splits and optional costs as zero values.

## What This Changes

The PR updates the model-usage skill script, docs, and tests so `--mode all` reports per-model token totals in text and JSON when `modelBreakdowns[]` exposes token fields.

## Best Possible Solution

Land a corrected skill enhancement that reports per-model total tokens when present, preserves unknown token splits and optional costs as `null` or omitted, and keeps broader cost normalization tracked by https://github.com/openclaw/openclaw/issues/37878.

## Reproduction Assessment

Yes for the blocking review findings: a CodexBar-shaped `modelBreakdowns[]` row with `totalTokens` but no split fields exercises the false zero split output, and an optional-cost row exercises the false zero cost output. This is source-reproducible from the PR code and current CodexBar payload structs.

## Solution Assessment

No, not yet. Parsing token data at the `modelBreakdowns[]` boundary is the right direction, but the data model needs per-field availability so absent upstream fields stay unknown instead of becoming zero.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve unknown per-model token splits:** `skills/model-usage/scripts/model_usage.py:320-323`
  - body: CodexBar per-model breakdowns can expose `totalTokens` without input/output/cache splits. This patch stores absent split fields as dataclass zeroes and emits them whenever `tokensAvailable` is true, so JSON falsely reports zero input/output/cache tokens instead of unknown; track field availability and emit `null` for missing split fields.
  - confidence: 0.88
- **[P2] Do not invent zero-cost model rows:** `skills/model-usage/scripts/model_usage.py:147-150`
  - body: CodexBar model breakdown costs are optional, but this creates a `ModelUsage` with cost `0.0` before proving a valid cost exists. A total-token-only row, or a row with an invalid cost, will be printed and serialized as a real zero-dollar model; keep cost nullable or preserve the prior skip behavior for rows without a valid cost.
  - confidence: 0.8
