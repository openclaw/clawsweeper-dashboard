## Summary

Keep open: the PR is a plausible model-usage skill enhancement with sufficient terminal proof, but it currently serializes unknown per-model split token fields as zero when CodexBar exposes only per-model `totalTokens`. That JSON correctness bug should be repaired before merge.

## What This Changes

The PR updates the model-usage skill docs, Python parser, output rendering, JSON serialization, and tests so `--mode all` can include per-model token usage when token fields are present.

## Best Possible Solution

Land a corrected skill enhancement that reports per-model total tokens when present, preserves unavailable split fields as `null`, and leaves cost normalization to https://github.com/openclaw/openclaw/issues/37878.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a skill feature PR. Current main source confirms the missing token summaries, and the PR body provides terminal output for the proposed behavior.

## Solution Assessment

No, not yet. Parsing token data at the `modelBreakdowns[]` boundary is the right direction, but JSON output should not turn unavailable per-model split fields into zeroes for current CodexBar total-only rows.
