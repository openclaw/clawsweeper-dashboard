## Summary

Keep open. Current main is still cost-only for `model-usage --mode all`, and this PR is useful, but the head still has a blocking JSON correctness bug for CodexBar rows that expose only `totalTokens`.

## What This Changes

This PR updates the `model-usage` skill script, docs, and tests to add per-model token usage to `--mode all` text and JSON summaries.

## Best Possible Solution

Land a corrected skill enhancement that reports per-model total tokens when present, preserves unavailable split fields as `null`, and leaves cost normalization to https://github.com/openclaw/openclaw/issues/37878.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a skill feature PR. Current main source confirms the missing token summaries, and the PR body provides terminal output for the proposed behavior.

## Solution Assessment

No, not yet. Parsing token data at the `modelBreakdowns[]` boundary is the right direction, but the JSON output should not turn unavailable per-model split fields into zeroes for current CodexBar total-only rows.

## Review Findings
