## Summary

Keep open for maintainer review: the patch fixes a real current-main and latest-release unbounded Google Chat media download path, and I found no concrete correctness or security regression in the diff. The remaining blocker is compatibility acceptance for changing direct helper calls that omit `maxBytes` from uncapped buffering to a 16 MiB fail-closed default.

## What This Changes

The PR adds a 16 MiB default cap to Google Chat media downloads when `maxBytes` is omitted and adds a regression test for the uncapped direct-helper path.

## Best Possible Solution

Merge after maintainers accept the 16 MiB omitted-`maxBytes` fallback as the direct helper contract; otherwise adjust the fallback to the configured Google Chat media cap before landing.

## Reproduction Assessment

Yes. Source inspection shows current `downloadGoogleChatMedia` can pass omitted `maxBytes` into `fetchBuffer`, where successful responses use unbounded `arrayBuffer()`; this read-only review did not run the test suite.

## Solution Assessment

Yes, mostly. Routing media responses through `readResponseWithLimit` at the Google Chat helper boundary is the narrow maintainable fix, while the exact fallback cap remains a maintainer compatibility choice.
