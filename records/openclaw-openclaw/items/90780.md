## What This Changes

The PR narrows QQBot bundled entry registration so tool discovery loads only tool registrations and defers sender auth module loading until `qqbot_channel_api` execution.

## Best Possible Solution

Land the narrow QQBot entry-path optimization after normal maintainer and CI gates, preserving full runtime registration and lazy sender loading.

## Solution Assessment

Yes. The existing plugin SDK already routes tool discovery through `registerFull` and suppresses channel runtime hydration, so narrowing QQBot's loader locally is the smallest maintainable fix; a core SDK change would be broader than needed.
