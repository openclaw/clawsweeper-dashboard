## Summary

Keep open. Current main still has real queued cron execution, no cron test/validate command, no input-driven skill test command, and no global tool interception contract for side-effect-free dry runs; related #41418 only covers a narrower global agent-tool dry-run proposal.

## What This Changes

The issue requests dry-run/test support for cron jobs and skills, including cron test/validate CLI commands, a skill test runner, mocked or read-only tool calls, JSON/CI output, and examples.

## Best Possible Solution

Design an additive shared testing surface: cron test/validate RPC plus CLI, an explicit skill test runner, and a coordinated tool-interception contract that composes with #41418.

## Reproduction Assessment

Not applicable as a feature request. The current-main verification path is high-confidence source inspection: the relevant CLI/protocol/runtime paths have no requested test/validate commands and cron.run still enqueues real work.

## Solution Assessment

Unclear as an implementation plan because the issue bundles cron, skills, CI output, and global tool mocking. The maintainable direction is to split the API contract deliberately while keeping the shared tool interception layer coordinated with #41418.
