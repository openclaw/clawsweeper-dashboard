## Summary

Keep this open. Current main and the latest release still do not provide side-effect-free cron test/validate commands, an input-driven skills test runner, or a general tool/API mock-readonly interception layer; the existing cron surfaces only support real queued runs, run history, and delivery previews.

## What This Changes

Requests dry-run/test/validate support for cron jobs and skills, including CI-friendly output and mocked or read-only tool/API execution.

## Best Possible Solution

Keep the issue open for an additive design: cron test/validate RPC plus CLI, a deterministic skills scenario runner, and a request-scoped tool-interception contract coordinated with the global dry-run work before adding any config surface.

## Reproduction Assessment

Yes. This is source-reproducible: current main has cron run/runs/show and skills list/info/check flows, but no cron test/validate command, no skills test runner, and no protocol dry-run mode for cron execution.

## Solution Assessment

Unclear as written. The requested direction is useful, but the maintainable solution needs maintainer decisions on the cron/skills boundary, config shape, and global tool-interception contract before implementation.
