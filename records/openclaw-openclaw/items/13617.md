## Summary

Keep this open: current main and the latest release still lack side-effect-free cron test/validate commands, an input-driven skills test runner, and the shared tool-interception contract needed to make dry-run safe.

## What This Changes

The issue requests new cron and skills dry-run/test CLI support with validation, mocked or read-only tool behavior, CI-friendly output, and optional testing configuration.

## Best Possible Solution

Define an additive core test contract: cron validate/test RPC plus CLI, an input-driven skill harness, and one request-scoped side-effect interception and state-isolation boundary shared with the global dry-run design.

## Reproduction Assessment

Yes for the source state: current main lacks cron test/validate commands and a skills test runner, while cron run still enqueues real work. This is a feature gap rather than a failing existing-contract bug reproduction.

## Solution Assessment

No, not as one broad persistent testing config and mock layer without a design decision. The safer path is a request-scoped cron/skills harness coordinated with the open global dry-run issue for shared tool interception.
