## Summary

Keep open. Current main still lacks side-effect-free cron test/validate commands, an input-driven skills test runner, and a shared dry-run/mock interception layer; the prior maintainer comment also kept this open and pointed to #41418 for the overlapping global tool dry-run design.

## What This Changes

The issue requests dry-run and validation support for cron jobs and skills, including mockable tool calls and CI-friendly output.

## Best Possible Solution

Keep this open and design an additive cron test/validate RPC plus CLI, a skills test runner with explicit input/JSON semantics, and a shared dry-run/tool-interception contract coordinated with #41418.

Not applicable as a bug reproduction; this is a feature request. The absence is high-confidence from source/docs inspection: current cron.run enqueues real work, skills has no test command, and tools.invoke dryRun is documented as ignored.
