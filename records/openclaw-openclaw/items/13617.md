## Summary

Current main still lacks side-effect-free cron test/validate commands and an input-driven skill test runner. The issue remains the canonical product-design item; https://github.com/openclaw/openclaw/issues/41418 covers the related universal tool-interception layer but does not replace the cron and skill workflows.

## What This Changes

Add side-effect-free cron and skill test/validation commands with mockable tool execution and CI-friendly output.

## Best Possible Solution

Implement static cron validation separately from request-scoped simulated agent execution, then coordinate with https://github.com/openclaw/openclaw/issues/41418 on one closed tool-interception contract that reports unsupported side effects instead of silently allowing them.

## Reproduction Assessment

Yes at the source level: current main exposes real cron enqueue execution and has no cron test/validate or input-driven skill test command. This is an absent capability rather than a failing existing contract.

## Solution Assessment

No, not as one bundled mode. Static validation, simulated model execution, tool mocking, read-only exemptions, delivery suppression, and duration estimation need separate contracts with explicit guarantees.

## Risks / Open Questions

- A simulation that fails to intercept a tool, plugin hook, delivery adapter, or session mutation could falsely assure users that a run is side-effect-free.
- The proposed persistent testing configuration, mocked API list, and read-only exemptions could become a competing execution-policy system unless their ownership and security semantics are explicit.
- Static validation cannot reliably predict model-selected tool calls or execution duration, so presenting inferred calls and time estimates as authoritative would be misleading.
