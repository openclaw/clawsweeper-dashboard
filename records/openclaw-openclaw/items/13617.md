## Summary

Keep open. Current main still lacks side-effect-free cron test/validate commands, an input-driven skill test runner, and a shared dry-run/mock/read-only tool interception layer; the recent discussion also explicitly kept this as maintainer design work.

## What This Changes

Request adds side-effect-free cron test/validate and skill test flows with CLI/RPC JSON output, mock/read-only tool behavior, and CI-friendly dry-run semantics.

## Best Possible Solution

Design an additive core implementation: cron test/validate RPC plus CLI, an input-driven skills test runner with JSON and exit-code semantics, and a coordinated dry-run/mock/read-only tool interception contract aligned with #41418.

## Reproduction Assessment

Not applicable as a feature request. The high-confidence verification path is static inspection of current CLI registrations, Gateway handlers, protocol schemas, docs, and the shared tool adapter, all of which show the requested framework is absent.

## Solution Assessment

Unclear until maintainers settle the cross-cutting design. The requested direction is valid, but the maintainable solution should coordinate cron, skills, protocol, and tool interception rather than adding isolated command stubs.
