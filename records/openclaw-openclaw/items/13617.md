## Summary

Keep open: current main still lacks the requested side-effect-free cron test/validate commands, an input-driven skills test runner, and a shared dry-run/mock contract. ## What This Changes

Request to add dry-run/test and validation support for cron jobs and skills, including mockable APIs, CI-friendly output, and optional testing configuration.

## Best Possible Solution

Define an additive core design for cron test/validate RPC plus CLI, a skills test runner with explicit input/output semantics, and a request-scoped tool interception contract coordinated with https://github.com/openclaw/openclaw/issues/41418.
