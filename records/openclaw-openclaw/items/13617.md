## Summary

Keep open. Current main still has no side-effect-free cron test/validate command, no input-driven skills test runner, and no general mock/dry-run interception layer; the open global dry-run discussion in #41418 is adjacent rather than a complete substitute.

## What This Changes

The issue requests side-effect-free test/validate commands for cron jobs and skills, plus a dry-run/mocking layer suitable for local debugging and CI.

## Best Possible Solution

Keep this issue open for an additive design covering cron test/validate RPC plus CLI, a skills test runner with explicit input and CI output, and a shared dry-run/tool-interception contract coordinated with #41418.
