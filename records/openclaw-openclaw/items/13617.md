## Summary

Keep this open: current main and the latest release still lack the requested side-effect-free cron test/validate commands, input-driven skill test runner, and shared dry-run/tool-interception contract. The open global dry-run issue and the open cron-add preview PR are related but only cover adjacent or narrower pieces.

## What This Changes

The issue requests a dry-run/test framework for cron jobs and skills so jobs, skill inputs, and tool calls can be validated without production side effects.

## Best Possible Solution

Define an additive core test contract: cron validate/test RPC plus CLI, an input-driven skill harness, and one request-scoped side-effect interception and state-isolation boundary shared with the global dry-run design.

## Reproduction Assessment

Yes for the source state: current main lacks cron test/validate commands and a skills test runner, while cron run still enqueues real work. This is a feature gap rather than a failing existing-contract bug reproduction.

## Solution Assessment

No, not as one broad persistent testing config and mock layer without a design decision. The safer path is a request-scoped cron/skills harness coordinated with https://github.com/openclaw/openclaw/issues/41418 for shared tool interception.
