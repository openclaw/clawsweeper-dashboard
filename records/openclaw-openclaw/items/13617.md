## Summary

Keep open: current main and v2026.6.8 still lack the requested side-effect-free cron test/validate commands, input-driven skills test runner, and shared dry-run/mock tool-interception contract. Existing cron delivery previews and the open cron-add preview PR cover only narrow adjacent behavior, while the global dry-run and scheduler-diagnostics issues remain related but not replacements.

## What This Changes

The issue requests side-effect-free cron test/validate commands, a skill test runner, and mockable dry-run tooling for CI and local development.

## Best Possible Solution

Keep this as a broad design issue and land an additive maintainer-approved contract that separates cron validation/preview from real queued execution, adds a skills test runner with explicit input/CI output semantics, and coordinates any shared tool interception with the global dry-run design.

## Reproduction Assessment

Yes. Source inspection on current main and v2026.6.8 shows cron exposes real run/history commands without test/validate modes, the Gateway schema lacks dry-run params, skills has no input-driven test command, and the shared tool adapter still executes or delegates tools.

## Solution Assessment

Unclear as proposed. The direction fits OpenClaw, but the safest solution needs a maintainer-approved public CLI/Gateway and security contract rather than isolated cron preview behavior or a process-global flag.
