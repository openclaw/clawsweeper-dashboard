## Summary

Keep this open: current main and the latest release still do not provide the requested side-effect-free cron test/validate commands, input-driven skill test runner, or shared dry-run/mock tool-interception layer. Existing cron delivery previews and the open cron-add preview PR cover only narrow adjacent pieces, while the open global dry-run issue covers a different but related layer.

## What This Changes

The issue requests side-effect-free cron and skill testing commands, CI/JSON output, dry-run configuration, and a mock/read-only tool layer for safe automation testing.

## Best Possible Solution

Keep this issue as the broad design tracker and land an additive contract that separates cron validation/preview from real queued execution, adds a skills test runner with explicit inputs and CI output, and coordinates shared tool interception with the global dry-run design.

## Reproduction Assessment

Yes. Source inspection on current main shows the requested commands and shared interception layer are missing: cron has real run/runs paths, skills has list/info/check/workshop flows, and the tool adapter calls tool.execute without dry-run interception.

## Solution Assessment

Unclear as proposed. The direction fits OpenClaw, but the best solution needs a maintainer-approved contract for public CLI/Gateway semantics and security-safe tool interception rather than a one-off cron add preview or ad hoc global flag.
