## Summary

## What This Changes

Requests side-effect-free cron and skill testing commands with validation, mockable tool calls, and CI-friendly output.

## Best Possible Solution

Add additive cron validation and delivery-preview RPC/CLI contracts plus an explicit input-driven skill harness, while defining one centrally enforced tool-interception and state-isolation contract together with https://github.com/openclaw/openclaw/issues/41418 rather than creating a parallel mocking layer.

## Reproduction Assessment

## Solution Assessment

No, not as one persistent-config testing framework. The narrower maintainable design is command-scoped cron validation and skill testing backed by one shared, explicitly designed tool-interception and state-isolation contract.

## Security Review

Status: needs_attention

Summary: The requested feature needs a security design that can prove all side-effecting tools and state mutations are intercepted rather than merely labeling a run as dry.

Concerns:

- **[medium] Define an enforceable side-effect boundary:**
  - body: A safe test mode must centrally distinguish blocked, simulated, and explicitly permitted operations across built-in, plugin, client-hosted, and transport tools; partial per-tool mocking would create a false safety guarantee.
  - confidence: 0.94
- **[medium] Specify session-state isolation:**
  - body: Even when external calls are blocked, normal agent execution may change transcripts or session state, so the design must define whether tests use disposable state or suppress persistence.
