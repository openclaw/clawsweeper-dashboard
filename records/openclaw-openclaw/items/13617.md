## What This Changes

The issue requests side-effect-free cron test/validate commands, a skills test command with explicit input, and a mock/read-only dry-run layer for CI and local debugging.

## Best Possible Solution

Keep this as the canonical cron/skills testing design item and design an additive cron test/validate RPC plus CLI, a skills test runner, and a coordinated tool-interception contract with the global dry-run work.
