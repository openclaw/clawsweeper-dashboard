## Summary

Keep open: the PR targets a useful Feishu document-comment workflow, but the proposed implementation is not mergeable because it calls a missing `docx.documentComment` SDK member and returns only one page of comment listings. Current main already has a partial, tested Drive comment surface under `feishu_drive`, so this needs maintainer API-surface judgment plus repair before merge.

## What This Changes

The PR adds `list_comments`, `get_comment`, `create_comment`, `update_comment`, and `delete_comment` actions to `feishu_doc`, updates the Feishu doc skill text, and adds docx tests for those comment actions.

## Best Possible Solution

Expose one coherent Feishu comment interface: either keep `feishu_drive` as canonical, or add `feishu_doc` wrappers that call verified Drive comment APIs, preserve pagination metadata or iterate pages, align unsupported update/delete semantics with Feishu's actual API, update docs, and add focused tests plus any required changelog entry.

## Reproduction Assessment

Yes. Current main can be checked by invoking or inspecting `feishu_doc` action dispatch, and the PR failure path is reproducible from the pinned SDK types: `client.docx.documentComment` is absent while Drive comment APIs are present.

## Solution Assessment

No. The PR's direct `docx.documentComment` path is not the best solution because it bypasses the dependency contract and duplicates an existing Drive comment surface; the safer direction is a maintainer-approved wrapper over the real Drive comment APIs.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Use the real Drive comment API:** `extensions/feishu/src/docx.ts:953`
  - body: The pinned Lark SDK exposes comment APIs under `client.drive.fileComment` / `fileCommentReply`, while `client.docx` has no `documentComment` member. Because this casts `docx` to `any`, a real client will dereference an absent API and every new comment action will fail at runtime. Route these actions through the verified Drive comment API or a typed wrapper before exposing them.
  - confidence: 0.91
- **[P2] Page through comment listings:** `extensions/feishu/src/docx.ts:964-972`
  - body: `list_comments` performs one list request and reports `total` from that page's item count. Feishu comment listing is paginated (`has_more` / `page_token`), so active documents can omit later comments and return a misleading count. Return pagination metadata or iterate pages before presenting this as a full list.
  - confidence: 0.86
