## Summary

Keep this PR open for maintainer review: it adds a new `feishu_doc` comment surface, but the implementation still calls an unsupported SDK path and overlaps with the existing Drive-backed Feishu comment workflow on current `main`. The external PR also lacks observable after-fix real behavior proof.

## What This Changes

Adds `feishu_doc` comment actions, schemas, docs, and mocked tests for listing, getting, creating, updating, and deleting Feishu Docx comments.

## Best Possible Solution

Keep one coherent Feishu comment contract: either keep `feishu_drive` canonical, or add maintainer-approved typed Drive-backed `feishu_doc` wrappers with pagination, supported reply/root semantics, docs, tests, and release-owned changelog coverage.

## Reproduction Assessment

Yes. The failure is source-reproducible by comparing the PR diff with the pinned SDK 1.63.1 contract: the PR calls `docx.documentComment`, while comments are exposed under `drive.fileComment` and `drive.fileCommentReply`.

## Solution Assessment

No. The direct `feishu_doc` implementation is not the best fix as written; a typed Drive-backed wrapper or keeping `feishu_drive` as the canonical comment tool is safer and matches current `main`.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Use the real Drive comment APIs:** `extensions/feishu/src/docx.ts:953`
  - body: The pinned Lark SDK exposes comments as `client.drive.fileComment` and `client.drive.fileCommentReply`; `client.docx` does not expose `documentComment`. Because this casts `docx` to `any`, real comment actions would dereference an absent SDK member and fail at runtime.
  - confidence: 0.94
- **[P2] Expose only supported edit and delete semantics:** `extensions/feishu/src/doc-schema.ts:208-218`
  - body: `update_comment` and `delete_comment` accept only a `comment_id`, but the verified Drive API updates/deletes replies via `fileCommentReply` and requires a `reply_id`; root comments expose solve-state patching, not arbitrary content update/delete through this shape. The schema, docs, and dispatcher need to match operations Feishu can actually call.
  - confidence: 0.86
- **[P2] Page through comment listings:** `extensions/feishu/src/docx.ts:963-972`
  - body: `list_comments` performs one list request and reports `total` from that page’s item count. Feishu comment listing is paginated with `has_more` and `page_token`, so active documents can omit later comments and return a misleading count unless the tool returns pagination metadata or iterates all pages.
  - confidence: 0.9
