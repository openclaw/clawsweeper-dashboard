## Summary

Keep open: the PR overlaps with shipped Feishu Drive comment support, but it still proposes extra `feishu_doc` edit/delete-style surface, lacks real Feishu proof, and the branch implementation has source-level correctness problems.

## What This Changes

The PR adds `feishu_doc` comment actions, schemas, mocked tests, and skill documentation for listing, reading, creating, updating, replying to, and deleting Feishu document comments.

## Best Possible Solution

Keep `feishu_drive` as the canonical comment surface, and only add narrow Drive-backed edit/delete/resolve actions if Feishu owners decide that tool surface should exist.

## Reproduction Assessment

Yes for the branch-level defects from source and dependency inspection; no live Feishu tenant reproduction or after-fix proof is attached. The SDK exposes Drive comment APIs while the PR calls `client.docx.documentComment`, and its listing path drops pagination metadata.

## Solution Assessment

No. The current PR duplicates shipped `feishu_drive` behavior and uses the wrong API owner boundary; any missing comment-management actions should be designed on the Drive comment endpoints.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Use the Drive comment API surface:** `extensions/feishu/src/docx.ts:955-958`
  - body: The new comment actions call `(client.docx as any).documentComment.*`, but the pinned Lark SDK exposes document comments under `client.drive.fileComment` and `client.drive.fileCommentReply`. In a real client this docx property is not a supported typed surface, so the tool can fail at runtime even though the mocks pass.
  - confidence: 0.86
- **[P2] Preserve pagination metadata for comment listings:** `extensions/feishu/src/docx.ts:959-968`
  - body: `list_comments` performs one list request and returns `total` from that page's `items.length`, so a multi-page comment thread is silently truncated and reported as complete. The shipped Drive tool returns `has_more` and `page_token`; this path needs equivalent pagination handling or metadata before it can be reliable.
  - confidence: 0.9
