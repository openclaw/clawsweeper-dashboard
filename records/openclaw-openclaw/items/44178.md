## Summary

Keep this PR open for maintainer review. Current main does not implement the requested direct `feishu_doc` comment actions, and the PR is not merge-ready because it calls a `docx.documentComment` SDK surface that is absent from the locked Lark SDK while also overlapping the shipped `feishu_drive` comment surface.

## What This Changes

The PR adds Feishu Docx comment actions to `feishu_doc`, with schema entries, implementation, tests, and skill documentation for listing, getting, creating, updating, and deleting comments.

## Best Possible Solution

Decide the public Feishu comment tool surface first, then implement it through typed, dependency-backed Drive comment APIs or a documented SDK seam, preserving pagination metadata and aligning return shapes with the shipped Drive comment behavior.

## Reproduction Assessment

Yes. Current main can be checked by searching the `feishu_doc` schema and dispatcher, and the PR failure path can be reproduced by comparing the diff to `@larksuiteoapi/node-sdk` 1.62.0, where `client.docx.documentComment` is not present.

## Solution Assessment

No. The PR's direct `docx.documentComment` path is not the best fix because it bypasses the SDK type contract and appears absent at runtime; the safer path is to build on the existing typed Drive comment resources or add a verified wrapper around them.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Use a real Feishu comment API:** `extensions/feishu/src/docx.ts:953`
  - body: The locked Lark SDK does not expose `client.docx.documentComment`; it exposes Drive comment resources (`drive.fileComment` / `fileCommentReply`). With a real client, this dereferences `undefined`, so every new comment action returns an error despite the mocked tests passing. Please route through the existing Drive comment API or add a verified typed SDK seam before exposing the actions.
  - confidence: 0.91
- **[P2] Page through comment listings:** `extensions/feishu/src/docx.ts:964`
  - body: The new `list_comments` action returns only the first comment-list response and sets `total` from that page's item count. Feishu comment listing is paginated, so active docs will silently omit later comments and report an incorrect total. Return pagination metadata or iterate pages before claiming a full result.
  - confidence: 0.86
