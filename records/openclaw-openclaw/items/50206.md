## Summary

Keep open. Current main still lacks this opt-in provider config/header path, and the PR is useful but needs normalized provider-config lookup, clearer disclosure wording, and changelog coverage before merge.

## What This Changes

The PR adds an opt-in `injectAgentHeader` provider config flag and wraps LLM stream requests to send `X-OpenClaw-Agent` with the active agent id.

## Best Possible Solution

Land a narrow opt-in implementation that reuses normalized provider config resolution, documents trusted-proxy disclosure, adds active-version changelog coverage, and preserves current default behavior.

## Reproduction Assessment

Yes for the blocking PR defect. Configure the provider under `z-ai` while the runtime provider is `z.ai`; current model resolution finds that config through normalized lookup, but the PR's direct lookup would skip header injection.

## Solution Assessment

No. The feature shape is close, but the mergeable path should share the existing normalized provider lookup and add trusted-provider documentation plus changelog coverage.
