## Summary

Keep open: the feature is not implemented on current main, and this PR is not merge-ready because it has current-main compatibility issues, skips normalized provider lookup, lacks trusted-endpoint disclosure, and still has no real request-path proof.

## What This Changes

This PR adds a provider-level `injectAgentHeader` config flag, schema/help metadata, a stream wrapper, and tests to send `X-OpenClaw-Agent` on outgoing LLM requests.

## Best Possible Solution

Either revise this narrow flag to use current dependencies, normalized provider resolution, trusted-endpoint disclosure, and real proxy/provider capture proof, or supersede it by landing a broader per-agent provider request header design.

## Reproduction Assessment

Yes for the proposed-patch defects by source inspection: the PR directly indexes provider config while current main resolves normalized provider aliases, and the new test imports a package name current main no longer depends on. This is not a current-main bug reproduction.

## Solution Assessment

No, not as-is. The feature direction is plausible, but the patch needs current-main dependency alignment, normalized lookup, trusted-endpoint disclosure, and real request-path proof before maintainers choose it over the broader provider-header design.
