## Summary

Keep open: current main still lacks this opt-in agent header feature, and the PR has blocking review issues plus missing real behavior proof. The broader provider-header work is relevant but remains open and does not yet make this exact branch safely obsolete.

## What This Changes

The PR adds an opt-in `models.providers.*.injectAgentHeader` config flag, schema/help metadata, a stream wrapper, and tests to send `X-OpenClaw-Agent: <agentId>` on matching provider requests.

## Best Possible Solution

Either update this narrow PR to use normalized provider lookup, document the trusted-endpoint disclosure boundary, and add redacted request-path proof, or close it once the broader provider-header design lands.

## Reproduction Assessment

Yes for the PR defects by source inspection: current model resolution accepts normalized provider aliases, while the PR's direct provider-key lookup would skip `injectAgentHeader` for alias-matched providers. This is a proposed-patch failure path, not a current-main bug reproduction.

## Solution Assessment

No, not as-is. The feature direction is plausible, but the implementation needs normalized provider lookup, trusted-endpoint disclosure docs, and real request-path proof before maintainers choose this narrow flag over the broader provider-header design.
