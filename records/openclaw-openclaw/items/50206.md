## Summary

Keep open: current main still lacks the requested opt-in agent header, and this PR remains the canonical narrow implementation for the closed duplicate reports, but the branch needs correctness, disclosure, and contributor proof work before merge.

## What This Changes

The PR adds an `injectAgentHeader` provider config option, schema/help metadata, a stream wrapper, and tests to send `X-OpenClaw-Agent: <agentId>` on matching model-provider requests.

## Best Possible Solution

Land a narrow opt-in implementation only after it reuses normalized provider config resolution, documents the disclosure boundary, and proves the header on an actual request path; alternatively, explicitly fold this use case into the broader provider-header design before closing this PR.

## Reproduction Assessment

Yes for the PR defect by source inspection: current model resolution accepts normalized provider aliases, while the PR's direct provider-key lookup would skip `injectAgentHeader` for an alias-matched provider. This is a proposed-patch failure path, not a current-main bug reproduction.

## Solution Assessment

No, not as-is. The feature direction is plausible, but the implementation should share normalized provider lookup and add trusted-endpoint disclosure before maintainers decide whether to land this narrow flag or the broader provider-header surface.
