## Summary

Keep open. Current main still lacks the requested opt-in provider setting and header injection, and this PR is useful but not ready because the implementation bypasses normalized provider lookup and the docs understate the disclosure risk.

## What This Changes

This PR adds an `injectAgentHeader` provider config field, schema/help/label entries, test support, targeted tests, and a stream wrapper that sends `X-OpenClaw-Agent` with the active agent ID on model requests.

## Best Possible Solution

Land a narrow, rebased implementation that remains explicitly opt-in, resolves provider config through the same normalized provider path as model selection, adds alias regression coverage, and documents that the header should be enabled only for trusted local or operator-controlled proxies.

## Reproduction Assessment

Yes. Current-main absence is reproducible by inspecting the strict provider schema and runner wrapper path; the PR bug is reproducible by configuring a provider under an alias such as `z-ai` while runtime resolution uses `z.ai`.

## Solution Assessment

No. The opt-in shape is close, but direct provider-map lookup is not the narrowest reliable implementation because current model resolution already supports normalized provider aliases; the docs also need a trusted-provider privacy note.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Resolve provider config through normalized lookup:** `src/agents/pi-embedded-runner/extra-params.ts:328`
  - body: The new gate reads `cfg?.models?.providers?.[provider]` directly. Model resolution falls back to normalized provider matching, so an alias-equivalent config such as `z-ai` can select the model while this option silently stays off for runtime provider `z.ai`. Please reuse the normalized provider helper and add alias coverage.
  - confidence: 0.92

  - body: The help text does not warn that enabling this sends the current agent ID to the configured provider or proxy. Please call out trusted local/operator-controlled use so users do not enable it on third-party providers without understanding the disclosure.
  - confidence: 0.8
