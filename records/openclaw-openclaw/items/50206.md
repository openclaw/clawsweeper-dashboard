## Summary

Keep open: current main does not implement this header feature, and the PR still has a source-reproducible provider-alias bug plus missing real behavior proof before maintainers can consider merge.

## What This Changes

The branch adds an `injectAgentHeader` provider config/schema/help surface, a stream wrapper, and tests to send `X-OpenClaw-Agent` from `applyExtraParamsToAgent`.

## Best Possible Solution

Land a narrow opt-in only after normalized provider lookup, trusted-endpoint disclosure, and real request proof are addressed, or fold this use case into the broader provider-header design.

## Reproduction Assessment

Yes for the PR defect by source inspection: current model resolution accepts normalized provider aliases, while the PR's direct provider-key lookup would skip `injectAgentHeader` for an alias-matched provider. This is a proposed-patch failure path, not a current-main bug reproduction.

## Solution Assessment

No, not as-is. The feature direction is plausible, but the implementation should reuse normalized provider lookup, document the disclosure boundary, and be reconciled with the broader provider-header PR before merge.
