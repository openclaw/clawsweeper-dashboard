## Summary

Keep this PR open: the feature is still useful and not implemented on current main, but the proposed patch still has the normalized-provider lookup defect plus documentation and changelog gaps identified in the existing review.

## What This Changes

The PR adds an opt-in `injectAgentHeader` provider config flag, schema/help/label entries, and a stream wrapper that sends `X-OpenClaw-Agent` on LLM requests when an agent id is available.

## Best Possible Solution

Land a narrow opt-in implementation that reuses normalized provider config resolution, warns that agent ids are disclosed to trusted local/operator-controlled proxies, adds active changelog coverage, and preserves the default no-header behavior.

## Reproduction Assessment

Yes for the blocking PR defect. Configure the provider under `z-ai` while the runtime provider is `z.ai`; current model resolution finds that config through normalized lookup, but the PR's direct lookup would skip header injection.

## Solution Assessment

No. The feature shape is close, but the mergeable path should share existing normalized provider lookup and add trusted-provider documentation, changelog coverage, and alias regression coverage.
