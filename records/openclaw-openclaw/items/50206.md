## Summary

Keep open: current main still lacks this feature, but the PR needs changes before merge because the proposed provider lookup misses normalized aliases and the disclosure/changelog coverage is incomplete.

## What This Changes

The PR adds an opt-in `injectAgentHeader` provider config flag, schema/help/label metadata, and an extra-params stream wrapper with tests to send `X-OpenClaw-Agent` for the active agent id.

## Best Possible Solution

Land a narrow opt-in implementation that reuses normalized provider config resolution, warns about trusted/operator-controlled proxy disclosure, adds an active changelog entry, and preserves the default no-header behavior.

## Reproduction Assessment

Yes for the blocking PR defect. A provider configured as `z-ai` can be found by normalized model resolution for runtime provider `z.ai`, but the PR's direct `cfg?.models?.providers?.[provider]` lookup would not find that config and would skip header injection.

## Solution Assessment

No. The feature direction is close, but the mergeable solution should share existing normalized provider config resolution and add disclosure/changelog coverage rather than introducing a parallel direct lookup.
