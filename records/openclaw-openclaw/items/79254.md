## Summary

Keep open: this still looks like a plausible Telegram forum-topic regression, and current source shows a concrete edge where persisted session activation can force mention gating ahead of group-level `requireMention=false`; the remaining blocker is proving whether that is the reporter's live path.

## What This Changes

The issue reports that Telegram forum/topic messages no longer trigger without an @mention when `requireMention=false`, BotFather privacy is disabled, and group/topic allowlist settings should permit plain messages.

## Best Possible Solution

Live-triage the Telegram topic ingress and session activation state, then either fix the Telegram `requireMention` precedence with regression coverage or document the `/activation`/session-state override clearly if current behavior is intentional.

## Reproduction Assessment

Partly. Source inspection gives a deterministic current-main path when a Telegram forum-topic session row has `groupActivation: "mention"`: it resolves `requireMention` true before group-level `requireMention=false` and can hit the `no-mention` skip gate, but the reporter's live session-store/update evidence is still missing.

## Solution Assessment

Unclear. A narrow code/test fix is plausible, but the best maintainable solution depends on the intended precedence between explicit group config and the user-facing `/activation` session override.
