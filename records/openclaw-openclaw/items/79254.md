## Summary

Keep open: current main still has a source-reproducible Telegram path where persisted `groupActivation: "mention"` can outrank group-level `requireMention: false` when no topic override exists, causing plain forum-topic messages to be skipped as `no-mention`.

## What This Changes

The issue reports that Telegram forum-topic group messages only trigger on direct @mentions despite `requireMention=false`, disabled BotFather privacy, and allowed group/user configuration.

## Best Possible Solution

Define and implement a clear precedence contract for explicit Telegram group/topic `requireMention` config versus persisted `/activation` state, then add focused regression coverage for the group-level false case.

## Reproduction Assessment

Yes for the likely current-main path: a Telegram forum-topic message with no topic override, group-level `requireMention=false`, and session `groupActivation="mention"` will resolve `requireMention=true` and hit the `no-mention` skip gate. The exact reporter deployment still needs a session/update capture to confirm that this is the live cause.

## Solution Assessment

Unclear until maintainers confirm the intended precedence between persisted `/activation` and explicit group config. The narrow code direction is likely in Telegram mention precedence plus a regression test, but the product contract should be settled first.
