## Summary

Current main and the stable v2026.5.18+ behavior no longer match the report: unmentioned Telegram forum-topic messages with `requireMention: false` are classified as actionable user requests by default, with quiet room-event handling only when explicitly configured.

## What This Changes

The issue reports that Telegram forum-topic group messages are ignored unless the bot is directly mentioned despite `requireMention: false`, BotFather privacy disabled, and group/topic allowlist cleanup.

## Best Possible Solution

Keep the shipped Telegram/group classification behavior and close this older release report; any new reproduction on v2026.5.18 or newer should be filed with fresh logs and `/activation` state.

## Reproduction Assessment

Yes for the reported release path: the issue includes concrete 2026.5.6 behavior and a matching duplicate with `ignoring group message without mention`, while current main has targeted regression coverage for the same unmentioned forum-topic admission path. I did not run a live Telegram forum repro in this read-only review.

## Solution Assessment

Yes. The current solution keeps the fix in the Telegram/channel inbound classification path, preserves `requireMention: false` as an actionable default, and leaves quiet room-event behavior behind an explicit opt-in setting.
