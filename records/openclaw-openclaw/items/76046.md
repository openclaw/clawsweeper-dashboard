## Summary

Keep open. Current main still lacks per-agent `contextInjection`, and the issue is now paired with open, unmerged PR #76054 using closing syntax, so the right path is to review/land or close that PR first.

## What This Changes

Add an `agents.list[].contextInjection` override that falls back to `agents.defaults.contextInjection` and then `"always"`.

## Best Possible Solution

Review and land #76054, or a narrow equivalent, so the schema, docs, resolver, and embedded run paths all honor the per-agent override while leaving broader bootstrap-budget overrides separate.

## Reproduction Assessment

Yes. A config with `agents.list[].contextInjection` is outside the strict current `AgentEntrySchema`, and even a typed bypass would be ignored because the resolver reads only `agents.defaults.contextInjection`.

## Solution Assessment

Yes. The requested per-agent override to defaults to `"always"` fallback is the narrowest maintainable fix for mixed strict and advisory agents.
