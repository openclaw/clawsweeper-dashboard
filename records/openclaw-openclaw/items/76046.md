## Summary

Keep open. Current main still lacks the requested per-agent `contextInjection` config, and the issue is already paired with open closing PR #76054, so the issue should stay open until that PR or an equivalent fix lands.

## What This Changes

Request per-agent `agents.list[].contextInjection` with fallback to `agents.defaults.contextInjection` and then the code default.

## Best Possible Solution

Land #76054 or an equivalent narrow change that adds the per-agent config contract, threads the active agent id into context-injection resolution, updates generated config/help/docs, and leaves broader bootstrap budget overrides for separate product review.

## Reproduction Assessment

Yes. A config containing `agents.list[].contextInjection` is outside the strict per-agent schema, and the runtime resolver ignores `agents.list` entirely even if the value bypassed validation.

## Solution Assessment

Yes. The proposed fallback order, per-agent override to defaults to `"always"`, is the smallest maintainable fix for mixed strict and advisory agents; broadening other bootstrap defaults should be a separate decision.
