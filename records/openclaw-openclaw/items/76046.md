## Summary

Keep open. Current main still has only the global `agents.defaults.contextInjection` runtime/schema/docs contract, and the linked implementation PR is open and unmerged, so this issue should remain open until that PR lands or is closed.

## What This Changes

Request `agents.list[].contextInjection` as a per-agent override with fallback to `agents.defaults.contextInjection` and then `"always"`.

## Best Possible Solution

Land or close #76054; the desired end state is a narrow config/schema/docs contract plus active-agent-id-aware resolution in normal embedded runs and compaction.

## Reproduction Assessment

Yes. Static current-main verification is enough: the strict per-agent schema rejects `agents.list[].contextInjection`, and the runtime resolver reads only `agents.defaults.contextInjection` even where the active agent id is already known.

## Solution Assessment

Yes. The requested per-agent override to defaults to `"always"` fallback is the narrow maintainable fix; the broader bootstrap budget knobs mentioned in the issue should remain a separate product decision.
