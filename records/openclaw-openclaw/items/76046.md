## Summary

Keep open. Current main still exposes `contextInjection` only through `agents.defaults`, and the embedded run paths do not pass an agent id when resolving whether to inject bootstrap context.

## What This Changes

Allow `agents.list[].contextInjection` to override `agents.defaults.contextInjection` for agents that need stricter prompt reinjection behavior.

## Best Possible Solution

Add a narrow per-agent config contract and make bootstrap context-injection resolution use the active agent id, while leaving broader bootstrap budget overrides for a separate maintainer decision.

## Reproduction Assessment

Yes. A config with `agents.list[].contextInjection` is outside the strict `AgentEntrySchema`, and even a typed bypass would be ignored because the resolver reads only `agents.defaults.contextInjection`.

## Solution Assessment

Yes. The proposed fallback order, per-agent override to defaults to `"always"`, is the smallest maintainable fix for mixed strict and advisory agents.
