## Summary

Keep open. Current main still does not expose or honor an `agents.list[].contextInjection` override, and the linked implementation PR is open with closing syntax, so this issue should remain open until that PR lands or is explicitly abandoned.

## What This Changes

Request adding a per-agent `contextInjection` config override that falls back to `agents.defaults.contextInjection` and then to `"always"`.

## Best Possible Solution

Review and land or close #76054; the desired end state is a narrow config/schema/docs/test update that resolves context injection by active agent id while preserving the existing default behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows `agents.list[].contextInjection` is outside the strict per-agent schema and ignored by the runtime resolver, which only reads `agents.defaults.contextInjection`.

## Solution Assessment

Yes. The requested fallback order, per-agent override to defaults to `"always"`, is the narrow maintainable implementation; broadening the same pattern to bootstrap budget fields should remain separate follow-up work.
