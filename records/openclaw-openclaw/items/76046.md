## Summary

Keep open. Current main still resolves context injection only from `agents.defaults`, the strict per-agent config schema omits `agents.list[].contextInjection`, and #76054 is the active open implementation PR with closing syntax.

## What This Changes

Add an `agents.list[].contextInjection` per-agent config override that falls back to `agents.defaults.contextInjection` and then `"always"`.

## Best Possible Solution

Review and land or close #76054; the end state should be a narrow resolver, schema, docs, and test update for `contextInjection` only.

## Reproduction Assessment

Yes. Source inspection on current main is enough for the missing config path: `AgentConfig` and `AgentEntrySchema` omit the field, and runtime callers resolve context injection without an agent id.

## Solution Assessment

Yes. The requested fallback order is the narrow maintainable design for this feature; the related bootstrap budget fields should be handled only as a separate product decision.
