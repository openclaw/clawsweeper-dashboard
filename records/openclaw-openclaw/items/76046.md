## Summary

Keep open. Current main only supports `contextInjection` at `agents.defaults`, and the runtime/config/schema/docs paths do not accept or resolve an `agents.list[].contextInjection` override.

## What This Changes

The issue requests a per-agent `contextInjection` config override with fallback to `agents.defaults.contextInjection` and then `always`.

## Best Possible Solution

Add a narrow per-agent `contextInjection` field across config types, validation, generated schema/help, docs, and the embedded run/compaction resolver path, without broadening this PR to the other bootstrap budget knobs.

## Reproduction Assessment

Yes. A static reproduction is enough: `agents.list[].contextInjection` is not in the strict `AgentEntrySchema`, and even if a config object were forced past validation, the runtime resolver reads only `agents.defaults.contextInjection`.

## Solution Assessment

Yes. The requested fallback order is the narrowest maintainable fix because it matches existing per-agent override patterns while preserving the current default behavior for all existing configs.
