## Summary

Keep open. Current main still lacks a per-agent `contextInjection` override, and the issue already has an open closing PR, so the right cleanup action is to leave the issue open until that PR is reviewed and either landed or closed.

## What This Changes

Allow `agents.list[].contextInjection` to override `agents.defaults.contextInjection` for individual agents.

## Best Possible Solution

Review and land a narrow PR that adds the strict per-agent schema/type/docs surface and resolves mode as per-agent entry, then defaults, then `"always"` across normal run and compaction paths.

## Reproduction Assessment

Yes. A source-level reproduction is enough here: current main has no accepted `agents.list[].contextInjection` schema field and the resolver cannot read one, while the run paths call the resolver without an agent id.

## Solution Assessment

Yes for the requested direction. A per-agent override with default fallback is the narrowest maintainable fix because the runtime already resolves session agent ids before building bootstrap context.
