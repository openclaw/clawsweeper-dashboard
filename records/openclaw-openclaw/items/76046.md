## Summary

Keep open. Current main still has only the global `agents.defaults.contextInjection` path, and the linked implementation PR is open and unmerged, so this issue should remain open until that PR lands or is replaced.

## What This Changes

The issue requests `agents.list[].contextInjection` with fallback to `agents.defaults.contextInjection` and then `"always"`.

## Best Possible Solution

Update and land #76054 or an equivalent narrow replacement that adds the per-agent schema/type/docs contract, resolver fallback, active-agent-id call sites, tests, and changelog entry.

## Reproduction Assessment

Yes. On current main, `agents.list[].contextInjection` is outside the strict per-agent schema, and even a typed bypass would be ignored because the resolver and both runtime call sites resolve only the global default.

## Solution Assessment

Yes. The requested fallback order is the narrow maintainable fix for mixed strict/general agents; broadening bootstrap budget overrides should remain a separate product decision.
