## Summary

Keep open. Current main still resolves `contextInjection` only from `agents.defaults`, and #76054 is an open, unmerged PR with closing syntax for this issue, so the issue should stay open until that PR lands or is closed.

## What This Changes

Allow `agents.list[].contextInjection` to override `agents.defaults.contextInjection` for bootstrap context injection mode.

## Best Possible Solution

Review and land or close #76054; the desired end state is a narrow per-agent config/schema/docs contract plus active-agent-id-aware bootstrap injection resolution.

## Reproduction Assessment

Yes. On current main, `agents.list[].contextInjection` is outside the strict per-agent schema, and even a typed bypass would be ignored because the resolver reads only `agents.defaults.contextInjection`.

## Solution Assessment

Yes. The requested fallback order, per-agent override to defaults to `"always"`, is the narrow maintainable fix; broader bootstrap budget overrides should remain a separate maintainer decision.
