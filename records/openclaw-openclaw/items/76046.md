## Summary

Keep open. Current main still lacks a per-agent `contextInjection` override, and open PR #76054 is already linked with closing syntax, so this issue should remain open until that implementation is reviewed and merged or closed.

## What This Changes

Allow `agents.list[]` entries to override `agents.defaults.contextInjection`, falling back to the default and then `"always"`.

## Best Possible Solution

Review and land or close PR #76054, keeping the fix narrowly scoped to per-agent `contextInjection` resolution and leaving broader bootstrap-budget overrides for separate maintainer judgment.

## Reproduction Assessment

Yes. Current main has a strict `AgentEntrySchema` without `contextInjection`, and the runtime resolver ignores the active agent even when the embedded run already has one.

## Solution Assessment

Yes. The requested fallback order, per-agent override to defaults to `"always"`, matches the existing per-agent override pattern and is the narrowest maintainable fix for mixed strict/advisory agents.
