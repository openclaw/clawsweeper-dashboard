Treat Ollama `:cloud` model refs routed through a local daemon as cloud-like for idle timeout/failover while preserving explicit timeout overrides and existing local-model behavior.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/79350. The observable bug is that Ollama `:cloud` models such as `ollama/glm-5.1:cloud` routed through a loopback/private Ollama daemon can be treated as local by `resolveLlmIdleTimeoutMs`, disabling the default cloud idle watchdog unless the user configured `models.providers.<id>.timeoutSeconds`; after tool results, a silent upstream request can then wait for a much later run/cron timeout. Thread provider/model identity into the idle-timeout resolver and keep the local-provider watchdog exemption only for non-cloud Ollama models. Preserve explicit `models.providers.<id>.timeoutSeconds`, explicit run timeout, `agents.defaults.timeoutSeconds`, and cron behavior. Add focused unit coverage for `baseUrl: http://127.0.0.1:11434` plus `modelId: glm-5.1:cloud` returning the default cloud idle timeout, non-cloud local Ollama still returning `0`, and explicit provider request timeout still winning. Update Ollama docs and `CHANGELOG.md` if the behavior change is user-facing. Do not change Ollama auth, tool-result conversion, tool schema normalization, or fallback list semantics.
