## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P1] Keep dry-run from submitting Codex turns:** `extensions/codex/src/continuity/bridge.ts:303-315`
  - body: `telegramDryRun` is checked only after the app-server request has already been sent. With writes enabled and methods confirmed, a dry-run `/codex goal` or steer request can still submit `turn/start` or `turn/steer`, so move the dry-run/no-op branch before creating the shared client or calling `client.request`.
  - confidence: 0.92
- **[P2] Avoid starting Codex on every gateway startup:** `extensions/codex/openclaw.plugin.json:22`
  - body: Changing the manifest to `onStartup: true` loads this plugin as a gateway sidecar, and the registered service immediately calls `checkWatches`; `snapshot()` reads the Codex app-server before checking whether any watches exist. That makes ordinary gateway startup probe or start Codex by default, so keep activation opt-in or make the bridge lazy until an explicit route/watch needs it.
  - confidence: 0.86
- **[P3] Publish the bridge docs in Codex docs navigation:** `docs/codex-continuity-bridge.md:1`
  - body: The new docs page is added at the docs root but is not added to `docs/docs.json` navigation or linked from the existing Codex harness docs, so users browsing the published docs will not discover it. Move or link it under the Codex plugin docs and add the navigation entry with the rest of the Codex harness pages.
  - confidence: 0.77
