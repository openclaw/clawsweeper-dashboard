  - confidence: 0.94

  - body: `telegramDryRun` is checked only after the bridge has submitted `turn/steer` or `turn/start` to Codex, so a dry-run configuration can still mutate the target thread. Return before acquiring the client or dispatching the write.
  - confidence: 0.97
- **[P1] Normalize tagged Codex thread statuses:** `extensions/codex/src/continuity/app-server-reader.ts:81-85`
  - body: Current Codex `thread/list` returns `Thread.status` as a tagged object such as `{ type: "active" }`, not a string. The string-only read path maps real active/idle/system-error states to `unknown`, making active-thread display and watch decisions unreliable.
  - confidence: 0.93
- **[P2] Read goals from the Codex goals DB:** `extensions/codex/src/continuity/sqlite-reader.ts:81-83`
  - body: The SQLite fallback opens `state_5.sqlite` and joins `thread_goals` there, but current Codex stores goals in `goals_1.sqlite` and drops `thread_goals` from the state DB. On current installs this fallback fails instead of providing the promised stale-labeled thread data.
  - confidence: 0.92
- **[P1] Keep the continuity bridge opt-in:** `extensions/codex/openclaw.plugin.json:22-23`
  - body: Changing the bundled Codex plugin to `onStartup: true` while `codexBridge.enabled` defaults to true makes existing installs start the bridge and poll Codex without an explicit user choice. Leave activation/default enablement opt-in or include an approved upgrade path and proof.
  - confidence: 0.89
