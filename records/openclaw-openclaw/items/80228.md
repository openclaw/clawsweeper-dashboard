Full review comments:

- **[P1] Do not shadow existing Codex status and threads commands:** `extensions/codex/src/command-handlers.ts:191-193`
  - body: This calls the continuity handler before the established `status` and `threads` branches, so `/codex status` loses the current app-server probe output and `/codex threads` loses the current resume-oriented listing. Put bridge output behind bridge-specific subcommands, or only intercept new bridge-only verbs.
  - confidence: 0.94
- **[P1] Honor dry run before sending Codex writes:** `extensions/codex/src/continuity/bridge.ts:343-344`
  - body: `telegramDryRun` is checked only after the bridge has submitted `turn/steer` or `turn/start` to Codex, so a dry-run configuration can still mutate the target thread. Return before acquiring the client or sending the write, while recording any audit state without dispatching the request.
  - confidence: 0.97
- **[P1] Normalize the real app-server thread status:** `extensions/codex/src/continuity/app-server-reader.ts:81-85`
  - body: Current Codex `thread/list` returns `Thread.status` as a tagged object such as `{ type: "active" }`, not a string. The bridge's string-only read path maps real active/idle/system-error states to `unknown`, which makes active-thread display and watch decisions unreliable.
  - confidence: 0.93
- **[P2] Read goals from the Codex goals DB or remove the fallback:** `extensions/codex/src/continuity/sqlite-reader.ts:81-83`
  - body: The SQLite fallback opens `state_5.sqlite` and joins `thread_goals` there, but current Codex stores goals in `goals_1.sqlite`. On current installs this fallback fails instead of providing the stale-labeled thread data promised by the bridge.
  - confidence: 0.92
- **[P1] Keep the bridge opt-in on startup:** `extensions/codex/openclaw.plugin.json:22`
  - body: Changing the bundled Codex plugin to `onStartup: true` while `codexBridge.enabled` defaults to true makes existing installs start the bridge and poll Codex without an explicit user choice. Leave startup activation/default bridge enablement opt-in, or include a maintainer-approved upgrade path and proof for the background behavior.
  - confidence: 0.89
