## Summary

This should stay open: current main and the latest release still use a fixed Dream Diary narrative timeout and fixed detached narrative concurrency, while the supported memory-core config/docs do not expose a way to tune or disable only narrative generation while keeping dreaming enabled.

## What This Changes

Request separate memory-core Dream Diary narrative controls for timeout, concurrency, or narrative disablement while keeping the broader dreaming sweep enabled.

## Best Possible Solution

Decide the supported memory-core narrative config shape, then add plugin-owned controls that preserve the existing default behavior while allowing slow local setups to tune or disable only Dream Diary narrative generation.

## Reproduction Assessment

Unclear for the live timeout: the source proves a fixed 60-second narrative wait and no separate narrative controls, but I did not reproduce the LM Studio/qwen cron timeout on current main.

## Solution Assessment

Unclear until maintainers choose the config contract; a narrow memory-core narrative config with existing defaults is safer than globally raising the hardcoded timeout or disabling all dreaming.
