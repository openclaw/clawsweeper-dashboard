## Summary

Keep open: current main still has a source-level reproduction for the confusing no-delivery preview, and the status summary path still presents cron session-store rows as ordinary recent sessions instead of distinguishing cron history.

## What This Changes

The issue asks for cron jobs with `delivery.mode: none` to show no-delivery preview/status output and for status to distinguish old cron session rows from current isolated cron runs.

## Best Possible Solution

Short-circuit explicit no-delivery previews to clear no-delivery text, then label cron session rows as cron/history in status while preserving existing runtime delivery semantics and session retention.

## Reproduction Assessment

Yes, from source inspection. Explicit `delivery: { mode: "none" }` does not hit the preview `not requested` branch and can resolve `last`, while status summary maps cron session-store rows without a cron classification.

## Solution Assessment

Yes, a narrow presentation fix is the best path. Keep `resolveCronDeliveryPlan` preserving explicit target context for message-tool use, but make preview/status rendering reflect no fallback delivery and cron session history clearly.
