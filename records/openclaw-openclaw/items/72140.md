## Summary

Keep open. Current main avoids displaying stale `totalTokens` as current context, but it still does not show the latest checkpoint `tokensAfter` value when totals are stale; open PR #72285 remains the active implementation path.

## What This Changes

The issue asks shared status output to show the current post-compaction context size instead of a stale pre-compaction total after compaction.

## Best Possible Solution

Land or replace #72285 with a narrow shared formatter fix that keeps fresh totals authoritative, uses the newest checkpoint by `createdAt` only when totals are stale or unavailable, validates finite non-negative `tokensAfter`, and adds focused regression coverage plus a changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to call `buildStatusMessage()` with `totalTokensFresh: false`, a stale `totalTokens`, and a latest compaction checkpoint containing `tokensAfter`; current main ignores that checkpoint and renders unknown context instead of the post-compaction size.

## Solution Assessment

Yes in direction. The fix belongs in the shared status formatter, with fresh totals preferred and checkpoint fallback used only for stale or missing totals.
