## Summary

Keep open. Current main hides stale `totalTokens` rather than showing them as current context, but the shared status formatter still does not use compaction checkpoint `tokensAfter`, and open PR #72285 remains the active implementation path.

## What This Changes

The issue requests that `/status` and shared status cards show the current post-compaction context size instead of a stale pre-compaction total.

## Best Possible Solution

Land or replace PR #72285 with a narrow shared formatter fix that keeps fresh totals authoritative, selects the newest checkpoint by `createdAt`, uses finite non-negative `tokensAfter` only when current totals are stale or unavailable, and adds focused regression coverage plus a changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to call `buildStatusMessage()` with `totalTokensFresh: false`, a stale `totalTokens`, and a latest compaction checkpoint containing `tokensAfter`; current main renders unknown context instead of that post-compaction size.

## Solution Assessment

Yes in direction, but not complete on current main. The maintainable fix belongs in the shared status formatter using existing freshness and checkpoint state, with PR #72285 as the active path once its review feedback is addressed.
