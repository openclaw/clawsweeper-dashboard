## Summary

Keep open. Current main prevents stale `totalTokens` from being shown as current context, but the shared formatter still does not use compaction checkpoints to show the post-compaction size when totals are stale; open PR #72285 remains the active implementation path.

## What This Changes

The issue requests that shared status output show the current post-compaction context size instead of stale pre-compaction totals.

## Best Possible Solution

Land or replace #72285 with a narrow shared formatter fix that keeps fresh totals authoritative, selects the newest checkpoint by `createdAt`, uses finite non-negative `tokensAfter` only when current totals are stale or unavailable, and adds focused regression coverage plus a changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to call `buildStatusMessage()` with `totalTokensFresh: false`, a stale `totalTokens`, and a latest compaction checkpoint containing `tokensAfter`; current main ignores that checkpoint and renders unknown context.

## Solution Assessment

Yes in direction. The fix belongs in the shared status formatter using existing session freshness and checkpoint state, with the important detail that the newest checkpoint should be selected before validating optional `tokensAfter`.
