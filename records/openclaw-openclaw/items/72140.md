- `P2`: This is a normal-priority user-visible status correctness bug with a clear but limited shared formatter surface.
- `impact:session-state`: The issue concerns incorrect reporting of session context usage and freshness state after compaction.

## Summary

Keep open. Current main suppresses known-stale pre-compaction totals, but it still does not use the latest compaction checkpoint `tokensAfter` as the post-compaction context size, and the open formatter PR remains the active implementation path.

## What This Changes

The issue asks shared `/status` output and status cards to show current post-compaction context size instead of stale pre-compaction totals.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/72285 with a corrected shared formatter fix that selects the latest checkpoint first, uses finite non-negative `tokensAfter` only when totals are stale or unavailable, and adds focused regression coverage.

## Reproduction Assessment

Yes, source-level. Calling `buildStatusMessage()` with `totalTokensFresh: false`, stale totals, and a latest compaction checkpoint containing `tokensAfter` cannot render the checkpoint-backed post-compaction context size on current main.

## Solution Assessment

Yes in direction, but it is not merged yet. The narrow maintainable fix belongs in the shared status formatter and should preserve latest-checkpoint ordering before validating optional `tokensAfter`.
