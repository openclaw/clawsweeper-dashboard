## Summary

Keep open. Current main still does not use the latest compaction checkpoint `tokensAfter` for the shared status `Context` line when stored totals are stale, and the open formatter PR remains the active implementation path rather than a reason to close this issue.

## What This Changes

Make shared status rendering show the current post-compaction context size instead of stale pre-compaction totals after compaction.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/72285 with a corrected shared formatter fix that selects the latest checkpoint first, uses finite non-negative `tokensAfter` only from that checkpoint when totals are stale or unavailable, and adds focused regression coverage.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to call `buildStatusMessage()` with `totalTokensFresh: false`, stale `totalTokens`, and a latest compaction checkpoint containing `tokensAfter`; current main suppresses the stale total to unknown and does not use the checkpoint.

## Solution Assessment

Yes in direction, but not yet on main. The narrow maintainable fix belongs in the shared status formatter using existing freshness and checkpoint state, with the active PR repaired or replaced if needed.
