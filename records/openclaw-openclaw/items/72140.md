## Summary

Keep open. Current main suppresses stale session totals instead of showing them as current context, but the shared status formatter still does not fall back to the latest compaction checkpoint `tokensAfter`, and open PR #72285 remains the active implementation path.

## What This Changes

The issue asks for `/status` and shared status cards to show the current post-compaction context size rather than a stale pre-compaction total.

## Best Possible Solution

Land or replace PR #72285 with a narrow shared formatter fix that keeps fresh totals authoritative, selects the newest checkpoint by `createdAt`, uses finite non-negative `tokensAfter` only when totals are stale or unavailable, and adds focused regression coverage plus a changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to call `buildStatusMessage()` with `totalTokensFresh: false`, a stale `totalTokens`, and a latest compaction checkpoint containing `tokensAfter`; current main renders unknown context instead of that post-compaction size.

## Solution Assessment

Yes in direction, but current main is incomplete. The maintainable fix belongs in `src/status/status-message.ts` because that shared formatter drives `/status` and status cards across channels.
