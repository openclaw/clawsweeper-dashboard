## Summary

Keep open. Current main hides stale `totalTokens` instead of showing the latest checkpoint `tokensAfter`, so the user-visible post-compaction context size is still not implemented and PR #72285 remains the active implementation path.

## What This Changes

The issue requests that `/status` and shared status cards show the current post-compaction context size, with the pre-compaction peak kept as compaction metadata.

## Best Possible Solution

Review and land or replace PR #72285 with a shared formatter fallback that keeps fresh totals authoritative, uses the newest valid checkpoint `tokensAfter` only for stale or missing totals, and includes focused regression coverage plus a changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to call `buildStatusMessage()` with `totalTokensFresh: false`, stale `totalTokens`, and a latest compaction checkpoint containing `tokensAfter`; current main renders unknown context because the formatter never reads checkpoint tokens.

## Solution Assessment

Yes in direction, but not complete on current main. The narrow maintainable fix belongs in the shared status formatter rather than in any channel-specific status surface.
