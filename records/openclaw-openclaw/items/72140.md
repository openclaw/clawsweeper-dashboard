## Summary

Keep open. Current main still treats stale session totals as unavailable and does not recover the post-compaction size from checkpoints, while open PR #72285 remains the active implementation path.

## What This Changes

Make `/status` and shared status cards show the current post-compaction context size instead of stale pre-compaction totals.

## Best Possible Solution

Land or replace #72285 with a narrow shared formatter fix that selects the newest checkpoint first, uses its finite non-negative `tokensAfter` only when totals are stale or unavailable, preserves fresh totals, and adds regression coverage plus changelog.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to call `buildStatusMessage()` with `totalTokensFresh: false`, a stale `totalTokens`, and a latest compaction checkpoint containing `tokensAfter`; current main ignores that checkpoint and renders unknown context.

## Solution Assessment

Yes in direction. The fix belongs in the shared status formatter using existing freshness and checkpoint state, with the important detail that the newest checkpoint should be selected before validating optional `tokensAfter`.
