## Summary

Keep open. Current main avoids treating stale totals as fresh, but the shared status formatter still does not show the latest post-compaction checkpoint `tokensAfter`, and https://github.com/openclaw/openclaw/pull/72285 remains the active implementation path.

## What This Changes

`/status` and shared status cards should show the current post-compaction context size instead of stale pre-compaction totals.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/72285 with a corrected shared formatter fix that preserves fresh totals, selects the latest checkpoint before validating `tokensAfter`, and adds focused regression coverage.

## Reproduction Assessment

Yes. A high-confidence source-level path is to call `buildStatusMessage()` with `totalTokensFresh: false`, stale token totals, and a latest compaction checkpoint containing `tokensAfter`; current main renders unknown context instead of checkpoint-backed usage.

## Solution Assessment

Yes in direction, but not yet on main. The narrow maintainable fix belongs in the shared formatter using existing freshness and checkpoint state, with the active PR repaired or replaced if needed.
