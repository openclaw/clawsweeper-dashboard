## Summary

Keep open. Current main prevents stale `totalTokens` from being shown as current context, but the shared status formatter still does not use compaction checkpoint `tokensAfter`, and open PR #72285 remains the active implementation path.

## What This Changes

Request shared `/status` and status-card context usage to show current post-compaction token size instead of stale pre-compaction totals.

## Best Possible Solution

Land or replace #72285 with a narrow formatter change that keeps fresh totals authoritative, falls back to the newest valid checkpoint `tokensAfter` for stale totals, and adds focused regression coverage plus changelog if user-facing.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to call `buildStatusMessage()` with `totalTokensFresh: false`, a stale `totalTokens`, and a latest compaction checkpoint containing `tokensAfter`; current main never reads that checkpoint and renders unknown context instead.

## Solution Assessment

Yes in direction. The narrow maintainable solution belongs in the shared status formatter using existing freshness and checkpoint state, with #72285 as the current review target rather than a separate repair lane.
