## Summary

Keep open. Current main still does not use the latest compaction checkpoint `tokensAfter` in the shared status formatter, and the linked open PR is the active narrow fix path for the remaining behavior.

## What This Changes

The issue asks for `/status` and shared post-message status cards to show the current post-compaction context size instead of a stale pre-compaction total.

## Best Possible Solution

Review and land PR #72285, or an equivalent narrow shared formatter change that prefers fresh totals and falls back to the latest valid checkpoint `tokensAfter` only when displayed totals are stale or missing, with regression coverage in the status formatter tests.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to call `buildStatusMessage` with stale `totalTokens`, `totalTokensFresh: false`, and a latest compaction checkpoint containing `tokensAfter`; current main has no code path that reads that checkpoint for the Context line.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the shared status formatter using existing session checkpoint data, not in a WhatsApp-specific workaround or a broader session-store migration.
