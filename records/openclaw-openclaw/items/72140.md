## Summary

Keep open. Current main no longer presents a stale `totalTokens` value as current context when totals are marked stale, but it still does not show the available post-compaction `tokensAfter` value that this issue requests; open PR #72285 remains the active implementation path for https://github.com/openclaw/openclaw/issues/72140.

## What This Changes

The issue asks for shared `/status` output to report the current post-compaction context size instead of a stale pre-compaction total.

## Best Possible Solution

Land or replace PR #72285 with a narrow shared formatter fix that keeps fresh totals authoritative, falls back to the newest valid checkpoint `tokensAfter` only when totals are stale or unavailable, and adds focused regression coverage plus a changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to call `buildStatusMessage()` on current main with `totalTokensFresh: false`, a stale `totalTokens`, and a latest compaction checkpoint containing `tokensAfter`; the formatter renders unknown context instead of the post-compaction size.

## Solution Assessment

Yes in direction, but current main is incomplete. The narrow maintainable fix belongs in the shared status formatter, using existing freshness and checkpoint state while keeping fresh totals authoritative.
