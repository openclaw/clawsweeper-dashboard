## Summary

Keep open. Current main prevents stale `totalTokens` from being shown as current context, but the shared status formatter still renders unknown context instead of using the latest compaction checkpoint `tokensAfter`; PR #72285 remains the active implementation path.

## What This Changes

The issue requests that `/status` and shared status cards show the current post-compaction context size instead of stale pre-compaction totals.

## Best Possible Solution

Land or replace PR #72285 with a narrow shared formatter fix that keeps fresh totals authoritative, falls back to the newest finite checkpoint `tokensAfter` only when totals are stale or unavailable, and includes regression coverage plus a changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to call `buildStatusMessage()` with `totalTokensFresh: false`, a stale `totalTokens`, and a latest compaction checkpoint containing `tokensAfter`; current main renders unknown context rather than that post-compaction size.

## Solution Assessment

Yes in direction, but current main is incomplete. The narrow maintainable fix belongs in the shared status formatter using existing freshness state and checkpoint metadata rather than channel-specific status handling.
