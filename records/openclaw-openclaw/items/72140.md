## Summary

Keep open. Current main suppresses stale pre-compaction totals, but it still renders unknown context instead of the current post-compaction `tokensAfter` value when session totals are stale; open PR #72285 remains the active implementation path.

## What This Changes

Report asks `/status` and shared status cards to show post-compaction context size instead of stale pre-compaction token totals.

## Best Possible Solution

Land or replace #72285 with a narrow shared formatter fix: keep fresh totals authoritative, fall back to the newest valid checkpoint `tokensAfter` only when totals are stale or unavailable, and add focused regression coverage plus a changelog entry.

## Reproduction Assessment

Yes, source-level. Calling `buildStatusMessage()` with `totalTokensFresh: false`, stale `totalTokens`, and a latest compaction checkpoint containing `tokensAfter` follows the current-main path that ignores checkpoints and renders unknown context.

## Solution Assessment

Yes in direction, but not complete on current main. The maintainable fix belongs in the shared status formatter rather than channel-specific code, with #72285 as the active implementation path once reviewed or replaced.
