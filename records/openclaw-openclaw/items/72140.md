## Summary

Keep open. Current main avoids showing stale `totalTokens` as current context, but the shared status formatter still does not use compaction checkpoint `tokensAfter` when totals are stale, and open PR #72285 remains the active implementation path.

## What This Changes

Request `/status` and shared status cards to show current post-compaction context tokens instead of stale pre-compaction totals.

## Best Possible Solution

Land or replace PR #72285 with a narrow shared formatter fix that keeps fresh totals authoritative, falls back to the newest valid checkpoint `tokensAfter` only when totals are stale or unavailable, and adds focused regression coverage plus a changelog entry.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to call `buildStatusMessage()` with `totalTokensFresh: false`, stale `totalTokens`, and a latest compaction checkpoint containing `tokensAfter`; current main renders unknown context instead of the post-compaction size.

## Solution Assessment

Yes in direction, but not complete on current main. The narrow maintainable fix belongs in the shared status formatter using existing freshness and checkpoint state, with PR #72285 as the active path once review feedback is handled.
