## Summary

Keep open. Current main suppresses stale `totalTokens` so it no longer presents the pre-compaction total as fresh truth, but the shared formatter still does not fall back to the latest compaction checkpoint `tokensAfter`; https://github.com/openclaw/openclaw/pull/72285 remains the active implementation path.

## What This Changes

Request that the shared `/status` context line show the current post-compaction context size instead of stale pre-compaction totals.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/72285 with a corrected shared formatter fix that keeps fresh totals authoritative, selects the latest checkpoint first, uses finite non-negative `tokensAfter` only when totals are stale or unavailable, and adds focused regression coverage.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to call `buildStatusMessage()` with `totalTokensFresh: false`, stale token totals, and a latest compaction checkpoint containing `tokensAfter`; current main renders unknown context because the formatter never reads the checkpoint data.

## Solution Assessment

Yes in direction, but not yet on current main. The narrow maintainable solution belongs in the shared status formatter using existing freshness and checkpoint state, while preserving fresh totals as authoritative.
