## Summary

Keep open. Current main still hides stale pre-compaction totals when `totalTokensFresh` is false, but it does not use the latest compaction checkpoint `tokensAfter` as the post-compaction context size, and the related formatter PR remains the active implementation path.

## What This Changes

The issue asks for `/status` and shared status cards to show current post-compaction context size instead of stale pre-compaction totals.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/72285 with a corrected shared formatter fix that selects the latest checkpoint first, uses finite non-negative `tokensAfter` only when totals are stale or unavailable, and adds focused regression coverage.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to call `buildStatusMessage()` with `totalTokensFresh: false`, stale totals, and a latest compaction checkpoint containing `tokensAfter`; current main renders unknown context instead of the checkpoint post-compaction size.

## Solution Assessment

Yes in direction, but it is not on main yet. The narrow maintainable fix belongs in the shared status formatter, selecting the latest compaction checkpoint first and using its valid `tokensAfter` only when stored totals are stale or unavailable.
