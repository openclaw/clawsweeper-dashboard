## Summary

Keep open. Current main no longer displays known-stale totals as fresh context, but it still does not use the existing latest compaction checkpoint `tokensAfter` to show the current post-compaction context size; the linked formatter PR is still open and unmerged.

## What This Changes

The issue asks shared `/status` output and status cards to show the current post-compaction context size instead of stale pre-compaction totals.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/72285 with a focused shared formatter fix that preserves fresh totals, selects the latest checkpoint before validating `tokensAfter`, and adds regression coverage.
