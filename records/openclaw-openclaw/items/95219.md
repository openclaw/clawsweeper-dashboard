## Summary

Keep open: current main and the latest release still use exact aggregate tool-result shaving during prompt projection, so older model-facing tool-result content can change as new tool results push the aggregate over budget. The open fix candidate at https://github.com/openclaw/openclaw/pull/93186 directly targets this behavior, but it is not merged and currently needs rebase plus maintainer acceptance of the cache-stability tradeoff.

## What This Changes

The issue requests stable model-facing projections for historical tool results so adjacent turns preserve prompt-cache prefixes during long tool-heavy channel sessions.

## Best Possible Solution

Review, rebase, and land https://github.com/openclaw/openclaw/pull/93186 or an equivalent stable live-projection fix, then close this issue only after the fix reaches main.

## Reproduction Assessment

Yes at source level: current main rebuilds request-local tool-result projections from the evolving aggregate total and older-first reductions, so a focused test can compare the same historical tool result before and after appending tail results. I did not run a fresh live provider cache trace in this read-only review.

## Solution Assessment

Yes, conditionally: the open PR’s live-projection/estimate boundary is a narrow fix direction, but it still needs rebase and maintainer acceptance before it can be treated as the solution.
