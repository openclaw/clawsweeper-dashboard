## Summary

Keep open: the fresh report is source-backed, and current main still has tail-based transcript append paths where a delayed inbound user mirror can be parented after an immediate delivery-mirror assistant entry.

## What This Changes

The issue asks OpenClaw to preserve chat turn causality when delayed inbound user transcript mirroring races immediate visible assistant delivery mirroring.

## Best Possible Solution

Add a shared causal ordering guard so same-rollout delivery mirrors wait for or attach after the triggering inbound user turn, with a regression for delayed user mirroring and immediate assistant delivery mirroring.

## Reproduction Assessment

No high-confidence executable repro was established in this read-only pass. Source inspection and the reporter's sanitized JSONL show a plausible current-main path: delivery mirrors and delayed user mirrors both append to the current transcript tail.

## Solution Assessment

Yes, a narrow transcript/session ordering repair is the best direction. The fix should make the inbound turn's causal anchor authoritative for same-rollout delivery mirrors rather than adding channel-specific policy.
