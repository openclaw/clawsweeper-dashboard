## Summary

Keep open: current main still appends delivery mirrors through the same tail-based transcript path, and there is no regression covering delayed inbound user persistence followed by immediate visible assistant mirroring.

## What This Changes

The issue reports that delayed inbound chat transcript mirroring can append the user turn after an assistant delivery mirror, inverting transcript order and parentId.

## Best Possible Solution

Add a core transcript-ordering guard that anchors or awaits the inbound user turn before same-rollout assistant delivery mirrors, with a regression for delayed user persistence and immediate mirror append.

## Reproduction Assessment

No high-confidence executable repro was established in this read-only review. Source inspection shows the vulnerable path: delivery mirrors append to the transcript tail, and delayed user persistence would also append using the current leaf parent.

## Solution Assessment

Yes, the best fix is a narrow core transcript/session ordering repair rather than channel-specific policy. The fix should make the inbound turn’s causal anchor authoritative for delivery mirrors from the same rollout and prove parentId/order with a focused regression test.
