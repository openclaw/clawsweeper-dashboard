## Summary

Current main already fixes the reported native chat approval loop. The merged fix keeps native chat exec approvals inline for Telegram and other bundled native approval channels, returns approval denials as foreground tool results, preserves explicit async follow-up modes, and is covered by focused regression tests; it is currently main-only and not in v2026.6.8.

## What This Changes

The issue asks native chat `/approve` for gateway exec approvals to resume the originating agent turn without requiring a manual follow-up message that can mint a new approval id.

## Best Possible Solution

Keep the current main inline native approval implementation and let it ship in the next release; reopen or file a fresh issue only with a repro on a build containing 1bfa2787b503515edc507b8019e511aa93974c74.

## Reproduction Assessment

## Solution Assessment

Yes. Reusing the existing inline approval wait path for bundled native approval clients is narrower than adding another post-hoc session injection path, and it preserves explicit follow-up and non-native fallback behavior.
