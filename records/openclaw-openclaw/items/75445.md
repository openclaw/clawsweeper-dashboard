## Summary

Keep open. The linked embedded-reply SecretRef bug is valid, but this PR still has correctness blockers: the Telegram fallbacks avoid the crash by discarding explicit prompt/reaction configuration, and the Discord portion is stale against current main's inspect-based discovery behavior.

## What This Changes

The PR wraps Discord and Telegram message-action discovery plus Telegram prompt capability/reaction guidance around unresolved SecretRef errors, adds regression tests, and adds a changelog entry for the embedded reply crash tracked by #75433.

## Best Possible Solution

Land a rebased plugin-owned fix that keeps runtime send/probe SecretRef resolution strict, uses config-only inspection or merged account config for best-effort Telegram prompt/action discovery, preserves explicit inline-button and reaction settings, and leaves current main's Discord inspect-based discovery intact.

## Reproduction Assessment

Yes. The issue has a high-confidence source-level reproduction: embedded prompt prep passes raw config into Telegram prompt/reaction hooks, those helpers call resolveTelegramAccount, and non-env SecretRefs throw in strict token resolution before model output.

## Solution Assessment

No, not as submitted. Catching only optional discovery errors is the right layer, but the Telegram fallbacks should preserve merged config state, and the Discord part should be rebased onto current main's inspect-based implementation instead of replacing it with empty discovery.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve Discord actions during SecretRef discovery:** `extensions/discord/src/channel-actions.ts:50`
  - body: Current main already inspects Discord accounts without resolving tokens, and its tests expect unresolved SecretRef-backed accounts to still advertise configured actions. Returning null here would regress that behavior after rebase, leaving prompt prep with no Discord actions even though the account is configured.
  - confidence: 0.86
- **[P2] Preserve disabled inline-button config during fallback:** `extensions/telegram/src/inline-buttons.ts:75`
  - body: When resolveTelegramAccount throws for an unresolved SecretRef, this returns the default scope instead of the configured capabilities. A SecretRef-backed account with capabilities.inlineButtons set to "off" will still advertise inlineButtons in prompt capabilities, so the fallback should inspect merged config without resolving the token or otherwise preserve "off".
  - confidence: 0.9
- **[P2] Preserve configured reaction level during fallback:** `extensions/telegram/src/reaction-level.ts:33-37`
  - body: The unresolved-SecretRef fallback drops account.config.reactionLevel and resolves the default minimal level. For a SecretRef-backed Telegram account configured with reactionLevel: "off", prompt prep will still emit reaction guidance, so the fallback needs to read the merged config state without resolving the token.
  - confidence: 0.9
