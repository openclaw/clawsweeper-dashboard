## Summary

Keep open: the linked bug is real and the PR is close, but the current head still drops explicit Telegram disabled settings when falling back from unresolved SecretRefs, so prompt prep can advertise inline buttons or reaction guidance that the config turned off.

## What This Changes

The PR adds Telegram unresolved-SecretRef fallbacks for channel-action, inline-button, and reaction-guidance discovery, adds Discord/Telegram regression tests, and updates the changelog for #75433.

## Best Possible Solution

Land a plugin-owned Telegram discovery fix that keeps runtime SecretRef resolution strict, uses read-only inspection or merged config for prompt discovery, preserves explicit disabled capability settings, and leaves the linked issue open until the PR merges.

## Reproduction Assessment

Yes by source inspection. Current main reaches strict Telegram token resolution from prompt capability/reaction discovery, and the PR findings are reproducible from the diff with a non-env SecretRef plus `capabilities.inlineButtons: "off"` or `reactionLevel: "off"`.

## Solution Assessment

No, not as submitted. Catching unresolved SecretRefs in discovery is the right layer, but the fallback should preserve merged Telegram config instead of substituting enabled defaults.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Preserve disabled inline-button config on fallback:** `extensions/telegram/src/inline-buttons.ts:74`
  - body: When `resolveTelegramAccount` throws for an unresolved SecretRef, this returns `DEFAULT_INLINE_BUTTONS_SCOPE`. Current main defines that default as `allowlist`, and the prompt hook treats anything except `off` as inline-button support, so a SecretRef-backed account with `capabilities.inlineButtons: "off"` will still advertise inline buttons. Use read-only inspection or merged config so explicit `off` survives the fallback.
  - confidence: 0.9
- **[P2] Preserve disabled reaction config on fallback:** `extensions/telegram/src/reaction-level.ts:33-37`
  - body: This unresolved-SecretRef fallback calls `resolveReactionLevel` with `value: undefined`, which produces the `minimal` default. For a SecretRef-backed Telegram account configured with `reactionLevel: "off"`, prompt prep will still emit agent reaction guidance, so the fallback should read the merged configured reactionLevel without resolving the credential.
  - confidence: 0.89
