## Summary

Keep open: the linked bug is real and the PR is directionally useful, but the latest patch still drops explicit Telegram capability settings when it falls back from unresolved SecretRefs.

## What This Changes

The PR adds Telegram unresolved-SecretRef fallbacks for channel-action, inline-button, and reaction-guidance discovery, adds Discord/Telegram regression tests, and updates the changelog.

## Best Possible Solution

Use a token-free Telegram account/config inspection path for best-effort prompt discovery so unresolved SecretRefs do not crash and explicit inlineButtons/reactionLevel settings remain authoritative, while leaving runtime send and probe SecretRef resolution strict.

## Reproduction Assessment

Yes. Source inspection of current main shows embedded prompt prep calls Telegram agentPrompt helpers that resolve credentials from raw config; the PR finding follows by combining a non-env SecretRef with explicit inlineButtons or reactionLevel off config.

## Solution Assessment

No, not as submitted. Catching unresolved SecretRefs in prompt discovery is the right layer, but substituting enabled defaults is not the narrowest correct fix because it ignores explicit Telegram disabled settings.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Preserve disabled inline buttons on fallback:** `extensions/telegram/src/inline-buttons.ts:74`
  - body: When resolveTelegramAccount throws for an unresolved SecretRef, this returns DEFAULT_INLINE_BUTTONS_SCOPE. That default is allowlist, and the prompt path treats anything except off as inlineButtons enabled, so a SecretRef-backed account configured with capabilities.inlineButtons: "off" will still advertise inline buttons. Read the merged capability config without resolving the token and preserve explicit off.
  - confidence: 0.9
- **[P2] Preserve disabled reaction guidance on fallback:** `extensions/telegram/src/reaction-level.ts:33-37`
  - body: This fallback calls resolveReactionLevel with value undefined, which produces the minimal default. For a SecretRef-backed Telegram account configured with reactionLevel: "off" or "ack", prompt prep will still emit agent reaction guidance, contradicting the configured reaction policy. Use the merged configured reactionLevel in the fallback path.
  - confidence: 0.89
