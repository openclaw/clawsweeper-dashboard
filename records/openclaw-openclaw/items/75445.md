## Summary

Keep open: this is the right bug to fix, but the latest PR head is not mergeable because the Telegram fallbacks can advertise explicitly disabled inline buttons and reactions when credentials are unresolved.

## What This Changes

The PR adds unresolved-SecretRef tolerance to Telegram message-action, inline-button, and reaction prompt discovery, adds Discord/Telegram regression tests, and updates the changelog.

## Best Possible Solution

Use a plugin-local read-only account inspection or merge path for prompt discovery so unresolved credentials do not crash and explicitly disabled Telegram features stay disabled.

## Reproduction Assessment

Yes. Source inspection shows current main prompt prep reaches Telegram helpers that resolve account credentials, and PR-head source shows a SecretRef-backed config with capabilities.inlineButtons or reactionLevel set to off would be converted to enabled defaults.

## Solution Assessment

No. Catching unresolved SecretRefs in plugin discovery is the right layer, but the fallback should preserve merged Telegram capability and reaction settings instead of substituting allowlist/minimal defaults.
