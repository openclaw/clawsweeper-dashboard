## Summary

Keep open: the linked crash is valid and this PR is in the right plugin-owned area, but the current head still turns Telegram inline-button and reaction guidance defaults back on when an unresolved SecretRef prevents token resolution.

## What This Changes

The PR adds best-effort unresolved-SecretRef handling for Discord/Telegram message-action and Telegram prompt/reaction discovery, with regression tests and a changelog entry.

## Best Possible Solution

Keep the discovery fix in the Telegram plugin, but derive fallback inline-button and reaction settings from merged or inspected Telegram config without resolving non-env SecretRefs, while leaving runtime send/probe paths strict.

## Reproduction Assessment

Yes. Source inspection shows current main prompt prep reaches Telegram helpers that call resolveTelegramAccount on raw config, and the PR-specific regressions follow from combining an unresolved exec/file SecretRef with capabilities.inlineButtons or reactionLevel set to off.

## Solution Assessment

No. Catching unresolved SecretRefs in discovery is the right layer, but substituting default enabled behavior is not the narrowest safe fix because it drops explicit disabled Telegram configuration.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

  - body: When resolveTelegramAccount fails on an unresolved SecretRef, this returns DEFAULT_INLINE_BUTTONS_SCOPE, which is treated as inline buttons enabled. A SecretRef-backed account with capabilities.inlineButtons: "off" will still advertise inlineButtons in prompt prep, so use merged/inspected config or a conservative off fallback without resolving the token.
  - confidence: 0.9

For a SecretRef-backed Telegram account configured with reactionLevel: "off", prompt prep will still tell the model to use reactions; preserve the merged configured reactionLevel without resolving the credential.
  - confidence: 0.89
