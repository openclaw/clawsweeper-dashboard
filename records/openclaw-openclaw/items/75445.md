## Summary

Keep open. The linked Telegram/Discord SecretRef discovery bug is real on current main, but this PR needs changes before merge because its Telegram fallbacks drop existing operator configuration and the branch overlaps a newer Discord discovery refactor already on main.

## What This Changes

The PR wraps Discord and Telegram message-action and prompt-discovery paths to tolerate unresolved SecretRef credentials, adds focused regression tests, and adds a changelog entry for the embedded reply crash.

## Best Possible Solution

Land a rebased plugin-owned fix that keeps runtime SecretRef resolution strict, uses config-only inspection for best-effort prompt/action discovery, preserves explicit Telegram inline-button and reaction settings when credentials are unresolved, and keeps the current-main Discord inspect-based behavior.

## Reproduction Assessment

Yes. On current main, a Telegram config with a non-env SecretRef botToken can reproduce at source level by calling resolveChannelPromptCapabilities or resolveChannelReactionGuidance for telegram; both reach resolveTelegramAccount and strict non-env SecretRef resolution before model output.

## Solution Assessment

No, not as submitted. The layer is right, but the Telegram fallback should preserve merged config state instead of replacing disabled inline-button or reaction settings with defaults; the Discord part also needs rebase alignment with current main's inspect-based discovery.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve disabled inline-button config during fallback:** `extensions/telegram/src/inline-buttons.ts:75`
  - body: When `resolveTelegramAccount` throws for an unresolved SecretRef, this returns `DEFAULT_INLINE_BUTTONS_SCOPE` instead of the configured capabilities. A SecretRef-backed account with `capabilities.inlineButtons: "off"` will still advertise `inlineButtons` because `messageToolCapabilities` treats any non-`off` scope as enabled. Use a config-only merge/inspection path, or return `off` for unavailable discovery.
  - confidence: 0.88
- **[P2] Preserve configured reaction level during fallback:** `extensions/telegram/src/reaction-level.ts:33-37`
  - body: The unresolved-SecretRef fallback drops `account.config.reactionLevel` and always resolves the default minimal level. For a SecretRef-backed Telegram account configured with `reactionLevel: "off"`, prompt prep will still emit reaction guidance, so the fallback should read the merged config without resolving the token or otherwise preserve the configured level.
  - confidence: 0.87
