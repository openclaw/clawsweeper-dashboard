## Summary

The PR is useful and mostly aimed at the right layer, but it should stay open because one Telegram reaction-guidance fallback still over-advertises reactions and the external PR does not include after-fix real behavior proof.

## What This Changes

The PR adds Discord/Telegram unresolved SecretRef tolerance in message-action discovery, Telegram prompt discovery, regression tests, and a changelog entry.

## Best Possible Solution

Land plugin-owned discovery hardening that avoids raw SecretRef resolution during prompt prep while preserving disabled Telegram inline/reaction config and leaving runtime send/probe paths strict.

## Reproduction Assessment

Yes. Source inspection shows current main reaches Telegram discovery helpers that resolve raw SecretRef-backed credentials before runtime snapshot resolution, and the PR-specific reaction regression follows from combining an unresolved SecretRef with `reactionLevel: "off"`.

## Solution Assessment

No. Catching unresolved SecretRefs in discovery is the right layer, but the reaction fallback should preserve inspected or merged Telegram config, or conservatively return off, instead of substituting the minimal default.
