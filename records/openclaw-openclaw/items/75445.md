## Summary

This PR is useful but not merge-ready. It addresses the SecretRef crash path, but it would drop existing Discord inspect-based discovery from current main and the Telegram fallbacks still ignore configured inline-button and reaction settings when credentials are unresolved.

## What This Changes

The PR catches unresolved SecretRef errors during Discord/Telegram message-action and Telegram prompt-capability discovery, adds regression tests, and adds a changelog entry for #75433.

## Best Possible Solution

Land a plugin-owned fix that keeps strict runtime SecretRef resolution, uses read-only config inspection for discovery-only paths, and preserves configured Telegram capability/reaction settings.

## Reproduction Assessment

Yes. The source path is clear: embedded prompt prep calls Telegram prompt capability and reaction guidance hooks, and those currently reach resolveTelegramAccount before the active runtime snapshot has resolved non-env SecretRefs.

## Solution Assessment

No, not as submitted. The layer is right, but the patch should inspect merged config without resolving credentials and should preserve current main's Discord inspect-based discovery.
