## Summary

Keep open. The PR targets a real Telegram/Discord SecretRef prompt-discovery crash, but the latest head still has blocking correctness regressions: Telegram fallbacks discard explicit inline-button and reaction settings, and the Discord change is behind current main's inspect-based discovery that already preserves configured actions for unresolved SecretRefs.

## What This Changes

The PR wraps Discord and Telegram message-action and Telegram prompt-capability/reaction discovery around unresolved SecretRef errors, adds focused regression tests, and adds a changelog entry for the embedded reply crash tracked by #75433.

## Best Possible Solution

Land a revised plugin-owned fix that keeps runtime SecretRef resolution strict, uses config-only inspection for best-effort prompt/action discovery, preserves explicit Telegram inline-button and reaction settings when credentials are unresolved, and keeps current main's Discord inspect-based action discovery behavior.

## Reproduction Assessment

Yes. Current main can be reproduced at source level by calling Telegram prompt capability or reaction guidance discovery with a non-env SecretRef botToken; the call path reaches resolveTelegramAccount and strict token resolution before model output.

## Solution Assessment

No, not as submitted. The layer is right, but the submitted fallback replaces configured Telegram states with defaults and regresses Discord from current main's inspect-based discovery; a config-only merge/inspection path is the safer narrow fix.
