## Summary

The PR is a plausible fix for a real current-main bug, but it is not merge-ready: the latest diff still aborts unscoped Telegram action discovery for all accounts when one account has an unresolved SecretRef, and the supplied proof is resolver-level Vitest output rather than a real embedded/channel run.

## What This Changes

The PR adds unresolved SecretRef fallbacks and regression tests for Telegram channel-action, inline-button, and reaction prompt discovery, adds a Discord discovery regression test, and updates the changelog.

## Best Possible Solution

Keep the plugin-owned SecretRef tolerance, but implement Telegram action discovery with per-account degradation so unresolved accounts are skipped while healthy accounts remain advertised, then require live embedded/channel proof before merge.

## Reproduction Assessment

Yes, by source inspection and the linked issue logs: current main's embedded prompt-prep path reaches Telegram discovery helpers that resolve raw non-env SecretRefs before model output. I did not run a live SecretRef-backed Telegram or Discord reproduction in this read-only review.

## Solution Assessment

No, not yet. Catching unresolved SecretRefs in plugin discovery is the right layer and the inline/reaction fallbacks are directionally correct, but unscoped Telegram action discovery still needs per-account degradation and real runtime proof.
