## Summary

Keep open: the branch addresses the right SecretRef crash class, but the latest diff still defaults Telegram prompt discovery to enabled inline buttons and minimal reactions when explicit config may have disabled them.

## What This Changes

The PR adds Telegram unresolved-SecretRef fallbacks for channel-action, inline-button, and reaction-guidance discovery, adds Discord/Telegram regression tests, and updates the changelog for #75433.

## Best Possible Solution

Land a plugin-owned fix that keeps runtime SecretRef resolution strict, uses inspection or merged config for best-effort Telegram prompt discovery, preserves explicit Telegram disabled capability settings, and keeps the linked bug open until the fix merges.

## Reproduction Assessment

Yes. Source inspection shows current main prompt prep reaches Telegram helpers that call strict resolveTelegramAccount on raw config, and the PR findings reproduce by combining an unresolved exec/file SecretRef with capabilities.inlineButtons or reactionLevel set to off.

## Solution Assessment

No, not as submitted. Catching unresolved SecretRefs in discovery is the right layer, but the fallback should inspect merged Telegram config and preserve configured off states instead of substituting enabled defaults.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Preserve disabled inline-button config on fallback:** `extensions/telegram/src/inline-buttons.ts:74`
  - body: When `resolveTelegramAccount` throws for an unresolved SecretRef, this returns `DEFAULT_INLINE_BUTTONS_SCOPE`; current main defines that default as `allowlist`, and `isTelegramInlineButtonsEnabled` treats anything except `off` as enabled. A SecretRef-backed account with `capabilities.inlineButtons: "off"` will still advertise inline buttons during prompt prep, so read the merged/inspected config without resolving the token and preserve the explicit off state.
  - confidence: 0.9
- **[P2] Preserve disabled reaction config on fallback:** `extensions/telegram/src/reaction-level.ts:33-37`
  - body: This unresolved-SecretRef fallback calls `resolveReactionLevel` with `value: undefined`, which produces the minimal default. For a SecretRef-backed Telegram account configured with `reactionLevel: "off"`, prompt prep will still emit agent reaction guidance, so the fallback should use the merged configured reactionLevel without resolving the credential.
  - confidence: 0.89
