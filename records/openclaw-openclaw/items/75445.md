## Summary

Keep open. The PR targets a real current-main Telegram/Discord SecretRef prompt-prep crash path, but the current head still over-advertises Telegram inline-button and reaction capabilities when explicitly disabled settings are combined with unresolved SecretRefs.

## What This Changes

The branch adds Telegram unresolved-SecretRef catch/fallbacks in action, inline-button, and reaction discovery, adds Discord/Telegram regression tests, and updates the changelog for the linked embedded prompt-prep crash.

## Best Possible Solution

Use Telegram's plugin-owned inspection or merged-config path for discovery so unresolved SecretRefs do not crash prompt prep while explicit inline-button and reaction-level settings remain honored.

## Reproduction Assessment

Yes for source-level reproduction: current main prompt prep reaches Telegram helpers that call resolveTelegramAccount and resolveTelegramToken on raw config, while the linked issue provides matching unresolved-SecretRef logs before model output. I did not execute the repro because this review is read-only.

## Solution Assessment

No. Catching unresolved SecretRefs in discovery is the right layer, but returning default enabled/minimal values is not the narrowest maintainable fix because it ignores explicit disabled Telegram capability settings.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Return conservative scope on unresolved SecretRef:** `extensions/telegram/src/inline-buttons.ts:74`
  - body: When resolveTelegramAccount throws on an unresolved SecretRef, this returns DEFAULT_INLINE_BUTTONS_SCOPE, which is allowlist and is treated as inline buttons enabled by prompt prep. A SecretRef-backed account with capabilities.inlineButtons: "off" will still advertise inlineButtons, so preserve the merged/inspected config or fall back to "off" when the configured scope cannot be read safely.
  - confidence: 0.92
- **[P2] Preserve disabled reaction level during fallback:** `extensions/telegram/src/reaction-level.ts:33-37`
  - body: This fallback passes value: undefined, which resolves to minimal and adds Telegram reaction guidance. A SecretRef-backed account configured with reactionLevel: "off" or "ack" can still prompt the model to react, so preserve the merged reactionLevel without resolving the token or fall back to no agent guidance.
  - confidence: 0.9
