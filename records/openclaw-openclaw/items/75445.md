## Summary

Keep open. The PR targets a valid source-reproducible bug, but the current head still has blocking Telegram correctness issues and only supplies resolver-test proof, not real embedded Telegram/Discord behavior proof.

## What This Changes

The PR adds fail-soft handling for unresolved SecretRefs in Discord/Telegram channel-action discovery and Telegram prompt capability/reaction guidance, with regression tests and a changelog entry.

## Best Possible Solution

Keep the PR open and repair it so Telegram discovery uses token-free per-account inspection while prompt helpers honor configured capabilities/reaction levels without resolving tokens, then require redacted real embedded-run proof.

## Reproduction Assessment

Yes, source-level. Current main passes raw params.config into Telegram prompt capability and reaction hooks before model output, and those hooks reach resolveTelegramAccount/resolveTelegramToken where non-env SecretRefs throw.

## Solution Assessment

No. Fail-soft plugin discovery is the right layer, but this branch still drops healthy Telegram accounts when one account is unresolved and can advertise reaction guidance despite an explicit off setting.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P2] Skip unresolved accounts during discovery:** `extensions/telegram/src/channel-actions.ts:76-77`
  - body: `listEnabledTelegramAccounts(cfg)` resolves every enabled account before filtering, so a single exec/file-backed account throws into this catch and returns `null` for the entire unscoped Telegram discovery surface. In multi-account installs that hides healthy env/plain accounts from prompt prep; degrade per account instead of aborting channel-wide discovery.
  - confidence: 0.94
- **[P2] Honor disabled reaction guidance on unresolved tokens:** `extensions/telegram/src/reaction-level.ts:32-37`
  - body: When the token is an unresolved SecretRef, this fallback returns the default minimal reaction level before reading account config. A config with `reactionLevel: "off"` will still prompt the agent to use Telegram reactions; read the reaction config without resolving the token or fail closed to `off` for unresolved credentials.
  - confidence: 0.88
