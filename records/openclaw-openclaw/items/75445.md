## Summary

Keep this PR open: it addresses a real SecretRef prompt-prep crash and the Telegram inline-button fallback is now conservative, but the current channel-actions diff still aborts unscoped Telegram discovery for all accounts when any one enabled account has an unresolved SecretRef. The supplied proof is also resolver-level Vitest output only, so the external PR still needs real embedded/channel behavior proof before merge.

## What This Changes

The branch adds Telegram unresolved-SecretRef fallbacks for channel-action, inline-button, and reaction prompt discovery, adds Discord/Telegram regression tests, and updates the changelog.

## Best Possible Solution

Keep the plugin-owned SecretRef tolerance, but degrade Telegram action discovery per account so unresolved accounts are skipped while healthy accounts still advertise actions, then require real embedded/channel proof before merge.

## Reproduction Assessment

Yes by source inspection. Current main's embedded prompt-prep path calls Telegram prompt/action discovery helpers with raw config, and those helpers can reach strict non-env SecretRef resolution before model output; I did not run a live reproduction in this read-only review.

## Solution Assessment

No. Catching unresolved SecretRefs in plugin discovery is the right layer and the conservative inline/reaction fallbacks are directionally correct, but the unscoped Telegram channel-actions fallback is too broad because it hides healthy accounts.
