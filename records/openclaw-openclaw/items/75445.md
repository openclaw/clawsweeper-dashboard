## Summary

Keep open: the PR addresses a valid SecretRef prompt-prep crash, but the latest diff still drops all Telegram action discovery when one account has an unresolved SecretRef and it still lacks required real behavior proof from an after-fix embedded reply run.

## What This Changes

The PR adds Telegram unresolved-SecretRef fallbacks for channel-action, inline-button, and reaction discovery, adds Discord/Telegram regression tests, and updates the changelog.

## Best Possible Solution

Keep the plugin-owned discovery hardening, but make Telegram discovery degrade per account so valid accounts still advertise actions while unresolved accounts are skipped or inspected conservatively, then require real embedded-run proof before merge.

## Reproduction Assessment

Yes, by source inspection: current main reaches Telegram prompt/action discovery helpers that resolve raw non-env SecretRef credentials before runtime snapshot resolution. I did not run a live reproduction because this review was constrained to read-only inspection.

## Solution Assessment

No. Catching unresolved SecretRefs in discovery is the right layer, but returning null for the whole Telegram discovery pass is too broad for multi-account configs; per-account filtering or inspection is the narrower maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Skip unresolved accounts before returning empty discovery:** `extensions/telegram/src/channel-actions.ts:74-79`

In a multi-account setup, that hides actions from other enabled accounts with usable env/plain tokens, so discovery should filter or inspect the failing account instead of disabling all Telegram actions.
