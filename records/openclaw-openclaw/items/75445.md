## Summary

Keep open: the PR targets a real SecretRef prompt-prep crash, but the current patch still over-advertises Telegram inline-button and reaction capabilities when unresolved credentials hide config that explicitly disables them.

## What This Changes

The PR catches unresolved Telegram SecretRef failures in message-action, inline-button, and reaction-guidance discovery, adds Discord/Telegram regression coverage, and updates the changelog.

## Best Possible Solution

Use Telegram's read-only inspection or merged-config path for discovery so unresolved SecretRefs do not crash prompt prep and configured inlineButtons/reactionLevel settings are preserved, while runtime send/probe paths stay strict.

## Reproduction Assessment

Yes. Source inspection shows current main prompt prep reaches Telegram helpers that call resolveTelegramAccount and strict non-env SecretRef resolution, and the linked issue logs match that chain; I did not run tests because this review is read-only.

## Solution Assessment

No. Catching unresolved SecretRefs in plugin-owned discovery is the right layer, but the fallback should inspect or preserve merged Telegram config instead of defaulting disabled capabilities on.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Return conservative inline-button scope:** `extensions/telegram/src/inline-buttons.ts:74`
  - body: When resolveTelegramAccount throws for an unresolved SecretRef, this returns DEFAULT_INLINE_BUTTONS_SCOPE, which prompt prep treats as inline-buttons enabled. A SecretRef-backed account with capabilities.inlineButtons: "off" will still advertise inlineButtons, so preserve the configured scope without resolving the token or fall back to "off".
  - confidence: 0.9
- **[P2] Preserve disabled reaction config:** `extensions/telegram/src/reaction-level.ts:33-37`
  - body: This fallback resolves reactionLevel with value: undefined, producing the minimal agent-reaction guidance. For a SecretRef-backed Telegram account configured with reactionLevel: "off", prompt prep will still tell the model to use reactions; preserve the merged configured reactionLevel without resolving the credential.
  - confidence: 0.89
