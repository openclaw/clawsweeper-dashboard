## Summary

Keep this PR open: the main SecretRef prompt-discovery direction is useful and the latest branch fixes the earlier per-account and reaction-config problems, but it still advertises Telegram message actions for token-file credentials that current runtime resolution treats as unavailable.

## What This Changes

The PR rewrites Telegram prompt-time action, inline-button, and reaction discovery to use read-only account inspection for unresolved SecretRef tokens, adds focused Telegram regression tests, and includes Codex/ACP troubleshooting docs updates.

## Best Possible Solution

Keep the read-only SecretRef-tolerant discovery path, but gate advertised Telegram actions on credential availability for token-file/env failures while preserving configured non-secret behavior for unresolved SecretRefs.

## Reproduction Assessment

Yes. Source inspection shows current main resolves Telegram prompt discovery through strict token paths, and the PR head changes that path; the remaining regression is source-reproducible because tokenFile configured_unavailable survives the new discovery filters while runtime token resolution returns source none.

## Solution Assessment

No, not yet. The read-only inspector is a maintainable direction for unresolved SecretRefs, but the current filter should distinguish unresolved SecretRef config from token-file/env credentials that are actually unavailable.
