## Summary

Keep open. The Telegram fix is still useful, but the current branch still drops all Telegram action discovery when any enabled account has an unresolved SecretRef, and the supplied proof is test output rather than a real embedded channel run.

## What This Changes

The branch adds Discord/Telegram SecretRef-tolerant discovery fallbacks, Telegram inline-button and reaction fallback handling, regression tests, and a changelog entry.

## Best Possible Solution

Keep the plugin-owned fail-soft discovery approach, but degrade per Telegram account or capability so healthy accounts still advertise actions, then require real SecretRef-backed embedded channel proof before landing.

## Reproduction Assessment

Yes. Current source shows embedded prompt prep can reach Telegram prompt capability and reaction hooks with raw config, and those helpers still resolve accounts into strict non-env SecretRef token resolution; I did not run a live repro because this review is read-only.

## Solution Assessment

No. Catching unresolved SecretRefs in optional plugin discovery is the right layer, but the branch still disables all Telegram action discovery when one account is unresolved and still needs real runtime proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Skip only the unresolved Telegram account:** `extensions/telegram/src/channel-actions.ts:77`
