## Summary

Keep open. The reported bug is valid and this PR is pointed at the right plugin-owned area, but the head is dirty against current main, would replace current Discord inspect-based discovery with an empty fallback, and still drops configured Telegram prompt behavior on unresolved SecretRef credentials.

## What This Changes

The PR catches unresolved SecretRef errors in Discord and Telegram message-action discovery, adds Telegram prompt inline-button and reaction fallbacks, adds focused regression tests, and adds a changelog entry for #75433.

## Best Possible Solution

Land a rebased plugin-owned fix that keeps strict runtime SecretRef resolution, uses read-only inspection for best-effort discovery, preserves configured Telegram inline-button/reaction settings, and keeps the current Discord inspect-based behavior from #75424.

## Reproduction Assessment

Yes. On current main, the source path is clear: embedded prompt prep calls collectRuntimeChannelCapabilities and resolveChannelReactionGuidance, Telegram prompt hooks call resolveTelegramInlineButtonsScope/resolveTelegramReactionLevel, and those helpers reach strict resolveTelegramAccount token resolution for non-env SecretRefs.

## Solution Assessment

No, not as submitted. The fix belongs in the plugin discovery/prompt layer, but the PR should be rebased onto current main and should inspect merged account config rather than replacing unresolved Telegram credentials with defaults or empty Discord discovery.

- **[P2] Preserve Discord inspect-based discovery:** `extensions/discord/src/channel-actions.ts:48-49`
  - body: Current main already avoids this crash for Discord by inspecting configured accounts without resolving tokens, and its tests assert unresolved SecretRef credentials still describe configured actions.   - confidence: 0.9

A SecretRef-backed account with capabilities.inlineButtons set to "off" will still advertise inlineButtons, so the fallback should read merged config without resolving the token or otherwise preserve the configured off state.
  - confidence: 0.88

  - body: The unresolved-SecretRef fallback drops account.config.reactionLevel and resolves the default minimal level. For a SecretRef-backed Telegram account configured with reactionLevel "off", prompt prep will still emit reaction guidance; preserve the merged config value without resolving the token.
  - confidence: 0.87
