## Summary

Keep open: the PR addresses the reported crash path, but current head still over-advertises Telegram inline-button and reaction capabilities when credential SecretRefs are unresolved, so it needs a narrow follow-up before merge.

## What This Changes

The PR catches unresolved Telegram SecretRef failures in channel-action, inline-button, and reaction-guidance discovery, adds Discord/Telegram regression tests, and updates the changelog.

## Best Possible Solution

Land a plugin-owned discovery fix that avoids raw SecretRef resolution while preserving merged Telegram capability and reaction settings, with runtime send/probe paths remaining strict.

## Reproduction Assessment

Yes. Source inspection shows current main prompt prep reaches Telegram helpers that call resolveTelegramAccount on raw config, and the PR-specific regressions follow from combining an unresolved exec/file SecretRef with capabilities.inlineButtons or reactionLevel set to off.

## Solution Assessment

No. Catching unresolved SecretRefs in discovery is the right layer, but the fallback should use merged or inspected Telegram config instead of substituting defaults that enable disabled capabilities.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Preserve disabled inline-button config:** `extensions/telegram/src/inline-buttons.ts:74`
  - body: When resolveTelegramAccount throws for an unresolved SecretRef, this returns DEFAULT_INLINE_BUTTONS_SCOPE, which current code treats as inline buttons enabled. A SecretRef-backed account with capabilities.inlineButtons: "off" will still advertise inlineButtons in prompt prep, so use merged/inspected config or a conservative off fallback without resolving the token.
  - confidence: 0.9

  - body: This fallback calls resolveReactionLevel with value: undefined, producing the minimal level with agent reaction guidance. For a SecretRef-backed Telegram account configured with reactionLevel: "off", prompt prep will still tell the model to use reactions; preserve the merged configured reactionLevel without resolving the credential.
  - confidence: 0.89
