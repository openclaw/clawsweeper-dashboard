## Label Justifications

- `P1`: Affected users can run an agent without intended prompt instructions while receiving no indication that the files were ignored.
- `impact:session-state`: The issue concerns which agent-specific instructions are included in the session’s system-prompt context.
- `impact:security`: Misplaced safety or access-control instructions can create a false belief that prompt-level guardrails are active.

## Summary

Current main and v2026.6.5 still silently ignore bootstrap-named files placed under agentDir. Per-agent bootstrap customization already belongs in each agent’s configured workspace, but the requested misplaced-file diagnostic remains useful and all linked implementation PRs closed unmerged; the product/security labels also require explicit maintainer handling.

## What This Changes

Add per-agent bootstrap loading from agentDir or warn and document that prompt bootstrap files must live in the agent’s workspace.

## Best Possible Solution

Preserve each configured agent workspace as the sole bootstrap source, keep agentDir state-only, and add a focused startup, setup, or doctor diagnostic that identifies recognized bootstrap files misplaced under agentDir and points to the correct per-agent workspace.

## Reproduction Assessment

Yes. Current main and v2026.6.5 load bootstrap files only from workspaceDir, while agentDir remains a separate state directory; placing SOUL.md or AGENTS.md only under agentDir therefore follows a clear ignored path with no diagnostic.

## Solution Assessment

No, loading or merging bootstrap files from agentDir is not the best default because it creates a second prompt source. The narrower maintainable repair is to preserve the documented per-agent workspace contract and warn clearly about misplaced recognized files.
