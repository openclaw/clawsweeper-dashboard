## Summary

Keep open: current main and v2026.5.20 still resolve bootstrap prompt files from the workspace only, with no diagnostic for bootstrap-named files placed under a configured agentDir. The related warning and agentDir-merge PRs are closed unmerged, so the reported silent failure remains valid and security-sensitive.

## What This Changes

The issue asks OpenClaw to warn about or support bootstrap files placed in per-agent agentDir because prompt injection currently reads only workspace bootstrap files.

## Best Possible Solution

Add a warning-only diagnostic for recognized bootstrap filenames found in agentDir but not used, cover CLI/embedded/compaction/system-prompt resolver paths, update docs, and leave agentDir merge or override semantics to a separate product decision.

## Reproduction Assessment

Yes. Source inspection shows a bootstrap file placed only under agentDir is never read because the resolver and loader only use workspaceDir, and current main has no warning helper for that path.

## Solution Assessment

Yes for the minimum fix: warning-only diagnostics are the narrow backward-compatible solution. Loading or merging agentDir bootstrap files is a broader prompt-context behavior change that needs maintainer product and security review.
