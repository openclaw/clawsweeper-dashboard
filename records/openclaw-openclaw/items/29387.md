## Summary

Keep open: current main and v2026.5.19 still load bootstrap files from workspaceDir only, and all related repair PRs are closed unmerged, so the silent-ignore report remains source-reproducible.

## What This Changes

The issue asks OpenClaw to warn about or support bootstrap files placed in per-agent agentDir paths because current prompt bootstrap injection only considers workspace files.

## Best Possible Solution

Add a warning-only diagnostic across every bootstrap resolver path, check the effective workspace file before warning, reuse bootstrap warning dedupe, and document that agentDir is state unless a separate product decision changes merge semantics.

## Reproduction Assessment

Yes, source-reproducible: a bootstrap file placed only under a configured agentDir is never read because the resolver and loader only use workspaceDir. Current main also has no warning helper for recognized bootstrap filenames in agentDir.

## Solution Assessment

Yes for the minimum warning-only fix: diagnostics are the narrow backward-compatible repair. Loading or merging agentDir bootstrap files is not the best automatic fix because it changes prompt-source semantics and needs explicit maintainer product judgment.
