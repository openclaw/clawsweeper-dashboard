## Summary

Keep open: current main still resolves bootstrap context from the workspace only, and the open closing PR has not landed, so this should remain open until that fix is corrected, merged, or explicitly superseded.

## What This Changes

The issue requests a warning or supported behavior for bootstrap-named files placed in per-agent `agentDir` because users can mistake them for active system-prompt context.

## Best Possible Solution

Land or supersede https://github.com/openclaw/openclaw/pull/72871 with a warning-only diagnostic that covers CLI, embedded, and compaction bootstrap resolver paths; keep merge/override semantics separate.

## Reproduction Assessment

Yes. Source inspection shows a setup with `SOUL.md` or `AGENTS.md` only under `agentDir` follows resolver paths that load `workspaceDir` files and never inspect `agentDir`, so the file is neither injected nor warned about.

## Solution Assessment

Yes for the minimum silent-failure fix: a warning-only diagnostic is the narrow backward-compatible solution. Loading or merging `agentDir` bootstrap files would be a separate product/API decision.
