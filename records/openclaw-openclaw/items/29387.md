## Summary

Keep open: current main still resolves bootstrap context from the configured workspace only and has no diagnostic for bootstrap-named files placed in agentDir; the linked closing PR is still open, so this issue should remain open until that PR lands or is superseded.

## What This Changes

The issue requests either per-agent bootstrap handling or a warning when SOUL.md, AGENTS.md, TOOLS.md, USER.md, and related bootstrap files are placed in agentDir instead of the configured workspace.

## Best Possible Solution

Land or supersede https://github.com/openclaw/openclaw/pull/72871 with a narrow warning diagnostic and keep agentDir merge or override semantics as a separate product discussion.

## Reproduction Assessment

Yes by source inspection. A setup with SOUL.md only in agentDir follows a resolver path that reads loadWorkspaceBootstrapFiles(workspaceDir) and never inspects agentDir, so the file is neither injected nor warned about.

## Solution Assessment

Yes for the minimum silent-failure fix. A warning-only diagnostic is the narrowest backward-compatible solution; loading or merging agentDir bootstrap files should remain separate if maintainers want that behavior.
