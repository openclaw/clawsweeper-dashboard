## Summary

Keep open: current main still resolves bootstrap context from `workspaceDir` only, emits no `agentDir` ignored-bootstrap warning, and open PR #72871 is the linked implementation candidate rather than a merged fix.

## What This Changes

Add a diagnostic or per-agent bootstrap behavior so bootstrap-named Markdown files placed under an `agentDir` are not silently ignored during workspace bootstrap injection.

## Best Possible Solution

Review and land or supersede PR #72871; the minimum end state should warn for ignored bootstrap-named files in `agentDir`, with any merge/override semantics documented explicitly.

## Reproduction Assessment

Yes. Source inspection on current main shows bootstrap resolution accepts only `workspaceDir` and reads standard bootstrap filenames under that workspace, while `agentDir` is not passed into the resolver or warning path.

## Solution Assessment

Unclear. The Option B warning in #72871 is the narrowest maintainable fix for silent failure; per-agent `agentDir`/workspace merge semantics need explicit maintainer product agreement before being treated as the preferred end state.
