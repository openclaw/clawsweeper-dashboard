## Summary

Keep open: current main and the latest release still build bootstrap context from `workspaceDir` only and do not warn when bootstrap-named files are present under `agentDir`. The active closing PR at https://github.com/openclaw/openclaw/pull/72871 is still open and has review feedback, so this issue should remain open until that PR lands or is superseded.

## What This Changes

Add a diagnostic or supported behavior for bootstrap-named files placed in per-agent `agentDir`, because prompt injection currently reads workspace files only.

## Best Possible Solution

Land or supersede https://github.com/openclaw/openclaw/pull/72871 with a corrected warning diagnostic, then handle agentDir merge/override semantics only as a separate product decision if maintainers want that behavior.

## Reproduction Assessment

Yes by source inspection. A setup with `SOUL.md` only in `agentDir` follows resolver paths that load `workspaceDir` files and never inspect `agentDir`, so the file is neither injected nor warned about.

## Solution Assessment

Yes for the minimum silent-failure fix. A warning-only diagnostic is the narrowest backwards-compatible solution; loading or merging `agentDir` bootstrap files should remain separate unless maintainers choose that product direction.
