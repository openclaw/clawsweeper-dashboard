## Summary

Keep open: current main and the latest release still resolve bootstrap context from the workspace only, with no agentDir warning path, and the related fix attempts are closed unmerged.

## What This Changes

The issue asks OpenClaw to stop silently ignoring bootstrap-named files placed in an agentDir, either by supporting per-agent bootstrap loading or by warning clearly.

## Best Possible Solution

Add a clear warning for bootstrap-named files found in agentDir but not used, update the public workspace/config docs, and only add agentDir merge or override semantics if maintainers approve that prompt-loading contract.

## Reproduction Assessment

Yes. Source inspection shows the bootstrap resolver loads from workspaceDir and has no agentDir input or scan, so agentDir bootstrap files cannot be injected or warned about by current main.

## Solution Assessment

No, not until maintainers choose the behavior. Warning-only is the narrowest maintainable bug mitigation, while per-agent agentDir merge or override semantics are a broader product and security contract.
