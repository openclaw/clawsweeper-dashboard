## Summary

Keep open: current main and the latest release still resolve bootstrap prompt files from workspaceDir only, with no diagnostic for bootstrap-named files under a configured agentDir. The warning-only and merge/override PR attempts linked in the discussion are all closed unmerged, so the silent-ignore report remains valid.

## What This Changes

The issue requests either per-agent bootstrap loading from agentDir or a warning when bootstrap-named files in agentDir are ignored.

## Best Possible Solution

Add a warning-only diagnostic for recognized bootstrap filenames found in configured agentDir but not used, cover all bootstrap resolver paths, reuse warning dedupe, and leave agentDir merge/override semantics to a separate product/API decision.

## Reproduction Assessment

Yes. Source inspection on current main shows bootstrap files are resolved from workspaceDir only, while agentDir is neither loaded nor inspected for ignored bootstrap filenames.

## Solution Assessment

Yes for the minimum fix: warning-only diagnostics are the narrow backward-compatible solution. Loading or merging agentDir bootstrap files would be a broader product/API decision and should not be bundled into the bug fix.
