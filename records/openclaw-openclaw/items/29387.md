## Summary

Keep open: current main and the latest release still load bootstrap context from the workspace only, while recognized bootstrap files under agentDir are neither injected nor warned about; the linked warning/override PRs are closed unmerged.

## What This Changes

The issue requests that OpenClaw stop silently ignoring recognized bootstrap files placed under agentDir, either by warning users or by supporting explicit per-agent bootstrap behavior.

## Best Possible Solution

Add a warning-only diagnostic for recognized bootstrap filenames found in agentDir, cover the bootstrap resolver call paths and docs, and leave agentDir merge/override semantics to a separate explicit product/security design.

## Reproduction Assessment

Yes at source level. The resolver and its main callers only accept workspaceDir, so a bootstrap file placed only in agentDir is not part of the injected context and there is no ignored-agentDir warning path in current main.

## Solution Assessment

Yes for the minimum fix: a warning-only diagnostic is the narrow backward-compatible repair. Loading or merging agentDir bootstrap files should remain a separate product/security decision because it changes prompt precedence and per-agent policy semantics.
