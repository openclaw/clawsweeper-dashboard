## Summary

Keep open: current main still resolves bootstrap context from `workspaceDir` only and does not inspect `agentDir` for ignored bootstrap-named files. The issue also has an open closing PR, https://github.com/openclaw/openclaw/pull/72871, so this should remain open until that PR is corrected and landed or explicitly superseded.

## What This Changes

The issue asks OpenClaw to stop silently ignoring bootstrap-named files placed under per-agent `agentDir`, either by loading them or warning that only workspace files are injected.

## Best Possible Solution

Land or supersede https://github.com/openclaw/openclaw/pull/72871 with a warning-only diagnostic that covers CLI, embedded, and compaction resolver paths; leave agentDir/workspace merge semantics to a separate product decision.

## Reproduction Assessment

Yes. By source inspection, a setup with `SOUL.md` or `AGENTS.md` only under `agentDir` follows resolver paths that load `workspaceDir` files and never inspect `agentDir`, so the file is neither injected nor warned about.

## Solution Assessment

Yes for the minimum silent-failure fix: a warning-only diagnostic is the narrowest backward-compatible repair. Loading or merging `agentDir` bootstrap files would be a separate product decision.
