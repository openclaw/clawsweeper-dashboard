## Summary

Keep open: current main and v2026.5.20 still load bootstrap context from `workspaceDir` only, have no ignored-`agentDir` bootstrap-file warning, and the directly linked fix PRs are closed unmerged.

## What This Changes

Warn or otherwise handle recognized bootstrap files placed under per-agent `agentDir` so users do not believe ignored instructions were injected.

## Best Possible Solution

Add a warning-only diagnostic for recognized bootstrap filenames found in `agentDir`, cover every bootstrap resolver path, reuse warning dedupe, update docs, and leave merge/override semantics to a separate product/security decision.

## Reproduction Assessment

Yes. Source inspection shows bootstrap resolution reads only `workspaceDir`, so a bootstrap file placed only in `agentDir` is not injected and no ignored-agentDir warning path exists on current main.

## Solution Assessment

Yes for the minimum fix. A warning-only diagnostic is the narrow backward-compatible repair; loading or merging `agentDir` bootstrap files should remain a separate product/security decision.
