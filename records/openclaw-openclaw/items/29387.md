## Summary

Keep open. Current main still resolves bootstrap prompt files from `workspaceDir` only and has no `agentDir` diagnostic, while the linked warning implementation is open and unmerged.

## What This Changes

The issue asks OpenClaw to either load or merge per-agent `agentDir` bootstrap markdown files, or warn when standard bootstrap files in `agentDir` are ignored, and clarify the docs.

## Best Possible Solution

Resolve the issue by adding a tested runtime diagnostic for ignored standard bootstrap files in `agentDir`, or by defining and implementing a product-approved merge/override contract across all bootstrap resolver call sites. The docs should continue to state that prompt bootstrap files belong in the workspace.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: put `SOUL.md` or `AGENTS.md` only in `agentDir`; current main passes only `workspaceDir` into bootstrap resolution, so the file is neither injected nor diagnosed.

## Solution Assessment

Yes for keeping this open. The narrowest maintainable fix is the warning-only Option B path already represented by open PR #72871; Option A needs explicit merge/override semantics before implementation.
