## Summary

Keep open: current main and the latest release still resolve bootstrap context from the effective workspace only, while the warning-only and agentDir-merge attempts linked in the discussion are closed unmerged. The remaining choice is security/product-sensitive because warning-only preserves today's workspace boundary, while loading agentDir files would change prompt precedence and per-agent state semantics.

## What This Changes

The issue asks OpenClaw to stop silently ignoring bootstrap-named markdown files placed in per-agent agentDir, either by warning or by supporting per-agent merge/override behavior.

## Best Possible Solution

Keep workspace as the canonical bootstrap source unless maintainers approve agentDir precedence; otherwise land a warning-only diagnostic plus docs/tests that cover every bootstrap resolver path.

Yes by source inspection. Current main loads bootstrap context from workspaceDir only and has no agentDir warning or merge path; I did not run a live setup because this review is read-only.

Unclear. Warning-only is the narrowest low-risk fix, but the reporter's preferred agentDir merge/override changes prompt authority and needs maintainer/security policy before implementation.
