## Summary

Keep open: current main and v2026.5.20 still resolve bootstrap files from workspaceDir only, and the direct warning/override PRs are closed unmerged, so the silent agentDir bootstrap-file failure remains source-reproducible.

## What This Changes

The issue asks OpenClaw to warn when recognized bootstrap files are placed in per-agent agentDir directories or to support per-agent bootstrap file merging instead of silently loading only workspace bootstrap files.

## Best Possible Solution

Add a warning-only diagnostic for recognized bootstrap filenames found in agentDir but not used, cover every bootstrap resolver path, reuse warning dedupe, update docs, and leave agentDir merge/override semantics to a separate product/API decision.

## Reproduction Assessment

Yes. Source inspection shows bootstrap resolution reads only workspaceDir, so a bootstrap file placed only in agentDir is not injected and no ignored-agentDir warning exists on current main.

## Solution Assessment

Yes for the minimum fix. A warning-only diagnostic is the narrow backward-compatible repair; loading or merging agentDir bootstrap files should remain a separate product/security decision.
