## Label Justifications

- `P1`: A reproducible silent bootstrap misconfiguration can leave real multi-agent workflows running without instructions users believe are active.
- `impact:security`: The issue concerns safety or access-control instructions being silently absent from the model prompt.
- `impact:session-state`: The affected bootstrap context changes the instructions associated with an agent’s sessions and can cause persistent expectation drift.

## Summary

Keep open: current `main` still loads bootstrap prompt files from each agent’s workspace, not `agentDir`, and all linked implementation attempts are closed unmerged. The report is source-reproducible, but the protected security/product-decision signals mean maintainers must choose between clarifying the directory contract with diagnostics or intentionally adding a new bootstrap precedence rule.

## What This Changes

Clarify or change bootstrap loading so files placed in an agent’s `agentDir` are not silently mistaken for active prompt instructions.

## Best Possible Solution

Define `agentDir` as state-only and add a narrowly scoped startup/doctor diagnostic for bootstrap-named files there, unless maintainers explicitly approve a documented per-agent merge/override contract with precedence, hook ordering, and security tests.

## Reproduction Assessment

Yes from source inspection: configure distinct workspace and `agentDir` paths, place a distinctive standard bootstrap file only in `agentDir`, and the current loader cannot select it because it resolves those names solely under `workspaceDir`. A live run was not performed in this read-only review.

## Solution Assessment

No single proposed implementation is approved as the best fix yet. A warning or doctor diagnostic preserves the current directory ownership contract, while loading from `agentDir` changes precedence and security semantics and therefore needs an explicit maintainer decision.
