## Label Justifications

- `P1`: Users can believe safety or persona instructions are active while the agent prompt silently omits them in multi-agent setups.

## Summary

Keep open: current main still resolves bootstrap context from workspace files only, the latest release has the same behavior, and all known PR attempts for this bug are closed unmerged.

## What This Changes

The issue asks OpenClaw to stop silently ignoring bootstrap-named files in per-agent `agentDir`, either by loading/merging them or by warning and documenting the workspace-only contract.

## Best Possible Solution

Add a warning-only diagnostic across every bootstrap resolver path, check the effective workspace file before warning, reuse bootstrap warning dedupe, and leave `agentDir` merge/override semantics to separate product/API work.

## Reproduction Assessment

Yes, source-reproducible: a bootstrap file placed only under a configured `agentDir` is never read because the resolver and loader only use `workspaceDir`, and there is no warning helper for that path.

## Solution Assessment

Yes for the minimum fix: warning-only diagnostics are the narrow backward-compatible solution. Loading or merging `agentDir` bootstrap files would be a broader product/API decision and should not be bundled into this bug fix.
