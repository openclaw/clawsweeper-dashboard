## Summary

Keep open: current main still loads bootstrap files from the workspace only and has no diagnostic for bootstrap-named files under agentDir. The linked closing PR remains open and conflicting, so this issue should stay open until that PR is fixed, landed, or superseded.

## What This Changes

The issue asks OpenClaw to warn about or support bootstrap-named files placed in per-agent agentDir instead of silently ignoring them.

## Best Possible Solution

Land or supersede https://github.com/openclaw/openclaw/pull/72871 with a warning-only diagnostic that covers every bootstrap resolver path while leaving agentDir merge semantics separate.

## Reproduction Assessment

Yes, by source inspection. A setup with SOUL.md or AGENTS.md only in agentDir follows bootstrap resolver paths that load workspaceDir files and never inspect agentDir, so the file is neither injected nor warned about.

## Solution Assessment

Yes for the minimum silent-failure fix. A warning-only diagnostic is the narrowest backward-compatible repair; loading or merging agentDir bootstrap files should remain a separate product decision.
