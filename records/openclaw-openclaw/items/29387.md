## Summary

Keep open: current `main` still resolves bootstrap context from the workspace only and has no source path that inspects `agentDir` for ignored bootstrap-named files. The issue also has an open `Fixes` PR, so it should remain open until that PR is corrected and landed or explicitly superseded.

## What This Changes

Warn or otherwise handle bootstrap-named files placed under an agent's `agentDir` because current prompt injection reads bootstrap files from the workspace only.

## Best Possible Solution

Land or supersede https://github.com/openclaw/openclaw/pull/72871 with a warning-only diagnostic that checks effective workspace files first, covers CLI/embedded/compaction paths, preserves workspace-only injection, and leaves merge/override semantics separate.

## Reproduction Assessment

Yes, by source inspection. A setup with `SOUL.md` or `AGENTS.md` only under `agentDir` follows resolver paths that load `workspaceDir` files and never inspect `agentDir`, so the file is neither injected nor warned about.

## Solution Assessment

Yes for the minimum silent-failure fix: a warning-only diagnostic is the narrowest backward-compatible repair. Loading or merging `agentDir` bootstrap files should remain a separate product decision.
