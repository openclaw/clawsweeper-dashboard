## Summary

Keep open: current main still resolves injected bootstrap context from the agent workspace, not `agentDir`, and the directly linked fix PRs are closed unmerged, so the silent `agentDir` bootstrap-file case still needs a maintainer decision.

## What This Changes

The issue asks OpenClaw to stop silently ignoring bootstrap-named files placed under a per-agent `agentDir`, either by loading them or warning/documenting that only workspace files are injected.

## Best Possible Solution

Keep workspace as the prompt-context boundary unless maintainers explicitly sponsor per-agent bootstrap files, and add a clear startup/doctor diagnostic for bootstrap-named files found under `agentDir`.

## Reproduction Assessment

Yes. Current main source shows bootstrap context is resolved from `workspaceDir`, while `agentDir` is a separate state/auth/session directory and is not checked for standard bootstrap filenames.

## Solution Assessment

No implementation is under review yet. Warning-only is the narrow compatibility-preserving repair; loading or merging `agentDir` bootstrap files would be a new prompt-ownership feature that needs maintainer direction.
