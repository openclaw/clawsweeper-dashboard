## Summary

Keep open: current main and the latest release still resolve bootstrap context from the workspace only, and the warning/override attempts linked in the discussion are closed unmerged. The minimum repair looks source-reproducible, but choosing warning-only versus agentDir merge/override semantics is a maintainer product/security decision.

## What This Changes

The issue requests either warning about recognized bootstrap files placed in `agentDir` or supporting per-agent agentDir bootstrap merge/override behavior.

## Best Possible Solution

Add a warning-only diagnostic for recognized bootstrap filenames found in agentDir, cover the resolver call paths and docs, and leave merge/override semantics to a separate explicit product/security design.

## Reproduction Assessment

Yes, at source level. The resolver and its main callers only accept workspaceDir, so a bootstrap file placed only in agentDir is not part of injected context and there is no current warning path for that ignored file.

## Solution Assessment

Yes for the minimum fix: a warning-only diagnostic is the narrow backward-compatible repair. Loading or merging agentDir bootstrap files should remain a separate product/security decision because it changes prompt precedence and per-agent policy semantics.
