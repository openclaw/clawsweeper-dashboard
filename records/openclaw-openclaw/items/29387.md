## Summary

Keep open: current main and the latest tagged release still load bootstrap context from workspace paths only, and the open closing PR has not merged.

## What This Changes

The issue reports that bootstrap files placed under a configured per-agent `agentDir` are silently ignored while only workspace bootstrap files enter the system prompt.

## Best Possible Solution

Land or supersede https://github.com/openclaw/openclaw/pull/72871 with a warning-only diagnostic that covers all bootstrap resolver paths and leaves merge/override semantics separate.

## Reproduction Assessment

Yes. Source inspection on current main shows a bootstrap file placed only under `agentDir` is never read by the workspace-only resolver and no warning helper exists for that path.

## Solution Assessment

Yes for the minimum silent-failure fix: a warning-only diagnostic is the narrow backward-compatible solution. Per-agent bootstrap merging would be broader behavior and should remain a separate decision.
