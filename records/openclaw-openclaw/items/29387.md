## Summary

Keep open: current main and v2026.6.10 still load bootstrap files from the configured workspace only, while two open PRs explicitly target this issue with different fixes. The remaining work is a maintainer/security policy choice between warning-only diagnostics and making agentDir a prompt-authority source.

## What This Changes

The issue asks OpenClaw to stop silently ignoring bootstrap-named files placed in a configured per-agent agentDir, either by warning or by defining agentDir bootstrap loading and precedence.

## Best Possible Solution

Land a maintainer-approved bootstrap policy: warning-only diagnostics that keep workspace as the canonical source, or explicit agentDir bootstrap loading with docs, diagnostics, and security/upgrade proof.

## Reproduction Assessment

Yes by source inspection: current main resolves bootstrap files from workspaceDir only and has no agentDir warning or loading path. I did not run a live setup because this review is read-only.

## Solution Assessment

Unclear until maintainers decide the policy. Warning-only is the lowest-risk fix; agentDir loading addresses per-agent customization but changes prompt authority.
