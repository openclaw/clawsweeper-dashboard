## Label Justifications

- `P1`: The report describes a silent prompt-rule failure that users can hit in multi-agent setups, and current source plus comments show the workflow remains unresolved.
- `impact:security`: Users can place safety or access-control instructions in `agentDir/AGENTS.md` and believe they are active when the resolver never injects them.
- `impact:session-state`: The affected surface is bootstrap prompt context for agent sessions, so the active session instructions can drift from the user's intended per-agent state.

## Summary

Current main and v2026.6.5 still resolve bootstrap context from the configured workspace only, while all directly linked fix PRs for this report are closed unmerged, so this issue remains necessary.

## What This Changes

The issue asks OpenClaw to stop silently ignoring bootstrap-named files placed in per-agent `agentDir`, either by loading them or warning that only workspace files are injected.

## Best Possible Solution

Make the bootstrap source of truth explicit: either keep workspace-only injection with a startup or doctor warning plus docs, or deliberately add per-agent override semantics with full resolver coverage and compatibility tests.

## Reproduction Assessment

Yes, by source inspection. `resolveBootstrapFilesForRun` only loads `workspaceDir` files, so a bootstrap file placed only in `agentDir` cannot be injected into the system prompt on current main.

## Solution Assessment

Unclear until maintainers choose the product boundary. Warning-only is the narrow compatibility-preserving fix, while agentDir merge/override would add a new per-agent bootstrap capability and needs explicit semantics.
