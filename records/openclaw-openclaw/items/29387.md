## Label Justifications

- `P1`: The issue describes a silent prompt-context failure that can leave expected safety rules out of active agent runs for real multi-agent users.
- `impact:security`: Users can place safety or access-control rules in agentDir/AGENTS.md and receive no warning that those rules are not enforced.
- `impact:session-state`: The affected behavior changes which bootstrap context is attached to an agent session and can make session instructions diverge from user expectations.

## Summary

Keep open: current main and v2026.6.6 still resolve bootstrap context from workspace files only, while prior warning/agentDir-merge PRs are closed unmerged. The remaining decision is security-sensitive because warning-only is narrow, but loading agentDir files changes prompt precedence and ownership semantics.

## What This Changes

The issue asks OpenClaw to warn when bootstrap-named files are placed in agentDir or deliberately load/merge those files for per-agent prompt customization.

## Best Possible Solution

Add a compatibility-preserving diagnostic and docs clarification for agentDir bootstrap-named files now, and only add agentDir merge/override behavior after maintainers explicitly choose prompt precedence and security semantics.

## Reproduction Assessment

Yes, at source level. Current main and v2026.6.6 route bootstrap injection through workspaceDir only, and the resolver/callers have no agentDir diagnostic or merge path.

## Solution Assessment

Unclear as an automatic fix. Warning-only is the narrowest compatibility-preserving path, while agentDir merge/override would change security-sensitive prompt precedence and needs maintainer product review.
