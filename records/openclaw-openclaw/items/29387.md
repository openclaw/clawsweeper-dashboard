## Summary

Keep open: current main still has no agentDir input or ignored-bootstrap diagnostic in bootstrap resolution, and the open closing PR is dirty/unlanded with outstanding review and proof work.

## What This Changes

The issue asks OpenClaw to warn about or support bootstrap-named files placed under per-agent agentDir instead of the configured workspace.

## Best Possible Solution

Land or supersede https://github.com/openclaw/openclaw/pull/72871 with a warning-only diagnostic that covers CLI, embedded, and compaction bootstrap resolver paths while keeping agentDir merge semantics separate.

## Reproduction Assessment

Yes, by source inspection. A setup with SOUL.md or AGENTS.md only in agentDir follows resolver paths that read workspaceDir files and never inspect agentDir, so the file is neither injected nor warned about.

## Solution Assessment

Yes for the minimum silent-failure fix. A warning-only diagnostic is the narrowest backward-compatible repair; loading or merging agentDir bootstrap files should remain a separate product decision.
