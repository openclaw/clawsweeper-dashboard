## Summary

Keep open: current `main` still resolves bootstrap files from `workspaceDir` only and has no diagnostic for bootstrap-named files under `agentDir`. The linked fixing PR is still open, conflicting, and has review feedback, so this issue should stay open until that PR lands or is superseded.

## What This Changes

The issue reports that bootstrap files placed in a configured per-agent `agentDir` are silently ignored while workspace bootstrap files are injected.

## Best Possible Solution

Land or supersede https://github.com/openclaw/openclaw/pull/72871 with a warning-only diagnostic that covers every bootstrap resolver path, while keeping `agentDir` merge/override semantics separate from this bug fix.

## Reproduction Assessment

Yes, by source inspection. A setup with `SOUL.md` or `AGENTS.md` only in `agentDir` follows resolver paths that load `workspaceDir` files and never inspect `agentDir`, so the file is neither injected nor warned about.

## Solution Assessment

Yes for the minimum silent-failure fix. A warning-only diagnostic is the narrowest backward-compatible repair; loading or merging `agentDir` bootstrap files should remain a separate maintainer product choice.
