## Summary

Keep open: current main still loads bootstrap context from the workspace only, has no ignored-`agentDir` bootstrap warning, and the linked closing PR is open rather than merged.

## What This Changes

The issue asks OpenClaw to diagnose or support bootstrap files such as `SOUL.md` and `AGENTS.md` when they are placed in an agent-specific `agentDir` instead of the workspace.

## Best Possible Solution

Review, repair, and land or supersede https://github.com/openclaw/openclaw/pull/72871; the minimum end state should warn for recognized bootstrap filenames in `agentDir` when the corresponding effective workspace file is absent, with docs making the workspace-only bootstrap contract explicit.

## Reproduction Assessment

Yes. Source inspection on current main shows the resolver and runtime call sites only pass `workspaceDir`, while the workspace loader reads standard bootstrap filenames from that workspace and no `agentDir` warning path exists.

## Solution Assessment

Yes for the minimum silent-failure fix. A warning-only diagnostic is the narrowest backward-compatible solution; loading or merging `agentDir` bootstrap files should be treated as a separate product decision if maintainers want it.
