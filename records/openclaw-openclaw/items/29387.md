## Summary

Keep open: current main and the latest release still resolve bootstrap prompt files from the workspace only, with no diagnostic for bootstrap-named files placed under a configured agentDir. The discussion’s warning-only and merge/override PR attempts are all closed unmerged, so the reported silent failure remains valid.

## What This Changes

The issue requests a warning or per-agent bootstrap behavior when SOUL.md, AGENTS.md, and similar bootstrap files are placed in a configured agentDir but are not injected into the system prompt.

## Best Possible Solution

Add a warning-only diagnostic for recognized bootstrap filenames found in agentDir but not used, cover all bootstrap resolver paths, reuse warning dedupe, and leave agentDir merge/override semantics to a separate product/API decision.

## Reproduction Assessment

Yes, source-reproducible: a bootstrap file placed only under a configured agentDir is never read because the resolver and loader only use workspaceDir, and current main has no warning helper for that path.

## Solution Assessment

Yes for the minimum fix: warning-only diagnostics are the narrow backward-compatible solution. Loading or merging agentDir bootstrap files would be a broader product/API decision and should not be bundled into the bug fix.
