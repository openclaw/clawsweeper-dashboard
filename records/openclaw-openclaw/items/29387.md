## Summary

Keep open: current main and the latest release still resolve prompt bootstrap files from the configured workspace, not agentDir, and the linked warning/merge PRs are closed unmerged. The report remains valid, but the final fix needs maintainer direction because warning-only and per-agent bootstrap merging have different prompt-precedence and compatibility implications.

## What This Changes

The issue reports that bootstrap-named markdown files placed in a configured per-agent agentDir are silently ignored while workspace bootstrap files are injected into the system prompt.

## Best Possible Solution

Keep this open for a maintainer-chosen fix that either emits a clear diagnostic when bootstrap-named files are found in agentDir or deliberately implements per-agent bootstrap merging with documented precedence and regression coverage across CLI, embedded-runner, and compaction paths.

## Reproduction Assessment

Yes, at source level. Current main and v2026.6.6 resolve bootstrap files from workspaceDir only, and the main CLI/embedded callers do not pass agentDir into the bootstrap resolver.

## Solution Assessment

No, not as an automatic cleanup close. Warning-only looks like the narrow compatibility-preserving repair, but agentDir file merging would change prompt precedence and should be an explicit maintainer product/security decision.
