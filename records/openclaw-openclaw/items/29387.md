## Summary

Keep open: current main and the latest release still build bootstrap context from `workspaceDir` only and have no diagnostic for bootstrap-named files under `agentDir`. The active closing PR at https://github.com/openclaw/openclaw/pull/72871 is still open, dirty, and has review feedback, so this issue should remain open until that PR lands or is superseded.

## What This Changes

The issue asks OpenClaw to diagnose or support bootstrap-named files placed in per-agent `agentDir` instead of silently loading only workspace bootstrap files.

## Best Possible Solution

Land or supersede https://github.com/openclaw/openclaw/pull/72871 with a warning-only diagnostic that covers every bootstrap resolver path; keep agentDir merge/override semantics separate unless maintainers explicitly choose that behavior.

## Reproduction Assessment

Yes, by source inspection. A setup with `SOUL.md` only in `agentDir` follows resolver paths that load `workspaceDir` files and never inspect `agentDir`, so the file is neither injected nor warned about.

## Solution Assessment

Yes for the minimum silent-failure fix. A warning-only diagnostic is the narrowest backward-compatible repair; loading or merging `agentDir` bootstrap files should remain separate unless maintainers choose that product direction.
