## Summary

Keep this open. Current main still resolves bootstrap context from the workspace only and has no `agentDir` warning path, while PR #72871 is open with closing syntax for this issue and should be reviewed or superseded before cleanup.

## What This Changes

The issue requests that bootstrap-named Markdown files in a per-agent `agentDir` either be loaded/merged into prompt context or produce a clear warning and documentation.

## Best Possible Solution

Keep this issue as the canonical tracker and review PR #72871 as the narrow diagnostic fix; if maintainers prefer Option A, first define the workspace-versus-`agentDir` merge contract and then implement it with focused tests and docs.

## Reproduction Assessment

Yes. Source inspection of current main shows bootstrap resolution loads only `workspaceDir` and never receives `agentDir`, so an `agentDir`-only `SOUL.md` or `AGENTS.md` will not be injected or warned about.

## Solution Assessment

Unclear for the preferred Option A because merge/override semantics need maintainer product agreement. Option B in PR #72871 is the narrowest maintainable fix for the confirmed silent-failure path.
