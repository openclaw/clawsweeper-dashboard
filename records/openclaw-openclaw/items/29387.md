## Summary

Keep open: current main and v2026.6.10 still resolve bootstrap context from `workspaceDir` only, while https://github.com/openclaw/openclaw/pull/94341 and https://github.com/openclaw/openclaw/pull/96720 are open closing fix candidates that should be reviewed before this issue closes.

## What This Changes

The issue asks OpenClaw to stop silently ignoring bootstrap-named files in a configured per-agent `agentDir`, either by warning or by supporting a defined per-agent merge/override behavior.

## Best Possible Solution

Review and land one chosen path: either a warning-only diagnostic and docs if `agentDir` remains state-only, or an explicit agentDir bootstrap precedence implementation with security and upgrade proof.

## Reproduction Assessment

Yes, source-level. Current main loads bootstrap metadata from `workspaceDir` only and does not inspect `agentDir` for warning or merge behavior; I did not run a live agent setup in this read-only review.

## Solution Assessment

Unclear. Warning-only is the narrowest low-risk repair, while agentDir merge/override satisfies the requested per-agent customization but needs maintainer/security approval for prompt-authority precedence.
