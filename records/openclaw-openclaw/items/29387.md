## Summary

Keep open: current main and v2026.5.19 still load bootstrap context from workspace files only, no ignored-agentDir diagnostic exists, and every PR that explicitly referenced this bug is closed unmerged.

## What This Changes

The issue requests a warning or per-agent bootstrap handling when SOUL.md, AGENTS.md, or similar bootstrap files are placed in agentDir instead of the configured workspace.

## Best Possible Solution

Add a warning-only diagnostic for recognized bootstrap filenames in agentDir across resolver paths, document that bootstrap injection comes from workspace, and leave merge/override semantics to a separate product/API decision.

## Reproduction Assessment

Yes, source-reproducible: the resolver does not accept agentDir and loads standard bootstrap files from workspaceDir only, so a file placed only under agentDir cannot be loaded or warned by this path.

## Solution Assessment

Unclear for the full requested behavior: warning-only is the narrow backward-compatible fix, while merging or overriding agentDir bootstraps is a broader prompt-contract decision.
