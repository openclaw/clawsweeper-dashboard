## Summary

Keep open: current main still loads bootstrap prompt files from the workspace path only and does not warn when bootstrap-named files are present only under a configured agentDir; the related fix attempts are closed unmerged, so the reported silent failure remains valid.

## What This Changes

The issue asks OpenClaw to warn about or support bootstrap-named files placed in a configured agentDir because prompt bootstrap injection currently reads workspace files only.

## Best Possible Solution

Add a warning-only diagnostic for recognized bootstrap filenames found in a configured agentDir but not used, cover all bootstrap resolver paths, update docs, and leave agentDir merge/override semantics to a separate product decision.

## Reproduction Assessment

Yes. Source inspection shows a bootstrap file placed only under agentDir is never read because the resolver and loader only use workspaceDir, and current main has no warning helper for that path.

## Solution Assessment

Yes for the minimum fix: warning-only diagnostics are the narrow backward-compatible repair. Loading or merging agentDir bootstrap files is a broader prompt-context behavior change that needs maintainer product and security review.
