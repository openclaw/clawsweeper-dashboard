## Summary

Keep open: current main and v2026.5.20 still load bootstrap files from workspaceDir only, and the direct warning/override PRs are closed unmerged, so the silent agentDir bootstrap-file failure remains source-reproducible.

## What This Changes

The issue requests a diagnostic or per-agent behavior for bootstrap-named files placed in agentDir, because only workspace bootstrap files are injected into the prompt today.

## Best Possible Solution

Add a warning-only diagnostic for recognized bootstrap filenames found in agentDir but not used, cover every bootstrap resolver path, reuse warning dedupe, and leave agentDir merge/override semantics to a separate product/API decision.

## Reproduction Assessment

Yes, source-reproducible: a bootstrap file placed only under a configured agentDir is never read because the resolver and loader only use workspaceDir, and current main has no ignored-agentDir warning path.

## Solution Assessment

Yes for the minimum fix: warning-only diagnostics are the narrow backward-compatible solution. Loading or merging agentDir bootstrap files would be a broader product/API decision and should not be bundled into this bug fix.
