## Summary

Keep open: current main still reads bootstrap context from the workspace only, has no ignored-agentDir bootstrap warning, and the linked closing PR is still open and conflicting rather than merged.

## What This Changes

The issue asks OpenClaw to warn about or support bootstrap files placed in per-agent agentDir so users do not silently miss expected system-prompt context.

## Best Possible Solution

Keep the workspace-only bootstrap contract and land or supersede the warning-only diagnostic, with docs clarifying that agentDir is runtime state rather than the bootstrap injection surface.

## Reproduction Assessment

Yes by source inspection. A setup with SOUL.md only in agentDir and not the workspace follows the current resolver path that reads loadWorkspaceBootstrapFiles(workspaceDir) and never inspects agentDir, so the file is neither injected nor warned about.

## Solution Assessment

Yes for the minimum silent-failure fix. A warning-only diagnostic is the narrowest backward-compatible solution; loading or merging agentDir bootstrap files should remain a separate product decision if maintainers want that behavior.
