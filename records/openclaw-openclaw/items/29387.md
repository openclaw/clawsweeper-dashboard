## Summary

Keep open: current main still loads bootstrap context from `workspaceDir` only and has no path that warns about ignored bootstrap-named files in `agentDir`. The issue also has an open closing PR, so this should remain open until that PR is corrected and landed or explicitly superseded.

## What This Changes

The issue requests warning or support for bootstrap-named files placed under a per-agent `agentDir` because current prompt injection only reads workspace bootstrap files.

## Best Possible Solution

Land or supersede https://github.com/openclaw/openclaw/pull/72871 with a warning-only diagnostic that checks the effective workspace file first, covers CLI/embedded/compaction paths, and leaves merge/override semantics to a separate product decision.

## Reproduction Assessment

Yes. By source inspection, a setup with `SOUL.md` or `AGENTS.md` only under `agentDir` follows resolver paths that load `workspaceDir` files and never inspect `agentDir`, so the file is neither injected nor warned about.

## Solution Assessment

Yes for the minimum silent-failure fix: a warning-only diagnostic is the narrowest backward-compatible repair. Loading or merging `agentDir` bootstrap files would be a separate product decision.
