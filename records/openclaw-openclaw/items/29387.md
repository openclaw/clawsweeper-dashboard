## Label Justifications

- `impact:session-state`: The issue changes which bootstrap context is associated with each agent session.
- `impact:security`: Users may incorrectly believe ignored AGENTS.md safety guidance is active.

## Summary

The issue remains necessary: current main and v2026.6.5 both use workspace-only bootstrap loading, all proposed fixes closed unmerged, and no newer canonical item owns the work. The supported per-agent customization seam is a per-agent workspace rather than agentDir, so maintainers should decide whether to add diagnostics and clearer docs or intentionally redefine agentDir as a prompt source.

## What This Changes

Add clear handling for bootstrap-named files placed in agentDir, either by warning users to use the per-agent workspace or by defining an intentional override contract.

## Best Possible Solution

Keep agentDir state-only, use each agent's configured workspace for distinct bootstrap context, and add a deduplicated diagnostic plus explicit docs when recognized bootstrap files are found only in a different agentDir.

## Reproduction Assessment

Yes. Current main deterministically loads bootstrap files only from workspaceDir, so a recognized file placed solely in agentDir is omitted without an agentDir-specific diagnostic.

## Solution Assessment

No to agentDir merging as the default fix; the narrowest maintainable solution is a warning and clearer per-agent workspace guidance, unless maintainers explicitly choose to redefine the agentDir trust and precedence contract.
