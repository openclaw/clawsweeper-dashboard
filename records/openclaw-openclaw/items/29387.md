## Summary

Keep this open: current main still loads bootstrap files from the workspace path only, has no agentDir ignored-bootstrap warning, and open PR #72871 is the active closing implementation candidate.

## What This Changes

The issue asks OpenClaw to either load per-agent agentDir bootstrap files or warn when bootstrap-named files in agentDir are ignored.

## Best Possible Solution

Review and land or supersede PR #72871; the minimum end state should warn for ignored bootstrap-named files in agentDir, with any merge/override semantics documented explicitly.

## Reproduction Assessment

Yes. Source inspection on current main shows bootstrap resolution accepts only workspaceDir and reads standard bootstrap filenames under that workspace, while agentDir is not passed into the resolver or warning path.

## Solution Assessment

Unclear. The warning-only approach in PR #72871 is the narrowest maintainable fix for the silent-failure path, while agentDir/workspace merge or override behavior needs maintainer product agreement.
