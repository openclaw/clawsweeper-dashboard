## Summary

Keep open. The PR targets a real current-main gap in embedded run finalization, but the proposed fallback emits directly from teardown and can bypass the existing serialized session-event chain; it also lacks the required changelog entry for a user-facing Control UI fix.

## What This Changes

The PR shares terminal lifecycle idempotency through embedded subscription state, emits a synthesized terminal lifecycle event from `unsubscribe()` when `agent_end` was missed, and adds unsubscribe fallback tests.

## Best Possible Solution

Keep the subscription-level terminal guarantee, but make teardown fallback participate in the same serialized session-event ordering as normal `agent_end` handling, or explicitly track terminal events as in-flight before deciding a fallback is needed. Add a focused regression for async pending message flush plus unsubscribe, and document the user-facing Control UI fix in the active changelog.

## Reproduction Assessment

Yes. The original bug has a code-level reproduction path through timeout-triggered `void activeSession.abort()`, cleanup `unsubscribe()`, and chat finalization depending on lifecycle `end` or `error`; the PR regression can be reproduced with a stub session that queues async `message_end` or `agent_end` work, then calls `unsubscribe()` before the promise settles.

## Solution Assessment

No. The subscription seam is the right place to guarantee a terminal lifecycle event, but this implementation is not the best fix until the fallback preserves the established pending-event ordering and the user-facing fix is recorded in the changelog.
