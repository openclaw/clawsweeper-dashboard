## Summary

Close as implemented on main: the stuck Control UI/Gateway chat terminal-lifecycle gap is already covered by the shipped runner-level backstop in current main, and this PR’s lower-level unsubscribe fallback is no longer needed. The proposed patch also has an ordering bug if merged because it emits outside the existing serialized session-event chain.

## What This Changes

The PR shares lifecycle terminal idempotency through embedded subscription state, emits a synthetic terminal lifecycle event from `subscribeEmbeddedPiSession.unsubscribe()`, and adds unsubscribe fallback unit tests.

## Best Possible Solution

Keep the shipped runner-level lifecycle backstop and close this PR. If maintainers still want a generic `subscribeEmbeddedPiSession` teardown contract, it should be designed through the serialized event-handler path rather than direct unsubscribe emission.

## Reproduction Assessment

Yes. The original bug has a high-confidence code path: a timeout can abort the active embedded session and complete without a delivered `agent_end`; current main has focused regression tests for the runner returning without a terminal lifecycle event.

## Solution Assessment

No, this PR is no longer the best way to solve it. Current main fixes the observable Gateway/Control UI finalization problem at the runner boundary, while this PR’s unsubscribe-level fallback can bypass pending `message_end` or `agent_end` work.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Queue fallback terminals behind pending session events:** `src/agents/pi-embedded-subscribe.ts:836`
  - body: The fallback is emitted directly from `unsubscribe()`, outside `createEmbeddedPiSessionEventHandler`'s `pendingEventChain`. If `message_end` or `agent_end` is already queued behind async flush work, this can finalize chat before pending assistant/block output finishes and then suppress the real terminal path via `lifecycleTerminalEmitted`. Route any subscription fallback through the serialized event path or track terminal events in flight before emitting.
  - confidence: 0.87
