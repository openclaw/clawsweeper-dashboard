## Summary

Keep open: current main still lacks a public per-candidate or provider/model retry-before-fallback budget for opaque proxy pools, and the matching provider-retry duplicate was closed in favor of this issue rather than implemented.

## What This Changes

The issue requests configurable retry counts per fallback candidate or provider/model so pooled proxy providers can retry before OpenClaw advances to the next fallback model.

## Best Possible Solution

Have maintainers choose a canonical provider/model or candidate-scoped retry policy that composes with auth.cooldowns, timeout classification, and replay safety, then add schema, docs, logs, and focused fallback tests.

## Reproduction Assessment

Yes, at source level for the missing capability: current main does one outer runFallbackAttempt per candidate and schemas reject retry budget fields. No live TicketPro reproduction was run.

## Solution Assessment

Unclear as proposed: per-candidate retries are plausible for opaque proxy pools, but provider/model or reason-scoped policy may be safer than ad hoc fallback-list fields.
