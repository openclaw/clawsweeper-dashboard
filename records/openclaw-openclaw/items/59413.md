## Summary

Keep open. Current main and the latest shipped release still provide auth-profile rotation and cooldown controls, but not a retry-before-fallback budget scoped to a fallback candidate or provider/model route; the related retry PR at https://github.com/openclaw/openclaw/pull/55118 was closed unmerged and the discussion identifies this remaining gap.

## What This Changes

Request configurable retry attempts before falling from a specific provider/model or fallback candidate to the next model, mainly for pooled proxy providers with transient 502/timeouts.

## Best Possible Solution

Add a backward-compatible retry policy at the provider/model or fallback-candidate boundary, preserve current single-attempt defaults, and document the operator-facing config with focused runtime and schema coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows a focused reproduction path: configure a primary plus fallback, make the primary throw timeout-classified failures, and assert the primary run function is called once before the fallback is attempted.

## Solution Assessment

Yes. A scoped provider/model or fallback-candidate retry budget is the narrow maintainable direction; increasing global retries would not handle heterogeneous chains where pooled proxy and direct providers need different retry budgets.
