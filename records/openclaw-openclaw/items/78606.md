## Summary

Keep this PR open, but it is not merge-ready. The privacy goal is useful, but current main and v2026.6.9 both implement and document unresolved-origin fallback to approver DMs; the PR changes that shipped contract to an empty target plan that also bypasses the existing failed-delivery notice path.

## What This Changes

The PR narrows shared native approval planning so origin-preferred approvals without a resolved origin no longer fall back to approver DMs, and updates a regression test, Discord docs, and CHANGELOG.md.

## Best Possible Solution

Keep the shipped DM fallback for existing `target: "channel"` users unless maintainers explicitly sponsor strict origin-only routing; if strict routing lands, make it opt-in and preserve a same-chat fallback notice when no native target can be planned.

## Reproduction Assessment

Yes. From source, current main and v2026.6.9 both add approver-DM fallback targets when origin resolution returns null, and the current Discord docs describe that fallback; I did not run a live Discord gateway in this read-only review.

## Solution Assessment

No. The shared planner is the right layer if maintainers want strict origin-only routing, but replacing a shipped fallback with an empty plan is not the safest default because it bypasses the current fallback-notice path.
