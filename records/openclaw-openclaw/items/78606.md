## Summary

Keep this PR open, but not merge-ready: it changes a documented and shipped Discord approval fallback into strict origin-only/no-target behavior, which needs maintainer acceptance and a conflict refresh before merge.

## What This Changes

The PR changes shared native approval delivery so Discord `target: "channel"` approvals do not fall back to approver DMs when the origin target cannot be resolved, and updates tests, Discord docs, and `CHANGELOG.md`.

## Best Possible Solution

Land only after maintainers explicitly accept strict origin-only fail-closed routing; otherwise preserve the shipped fallback and pursue privacy behavior through a separately approved contract.

## Reproduction Assessment

Yes, from source. Current main and `v2026.6.8` both fall back to approver DMs when origin delivery is unavailable, and the current Discord docs describe that fallback; I did not run the runtime in this read-only review.

## Solution Assessment

No, not as an unapproved bug fix. The shared planner is the right layer if maintainers choose strict origin-only routing, but replacing a shipped fallback with no planned target is a compatibility decision.
