## Summary

Keep open: current main still has the claimed origin-miss-to-DM fallback, and the one-line fix is plausible, but this PR should not merge as-is because it lacks real behavior proof and leaves the public Discord fallback docs and changelog out of sync.

## What This Changes

The PR changes native approval delivery planning so an origin-preferred approval with no resolved origin target returns no targets instead of falling back to approver DMs, and updates the core unit test around that case.

## Best Possible Solution

Ship the channel-only no-DM behavior with aligned docs/changelog, focused regression coverage, and real native-runtime proof that unresolved origin delivery does not send approval prompts to approver DMs.

## Reproduction Assessment

Yes, at source level: current main's resolver falls back to approver DMs when preferredSurface is "origin" and originTarget is null, and the current unit test asserts that behavior. I did not run tests because this review is read-only.

## Solution Assessment

Unclear as submitted: the code change is a narrow way to stop the fallback, but the mergeable solution also needs docs/changelog alignment and real behavior proof. Maintainers should also confirm whether the existing documented Discord fallback is intentionally retired.
