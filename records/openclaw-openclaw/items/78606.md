## Summary

Keep open: the no-DM intent is valid, but this patch can leave origin-only Discord approvals with no visible approval path because an empty native plan skips both native delivery and the existing same-chat manual notice.

## What This Changes

The PR changes native approval planning so origin-preferred Discord channel delivery no longer falls back to approver DMs when no origin target resolves, and updates planner tests, Discord docs, and the changelog.

## Best Possible Solution

Keep channel/origin-only approvals fail-closed to approver DMs, but ensure a same-chat manual approval notice or unsuppressed local prompt remains whenever native planning resolves no delivery target.

## Reproduction Assessment

Yes, at source level: `target: "channel"` maps to origin delivery, current main falls back to approver DMs when origin resolution returns null, and the PR’s empty-plan behavior follows directly from the changed guard. I did not run tests in this read-only review.

## Solution Assessment

No: the one-line planner guard prevents the unwanted DM fallback, but it also creates an empty plan that bypasses the current manual fallback notice. The safer fix preserves no-DM routing while keeping same-chat/manual approval instructions visible.
