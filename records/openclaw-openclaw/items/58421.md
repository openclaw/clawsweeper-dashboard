## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.95

Full review comments:

- **[P1] Gate resetSession to full registration mode:** `src/plugins/registry.ts:3048-3050`
  - body: This spread is inside `registrationCapabilities.capabilityHandlers`, and current main sets that flag for `full`, `discovery`, and `tool-discovery`. When `sessions.reset` is present, discovery-mode plugin code receives a session-mutating helper even though the docs and PR body say it is full-mode only; gate this helper on `registrationMode === "full"` and cover discovery/tool-discovery absence.
  - confidence: 0.96
- **[P1] Remove the direct reset-handler sessions.changed emit:** `src/gateway/server-methods/sessions.ts:2063-2066`
  - body: `performGatewaySessionReset` now emits a lifecycle event that the gateway forwards as `sessions.changed`, but the RPC handler still sends a direct change event. The latest direct payload also fails `check-prod-types` because `emitSessionsChanged` requires `reason`, so remove this direct emit and let the lifecycle path own reset notifications.
  - confidence: 0.95
