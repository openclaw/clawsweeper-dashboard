## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Gate resetSession to full mode:** `src/plugins/registry.ts:3048-3050`
  - body: This helper is added inside the `capabilityHandlers` block, and current main sets that block true for `full`, `discovery`, and `tool-discovery`. In those discovery modes a plugin can now receive `resetSession` and call `performGatewaySessionReset`, which violates the read-only discovery contract and can mutate session state; gate it on `registrationMode === "full"` and cover discovery/tool-discovery absence.
  - confidence: 0.93
- **[P1] Keep reset notifications on one path:** `src/gateway/server-methods/sessions.ts:2063-2066`
  - body: `performGatewaySessionReset` now emits a session lifecycle event, and the gateway runtime forwards lifecycle events as `sessions.changed`. This direct emit means one successful `sessions.reset` RPC can send two `sessions.changed` updates; pick one path and add exactly-once coverage for gateway and plugin reset callers.
  - confidence: 0.9
