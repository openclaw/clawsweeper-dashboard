## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Gate resetSession behind explicit plugin authority:** `src/plugins/registry.ts:3177-3179`
  - body: This adds a public plugin SDK method that can rotate gateway sessions, but the only gate is whether the gateway exposes `sessions.reset`. That gives every full plugin the same admin-scoped reset authority with no manifest, trust, or capability contract, so external plugins can mutate session state beyond an approved boundary.
  - confidence: 0.89
- **[P1] Guard late resetSession calls against inactive registries:** `src/plugins/registry.ts:3179-3199`
  - body: `resetSession` is made late-callable, but the method body does not check `activateGlobalSideEffects` or whether this record still belongs to the active loaded registry before calling `performGatewaySessionReset`. A plugin that captured the API can reset sessions after its registry is retired or side effects are disabled, unlike adjacent late-callable APIs such as `sendSessionAttachment` and `emitAgentEvent`.
  - confidence: 0.91
