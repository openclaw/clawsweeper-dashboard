This PR adds optional full-mode plugin SDK `api.resetSession(key, reason?)` support, delegates resets through the gateway reset service, documents the helper, updates SDK API hashes, and adds focused plugin contract tests.

## Best Possible Solution

Expose `api.resetSession` only after reset notifications have one canonical event path, the full-plugin trust boundary is explicitly accepted, and the PR includes redacted real plugin-to-gateway proof.

No. The SDK helper direction is plausible, but the current implementation should not merge until reset event delivery is single-sourced and maintainers accept the full-plugin session reset boundary.
