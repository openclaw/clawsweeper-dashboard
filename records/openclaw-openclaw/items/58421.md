Full review comments:

- **[P1] Make resetSession survive plugin registration close:** `src/plugins/registry.ts:3179`
  - body: This adds `resetSession` to the plugin API, but the loader wraps plugin APIs in a guard that returns `undefined` after `register` closes unless the method is listed as late-callable in `src/plugins/api-lifecycle.ts`. Since this PR does not update that policy, a plugin that captures `api.resetSession` for the advertised runtime use will not receive the promised `Promise<PluginResetSessionResult>` after registration; either mark it late-callable with stale-registry tests or keep it out of the runtime contract.
  - confidence: 0.9
- **[P1] Gate resetSession behind an explicit authority contract:** `src/plugins/registry.ts:3195-3199`
  - body: `sessions.reset` is an `operator.admin` gateway method, but this helper delegates to it for any full plugin whenever the gateway exposes the method. That lets plugin code rotate arbitrary session state without a manifest/trust/config boundary, so the PR needs an explicit maintainer-approved authority model or a narrower scoped/bundled-only API before this becomes public SDK surface.
  - confidence: 0.86
