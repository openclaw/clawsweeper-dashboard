## Summary

Keep open. The PR is a useful additive plugin SDK feature, but the current head still exposes a session-mutating helper through the broad capability-handler surface used by discovery modes, and the external PR still lacks after-fix real behavior proof.

## What This Changes

The PR adds optional plugin SDK `api.resetSession(key, reason?)` support, gateway reset in-flight/config snapshot changes, lifecycle broadcast wiring, docs/changelog/API-baseline updates, and focused contract tests.

## Best Possible Solution

Expose the helper only in full native plugin registration, add discovery and tool-discovery absence coverage, preserve one reset notification path, and require redacted real behavior proof plus maintainer approval of the SDK boundary.

## Reproduction Assessment

Yes. Source inspection is enough: current main makes `capabilityHandlers` true for discovery/tool-discovery, and the PR head places `resetSession` inside that spread.

## Solution Assessment

No for the current patch. The direction is reasonable, but the implementation should full-mode-gate the mutating helper and add discovery/tool-discovery absence tests before merge.
