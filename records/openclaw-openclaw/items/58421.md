Keep open. Current main still lacks the proposed plugin reset API, but the latest PR head is not merge-ready because exact-head lint is failing and the mutating helper is exposed through discovery/tool-discovery registration modes instead of full mode only.

The PR adds optional plugin-facing `api.resetSession(key, reason?)`, exports its result type, wires it to the gateway reset service, updates reset internals/events, and adds docs, changelog, SDK baseline, and focused tests.

## Best Possible Solution
