## What This Changes

This PR adds a full-mode, capability-gated plugin SDK `api.resetSession(key, reason?)` helper with reset-service in-flight/config-snapshot changes, lifecycle event emission, docs/changelog updates, SDK API baseline refresh, and focused tests.

## Best Possible Solution

Expose the helper only if maintainers approve full-plugin session mutation, then route reset success through exactly one `sessions.changed` path that preserves reason semantics and back it with real plugin proof.

## Reproduction Assessment

Yes for the PR defect: PR-head CI reports TS2353 at the changed `sessions.reset` broadcast line, and source inspection confirms the helper still requires `reason`. No real end-to-end plugin behavior has been reproduced because the PR lacks real plugin proof.

## Solution Assessment

No. The feature direction may be acceptable, but the current patch is not the best merge shape until it fixes the reset event path, preserves reason semantics, supplies real proof, and gets boundary acceptance.
