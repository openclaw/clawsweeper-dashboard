## Summary

Keep open: current main still has a source-visible path where inline Control UI HEIC/HEIF attachments can reach the current-run provider handoff without JPEG normalization, and #50320 is already the open implementation candidate.

## What This Changes

Normalize HEIC/HEIF Control UI chat attachments to JPEG before the current-run `activeSession.prompt(..., { images })` provider call.

## Best Possible Solution

Land #50320 or a focused replacement that converts HEIC/HEIF chat attachments to JPEG before the active prompt call, rechecks converted image limits, preserves non-HEIC behavior, and adds gateway attachment regression coverage.

## Reproduction Assessment

Yes. The reporter gave concrete Control UI HEIC upload steps and hot-patch proof, and current source still preserves inline HEIC MIME through the current-run prompt image path when no prompt refs force later loading.

## Solution Assessment

Yes. Normalizing before the current-run provider send is the narrow fix boundary because persistence/history normalization and other media loaders occur too late for the in-flight prompt image payload.
