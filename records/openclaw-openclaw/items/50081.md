## Summary

Keep open. Current main still allows inline Control UI/WebChat HEIC or HEIF attachments to remain `image/heic`/`image/heif` through the current-run prompt image path, and the related implementation candidate is still open and unmerged in the provided GitHub context.

## What This Changes

Normalize HEIC/HEIF images uploaded through Control UI/WebChat before the current-run OpenAI prompt handoff so they are sent as a supported JPEG image payload.

## Best Possible Solution

Land #50320 or a focused replacement that converts inline HEIC/HEIF chat attachments to JPEG before they reach `activeSession.prompt`, rechecks converted size against image limits, preserves non-HEIC and offloaded-media behavior, and adds focused parser plus prompt-dispatch regression coverage.

## Reproduction Assessment

Yes. The reporter provided concrete Control UI HEIC upload steps and hot-patch confirmation, and current main statically preserves inline HEIC/HEIF images through the current-run prompt handoff for prompts without image refs.

## Solution Assessment

Yes. Normalizing before the current-run provider call is the right boundary because persistence/history normalization happens too late for the in-flight model request; the remaining maintainer choice is whether to land, split, or replace #50320.
