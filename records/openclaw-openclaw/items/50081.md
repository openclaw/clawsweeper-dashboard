## Summary

Keep open: current main still has a source-visible path where inline Control UI/WebChat HEIC attachments can remain `image/heic` through the current-run prompt handoff, and open PR #50320 already uses closing syntax for this issue.

## What This Changes

Normalize Control UI/WebChat HEIC and HEIF prompt images to JPEG before the current-run provider submission.

## Best Possible Solution

Land #50320 or a focused replacement that converts HEIC/HEIF chat attachments to JPEG before `activeSession.prompt`, enforces post-conversion image limits, preserves non-HEIC behavior, and adds gateway attachment regression coverage.

## Reproduction Assessment

Yes, source-reproducible with high confidence. The reporter provided concrete Control UI HEIC upload steps and hot-patch confirmation, and current main still preserves inline HEIC MIME through the prompt image path when no prompt refs force later sanitization.

## Solution Assessment

Yes. Normalizing before the current-run provider call is the narrow fix boundary because persistence/history normalization happens too late for the in-flight prompt image payload.
