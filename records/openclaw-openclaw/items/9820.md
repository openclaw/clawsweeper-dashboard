## Summary

Keep open. Current main still excludes plain JSONL from builtin memorySearch extra paths, while the separate session-transcript JSONL pipeline does not satisfy arbitrary hook-log ingestion from extraPaths.

## What This Changes

The issue asks for JSONL files in memorySearch extra paths to be discovered, indexed, watched, and readable without converting them to Markdown.

## Best Possible Solution

Either implement first-class JSONL extra-path support across admission, line-aware parsing/chunking, watcher/read behavior, docs, and tests, or explicitly document the supported QMD/session-memory/converter/plugin workflow.

## Reproduction Assessment

Yes, at source level. Current main still filters builtin memory extra paths to Markdown or enabled multimodal files and rejects non-Markdown paths in memory_get; I did not run a live indexer because this review is read-only.

## Solution Assessment

Unclear until maintainers decide the product boundary. Native JSONL support is plausible, but QMD, session-memory, converter, or plugin workflows may be the intended supported path for raw logs.
