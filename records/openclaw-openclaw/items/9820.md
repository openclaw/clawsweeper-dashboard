## Summary

Keep open: current main and v2026.6.8 still exclude ordinary JSONL from builtin memory extra paths, while the existing JSONL path is a separate session-transcript source with its own filtering, redaction, and line mapping.

## What This Changes

The issue requests native JSONL ingestion for memorySearch.extraPaths so raw message log files can be indexed without conversion.

## Best Possible Solution

Adopt one explicit path: implement schema-aware JSONL extraction, redaction, line mapping, watching, retrieval, docs, and tests across builtin memory, or document QMD/session-memory/converter plugin as the supported workflow.

## Reproduction Assessment

Yes, at source level: current main excludes plain JSONL from builtin memory discovery, retrieval, and watching, while session transcript JSONL follows a separate line-mapped parser. I did not run a live repro because this review is read-only.

## Solution Assessment

No, not as the issue body's extension-list-only sketch. The maintainable solution needs a JSONL extraction/redaction/citation contract and consistent updates across admission, indexing, watching, retrieval, docs, and tests, or a documented non-builtin alternative.
