## Summary

Keep open. Current main still excludes arbitrary JSONL files from builtin memory extra-path discovery and memory_get, and existing session-transcript JSONL support is a separate opt-in source rather than native raw-log ingestion.

## What This Changes

The issue requests native JSONL support in builtin memorySearch.extraPaths so raw message hook logs can be indexed without converting them to Markdown.

## Best Possible Solution

Either implement first-class JSONL extra-path support across admission, line-aware parsing/chunking, watcher/read behavior, docs, and tests, or explicitly document the supported QMD/session-memory/converter/plugin workflow.

## Reproduction Assessment

Yes, at source level. Current main filters builtin memory extra paths to Markdown or enabled multimodal files and rejects non-Markdown paths in memory_get; no live indexer run was appropriate for this read-only review.

## Solution Assessment

Unclear until maintainers decide the product boundary. Native JSONL support is plausible, but the maintainable fix is broader than adding an extension check and may instead be QMD, session-memory, converter, or plugin documentation.
