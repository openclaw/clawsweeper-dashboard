## Summary

Keep open. Current main still lacks native JSONL support for builtin `memorySearch.extraPaths`, and the recent maintainer-side discussion already identified this as a product follow-up rather than closeable cleanup.

## What This Changes

Request adds native `.jsonl` ingestion to memory indexing so raw message hook logs in `memorySearch.extraPaths` can be searched without conversion.

## Best Possible Solution

Adopt one documented path: either implement JSONL as a first-class extra-path source across discovery, chunking, watching, exact reads, docs, and tests, or document the supported raw-log workflow outside builtin extra-path indexing.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is an enhancement request. Source inspection confirms current main only admits Markdown plus enabled multimodal extra files, so arbitrary `.jsonl` extra paths remain unsupported.

## Solution Assessment

Unclear. Adding `.jsonl` support is plausible, but the maintainable fix is not just an extension whitelist because line semantics, chunking, `memory_get`, watcher behavior, docs, and the raw-hook-log product boundary all need a coherent decision.
