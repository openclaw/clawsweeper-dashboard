## Summary

Keep open. Current main still does not provide native builtin JSONL ingestion for memorySearch.extraPaths, and the remaining decision is whether raw JSONL logs belong in builtin memory or should be handled through QMD, session memory, a converter, or a plugin workflow.

## What This Changes

Requests native JSONL ingestion for memorySearch.extraPaths so raw message hook logs can be indexed without converting them to Markdown.

## Best Possible Solution

Either implement first-class JSONL extra-path support across admission, line-aware parsing/chunking, watcher/read behavior, docs, and tests, or explicitly document the supported QMD/session-memory/converter/plugin workflow.

## Reproduction Assessment

Yes, at source level. Current main filters builtin memory extra paths to Markdown or enabled multimodal files, rejects non-Markdown memory_get targets, and only has JSONL parsing for the separate session-memory source.

## Solution Assessment

Unclear until maintainers decide the product boundary. Native JSONL support is plausible, but the maintainable fix is broader than adding an extension check and may instead be QMD, session-memory, converter, or plugin documentation.
