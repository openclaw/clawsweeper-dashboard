## Summary

Keep open. Current main and the latest release still exclude ordinary JSONL files from builtin memory extra paths, and the existing JSONL support is limited to the separate opt-in session transcript path with its own filtering, redaction, and line mapping.

## What This Changes

The issue requests native JSONL ingestion for builtin memory search extra paths so raw message logs can be indexed without converting them to Markdown.

## Best Possible Solution

Adopt one canonical path: either implement schema-aware JSONL admission, extraction, redaction, line mapping, watching, retrieval, docs, and tests across builtin memory, or document session memory, QMD, or an external converter/plugin as the supported workflow.

## Reproduction Assessment

Yes, at source level. Current main directly excludes plain JSONL from ordinary memory discovery, retrieval, and watching, while session transcript JSONL follows a separate line-mapped path.

## Solution Assessment

No, not as the proposed extension-list-only change. The maintainable solution needs a JSONL record-to-text and redaction contract plus consistent updates across admission, indexing, watching, retrieval, citations/docs, and tests, or a documented non-builtin alternative.
