## Summary

Keep open. Current main still excludes arbitrary `.jsonl` files from builtin `memorySearch.extraPaths`, and the existing discussion frames this as a maintainer product/API decision rather than a cleanup close.

## What This Changes

Request native `.jsonl` ingestion for memory indexing so raw hook message logs in `memorySearch.extraPaths` can be searched without converting them to Markdown.

## Best Possible Solution

Adopt one documented path: either implement JSONL as a first-class extra-path source across discovery, parsing, watching, reads, docs, and tests, or document QMD/session-memory/converter-plugin as the supported workflow.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is an enhancement request. Source inspection gives high confidence that arbitrary `.jsonl` extra paths are still filtered or rejected on current main.

## Solution Assessment

Unclear. Native JSONL support may be reasonable, but the proposed extension whitelist is not a complete fix without JSONL parsing/chunking semantics, line mapping, `memory_get`, watcher behavior, docs, and tests.
