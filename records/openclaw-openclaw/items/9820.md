- `impact:session-state`: The request concerns persisted conversation/message records becoming searchable memory context.

## Summary

Keep open. Current main and v2026.6.8 still exclude ordinary JSONL from builtin memory extra paths, and existing JSONL support is limited to opt-in session transcript indexing with separate filtering, redaction, and line mapping.

## What This Changes

Add native JSONL ingestion to builtin `memorySearch.extraPaths` so raw message-hook logs can be indexed without converting them to Markdown.

## Best Possible Solution

Adopt one explicit path: implement schema-aware JSONL extraction, redaction, line mapping, watching, retrieval, docs, and tests across builtin memory, or document QMD, session-memory, or a converter/plugin as the supported workflow.

## Reproduction Assessment

Yes, at source level. Current main excludes plain JSONL from builtin memory discovery, retrieval, and watching, while session transcript JSONL follows a separate line-mapped parser.

## Solution Assessment

No, not as the proposed extension-list-only change. The maintainable solution needs a JSONL extraction/redaction/citation contract and consistent updates across admission, indexing, watching, retrieval, docs, and tests, or a documented non-builtin alternative.
