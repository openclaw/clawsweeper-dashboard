## Summary

Keep open. Current main still lacks native JSONL support for builtin `memorySearch.extraPaths`; existing JSONL handling is limited to session transcript indexing, and recent maintainer-side discussion already kept this feature open for that gap.

## What This Changes

Request native `.jsonl` ingestion in the builtin memory indexer so `extraPaths` can index raw message logs without a conversion step.

## Best Possible Solution

Add documented native JSONL handling for builtin `extraPaths` across discovery, parsing/chunking, reads, watcher updates, docs, changelog, and tests, or document the supported converter/QMD/session-memory path.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to configure a `.jsonl` file or directory in `agents.defaults.memorySearch.extraPaths`; current discovery, read, and watcher paths still admit `.md` or enabled multimodal files only.

## Solution Assessment

No. The title's extension-list-only patch would be incomplete; the maintainable solution needs format-aware JSONL parsing plus aligned `memory_get`, watch behavior, docs, changelog, and regression coverage.
