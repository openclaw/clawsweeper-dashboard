## Summary

Keep open. Current main still limits builtin `memorySearch.extraPaths` discovery, exact reads, watcher admission, and docs to Markdown plus enabled multimodal files; the existing JSONL handling is specific to session transcripts, and recent maintainer-side discussion already kept this issue open for the same gap.

## What This Changes

Add native JSONL support so raw message-log files in `memorySearch.extraPaths` can be indexed and searched without converting them to Markdown first.

## Best Possible Solution

Define the supported JSONL ingestion contract, then either implement format-aware support across admission, parsing/chunking, `memory_get`, watcher filtering, docs, changelog, and tests, or document the intended QMD/session/converter workflow.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to configure a `.jsonl` file or directory in `agents.defaults.memorySearch.extraPaths`; current main admits `.md` or enabled multimodal files, not arbitrary JSONL.

## Solution Assessment

No. The extension-list-only proposal is not the best complete fix because JSONL needs format-aware parsing/chunking plus aligned read, watch, backend, docs, changelog, and regression coverage.
