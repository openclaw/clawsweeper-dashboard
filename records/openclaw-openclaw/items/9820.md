## Summary

Keep open. Current main still excludes plain JSONL from builtin memory extra paths and the recent maintainer-side discussion already kept this open; the remaining decision is product scope, not stale cleanup.

## What This Changes

The issue requests native JSONL ingestion for builtin `memorySearch.extraPaths` so raw message-hook logs can be indexed without converting them to Markdown.

## Best Possible Solution

Either add first-class JSONL extra-path support across admission, line-aware chunking, watcher/read behavior, docs, and tests, or document the supported QMD/session-memory/converter workflow as the intended path.

## Reproduction Assessment

Yes, at source level. Current main still filters builtin memory extra paths to Markdown or enabled multimodal files and rejects non-Markdown paths in `memory_get`; I did not run a live indexer because the checkout is read-only.

## Solution Assessment

Unclear until maintainers decide the product boundary. Native JSONL support is plausible, but QMD/session-memory/converter-plugin workflows may be the intended supported alternative for raw logs.
