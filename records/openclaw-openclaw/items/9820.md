## Summary

Keep open. Current main and the latest release still do not support arbitrary JSONL files in builtin `memorySearch.extraPaths`; existing JSONL handling is limited to session transcript indexing, and the requested broader ingestion needs maintainer product/security direction.

## What This Changes

Add native JSONL ingestion for memory indexing so raw message-log files in `memorySearch.extraPaths` can be searched without converting them to Markdown.

## Best Possible Solution

Keep this as the canonical JSONL extraPaths request until maintainers either approve a full JSONL ingestion contract or document the supported QMD/session-memory/converter/plugin path.

## Reproduction Assessment

Yes, at source level: current main filters memory files to Markdown or enabled multimodal files, reads memory_get targets only when they end in `.md`, and reserves JSONL line mapping for session transcripts. I did not run a live gateway scenario because the read-only source path is clear.

## Solution Assessment

No, the proposed extension-list change is not the best complete fix by itself. A maintainable solution needs an approved JSONL parsing/security contract plus coordinated discovery, indexing, watcher, memory_get, docs, and tests, or an explicit documented alternative.
