## Summary

Keep open. Current main still filters builtin memorySearch.extraPaths to Markdown or enabled multimodal files, indexes non-session text through Markdown chunking, rejects non-.md memory_get paths, and documents/tests Markdown-only extra path behavior; the separate session transcript JSONL path does not satisfy arbitrary raw hook logs.

## What This Changes

Add native JSONL ingestion for memorySearch.extraPaths so raw message-hook logs can be indexed and retrieved without converting to Markdown.

## Best Possible Solution

Keep this open until maintainers either implement JSONL extraPaths across discovery, parsing/chunking, watcher, memory_get, docs, and tests, or document the supported alternative workflow.

## Reproduction Assessment

Yes. Source inspection on current main shows JSONL extraPaths are filtered out or rejected while session transcript JSONL uses a separate parser; I did not run a live filesystem repro in the read-only checkout.

## Solution Assessment

Unclear. Native JSONL support may be the right direction, but the proposed extension-list snippet is not sufficient because the maintainable fix must define JSONL parsing, chunk line mapping, watcher/read behavior, and docs/tests or choose a supported alternative.
