## Summary

Keep open. Current main still excludes `.jsonl` from builtin `memorySearch.extraPaths` discovery, `memory_get`, watcher admission, and docs; the JSONL parsing that exists today is scoped to session transcripts, not arbitrary extra-path log files.

## What This Changes

Request native JSONL ingestion for `memorySearch.extraPaths` so raw message-hook logs can be semantically indexed without conversion to Markdown.

## Best Possible Solution

Add documented native JSONL support for builtin `memorySearch.extraPaths` across admission, parsing/chunking, `memory_get`, watcher filtering, docs, changelog, and focused tests; otherwise document the supported alternative workflow.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to configure a `.jsonl` file or directory in `agents.defaults.memorySearch.extraPaths`; current discovery, read, and watcher paths admit `.md` or enabled multimodal files only.

## Solution Assessment

No. The proposed extension-list-only change would be incomplete; the maintainable fix needs format-aware JSONL parsing plus aligned read, watch, docs, changelog, and regression coverage.
