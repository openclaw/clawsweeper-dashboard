## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Defer fast replies until plugins can claim callbacks:** `extensions/telegram/src/bot-core.ts:241`
  - body: This middleware answers any `action:proceed`/`action:status`/`action:recommend*` callback before the normal handler reaches `dispatchTelegramPluginInteractiveHandler`. Plugin interactive callbacks are namespace based, so a plugin that owns the `action` namespace would still receive an OpenClaw final-button toast before its handler can claim the callback; gate the fast path on plugin namespace ownership or move this acknowledgement behind the plugin-claim check.
  - confidence: 0.9
- **[P2] Keep pre-auth fast acknowledgements neutral:** `extensions/telegram/src/bot-core.ts:241`
  - body: The fast middleware runs before `authorizeTelegramEventSender`, and the new unauthorized test expects an unauthorised user to receive `Proceeding...`. That changes unauthorized callbacks from a generic acknowledgement into action-specific feedback; send a neutral fast toast before auth, or defer action-specific text until the handler has authorized the sender.
  - confidence: 0.84
- **[P2] Skip the handler answer after a fast-lane success:** `extensions/telegram/src/bot-handlers.runtime.ts:1984-1988`
  - body: For final-button data, the new middleware answers the callback query and the normal handler now answers the same callback id again with `callbackAckText`. Telegram callback queries are single-use, so a successful fast answer can make the later handler answer stale; carry fast-lane success in the context and skip the second answer while keeping the visible fallback for a failed/stale fast attempt.
  - confidence: 0.82
