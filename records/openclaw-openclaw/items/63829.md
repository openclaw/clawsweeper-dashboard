## Summary

Keep open: current main and the latest stable release still route Memory Wiki through one captured global vault config, while issue comments show this remains a real multi-agent state and security isolation gap rather than cleanup.

## What This Changes

The issue requests per-agent Memory Wiki vault configuration so multi-agent setups can isolate wiki storage instead of sharing one global vault path.

## Best Possible Solution

Define a maintainer-approved Memory Wiki/plugin config isolation contract, then implement vault routing with docs, doctor/upgrade behavior, and two-agent regression coverage across tools, prompt/corpus supplements, CLI, gateway methods, and background jobs.
