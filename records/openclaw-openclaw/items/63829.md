## Summary

Keep open: current main and the latest release still use one process-wide `memory-wiki` vault config, while the issue describes a real multi-agent memory isolation and security gap that now overlaps an open maintainer memory-config PR rather than a merged fix.

## What This Changes

The issue requests per-agent `memory-wiki` vault configuration so multi-agent setups can isolate wiki storage and search instead of sharing one global vault.

## Best Possible Solution

Land or narrow a maintainer-approved memory configuration contract that gives Memory Wiki agent-local vault state by default or explicit per-agent overrides, with doctor migration/security review and regression tests for cross-agent isolation.

## Reproduction Assessment

Yes, source-level reproduction is high-confidence: current main has one memory-wiki plugin config and no agent-level plugin config path, so two agents cannot select separate wiki vault roots through `agents.list[].plugins` or a `per-agent` vault mode.

## Solution Assessment

Unclear as an implementation choice: the requested behavior is valid, but the best solution should be the maintainer-approved memory/plugin config contract rather than a memory-wiki-only special case unless that broader contract is rejected.
