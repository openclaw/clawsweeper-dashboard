## Summary

Keep open: current main and the latest release still use one global memory-wiki vault config, while the issue discussion and source show a real multi-agent memory isolation and security gap with no merged viable PR now owning the fix.

## What This Changes

The issue requests per-agent memory-wiki vault isolation so multi-agent gateways can avoid sharing one wiki vault and search index.

## Best Possible Solution

Define a maintainer-approved memory-wiki/plugin scoping contract with security isolation rules, schema/docs updates, and two-agent regression coverage, then coordinate it with the broader per-agent plugin config tracker.

## Reproduction Assessment

Yes, at source level. Current main and `v2026.6.8` still resolve one memory-wiki config from global `plugins.entries.memory-wiki.config`, the agent schema has no `plugins` override, and memory-wiki write/read surfaces use the same captured vault path.

## Solution Assessment

Unclear. The requested directions are valid, but the best fix needs maintainer choice between a generic per-agent plugin config contract, memory-wiki-owned vault resolution, or another scoped vault model with explicit security and lifecycle semantics.
