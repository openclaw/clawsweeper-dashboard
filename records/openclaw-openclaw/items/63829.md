## Summary

Keep open: current main and v2026.6.6 still use one top-level memory-wiki vault config, while multiple users report real multi-agent isolation pain and the remaining fix needs maintainer product/security review rather than cleanup closure.

## What This Changes

The issue requests per-agent memory-wiki vault isolation so agents on one gateway can keep separate wiki roots, indexes, and search results.

## Best Possible Solution

Keep this as the canonical memory-wiki vault-isolation tracker, then land or replace the open memory-wiki PR with an equivalent scoped vault solution plus docs, two-agent regression coverage, and coordination with the broader per-agent plugin-config issue.

## Reproduction Assessment

Yes, at source level: current main rejects agents.list[].plugins and memory-wiki uses one captured top-level vault config for wiki reads and writes. I did not run a live two-agent gateway in this read-only pass.

## Solution Assessment

Unclear. The requested direction is valid, but maintainers still need to choose whether the best fix is memory-wiki vault templating, a generic per-agent plugin override contract, prompt/corpus supplement scoping, or some combination.
