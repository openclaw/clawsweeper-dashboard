## Summary

Keep open. Current main still captures one top-level memory-wiki config at plugin registration, has no per-agent plugin config or per-agent vault mode, and the active implementation path is still the open PR at https://github.com/openclaw/openclaw/pull/67584 rather than merged behavior.

## What This Changes

The issue requests per-agent memory-wiki vault configuration so agents on one gateway can use isolated wiki vaults instead of one shared global vault.

## Best Possible Solution

Keep this as the memory-wiki vault isolation tracker and land https://github.com/openclaw/openclaw/pull/67584 or an equivalent plugin-owned fix with per-invocation vault resolution, docs, and two-agent regression coverage.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: configure two agents on one gateway and the current schema rejects per-agent plugin config while memory-wiki read/write paths use the single captured top-level vault config.

## Solution Assessment

No current-main solution exists yet. The narrowest maintainable direction appears to be memory-wiki-owned ctx-aware vault resolution or explicit mapping; a generic agents.list[].plugins config surface is broader API/product work.
