## Summary

Keep open. Current main and v2026.5.12 still resolve one global memory-wiki vault config for the gateway, while the focused implementation remains the open PR at https://github.com/openclaw/openclaw/pull/67584 and the broader generic plugin-override API remains tracked separately at https://github.com/openclaw/openclaw/issues/55401.

## What This Changes

Request support for isolating memory-wiki vaults per agent on a shared gateway, either through agent-level plugin config overrides or a per-agent/templated vault mode.

## Best Possible Solution

Keep this as the canonical memory-wiki vault-isolation tracker and land or replace https://github.com/openclaw/openclaw/pull/67584 with a plugin-owned, context-aware vault resolution fix plus docs and two-agent regression coverage; leave the broader generic override API to https://github.com/openclaw/openclaw/issues/55401.

## Reproduction Assessment

Yes, at source level. Configure two agents on one gateway and current main has no valid per-agent memory-wiki plugin config while memory-wiki read/write paths use the single captured top-level vault config.

## Solution Assessment

Unclear. The requested direction is valid, but maintainers still need to choose between the narrow memory-wiki templated-vault approach and the broader agent-level plugin config override contract.
