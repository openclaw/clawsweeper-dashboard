## Summary

Keep open. Current main and the latest release still resolve one memory-wiki vault from the top-level plugin config, agent entries still reject per-agent plugin config, and the active implementation path remains the open PR at https://github.com/openclaw/openclaw/pull/67584 rather than merged behavior.

## What This Changes

Requests per-agent memory-wiki vault isolation so multi-agent gateways can use separate wiki roots and indexes instead of one shared plugin-level vault.

## Best Possible Solution

Keep this as the canonical memory-wiki vault isolation tracker and review, land, or replace https://github.com/openclaw/openclaw/pull/67584 with an equivalent context-aware vault resolution fix plus docs and two-agent regression coverage.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: configure two agents on one gateway and current main has no valid per-agent memory-wiki plugin config while memory-wiki read/write paths use one captured top-level vault config.

## Solution Assessment

Unclear. The narrow memory-wiki-owned vault templating approach in the open PR looks plausible, but maintainers still need to choose how it relates to the broader per-agent plugin override contract tracked in https://github.com/openclaw/openclaw/issues/55401.
