## Summary

Keep open. Current main still treats any non-empty requester-side announce payload as delivered, so a prefix-only completion announce can suppress fallback even though the full child result is already captured in the internal completion event.

## What This Changes

The issue asks OpenClaw to stop silently truncating or dropping sub-agent completion announce bodies delivered back to the requester session.

## Best Possible Solution

Add integrity detection comparing requester-side visible announce output to the captured task_completion result, then fall back to the preserved result when the visible payload is missing or materially incomplete.

## Reproduction Assessment

Yes at source level, but not as a live Telegram replay. A mocked gateway result with a short visible payload and a longer task_completion result follows the current-main branch that returns direct delivery without using the captured fallback.

## Solution Assessment

No; the current implementation is not the best solution because it only falls back on thrown delivery or no visible payload. The narrower maintainable fix is an integrity check that can distinguish complete requester output from prefix-only delivery without changing the subagent API.
