## Summary

Keep open. Current main preserves the child transcript and frozen result, but the requester-side announce path still treats any visible requester-agent output as successful, so a short prefix-only response can be accepted without falling back to the full child result or marking truncation.

## What This Changes

Ensure sub-agent completion announces preserve or signal the full child result instead of delivering a short prefix to the requester session.

## Best Possible Solution

Keep the typed completion handoff, but add deterministic integrity detection for prefix-only or materially shorter announce outputs and fall back to the frozen child result with regression coverage.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? No live Telegram reproduction was established in this read-only review, but source inspection shows a current-main path where a short visible requester payload is accepted as successful without comparing it to the captured child result.

## Solution Assessment

Is this the best way to solve the issue? No; the current implementation is not sufficient because it treats any visible requester-agent output as complete delivery. A safer fix is a narrow integrity check plus fallback to the already-frozen child result, with tests that preserve legitimate concise rewrites.
