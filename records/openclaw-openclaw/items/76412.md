## Summary

Keep open. Current main still has the source-level lossy-success path: a requester-visible prefix is treated as successful delivery, and the open draft PR #76528 is the active implementation candidate rather than a reason to close this issue before merge.

## What This Changes

Report asks OpenClaw to preserve full subagent completion announce bodies, or at least detect truncation, when results are delivered back to requester sessions.

## Best Possible Solution

Land a narrow integrity fallback in subagent announce delivery that detects materially incomplete requester output and sends the already-captured child result, with regression coverage and docs; #76528 is the active candidate for that end state.

## Reproduction Assessment

No live Telegram reproduction was established. Source inspection gives a high-quality reproduction shape: make the requester agent return any short visible payload while the internal task_completion result contains a longer child report, and current main will mark direct delivery successful without fallback.

## Solution Assessment

No; current main is not the best solution because it treats any visible requester output as complete delivery. The narrow maintainable fix is to compare suspicious prefix-only output against the captured child result and fall back only for clearly incomplete delivery, preserving legitimate concise rewrites.
