## Summary

Keep open. Current main still has a source-reproducible lossy-success path for sub-agent completion delivery, and open draft PR #76528 already targets this issue with closing syntax, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

Ensure sub-agent completion announce events deliver or detect the full child result instead of silently accepting prefix-only requester output.

## Best Possible Solution

Land #76528 or an equivalent narrow fix that detects prefix-only requester completion output and falls back to the captured child result while preserving legitimate concise rewrites.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes from source inspection: a dormant completion requester with any visible gateway-agent payload suppresses fallback without checking whether that payload is only a prefix of the captured child result.

## Solution Assessment

Is this the best way to solve the issue? The best direction is the narrow integrity-check plus fallback approach described by #76528; current main is not sufficient because it treats any visible requester output as complete delivery.
