## Summary

Keep open. Current main still has the lossy-success path described here, and open PR #76528 is a draft fix that references this issue with closing intent, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue asks OpenClaw to stop silently truncating or dropping sub-agent completion announce bodies delivered back to the requester session.

## Best Possible Solution

Review and land PR #76528 or an equivalent narrow integrity fallback, then let the issue close after the fix merges.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes at source level: a mocked gateway response with a short visible requester payload and a longer task_completion result follows the current branch that returns direct delivery without fallback.

## Solution Assessment

Is this the best way to solve the issue? Yes for the linked fix direction: compare requester-visible completion output with the captured result and fall back only when the visible output is materially incomplete, while preserving legitimate rewrites.
