Yes, from source inspection: the PR excludes killed sessions from `resumable`, while current `sessions_send` still reaches reactivation for any ended subagent run. I did not run a live killed-session reproduction in this read-only review.

## Solution Assessment

No. The additive field is the right shape for the reported done/failed/timeout problem, but the killed-session branch is not the best fix until runtime send behavior is either preserved in metadata or explicitly rejected.
