## Summary

## Best Possible Solution

Keep the additive metadata approach, but make `resumable` match the actual send contract for all ended session statuses or deliberately change the send contract, docs, and tests together before merge.

## Reproduction Assessment

## Solution Assessment

No, not as submitted. The additive field is the right maintainable shape, but the implementation needs to align the killed-session metadata and model-facing wording with the existing `sessions_send` behavior before this is the best fix.
