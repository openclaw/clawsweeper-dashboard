## Summary

Keep open. The additive `resumable` field is the right fix shape for the linked session-status bug, but the current patch says killed sessions are not resumable while current `sessions_send` reactivation code can restart any ended latest subagent run, including killed rows.

## What This Changes

This PR adds optional `resumable` metadata to gateway and agent-facing session rows, passes it through `sessions_list`, updates model-facing guidance, and refreshes focused tests and prompt snapshots.

## Best Possible Solution

Keep the additive `resumable` field, but make killed-session metadata and prompt text match the chosen `sessions_send` contract; if killed should be non-resumable, enforce that in send/reactivation code with tests before merge.

## Reproduction Assessment

Yes, at source level. Current main exposes terminal-looking statuses through `sessions_list` while `sessions_send` reactivates ended subagent sessions, and the PR-specific killed mismatch is visible from the send/reactivation path.

## Solution Assessment

No, not yet. Additive metadata is the right maintainable direction, but the killed-session contract must be aligned with current send behavior or enforced as a deliberate behavior change before merge.
