## Summary

Keep open: this is a useful, bounded provider-discovery fix attempt, but it is not merge-ready because marker-only self-hosted discovery can still reach live `/models` probes without Authorization and the contributor proof is still mock/unit-only.

## What This Changes

The PR changes self-hosted provider discovery to fall back from `discoveryApiKey` to `toDiscoveryApiKey(apiKey)` and adds tests for concrete keys and non-secret markers.

## Best Possible Solution

Land a provider discovery fix that forwards only concrete discovery credentials and skips marker-only dynamic probes unless an explicit authless-local provider path is documented, covered, and live-proven.

## Reproduction Assessment

Yes, source-level: marker-only `apiKey` with missing `discoveryApiKey` produces `discoveryKey: undefined`, then vLLM/SGLang builders can call `/models` without Authorization. I did not run a live self-hosted endpoint in this read-only review.

## Solution Assessment

No. `toDiscoveryApiKey` is the right primitive for avoiding marker text in bearer headers, but this PR should also stop marker-only dynamic discovery or define and prove an explicit authless-local contract.
