## Summary

Keep open: this PR is a focused candidate for the canonical provider discovery auth bug, but the current diff still builds marker-only self-hosted providers with no discovery credential and lacks live endpoint proof, so it should not merge yet.

## What This Changes

The PR changes self-hosted provider discovery to fall back from a missing `discoveryApiKey` to `toDiscoveryApiKey(apiKey)` and adds tests for discovery-key precedence and marker filtering.

## Best Possible Solution

Land a provider discovery fix that forwards concrete discovery credentials and skips marker-only dynamic probes unless an explicitly supported authless-local path is documented, tested, and live-proven.

## Reproduction Assessment

Yes, source-reproducible: on PR head, marker-only `apiKey` with missing `discoveryApiKey` produces `discoveryKey: undefined`, then vLLM/SGLang builders call `/models` without Authorization. I did not run a live self-hosted endpoint in this read-only review.

## Solution Assessment

No. `toDiscoveryApiKey` is the right primitive for avoiding marker text in bearer headers, but this is not yet the best fix because marker-only discovery should stop before the provider builder or be an explicit authless-local contract with live proof.
