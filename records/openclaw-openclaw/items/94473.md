## Summary

Keep open: the linked provider-discovery bug is real, but this branch is not a safe fix because it would send non-secret marker and placeholder values as bearer tokens during self-hosted `/models` discovery.

## What This Changes

The branch changes self-hosted provider discovery to pass `discoveryApiKey ?? apiKey` into `buildProvider` and adds regression tests for env-backed, marker-backed, and missing-key cases.

## Best Possible Solution

Keep discovery Authorization based only on concrete resolved discovery credentials, skipping marker-only probes when necessary, and add focused coverage plus redacted real endpoint proof.

## Reproduction Assessment

Yes, at source level. The resolver can expose a truthy public marker such as `custom-local` with `discoveryApiKey` undefined, and this PR would pass that marker into the `/models` discovery builder.

## Solution Assessment

No. The PR touches the right self-hosted discovery boundary, but the fallback should preserve the concrete-secret versus marker separation rather than treating markers as bearer tokens.
