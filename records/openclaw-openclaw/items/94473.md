## Summary

Keep open: this remains the active candidate fix for the linked provider-catalog auth issue, but the branch is not merge-ready because marker-only discovery still reaches provider builders with no credential and the supplied proof is script/unit-test evidence rather than real endpoint behavior.

## What This Changes

The PR changes self-hosted provider catalog discovery to fall back from `discoveryApiKey` to `toDiscoveryApiKey(apiKey)` and adds focused tests plus a local proof script.

## Best Possible Solution

Land a credential-aware discovery fix that forwards concrete credentials, skips or explicitly scopes marker-only unauthenticated discovery, and includes redacted live endpoint proof.

## Reproduction Assessment

Yes, at source level: marker-only `apiKey` plus missing `discoveryApiKey` computes no discovery credential, and vLLM/SGLang builders then call `/models` without Authorization. I did not establish a live endpoint reproduction in this read-only review.

## Solution Assessment

No, not yet. `toDiscoveryApiKey` is the right primitive for avoiding marker text in bearer headers, but the patch still needs marker-only policy resolution and live endpoint proof before it is the best fix.
