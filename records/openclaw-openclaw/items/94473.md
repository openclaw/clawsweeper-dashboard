## Summary

Keep open: the rebase fixes the earlier marker-as-bearer regression, but marker-only self-hosted discovery can still call the provider builder with no discovery credential and issue unauthenticated `/models` probes, while the supplied proof is still source-level/mock-only rather than a real endpoint after-fix run.

## What This Changes

The PR imports `toDiscoveryApiKey`, passes `discoveryApiKey ?? toDiscoveryApiKey(apiKey)` into self-hosted provider discovery, and adds tests for discovery credential precedence and marker filtering.

## Best Possible Solution

Land a provider discovery fix that uses concrete discovery credentials when available and otherwise either skips marker-only probes or documents and proves an explicit authless local discovery path.

## Reproduction Assessment

Yes, at source level: `apiKey: "custom-local"` with `discoveryApiKey: undefined` flows through `toDiscoveryApiKey` to `undefined`, then the PR still calls the provider builder and the local model discovery path sends no Authorization header. I did not run a live LiteLLM/vLLM/SGLang endpoint in this read-only review.

## Solution Assessment

No, not yet. Filtering fallback credentials is the right primitive, but the patch still needs a settled marker-only behavior: skip unauthenticated marker-backed probes, or explicitly preserve authless local discovery with live proof.
