## Summary

Keep open: the patch touches the right self-hosted discovery boundary, but its fallback can turn non-secret auth markers into bearer credentials and does not prove the real `/models` symptom is fixed.

## What This Changes

The PR changes self-hosted OpenAI-compatible provider discovery to pass `discoveryApiKey ?? apiKey` into `buildProvider` and adds focused regression tests for that fallback.

## Best Possible Solution

Keep discovery Authorization sourced only from concrete resolved discovery credentials, or skip marker-only discovery, then prove both auth-required and authless local behavior.

## Reproduction Assessment

Yes, at source level. Resolver paths can expose a truthy public marker with `discoveryApiKey` undefined, and this PR would pass that marker into discovery code that builds bearer headers from any non-empty key.

## Solution Assessment

No. The PR touches the right self-hosted discovery boundary, but the fallback bypasses the marker-aware discovery-key contract instead of preserving concrete discovery credentials or skipping marker-only discovery.
