## Summary

## What This Changes

The PR changes self-hosted provider discovery to fall back from missing `discoveryApiKey` through `toDiscoveryApiKey(apiKey)` and adds regression tests for discovery-key precedence and marker filtering.

## Best Possible Solution

Keep discovery credentials concrete-only: pass resolved `discoveryApiKey` or another proven secret, skip or explicitly preserve marker-only authless discovery with tests, and require redacted live endpoint proof before merge.

## Reproduction Assessment

Yes, source-level. Current resolver contracts and tests show an env-ref profile can expose an env var name with no discovery key, and marker-only resolver output still reaches local-model discovery without a concrete key.

## Solution Assessment

No. The helper is plausible for known markers, but this patch can still forward unresolved env-ref names as bearer tokens and does not stop marker-only unauthenticated discovery for the linked symptom.
