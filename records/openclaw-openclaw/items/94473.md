## Summary

Keep open: the PR improves the concrete-credential fallback path, but marker-only self-hosted discovery can still reach provider builders with no discovery credential and the PR still lacks after-fix live endpoint proof.

## What This Changes

The PR changes self-hosted provider discovery to pass `discoveryApiKey ?? toDiscoveryApiKey(apiKey)` into provider builders and adds unit coverage for concrete keys and non-secret markers.

## Best Possible Solution

Land a provider discovery fix that forwards concrete discovery credentials, skips or explicitly documents marker-only dynamic probes, and includes redacted live endpoint proof.

## Reproduction Assessment

Yes, source-reproducible: marker-only `apiKey` plus missing `discoveryApiKey` produces no discovery credential, and the vLLM/SGLang builders can still call `/models` without Authorization. I did not run a live endpoint in this read-only review.

## Solution Assessment

No, not yet. `toDiscoveryApiKey` is the right primitive for avoiding marker text in bearer headers, but the PR still needs marker-only policy resolution and live endpoint proof before it is the best fix.
