## Summary

Keep open: this is a useful fix direction, but the branch still lets marker-only self-hosted discovery build vLLM/SGLang providers without a discovery credential, preserving unauthenticated `/models` probes, and the PR still lacks redacted after-fix live endpoint proof.

## What This Changes

The branch imports `toDiscoveryApiKey`, passes `discoveryApiKey ?? toDiscoveryApiKey(apiKey)` into self-hosted provider builders, and adds marker-matrix tests.

## Best Possible Solution

Forward concrete discovery credentials, skip or explicitly scope marker-only live discovery, preserve authless local behavior, and require focused tests plus redacted endpoint proof.

## Reproduction Assessment

Yes, at source level: marker-only `apiKey` plus absent `discoveryApiKey` still reaches vLLM/SGLang builders and then `/models` discovery without Authorization. I did not run a live LiteLLM, vLLM, or SGLang endpoint in this read-only review.

## Solution Assessment

No. `toDiscoveryApiKey` is the right primitive for avoiding marker text in bearer headers, but this patch still needs marker-only discovery handling and real endpoint proof before it is the best fix.
