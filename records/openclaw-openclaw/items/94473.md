## Summary

Keep this PR open, but it is not merge-ready: the proposed fallback can send non-secret marker strings as bearer credentials during self-hosted `/models` discovery, and the supplied proof is only a mocked expression capture rather than a real endpoint run.

## What This Changes

The PR changes self-hosted provider discovery to pass `discoveryApiKey ?? apiKey` into provider construction and adds tests that expect this fallback.

## Best Possible Solution

Keep discovery Authorization sourced only from concrete resolved discovery credentials, or explicitly skip marker-only dynamic discovery, with focused marker/env/SecretRef/authless tests and redacted live `/models` proof.

## Reproduction Assessment

Yes. Source inspection traces `apiKey` marker values from `discoverOpenAICompatibleSelfHostedProvider` through vLLM/SGLang provider construction into `discoverOpenAICompatibleLocalModels`, where any non-empty key becomes an Authorization header.

## Solution Assessment

No. The PR edits the right self-hosted discovery boundary, but it should preserve the concrete `discoveryApiKey` separation or filter the fallback instead of treating non-secret markers as bearer tokens.
