## Summary

Keep open: this PR targets a real linked provider discovery auth bug, but the proposed fallback can turn non-secret markers or env placeholders into `/models` Authorization values, and the PR body still only supplies focused test output rather than real self-hosted endpoint proof.

## What This Changes

The PR changes self-hosted provider catalog discovery to pass `discoveryApiKey ?? apiKey` into the provider builder and adds regression tests for that fallback.

## Best Possible Solution

Keep discovery Authorization sourced only from concrete resolved discovery credentials, while preserving marker-backed authless behavior or changing the resolver contract with focused tests and live endpoint proof.

## Reproduction Assessment

Yes, at source level. Current main can carry a truthy marker or env-var-name `apiKey` with `discoveryApiKey` undefined, and this PR would send that placeholder through the discovery builder.

## Solution Assessment

No. The fix is in the right surface, but falling back from `discoveryApiKey` to public `apiKey` treats marker values as concrete secrets instead of preserving the resolver contract.
