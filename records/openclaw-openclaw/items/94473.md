## Summary

Keep open: this is a useful partial fix, but the branch still lets marker-only self-hosted discovery invoke providers with no discovery credential, and the submitted proof is unit/simulation-only rather than real endpoint behavior.

## What This Changes

The PR changes self-hosted provider discovery to pass `discoveryApiKey ?? toDiscoveryApiKey(apiKey)` into the provider builder and adds marker-matrix tests.

## Best Possible Solution

Land a credential-aware fix that forwards concrete discovery credentials, skips or explicitly scopes marker-only self-hosted discovery, and includes focused tests plus redacted live endpoint proof.

## Reproduction Assessment

Yes, at source level: marker-only `apiKey` plus absent `discoveryApiKey` still reaches vLLM/SGLang builders and then `/models` discovery without Authorization. I did not run a live LiteLLM, vLLM, or SGLang endpoint in this read-only review.

## Solution Assessment

No. `toDiscoveryApiKey` is the right primitive for avoiding marker text in bearer headers, but this patch still needs marker-only discovery handling and real endpoint proof before it is the best fix.
