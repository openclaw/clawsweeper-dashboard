## Summary

Keep open: the branch has a useful concrete-key fallback, but marker-only self-hosted discovery still reaches provider builders with no credential and the contributor has not supplied live after-fix `/models` proof.

## What This Changes

The PR changes self-hosted provider discovery to fall back through `toDiscoveryApiKey(apiKey)` when `discoveryApiKey` is missing and adds tests for discovery-key precedence and marker filtering.

## Best Possible Solution

Land a provider discovery fix that forwards concrete discovery credentials and skips marker-only dynamic probes unless an explicit authless-local contract is documented and live-proven.

## Reproduction Assessment

Yes, at source level: marker-only `apiKey` with missing `discoveryApiKey` flows through this PR to `buildProvider({})`, and vLLM/SGLang builders then call `/models` without Authorization. I did not run a live LiteLLM/vLLM/SGLang endpoint in this read-only review.

## Solution Assessment

No. `toDiscoveryApiKey` is the right primitive for avoiding marker text in bearer headers, but this is not yet the best fix because marker-only discovery still needs to stop before the provider builder or be an explicit authless-local contract with live proof.
