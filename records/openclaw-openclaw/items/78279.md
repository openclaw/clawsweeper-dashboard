## Summary

Current main already supports the requested Slack DM workflow through the shared agent `message` tool and the bundled Slack plugin, and the proactive Slack DM send fix shipped in `v2026.5.4`. A separate Slack-only DM skill/tool with custom naming or policy behavior belongs in the existing plugin/ClawHub surface, not as a second core tool.

## What This Changes

Request to add Slack direct-message capability so an OpenClaw agent can privately notify team members.

## Best Possible Solution

Keep using the shipped shared `message` tool plus Slack `user:<id>` targets; route any richer Slack DM workflow, name shortcuts, or custom opt-out/rate-limit policy through a plugin/ClawHub helper or the existing narrower policy issues.

## Reproduction Assessment

Not applicable: this is a feature request, not a bug report. Source, docs, tests in the fixing commit, and release provenance show the Slack DM send path is already available on current main and shipped in `v2026.5.4`.

## Solution Assessment

Yes: the shared `message(action="send")` path is the maintainable solution because core owns one message tool while the Slack plugin owns target parsing and delivery. A separate Slack-specific tool would duplicate that boundary unless it is packaged externally as an optional plugin or skill.
