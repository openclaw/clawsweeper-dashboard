## Summary

Keep open. Current main has prompt guidance, tool-shaped-text diagnostics, and proxy `tool_choice: auto` handling, but source and tests still show final assistant text can be delivered without matching search/tool telemetry, so the requested source-verification guarantee is not implemented.

## What This Changes

The issue asks OpenClaw to prevent agents from claiming search-backed property research unless real web/search/tool evidence exists, or to refuse the task.

## Best Possible Solution

Add a core telemetry-backed claim/source verification policy that checks final claims about searches, citations, browser activity, and external-source research against recorded `web_search`, `web_fetch`, browser, MCP, or tool events, then warns or blocks according to maintainer-chosen policy while coordinating with #45049 and #49876.

## Reproduction Assessment

Unclear for the exact property dataset without replaying the reporter's OpenRouter/Google Hotels setup. Source inspection gives a high-confidence missing guarantee: current main can deliver final-answer text without matching web/search/tool evidence.

## Solution Assessment

No, the current mitigations are not a complete solution. Prompt guidance, diagnostics, and `tool_choice: auto` improve observability and tool selection but do not verify that claimed research or source use actually happened.
