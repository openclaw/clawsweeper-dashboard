## Summary

Keep open. Current main still tracks completed compaction but the terminal embedded-run failure path can format mixed timeout plus unavailable-fallback exhaustion through the generic `/new` message when verbose details are off, so the reported cause loss remains source-backed.

## What This Changes

The issue asks OpenClaw to preserve post-compaction timeout and fallback-exhaustion details instead of returning the generic `/new` fallback message.

## Best Possible Solution

Add a narrow, sanitized post-compaction failure formatter that uses completed-compaction state and `FallbackSummaryError` attempts to report timeout and fallback exhaustion without changing reset or fallback policy.

## Reproduction Assessment

Yes, source-level. A focused test can emit a completed compaction event, then reject with a mixed timeout plus unavailable-fallback `FallbackSummaryError` under `resolvedVerboseLevel: "off"`; current formatter paths still resolve that shape to the generic `/new` text.

## Solution Assessment

Yes. The narrow maintainable fix is to plumb completed-compaction state into the existing failure formatter and use structured fallback attempts; auto-resetting sessions or changing global fallback policy is broader than this bug requires.
