## Summary

Keep open. Current main still allows the reported failure shape: completed compaction is tracked in the run, but the final fallback-exhaustion formatter does not use that state and existing coverage still expects the generic `/new` text for mixed fallback failures with verbose details off.

## What This Changes

The issue asks for post-compaction timeout and fallback-exhaustion replies to preserve compaction and per-attempt failure details instead of collapsing to the generic `/new` message.

## Best Possible Solution

Preserve completed-compaction state through the embedded failure path and use structured fallback attempts to produce a compaction-aware timeout/fallback-exhaustion reply with focused regression coverage.

## Reproduction Assessment

Yes, source-level. Mock `runWithModelFallback` to reject with a mixed `FallbackSummaryError` after completed compaction has been observed and run with `resolvedVerboseLevel: "off"`; current tests already prove that mixed fallback summaries format to the generic text.

## Solution Assessment

Yes. The narrow maintainable fix is to plumb completed-compaction state into the existing failure formatter and summarize `FallbackSummaryError.attempts`; session auto-reset or global failover-policy changes would be broader than this bug requires.
