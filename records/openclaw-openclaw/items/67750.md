## Summary

Keep open. Current main still has the narrow source-backed gap: completed compaction is counted, but terminal embedded-run failure formatting does not receive that compaction state, and mixed timeout plus unavailable-fallback exhaustion can still resolve to the generic `/new` message when verbose details are off.

## What This Changes

The issue asks for successful-compaction-then-timeout fallback failures to preserve the compaction, timeout, and fallback-exhaustion cause chain in the user-facing reply.

## Best Possible Solution

Add a narrow sanitized post-compaction failure formatter that combines completed-compaction state with structured fallback attempt categories, without changing reset policy, fallback selection, or global compaction behavior.

## Reproduction Assessment

Yes, source-level. A focused test can emit a completed compaction event, reject the fallback runner with timeout plus unavailable-fallback `FallbackSummaryError` under verbose-off settings, and observe that current main still resolves through the generic failure text path.

## Solution Assessment

Yes for the narrow issue. The maintainable fix is to thread completed-compaction state into the existing failure reply decision and consume structured fallback attempt reasons; auto-resetting sessions or changing global fallback policy is broader than this report requires.
