## Summary

Keep open. Current main still has a source-backed path where a completed compaction followed by mixed timeout and unavailable-fallback exhaustion is formatted through the generic `/new` failure path when verbose details are off.

## What This Changes

The issue asks for embedded-agent failures after successful auto-compaction to preserve the post-compaction timeout and fallback-exhaustion cause instead of returning the generic `/new` fallback.

## Best Possible Solution

Preserve completed-compaction state and structured FallbackSummaryError attempt causes in embedded failure formatting, with focused regression coverage for post-compaction timeout plus unavailable fallback exhaustion.

## Reproduction Assessment

Yes, source-level. A focused test can emit a completed compaction event or set completed-compaction state, then reject with a mixed timeout plus billing FallbackSummaryError under `resolvedVerboseLevel: "off"`; current formatter paths still resolve that class to the generic `/new` text.

## Solution Assessment

Yes. The narrow maintainable fix is to plumb completed-compaction state into the existing failure formatter and use structured fallback attempts; auto-resetting sessions or changing global fallback policy would be broader than this bug requires.
