## Summary

Keep open. Current main still separates completed compaction tracking from terminal embedded-run failure formatting, so a post-compaction timeout with mixed fallback exhaustion can still collapse to the generic `/new` message when verbose details are off.

## What This Changes

The issue asks OpenClaw to preserve post-compaction timeout and fallback-exhaustion context in the user-facing embedded-run failure message instead of returning the generic `/new` fallback.

## Best Possible Solution

Add a narrow, sanitized post-compaction failure formatter that uses completed-compaction state and structured fallback attempts without changing reset or fallback policy.

## Reproduction Assessment

Yes, source-level. Current main can track a completed compaction, then route a mixed timeout plus unavailable-fallback `FallbackSummaryError` through the generic failure formatter when verbose details are off.

## Solution Assessment

Yes. The narrow maintainable fix is to plumb completed-compaction state into the existing failure formatter and consume structured fallback attempts; auto-resetting sessions or changing global fallback policy is broader than this bug requires.
