## Summary

Keep open. Current main still has a source-backed path where successful compaction followed by timeout plus unavailable fallback exhaustion is formatted as the generic `/new` failure when verbose details are off.

## What This Changes

The issue asks for post-compaction timeout and fallback-exhaustion replies to preserve the compaction, timeout, and fallback-cause chain.

## Best Possible Solution

Preserve completed-compaction state and structured fallback-attempt causes in embedded failure formatting, with regression coverage for post-compaction timeout plus unavailable fallback exhaustion.

## Reproduction Assessment

Yes, source-level. A focused test can emit a completed compaction event or otherwise set completed-compaction state, then reject with a timeout plus unavailable-fallback `FallbackSummaryError` under `resolvedVerboseLevel: "off"`; current source still resolves that class to generic text.

## Solution Assessment

Yes. The narrow maintainable fix is to plumb completed-compaction state into the existing failure formatter and use structured fallback attempts; auto-resetting sessions or changing global fallback policy would be broader than this bug requires.
