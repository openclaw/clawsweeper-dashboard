## Summary

Keep open. Current main still tracks completed compaction separately from the terminal embedded-run failure formatter, so a post-compaction timeout with mixed fallback exhaustion can still collapse into the generic `/new` failure text when verbose details are off.

## What This Changes

The issue asks OpenClaw to preserve post-compaction timeout and fallback-exhaustion details instead of returning the generic `/new` failure message.

## Best Possible Solution

Add a narrow sanitized post-compaction failure formatter that combines completed-compaction state with structured fallback attempts, without changing reset, fallback-selection, or global compaction policy.

## Reproduction Assessment

Yes, source-level. A focused test can emit a completed compaction event, then reject with a timeout plus unavailable-fallback `FallbackSummaryError` under verbose-off settings and observe that current `main` still resolves to the generic `/new` text.

## Solution Assessment

Yes. The narrow maintainable fix is to pass completed-compaction state into the existing failure reply decision and consume structured fallback attempt reasons; auto-resetting sessions or changing global fallback policy would be broader than this bug requires.
