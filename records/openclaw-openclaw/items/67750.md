## Summary

Keep open. Current `main` still separates completed-compaction state from the terminal embedded-run failure formatter, so a post-compaction timeout with mixed fallback exhaustion can still end as the generic external-run failure text when verbose details are off.

## What This Changes

The issue asks for post-compaction embedded-run timeouts with exhausted fallbacks to surface a compaction/timeout-aware user message instead of the generic `/new` fallback.

## Best Possible Solution

Add a narrow sanitized post-compaction failure formatter that uses completed-compaction state plus `FallbackSummaryError` attempts to report timeout and fallback exhaustion without changing reset or fallback policy.

## Reproduction Assessment

Yes, source-level. A focused test can emit a completed compaction event, then reject with a timeout plus unavailable-fallback `FallbackSummaryError` under verbose-off settings and observe that current formatter paths still resolve to the generic `/new` text.

## Solution Assessment

Yes. The narrow maintainable fix is to plumb completed-compaction state into the existing failure formatter and consume structured fallback attempts; auto-resetting sessions or changing global fallback policy is broader than this bug requires.
