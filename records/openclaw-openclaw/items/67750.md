## Summary

Keep open. Current main still has a source-backed path where a post-compaction retry can end in mixed timeout plus unavailable fallback exhaustion and be formatted as the generic `/new` failure when verbose details are off.

## What This Changes

The issue requests a compaction/timeout-aware user-facing failure message when an embedded retry times out after successful auto-compaction and fallback models are unavailable.

## Best Possible Solution

Preserve completed-compaction state and `FallbackSummaryError` attempt causes in embedded failure formatting, with focused regression coverage for post-compaction timeout plus unavailable fallback exhaustion.

## Reproduction Assessment

Yes, source-level. Mock a completed compaction event followed by a mixed `FallbackSummaryError` with `resolvedVerboseLevel: "off"`; current formatter and tests show that class of failure still resolves to the generic `/new` text.

## Solution Assessment

Yes. The narrow maintainable fix is to plumb completed-compaction state into the existing failure formatter and use structured fallback attempts; auto-resetting sessions or changing global fallback policy would be broader than this bug requires.
