## Summary

Keep open. Current main still has the reported source-level failure shape: completed compaction is tracked, but a later mixed timeout/billing fallback exhaustion can still fall into the generic external-run failure copy when verbose details are off.

## What This Changes

The issue requests a compaction/timeout-aware embedded-agent failure message after successful auto-compaction when the retried run times out and fallback models are unavailable.

## Best Possible Solution

Preserve completed-compaction state and fallback-attempt causes in embedded reply failure formatting, with focused regression coverage for post-compaction timeout plus unavailable or billing fallback exhaustion.

## Reproduction Assessment

Yes, source-level. Mock `runWithModelFallback` to throw a mixed `FallbackSummaryError` after a completed compaction has been observed with `resolvedVerboseLevel: "off"`; current tests already prove mixed fallback exhaustion formats to the generic `/new` text.

## Solution Assessment

Yes. The narrow maintainable fix is to plumb completed-compaction state into the existing failure formatter and use `FallbackSummaryError.attempts`; auto-resetting sessions or changing global fallback policy would be broader than this bug requires.
