## Summary

Keep open. Current main still has a source-backed path where mixed timeout plus unavailable-fallback exhaustion is collapsed to the generic run-failure text when verbose details are off, and existing regression coverage still pins that generic reply.

## What This Changes

The issue asks OpenClaw to preserve successful-compaction, post-compaction timeout, and unavailable-fallback details in the embedded-agent failure reply instead of returning the generic `/new` fallback.

## Best Possible Solution

Preserve completed-compaction state and fallback-attempt causes in embedded reply failure formatting, with focused regression coverage for post-compaction timeout plus unavailable or billing fallback exhaustion.

## Reproduction Assessment

Yes, source-level. Mock `runWithModelFallback` to throw a mixed `FallbackSummaryError` after a completed compaction path and keep `resolvedVerboseLevel: "off"`; current tests already prove that class of mixed fallback exhaustion formats to the generic `/new` text.

## Solution Assessment

Yes. The narrow maintainable fix is to plumb completed-compaction state into the existing failure formatter and use `FallbackSummaryError.attempts`; auto-resetting sessions or changing global fallback policy would be broader than this bug requires.
