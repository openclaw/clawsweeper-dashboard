## Summary

Keep open. Current main still has a source-backed path where a completed auto-compaction can be followed by a timeout plus exhausted/unavailable fallbacks and end in the generic external-run `/new` message, so the reported cause-loss bug remains valid.

## What This Changes

The issue requests post-compaction embedded-run failures to preserve the successful compaction, retried timeout, and unavailable-fallback cause chain instead of collapsing to the generic `/new` fallback message.

## Best Possible Solution

Add a narrow, sanitized post-compaction failure formatter that uses completed-compaction state plus structured `FallbackSummaryError` attempts to report timeout and fallback exhaustion without changing reset or fallback policy.

## Reproduction Assessment

Yes, source-level. A focused regression can emit a completed compaction event, reject with a mixed timeout plus unavailable-fallback `FallbackSummaryError`, and assert that `resolvedVerboseLevel: "off"` no longer returns the generic `/new` text.

## Solution Assessment

Yes. The narrow maintainable repair is to plumb completed-compaction state into the existing failure formatter and consume structured fallback attempts; auto-resetting sessions or changing fallback selection would be broader policy work than this bug requires.
