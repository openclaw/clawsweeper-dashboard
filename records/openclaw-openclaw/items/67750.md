## Summary

Keep open. Current main still preserves completed compaction state internally but does not use it when a later mixed timeout/unavailable-fallback failure is formatted, so the reported generic `/new` fallback remains reachable.

## What This Changes

The issue requests compaction-aware timeout/fallback messaging for embedded agent runs after successful auto-compaction.

## Best Possible Solution

Carry completed-compaction state into the embedded final-failure formatter and add focused copy for post-compaction timeout plus exhausted or unavailable fallbacks, leaving broader model-switch UX and subagent races to their own reports.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists by making `runAgentTurnWithFallback` reject with a mixed `FallbackSummaryError` after completed compaction while `resolvedVerboseLevel` is off; current tests already prove that mixed fallback exhaustion becomes the generic text.

## Solution Assessment

Yes. A narrow reply-runner state-plumbing and formatter fix with regression tests is the maintainable path; auto-resetting sessions or changing global fallback policy would be broader than this messaging bug.
