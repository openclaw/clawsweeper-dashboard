## Summary

Keep open. Current `main` still tracks completed compaction separately from the terminal embedded-run failure formatter, so a post-compaction timeout plus unavailable-fallback exhaustion can still collapse to the generic `/new` message when verbose details are off.

## What This Changes

The issue asks for post-compaction embedded-agent failures to preserve the successful-compaction plus timeout plus fallback-exhaustion cause chain instead of returning the generic `/new` fallback message.

## Best Possible Solution

Add a narrow, sanitized post-compaction failure formatter that uses completed-compaction state and `FallbackSummaryError` attempts to report timeout and fallback exhaustion without changing reset or fallback policy.

## Reproduction Assessment

Yes, source-level. A focused test can emit a completed compaction event, then reject with a mixed timeout plus unavailable-fallback `FallbackSummaryError` under `resolvedVerboseLevel: "off"`; current formatter paths still resolve that shape to the generic `/new` text.

## Solution Assessment

Yes. The narrow maintainable fix is to plumb completed-compaction state into the existing failure formatter and consume structured fallback attempts; auto-resetting sessions or changing global fallback policy is broader than this bug requires.
