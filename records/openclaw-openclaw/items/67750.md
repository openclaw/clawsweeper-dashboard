## Summary

Keep open. Current main still has the reported failure shape: completed compaction is tracked, but a later mixed timeout plus unavailable fallback summary still reaches the generic external-run failure text when verbose details are off.

## What This Changes

The issue requests compaction/timeout-aware user-facing failure text when a retried embedded run times out after successful auto-compaction and fallback candidates are exhausted or unavailable.

## Best Possible Solution

Preserve completed-compaction state and fallback-attempt causes in embedded reply failure formatting, with focused regression coverage and changelog, while leaving broader model-switch UX and subagent races to their own reports.

## Reproduction Assessment

Yes, source-level. Mock `runAgentTurnWithFallback` so a completed compaction is observed and then a mixed `FallbackSummaryError` with timeout plus unavailable or billing fallback attempts is thrown while verbose details are off; current tests already prove that mixed-cause summaries format to the generic text.

## Solution Assessment

Yes. The narrow maintainable fix is to plumb completed-compaction state into the external-run failure formatter and add regression coverage; auto-resetting sessions or changing global fallback policy would be broader than this messaging bug requires.
