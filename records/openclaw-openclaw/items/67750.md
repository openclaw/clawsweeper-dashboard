Keep open. Current main still has a source-backed path where a completed compaction followed by mixed timeout plus unavailable fallback exhaustion can be formatted as the generic `/new` failure when verbose details are off.

## What This Changes

Requests a compaction/timeout-aware embedded-agent failure message after successful auto-compaction, retried LLM idle timeout, and exhausted unavailable fallbacks.

## Best Possible Solution

Preserve completed-compaction state and structured FallbackSummaryError attempt causes in the embedded failure formatter, with focused regression coverage for post-compaction timeout plus unavailable fallback exhaustion.

## Reproduction Assessment

Yes, source-level. A mocked completed compaction event followed by a mixed FallbackSummaryError with resolvedVerboseLevel off follows the current catch path to the generic `/new` text, and existing tests already pin the generic mixed-failure behavior.

## Solution Assessment

Yes. The narrow maintainable fix is to use the existing completed-compaction counter and structured fallback attempts in agent-runner-execution formatting; auto-resetting sessions or changing fallback selection policy would be broader than this bug requires.
