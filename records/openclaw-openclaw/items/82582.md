The issue requests that slash/native command replies such as `/active-memory status` remain text-only when TTS is configured to speak final replies.

## Best Possible Solution

Land a scoped command-output TTS exclusion in the linked fix PR or a narrow replacement, while preserving configured final TTS for command continuations and ordinary assistant replies.

## Reproduction Assessment

Yes. Source inspection on current main shows terminal command replies are marked for delivery but not as status notices, and visible final payloads still pass through `maybeApplyTtsToReplyPayload` when TTS auto mode is enabled.

## Solution Assessment

Yes for the issue direction, but the safest implementation is a command-output-level marker or equivalent guard rather than a broad command-turn TTS bypass.
