The issue reports that Auto-TTS set to `always` synthesizes slash command/status responses such as `/active-memory status`.

## Best Possible Solution

## Reproduction Assessment

Yes. Source inspection shows `/active-memory status` returns plain text, command delivery does not mark it as command output, and final dispatch applies Auto-TTS whenever the effective mode is enabled.
