## Summary

Keep open: current main still treats slash-command replies as ordinary final replies for Auto-TTS, and the only linked fix PR is closed unmerged.

## What This Changes

Slash-command/status responses such as `/active-memory status` should stay text-only when Auto-TTS is set to `always`.

## Best Possible Solution

Add a central command-response marker for all slash-command reply paths, skip Auto-TTS in final dispatch for that marker, preserve explicit `/tts audio` and `/tts latest`, and cover the direct skill command path with tests.

## Reproduction Assessment

Yes. Source inspection on current main shows `/active-memory status` returns text, command delivery does not mark it as command output, and final dispatch applies Auto-TTS whenever `ttsAuto` resolves enabled.

## Solution Assessment

Yes. A central command-response marker plus one central TTS-dispatch guard is the narrow maintainable fix, but it should include direct skill slash-command replies as well as the normal command handler path.
