## Summary

Keep this issue open: current main still leaves reused `claude-cli` session bindings intact after `AbortError` and non-`session_expired` timeout failovers, and the linked open draft PR should be reviewed rather than closing the issue early.

## What This Changes

Invalidate stored `claude-cli` resume session IDs after abort or non-expired failover failures so the next turn starts from a fresh CLI session.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81247, or an equivalent narrow fix, with regression coverage and redacted real runtime proof for the reused `claude-cli` abort-or-timeout path.

## Reproduction Assessment

Yes. A high-confidence source repro is to seed a stored `claude-cli` binding with an existing transcript, make the resumed turn throw `AbortError` or `FailoverError` reason `timeout`, and observe that current main rethrows without clearing the binding; the issue comments add live log evidence for that path.

## Solution Assessment

Yes. Clearing only failed reused `claude-cli` bindings through the existing session-store helper is the narrow maintainable fix; raising watchdog caps or adding broad fallback would not stop reuse of the same poisoned resume session.
