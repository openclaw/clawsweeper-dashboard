## Summary

Keep this issue open: current main still reuses valid-looking `claude-cli` bindings after `AbortError` or non-`session_expired` timeout failovers, and an open draft PR explicitly closes it, so the issue should wait for that PR or an equivalent fix to land.

## What This Changes

The issue requests invalidating stored `claude-cli` resume session IDs after `AbortError` or `FailoverError` so later turns do not reuse poisoned CLI state.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/81247, or an equivalent narrow fix, with regression coverage and acceptable redacted real-runtime proof for the reused `claude-cli` abort-or-timeout path.

## Reproduction Assessment

Yes. A high-confidence source repro is to seed a stored `claude-cli` binding with an existing transcript, make the resumed turn throw `AbortError` or `FailoverError` reason `timeout`, and observe that current main rethrows without clearing the binding; the issue comments add live log evidence for that path.

## Solution Assessment

Yes. Clearing only failed reused `claude-cli` bindings through the existing session-store helper is the narrow maintainable fix; raising watchdog caps or adding broad fallback would not stop reuse of the same poisoned resume session.
