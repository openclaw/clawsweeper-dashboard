## Summary

Keep this open: current main still has the reported `claude-cli` reuse failure path, and the active fix path is an open draft PR that directly closes this issue.

## What This Changes

The issue asks OpenClaw to invalidate reused `claude-cli` stored session ids after `AbortError` or non-`session_expired` `FailoverError` so the next turn starts fresh.

## Best Possible Solution

Review and land, or replace, https://github.com/openclaw/openclaw/pull/81247 with a focused clear-only fix that preserves the existing `session_expired` retry path and scopes cleanup to failed reused `claude-cli` sessions.

## Reproduction Assessment

Yes. Source inspection on current main plus the live production traces give a high-confidence path: reuse a stored `claude-cli` binding, hit `AbortError` or timeout `FailoverError`, and observe that current main does not clear the binding.

## Solution Assessment

Yes. Clearing the stored reused `claude-cli` binding through `clearCliSessionInStore` before rethrowing abort or non-expired failover is the narrow maintainable fix; broad provider-wide cleanup would be riskier.
