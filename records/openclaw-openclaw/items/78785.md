## Summary

Keep this open: current main still leaves an active `claude-cli` binding in the session store after AbortError and non-`session_expired` FailoverError paths, and the related fix PR is closed unmerged. The reporter added production live-runtime traces after that PR closed, so this is a good narrow repair candidate rather than cleanup-close material.

## What This Changes

The issue requests invalidating the persisted `claude-cli` resume session after aborts or non-`session_expired` failover errors so later turns do not reuse a wedged CLI transcript.

## Best Possible Solution

Add a narrow clear-only path for active reused `claude-cli` bindings on AbortError and session-tainting FailoverError, using the existing store helper with focused regression coverage and no new config option.

## Reproduction Assessment

Yes from source and logs: seed a stored `claude-cli` binding, make the reused turn fail with AbortError or timeout FailoverError, and current main leaves the binding eligible for the next reuse. I did not execute the live Claude path in this read-only review.

## Solution Assessment

Yes: clearing through the existing session-store helper on active reused CLI sessions is the narrow maintainable fix. A broader watchdog configuration change or retry-policy change would be a separate product surface.
