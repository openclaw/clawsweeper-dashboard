## Summary

Keep open. Current main still lacks the proposed `agent.timeline` surface, but the PR branch is a broad XL/dirty implementation with unresolved correctness and security findings, and the canonical trace-schema direction is still tracked separately at https://github.com/openclaw/openclaw/issues/68204.

## What This Changes

The PR branch adds an in-memory agent run-trace store and `agent.timeline` gateway/protocol/client/docs surface, while also adding block reply/chunker, plan-search, sandbox execution, cron critic/playbook memory, simulation-lane, and roadmap changes.

## Best Possible Solution

Use https://github.com/openclaw/openclaw/issues/68204 to settle the canonical trace schema, then land a narrow trace-only PR with store, RPC, protocol schema, generated clients, docs, object-level authorization, redaction, retention/span caps, and focused tests; split planner, sandbox, cron, simulation, and roadmap work into separate PRs.

## Reproduction Assessment

Yes for the PR review issues: the PR diff shows caller-supplied run-id lookup, no timeline ownership check, no hard running/span caps, critic early return before later failure handling, and static-import-only sandbox validation. I did not run tests because this was a read-only review.

## Solution Assessment

No, this PR is not the best way to solve the issue as currently shaped. The maintainable path is a smaller trace-only implementation aligned with the canonical trace-schema issue and hardened before returning run traces to operator clients.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Authorize timeline reads before returning spans:** `src/gateway/server-methods/agent.ts:710-724`
  - body: `agent.timeline` looks up a caller-supplied `runId` and returns `sessionKey` plus raw spans without verifying that the requester owns or started that run. A read-scoped operator client that learns another run id could retrieve another session's trace details, so the handler needs object-level authorization before returning the timeline.
  - confidence: 0.93
- **[P1] Cap running trace state before accepting new runs:** `src/agents/agent-run-trace.ts:80-105`
  - body: Every new run id is inserted into the process-global timeline map, but pruning only age-evicts running timelines and caps terminal timelines. Since agent run ids come from caller-provided idempotency keys, many incomplete runs can accumulate during the retention window; add global or per-session running caps and a span cap.
  - confidence: 0.88
- **[P1] Classify cron failures before critic returns:** `src/cron/isolated-agent/run.ts:797-805`
  - body: The `needs_replan` critic branch returns `status: "ok"` before later fatal-error payload and run-level error classification runs. With critic mode enabled, a failed embedded run can be reported as a successful replan request and skip normal failure handling.
  - confidence: 0.9
- **[P1] Reject dynamic Python import escapes:** `src/agents/sandbox/execution-policy.ts:170-171`
  - body: The sandbox execution validator only checks modules returned by the static import parser. Code such as `__import__("subprocess")` produces no usage entry, so denied modules can bypass the new policy; detect or reject dynamic import primitives before rendering the template.
  - confidence: 0.88
- **[P2] Strip inline comments from import segments:** `src/agents/sandbox/execution-policy.ts:54-59`
  - body: `parseImportSegment` strips aliases but leaves trailing comments in the module name. Valid Python like `import math  # used below` can be parsed with the comment included, which can misclassify or reject otherwise allowed imports.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: Security review needs attention because the PR exposes raw trace data and adds sandbox execution-policy code with bypassable validation.

Concerns:

- **[high] Timeline RPC lacks object-level authorization:** `src/gateway/server-methods/agent.ts:710`
  - body: The new handler fetches by caller-provided `runId` and returns `sessionKey` plus raw spans without verifying the requester owns the run, creating an information-disclosure path for trace metadata and error strings.
  - confidence: 0.93
- **[medium] Running timeline cache lacks hard caps:** `src/agents/agent-run-trace.ts:80`
  - body: Running timelines are stored globally and retained by age without a count or span cap, so repeated distinct idempotency keys can grow in-memory trace state during the retention window.
  - confidence: 0.88
- **[high] Sandbox policy misses dynamic imports:** `src/agents/sandbox/execution-policy.ts:170`
  - body: The new Python execution validator enforces allow/deny lists only for static `import` and `from` statements, leaving dynamic import paths able to load blocked modules unless separately detected or prohibited.
  - confidence: 0.88
