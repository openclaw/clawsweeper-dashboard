## Summary

Keep open. Current main still does not expose an `agent.timeline` RPC or trace timeline schema, so the central contribution is not implemented, but the branch is too broad and introduces concrete security/correctness blockers that need maintainer judgment and a split before merge.

## What This Changes

The PR branch adds an in-memory agent run-trace timeline store and `agent.timeline` gateway/protocol/client/docs coverage, while also changing block reply and chunker behavior and adding unrelated plan-search, sandbox execution, cron critic/procedural-memory, simulation, roadmap, and test updates.

## Best Possible Solution

Settle the canonical trace contract in #68204, then land a narrow trace-only implementation that includes the store, RPC, protocol schema, generated clients, docs, object-level authorization, redaction, retention limits, and focused tests. Keep already-shipped block reply behavior separate and split autonomy/sandbox/cron experiments into their own reviewed PRs.

Yes for the review issues: source inspection on current main shows no `agent.timeline` method/schema, and the PR diff shows the unauthorised `runId` lookup, uncapped running trace state, critic early return, and static-only sandbox import validation. ## Solution Assessment

No, the current PR branch is not the best way to solve the issue. The maintainable path is a smaller trace-only PR aligned with #68204 and hardened before returning run traces to operator clients.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Authorize timeline reads before returning spans:** `src/gateway/server-methods/agent.ts:710-724`
  - body: `agent.timeline` looks up a caller-supplied `runId` and returns `sessionKey` plus raw spans without checking that the requester owns or started that run. A read-scoped operator client that learns another run id could retrieve another session's trace details, so the handler needs object-level authorization before returning the timeline.
  - confidence: 0.93
- **[P1] Cap running trace state before accepting new runs:** `src/agents/agent-run-trace.ts:104-106`
  - body: Every new run id is inserted into the process-global timeline map, but pruning only caps terminal timelines and evicts running timelines by age. Since agent run ids come from caller-provided idempotency keys, many incomplete runs can accumulate for the retention window; add global/per-session running caps and a span cap.
  - confidence: 0.87
- **[P1] Classify cron failures before critic returns:** `src/cron/isolated-agent/run.ts:797-805`
  - body: The `needs_replan` critic branch returns `status: "ok"` before later fatal-error payload and run-level error classification runs. With critic mode enabled, a failed embedded run can be reported as a successful replan request and skip normal failure handling.
  - confidence: 0.9
- **[P1] Reject dynamic Python import escapes:** `src/agents/sandbox/execution-policy.ts:170-171`
  - body: The sandbox execution validator only checks modules returned by the static import parser. Code such as `__import__("subprocess")` produces no usage entry, so denied modules can bypass the new policy; detect or reject dynamic import primitives before rendering the template.
  - confidence: 0.88
- **[P2] Strip inline comments from import segments:** `src/agents/sandbox/execution-policy.ts:54-59`
  - body: `parseImportSegment` strips aliases but leaves trailing comments in the module name. Valid Python like `import math  # used below` is parsed with the comment included, which can misclassify or reject otherwise allowed imports.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The PR introduces security-sensitive trace disclosure and sandbox policy bypass risks that should block merge until hardened.

Concerns:

- **[high] Timeline RPC lacks object-level authorization:** `src/gateway/server-methods/agent.ts:710`
  - body: The new handler fetches by caller-provided `runId` and returns `sessionKey` plus raw spans without verifying the requester owns the run, creating an information-disclosure path for trace metadata and error strings.
  - confidence: 0.93
- **[medium] Running timeline cache lacks hard caps:** `src/agents/agent-run-trace.ts:104`
  - body: Running timelines are stored globally and retained by age without a count or span cap, so repeated distinct idempotency keys can grow in-memory trace state during the retention window.
  - confidence: 0.87
- **[high] Sandbox policy misses dynamic imports:** `src/agents/sandbox/execution-policy.ts:170`
  - body: The new Python execution validator enforces allow/deny lists only for static `import` and `from` statements, leaving dynamic import paths able to load blocked modules unless separately detected or prohibited.
  - confidence: 0.88
