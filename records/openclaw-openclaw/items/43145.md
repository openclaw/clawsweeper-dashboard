## Summary

Keep open. The tracing idea is not implemented on current main, but this XL branch should not merge as-is because it has missing real behavior proof plus source-proven security and correctness blockers in the new timeline endpoint, sandbox validation, cron critic handling, and retained diagnostic storage.

## What This Changes

The branch adds an in-memory agent run trace timeline and `agent.timeline` gateway method, generated Swift protocol models, docs/tests, and additional autonomy/sandbox/cron features including plan search, critic-loop config, simulation lanes, and procedural playbook memory.

## Best Possible Solution

Split the branch into narrow PRs, start with an authorized and redacted trace API that aligns with the broader schema work in https://github.com/openclaw/openclaw/issues/68204, and move unrelated autonomy/sandbox/cron experiments through separate design review.

## Reproduction Assessment

Not applicable for the requested feature as a user bug, but the review blockers are source-reproducible from the PR diff: the new RPC reads by `runId` without ownership checks, the Python validator only checks static imports, and the cron critic return precedes fatal-result classification.

## Solution Assessment

No. A broad trace-plus-autonomy bundle is not the narrowest maintainable path; the safer solution is a small trace contract with authorization/redaction first, then separate product review for plan search, critic loops, sandbox execution, and simulation lanes.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Authorize timeline reads before returning spans:** `src/gateway/server-methods/agent.ts:710-724`
  - body: `agent.timeline` accepts an arbitrary `runId` and returns the stored trace, including `sessionKey` and span details, without checking that the caller owns or is allowed to inspect that run. This exposes cross-session trace data to any client with the method scope if it can learn or guess a run id, so bind traces to the requesting principal or require an elevated/redacted access path before returning them.
  - confidence: 0.94
- **[P1] Reject dynamic imports in Python sandbox validation:** `src/agents/sandbox/execution-policy.ts:170-171`
  - body: `validateSandboxPythonExecution` only enforces allow/deny policy on static `import`/`from` lines returned by `extractPythonImportUsages`. Generated code can still call `__import__("subprocess")` and no usage is checked, so blocked modules bypass the sandbox execution policy; detect/prohibit dynamic import primitives before accepting the rendered code.
  - confidence: 0.92
- **[P1] Classify run errors before returning needs_replan:** `src/cron/isolated-agent/run.ts:797-805`
  - body: The critic-loop `needs_replan` branch returns `status: "ok"` before the later fatal-error payload classification runs. If the embedded agent returned only error payloads, enabling the critic can suppress the normal error result and downstream failure handling; compute the run error outcome first or gate this early return on no fatal run error.
  - confidence: 0.9
- **[P2] Cap running timelines and per-run spans:** `src/agents/agent-run-trace.ts:104-105`
  - body: New timelines and spans are appended to process-global arrays/maps, but only terminal timelines have a count cap. A client that starts many unique in-flight runs, or one run that emits many events, can grow memory for the full running retention window; add global/per-session running caps and a per-timeline span cap or eviction policy.
  - confidence: 0.86
- **[P2] Redact and bound stored cron errors:** `src/cron/procedural-playbook-memory-v0.ts:236`
  - body: The playbook memory layer stores `signal.error` verbatim in `lastError` before writing the JSON state file. Runtime errors can include URLs, tokens, user content, stack traces, or very large strings, so redact and truncate before persistence or store only a classification/hash by default.
  - confidence: 0.87
- **[P2] Strip inline comments from Python import segments:** `src/agents/sandbox/execution-policy.ts:54-59`
  - body: `parseImportSegment` removes aliases but leaves trailing comments in the module token, so valid code such as `import os  # needed` is parsed as a different root and produces the wrong validation result. Strip inline comments before splitting the root module.
  - confidence: 0.83

## Security Review

Status: needs_attention

Summary: The diff introduces concrete security-sensitive issues in the new trace RPC, sandbox execution validation, in-memory retention, and persisted cron diagnostics.

Concerns:

- **[high] Trace RPC exposes cross-run details:** `src/gateway/server-methods/agent.ts:710`
  - body: `agent.timeline` returns timelines by caller-supplied `runId` without object-level authorization and includes sensitive fields such as `sessionKey`, span metadata, and error/note strings.
  - confidence: 0.94
- **[high] Dynamic Python imports bypass policy:** `src/agents/sandbox/execution-policy.ts:170`
  - body: The sandbox Python validator only parses static import lines, so dynamic import primitives can load denied modules without being checked against `sandbox.execution` allow/deny policy.
  - confidence: 0.92
- **[medium] Run trace store lacks hard caps:** `src/agents/agent-run-trace.ts:104`
  - body: Running timelines and span arrays are retained in process memory without a hard count or per-run span limit, leaving a memory growth path for many in-flight run IDs or noisy runs.
  - confidence: 0.86
- **[low] Cron errors are persisted unredacted:** `src/cron/procedural-playbook-memory-v0.ts:236`
  - body: The procedural playbook memory state writes raw `signal.error` strings to disk, which can retain secrets, PII, stack traces, or large attacker-controlled content.
  - confidence: 0.87
- **[low] Decision log hash chain is unauthenticated:** `src/agents/sandbox/simulation-lane.ts:298`
  - body: The simulation lane verifier uses plain SHA-256 over log entries; if this is presented as immutable/tamper-proof audit evidence, an editor can rewrite entries and recompute the chain.
  - confidence: 0.74
