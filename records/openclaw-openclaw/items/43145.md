## Summary

Keep open. Current main still does not implement the proposed `agent.timeline` surface, and the branch has source-proven correctness/security blockers, missing real behavior proof, and overlap with the open trace-contract architecture issue.

## What This Changes

The PR adds an in-memory agent run trace timeline and `agent.timeline` endpoint while also changing callback/chunking behavior, cron critic/playbook memory, sandbox execution helpers, plan search, protocol models, docs, and tests.

## Best Possible Solution

Split this into focused maintainer-reviewed work: first a versioned, authorized, redacted trace contract aligned with https://github.com/openclaw/openclaw/issues/68204, then separate small PRs for callback/chunking, cron, sandbox, planner, and docs changes with their own proof.

Overall confidence: 0.88

Full review comments:

- **[P1] Authorize and redact timeline responses:** `src/gateway/server-methods/agent.ts:715-724`
  - body: `agent.timeline` returns any stored trace by caller-supplied `runId`, including `sessionKey` and raw spans with error, note, tool, provider, and model details. Bind reads to the caller's authorized run/session or require an elevated trace scope, and return a redacted view by default.
  - confidence: 0.88
- **[P1] Classify run errors before critic replan:** `src/cron/isolated-agent/run.ts:797-805`
  - body: The `needs_replan` branch returns `status: "ok"` before later error-payload and `runLevelError` checks run. With critic loop enabled, a failed isolated-agent run can be reported as a successful replan request and skip normal failure handling.
  - confidence: 0.9
- **[P1] Reject dynamic imports in sandbox validation:** `src/agents/sandbox/execution-policy.ts:170-171`
  - body: The validator enforces allow/deny lists only on static imports from `extractPythonImportUsages`. Python code using dynamic import primitives such as `__import__("subprocess")` can bypass the policy and still be rendered for execution.
  - confidence: 0.88
- **[P2] Cap retained running traces and spans:** `src/agents/agent-run-trace.ts:104-106`
  - body: Running timelines and per-run spans have retention but no hard count or span limit. A client that starts many distinct or long-running runs can grow the process-global trace store until retention catches up.
  - confidence: 0.81
- **[P2] Redact stored cron playbook errors:** `src/cron/procedural-playbook-memory-v0.ts:236`
  - body: Persisting `signal.error` verbatim can write provider URLs, identifiers, tokens, stack traces, or user content into playbook memory state. Redact and bound this string before saving it.
  - confidence: 0.84
- **[P3] Strip inline comments when parsing imports:** `src/agents/sandbox/execution-policy.ts:54-59`
  - body: `parseImportSegment` strips aliases but not trailing comments, so valid code like `import os  # needed` is parsed as a different root and misclassified instead of matching the configured import policy.
  - confidence: 0.78
- **[P3] Avoid presenting unkeyed log hashes as immutable:** `src/agents/sandbox/simulation-lane.ts:376-387`
  - body: The simulation-lane log chain uses plain SHA-256, so anyone who can edit the log can recompute the chain. If this is only non-adversarial integrity checking, avoid wording/tests that imply immutable audit semantics; otherwise use a keyed MAC or signature.
  - confidence: 0.74

## Security Review

Status: needs_attention

Summary: The diff introduces trace exposure, sandbox-validation, in-memory-retention, persisted-error, and hash-chain semantics risks that need fixes before merge.

Concerns:

- **[high] Trace endpoint exposes unredacted run data:** `src/gateway/server-methods/agent.ts:715`
  - body: `agent.timeline` is read-scoped and returns traces by `runId` without caller/session binding or redaction, exposing session keys and raw error/tool metadata to any authorized reader that can obtain a run id.
  - confidence: 0.88
- **[high] Sandbox policy can be bypassed with dynamic imports:** `src/agents/sandbox/execution-policy.ts:170`
  - body: The Python execution validator checks only static import statements, so dynamic import primitives can evade deny/allow lists for blocked modules.
  - confidence: 0.88
- **[medium] Trace store can grow without hard running/span caps:** `src/agents/agent-run-trace.ts:104`
  - body: The process-global trace store has no hard cap for running timelines or per-run spans, allowing many distinct or long-running runs to accumulate memory until retention catches up.
  - confidence: 0.81
- **[medium] Cron playbook memory persists raw error text:** `src/cron/procedural-playbook-memory-v0.ts:236`
  - body: `signal.error` is stored directly in JSON state, which can retain secrets, private identifiers, stack traces, or oversized strings from upstream failures.
  - confidence: 0.84
- **[low] Unkeyed hash chain can imply stronger integrity than it has:** `src/agents/sandbox/simulation-lane.ts:376`
  - body: The simulation lane hashes log entries with plain SHA-256; anyone who can edit the log can recompute the chain, so the code and wording should avoid audit-strength immutability claims unless keyed authentication is added.
  - confidence: 0.74
