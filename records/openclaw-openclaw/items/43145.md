## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P1] Authorize timeline reads before returning spans:** `src/gateway/server-methods/agent.ts:710-724`
  - body: `agent.timeline` looks up any caller-supplied `runId` and returns `sessionKey` plus the full span list without checking that the requesting client owns that run. A paired read client that learns another run id can read tool names, provider/model metadata, and free-form error/note fields, so bind traces to the caller/session or require an elevated scope and redact the response.
  - confidence: 0.9
- **[P1] Classify run errors before returning critic replan:** `src/cron/isolated-agent/run.ts:797-805`
  - body: The new critic `needs_replan` branch returns `status: "ok"` before this function computes `runLevelError` and fatal error payloads. With critic loop enabled, an embedded-agent run that only produced error payloads can be reported as ok and skip normal cron failure handling, alerts, and playbook signals; move error classification before the critic return or gate the critic branch on no fatal error.
  - confidence: 0.88
- **[P1] Reject dynamic Python imports in sandbox validation:** `src/agents/sandbox/execution-policy.ts:170-171`
  - body: `validateSandboxPythonExecution` only validates modules found by the static `extractPythonImportUsages` parser. Code such as `__import__("subprocess")` produces no parsed import and can bypass the deny lists, so the validator needs to detect or prohibit dynamic import primitives before accepting generated code.
  - confidence: 0.86
- **[P2] Cap running trace timelines and span growth:** `src/agents/agent-run-trace.ts:104-107`
  - body: The trace store caps terminal timelines, but new running timelines are added to a process-global map with no hard running-count cap and no per-timeline span cap. A caller that creates many incomplete runs can hold many timelines and spans until the age cutoff, so add running/per-session and per-run span limits or bounded eviction.
  - confidence: 0.82
- **[P2] Redact cron errors before persisting playbook memory:** `src/cron/procedural-playbook-memory-v0.ts:236`
  - body: The playbook memory layer persists `signal.error` verbatim as `lastError`. Those strings can contain URLs, tokens, user content, or stack details and are also unbounded in length, so redact and truncate before saving to the JSON state file.
  - confidence: 0.82
- **[P2] Strip inline comments from import segments:** `src/agents/sandbox/execution-policy.ts:54-59`
  - body: `parseImportSegment` only removes aliases, so valid Python like `import os  # path ops` is parsed as a module name containing the comment and then rejected as an unallowlisted dependency instead of matching the `os` deny/import policy. Strip inline comments before classifying the segment.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: The diff introduces concrete gateway trace-disclosure, sandbox policy-bypass, memory-retention, and persisted-error risks.

Concerns:

- **[high] Timeline endpoint can disclose other run traces:** `src/gateway/server-methods/agent.ts:710`
  - body: `agent.timeline` returns a trace solely by supplied `runId` and includes session/span details without caller ownership checks or redaction.
  - confidence: 0.9
- **[high] Sandbox execution policy misses dynamic imports:** `src/agents/sandbox/execution-policy.ts:170`
  - body: The new validator checks static imports only, so dynamic import primitives can bypass allow/deny lists for generated Python code.
  - confidence: 0.86
- **[medium] Running trace store is not count bounded:** `src/agents/agent-run-trace.ts:104`
  - body: The in-memory timeline store has no hard cap for running timelines or spans, leaving an avoidable memory-growth path for many incomplete runs.
  - confidence: 0.82
- **[low] Cron playbook memory stores raw errors:** `src/cron/procedural-playbook-memory-v0.ts:236`
  - body: Persisting raw, unbounded error strings can retain secrets, private paths, user content, or large stack traces on disk.
  - confidence: 0.82
