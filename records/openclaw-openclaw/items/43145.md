## Summary

Keep open for human review: current main has not implemented the trace endpoint or retained run-trace store, and this broad external PR still has blocking correctness, security, and real-behavior-proof gaps rather than a cleanup-safe close reason.

## What This Changes

The PR adds an `agent.timeline` trace endpoint and in-memory run timeline store while also changing agent block replies/chunking, plan search, cron critic/playbook memory, sandbox execution helpers, generated protocol models, docs, and tests.

## Best Possible Solution

Use this branch as prior art, but split the work into focused maintainer-reviewed PRs: first a hardened, versioned run-trace contract aligned with `https://github.com/openclaw/openclaw/issues/68204`, then separate plan-search, cron, sandbox, and chat-formatting changes with their own proof and security review.

## Reproduction Assessment

Not applicable as a feature PR, but the blocking review findings are source-reproducible from the PR head: the unbound timeline read, early critic return, static-only sandbox validation, unbounded trace retention, and raw error persistence are visible in the changed source.

## Solution Assessment

No. The current branch is not the best merge unit because it combines an observability feature with unrelated cron, sandbox, planning, docs, protocol, and chat-delivery changes before the trace contract and security boundary are settled.

Full review comments:

- **[P1] Authorize and redact timeline responses:** `src/gateway/server-methods/agent.ts:715-724`
  - body: `agent.timeline` returns any stored trace by caller-supplied `runId`, including `sessionKey` and raw spans with error, note, tool, provider, and model details. Bind reads to the caller’s authorized run/session or require an elevated trace scope, and return a redacted view by default.
  - confidence: 0.87
- **[P1] Classify run errors before critic replan:** `src/cron/isolated-agent/run.ts:797-805`
  - body: The `needs_replan` branch returns `status: "ok"` before later error-payload and `runLevelError` checks run. With critic loop enabled, a failed isolated-agent run can be reported as a successful replan request and skip normal failure handling.
  - confidence: 0.9
- **[P1] Reject dynamic imports in sandbox validation:** `src/agents/sandbox/execution-policy.ts:170`
  - body: The validator enforces allow/deny lists only on static imports from `extractPythonImportUsages`. Python code using dynamic import primitives such as `__import__("subprocess")` can bypass the policy and still be rendered for execution.
  - confidence: 0.88
- **[P2] Cap retained running traces and spans:** `src/agents/agent-run-trace.ts:104-106`
  - body: Running timelines and per-run spans have retention but no hard count or span limit. A client that starts many distinct or long-running runs can grow the process-global trace store until retention catches up.
  - confidence: 0.81
- **[P2] Redact stored cron playbook errors:** `src/cron/procedural-playbook-memory-v0.ts:236`
  - body: Persisting `signal.error` verbatim can write provider URLs, identifiers, tokens, stack traces, or user content into playbook memory state. Redact and bound this string before saving it.
  - confidence: 0.83
- **[P3] Strip inline comments when parsing imports:** `src/agents/sandbox/execution-policy.ts:54-59`
  - body: `parseImportSegment` strips aliases but not trailing comments, so valid code like `import os  # needed` is parsed as a different root and misclassified instead of matching the configured import policy.
  - confidence: 0.79
- **[P3] Avoid presenting unkeyed log hashes as immutable:** `src/agents/sandbox/simulation-lane.ts:376-387`
  - body: The simulation-lane log chain uses plain SHA-256, so anyone who can edit the log can recompute the chain. If this is only non-adversarial integrity checking, avoid wording/tests that imply immutable or tamper-proof audit semantics; otherwise use a keyed MAC or signature.
  - confidence: 0.74

## Security Review

Status: needs_attention

Summary: The diff introduces new trace exposure, sandbox-validation, in-memory-retention, and persisted-error risks that need fixes before merge.

Concerns:

- **[medium] Trace endpoint exposes unredacted run data:** `src/gateway/server-methods/agent.ts:715`
  - body: `agent.timeline` is read-scoped and returns traces by `runId` without caller/session binding or redaction, exposing session keys and raw error/tool metadata to any authorized reader that can obtain a run id.
  - confidence: 0.87
- **[medium] Sandbox policy can be bypassed with dynamic imports:** `src/agents/sandbox/execution-policy.ts:170`
  - body: The Python execution validator checks only static import statements, so dynamic import primitives can evade deny/allow lists for blocked modules.
  - confidence: 0.88
- **[medium] Trace store can grow without hard caps:** `src/agents/agent-run-trace.ts:104`
  - body: Running timelines and per-run spans have retention but no hard count or span limit, creating a memory-growth path during repeated incomplete or long-running runs.
  - confidence: 0.81
- **[low] Cron memory persists raw error text:** `src/cron/procedural-playbook-memory-v0.ts:236`
  - body: The new playbook memory layer stores raw error strings to disk without redaction or length limits, which can retain secrets or private user/provider details.
  - confidence: 0.83
- **[low] Simulation log hash chain is not authenticated:** `src/agents/sandbox/simulation-lane.ts:456`
  - body: The decision log uses unkeyed SHA-256 hashes; this can detect accidental edits but does not prove authenticity if an attacker can rewrite the log and recompute hashes.
  - confidence: 0.72
