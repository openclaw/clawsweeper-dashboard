Full review comments:

- **[P1] Authorize and redact timeline responses:** `src/gateway/server-methods/agent.ts:715-724`
  - body: `agent.timeline` returns any stored trace by caller-supplied `runId`, including `sessionKey` and raw spans with errors, notes, tools, provider, and model details. Bind reads to the caller's authorized run/session or require an elevated trace scope, and return a redacted view by default.
  - confidence: 0.88
- **[P1] Classify run errors before critic replan:** `src/cron/isolated-agent/run.ts:797-805`
  - body: The `needs_replan` branch returns `status: "ok"` before later error-payload and run-level error checks run. With critic loop enabled, a failed isolated-agent run can be reported as a successful replan request and skip normal failure handling.
  - confidence: 0.9
- **[P1] Reject dynamic Python imports in sandbox validation:** `src/agents/sandbox/execution-policy.ts:170-171`
  - body: The validator enforces allow/deny lists only on static imports returned by `extractPythonImportUsages`. Python code using dynamic import primitives such as `__import__("subprocess")` can bypass the policy and still be rendered for execution.
  - confidence: 0.88
- **[P2] Cap retained running traces and spans:** `src/agents/agent-run-trace.ts:74-79`
  - body: Running timelines have only time-based retention and no hard count cap, and individual timelines do not cap span growth. A client that starts many distinct or long-running runs can grow the process-global trace store until retention catches up.
  - confidence: 0.81
- **[P2] Redact stored cron playbook errors:** `src/cron/procedural-playbook-memory-v0.ts:236`
  - body: Persisting `signal.error` verbatim can write provider URLs, identifiers, tokens, stack traces, or user content into playbook memory state. Redact and bound this string before saving it.
  - confidence: 0.84
- **[P3] Strip inline comments when parsing imports:** `src/agents/sandbox/execution-policy.ts:54-59`
  - body: `parseImportSegment` strips aliases but not trailing comments, so valid code like `import os  # needed` is parsed as a different root and misclassified instead of matching the configured import policy.
  - confidence: 0.78
- **[P3] Avoid presenting unkeyed log hashes as immutable:** `src/agents/sandbox/simulation-lane.ts:456-457`
  - body: The simulation-lane log chain uses plain SHA-256, so anyone who can edit the log can recompute the chain. If this is only non-adversarial integrity checking, avoid wording/tests that imply immutable audit semantics; otherwise use a keyed MAC or signature.
  - confidence: 0.74

## Security Review

Status: needs_attention

Summary: The diff introduces concrete trace exposure, sandbox-validation, in-memory-retention, persisted-error, and hash-chain semantics risks that need fixes before merge.

Concerns:

- **[high] Trace endpoint exposes unredacted run data:** `src/gateway/server-methods/agent.ts:715`
  - body: `agent.timeline` is read-scoped and returns traces by `runId` without caller/session binding or redaction, exposing session keys and raw error/tool metadata to any authorized reader that can obtain a run id.
  - confidence: 0.88
- **[high] Sandbox policy can be bypassed with dynamic imports:** `src/agents/sandbox/execution-policy.ts:170`
  - body: The Python execution validator checks only static import statements, so dynamic import primitives can evade deny/allow lists for blocked modules.
  - confidence: 0.88
- **[medium] Trace retention lacks hard caps:** `src/agents/agent-run-trace.ts:74`
  - body: Running timelines and per-run spans are not hard-capped, creating a memory growth path for many distinct or long-running traces within the retention window.
  - confidence: 0.81
- **[medium] Cron memory persists raw error text:** `src/cron/procedural-playbook-memory-v0.ts:236`
  - body: Raw cron error strings can include secret-shaped provider URLs, identifiers, user content, or stack traces and are persisted without redaction or bounding.
  - confidence: 0.84
- **[low] Log hash chain is not tamper resistant:** `src/agents/sandbox/simulation-lane.ts:456`
  - body: Plain SHA-256 can catch accidental corruption but does not provide adversarial immutability; either avoid audit-style claims or use a keyed/signature scheme.
  - confidence: 0.74
