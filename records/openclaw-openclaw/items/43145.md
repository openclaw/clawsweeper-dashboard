## Summary

Keep open. Current main does not implement `agent.timeline` or the related run-trace surface, but this PR is too broad, currently conflicting, lacks real behavior proof, and has blocking security and correctness issues in the proposed patch.

## What This Changes

The PR adds an `agent.timeline` gateway endpoint with in-memory agent run traces, plus related plan-search, cron critic-loop, sandbox execution, simulation-lane, docs, generated protocol, and test changes.

## Best Possible Solution

Split this into focused maintainer-reviewed PRs: first a hardened, versioned run-trace contract aligned with https://github.com/openclaw/openclaw/issues/68204, then separate plan-search, cron, and sandbox changes with their own docs, security review, and real behavior proof.

## Reproduction Assessment

Not applicable as a feature PR, but the blocking review findings are source-reproducible from the proposed head: the critic return order, static-only sandbox import validation, raw error persistence, and timeline response shape are visible in the diff.

## Solution Assessment

No. The run-trace idea may be useful prior art, but this broad branch is not the best merge unit until the trace endpoint is hardened, unrelated surfaces are split or justified, and the contributor provides real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Authorize and redact timeline responses:** `src/gateway/server-methods/agent.ts:715-724`
  - body: `agent.timeline` returns any stored trace by caller-supplied `runId`, including `sessionKey` and raw spans with error/note/tool/provider/model details. Bind reads to the caller’s authorized run/session or require a narrower elevated trace scope, and return a redacted view by default.
  - confidence: 0.86
- **[P1] Classify run errors before returning critic replan:** `src/cron/isolated-agent/run.ts:797-805`
  - body: The `needs_replan` branch returns `status: "ok"` before the later fatal error-payload and `runLevelError` classification runs. With critic loop enabled, a failed isolated agent run can be reported as a successful replan request and skip normal failure handling.
  - confidence: 0.9
- **[P1] Reject dynamic imports in sandbox validation:** `src/agents/sandbox/execution-policy.ts:170`
  - body: The validator enforces allow/deny lists only on static imports returned by `extractPythonImportUsages`. Python code such as `__import__("os")` produces no entries here, so blocked modules can bypass the policy and still be rendered for execution.
  - confidence: 0.88
- **[P2] Cap retained running traces and spans:** `src/agents/agent-run-trace.ts:241`
  - body: Running timelines are retained for up to 30 minutes, but there is no hard cap on the number of running timelines or spans per timeline. A client that starts many distinct runs can grow the process-global trace store until retention catches up.
  - confidence: 0.8
- **[P2] Redact stored cron playbook errors:** `src/cron/procedural-playbook-memory-v0.ts:236`
  - body: Persisting `signal.error` verbatim can write provider/tool URLs, identifiers, tokens, stack traces, or user content into `playbook-memory-v0` state. Redact and bound this string before saving it.
  - confidence: 0.83
- **[P3] Strip inline comments when parsing imports:** `src/agents/sandbox/execution-policy.ts:54-60`
  - body: `parseImportSegment` strips aliases but not trailing comments, so valid code like `import os  # needed` is parsed as a different root and misclassified instead of matching the configured import policy.
  - confidence: 0.79

## Security Review

Status: needs_attention

Summary: The patch introduces new trace exposure, sandbox-validation, in-memory retention, and persisted-error risks that need fixes before merge.

Concerns:

- **[medium] Trace endpoint exposes unredacted run data:** `src/gateway/server-methods/agent.ts:715`
  - body: `agent.timeline` is read-scoped and returns traces by `runId` without caller/session binding or redaction, exposing session keys and raw error/tool metadata to any authorized reader that can obtain a run id.
  - confidence: 0.86
- **[medium] Sandbox policy can be bypassed with dynamic imports:** `src/agents/sandbox/execution-policy.ts:170`
  - body: The Python execution validator only checks static import statements, so dynamic import primitives can evade deny/allow lists for blocked modules.
  - confidence: 0.88
- **[medium] Trace store can grow without hard caps:** `src/agents/agent-run-trace.ts:241`
  - body: Running timelines and per-run spans have retention but no hard count or span limit, creating a new memory-growth path during repeated incomplete runs.
  - confidence: 0.8
- **[low] Cron memory persists raw error text:** `src/cron/procedural-playbook-memory-v0.ts:236`
  - body: The new playbook memory layer stores raw error strings to disk without redaction or length limits, which can retain secrets or private user/provider details.
  - confidence: 0.83
