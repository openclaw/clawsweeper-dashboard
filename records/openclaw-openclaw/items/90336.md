## Summary

Keep open. Current main and the latest release still allow providerless memory search to degrade into keyword-only results, while this PR is a useful targeted fix but remains compatibility-sensitive because it changes explicit remote embedding-provider failures from fail-open to fail-closed.

## What This Changes

The branch adds an internal memory-core provider requirement, fail-fast guards for required-provider search and sync, and focused tests while keeping the public resolved memory-search config shape unchanged.

## Best Possible Solution

Keep the fail-fast behavior for truly explicit remote providers, but have maintainers decide whether already-migrated legacy-auto configs should share that behavior and update docs or release-note wording if needed.

## Reproduction Assessment

No live current-main reproduction was established in this read-only review. The source path is clear: current main can catch missing-provider embedding failures and return keyword-only FTS results, and the PR adds simulated regression coverage for that path.

## Solution Assessment

This is an acceptable mitigation and likely the right memory-core guard for explicit providers, not a complete proof that the underlying embedded provider registry race is gone. It becomes the best fix once maintainers accept the fail-closed upgrade behavior or narrow it for legacy-auto migrations.
