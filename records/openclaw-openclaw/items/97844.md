## Summary

Keep open. Current main has the live Copilot `/models` mapper and refreshable-catalog runtime loading, but the bundled Copilot manifest still carries 128k Claude rows and the context/status cache path can still combine live rows with stale static context rows, so I cannot safely close this as implemented-on-main.

## What This Changes

The issue reports that published v2026.6.10 caps GitHub Copilot Claude models at 128k despite live Copilot metadata showing 1M, and asks for a release or narrow workaround.

## Best Possible Solution

Make live GitHub Copilot catalog metadata authoritative for both model display and session context budgeting, while keeping static rows only as a true offline fallback that cannot lower verified live context.

## Reproduction Assessment

## Solution Assessment

No. Adding a temporary `agents.defaults.models.<id>.contextWindow` schema override is not the best fix; the maintainable repair is to stop static fallback metadata from lowering verified live provider metadata.
