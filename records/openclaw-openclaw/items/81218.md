## What This Changes

The issue requests that vLLM/SGLang provider wildcards discover models from explicitly configured self-hosted provider endpoints instead of making those providers manual-only.

## Best Possible Solution

Review and land the linked provider-discovery fix so exact `provider/model` entries stay manual allowlists while `provider/*` visibility queries the configured endpoint through the existing catalog builders.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure `models.providers.vllm` or `models.providers.sglang` together with `agents.defaults.models["provider/*"]`, and the shared helper returns before catalog discovery can query the configured `/models` endpoint.

## Solution Assessment

Yes. The narrow maintainable solution is in the shared self-hosted discovery path, distinguishing wildcard-driven dynamic discovery from exact manual model entries and passing explicit endpoint details into the existing provider builders.
