## Summary

Keep open. Current main still lacks builtin-specific allowlist handling or documentation, and the open linked implementation PR at https://github.com/openclaw/openclaw/pull/79363 uses closing linkage for this issue, so this should remain open until that PR lands or maintainers choose a docs-only policy.

## What This Changes

Request that exec allowlist mode document or optionally auto-allow selected pathless shell builtins so commands like `cd /path && git status` do not always prompt.

## Best Possible Solution

Review and either land https://github.com/openclaw/openclaw/pull/79363 after security/product approval, or reject builtin auto-allow and add focused docs for the builtin limitation and workaround.

## Reproduction Assessment

Yes, source-reproducible. Current main only satisfies segments through allowlist, safeBins, skill auto-allow, or inline-chain handling; a pathless builtin segment such as `cd` cannot match normal allowlist paths and causes a chained command to fail allowlist evaluation.

## Solution Assessment

Unclear. The linked closed-set opt-in `safeBuiltins` direction is plausible, but it is a new exec approval policy knob and should remain under maintainer security/product review; docs-only is the safer fallback if that policy is rejected.
