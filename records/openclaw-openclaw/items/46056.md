## Summary

Keep open. Current main still has the reported pathless-builtin allowlist gap, and an open PR with closing syntax now owns the implementation path, so this issue should close only after that PR is fixed and merged or explicitly abandoned.

## What This Changes

The issue asks for shell builtins such as `cd` to be documented or optionally allowed in exec allowlist mode so chained commands do not unexpectedly prompt when later binaries are allowlisted.

## Best Possible Solution

Review and land a corrected version of https://github.com/openclaw/openclaw/pull/79363, or split out a docs-only clarification if maintainers decide not to add the new exec-approval config knob.

## Reproduction Assessment

Yes. The high-confidence reproduction is source-level on current main: pathless builtin segments cannot satisfy ordinary allowlist or safeBins branches, so a chain starting with `cd` remains unsatisfied.

## Solution Assessment

Unclear for the runtime path until maintainers accept the policy. The opt-in `safeBuiltins` direction is plausible, but the open PR needs its security and docs findings resolved before it is the best mergeable solution.
