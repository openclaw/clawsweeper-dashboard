## Summary

Keep open. Current main still has no builtin-specific allowlist/config path for pathless shell builtins, and the open implementation PR at https://github.com/openclaw/openclaw/pull/79363 uses closing linkage for this issue, so the issue should stay open until that PR lands or maintainers choose a docs-only/product-policy resolution.

## What This Changes

The issue requests documentation or an opt-in runtime policy so shell builtins such as `cd`, `pwd`, `export`, and `unset` do not always trigger exec approval prompts under allowlist mode.

## Best Possible Solution

Review and either land the closed-set opt-in implementation in https://github.com/openclaw/openclaw/pull/79363 after its remaining concerns are resolved, or reject that policy and add focused docs explaining builtin behavior and workarounds.

## Reproduction Assessment

Yes. Source inspection on current main shows pathless builtin segments cannot satisfy normal allowlist, safeBins, or skill auto-allow, and shell-chain evaluation returns false on the first unsatisfied segment.

## Solution Assessment

Unclear. The linked closed-set `tools.exec.safeBuiltins` direction is plausible, but it changes exec approval policy and should be decided through the open PR rather than closing this issue early.
