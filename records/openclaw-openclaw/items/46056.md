## Summary

Keep open. Current main still lacks builtin-specific allowlist handling, and the open implementation PR at https://github.com/openclaw/openclaw/pull/79363 uses closing linkage for this issue.

## What This Changes

The issue requests documented or opt-in allowlist handling for pathless shell builtins such as cd/export so allowlisted command chains stop prompting unexpectedly.

## Best Possible Solution

Review and either land the closed-set opt-in implementation in https://github.com/openclaw/openclaw/pull/79363, or reject that policy and add focused docs for the builtin limitation and workaround.

## Reproduction Assessment

Yes, source-reproducible. Current main has no builtin satisfaction branch, pathless segments cannot match ordinary allowlist paths, and shell-chain evaluation fails at the first unsatisfied segment.

## Solution Assessment

Unclear. The linked opt-in tools.exec.safeBuiltins direction is plausible, but it changes exec approval policy and should be decided through the open PR rather than closing this issue early.
