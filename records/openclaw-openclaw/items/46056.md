## Summary

Keep open. Current main still has the pathless shell-builtin allowlist gap, and the open implementation PR at https://github.com/openclaw/openclaw/pull/79363 still references this issue with closing linkage, so the issue should remain open until that PR lands or is closed.

## What This Changes

Allow or document shell builtin handling in exec allowlist mode so commands such as `cd /path && git status` do not unexpectedly require approval when the relevant binaries are allowlisted.

## Best Possible Solution

Review and either land the linked opt-in `tools.exec.safeBuiltins` implementation after policy and proof concerns are resolved, or land a focused docs clarification if maintainers reject the runtime policy change.

## Reproduction Assessment

Yes. Source inspection on current main shows pathless builtin segments cannot satisfy allowlist, safeBins, or skill branches, and shell chains stop at the first unsatisfied segment.

Unclear. The linked opt-in config direction is plausible, but it changes exec approval semantics and should be decided through the open PR rather than by closing this issue early.
