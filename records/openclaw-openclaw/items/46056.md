## Summary

Keep open. Current main and the latest release still lack shell-builtin-aware allowlist handling or docs, and the open linked implementation PR with closing linkage means this issue should remain the canonical tracker until maintainers land or reject that policy change.

## What This Changes

The issue requests documentation or opt-in allowlist support for shell builtins such as `cd` so builtin segments in chained exec commands do not always trigger approval.

## Best Possible Solution

Review and either land https://github.com/openclaw/openclaw/pull/79363 after security/product approval and configured exec-flow proof, or reject builtin auto-allow and add focused docs for the current limitation and workaround.

## Reproduction Assessment

Yes. Source inspection shows builtin segments have no `safeBuiltins` branch and pathless command resolutions cannot match normal allowlist entries, so a command like `cd /tmp && git status` misses on the `cd` segment.

## Solution Assessment

Unclear. A closed-set opt-in builtin allow config is plausible, but it is a new exec approval policy knob; docs-only is the safer fallback if maintainers reject the policy change.
