## Summary

Keep open: the patch catches the localhost:1455 conflict, but the busy-port branch drops the authorization URL before prompting for manual entry, so the target login path cannot complete as written.

## What This Changes

The PR adds an OpenAI Codex OAuth localhost:1455 availability preflight, suppresses local browser opening on conflict, prompts immediate manual entry, mirrors the bundled OpenAI runtime copy, adds regression coverage, and updates the changelog.

## Best Possible Solution

Keep the narrow port preflight, but when automatic opening is suppressed, show or log the authorize URL with manual-open instructions in both OpenAI OAuth wrappers and cover that behavior in the regression test.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path for the original bug by occupying 127.0.0.1:1455 and running the OpenAI Codex OAuth login, and it also shows the PR defect because the busy-port onAuth branch returns before any code exposes event.url.

## Solution Assessment

No, not as written. The local port preflight is the right narrow direction, but the solution must preserve a manual authorization URL while still suppressing automatic browser opening.
