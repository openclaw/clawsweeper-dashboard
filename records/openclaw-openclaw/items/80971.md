## Summary

Keep open: this is a focused bug-fix PR with sufficient real terminal proof, but the current patch drops the only authorization URL on the occupied-port path, so a fresh manual OAuth login cannot complete from the prompt it shows.

## What This Changes

The PR adds a localhost:1455 availability preflight to the OpenAI Codex OAuth core wrapper and bundled OpenAI plugin runtime, updates one regression test, and adds a changelog entry.

## Best Possible Solution

Keep the port-conflict preflight, suppress automatic browser opening, and expose the authorize URL with manual-open instructions in both OpenAI OAuth wrappers, backed by regression coverage for URL availability.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: with 127.0.0.1:1455 occupied, the pinned upstream helper still calls onAuth with the authorize URL, and this PR returns before exposing that URL while immediately prompting for manual input.

## Solution Assessment

No, not as written. The preflight is the right narrow approach, but the implementation must preserve a manual authorization URL while still avoiding openUrl on the occupied callback port.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Preserve the manual OAuth URL when the port is busy:** `src/plugins/provider-openai-codex-oauth.ts:243-245`
  - body: When localCallbackAvailable is false, this returns before baseOnAuth can log or expose event.url. The user is immediately asked to paste a code or redirect URL but is never shown the authorize URL, so a fresh busy-port login cannot complete from this prompt. Log or display the URL for manual opening while still skipping openUrl.
  - confidence: 0.93

  - body: The bundled OpenAI runtime copy has the same early return before the SDK OAuth handler can expose the authorization URL. Users going through the bundled plugin would reach the manual prompt with no URL to open, so this copy needs the same manual-URL behavior as the core wrapper.
  - confidence: 0.9
