## Summary

Keep open: the PR is aimed at the right OpenAI Codex OAuth bug, but the current head suppresses the browser opener by returning before the authorization URL is shown anywhere, so the busy-port manual path cannot be completed as written.

## What This Changes

The PR adds an OpenAI Codex OAuth localhost:1455 preflight, busy-port diagnostics, immediate manual prompting, mirrored bundled OpenAI runtime changes, a regression test, and a changelog entry.

## Best Possible Solution

Keep the local port preflight, but when automatic opening is suppressed, show or log the authorize URL with manual-open instructions in both OAuth wrappers and cover that behavior in the regression test.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: occupy 127.0.0.1:1455, run the OpenAI Codex OAuth login, and the PR head will prompt immediately while never exposing the authorize URL needed for manual completion.

## Solution Assessment

No, not as written. The preflight is the narrow maintainable direction, but the busy-port branch must preserve manual URL exposure while avoiding automatic browser opening.
