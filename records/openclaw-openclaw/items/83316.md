## What This Changes

The PR teaches `openclaw update` to redirect package-manager updates to the managed gateway service package root when it differs from the invoking CLI root, adds user-facing logs/dry-run notes, and adds a CLI regression test.

## Best Possible Solution

Land this focused root-selection fix after redacted real managed-service proof or an explicit maintainer proof override, preserving the existing service-layout and update-global helper boundaries.

## Reproduction Assessment

Yes by source inspection: current main uses the invoking CLI package root throughout the package update path, while the existing service layout helper can point at a different managed-service package root. I did not run the destructive live update path in this read-only review.

## Solution Assessment

Yes: the proposed approach is the narrowest maintainable fix because it reuses the existing service-layout summary and only redirects package updates when the service package root differs and is not a source checkout. The remaining blocker is real managed-service proof, not a different implementation direction.
