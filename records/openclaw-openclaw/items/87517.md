## Summary

Keep open: current main and `v2026.6.6` already skip gateway-token generation when a plaintext token resolves, but the production audit hashes do not include redacted changed-path or before/after config proof tying the change to `gateway.auth.token`. Because the report is credential-sensitive and nearby doctor service repair can write gateway-token config, maintainer/security review is still the safest outcome.

## What This Changes

The issue reports that interactive `openclaw doctor --fix` may rotate an already-valid `gateway.auth.token`, disconnecting existing gateway consumers.

## Best Possible Solution

Preserve the current token-preservation contract, obtain redacted changed-path proof, then patch the exact writer with regression coverage for existing-token preservation and intentional token-generation logging.
