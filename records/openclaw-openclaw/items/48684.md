## Summary

Keep open. Current main and the latest release still route debounce bypass through slash-command detection, so documented standalone abort phrases such as `stop`, `abort`, and `wait` remain debounceable; the linked fix PR is closed unmerged.

## What This Changes

Bare abort phrases should bypass inbound message debounce and flush immediately across debounced chat channels.

## Best Possible Solution

Use abort-aware command detection at shared and plugin-local debounce gates, preserve intentional coalescing exceptions, and add focused regression coverage for bare abort phrases.

## Reproduction Assessment

Yes, by source-level reproduction on current main. `shouldDebounceTextInbound({ text: "stop", cfg })` reaches `hasControlCommand("stop")`, which does not consult `ABORT_TRIGGERS`, so the predicate treats the abort phrase as debounceable text.

## Solution Assessment

Yes. Reusing `isControlCommandMessage()` or the plugin-runtime equivalent at debounce boundaries is the narrowest maintainable fix, as long as command authorization and mention gates keep their current narrower semantics.
