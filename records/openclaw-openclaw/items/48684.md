## Summary

Keep open. Current main and the latest release still route shared inbound debounce bypass through slash-command-only detection, and the open draft fix PR at https://github.com/openclaw/openclaw/pull/82873 uses closing syntax for this issue, so the issue should stay open until that PR lands or is replaced.

## What This Changes

The issue asks for bare abort phrases such as `stop`, `abort`, and `wait` to bypass inbound debounce immediately across debounced channels.

## Best Possible Solution

Review and land or replace the focused shared-helper fix, then explicitly include or split the remaining plugin-local debounce gaps before closing this issue.

## Reproduction Assessment

Yes for source reproduction: current main routes bare `stop` through `hasControlCommand()`, which does not consult the abort trigger set, so the shared debounce predicate treats it as debounceable. I did not run a live channel gateway in this read-only review.

## Solution Assessment

Yes, reusing `isControlCommandMessage()` at debounce boundaries is the narrowest maintainable fix because it preserves slash-command handling and adds existing abort-trigger detection. The merge path still needs maintainers to decide whether plugin-local predicates are part of the same fix or a tracked follow-up.
