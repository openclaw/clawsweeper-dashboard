- `impact:message-loss`: The report is about debounce merging or swallowing a user control message before it reaches session handling.

## Summary

Keep open because current main still has the reported debounce gap, and two open PRs explicitly reference this issue as their fix path. The issue should stay open until one canonical PR lands or the linked PRs are closed without a replacement.

## What This Changes

The issue requests that bare abort phrases such as `stop`, `abort`, and `wait` bypass inbound debounce immediately instead of being delayed or merged.

## Best Possible Solution

Review and land one canonical fix that moves shared and plugin-local debounce bypasses to abort-aware command detection while preserving auth, mention-gating, and documented coalescing exceptions.

## Reproduction Assessment

Yes. Source inspection on current main shows `shouldDebounceTextInbound({ text: "stop", cfg })` reaches `hasControlCommand("stop")`, while bare stop/abort/wait are only recognized by the separate abort-aware path.

## Solution Assessment

Yes. Reusing the existing abort-aware command detector at debounce gates is the narrow maintainable fix, provided auth and mention-gating call sites that need slash-command semantics stay unchanged.
