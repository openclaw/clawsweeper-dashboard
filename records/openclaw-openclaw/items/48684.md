## Summary

Keep open. Current main and the latest release still use slash-command-only debounce bypasses for the shared inbound debounce path, while the abort-aware helper and docs already cover standalone abort phrases; the related fix PR at https://github.com/openclaw/openclaw/pull/48916 is closed unmerged.

## What This Changes

Bare standalone abort phrases should bypass inbound debounce so `stop`, `abort`, and `wait` flush immediately instead of being batched or merged.

## Best Possible Solution

Use abort-aware command detection at shared and plugin-local debounce boundaries, preserve documented coalescing exceptions, and add focused regression coverage for bare abort phrases.

## Reproduction Assessment

Yes. Source inspection on current main shows `shouldDebounceTextInbound({ text: "stop", cfg })` reaches `hasControlCommand("stop")`, which does not consult the abort trigger set, so the predicate returns debounceable text.

## Solution Assessment

Yes. Reusing `isControlCommandMessage()` or the plugin-runtime equivalent at debounce gates is the narrowest maintainable fix, as long as authorization and mention-gating paths that intentionally need slash-command semantics are left alone.
