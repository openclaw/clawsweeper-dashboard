- **[P1] Preserve the compact session file:** `src/context-engine/delegate.ts:67`
  - body: `ContextEngine.compact` still receives a concrete `sessionFile`, and current main forwards it to the built-in compaction bridge. This branch drops both the top-level and runtimeContext file whenever a `sessionKey` exists, so file-backed/custom engines can compact a store-derived/default transcript instead of the active file they asked to compact.
  - confidence: 0.86
- **[P1] Use no-create opens for SQLite reads:** `src/config/sessions/session-accessor.sqlite.ts:104`
  - body: These read helpers call the normal agent database opener, which creates directories/files, runs schema DDL, and registers the DB. The migration tracker requires doctor/startup preview reads not to materialize missing DBs, so add a no-create/read-only path that returns empty results when the database is absent.
  - confidence: 0.9
- **[P1] Require export before deleting transcript rows:** `src/config/sessions/session-accessor.sqlite.ts:751`
  - body: Deleting the `sessions` row cascades `transcript_events`, but this cleanup path only counts the state as archived and does not require a durable export receipt first. Once SQLite transcripts are live, this can silently drop transcript history during lifecycle cleanup.
  - confidence: 0.88
