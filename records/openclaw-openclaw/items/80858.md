## Summary

Keep open. Current main and the latest release still emit the empty REM candidate placeholder and the promotion filter still lacks a placeholder-specific rejection; the focused open fix PR already references this issue with closing syntax, so this should stay open until that PR or an equivalent fix lands.

## What This Changes

Fix the memory-core dreaming pipeline so empty REM candidate-truth placeholders are not written, re-ingested, or promoted into MEMORY.md.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/80916 after normal review, or apply the same narrow source and regression-test changes on main.

## Reproduction Assessment

Yes. Source inspection on current main shows the REM preview still emits the empty placeholder and the promotion path still depends on a contamination filter that does not classify that placeholder shape.

## Solution Assessment

Yes. The narrow maintainable solution is to stop producing candidate-like REM markdown when there are no candidate truths and defensively filter already-written historical placeholders during promotion.
