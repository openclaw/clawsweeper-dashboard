## Summary

Keep open. Current main still emits the empty REM placeholder and does not filter that exact placeholder before promotion; the issue is already paired with an open fixing PR, so it should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue asks for empty REM dream days not to write or promote the `No strong candidate truths surfaced` placeholder into `MEMORY.md`.

## Best Possible Solution

Review, update if needed, and land https://github.com/openclaw/openclaw/pull/80916 or an equivalent narrow fix that removes empty REM candidate markdown and filters historical placeholder snippets.

## Reproduction Assessment

Yes. Source inspection shows `previewRemDreaming` emits the empty placeholder, daily ingestion can capture list bullets as snippets, and `applyShortTermPromotions` currently filters through a contamination predicate that does not reject this exact placeholder.

## Solution Assessment

Yes. The narrow maintainable fix is to stop emitting candidate-like markdown when REM has no candidate truths and add a defensive placeholder filter for already-written daily memory artifacts.
