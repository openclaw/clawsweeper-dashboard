## Summary

Keep open. Current main still emits the empty REM placeholder and the promotion filter does not reject that exact placeholder shape, while an open linked PR explicitly targets this issue and should be reviewed or landed before the issue closes.

## What This Changes

The issue asks memory-core to stop promoting empty REM placeholder text into long-term `MEMORY.md`.

## Best Possible Solution

Review and land or revise https://github.com/openclaw/openclaw/pull/80916, preserving the narrow REM output fix, historical placeholder filter, and regression coverage while coordinating overlap with https://github.com/openclaw/openclaw/pull/68774.

## Reproduction Assessment

Yes, source-level: current `previewRemDreaming` emits the empty-result placeholder and `applyShortTermPromotions` depends on a filter that does not classify that placeholder shape. I did not run the full cron pipeline in this read-only review.

## Solution Assessment

Yes. The narrowest maintainable fix is to avoid writing candidate-like markdown when REM has no candidate truths and to reject already-written placeholder snippets before promotion; the open linked PR implements that direction.
