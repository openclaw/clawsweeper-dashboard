## What This Changes

The issue reports that empty REM dreaming output is written as a candidate-like placeholder and later promoted into long-term MEMORY.md.

## Best Possible Solution

Land the focused memory-core fix in https://github.com/openclaw/openclaw/pull/80916 after resolving conflicts and normal maintainer validation, then close this issue as implemented.

## Reproduction Assessment

Yes. Source inspection on current main shows `previewRemDreaming` still emits the empty-result placeholder and `applyShortTermPromotions` depends on a contamination filter that does not classify that placeholder shape.

## Solution Assessment

Yes. The narrow maintainable solution is to stop writing empty candidate-truth markdown at the REM source and defensively reject already-written placeholder snippets before promotion, which is what the linked PR attempts.
