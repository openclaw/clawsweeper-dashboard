## Summary

Keep open. Current `main` and the latest release still emit the empty REM placeholder and the promotion path has no placeholder-specific guard, so this remains a valid memory-core data-pollution bug rather than a cleanup close.

## What This Changes

The issue asks for empty REM dreaming days to stop writing and later promoting the `No strong candidate truths surfaced` placeholder into long-term memory.

## Best Possible Solution

Stop writing empty-result prose into machine-ingested REM candidate sections, defensively reject historical placeholder snippets before promotion, and add focused memory-core regression coverage.

## Reproduction Assessment

Yes, source-level: current `previewRemDreaming` emits the empty-result placeholder, daily ingestion can convert list bullets into recall snippets, and promotion filtering does not reject that exact text. A full cron run was not executed in this read-only pass.

## Solution Assessment

Yes. The narrow maintainable fix is to avoid generating candidate-like placeholder markdown and to defensively skip already-written placeholder candidates during promotion, while coordinating with the existing broader dreaming-contamination PRs.
