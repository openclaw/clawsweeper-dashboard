## Summary

Keep open. Current main still writes plugin policy config before validating the requested id, and open PR #73554 is already the active closing implementation candidate.

## What This Changes

The issue requests registry validation for `plugins enable` and `plugins disable` so nonexistent plugin ids fail non-zero without writing `plugins.entries`.

## Best Possible Solution

Land or replace the existing implementation PR with registry-backed pre-write guards for both policy commands plus no-write regression tests.

## Reproduction Assessment

Yes. I did not run the mutating CLI in this read-only review, but the current source path shows an unknown id flowing into the config mutation helpers and `replaceConfigFile` before any discovered-registry lookup.

## Solution Assessment

Yes. A pre-write check against the discovered plugin snapshot used by plugin listing/inspection is the narrow maintainable fix, while leaving legacy stale-entry validation warn-only for startup resilience.
