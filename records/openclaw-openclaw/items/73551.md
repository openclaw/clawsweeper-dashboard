## Summary

Keep open. Current main still writes plugin config for unknown `plugins enable` and `plugins disable` ids before any registry lookup, and open PR #73554 is the active closing implementation candidate, so this issue should remain open until that PR lands or is replaced.

## What This Changes

Make `openclaw plugins enable/disable <id>` reject undiscovered plugin ids without mutating plugin config.

## Best Possible Solution

Land or replace #73554 with a registry-backed pre-write guard for both policy commands plus no-write regression tests.

## Reproduction Assessment

Yes. Without running the mutating command, the current source shows an unknown id flowing into config mutation helpers and `replaceConfigFile` before any discovered-registry lookup.

## Solution Assessment

Yes. The narrow maintainable fix is to check the requested id against the same discovered plugin snapshot used by list/inspect before calling the config mutation helpers, while keeping legacy stale-entry validation warn-only for startup resilience.
