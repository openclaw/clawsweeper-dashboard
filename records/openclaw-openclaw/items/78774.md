## Summary

Keep open: the report maps to current source behavior for a broken managed plugin install, and the exact `Unsupported channel` symptom should not be closed until the existing repair path is validated against Weixin login.

## What This Changes

The issue asks OpenClaw to stop failing `openclaw channels login --channel openclaw-weixin` with a compiled-runtime plugin warning followed by `Unsupported channel`.

## Best Possible Solution

Make OpenClaw repair or clearly route broken managed Weixin installs through the existing plugin update/doctor path, then validate that `channels login` either reloads a good `@tencent-weixin/openclaw-weixin@2.4.1` install or gives an actionable reinstall command instead of ending at `Unsupported channel`.

## Reproduction Assessment

Yes from source, but not with a live Weixin account: a global installed plugin with a TypeScript entry and no compiled runtime output is skipped with the same warning, and channel login throws `Unsupported channel` when no plugin resolves. The current npm 2.4.1 artifact itself contains compiled output, so the remaining repro data is the reporter's installed plugin state.

## Solution Assessment

Unclear. The right direction is likely the generic broken-runtime-entry repair path in https://github.com/openclaw/openclaw/pull/77219 or an equivalent targeted reinstall/diagnostic fix, but it needs validation against this exact Weixin login path before this issue is closed.
