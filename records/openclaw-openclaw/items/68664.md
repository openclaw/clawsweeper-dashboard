## What This Changes

The issue reports that legacy `memorySearch` keys can make config validation reject a user config before cleanup can remove them, causing gateway startup failures after schema changes.

## Best Possible Solution

Refresh and review the linked fix, or replace it with a narrow current-main migration that strips obsolete `memorySearch` keys while preserving strict runtime validation and adding focused upgrade-path proof.

## Solution Assessment

Partly. A targeted doctor migration is the policy-aligned minimal repair for shipped legacy keys, while automatic startup/update migration remains a maintainer product decision.
