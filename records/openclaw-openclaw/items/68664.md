## What This Changes

The issue asks OpenClaw to prevent obsolete memorySearch config keys from blocking gateway startup after schema changes by migrating or stripping them before strict validation.

## Best Possible Solution

Resolve this through a maintainer-approved migration boundary: either land a narrow doctor/update cleanup that strips obsolete memorySearch keys without claiming startup self-healing, or explicitly add startup-safe migration coverage and real runtime proof before closing the issue.

## Reproduction Assessment

Yes, by source inspection: a config with obsolete agents.defaults.memorySearch keys reaches strict validation before any doctor-only cleanup can run in startup/read paths. I did not run a live service repro in this read-only review.

## Solution Assessment

Unclear as filed. Stripping the obsolete keys is a narrow recovery fix, but moving doctor migrations into startup or update changes a documented migration boundary and needs maintainer approval.
