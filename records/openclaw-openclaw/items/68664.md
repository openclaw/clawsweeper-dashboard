## Summary

Keep this issue open. Current `origin/main` still rejects the reported stale `agents.defaults.memorySearch` subkeys and the remaining work is already paired with open PR #68685, which uses closing syntax for this exact bug; policy says the issue should stay open until that PR merges, is replaced, or is closed.

## What This Changes

The issue requests that legacy config handling strip obsolete `agents.defaults.memorySearch` keys before strict schema validation can prevent gateway startup.

## Best Possible Solution

Land or replace #68685 with a focused runtime legacy migration that strips only the obsolete `agents.defaults.memorySearch` keys and adds regression coverage for already-migrated configs; track update-command automation separately if maintainers still want that broader behavior.

## Solution Assessment
