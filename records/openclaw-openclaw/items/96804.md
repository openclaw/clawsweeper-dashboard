## Summary

This issue is still necessary: current main and the latest release still replace oversized trajectory event `data` wholesale, while the open linked PR is the active fix candidate and should land or be replaced before the issue closes.

## What This Changes

The issue requests preserving small accounting fields such as `usage` when oversized `model.completed` trajectory events are truncated instead of replacing the whole `data` block.

## Best Possible Solution

Land the canonical trajectory fix that preserves accounting fields inside bounded truncation stubs for both core and bundled Codex recorder paths without raising the event cap or adding configuration.

## Reproduction Assessment

Yes from source inspection: current main stores `usage` inside `model.completed.data` and then replaces that entire `data` object when the serialized event exceeds the event byte cap. I did not run a write-path reproduction because this was a read-only review.

## Solution Assessment

Yes in direction: preserving small accounting fields while still dropping large payloads is the narrow runtime-layer fix. The open linked PR should be reviewed and landed or replaced before this issue closes.
