## Summary

Keep open: current main and the latest release still replace oversized trajectory `data` with a truncation stub, so `model.completed.data.usage` is lost; https://github.com/openclaw/openclaw/pull/96811 is the active closing fix candidate.

## What This Changes

Preserve small accounting fields such as `usage` when oversized `model.completed` trajectory events are truncated instead of replacing the whole `data` block.

## Best Possible Solution

Land the canonical trajectory fix that preserves bounded accounting fields under the existing event cap across the core and bundled Codex recorder paths, without adding a new config surface.

## Reproduction Assessment

Yes. Current-main source stores `usage` inside `model.completed.data` and replaces that whole object when the serialized event exceeds 262144 bytes; I did not run a write-path replay because this review is read-only.

## Solution Assessment

Yes in direction. Preserving small structured accounting fields inside the existing truncation helpers is the narrow maintainable fix, and the open linked PR covers both core and bundled Codex recorder paths.
