## Summary

This issue is still necessary: current main and the latest release still replace the whole oversized trajectory event data object, while the open linked PR is the active fix candidate and has not merged yet.

## What This Changes

The issue reports that oversized `model.completed` trajectory events drop the entire `data` block, including `usage`, causing heavy-turn token accounting to undercount or read as zero.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/96811 or an equivalent narrow trajectory recorder fix, then close this issue once current main preserves accounting fields under the event cap.

## Reproduction Assessment

Yes, from source inspection: current main records `usage` inside `model.completed.data`, then replaces that entire object when the serialized row exceeds 262144 bytes. I did not run a write-path replay because this review is read-only.

## Solution Assessment

Yes in direction: preserving small structured accounting fields inside the existing truncation stub is the narrow maintainable fix. The active path is to review and land the linked PR rather than queue a duplicate repair job.
