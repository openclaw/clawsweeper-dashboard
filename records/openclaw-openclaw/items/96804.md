## What This Changes

The issue reports that oversized `model.completed` trajectory events lose token usage when event-size truncation replaces the whole `data` object.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/96811, or an equivalent narrow fix that preserves small accounting fields while keeping the existing event byte cap.

## Solution Assessment

Yes in direction: preserving small structured accounting fields inside the existing truncation stub is the narrow runtime-layer fix. The open linked PR should be reviewed and landed or replaced before this issue closes.
