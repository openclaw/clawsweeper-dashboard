## What This Changes

The issue reports that Open WebUI image uploads routed through OpenClaw to LM Studio fail because the generated `placeholder.openclaw.cloud` media URL is unreachable on WSL2 with a custom gateway port.

## Best Possible Solution

Define a narrow OpenAI-compatible media-ingest contract that avoids unresolved placeholder hosts, preserves SSRF-safe defaults, and documents any explicit WSL/local trust configuration needed for private gateway origins.

## Reproduction Assessment

Source-reproducible: the reported placeholder-host failure was not live-recreated on WSL2, but current code clearly rejects remote image URLs by default and blocks loopback/private resolution when enabled.

## Solution Assessment
