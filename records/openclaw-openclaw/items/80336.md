## Summary

Keep open for maintainer review: current main still does not make the reported Open WebUI placeholder media URL flow work, and no canonical duplicate or merged fix was found. The remaining decision is about the supported Open WebUI image-upload contract and how to preserve the existing private-network/SSRF boundary.

## What This Changes

The issue requests that Open WebUI image uploads routed through OpenClaw to LM Studio work on WSL2 when the gateway uses a custom port and the media URL host is unreachable.

## Best Possible Solution

Define and document a narrow Open WebUI image-upload contract, preferably data URI or authenticated gateway media proxy behavior, that works across WSL2/custom ports without broadly allowing private-network URL fetches.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection shows a remote image_url fails under the default chat-completions config and a loopback/private mapping would hit the existing SSRF guard, but the exact Open WebUI -> OpenClaw -> LM Studio payload and logs are missing.

## Solution Assessment

Unclear. Editing hosts files or broadly allowing private-network image fetches is not the best fix; maintainers should choose a narrow media-ingest/proxy contract that preserves URL-fetch hardening.
