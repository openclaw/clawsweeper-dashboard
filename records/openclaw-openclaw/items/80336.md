## Summary

Keep open: current main and the latest shipped release still do not define a safe Open WebUI placeholder-media contract for WSL2/custom-port image uploads, and the remaining fix intersects Chat Completions media handling plus SSRF/private-network policy.

## What This Changes

The issue requests that OpenAI-compatible Open WebUI image uploads remain accessible to the vision pipeline on WSL2 when the gateway uses a custom local port.

## Best Possible Solution

Define a documented Open WebUI media contract: require data URI or reachable public URLs, or add a narrow authenticated local media ingestion/proxy path that works across WSL2/custom ports while preserving SSRF protections.

## Reproduction Assessment

No high-confidence live reproduction was run. Source inspection does show the reported placeholder/loopback URL path is rejected by the Chat Completions default or by the private-network guard when URL fetching is enabled, but the exact Open WebUI/WSL2/lmstudio request and logs are missing.

## Solution Assessment

Unclear. A hosts-file workaround or broad private-network allow is not the right fix; maintainers should choose a narrow media-ingest/proxy contract or document the supported payload shape before code changes.
