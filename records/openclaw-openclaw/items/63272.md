## Summary

Keep open: current main and v2026.6.10 still expose `openclaw skills` search/install/update/verify but no `skills uninstall` command, and the previously linked implementation PR is closed unmerged. The issue remains a valid product/design tracker for clean ClawHub skill removal and lockfile cleanup.

## What This Changes

Request to add `openclaw skills uninstall <slug>` so ClawHub-installed skills can be removed from the workspace and `.clawhub/lock.json` together.

## Best Possible Solution

Keep this as the canonical CLI uninstall design tracker until maintainers either sponsor a narrow CLI/lifecycle implementation or decide the feature should wait for a different ClawHub lifecycle surface.

## Reproduction Assessment

Yes. Source inspection shows ClawHub install writes lockfile/origin metadata and `skills update --all` consumes the lockfile to update tracked skills, while the CLI lacks an uninstall command to remove both pieces of state.

## Solution Assessment

Unclear until maintainers decide the destructive CLI contract. A narrow command built on the current lifecycle helpers is the plausible right layer, but manual-skill handling and global/agent semantics need product review first.
