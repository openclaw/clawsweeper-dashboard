## Summary

Keep open: current main still lacks the requested `openclaw skills uninstall` command and a safe ClawHub skill removal helper, and the related Gateway RPC PR is open, conflicting, and does not cover the CLI surface.

## What This Changes

The issue proposes a new `openclaw skills uninstall <slug>` CLI command that removes a ClawHub-installed workspace skill and its `.clawhub/lock.json` entry.

## Best Possible Solution

Add a narrow CLI/workspace uninstall path that removes only ClawHub-originated or lockfile-tracked skill state, updates docs and changelog, and leaves dashboard, bundled-skill, sandbox purge, `.gitignore`, and plugin-config behavior out of scope.

## Reproduction Assessment

Yes. Static inspection of current main shows the CLI, backend, tests, and docs still have install/update coverage but no uninstall command or lockfile-removal helper.

## Solution Assessment

Yes. The proposed narrow CLI/workspace direction is maintainable, with the safer adjustment that deletion should be limited to ClawHub-proven or lockfile-tracked skill state rather than arbitrary manual skill folders.
