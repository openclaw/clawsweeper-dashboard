## Summary

Keep open. Current main still lacks the requested CLI command and backend removal helper, while the related closed request and closed dashboard PR do not implement this scope; the open Gateway RPC PR is useful context but is not a merged current-main CLI implementation.

## What This Changes

Request adds an `openclaw skills uninstall <slug>` CLI flow that removes a workspace ClawHub skill directory and its `.clawhub/lock.json` entry.

## Best Possible Solution

Implement the CLI-facing inverse of ClawHub install/update: safely resolve the active workspace, remove only ClawHub-proven or lockfile-tracked skill state, delete the matching lockfile entry, and update focused CLI docs, changelog, and tests.
