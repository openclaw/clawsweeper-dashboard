## Summary

Keep open. Current main still lacks the requested `openclaw skills uninstall <slug>` CLI command and still has no backend helper that removes both the workspace skill directory and the `.clawhub/lock.json` entry; the closed prior request and closed dashboard PR do not implement this CLI surface, and the overlapping open RPC PR is not a current-main implementation of the requested command.

## What This Changes

Add an `openclaw skills uninstall <slug>` CLI command that safely removes a workspace ClawHub skill directory and deletes its `.clawhub/lock.json` entry, with dry-run, confirmation, and JSON output support.

## Best Possible Solution

Implement the CLI-facing inverse of ClawHub skill install/update in the skills subsystem: resolve the active agent workspace, safely plan and remove the workspace skill folder, delete only the matching `.clawhub/lock.json` entry, support `--agent`, `--yes`, `--dry-run`, and `--json`, update the user-facing docs and changelog, and share any landed backend primitive from #69319 if it matches these semantics.

## Reproduction Assessment

Yes. The missing feature is reproducible by static current-main inspection: the skills CLI and public docs list no uninstall command, and the ClawHub skills backend has install/update lockfile code but no delete helper.

## Solution Assessment

Yes, the requested CLI/workspace scope is the narrow maintainable fix for the lockfile reappearing problem. The only sequencing caveat is to reuse a merged backend primitive from #69319 if it lands before the CLI work starts.
