## Summary

Keep open. Current main still lacks the requested `openclaw skills uninstall <slug>` CLI command and still has no skills backend helper that removes both the workspace skill directory and the `.clawhub/lock.json` entry; the closed prior request and closed dashboard PR do not implement this CLI surface, and the related open RPC PR is not a current-main CLI implementation.

## What This Changes

Add an `openclaw skills uninstall <slug>` command that removes a workspace ClawHub skill directory and deletes the matching `.clawhub/lock.json` entry with dry-run, confirmation, and JSON output support.

## Best Possible Solution

Implement the CLI-facing inverse of ClawHub skill install/update in the skills subsystem: resolve the active agent workspace, safely plan and remove the workspace skill folder, delete only the matching lockfile entry, update user-facing docs and changelog, and reuse any compatible landed backend primitive from #69319.

## Reproduction Assessment

Yes. Static current-main inspection reproduces the missing feature: the skills CLI and docs list no uninstall command, and the ClawHub skills backend has install/update lockfile code but no delete helper.

## Solution Assessment

Yes. The requested CLI/workspace scope is the narrow maintainable fix for the lockfile reappearing problem; the safer sequencing detail is to reuse #69319 if it lands before CLI work starts.
