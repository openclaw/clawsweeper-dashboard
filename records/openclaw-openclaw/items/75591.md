## Summary

Keep open. The report is fresh, detailed, corroborated by multiple users, and current main still has request-time paths that can rebuild plugin manifest registries instead of consistently reusing the gateway-owned metadata snapshot.

## What This Changes

The issue asks OpenClaw to stop rereading bundled plugin manifests dozens of times during each embedded-run preparation and reuse startup plugin metadata so per-turn prep latency drops back to seconds.

## Best Possible Solution

Reuse the gateway-owned PluginMetadataSnapshot or PluginLookUpTable across embedded-run prep, plugin skill resolution, provider ownership/runtime hook resolution, and plugin tool discovery where compatible, preserving snapshot invalidation on config, workspace, and installed-index changes.

## Reproduction Assessment

Yes. The issue includes a concrete OPENCLAW_DEBUG_INGRESS_TIMING plus strace reproduction, and current main still exposes cold manifest-registry fallback paths that match the repeated openclaw.plugin.json reads.

## Solution Assessment

Yes. The best fix is to thread the existing snapshot/lookup-table seam through hot request paths rather than adding hidden TTL-style manifest caches, which would conflict with the documented plugin cache boundary.

## Repair Work Prompt

Fix the per-request plugin manifest reread regression tracked by https://github.com/openclaw/openclaw/issues/75591. Keep the scope to reusing existing PluginMetadataSnapshot / PluginLookUpTable / manifestRegistry products in embedded-run request prep; do not add hidden TTL caches, persistent global manifest caches, or the broader fastPath API proposed in #75412. Thread or borrow the current gateway-compatible metadata snapshot through plugin skill resolution, provider ownership/runtime-hook resolution, and plugin tool discovery so repeated core-plugin-tools, system-prompt, stream-setup, bundle-tools, and skills prep do not rebuild the installed plugin manifest registry. Preserve invalidation on config, plugins.load.paths, workspaceDir, installed-index writes, and plugin inventory changes. Add regression tests that fail if the hot callers reread manifests when a compatible lookup table/snapshot is available. Add a user-facing CHANGELOG.md fix entry crediting the reporters, excluding bot/maintainer handles if they are the only source authors.
