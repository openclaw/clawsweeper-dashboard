Full review comments:

- **[P1] Make the QMD GBK test drive the Windows decoder path:** `packages/memory-host-sdk/src/host/qmd-process.test.ts:154-160`
  - body: `runCliCommand` calls the new SDK decoder without injecting platform or encoding, and that decoder returns UTF-8 before codepage detection when `process.platform` is not `win32`. On Linux/macOS this test never observes the mocked `chcp` result, so the GBK bytes decode as replacement characters instead of the expected Chinese output.
  - confidence: 0.92
- **[P2] Reject non-local SystemRoot before executing cmd.exe:** `src/infra/windows-encoding.ts:70`
  - body: This path builds an executable from `process.env.SystemRoot` and runs it for `chcp`, but the new validation accepts Windows absolute roots without requiring a local drive root. Node treats UNC paths as absolute, so a tainted environment can redirect this lookup away from the local Windows install.
  - confidence: 0.9
- **[P2] Reuse hardened root validation in the SDK decoder:** `packages/memory-host-sdk/src/host/windows-output-decoder.ts:143`
  - body: The SDK copy adds the same SystemRoot-derived cmd.exe execution path, so it needs the same local-drive-only validation or a shared hardened helper. Leaving the duplicate weaker check here preserves the executable trust-boundary issue even if core is fixed.
  - confidence: 0.9
