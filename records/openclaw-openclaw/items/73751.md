## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Reject UNC SystemRoot before spawning cmd.exe:** `src/infra/windows-encoding.ts:70-78`
  - body: `resolveTrustedWindowsCmdPath()` now derives an executable path from `process.env.SystemRoot`, but `normalizeWindowsSystemRoot()` accepts UNC roots such as `\\server\share\Windows` because they are absolute and pass the basename/dirname check. A tainted environment can therefore make codepage detection run `\\server\share\Windows\System32\cmd.exe`; require a local drive root before using this path. Apply the same fix to the memory-host copy.
  - confidence: 0.86
- **[P1] Make the qmd encoding test exercise Windows decoding:** `packages/memory-host-sdk/src/host/qmd-process.test.ts:140-161`
  - body: This test only mocks `spawnSync`, but `decodeWindowsOutputBuffer()` returns UTF-8 immediately unless `process.platform` is `win32`, so Linux/macOS CI never calls the mock and the GBK bytes decode as mojibake instead of `小试`. Inject a decoder/platform override or mock `process.platform` for this test before asserting the codepage path.
  - confidence: 0.88
