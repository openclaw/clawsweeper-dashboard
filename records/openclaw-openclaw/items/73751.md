## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Reject non-local SystemRoot before cmd.exe execution:** `src/infra/windows-encoding.ts:70`
  - body: `path.win32.isAbsolute()` accepts UNC and device-style roots, and this value is later joined with `System32\cmd.exe` and executed for `chcp`. Require a local drive root such as `C:\` before trusting `SystemRoot`, or a tainted environment can redirect codepage detection to a network/device executable path.
  - confidence: 0.86
- **[P2] Apply the same cmd.exe root hardening in the SDK helper:** `packages/memory-host-sdk/src/host/windows-output-decoder.ts:143`
  - body: The memory-host SDK copy has the same SystemRoot validation and execution sink as the core helper. If this SDK helper remains, it needs the same local-drive-root check so QMD codepage detection cannot execute `cmd.exe` from a UNC or device root.
  - confidence: 0.86
