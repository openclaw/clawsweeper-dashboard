## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Reject UNC SystemRoot before spawning cmd.exe:** `src/infra/windows-encoding.ts:70-77`
  - body: `path.win32.isAbsolute()` accepts UNC and extended-length roots, and these checks still allow values like `\\server\share\Windows`. Because this value is used to execute `System32\cmd.exe` for the codepage probe, a tainted environment can redirect execution; require a local drive root or use the existing Windows install-root normalizer.
  - confidence: 0.92
- **[P2] Apply the same SystemRoot hardening in the SDK decoder:** `packages/memory-host-sdk/src/host/windows-output-decoder.ts:143-150`
  - body: The memory-host SDK copy has the same UNC-accepting SystemRoot validation before spawning the generated `cmd.exe` path. Harden this helper too, otherwise the SDK path keeps the same attacker-controlled executable sink.
