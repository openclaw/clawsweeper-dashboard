## Summary

Keep open: the PR contains a still-relevant Android MMS performance idea, but the proposed branch is not mergeable as-is because it broadens Play flavor SMS/call-log PII access, lacks real-device proof, and drops MMS candidates before hydrated filtering and pagination.

## What This Changes

The PR enables SMS/MMS and call-log access in the Android Play flavor, updates SMS UI copy, and rewrites mixed SMS/MMS by-phone search to batch MMS hydration with a timeout.

## Best Possible Solution

Keep the Play flavor restricted unless maintainers approve a policy change, and port a candidate-preserving MMS batching fix into the current thirdParty SMS manager with focused regression coverage and real Android proof.

## Reproduction Assessment

Yes by source review. Current main still performs per-row MMS address/body/meta queries in mixed by-phone search, and the PR-introduced truncation is visible in the diff where only the first 50 MMS candidates are hydrated before filtering and pagination.

## Solution Assessment

No. Batch hydration is the right direction for the timeout, but this branch combines it with a sensitive Play flavor access change and implements the cap by discarding searchable candidates.
