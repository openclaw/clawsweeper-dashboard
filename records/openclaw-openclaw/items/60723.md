## Summary

Keep open: the MMS timeout problem still exists on current main, but this branch is not merge-ready because the batching drops candidates beyond the cap and the Play permission expansion conflicts with the current restricted-flavor policy without approval.

## What This Changes

The PR enables Play-flavor SMS/MMS and call-log access, updates SMS UI copy, and rewrites Android mixed SMS/MMS by-phone search to batch MMS hydration with a timeout.

## Best Possible Solution

Keep the Play flavor restricted unless maintainers approve a policy change, and port a candidate-preserving MMS batching fix into the current thirdParty SMS manager with focused regression coverage and real Android proof.

## Reproduction Assessment

Yes by source review. Current main still performs per-row MMS address/body/meta queries in mixed by-phone search, and the PR truncates hydrated MMS candidates before applying hydrated filters and pagination.

## Solution Assessment

No. Batch hydration is the right direction for the timeout, but this branch combines it with a sensitive Play-flavor policy change, a stale flavor implementation, and a candidate-dropping cap.
