## Summary

This PR should remain open because the MMS timeout/performance concern still appears real on current main, but the branch mixes that fix with a security-sensitive Play flavor policy change and still has a candidate-truncation regression that can drop valid MMS results.

## What This Changes

The PR enables Android Play flavor SMS/call-log access, updates SMS UI copy to mention MMS, and rewrites mixed SMS/MMS by-phone querying to batch MMS body and metadata hydration with a timeout.

## Best Possible Solution

Keep the Play flavor restricted unless maintainers explicitly approve a release-policy change, and port a candidate-preserving MMS batching fix into the thirdParty SMS manager with focused regression coverage and real Android proof.

## Reproduction Assessment

Yes for the PR-introduced regression by source review: the branch truncates MMS candidates before hydrated filters and pagination. The original timeout is source-reproducible on current main because mixed by-phone MMS hydration still performs per-row body/meta/address queries, but I did not run a real-device reproduction.

## Solution Assessment

No. Batch hydration is the right direction for the timeout, but this PR combines it with a sensitive Play flavor expansion and implements the cap by discarding searchable candidates.
