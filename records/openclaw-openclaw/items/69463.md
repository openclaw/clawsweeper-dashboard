## Summary

Keep open: this is a useful feature direction, but the PR still needs human privacy/API review, contributor-supplied real behavior proof, and fixes for raw session-key export, stale Pi package imports, and public config documentation before merge.

## What This Changes

This PR adds an opt-in `diagnostics.sessionTracing` config block and stream wrapper to inject configurable session-id and session-name headers into embedded LLM provider requests.

## Best Possible Solution

Define one privacy-safe, documented LLM request-correlation contract using explicit non-sensitive values or trusted-target gating, then reconcile it with the related gateway/header work before exposing config.

## Reproduction Assessment

Not applicable as a current-main bug; this is a feature PR. Yes for the PR defects: source inspection of PR head shows raw session-key defaults, the stale Pi import, and the undocumented public config surface.

## Solution Assessment

No, not as-is. The request-correlation goal is useful, but the implementation should avoid raw session keys by default, document the outbound privacy contract, use the current Pi namespace, and align with the overlapping gateway header design.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Avoid exporting raw session keys by default:** `src/agents/observability-session-headers.ts:36-40`
  - body: When tracing is enabled and `sessionName` is omitted, this sends the raw internal `sessionKey` as `x-session-name`; when `runId` is absent, it also sends it as `x-session-id`. Current session keys can include channel, account, peer kind, and peer id components, while the existing OTEL privacy contract avoids exporting session keys. Require explicit non-sensitive values, a trusted target, or bounded/hash identifiers before sending these headers out.
  - confidence: 0.9
- **[P2] Use the current Pi package namespace:** `src/agents/observability-session-headers.ts:1`
  - body: Current main depends on and imports `@earendil-works/pi-agent-core`, but this new helper imports `@mariozechner/pi-agent-core`. After rebasing onto current main, this added file should use the current package namespace or it will not match the repository's Pi runtime surface.
  - confidence: 0.88
- **[P3] Document the public tracing config before merge:** `src/config/zod-schema.ts:310-323`
  - body: This adds a user-facing `diagnostics.sessionTracing` config key, but the configuration reference, OpenTelemetry/privacy docs, and schema help/labels do not explain what data leaves the process or how operators should choose safe values. Add those docs before exposing the config surface.
  - confidence: 0.8
