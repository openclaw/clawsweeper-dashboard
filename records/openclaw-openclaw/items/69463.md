## Summary

This PR is a useful observability feature direction, but it should stay open because the current patch still has privacy/API, build-compatibility, docs, proof, and rebase blockers before merge.

## What This Changes

The PR adds an opt-in `diagnostics.sessionTracing` config block and wraps outbound LLM provider streams to add configurable session-id and session-name HTTP headers.

## Best Possible Solution

Define one documented, privacy-safe LLM request-correlation contract that is explicit opt-in or trusted-target scoped, uses bounded non-sensitive identifiers by default, and coordinates with the open provider metadata work.

## Reproduction Assessment

Not applicable as a current-main bug because this is a feature PR. The blocking PR defects are source-reproducible from PR-head code and current-main session-key, telemetry, dependency, and docs contracts.

## Solution Assessment

No, not as-is. The request-correlation direction is useful, but the implementation should avoid raw session-key defaults, use the current Pi package namespace, document the privacy/config contract, and coordinate with the overlapping provider metadata proposals.
