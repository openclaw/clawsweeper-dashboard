## Summary

Keep this PR open, but it should not merge as-is: the feature goal is valid, yet the current patch exports raw internal session keys by default, uses a stale Pi package namespace against current main, lacks public config/privacy docs, and still has no inspectable real behavior proof.

## What This Changes

The PR adds an opt-in `diagnostics.sessionTracing` config block and stream wrapper to attach configurable session-id/name headers to embedded LLM provider requests.

## Best Possible Solution

Define one privacy-safe, documented LLM request-correlation contract using explicit operator-provided values or bounded non-sensitive identifiers, and reconcile it with https://github.com/openclaw/openclaw/pull/54985 before merging.

## Reproduction Assessment

Not applicable as a current-main bug; this is a feature PR. Yes for the blocking review findings: source inspection of PR head shows raw `sessionKey` defaults and a stale Pi package namespace against current main.

## Solution Assessment

No, not as-is. The request-correlation direction is useful, but the patch should avoid raw session-key defaults, use the current Pi namespace, document the privacy/config contract, and align with the overlapping Gateway header proposal.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Avoid exporting raw session keys by default:** `src/agents/observability-session-headers.ts:36-40`
  - body: When tracing is enabled and `sessionName` is omitted, this sends the raw internal `sessionKey` as `x-session-name`; when `runId` is absent, it also sends it as `x-session-id`. Current session keys can include channel, account, peer kind, and peer id components, while the existing OTEL privacy contract avoids exporting session keys. Require explicit non-sensitive values or derive bounded/hash identifiers before sending these headers out.
  - confidence: 0.9
- **[P2] Use the current Pi package namespace:** `src/agents/observability-session-headers.ts:1`
  - body: Current main depends on and imports `@earendil-works/pi-agent-core`, but this new helper imports `@mariozechner/pi-agent-core`. After rebasing onto current main, this added file should use the current package namespace or it will not match the repository's Pi runtime surface.
  - confidence: 0.88
- **[P3] Document the public tracing config before merge:** `src/config/zod-schema.ts:310-323`
  - body: This adds a user-facing `diagnostics.sessionTracing` config key, but the configuration reference, OpenTelemetry/privacy docs, and schema help do not explain what data leaves the process or how operators should choose safe values. Add those docs before exposing the config surface.
  - confidence: 0.8
