## Summary

Keep this PR open for maintainer review. Current main still lacks the proposed configurable session-tracing headers, so the work is not obsolete, but the branch is not merge-ready as-is because it defaults to exporting raw internal session keys to provider/observability endpoints and needs a core-vs-provider direction, docs, and changelog review.

## What This Changes

The branch adds a `diagnostics.sessionTracing` config surface, a helper and tests for resolving configurable session headers, and wires the wrapper into embedded LLM stream requests.

## Best Possible Solution

If core-level request correlation headers are desired, implement them as a privacy-safe diagnostics contract that uses stable non-sensitive run/model-call identifiers or explicit operator-provided names, preserves trusted `traceparent` behavior, reuses existing header merge protections, and updates config docs, schema metadata, tests, and changelog. If arbitrary observability-provider headers are the goal, route the feature through provider/plugin configuration instead of a broad core default.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature PR. The behavior can be verified from the diff and helper tests by enabling `diagnostics.sessionTracing` and inspecting stream options, but this read-only review did not run tests or live provider traffic.

## Solution Assessment

No, not as-is. The PR solves a real tracing need, but the safest maintainable path should avoid raw session-key export by default, align with the existing diagnostic model-call/`traceparent` surface, and resolve the overlap with #54985 before merge.

## Security Review

Status: needs_attention

Summary: The diff introduces an opt-in but concrete privacy risk by exporting raw internal session keys in outbound LLM request headers by default.
