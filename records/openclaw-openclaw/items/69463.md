## Summary

Keep open: the correlation feature is useful, but this PR still needs maintainer privacy/API review and contributor-supplied real behavior proof before merge. Source inspection also shows raw session-key defaults, a stale Pi package import, and missing user-facing documentation for the new config surface.

## What This Changes

The PR adds a `diagnostics.sessionTracing` config block, schema/types/tests, and an embedded LLM stream wrapper that injects configurable session-id/name headers into provider requests.

## Best Possible Solution

Define one documented, privacy-safe LLM request-correlation contract that is explicit opt-in or trusted-target scoped, uses bounded non-sensitive identifiers by default, and aligns generic config with provider-owned header hooks.

## Reproduction Assessment

Not applicable as a current-main bug because this is a feature PR. Yes for the blocking review findings: PR-head source inspection shows raw session-key defaults, a stale Pi namespace import, and missing docs for the new config surface.

## Solution Assessment

No, not as-is. The request-correlation direction is useful, but the implementation should avoid raw session-key defaults, use the current package namespace, document the privacy/config contract, and align with the overlapping gateway-header proposals.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Avoid exporting raw session keys by default:** `src/agents/observability-session-headers.ts:36-40`
  - body: When tracing is enabled and `sessionName` is omitted, this sends the raw internal `sessionKey` as `x-session-name`; when `runId` is absent, it also sends it as `x-session-id`. Current session keys can include channel, account, peer kind, and peer id components, while the existing diagnostics privacy contract avoids exporting session keys. Require explicit non-sensitive values or derive bounded identifiers before sending these headers out.
  - confidence: 0.9
- **[P2] Use the current Pi package namespace:** `src/agents/observability-session-headers.ts:1`
  - body: Current main depends on and imports `@earendil-works/pi-agent-core`, but this new helper imports `@mariozechner/pi-agent-core`. Rebase this file onto the current package namespace so the added helper matches the repository's Pi runtime surface.
  - confidence: 0.88
- **[P3] Document the tracing config privacy contract:** `src/config/zod-schema.ts:310-323`
  - body: This adds a user-facing `diagnostics.sessionTracing` config key, but the configuration reference and OpenTelemetry/privacy docs do not explain what data leaves the process or how operators should choose safe values. Add those docs before exposing the config surface.
  - confidence: 0.8
