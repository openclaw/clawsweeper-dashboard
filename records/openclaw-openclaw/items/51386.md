## Summary

Keep open. Current main and the latest stable release still keep session transcript indexing experimental and opt-in, while the recent discussion and linked issues show unresolved source-label, visibility, QMD export, duplicate-indexing, privacy, and cost questions that need maintainer product judgment before graduation.

## What This Changes

Request to graduate session transcript indexing from experimental, define privacy/source/cost guidance, and potentially enable it by default for new installs to improve cross-channel continuity.

## Best Possible Solution

Keep this as the sessionMemory graduation tracker until maintainers define the stable source/visibility contract, QMD/export behavior, duplicate-hit guidance, privacy defaults, cost guidance, and public docs, then land one coordinated code/test/docs change.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is a graduation/defaults request. Static source, tests, docs, release-tag inspection, and related live issues give a high-confidence check that the feature remains experimental and opt-in.

## Solution Assessment

No, a direct default flip is not the best way to solve it today. The maintainable path is a maintainer-owned graduation that handles privacy boundaries, source labels, QMD behavior, duplicate indexing, retrieval precedence, tests, and docs together.
