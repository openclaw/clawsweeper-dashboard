## Summary

Keep open. Current main still treats session transcript recall as an opt-in experimental memory-search source, with source code, tests, and public docs all preserving the experimental gate. The implementation pieces cited by the report do exist, and the KeepMyClaw comment is useful external recovery context, but neither replaces the OpenClaw product decision around default-on behavior, privacy scope, cost guidance, and the still-open related gateway/session sync work.

## What This Changes

The issue requests graduating `sessionMemory` from experimental to stable/default-on for new installs, with stability confirmation, DM-only privacy defaults, opt-out documentation, and embedding cost guidance.

## Best Possible Solution

Memory/core maintainers should keep this as the graduation tracker: first resolve or explicitly scope gateway-dispatched session recall and incremental sync/cost work, then make a deliberate default/privacy decision, add any chosen config materialization defaults with focused resolver and memory-manager coverage, and update the public memory docs with scope, opt-out, staleness, and embedding-cost guidance.

## Reproduction Assessment

Not applicable as a runtime bug reproduction; this is a feature/defaults request. The current behavior is high-confidence from static verification: default config resolves to memory-only, tests enforce the experimental session gate, and docs still describe session transcript indexing as opt-in.

## Solution Assessment

No, not as an automated default flip today. The maintainable path is a maintainer-led graduation after the related stability and cost issues are resolved or explicitly scoped, with docs and privacy defaults changed in the same PR.
