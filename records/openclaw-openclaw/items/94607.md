## Best Possible Solution

Land one canonical fix that makes the preflight and the actual SSH local bind address match, preserves `PortInUseError` fallback, and closes the open canonical issue.

No, not yet: scoping the preflight to `127.0.0.1` is only the best fix if the actual SSH local listener is also explicitly bound there. OpenSSH `-L` syntax shows the current forward spec omits `bind_address`, so the patch should align the `-L` bind and the preflight before merge.
