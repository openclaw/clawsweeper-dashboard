## Summary

Keep open. Current main still has the agents overview picker gap this PR targets: configured provider-qualified primary, fallback, or per-agent refs outside `agents.defaults.models` and the catalog can still render as a synthetic current option. The chat-header side is already covered by newer main code, so the useful remaining work is the narrower overview picker refresh.

## What This Changes

The PR updates the Control UI model picker helpers to collect broader configured model refs while preserving alias labels, and adds agents overview plus chat header regression coverage for provider-qualified model selections.

## Best Possible Solution

Refresh the PR around the remaining agents overview behavior: make `resolveConfiguredModels` reuse the broader configured-model collector, preserve aliases from `agents.defaults.models`, add the overview regression test, and leave the already-shipped chat-header model-ref path intact except for test reconciliation.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a config with `agents.defaults.model.primary` or `fallbacks` set to provider-qualified refs that are not in `agents.defaults.models` and not present in `modelCatalog`; current `resolveConfiguredModels` omits them and `buildModelOptions` falls back to `Current (...)`.

## Solution Assessment

Unclear for the PR as a whole. Reusing the existing configured-model collector is the narrow maintainable fix for the overview picker, but the chat-header changes are stale because current main already has centralized chat model-ref handling.
