## Summary

Keep open. Current main and the latest release still enqueue truncated Slack and Microsoft Teams inbound message bodies into model-visible system-event text, while the historical fix PR is closed unmerged and no open canonical PR currently owns the repair.

## What This Changes

The issue asks to remove or safely separate truncated inbound message-body previews from system-event/runtime-context prompt blocks so long Slack and Teams messages are not duplicated or cut mid-word.

## Best Possible Solution

Remove user message content from arrival-only Slack and Microsoft Teams system-event labels, keep the full body only in the normal inbound payload/context, and add focused regression tests while preserving Feishu and Mattermost's current non-enqueue behavior.

## Reproduction Assessment

Yes, by source inspection: Slack and Microsoft Teams still enqueue a 160-character body preview into `System:` event text while the full body is separately delivered to the agent. I did not run live channel proof in this read-only review.

## Solution Assessment

Yes. The narrow maintainable fix is to keep system events as notification labels only and leave user-authored content in the existing inbound body/context path, with Slack and Teams regression coverage.
