## Summary

Keep open: this maintainer-labeled PR is active and the earlier TTS export blocker is fixed, but source review found a web_fetch provider fallback regression that should block merge.

## What This Changes

The branch exposes loaded Gateway/runtime plugin metadata, pins gateway runtime registries, widens startup plugin selection, adds public/runtime provider lookup paths, prewarms TTS, and updates focused tests and changelog.

## Best Possible Solution

Keep the loaded-runtime metadata refactor, but preserve web_fetch provider resolution by adding a web-fetch runtime public artifact path or retaining the previous provider fallback until fetch providers are startup-loaded.

## Reproduction Assessment

Yes for the review finding: source inspection shows default web_fetch execution can prefer runtime providers, then the PR's resolveRuntimeWebFetchProviders path returns [] when no loaded registry exists and no fetch runtime artifact resolver is wired.

## Solution Assessment

No: the metadata surface direction is reasonable and the TTS export fix is present, but this patch narrows the runtime web-provider fallback before web_fetch has an equivalent loaded/public provider path.

## Review Findings
