## Summary

Keep open. Current main still has a source-reproducible embedded-runner bypass: Pi creates the session with an auth-wrapped `streamSimple`, and OpenClaw only swaps to its boundary-aware transport when the stream function is `undefined` or literally `streamSimple`, so custom `openai-completions` embedded sessions can bypass the transport that adds streaming usage. This is a focused follow-up to https://github.com/openclaw/openclaw/issues/75357 and distinct from the broader fallback-estimation work in https://github.com/openclaw/openclaw/issues/73990.

## What This Changes

Custom OpenAI-compatible embedded sessions should send `stream_options.include_usage: true` by default so streamed usage is recorded without per-model compat overrides.

## Best Possible Solution

Route Pi's default OpenAI-compatible embedded fallback stream through OpenClaw's boundary-aware transport, while preserving provider-owned and explicit custom stream functions, and add regression coverage for custom `openai-completions` embedded sessions.

## Reproduction Assessment

Yes. A high-confidence source-level path exists: create an embedded session for a custom `openai-completions` model with normalized streaming usage compat false, then observe that the resolver treats Pi's auth-wrapped default stream as `session-custom` and bypasses OpenClaw's usage-aware transport.

## Solution Assessment

Yes. The narrow maintainable fix belongs in embedded stream resolution or its session-creation metadata, not in a new config knob or in asking every custom model to set `compat.supportsUsageInStreaming`.
