## Summary

Keep open. Current main still has a source-reproducible embedded-runner bypass: OpenClaw only swaps to its boundary-aware transport when the session stream is undefined or exactly `streamSimple`, while PI installs an auth-wrapped default stream that is not identity-equal to `streamSimple`. This is a focused follow-up to https://github.com/openclaw/openclaw/issues/75357, distinct from the fallback-estimation work in https://github.com/openclaw/openclaw/issues/73990, and it is already paired with an open implementation candidate at https://github.com/ly85206559/openclaw/pull/3.

## What This Changes

The issue requests that custom OpenAI-compatible embedded streaming sessions send `stream_options.include_usage` by default without per-model `compat.supportsUsageInStreaming` overrides.

## Best Possible Solution

Route PI's default OpenAI-compatible embedded stream through OpenClaw's boundary-aware transport while preserving explicit provider-owned and custom session streams, then land regression coverage in `stream-resolution.test.ts`.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction path exists: use a custom `openai-completions` embedded session whose PI-created session stream is the default auth wrapper, and the resolver will treat it as custom instead of routing through OpenClaw's usage-aware transport.

## Solution Assessment

Yes. The narrow maintainable fix belongs in embedded stream resolution or its default-stream metadata, not in a new config knob or in requiring every custom model to set `compat.supportsUsageInStreaming`.
