## Summary

Keep open. Current main supports Slack ambient room events for inbound room chatter and now emits Slack outbound message hooks, but it still has no same-app internal fanout from an agent's successful Slack post to other bound agents; the linked PR only covers cross-app bot allowlisting and explicitly leaves this same-app case as follow-up.

## What This Changes

Request internal distribution of same-app Slack agent outbound posts as room events to other agents bound to the same Slack room.

## Best Possible Solution

Add a maintainer-approved Slack/gateway fanout that reuses existing outbound receipt or message_sent infrastructure to create room_event-style context for other bound agents, gated by existing ambient-room policy where possible and excluding the origin session.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature request. The reporter supplied credible Slack same-app behavior details, and source inspection confirms current main lacks an internal outbound-to-room-event fanout.

## Solution Assessment

Unclear as a final design. The requested direction is plausible, but the best fix should reuse the current Slack message_sent/outbound receipt and ambient room-event path with explicit loop and authorization rules, not the cross-app allowBotsFrom PR.
