## Summary

Keep open. Current main has Slack ambient room events for inbound room chatter and now emits Slack outbound `message_sent` hooks, but it still lacks the requested same-app internal fanout from one agent's successful Slack post to other agents bound to the same room; the linked PR is open, conflicting, and explicitly cross-app only.

## What This Changes

The issue requests gateway-level internal fanout of outbound Slack posts from same-app agents to other agents bound to the same Slack room.

## Best Possible Solution

Add a maintainer-approved Slack/gateway fanout that reuses the existing outbound receipt or `message_sent` seam to create `room_event`-style context for other eligible bound agents while excluding the origin session.

## Reproduction Assessment

No, not as a strict bug reproduction because this is a feature request. Source inspection gives a high-confidence missing-path check: current Slack outbound emits hooks/receipts but has no co-bound room-event fanout.

## Solution Assessment

Unclear as a final design. The requested gateway-internal direction is plausible, but maintainers need to approve the exact loop, authorization, and session-routing semantics before a fix PR is safe.
