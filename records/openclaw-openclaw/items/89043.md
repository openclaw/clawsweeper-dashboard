## Summary

Keep open. Current main and the latest release still build Slack room-event context only from inbound Slack events, while Slack outbound delivery only posts messages, records receipts/thread participation, and emits sent hooks; the linked pull request is open and explicitly covers only cross-app bot allowlisting, not same-app internal fanout.

## What This Changes

Request same-app Slack agent posts to be internally distributed as room events to other agents bound to the same channel in multi-agent single-app deployments.

## Best Possible Solution

Add a maintainer-approved Slack plugin/core seam that turns successful same-app Slack outbound posts into quiet room events for other eligible bound agent sessions, with explicit origin exclusion, authorization, threading, and loop-protection tests.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Partly: source inspection shows current main has no outbound-to-co-bound-room-event path, but a live multi-agent single-app Slack Socket Mode reproduction is still missing.

## Solution Assessment

Is this the best way to solve the issue? Unclear as a final design: internal fanout is plausible, but maintainers need to choose the routing, authorization, origin-exclusion, threading, and loop-protection contract first.
