## Summary

Keep open. Current main has Slack inbound ambient room-event handling and Slack outbound `message_sent` hooks, but it still does not synthesize same-app outbound Slack posts into room events for other agents bound to the same channel; the linked PR is open, partial, and cross-app only.

## What This Changes

Request internal distribution of same-Slack-app outbound agent posts as `room_event` context to other agents bound to the same Slack channel.

## Best Possible Solution

Add a maintainer-approved Slack/gateway fanout that reuses the successful outbound delivery seam to create room-event context for other eligible bound agents while excluding the origin session and preserving Slack authorization rules.

## Reproduction Assessment

No high-confidence live reproduction was run. The issue includes detailed empirical Slack evidence, and source inspection gives a high-confidence missing-path check: outbound Slack posts emit hooks/receipts, but no current path fans them out as room events to co-bound agents.

## Solution Assessment

Unclear as a final design. Gateway-internal fanout is plausible, but maintainers need to approve the routing, authorization, origin-exclusion, and loop-protection semantics before a fix PR is safe.
