Keep open: current main still treats `[[reply_to_current]]` as a shared reply directive, and the strict per-agent config surface still has no `directives.replyToCurrent` opt-out. Add a per-agent configuration option to disable runtime processing of the `[[reply_to_current]]` directive.

## Best Possible Solution

Add a backward-compatible per-agent directive policy that defaults to enabled, resolves through the agent config surface, and enforces disabled `replyToCurrent` handling at shared parsing and reply-normalization boundaries.
