## Summary

Keep open: the Codex streaming fix is in the right area and has useful TUI proof, but the branch still emits `replace: true` with a non-empty `delta`; current Gateway/TUI merge code ignores that marker and appends the delta, so switched Codex assistant items can still show concatenated text.

## What This Changes

The branch streams Codex app-server non-commentary assistant deltas through partial replies and assistant agent events, and adds replace markers plus projector tests for switched assistant items.

## Best Possible Solution

Keep the projector-level streaming fix, but make replacement events safe for live consumers by either omitting appendable delta on replacement or plumbing `replace` through Gateway/TUI merge logic with focused regression coverage.

## Solution Assessment

No, not yet. The Codex projector is the right boundary for unphased deltas, but the current replacement payload shape is not safe until live consumers honor replacement or receive non-appendable replacement events.
