## Summary

Keep this PR open: it fixes the main unphased Codex delta path and has live TUI proof, but the switched-item replacement path still breaks live chat consumers because the emitted replacement payload also carries a non-empty delta that Gateway/TUI append while ignoring `replace`.

## What This Changes

The PR changes the Codex app-server projector to stream non-commentary assistant deltas through both `onPartialReply` and assistant agent events, adds replacement markers on assistant item switches, and updates projector tests.

## Best Possible Solution

Keep the projector fix, but make replacement events safe for every live consumer by either clearing `delta` on replacement payloads or plumbing `replace` through Gateway/TUI merge logic, with focused regression coverage for switched assistant items.

## Reproduction Assessment

Yes, source-level reproduction is high confidence: current main gates Codex partials on `final_answer`, while upstream Codex delta notifications have no phase and Gateway/TUI consume assistant events for live deltas. I did not run a fresh live Codex turn in this read-only review.

## Solution Assessment

No, not yet. The projector is the right owner boundary, but the current payload shape is not the best fix until replacement semantics are honored by Gateway/TUI or replacement deltas stop being appendable.
