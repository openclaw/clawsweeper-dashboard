## Summary

Keep this PR open: it addresses a real Codex app-server streaming bug and now includes plausible live TUI proof, but the current patch still sends superseded non-commentary assistant items through the broader partial-reply path without replacement semantics, which can corrupt streaming for replacement-unaware channel consumers.

## What This Changes

The PR changes the Codex app-server event projector to stream every non-commentary assistant delta through both `onPartialReply` and `stream: "assistant"` agent events, with projector tests for unphased deltas.

## Best Possible Solution

Keep the projector fix at this layer, but preserve item-boundary replacement for all partial consumers while retaining the new assistant agent events that make TUI/WebChat stream incrementally.

## Solution Assessment

No, not as submitted. The projector is the right layer and the assistant agent-event fanout is needed for TUI/WebChat, but `onPartialReply` must preserve replacement semantics when Codex switches assistant item IDs.
