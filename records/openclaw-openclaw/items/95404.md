## Summary

Keep open: the projector direction addresses the Codex TUI/WebChat source gap, but the new replace:true empty-delta assistant events are not handled by OpenAI-compatible Chat Completions streaming, so a multi-item Codex turn can still leak stale intermediate text. The linked recordings are useful for ordinary TUI streaming, but they do not prove the latest replacement behavior or the affected SSE path.

## What This Changes

The branch updates the Codex app-server projector and tests to stream unphased non-commentary assistant deltas through partial replies and assistant agent events, mark assistant item switches as replacements, and carry incidental formatting/conflict-resolution changes in QA, Telegram, and script test files.

## Best Possible Solution

Keep the projector fix, but make every assistant-event streaming consumer that can see replace:true snapshots preserve final-text authority, especially OpenAI-compatible Chat Completions, then add focused regression coverage and refreshed proof.

## Reproduction Assessment

Yes, at source level: current main gates Codex partials on final_answer while the inspected Codex app-server delta notification has no phase field, and TUI/WebChat consume stream:"assistant" events for live deltas. The new OpenAI-compatible streaming regression is also source-reproducible from the empty-delta replacement payload and openai-http's empty-content return path.

## Solution Assessment

No, not yet. The Codex projector is the right owner boundary for restoring unphased assistant deltas, but the branch is incomplete until public streaming consumers that cannot rewrite prior chunks handle replace:true snapshots safely.
