## Summary

Keep open: the latest commit adds a replacement marker at the Codex projector, but the patch still sends replace-marked payloads with `delta` into live-chat/TUI and append-oriented partial consumers that do not honor `replace` before appending. A switched Codex assistant item can still concatenate superseded prose or suppress the final answer on some surfaces.

## What This Changes

The PR changes the Codex app-server event projector and tests so non-commentary assistant deltas stream through both `onPartialReply` and `stream: "assistant"` agent events, with `replace: true` when the assistant item id changes.

## Best Possible Solution

Keep the projector-layer Codex streaming fix, but make replacement semantics a real cross-surface contract before merge.

## Reproduction Assessment

Yes, at source level: current main gates Codex partials on `final_answer` while Codex delta notifications do not carry phase, and the PR's replacement path can be traced through live-chat/TUI and Teams consumers without executing a live run.

## Solution Assessment

No, not yet. The projector is the right layer for emitting Codex deltas, but the patch only marks replacements at the source while downstream merge helpers and at least one append-oriented channel ignore that marker when a delta is present.
