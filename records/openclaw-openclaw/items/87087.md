## Summary

This PR is still useful and should stay open: current main still sends the Claude CLI skill catalog in the appended prompt while also preparing the native Claude skills plugin, and the PR narrows that duplication with focused fallback coverage plus new live Claude CLI proof.

## What This Changes

The PR suppresses the Claude CLI `<available_skills>` prompt block when selected skills can be carried by the native Claude Code plugin, adds prepare-context regression tests, and updates the CLI backend docs.

## Best Possible Solution

Land the plugin-first Claude CLI skill path after required checks are settled, keeping the prompt catalog only as the documented fallback when plugin skills cannot be prepared.
