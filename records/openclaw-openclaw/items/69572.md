## Summary

Keep open. Current main and the latest release still implement Feishu typing feedback through Feishu/Lark reaction APIs, while the SDK-backed `Typing` surface I found is a user/system-status API rather than a proven per-chat native typing indicator.

## What This Changes

The issue requests replacing Feishu's reaction-based typing feedback with a native Feishu/Lark typing-status API.

## Best Possible Solution

Confirm the Feishu/Lark native typing or system-status API and permission model, then update the Feishu plugin with tests, docs, and live proof; keep reaction feedback only as an explicit fallback if native typing is unavailable.

## Reproduction Assessment

Yes for the source-level behavior: current main routes Feishu typing feedback through `im.messageReaction.create`. No live-client proof currently confirms which Feishu/Lark API produces native `正在输入...` text for a bot.

## Solution Assessment

Unclear. Replacing reactions may be the right UX direction, but the SDK `Typing` surface appears to be tenant/user system status rather than a proven per-chat typing API, so the implementation needs API and live-client confirmation first.
