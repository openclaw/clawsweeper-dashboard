## Summary

Keep open. Current main and the latest release still implement Feishu typing feedback as Feishu/Lark reactions, and the related open PRs adjust reaction behavior rather than replacing it with native typing status.

## What This Changes

Replace Feishu typing feedback that currently uses message/comment reactions with the native Feishu/Lark typing status API.

## Best Possible Solution

Keep this as a bundled Feishu plugin behavior bug and replace the reaction-based typing lifecycle with a confirmed native typing/status implementation, with docs and tests updated around the chosen API and any deliberate fallback.

## Reproduction Assessment

Yes. With Feishu typing enabled, the source-level path is deterministic: reply start calls `addTypingIndicator()`, which sends `im.messageReaction.create` with the `Typing` emoji instead of a native typing-status call.

## Solution Assessment

No current fix is present. The safer path is a Feishu-owned native typing/status implementation after API and permission semantics are confirmed; the currently linked PRs keep building on reactions.
