## Summary

Keep open. Current main and the latest release still use Feishu message reactions for typing feedback, while the requested native Feishu/Lark typing/status behavior remains unimplemented and needs API, permission, fallback, and live-client validation before a safe change.

## What This Changes

The issue asks Feishu typing feedback to use a native Feishu/Lark typing indicator instead of the current `Typing` emoji message reaction.

## Best Possible Solution

Confirm the supported Feishu/Lark native typing or status API and permission model, then update the Feishu plugin lifecycle, tests, docs, and fallback policy with live proof.

## Reproduction Assessment

Yes for source-level behavior: current main routes Feishu typing feedback through `im.messageReaction.create` and documents it as typing reactions. No live client proof was run to validate the native `正在输入...` expectation or candidate replacement API.

## Solution Assessment

Unclear. Replacing reactions may be the right UX direction, but the checked SDK surface only proves message reactions and tenant/user system status with a `Typing` icon, not a confirmed per-chat bot typing API.
