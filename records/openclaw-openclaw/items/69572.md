## Summary

Keep open. Current main and the latest release still implement Feishu typing feedback through reaction APIs, while the plausible SDK `Typing` surface is a tenant/user system-status API rather than a confirmed native per-chat bot typing API.

## What This Changes

Request replacing Feishu's reaction-based typing feedback with a native Feishu/Lark typing or status mechanism.

## Best Possible Solution

Confirm the native Feishu/Lark typing or status API and permission model, then update the Feishu plugin with tests, docs, and live proof; keep reaction feedback only as an explicit fallback if native typing is unavailable.

## Reproduction Assessment

Yes for the source-level behavior: current main routes Feishu typing feedback through `im.messageReaction.create`. No live-client proof currently confirms which Feishu/Lark API produces native per-chat `正在输入...` text for a bot.

## Solution Assessment

Unclear. Replacing reactions may be the right UX direction, but the SDK `Typing` surface found here appears to be tenant/user system status rather than a proven per-chat typing API, so implementation needs API and live-client confirmation first.
