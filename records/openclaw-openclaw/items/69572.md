## Summary

Keep open: current main and v2026.6.10 still implement Feishu processing feedback as a persistent Typing message reaction, while replacing it with a native Feishu/Lark typing-status path remains an unconfirmed external API and product decision.

## What This Changes

The issue requests replacing Feishu's Typing emoji reaction feedback with a native Feishu/Lark typing indicator when the bot is processing a message.

## Best Possible Solution

Keep the issue open until maintainers confirm the supported Feishu/Lark typing API contract; then either implement native typing inside the Feishu plugin lifecycle with fallback docs and live DM/group/thread proof, or document the reaction/streaming-card fallback as intentional.

## Reproduction Assessment

Partially yes: source inspection reproduces that current main and v2026.6.10 still use Feishu message reactions for typing feedback. No live Feishu/Lark client repro was run to prove the native-status rendering gap or a supported replacement endpoint.

## Solution Assessment

Unclear: a native Feishu typing endpoint would be the cleanest fix if it is supported for bot messages, but current SDK/source evidence does not prove that path. Without that contract, the safer solution is a maintainer-approved fallback UX and docs update rather than an automated API swap.
