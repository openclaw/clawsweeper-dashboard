## Summary

Keep open. Current main and the latest release still implement Feishu typing feedback as message reactions, and the 2026-05-21 product update makes a clearer in-progress indicator a near-term Feishu UX requirement, so the remaining work is unresolved and should not be auto-closed.

## What This Changes

The issue asks Feishu to replace or redesign reaction-based typing feedback so users see an obvious native or intentionally designed working indicator during agent turns.

## Best Possible Solution

Confirm the supported Feishu/Lark native typing or status API; if none exists for some surfaces, design and document the best visible fallback in the Feishu plugin with lifecycle cleanup, anti-spam behavior, tests, docs, and live Feishu/Lark proof.

## Reproduction Assessment

Yes for source-level behavior: current main routes Feishu typing feedback through `im.messageReaction.create` and documents it as typing reactions. No live Feishu/Lark proof was run to validate the native `正在输入...` expectation or a replacement API.

## Solution Assessment

Unclear. The product target is now clear, but the inspected SDK surface only proves reaction and system-status APIs, so the maintainable fix needs a Feishu API, fallback-policy, and live-proof decision first.
