## Summary

Keep open. Current main and the latest release still implement Feishu typing feedback with message reactions, while the requested native Feishu/Lark typing or working indicator remains a plugin-owner/API decision; the Agenteam product note was corrected as out-of-scope for OpenClaw upstream, but no merged OpenClaw fix replaces the reaction path.

## What This Changes

The issue asks for Feishu typing feedback to use a native Feishu/Lark typing or working indicator instead of the current message-reaction emoji path.

## Best Possible Solution

Have the Feishu plugin owner decide and document the supported typing/working indicator strategy, then implement the native API path if Feishu/Lark supports it for the required chat surfaces with a deliberate fallback and live client proof.

## Reproduction Assessment

No, not for the full Feishu client behavior. Source inspection does verify that current OpenClaw uses Feishu message reactions for typing feedback, but the report does not include live Feishu proof of the missing native status or an inspected Feishu API contract for the replacement path.

## Solution Assessment

Unclear. Replacing reactions with a native typing/working API may be the right outcome if Feishu supports the required surfaces, but the safe solution needs plugin-owner API, permission, fallback, and live-validation decisions first.
