## Summary

Keep open. Current main and the latest release still implement Feishu typing feedback as Feishu/Lark reaction calls, and the related open PRs adjust reaction behavior rather than replacing it with native typing status.

## What This Changes

Replace Feishu's reaction-based typing indicator with native Feishu/Lark typing status so users see the client typing text instead of a Typing emoji reaction.

## Best Possible Solution

Implement the fix inside the bundled Feishu plugin after confirming official API and permission semantics, with tests for the new lifecycle and an explicit fallback when native typing is unsupported.

## Reproduction Assessment

Yes. With Feishu typing enabled, the source-level path is deterministic: reply start calls `addTypingIndicator()`, which sends `im.messageReaction.create` with the `Typing` emoji instead of a native typing-status call.

## Solution Assessment

Unclear. The requested direction is likely the right product behavior, but the exact native Feishu/Lark API contract is not confirmed from the current repo or pinned SDK surface, so a maintainer should verify it before implementation.
