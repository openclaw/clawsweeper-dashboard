## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Use the actual failed edit for diagnostics:** `src/agents/sessions/tools/edit.ts:181-182`
  - body: `applyEditsToNormalizedContent` accepts fuzzy matches after normalizing spaces, quotes, dashes, and related characters, but this selector only checks whether the raw LF-normalized oldText is an exact substring. In a multi-edit call where an earlier edit fuzzy-matches and a later edit is the one that misses, the closest-lines block will be generated for the earlier successful edit, so the new diagnostic points callers at the wrong text. Carry the failed edit index from the matcher or reuse the same match semantics here.
  - confidence: 0.87
- **[P2] Bound closest-line scanning before computing distances:** `src/agents/sessions/tools/edit-diff.ts:375-381`
  - body: The new mismatch path walks every nonblank file line and runs a capped but still O(n*m) Levenshtein comparison before rejecting dissimilar rows. A failed edit in a large file can turn an error response into seconds of CPU or a stalled agent turn even though only three candidates are displayed. Add a file-size, line-count, time, or cheap prefilter bound before the full distance calculation.
  - confidence: 0.82
