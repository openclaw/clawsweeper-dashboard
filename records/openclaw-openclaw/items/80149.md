## Summary

Keep open. The node wake-budget direction may be useful, but this branch is not mergeable as-is because it reverts current-main `sessions_spawn` behavior, leaves finite nudge-budget semantics inconsistent, and lacks required real behavior proof.

## What This Changes

The PR adds wake nudge budget fields to `node.list`/`node.describe`, introduces `OPENCLAW_NODE_WAKE_NUDGE_BUDGET=0` for unlimited nudges, updates wake tests, and changes `sessions_spawn` handling.

## Best Possible Solution

Rebase onto current main, keep the existing `sessions_spawn` contract untouched, implement node wake nudge budgets with clear finite/unlimited semantics plus focused tests/docs, and add redacted real setup proof.

## Reproduction Assessment

Yes for the review findings: source inspection against current main shows the PR removes documented and tested `sessions_spawn` behavior. The node wake-budget runtime path itself was not live-reproduced; the PR only provides test command names.

## Solution Assessment

No. The node budget change should be isolated from the stale `sessions_spawn` rollback, and the budget API should define and enforce finite counts or clamp the supported modes before merge.
