## Summary

Keep open: the useful discoverability hint is not merge-ready because the added docs still promise `Keep current values` preserves provider keys, while current setup continues into the selected provider auth flow whenever a non-skip `--auth-choice` is supplied.

## What This Changes

The branch adds provider-key update hints to onboarding wizard copy, onboarding docs, and regression tests for rerunning onboarding with an auth choice.

## Best Possible Solution

Keep the new `Review and update` hint, but make docs and runtime agree: either remove the Keep-preserves-provider-keys promise or implement and test a true provider-auth skip for Keep.

## Reproduction Assessment

Yes from source inspection. Current main only special-cases `reset` in the existing-config menu, then later calls `applyAuthChoice` for any supplied non-skip `--auth-choice`, so the added Keep-preserves-keys wording does not match runtime behavior.

## Solution Assessment

No as-is. The `Review and update` hint is a good narrow fix, but the docs must stop promising provider-key preservation unless the PR also changes and tests the Keep path as a true auth skip.
