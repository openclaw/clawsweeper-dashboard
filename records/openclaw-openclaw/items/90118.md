## What This Changes

## Best Possible Solution

Keep the targeted hint/docs/test approach, but gate or reword the hint so non-credential auth choices such as `skip` do not promise a provider-key prompt.

## Reproduction Assessment

Yes, source-reproducible for the review finding: `skip` is a valid documented auth choice, the existing-config hint condition is based on any defined `opts.authChoice`, and the explicit skip branch avoids provider credential prompts.

## Solution Assessment

No, not quite: the hint belongs at the right decision point, but checking only whether `opts.authChoice` is defined is too broad; the best fix is to limit the hint to credential-updating choices or use wording that stays true for all explicit choices.
