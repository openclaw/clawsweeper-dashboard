## Summary

Current main and shipped release code no longer have the reported gateway-runtime dependency install path, and the affected plugins are now handled through package-manager-owned or external plugin install paths rather than `dist/extensions/*/node_modules`.

## What This Changes

The issue reports a non-root gateway EACCES regression when bundled plugin runtime dependencies are installed under root-owned packaged extension directories.

## Best Possible Solution

Keep the shipped package-manager-owned plugin dependency model and ask for a fresh report only if a current release still launches a package-manager child process under `dist/extensions/*`.

## Reproduction Assessment

Yes for the historical bug: the report gives a concrete 2026.4.26 root-owned global install plus non-root systemd-user gateway path and EACCES logs. I did not reproduce a current-main failure because source and release evidence show the runtime install path is gone.

## Solution Assessment

Yes: the maintainable fix is the shipped package-manager-owned model, with explicit plugin install/update handling and postinstall cleanup of legacy runtime-deps state rather than gateway startup writes into packaged extension directories.
