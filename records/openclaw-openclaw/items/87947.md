## Label Justifications

- `P1`: The original regression silently removed gateway plugin capability for non-root production-style installs.
- `impact:other`: The issue is about gateway plugin availability at startup, which is maintainer-visible but outside the specific owned impact labels.

## Summary

Current main and the v2026.6.6 release no longer have the reported gateway-runtime dependency install path under root-owned `dist/extensions/*`; the affected plugin surfaces now resolve through package-owned deps, external plugin catalogs, or root-owned package dependencies without runtime `npm install`.

## What This Changes

The issue asks OpenClaw to stop silently dropping bundled plugins when a non-root gateway cannot create `node_modules` under root-owned packaged plugin directories.

## Best Possible Solution

Keep the current package/plugin ownership model and close this issue; ask for a new report only if a current release again writes plugin deps under packaged `dist/extensions`.

## Reproduction Assessment

No high-confidence current-main repro remains. The original 2026.4.x logs were concrete, but current main and v2026.6.6 source no longer contain the gateway runtime package-manager install path under `dist/extensions/*/node_modules`.

## Solution Assessment

Yes. The current solution removes the old packaged write target, routes ACPX/WhatsApp through official external plugin catalogs, keeps browser deps in the root package, and confines dependency install work to explicit plugin install/update flows.
