## Summary

Current main and shipped releases have removed the reported gateway-runtime dependency install path; plugin deps now belong to package-manager-owned plugin installs or root-owned bundled package deps instead of `dist/extensions/*/node_modules`.

## What This Changes

The issue asks OpenClaw to stop failing bundled plugin loading with EACCES when a root-installed package is run by a non-root gateway user.

## Best Possible Solution

Keep the shipped package-manager-owned plugin dependency model and ask for a fresh report only if a current release still launches a package-manager child process under `dist/extensions/*`.

## Reproduction Assessment

Yes for the historical bug: the report gives concrete 2026.4.26 root-owned global-install steps and EACCES logs. I did not reproduce a current-main failure because source and release evidence show the runtime install path is gone.

## Solution Assessment

Yes. The shipped package-manager-owned model is the maintainable fix because plugin install/update/doctor own dependency repair while gateway runtime loading stays read/import-only for packaged plugin surfaces.
