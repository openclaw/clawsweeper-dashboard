Keep open. Current main and v2026.6.6 still have the single-registry global hook runner and preserve-gate split, while the linked fixing PR remains open and unmerged, so this issue should continue tracking the regression until that fix or an equivalent lands.

## What This Changes

The issue asks OpenClaw to restore `before_tool_call` and `after_tool_call` hook dispatch for enabled local/global-dir plugins after the 2026.6.5 upgrade regression.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/92016 with a live composed or request-scoped hook registry fix that keeps pinned/local hooks visible for both tool-hook phases, with regression and upgrade proof.

## Reproduction Assessment

Yes. The source path is clear: activate a gateway-bindable registry, then a distinct default/scoped registry; current main can preserve the old runner while before/after tool dispatch consults only that runner.

## Solution Assessment

Yes. Repairing registry and runner lifecycle ownership is the maintainable fix; adding more activation, timeout, or docs-only configuration would not restore fail-closed tool-hook dispatch.
