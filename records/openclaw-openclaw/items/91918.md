## What This Changes

The issue asks OpenClaw to restore `before_tool_call` and `after_tool_call` dispatch for enabled local global-dir plugins after the 2026.5.27 to 2026.6.5 upgrade regression.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/92016, or an equivalent maintainer-approved core runtime fix, so tool-call dispatch sees live local plugin hooks without relying on order-sensitive runner preservation.

## Reproduction Assessment

Yes. Source inspection shows a current-main path where a later active registry can be invisible to the global hook runner that tool-call dispatch consults; I did not run a live upgraded install in this read-only review.

## Solution Assessment

Yes. The best solution is a core registry/runner lifecycle repair, not extra plugin timeout or activation configuration, because the documented hook contract already expects enabled plugins to participate in tool-call dispatch.
