## What This Changes

The issue reports a released regression where enabled local extension `before_tool_call` and `after_tool_call` hooks register but do not fire during agent runs after upgrading to 2026.6.5.

## Best Possible Solution

Review, adjust if needed, and land the linked live-registry composition fix so tool dispatch sees registered local extension hooks across gateway and scoped plugin activations.

## Reproduction Assessment

Yes. Source inspection shows current main can separate the active plugin registry from the global runner consulted by before/after tool-call dispatch, and the reporter provided a concrete 2026.6.5 versus 2026.5.27 regression path.

## Solution Assessment

Yes. Repairing hook-runner registry ownership or composing live active/pinned registries is the narrow maintainable fix; adding more timeout, permission, or activation config would not address the split source of truth.
