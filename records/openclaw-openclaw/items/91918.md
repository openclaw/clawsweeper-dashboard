## Summary

Keep this open: current main and the latest release still have the global hook-runner/active-registry split, and the linked fix is an open unmerged PR rather than shipped behavior.

## What This Changes

The issue reports a stable-release regression where enabled local extension `before_tool_call` and `after_tool_call` hooks register but do not fire during agent tool execution.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/92016, or an equivalent core plugin-runtime fix, so tool-call dispatch sees the canonical live hook registry without making hook plugins rely on new activation or timeout configuration.

## Reproduction Assessment

Yes, at source level with high confidence. Current main preserves a gateway-bound global runner across a later default activation, and tool execution asks only that runner whether `before_tool_call` hooks exist.

## Solution Assessment

Yes, the best path is to fix hook-registry lifecycle ownership rather than add more activation, timeout, or permission configuration. The open linked PR appears to target that owner boundary, but this issue should remain open until that work is merged or replaced.
