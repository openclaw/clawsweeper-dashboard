## Summary

Keep open: the proposed behavior is focused, but this patch would fail TypeScript as written and the external PR still lacks after-fix real behavior proof. The `PAUSED` sentinel is not part of the documented current heartbeat contract, so maintainers should also decide whether to accept and document that new file-level disable state.

## What This Changes

This PR adds an exact, case-insensitive `PAUSED` check in heartbeat preflight so scheduled heartbeat runs skip before making a model call.

## Best Possible Solution

If maintainers want a `PAUSED` sentinel, land it as a documented heartbeat-file contract with the skip-reason type updated and focused regression coverage; otherwise keep `0m` as the explicit disable path.

## Reproduction Assessment

Yes for the observed current behavior: source inspection shows a scheduled heartbeat with `HEARTBEAT.md` containing `PAUSED` is non-empty and falls through to the normal prompt path. I did not run a live provider reproduction.

## Solution Assessment

No as submitted: the new skip reason is not included in the `HeartbeatSkipReason` type, so the branch should fail TypeScript. The maintainable version also needs a documented sentinel contract and a focused regression test.
