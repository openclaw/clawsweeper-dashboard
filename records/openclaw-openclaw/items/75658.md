## Summary

Keep open: current main still does not give `openclaw sessions cleanup` a first-class dry-run/apply plan for archived and orphaned session artifacts, and open PR #75672 is already the active closing implementation candidate.

## What This Changes

The request is for `openclaw sessions cleanup` to report and remove physical `.deleted`, `.reset`, and orphan checkpoint files instead of only showing index-entry cleanup.

## Best Possible Solution

Review and land or reject #75672; the desired end state is explicit artifact cleanup planning that reports bytes/counts in dry-run and JSON output and applies removals under `--enforce`.

## Reproduction Assessment

Yes. A source-level reproduction exists: with no disk budget configured, archive-only and orphan-checkpoint files are not represented in `previewStoreCleanup` output, so dry-run can report zero removals for the reported physical-file buildup.

## Solution Assessment

Yes on direction, pending PR review. Wiring artifact planning into `sessions cleanup` is the narrow maintainable fix; relying on write-time maintenance or optional disk-budget pressure leaves the command behavior incomplete.
