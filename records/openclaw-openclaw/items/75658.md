## Summary

Keep open: current main only partially covers the disk-bloat problem. Enforced store maintenance can prune old archive files, but `openclaw sessions cleanup --dry-run` and the JSON/text summary still lack archive-file planning/reporting for `.deleted` and `.reset` files, and the open closing PR remains the implementation candidate.

## What This Changes

Request `openclaw sessions cleanup` to physically remove and report `.deleted`, `.reset`, and orphan checkpoint session files instead of only updating `sessions.json` entries.

## Best Possible Solution

## Solution Assessment
