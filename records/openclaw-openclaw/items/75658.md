## Summary

Keep open: current main still lacks a first-class `sessions cleanup` phase that plans/reports/applies ordinary archived-file cleanup for dry-run/JSON output and orphan checkpoints, and open PR #75672 remains the active closing implementation candidate.

## What This Changes

The issue asks `openclaw sessions cleanup` to remove physical `.deleted`, `.reset`, and orphan `.checkpoint` session files instead of only pruning `sessions.json` entries.

## Best Possible Solution

Review and land or replace #75672 with an equivalent command-level archived-file cleanup plan that reports dry-run/JSON counts and bytes, applies under enforcement, preserves retention semantics, and handles orphan checkpoints safely.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: with no disk budget configured, the dry-run path reports only entry pruning/capping and `diskBudget: null`, so archived `.deleted`, `.reset`, and orphan checkpoint files are not surfaced as planned removals.

## Solution Assessment

Yes on direction, pending PR review. The maintainable fix belongs in the `sessions cleanup` planning/reporting path, reusing existing archive/checkpoint classifiers and writer-safe apply paths rather than relying only on write-time maintenance or optional disk-budget pressure.
