## Summary

Keep open: current main still lacks a first-class `sessions cleanup` plan/report/apply path for ordinary archived `.deleted`/`.reset` files and orphan checkpoints, and open PR #75672 is the active closing implementation candidate.

## What This Changes

The issue requests that `openclaw sessions cleanup` remove physical `.deleted`, `.reset`, and orphan checkpoint session artifacts instead of only pruning session index entries.

## Best Possible Solution

Review and land or close #75672; the desired end state is a cleanup-command archive maintenance phase that reports counts/bytes in dry-run and JSON output, applies under `--enforce`, and reuses the existing archive/checkpoint classifiers.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: with no disk budget configured, `enforceSessionDiskBudget` returns `null`, while `previewStoreCleanup` only plans index pruning/capping/missing-entry work, so a store containing only `.deleted`, `.reset`, and orphan checkpoint files can report zero removals.

## Solution Assessment

Unclear until #75672 is reviewed. The requested direction is valid, but the maintainable fix should wire physical artifact planning into `openclaw sessions cleanup` itself rather than depending on hidden write-time retention or optional disk-budget pressure.
