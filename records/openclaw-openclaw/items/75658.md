## Summary

Keep open: current main still lacks a `sessions cleanup` phase that reports or removes `.deleted` and `.reset` archive files outside optional disk-budget pressure, and open PR #75672 is already the active closing implementation candidate for https://github.com/openclaw/openclaw/issues/75658.

## What This Changes

The issue requests that `openclaw sessions cleanup` report and remove physical `.deleted`, `.reset`, and orphan checkpoint session artifacts, not only prune `sessions.json` index entries.

## Best Possible Solution

Review and land or reject #75672; the desired end state is a cleanup-command artifact plan that reports counts and bytes in dry-run/JSON output and applies removals under `--enforce` using existing classifiers and retention rules.

## Reproduction Assessment

Yes. A source-level reproduction exists: with no disk budget configured, `enforceSessionDiskBudget` returns `null`, while `previewStoreCleanup` has no standalone archived-file phase, so a store containing only `.deleted`, `.reset`, and orphan checkpoint files can report zero removals.

## Solution Assessment

Yes on direction, pending PR review. The narrow maintainable fix is to wire physical artifact planning into `openclaw sessions cleanup` itself rather than relying on write-time maintenance or optional disk-budget pressure.
