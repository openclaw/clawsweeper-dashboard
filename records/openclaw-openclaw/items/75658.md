## Summary

Keep open: current main still lacks a first-class `sessions cleanup` phase for reporting or removing ordinary `.deleted`, `.reset`, and orphan checkpoint files, and #75672 is the open closing implementation candidate.

## What This Changes

Request `openclaw sessions cleanup` to report and remove archived `.deleted`, `.reset`, and orphan checkpoint session files in addition to index entries.

## Best Possible Solution

Review and land or reject #75672; the desired end state is a cleanup-command archive maintenance phase that reports counts/bytes in dry-run and JSON output and applies under `--enforce`.

## Reproduction Assessment

Yes. A source-level reproduction exists: with no disk budget configured, `enforceSessionDiskBudget` returns `null`, while `previewStoreCleanup` only plans missing/stale/capped/index work, so a store containing only archived files can report zero removals.

## Solution Assessment

Unclear until #75672 is reviewed. The maintainable direction is to wire physical artifact planning into `openclaw sessions cleanup` using the existing archive/checkpoint classifiers and retention settings.
