## Summary

Keep open: current main still lacks an explicit `sessions cleanup` archive-file plan/report/apply path for ordinary `.deleted`, `.reset`, and orphan checkpoint artifacts, and open PR #75672 is the active closing implementation candidate.

## What This Changes

Add `openclaw sessions cleanup` support for reporting and removing physical `.deleted`, `.reset`, and orphan checkpoint session files instead of only pruning `sessions.json` entries.

## Best Possible Solution

Review and land or close #75672; the desired end state is a cleanup-command archive maintenance phase that reports counts/bytes in dry-run and JSON output, applies under `--enforce`, and reuses the existing archive/checkpoint classifiers.

## Reproduction Assessment

Yes. With no disk budget configured, `enforceSessionDiskBudget` returns `null`, while `previewStoreCleanup` only plans missing/stale/capped entry mutations, so a store containing only `.jsonl.deleted.*`, `.jsonl.reset.*`, and unreferenced checkpoint files can report zero removals.

## Solution Assessment

Unclear until #75672 is reviewed. The requested direction is valid, but the maintainable fix should wire physical artifact planning into `openclaw sessions cleanup` itself rather than depending on hidden write-time retention or optional disk-budget pressure.
