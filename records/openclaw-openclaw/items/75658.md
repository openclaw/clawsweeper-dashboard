## Summary

Keep open: current main and the latest release still leave stale `.deleted.*` and `.reset.*` archive pruning outside explicit `openclaw sessions cleanup`, and the open draft implementation candidate at https://github.com/openclaw/openclaw/pull/89288 uses closing syntax for this issue.

## What This Changes

`openclaw sessions cleanup` should remove stale physical deleted/reset session archives and orphaned checkpoint artifacts instead of only pruning `sessions.json` entries.

## Best Possible Solution

Land or replace the linked PR so explicit session cleanup uses the same archive-retention contract as normal maintenance and proves dry-run/apply behavior for deleted and reset archives.

## Reproduction Assessment

Yes, at source level: current `sessions cleanup` calls the generic unreferenced-artifact sweep, and that classifier excludes archive-suffixed `.deleted.*` and `.reset.*` transcripts. I did not run a live filesystem repro because this review is read-only.

## Solution Assessment

Yes for keeping it open: current main and `v2026.6.6` do not solve the explicit cleanup gap, and the open linked PR is the right coordination point. The final implementation should share the existing archive-retention owner rather than create a drifting duplicate policy.
