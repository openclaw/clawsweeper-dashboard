## Summary

Keep open: current main still lacks explicit `sessions cleanup` artifact planning/reporting for `.deleted`, `.reset`, and orphan checkpoint files outside optional disk-budget pressure, and #75672 is already the open closing implementation candidate.

## What This Changes

The issue requests that `openclaw sessions cleanup` report and remove physical deleted/reset transcript archives and orphaned checkpoint artifacts, not only prune `sessions.json` entries.

## Best Possible Solution

Review and land or reject #75672; the desired end state is a cleanup-command artifact plan that reports counts/bytes in dry-run and JSON output and applies removals under `--enforce` using existing retention and reference rules.

## Reproduction Assessment

Yes. A source-level reproduction exists: with no disk budget configured, `enforceSessionDiskBudget` returns `null`, while `previewStoreCleanup` only plans index pruning/capping/missing-entry work, so archive-only stores can report zero removals.

## Solution Assessment

Yes on direction, pending PR review. The narrow maintainable fix is to wire standalone physical artifact planning into `openclaw sessions cleanup` instead of relying on optional disk-budget pressure or unreported write-time cleanup.
