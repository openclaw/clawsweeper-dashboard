## Summary

Keep open: current main still only partially covers the reported cleanup gap, and there is an open PR that explicitly closes this issue. The issue should remain open until that PR is reviewed and merged, replaced, or closed.

## What This Changes

The issue requests that `openclaw sessions cleanup` report and remove physical `.deleted`, `.reset`, orphan checkpoint, and related session artifact files instead of only pruning `sessions.json` entries.

## Best Possible Solution

Review and land PR #75672, or a narrow replacement, that adds explicit archived/reset/checkpoint artifact planning to `openclaw sessions cleanup`, reports counts and bytes in dry-run/JSON output, and preserves the existing retention and safe-directory boundaries.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: with default `maxDiskBytes` and `highWaterBytes` unset, the disk-budget path returns `null`, and the dry-run path reports only missing/stale/capped index work, so archived files can remain invisible to cleanup output.

## Solution Assessment

Yes, the linked PR direction is the right shape, but current main is not the solution yet. The maintainable fix is a first-class artifact cleanup plan in the CLI rather than relying on retention cleanup hidden inside store writes or budget-pressure cleanup.
