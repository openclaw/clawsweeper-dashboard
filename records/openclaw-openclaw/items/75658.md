## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/97103, or an equivalent replacement, reusing the existing archive-retention helper and preserving `resetArchiveRetention` semantics.

## Reproduction Assessment

Yes. Source inspection shows current dry-run cleanup uses the unreferenced-artifact sweep, whose predicate excludes deleted/reset archive names, and the CLI renderer has no deleted/reset archive count; I did not run a live filesystem repro in this read-only review.

## Solution Assessment

Yes for the narrow issue. The best fix is to reuse the existing archive-retention helper for preview/apply reporting rather than adding a parallel expiration policy; the open linked PR follows that direction and should be reviewed or replaced.
