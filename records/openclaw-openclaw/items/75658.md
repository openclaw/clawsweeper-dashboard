## Summary

Keep open: current `main` still lacks standalone `sessions cleanup` planning/reporting/apply behavior for `.deleted`, `.reset`, and orphan checkpoint files when no disk budget is configured, and #75672 is already the open closing implementation candidate.

## What This Changes

Make `openclaw sessions cleanup` report and remove archived deleted/reset transcript files plus orphan checkpoint files from disk, not only prune `sessions.json` entries.

## Best Possible Solution

Review and land or reject #75672; the desired end state is a cleanup-command artifact plan that reports counts/bytes in dry-run and JSON output and applies removals under existing retention/reference rules.

## Reproduction Assessment

Yes. A source-level reproduction exists: with no disk budget configured, `enforceSessionDiskBudget` returns `null`, while `previewStoreCleanup` only plans index pruning/capping/missing-entry work, so archive-only stores can report zero removals.

## Solution Assessment

Yes on direction, pending PR review. The narrow maintainable fix is to wire standalone physical artifact planning into `openclaw sessions cleanup` instead of relying on optional disk-budget pressure or unreported write-time cleanup.
