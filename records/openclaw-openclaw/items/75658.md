## Summary

Keep open. Current main still lacks first-class physical artifact cleanup in `openclaw sessions cleanup`, and open PR #75672 uses closing syntax for this issue, so the issue should remain open until that PR is merged, replaced, or closed.

## What This Changes

The issue requests that `openclaw sessions cleanup` report and remove physical `.deleted`, `.reset`, trajectory, and orphan checkpoint session artifacts instead of only pruning `sessions.json` index entries.

## Best Possible Solution

Review and land or replace #75672 with a cleanup implementation that adds an explicit archived-file planning phase to the CLI, reports counts/bytes in dry-run and JSON output, applies the same plan under `--enforce`, and preserves referenced checkpoint files by exact path.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: with no session disk budget configured, `enforceSessionDiskBudget` returns `null` while `previewStoreCleanup` reports only index and budget changes, so a store containing only `.deleted`, `.reset`, and orphan checkpoint artifacts can dry-run with zero removals.

## Solution Assessment

Yes for the requested product direction. The maintainable fix should wire physical artifact planning into `openclaw sessions cleanup` itself, reusing the existing archive parsers and exact checkpoint-reference logic rather than relying on hidden write-time retention or optional budget pressure.
