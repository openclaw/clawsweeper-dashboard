## Summary

Keep open: current main still lacks a first-class `sessions cleanup` report/apply path for ordinary archived `.deleted`/`.reset` files and orphan checkpoints, and #75672 is the active open closing PR.

## What This Changes

The issue asks for `openclaw sessions cleanup` to report and remove archived `.deleted`, `.reset`, and orphan checkpoint files in addition to pruning `sessions.json` index entries.

## Best Possible Solution

Review and land or close #75672; the desired end state is a cleanup-command archive maintenance phase that reports counts and bytes in dry-run/JSON output and applies under `--enforce`.

## Reproduction Assessment

Yes. A source-level reproduction exists: with no disk budget configured, `enforceSessionDiskBudget` returns `null`, while `previewStoreCleanup` only plans missing/stale/capped entry work, so a store containing only archived files can report zero removals.

## Solution Assessment

Unclear until #75672 is reviewed. The requested direction is valid, and the maintainable fix should wire physical artifact planning into `openclaw sessions cleanup` instead of relying only on write-time maintenance or optional disk-budget pressure.
