## Summary

Keep open: current `main` still has no standalone `sessions cleanup` phase that reports or removes `.deleted`, `.reset`, and orphan checkpoint artifacts, and PR #75672 is already the active closing implementation candidate.

## What This Changes

The issue requests that `openclaw sessions cleanup` remove and dry-run report physical archived/reset/deleted/orphaned session files, not only session index entries.

## Best Possible Solution

Add command-level artifact planning/reporting/apply wiring to `runSessionsCleanup`, reusing the existing artifact parsers and retention config instead of relying on disk-budget pressure.

## Reproduction Assessment

Yes. A source-level reproduction exists: with no disk budget configured, `enforceSessionDiskBudget` returns `null`, and `previewStoreCleanup` only plans index pruning/capping/missing-entry work.

## Solution Assessment

Yes on direction, pending PR review. The narrow maintainable fix is to wire physical artifact cleanup into `openclaw sessions cleanup` itself and review or replace PR #75672 rather than closing the issue early.
