## Summary

Keep open: current main partly addresses orphaned primary/checkpoint/trajectory artifacts, but it still does not provide standalone `sessions cleanup` planning/reporting for `.deleted` and `.reset` archive files; an open linked PR is already the closing implementation candidate.

## What This Changes

The issue requests that `openclaw sessions cleanup` remove and report physical `.deleted`, `.reset`, and orphan checkpoint session artifacts instead of only pruning `sessions.json` entries.

## Best Possible Solution

Review and land or reject https://github.com/openclaw/openclaw/pull/75672, with dry-run text, JSON summary, and enforce behavior covering archive files while preserving configured retention rules.

## Reproduction Assessment

Yes. A high-confidence source-level path exists: with no disk budget configured, disk-budget cleanup returns `null`, and the standalone unreferenced sweep excludes `.deleted` and `.reset` archive artifacts.

## Solution Assessment

Yes on direction. The maintainable fix is to add archive-aware planning/reporting to `openclaw sessions cleanup`, not to depend on optional disk-budget pressure or unreported store-write maintenance.
