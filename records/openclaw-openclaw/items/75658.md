## Summary

Keep open: current main still lacks command-level artifact planning/reporting for archive-only `.deleted`, `.reset`, and orphan checkpoint cleanup, and #75672 is already the open closing implementation candidate.

## What This Changes

Request that `openclaw sessions cleanup` report and remove physical `.deleted`, `.reset`, and orphan checkpoint artifacts instead of only pruning `sessions.json` entries.

## Best Possible Solution

Review and land or reject #75672; the desired end state is a cleanup command that reports artifact counts/bytes in dry-run and JSON output and removes eligible files under `--enforce` using existing retention and reference rules.

## Reproduction Assessment

Yes. A source-level reproduction exists: with no disk budget configured, `enforceSessionDiskBudget` returns `null` and `previewStoreCleanup` only plans index changes, so archive-only stores can report no cleanup.

## Solution Assessment

Yes on direction, pending PR review. The narrow maintainable fix is to wire standalone physical artifact planning into `openclaw sessions cleanup` instead of relying on optional disk-budget pressure or unreported write-time archive cleanup.
