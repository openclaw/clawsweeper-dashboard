## Summary

Keep open. Current `main` still only partially addresses the reported disk growth: archived `.deleted`/`.reset` cleanup exists in write-time maintenance and disk-budget enforcement, but `openclaw sessions cleanup --dry-run` does not independently plan/report ordinary archived file cleanup, and orphan checkpoints are only removed when a disk budget is configured and exceeded. An open implementation PR, #75672, already uses closing syntax for this issue, so the issue should remain open until that PR is reviewed and merged or closed.

## What This Changes

The issue requests that `openclaw sessions cleanup` report and remove physical archived/reset/deleted session files and orphan checkpoints, not only prune `sessions.json` index entries.

## Best Possible Solution

Review and land or close #75672. The desired end state is a first-class physical artifact maintenance phase in `openclaw sessions cleanup` that reports counts/bytes in dry-run and JSON output, applies the same plan under `--enforce`, uses the existing archive/checkpoint classifiers, and keeps any full-purge behavior behind an explicit documented option.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: with no session disk budget configured, `enforceSessionDiskBudget` returns `null`, while `previewStoreCleanup` only reports index pruning/capping/missing counts, so a store containing only `.deleted`, `.reset`, and orphan checkpoint artifacts can dry-run with zero file removals.

## Solution Assessment

Unclear until #75672 is reviewed. The requested direction is valid, but the maintainable implementation should wire physical artifact planning into the cleanup command itself rather than relying on hidden write-time retention or optional disk-budget pressure.
