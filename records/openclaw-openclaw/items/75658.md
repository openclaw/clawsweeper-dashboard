## Summary

Keep open: current main still lacks a `sessions cleanup` phase that reports and removes `.deleted`, `.reset`, and orphan checkpoint files outside optional disk-budget pressure, and open PR #75672 is already the active closing implementation candidate.

## What This Changes

`openclaw sessions cleanup` should remove physical `.deleted`, `.reset`, and orphan checkpoint session artifacts, not only prune `sessions.json` index entries.

## Best Possible Solution

Review and land or reject #75672; the desired end state is a cleanup-command artifact plan that reports counts/bytes in dry-run and JSON output and applies removals under `--enforce` using the existing classifiers and retention rules.

## Reproduction Assessment

Yes. A source-level reproduction exists: with no disk budget configured, `enforceSessionDiskBudget` returns `null`, while `previewStoreCleanup` only plans index pruning/capping/missing-entry work, so a store containing only `.deleted`, `.reset`, and orphan checkpoint files can report zero removals.

## Solution Assessment

Yes on direction, pending PR review. The narrow maintainable fix is to wire physical artifact planning into `openclaw sessions cleanup` itself rather than relying on write-time maintenance or optional disk-budget pressure.
