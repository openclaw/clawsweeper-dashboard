## Summary

Keep open: current main still lacks a first-class `sessions cleanup` phase for ordinary `.deleted`, `.reset`, and orphan checkpoint files, and PR #75672 is the open closing implementation candidate.

## What This Changes

Request `openclaw sessions cleanup` to plan, report, and apply physical cleanup of archived `.deleted`, `.reset`, orphan checkpoint, and optional old trajectory session files instead of only cleaning session index entries.

## Best Possible Solution

Review and land or reject PR #75672; the desired end state is a cleanup-command archive maintenance phase with dry-run and JSON byte/count reporting plus guarded `--enforce` deletion.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: with disk-budget thresholds unset, `enforceSessionDiskBudget` returns `null`, while `previewStoreCleanup` only plans index pruning/capping/missing-entry work.

## Solution Assessment

Yes on direction, pending PR review. The narrow maintainable fix is to wire archived-file planning into `openclaw sessions cleanup` using existing classifiers and retention thresholds, rather than relying on write-time maintenance or budget pressure.
