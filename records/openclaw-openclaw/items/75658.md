## Summary

Keep open: current main only partially covers this. Physical archive cleanup exists for aged `.deleted`/`.reset` files during enforced store writes, and orphan artifacts can be removed under a configured disk budget, but the CLI dry-run path still does not preview ordinary archive-retention cleanup and orphan checkpoints are not cleaned unless disk-budget pressure is configured.

## What This Changes

The issue requests that `openclaw sessions cleanup` remove and report physical archived/reset session files and orphaned checkpoint/trajectory artifacts, not only prune `sessions.json` entries.

## Best Possible Solution

Add a dedicated physical-artifact maintenance phase for `openclaw sessions cleanup` that uses the existing archive/checkpoint/trajectory classifiers, reports counts and bytes in dry-run and JSON output, applies the same result under `--enforce`, and only purges all retained archives behind an explicit, documented option if maintainers want that behavior.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: with default `maxDiskBytes: null`, `enforceSessionDiskBudget` returns `null`, so a dry run over a store containing only physical `.deleted`/`.reset` archives and orphan checkpoints can report zero file removals unless index pruning/capping also happens.

## Solution Assessment

No. The current implementation is a useful partial solution, but the maintainable fix is to make physical artifact sweeping a first-class cleanup report/apply path instead of relying on age-based cleanup hidden inside store writes or disk-budget pressure.
