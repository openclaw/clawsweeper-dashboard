## What This Changes

The issue reports that isolated cron sessions on v2026.5.18 downgrade `thinkingDefault: low` to `off` for `google/gemini-3-flash-preview`, despite the same configuration reportedly working in v2026.4.23.

## Best Possible Solution

Make Google Gemini 3 Flash thinking support authoritative at the Google provider/catalog boundary, then cover isolated cron `thinkingDefault: low` so cron keeps `low` for `google/gemini-3-flash-preview` without weakening off-only semantics for unrelated models.

## Reproduction Assessment

Yes, at source level: a catalog row with `reasoning: false` makes `low` unsupported before the Google provider profile can advertise Gemini 3 Flash levels, and the isolated cron path then downgrades to `off`. I did not run the live Google cron path.

## Solution Assessment

Yes: a narrow provider/catalog capability fix is the maintainable solution. A fail-closed cron policy for truly unsupported thinking configs is a separate behavior choice and should not be required to fix this supported Gemini 3 Flash case.
