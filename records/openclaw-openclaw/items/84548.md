## What This Changes

The issue reports that isolated cron sessions in v2026.5.18 downgrade `thinkingDefault: low` to `off` for `google/gemini-3-flash-preview`, although the same configuration produced thinking blocks in v2026.4.23.

## Best Possible Solution

Restore Google Gemini 3 Flash thinking support at the provider/catalog boundary so `thinkingDefault: low` remains valid in isolated cron, with regression coverage for catalog rows that would otherwise report `reasoning: false`.

## Reproduction Assessment

Yes, at source level: a catalog row with `reasoning: false` makes `low` unsupported before the Google provider profile is considered, and isolated cron then downgrades to `off`. I did not run the live cron path against Google credentials.

## Solution Assessment

Yes for the central regression: the narrowest maintainable fix is at the Google provider thinking/catalog boundary, with cron regression coverage. Startup-time refusal for unsupported explicit thinking is a separate policy decision and should not be coupled to restoring valid Gemini 3 Flash levels.
