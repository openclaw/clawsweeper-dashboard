## What This Changes

The issue reports that isolated cron sessions on `google/gemini-3-flash-preview` downgrade `thinkingDefault: low` to `off` in v2026.5.18 despite earlier cron runs producing Gemini thinking blocks.

## Best Possible Solution

Make Google Gemini 3 thinking support authoritative at the provider profile/catalog boundary, preserve explicit `reasoning:false` semantics for unrelated providers, and add cron regression coverage for `google/gemini-3-flash-preview` with conflicting catalog metadata.

## Solution Assessment

Yes for the central regression: a narrow provider-profile/catalog precedence fix with cron coverage is the maintainable path. The warn-versus-refuse behavior should be treated as a separate maintainer policy choice if pursued.
