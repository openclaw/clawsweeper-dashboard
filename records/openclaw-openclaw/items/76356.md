## Summary

Keep open. The PR targets a real subagent lifecycle bug from #58776, but the current diff has blocking correctness issues: the new startup failure event can be followed by a normal lifecycle end, and one added import creates a runtime static import cycle.

## What This Changes

The PR adds first-progress and startup-failed embedded subagent lifecycle signals, wires consumers to treat startup-failed as terminal failure, and updates docs, changelog, and focused tests.

## Best Possible Solution

Keep the branch direction, but make startup-failed an exclusive terminal lifecycle event and move first-progress emission into a leaf helper before rerunning lifecycle and import-cycle checks.

## Reproduction Assessment

Yes. Source inspection shows current main lacks first-progress/startup-failed, and the PR's new startup-failed branch would still fall through to the existing normal end emission path.

## Solution Assessment

No. The proposed lifecycle phases are the right shape, but the patch needs to avoid the terminal fallthrough and the new static import cycle before it is a maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P2] Return after emitting startup-failed:** `src/agents/pi-embedded-subscribe.handlers.lifecycle.ts:147`
  - body: The startup-aborted path emits `startup-failed` and then falls through to the existing `end` emission. Because lifecycle listeners are synchronous fanout into async handlers, the later `end` can be processed as a successful terminal event and overwrite the startup failure for `agent.wait` or subagent completion.
  - confidence: 0.92
- **[P2] Move the progress helper out of the lifecycle module:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:23`
  - body: `lifecycle.ts` already value-imports helpers from `handlers.messages.ts`, so adding a value import from `handlers.messages.ts` back to `lifecycle.ts` creates a runtime static import cycle. Please put the shared progress emitter in a leaf helper that both modules can import.
  - confidence: 0.88
