## What This Changes

The issue reports that Claude CLI approval responses still return the deprecated `{ behavior: "allow" }` permission result shape, causing Claude Code 2.1.156 to reject tool calls with a ZodError.

## Best Possible Solution

Land one complete linked fix that updates all Claude permission allow response paths to return an `updatedInput` record carrying the tool input where available, keeps deny behavior unchanged, and refreshes regression coverage for both response surfaces.
