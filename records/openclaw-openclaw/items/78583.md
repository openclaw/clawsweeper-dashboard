## Summary

Keep open. This is a broad new opt-in AgentKit/World ID feature that is not on current main, and the supplied real-behavior proof is meaningful, but the patch introduces a blocking security/API-boundary issue by exporting an operator.admin gateway client through the public plugin SDK while `resolveVerified` trusts a caller-supplied plugin id.

## What This Changes

Adds an opt-in bundled AgentKit plugin with World ID HITL approvals, proof-backed approval resolution, TUI/chat/channel approval actions, CLI/status helpers, docs, dependencies, and focused tests.

## Best Possible Solution

Keep AgentKit opt-in, but replace the public operator.admin helper exposure with a narrower plugin-owned proof resolver or caller-bound capability before merging the World HITL path.

## Reproduction Assessment

Not applicable as a feature PR rather than a current-main bug. The after-fix behavior has a high-confidence proof path from the PR body's isolated hosted HITL run, copied TUI/gateway output, MP4 attachment, and successful Real behavior proof check.

## Solution Assessment

No. The feature direction may be reasonable, but exporting a general operator.admin gateway helper through the public plugin SDK is not the narrowest maintainable way to solve proof-backed approvals; the safer path is a plugin-owned resolver that binds the caller to the pending approval owner/proof context.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.78

Full review comments:

- **[P1] Keep admin gateway helpers out of the public SDK:** `src/plugin-sdk/gateway-runtime.ts:38-41`
  - body: This export makes `withOperatorAdminGatewayClient` available from the public `openclaw/plugin-sdk/gateway-runtime` subpath, so any plugin can open an `operator.admin` client and call `plugin.approval.resolveVerified` with a matching `pluginId`. Since the gateway resolver only compares that caller-supplied id to the pending request, a different plugin can bypass the proof-owned AgentKit allow path. Keep the admin helper internal or add a resolver capability that binds the caller to the owning plugin/proof flow.
  - confidence: 0.82
