## Summary

Keep open: the feature is not obsolete on current main, but the PR still has a security-sensitive SDK trust-boundary blocker around public operator-admin gateway access and proof-owned approval resolution.

## What This Changes

Adds an opt-in bundled AgentKit plugin with World ID human-in-the-loop approvals, proof-backed plugin approval resolution, approval actions across TUI/chat/channels, CLI/status helpers, docs, tests, and World/viem dependencies.

## Best Possible Solution

Keep AgentKit opt-in, but replace the public operator-admin helper exposure with a narrow caller-bound proof resolver before merge.

## Reproduction Assessment

Not applicable as a feature/security-hardening PR rather than a current-main bug. The after-fix behavior has a strong proof path from the PR body, redacted live output, the MP4 attachment, and passing Real behavior proof checks.

## Solution Assessment

No. The feature direction may be reasonable, but exporting general operator-admin gateway helpers through a public plugin SDK subpath is not the narrowest maintainable way to solve proof-backed approvals.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.78

Full review comments:

- **[P1] Keep admin gateway helpers out of the public SDK:** `src/plugin-sdk/gateway-runtime.ts:38-41`
  - body: This export makes `withOperatorAdminGatewayClient` available from the public `openclaw/plugin-sdk/gateway-runtime` subpath, while `plugin.approval.resolveVerified` trusts a caller-provided `pluginId` after only comparing it to the pending request.   - confidence: 0.82
