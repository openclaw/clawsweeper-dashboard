## Review Findings

Overall confidence: 0.88

Full review comments:

- **[P2] Ship AgentKit runtime deps with packaged OpenClaw:** `extensions/agentkit/package.json:7-10`
  - body: This adds AgentKit as an included bundled plugin, but its World/viem runtime modules are only declared in the plugin package and `world-id.runtime.ts` still resolves `@worldcoin/idkit-core` plus its WASM file at runtime. Packaged OpenClaw excludes `dist/extensions/*/node_modules/**` and startup does not install bundled plugin deps, so npm-installed OpenClaw can ship AgentKit code without the modules/assets it needs. Either externalize AgentKit or root-own/internalize these deps with package-acceptance proof.
  - confidence: 0.9
- **[P2] Raise AgentKit host compatibility metadata:** `extensions/agentkit/package.json:17-34`
  - body: The package still advertises 2026.4.15-era host compatibility even though it depends on APIs added by this PR, including `plugin.approval.resolveVerified`, command approval actions, and richer `chat.inject` payloads. Older hosts satisfying this range cannot run the plugin correctly, so align the package version, peer range, plugin API range, and min host version with the first OpenClaw release that includes those APIs.
  - confidence: 0.87
