## Summary

Keep this PR open: the AgentKit feature is not already on current main and has useful proof, but the current patch still has a blocking SDK/security trust-boundary issue around proof-backed approval resolution.

## What This Changes

The PR adds an opt-in bundled AgentKit plugin with World-backed HITL approvals, proof-backed approval resolution, TUI/chat/channel approval actions, docs, dependencies, generated baselines, and tests.

## Best Possible Solution

Keep AgentKit opt-in, but replace the public operator-admin helper exposure with a narrow caller-bound or internal proof resolver before merge.

## Reproduction Assessment

Not applicable: this is a feature/security-hardening PR rather than a current-main bug report. The after-fix behavior has supplied proof through redacted live TUI/gateway output and a passing Real behavior proof check.

## Solution Assessment

No. The feature direction may be reasonable, but exporting general operator-admin gateway helpers through a public plugin SDK subpath is not the narrowest maintainable way to solve proof-backed approvals.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.78

Full review comments:

- **[P1] Keep admin gateway helpers out of the public SDK:** `src/plugin-sdk/gateway-runtime.ts:38-41`
  - body: This export makes withOperatorAdminGatewayClient available from the public openclaw/plugin-sdk/gateway-runtime subpath, while plugin.approval.resolveVerified trusts a caller-provided pluginId after only comparing it to the pending request.
