## Summary

This PR is useful but should stay open: current head still ships AgentKit as an included bundled plugin while its World runtime dependencies are only plugin-local, and the package metadata says older hosts can run APIs introduced by this PR.

## What This Changes

Adds an opt-in bundled AgentKit plugin with World-backed HITL approvals, gateway approval API extensions, TUI/channel approval actions, docs, dependencies, and tests.

## Best Possible Solution

Decide the distribution boundary first; if AgentKit remains bundled, root-own or internalize the World runtime dependencies, raise compatibility metadata, and require package-acceptance plus security review.

## Reproduction Assessment

Not applicable for an existing bug; this is a new AgentKit HITL feature and security boundary. The merge blockers are source-reproducible from package metadata, current-main API absence, and the documented bundled-plugin dependency contract.

## Solution Assessment

No, not yet. The approval seam may be a reasonable direction, but the current PR is not merge-safe until dependency ownership and host compatibility metadata match the intended distribution path.
