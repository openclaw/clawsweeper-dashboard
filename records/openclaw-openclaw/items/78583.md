## Summary

Keep this PR open: it is active and not implemented on current main, but the current head still exposes an admin-scoped gateway helper through a public Plugin SDK subpath, so it needs a security/SDK boundary decision before merge.

## What This Changes

The PR adds an opt-in bundled AgentKit plugin with World-backed HITL approvals, gateway/plugin approval protocol changes, TUI/chat/channel approval actions, docs, generated baselines, and tests.

## Best Possible Solution

Keep AgentKit opt-in, but replace the public operator-admin helper exposure with a narrow proof resolver that binds resolution to the owning plugin/proof context.

## Reproduction Assessment

Not applicable: this is a feature PR, not a current-main bug report. The after-fix behavior has supplied proof through redacted live TUI/gateway output and a GitHub-hosted MP4 attachment.

## Solution Assessment

No. The feature direction may be reasonable, but exporting a general operator-admin gateway helper through a public Plugin SDK subpath is not the narrowest maintainable way to solve proof-backed approvals.
