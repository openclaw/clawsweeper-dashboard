## What This Changes

The PR adds an opt-in bundled AgentKit plugin for World-backed HITL approvals, a verified plugin-approval resolver, TUI/channel approval actions, docs, changelog, dependencies, and tests.

## Best Possible Solution

Decide the AgentKit distribution boundary first; if it remains bundled, make the runtime dependency path package-safe, align compatibility metadata to the first supporting host release, and run package acceptance plus security review for the verified resolver.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Ship AgentKit runtime deps with packaged OpenClaw:** `extensions/agentkit/package.json:7-10`
  - body: This adds AgentKit as an included bundled plugin, but its World/viem runtime modules are declared only in the plugin package. Packaged OpenClaw excludes extension node_modules and gateway startup/reload do not install plugin deps, so an npm-installed OpenClaw can fail to load AgentKit; either externalize the plugin or root-own/internalize these deps with package-acceptance proof.
  - confidence: 0.89
- **[P2] Raise AgentKit host compatibility metadata:** `extensions/agentkit/package.json:16-35`
  - body: The package still advertises 2026.4.15-era host compatibility even though it depends on APIs added by this PR, including plugin.approval.resolveVerified, command approval actions, and operator-admin gateway helpers. Align package version, peer range, pluginApi range, and minHostVersion with the first host release that includes those APIs.
  - confidence: 0.86
