## Summary

Keep open. The PR is active and has useful real HITL proof, but it introduces a blocking package compatibility metadata bug: the new AgentKit plugin advertises compatibility with older OpenClaw/plugin API versions while importing SDK and gateway surfaces added by this PR.

## What This Changes

The PR adds an opt-in AgentKit plugin with World-backed human-in-the-loop approvals across gateway, TUI, chat/channel approval rendering, docs, and tests.

## Best Possible Solution

Keep the opt-in plugin direction, but align the package compatibility/version metadata with the first OpenClaw release that includes the new SDK and gateway method before merge.

## Reproduction Assessment

Yes, by source inspection. The PR head package metadata allows older hosts, while the new plugin imports an SDK export and gateway method that are absent from current main before this PR.

## Solution Assessment

No. The feature path is plausible, but the package should not advertise compatibility with hosts that cannot load its required SDK/gateway surface.
