## Summary

Keep open: the feature is not on current main and the PR has sufficient real behavior proof, but the current head still leaves a security boundary problem in the proof-backed approval resolver path.

## What This Changes

Adds an opt-in bundled AgentKit plugin with World ID HITL approvals, proof-backed plugin approval resolution, TUI/chat/channel approval actions, CLI/status helpers, docs, generated baselines, and new World/viem dependencies.

## Best Possible Solution

Keep AgentKit opt-in, but replace the public operator-admin helper exposure with a narrow proof resolver that binds the caller to the owning plugin/proof context before merging.

## Reproduction Assessment

Not applicable as a feature PR rather than a current-main bug. The after-fix behavior has a strong proof path from the PR body, redacted live TUI/gateway output, the MP4 attachment, and the passing Real behavior proof check.

## Solution Assessment

No. The feature direction may be reasonable, but exporting a general operator-admin gateway helper through a public plugin SDK subpath is not the narrowest maintainable way to solve proof-backed approvals.

  - body: This export makes `withOperatorAdminGatewayClient` available from the public `openclaw/plugin-sdk/gateway-runtime` subpath, while `plugin.approval.resolveVerified` trusts a caller-provided `pluginId` after only comparing it to the pending request.
