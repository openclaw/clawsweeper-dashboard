## Summary

Keep open: the latest head still exposes a generic `operator.admin` gateway client through the public plugin SDK for AgentKit's narrower proof-backed approval flow, which is a blocking security and product-boundary issue despite supplied real behavior proof.

## What This Changes

This PR adds an opt-in bundled AgentKit plugin with World-backed HITL approvals, approval protocol/rendering extensions, TUI/channel handling, docs, tests, and World dependency metadata.

## Best Possible Solution

Keep AgentKit opt-in, but replace the public generic admin SDK helper with a narrow, versioned proof-backed approval and controlled session-message capability before landing.

## Reproduction Assessment

Not applicable as a current-main bug reproduction: this PR adds a new opt-in integration. The blocker is source-reproducible on the PR head from the public SDK export, the `operator.admin` helper implementation, and AgentKit call sites.

## Solution Assessment

No as written: the proof-backed AgentKit approval direction is coherent, but exporting a generic `operator.admin` gateway helper through the public plugin SDK is broader than the feature needs. A narrow SDK capability for verified approval resolution and controlled session-message injection is safer.
