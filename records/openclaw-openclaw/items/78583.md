## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Ship AgentKit dependencies in packaged OpenClaw:** `extensions/agentkit/package.json:7-10`
  - body: This PR adds AgentKit as an included bundled plugin, but its runtime modules are declared only in the extension package. Packaged OpenClaw installs do not install plugin package dependencies during startup/postinstall, and root package.json neither declares these World/viem deps nor excludes `dist/extensions/agentkit`, so enabling the bundled plugin or CLI from an npm-installed OpenClaw can fail with missing modules. Either make AgentKit external/downloadable or root-own/internalize these runtime deps with package-acceptance coverage.
  - confidence: 0.88
- **[P2] Raise AgentKit host compatibility metadata:** `extensions/agentkit/package.json:17-34`
  - body: The package still advertises OpenClaw 2026.4.15-era compatibility, but the plugin uses SDK/gateway seams introduced by this PR such as `plugin.approval.resolveVerified`, approval action templates, and operator-admin gateway helpers. An external npm/ClawHub install can pass compatibility checks on an older host and then fail at runtime, so align the package version, peer range, pluginApi range, and minHostVersion with the first host version that includes these APIs.
  - confidence: 0.86
