## Best Possible Solution

Decide the distribution boundary first; if AgentKit remains bundled, root-own or internalize the World runtime dependencies, raise compatibility metadata to the first supporting OpenClaw release, and require package-acceptance plus security review before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Ship AgentKit runtime deps with packaged OpenClaw:** `extensions/agentkit/package.json:7-10`
  - body: This adds AgentKit as an included bundled plugin, but its World/viem runtime modules are only declared in the plugin package. Packaged OpenClaw excludes extension node_modules and does not install bundled plugin deps at startup, so npm-installed OpenClaw can ship AgentKit code without the modules it imports; either externalize the plugin or root-own/internalize these deps with package-acceptance proof.
  - confidence: 0.9
- **[P2] Raise AgentKit host compatibility metadata:** `extensions/agentkit/package.json:17-34`
  - body: The package still advertises 2026.4.15-era host compatibility even though it depends on APIs added by this PR, including plugin.approval.resolveVerified, command approval actions, and richer chat.inject payloads. Older hosts satisfying this range cannot run the plugin correctly, so align the package version, peer range, pluginApi range, and minHostVersion with the first host release that includes those APIs.
  - confidence: 0.86
