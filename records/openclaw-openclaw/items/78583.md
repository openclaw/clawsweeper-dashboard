## What This Changes

The PR adds a bundled opt-in AgentKit plugin with World-backed HITL approvals plus gateway approval APIs, TUI/chat/channel approval actions, docs, dependency metadata, generated artifacts, and tests.

Decide whether AgentKit belongs bundled or external; if bundled, root-own the runtime dependencies, raise compatibility metadata to the first release with the new APIs, add package-acceptance proof, and keep the proof-resolver trust boundary reviewed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Ship AgentKit deps with bundled package:** `extensions/agentkit/package.json:7-10`
  - body: This adds AgentKit as an included bundled plugin, but its runtime modules are declared only in the plugin package. Packaged OpenClaw includes dist/extensions/agentkit while excluding extension node_modules, and the root package does not declare the World/viem deps, so an npm-installed OpenClaw can fail to load AgentKit with missing modules. Either externalize the plugin or root-own/internalize these deps with package-acceptance proof.
  - confidence: 0.89
- **[P2] Raise AgentKit host compatibility metadata:** `extensions/agentkit/package.json:17-34`
  - body: The package still advertises 2026.4.15-era host compatibility even though it depends on new host APIs added by this PR, including plugin.approval.resolveVerified, approval action templates, and operator-admin gateway helpers. An external install could pass compatibility checks on an older host and then fail at runtime; align package version, peer range, pluginApi range, and minHostVersion with the first host release that includes those APIs.
  - confidence: 0.86
