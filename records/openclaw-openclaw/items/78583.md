## Summary

Keep open: the latest head has real behavior proof and green checks, but it still leaves AgentKit's World/viem runtime dependencies package-unsafe and advertises compatibility with host versions that do not contain the new approval APIs.

## Best Possible Solution

Decide the AgentKit distribution boundary first; if it remains bundled, root-own or internalize the World runtime dependencies, align compatibility metadata to the first supporting host release, and require package-acceptance plus security review before merge.
