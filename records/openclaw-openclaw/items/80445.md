## Summary

Keep this PR open: it targets a valid duplicate visible-delivery bug, but the latest head still has a blocking outbound idempotency correctness issue and the supplied proof is explicitly mocked/synthetic rather than real Gateway/Discord behavior.

## What This Changes

This PR adds same-turn current-channel `message.send` tracking, exact final-delivery duplicate suppression, explicit-send idempotency, and focused agent/outbound tests.

## Best Possible Solution

Keep the exact route/text duplicate-final predicate, but revise outbound idempotency to wait for or share the in-flight result, or suppress only after a successful send is remembered, then provide live proof or a maintainer override.

## Reproduction Assessment

Yes, source-reproducible: current main has separate visible `message.send` and final durable delivery paths with no same-turn tracker. I did not establish a live Discord reproduction.

## Solution Assessment

Not yet. Exact normalized text plus full route equality is the right suppression shape, but the idempotency helper should not mark an in-flight replay as suppressed before the original send has succeeded.
