## Summary

Keep open: the PR targets a source-reproducible duplicate visible delivery bug, but it still has a merge-blocking idempotency race and the supplied proof is mocked seam output rather than live Gateway/Discord behavior.

## What This Changes

The PR adds same-turn explicit message-send tracking, duplicate final suppression, explicit-send idempotency, and regression tests across agent delivery, message-tool, and outbound delivery paths.

## Best Possible Solution

Keep the exact normalized route/text duplicate-final predicate, but make outbound idempotency wait for or share the in-flight result, or suppress only after a successful send is remembered, then add live Gateway/Discord proof or an explicit maintainer override.

## Reproduction Assessment

Yes, source-reproducible: current main has separate visible `message.send` and final durable delivery paths without a same-turn tracker in agent command delivery. I did not run a live Discord reproduction under this read-only review.

## Solution Assessment

No. Exact normalized route/text suppression is the right direction, but the idempotency helper should not report an in-flight replay as suppressed before the original send has succeeded.

## Review Findings
