## Summary

Keep open: the updated branch appears to address the earlier near-duplicate suppression concern, but this external PR still has only mocked validation and needs redacted real gateway/Discord behavior proof before merge.

## What This Changes

The branch threads a same-turn explicit `message.send` tracker through gateway agent runs and suppresses automatic final delivery when normalized visible text matches a same-channel/target explicit send.

## Best Possible Solution

Land the narrow exact-duplicate suppression after real gateway/Discord proof is attached, while keeping detached longwork ownership policy in https://github.com/openclaw/openclaw/issues/80450 separate.

## Reproduction Assessment

Yes from source inspection, but not from a live run: current main routes explicit `message.send` through `runMessageAction` and final replies through `sendDurableMessageBatch` independently. A same-turn identical current-channel send and final reply can therefore traverse both paths.

## Solution Assessment

Yes, the updated approach is the right narrow boundary: track same-turn explicit sends and suppress only exact normalized text duplicates on the same route while preserving warnings, different targets, media, and threads. It still needs real behavior proof before merge.
