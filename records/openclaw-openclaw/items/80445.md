## Summary

Keep open: the PR targets a real duplicate visible-delivery path, but the proposed route matcher can still suppress a final reply across different account/thread surfaces, and the supplied proof is isolated mocked validation rather than real Gateway/Discord behavior proof.

## What This Changes

The PR adds agent-run `message.send` tracking, exact-text duplicate final-delivery suppression, direct outbound idempotency for replayed explicit sends, and focused delivery regression tests.

## Best Possible Solution

Land the runtime duplicate suppression and replay idempotency only after route matching treats account/thread as exact route fields, redacted real Gateway/Discord proof is supplied, and the detached longwork ownership question stays in https://github.com/openclaw/openclaw/issues/80450.

## Reproduction Assessment

Yes from source inspection for the original same-turn duplicate path: current main runs explicit `message.send` and final delivery independently. The replayed explicit-send shape is backed by the contributor's live observation but was not live-reproduced here.

## Solution Assessment

No, not as currently implemented. Exact visible-text suppression plus direct-send idempotency is the right narrow direction, but the route comparison needs to reject one-sided account/thread matches before this is safe to merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

- **[P2] Require route fields to match when only one side has them:** `src/agents/command/delivery.ts:294`
