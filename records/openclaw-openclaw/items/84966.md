The PR changes agent harness selection so CLI runtime aliases such as claude-cli and google-gemini-cli route in-process compaction through the PI harness, with regression tests for selector and compaction entrypoint behavior.

## Best Possible Solution

Keep this PR open for review, land the narrow selector fallback after real CLI-backed compaction proof, and leave the broader helper-routing cluster with https://github.com/openclaw/openclaw/pull/72499.

## Solution Assessment

Likely yes. Routing only known CLI runtime aliases to PI for in-process harness selection matches the documented boundary that CLI aliases are not embedded harness ids, while preserving normal runCliAgent dispatch upstream.
