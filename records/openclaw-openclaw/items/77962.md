## Summary

Keep open. Current main and v2026.5.4 already put `MessageContent` into the Discord IDENTIFY intent mask, and `content=limited` is treated as an informational application-flag state rather than disabled, but the current-release guild delivery failure still needs live confirmation before cleanup.

## What This Changes

The issue reports that Discord guild messages are not received because OpenClaw shows Message Content Intent as limited even when the portal intent is enabled.

## Best Possible Solution

Keep this open until a maintainer reproduces with a live Discord guild and narrows whether the failure is inbound gateway frames, guild access/mention config, or visible reply/tool-calling behavior.
