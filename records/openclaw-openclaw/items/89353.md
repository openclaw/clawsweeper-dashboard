## What This Changes

The PR changes `src/plugin-sdk/provider-tools.ts` and its test so unreadable nested tool-parameter schema properties are reported as provider diagnostics instead of aborting schema inspection.

## Best Possible Solution

Refresh and consolidate the provider schema hardening into one current-main branch that preserves the shared SDK helper behavior and focused regression coverage.

## Reproduction Assessment

Yes. Current main and `v2026.6.6` call `Object.entries` and read schema properties during unsupported-keyword inspection, so an enumerable getter-backed schema child can throw before `inspectToolSchemas` returns diagnostics; I verified this by source inspection rather than executing the probe.

## Solution Assessment

Unclear as the final landing path. The patch is an acceptable narrow mitigation for child getters and does not add SDK API surface, but overlapping broader draft PRs now cover adjacent hostile-schema traversal paths, so maintainers should refresh and consolidate before merge.
