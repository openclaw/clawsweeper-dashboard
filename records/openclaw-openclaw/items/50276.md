## Summary

Keep open: the branch targets a useful Slack threading gap, but the current patch is not mergeable because it redeclares an imported resolver, adds non-runnable tests, and bypasses the established Slack off-mode threading policy.

## What This Changes

The PR adds Slack-only outbound message-action handling for `[[reply_to_current]]`, maps it to `threadId`, and adds same-channel and cross-channel threading tests.

## Best Possible Solution

Repair the branch to reuse the existing outbound threading seam, make the tests runnable, and keep Slack explicit-thread behavior aligned with the documented `replyToMode` policy.

## Reproduction Assessment

Yes. Static inspection of the PR diff and current main is enough: the patch redeclares an imported ESM binding and the added tests reference helpers that are not defined in the file.

## Solution Assessment

No. The branch should not shadow the shared outbound threading helper or force Slack off-mode into threading; the narrower maintainable path is to extend the existing seam while preserving current Slack policy.
