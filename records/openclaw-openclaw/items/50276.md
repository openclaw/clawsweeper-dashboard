The PR adds Slack-only resolution for message-tool `[[reply_to_current]]` so same-channel sends can target the active Slack thread, plus same-channel and cross-channel tests.

## Best Possible Solution

Resolve explicit current-thread handling through the Slack/channel threading contract, align the off-mode policy and docs intentionally, then validate with focused tests plus redacted live Slack proof.
