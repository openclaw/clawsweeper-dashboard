## Summary

Keep open. Current main still lacks the explicit Slack `[[reply_to_current]]` outbound mapping, but this PR should not merge as-is because it redeclares an imported helper, adds tests with undefined helpers, and changes documented Slack `replyToMode=off` behavior without maintainer policy sign-off.

## What This Changes

This PR adds Slack-only outbound handling for `[[reply_to_current]]` by mapping the active thread target to `threadId` and adds same-channel/cross-channel threading tests.

## Best Possible Solution

Use the existing outbound threading seam or a Slack adapter helper, avoid redeclaring imported helpers, add runnable same-channel/cross-channel/explicit-precedence tests, and only change off-mode semantics with matching docs/config updates if maintainers choose that policy.

## Reproduction Assessment

Yes. Static inspection of the PR diff gives a high-confidence reproduction: the module redeclares an imported ESM binding, and the new tests call helper names that are not defined or imported. Tests were not run because this review was read-only.

## Solution Assessment

No. The intended capability is plausible, but this patch is not the best fix until it uses the existing threading seam and resolves the documented Slack off-mode policy conflict.
