## Summary

Keep open. The requested Slack behavior is still not implemented on current main, and this contributor PR remains the focused implementation thread for closed issue #50031, but the current patch is not mergeable as written because it redeclares an imported resolver, adds tests that call undefined helpers, and changes Slack off-mode threading semantics without aligned docs/plugin policy.

## What This Changes

This PR adds Slack-only outbound handling for `[[reply_to_current]]` by mapping it to the active Slack thread target and adds same-channel/cross-channel threading tests.

## Best Possible Solution

Rework the change through the existing `message-action-threading` seam or a Slack adapter helper, avoid redeclaring imported helpers, add runnable same-channel/cross-channel/explicit-precedence tests, and align Slack off-mode behavior with docs and channel metadata if maintainers decide explicit tags should override it.

## Reproduction Assessment

Yes. Applying the PR diff has a static ESM/TypeScript failure from redeclaring an imported binding, and the added tests reference helper names that are not defined or imported in the file.

## Solution Assessment

No. The intended Slack support is plausible, but this patch is not the best fix because it duplicates an existing threading helper, adds unrunnable tests, and silently changes documented Slack off-mode semantics.
