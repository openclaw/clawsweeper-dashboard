## What This Changes

The PR adds Slack-only outbound message-action handling that maps `[[reply_to_current]]` to the active Slack thread id and adds same-channel/cross-channel threading tests.

## Best Possible Solution

Repair the branch to reuse the existing outbound threading seam, add runnable coverage, and preserve Slack `replyToMode: "off"` behavior unless maintainers deliberately change that policy.

## Solution Assessment

No. The proposed approach is not the best fix because it redeclares an imported helper and forces Slack off-mode into threading; a maintainable fix should use the existing threading helper and current Slack policy.
