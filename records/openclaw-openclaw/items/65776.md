## Summary

Keep open: current main and v2026.6.10 still expose static remote MCP headers plus OAuth, but no generic request-scoped header resolver for session/runtime authorization or routing metadata. The related open issues and PRs cover narrower slices, so this remains a security-sensitive product/API decision rather than a cleanup close.

## What This Changes

Request support for dynamic per-request headers on remote MCP HTTP/streamable-HTTP requests, including runtime auth, tenant, session, and correlation metadata.

## Best Possible Solution

Define one maintainer-approved remote MCP auth/header contract that reuses OAuth, dynamic env substitution, or caller-context injection where they fit, and only adds a typed opt-in request-context resolver for the remaining cases with explicit scoping, redaction, docs, and tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Current-main absence is high-confidence from the MCP config type, schema, transport, docs, latest-release check, dependency contract, and related open work.

## Solution Assessment

Unclear until maintainers choose the API and security boundary. A request-time resolver is plausible, but the maintainable shape should be narrowed against OAuth, client_credentials, dynamic env substitution, and caller-context alternatives first.
