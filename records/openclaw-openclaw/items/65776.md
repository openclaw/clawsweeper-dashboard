## Summary

Keep this open. Current main still supports only static remote MCP headers, and the requested per-request authorization/routing header resolver would add a new public MCP runtime/config API with security-sensitive context, precedence, redaction, and OAuth coordination decisions.

## What This Changes

Support runtime-generated per-request headers for remote MCP server calls, merged with existing static headers.

## Best Possible Solution

Add a typed request-scoped MCP header resolver or plugin/runtime API for both SSE and streamable-http, with explicit context exposure, merge precedence, redaction, OAuth interaction, and focused tests/docs.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection against current main gives a high-confidence verification path: config, schema, transport construction, docs, tests, and SDK integration expose only static headers, not a per-request arbitrary header resolver.

## Solution Assessment

Unclear pending maintainer design. A hook or headersProvider is plausible, but the maintainable solution needs explicit API shape, allowed context, precedence, redaction, and OAuth coordination decisions before implementation.
