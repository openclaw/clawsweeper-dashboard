## Summary

Keep open: current main and v2026.6.10 still provide static remote MCP headers plus OAuth, but no generic request-scoped header resolver for runtime/session authorization or tenant routing. The linked issues and PRs cover narrower OAuth, env-substitution, and caller-context slices, so this remains a security-sensitive product/API decision rather than conservative cleanup.

## What This Changes

Request a generic request-time mechanism for remote MCP HTTP/SSE headers so auth, tenant/routing, and session metadata can be generated from runtime context per outbound MCP request.

## Best Possible Solution

Define one maintainer-approved remote MCP auth/header contract that reuses OAuth, SecretRef, dynamic env substitution, or caller-context injection where they fit, and only adds a typed opt-in request-context resolver for remaining cases with explicit scoping, redaction, docs, and tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Current-main absence is high-confidence from the MCP config type, schema, transport wiring, docs, latest-release check, dependency contract, and related open work.

## Solution Assessment

Unclear until maintainers choose the API and security boundary. A request-time resolver is plausible, but the maintainable shape should be narrowed against OAuth, SecretRef, dynamic env substitution, and caller-context alternatives first.
