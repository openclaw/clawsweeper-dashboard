## Summary

Keep open: current main and v2026.6.10 still expose static remote MCP headers plus OAuth, but not a generic request-scoped header resolver for runtime auth, tenant routing, or session metadata.

## What This Changes

The issue requests dynamic per-request headers for HTTP/streamable HTTP MCP servers so outbound MCP calls can include runtime-specific auth and routing metadata.

## Best Possible Solution

Define one maintainer-approved MCP remote auth/header contract, reuse OAuth or narrower caller-context/env mechanisms where they fit, and add a typed opt-in request-context resolver only for the remaining cases with explicit URL scoping, precedence, redaction, docs, and tests.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Current-main absence is high-confidence from the MCP type, schema, transport, docs, release tag, and related open work.

## Solution Assessment

Unclear until maintainers choose the API and security boundary. A request-time resolver is plausible, but the maintainable OpenClaw shape should be narrowed against OAuth, dynamic env substitution, and caller-context alternatives first.
