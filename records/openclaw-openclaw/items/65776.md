## Summary

Keep open. Current main and v2026.5.12 still expose remote MCP headers as static config/RequestInit values only; adding request-scoped auth/routing headers would require a new public MCP runtime/config API plus security design.

## What This Changes

Request support for dynamic per-request headers on remote MCP SSE/streamable-http requests so runtime/session auth and routing metadata can be merged with static headers.

## Best Possible Solution

Add a typed request-scoped MCP header resolver or runtime API for both SSE and streamable-http with explicit context limits, merge precedence, redaction, OAuth coordination, and focused tests/docs.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source and docs inspection give a high-confidence verification path: current config, schema, transport construction, and tests expose only static headers.

## Solution Assessment

Unclear pending maintainer design. A hook or headersProvider is plausible, but the maintainable solution needs explicit API shape, allowed context, precedence, redaction, and OAuth coordination before implementation.
