## Summary

Keep open. Current main still only supports static remote MCP header maps, and the adjacent OAuth PR does not cover arbitrary per-request tenant, user, refreshed-token, routing, or correlation headers.

## What This Changes

The issue requests a request-scoped mechanism to generate and merge dynamic headers for remote MCP SSE and streamable-http requests.

## Best Possible Solution

Add a typed request-scoped header resolver or public runtime/plugin API for remote MCP transports, covering SSE and streamable-http with explicit context, merge precedence, redaction, OAuth interaction, tests, and docs.

## Reproduction Assessment

Not applicable as a bug reproduction because this is a feature request. Source inspection gives a high-confidence verification path: config, schema, launch resolution, transport construction, and callTool all lack a request-scoped header hook.

## Solution Assessment

Unclear pending maintainer design. A headersProvider-style hook is plausible, but the maintainable solution needs explicit API, security, redaction, precedence, and OAuth coordination decisions first.
