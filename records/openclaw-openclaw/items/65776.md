## Summary

Keep open: current main and v2026.6.8 still provide static remote MCP headers, shipped OAuth, and narrow Codex env-header projection, but no generic request-scoped MCP header resolver for session/runtime auth or routing metadata.

## What This Changes

Request support for dynamic per-request headers on remote MCP HTTP servers, with runtime/session context merged with static configured headers.

## Best Possible Solution

Define one maintainer-approved MCP auth/header contract that reuses OAuth or narrower caller-context/env mechanisms where they fit, and only adds a typed opt-in request-context header resolver with explicit URL scoping, precedence, redaction, docs, and tests for the remaining cases.

## Reproduction Assessment

Not applicable as a bug reproduction; this is a feature/API request. Current-main absence is high-confidence from the MCP type, schema, transport, docs, latest-release checks, and related open work.

## Solution Assessment

Unclear until maintainers choose the API and security boundary. A request-time resolver is plausible, but the maintainable OpenClaw shape should be narrowed against OAuth, client_credentials, dynamic env substitution, and caller-context alternatives first.
