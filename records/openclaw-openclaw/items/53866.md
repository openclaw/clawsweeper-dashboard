## Summary

Keep open as a valid feature request, but this branch is not merge-ready: current main now routes provider setup through plugin-owned `runSetup`, while the PR hard-codes Firecrawl auth in the command wrapper, writes legacy fetch config, is merge-conflicting, and still lacks real behavior proof.

The PR adds a Firecrawl PKCE browser-auth helper under `src/commands`, wires it into web-search onboarding, syncs Firecrawl fetch credentials, and adds command tests.

## Best Possible Solution

Implement browser sign-in as Firecrawl plugin-owned `WebSearchProvider.runSetup`, persist SecretRef-aware plugin-scoped `webSearch` and `webFetch` credentials, update focused provider/setup tests and docs, and include redacted real behavior proof.

## Solution Assessment

No. The feature is useful, but the current branch is not the best solution because Firecrawl-specific browser auth belongs in the Firecrawl provider `runSetup` hook with plugin-scoped credential writes, not in the core command wrapper.
