## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Honor skill eligibility before loading subscribers:** `extensions/feishu/src/event.runtime.ts:145-146`
  - body: This path scans configured workspace/global/extra skill roots and loads subscriber sidecars directly, bypassing the existing skill eligibility pipeline. A skill disabled with `skills.entries.<key>.enabled = false` or excluded for a `targetAgentId` with `agents.list[].skills: []` can still run its handler or trigger when Feishu events arrive; build subscribers from canonical eligible skill entries for the target agent instead.
  - confidence: 0.9
- **[P2] Use event IDs for publish-event dedupe:** `extensions/feishu/src/event.model.ts:387-393`
  - body: The hijacker dedupes publish-path events on `normalizedEvent.sourceId`, but Bitable normalization chooses `record_id` or `field_id` before `event_id`. That drops the second legitimate update to the same record or field for the Feishu dedupe TTL, so use the unique event id for dedupe and keep record/field ids as subject tokens.
  - confidence: 0.88
