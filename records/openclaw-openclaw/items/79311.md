## Summary

Keep open: the feature direction is plausible, but the current patch is not mergeable because it bypasses canonical skill visibility for a new execution surface, can miss category/topic-only subscribers, misuses subject ids for dedupe, fails type/lint checks, and lacks sufficient real Feishu proof.

## What This Changes

The PR adds a Feishu declarative event subscriber runtime that loads skill sidecar specs, publishes normalized events to a topic bus, and dispatches handlers or agent triggers from monitor startup.

## Best Possible Solution

Land a Feishu-owned subscriber runtime only after it compiles, uses the canonical skill snapshot and allowlist contract, registers concrete SDK event types reliably, preserves per-delivery dedupe, documents the sidecar contract, and includes redacted live Feishu proof.

## Reproduction Assessment

Yes for the PR blockers by source inspection and CI: the raw skill-root scan, exact SDK dispatch contract, sourceId dedupe behavior, type errors, and failed real-proof gate are all visible. I did not run a live Feishu account in this read-only review.

## Solution Assessment

No. The runtime direction may be viable, but this patch is not the best maintainable solution until it preserves OpenClaw's skill visibility contract, exact Feishu SDK registration, unique event dedupe, and a compiling validated implementation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Respect skill visibility before executing subscribers:** `extensions/feishu/src/event.runtime.ts:145-147`
  - body: `startFeishuEventRuntime` scans configured skill roots and later imports handlers from those entries, bypassing the effective skill snapshot that enforces precedence and agent allowlists. A shadowed, disabled, or agent-disallowed skill could still receive Feishu events and execute host code, so derive subscribers from the canonical visible skill set or enforce the same filters first.
  - confidence: 0.9
- **[P1] Use an exported ingress option type:** `extensions/feishu/src/event.executor.ts:3`
  - body: `AgentCommandIngressOpts` is imported from `openclaw/plugin-sdk/agent-runtime`, but that subpath does not export the type, and `check-prod-types` fails on this file. Infer the parameter type from `agentCommandFromIngress` or export the type intentionally before this can build.
  - confidence: 0.96
- **[P1] Pass a full runtime to trigger execution:** `extensions/feishu/src/event.runtime.ts:156`
  - body: `createFeishuEventSubscriptionExecutionHandler` expects a `RuntimeEnv`, but `startFeishuEventRuntime` narrows its runtime to only `log` and `error` before passing it through. `check-prod-types` rejects this because required runtime methods such as `exit` may be missing.
  - confidence: 0.94
- **[P1] Narrow numeric delivery limits before comparing:** `extensions/feishu/src/event.skill-spec.ts:148-152`
  - body: `parseOptionalPositiveInteger` compares and returns `value` while it is still typed as `unknown`, so the PR fails strict typechecking on null/unknown numeric cases. Narrow to a number before applying the safe-integer and positive checks.
  - confidence: 0.93
- **[P2] Require exact event types for SDK registration:** `extensions/feishu/src/event.runtime.ts:204-209`
  - body: Subscriber specs can match only topics, categories, or subtypes, but monitor startup only registers explicit `eventTypes` with the Feishu SDK. Because the SDK invokes handlers by exact parsed event type, those otherwise-valid subscribers never reach the topic bus unless the spec requires concrete event types or the runtime derives a concrete registration list.
  - confidence: 0.88
- **[P2] Deduplicate on the event delivery id:** `extensions/feishu/src/event.hijacker.ts:75-79`
  - body: The hijacker dedupes publish-path events with `normalizedEvent.sourceId`, but the normalizer chooses subject identifiers such as bitable `record_id` before `event_id`. That treats later updates for the same record as duplicate deliveries, so keep a separate delivery id for dedupe and leave subject ids for routing/session hints.
  - confidence: 0.84
