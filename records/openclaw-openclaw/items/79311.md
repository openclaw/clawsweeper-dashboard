- **[P1] Honor skill eligibility before loading subscribers:** `extensions/feishu/src/event.runtime.ts:145-146`
  - body: `startFeishuEventRuntime` passes raw workspace/home/extra/plugin roots into the subscriber loader, and that loader accepts any `SKILL.md` sidecar it finds. A skill disabled with `skills.entries.<key>.enabled = false`, blocked by requirements, blocked by bundled allowlists, or excluded by an agent skill filter can still import a handler or trigger an agent run from Feishu events; build subscribers from canonical eligible skill entries for the target agent instead.
  - confidence: 0.9
- **[P2] Deduplicate publish events by event ID:** `extensions/feishu/src/event.model.ts:387-393`
  - body: Bitable normalization chooses `record_id` or `field_id` before `event_id`, and the hijacker dedupes publish events on `normalizedEvent.sourceId`. Two legitimate updates to the same record or field can drop the second event for the dedupe TTL; use the unique Feishu event ID for dedupe and keep record and field IDs as subject tokens.
