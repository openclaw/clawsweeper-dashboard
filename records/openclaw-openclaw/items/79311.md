## Summary

Keep open: the PR adds a useful Feishu subscriber feature, but the head does not build, the new skill-handler execution path bypasses established skill visibility controls, event registration and dedupe semantics are unsafe, and the external real-behavior proof is still too weak for merge.

## What This Changes

The PR adds a Feishu declarative event subscriber runtime, skill sidecar parsing, topic-bus dispatch, handler/trigger execution, monitor startup wiring, and related tests.

## Best Possible Solution

Keep the Feishu-owned subscriber runtime direction, but land it only after it compiles, uses the canonical visible-skill contract, registers concrete Feishu event types reliably, deduplicates by delivery id, documents the sidecar contract, and includes real Feishu proof.

## Reproduction Assessment

Yes for the PR blockers: CI logs reproduce the typecheck failures, and source inspection shows the raw skill-root scan, exact-event registration gap, and sourceId-based dedupe. I did not run a live Feishu account in this read-only review.

## Solution Assessment

No. A Feishu-owned declarative runtime is plausible, but this patch is not the narrowest maintainable solution until it aligns with existing skill visibility, SDK event registration, and delivery-id dedupe contracts.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Fix the non-exported ingress type import:** `extensions/feishu/src/event.executor.ts:3`
  - body: `AgentCommandIngressOpts` is not exported from `openclaw/plugin-sdk/agent-runtime`, and the PR head fails prod/test typecheck before it can build. Use an exported public type, export the type through the SDK surface, or derive the parameter type from `agentCommandFromIngress`.
  - confidence: 0.96
- **[P1] Narrow positive integers before returning them:** `extensions/feishu/src/event.skill-spec.ts:148-152`
  - body: `parseOptionalPositiveInteger` compares and returns `value` while it is still `unknown`, which produces the current `tsgo:prod` errors for possibly-null and `{}` values. Add an explicit number narrowing before `Number.isSafeInteger` and the `<= 0` comparison.
  - confidence: 0.94
- **[P1] Pass a full runtime env to trigger execution:** `extensions/feishu/src/event.runtime.ts:156`
  - body: `startFeishuEventRuntime` accepts only a `RuntimeLogger`, but `createFeishuEventSubscriptionExecutionHandler` expects `RuntimeEnv`; the PR head fails typecheck here, and trigger execution would not receive the runtime services its contract asks for. Either widen the runtime parameter or narrow the executor contract.
  - confidence: 0.91
- **[P1] Respect visible skills before executing subscribers:** `extensions/feishu/src/event.runtime.ts:145-147`
  - body: `startFeishuEventRuntime` scans raw skill roots and later imports handler files from those entries, bypassing the canonical skill snapshot that enforces precedence and agent allowlists. A shadowed, disabled, or agent-disallowed skill can still receive Feishu events and execute host code, so derive subscribers from the visible skill contract or enforce the same filters first.
  - confidence: 0.89
- **[P2] Register concrete SDK event types for subscribers:** `extensions/feishu/src/event.runtime.ts:204-209`
  - body: Subscriber specs can match only `topics`, `categories`, or `subtypes`, but this helper exposes only explicit `eventTypes` to monitor registration. The Feishu SDK dispatches by exact registered type, so those otherwise-valid subscribers never reach the topic bus; require concrete event types or add an exact registration list.
  - confidence: 0.88
- **[P2] Deduplicate publish events by delivery id:** `extensions/feishu/src/event.hijacker.ts:75-79`
  - body: The hijacker checks duplicates using `normalizedEvent.sourceId`, while the normalizer uses subject identifiers such as a bitable `record_id` before `event_id`. That drops later updates for the same record as duplicates; keep a separate unique delivery id for dedupe and leave subject ids for routing/session hints.
  - confidence: 0.83
