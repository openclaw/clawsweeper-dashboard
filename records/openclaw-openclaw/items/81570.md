## Summary

Keep open: the draft patch still has a blocking freshness bug in the outer metadata snapshot memo, and the PR body still lacks exact PR-head real behavior proof for the performance claim.

## What This Changes

The PR adds a single-slot process memo around `loadPluginMetadataSnapshot`, clone-on-return behavior, lifecycle cache clearing, and focused memo regression tests.

## Best Possible Solution

Keep the performance optimization only with a derived discovery/install stat fingerprint equivalent to the broader related PR, or restrict reuse to explicitly owned current snapshots, then add redacted PR-head CLI timing proof.

## Reproduction Assessment

Yes, at source level: with the same config/env and a derived registry path, a plugin install/removal/manifest edit under configured, workspace, bundled, or global roots can occur between two calls, and the second call can hit the memo before discovery runs. I did not mutate the checkout to run a filesystem repro.

## Solution Assessment

No; an outer process memo is plausible only if its key carries the same plugin inventory and manifest freshness facts that current main gets by rescanning. The safer fix is the stat-based fingerprint from the broader related PR or passing explicit current snapshots through hot callers.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Fingerprint derived discovery before memo hits:** `src/plugins/plugin-metadata-snapshot-memo.ts:296-298`
  - body: The memo key only fingerprints persisted registry files, but when persisted reads are disabled, missing, or stale, the snapshot is derived by scanning `plugins.load.paths`, workspace, bundled, and global roots. Those directory entries and manifests can change without changing this key, so a second same-input call returns the old snapshot instead of making installs or manifest edits visible on the next metadata read. Include a stat-based discovery/install fingerprint for the derived path or bypass the memo there.
  - confidence: 0.9
