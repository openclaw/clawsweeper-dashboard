## Summary

Keep this PR open: it is a plausible narrow perf hotfix with sufficient real-environment proof, but the latest head still has blocking code issues in `src/plugins/plugin-metadata-snapshot.ts` around the cache-hit clone path and installed-index mutation isolation.

## What This Changes

This PR adds process-local memoization for plugin metadata snapshots, narrows memo eligibility and fingerprints, ignores vanished recovered install records, and adds focused plugin metadata/registry tests.

## Best Possible Solution

Keep the narrow release hotfix shape, but make the memo cache-hit path type-clean and mutation-isolated while leaving broader freshness hardening to https://github.com/openclaw/openclaw/pull/81064 or a follow-up.

## Reproduction Assessment

Yes, at source/proof level: current main rebuilds plugin metadata snapshots on repeated calls, and the PR body supplies real beta6 terminal timing proof. The latest blocking review findings are source-reproducible from the PR head; I did not run tests in this read-only checkout.

## Solution Assessment

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Remove the stale clone options argument:** `src/plugins/plugin-metadata-snapshot.ts:352`
  - body: The PR head defines `clonePluginMetadataSnapshot` with one `snapshot` parameter, but the cache-hit path still passes a second options object. That introduces a TypeScript arity error in production code, so the repo's `tsgo` checks will fail once non-draft validation runs.
  - confidence: 0.93
- **[P2] Deep-clone cached installed-index records:** `src/plugins/plugin-metadata-snapshot.ts:120-122`
  - body: The memo clone only shallow-copies `installRecords` and `index.plugins` records. Nested fields such as install-record entries, `startup`, `compat`, and package metadata remain shared with the memoized snapshot, so a caller mutating a returned index can poison later cache hits; clone or freeze the nested index data and cover that path.
  - confidence: 0.86
