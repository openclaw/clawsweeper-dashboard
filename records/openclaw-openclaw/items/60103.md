## Summary

Keep open. Current main still lacks a Feishu outbound normalizer for agent-authored mention tags, so text-style `<at user_id="...">...</at>` can still be sent unchanged through card and streaming-card paths; related #73469 is useful context but narrower than this cross-format request.

## What This Changes

Request automatic Feishu outbound @mention syntax conversion so agent-authored text/card/post mention tags match the selected message format.

## Best Possible Solution

Add a Feishu-owned outbound mention-normalization helper used by text/post, card, outbound, and streaming-card send paths while keeping explicit `mentionTargets` behavior intact.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a Feishu reply or outbound send using card renderMode or streaming where the agent text contains `<at user_id="ou_xxx">Name</at>`; current main forwards that text into card content without converting it.

## Solution Assessment

Yes. A localized Feishu plugin helper plus focused tests is the narrowest maintainable fix; a core channel abstraction or streaming notification redesign is larger than this syntax-format bug requires.
