## Summary

Keep open. Current main still has Feishu format-specific mention helpers and explicit-target prepending, but no outbound normalizer for agent-authored `<at user_id="...">...</at>` or `<at id=...></at>` tags across text/post, card, outbound adapter, or streaming-card delivery. The related open #73469 is useful streaming-notification context, but it is narrower than this cross-format syntax-normalization request.

## What This Changes

The issue requests automatic normalization of Feishu agent-authored @mention tags into the correct text, card, and post syntaxes before outbound delivery.

## Best Possible Solution

Implement a Feishu-owned outbound mention-normalization helper used by text/post, structured-card, markdown-card, outbound-adapter, and streaming-card update/close paths. Keep explicit mentionTargets behavior intact, preserve card sanitization boundaries, and treat #73469 as a separate notification-semantics follow-up unless the same change can safely prove notification delivery.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a Feishu reply or outbound send with `renderMode: "card"` or streaming enabled where the agent final text contains `<at user_id="ou_xxx">Name</at>`; current main forwards that text into card/post content without converting it to the destination format.

## Solution Assessment

Yes. A small Feishu plugin helper with focused tests is the narrowest maintainable fix; a core channel abstraction or broad streaming-notification redesign would be larger than this syntax-format bug requires.
