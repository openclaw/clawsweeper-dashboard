## Summary

Keep open. Current main still lacks outbound Feishu mention-tag normalization: agent-authored text-style `<at user_id="...">...</at>` can still be sent through card, streaming-card, post/text, and proactive outbound paths without conversion to the format-specific Feishu syntax, so the previously identified source-level bug remains actionable.

## What This Changes

The issue requests automatic Feishu outbound mention syntax normalization across card, text, and post message formats so agent-written mention tags render as real mentions in the selected delivery format.

## Best Possible Solution

Add a Feishu-owned outbound normalization helper at the card, text/post, proactive outbound, and streaming-card send boundaries, with regression tests that preserve explicit `mentionTargets` and existing escaping behavior.

## Reproduction Assessment

Yes, at source level. A Feishu reply or proactive send with card render mode or streaming enabled and agent text containing `<at user_id="ou_xxx">Name</at>` still flows into outbound card/post delivery without conversion on current main.

## Solution Assessment

Yes. A localized Feishu plugin helper plus focused regression tests is the narrowest maintainable fix; a core channel abstraction, new config option, or product-policy decision is not needed for this format-specific bug.
