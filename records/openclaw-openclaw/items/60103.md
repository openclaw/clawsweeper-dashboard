## Summary

Keep open. Current main still has separate Feishu text/card mention formatters and outbound send paths that only prepend structured `mentionTargets`; agent-authored `<at user_id="...">...</at>` text is still forwarded unchanged through card, streaming-card, post/text, and proactive outbound paths.

## What This Changes

The issue asks Feishu outbound delivery to normalize manually written @mention tags between text, card, and post message formats.

## Best Possible Solution

Add a Feishu-owned outbound mention-normalization helper used at text/post, card, outbound, and streaming-card send boundaries, with focused tests that preserve explicit `mentionTargets` behavior and do not rewrite escaped or caller-supplied native-card content unexpectedly.

## Reproduction Assessment

Yes, at source level. A Feishu reply or proactive send with card render mode or streaming enabled and agent text containing `<at user_id="ou_xxx">Name</at>` still flows into card markdown or Card Kit updates without conversion on current main.

## Solution Assessment

Yes. A localized Feishu plugin helper plus targeted tests is the narrowest maintainable solution; a core channel abstraction or new config option is not needed for this format-specific bug.
