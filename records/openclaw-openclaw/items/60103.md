## Summary

Keep open. Current main still only prepends explicit Feishu `mentionTargets` and does not rewrite agent-authored `<at>` tags when routing through post/text, card, outbound, or streaming-card paths, so the cross-format mention mismatch remains actionable.

## What This Changes

Request Feishu outbound delivery to normalize manually written @mention tags to the syntax required by text/post/card message formats.

## Best Possible Solution

Add a Feishu-owned outbound mention-normalization helper and apply it consistently at send boundaries while preserving explicit `mentionTargets` behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a Feishu reply or proactive send with `renderMode: "card"` or streaming enabled where the agent text contains `<at user_id="ou_xxx">Name</at>`; current main forwards that text into card content unchanged.

## Solution Assessment

Yes. A localized Feishu plugin helper plus focused tests is the narrowest maintainable fix; a core channel abstraction is not needed for this format-specific syntax bug.
