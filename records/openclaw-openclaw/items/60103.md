## Summary

Keep open. Current main still forwards agent-authored Feishu text-style mention tags through card, streaming-card, and outbound render-mode paths without converting them; the existing helpers only prepend explicit mentionTargets.

## What This Changes

Normalize Feishu agent-authored @mention tags between text, post, and card outbound formats so mentions survive render-mode and streaming delivery.

## Best Possible Solution

Add a Feishu-owned outbound mention-normalization helper shared by post/text, non-streaming card, streaming-card update/close, and proactive outbound paths while preserving explicit mentionTargets behavior.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is a Feishu reply or outbound send using card renderMode or streaming where the agent text contains `<at user_id="ou_xxx">Name</at>`; current main forwards that text into card content without conversion.

## Solution Assessment

Yes. A localized Feishu plugin helper with focused send/reply/streaming/outbound tests is the narrowest maintainable solution; a core channel API change is larger than this format-specific bug requires.
