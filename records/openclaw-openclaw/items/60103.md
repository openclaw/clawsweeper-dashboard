## Summary

Keep open. Current main still lacks a Feishu outbound normalizer for agent-authored mention tags, so text-style `<at user_id="...">...</at>` can flow unchanged into card, outbound card, and streaming-card content.

## What This Changes

The issue asks Feishu to normalize agent-authored `<at>` mention syntax before sending card, text/post, and streaming replies.

## Best Possible Solution

Add a Feishu-owned, format-aware outbound mention-normalization helper and use it consistently in post/text, static card, outbound, and streaming-card send paths while preserving explicit `mentionTargets` behavior.

## Reproduction Assessment

Yes, at source level. A Feishu reply or outbound send using card render mode or streaming with agent text containing `<at user_id="ou_xxx">Name</at>` reaches card markdown or Card Kit update content without conversion on current main.

## Solution Assessment

Yes. A localized Feishu plugin helper plus focused tests is the narrowest maintainable fix; a core channel abstraction or broad streaming redesign is larger than this syntax-format bug requires.
