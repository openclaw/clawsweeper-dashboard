## Summary

Keep open: current main and the latest release still route agent-authored Feishu text-style `<at user_id="...">name</at>` tags through post, card, proactive outbound, and streaming-card paths without a format-specific outbound normalizer.

## What This Changes

Request Feishu outbound delivery to normalize agent-authored mention tags into the correct text, card, or post syntax before sending.

## Best Possible Solution

Add one Feishu-owned outbound normalization helper and apply it at the post, card, proactive outbound, and streaming-card text boundaries while preserving the current no-automatic-mention-cascade behavior.

## Reproduction Assessment

Yes, at source level. A Feishu reply or proactive send with card render mode or streaming enabled and agent text containing `<at user_id="ou_xxx">Name</at>` still flows into card/post delivery without format-specific conversion on current main.

## Solution Assessment

Yes. A localized Feishu plugin normalizer with focused regression tests is the narrow maintainable fix; the separate push-notification limitation should stay tracked in https://github.com/openclaw/openclaw/issues/73469.
