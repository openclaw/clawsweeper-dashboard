## Summary

Keep open. Current main still forwards agent-authored Feishu text-style mention tags through card, streaming-card, post/text, and proactive outbound paths without format-specific normalization; the related streaming notification report is adjacent, not a complete replacement.

## What This Changes

The issue requests Feishu outbound mention normalization so agent-authored text-style mention tags are converted to the message-format-specific syntax for card, text, and post sends.

## Best Possible Solution

Add a Feishu-owned outbound mention-normalization helper at the text/post, card, proactive outbound, and streaming-card send boundaries, with focused regression tests that preserve explicit `mentionTargets` and native-card escaping behavior.

## Reproduction Assessment

Yes, at source level. A Feishu reply or proactive send with card render mode or streaming enabled and agent text containing `<at user_id="ou_xxx">Name</at>` still flows into card/post delivery without conversion on current main.

## Solution Assessment

Yes. A localized Feishu plugin helper plus targeted tests is the narrowest maintainable fix; a core channel abstraction or new config option is not needed for this format-specific bug.
