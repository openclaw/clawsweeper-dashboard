## Summary

Keep open. The Cursor SDK plugin work is a real feature candidate, but the latest branch still has blocking OpenClaw-path auth/startup issues, insufficient real behavior proof for the actual plugin route, and dependency/native-build changes that need maintainer security/product review.

## What This Changes

Adds a bundled `extensions/cursor-sdk` plugin that registers a Cursor SDK provider and AgentHarness, plus unit/live tests and workspace dependency changes for `@cursor/sdk`, `undici`, and `sqlite3`.

## Best Possible Solution

Land this only after the plugin-shaped integration fixes auth and startup handling, adds OpenClaw-path proof, and gets maintainer acceptance for the dependency surface; otherwise redirect the plugin to ClawHub/npm distribution.

## Reproduction Assessment

Yes, source-reproducible for the blocking PR behavior: current main skips generic auth bootstrap for plugin harnesses and only has Codex-specific automatic harness auth forwarding, while the PR runner requires `params.resolvedApiKey`. I did not run a live Cursor/OpenClaw turn in this read-only review.

## Solution Assessment

No. The AgentHarness plugin seam is the right technical boundary if maintainers accept a bundled plugin, but this branch still needs plugin-safe credential resolution, guarded SDK startup, OpenClaw-path proof, and dependency security review.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Resolve Cursor credentials before checking resolvedApiKey:** `extensions/cursor-sdk/src/run-attempt.ts:62-66`
  - body: A normal `cursor-sdk` run advertised with `CURSOR_API_KEY` can reach this branch with `params.resolvedApiKey` unset. Current main skips PI auth bootstrap for plugin-owned harnesses, and automatic plugin-harness profile ordering only runs when the runtime auth plan provides a harness auth provider, which is currently Codex-only. Resolve the Cursor credential through the plugin/profile/env contract before returning this missing-key result.
  - confidence: 0.91
- **[P2] Guard SDK startup with timeout and classification:** `extensions/cursor-sdk/src/run-attempt.ts:110-115`
  - body: `Agent.create()` is awaited before the `try` block and is not raced against `params.timeoutMs`. If Cursor startup/auth/network setup hangs or rejects, the harness can block past OpenClaw's timeout or throw out of `runAttempt` instead of returning a classified harness result. Move startup into the guarded lifecycle and make timeout/abort handling cover it.
  - confidence: 0.88
