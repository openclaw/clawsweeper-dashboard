## Summary

Keep this PR open for maintainer handling: the Cursor SDK AgentHarness/plugin shape is plausible, but this head still has blocking cancellation/replay bugs and needs explicit product/security approval before OpenClaw bundles a trusted Cursor local/cloud runtime.

## What This Changes

The PR adds a bundled `cursor-sdk` plugin that registers a Cursor SDK provider and AgentHarness, adds plugin config/tests, and changes the lockfile plus a workspace `undici` override.

## Best Possible Solution

If maintainers approve official bundling, keep the AgentHarness plugin direction but fix full-lifecycle cancellation, replay/side-effect reporting, dependency policy, docs/changelog, and targeted regression tests before merge; otherwise publish this integration externally through ClawHub/npm.

## Reproduction Assessment

Yes. Static inspection of the PR head plus the `@cursor/sdk@1.0.11` type declarations reproduces the blocking review findings without a live Cursor API key.

## Solution Assessment

No as-is. The plugin seam is the narrow maintainable integration point, but the current implementation does not yet preserve OpenClaw cancellation/replay semantics or document the trusted Cursor execution boundary.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Guard startup and first send with abort racing:** `extensions/cursor-sdk/src/run-attempt.ts:115-126`
  - body: `Agent.create()` and `agent.send()` are awaited directly, but the Cursor SDK types do not accept an `AbortSignal` for either call. If `/stop` or `params.timeoutMs` fires before `send()` returns, this only aborts the local controller and the attempt can remain stuck past OpenClaw's deadline because no `Run` exists to cancel yet.
  - confidence: 0.9
- **[P2] Keep cancellation active through wait:** `extensions/cursor-sdk/src/run-attempt.ts:142-161`
  - body: The abort listener that calls `run.cancel()` is removed as soon as streaming finishes, before `run.wait()` is awaited. A timeout, gateway shutdown, or `/stop` during that final wait no longer cancels the Cursor run and can hang or return a late success after OpenClaw's deadline.
  - confidence: 0.9
- **[P2] Mark hidden Cursor execution replay-unsafe:** `extensions/cursor-sdk/src/run-attempt.ts:242-249`
  - body: The harness intentionally drops Cursor tool events, yet successful attempts always report `toolMetas: []` and `replayMetadata: { hadPotentialSideEffects: false, replaySafe: true }`. A Cursor run that edited files, ran shell, used MCP, or spawned subagents without visible text can therefore be submitted again by retry paths that trust replay metadata.
  - confidence: 0.86
- **[P3] Add the required changelog entry:** `extensions/cursor-sdk/index.ts:7-8`
  - body: This adds a user-facing selectable Cursor SDK runtime, but the branch does not update `CHANGELOG.md`. Repo policy requires a changelog entry for feature work before merge.
  - confidence: 0.88
