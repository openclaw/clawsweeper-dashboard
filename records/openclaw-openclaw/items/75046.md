## Summary

Keep this PR open, but do not merge it as-is. The branch is active, closes the same-author feature request #75044, and uses the right broad plugin direction, but the current diff still has blocking harness/provider integration issues plus security and dependency-policy questions for a bundled trusted Cursor runtime.

## What This Changes

The branch adds a bundled `extensions/cursor-sdk` plugin with a manifest, AgentHarness registration, Cursor SDK runner, error-classification tests, an isolated `@cursor/sdk` dependency, lockfile updates, and a workspace `undici` override.

## Best Possible Solution

If maintainers approve bundling, land this as a provider-plus-harness plugin with model/catalog/auth visibility, explicit Cursor runtime policy documentation, OpenClaw abort/timeout semantics, reviewed exact dependency pins, and focused tests for the provider-resolution and cancellation paths. Otherwise, keep the implementation external through ClawHub/npm until the runtime boundary is proven.

## Reproduction Assessment

Yes for the review findings. Static tracing shows `cursor-sdk/composer-2` reaches current main’s provider/model resolution before `runAttempt`, while the PR head has no provider runtime registration; the runner code also leaves `agent.send()` and `run.wait()` outside its timeout and never observes `params.abortSignal`.

## Solution Assessment

No, this is not the best way to solve the linked feature request as-is. The plugin boundary is directionally right, but the implementation needs provider/auth/catalog wiring, full cancellation handling, and maintainer-approved security/dependency policy before it can safely merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Register cursor-sdk as a real provider:** `extensions/cursor-sdk/index.ts:9`
  - body: The plugin claims the `cursor-sdk` provider but only registers an AgentHarness. Current main still requires a resolved model before invoking the harness, so `cursor-sdk/composer-2` can fail with `model_not_found` before this runner is reached; add provider/catalog/auth or dynamic-model registration for this provider.
  - confidence: 0.9
- **[P2] Keep the timeout active through send and wait:** `extensions/cursor-sdk/src/run-attempt.ts:99-106`
  - body: The timeout starts only after `agent.send()` returns and is cleared before `run.wait()`. Cursor startup/send or final wait can therefore exceed `params.timeoutMs` without cancellation or failover; the guard should cover create/send/stream/wait or enforce a shared deadline.
  - confidence: 0.88
- **[P2] Cancel Cursor runs on OpenClaw abort:** `extensions/cursor-sdk/src/run-attempt.ts:99`
  - body: `runCursorSdkAttempt` never observes `params.abortSignal`. `/stop`, gateway shutdown, or caller cancellation can leave Cursor work running until upstream completion; bridge OpenClaw aborts to `run.cancel()` and report `externalAbort` correctly.
  - confidence: 0.86
