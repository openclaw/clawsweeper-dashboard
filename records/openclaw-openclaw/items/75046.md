## Summary

Keep this PR open. It is an active contributor PR paired with the same-author feature issue, and current main does not already ship a Cursor SDK native harness; however, the latest diff still has blocking lifecycle/replay bugs and needs maintainer product/security approval before bundling a trusted Cursor SDK runtime.

## What This Changes

The PR adds a bundled `extensions/cursor-sdk` plugin that registers a `cursor-sdk` provider and AgentHarness, stages `@cursor/sdk`, adds plugin config/tests, and updates the lockfile plus a workspace-wide `undici` override.

## Best Possible Solution

If maintainers approve official bundling, keep the provider-plus-AgentHarness shape but make the full Cursor lifecycle abortable, keep cancellation active until final wait completes, represent hidden Cursor tool side effects accurately, document the Cursor-owned execution boundary, use an owner-approved deterministic dependency plan, and add the required changelog/tests. Without that approval, this integration should live as an external ClawHub/npm plugin using the existing plugin seams.

## Reproduction Assessment

Yes for the blocking review findings. Static inspection of the PR head plus `@cursor/sdk@1.0.11` types shows create/send/wait are not wired to AbortSignal, cancellation is removed before `run.wait()`, and Cursor `tool_call` stream events are ignored while the harness reports replay-safe metadata.

## Solution Assessment

No as-is. AgentHarness is the right technical seam for a native Cursor runtime if it is approved as trusted/bundled, but this patch does not yet preserve OpenClaw abort, timeout, replay, policy, and dependency expectations.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Keep startup and first send abortable:** `extensions/cursor-sdk/src/run-attempt.ts:115-126`
  - body: `Agent.create()` and `agent.send()` are awaited directly before a Cursor `Run` exists. The SDK types expose no AbortSignal option for either call, so `/stop` or `params.timeoutMs` can only flip the local controller while startup or first send remains stuck past OpenClaw's deadline.
  - confidence: 0.9
- **[P2] Keep cancellation active through wait:** `extensions/cursor-sdk/src/run-attempt.ts:142-161`
  - body: The abort listener that calls `run.cancel()` is removed as soon as streaming finishes, before `run.wait()` resolves. A timeout, shutdown, or `/stop` during that final wait will no longer cancel the Cursor run and can hang or return a late result after OpenClaw's deadline.
  - confidence: 0.9
- **[P2] Mark hidden Cursor tool activity replay-unsafe:** `extensions/cursor-sdk/src/run-attempt.ts:242-249`
  - body: Cursor stream events include `tool_call` messages, but the harness drops non-assistant events and always returns `toolMetas: []` with `replaySafe: true`. A Cursor run that edited files, ran shell, or spawned subagents without visible text can therefore be retried as if no side effects happened.
  - confidence: 0.84
- **[P3] Add the required changelog entry:** `extensions/cursor-sdk/index.ts:7-8`
  - body: This adds a user-facing Cursor SDK runtime, but the branch does not update `CHANGELOG.md`. Repo policy requires a changelog entry before merging feature work.
  - confidence: 0.87
