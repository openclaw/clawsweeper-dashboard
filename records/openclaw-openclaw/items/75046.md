## Summary

Keep this PR open. The plugin-shaped AgentHarness approach is the right technical seam for a Cursor SDK experiment, but the current PR still has blocking abort/timeout lifecycle bugs and introduces a trusted local/cloud runtime plus dependency override surface that needs maintainer product/security approval before bundling.

## What This Changes

The PR adds a bundled `extensions/cursor-sdk` plugin with provider auth metadata, an AgentHarness runner, Cursor SDK runtime dependency staging, unit tests, lockfile changes, and a workspace-wide `undici` override.

## Best Possible Solution

If maintainers approve official bundling, keep the provider-plus-AgentHarness architecture but fix cancellation through create/send/stream/wait, document the Cursor-owned execution and diagnostic boundary, replace the broad dependency override with a deterministic reviewed dependency plan, add focused lifecycle tests, and add the required changelog entry. If that approval is not granted, publish the Cursor SDK integration externally through ClawHub/npm using the existing plugin seams.

## Reproduction Assessment

Yes for the blocking review findings. Static inspection of the PR head plus `@cursor/sdk@1.0.11` types shows `agent.send()` has no AbortSignal option, `run.wait()` is a separate operation, and the PR removes cancellation before waiting.

## Solution Assessment

No as-is. The AgentHarness seam is the narrow maintainable integration point, but this patch is not the best complete solution until it preserves OpenClaw abort/timeout semantics and receives explicit approval for the bundled trusted-runtime and dependency surface.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Race first send against abort and timeout:** `extensions/cursor-sdk/src/run-attempt.ts:126`
  - body: `Agent.send()` has no AbortSignal option, and before it resolves there is no Cursor `Run` to cancel. The local timeout only aborts `runAbortController`, so a local startup or cloud reservation stall can still block past `params.timeoutMs` or `/stop`; race this phase and dispose promptly.
  - confidence: 0.9
- **[P2] Keep cancellation active through wait:** `extensions/cursor-sdk/src/run-attempt.ts:143-161`
  - body: The abort listener that calls `run.cancel()` is removed when streaming finishes, before `run.wait()` is awaited. If timeout, gateway shutdown, or `/stop` fires during that final wait, the Cursor run is no longer cancelled and the attempt can hang past OpenClaw's deadline.
  - confidence: 0.9
- **[P3] Add the required changelog entry:** `extensions/cursor-sdk/index.ts:7-8`
  - body: This user-facing feature adds a selectable Cursor SDK runtime, but the branch has no `CHANGELOG.md` entry. Repo policy requires a changelog entry before merging feature work.
  - confidence: 0.86
