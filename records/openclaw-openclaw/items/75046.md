## Summary

Keep open: this is an active feature PR paired with an open same-author feature request, but the branch is not merge-ready because the normal Cursor API key path is source-reproducibly broken, SDK startup is not fully guarded, OpenClaw-path proof is still missing, and the new dependency/native install surface needs maintainer acceptance.

## What This Changes

Adds a bundled `extensions/cursor-sdk` plugin registering a Cursor SDK provider and AgentHarness, live/unit tests, and workspace dependency policy changes for `@cursor/sdk`, `undici`, and `sqlite3`.

## Best Possible Solution

Land a plugin-scoped Cursor SDK harness only after fixing credential forwarding and startup handling, adding OpenClaw-path proof, and accepting the dependency surface; otherwise redirect the integration to ClawHub/npm distribution.

## Reproduction Assessment

Yes, source-reproducible for the blocking PR behavior: current main only auto-forwards plugin-harness auth for Codex while the branch requires `params.resolvedApiKey`, and the SDK startup path is outside the guarded lifecycle. I did not run a live OpenClaw/Cursor turn in this read-only review.

## Solution Assessment

No. The AgentHarness plugin seam is the right technical boundary if maintainers accept a bundled plugin, but this branch still needs credential forwarding, guarded SDK startup, OpenClaw-path proof, and dependency security/product acceptance.
