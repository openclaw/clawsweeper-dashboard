## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Pin the new runtime dependencies exactly:** `extensions/cursor-sdk/package.json:7`
  - body: OpenClaw's dependency guard rejects non-exact package specs and workspace overrides. This new package uses `@cursor/sdk: ^1.0.7`, and the same PR adds an `undici` range override, so `check-dependencies` can fail and future installs can resolve code that was not reviewed with this PR.
  - confidence: 0.93
- **[P2] Race the first send against the OpenClaw timeout:** `extensions/cursor-sdk/src/run-attempt.ts:126`
  - body: The timeout only aborts a local controller until `run.cancel()` is registered after `agent.send()` returns. Since the Cursor SDK defers runtime initialization until the first send and `send` has no abort option, a stuck local/cloud startup can hang past `params.timeoutMs` instead of returning a timed-out harness result.
  - confidence: 0.86
- **[P2] Mirror assistant replies into the harness transcript:** `extensions/cursor-sdk/src/run-attempt.ts:221-223`
  - body: Successful runs return a `messagesSnapshot` containing only the user prompt and leave `lastAssistant` undefined. That omits the Cursor reply from OpenClaw's transcript/context snapshot, so follow-up turns, context-engine ingestion, and harness switching lose the assistant response even though `assistantTexts` was returned for delivery.
  - confidence: 0.84
- **[P2] Choose an explicit sqlite3 build-script policy:** `pnpm-workspace.yaml:85`
  - body: This commits pnpm's approval prompt text under `allowBuilds` instead of a boolean decision. Because the new SDK dependency brings in native `sqlite3`, the PR needs an intentional `true` or `false` build-script policy rather than leaving the lifecycle-script approval unresolved.
  - confidence: 0.88
