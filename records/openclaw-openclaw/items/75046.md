## Summary

Keep open. The PR is still a live implementation candidate for a capability current main does not have, but the patch has blocking OpenClaw-path credential handling and timeout/error-lifecycle issues, plus contributor proof and dependency/security review gaps.

## What This Changes

Adds a bundled `extensions/cursor-sdk` plugin that registers a Cursor SDK provider and AgentHarness, config schema, unit/live tests, `@cursor/sdk`, lockfile changes, a scoped `undici` override, and `sqlite3` build allowance.

## Best Possible Solution

Land a plugin-scoped Cursor SDK harness only if maintainers accept the bundled dependency/trust boundary; otherwise redirect it to ClawHub/npm while preserving the existing Cursor ACP route.

## Reproduction Assessment

Yes, source-reproducible for the blocking PR behavior: current main skips generic auth bootstrap for plugin-owned harnesses and only forwards Codex harness auth, while the PR runner fails when `params.resolvedApiKey` is absent. I did not run a live OpenClaw Cursor turn.

## Solution Assessment

No. The AgentHarness plugin seam is the right technical boundary if maintainers accept bundling, but this implementation still needs plugin-safe credential resolution, guarded SDK startup/send timeout handling, OpenClaw-path proof, and supply-chain review.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Resolve Cursor credentials before checking resolvedApiKey:** `extensions/cursor-sdk/src/run-attempt.ts:62-66`
  - body: For plugin-owned harnesses, current main skips the generic auth bootstrap and only maps Codex to a harness auth provider. A normal `cursor-sdk` run with `CURSOR_API_KEY` or a profile can therefore reach this branch with `params.resolvedApiKey` unset and fail before the SDK call. Resolve the plugin provider credential from the forwarded profile/env/model auth contract before returning this error.
  - confidence: 0.91
- **[P2] Guard SDK startup and send with the timeout:** `extensions/cursor-sdk/src/run-attempt.ts:115-126`
  - body: `@cursor/sdk` exposes `Agent.create()` and `agent.send()` without an AbortSignal option, so the current timer only aborts a local signal that neither await observes. `Agent.create()` is also outside the `try/finally`, so startup failures bypass classified harness errors and cleanup. Race startup/send against the harness timeout inside the guarded lifecycle before streaming or waiting.
  - confidence: 0.87
