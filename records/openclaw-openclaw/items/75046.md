## Summary

Keep open. Current main does not implement the Cursor SDK runtime, this PR is paired with an open same-author feature issue, and the PR still has blocking lifecycle/replay defects plus a product/security dependency decision before it can merge.

## What This Changes

The PR adds a bundled `extensions/cursor-sdk` native provider and AgentHarness plugin with `@cursor/sdk` runtime dependencies, auth/config metadata, tests, lockfile changes, and a workspace-wide `undici` override.

## Best Possible Solution

If maintainers approve bundling, keep the provider-plus-AgentHarness shape but make the full Cursor lifecycle deadline/abort aware, keep cancellation wired through final wait, conservatively record hidden tool execution as replay-unsafe unless it is observed precisely, document the Cursor-owned execution boundary, use an owner-approved dependency plan, and add the required changelog/tests. If maintainers do not approve bundling, publish the same integration as an external ClawHub/npm plugin using the existing seam.

## Reproduction Assessment

Yes for the blocking PR defects. Static inspection of PR head `729967eeb4828454596dc6450acf7474b6998a3e` plus published `@cursor/sdk@1.0.11` types shows the abort/cancel/replay contract gaps without needing a live API key.

## Solution Assessment

No as-is. AgentHarness is the right maintainable seam for this kind of native runtime, but this patch is not the best complete solution until it preserves OpenClaw lifecycle/replay semantics and receives explicit maintainer product/security approval for the bundled Cursor SDK dependency surface.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Make startup and first send abortable:** `extensions/cursor-sdk/src/run-attempt.ts:115-126`
  - body: `Agent.create()` and `agent.send()` are awaited directly before there is any `Run` to cancel, and the SDK types expose no AbortSignal option for either call. If local startup or cloud reservation stalls, `/stop` or `params.timeoutMs` only aborts the local controller while the OpenClaw turn remains stuck past its deadline.
  - confidence: 0.9
- **[P2] Keep cancellation active through wait:** `extensions/cursor-sdk/src/run-attempt.ts:142-161`
  - body: The abort listener that calls `run.cancel()` is removed as soon as streaming finishes, before `run.wait()` is awaited. A timeout, gateway shutdown, or `/stop` during that final wait no longer cancels the Cursor run and can hang or return a late success after OpenClaw's deadline.
  - confidence: 0.9
- **[P2] Mark hidden Cursor tool runs replay-unsafe:** `extensions/cursor-sdk/src/run-attempt.ts:242-249`
  - body: The harness drops Cursor `tool_call` events and returns `toolMetas: []` with `replayMetadata: { hadPotentialSideEffects: false, replaySafe: true }`. Current retry gates rely on that metadata, so an empty or reasoning-only Cursor run that edited files, ran shell, or created a PR can be resubmitted and duplicate side effects.
  - confidence: 0.86
- **[P3] Add the required changelog entry:** `extensions/cursor-sdk/index.ts:7-8`
  - body: This user-facing feature adds a selectable Cursor SDK runtime, but the branch does not update `CHANGELOG.md`. Repo policy requires a changelog entry before merging feature work.
  - confidence: 0.87
