## Summary

Keep this PR open for maintainer review. The AgentHarness direction is a plausible seam, but the current branch still has blocking lifecycle/replay bugs, adds an official trusted Cursor SDK runtime with unresolved security and dependency review, and is paired with an open same-author feature issue.

## What This Changes

The PR adds a bundled `extensions/cursor-sdk` native plugin that registers a Cursor provider and AgentHarness, adds Cursor SDK config/tests, updates the lockfile, and adds a workspace-wide `undici` override.

## Best Possible Solution

If maintainers approve official bundling, keep the provider-plus-AgentHarness shape but preserve OpenClaw abort/timeout/replay semantics across the whole Cursor lifecycle, document the Cursor-owned execution boundary, use an owner-approved deterministic dependency plan, and add the required changelog/tests. Without that approval, the integration should start as an external ClawHub/npm plugin using the existing plugin seams.

## Reproduction Assessment

Yes for the blocking review findings. Static inspection of the PR head plus the published `@cursor/sdk@1.0.11` types shows create/send/wait are awaited without AbortSignal support, cancellation is removed before `run.wait()`, and hidden Cursor tool execution is marked replay-safe.

## Solution Assessment

No as-is. AgentHarness is the narrow maintainable seam for this kind of native runtime, but this patch is not ready until it preserves lifecycle and replay policy and receives explicit product/security approval for bundling a trusted Cursor SDK runtime.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep startup and first send abortable:** `extensions/cursor-sdk/src/run-attempt.ts:115-126`
  - body: `Agent.create()` and `agent.send()` are awaited before a cancellable `Run` exists, and the Cursor SDK types do not expose an AbortSignal for either call. If local startup or cloud reservation stalls, `/stop` or `params.timeoutMs` only aborts the local controller and the OpenClaw attempt can remain stuck past its deadline.
  - confidence: 0.9
- **[P2] Keep cancellation active through wait:** `extensions/cursor-sdk/src/run-attempt.ts:142-161`
  - body: The abort listener that calls `run.cancel()` is removed after streaming but before `run.wait()`. A timeout, gateway shutdown, or `/stop` during that final wait no longer cancels Cursor and can either hang or return a late success after OpenClaw's deadline.
  - confidence: 0.9
- **[P2] Mark hidden Cursor tool runs replay-unsafe:** `extensions/cursor-sdk/src/run-attempt.ts:242-249`
  - body: The stream includes Cursor `tool_call` events, but the harness drops them and always reports `hadPotentialSideEffects: false, replaySafe: true`. A Cursor run that edited files, ran shell, or spawned subagents without visible text can therefore be resubmitted by OpenClaw retry gates.
  - confidence: 0.84
- **[P3] Add the required changelog entry:** `extensions/cursor-sdk/index.ts:7-8`
  - body: This adds a user-facing selectable Cursor SDK runtime, but the branch does not update `CHANGELOG.md`. Repo policy requires a changelog entry before feature work merges.
  - confidence: 0.87

## Security Review

Status: needs_attention

Summary: The diff introduces a trusted bundled Cursor local/cloud agent runtime and new SDK/native dependency surface that needs maintainer security and supply-chain approval before merge.

Concerns:

  - body: The harness starts Cursor local/cloud execution while Cursor owns file edits, shell, MCP, and subagent behavior. The branch does not yet document or enforce how OpenClaw sandbox, approval, timeout, abort, diagnostic, and tool-observation policy maps onto Cursor SDK options/events.
  - confidence: 0.86
