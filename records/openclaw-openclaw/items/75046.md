## Summary

Keep open. Current main still lacks Cursor SDK support, and this same-author implementation is paired with an open linked feature issue, but the PR still has source-reproducible auth/error-handling blockers plus insufficient OpenClaw-path proof and unresolved supply-chain review.

## What This Changes

Adds a bundled `extensions/cursor-sdk` plugin that registers a `cursor-sdk` provider and AgentHarness backed by `@cursor/sdk`, with config metadata, tests, lockfile changes, an `undici` override, and `sqlite3` build permission.

## Best Possible Solution

Land only a plugin-scoped Cursor SDK harness that resolves credentials through the supported plugin auth contract, contains SDK startup failures inside the harness result path, has redacted OpenClaw-path proof, and has maintainer product/security approval for bundled distribution.

## Reproduction Assessment

Yes for the blocking review findings via source inspection: current main skips generic auth bootstrap for plugin-owned harnesses and the PR runner only reads `params.resolvedApiKey`. I did not run a live OpenClaw Cursor turn.

## Solution Assessment

No. The plugin boundary is the right technical shape, but this implementation still needs plugin-safe credential resolution, guarded SDK startup/error handling, OpenClaw-path proof, and maintainer security/product acceptance before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Resolve Cursor credentials before checking `resolvedApiKey`:** `extensions/cursor-sdk/src/run-attempt.ts:62-66`
  - body: Current main skips PI auth bootstrap for plugin-owned harnesses and only fills `params.resolvedApiKey` from `apiKeyInfo`. This runner immediately returns the missing-key error when that field is unset, so ordinary `CURSOR_API_KEY` or profile setup for `cursor-sdk` can fail before the SDK is called. Resolve the credential through the plugin-safe auth/profile contract first.
  - confidence: 0.92
- **[P2] Catch SDK startup failures inside the harness result:** `extensions/cursor-sdk/src/run-attempt.ts:115`
  - body: `Agent.create(agentOptions)` runs before the `try` block. If Cursor rejects during startup, auth validation, or network setup, the error escapes to harness selection and is rethrown, bypassing the PR's error classification and normal prompt failure result. Move SDK creation into the guarded lifecycle and only dispose after an agent exists.
  - confidence: 0.87
