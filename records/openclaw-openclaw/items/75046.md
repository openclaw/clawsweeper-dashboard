## Summary

Keep open. The PR is a real implementation candidate paired with open issue #75044, but the current diff still has blocking lifecycle/replay defects and needs maintainer product, security, and dependency approval before bundling Cursor SDK as an official trusted runtime.

## What This Changes

The PR adds a bundled `cursor-sdk` provider and AgentHarness plugin backed by `@cursor/sdk`, runtime config/tests, lockfile changes, and a workspace-wide `undici` override.

## Best Possible Solution

If maintainers approve official bundling, keep the provider-plus-AgentHarness shape but fix full-lifecycle abortability, replay/side-effect reporting, docs for Cursor-owned execution, deterministic dependency pinning, changelog coverage, and targeted tests; otherwise keep this integration as an external ClawHub/npm plugin using the existing seams.

## Reproduction Assessment

Yes for the blocking review findings. Static inspection of the PR head plus the published `@cursor/sdk@1.0.11` types shows the unabortable create/send/wait windows, listener removal before `run.wait()`, and replay-safe metadata despite hidden Cursor execution.

## Solution Assessment

No as-is. AgentHarness is the narrow maintainable seam for a native Cursor runtime, but this implementation is not ready until it preserves OpenClaw cancellation/replay semantics and maintainers explicitly approve the bundled trusted runtime and dependency footprint.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Race startup and first send against abort:** `extensions/cursor-sdk/src/run-attempt.ts:115-126`
  - body: `Agent.create()` and `agent.send()` are awaited directly even though the SDK types expose no `AbortSignal` option and there is no `Run` to cancel before `send()` resolves. If local startup or cloud reservation stalls, `/stop` or `params.timeoutMs` only aborts the local controller and the attempt can remain stuck past OpenClaw's deadline.
  - confidence: 0.9
- **[P2] Keep cancellation active through wait:** `extensions/cursor-sdk/src/run-attempt.ts:142-161`
  - body: The abort listener that calls `run.cancel()` is removed as soon as streaming finishes, before `run.wait()` is awaited. A timeout, gateway shutdown, or `/stop` during that final wait no longer cancels the Cursor run and can hang or return a late success after OpenClaw's deadline.
  - confidence: 0.9
- **[P2] Mark hidden Cursor execution as replay-unsafe:** `extensions/cursor-sdk/src/run-attempt.ts:242-249`
  - body: The harness intentionally drops Cursor tool events, yet `buildSuccessResult()` always reports `toolMetas: []` and replay metadata as safe. Current retry gates use that metadata to avoid resubmitting turns after side effects, so a Cursor run that edited files or ran shell without visible text can be submitted again.
  - confidence: 0.84
- **[P3] Add the required changelog entry:** `extensions/cursor-sdk/index.ts:7-8`
  - body: This is a user-facing feature that adds a selectable Cursor SDK runtime, but the branch does not update `CHANGELOG.md`. Repo policy requires a changelog entry before merging feature work.
  - confidence: 0.88
