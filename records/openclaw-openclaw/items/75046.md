## Summary

Keep open. The PR is a real implementation candidate for the linked Cursor SDK feature, but current main does not already implement it and the patch still has merge-blocking lifecycle, replay-safety, dependency-shape, changelog, and security-review issues.

## What This Changes

The PR adds a bundled `cursor-sdk` plugin with provider registration, AgentHarness runtime code, Cursor SDK configuration, unit tests, and root dependency/lockfile updates.

## Best Possible Solution

Decide first whether OpenClaw should officially bundle this trusted Cursor SDK runtime. If yes, keep the provider plus AgentHarness shape but fix abort/timeout semantics, cancellation through `wait()`, replay metadata for hidden Cursor tools, runtime-dependency packaging, security documentation, and changelog coverage before merge; if not, keep this as an external ClawHub/npm plugin using the existing harness/provider seams.

## Reproduction Assessment

Yes for the review findings. Static inspection of PR head plus current main shows the uncancellable create/send/wait windows, the replay metadata mismatch, and the existing bundled-plugin guard that would reject `stageRuntimeDependencies: true`.

## Solution Assessment

No as-is. AgentHarness is the right narrow seam for a native Cursor runtime, but this implementation is not yet the safest maintainable solution because it does not preserve OpenClaw lifecycle/replay policy and still needs explicit bundled-runtime security and dependency approval.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Remove the staged runtime-deps marker:** `extensions/cursor-sdk/package.json:16-18`
  - body: This package sets `openclaw.bundle.stageRuntimeDependencies: true`, but current main has a bundled-plugin shape guard that collects any package with that legacy marker as an offender and expects the list to be empty. The new plugin will therefore fail the existing guard unless the dependency plan is moved to the current runtime-deps path or the guard is intentionally updated with maintainer approval.
  - confidence: 0.93
- **[P2] Make startup and first send abortable:** `extensions/cursor-sdk/src/run-attempt.ts:115-126`
  - body: `Agent.create()` and `agent.send()` are awaited before there is a `Run` to cancel, and the Cursor SDK types expose no AbortSignal option for either call. If local startup or cloud reservation stalls, OpenClaw's timeout or `/stop` only flips the local controller and the attempt can remain stuck past the deadline.
  - confidence: 0.9
- **[P2] Keep cancellation wired through wait:** `extensions/cursor-sdk/src/run-attempt.ts:142-161`
  - body: The abort listener that calls `run.cancel()` is removed as soon as streaming exits, before `run.wait()` runs. A timeout, shutdown, or user stop during that final wait no longer cancels the Cursor run and can hang or return a late result after OpenClaw considers the turn expired.
  - confidence: 0.91
- **[P2] Mark hidden Cursor tool runs as replay-unsafe:** `extensions/cursor-sdk/src/run-attempt.ts:249`
  - body: The harness intentionally drops Cursor tool-call events, but still returns empty `toolMetas` and `replayMetadata` saying there were no side effects. Current retry gates use that metadata to resubmit empty/error turns only when no side effects occurred, so a Cursor run that edited files or ran shell without final text can be duplicated.
  - confidence: 0.86
- **[P3] Add the required changelog entry:** `extensions/cursor-sdk/index.ts:5-8`
  - body: This is a user-facing feature that adds a selectable Cursor SDK runtime, but the branch does not add a `CHANGELOG.md` entry. Repo policy requires changelog coverage for feature work before merge, with allowed contributor credit rather than bot or maintainer handles when applicable.
  - confidence: 0.86
