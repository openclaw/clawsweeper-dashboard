## Summary

Keep this PR open for maintainer review. The AgentHarness/plugin direction is the right seam if OpenClaw wants an official Cursor SDK runtime, but the current branch still has blocking cancellation/replay defects and introduces a trusted local/cloud execution dependency that needs explicit product and security approval.

## What This Changes

Adds a bundled `extensions/cursor-sdk` plugin that registers a Cursor SDK provider and AgentHarness, adds focused harness/error tests, and updates the lockfile plus a workspace `undici` override.

## Best Possible Solution

If maintainers approve official bundling, keep the plugin-based AgentHarness shape but fix lifecycle cancellation, replay metadata, dependency policy, docs/changelog, and labeler coverage before merge; otherwise publish the integration externally through ClawHub/npm.

## Reproduction Assessment

Yes. Static inspection of the PR head plus the `@cursor/sdk@1.0.12` type declarations reproduces the cancellation and replay issues without a live Cursor API key.

## Solution Assessment

No as-is. The plugin seam is the narrow maintainable integration point, but the current implementation does not yet preserve OpenClaw abort/replay semantics or document the trusted Cursor execution boundary.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Guard Cursor startup and send against aborts:** `extensions/cursor-sdk/src/run-attempt.ts:115-126`
  - body: `Agent.create()` and `agent.send()` are awaited without an abortable race, and the SDK types do not accept an `AbortSignal` there. If `/stop` or `params.timeoutMs` fires before a `Run` exists, OpenClaw can remain stuck past its deadline because there is nothing to cancel yet.
  - confidence: 0.9
- **[P2] Keep cancellation active through wait:** `extensions/cursor-sdk/src/run-attempt.ts:142-161`
  - body: The abort listener that calls `run.cancel()` is removed as soon as streaming finishes, before `run.wait()` is awaited. A timeout, shutdown, or `/stop` during that final wait no longer cancels the Cursor run and can hang or return a late success after OpenClaw's deadline.
  - confidence: 0.9
- **[P2] Mark hidden Cursor execution replay-unsafe:** `extensions/cursor-sdk/src/run-attempt.ts:242-249`
  - body: The harness intentionally drops Cursor `tool_call` events, yet successful attempts always return empty `toolMetas` and `replaySafe: true`. A run that edited files, ran shell, used MCP, or spawned subagents can therefore be retried by paths that trust replay metadata.
  - confidence: 0.87
- **[P3] Add the required changelog entry:** `extensions/cursor-sdk/index.ts:5-8`
  - body: This adds a user-facing selectable Cursor SDK runtime, but the branch does not update `CHANGELOG.md`. Repo policy requires a changelog entry for feature work before merge.
  - confidence: 0.88
- **[P3] Add labeler coverage for the new plugin:** `extensions/cursor-sdk/openclaw.plugin.json:2-5`
  - body: New plugin surfaces need `.github/labeler.yml` coverage. Without an `extensions: cursor-sdk` entry, future Cursor SDK changes will not route cleanly; this PR already picked up an unrelated memory-core label from formatting churn.
  - confidence: 0.8
