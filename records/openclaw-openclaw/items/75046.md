## Summary

Keep open. Current main does not implement a Cursor SDK harness, and the PR uses the intended AgentHarness/plugin seam, but the patch still has lifecycle, replay-safety, changelog, and maintainer product/security approval blockers before it should merge.

## What This Changes

Adds a bundled `extensions/cursor-sdk` plugin that registers a `cursor-sdk` provider and AgentHarness backed by `@cursor/sdk`, with plugin config, auth metadata, tests, lockfile updates, and a workspace-wide `undici` override.

## Best Possible Solution

If maintainers approve official bundling, keep the provider-plus-AgentHarness shape but make the full Cursor lifecycle abortable, keep cancellation live through final wait, conservatively report hidden tool side effects, document the Cursor-owned execution boundary, use an owner-approved deterministic dependency plan, and add the required changelog and focused lifecycle tests. Without that approval, the integration should live as an external ClawHub/npm plugin using the existing plugin seams.

## Reproduction Assessment

Yes. Static inspection plus the published `@cursor/sdk@1.0.11` types gives a high-confidence repro path: fake or wrap SDK `Agent.create()`, `agent.send()`, or `run.wait()` so they hang, then observe that the PR's timeout/abort path cannot cancel before a `Run` exists or after the stream listener is removed.

## Solution Assessment

No as-is. AgentHarness is the narrow maintainable seam for a native Cursor runtime, but this implementation is not the best solution until it preserves OpenClaw abort/timeout/replay semantics and receives explicit maintainer approval for the trusted runtime and dependency footprint.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Make startup and first send abortable:** `extensions/cursor-sdk/src/run-attempt.ts:115-126`
  - body: `Agent.create()` and `agent.send()` are awaited directly even though the SDK types expose no AbortSignal option and there is no `Run` to cancel before `send()` resolves. If local startup or cloud reservation stalls, `/stop` or `params.timeoutMs` only aborts the local controller and the attempt can remain stuck past OpenClaw's deadline.
  - confidence: 0.9
- **[P2] Keep cancellation active through wait:** `extensions/cursor-sdk/src/run-attempt.ts:143-161`
  - body: The abort listener that calls `run.cancel()` is removed as soon as streaming finishes, before `run.wait()` is awaited. A timeout, gateway shutdown, or `/stop` during that final wait no longer cancels the Cursor run and can either hang or return a late success after OpenClaw's deadline.
  - confidence: 0.9
- **[P2] Mark hidden Cursor tool activity replay-unsafe:** `extensions/cursor-sdk/src/run-attempt.ts:242-249`
  - body: The harness intentionally drops Cursor tool events, yet `buildSuccessResult()` always reports no tools and `replaySafe: true`. Current retry gates use that metadata to resubmit empty/planning/reasoning turns only when no side effects occurred, so a Cursor run that edited files or ran shell without visible text can be submitted again.
  - confidence: 0.82
- **[P3] Add the required changelog entry:** `extensions/cursor-sdk/index.ts:7-8`
  - body: This is a user-facing feature that adds a selectable Cursor SDK runtime, but the branch does not update `CHANGELOG.md`. Repo policy requires a changelog entry before merging feature work.
  - confidence: 0.87
