## Summary

Keep this PR open. Current main does not implement a native Cursor SDK AgentHarness, the PR is paired with the still-open feature issue #75044, and the latest head still has blocking lifecycle/replay defects plus a security/dependency approval question before this trusted runtime should be bundled.

## What This Changes

The PR adds a bundled `extensions/cursor-sdk` native plugin that registers a `cursor-sdk` provider and AgentHarness, stages `@cursor/sdk`, adds plugin config/tests, and changes the workspace lockfile plus a workspace-wide `undici` override.

## Best Possible Solution

If maintainers explicitly approve official bundling, keep the provider-plus-AgentHarness shape but fix full-lifecycle abort/timeout cleanup, preserve cancellation through `run.wait()`, report opaque Cursor execution as replay-unsafe or surface sufficient tool telemetry, use an owner-approved deterministic dependency plan, document the Cursor-owned execution boundary, add the required changelog, and validate the plugin tests plus changed gate. Without that approval, this should remain an external ClawHub/npm plugin built on the existing AgentHarness/provider seams.

## Reproduction Assessment

Yes for the blocking review findings. Static inspection of the latest PR head shows the non-abortable `Agent.create()` and cleanup scope, removal of cancellation before `run.wait()`, and replay-safe metadata despite intentionally hidden Cursor tool execution.

## Solution Assessment

No as-is. AgentHarness is the narrow maintainable seam for a native Cursor runtime, but this patch does not yet preserve OpenClaw's abort, timeout, replay, and security-boundary expectations, and it needs maintainer approval before bundling the dependency/runtime.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Move agent creation under abort cleanup:** `extensions/cursor-sdk/src/run-attempt.ts:115`
  - body: `Agent.create()` is awaited after the timeout/upstream abort handlers are installed but before the `try/finally`. If Cursor startup hangs, the timeout only aborts the local controller with no run to cancel; if startup rejects, the timeout and upstream listener are not cleaned up and the error bypasses the SDK classification path.
  - confidence: 0.88
- **[P2] Keep cancellation active through wait:** `extensions/cursor-sdk/src/run-attempt.ts:143`
  - body: The abort listener that calls `run.cancel()` is removed as soon as streaming ends, before `run.wait()` is awaited. A timeout, gateway shutdown, or `/stop` during that final wait will no longer cancel the Cursor run and can hang or return a late result.
  - confidence: 0.9
- **[P2] Mark opaque Cursor execution replay-unsafe:** `extensions/cursor-sdk/src/run-attempt.ts:249`
  - body: The harness intentionally drops Cursor tool events, but `buildSuccessResult()` reports no side effects and `replaySafe: true`. A run that edited files or ran shell without visible assistant text can then be retried as if nothing happened.
  - confidence: 0.84
- **[P3] Add the required changelog entry:** `extensions/cursor-sdk/index.ts:7-8`
  - body: This adds a user-facing selectable Cursor SDK runtime, but the PR does not update `CHANGELOG.md`. Repo policy requires a changelog entry before merging user-facing feature work.
  - confidence: 0.86
