## Summary

Keep open: this is a useful feature proposal, but the current patch is not merge-ready because it adds a trusted bundled Cursor runtime with cancellation/replay blockers, missing real behavior proof, and a product/security approval decision still pending.

## What This Changes

Adds a bundled `extensions/cursor-sdk` plugin that registers a Cursor SDK provider and AgentHarness, package-local `@cursor/sdk` dependency, unit tests, and lockfile/root override changes.

## Best Possible Solution

Either publish this integration externally through ClawHub/npm, or, if maintainers want it bundled, land it only after the runtime boundary is documented, cancellation/replay semantics are fixed, dependencies are explicitly approved, and real Cursor SDK proof is attached.

## Reproduction Assessment

Yes. Static inspection of the PR head plus `@cursor/sdk@1.0.12` declarations gives a high-confidence source reproduction for the cancellation and replay blockers without needing a live Cursor API key.

## Solution Assessment

No as-is. The AgentHarness seam is the right narrow integration point if this is approved for bundling, but this patch does not yet preserve OpenClaw cancellation/replay semantics or document the trusted execution boundary.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Guard startup and first send with abort:** `extensions/cursor-sdk/src/run-attempt.ts:115-126`
  - body: `Agent.create()` and `agent.send()` are awaited directly even though the Cursor SDK types do not accept an AbortSignal. If `/stop` or the attempt timeout fires before `send()` resolves, the harness cannot return at OpenClaw's deadline and no `Run` exists yet to cancel.
  - confidence: 0.9
- **[P2] Keep cancellation active through wait:** `extensions/cursor-sdk/src/run-attempt.ts:142-161`
  - body: The listener that calls `run.cancel()` is removed as soon as streaming finishes, before `run.wait()` starts. A timeout, shutdown, or `/stop` during that final wait no longer cancels the Cursor run and can hang or return a late success.
  - confidence: 0.91
- **[P2] Mark hidden Cursor side effects as replay-unsafe:** `extensions/cursor-sdk/src/run-attempt.ts:242-249`
  - body: The harness intentionally drops Cursor tool events, but successful attempts always return empty tool metadata with `replaySafe: true`. Cursor runs can edit files, run shell commands, use MCP, or spawn subagents invisibly, so retry paths may duplicate hidden side effects.
  - confidence: 0.88
- **[P3] Add the required changelog entry:** `extensions/cursor-sdk/index.ts:5-11`
  - body: This adds a user-facing selectable Cursor SDK runtime, but the branch does not update `CHANGELOG.md`. Repo policy requires a changelog entry for feature work before merge.
  - confidence: 0.87

## Security Review

Status: needs_attention

Summary: The diff introduces a trusted Cursor execution runtime plus new dependency and override surface that needs maintainer security review before bundling.

Concerns:

- **[medium] Map OpenClaw policy before local Cursor runs:** `extensions/cursor-sdk/src/run-attempt.ts:82`
  - body: The harness starts Cursor local/cloud execution while Cursor owns file edits, shell, MCP, and subagent behavior. The branch does not yet document or enforce how OpenClaw sandbox, approval, timeout, abort, diagnostics, and tool-observation policy map onto that execution boundary.
  - confidence: 0.86
- **[medium] Review the Cursor SDK dependency footprint:** `extensions/cursor-sdk/package.json:7`
  - body: Bundling `@cursor/sdk` stages a new SDK with platform optional packages and runtime dependencies such as sqlite3, connectrpc, statsig, and Cursor native packages under an official plugin package. That expands the trusted install and execution surface for users who enable the plugin.
  - confidence: 0.84
- **[low] Use a deterministic undici override:** `package.json:1803`
  - body: The PR adds a workspace-wide `undici: >=6.24.0` override. A broad range can change unrelated dependency resolution over time; use an exact reviewed pin or an owner-approved dependency plan.
  - confidence: 0.8
