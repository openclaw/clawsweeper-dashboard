## Summary

Keep this PR open for normal review. Current main does not contain a Cursor SDK harness, the PR is paired with the still-open feature request, and the branch has blocking lifecycle/replay issues plus a trusted-runtime dependency/security boundary that needs maintainer approval before merge.

## What This Changes

The PR adds a bundled `extensions/cursor-sdk` plugin that registers Cursor SDK provider auth/model passthrough plus an AgentHarness, with plugin tests and workspace dependency/lockfile changes.

## Best Possible Solution

If maintainers approve a bundled Cursor SDK runtime, land it as a trusted AgentHarness plugin only after lifecycle cancellation, replay metadata, dependency policy, docs/changelog, and labeler coverage are fixed; otherwise keep Cursor SDK as an external ClawHub/npm plugin while the current ACPX Cursor CLI route remains documented.

## Reproduction Assessment

Yes. Static inspection of PR head `605e1aeda1e68a0119bc7c15db877babd1c7a287` plus the published `@cursor/sdk@1.0.12` type declarations reproduces the cancellation and replay blockers without a live Cursor key.

## Solution Assessment

No as-is. The AgentHarness plugin seam is the right narrow integration point if maintainers approve bundling, but the current patch does not preserve OpenClaw lifecycle, replay, dependency, and release-note expectations.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Guard Cursor startup before a Run exists:** `extensions/cursor-sdk/src/run-attempt.ts:115-126`
  - body: The timeout/upstream abort controller is installed before `Agent.create()`, but the SDK exposes no `AbortSignal` for `Agent.create()` or `agent.send()`. If timeout or `/stop` fires while either await is pending, OpenClaw cannot cancel a Cursor run yet; `Agent.create()` errors also bypass the later cleanup path.
  - confidence: 0.91
- **[P2] Keep abort cancellation active through wait:** `extensions/cursor-sdk/src/run-attempt.ts:142-161`
  - body: The listener that calls `run.cancel()` is removed as soon as streaming ends, before `run.wait()` runs. A timeout, shutdown, or `/stop` during that final wait will no longer cancel the Cursor run and can return late or hang past OpenClaw's deadline.
  - confidence: 0.9
- **[P2] Mark hidden Cursor work as replay-unsafe:** `extensions/cursor-sdk/src/run-attempt.ts:242-249`
  - body: The harness drops Cursor tool events but always returns `toolMetas: []` and `replayMetadata: { hadPotentialSideEffects: false, replaySafe: true }`. Cursor shell/file/MCP/subagent work can therefore be retried by OpenClaw as if no side effect happened.
  - confidence: 0.88
- **[P3] Add the required changelog entry:** `extensions/cursor-sdk/index.ts:5-8`
  - body: This adds a user-facing selectable Cursor SDK runtime, but `CHANGELOG.md` is unchanged. Repo policy requires user-facing feature work to have an Unreleased changelog entry before merge.
  - confidence: 0.88
- **[P3] Add a labeler rule for the new plugin:** `extensions/cursor-sdk/openclaw.plugin.json:2-5`
  - body: New plugin surfaces need labeler coverage, but this branch does not add an `extensions/cursor-sdk/**` rule. Future Cursor SDK-only PRs would miss the plugin label and owner-routing signal.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: The patch introduces a new trusted local/cloud Cursor execution and dependency surface that needs maintainer security review before bundling.

Concerns:

- **[medium] Map OpenClaw policy before Cursor execution:** `extensions/cursor-sdk/src/run-attempt.ts:82`
  - body: The harness starts Cursor local/cloud agents while Cursor owns file edits, shell, MCP, subagents, and optional cloud PR creation. The branch does not yet document or enforce how OpenClaw sandbox, approvals, timeout, abort, diagnostics, tool observation, and replay policy map onto that execution boundary.
  - confidence: 0.87
- **[medium] Review the Cursor SDK dependency footprint:** `extensions/cursor-sdk/package.json:7`
  - body: Bundling `@cursor/sdk` stages a new SDK with platform optional packages and runtime dependencies such as sqlite3, connectrpc, and statsig under an official plugin package. That expands the trusted install and execution surface for users who enable the plugin.
  - confidence: 0.84
- **[low] Use a deterministic undici override:** `package.json:1772`
  - body: The workspace-wide `undici: >=6.24.0` override can change future unrelated dependency resolution. Use an exact reviewed pin or an owner-approved dependency plan instead of a broad range.
  - confidence: 0.8
