## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Mark opaque Cursor runs replay-unsafe:** `extensions/cursor-sdk/src/run-attempt.ts:249`
  - body: The harness always returns `toolMetas: []` and `replayMetadata: { hadPotentialSideEffects: false, replaySafe: true }` even though Cursor can run file, shell, MCP, and subagent tools internally and this wrapper does not capture those steps. The outer runner uses this metadata to decide retry guards and user warnings, so an empty or error Cursor turn after side effects can be retried or surfaced as safe. Capture SDK tool telemetry via callbacks such as `onStep`/`onDelta`, or mark the attempt replay-unsafe while effects are opaque.
  - confidence: 0.88
- **[P2] Enforce timeout before awaiting create/send:** `extensions/cursor-sdk/src/run-attempt.ts:115-126`
  - body: The timeout only aborts `runAbortController`, but `Agent.create()` and `agent.send()` are awaited before the code can observe that abort or install `run.cancel()`. If either SDK call hangs, `runCursorSdkAttempt` never returns a timed-out result despite `params.timeoutMs`. Wrap these awaits in a timeout race or otherwise make the create/send phases bounded.
  - confidence: 0.84
