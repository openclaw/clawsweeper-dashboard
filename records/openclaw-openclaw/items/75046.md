## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Stop marking Cursor runs replay-safe:** `extensions/cursor-sdk/src/run-attempt.ts:249`
  - body: Both result builders always set `hadPotentialSideEffects: false` and `replaySafe: true`, but Cursor SDK streams `tool_call` events and this wrapper drops them while Cursor may edit files or run tools. OpenClaw uses this flag to decide whether empty/error turns may be retried, so a Cursor run that already mutated the workspace can be replayed and duplicate side effects.
  - confidence: 0.89
- **[P2] Return harness errors when Agent.create fails:** `extensions/cursor-sdk/src/run-attempt.ts:115`
  - body: `Agent.create(...)` is awaited before the guarded `try/finally`, so auth/config/network failures reject out of `runCursorSdkAttempt` instead of returning an `AgentHarnessAttemptResult`, and the timeout plus upstream abort listener are not cleaned up. Move creation inside the guarded block and dispose only after an agent exists.
  - confidence: 0.86
- **[P2] Race send setup against the harness timeout:** `extensions/cursor-sdk/src/run-attempt.ts:126`
  - body: The timeout only aborts `runAbortController`; before `agent.send()` resolves there is no `Run` to cancel, and `SendOptions` has no `AbortSignal`, so a hung send can block past `params.timeoutMs` instead of returning the timeout result this harness promises. Wrap create/send setup in a timeout race or make the unsupported gap explicit and fail closed.
  - confidence: 0.78
