## Summary

Keep this PR open: it targets a real Cursor SDK agent-runtime gap through the right general AgentHarness/plugin seam, but the current branch is not merge-ready and needs maintainer security/product approval before OpenClaw bundles a trusted Cursor local/cloud runtime.

## What This Changes

The branch adds a bundled `cursor-sdk` provider plus AgentHarness plugin, Cursor SDK runtime/tests, lockfile updates, a workspace `undici` override, and minor formatting-only test churn.

## Best Possible Solution

If maintainers approve an official Cursor SDK runtime, keep the provider-plus-AgentHarness shape but fix full-lifecycle cancellation, replay/side-effect metadata, support-contract docs, changelog coverage, and dependency pinning; otherwise publish this as an external ClawHub/npm plugin.

## Reproduction Assessment

Yes for the review findings. Static inspection of the final PR diff plus the published `@cursor/sdk@1.0.11` types is enough to show the abort, wait-cancellation, and replay-metadata defects without a live Cursor key.

## Solution Assessment

No as-is. AgentHarness is the narrow maintainable seam for a native Cursor runtime, but the PR needs lifecycle/replay fixes and explicit maintainer approval for the trusted bundled dependency/runtime footprint.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Cover create/send with abortable timeout:** `extensions/cursor-sdk/src/run-attempt.ts:101-110`
  - body: `Agent.create()` and `agent.send()` are awaited directly while the PR's timeout and upstream abort only abort a local controller. The published SDK types expose no `AbortSignal` for either call, so a stalled local startup or cloud reservation can ignore `/stop` and `params.timeoutMs` until the promise resolves.
  - confidence: 0.9
- **[P2] Keep cancellation active through wait:** `extensions/cursor-sdk/src/run-attempt.ts:125-141`
  - body: The abort listener that calls `run.cancel()` is removed as soon as streaming ends, before `run.wait()` completes. A timeout, gateway shutdown, or `/stop` during that final wait no longer cancels the Cursor run and can hang or return a late success after OpenClaw's deadline.
  - confidence: 0.9
- **[P2] Mark hidden Cursor execution as replay-unsafe:** `extensions/cursor-sdk/src/run-attempt.ts:213-220`
  - body: The harness intentionally drops Cursor `tool_call` events and assistant `tool_use` blocks, but returns `toolMetas: []` and `replaySafe: true`. Retry gates rely on that metadata to avoid resubmitting turns after side effects, so a Cursor run that edited files, ran shell, or spawned agents can be replayed.
  - confidence: 0.86
- **[P3] Add the required changelog entry:** `extensions/cursor-sdk/index.ts:6-10`
  - body: This adds a user-facing selectable Cursor SDK runtime/plugin, but the branch does not update `CHANGELOG.md`. Repo policy requires changelog coverage before feature work like this can merge.
  - confidence: 0.88
