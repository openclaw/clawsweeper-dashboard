## Summary

Keep open: current main does not implement a Cursor SDK harness, the PR is paired with an open same-author feature issue, and the branch still has blocking lifecycle/replay issues plus a bundled-runtime security/product decision before merge.

## What This Changes

The PR adds a bundled `extensions/cursor-sdk` plugin that registers a Cursor SDK provider and AgentHarness, adds plugin config/tests, stages `@cursor/sdk`, and changes lockfile plus workspace `undici` resolution.

## Best Possible Solution

Approve the official bundled Cursor runtime first; if approved, keep the AgentHarness plugin direction but fix lifecycle cancellation, replay metadata, docs/changelog, and dependency pinning before merge, otherwise publish it externally through ClawHub/npm.

## Reproduction Assessment

Yes. Static inspection of the PR head plus `@cursor/sdk@1.0.11` type declarations reproduces the cancellation and replay findings without a live Cursor API key.

## Solution Assessment

No as-is. The plugin seam is the right narrow integration point if OpenClaw wants an official Cursor SDK runtime, but the current implementation does not yet preserve OpenClaw timeout, stop, replay, or trusted-runtime policy semantics.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Guard startup and first send with abort racing:** `extensions/cursor-sdk/src/run-attempt.ts:115-126`
  - body: `Agent.create()` and `agent.send()` are awaited directly, but the Cursor SDK types do not accept an AbortSignal for either call. If `/stop` or the timeout fires before `send()` returns, OpenClaw has no `Run` to cancel and the attempt can remain stuck past its deadline.
  - confidence: 0.9
- **[P2] Keep cancellation active through wait:** `extensions/cursor-sdk/src/run-attempt.ts:142-161`
  - body: The abort listener that calls `run.cancel()` is removed after streaming and before `run.wait()`. A timeout, gateway shutdown, or `/stop` during that final wait no longer cancels the Cursor run and can hang or return a late success.
  - confidence: 0.9
- **[P2] Mark hidden Cursor execution replay-unsafe:** `extensions/cursor-sdk/src/run-attempt.ts:242-249`
  - body: The harness drops Cursor tool events but always reports `toolMetas: []` and `replaySafe: true`. A Cursor run that edited files, ran shell, used MCP, or spawned subagents can therefore be retried by paths that trust replay metadata.
  - confidence: 0.86
- **[P3] Add the required changelog entry:** `extensions/cursor-sdk/index.ts:7-8`
  - body: This adds a user-facing selectable Cursor SDK runtime, but the branch does not update `CHANGELOG.md`. Repo policy requires a changelog entry for feature work before merge.
  - confidence: 0.88
