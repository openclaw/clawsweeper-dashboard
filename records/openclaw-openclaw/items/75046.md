## Summary

Keep this PR open for maintainer review. Current main does not already include a bundled Cursor SDK harness, and the PR is paired with the still-open feature request #75044, but the proposed patch still has blocking lifecycle/replay bugs plus a security and supply-chain approval question before it should merge as an official bundled plugin.

## What This Changes

The PR adds a bundled `extensions/cursor-sdk` native plugin that registers Cursor as a provider and AgentHarness, stages `@cursor/sdk`, adds config/tests, and updates the workspace lockfile plus a workspace-wide `undici` override.

## Best Possible Solution

If maintainers approve official bundling, keep the provider-plus-AgentHarness shape but make the full Cursor lifecycle abortable, keep cancellation active through wait, mark or observe side effects accurately, document the Cursor-owned execution boundary, use an owner-approved deterministic dependency plan, and add the required changelog/tests. Without that approval, the same integration should live as an external ClawHub/npm plugin using the existing plugin seams.

## Reproduction Assessment

Yes for the blocking review findings. Static inspection of the PR head plus the published `@cursor/sdk@1.0.11` types shows create/send/wait are awaited without AbortSignal support, cancellation is removed before `run.wait()`, and the harness marks hidden Cursor execution as replay-safe.

## Solution Assessment

No as-is. AgentHarness is the narrow maintainable seam for this kind of native runtime, but this patch is not the best complete solution until it preserves OpenClaw abort/timeout/replay semantics and receives explicit maintainer product/security approval for bundling Cursor SDK as a trusted runtime.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Race startup and first send against abort:** `extensions/cursor-sdk/src/run-attempt.ts:115-126`
  - body: `Agent.create()` and `agent.send()` are awaited directly even though the SDK types expose no AbortSignal option and there is no `Run` to cancel before `send()` resolves. If local startup or cloud reservation stalls, `/stop` or `params.timeoutMs` only aborts the local controller and the attempt can remain stuck past OpenClaw's deadline.
  - confidence: 0.9
- **[P2] Keep cancellation active through wait:** `extensions/cursor-sdk/src/run-attempt.ts:143-161`
  - body: The abort listener that calls `run.cancel()` is removed as soon as streaming finishes, before `run.wait()` is awaited. A timeout, gateway shutdown, or `/stop` during that final wait no longer cancels the Cursor run and can either hang or return a late success after OpenClaw's deadline.
  - confidence: 0.9
- **[P2] Treat hidden Cursor tool runs as replay-unsafe:** `extensions/cursor-sdk/src/run-attempt.ts:229-232`
  - body: The harness intentionally drops Cursor tool events, yet `buildSuccessResult()` always reports `toolMetas: []` and `replayMetadata: { hadPotentialSideEffects: false, replaySafe: true }`. Current retry gates use that metadata to resubmit empty/planning/reasoning turns only when no side effects occurred, so a Cursor run that edited files or ran shell without visible text can be submitted again.
  - confidence: 0.82
- **[P3] Add the required changelog entry:** `extensions/cursor-sdk/index.ts:7-8`
  - body: This is a user-facing feature that adds a selectable Cursor SDK runtime, but the branch does not update `CHANGELOG.md`. Repo policy requires a changelog entry before merging feature work.
  - confidence: 0.87
