## Best Possible Solution

Land this only after auth and timeout handling are fixed, OpenClaw-path proof is supplied, plugin docs/label routing are aligned, and maintainers accept the dependency surface; otherwise redirect the plugin to ClawHub/npm distribution.

## Solution Assessment

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Resolve Cursor credentials before checking resolvedApiKey:** `extensions/cursor-sdk/src/run-attempt.ts:62-66`
  - body: A normal `cursor-sdk` run advertised with `CURSOR_API_KEY` can reach this branch with `params.resolvedApiKey` unset. Current main skips PI auth bootstrap for plugin-owned harnesses, and automatic plugin-harness profile ordering only runs when the runtime auth plan provides a harness auth provider, which is currently Codex-only. Resolve the Cursor credential through the plugin/profile/env contract before returning this missing-key result.
  - confidence: 0.91
- **[P2] Race SDK startup and send against the harness timeout:** `extensions/cursor-sdk/src/run-attempt.ts:115-126`
  - body: `Agent.create()` and `agent.send()` are awaited without a timeout race. The timer only aborts a local signal, but the Cursor SDK `send` options have no AbortSignal and no `Run` exists to cancel yet, so startup or initial send can still block past `params.timeoutMs` or throw outside the classified result path. Guard these promises and always clean up through the same lifecycle.
  - confidence: 0.88
- **[P3] Add label routing for the new plugin:** `extensions/cursor-sdk/openclaw.plugin.json:2`
  - body: Adding a bundled plugin under `extensions/cursor-sdk` creates a new plugin surface, but this PR does not add a `.github/labeler.yml` route. Future Cursor SDK changes will not get a plugin label for routing; add the labeler entry and have maintainers create the matching label if needed.
  - confidence: 0.86
