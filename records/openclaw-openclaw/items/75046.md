## Summary

Keep this PR open. Current main does not already implement a Cursor SDK harness, and this is the active implementation paired with open #75044, but the patch still has blocking timeout/abort lifecycle bugs plus security and supply-chain questions for a bundled trusted agent runtime.

## What This Changes

The PR adds a bundled `extensions/cursor-sdk` provider and AgentHarness plugin with Cursor SDK local/cloud execution, plugin config, tests, `@cursor/sdk` runtime dependencies, lockfile updates, a workspace `undici` override, and one formatting-only config schema change.

## Best Possible Solution

If maintainers want Cursor SDK bundled, land it as a trusted provider-plus-harness plugin only after send/stream/wait cancellation is fixed, Cursor-owned shell/file/MCP/subagent policy is documented, dependency/override policy is reviewed, and focused tests cover timeout and upstream abort phases. If that bar is not met, keep the implementation external via ClawHub/npm while preserving the existing plugin API.

## Reproduction Assessment

Yes for the review findings. Static tracing of the PR head plus `@cursor/sdk@1.0.7` docs/types shows first runtime initialization happens in `agent.send()`, while the PR only cancels after a `run` exists and removes that cancellation hook before `run.wait()`.

## Solution Assessment

No as-is. The provider-plus-harness shape is a reasonable direction, but this implementation is not yet the safest maintainable solution because it does not fully preserve OpenClaw timeout/abort semantics and still needs trusted-runtime security approval.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Race first send against abort and timeout:** `extensions/cursor-sdk/src/run-attempt.ts:126`
  - body: `Agent.create()` defers runtime initialization until the first send, but this awaits `agent.send(params.prompt)` without racing it against `runAbortController.signal` or `params.timeoutMs`. If Cursor local startup or cloud run creation stalls, the timer only aborts an internal controller and no `run` exists to cancel, so failover can hang past the configured timeout.
  - confidence: 0.9
- **[P2] Keep cancellation active until wait completes:** `extensions/cursor-sdk/src/run-attempt.ts:143-161`
  - body: The abort listener that calls `run.cancel()` is removed immediately after `run.stream()` finishes, before `run.wait()` runs. If the final wait stalls or an upstream abort arrives in that phase, the timeout/abort signal no longer cancels the Cursor run and this attempt can block instead of returning a timed-out or aborted result.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The diff introduces a bundled trusted Cursor local/cloud runtime plus a new SDK/native dependency footprint and workspace override that need maintainer security review.

  - body: The local runtime starts Cursor in the workspace and lets Cursor own file edits, shell, MCP, and subagent behavior, but the PR does not yet document or enforce how OpenClaw sandbox, approval, abort, timeout, and diagnostic policy apply to that execution surface.
  - confidence: 0.86
- **[medium] Review the Cursor SDK dependency footprint:** `extensions/cursor-sdk/package.json:7`
