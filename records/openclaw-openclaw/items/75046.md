## Summary

Keep this PR open. Current main does not yet include a Cursor SDK local/cloud agent harness, and the branch uses the intended plugin/provider harness seam, but the latest head is not merge-ready because timeout/abort handling can still hang, the trusted Cursor runtime and dependency footprint need maintainer security/product review, and the user-facing feature is missing a changelog entry.

## What This Changes

The branch adds a bundled `extensions/cursor-sdk` plugin that registers a `cursor-sdk` provider and AgentHarness, stages `@cursor/sdk`, adds unit tests, updates the lockfile, adds a workspace `undici` override, and includes one incidental config-schema formatting change.

## Best Possible Solution

If maintainers want Cursor SDK bundled, keep the provider-plus-harness shape but fix lifecycle cancellation across send, stream, and wait; document the Cursor-owned tool/sandbox/approval boundary; replace broad dependency overrides with reviewed pins or a dependency-owner plan; add focused lifecycle tests and the required changelog. If that product/security bar is not approved, publish this as an external ClawHub/npm plugin using the existing SDK surface.

## Reproduction Assessment

Yes for the blocking review findings. Static tracing of the PR head plus `@cursor/sdk@1.0.7` README/types shows initialization happens on first `send`, `send` has no abort option, and `wait` is a separate cancellable phase after streaming.

## Solution Assessment

No as-is. The plugin/provider direction is maintainable, but the current implementation does not yet preserve OpenClaw timeout/abort semantics and the bundled trusted-runtime/dependency decision needs explicit maintainer approval.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Race first send against abort and timeout:** `extensions/cursor-sdk/src/run-attempt.ts:126`
  - body: `Agent.create()` returns immediately, but the Cursor SDK defers runtime startup until the first `send`, and `SendOptions` has no abort signal. Awaiting `agent.send(params.prompt)` here means a local startup or cloud reservation stall can run past `params.timeoutMs` because no `Run` exists yet for `run.cancel()`. Race this phase against timeout/upstream abort and return promptly.
  - confidence: 0.9
- **[P2] Keep cancellation active through wait:** `extensions/cursor-sdk/src/run-attempt.ts:143`
  - body: The abort listener that calls `run.cancel()` is removed as soon as streaming finishes, before `run.wait()` runs. If the final wait stalls or the user aborts in that phase, the Cursor run is no longer cancelled and the attempt can block past OpenClaw's timeout. Keep the listener until after `wait()` or race `wait()` against the abort signal.
  - confidence: 0.88
- **[P3] Add the required changelog entry:** `extensions/cursor-sdk/index.ts:7`
  - body: This is a user-facing `feat` that adds a new selectable Cursor SDK runtime, but the branch does not update `CHANGELOG.md`. Repo policy requires user-facing feat/fix/perf changes to include a changelog entry before merge.
  - confidence: 0.86
