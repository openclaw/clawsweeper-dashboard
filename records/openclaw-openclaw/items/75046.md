## Summary

Keep open. Current main does not have a native `@cursor/sdk` AgentHarness plugin, and this PR is the active implementation paired with open issue #75044, but the patch is not mergeable as-is because it leaves OpenClaw abort/replay semantics incomplete and needs explicit product/security/dependency approval for bundling Cursor's trusted local/cloud runtime.

## What This Changes

The PR adds a bundled `extensions/cursor-sdk` native plugin that registers a Cursor SDK provider and AgentHarness, wires `CURSOR_API_KEY` auth, stages `@cursor/sdk`, adds focused tests, and updates workspace dependency resolution plus the lockfile.

## Best Possible Solution

If maintainers approve official bundled Cursor support, keep the provider-plus-AgentHarness shape but make the full SDK lifecycle abortable, preserve OpenClaw replay/side-effect metadata, document the Cursor-owned execution boundary, use a deterministic reviewed dependency plan, and add the required changelog/tests. Without that approval, the integration should remain an external ClawHub/npm plugin using the existing plugin seams.

## Reproduction Assessment

Yes for the blocking review findings. Static inspection of the PR head plus `@cursor/sdk@1.0.11` types shows the unabortable create/send/wait windows, hidden SDK tool events, and replay-safe result metadata; current-main inspection also confirms there is no native `cursor-sdk` plugin already present.

## Solution Assessment

No as-is. AgentHarness is the narrow maintainable seam if OpenClaw chooses to trust and bundle Cursor SDK, but this patch needs lifecycle, replay, changelog, and dependency/security work before it is the right implementation.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Keep create and send within the abortable lifecycle:** `extensions/cursor-sdk/src/run-attempt.ts:115-126`
  - body: `Agent.create()` and `agent.send()` are awaited without any SDK abort option and before a cancellable `Run` exists. If local startup or cloud reservation stalls, `/stop`, gateway shutdown, or `params.timeoutMs` only aborts the local controller and the attempt can remain stuck past OpenClaw's deadline.
  - confidence: 0.9
- **[P2] Keep cancellation active through run.wait():** `extensions/cursor-sdk/src/run-attempt.ts:142-161`
  - body: The abort listener that calls `run.cancel()` is removed as soon as streaming finishes, before `run.wait()` is awaited. A timeout or upstream abort during that final wait no longer cancels the Cursor run and can hang or return a late success after OpenClaw already stopped the turn.
  - confidence: 0.9
- **[P2] Mark hidden Cursor tool activity replay-unsafe:** `extensions/cursor-sdk/src/run-attempt.ts:242-249`
  - body: The harness intentionally drops Cursor tool-call events, then returns `toolMetas: []` and `replayMetadata: { hadPotentialSideEffects: false, replaySafe: true }`. OpenClaw retry gates use that metadata, so a Cursor run that edited files, ran shell, or spawned subagents without visible text can be submitted again.
  - confidence: 0.84
- **[P3] Add the required changelog entry:** `extensions/cursor-sdk/index.ts:5-8`
  - body: This adds a user-facing selectable Cursor SDK runtime, but the branch does not update `CHANGELOG.md`. OpenClaw policy requires a changelog entry for feature work before merge.
  - confidence: 0.87
