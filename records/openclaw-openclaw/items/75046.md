## Best Possible Solution

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Resolve Cursor credentials before checking resolvedApiKey:** `extensions/cursor-sdk/src/run-attempt.ts:62-66`
  - body: Current main skips PI generic auth bootstrap for plugin-owned harnesses, and `params.resolvedApiKey` is only populated from `apiKeyInfo`. With ordinary `CURSOR_API_KEY` or profile setup, this runner can return the missing-key error before Cursor is called; resolve the forwarded plugin auth/profile/env credential through the harness-safe contract first.
  - confidence: 0.92
- **[P2] Catch SDK creation failures inside the harness result:** `extensions/cursor-sdk/src/run-attempt.ts:115`
  - body: `Agent.create()` runs before the `try` block. If Cursor rejects during startup, auth validation, or network setup, the exception escapes harness selection instead of becoming a classified `AgentHarnessAttemptResult`, so normal prompt failure handling is bypassed. Move creation into the guarded lifecycle and dispose only after an agent exists.
  - confidence: 0.86
