## Summary

Keep this PR open. Current main does not implement a Cursor SDK AgentHarness, and this PR is paired with the same-author open feature request #75044, but the branch still has concrete abort/timeout/replay correctness bugs and needs explicit maintainer product/security/dependency approval before bundling a trusted Cursor local/cloud runtime. The discussion usefully clarified that tool/subagent output is intentionally flattened to final text, but that does not yet resolve OpenClaw's replay-safety or diagnostic boundary concerns.

## What This Changes

The PR adds a bundled `extensions/cursor-sdk` plugin that registers a Cursor SDK provider and AgentHarness, stages `@cursor/sdk`, adds manifest/config/tests, and changes the workspace lockfile plus a workspace-wide `undici` override.

## Best Possible Solution

If maintainers approve official bundling, keep the provider-plus-AgentHarness shape but make the full Cursor lifecycle abortable, keep cancellation active through final wait, mark or observe side effects accurately, document the Cursor-owned execution boundary, use an approved deterministic dependency plan, and add changelog/labeler/docs coverage. If that approval is not granted, the same integration should live as an external ClawHub/npm plugin using the existing plugin seams.

## Reproduction Assessment

Yes for the blocking review findings. Static inspection of PR head 729967e plus the published `@cursor/sdk@1.0.11` types reproduces the create/send/wait abort gaps and the replay metadata mismatch without needing a live Cursor credential.

## Solution Assessment

No as-is. AgentHarness is the narrow maintainable seam for this kind of native runtime, but this patch is not a complete solution until it preserves OpenClaw abort/timeout/replay semantics and receives explicit maintainer product/security approval for bundling Cursor SDK as a trusted runtime.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Guard startup and first send with abortable timeout:** `extensions/cursor-sdk/src/run-attempt.ts:115-126`
  - body: `Agent.create()` and `agent.send()` are awaited directly, but the SDK types do not expose an AbortSignal for either call and no `Run` exists to cancel until `send()` resolves. If local startup or cloud reservation stalls, `/stop` or `params.timeoutMs` only aborts the local controller and the attempt can remain stuck past OpenClaw's deadline.
  - confidence: 0.9
- **[P2] Keep cancellation active through wait:** `extensions/cursor-sdk/src/run-attempt.ts:142-161`
  - body: The abort listener that calls `run.cancel()` is removed as soon as streaming finishes, before `run.wait()` is awaited. A timeout, gateway shutdown, or `/stop` during that final wait no longer cancels the Cursor run and can either hang or return a late success after OpenClaw's deadline.
  - confidence: 0.9
- **[P2] Mark hidden Cursor tool execution replay-unsafe:** `extensions/cursor-sdk/src/run-attempt.ts:249`
  - body: The harness intentionally drops Cursor `tool_call` events, yet successful attempts always return `toolMetas: []` and `replayMetadata` marked safe. Current retry gates use this metadata to decide whether empty/planning/reasoning turns can be resubmitted, so a Cursor run that edited files or ran shell without visible text can be replayed.
  - confidence: 0.84
- **[P3] Add the required changelog entry:** `extensions/cursor-sdk/index.ts:7-8`
  - body: This is a user-facing feature that adds a selectable Cursor SDK runtime, but the branch does not update `CHANGELOG.md`. Repo policy requires a changelog entry before merging feature work.
  - confidence: 0.87
