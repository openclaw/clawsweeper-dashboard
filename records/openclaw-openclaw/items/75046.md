## Summary

Keep this PR open. Current main still has no Cursor SDK harness, and the linked feature issue is paired with this implementation PR, but the patch still has blocking timeout/abort lifecycle bugs plus a trusted-runtime and dependency-policy decision that needs maintainer review before merge.

## What This Changes

The PR adds a bundled `extensions/cursor-sdk` plugin with provider/auth registration, an AgentHarness runner, Cursor SDK runtime dependency staging, unit tests, lockfile changes, and a workspace-wide `undici` override.

## Best Possible Solution

If maintainers approve official bundling, keep the plugin/provider/harness architecture but fix cancellation across send, stream, and wait; document the Cursor-owned execution boundary; replace the broad dependency override with a reviewed deterministic dependency plan; add lifecycle tests and a changelog. If that approval is not granted, publish the Cursor SDK integration externally through ClawHub/npm using the existing AgentHarness and provider seams.

## Reproduction Assessment

Yes for the blocking review findings. Static inspection of the PR head plus `@cursor/sdk@1.0.7` types shows `agent.send()` has no abort option, `run.wait()` is a separate operation, and the PR removes cancellation before waiting.

## Solution Assessment

No as-is. The plugin-shaped AgentHarness approach is the right technical seam, but the current implementation does not yet preserve OpenClaw abort/timeout guarantees and the bundled trusted-runtime/dependency decision is not resolved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Race first send against abort and timeout:** `extensions/cursor-sdk/src/run-attempt.ts:126`
  - body: The timeout only aborts the local controller, but before `agent.send` resolves there is no `Run` to cancel and Cursor `SendOptions` has no AbortSignal. A local startup or cloud reservation stall can therefore block past `params.timeoutMs` or `/stop`; race this phase against timeout/upstream abort and dispose promptly.
  - confidence: 0.9
- **[P2] Keep cancellation active through wait:** `extensions/cursor-sdk/src/run-attempt.ts:143`
  - body: The listener that calls `run.cancel()` is removed as soon as streaming finishes, then `run.wait()` is awaited. If timeout or upstream abort fires during that final wait, the Cursor run is no longer cancelled and the attempt can hang; keep cancellation active until wait completes or race `wait()`.
  - confidence: 0.88
- **[P3] Add the required changelog entry:** `extensions/cursor-sdk/index.ts:7`
  - body: This user-facing feature adds a selectable Cursor SDK runtime, but the branch has no `CHANGELOG.md` entry for `cursor-sdk`, `Cursor SDK`, `@cursor/sdk`, or the linked refs. Repo policy requires a changelog entry before merging feature work.
  - confidence: 0.86
