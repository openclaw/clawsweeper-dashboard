## Summary

Keep open. The PR is a plausible Cursor SDK plugin, but the branch still has source-reproducible OpenClaw-path blockers, insufficient contributor proof for the actual plugin path, and a dependency/security product decision before it can merge.

## What This Changes

Adds a bundled `extensions/cursor-sdk` plugin with provider and AgentHarness registration, Cursor SDK unit/live tests, and workspace dependency policy changes for `@cursor/sdk`, `undici`, and `sqlite3`.

## Best Possible Solution

Land a plugin-scoped Cursor SDK harness only after fixing OpenClaw-path auth/activation/timeout behavior, adding real OpenClaw-path proof, and accepting the dependency surface; otherwise publish it through ClawHub/npm instead of bundling it.

## Reproduction Assessment

Yes, source-reproducible for the blocking PR behavior: current main skips generic auth bootstrap for plugin harnesses and only forwards harness auth for Codex, while the proposed runner requires `params.resolvedApiKey`. I did not run a live Cursor/OpenClaw turn in this read-only review.

## Solution Assessment

No. The AgentHarness plugin seam is the right technical boundary if maintainers want this bundled, but this branch still needs OpenClaw-path credential handling, guarded SDK startup/send behavior, real proof, and dependency/product acceptance.

  - confidence: 0.88
