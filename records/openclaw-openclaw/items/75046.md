## Summary

Keep open. Current main does not include Cursor SDK support, and this PR still has blocking adapter issues: `Agent.create` failures escape cleanup/result conversion, Cursor tool side effects are reported as replay-safe, and the supplied proof exercises the SDK directly rather than the OpenClaw harness. Bundling this optional plugin and its native dependency surface also needs maintainer approval.

## What This Changes

The PR adds a bundled `extensions/cursor-sdk` plugin with provider registration, an AgentHarness runner, Cursor SDK dependency and lockfile changes, and unit/live tests.

## Best Possible Solution

If maintainers want Cursor SDK bundled, land it only after fixing lifecycle/replay semantics, adding label routing and any required docs, approving the dependency/build surface, and collecting real OpenClaw harness proof; otherwise redirect the plugin to ClawHub/npm while keeping the core AgentHarness seam unchanged.

## Reproduction Assessment

Yes, for the review findings. Source inspection shows `Agent.create` can reject before the guarded attempt block, and the Cursor SDK stream contract exposes `tool_call` events while the adapter always returns replay-safe metadata.

## Solution Assessment

No. The AgentHarness plugin seam is the right technical boundary if maintainers accept a bundled plugin, but this patch is not ready because it mishandles create failures, under-reports side effects, and lacks real OpenClaw harness proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Handle Agent.create failures inside the attempt guard:** `extensions/cursor-sdk/src/run-attempt.ts:115`
  - body: The timeout and upstream abort listener are installed before `Agent.create`, but `Agent.create` is awaited before the `try/catch/finally`. If the SDK rejects during agent creation, the harness throws instead of returning a prompt error and never clears the timer or abort listener. Move creation into the guarded block, use a nullable `agent`, and dispose only after creation succeeds.
  - confidence: 0.91
- **[P2] Do not mark Cursor tool runs replay-safe:** `extensions/cursor-sdk/src/run-attempt.ts:249-250`
  - body: Cursor SDK can emit `tool_call` events and the PR states Cursor owns file-editing and terminal tools, but the adapter drops all tool events and always returns `hadPotentialSideEffects: false` and `replaySafe: true`. That lets timeout or empty-response retry paths resubmit a turn after side effects, potentially duplicating file edits or commands. Track tool-call side effects or conservatively mark these runs replay-unsafe when tool activity is possible or observed.
  - confidence: 0.88
- **[P3] Add the new plugin to label routing:** `extensions/cursor-sdk/openclaw.plugin.json:2`
  - body: This introduces a new `cursor-sdk` plugin surface, but `.github/labeler.yml` has no `extensions: cursor-sdk` entry. Root policy requires new plugin surfaces to update label routing, otherwise future Cursor SDK changes will not get the expected review label.
  - confidence: 0.84
