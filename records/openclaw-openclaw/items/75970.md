## Summary

Keep this PR open with changes requested. The store-boundary quarantine/preservation work matches the linked cron dashboard bug, but the PR's Control UI fallback still lets a schedule-less row through the default list filter if malformed data reaches the browser.

## What This Changes

The PR adds cron store validation/quarantine for malformed persisted jobs, preserves quarantined rows on saves, normalizes string interval schedules, adds Control UI guards, tests, and a changelog entry.

## Best Possible Solution

Keep the backend quarantine/preservation path, add a client-side schedule-shape boundary before any cron row reaches render/edit, then land the PR to close #51871.

## Reproduction Assessment

Yes. Source inspection of PR head shows a cron.list row with a valid payload but missing schedule can pass the payload-only load boundary and the default all filter, then hit formatCronSchedule and dereference schedule.kind.

## Solution Assessment

No, not yet. The backend quarantine is the right narrow fix, but the PR's stated UI fallback should also drop unsupported schedules at the list boundary or render them safely.
