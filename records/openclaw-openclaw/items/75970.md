## Summary

Keep open: this is a plausible narrow bug fix for the linked cron dashboard regression, with current CI green, but it still needs contributor-supplied real behavior proof before merge and has one small log-noise cleanup finding.

## What This Changes

The PR quarantines malformed persisted cron jobs during store load, preserves quarantined rows on later saves, adds Control UI schedule guards, regression tests, and a changelog entry.

## Best Possible Solution

Merge the store-boundary quarantine/preservation and UI guard after contributor-supplied real behavior proof is added and the repeated malformed-row warning is deduped.

## Reproduction Assessment

Yes. Source inspection shows current main can pass a valid-payload row with a missing schedule through cron.list into Control UI paths that dereference schedule.kind; I did not run a browser scenario because this review is read-only.

## Solution Assessment

Mostly yes. Quarantining unrecoverable persisted rows at the cron store boundary while preserving them for repair, plus a defensive UI schedule guard, is the narrow maintainable fix; the safer remaining cleanup is deduping the warning emitted for persistent bad rows.
