## Summary

Keep open: the direction is useful, but the PR is dirty against main and would regress existing `sessions_send` safeguards around delegated reply reads, parent-owned child skips, provenance, target lanes, and thread-target rejection.

## What This Changes

The PR adds a typed plugin `sessions_send` hook, routes `sessions_send` through hook/delegated dispatch before fallback, strips caller-supplied `roundOneReply`, updates SDK exports/API hashes, and adds tests.

## Best Possible Solution

Land one clean rebased implementation that adds the typed hook while preserving current-main thread-target rejection, parent-owned child A2A skips, provenance markers, target-scoped lanes, current TypeBox imports, SDK exports, docs, changelog, and regression coverage.

Yes. A high-confidence static reproduction exists in the PR head: delegated dispatch can finish before the post-hook baseline read, and the stale branch also omits current-main guards visible in source.

## Solution Assessment

No. The hook direction is plausible, but this is not the best mergeable solution until it is rebased and consolidated with #68622 without dropping current safeguards.
