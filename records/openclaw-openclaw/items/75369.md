## Summary

Keep open. The PR is a useful follow-up to #68622 and moves the trust boundary in the right direction, but the branch is still not merge-ready because it can suppress fast delegated replies and it predates current-main safeguards for ACP-child A2A skipping, inter-session provenance, and target-scoped lanes.

## What This Changes

The PR adds a typed plugin `sessions_send` hook, SDK/API baseline exports, delegated dispatch metadata handling, trust-boundary hardening for delegated `roundOneReply`, and sessions tool regression tests.

## Best Possible Solution

Land a clean, rebased implementation that keeps the `sessions_send` hook and trust hardening, captures delegated baselines before plugin-started work or relies on trusted plugin-owned replies, preserves current-main ACP/provenance/lane behavior, exports the complete hook type contract, documents the hook, and then supersedes or closes #68622 after merge.

## Reproduction Assessment

Yes. The PR head plus current `run-wait` logic provide a high-confidence static reproduction: if a plugin finishes the delegated run before the PR reads the baseline, the later wait/read helper filters the new reply as unchanged.

## Solution Assessment

No. The direction is plausible, but this branch is not the best merge shape until it is rebased and the delegated baseline, ACP-child skip, provenance, lane, SDK export, and docs gaps are fixed.

Full review comments:

- **[P1] Snapshot delegated runs before handing control to hooks:** `src/agents/tools/sessions-send-tool.ts:450-455`
  - body: In delegated mode, this reads the baseline only after `runSessionsSend` returns. A plugin can start and complete the delegated run before returning `waitRunId`, so the baseline can capture the new assistant reply and `waitForAgentRunAndReadUpdatedAssistantReply` will return no reply. Capture the baseline before plugin dispatch, or use a trusted plugin reply without filtering it as the baseline.
  - confidence: 0.9
- **[P1] Preserve the ACP-child A2A skip on rebase:** `src/agents/tools/sessions-send-tool.ts:386-388`
  - body: Current main skips `runSessionsSendA2AFlow` when the requester owns a background ACP child and reports `delivery.status="skipped"`; this branch always marks delivery pending and starts A2A. Rebasing this as-is would reintroduce the parent-child ping-pong path current main already prevents.
  - confidence: 0.86
- **[P2] Keep inter-session provenance and target lanes:** `src/agents/tools/sessions-send-tool.ts:491-496`
  - body: This rebuilt `sendParams` block uses the raw message and unscoped nested lane, while current main annotates inter-session messages and scopes nested lanes with `resolveNestedAgentLaneForSession(resolvedKey)`. Preserve those fields so forwarded prompts stay marked and different target sessions do not block each other.
  - confidence: 0.84
- **[P3] Export the nested sessions_send hook types:** `src/plugin-sdk/core.ts:102-106`
  - body: The new public event/result types expose `PluginHookSessionsSendTask` and `PluginHookSessionsSendCancelTarget`, but `plugin-sdk/core` only re-exports the context/event/result names.   - confidence: 0.74
