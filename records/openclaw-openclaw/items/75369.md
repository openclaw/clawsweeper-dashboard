## Summary

Keep open: the hook work is still useful and is not implemented on current main, but this PR is not mergeable as-is because the head still has a trust-boundary bug and is stale enough to drop recent current-main sessions/plugin-hook behavior.

## What This Changes

This PR adds a typed plugin `sessions_send` hook for direct or delegated delivery, exports SDK hook types, documents the hook, updates API/changelog files, and adds sessions regression tests.

## Best Possible Solution

Consolidate this into one current-main `sessions_send` hook PR that preserves recent sessions and hook API changes, keeps requester identity host-owned, and lands after owner review of the public plugin contract.

## Reproduction Assessment

Yes. Source inspection of the PR head reproduces the review blockers: caller-controlled requester metadata reaches `event.task.requester`, and the stale branch lacks current-main sessions/plugin hook additions.

## Solution Assessment

No. The hook direction is plausible, but the mergeable solution must rebase over current main and keep requester identity in host-owned context or sanitized defaults instead of accepting raw task metadata.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Keep requester identity host-owned:** `src/agents/tools/sessions-send-tool.ts:153-160`
  - body: These fields prefer `params.task.requester.*` over the trusted requester values passed by the host. A tool caller can spoof `event.task.requester.sessionKey` or `channel`, and plugin authors may treat the sanitized task as trusted metadata. Populate `task.requester` only from host-owned defaults, or leave trusted requester identity only in `ctx`.
  - confidence: 0.86
- **[P2] Preserve configured-agent session bootstrap:** `src/agents/tools/sessions-send-tool.ts:257`
  - body: This branch predates current main's configured-agent main-session bootstrap. In the PR head, `agentId` is only used inside the `label` path, so an `agentId`-only `sessions_send` call falls through to the missing-target error and the later `ensureConfiguredAgentMainSession` guard from main is absent. Rebase the hook path on top of current main and keep that bootstrap behavior.
  - confidence: 0.88
- **[P2] Preserve current plugin hook API entries:** `src/plugins/hook-types.ts:107`
  - body: The PR head's hook-name union/list still stops at `before_install`, while current main exposes `before_agent_run`. Merging this stale `hook-types` surface without carrying current main forward would remove an existing public plugin hook from the SDK contract.
  - confidence: 0.8
