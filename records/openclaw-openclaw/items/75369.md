## Best Possible Solution

Refresh this PR onto current `main`, keep requester identity host-owned, preserve existing sessions and hook contracts, and land only after maintainer API/security review.

Yes for the review blockers: source inspection of the PR head shows spoofable requester metadata and missing current-main API/session behavior. This is not a current-main bug reproduction because the hook itself is a new public capability.

## Solution Assessment

No, not as the current head. The hook direction is plausible, but the mergeable solution must preserve current `main` behavior and keep requester identity in host-owned context before the SDK contract is accepted.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Keep requester identity host-owned:** `src/agents/tools/sessions-send-tool.ts:152-160`
  - body: `readSessionsSendHookTask` prefers `params.task.requester.*` over trusted host defaults, so a tool caller can spoof `event.task.requester.sessionKey` or `channel` in metadata that plugins may treat as sanitized. Populate requester identity only from host-owned context/defaults, or leave it only in `ctx`.
  - confidence: 0.9
- **[P2] Preserve configured-agent session bootstrap:** `src/agents/tools/sessions-send-tool.ts:256-257`
  - body: Current `main` resolves `sessions_send({ agentId, message })` to the configured agent main session and ensures that session exists before delivery. This head only enters label lookup when `label` is present, so an `agentId`-only send regresses to the missing-target error instead of bootstrapping the configured agent session.
  - confidence: 0.88
- **[P2] Preserve the existing hook name contract:** `src/plugins/hook-types.ts:90-107`
  - body: The branch adds `sessions_send` from a stale hook list that omits current-main `before_agent_run`. Merging this version would remove an existing public Plugin SDK hook from `PluginHookName` and `PLUGIN_HOOK_NAMES`, so refresh the list instead of replacing it from the older base.
  - confidence: 0.86
