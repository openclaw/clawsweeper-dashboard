## Summary

Keep open. The PR addresses a real `sessions_send` delegated trust boundary, but the branch is dirty, overlaps open #68622, and the patch would drop several current-main inter-session safeguards before it is safe to merge.

## What This Changes

The PR adds a typed `sessions_send` plugin hook, routes `sessions_send` through direct/delegated hook handling, strips caller-supplied delegated `roundOneReply`, updates SDK API hashes, and adds tests.

## Best Possible Solution

Land one clean rebased implementation that adds the typed hook while preserving current thread-target rejection, parent-owned child A2A skips, provenance wrapping, target-scoped lanes, soft-timeout announce behavior, SDK exports, changelog, and regression coverage.

## Reproduction Assessment

Yes. A high-confidence static reproduction exists in the PR head: delegated dispatch can finish before the post-hook baseline read, and the stale branch also omits current-main guards visible in source.

## Solution Assessment

No. The hook direction is plausible, but this is not the best mergeable solution until it is rebased and consolidated with #68622 without dropping current safeguards.

Overall confidence: 0.91

Full review comments:

- **[P1] Snapshot reply history before delegated dispatch:** `src/agents/tools/sessions-send-tool.ts:450-455`
  - body: In delegated mode, `delegatedBaselineReply` is read after `runSessionsSend` returns. A fast plugin can start and finish before returning `waitRunId`, causing the new reply to become the baseline and suppressing the caller-visible reply and announce seed.
  - confidence: 0.91
- **[P1] Preserve parent-owned child A2A skips:** `src/agents/tools/sessions-send-tool.ts:386-388`
  - body: Current main skips A2A and reports `delivery.status="skipped"` when the requester owns the ACP/native child result path. This branch rebuilds delivery as always pending and starts A2A unconditionally, which can reintroduce duplicate parent-child delivery and echo loops on rebase.
  - confidence: 0.88
- **[P2] Keep delayed A2A alive after soft timeouts:** `src/agents/tools/sessions-send-tool.ts:463-469`
  - body: Current main treats nonterminal wait timeouts as accepted and starts the delayed A2A flow. The delegated path here returns `timeout` immediately, so delayed delegated replies can lose the announce path instead of completing asynchronously.
  - confidence: 0.84
- **[P2] Keep inter-session provenance and target lanes:** `src/agents/tools/sessions-send-tool.ts:490-496`
  - body: The rebuilt fallback `sendParams` sends raw text and uses the shared nested lane. Current main annotates forwarded prompts as inter-session content and scopes the nested lane by target session, so preserving those fields is required for prompt provenance and isolation.
  - confidence: 0.87
- **[P2] Keep thread-session target rejection:** `src/agents/tools/sessions-send-tool.ts:370-374`
  - body: Current main rejects thread-scoped session keys before building delivery state. This branch lacks that guard, so a rebase that keeps this flow can send tool-routed messages into human-facing thread sessions again.
  - confidence: 0.8
- **[P2] Keep the current TypeBox import:** `src/agents/tools/sessions-send-tool.ts:2`
  - body: The branch still imports `@sinclair/typebox`, while current main has migrated this file to `typebox` and the package manifests declare `typebox`. Preserve the current import during rebase to avoid dependency resolution drift.
  - confidence: 0.78
- **[P3] Export the nested hook types:** `src/plugin-sdk/core.ts:102-106`
  - body: The new public result/event shapes expose task and cancel-target structures, but the SDK core barrel only exports the context, event, and result names. Export the nested task and cancel-target types too so plugin authors can name the full contract.
  - confidence: 0.74
- **[P3] Add the required changelog entry:** `src/plugins/hook-types.ts:328-345`
  - body: This exposes a new public plugin hook and changes `sessions_send` behavior, but the PR does not update `CHANGELOG.md`. Repo policy requires user-facing fix, feature, and perf changes to include a changelog entry before merge.
  - confidence: 0.72
