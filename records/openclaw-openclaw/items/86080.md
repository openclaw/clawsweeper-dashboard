## Summary

Keep open: current main and v2026.6.8 still leave memory visibly idle when dreaming is off and the flush threshold is not reached, and no single merged related item covers the full diagnostics, onboarding, reset-flush, and threshold-policy gap.

Request to make default memory behavior visibly explain its idle state or create useful memory artifacts sooner when dreaming is disabled and compaction thresholds are not reached.

## Best Possible Solution

Define one maintainer-approved memory UX contract that explains idle state and chooses which lifecycle or threshold path should create artifacts sooner without adding competing memory behaviors.
