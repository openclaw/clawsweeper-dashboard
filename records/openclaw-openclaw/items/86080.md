## Summary

Keep open: current main and v2026.6.9 still leave visible memory artifacts idle when dreaming is disabled and the pre-compaction flush thresholds are not reached, so the remaining work is a maintainer product/defaults decision rather than conservative cleanup.

## What This Changes

Request clearer or more active default memory behavior when dreaming is off and memory flush or compaction thresholds have not been reached.

## Best Possible Solution

Keep this as the umbrella memory UX/defaults decision, then choose a maintainer-approved path that either explains the idle state clearly or intentionally changes when memory artifacts are written, while narrower reset-flush and threshold-scaling work stay in their linked issues.
