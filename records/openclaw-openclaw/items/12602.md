## Summary

Keep open. Current main supports explicit Slack Block Kit payloads, message-tool blocks, semantic presentation blocks, and opt-in interactive controls, but ordinary markdown-table/rich-output replies still flow through text rendering and the remaining work is already being tracked by active Slack follow-ups.

## What This Changes

Request to let Slack agents send richer Block Kit output, including authored blocks, buttons/selects, and automatic table or rich rendering instead of plain mrkdwn text.

## Best Possible Solution

Land or replace the native Slack markdown-table Block Kit send path, then keep broader Markdown-to-rich_text conversion scoped in #57708 with conservative fallbacks, docs, and changelog coverage.

## Reproduction Assessment

Yes for the remaining observable gap. Source inspection shows markdown.tables: "block" is still coerced to "code", and ordinary Slack replies then render mrkdwn text chunks instead of native table or rich_text blocks.

## Solution Assessment

No, closing as implemented is not the best current action. Explicit Block Kit pass-through is implemented, but the remaining automatic table/rich-output behavior should be resolved through the active table follow-up and #57708 rather than swept closed.
