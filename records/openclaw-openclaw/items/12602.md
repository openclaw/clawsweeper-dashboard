## Summary

Keep open. Current main supports explicit Slack Block Kit payloads, message-tool blocks, presentation blocks, and opt-in interactive controls, but ordinary markdown-table/rich output still resolves to text/code rendering, so the umbrella request is not fully implemented.

## What This Changes

Request Slack agent messages support richer Block Kit output, including arbitrary blocks, interactive controls, and native table/rich rendering instead of plain mrkdwn text.

## Best Possible Solution

Land or replace the native Slack markdown-table Block Kit send path, then keep broader Markdown-to-rich_text conversion scoped in #57708 with conservative fallbacks, docs, and changelog coverage.

## Reproduction Assessment

Yes. Source inspection of current main shows `channels.slack.markdown.tables: "block"` is schema-valid but `resolveMarkdownTableMode` coerces it to `code`, and `sendMessageSlack` then posts mrkdwn text chunks without blocks.

## Solution Assessment

No, closing as implemented is not the best current action. The explicit Block Kit path is implemented, but the remaining automatic table/rich-output behavior should be resolved through the active table follow-up and #57708 rather than swept closed.
