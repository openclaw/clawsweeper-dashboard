## Summary

Keep open: current main has implemented explicit Slack Block Kit payloads, presentation rendering, and interactive controls, but this broader Slack rich-output umbrella still has unresolved product/API scope around automatic table/rich_text rendering and native-block producer support.

## What This Changes

Request Slack agent responses to support richer Block Kit output such as structured sections, buttons/selects, context, and table-style rendering instead of plain text only.

## Best Possible Solution

Keep this as the Slack rich-output umbrella until maintainers narrow the remaining Slack-owned rendering/API contract; continue using `presentation` for portable controls and document native Slack blocks only if that provider-specific boundary is approved.

## Reproduction Assessment

Yes at the source level: explicit Slack blocks work, but ordinary Markdown replies still resolve table mode without block-renderer capability and flow through text chunk rendering. No live Slack failure was reproduced because this is a feature/product-scope gap, not a broken current contract.

## Solution Assessment

No, the original broad tool/markup/hybrid proposal is not yet the best single implementation shape. The maintainable path is a maintainer-approved Slack plugin/rendering boundary, with portable `presentation` kept as the shared default.
