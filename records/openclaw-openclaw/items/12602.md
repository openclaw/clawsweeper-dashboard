## Summary

Keep open: current main supports explicit Slack Block Kit payloads and interactive reply directives, but ordinary Slack markdown replies still render through text chunks, `markdown.tables: "block"` is still coerced back to code rendering, and the remaining table-send and automatic `rich_text` work is tracked by active follow-ups.

## What This Changes

Request to let Slack agent messages use Block Kit for richer structured responses such as sections, fields, action buttons, selects, context metadata, and better table rendering.

## Best Possible Solution

Keep this as the Slack rich-output umbrella until https://github.com/openclaw/openclaw/pull/57717 or a replacement lands the table send path, and https://github.com/openclaw/openclaw/issues/57708 settles the automatic `rich_text` direction with docs and changelog coverage.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature umbrella. Source inspection gives high confidence that explicit blocks work on current main while table Block Kit delivery and automatic `rich_text` rendering remain incomplete.

## Solution Assessment

No, closing this as implemented is not the best current action. The maintainable path is to review or replace the table-send PR and make the `rich_text` product decisions before closing the umbrella.
