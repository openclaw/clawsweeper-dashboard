## Summary

Keep open: explicit Slack Block Kit blocks and presentation controls are on current main and in the latest release, but ordinary Slack markdown still sends mrkdwn/text chunks, `markdown.tables: "block"` is still coerced to `code`, the table-send PR closed unmerged, and automatic rich_text conversion remains an open product RFC.

## What This Changes

Request to let agents send richer Slack Block Kit output, including explicit blocks, tables, buttons, selects, context metadata, and automatic formatting conversions.

## Best Possible Solution

Keep this umbrella open until maintainers either split the remaining table work to a new canonical item or land a focused replacement, while using https://github.com/openclaw/openclaw/issues/57708 for the broader automatic rich_text/defaulting decision.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature umbrella. Source inspection gives a high-confidence boundary: explicit blocks and presentation controls exist, while automatic rich_text conversion and native markdown-table Block Kit delivery are not wired into current main.

## Solution Assessment

No, closing this as implemented is not the best path. The maintainable direction is to keep the existing explicit-block surfaces, make a maintainer decision on automatic Slack rich output, and use a focused replacement if table-block delivery is still desired after the closed table PR.
