## Summary

Keep open: current main supports explicit Slack Block Kit payloads and interactive directives, but ordinary markdown Slack replies still post mrkdwn text, `markdown.tables: "block"` is still coerced to code, and open follow-ups own the remaining table and automatic `rich_text` work.

## What This Changes

Requested richer Slack agent output through Block Kit payloads, including structured sections, buttons, selects, context blocks, and better table rendering.

## Best Possible Solution

Keep this as the Slack rich-output umbrella until the table send path lands or is replaced, then let https://github.com/openclaw/openclaw/issues/57708 carry the automatic `rich_text` product decision.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature umbrella. Source inspection gives high confidence that explicit blocks work on current main while table Block Kit delivery and automatic `rich_text` output remain incomplete.

## Solution Assessment

No, closing this as implemented is not the best current action. The maintainable path is to review, rebase, or replace https://github.com/openclaw/openclaw/pull/57717 for table delivery and resolve https://github.com/openclaw/openclaw/issues/57708 for automatic rich output.
