## Summary

Keep open: current main covers raw Slack Block Kit delivery, shared presentation rendering inside the Slack adapter, and interactive blocks, but the broader producer/API boundary plus ordinary markdown/table rich rendering remain unresolved and a narrower Slack presentation bug is already tracked separately.

## What This Changes

The issue requests richer Slack Block Kit output from agent messages, including structured sections, controls, native blocks, and table/report-style rendering in Slack.

## Best Possible Solution

Keep this as the Slack rich-output umbrella, land or close the narrower presentation downgrade candidates, then have maintainers decide and document the Slack-native producer boundary versus portable presentation.

## Reproduction Assessment

Yes, at source level. Raw blocks and shared presentation pieces exist, but ordinary Slack markdown replies still render through text chunking and native table/rich producer support remains gated or undecided.

## Solution Assessment

No, the original broad tool/markup/hybrid proposal is not yet the best single fix. The maintainable path is a maintainer-approved Slack plugin producer/rendering boundary that preserves portable presentation as the shared default.
