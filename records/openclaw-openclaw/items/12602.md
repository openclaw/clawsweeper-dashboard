## Summary

Keep open. Current main supports explicit Slack Block Kit payloads, shared presentation blocks, and opt-in interactive controls, but the ordinary markdown-table/rich-output path still falls back to text/code rendering and the active follow-ups #57717 and #57708 remain the right place to resolve that remaining scope.

## What This Changes

Request Slack agent messages to support richer Block Kit output, including structured sections, dividers, buttons, selects, context blocks, and better table rendering.

## Best Possible Solution

Keep this as the Slack rich-output umbrella until maintainers land or explicitly replace the native markdown-table Block Kit send path, then track broader rich_text conversion under #57708 with conservative fallback behavior and docs/changelog updates.

## Reproduction Assessment

Yes. Configure channels.slack.markdown.tables: "block" and send an ordinary Slack markdown table reply without explicit blocks; current source and tests show the resolver returns code mode and the send path posts mrkdwn text chunks instead of native Block Kit table blocks.

## Solution Assessment

No, closing this as implemented is not the best current action. The explicit Block Kit path is implemented, but the remaining automatic table/rich-output behavior should be resolved by reviewing #57717 and scoping #57708 rather than duplicating work.
