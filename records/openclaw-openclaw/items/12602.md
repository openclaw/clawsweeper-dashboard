## Summary

Keep open: current main supports explicit Slack Block Kit payloads, shared presentation controls, and opt-in interactive replies, but the broader rich-output umbrella still has unresolved work because ordinary Slack replies remain on the mrkdwn/text path and `markdown.tables: "block"` is still coerced back to code until a safe send/defaulting design lands.

## What This Changes

The issue requests richer Slack agent output by allowing agents to send Slack Block Kit payloads for structured blocks, buttons, selects, context, and table-style presentation.

## Best Possible Solution

Keep this as the Slack rich-output umbrella until maintainers either land or defer native table/rich_text conversion, with https://github.com/openclaw/openclaw/issues/57708 carrying the detailed Markdown-to-rich_text/defaulting decision and Slack interaction wake work staying separate under https://github.com/openclaw/openclaw/issues/61502.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature umbrella. Source inspection gives a high-confidence current boundary: explicit blocks and presentation controls work, while ordinary replies still render as mrkdwn/text and block table mode is coerced to code.

## Solution Assessment

No, closing this as fully implemented is not the best path. The maintainable route is to keep the umbrella open while maintainers settle native table/rich_text defaulting and explicit-block precedence in the narrower related work.
