## Summary

Keep open: current main supports explicit Slack Block Kit payloads and interactive directives, but ordinary markdown Slack replies still render text, `markdown.tables: "block"` is still coerced to code, and the remaining table/rich-output work is still active in related follow-ups rather than merged on main.

## What This Changes

Add Slack Block Kit support for richer agent messages, including structured payloads, buttons/selects, context blocks, and better table or rich formatting.

## Best Possible Solution
