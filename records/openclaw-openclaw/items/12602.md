## Summary

Keep open: current main and the latest release support explicit Slack blocks and shared presentation controls, but native markdown-table Block Kit delivery and automatic rich_text conversion are still unfinished and already have active follow-up work.

## What This Changes

The issue requests richer Slack agent messages using Block Kit JSON, tables, buttons, selects, context blocks, and fallback text instead of plain text-only replies.

## Best Possible Solution

Keep this umbrella open until the table-send path lands or is replaced, use existing channelData.slack.blocks and presentation payloads for explicit Block Kit today, and let the rich_text RFC carry the broader defaulting decision.

Not applicable as a strict bug reproduction because this is a feature umbrella. Source inspection gives a high-confidence boundary: explicit blocks and presentation controls exist, while markdown-table Block Kit delivery and automatic rich_text conversion remain incomplete.
