## Summary

Keep open: current main and the latest release support explicit Slack Block Kit payloads, shared presentation controls, and opt-in interactive replies, but ordinary Slack replies still render through the mrkdwn/text path and native table/rich_text defaulting remains an unresolved product decision tracked by narrower related work.

## What This Changes

Requested richer Slack Block Kit output from agent messages, including explicit blocks, tables, interactive controls, and fallback text behavior.

## Best Possible Solution

Keep this as the Slack rich-output umbrella until maintainers either land or defer native table/rich_text conversion, with https://github.com/openclaw/openclaw/issues/57708 owning the detailed rendering/defaulting decision and Slack interaction wake behavior staying separate under https://github.com/openclaw/openclaw/issues/61502.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature umbrella. Source inspection gives a high-confidence current boundary: explicit blocks and presentation controls work, while ordinary replies still render as mrkdwn/text and block table mode is coerced to code.

## Solution Assessment

No, closing or auto-implementing this now is not the best path. The maintainable route is maintainer product review for native table/rich_text defaults, streaming/chunking behavior, fallback text, and explicit-block precedence in the narrower related work.
