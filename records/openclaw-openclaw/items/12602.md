## Summary

Keep open. Current main supports explicit Slack Block Kit payloads, message-tool blocks, semantic presentation blocks, and opt-in interactive controls, but the ordinary markdown-table/rich-output path still renders through text/code, and the discussion points to active follow-ups #57717 and #57708 for the remaining scope.

## What This Changes

Request richer Slack agent messages via Block Kit JSON, interactive controls, structured sections, and automatic table/rich rendering instead of plain mrkdwn text.

## Best Possible Solution

Land or replace the native Slack markdown-table Block Kit send path, then keep broader Markdown-to-rich_text conversion scoped in #57708 with conservative fallbacks, docs, and changelog coverage.

## Reproduction Assessment

Yes for the remaining observable gap. Source inspection shows markdown.tables: "block" is schema-valid but coerced to "code", and ordinary Slack replies then go through mrkdwn text chunks rather than native table blocks.

## Solution Assessment

No, closing as implemented is not the best current action. The explicit Block Kit path is implemented, but the remaining automatic table/rich-output behavior should be resolved through the active table follow-up and #57708 rather than swept closed.
