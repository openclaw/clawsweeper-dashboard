## Summary

Keep open. Current main implements explicit Slack Block Kit payloads, shared presentation blocks, and interactive controls, but the ordinary markdown-table Block Kit send path remains unmerged and broader rich_text conversion is still tracked by an open RFC.

## What This Changes

The issue requests richer Slack agent messages using Block Kit features such as structured sections, fields, dividers, buttons, selects, context metadata, and table-style output.

## Best Possible Solution

Keep this as the Slack rich-output umbrella until the table send path lands or is replaced, then route broader automatic rich_text conversion through the focused RFC with conservative fallbacks and docs/changelog coverage.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature umbrella. The remaining gap is source-verifiable: current main still converts ordinary Slack markdown replies to mrkdwn text and still coerces `markdown.tables: "block"` back to code.

## Solution Assessment

No, closing as implemented is not the best current action. Explicit Block Kit and semantic presentation support are implemented, but the remaining table and automatic rich-output work should continue through the active Slack follow-ups rather than being swept closed.
