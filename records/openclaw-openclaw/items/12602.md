## Summary

Keep open. Current main supports explicit Slack Block Kit payloads, shared presentation blocks, and interactive reply directives, but the ordinary markdown-table Block Kit send path, broader automatic rich output, and interaction wake path are still active follow-ups rather than cleanup closure.

## What This Changes

The issue requests richer Slack agent messages using Block Kit JSON, interactive controls, structured sections, context, buttons, selects, and optional automatic markdown-to-rich-output conversion.

## Best Possible Solution

Keep this as the Slack rich-output umbrella until the table send path and interaction wake follow-up are resolved, then route broader automatic rich_text conversion through the focused RFC with docs and changelog coverage.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature umbrella. The remaining gaps are source-verifiable on current main: ordinary Slack replies still render through mrkdwn text, block table mode is coerced back to code, and the related rich-output follow-ups are open.

## Solution Assessment

No, closing as implemented is not the best current action. Explicit Block Kit plumbing is implemented, but the remaining table, rich_text, and interaction-wake slices should continue through their active Slack follow-ups rather than being swept closed.
