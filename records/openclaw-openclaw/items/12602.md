## Summary

Keep open. Current main implements explicit Slack Block Kit payload delivery, message-tool blocks, shared presentation blocks, and opt-in interactive controls, but the ordinary markdown-table/rich-output path still falls back to text/code rendering, so the umbrella request is not fully implemented.

## What This Changes

The issue requests richer Slack agent responses using Block Kit structures such as sections, fields, buttons, selects, context blocks, and native table-style rendering.

## Best Possible Solution

Land or replace the native Slack markdown-table Block Kit send path, then keep broader Markdown-to-rich_text conversion scoped in #57708 with conservative fallbacks, docs, and changelog coverage.

## Reproduction Assessment

Yes. Set channels.slack.markdown.tables to block and send an ordinary Slack markdown table reply without explicit blocks; current source and tests show the resolver coerces block to code and the sender posts mrkdwn text chunks.

## Solution Assessment

No, closing as implemented is not the best current action. The explicit Block Kit path is implemented, but the remaining automatic table/rich-output behavior should be resolved by reviewing #57717 and scoping #57708 rather than duplicating work.
