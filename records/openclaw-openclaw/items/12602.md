## Summary

Keep open. Current main supports explicit Slack Block Kit payloads and opt-in interactive controls, but the ordinary markdown-table/rich-output path is still incomplete because `markdown.tables: "block"` is coerced back to code rendering and Slack sends text chunks when no explicit blocks are supplied.

## What This Changes

The issue requests richer Slack Block Kit support for Slack agent messages, including structured blocks, buttons/selects, context, and automatic richer rendering for common outputs like tables.

## Best Possible Solution

Keep this as the Slack rich-output umbrella until the native markdown-table Block Kit send path lands or is explicitly replaced, then move broader rich_text conversion to #57708 with conservative fallbacks plus docs and changelog updates.

## Reproduction Assessment

Yes. Configure `channels.slack.markdown.tables: "block"` and send an ordinary Slack markdown table reply without explicit blocks; current source and tests show the resolver returns `code` and the send path posts mrkdwn text chunks.

## Solution Assessment

No, closing this as implemented is not the best current action. Explicit Block Kit payloads are implemented, but the remaining automatic table/rich-output behavior should be resolved through #57717 and #57708 rather than hidden by this umbrella cleanup.
