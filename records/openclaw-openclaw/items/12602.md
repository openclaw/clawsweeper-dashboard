## Summary

Keep open: current main and the latest release support explicit Slack Block Kit payloads, presentation controls, and interactive reply blocks, but ordinary Slack replies still render through the mrkdwn/text path and `markdown.tables: "block"` is still coerced back to code rendering. The remaining native table/rich_text defaulting work is product-sensitive and is still tracked by narrower related work.

## What This Changes

The issue requests richer Slack agent output through Block Kit payloads, native table/rich_text rendering, and interactive controls instead of plain mrkdwn-only replies.

## Best Possible Solution

Keep this as the Slack rich-output umbrella until maintainers either land or explicitly defer native table/rich_text conversion in https://github.com/openclaw/openclaw/issues/57708, while keeping Slack interaction wake reliability separate under https://github.com/openclaw/openclaw/issues/61502.

Not applicable as a strict bug reproduction because this is a feature umbrella. Source inspection gives a high-confidence current boundary: explicit blocks and presentation controls work, while ordinary Slack replies still render through mrkdwn/text chunks and block table mode is coerced to code.
