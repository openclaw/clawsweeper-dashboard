## Summary

Keep open: current main and the latest release support explicit Slack Block Kit payloads, message-tool `blocks`, presentation controls, and interactive reply blocks, but ordinary Slack replies still render through the mrkdwn/text path and `markdown.tables: "block"` is still coerced back to code rendering. The remaining native table/rich_text defaulting work is product-sensitive and is still tracked by narrower related work.

## What This Changes

The issue requests richer Slack agent messages using Block Kit JSON, sections, buttons, selects, context blocks, and markdown-to-native Slack formatting rather than only plain text replies.

## Best Possible Solution

Keep this as the Slack rich-output umbrella until maintainers either land or explicitly defer native table/rich_text conversion in https://github.com/openclaw/openclaw/issues/57708, while keeping Slack interaction wake reliability separate from this issue.
