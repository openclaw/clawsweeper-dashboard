## Summary

Request to let agents send richer Slack Block Kit messages for structured summaries, tables, buttons, selects, and other interactive responses instead of plain markdown text.

## Best Possible Solution

Keep this as the Slack rich-output umbrella until maintainers either land the native table/rich_text send path or explicitly move the remaining conversion work to the narrower open Slack rich_text RFC.

## Reproduction Assessment

## Solution Assessment

Unclear. Explicit `blocks` support is already the right narrow slice for raw payloads, but automatic table/rich_text conversion needs maintainer product/API direction before a new implementation PR is safe.
