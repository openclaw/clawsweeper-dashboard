## Summary

Keep open. Current main covers explicit Slack Block Kit payloads and portable presentation controls, but source and history show the ordinary markdown-table/native rich-output path is still incomplete, while the related rich_text RFC remains open and the table-send PR was closed unmerged.

## What This Changes

The issue requests richer Slack Block Kit support for general Slack agent messages instead of plain text-only responses.

## Best Possible Solution

Keep this as the Slack rich-output umbrella until maintainers either land a current-main Slack table/rich-output send path or explicitly move the remaining scope to a narrower canonical issue such as https://github.com/openclaw/openclaw/issues/57708.

## Reproduction Assessment

Yes at source level. Current Slack send can deliver explicit blocks, but ordinary markdown tables still resolve to code/text rendering because `block` table mode is coerced to `code`.

## Solution Assessment

No final solution is on main yet. The shipped explicit-block and presentation paths are useful slices, but automatic table or rich_text conversion still needs a maintainer-approved design and implementation.
