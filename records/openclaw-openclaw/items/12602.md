## Summary

Keep open: current main supports explicit Slack Block Kit payloads, message-tool blocks, and opt-in interactive directives, but the ordinary Slack markdown reply path still posts mrkdwn/text chunks, table-block delivery is still in the open https://github.com/openclaw/openclaw/pull/57717 follow-up, and broader automatic rich output remains in https://github.com/openclaw/openclaw/issues/57708.

## What This Changes

The issue requests richer Slack agent output through Block Kit payloads, including structured blocks, buttons, selects, context blocks, and table-style rendering.

## Best Possible Solution

Keep this umbrella open until the native table send path lands or is intentionally replaced, then let https://github.com/openclaw/openclaw/issues/57708 carry the broader automatic Slack rich-output decision with docs, regression coverage, and redacted real Slack proof.

## Solution Assessment
