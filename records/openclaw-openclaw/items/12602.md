## Summary

Keep this feature umbrella open for maintainer follow-up: authored Block Kit payloads and interactive controls are shipped, but ordinary agent markdown still follows the mrkdwn text-chunk path, and the remaining automatic rich-output direction is under active product discussion in https://github.com/openclaw/openclaw/issues/57708.

## What This Changes

Add richer Slack agent output using Block Kit, including authored blocks, interactive controls, and automatic structured rendering for content such as tables.

## Best Possible Solution

Keep the shipped explicit-block and typed interactive-action paths, then define one Slack-owned automatic rendering pipeline in the narrower RFC that converts only supported structures, preserves accessible text fallbacks, and avoids duplicating portable presentation semantics.

## Reproduction Assessment

Yes at source level for the remaining gap: explicit blocks use Slack's structured send branch, while ordinary agent markdown is rendered into mrkdwn text chunks. This establishes the missing automatic rich-rendering path without claiming a live Slack failure.

## Solution Assessment

No, unrestricted raw Block Kit JSON alone is not the best complete solution; typed portable presentation/actions plus Slack-owned rendering and fallback behavior better match the repository's channel boundary and avoid competing message semantics.
