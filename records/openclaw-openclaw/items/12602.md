## Summary

Keep open. Current main supports explicit Slack Block Kit payloads and interactive directives, but the broader rich Slack output request is still incomplete: ordinary replies still render through mrkdwn text chunks, `markdown.tables: "block"` is still coerced back to code, and the table-send PR plus rich_text RFC remain active follow-ups.

## What This Changes

The issue requests richer Slack agent messages through Block Kit payloads, table rendering, interactive controls, and possible automatic Markdown-to-Block-Kit conversion.

## Best Possible Solution

Keep this as the Slack rich-output umbrella until https://github.com/openclaw/openclaw/pull/57717 or a replacement lands the table send path, and https://github.com/openclaw/openclaw/issues/57708 resolves the automatic rich_text direction with docs and changelog coverage.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature umbrella. Source inspection gives high confidence that explicit blocks work on current main while table Block Kit delivery and automatic rich_text rendering are not complete.

## Solution Assessment

No, closing this as implemented is not the best current action. The maintainable path is to review or replace the table-send PR and make the rich_text product decisions before closing the umbrella.
