## Summary

Keep open: current main already supports explicit Slack Block Kit payloads, message-tool blocks, presentation blocks, and interactive reply controls, but ordinary Slack markdown replies still go through mrkdwn text chunks and native table/rich_text rendering remains owned by active focused follow-ups.

## What This Changes

The issue requests richer Slack agent output by allowing agents to send Slack Block Kit blocks and related structured/interactivity surfaces instead of plain markdown text.

## Best Possible Solution

Keep this umbrella open until the native Slack table send path lands or is replaced, then let https://github.com/openclaw/openclaw/issues/57708 carry the broader automatic rich_text conversion decision with docs, tests, and redacted Slack proof.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is a feature umbrella. Current-main source inspection gives a high-confidence verification path: explicit blocks and interactive directives work, while native table delivery and automatic rich_text conversion are not complete on main.

## Solution Assessment
