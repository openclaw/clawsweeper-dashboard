## Summary

Keep this open: current main still implements inbound audio as transcription/media-understanding text, and the only related implementation candidate, #48900, remains open and unmerged with provider-contract and raw-media egress decisions still unresolved.

## What This Changes

Requests an option to pass inbound raw audio to native multimodal models alongside or instead of STT transcripts.

## Best Possible Solution

Keep this issue as the product tracker and resolve it through #48900 or a successor that adds a typed prompt-media contract, capability-gated provider adapters, explicit transcript-only versus audio-plus-transcript config, size/privacy limits, docs, and regression coverage.

## Reproduction Assessment

Yes. Source and regression tests give a high-confidence path: a successfully transcribed inbound audio attachment is stripped from prompt media notes, and the embedded agent prompt contract has no raw audio payload field.

## Solution Assessment

Unclear for the open implementation candidate because it is unmerged and depends on provider content-block mapping outside this code path. The maintainable solution should be typed and capability-gated rather than tunneling audio through an image-shaped prompt path.
