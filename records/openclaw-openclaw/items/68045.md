## Summary

Keep open. The reported leak is valid on current main, but the latest PR head still has blocking sanitizer/content-integrity issues and lacks inspectable real behavior proof from the contributor's setup.

## What This Changes

The PR exports shared chat text post-processing, routes grouped Control UI message text through it with empty-group suppression, extends inbound metadata stripping for system/internal lines, and adds focused tests plus changelog entries.

## Best Possible Solution

Revise and rebase the PR so grouped rendering shares user/assistant cleanup while preserving runtime-context stripping, normalizer directive/media handling, all-empty group suppression, and system-event filtering that matches current timestamp and continuation formats.

## Reproduction Assessment

Yes, source-level. Current main still bypasses shared assistant/user extraction in grouped bubbles, lacks all-empty group suppression, and does not strip System/System (untrusted) or internal instruction lines while session-system-events emits those prefixes.

## Solution Assessment

No. The PR targets the right bug boundary, but the current head is not the narrow maintainable fix until it preserves current runtime-context stripping, normalizer directive/media cleanup, and system-event matching for current timestamp and continuation shapes.

Full review comments:

- **[P1] Preserve runtime-context stripping in processMessageText:** `ui/src/ui/chat/message-extract.ts:10-17`
  - body: Current main strips internal runtime-context blocks before display cleanup, but this exported `processMessageText` has no `stripInternalRuntimeContext` import or call. Rebasing this version would make those internal blocks visible through the shared Control UI extraction path.
  - confidence: 0.88
- **[P1] Match current system-event timestamps:** `src/auto-reply/reply/strip-inbound-meta.ts:35-36`
  - body: `SYSTEM_EVENT_LINE_RE` only matches weekday-prefixed timestamps such as `[Wed 2026-...]`, but `session-system-events.ts` formats current timestamps as `2026-...T...Z` or `2026-... ... TZ`. The new filter therefore misses the emitted `System:` lines it is meant to remove.
  - confidence: 0.92
- **[P2] Strip system-event continuation lines:** `src/auto-reply/reply/strip-inbound-meta.ts:35-36`
  - body: `drainFormattedSystemEventBlock` prefixes every subline with `System:` or `System (untrusted):`, but only the first subline carries a timestamp. This regex drops only timestamped first lines, leaving continuation fragments like `System (untrusted): <output>` visible.
  - confidence: 0.91
- **[P1] Keep normalized assistant directive cleanup:** `ui/src/ui/chat/grouped-render.ts:1114`
  - body: Using `extractTextCached(message)` as the grouped markdown source bypasses `normalizeMessage`'s reply/media/audio directive parsing while attachments and reply state still come from `normalizeMessage`.   - confidence: 0.89
