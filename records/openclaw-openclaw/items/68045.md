## Summary

Keep open: current main still has the reported UI/sanitizer gaps, but this PR is not ready because the latest patch can reintroduce assistant directive tokens, leaves multiline system-event continuations visible, and needs a rebase that preserves current internal-context stripping plus changelog credit.

## What This Changes

This PR exports chat text post-processing, changes grouped Control UI rendering and empty-group suppression, extends inbound metadata stripping for system-event/internal-instruction lines including the TUI path, and adds tests plus changelog entries.

## Best Possible Solution

Revise and rebase this PR so grouped rendering shares user/assistant stripping while preserving normalizeMessage directive/media cleanup, strips multiline injected system-event preludes safely across Control UI and TUI, preserves current internal-runtime-context stripping, and lands with focused regression tests plus compliant changelog attribution.

## Reproduction Assessment

Yes. The grouped-render regression is reproducible with an assistant message containing [[reply_to_current]], MEDIA:, and [[audio_as_voice]]; the sanitizer leak is reproducible with a multiline drainFormattedSystemEvents payload whose continuation line lacks a timestamp.

## Solution Assessment

No. The PR targets the right product boundary, but the current patch is not the best solution because it trades one class of leaked scaffolding for directive/media regressions and leaves part of the reported system-event leak unresolved.
