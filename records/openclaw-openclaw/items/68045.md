Overall confidence: 0.91

Full review comments:

- **[P1] Preserve normalized assistant directive cleanup:** `ui/src/ui/chat/grouped-render.ts:1105-1112`
  - body: Using `extractTextCached(message)` as the grouped bubble markdown source bypasses `normalizeMessage`'s assistant directive/media cleanup. Assistant messages containing `[[reply_to_current]]`, `[[audio_as_voice]]`, or `MEDIA:` lines can render those control tokens in the bubble while the structured reply pill or media attachment is also rendered.
  - confidence: 0.94
- **[P2] Strip system-event continuation lines:** `src/auto-reply/reply/strip-inbound-meta.ts:36-37`
  - body: `SYSTEM_EVENT_LINE_RE` only matches prefixed lines that include the timestamp, but `drainFormattedSystemEvents` prefixes every subline and adds the timestamp only to subline index 0. Multiline events therefore leave continuation lines like `System (untrusted): <output>` visible in user-facing text.
  - confidence: 0.92
- **[P2] Preserve internal runtime context stripping:** `ui/src/ui/chat/message-extract.ts:11-17`
  - body: Current main strips internal runtime context before role-specific display cleanup, and tests assert that user text behavior. The PR branch is based on an older extractor, so the rebase must keep `stripInternalRuntimeContext` or internal context blocks can reappear in displayed text.
  - confidence: 0.88
