## Summary

Keep open. The PR addresses a real current-main leak class, but the head is dirty, lacks inspectable real behavior proof, and still has sanitizer/UI regressions that would leave or reintroduce user-visible internal scaffolding.

## What This Changes

The PR exports shared chat text post-processing, changes Control UI grouped rendering and empty-group suppression, extends inbound metadata stripping for system-event/internal-instruction lines, and adds tests plus changelog entries.

## Best Possible Solution

Rebase and revise the PR so grouped rendering shares user/assistant cleanup while preserving normalizer directive/media handling, runtime-context stripping, all-empty group suppression, and system-event filtering that matches current emitted timestamps and continuations.

## Reproduction Assessment

Yes. Source inspection on current main shows grouped bubbles still bypass shared assistant/user text extraction, renderMessageGroup has no all-empty group precheck, and stripInboundMetadata does not recognize System/System (untrusted) or internal instruction lines while session-system-events still emits those prefixes.

## Solution Assessment

No. The PR targets the right bug boundary, but the current head is not the narrow maintainable fix until it is rebased and preserves current runtime-context stripping, assistant directive/media cleanup, and multiline/current-format system-event scrubbing.
