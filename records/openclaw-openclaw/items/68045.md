## Summary

Keep open. Current main still has the underlying Control UI and inbound metadata leak surfaces, but this PR is not ready to merge because the sanitizer patch no longer matches current main’s system-event timestamp format, still leaves continuation lines visible, can reintroduce assistant directive text in grouped bubbles, and needs changelog credit.

## What This Changes

The PR exports chat text post-processing, changes grouped Control UI rendering and empty-group suppression, extends inbound metadata stripping for system-event/internal-instruction lines including the TUI leading stripper, and adds focused tests plus changelog entries.

## Best Possible Solution

Revise the PR against current main so grouped bubbles combine shared assistant/user stripping with `normalizeMessage` directive/media cleanup, all-empty groups are suppressed after the same visibility rules the renderer uses, both inbound stripping variants remove only actual injected system-event/internal-instruction preludes including continuations, and the user-facing changelog entries include valid contributor credit.

## Reproduction Assessment

Yes. The system-event leak can be reproduced by feeding the PR sanitizer a current-main `drainFormattedSystemEvents` block such as `System (untrusted): [2026-04-17 06:11:17 UTC] ...` plus a non-timestamped continuation; the grouped-render regression can be reproduced with the existing inline `[[reply_to_current]]`/`MEDIA:` grouped-render fixture. I did not run tests because this was a read-only review.

## Solution Assessment

No. The PR targets the right product boundary, but the current patch is not the best fix because it uses stale timestamp assumptions, leaves multiline event continuations behind, and trades the normalized assistant directive/media cleanup path for raw shared extraction in grouped bubbles.
