## Summary

Keep open. The PR addresses a real gateway shutdown reliability gap and the refreshed head fixes the earlier late-rejection pattern, but it still leaves `agent-harnesses` unbounded and marks an unrelated issue as fixed in the changelog.

## What This Changes

This PR adds a 5s gateway shutdown timeout helper, applies it to several non-HTTP teardown steps, adds timeout tests, and adds an Unreleased changelog entry.

## Best Possible Solution

Land the bounded shutdown timeout approach after adding the same coverage to agent-harness disposal and correcting the changelog reference while preserving current HTTP/WebSocket close semantics.

## Reproduction Assessment

Yes for the reviewable defect: on PR head, mock `disposeRegisteredAgentHarnesses()` or a registered harness `dispose()` to never settle, advance fake timers, and shutdown remains blocked at `agent-harnesses` because that step has no `timeoutMs`. The production Telegram stall itself was not live-reproduced.

## Solution Assessment

No as-is. The helper and late-rejection observer are a maintainable direction, but the current application is incomplete for async plugin-owned agent harness cleanup and the changelog points at the wrong tracker.
