## Summary

Keep open: current main still lacks the requested per-subsystem close-handler timeout, and the refreshed patch is directionally sound, but the changelog references unrelated #43851 and exact-head CI is still red.

## What This Changes

The PR adds a gateway shutdown timeout helper, applies 5 s timeouts to non-HTTP close-handler subsystem stops, adds tests, and adds a changelog entry.

## Best Possible Solution

Land the bounded non-HTTP shutdown timeout implementation after correcting the release-note reference and validating the exact head, while keeping HTTP listener force-close semantics separate.

## Reproduction Assessment

Yes for the reviewable behavior: unit fakes can make a close-handler subsystem hang, advance fake timers, and verify later cleanup plus late-rejection observation. The production Telegram polling stall was not live-reproduced in this review.

## Solution Assessment

Mostly yes. Extending the existing shutdown-step warning model for non-HTTP teardown is a narrow maintainable fix, and preserving the dedicated HTTP close path avoids the earlier restart binding regression; the changelog reference should be corrected before merge.
