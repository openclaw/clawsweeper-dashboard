## What This Changes

The branch adds a `shutdown-timeout` helper, wraps selected non-HTTP gateway teardown steps with 5s timeouts, adds regression tests, and records a changelog entry.

## Best Possible Solution

Land the bounded non-HTTP shutdown timeout if maintainers accept the 5s subsystem budget, keeping the existing HTTP listener force-close path separate.

## Reproduction Assessment

Yes for source-level reproduction: current main serially awaits non-HTTP teardown calls without per-subsystem timeouts, while `run-loop.ts` only provides a 25s outer force-exit backstop. A non-settling stop promise can therefore block later cleanup until that outer budget is consumed.

## Solution Assessment

Yes. The PR is a narrow maintainable fix because it bounds the existing shutdown-step path for non-HTTP subsystems, observes late promise rejections, and preserves the specialized WebSocket and HTTP close semantics.
