## Summary

## What This Changes

## Best Possible Solution

Port the per-subsystem timeout design onto current main while letting channel manager stop finalization remain authoritative, preserve current restart/session drains and HTTP close semantics, and merge only after redacted real shutdown proof.

## Solution Assessment

No, not as currently proposed. Per-subsystem timeouts are the right direction, but the channel-stop wrapper needs to respect the channel manager's own timeout/final-state contract and the branch needs rebase plus real proof.
