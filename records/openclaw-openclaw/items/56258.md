## What This Changes

This PR adds a 5s `raceTimeout` helper, applies it to several non-HTTP gateway close-handler teardown steps, adds focused shutdown timeout tests, and adds a changelog entry.

## Best Possible Solution

Land the bounded shutdown timeout approach after applying the same timeout coverage to async plugin-owned agent harness disposal and correcting the release-note issue reference while preserving current HTTP force-close semantics.

## Reproduction Assessment

Yes for the reviewable defect: mock `disposeRegisteredAgentHarnesses()` or a registered harness `dispose()` to never settle, advance fake timers, and the PR branch will remain blocked at `agent-harnesses` because that step has no `timeoutMs`. The production Telegram polling stall itself was not live-reproduced here.

## Solution Assessment

No as-is. The timeout helper and late-rejection handling are a maintainable pattern, but the current application is incomplete for an async plugin-owned teardown step and the changelog metadata needs correction.
