## Summary

Keep open. The PR is directionally useful and the refreshed head preserves HTTP/WebSocket shutdown semantics, but it still leaves agent-harness disposal unbounded and the changelog marks an unrelated response-watchdog issue as fixed.

## What This Changes

This PR adds a 5s gateway shutdown timeout helper, applies it to several non-HTTP teardown steps, adds timeout tests, and adds an Unreleased changelog entry.

## Best Possible Solution

Land the bounded shutdown timeout approach after applying the same timeout coverage to agent-harness disposal and correcting the changelog reference, while preserving current HTTP listener force-close behavior.

## Reproduction Assessment

Yes for the reviewable defect: on PR head, mock `disposeRegisteredAgentHarnesses()` or a registered harness `dispose()` to never settle, advance fake timers past `SUBSYSTEM_STOP_TIMEOUT_MS`, and shutdown remains blocked at `agent-harnesses`. The production Telegram stall itself was not live-reproduced.

## Solution Assessment

No as-is. The helper and late-rejection observer are a maintainable direction, but the timeout application is incomplete for async plugin-owned agent harness cleanup and the changelog points to the wrong tracker.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

Apply `SUBSYSTEM_STOP_TIMEOUT_MS` here and cover it with a fake-timer test.
