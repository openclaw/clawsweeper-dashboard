## What This Changes

The issue requests native macOS TUI clipboard image paste that inserts inline `[imgN]` tokens and expands them into media attachments on submit.

## Best Possible Solution

Have maintainers choose the TUI media-paste UX and platform policy, then implement the selected path in the TUI editor/submit/backend boundary with focused tests and real macOS paste proof.

## Reproduction Assessment

Yes for current-main source behavior: the TUI editor has no Ctrl+V image branch, and the submit/backend path sends only text. It is not a strict bug reproduction because native TUI clipboard-image paste is not an existing documented contract.

## Solution Assessment

Unclear: the marker approach fits existing pi-tui and downstream media parsing, but maintainers should choose between inline text markers and structured attachments, and between macOS-only and cross-platform support, before implementation.
