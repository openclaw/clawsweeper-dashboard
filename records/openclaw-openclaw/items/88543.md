## Summary

Keep open: current main and v2026.6.9 still lack native TUI clipboard-image paste or an inline token-to-attachment handoff, while the remaining implementation shape needs maintainer UX, privacy, and platform-scope decisions.

## What This Changes

The issue requests native macOS TUI clipboard image paste that inserts inline `[imgN]` tokens and submits the pasted images with the message.

## Best Possible Solution

Keep this open and design a bounded core TUI image-paste flow that uses pi-tui insertion for visible tokens but sends images through the existing structured attachment/media pipeline with explicit privacy and platform behavior.

## Reproduction Assessment

Yes, source-level: current main has no `Ctrl+V` image branch in `CustomEditor` and the TUI send contract still passes only message text. I did not run a live macOS clipboard proof because the missing code path is clear from source.

## Solution Assessment

No for the exact reference shape: the editor hook is plausible, but appending absolute host paths as prose is weaker than using the existing structured `chat.send` attachment and media claim-check pipeline. Maintainers should choose the UX and privacy boundary before implementation.
