## Risks / Open Questions

- The proposed `[imgN]` plus `[media attached: <absolute path>]` shape could expose local media paths in prompt text, history, logs, or transcripts unless maintainers explicitly accept and bound that behavior.
- A macOS-first `pngpaste` path would create platform-specific operator expectations; maintainers still need to choose whether this lands as macOS-only, cross-platform, or behind a documented dependency/fallback policy.
- There are two plausible transport shapes, inline text markers versus structured `chat.send` attachments, and choosing the wrong owner boundary could create later drift between Gateway, embedded TUI, and chat attachment handling.
