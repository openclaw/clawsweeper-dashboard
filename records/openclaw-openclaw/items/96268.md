## Summary

Keep this PR open: current main and the latest release still contain the web_fetch entity-decoding bug, while this branch is the canonical open fix with no blocking code, security, or proof findings from this review.

## What This Changes

The PR routes web_fetch HTML entity decoding through the shared decoder, tightens that decoder for web_fetch’s prior entity contract, and adds regression tests for astral, escaped-ampersand, uppercase, malformed, and BMP entities.

## Best Possible Solution

Land this PR or an equivalent shared-decoder fix after normal maintainer review, because it removes the duplicated web_fetch decoder and preserves the prior entity contracts with focused tests.

## Reproduction Assessment

Yes. Current main and v2026.6.10 still show the `&amp;`-first replacement chain and `String.fromCharCode` numeric decoding path, and the PR body provides before/after output through exported `htmlToMarkdown`.

## Solution Assessment

Yes. Reusing `decodeHtmlEntityAt` with a single left-to-right pass is the narrow maintainable fix; a local `fromCodePoint` helper or replacement-chain reorder would leave duplicate decoder behavior in place.
