## Summary

Keep open: current main and the latest release still requeue Telegram spooled handler errors unconditionally, and the linked closing PR is still open rather than merged.

## What This Changes

The issue reports that Telegram slash-command spooled updates can retry MissingAgentHarnessError forever and block later inbound messages.

## Best Possible Solution

Review and land or replace the linked fix PR with a supported missing-harness error classifier, focused regression tests, and proof that the poison update is dead-lettered while later Telegram updates drain.

Yes, source-reproducible: current main requeues every Telegram spooled bot.handleUpdate failure and existing tests expect that retry behavior. I did not run a live Telegram repro in this read-only pass.

## Solution Assessment

Yes for the narrow spool-classification direction: use the existing failed tombstone path for non-retryable missing-harness errors while preserving transient retries. The open linked PR is the right workstream to review or replace before closing this issue.
