## What This Changes

The PR adds a module-level in-memory per-session chat draft cache and save/restore helpers wired into several Control UI session and agent switch paths.

## Best Possible Solution

Land one rebased Control UI change that preserves drafts in the shared session-switch lifecycle, scopes or clears cached drafts on context/deletion, keeps queued-message behavior intact, and adds focused tests plus real behavior proof.

## Reproduction Assessment

Yes, at source level. On current main, setting state.chatMessage and switching sessions through switchChatSession reaches resetChatStateForSessionSwitch(), which clears the composer with state.chatMessage = "".

## Solution Assessment

No, not as submitted. The save/restore direction is reasonable, but the branch should be rebased onto current main's centralized switch helper, cover the visible sidebar selection path, and clear or scope the cache before merge.

Status: needs_attention
