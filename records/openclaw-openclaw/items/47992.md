## Summary

Keep this PR open for now: current main still drops Control UI composer drafts on session switches, and the related #54655 was closed unmerged, but this branch is not ready as posted because it misses a visible switch path and has draft-isolation, cleanup, test, and changelog gaps.

## What This Changes

The PR adds an in-memory Control UI chat composer draft cache and wires save/restore helpers into selected session and agent switch callbacks.

## Best Possible Solution

Ship one rebased or replacement Control UI change that preserves composer drafts through the centralized session-switch helper, scopes draft storage to the active gateway/session context, evicts empty and deleted-session entries, preserves existing queue behavior, and adds focused tests plus the required changelog entry.

## Reproduction Assessment

Yes. On current main, set a non-empty `state.chatMessage` and switch sessions through `switchChatSession()`; `resetChatStateForSessionSwitch()` clears the draft before loading the destination session.

## Solution Assessment

No, not as posted. An in-memory cache is a reasonable narrow approach, but it needs to be centralized, workspace-scoped, deletion-aware, tested, and documented in the changelog before merge.
