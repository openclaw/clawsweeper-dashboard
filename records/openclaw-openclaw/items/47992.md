## Summary

Keep open. Current main still clears the Control UI chat composer on session switches, and this PR targets a real UX gap, but the branch is incomplete against the current session-switch path and lacks required after-fix real behavior proof.

## What This Changes

The PR adds an in-memory per-session chat draft cache and wires save/restore calls into Control UI session and agent switching callbacks.

## Best Possible Solution

Ship one rebased Control UI change centered in switchChatSession()/resetChatStateForSessionSwitch(), with per-session draft restore, deleted-session cleanup, and focused regression coverage while preserving existing queue behavior.

## Reproduction Assessment

Yes, by source inspection. Current main sends visible session and agent switches through switchChatSession(), and that helper still clears state.chatMessage instead of restoring a per-session composer draft.

## Solution Assessment

No, not as this branch currently stands. The save/restore direction is reasonable, but the branch misses the chat sidebar selection path and needs to be rebased around the centralized helper with deletion cleanup and tests.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve drafts for sidebar session selections:** `ui/src/ui/app-render.ts:1438`
  - body: The PR wires save/restore into several callbacks, but the PR head still switches chat-sidebar selections with state.setSessionKey(key) directly. Picking a session from the sidebar can therefore carry the outgoing draft into the destination session instead of restoring that session's draft.
  - confidence: 0.88

  - body: chatDraftCache is module-level state, but the patch adds no clear helper or deletion hook. Drafts for removed sessions would stay in memory until the page reloads, so the session deletion flow should evict those keys.
  - confidence: 0.78
