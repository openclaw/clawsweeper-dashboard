## Summary

Keep this PR open: the user problem is still present on current main and the PR is useful, but the submitted branch is incomplete against a visible session-selection path, lacks real behavior proof, and needs a scoped draft-cache cleanup before merge.

## What This Changes

The PR adds an in-memory per-session Control UI chat draft cache and wires save/restore calls into several session and agent switch paths.

## Best Possible Solution

Ship one rebased Control UI change that preserves composer drafts in the shared session-switch helper, clears or scopes cached drafts on deletion/context changes, and adds focused session/agent/sidebar switch coverage.

## Reproduction Assessment

Yes, at source level. On current main, setting state.chatMessage and calling switchChatSession() reaches resetChatStateForSessionSwitch(), which clears the composer draft with state.chatMessage = "".

## Solution Assessment

No, not as submitted. The save/restore direction is reasonable, but the branch should be rebased onto the centralized switch helper, cover the sidebar selection path, and add cache cleanup/scoping before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Preserve drafts from sidebar session selections:** `ui/src/ui/app-render.ts:1438`
  - body: The PR head still switches sessions in `onSessionSelect` with `state.setSessionKey(key)` and no draft save/restore call. Picking a session from the chat sidebar can carry the outgoing draft into the destination session and skip restoring the destination draft, so the visible feature remains incomplete.
  - confidence: 0.86
- **[P3] Clear deleted sessions from the draft cache:** `ui/src/ui/app-render.helpers.ts:37`
  - body: `chatDraftCache` is module-level state with no eviction hook for deleted sessions. Drafts for removed sessions would stay in memory until page reload; expose a clear helper and call it after successful session deletion to keep the cache bounded.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: No supply-chain or code-execution risk was found, but the proposed local draft cache needs scoping or cleanup to avoid restoring stale draft text in the wrong UI context.

Concerns:

- **[low] Scope cached drafts to the active UI context:** `ui/src/ui/app-render.helpers.ts:37`
  - body: The PR stores unsent text in a module-level Map keyed only by sessionKey. In a long-lived browser tab, reused keys such as `main` after reconnecting to another gateway or workspace could restore prior draft text in the wrong context unless the cache is scoped or cleared.
  - confidence: 0.7
