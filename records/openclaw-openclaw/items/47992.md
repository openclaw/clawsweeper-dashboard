## Summary

Keep open: current main still clears the Control UI composer draft during session switches, while this PR remains incomplete, lacks required real behavior proof, and needs rework onto the current centralized switch path.

## What This Changes

The PR adds an in-memory Map-backed per-session composer draft cache and wires save/restore helpers into several Control UI session and agent switching paths.

## Best Possible Solution

Ship a rebased, focused Control UI implementation that preserves composer drafts in the shared session-switch reset path, scopes or clears cached drafts on deletion/context changes, keeps queued-message behavior intact, and adds focused draft restoration coverage.

## Reproduction Assessment

Yes, at source level. On current main, setting state.chatMessage and calling switchChatSession reaches resetChatStateForSessionSwitch, which assigns state.chatMessage = "".

## Solution Assessment

No, not as submitted. The save/restore direction is reasonable, but the branch should be rebased onto the centralized switch helper, cover sidebar selection, and add cache cleanup or context scoping before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve drafts through the sidebar session picker:** `ui/src/ui/app-render.ts:1438`
  - body: The PR head still switches sessions in onSessionSelect with state.setSessionKey(key) and no save/restore call. Picking a session from the chat sidebar can leave the outgoing draft attached to the new session and skip restoring the destination draft, so the visible feature remains incomplete.
  - confidence: 0.86
- **[P3] Clear draft cache entries for deleted sessions:** `ui/src/ui/app-render.helpers.ts:37`
  - body: chatDraftCache is module-level state with no eviction hook for deleted sessions. Drafts for removed sessions would remain in memory until page reload; expose a clear helper and call it after successful session deletion to keep the cache bounded.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: No supply-chain risk was found, but the module-level session-key-only draft cache has a low-severity local draft isolation and retention concern.

Concerns:

- **[low] Scope cached drafts to the active UI context:** `ui/src/ui/app-render.helpers.ts:37`
  - body: The PR stores unsent text in a module-level Map keyed only by session key. In a long-lived browser tab, reused keys such as main after reconnecting to another gateway or workspace could restore prior draft text in the wrong context unless the cache is scoped or cleared.
  - confidence: 0.7
