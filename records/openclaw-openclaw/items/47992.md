## Summary

Keep this PR open. Current main still discards the Control UI composer draft during session switches, but the posted branch is dirty against current main and needs a rebased or replacement implementation before merge because it misses one visible switch path and keys draft state too broadly.

## What This Changes

The PR adds an in-memory Control UI chat draft cache plus save/restore helpers and wires them into selected session and agent switching callbacks.

## Best Possible Solution

Ship one rebased or replacement Control UI change that preserves composer drafts in the centralized session-switch helper, scopes draft storage to the active gateway/session context, evicts empty and deleted-session entries, preserves existing per-session queue behavior, and adds focused tests plus the required changelog entry.

## Reproduction Assessment

Yes. On current main, type a draft in the Control UI composer and switch sessions through a path that calls switchChatSession(); resetChatStateForSessionSwitch() clears state.chatMessage before the original draft can be restored.

## Solution Assessment

No, not as posted. The in-memory cache is a reasonable narrow boundary, but the branch should be rebased onto the centralized helper and adjusted for gateway scoping, sidebar coverage, deletion cleanup, tests, and changelog policy.
