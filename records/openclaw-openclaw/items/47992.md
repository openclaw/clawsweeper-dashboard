## Summary

Keep this PR open: the draft-loss behavior is still present on current main, but this branch is stale/dirty, lacks required real Control UI proof, and needs cache scoping/cleanup before it is a safe merge path.

## What This Changes

Adds an in-memory per-session Control UI chat draft cache and wires save/restore around session and agent switching paths.

## Best Possible Solution

Ship one rebased Control UI change that preserves composer drafts in the shared session-switch lifecycle, scopes or clears cached drafts on context/deletion changes, keeps queued-message behavior intact, and adds focused tests plus real UI proof.

## Reproduction Assessment

Yes, at source level. On current main, switchChatSession calls resetChatStateForSessionSwitch(), which assigns state.chatMessage = "" and drops the unsent composer draft.

## Solution Assessment

No, not as submitted. The save/restore direction is reasonable, but the branch needs a rebase onto the shared switch helper, draft cache scoping/cleanup, tests, and real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve drafts from the sidebar session picker:** `ui/src/ui/app-render.ts:1438`
  - body: The PR head still switches sessions in onSessionSelect with state.setSessionKey(key) and no save/restore call. Picking a session from the chat sidebar can leave the outgoing draft attached to the wrong visible session and does not restore the destination draft; current main has since centralized this path through switchChatSession, so the branch should rebase or cover that shared path.
  - confidence: 0.84
- **[P3] Clear cached drafts when sessions are deleted:** `ui/src/ui/app-render.helpers.ts:37`
  - body: chatDraftCache is module-level state with no eviction hook for deleted sessions. Drafts for removed sessions would remain in memory until page reload; expose a clear helper and call it after successful session deletion to keep the cache bounded.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: No supply-chain risk was found, but the session-key-only draft cache has a low-severity local draft isolation and retention concern.

Concerns:

- **[low] Scope cached drafts to the active UI context:** `ui/src/ui/app-render.helpers.ts:37`
  - body: The PR stores unsent text in a module-level Map keyed only by session key. In a long-lived browser tab, reused keys such as main after reconnecting to another gateway or workspace could restore prior draft text in the wrong context unless the cache is scoped or cleared.
  - confidence: 0.72
