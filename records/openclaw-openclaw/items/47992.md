## Summary

Keep this PR open: current main still loses the composer draft on session switch, but this branch is conflicting, misses one visible switch path on its own head, lacks real behavior proof, and needs a scoped cache cleanup before merge.

## What This Changes

The PR adds an in-memory per-session chat draft cache plus save/restore calls for several Control UI session and agent switch paths.

## Best Possible Solution

Ship one rebased Control UI change that preserves composer drafts in the centralized session-switch helper, scopes or clears cached drafts on deletion/context changes, keeps queue behavior intact, and adds focused session/agent/sidebar switch coverage.

## Reproduction Assessment

Yes, at source level. On current main, setting `state.chatMessage` and calling `switchChatSession()` reaches `resetChatStateForSessionSwitch()`, which clears the draft with `state.chatMessage = ""`.

## Solution Assessment

No, not as submitted. The save/restore direction is reasonable, but the branch should be rebased onto the centralized switch helper, cover the sidebar selection path, and add cache cleanup/scoping before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.87

Full review comments:

- **[P2] Preserve drafts from sidebar session selections:** `ui/src/ui/app-render.ts:1438`
  - body: The PR head still switches sessions in `onSessionSelect` with `state.setSessionKey(key)` and no draft save/restore call. Picking a session from the chat sidebar can carry the outgoing draft into the destination session and skip restoring the destination draft, so the visible feature remains incomplete on this branch.
  - confidence: 0.86
- **[P3] Clear deleted sessions from the draft cache:** `ui/src/ui/app-render.helpers.ts:37`
  - body: `chatDraftCache` is module-level state with no eviction hook for deleted sessions. Drafts for removed sessions would stay in memory until page reload; expose a clear helper and call it after successful session deletion to keep the cache bounded.
  - confidence: 0.78

## Security Review

Status: needs_attention
