## Summary

Keep open: current main still drops the composer draft on session switches, but this branch is not merge-ready because it is stale against the centralized switch path, misses a visible switch entrypoint on the PR head, and lacks real behavior proof.

## What This Changes

The branch adds a module-level Map-backed draft cache and save/restore calls around several Control UI session and agent switch callbacks.

## Best Possible Solution

Ship one rebased Control UI change centered in the shared session-switch helper that preserves drafts per session with context scoping/eviction, covers every session/agent switch entrypoint, and adds focused tests plus real UI proof.

## Reproduction Assessment

Yes, at source level. On current main, switchChatSession reaches resetChatStateForSessionSwitch, which assigns state.chatMessage = "" before loading the destination session.

## Solution Assessment

No, not as submitted. The save/restore direction is reasonable, but the branch should be rebased onto the centralized switch helper, cover sidebar session selection, and add cache scoping or cleanup before merge.
