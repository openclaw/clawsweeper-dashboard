## Summary

Keep open: current main still clears the composer draft during session switches, and this PR is useful but incomplete and stale against the newer centralized switch helper.

## What This Changes

The PR adds an in-memory per-session chat draft cache and wires save/restore calls into several Control UI session and agent switch handlers.

## Best Possible Solution

Implement draft preservation once in the current centralized session-switch helper, clear cached drafts after successful session deletion, preserve existing queue behavior, and add focused regression tests.

## Reproduction Assessment

Yes, from source inspection. Current main routes session changes through `switchChatSession()`, which calls a reset helper that assigns `state.chatMessage = ""`, and the PR head leaves one visible sidebar selection path uncovered.

## Solution Assessment

No. The direction is right, but the PR should be rebased onto current main's centralized switch path and include delete-cache cleanup instead of patching older scattered callbacks only.
