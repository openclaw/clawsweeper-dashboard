## Summary

Keep open: current main still clears the composer draft during session switches, but this PR is not merge-ready because it lacks real behavior proof and leaves visible switch/cache cleanup gaps.

## What This Changes

The branch adds an in-memory session-keyed chat draft cache and save/restore calls around Control UI session and agent switching.

## Best Possible Solution

Land one focused Control UI implementation in the shared session-switch path with scoped draft storage, deletion cleanup, and regression coverage for session, agent, and sidebar switches.

## Reproduction Assessment

Yes. By source inspection, setting state.chatMessage and calling switchChatSession() on current main reaches resetChatStateForSessionSwitch(), which clears the draft instead of restoring one per session.

## Solution Assessment

No, not as written. The save/restore direction is reasonable, but the branch needs a rebase around the current centralized helper, sidebar coverage, cache cleanup/scoping, tests, and real behavior proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Preserve drafts through sidebar session selection:** `ui/src/ui/app-render.ts:1438`
  - body: The PR head still switches chat-sidebar selections with `state.setSessionKey(key)` directly. Selecting a session from the sidebar can carry the outgoing draft into the destination session instead of restoring that session's saved draft.
  - confidence: 0.88
- **[P3] Clear deleted sessions from the draft cache:** `ui/src/ui/app-render.helpers.ts:37-50`
  - body: `chatDraftCache` is module-level state, but the patch adds no eviction hook for removed sessions. Drafts for deleted sessions would stay in memory until page reload, so the deletion flow should clear those keys.
  - confidence: 0.78

## Security Review

Status: needs_attention

Summary: Needs attention: no supply-chain or code-execution risk was found, but the new in-memory draft cache needs scoping/cleanup to avoid local draft leakage.

Concerns:

- **[low] Scope cached drafts to the active UI context:** `ui/src/ui/app-render.helpers.ts:37`
  - body: The PR stores unsent text in a module-level Map keyed only by sessionKey. In a long-lived browser tab, reused keys such as `main` after reconnecting or changing gateway/workspace could restore a prior draft in the wrong context unless the cache is scoped or cleared.
  - confidence: 0.7
