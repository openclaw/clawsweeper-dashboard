## Summary

Keep open. Current main still treats Claude CLI as a supported Anthropic backend, but the official Docker image does not install Claude Code and the Docker/provider/backend docs still do not give a Docker-specific persistent binary plus `~/.claude` credential recipe.

## What This Changes

Request to make Docker Claude CLI reuse survive container upgrades by either preinstalling Claude Code in the official image or documenting a persistent install and credential path.

## Best Possible Solution

Add Docker-specific Claude CLI documentation using the existing home-volume/extra-mount and `cliBackends.claude-cli.command` surfaces, while leaving official image preinstallation as a separate maintainer decision.

## Reproduction Assessment

Not applicable as a failing runtime reproduction because this is a feature/docs gap. The gap is high-confidence by source inspection: current main requires a host `claude` binary and documents only host-oriented setup, while Docker docs do not cover persistent Claude CLI install or credentials.

## Solution Assessment

Yes for a docs-only Option B repair: it is the narrowest maintainable path and uses existing Docker mount plus CLI backend command override surfaces. Preinstalling Claude Code in the official image is not the best automated cleanup path because it changes release artifact policy and supply-chain scope.
