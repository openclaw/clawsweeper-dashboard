## Summary

Keep this open. Current main and the latest shipped release still support the Anthropic Claude CLI backend as an external `claude` command, but the official Docker image does not ship Claude Code and the Docker/provider/backend docs still lack a Docker-specific persistent binary plus `~/.claude` credential recipe.

## What This Changes

The issue requests official Docker support for Claude CLI reuse by either baking Claude Code into the image or documenting a persistent Docker install and credential setup.

## Best Possible Solution

Add Docker-specific Claude CLI persistence docs and leave image preinstallation as a separate maintainer decision.

## Reproduction Assessment

Not applicable as a failing runtime reproduction because this is a Docker setup documentation gap. Source and docs inspection gives high confidence that current main and v2026.5.7 expect an external `claude` binary while lacking Docker-specific persistence guidance.

## Solution Assessment

Yes for a docs-only Option B repair: it uses existing `OPENCLAW_HOME_VOLUME` / `OPENCLAW_EXTRA_MOUNTS` and `agents.defaults.cliBackends.claude-cli.command` surfaces without changing the release image supply chain. Preinstalling Claude Code should not be the automated cleanup path.
