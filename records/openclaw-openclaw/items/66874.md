## Summary

Keep this open. Current main still supports the Anthropic Claude CLI backend, but the official Docker image and release workflow do not install Claude Code, and the Docker/provider/backend docs still do not give a Docker-specific persistent install plus `~/.claude` credential recipe.

## What This Changes

The issue asks OpenClaw to either preinstall Claude Code in the official Docker image or document a Docker-persistent Claude CLI install and credentials path.

## Best Possible Solution

Add Docker-specific Claude CLI persistence docs using existing mount and command-override surfaces, while leaving image preinstallation as a separate maintainer decision.

## Reproduction Assessment

Not applicable as a failing runtime reproduction because this is a Docker setup documentation gap. The gap is high-confidence by source inspection: current main expects an external `claude` binary, while Docker docs do not explain a persistent binary or `~/.claude` credential path.

## Solution Assessment

Yes for a docs-only Option B repair: it is the narrowest maintainable solution and uses existing Docker mount plus `agents.defaults.cliBackends.claude-cli.command` surfaces. Preinstalling Claude Code is not the best automated cleanup path because it changes release artifact and supply-chain policy.
