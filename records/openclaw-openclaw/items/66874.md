## Summary

Keep this open: current main still treats Claude Code as an external `claude` command, the official Docker image path does not install it, and Docker docs still lack a Claude-specific persistent install plus `~/.claude` credential recipe. A docs-only Option B repair remains the narrow safe path; preinstalling Claude Code in the image is a separate release artifact and supply-chain decision.

## What This Changes

The issue requests either preinstalling the Claude Code CLI in the official Docker image or documenting a persistent Docker install and credential path for the Anthropic Claude CLI backend.

## Best Possible Solution

Add Docker-specific Claude CLI guidance for persistent binary storage, Claude Code credential persistence, command override config, onboarding verification, and API-key fallback; leave image preinstallation for an explicit maintainer release-artifact decision.

## Reproduction Assessment

Not applicable as a failing runtime bug. Current-main source and docs inspection does verify the setup gap: `claude-cli` still expects an external `claude` command and Docker docs do not provide a Claude-specific persistent install and credential recipe.

## Solution Assessment

Yes for a docs-only Option B repair. It addresses the actionable setup gap through existing Docker mounts and `agents.defaults.cliBackends.claude-cli.command` without changing official image contents or package supply-chain ownership.
