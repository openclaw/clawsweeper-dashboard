## Summary

Keep open. Current main still supports the Anthropic Claude CLI backend, but the official Docker image/release path does not install Claude Code and the Docker/provider/backend docs still lack a Docker-specific persistent install and credential recipe; related #2440 is adjacent auth-detection history, not the canonical tracker for this Docker gap.

## What This Changes

Request official Docker support for Claude CLI reuse by either preinstalling Claude Code or documenting persistent Docker install and credential storage.

## Best Possible Solution

Ship a docs-only Docker Claude CLI recipe using the existing command override and persistence knobs, while leaving official-image preinstallation as a separate maintainer decision.

## Reproduction Assessment

Not applicable as a failing runtime reproduction because this is a feature/docs gap. The gap is high-confidence by inspection: current main documents Claude CLI as an installed/logged-in host dependency while the official Docker image and Docker docs do not provide a persistent Docker path.

## Solution Assessment

Yes for a docs-only Option B repair: it is the narrowest maintainable solution and uses existing Docker mount, npm-prefix, and cliBackends command surfaces. Automatic image preinstallation is not the best automated cleanup path because it changes official release supply chain and policy.

## Repair Work Prompt

Fix the Docker Claude CLI docs gap for https://github.com/openclaw/openclaw/issues/66874 with a docs-only PR. Add Docker-specific guidance for using the Anthropic `claude-cli` backend without changing Dockerfile, release workflow, dependency pins, or package installs in the official image. Cover: persistent Claude Code installation under a mounted npm prefix or home volume; persistence of Claude Code OAuth state under `$HOME/.claude`; an explicit `agents.defaults.cliBackends.claude-cli.command` example plus `agents.defaults.agentRuntime.id: "claude-cli"`; onboarding/verification commands; and Anthropic API-key fallback/rollback guidance. Prefer `docs/install/docker.md` as the main Docker recipe and link or mirror only the necessary short notes from `docs/providers/anthropic.md`, `docs/gateway/cli-backends.md`, and `docs/gateway/authentication.md`. Include a concise `CHANGELOG.md` fix entry for the user-facing docs issue, omitting bot/maintainer thanks if no human contributor attribution is appropriate. Do not preinstall `@anthropic-ai/claude-code`, do not alter Docker release workflows, and do not add new runtime APIs.
