## Summary

Keep this open: current main still treats Claude Code as an external `claude` command and neither the official Docker image path nor the Docker/provider/backend docs provide a Docker-specific persistent Claude Code install and credential recipe.

## What This Changes

Request to either pre-install Claude Code in the official Docker image or document a persistent Docker install, credential, and command path for the Anthropic Claude CLI backend.

## Best Possible Solution

Add Docker-specific Claude CLI docs for persistent binary/npm-prefix storage, `~/.claude` credential persistence, explicit `claude-cli.command`, onboarding verification, and API-key fallback; leave image preinstallation as a separate release-artifact decision.

## Reproduction Assessment

Not applicable as a failing runtime bug; source and docs inspection show the supported Claude CLI backend still requires an external `claude` binary while Docker-specific persistence guidance is missing.

## Solution Assessment

Yes for a docs-only Option B fix: it uses existing Docker mounts and `agents.defaults.cliBackends.claude-cli.command` without changing official image contents or package supply-chain ownership.
