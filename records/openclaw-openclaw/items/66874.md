## Summary

Keep this open. Current main still supports the Anthropic Claude CLI backend, but the official Docker image and Docker release workflow do not install Claude Code, and the Docker/provider/backend docs still lack Docker-specific persistent install plus `~/.claude` credential guidance; the adjacent closed credential issue does not track this Docker provisioning gap.

## What This Changes

The issue requests either preinstalling Claude Code in the official Docker image or documenting a persistent Docker install and credential path for the Anthropic Claude CLI backend.

## Best Possible Solution

Add Docker-specific Claude CLI guidance for persistent installation, `~/.claude` credential persistence, `cliBackends.claude-cli.command`, onboarding verification, and API-key fallback, while leaving official image preinstallation to a separate maintainer decision.

## Reproduction Assessment

Not applicable as a failing runtime reproduction because this is a documentation/setup gap. The gap is high-confidence by source inspection: current main requires an external `claude` binary and host-oriented Claude CLI setup, while Docker docs do not cover persistent binary or credential storage.

## Solution Assessment

Yes for a docs-only Option B repair: it is the narrowest maintainable path and uses existing Docker mount plus CLI backend command override surfaces. Preinstalling Claude Code in the official image is not the best automated cleanup path because it changes release artifact policy and supply-chain scope.
