## Summary

Keep open. Current main still supports the Anthropic Claude CLI backend, but the official Docker image and Docker release workflow do not install Claude Code, and the Docker/provider/backend docs still lack Docker-specific persistent binary plus `~/.claude` credential guidance. The earlier ClawSweeper keep-open comments remain accurate.

## What This Changes

The issue requests that Docker users either get Claude Code preinstalled in the official image or receive documented persistent install and credential guidance for the `claude-cli` backend.

## Best Possible Solution

Add Docker-specific Claude CLI guidance for a persisted install path, `~/.claude` credential persistence, `cliBackends.claude-cli.command`, onboarding verification, and API-key fallback; leave official image preinstallation to a separate maintainer product/security decision.

## Reproduction Assessment

Not applicable as a failing runtime reproduction because this is a feature/docs gap. The gap is high-confidence by source inspection: current main requires a host `claude` binary and documents only host-oriented setup, while Docker docs do not cover persistent Claude CLI install or credentials.

## Solution Assessment

Yes for a docs-only Option B repair: it is the narrowest maintainable path and uses existing Docker mount plus CLI backend command override surfaces. Preinstalling Claude Code in the official image is not the best automated cleanup path because it changes release artifact policy and supply-chain scope.
