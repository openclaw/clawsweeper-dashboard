## Summary

Keep open. Current main still supports Claude CLI as a first-class Anthropic backend, but the official Docker image and Docker release workflow still do not install Claude Code, and the user-facing Docker/provider/backend docs still lack a persistent Docker install plus `~/.claude` credential recipe. The adjacent closed credential-detection issue is useful history but does not supersede this Docker image/docs gap.

## What This Changes

The issue requests that the official Docker image either include Claude Code CLI or document a persistent Docker install and credential path for the Claude CLI backend.

## Best Possible Solution

Add Docker-specific Claude CLI guidance covering a persistent binary install location, `~/.claude` credential persistence, explicit `agents.defaults.cliBackends.claude-cli.command` plus `agentRuntime.id: "claude-cli"` config, onboarding verification, and Anthropic API-key fallback. Keep official-image preinstallation as a separate maintainer decision.

## Reproduction Assessment

Yes for the reported image/docs gap by static inspection, though this is not a failing runtime bug. Current main lacks a Dockerfile or release-workflow Claude Code install, and the relevant docs still require a host-local logged-in Claude CLI without Docker persistence steps.

## Solution Assessment

Yes, a docs-only Option B repair is the narrowest maintainable next step. Preinstalling Claude Code may still be a product choice later, but it should not be folded into automated cleanup because it changes the official release artifact and supply-chain surface.
