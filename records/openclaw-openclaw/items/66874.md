## Summary

Keep this open. Current main still expects an external `claude` command for the Anthropic CLI backend, while the Docker image/release path does not install Claude Code and the user-facing Docker docs still lack a Claude-specific persistent binary plus `~/.claude` credential recipe.

## What This Changes

Request to either preinstall Claude Code in the official Docker image or document a persistent Docker install, command override, and `~/.claude` credential path.

## Best Possible Solution

Add Docker-specific Claude CLI guidance for persistent installation, `~/.claude` credential persistence, command override config, onboarding verification, and API-key fallback; leave image preinstallation as a separate release-artifact decision.

## Reproduction Assessment

Not applicable as a failing runtime reproduction because this is a Docker setup documentation and image-provisioning request. Source and docs inspection gives high confidence that OpenClaw expects an external `claude` binary while lacking Docker-specific persistence guidance.

## Solution Assessment

Yes for a docs-only Option B repair: it uses existing Docker mount and `agents.defaults.cliBackends.claude-cli.command` surfaces without changing the official image supply chain. Preinstalling Claude Code is a broader product and supply-chain decision.
