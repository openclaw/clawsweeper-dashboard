## Summary

Keep this open. Current main and the latest shipped release still treat the Anthropic Claude CLI backend as an external `claude` command, while the official Docker image does not install Claude Code and the Docker/provider/backend docs still do not give Docker users a persistent binary plus `~/.claude` credential recipe.

## What This Changes

Request that the official Docker image either pre-install the Claude Code CLI or document a persistent Docker install and credential path for the `claude-cli` backend.

## Best Possible Solution

Add Docker-specific Claude CLI persistence documentation and leave official-image preinstallation as a separate maintainer product and supply-chain decision.

## Reproduction Assessment

Not applicable as a failing runtime reproduction because this is a Docker setup documentation and image-provisioning request. Source, docs, tests, and latest-release inspection give high confidence that OpenClaw expects an external `claude` binary while lacking Docker-specific persistence guidance.

## Solution Assessment

Yes for a docs-only Option B repair: it uses the existing Docker volume/mount and `agents.defaults.cliBackends.claude-cli.command` surfaces without changing the release image supply chain. Preinstalling Claude Code is not the narrow automated cleanup path.
