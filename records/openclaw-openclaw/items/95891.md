## Best Possible Solution

Add a provider-owned opencode-go MiniMax Anthropic Messages thinking guard or parser wrapper in the opencode-go plugin, with focused `minimax-m3` regression coverage and no core/channel policy change unless a plugin SDK seam is truly missing.
