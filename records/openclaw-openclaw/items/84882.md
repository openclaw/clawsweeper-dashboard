Keep open. ## What This Changes

The issue reports memory-core Dreaming silently deleting date-only daily memory files under `memory/YYYY-MM-DD.md` during the `normalized recall artifacts before dreaming` flow.

## Best Possible Solution

Identify the concrete delete/truncate path, then add a regression that runs Dreaming with `separate` storage and proves seeded `memory/YYYY-MM-DD.md` files remain intact while dangling recall sources are skipped or repaired safely.

## Reproduction Assessment

No. The report has detailed production and backup evidence, but I did not establish a deterministic current-main reproduction; source inspection shows the named normalization step is likely correlated, not proven as the direct daily-file deleter.

## Solution Assessment

Unclear. The suggested guard against daily-file mutation is directionally useful, but the best fix first needs to locate the actual delete/truncate path and then protect that path with focused preservation and audit coverage.
