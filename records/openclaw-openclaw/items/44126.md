## Summary

Keep open. Current main still lacks the proposed agent-callable thinking-level tool, so the PR is not obsolete, but the current branch is not safe to merge because it breaks the provider stream-wrapper thinking contract and adds adaptive-thinking guidance to minimal prompts.

## What This Changes

This PR adds a `set_thinking_level` agent tool, adaptive-thinking prompt guidance, mutable run/session thinking state plumbing, provider wrapper resolver changes, and regression tests.

## Best Possible Solution

Port the useful idea onto the current provider-profile architecture: validate through `resolveThinkingProfile`, preserve run/session state semantics, keep provider hook contracts coherent, and omit adaptive guidance from minimal prompt mode; broader automatic strategy work should stay coordinated with #71350.

## Reproduction Assessment

Yes for the review findings. A minimal `buildAgentSystemPrompt` call with `set_thinking_level` available would include the new Adaptive Thinking section, and current provider wrappers such as Z.AI and DeepSeek treat `ctx.thinkingLevel` as a concrete string.

## Solution Assessment

No. The direction is plausible, but this PR is not the narrowest maintainable solution until provider-wrapper thinking resolution and minimal-prompt gating are corrected against current main.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Resolve thinking before provider hooks:** `src/plugins/types.ts:429-431`
  - body: This widens `ProviderWrapStreamFnContext.thinkingLevel` to accept a function, but existing bundled provider helpers still treat `ctx.thinkingLevel` as a string. Z.AI, DeepSeek, OpenRouter/DeepSeek, and shared SDK wrappers compare or normalize the value directly, so passing a resolver can make `set_thinking_level` report success while later provider payloads keep the wrong thinking state. Keep the plugin context concrete or adapt every wrapper to resolve lazily at payload time.
  - confidence: 0.9
- **[P2] Skip adaptive guidance in minimal prompts:** `src/agents/system-prompt.ts:168-169`
  - body: `buildAdaptiveThinkingSection` only checks whether `set_thinking_level` is available, so subagent/cron minimal prompts also get this long policy block. Minimal mode exists to keep those prompts compact; gate this section on non-minimal prompt mode before adding it to the prompt.
  - confidence: 0.86
