## Summary

## What This Changes

The PR adds a `set_thinking_level` agent tool, adaptive-thinking prompt guidance, provider-aware thinking resolution, run/session thinking-state plumbing, and regression tests.

## Best Possible Solution

Port the useful tool onto the current provider-profile architecture: keep provider hook contracts coherent, validate through `resolveThinkingProfile`, preserve live run/session state semantics, skip adaptive guidance in minimal prompts, add the changelog entry, and coordinate broader strategy work with #71350.

## Reproduction Assessment

Yes. Source inspection reproduces the review findings: the PR passes a lazy thinking value through a hook whose existing consumers compare strings, and its prompt section lacks the minimal-mode guard used by adjacent long sections.

## Solution Assessment

No. The direction is plausible, but this patch is not the best mergeable form until provider-wrapper thinking resolution and minimal-prompt gating are corrected against current main and the required changelog entry is added.
