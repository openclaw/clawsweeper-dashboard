## Summary

Keep open. Current main has provider-profile thinking support and `/think` session controls, but it does not implement this PR's `set_thinking_level` agent tool. The PR still targets a real agent-control gap, but the latest diff is not mergeable as-is because live thinking updates do not propagate correctly through plugin-owned stream wrappers and the prompt guidance regresses minimal prompt mode.

## What This Changes

This PR adds a `set_thinking_level` agent tool, adaptive-thinking prompt guidance, provider-aware requested/effective thinking resolution, mid-run thinking state plumbing, and regression tests for tool availability and provider wrapper behavior.

## Best Possible Solution

Preserve the idea, but port it onto the current provider thinking-profile architecture: the tool should validate against `resolveThinkingProfile`/provider-owned levels, update run and session state consistently, pass live thinking changes through every plugin stream hook that consumes thinking, and keep adaptive prompt guidance out of minimal prompt mode.

## Reproduction Assessment

Yes for the review findings. A minimal `buildAgentSystemPrompt` call with `set_thinking_level` available would include the new long Adaptive Thinking section, and a provider plugin hook that reads `ctx.thinkingLevel` once or compares it to string literals will not observe later resolver changes in the same run.

## Solution Assessment

No, not as currently proposed. The PR's direction is plausible, but the implementation needs a rebase and a narrower integration with current provider-profile and plugin-wrapper contracts before it is the maintainable fix.
