## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep provider thinking hooks concrete:** `src/plugins/types.ts:436`
  - body: This widens `ProviderWrapStreamFnContext.thinkingLevel` to a resolver, but provider hooks still resolve or compare it as if it were a concrete value. For example, the Moonshot plugin computes `thinkingType` once while wrapping, so later `set_thinking_level` calls can report success while provider payloads keep the stale startup level. Keep this hook concrete or update every provider hook/helper to resolve lazily at payload time.
  - confidence: 0.9
- **[P2] Gate adaptive guidance on minimal mode:** `src/agents/system-prompt.ts:171-175`
  - body: `buildAdaptiveThinkingSection` only checks tool availability, so minimal subagent/cron prompts still get the new multi-line policy block. Those prompt modes intentionally skip adjacent long sections to stay compact, so pass `isMinimal` into this section and return nothing for minimal prompts.
  - confidence: 0.86
