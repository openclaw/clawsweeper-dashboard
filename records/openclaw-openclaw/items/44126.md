## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Keep provider thinking hooks concrete or fully lazy:** `src/plugins/types.ts:436`
  - body: This widens `ProviderWrapStreamFnContext.thinkingLevel` to a resolver, but bundled provider hooks can still collapse it to a one-time value. For example, the Moonshot plugin computes `thinkingType` before wrapping, so later `set_thinking_level` calls can report success while provider payloads keep the stale startup level; either keep the hook concrete or update provider hooks/helpers to resolve lazily at payload time.
  - confidence: 0.9
- **[P2] Gate adaptive guidance on minimal mode:** `src/agents/system-prompt.ts:171-175`
  - body: `buildAdaptiveThinkingSection` only checks tool availability, so minimal subagent/cron prompts still get the new multi-line policy block. Those modes intentionally skip adjacent long sections to stay compact, so pass `isMinimal` into this section and return nothing for minimal prompts.
  - confidence: 0.86
