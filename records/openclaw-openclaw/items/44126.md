## Summary

Keep open. Current main still lacks the proposed agent-callable thinking-level tool, so the PR is not obsolete, but the current branch is not safe to merge because it widens the provider stream-wrapper thinking contract without updating all string-oriented consumers and it injects a long adaptive-thinking policy into minimal prompts.

## What This Changes

This PR adds a `set_thinking_level` agent tool, adaptive-thinking prompt guidance, run/session thinking state plumbing, provider wrapper resolver changes, and regression tests.

## Best Possible Solution

Port the useful idea onto the current provider-profile architecture: keep tool validation aligned with `resolveThinkingProfile`, preserve run/session state semantics, pass live thinking changes through provider wrappers without breaking plugin contracts, and keep adaptive prompt guidance out of minimal prompt mode. Broader automatic strategy work should remain coordinated with #71350.

## Reproduction Assessment

Yes for the review findings. A minimal `buildAgentSystemPrompt` call with `set_thinking_level` available would include the new Adaptive Thinking section, and current provider wrappers such as DeepSeek/Bedrock compare `ctx.thinkingLevel` as a concrete string, so a function resolver changes their behavior.

## Solution Assessment

No. The direction is plausible, but the current PR is not the narrowest maintainable solution until the provider-wrapper contract and minimal-prompt behavior are corrected against current main.

- **[P2] Resolve thinking before provider hooks:** `src/plugins/types.ts:429-431`
  - body: This widens `ProviderWrapStreamFnContext.thinkingLevel` to accept a function, but existing bundled provider helpers still treat `ctx.thinkingLevel` as a string.
