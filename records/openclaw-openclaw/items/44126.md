- **[P2] Resolve thinking before provider hooks:** `src/plugins/types.ts:429-431`
  - body: This widens `ProviderWrapStreamFnContext.thinkingLevel` to accept a function, but existing bundled and SDK provider wrappers still treat it as a concrete string. Providers such as Z.AI and shared stream-family wrappers can keep the wrong payload state after `set_thinking_level` reports success, so keep the hook context concrete or update every wrapper/helper to resolve lazily at payload time.
  - confidence: 0.9

  - body: `buildAdaptiveThinkingSection` is emitted whenever `set_thinking_level` is available, but minimal prompts are used for subagent/cron sessions to stay compact and adjacent long sections skip them with `isMinimal`. Gate this section on non-minimal mode so lean prompts do not gain the new multi-line policy block.
  - confidence: 0.86
