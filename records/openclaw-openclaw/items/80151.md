## Summary

Keep open: the status projection is useful, but the latest head still drops the legacy success signal for terminal suppressed no-visible payloads, so main-session pending final delivery can remain uncleared.

## What This Changes

This PR adds an optional `deliveryStatus` object to agent delivery results, CLI/gateway JSON output, docs, and tests while retaining `deliverySucceeded` as the legacy compatibility field.

## Best Possible Solution

Keep the `sendDurableMessageBatch`-based status projection, but make terminal suppressed no-visible outcomes clear pending final delivery via the legacy success signal or an equivalent cleanup path.

## Reproduction Assessment

Yes. A delivered main-session turn that produces only a silent payload such as `NO_REPLY` records `pendingFinalDelivery`, then the PR normalizes to an empty outbound plan and returns without the legacy success bit needed by the cleanup gate.

## Solution Assessment

Mostly yes, but not mergeable as-is. Projecting durable send status is the right boundary; the suppressed no-visible branch should preserve the old terminal-success cleanup contract.
