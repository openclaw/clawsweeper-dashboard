## Summary

Keep open: the direction matches the durable-send contract and the PR has real behavior proof, but the no-visible-payload path can leave main-session pending final delivery uncleared for terminal suppressed replies.

## What This Changes

The PR adds a documented `deliveryStatus` object to `deliverAgentCommandResult` and `openclaw agent --json --deliver`, maps durable send outcomes into that object, and adds focused delivery tests plus a changelog entry.

## Best Possible Solution

Keep the `sendDurableMessageBatch` projection, but preserve legacy retry cleanup for terminal suppressed outcomes before maintainer approval of the JSON contract.

## Reproduction Assessment

Yes, source-reproducible. Current main emits JSON before delivery and lacks `deliveryStatus`; the blocking PR edge case is traceable from `NO_REPLY` normalization to the pending-final-delivery cleanup check that requires `deliverySucceeded === true`.

## Solution Assessment

Mostly yes, but not yet mergeable. Projecting `sendDurableMessageBatch` is the right boundary; the no-visible-payload branch should also preserve the legacy success signal for terminal suppressed delivery cleanup.
