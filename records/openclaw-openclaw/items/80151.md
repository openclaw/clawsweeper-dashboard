## Summary

Keep open: the PR is on the right durable-delivery boundary and has useful proof, but the current head still leaves a legacy pending-final-delivery cleanup path broken for terminal no-visible replies.

## What This Changes

The PR adds a documented `deliveryStatus` object to agent delivery results and JSON output, maps durable send outcomes into it, and teaches subagent announcement delivery to treat failed agent delivery status as failure evidence.

## Best Possible Solution

Keep the durable status projection, but preserve the legacy terminal-success signal or explicitly clear pending final delivery for suppressed no-visible outcomes.

## Reproduction Assessment

Yes, source-reproducible. A main-session delivered turn with a final silent payload stores `pendingFinalDelivery`, then the PR normalizes to an empty outbound plan and returns without `deliverySucceeded`, so the existing cleanup gate cannot clear the marker.

## Solution Assessment

Mostly yes, but not mergeable as-is. Projecting `sendDurableMessageBatch` is the maintainable boundary; the suppressed no-visible branch should also preserve the old cleanup contract.
