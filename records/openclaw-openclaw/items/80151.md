## Summary

Keep open: the durable delivery status projection is useful, but the latest head still drops the legacy success signal for terminal silent delivery, so a main-session `NO_REPLY` final payload can leave `pendingFinalDelivery` uncleared.

## What This Changes

This PR adds an optional durable `deliveryStatus` object to agent delivery results and CLI/gateway JSON output, updates related docs/changelog/generated baseline, and adds focused delivery and subagent announcement tests.

## Best Possible Solution

Keep the `sendDurableMessageBatch`-based status projection, but make terminal suppressed no-visible outcomes clear pending final delivery via `deliverySucceeded: true` or an equivalent explicit cleanup path.

## Reproduction Assessment

Yes, source-reproducible. A delivered main-session turn with a raw silent payload records `pendingFinalDelivery`, then the PR normalizes it to an empty delivery plan and returns without the legacy success bit that the cleanup gate requires.

## Solution Assessment

Mostly yes, but not mergeable as-is. Projecting durable send status is the right boundary; the silent suppressed branch needs to preserve the existing cleanup contract.
