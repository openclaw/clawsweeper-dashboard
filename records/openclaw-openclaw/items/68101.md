## Summary

Keep open: current main still swallows Lobster workflow-file resolution errors and falls back to inline pipeline execution, while https://github.com/openclaw/openclaw/pull/68106 remains open and unmerged as the closing implementation candidate.

## What This Changes

The issue asks Lobster to treat relative workflow-file-like `pipeline` inputs as workflow files or clear path errors instead of reparsing them as inline pipeline commands.

## Best Possible Solution

Review, refresh, and land https://github.com/openclaw/openclaw/pull/68106, or replace it with an equivalent focused Lobster runner fix plus regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows a missing relative `.lobster` path reaches `resolveWorkflowFile()`, has the thrown path error caught as `null`, and then falls through to inline pipeline execution; I did not run a live repro in the read-only checkout.

## Solution Assessment

Yes. A narrow `extensions/lobster/src/lobster-runner.ts` fix that rethrows workflow-like path resolution errors, with focused tests preserving normal inline pipelines, is the maintainable solution.
