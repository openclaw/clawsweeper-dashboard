## Summary

Keep open: current main still swallows Lobster workflow-file resolution errors and falls back to inline pipeline execution, and https://github.com/openclaw/openclaw/pull/68106 remains the open closing implementation candidate.

## What This Changes

The issue requests that Lobster treat relative workflow-file-like `pipeline` inputs as workflow paths or clear path errors instead of falling back to inline pipeline parsing.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/68106 with a focused Lobster runner/test fix that preserves normal inline pipelines while surfacing workflow path errors.

## Reproduction Assessment

Yes. Source inspection on current main shows a missing relative `.lobster` pipeline reaches `resolveWorkflowFile()`, has the thrown file/path error caught as `null`, and then falls through to inline pipeline execution.

## Solution Assessment

Yes. A narrow Lobster plugin runner change plus focused regression coverage is the maintainable fix, and https://github.com/openclaw/openclaw/pull/68106 already owns that implementation candidate.
