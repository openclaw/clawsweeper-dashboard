## Summary

Keep open: current main and the latest release still swallow Lobster workflow-file resolution failures and then run the original file-looking string as an inline pipeline. ## What This Changes

The issue asks Lobster to treat unresolved relative workflow-file paths as workflow file/path errors instead of falling back to inline pipeline parsing.
