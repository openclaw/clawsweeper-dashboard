## Summary

Keep open: current main and latest v2026.5.18 still swallow Lobster workflow-file resolution errors for missing relative workflow paths and fall through to inline pipeline execution; the open closing PR remains unmerged.

## What This Changes

Make the Lobster tool treat relative workflow file paths as workflow files or path errors instead of reparsing them as inline pipelines.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/68106 with a focused runner change that rethrows unresolved workflow-file-like paths and adds regression coverage while preserving inline pipelines.

## Reproduction Assessment

Yes. Source inspection on current main shows a missing relative workflow path reaches `resolveWorkflowFile()`, has its error swallowed as `null`, and then falls through to inline pipeline execution.

## Solution Assessment

Yes as a code direction. A narrow Lobster runner fix that rethrows workflow-file-like path resolution errors, with tests preserving normal inline pipelines, is the maintainable solution.
