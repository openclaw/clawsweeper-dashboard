## What This Changes

The issue requests that relative Lobster workflow-file paths be resolved as workflow files or reported as clear path errors instead of being reinterpreted as inline pipeline commands.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/68106, or an equivalent focused Lobster plugin fix, with regression coverage for missing relative workflow files and unchanged normal inline pipeline behavior.

Yes. Source inspection shows a run request with `pipeline: "lobster/missing-workflow.lobster"` reaches `resolveWorkflowFile()`, has the stat/path error caught as `null`, and then falls through to inline pipeline execution.

## Solution Assessment
