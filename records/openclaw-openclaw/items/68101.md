## Summary

Keep open: current main still swallows Lobster workflow-file resolution errors and falls back to inline pipeline execution, and the open closing implementation candidate has not merged.

## What This Changes

The issue asks Lobster to treat relative workflow-file-looking `pipeline` values as workflow files or clear path errors instead of falling back to inline pipeline parsing.

## Best Possible Solution

Land or replace the focused fix from https://github.com/openclaw/openclaw/pull/68106 in the Lobster runner with regression coverage that preserves normal inline pipelines.

## Reproduction Assessment

Yes. Source inspection on current main shows a missing relative workflow path reaches `resolveWorkflowFile()`, has the thrown path error caught as `null`, and then falls through to inline pipeline execution.

## Solution Assessment

Yes as a code direction. A narrow Lobster plugin runner fix that rethrows workflow-file-like path resolution errors, with tests preserving inline pipelines, is the maintainable solution; the issue should stay open until that PR or an equivalent fix merges.
