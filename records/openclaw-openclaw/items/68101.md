## What This Changes

The issue reports that Lobster treats an unresolved relative workflow file path as an inline pipeline command and returns a misleading unknown-command error.

## Best Possible Solution

Land or replace the linked Lobster plugin fix so workflow-looking paths surface resolution errors directly while normal inline pipelines continue to run unchanged.

## Reproduction Assessment

Yes, as source-reproducible rather than live-reproduced: current main resolves candidate workflow files, catches any stat/path error, returns null, and then executes the same string as a pipeline. I did not run a live tool invocation because this was a read-only checkout review.

## Solution Assessment

Yes, the best fix belongs in the Lobster runner path: preserve workflow-like resolution failures instead of silently falling back, with regression coverage for missing relative files and unchanged inline pipeline behavior.
