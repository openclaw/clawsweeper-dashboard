## Summary

Keep open: current main still catches Lobster workflow-file resolution errors and falls back to inline pipeline execution, and the live closing implementation candidate remains open rather than merged.

## What This Changes

The issue asks Lobster to treat file-like relative workflow paths as workflow files or clear path errors instead of reparsing them as inline pipeline commands.

## Best Possible Solution

Land or replace the focused fix from https://github.com/openclaw/openclaw/pull/68106 in the Lobster runner with regression coverage that preserves normal inline pipelines.

## Reproduction Assessment

Yes. Source inspection on current main shows a missing relative .lobster path reaches resolveWorkflowFile(), has the thrown path error caught as null, and then falls through to inline pipeline execution.

## Solution Assessment

Yes as a code direction. A narrow Lobster plugin runner fix that rethrows workflow-file-like path resolution errors, with tests preserving inline pipelines, is the maintainable solution; the issue should stay open until that PR or an equivalent fix merges.
