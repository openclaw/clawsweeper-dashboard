## Summary

Keep open: current main and the latest release still swallow Lobster workflow-file resolution errors and fall back to inline pipeline execution, and the open closing PR remains the active implementation candidate.

## What This Changes

The issue requests that Lobster treat relative workflow-file-looking pipeline input as a workflow path or clear path error instead of silently falling back to inline pipeline parsing.

## Best Possible Solution

Land or replace the focused fix from https://github.com/openclaw/openclaw/pull/68106 in the Lobster runner with regression coverage, keeping normal inline pipelines unchanged.

## Reproduction Assessment

Yes at source level. A missing relative workflow path reaches resolveWorkflowFile(), has the thrown path error caught as null, and then falls through to inline pipeline execution; I did not run a live repro in this read-only review.

## Solution Assessment

Yes as a code direction. A narrow Lobster runner fix that rethrows workflow-file-like path resolution errors, with tests preserving inline pipelines, is the maintainable solution; this issue should stay open until that PR or an equivalent fix merges.
