## Summary

Keep open: current main still catches Lobster workflow-file resolution failures and falls back to inline pipeline execution, and the linked closing PR is open and unmerged.

## What This Changes

The issue requests that Lobster treat relative workflow-file-looking `pipeline` values as workflow paths or clear path errors instead of reparsing them as inline pipeline commands.

## Best Possible Solution

Review and land PR #68106, or an equivalent focused Lobster plugin fix, with regression coverage for missing relative workflow files and unchanged normal inline pipeline behavior.

## Reproduction Assessment

Yes. A run request with `pipeline: "lobster/missing-workflow.lobster"` reaches `resolveWorkflowFile()`, the stat/path error is caught as `null`, and the original string then goes to inline pipeline execution.

## Solution Assessment

Yes. A narrow extension-owned runner/test change is the best maintainable solution; no core API or broader product decision appears necessary from the current evidence.
