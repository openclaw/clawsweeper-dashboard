Keep open: current main and latest v2026.5.12 still swallow Lobster workflow-file resolution errors and fall back to inline pipeline execution, and https://github.com/openclaw/openclaw/pull/68106 remains open with closing syntax as the implementation candidate.

## What This Changes

The issue asks Lobster to preserve workflow-file/path errors for relative workflow paths instead of reparsing them as inline pipeline commands.

## Best Possible Solution

Refresh and land https://github.com/openclaw/openclaw/pull/68106, or replace it with an equivalent focused Lobster runner/test fix that rethrows workflow-file-like path resolution errors while preserving normal inline pipeline execution.
