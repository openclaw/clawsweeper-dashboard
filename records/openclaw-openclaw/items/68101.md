## Summary

Keep open: current main and the latest release still swallow Lobster workflow-file resolution failures and then pass the original file-looking string to inline pipeline execution. The linked fix PR is open and unmerged with closing syntax, so this issue should remain open until that PR or an equivalent focused fix lands.

## What This Changes

Request to make the Lobster tool treat relative workflow-file-looking `pipeline` values as workflow paths and surface path errors instead of falling back to inline pipeline parsing.

## Best Possible Solution

Review and land PR #68106, or land an equivalent focused Lobster plugin change that rethrows workflow resolution errors for workflow-file-like inputs and adds regression coverage for missing relative workflow files while preserving normal inline pipelines.

## Reproduction Assessment

Yes. A run request with `pipeline: "lobster/missing-workflow.lobster"` reaches `resolveWorkflowFile()`, the resulting stat/path error is caught and converted to `null`, and the original string is then passed to inline pipeline execution.

## Solution Assessment

Yes. A narrow extension-owned runner/test fix is the best maintainable solution; no core API or broader product decision appears necessary from the current evidence.
