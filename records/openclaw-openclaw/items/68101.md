## Summary

Keep open: current main and latest v2026.5.18 still catch Lobster workflow-file resolution errors and fall back to inline pipeline parsing, while the open closing PR remains unmerged and conflicting.

## What This Changes

The issue asks the Lobster tool to treat relative workflow-file-looking `pipeline` values as workflow paths and return path errors instead of reinterpreting them as inline pipelines.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/68106, or a refreshed equivalent in the Lobster runner, with regression coverage for missing relative workflow files and preserved inline-pipeline behavior.

## Reproduction Assessment

Yes. Source inspection on current main shows a missing relative workflow path reaches `resolveWorkflowFile()`, has its thrown path error caught as `null`, and then falls through to inline pipeline execution.

## Solution Assessment

Yes as a code direction. A narrow Lobster plugin runner fix that rethrows workflow-file-like path resolution errors, with tests preserving inline pipelines, is the maintainable solution; this issue should stay open until that PR or an equivalent fix merges.
