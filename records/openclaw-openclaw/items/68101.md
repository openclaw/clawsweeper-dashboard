## Summary

Keep open: current main and latest release still discard Lobster workflow-file resolution errors and fall back to inline pipeline execution, while https://github.com/openclaw/openclaw/pull/68106 remains open with closing syntax as the implementation candidate.

## What This Changes

Report clear workflow path errors when Lobster `pipeline` is a relative workflow-file path that cannot be resolved instead of treating it as an inline pipeline command.

## Best Possible Solution

Land or replace the focused Lobster runner/test fix from https://github.com/openclaw/openclaw/pull/68106 after branch refresh and proof, keeping workflow-file handling inside the Lobster plugin.

## Reproduction Assessment

Yes. Source inspection on current main shows a missing relative `.lobster` path reaches `resolveWorkflowFile()`, has the thrown path error caught as `null`, and then falls through to inline pipeline execution.

## Solution Assessment

Yes. A narrow `extensions/lobster/src/lobster-runner.ts` fix that rethrows workflow-like path resolution errors, with focused tests preserving normal inline pipelines, is the maintainable direction.
