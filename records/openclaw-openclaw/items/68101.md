## Summary

Keep open: current main and v2026.5.12 still catch Lobster workflow-file resolution errors and fall back to inline pipeline execution, while https://github.com/openclaw/openclaw/pull/68106 remains the open closing implementation candidate.

## What This Changes

The issue requests that Lobster treat workflow-looking relative paths as workflow files or path errors instead of falling back to inline pipeline parsing.

## Best Possible Solution

Land or replace the focused Lobster runner fix from https://github.com/openclaw/openclaw/pull/68106 with regression coverage, while keeping ordinary inline pipelines unchanged.

## Reproduction Assessment

Yes. Source inspection on current main shows a missing relative workflow path reaches resolveWorkflowFile(), has the thrown path error caught as null, and then falls through to inline pipeline execution; I did not run a live Lobster invocation in this read-only review.

## Solution Assessment

Yes as a code direction. A narrow Lobster plugin runner change that rethrows workflow-file candidate resolution errors, with tests preserving inline pipelines, is the maintainable fix; this issue should stay open until that PR or an equivalent fix merges.
