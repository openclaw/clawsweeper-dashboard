## Summary

Keep open: current main and the latest release still swallow Lobster workflow-file resolution errors and fall back to inline pipeline execution, and https://github.com/openclaw/openclaw/pull/68106 remains the open closing implementation candidate.

## What This Changes

The issue asks Lobster to treat relative workflow-file paths as file inputs and surface path resolution errors instead of reparsing them as inline pipelines.

## Best Possible Solution

Land or replace the focused fix from https://github.com/openclaw/openclaw/pull/68106 in the Lobster runner with regression coverage, keeping normal inline pipelines unchanged.

## Reproduction Assessment

Yes. Source inspection on current main shows a missing relative workflow path reaches resolveWorkflowFile(), has the thrown path error caught as null, and then falls through to inline pipeline execution.

## Solution Assessment

Yes as a code direction. A narrow Lobster plugin runner fix that rethrows workflow-file-like path resolution errors, with tests preserving inline pipelines, is the maintainable solution; this issue should stay open until that PR or an equivalent fix merges.
