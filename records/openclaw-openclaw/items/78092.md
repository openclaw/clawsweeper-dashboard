## Summary

Keep open. Current `main` and the latest tagged release still call the MCP SDK `Client.callTool` without `RequestOptions`, and open PR https://github.com/openclaw/openclaw/pull/78160 references this issue with closing syntax as the active fix path.

## What This Changes

Allow bundled MCP tool calls to use a per-server request timeout instead of inheriting the MCP SDK 60 second default.

## Best Possible Solution

Review and either land or close https://github.com/openclaw/openclaw/pull/78160; the final fix should cover runtime plumbing, public config schema/types, docs, focused regression tests, and real long-running MCP proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: bundled MCP tools execute through `params.runtime.callTool`, which reaches SDK `Client.callTool` without `RequestOptions`, and SDK 1.29.0 defaults omitted request timeouts to 60000 ms.

## Solution Assessment

Unclear until the timeout policy is accepted. Passing per-server request timeout options into the SDK is the narrow code direction, but the public config contract and progress-reset/default behavior need maintainer review.
