## Summary

Keep open for maintainer judgment: current main and v2026.5.12 still implement documented same-turn steering for rapid same-session messages, while the shipped orphaned-user prompt repair does not decide whether Feishu/direct messages should default to independent followups.

## What This Changes

The issue asks for rapid queued Feishu messages to produce independent replies instead of letting queued message content influence an active or later reply.

## Best Possible Solution

Maintainers should decide whether Feishu/direct-message traffic should keep the documented `steer` default with guidance to use `messages.queue.byChannel.feishu: "followup"`, or intentionally change that default with regression coverage.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the documented merge path: current source and docs show default `steer` routes rapid same-session messages into the active run. No live Feishu proof confirms a separate current-main prompt-builder bleed beyond that behavior.

## Solution Assessment

Is this the best way to solve the issue? Unclear. The existing `followup` mode is the narrow maintainable path for independent later turns, while changing Feishu or global defaults needs maintainer product judgment.
