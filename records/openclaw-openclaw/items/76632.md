## Summary

Keep open for maintainer judgment. Current main deliberately defaults busy-session handling to `steer`, which can include queued messages in the active run; the reported symptom is source-supported, but changing that default would be a product/UX decision rather than a narrow bug fix.

## What This Changes

The issue requests that rapid queued Feishu direct messages be processed as independent turns instead of later messages being included in an earlier reply.

## Best Possible Solution

Decide whether this should remain documented `steer` behavior with `/queue followup` or global `messages.queue.mode: "followup"` guidance, or change Feishu/direct-message queue defaults after maintainer review.

## Reproduction Assessment

Yes for the observed merge path: current source shows default `steer` queues follow-up text into the active streaming run, and the docs say that mode batches pending messages before the next model call. I did not run a live Feishu test.

## Solution Assessment

Unclear. A code fix is not obviously the best answer because current docs define this as expected `steer` behavior and `followup` already provides independent later turns; changing defaults needs maintainer product judgment.
