## Summary

Keep open for maintainer judgment. Current main still defaults busy-session arrivals to `steer`, which can inject queued text into the active run; independent later replies are available through `followup`, but changing Feishu/direct-message defaults is a product decision rather than a narrow bug fix.

## What This Changes

Request that rapid queued Feishu messages be processed and replied to independently instead of being steered into the active session response.

## Best Possible Solution

Maintainers should decide whether Feishu or direct-message traffic should keep the documented `steer` default with guidance to use `messages.queue.byChannel.feishu: "followup"`, or deliberately change that default with regression coverage.

## Reproduction Assessment

Yes for the documented merge path: source and docs show default `steer` routes rapid follow-up text into the active run. No live Feishu run confirms a separate prompt-builder bleed beyond that behavior.

## Solution Assessment

Unclear. Existing `followup` mode is the narrowest way to get independent later turns, while changing Feishu or global defaults needs maintainer product judgment.
