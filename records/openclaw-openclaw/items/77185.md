## Summary

Current main and the v2026.5.3 release already include the startup-load fix for selected external context-engine plugins, so the reported 2026.5.2 lossless-claw fallback is obsolete.

## What This Changes

The issue reports that OpenClaw 2026.5.2 loads the lossless-claw plugin but does not register it as the selected context engine, falling back to legacy when duplicate plugin ids are present.

## Best Possible Solution

Keep the shipped v2026.5.3 startup-load behavior and route any remaining duplicate-plugin warning to plugins doctor or a new issue with fresh v2026.5.3 logs if it still affects runtime registration.

## Reproduction Assessment

Yes, from source and the report: a selected external context-engine plugin without startup activation could be absent from the gateway startup scope, after which resolveContextEngine logs the unregistered-engine fallback. Current main and v2026.5.3 source include the selected slot in startup/preload coverage, so the old path is no longer expected to reproduce.

## Solution Assessment

Yes, including the selected non-legacy context-engine plugin in the gateway startup load plan is the narrow fix because it loads the plugin before agent turns resolve the configured engine without special-casing lossless-claw.
