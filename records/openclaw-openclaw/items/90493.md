## What This Changes

This PR changes Skill Workshop list and inspect service paths to mark pending create proposals stale when the target skill file already exists, and adds service regression tests for that behavior and terminal-status preservation.

## Best Possible Solution

Use https://github.com/openclaw/openclaw/pull/90421 as the canonical fix path, keeping proposal list and inspect read-only while projecting stale status consistently across service, gateway, tool, and UI consumers.

## Solution Assessment

No. Persisting stale from list/inspect is not the best fix because those are documented read-only surfaces; the safer solution is the read-only stale projection already implemented in https://github.com/openclaw/openclaw/pull/90421.
