## Summary

Keep open. Current main still builds light dreaming from a lookback window and appends every successful Dream Diary narrative without body-level duplicate suppression; open PR #70332 remains the active fix candidate rather than a merged implementation.

## What This Changes

The issue reports repeated byte-identical light-phase Dream Diary work summaries across managed dreaming cycles and asks for incremental emission or duplicate-body suppression.

## Best Possible Solution

Land or replace PR #70332 with a narrow memory-core fix that suppresses unchanged automatic diary bodies or tracks an incremental light-summary emission boundary, with regression coverage for repeated unchanged light sweeps.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to run repeated light dreaming over unchanged lookback data, or append the same Dream Diary body at different timestamps; current main keeps both dated entries by design.

## Solution Assessment

Unclear. The issue's `lastEmittedAt` cutoff is plausible, but PR #70332's body-level skip may be the narrower user-visible fix if it avoids repeated diary bodies without dropping genuinely changed prose.
