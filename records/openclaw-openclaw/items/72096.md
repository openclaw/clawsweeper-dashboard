## Summary

Keep open. Current main still appends repeated Dream Diary narratives and its dedupe fingerprint includes the timestamp, so unchanged light-phase prose at different times is preserved; open PR #70332 is the active repair path but has not merged.

## What This Changes

The issue asks memory-core dreaming to stop appending repeated verbatim light-phase work summaries to `DREAMS.md` across hourly cycles.

## Best Possible Solution

Memory-core should either skip unchanged automatic diary bodies or persist a per-phase light-summary emission boundary, with regression coverage for repeated light dreaming over unchanged input.

## Reproduction Assessment

Yes. A focused reproduction is to append the same Dream Diary body at two different timestamps, or run light dreaming repeatedly over unchanged deterministic input; current main preserves both dated entries.

## Solution Assessment

Unclear. The proposed `lastEmittedAt` cutoff is plausible, but body-level duplicate suppression as in #70332 may be the narrower user-visible fix if it avoids repeated diary prose without dropping genuinely changed summaries.
