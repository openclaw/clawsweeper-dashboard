## Summary

Keep this open: current main still stages light dreaming from the full configured lookback window and then appends a diary narrative from those staged snippets, so unchanged entries can be emitted again on later cycles instead of being treated as already emitted.

## What This Changes

The issue asks for light-phase dreaming diary summaries to stop repeating the same work-summary content across scheduled dreaming cycles.

## Best Possible Solution

Light dreaming should suppress already-emitted unchanged entries before writing reports or feeding the narrative, preferably by reusing phase-signal state or a narrow emitted-at marker rather than cleaning up DREAMS.md after the fact.

## Reproduction Assessment

Yes, source-reproducible: current light dreaming selects from the whole lookback window and appends a diary entry from those snippets without checking whether the same entries were already emitted by an earlier light cycle. I did not run an overnight live reproduction because this review is read-only.

## Solution Assessment

Yes: the best fix is to stop duplicate light emission at the memory-core selection/state boundary. Post-hoc DREAMS.md string dedupe would be too late because it would not fix repeated phase signals or REM/deep inputs.
