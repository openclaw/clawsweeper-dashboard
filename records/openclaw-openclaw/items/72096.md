## Summary

Keep open. Current main and the latest shipped tag still use a rolling light-dreaming lookback plus timestamp-sensitive Dream Diary dedupe, so unchanged light-phase input can still append the same narrative body under a new timestamp; the overlapping fix remains unmerged in https://github.com/openclaw/openclaw/pull/70332.

## What This Changes

Light-phase dreaming should stop appending identical work-summary or Dream Diary narrative blocks across repeated overnight cycles over unchanged input.

## Best Possible Solution

Implement one canonical memory-core behavior that prevents repeated unchanged automatic diary bodies, with regression coverage for repeated light sweeps and without hiding genuinely distinct diary entries.

## Reproduction Assessment

Yes, by source inspection. Repeated light dreaming over unchanged lookback input can feed the same capped snippets into narrative generation, and current append/dedupe logic keeps identical bodies when their timestamps differ.

## Solution Assessment

Unclear as a final implementation choice. The proposed last-emitted cutoff is plausible, but body-level automatic diary duplicate suppression may be the narrower user-visible fix, and https://github.com/openclaw/openclaw/pull/70332 already targets that direction with remaining blockers.
