## Summary

Keep open. Current main still builds light Dream Diary input from the configured lookback window and appends successful narratives without comparing prior diary bodies; the existing dedupe path and regression test explicitly keep identical prose when timestamps differ.

## What This Changes

Make light-phase Dream Diary/work-summary output incremental or dedupe unchanged summaries so identical commit-list content is not appended across repeated dreaming cycles.

## Best Possible Solution

Land or rework the focused memory-core diary hardening so unchanged automatic Dream Diary output is skipped while genuine new light-phase snippets still produce entries, with regression coverage for repeated light sweeps over unchanged input and for new snippets that must append.

## Reproduction Assessment

Yes. The issue body gives concrete repeated DREAMS.md excerpts, and the current source-level path is clear: stable lookback input can produce the same narrative body, while append and dedupe allow that body to remain under different timestamps.

## Solution Assessment

Unclear. The proposed `lastEmittedAt` cutoff is plausible for light summaries, but body-level duplicate skipping in PR #70332 may be the narrower central guard if it preserves legitimate distinct diary prose.
