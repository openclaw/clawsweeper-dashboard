## Summary

Keep open. Current main still allows identical light-phase Dream Diary bodies to be appended under different timestamps, and the already-linked #70332 remains the active open repair path rather than a merged fix.

## What This Changes

The issue asks memory-core dreaming to make light-phase work summaries incremental or suppress repeated identical diary content across repeated overnight cycles.

## Best Possible Solution

Land #70332 or an equivalent memory-core change that suppresses unchanged automatic diary bodies or records a light-phase emission boundary, with regression coverage for repeated unchanged light sweeps.

## Reproduction Assessment

Yes. Source inspection shows repeated light dreaming over unchanged lookback input can append the same body under a new timestamp, and the existing dedupe test explicitly preserves identical body text with distinct timestamps.

## Solution Assessment

Unclear. A `lastEmittedAt` cutoff is plausible, but body-level duplicate suppression may be the narrower maintainable fix because it prevents repeated diary prose without dropping genuinely changed summaries; #70332 is already exploring that direction.
