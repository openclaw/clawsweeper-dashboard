## Summary

Keep open. Current main still allows repeated light dreaming over the same rolling lookback to append the same Dream Diary body under new timestamps, and the overlapping repair path in #70332 is still separate from main rather than shipped.

## What This Changes

The issue asks for light-phase Dream Diary/work-summary output to stop repeating identical commit-list content across hourly dreaming cycles.

## Best Possible Solution

Land #70332 or an equivalent memory-core change that suppresses unchanged automatic diary bodies, or tracks a light-phase emission boundary, while preserving genuinely changed phase summaries.

## Reproduction Assessment

Yes. Source inspection shows repeated light dreaming can reuse unchanged lookback input and append the same narrative body under a new timestamp, while the current dedupe test explicitly keeps identical bodies at different timestamps.

## Solution Assessment

Unclear. The proposed `lastEmittedAt` cutoff is plausible, but the open #70332 body-level duplicate suppression may be the narrower fix for the visible `DREAMS.md` duplication without changing input selection semantics.
