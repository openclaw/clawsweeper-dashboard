## Summary

Keep open. Current main still builds light dreaming from a rolling lookback and appends timestamped Dream Diary entries without same-body suppression, while the active overlapping repair PR remains open and unmerged.

## What This Changes

The issue asks memory-core dreaming to stop repeated identical light-phase work-summary content from being emitted into DREAMS.md across scheduled cycles.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/70332 or an equivalent memory-core fix that suppresses unchanged automatic diary bodies or records a light-phase emission boundary, with regression coverage for repeated unchanged light sweeps.

## Reproduction Assessment

Yes. Source inspection shows repeated light dreaming over unchanged lookback input can feed the same snippets into narrative generation, and current append/dedupe logic preserves identical bodies when timestamps differ.

## Solution Assessment

Unclear. The proposed lastEmittedAt cutoff is plausible, but body-level duplicate suppression may be the narrower maintainable fix for the user-visible DREAMS.md duplication; the active repair PR targets that direction but is still open.
