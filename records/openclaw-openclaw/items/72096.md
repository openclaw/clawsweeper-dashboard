## Summary

Keep open. Current main still uses a rolling light-dreaming lookback and timestamp-sensitive Dream Diary dedupe, so unchanged light-phase narrative bodies can still be appended with different timestamps; the overlapping fix remains an open PR rather than merged or shipped.

## What This Changes

The issue asks for light-phase dreaming summaries in `DREAMS.md` to stop repeating the same work-summary body across hourly cycles, either by tracking an emit cutoff or suppressing duplicate diary bodies.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/70332 after refresh/proof or replace it with an equivalent memory-core fix that suppresses unchanged automatic diary bodies or records a light-phase emission boundary, with regression coverage for repeated unchanged light sweeps.

## Reproduction Assessment

Yes, by source inspection. Repeated light dreaming over unchanged lookback input can feed the same capped snippets into narrative generation, and current append/dedupe logic preserves identical bodies when timestamps differ.

## Solution Assessment

Unclear as a final implementation choice. The proposed `lastEmittedAt` cutoff is plausible, but body-level duplicate suppression may be the narrower user-visible fix, and https://github.com/openclaw/openclaw/pull/70332 already targets that path with remaining review/proof work.
