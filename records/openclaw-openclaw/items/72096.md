## Summary

Keep open. Current main still uses a rolling light-dreaming lookback and timestamp-sensitive Dream Diary dedupe, so unchanged light-phase narrative bodies can still be appended with different timestamps; the overlapping fix remains an open PR rather than merged or shipped.

## What This Changes

The issue reports repeated light-phase dreaming work summaries being appended verbatim to DREAMS.md across multiple managed cycles.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/70332 or an equivalent memory-core fix that suppresses unchanged automatic diary bodies or records a light-phase emission boundary, with regression coverage for repeated unchanged light sweeps.

## Reproduction Assessment

Yes, by source inspection. Repeated light dreaming over unchanged lookback input can feed the same capped snippets into narrative generation, and current append/dedupe logic preserves identical bodies when timestamps differ.

## Solution Assessment

Unclear as a final implementation choice. The proposed lastEmittedAt cutoff is plausible, but body-level duplicate suppression may be the narrower user-visible fix; https://github.com/openclaw/openclaw/pull/70332 targets that path but still needs review fixes and proof.
