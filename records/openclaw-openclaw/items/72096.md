## Summary

Keep open. Current main still builds light Dream Diary content from a lookback window and appends each successful narrative without body-level duplicate suppression, while the active duplicate-diary repair path remains open in PR #70332.

## What This Changes

Make light-phase Dream Diary work summaries incremental or suppress unchanged repeated summary content across dreaming cycles.

## Best Possible Solution

Memory-core should skip unchanged automatic diary bodies or persist a light-summary emission boundary, with regression coverage for repeated unchanged sweeps.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to append the same Dream Diary body at two different timestamps, or run repeated light dreaming over unchanged deterministic input; current main preserves both dated entries.

## Solution Assessment

Unclear. The proposed lastEmittedAt cutoff is plausible, but body-level duplicate suppression may be the narrower user-visible fix if it avoids repeated diary prose without dropping genuinely changed summaries.
