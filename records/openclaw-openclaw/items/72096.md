## Summary

Keep open. Current main still builds light dreaming from a rolling lookback and appends timestamped Dream Diary entries without body-level duplicate suppression; #70332 remains open as the active overlapping repair path rather than a merged fix.

## What This Changes

The issue asks memory-core dreaming to stop re-emitting identical light-phase Dream Diary/work-summary content across repeated managed cycles.

## Best Possible Solution

Land #70332 or an equivalent memory-core change that suppresses unchanged automatic diary bodies or tracks a light-phase emission boundary, with regression coverage for repeated unchanged light sweeps.

## Reproduction Assessment

Yes. Source inspection shows repeated light dreaming over unchanged lookback input can append the same body under a new timestamp, and the existing dedupe test explicitly preserves identical body text with distinct timestamps.

## Solution Assessment

Unclear. The proposed lastEmittedAt cutoff is plausible, but body-level duplicate suppression may be narrower if it prevents repeated diary prose without dropping genuinely changed summaries; #70332 targets that direction but is still open.
