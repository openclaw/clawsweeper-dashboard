## Summary

Keep open. Current main still builds light dreaming from a rolling lookback and appends timestamped Dream Diary entries without body-level duplicate suppression; open PR #70332 remains the active overlapping repair path rather than a merged fix.

## What This Changes

The issue asks memory-core dreaming to avoid emitting identical light-phase Dream Diary/work-summary content across repeated overnight cycles, either by tracking last-emitted input or suppressing duplicates.

## Best Possible Solution

Review and land #70332 or an equivalent memory-core change that suppresses unchanged automatic diary bodies, with regression coverage for repeated unchanged light sweeps.

## Reproduction Assessment

Yes. Source inspection shows repeated light dreaming over unchanged lookback input can append the same narrative body under a new timestamp, and current tests explicitly preserve identical body text with distinct timestamps.

## Solution Assessment

Unclear. The issue's `lastEmittedAt` cutoff is plausible, but #70332's body-level duplicate suppression may be narrower because it prevents repeated diary prose without dropping genuinely changed summaries.
