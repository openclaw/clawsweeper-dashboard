## Summary

Keep open. Current main still builds light dreaming from a rolling lookback and appends timestamped Dream Diary entries without same-body suppression, while https://github.com/openclaw/openclaw/pull/70332 remains the active unmerged fix path.

## What This Changes

The issue asks memory-core dreaming to stop repeating identical light-phase work summaries in DREAMS.md across recurring overnight cycles.

## Best Possible Solution

Land or rework https://github.com/openclaw/openclaw/pull/70332, or an equivalent memory-core fix that skips unchanged automatic diary bodies or records a light-phase emit boundary with regression coverage.

## Reproduction Assessment

Yes, source-level. Repeated light runs over unchanged rolling lookback input can select the same snippets, appendNarrativeEntry always inserts a timestamped diary block, and the current dedupe test preserves identical body text across different timestamps.

## Solution Assessment

Unclear. The proposed lastEmittedAt cutoff is plausible, but body-level duplicate suppression may be the narrower maintainable fix for the user-visible DREAMS.md duplication; the active repair PR targets that direction but still needs review/proof before merge.
