## Summary

Keep open. Current main still selects light dreaming input from a lookback window and appends timestamped diary entries without body-level duplicate suppression, while open PR #70332 remains the active unmerged fix candidate.

## What This Changes

The issue asks memory-core dreaming to stop repeating identical light-phase Dream Diary/work-summary content across scheduled cycles, either with an incremental emission boundary or duplicate suppression.

## Best Possible Solution

Land or replace #70332 with a narrow memory-core fix that suppresses unchanged automatic diary bodies or tracks a per-phase emission cutoff, with regression coverage for repeated light sweeps over unchanged input.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to append identical narrative text at two different timestamps or run light dreaming twice over unchanged recall input; current code creates distinct timestamped entries and the dedupe test preserves that case.

## Solution Assessment

Unclear. The issue's lastEmittedAt cutoff is plausible, but body-level duplicate suppression in #70332 may be the narrower user-visible fix if it avoids repeats without dropping genuinely changed summaries.
