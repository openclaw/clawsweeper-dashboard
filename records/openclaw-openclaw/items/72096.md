## Summary

Keep open. Current main still builds light dreaming from a lookback window and appends every successful Dream Diary narrative without body-level duplicate suppression; open PR #70332 remains the active fix candidate rather than a merged implementation.

## What This Changes

Make light-phase Dream Diary/work-summary output incremental, or suppress unchanged repeated blocks across dreaming cycles.

## Best Possible Solution

Land or replace PR #70332 with a narrow memory-core fix that suppresses unchanged automatic diary bodies or tracks an incremental light-summary emission boundary, with regression coverage for repeated unchanged sweeps.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to append the same Dream Diary body at two different timestamps, or run repeated light dreaming over unchanged deterministic input; current main preserves both dated entries because append always writes and dedupe includes the date line.

## Solution Assessment

Unclear. The proposed `lastEmittedAt` cutoff is plausible, but PR #70332's body-level skip may be the narrower user-visible fix if it avoids repeated diary bodies without dropping genuinely changed prose.
