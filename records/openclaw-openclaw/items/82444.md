## Summary

Keep this PR open: the code change is narrow and matches a real source-level crash path, but the external PR still needs after-fix real behavior proof before merge.

## What This Changes

The PR adds a client-side socket error handler for debug proxy CONNECT tunnels and changes upstream-error cleanup from ending to destroying the client socket.

## Best Possible Solution

The debug proxy CONNECT tunnel should handle socket errors from both sides, record the failure, and tear down the paired socket without crashing the OpenClaw process.

## Reproduction Assessment

Yes, at source level: current main has no clientSocket error listener in the CONNECT tunnel, and Node's EventEmitter/net.Socket contract makes an unhandled socket error a process-ending path. I did not run a live current-main crash repro in this read-only review.

## Solution Assessment

Yes, the proposed direction is the narrow maintainable fix: add a regular error listener on the accepted client socket and destroy the paired upstream socket on failure. The remaining merge blocker is proof, not a different code direction.
