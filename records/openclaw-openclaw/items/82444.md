## Summary

Keep open: this is a plausible focused bug fix for current `main`. The CONNECT tunnel currently pipes `clientSocket` and `upstreamSocket` but only handles upstream socket errors, and Node treats an unhandled `error` event as process-fatal; I found no blocking code-review finding in the PR diff, but the PR still lacks after-fix real behavior proof before merge.

## What This Changes

The PR adds a `clientSocket` error handler to the debug proxy CONNECT tunnel and changes upstream-error cleanup from `clientSocket.end()` to `clientSocket.destroy()`.

## Best Possible Solution

Land this or an equivalent client-socket error cleanup after real debug proxy proof is posted, CI is green, and ideally a focused regression covers the crash path.

## Reproduction Assessment

## Solution Assessment

Yes, the PR is the narrow maintainable fix direction: handle the client socket error symmetrically and tear down the peer socket. The remaining merge gate is proof and, preferably, regression coverage rather than a different product design.
