## Summary

Keep open: this is an open, maintainer-labeled, dirty PR, and current head still has blocking correctness issues plus a concrete plugin-install trust-boundary concern. The uninstall-safe managed npm root issue from the earlier review appears addressed, but the remaining findings block merge rather than support cleanup closure.

## What This Changes

The PR moves plugin dependency ownership to install-time npm/git roots, removes bundled runtime dependency staging and repair paths plus related CI/docs/tests, changes packaged plugin loading and SDK alias generation, adds legacy dependency cleanup/repair flows, and externalizes QQ Bot as a downloadable plugin.

## Best Possible Solution

Keep the install-time dependency ownership direction only after revising the branch so SDK alias generation preserves named-only subpath compatibility, doctor cleanup removes only known OpenClaw-generated legacy artifacts, git plugin source is validated before package-manager work, and the rebased branch passes focused plugin install/doctor tests plus the broad changed gate in Testbox.

## Reproduction Assessment

Yes. The blockers reproduce by deterministic source review: import a generated alias for named-only `openclaw/plugin-sdk/outbound-runtime`, run the cleanup target collector against a plugin root with a legitimate `node_modules`, and trace git install order from clone to npm install to OpenClaw validation.

## Solution Assessment

No. The dependency-model direction may be maintainable, but this implementation is not the best fix until it preserves SDK compatibility and keeps cleanup and git-install trust boundaries narrow.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve default-safe SDK alias wrappers:** `src/plugins/plugin-sdk-dist-alias.ts:13`
  - body: The wrapper now emits `export { default }` for every SDK subpath. Named-only SDK files such as `outbound-runtime` do not export default, so importing the generated alias can fail during ESM instantiation; keep the current namespace fallback or only default-forward modules that actually export one.
  - confidence: 0.94
- **[P1] Stop deleting valid plugin node_modules:** `src/commands/doctor/shared/plugin-dependency-cleanup.ts:39`
  - body: Doctor cleanup marks every plugin-local `node_modules` under plugin roots as legacy debris. This branch also makes local/path/archive plugins rely on preinstalled dependencies, so `doctor --fix` can delete a valid dependency tree and break the plugin on the next load.
  - confidence: 0.91
- **[P2] Scan git plugins before npm install:** `src/plugins/git-install.ts:292-309`
  - body: This path runs `npm install` in the cloned repository before `installPluginFromInstalledPackageDir` performs OpenClaw's install validation. Even with lifecycle scripts disabled, dependency resolution and network work happen for unvalidated git plugin source; validate the source first.
  - confidence: 0.86
