## Summary

Keep this PR open: the latest head still has concrete merge blockers in plugin SDK aliasing, npm/git install discovery, git install scan ordering, doctor cleanup, and uninstall cleanup. The PR also carries the protected `maintainer` label, so it is not a cleanup-close candidate; it needs revision before normal maintainer handling can continue.

## What This Changes

This PR removes bundled runtime dependency staging and related commands/tests, moves npm/git plugin installs into OpenClaw-owned user roots, externalizes QQ Bot as downloadable, and updates loader, doctor, docs, CI, Docker, and tests for install-time plugin dependency ownership.

## Best Possible Solution

Revise the PR while preserving the install-time dependency model: keep startup/load-time package-manager work removed, but restore default-safe SDK alias compatibility, make npm/git install records discoverable and loadable, keep git source safety scans before package-manager work, narrow doctor cleanup to OpenClaw-generated legacy artifacts, and teach uninstall/update paths about the new managed package roots.

## Reproduction Assessment

Yes. The SDK alias failure has a direct Node ESM reproduction for re-exporting `default` from a named-only module, and the install-root, doctor-cleanup, scan-order, and uninstall issues follow directly from the latest PR source against current-main discovery/uninstall behavior.

## Solution Assessment

No. The product direction may be maintainable, but this patch is not the safest implementation until it preserves compatibility and scan ordering and updates the registry, cleanup, and uninstall contracts around the new roots.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Preserve default-safe SDK alias wrappers:** `src/plugins/plugin-sdk-dist-alias.ts:12-13`
  - body: The generated alias now unconditionally re-exports `default` from every SDK subpath. Most `dist/plugin-sdk/*.js` files only have named exports, and ESM rejects that wrapper during module instantiation, so packaged plugins importing named SDK helpers through `openclaw/plugin-sdk/*` can fail before registration. Keep the old namespace fallback or only re-export `default` when the source module actually has one.
  - confidence: 0.94
- **[P1] Add discovery for package install roots:** `src/plugins/install.ts:1146-1147`
  - body: Npm plugins are now installed under `~/.openclaw/npm/node_modules/<pkg>` and git plugins under `~/.openclaw/git/...`, but plugin discovery still scans `~/.openclaw/extensions`, workspace, bundled, and configured load paths. Install records are only used after a candidate is found, so a successful npm/git install can be recorded and enabled without appearing in the refreshed registry or Gateway load set.
  - confidence: 0.9
- **[P2] Scan git plugin sources before npm install:** `src/plugins/git-install.ts:292-309`
  - body: This path runs `npm install` inside the cloned git repository before `installPluginFromInstalledPackageDir` performs the package source safety scan. Even with lifecycle scripts disabled, a source that the scanner would reject can now trigger dependency resolution and network/package-manager work first; keep pre-install validation/scan ahead of dependency convergence for git installs.
  - confidence: 0.84
- **[P1] Stop deleting plugin node_modules during doctor cleanup:** `src/commands/doctor/shared/plugin-dependency-cleanup.ts:37-46`
  - body: The new cleanup marks any `node_modules` directory under `extensions/*` as removable legacy state, but this PR also makes local/path/archive installs rely on preinstalled dependencies. Running `openclaw doctor --fix` after such an install can delete valid runtime dependencies and break the plugin on next load.
  - confidence: 0.9
- **[P2] Delete managed npm installs on uninstall:** `src/plugins/install.ts:1146-1147`
  - body: The new npm install target is `.openclaw/npm/node_modules/<pkg>`, but uninstall safety still only trusts the managed `extensions` layout. Uninstall will fall back to removing `.openclaw/extensions/<pluginId>` and leave the actual npm package directory behind, so repeated install/uninstall cycles accumulate stale plugin code.
  - confidence: 0.86
