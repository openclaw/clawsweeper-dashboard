## Summary

Keep open: the latest PR head is still an open, unmerged, maintainer-labeled dependency-model rewrite with blocking correctness and security findings. The later uninstall fix addresses one earlier review point, but the SDK alias, doctor cleanup, and git-install trust-boundary regressions remain on the branch.

## What This Changes

The PR moves plugin dependency ownership to explicit npm/git install roots, removes bundled runtime dependency staging and repair paths plus related CI/docs/tests, prefers native packaged JavaScript loading, externalizes QQ Bot, and adds doctor cleanup plus downloadable-plugin repair behavior.

## Best Possible Solution

Revise the branch while preserving the install-time dependency ownership direction: keep SDK alias compatibility, limit doctor cleanup to known OpenClaw-generated legacy artifacts, scan git plugin source before package-manager work, keep the managed npm uninstall fix, and rerun the focused plugin, doctor, Docker, and release-path gates.

## Reproduction Assessment

Yes. The blockers reproduce by deterministic source review: generate an alias for a named-only SDK subpath, follow doctor cleanup target collection for a valid plugin node_modules directory, and trace git install order before the package source scan.

## Solution Assessment

No. The direction may be maintainable, but this implementation is not the best fix until it preserves existing SDK compatibility and keeps cleanup and git-source trust boundaries intact.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve default-safe SDK alias wrappers:** `src/plugins/plugin-sdk-dist-alias.ts:13`
  - body: The new wrapper always emits `export { default }` for every SDK subpath. Named-only SDK files such as `outbound-runtime` do not export default, so importing the generated wrapper can fail during ESM instantiation; keep the current namespace fallback or only default-forward modules that actually have a default export.
  - confidence: 0.94
- **[P1] Stop deleting valid plugin node_modules:** `src/commands/doctor/shared/plugin-dependency-cleanup.ts:39`
  - body: Doctor cleanup now treats every plugin-local `node_modules` under extension roots as legacy debris. With this branch making plugin packages own their dependencies, `doctor --fix` can delete a valid runtime dependency tree and break the plugin on the next load.
  - confidence: 0.91
- **[P2] Scan git plugins before npm install:** `src/plugins/git-install.ts:292-309`
  - body: This path runs `npm install` in the cloned repository before the source safety scan in `installPluginFromInstalledPackageDir`. Even with lifecycle scripts disabled, OpenClaw is doing dependency resolution and network work for unapproved source; validate the plugin source before package-manager work.
  - confidence: 0.86
