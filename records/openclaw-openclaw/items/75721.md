## Summary

Keep open: this is a protected maintainer-labeled, broad plugin dependency refactor and the latest PR head still has concrete correctness and security-sensitive blockers around SDK alias compatibility, doctor cleanup, git install scan ordering, and managed npm uninstall behavior.

## What This Changes

The PR removes bundled/runtime dependency staging and related CLI/Docker coverage, moves npm and git plugin installs into OpenClaw-owned user roots, loads packaged JS natively first, adds doctor cleanup/repair behavior, externalizes QQ Bot as downloadable, and updates related docs/tests/workflows.

## Best Possible Solution

Revise the PR while preserving the install-time dependency ownership direction: keep SDK alias compatibility, scan git sources before package-manager work, make npm/git roots first-class managed install targets for uninstall/discovery, and narrow doctor cleanup to OpenClaw-generated legacy artifacts only.

## Reproduction Assessment

Yes. The blockers reproduce by deterministic source review of the PR head against current main: wrapper export shape, cleanup target selection, install ordering, and uninstall trust boundaries are visible without live credentials.

## Solution Assessment

No. The direction may be maintainable, but this implementation is not the best fix until it preserves current SDK compatibility and the safety boundaries around new dependency roots.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve default-safe SDK alias wrappers:** `src/plugins/plugin-sdk-dist-alias.ts:12-13`
  - body: The new wrapper always emits `export { default }` for every SDK subpath. Named-only SDK modules such as `src/plugin-sdk/outbound-runtime.ts` have no default export, so ESM instantiation rejects the wrapper; keep the namespace fallback or only default-forward modules that actually provide one.
  - confidence: 0.94
- **[P1] Stop deleting plugin-local node_modules:** `src/commands/doctor/shared/plugin-dependency-cleanup.ts:37-46`
  - body: The cleanup treats plugin-local `node_modules` directories as legacy debris, but this PR also makes local/path/archive plugins rely on preinstalled dependencies. Running `doctor --fix` can delete valid runtime deps and break those plugins after repair.
  - confidence: 0.9
- **[P2] Scan git plugin sources before npm install:** `src/plugins/git-install.ts:292-309`
  - body: This path runs package-manager work inside the cloned git repository before the plugin source safety scan. Even with lifecycle scripts disabled, dependency resolution and network activity now happen for unapproved source; keep source validation ahead of dependency convergence.
  - confidence: 0.84
- **[P2] Delete managed npm installs on uninstall:** `src/plugins/install.ts:1146-1147`
  - body: Npm plugins now install under `.openclaw/npm/node_modules/<pkg>`, but uninstall safety still only trusts the managed extensions layout before falling back to `extensions/<id>`. Uninstall can leave the actual package root behind and accumulate stale plugin code.
  - confidence: 0.86
