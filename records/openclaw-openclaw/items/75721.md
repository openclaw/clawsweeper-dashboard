## Summary

Keep open: this maintainer-labeled dependency-model PR is not safe to close, and latest head still has blocking correctness and security issues around SDK alias compatibility, cleanup scope, git install scanning order, and uninstall cleanup for the new npm root.

## What This Changes

The PR removes bundled/runtime plugin dependency staging, moves plugin dependency ownership to npm/git install roots with local plugins requiring preinstalled deps, changes packaged JS loading, adds legacy cleanup/repair behavior, and externalizes QQ Bot as downloadable.

## Best Possible Solution

Revise the branch while preserving the install-time dependency ownership direction: keep SDK alias compatibility, limit doctor cleanup to known OpenClaw-generated legacy artifacts, scan git plugin source before package-manager work, and make npm/git install roots first-class uninstall targets.

## Reproduction Assessment

Yes. The blockers reproduce by deterministic latest-head source review: wrapper export shape, cleanup target selection, git install order, and uninstall target trust boundaries are visible without live credentials.

## Solution Assessment

No. The overall direction may be maintainable, but this implementation is not the best fix until it preserves existing SDK compatibility and the plugin install security and cleanup boundaries.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve default-safe SDK alias wrappers:** `src/plugins/plugin-sdk-dist-alias.ts:13`
  - body: The new wrapper unconditionally emits `export { default }` for every SDK subpath. Named-only SDK files such as `src/plugin-sdk/outbound-runtime.ts` do not export default, so ESM instantiation can fail; keep the namespace fallback or only default-forward modules that actually provide one.
  - confidence: 0.94
- **[P1] Stop deleting plugin-local node_modules:** `src/commands/doctor/shared/plugin-dependency-cleanup.ts:39`
  - body: The cleanup marks every plugin-local `node_modules` under `extensions/*` or `dist/extensions/*` as removable legacy debris, but this PR makes local/path/archive plugins rely on already-installed dependencies. Running `doctor --fix` can delete valid runtime deps and break those plugins.
  - confidence: 0.91
- **[P2] Scan git plugin sources before npm install:** `src/plugins/git-install.ts:292-309`
  - body: This path runs `npm install` in the cloned git repository before the package install path performs the source safety scan. Even with lifecycle scripts disabled, OpenClaw is doing dependency resolution and network work for unapproved source; validate the plugin source before package-manager work.
  - confidence: 0.85
- **[P2] Teach uninstall about managed npm roots:** `src/plugins/install.ts:1158`
  - body: Npm plugins now install under `~/.openclaw/npm/node_modules/<pkg>`, but uninstall still only trusts extension-root paths before falling back to `extensions/<id>`. Uninstalling these plugins leaves the actual npm package root behind as stale code on disk.
  - confidence: 0.86
