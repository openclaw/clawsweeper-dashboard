## Summary

Keep open: this PR is protected by the maintainer label and the latest head still has blocking plugin dependency and install-safety regressions. Source inspection confirms the SDK alias wrapper, doctor cleanup target selection, git install ordering, and managed npm uninstall path need revision before merge.

## What This Changes

The PR removes bundled/runtime dependency staging and related CLI/Docker coverage, moves npm and git plugin installs into OpenClaw-owned user roots, loads packaged JS natively first, adds legacy cleanup/downloadable-plugin repair behavior, and externalizes QQ Bot as downloadable.

## Best Possible Solution

Revise this branch while preserving the install-time dependency ownership direction: retain SDK alias compatibility, constrain doctor cleanup to known OpenClaw-generated legacy artifacts, validate git plugin source before package-manager work, and make npm/git install roots first-class uninstall/discovery targets.

## Reproduction Assessment

Yes. The blockers reproduce by deterministic source review of the latest PR head: wrapper export shape, cleanup target selection, git install order, and uninstall target trust boundaries are visible without live credentials.

## Solution Assessment

No. The direction may be maintainable, but this implementation is not the narrowest safe solution until it preserves existing SDK compatibility and plugin install security boundaries.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve default-safe SDK alias wrappers:** `src/plugins/plugin-sdk-dist-alias.ts:13`
  - body: The new wrapper unconditionally emits `export { default }` for every SDK subpath. Named-only SDK files such as `src/plugin-sdk/outbound-runtime.ts` do not export default, so ESM instantiation will fail; keep the namespace fallback or only default-forward modules that actually provide one.
  - confidence: 0.94
- **[P1] Stop deleting plugin-local node_modules:** `src/commands/doctor/shared/plugin-dependency-cleanup.ts:39`
  - body: The cleanup marks every plugin-local `node_modules` under `extensions/*` or `dist/extensions/*` as removable legacy debris, but this PR makes local plugins rely on already-installed dependencies. Running `doctor --fix` can delete valid runtime deps and break those plugins.
  - confidence: 0.9
- **[P2] Scan git plugin sources before npm install:** `src/plugins/git-install.ts:292-309`
  - body: This path runs `npm install` in the cloned git repository before the package install path performs the source safety scan. Even with lifecycle scripts disabled, OpenClaw is doing dependency resolution and network work for unapproved source; validate the plugin source before package-manager work.
  - confidence: 0.84
- **[P2] Teach uninstall about managed npm roots:** `src/plugins/install.ts:1158`
  - body: Npm plugins now install under `~/.openclaw/npm/node_modules/<pkg>`, but uninstall still only trusts extension-root paths before falling back to `extensions/<id>`. Uninstalling these plugins leaves the actual npm package root behind as stale code on disk.
  - confidence: 0.86
