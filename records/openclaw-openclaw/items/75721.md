## Summary

Keep open: this protected maintainer-labeled PR still has blocking correctness findings and one concrete plugin-install trust-boundary concern. The latest PR context shows the earlier uninstall-target issue was addressed, but the SDK alias, doctor cleanup, and git install-order problems remain merge blockers.

## What This Changes

The PR moves plugin dependency ownership to install-time npm/git roots, removes bundled runtime dependency staging and repair paths, externalizes QQ Bot, and updates Docker, CI, docs, tests, and release validation around that model.

## Best Possible Solution

Keep the install-time dependency ownership direction only after revising the branch so generated SDK aliases preserve named-only subpath compatibility, doctor cleanup removes only known OpenClaw-generated legacy artifacts, and git plugin source is validated before package-manager work. The revised branch should pass focused plugin install/doctor tests plus the broad changed gate in Testbox.

## Reproduction Assessment

Yes. The blockers have high-confidence deterministic reproduction paths: import a generated alias for named-only `openclaw/plugin-sdk/outbound-runtime`, run the cleanup collector against a plugin root with a legitimate `node_modules`, and trace the git install flow from clone to npm install to OpenClaw validation.

## Solution Assessment

No. The dependency-model direction may be maintainable, but this implementation is not the best fix until it preserves SDK compatibility and keeps cleanup and git-install trust boundaries narrow.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve default-safe SDK alias wrappers:** `src/plugins/plugin-sdk-dist-alias.ts:13`
  - body: The wrapper now emits `export { default }` for every SDK subpath. Named-only SDK files such as `outbound-runtime` do not export default, so importing the generated alias can fail during ESM instantiation; keep the current namespace fallback or only default-forward modules that actually export one.
  - confidence: 0.94
- **[P1] Stop deleting valid plugin node_modules:** `src/commands/doctor/shared/plugin-dependency-cleanup.ts:39`
  - body: Doctor cleanup marks plugin-local `node_modules` under plugin roots as legacy debris. This branch also makes local/path/archive plugins rely on preinstalled dependencies, so `doctor --fix` can delete a valid dependency tree and break the plugin on the next load.
  - confidence: 0.91
- **[P2] Scan git plugins before npm install:** `src/plugins/git-install.ts:292-309`
  - body: This path runs npm dependency resolution in the cloned repository before `installPluginFromInstalledPackageDir` performs OpenClaw install validation. Even with lifecycle scripts disabled, dependency resolution and network work happen for unvalidated git plugin source; validate the source first.
  - confidence: 0.86
