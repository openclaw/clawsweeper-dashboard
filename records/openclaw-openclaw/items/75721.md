## Summary

Keep open: this is a protected maintainer-labeled PR, and the latest head still has blocking plugin dependency-model regressions. Two earlier findings remain reproducible on the current PR head, the git install trust-boundary issue remains, and the prior npm-uninstall finding appears addressed by a later commit.

## What This Changes

The PR moves plugin dependency ownership to explicit npm/git install roots, removes bundled runtime dependency staging and repair paths, prefers native packaged JavaScript loading, externalizes QQ Bot as downloadable, and updates related doctor, Docker, CI, docs, and tests.

## Best Possible Solution

Keep the install-time dependency ownership direction, but revise the branch so SDK alias generation preserves named-only subpath compatibility, doctor cleanup only removes known OpenClaw-generated legacy artifacts, and git plugin source validation happens before package-manager work.

## Reproduction Assessment

Yes. The blockers reproduce by deterministic source review against the latest PR head: generate an alias for a named-only SDK subpath, follow doctor cleanup collection for a valid plugin-local `node_modules`, and trace git install order before the install-source scan.

## Solution Assessment

No. The proposed direction may be maintainable, but this implementation is not the best fix until it preserves SDK compatibility and keeps cleanup and git-install trust boundaries narrow.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve default-safe SDK alias wrappers:** `src/plugins/plugin-sdk-dist-alias.ts:13`
  - body: The wrapper now emits `export { default }` for every SDK subpath. Named-only SDK files such as `outbound-runtime` do not export default, so importing the generated alias can fail during ESM instantiation; keep the current namespace fallback or only default-forward modules that actually export one.
  - confidence: 0.94
- **[P1] Stop deleting valid plugin node_modules:** `src/commands/doctor/shared/plugin-dependency-cleanup.ts:39`
  - body: Doctor cleanup marks every plugin-local `node_modules` under extension roots as legacy debris. This branch also makes local/path/archive plugins rely on preinstalled dependencies, so `doctor --fix` can delete a valid dependency tree and break the plugin on the next load.
  - confidence: 0.91
- **[P2] Scan git plugins before npm install:** `src/plugins/git-install.ts:292-309`
  - body: This path runs `npm install` in the cloned repository before `installPluginFromInstalledPackageDir` performs OpenClaw's install validation. Even with lifecycle scripts disabled, dependency resolution and network work happen for unvalidated git plugin source; validate the source first.
  - confidence: 0.86
