## Summary

Keep open: this PR is protected by the `maintainer` label and the latest reviewed diff still has blocking correctness and security issues in the plugin install path. The core direction may be viable, but the SDK alias wrapper, doctor cleanup, uninstall cleanup, and git-install scan ordering need revision before merge.

## What This Changes

The PR removes bundled/runtime plugin dependency staging, moves npm and git plugin installs into OpenClaw-owned user roots, removes `plugins deps` and bundled dependency Docker coverage, makes packaged JS load natively first, adds doctor cleanup/repair behavior, externalizes QQ Bot as downloadable, and updates related docs/tests/workflows.

## Best Possible Solution

Revise this PR while preserving the install-time dependency ownership model: keep SDK alias compatibility, scan git plugin source before any package-manager work, make npm/git install roots discoverable and uninstallable as managed roots, and narrow doctor cleanup to OpenClaw-generated legacy artifacts only.

## Reproduction Assessment

Yes. The blockers reproduce by deterministic source review against the latest PR context and current main: the changed wrapper shape, cleanup target collection, uninstall trust boundary, and git install ordering are visible without needing live credentials.

## Solution Assessment

No. The dependency model can be maintainable, but this implementation is not the best fix until it preserves compatibility and install safety around the new roots.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve default-safe SDK alias wrappers:** `src/plugins/plugin-sdk-dist-alias.ts:12-13`
  - body: The wrapper now always emits `export { default }` for every SDK subpath. Most generated SDK subpath modules only expose named exports, so ESM rejects the wrapper when the source has no default export. Keep the current namespace-safe fallback or only default-forward modules that actually provide one.
  - confidence: 0.94
- **[P1] Stop deleting plugin node_modules during doctor cleanup:** `src/commands/doctor/shared/plugin-dependency-cleanup.ts:37-46`
  - body: The cleanup treats plugin-local `node_modules` directories as legacy debris, but the PR's contract says local/path/archive plugins must bring preinstalled dependencies. Running `doctor --fix` can delete valid runtime deps and break those plugins after repair.
  - confidence: 0.9
- **[P2] Scan git plugin sources before npm install:** `src/plugins/git-install.ts:292-309`
  - body: This path runs package-manager work inside the cloned git repository before the plugin source safety scan. Even with lifecycle scripts disabled, dependency resolution and network activity now happen for unapproved source; keep source validation ahead of dependency convergence.
  - confidence: 0.84
- **[P2] Delete managed npm installs on uninstall:** `src/plugins/install.ts:1146-1147`
  - body: Npm plugins now install under `.openclaw/npm/node_modules/<pkg>`, but uninstall safety still only trusts the managed extensions layout before falling back to `extensions/<id>`. Uninstall can leave the actual package root behind and accumulate stale plugin code.
  - confidence: 0.86
