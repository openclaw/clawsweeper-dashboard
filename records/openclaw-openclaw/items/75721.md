## Summary

Keep open: this is a protected maintainer-labeled dependency-model PR, and the current head still has blocking correctness findings plus a concrete git-install trust-boundary concern. The later force-push fixed the earlier uninstall-safe managed npm root concern, but the SDK alias, doctor cleanup, and git install ordering problems remain on the reviewed head.

## What This Changes

The PR moves plugin dependency ownership to install-time npm/git roots, removes bundled/runtime dependency staging and repair paths, externalizes QQ Bot as downloadable, and updates related docs, Docker/CI lanes, scripts, tests, and release checks.

## Best Possible Solution

Revise or split the branch so the install-time dependency model preserves SDK subpath compatibility, removes only known OpenClaw-generated legacy artifacts, and validates git plugin source before package-manager work. After those fixes, rerun focused plugin install/doctor coverage and the broad changed gate in Testbox.

## Reproduction Assessment

Yes. The blockers have deterministic reproduction paths: import a generated alias for a named-only SDK subpath, run the cleanup collector against a plugin root with a legitimate `node_modules`, and trace git install from clone through npm install before OpenClaw validation.

## Solution Assessment

No. The direction may be maintainable, but this implementation is not the best fix until SDK compatibility, cleanup scope, and the git-install trust boundary are narrowed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve default-safe SDK alias wrappers:** `src/plugins/plugin-sdk-dist-alias.ts:13`
  - body: The wrapper now emits `export { default }` for every SDK subpath. Named-only SDK files such as `outbound-runtime` do not export default, so importing the generated alias can fail during ESM instantiation; keep the current namespace fallback or only default-forward modules that actually export one.
  - confidence: 0.94
- **[P1] Stop deleting valid plugin node_modules:** `src/commands/doctor/shared/plugin-dependency-cleanup.ts:39`
  - body: Doctor cleanup marks plugin-local `node_modules` under plugin roots as legacy debris. This branch also makes local/path/archive plugins rely on preinstalled dependencies, so `doctor --fix` can delete a valid dependency tree and break the plugin on the next load.
  - confidence: 0.91
- **[P2] Scan git plugins before npm install:** `src/plugins/git-install.ts:292-309`
  - body: This path runs npm dependency resolution in the cloned repository before `installPluginFromInstalledPackageDir` performs OpenClaw install validation. Even with lifecycle scripts disabled, dependency resolution and network work happen for unvalidated git plugin source; validate the source first.
  - confidence: 0.87
