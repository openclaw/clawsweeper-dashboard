## Summary

Keep open: this maintainer-labeled XL PR is not a cleanup-close candidate, and the proposed patch has blocking regressions in the packaged plugin SDK alias and new npm/git install-root model. Current main does not already implement this refactor, so the right outcome is revision on the PR rather than closure.

## What This Changes

This PR removes bundled runtime dependency staging and related commands/tests, moves npm and git plugin dependencies into OpenClaw-owned user state roots, externalizes QQ Bot as downloadable, and updates loader, doctor, Docker, CI, docs, and tests for install-time plugin dependency ownership.

## Best Possible Solution

Revise this PR before merge while preserving the intended install-time dependency model: keep startup/load-time package-manager work removed, but restore SDK alias compatibility, make npm/git install-record paths discoverable and loadable, and keep safety scans before package-manager work for git-sourced plugins.

## Reproduction Assessment

Yes. The SDK alias failure is reproducible with Node ESM by re-exporting `default` from a named-only module, and the npm/git registry gap follows directly from the PR's new install roots versus current discovery roots.

## Solution Assessment

No. The product direction may be maintainable, but this patch is not the best safe implementation until it preserves the old default-safe SDK wrappers, wires install records into discovery or registry candidates, and avoids running npm in untrusted git sources before source scanning.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve default-safe SDK alias wrappers:** `src/plugins/plugin-sdk-dist-alias.ts:12-13`
  - body: The generated alias now unconditionally re-exports `default` from every SDK subpath. Most `dist/plugin-sdk/*.js` files only have named exports, and ESM rejects that wrapper during module instantiation, so packaged bundled plugins importing named SDK helpers through `openclaw/plugin-sdk/*` can fail before registration. Keep the old namespace fallback or only re-export `default` when the source module actually has one.
  - confidence: 0.94
- **[P1] Add discovery for package install roots:** `src/plugins/install.ts:1146-1147`
  - body: Npm plugins are now installed under `~/.openclaw/npm/node_modules/<pkg>` and git plugins under `~/.openclaw/git/...`, but plugin discovery still scans `~/.openclaw/extensions`, workspace, bundled, and configured load paths. Install records are only used for precedence after a candidate is found, so a successful npm/git install can be recorded and enabled without appearing in the refreshed registry or Gateway load set.
  - confidence: 0.9
- **[P2] Scan git plugin sources before npm install:** `src/plugins/git-install.ts:292-309`
  - body: This path runs `npm install` inside the cloned git repository before `installPluginFromInstalledPackageDir` performs the package source safety scan. Even with lifecycle scripts disabled, a source that the scanner would reject can now trigger dependency resolution and network/package-manager work first; keep the pre-install validation/scan step ahead of dependency convergence for git installs.
  - confidence: 0.82
