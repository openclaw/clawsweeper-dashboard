## Summary

Keep open: current main does not contain this dependency-model rewrite, and the open PR is protected by the `maintainer` label while still carrying source-confirmed blocking correctness and security issues. The latest head changes SDK alias generation, doctor cleanup, git install order, and npm install roots in ways that can break plugin loading/cleanup or broaden pre-scan package-manager work, so this needs maintainer revision rather than cleanup closure.

## What This Changes

The PR moves plugin dependency ownership to install-time npm/git roots, removes bundled runtime dependency staging and repair paths, prefers native packaged JS loading, adds legacy cleanup and downloadable-plugin repair paths, and externalizes QQ Bot as a downloadable plugin.

## Best Possible Solution

Revise the branch while preserving the install-time dependency ownership direction: keep SDK alias compatibility, limit doctor cleanup to known OpenClaw-generated legacy artifacts, scan git plugin source before package-manager work, add uninstall coverage for npm/git managed roots, then rerun the changed plugin, doctor, Docker, and release-path gates.

## Reproduction Assessment

Yes. The blockers reproduce by deterministic source review: generate an alias for a named-only SDK subpath, follow the cleanup target collection for a plugin `node_modules`, and trace the git install order before `installPluginFromInstalledPackageDir` scans the source.

## Solution Assessment

No. The overall direction may be maintainable, but this implementation is not the best fix until it preserves existing SDK compatibility and keeps cleanup, uninstall, and source-scan trust boundaries intact.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve default-safe SDK alias wrappers:** `src/plugins/plugin-sdk-dist-alias.ts:13`
  - body: The new wrapper always emits `export { default }` for every SDK subpath. Named-only SDK files such as `outbound-runtime` do not export default, so importing the generated wrapper can fail during ESM instantiation; keep the current namespace fallback or only default-forward modules that actually have a default export.
  - confidence: 0.94
- **[P1] Stop deleting valid plugin node_modules:** `src/commands/doctor/shared/plugin-dependency-cleanup.ts:39`
  - body: Doctor cleanup now treats every plugin-local `node_modules` under extension roots as legacy debris. With this branch making plugin packages own their dependencies, `doctor --fix` can delete a valid runtime dependency tree and break the plugin on the next load.
  - confidence: 0.91
- **[P2] Scan git plugins before npm install:** `src/plugins/git-install.ts:292-309`
  - body: This path runs `npm install` in the cloned repository before the source safety scan in `installPluginFromInstalledPackageDir`. Even with lifecycle scripts disabled, OpenClaw is doing dependency resolution and network work for unapproved source; validate the plugin source before package-manager work.
  - confidence: 0.86
- **[P2] Teach uninstall to remove managed npm roots:** `src/plugins/install.ts:1157-1158`
  - body: Npm plugins now install under the OpenClaw npm root, but uninstall target resolution still only trusts extension-root managed paths before falling back to `extensions/<id>`. Uninstalling these plugins can leave the actual npm package root behind as stale code on disk.
  - confidence: 0.86
