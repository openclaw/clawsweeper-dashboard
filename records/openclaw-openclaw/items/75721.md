## Summary

Keep open: current main does not implement this broad dependency-handling refactor, and the open PR is protected by the `maintainer` label. Latest head still has blocking regressions in SDK alias compatibility, doctor cleanup scope, and the git-plugin install trust boundary, so it needs maintainer revision before merge.

## What This Changes

The PR moves plugin dependency ownership to install/update-time npm and git roots, removes bundled runtime dependency staging and `plugins deps`, externalizes QQ Bot as downloadable, and updates related docs, CI, Docker, installer, doctor, uninstall, release, and test surfaces.

## Best Possible Solution

Revise or split the branch so the install-time dependency model preserves SDK subpath compatibility, removes only known OpenClaw-generated legacy artifacts, validates git plugin source before npm work, and includes the required user-facing changelog before a broad Testbox changed gate.

## Reproduction Assessment

Yes. The blockers are reproducible by inspecting the generated alias for a named-only SDK subpath, collecting cleanup targets for a plugin root with `node_modules`, and tracing the git install order from clone to npm install to OpenClaw validation.

## Solution Assessment

No. The product direction may be viable, but this implementation is not the narrowest maintainable solution until SDK compatibility, cleanup scope, and the git-install trust boundary are corrected.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve default-safe SDK alias wrappers:** `src/plugins/plugin-sdk-dist-alias.ts:13`
  - body: The wrapper now emits `export { default }` for every SDK subpath. Named-only SDK files such as `outbound-runtime` do not export default, so importing the generated alias can fail during ESM instantiation; keep the current namespace fallback or only default-forward modules that actually export one.
  - confidence: 0.94
- **[P1] Stop deleting valid plugin node_modules:** `src/commands/doctor/shared/plugin-dependency-cleanup.ts:39`
  - body: Doctor cleanup marks plugin-local `node_modules` under plugin roots as legacy debris. This branch also makes local/path/archive plugins rely on preinstalled dependencies, so `doctor --fix` can delete a valid dependency tree and break the plugin on the next load.
  - confidence: 0.91
- **[P2] Scan git plugins before npm install:** `src/plugins/git-install.ts:292-309`
  - body: This path runs npm dependency resolution in the cloned repository before `installPluginFromInstalledPackageDir` performs OpenClaw install validation. Even with lifecycle scripts disabled, dependency resolution and network work happen for unvalidated git plugin source; validate the source first.
  - confidence: 0.88
