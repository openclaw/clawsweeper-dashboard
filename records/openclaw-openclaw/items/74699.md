## Summary

Current main contains a partial completion-flush fix for the original debounce race, but it still treats Codex steering as accepted before delivery and still swallows rejected `turn/steer` requests, so the PR remains a relevant implementation candidate that needs maintainer reconciliation with current main.

The PR makes Codex app-server queued steering report delivery success, adds a durable embedded-run queue helper, awaits it in `runReplyAgent()` for followup fallback, and adds regression coverage plus a changed-target routing test adjustment.

## Best Possible Solution

Land a rebased narrow fix that makes every default Codex queued user message either reach accepted live `turn/steer` or fall back through the existing followup queue when the turn is terminal or steering rejects.

Yes. A high-confidence source-level path is a default `steer` message queued during an active Codex app-server run where `turn/steer` is delayed until completion or rejects; current main acknowledges the message synchronously and has no delivery-result path back to followup queueing.

## Solution Assessment

Unclear as-is. Durable delivery is the right direction, but the branch must be reconciled with the current completion-flush implementation so successful flushes are preserved and rejected flushes fall back cleanly.
