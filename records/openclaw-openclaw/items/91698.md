## What This Changes

## Best Possible Solution

Land a Feishu-local final-chunk merge that preserves complete re-render behavior without dropping same-prefix deltas, then verify it with live Feishu CardKit/DeepSeek proof or an explicit maintainer proof override.

## Reproduction Assessment

Yes from source inspection: current main and v2026.6.1 overwrite `streamText` on each active Feishu streamed `final` payload, so the reported incremental-final sequence closes with only the last fragment. I did not run a live Feishu tenant reproduction in this read-only review.

## Solution Assessment

No as submitted. The Feishu channel buffer is the right owner boundary, but the shared-prefix re-render heuristic is too broad and should be fixed before live proof or a proof override is accepted.
