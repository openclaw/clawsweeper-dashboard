## Summary

Keep this open: current main still treats unmarked or localized Gemini reasoning preambles as visible assistant text, and the newer Gemini incomplete-turn retry is only a partial mitigation. The open PR #60551 remains the implementation candidate rather than proof that the issue is fixed.

## What This Changes

Gemini reasoning or planning text should be hidden from user-visible channel replies, including plain or localized preambles that are not provider-marked thought parts or XML thinking tags.

## Best Possible Solution

Keep this as the Gemini/plain-text reasoning leakage tracker and review PR #60551 or a replacement that adds conservative shared outbound sanitization with regression coverage for the reported Gemini examples, localized headers, tagged fragments, normal prose/code pass-through, and Telegram plus generic message-tool delivery paths.

## Reproduction Assessment

Yes for the source-level path: an unmarked Gemini `part.text` containing `Thinking Process:` or a localized reasoning header is emitted as visible text and is not removed by the current shared or Telegram sanitizers. I did not run a live Gemini/Telegram repro, but the current source path is enough to reproduce this leak shape without network access.

## Solution Assessment

Yes, keeping this open and routing through #60551 or a replacement is the best current path. The narrow maintainable fix belongs in shared outbound sanitization while provider-flagged thought parts and XML reasoning tags should continue using the existing paths.
