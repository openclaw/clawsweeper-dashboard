## Summary

Keep this open. Current main hides provider-marked Gemini thought parts and XML-style reasoning tags, but it still has no conservative classifier for unmarked/localized natural-language reasoning preambles like the reported `Thinking Process:`, `Analysis:`, `Tankegang:`, or `The SOUL.md says...` shapes; the related open PR #60551 remains the implementation candidate rather than proof that the issue is fixed.

## What This Changes

The issue requests preventing Gemini reasoning or planning text from leaking as visible assistant output in delivered chat messages, especially Telegram.

## Best Possible Solution

Keep this as the Gemini/plain-text reasoning leakage tracker and review PR #60551 or a replacement that adds conservative shared outbound sanitization for untagged reasoning/planning preambles, with regression coverage for the reported Gemini examples, localized headers, tagged fragments, code/prose pass-through, normal `/reasoning` visibility, and Telegram plus generic message-tool delivery paths.

## Reproduction Assessment

Yes for the source-level path: an unmarked Gemini `part.text` containing `Thinking Process:` or a localized reasoning header is treated as normal visible text because current main only hides `part.thought === true`, exact `Reasoning:\n` payloads, or XML-style tags. I did not run a live Gemini/Telegram repro, but the current source path is enough to reproduce the leak shape without network access.

## Solution Assessment

Yes, keeping this open and routing through the existing sanitizer PR is the best current path. The narrow maintainable fix belongs in a shared delivery sanitizer rather than a Telegram-only workaround, while provider-flagged thought parts should continue to use the existing Google stream classification.
