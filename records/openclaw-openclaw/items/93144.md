## What This Changes

## Best Possible Solution

Keep this issue open while maintainers review the linked candidate PR and settle a transport-layer design that preserves stdio env safety, OAuth/header scoping, and documented MCP reload semantics.

Unclear. A fetch-wrapper approach is plausible because the pinned MCP SDK supports custom fetch hooks, but the best fix still needs maintainer review for security, stdio constraints, OAuth scoping, and config snapshot semantics.
