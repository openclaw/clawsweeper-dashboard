The issue requests dynamic `${ENV_VAR}` re-evaluation for MCP `headers` and stdio `env` at request, connection, or spawn time instead of config-load time.

Keep the issue open while maintainers review the linked PR and decide whether MCP token rotation should be implemented through dynamic env substitution, OAuth/client_credentials, SecretRef, or a narrower approved combination.
