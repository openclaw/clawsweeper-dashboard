Keep open: current main and the latest release still resolve `${ENV_VAR}` during config activation and pass fixed MCP header/env values into transports, while the open implementation candidate still needs maintainer security/product review.

## What This Changes

Request dynamic re-evaluation of MCP `${ENV_VAR}` templates in headers and env at request, connection, or spawn time instead of only at config activation.

## Best Possible Solution

Settle one owner-approved MCP dynamic credential contract, then either land a narrowed version of https://github.com/openclaw/openclaw/pull/94167 with security/docs/tests or close it in favor of OAuth/SecretRef and the broader remote-header design.
