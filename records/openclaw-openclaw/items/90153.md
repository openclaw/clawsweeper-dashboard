## Summary

Keep open: the central doctor hook failure still exists on current main, and this maintainer-authored draft is protected from cleanup closing. The patch looks like a valid bounded mitigation, but the stricter plugin install recovery behavior is compatibility-sensitive and now overlaps with the broader open hardening work at https://github.com/openclaw/openclaw/pull/92014.

## What This Changes

The branch wraps channel doctor hooks so plugin callback failures become channel-scoped warnings, surfaces warning-only stale cleanup diagnostics, and makes plugin install recovery reject stale cleanup warnings.

## Best Possible Solution

Resolve the overlap with https://github.com/openclaw/openclaw/pull/92014, rebase against current main, and land one maintainer-approved path that preserves doctor warning surfacing while explicitly accepting or revising the install recovery blocking behavior.

## Reproduction Assessment

Yes. From current-main source inspection, several channel doctor adapter hooks are invoked without per-entry failure isolation, so a throwing plugin callback can abort doctor or install recovery before sibling diagnostics are collected; I did not run tests in this read-only review.

## Solution Assessment

Yes, as a bounded mitigation: catching at the channel doctor adapter boundary keeps plugin-owned hook failures from aborting core doctor flows. The install recovery stop remains a compatibility decision, and the broader https://github.com/openclaw/openclaw/pull/92014 may be the better canonical landing path for the shared hook-hardening work.
