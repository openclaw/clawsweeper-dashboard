## What This Changes

The issue asks for `memory_search` to surface gateway-dispatched session transcript content when experimental session memory and session sources are enabled.

## Best Possible Solution

Define the transcript recall contract, then either intentionally wire QMD export and visibility for gateway-style same-agent recall with regression coverage, or keep the conservative gates and land clear doctor/docs diagnostics.

## Reproduction Assessment

Yes, at source level. Current main still requires separate QMD session export and session-tool visibility gates, so `experimental.sessionMemory` plus `sources: ["sessions"]` alone does not make unrelated gateway sessions visible from a DM query.

## Solution Assessment

Unclear. The requested continuity is valid, but an unconditional behavior change is not the narrowest safe fix until maintainers choose the transcript export, visibility, diagnostics, and cost policy.
