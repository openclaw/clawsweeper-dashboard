## Summary

The issue should stay open because current main still has several independent gates that can make indexed session transcripts disappear from the agent `memory_search` path: experimental session sources, QMD transcript export, and privacy/session visibility filtering. Narrow related fixes and docs exist, but they do not resolve the broader gateway-dispatched session recall contract.

## What This Changes

The issue requests that enabling experimental session memory make gateway-dispatched session transcripts available through the agent `memory_search` tool when visibility permits.

## Best Possible Solution

Define the session recall contract first, then land narrow fixes and diagnostics that preserve privacy defaults while making configured same-agent session hits reliably explainable and searchable.

## Reproduction Assessment

Yes, source inspection supports the reported failure class: current main can index sessions while the agent tool later drops session hits through visibility/export gates, but I did not run the original live Gemini/QMD/gateway reproduction.

## Solution Assessment

No single code-only fix is clearly best yet because the remaining behavior crosses privacy defaults, QMD export policy, gateway session ownership, and user-facing diagnostics.
