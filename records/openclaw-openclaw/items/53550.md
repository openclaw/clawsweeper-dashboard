## Summary

Keep open. Current main still leaves the gateway/QMD session recall path split across separate memory-source, QMD export, and session-visibility gates, while the related open items only cover narrower diagnostics or the archived-path sub-bug.

## What This Changes

The issue requests that `experimental.sessionMemory` with session sources return gateway-dispatched session transcript content through `memory_search`, including QMD-backed setups.

## Best Possible Solution

Keep this issue open for the gateway/QMD recall contract, review the diagnostics path in https://github.com/openclaw/openclaw/pull/80947, land or close the narrower archived-path fix in https://github.com/openclaw/openclaw/pull/83518, and make any broader default or visibility change only after a maintainer privacy decision.

## Reproduction Assessment

Yes, at source level. With QMD backend and only `experimental.sessionMemory` plus `sources: ["memory", "sessions"]`, QMD transcript export remains disabled; when QMD sessions are enabled, session hits are still filtered by `tools.sessions.visibility` and sandbox visibility.

## Solution Assessment

Unclear as a single automatic fix. Narrow bugs such as archived QMD path mapping can be fixed directly, but the broader gateway-to-DM recall behavior needs an explicit maintainer decision on transcript export, visibility, diagnostics, and privacy defaults.
