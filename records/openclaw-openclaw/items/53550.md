## Summary

Keep open. Current main still leaves the reported QMD/gateway recall path unresolved: builtin session memory and QMD transcript export remain separate switches, and the default session visibility still hides unrelated same-agent sessions from a DM query. The related #51386 is a broader graduation/policy item, not a terminal duplicate for this concrete QMD plus gateway behavior.

## What This Changes

The issue requests that `experimental.sessionMemory` make `memory_search` return gateway-dispatched session transcript content across sessions, including with QMD/Gemini-backed memory search.

## Best Possible Solution

Define one explicit transcript recall contract: either make QMD honor `experimental.sessionMemory` plus `sources: ["sessions"]` with clear export/status logs, or keep QMD opt-in separate while surfacing actionable diagnostics and docs for `memory.qmd.sessions.enabled: true` plus an intentional visibility setting such as `tools.sessions.visibility: "agent"`. Add regression coverage for a gateway-style same-agent session queried from an unrelated DM session.

## Reproduction Assessment

Yes. A source-level reproduction is high confidence: with `memory.backend: "qmd"` and only `experimental.sessionMemory: true`, QMD resolves `sessions.enabled` to false, so no session collection/export is created; with default `tools.sessions.visibility: "tree"`, unrelated same-agent session hits are filtered even if indexed.

## Solution Assessment

Unclear. The requested behavior is valid, but the narrowest maintainable fix needs maintainer judgment on whether QMD should follow the experimental session-memory flag automatically or remain a separate transcript-export opt-in with stronger diagnostics and documentation.
