## Summary

Keep open. Current main still leaves the reported gateway-to-DM recall path unresolved: `experimental.sessionMemory` only enables the builtin session source, QMD transcript export is still a separate `memory.qmd.sessions.enabled` opt-in, and session hits are filtered by the default `tree` visibility policy before they can surface in `memory_search`.

## What This Changes

The issue asks for gateway-dispatched session transcripts to be indexed and returned by `memory_search` when `agents.defaults.memorySearch.experimental.sessionMemory` is enabled with `sources: ["memory", "sessions"]`.

## Best Possible Solution

Define one explicit transcript recall contract: either make QMD honor `experimental.sessionMemory` plus `sources: ["sessions"]` with session export, logging, docs, and regression coverage, or keep QMD opt-in separate while surfacing actionable diagnostics/docs requiring `memory.qmd.sessions.enabled: true` and an intentional visibility setting such as `tools.sessions.visibility: "agent"` for same-agent gateway-to-DM recall.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: with `memory.backend: "qmd"` and only `experimental.sessionMemory: true`, QMD resolves `sessions.enabled` to false, so no session export collection is configured; with default `tools.sessions.visibility: "tree"`, unrelated same-agent gateway hits are filtered even if indexed.

## Solution Assessment

Unclear. The requested behavior is valid, but the best maintainable fix depends on whether maintainers want the experimental flag to implicitly enable QMD transcript export or instead preserve QMD as a separate opt-in with stronger diagnostics, docs, and explicit visibility configuration.
