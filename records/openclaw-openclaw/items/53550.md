## Summary

Keep open. Current main still leaves the gateway-to-DM recall path split across the experimental session source flag, the separate QMD transcript export gate, and session-tool visibility filtering, so closing this as implemented would hide an unresolved product/privacy decision.

## What This Changes

The issue requests that gateway-dispatched session transcripts appear in `memory_search` when session memory and session sources are enabled.

## Best Possible Solution

Define the gateway/QMD session-recall contract explicitly, land or close the diagnostics path in https://github.com/openclaw/openclaw/pull/80947, keep the archived-path fix from https://github.com/openclaw/openclaw/pull/83518, and make any broader export or visibility change only after maintainer privacy review.

## Reproduction Assessment

Yes, at source level. With QMD backend, `experimental.sessionMemory` plus `sources: ["memory", "sessions"]` does not enable QMD transcript export, and when QMD sessions are enabled the tool still filters hits through `tools.sessions.visibility` and sandbox visibility.

## Solution Assessment

Unclear as a single automatic fix. The narrow archived-path bug is already fixed separately, while broader gateway-to-DM recall needs an explicit maintainer decision on transcript export, visibility defaults, diagnostics, and privacy boundaries.
