## Summary

Keep open. Current main and the latest shipped release still leave this recall path split across session-memory sources, QMD transcript export, session-tool visibility, sandbox clamping, and a newer archived-path mapping bug, so there is not a clean close reason yet.

## What This Changes

The issue asks for gateway-dispatched session transcripts to be searchable through `memory_search` when session memory and session sources are enabled.

## Best Possible Solution

Keep this as the broader gateway/session-memory recall tracker until maintainers define the recall contract, land or close the diagnostic path in https://github.com/openclaw/openclaw/pull/80947, and handle the focused archived-path bug in https://github.com/openclaw/openclaw/issues/83506.

## Reproduction Assessment

Yes, at source level. With QMD backend and only `experimental.sessionMemory`, QMD export remains disabled; with QMD export enabled, session hits still pass through visibility and transcript-key resolution gates that can hide unrelated or unmapped sessions.

## Solution Assessment

Unclear as a single implementation change. The maintainable path is to keep visibility conservative, add diagnostics/docs for the current gates, fix narrow mapping bugs separately, and let maintainers decide any broader recall/default change.
