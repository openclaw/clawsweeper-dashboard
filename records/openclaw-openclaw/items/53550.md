## Summary

Keep open. Current main has fixed the archived QMD path sub-bug via the merged narrower PR, but the central gateway-to-DM recall problem still spans separate QMD export, memory source, session visibility, and sandbox gates; the remaining behavior needs maintainer privacy/product review rather than cleanup closure.

## What This Changes

The issue asks for `memory_search` with session memory enabled to return gateway-dispatched session transcript content across sessions, including QMD-backed setups.

## Best Possible Solution

Keep the issue open until maintainers define the transcript recall contract: either document and diagnose the required opt-in combination, or intentionally bridge session-memory and QMD/session visibility behavior while preserving privacy and adding gateway-style regression coverage.

## Reproduction Assessment

Yes, at source level. With QMD backend and only `experimental.sessionMemory` plus `sources: ["memory", "sessions"]`, QMD transcript export remains disabled; when QMD sessions are enabled, unrelated same-agent gateway hits are still filtered by `tools.sessions.visibility` and sandbox visibility.

## Solution Assessment

Unclear as an automatic fix. The archived-path sub-bug is fixed, but the broader gateway-to-DM recall behavior needs an explicit maintainer decision on QMD export defaults, diagnostics, visibility semantics, and privacy.
