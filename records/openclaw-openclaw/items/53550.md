## Summary

Keep open. Current main still leaves the reported QMD gateway-to-DM recall path unresolved: `experimental.sessionMemory` enables the builtin session source, but QMD transcript export remains a separate disabled-by-default switch, and session-sourced hits still pass through `tools.sessions.visibility`, whose default `tree` policy hides unrelated same-agent gateway sessions from a DM query.

## What This Changes

The issue asks for `memory_search` with `experimental.sessionMemory` and `sources: ["memory", "sessions"]` to include gateway-dispatched session transcripts, including with QMD/Gemini-backed memory search.

## Best Possible Solution

Define the transcript recall contract explicitly: either make QMD honor `experimental.sessionMemory` plus `sources: ["sessions"]` with clear export/status diagnostics, or keep QMD opt-in separate while surfacing actionable docs and diagnostics for `memory.qmd.sessions.enabled: true` plus an intentional same-agent visibility setting such as `tools.sessions.visibility: "agent"`; then add a gateway-style same-agent DM regression test.

## Reproduction Assessment

Yes, at source level. With QMD backend and only `experimental.sessionMemory: true`, QMD session export stays disabled, and default `tree` visibility filters unrelated same-agent gateway sessions from a DM query even if session hits exist.

## Solution Assessment

Unclear. The requested behavior is valid, but the best fix depends on maintainer policy for QMD export defaults, cross-session privacy, diagnostics, and cost behavior rather than a narrow bug-only patch.
