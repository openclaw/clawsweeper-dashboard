## Summary

Keep open. Current main still leaves the reported QMD gateway-to-DM recall path unresolved because builtin session-memory sources, QMD transcript export, and session-tool visibility remain separate gates, while related PRs only document or diagnose the mismatch so far.

## What This Changes

The issue asks for `memory_search` with experimental session memory to return gateway-dispatched session transcript content across sessions, including QMD-backed setups.

## Best Possible Solution

Define the transcript recall contract, then either keep the conservative gates with clear docs/doctor diagnostics or intentionally widen QMD export and visibility with gateway-style regression coverage.

## Reproduction Assessment

Yes, at source level. With QMD backend and only `experimental.sessionMemory: true`, QMD session export stays disabled, and default `tree` visibility filters unrelated same-agent gateway sessions from a DM query even if session hits exist.

## Solution Assessment

Unclear. The requested continuity is valid, but the best fix depends on maintainer policy for transcript export defaults, cross-session privacy, diagnostics, and cost rather than a narrow bug-only patch.
