## What This Changes

The issue reports that opt-in session memory search can fail to surface gateway-dispatched session transcript hits through the agent `memory_search` tool even when backend or CLI search can see the sessions.

## Best Possible Solution

Keep the issue open until maintainers land or reject the targeted live-orphan visibility fix, then finish the diagnostic/docs path without weakening the default transcript privacy model.

## Reproduction Assessment

Yes, at source level: a `sessions/main/live-orphan.jsonl` hit with an empty combined session store has owner metadata, but current main only synthesizes fallback keys for archived hits, so the live hit is filtered out. I did not run a live gateway repro in this read-only pass.

## Solution Assessment

No, current main is not the complete solution yet. The best fix is a narrow owner-scoped live fallback that preserves the existing visibility guard, plus diagnostics/docs for the QMD/session recall gates rather than broadening defaults.
