## Summary

Keep open: current main and the latest release still split session recall across session-source opt-in, QMD transcript export, and post-search visibility filtering, and current tests explicitly keep live mapped QMD orphan session hits dropped when no session-store key remains. The linked merged QMD archive fix covers one narrower path, while the open live-orphan and diagnostic PRs have not yet resolved the full gateway-to-agent recall contract.

## What This Changes

The issue asks OpenClaw to make session-backed memory_search surface gateway-dispatched session transcripts across sessions when session memory is enabled, with clear behavior for QMD, builtin indexing, visibility, and diagnostics.

## Best Possible Solution

Keep this as the canonical concrete bug until maintainers define the privacy-preserving session recall contract, land or reject the narrow live-orphan and diagnostic PRs, and verify a redacted live QMD/Gateway path from a gateway-dispatched session to an unrelated DM query.
