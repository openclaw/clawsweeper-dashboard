## Summary

Keep open. Current main still leaves the reported recall path split between the experimental memory-search source flag, a separate QMD transcript-export gate, and session-tool visibility filtering, so the remaining behavior needs maintainer product/security review rather than cleanup closure.

## What This Changes

The issue requests that gateway-dispatched session transcripts become discoverable through `memory_search` across sessions, including QMD-backed setups.

## Best Possible Solution

Define the gateway/QMD session-recall contract explicitly, land or close the diagnostics path in https://github.com/openclaw/openclaw/pull/80947, keep the archived-path fix from https://github.com/openclaw/openclaw/pull/83518, and make any broader export or visibility change only after a maintainer privacy decision.

## Reproduction Assessment

Yes, at source level. With QMD backend, `experimental.sessionMemory` plus `sources: ["memory", "sessions"]` does not enable QMD transcript export, and when QMD sessions are enabled the tool still filters hits through `tools.sessions.visibility` and sandbox visibility.

## Solution Assessment

Unclear as a single automatic fix. The narrow archived-path bug is already fixed separately, while the broader gateway-to-DM recall behavior needs an explicit maintainer decision on transcript export, visibility defaults, diagnostics, and privacy boundaries.
