## Summary

Keep open: current main still has the reported limit edge case in the diagnostic phase helper, and the fix is narrow enough for a focused bug-fix PR rather than a close-out.

## What This Changes

The issue reports that `getRecentDiagnosticPhases(0)` returns the full recent phase buffer instead of an empty list.

## Best Possible Solution

Normalize the limit once, return an empty array unless it is finite and positive, then keep positive limits returning the most recent snapshots with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: record phases, call `getRecentDiagnosticPhases(0)`, and the current `slice(-Math.max(0, limit))` expression returns the retained buffer.

## Solution Assessment

Yes. The narrow maintainable fix is to guard for a finite positive limit before slicing, without changing diagnostic event schema or liveness warning formatting.
