## What This Changes

The issue asks for `getRecentDiagnosticPhases(limit)` to return an empty array for zero, negative, or non-finite limits while preserving positive-limit behavior.

## Best Possible Solution

Normalize the limit once, return an empty array unless it is finite and positive, and keep positive limits returning copied snapshots of the most recent phases.

## Reproduction Assessment

Yes. Source inspection on current main plus a Node slice check gives a high-confidence path: record phases, call `getRecentDiagnosticPhases(0)`, and the retained buffer is returned.

## Solution Assessment

Yes. The narrow maintainable fix is to guard for a finite positive limit before slicing, without changing the diagnostic event schema or liveness warning formatting.
