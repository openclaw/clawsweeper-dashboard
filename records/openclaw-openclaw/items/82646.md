## Summary

Keep open: current `main` still has the reported edge case, and an open linked fix PR is already attached, so this issue should close only after that PR merges or is replaced.

## What This Changes

`getRecentDiagnosticPhases` should return an empty array for zero, negative, or non-finite limits instead of returning the retained diagnostic phase buffer.

## Best Possible Solution

Review and land the linked narrow fix, or replace it with an equivalent guard that rejects non-finite and non-positive limits before slicing and adds direct regression coverage.

## Reproduction Assessment

Yes. Source inspection on current `main` shows the helper still uses `slice(-Math.max(0, limit))`, and a focused Node check confirms the reported JavaScript slice behavior for zero, negative, and non-finite inputs.

## Solution Assessment

Yes. The narrow maintainable fix is to return `[]` unless the limit is finite and positive, then keep positive limits returning the most recent snapshots; the open linked PR follows that direction but still needs normal PR review before merge.
