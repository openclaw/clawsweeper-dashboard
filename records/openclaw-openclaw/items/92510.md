## Summary

The shipped gateway behavior can silently route an explicit unknown selector into phantom agent-scoped state, and this PR applies a coherent shared fix across all affected OpenAI-compatible endpoints. The patch has no concrete correctness finding, but it is not merge-ready because the external-contributor proof is repository test-harness output rather than after-fix requests against an actual configured gateway, and maintainers must explicitly accept the intentional compatibility change from successful fallback to HTTP 400.

## What This Changes

The PR validates explicit OpenAI-compatible header and model agent selectors against the effective configured roster and maps unknown selectors to structured HTTP 400 responses across chat completions, responses, and embeddings.

## Best Possible Solution

Keep one shared effective-roster check in `resolveAgentIdForRequest`, preserve omitted and `openclaw/default` routing, map the typed failure consistently at every OpenAI-compatible HTTP boundary, and demonstrate unknown, default, and configured selectors against a real gateway before merge.
