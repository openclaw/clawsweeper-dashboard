## Summary

Keep open: current main still rejects keyless Bedrock `aws-sdk` auth in the media-understanding image/fallback and audio-video paths, and the focused open PR that explicitly closes this issue has not landed yet.

## What This Changes

Allow media-understanding image and audio/video Bedrock requests using `auth: "aws-sdk"` to proceed without a literal API key, matching chat completion auth behavior.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/72092, or an equivalent narrow fix, so empty `aws-sdk` media auth can reach Bedrock while other auth modes keep strict missing-key errors.

## Reproduction Assessment

Yes. Source inspection on current main is high-confidence: an image-capable `amazon-bedrock` model with `auth: "aws-sdk"` and no `apiKey` reaches strict `requireApiKey` in the media path before the AWS SDK credential chain can run.

## Solution Assessment
