## Summary

Keep open. Current main still has the reported media-understanding auth gap, and the active linked fix PR is open rather than merged.

## What This Changes

The issue requests that Bedrock `aws-sdk` auth work for image and related media-understanding paths without requiring a static API key.

## Best Possible Solution

Land #72092 or an equivalent narrow fix that allows empty `aws-sdk` auth only in the affected media paths, preserves strict errors for other auth modes, avoids writing empty runtime secrets, and adds regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows Bedrock `aws-sdk` auth can resolve with no `apiKey`, then the image and media paths call `requireApiKey`, which throws before the provider call.

## Solution Assessment

Yes. The linked PR direction is the narrow maintainable fix because it aligns media-understanding with the existing chat `aws-sdk` allowance instead of weakening `requireApiKey` globally.
