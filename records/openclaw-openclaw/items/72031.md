## Summary

Current main still has the reported Bedrock `aws-sdk` media-auth failure, and the linked implementation PR remains open and unmerged, so this issue is still necessary as the canonical tracker.

## What This Changes

The issue reports that Bedrock media-understanding tools reject `auth mode: aws-sdk` because API-key-only guards run before AWS SDK credentials can resolve.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/72092 with a focused current-main fix that lets Bedrock `aws-sdk` media-understanding calls proceed without a static API key while preserving non-`aws-sdk` missing-key errors and retry behavior.

## Reproduction Assessment

Yes. Source inspection is enough to reproduce the failure path: configure Bedrock with `auth.mode: "aws-sdk"` and no literal `apiKey`, then invoke image or media-understanding so `requireApiKey` throws before SDK credential resolution.

## Solution Assessment

Yes, the linked fix direction is the right shape: extend the existing chat-path missing-key allowance to media-understanding, without adding a new config option or asking users to convert AWS credentials into OpenClaw API keys.
