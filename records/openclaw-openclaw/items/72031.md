## Summary

Keep open. Current main still sends Bedrock `aws-sdk` media-understanding auth through strict API-key checks, and open PR #72092 is already linked with closing syntax as the active implementation candidate.

## What This Changes

The issue asks for Bedrock media-understanding image and audio/video auth to allow keyless `aws-sdk` credentials the same way chat/completion does.

## Best Possible Solution

Media auth resolvers should permit empty credential material only for resolved `aws-sdk` auth while preserving strict missing-key errors for every other mode.

## Reproduction Assessment

Yes. Configure an `amazon-bedrock` image model with `auth: "aws-sdk"` and no `apiKey`; current main reaches `requireApiKey` in `resolveImageRuntime` before the AWS SDK credential chain can run.

## Solution Assessment

Yes. Mirroring the chat path's missing-key allowance in the media auth resolvers is the narrowest maintainable fix; relaxing `requireApiKey` globally would broaden behavior beyond the Bedrock `aws-sdk` case.
