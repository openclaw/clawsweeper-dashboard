## Summary

Keep open. Current main still routes Bedrock `aws-sdk` media-understanding auth through strict API-key checks, and the open unmerged PR #72092 has closing syntax for this issue, so cleanup should wait for that PR to land or be superseded.

## What This Changes

The issue asks for media-understanding image and provider-execution auth to allow Bedrock `aws-sdk` credential-chain auth without requiring a literal API key.

## Best Possible Solution

Land or supersede #72092 with a narrow media-understanding fix that permits empty API-key material only for `aws-sdk` auth, avoids storing empty runtime keys, preserves strict missing-key errors for other modes, and adds focused regression coverage for Bedrock image plus provider-execution auth.

## Reproduction Assessment

Yes. Configure a Bedrock image model with `auth: "aws-sdk"` and no literal `apiKey`, then invoke the image tool; current main reaches `requireApiKey` in `resolveImageRuntime`, which deterministically throws before the AWS SDK credential chain can run.

## Solution Assessment

Yes. Mirroring the chat path's missing-key allowance inside the media auth resolvers is the narrowest maintainable fix; changing `requireApiKey` globally would relax behavior outside the reported Bedrock `aws-sdk` case.
