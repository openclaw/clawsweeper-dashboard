## Summary

Keep this issue open: current main and latest release still route Bedrock media-understanding auth through strict API-key guards, and the linked closing PR is open rather than merged.

## What This Changes

The issue asks for Bedrock media-understanding tools to accept `aws-sdk` auth mode without a static API key, matching chat/completion behavior.

## Best Possible Solution

Review and land, replace, or extend https://github.com/openclaw/openclaw/pull/72092 so Bedrock media-understanding paths can use keyless `aws-sdk` auth while preserving strict missing-key errors for non-`aws-sdk` modes.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure Bedrock with `auth: "aws-sdk"` and no literal `apiKey`, then invoke an affected media-understanding path so `requireApiKey` throws before SDK credentials can resolve; issue comments also include live reports on 2026.5.20 and 2026.5.28.

## Solution Assessment

Partly. The linked PR is the best visible candidate for the original image/audio/video bug, but later PDF reports mean the final solution should either cover that helper path too or leave a clear follow-up before this issue closes.
