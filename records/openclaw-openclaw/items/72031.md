## Summary

Keep open. Current main still routes Bedrock `aws-sdk` media auth through unconditional API-key requirements, and the linked closing PR #72092 is still open rather than merged.

## What This Changes

The issue reports that Bedrock image/media-understanding calls using `auth: "aws-sdk"` fail because media resolvers require a static API key even though chat allows SDK-resolved credentials.

## Best Possible Solution

Land or supersede #72092 with a narrow media-understanding fix: allow missing API-key material only for `aws-sdk` auth in the image and media provider auth resolvers, avoid storing empty runtime secrets, preserve missing-key failures for every other auth mode, and add regression coverage for the Bedrock image path plus provider execution auth.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: configure an `amazon-bedrock` image-capable model with `auth.mode === "aws-sdk"` and no `apiKey`, then call the image tool; current main reaches `requireApiKey` and throws before AWS SDK credentials can be resolved.

## Solution Assessment

Yes. Mirroring the chat pathŌĆÖs missing-key allowance in media-specific auth resolution is the narrow maintainable fix; weakening `requireApiKey` globally would broaden behavior beyond the Bedrock `aws-sdk` case.
