## Summary

Keep open. Current main still routes Bedrock `aws-sdk` media-understanding auth through strict literal API-key checks, and open PR #72092 is the active closing implementation candidate.

## What This Changes

The report asks for Bedrock `aws-sdk` auth to work in the image and related media-understanding paths without requiring a literal API key.

## Best Possible Solution

Review, land, or supersede PR #72092 with a narrow media-auth fix that permits empty key material only for resolved `aws-sdk` auth while preserving strict missing-key errors for other modes.

## Reproduction Assessment

Yes. Configure an `amazon-bedrock` image model with `auth: "aws-sdk"` and no `apiKey`; current main reaches `requireApiKey` in `resolveImageRuntime` before the AWS SDK credential chain can run.

## Solution Assessment

Yes. Mirroring the chat path's missing-key allowance in the media auth resolvers is the narrowest maintainable direction; relaxing `requireApiKey` globally would broaden behavior beyond the Bedrock `aws-sdk` case.
