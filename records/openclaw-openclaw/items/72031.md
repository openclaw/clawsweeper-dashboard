## Summary

Keep open. Current main still routes Bedrock `aws-sdk` media-understanding auth through strict API-key checks, and open PR #72092 uses closing syntax for this issue but has not merged.

## What This Changes

Allow Bedrock media-understanding image and audio/video calls to use `aws-sdk` auth without a static API key, matching the chat path.

## Best Possible Solution

Review and land or supersede PR #72092 with a narrow media-auth fix that permits empty credential material only for resolved `aws-sdk` auth, plus regression coverage for image and audio/video auth resolution.

## Reproduction Assessment

Yes. Configure an `amazon-bedrock` image model with `auth: "aws-sdk"` and no `apiKey`; current main reaches `requireApiKey` in `resolveImageRuntime` before the AWS SDK credential chain can run.

## Solution Assessment

Yes. Mirroring the chat pathŌĆÖs missing-key allowance in the media auth resolvers is the narrowest maintainable fix; relaxing `requireApiKey` globally would broaden behavior beyond the Bedrock `aws-sdk` case.
