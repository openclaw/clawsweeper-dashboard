## Summary

Keep open. Current main still sends Bedrock `aws-sdk` media-understanding auth through unconditional API-key checks, and #72092 is an open unmerged PR with closing syntax for this issue, so cleanup should wait for that PR to land or be superseded.

## What This Changes

Allow Bedrock `aws-sdk` auth without a static API key in the image/media-understanding paths so AWS SDK credentials can be resolved at provider call time.

## Best Possible Solution

Land or supersede #72092 with a narrow media-understanding fix that permits empty API-key material only for `aws-sdk` auth, avoids storing empty runtime keys, preserves missing-key errors for other modes, and adds regression coverage for Bedrock image plus provider-execution auth.

## Reproduction Assessment

Yes. Configure a Bedrock image model so auth resolves to `mode: "aws-sdk"` with no `apiKey`, then invoke the image tool; current main reaches `requireApiKey` in `resolveImageRuntime` before the AWS SDK credential chain can run.

## Solution Assessment

Yes. Mirroring the chat path's missing-key allowance inside the media auth resolvers is the narrowest maintainable fix; changing `requireApiKey` globally would broaden behavior beyond the reported Bedrock `aws-sdk` case.
