- `impact:auth-provider`: The failure is in provider authentication: `aws-sdk` mode is routed through API-key-only guards before credential-chain auth can run.

## Summary

Current main and the latest release still route Bedrock media-understanding auth through strict API-key guards, while the open linked PR remains unmerged, so this issue is still necessary as the canonical tracker.

## What This Changes

The issue reports that Bedrock image/PDF/media-understanding tools fail in `aws-sdk` auth mode because they require a static API key before AWS SDK credentials can resolve.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/72092 with a focused fix that lets Bedrock `aws-sdk` media calls proceed keylessly while preserving strict missing-key errors for non-`aws-sdk` modes.

## Reproduction Assessment

Yes from source: configure Bedrock with `auth: "aws-sdk"` and no literal `apiKey`, then invoke image or another affected media path so `requireApiKey` throws before SDK credentials can resolve. I did not run a live Bedrock repro in this read-only review.

## Solution Assessment

Yes, the linked fix direction is the best narrow path: apply the existing chat-path missing-key allowance to media-understanding without adding a new config option or asking users to convert AWS credentials into OpenClaw API keys.
