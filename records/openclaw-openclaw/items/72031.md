## Label Justifications

- `P1`: This is a concrete provider-auth regression that blocks Bedrock media workflows for users relying on AWS SDK credentials.
- `impact:auth-provider`: The failure is in auth-mode handling and prevents keyless `aws-sdk` Bedrock provider routing from reaching the provider.

## Summary

Keep open: current main and the latest release still require a literal API key in the Bedrock media-understanding paths, and the focused fixing pull request is open but unmerged.

## What This Changes

The issue reports that Bedrock image/media understanding should honor `auth: "aws-sdk"` keyless credentials instead of failing in `requireApiKey`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/72092, or an equivalent narrow fix, with regression coverage for image plus audio/video empty-key Bedrock auth.

## Reproduction Assessment

Yes. Source inspection on current main shows a Bedrock image/media model with `auth: "aws-sdk"` and no `apiKey` reaches strict `requireApiKey` before the AWS SDK credential chain can run, and commenters confirmed the same failure on `2026.5.20`.

## Solution Assessment

Yes. The best fix direction is the linked narrow patch or equivalent: allow empty-key execution only for resolved `aws-sdk` auth while preserving existing errors for other auth modes and covering the rotation path.
