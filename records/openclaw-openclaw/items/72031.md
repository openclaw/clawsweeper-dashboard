## Summary

Keep open. Current main still sends Bedrock `aws-sdk` media-understanding auth through unconditional API-key requirements, while the linked closing PR #72092 is open and unmerged, so this issue should remain open until that PR lands or is superseded.

## What This Changes

Allow Bedrock media-understanding image and audio/video paths using `auth: "aws-sdk"` to run without a static API key while preserving missing-key failures for other auth modes.

## Best Possible Solution

Land or supersede #72092 with a narrow media-understanding fix that allows empty API-key material only for `aws-sdk` auth, skips storing empty runtime keys, preserves missing-key errors for all other modes, and adds regression coverage for Bedrock image plus provider-execution auth.

## Reproduction Assessment

Yes. Configure a Bedrock image model so auth resolves to `mode: "aws-sdk"` with no `apiKey`, then invoke the image tool; current main reaches `requireApiKey` in `resolveImageRuntime` and throws before the AWS SDK credential chain can run.

## Solution Assessment

Yes. Mirroring the chat path's missing-key allowance inside the media auth resolvers is the narrowest maintainable fix; changing `requireApiKey` globally would broaden behavior beyond the reported Bedrock `aws-sdk` case.
