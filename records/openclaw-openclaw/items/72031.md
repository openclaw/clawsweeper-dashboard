## Summary

Keep open. Current `main` still routes Bedrock `aws-sdk` media-understanding auth through strict API-key checks, and the linked fix PR #72092 is open and unmerged.

## What This Changes

Permit keyless `aws-sdk` auth in the Bedrock image and related media-understanding provider-execution paths while preserving API-key errors for other auth modes.

## Best Possible Solution

Land or supersede #72092 with a narrow media-understanding fix that allows empty API-key material only for `aws-sdk`, avoids storing empty runtime keys, and adds regression coverage for image plus provider-execution paths.

## Reproduction Assessment

Yes. Configure an `amazon-bedrock` image model with `auth: "aws-sdk"` and no `apiKey`; current `main` reaches `requireApiKey` in `resolveImageRuntime` before AWS SDK credential resolution can run.

## Solution Assessment

Yes. Mirroring the chat path's missing-key allowance in the media auth resolvers is the narrowest maintainable fix; changing `requireApiKey` globally would broaden behavior beyond the Bedrock `aws-sdk` case.
