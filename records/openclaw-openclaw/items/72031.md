## Summary

Keep open. Current `main` still sends Bedrock `aws-sdk` media-understanding auth through strict API-key checks, while the chat path already has a missing-key allowance and the linked fix PR is still open and unmerged.

## What This Changes

Allow Bedrock media-understanding image, fallback, and audio/video provider-execution paths to use AWS SDK credentials without a static API key.

## Best Possible Solution

Land or supersede PR #72092 with a narrow media-understanding fix that permits empty API-key material only for `aws-sdk` auth, avoids storing empty runtime keys, preserves missing-key errors for other modes, and adds regression coverage.

## Reproduction Assessment

Yes. Configure an `amazon-bedrock` image model with `auth: "aws-sdk"` and no `apiKey`, then invoke the image tool; current `main` reaches `requireApiKey` in `resolveImageRuntime` before the AWS SDK credential chain can run.

## Solution Assessment

Yes. Mirroring the chat path's missing-key allowance in the media auth resolvers is the narrowest maintainable fix; changing `requireApiKey` globally would broaden behavior beyond the Bedrock `aws-sdk` case.
