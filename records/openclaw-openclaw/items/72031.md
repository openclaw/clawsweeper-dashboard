## Summary

Keep open. Current main still rejects keyless Bedrock `aws-sdk` auth in the media-understanding image and runner paths, and an open implementation PR explicitly closes this issue, so this should stay open until that PR or an equivalent fix lands.

## What This Changes

Allow Bedrock media-understanding image and audio/video paths to use `aws-sdk` credentials without a literal API key.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/72092, or an equivalent narrow fix, after redacted real Bedrock media proof shows keyless `aws-sdk` execution reaches the provider path.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: configure `amazon-bedrock` media auth as `aws-sdk` with no literal `apiKey`, then the current image, fallback, and runner paths call strict `requireApiKey` before Bedrock can use the SDK chain.

## Solution Assessment

Yes. The narrow maintainable solution is to mirror the existing chat missing-key allowance for `aws-sdk` in media-understanding while preserving strict missing-key errors for other auth modes.
