## Summary

Keep open. Current main still sends the Bedrock `aws-sdk` media-understanding paths through strict API-key checks, and open PR #72092 already carries the closing implementation candidate.

## What This Changes

The issue requests that Bedrock image and related media-understanding auth paths allow `auth: "aws-sdk"` without requiring a literal API key.

## Best Possible Solution

Review, land, or supersede PR #72092 with a narrow media-auth fix that permits empty key material only for resolved `aws-sdk` auth and preserves strict missing-key behavior for other modes.

## Reproduction Assessment

Yes. Configure an `amazon-bedrock` image model with `auth: "aws-sdk"` and no `apiKey`; current main reaches `requireApiKey` in `resolveImageRuntime`, and the helper throws before AWS SDK credential resolution can run.

## Solution Assessment

Yes. Mirroring the chat path's missing-key allowance in the media auth resolvers is the narrowest maintainable fix; relaxing `requireApiKey` globally would broaden behavior beyond this provider auth mode.
