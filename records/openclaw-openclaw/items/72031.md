## Summary

Keep open. Current main still rejects keyless Bedrock `aws-sdk` auth in the media-understanding paths, and an open PR explicitly closes this issue, so this should stay open until that PR or an equivalent fix lands.

## What This Changes

The issue requests that Bedrock media-understanding paths allow `aws-sdk` auth without a literal API key, matching the existing chat/completion behavior.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/72092, or an equivalent narrow fix, so only resolved `aws-sdk` media auth can proceed without a literal key while other auth modes keep strict missing-key errors.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: configure Bedrock media auth as `aws-sdk` without a literal `apiKey`, then the image and media runner paths call strict `requireApiKey` before provider execution.

## Solution Assessment

Yes. The maintainable fix is to align media-understanding with the existing chat `aws-sdk` missing-key allowance, keep the strict helper semantics for other modes, and add focused regression coverage for image and runner media paths.
