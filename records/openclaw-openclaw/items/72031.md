## Summary

Keep open: current main and the latest release still route Bedrock `aws-sdk` media auth through strict API-key checks, and the focused closing PR remains open and currently conflicting.

## What This Changes

The issue asks OpenClaw media tools to honor Bedrock `auth: "aws-sdk"` without requiring a literal API key, matching the chat path's AWS SDK credential-chain behavior.

## Best Possible Solution

Land or replace the linked fix with a narrow media-auth change covering image, fallback, audio/video, and PDF paths for empty-key `aws-sdk` auth while preserving fail-closed behavior for other modes and adding focused regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows `amazon-bedrock` `aws-sdk` media auth reaches strict `requireApiKey` with no literal key before the AWS SDK credential chain can run, and multiple issue comments report matching live failures.

## Solution Assessment

Yes, the best fix shape is to mirror the existing chat-path missing-key allowance in the media auth paths without adding new config. The linked PR is the right lane but must be made mergeable and should cover the later PDF evidence too.
