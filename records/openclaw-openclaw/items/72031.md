## Summary

Keep this issue open: current main and the latest release still route Bedrock media auth through strict API-key guards, and the linked closing PR https://github.com/openclaw/openclaw/pull/72092 is open rather than merged.

## What This Changes

The issue reports that Bedrock image/media tools fail with `auth: "aws-sdk"` because media auth requires a static API key even though AWS SDK credentials are available.

## Best Possible Solution

Review and land, replace, or extend https://github.com/openclaw/openclaw/pull/72092 so Bedrock media paths use keyless `aws-sdk` auth while non-`aws-sdk` modes keep strict missing-key errors.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: configure Bedrock with `auth: "aws-sdk"` and no literal `apiKey`, then invoke an affected media path so `requireApiKey` throws before SDK credentials can resolve; comments also include live reports on 2026.5.20 and 2026.5.28.

## Solution Assessment

Yes for the core bug boundary. The best fix is to reuse the existing keyless `aws-sdk` behavior already used by chat/media runner paths, with maintainers deciding whether the later PDF reports belong in the same PR or a follow-up.
