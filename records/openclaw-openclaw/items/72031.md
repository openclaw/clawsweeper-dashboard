## Summary

Keep open. Current main still throws before Bedrock AWS SDK credentials can resolve in the image and media runner auth paths, and open PR #72092 is already tracking the narrow fix with closing syntax.

## What This Changes

The issue asks OpenClaw's media-understanding image and audio/video auth paths to allow Bedrock `aws-sdk` mode without a literal API key, matching chat completion behavior.

## Best Possible Solution

Review, land, or supersede PR #72092 with a narrow media-auth fix that permits empty key material only for resolved `aws-sdk` auth and preserves strict missing-key errors for other modes.

## Reproduction Assessment

Yes. Configure an `amazon-bedrock` image model with `auth: "aws-sdk"` and no `apiKey`; current main reaches `requireApiKey` in `resolveImageRuntime` before the AWS SDK credential chain can run.

## Solution Assessment

Yes. Mirroring the chat path's missing-key allowance in the media auth resolvers is the narrowest maintainable fix; relaxing `requireApiKey` globally would broaden behavior beyond the Bedrock `aws-sdk` case.
