## Summary

Keep open. Current `main` still routes Bedrock `aws-sdk` media auth through unconditional API-key checks, and the open closing PR #72092 has not merged yet, so this issue should remain the canonical tracker until that fix lands or is superseded.

## What This Changes

The issue asks for Bedrock `aws-sdk` media-understanding auth to allow image, audio, and video requests without a static API key, matching chat completion behavior.

## Best Possible Solution

Land or supersede #72092 with the narrow media auth fix: allow empty API-key material only when resolved auth mode is `aws-sdk`, preserve missing-key failures for all other modes, avoid storing empty runtime keys, and keep focused regression coverage for Bedrock image plus provider execution auth.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: configure an `amazon-bedrock` image model with `auth.mode === "aws-sdk"` and no `apiKey`; current `main` reaches `requireApiKey` in media auth resolution and throws before the Bedrock SDK credential chain can run.

## Solution Assessment

Yes. Mirroring the chat path's missing-key allowance in the media-specific auth resolvers is the narrow maintainable solution; changing `requireApiKey` globally would broaden behavior beyond the reported Bedrock `aws-sdk` case.
