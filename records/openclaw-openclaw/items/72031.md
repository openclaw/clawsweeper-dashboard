## Summary

Keep open: current main still rejects keyless Bedrock `aws-sdk` auth in media-understanding, and a focused open PR explicitly closes this issue, so the issue should remain open until that PR or an equivalent fix lands.

## What This Changes

The issue asks OpenClaw media-understanding to let Bedrock `auth: "aws-sdk"` image and media requests use AWS SDK credentials without a static API key.

## Best Possible Solution

Land https://github.com/openclaw/openclaw/pull/72092, or an equivalent narrow fix, so only resolved `aws-sdk` media auth can proceed without a literal key while other auth modes keep strict missing-key errors.

## Reproduction Assessment

Yes. Source-level reproduction is high confidence: configure Bedrock media auth as `aws-sdk` without a literal `apiKey`, then current main reaches strict `requireApiKey` in the image/fallback and media runner paths before provider execution.

## Solution Assessment

Yes. The maintainable fix is to align media-understanding with the existing text-runtime `aws-sdk` missing-key allowance, add focused regression coverage, and keep strict missing-key errors for other auth modes.
