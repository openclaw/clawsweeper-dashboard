## Summary

Keep open. Current `origin/main` still routes Bedrock `aws-sdk` media-understanding auth through strict API-key checks before the AWS SDK credential chain can run, and PR #72092 remains the open implementation candidate for this issue.

## What This Changes

Allow media-understanding image and audio/video auth resolution to accept Bedrock `aws-sdk` mode without a static API key, matching the chat/completion path.

## Best Possible Solution

Review, land, or supersede PR #72092 with a narrow media-understanding fix that permits empty API-key material only for `aws-sdk`, preserves strict errors for other auth modes, skips storing empty runtime keys, adds regression tests, and includes the required changelog entry.

## Reproduction Assessment

Yes. Configure an `amazon-bedrock` image model whose resolved auth is `mode: "aws-sdk"` with no `apiKey`, then invoke the image tool; current `origin/main` reaches `requireApiKey` in `resolveImageRuntime` before provider execution.

## Solution Assessment

Yes. Mirroring the chat path's missing-key allowance in the media auth resolvers is the narrow maintainable fix; changing `requireApiKey` globally would relax every caller beyond the Bedrock SDK-auth case.
