## Summary

Keep open. The PR targets a real ClawHub archive-download hang and now includes acceptable local-stub live output, but the implementation cancels `response.body` after `arrayBuffer()` has already locked it, so the stalled body read is not actually aborted.

## What This Changes

The PR adds a shared body-read timeout helper for ClawHub skill, package, and ClawPack archive downloads and adds stalled-body regression tests.

## Best Possible Solution

Rework the helper so the timeout owns the active body reader or an AbortController through body consumption, and cover skill, package, and ClawPack archives with real `ReadableStream` stalled-body tests.

## Reproduction Assessment

Yes. Current main’s ClawHub archive paths read the body after the request timeout is cleared, and a direct Node Web Streams check confirms the PR’s cancellation path fails once `arrayBuffer()` locks the stream.

## Solution Assessment

No. The direction is right, but cancelling `response.body` after starting `arrayBuffer()` is not the narrow maintainable fix; the timeout should own a reader or abort signal for the in-flight body read.

## Review Findings

Overall correctness: patch is incorrect
