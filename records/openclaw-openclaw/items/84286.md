## Best Possible Solution

Keep the current managed-cron reconciliation model, then define a bounded memory-core recovery path for stale low-signal session-corpus entries that preserves promotion quality and gives operators a documented reset or backfill route.

## Solution Assessment

Unclear: automatic age or dailyCount fallback may solve the reported stuck state, but it changes durable-memory promotion policy. The cron reconciliation shape itself already looks correct on current main.
