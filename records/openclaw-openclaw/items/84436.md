- `impact:session-state`: The issue concerns unwanted memory recall state and `.dreams` artifact churn when a memory-state feature is disabled.

## Summary

Keep open: current main still performs dreaming recall tracking from normal memory search when the documented master switch is off, and two open PRs already use closing syntax for the same bug. The issue should close only after one linked fix lands.

## What This Changes

Stop ordinary memory search and CLI search from writing `memory/.dreams` recall artifacts when `plugins.entries.memory-core.config.dreaming.enabled=false` while preserving normal search results.

## Best Possible Solution

Land one canonical linked fix that gates tool and CLI recall-tracking writes on the resolved dreaming enablement while preserving normal memory search and adding regression coverage for disabled and enabled dreaming.

## Reproduction Assessment

Yes. Source inspection on current main shows both the tool and CLI memory search paths can call `recordShortTermRecalls(...)` without checking `dreaming.enabled`, and that helper writes the reported `.dreams` files; I did not run a live filesystem repro because this review is read-only.

## Solution Assessment

Yes. The maintainable fix is the narrow call-site guard described by the issue and attempted by the linked PRs, with tests proving search still returns results and recall artifacts are skipped only when dreaming is disabled.
