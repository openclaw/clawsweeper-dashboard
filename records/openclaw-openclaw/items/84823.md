The PR filters `session.tool` mirror recipients in `createAgentEventHandler` so connections already receiving a run-scoped tool `agent` event are skipped, and adds a gateway event-handler regression test.

## Best Possible Solution

Land a gateway-owned dedupe that preserves session-only `session.tool` delivery, after maintainer review confirms overlapping tool-events clients may rely on `agent` frames and after-fix real behavior proof is added.
