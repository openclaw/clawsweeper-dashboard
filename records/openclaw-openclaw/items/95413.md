The PR adds a Telegram rich HTML preparation helper that converts non-whitespace text-node newlines to `<br>` for persistent rich sends/edits and adds two send-path regression tests.

## Best Possible Solution

Use one canonical rich newline normalization that distinguishes structural block whitespace from semantic inline/text breaks, protects pre/code/math regions, and merge only after exact-head Telegram proof.
