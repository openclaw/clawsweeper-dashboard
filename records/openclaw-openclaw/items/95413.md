Full review comments:

- **[P2] Convert inline-only newline segments too:** `extensions/telegram/src/rich-message.ts:210-214`
  - body: This condition preserves every whitespace-only text node, so rich HTML produced from common formatted Markdown like `**one**\n**two**` becomes `<b>one</b>\n<b>two</b>` and still sends a raw newline between inline tags. That is the same collapse class the PR is meant to fix; only structural whitespace between block/container tags should be exempted.
  - confidence: 0.9
- **[P2] Preserve rich math text while rewriting newlines:** `extensions/telegram/src/rich-message.ts:225-231`
  - body: The depth tracking only protects `<pre>` and `<code>`, but OpenClaw and the Bot API support rich math tags such as `<tg-math-block>`. A multiline formula sent as explicit rich HTML would now get `<br>` inserted inside the math content, so the helper should protect `tg-math` and `tg-math-block` and add focused coverage.
  - confidence: 0.84
