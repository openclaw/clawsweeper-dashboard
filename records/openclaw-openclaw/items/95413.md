Full review comments:

- **[P2] Convert separator newlines between inline tags:** `extensions/telegram/src/rich-message.ts:211-214`
  - body: `appendText` skips every whitespace-only text node, so rich HTML such as `<b>one</b>\n<i>two</i>` still sends a raw newline between inline tags and can collapse in the same persistent rich-message path this PR is fixing. Limit the skip to structural block/container whitespace and add a regression test for inline-formatted line breaks.
  - confidence: 0.87
- **[P2] Protect rich math tags from newline rewriting:** `extensions/telegram/src/rich-message.ts:226-232`
  - body: The helper only treats `pre` and `code` as protected regions, but current rich HTML support also allows `tg-math` and `tg-math-block`. A multiline math block would have its internal newline rewritten to `<br>`, changing supported rich math content; include those tags in the protected-region logic or move this normalization to a boundary that already knows rich tag classes.
  - confidence: 0.84
