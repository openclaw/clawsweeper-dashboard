## Summary

Keep open. Current main has partial hardening for this incident class, especially runtime config use, model-cache invalidation, bounded timeout compaction, and subagent direct-to-queue fallback, but the reported umbrella chain is not fully resolved. Gateway drain still rejects new work, reply handling still surfaces a restart message for GatewayDrainingError instead of retrying the original turn, subagent announce cleanup still has retry-limit/expiry give-up paths, and timeout compaction still falls through after the retry cap; merged #70743 is relevant partial hardening, not a complete superseding fix.

## What This Changes

The issue asks OpenClaw to investigate and harden a large-session failure chain involving compaction timeout, gateway draining, subagent announce loss, and stale runtime fallback config after provider removal.

## Best Possible Solution

Keep this as the umbrella until maintainers either split it into narrower canonical follow-ups or land a coordinated reliability fix covering restart/drain retry semantics, durable or explicitly bounded subagent completion delivery, compaction timeout circuit-breaker/reset policy, and documented live fallback/session-override behavior after config changes.
