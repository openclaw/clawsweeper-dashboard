## Summary

Keep open. Current main has useful hardening, but the reported cascade is not fully resolved: drain windows still reject new enqueues, reply handling terminates with restart text, subagent announce delivery remains bounded best-effort, and compaction recovery still has capped fallthrough paths.

## What This Changes

Investigate a v2026.4.8 regression where large Telegram sessions with compaction timeouts may cascade into gateway drain rejections, lost subagent announces, and stale fallback routing after config changes.

## Best Possible Solution

Keep this as the umbrella until maintainers split or coordinate reliability work for drain-window replay, durable or explicitly bounded subagent completion delivery, compaction timeout/overflow escalation, and documented live fallback reload semantics.
