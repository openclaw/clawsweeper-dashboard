## What This Changes

The issue requests allowing `channels.whatsapp.groups` keys to match exact WhatsApp group names as well as stable group JIDs.

## Best Possible Solution

Keep this issue open while maintainers review the linked PR; land a default-off, JID-first implementation only if the mutable-name security tradeoff is accepted.

## Reproduction Assessment

Yes, at source level. Current main carries `group.subject` from WhatsApp metadata, but admission, requireMention, tool policy, and system prompts still resolve from stable group IDs only.

## Solution Assessment

Unclear until maintainer review. A WhatsApp-local, default-off, JID-first opt-in looks like the safest shape; implicit mutable-name authorization would not be the best solution.
