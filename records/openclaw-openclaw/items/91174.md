## Summary

Keep open. Current main and v2026.6.6 still resolve WhatsApp group config by stable group JID only, while the linked implementation PR remains open and conflicting, so the issue should stay open until maintainers accept, land, or reject that security-sensitive path.

## What This Changes

Allow WhatsApp `channels.whatsapp.groups` entries to match an exact current group subject/name as well as the stable group JID, while preserving JID and wildcard behavior.

## Best Possible Solution

Review and refresh https://github.com/openclaw/openclaw/pull/91180 as the canonical implementation path; land only a default-safe, explicit opt-in exact-subject matcher where stable JID keys continue to win.

## Reproduction Assessment

Yes, at source level. Current main carries `groupSubject`, but admission, require-mention, tool policy, diagnostics, and system prompt resolution still use the stable WhatsApp group JID.

## Solution Assessment

Unclear until maintainer review finishes. Exact subject matching is plausible, but the maintainable shape should be explicit opt-in with JID precedence, duplicate/rename semantics, docs, diagnostics, and focused WhatsApp proof.
