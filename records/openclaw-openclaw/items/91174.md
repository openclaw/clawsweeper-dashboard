The issue requests allowing `channels.whatsapp.groups` keys to match either the stable WhatsApp group JID or the exact current WhatsApp group subject/name.

## Best Possible Solution

Yes, at source level. Current main carries WhatsApp group subjects from Baileys metadata, but group admission, require-mention defaults, tool policy, and group system prompts still resolve config using only the WhatsApp group JID.

## Solution Assessment

Unclear until maintainer approval. The safest apparent shape is a WhatsApp-local opt-in with stable JIDs taking precedence, but duplicate-name and rename behavior are product/security decisions.
