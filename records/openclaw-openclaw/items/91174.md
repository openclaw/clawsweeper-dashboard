## What This Changes

Allow WhatsApp `channels.whatsapp.groups` entries to match either the stable group JID or the exact current group subject/name, while keeping JID matches preferred.

## Best Possible Solution

Review https://github.com/openclaw/openclaw/pull/91180 as the canonical implementation path; land it only if JID-first behavior stays default and exact-name matching remains an explicit audited opt-in, otherwise keep WhatsApp group config JID-only.

## Solution Assessment

Unclear until maintainer review finishes. Exact subject matching is a plausible solution, but the safest shape needs explicit opt-in, JID precedence, duplicate-name/rename semantics, docs, diagnostics, and live WhatsApp proof.
