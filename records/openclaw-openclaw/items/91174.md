## Summary

Keep open. Current main and v2026.6.9 still resolve WhatsApp group config from stable group JIDs only, and the open linked PR is the active implementation candidate for the requested exact-name matching behavior.

## What This Changes

Allow `channels.whatsapp.groups` keys to match either the stable WhatsApp group JID or the exact current WhatsApp group subject/name, with JID keys preferred.

## Best Possible Solution

Keep this issue open while maintainers review https://github.com/openclaw/openclaw/pull/91180; land a default-off, JID-first, audited implementation or explicitly keep WhatsApp group config JID-only and leave readability to https://github.com/openclaw/openclaw/issues/11487.

## Reproduction Assessment

Yes, at source level. Current main captures `groupSubject` from WhatsApp metadata, but admission, requireMention, tool policy, and system prompts still resolve from stable group IDs only.

## Solution Assessment

Unclear until maintainer review. A WhatsApp-local, default-off, JID-first opt-in with dangerous-flag auditing looks like the safest shape; implicit mutable-name authorization would not be the best solution.
