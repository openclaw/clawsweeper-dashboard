## Summary

Keep open. Current main still allows the outbound route/session-key owner and delivery-mirror agent/store owner to diverge, while an existing gateway test encodes explicit-agent precedence, so closing this as fixed would be premature.

## What This Changes

The issue reports that delivery-mirror transcript entries can be written under the main agent instead of the writer agent in multi-account, multi-agent Feishu configurations.

## Best Possible Solution

Define the transcript ownership invariant, then update message-action and gateway mirror construction or transcript append validation so mirror.sessionKey and mirror.agentId cannot target different agent stores, with focused regression coverage.

## Reproduction Assessment

Yes, with source-level confidence. The issue gives concrete Feishu steps, and current main still routes mirror.sessionKey from the outbound route while selecting the transcript store from mirror.agentId.

## Solution Assessment

Unclear. Deriving mirror.agentId from outboundRoute.sessionKey is plausible for the reported ghost sessions, but it conflicts with the existing gateway explicit-agent precedence test, so the invariant should be settled before patching.
