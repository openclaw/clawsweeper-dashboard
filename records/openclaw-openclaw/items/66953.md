## Summary

Keep open. Current main still allows the delivery route session key and delivery-mirror transcript owner agent to diverge, and the existing gateway test suite currently encodes explicit-agent precedence that conflicts with the reporter's suggested route-owner fix.

## What This Changes

The issue requests that delivery-mirror transcript entries for multi-account, multi-agent outbound sends be written under the agent identified by the routed session key instead of the default/request agent.

## Best Possible Solution

Add a focused multi-agent, multi-account regression and enforce one invariant: delivery-mirror transcript storage must follow the same owner as the route/session key, or mismatched explicit agent/session inputs must be rejected before delivery.

## Reproduction Assessment

Yes. The issue provides concrete Feishu multi-account steps, and current-main source inspection shows the same reproducing shape: mirror.sessionKey can be route-owned while mirror.agentId remains request/effective-agent-owned before transcript store selection.

## Solution Assessment

Unclear. Deriving mirror.agentId from the outbound route session key looks like the narrow transcript-pollution fix, but it conflicts with an existing gateway test that says explicit agentId should override the sessionKey agent for mirror delivery.
