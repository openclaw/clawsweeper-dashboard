## Summary

Current `main` still lacks WhatsApp configured-binding hooks while ACP materialization is explicitly adapter-gated, so the reported silent local-execution path remains necessary to fix. The older route-binding report is related but does not supersede this narrower failure after agent selection.

## What This Changes

Add WhatsApp ACP configured-binding materialization so matching direct conversations execute on the configured ACP backend instead of falling through to embedded local execution.

## Best Possible Solution

Implement the standard configured-binding adapter on `whatsappPlugin`, run the canonical ACP target readiness/materialization step before inbound reply dispatch, and prove direct-message matching selects the ACP binding session without an embedded fallback.

## Reproduction Assessment

Yes. The report supplies a concrete resolver probe and runtime steps, while current source confirms that WhatsApp lacks the adapter hooks required to compile and resolve a configured ACP binding.

## Solution Assessment

Yes. Extending WhatsApp through the existing channel binding contract and canonical ACP lifecycle is the narrowest maintainable repair; changing generic agent routing or adding another fallback would duplicate policy.
