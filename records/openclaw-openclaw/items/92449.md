The issue requests that WhatsApp type=acp configured bindings materialize configured ACP sessions, or fail loudly, instead of falling back to embedded local execution.

## Solution Assessment

Yes. The best fix direction is to reuse the established channel configured-binding adapter and shared route/readiness lifecycle in the WhatsApp plugin, not to add a new config surface or a core WhatsApp special case.
