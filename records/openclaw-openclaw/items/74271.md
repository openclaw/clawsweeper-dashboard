## Summary

Keep open. Current main still has the gateway credential precedence split and existing diagnostics cover launchctl/service drift but not the requested state-dir `.env` token mismatch; open PR #74433 is the unmerged implementation candidate, so this issue should stay open until that PR lands or is replaced.

## What This Changes

Request a redacted doctor/status diagnostic when state-dir `OPENCLAW_GATEWAY_TOKEN` shadows a configured local `gateway.auth.token`.

## Best Possible Solution

Keep this as the canonical report and resolve it through #74433 or a replacement that adds one shared redacted mismatch diagnostic for differing non-empty state-dir `OPENCLAW_GATEWAY_TOKEN` and configured `gateway.auth.token`.

## Reproduction Assessment

Yes. A high-confidence source-backed reproduction is local gateway mode with `gateway.auth.token` in config and a different non-empty `OPENCLAW_GATEWAY_TOKEN` in state-dir `.env`, which exercises env-first client/probe credentials against config-first gateway auth.

## Solution Assessment

Yes. A warning is the narrowest maintainable solution because it preserves intentional env-based auth, avoids deleting credentials, and can point to `~/.openclaw/.env` and gateway restart steps without printing secrets.
