## Summary

Keep open. Current main still lacks the requested redacted state-dir `.env` token mismatch warning, and open PR #74433 is an unmerged closing implementation candidate that should be reviewed, landed, or replaced before this issue closes.

## What This Changes

Warn in doctor/status surfaces when local gateway mode has `gateway.auth.token` configured and state-dir `.env` contains a different non-empty `OPENCLAW_GATEWAY_TOKEN`, without printing secrets.

## Best Possible Solution

Keep this as the canonical report and resolve it through PR #74433 or a replacement that adds one shared redacted diagnostic for differing non-empty state-dir `OPENCLAW_GATEWAY_TOKEN` and configured `gateway.auth.token` in local mode.

## Reproduction Assessment

Yes. A high-confidence source-backed reproduction is local gateway mode with `gateway.auth.token` set in config and a different non-empty `OPENCLAW_GATEWAY_TOKEN` loaded from state-dir `.env`; current credential code and parity tests show the env/config disagreement.

## Solution Assessment

Yes for the requested direction, but it is not implemented on main yet. A warning is safer than deleting or rewriting credentials because env-based auth is a supported configuration.
