## Summary

## What This Changes

The issue asks for `doctor`/status diagnostics that warn when `OPENCLAW_GATEWAY_TOKEN` in `~/.openclaw/.env` can shadow a configured `gateway.auth.token` in local gateway mode.

## Best Possible Solution

Keep this as the canonical report and resolve it through #74433 or an equivalent replacement that adds one shared redacted mismatch diagnostic without deleting or printing credentials.

## Reproduction Assessment

Yes. Configure local gateway mode with `gateway.auth.token`, load a different non-empty `OPENCLAW_GATEWAY_TOKEN` from the state-dir `.env` into the command environment, then exercise local call/probe paths against config-first gateway auth.

## Solution Assessment

Yes. A warning is the narrowest maintainable direction because it preserves intentional env-based auth while making stale state-dir credentials visible without exposing secret values.
