## What This Changes

Request a diagnostic warning when local gateway mode has `gateway.auth.token` configured while `OPENCLAW_GATEWAY_TOKEN` in the state-dir `.env` can override or disagree with it.

## Best Possible Solution

Keep this as the canonical report and resolve it through PR #74433 or a replacement that adds one shared redacted diagnostic: parse the state-dir `.env`, compare non-empty `OPENCLAW_GATEWAY_TOKEN` against configured local `gateway.auth.token`, explain env precedence, and suggest removing or syncing the env key plus restarting the gateway without printing secrets.

## Reproduction Assessment

Yes. A source-backed reproduction is local gateway mode with `gateway.auth.token` set in config and a different non-empty `OPENCLAW_GATEWAY_TOKEN` loaded from the state-dir `.env`; current parity coverage documents the resulting credential disagreement.

## Solution Assessment

Yes for the warning direction: warning is safer than deleting or rewriting credentials that may be intentional. The implementation should be reviewed for shared redaction and requested surface coverage rather than separate ad hoc checks.
