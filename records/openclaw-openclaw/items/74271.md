## Summary

Keep open. Current main still lacks the requested redacted state-dir `.env` mismatch warning, and open PR #74433 is an unmerged closing implementation candidate that should be reviewed, landed, or replaced before this issue closes.

## What This Changes

Warn doctor/status surfaces when local gateway config has `gateway.auth.token` while `~/.openclaw/.env` contains a different non-empty `OPENCLAW_GATEWAY_TOKEN`.

## Best Possible Solution

Land or rework PR #74433 into one shared redacted diagnostic that compares state-dir `.env` token presence/value against configured `gateway.auth.token` and surfaces it in the relevant doctor/status paths.

## Reproduction Assessment

Yes. A high-confidence source-backed reproduction is local gateway mode with `gateway.auth.token` set in config and a different non-empty `OPENCLAW_GATEWAY_TOKEN` loaded from the state-dir `.env`; current precedence tests and auth code show that disagreement.

## Solution Assessment

Yes for the requested direction: warning is safer than rewriting or deleting credentials that may be intentional. Current main does not implement it yet, so the open PR or a narrow replacement is the maintainable path.
