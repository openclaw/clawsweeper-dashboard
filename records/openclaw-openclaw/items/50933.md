This PR adds `session:idle_reset` and `session:daily_reset` internal hook events for stale session rollover, updates the bundled session-memory hook to handle them, and adds related tests/docs.

## Best Possible Solution

Keep implementation review on https://github.com/openclaw/openclaw/pull/61675, land or reject that canonical hook contract, and close competing branches against the shipped decision.

Yes. Source inspection gives a high-confidence path: configure a stale daily or idle session and send the next inbound message; current main rotates and archives the prior session while session-memory remains registered only for command new/reset events.

## Solution Assessment

No. The direction is valid, but this branch is no longer the best path because it awaits hook dispatch in session initialization and is superseded by a maintained canonical branch that moves this work off the reply path.
