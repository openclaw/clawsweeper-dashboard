## Summary

Keep open. Current main still has the source-reproducible duplicate `path | path` media-header behavior when Telegram mirrors local media paths into URLs, and the open closing PR https://github.com/openclaw/openclaw/pull/77279 is the active fix path; changing the all-image transcription default is a separate product/config decision.

## What This Changes

The issue reports that Telegram album messages only describe the first image, omit that image from the media header, duplicate paths in other header entries, and do not describe the remaining images.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/77279 for the duplicate path/url formatter bug, then only revisit all-image transcription if maintainers want to change the documented attachment policy.

## Reproduction Assessment

Yes, at source level. Passing equal `MediaPaths` and `MediaUrls` into `buildInboundMediaNote` reproduces the duplicate `path | path` header shape that Telegram creates by setting both fields to `media.path`.

## Solution Assessment

Yes for the narrow remaining bug. The shared formatter should dedupe identical sanitized path/url values; changing the first-image media-understanding default to all images is a separate product decision because `mode: "all"` already exists.
