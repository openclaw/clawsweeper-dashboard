## Summary

Keep open. Current `main` still has the source-reproducible duplicate `path | path` media-header bug, and the open closing PR https://github.com/openclaw/openclaw/pull/77279 is the active fix path.

## What This Changes

The issue reports that Telegram multi-image messages only describe the first image, list the remaining image paths in the media header, and duplicate local paths as `path | path`.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/77279 for the duplicate path/url formatter bug, while keeping all-image transcription under the documented attachment policy unless maintainers decide to change that default separately.

## Reproduction Assessment

Yes. Pass equal `MediaPaths` and `MediaUrls` into `buildInboundMediaNote`, which Telegram creates by setting both fields to `media.path`, and current `main` renders duplicated `path | path` header lines.

## Solution Assessment

Yes for the remaining narrow bug. The shared formatter should dedupe identical sanitized path/url values; changing the first-image media-understanding default to all images would be a separate policy decision.
