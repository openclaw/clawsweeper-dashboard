- `impact:message-loss`: Some user-sent image context can be absent or misleading in the model-visible prompt when only one image is described and remaining media lines duplicate local references.

## Summary

Keep open. Current main and the latest release still render duplicated identical media path/URL headers, while https://github.com/openclaw/openclaw/pull/77279 is the open focused fix path and the all-image transcription default remains a product-policy question.

## What This Changes

The issue asks OpenClaw to handle Telegram multi-image messages so every image is represented once in the media header and image understanding does not leave album images unseen.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/77279 for the duplicate media-note suffix, while keeping all-image descriptions on `tools.media.image.attachments` with `mode: "all"` unless maintainers choose a separate default change.

## Reproduction Assessment

Yes, at source level. Current main maps Telegram album media path to both `path` and `url`, then the shared formatter appends the mirrored URL and current tests expect `path | path` output.

## Solution Assessment

Yes for the narrow formatter bug: dedupe identical sanitized path and URL in the shared formatter. Changing the default from first-image to all-image understanding is not the same fix because the existing config already supports `mode: "all"` and a default change needs maintainer approval.
