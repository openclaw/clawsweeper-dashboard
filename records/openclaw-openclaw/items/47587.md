## Summary

Keep open. Current main preserves all Telegram album media paths, but the issue is not fully resolved because Telegram still mirrors local paths into URL fields, the media-note formatter still renders duplicated `path | path`, and shared media understanding still processes only the first image by default.

## What This Changes

The issue requests that Telegram album messages expose every downloaded image once in the media header and run image understanding for every image.

## Best Possible Solution

Make Telegram album behavior explicit: de-duplicate local path/url rendering, keep all album media paths available, and have maintainers choose whether Telegram albums should override the shared first-image media-understanding default or require documented `tools.media.image.attachments.mode = "all"` configuration.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists by constructing a Telegram album with multiple image paths: current context assembly creates identical `MediaPaths`/`MediaUrls`, the prompt formatter renders both, and the shared selector chooses only the first image under default policy.

## Solution Assessment

Unclear. Header de-duplication is a narrow maintainable fix, but all-image transcription by default is a product/cost policy decision because it would diverge from the documented shared media-understanding default.
