## Summary

Keep open. Current main still has a source-reproducible duplicate `path | path` media-header bug, and the open closing PR https://github.com/openclaw/openclaw/pull/77279 is the active fix path.

## What This Changes

The issue reports that Telegram album image messages describe only the first image, omit that image from the attachment header, list other images without descriptions, and duplicate local paths in media header lines.

## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/77279 for the identical path/url formatter bug, while leaving broader all-image transcription defaults to the existing attachment policy or a separate product decision.

## Reproduction Assessment

Yes. Pass equal `MediaPaths` and `MediaUrls` into `buildInboundMediaNote`, which Telegram creates for local album media, and current main renders duplicated `path | path` header lines.

## Solution Assessment

Yes for the remaining narrow bug. The shared formatter should dedupe identical sanitized path/url values; changing the default from first-image media understanding to all-image understanding would be a separate policy change.
