## Summary

Keep open. Current main still has a source-reproducible duplicated local path/url media header for Telegram album media, and https://github.com/openclaw/openclaw/pull/77279 is the open closing PR carrying the narrow formatter fix.

## What This Changes

The issue requests that Telegram multi-image messages keep every image path in the media header once and make image descriptions consistent instead of splitting first-image description from the remaining image paths.

## Best Possible Solution

Land or replace the open formatter PR so identical local path/url media-note entries render once, while keeping multi-image description behavior controlled by the documented attachment policy.

## Reproduction Assessment

Yes. A high-confidence source reproduction exists by passing equal MediaPaths and MediaUrls arrays, as Telegram does for local album media, into buildInboundMediaNote and observing the current duplicated `path | path` output.

## Solution Assessment

Yes for the remaining narrow bug. Dedupe identical path/url rendering in the shared media-note formatter; do not change the shared first-image default because all-image processing is already configurable.
