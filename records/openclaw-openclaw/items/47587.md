## Summary

Keep open. Current main still has a source-reproducible duplicated local path/url media header for Telegram album media, and https://github.com/openclaw/openclaw/pull/77279 is an open closing PR carrying the narrow formatter fix; this issue should stay open until that PR lands or is replaced.

## What This Changes

The issue asks for Telegram album image messages to include every image once in the media header and make image understanding cover the album consistently.

## Best Possible Solution

Land or replace the open formatter fix so identical local path/url media-note entries render once, while leaving the all-image transcription default to the documented attachment policy unless a maintainer deliberately changes it.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction: pass equal `MediaPaths` and `MediaUrls` arrays into `buildInboundMediaNote`, which is what Telegram does for local album media, and current main renders duplicated `path | path` header lines.

## Solution Assessment

Yes for the remaining narrow bug. The maintainable fix is to dedupe identical sanitized path/url rendering in the shared formatter; changing the shared first-image understanding default would be a separate product decision.
