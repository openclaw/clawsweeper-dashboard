## Label Justifications

- `P2`: The bug affects Telegram/media prompt context for album users, but the blast radius is bounded to media prompt assembly rather than core availability.
- `impact:message-loss`: Some user-sent image context can be absent or misleading in the model-visible prompt when only one image is described and remaining media lines duplicate local references.
- `impact:session-state`: Conversation history can preserve incomplete visual context, causing later session state to diverge from what the user actually sent.

## Summary

Keep open. Current main and the latest release still duplicate identical media path/URL headers, while https://github.com/openclaw/openclaw/pull/77279 is the open focused repair and the all-image transcription default remains a maintainer product decision.

## What This Changes

The issue asks for Telegram album images to all be described and listed once in the media header instead of one image being described while the remaining header entries duplicate path/URL data.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/77279 for the duplicate media-note suffix, and handle any default change to all-image descriptions as separate product work.

## Reproduction Assessment

Yes, at source level. Current main maps Telegram album media path to both `path` and `url`, then the shared formatter appends the mirrored URL and current tests expect `path | path` output.

## Solution Assessment

Yes for the narrow formatter bug: dedupe identical sanitized path and URL in the shared formatter. Changing the default from first-image to all-image understanding is separate because current docs already expose `tools.media.image.attachments` with `mode: "all"`.
