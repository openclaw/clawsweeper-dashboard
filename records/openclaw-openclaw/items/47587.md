## Summary

Keep open. Current main still source-reproduces the duplicated `path | path` media header, while the broader all-image transcription default is a documented attachment-policy choice and the focused formatter fix is already carried by https://github.com/openclaw/openclaw/pull/77279.

## What This Changes

Fix Telegram multi-image prompts so downloaded images are represented consistently, without duplicated local path/URL headers or missing image descriptions.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/77279 for the duplicate identical path/url header, while keeping all-image descriptions on the existing `tools.media.image.attachments` configuration path unless maintainers choose a separate default change.

## Reproduction Assessment

Yes, at source level. Passing equal `MediaPaths` and `MediaUrls` into `buildInboundMediaNote` follows the Telegram/shared payload shape and renders duplicated `path | path` header lines on current main.

## Solution Assessment

Yes for the narrow duplicate-header bug: the shared formatter should dedupe identical sanitized path/url values. Changing all-image transcription defaults is not the same fix because current docs already expose `mode: "all"` and the default policy needs maintainer approval.
