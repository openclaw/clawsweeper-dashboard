## Summary

Keep open: current main still has unbridged compile-cache launcher respawns, and an open closing PR already targets the reported orphan-worker bug, so this issue should remain the tracker until that fix lands.

## What This Changes

Killing `openclaw security audit --deep` should terminate launcher/worker descendants instead of leaving an `openclaw-security` child reparented to init.

## Best Possible Solution

Land PR #77481 or an equivalent narrow fix that puts compile-cache launcher respawns under child signal forwarding with regression coverage.

## Reproduction Assessment

Yes, at source level: current main still uses blocking `spawnSync` in the compile-cache launcher respawn paths while the documented and existing CLI pattern uses a child-process bridge. I did not live-run the long macOS audit command in this read-only pass.

## Solution Assessment

Yes in direction: direct launcher child signal forwarding is the narrowest maintainable fix and is safer than broad process-group killing in service-managed runtimes. Current main has not landed that fix yet; the open linked PR is the right item to review.
