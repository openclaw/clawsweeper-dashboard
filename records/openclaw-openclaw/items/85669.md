## Summary

Keep open: current main still has source paths where OpenClaw transcript-only assistant rows can be returned through `sessions_history` and dashboard `chat.history`, and an open fixing PR references this issue with closing syntax. ## What This Changes

The issue asks for `sessions_history` and the dashboard session viewer to hide OpenClaw transcript-only assistant rows such as `delivery-mirror` and `gateway-injected`.

## Best Possible Solution

Filter or collapse transcript-only OpenClaw assistant rows in the shared history projection where they duplicate canonical assistant output, while preserving legitimate visible mirror records and adding regressions for `sessions_history`, `chat.history`/HTTP dashboard history, and the existing message-tool mirror cases.

## Reproduction Assessment

## Solution Assessment

Unclear as written. Filtering every transcript-only OpenClaw assistant row conflicts with current tests that keep some delivery mirrors visible, so the safer solution is shared duplicate suppression or an explicit maintainer decision to change that history contract.
