## Reproduction Assessment

Yes, source-reproducible: if `chat.history` returns an assistant message with `provider: "openclaw"` and `model: "delivery-mirror"` or `"gateway-injected"`, `sessions_history` selects it because it only strips tool/toolResult messages. I did not run a live dashboard repro in this read-only review.

## Solution Assessment

Yes. The narrow maintainable solution is to filter transcript-only OpenClaw assistant rows in the shared history projection/tool output while preserving persisted transcripts and message-tool mirror display rows.
