## Summary

Keep open: current main and the latest release still let transcript-only OpenClaw assistant rows pass through `sessions_history`, and there is an open fixing PR with closing syntax that should be reviewed or landed before this issue closes.

## What This Changes

The issue asks OpenClaw to filter `delivery-mirror` and `gateway-injected` assistant rows from `sessions_history` and dashboard-facing history so duplicate assistant messages are not shown.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/85676 or an equivalent narrow fix that filters transcript-only OpenClaw assistant rows from returned history while preserving real assistant and message-tool-visible records.

## Reproduction Assessment

Yes, source-reproducible: a `chat.history` response containing assistant rows with `provider: "openclaw"` and `model: "delivery-mirror"` or `"gateway-injected"` passes through `sessions_history` because only tool roles are stripped. I did not run a live dashboard session in this read-only review.

## Solution Assessment

Yes. The maintainable fix is to filter transcript-only OpenClaw assistant rows in the history output path and add regression coverage for both default and `includeTools` behavior; optional relevance or cron filters are separate product work.
