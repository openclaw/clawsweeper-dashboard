## What This Changes

Filter `delivery-mirror` and `gateway-injected` assistant transcript rows out of `sessions_history` and dashboard-visible session history.

## Best Possible Solution

Land or replace the linked PR with a shared transcript-only assistant predicate used by both the session tool and dashboard/chat history projection while leaving persisted transcript storage intact.

## Reproduction Assessment

Yes, source-reproducible: a `chat.history` result containing an assistant row with `provider: openclaw` and model `delivery-mirror` or `gateway-injected` can pass through `sessions_history` because current main only strips tool roles. I did not run a live dashboard repro in this read-only review.

## Solution Assessment

Yes for the bug shape. The narrow maintainable solution is to share the transcript-only OpenClaw assistant filter with the session history tool and dashboard history projection rather than changing persistence or Telegram delivery behavior.
