## Best Possible Solution

Land or replace https://github.com/openclaw/openclaw/pull/85676 with a narrow shared predicate that filters delivery-mirror and gateway-injected assistant rows from sessions_history and dashboard history projection, with regression coverage for default and includeTools paths.

## Solution Assessment

Yes, the narrow maintainable fix is to filter transcript-only OpenClaw assistant rows in the shared history projection/tool output while preserving persisted transcript storage and visible message-tool mirrors.
