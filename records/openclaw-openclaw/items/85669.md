## What This Changes

The issue asks for `sessions_history` and dashboard history to filter `provider: openclaw` assistant rows whose model is `delivery-mirror` or `gateway-injected` so duplicate assistant messages are not shown.

## Best Possible Solution

Filter transcript-only OpenClaw assistant rows in the shared history/tool projection path while preserving visible message-tool mirrors, and cover both `sessions_history` and dashboard `chat.history` with regression tests.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: any `chat.history` result containing an assistant row with `provider: openclaw` and model `delivery-mirror` or `gateway-injected` can pass through `sessions_history` because only tool roles are stripped.

## Solution Assessment

Yes, the narrow maintainable fix is to share the transcript-only OpenClaw assistant predicate with history projection/tool output. The open PR should either cover the dashboard `chat.history` path too or prove that the reported dashboard duplicate comes from `sessions_history` alone.
