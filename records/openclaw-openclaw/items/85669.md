## Summary

Keep open: current main and the latest release still let transcript-only OpenClaw assistant rows reach session history surfaces, and an open closing PR is already linked but not merged. Recent comments also confirm the user-visible confusion in OpenClaw 2026.5.20.

## What This Changes

Filter `delivery-mirror` and `gateway-injected` OpenClaw assistant transcript-only rows from `sessions_history` and dashboard session history so duplicate assistant messages are not shown.

## Best Possible Solution

Review and land or replace https://github.com/openclaw/openclaw/pull/85676 with a shared transcript-only OpenClaw assistant filter, regression coverage for both `sessions_history` and dashboard/session-history projection, and real behavior proof.

## Reproduction Assessment

Yes. A `chat.history` result containing an assistant row with `provider: openclaw` and `model: delivery-mirror` or `gateway-injected` reaches current `sessions_history` output because only `tool` and `toolResult` roles are stripped; user comments also confirm the symptom on OpenClaw 2026.5.20.

## Solution Assessment

Yes. The narrow maintainable solution is to filter transcript-only OpenClaw assistant rows in the shared history projection/tool output while preserving persisted transcripts and Telegram delivery behavior; this issue should close only after the linked implementation or an equivalent fix merges.
