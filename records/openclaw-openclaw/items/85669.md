## What This Changes

The issue requests filtering `delivery-mirror` and `gateway-injected` assistant rows from `sessions_history` and dashboard-visible session history so duplicate assistant messages are not shown.

## Best Possible Solution

Land or replace the open fix with a narrow shared transcript-only assistant filter that covers the tool output and any dashboard-visible history projection without changing persisted transcripts or delivery behavior.

## Reproduction Assessment

Yes, source-reproducible: a `chat.history` response containing an assistant row with `provider: openclaw` and `model: delivery-mirror` or `gateway-injected` passes through `sessions_history` because only tool roles are stripped. ## Solution Assessment

Yes, the issue direction is the right fix shape: filter transcript-only OpenClaw assistant rows in the shared history output path. The open PR should be reviewed for whether it also covers the dashboard-visible history path or needs a small extension there.
