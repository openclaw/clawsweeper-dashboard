## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve remote attachment URLs:** `ui/src/ui/chat/grouped-render.ts:963-965`
  - body: For non-local sources, `buildAssistantAttachmentUrl` returns the original URL, but this helper then appends `?download=1` to every non-image attachment link. That changes external or signed URLs and can make existing assistant document/audio/video links fail; only add the query to the Control UI assistant-media route, or leave remote hrefs unchanged.
  - confidence: 0.88
- **[P2] Encode filenames before setting Content-Disposition:** `src/gateway/control-ui.ts:520-521`
  - body: This interpolates the basename directly into an HTTP header after only replacing quotes and CR/LF. Node rejects non-Latin1 header values, so a markdown/download file named with CJK or Cyrillic characters can throw during `setHeader` and fall into the route's 404 path; build an ASCII fallback plus RFC 5987 `filename*` before setting the header.
  - confidence: 0.86
