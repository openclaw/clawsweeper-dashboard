## Summary

Keep open: the PR is still useful, but it lacks required real behavior proof and still has a correctness issue where the new download path can mutate external or data attachment URLs.

## What This Changes

The PR renders assistant top-level `mediaUrl`/`mediaUrls` in the Control UI, adds download links/headers for non-image assistant attachments, and adds focused UI/Gateway tests.

## Best Possible Solution

Land a rebased patch that renders assistant media fields and forces downloads only for OpenClaw-owned local media routes while preserving external/data URLs and media-ticket auth.

## Reproduction Assessment

Yes. Source inspection shows a signed external document URL mirrored into `mediaUrls` is preserved on current main, while this PR would render that non-image attachment through a helper that appends `download=1`.

## Solution Assessment

No. The local markdown/download direction is maintainable, but the patch should scope query-parameter forcing to OpenClaw-owned media routes and leave external/provider/data hrefs unchanged.
