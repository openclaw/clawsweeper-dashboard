## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Encode Content-Disposition filenames safely:** `src/gateway/control-ui.ts:521-522`
  - body: This interpolates the raw basename into an HTTP header. Node rejects non-Latin header content, so a markdown file like `報告.md` will throw `ERR_INVALID_CHAR` inside the handler and fall into the 404 path instead of downloading. Build an ASCII fallback plus `filename*` value, and keep the existing CR/LF/header-injection sanitization.
  - confidence: 0.92
- **[P2] Leave external attachment URLs unchanged:** `ui/src/ui/chat/grouped-render.ts:963-964`
  - body: `buildAssistantAttachmentDownloadUrl` is used for every non-image attachment, but `buildAssistantAttachmentUrl` returns non-local URLs unchanged. Appending `?download=1` or `&download=1` to external/presigned attachment URLs can invalidate signatures or change the resource, regressing existing assistant attachments. Only add the query parameter for OpenClaw routes that honor it, and rely on `download` without mutating external URLs.
  - confidence: 0.88
