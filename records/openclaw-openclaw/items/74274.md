## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Encode filenames before setting the header:** `src/gateway/control-ui.ts:519-520`
  - body: This sets `Content-Disposition` with `path.basename(localPath)` after only stripping quotes and newlines. Node rejects non-Latin-1 header values, so a local file such as `報告.md` can throw inside the route and turn the download into a 404; use an ASCII fallback plus `filename*` encoding before setting the header.
  - confidence: 0.9
- **[P2] Do not mutate external attachment URLs:** `ui/src/ui/chat/grouped-render.ts:957-959`
  - body: `buildAssistantAttachmentUrl` returns the original URL for non-local attachments, but this helper then appends `download=1` to every non-image URL. That can break signed URLs or URLs with fragments; keep external URLs unchanged and rely on the `download` attribute, or add the query flag only for the assistant-media route that handles it.
  - confidence: 0.83
