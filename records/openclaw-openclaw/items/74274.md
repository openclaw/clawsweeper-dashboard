## Review Findings

Overall correctness: patch is incorrect

Full review comments:

- **[P1] Preserve non-local attachment URLs:** `ui/src/ui/chat/grouped-render.ts:1080-1082`
  - body: `buildAssistantAttachmentUrl` returns external and data sources unchanged, but this new helper runs after that boundary and appends `download=1` to every available audio/video/document URL. That can corrupt `data:` media URLs or invalidate signed external URLs, so add the download marker only for local or same-origin OpenClaw media routes and leave non-local URLs unchanged.
  - confidence: 0.9
- **[P1] Encode Content-Disposition filenames safely:** `src/gateway/control-ui.ts:518-519`
  - body: The basename is interpolated into `Content-Disposition` after only quote/CR/LF stripping. Node rejects raw non-Latin-1 header values, so a Unicode markdown filename such as `報告.md` throws `ERR_INVALID_CHAR` and falls into the 404 path; build an ASCII fallback plus `filename*` before setting the header.
  - confidence: 0.86
