  - body: The PR head's auto predicate is based on a hard-coded Markdown-capable set, so plugin channels such as Matrix are treated as plain text even though current Matrix outbound uses `chunkerMode: "markdown"`. That downgrades links, code, and emphasis before Matrix formatting can render them.
  - confidence: 0.9
- **[P2] Resolve markdown.strip with account scope:** `src/infra/outbound/deliver.ts:371`
  - body: `resolveChannelMarkdownConfig()` only reads `channels[channel].markdown`, but existing Markdown table config resolution checks `channels[channel].accounts[accountId].markdown` first. Multi-account operators would have their account-specific `strip` override silently ignored.
  - confidence: 0.88
- **[P1] Avoid mutating identifiers in the default strip path:** `src/shared/text/strip-markdown.ts:17`
  - body: This branch puts `stripMarkdown()` on a default outbound path, but its single-underscore regex predates current main's identifier-safe behavior and can remove underscores from text like `foo_bar_baz` or URL paths. A rebased implementation must preserve the current word-boundary fix before using the helper broadly.
  - confidence: 0.9
- **[P2] Consume complete fenced-code fences when stripping:** `src/shared/text/strip-markdown.ts:9`
  - body: The fenced-code regex accepts only lowercase language tags and leaves the newline after the closing fence outside the match, so fences such as `c++` can leak suffix text and mid-message blocks can gain extra blank lines in delivered plain text.
  - confidence: 0.86
