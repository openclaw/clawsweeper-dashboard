## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.92

Full review comments:

- **[P1] Preserve lightweight-format channels during auto-strip:** `src/infra/outbound/sanitize-text.ts:86`
  - body: `shouldStripMarkdown("whatsapp")` defaults to true because WhatsApp is not marked Markdown-capable, but existing WhatsApp delivery intentionally converts HTML and Markdown into WhatsApp markers such as `*bold*`, `_italic_`, and backticks. The new post-sanitize strip pass would remove that formatting by default.
  - confidence: 0.92
- **[P1] Honor Markdown-rendering plugin channels before auto-strip:** `src/infra/outbound/sanitize-text.ts:86`
  - body: The auto path only checks channel metadata before stripping. Matrix currently declares `chunkerMode: "markdown"` and renders Markdown downstream while its package metadata inspected here lacks `markdownCapable`, so this would strip links, code, and emphasis before Matrix can render them.
  - confidence: 0.88
- **[P2] Resolve markdown.strip with account scope:** `src/infra/outbound/deliver.ts:369-371`
  - body: The new config lookup only reads `channels[channel].markdown`, but existing Markdown config resolves `accounts.<id>.markdown` before channel-level settings. Multi-account users would silently lose per-account `markdown.strip` overrides even though the sibling `tables` option has account precedence.
  - confidence: 0.87
- **[P1] Avoid mutating identifiers in the default strip path:** `src/infra/outbound/deliver.ts:343-346`
  - body: Wiring `stripMarkdown()` into default delivery exposes the PR branch's single-underscore emphasis regex to iMessage/SMS/IRC text, so copyable identifiers like `foo_bar_baz` can be delivered as `foobarbaz`. Keep the current-main boundary-aware behavior or add equivalent coverage before making this automatic.
  - confidence: 0.9
- **[P2] Consume full fenced-code info strings when stripping:** `src/shared/text/strip-markdown.ts:9`
  - body: The fenced-code regex only accepts lowercase info strings and leaves the newline after the closing fence outside the match. Common fences such as `c++` or `c#` can leak remnants, and mid-message fenced blocks can gain an extra blank line in plain-text output.
  - confidence: 0.9
