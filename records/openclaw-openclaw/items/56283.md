Full review comments:

- **[P1] Preserve lightweight-format channels during auto-strip:** `src/infra/outbound/sanitize-text.ts:86`
  - body: `shouldStripMarkdown("whatsapp")` falls through to the default strip path, but `sanitizeForPlainText()` intentionally converts HTML to lightweight markers like `*bold*`, `_italic_`, and backtick code for channels that render that syntax. Running `stripMarkdown()` after that erases formatting owned by those channels, so WhatsApp-style output is silently downgraded.
  - confidence: 0.93
- **[P1] Honor renderer-owned plugin channels before auto-strip:** `src/infra/outbound/sanitize-text.ts:86`
  - body: The PR head's auto predicate is based on a hard-coded Markdown-capable set, so plugin channels such as Matrix are treated as plain text even though their outbound path expects Markdown and renders it to native HTML. That strips links, code, lists, and emphasis before the plugin renderer can handle them.
  - confidence: 0.9
- **[P2] Resolve markdown.strip with account scope:** `src/infra/outbound/deliver.ts:371`
  - body: `resolveChannelMarkdownConfig()` only reads `channels[channel].markdown`, while the existing Markdown table config resolver checks `channels[channel].accounts[accountId].markdown` first. Multi-account operators would have account-specific `strip` overrides silently ignored.
  - confidence: 0.88
- **[P1] Avoid mutating identifiers in the default strip path:** `src/infra/outbound/deliver.ts:345`
  - body: This branch puts `stripMarkdown()` on a default outbound path, but the PR head predates current main's identifier-safe underscore behavior and can remove underscores from text like `foo_bar_baz` or URL paths. A rebased implementation must preserve the current word-boundary fix before using the helper broadly.
  - confidence: 0.9
- **[P2] Consume complete fenced-code fences when stripping:** `src/shared/text/strip-markdown.ts:9`
  - body: The fenced-code regex accepts only lowercase language tags and leaves the newline after the closing fence outside the match, so fences such as `c++` can leak suffix text and mid-message blocks can gain extra blank lines in delivered plain text.
  - confidence: 0.86
