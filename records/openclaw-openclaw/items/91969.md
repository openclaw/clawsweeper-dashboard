## Summary

Keep this PR open: the stale-cache clear was repaired, but the exporter still stores session IDs and caller-selectable session keys in one raw-string namespace, allowing cross-run attribution collisions; direct live proof of the bundled exporter path is also still missing.

## What This Changes

Adds an opt-in gateway and SDK `clientContext` bag, forwards it through trusted diagnostic private data, and exports it as `openclaw.client.*` attributes on diagnostics-otel model-call spans.

## Best Possible Solution

Namespace cache aliases by identity type, preserve clear-on-unseeded behavior, document the bounded no-secrets/cardinality contract, and demonstrate the exact bundled exporter against a live OTel backend before merge.

## Reproduction Assessment

Yes. Seed context for a run whose session key equals another run's session ID, then resolve or clear the second run; the shared raw-string cache selects the wrong bag or deletes the wrong entry.

## Solution Assessment

No. The trusted private-data route is coherent, but the cache must make the two session identity types distinct and the bundled exporter needs direct live proof before this is the safest implementation.
