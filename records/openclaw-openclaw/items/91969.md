## Label Justifications

- `P1`: The patch can silently attribute a later run to the wrong upstream caller, corrupting a user-visible observability workflow.
- `merge-risk: 🚨 compatibility`: The PR expands gateway protocol, generated Swift, SDK, and plugin diagnostic contracts without complete public contract documentation.

- `merge-risk: 🚨 security-boundary`: Caller-controlled opaque metadata crosses into trusted diagnostics and exported telemetry, where stale state can expose or misassociate identity information.

## Summary

Keep this PR open for repair: the approach is plausible and the public event payload remains private, but the bundled exporter can silently reuse a prior run’s attribution when a session is reused without context. The supplied live evidence validates the trusted forwarding seam, not the new in-tree cache and attribute-stamping path.

## What This Changes

The PR adds an opt-in gateway and SDK `clientContext` field, carries it through trusted diagnostic private data, and maps it onto diagnostics-otel model-call span attributes.

## Best Possible Solution

Use explicit set-and-clear semantics for every cached session alias, cover session reuse at the exporter level, document the public contract, and demonstrate a redacted live trace produced by the bundled diagnostics-otel exporter.

## Reproduction Assessment

Yes, at source level: seed context for a session, reuse the same aliases without context, then emit a model-call event; the exporter retains the old value because undefined lifecycle updates are ignored. Tests were not executed because this review is read-only.

## Solution Assessment

No, not yet. The trusted-private-data seam is reasonable, but the exporter needs symmetric clear semantics, public documentation, and direct live proof before this is the narrowest safe implementation.
