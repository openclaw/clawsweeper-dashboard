## Summary

Keep this PR open, but do not merge it yet: the new capability is not present on current main and the live proof is strong, yet raw session alias keys can silently apply one run’s attribution to another, while the new cross-language telemetry contract still needs explicit safety guidance and owner approval.

## What This Changes

Adds an optional caller-supplied `clientContext` to gateway and SDK agent requests, propagates it through trusted diagnostic private data, and exports it as `openclaw.client.*` attributes on diagnostics-otel model-call spans.

## Best Possible Solution

Keep the trusted private-data design, but namespace cache aliases by identity type, add cross-type collision and clearing tests, document that callers must use stable bounded non-sensitive attribution values, and obtain explicit gateway/diagnostics owner approval before merge.

## Reproduction Assessment

Yes for the blocking defect. Source inspection gives a deterministic path where one run’s `sessionId` equals another run’s `sessionKey`, causing the shared raw-string cache to resolve or clear the wrong attribution without malformed input.

## Solution Assessment

No, not as written. The trusted private-data seam is the narrowest existing architecture, but identity namespaces must remain distinct and the new public telemetry contract needs explicit safety guidance and owner approval.

Full review comments:

- **[P1] Namespace session IDs and keys before caching:** `extensions/diagnostics-otel/src/client-context-attributes.ts:16-23`
  - body: `clientContextKeys` returns bare values for two independent identity fields and stores them in the same map. If run A has `sessionId = "x"` and run B has `sessionKey = "x"`, B can overwrite, resolve, or clear A’s attribution, silently stamping the wrong caller on model spans. Return typed aliases such as `id:${sessionId}` and `key:${sessionKey}`, and add cross-type resolve and clearing coverage.
  - confidence: 0.98
- **[P1] Document safe clientContext value requirements:** `docs/plugins/reference/diagnostics-otel.md:19-26`
  - body: The documented opaque bag accepts arbitrary caller values that become exported span attributes, but does not tell users to exclude secrets, personal data, or uncontrolled-cardinality identifiers. Add explicit safety and cardinality guidance before establishing this public gateway and SDK contract.
  - confidence: 0.96
