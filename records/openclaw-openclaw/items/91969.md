## Summary

This PR has strong single-run live proof and addresses the prior cache-collision and telemetry-safety feedback, but it is not merge-ready because request admission writes attribution into mutable session-scoped state before the run enters its serialized session lane. A later request for the same session can therefore replace the active run’s context and misattribute its subsequent model telemetry.

## What This Changes

The PR adds an optional caller-supplied `clientContext` to gateway and SDK agent requests, propagates it through trusted diagnostic private data, and exports it as `openclaw.client.*` attributes on diagnostics-otel model-call spans.

## Best Possible Solution

Carry the normalized context as a prepared run-owned fact through execution and diagnostic emission, or index it by `runId` and resolve model events by that identifier; then prove two differently attributed requests sharing one session remain isolated and obtain explicit approval for the cross-client contract.

## Reproduction Assessment

Yes from source: admit two gateway agent requests with different `clientContext` values for the same session while the first run occupies the session lane; the second request mutates shared diagnostic context before its own run starts.

## Solution Assessment

No; per-run attribution should be owned by the concrete run rather than one mutable session-global value written at request admission.
