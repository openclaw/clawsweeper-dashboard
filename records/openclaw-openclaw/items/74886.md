## Summary

Keep open. The report is credible and still not proven fixed on current main: WhatsApp now suppresses payloads already marked as reasoning, but its dispatch options still do not provide `onReasoningStream`, and the paired open PR #75258 targets exactly that remaining stream-boundary gap. ## What This Changes

The issue reports a 2026.4.27 regression where a WhatsApp direct session leaked internal reasoning, became unstable, and appeared to stick to a MiniMax fallback instead of the configured Codex primary.

## Best Possible Solution

Treat the WhatsApp reasoning disclosure as the first active fix: review and land or revise PR #75258 with a regression test that proves WhatsApp supplies a reasoning-stream sink and does not deliver reasoning chunks. Then split any remaining fallback-status or gateway-stall work into narrower follow-up issues using sanitized artifacts.
