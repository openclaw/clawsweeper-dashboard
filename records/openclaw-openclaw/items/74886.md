## Summary

Keep open. Current main still lacks the WhatsApp `onReasoningStream` reply option, the existing guards only prove suppression for payloads already marked as reasoning, and the reporter has current 2026.4.29 evidence plus an open paired PR targeting the first suspected leak path. ## What This Changes

The issue reports a 2026.4.27 WhatsApp direct-session regression where reasoning text became user-visible, the active model appeared to stick to MiniMax instead of Codex, and rollback to 2026.4.23 restored behavior.

## Best Possible Solution

Keep this as an active release-regression issue until PR #75258 is reviewed and either landed or revised with a WhatsApp regression test, then split any remaining MiniMax/status fallback and gateway-stability evidence into the narrow canonical issues that own those paths.
