## Summary

Keep open. Current main still lacks the WhatsApp `onReasoningStream` reply option that the report and paired open PR identify as the likely reasoning-leak boundary, and the broader MiniMax fallback/session-stability symptoms are not proven fixed by current source or related issue state.

## What This Changes

The issue reports a WhatsApp direct-session regression after updating to 2026.4.27 where reasoning text leaked to chat, the session appeared to stick to MiniMax instead of the configured Codex primary, and rollback to 2026.4.23 restored normal behavior.

## Best Possible Solution

Land or revise the WhatsApp-owned `onReasoningStream` hardening with a regression test proving reasoning chunks are not delivered through WhatsApp, then retest a current build to separate any remaining MiniMax fallback/status persistence or gateway-stability behavior into narrower follow-up issues.

## Reproduction Assessment

Partial. The reasoning-leak boundary has a high-confidence source reproduction path because current main omits the WhatsApp reasoning-stream callback, but the complete rollback/fallback/stability sequence has not been reproduced live against current main.

## Solution Assessment

Unclear for the full issue. The paired PR is the narrowest maintainable first fix for the WhatsApp reasoning disclosure, but it is not sufficient evidence that the MiniMax fallback state or broader session instability is solved.
