## Summary

Keep open. The reporter provided credible sanitized rollback and transcript evidence for a privacy-sensitive WhatsApp reasoning leak plus model/fallback instability, and current main only proves suppression for payloads already marked as reasoning; the WhatsApp-specific reasoning-stream boundary remains paired with open PR #75258 and the MiniMax/status/stability parts are not proven fixed.

## What This Changes

The issue reports a regression after upgrading from 2026.4.23 to 2026.4.27 where a WhatsApp direct session became unstable, exposed internal thinking text, and appeared to stick on MiniMax instead of the configured Codex primary model.

## Best Possible Solution

Treat this as an active release-regression cluster: review and either land or revise PR #75258 for the WhatsApp reasoning-stream suppression boundary, then use sanitized artifacts or a maintainer debug export path to split any remaining fixes across WhatsApp delivery, shared reply assembly, MiniMax fallback/status state, and gateway stall regressions.
