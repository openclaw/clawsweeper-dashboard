## Summary

Keep open: current main and v2026.6.10 still expose the reported compaction budgets as absolute numeric values, and an open linked PR plus competing candidate PRs need maintainer API-shape review before this issue can close.

## What This Changes

Request context-window-relative compaction threshold configuration so memory flush, reserve, keep-recent, and reserve-floor budgets scale when a session switches between models with different context windows.

## Best Possible Solution

Keep this as the canonical issue, choose one backward-compatible compaction config contract, then land one implementation covering schema, types, all runtime readers, docs/help metadata, tests, and real behavior proof.

## Reproduction Assessment

Yes at source level: current schema, types, docs, and runtime readers are numeric-only for the reported fields and subtract absolute values from the active context window. I did not run a live gateway session switching between 200K and 1M-token models.

## Solution Assessment

Unclear: percentage strings are plausible, but additive share fields and default scaling are also open candidates. The best path is a maintainer decision on one public config contract, followed by one complete implementation.
