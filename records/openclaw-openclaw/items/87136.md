## Label Justifications

- `P2`: The issue can make mixed-context-window sessions unreliable, but it is a bounded compaction config/runtime improvement rather than an emergency outage.
- `impact:session-state`: Compaction thresholds control when session context is flushed, summarized, and retained.

## Summary

Keep open: current main and v2026.6.8 still expose the reported compaction thresholds as numeric token counts only, and the issue already has open implementation candidates with competing public config shapes.

## What This Changes

The issue requests context-window-relative compaction thresholds so absolute token budgets do not become unusable when sessions switch between models with different context windows.

## Best Possible Solution

Keep this as the canonical issue, choose one backward-compatible compaction config contract, then land a single implementation covering schema, types, all runtime readers, docs/metadata, tests, and real behavior proof.

## Reproduction Assessment

Partially: source inspection gives a high-confidence path to reproduce the missing relative-threshold capability because current schema, types, docs, and runtime readers are numeric-only. I did not run a live gateway session switching between 200K and 1M-token models.

## Solution Assessment

Unclear: percentage strings and additive share fields are both plausible, but this is a public config/API decision and needs one maintainer-approved shape before implementation is best.
