## Summary

Keep open: current main still defines the requested compaction thresholds as numeric token counts only. Existing code has shipped context-window guardrails for some reserve-token paths, but it does not implement percentage strings or a single relative-threshold contract across memory flush, Pi settings, and docs.

## What This Changes

The issue requests relative, active-model-context-aware compaction thresholds instead of absolute token counts for compaction and memory-flush settings.

## Best Possible Solution

Keep numeric values backward-compatible, choose one canonical relative-threshold representation, and resolve it at runtime against the active session model context window across Pi compaction, preflight compaction, and memory flush with focused docs and tests.

## Reproduction Assessment

## Solution Assessment

Unclear. Percentage strings are a plausible solution, but maintainers should first decide the canonical config semantics and whether this remains config-only or touches plugin memory-flush contracts.
