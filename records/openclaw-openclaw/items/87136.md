- `impact:session-state`: Compaction thresholds control when session context is flushed, summarized, retained, or reset.

## Summary

Keep open: current main and v2026.6.9 still expose the reported compaction thresholds as absolute numeric token counts, and open implementation PRs are waiting on maintainer choice of the public config shape.

## What This Changes

The issue requests context-window-relative compaction thresholds so reserve, keep-recent, floor, and memory-flush soft thresholds scale with the active model after model switches.

## Best Possible Solution

Keep this as the canonical issue, choose one backward-compatible compaction config contract, then land a single implementation covering schema, types, all runtime readers, docs/help metadata, tests, and real behavior proof.

## Reproduction Assessment

Yes at source level: current schema, types, docs, and runtime readers are numeric-only for the reported fields. I did not run a live gateway session switching between 200K and 1M-token models.

## Solution Assessment

Unclear: percentage strings and additive share fields are both plausible, but this is a public config/API decision. The best path is to pick one contract and update every reader and public surface together.
