## Summary

Keep open: current main still uses numbered config backup rotation, and this PR is a plausible implementation path, but the PR head still has blocking migration-retention and collision-retention bugs plus missing real behavior proof before merge.

## What This Changes

The PR changes config backup rotation from numbered `.bak.N` slots to UTC datetime-stamped backup files, updates pruning and permission hardening, and expands backup-rotation tests.

## Best Possible Solution

Choose one timestamp-suffix implementation for the linked issue that preserves or converts legacy numeric backups, caps retention, handles collisions deterministically, hardens permissions, and includes focused tests plus real config-write proof.

## Reproduction Assessment

Yes for the review findings: PR-head source inspection shows legacy numbered backups are unlinked as orphans, collision allocation reuses the first missing suffix, and the matcher accepts `-00`/`-01`. I did not run tests because this was a read-only review.

## Solution Assessment

No, not yet. The file/test boundary is appropriate, but the PR should preserve or convert existing rollback history, allocate from the highest existing collision suffix plus one, tighten suffix validation, and add real behavior proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve legacy numbered backups during migration:** `src/config/backup-rotation.ts:153-159`
  - body: On the first post-upgrade maintenance cycle, existing `.bak.1` through `.bak.4` files are classified as orphans and unlinked. The linked request calls out migration safety and current main treats those files as the recovery ring, so this can discard known-good rollback states immediately after upgrading. Preserve or convert the legacy slots under the same total backup cap instead of deleting them as orphans.
  - confidence: 0.92
- **[P2] Choose collision suffixes from the highest existing value:** `src/config/backup-rotation.ts:69-73`
  - body: When cleanup has pruned lower same-second suffixes, this loop reuses the first missing value such as `-02` even if newer `-03..-06` backups remain. The next cleanup sorts that freshly rotated backup as oldest and can delete it immediately, so rapid writes stop preserving the newest rollback points. Parse existing suffixes for the timestamp and append max plus one.
  - confidence: 0.91
- **[P3] Reject zero collision suffixes:** `src/config/backup-rotation.ts:32`
  - body: The matcher accepts `-00` and `-01` because `\d{2,}` includes them, even though the allocator starts at `-02`. User-created files with those suffixes would be treated as managed backups and count toward pruning, so tighten the pattern and add assertions for `-00` and `-01`.
  - confidence: 0.86
- **[P3] Make the collision test deterministic:** `src/config/config.backup-rotation.test.ts:231-243`
  - body: The test derives a timestamp from the first rotation, then calls `rotateConfigBackups` again using the wall clock. If the second changes between those calls, the second rotation uses a different timestamp and the test fails even when the collision logic is correct. Freeze time or inject the timestamp source for this test.
  - confidence: 0.78
