## Summary

Keep open: current main still uses numeric config backup rotation, and this PR is a plausible implementation path, but the latest head still has data-retention correctness issues and lacks required real behavior proof.

## What This Changes

The PR changes config backup rotation from numeric `.bak.N` slots to UTC datetime-stamped backup files and updates the focused backup-rotation tests.

## Best Possible Solution

Land one chosen datetime-backup implementation that preserves or converts existing numbered backups, uses collision-safe names, keeps bounded retention and owner-only permissions, and updates the focused tests together.

## Reproduction Assessment

Yes for the PR defects: latest PR-head source shows legacy numbered backups are classified as orphans, collision allocation uses the first missing suffix, and the matcher accepts `-00`/`-01`. I did not run tests because this is a read-only review.

## Solution Assessment

No, not yet. The implementation touches the right files, but it should preserve or convert legacy rollback history, allocate collision suffixes after the highest existing value, tighten suffix validation, and add real config-write proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve legacy numbered backups during migration:** `src/config/backup-rotation.ts:153`
  - body: On the first post-upgrade cleanup, existing `.bak.1` through `.bak.4` files enter the orphan list and are unlinked, even though current main treats them as the recovery ring and the linked feature request calls out migration safety. Preserve or convert those slots under the same retention cap instead of deleting rollback history.
  - confidence: 0.93
- **[P2] Choose collision suffixes from the highest existing value:** `src/config/backup-rotation.ts:69-73`
  - body: When cleanup has already pruned a lower same-second suffix, this loop can reuse `-02` while newer `-03` and later backups remain. The next cleanup then sorts the freshly rotated backup as oldest, so rapid writes can discard the newest rollback point; parse existing suffixes and append max plus one.
  - confidence: 0.91
- **[P3] Reject zero collision suffixes:** `src/config/backup-rotation.ts:32`
  - body: The matcher accepts `-00` and `-01` because `\d{2,}` includes them, even though the allocator starts at `-02`. User-created files with those suffixes would be treated as managed backups and count toward pruning, so tighten the pattern and assert these cases.
  - confidence: 0.86
- **[P3] Make the collision test deterministic:** `src/config/config.backup-rotation.test.ts:243`
  - body: This test derives the timestamp from a first rotation, then calls `rotateConfigBackups` again using the wall clock. If the second call crosses a second boundary, it creates a different timestamp and the test fails even when collision handling is correct; freeze or inject the timestamp source.
  - confidence: 0.78
