## Summary

Keep open: current main still uses numbered config backups, and this PR is a relevant implementation, but the branch still has a collision-rotation bug that can drop newer rollback points during same-second bursts.

## What This Changes

The PR changes config backup rotation from numbered `.bak.N` files to UTC datetime-stamped backups, with updated pruning, permission hardening, collision handling, and focused tests.

## Best Possible Solution

Keep the timestamp suffix implementation scoped to the config backup module, but fix collision suffix selection to append after the highest existing same-timestamp suffix, make the collision tests deterministic, add the required changelog entry, and have maintainers confirm legacy-backup migration policy before merge.

## Reproduction Assessment

Yes, from source inspection: with a fixed timestamp and existing `-03..-06` files but no `-02`, the PR loop reuses `-02`, and the same cleanup order prunes that newly-created backup. I did not run the test locally because `pnpm` is unavailable.

## Solution Assessment

No, not as-is. The module boundary is appropriate, but the collision logic should choose `max(existingSuffix)+1`, the wall-clock collision test should be deterministic, and the user-facing filename change needs a changelog entry.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Choose collision suffix from the highest existing value:** `src/config/backup-rotation.ts:70-73`
  - body: When cleanup has already pruned a low suffix such as `-02` while newer siblings like `-03..-06` remain, this loop reuses the first missing suffix instead of continuing after the highest one. The next prune then treats that newly-created `-02` as oldest, so rapid same-second config writes stop advancing the backup ring and drop recent rollback states.
  - confidence: 0.94
- **[P2] Freeze the timestamp in collision tests:** `src/config/config.backup-rotation.test.ts:243`
  - body: This test captures the timestamp from one rotation and then calls `rotateConfigBackups` again using the real wall clock. If the second rolls over between those calls, the second rotation creates a different bare timestamp instead of `stampSuffix-03`, making the test fail nondeterministically; freeze time or inject the timestamp source for this case.
  - confidence: 0.86
- **[P3] Add the changelog entry for the backup filename change:** `src/config/backup-rotation.ts:40-44`
  - body: This changes a user-visible config backup filename and migration behavior, but the PR only touches the implementation and test files. Repo policy requires user-facing `feat` changes to include a `CHANGELOG.md` entry before merge.
  - confidence: 0.84
