  - body: `cleanOrphanBackups` treats every non-datetime `.bak.*` entry as an orphan, so users upgrading with `.bak.1` through `.bak.4` lose rollback snapshots on the first maintenance cycle. Preserve or convert the numeric ring before deleting it.
  - confidence: 0.95

  - body: Current main exports `createPreUpdateConfigSnapshot` and the update command imports it before repair writes. This stale branch's backup module lacks that export and its tests, so conflict resolution must preserve the current snapshot surface instead of replacing the file with the branch version.
  - confidence: 0.9
- **[P2] Allocate collision suffixes after the highest sibling:** `src/config/backup-rotation.ts:69-73`
  - body: When cleanup has pruned a low same-second suffix while higher siblings remain, this loop reuses the first missing value such as `-02`. That new backup then sorts older than existing siblings and can be pruned immediately, dropping the newest rollback point.
  - confidence: 0.91
- **[P3] Reject unmanaged low collision suffixes:** `src/config/backup-rotation.ts:32`
  - body: The matcher accepts `-00` and `-01` via `\d{2,}`, but the allocator starts at `-02`. User-created files with those suffixes would be treated as managed backups and count toward pruning, so tighten the matcher and cover those cases.
  - confidence: 0.84
- **[P3] Make the collision regression test deterministic:** `src/config/config.backup-rotation.test.ts:243`
  - body: The test learns a timestamp from one rotation and then calls `rotateConfigBackups` again using a fresh wall clock. If the second ticks between calls, the expected `-03` sibling is not created even when collision handling works.
  - confidence: 0.78
