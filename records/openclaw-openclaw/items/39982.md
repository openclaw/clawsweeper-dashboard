## Summary

Keep open. Current main still uses numeric config backup rotation, and the PR branch is a real implementation candidate, but it still has a concrete same-second collision retention bug and overlaps with the open sibling implementation #40392, so maintainers should choose and finish one canonical design before merge.

## What This Changes

This PR changes config backup rotation from numeric `.bak.1` slots to UTC datetime-stamped `.bak.YYYYMMDD-HHmmss` files, updates cleanup and permission hardening for datetime backups, removes non-datetime backup orphans, and expands backup-rotation tests.

## Best Possible Solution

Merge one canonical datetime-backup implementation only after collision allocation is monotonic from the highest existing suffix or otherwise collision-free, retention remains bounded, permission hardening covers every retained backup, legacy numbered backups have an explicit migration policy, and the user-facing change is recorded in the changelog.

## Reproduction Assessment

Yes. A static trace is enough: with same-second entries such as `openclaw.json.bak.T-03` through `T-06`, this PR chooses the first missing `T-02`; cleanup then sorts `T-02` as oldest and removes the freshly rotated backup.

## Solution Assessment

No. The direction is plausible, but this branch is not yet the best fix because collision numbering should continue after the highest retained suffix, the test needs deterministic time control, the changelog is missing, and maintainers need to settle the canonical design against #40392.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Continue after the highest collision suffix:** `src/config/backup-rotation.ts:69-73`
  - body: When lower same-second suffixes have already been pruned, this loop reuses the first missing suffix such as `-02` even if newer siblings like `-03..-06` still exist. The following cleanup pass sorts by numeric suffix and removes that newly rotated backup as the oldest, so burst writes can stop advancing recent rollback points. Pick `max(existingSuffix) + 1` instead.
  - confidence: 0.92
- **[P3] Stabilize the collision regression test:** `src/config/config.backup-rotation.test.ts:231-244`
  - body: This test learns the current timestamp from one rotation and then expects the next rotation to use the same second. If execution crosses a one-second boundary between those calls, the second rotation gets a new timestamp and the assertion for the old `-03` suffix fails. Inject a clock or timestamp parameter for this regression path.
  - confidence: 0.78

  - body: This is a user-facing `feat(config)` change to backup filenames and migration behavior, but the PR only changes code and tests. OpenClaw policy requires user-facing `feat`, `fix`, and `perf` changes to add a `CHANGELOG.md` entry before merge.
  - confidence: 0.87
