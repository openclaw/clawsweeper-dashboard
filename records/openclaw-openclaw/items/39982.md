## Summary

Keep open. Current main still uses numeric config backup rotation; this PR targets the right module but still deletes legacy numbered backups, has a collision suffix reuse bug, and lacks real behavior proof, with a competing open implementation requiring maintainer choice.

## What This Changes

The PR changes config backup rotation from numeric `.bak.N` slots to UTC datetime-stamped `.bak.YYYYMMDD-HHmmss` files and updates cleanup, permission hardening, and backup-rotation tests.

## Best Possible Solution

Land one canonical timestamp-backup implementation that preserves or converts existing numbered backups, allocates collision suffixes from the highest existing suffix, uses deterministic tests, and includes real config-write proof.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the PR defects by source inspection: legacy numbered backups are classified as orphans, collision suffix allocation can reuse pruned lower suffixes, and one collision test depends on the wall clock; the underlying request is a feature change, not a current-main bug.

## Solution Assessment

Is this the best way to solve the issue? No. The backup-rotation module is the right boundary, but this branch still needs safe legacy migration, max-suffix collision allocation, tighter suffix validation/tests, real behavior proof, and maintainer choice versus https://github.com/openclaw/openclaw/pull/40392.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Preserve legacy backup snapshots during migration:** `src/config/backup-rotation.ts:157-159`
  - body: `cleanOrphanBackups` pushes every non-datetime `.bak.*` entry into `orphans` and unlinks it, so users upgrading with current `.bak.1` through `.bak.4` rollback files lose those snapshots on the first config write. The linked feature request calls out migration safety, so preserve or convert the numbered backups under the same cap instead of deleting them as stale.
  - confidence: 0.92
- **[P2] Allocate collision suffixes after the highest existing suffix:** `src/config/backup-rotation.ts:69-73`
  - body: This loop chooses the first missing suffix starting at `-02`. After cleanup has pruned low same-second suffixes while newer higher ones remain, the next rotation can reuse `-02`; pruning then treats that fresh backup as older and can delete it immediately, dropping the newest rollback state.
  - confidence: 0.9
- **[P3] Reject managed suffixes below the allocator range:** `src/config/backup-rotation.ts:32`
  - body: The matcher accepts `-00` and `-01` because `\d{2,}` includes them, even though the allocator starts at `-02`. User-created files with those suffixes would be treated as managed backups and count toward pruning, so tighten the regex and cover those exact cases.
  - confidence: 0.86
- **[P3] Make the collision test independent of the wall clock:** `src/config/config.backup-rotation.test.ts:231-243`
  - body: The test learns a timestamp from the first rotation and then calls `rotateConfigBackups` again using the real clock. If the second changes between those calls, the second rotation uses a new timestamp and the expected `-03` sibling is not created even when collision handling works.
  - confidence: 0.78
