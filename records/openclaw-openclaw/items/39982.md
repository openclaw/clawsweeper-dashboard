## Summary

Keep open: current main still uses numeric config backup rotation, and this PR is useful but still has a source-reproducible collision-retention bug plus small matcher/changelog blockers before it is safe to merge.

## What This Changes

The PR rewrites config backup rotation to use UTC datetime-stamped `.bak.YYYYMMDD-HHmmss` files, prune and chmod timestamped backups, remove legacy numbered backups, and update backup-rotation tests.

## Best Possible Solution

Repair the collision allocator and suffix matcher, add the changelog entry, then have maintainers choose one timestamped-backup implementation and close the competing path after the selected PR lands.

## Reproduction Assessment

Yes for the review finding: source inspection of the PR head shows a gap-filling suffix loop that can reuse `-02` while higher same-second suffixes remain, causing cleanup to prune the freshly rotated backup. I did not execute tests because the checkout had to remain read-only.

## Solution Assessment

No, not yet. The module and test boundary is appropriate, but the PR should allocate from the highest existing collision suffix, tighten accepted suffixes, and add the required changelog entry before maintainer product review.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.89

Full review comments:

- **[P2] Choose suffixes after the highest existing value:** `src/config/backup-rotation.ts:69-73`
  - body: When cleanup has pruned lower same-second suffixes, this loop stops at the first gap. With `-03..-06` present and `-02` missing, the next rotation writes `-02`; `cleanOrphanBackups` then sorts it before the existing siblings and removes the fresh backup, so rapid writes stop preserving recent rollback points. Parse existing same-timestamp suffixes and append max+1 instead.
  - confidence: 0.91
- **[P3] Reject zero and one collision suffixes:** `src/config/backup-rotation.ts:32`
  - body: The regex accepts `-00` and `-01` even though the allocator starts at `-02`. User-created files with those names are treated as managed backups and can consume or prune slots, so tighten the matcher and add assertions for those cases.
  - confidence: 0.84
- **[P3] Add the changelog entry:** `src/config/backup-rotation.ts:46-49`
  - body: This user-facing feature changes config backup filenames and upgrade cleanup behavior, but the PR modifies only code and tests. OpenClaw policy requires a `CHANGELOG.md` entry for user-facing feature changes before merge.
  - confidence: 0.86
