## Summary

Keep this PR open: current main still uses numeric config-backup rotation, but this branch is not merge-ready because the collision allocator can reuse a lower suffix after pruning and then let cleanup delete the newly rotated backup.

## What This Changes

This PR replaces numeric config backup slots with UTC datetime-stamped `.bak.YYYYMMDD-HHmmss` files, updates cleanup and permission hardening for those files, and expands config backup tests.

## Best Possible Solution

Merge one canonical datetime-backup implementation after collision allocation is monotonic or collision-free, retention remains bounded, permission hardening covers retained backups, legacy numbered backup migration is explicit, and the changelog records the user-facing change.

## Reproduction Assessment

Yes for the PR-introduced retention bug by source trace: with same-second backups such as `T-03..T-06` present and `T-02` pruned, this branch chooses `T-02`, after which cleanup can rank that fresh backup as oldest. I did not run tests because this review was read-only.

## Solution Assessment

No. The direction is plausible, but this branch is not yet the best fix until collision allocation continues after the highest existing suffix, the time-dependent test is stabilized, the changelog is added, and maintainers reconcile it with #40392 and the legacy-backup policy.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Continue after the highest collision suffix:** `src/config/backup-rotation.ts:68-73`
  - body: When lower same-second suffixes have already been pruned, this loop reuses the first missing suffix such as `-02` even if newer siblings like `-03..-06` still exist. The following cleanup pass sorts by numeric suffix and can remove the freshly rotated backup as the oldest, so burst writes can stop advancing recent rollback points. Pick `max(existingSuffix) + 1` instead.
  - confidence: 0.92
- **[P3] Stabilize the collision regression test:** `src/config/config.backup-rotation.test.ts:231-243`
  - body: This test learns the current timestamp from one rotation and then expects the next rotation to use the same second. If execution crosses a one-second boundary between those calls, the second rotation gets a new timestamp and the assertion for the old `-03` suffix fails. Inject a clock or timestamp parameter for this regression path.
  - confidence: 0.78
- **[P3] Reject unproduced zero collision suffixes:** `src/config/backup-rotation.ts:27-32`
  - body: The comment says suffixes below `-02` are excluded, but `(?:-\d{2,})?` still accepts `-00` and `-01`. Those user-created files would be managed as legitimate datetime backups even though rotation never writes them. Tighten the regex or update the stated contract.
  - confidence: 0.79
- **[P3] Add the required changelog entry:** `src/config/backup-rotation.ts:16-24`
  - body: This is a user-facing config feature that changes backup filenames and cleanup behavior, but the diff only changes code and tests. OpenClaw policy requires user-facing `feat`, `fix`, and `perf` changes to add a `CHANGELOG.md` entry before merge.
  - confidence: 0.87
