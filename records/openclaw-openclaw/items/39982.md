## Summary

Keep open: current main and v2026.5.7 still use numbered config backup rotation, and this PR is a plausible implementation path, but the latest head still deletes legacy numbered recovery backups, can recycle lower same-second collision suffixes, and lacks contributor real behavior proof.

## What This Changes

The PR replaces the numbered config backup ring with UTC datetime-stamped backup files and updates backup cleanup, permission hardening, collision handling, and focused tests.

## Best Possible Solution

Land one canonical timestamp-backup implementation that preserves or converts legacy numbered backups, allocates collision names from the max existing suffix, tightly validates managed suffixes, keeps bounded retention and owner-only permissions, and includes focused tests plus real config-write proof.

## Reproduction Assessment

Yes for the PR defects by source inspection: current head deletes legacy numbered backups via the orphan path, reuses the first missing collision suffix, accepts invalid zero-padded suffixes, and has a wall-clock-dependent collision test. The underlying request is a feature change, not a current-main bug reproduction.

## Solution Assessment

No. `src/config/backup-rotation.ts` and its focused tests are the right boundary, but this patch is not the best mergeable solution until legacy migration, collision allocation, suffix validation, deterministic tests, proof, and canonical PR choice are resolved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve legacy backups during migration:** `src/config/backup-rotation.ts:157-159`
  - body: On the first post-upgrade maintenance cycle, existing `.bak.1` through `.bak.4` files are classified as orphans and unlinked. The linked request calls out migration safety and current main treats those files as the recovery ring, so preserve or convert them under the same total cap instead of deleting them.
  - confidence: 0.92
- **[P2] Choose collision suffixes from the max existing value:** `src/config/backup-rotation.ts:69-73`
  - body: When cleanup has pruned lower same-second suffixes, this loop can reuse `-02` while newer `-03..-06` backups remain. The next cleanup can then sort that freshly rotated backup as oldest and delete it immediately, so parse existing suffixes for the timestamp and append max plus one.
  - confidence: 0.9
- **[P3] Reject zero-padded collision suffixes below 02:** `src/config/backup-rotation.ts:32`
  - body: The matcher accepts `-00` and `-01` because `\d{2,}` includes them, even though the allocator starts at `-02`. User-created files with those suffixes would be treated as managed backups and count toward pruning, so tighten the pattern and test those values.
  - confidence: 0.86
- **[P3] Make the collision test deterministic:** `src/config/config.backup-rotation.test.ts:231-243`
  - body: The test learns a timestamp from the first rotation, then calls `rotateConfigBackups` again using the wall clock. If the second changes between those calls, the second rotation uses a different timestamp and the test fails even when collision handling is correct; freeze time or inject the timestamp source.
  - confidence: 0.78
