## Summary

Keep open: the requested timestamped backup behavior is not on current main, and this PR still has blocking backup-retention defects plus missing real behavior proof and overlap with another open implementation.

## What This Changes

The PR replaces numeric config backup rotation with UTC datetime-suffixed backups and updates cleanup, permission hardening, and focused backup-rotation tests.

## Best Possible Solution

Land one canonical timestamp-backup implementation that preserves or converts legacy numbered snapshots, allocates collision-safe bounded names, keeps owner-only permissions, updates focused tests, and includes real config-write proof.

## Reproduction Assessment

Yes for the PR defects by source inspection: cleanup unlinks legacy numbered backups, the allocator can reuse pruned low collision suffixes, the matcher accepts `-00` and `-01`, and the collision test depends on the wall clock. The underlying timestamp-backup request is a feature change, not a current-main bug reproduction.

## Solution Assessment

No. `src/config/backup-rotation.ts` is the right boundary, but this branch should not merge until legacy migration, collision allocation, suffix validation, deterministic testing, real proof, and overlap with https://github.com/openclaw/openclaw/pull/40392 are resolved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve legacy backup snapshots during migration:** `src/config/backup-rotation.ts:157-159`
  - body: The linked request calls out that existing `.bak.1` through `.bak.4` files need to be preserved or converted. This cleanup path classifies every non-datetime `.bak.*` entry as an orphan, so users lose rollback snapshots on the first maintenance cycle after upgrade.
  - confidence: 0.93
- **[P2] Allocate collision suffixes after the highest existing value:** `src/config/backup-rotation.ts:69-73`
  - body: When cleanup has already pruned a low same-second suffix while higher siblings remain, this loop reuses the first missing value such as `-02`. That new backup then sorts older than existing siblings and can be pruned immediately, dropping the newest rollback point.
  - confidence: 0.9
- **[P3] Reject suffixes below the allocator range:** `src/config/backup-rotation.ts:32`
  - body: The matcher accepts `-00` and `-01` via `\d{2,}`, but the allocator starts at `-02`. User-created files with those suffixes would be treated as managed backups and count toward pruning, so tighten the regex and cover those cases.
  - confidence: 0.84
- **[P3] Freeze the timestamp used by the collision test:** `src/config/config.backup-rotation.test.ts:243`
  - body: The test learns a timestamp from one rotation and then calls `rotateConfigBackups` again using a fresh clock. If the second ticks between calls, the expected `-03` sibling is not created even when collision handling works.
  - confidence: 0.78
