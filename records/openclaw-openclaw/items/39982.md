## Summary

Keep open: current main and the latest release still use numbered config backups, but this PR still has blocking backup-retention defects and lacks external real behavior proof. It also overlaps with another open implementation, so maintainers need to choose the canonical migration and collision policy.

## What This Changes

The PR replaces numeric `.bak.N` config backup rotation with UTC datetime-stamped backup files and updates cleanup, permission hardening, and focused backup-rotation tests.

## Best Possible Solution

Land one canonical timestamp-backup implementation that preserves or converts existing numbered backups, allocates collision suffixes after the highest existing suffix, keeps bounded retention and owner-only permissions, uses deterministic tests, and includes real config-write proof.

## Reproduction Assessment

Yes for the PR defects by source inspection: legacy numbered backups are classified as orphans, collision suffix allocation can reuse pruned lower suffixes, and one collision test depends on the wall clock. The underlying timestamp-backup request is a feature change, not a current-main bug.

## Solution Assessment

No. The backup-rotation module is the right boundary, but this branch still needs safe legacy migration, max-suffix collision allocation, tighter suffix validation/tests, real behavior proof, and maintainer choice versus https://github.com/openclaw/openclaw/pull/40392.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve legacy backup snapshots during migration:** `src/config/backup-rotation.ts:157-159`
  - body: `cleanOrphanBackups` sends every non-datetime `.bak.*` file to `orphans`, so existing `.bak.1` through `.bak.4` rollback files are unlinked on the first write after upgrade. The linked request calls for preserving or converting legacy numeric backups; keep those snapshots under the retention cap instead of deleting them as stale.
  - confidence: 0.93
- **[P2] Allocate collision suffixes after the highest existing one:** `src/config/backup-rotation.ts:69-73`
  - body: When cleanup has pruned low same-second suffixes while higher ones remain, this loop reuses the first missing value such as `-02`. The fresh backup can then sort older than existing siblings and be pruned immediately, dropping the newest rollback point; parse existing suffixes and append after the maximum.
  - confidence: 0.9
- **[P3] Reject suffixes below the allocator range:** `src/config/backup-rotation.ts:32`
  - body: The matcher accepts `-00` and `-01` via `\d{2,}`, but the allocator starts at `-02`. User-created files with those suffixes would be treated as managed backups and count toward pruning, so tighten the regex and cover those exact cases.
  - confidence: 0.86
- **[P3] Make collision tests independent of the wall clock:** `src/config/config.backup-rotation.test.ts:231-243`
  - body: The test records the first rotation's current-second stamp, then calls `rotateConfigBackups` again using a fresh clock. If the second ticks between calls, no `-03` sibling is created even when collision handling works; inject or freeze the timestamp for this path.
  - confidence: 0.78
