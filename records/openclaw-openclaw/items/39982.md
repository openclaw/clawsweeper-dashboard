## Summary

Keep open: current main still uses numbered config backups, and this PR is a plausible implementation but remains blocked by legacy-migration, collision-ordering, validation, and proof gaps.

## What This Changes

This PR changes config backup rotation from numbered `.bak.N` slots to UTC datetime-stamped `.bak.YYYYMMDD-HHmmss` files and updates cleanup, permission hardening, and tests.

## Best Possible Solution

Land one consolidated datetime-backup implementation that preserves or converts legacy numbered backups, uses collision-safe names, enforces bounded retention and owner-only permissions, and includes focused tests plus real config-write proof.

## Reproduction Assessment

Yes for the PR defects by source inspection: legacy numbered backups enter the orphan unlink path, same-second collision numbering can reuse the first missing suffix, and the matcher accepts unproduced zero-padded suffixes. The underlying request is a feature change, not a broken current-main bug.

## Solution Assessment

No, not yet. The file boundary is appropriate, but the branch should preserve or migrate legacy rollback history, allocate after the highest same-second suffix, tighten suffix validation, make the collision test deterministic, and add real behavior proof before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Preserve legacy backups during migration:** `src/config/backup-rotation.ts:157-159`
  - body: On the first post-upgrade maintenance cycle, existing `.bak.1` through `.bak.4` files are classified as orphans and unlinked. The linked request calls out migration safety and current main treats those files as the recovery ring, so preserve or convert them under the same total cap instead of deleting rollback history.
  - confidence: 0.92
- **[P2] Choose collision suffixes from the max existing value:** `src/config/backup-rotation.ts:69-73`
  - body: When cleanup has pruned lower same-second suffixes, this loop can reuse `-02` while newer `-03..-06` backups remain. The next cleanup then sorts that freshly rotated backup as oldest and can delete it immediately, so parse existing suffixes for the timestamp and append max plus one.
  - confidence: 0.9
- **[P3] Reject zero-padded invalid collision suffixes:** `src/config/backup-rotation.ts:32`
  - body: The matcher accepts `-00` and `-01` because `\d{2,}` includes them, even though the allocator starts at `-02`. User-created files with those suffixes would be treated as managed backups and count toward pruning, so tighten the pattern and test those values.
  - confidence: 0.86
- **[P3] Freeze time in the collision test:** `src/config/config.backup-rotation.test.ts:231-243`
  - body: The test learns a timestamp from the first rotation, then calls `rotateConfigBackups` again using the wall clock. If the second changes between those calls, the second rotation uses a different timestamp and the test fails even when collision handling is correct; inject or freeze the timestamp source.
  - confidence: 0.78
