## Summary

Keep open. Current main still uses numeric config backup rotation, and this PR is a plausible implementation of the requested feature, but the latest head still has a retention bug plus unresolved changelog and migration-design work before merge.

## What This Changes

This PR changes config backup rotation from numbered `.bak.N` files to UTC datetime-stamped `.bak.YYYYMMDD-HHmmss` files, updating cleanup, permission hardening, and backup-rotation tests.

## Best Possible Solution

Merge one canonical datetime-backup implementation after collision allocation remains monotonic, retention stays bounded, permission hardening covers retained backups, legacy numbered backups have an explicit migration policy, and the changelog records the user-facing change.

## Reproduction Assessment

Yes for the PR-introduced bug by source trace: with same-second backups like `T-03` through `T-06` present and `T-02` absent, this branch chooses `T-02`, then cleanup ranks it as oldest and can remove the freshly rotated backup.

## Solution Assessment

No. The direction is plausible, but this branch is not yet the best fix until suffix allocation continues after the highest existing collision, the time-sensitive test is stabilized, the changelog is added, and maintainers reconcile it with #40392.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Continue after the highest collision suffix:** `src/config/backup-rotation.ts:69-73`
  - body: When lower same-second suffixes have already been pruned, this loop reuses the first missing suffix such as `-02` even if newer siblings like `-03..-06` still exist. The cleanup pass then sorts by numeric suffix and can remove that newly rotated backup as the oldest, so burst writes stop advancing recent rollback points. Pick `max(existingSuffix) + 1` instead.
  - confidence: 0.92
- **[P3] Stabilize the collision regression test:** `src/config/config.backup-rotation.test.ts:231-243`
  - body: This test learns the current timestamp from one rotation and then expects the next rotation to use the same second. If execution crosses a one-second boundary between those calls, the second rotation gets a new timestamp and the assertion for the old `-03` suffix fails. Inject a clock or timestamp parameter for this regression path.
  - confidence: 0.78
- **[P3] Reject unproduced zero-padded suffixes:** `src/config/backup-rotation.ts:32`
  - body: The comment says collision suffixes start at `-02`, but this regex also accepts user-created `-00` and `-01` files as managed datetime backups. Tighten the optional suffix so only produced collision values are retained and counted against capacity.
  - confidence: 0.72
- **[P3] Add the required changelog entry:** `src/config/backup-rotation.ts:40-44`
  - body: This is a user-facing config feature that changes backup filenames and legacy cleanup behavior, but the PR only changes code and tests. OpenClaw policy requires a `CHANGELOG.md` entry for user-facing `feat`, `fix`, and `perf` changes before merge.
  - confidence: 0.87
