## Summary

Keep this PR open, but it is not ready to merge: current main still uses numeric config backups, while this branch still has a collision-allocation bug, lacks the required changelog entry, and overlaps with another open implementation.

## What This Changes

This PR changes config backup rotation from numeric `.bak.1` slots to UTC datetime-stamped `.bak.YYYYMMDD-HHmmss` files, updates cleanup and permission hardening for datetime backups, and expands backup-rotation tests.

## Best Possible Solution

Merge one canonical datetime-backup implementation after collision allocation remains monotonic or otherwise collision-free, retention stays bounded, permission hardening covers retained backups, legacy numbered backups have an explicit migration policy, and the changelog records the user-facing behavior change.

## Reproduction Assessment

Yes for the PR-introduced bug by source trace: with same-second entries such as `T-03` through `T-06` and `T-02` absent, this branch chooses `T-02`, then cleanup ranks it as oldest and can remove the freshly rotated backup. I did not execute tests because the review was read-only and `pnpm` is unavailable.

## Solution Assessment

No. The direction is plausible, but this branch is not yet the best fix until it allocates after the highest existing collision suffix, stabilizes the time-dependent regression test, adds the required changelog entry, and is reconciled with #40392.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Continue after the highest collision suffix:** `src/config/backup-rotation.ts:69-73`
  - body: When lower same-second suffixes have already been pruned, this loop reuses the first missing suffix such as `-02` even if newer siblings like `-03..-06` still exist. The following cleanup pass sorts by numeric suffix and removes that newly rotated backup as the oldest, so burst writes can stop advancing recent rollback points. Pick `max(existingSuffix) + 1` instead.
  - confidence: 0.92
- **[P3] Stabilize the collision regression test:** `src/config/config.backup-rotation.test.ts:231-244`
  - body: This test learns the current timestamp from one rotation and then expects the next rotation to use the same second. If execution crosses a one-second boundary between those calls, the second rotation gets a new timestamp and the assertion for the old `-03` suffix fails. Inject a clock or timestamp parameter for this regression path.
  - confidence: 0.78
- **[P3] Add the required changelog entry:** `src/config/backup-rotation.ts:16-24`
  - body: This is a user-facing config feature that changes backup filenames and migration behavior, but the PR only changes code and tests. OpenClaw policy requires user-facing `feat`, `fix`, and `perf` changes to add a `CHANGELOG.md` entry before merge.
  - confidence: 0.87
