## Summary

Keep open: current main still has the Xiaomi MiMo visible-output gap, and the PR is a plausible narrow fix, but the branch is not mergeable in its current form because it uses stale Pi package/test-helper contracts and has no after-fix real MiMo proof. The newer Xiaomi replay work in https://github.com/openclaw/openclaw/pull/81589 is related but targets multi-turn `reasoning_content` replay, not this PR's final-answer-as-visible-text behavior.

## What This Changes

The PR adds a Xiaomi-only stream wrapper and regression tests to convert reasoning-only final messages from `mimo-v2-pro` and `mimo-v2-omni` into visible text.

## Best Possible Solution

Update the Xiaomi-owned stream normalization onto current plugin contracts, keep it scoped to visible reasoning-only final answers, add redacted real MiMo proof, and land it separately from the multi-turn replay work if both behaviors remain needed.

## Reproduction Assessment

Yes, at source level. Current main exposes the affected Xiaomi MiMo models through OpenAI completions, maps `reasoning_content` to thinking blocks, and visible assistant extraction ignores thinking-only content; I did not run a live Xiaomi request in this read-only review.

## Solution Assessment

Mostly yes. A Xiaomi provider-owned stream wrapper is the narrow maintainable fix for the blank final-output case, but the branch must be rebased onto current package/test contracts and should not be conflated with the separate replay repair.
