## Summary

Keep open. Current main still has the Xiaomi provider on the OpenAI-compatible path without a Xiaomi stream normalization hook, while the shared parser maps `reasoning_content` into thinking blocks and the TUI visible reply path reads assistant text blocks. The PR is pointed at a real user-visible blank-reply bug, but its head is dirty against main and the added test uses an obsolete helper shape, so it needs a small repair before merge.

## What This Changes

The PR adds a Xiaomi-only stream wrapper and regression tests to convert thinking-only final messages for `mimo-v2-pro` and `mimo-v2-omni` into text while leaving non-target Xiaomi models unchanged.

## Best Possible Solution

Keep the fix in the Xiaomi plugin boundary: rebase or replace the PR on current main, update the tests to the current public plugin test runtime, add the required changelog entry, and validate the Xiaomi stream normalization without changing the generic OpenAI-compatible parser or non-Xiaomi providers.

Yes. The linked report gives concrete Xiaomi MiMo steps and sample `delta.reasoning_content` output with empty `content`; current main still maps that field to thinking and visible rendering reads assistant text blocks, matching the blank-reply failure path.

## Solution Assessment
