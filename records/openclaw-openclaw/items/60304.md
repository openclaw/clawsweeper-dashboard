## What This Changes

The PR adds a Xiaomi provider stream wrapper and regression tests to promote reasoning-only final messages from `mimo-v2-pro` and `mimo-v2-omni` into visible assistant text.

## Best Possible Solution

Rebase and repair the Xiaomi-owned wrapper on current main, use the renamed pi packages and public async SDK test helper, then add redacted real Xiaomi MiMo proof before merge.

## Reproduction Assessment

Yes, from source and the linked report rather than a live run in this review. Current main still maps `delta.reasoning_content` to thinking while Xiaomi MiMo Pro/Omni are OpenAI-completions reasoning models and visible rendering reads text blocks.

## Solution Assessment

Mostly yes. A Xiaomi plugin-owned `wrapStreamFn` is the narrow maintainable boundary, but this branch must be rebased and repaired for current package/test-helper contracts before it is a mergeable implementation.
