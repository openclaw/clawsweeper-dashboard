## Summary

Keep open: this is still the canonical PR for a real Xiaomi MiMo blank-response bug, but the branch is not mergeable against current main because it uses retired pi package names and a removed sync test helper, and it still lacks contributor-supplied real behavior proof.

## What This Changes

The PR adds a Xiaomi provider stream wrapper and regression tests to promote reasoning-only final messages from `mimo-v2-pro` and `mimo-v2-omni` into visible text.

## Best Possible Solution

Rebase the Xiaomi-owned wrapper onto current main using the current pi package names and async public test helper, keep the normalization scoped to MiMo Pro/Omni, then merge after redacted real Xiaomi MiMo proof is added.

## Reproduction Assessment

Yes, from source and the linked report rather than a live run in this review. Current main still maps `reasoning_content` to thinking for OpenAI completions while the Xiaomi MiMo Pro/Omni catalog entries use that path and visible rendering reads text blocks.

## Solution Assessment

Mostly yes. A Xiaomi plugin-owned `wrapStreamFn` is the narrow maintainable boundary, but this branch must be updated for current dependency and test-helper contracts before it is a mergeable implementation.
