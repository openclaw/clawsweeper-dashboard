## Summary

Keep open: current main only partially covers the request, and this PR is still a useful implementation candidate, but it is not mergeable as-is because the proposed public plugin API has type/build integration gaps and the thinking lifecycle handling still has concrete correctness bugs.

## What This Changes

The PR adds plugin-facing thinking lifecycle hooks, a filtered agent-event subscription API, `agent_end` token/tool metadata, generated SDK baselines, Lobster/test-helper updates, and focused agent-event tests.

## Best Possible Solution

Rebase on current main, settle the plugin event API shape against the existing host-owned event subscription model, fix the thinking lifecycle edge cases with focused regression tests, wire all API builders/test helpers, and add the required changelog entry before merge.

## Reproduction Assessment

Yes for the PR findings: the missing builder wiring and the duplicate/fallback thinking paths are reproducible from the proposed source diff. I did not execute tests because this review is read-only.

## Solution Assessment

No, not as-is. The direction is plausible, but the safest solution should reuse or deliberately extend the existing plugin event contract, keep the generated/public SDK builders type-safe, and make thinking lifecycle events one-to-one before exposing the surface.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Thread the required event API through builders:** `src/plugins/types.ts:1378-1395`
  - body: Adding `onAgentEvent` as a required `OpenClawPluginApi` member leaves the central `buildPluginApi(...)` and public `createTestPluginApi(...)` helpers without that property. Those object literals are typed as `OpenClawPluginApi`, so this will break typechecking and any test/plugin helper code that relies on the builders until a real or noop implementation is added there too.
  - confidence: 0.88
- **[P2] Deduplicate repeated native thinking_end events:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:223-241`
  - body: This branch synthesizes `thinking_start` whenever a native `thinking_end` arrives while `reasoningStreamOpen` is false. After a normal start/end closes the stream, a duplicate terminal event follows the same branch and plugins receive a second start/end pair for one reasoning block; track the native block closure or ignore duplicate terminal events after a matched end.
  - confidence: 0.86
- **[P2] Respect code spans in message-end fallback reasoning:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:463-465`
  - body: The `message_end` safety net falls back to tagged-text extraction that does not honor code-span filtering. If native thinking is open and the final answer contains a literal `<think>...</think>` example, the hook can emit bogus reasoning text to plugins; reuse the code-span-aware parser or avoid parsing user-visible code as fallback reasoning.
  - confidence: 0.78
- **[P3] Add the required changelog entry:** `src/plugins/types.ts:1378-1395`
  - body: This PR adds public plugin SDK hooks and a new event subscription API, but the changed files do not include `CHANGELOG.md`. Repo policy requires user-facing `feat` changes to be recorded before merge.
  - confidence: 0.9
