## Summary

Keep open. Current main and the maintainer lifecycle PR cover adjacent pieces, but this PR still contains distinct plugin-facing API and metadata work; it is not mergeable as-is because the patch has source-reproducible lifecycle/API defects and lacks real behavior proof.

## What This Changes

Adds plugin-facing thinking hooks, a session-filtered `api.onAgentEvent` subscription API, `agent_end` token/tool metadata, event-stream filtering, and related tests/generated SDK baselines.

## Best Possible Solution

Rebase onto current main, align with the host-owned plugin event subscription contract or document a deliberate new API, then make thinking lifecycle delivery deterministic with focused tests and real plugin/channel proof.

## Reproduction Assessment

Yes for the review findings: the duplicate native `thinking_end` path, regex-only fallback extraction, and current-main API-builder mismatch are reproducible from PR-head/current-main source inspection. I did not run tests because this was a read-only review.

## Solution Assessment

No, not as-is. The direction is plausible, but the maintainable solution should align with the existing host-owned subscription contract and guarantee one-to-one lifecycle edges before exposing a public SDK surface.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Wire the new event API through current builders:** `src/plugins/types.ts:1392-1395`
  - body: Adding required `OpenClawPluginApi.onAgentEvent` means the current `buildPluginApi(...)` and public test-helper builders on main must provide it after rebase, or this should use the existing `api.agent.events.registerAgentEventSubscription(...)` contract instead. Leaving the required member only in older registry/test fakes will break SDK/helper consumers on current main.
  - confidence: 0.86
- **[P2] Ignore duplicate native thinking_end events:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:223-241`
  - body: After a normal native thinking start/end closes the reasoning stream, a repeated provider `thinking_end` enters this synthetic-start branch and emits another start/end pair for the same reasoning block. Track a closed native block or ignore duplicate terminal events so plugin UIs do not double-count or flicker.
  - confidence: 0.88
- **[P2] Use code-span-aware fallback extraction:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:463-465`
  - body: The `message_end` safety net extracts tagged thinking with a regex-only helper. If native thinking is open and the final answer contains a literal `<think>...</think>` code example, plugins can receive fake reasoning text; reuse the code-span-aware parser or avoid parsing user-visible code in this fallback.
  - confidence: 0.79
