## Summary

Keep open: current main only partially covers this area through the host-owned event stream/subscription surfaces, while the PR still contains unique plugin-hook and metadata work. The branch is not mergeable as-is because the proposed public SDK/event behavior has source-reproducible P2 lifecycle/API bugs and lacks contributor real-behavior proof.

## What This Changes

The PR adds plugin-facing thinking lifecycle hooks, `agent_end` token/tool metadata, a session-filtered top-level plugin agent-event subscription API, and related tests/generated SDK baselines.

## Best Possible Solution

Rebase onto current main, decide whether this belongs in the existing host-owned plugin event subscription contract or a new documented hook/API, then make thinking lifecycle events one-to-one with focused regression tests and real channel/plugin proof.

## Reproduction Assessment

Yes for the PR review findings: the duplicate native `thinking_end` path, regex-only fallback extraction, and current-main API-builder gap are reproducible from source inspection. I did not execute tests because this was a read-only review.

## Solution Assessment

No, not as-is. The direction is plausible, but the maintainable path should align with the existing plugin event subscription contract and make lifecycle hook delivery deterministic before exposing a public SDK surface.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Wire the new event API through current builders:** `src/plugins/types.ts:1392-1395`
  - body: Adding `onAgentEvent` as a required `OpenClawPluginApi` member means the current `buildPluginApi(...)` and public test-helper builders on main must provide it too. Rebase this branch and either reuse the existing host-owned subscription contract or add the real/noop builder implementations, otherwise plugin SDK typechecking and helper consumers will break.
  - confidence: 0.86
- **[P2] Ignore duplicate native thinking_end events:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:223-241`
  - body: After a normal native start/end closes the reasoning stream, a repeated provider `thinking_end` enters this synthetic-start branch and emits another start/end pair for the same reasoning block. Track the closed native block or ignore duplicate terminal events so plugin UIs do not double-count or flicker.
  - confidence: 0.88
- **[P2] Use code-span-aware fallback extraction:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:463-465`
  - body: The `message_end` safety net extracts tagged thinking with a regex-only helper. If native thinking is open and the final answer contains a literal `<think>...</think>` code example, plugins can receive fake reasoning text; reuse the code-span-aware parser or avoid parsing user-visible code in this fallback.
  - confidence: 0.79
