## Summary

Keep open: this external feature PR still contains meaningful plugin SDK and agent-event work, but it is not mergeable as-is. ## What This Changes

Adds plugin-facing thinking lifecycle hooks, a filtered agent-event subscription API, agent_end token/tool metadata, event-stream filtering, tests, and generated SDK baselines.

## Best Possible Solution

Rebase onto current main, use the host-owned plugin event subscription contract unless maintainers approve a new top-level API, and make thinking lifecycle delivery deterministic with focused tests and real plugin/channel proof.

## Reproduction Assessment

Yes for the review findings: PR-head and current-main source inspection reproduces the API-builder mismatch, duplicate native `thinking_end` path, and regex-only fallback extraction. I did not run tests because this was a read-only review.

## Solution Assessment

No, not as-is. The direction is plausible, but the maintainable solution should align with the existing host-owned plugin event contract and guarantee one-to-one thinking lifecycle edges before exposing the SDK surface.

- **[P2] Wire the event API through current builders:** `src/plugins/types.ts:1392-1395`
  - body: Adding required `OpenClawPluginApi.onAgentEvent` means current main's `buildPluginApi(...)` and helper builders must provide it after rebase, or this should use the existing `api.agent.events.registerAgentEventSubscription(...)` contract.   - confidence: 0.86
- **[P2] Ignore duplicate native thinking_end events:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:223-241`
  - body: After a normal native thinking start/end closes the reasoning stream, a repeated provider `thinking_end` enters this synthetic-start branch and emits another start/end pair for the same reasoning block. Track a closed native block or ignore duplicate terminal events so plugin UIs do not double-count or flicker.
  - confidence: 0.88
- **[P2] Use code-span-aware fallback extraction:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:463-465`
  - body: The `message_end` safety net extracts tagged thinking with a regex-only helper. If native thinking is open and the final answer contains a literal `<think>...</think>` code example, plugins can receive fake reasoning text; reuse the code-span-aware parser or avoid parsing user-visible code in this fallback.
  - confidence: 0.79
