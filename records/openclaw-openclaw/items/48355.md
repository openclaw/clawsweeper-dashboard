## Summary

Keep open. Current main only covers part of the requested streaming surface, and this PR is still the implementation candidate, but it is not merge-ready because the proposed thinking lifecycle hooks can duplicate terminal events and can mis-extract literal code examples as reasoning.

## What This Changes

The PR adds plugin-facing thinking lifecycle hooks, a filtered `api.onAgentEvent` stream subscription, thinking event delivery, and `agent_end` token/tool-count metadata across agent runner, plugin SDK, generated baselines, and tests.

## Best Possible Solution

Rebase on current main, align the public contract with the existing host-owned event subscription/runtime-events model, fix the thinking lifecycle edge cases with focused tests, and add the required changelog entry before merge.

## Reproduction Assessment

Yes for the PR findings: focused subscribe-harness tests can send a normal native thinking start/end followed by a duplicate `thinking_end`, and can close an open native reasoning lifecycle at `message_end` with a literal `<think>...</think>` code example. I did not run tests because this review is read-only.

## Solution Assessment

No, the PR is not the best mergeable solution as-is. The direction is plausible, but the safest implementation should preserve one plugin event contract, dedupe native terminal thinking events, and use code-span-aware reasoning extraction before exposing the SDK surface.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.84

Full review comments:

- **[P2] Deduplicate repeated native thinking_end events:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:223-241`
  - body: This branch synthesizes `thinking_start` whenever a native `thinking_end` arrives while `reasoningStreamOpen` is false. After a normal start/end closes the stream, a duplicate terminal event follows the same branch and plugins receive a second start/end pair for one reasoning block. Track that the current message's native thinking block has already closed, or ignore duplicate terminal events after a matched end.
  - confidence: 0.86
- **[P2] Respect code spans in message-end fallback reasoning:** `src/agents/pi-embedded-subscribe.handlers.messages.ts:463-465`
  - body: The `message_end` safety net falls back to `extractThinkingFromTaggedText(extractAssistantTextWithThinkingTags(msg))`, but that extractor does not honor code-span filtering. If native thinking is open and the final answer contains a literal code example like `<think>...</think>`, the hook can emit bogus reasoning text to plugins. Reuse the code-span-aware tag parser or avoid parsing user-visible code as fallback reasoning.
  - confidence: 0.78
- **[P3] Add the required changelog entry:** `src/plugins/types.ts:1378-1395`
  - body: This PR adds public plugin SDK hooks and a new event subscription API, but the changed files do not include `CHANGELOG.md`. Repo policy requires user-facing `feat` changes to be recorded before merge.
  - confidence: 0.9
