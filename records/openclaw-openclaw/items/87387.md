## Summary

Keep open: current main and v2026.6.9 still route combined local/session-list abortability into the visible Control UI composer busy and Stop controls, and https://github.com/openclaw/openclaw/pull/91606 is an open, unmerged fix candidate with closing linkage.

The issue asks for Control UI webchat to return the composer badge and button to idle/send after a completed response when no local stream is active.

## Best Possible Solution

Land a repaired Control UI fix that separates session-list-only stale abortability from local active-run UI while preserving Stop, Queue, and Steer for genuine active local runs.
