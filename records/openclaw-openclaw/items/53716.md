## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Ship the watchdog script with the packaged CLI:** `src/cli/gateway-cli/run.ts:571`
  - body: The new command resolves `scripts/openclaw-watchdog.sh` at runtime, but the npm package file list does not include that repository script path. Installed users would get a `gateway watchdog` command that points at a missing file unless the script is moved into a shipped runtime asset or package metadata is updated.
  - confidence: 0.94
- **[P1] Read the watchdog token from environment defaults:** `scripts/openclaw-watchdog.sh:22`
  - body: The CLI forwards `OPENCLAW_GATEWAY_TOKEN` to the child process, but the script initializes `TOKEN` to an empty string and only changes it when `--token` is passed. Env-authenticated gateways will be probed without the token and can be incorrectly treated as down.
  - confidence: 0.9
- **[P2] Clear startup errors after startup actually succeeds:** `src/cli/gateway-cli/run.ts:460-466`
  - body: The cleanup runs after `await startLoop()`, but the Gateway loop is long-running and normally does not return after successful startup. That leaves stale `startup-error.txt` content visible after recovery.
  - confidence: 0.88
- **[P2] Reject invalid watchdog numeric options before spawning:** `src/cli/gateway-cli/run.ts:575-583`
  - body: The option parsers accept invalid input as `NaN`, then forward it into shell arithmetic. Inputs like `--interval foo` or `--max-attempts foo` should fail in Commander before starting the watchdog script.
  - confidence: 0.9
- **[P2] Handle watchdog spawn failures explicitly:** `src/cli/gateway-cli/run.ts:608-621`
  - body: Wrapping `spawn("bash", ...)` in `try/catch` does not catch normal child-process spawn failures such as `ENOENT`; Node emits those through the child `error` event. Add an `error` listener so the CLI reports a controlled failure.
  - confidence: 0.86
- **[P3] Add the required changelog entry:** `src/cli/gateway-cli/run.ts:574-584`
  - body: This is a user-facing `feat` that adds a new `gateway watchdog` command, but the PR does not update `CHANGELOG.md`. OpenClaw policy requires user-visible feature changes to appear in the changelog before merge.
  - confidence: 0.82
