Overall confidence: 0.9

Full review comments:

- **[P1] Ship the watchdog script with the packaged CLI:** `src/cli/gateway-cli/run.ts:573`
  - body: The command resolves `scripts/openclaw-watchdog.sh` at runtime, but the package `files` list does not include that script. Installed package users would get a `gateway watchdog` command that points at a missing file unless the script is moved into a shipped runtime asset or packaging is updated.
  - confidence: 0.94
- **[P1] Read the watchdog token from the environment default:** `scripts/openclaw-watchdog.sh:22`
  - body: The CLI advertises and forwards `OPENCLAW_GATEWAY_TOKEN`, but the Bash script initializes `TOKEN` to an empty string and only updates it from `--token`. Env-authenticated gateways will be probed without the configured token and may be treated as down.
  - confidence: 0.9
- **[P2] Clear startup errors only after successful startup:** `src/cli/gateway-cli/run.ts:462-466`
  - body: This cleanup runs after `await startLoop()`, but the Gateway loop is designed to keep running until shutdown or restart. On normal successful startup this block never executes, so stale `startup-error.txt` content can remain visible after recovery.
  - confidence: 0.88
- **[P2] Reject invalid watchdog numeric options before spawning:** `src/cli/gateway-cli/run.ts:585-590`
  - body: Both watchdog numeric parsers can return `NaN`, and that value is forwarded into shell arithmetic. The command should reject invalid `--interval` and `--max-attempts` input in Commander before launching the script.
  - confidence: 0.9
- **[P2] Handle watchdog spawn failures explicitly:** `src/cli/gateway-cli/run.ts:615-623`
  - body: Wrapping `spawn("bash", ...)` in try/catch does not handle normal spawn failures emitted on the returned `ChildProcess`. Add an `error` listener so a missing Bash runtime reports a controlled CLI failure instead of an uncaught exception.
  - confidence: 0.86
- **[P2] Keep gateway tokens out of child-process argv:** `src/cli/gateway-cli/run.ts:603-604`
  - body: When `--token` is supplied, the token is appended to the long-running Bash child process arguments, which can be visible in local process listings. Pass the token through the child environment or another non-argv secret path instead.
  - confidence: 0.82
