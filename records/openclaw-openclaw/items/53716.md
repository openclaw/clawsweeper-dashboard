## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Ship the watchdog script with the packaged CLI:** `src/cli/gateway-cli/run.ts:572-573`
  - body: The new command resolves `scripts/openclaw-watchdog.sh` at runtime, but `package.json` does not include that script in the published package. Installed users would get a `gateway watchdog` command whose target file is missing unless the script is moved into a shipped asset or package metadata is updated.
  - confidence: 0.94
- **[P1] Read the watchdog token from environment defaults:** `scripts/openclaw-watchdog.sh:21`
  - body: The CLI says `OPENCLAW_GATEWAY_TOKEN` is the default and forwards it to the child environment, but the script initializes `TOKEN` to an empty string and only changes it for `--token`. Env-authenticated gateways will be probed without the token and can be incorrectly treated as down.
  - confidence: 0.9
- **[P2] Clear startup errors after startup succeeds:** `src/cli/gateway-cli/run.ts:466-467`
  - body: This cleanup runs after `await startLoop()`, but the Gateway loop is long-running and normally does not return after successful startup. That leaves stale `startup-error.txt` content visible after the Gateway has recovered.
  - confidence: 0.88
- **[P2] Reject invalid watchdog numeric options before spawning:** `src/cli/gateway-cli/run.ts:584-589`
  - body: Both numeric parsers can return `NaN`, and that value is forwarded into shell arithmetic. Inputs like `--interval foo` or `--max-attempts foo` should fail in Commander before starting the watchdog script.
  - confidence: 0.9
- **[P2] Handle watchdog spawn failures explicitly:** `src/cli/gateway-cli/run.ts:612`
  - body: Wrapping `spawn("bash", ...)` in `try/catch` does not handle normal child-process spawn failures such as `ENOENT`; Node emits those through the child `error` event. Add an `error` listener so the CLI reports a controlled failure.
  - confidence: 0.86
- **[P3] Add the required changelog entry:** `src/cli/gateway-cli/run.ts:575-576`
  - body: This is a user-facing feature that adds a new `gateway watchdog` command, but the PR diff does not update `CHANGELOG.md`. OpenClaw policy requires user-visible feature changes to appear in the changelog before merge.
  - confidence: 0.82
