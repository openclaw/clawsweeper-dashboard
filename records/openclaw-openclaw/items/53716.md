Full review comments:

- **[P1] Ship the watchdog script with the packaged CLI:** `src/cli/gateway-cli/run.ts:573`
  - body: The new command resolves `scripts/openclaw-watchdog.sh` at runtime, but the package file list does not include that script. Installed package users would get a `gateway watchdog` command that points at a missing file unless the script is moved into a shipped runtime asset or packaging is updated.
  - confidence: 0.94
- **[P1] Read the watchdog token from the environment default:** `scripts/openclaw-watchdog.sh:22`
  - body: The CLI advertises and forwards `OPENCLAW_GATEWAY_TOKEN`, but the script initializes `TOKEN` to an empty string and only updates it from `--token`. Env-authenticated gateways will be probed without the configured token and may be treated as down.
  - confidence: 0.9
- **[P2] Clear startup errors after startup actually succeeds:** `src/cli/gateway-cli/run.ts:463-466`
  - body: This cleanup runs only after `await startLoop()` returns, but the Gateway loop is designed to stay running until shutdown or exit. On normal successful startup this block never executes, leaving stale `startup-error.txt` content visible after recovery.
  - confidence: 0.88
- **[P2] Reject invalid watchdog numeric options before spawning:** `src/cli/gateway-cli/run.ts:585-590`
  - body: Both numeric parsers can return `NaN`, and that value is forwarded into shell arithmetic. The command should reject invalid `--interval` or `--max-attempts` input before launching the script.
  - confidence: 0.9
- **[P2] Handle child-process spawn errors with an error listener:** `src/cli/gateway-cli/run.ts:615`
  - body: Wrapping `spawn("bash", ...)` in try/catch does not handle normal spawn failures emitted on the returned `ChildProcess`. Add an `error` listener so a missing Bash runtime reports a controlled CLI failure instead of an uncaught exception.
  - confidence: 0.86
- **[P2] Keep gateway tokens out of child-process argv:** `src/cli/gateway-cli/run.ts:603-604`
  - body: When `--token` is supplied, the token is appended to the long-running Bash child process arguments, which can be visible in local process listings. Pass the token through the child environment and make the script read that env default instead.
  - confidence: 0.82
