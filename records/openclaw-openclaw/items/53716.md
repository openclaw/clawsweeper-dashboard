Overall confidence: 0.91

Full review comments:

- **[P1] Ship the watchdog script with the packaged CLI:** `src/cli/gateway-cli/run.ts:572`
  - body: The new command resolves `scripts/openclaw-watchdog.sh` at runtime, but the package `files` list does not include that script. Installed package users would get a `gateway watchdog` command that points at a missing file unless the script is moved into a shipped runtime asset or package metadata is updated.
  - confidence: 0.94
- **[P1] Read the watchdog token from environment defaults:** `scripts/openclaw-watchdog.sh:22`
  - body: The CLI advertises and forwards `OPENCLAW_GATEWAY_TOKEN`, but the script initializes `TOKEN` to an empty string unless `--token` is passed. Env-authenticated gateways will be probed without the token and can be treated as down, triggering incorrect restart or manual-intervention flows.
  - confidence: 0.9
- **[P2] Clear startup errors after startup actually succeeds:** `src/cli/gateway-cli/run.ts:460-463`
  - body: The cleanup runs after `await startLoop()`, but the gateway loop is designed to keep running until shutdown or restart handling. On normal successful startup this block is not reached, leaving stale `startup-error.txt` content visible after recovery.
  - confidence: 0.88
- **[P2] Reject invalid watchdog numeric options before spawning:** `src/cli/gateway-cli/run.ts:580-586`
  - body: The option parsers can return `NaN` for inputs like `--interval foo` or `--max-attempts foo`, and those values are forwarded into shell arithmetic. These should fail with Commander validation errors before spawning the watchdog script.
  - confidence: 0.9
- **[P2] Handle watchdog spawn errors explicitly:** `src/cli/gateway-cli/run.ts:608-620`
  - body: Wrapping `spawn("bash", ...)` in `try/catch` does not catch normal child-process spawn failures such as missing Bash; Node emits those through the child `error` event. Add an `error` listener so the CLI fails predictably instead of taking the uncaught-exception path.
  - confidence: 0.86
- **[P3] Add the required changelog entry for the new command:** `src/cli/gateway-cli/run.ts:575-576`
  - body: This is a user-facing `feat` that adds a new `gateway watchdog` command, but the PR does not update `CHANGELOG.md`. OpenClaw policy requires user-visible feature changes to appear in the changelog before merge.
  - confidence: 0.82
