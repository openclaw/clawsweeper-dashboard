## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Ship the watchdog script with the packaged CLI:** `src/cli/gateway-cli/run.ts:573`
  - body: The new command resolves `scripts/openclaw-watchdog.sh` at runtime, but the package file list does not include that script. Installed package users would get a `gateway watchdog` command that points at a missing file unless the script is moved into a shipped runtime asset or packaging is updated.
  - confidence: 0.94
- **[P1] Read the watchdog token from the environment default:** `scripts/openclaw-watchdog.sh:22`
  - body: The CLI advertises and forwards `OPENCLAW_GATEWAY_TOKEN`, but the script initializes `TOKEN` to an empty string and only updates it from `--token`. Env-authenticated gateways will be probed without the configured token and may be treated as down.
  - confidence: 0.9
- **[P2] Clear startup errors only after startup succeeds:** `src/cli/gateway-cli/run.ts:463-466`
  - body: This cleanup runs after `await startLoop()`, but the Gateway loop is long-running and normally does not resolve after a successful startup. Stale `startup-error.txt` content can therefore remain visible after recovery.
  - confidence: 0.88
- **[P2] Reject invalid watchdog numeric options before spawning:** `src/cli/gateway-cli/run.ts:583-590`
  - body: The parsers accept `NaN` for values like `--interval foo` or `--max-attempts foo`, then forward that value into shell arithmetic. Commander should reject invalid numbers before launching the script.
  - confidence: 0.9
- **[P2] Handle child-process spawn errors explicitly:** `src/cli/gateway-cli/run.ts:615-623`
  - body: Wrapping `spawn("bash", ...)` in try/catch does not catch normal spawn failures emitted on the returned `ChildProcess`. Add an `error` listener so a missing Bash runtime reports a controlled CLI failure instead of an uncaught exception.
  - confidence: 0.86
- **[P2] Keep gateway tokens out of child-process argv:** `src/cli/gateway-cli/run.ts:603-604`
  - body: When `--token` is supplied, the token is appended to the long-running Bash child process arguments, which can be visible in local process listings. Pass the token through the child environment and make the script read that env default instead.
  - confidence: 0.82

## Security Review

Status: needs_attention

Summary: Needs attention: the diff introduces a concrete local secret-exposure risk by forwarding a gateway token through long-running child-process argv.

Concerns:

- **[medium] Gateway token can be exposed in watchdog argv:** `src/cli/gateway-cli/run.ts:604`
  - body: `opts.token` is appended as `--token <value>` when spawning the Bash watchdog. Because the watchdog is intended to run continuously, the token can remain visible through local process listings; use `OPENCLAW_GATEWAY_TOKEN` or another non-argv secret path instead.
  - confidence: 0.82
