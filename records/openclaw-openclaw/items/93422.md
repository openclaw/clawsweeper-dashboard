Keep open: current main has the shared `/name <title>` label path and backend label storage, but it still lacks the requested Control UI `/label <name>` and named `/new <name>` workflow. ## What This Changes

The issue requests user-friendly WebChat/Control UI session naming through `/label <name>` and/or `/new <name>`, with the label visible in session selectors and session JSON/listing surfaces.

## Best Possible Solution

Review and either land or reject https://github.com/openclaw/openclaw/pull/93587 for the Control UI command slice, while keeping the broader CLI/cross-surface label contract consolidated under https://github.com/openclaw/openclaw/issues/55249.

## Reproduction Assessment

Yes, by source inspection rather than a failing bug path. Current main dispatches typed `/new` without carrying args, has no Control UI `/label` local command, and documents `/name` as the existing shared rename path.

## Solution Assessment

Unclear until the product direction is accepted.
