## Summary

Keep open: current main has the backend label/display primitives and the merged `/name <title>` chat command, but it still lacks the requested `/new <name>` flow and any `/label` alias or explicit naming contract for this WebChat/Control UI path. The active follow-up at https://github.com/openclaw/openclaw/pull/95700 and the title/label contract work at https://github.com/openclaw/openclaw/pull/95182 make this a maintainer product-review item, not a cleanup close.

## What This Changes

Add chat/Control UI session naming through `/label <name>` and/or named `/new` sessions so friendly session names persist and display in session selectors and CLI output.

## Best Possible Solution

Review https://github.com/openclaw/openclaw/pull/95700 alongside the title/label contract in https://github.com/openclaw/openclaw/pull/95182, then land a single canonical chat naming path and add `/label` only if maintainers want it as an alias for `/name`.

## Reproduction Assessment

Yes, source-level for the missing feature path: Control UI `/new` ignores trailing args, the create-session helper does not send a label, and no `/label` local command is registered. This is a feature gap, not a broken existing bug contract.

## Solution Assessment

Unclear until maintainers choose the canonical naming contract. The maintainable path is to reuse the existing `/name` and label/title primitives through the active PR rather than adding a separate `displayName`-only command path.
