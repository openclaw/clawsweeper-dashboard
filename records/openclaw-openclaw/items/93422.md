## What This Changes

Request to add WebChat/Control UI session naming through `/label <name>` and/or `/new <name>`, with names persisted and surfaced in session selection and session JSON output.

## Best Possible Solution

Settle one canonical session naming contract, then implement the Control UI/WebChat command path and session JSON/list projection against that contract.

## Reproduction Assessment

Yes. Source inspection on current main shows `/new` ignores trailing text, no local `/label` command is registered, and the create-session request does not send a naming field.

## Solution Assessment

Unclear. The existing label storage and `/name <title>` command are the right foundation, but maintainers still need to choose whether `/label` and `/new <name>` should exist or whether the UX should stay on `/name` plus another UI affordance.
