## Summary

Keep open: current main still has per-run model resolution and per-request provider dispatcher/client construction, while the requested pooling/cache/warmup behavior needs maintainer product and network-security design before implementation.

## What This Changes

Request HTTP connection pooling, model-resolution/auth caching, and provider warmup to reduce slow OpenAI-compatible provider startup on each agent run.

## Best Possible Solution

Add focused timing instrumentation and a controlled provider benchmark first, then implement bounded provider transport reuse or caching keyed by request policy without weakening auth or network protections.
