Keep open: current main still has the repeated per-run model-resolution and per-request provider transport shape, and multiple commenters added matching provider/channel signals; the requested pooling/cache/warmup change needs provider-runtime and network-security design before implementation.

## What This Changes

Request provider HTTP connection pooling, resolved-model/auth caching, and gateway warmup to reduce repeated per-run model-resolution latency for OpenAI-compatible providers.

## Best Possible Solution

Design a bounded provider-transport reuse layer keyed by provider request policy, preserve SSRF/proxy/TLS cleanup guarantees, benchmark it, then separately decide whether model metadata TTL caching or gateway warmup should become user-facing behavior.

## Solution Assessment
