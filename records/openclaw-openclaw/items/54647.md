## What This Changes

The PR changes gateway session model resolution and tests so ACP session rows read `entry.acp.runtimeOptions.model` and omit default-model fallback when no ACP model is recorded.

## Best Possible Solution

Rework the current gateway session-row resolver so ACP rows use ACP runtime metadata when present, omit model fields when ACP runtime model is unknown, and preserve existing generic-session provider, transcript, and display normalization behavior with focused regressions and real proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: an ACP session key with only `entry.acp.runtimeOptions.model`, or with no ACP runtime model, still flows through generic session model resolution and default fallback on current main.

## Solution Assessment

No. The PR targets the right bug, but the current branch reparses prefixed models even when a stored provider is explicit and needs rework against current main's session-list resolver semantics.
