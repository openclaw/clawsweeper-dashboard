## Summary

Keep open: the underlying gateway bug is still visible on current main, but this PR has a blocking provider-preservation regression, is now conflicting with current session-list changes, and still lacks after-fix real behavior proof.

## What This Changes

The PR changes gateway session model resolution and tests so ACP session rows read `entry.acp.runtimeOptions.model` and avoid default-model fallback when the ACP runtime model is unknown.

## Best Possible Solution

Rework the current gateway row resolver so ACP rows use explicit ACP runtime metadata when present, omit misleading model fields when unknown, and preserve explicit providers plus current selected-model, transcript, cache, and ACP runtime metadata behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: an ACP session row with only `entry.acp.runtimeOptions.model`, or with no ACP runtime model, still flows through the generic gateway resolver and default-model fallback on current main.

## Solution Assessment

No. The PR targets the right bug, but it re-parses prefixed model strings even when an explicit provider is stored and needs to be rebased over current session-list behavior before merge.

## Review Findings

Overall confidence: 0.9

- **[P1] Preserve explicit providers for prefixed models:** `src/gateway/session-utils.ts:1058-1060`
