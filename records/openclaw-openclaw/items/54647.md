## Summary

Keep this PR open. Current main still has the ACP session model-reporting bug, and the PR targets the right gateway surface, but the proposed helper introduces a provider-preservation regression and still needs release-note coverage before it is safe to merge.

## What This Changes

The PR changes gateway session model resolution and tests so ACP session rows read `entry.acp.runtimeOptions.model` and avoid default-model fallback when no explicit ACP runtime model is known.

## Best Possible Solution

Gateway session-row serialization should use explicit ACP runtime metadata for ACP keys, omit `modelProvider` and `model` when the ACP runtime model is unknown, and preserve existing provider, transcript, live subagent, selected-model, and display-normalization behavior with focused regressions.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: an ACP session key with `entry.acp.runtimeOptions.model` is invisible to current main's generic resolver, and an ACP row without generic model metadata falls through to `DEFAULT_MODEL`.

## Solution Assessment

No. The PR is aimed at the narrowest useful surface, but it is not the best implementation yet because it re-parses prefixed model IDs even when an explicit provider is already stored and it lacks the required changelog entry.
