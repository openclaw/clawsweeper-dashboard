## Summary

Keep this PR open. Current main still has the ACP sessions-list model reporting bug, and the PR targets the right narrow gateway surface, but the proposed helper introduces a blocking provider-preservation regression and also needs the required changelog entry before merge.

## What This Changes

The PR updates gateway session model resolution and tests so ACP session rows read `entry.acp.runtimeOptions.model` and avoid default-model fallback when no explicit ACP runtime model is known.

## Best Possible Solution

Update the gateway session-row resolver so ACP rows use explicit ACP runtime metadata when present, leave `modelProvider` and `model` absent when the ACP runtime model is unknown, and preserve existing generic-session provider handling, transcript fallback, live subagent model handling, selected-model display, and display normalization with focused regressions.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is an ACP session key with `entry.acp.runtimeOptions.model`; current main never reads that field, and an ACP row with no generic or ACP model metadata falls through to `DEFAULT_MODEL`. The PR regression is likewise reproducible by passing `modelProvider: "openrouter"` with a prefixed `model: "anthropic/claude-haiku-4.5"` through the new helper.

## Solution Assessment

No. The PR targets the correct narrow surface, but it is not yet the best implementation because it re-parses prefixed model ids even when an explicit provider is already stored. The safer fix keeps explicit providers authoritative while adding the ACP-only no-default path and changelog coverage.
