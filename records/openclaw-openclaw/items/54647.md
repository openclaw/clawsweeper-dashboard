## Summary

Keep this PR open. Current main still has the reported ACP session model-reporting bug, and the PR targets the right gateway surface, but the branch should not merge as-is because it regresses explicit provider preservation for slash-prefixed model ids and is currently dirty against main.

## What This Changes

The PR modifies `src/gateway/session-utils.ts` and `src/gateway/session-utils.test.ts` so ACP session rows read `entry.acp.runtimeOptions.model` and omit the generic default-model fallback when no explicit ACP runtime model is recorded.

## Best Possible Solution

Update the current gateway session-row resolver so ACP rows use explicit ACP runtime metadata when present, leave `modelProvider` and `model` absent when the ACP runtime model is unknown, and preserve existing generic-session provider handling, selected-model display, transcript fallback, live subagent model handling, and display normalization with focused regressions.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction is to call `listSessionsFromStore` on current main with an ACP session key and `entry.acp.runtimeOptions.model`; the current row path never reads that ACP metadata. A second check is an ACP row with no generic or ACP runtime model metadata, which currently falls through to `DEFAULT_MODEL`.

## Solution Assessment

No. The PR targets the right narrow surface, but the implementation is not the best fix because it re-parses prefixed model ids even when an explicit provider is stored. The safer repair keeps explicit providers authoritative while adding the ACP-only no-default path on top of current main.
