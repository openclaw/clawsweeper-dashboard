## What This Changes

## Best Possible Solution

Rework the current-main gateway session-row resolver so ACP rows use `entry.acp.runtimeOptions.model` when known, suppress default-model fallback when the ACP runtime model is unknown, and preserve generic explicit-provider behavior.

## Reproduction Assessment

No. The direction is right, but this branch is not the best merge shape because it reparses explicit providers and predates current main's selected/transcript display model logic; a narrow current-main rework is safer.
