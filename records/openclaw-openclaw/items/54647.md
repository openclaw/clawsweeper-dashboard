## Summary

Keep open: the ACP `sessions.list` bug is still present on current `main`, and this PR targets the right gateway surface, but the branch still has a blocking provider-preservation regression and lacks required real behavior proof.

## What This Changes

The PR changes gateway session model resolution and tests so ACP session rows read ACP runtime model metadata and avoid default-model fallback when no ACP model is known.

## Best Possible Solution

Rework the gateway row resolver on current main so ACP rows use `entry.acp.runtimeOptions.model` when present, omit `modelProvider`/`model` when the ACP runtime model is unknown, and preserve explicit providers plus current session-list model identity behavior with focused regressions and real proof.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: an ACP session row with only `entry.acp.runtimeOptions.model`, or with no ACP runtime model, still flows through the generic resolver and default-model fallback on current main.

## Solution Assessment

No. The PR targets the right surface, but it should treat an explicit stored provider as authoritative before parsing slash-prefixed model strings and should be rebased over current session-list model identity behavior.
