## Summary

Keep open: current main still has the ACP `sessions.list` model-reporting bug, but this PR is not mergeable as-is because it can rewrite explicit wrapper providers for prefixed model ids and it lacks after-fix real behavior proof.

## What This Changes

The PR changes gateway session model resolution and tests so ACP session rows read `entry.acp.runtimeOptions.model` and avoid default-model fallback when no explicit ACP runtime model is known.

## Best Possible Solution

Rework the gateway session-row resolver so ACP rows use explicit ACP runtime metadata, omit model fields when the ACP runtime model is unknown, and preserve existing generic-session provider and transcript fallback behavior with focused regressions.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: current main never reads `entry.acp.runtimeOptions.model` when building session rows, then falls through to the generic default-model resolver.

## Solution Assessment

No. The PR targets the right gateway surface, but it should treat an explicit stored `modelProvider` as authoritative before parsing slash-prefixed model strings and should be rebased around current session-list model identity behavior.
