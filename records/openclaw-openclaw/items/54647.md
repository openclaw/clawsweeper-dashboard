## Summary

Keep open. Current main still has the ACP `sessions.list` model-reporting bug, but this PR is not merge-ready because its helper can overwrite explicit wrapper providers for prefixed model ids and the branch needs the required changelog/rebase work.

## What This Changes

The PR changes gateway session-row model resolution and tests so ACP session rows read `entry.acp.runtimeOptions.model` and omit default-model fallback when ACP has no explicit runtime model.

## Best Possible Solution

Gateway session-row serialization should use ACP runtime metadata for ACP rows, return no model identity when ACP runtime model is unknown, preserve explicit provider semantics, and ship focused regressions plus a single-line changelog fix entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: current main resolves all session rows through the generic model resolver and default fallback, while ACP runtime model identity is stored under `entry.acp.runtimeOptions.model`.

## Solution Assessment

No. The PR targets the right gateway surface, but it should treat stored `modelProvider` as authoritative before parsing slash-prefixed model ids, then rebase or replace the stale branch and add the required changelog entry.
