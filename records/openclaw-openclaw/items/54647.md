## Summary

Keep open: current main still has the ACP session model-reporting bug, and this PR is a useful focused fix, but the branch is conflicting and still has a provider-preservation regression that should block merge. The PR also lacks after-fix real behavior proof from a real gateway/client path.

## What This Changes

The PR changes gateway session model resolution and tests so ACP session rows read ACP runtime metadata and avoid default-model fallback when no ACP runtime model is known.

## Best Possible Solution

Rework the gateway session-row resolver on current main so ACP rows read `entry.acp.runtimeOptions.model`, omit model fields when the ACP runtime model is unknown, and preserve explicit providers plus transcript/live model identity behavior with focused regressions.

## Reproduction Assessment

Yes. A current-main ACP session store row with an ACP session key and only `entry.acp.runtimeOptions.model` follows a generic resolver that never reads `entry.acp`, while unknown ACP model rows fall through to `DEFAULT_MODEL`; this is source-reproducible without running tests.

## Solution Assessment

No. The PR targets the right gateway surface, but it should be rebased or reworked on current main so explicit providers remain authoritative and the newer transcript/live model identity selection is preserved.
