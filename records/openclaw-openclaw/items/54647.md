## Summary

Keep open: current main still reports ACP session rows through the generic model resolver, but this PR needs changes because it can rewrite an explicit wrapper provider for slash-prefixed models and it lacks the required changelog entry.

## What This Changes

The PR updates gateway session model resolution and tests so ACP session rows read `entry.acp.runtimeOptions.model` and avoid default-model fallback when no ACP runtime model is known.

## Best Possible Solution

Gateway session-row serialization should use explicit ACP runtime metadata for ACP rows, omit `modelProvider` and `model` when the ACP runtime model is unknown, preserve existing explicit provider semantics, and carry focused regressions plus a single-line changelog fix entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: an ACP session key with `entry.acp.runtimeOptions.model` is invisible to current main's generic resolver, and an ACP row without generic model metadata falls through to `DEFAULT_MODEL`.

## Solution Assessment

No. The PR targets the right gateway surface, but the implementation should treat stored `modelProvider` as authoritative before parsing slash-prefixed models and should add the required changelog entry.
