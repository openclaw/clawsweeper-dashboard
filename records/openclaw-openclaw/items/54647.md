## Summary

Keep open: current main still has the ACP `sessions.list` model-reporting bug, and this PR is the canonical fix path, but the branch has a blocking provider-preservation regression and lacks required real behavior proof.

## What This Changes

The PR changes gateway session model resolution and tests so ACP session rows read `entry.acp.runtimeOptions.model` and avoid configured-default fallback when no ACP runtime model is known.

## Best Possible Solution

Update the current gateway session-row resolver so ACP rows use explicit ACP runtime metadata when present, leave `modelProvider` and `model` absent when the ACP runtime model is unknown, and preserve explicit provider, selected-model, transcript fallback, and display-normalization behavior with focused regressions.

## Reproduction Assessment

Yes. Source inspection is high-confidence: current main's row builder calls the generic model resolver and falls back to `DEFAULT_MODEL`, while ACP runtime model identity is stored separately under `entry.acp.runtimeOptions.model`.

## Solution Assessment

No. The PR targets the right gateway surface, but it should treat an explicit stored `modelProvider` as authoritative before parsing slash-prefixed model strings and should be rebased over current session-list model identity behavior.
