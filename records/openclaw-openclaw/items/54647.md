## Summary

Keep open: the ACP session-list bug still exists on current main and this PR is a useful direction, but the branch is not merge-ready because it regresses explicit provider preservation, conflicts with current main, and lacks real behavior proof.

## What This Changes

The PR changes gateway session model resolution and tests so ACP session rows use `entry.acp.runtimeOptions.model` and stop defaulting unknown ACP models to the configured default.

## Best Possible Solution

Rework this on current main so ACP rows read `entry.acp.runtimeOptions.model` and omit default fallback only when ACP runtime model is unknown, while preserving explicit providers and current selected/transcript display behavior.

## Reproduction Assessment

Yes at source level. Current main routes `sessions.list` rows through the generic model identity resolver, while ACP runtime model data is stored under `entry.acp.runtimeOptions.model`; no tests were run in this read-only review.

## Solution Assessment

No. The direction is correct, but this branch reparses explicit providers and predates current main's session-list model display behavior, so the best fix is a narrow current-main rework.
