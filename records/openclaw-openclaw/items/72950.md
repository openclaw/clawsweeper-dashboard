## Summary

Keep open: current main still lacks a generic plugin config overlay or plugin-declared env override contract. Existing config env substitution and OTel-specific env fallbacks help some deployments, but they do not satisfy the requested fast reconfiguration path for locked sandboxes.

## What This Changes

Request a generic way for plugin config under `plugins.entries.<id>.config` to be overridden by declared environment variables and/or a writable overlay in policy-locked sandbox deployments.

## Best Possible Solution

Design a generic, schema-validated plugin config resolver that merges baked config with an approved writable overlay and explicit plugin-declared env mappings, then document precedence and locked-sandbox behavior.

## Reproduction Assessment

Yes for the OpenClaw side: source inspection on current main shows the resolved, validated, and exposed plugin config comes from `plugins.entries.<id>.config` without overlay or env mapping. The external NemoClaw filesystem denial is supported by the report but was not live-tested here.

## Solution Assessment

Unclear for the exact implementation. A generic resolver with documented precedence is likely safer than plugin-specific env hacks, and the reporter clarified that env-only support would not cover the full runtime reconfiguration loop.
