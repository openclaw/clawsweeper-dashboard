## Summary

Keep open: current main still lacks a generic plugin-config env override or writable overlay contract; the core path validates and exposes only baked `plugins.entries.<id>.config`, and the follow-up discussion clarifies that env-only support would not fully solve policy-locked runtime reconfiguration.

## What This Changes

The issue requests a runtime-adjustable plugin config path for policy-locked deployments, using explicit env mappings and/or a writable overlay merged with `plugins.entries.<id>.config`.

## Best Possible Solution

Add a generic, schema-validated plugin config resolution layer that merges baked config with an approved writable overlay and explicit plugin-declared env mappings, then document precedence and locked-sandbox behavior.

## Reproduction Assessment

Yes for the OpenClaw side: current-main source inspection shows the only resolved, validated, and exposed plugin config is `plugins.entries.<id>.config`. The external NemoClaw `EACCES` path is documented in the issue, but this review did not live-run that sandbox.

## Solution Assessment

Unclear for the exact implementation. The narrowest maintainable direction appears to be a generic overlay plus explicit env mapping contract validated after merge; env-only support would not fully cover the reporter’s runtime reconfiguration loop.
