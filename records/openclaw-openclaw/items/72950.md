## Summary

Keep open: current main still lacks a generic plugin-config env override or writable-overlay contract, and the comments add evidence that this affects more than one locked-down plugin deployment.

## What This Changes

Request a generic env-variable and/or writable-overlay path for plugin config so policy-locked sandbox deployments can adjust `plugins.entries.<id>.config` without rebuilding images.

## Best Possible Solution

Add a generic, schema-validated plugin config resolution layer that merges baked config with an approved writable overlay and explicit plugin-declared env mappings, then document precedence and locked-sandbox behavior.

## Reproduction Assessment

Yes for the OpenClaw side: current-main source inspection shows only `plugins.entries.<id>.config` is resolved, validated, and exposed to plugins. The external NemoClaw Landlock `EACCES` path is documented in the issue, but this review did not live-run that sandbox.

## Solution Assessment

Unclear for the exact implementation. Generic `${VAR}` substitution is already available as a partial workaround, but it does not replace baked values or provide a runtime writable overlay, so the maintainable fix needs explicit core contract design.
