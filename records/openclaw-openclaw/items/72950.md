## Summary

Keep open: current main still lacks a generic plugin-config env override or writable overlay contract; existing global env substitution and plugin-specific OTel fallbacks do not solve writable runtime reconfiguration for arbitrary plugin config in policy-locked sandboxes.

## What This Changes

Request to let `plugins.entries.<id>.config` be overridden through declared environment variables and/or a writable overlay for policy-locked sandbox deployments.

## Best Possible Solution

Add a generic, schema-validated plugin config resolution layer that merges baked config with an approved writable overlay and explicit plugin-declared env mappings, then document precedence and locked-sandbox behavior.

## Reproduction Assessment

Yes for the OpenClaw side: current-main source inspection shows only `plugins.entries.<id>.config` is validated and exposed to plugins. The NemoClaw filesystem denial is evidenced by the reporter’s command output but was not live-run here.

## Solution Assessment

Unclear for the exact implementation. The maintainable direction appears to be a generic overlay plus explicit env mapping contract validated after merge; env-only support would not cover the reporter’s fast runtime reconfiguration loop.
