## Summary

Keep open: current main still lacks a generic plugin-config env override or writable overlay contract. Existing `${VAR}` substitution and `$include` support can help only when the baked config already planned for them, but they do not provide the requested plugin-declared override or runtime overlay path for policy-locked sandboxes.

## What This Changes

Request a generic plugin config path that can merge baked `plugins.entries.<id>.config` with declared env-variable overrides and/or a writable runtime overlay for policy-locked sandboxes.

## Best Possible Solution

Design a generic, schema-validated plugin config resolver that merges baked config with an approved writable overlay and explicit plugin-declared env mappings, then document precedence and locked-sandbox behavior.

## Reproduction Assessment

Yes for the OpenClaw side: current-main source inspection shows the resolved, validated, and exposed plugin config comes from `plugins.entries.<id>.config`. The external NemoClaw filesystem denial is supported by the report but was not live-tested here.

## Solution Assessment

Unclear for the exact implementation. The safest direction is a generic resolver and documented precedence model rather than plugin-specific env hacks; env-only support would not cover the reporter's full runtime reconfiguration loop.
