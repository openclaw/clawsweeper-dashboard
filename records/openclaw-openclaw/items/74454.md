## Summary

Keep open. Current main already has the workspace `.env` blocklist half from related merged work, but the registry helper still builds `reg.exe` candidates from env-derived `SystemRoot` and `WINDIR`; the PR remains useful and I found no blocking patch defect, but it is security-sensitive Windows command-execution hardening that should get maintainer/security approval before merge.

## What This Changes

The PR adds a changelog entry, extends workspace dotenv and Windows install-root unit tests, and changes the Windows registry helper to resolve only the fixed `C:\Windows\System32\reg.exe` path instead of env-derived roots.

## Best Possible Solution

Land a maintainer/security-approved version that keeps the shipped workspace `.env` blocklist, prevents registry probing from selecting untrusted env-derived `reg.exe` paths, preserves the intended Windows install-root fallback behavior, and coordinates with the adjacent Windows command-hardening PRs.
