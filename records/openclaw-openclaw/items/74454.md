## Summary

Keep open: current main still lets the registry helper choose env-derived `reg.exe` candidates before executing registry queries, and this PR applies the remaining focused hardening after related dotenv/ACL work already landed.

## What This Changes

The PR pins Windows registry probing to a fixed `C:\Windows\System32\reg.exe`, removes env-derived registry-client candidates, updates focused infra tests, and adds a changelog entry.

## Best Possible Solution

Land a rebased, maintainer/security-approved patch that prevents registry probing from selecting environment-derived `reg.exe` paths while preserving the current host-env policy and Windows install-root fallback behavior.

## Solution Assessment
