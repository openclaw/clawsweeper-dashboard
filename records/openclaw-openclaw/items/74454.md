## Summary

Keep open. Current main has the workspace dotenv blocklist portion from related Windows hardening, but the registry helper still derives `reg.exe` candidates from `SystemRoot`/`WINDIR`; the latest PR head appears to address that remaining command-resolution surface and the earlier review feedback, so this should proceed through maintainer/security review rather than cleanup closure.

## What This Changes

The PR adds a changelog entry, extends dotenv and Windows install-root tests, and changes Windows registry probing so `locateWindowsRegExe`/`getWindowsRegExeCandidates` ignore env-derived roots and only select the fixed `C:\Windows\System32\reg.exe` path.

## Best Possible Solution

Land a maintainer/security-approved version that keeps workspace dotenv system-root keys blocked, prevents registry probing from selecting env-derived `reg.exe` paths, preserves the existing install-root fallback contract, and coordinates with the related Windows command-hardening PRs.

Yes.
