## Summary

Keep open: the PR addresses a real skill symlink warning class, but the proposed patch is not mergeable because it can relabel bundled skills as non-bundled and bypass `skills.allowBundled`.

## What This Changes

The branch broadens skill realpath containment to accept symlink targets inside other configured skill roots and adds regression tests for those cross-root symlinks.

## Best Possible Solution

Accept safe cross-root symlinks only when the target policy preserves the original source semantics, with bundled targets either rejected or classified as bundled so `skills.allowBundled` remains enforceable.

## Reproduction Assessment

Yes for the blocking PR defect: source inspection shows a non-bundled scan root can accept a symlink into `bundledSkillsDir`, while the loaded entry keeps the non-bundled caller source. I did not execute the test path because this review is read-only.

## Solution Assessment

No. The cross-root symlink direction is plausible, but the implementation trusts bundled targets without preserving bundled-source filtering; a narrower repair should keep `skills.allowBundled` enforceable.
