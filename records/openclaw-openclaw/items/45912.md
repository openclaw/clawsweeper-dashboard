## Summary

Keep open: current main still has the reported noisy cross-root symlink limitation, but this PR is not merge-ready because it treats bundled skills as ordinary trusted symlink targets and can bypass `skills.allowBundled`.

## What This Changes

The PR threads configured skill-root realpaths through skill containment checks and adds symlink regression tests for workspace/project-agent and managed/extraDirs skill roots.

## Best Possible Solution

Implement one source-preserving symlink policy: allow safe cross-root links among explicitly trusted non-bundled roots, keep external escapes rejected, and preserve bundled-source semantics or exclude bundled targets so `skills.allowBundled` remains enforceable.

## Reproduction Assessment

Yes. On the PR head, configure `skills.allowBundled` to exclude a bundled skill, then create a workspace or managed symlink to that bundled skill; the new allowed-root check accepts the target while the local loader keeps a non-bundled source.

## Solution Assessment

No. Allowing selected cross-root symlinks may be the right direction, but including `bundledSkillsDir` without preserving bundled classification is not the narrowest maintainable fix.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9
