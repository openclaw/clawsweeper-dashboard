## Summary

Keep open: the cross-root skill symlink problem is still relevant, but this PR should not merge as-is because the diff trusts bundled skill targets without preserving bundled-source classification, which can bypass `skills.allowBundled`. Current `main` still enforces active-root realpath containment, so the branch is not obsolete.

## What This Changes

The branch adds an allowed-root realpath list to skill containment checks and regression tests for symlinks between workspace, managed, and configured skill roots.

## Best Possible Solution

Adopt one source-preserving symlink policy that accepts safe cross-root links among trusted non-bundled roots while keeping `skills.allowBundled` enforceable for bundled targets.

## Reproduction Assessment

Yes for the blocking PR defect: source inspection shows a symlink from a non-bundled scan root to an excluded bundled skill would pass the new allowed-root check, then load with the caller source and skip the bundled allowlist. I did not execute it because this was a read-only review.

## Solution Assessment

No. Allowing selected cross-root symlinks may be the right direction, but the proposed implementation broadens trusted roots without preserving bundled-source semantics or excluding bundled targets.

## Review Findings

Overall correctness: patch is incorrect
