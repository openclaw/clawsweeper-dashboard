## Label Justifications

- `P1`: The reported release bug can replace normal channel replies with a visible harness error once memory flush triggers for CLI-backed sessions.
- `impact:message-loss`: The failure surfaces as the assistant reply instead of the expected response in chat channels.
- `impact:session-state`: The failure occurs in pre-compaction memory flush and affects session continuity work near context limits.
- `impact:auth-provider`: The root cause is provider/runtime routing for persisted CLI runtime aliases such as `claude-cli`.

## Summary

Current main already contains the central fix: provider-owned CLI runtime aliases are recognized before `MissingAgentHarnessError`, and regression coverage confirms `claude-cli` selects PI instead of throwing. The fix is not in `v2026.5.20` yet, so this is a current-main-only implemented close.

## What This Changes

Fix memory flush for CLI-backed sessions whose persisted runtime override is a provider-owned CLI alias such as `claude-cli`.

## Best Possible Solution

Keep the current harness-selection fix and ship it in the next OpenClaw release; no additional core repair is needed for this report.

## Reproduction Assessment

Yes for the reported release behavior by source/history: memory flush passes persisted runtime overrides into fallback execution, and the pre-fix harness selector threw for `claude-cli`. Current main no longer has that failing path because the provider-owned CLI alias guard runs before `MissingAgentHarnessError`.

## Solution Assessment

Yes. Centralizing the provider-owned CLI alias passthrough in `selectAgentHarnessDecision` fixes memory flush and other inherited-runtime callers without weakening fail-closed behavior for unknown or wrong-provider runtimes.
