## Summary

Keep open. Current main still sends Feishu streaming replies as CardKit card references, while the quoted-message and thread-history read paths neither request raw CardKit content nor parse `json_card`; the related narrow PR is closed unmerged and the overlapping open PR still needs maintainer review.

## What This Changes

Feishu should let agents read or quote streaming CardKit replies as actual text instead of `[Interactive Card]` in shared group chats.

## Best Possible Solution

Land one Feishu-owned read-path fix that requests validated raw CardKit content for both `getMessageFeishu()` and `listFeishuThreadMessages()`, parses returned card summary/body text before existing fallbacks, and adds focused quoted-message and thread-history regression tests. Coordinate #56795 and #55466 so maintainers choose one implementation path.
