## Summary

Keep open. Current main still exposes only the final Active Memory summary cap and QMD mode, while shared `memory_search` still merges memory and wiki supplement hits under one score sort and `maxResults` slice, so the requested configurable or reserved memory/wiki allocation remains missing.

## What This Changes

Request to make Active Memory's memory/wiki recall allocation tunable so wiki results are not crowded out by memory under the default recall budget.

## Best Possible Solution

Add an Active Memory-owned recall allocation policy that preserves wiki coverage when wiki supplements are enabled, keeps `maxSummaryChars` as the final injected-summary cap, and updates schema, manifest UI hints, docs, focused tests, and changelog together.

## Reproduction Assessment

Yes. A focused mocked test can register a wiki supplement, mock more higher-scored memory hits than the requested `maxResults`, call `memory_search` with `corpus: "all"`, and observe wiki hits sliced out of the combined sorted result list.

## Solution Assessment

Unclear. The requested knobs are a viable additive fix, but the narrowest maintainable solution may be a balanced per-source reservation with optional caps rather than exposing every internal snippet and formatting budget directly.
