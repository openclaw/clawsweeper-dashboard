## Summary

Close as implemented. Current main and the shipped v2026.5.4 release implement the issue's proposed alternative: balanced memory/wiki corpus selection with backfill for `corpus=all`, so one corpus no longer consumes the bounded recall result set before the other can appear.

## What This Changes

Make Active Memory and memory-wiki recall avoid memory/wiki starvation by configuring or otherwise splitting recall capacity between memory and wiki results.

## Best Possible Solution

Keep the shipped balanced `corpus=all` implementation and avoid adding low-level Active Memory formatter knobs unless a separate tuning need is filed.

## Reproduction Assessment

No current-main failing reproduction remains. The historical source path was covered by the merged regression tests: mixed memory/wiki hits under a small `maxResults` now retain both corpora.

## Solution Assessment

Yes. The balanced corpus-selection and backfill path is narrower than exposing formatter constants as public config, and it implements the issue's proportional-split alternative without changing plugin APIs.
