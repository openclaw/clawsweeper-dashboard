## Summary

Keep open. Current main still lacks any configurable Active Memory memory/wiki recall allocation, and the shared memory/wiki paths still combine sources under one score sort and result cap; the prior ClawSweeper keep-open comment remains consistent with the current source tree.

## What This Changes

Request to make Active Memory recall allocation between memory and wiki results configurable or balanced so wiki context can reach the injected Active Memory summary.

## Best Possible Solution

Add an Active Memory-owned recall policy that preserves wiki coverage when wiki supplements are enabled, keeps maxSummaryChars as the final injected-summary cap, and updates config schema, manifest UI hints, docs, focused tests, and changelog together.

## Reproduction Assessment

Yes. A focused mocked test can seed higher-scored memory hits plus lower-scored wiki supplement hits, call memory_search with corpus=all and a small maxResults, and observe wiki hits dropped after the combined sort/slice; static source inspection confirms that path on current main.

## Solution Assessment

Unclear. The requested knobs would solve the operator tuning need, but the narrowest maintainable fix may be a balanced per-source recall policy with additive optional caps rather than directly exposing every snippet and wiki formatting budget.
