## Summary

Keep open. Current main still lacks Active Memory-owned recall allocation knobs or a source reservation policy, and the shared memory/wiki search path can still let memory hits crowd wiki supplement hits out under one combined result cap.

## What This Changes

Request to make Active Memory's recall allocation between memory and wiki configurable or proportionally balanced.

## Best Possible Solution

Add an Active Memory-owned recall allocation policy that preserves wiki coverage when wiki supplements are enabled, keeps maxSummaryChars as the final injected-summary cap, and documents minimal optional tuning config.

## Reproduction Assessment

Yes. A focused mocked test can register a wiki corpus supplement, mock higher-scored memory hits, call memory_search with corpus=all and a small maxResults, and observe wiki hits being dropped by the combined sort/slice path.

## Solution Assessment

Yes, if solved as a narrow plugin-owned policy with generic memory-core support only where needed. A balanced per-source default plus optional caps is more maintainable than exposing every low-level formatter constant directly.

## Repair Work Prompt

Fix https://github.com/openclaw/openclaw/issues/75607. Active Memory plus memory-wiki recall currently lets memory hits consume the shared memory_search corpus=all result/summary budget because Active Memory exposes only maxSummaryChars and memory-core merges, sorts, and slices memory and wiki supplement hits under one maxResults cap. Implement a narrow Active Memory-owned recall allocation policy that preserves wiki coverage when wiki supplements are enabled, keeping maxSummaryChars as the final injected-summary cap. Prefer balanced per-source defaults with minimal optional config documented in the plugin manifest/docs; do not add bundled plugin ids to core and do not move owner-specific behavior out of the plugins. Add regression coverage for config normalization/schema and for memory_search corpus=all when memory hits outrank wiki supplement hits. Update docs/concepts/active-memory.md and CHANGELOG.md for the user-facing behavior.
