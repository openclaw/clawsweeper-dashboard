## Summary

Keep open: the launchd direction is well-supported by current-main source evidence, but latest PR head still bypasses the new persistent-disable mode after a prior bootout and the supplied macOS proof is narrative-only rather than inspectable current-head output.

## What This Changes

The PR changes macOS LaunchAgent stop to use `launchctl bootout` by default, adds `openclaw gateway stop --disable`, skips repair kickstart when an already-loaded service is running, and updates tests, docs, and changelog.

## Best Possible Solution

Keep the bootout-by-default and skip-kickstart direction, route `--disable` through the not-loaded launchd stop path with lifecycle coverage, then attach inspectable latest-head macOS proof.

## Reproduction Assessment

Yes for source-level reproduction: current main unconditionally disables before stop and kickstarts in the already-loaded repair branch. I did not run a live macOS launchd reproduction in this read-only review.

## Solution Assessment

No, not as written. The launchd implementation direction is narrow, but the CLI lifecycle layer must honor `--disable` after a prior bootout/not-loaded state and the PR still needs inspectable latest-head real behavior proof.
