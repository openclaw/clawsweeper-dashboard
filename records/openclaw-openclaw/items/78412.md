## Summary

Keep open: the current-main launchd bugs are real and the PR direction is useful, but latest head still bypasses `--disable` in the not-loaded stop path and the external real-behavior proof is narrative-only and not for the latest head.

## What This Changes

This PR changes macOS Gateway LaunchAgent stop to bootout by default, adds `gateway stop --disable`, skips repair kickstart for already-running LaunchAgents, and updates launchd tests, docs, and changelog.

## Best Possible Solution

Keep the bootout-by-default and skip-kickstart fixes, make `gateway stop --disable` persist the disable state even after a prior bootout/not-loaded state, and attach inspectable latest-head macOS launchd proof.

## Reproduction Assessment

Yes for source-level reproduction: current main unconditionally disables before stop and kickstarts in the already-loaded repair branch. I did not run a live macOS launchd reproduction in this read-only review.

## Solution Assessment

No, not as latest head stands. The launchd implementation direction is narrow, but the lifecycle layer still bypasses `--disable` after a not-loaded state and the PR still needs inspectable latest-head real behavior proof.

## Review Findings

Full review comments:

- **[P2] Route --disable through not-loaded stops:** `src/cli/daemon-cli/lifecycle-core.ts:417`
