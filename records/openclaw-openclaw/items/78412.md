## Summary

Keep open: the PR targets real current-main macOS launchd bugs and is directionally useful, but it is not merge-ready because the new `--disable` path misses installed-but-unloaded LaunchAgents and the after-fix macOS proof is still prose-only.

## What This Changes

The PR changes macOS Gateway stop to boot out the LaunchAgent by default, adds `gateway stop --disable`, skips repair kickstart when an already-loaded service is running, and updates launchd tests, Gateway docs, and the changelog.

## Best Possible Solution

Keep the bootout-by-default and running-state repair direction, fix `--disable` so it persists disable state even for installed-but-unloaded LaunchAgents, then merge only after raw macOS launchd proof and focused checks are clean.
