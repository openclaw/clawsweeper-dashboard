## Summary

This PR should stay open: current main source confirms the reported macOS launchd stop and repair problems, but the latest PR head still lets `gateway stop --disable` return from the not-loaded path without persisting launchd disable, and the live macOS proof is text-only rather than inspectable output.

## What This Changes

The PR changes macOS gateway stop to `launchctl bootout` by default, adds `gateway stop --disable`, skips repair kickstart when an already-loaded LaunchAgent is already running, and updates tests, docs, and changelog.

## Best Possible Solution

Fix the not-loaded `--disable` routing, keep the repair skip-kickstart change, and get maintainer-approved macOS lifecycle semantics with redacted latest-head launchd proof before merge.

## Reproduction Assessment

Yes for source-level reproduction: current main unconditionally disables before stop and unconditionally kickstarts in the already-loaded repair branch. I did not run a live macOS launchd reproduction in this read-only review.

## Solution Assessment

No, not as current PR head stands. The launchd implementation direction is focused, but `--disable` is bypassed when the service is already not loaded and the changed stop contract still needs maintainer acceptance plus inspectable live proof.
