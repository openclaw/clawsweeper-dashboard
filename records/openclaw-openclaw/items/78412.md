## Summary

Keep open: current main source supports the reported launchd bugs and the PR direction is reasonable, but the latest head still bypasses the new persistent-disable path after a default bootout and lacks current-head diagnostic macOS proof.

## What This Changes

The PR changes macOS gateway stop to boot out the LaunchAgent by default, adds a launchd `gateway stop --disable` path, skips repair kickstart for already-running LaunchAgents, and updates tests, docs, and the changelog.

## Best Possible Solution

Keep the bootout-by-default and skip-kickstart direction, but route `--disable` through launchd even when the service is not loaded, add lifecycle-core coverage for that path, and attach current-head macOS launchd proof before merge.

## Reproduction Assessment

Yes for source-level reproduction: current main unconditionally disables before stop and kickstarts in the already-loaded repair branch. I did not establish a live current-main macOS launchd repro in this read-only review.

## Solution Assessment

No, not as written. The launchd implementation direction is narrow, but the CLI lifecycle layer must honor `--disable` after a prior bootout/not-loaded state and the PR needs current-head real behavior proof.
