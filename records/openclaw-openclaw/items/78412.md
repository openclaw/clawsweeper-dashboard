## Summary

Keep open: the patch fixes the right launchd branches, but it still bypasses `--disable` whenever the LaunchAgent has already been booted out, and the supplied live proof is not current-head diagnostic output.

## What This Changes

The PR changes macOS Gateway stop to boot out the LaunchAgent by default, adds a launchd-only `gateway stop --disable` path, skips repair kickstart when an already-loaded LaunchAgent is running, and updates tests, docs, and the changelog.

## Best Possible Solution

Keep the bootout-by-default and skip-kickstart direction, route `--disable` through the launchd stop/disable path even when the service is already booted out, add lifecycle coverage for that path, and attach redacted current-head macOS launchd proof before merge.

## Reproduction Assessment

Yes for source-level reproduction: current main unconditionally disables before stop and kickstarts in the already-loaded repair branch. I did not establish a live current-main macOS launchd repro in this read-only review.

## Solution Assessment

No, not as written. The implementation direction is narrow, but the PR must honor `--disable` after a prior bootout and attach current-head real macOS proof before merge.
