## Summary

Keep this PR open and do not merge yet: current `main` source confirms the two macOS launchd bugs and the PR mostly targets the right code paths, but the new `--disable` flag misses an installed-but-unloaded LaunchAgent case and the live proof comment is narrative-only rather than terminal/log output.

## What This Changes

The PR changes macOS Gateway stop to boot out the LaunchAgent by default, adds `gateway stop --disable`, skips launchd repair kickstart when an already-loaded service is running, and updates launchd tests, Gateway docs, and the changelog.

## Best Possible Solution

Update the PR so `--disable` can persist launchd disable state even for installed-but-unloaded LaunchAgents, then land after macOS terminal/log proof covers default stop, start recovery, KeepAlive respawn, `--disable`, and the repair-running path.

## Reproduction Assessment

Yes for source-level reproduction: current `main` unconditionally disables before stop and unconditionally kickstarts in the already-loaded repair branch. I did not run a live macOS launchd repro in this read-only review.

## Solution Assessment

No, not merge-ready as written: bootout-by-default plus a running-state repair check is the right direction, but the `--disable` CLI semantics have an uncovered not-loaded case and the real behavior proof needs raw observable output.

- **[P2] Honor --disable when the LaunchAgent is booted out:** `src/cli/daemon-cli/lifecycle-core.ts:414`
