## Summary

Keep open: the reported main-branch launchd bugs are source-reproducible and the PR targets the right area, but latest head still has a `--disable` lifecycle blocker and lacks inspectable latest-head macOS launchd proof.

## What This Changes

The PR changes macOS Gateway LaunchAgent stop to use `launchctl bootout` by default, adds `gateway stop --disable`, skips repair kickstart for already-running LaunchAgents, and updates launchd tests, docs, and changelog.

## Best Possible Solution

Keep the bootout-by-default direction only after making `--disable` persistently disable even from the not-loaded state, adding focused regression coverage, and collecting redacted latest-head macOS launchd proof.

## Reproduction Assessment

Yes for source-level reproduction: current main unconditionally calls `launchctl disable` in `stopLaunchAgent` and unconditionally kickstarts after an already-loaded bootstrap. I did not establish a live macOS failure in this read-only review.

## Solution Assessment

No, not as latest head stands. The implementation direction is plausible, but the new `--disable` path is bypassed for an already booted-out LaunchAgent and the lifecycle contract still needs inspectable live proof.

## Review Findings

Overall correctness: patch is incorrect
