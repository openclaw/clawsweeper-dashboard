## Summary

The PR is a plausible fix direction, but it should stay open because the latest head still leaves the new `--disable` behavior ineffective for an installed-but-booted-out LaunchAgent, and the supplied macOS proof is narrative rather than raw observable output.

## What This Changes

The PR changes macOS Gateway stop to boot out the LaunchAgent by default, adds `gateway stop --disable`, skips already-loaded repair kickstart when the service is running, and updates launchd tests, Gateway docs, and the changelog.

## Best Possible Solution

Keep the bootout-by-default and skip-kickstart direction, route `--disable` through the launchd not-loaded case, and merge only after raw macOS launchd proof is attached.
