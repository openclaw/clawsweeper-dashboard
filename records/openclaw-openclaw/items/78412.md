## Summary

Keep this PR open: the launchd fix direction is useful and source-backed, but the latest head still has a blocking `--disable` lifecycle gap and the external macOS proof is not strong current-head diagnostic evidence.

## What This Changes

This PR changes macOS Gateway service stop to boot out the LaunchAgent by default, adds a launchd-only `gateway stop --disable` path, skips repair kickstart for already-running LaunchAgents, and updates tests, docs, and the changelog.

## Best Possible Solution

Keep the bootout-by-default and skip-kickstart direction, route `--disable` through launchd even when the service is already booted out, add lifecycle-core coverage for that path, and attach redacted current-head macOS launchd proof before merge.

## Reproduction Assessment

Yes for source-level reproduction: current main unconditionally disables before stop and kickstarts in the already-loaded repair branch. I did not establish a live current-main macOS launchd repro in this read-only review.

## Solution Assessment

No, not as written. The implementation direction is narrow, but the PR must honor `--disable` after a prior bootout and attach current-head real macOS proof before merge.
