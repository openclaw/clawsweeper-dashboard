## Summary

Keep open: the source-reproduced launchd bugs are real, but the latest PR still bypasses the new persistent-disable path when the LaunchAgent is already booted out, and the external proof is not inspectable current-head diagnostics.

## What This Changes

The PR changes macOS launchd stop to boot out by default, adds `gateway stop --disable`, skips kickstart when an already-loaded service is running, and updates tests, docs, and changelog.

## Best Possible Solution

Keep the bootout-by-default and skip-kickstart direction, but make `--disable` reach the launchd disable path even when the service is already not loaded, cover that lifecycle case, and attach inspectable current-head macOS proof.

## Reproduction Assessment

Yes for source-level reproduction: current main unconditionally disables before stop and kickstarts in the already-loaded repair branch, and the PR’s remaining `--disable` gap is visible in the current-head control flow. I did not establish a live current-main macOS launchd repro in this read-only review.

## Solution Assessment

No, not as written. The launchd implementation direction is narrow, but the CLI lifecycle layer must honor `--disable` after a prior bootout/not-loaded state and the PR needs inspectable real behavior proof for the latest head.
