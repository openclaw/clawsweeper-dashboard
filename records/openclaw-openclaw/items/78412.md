## Summary

Keep this PR open: the current-main bug is source-reproducible and the launchd-side direction is plausible, but latest head still has a blocking `--disable` lifecycle bug and lacks inspectable latest-head macOS behavior proof.

## What This Changes

This PR changes macOS Gateway LaunchAgent stop to bootout by default, adds `gateway stop --disable`, skips kickstart when an already-loaded service is running, and updates launchd tests, docs, and changelog.

## Best Possible Solution

Keep the bootout-by-default direction only after routing `--disable` through the booted-out/not-loaded case, adding daemon-CLI regression coverage, and collecting redacted latest-head macOS launchd proof.

## Reproduction Assessment

Yes for source-level reproduction: current main unconditionally disables launchd on stop and unconditionally kickstarts after an already-loaded bootstrap. I did not establish a live current-main macOS launchd failure in this read-only review.

## Solution Assessment

No, not as latest head stands. The launchd-side fix is plausible, but `--disable` is bypassed when the LaunchAgent is already booted out, so the documented persistent-disable command can report success without persisting the disable.
