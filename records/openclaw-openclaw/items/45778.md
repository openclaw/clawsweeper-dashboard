## Summary

Keep open. Current main still lacks both requested platform seams: cron has no zero-LLM direct command payload, and skill env injection does not expose foreground reply-route variables. The SignalRadar ClawHub skill and webhook workaround are useful context, but this is still a core cron/routing contract decision rather than ClawHub-only cleanup.

## What This Changes

The issue requests a zero-LLM way for skills running from system crontab to send OpenClaw chat messages, either through a directExec cron payload or foreground reply-route environment variables.

## Best Possible Solution

Expose one explicit zero-LLM notification contract that preserves channel permissions, either scoped reply-route injection for user-authorized skill runs or an opt-in directExec cron payload limited to approved installed-skill commands and wired through existing cron delivery, logging, timeout, and failure semantics.

## Reproduction Assessment

Yes for the missing capability. Source inspection on current main shows cron only supports systemEvent and agentTurn payloads, detached delivery jobs must be agentTurn, and skill env injection does not expose OPENCLAW_REPLY_* route variables.

## Solution Assessment

Unclear until maintainers choose the contract. Option A is the broader scheduler solution but needs host-exec controls; Option B is smaller but still needs route permission, lifetime, and revocation semantics.
