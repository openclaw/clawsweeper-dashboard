## Summary

Keep open: current main still lacks both requested seams for SignalRadar-style zero-LLM background alerts. The direct-exec cron half is covered by the open canonical tracker https://github.com/openclaw/openclaw/issues/18160, but this issue also preserves the distinct foreground reply-route env-injection option and the reporter confirmed it remains relevant on April 29, 2026.

## What This Changes

The issue requests a zero-LLM way for skills using system crontab to deliver chat messages, either through a direct cron exec payload or scoped reply-route environment variables.

## Best Possible Solution

Keep native cron direct-exec design aligned with https://github.com/openclaw/openclaw/issues/18160, and separately decide whether OpenClaw wants a scoped, authorized reply-route env contract for skills that persist a user-approved chat route.

## Reproduction Assessment

Yes for source-level capability absence, not as a strict bug reproduction. Current main has no direct cron exec payload and no OPENCLAW_REPLY_CHANNEL/OPENCLAW_REPLY_TARGET env contract; I did not run a live SignalRadar or system crontab scenario.

## Solution Assessment

Unclear until maintainers choose the product and security contract. A native direct-exec cron payload belongs with the open canonical tracker, while reply-route env injection is smaller but needs explicit authorization, scoping, and stale-target behavior.
