## Summary

This remains a valid platform feature request, not stale cleanup: SignalRadar already exists as a ClawHub skill, but the missing piece is an OpenClaw cron/reply-route contract, and current main still has no zero-LLM direct cron execution or foreground reply-route environment injection.

## What This Changes

The issue requests a zero-LLM background notification path for skills, either via a direct cron execution payload with announce delivery or foreground reply-route environment injection.

## Best Possible Solution

A maintainer-owned design should choose one scoped platform contract, then update cron protocol/types, scheduler dispatch, CLI/docs, and focused regression coverage around delivery and authorization.

## Reproduction Assessment

Not applicable as a feature request. Source inspection confirms the current cron contract only supports `systemEvent` and `agentTurn` payloads, and the requested reply-route env vars are absent.

## Solution Assessment

Unclear until maintainers choose the platform seam. Option B is smaller, while Option A keeps delivery in the scheduler but needs a stronger security model for command execution.
