## Summary

Keep open: current main and the latest release still lack a zero-LLM cron delivery path or foreground reply-route environment injection, and the reporter added recent confirmation that the ClawHub skill workaround is still needed. This is a real feature/API gap, but the direct execution and persisted route options need maintainer product and security design before implementation.

## What This Changes

Request a zero-LLM background skill notification path, either through a direct cron command payload with announce delivery or foreground reply-route env vars for later `openclaw message send`.

## Best Possible Solution

Define a narrow, documented core/plugin contract for authorized background delivery, then implement either scoped reply-route injection or an opt-in direct execution cron mode with regression coverage and docs.

## Reproduction Assessment

Not applicable as a strict bug reproduction. Source inspection confirms the current cron, CLI, docs, and release surfaces only support system events or LLM-backed agent-turn jobs, and the requested reply-route env vars are absent.

## Solution Assessment

Unclear until maintainers choose the product and security contract. Reply-route env injection is the smaller delivery seam, while `directExec` needs a stronger explicit opt-in and command-execution policy before it is safe to add.
