## Summary

Keep open: current main still lacks both a zero-LLM cron delivery path and foreground reply-route env injection, and this is a feature/API gap that needs maintainer product and security design rather than stale cleanup.

## What This Changes

Requests a platform seam that lets skills send background notifications without an LLM turn, either through direct command cron delivery or injected reply-route environment variables.

## Best Possible Solution

Define a narrow, documented core/plugin contract for authorized background delivery, then implement either scoped reply-route injection or an opt-in direct execution cron mode with regression coverage and docs.

## Reproduction Assessment

Not applicable as a strict bug reproduction. Source inspection confirms the current cron, CLI, plugin scheduling, docs, and environment surfaces only support system events or LLM-backed agent-turn jobs, while the requested reply-route env vars are absent.

## Solution Assessment

Unclear until maintainers choose the product and security contract. Reply-route env injection is the smaller delivery seam, while `directExec` needs an explicit opt-in and command-execution policy before it is safe to add.
