## Summary

Keep open. Current main still lacks both requested platform seams: cron has no zero-LLM direct command payload, and skill env injection does not expose a foreground reply route that a later system crontab job can reuse. The SignalRadar ClawHub skill is useful context, but this is not a ClawHub-only request because the missing piece is a core cron/routing contract.

## What This Changes

The issue asks OpenClaw to let skills send zero-LLM background notifications from system cron, either through a directExec cron payload or scoped foreground reply-route environment injection.

## Best Possible Solution

Expose one explicit zero-LLM notification contract that preserves channel permissions: either scoped reply-route injection for user-authorized skill runs, or an opt-in directExec cron payload restricted to approved installed-skill commands and wired through existing cron delivery, logging, timeout, and failure-notification semantics.

## Reproduction Assessment

Yes. Static inspection of current main is enough: cron types/schema/validation/execution accept only systemEvent or agentTurn, and the skill env path injects configured skill env/apiKey values rather than OPENCLAW_REPLY_* route variables.

## Solution Assessment

Unclear until maintainers choose the contract. Option A is the broader scheduler fix but needs host-exec controls; Option B is smaller but still needs permission, account/thread scoping, persistence, and revocation semantics.
