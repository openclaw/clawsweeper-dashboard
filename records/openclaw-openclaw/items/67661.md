## What This Changes

The issue asks OpenClaw to persist the active modelId when incomplete-turn exhaustion paths mark an auth-profile failure.

## Best Possible Solution

Review and either land or consciously reject https://github.com/openclaw/openclaw/pull/67662; the desired code state is for both terminal incomplete-turn failure writes to pass the active modelId with focused regression coverage.

## Reproduction Assessment

Yes, at source level. Current main shows both terminal incomplete-turn branches call maybeMarkAuthProfileFailure without modelId while cooldown storage and lookup only preserve model-scoped behavior when modelId/cooldownModel is present.

## Solution Assessment

Yes. Passing the already available active modelId at the two missing call sites is the narrow maintainable fix; broader session-scoped model exhaustion persistence belongs to separate PRs.
