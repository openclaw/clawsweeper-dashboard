## Summary

Keep open: current main still has the reported call-site propagation gap, and live GitHub shows this issue is still paired with an open closing PR rather than a merged fix.

## What This Changes

The issue asks the embedded runner to include the active modelId when incomplete-turn cooldown failures are persisted so same-session model-scoped fallback does not retry the exhausted model.

## Best Possible Solution

Review and either land or consciously reject the linked narrow PR; the desired code state is for both terminal incomplete-turn failure writes to pass the already available active modelId with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows both terminal incomplete-turn branches call maybeMarkAuthProfileFailure without modelId while the storage and lookup code only preserve model-scoped cooldowns when modelId/cooldownModel is present.

## Solution Assessment

Yes. Passing the already available modelId at the two missing call sites is the narrow maintainable fix; broader session model-exhaustion persistence in other open PRs is separate.
