## Summary

Current main still omits the active model id from both incomplete-turn auth-profile failure writes, while an open PR with closing syntax already targets this exact issue; keep this open until that PR is reviewed and landed or closed.

## What This Changes

The issue requests that incomplete-turn auth-profile failure writes include the active model id so same-run model-scoped cooldowns prevent retrying an exhausted model.

## Best Possible Solution

Review and either land or close PR #67662; the desired end state is a narrow runtime fix that passes the active model id into both incomplete-turn failure writes with regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence reproduction path: both incomplete-turn failure writes omit modelId while the cooldown store records cooldownModel only when modelId is supplied.

## Solution Assessment

Yes. Passing the already available active model id at the two incomplete-turn call sites is the narrowest maintainable fix because the helper and storage contract already support model-scoped cooldown writes.
