## Summary

Keep open: current `main` still has both incomplete-turn failure writes that can persist rate-limit cooldown state without the active model id, and the issue is already paired with open closing implementation work at https://github.com/openclaw/openclaw/pull/67662. The newer related PRs cover broader session-cooldown persistence, but they are still open and do not make this narrow runner bug implemented on `main`.

## What This Changes

The issue requests that incomplete-turn exhaustion handling persist auth-profile failures with the active `modelId` so model-scoped cooldown checks do not retry the same exhausted model in the same run/session.

## Best Possible Solution

Review and land or reject the linked narrow PR; the intended end state is for both incomplete-turn failure writes to pass the active `modelId` with regression coverage for reasoning-only and generic incomplete-turn exhaustion paths.

## Reproduction Assessment

Yes. Source inspection on current `main` shows both terminal incomplete-turn branches can persist a rate-limit auth-profile failure without `modelId`, while the cooldown store records model scope only when `modelId` is supplied.

## Solution Assessment

Yes. Passing the already available active model id at the two missing runner call sites is the narrow maintainable fix because the helper, storage layer, lookup path, tests, and docs already support model-scoped cooldowns.
