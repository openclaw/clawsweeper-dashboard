## Summary

Keep open. Current main still omits the active model id from both incomplete-turn auth-profile failure writes, while the cooldown store only records a model-scoped cooldown when `modelId` is supplied; PR #67662 is open and unmerged with closing syntax for this issue.

## What This Changes

Persist the active `modelId` when incomplete-turn exhaustion paths mark an auth-profile failure, so same-run model-scoped cooldowns can avoid retrying the exhausted model.

## Best Possible Solution

Both incomplete-turn failure writes should pass the active `modelId`, with regression coverage proving reasoning-only exhaustion and general incomplete-turn cooldown persistence stay model-scoped.

## Reproduction Assessment

Yes. Static inspection on current main reproduces the issue: both incomplete-turn writes omit `modelId`, and `computeNextProfileUsageStats` only records `cooldownModel` from `params.modelId`.

## Solution Assessment

Yes. Passing the already available active `modelId` at the two call sites is the narrowest maintainable fix because the helper and storage contracts already support model-scoped cooldown writes.
