## Summary

Keep open. Current main still omits the active model id from the two incomplete-turn auth-profile failure writes, and PR #67662 is an open, unmerged implementation candidate that explicitly closes this issue.

## What This Changes

Ensure incomplete-turn auth-profile failure writes in `runEmbeddedPiAgent` include the active `modelId` so model-scoped cooldowns prevent same-run retries of an exhausted model.

## Best Possible Solution

Review and either land or close PR #67662. The desired end state is a focused runtime change that passes `modelId` into both incomplete-turn failure writes, with regression coverage for reasoning-only exhaustion and general incomplete-turn cooldown persistence.

## Reproduction Assessment

Yes. The issue is reproducible by static inspection on current main: both incomplete-turn branches omit `modelId`, while the auth-profile cooldown store only records `cooldownModel` when `modelId` is supplied.

## Solution Assessment

Yes. Passing the already-available active `modelId` at the two incomplete-turn call sites is the narrowest maintainable fix because the helper and storage contract already support model-scoped cooldown writes.
