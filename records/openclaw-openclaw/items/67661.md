## Summary

Keep open: current main still omits the active model id from both incomplete-turn auth-profile failure writes, and https://github.com/openclaw/openclaw/pull/67662 is already the open closing implementation candidate for the same bug.

## What This Changes

The issue asks OpenClaw to persist the active `modelId` when incomplete-turn exhaustion paths record model-scoped auth-profile cooldown failures.

## Best Possible Solution

Land or reject the linked PR; the desired end state is that both incomplete-turn failure writes pass the active `modelId` and have regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows both terminal incomplete-turn branches can persist a rate-limit cooldown without `modelId`, while the cooldown store records model scope only when `modelId` is supplied.

## Solution Assessment

Yes. Passing the already available active `modelId` at the two missing call sites is the narrow maintainable fix because the helper, storage layer, docs, and lookup path already support model-scoped cooldowns.
