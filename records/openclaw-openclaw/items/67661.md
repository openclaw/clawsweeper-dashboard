## Summary

Keep open: current main still has the reported modelId propagation gap in the incomplete-turn terminal paths, and the issue is already paired with an open closing PR rather than a merged fix.

## What This Changes

Persist the active modelId when incomplete-turn exhaustion marks auth-profile failures so same-session model-scoped cooldowns can prevent retrying the exhausted model.

## Best Possible Solution

Review, rebase, or port https://github.com/openclaw/openclaw/pull/67662 so both terminal incomplete-turn failure writes pass the active modelId with focused regression coverage.

## Reproduction Assessment

Yes. Source inspection on current main shows both terminal incomplete-turn branches call maybeMarkAuthProfileFailure without modelId while cooldown storage only records cooldownModel when modelId is supplied.

## Solution Assessment

Yes. Passing the already available active modelId at the two missing call sites is the narrow maintainable fix; the broader session-scoped exhaustion PRs are separate feature work.
