## Summary

Keep open. Current main still omits the active model id from both incomplete-turn auth-profile failure writes, while the cooldown store only records a model-scoped cooldown when `modelId` is supplied. The linked PR #67662 is open and unmerged with closing syntax for this issue, so repository policy keeps the issue open until that PR lands or is closed.

## What This Changes

Persist the active `modelId` when incomplete-turn exhaustion paths mark an auth-profile failure so model-scoped cooldowns prevent same-run retries of the exhausted model.

## Best Possible Solution

Review and either land or close PR #67662. The desired end state is a focused runtime change that passes `modelId` into both incomplete-turn failure writes, with regression coverage for reasoning-only exhaustion and general incomplete-turn cooldown persistence.

## Reproduction Assessment

Yes. Static inspection on current main reproduces the issue: both incomplete-turn failure writes omit `modelId`, and the auth-profile cooldown store records `cooldownModel` only when a model id is supplied.

## Solution Assessment

Yes. Passing the already available active `modelId` at the two call sites is the narrowest maintainable fix because the helper and storage contract already support model-scoped cooldown writes.
