## Summary

Keep open. Current main still rejects Telegram account-level `agentId` in strict account config, and #63380 remains the open unmerged implementation candidate linked with closing syntax for this regression.

## What This Changes

Restore or migrate the previously working Telegram multi-account config shape that used `channels.telegram.accounts.<id>.agentId` for per-account agent routing.

## Best Possible Solution

Review #63380 as the active implementation candidate, with the final fix either accepting and routing or migrating legacy account-level `agentId`, plus regression coverage for the reported multi-account config.

## Reproduction Assessment

Yes. The issue provides a concrete multi-account config, and current source shows each account is parsed by a strict schema that omits `agentId`, so the validation failure is source-reproducible on current main.

## Solution Assessment

Unclear until #63380 is reviewed. The narrow durable fix needs routing semantics or a doctor/docs migration path, not only allowing the field through schema validation.
