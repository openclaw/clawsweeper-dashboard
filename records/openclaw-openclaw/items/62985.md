## Summary

Keep open. Current main still rejects Telegram account-level `agentId`, the documented account-routing path is top-level `bindings[]`, and #63380 remains the paired unmerged implementation candidate from the provided GitHub context.

## What This Changes

The issue reports a regression where Telegram multi-account configs with `agentId` under `channels.telegram.accounts.<id>` became invalid after updating from 2026.4.5 to 2026.4.8.

## Best Possible Solution

Review #63380 as the active implementation candidate; the final outcome should either preserve legacy account-level `agentId` routing with schema, type, runtime, and regression-test coverage or provide an explicit doctor/docs migration to top-level `bindings[]`.

## Reproduction Assessment

Yes. The provided config shape is source-reproducible on current main because strict Telegram account schema rejects unknown `agentId`, while only topic config accepts `agentId`.

## Solution Assessment

Unclear until #63380 is reviewed. Restoring compatibility is plausible, but the maintainable fix must either route the legacy account field correctly or guide users to an equivalent `bindings[]` migration.
