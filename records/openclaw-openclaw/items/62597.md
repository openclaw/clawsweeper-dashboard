## Summary

Keep open. The PR addresses a source-reproducible memory-core performance issue that is still present on current main, but it needs after-fix real behavior proof and a refresh against current main before merge.

## What This Changes

Adds a memory-core cache for `hasIndexedContent()`, sets it after chunk writes, and clears it on chunk deletes, reset, and safe-reindex DB swaps.

## Best Possible Solution

Refresh the branch on current main, invalidate the cache on every chunk/table mutation and DB handle replacement, add focused regression coverage, and provide redacted runtime proof that repeated searches avoid redundant probes.

## Reproduction Assessment

Yes, source-reproducible. Current main reaches `hasIndexedContent()` from `search()`, and that method prepares direct `chunks` and FTS SQLite probes with no cache; I did not run a benchmark in this read-only review.

## Solution Assessment

Yes, with a rebase adjustment. A manager-local cache is the narrow maintainable fix, but it must be invalidated on all mutation and DB replacement paths, including readonly recovery.
