## Summary

Keep open: current main and v2026.5.12 still run the uncached SQLite probes, and this PR is a focused optimization, but it lacks required real behavior proof and still needs a small DB-reopen invalidation refresh.

## What This Changes

Adds a manager-local memory-core `hasIndexedContent()` cache and invalidates it on chunk writes, chunk deletes, index reset, and safe-reindex DB swaps.

## Best Possible Solution

Keep the manager-local cache, invalidate it on every chunk/table mutation and DB handle replacement, add focused regression coverage, and attach redacted runtime proof showing repeated searches avoid redundant probes.

## Reproduction Assessment

Yes. Source inspection on current main shows each `search()` reaches `hasIndexedContent()`, which prepares SQLite existence probes; a focused regression can count prepare/get calls across repeated searches.

## Solution Assessment

Yes, with a refresh. A memory-core-local indexed-content cache is the narrow maintainable direction, provided it is invalidated across writes, deletes, resets, safe reindex, and readonly-recovery DB replacement.
