## Summary

Keep open: the patch looks correct and has sufficient real behavior proof, but it introduces fail-closed proxy limits and overlaps two open proxy hardening PRs, so the remaining decision is maintainer acceptance or tuning rather than cleanup closure.

## What This Changes

The PR bounds proxy error-body reads, caps total proxy SSE bytes with the shared SSE guard, adds a 120 second SSE read idle timeout, and adds focused proxy runtime tests.

## Best Possible Solution

Land this bounded proxy response guard only after maintainers accept or tune the 16 MiB caps and 120 second idle timeout, then coordinate or rebase the overlapping proxy hardening PRs.

## Reproduction Assessment

Yes. Current main and v2026.6.10 have source-visible unbounded non-2xx response.json() and direct SSE reader.read() paths, and the PR body includes real local HTTP proxy-style output for normal, overflow, error-body overflow, and idle-timeout paths.

## Solution Assessment

Yes, conditionally. The proxy response boundary is the right layer and the total SSE cap reuses the shared guard, but maintainers still need to accept or tune the fail-closed cap and idle-time policy.
