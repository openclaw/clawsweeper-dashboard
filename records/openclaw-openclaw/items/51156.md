## Summary

Keep open. Current main still lacks the requested opt-in final-tag config for non-tag providers, and the linked leak issue remains active, but this PR is not merge-ready because it needs a current-main rebase, fixes for the provider-context and cron paths, a failing test expectation, and contributor real-behavior proof.

## What This Changes

Adds an `agents.defaults.enforceFinalTag` config option and threads it through reply, cron, prompt, docs, and tests so final-tag filtering can be forced for any provider.

## Best Possible Solution

Rebase the opt-in config surface onto current main, preserve provider-runtime context and the current cron executor path, fix the helper test, and keep the broader default-delivery policy on the canonical leak issue.

## Reproduction Assessment

Yes, source-reproducible. Current main only enables `enforceFinalTag` for reasoning-tag providers, while the existing subscriber path already filters to `<final>` content when that run flag is true.

## Solution Assessment

Mostly yes, but not as this branch currently stands. A default-off agent defaults option is a narrow mitigation, but the mergeable version must use current provider context, current cron executor plumbing, passing tests, and redacted real behavior proof.
