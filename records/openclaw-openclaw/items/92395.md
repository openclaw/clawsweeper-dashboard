## Summary

Close as implemented on current main: the central lock-lifetime problem is fixed by current code that releases the session JSONL write lock before provider streaming and before post-turn async/compaction waits, then reacquires it only for bounded transcript write sections. ## What This Changes

The issue asks OpenClaw to stop holding the session JSONL write lock across long agent turns so queued follow-up messages are not lost.

## Best Possible Solution
