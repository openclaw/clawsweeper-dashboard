## Summary

This should stay open: current main and the latest shipped release still treat nonzero Codex token usage as a known zero-dollar cost, so `usage-cost` can report confident `$0` with `missingCostEntries: 0`. The report is actionable and the remaining fix is not covered by the related open cost-trust items.

## What This Changes

The issue asks OpenClaw to stop reporting Codex `gpt-5.5` API-key usage as complete zero-dollar spend when pricing is unknown or missing.

## Best Possible Solution

Treat Codex zero pricing as unknown unless it comes from an explicit trusted pricing source, so nonzero token usage either computes from configured/cache pricing or increments `missingCostEntries` instead of reporting complete free usage.

## Reproduction Assessment

Yes from source inspection: current main records nonzero Codex token usage with a present zero `usage.cost`, and the usage-cost aggregator treats present zero cost as complete rather than missing. I did not run a live Codex API-key session in this read-only sweep.

## Solution Assessment

Yes, the maintainable fix is to make unknown Codex pricing explicit and only report dollar cost when a real pricing source is available. Hardcoding API spend without billing-mode awareness would risk worsening the related OAuth/subscription confusion.
