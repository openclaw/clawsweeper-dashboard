## Label Justifications

- `P1`: The bug can silently disable budget and spike monitoring for real Codex API-key spend in active agent workflows.
- `impact:data-loss`: The usage-cost summary corrupts cost telemetry by recording nonzero token usage as complete zero-dollar spend.
- `impact:auth-provider`: The failure depends on Codex/OpenAI provider attribution, auth mode, and model pricing resolution.

## Summary

Keep open: current main still records nonzero Codex token usage with present zero cost data, and `usage-cost` treats that as complete rather than missing, so API-key spend can remain invisible to cost alerts.

## What This Changes

Report that Codex `gpt-5.5` usage-cost accounting returns confident zero-dollar totals for nonzero token usage because the Codex model catalog and transcript path carry zero pricing.

## Best Possible Solution

Treat Codex zero pricing as unknown unless it comes from a trusted pricing source, so nonzero token usage either computes from valid metered pricing or increments `missingCostEntries` instead of reporting complete free usage.

## Reproduction Assessment

Yes, at source level: current main records Codex token counts with a present zero `usage.cost`, and the usage-cost aggregator treats present zero cost as complete. I did not run a live API-key Codex turn in this read-only review.

## Solution Assessment

Yes, the narrow maintainable direction is to represent unknown Codex pricing explicitly or resolve trusted metered pricing, while preserving separate OAuth/subscription billing semantics.
