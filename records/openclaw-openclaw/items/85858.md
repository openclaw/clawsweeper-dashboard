## Label Justifications

- `P1`: The bug can silently disable budget alerts and spike detection for real API-key spend on active agent installs.
- `impact:auth-provider`: The failure depends on Codex/OpenAI auth routing and model pricing resolution being trusted for cost reporting.

## Summary

This should stay open: current main and the latest release still have a source-backed path where tokenized Codex usage can carry a present zero cost, and no merged fix has landed. The active related implementation is https://github.com/openclaw/openclaw/pull/85882, but it is still open and has a compatibility review blocker around intentional zero-cost configs.

## What This Changes

The issue asks OpenClaw to stop reporting confident `$0` usage-cost totals for unpriced Codex/gpt-5.5 token usage and surface unknown pricing to budget and spike tooling.

## Best Possible Solution

Represent unknown pricing explicitly or source-aware, so tokenized turns with untrusted zero pricing increment `missingCostEntries` while documented intentional zero-cost/local provider configs remain complete `$0` data.

## Reproduction Assessment

Yes. Source inspection shows current main treats present zero cost as complete and never increments `missingCostEntries`, and the reporter supplied live output from a real host showing nonzero token volume with `totalCost: 0` and `missingCostEntries: 0`.

## Solution Assessment

Yes in direction, but not every proposed implementation is safe yet. The durable fix should distinguish unknown pricing from intentionally free pricing rather than treating every zero row as unknown.
