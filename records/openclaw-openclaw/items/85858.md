## Label Justifications

- `P1`: The bug can silently disable budget and spike monitoring for real Codex/OpenAI API-key spend in active agent workflows.
- `impact:data-loss`: The reported behavior corrupts or drops cost information in session usage summaries by recording unknown spend as complete zero-dollar data.
- `impact:auth-provider`: The failure depends on provider/model attribution and pricing resolution for OpenAI/Codex auth paths.

## Summary

This should stay open: current main and the latest shipped release still let Codex app-server turns write a present zero-dollar cost for nonzero token usage, and the usage-cost path treats that as complete rather than missing. No open PR appears to own this exact fix, and the issue is narrower than the adjacent Codex OAuth billing-mode discussion.

## What This Changes

The issue asks OpenClaw to stop reporting confident $0 usage-cost totals for Codex or otherwise unpriced model usage when real API-key spend may exist.

## Best Possible Solution

Keep the issue open and make transcript cost unknown unless the runtime has authoritative pricing; usage-cost should then estimate from trusted config/cache or increment `missingCostEntries` instead of treating synthetic zero as free usage.

## Reproduction Assessment

Yes from source inspection: Codex app-server turns can persist nonzero token usage with `cost.total: 0`, and usage-cost only treats undefined cost as missing. I did not run a live Codex API-key session in this read-only sweep.

## Solution Assessment

Yes: the narrow fix is to preserve unknown cost through Codex transcript emission and cost aggregation, then estimate only from trusted pricing. The broader Codex OAuth/subscription billing semantics should remain with https://github.com/openclaw/openclaw/issues/78760.
