## Label Justifications

- `P1`: The bug can silently disable budget and spike monitoring for real API-key spend on active Codex/gpt-5.5 usage.
- `impact:data-loss`: The affected usage-cost path records misleading complete zero-cost totals instead of missing-cost data for nonzero usage.
- `impact:auth-provider`: The failure depends on provider/model pricing resolution and Codex/OpenAI API-key versus OAuth billing semantics.

## Summary

Keep this open: current main and the latest shipped release still let nonzero unpriced Codex/gpt-5.5 usage appear as complete $0 usage, and a same-author implementation PR now targets the issue but has not merged. The reporter's follow-up usefully sharpened the root cause to unknown pricing being collapsed into zero rather than tracked as missing.

## What This Changes

Make usage-cost distinguish unknown model pricing from genuinely free pricing so unpriced Codex/gpt-5.5 usage increments missingCostEntries instead of reporting a confident $0.

## Best Possible Solution

Preserve unknown pricing as missing while retaining explicit free-model semantics, then review and land or revise https://github.com/openclaw/openclaw/pull/85882.

## Reproduction Assessment

Yes, by source inspection: current main records nonzero Codex usage with present zero cost data and usage-cost treats defined zero as complete rather than missing. I did not run a live API-key Codex session in this read-only review.

## Solution Assessment

Yes for the issue direction: unknown pricing should stay unknown and increment missingCostEntries rather than fabricating spend. The open PR's all-zero heuristic still needs review so explicit free pricing and Codex OAuth semantics are not flattened incorrectly.
