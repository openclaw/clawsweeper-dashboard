## Label Justifications

- `P1`: The bug can silently disable cost alerts and spike detection for real API-key spend in active Gateway/Codex workflows.
- `impact:auth-provider`: The failure depends on provider/model pricing resolution and Codex/OpenAI auth-mode billing semantics.

## Summary

This should stay open: current main and v2026.5.22 still treat nonzero usage with a present all-zero cost as complete zero-dollar spend, and the open repair PR has not merged yet.

## What This Changes

The issue asks OpenClaw to distinguish unknown/unpriced model cost from genuinely free usage so `gateway usage-cost` does not report a confident `$0` for billable Codex/gpt-5.5 token usage.

## Best Possible Solution

Land a contract-safe fix that marks source-missing Codex or pricing-cache rates as unknown/missing for nonzero usage while preserving explicit free/local zero-cost configs.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path: nonzero Codex usage can carry `cost.total: 0`, and current aggregation treats that present zero as complete rather than missing. I did not run a live API-key session in this read-only review.

## Solution Assessment

Yes on direction: unknown pricing should remain unknown so downstream budget tooling can warn instead of trusting `$0`. The safest implementation should distinguish source-missing pricing from explicit free/local zero-cost configuration.
