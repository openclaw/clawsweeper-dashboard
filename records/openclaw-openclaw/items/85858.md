## Label Justifications

- `P1`: The bug can silently disable spend alerts and spike detection for real API-key usage, affecting active operators before they notice charges elsewhere.
- `impact:auth-provider`: The failure depends on provider/model pricing resolution and Codex/OpenAI billing semantics for API-key versus OAuth-backed usage.

## Summary

This should stay open: current main still collapses unknown or all-zero model pricing into a defined $0 cost, and the linked repair PR is open rather than merged. The report is source-reproducible and matches the docs expectation that missing pricing should show token usage without pretending cost is complete.

## What This Changes

The issue reports that `openclaw gateway usage-cost` treats unpriced Codex/OpenAI `gpt-5.5` token usage as complete $0 spend instead of missing pricing, disabling cost-based budget and spike monitoring.

## Best Possible Solution

Land the open repair PR or an equivalent narrow fix that preserves unknown pricing through usage-cost aggregation, counts nonzero-token unpriced turns as missing, and tests zero/free compatibility plus positive recorded costs.

## Reproduction Assessment

Yes from source inspection: all-zero or missing pricing becomes a defined zero cost, and the aggregator only treats `undefined` as missing. I did not run a live Codex API-key session or local test because this sweep is read-only.

## Solution Assessment

Yes: preserving unknown pricing and surfacing nonzero-token unpriced turns as `missingCostEntries` is the narrow maintainable fix. The linked PR is the current implementation candidate, with compatibility around intentional zero pricing as the main review point.
