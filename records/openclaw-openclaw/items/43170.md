## Summary

Keep open: current main supersedes the exact-peer Feishu/plugin-channel parts, but this PR still carries unique catch-all/defaultChannels behavior, lacks real Feishu/QQ Bot proof, and has two P2 correctness blockers.

## What This Changes

The PR extends configured ACP bindings toward Feishu and QQ Bot, adds catch-all `acp.defaultChannels`, auto-enables acpx for ACP config, exports route helpers, wires Feishu routing, and updates docs/tests.

## Best Possible Solution

Rebase onto the generic configured-binding registry, fix the `defaultChannels` agent-selection and Feishu failure semantics, and only keep catch-all/defaultChannels after maintainer product/API approval and live channel proof.

## Reproduction Assessment

Not applicable for the requested feature as a user bug. The blocking PR findings are source-reproducible from PR head: `defaultChannels` synthesizes `main`/runtime-only ACP targets, and Feishu logs then continues after ACP setup failure.

## Solution Assessment

No, not as-is. Current main has a cleaner generic configured-binding seam for exact-peer support, while this branch still needs correctness fixes, a rebase, maintainer approval for catch-all/defaultChannels semantics, and real channel proof.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Honor acp.defaultAgent for default channel sessions:** `src/acp/persistent-bindings.resolve.ts:460`
  - body: The quick start config sets `acp.defaultAgent`, but synthesized `defaultChannels` specs only use per-agent runtime defaults and otherwise fall back to `agentId` (`main`). `ensureConfiguredAcpBindingSession` then initializes with `spec.acpAgentId ?? spec.agentId`, so the documented config can start the wrong ACP harness.
  - confidence: 0.92
- **[P2] Abort Feishu dispatch when ACP setup fails:** `extensions/feishu/src/bot.ts:1226-1228`
  - body: After overwriting the route for a configured ACP binding, an init failure only logs and continues. That lets the message proceed on a route whose ACP session was not created instead of stopping or surfacing the configured binding failure like current configured-binding paths do.
  - confidence: 0.86
