## Summary

Keep open: current main already covers the exact-peer Feishu/plugin-channel ACP binding part, but this PR still proposes unique catch-all and `acp.defaultChannels` behavior that is not implemented on main and still has correctness, proof, and product/API review blockers.

## What This Changes

The PR extends ACP persistent bindings toward Feishu and QQ Bot, adds `acp.defaultChannels` catch-all synthesis, auto-enables the ACP backend plugin for ACP config, exports ACP route helpers, wires Feishu routing, and updates docs/tests.

## Best Possible Solution

Rebase onto current main's generic configured-binding registry and either drop the shipped exact-peer overlap or carry a maintainer-approved `defaultChannels`/catch-all design with corrected agent selection, failure semantics, docs, tests, and live channel proof.

## Reproduction Assessment

Not applicable as a user bug; this is a feature PR. The blocking review findings are source-reproducible from PR head: synthesized `defaultChannels` ignores `acp.defaultAgent`, and Feishu continues after ACP setup failure.

## Solution Assessment

No, not as-is. Current main has the cleaner generic exact-peer binding seam, while this branch still needs a product decision for catch-all/defaultChannels plus correctness fixes and live proof.
