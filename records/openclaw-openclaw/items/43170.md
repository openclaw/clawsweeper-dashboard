## Best Possible Solution

Rebase or split the work onto the current generic configured-binding architecture: keep shipped Feishu explicit binding behavior, add QQ Bot through a channel-owned provider plus inbound readiness handling, and decide defaultChannels/acpx auto-enable with tests that honor acp.defaultAgent.

## Solution Assessment
