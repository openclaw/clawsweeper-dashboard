## Summary

Keep open: current main already covers much of the Feishu/plugin-channel ACP binding direction through the newer configured-binding registry, but this branch still proposes unique `acp.defaultChannels`/catch-all behavior and has blocking correctness and proof gaps before it should merge.

## What This Changes

The branch adds Feishu/QQ ACP persistent binding resolution, `acp.defaultChannels`, Feishu handler integration, SDK exports, docs, config schema updates, and auto-enable tests.

## Best Possible Solution

Rebase onto the current generic configured-binding registry, drop superseded hard-coded channel plumbing, and pursue a focused defaultChannels/catch-all design only if maintainers want that new config surface.

## Reproduction Assessment

Not applicable for the original feature request as a runtime bug report. The review findings are source-reproducible from PR head: `acp.defaultAgent` is not used during synthesized binding initialization, and Feishu continues after an ACP init failure.

## Solution Assessment

No, not as-is. Current main has a more maintainable generic binding seam, while this branch still needs a rebase, a defaultAgent fix, Feishu failure handling aligned with main, and real behavior proof.
