## What This Changes

The PR replaces unbounded APNs relay `response.json()` parsing with a 16 MiB bounded read, fails oversized relay replies as `RelayResponseTooLarge`, and adds focused relay response tests.

## Best Possible Solution

Land the bounded APNs relay read after maintainers accept the intentional over-cap fail-closed behavior and required checks complete.

## Reproduction Assessment

Yes. Current main and `v2026.6.10` show `sendApnsRelayRequest` buffering relay replies through `response.json()`, and the PR body includes real local relay output showing the bounded path cancels an oversized body while the old path reads it fully.

## Solution Assessment

Yes, with a maintainer compatibility decision. Reusing `readResponseWithLimit` in the APNs relay request path is the narrow maintainable fix; the only remaining decision is accepting fail-closed behavior for oversized custom relay replies.
