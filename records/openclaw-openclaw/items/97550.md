## Summary

Keep open: this is a focused APNs relay hardening patch and the code direction looks correct, but the external PR still needs real behavior proof before merge. Current main still buffers the relay response with `response.json()`, while the PR moves that path to the existing bounded-response helper and fails closed on overflow.

## What This Changes

The PR changes the APNs relay send path to read relay JSON responses through a 16 MiB bounded buffer, fail closed on overflow, and updates relay response tests to use real `Response` bodies plus an oversized-body case.

## Best Possible Solution

Land the bounded-read fail-closed fix after maintainer acceptance of the oversized-response compatibility change and contributor-provided real gateway or local-relay proof.

## Reproduction Assessment

Yes by source inspection: current main calls `response.json()` on the APNs relay fetch, and a relay returning an oversized body would be buffered before parsing. I did not run a live reproduction because this review is read-only.

## Solution Assessment

Yes, this is the right boundary and a maintainable fix shape: the relay response is bounded where it is read, using the existing media-core helper and preserving small malformed-body fallback behavior. The remaining gap is proof, not a code-shape blocker.
