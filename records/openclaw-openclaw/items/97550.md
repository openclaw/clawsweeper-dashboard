## What This Changes

The PR replaces APNs relay `response.json()` parsing with a 16 MiB bounded response read and adds relay response regression tests for oversized, empty, BOM-prefixed, and normal bodies.

## Best Possible Solution

Merge the bounded relay read after maintainers accept the over-cap fail-closed behavior and required checks complete cleanly.

## Reproduction Assessment

Yes from source inspection: current main and `v2026.6.10` call `response.json()` in the APNs relay response path, so an oversized relay body is buffered before parsing. I did not run local tests because this review is read-only, but the PR body and Real behavior proof check provide loopback relay proof for the bounded path.

## Solution Assessment

Yes, with a maintainer decision: using the shared bounded response helper at the APNs relay boundary is the narrowest maintainable fix, and the follow-up commit preserves shipped BOM decoding semantics. The remaining question is whether maintainers accept the intentional fail-closed behavior for oversized custom relay replies.
