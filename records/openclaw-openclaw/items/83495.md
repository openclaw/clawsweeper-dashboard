## Summary

Keep this PR open, but the current patch should not merge as-is because the new governed-flow suppression can drop terminal child delivery when the flow explicitly opts out of auto-close and therefore never queues the replacement parent closeout.

## What This Changes

The PR adds governed managed TaskFlow finalization and parent closeout delivery logic in the task registry with regression tests.

## Best Possible Solution

Keep the governed parent closeout finalizer, but make child terminal suppression depend on the same ownership condition that guarantees a parent summary is queued or delivered, and add regression coverage for manual/disabled auto-close flows.

## Reproduction Assessment

Yes, source inspection gives a high-confidence reproduction path for the PR-introduced bug: create a governed managed flow with `autoCloseOnAllChildrenTerminal=false` or `terminalCloseoutPolicy="manual"`, terminalize a linked child, and `maybeDeliverTaskTerminalUpdate` will mark the child `not_applicable` while the parent finalizer returns before queuing a summary. I did not run tests because this review is read-only.

## Solution Assessment

No, not as currently written. The finalizer direction is narrow and useful, but the suppression predicate must be tied to actual parent-summary ownership so the patch does not create a competing message-loss path.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P1] Preserve child delivery when auto-close is disabled:** `src/tasks/task-registry.ts:1649`
  - body: `isTaskDeliveryOwnedByGovernedFlow(latest)` suppresses terminal delivery for every task linked to a governed flow, but the new finalizer deliberately returns without queuing a parent closeout when the flow state has `autoCloseOnAllChildrenTerminal === false` or `terminalCloseoutPolicy === "manual"`. In that opt-out case this line marks the child `not_applicable` and no governed parent summary is queued, so the parent loses the terminal closeout entirely. Gate suppression on the same auto-close/parent-summary ownership predicate, or let normal child delivery proceed when the governed flow is manual.
  - confidence: 0.9
