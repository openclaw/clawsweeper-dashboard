## Summary

Keep open: the narrow finalizer direction is useful, but the patch currently can drop terminal messages for manual governed flows and can leave parent closeout delivery stuck if the delivery runtime fails to load.

## What This Changes

This PR adds a governed TaskFlow finalizer in `src/tasks/task-registry.ts` and regression tests in `src/tasks/task-registry.test.ts` for terminal child closeout behavior.

## Best Possible Solution

Keep the narrow finalizer direction, but merge only after suppression is tied to an actually queued parent closeout, fallback failures remain durable, and source-branch live proof covers automatic and manual policies.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: create a governed managed flow with manual closeout opt-out, terminalize a linked child, then `maybeDeliverTaskTerminalUpdate` marks the child `not_applicable` even though no parent summary was queued.

## Solution Assessment

No. The finalizer is a plausible narrow direction, but the implementation needs to preserve manual-policy child delivery and durable fallback behavior before it is the safest solution.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.91

Full review comments:

- **[P1] Preserve child delivery for manual governed flows:** `src/tasks/task-registry.ts:1649-1653`
  - body: This suppression runs for every governed flow, but `syncManagedGovernedFlowFromTask` deliberately returns early for `autoCloseOnAllChildrenTerminal=false` and `terminalCloseoutPolicy="manual"`. In those cases no parent summary is queued, so marking the terminal child `not_applicable` means the parent receives neither the child terminal update nor a governed closeout.
  - confidence: 0.93
- **[P2] Queue fallback when delivery runtime loading fails:** `src/tasks/task-registry.ts:1286-1291`
  - body: If `loadTaskRegistryDeliveryRuntime()` rejects, this catch only logs after the flow has already been terminalized with a `parentSummaryDelivery` context key. Future finalizer attempts then skip and the child terminal update is suppressed, so the parent closeout can be lost; handle this like send failures by marking failed and queuing the parent-session fallback.
  - confidence: 0.86
