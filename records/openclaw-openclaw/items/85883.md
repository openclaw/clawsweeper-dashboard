## Summary

Keep open. Current main has the structured OpenAI Chat Completions tool-call path, but the final channel-delivery path still explicitly treats detected tool-shaped assistant text as visible text, so the reported raw JSON leak is plausible and not proven fixed.

## What This Changes

The issue reports that the openai-completions provider sends OpenClaw channel-output envelope JSON to Discord instead of unwrapping it to the intended message text.

## Best Possible Solution

Fix the OpenAI-compatible tool-call path so message-tool output arrives as structured tool execution, and add a guarded final-delivery test that prevents raw message-tool JSON from reaching channels.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path for the failure class: tool-shaped assistant text is detected and then still delivered as text. I did not establish a full live Discord/OpenAI-compatible reproduction on current main.

## Solution Assessment

Unclear; the requested provider parity is correct, but the safest fix is likely at the transport/tool-call normalization boundary rather than a broad raw-JSON unwrap fallback at channel delivery.
