## What This Changes

This PR changes sandbox env value validation to warn on long structured base64-like values only when the env var name looks credential-related, updates the skill override caller, and adjusts sanitizer tests.

## Best Possible Solution

Land a secops-reviewed sanitizer update that reduces non-sensitive long-value false positives while preserving warnings for all credential-looking explicit sandbox and skill env names, with focused regression tests and exact runtime proof.

## Reproduction Assessment

Yes, at source level. Current main has the broad value-only 80-character base64-character regex and the existing test still expects SAFE_TEXT to warn despite the non-sensitive key.

## Solution Assessment

Not yet. The direction is maintainable, but the proposed sensitive-name gate is too narrow for existing credential env names and needs security-owner agreement plus regression coverage before merge.
