## Summary

Keep open: current main has internal usage-family metadata and alias-aware sessionId selection, but the public Gateway/SDK surface still returns only canonical keys and omits the requested durable family/lineage projection. The same author has an open implementation PR at https://github.com/openclaw/openclaw/pull/79970, so this should stay open for review instead of being closed or sent to a parallel repair lane.

## What This Changes

Expose a public Gateway/SDK seam that resolves `{agentId, sessionId}` to canonical session ownership and durable family membership across reset or rotation.

## Best Possible Solution

Consolidate the narrow public resolver and family projection into the Gateway/SDK contract, then land the selected implementation under the umbrella at https://github.com/openclaw/openclaw/issues/79902 once the SQLite runtime boundary is settled.
