## Summary

Keep open: current main has internal usage-family metadata, alias-aware sessionId selection, and usage lineage rollups, but it still does not expose the requested public durable lineage/family discovery contract. The issue is now paired with an open implementation candidate at https://github.com/openclaw/openclaw/pull/79970, so the right path is PR review rather than closing or starting a separate repair lane.

## What This Changes

Request to add a public Gateway/SDK helper or projection that resolves `{agentId, sessionId}` to canonical current session ownership and session-family membership across rotations.

## Best Possible Solution

Land or supersede a narrow, documented public Gateway/SDK seam that returns explicit resolved/ambiguous/not_found outcomes plus canonical family identity and known member session IDs without exposing raw row blobs.
