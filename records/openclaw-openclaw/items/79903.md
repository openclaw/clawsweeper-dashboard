## Summary

Keep open: current main has internal usage-family tracking and a narrow `sessions.resolve`, but it does not expose the requested public durable lineage/sessionId discovery seam, and the API shape is tied to active SQLite runtime design work.

## What This Changes

Expose a stable public helper surface keyed by `{agentId, sessionId}` for canonical session resolution, family membership, predecessor/successor lookup, and live continuation across rotations.

## Best Possible Solution

Settle the SQLite runtime shape, then add a narrow documented Gateway/SDK/plugin-safe helper that returns canonical key, live continuation, and lineage IDs without exposing raw row blobs.

## Reproduction Assessment

Not applicable: this is a feature/API seam request, not a reproducible bug. Source inspection confirms related internal fields exist, but the requested public lineage helper surface is absent.

## Solution Assessment

Yes on direction, but not as an already-solved cleanup item. The maintainable path is a narrow public contract aligned with https://github.com/openclaw/openclaw/issues/79902 and https://github.com/openclaw/openclaw/pull/78595 rather than requiring companions to scan rows or parse blobs.
