## Summary

Keep open: current main has internal usage-family metadata and private alias-aware sessionId selection, plus ACP parent/child lineage metadata, but it still lacks the requested public Gateway/SDK contract for durable family discovery, predecessor/successor lookup, and explicit reusable selection semantics. The linked implementation work remains open and needs maintainer API/storage-boundary review rather than cleanup closure.

## What This Changes

The issue requests a public OpenClaw session lineage and sessionId discovery seam for canonical session ownership, rotation-family membership, live continuation selection, and ambiguity handling.

## Best Possible Solution

Keep this issue as the narrow tracker and land a small maintainer-reviewed Gateway/SDK seam that exposes canonical selection, explicit none/ambiguous/selected outcomes, and usage-family member ids without requiring raw session-row parsing.
