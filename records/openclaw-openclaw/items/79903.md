## Summary

Keep open: current main has internal usage-family metadata, private alias-aware resolution, and usage rollups, but it still lacks the requested public durable lineage and sessionId discovery contract; the related SQLite runtime/API design work is still active.

## What This Changes

The issue requests a public helper/projection surface to resolve canonical session ownership, live continuation, and family membership across sessionId rotations.

## Best Possible Solution

Add a narrow documented Gateway/SDK helper after the SQLite runtime contract settles, returning canonical key, live continuation or explicit ambiguity, and known family session IDs without exposing raw row blobs.

Not applicable: this is a feature/API seam request, not a reproducible bug. Source inspection confirms the internal lineage data and resolver policy exist, while the requested public helper surface is absent.

## Solution Assessment
