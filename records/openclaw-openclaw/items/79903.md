## Summary

Keep open: current main has internal usage-family fields and a narrow `sessions.resolve` key resolver, but it does not expose the requested durable public lineage/discovery API, and the request is tied to active SQLite runtime design work.

## What This Changes

Request to add a public helper/API, keyed by `{agentId, sessionId}`, that resolves canonical session ownership and exposes session-family lineage across rotations.

## Best Possible Solution

Settle the SQLite runtime shape, then add a narrow documented public helper that returns canonical key, live continuation, ambiguity, and known family session IDs without exposing raw row blobs.
