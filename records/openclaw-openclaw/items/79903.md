## Summary

Keep open: current main has internal usage-family metadata and an alias-aware selection helper, but it still lacks the requested public contract for family membership, live continuation, and predecessor/successor discovery. The issue is also paired with open implementation PRs, especially https://github.com/openclaw/openclaw/pull/79970, so it should remain open until maintainers review and land or reject that API shape.

## What This Changes

Request to expose a stable public Gateway/SDK seam for resolving `{agentId, sessionId}` to canonical session ownership and session-family lineage across rotations.

## Best Possible Solution

Converge the open resolver PRs into one narrow, documented Gateway/SDK contract that returns canonical key selection, explicit ambiguity, and known usage-family session IDs without exposing raw row blobs.

## Reproduction Assessment

Not applicable: this is a feature/API seam request, not a bug with a runtime reproduction path. Source inspection confirms both the internal lineage data and the missing public discovery surface.

## Solution Assessment

Yes directionally, but not solved on current main. The maintainable path is to review and land a single versioned Gateway/SDK surface from the open implementation PRs, or explicitly close those PRs if maintainers choose not to expose the seam.
