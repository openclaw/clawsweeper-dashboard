## Summary

Keep open: current main has internal lineage metadata and alias-aware sessionId selection, but it still lacks the requested public Gateway/SDK lineage discovery contract. The related implementation work is active but unmerged in https://github.com/openclaw/openclaw/pull/79970 and https://github.com/openclaw/openclaw/pull/79962, so this should stay open for maintainer API review rather than be swept closed.

## What This Changes

Expose a public sessionId-to-canonical-session and family lineage discovery seam so consumers can follow rotations without scanning raw session rows.

## Best Possible Solution

Converge the open helper/protocol work into a small versioned public Gateway/SDK contract that returns canonical ownership, explicit ambiguity, and family membership while keeping companion-specific models outside core.
