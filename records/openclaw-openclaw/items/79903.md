## Summary

Keep open: current main and the latest release have the internal lineage data, alias-aware selection policy, usage rollups, and ACP parent/child metadata, but they still do not expose the requested public durable session-family discovery contract. The issue also has active implementation candidates at https://github.com/openclaw/openclaw/pull/79970 and https://github.com/openclaw/openclaw/pull/79962, so cleanup should not close the tracking issue before maintainers decide which API surface should land.

## What This Changes

The issue requests a public Gateway/SDK seam for canonical sessionId resolution, ambiguity reporting, and durable session-family discovery across resets and rotations.

## Best Possible Solution

Review the open implementation candidates and land one narrow, documented Gateway/SDK session lineage contract that returns canonical resolution, explicit ambiguity, and known family sessionIds without forcing consumers to parse raw session rows.
