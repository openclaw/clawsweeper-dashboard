## Summary

Keep open: current main has internal usage-family metadata and alias-aware sessionId selection, but it still lacks the requested public durable lineage/family discovery contract. The issue is also actively paired with open implementation PRs, especially https://github.com/openclaw/openclaw/pull/79970 and https://github.com/openclaw/openclaw/pull/79962, so cleanup should not close it before those are reviewed or resolved.

## What This Changes

Expose a public durable session lineage and sessionId discovery surface for canonical/current session ownership and family membership across rotations.

## Best Possible Solution

Review and sequence a narrow Gateway/SDK lineage seam on top of the SQLite runtime work, exposing canonical key resolution, explicit ambiguity, live continuation, and known family session IDs without requiring consumers to parse raw session rows.
