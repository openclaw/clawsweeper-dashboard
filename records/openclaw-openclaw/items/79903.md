## Summary

Keep open: current main has internal session-lineage fields and limited sessionId-to-key resolution, but it does not expose the requested public durable lineage/discovery seam, and the request is a focused slice of the open SQLite runtime design work.

## What This Changes

Request exposing stable public helpers to resolve canonical/current session identity, live continuation, and predecessor/successor lineage for `{agentId, sessionId}` across resets or rotations.

## Best Possible Solution

Land or settle the database-backed runtime shape, then add a narrow documented helper/API that returns canonical key, live continuation, and lineage IDs without exposing raw row blobs.

## Reproduction Assessment

Not applicable: this is a feature/API seam request, not a reproducible bug. Source inspection confirms related internal fields exist, but the requested public lineage helper surface is absent.

## Solution Assessment

Yes on direction, but not as an already-solved cleanup item. The maintainable path is a narrow public contract aligned with https://github.com/openclaw/openclaw/pull/78595 and https://github.com/openclaw/openclaw/issues/79902, rather than asking companions to parse raw session rows.
