## Summary

Keep open: current main has internal lineage metadata and alias-aware session-id selection, but no merged public contract for family membership, live continuation, or predecessor/successor discovery; the open implementation PRs should be reviewed rather than using cleanup to close the issue.

## What This Changes

Expose a public Gateway/SDK seam for resolving canonical session ownership and lineage/family membership from `{agentId, sessionId}` across rotations.

## Best Possible Solution

Add one narrow, documented, additive Gateway/SDK contract that returns canonical ownership, explicit ambiguity, live continuation, and known family session ids without exposing raw session-row blobs.
