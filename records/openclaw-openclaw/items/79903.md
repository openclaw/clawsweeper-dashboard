## Summary

Keep open: current main has internal usage-family metadata and alias-aware session-id selection, but it still does not expose the requested public durable lineage/family discovery contract. The same author has an open implementation PR at https://github.com/openclaw/openclaw/pull/79970, and a related open Gateway API PR at https://github.com/openclaw/openclaw/pull/79962, so the issue should remain open until that work is reviewed, landed, or superseded.

## What This Changes

Request a public Gateway/SDK session lineage and sessionId discovery seam that resolves canonical keys, ambiguity, family membership, and predecessor/successor ids across rotations.

## Best Possible Solution

Converge the open Gateway/SDK work into one narrow, documented public contract that preserves current selection semantics and exposes family membership without forcing consumers to parse raw session rows.

## Reproduction Assessment

Not applicable: this is a feature/API seam request, not a reproducible bug. Source inspection confirms the internal data and selection policy exist while the requested public contract is still missing on current main.

## Solution Assessment

Yes directionally, but not complete on main. The maintainable path is maintainer review of the open PRs and a small versioned Gateway/SDK surface rather than closing this tracker early.
