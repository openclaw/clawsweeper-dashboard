## Summary

Keep open: current main has internal usage-family metadata, internal alias-aware selection, usage rollups, and ACP parent/child lineage metadata, but it still lacks the requested public Gateway/SDK contract for durable session-family discovery across rotations. The work is active in https://github.com/openclaw/openclaw/pull/79970 and overlaps with https://github.com/openclaw/openclaw/pull/79962, so the issue should remain open for maintainer API review rather than cleanup closure.

## What This Changes

Request to expose a public `{agentId, sessionId}` session-lineage and selection surface that returns canonical ownership, ambiguity, family membership, and predecessor/successor information across rotations.

## Best Possible Solution

Add a narrow, documented, backward-compatible Gateway/SDK contract that resolves `{agentId, sessionId}` to canonical selection plus explicit ambiguity and family membership without making consumers parse raw session rows.

## Reproduction Assessment

Not applicable: this is a feature/API seam request, not a reproducible bug. Source inspection confirms the internal data and resolver policy exist while the requested public helper/projection remains absent.

## Solution Assessment

Unclear until maintainer API review completes. The requested direction is maintainable, but the safest solution is to choose one small versioned Gateway/SDK shape across the open related PRs rather than merge competing partial seams.
