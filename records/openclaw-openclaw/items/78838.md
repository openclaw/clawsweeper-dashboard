## Summary

Close: current main and the shipped provider path already address the reported Gemini post-tool empty final text by routing direct Google/Vertex Gemini reasoning to native mode instead of tagged text mode.

## What This Changes

Issue reports Gemini final assistant text becoming empty in `assistant_message_end` after a tool call when the response is longer than roughly 150-200 tokens, leading to embedded run timeout.

## Best Possible Solution

Keep the provider-owned native reasoning override for direct Google/Vertex Gemini and close the report as fixed in the shipped provider implementation.

## Reproduction Assessment

Yes from source and merged PR proof: the old direct Gemini tagged-reasoning conflict produced empty post-tool turns, and the merged fix includes a live `google/gemini-2.5-flash` tool-call run returning final text. I did not run a new paid live Gemini request during this read-only review.

## Solution Assessment

Yes. The narrow provider-owned fix is to use native reasoning mode for direct Google/Vertex Gemini while keeping Gemini CLI tagged, which matches the different runtime output contracts.
