## Summary

Keep open: the Feishu localization work is still useful and not implemented on current main, but the current branch would regress workspace-only file-read enforcement in Feishu docx uploads.

## What This Changes

The PR localizes Feishu bot, wiki, bitable, media, send-result, and docx error messages to Chinese, adds contextual IDs/URLs/tokens to failures, and updates Feishu tests.

## Best Possible Solution

Rebase the localization onto current main while preserving the `[System: ...]` permission-notice contract and the docx `loadWebMedia(..., { localRoots })` sandbox path, then add focused regression coverage for localized copy and workspace-only uploads.

## Reproduction Assessment

Yes. Static reproduction is high-confidence: the PR diff reads local docx upload paths with `fs.readFile`, while current main routes those paths through `loadWebMedia` with `localRoots` and has tests asserting that behavior.

## Solution Assessment

No. The localization direction is reasonable, but the current patch is not the best merge path because it bypasses active workspace-only enforcement; a safer fix keeps the current file-read flow and changes only the approved error copy around it.
