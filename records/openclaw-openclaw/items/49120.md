## Summary

Keep open. The PR is a focused security hardening change that is not obsolete on current main, but it still needs maintainer/security policy reconciliation with the private-network opt-in direction and a test contract update before merge; the external PR also lacks real after-fix behavior proof.

## What This Changes

The branch imports the shared SSRF host/IP classifier into cron webhook URL normalization and adds unit tests that reject private/internal webhook targets while accepting public HTTP(S) URLs.

## Best Possible Solution

Choose the cron webhook SSRF posture explicitly, then land either create/patch-time rejection with updated runtime coverage or a documented operator opt-in while keeping the runtime DNS-pinned guard in place.

## Reproduction Assessment

Yes. Source inspection shows the PR guard would reject the 127.0.0.1 URL in src/gateway/server-cron.test.ts during state.cron.add, before the existing runtime fetch assertion can run.

## Solution Assessment

No, not as submitted. Reusing the shared SSRF classifier is the narrow hardening mechanism, but the PR needs the runtime test contract and the related private-network policy decision reconciled first.
