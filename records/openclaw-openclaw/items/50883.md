## Summary

Keep open: current main still lacks `openclaw sessions --kind`, and the PR is useful but not merge-ready because the normal route-first CLI path would drop the flag, the new parser rejects displayed `cron` rows, and the external real-behavior proof gate is still unmet.

## What This Changes

The PR adds a repeatable/comma-separated `--kind` filter to `openclaw sessions`, with a parser helper, command wiring, JSON/text output metadata, and focused tests.

## Best Possible Solution

Land a repaired version that shares one canonical session-kind contract, parses repeated/comma-separated `--kind` in both route-first and Commander paths, filters before limiting, and includes real CLI proof.

## Reproduction Assessment

Yes. Source inspection shows the default route-first sessions parser has no `kind` field while the routed command uses it, and current classifiers/types include `cron` even though the PR helper excludes it.

## Solution Assessment

No. The helper direction is reasonable, but the patch needs route-first parsing, the canonical kind list including `cron`, and real behavior proof before it is the best merge form.
