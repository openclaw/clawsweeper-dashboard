## Best Possible Solution

Land a rebased sessions kind filter that is wired through both route-first and Commander paths, derives valid choices from the canonical classifier, filters before limit selection, updates docs/tests, and includes redacted real CLI proof.

## Reproduction Assessment

Yes, for the PR defects by source inspection: normal sessions CLI routing goes through `parseSessionsRouteArgs`, which does not return `kind`, and the current classifier exposes `cron` and `spawn-child` beyond the PR helper's hardcoded list.

## Solution Assessment

No, not as currently implemented. The maintainable path is a rebased filter that shares the existing classifier contract, is parsed by both CLI paths, and filters before the current default/newest limit is applied.
