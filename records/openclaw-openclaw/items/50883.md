## Summary

Keep open: the feature is useful, but this PR is not ready to merge because it misses the route-first CLI path, omits the current `cron` session kind, and lacks real behavior proof from an external contributor.

## What This Changes

The PR adds a repeatable/comma-separated `--kind` option for `openclaw sessions`, a parsing helper, command filtering, JSON/text output metadata, and focused tests.

## Best Possible Solution

Land a repaired implementation that shares one canonical session-kind contract including `cron`, parses `--kind` in both route-first and Commander paths, filters before limit selection, updates focused tests/docs, and includes real CLI proof.

## Reproduction Assessment

Yes. Source inspection is enough for the review findings: current main's route-first sessions parser forwards no `kind`, while current sessions/status contracts include `cron`.

## Solution Assessment

No. The helper direction is reasonable, but the proposed patch is not the best merge form until it covers route-first parsing, includes `cron`, stays compatible with current limit handling, and provides real behavior proof.
