## Summary

Keep open. The `--kind` filter is a useful CLI feature that current main still lacks, but this branch is not merge-ready because it only wires the Commander-visible sessions command, not the route-first parser used by normal sessions invocations, and it hardcodes an obsolete kind list after merged classifier work added `cron` and `spawn-child`. The PR also still lacks required real behavior proof.

## What This Changes

The PR adds a repeatable/comma-separated `openclaw sessions --kind` filter with a parser helper, CLI option wiring, command filtering, JSON metadata, and focused tests.

## Best Possible Solution

Land a repaired implementation that parses `--kind` in both route-first and Commander paths, reuses the canonical `src/sessions/classify-session-kind.ts` kind contract, filters before limit selection, updates focused tests/docs as needed, and includes real CLI proof.

## Reproduction Assessment

Yes. Source inspection reproduces the review findings: `parseSessionsRouteArgs` does not parse `--kind`, routed sessions pass those parsed args directly to `sessionsCommand`, and current `SessionKind` includes `cron` and `spawn-child`.

## Solution Assessment

No. The parser-helper direction is reasonable, but this PR is not the best merge form until it covers the route-first parser, reuses the canonical kind contract, and integrates with the current limit flow.
