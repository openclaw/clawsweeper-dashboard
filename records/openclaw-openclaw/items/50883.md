The PR adds a repeatable/comma-separated `openclaw sessions --kind` filter, a parser helper, sessions output metadata, and focused tests.

## Best Possible Solution

Land a repaired sessions kind filter that is parsed by both CLI paths, derives allowed values from the canonical session classifier, filters before limit selection, updates focused tests/docs, and includes real CLI proof.

## Reproduction Assessment

Yes, for the PR defects. Source inspection shows routed `sessions` uses `parseSessionsRouteArgs` and forwards that result, while the PR only adds `kind` to the Commander path; current `main` also emits `cron` and `spawn-child` kinds.

## Solution Assessment

No. The parser-helper direction is reasonable, but this patch should not merge until it covers route-first parsing, reuses the canonical `SessionKind` contract, and preserves current limit flow.
