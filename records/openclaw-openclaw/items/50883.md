## Summary

Keep open: the feature is useful, but the patch is not mergeable as-is because normal route-first `openclaw sessions` invocations would not receive the new filter, the helper omits current session kinds, and the external PR still lacks real behavior proof.

## What This Changes

The PR adds a repeatable/comma-separated `openclaw sessions --kind` CLI filter, a session-kind parser helper, sessions command filtering/JSON metadata, and focused tests.

## Best Possible Solution

Land a repaired sessions kind filter that is parsed by both CLI paths, reuses canonical `SessionKind` values, filters before selecting limited rows, updates focused tests/docs, and includes real CLI proof.

## Reproduction Assessment

Yes. Source inspection is enough: route-first `sessions` uses `parseSessionsRouteArgs` and forwards those args, while the PR only adds `kind` to the Commander path; current main also emits `cron` and `spawn-child` kinds.

## Solution Assessment

No. The parser-helper direction is reasonable, but this patch should not merge until it covers route-first parsing, derives allowed values from the canonical classifier contract, and fits the current limit flow.
