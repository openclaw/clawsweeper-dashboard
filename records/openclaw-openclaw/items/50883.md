## Summary

Keep open: the feature is useful, but the proposed patch is not safe to merge because normal route-first `openclaw sessions` runs would drop `--kind`, the new helper has a stale session-kind list, and the external PR still lacks real behavior proof.

## What This Changes

The PR adds a repeatable/comma-separated `openclaw sessions --kind` filter with a parser helper, CLI option wiring, output metadata, and focused tests.

## Best Possible Solution

Land a repaired implementation that reuses the canonical `SessionKind` contract, parses `--kind` in both Commander and route-first paths, filters before limit selection, updates focused tests/docs as needed, and includes real CLI proof.

## Reproduction Assessment

Yes. Source inspection is enough for the review findings: current main's route-first sessions parser forwards no `kind`, and the current session/status kind contract includes `cron` and `spawn-child`.

## Solution Assessment

No. The parser-helper direction is reasonable, but this PR is not the best merge form until it covers route-first parsing, uses the canonical kind contract, integrates with current limit handling, and provides real behavior proof.
