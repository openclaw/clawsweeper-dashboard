## Summary

Keep open: current main still lacks a sessions kind filter, so the PR is not obsolete, but the proposed patch is not mergeable as-is because it misses the route-first CLI path, excludes the current `cron` session kind, and lacks real behavior proof.

## What This Changes

The branch adds a `sessions-kind` parser/test, filters `sessionsCommand` output by kind, and wires a Commander `--kind` option with tests.

## Best Possible Solution

Land a repaired version that shares one canonical session-kind contract, parses repeated/comma-separated `--kind` in both route-first and Commander paths, filters before limiting, and includes real CLI proof.

## Reproduction Assessment

Yes. Source inspection shows the PR only forwards `--kind` through Commander while normal `openclaw sessions` uses the route-first parser, and the PR helper rejects the currently displayed `cron` rows.

## Solution Assessment

No. The parser/filter helper direction is reasonable, but the patch needs route-first parsing, the canonical kind list including `cron`, and real behavior proof before it is the best merge form.
