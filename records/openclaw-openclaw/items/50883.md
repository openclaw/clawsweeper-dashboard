## What This Changes

## Best Possible Solution

Land a repaired version that uses one typed session-kind contract across classification, route-first parsing, Commander, text/JSON filtering, tests, docs, and changelog.

Yes. Source inspection gives a high-confidence path: exact `openclaw sessions` invocations use route-first parsing, and the proposed diff adds only Commander forwarding for `--kind`.

## Solution Assessment

No. The parser/filter helper direction is reasonable, but the patch is not the best merge form until route-first parsing, the current `cron` kind, docs, and changelog share the same contract.
