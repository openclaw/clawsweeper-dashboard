## Summary

Keep open: the feature is useful and not already on main, but the PR still has blocking correctness defects in the normal route-first CLI path and in the session-kind contract, plus missing real behavior proof.

## What This Changes

The PR adds a repeatable/comma-separated `openclaw sessions --kind` option, a parser helper, sessions output metadata, and focused tests.

## Best Possible Solution

Land a repaired sessions kind filter that is parsed by both route-first and Commander paths, derives allowed values from the shared classifier, filters before limit selection, updates focused tests/docs, and includes real CLI proof.

## Reproduction Assessment

Yes, for the PR defects. Source inspection shows routed `sessions` uses `parseSessionsRouteArgs` and forwards that result, while the PR only adds `kind` to the Commander path; current main also emits `cron` and `spawn-child` kinds that the helper omits.

## Solution Assessment

No. The parser-helper direction is reasonable, but this patch should not merge until it covers route-first parsing, reuses the canonical `SessionKind` contract, and preserves current limit flow.
