## Summary

Keep open: the feature is still useful, but the current PR branch is not mergeable as-is because normal route-first `openclaw sessions` runs would drop `--kind`, the allowed-kind contract omits current displayed kinds, and the external PR lacks after-fix real behavior proof.

## What This Changes

The PR adds a repeatable `openclaw sessions --kind` filter, a parsing helper, JSON/text filter metadata, Commander wiring, and focused tests.

## Best Possible Solution

Land a repaired version that uses one canonical session-kind contract, parses repeated/comma-separated `--kind` through both route-first and Commander paths, filters before limiting, and covers text/JSON behavior plus route parser tests.

## Reproduction Assessment

Yes. Source inspection shows the proposed `--kind` option is only forwarded by Commander, while normal exact `openclaw sessions` invocations use `parseSessionsRouteArgs`; the helper also rejects currently displayed `cron` rows.

## Solution Assessment

No. The parser/filter helper direction is reasonable, but the PR is not the best merge form until route-first parsing and the canonical kind list are aligned, and after-fix real behavior proof is added.
