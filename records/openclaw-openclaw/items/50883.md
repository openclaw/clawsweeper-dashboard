## Summary

Keep open: the feature is useful, but this branch is not merge-ready because the new filter is only wired through the Commander path while normal route-first `sessions` invocations would drop it, and the helper hardcodes an outdated subset of current session kinds.

## What This Changes

The PR adds a repeatable/comma-separated `openclaw sessions --kind` filter, a parser helper, sessions output metadata, and focused tests.

## Best Possible Solution

Land a repaired sessions kind filter that is parsed by both route-first and Commander paths, derives allowed values from the shared classifier, preserves current limit semantics, updates focused tests/docs, and includes real CLI proof.

## Reproduction Assessment

Yes, for the PR defects by source inspection. Normal CLI runs use route-first parsing, and the PR does not add `kind` to `parseSessionsRouteArgs`; current main also exposes `cron` and `spawn-child` session kinds that the helper omits.

## Solution Assessment

No. The parser-helper direction is reasonable, but this patch should not merge until it covers route-first parsing, reuses the canonical `SessionKind` contract, and preserves current limit flow.
