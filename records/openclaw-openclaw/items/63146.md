## What This Changes

The branch sorts memory corpus supplement registrations by `pluginId` and adds two unit tests for out-of-order and repeated registration.

## Best Possible Solution

Land deterministic `pluginId` ordering with the focused regression tests after contributor proof is added and maintainers refresh changed-surface validation.

## Reproduction Assessment

Yes, source-reproducible: on current main, registering `plugin-z` before `plugin-a` would return insertion order because `listMemoryCorpusSupplements()` only copies the backing array. The memory get path then consumes the list sequentially and returns the first hit.

## Solution Assessment

Yes. Sorting returned corpus supplement registrations by `pluginId` is the narrowest maintainable fix and matches the existing prompt supplement ordering pattern without changing the plugin API.
