## What This Changes

The PR sorts `listMemoryCorpusSupplements()` results by `pluginId` and adds two unit tests for out-of-order registration and same-plugin re-registration.

## Best Possible Solution

Land the focused deterministic `pluginId` ordering and regression tests after contributor proof and maintainer validation, unless maintainers choose to define a richer explicit corpus supplement priority contract.

## Solution Assessment

Yes. Sorting returned corpus supplement registrations by `pluginId` is the narrowest maintainable fix and matches the adjacent prompt supplement pattern; a priority field is only preferable if maintainers want an explicit conflict-resolution policy.
