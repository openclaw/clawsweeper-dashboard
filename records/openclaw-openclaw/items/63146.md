The PR sorts `listMemoryCorpusSupplements()` by `pluginId` and adds two unit tests for deterministic corpus supplement ordering after out-of-order registration and re-registration.

## Best Possible Solution

## Reproduction Assessment

Yes from source inspection. Current main returns corpus supplements in registration order, and `getMemoryCorpusSupplementResult()` returns the first non-null supplement result, so the proposed ordering tests would fail before the patch.

## Solution Assessment

Yes, with maintainer acceptance of the compatibility risk. Sorting by `pluginId` is narrow, matches the adjacent prompt supplement pattern, and avoids adding a config knob unless insertion order is intentionally part of the contract.
