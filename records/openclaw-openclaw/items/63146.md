## Summary

Keep open: current main still returns corpus supplements in registration order, while this PR is a small, correct-looking deterministic ordering fix that matches the adjacent prompt supplement pattern; the remaining blocker is contributor real behavior proof plus refreshed validation.

## What This Changes

The branch sorts `listMemoryCorpusSupplements()` results by `pluginId` and adds two regression tests for out-of-order registration and same-plugin re-registration.

## Best Possible Solution

Land the focused deterministic `pluginId` ordering and regression tests after contributor proof and refreshed validation, unless maintainers decide to add an explicit corpus supplement priority contract.

## Reproduction Assessment

Yes. Source inspection on current main shows registration appends corpus supplements and listing returns that array unsorted, so registering `plugin-z` before `plugin-a` would reproduce insertion-order output without needing a full app run.

## Solution Assessment

Yes. Sorting returned corpus supplement registrations by `pluginId` is the narrowest maintainable fix and matches the existing prompt supplement deterministic ordering pattern; a priority field would only be needed for a broader policy change.
