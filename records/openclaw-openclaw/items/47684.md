## Summary

Keep open. Current main and v2026.5.22 still lack a bulk auth rotation/sync command or config-set/status warning for stale per-agent auth profiles shadowing env-backed provider keys; the closed explicit-config-key fix and the open active-profile migration issue are adjacent but do not fully own this exact credential-rotation UX.

## What This Changes

Request a bulk auth sync, auto-sync, or shadow-warning path for rotating provider API keys across per-agent auth profile stores.

## Best Possible Solution

Define an owner-reviewed explicit auth rotation path with dry-run/reporting and shadow warnings, using existing auth-store locks and preserving explicit credential precedence; avoid automatic 401 fallback unless maintainers approve that security policy.

## Reproduction Assessment

Yes, at source level. Current main has per-agent auth commands and profile-first provider resolution by default, while no bulk sync/migration command or config-set shadow warning exists for rotated env keys.

## Solution Assessment

Unclear as proposed. An explicit sync/migration command plus warnings looks safer than automatic cross-agent rewrites or automatic 401 fallback, but maintainers need to choose the credential policy before implementation.
