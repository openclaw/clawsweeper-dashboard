## Summary

Keep open. Current main and v2026.5.22 still lack a bulk auth rotation/sync command or config-set shadow warning for inline env keys, and the implemented provider-apiKey precedence fix does not cover stale per-agent auth profiles shadowing `env.ANTHROPIC_API_KEY`.

## What This Changes

Request adding a bulk auth sync/rotation command, auto-sync path, warning, or fallback so API key rotation does not leave per-agent auth profiles stale.

## Best Possible Solution

Define an owner-reviewed explicit auth rotation path with dry-run/reporting and audit/status warnings, using existing auth-store locks and preserving explicit credential precedence; avoid implicit 401 fallback unless maintainers approve that security policy.

## Reproduction Assessment

Yes, at source level. Current main has per-agent auth commands and profile-first provider resolution by default, while no bulk sync/migration command or config-set shadow warning exists for rotated env keys.

## Solution Assessment

Unclear as proposed. An explicit sync/migration command plus warnings looks safer than automatic cross-agent rewrites or automatic 401 fallback, but maintainers need to choose the credential policy before implementation.
