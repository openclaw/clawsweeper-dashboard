## Summary

Keep open. Current main still has a profile-first provider auth path, `REF_SHADOWED` is only surfaced through `openclaw secrets audit`, and the related reconciliation PR was closed unmerged, so the requested rotation/sync UX is not implemented yet.

## What This Changes

Request to add an auth sync command, auto-sync, fallback, or warning so API key rotation in config/env does not leave stale per-agent `auth-profiles.json` credentials in use.

## Best Possible Solution

Define a non-destructive rotation path: likely an explicit, dry-run-capable sync or repair command plus `config set`/`models status` shadowing warnings, while avoiding silent secret copying or fallback behavior unless maintainers explicitly approve that precedence change.

## Reproduction Assessment

Yes. A source-level reproduction is clear: with an Anthropic auth profile present and `ANTHROPIC_API_KEY` supplied through config/env, `createProviderAuthResolver` returns the stored profile before checking the env/config key.

## Solution Assessment

Unclear. The issue lists several possible fixes, but the safest maintainable direction needs maintainer choice because bulk credential sync and fallback-on-401 both affect secret handling and per-agent auth semantics.
