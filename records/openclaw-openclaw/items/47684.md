## Summary

Keep open. Current main still implements profile-first provider auth, only reports shadowing through secrets audit, and has no bulk auth sync or rotation command; the related reconciliation PR was closed unmerged, so the requested UX remains unsolved.

## What This Changes

Request to add an auth sync, auto-sync, warning, or fallback path so API key rotation propagates across agent auth profiles.

## Best Possible Solution

Ship an explicit, non-destructive rotation path, likely dry-run-capable auth sync or doctor/secrets repair plus clear shadow warnings, while preserving documented profile precedence unless maintainers intentionally change it.

## Reproduction Assessment

Yes, at source level. With a matching Anthropic auth profile present and `ANTHROPIC_API_KEY` supplied through env/config, `createProviderAuthResolver` returns the stored profile before it checks env or config-backed auth.

## Solution Assessment

Unclear. The narrowest maintainable direction is probably an explicit sync or repair flow with warnings, but maintainers should choose the credential-rotation policy before code changes.
