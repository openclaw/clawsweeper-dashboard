## Summary

Keep open: current main still has profile-first auth precedence and only exposes audit/per-agent configuration paths, so the requested bulk rotation/sync or inline shadowing warning remains a valid enhancement that needs maintainer UX/security direction.

## What This Changes

Request to add a user-friendly way to rotate provider API keys across per-agent auth profiles or warn when env/config credentials are shadowed.

## Best Possible Solution

Design an explicit auth rotation path or targeted shadowing warning that preserves per-agent credential ownership, updates docs/changelog, and keeps existing audit signals intact.

## Reproduction Assessment

Yes for the reported precedence behavior: source inspection shows stored auth profiles win before env/config fallback, and the docs already describe that as `REF_SHADOWED`. I did not run a live gateway/config-set flow in this read-only review.

## Solution Assessment

Unclear: the issue lists several possible solutions, and the best fix needs maintainer choice because bulk sync or fallback could overwrite or bypass intentionally per-agent credentials. A warning or explicit sync command is safer than silent automatic mutation until that policy is decided.
