## Summary

Keep open. Current main and v2026.5.22 still lack a bulk auth sync/migrate command, config-set shadowing warning, or automatic cross-agent credential rotation path; related fixes cover explicit configured API-key precedence and missing non-main auth inheritance, but not this env/ref rotation UX.

## What This Changes

Requests a bulk or automatic credential-sync path so API key rotation propagates across per-agent auth profile stores or at least warns when stored profiles shadow config/env refs.

## Best Possible Solution

Define an owner-reviewed explicit auth rotation path with dry-run/reporting and shadow warnings, preserving explicit credential precedence unless maintainers approve a fallback or automatic rewrite policy.

## Reproduction Assessment

Yes, at source level for the missing capability. Current main has per-agent auth commands, profile-first provider resolution by default, and audit-only shadowing diagnostics, while no bulk sync/migration command or config-set/status shadow warning exists for rotated env keys.

## Solution Assessment

Unclear as proposed. An explicit sync or migration command plus warnings looks safer than automatic cross-agent rewrites or automatic 401 fallback, but maintainers need to choose the credential policy before implementation.
