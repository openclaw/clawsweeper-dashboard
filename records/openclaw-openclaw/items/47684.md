## Summary

Keep open. Current main and the latest release still expose per-agent auth profile writes, profile-first provider resolution by default, and audit-only shadowing diagnostics, while no bulk auth sync/rotation command or config-set/status warning owns this credential-rotation UX.

## What This Changes

The issue requests a command, automatic sync, warning, or fallback path for rotating provider API keys across per-agent auth profile stores without manually editing each agent file.

## Best Possible Solution

Define an owner-reviewed explicit auth rotation path with dry-run/reporting and shadow warnings, using existing auth-store locks and preserving explicit credential precedence; avoid automatic 401 fallback unless maintainers approve that security policy.

## Reproduction Assessment

Yes, at source level. Current main has per-agent auth commands and profile-first provider resolution by default, while no bulk sync/migration command or config-set/status shadow warning exists for rotated env keys.

## Solution Assessment

Unclear as proposed. An explicit sync or migration command plus warnings looks safer than automatic cross-agent rewrites or automatic 401 fallback, but maintainers need to choose the credential policy before implementation.
