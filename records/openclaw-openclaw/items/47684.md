## Summary

Keep open. Current main and v2026.5.22 still expose per-agent auth profile writes, per-agent auth-order controls, and audit-only REF_SHADOWED diagnostics, but no bulk auth sync/migrate command, config-set warning, models-status warning, or owner-approved fallback policy for rotated env/config keys.

## What This Changes

The issue requests a CLI command, auto-sync, warning, or fallback path for rotating provider API keys across per-agent auth profile stores.

## Best Possible Solution

Define an owner-reviewed explicit auth rotation path with dry-run/reporting and shadow warnings, preserving stored credential precedence unless maintainers approve a fallback policy.

## Reproduction Assessment

Yes, at source level for the missing capability. Current main has per-agent auth commands, profile-first provider resolution by default, and audit-only shadowing diagnostics, while no bulk sync/migration command or config-set/status shadow warning exists for rotated env keys.

## Solution Assessment

Unclear as proposed. An explicit sync or migration command plus warnings looks safer than automatic cross-agent rewrites or automatic 401 fallback, but maintainers need to choose the credential policy before implementation.
