## Summary

Keep open. Current main still lets stored auth profiles shadow env/config-backed provider auth, `REF_SHADOWED` remains audit-only, and the related reconciliation PR was closed unmerged.

## What This Changes

The issue requests an auth key rotation or synchronization path so config-updated API keys do not remain shadowed by stale per-agent auth profiles.

## Best Possible Solution

Ship an explicit, non-destructive rotation path, likely a dry-run-capable auth sync or repair flow plus shadow warnings, while preserving documented profile precedence unless maintainers intentionally change it.

## Reproduction Assessment

Yes, at source level. With an Anthropic auth profile present and `ANTHROPIC_API_KEY` supplied through env/config, `createProviderAuthResolver` returns the profile credential before checking env or config-backed provider auth.

## Solution Assessment

Unclear. The issue lists several viable approaches, but bulk syncing or fallback-on-401 would change secret handling and per-agent isolation semantics, so maintainer policy should lead the implementation.
