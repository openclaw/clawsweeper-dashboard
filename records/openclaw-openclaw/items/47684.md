## Summary

Keep open. Current main still treats stored auth profiles as higher precedence than env/config-backed provider auth, `REF_SHADOWED` remains audit-only, and the related reconciliation PR was closed unmerged, so the requested rotation/sync UX is still missing.

## What This Changes

Request an auth rotation/sync workflow, warning, or fallback so API key changes are not hidden by stale per-agent auth profiles.

## Best Possible Solution

Ship a maintainer-approved, non-destructive rotation path such as a dry-run-capable sync or repair command plus shadow warnings, while preserving documented profile precedence unless that policy is intentionally changed.

## Reproduction Assessment

Yes, at source level. With a stored Anthropic auth profile and `ANTHROPIC_API_KEY` supplied through env/config, the resolver returns the stored profile before it checks env or config-backed auth; I did not live-rotate a real credential because this review is read-only.

## Solution Assessment

Unclear. The issue lists several plausible approaches, but bulk sync and fallback-on-401 affect secret propagation and auth precedence, so maintainers should choose the policy before implementation.
