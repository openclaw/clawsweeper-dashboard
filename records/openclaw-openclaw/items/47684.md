## Summary

Keep open. Current main and the latest release still have per-agent auth profile management, REF_SHADOWED diagnostics, and SecretRef migration tooling, but no bulk API-key rotation/sync surface for stale per-agent auth profiles; the closest related work is an open, broader active-profile migration issue rather than a merged fix for this request.

## What This Changes

The issue requests a safer way to rotate provider API keys across all per-agent auth profile stores, either through sync automation, a CLI command, shadowing warnings, or fallback behavior.

## Best Possible Solution

Define an owner-reviewed auth rotation path that keeps credential precedence explicit, likely an explicit sync/migration command plus status/audit warnings, using existing auth-store locks and SecretRef-aware rules rather than hidden 401 fallback.

## Reproduction Assessment

Yes, at source level. Current main has per-agent auth commands and auth-profile-first provider resolution, while no bulk sync/migrate command or `config set` shadow warning exists for rotated provider API keys.

## Solution Assessment

Unclear as proposed. An explicit sync/migration command plus warnings looks safer than automatic cross-agent rewrites or automatic 401 fallback, but maintainers need to choose the credential policy before implementation.
