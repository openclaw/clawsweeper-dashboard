## Summary

Keep open. Current main and the latest release still lack a bulk auth sync, migrate, or shadowing-warning command; the closed explicit-config-key bug is fixed, and https://github.com/openclaw/openclaw/issues/61709 is related, but neither fully owns this env/ref credential-rotation UX.

## What This Changes

The issue asks for a user-facing way to sync, migrate, warn about, or fall back from rotated provider API keys across per-agent auth profile stores.

## Best Possible Solution

Define an owner-reviewed explicit auth rotation path with dry-run/reporting and shadow warnings, using existing auth-store locks and preserving explicit credential precedence unless maintainers approve a fallback policy.

## Reproduction Assessment

Yes, at source level for the missing capability. Current main has per-agent auth commands, profile-first provider resolution by default, and audit-only shadowing diagnostics, while no bulk sync/migration command or config-set/status shadow warning exists for rotated env keys.

## Solution Assessment

Unclear as proposed. An explicit sync or migration command plus warnings looks safer than automatic cross-agent rewrites or automatic 401 fallback, but maintainers need to choose the credential policy before implementation.
