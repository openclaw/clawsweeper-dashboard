## Summary

Keep open. Current main still leaves the env/config-backed key-rotation case unresolved: stored auth profiles can win before env/config-backed auth, `REF_SHADOWED` is audit-only, and the related reconciliation PR was closed unmerged.

## What This Changes

The issue requests a bulk auth sync, auto-sync, shadowing warning, or fallback behavior so rotated provider API keys do not remain shadowed by stale per-agent auth profiles.

## Best Possible Solution

Ship a maintainer-approved, non-destructive rotation path, likely a dry-run-capable sync/repair command plus clearer shadow warnings, while preserving documented profile precedence unless that policy intentionally changes.

## Reproduction Assessment

Yes, at source level. With a stored provider auth profile and a rotated `ANTHROPIC_API_KEY` available through env/config, current auth resolution can return the stored profile before env/config-backed auth; I did not run a live credential rotation because this review is read-only.

## Solution Assessment

Unclear. The issue lists several plausible fixes, but bulk sync and fallback-on-401 change credential propagation and auth precedence, so maintainers should choose the policy before implementation.
