## Security Review

Status: needs_attention

Summary: Credential rotation is security-sensitive because it changes where API keys are stored and which credential source wins at runtime.

Concerns:

- **[medium] Stale stored credentials can shadow rotated env credentials:** `src/agents/model-auth.ts:677`
  - body: Normal runtime resolution can select an existing auth-profile credential before the updated env/config credential, so users may believe rotation succeeded while OpenClaw still tries the old stored key.
  - confidence: 0.9
- **[medium] Bulk sync could expand secret copies without policy:**
  - body: A sync or auto-sync feature could write credential material across multiple agent stores unless maintainers require a narrow, dry-run-capable, ref-aware, or otherwise non-destructive policy.
  - confidence: 0.82
