## Summary

Keep open. Current main has partial adjacent fixes for explicit provider-config precedence, inherited main auth, and diagnostics, but it still lacks the requested bulk/env key-rotation command or automatic all-agent sync, and the remaining auth-precedence behavior needs maintainer/security product design.

## What This Changes

The issue requests a CLI command, automatic sync, or warning so rotated provider API keys do not remain shadowed by per-agent stored auth profiles.

## Best Possible Solution

Keep the current explicit-provider-config precedence and inherited-main fallback behavior, then design an explicit, audited SQLite-backed bulk auth update or doctor/status warning path; avoid unconditional fallback-on-401 unless maintainers explicitly choose that security tradeoff.

## Reproduction Assessment

Yes at source level: real stored profiles are still evaluated before env fallback, and tests cover that a real stored profile stays ahead of env auth. I did not run a live key-rotation flow because the review is read-only and has no provider credentials.

## Solution Assessment

Unclear. A sync command or warning path is plausible, but the best fix must be chosen by maintainers because automatic secret propagation, auth precedence, and fallback-on-401 have different security and upgrade tradeoffs.
