## Summary

Keep open. Current main and the latest release still lack a bulk auth sync/migrate command or a config/status shadowing warning for env/ref key rotation across agent auth stores; https://github.com/openclaw/openclaw/issues/61709 is related but does not fully own this env/ref credential-rotation UX.

## What This Changes

The issue requests a user-facing way to rotate provider API keys across multiple per-agent auth profile stores, either through sync, auto-sync, shadow warnings, or fallback behavior.

## Best Possible Solution

Define an owner-reviewed explicit auth rotation path with dry-run/reporting and shadow warnings, using existing auth-store locks and preserving profile precedence unless maintainers approve fallback behavior.

## Reproduction Assessment

Yes, source-level. Current main has per-agent auth commands, profile-first provider resolution by default, and audit-only shadowing diagnostics, while no bulk sync/migration command or config-set/status shadow warning exists for rotated env keys.

## Solution Assessment

Unclear as proposed. An explicit sync or migration command with warnings looks safer than automatic cross-agent rewrites or automatic 401 fallback, but maintainers need to choose the credential policy before implementation.
