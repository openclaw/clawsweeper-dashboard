## Summary

Keep open. Current main and the latest release still lack a bulk auth sync or rotation command and keep shadowing diagnostics as audit-only; the explicit-config-key bug in https://github.com/openclaw/openclaw/issues/40363 is fixed, and https://github.com/openclaw/openclaw/issues/61709 is related, but neither fully owns this env/ref credential-rotation UX.

## What This Changes

The issue requests a supported way to rotate or synchronize provider credentials across per-agent auth profile stores so config/env key changes do not leave agents using stale credentials.

## Best Possible Solution

Define an owner-reviewed explicit auth rotation path with dry-run/reporting and shadow warnings, using existing auth-store locks and preserving explicit credential precedence; avoid automatic 401 fallback unless maintainers approve that security policy.

## Reproduction Assessment

Yes, at source level. Current main has per-agent auth commands, profile-first provider resolution by default, and audit-only shadowing diagnostics, while no bulk sync/migration command or config-set/status shadow warning exists for rotated env keys.

## Solution Assessment

Unclear as proposed. An explicit sync or migration command plus warnings looks safer than automatic cross-agent rewrites or automatic 401 fallback, but maintainers need to choose the credential policy before implementation.
