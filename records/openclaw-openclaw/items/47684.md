- `impact:security`: The issue concerns API key rotation, stale stored credentials, and whether OpenClaw should copy or fall back between secret sources.

## Summary

Keep open: current main and v2026.5.20 still lack a bulk auth sync, auto-sync, or shadow-warning path for env-backed API key rotation across agent auth profiles. The closed explicit `models.providers.*.apiKey` rotation issue and the newer active-profile migration request are adjacent, but neither fully owns this env/preference-shadowing workflow decision.

## What This Changes

Request a user-facing auth sync, auto-sync, warning, or fallback flow so env-backed Anthropic API key rotation is reflected across per-agent auth profiles or clearly reported.

## Best Possible Solution

Define an explicit, non-destructive credential-rotation policy, likely a dry-run-capable sync or stronger shadow-warning flow, while preserving documented profile precedence unless security owners intentionally change it.

## Reproduction Assessment

Yes, at source level. Current main persists `config set env.*` changes to config only, while default provider auth resolution still selects stored auth profiles before env/config-aware keys unless a caller explicitly opts into env-first resolution.

## Solution Assessment

Unclear until maintainer/security review chooses the policy. The issue lists several plausible directions, and copying secrets, warning, explicit sync, or fallback-on-401 each carries different security and credential-authority tradeoffs.
