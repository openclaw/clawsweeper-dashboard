## Summary

Keep open. Current main still resolves stored auth profiles before env/config-backed provider auth, `REF_SHADOWED` remains an audit-only signal, and the related reconciliation PR #57371 was closed unmerged, so the requested rotation or sync UX is not implemented.

## What This Changes

Add a way to sync, warn about, or otherwise handle provider API-key rotation across per-agent auth profiles when config/env credentials are shadowed.

## Best Possible Solution

Ship an explicit, non-destructive rotation path, likely a dry-run-capable auth sync or repair flow plus shadow warnings, while preserving documented profile precedence unless maintainers choose to change it.

## Reproduction Assessment

Yes, at source level. With an Anthropic profile present and `ANTHROPIC_API_KEY` supplied through env/config, `createProviderAuthResolver` returns the stored profile before it checks env or config-backed auth.

## Solution Assessment

Unclear. The issue lists several viable approaches, but bulk syncing or fallback-on-401 changes secret handling and per-agent isolation semantics, so a maintainer should choose the policy before implementation.
