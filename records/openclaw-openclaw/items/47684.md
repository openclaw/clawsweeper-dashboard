## Summary

Keep open. ## What This Changes

The issue requests a user-facing way to sync, migrate, warn about, or otherwise handle rotated provider API keys across per-agent auth profile stores.

## Best Possible Solution

Design an explicit, audited SQLite-backed bulk auth update or doctor/status warning path, and avoid unconditional fallback-on-401 unless maintainers deliberately choose that security tradeoff.

## Reproduction Assessment

Yes, at source level: stored real profiles are still resolved before env fallback, and current CLI commands do not provide all-agent auth sync or migration. I did not run a live credential-rotation flow because this review is read-only and has no provider credentials.

## Solution Assessment

Unclear. A sync command or warning path is plausible, but the best fix needs maintainer/security choice because automatic secret propagation, auth precedence, and fallback-on-401 carry different upgrade and safety tradeoffs.
