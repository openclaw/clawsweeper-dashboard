## Summary

Keep open. Current main and v2026.5.22 still lack a cross-agent auth sync or auto-sync flow, and source shows real stored auth profiles can still take precedence over config env credentials; the related auth-migration issue is adjacent but does not fully replace this API-key rotation UX/security decision.

## What This Changes

The issue requests a user-facing way to rotate or synchronize provider API keys across per-agent auth profile stores when config-level env keys are updated.

## Best Possible Solution

Define an owner-reviewed auth rotation path that preserves SecretRef/keyRef behavior, reports shadowed plaintext profiles, and offers an explicit dry-run/apply flow for cross-agent credential or active-profile migration.

## Reproduction Assessment

Yes, as a source-level current-main check rather than a live credential repro. The CLI has no bulk auth sync/migrate command, auth-order writes are per-agent, and the resolver still checks real stored auth profiles before env fallback.

## Solution Assessment

Unclear until maintainer product and security review. A sync or migrate command is plausible, but automatic config-set mutation would be risky unless target scope, locking, audit, and rollback semantics are explicit.
