## Summary

Keep open. Current main and the latest release still preserve real stored auth profiles ahead of env/config env fallback, have no bulk auth sync/migrate command, and only provide narrower fixes for explicit provider apiKey precedence, OAuth stale-shadow cleanup, and rate-limit key rotation.

## What This Changes

The issue asks for a user-facing way to synchronize or warn about provider API key rotation when per-agent auth profiles shadow updated config/env credentials.

## Best Possible Solution

Add an explicit, security-reviewed auth maintenance path, most likely doctor/status diagnostics plus a deliberate sync or migrate command, without making runtime silently mutate or fall back across credential sources on 401.

## Reproduction Assessment

Yes, source-level only: current main still preserves real stored-profile precedence over env auth and the existing stale-shadow repair is OAuth-only. I did not run a live credential-rotation repro because this review is read-only and should not create or mutate auth state.

## Solution Assessment

Unclear. The requested direction is valid, but the best fix needs maintainer/security choice between diagnostics, doctor repair, an explicit bulk command, or changing precedence; unconditional fallback-on-401 is not clearly safe.
