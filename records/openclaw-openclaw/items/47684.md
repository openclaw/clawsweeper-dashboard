## Summary

Keep open. Current main still has profile-first auth resolution for normal runtime/status paths, `REF_SHADOWED` remains audit-only, and there is no bulk sync/rotate command; the direct `models.providers.*.apiKey` case was partly addressed elsewhere, but this env/config-backed rotation UX remains a policy-sensitive feature request.

## What This Changes

Add a user-friendly way to rotate or reconcile provider API keys across per-agent auth-profile stores, or warn when stored profiles shadow updated config/env credentials.

## Best Possible Solution

Define a non-destructive credential-rotation policy, likely a dry-run-capable sync/doctor repair plus clearer shadow warnings, while preserving or deliberately revising documented auth-profile precedence.

## Reproduction Assessment

Yes, at source level. Current source and tests show stored auth profiles win over env credentials by default, with env-first only available when a caller opts in.

## Solution Assessment

Unclear. The issue lists several plausible directions, but auto-sync, fallback-on-401, and warning-only behavior have different security and precedence tradeoffs that maintainers should choose first.

## Security Review

Status: needs_attention

Summary: The issue is security-sensitive because API-key rotation and any sync repair affect where secrets are stored and which credentials are selected at runtime.

Concerns:

- **[medium] Stale stored credentials can shadow rotated env credentials:** `src/agents/model-auth.ts:688`
  - body: Normal runtime resolution can select a stored auth profile before the updated env/config credential, so users may believe rotation succeeded while OpenClaw continues trying the old stored credential.
  - confidence: 0.9
- **[medium] Bulk sync could expand secret copies without policy:**
  - body: A sync command or auto-sync path would write credential material across agent stores unless it uses refs or a narrow repair policy, so secops should define the intended authority and dry-run behavior first.
  - confidence: 0.82
