## Summary

Keep open. Current main and v2026.5.22 still have per-agent auth-profile writes, profile-first credential resolution, and only audit-level REF_SHADOWED reporting, so the requested bulk rotation/sync or config-set/status warning remains unresolved.

## What This Changes

The issue requests a user-facing way to rotate or synchronize provider API keys across agent-scoped auth profiles, or at least warn when stale profiles shadow updated env config.

## Best Possible Solution

Define an owner-reviewed explicit auth rotation path with dry-run/reporting and shadowing warnings, using existing auth-store locks and preserving explicit credential precedence; avoid implicit 401 fallback unless maintainers approve that security policy.

## Reproduction Assessment

Yes, at source level. Current main has per-agent auth commands and profile-first provider resolution by default, while no bulk sync/migration command or config-set shadow warning exists for rotated env keys.

## Solution Assessment

Unclear as proposed. An explicit sync or migration command plus warnings looks safer than automatic cross-agent rewrites or automatic 401 fallback, but maintainers need to choose the credential policy before implementation.
