## Label Justifications

- `P2`: The issue is a real credential-rotation UX and auth-provider improvement with limited blast radius, not an emergency outage.
- `impact:security`: The item is about API key rotation, stale credentials, and whether OpenClaw should copy or fall back between secret sources.
- `impact:auth-provider`: The affected behavior is provider credential precedence between auth profiles, env/config credentials, and SecretRef-backed auth.

## Summary

Keep open. Current main and the latest release still preserve profile-first auth resolution by default, REF_SHADOWED remains audit-only, and no bulk auth sync/rotation command exists, so the remaining work needs maintainer security/product policy rather than cleanup closure.

## What This Changes

The issue requests a way to sync, warn about, or otherwise handle API key rotation when per-agent auth profiles shadow updated config/env credentials.

## Best Possible Solution

Define an explicit, non-destructive credential-rotation policy, likely a dry-run-capable sync or stronger shadow warning flow, while preserving documented profile precedence unless security owners deliberately change it.

## Reproduction Assessment

Yes, at source level. `env.*` config values are applied to the runtime environment, but current main and `v2026.5.12` resolve stored auth profiles before env credentials by default and only expose REF_SHADOWED through secrets audit.

## Solution Assessment

Unclear. The issue lists several viable directions, and maintainers need to choose whether credential rotation should copy secrets, warn, offer an explicit sync, or alter fallback/precedence semantics.

## Security Review

Status: needs_attention

Summary: Credential rotation policy is security-sensitive because the fix could copy API keys across agent stores or change which credential source wins at runtime.

Concerns:

- **[medium] Stale stored credentials can shadow rotated env credentials:** `src/agents/model-auth.ts:693`
  - body: Normal profile-first resolution can select an existing auth-profile credential before the updated env/config credential, leaving users with a stale key even after they update `env.ANTHROPIC_API_KEY`.
  - confidence: 0.9
- **[medium] Bulk sync could expand secret copies without policy:**
  - body: A sync or auto-sync implementation could write credential material across multiple agent stores unless maintainers require a narrow, auditable, dry-run-capable, or SecretRef-aware flow.
  - confidence: 0.82
