- **[P1] Keep auto image backend compatibility:** `src/media/image-ops.ts:138`
  - body: Current main only fails closed on unknown image dimensions when an operator explicitly selects a backend; in auto mode it can still try the bundled backend first.   - confidence: 0.87

  - body: The PR adds BMP/TIFF header parsing, but this gate still only allows PNG/GIF/WebP/JPEG into Photon. As a result, BMP/TIFF inputs skip the bundled Photon backend and require ImageMagick/GraphicsMagick even though the pinned Photon WASM package includes BMP and TIFF decoder support, so installs without those external tools regress unnecessarily.
  - confidence: 0.86

  - body: This PR removes Sharp from the root optionalDependencies, but the install docs in this branch still tell npm users how to work around Sharp/libvips build failures with SHARP_IGNORE_GLOBAL_LIBVIPS. That guidance becomes misleading once this package no longer installs Sharp, so the install troubleshooting docs should be removed or rewritten with the new system-converter requirements.
