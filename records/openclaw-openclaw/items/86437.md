## Summary

The PR replaces Sharp/Jimp image processing surfaces with Photon, updates media and WhatsApp image handling, dependency metadata, installers, docs, changelog, and image tests.

## Best Possible Solution

Land the Photon replacement only after preserving or explicitly accepting the HEIC/default-backend upgrade change, keeping pinned-install compatibility, allowing Photon-supported BMP/TIFF inputs, and recording maintainer acceptance of the new parser dependency.

## Solution Assessment

No; replacing Sharp with Photon is plausible, but this branch is not the best merge shape until it preserves existing HEIC and installer compatibility or gets explicit maintainer acceptance for those breaks.

  - body: Current main tries the packaged Sharp backend before system tools and the docs promise HEIC/HEIF input images are accepted.   - body: The installer still supports `--version` and `OPENCLAW_VERSION`, so it can install older OpenClaw releases that still depend on Sharp. Removing `SHARP_IGNORE_GLOBAL_LIBVIPS=1` from those npm/pnpm paths reopens the known global-libvips build failure for rollbacks and pinned installs.
  - confidence: 0.88

  - body: The PR adds BMP/TIFF metadata preflight, and Photon 0.3.4 can decode valid BMP/TIFF buffers, but this gate still lets only PNG/GIF/WebP/JPEG reach Photon resize operations. With `OPENCLAW_IMAGE_BACKEND=photon`, or auto mode on hosts without external tools, those supported inputs are rejected before the bundled backend can process them.
  - confidence: 0.9
