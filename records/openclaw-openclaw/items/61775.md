## What This Changes

The PR expands the root Makefile into a Make-based wrapper around existing pnpm build, test, format, lint, typecheck, development, dependency, and cleanup commands.

## Best Possible Solution

If maintainers want this developer surface, rebase it over current main, keep it as a thin pnpm wrapper, sequence ordered workflows explicitly, and require redacted real command output before merge.

## Reproduction Assessment

Yes for the review findings: static inspection of the PR head plus GNU Make's documented parallel execution model shows `rebuild` and `landing-gate` can run their prerequisites out of the intended order under parallel make.

## Solution Assessment

No. The wrapper idea is only maintainable if maintainers want the root Makefile back, and the ordered workflows should be recipes that invoke steps sequentially rather than ordinary prerequisite lists.
