## Summary

Keep open. The follow-up addressed the earlier `.PHONY` and `format` split feedback, but the latest head still models ordered workflows as ordinary Make prerequisites, so parallel Make can interleave clean/build and check/test/build steps.

## What This Changes

This PR expands the root Makefile from a single build target into a developer workflow wrapper around existing pnpm scripts for deps, build/UI, clean/rebuild, checks, tests, dev, version, and landing-gate.

## Best Possible Solution

Keep the Makefile convenience layer, but convert ordered compound targets into sequential recipes that invoke `$(MAKE)` one step at a time.

## Reproduction Assessment

Yes. Static inspection of the latest PR head plus GNU Make behavior is enough: ordinary prerequisites may run concurrently under `make -j` or inherited `MAKEFLAGS`.

## Solution Assessment

No. Delegating to existing pnpm scripts is a reasonable wrapper, but ordered workflows should be recipes rather than prerequisite lists so their sequence is preserved.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P2] Sequence rebuild after cleaning:** `Makefile:35`
  - body: `rebuild` lists `clean-dist build` as ordinary prerequisites, so `make -j rebuild` can run clean and build concurrently. That can remove `dist` while the build is producing it; make this target a sequential recipe that runs clean before build.
  - confidence: 0.9
- **[P2] Sequence landing gate steps:** `Makefile:87`
  - body: `landing-gate` lists `check test build` as ordinary prerequisites, which GNU Make may schedule concurrently under `-j` or inherited `MAKEFLAGS`. That defeats the promised check-then-test-then-build gate and can overlap broad commands in one worktree.
  - confidence: 0.92
