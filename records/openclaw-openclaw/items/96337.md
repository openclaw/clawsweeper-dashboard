## Summary

Keep open: current main still routes Anthropic Vertex through the Vertex SDK raw stream into the shared Anthropic SSE parser, and I found no same-repo merged fix or canonical open PR that specifically covers native pure-text Vertex streaming.

## What This Changes

The issue reports a v2026.6.10 regression where `anthropic-vertex` native streaming returns `non-visible-output` for first pure-text Claude responses that worked through proxy-like routing in v2026.6.8.

## Best Possible Solution

Fix and live-validate the provider-owned Anthropic Vertex stream path so native pure-text responses produce visible text, using a documented maintainer-approved proxy-like fallback only if native cannot be stabilized quickly.
