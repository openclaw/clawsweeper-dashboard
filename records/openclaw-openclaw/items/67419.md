## Summary

Keep open: current main and the latest release still default omitted `agents.defaults.contextInjection` to `always`, keep `MEMORY.md` in the stable bootstrap set, and only treat `HEARTBEAT.md` as dynamic. The merged work gives users opt-in mitigations, but the requested default, MEMORY stub, dynamic bootstrap-file policy, and restart/session-state edge cases still need maintainer product direction.

## What This Changes

The issue requests reducing repeated bootstrap/session context injection by changing context-injection defaults, stubbing `MEMORY.md`, and adding configurable dynamic bootstrap files.

## Best Possible Solution

Keep this as the concrete bootstrap/context-bloat tracker and have maintainers split or sponsor the default-policy, MEMORY-stub, and dynamic-file pieces with upgrade-safe docs and regression coverage.

## Reproduction Assessment

Yes for the default/config portion: source and tests show omitted `contextInjection` still resolves to `"always"`, and docs/release text match that behavior. The restart-specific and MEMORY-stub savings need live/product validation before a final implementation shape.

## Solution Assessment

No, not as one automatic bug-fix PR. The maintainable path is a maintainer-sponsored policy split for defaults, MEMORY loading, and dynamic cache-boundary control because each can change prompt behavior for existing users.
