## Summary

Keep open. Current main still defaults omitted `agents.defaults.contextInjection` to per-turn bootstrap injection, docs/UI/tests encode that behavior, `MEMORY.md` is still normal bootstrap context, and the requested dynamic bootstrap-file config is absent; the open draft PR only covers one `continuation-skip` edge case.

## What This Changes

The issue requests reducing session-token bloat by making `continuation-skip` the default, lazy-loading `MEMORY.md` through a stub, and adding configurable dynamic bootstrap-file placement.

## Best Possible Solution

Keep this open until maintainers decide the default bootstrap reinjection policy, land or close #74813 for the narrow continuation evidence gap, and split `MEMORY.md` lazy loading plus dynamic bootstrap-file configuration into scoped follow-ups with #65438 covering the cache-boundary config piece.

## Reproduction Assessment

Yes. Source and tests on current main show the default still resolves to `"always"`, assistant-only transcripts without the full-bootstrap marker do not count as completed, `MEMORY.md` is injected when present, and no dynamic bootstrap-file config exists.

## Solution Assessment

No patch is proposed here, and draft #74813 is only a partial repair for `continuation-skip`. The maintainable path needs a maintainer policy decision before changing defaults, then separate runtime/config/docs/UI/test work for each scoped behavior.
