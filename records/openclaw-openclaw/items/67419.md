## Summary

Keep open. Current main still defaults bootstrap context injection to `always`, still documents and tests that default, still injects `MEMORY.md` when present, and still has hard-coded dynamic bootstrap file handling. The maintainer umbrella preserves concrete unresolved reports, and draft PR #74813 is an implementation candidate for only one continuation-skip slice, not a merged fix for the whole issue.

## What This Changes

The issue requests reducing bootstrap/session context bloat by making continuation-safe bootstrap injection the default, lazy-loading or stubbing `MEMORY.md`, and adding configurable dynamic bootstrap-file behavior.

## Best Possible Solution

Keep this issue open until maintainers decide the default bootstrap reinjection policy. If the lower-token direction is accepted, land a coordinated runtime/config/docs/tests/UI default change, review #74813 for the restart/assistant-turn continuation gap, and track `MEMORY.md` lazy loading plus dynamic bootstrap-file ordering under scoped follow-ups such as #65438.

## Reproduction Assessment

Yes. Not applicable as a patch review. The best fix is not to close this from the existing opt-in mechanism alone; the maintainable path is a scoped maintainer decision plus aligned implementation across runtime, config schema/help, docs, UI defaults, and regression tests.
