## Summary

Keep open. Current main and the latest release still default missing `agents.defaults.contextInjection` to per-turn bootstrap injection, `continuation-skip` remains opt-in, `MEMORY.md` is still normal bootstrap context, and the configurable dynamic-file/cache-boundary part is still tracked separately. The linked default-switch and assistant-turn PRs are closed unmerged, while the maintainer umbrella keeps this concrete report as unresolved evidence.

## What This Changes

The issue requests reducing recurring bootstrap context bloat by defaulting to continuation-skip, replacing full MEMORY.md injection with on-demand loading, and adding configurable dynamic bootstrap file handling.

## Best Possible Solution

Maintainers should choose one canonical context-injection policy with upgrade proof, keep an explicit `always` escape hatch if the default changes, and split the MEMORY.md stub plus dynamic-file cache-boundary work into the existing narrower trackers where appropriate.

## Reproduction Assessment

Yes by source inspection for the default behavior: omitted `contextInjection` still resolves to `always`, and tests assert that default. I did not live-run a gateway session, so the exact percentage savings and restart-specific repro are not fully proven here.

## Solution Assessment

No, not as one autonomous fix. The maintainable path needs a maintainer decision on default/upgrade behavior, while dynamic-file ordering belongs with https://github.com/openclaw/openclaw/issues/65438 and duplicate transcript/session assembly remains tied to https://github.com/openclaw/openclaw/issues/69208.
