## Summary

Keep open. Current main still defaults missing `agents.defaults.contextInjection` to per-turn bootstrap injection, documents and exposes that default, injects `MEMORY.md` as normal bootstrap context when present, and has no dynamic bootstrap-file config; the linked umbrella explicitly says concrete reports with unresolved evidence should remain open.

## What This Changes

The issue asks OpenClaw to reduce bootstrap context token bloat by making `continuation-skip` the default, lazy-loading or stubbing `MEMORY.md`, and adding dynamic bootstrap-file cache-boundary configuration.

## Best Possible Solution

Define an explicit bootstrap-context policy that covers the default mode, memory injection behavior, and cache-boundary configuration, then land narrow runtime/config/docs/tests changes with transcript-duplication follow-ups tracked separately.

## Reproduction Assessment

Yes. Source inspection on current main shows the default still resolves to `always`, assistant-only transcripts without the marker remain incomplete, `MEMORY.md` is loaded when present, and dynamic bootstrap-file config is not exposed.

## Solution Assessment

No patch is proposed here, and the linked draft PR is only partial. The maintainable path is an explicit maintainer decision on defaults and scope, then coordinated runtime, config, docs, UI, and regression coverage.
