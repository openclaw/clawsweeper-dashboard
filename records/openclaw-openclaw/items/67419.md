## Summary

Keep open. Current main still has the reported default bootstrap reinjection behavior, still loads MEMORY.md as normal bootstrap context, and still lacks the requested dynamic bootstrap-file config surface; the linked umbrella and draft PR do not fully replace this concrete report.

## What This Changes

The issue asks OpenClaw to reduce repeated bootstrap context cost by changing the default contextInjection behavior, lazy-loading MEMORY.md, and adding dynamic bootstrap-file cache-boundary controls.

## Best Possible Solution

Have maintainers decide the default bootstrap reinjection policy, then land a coordinated runtime/config/docs/UI/test change if lower default token use is desired; keep MEMORY.md lazy loading and dynamic-file ordering as scoped follow-ups or canonical linked issues.

## Reproduction Assessment

Yes. Source inspection on current main shows omitted contextInjection still resolves to "always", MEMORY.md is still loaded as a bootstrap file when present, and no bootstrapDynamicFiles/bootstrapFileOrder config surface exists.

## Solution Assessment

No patch is proposed here, and the open draft PR only addresses one continuation-skip subcase. The narrow maintainable path is a maintainer policy decision followed by split implementation slices with focused regression coverage.
