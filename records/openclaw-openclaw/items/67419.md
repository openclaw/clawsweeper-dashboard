## Summary

Keep open. Current main still uses per-turn bootstrap injection when `agents.defaults.contextInjection` is omitted, still injects `MEMORY.md` when present, and still lacks a `bootstrapDynamicFiles` or `bootstrapFileOrder` config surface; the maintainer umbrella also explicitly keeps this as the canonical bootstrap/context-cost track.

## What This Changes

The issue asks OpenClaw to reduce repeated bootstrap context cost by defaulting safe continuations to skip reinjection, lazy-loading or stubbing `MEMORY.md`, and adding dynamic bootstrap-file cache-boundary configuration.

## Best Possible Solution

Keep this open as the bootstrap/context-cost tracker, decide the default policy explicitly, land the narrow continuation-skip correctness work only after review, and handle dynamic file classification through the canonical config/API discussion at https://github.com/openclaw/openclaw/issues/65438.

## Reproduction Assessment

Yes for the source-level behavior: current main still defaults omitted `contextInjection` to `"always"`, includes `MEMORY.md` as bootstrap context, and has no dynamic bootstrap-file config. The restart-specific continuation-skip subcase has reporter transcript evidence but was not live-replayed here.

## Solution Assessment

Unclear until maintainers choose the product/API boundary. The requested direction is plausible, but the maintainable path should split the default behavior decision, `MEMORY.md` lazy-loading, dynamic cache-boundary API, and the narrow continuation-skip fix.
