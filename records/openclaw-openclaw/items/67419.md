## Summary

Keep open. Current main and the latest release still default missing `agents.defaults.contextInjection` to `always`, the `continuation-skip` path remains opt-in, `MEMORY.md` and dynamic bootstrap-file handling are not fully addressed for non-Codex paths, and the related implementation PRs are closed unmerged while the maintainer umbrella preserves this issue's concrete evidence.

## What This Changes

The issue requests reducing session context bloat by avoiding repeated bootstrap-file injection, shrinking or lazily loading `MEMORY.md`, and adding cache-boundary controls for dynamic bootstrap files.

## Best Possible Solution

Decide the default bootstrap reinjection policy, implement the chosen runtime/config/docs/UI/tests together, keep dynamic-file controls under https://github.com/openclaw/openclaw/issues/65438, and keep broader duplicate transcript analysis under https://github.com/openclaw/openclaw/issues/69208.

## Reproduction Assessment

No high-confidence live reproduction was established in this read-only pass. Source inspection does reproduce the default `always` behavior and the missing dynamic-file config surface, but the exact token percentage and restart cache-bust path still need live proof.

## Solution Assessment

Unclear. Defaulting to `continuation-skip` is plausible, but it is not automatically the best fix because it changes default prompt behavior; `MEMORY.md` stubbing and dynamic cache-boundary controls need separate product/API decisions.
