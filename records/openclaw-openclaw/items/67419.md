## Summary

Keep open: current main still leaves bootstrap/context injection defaulted to `always`, keeps `continuation-skip` as explicit opt-in, and the MEMORY.md/dynamic bootstrap policy portions remain unresolved. The linked umbrella tracks the broader family but explicitly preserves concrete reports like this until their exact symptom is verified fixed.

## What This Changes

The issue requests reducing repeated bootstrap and memory context injection across multi-turn sessions, including default continuation skipping, lighter memory bootstrap, and dynamic bootstrap-file cache boundary controls.

## Best Possible Solution

Maintainers should split the remaining policy: decide whether safe continuation should become the default, preserve explicit per-turn injection for users who need it, and track dynamic bootstrap cache-boundary config in https://github.com/openclaw/openclaw/issues/65438.

Yes for the source-level default behavior: `resolveContextInjectionMode()` returns `always` when the setting is omitted, and tests assert that default. No live restart repro was run in this read-only review.

## Solution Assessment

Unclear as a single fix: making `continuation-skip` the default is plausible but compatibility-sensitive, while MEMORY.md stubbing and dynamic bootstrap ordering are broader product/config decisions.
