## Summary

Keep open. Current main still defaults missing `agents.defaults.contextInjection` to `"always"`, keeps `continuation-skip` opt-in, injects `MEMORY.md` as normal bootstrap context, and has no configurable dynamic bootstrap-file surface; the remaining scope needs maintainer policy decisions rather than conservative cleanup closure.

## What This Changes

The issue asks OpenClaw to reduce bootstrap context bloat by changing the default context-injection mode, shrinking MEMORY.md bootstrap payloads, and adding configurable dynamic bootstrap files.

## Best Possible Solution

Keep a scoped tracking split: decide the default context-injection policy, land or close #74813 for the marker gap, and handle `MEMORY.md` lazy-loading plus dynamic bootstrap-file controls as separate follow-ups, with #65438 covering the cache-boundary config piece.

## Solution Assessment
