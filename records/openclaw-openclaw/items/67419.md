## Summary

Keep open. Current main and the latest release still default omitted `agents.defaults.contextInjection` to per-turn bootstrap injection, keep `continuation-skip` opt-in, keep `MEMORY.md` in the normal non-Codex bootstrap path, and expose no `bootstrapDynamicFiles` or `bootstrapFileOrder` config surface. The linked default-switch and assistant-turn mitigation PRs are closed unmerged, while the open umbrella and dynamic-file tracker preserve remaining work.

## What This Changes

The issue requests reducing session context bloat by changing bootstrap reinjection defaults, shrinking or lazy-loading `MEMORY.md`, and adding dynamic bootstrap-file cache-boundary controls.

## Best Possible Solution

Keep this issue open until maintainers choose the canonical default/context policy, then land one scoped implementation for the approved default while leaving dynamic bootstrap-file controls under https://github.com/openclaw/openclaw/issues/65438 and transcript/idempotency work under https://github.com/openclaw/openclaw/issues/69208.
