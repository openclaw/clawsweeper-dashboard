## Summary

Keep open. Current main and v2026.6.6 still default omitted `agents.defaults.contextInjection` to per-turn bootstrap injection, `continuation-skip` remains opt-in, `MEMORY.md` is still normal prompt context for non-Codex harnesses, and the dynamic bootstrap-file/cache-boundary config is not implemented; the related default-switch and assistant-turn PRs are closed or superseded back to this issue.

## What This Changes

Request to reduce session context bloat by changing bootstrap re-injection defaults, making `MEMORY.md` lighter, and adding dynamic bootstrap cache-boundary control.

## Best Possible Solution

Maintainers should choose one canonical bootstrap-context policy, keep an explicit per-turn override if the default changes, and split the `MEMORY.md` and dynamic-file ordering work to narrower follow-ups such as https://github.com/openclaw/openclaw/issues/65438.

## Reproduction Assessment

Yes, at source level: omitted `contextInjection` still resolves to `always`, and tests/docs assert that current behavior. I did not live-run the gateway restart scenario or independently measure the reported token percentages.

## Solution Assessment

No, not as one autonomous fix. The default flip is a compatibility/product decision, while `MEMORY.md` stubbing and dynamic bootstrap-file ordering are separate scopes that should be split or explicitly approved together.
