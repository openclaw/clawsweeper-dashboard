## Summary

Keep open: current main and the latest release still default omitted `agents.defaults.contextInjection` to `always`, keep `continuation-skip` opt-in, still inject `MEMORY.md` as normal bounded bootstrap context on non-Codex paths, and have no `bootstrapDynamicFiles`/`bootstrapFileOrder` config surface. The issue remains useful as the canonical bootstrap/context-bloat tracker, but the next change is compatibility-sensitive and needs maintainer product direction.

## What This Changes

The issue asks OpenClaw to reduce repeated session-context bootstrap cost by making `continuation-skip` the default, shrinking or lazy-loading `MEMORY.md`, and adding dynamic bootstrap-file cache-boundary controls.

## Best Possible Solution

Keep this as the canonical bootstrap/context-bloat policy issue, let maintainers choose the default behavior, then land scoped runtime/config/docs/UI/test changes while splitting dynamic bootstrap-file ordering to the narrower tracker.

## Reproduction Assessment

Yes at source level: current main and `v2026.6.8` still default omitted `agents.defaults.contextInjection` to `"always"`, and the requested dynamic bootstrap-file config surface is absent. I did not live-run the gateway restart/cache-bust scenario.

## Solution Assessment

Unclear as a bundled change: the default switch is mechanically narrow, but it changes behavior for every default-configured agent and the `MEMORY.md` plus dynamic-file portions need separate scope decisions. The safest path is a maintainer-approved default policy plus split follow-ups for dynamic ordering and memory prompt shape.
