## Summary

Keep open. Current main still leaves the reported bootstrap/context-cost behavior unresolved: omitted `contextInjection` resolves to per-turn injection, `MEMORY.md` is still normal bootstrap context, assistant-only continuations still do not count as completed bootstrap turns, and dynamic bootstrap-file configuration is still absent.

## What This Changes

The issue asks OpenClaw to reduce session context bloat by making continuation skipping the default, lazy-loading or stubbing `MEMORY.md`, and adding dynamic bootstrap-file cache-boundary configuration.

## Best Possible Solution

Keep this open as the bootstrap/context-cost tracker, decide the default policy explicitly, land or replace the narrow continuation-skip correctness fix, and handle dynamic file classification through https://github.com/openclaw/openclaw/issues/65438.

## Solution Assessment

Unclear until maintainers choose the product/API boundary. The requested direction is plausible, but the maintainable path should split the default behavior decision, `MEMORY.md` lazy-loading, dynamic cache-boundary API, and the narrow continuation-skip fix.
