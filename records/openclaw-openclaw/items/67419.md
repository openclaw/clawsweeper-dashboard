## Summary

Keep open. Current main still preserves the reported default reinjection behavior, keeps `continuation-skip` opt-in, still injects `MEMORY.md` as normal bootstrap context, and lacks the requested dynamic bootstrap-file config; the open draft PR only covers one continuation-skip edge case while the maintainer umbrella explicitly keeps concrete reports with unresolved evidence open.

## What This Changes

The issue requests reducing repeated bootstrap context injection by changing the default contextInjection mode, shrinking `MEMORY.md` bootstrap cost, and adding dynamic bootstrap-file cache-boundary controls.

## Best Possible Solution

Keep this issue open until maintainers split and decide the policy: land or close #74813 for the continuation-skip marker gap, decide whether the default should change, and track `MEMORY.md` lazy-loading plus `bootstrapDynamicFiles` separately with #65438 covering the cache-boundary config piece.

## Reproduction Assessment

Yes. Source and tests show omitted `agents.defaults.contextInjection` resolves to `"always"`, assistant-only transcripts without the full-bootstrap marker do not count as completed, `MEMORY.md` is still injected, and no dynamic bootstrap-file config exists on current main.

## Solution Assessment

No patch is proposed here, and the linked draft PR is only a partial solution. The maintainable path is to make an explicit default-policy decision, then implement runtime/config/docs/UI/tests together while splitting `MEMORY.md` and dynamic-file ordering into scoped follow-ups.
