## Summary

Keep open. Current main still preserves the reported default reinjection behavior, and the remaining work spans a product default decision plus separate MEMORY.md and dynamic-bootstrap-file scopes; the open draft PR only covers one continuation-skip edge case.

## What This Changes

Reduce session token bloat by skipping repeated bootstrap file injection, making MEMORY.md less costly, and adding dynamic bootstrap-file cache-boundary control.

## Best Possible Solution

Keep this issue open until maintainers choose the default bootstrap injection policy, then align runtime, config schema/help, docs, UI defaults, and tests; treat MEMORY.md stubbing and dynamic file ordering as scoped follow-ups, with #65438 covering the cache-boundary config piece.
