## Summary

Keep open. Current main still defaults missing `agents.defaults.contextInjection` to per-turn bootstrap injection, still injects `MEMORY.md` when present, and has no dynamic bootstrap-file ordering/config surface; draft PR #74813 addresses only one continuation-skip edge case.

## What This Changes

The issue asks OpenClaw to reduce session context bloat by making continuation bootstrap injection leaner by default, lazy-loading or stubbing `MEMORY.md`, and adding configurable dynamic bootstrap file cache-boundary control.

## Best Possible Solution

Keep this issue open until maintainers choose the default bootstrap-injection policy, then align runtime, schema/help, docs, UI presets, and tests; handle `MEMORY.md` and dynamic-file ordering as scoped follow-ups, with #65438 covering the cache-boundary config piece.

## Reproduction Assessment

Yes. The default-config path reproduces from source and tests: omitted config resolves to `always`, docs/UI expose that default, and marker-only completion detection leaves the restart/assistant-turn continuation case unresolved on main.

## Solution Assessment

Not applicable as a patch review. The existing opt-in `continuation-skip` mechanism is useful but only partial; #74813 is a narrow draft fix, while the default, `MEMORY.md`, and dynamic-file changes need maintainer scoping.
