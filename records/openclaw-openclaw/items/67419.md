The issue asks OpenClaw to reduce repeated bootstrap/session context injection and token bloat by changing the default injection mode and improving memory/cache-boundary behavior.

## Best Possible Solution

Maintainers should choose the default bootstrap policy first, then land a scoped runtime/config/docs/test change for that decision while leaving dynamic-file ordering to the canonical cache-boundary tracker and treating non-Codex memory compaction as a separate follow-up if needed.
