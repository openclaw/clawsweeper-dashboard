Full review comments:

- **[P1] Keep serviceAccountAdc account-local:** `extensions/googlechat/src/accounts.ts:159-160`
  - body: `accounts.default` currently strips credential fields before sharing defaults with named accounts, but this new credential selector is not stripped. With `accounts.default.serviceAccountAdc: true`, a named account without its own key can resolve to ADC and send as the ambient identity, so exclude the flag from shared defaults and add a named-account regression.
  - confidence: 0.92
- **[P2] Register serviceAccountAdc for implicit default accounts:** `extensions/googlechat/src/accounts.ts:159-160`
  - body: The PR documents the top-level `channels.googlechat.serviceAccountAdc` form, but Google Chat account listing only treats `serviceAccount`, `serviceAccountRef`, and `serviceAccountFile` as implicit default-account keys. In a multi-account config this can leave the top-level ADC default account out of startup/action enumeration, so add the new flag to the implicit credential keys and test it with a named account present.
  - confidence: 0.88
- **[P1] Resolve the metadata-server ADC boundary:** `extensions/googlechat/src/accounts.ts:159-160`
  - body: Returning `adc` lets `GoogleAuth` use Compute ADC; `google-auth-library` then calls `gcp-metadata`, which performs direct `gaxios.request` calls to metadata hosts instead of the plugin's `createGoogleAuthFetch` guard. Route that metadata access through an approved OpenClaw guard or get explicit maintainer/security approval with focused coverage before enabling it.
  - confidence: 0.87
