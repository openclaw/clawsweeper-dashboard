## Summary

Keep this PR open: the keyless Google Chat auth idea is useful, but the proposed ADC source opens a security-sensitive metadata-token path outside the existing guarded Google auth transport, and the PR still needs stronger real behavior proof plus conflict cleanup before merge.

## What This Changes

The PR adds an opt-in `channels.googlechat.serviceAccountAdc` config path, docs, generated config metadata, and resolver tests so Google Chat outbound auth can use ADC when no explicit service-account credential is configured.

## Best Possible Solution

Keep the opt-in keyless Google Chat direction, but make metadata-token ADC an explicit bounded auth path or maintainer-approved exception with tests and redacted live proof before merge.
