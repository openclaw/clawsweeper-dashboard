The PR adds an opt-in `channels.googlechat.serviceAccountAdc` config key, schema/docs/generated metadata, and resolver tests so Google Chat outbound auth can use Application Default Credentials when no explicit service-account credential is configured.

## Best Possible Solution

Keep keyless Google Chat auth opt-in only after named-account credential isolation is fixed, the ADC metadata boundary is either guarded or explicitly approved, and redacted proof shows the outbound reply path.
