Full review comments:

- **[P1] Keep serviceAccountAdc out of shared default credentials:** `extensions/googlechat/src/accounts.ts:159-160`
  - body: `accounts.default` intentionally shares webhook/audience defaults while stripping credential fields before named accounts inherit them. Because this patch adds `serviceAccountAdc` but does not add it to that exclusion list, `accounts.default.serviceAccountAdc: true` can make every named account without its own key resolve as `credentialSource: "adc"`, changing which identity signs outbound Chat messages. Please strip this field with the other credential fields and add a named-account regression test.
  - confidence: 0.91
- **[P1] Route ADC metadata access through an approved boundary:** `extensions/googlechat/src/accounts.ts:159-160`
  - body: Returning `"adc"` lets GoogleAuth fall through to Compute ADC, where `gcp-metadata` refreshes tokens with direct `gaxios.request` calls to the metadata host rather than the plugin's guarded `createGoogleAuthFetch()` transport. Before this source ships, please route metadata ADC through an OpenClaw-owned guard or get an explicit maintainer/security exception and add focused coverage.
  - confidence: 0.87
