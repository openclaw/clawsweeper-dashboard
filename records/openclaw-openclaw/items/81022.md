## Summary

Keep this PR open for normal maintainer review. The patch is focused, matches Robolectric and Conscrypt dependency contracts, has supplied after-fix terminal proof from Linux ARM64, and I found no blocking review findings.

## What This Changes

The PR disables Robolectric Conscrypt for Android Gradle `Test` tasks so JVM unit tests avoid loading Conscrypt native code on Linux ARM64.

## Best Possible Solution

Merge the focused Gradle test override after normal Android maintainer review and CI, keeping the change limited to JVM test configuration rather than runtime code or dependency resolution.

## Reproduction Assessment

Yes, as a source-backed path: run the Android JVM unit tests on Linux aarch64 from current main, where Robolectric 4.16.1 defaults Conscrypt ON and Conscrypt 2.5.2 lacks a Linux aarch64 native library. I did not execute that host path in this read-only review.

## Solution Assessment

Yes. Setting `robolectric.conscryptMode=OFF` on Gradle `Test` tasks uses Robolectric's supported configuration surface and is narrower than changing dependencies or individual tests.
