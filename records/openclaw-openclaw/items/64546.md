## Summary

Keep this PR open: it targets a real Mattermost callback-token security flaw, but the branch is not merge-ready because the replacement secret can be process-local and non-durable, the new config surface is not covered by setup/docs/provisioning, and the live PR is currently dirty against main.

## What This Changes

This PR changes Mattermost interactive-button signing from bot-token-derived HMAC secrets to configured or generated per-account interaction secrets, with related config and test updates.

## Best Possible Solution

Land a mergeable security fix that removes bot-token-derived signing while either provisioning a durable per-account interaction secret or failing closed with explicit setup/migration guidance, then update the Mattermost docs and prove upgrade, restart, separate-process, and old-forgery behavior.

## Reproduction Assessment

Yes. Source inspection shows current main and v2026.6.9 still derive Mattermost interaction signing from the bot token with a public HMAC key, and PR-head proof shows the proposed code rejects an old bot-token-derived forged token.

## Solution Assessment

No. Removing the bot-token-derived signing root is the right direction, but a process-local generated fallback is not the best fix because it is not durable across the existing CLI/gateway process boundary or restarts.
