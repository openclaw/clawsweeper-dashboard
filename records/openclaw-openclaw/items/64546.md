## Summary

Keep this PR open: it fixes the right Mattermost security bug directionally, but the replacement secret is process-local when unset and the new credential is not wired through setup, secret discovery, plugin metadata, or docs, so the branch is not merge-ready.

## What This Changes

The PR adds a Mattermost `interactions.secret` config field and changes interaction signing from a bot-token-derived HMAC secret to a configured or generated per-account secret, with related runtime and tests.

## Best Possible Solution

Land a refreshed security fix that removes bot-token-derived signing and either provisions a durable per-account interaction secret through setup, secret contracts, docs, and tests, or explicitly fails closed with clear upgrade behavior and proof.

## Reproduction Assessment

Yes, source-level: current main derives the signing secret from the Mattermost bot token with a public HMAC key, and the PR-head replacement fallback is process-local when no configured secret exists. I did not run a live Mattermost server in this read-only review.

## Solution Assessment

No. Removing bot-token-derived signing is the right direction, but this is not the best merge shape until the replacement secret is durable or explicitly required across setup, docs, upgrade, and restart/split-process behavior.
