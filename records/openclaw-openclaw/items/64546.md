## Summary

This PR should stay open as the candidate fix for the Mattermost interaction-token security issue, but it is not merge-ready: the replacement signing secret is process-local when unset, the new credential is not wired through setup/secret discovery/docs, and GitHub currently reports the branch as conflicting.

## What This Changes

The PR adds a Mattermost `interactions.secret` config field and changes interaction signing from bot-token-derived HMAC secrets to configured or generated per-account secrets, with runtime and test updates.

## Best Possible Solution

Land a refreshed security fix that removes bot-token-derived signing and either provisions a durable per-account interaction secret through setup, secret contracts, docs, and tests, or explicitly fails closed with clear upgrade behavior and proof.

## Reproduction Assessment

Yes, source-level. Current main and `v2026.6.10` derive the Mattermost interaction signing secret from the bot token with a public HMAC key, and the PR head shows the replacement fallback is process-local when no configured secret exists.

## Solution Assessment

No. Removing bot-token-derived signing is the right direction, but this is not the best merge shape until the replacement secret is durable or explicitly required across setup, docs, upgrade, and restart behavior.
