## Summary

Keep open. ## What This Changes

The branch adds Mattermost `interactions.secret` SecretRef support and changes button HMAC signing from bot-token-derived secrets to configured or generated per-account secrets.

## Best Possible Solution

Land a refreshed security fix that removes bot-token-derived signing without a process-local fallback: persist/provision a per-account secret or fail closed with explicit setup/docs/metadata, then prove fresh install, upgrade, restart/separate-process callbacks, and old-forgery rejection.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.9

Full review comments:

- **[P1] Persist or require the interaction secret:** `extensions/mattermost/src/mattermost/interactions.ts:198-200`
  - body: When `interactions.secret` is unset, this generates a random process-local secret. The send path signs buttons and the gateway monitor verifies callbacks, and current main deliberately keeps those stable across CLI/gateway processes; after this change, CLI-sent buttons, post-restart clicks, or separate processes can fail `_token` verification. Persist/provision the secret or fail closed with explicit setup/docs/tests instead of silently generating an in-memory fallback.
  - confidence: 0.91
- **[P1] Deep-merge interactions before reading account secrets:** `extensions/mattermost/src/mattermost/accounts.ts:118-123`
  - body: Adding `interactions.secret` makes account-level `interactions` overrides likely, but `mergeMattermostAccountConfig` still only deep-merges `commands`. An account that sets `{ interactions: { secret } }` drops inherited `callbackBaseUrl` and `allowedSourceIps`, which can send callbacks back to localhost or reject non-loopback Mattermost requests. Include `interactions` in the nested merge and add coverage for secret plus inherited reachability config.
  - confidence: 0.86
- **[P1] Publish the new signing-secret contract:** `extensions/mattermost/src/config-schema-core.ts:150`
  - body: This adds a security-sensitive SecretRef field, but the PR does not update the public Mattermost docs, setup/plugin metadata, env-var hints, or generated channel metadata. Operators cannot discover the durable secret required to avoid the process-local fallback, and current docs still describe HMAC verification as automatic with no config. Surface the new contract everywhere config users and setup flows learn Mattermost interaction settings.
  - confidence: 0.88
