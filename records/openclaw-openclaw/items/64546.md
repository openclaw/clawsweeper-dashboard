## Summary

Keep open: this is a real Mattermost interaction-token security fix directionally, but the patch is not merge-ready because the unset `interactions.secret` path becomes process-local random state, the new credential is not wired through setup/SecretRef/docs, and GitHub currently reports the head as conflicting.

## What This Changes

The branch adds `interactions.secret` config resolution for Mattermost accounts and switches button callback HMAC signing to a configured or generated per-account secret.

## Best Possible Solution

Choose and implement an upgrade-safe interaction-secret model, preferably durable configured/generated provisioning or an explicit fail-closed setup requirement, then refresh setup, SecretRef discovery, docs, restart/split-process tests, and the branch merge state together.

## Reproduction Assessment

Yes. Source inspection shows current main derives the callback HMAC key from a public constant plus the bot token, and the PR's upgrade regression is also source-reproducible because the unset-secret fallback is process-local only.

## Solution Assessment

No. Replacing the forgeable derivation is necessary, but an in-memory random fallback plus hidden config is not the best upgrade-safe solution for a callback credential that must be shared across sender/verifier processes and restarts.
