## What This Changes

This PR adds a Mattermost `interactions.secret` config field and replaces bot-token-derived interaction signing with configured or generated per-account secrets in the Mattermost interaction, monitor, and send paths.

## Best Possible Solution

Land a refreshed security fix that removes bot-token-derived signing and either provisions a durable per-account interaction secret through setup, SecretRef/docs/tests, or explicitly fails closed with clear upgrade behavior and restart or split-process proof.

## Reproduction Assessment

Yes, source-level. Current main derives the Mattermost interaction secret from the bot token with a public HMAC key, and PR head replaces the unconfigured path with a process-local random secret.

## Solution Assessment

No. Removing bot-token-derived signing is the right direction, but this is not the best merge shape until the replacement secret is durable or explicitly required across setup, docs, upgrade, and restart or split-process behavior.
