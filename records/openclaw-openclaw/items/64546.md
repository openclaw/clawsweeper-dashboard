## Summary

Keep open. The PR addresses a real Mattermost callback-signing security issue, but it is not merge-ready because the unconfigured fallback is process-local, the new secret contract is not wired through setup/docs/secret management, and GitHub currently reports the branch as conflicting with main.

## What This Changes

The branch adds optional Mattermost account `interactions.secret` config and changes interactive-button HMAC signing to use configured or generated per-account secrets instead of deriving them from the bot token.

## Best Possible Solution

Land a maintainer-approved version that uses a separate durable per-account Mattermost interaction secret, with setup/docs/secret-contract/doctor or explicit fail-closed migration behavior plus proof for upgrade, restart or separate-process callbacks, and old-forgery rejection.

Yes. Source inspection shows current main derives Mattermost interaction signing from the bot token, and PR head generates an unconfigured process-local fallback; I did not run a live Mattermost server reproduction in this read-only review.

## Solution Assessment

No. Removing the bot-token-derived signing root is the right direction, but a process-local generated fallback is not the best fix because it is not durable across the existing CLI/gateway process boundary or restarts.
