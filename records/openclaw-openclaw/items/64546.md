## Summary

This PR should stay open: it addresses a real Mattermost callback-forgery issue, but the current patch is not merge-ready because the replacement signing secret can be process-local, the new credential is not fully provisioned, public docs still describe the old forgeable algorithm, and GitHub reports the branch as conflicting.

## What This Changes

The PR changes Mattermost interaction signing to use a configured or generated per-account interaction secret instead of deriving the secret from the bot token, and updates related schema, runtime, and tests.

## Best Possible Solution

Land a refreshed security fix that removes bot-token-derived signing while making the replacement secret durable or explicitly required across setup, docs, and upgrade behavior.

## Reproduction Assessment

Yes, source-level: current main derives the Mattermost interaction signing secret from a bot token using a public HMAC key, and PR-head source shows the fallback replacement is process-local when no configured secret exists.

## Solution Assessment

No. Removing bot-token-derived signing is the right direction, but the merge shape is incomplete until the replacement secret is durable or explicitly required across setup, docs, upgrade, and restart behavior.

  - body: When `interactions.secret` is unset, this generates a random value held only in the current process.
