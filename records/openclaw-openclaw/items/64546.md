## Summary

Keep this PR open, but not merge-ready: it addresses the Mattermost HMAC forgeability in the right direction while introducing an unproven process-local secret fallback and a new security-sensitive config surface without the required setup, docs, upgrade, and conflict resolution work.

## What This Changes

The PR adds optional Mattermost account `interactions.secret` support and changes button HMAC signing to use a configured or generated per-account secret instead of deriving it from the bot token.

## Best Possible Solution

Land a maintainer-approved version that uses a separate durable per-account Mattermost interaction secret, with setup/docs/secret-contract/doctor or explicit fail-closed migration behavior plus proof for upgrade, restart or separate-process callbacks, and old-forgery rejection.

## Reproduction Assessment

Yes. Source inspection shows current main and `v2026.6.9` still derive Mattermost interaction signing from the bot token with a public HMAC key, and PR head replaces that with configured or generated secrets.

## Solution Assessment

No. Removing the bot-token-derived signing root is the right direction, but a process-local generated fallback is not the best fix because it is not durable across the existing CLI/gateway process boundary or restarts.
