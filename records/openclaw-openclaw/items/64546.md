## Summary

Keep open: the PR targets a real Mattermost callback authorization bug, but the current patch is not safe to merge because its unconfigured random fallback is process-local and the new secret contract lacks setup, docs, doctor/provisioning, and secret-registry coverage.

## What This Changes

The branch adds optional Mattermost `interactions.secret` config and switches button HMAC signing from a bot-token-derived secret to configured or generated per-account secrets.

## Best Possible Solution

Land a maintainer-approved fix that uses a separate durable per-account Mattermost interaction secret, with setup, doctor/docs, secret registry coverage, upgrade behavior, and restart or cross-process proof.

## Reproduction Assessment

Yes. Source inspection shows current main and the latest release derive Mattermost interaction signing from the bot token with a public HMAC key; PR head removes that path but introduces an unconfigured process-local fallback.

## Solution Assessment

No. Removing the bot-token-derived signing root is the right direction, but a process-local generated fallback is not the best fix because it is not durable across the existing CLI/gateway process boundary or restarts.
