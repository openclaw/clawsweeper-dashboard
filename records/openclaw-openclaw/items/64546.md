## Summary

Keep this PR open, but do not merge it as-is: it removes the forgeable bot-token-derived Mattermost HMAC path, yet replaces the default with a process-local random secret and does not wire the new credential through setup, metadata, or docs.

## What This Changes

The PR changes Mattermost interaction signing from a bot-token-derived HMAC secret to a configured or generated per-account interaction secret, with schema, runtime, and test updates.

## Best Possible Solution

Land a refreshed security fix that replaces bot-token-derived signing with a durable configured or provisioned per-account secret, or fails closed with explicit setup and upgrade docs before any buttons are signed.

## Reproduction Assessment

Yes. Source inspection shows current main derives the Mattermost interaction signing secret from the bot token using a public HMAC key, and the PR head shows the unconfigured replacement is process-local.

## Solution Assessment

No. Removing bot-token-derived signing is the right direction, but this is not the best merge shape until the replacement secret is durable or explicitly required across setup, docs, upgrade, restart, and split-process behavior.
