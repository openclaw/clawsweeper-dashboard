## Label Justifications

- `P1`: The PR addresses a real Mattermost callback authorization weakness but currently risks breaking legitimate interactive button workflows.
- `merge-risk: 🚨 compatibility`: Existing Mattermost users without a new `interactions.secret` can lose working callbacks after upgrade, restart, or split CLI/gateway operation.
- `merge-risk: 🚨 message-delivery`: Mismatched sender and gateway secrets can cause valid Mattermost button clicks to be rejected as invalid tokens.
- `merge-risk: 🚨 security-boundary`: The PR changes the Mattermost callback signing boundary and leaves durable-secret provisioning incomplete.

## Summary

Keep this PR open: it addresses the right Mattermost interaction-token security problem and the supplied terminal proof shows old bot-token-derived forgeries are rejected, but the patch is not merge-ready because the fallback secret is process-local and the new durable-secret surface is not wired through setup, secret contracts, or docs.

## What This Changes

The PR replaces Mattermost bot-token-derived interaction signing with configured or generated per-account secrets and updates the schema, tests, send path, monitor path, and account resolution for `interactions.secret`.

## Best Possible Solution

Land a refreshed security fix that removes bot-token-derived signing, uses either a durable per-account interaction secret or an explicit fail-closed setup requirement, wires setup/secret-target/docs coverage, and proves upgrade, restart or split-process callbacks, and old-forgery rejection.

## Reproduction Assessment

Yes, source-level. Current main derives the Mattermost interaction signing secret from the bot token with a public HMAC key, and PR-head source shows the unset-secret path can create process-local secrets independently in sender and gateway paths.

## Solution Assessment

No. Stopping bot-token-derived signing is the right direction, but this is not the best merge shape until the new signing secret is durable or explicitly required and setup, SecretRef, docs, upgrade, and restart behavior are covered.
