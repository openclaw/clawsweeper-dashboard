## Label Justifications

- `P1`: This PR addresses a real Mattermost callback authorization weakness but currently risks breaking legitimate interactive button workflows.
- `merge-risk: 🚨 compatibility`: Existing Mattermost users without a new `interactions.secret` can lose working callbacks after upgrade, restart, or split CLI/gateway operation.
- `merge-risk: 🚨 message-delivery`: Mismatched sender and gateway secrets can cause valid Mattermost button clicks to be rejected as invalid tokens.
- `merge-risk: 🚨 security-boundary`: The PR changes the interaction callback signing boundary and adds a new secret-bearing config field without complete secret-contract, setup, or docs coverage.

## Summary

Keep this PR open, but it is not merge-ready: the security direction is correct and proof covers the local signing primitive, but the patch still leaves a process-local generated secret as the default and does not wire the new Mattermost signing secret through setup, docs, or the secret-target contract.

## What This Changes

The PR replaces Mattermost interaction token signing from a bot-token-derived HMAC secret with configured or generated per-account interaction secrets, updating schema, send/monitor wiring, and tests.

## Best Possible Solution

Land a refreshed security fix that stops bot-token-derived signing and either provisions a durable per-account Mattermost interaction secret through setup/SecretRef/docs or intentionally fails closed with explicit upgrade guidance and proof.

## Reproduction Assessment

Yes, source-level. Current main and v2026.6.10 derive the Mattermost interaction secret from the bot token with a public HMAC key, and PR-head source shows unset `interactions.secret` generates independent in-memory secrets at sender and gateway call sites; I did not run a live Mattermost server.

## Solution Assessment

No. Stopping bot-token-derived signing is the right direction, but this PR is not the best merge shape until the secret is durable or explicitly required and setup, secret-contract, docs, upgrade, and restart behavior are covered.
