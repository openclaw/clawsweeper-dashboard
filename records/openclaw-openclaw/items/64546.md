## Summary

Keep open: the PR targets a real Mattermost interaction-token forgery still present on current main, but the branch is not merge-ready because the replacement unset-secret path is process-local, the new credential is not wired through setup/SecretRef/docs, and GitHub currently reports the branch as conflicting.

## What This Changes

The branch adds Mattermost `interactions.secret` config resolution and switches button callback HMAC signing from bot-token-derived secrets to configured or generated per-account secrets.

## Best Possible Solution

Choose and implement an upgrade-safe Mattermost interaction-secret model, then wire setup, SecretRef targets, docs, restart/split-process tests, and a branch refresh together before merge.

## Reproduction Assessment

Yes. Source inspection shows current main derives the Mattermost interaction HMAC secret from the bot token with a public key, and the PR's unset-secret regression is source-reproducible because generated secrets are process-local.

## Solution Assessment

No. Removing bot-token-derived signing is the right direction, but an in-memory random fallback plus hidden config is not the best upgrade-safe solution for a credential shared by sender and callback verifier.
