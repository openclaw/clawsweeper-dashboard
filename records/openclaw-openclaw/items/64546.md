## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P1] Persist the fallback secret before signing buttons:** `extensions/mattermost/src/mattermost/interactions.ts:198-200`
  - body: When `interactions.secret` is unset, this generates a random secret and keeps it only in this process. `sendMessageMattermost` signs buttons in the sender process, while `monitorMattermostProvider` verifies clicks in the gateway process and calls the same helper separately; with no configured secret, those maps diverge across a normal CLI/gateway split or restart and clicks are rejected as `Invalid token`. Please make the fallback stable via a persisted/generated secret, or fail closed with an explicit configured-secret requirement and upgrade coverage.
  - confidence: 0.88
- **[P2] Update the public button HMAC docs:** `extensions/mattermost/src/types.ts:95`
  - body: This adds `interactions.secret`, but the Mattermost docs still say HMAC verification is automatic with no config and tell external scripts to derive the signing secret from the bot token using the old hardcoded key. After this patch, those tokens no longer match the gateway, and the old derivation remains published as the integration path. Please update the docs for configured/persisted secrets and external script signing.
  - confidence: 0.83
