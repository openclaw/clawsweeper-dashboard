## What This Changes

This PR adds a Mattermost `interactions.secret` config field and changes button interaction signing from bot-token-derived HMAC secrets to configured or generated per-account secrets, with runtime and tests.

## Best Possible Solution

Land a refreshed security fix that removes bot-token-derived signing and either provisions a durable per-account interaction secret through setup, SecretRef contracts, docs, and tests, or explicitly fails closed with clear upgrade behavior and proof.

## Reproduction Assessment

Yes, source-level. Current main derives the Mattermost interaction secret from the bot token with a public HMAC key, and the PR-head replacement fallback is process-local when no configured secret exists.

## Solution Assessment

No. Removing bot-token-derived signing is the right direction, but this is not the best merge shape until the replacement secret is durable or explicitly required across setup, docs, upgrade, and restart or split-process behavior.
