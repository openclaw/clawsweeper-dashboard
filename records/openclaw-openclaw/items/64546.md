## Summary

This PR addresses a real Mattermost callback-token forgeability bug, but it is still not merge-ready because the unconfigured fallback is process-local and can break legitimate button callbacks across CLI/gateway process boundaries or gateway restarts.

## What This Changes

The branch adds optional Mattermost `interactions.secret` config and changes interaction signing from bot-token-derived HMAC secrets to configured or generated per-account secrets.

## Best Possible Solution

Land a maintainer-approved version that uses a separate durable per-account Mattermost interaction secret, with setup/docs/doctor or explicit fail-closed migration behavior and proof for old-forgery rejection plus restart or separate-process verification.

## Reproduction Assessment

Yes. Source inspection shows current main still derives Mattermost interaction signing from the bot token with a public HMAC key, and PR head replaces that with a process-local fallback when no configured secret exists; I did not run a live Mattermost server reproduction in this read-only review.

## Solution Assessment

No. Removing the bot-token-derived signing root is the right direction, but a process-local fallback is not the best fix because it is not durable across the existing CLI/gateway process boundary or restarts.
