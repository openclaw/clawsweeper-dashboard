## Summary

Keep open: current main and the latest release still contain the reported Mattermost bot-token-derived HMAC, but this PR is not merge-ready because the replacement uses a process-local fallback and adds an unprovisioned secret surface.

## What This Changes

The branch changes Mattermost interactive-button signing to use a configured or generated interaction secret, adds an account `interactions.secret` schema/type, and updates Mattermost interaction/send tests.

## Best Possible Solution

Land a mergeable security fix that removes bot-token-derived signing while either provisioning a durable per-account interaction secret or failing closed with explicit setup, doctor, docs, upgrade, and restart proof.

## Reproduction Assessment

Yes. Source inspection shows current main and `v2026.6.9` still derive Mattermost interaction signing from the bot token with a public HMAC key; I did not run a live Mattermost server reproduction in this read-only review.

## Solution Assessment

No. Removing the bot-token-derived signing root is the right direction, but process-local generation is not the best fix because it is not durable across the existing send/gateway process boundary or restarts.
