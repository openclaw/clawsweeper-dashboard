## Summary

## What This Changes

The branch changes Mattermost interaction signing to use a configured or generated per-account interaction secret instead of deriving it from the bot token, and updates related schema, runtime, and tests.

## Best Possible Solution

Land a refreshed security fix that removes bot-token-derived signing while making the replacement secret durable or explicitly required through setup, secret registry, docs, and upgrade proof.

## Reproduction Assessment

Yes, source-level. Current main derives the Mattermost interaction signing secret from the bot token with a public HMAC key, and PR-head source shows the replacement fallback is process-local when no configured secret exists.

## Solution Assessment

No. Removing bot-token-derived signing is the right direction, but this is not the best merge shape until the replacement secret is durable or explicitly required across setup, docs, upgrade, and restart behavior.
