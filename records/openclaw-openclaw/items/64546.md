## What This Changes

This PR changes Mattermost interaction signing from a bot-token-derived HMAC secret to a configured or generated per-account interaction secret, with related schema/runtime/test updates.

## Best Possible Solution

Land a refreshed security fix that removes bot-token-derived signing and either provisions a durable per-account interaction secret through setup/secret/docs, or explicitly fails closed with clear upgrade behavior and proof.

Yes, source-level: current main and `v2026.6.10` derive the Mattermost interaction signing secret from the bot token with a public HMAC key, and the PR head shows the fallback replacement is process-local when no configured secret exists.

No. Removing bot-token-derived signing is the right direction, but this is not the best merge shape until the replacement secret is durable or explicitly required across setup, docs, upgrade, and restart behavior.
