## Summary

Keep open: the security direction is valid, but the PR is not merge-ready because the unset-secret fallback becomes process-local and can break Mattermost button callbacks after restart or across split processes, while the new signing credential is not wired through setup, docs, or SecretRef contracts.

## What This Changes

The PR adds Mattermost `interactions.secret` config support and changes interaction token signing from bot-token-derived HMAC secrets to configured or generated per-account secrets.

## Best Possible Solution

Land a refreshed security fix that removes bot-token-derived signing and either provisions a durable per-account interaction secret through setup, SecretRef contracts, docs, and tests, or explicitly requires one with clear upgrade proof.

## Reproduction Assessment

Yes. Current main source still derives the Mattermost interaction secret from the bot token with a public HMAC key, and the PR-head replacement fallback is clearly process-local when no configured secret exists.

## Solution Assessment

No. Removing bot-token-derived signing is the right direction, but the merge shape is not safe until the replacement secret is durable or explicitly required across setup, docs, upgrade, and restart or split-process behavior.
