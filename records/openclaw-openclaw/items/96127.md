## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Do not forward wrapper fields as target args:** `src/agents/tool-search.ts:1690`
  - body: The fallback only removes `id`, but `readId` also accepts `toolId` and `name`, and nullish wrapper keys like `args: null` can remain in `rest`. A flattened call such as `{ toolId: "openclaw:core:exec", command: "pwd" }` would now pass `toolId` through to the target input, which can break strict MCP/client tools or tools with colliding parameter names. Track the selector key that was used and drop wrapper metadata before forwarding the target args, with regression coverage for the alias/nullish cases.
  - confidence: 0.84
