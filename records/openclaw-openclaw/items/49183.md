## Summary

Keep open. Current main has useful sender-scoped, group-scoped, inherited, and owner-only tool policy primitives, but it still does not define a scope-keyed config contract or thread authenticated Gateway operator scopes into the effective agent tool-policy resolver.

## What This Changes

Request a session/operator-scope policy layer that links Gateway auth scopes with agent tool availability.

## Best Possible Solution

Define a reviewed operator/session-scope policy contract, thread authenticated Gateway scopes into the shared tool-policy path, document precedence against sender, inherited, and owner-only gates, and add regression coverage for distinct read/write/admin tool surfaces.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an RFC for a new authorization/config surface. Source inspection gives a high-confidence verification path: current main lacks `toolsByScope` and has no caller-scope parameter in effective tool-policy resolution.

## Solution Assessment

Unclear. The proposed direction is plausible, but maintainers still need to choose the config home, precedence, and fail-closed enforcement layer before implementation.
