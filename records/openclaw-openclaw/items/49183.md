## Summary

Keep open: current main and v2026.6.10 still have gateway operator scopes and layered agent tool policies, but no `toolsByScope` config or operator-scope input in runtime tool-policy resolution. The remaining work is a security/product decision about whether gateway scopes should become a tool-authorization boundary and how that composes with existing sender, group, provider, sandbox, subagent, and inherited policy layers.

## What This Changes

Requested session-level tool filtering that connects gateway operator scopes to agent tool policy resolution for multi-user deployments.

## Best Possible Solution

Keep this RFC open until maintainers decide the operator-scope trust boundary and policy precedence; if accepted, split it into additive config/schema, gateway scope handoff, policy-pipeline, docs, and regression-test slices.
