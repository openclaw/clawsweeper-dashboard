## Summary

Keep open. Current main now has useful sender-scoped tool policies, but it still does not bridge authenticated gateway operator scopes into the shared agent tool-policy resolver; the remaining RFC needs maintainer security/product direction.

## What This Changes

The issue requests session-level tool policy overrides that map gateway operator scopes such as operator.read/write/admin to effective agent tool access.

## Best Possible Solution

Define a reviewed operator/session-scope policy contract, thread authenticated gateway scopes into the shared tool-policy path, document precedence against sender and owner-only gates, and add regression coverage for distinct read/write/admin tool surfaces.

## Reproduction Assessment

Not applicable as a bug reproduction; this is an RFC for a new policy surface. Source inspection gives a high-confidence verification path: current main has sender-scoped policies, but no scope-keyed config field or caller-scope parameter in the effective tool-policy resolver.

## Solution Assessment

Unclear. The proposed toolsByScope shape is plausible, but maintainers still need to choose the config home, merge precedence, enforcement layer, and interaction with sender identity and owner-only tools.
