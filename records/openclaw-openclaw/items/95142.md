## Summary

Keep open. Current main and v2026.6.10 still lack WhatsApp delivered-message pinning and a WhatsApp group-description message action, and the remaining blocker is the security/product policy for exposing persistent WhatsApp admin mutations through the message tool.

Request support for WhatsApp/Baileys delivered-message pinning with duration selection and a message action that updates a WhatsApp group description by JID.

## Best Possible Solution

Implement this in the WhatsApp plugin with explicit admin/action authorization, duration validation, group-JID validation, docs, focused tests, and redacted live WhatsApp proof.

## Reproduction Assessment

Yes, at source level: current main has no WhatsApp `pinDeliveredMessage` hook and no WhatsApp group-description message action. I did not run live WhatsApp proof because the missing surfaces are clear in source and in v2026.6.10.

## Solution Assessment

Unclear until maintainers choose the policy gate. The maintainable shape is WhatsApp-plugin-owned support with explicit authorization, not a core workaround or ungated message-tool mutation.
