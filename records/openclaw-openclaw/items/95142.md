## Summary

Keep open: current main and v2026.6.8 still lack WhatsApp delivered-message pinning, pin duration support, and a WhatsApp group-description message action. The request is valid feature work, but it needs maintainer API and authorization direction before an automated fix PR should start.

## What This Changes

Request adding WhatsApp/Baileys support for delivery-pin duration and a message action to update a group description.

## Best Possible Solution

Keep this in WhatsApp plugin/product review: implement pinning through the existing outbound hook with an agreed duration contract, then add a gated group-description action backed by Baileys listener APIs with docs, tests, and live proof.

## Reproduction Assessment

Yes at source level: current main has the generic delivery-pin hook and degradation path, but WhatsApp does not implement the hook or advertise pin support, and no action path reaches groupUpdateDescription. I did not run a live WhatsApp account, so this is source-reproducible rather than live-reproduced.

## Solution Assessment

Unclear as a single automated fix: the pin hook belongs cleanly in the WhatsApp plugin, but duration semantics and group-description exposure need maintainer decisions about API shape, action naming, and admin authorization.
