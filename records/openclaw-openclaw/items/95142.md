## Summary

Keep open: current main and the latest release still lack WhatsApp delivery-pin support and a WhatsApp group-description message action. ## What This Changes

Request support for WhatsApp/Baileys delivered-message pinning with selectable duration and a message action to update a WhatsApp group description.

## Best Possible Solution

Keep this issue open for maintainer-scoped WhatsApp plugin work: add delivered-message pinning through the existing outbound hook, then design a gated group-description action with explicit admin authorization and tests.

## Reproduction Assessment

Yes at source level: current main has the generic delivery-pin hook and degradation path, but WhatsApp does not implement the hook or advertise pin support, and no action path reaches groupUpdateDescription. I did not run a live WhatsApp account, so this is source-reproducible rather than live-reproduced.

## Solution Assessment

Unclear as a single automated fix: the pin hook belongs cleanly in the WhatsApp plugin, but duration semantics and the group-description action need maintainer decisions about shared API shape, action naming, and group-admin authorization.
