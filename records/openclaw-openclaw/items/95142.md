The issue requests WhatsApp/Baileys support for 24h/7d/30d delivered-message pins and setting WhatsApp group descriptions through OpenClaw messaging surfaces.

## Best Possible Solution

Add WhatsApp-plugin-owned pin and group-description support behind an explicit admin/action authorization gate, reusing the existing core delivery-pin seam and channel message-action discovery, with focused tests and live WhatsApp proof before broad enablement.

## Solution Assessment

Unclear until maintainers choose the policy gate. The maintainable shape appears to be WhatsApp-plugin-owned support through existing channel/message seams, not a core workaround or ungated mutation action.
