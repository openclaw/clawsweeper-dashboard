## Summary

Keep open. Current main and latest release still lack the WhatsApp delivery-pin hook and any WhatsApp group-description message action; related discussion includes useful references in a closed unmerged PR and a fork, but the remaining work needs maintainer product/security direction before exposing persistent WhatsApp group mutations through a linked account.

## What This Changes

Request WhatsApp/Baileys support for delivered-message pinning with duration selection and a message action to update group descriptions.

## Best Possible Solution

Implement this in the WhatsApp plugin with an explicit admin/action authorization gate, a narrow delivery-pin duration contract, focused tests, docs, and redacted live WhatsApp proof.

## Reproduction Assessment

Yes, at source level: current main exposes no WhatsApp `pinDeliveredMessage` hook and rejects non-`react`/`upload-file` WhatsApp actions, while v2026.6.9 has the same gap. I did not run a live WhatsApp probe because the missing implementation surfaces are clear from source and tag inspection.

## Solution Assessment

Unclear until maintainers choose the policy gate. The maintainable path is a WhatsApp-plugin implementation with explicit admin/action authorization and a narrow delivery-pin duration contract, not an ungated core workaround.
