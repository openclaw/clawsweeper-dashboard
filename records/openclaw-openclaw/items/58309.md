## Summary

Keep open. Current main still has the raw JSON parse behavior this PR is trying to improve, but the patch is not merge-ready because the added device identity atomic-write helper defaults to world-readable permissions and the remaining Tailscale funnel parse path is still outside the new diagnostic handling.

## What This Changes

The PR wraps selected archive, Tailscale, and delivery-queue JSON.parse calls with contextual errors and adds atomic writes for device identity files.

## Best Possible Solution

Keep the useful JSON diagnostic work, but finish it as a focused infra fix: make identity writes private-by-default or reuse the existing secure atomic JSON helper, cover the funnel parse path, add focused regression tests, and include the required changelog entry if the behavior stays user-facing.

## Reproduction Assessment

Yes. Current main has direct JSON.parse calls in the targeted helpers, so malformed archive JSON, corrupt queue files loaded through delivery queue APIs, and malformed Tailscale JSON output give a high-confidence reproduction path for the diagnostic gap.

## Solution Assessment

No. The proposed direction is reasonable, but this exact patch is not the best final form until the key-material helper cannot default to 0o644 and the remaining Tailscale funnel parse path, tests, and changelog coverage are addressed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.85
