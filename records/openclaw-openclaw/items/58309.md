## Summary

Keep this PR open, but do not merge it as-is. The JSON diagnostic changes address real current-main gaps, while the added device-identity atomic-write helper is security-sensitive and defaults omitted modes to `0o644`.

## What This Changes

The PR wraps selected archive, Tailscale, and outbound delivery queue JSON parsing with contextual errors and adds synchronous atomic writes for device identity persistence.

## Best Possible Solution

Keep the useful JSON diagnostics, but finish the patch as a focused infra fix with private-by-default identity writes, the remaining Tailscale funnel parse path covered, regression tests, and a changelog entry if the behavior remains user-facing.

## Reproduction Assessment

Yes. Current main has direct `JSON.parse` calls in the targeted archive helper, outbound queue reader, and Tailscale helpers, so malformed files or mocked malformed Tailscale stdout provide a high-confidence reproduction path for the diagnostic gap.

## Solution Assessment

No. The direction is reasonable, but this exact patch is not the best final form until the key-material helper cannot default to `0o644` and the remaining parse/test/changelog gaps are handled.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.85

Full review comments:

- **[P2] Use a private default for identity writes:** `src/infra/device-identity.ts:47`
  - body: The new helper is documented for cryptographic key material but falls back to `0o644`. Current callers pass `0o600`, but any future caller that omits `mode` would create identity data with a world-readable mode, so the helper should default to `0o600` or require an explicit mode.
  - confidence: 0.84

## Security Review
