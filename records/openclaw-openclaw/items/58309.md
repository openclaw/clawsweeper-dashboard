## Summary

Keep open. The JSON diagnostic direction is useful, but current `main` has already moved several targeted paths to `fs-safe`/private file-store helpers, while this PR still adds a security-sensitive device-identity helper with a `0o644` default and has no real after-fix behavior proof.

## What This Changes

This PR wraps selected archive, Tailscale, and outbound delivery queue JSON parsing with contextual errors and adds a synchronous atomic-write helper for device identity persistence.

## Best Possible Solution

Rebase and narrow onto current `main`: keep any still-needed Tailscale JSON diagnostics, rely on `fs-safe`/private file-store for the superseded paths, remove the `0o644` key-material helper footgun, and add focused regression proof.

## Reproduction Assessment

Yes, source-reproducible. Current `main` still has direct Tailscale `JSON.parse` paths that can be exercised with malformed mocked stdout, while other targeted paths are now partly covered by shared helpers.

## Solution Assessment

No, not as-is. The intended diagnostic fix is reasonable, but this patch mixes in key-material persistence changes and overlaps with newer `main` infrastructure, so a smaller rebased fix is the maintainable path.
