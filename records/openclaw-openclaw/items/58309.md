## Summary

Keep open. The JSON diagnostic work is useful and still absent from current main, but this PR also touches private-key persistence and introduces a reusable atomic-write helper whose fallback mode is too permissive for key material.

## What This Changes

This PR wraps selected archive, Tailscale, and delivery-queue JSON parsing with contextual errors and refactors device identity writes through a sync temp-file rename helper.

## Best Possible Solution

Keep the JSON diagnostics, but either split the device-identity refactor for explicit security review or make it private-by-default, cover the remaining Tailscale funnel parse path, add focused tests, and add the changelog entry.

## Reproduction Assessment

Yes. Current main has direct `JSON.parse` calls in the targeted archive, Tailscale, and delivery queue helpers, so malformed JSON in those inputs gives a high-confidence reproduction path for the diagnostic gap.

## Solution Assessment

No. The direction is reasonable, but this exact patch is not the best final form until key-material writes cannot fall back to `0o644` and the remaining parse path, tests, and changelog coverage are handled.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Use a private default for identity writes:** `src/infra/device-identity.ts:47`
  - body: The new helper is documented for cryptographic key material but falls back to `0o644` when `options.mode` is omitted. Current callers pass `0o600`, but the helper is reusable and a future caller could create readable private-key material by omission, so default to `0o600` or require a mode.
