## What This Changes

The issue reports that Matrix room messages with valid `m.mentions.user_ids` can be skipped when the formatted-body visible label is `@[Display Name]`.

## Best Possible Solution

Review and land the linked focused Matrix PR if it remains correct and green, then close this issue through that merged fix.

## Reproduction Assessment

Yes. From source inspection, a message whose body and formatted-body anchor label are `@[Display Name]` with `m.mentions.user_ids` containing the bot MXID misses current visible-label candidates and reaches the `no-mention` room skip path.

## Solution Assessment

Yes. The narrow maintainable solution is to add the bracketed display-name candidate while preserving the existing visible-label guard; treating `m.mentions.user_ids` alone as sufficient is broader than needed for this issue.
