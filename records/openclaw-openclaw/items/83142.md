## Summary

Keep open. Current main still has the source-reproducible Matrix mention gap, and the issue is already paired with an open fixing PR that uses closing syntax, so the issue should remain open until that PR is reviewed and merged or explicitly rejected.

## What This Changes

Matrix room messages with valid `m.mentions.user_ids` can be skipped when the formatted `matrix.to` label is `@[Display Name]`.

## Best Possible Solution

Review and land the existing open fix PR https://github.com/openclaw/openclaw/pull/83156 if CI and maintainer review stay clean, keeping the visible-label guard while accepting the bracketed display-name form.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: `@[Display Name]` is absent from the accepted Matrix visible-label candidates, so a required room mention with that label can reach the `no-mention` skip path on current main.

## Solution Assessment

Yes. The narrow maintainable solution is to add `@[displayName]` to the existing visible-label and prefix candidate handling; making `m.mentions.user_ids` sufficient by itself would broaden the current anti-spoof policy.
