## Summary

Keep open: current main and the latest v2026.5.12 release still accept `Display Name` and `@Display Name` visible Matrix mention labels, but not `@[Display Name]`, so a valid `m.mentions.user_ids` event can still be dropped as `no-mention` in require-mention rooms.

## What This Changes

The issue reports that Matrix room messages with a valid `m.mentions.user_ids` entry can be ignored when the visible mention label is formatted as `@[Display Name]`.

## Best Possible Solution

Add bracketed display-name support in Matrix's visible-label and mention-prefix helpers, with regression tests, while preserving the current hidden-metadata-only and `@room` safeguards unless maintainers explicitly revise that policy.

## Reproduction Assessment

Yes, at source level. A formatted-body anchor whose href targets the bot MXID but whose visible label normalizes to `@[display name]` misses the current candidate list, leaving `wasMentioned` false and causing the room handler to skip the message as `no-mention`.

## Solution Assessment

Yes for the narrow repair: accepting `@[displayName]` alongside existing display-name candidates is the smallest maintainable fix. Making `m.mentions.user_ids` authoritative without visible-label validation would be a broader policy change that conflicts with current tests and changelog wording.
