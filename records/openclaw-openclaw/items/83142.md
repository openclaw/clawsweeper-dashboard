## Summary

Keep open. Current main still misses the reported `@[Display Name]` Matrix mention shape, and https://github.com/openclaw/openclaw/pull/83156 is an open fix PR that references this issue with closing syntax, so this should remain open until that PR lands or is replaced.

## What This Changes

The issue asks Matrix room mention handling to accept events whose `m.mentions.user_ids` contains the bot MXID when the visible Matrix URI label is `@[Display Name]`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/83156, or replace it with an equivalent narrow fix that accepts `@[displayName]` while preserving metadata-only spoof protection.

## Reproduction Assessment

Yes. From source inspection, a Matrix room event with the bot MXID in `m.mentions.user_ids` and an anchor label of `@[Display Name]` misses the current visible-label candidates and reaches the room `no-mention` skip path when mention gating is required.

## Solution Assessment

Yes. The narrow maintainable fix is to accept the bracketed display-name label while preserving the existing visible-label guard; making `m.mentions.user_ids` sufficient by itself is broader than needed for this report.
