## What This Changes

The issue reports Matrix room messages being skipped as `no-mention` when `m.mentions.user_ids` contains the bot MXID but the visible link label is `@[Display Name]`.

## Best Possible Solution

Land or close the linked open PR after normal maintainer/CI review; the end state should add `@[displayName]` to Matrix visible-label and prefix candidates while preserving metadata-only spoof protection.

## Reproduction Assessment

Yes. Source inspection on current main shows the exact `@[Display Name]` label is absent from the candidate lists, and the handler still takes the `no-mention` skip path when `resolveMentions` returns false.

## Solution Assessment

Yes. The narrow maintainable solution is to accept the bracketed display-name label in the existing candidate lists and regression tests, rather than making protocol metadata alone bypass the visible-label guard.
