## Summary

Keep open. Current main still lacks the bracketed display-name Matrix mention candidate, so the reported room message can be skipped as `no-mention`; the open linked pull request is the right implementation path and should close this only after it merges.

## What This Changes

The issue asks Matrix mention detection to accept a valid bot mention when `m.mentions.user_ids` names the bot but the visible formatted link label is `@[Display Name]`.

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/83156 or an equivalent narrow parser/test fix that accepts `@[displayName]` while preserving the existing visible-label guard.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path: a Matrix room event with `m.mentions.user_ids` for the bot and formatted label `@[Display Name]` misses the candidate list, leaving `wasMentioned` false and reaching the `no-mention` skip path.

## Solution Assessment

Yes. The narrow maintainable fix is to add `@[displayName]` to the existing Matrix visible-label and prefix candidates, with regression coverage, instead of changing the broader metadata-only trust policy.
