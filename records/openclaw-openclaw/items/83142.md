## What This Changes

## Best Possible Solution

Review and land https://github.com/openclaw/openclaw/pull/83156, preserving the existing visible-label guard while adding `@[displayName]` parser and prefix-stripping coverage.

## Reproduction Assessment

Yes. The sanitized event is source-reproducible because current main excludes `@[Display Name]` from visible mention candidates and the Matrix room handler drops unrecognized required room mentions as `no-mention`.

## Solution Assessment

Yes. The narrow maintainable fix is to add `@[displayName]` to the existing visible-label and prefix candidate lists while keeping the metadata-only spoof guard intact.
