## Summary

Keep open. Current main and the reported v2026.5.12 release can skip the sanitized Matrix event because the mention parser requires a recognized visible label but does not include the `@[Display Name]` form among accepted display-name labels.

## What This Changes

Treat Matrix room messages as bot mentions when valid `m.mentions.user_ids` metadata is paired with a visible `@[Display Name]` Matrix link label.

## Best Possible Solution

Keep the existing visible-label safety guard, add `@[displayName]` as an accepted Matrix URI label, and cover the sanitized event in focused parser and handler tests.

## Reproduction Assessment

Yes, from source inspection: with `body` and anchor label `@[Display Name]`, `m.mentions.user_ids` containing the bot MXID, and no matching custom regex, current candidates miss the bracketed label so the room handler reaches the `no-mention` skip path. I did not execute tests because this review was read-only.

## Solution Assessment

Yes, the narrow maintainable fix is to accept the bracketed display-name label while preserving the existing metadata-only spoof protection. Making `m.mentions.user_ids` sufficient by itself is broader than needed here.
