## Summary

Keep this open: current main still supports/recommends Node 24, still pins the affected Pi package family, and the published Pi package metadata matches the Node 24 `ERR_PACKAGE_PATH_NOT_EXPORTED` failure mode when an import-only export is reached through CommonJS resolution.

## What This Changes

The issue reports Node.js v24 WebChat failures caused by `@mariozechner/pi-*` package export metadata when the embedded Pi runtime is loaded through a CommonJS path.

## Best Possible Solution

Keep the issue open and fix Node 24 support at the Pi package boundary with an approved export/dependency change, then prove the installed WebChat or Control UI chat path on Node 24.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Not as a full WebChat run in this checkout, but yes at the source/dependency boundary: Node v24.14.1 rejects the same import-only export shape through `createRequire()`, and current main still depends on those packages.

## Solution Assessment

Is this the best way to solve the issue? No for the suggested `node_modules` patch, because it is temporary and lost on reinstall; the maintainable fix is an approved upstream/default export update, dependency migration, or runtime import-path change with installed-package proof.
