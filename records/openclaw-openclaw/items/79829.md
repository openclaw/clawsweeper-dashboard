## What This Changes

## Best Possible Solution

Keep this open and fix Node 24 support at the Pi package boundary with an approved upstream package/export change or a maintainer-approved dependency patch, plus a regression check that exercises the installed-package WebChat agent path.

Do we have a high-confidence way to reproduce the issue? Not as a full WebChat run in this checkout, but the source/dependency evidence is strong: current main still uses the affected Pi packages and Node v24 demonstrably rejects import-only exports when they are loaded through `createRequire()`.

## Solution Assessment

Is this the best way to solve the issue? No for the temporary `node_modules` patch, because it is lost after reinstall; the durable fix should be an approved dependency/export update or upstream package replacement with installed-package runtime proof.
