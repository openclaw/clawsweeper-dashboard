## Summary

Keep open: current main and the latest published release still use the affected Pi package family, Node 24 is a supported/recommended runtime, and the published Pi package metadata reproduces the import-only export failure when resolved through CJS interop.

## What This Changes

The issue asks OpenClaw to fix Node 24 WebChat/embedded-agent failures caused by `@mariozechner/pi-*` package exports that lack CJS-compatible `default` or `require` conditions.

## Best Possible Solution

Fix Node 24 support at the Pi package boundary by eliminating CJS resolution of ESM-only Pi packages or by using an approved upstream/package-export update, then prove the installed WebChat or Control UI chat path on Node 24.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Yes for the dependency-resolution failure: Node v24.14.1 with the actual 0.73.1 package metadata throws `ERR_PACKAGE_PATH_NOT_EXPORTED` through `createRequire()`, though a full WebChat run was not performed in this read-only checkout.

## Solution Assessment

Is this the best way to solve the issue? No for the proposed `node_modules` patch because it is reinstall-fragile; the maintainable fix should be an approved dependency/export or loader-path change with installed-package runtime proof.
