Keep open: Node 24 is a documented OpenClaw runtime, current main still depends on the affected Pi package family, and the registry export metadata matches the reported Node resolver failure when CJS interop is used.

## What This Changes

The issue reports that WebChat fails under Node.js 24 because bundled `@mariozechner/pi-*` packages cannot be loaded through CJS interop due to package export metadata.

## Best Possible Solution

Approve a Pi package-boundary strategy that avoids CJS resolution of import-only Pi exports or updates the package export maps upstream, then prove WebChat or Control UI chat sends under Node 24 from an installed package.

## Reproduction Assessment

Do we have a high-confidence way to reproduce the issue? Partially: the Node 24 `createRequire()` failure is reproducible against the same import-only export shape, and source inspection shows current main still uses those Pi packages. A full WebChat turn still needs installed-package runtime proof.

## Solution Assessment

Is this the best way to solve the issue? No for the reported temporary `node_modules` patch because reinstalling loses it. The maintainable fix should be an approved dependency/export or loader-boundary change with package-install runtime proof.
