## Summary

Current main already implements the requested cron Telegram topic support: cron add/create and cron edit expose `--thread-id`, pass it as `delivery.threadId`, validate numeric values, document the user flow, and have regression coverage. The earlier contributor PRs were carried forward into merged PR #73379, so the open fork follow-up is superseded by the implementation now on main.

## What This Changes

Add first-class `--thread-id` support to `openclaw cron add`/`create` and `openclaw cron edit` so Telegram cron deliveries can target forum topics.

## Best Possible Solution

Close this as implemented and keep the merged main-line cron CLI behavior, with any remaining adjacent ideas such as malformed slash-target validation or per-run topic creation tracked as separate focused issues.
