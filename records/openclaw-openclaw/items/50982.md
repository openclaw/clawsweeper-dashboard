## Summary

Current main and the latest stable release already implement the requested cron Telegram topic flag, with CLI add/edit support, typed delivery.threadId plumbing, docs, and regression coverage. The related contributor PRs were carried forward into a merged maintainer PR, so the remaining open issue is cleanup rather than active work.

## What This Changes

Add Telegram topic `--thread-id` support to cron add/edit delivery so scheduled Telegram messages can target forum topics.

## Best Possible Solution

Keep the shipped v2026.5.18 implementation; if cron list display or the failure-notification cascade from the later comment still reproduces, track that as a narrower follow-up with redacted job config and logs.

## Reproduction Assessment

Yes from source and release history. Older release v2026.4.19-beta.2 lacked the cron flag, while current main and v2026.5.18 expose `--thread-id` on cron add/edit with regression tests.

## Solution Assessment

Yes. The shipped fix adds the requested CLI flag and uses the existing typed `delivery.threadId` delivery contract, which is the narrow maintainable path rather than requiring users to hand-encode Telegram target strings.
