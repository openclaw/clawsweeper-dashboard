## Summary

Current main and the latest stable release already support cron `--thread-id` for Telegram topic delivery, with schema acceptance, CLI coverage, and delivery-path propagation tests; the older unmerged PR attempts are superseded.

## What This Changes

The issue requests `--thread-id` support for cron add/edit so Telegram cron delivery can target forum topics.

## Best Possible Solution

Keep the shipped cron CLI and delivery-thread implementation; track any remaining invalid-target or failure-notification edge case under a narrower follow-up rather than this completed flag request.

## Reproduction Assessment

Not applicable for current main: this is a feature gap report, and source inspection shows the requested cron `--thread-id` flag and delivery propagation are now present. I did not run the CLI because this review was required to keep the checkout read-only.

## Solution Assessment

Yes. The shipped implementation adds the CLI flag, keeps the value in the existing `delivery.threadId` contract, validates it through the gateway schema, and reuses the established cron delivery path instead of adding Telegram-specific core policy.
