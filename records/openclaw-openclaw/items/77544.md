## Summary

Keep open: current main has the reported npm update skip path, and the PR is headed in the right direction, but the patch should verify the actual `openclaw` peer link target before skipping and add the required changelog entry.

## What This Changes

The PR adds a package-manifest helper and uses it during npm plugin updates to bypass unchanged-install skips when an installed plugin declares `peerDependencies.openclaw` but lacks `node_modules/openclaw`, with regression tests.

## Best Possible Solution

Keep the PR open, require the peer-link health check to prove the path resolves to the current host OpenClaw package before skipping, and add focused regression coverage plus a changelog entry.

## Reproduction Assessment

Yes from source inspection: current main can skip the npm installer when version and integrity match, and the installer is the path that creates the `openclaw` peer link. I did not execute the repro because this review is read-only.

## Solution Assessment

No. Bypassing the unchanged skip is the right fix shape, but the proposed helper should validate the peer link resolves to the host OpenClaw package rather than accepting any existing path.
