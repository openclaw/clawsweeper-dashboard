## What This Changes

The PR wraps status text/reply generation with degraded fallbacks, changes five status lazy imports to catch failures as `undefined`, and adds two status-text smoke tests.

## Best Possible Solution

Land a type-safe status fallback that clears failed lazy-import caches for retry, handles optional status sections explicitly, and either covers or defers the linked LINE timeout/fallback work while keeping the issue open until real LINE/runtime proof exists.

## Reproduction Assessment

Yes for source-level reproduction, no for live LINE. Current main has an await chain from `handleStatusCommand` to `buildStatusReply` to uncaught lazy status imports, so an import rejection can escape before a reply payload.

## Solution Assessment

No. The top-level guard direction is useful, but the PR is not the best fix until failed lazy imports retry instead of caching `undefined`, optional modules are handled type-safely, and the linked LINE proof/scope is addressed.
