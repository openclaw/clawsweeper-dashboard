## Summary

Keep open: the PR addresses a real Codex app-server stall gap, but the new transport watchdog is still refreshed by unscoped shared-client traffic, so a concurrent turn can mask the stalled turn it is meant to catch; it also needs the required changelog entry.

## What This Changes

The branch adds a 5-minute Codex app-server transport-idle watchdog, timeout flag propagation for retry policy, and regression tests for idle, wallclock, and normal completion outcomes.

## Best Possible Solution

Land a current-turn-scoped 5-minute transport-idle watchdog with idle timeout flag plumbing, concurrency regression coverage, and an `Unreleased` Fixes changelog entry.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: start run A, deliver a notification or request for thread B through the shared Codex app-server client before the new deadline, and run A's transport timer is refreshed even though its own turn is silent.

## Solution Assessment

No. The watchdog and timeout-flag direction is sound, especially after the 5-minute retune, but the reset path must be scoped to current-turn activity and the user-facing fix needs a changelog entry before merge.
