## Summary

Current main and shipped releases now include the Apple Silicon Homebrew paths in generated macOS LaunchAgent environments, which resolves the plausible `imsg` PATH regression behind the reported iMessage polling failure after upgrade.

## What This Changes

The issue reports that iMessage inbound messages reach `chat.db` after upgrading to 2026.5.3-1, but the gateway no longer polls or replies automatically.

## Best Possible Solution

Keep the shipped LaunchAgent PATH fix and ask for a fresh report only if current releases still fail with `imsg` resolvable from the gateway service environment or an explicit `channels.imessage.cliPath`.

## Reproduction Assessment

Yes for the historical regression: source and release history show affected macOS service envs could omit `/opt/homebrew/bin`, and the related iMessage report confirms that made `imsg` unavailable to the LaunchAgent. I did not live-run a macOS Messages account for this cleanup review.

## Solution Assessment

Yes. The narrow maintainable fix is to include stable Apple Silicon Homebrew directories in generated macOS service PATHs while preserving the existing guard against inheriting arbitrary shell or workspace PATH entries.
