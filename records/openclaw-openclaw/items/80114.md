## Summary

Keep open: current main fixes the older schema-validator mismatch, but the reported agent-facing `gateway config.patch` path is still protected by design, the linked fix PR was closed unmerged after a security-boundary review, and the upgrade/local-plugin policy preservation part is not clearly implemented by a shipped change.

## What This Changes

The issue asks OpenClaw to keep local plugin `hooks.allowConversationAccess` working across upgrades and provide a supported way to set it when `agent_end` needs raw conversation access.

## Best Possible Solution

Preserve or restore local plugin hook policy through upgrade/doctor/install config-management paths while keeping raw conversation access grants explicit, host-approved, and not agent-mutable without a maintainer security decision.

## Reproduction Assessment

Partially yes. Source and existing tests reproduce the core blocked `agent_end`/agent-facing `config.patch` behavior, but the exact upgrade reset was not live-reproduced under the read-only review constraints.

## Solution Assessment

No for the linked allowlist PR direction. The safer fix is to preserve host-approved hook policy during upgrade/config repair, or settle the manifest/state-access contract separately, rather than making the agent-facing Gateway tool able to grant raw conversation access.
