Keep open: current main still blocks the reported agent-driven `gateway config.patch` path for this hook policy field, and an open linked PR with closing syntax should be reviewed or replaced before this issue closes.

The issue asks OpenClaw to preserve or allow configuration of `plugins.entries.<id>.hooks.allowConversationAccess` for local/community plugins so conversation hooks such as `agent_end` are not blocked after upgrades.

## Best Possible Solution

Resolve the open linked PR with an explicit maintainer boundary decision: preserve local plugin hook policy through upgrade/config-management paths, or deliberately allow a narrowly scoped human-approved config.patch path for this security gate.

## Solution Assessment
