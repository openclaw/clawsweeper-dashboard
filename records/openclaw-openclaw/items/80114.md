## Summary

Keep open. Current main still has a real conflict between host-approved conversation-hook policy, the agent-facing protected config write surface, and doctor/plugin repair paths that can remove unknown plugin entries; this is not safely closeable as implemented, duplicate, or external-only.

## What This Changes

The issue asks OpenClaw to preserve `plugins.entries.rl-training-headers.hooks.allowConversationAccess` across upgrades or provide a safe supported way to restore it for a local plugin.

## Best Possible Solution

Preserve existing operator-approved `plugins.entries.<id>.hooks` for local/private plugins across update, doctor, and plugin repair flows, add targeted warnings/docs, and keep manifest or agent-written trust changes behind explicit maintainer security review.

## Reproduction Assessment
