## Summary

Keep open. The performance goal is valid, but this PR is not merge-ready: the installed-index memo can return plugin metadata for the wrong discovery scope, and the item is also protected by a maintainer label/member author signal.

## What This Changes

This PR adds a process-scoped installed-plugin-index memo with invalidation on refresh/write paths and tests for memo reuse, bypass, and invalidation.

## Best Possible Solution

Keep the performance goal, but implement reuse at the caller-owned snapshot/control-plane layer or key the lower memo by the canonical discovery and freshness fingerprint with regression coverage.

## Reproduction Assessment

Yes for the review finding: source inspection shows the PR cache key excludes discovery roots/load paths and freshness inputs that current main uses to build the installed index. I did not run tests because this review is read-only.

## Solution Assessment

No. The proposed lower-level memo is too broad as keyed; the safer solution is to use the existing control-plane/snapshot fingerprinting model or include equivalent discovery and watched-file freshness in this memo.
