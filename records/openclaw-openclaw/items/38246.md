## Summary

Keep this open. Current main still routes WhatsApp visible auto-replies through the inbound message’s bound reply/sendMedia path, and the existing main-DM owner pinning only protects lastRoute metadata rather than blocking implicit replies to allowlisted non-owner DMs.

## What This Changes

The issue requests an opt-in WhatsApp routing safeguard that prevents implicit plain-text replies from being sent back to non-owner DM senders unless the agent uses an explicit target.

## Best Possible Solution

Add an opt-in WhatsApp root/account routing policy that detects admitted non-owner direct chats before visible auto-reply delivery, prevents implicit text/media replies unless the agent uses an explicit outbound target, preserves normal owner-DM behavior, and documents the config with regression coverage.
