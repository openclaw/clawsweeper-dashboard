## Summary

Keep open: current main still treats visible WhatsApp DM replies as same-chat replies, and the existing owner pinning only prevents `lastRoute` from being overwritten rather than blocking the current reply to a third-party sender.

## What This Changes

Request an opt-in WhatsApp DM routing guard that blocks or reroutes implicit plain-text replies when the inbound sender is not the session owner.

## Best Possible Solution

Design an explicit owner-aware reply policy for WhatsApp DMs, then add schema, docs, and regression tests around the chosen block or private-owner fallback semantics.

## Reproduction Assessment

Yes, source inspection gives a high-confidence path for the current behavior: WhatsApp auto-reply text is sent with `msg.reply(...)` to the inbound chat, and no matching config guard exists. I did not run a live WhatsApp session.

## Solution Assessment

Unclear. An opt-in guard is a plausible boundary, but maintainers need to choose the exact security and UX semantics before implementation, especially block versus auto-route-to-owner behavior.
