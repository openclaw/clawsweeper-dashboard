## Summary

Keep open: current main has a broad tool-only visible-reply mitigation for all direct chats, but WhatsApp still defaults direct-chat final replies to the inbound chat and lacks the requested owner-aware guard for third-party DM senders.

## What This Changes

Request an opt-in WhatsApp DM routing safeguard that blocks or reroutes implicit plain replies when the inbound sender is not the configured owner.

## Best Possible Solution

Design an explicit owner-aware reply policy, then add schema, docs, and regression coverage around the chosen block or owner-private fallback semantics while reusing the shared visible-reply machinery where possible.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: default WhatsApp direct replies remain automatic and `deliverWebReply` sends final text to `msg.from`; the existing tool-only option is a broad all-direct mitigation, not the requested owner-aware guard.

## Solution Assessment

Unclear. A per-channel config is plausible, but the maintainable fix should first settle block versus owner-reroute semantics and whether this belongs in shared source-reply policy instead of a WhatsApp-only knob.
