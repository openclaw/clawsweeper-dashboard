## Summary

Close as implemented: current main and v2026.5.2 ship a broader `messages.visibleReplies: "message_tool"` policy that suppresses automatic source-chat final replies for direct chats too, including WhatsApp. It does not add the exact owner-only exception proposed here, but it covers the reported leak path where plain assistant text is implicitly posted back to a third-party DM.

## What This Changes

Request an opt-in WhatsApp/DM routing safeguard that blocks or reroutes implicit plain-text replies when the last inbound sender is not the owner.

## Best Possible Solution

Keep the shipped global visible-reply guard and use `messages.visibleReplies: "message_tool"` for hardened DM deployments; any owner-only convenience exemption should be tracked as a separate UX refinement.

## Reproduction Assessment

Not applicable as a current bug reproduction: this is a security hardening feature request, and current source plus release inspection show a shipped opt-in mode that suppresses implicit direct-chat final delivery.

## Solution Assessment

Yes for the reported leak path: the shipped global policy is broader than the proposed WhatsApp-only owner-aware setting, but it is a maintainable shared solution that avoids owner-intended plain final replies being auto-posted to a third-party source chat.
