## Summary

Keep open. Current main has DM isolation, main-route pinning, message-tool explicit-target checks, and sendPolicy guardrails, but WhatsApp visible auto-replies still route to the inbound chat and there is no owner-aware config guard for non-owner DM senders.

## What This Changes

Request a configuration guard that blocks or reroutes implicit plain-text replies in WhatsApp DM sessions when the inbound sender is not the session owner.

## Best Possible Solution

Add an owner-aware delivery guard or policy that can suppress visible auto-replies for non-owner DM senders while preserving explicit message-tool sends and documenting the WhatsApp/shared-channel semantics.

## Reproduction Assessment

Not applicable as a feature request. Source and docs show the current-main behavior: inbound WhatsApp replies are intentionally delivered back to the inbound chat, and no owner-aware blocking option exists.

## Solution Assessment

Unclear. The proposed config direction is plausible, but maintainers should decide whether the right policy is block, owner reroute, or a shared sendPolicy extension before implementation.
