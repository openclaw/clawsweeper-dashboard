## Summary

Current main and the latest release already provide a shipped safeguard for this leak path: `messages.visibleReplies: "message_tool"` keeps normal source replies private and requires explicit visible output through the message tool, while WhatsApp also has main-DM route pinning to avoid non-owner DMs overwriting the main session route.

## What This Changes

The issue requests a WhatsApp DM routing safeguard that prevents implicit plain-text replies from being sent back to third-party senders unless an explicit target is used.

## Best Possible Solution

Keep the shipped generic visible-reply policy and direct users who need this safeguard to set `messages.visibleReplies: "message_tool"`, instead of adding a WhatsApp-only routing knob.

## Reproduction Assessment

Not applicable as a feature request. The current-main source, docs, and tests establish the shipped safeguard behavior without needing a live WhatsApp reproduction.

## Solution Assessment

Yes. The implemented generic source-reply policy covers WhatsApp through the shared dispatch path and is a better boundary than a WhatsApp-specific routing setting.
