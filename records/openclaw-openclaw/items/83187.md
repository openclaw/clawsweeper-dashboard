## Best Possible Solution

Keep the heartbeat clear-on-success fix, but gate both clear sites to pending-final state that the current heartbeat run created or satisfied, with regression coverage for message_tool_only and heartbeat response-tool preservation of older recovery.

No, not yet. Clearing in heartbeat-runner is the right owner-boundary fix, but the latest patch must preserve older pending-final state when the current heartbeat delivery did not create or satisfy it.
