## Summary

Keep open: the narrow bug fix is useful and backed by live proof, but the current matcher checks only the primary origin before deleting session fallback state, so it can erase a newer same-origin provider/model/auth fallback pin.

## What This Changes

The PR adds auto-reply fallback cleanup and tests to clear auto-owned model fallback session pins after restart/drain/user-abort exits or successful fallback from ephemeral primary failures.

## Best Possible Solution

Keep the narrow ephemeral-failure cleanup, but clear only when the current session entry still exactly matches the fallback provider/model/auth state selected by this run; leave TTL/config policy to the related items.

## Reproduction Assessment

Yes. Current source persists fallback selections and has no current-main cleanup on these ephemeral success/abort paths, while the PR body supplies live terminal proof of stuck Telegram session pins and after-fix cleanup.

## Solution Assessment

No, not as currently patched. The narrow cleanup direction is right, but the matcher must verify the exact fallback provider/model/auth state before deleting session state.
