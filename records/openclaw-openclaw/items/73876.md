## Summary

Keep open. Current main still lacks a direct `moonshotai` alias for Moonshot, and the issue is paired with open closing PR #74946, so it should remain open until that PR or a replacement merges.

## What This Changes

Accept `moonshotai/<model>` as a direct Moonshot API model reference while preserving existing `moonshot/<model>` configs.

## Best Possible Solution

Land #74946 if it proves `moonshotai/kimi-k2.6` resolves through the actual runtime model path, or replace it with a narrow alias fix at the provider-normalization/catalog layer plus focused tests.

## Reproduction Assessment

Yes. Current source gives a high-confidence reproduction path: `moonshotai/kimi-k2.6` keeps provider `moonshotai`, while the registered Moonshot catalog/provider remains `moonshot`.

## Solution Assessment

Partly. A backward-compatible alias is the right direction and a provider rename is not, but the fix should include runtime model-resolution coverage in addition to catalog alias planning.
