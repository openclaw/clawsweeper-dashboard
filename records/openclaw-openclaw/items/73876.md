## Summary

Keep open. Current main still exposes direct Moonshot as `moonshot/...` without a `moonshotai` provider alias, and open PR #74946 is the active unmerged closing candidate for this issue.

## What This Changes

Accept `moonshotai/<model>` as a direct Moonshot API model reference while preserving existing `moonshot/<model>` configs.

## Best Possible Solution

Land a reviewed backward-compatible alias from `moonshotai` to canonical `moonshot`, with regression coverage that proves the runtime resolver accepts `moonshotai/kimi-k2.6`.

## Reproduction Assessment

Yes. Source inspection on current main shows `moonshotai/kimi-k2.6` keeps provider `moonshotai`, while the bundled Moonshot catalog only exposes `moonshot/kimi-k2.6`, matching the reported `Unknown model` path.

## Solution Assessment

Yes. A backward-compatible alias is narrower than renaming the provider because it preserves existing `moonshot/...` configs, auth profiles, plugin ids, and docs while accepting the OpenRouter-style slug.
