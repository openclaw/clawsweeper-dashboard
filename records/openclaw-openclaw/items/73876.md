## Summary

Keep open. Current main still lacks runtime normalization for `moonshotai` or `moonshot-ai`, and the open closing PR needs runtime-resolution coverage before this can close.

## What This Changes

Accept OpenRouter-style direct Moonshot model refs such as `moonshotai/kimi-k2.6` without breaking existing `moonshot/...` configs.

## Best Possible Solution

Land a backward-compatible provider-id normalization alias from `moonshotai` and `moonshot-ai` to canonical `moonshot`, with regression coverage for direct model resolution and the required changelog entry.

## Reproduction Assessment

Yes by source inspection. Current main parses `moonshotai/kimi-k2.6` through `normalizeProviderId`, leaves provider `moonshotai` unchanged, and then looks for catalog/runtime rows that are keyed only under `moonshot`.

## Solution Assessment

No for a manifest-only alias or provider rename. The narrow maintainable fix is runtime provider-id normalization that preserves existing `moonshot/...` configs while accepting the OpenRouter-style aliases.
