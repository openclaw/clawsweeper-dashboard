## Summary

Keep open. Current main still exposes direct Moonshot as `moonshot/...` with no `moonshotai` provider normalization or manifest alias, and open PR #74946 is the active closing candidate to review before this issue closes.

## What This Changes

Allow `moonshotai/<model>` direct Moonshot refs to resolve to the existing `moonshot` provider while preserving existing `moonshot/...` configs.

## Best Possible Solution

Land a reviewed backward-compatible alias that maps direct `moonshotai` refs to canonical `moonshot` and verifies the runtime resolver path that currently returns `Unknown model`.

## Reproduction Assessment

Yes. Source inspection on current main shows `moonshotai/kimi-k2.6` keeps provider `moonshotai`, while the built-in catalog only exposes `moonshot/kimi-k2.6`, matching the reporter's runtime logs.

## Solution Assessment

Yes. A backward-compatible alias is narrower than renaming the provider because it preserves existing `moonshot/...` configs, auth profiles, plugin ids, and docs while accepting the OpenRouter-style slug.
