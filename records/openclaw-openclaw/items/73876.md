## Summary

Keep open. Current main still lacks a direct `moonshotai` alias for the Moonshot provider, and open PR #74946 is the active closing candidate that should be reviewed or landed before this issue closes.

## What This Changes

Accept direct `moonshotai/<model>` model refs as aliases for the canonical direct Moonshot provider while preserving existing `moonshot/...` configs.

## Best Possible Solution

Land a reviewed backwards-compatible alias that accepts `moonshotai/<model>` for direct Moonshot while keeping `moonshot/...` as the canonical provider id.

## Reproduction Assessment

Yes. Source inspection on current main shows `moonshotai/kimi-k2.6` keeps provider `moonshotai`, while the built-in catalog only exposes `moonshot/kimi-k2.6`, matching the reporter's logs.

## Solution Assessment

Yes. A backwards-compatible alias is narrower than renaming the provider because it preserves existing `moonshot/...` configs, auth profiles, plugin ids, and docs.
