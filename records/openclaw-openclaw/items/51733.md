## Summary

Keep this PR open. Current main supersedes the watcher-glob part by watching skill-root directories, but it still does not implement the PR's repo-style nested bundle loading for paths like `<skillRoot>/<repo>/skills/<skill>/SKILL.md`; the branch needs a focused rebase and one remaining precedence bug fixed before merge.

## What This Changes

The PR adds watcher coverage, loader discovery, and regression tests for repo-style nested skill bundles under configured skill roots.

## Best Possible Solution

Land a rebased, loader-focused fix that discovers repo-style nested bundles under workspace, project-agent, personal-agent, and managed skill roots while keeping the current root watcher and bounded discovery model. The fix should align hybrid bundle precedence with the existing nested-root behavior or document and test any intentional difference.

## Reproduction Assessment

Yes. Static inspection gives a high-confidence reproduction path: place a skill at `<workspace>/skills/<repo>/skills/<skill>/SKILL.md`; current main scans `<repo>/SKILL.md` and then `<repo>/<child>/SKILL.md`, but never reaches `<repo>/skills/<skill>/SKILL.md`.

## Solution Assessment

No, the current PR branch is not yet the best merge shape. A safer solution is to drop the obsolete watcher-glob changes, rebase onto current main, and keep the patch focused on bounded repo-style loader discovery plus precedence tests.

## Review Findings

Full review comments:

- **[P2] Resolve nested roots before loading top-level skills:** `src/agents/skills/workspace.ts:303-320`
