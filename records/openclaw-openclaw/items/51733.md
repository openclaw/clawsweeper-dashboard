## Summary

Keep this PR open: current main fixed the adjacent two-level grouped skill layout, but it still does not cover this PR's repo-style `<root>/<repo>/skills/<skill>/SKILL.md` case under managed/workspace roots. The branch is a plausible fix path, but it needs one correctness change and real behavior proof before merge.

## What This Changes

The PR adds repo-style nested skill discovery under managed/workspace skill roots, adds watcher targets for `*/skills/*/SKILL.md`, and extends loader/watcher tests.

## Best Possible Solution

Preserve the current bounded discovery model, but add the missing repo-style child-root case by resolving nested roots before direct child loading, align watcher coverage with the loader, and add regression coverage for both plain repo-style and hybrid top-level-plus-nested bundles.

## Reproduction Assessment

Yes, from source inspection rather than an executed local test. Current main handles `skills/<group>/<skill>/SKILL.md` but not `<root>/<repo>/skills/<skill>/SKILL.md` under managed/workspace roots, and the PR's added tests model that missing path.

## Solution Assessment

No, not as currently written. The direction is right, but the branch should preserve nested-root precedence for child repo bundles and include real behavior proof before merge.
