## Summary

Keep open: current main still misses repo-style nested skill bundles under configured skill roots, and the PR branch is useful but not mergeable because it resolves a direct child `SKILL.md` before nested `skills/*/SKILL.md` entries.

## What This Changes

The PR updates the skills watcher, workspace skill loader, and loader/watcher tests to discover repo-style nested skill bundles under configured skill roots.

## Best Possible Solution

Land a rebased loader-focused fix that discovers repo-style nested bundles under workspace, project-agent, personal-agent, managed, and extra skill roots before direct-child fallback while preserving caps, containment checks, and the current watcher-root design.

## Reproduction Assessment

Yes. Source inspection shows current main only resolves a nested skills root at the configured root and then scans direct children plus one grouped level, so `<skillRoot>/<repo>/skills/<skill>/SKILL.md` is not reached.

## Solution Assessment

No. The direction is right, but the current PR is not the best fix because it mixes stale watcher-glob changes with a loader precedence bug; a current-main loader patch with targeted precedence tests is safer.
