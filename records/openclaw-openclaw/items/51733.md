## Summary

Keep this PR open: it targets a real repo-style skill-bundle discovery gap, but the branch is stale against current main and still has correctness blockers in both loader precedence and watcher strategy.

## What This Changes

The PR changes skills loading, skills watcher targets, and tests to discover repo-style nested skill bundles under configured skill roots.

## Best Possible Solution

Land a rebased current-main fix that discovers and refreshes `<skillRoot>/<repo>/skills/<skill>/SKILL.md` while preserving bounded traversal, realpath containment, and current grouped-skill semantics.

## Reproduction Assessment

Yes. Source inspection shows current main skips `<repo>/skills/<skill>/SKILL.md` under managed/workspace skill roots, while the PR head still has a source-reproducible hybrid-repo precedence defect.

## Solution Assessment

No. The PR points at the right gap, but the branch is not the best merge as-is because it adds stale glob watcher targets and resolves direct child skills before nested repo roots.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.86

Full review comments:

- **[P2] Resolve nested roots before direct-child skills:** `src/agents/skills/workspace.ts:436-443`
  - body: When a repo under a configured skills root contains both `<repo>/SKILL.md` and `<repo>/skills/<skill>/SKILL.md`, this branch loads the direct child first and never resolves the nested root. A repo passed directly through `skills.load.extraDirs` resolves the nested root first, so the same bundle exposes different skills by install location.
  - confidence: 0.84
- **[P2] Use the current directory watcher model:** `src/agents/skills/refresh.ts:94-97`
  - body: These added watch targets are glob patterns, but the current dependency is Chokidar 5, where glob support was removed. A rebased fix should extend the current literal-directory-plus-depth watcher model for repo-style bundles instead of relying on `*` patterns that will not be interpreted as globs.
  - confidence: 0.78
