## Summary

## What This Changes

This PR updates skills discovery, skills watcher targets, and tests so repo-style `skills/<repo>/skills/<skill>/SKILL.md` bundles under configured roots can load and refresh the skills snapshot.

## Best Possible Solution

Land a rebased patch that supports repo-style bundles under configured skill roots while preserving bounded scans, containment checks, cycle protection, and watcher/loader alignment.

## Reproduction Assessment

Yes, by source inspection. Current main resolves repo-style `dir/skills/*/SKILL.md` only when the configured root is the repo itself, while managed/workspace root scans and watcher depth stop at the direct or one-grouped-level layouts.

## Solution Assessment

No. The direction is useful, but this PR head should resolve nested roots before loading direct child skills and needs a rebase onto current main's watcher model before merge.
