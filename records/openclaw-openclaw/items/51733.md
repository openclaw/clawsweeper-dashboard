## Summary

Keep this PR open: the repo-style nested skill use case still is not covered by current main, but the branch is stale and still has a loader precedence bug that would expose different skills depending on install location.

## What This Changes

The PR changes skills loading, skills watcher targets, and tests to discover repo-style nested skill bundles under configured skill roots.

## Best Possible Solution

Land a current-main repair that deliberately supports repo-style bundles under configured skill roots while preserving bounded scans, containment checks, cycle protection, and watcher/loader alignment.

## Reproduction Assessment

Yes, by source inspection. Current main can load repo-style bundles when the configured root is the repo itself, but configured roots scan only one grouped level and do not reach `<repo>/skills/<skill>/SKILL.md`.

## Solution Assessment

No. The PR targets the right gap, but the branch should be repaired or replaced on current main because it loads direct child skills before nested roots and its watcher changes predate the current root-directory watcher model.
