## What This Changes

The PR changes skills discovery, skills watcher targets, and tests so repo-style nested skill bundles under configured skill roots are loaded and refreshed.

## Best Possible Solution

Land a current-main repair that supports repo-style bundles under configured skill roots while preserving bounded scans, containment checks, cycle protection, and watcher/loader alignment.

## Reproduction Assessment

Yes, by source inspection. Current main's loader resolves nested roots only at the configured root and then scans direct children plus one grouped level, so managed/workspace `<repo>/skills/<skill>/SKILL.md` is not discovered.

## Solution Assessment

No. The direction is right, but the PR head should resolve nested roots before direct child loading and be refreshed for current main's watcher model before merge.
