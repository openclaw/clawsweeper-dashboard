## Summary

Keep open. Current main and v2026.5.3 still contain the reported restart-recovery and WebChat delivery constraints, while the WebChat docs promise replies back to WebChat, so this needs a focused regression fix rather than cleanup closure.

## What This Changes

Restore WebChat restart auto-resume so a response generated after gateway restart is visible in the WebChat UI without manual follow-up.

## Best Possible Solution

Restore restart-recovery visibility through WebChat’s internal chat event and transcript/history mechanisms, with regression coverage, while keeping WebChat out of generic outbound delivery.

## Reproduction Assessment

No live end-to-end reproduction was run. Source inspection matches the reported path: restart recovery resumes with `deliver: false`, and WebChat agent delivery is still rejected or downgraded to session-only in tests and code.

## Solution Assessment

No, not as broadly proposed. The requested outcome is valid, but the safer fix is to repair WebChat-specific resume visibility without removing the internal-only outbound channel boundary.
