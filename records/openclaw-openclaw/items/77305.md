## Summary

Keep open. Current main still contains the reported restart-recovery and WebChat delivery constraints, and the WebChat docs promise replies back to WebChat, so there is no safe cleanup close here.

## What This Changes

Restore WebChat restart recovery so auto-resumed agent responses after Gateway restart become visible in WebChat without manual prompting.

## Best Possible Solution

Make restart recovery visibly complete WebChat sessions through the internal chat event and transcript/history mechanisms, with regression coverage, while preserving external outbound delivery rules.

## Reproduction Assessment

No live end-to-end restart/WebChat run was performed. Source inspection matches the reported failure boundary: recovery sends a resume agent turn with `deliver: false`, while current agent delivery keeps WebChat session-only even when delivery is requested.

## Solution Assessment

The requested outcome is the right direction, but the issue's broad `deliver: true` or WebChat exclusion removal is not the safest fix. The narrower maintainable solution is to repair WebChat resume visibility through the existing internal chat/history path.
