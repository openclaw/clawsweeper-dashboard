## Summary

Keep open. Current main still has the reported restart-recovery and WebChat delivery constraints, and the WebChat docs promise deterministic replies back to WebChat, so this needs a focused fix rather than cleanup closure.

## What This Changes

Restore gateway-restart auto-resume so WebChat sessions receive the recovered assistant response without manual follow-up.

## Best Possible Solution

Make restart recovery visibly complete WebChat sessions through the existing internal chat event and transcript/history mechanisms, with regression coverage, while leaving external outbound delivery rules unchanged.

## Reproduction Assessment

No live end-to-end reproduction was run. Source inspection matches the reported path: restart recovery resumes with `deliver: false`, and agent delivery currently keeps WebChat session-only even when delivery is requested.

## Solution Assessment

The proposed broad `deliver: true` or removal of the WebChat exclusion is not clearly the safest fix. The maintainable direction is to repair WebChat-specific resume visibility without turning the internal WebChat surface into an external message channel target.
