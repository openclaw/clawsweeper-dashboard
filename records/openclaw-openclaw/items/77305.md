## Summary

Keep open. Current main still contains the reported restart-recovery and WebChat delivery constraints, while docs and adjacent tests show this is a narrow WebChat resume-visibility gap rather than cleanup closure.

## What This Changes

WebChat restart recovery should auto-resume an interrupted turn after gateway restart and make the completed response visible in the WebChat UI without manual user prompting.

## Best Possible Solution

Make restart recovery visibly complete WebChat sessions through WebChat’s internal chat event and transcript/history mechanisms, with regression coverage, while keeping WebChat out of generic outbound channel delivery.

## Reproduction Assessment

No live end-to-end reproduction was run. Source inspection gives a clear current-main path matching the report: restart recovery calls the agent RPC with `deliver: false`, and WebChat remains excluded from outbound delivery even when delivery is requested.

## Solution Assessment

The issue’s broad suggested fix is not clearly the safest solution. The narrow maintainable fix is WebChat-specific resume visibility through internal chat events/history, not making `webchat` a generic outbound deliverable channel.

## Repair Work Prompt

Fix the WebChat gateway-restart auto-resume regression reported in openclaw/openclaw#77305. Current main resumes interrupted main sessions with `deliver: false` in `src/agents/main-session-restart-recovery.ts`, while `src/gateway/server-methods/agent.ts` keeps WebChat out of generic outbound delivery. Do not make `webchat` a generic outbound deliverable channel or remove the WebChat delivery guard broadly. Instead, make recovered WebChat turns become visible through the existing internal chat event/transcript/history mechanisms so a restarted WebChat session sees the completed resumed response without manual prompting. Add focused regression coverage around restart recovery and WebChat agent/chat event delivery. Include a single-line user-facing changelog fix entry unless maintainers decide this is test-only.
