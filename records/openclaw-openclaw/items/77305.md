## Summary

Keep open. Current main still has the source-backed WebChat restart-recovery delivery gap, and the documented WebChat contract says replies should return to WebChat.

## What This Changes

The issue asks OpenClaw to restore automatic delivery of resumed WebChat responses after a gateway restart, matching the 5.2 behavior.

## Best Possible Solution

Make restart recovery visibly complete WebChat sessions through the existing internal chat event and durable transcript/history mechanisms while preserving WebChat’s internal-only outbound boundary.

## Reproduction Assessment

No live high-confidence reproduction was run. Source inspection matches the report: restart recovery resumes with `deliver: false`, WebChat is excluded from outbound delivery, and tests assert both current behaviors.

## Solution Assessment

No, the issue’s broad suggested removal of the WebChat exclusion is not the safest fix. The narrow maintainable solution is WebChat-specific resume visibility through chat events/history without turning WebChat into an external channel target.
