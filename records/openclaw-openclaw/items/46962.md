## Summary

Keep open: the underlying Markdown IR bug is still present on current main, but this PR is not merge-ready because it would drop current loose-list paragraph handling and still lacks real after-fix behavior proof.

## What This Changes

The PR adds blockquote-depth tracking in `markdownToIR` to reapply configured blockquote prefixes across soft/hard breaks, nested quotes, and paragraph transitions, with regression tests.

## Best Possible Solution

Merge a rebased patch that adds quote-depth-aware prefix restoration for Markdown IR soft/hard breaks and paragraph transitions while retaining current list/table separator rules and focused regression coverage.

## Reproduction Assessment

Yes, by source inspection. Current main handles `softbreak` and `hardbreak` by appending only a bare newline while Slack and Signal configure `blockquotePrefix: "> "`, so continuation lines lose the quote prefix.

## Solution Assessment

No. The direction is right, but the branch should carry forward current main's token/open-level loose-list paragraph separator while adding quote-prefix restoration.
