## Summary

Keep open: current main still has the Markdown IR blockquote-prefix continuation gap, and the prior review blocker still holds because this PR predates the current loose-list paragraph separator logic and lacks after-fix real behavior proof.

## What This Changes

The PR adds blockquote-depth-aware Markdown IR rendering and regression tests so quote-prefixed outputs repeat the blockquote prefix after soft/hard breaks and blockquote paragraph transitions.

## Best Possible Solution

Merge a rebased patch that restores quote prefixes for Markdown IR soft/hard breaks and blockquote paragraph transitions while preserving current loose-list and table separator behavior with focused regression coverage.

## Reproduction Assessment

Yes, by source inspection. Current main appends a bare newline for Markdown `softbreak` and `hardbreak` while Slack and Signal configure `blockquotePrefix: "> "`, so continuation lines lose the quote marker.

## Solution Assessment

No. The PR's direction is right, but as-is it would regress current main's loose-list paragraph separators; the safer fix is to rebase and carry forward the token/open-level guard while adding quote-depth prefix restoration.
