## Summary

Keep open. Current main still has the memory-wiki broken-wikilink false-positive path, and this PR is a focused implementation candidate, but the patch should not merge as-is because it lowercases path-like link targets and lacks after-fix real behavior proof.

## What This Changes

The PR expands memory-wiki lint target lookup to accept normalized title, slug, fragment-stripped, path, basename, id, and imported source-path suffix wikilinks, with a focused regression test.

## Best Possible Solution

Land a narrow lint resolver fix that resolves titles, slug variants, fragments, and imported source-path suffixes while preserving case-sensitive validation for path-like links, backed by focused tests and real behavior proof.

## Reproduction Assessment

Yes, by source inspection. A page titled `Alpha Database` linked as `[[Alpha Database#Evidence]]` reaches lint with the fragment intact, while current main only accepts path and basename targets.

## Solution Assessment

No. The direction is right, but this implementation should separate case-folded title/metadata matching from case-preserving path matching before merge.
