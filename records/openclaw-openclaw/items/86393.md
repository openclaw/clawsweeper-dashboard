## Summary

Keep open: current main still resolves the `/btw` transcript directly from the stored session file, while Codex side-question binding lookup is tied to that exact transcript sidecar; compaction metadata has enough pre/post transcript data to repair this without a product decision.

## What This Changes

The issue requests that Codex `/btw` side questions remap a pre-compaction checkpoint transcript back to the active post-compaction transcript before reading the Codex app-server binding.

## Best Possible Solution

Add a bounded resolver helper in `src/agents/btw-transcript.ts` that maps matched pre-compaction checkpoint transcripts to their `postCompaction.sessionFile`, with focused regression coverage and no new config surface.

## Reproduction Assessment

Yes, source-reproducible: construct a session entry whose `sessionFile` points at `preCompaction.sessionFile`, and current main returns that checkpoint path before Codex reads the matching sidecar. I did not run tests because this was a read-only review.

## Solution Assessment

Yes, the proposed direction is the narrow maintainable fix: resolve the active post-compaction transcript before invoking the harness, and leave behavior unchanged when no matching checkpoint exists.
