## Summary

Keep open: current main has a plugin-level `authEpochMode` hook and token-rotation-stable auth-epoch logic, but it does not expose `authEpochMode` through `openclaw.json`; this is a new config/API surface rather than a conservative cleanup close.

## What This Changes

The issue asks to expose CLI backend `authEpochMode` in `openclaw.json`, with backend defaults and per-agent overrides so OAuth CLI runtimes can avoid auth-epoch session resets.

## Best Possible Solution

Decide whether auth-epoch policy should remain backend-owned or become a documented user config contract; if accepted, add schema/types/help/docs/tests and resolve the global-versus-per-agent precedence deliberately.

## Reproduction Assessment

Not applicable as a strict bug reproduction because this is primarily a request for a new config surface. Source inspection does confirm the requested `openclaw.json` field is absent, while the reported token-refresh reset behavior was not live-reproduced on current main.

## Solution Assessment

Unclear: exposing `authEpochMode` may be useful, but current main already treats OAuth token refreshes as epoch-stable. A maintainer should first decide whether the remaining problem is a public config gap or a narrower auth identity/session bug.
