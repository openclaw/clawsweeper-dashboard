## Summary

Keep open: current main still has a source-level WebChat terminalization gap that can suppress the final chat error after a provider 429/idle-timeout, and the related fix PR was closed unmerged for queue-limit automation rather than because the bug was resolved.

## What This Changes

WebChat should stop polling and show the failed assistant turn live when a provider 429 or idle-timeout fails before producing content.

## Best Possible Solution

Fix the generic gateway/WebChat terminal error path so retry-grace suppression does not outlive the retry window, then add regression coverage for failed no-content chat.send runs and the deferred history reload behavior.

## Reproduction Assessment

Yes, at source level: current main still schedules lifecycle error finalization with skipChatErrorFinal and the UI waits on a terminal chat event to clear chatRunId. The reporter also supplied live logs and before/after screenshots, but I did not rerun the live provider 429 locally.

## Solution Assessment

Yes: the narrow maintainable fix is in the generic gateway/WebChat terminalization path, not in OpenRouter-specific handling. The prior closed PR points at a plausible one-line core change, but it should land with regression coverage and validation on current main.
