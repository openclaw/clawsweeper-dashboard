Keep open: current main still has the source-level gap described in the report, and the two linked fix PRs were closed without merging. The delayed lifecycle-error path still carries `skipChatErrorFinal` through the grace timer, and the adjacent test still expects no terminal chat error for the active no-link `chat.send` case.

## What This Changes

WebChat should stop the active polling indicator and surface the failed assistant turn after provider 429 or idle-timeout failures without requiring a page refresh.

## Best Possible Solution

Land a narrow gateway lifecycle fix that preserves the retry/fallback grace window but emits a terminal chat error, and reconciles the persisted failed turn, once the grace expires with no successful retry.
