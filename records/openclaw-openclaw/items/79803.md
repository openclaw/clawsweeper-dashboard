## Summary

Keep open: current main still has the WebChat/gateway terminalization gap, and the two linked fix attempts are closed without merge rather than superseding the report.

## What This Changes

WebChat should stop polling and surface the failed assistant turn live when a provider 429 or idle-timeout failure happens before content is produced.

## Best Possible Solution

Fix the gateway lifecycle terminalization path so an unrecovered provider failure emits or reconciles a terminal chat error after the retry/grace window, while preserving per-attempt fallback suppression.

## Reproduction Assessment

Yes, at source level: current main preserves `skipChatErrorFinal` through the delayed lifecycle-error finalizer, and an adjacent test asserts no chat error after the grace timer for the active no-link `chat.send` case. The reporter also supplied live logs and before/after screenshots, but I did not rerun a live provider 429 locally.

## Solution Assessment

Yes: the narrow maintainable fix belongs in the gateway/WebChat terminal event path, not in provider-specific OpenRouter handling. The fix should update the existing gateway lifecycle regression test and only touch UI code if event reconciliation semantics change.
