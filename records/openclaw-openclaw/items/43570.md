## Summary

Keep open. Current main and latest release still have the Discord `thread-reply` attachment alias gap, and #43574 is the paired open fix candidate, so this issue should stay open until a fix merges.

## What This Changes

Discord `thread-reply` should honor `path` and `filePath` attachment aliases the same way `send` already does.

## Best Possible Solution

Land #43574 or an equivalent Discord-owned patch that resolves `thread-reply` attachments as `media`, then `path`, then `filePath`, with focused regression coverage and a user-facing changelog entry.

## Reproduction Assessment

Yes. Source inspection on current main is high confidence: a `thread-reply` call with only `filePath` or `path` leaves `mediaUrl` undefined before the Discord runtime send path.

## Solution Assessment

Yes. The narrow maintainable fix belongs in the Discord `thread-reply` handler because downstream delivery already attaches media once `mediaUrl` is populated, and shared hydration does not cover this action.
