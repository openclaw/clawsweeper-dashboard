## Summary

Keep open. Current main and latest release `v2026.5.2` still have the Discord `thread-reply` attachment alias gap, and open PR #43574 already tracks the focused fix.

## What This Changes

The issue asks Discord `thread-reply` to honor `media`, `path`, and `filePath` attachment aliases consistently with Discord `send`.

## Best Possible Solution

Land #43574 or an equivalent Discord-owned patch that maps `thread-reply` attachments with `media` first, then `path`, then `filePath`, with focused regression coverage for alias handling and precedence.

## Reproduction Assessment

Yes. The source path is high confidence: a `thread-reply` call with only `filePath` leaves `mediaUrl` undefined before runtime delivery, so no attachment is forwarded.

## Solution Assessment

Yes. The narrowest maintainable fix is in the Discord `thread-reply` handler because shared hydration does not cover that action and the downstream runtime already sends the attachment once `mediaUrl` is populated.
