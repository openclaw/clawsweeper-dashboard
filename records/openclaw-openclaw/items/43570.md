## Summary

Keep open. Current main and latest release v2026.5.2 still have the Discord thread-reply attachment alias gap, and open PR #43574 remains the focused fix candidate; closing this issue before that lands would drop a valid tracked bug.

## What This Changes

Discord message action `thread-reply` should attach files when callers use `path` or `filePath`, matching `send`'s `media`/`path`/`filePath` fallback.

## Best Possible Solution

Land #43574 or an equivalent Discord-owned patch that maps `thread-reply` attachments with `media` first, then `path`, then `filePath`, with regression coverage for alias handling and precedence.

## Reproduction Assessment

Yes. Source inspection on current main shows a `thread-reply` call with only `filePath` or `path` leaves `mediaUrl` undefined before runtime delivery, while `send` accepts those aliases.

## Solution Assessment

Yes. The narrowest maintainable fix is in the Discord `thread-reply` handler because shared hydration does not cover that action and the downstream runtime already sends the attachment once `mediaUrl` is populated.
