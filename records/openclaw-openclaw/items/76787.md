## Summary

Keep open. Current main still treats Telegram `t.me/c/...` links as ordinary inbound text and exposes no Telegram read/fetch/link-resolution action, while the requested historical-message injection needs maintainer privacy and side-effect policy before implementation.

## What This Changes

The issue requests resolving Telegram `t.me/c/<chat>/<thread>/<message>` links and injecting the referenced message content into the agent context.

## Best Possible Solution

Design a Telegram-owned, opt-in resolver that prefers already-ingested history when possible, documents any Bot API fallback, and gates historical content injection by clear access and retention policy.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence reproduction path for the current behavior: a Telegram message containing a `t.me/c/...` URL flows through text expansion into `BodyForAgent`, and no Telegram read/fetch resolver is exposed; I did not run a live bot in this read-only review.

## Solution Assessment

No. The requested copy/forward/delete retrieval path is not yet the best maintainable fix because maintainers first need to choose the privacy, authorization, retention, and Telegram-side-effect policy.

## Security Review

Status: needs_attention

Summary: Resolving Telegram message links is security-sensitive because it can inject historical chat content that was not directly submitted in the current turn.

Concerns:

- **[medium] Define linked-message access policy first:**
  - body: Automatic `t.me/c` resolution can expose prior group or DM content to the agent, so maintainers should define who may trigger it, what chats/messages are eligible, retention behavior, and the safe default before implementation.
  - confidence: 0.86
- **[medium] Validate Bot API retrieval side effects:**
  - body: The proposed copy/forward/delete retrieval path can create visible messages or leave cleanup/audit failures, so any implementation should handle permissions, deletion failures, and observable side effects explicitly.
  - confidence: 0.75
