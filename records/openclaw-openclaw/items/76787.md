## Summary

Keep open. Current main still treats Telegram links as inbound text and exposes no Telegram message-link fetch/read resolver in the action, config, or docs surface. This is a new, security-sensitive Telegram plugin capability that needs maintainer policy before implementation.

## What This Changes

The issue requests parsing `t.me/c/<chat>/<thread>/<message>` links in Telegram messages, fetching the referenced content, and injecting it into session context.

## Best Possible Solution

Design a Telegram-owned, opt-in or policy-gated resolver that prefers already-ingested history and documents any accepted Bot API fallback before injecting linked content.

## Reproduction Assessment

Yes. Source inspection shows inbound Telegram text flows through `expandTextLinks` into `BodyForAgent`, while the Telegram action/docs/config surfaces have no message-link resolver; I did not run a live Telegram bot in this read-only review.

## Solution Assessment

No. The requested copy/forward/delete retrieval path is not clearly the best solution until maintainers decide the privacy, permission, retention, and side-effect policy; a gated Telegram plugin resolver is the safer direction.

## Security Review

Status: needs_attention

Summary: Resolving Telegram message links is security-sensitive because it can inject historical chat content that was not directly pasted into the current turn.

Concerns:

- **[medium] Define linked-message access policy first:**
  - body: Automatic `t.me/c` resolution can expose past group or DM content to the agent, so maintainers should define who may trigger it, what history may be used, whether content is retained, and what default is safe before implementation.
  - confidence: 0.82
- **[medium] Validate any Bot API retrieval side effects:**
  - body: The issue proposes copy/forward-style retrieval; if that path is chosen, the Telegram plugin should explicitly handle visible messages, deletion failures, permissions, and auditability rather than making resolution implicit.
  - confidence: 0.72
