## Label Justifications

- `P3`: This is a useful Telegram feature/design request, but it is not an outage or regression in an existing documented read/link-resolution contract.
- `impact:session-state`: The requested behavior would inject referenced Telegram message content into the agent session context.

## Summary

Keep open: current main and v2026.6.10 still do not resolve Telegram `t.me/c/...` links, expose a Telegram read/fetch action, or inject linked message content into session context. The request is valid, but automatic link expansion needs maintainer product/security direction before implementation.

## What This Changes

Resolve Telegram `t.me/c/` message links by fetching the referenced message content and injecting it into Telegram session context.

## Best Possible Solution

Define a Telegram-plugin-owned, authorization-aware link/read design that resolves only approved message scopes, aligns with the broader Telegram read-action work, documents retention limits, and includes focused tests plus live Telegram proof.

## Reproduction Assessment

Yes, at source level. Current main has no `t.me/c` parser, Telegram read/fetch action, or supplemental linked-message context path, so these links remain ordinary inbound text.

## Solution Assessment

Unclear. The maintainable path is likely a bounded observed-message/read surface with explicit authorization, not an ad hoc automatic copy/forward/delete injection path without a security contract.
