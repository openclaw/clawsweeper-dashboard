## Summary

Keep open. Current main does not implement Telegram `t.me/c/...` resolution: inbound Telegram text is passed into `BodyForAgent` after text-link expansion, and the Telegram action/docs surface has no read/fetch resolver. Resolving historical messages needs maintainer policy on cached history versus Bot API forward/delete side effects, so this should not be closed by cleanup.

## What This Changes

Request automatic resolution of Telegram `t.me/c/...` message links into referenced message content in Telegram session context.

## Best Possible Solution

Design a Telegram-owned, policy-gated resolver that prefers already-ingested/cached messages when possible and only uses Bot API forwarding/copy behavior if maintainers accept the privacy and side-effect tradeoffs, with docs and regression coverage.

## Reproduction Assessment

Yes, from source inspection: Telegram inbound text flows through `expandTextLinks` into `BodyForAgent`, and neither the Telegram action map nor docs expose a fetch/read resolver. I did not run a live Telegram bot because this review is read-only.

## Solution Assessment

No, the suggested Bot API copy/forward-and-delete path is not clearly the best fix yet because it may create side effects and needs privacy/permission policy. A safer direction is a Telegram-owned cached-history resolver or explicit gated read action.
