## Summary

Keep open. Current main still treats Telegram `t.me/c/...` links as inbound text, and adding linked-message hydration would be a new Telegram capability with privacy, authorization, and Bot API side-effect decisions.

## What This Changes

Resolve Telegram `t.me/c/...` message links in inbound Telegram context by retrieving referenced message content and injecting it into the session context.

## Best Possible Solution

Design a Telegram-owned, opt-in resolver that prefers already-observed message history, defines access and retention policy, and uses any Bot API copy/forward fallback only with explicit side-effect handling.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: a Telegram message containing a `t.me/c/...` URL flows through text expansion into `rawBody` and `bodyForAgent`, with no resolver or read/fetch action on current main.

## Solution Assessment

No. The suggested copy/forward/delete retrieval path is not yet the best maintainable fix because privacy, authorization, retention, and Telegram-visible side effects need maintainer policy first.
