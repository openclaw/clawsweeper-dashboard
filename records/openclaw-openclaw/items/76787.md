## Summary

## What This Changes

The issue requests resolving Telegram `t.me/c/...` message links and injecting the referenced Telegram message content into agent session context.

## Best Possible Solution

Design this as Telegram-plugin-owned work: first settle the read/cache or Bot API copy/forward contract and authorization boundary, then add gated `t.me/c` parsing and context injection with docs, focused tests, and live Telegram proof.

## Reproduction Assessment

Yes, at source level. Current main lacks a `t.me/c` parser, Telegram read/fetch action discovery, runtime dispatch, action/config gate, and linked-message supplemental context path, so such links remain ordinary inbound text.

## Solution Assessment

Unclear until maintainers choose the product/security contract. The maintainable path is likely a Telegram-plugin read/cache or Bot API copy/forward design with explicit authorization, then a gated automatic link-expansion layer rather than a direct unconditional context injection path.
