## Summary

Keep open. Current main still treats Telegram message links as ordinary inbound text, and adding automatic linked-message resolution is a new Telegram plugin capability with privacy and side-effect policy decisions.

## What This Changes

The issue asks OpenClaw to resolve Telegram `t.me/c/...` message links and inject the referenced message content into Telegram session context.

## Best Possible Solution

Design a Telegram-owned, opt-in or policy-gated resolver that prefers already-ingested history, documents any accepted Bot API fallback, and injects linked content only when the chat/account policy allows it.

## Reproduction Assessment

Yes. Source inspection on current main shows a `t.me/c` URL in inbound text would pass through `expandTextLinks` into `rawBody` and `BodyForAgent`, while no Telegram fetch/read action exists; I did not run a live Telegram bot.

## Solution Assessment

No. The requested copy/forward/delete retrieval path is not clearly the best solution until maintainers decide the privacy, permission, retention, and visible-side-effect policy; a gated Telegram plugin resolver is the safer direction.
