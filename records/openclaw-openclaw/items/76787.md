## Summary

Keep open. Current main still treats Telegram `t.me/c/...` links as ordinary inbound text and exposes no Telegram read/fetch/link-resolution action, while resolving historical messages into context is a new security-sensitive Telegram capability that needs maintainer policy first.

## What This Changes

Request Telegram to resolve `t.me/c/...` message links and inject the referenced message content into the agent context.

## Best Possible Solution

Design a Telegram-owned, opt-in resolver that prefers already-ingested reply/history context when possible and documents any Bot API fallback with clear authorization, retention, and side-effect behavior.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence current-main path: a Telegram message containing a `t.me/c/...` URL flows through text expansion into `BodyForAgent`, and no Telegram read/fetch resolver is exposed; I did not run a live bot in this read-only review.

## Solution Assessment

No. The requested copy/forward/delete retrieval path is not yet the best maintainable fix because maintainers first need to choose the privacy, authorization, retention, and Telegram-side-effect policy.
