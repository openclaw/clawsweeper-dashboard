## Summary

Keep open. Current main still has no Telegram `t.me/c/...` resolver in the inbound context path, message action surface, config type, or Telegram docs; this is a new Telegram plugin capability with privacy/security policy to settle before implementation.

## What This Changes

Add Telegram `t.me/c/...` message-link parsing and referenced-message context injection for Telegram group/DM turns.

## Best Possible Solution

Design a Telegram-owned, explicitly gated resolver that prefers already-ingested same-chat history and documents any Bot API fallback before injecting linked historical content.

## Reproduction Assessment

Yes. Source inspection shows Telegram link text flows through `expandTextLinks` into `BodyForAgent`, while action/config/docs surfaces expose no message-link resolver; I did not run a live Telegram bot in this read-only review.

## Solution Assessment

No. The requested copy/forward/delete retrieval path is not clearly the best fix until maintainers decide privacy, permission, retention, and visible-side-effect policy; a gated Telegram plugin resolver is the safer direction.

## Security Review

Status: needs_attention

Summary: No patch was reviewed, but the requested capability is security-sensitive because historical Telegram content could enter agent context without being pasted into the current turn.

Concerns:

- **[medium] Define linked-message access policy first:**
  - body: Automatic `t.me/c` resolution can expose prior group or DM content to the agent, so maintainers should decide who may trigger it, which chats/messages are eligible, how much is retained, and what default is safe.
  - confidence: 0.84
- **[medium] Validate Bot API retrieval side effects:**
  - body: The proposed copy/forward-style retrieval can create visible messages or deletion/audit failure modes unless the Telegram plugin explicitly handles permissions, cleanup, failures, and logging.
  - confidence: 0.73
