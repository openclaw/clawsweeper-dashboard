## Summary

Keep open. Current main still treats Telegram `t.me/c/...` links as inbound text and does not expose a Telegram read/fetch/link-resolution action; implementing this is a new Telegram capability with privacy and side-effect policy questions.

## What This Changes

Request to make the Telegram plugin resolve `t.me/c` message links and inject the referenced Telegram message content into agent context.

## Best Possible Solution

Design a Telegram-owned, opt-in resolver that prefers already-ingested history when possible and documents any Bot API fallback with explicit access, retention, and side-effect rules.

## Reproduction Assessment

Yes. Source inspection on current main gives a high-confidence path: a Telegram message containing a `t.me/c/...` URL flows through text expansion into `BodyForAgent`/`RawBody`, with no resolver or fetch action; no live bot was run in this read-only review.

## Solution Assessment

No. The requested copy/forward/delete retrieval path is not yet the best maintainable implementation because maintainers first need to choose the privacy, authorization, retention, and Telegram-side-effect policy.
