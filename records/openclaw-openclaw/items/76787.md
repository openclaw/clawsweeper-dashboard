## Summary

Keep open: current main and the latest release still lack Telegram t.me/c link parsing, message fetch/read dispatch, and automatic link-to-context injection. The related Telegram read-action issue overlaps, but this item remains a distinct product/security decision about automatically expanding session context from referenced Telegram messages.

## What This Changes

Request to resolve Telegram t.me/c message links from inbound Telegram messages and inject the referenced message content into the agent session context.

## Best Possible Solution

Keep this open for a Telegram-plugin-owned design that aligns t.me/c link resolution with the broader read-action surface, explicit authorization, bounded retention, docs, focused tests, and live Telegram proof.

## Reproduction Assessment

Yes, at source level. Current main action discovery, runtime dispatch, inbound body construction, and session context assembly lack a Telegram read/fetch or t.me/c resolution path, so such links remain ordinary inbound text.

## Solution Assessment

Unclear until maintainers choose the product/security contract. The likely maintainable direction is to align link resolution with the broader Telegram read/cache design instead of adding an independent automatic fetch path first.
