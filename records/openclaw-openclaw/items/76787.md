## Summary

Keep open. Current main still passes Telegram `t.me/c/...` links through as text and exposes no Telegram-owned fetch/link-resolution action; this is a new, security-sensitive plugin capability that needs maintainer policy before implementation.

## What This Changes

The issue requests parsing Telegram `t.me/c/<chat>/<thread>/<message>` links, retrieving the referenced message through Telegram Bot API behavior, and injecting the resolved content into session context.

## Best Possible Solution

Design a Telegram-owned, opt-in or policy-gated resolver that prefers already-ingested history when possible and documents any accepted Bot API fallback before injecting linked content.

## Reproduction Assessment

Yes. Source inspection shows a `t.me/c/...` URL remains inbound text through `expandTextLinks` and `BodyForAgent`, while the Telegram action, config, and docs surfaces have no message-link resolver; I did not run a live Telegram bot in this read-only review.

## Solution Assessment

No. The requested copy/forward/delete retrieval path is plausible but not yet the best maintainable solution until maintainers define privacy, permission, retention, default-off gating, and side-effect policy.

## Security Review

Status: needs_attention

Summary: Needs attention because automatic Telegram linked-message resolution would allow a current turn to inject historical chat content that was not directly sent in that turn.

Concerns:

- **[medium] Define linked-message access policy first:**
  - body: A resolver could expose prior group or DM content to an agent based only on a link in the current turn, so maintainers should define who may trigger it, what history may be read, retention behavior, and the default policy before implementation.
  - confidence: 0.84
- **[medium] Validate Bot API retrieval side effects:**
  - body: The proposed copy/forward/delete approach may create visible messages, fail deletion, or behave differently across permissions and protected content, so any implementation needs explicit side-effect handling and tests.
  - confidence: 0.74
