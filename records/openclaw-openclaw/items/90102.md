## What This Changes

The PR guards QA runner manifest metadata reads before sorting/loading runtime surfaces and adds regression coverage for poisoned `qaRunners` and `rootDir` accessors.

## Best Possible Solution

Land the guarded manifest-row normalization with the focused regression test after normal maintainer review and CI, keeping the existing QA runner SDK subpath and runtime contract unchanged.

## Reproduction Assessment

Yes. From source, current main reads `plugin.qaRunners` during filtering and `rootDir` during sorting, so throwing accessors abort discovery before healthy QA runner plugins are reached.

## Solution Assessment

Yes. Normalizing only the manifest metadata fields needed for discovery is the narrowest maintainable fix, and the patch preserves the existing duplicate-command and runtime-registration mismatch failures.
