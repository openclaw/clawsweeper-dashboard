The PR changes Plugin SDK QA runner discovery to normalize only the manifest metadata needed for QA runner CLI registration and skip unreadable rows before sorting/loading runtime surfaces.

## Best Possible Solution

## Solution Assessment

Yes. Normalizing the small discovery metadata slice before sorting/loading is the narrowest maintainable fix I found, and it preserves existing failures for duplicate commands and manifest/runtime registration mismatches.
