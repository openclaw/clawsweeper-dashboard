## Summary

Keep open. Current main and the latest release still manage the macOS gateway lifecycle as a user LaunchAgent only, while this issue asks for a system LaunchDaemon lifecycle contract that remains unresolved and security-sensitive.

## What This Changes

Request to make macOS gateway stop and uninstall detect or manage system-level LaunchDaemons in addition to user LaunchAgents.

## Best Possible Solution

Define one documented opt-in macOS user/system launchd model, then align install, status, stop, uninstall, doctor guidance, docs, and regression tests around that contract.

## Reproduction Assessment

Yes for source-level reproduction: current main still resolves managed Darwin gateway lifecycle to `$HOME/Library/LaunchAgents` and `gui/<uid>`, with no stop/uninstall system-scope option. I did not run a live macOS LaunchDaemon scenario.

## Solution Assessment

Unclear as a final implementation plan. Detecting system daemons and giving actionable elevated guidance is the right direction, but the scope flag, privilege model, ownership, environment, and lifecycle semantics need maintainer design first.
