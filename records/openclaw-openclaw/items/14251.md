## Summary

Keep open: current main and the latest release still do not provide macOS LaunchDaemon lifecycle management for gateway stop/uninstall, and the remaining work needs a maintainer product/security decision before automation.

## What This Changes

The issue requests macOS gateway lifecycle support for detecting and managing system-level LaunchDaemons alongside user LaunchAgents.

## Best Possible Solution

Ship one coherent macOS launchd user/system scope model that covers this lifecycle gap and the install/status work in https://github.com/openclaw/openclaw/issues/51860, with explicit privilege UX, docs, and regression tests.

## Reproduction Assessment

Yes at source level: current main registers no system lifecycle flag, the Darwin service adapter is LaunchAgent-only, and stop/uninstall target the user gui/<uid> LaunchAgent path. I did not run a live macOS LaunchDaemon repro in this read-only review.

## Solution Assessment

Unclear: detecting both scopes and avoiding false lifecycle success is the right end state, but the exact --system/--daemon, sudo, ownership, and status semantics need maintainer direction before implementation.
