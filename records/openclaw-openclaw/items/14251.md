## Summary

Keep open: current main still manages the macOS gateway lifecycle through the per-user LaunchAgent path and gui launchd domain. `gateway status --deep` has diagnostic scanning, but stop/uninstall have no system scope and the remaining user/system launchd contract overlaps active related work.

## What This Changes

The issue requests macOS gateway stop, uninstall, and status behavior that can detect system-level LaunchDaemons and provide safe management or guidance.

## Best Possible Solution

Define one explicit macOS user/system launchd model, then align install, status, stop, uninstall, docs, and regression tests so a matching system LaunchDaemon is surfaced accurately and managed or guided safely.

## Reproduction Assessment

Yes, from source inspection rather than a live macOS run. Current main still routes lifecycle operations through the user LaunchAgent plist and `gui/<uid>` launchd target, with no system-scope option.

## Solution Assessment

Unclear. The requested direction is sound, but the exact `--system`/`--daemon` UX and privilege model should be decided once across this issue and #51860 before implementation.
