## Summary

Keep open. Current main still models the macOS managed gateway as a per-user LaunchAgent, while the related discussion keeps LaunchDaemon install/status in https://github.com/openclaw/openclaw/issues/51860 and leaves this issue as the stop/uninstall lifecycle tracker.

## What This Changes

The issue requests macOS gateway lifecycle commands that can detect and handle system-level LaunchDaemons instead of reporting success against only the user LaunchAgent path.

## Best Possible Solution

Define one documented opt-in macOS LaunchDaemon scope that covers install/status/stop/uninstall, privilege guidance, `UserName`, durable environment handling, doctor behavior, docs, and regression tests coordinated with the open install/status tracker.

## Reproduction Assessment

Yes for source reproduction: current main still resolves managed macOS lifecycle to `$HOME/Library/LaunchAgents` and `gui/<uid>`, with no stop/uninstall system-scope option. I did not run a live macOS LaunchDaemon scenario.

## Solution Assessment

Unclear as a final implementation plan. Detecting system daemons and providing actionable elevated guidance is the right direction, but the exact flag, privilege, ownership, and status/lifecycle semantics should be decided with the broader LaunchDaemon design thread.
