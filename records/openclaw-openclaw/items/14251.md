## Summary

Keep open. Current main and the latest release still manage the macOS gateway lifecycle as a user LaunchAgent; `status --deep` is only diagnostic, and the remaining LaunchDaemon lifecycle behavior needs maintainer product and security review.

## What This Changes

Requests that gateway stop/uninstall detect or manage `/Library/LaunchDaemons` macOS LaunchDaemons instead of reporting success from the user LaunchAgent path only.

## Best Possible Solution

Define one documented macOS user/system launchd model, then update lifecycle operations, status/doctor guidance, docs, and regression tests around that contract.

## Reproduction Assessment

Yes, source-level. Current main still has no launchd user/system scope on gateway stop or uninstall, so a system-domain LaunchDaemon is outside the managed lifecycle path.

## Solution Assessment

Unclear. LaunchDaemon detection and management is the right direction, but the exact `--system` or `--daemon` flag, privilege flow, and ownership model need maintainer approval before implementation.
