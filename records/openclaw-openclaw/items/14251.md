## Summary

Keep open. Current main still manages macOS gateway lifecycle as a per-user LaunchAgent, while system-level LaunchDaemon stop/uninstall support remains unresolved and product-sensitive; https://github.com/openclaw/openclaw/issues/51860 tracks the related install/status design rather than replacing this lifecycle-specific issue.

## What This Changes

The issue requests macOS gateway stop/uninstall support for `/Library/LaunchDaemons` services, including detection, clearer guidance, and possibly a `--system` lifecycle flag.

## Best Possible Solution

Define one documented opt-in macOS LaunchDaemon model, then align install, status, stop, uninstall, doctor guidance, docs, and regression tests around that contract.

## Reproduction Assessment

Yes for source reproduction: current main still resolves managed macOS lifecycle to `$HOME/Library/LaunchAgents` and `gui/<uid>`, with no stop/uninstall system-scope option. I did not run a live macOS LaunchDaemon scenario.

## Solution Assessment

Unclear as a final implementation plan. Detecting system daemons and giving actionable elevated guidance is the right direction, but the exact flag, privilege, ownership, environment, status, and lifecycle semantics should be decided with the broader LaunchDaemon design thread.
