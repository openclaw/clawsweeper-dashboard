## Label Justifications

- `P2`: This is a normal-priority lifecycle improvement affecting non-default custom macOS system deployments with a bounded platform blast radius.
- `impact:security`: Supporting root-owned LaunchDaemons crosses service ownership, user identity, secret environment, and elevated-command boundaries.
- `impact:crash-loop`: A KeepAlive system daemon can respawn after ineffective user-scope stop attempts and continuously preserve gateway port conflicts.

## Summary

Current main still has a source-verifiable lifecycle gap: macOS gateway stop and uninstall manage only the user LaunchAgent, while system-level services are merely discovered by `status --deep`. This should remain open as the lifecycle companion to the broader install/status request because direct LaunchDaemon control requires a shared maintainer decision about ownership, service identity, and elevation.

## What This Changes

Add accurate detection, reporting, and safe stop/uninstall handling for gateways running as macOS system LaunchDaemons.

## Best Possible Solution

Define one explicit macOS user/system launchd model shared with https://github.com/openclaw/openclaw/issues/51860, keep the managed user LaunchAgent as the safe default, reuse deep-scan facts for truthful warnings, and add operator-controlled system lifecycle operations only with verified ownership, user identity, privilege guidance, docs, and regression coverage.

## Reproduction Assessment

Yes, at source level: current main routes Darwin lifecycle operations only through LaunchAgent helpers in the user GUI domain, while `status --deep` separately performs diagnostic system-service discovery. A live macOS system-domain run was not performed in this read-only review.

## Solution Assessment

Unclear as proposed: detecting the system daemon and avoiding a misleading success result are sound, but the proposed flags and direct mutation are not the best fix until maintainers choose one shared ownership, identity, and elevation contract with the companion install/status work.
