## Summary

Keep open. Current main and the latest release still manage macOS gateway lifecycle as a user LaunchAgent; `gateway status --deep` is diagnostic discovery, not system LaunchDaemon stop/uninstall support, and the remaining work needs a shared product/security decision with https://github.com/openclaw/openclaw/issues/51860.

## What This Changes

Issue requests better macOS gateway lifecycle handling for system-level LaunchDaemons so stop/uninstall/status do not report user LaunchAgent success while a system daemon remains active.

## Best Possible Solution

Define one explicit macOS launchd scope contract shared by install, status, stop, uninstall, doctor, docs, and tests; keep user LaunchAgent as the safe default and add only an explicit, audited system LaunchDaemon path or guidance flow.

## Reproduction Assessment

Yes, at source level with high confidence. Current main manages only the user LaunchAgent plist and GUI domain, while system LaunchDaemon discovery is diagnostic and no system-scope lifecycle flag exists.

## Solution Assessment

No, the issue's isolated helper/flag sketch is only a plausible direction. The best fix is a shared launchd scope model with explicit privilege, execution-user, coexistence, lifecycle, status, doctor, docs, and tests.
