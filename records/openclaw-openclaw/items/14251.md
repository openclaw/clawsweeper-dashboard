## Summary

Keep open. Current main still manages the macOS Gateway lifecycle as a per-user LaunchAgent, while system LaunchDaemon stop/uninstall support remains unresolved and should be coordinated with the open install/status tracker at https://github.com/openclaw/openclaw/issues/51860; the earlier LaunchDaemon PR at https://github.com/openclaw/openclaw/pull/33489 was closed unmerged after review found unsafe auto-scope behavior.

## What This Changes

The issue requests macOS Gateway lifecycle commands that detect and manage or guide users through stopping/uninstalling system LaunchDaemons under `/Library/LaunchDaemons`.

## Best Possible Solution

Define one documented opt-in macOS user/system launchd model, then align install, status, stop, uninstall, doctor guidance, docs, and regression tests around that contract.

## Reproduction Assessment

Yes, for source-level reproduction: current main still routes managed Darwin Gateway lifecycle operations to `$HOME/Library/LaunchAgents` and `gui/<uid>`, with no stop/uninstall system-scope option. I did not run a live macOS LaunchDaemon scenario.

## Solution Assessment

Unclear as a final implementation plan. Detecting system daemons and giving actionable elevated guidance is directionally right, but the scope flag, privilege model, ownership, environment, and lifecycle semantics need maintainer design first.
