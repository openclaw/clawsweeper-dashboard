## Summary

Keep open: current main still routes macOS gateway stop/uninstall through the per-user LaunchAgent path and `gui/<uid>` launchd domain. The remaining work is a LaunchDaemon lifecycle feature and privilege-UX decision tied to #51860, not an implemented cleanup.

## What This Changes

Request support for detecting and managing macOS system-level gateway LaunchDaemons in gateway stop, uninstall, and status lifecycle commands.

## Best Possible Solution

Define one macOS user/system launchd model, then align install, status, stop, uninstall, docs, and regression tests so system LaunchDaemons are surfaced accurately and managed or guided safely.

## Reproduction Assessment

Yes, from source inspection rather than a live macOS run. Current main still routes lifecycle operations through the user LaunchAgent plist and `gui/<uid>` target, with no system-scope lifecycle option.

## Solution Assessment

Unclear. The proposed detection and guidance are directionally sound, but the exact CLI flag, privilege model, and managed-versus-external daemon ownership should be decided once with #51860 before implementation.
