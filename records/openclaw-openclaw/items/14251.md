## Summary

Keep open. Current main still performs macOS gateway stop/uninstall through user LaunchAgent helpers only; `status --deep` adds diagnostic discovery, but not system-scope lifecycle management, and #51860 still needs a unified LaunchDaemon contract.

## What This Changes

Request gateway stop/uninstall/status support for detecting and safely handling macOS system-level LaunchDaemons instead of only user LaunchAgents.

## Best Possible Solution

Define one explicit macOS user/system launchd scope model, then update lifecycle, status, docs, and regression tests so system LaunchDaemons are detected accurately and privileged actions require explicit operator intent.

## Reproduction Assessment

Yes, source-level. With `ai.openclaw.gateway` loaded in the `system` launchd domain and no matching user LaunchAgent, current main checks `gui/<uid>` and `~/Library/LaunchAgents` for lifecycle operations.

## Solution Assessment

Unclear. The requested direction is valid, but the maintainable fix should be chosen with #51860 so install, status, stop, uninstall, sudo guidance, and ownership semantics stay consistent.
