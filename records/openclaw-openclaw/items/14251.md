## Summary

Keep open. Current main still uses the macOS user LaunchAgent path for gateway stop/uninstall, while `status --deep` is only diagnostic and the broader system LaunchDaemon contract remains open in #51860.

## What This Changes

Request to make gateway stop, uninstall, and status detect and guide management of macOS system LaunchDaemons in addition to user LaunchAgents.

## Best Possible Solution

Define one macOS user/system launchd scope contract, then update lifecycle, status, docs, and regression tests so system LaunchDaemons are detected accurately and privileged actions require explicit operator intent.

## Reproduction Assessment

Yes. A high-confidence source-level reproduction exists: with `ai.openclaw.gateway` loaded in the `system` launchd domain and no matching user LaunchAgent, current main checks `gui/<uid>` and `~/Library/LaunchAgents` for lifecycle operations.

## Solution Assessment

Unclear. The requested direction is valid, but the maintainable solution should be chosen with #51860 so install, status, stop, uninstall, sudo guidance, and ownership semantics stay consistent.
