## Summary

Keep open. Current main still manages the macOS gateway lifecycle through the user LaunchAgent adapter, so a system LaunchDaemon can remain outside stop/uninstall; the requested scope model is valid but needs maintainer design before implementation.

## What This Changes

Request gateway stop/uninstall/status lifecycle handling for system-scope macOS LaunchDaemon installs under /Library/LaunchDaemons.

## Best Possible Solution

Define one documented macOS launchd user/system scope model, then make gateway status, stop, uninstall, and install-mode guidance consistently detect default LaunchDaemons, avoid false user-scope success, and provide safe elevated-command guidance or explicit system lifecycle flags.

## Reproduction Assessment

Yes for source reproduction: current main resolves the macOS gateway service to ~/Library/LaunchAgents and gui/<uid>, while stop/uninstall register no system scope flag. I did not run a live macOS LaunchDaemon scenario.

## Solution Assessment

Unclear as an implementation plan: detecting the system daemon and printing actionable guidance is the right boundary, but the exact --system/--daemon flag and privilege model should be decided with the related LaunchDaemon scope work before coding.
