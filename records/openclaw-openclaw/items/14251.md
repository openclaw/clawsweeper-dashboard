## Summary

Keep open. Current main and v2026.6.10 still manage macOS gateway stop/uninstall through the user LaunchAgent path and GUI launchd domain, while system LaunchDaemon handling remains a security/product decision shared with the open install/status companion.

## What This Changes

Request to make gateway stop/uninstall detect and safely handle macOS system-level LaunchDaemons instead of only user LaunchAgents.

## Best Possible Solution

Define one explicit macOS launchd user/system scope contract across install, status, stop, uninstall, doctor, docs, and tests, keeping user LaunchAgent as the safe default and adding only an audited explicit LaunchDaemon path or guidance flow.

## Reproduction Assessment

Yes. Source inspection shows the current Darwin gateway lifecycle is wired to user LaunchAgent paths and gui/<uid> launchctl targets with no lifecycle system-scope option; I did not run a live macOS LaunchDaemon scenario in this read-only review.

## Solution Assessment

No. The issue's isolated --system helper sketch is plausible, but the maintainable fix is a shared launchd scope model covering privilege, execution user, coexistence, lifecycle, status, doctor, docs, and tests.
