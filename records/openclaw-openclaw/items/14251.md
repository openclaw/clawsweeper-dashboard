## Summary

Keep open. Current main still manages the macOS gateway lifecycle as a user LaunchAgent only; system LaunchDaemons are only partially visible through status diagnostics and are not stop/uninstall targets.

## What This Changes

Request lifecycle support or guidance for stopping and uninstalling macOS system-level gateway LaunchDaemons in addition to user LaunchAgents.

## Best Possible Solution

Define one documented macOS launchd user/system scope model, then make gateway status, stop, uninstall, and any install-mode guidance consistently detect default LaunchDaemons, avoid false user-scope success, and provide safe elevated-command guidance or explicit system lifecycle flags.

## Reproduction Assessment

Yes for source reproduction: current main resolves the macOS gateway service to ~/Library/LaunchAgents and gui/<uid>, while stop/uninstall register no system scope flag. A default /Library/LaunchDaemons/ai.openclaw.gateway.plist remains outside those lifecycle commands, though I did not run a live macOS daemon scenario.

## Solution Assessment

Unclear as an implementation plan: detecting the system daemon and printing actionable guidance is the right boundary, but the exact --system/--daemon flag and privilege model should be decided with the related LaunchDaemon scope work before coding.
