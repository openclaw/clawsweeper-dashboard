## Summary

Keep open. Current main still intentionally treats Control UI/WebChat sends as internal and prevents WebChat clients from inheriting Feishu delivery routes, so the remaining Feishu plus dashboard sync request is a real product/routing-boundary decision rather than an already-fixed bug.

## What This Changes

The issue requests real-time bidirectional message and task-status synchronization between the local Control UI dashboard and the Feishu bot channel.

## Best Possible Solution

Keep this as a maintainer-owned cross-surface routing decision: either add an explicit opt-in Feishu/WebChat mirror or delivery mode with route-safety tests, or document that Feishu and WebChat are separate deterministic surfaces.

## Reproduction Assessment

Yes for the code-level boundary: Control UI sends are deliver:false WebChat turns, and current main blocks WebChat route inheritance to Feishu. The full live delay and task-status claim still needs Feishu runtime logs or a live DM/group/topic reproduction.

## Solution Assessment

Unclear as a bug fix. The safest solution is not implicit always-on sync, but a maintainer-approved opt-in mirror/delivery contract or clearer docs for the current routing boundary.
