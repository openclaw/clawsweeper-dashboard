## Summary

Keep open. Current main still preserves a WebChat/internal delivery boundary: Control UI sends use `deliver: false`, WebChat clients cannot inherit external channel routes, and Feishu inbound dispatch is routed through Feishu session context rather than a bidirectional dashboard mirror. There is active-session refresh plumbing for `session.message`, but it does not satisfy the reported expectation that both Feishu and the dashboard always have identical real-time history and delivery.

## What This Changes

The issue requests real-time bidirectional message and task-status synchronization between the local dashboard and the Feishu bot channel.

## Best Possible Solution

Decide the product contract for Feishu-backed dashboard sessions. If bidirectional mirroring is in scope, add an explicit opt-in Feishu/WebChat mirror or delivery mode that preserves channel route safety, covers Feishu DM/group/topic sessions, updates Control UI state handling, and documents the setting. If it is intentionally out of scope, make the boundary clear in Control UI, WebChat, channel-routing, and Feishu docs before closing.
