## What This Changes

The PR infers an effective provider from provider-qualified `modelId` values in `resolveModelRuntimePolicy` and adds regression tests for provider-model and provider-wide runtime policy lookup.

## Best Possible Solution

Land the narrow resolver fix if maintainers accept that documented provider/model runtime policy should apply to provider-qualified model ids even when the caller omits `provider`.

## Reproduction Assessment

Yes. Source inspection shows current main cannot reach `models.providers.<provider>` when `provider` is empty and the provider appears only inside `modelId`, while the PR adds targeted resolver cases for that path.

## Solution Assessment

Yes. Inferring one normalized effective provider at the resolver entry and reusing existing match helpers is the narrow owner-boundary fix; adding provider-catalog special cases would duplicate policy logic.
