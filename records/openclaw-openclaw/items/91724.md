The PR infers an effective provider from provider-qualified model ids in model runtime policy resolution and adds regression tests for provider model and provider fallback runtime policies.

## Best Possible Solution

Land the narrow resolver fix if maintainers accept that documented provider/model runtime policy should apply to qualified model ids even when the caller omits provider.

Yes. Inferring one normalized effective provider at the resolver entry and reusing existing provider/model match helpers is the narrow owner-boundary fix; changing each caller would duplicate policy and miss direct resolver users.
