## Summary

Keep open: the PR fixes fresh self-signed cert generation, but the advertised existing-cert upgrade path is not wired into the runtime and the new test can pass with a CN-only cert, so it is not merge-ready.

## What This Changes

The PR adds SANs to newly generated gateway self-signed certificates and adds helper/test coverage intended to upgrade older CN-only generated certs.

## Best Possible Solution

Generate SAN-bearing certs for fresh installs, invoke a narrowly guarded upgrade before loading older OpenClaw CN-only certs, assert the SAN and Node identity check in tests, and document or explicitly accept any fingerprint rotation behavior.
