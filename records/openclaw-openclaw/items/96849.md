## Best Possible Solution

Verify the main-session Discord path on current main/latest release with a structured thinking-block transcript or live MiniMax-style repro, then fix at the shared outbound visible-text boundary so type:"thinking" blocks cannot enter channel payloads, with a Discord regression test.

## Reproduction Assessment

No high-confidence current-main reproduction yet. The issue gives concrete release steps and transcript shape, but this read-only review did not run live Discord/MiniMax and current main has visible-text extraction plus later Discord sanitization that may change the outcome.

## Solution Assessment

Unclear. The best fix is likely a shared outbound/assistant visible-text invariant with Discord coverage, not a Discord-only string scrubber, because plain thinking prose cannot be reliably removed once it has been flattened into payload.text.
