## Summary

Keep open: the help-text direction is useful, but the branch currently misdocuments the cron default as UTC and lacks required rendered-help proof from the contributor.

## What This Changes

The PR updates `cron add` and `cron edit` help text so `--tz` mentions `--cron` plus offset-less `--at`, and adjusts `cron edit --at` wording.

## Best Possible Solution

Revise the help strings to mention both cron expressions and offset-less one-shot datetimes while preserving their separate defaults, then attach rendered `cron add` and `cron edit` help proof.

## Reproduction Assessment

Yes. Source inspection is enough: the PR adds a combined UTC default, while current runtime, tests, and docs keep cron's omitted-`tz` default as the gateway host timezone.

## Solution Assessment

No, not as written. The scope is narrow and maintainable, but the wording needs to separate cron's host-local default from one-shot `--at` UTC/default behavior.
