## Summary

Keep open: the help-text request is still valid, but this PR should not merge as-is because it documents a UTC default for cron schedules even though current main keeps cron-without-`--tz` on the Gateway host timezone. The external PR also lacks after-fix real behavior proof.

## What This Changes

The PR changes `cron add` and `cron edit` CLI help strings so `--tz` mentions both `--cron` expressions and offset-less `--at` values.

## Best Possible Solution

Refresh the two help strings so `--tz` mentions cron expressions and offset-less one-shot datetimes while preserving the distinct defaults: cron without `--tz` uses the Gateway host timezone, and offset-less `--at` without `--tz` is UTC.

## Reproduction Assessment

Yes. Source inspection gives a high-confidence path: the PR head adds `default: UTC` to both `--tz` help strings, while current main's runtime, tests, and docs keep cron's omitted-`tz` default as the Gateway host timezone.

## Solution Assessment

No. The requested help-text direction is narrow and valid, but the current PR wording is not the best solution because it collapses cron and one-shot defaults into a single UTC claim.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.94

Full review comments:

- **[P2] Remove the UTC default from cron add help:** `src/cli/cron-cli/register.cron-add.ts:89`
  - body: The new `--tz` help says the default is UTC for both `--cron` and offset-less `--at`, but cron schedules without `tz` fall back to the Gateway host local timezone via `resolveCronTimezone`. This can lead users to omit `--tz` when they need UTC cron execution and then get local-time scheduling instead.
  - confidence: 0.94
- **[P2] Remove the UTC default from cron edit help:** `src/cli/cron-cli/register.cron-edit.ts:59`
  - body: The edit command gets the same incorrect UTC default claim. Editing a job with `--cron` and no `--tz` keeps cron evaluation in the Gateway host local timezone, so this help would document the wrong behavior for recurring schedules.
  - confidence: 0.94
