## Summary

Keep open: this is a focused help-text PR for a real cron CLI discoverability gap, but the current branch documents the wrong cron default and lacks required rendered-help proof before merge.

## What This Changes

The branch updates `cron add` and `cron edit` help so `--tz` says it applies to `--cron` and offset-less `--at` values, and makes `cron edit --at` mention `--tz`.

## Best Possible Solution

Revise the help strings to mention both cron expressions and offset-less one-shot datetimes while preserving separate defaults, then show rendered `cron add` and `cron edit` help output.

## Reproduction Assessment

Yes. Source inspection is enough: the PR adds a combined UTC-default claim, while current runtime, tests, and docs keep cron's omitted-`tz` default as the Gateway host timezone.

## Solution Assessment

No, not as written. The help-text direction is narrow and maintainable, but this PR should preserve the separate cron and one-shot defaults instead of collapsing them into one UTC default.
