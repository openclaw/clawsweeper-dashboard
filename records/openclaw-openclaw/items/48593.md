## Summary

Keep open: current main still has the reported final-tag leak, but this PR should not merge as-is because the widened scanners can misclassify custom or malformed tags in a strict suppression path and the contributor has not provided after-fix real behavior proof.

## What This Changes

The PR broadens final/thinking tag regexes in embedded streaming, sanitizer/fallback paths, live-test helpers, and regression tests, including self-closing tag handling.

## Best Possible Solution

Land one reconciled constrained or shared tag parser that strips intended bare, attributed, split, and self-closing reasoning/final tags while failing closed on malformed or custom tags, with focused tests and redacted real chat proof.

## Reproduction Assessment

Yes, at source level: current main leaves attributed and self-closing final tags visible in duplicated streaming/sanitizer patterns, and a May 14 comment reports the same leak on v2026.5.7 Telegram. I did not run a live Telegram or Gemini session in this read-only review.

## Solution Assessment

No, not as-is. The touched surfaces are right, but the widened regex should use a constrained tag-name/attribute grammar or shared helper so strict final-tag enforcement fails closed before merge.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88
