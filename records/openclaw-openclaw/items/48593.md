## Summary

Keep open: current main still has source-level final-tag leak gaps, but this PR cannot merge as-is because it is conflicting, lacks after-fix real behavior proof, and widens a strict suppression parser in a way that can emit malformed-tag tail text.

## What This Changes

The PR broadens embedded reasoning/final tag matching for Gemini-style attributed and self-closing tags across streaming, sanitizer/fallback paths, live-test helper regexes, and regression tests.

## Best Possible Solution

Land one reconciled shared or constrained tag parser that strips intended bare, attributed, split, and self-closing reasoning/final tags while failing closed on malformed/custom tags, with focused regression coverage and redacted real chat proof.

## Reproduction Assessment

Yes, at source level: current main's streaming and sanitizer regexes strip bare `<final>` tags but leave attributed and self-closing variants visible. I did not run a live Telegram or Gemini session in this read-only review.

## Solution Assessment

No, not as-is. The touched surfaces are right, but the broadened regex needs a constrained tag-name/attribute grammar or shared helper so strict final-tag enforcement still fails closed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88
