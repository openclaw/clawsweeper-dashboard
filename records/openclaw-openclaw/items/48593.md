## Summary

Keep open: the PR addresses a real Gemini final/thinking tag leak, but the widened tag regex can misclassify malformed or custom tags in a strict content-suppression path, and the external contributor still has not provided after-fix real behavior proof.

## What This Changes

The PR broadens embedded final/thinking tag stripping for attributed and self-closing tags and updates sanitizer, streaming, live-test helper, and regression tests.

## Best Possible Solution

Land one reconciled shared or constrained tag parser that accepts intended bare, attributed, split, and self-closing reasoning/final tags while failing closed on malformed/custom tags, with focused regression tests and redacted real chat proof.

## Reproduction Assessment

Yes, at source level: current main's streaming and sanitizer regexes strip bare `<final>` tags but leave attributed and self-closing variants. I did not run a live Telegram or Gemini session in this read-only review.

## Solution Assessment

No, not as-is. The touched paths are right, but the broadened regex needs a constrained tag-name/attribute grammar or shared helper so strict final-tag enforcement still fails closed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Constrain reasoning tag matches to complete tag names:** `src/agents/pi-embedded-subscribe.ts:24-25`
  - body: The widened `\b[^<>]*` scanners also match custom tags like `<final-result>` and partial malformed text like `<final reason="a>b">`. In enforced final mode that can mark a final block as seen and emit content that should remain suppressed, so the parser should require a real tag-name terminator and fail closed on malformed attributes.
  - confidence: 0.88
