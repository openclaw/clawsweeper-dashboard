## Summary

Keep this PR open: current main still has the narrow final-tag matcher in the streaming and sanitizer paths, but the proposed broadened regex has a strict-mode content-suppression bug and the external contributor has not provided real after-fix behavior proof.

## What This Changes

The PR broadens embedded final/thinking tag regexes for attributed and self-closing tags, adds self-closing handling, and updates related regression and live-test helpers.

## Best Possible Solution

Land one reconciled parser/helper fix that strips intended bare, attributed, split, and self-closing reasoning/final tags while failing closed on malformed/custom tags, with redacted real Telegram or equivalent live proof.

## Reproduction Assessment

Yes, at source level: current main's final-tag regex strips bare tags but leaves attributed and self-closing final tags in the streaming/sanitizer paths. I did not run a live Telegram/Gemini session in this read-only pass.

## Solution Assessment

No, not as-is. The touched surfaces are right, but the broadened tag matching needs constrained tag-name/attribute parsing or a shared helper so strict final-tag enforcement still fails closed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Constrain reasoning tag matches to complete tag forms:** `src/agents/pi-embedded-subscribe.ts:24-25`
  - body: The widened `\b[^<>]*` scanners match custom tags like `<final-result>` and partially match malformed text such as `<final reason="a>b">`. In enforced final mode that marks a block as seen and emits trailing text instead of suppressing hidden content, so the parser needs a real tag terminator/attribute grammar that fails closed.
