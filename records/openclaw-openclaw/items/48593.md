## Summary

Keep open: the final-tag leak is still source-reproducible on current main, but this PR should not merge as-is because its broadened matcher can fail open in strict final-tag suppression and it overlaps https://github.com/openclaw/openclaw/pull/66225 plus https://github.com/openclaw/openclaw/issues/65867.

## What This Changes

The PR broadens Gemini final/thinking tag matching in embedded streaming, error/message cleanup, shared reasoning-tag stripping, live checks, and regression tests for attributed and self-closing tags.

## Best Possible Solution

Land one reconciled parser/helper fix that strips intended bare, attributed, split, and self-closing reasoning/final tags while rejecting custom or malformed tag names and preserving the broader plain-text Gemini reasoning leak for its own tracker.

## Reproduction Assessment

Yes. A source-level regex probe against current main reproduces the narrow matcher leaving `<final id="1">` and `<final/>`, and the same probe demonstrates the PR's strict-mode malformed-tag regression.

## Solution Assessment

No, not as-is. The touched surfaces are right, but the safest fix should centralize or tighten tag parsing so strict final-tag enforcement only accepts complete intended tags and fails closed on malformed markup.

## Review Findings

Overall correctness: patch is incorrect
