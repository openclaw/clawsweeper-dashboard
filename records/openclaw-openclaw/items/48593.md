## Summary

Keep this PR open because current main still has the delivery-path `<final>` leak shape the PR is trying to address, but the proposed widened matcher is not safe to merge as-is. The PR overlaps the open #66225/#65867 work and still needs an exact tag-name delimiter fix before maintainers choose the canonical implementation.

## What This Changes

The PR widens final/thinking tag regexes in embedded streaming, final-tag cleanup, message-end fallback, shared reasoning-tag cleanup, and live-test leak checks, while adding self-closing/attributed reasoning-tag regression coverage.

## Best Possible Solution

Land or replace this with a current-main patch that uses one exact reasoning-tag matcher/parser across streaming cleanup, user-facing sanitization, and leak assertions, supports attributed and self-closing `<final>`/`<think>` tags, and rejects hyphenated custom tag names. Keep the broader plain-text Gemini reasoning leak tracked separately.

## Reproduction Assessment

Yes. A focused reproduction is to stream or sanitize `<final id="1">Visible</final>` or `<final/>` through the current delivery paths; code inspection and a regex probe show current main misses those opening/self-closing forms.

## Solution Assessment

No, not as-is. The direction is right, but `\b[^<>]*` is too broad for tag-name termination and the branch needs rebasing onto the current scanner layout before it is a maintainable fix.
