## Summary

Keep open: the PR targets a real Gemini final/thinking tag leak that is still visible in current `main`, but the current patch has a blocking strict-parser regression and lacks after-fix real behavior proof.

## What This Changes

The PR broadens final/thinking tag stripping for Gemini-style attributed and self-closing tags across embedded streaming, sanitizer/fallback paths, live-test helpers, and regression tests.

## Best Possible Solution

Land one reconciled shared or constrained tag parser that strips intended bare, attributed, split, and self-closing reasoning/final tags while failing closed on malformed or custom tags, with focused coverage and redacted real chat proof.

## Reproduction Assessment

Yes, at source level. Current `main` leaves attributed and self-closing final tags visible in the duplicated streaming/sanitizer paths, though I did not run a live Telegram or Gemini session in this read-only review.

## Solution Assessment

No, not as-is. The touched surfaces are right, but the widened regex should use a constrained tag-name/attribute grammar or shared helper so strict final-tag enforcement fails closed.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Constrain reasoning tag matches to complete tag names:** `src/agents/pi-embedded-subscribe.ts:24-25`
  - body: The widened `\b[^<>]*` scanners also match custom or malformed tag text such as `<final-result>` and `<final reason="a>b">`. In `enforceFinalTag` mode that can mark a final block as seen and emit text that should stay suppressed, so the tag terminator and attribute grammar need to be constrained.
  - confidence: 0.88

## Security Review

Status: needs_attention

Summary: The diff does not touch dependencies or workflows, but it changes a strict content-suppression parser in a way that can leak text after malformed or custom final tags.
