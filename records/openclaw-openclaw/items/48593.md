## Summary

Keep open: current main still misses the attributed/self-closing final-tag leak class, but this PR is not ready to merge because its widened matcher also accepts hyphenated custom tag names and overlaps newer #66225/#65867 work.

## What This Changes

The branch widens final/thinking tag cleanup in embedded delivery, user-facing sanitization/error cleanup, shared reasoning-tag handling, and live/test coverage to handle attributed and self-closing Gemini tags.

## Best Possible Solution

Land or replace this with a current-main patch that uses one exact reasoning-tag matcher across streaming cleanup, sanitization, and leak assertions, supports attributed/self-closing `<final>` and `<think>` tags, and rejects hyphenated custom tag names.

## Reproduction Assessment

Yes. A focused reproduction is to stream or sanitize `<final id="1">Visible</final>` and `<final/>hidden<final>Visible</final>` through the current delivery paths; code inspection plus the regex probe show current main misses those forms.

## Solution Assessment

No, not as-is. The intended direction is right, but `\b[^<>]*` is too broad for tag-name termination; the safer fix should accept only real tag delimiters after the tag name, such as whitespace, `/`, or `>`.

## Review Findings
