## Summary

Keep this PR open: it targets a real Gemini final/reasoning tag leak that still exists on current main, but the patch is not merge-ready because the broadened regex can weaken strict suppression and the external PR still lacks after-fix real behavior proof.

## What This Changes

The branch broadens embedded final/thinking tag matching for attributed and self-closing Gemini wrappers, updates related final-tag cleanup/test matchers, and adds focused regression tests.

## Best Possible Solution

Land one constrained or shared tag parser that strips intended bare, attributed, split, and self-closing reasoning/final wrappers while failing closed on malformed/custom tags, with focused regression coverage and redacted real Telegram or equivalent proof.

## Reproduction Assessment

Yes, at source level: current main's streaming and sanitizer regexes leave attributed and self-closing final tags visible, and a later user comment reports the same family on OpenClaw 2026.5.7 Telegram group delivery. I did not run a live Telegram/Gemini session in this read-only review.

## Solution Assessment

No, not as-is. The touched surfaces are right, but the widened regex should use a constrained tag-name/attribute grammar or shared helper so strict final-tag enforcement fails closed before merge.
