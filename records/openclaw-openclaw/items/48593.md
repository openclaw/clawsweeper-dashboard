## Summary

Keep open: current main still has the narrow final-tag matchers, but this PR is blocked by a strict parser regression, missing contributor real-behavior proof, and overlap with another open implementation.

## What This Changes

The PR broadens embedded final/thinking tag regexes for attributed and self-closing tags, adds self-closing handling, and updates related regression/live-test helpers.

## Best Possible Solution

Land one reconciled parser/helper fix that strips intended bare, attributed, split, and self-closing reasoning/final tags while failing closed on malformed/custom tags.

## Reproduction Assessment

Yes, at source level: the current regex leaves attributed and self-closing final tags, and the PR-style matcher can be probed to expose the malformed-tag strict-mode regression. I did not run a live Telegram/Gemini session in this read-only pass.

## Solution Assessment

No, not as-is. The touched surfaces are right, but the tag parsing needs to be constrained or centralized before this can safely land.

## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Constrain reasoning tag matches to real delimiters:** `src/agents/pi-embedded-subscribe.ts:24-25`
  - body: The widened `\b[^<>]*` scanners also match custom tags such as `<final-result>` and can partially match malformed tags such as `<final reason="a>b">`. In `enforceFinalTag` mode that can mark a final block as seen and emit text that should stay suppressed, so require real tag-name delimiters and fail closed on malformed tag text.
  - confidence: 0.88
