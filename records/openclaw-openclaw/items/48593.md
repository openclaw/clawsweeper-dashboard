## Summary

Keep open: the PR targets a valid current-main Gemini final-tag leak, but the branch is currently dirty/unmergeable and the widened matcher can misclassify non-final or malformed tags in the strict suppression path.

## What This Changes

The PR broadens final/thinking tag cleanup in embedded streaming, message/error cleanup, shared reasoning-tag handling, live leak checks, and tests for attributed and self-closing Gemini tags.

## Best Possible Solution

Land a current-main repair or replacement that uses one exact, shared reasoning-tag parser for streaming and user-facing sanitization, supports intended attributed/self-closing tags, rejects custom or malformed tag names, and leaves the broader plain-text Gemini reasoning leak to its umbrella tracker.

## Reproduction Assessment

Yes. Source inspection plus a read-only regex probe show current main misses attributed and self-closing final tags in the streaming/sanitizer paths, and the same probe shows the PR's widened pattern misclassifies custom or malformed tags.

## Solution Assessment

No, not as-is. The intended surfaces are right, but the matcher should require real tag delimiters or a shared parser instead of `\b[^<>]*`, and the stale branch needs current-main sanitizer coverage and conflict resolution.
