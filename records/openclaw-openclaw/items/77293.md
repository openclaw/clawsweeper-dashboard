## Summary

Keep open: current main still routes CalVer correction host versions through the generic plugin API semver range check, and the linked open PR #77450 is the right vehicle to resolve that half before this issue closes.

## What This Changes

The issue reports that OpenClaw 2026.5.3-1 breaks plugin install/update flows because a hashed install runtime chunk is missing from the npm package and `2026.5.3-1` is rejected against plugin API ranges like `>=2026.5.3`.

## Best Possible Solution

Review and land #77450 for the plugin API comparison bug, then close this issue only after package artifact verification confirms the runtime-chunk report is not still present.

## Reproduction Assessment

Yes for the plugin API mismatch: source inspection shows a correction runtime can flow into the install compatibility gate and be compared as a prerelease below the base stable version. No high-confidence reproduction was established here for the missing hashed runtime chunk.

## Solution Assessment

Yes: the narrowest maintainable path is to normalize OpenClaw stable correction versions only in the plugin API compatibility check, as #77450 proposes, while keeping package tarball inventory checks as the artifact guardrail.
