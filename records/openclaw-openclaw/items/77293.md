## Summary

Keep open: the missing runtime chunk claim is not confirmed from current source evidence, but the plugin API mismatch remains source-reproducible for stable correction versions such as `2026.5.3-1`.

## What This Changes

The issue requests package fixes so `2026.5.3-1` includes required `install.runtime-*` chunks and treats stable correction versions as compatible with same-base plugin API floors.

## Best Possible Solution

Make plugin API compatibility correction-version-aware, or expose the base plugin API contract for stable correction releases, while preserving prerelease rejection and existing tarball dist-closure checks.

## Reproduction Assessment

Yes for the plugin API half: current source passes a correction-suffixed runtime version into a generic semver range check, so `2026.5.3-1` fails `>=2026.5.3`. The missing runtime chunk half is not high-confidence reproduced here.

## Solution Assessment

No, the current `2026.5.4` version bump only avoids this exact suffix and does not durably handle future stable correction releases. The narrow maintainable fix is to use OpenClaw release-version comparison or normalize the exposed plugin API contract for correction builds.
