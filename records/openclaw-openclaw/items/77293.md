The issue reports that the 2026.5.3-1 npm package breaks plugin installation/update because an install runtime chunk is missing and the runtime plugin API version is rejected by plugins requiring >=2026.5.3.

## Best Possible Solution

Teach plugin API compatibility to treat OpenClaw stable correction versions as compatible with same-date base floors while preserving real prerelease rejection, then verify the package dist inventory path separately.

## Reproduction Assessment

## Solution Assessment

No. Bumping current main to 2026.5.4 avoids this specific suffix, but the narrow maintainable fix is to make the compatibility check understand OpenClaw stable correction versions or expose a base plugin API version for them.
