## Summary

Keep open: current main still routes correction host versions such as `2026.5.3-1` through generic semver plugin API range checks, and an open closing PR already targets the remaining compatibility bug.

## What This Changes

The issue reports that `openclaw@2026.5.3-1` breaks plugin install/update flows because a hashed `install.runtime-*` chunk is missing and the runtime plugin API version is rejected against `>=2026.5.3`.

## Best Possible Solution

Review and land PR #77450 or an equivalent narrow fix that normalizes only stable CalVer correction host versions for plugin API range checks while keeping dist inventory validation as the package artifact guardrail.

## Reproduction Assessment

Yes for the plugin API mismatch by source inspection: a `2026.5.3-1` compatibility host version reaches `satisfiesPluginApiRange` and is compared as a prerelease below `2026.5.3`. The missing runtime chunk half is artifact-specific and was not live-reproduced here.

## Solution Assessment

Yes, the open PR direction is the narrow maintainable fix: normalize stable correction versions in the plugin API compatibility path without changing the visible runtime version or prerelease rejection semantics.
