## Summary

Keep open: current main still routes stable correction host versions through the generic plugin API semver range check, and open PR #77450 already targets that remaining compatibility bug; the missing runtime-chunk side has current main guardrails but was not live-revalidated against the published 2026.5.3-1 tarball.

## What This Changes

Report that npm global OpenClaw 2026.5.3-1 breaks plugin install/update because a hashed install runtime chunk is missing and the correction runtime version fails plugin API range checks.

## Best Possible Solution

Review and land #77450 or an equivalent narrow fix that normalizes stable CalVer correction host versions only for plugin API compatibility checks while retaining the current package tarball inventory guardrail.

## Reproduction Assessment

Yes for the plugin API mismatch by source inspection: a correction host version such as 2026.5.3-1 reaches satisfiesPluginApiRange and compares as a prerelease below >=2026.5.3. The missing runtime chunk half is artifact-specific and was not live-reproduced here.

## Solution Assessment

Yes, the linked PR direction is the narrow maintainable fix: normalize stable correction versions in the plugin API compatibility path without changing the visible runtime version or prerelease rejection semantics.
