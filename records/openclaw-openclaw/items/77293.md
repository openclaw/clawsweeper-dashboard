The issue reports npm plugin install/update failures in openclaw 2026.5.3-1 from a missing `install.runtime-*` dist chunk and a plugin API version mismatch against official plugins.

## Best Possible Solution

Patch plugin API compatibility so OpenClaw stable correction releases satisfy same-date plugin API floors, while keeping the existing package tarball inventory and import-closure checks intact.

Partially. Source inspection gives a high-confidence path for the plugin API failure through `satisfiesPluginApiRange("2026.5.3-1", ">=2026.5.3")`, but the missing runtime chunk was not reproduced from the current registry tarball listing.

## Solution Assessment
