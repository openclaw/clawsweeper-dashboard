## Summary

Current main and the v2026.6.6 release already carry Google Chat room thread context through the canonical inbound/threading/message-tool path, while this branch adds a hidden process-global fallback that can misroute messages and is no longer a useful landing candidate.

## What This Changes

The PR stores the most recent Google Chat room thread per space in process-global state and falls back to that thread when Google Chat sendText receives no explicit threadId or replyToId.

## Best Possible Solution

## Solution Assessment
