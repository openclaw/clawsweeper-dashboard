## Summary

Close as implemented: current main and shipped releases now escalate assistant timeouts to configured model fallbacks after one-way auth-profile rotation is exhausted, while intentionally keeping pure timeouts out of shared auth-profile cooldown state.

## What This Changes

The issue asks for model timeouts to trigger faster auth-profile or model fallback instead of spending the full turn timeout on repeated primary-provider attempts.

## Best Possible Solution

Keep the shipped timeout fallback behavior and leave broader retry-count, failure-class, or per-agent timeout policy changes to narrower open policy issues.

## Reproduction Assessment

No current failing reproduction remains. Source and E2E coverage show current main rotates profiles one-way and escalates timeout failures to configured model fallbacks after rotation is exhausted.

## Solution Assessment

Yes. The shipped fix addresses the observable slow timeout-to-fallback behavior without treating every timeout as shared auth-profile health, which avoids poisoning other sessions that share the same profile.
