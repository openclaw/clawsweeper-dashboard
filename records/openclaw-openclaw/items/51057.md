## Summary

Keep open. Current main still has the reported timeout/profile behavior, and nearby tests explicitly codify “timeout means rotate without cooldown,” so changing it needs a maintainer failover-policy decision rather than cleanup closure.

## What This Changes

The issue requests faster model fallback when auth profile timeouts keep cycling through profiles instead of cooling them down or escalating promptly.

## Best Possible Solution

Define a narrow timeout-as-transient-failure policy in the embedded failover/auth-profile path, with explicit threshold/cooldown semantics and focused tests, while leaving broader liveness/config work to the related timeout-design issues.

## Reproduction Assessment

Yes from source and tests, but not from a live outage run. Current main still skips timeout failure marking and has an e2e test asserting that timed-out profiles rotate without cooldown.

## Solution Assessment

Unclear. Reusing the existing transient cooldown/fallback mechanisms looks plausible, but the threshold, cooldown duration, and interaction with broader timeout/liveness work need maintainer policy before implementation.
