Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Clear pending channel refreshes after disconnect:** `ui/src/ui/controllers/channels.ts:171-179`
  - body: When a stronger request is queued during an in-flight load, this branch only clears it if the state is still connected. If the UI disconnects first, the stale request can replay after reconnect during an unrelated load, causing avoidable heavy status work and unexpected state replacement.
  - confidence: 0.87
- **[P2] Gate missing-gateway guidance on failed probes:** `src/wizard/setup.finalize.ts:295-305`
  - body: This `else` branch also runs when the gateway probe succeeds, so successful setup runs can show “Gateway not detected yet” even though the Gateway is reachable. Only show this note for the failed-probe/no-daemon case.
  - confidence: 0.9
- **[P2] Reuse the configured result in summary snapshots:** `src/gateway/server-methods/channels.ts:347-354`
  - body: The summary-only path computes `configured` before probe/audit, but `buildChannelAccountSnapshot` can call `plugin.config.isConfigured` again for accounts without an inline `configured` field. That breaks the new lightweight status path's single configuration-check guarantee.
  - confidence: 0.8
