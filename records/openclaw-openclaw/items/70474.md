## Review Findings

Overall correctness: patch is incorrect

Overall confidence: 0.88

Full review comments:

- **[P2] Clear queued channel refreshes when disconnecting:** `ui/src/ui/controllers/channels.ts:171-180`
  - body: When a stronger request is queued during an in-flight load, this branch clears it only if the UI is still connected. If the UI disconnects first, the stale pending request can replay after reconnect during an unrelated load, causing avoidable heavy status work and unexpected state replacement.
  - confidence: 0.87
- **[P2] Gate missing-gateway guidance on failed probes:** `src/wizard/setup.finalize.ts:295-305`
  - body: This else branch also runs when the gateway probe succeeds, so successful no-daemon setup runs can show "Gateway not detected yet" even though the Gateway is reachable. Only show this note for the failed-probe/no-daemon case.
  - confidence: 0.9
- **[P2] Reuse configured status in summary snapshots:** `src/gateway/server-methods/channels.ts:347-354`
  - body: The summary-only path computes `configured` once, but this call can invoke `plugin.config.isConfigured` again through `buildChannelAccountSnapshot` when the account lacks an inline configured field. That breaks the new lightweight status path's single configuration-check guarantee.
  - confidence: 0.8
