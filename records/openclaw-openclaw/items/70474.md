Full review comments:

- **[P2] Clear pending refreshes when disconnecting:** `ui/src/ui/controllers/channels.ts:171-180`
  - body: When a stronger request is queued during an in-flight load, this branch only clears `privateState.pending` if the UI is still connected. If the UI disconnects first, that stale stronger request can replay during a later unrelated load, causing avoidable heavy status work and unexpected state replacement.
  - confidence: 0.87
- **[P2] Gate missing-gateway guidance on failed probes:** `src/wizard/setup.finalize.ts:295-305`
  - body: This `else` branch also runs when `gatewayProbe.ok` is true and `installDaemon` is false, so successful no-daemon setup can show "Gateway not detected yet" even though the Gateway is reachable. Only show this guidance for the failed-probe/no-daemon path.
  - confidence: 0.9
- **[P2] Reuse configured state in summary snapshots:** `src/gateway/server-methods/channels.ts:347-354`
  - body: The summary-only path computes `configured` once before probe/audit, but `buildChannelAccountSnapshot` can call `plugin.config.isConfigured` again for accounts without an inline `configured` field. Pass the computed configured value through the snapshot path so the lightweight request keeps its single-check guarantee.
  - confidence: 0.8
