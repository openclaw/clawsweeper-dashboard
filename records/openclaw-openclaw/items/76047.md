## Summary

Keep open: current main still does not prove the reported gateway saturation is fixed. The shared JSON writer now cleans its temp path, but cron's separate atomic writer still lacks failure cleanup and Control UI still polls `node.list` while connected, while the claimed 2-second `node.list` probe is not visible in the current handler.

## What This Changes

Fix gateway event-loop saturation attributed to orphaned atomic-write temp files and repeated slow `node.list` polling with no paired nodes.

## Best Possible Solution

Profile a live macOS gateway with Control UI connected and no paired nodes, then land the confirmed narrow fixes: cron orphan-temp cleanup and scoped or cached node polling only if `node.list` remains a measured hot path.

## Reproduction Assessment

Unclear. The issue has concrete logs, file counts, version, and environment, but this read-only review did not run a live gateway and current main does not expose the claimed `node.list` probe timeout in source.

## Solution Assessment

Unclear. Cron temp cleanup is a narrow maintainable improvement, but returning early from `node.list` is not grounded in the current handler; profiling should decide whether the UI poller, pairing-file reads, or another gateway stall is the real fix point.
